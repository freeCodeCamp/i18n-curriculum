# --description--

前のレッスンでは`localStorage`の操作について学び、`sessionStorage`についても簡単に紹介しました。

`sessionStorage`は、ウェブアプリケーションが動作しているタブやウィンドウをユーザーが閉じるとすぐにデータがクリアされることを思い出してください。これは、フォームデータのナビゲーション中の保持やチェックアウトプロセス中の一時的な状態情報の保存など、単一セッションの間だけデータを保持する必要がある場合に最適です。

`localStorage`と同様に、`sessionStorage`もキーと値のペアを使ってデータを保存および取得します。`sessionStorage`で使われるメソッドは`localStorage`と同じで、実際の違いはデータが保存される期間だけです。

以下は、さまざまなメソッドを使った例です：

- `sessionStorage.setItem()`：`sessionStorage`にキーと値のペアを保存します。

```js
sessionStorage.setItem('cart', '3 items');
```

- `sessionStorage.getItem()`：`sessionStorage`から指定したキーの値を取得します。

```js
let cart = sessionStorage.getItem('cart');
console.log(cart); // Outputs: '3 items'
```

- `sessionStorage.removeItem()`：キーを使って`sessionStorage`から特定のアイテムを削除します。

```js
sessionStorage.removeItem('cart');
```

- `sessionStorage.clear()`：`sessionStorage`に保存されているすべてのデータをクリアします。

```js
sessionStorage.clear();
```

ブラウザのタブやウィンドウが開いている間だけ持続する`sessionStorage`にデータを保存する例を見てみましょう：

```js
// Store data in sessionStorage
sessionStorage.setItem('currentUser', 'JohnDoe');

// Retrieve the stored data
const user = sessionStorage.getItem('currentUser');
console.log(user); // 'JohnDoe'

// Remove a specific key from sessionStorage
sessionStorage.removeItem('currentUser');

// Clear all sessionStorage data
sessionStorage.clear();
```

この例では、

1. 現在のユーザー名（`JohnDoe`）を`sessionStorage`に保存します。

2. それを取得して表示します。

3. キー`currentUser`に関連付けられたアイテムを削除します。

4. `sessionStorage`のすべてのデータをクリアします。

`localStorage`との主な違いは、ユーザーがタブを閉じるとすぐに保存されたセッションデータがすべて失われることです。

`sessionStorage`は特に以下のようなシナリオで役立ちます：

- 複数ページにわたるフォーム処理中の一時的なフォームエントリーの保存。

- セッションをまたいで保持する必要のない一時的な選択や設定の保存。

- タブを閉じたら記憶する必要のないシングルページアプリケーションの状態の維持。

`sessionStorage`は、ユーザーがページを離れるとセッションデータがクリアされることを保証するため、現在のセッションを超えて情報を保持したくない場合に最適です。
