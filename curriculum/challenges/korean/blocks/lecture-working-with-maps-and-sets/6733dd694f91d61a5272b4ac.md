# --description--

JavaScript에서 `Map`는 객체와 유사하게 키-값 쌍을 저장하는 내장 객체입니다. 하지만 표준 JavaScript 객체와 달리 객체와 함수 같은 모든 유형의 키를 허용한다는 점에서 다릅니다.

`WeakMap`는 `Map`와 유사한 키-값 쌍의 모음이지만, 키에 대해 약한 참조를 사용합니다. 키는 반드시 객체여야 하며, 값은 어떤 유형이든 될 수 있습니다.

이전 수업에서 배운 것처럼 `WeakMap`와 `Map`의 관계는 `WeakSet`와 `Set`의 관계와 비슷하다고 생각하세요.

`Map`를 생성하려면 `Map()` 키워드 앞에 `new` 생성자를 사용하세요:

```js
const myFirstMap = new Map();
```

Map을 값으로 초기화할 수도 있습니다:

```js
const myTreesMap = new Map([
 [{ type: 'deciduous' }, 'Maple tree'],
 [['forest', 'grove'], 'Pine tree'],
 [42, 'Oak tree'],
 [true, 'Birch tree'],
 [function() { return 'I am a function key'; }, 'Willow tree'],
]);
```

만약 `Map`를 값으로 초기화하지 않았다면, `set()` 메서드를 사용해 값을 추가할 수 있습니다:

```js
const myTreesMap = new Map();
myTreesMap.set({ type: 'deciduous' }, 'Maple tree');
myTreesMap.set([1, 2], 'Pine tree');
myTreesMap.set(42, 'Oak tree');
myTreesMap.set(true, 'Birch tree');
myTreesMap.set(function() { return "I'm a function key"; }, 'Willow tree');

console.log(myTreesMap);
```

콘솔에서 `Map`가 어떻게 보이는지 예를 들어보겠습니다:

```js
/*
Map(5) {{…} => 'Maple tree', Array(2) => 'Pine tree', 42 => 'Oak tree', true => 'Birch tree', ƒ => 'Willow tree'}
  [[Entries]]
    0:{Object => "Maple tree"}
      key: {type: 'deciduous'}
      value: "Maple tree"
    1:{Array(2) => "Pine tree"}
      key: (2)
      value: "Pine tree"
    2:{42 => "Oak tree"}
      key: 42
      value: "Oak tree"
    3:{true => "Birch tree"}
      key: true
      value: "Birch tree"
    4:{function () { return "I'm a function key"; } => "Willow tree"}

      value: "Willow tree"

```

`Map`를 다룰 때 사용할 수 있는 다른 메서드는 다음과 같습니다:

- 지정한 `get(key)`와 연관된 값을 가져오는 `key`.
- `has(key)`에 `key`가 존재하는지 확인하는 `Map`.
- `delete(key)`에서 키-값 쌍을 제거하는 `Map`.
- 모든 키-값 쌍을 제거하는 `clear()`.
- `entries()`의 항목을 확인하는 `Map` (`MapIterator`로 항목을 반환).
- `forEach()`의 항목을 반복하는 `Map`.
- `size`에 있는 키-값 쌍의 수를 나타내는 `Map`.
    

`WeakMap()` 생성자를 사용해 `WeakMap`를 생성할 수도 있습니다:

```js
const myFirstWeakMap = new WeakMap();
```

`set()`, `get()`, `has()`, 그리고 `delete()` 메서드는 `WeakMap`에서도 모두 사용할 수 있습니다. 예를 들어 `Map` 메서드로 `set()`에 항목을 할당하는 방법은 다음과 같습니다:

```js
const myTreeWeakMap = new WeakMap();

myTreeWeakMap.set({ id: 1 }, 'Maple tree');
myTreeWeakMap.set({ id: 2 }, 'Pine tree');
myTreeWeakMap.set({ id: 3 }, 'Oak tree');
myTreeWeakMap.set({ id: 4 }, 'Birch tree');
myTreeWeakMap.set({ id: 5 }, 'Willow tree');

console.log(myTreeWeakMap);
```

콘솔에서 `WeakMap`가 어떻게 보이는지 예를 들어보겠습니다:

```js
/*
WeakMap {{…} => 'Willow tree', {…} => 'Maple tree', {…} => 'Pine tree', {…} => 'Oak tree'}
  [[Entries]]
    No properties
  [[Prototype]]: WeakMap
*/
```

`Map`와 `WeakMap`의 차이점은 다음과 같습니다:

| **기능** | **Map** | **WeakMap** |
| --- | --- | --- |
| **키 유형** | 키는 문자열, 숫자, 객체, 함수 등 모든 데이터 유형이 될 수 있습니다. | 키는 반드시 객체여야 합니다. |
| **사용 사례** | 모든 유형의 키와 데이터를 연관시켜야 할 때 `Map`를 사용하세요. | 객체와만 데이터를 연관시켜야 할 때 `WeakMap`를 사용하세요. |
| **반복** | `Map`, `forEach()`, `keys()`, 또는 `values()`를 사용해 `entries()`를 반복할 수 있습니다. | `WeakMap`는 반복할 수 없습니다. |
| **크기 속성** | `Map`는 키-값 쌍의 수를 얻기 위한 `size` 속성이 있습니다. | `WeakMap`는 `size` 속성이 없습니다. |

위 표에서 키 유형, 사용 사례, 반복, 크기 속성을 포함한 차이점을 요약해 보여줍니다. 이 표 내용을 잠시 읽으며 차이점을 더 자세히 익히세요.
