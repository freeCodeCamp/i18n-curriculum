# --description--

Wenn Sie Abhängigkeiten mit npm installieren, insbesondere in einem bestehenden Projekt, schaut npm in Ihre `package.json`-Datei, um zu sehen, welche Pakete Ihr Projekt benötigt.

Jede Abhängigkeit in `package.json` enthält eine Versionsnummer wie diese:

```bash
"lodash": "^4.17.21"
```

Wie Sie zuvor gelernt haben, erlaubt das **Caret-Zeichen (^)** npm, neuere Minor- und Patch-Versionen wie `4.18.0` oder `4.17.22` zu installieren. Diese Flexibilität ist hilfreich, kann aber auch zu Problemen führen.

Wenn zwei Entwickler `npm install` zu unterschiedlichen Zeiten ausführen, könnte npm leicht unterschiedliche Versionen desselben Pakets installieren. Selbst kleine Unterschiede können Bugs oder unerwartetes Verhalten verursachen.

Die `package-lock.json`-Datei löst dieses Problem. Sie wird automatisch erstellt, wenn Sie `npm install` zum ersten Mal ausführen, und sie sperrt die genauen Versionen jeder installierten Abhängigkeit – einschließlich aller Unterabhängigkeiten, also Abhängigkeiten von Abhängigkeiten.

Das garantiert, dass alle, die am Projekt arbeiten, jedes Mal dieselben Paketversionen erhalten.

Hier ist eine einfache `package.json`-Datei mit `lodash` als Abhängigkeit:

```bash
{
  "name": "package-lock-json",
  "version": "1.0.0",
  "description": "",
  "main": "index.js",
  "scripts": {
    "test": "echo \"Error: no test specified\" && exit 1"
  },
  "keywords": [],
  "author": "",
  "license": "ISC",
  "type": "commonjs",
  "dependencies": {
    "lodash": "^4.17.21"
  }
}
```

Und hier ist der passende Abschnitt aus der `package-lock.json`:

```bash
"node_modules/lodash": {
  "version": "4.17.21",
  "resolved": "https://registry.npmjs.org/lodash/-/lodash-4.17.21.tgz",
  "integrity": "sha512-v2kDEe57..."
}
```

Beachten Sie den Unterschied:

* In `package.json` ist `lodash` als `^4.17.21` aufgeführt – eine flexible Regel.
    
* In `package-lock.json` ist `lodash` auf Version `4.17.21` festgelegt.
    

Egal, wann jemand `npm install` ausführt, er erhält genau diese Version.

So bekommt jeder, der `npm install` ausführt, immer genau dieselbe Version von Lodash.

Beachten Sie, dass dies auch für Unterabhängigkeiten gilt, also Abhängigkeiten von allen Abhängigkeiten, die Sie für Ihr Projekt verwenden.

Beachten Sie auch, dass während die `dependencies` in `node_modules/express` Caret-Zeichen für Bereiche verwenden, die Einträge für jede Unterabhängigkeit wie `node_modules/etags` auf eine bestimmte Version festgelegt sind.

Kurz gesagt, die `package-lock.json`-Datei sorgt dafür, dass alle auf dem gleichen Stand sind und gewährleistet Konsistenz auf den Rechnern der Entwickler, CI-Servern, bei Deployments und in anderen Umgebungen.

Es ist wichtig, Ihre `package-lock.json`-Datei immer mit Git zu committen, damit Ihr Team dieselbe Konfiguration teilt. Bearbeiten Sie die `package-lock.json`-Datei auch nicht manuell – npm aktualisiert sie automatisch, wann immer sich Abhängigkeiten ändern.
