# --interactive--

تُتيح لك الخاصية `grid-template-areas` تصميم تخطيط شبكي بصري باستخدام مسميات مسماة.

ثم تعيّن هذه المسميات لعناصر شبكية محددة باستخدام الخاصية grid-area. بعبارة أخرى، تُسمى هذه المسميات أيضًا "أسماء مساحة الشبكة".

إليك الصيغة النحوية الأساسية للخاصية `grid-template-areas`:

```css
grid-template-areas:
 'header header header'
 'left-sidebar main right-sidebar'
 'footer footer footer';
```

إليك النقاط التي يجب أن تتذكرها من الصيغة النحوية الأساسية:

- القيم مثل `header` و `main` هي أسماء مساحات الشبكة.

- كل قيمة مفصولة بمسافة داخل سلسلة تمثل عمودًا.

- كل سلسلة تمثل صفًا في الشبكة.

لذا، في الصيغة، لدينا حاوية شبكة بحجم 3 في 3.

بعد تعريف القالب، عليك استخدام الخاصية `grid-area` والمسميات المسماة (أو المساحات) كقيم لإخبار CSS بأن عنصرًا معينًا ينتمي إلى تلك المساحة في الشبكة.

تربط الخاصية `grid-area` العنصر الشبكي بالمنطقة المسماة التي تعرفها في الخاصية `grid-template-areas`.

طريقة شائعة لعرض قدرات الخاصية `grid-template-areas` هي بإنشاء تخطيط Holy Grail الكلاسيكي.

إذا لم تكن تعرفه، فإن تخطيط Holy Grail هو نمط تصميم ويب يحتوي على قسم علوي، تذييل، شريطان جانبيان، ومنطقة محتوى رئيسية. يضمن أن يأخذ المحتوى الرئيسي الأولوية، بينما تتكيف الأشرطة الجانبية والأقسام الأخرى بشكل متجاوب داخل التخطيط.

توجد حلول كثيرة لتنفيذ تخطيط Holy Grail، لكن استخدام `grid-template-areas` والخاصية `grid-area` هو أبسط طريقة لإنشائه.

إليك مثالًا لتخطيط Holy Grail:

:::محرر_تفاعلي

```html
<link rel="stylesheet" href="styles.css">
<div class="grid-container">
  <div class="header">
    <h2>Header</h2>
  </div>
  <div class="sidebar-left">
    <h2>Left Sidebar</h2>
  </div>
  <div class="main"><h2>Main Content</h2></div>
  <div class="sidebar-right">
    <h2>Right Sidebar</h2>
  </div>
  <div class="footer">
    <h2>Footer</h2>
  </div>
</div>
```

```css
.grid-container {
  display: grid;
  grid-template-areas:
    'header header header'
    'sidebar-left main sidebar-right'
    'footer footer footer';
  gap: 10px;
  background-color: #2196F3;
  padding: 10px;
}

.header {
  grid-area: header;
  background-color: rgba(255, 255, 255, 0.8);
  padding: 20px;
  text-align: center;
}

.sidebar-left {
  grid-area: sidebar-left;
  background-color: rgba(255, 255, 255, 0.8);
  padding: 20px;
  text-align: center;
}

.main {
  grid-area: main;
  background-color: rgba(255, 255, 255, 0.8);
  padding: 20px;
  text-align: center;
}

.sidebar-right {
  grid-area: sidebar-right;
  background-color: rgba(255, 255, 255, 0.8);
  padding: 20px;
  text-align: center;
}

.footer {
  grid-area: footer;
  background-color: rgba(255, 255, 255, 0.8);
  padding: 20px;
  text-align: center;
}
```

:::

يرجى ملاحظة أن الخاصيتين `grid-template-areas` و `grid-area` يمكن استخدامهما بشكل مستقل عن بعضهما البعض.

تُستخدم الخاصية `grid-template-areas` بشكل خاص لتعريف تخطيط بصري من خلال رسم مساحات شبكية مسماة داخل حاوية الشبكة.

من ناحية أخرى، تُستخدم الخاصية `grid-area` لتحديد مواضع العناصر الشبكية الفردية، إما بتحديد مواقع الصف والعمود أو بالإشارة إلى المساحات المسماة التي عُرفت باستخدام الخاصية `grid-template-areas`.
