# --description--

توفر واجهة برمجة تطبيقات تحديد الموقع الجغرافي طريقة للمواقع الإلكترونية لطلب موقع المستخدم. من المهم ملاحظة أنه لأسباب تتعلق بالخصوصية، يجب أن يمنح المستخدم الإذن قبل أن يمكن الوصول إلى موقعه عبر الموقع الإلكتروني.

الطريقة الرئيسية التي سنركز عليها اليوم هي `getCurrentPosition`. تُستخدم هذه الطريقة لجمع الموقع الجغرافي للجهاز. إليك مثالًا على كيفية استخدام `getCurrentPosition`:

```js
navigator.geolocation.getCurrentPosition(
  (position) => {
    console.log("Latitude: " + position.coords.latitude);
    console.log("Longitude: " + position.coords.longitude);
  },
  (error) => {
    console.log("Error: " + error.message);
  }
);
```

في هذا الكود، نستدعي `getCurrentPosition` ونمرر لها دالة ستُستدعى عند الحصول على الموقع بنجاح. يحتوي كائن الموقع هذا على معلومات متعددة، لكننا نركز فقط على `latitude` و`longitude`.

إذا حدثت مشكلة في الحصول على الموقع، فسيتم تسجيل الخطأ في وحدة التحكم.

تستخدم طريقة `getCurrentPosition` نظام تحديد المواقع العالمي (GPS)، أو شبكات Wi-Fi، أو تحديد الموقع عبر عنوان IP، حسب الجهاز وإعداداته. بمجرد العثور على الموقع، تُستدعى دالة الاستدعاء الراجع الناجحة مع كائن الموقع.

يحتوي كائن الموقع على خصائص متعددة، وأكثرها استخدامًا هي `latitude` و`longitude`، لكنه قد يشمل أيضًا `altitude` و`accuracy` و`speed` و`heading`، وغيرها.

اعتبار مهم عند استخدام تحديد الموقع هو خصوصية المستخدم. تأكد من شرح سبب حاجتك إلى بيانات موقعهم وكيف ستستخدمها.
