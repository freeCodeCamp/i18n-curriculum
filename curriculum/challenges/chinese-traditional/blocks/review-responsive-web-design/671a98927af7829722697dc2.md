## 響應式網頁設計

- **定義**：響應式設計的核心原則是適應性－網站能根據所瀏覽的裝置的螢幕大小和功能調整其版面配置和內容的能力。
- **流體格線**：這些使用相對單位如百分比，而非固定單位如像素，允許內容根據螢幕大小調整大小和重新排列。
- **彈性影像**：這些影像設定為在其包含元素內調整大小，確保它們在較小的螢幕上不會超出其容器的上限溢位。

## 媒體查詢

- **定義**：這允許開發者根據裝置的特性，主要是視窗寬度，套用不同的樣式。

```css
@media screen and (min-width: 768px) {
  /* Styles for screens at least 768px wide */
}
```

- **`all` 媒體型別**：這適用於所有裝置。如果未指定媒體型別，則為預設值。
- **`print` 媒體型別**：這是針對分頁資料和在列印預覽模式下於螢幕上檢視的文件。
- **`screen` 媒體型別**：這主要是針對螢幕。
- **`aspect-ratio`**：這描述了視窗寬度與高度之間的比例。

```css
@media screen and (aspect-ratio: 16/9) {
  /* Styles for screens with a 16:9 aspect ratio */
}
```

- **`orientation`**：這用來顯示裝置是處於橫向還是直向的 `orientation`。

```css
@media screen and (orientation: landscape) {
  /* Styles for landscape orientation */
}
```

- **`resolution`**：這用來描述輸出裝置的解析度，單位為每英吋點數（dpi）或每公分點數（dpcm）。

```css
@media screen and (min-resolution: 300dpi) {
  /* Styles for high-resolution screens */
}
```

- **`hover`**：這用於測試主要輸入機制是否能暫留在元素上。

```css
@media (hover: hover) {
  /* Styles for devices that support hover */
}
```

- **`prefers-color-scheme`**：這用來偵測使用者是否要求淺色或深色主題。
- **媒體查詢和邏輯運算子**：`and` 運算子用於結合多個媒體特性，而 `not` 和 `only` 可用於否定或隔離媒體查詢。

```css
@media screen and (min-width: 768px) and (orientation: landscape) {
  /* Styles for landscape screens at least 768px wide */
}
```

## 常見媒體中斷點

- **定義**：媒體中斷點是在網站設計中，版面配置和內容會調整以適應不同螢幕尺寸的特定點。你可以使用一些通用的中斷點來標的手機、平板和桌面螢幕。但嘗試追蹤每一種不同裝置的所有可能螢幕尺寸並不明智。

```css
/* Styles for screens wider than 768px */
@media screen and (min-width: 768px) {
  body {
    font-size: 1.125rem;
  }
}
```

- **小型裝置（智慧型手機）**：最多 640px
- **中型裝置（平板）**：641px 到 1024px
- **大型裝置（桌上型電腦）**：1025px 及以上

## 行動優先方法

- **定義**：行動優先方法是一種設計理念和響應式網頁設計中的開發策略，優先為行動裝置建立網站，再為較大螢幕設計。

```css
/* Base styles for mobile */
.container {
  width: 100%;
  padding: 10px;
}

/* Styles for larger screens */
@media screen and (min-width: 768px) {
  .container {
    width: 750px;
    margin: 0 auto;
    padding: 20px;
  }
}

@media screen and (min-width: 1024px) {
  .container {
    width: 960px;
  }
}
```
