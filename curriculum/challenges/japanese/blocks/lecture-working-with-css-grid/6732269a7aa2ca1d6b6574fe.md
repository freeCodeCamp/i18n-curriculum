# --interactive--

暗黙のグリッドとは、グリッドレイアウトでアイテムを配置するときにブラウザーが自動的に作成する行と列のことです。これは `grid-template-rows` や `grid-template-columns` を使って明示的に定義されていない部分を指します。

ブラウザーが暗黙的に作成する列と行を制御するプロパティは `grid-auto-columns` と `grid-auto-rows` です。

暗黙のグリッドは、明示的に定義された行や列の外にアイテムを配置したときにブラウザーが自動的に生成する追加の行や列も指します。

例えば、次のようにグリッドレイアウトで明示的に2つの列だけを定義したとします。

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<div class="grid-container">
  <div class="grid-item">Item 1</div>
  <div class="grid-item">Item 2</div>
  <div class="grid-item">Item 3</div>
  <div class="grid-item">Item 4</div>
  <div class="grid-item">Item 5</div>
  <div class="grid-item">Item 6</div>
</div>
```

```css
.grid-container {
  display: grid;
  grid-template-columns: 100px 100px; /* Only 2 explicit columns */
}

.grid-item {
  background-color: burlywood;
  border: 1px solid orangered;
  padding: 0.5rem;
  margin: 0.5rem;
}
```

:::

2つのアイテムが最初の行の2つの明示的な列に配置されました。`Item 1` は1列目に、`Item 2` は2列目にあります。

次のアイテムは新しい行を開始し、`Item 3` は2行目の1列目に、`Item 4` は2行目の2列目に配置されます。このように続きます。

すでに見たように、明示的なグリッドとは意図的に設定したグリッドのエリアのことです。つまり、`grid-template-rows` と `grid-template-columns` のプロパティを使ってグリッドレイアウトの行と列を明示的に定義した部分です。

以下は暗黙のグリッドと明示的なグリッドの違いをまとめたテーブルです。

| **機能**              | **明示的なグリッド**                                                                | **暗黙のグリッド**                                                                        |
|----------------------|------------------------------------------------------------------------------------|--------------------------------------------------------------------------------------------|
| **サイズ制御**       | ` grid-template-rows ` と ` grid-template-columns ` を使って完全にカスタマイズ可能。                         | ` grid-auto-rows ` と ` grid-auto-columns ` で制御されるか、` auto ` がデフォルトになる。                 |
| **デフォルトの動作** | 明示的に定義しない限り変わらない。                                               | 明示的なグリッドの外に配置されたアイテムに合わせて自動的に調整される。                   |
| **複雑さ**           | レイアウト構造の計画がより必要。                                                   | 構造化されていない、または可変のコンテンツに対して実装が簡単。                           |
| **柔軟性**           | 定義された行と列で堅い構造。                                                       | 動的に配置されたコンテンツに柔軟に対応。                                                 |
| **パフォーマンス**   | レイアウトが事前定義されているため、よりパフォーマンスが良い可能性がある。         | 暗黙のグリッド生成に追加のブラウザー計算が必要になる場合がある。                         |
| **使用例**           | グリッド構造が予測可能で事前に定義されている場合に有用。                           | コンテンツが不明または頻繁に変わる動的なレイアウトに有用。                             |
