# --description--

Tujifunze jinsi ya kufananisha au kubadilisha matukio yote ya muundo katika mfuatano wa herufi.

Umejifunza awali kuhusu njia za `replace()` na `match()`, pamoja na kiimarishaji cha `g` cha kimataifa. Sasa unaweza kuunganisha maarifa hayo kushughulikia mifumo yote katika mfuatano wa herufi. Tukumbuke msimbo wetu wa awali wa kufananisha:

```js
const regex = /freeCodeCamp/;
const match = "freeCodeCamp".match(regex);
console.log(match);
```

Na kitu chetu cha matokeo cha kufananisha:

```js
// [
//   'freeCodeCamp',
//   index: 0,
//   input: 'freeCodeCamp',
//   groups: undefined
// ]
```

Lakini vipi ikiwa tuna mfuatano wa herufi wenye matukio mengi ya `freecodecamp` ya kufananisha? Tuchunguze jinsi `match()` inavyotenda na hiyo. Tutaongeza mfano wetu wa zamani wa `replace()` pia, kwa kulinganisha:

```js
const regex = /freecodecamp/;
const str = "freecodecamp is the best we love freecodecamp";
const matched = str.match(regex);
const replaced = str.replace(regex, "freeCodeCamp");
console.log(matched);
console.log(replaced);
```

Na matokeo ni haya:

```js
// [
//   'freecodecamp',
//   index: 0,
//   input: 'freecodecamp is the best we love freecodecamp',
//   groups: undefined
// ]
// freeCodeCamp is the best we love freecodecamp
```

Aibu! `match()` ilirudisha tu mechi ya kwanza, na `replace()` ilibadilisha tu mechi ya kwanza. Hii ni kwa sababu, kwa chaguo-msingi, `match()` na `replace()` hufanya kazi tu dhidi ya tukio la kwanza la muundo.

Kwa bahati nzuri, unaweza kuepuka hili kwa kutumia kiimarishaji cha kimataifa kwenye usemi wako wa kawaida. Tuweke hiyo kwenye yetu:

```js
const regex = /freecodecamp/g;
const str = "freecodecamp is the best we love freecodecamp";
const matched = str.match(regex);
const replaced = str.replace(regex, "freeCodeCamp");
console.log(matched);
console.log(replaced);
```

Na thibitisha matokeo:

```js
// [ 'freecodecamp', 'freecodecamp' ]
// freeCodeCamp is the best we love freeCodeCamp
```

Ilienda sawa! Wito wetu wa kubadilisha ulibadilisha mfuatano wote wa herufi ndogo za `freecodecamp`, na njia yetu ya `match()` ilifananishwa zote mbili.

Kinachovutia hapa ni kwamba unapotumia kiimarishaji cha kimataifa na `match()`, hupoteza taarifa za ziada kuhusu makundi ya kunasa na viashiria vya mfuatano wa herufi ambavyo vingepatikana kwenye safu ya mechi.

Kwa bahati nzuri, sasisho la ECMAScript la mwaka 2019 lilileta njia mbili mpya: `matchAll()` na `replaceAll()`. Kama wenzao wa pekee, njia hizi zinakubali mfuatano wa herufi au usemi wa kawaida, na `replaceAll()` pia inakubali hoja ya pili kama mfuatano wa herufi wa kubadilisha.

Lakini tofauti na njia zilizotangulia, `replaceAll()` na `matchAll()` zitatupa kosa ikiwa utawapa usemi wa kawaida usio na kiimarishaji cha kimataifa. Tuweke msimbo wetu kutumia njia hizi mpya:

```js
const pattern = "freecodecamp";
const str = "freecodecamp is the best we love freecodecamp";
const matched = str.matchAll(pattern);
const replaced = str.replaceAll(pattern, "freeCodeCamp");
console.log(matched);
console.log(replaced);
```

Na matokeo yetu:

```js
// {}
// freeCodeCamp is the best we love freeCodeCamp
```

Habari njema! `replaceAll()` yetu ilifanya kazi kama tulivyotaka: ilibadilisha matukio yote ya `freecodecamp` yenye herufi ndogo kuwa toleo lililokuwa na camelcase ipasavyo.

Lakini ni nini hicho kitu tupu? `matchAll()` inarudisha aina maalum ya kitu kinachoitwa `Iterator`, ambacho konsoli ya freeCodeCamp haijajiandaa kukishughulikia.

Tukitazama kwenye konsoli ya kivinjari chetu, `Iterator` ina njia ya `next()`, ambayo tunaweza kuitisha kupata thamani inayofuata:

```js
// RegExpStringIterator {  }
//  <prototype>: RegExp String Iterator {
//    next: ƒ next(),
//    Symbol(Symbol.toStringTag): "RegExp String Iterator"
//    <prototype>: Object { ... }
//  }
```

Tuitishe `matched.next()`, na andika matokeo:

```js
// {
//   "done": false,
//   "value": [
//     0: "freecodecamp"
//     groups: undefined
//     index: 0
//     input: "freecodecamp is the best we love freecodecamp"
//   ]
// }
```

Hapo kuna safu yetu ya mechi! `next()` hutupa kitu chenye thamani mbili: `done`, ambayo ni `false` wakati kuna vipengele zaidi vinavyopatikana kwenye mzunguko, na `value` ambayo ni thamani tuliyorudia hivi sasa. Hivyo, tukiiita tena mara moja:

```js
const regex = /freecodecamp/g;
const str = "freecodecamp is the best we love freecodecamp";
const matched = str.matchAll(regex);
const replaced = str.replaceAll(regex, "freeCodeCamp");
console.log(matched);
console.log(replaced);
console.log(matched.next());
console.log(matched.next());
```

```js
// {
//   "done": false,
//   "value": Array [ "freecodecamp"]
// }
//
// {
//   "done": false,
//   "value": Array [ "freecodecamp"]
// }
```

Subiri, kwa nini inasema `done` bado ni `false`? Inapaswa kuwa na mechi mbili tu kwenye safu, sivyo? Tuitishe mara ya tatu tuone tunapata nini:

```js
// {
//   "done": false,
//   "value": Array [ "freecodecamp"]
// }
//
// {
//   "done": false,
//   "value": Array [ "freecodecamp"]
// }
//
// {
//   "done": true,
//   "value": undefined
// }
```

`done` hatimaye ni `true`, lakini kwa nini thamani hiyo ni `undefined`? Kweli, mzunguko wa `matchAll()` ni mzunguko wa mvivu. Haufuatilii mechi zako zote mara moja. Hupata mechi tu unapoamuru kwa kuitisha `next()`.

Mara tu inapopata mechi, si `done`. Mara inaposhindwa kupata mechi na kurudisha `undefined`, ni `done`. Hii inaweza kuonekana kuwa isiyofaa, lakini inaweza kusaidia sana wakati usemi wako wa kawaida unahitaji nguvu nyingi za kompyuta.

Ikiwa mfano wako ni mdogo zaidi, kama wetu, unaweza kupuuza kipengele hicho na kutoa mechi zote mara moja kwa kuibadilisha kuwa safu ya data. Hii inafanikishwa kwa kuitisha `Array.from()` na kupitisha mzunguko wako kama hoja.

Hebu sahihisha msimbo wetu kutumia hiyo: tutaendelea na kusafisha wito zetu za `replaceAll` kwa sababu tunajua zinafanya kazi:

```js
const regex = /freecodecamp/g;
const str = "freecodecamp is the best we love freecodecamp";
const matched = str.matchAll(regex);
console.log(Array.from(matched));
```

Na hatimaye tunapata safu yetu ya mechi:

```js
// [
//   'freecodecamp',
//   index: 0,
//   input: 'freecodecamp is the best we love freecodecamp',
//   groups: undefined
// ]
//
// [
//   'freecodecamp',
//   index: 33,
//   input: 'freecodecamp is the best we love freecodecamp',
//   groups: undefined
// ]
```

Njia hizi zenye nguvu zinaweza kusaidia kudhibiti na kutoa data kutoka kwa mfuatano wa herufi bila kuathiri utendaji au usomaji.
