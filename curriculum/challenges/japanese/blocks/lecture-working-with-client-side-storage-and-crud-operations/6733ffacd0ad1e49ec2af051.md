# --description--

前のレッスンで学んだように、クッキーはウェブサイトがユーザーのデバイスに保存できるシンプルなデータです。

しかし、実際にはクッキーにもっと複雑なデータ構造を保存することもできます。よく使われる方法の一つは、JSONを使ってオブジェクトや配列を保存することです。例を示します。

```js
const userData = {
  name: "John Doe",
  age: 30,
  role: "admin"
};

document.cookie = "userInfo=" + JSON.stringify(userData) + "; path=/";
```

この例では、ユーザーデータのオブジェクトを作成し、それをJSON文字列に変換してからクッキーに保存しています。このデータを取得したいときは、`JSON.parse()`を使ってJSON文字列をオブジェクトに解析できます。

さて、トピックの「HTTPヘッダー」について気になるかもしれません。通常、クッキーはサーバーがHTTPヘッダーを使って設定します。

例えば、サーバーは次のようなヘッダーを送信することがあります。

```http
Set-Cookie: username=John Doe; expires=Thu, 31 Dec 2024 6:00:00 IST; path=/
```

このヘッダーはブラウザにクッキーを設定するよう指示しています。

また、JavaScriptを使ってブラウザ内で直接クッキーを設定することもできます。これは、すぐにサーバーに送信する必要のないデータを保存するのに便利です。

クッキーには約4KBのサイズ制限があり、あまり多くのデータを保存するとウェブアプリの動作が遅くなる場合があります。

大量のデータをクッキーに保存すると、クッキーがすべてのHTTPリクエストに送信されるため、ネットワークのトラフィックが増加します。
