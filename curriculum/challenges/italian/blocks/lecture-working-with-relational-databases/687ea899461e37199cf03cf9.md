# --description--

Per definire una tabella in SQL, devi specificare il tipo di dato di ogni colonna.

Ricorda che questa è la sintassi base per creare una tabella in SQL. Ogni colonna ha il suo tipo di dato.

```sql
CREATE TABLE table_name(
  column1 data_type column_constraint,
  column2 data_type column_constraint,
  column3 data_type column_constraint,
  ... etc
);
```

Come sviluppatore, il tuo obiettivo è scegliere questi tipi di dati nel modo più preciso possibile.

Ci sono sei categorie popolari di tipi di dati in SQL:

- Numerici, come `INTEGER`, `FLOAT`, `SERIAL` e `DECIMAL`.
- Data e ora, come `TIMESTAMP`, `DATE` e `TIME`.
- Carattere e stringa. Questi includono `CHAR`, `VARCHAR` e `TEXT`.
- Unicode, inclusi `NTEXT` e `NVARCHAR`. Questi sono usati per assicurare che il testo venga archiviato e recuperato correttamente, indipendentemente dall'origine dei caratteri.
- Binari, usati per archiviare dati non testuali, come immagini, audio e file video.
- E altri tipi di dati vari, come `XML` e `TABLE`.

In questa lezione, copriremo alcuni dei più usati con PostgreSQL. I tipi di dati variano tra i sistemi di gestione database, ma sono generalmente abbastanza simili. Quindi dovrai consultare la lista completa nella documentazione.

Inizieremo con i valori numerici.

Per prima cosa, abbiamo il tipo di dato `INTEGER`:

```sql
units_sold INTEGER
```

In questo esempio, assegniamo questo tipo di dato a una colonna `units_sold`.

I tipi `INTEGER` occupano 4 byte nel database e possono avere valori da -2.147.483.648 a 2.147.483.647. La documentazione ufficiale di PostgreSQL lo descrive come la "scelta tipica per numeri interi".

Abbiamo anche `SMALLINT` e `BIGINT`, che sono praticamente uguali a `INTEGER` ma hanno rispettivamente un intervallo di numeri più piccolo e più grande, a causa della dimensione che occupano nel database.

Una funzionalità utile disponibile in PostgreSQL è creare una colonna usando la parola chiave `SERIAL`. Qui, una colonna `id` viene creata usando `SERIAL`:

```sql
id SERIAL
```

Anche se non è un vero tipo di dato, è molto utile per creare colonne con identificatori unici perché la colonna avrà un tipo `INTEGER`, non permetterà valori `NULL` e si incrementerà automaticamente quando si aggiungono righe.

Per esempio, la prima riga avrà automaticamente un `id` di `1`, la seconda riga avrà un `id` di `2` e così via, creando un ID unico per ogni record.

In MySQL, l'equivalente di `SERIAL` sarebbe l'attributo `AUTO_INCREMENT`, usato per generare automaticamente numeri interi sequenziali:

```sql
id INT AUTO_INCREMENT
```

Puoi vedere che diversi sistemi di gestione database possono avere modi diversi per ottenere esattamente la stessa funzionalità nel tuo database.

Questi sono i tipi di dati numerici più comunemente usati, ma a volte potresti aver bisogno di rappresentare testo o sequenze di caratteri.

Per questo, hai il tipo di dato `VARCHAR`. Questo tipo di dato è usato per stringhe di lunghezza variabile. Puoi impostare la lunghezza massima dei caratteri tra parentesi tonde:

```sql
name VARCHAR(50)
```

In questo esempio, definiamo una colonna `name`, dove i valori possono essere lunghi fino a 50 caratteri.

Questo tipo di dato imposta una lunghezza massima per le stringhe, ma se hai bisogno di archiviare stringhe di qualsiasi lunghezza, puoi usare invece `TEXT`:

```sql
name TEXT
```

Oltre ai numeri e alle stringhe, è anche comune archiviare date e orari.

Per esempio, se crei una tabella per archiviare eventi, potresti aver bisogno di archiviare la data di ogni evento in un formato specifico. Per questo, hai il tipo di dato `DATE`:

```sql
event_date DATE
```

Per archiviare l'ora, puoi anche usare il tipo di dato `TIME`. Per esempio, potresti archiviare l'ora in cui un evento inizia:

```sql
start_time TIME
```

Il tipo di dato `TIMESTAMP` combina entrambi. Include sia la data che l'ora. Può anche includere il fuso orario se scrivi `TIMESTAMP WITH TIME ZONE`:

```sql
event_timestamp TIMESTAMP
```

```sql
event_timestamp TIMESTAMP WITH TIME ZONE
```

E infine, se hai bisogno di archiviare un valore booleano, `TRUE` o `FALSE`, puoi usare il tipo di dato `BOOLEAN`:

```sql
is_active BOOLEAN
```

Ora conosci alcuni dei tipi di dati più comuni in SQL, ma ce ne sono molti altri. Puoi consultare la documentazione del tuo sistema di gestione database per trovare più informazioni sui tipi di dati che puoi usare.
