# --description--

웹은 방대한 정보와 활동으로 가득한 강력한 공간입니다. 하지만 조심하지 않으면 위험한 장소가 될 수도 있습니다. 수많은 해커와 사이버 범죄자가 웹사이트를 침해하고 데이터를 탈취할 새로운 방법을 찾고 있습니다.

메타나 마이크로소프트 같은 수십억 달러 규모 기업에 영향을 준 몇몇 대규모 데이터 유출 사건을 들어본 적이 있을 겁니다. 웹 애플리케이션 작업을 할 때는 취약점과 약점을 찾아내기 위해 철저한 보안 테스트를 해야 합니다.

보안 감사와 테스트를 수행할 때 알아야 할 몇 가지 원칙은 다음과 같습니다:

- 기밀성: 민감한 정보가 의도된 수신자가 아닌 다른 사람에게 유출되는 것을 방지합니다.

- 무결성: 악의적인 사용자가 사용자 정보를 변경하지 못하도록 막습니다.

- 인증: 사용자의 신원을 확인하여 해당 시스템을 사용할 권한이 있는지 검증합니다.

- 권한 부여: 인증된 사용자가 수행할 수 있는 작업이나 접근할 수 있는 시스템 부분을 결정하는 과정입니다.

- 가용성: 정보와 서비스가 필요할 때 인증된 사용자에게 제공되도록 보장합니다.

- 부인 방지: 송신자와 수신자 모두가 전달 증명과 송신자 신원 확인을 갖도록 하여, 송신자가 정보를 보냈다는 사실을 부인하지 못하게 합니다.

이제 보안 테스트의 이유를 이해했으니, 몇 가지 일반적인 보안 위협을 살펴보겠습니다:

- 크로스 사이트 스크립팅(XSS): 이전 수업에서 배웠습니다. 간단히 말해, 공격자가 악성 스크립트를 웹 페이지에 주입하면 다른 사용자가 그 페이지를 볼 때 해당 스크립트가 피해자의 브라우저에서 실행되어 쿠키나 세션 데이터를 훔치거나 사용자의 동의 없이 악의적인 행동을 할 수 있습니다.

- SQL 인젝션: SQL은 관계형 데이터베이스에서 정보를 관리하고 저장하는 언어입니다. SQL 인젝션은 악의적인 사용자가 데이터베이스에 악성 코드를 주입하는 공격입니다. 앞으로의 모듈에서 SQL, 관계형 데이터베이스, SQL 인젝션에 대해 더 배울 것입니다.

- 서비스 거부(DoS) 공격: 악의적인 사용자가 웹사이트에 대량의 요청이나 트래픽을 보내 서버가 느려지거나 다운되어 사이트를 사용할 수 없게 만드는 공격입니다.

보안 위협 유형은 더 많지만, 웹 애플리케이션을 보호하고 보안을 강화하는 데 도움이 되는 다양한 도구가 있습니다. 보안 테스트 도구는 다음과 같은 넓은 범주로 나뉩니다:

- 정적 애플리케이션 보안 테스트: 애플리케이션의 소스 코드를 평가하여 보안 취약점을 찾는 도구입니다.

- 동적 애플리케이션 보안 테스트(DAST): 애플리케이션의 프런트엔드와 상호작용하여 잠재적 보안 약점을 찾아내는 도구입니다. 정적 테스트와 달리 소스 코드에 접근하지 않습니다.

또 다른 보안 테스트 유형은 침투 테스트(펜테스트)입니다. 이는 애플리케이션에 대한 모의 사이버 공격을 만들어 시스템의 취약점을 찾아내는 작업입니다. 중대형 기업은 정기적인 보안 감사의 일환으로 전담 사이버보안 전문가 팀이 펜테스트를 수행하기도 합니다.

사이버보안 분야는 매우 방대하고 세부적입니다. 하지만 모든 개발자는 웹 애플리케이션에 대한 몇 가지 일반적인 위협과 이를 방어하는 방법을 이해해야 합니다.
