# --description--

讓我們來看看你可以用來插入和檢視資料的基本指令。

首先，要查看資料，你需要將它插入。所以我們將從插入開始。

本課程將涵蓋的範例將基於這張狗狗記錄的表格：

```sql
CREATE TABLE dogs(
  id SERIAL,
  name VARCHAR(100),
  age INTEGER
);
```

該表格將儲存各種狗的名字和年齡。

如果我們假設正在使用 PostgreSQL，我們可以為 `id` 使用 `SERIAL` 資料型別。由於 `id` 是使用 `SERIAL` 創建的，其值將會是 `INTEGER`，從 `1` 開始，並在插入新記錄時自動遞增。因此，當你插入記錄時，不需要為它傳遞值。

讓我們先使用 `INSERT INTO` 述語將記錄插入到我們的 `dogs` 資料表中。有很多方法可以做到這一點。第一種方法是插入單一列，並指定將要插入的值的行。

你寫 `INSERT INTO`，接著是資料表的名稱（在此例中為 `dogs`），然後是小括弧內的行，再來是 `VALUES`，以及其對應值的小括弧內。

這是一個範例，我們正在插入一筆 `name` 為 `'Gino'`、`age` 為 `3` 的狗記錄，請記得 `id` 將會自動指定：

```sql
INSERT INTO dogs (name, age)
VALUES ('Gino', 3);
```

這通常是最安全的選項，因為你明確指定了行，所以值會依序指定到那些行。

請注意，`'Gino'` 需要用單引號括起來，因為它是 `VARCHAR` 型別。

另一種替代方法是不指定行就插入記錄，如下所示：

```sql
INSERT INTO dogs
VALUES ('Gino', 3);
```

這個語法是有效的 SQL，但容易出錯。

由於值是根據行順序指定，PostgreSQL 將嘗試將 `'Gino'` 插入期望整數的 `id` 行。這會導致像是 `invalid input syntax for type integer` 的錯誤。

因此，在插入資料時明確指定行名稱會更安全。

你也可以透過逗號分隔，在同一個 SQL 指令中插入多筆記錄。

這裡，我們正在插入兩筆狗的記錄，一筆是 `'Gino'`，另一筆是 `'Nora'`。

```sql
INSERT INTO dogs (name, age)
VALUES
  ('Gino', 3),
  ('Nora', 2);
```

就像你可以插入記錄一樣，你也可以從資料庫查詢資訊。在資料庫的背景關係中，查詢是對資料的請求。

在 SQL 中，`SELECT` 述語用於從一個或多個資料表查詢資料。你可以自訂查詢以取得你所需的精確資訊。

如果你需要查詢 `dogs` 表格中的所有資訊，只需要寫 `SELECT`，接著是一個星號（`*`），然後是 `FROM`，以及表格名稱 `dogs`：

```sql
SELECT *
FROM dogs;
```

在此指令中，星號是表述「所有行」的萬用字元。

你將取得該資料表中所有記錄的所有資料：

```sql
 id | name | age 
----+------+-----
  1 | Gino |   3
  2 | Nora |   2
```

若要查詢特定行，你可以在指令中 `SELECT` 後面寫出那些行的名稱，並以逗號分隔。在此範例中，我們查詢 `dogs` 資料表的 `name` 和 `age` 行：

```sql
SELECT name, age
FROM dogs;
```

結果將只包含 `name` 和 `age` 行：

```sql
 name | age 
------+-----
 Gino |   3
 Nora |   2
```

有時候，你可能需要根據特定條件查詢資料。 例如，要取得所有小於 3 歲的狗，你可以使用 `WHERE` 關鍵字和小於 (`<`) 比較運算子：

```sql
SELECT *
FROM dogs
WHERE age < 3;
```

結果將只包含年齡小於 3 的狗：

```sql
 id | name | age 
----+------+-----
  2 | Nora |   2
```

如果你只是想找到 `'Gino'` 的 `age`，你可以使用相等（`=`）比較運算子：

```sql
SELECT age
FROM dogs
WHERE name = 'Gino';
```

結果如下：

```sql
 age 
-----
  3
```

這些是在 SQL 中插入和查看資料的常見且簡單的方法，但你可以選擇許多不同的選項，可選方案，其中有些是更進階的。你將在接下來的課程中學習它們。
