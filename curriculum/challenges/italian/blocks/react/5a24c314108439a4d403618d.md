# --description--

Finora hai renderizzato i componenti React sul client. Normalmente, è quello che farai sempre. Tuttavia, ci sono alcuni casi in cui ha senso renderizzare un componente React sul server. Poiché React è una libreria JavaScript per le viste e puoi eseguire JavaScript sul server con Node, questo è possibile. Infatti, React fornisce un metodo `renderToString()` che puoi usare a questo scopo.

Ci sono due motivi principali per cui il rendering sul server può essere usato in un'app reale. Primo, senza farlo, le tue app React consisterebbero in un file HTML relativamente vuoto e un grande bundle di JavaScript quando viene caricato inizialmente nel browser. Questo potrebbe non essere ideale per i motori di ricerca che cercano di indicizzare il contenuto delle tue pagine così le persone possono trovarti. Se renderizzi il markup HTML iniziale sul server e lo invii al client, il caricamento iniziale della pagina contiene tutto il markup della pagina che può essere scansionato dai motori di ricerca. Secondo, questo crea un'esperienza di caricamento iniziale della pagina più veloce perché l'HTML renderizzato è più piccolo del codice JavaScript dell'intera app. React sarà comunque in grado di riconoscere la tua app e gestirla dopo il caricamento iniziale.

# --instructions--

Il metodo `renderToString()` è fornito su `ReactDOMServer`, che è disponibile qui come oggetto globale. Il metodo prende un argomento che è un elemento React. Usalo per renderizzare `App` in una stringa.
