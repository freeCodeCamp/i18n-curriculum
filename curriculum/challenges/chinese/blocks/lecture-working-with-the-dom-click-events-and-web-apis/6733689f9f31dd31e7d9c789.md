# --description--

让我们了解 DOM 节点及其在 DOM 树 中的关系。

正如真实的树有大小分支以层级结构连接，DOM 节点之间也存在直接和间接的关系。我们将用这个例子来说明这些关系：

```html
<!DOCTYPE html>
<html>
  <head>
    <title>DOM Tree Example</title>
  </head>
  <body>
    <h1>Heading 1</h1>
    <p>Paragraph 1</p>
    <ul>
      <li>List item 1</li>
      <li>List item 2</li>
    </ul>
  </body>
</html>
```

让我们从顶层开始。DOM 树的根是 `html` 元素。它是 HTML 文档所有内容的顶层容器。所有其他节点都是该根节点的后代。

然后，在根节点下方，我们查找层次结构中的其他节点。父节点是包含其他元素的元素。子节点是包含在另一个元素内的元素。

在此示例中，`body` 元素是段落元素的父元素，而段落元素是 `body` 元素的子元素。这些元素作为节点表现于 DOM 树中，具有相同的关系。

就像我们有父节点和子节点一样，我们也有兄弟节点。兄弟节点是共享相同父节点的元素。在我们的示例中，两个列表项元素共享相同的无序列表父节点，因此它们是兄弟节点。 

同样，`h1` 和段落元素是兄弟元素，因为它们共享相同的父元素，即 `body` 元素。

我们在层次结构的不同级别之间也有间接关系。后代节点是包含在另一个元素内的元素，可以是直接包含也可以是间接包含。如果一个元素可以通过从其祖先沿着 DOM `tree` 向下遍历到达，则该元素被视为另一个元素的后代。

在我们的示例中，列表项 `li` 元素是 `body` 元素的后代，因为它们间接包含在其中。

祖先节点是 DOM 树层次结构中位于另一个元素之上的元素。在我们的示例中，`body` 元素是列表项 `li` 元素的祖先。

理解这些关系对于使用 JavaScript 操作和导航 DOM 树至关重要。
