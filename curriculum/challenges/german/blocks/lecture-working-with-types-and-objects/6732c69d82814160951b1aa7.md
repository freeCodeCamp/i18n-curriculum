# --interactive--

Der `Number()`-Konstruktor wird verwendet, um ein Number-Objekt zu erstellen. Das Number-Objekt enthält einige hilfreiche Eigenschaften und Methoden wie die `isNaN` und die `toFixed`-Methode. Hier ist ein Beispiel, das den `Number()`-Konstruktor mit dem `new`-Schlüsselwort verwendet:

:::interactive_editor

```js
const myNum = new Number("34");
console.log(typeof myNum); // "object" 
```

:::

In diesem Beispiel übergeben wir einen String-Literal an den `Number()`-Konstruktor, und der Rückgabewert ist vom Typ `object` anstelle eines `string`.

Wenn der `Number()`-Konstruktor als Funktion ohne das `new`-Schlüsselwort aufgerufen wird, ist der Rückgabewert der primitive Zahlentyp. Meistens verwenden Sie den `Number()`-Konstruktor, um andere Datentypen in den Zahlentyp zu konvertieren. Hier ist ein Beispiel, wie ein String in eine Zahl umgewandelt wird:

:::interactive_editor

```js
const myNum = Number("100");
console.log(myNum); // 100

console.log(typeof myNum); // number
```

:::

Das ist hilfreich, wenn Sie Eingaben vom Benutzer erhalten und diese String-Eingabe in eine Zahl umwandeln müssen, um mathematische Berechnungen durchzuführen.

Wenn Sie versuchen, den `Number()`-Konstruktor mit einem leeren String aufzurufen, ist das Ergebnis die Zahl `0`:

:::interactive_editor

```js
const num = Number("");
console.log(num); // 0
```

:::

Das liegt daran, dass JavaScript versucht, den String zu parsen, und da er keine Ziffern enthält, ist das Ergebnis null.

Wenn Sie versuchen, einen String mit zufälligen Zeichen zu übergeben, ist das Ergebnis `NaN` oder „Not a Number“.

:::interactive_editor

```js
const num = Number("random");
console.log(num); // NaN
```

:::

Beim Arbeiten mit Booleans gibt `true` `1` zurück, weil `true` in eins umgewandelt wird, und `false` gibt `0` zurück, weil `false` in null umgewandelt wird.

:::interactive_editor

```js
const boolTrue = Number(true);
const boolFalse = Number(false);

console.log(boolTrue); // 1
console.log(boolFalse); // 0
```

:::

Wenn Sie `null` übergeben, ist das Ergebnis `0`, und wenn Sie `undefined` übergeben, ist das Ergebnis `NaN`.

:::interactive_editor

```js
const undefinedNum = Number(undefined);
const nullNum = Number(null);

console.log(undefinedNum); // NaN
console.log(nullNum); // 0
```

:::

Beim Arbeiten mit Arrays gibt es einige Dinge zu beachten.

Ein leeres Array gibt `0` zurück. Ein Array mit einer einzelnen Zahl gibt diese Zahl zurück. Ein Array mit mehreren Zahlen gibt `NaN` zurück. Und ein Array mit String(s) gibt ebenfalls `NaN` zurück.

:::interactive_editor

```js
const emptyArr = Number([]);
const arrOneNum = Number([7]);
const arrMultiNum = Number([7, 36, 12]);
const arrStr = Number(["str1"]);
const arrMultiStr = Number(["str1", "str2"]);

console.log(emptyArr); // 0
console.log(arrOneNum); // 7
console.log(arrMultiNum); // NaN
console.log(arrStr); // NaN
console.log(arrMultiStr); // NaN
```

:::

Beim Arbeiten mit Objekten ist das Ergebnis immer `NaN`.

:::interactive_editor

```js
const obj1 = Number({});
const obj2 = Number({2: 2});
const obj3 = Number({key: "val"});
const obj4 = Number({key: true});

console.log(obj1); // NaN
console.log(obj2); // NaN
console.log(obj3); // NaN
console.log(obj4); // NaN
```

:::

Zusammenfassend verwenden Sie den `Number()`-Konstruktor meistens für die Typumwandlung und weniger, um eine Zahl oder ein Number-Objekt zu erstellen.
