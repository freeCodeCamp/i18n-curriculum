# --description--

Il middleware a livello di applicazione è un middleware associato a un'istanza di un'app Express. Viene tipicamente usato per gestire richieste che richiedono elaborazioni su più rotte. Può essere applicato globalmente o a rotte specifiche.

Questo middleware si registra usando `app.use()` o metodi di rotta specifici come `app.get()`, `app.post()`, ecc. Viene eseguito per ogni richiesta in arrivo a meno che non sia limitato a rotte specifiche.

```js
const express = require('express')
const app = express()

// Application-level middleware that logs all incoming requests
app.use((req, res, next) => {
  console.log('Request received at:', new Date())
  next(); // Passes control to the next middleware
})

app.get('/', (req, res) => {
  res.send('Hello World!')
})

app.listen(3000, () => {
  console.log('Server running on http://localhost:3000')
})
```

* `app.use()`: Questo è un middleware a livello di applicazione. Registra ogni richiesta fatta al server, indipendentemente dalla rotta.
* Il middleware viene eseguito per ogni richiesta in arrivo, fornendo al server dettagli come il timestamp corrente.

Il middleware a livello di applicazione è utile per attività che devono essere applicate su molte o tutte le rotte dell'app. Questo può includere logging, autenticazione e gestione degli errori.
