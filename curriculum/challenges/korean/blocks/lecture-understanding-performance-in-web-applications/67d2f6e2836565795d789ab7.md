# --description--

URL을 입력하고 엔터를 눌렀을 때 무슨 일이 일어나는지 궁금했던 적 있나요? 브라우저가 코드를 우리가 보는 인터랙티브한 페이지로 어떻게 변환하는지 알고 싶나요?

브라우저가 웹 페이지를 렌더링하는 방식을 이해하는 것은 성능 최적화와 사용자 경험 향상을 목표로 하는 개발자에게 필수적입니다.

이번 수업에서는 렌더링 과정을 명확하고 이해하기 쉽게 단계별로 설명하겠습니다.

먼저 브라우저가 HTML을 파싱하고 DOM을 만듭니다.

과정은 브라우저가 페이지의 HTML 콘텐츠를 가져오는 것으로 시작합니다. 이 HTML을 파싱하여 문서 객체 모델(DOM)을 구성하는데, 이는 페이지 내용을 나타내는 트리 구조입니다.

다음은 간단한 HTML 예시입니다:

```html
<html>
  <body>
    <h1>Hello, World!</h1>
    <p>Welcome to our website.</p>
  </body>
</html>
```

브라우저는 이것을 파싱하여 `html`를 루트로 하는 DOM 트리를 만듭니다. 이 루트는 `body`를 포함하고, `body`는 다시 `h1`와 `p` 요소를 포함합니다.

다음으로 브라우저는 CSS를 처리하여 CSS 객체 모델(CSSOM)을 만듭니다. 이것도 요소들이 어떻게 스타일링되어야 하는지 결정하는 또 다른 트리 구조입니다.

다음 CSS를 예로 들어보겠습니다:

```css
h1 {
  color: blue;
}
p {
  font-size: 16px;
}
```

CSS 객체 모델은 `h1` 요소가 파란색이고 `p` 요소는 폰트 크기가 16픽셀임을 지정합니다.

그다음 브라우저는 DOM과 CSS 객체 모델을 결합하여 렌더 트리를 만듭니다. 이 트리는 페이지를 렌더링하는 데 필요한 노드만 포함하며, 각 노드는 해당 스타일과 짝을 이룹니다.

예시에서는 렌더 트리가 `h1`와 `p` 요소로 구성되며, 적절한 스타일이 적용되어 있습니다.

렌더 트리를 바탕으로 브라우저는 레이아웃 또는 리플로우라 불리는 과정에서 각 요소의 정확한 위치와 크기를 계산합니다. 이를 통해 요소들이 화면에서 올바른 위치와 크기로 나타납니다.

마지막으로 브라우저는 픽셀을 화면에 그려서 계산된 스타일과 레이아웃에 따라 각 요소를 렌더링합니다. 복잡한 페이지에서는 여러 레이어가 합성되어 최종 시각적 결과를 만듭니다.

결론적으로, HTML과 CSS를 파싱하는 것부터 화면에 픽셀을 그리는 것까지, 브라우저의 렌더링 과정은 정교한 단계들의 수열입니다.
