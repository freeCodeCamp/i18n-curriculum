# --interactive--

**注意**: いくつかのインタラクティブな例では、まだ学習していないCSSが使われている場合があります。コードのすべてを理解しようと心配しないでください。これらの例の目的は、デザインの概念をプレビューで示し、仕組みをよりよく理解してもらうことです。プレビューを見るには、インタラクティブエディターを有効にしてください。

インターネット上には何千ものeコマースサイトがあり、ショッピングカートはeコマース体験の重要な部分です。良いデザインはショッピングカートの体験をより楽しくし、売上を増やすことができます。逆に悪いデザインはカート放棄や売上の損失につながります。

このレッスンでは、ショッピングカート設計のベストプラクティスをいくつか紹介します。最初の設計上の考慮点は、ショッピングカートが常にユーザーに見えるようにすることです。ほとんどのショッピングカートデザインでは、ページの右上にカートが表示されます。ユーザーはカートアイコンの隣にカート内のアイテム数が表示され、それをクリックすれば購入するアイテムの詳細を確認できます。

もう一つの考慮点は、ユーザーがカート内のアイテムの数量を簡単に更新できるようにすることです。これは、カート内の各アイテムの隣に数量入力フィールドを設けることで実現できます。ユーザーは入力フィールドの数字を変えるだけで簡単に数量を更新できます。

また、カート内の各アイテムの隣に「削除」ボタンを設置してください。これによりユーザーは簡単にアイテムをカートから取り除けます。アイテムの削除を難しくすると、ユーザーのフラストレーションやカート放棄につながるため避けるべきです。

さらに、ショッピングカートのアイコン自体も重要です。アイコンはすべてのユーザーにとってわかりやすいものであるべきです。一般的なアイコンは、取っ手と車輪のついたショッピングカートです。他にはショッピングバッグやバスケットのアイコンもあります。ただし、あまり抽象的すぎたり理解しにくいアイコンは避けてください。

ユーザーがカート内の合計金額を確認したいときに、合計金額がすぐにわかるようにしてください。ページ上で目立つ場所に表示し、ユーザーが探さなくても済むようにします。

最後に、ユーザーがチェックアウトに進むための明確なコールトゥアクションボタンを設置してください。このボタンはページ上で目立つように表示し、ユーザーが探さなくてもすぐにわかるようにします。

ページにボタンが多すぎると混乱を招くため避けてください。コールトゥアクションボタンはページで最も目立つボタンにして、ユーザーが次に何をすべきかはっきりわかるようにします。ブランドのメインカラーを使って、ページの他の部分から際立たせてください。
