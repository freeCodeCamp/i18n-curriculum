# --interactive--

`break` 文はループを早期に終了するために使われ、`continue` 文はループの現在の繰り返しをスキップして次の繰り返しに進むために使われます。

以下は `break` ループで `for` 文を使った例です。

:::interactive_editor

```js
for (let i = 0; i < 10; i++) {
  if (i === 5) {
    break;
  }
  console.log(i);
}
```

:::

上の例では、ループは `0` からカウントを始め、`i` が `10` より小さい間ループが続きます。

ループ内で `i` が `5` と等しいかどうかをチェックします。等しい場合は `break` 文を使ってループを早期に終了します。そうでなければ `i` の値をコンソールに出力します。したがって、このコードの出力は `0`、`1`、`2`、`3`、`4` の数字を出力します。

`break` 文は、特定の条件に基づいてループを早期に終了したい場合に便利です。例えば、配列の中で特定の値を検索しているときに、その値が見つかったら `break` 文を使ってループを終了できます。

ループを完全に終了せずに特定の繰り返しだけをスキップしたい場合もあります。ここで `continue` 文が役立ちます。以下は `continue` ループで `for` 文を使った例です。

:::interactive_editor

```js
for (let i = 0; i < 10; i++) {
  if (i === 5) {
    continue;
  }
  console.log(i);
}
```

:::

前と同様に、`i` を `0` に初期化し、`i` が `10` より小さい間ループが実行されます。

ループ内で `i` が `5` と等しいとき、`continue` 文を使って現在の繰り返しをスキップし、次の繰り返しに進みます。

このコードの出力は `0`、`1`、`2`、`3`、`4`、`6`、`7`、`8`、`9` の数字を出力します。`5` の数字は `continue` 文のためにスキップされます。

`break` と `continue` の両方の文でできるもう一つのことは、ラベルを使ってどのループを break または continue するかを指定することです。

これはネストされたループがある場合に、内側のループから外側のループの制御をしたいときに便利です。

以下は `break` 文でラベルを使った例です。

:::interactive_editor

```js
outerLoop: for (let i = 0; i < 3; i++) {
  innerLoop: for (let j = 0; j < 3; j++) {
    if (i === 1 && j === 1) {
      break outerLoop;
    }
    console.log(`i: ${i}, j: ${j}`);
  }
}
```

:::

この例では、`for` というラベルが付いた外側の `outerLoop` ループと、`for` というラベルが付いた内側の `innerLoop` ループがあります。

`i` が `1` と等しく、かつ `j` が `1` と等しいとき、`break` ラベルを付けた `outerLoop` 文を使って外側のループを早期に終了します。これにより内側と外側の両方のループが終了します。

このコードの出力はコンソールに以下を記録します。

```md
"i: 0, j: 0"
"i: 0, j: 1"
"i: 0, j: 2"
"i: 1, j: 0"
```

ほとんどの場合、`break` と `continue` 文でラベルを使う必要はありませんが、必要になったときにその選択肢があることを知っておくのは良いことです。
