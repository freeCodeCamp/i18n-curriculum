# --description--

Come promemoria, questo progetto si basa sul seguente progetto starter clonato da <a href="https://github.com/freeCodeCamp/boilerplate-infosec/" target="_blank" rel="noopener noreferrer nofollow">GitHub</a>.

Questa sfida mette in evidenza una nuova promettente difesa che può ridurre significativamente il rischio e l'impatto di molti tipi di attacchi nei browser moderni. Impostando e configurando una content security policy puoi prevenire l'iniezione di qualsiasi contenuto non intenzionale nella tua pagina. Questo proteggerà la tua app da vulnerabilità XSS, tracciamenti indesiderati, frame dannosi e molto altro. La CSP funziona definendo una lista di fonti di contenuto consentite e affidabili. Puoi configurarle per ogni tipo di risorsa di cui una pagina web può aver bisogno (script, fogli di stile, caratteri tipografici, frame, media e così via...). Sono disponibili molteplici direttive, così il proprietario di un sito web può avere un controllo granulare. Consulta HTML 5 Rocks, KeyCDN per maggiori dettagli. Purtroppo la CSP non è supportata dai browser più vecchi.

Per impostazione predefinita, le direttive sono molto aperte, quindi è importante impostare la direttiva defaultSrc come fallback. Helmet supporta sia lo stile di denominazione defaultSrc sia default-src. Il fallback si applica alla maggior parte delle direttive non specificate.

# --instructions--

In questo esercizio, usa `helmet.contentSecurityPolicy()`. Configuralo aggiungendo un oggetto `directives`. Nell'oggetto, imposta `defaultSrc` su `["'self'"]` (la lista delle fonti consentite deve essere in un array), in modo da fidarti per impostazione predefinita solo dell'indirizzo del tuo sito web. Imposta anche la direttiva `scriptSrc` in modo da permettere di scaricare script solo dal tuo sito (`'self'`) e dal dominio `'trusted-cdn.com'`.

Suggerimento: nella parola chiave `'self'` le virgolette singole fanno parte della parola chiave stessa, quindi devono essere racchiuse tra virgolette doppie per funzionare.

La tua configurazione csp non è corretta. defaultSrc dovrebbe essere ["'self'"] e scriptSrc dovrebbe essere ["'self'", 'trusted-cdn.com']

```js
const response = await fetch(code + '/_api/app-info');
if (!response.ok) {
  throw Error(await response.text());
}
const data = await response.json();
var cspHeader = Object.keys(data.headers).filter(function (k) {
  return (
    k === 'content-security-policy' ||
    k === 'x-webkit-csp' ||
    k === 'x-content-security-policy'
  );
})[0];
assert.equal(
  data.headers[cspHeader],
  "default-src 'self'; script-src 'self' trusted-cdn.com"
);
```
