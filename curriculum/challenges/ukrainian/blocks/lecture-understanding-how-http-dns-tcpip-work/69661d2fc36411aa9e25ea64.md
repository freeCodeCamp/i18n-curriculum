# --description--

Давайте дізнаємося про загальні коди відповіді HTTP.

Коди відповіді HTTP — це трицифрові числові коди, які вебсервери використовують для спілкування з клієнтами, щоб повідомити, що сталося з їхніми запитами.

Вони надають інформацію про те, чи було оброблено запит успішно, чи сталася помилка під час обробки запиту.

У цій лекції ми розглянемо деякі з найпоширеніших кодів.

## Коди успіху

Почнемо з **кодів успіху**. Ці коди починаються з цифри 2.

Код **200 OK** — найпоширеніший код відповіді HTTP. Він означає, що запит було успішно виконано, і сервер повернув інформацію, яку запитував клієнт.

Код **201 Created** повертається сервером, коли клієнт запитує створення ресурсу, і процес успішно завершено. Наприклад, якщо новий обліковий запис користувача створено успішно, сервер повертає цей код відповіді клієнту.

Код **204 No Content** повертається, коли запит було успішно оброблено, але сервер не має вмісту для повернення. Це може використовуватися для процесів, які видаляють або оновлюють ресурс у базі даних, але не мають даних для повернення.

## **Коди перенаправлення**

Тепер поговоримо про **коди перенаправлення**.

Ці коди повертає сервер, коли ресурс, запитаний клієнтом, було переміщено в інше місце або коли клієнту потрібно виконати додаткові кроки для завершення запиту.

Ці коди починаються з цифри 3.

Код **301 Moved Permanently** повертається, коли ресурс, запитаний клієнтом, було назавжди переміщено в інше місце. У цьому випадку сервер повинен включити URL нової локації у відповідь, і після отримання відповіді клієнт має перенаправити користувача на цей URL.

Код **302 Found** означає, що ресурс, запитаний клієнтом, тимчасово переміщено в інше місце. Сервер повинен надати новий URL у відповіді, і клієнт має перенаправити користувача на цей URL. Однак оригінальний URL може бути доступним у майбутньому, оскільки зміна тимчасова. Це основна різниця між кодами відповіді 301 і 302.

Код **304 Not Modified** повертається сервером, щоб повідомити клієнту, що ресурс, який запитували, не змінювався з моменту останнього запиту. Сервер надсилає цей код, щоб уникнути повторної відправки даних, які вже були надіслані, якщо вміст не змінився. Це техніка оптимізації продуктивності.

## Коди помилок клієнта

Тепер почнемо розглядати **коди помилок**.

Помилки можуть бути спричинені клієнтом або сервером. Спочатку поговоримо про коди помилок клієнта.

Коди помилок клієнта повертаються, коли запит не вдалося виконати через проблему, створену клієнтом.

Ці коди починаються з цифри 4.

Код **400 Bad Request** повертається, коли сервер не може зрозуміти запит, надісланий клієнтом. Це може статися, коли клієнт надсилає запит з неправильним синтаксисом, недійсними параметрами або некоректними даними.

Код **401 Unauthorized** пов’язаний з автентифікацією. Він означає, що клієнт має надати певні автентифікаційні дані для доступу до запитуваного ресурсу. Ці дані можуть включати ім’я користувача та пароль.

Код **403 Forbidden** повертається, коли клієнт не має права доступу до запитуваного ресурсу. Різниця між 401 і 403 полягає в тому, що у випадку 403 клієнт не має необхідних дозволів для конкретного ресурсу, навіть якщо має дійсні автентифікаційні дані.

Код **404 Not Found** повертається, коли ресурс не знайдено на сервері. Можливо, його видалили або в URL є помилка, яка заважає серверу знайти ресурс. Сервер також може тимчасово мати проблему, що призводить до цього коду помилки.

## Коди помилок сервера

Далі йдуть **коди помилок сервера**. Ці коди повертає сервер, коли запит не вдалося обробити через проблему на стороні сервера.

Код помилки **500 Internal Server Error** означає, що на сервері сталася несподівана помилка. Цю помилку можуть спричинити різні проблеми — від налаштувань до тимчасових збоїв.

Код помилки **502 Bad Gateway** повертається, коли сервер намагається зв’язатися з іншим сервером, але отримує від нього недійсну відповідь. Причин для такої помилки може бути багато.

Код помилки **503 Service Unavailable** означає, що сервер тимчасово не зможе обробити запит, тому клієнту слід спробувати пізніше. Поширені причини цієї помилки — роботи з обслуговування та великий обсяг запитів, які сервер наразі не може обробити.

Коди відповіді HTTP є важливими для розуміння комунікації між клієнтами та вебсервером. Розуміючи ці коди, ви зможете ефективніше усувати неполадки та налагоджувати свої вебзастосунки.
