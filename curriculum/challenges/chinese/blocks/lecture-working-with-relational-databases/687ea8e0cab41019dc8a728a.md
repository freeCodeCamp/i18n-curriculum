# --description--

正如你在之前的课程中学到的，关系数据库将数据保存在由行和列组成的表格中。

“关系数据库”中的“关系”方面来自于不同表格中的记录可以具有不同类型的关系。

关系有五种主要类型：

- 一对一
- 一对多
- 多对一
- 多对多
- 以及自引用关系（也称为递归的关系）。

当第一个表格中的每条记录最多只能与第二个表格中的一条记录关联，反之亦然时，这两个表格具有**一对一**关系。

例如，假设一家公司每个员工只赋值一辆车辆。

在这种情况下，数据库可能有一个 `employees` 表格和一个 `vehicles` 表格，其中每个员工记录对应数据库中唯一的一个车辆记录。

相比之下，**一对多** 关系发生在一个表格中的一条记录可以与另一个表格中的一条或多条记录相关联时。

例如，如果我们有一个 `customers` 表格和一个 `orders` 表格，每个客户可以关联一个或多个订单，但每个订单只能关联一个客户。

这与**多对一**关系相同，但视角相反。一个或多个订单可以关联到一个客户。

**多对多** 关系发生在一个表格中的记录可以与另一个表格中的多个记录相关联，反之亦然。

例如，让我们分析一个用于 `library` 的数据库。为简化起见，假设该数据库只有两个 `table`：`books` 和 `authors`。

一个作者可以写多本书，一本书也可以由多个作者共同撰写。

实现这种类型的关系稍微复杂一些，因为关系模型在其核心并不直接支持多对多关系。

为此，你通常会创建一个中间表格，也称为连接表格。

此表格通过将多对多关系转换为两个一对多关系来解决这些初始限制。

它在第一个表格和连接表格之间创建一对多关系，并在第二个表格和连接表格之间创建一对多关系。

连接表的名称通常是实体的组合。在我们关于 `books` 和 `authors` 的示例上下文中，你可以将其称为 `books_authors`。

在连接表本身，你会通过创建两个列来关联作者和他们的书籍：一个用于作者的 id（`author_id`）的列，另一个用于书籍的 id（`book_id`）的列。

这些将是外键，分别引用 `authors` 和 `books` 表格中对应的主键。

通过这个连接表，你将能够查询给定书籍的所有作者或给定作者写的所有书籍。

使用连接表可以更轻松地查询数据，防止冗余，并简化数据库模式。它们对于实现多对多关系非常有帮助。

最后，我们查找**自引用**关系，这种关系发生在一个表格的记录可以与同一表格上的其他记录相关联时。这也称为递归关系。

理解这些关系对于设计和建模复杂且高效的关系数据库至关重要。
