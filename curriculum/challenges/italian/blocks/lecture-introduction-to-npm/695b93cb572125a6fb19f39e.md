# --description--

Quando installi dipendenze con npm, soprattutto in un progetto esistente, npm guarda il tuo file `package.json` per vedere quali pacchetti servono al progetto.

Ogni dipendenza in `package.json` include un numero di versione come questo:

```bash
"lodash": "^4.17.21"
```

Come hai imparato prima, il **caret (^)** permette a npm di installare versioni minori e patch più recenti come `4.18.0` o `4.17.22`. Questa flessibilità è utile, ma può anche causare problemi.

Se due sviluppatori eseguono `npm install` in momenti diversi, npm potrebbe installare versioni leggermente diverse dello stesso pacchetto. Anche piccole differenze possono causare bug o comportamenti inattesi.

Il file `package-lock.json` risolve questo problema. Viene creato automaticamente la prima volta che esegui `npm install` e blocca le versioni esatte di ogni dipendenza installata—incluse tutte le dipendenze figlie, cioè le dipendenze delle dipendenze.

Questo garantisce che tutte le persone che lavorano al progetto abbiano sempre le stesse versioni dei pacchetti.

Ecco un semplice file `package.json` con `lodash` come dipendenza:

```bash
{
  "name": "package-lock-json",
  "version": "1.0.0",
  "description": "",
  "main": "index.js",
  "scripts": {
    "test": "echo \"Error: no test specified\" && exit 1"
  },
  "keywords": [],
  "author": "",
  "license": "ISC",
  "type": "commonjs",
  "dependencies": {
    "lodash": "^4.17.21"
  }
}
```

E qui c’è la sezione corrispondente dal `package-lock.json`:

```bash
"node_modules/lodash": {
  "version": "4.17.21",
  "resolved": "https://registry.npmjs.org/lodash/-/lodash-4.17.21.tgz",
  "integrity": "sha512-v2kDEe57..."
}
```

Nota la differenza:

* In `package.json`, `lodash` è elencato come `^4.17.21` — una regola flessibile.
    
* In `package-lock.json`, `lodash` è bloccato alla versione `4.17.21`.
    

Non importa quando qualcuno esegue `npm install`, otterrà quella versione esatta.

In questo modo, non importa quando qualcuno esegue `npm install`, otterrà sempre la stessa versione di Lodash.

Nota che questo vale anche per le dipendenze figlie, cioè le dipendenze di qualsiasi dipendenza che usi nel progetto.

Puoi vedere che oltre a Express stesso, ci sono altri pacchetti come `accepts`, `mime-types` e `negotiator`. Questi sono dipendenze figlie, cioè pacchetti di cui Express dipende internamente per funzionare.

Nota anche che, mentre le `dependencies` in `node_modules/express` usano caratteri caret per gli intervalli, se trovi la voce per ogni dipendenza figlia come `node_modules/etags`, sono bloccate a una versione specifica.

In breve, il file `package-lock.json` mantiene tutti allineati e garantisce coerenza tra le macchine degli sviluppatori, i server CI, le distribuzioni e altri ambienti.

È importante fare sempre il commit del file `package-lock.json` con Git così il tuo team condivide la stessa configurazione. Inoltre, non modificare manualmente il file `package-lock.json`—npm lo aggiornerà automaticamente ogni volta che cambiano le dipendenze.
