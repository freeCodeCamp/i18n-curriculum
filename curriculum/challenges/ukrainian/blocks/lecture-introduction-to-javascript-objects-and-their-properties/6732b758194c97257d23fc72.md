# --interactive--

У JavaScript конструктор — це спеціальний тип функції, який використовується для створення та ініціалізації об’єктів. Його викликають за допомогою ключового слова `new` і він може ініціалізувати властивості та методи у новоствореному об’єкті.

У цьому уроці ми розглянемо, як працювати з конструктором `Object()`. Конструктор `Object()` створює новий порожній об’єкт. Ось приклад:

```js
new Object()
```

Коли ви викликаєте `new Object()`, він повертає новий об’єкт, який можна використовувати для збереження значень.

Конструктор `Object()` можна використовувати з ключовим словом `new` або без нього. Коли його викликають як функцію без `new`, він поводиться по-різному залежно від типу переданого значення. Ось приклад використання конструктора `Object()` без ключового слова `new`:

:::interactive_editor

```js
const num = 42;
const numObj = Object(num); // Creates an object wrapper for the number

console.log(numObj);
console.log(typeof numObj); // "object"
```

:::

Як ви бачите у другому `console.log`, `numObj` є об’єктом. Це відбувається тому, що ми використали конструктор `Object()`, щоб перетворити вхідне число на об’єкт.

Що станеться, якщо ми спробуємо передати `null` або `undefined` у конструктор `Object()`?

:::interactive_editor

```js
const newObj = new Object(undefined);
console.log(newObj); // {}
```

:::

Результатом буде порожній об’єкт. Ще одним випадком використання конструктора `Object()` є ситуація, коли ви працюєте зі значенням невідомого типу і потрібно переконатися, що це об’єкт. Розглянемо наступний приклад:

:::interactive_editor

```js
function toObject(value) {
  if (value === null || value === undefined) {
    return {};
  }

  if (typeof value === "object") {
    return value;
  }

  return Object(value);
}

console.log(toObject(null));

console.log(toObject(true));

console.log(toObject([1, 2, 3]));
```

:::

У цьому прикладі у нас є функція з назвою `toObject`. Друга умова перевіряє, чи є значення типом об’єкта, і повертає це значення, якщо умова `true` виконується. Ця умова перевіряє як об’єкти, так і масиви, оскільки масиви є спеціальними типами об’єктів.

Якщо жодна з умов не є істинною, функція повертає `Object(value)`, який перетворює вхідні дані на об’єкт. Це працює для значень типу число, рядок і булевий.

Здебільшого ви не будете використовувати конструктор `Object()` для створення нових об’єктів, бо замість цього будете використовувати літеральний синтаксис об’єктів (наприклад, `const objectLiteral = { name: "Beau" }`). Але все одно корисно розуміти основи роботи з конструктором Object.
