# --description--

Node.js haina seva ya WebSocket iliyojengwa ndani. Wasanidi programu kawaida hutumia vifurushi vya nje kama `ws` kutekeleza itifaki ya WebSocket na kushughulikia mawasiliano ya wakati halisi kati ya wateja na seva.

Maktaba ya `ws` ni nyepesi, haraka, na rahisi kutumia. Inatekeleza itifaki ya WebSocket ili uweze kuunda na kusimamia muunganisho wa WebSocket.

Unaweza kuiweka kwa kutumia npm:

```bash
npm install ws
```

Mara baada ya kuwekwa, tengeneza seva ya WebSocket kwa kuimport maktaba ya `ws`, kuita njia yake ya `Server`, na kupitisha port:

```js
const WebSocket = require("ws");

const server = new WebSocket.Server({
  port: 3000,
});
```

Hii inaanzisha seva ya WebSocket inayosikiliza muunganisho kwenye port `3000`.

Mteja anapounganishwa, seva hutuma tukio la `connection`.

```js
server.on("connection", (socket) => {
  console.log("Client connected");
});
```

Fikiria tukio hili kama njia ya seva kusema "mteja mpya amejiunga".

Kigezo cha `socket` kinawakilisha muunganisho kati ya seva na mteja huyo. Unakitumia kutuma na kupokea ujumbe.

Mteja aliyeunganishwa anapotuma data, seva hutuma tukio la `message`.

```js
server.on("connection", (socket) => {
  socket.on("message", (message) => {
    console.log(message.toString());
  });
});
```

Katika mfano huu, seva inasikiliza ujumbe unaokuja na kuandika kwenye konsoli.

Unaweza kutuma data kurudi kwa mteja kwa kutumia njia ya `send()`:

```js
socket.send("Hello from the server");
```

Hii inatuma ujumbe kupitia muunganisho wa wazi wa WebSocket.

Muda muunganisho unapoendelea kuwa wazi, mteja na seva wanaweza kuendelea kubadilishana ujumbe kwa wakati halisi.

Mbali na `connection` na `message`, muunganisho wa WebSocket pia hutuma matukio ya `close` na `error`.

`close` hutokea wakati muunganisho unafungwa:

```js
socket.on("close", () => {
  console.log("Disconnected");
});
```

`error` hutokea wakati kuna tatizo:

```js
socket.on("error", (error) => {
  console.error(error);
});
```

Pamoja, matukio haya ni msingi wa programu nyingi za WebSocket.

Hitaji la kawaida katika programu za mazungumzo ni utangazaji. Utangazaji maana yake ni kutuma ujumbe kwa wateja wengi waliounganishwa badala ya mmoja tu.

Fikiria chumba cha mazungumzo chenye watumiaji watatu wanaoitwa **John Doe**, **Jane Doe**, na **Wes Smith**.

Ikiwa John Doe atatuma ujumbe, seva inaweza kuhitaji kupeleka ujumbe huo kwa Jane Doe na Wes Smith.

Ili kufanya hivyo, seva huhifadhi kumbukumbu za wateja waliounganishwa na kuzunguka kwao wakati ujumbe unakuja.

Kabla ya kutuma ujumbe, seva hukagua kama muunganisho wa mteja bado uko wazi:

```js
if (client.readyState === WebSocket.OPEN) {
  // do something
}
```

Kila muunganisho wa WebSocket una sifa ya `readyState` inayobainisha hali yake ya sasa.

`WebSocket.OPEN` ina maana muunganisho ni hai na uko tayari kutuma na kupokea data.

Kwa kukagua hili, seva inazuia kutuma ujumbe kwa wateja waliotoka au ambao muunganisho wao bado unaanzishwa:

```js
server.on("connection", (socket) => {
  socket.on("message", (message) => {
    server.clients.forEach((client) => {
      if (client.readyState === WebSocket.OPEN) {
        client.send(message.toString());
      }
    });
  });
});
```

Katika mfano huu, kila mteja aliyeunganishwa anapokea ujumbe unaokuja. Hii inaitwa utangazaji.

Programu nyingi za wakati halisi hutumia utangazaji kusambaza data kama ujumbe wa mazungumzo, arifa za moja kwa moja, taarifa za michezo, na matukio ya michezo ya wachezaji wengi.

Kwa maktaba ya `ws`, ni rahisi kujenga seva za WebSocket katika Node.js.

Unachohitaji ni kuunda seva, kusikiliza matukio, kubadilishana ujumbe, na kutangaza masasisho kwa wateja waliounganishwa.

Vifunguo hivi ni msingi wa programu nyingi za wakati halisi.
