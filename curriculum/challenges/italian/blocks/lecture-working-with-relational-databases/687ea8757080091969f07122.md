# --description--

SQL sta per Structured Query Language. È stato sviluppato negli anni ’70, quando era inizialmente noto come structured English query language (SEQUEL). Questo termine è stato poi abbreviato in SQL.

SQL è un linguaggio di programmazione usato per archiviare e gestire dati in database relazionali.

Come hai imparato in una lezione precedente, i database relazionali organizzano i dati in tabelle. Ogni tabella ha un insieme di righe e colonne, dove sono rappresentati gli attributi dei dati e le loro relazioni. Ogni riga rappresenta un record mentre ogni colonna rappresenta un attributo o campo specifico di quel record.

Con SQL puoi aggiungere, modificare, eliminare, trovare e recuperare dati da un database relazionale. Fornisce un insieme completo di comandi per interrogare, filtrare, ordinare e aggregare i dati.

SQL può aiutare a ottimizzare le prestazioni del database, interrogando solo i dati necessari, e può essere facilmente integrato con un’ampia gamma di linguaggi di programmazione, così puoi usarlo per interagire con i database direttamente dalle tue applicazioni.

È stato anche adottato come standard industriale. Garantisce l’integrità dei dati e include molte funzionalità di sicurezza, come il supporto per l’autenticazione degli utenti e la crittografia dei dati.

È inoltre scalabile, portabile e compatibile con un’ampia gamma di sistemi di gestione di database.

SQL si basa su comandi, comunemente noti come istruzioni SQL o query SQL.

Come sviluppatore, scriverai queste istruzioni usando specifici elementi del linguaggio SQL o parole chiave. Ti permettono di eseguire le operazioni necessarie sul tuo database.

Prima di poter inserire comandi SQL, devi aprire la shell psql per poter interagire direttamente con il database PostgreSQL. Nel prompt dei comandi o nel terminale, inserisci:

```bash
psql -U <username> -d <database_name>
```

Sostituisci `username` con il tuo nome utente e `database_name` con il database a cui vuoi connetterti.

Se non hai ancora creato un tuo database, PostgreSQL include automaticamente un database predefinito chiamato `postgres` quando viene installato. Puoi usare questo database `postgres` per seguire la lezione.

Una volta connesso, vedrai che il prompt cambia in:

```bash
postgres=#
```

Mostra il nome del database a cui sei connesso e attende i comandi SQL.

Passiamo ad alcuni dei comandi SQL più fondamentali che dovresti conoscere. Noterai che, per convenzione, i comandi SQL terminano con un punto e virgola (`;`).

Per prima cosa, puoi usare questo comando per creare un database chiamato `my_database`:

```sql
CREATE DATABASE my_database;
```

Puoi digitare `CREATE` seguito da `DATABASE` e dal nome del database che vuoi creare. Nota che le parole chiave SQL del comando sono scritte in lettere maiuscole.

Poi, una volta che hai il tuo database, puoi connetterti a esso.

Il modo in cui ti connetti al database dipende davvero dall’ambiente e dallo strumento che stai usando.

Nel prompt dei comandi o nel terminale, puoi usare il metodo mostrato prima. Nella shell interattiva psql, puoi cambiare database con il comando di scelta rapida `\c` seguito dal nome del database così:

```bash
\c my_database
```

Nota che il comando `\c` e altri comandi di scelta rapida (quelli con una barra rovesciata davanti) fanno parte della shell psql e non del linguaggio SQL stesso, quindi non richiedono un punto e virgola per completare il comando.

Ora che ti sei connesso al tuo nuovo database, vedrai che il prompt è cambiato:

```bash
my_database=#
```

Una volta creato e connesso a un database, puoi creare una tabella con le parole chiave `CREATE TABLE`:

```sql
CREATE TABLE products (
  id SERIAL,
  name VARCHAR(255)
);
```

Nel comando scrivi `CREATE TABLE`, seguito dal nome della tabella che vuoi creare. In questo caso, creerà una tabella chiamata `products`.

Poi, tra parentesi tonde, scrivi i nomi delle colonne che la tabella deve avere e specifichi il tipo di dato dei valori che saranno archiviati in ogni colonna.

Nell’esempio, la tabella avrà due colonne, una per l’id del prodotto e un’altra per il nome del prodotto.

La convenzione standard per i nomi di tabelle e colonne è lo snake case, scrivendo le parole in minuscolo e separandole con un underscore (`_`). Per esempio: `delivery_orders`

Questi sono alcuni comandi SQL essenziali che dovresti conoscere per iniziare.

Con SQL puoi facilmente interrogare, manipolare e analizzare dati per prendere decisioni informate e risolvere problemi reali.
