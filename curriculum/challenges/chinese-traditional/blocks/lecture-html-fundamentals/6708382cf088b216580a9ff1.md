# --interactive--

`id` 屬性為 HTML 元素添加唯一的識別字。 

這是一個具有 `id` 為 `title` 的 `h1` 元素範例。 

在 `h1` 元素下方，添加一個 `h2` 元素，並將 `id` 設定為 `"subtitle"`。你可以為 `h2` 撰寫任何你喜歡的文字，並且你會在預覽視窗中看到變更。若要與範例互動，你需要啟用互動式編輯器。

:::interactive_editor

```html
<h1 id="title">Movie Review Page</h1>

```

:::

你可以在你的 JavaScript 或 CSS 中取用 `title` 的 `id` 名稱。以下是一個 CSS 範例，引用 `title` 的 `id`，將文字的 `color` 改為 `red`。

**注意**：在這個互動範例中已為你提供了一些 CSS。不要擔心試著理解 CSS 程式碼，因為你會在未來的課程中學到更多。但如果你想看到文字顏色變成藍色，請啟用互動編輯器，點擊 `styles.css` 分頁，並將 `color: red;` 改成 `color: blue;`。 

:::interactive_editor

```html
<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="utf-8" />
    <meta
       name="viewport"
       content="width=device-width, initial-scale=1.0" />
    <title>Review page Example</title>
    <link rel="stylesheet" href="./styles.css" />
  </head>
  <body>
    <h1 id="title">Movie Review Page</h1>
  </body>
</html>
```

```css
#title {
  color: red;
}
```

:::

在 `title` 前面的雜湊符號（`#`）告訴電腦你想要標的為該值的 `id`。`id` 名稱不應該被使用超過一次，且它們應該始終是唯一的。另一件關於 `id` 值需要注意的事是，它們不能包含空格。以下是將字 `main` 和 `heading` 套用到 `id` 屬性值的範例：

```html
<h1 id="main heading">Main heading</h1>
```

瀏覽器會將此空格視為 `id` 的一部分，這將導致在樣式設計和腳本編寫時出現不必要的問題。`id` 屬性值應僅包含字母、數字、底線和破折號。

與 `id` 屬性相反，`class` 屬性值不需要是唯一的，且可以包含空格。 

這裡是一個將名為 `box` 的類別套用到 `div` 元素的範例。

```html
<div class="box"></div>
```

如果你想為一個元素添加多個類別名稱，可以透過空格分隔名稱來達成。以下是一個將多個類別套用到 `div` 元素的更新範例。

```html
<div class="box red-box"></div>
```

這是將多個類別套用到多個 `div` 元素的另一個範例。 

**注意**：在這個互動範例中已經為你提供了一些 CSS。不要擔心試著理解 CSS 程式碼，因為你會在未來的課程中學到更多。但如果你想改變第一個和第三個方塊的顏色，請啟用互動編輯器並點擊 `styles.css` 標籤，然後將 `background-color: red;` 改成 `background-color: black;`。

:::interactive_editor

```html
<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="utf-8" />
    <meta
       name="viewport"
       content="width=device-width, initial-scale=1.0" />
    <title>Colored boxes example</title>
    <link rel="stylesheet" href="./styles.css" />
  </head>
  <body>
    <div class="box red-box"></div>
    <div class="box blue-box"></div>
    <div class="box red-box"></div>
    <div class="box blue-box"></div>
  </body>
</html>
```

```css
.box {
  width: 100px;
  height: 100px;
}

.red-box {
  background-color: red;
}

.blue-box {
  background-color: blue;
}
```

:::

總結來說，什麼時候應該使用 `id`，什麼時候應該使用 `class`？當你想將一組樣式套用到多個元素時，`class` 是最適合的。如果你想標的特定元素，最好使用 `id`，因為那些值必須是唯一的。

# --questions--

## --text--

你應該何時使用 `id` 而非 `class`？

## --answers--

當你想要一個只應用於一個元素的唯一識別符時，請使用 `class`。

### --feedback--

請記住，`id` 名稱是唯一的，且每個 HTML 文件中只應使用一次。

---

當你需要特定元素的唯一識別字時，使用 `id`，當你想將樣式或行為套用到多個元素時，使用 `class`。

---

當你想要在網頁的不同零件中一致地為多個元素設計樣式時，請使用 `id`。

### --feedback--

請記住，`id` 名稱是唯一的，且每個 HTML 文件中只應使用一次。

---

當你想套用可以被你 CSS 中其他樣式輕易覆寫的樣式時，請使用 `id`。

### --feedback--

請記住，`id` 名稱是唯一的，且每個 HTML 文件中只應使用一次。
