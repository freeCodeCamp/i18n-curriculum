# --interactive--

## オブジェクトの基本

- **定義**: オブジェクトはプロパティで構成されるデータ構造です。プロパティはキーと値で成り立っています。オブジェクトからデータにアクセスするには、ドット記法かブラケット記法のいずれかを使えます。

:::interactive_editor

```js
const person = {
  name: "Alice",
  age: 30,
  city: "New York"
};

console.log(person.name);  // Alice
console.log(person["name"]); // Alice
```

:::

既存のオブジェクトのプロパティを設定するには、ドット記法かブラケット記法を代入演算子と一緒に使います。

:::interactive_editor

```js
const person = {
  name: "Alice",
  age: 30
};

person.job = "Engineer"
person["hobby"] = "Knitting"
console.log(person);  // {name: 'Alice', age: 30, job: 'Engineer', hobby: 'Knitting'}
```

:::

## オブジェクトからプロパティを削除する

- **`delete` 演算子**: この演算子はオブジェクトからプロパティを削除するために使います。

:::interactive_editor

```js
const person = {
  name: "Alice",
  age: 30,
  job: "Engineer"
};

delete person.job;

console.log(person.job); // undefined
```

:::

## オブジェクトにプロパティがあるか確認する

- **`hasOwnProperty()` メソッド**: このメソッドは、オブジェクトが指定したプロパティを自身のプロパティとして持っているかどうかを示すブール値を返します。

:::interactive_editor

```js
const person = {
  name: "Alice",
  age: 30
};

console.log(person.hasOwnProperty("name")); // true
console.log(person.hasOwnProperty("job")); // false
```

:::

- **`Object.hasOwn()` メソッド**: これはオブジェクトが自身のプロパティ（継承されたものではない）を持っているか確認する、現代的で推奨される方法です。構文は `Object.hasOwn(object, propertyName)` です。プロパティがオブジェクトに存在すれば `true` を返し、存在しなければ `false` を返します — プロパティの値に関係なくです。これにより `hasOwnProperty()` より安全で、値が `if (obj.prop)`、`0`、`false`、または `null` の場合でも `undefined` より信頼性があります。

:::interactive_editor

```js
const person = {
  name: "Alice",
  age: 30
};

console.log(Object.hasOwn(person, "name")); // true
console.log(Object.hasOwn(person, "job")); // false
```

:::

以下は、なぜ `Object.hasOwn()` が値を直接チェックするより信頼できるかを示す詳しい例です。

:::interactive_editor

```js
const settings = {
  darkMode: false,
  fontSize: 0,
  language: null
};

// Object.hasOwn() correctly sees these properties exist
console.log(Object.hasOwn(settings, "darkMode")); // true (value is false, but exists)
console.log(Object.hasOwn(settings, "fontSize")); // true (value is 0, but exists)
console.log(Object.hasOwn(settings, "theme"));    // false (property was never added)

// Using if() directly is unsafe for falsy values!
if (settings.darkMode) {
  console.log("Dark mode on"); // Does NOT print — misleading!
}

// Object.hasOwn() is the safe way
if (Object.hasOwn(settings, "darkMode")) {
  console.log("darkMode exists, value is:", settings.darkMode); // darkMode exists, value is: false
}
```

:::

- **`in` 演算子**: この演算子は、プロパティがオブジェクトに存在すれば `true` を返します。

:::interactive_editor

```js
const person = {
  name: "Bob",
  age: 25
};

console.log("name" in person);  // true
```

:::

## ネストされたオブジェクトからプロパティにアクセスする

- **データへのアクセス**: ネストされたオブジェクトからプロパティにアクセスするには、単純なオブジェクトのプロパティにアクセスするのと同様にドット記法かブラケット記法を使います。ただし、ネストされた構造を掘り下げるためにこれらのアクセス方法をチェーンする必要があります。

:::interactive_editor

```js
const person = {
  name: "Alice",
  age: 30,
  contact: {
    email: "alice@example.com",
    phone: {
      home: "123-456-7890",
      work: "098-765-4321"
    }
  }
};

console.log(person.contact.phone.work); // "098-765-4321"
```

:::

## プリミティブ型と非プリミティブ型のデータ型

- **プリミティブ型データ**: これらのデータ型には数値、文字列、ブール値、`null`、`undefined`、およびシンボルが含まれます。これらの型は「プリミティブ」と呼ばれ、単一の値を表しオブジェクトではありません。プリミティブ値は不変で、一度作成されると値を変更できません。
- **非プリミティブ型データ**: JavaScriptでは、これらはオブジェクトであり、通常のオブジェクト、配列、関数が含まれます。プリミティブとは異なり、非プリミティブ型は複数の値をプロパティや要素として保持できます。

## オブジェクトのメソッド

- **定義**: オブジェクトのメソッドはオブジェクトに関連付けられた関数です。オブジェクトのプロパティとして定義され、オブジェクトのデータにアクセスし操作できます。メソッド内の `this` キーワードはオブジェクト自身を指し、そのプロパティにアクセス可能にします。

:::interactive_editor

```js
const person = {
  name: "Bob",
  age: 30,
  sayHello: function() {
    return "Hello, my name is " + this.name;
  }
};

console.log(person.sayHello()); // "Hello, my name is Bob"
```

:::

## オブジェクトコンストラクタ

- **定義**: JavaScriptにおけるコンストラクタは、オブジェクトを作成し初期化するための特別な種類の関数です。`new` キーワードで呼び出され、新しく作成されたオブジェクトにプロパティやメソッドを初期化できます。`Object()` コンストラクタは新しい空のオブジェクトを作成します。

```js
new Object()
```

## オプショナルチェーン演算子（`?.`）の使い方

- **定義**: この演算子は、存在するかどうかを気にせずにオブジェクトのプロパティに安全にアクセスしたりメソッドを呼び出したりできます。

:::interactive_editor

```ts
const user = {
  name: "John",
  profile: {
    email: "john@example.com",
    address: {
      street: "123 Main St",
      city: "Somewhere"
    }
  }
};

console.log(user.profile?.address?.street); // "123 Main St"
console.log(user.profile?.phone?.number);   // undefined
```

:::

## オブジェクトの分割代入

- **定義**: オブジェクトの分割代入は、オブジェクトから値を抽出し、より簡潔で読みやすい方法で変数に代入できます。

:::interactive_editor

```js
const person = { name: "Alice", age: 30, city: "New York" };

const { name, age } = person;

console.log(name); // Alice
console.log(age);  // 30
```

:::

## JSONの扱い方

- **定義**: JSONはJavaScript Object Notationの略です。これは軽量でテキストベースのデータ形式で、サーバーとウェブアプリケーション間でデータを交換する際によく使われます。

```js
{
  "name": "Alice",
  "age": 30,
  "isStudent": false,
  "list of courses": ["Mathematics", "Physics", "Computer Science"]
}
```

- **`JSON.stringify()`**: このメソッドはJavaScriptオブジェクトをJSON文字列に変換するために使います。これはデータを簡単に共有または転送できる形式で保存や送信したい場合に便利です。

:::interactive_editor

```js
const user = {
  name: "John",
  age: 30,
  isAdmin: true
};

const jsonString = JSON.stringify(user);
console.log(jsonString); // '{"name":"John","age":30,"isAdmin":true}'
```

:::

- **`JSON.parse()`**: このメソッドはJSON文字列をJavaScriptオブジェクトに戻します。ウェブサーバーやlocalStorageからJSONデータを取得し、アプリケーション内でデータを操作したい場合に便利です。

:::interactive_editor

```js
const jsonString = '{"name":"John","age":30,"isAdmin":true}';
const userObject = JSON.parse(jsonString);

// result: { name: 'John', age: 30, isAdmin: true }
console.log(userObject);
```

:::
