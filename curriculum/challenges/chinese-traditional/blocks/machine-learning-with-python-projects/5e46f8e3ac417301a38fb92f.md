# --instructions--

在這個挑戰中，你將使用 **K-Nearest Neighbors** 創建書籍推薦演算法。

在這個專案中，你將使用 Book-Crossings 資料集，該資料集包含 90,000 位使用者對 270,000 本書的 1.1 百萬筆評分（評分範圍為 1-10）。該資料集已經在筆記本中匯入，因此不需要額外下載。

使用來自 `sklearn.neighbors` 的 `NearestNeighbors` 開發一個模型，顯示與給定書籍相似的書籍。最近鄰演算法透過測量距離來判斷實體的「接近度」。

創建一個名為 `get_recommends` 的函式（程式），該函式以書名（來自資料集）作為引數，並傳回包含 5 本相似書籍及其與該書名引數距離的列表。

這段程式碼：

```py
get_recommends("The Queen of the Damned (Vampire Chronicles (Paperback))")
```

應該傳回：

```py
[
  'The Queen of the Damned (Vampire Chronicles (Paperback))',
  [
    ['Catch 22', 0.793983519077301], 
    ['The Witching Hour (Lives of the Mayfair Witches)', 0.7448656558990479], 
    ['Interview with the Vampire', 0.7345068454742432],
    ['The Tale of the Body Thief (Vampire Chronicles (Paperback))', 0.5376338362693787],
    ['The Vampire Lestat (Vampire Chronicles, Book II)', 0.5178412199020386]
  ]
]
```

請注意，從 `get_recommends()` 傳回的資料是一個列表。列表中的第一個元素是傳入函式的書名。列表中的第二個元素是一個包含五個列表的列表。這五個列表中的每一個都包含一本推薦書籍以及該推薦書籍與傳入函式的書籍之間的距離。

如果你繪製資料集（選用），你會注意到大多數書籍的評分並不頻繁。為了確保統計顯著性，請從資料集中移除評分少於 200 次的使用者和評分少於 100 次的書籍。
