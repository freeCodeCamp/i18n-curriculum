# --description--

Lassen Sie uns einige der wichtigsten Techniken zur Verbesserung der JavaScript-Leistung im Speziellen durchgehen.

Es mag Sie überraschen, aber die erste Technik besteht darin, so wenig JavaScript wie möglich zu verwenden.

Wenn Sie eine einfache statische Website entwickeln, können Sie möglicherweise genau dasselbe ohne Frameworks erreichen, die auf JavaScript angewiesen sind.

Sie sollten auch versuchen, ungenutzten JavaScript-Code zu entfernen und, wenn möglich, einfachere Lösungen zu verwenden.

Zum Beispiel könnten Sie in Betracht ziehen, eingebaute Browserfunktionen anstelle von benutzerdefinierten zu verwenden. Zu diesen eingebauten Browserfunktionen gehören die Formularvalidierung, der eingebaute Videoplayer des Browsers und CSS-Animationen anstelle von benutzerdefinierten JavaScript-Animationen.

Verwenden Sie so wenige Animationen wie möglich.

Reduzieren Sie die Menge der DOM-Manipulationen in Ihrem JavaScript-Code, da diese rechenintensiv sind.

Es ist auch hilfreich, Ihren JavaScript-Code in Module aufzuteilen, die kritische und nicht-kritische Aufgaben ausführen. So können Sie die kritischen Module so früh wie möglich vorladen und die nicht-kritischen aufschieben, um die Seite so schnell wie möglich zu rendern.

Das Vorladen von Dateien garantiert nicht, dass sie verfügbar sind, wenn Sie sie verwenden, aber der Download beginnt früher, sodass die Gesamtwartezeit kürzer sein sollte.

Sobald Ihr Code in mehrere Dateien aufgeteilt ist, können Sie diese während des Build-Prozesses automatisch minifizieren, um ihre Größe zu reduzieren und die Ladezeit zu optimieren.

Wenn ein Ereignis-Listener nicht mehr benötigt wird, sollten Sie ihn entfernen:

```js
element.removeEventListener("mousemove", handleMouseMove);
```

In diesem Beispiel entfernen wir den Ereignis-Listener für das `mousemove`-Ereignis, sodass die `handleMouseMove`-Funktion nicht mehr aufgerufen wird, wenn das Ereignis ausgelöst wird. Wenn Sie den Ereignis-Listener beibehalten, führt das Programm weiterhin Berechnungen aus, die nicht mehr notwendig sind, was die Gesamtleistung beeinträchtigt. Dieser Effekt wird noch deutlicher, wenn Sie mehrere aktive Ereignis-Listener gleichzeitig haben.

Und da wir gerade beim Thema Ereignisse sind: Es wird empfohlen, wann immer möglich Event Delegation zu verwenden.

Das bedeutet, dass Sie die Ereignis-Listener nicht auf einzelnen Kindelementen setzen müssen. Sie können sie direkt auf dem Elternelement setzen, und das Ereignis wird nach oben weitergereicht, bis es ein Elternteil findet, das es verarbeiten kann.

Die Optimierung von JavaScript ist entscheidend für die Entwicklung schneller und responsiver Webanwendungen. In einem realen Projekt analysieren Sie ständig die Leistung Ihrer Website, um Verbesserungsmöglichkeiten zu finden.
