# --interactive--

Currying ist eine Technik, bei der wir eine Funktion, die mehrere Argumente annimmt, in eine Folge von Funktionen umwandeln, von denen jede ein einzelnes Argument annimmt.

Beginnen wir mit einem einfachen Beispiel. Stellen Sie sich vor, wir haben eine Funktion, die zwei Zahlen addiert:

:::interactive_editor

```js
function add(a, b) {
  return a + b;
}

console.log(add(3, 4)); // 7
```

:::

Dies ist eine Funktion, die zwei Argumente annimmt und deren Summe zurückgibt. Sehen wir uns nun an, wie wir diese Funktion curry-förmig umwandeln können:

:::interactive_editor

```js
function curriedAdd(a) {
  return function(b) {
    return a + b;
  }
}

console.log(curriedAdd(3)(4)); // 7
```

:::

In diesem curry-konvertierten Code nimmt die Funktion nicht zwei Argumente auf einmal, sondern eine Funktion nimmt das erste Argument und gibt eine weitere Funktion zurück. Diese zurückgegebene Funktion nimmt dann das zweite Argument und führt die Addition aus. Wir rufen sie so auf `curriedAdd(3)(4)`, wobei jedes Paar runder Klammern einen Funktionsaufruf darstellt.

Aber warum sollten wir das tun wollen?

Currying ermöglicht es uns, einige spezielle Funktionen einfach zu erstellen. Zum Beispiel könnten wir eine Funktion erstellen, die immer fünf zu einer beliebigen Zahl addiert:

:::interactive_editor

```js
function curriedAdd(a) {
  return function(b) {
    return a + b;
  }
}

const addFive = curriedAdd(5);
console.log(addFive(10)); // 15
console.log(addFive(20)); // 25
```

:::

Hier ist `addFive` eine Funktion, die immer bereit ist, fünf zu jeder Zahl hinzuzufügen, die wir ihr geben. Dies ist ein einfaches Beispiel für partielle Anwendung, bei der wir eine bestimmte Anzahl von Argumenten an eine Funktion binden und dadurch eine andere Funktion erzeugen, die weniger Argumente benötigt.

Während sich unsere Beispiele auf Funktionen mit zwei Argumenten konzentriert haben, kann Currying auf Funktionen mit beliebig vielen Argumenten angewendet werden.

Im Verlauf Ihrer JavaScript-Reise werden Sie wahrscheinlich viele Situationen finden, in denen Currying Ihren Code sauberer und ausdrucksstärker machen kann.

## --text--

Welches der folgenden Beispiele ist eine curried Funktion in JavaScript?

## --answers--

```js
function add(a, b, c) {
  return a + b + c;
}
```

### --feedback--

Dies ist eine reguläre Funktion, die alle Parameter auf einmal annimmt.

---

```js
function add(a) {
  return function (b) {
    return function (c) {
      return a + b + c;
    };
  };
}
```

---

```js
function add(a, b, c) {
  return a() + b() + c();
}
```

### --feedback--

Diese Funktion akzeptiert alle Parameter auf einmal und ruft sie wie Funktionen auf, was nicht der Definition einer curried Funktion entspricht.

---

```js
function add(a) {
  function inner(b) {
    return a + b;
  }
}
```

### --feedback--

Die innere Funktion wird nie zurückgegeben, daher wird `add(2)` `undefined` zurückgeben und `add(2)(3)` wird einen Fehler werfen.
