# --description--

Urithi ni dhana kuu inayofuata katika programu za kuzingatia darasa la vitu (OOP) tutakayojifunza.

Tuchunguze kwa undani zaidi dhana hii na jinsi inavyokuwezesha kuandika msimbo unaoweza kutumika tena.

Kwa urithi, darasa la mtoto (au darasa la kijicho) linaweza kutumia sifa za kipengele na njia za darasa la msingi (au darasa la mzazi). Hii inakuwezesha kutumia msimbo tena, kuunda mlinganyo wa darasa ulio wazi, na kubinafsisha tabia bila kuandika msimbo mzima upya. Unaweza kubinafsisha kwa kuongeza njia zilizopo au kuzighairi katika darasa la mtoto.

Hii ndiyo sintaksia ya msingi ya urithi:

```py
class Parent:
    # Parent attributes and methods

class Child(Parent):
    # Child inherits, extends, and/or overrides where necessary
```

Ili darasa la `Child` kurithi kutoka darasa la `Parent`, lazima upitishe Mzazi kwa Mtoto.

Mtindo huu unaitwa urithi wa moja kwa moja, kwa sababu darasa la mtoto hurithi kutoka kwa darasa moja tu la mzazi.

Hapa kuna mfano:

```py
class Animal:
    def __init__(self, name):
        self.name = name

    def sound(self):
        return f'{self.name} makes a sound'

class Dog(Animal):
    bark = 'woof! woof!! woof!!!'

jack = Dog('Jack')
print(jack.sound())  # Jack makes a sound
print(jack.bark)  # woof! woof!! woof!!!
```

Unaweza kuona kwamba tunaweza kutumia tena sifa ya kipengele ya `self.name` na njia ya `sound()` kutoka kwa darasa la mzazi la `Animal` katika darasa la mtoto la `Dog`.

Tughairi njia ya `sound()` kutoka kwa darasa la mzazi la `Animal` katika darasa la mtoto la `Dog` ili tuweze kutumia `sound()` thamani ya darasa la `bark`:

```py
class Animal:
    def __init__(self, name):
        self.name = name

    def sound(self):
        return f'{self.name} makes a sound.'

class Dog(Animal):
    bark = 'woof! woof!! woof!!!'

    # Override sound() to use bark class variable
    def sound(self):
        return f'{self.name} barks {self.bark}'

jack = Dog('Jack')
print(jack.sound())  # Jack barks woof! woof!! woof!!!
```

Ikiwa unataka kuhifadhi thamani ya return ya `sound()` na kuongeza thamani ya darasa ya bark baadaye, unaweza kuongeza `sound()` kwa kutumia kitendakazi cha `super()`:

```py
class Animal:
    def __init__(self, name):
        self.name = name

    def sound(self):
        return f'{self.name} makes a sound'

class Dog(Animal):
    bark = 'woof! woof!! woof!!!'

    # Call Animal.sound(), then append bark
    def sound(self):
        base = super().sound()
        return f'{base}, then {self.name} barks {self.bark}'

jack = Dog('Jack')
print(jack.sound())  # Jack makes a sound, then Jack barks woof! woof!! woof!!!
```

Katika mfano huu, `base` ni matokeo ya wito wa njia ya `sound()` kutoka kwa darasa la `Animal`, kisha tunaongeza sauti maalum ya darasa la `Dog`. Kwa njia hii, unaweza kuongeza utendaji wa darasa la mzazi la `Animal` huku ukihifadhi tabia yake ya awali.

Pia kuna urithi wa mara nyingi, ambapo darasa la mtoto linaweza kurithi kutoka kwa madarasa zaidi ya moja ya mzazi.

Hii ndiyo sintaksia ya msingi ya urithi wa mara nyingi:

```py
class Parent:
    # Attributes and methods for Parent

class Child:
    # Attributes and methods for Child

class GrandChild(Parent, Child):
    # GrandChild inherits from both Parent and Child
    # GrandChild can combine or override behavior from each
```

Njia rahisi ya kuonyesha urithi wa mara nyingi ni kwa mfano wa chura, ambaye anaweza kutembea ardhini na kuogelea katika maji:

```py
class Walker:
    def walk(self):
        return 'I can walk on land'

class Swimmer:
    def swim(self):
        return 'I can swim in water'

# Amphibian inherits from both Walker and Swimmer
class Amphibian(Walker, Swimmer):
    def __init__(self, name):
        self.name = name

    def introduce(self):
        return f"I'm {self.name} the frog. {self.walk()} and {self.swim()}."

frog = Amphibian('Freddy')
print(frog.introduce())
# Output: I'm Freddy the frog. I can walk on land and I can swim in water.
```
