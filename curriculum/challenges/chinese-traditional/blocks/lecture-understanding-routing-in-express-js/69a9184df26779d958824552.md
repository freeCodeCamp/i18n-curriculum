# --description--

在網頁應用程式中，提供靜態檔案是常見的需求。靜態檔案包括影像、CSS 檔案、客戶端 JavaScript、字型和其他不會動態變化的資源。

Express 透過內建的 `express.static()` 中介層，讓提供這些檔案變得簡單。

在 Express 中，靜態檔案提供是指直接從伺服器的檔案系統將檔案傳送到客戶端瀏覽器，而不需額外的伺服器端處理。

舉例來說，當瀏覽器請求像 `/styles.css`、`/script.js` 或 `/images/logo.png` 這類檔案時，Express 可以在指定的資料夾中找到這些檔案並直接提供。如果找不到檔案，Express 會自動回應 404。由於這些檔案不需要在伺服器端產生或修改，因此稱為*靜態*檔案。

這讓提供像樣式表和影像等資產變得容易，且不需要為每個檔案手動定義路由。

以下是你如何在 Express 中設定靜態檔案提供：

```javascript
const express = require("express");
const path = require("path");
const app = express();
const port = 3000;

// Serve static files from the "public" directory
app.use(express.static(path.join(__dirname, "public")));

app.get("/", (req, res) => {
  res.send("Welcome to the static file demo!");
});

app.listen(port, () => {
  console.log(`Server running at http://localhost:${port}`);
});
```

首先，我們匯入 `express` 來創建伺服器，並匯入 `path` 以跨不同作業系統安全地構建檔案路徑。

接著，`app.use(express.static(path.join(__dirname, "public")))` 告訴 Express 從 `public` 資料夾提供檔案。`express.static()` 中介層告訴 Express 從 `public` 資料夾提供檔案。該目錄內的任何檔案都能被瀏覽器存取。

我們也定義了一個簡單的 `GET` 路由給 `/`，傳送歡迎訊息。這表示靜態檔案提供可以與動態路由共存。

最後，`app.listen()` 在埠 3000 啟動伺服器。伺服器啟動後，對 `public` 資料夾中靜態資產的請求會自動被處理。

要讓這個功能運作，你需要一個名為 `public` 的資料夾來存放靜態檔案。典型的專案結構可能長這樣：

```md
my-app/
│
├── app.js
└── public/
    ├── index.html
    ├── styles.css
    └── image.png
```

當使用者發出請求，例如造訪 `http://localhost:3000/index.html` 時，Express 會檢查請求的檔案是否存在於 `public` 目錄中。如果檔案存在，Express 會直接傳送給瀏覽器；如果不存在，Express 會繼續執行下一個中介層或回應 404。

記住，你不需要為每個檔案定義路由，因此對 `http://localhost:3000/styles.css` 的請求表示 Express 會尋找 `public/styles.css` 並自動回應。

你也可以自訂靜態檔案的提供方式。舉例來說，你可以從多個目錄提供檔案，或設定快取行為。

假設你的應用程式有以下檔案結構：

```md
my-app/
│
├── app.js
│
├── public/
│   ├── index.html
│   └── styles.css
│
└── images/
    └── image.png
```

以下是如何註冊多個目錄來提供靜態檔案：

```javascript
app.use(express.static(path.join(__dirname, "public")));
app.use("/images", express.static(path.join(__dirname, "images")));
```

這裡我們像之前一樣使用 `app.use(express.static(path.join(__dirname, "public")));`，表示 `public` 目錄內的檔案會從根路徑提供。

但如果你看下一行，你會看到我們將 `"/images"` 作為第一個引數傳給 `app.use()`，接著是 `express.static(path.join(__dirname, "images"))`。在這種情況下，`"/images"` 是掛載路徑，告訴 Express 只有當收到 `/images` 的請求時才使用這個中介層。當觸發時，`express.static()` 會尋找 `images` 目錄內的檔案。

因此，根據這個組態，以下請求會這樣運作：

*   GET `http://localhost:3000/styles.css`：Express 會在 `public` 目錄尋找 `styles.css`，找到檔案後傳送給客戶端。
    
*   GET `http://localhost:3000/images/image.png`：由於此請求以 `/images` 開頭，Express 會在 `images` 目錄尋找 `image.png`，找到檔案後傳送給客戶端。
    

使用 Express 提供靜態檔案既簡單又強大。透過 `express.static()`，你可以輕鬆提供影像、樣式表和客戶端 JavaScript 等資產。

將靜態檔案放在像 `public` 這樣的專用資料夾，或使用多個檔案目錄和掛載路徑，有助於讓你的應用程式組織良好、易於維護且方便擴充。
