# --interactive--

Ланцюжок методів — це техніка, коли ви викликаєте кілька методів один за одним. Ви можете використовувати ланцюжок методів для багатьох типів значень у JavaScript, включно з рядками, масивами та об’єктами. Хоча рядки є примітивними значеннями, JavaScript тимчасово обгортає їх в об’єкт `String`, коли ви використовуєте метод рядка.

Розглянемо приклад із різними методами рядків:

:::interactive_editor

```js
const result = "  Hello, World!  "
  .trim()
  .toLowerCase()
  .replace("world", "JavaScript");

console.log(result); // "hello, JavaScript!"
```

:::

У цьому прикладі ми починаємо з рядка і виконуємо три операції послідовно: обрізаємо пробіли, переводимо в нижній регістр і замінюємо рядок `world` на `JavaScript`. Кожен метод повертає новий рядок, який стає ціллю для наступного виклику методу.

Ланцюжок методів може значно покращити читабельність коду при роботі зі складними операціями.

Наприклад, розглянемо цей приклад із методами `filter`, `map` і `reduce`:

:::interactive_editor

```js
const transactions = [
  { amount: 100, type: "credit" },
  { amount: 20, type: "cash" },
  { amount: 150, type: "credit" },
  { amount: 50, type: "cash" },
  { amount: 75, type: "credit" }
];

const totalCreditWithBonus = transactions
  .filter((transaction) => transaction.type === "credit")
  .map((transaction) => transaction.amount * 1.1)
  .reduce((sum, amount) => sum + amount, 0);

console.log(totalCreditWithBonus); // 357.5
```

:::

У цьому прикладі ми маємо масив транзакцій, де кожен об’єкт має суму та тип — кредитна картка або готівка.

Спочатку ми фільтруємо транзакції, створюючи новий масив лише з транзакціями кредитних карток. Потім ми ланцюжком викликаємо метод `map` для відфільтрованого результату і для кожної суми транзакції помножуємо її на `1.1`, що представляє бонус `10%`.

Потім беремо цей результат і ланцюжком викликаємо метод `reduce`, щоб підсумувати всі суми, що дає `357.5`.

Хоча ланцюжок методів робить код більш лаконічним і читабельним, важливо використовувати його розумно.

Дуже довгі ланцюжки можуть бути складними для налагодження, оскільки не одразу зрозуміло, який крок у ланцюжку може спричиняти проблему. Часто корисно розбивати дуже довгі ланцюжки на кілька кроків для кращої зрозумілості та простішого налагодження.
