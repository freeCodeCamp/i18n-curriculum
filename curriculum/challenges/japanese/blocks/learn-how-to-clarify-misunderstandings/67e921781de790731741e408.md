# --explanation--

`Clarifying`は、混乱を取り除くように説明することを意味します。これにより、誰かが概念や状況をよりよく理解できるようになります。例えば：

`Thanks for clarifying how the API works.` - 話者は説明によってAPIが理解しやすくなったことに感謝しています。

`Abstraction`は、複雑な内部の仕組みを隠して重要な機能だけを示すことで単純化することを指します。例えば：

`Using a login function is an example of abstraction - you don't need to know how the password is checked, just that it works.` - これは、ユーザーがシンプルなインターフェースとやり取りしながら技術的なロジックは隠されている様子を示しています。
