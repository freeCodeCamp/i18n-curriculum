# --description--

HTTPS sta per Hypertext Transfer Protocol Secure. È la versione sicura di HTTP.

Come hai imparato in alcune lezioni precedenti, HTTP trasferisce dati tra il tuo browser e un sito web. HTTP standard invia questi dati in testo semplice, quindi chiunque può intercettarli.

Per questo motivo, HTTPS usa un protocollo crittografico chiamato Transport Layer Security (TLS) per proteggere questo trasferimento di dati. TLS cripta il canale di comunicazione tra due punti. I sistemi più vecchi usavano Secure Sockets Layer (SSL), che molte piattaforme di hosting web usano ancora oggi.

La crittografia trasforma quei dati in un codice illeggibile che solo il browser dell'utente e il server web possiedono le chiavi per decifrare.

Per esempio, quando digiti una password su un sito web, HTTPS la cripta immediatamente e protegge le informazioni della carta di credito, gli indirizzi personali, le credenziali di accesso e altre informazioni sensibili. Gli hacker che intercettano i dati vedono solo caratteri casuali, quindi non possono usare le informazioni rubate.

Questa protezione è fondamentale sulle reti Wi-Fi pubbliche. Le reti Wi-Fi aperte in caffè, aeroporti e altri luoghi pubblici mancano di sicurezza di base, permettendo ai criminali informatici di monitorare il traffico non criptato su queste reti. HTTPS crea un tunnel privato attraverso la rete pubblica per mantenere la tua cronologia di navigazione al sicuro da malintenzionati nelle vicinanze.

HTTPS fornisce anche autenticazione, che dimostra che un sito web è legittimo. Per usare HTTPS, il proprietario di un sito deve ottenere un certificato digitale rilasciato da un'organizzazione affidabile. Il certificato verifica la vera identità del proprietario del sito.

Quando visiti un sito e l'indirizzo inizia solo con "http", è un modo per sapere che il proprietario del sito non ha ottenuto quel certificato, rendendo il sito non sicuro. I browser moderni ti avvertono anche riguardo a questi siti.

Infine, l'integrità dei dati è un altro beneficio chiave di HTTPS. Questo significa che nessuno può modificare i tuoi dati durante il trasferimento. Un hacker non può inserire codice dannoso nei file del sito mentre viaggiano verso il tuo dispositivo. Ricevi esattamente i dati che il proprietario del sito ha inviato.

I motori di ricerca come Google danno priorità ai siti HTTPS, poiché i siti sicuri si posizionano più in alto nei risultati di ricerca. I browser moderni richiedono anche HTTPS per funzionalità avanzate come geolocalizzazione, notifiche push e webcam.
