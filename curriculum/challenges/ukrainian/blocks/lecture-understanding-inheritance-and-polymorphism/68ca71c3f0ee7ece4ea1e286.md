# --description--

У попередньому уроці ви дізналися про префіксування атрибутів одним підкресленням і подвійним підкресленням.

Щоб нагадати різницю між ними, одинарне підкреслення — це конвенція, яка означає, що атрибут призначений для внутрішнього використання в класі і не повинен безпосередньо доступатися поза межами класу. Подвійне підкреслення, навпаки, забороняє безпосередній доступ до цього атрибута поза межами класу.

Ось приклад, який демонструє, як працюють обидва варіанти:

```py
class Example:
    def __init__(self):
        self._internal = 'I can be accessed from outside the class, but should not'
        self.__private = 'You cannot access me directly from outside the class'

obj = Example()

print(obj._internal) # I can be accessed from outside the class, but should not
print(obj.__private)  # AttributeError: 'Example' object has no attribute '__private'
```

Префіксування атрибута подвійним підкресленням запускає процес name mangling у Python, під час якого Python внутрішньо перейменовує атрибут, додаючи підкреслення та ім’я класу як префікс, перетворюючи `__attribute` на `_ClassName__attribute`.

Щоб побачити це на практиці, створіть екземпляр класу і використайте спеціальний атрибут `__dict__` цього екземпляра, який є словником, що містить атрибути об’єкта:

```py
class Example:
    def __init__(self, internal, private):
        self._internal = internal
        self.__private = private

example1 = Example(
    'I can be accessed from outside the class, but should not',
    'I cannot be accessed directly from outside the class'
)

print(example1.__dict__)
```

Результат буде таким:

```py
{
  '_internal': 'I can be accessed from outside the class, but should not',
  '_Example__private': 'I cannot be accessed directly from outside the class'
}
```

Як бачите, атрибут `__private` зберігається як `_Example__private`. Це означає, що ви все ще можете отримати доступ до цього атрибута поза межами класу таким чином:

```py
class Example:
    def __init__(self, internal, private):
        self._internal = internal
        self.__private = private

example1 = Example(
    'I can be accessed from outside the class, but should not',
    'I cannot be accessed directly from outside the class'
)
example2 = Example(
    'I should not be accessed from outside the class',
    'But I can be accessed from outside the class with name mangling'
)

print(example1._Example__private) # I cannot be accessed directly from outside the class
print(example2._Example__private) # But I can be accessed from outside the class with name mangling
```

Отже, навіщо Python потрібен name mangling?

Головна мета name mangling — запобігти випадковому заміщенню атрибутів і методів під час використання успадкування. Ось приклад, який це проілюструє:

```py
class Parent:
    def __init__(self):
        self.__data = 'Parent data'

class Child(Parent):
    def __init__(self):
        super().__init__()
        self.__data = 'Child data'

c = Child()
print(c.__dict__) # {'_Parent__data': 'Parent data', '_Child__data': 'Child data'}
```

Ви бачите, що і клас `Parent`, і `Child`, який від нього успадковується, мають окремі атрибути `_class__data`. Це стало можливим завдяки name mangling. Інакше `Child` випадково перезаписав би дані батьківського класу.

Ось що сталося б, якби Python не виконував name mangling, тобто якщо не префіксувати атрибути в обох класах подвійним підкресленням:

```py
class Parent:
   def __init__(self):
       self.data = 'Parent data'

class Child(Parent):
   def __init__(self):
       super().__init__()
       self.data = 'Child data'

c = Child()
print(c.__dict__)  # {'data': 'Child data'}
```

Отже, який префікс слід використовувати для атрибутів: одинарне підкреслення (`_`) чи подвійне підкреслення (`__`)? Це залежить. Якщо атрибут призначений лише для внутрішнього використання в класі, використовуйте одинарне підкреслення.

Але якщо ви працюєте з класом, який буде успадковуватися, слід використовувати подвійне підкреслення, щоб атрибут батьківського класу не був заміщений.
