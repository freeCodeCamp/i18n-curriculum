## Sets in JavaScript

- Ein `Set` ist eine eingebaute Möglichkeit zur Verwaltung von Datensammlungen.
- Sets stellen sicher, dass jeder Wert nur einmal darin vorkommt, was sie nützlich macht, um Duplikate aus einem Array zu entfernen oder Sammlungen von unterschiedlichen Werten zu verwalten.
- Sie können ein `Set` mit dem `Set()`-Konstruktor erstellen:

:::interactive_editor

```js
const set = new Set([1, 2, 3, 4, 5]);
console.log(set); // Set { 1, 2, 3, 4, 5 }
```

:::

- Sets können mit diesen Methoden bearbeitet werden:

  - `add()`: Fügt dem `Set` ein neues Element hinzu.
  - `delete()`: Entfernt ein Element aus dem `Set`.
  - `has()`: Prüft, ob ein Element im `Set` vorhanden ist.
  - `clear()`: Entfernt alle Elemente aus dem `Set`.
  - `keys()` und `values()`: Beide geben ein `SetIterator` zurück, das die Werte des `Set` enthält. Sie sind identisch, da `keys()` ein Alias für `values()` ist.
  - `forEach()`: Zum Durchlaufen der Werte des `Set`.

## Weaksets in JavaScript

- `WeakSet` ist eine Sammlung von Objekten, die es erlaubt, schwach gehaltene Objekte zu speichern.

## Sets vs WeakSets

- Im Gegensatz zu Sets unterstützt ein `WeakSet` keine primitiven Werte wie Zahlen oder Strings.
- Ein `WeakSet` speichert nur Objekte, und die Referenzen auf diese Objekte sind „schwach“, was bedeutet, dass das Objekt automatisch entfernt wird, wenn es sonst nirgendwo im Code verwendet wird, um Speicher freizugeben.

## Maps in JavaScript

- Ein `Map` ist ein eingebautes Objekt, das Schlüssel-Wert-Paare hält, genau wie ein Objekt.
- Maps unterscheiden sich von den Standard-JavaScript-Objekten durch ihre Fähigkeit, Schlüssel beliebigen Typs zu erlauben, einschließlich Objekten und Funktionen.
- Ein `Map` bietet bessere Leistung als das Standardobjekt, wenn es um häufiges Hinzufügen und Entfernen von Schlüssel-Wert-Paaren geht.
- Sie können ein `Map` mit dem `Map()`-Konstruktor erstellen:

:::interactive_editor

```js
const map = new Map([
  ['flower', 'rose'],
  ['fruit', 'apple'],
  ['vegetable', 'carrot']
]);
console.log(map); // Map(3) { 'flower' => 'rose', 'fruit' => 'apple', 'vegetable' => 'carrot' }
```

:::

- Maps können mit diesen Methoden bearbeitet werden:

  - `set()`: Fügt dem `Map` ein neues Schlüssel-Wert-Paar hinzu.
  - `get()`: Ruft den Wert eines Schlüssels aus dem `Map` ab.
  - `delete()`: Entfernt ein Schlüssel-Wert-Paar aus dem `Map`.
  - `has()`: Prüft, ob ein Schlüssel im `Map` vorhanden ist.
  - `clear()`: Entfernt alle Schlüssel-Wert-Paare aus dem `Map`.

Beachten Sie, dass sowohl Maps als auch Sets die `size`-Eigenschaft besitzen, die die Anzahl der eindeutigen Elemente darin zurückgibt.

## WeakMaps in JavaScript

- Ein `WeakMap` ist eine Sammlung von Schlüssel-Wert-Paaren, genau wie `Map`, aber mit schwachen Referenzen auf die Schlüssel. Die Schlüssel müssen Objekte sein, und die Werte können beliebig sein.

## Maps vs WeakMaps

- WeakMaps ähneln WeakSets darin, dass sie nur Objekte speichern und die Referenzen auf diese Objekte „schwach“ sind.
