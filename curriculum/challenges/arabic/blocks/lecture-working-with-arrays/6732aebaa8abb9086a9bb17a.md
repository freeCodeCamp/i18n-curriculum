# --interactive--

في الدرس السابق، تعرّفت على العمل مع المصفوفات وكيفية الوصول إلى عناصر مختلفة فيها. إليك تذكير بكيفية الوصول إلى العنصر الثاني في المصفوفة:

:::interactive_editor

```js
const fruits = ["apple", "banana", "cherry"];

console.log(fruits[1]); // "banana"
```

:::

بما أن المصفوفات تبدأ فهارسها من الصفر، سيكون العنصر الأول عند الفهرس `0`، والعنصر الثاني عند الفهرس `1`، وهكذا. من المهم أن تعرف أنه إذا حاولت الوصول إلى فهرس غير موجود في المصفوفة، ستُرجع JavaScript القيمة `undefined`.

:::interactive_editor

```js
let fruits = ["apple", "banana", "cherry"];
console.log(fruits[3]); // undefined
```

:::

في هذا المثال، لا يوجد عنصر عند الفهرس `3` في مصفوفة `fruits`. لذلك ستظهر في السجل القيمة `undefined`. الآن، لننظر كيف نُحدّث العناصر في المصفوفة. يمكنك تحديث عنصر بتعيين قيمة جديدة لفهرس محدد.

:::interactive_editor

```js
let fruits = ["apple", "banana", "cherry"];
fruits[1] = "blueberry";
console.log(fruits); // ["apple", "blueberry", "cherry"]
```

:::

في هذا المثال، استبدلنا `banana` بـ `blueberry` عند الفهرس `1`. تتيح لك هذه الطريقة تغيير أي عنصر في المصفوفة طالما تعرف فهرسه. يمكنك أيضًا إضافة عناصر جديدة إلى المصفوفة بتعيين قيمة لفهرس غير موجود بعد:

:::interactive_editor

```js
let fruits = ["apple", "banana", "cherry"];
fruits[3] = "date";
console.log(fruits); // ["apple", "banana", "cherry", "date"]
```

:::

لكن كن حذرًا عند القيام بذلك. إذا عيّنت قيمة لفهرس أكبر بكثير من طول المصفوفة الحالي، ستُنشئ عناصر غير معرفة للفهارس بينهما، مما قد يؤدي إلى سلوك غير متوقع. مع استمرارك في العمل مع JavaScript، ستجد أن هذه الطرق للوصول إلى عناصر المصفوفة وتحديثها أساسية للعديد من مهام البرمجة. سواء كنت تبني قائمة مهام بسيطة أو تعالج هياكل بيانات معقدة، ستكون هذه المهارات لا تقدر بثمن.
