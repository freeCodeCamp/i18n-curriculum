## WebSockets와 HTTP 비교

- **HTTP**는 요청-응답 모델을 따릅니다: 클라이언트가 요청을 보내면 서버가 응답을 보내고, 그 후 연결이 종료됩니다.
- **WebSockets**는 클라이언트와 서버 사이에 영구적이고 양방향 연결을 만들어 어느 한쪽이 종료 신호를 보낼 때까지 연결이 유지됩니다.
- HTTP는 페이지를 불러오거나 데이터를 가져오는 데 적합하지만, 실시간 애플리케이션에는 느리고 비효율적입니다.
- 모든 WebSocket 연결은 **HTTP 핸드셰이크**로 시작합니다: 클라이언트가 업그레이드를 요청하고 서버가 `101 Switching Protocols`로 응답합니다.

### WebSocket 연결 흐름

1. 클라이언트가 HTTP를 통해 서버에 연결하고 업그레이드를 요청합니다.
2. 서버가 이를 수락하고 `101 Switching Protocols`로 응답합니다.
3. 연결이 열린 상태로 유지됩니다.
4. 연결이 종료될 때까지 양쪽이 언제든지 데이터를 주고받습니다.

### HTTP와 WebSocket 통신 비교

- **HTTP**: `Client → Request → Server → Response` (한 번의 사이클 후 종료)
- **WebSocket**: `Client ↔ Server` (영구적이고 양방향)

### 도움이 되는 비유

- **HTTP**는 편지를 주고받는 것과 같습니다: 한 통을 쓰고 답장을 기다리며 대화가 끝납니다.
- **WebSockets**는 전화 통화와 같습니다: 양쪽이 필요할 때까지 실시간으로 대화합니다.

### WebSockets의 실제 사용 사례

- 채팅 애플리케이션
- 온라인 멀티플레이어 게임
- 실시간 스포츠 점수 업데이트
- 주식 시장 대시보드
- 협업 도구 (예: Google Docs)

## Pub/Sub 아키텍처

- **Pub/Sub** (게시/구독)는 시스템의 구성 요소들이 서로 직접 알지 못해도 통신할 수 있게 하는 메시징 아키텍처입니다.
- 세 가지 주요 구성 요소가 있습니다: **게시자**, **주제**, 그리고 **구독자**.
  - **게시자**는 메시지를 주제에 보냅니다.
  - **주제**(채널 또는 그룹이라고도 함)는 중개자 역할을 합니다.
  - **구독자**는 자신이 구독한 주제에서 메시지를 받습니다.

### Pub/Sub 메시지 흐름

1. 구독자가 주제를 구독합니다.
2. 게시자가 그 주제에 메시지를 보냅니다.
3. 메시징 시스템이 해당 주제의 모든 구독자에게 메시지를 전달합니다.

### 직접 통신과 Pub/Sub 비교

- **Pub/Sub 없이**: `Application A → Application B` — 밀접하게 결합되어 있어 B의 변경이 A를 망가뜨릴 수 있습니다.
- **Pub/Sub 사용 시**: `Application A → Topic → Subscribers` — A는 단지 게시만 하고 누가 받는지는 알 필요가 없습니다.

### Pub/Sub의 실제 사용 사례

- 여러 방이 있는 채팅 애플리케이션
- 온라인 멀티플레이어 게임
- 알림 시스템
- 실시간 대시보드
- 이벤트 기반 애플리케이션

### WebSockets와 Pub/Sub 비교

- **WebSockets**는 통신 채널(도로)을 제공합니다.
- **Pub/Sub**는 특정 메시지를 누가 받는지 결정하는 메시징 패턴(교통 시스템)을 제공합니다.
- 많은 실시간 애플리케이션이 두 가지를 함께 사용합니다: WebSockets는 연결을 유지하고, Pub/Sub는 메시지를 올바른 구독자에게 라우팅합니다.

## Node.js에서 WebSockets 사용하기

- Node.js에는 내장 WebSocket 서버가 없습니다. `ws` 라이브러리가 표준 외부 패키지로 사용됩니다.
- 설치 명령: `npm install ws`

### WebSocket 서버 생성하기

```js
const WebSocket = require("ws");

const server = new WebSocket.Server({ port: 3000 });
```

### WebSocket 이벤트

| 이벤트 | 트리거 |
|---|---|
| `connection` | 새로운 클라이언트가 서버에 연결됨 |
| `message` | 연결된 클라이언트가 데이터를 보냄 |
| `close` | 연결이 종료됨 |
| `error` | 연결에 문제가 발생함 |

### 클라이언트에게 메시지 보내기

```js
socket.send("Hello from the server");
```

### 모든 연결된 클라이언트에게 방송하기

- `server.clients`는 모든 연결된 클라이언트를 보관합니다.
- 연결이 끊긴 클라이언트에 오류가 발생하지 않도록 `readyState === WebSocket.OPEN`를 확인하세요.

```js
server.on("connection", (socket) => {
  socket.on("message", (message) => {
    server.clients.forEach((client) => {
      if (client.readyState === WebSocket.OPEN) {
        client.send(message.toString());
      }
    });
  });
});
```

- **방송**은 여러 연결된 클라이언트에게 동시에 메시지를 보내는 패턴으로, 채팅 앱, 실시간 알림, 스포츠 업데이트, 멀티플레이어 게임 이벤트에 사용됩니다.
