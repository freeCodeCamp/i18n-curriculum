# --description--

`debugger` 语句是一个强大的 JavaScript 工具，它允许你在特定行暂停你的代码，以便调查程序中发生了什么。正确使用时，`debugger` 语句可以为你节省大量时间，帮助你找出为什么某些东西没有按预期工作。

JavaScript 从上到下执行你的代码。当 JavaScript 执行你的代码并遇到 `debugger` 语句时，它会立即在该行暂停执行。这让你有机会检查变量、查看函数以及代码的整体流程。

通过这样，你可以准确看到哪里是正确的，哪里是错误的。然而，这只有在你的浏览器开发者工具打开时才会发生。否则，`debugger` 语句会被忽略，代码将照常运行。

此外，当你使用 `debugger` 语句时，现代浏览器会在指定行暂停代码执行。它们还允许你通过点击播放按钮恢复执行，但页面本身不会自动重新加载，除非手动触发。

要使用 `debugger` 语句，请将其放置在你希望代码执行暂停的那一行。以下是一个示例：

```js
let firstNumber = 5;
let secondNumber = 10;
debugger; // Code execution pauses here
let sum = firstNumber + secondNumber;
console.log(sum);
```

通过这个示例，如果你在代码运行前没有打开控制台，最终打开时你会看到 `15`。当控制台打开且你重新加载页面时，执行会暂停，以便你检查代码。 

这是一个更复杂的示例：

```js
function calculateTotalPrice(price, discountPercentage) {
 debugger
 let discountAmount = (price * discountPercentage) / 100
 let totalPrice = price - discountAmount

 console.log(`Original Price: ${price}`)
 console.log(`Discount Amount: ${discountAmount}`)
 console.log(`Total Price after Discount: ${totalPrice}`)

 return totalPrice
}

let price = 100
let discount = 15

let finalPrice = calculateTotalPrice(price, discount)
console.log(`Final Price: ${finalPrice}`);
```

在此示例中，`debugger` 语句被放置在 `calculateTotalPrice` 的顶部。当调用该函数时，函数执行将被暂停。

如果你在控制台打开的情况下重新加载页面，你会看到页面不断重新加载，这确认了代码的执行只是暂停，而不是停止。如果你想让执行继续，可以点击播放按钮。

随着你继续构建你的 JavaScript 程序，测试代码中的 `debugger` 语句，看看它如何帮助你更好地理解执行流程、实时检查变量，并快速定位可能出错的地方。
