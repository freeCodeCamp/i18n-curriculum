# --description--

التعامل مع الأحداث جزء أساسي من كل موقع تفاعلي.

يوفر React طريقة قوية ومتسقة للتعامل مع الأحداث عبر نظام الأحداث الاصطناعية الخاص به، وهو غلاف حول الأحداث الأصلية مثل `click` و`keydown` و`submit` التي تعلمتها في الدروس السابقة.

يضمن هذا الغلاف عبر المتصفحات أن تعمل الأحداث بنفس الطريقة في جميع المتصفحات فلا توجد اختلافات.

لنرَ كيف تعمل الأحداث في React حتى تبدأ باستخدامها في مشاريعك.

في React، تعمل معالجات الأحداث بطريقة مشابهة لأحداث المتصفح الأصلية، لكن مع بعض التعديلات.

بدلاً من استخدام أسماء خصائص الأحداث بأحرف صغيرة مثل `onclick` و`onsubmit`، يستخدم React تنسيق سنام الجمل مثل `onClick` و`onSubmit`.

بالإضافة إلى ذلك، بدلاً من استخدام سلاسل نصية لتحديد نوع الحدث، يتوقع React دالة لمعالج الحدث.

تُمرر دالة معالج الحدث إلى العنصر كخاصية، ويُستخدم نوع الحدث مثل `onClick` أو `onSubmit` كخاصية في JSX.

إليك تذكير بكيفية التعامل مع حدث النقر في HTML العادي:

```html
<button onclick="alert('Button clicked!')">Click Me</button>
```

وهنا كيف تفعل نفس الشيء في React:

```jsx
function handleClick() {
  console.log("Button clicked!");
}

<button onClick={handleClick}>Click Me</button>;
```

في هذا المثال، تقوم الدالة `handleClick` بتسجيل رسالة في وحدة التحكم عند نقر المستخدم على الزر.

لاحظ أنك لا تحتاج إلى أقواس عادية بعد `handleClick` في خاصية `onClick`، لأنك تمرر مرجعًا إلى الدالة وليس تستدعيها.

في React، عادةً ما تبدأ دوال معالجات الأحداث بالبادئة `handle` للدلالة على مسؤوليتها عن التعامل مع الأحداث، مثل `handleClick` أو `handleSubmit`.

عندما يطلق فعل المستخدم حدثًا، يمرر React كائن حدث اصطناعي إلى معالجك. يتصرف هذا الكائن مثل كائن الحدث الأصلي في جافاسكريبت العادية، ويوفر خصائص مثل `type` و`target` و`currentTarget`.

يمكنك تمرير `event` كمعلمة إلى دالة المعالج وتسجيله في وحدة التحكم لتفحص هيكله:

```js
function handleClick(event) {
  console.log(event);
}
```

لمنع السلوكيات الافتراضية مثل تحديث المتصفح أثناء حدث `onSubmit`، يمكنك استدعاء طريقة `preventDefault()`:

```jsx
function handleSubmit(event) {
  event.preventDefault();
  console.log("Form submitted!");
}

<form onSubmit={handleSubmit}>
  <input type="text" />
  <button>Submit</button>
</form>;
```

يمكنك أيضًا إيقاف حدث من التصعيد إلى العناصر الأصلية باستدعاء `event.stopPropagation()`.

أحيانًا، أثناء التعامل مع حالات خاصة مثل ميزات الحذف والتعديل، قد ترغب في تمرير بيانات إضافية إلى معالج الحدث. يمكنك فعل ذلك بتغليف المعالج في دالة سهمية مضمّنة:

```jsx
function handleDelete(id) {
  console.log("Deleting item:", id);
}

<button onClick={() => handleDelete(1)}>Delete Item</button>;
```

لا بأس باستخدام معالجات الأحداث المضمّنة في React لأن React يدير عمليات إعادة العرض بكفاءة ويتجنب مشاكل الأداء.

في جافاسكريبت العادية، قد تؤدي معالجات الأحداث المضمّنة إلى مشاكل في الأداء بسبب إنشاء دوال جديدة في كل إعادة عرض، إذ لا يوجد DOM افتراضي لتحسين العملية.
