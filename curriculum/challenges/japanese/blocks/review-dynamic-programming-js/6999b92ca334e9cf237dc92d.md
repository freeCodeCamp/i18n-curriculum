## 動的計画法の紹介

- **定義**：動的計画法は、複雑な問題をより簡単な部分問題に分解し、その結果を保存して重複した計算を避けるアルゴリズムの手法です。 
- **重複部分問題**：大きな問題を解く際に同じ小さな問題が何度も現れます。これらの部分問題を繰り返し計算する代わりに、その解を保存します。
- **最適部分構造**：問題の最適解は、その部分問題の最適解を含みます。つまり、小さな部分の最適解を組み合わせて最良の解を作り上げることができます。

## 動的計画法の解法

- **メモ化（トップダウンアプローチ）**：メモ化は高コストな関数呼び出しの結果を保存し、同じ入力が再度現れたときにキャッシュされた結果を返します。

```js
function climbStairsMemo(n, memo = {}) {
  // If already calculated, return cached value
  if (memo[n] !== undefined) {
    return memo[n];
  }

  // Base cases: 1 way for 1 step, 2 ways for 2 steps
  if (n <= 2) {
    return n;
  }

  // Calculate once and store in memo for future use
  memo[n] =
    climbStairsMemo(n - 1, memo) +
    climbStairsMemo(n - 2, memo);

  return memo[n];
}
```

- **タビュレーション（ボトムアップアプローチ）**：タビュレーションは下から解を構築し、部分問題の解をテーブルに埋めていきます。

```js
function climbStairsTabulation(n) {
  if (n <= 2) {
    return n;
  }

  // Create an array to store solutions from 0 to n
  const dp = new Array(n + 1).fill(0);

  // Base cases
  dp[1] = 1; // 1 way to reach step 1
  dp[2] = 2; // 2 ways to reach step 2

  // Build up the solution iteratively
  for (let i = 3; i <= n; i++) {
    // Ways to reach step i = ways to reach (i-1) + ways to reach (i-2)
    dp[i] = dp[i - 1] + dp[i - 2];
  }

  return dp[n];
}
```

## 動的計画法を使った実世界の応用

- **ルート最適化**：GPSシステムは動的計画法のアルゴリズムを使って場所間の最短経路を見つけます。
- **テキスト処理**：スペルチェッカーやオートコンプリート機能は単語間の編集距離を計算するために動的計画法をよく利用します。
- **金融モデリング**：投資戦略やポートフォリオ最適化には動的計画法の手法が頻繁に使われます。
- **資源配分**：ナップサック問題やその変種はスケジューリング、予算管理、資源管理に現れます。

## 動的計画法を使うべき場合

次のような場合に動的計画法の利用を検討してください：

- 問題が重複する部分問題に分解できる。
- 問題に最適部分構造がある。
- 単純な再帰的解法では計算が繰り返される。
- 時間計算量を優先し、空間計算量を犠牲にしてもよい。
