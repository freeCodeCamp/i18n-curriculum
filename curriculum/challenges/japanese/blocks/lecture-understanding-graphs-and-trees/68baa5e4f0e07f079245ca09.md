# --description--

グラフは、ノード（頂点とも呼ばれます）とそれらをつなぐエッジの集合で構成される非常に強力なデータ構造です。

グラフをコードで実装する一般的な方法は2つあります：

* 隣接行列
    
* 隣接リスト
    

これらについて詳しく見ていき、それぞれの利点と制限を説明します。

## 隣接行列

まずは隣接行列から始めましょう。

隣接行列は、行と列がグラフの頂点を表す二次元のリストです。

行列の値はノード間のエッジや接続を表します。

例えば、`matrix`という変数に格納された行列がある場合、`matrix[i][j]`に格納された値は、`i`が行、`j`が列を表し、ノード`i`とノード`j`の間のエッジや接続を表します。

値は、グラフが重み付きか非重み付きかによって意味が異なります：

* グラフが非重み付きの場合、`1`の値はこれらのノード間にエッジがあることを意味し、`0`の値はエッジがないことを意味します。
    
* グラフが重み付きの場合、その値はノードをつなぐエッジの重みを表します。
    

隣接行列を使う大きな利点の一つは、2つのノード間にエッジがあるかどうかを定数時間`O(1)`で確認できることです。これは、プログラムがその特定の値を二次元リストの中から見つけるだけで済むためです。

しかし、このエッジの検出の効率性にはトレードオフがあります。隣接行列は、グラフのノード数`O(V²)`に対して二次の空間を必要とするため、空間効率が悪いです`V`。

これは、エッジが少ないスパースグラフに対して非効率的です。なぜなら、スパースグラフの場合、ノード間にエッジがないことを表す多くの0を行列に保存することになり、これらの0もメモリ上のスペースを消費するからです。

また、隣接行列はノードの隣接ノードを見つけるのにも非効率的です。プログラムはエッジを表す0や1を見つけるために行または列全体を反復する必要があり、最悪の場合この処理はグラフのノード数Vに対してO(V)の時間がかかります。

この特定のグラフの隣接行列の例を見てみましょう：

<img src="https://cdn.freecodecamp.org/curriculum/lecture-transcripts/how-do-matrices-and-adjacency-lists-work-1.png" alt="A graph with four nodes labeled A, B, C, and D. Node A connects to B, C, and D. Node B connects to A and D. Node C connects to A. Node D connects to A and B.">

隣接行列では：

* 各行はノードを表します。最初の行はノードA、2行目はノードB、以下同様です。
    
* 各列もノードを表します。
    
* 行列の各値は、各ノードのペア間にエッジがあるかどうかを表します。値が0の場合はエッジがなく、1の場合はエッジがあります。
    

対角線上の値は、各ノードが自己ループ（ノード自身に接続するエッジ）を持つかどうかを表します。例では、自己ループがないためすべて0です。

これは、行と列が対応するノードをどのように表すかを示す隣接行列の視覚的な表現です。

例えば、最初の行は`[0, 1, 1, 1]`です。なぜならノードAはノードB、C、Dに接続するエッジを持っているからです：

```markdown
#      A  B  C  D
# A   [0, 1, 1, 1],
# B   [1, 0, 0, 1],
# C   [1, 0, 0, 0],
# D   [1, 1, 0, 0]
```

そしてこれは同じ隣接行列をPythonコードで実装したものです：

```python
adjacency_matrix = [
    [0, 1, 1, 1],  # The neighbors of A are B, C, and D
    [1, 0, 0, 1],  # The neighbors of B are A and D
    [1, 0, 0, 0],  # The only neighbor of C is A
    [1, 1, 0, 0]   # The neighbors of D are A and B
]
```

## 隣接リスト

グラフを表すもう一つの一般的な方法は隣接リストを使うことです。

隣接リストは、各ノードの隣接ノードすべてを保存する配列または辞書です。

隣接リストの実装方法は2つあります：

* 配列として、各インデックスがノードを表し、そのインデックスに保存されたリストが隣接ノードを含む方法。
    
* 辞書として、各キーがノードを表し、そのキーに対応する値（リスト）が隣接ノードを含む方法。
    

隣接リストは隣接行列よりも空間効率が良く、空間計算量は`O(V + E)`で、`V`は頂点（ノード）の数、`E`はエッジの数です。

また、隣接ノードをすべて見つける操作も効率的です。この操作はノードに対応するリストにアクセスするだけで済むからです。

しかし、ここにもトレードオフがあります。

隣接リストは、2つのノード間にエッジがあるかどうかを判定する際、隣接行列より効率が悪いです。

最悪の場合、ノードがグラフ内のすべての他のノードに接続されている場合、非常に長い隣接ノードのリストを検索する必要があり、検索に`O(V)`の時間がかかる場合があります。

このグラフの隣接リストの例を示します：

<img src="https://cdn.freecodecamp.org/curriculum/lecture-transcripts/how-do-matrices-and-adjacency-lists-work-1.png" alt="A graph with four nodes labeled A, B, C, and D. Node A connects to B, C, and D. Node B connects to A and D. Node C connects to A. Node D connects to A and B. It's the same image as before.">

この隣接リストは辞書として実装されています。辞書の各キーはノードを表し、そのキーに対応する値は対応するノードのすべての隣接ノードのリストです：

```python
adjacency_list = {
    'A': ['B', 'C', 'D'],
    'B': ['A', 'D'],
    'C': ['A'],
    'D': ['A', 'B']
}
```

あるいは、2次元リストとして実装することもできます。各インデックスがノードを表します。例えば、インデックス0はノードA、インデックス1はノードB、以下同様です：

```python
adjacency_list = [
    ['B', 'C', 'D'],  # Neighbors of A (index 0)
    ['A', 'D'],       # Neighbors of B (index 1)
    ['A'],            # Neighbors of C (index 2)
    ['A', 'B']        # Neighbors of D (index 3)
]
```

この2次元リストは隣接行列に似ているように見えますが、実際はかなり異なります。

* 隣接行列は、グラフ内のエッジやエッジの重みを表す0、1、または他の値を保存します。
    
* 隣接リストは、各ノードの実際の隣接ノードのリストを保存します。
    

これは非常に重要な違いであり、よく理解しておくべきです。

隣接行列と隣接リストの両方はグラフを実装する上で非常に重要です。どちらを選ぶかはグラフのサイズやデータの使い方によります。隣接行列はエッジが多い密なグラフに適しており、隣接リストはスパースグラフが多い現実のシナリオで一般的に好まれます。
