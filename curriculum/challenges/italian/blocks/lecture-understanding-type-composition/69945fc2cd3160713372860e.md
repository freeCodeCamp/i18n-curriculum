# --description--

Nella lezione precedente, hai imparato a lavorare con i tipi union. Un modo comune per lavorare con i tipi union è usare un alias di tipo.

Un alias di tipo è un nome per un tipo. Ecco un esempio di come assegnare un nome a un tipo union:

```ts
type ID = number | string;
```

Spesso nei tuoi progetti creerai alias di tipo e li userai in tutto il programma in questo modo:

```ts
type ID = number | string;

type User = {
  id: ID;
  name: string;
};

function getUserId(user: User): ID {
  return user.id;
}
```

Questo esempio usa due tipi chiamati `ID` e `User`. `ID` sarebbe un tipo union per `number` o `string` mentre `User` è un tipo oggetto con le proprietà `id` e `name`. La `id` è specificata con il tipo `ID` mentre la `name` ha il tipo `string`.

La funzione `getUserId` ha un parametro chiamato `user` e il suo tipo è `User`. Il tipo di ritorno della funzione `getUserId` sarebbe `ID`.

Gli alias di tipo sono un ottimo modo per descrivere la forma di un oggetto. È comune creare tipi oggetto e usarli in array in questo modo:

```ts
type User = {
  id: number | string;
  name: string;
};

const users: User[] = [
  { id: 101, name: "Alice" },
  { id: "A102", name: "Bob" },
  { id: 103, name: "Charlie" }
];
```

In questo esempio, l’array `users` deve essere riempito solo con oggetti `User`. Se dimentichi una proprietà in uno degli oggetti, TypeScript lancerà un errore come questo:

```ts
type User = {
  id: number | string;
  name: string;
};

const users: User[] = [
  // Property 'name' is missing in type '{ id: number; }' but required in type 'User'.
  { id: 101 },
  { id: "A102", name: "Bob" },
  { id: 103, name: "Charlie" }
];
```

Avrai notato l’uso del PascalCase quando si tratta di nominare gli alias di tipo. Questo perché, per convenzione, usare PascalCase aiuta a distinguere i tipi dalle variabili. Aiuta anche la leggibilità. Quando vedi `User[]`, saprai che è un array di oggetti di tipo `User`.
