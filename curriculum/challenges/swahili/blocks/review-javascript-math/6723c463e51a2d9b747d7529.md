## Kufanya kazi na aina ya data ya Number

- **Ufafanuzi**: Aina ya `Number` ya JavaScript inajumuisha nambari kamili, nambari za desimali, `Infinity` na `NaN`. Nambari za desimali ni zile zenye sehemu ya desimali. `Infinity` chanya ni nambari iliyo kubwa kuliko nambari nyingine yoyote ilhali `-Infinity` ni nambari ndogo kuliko nyingine yoyote. `NaN` (`Not a Number`) inawakilisha thamani ya nambari isiyo sahihi kama mfuatano wa herufi `"Jessica"`.

## Operesheni za kawaida za kihesabu

- **Kipengele cha kuongeza**: Kipengele hiki (`+`) hutumika kuhesabu jumla ya nambari mbili au zaidi.
- **Kipengele cha kutoa**: Kipengele hiki (`-`) hutumika kuhesabu tofauti kati ya nambari mbili.
- **Kipengele cha kuzidisha**: Kipengele hiki (`*`) hutumika kuhesabu bidhaa ya nambari mbili au zaidi.
- **Kipengele cha kugawanya**: Kipengele hiki (`/`) hutumika kuhesabu gawio kati ya nambari mbili.
- **Mgawanyiko kwa sifuri**: Ukijaribu kugawanya kwa sifuri, JavaScript itarudisha `Infinity`.
- **Kipengele cha mabaki**: Kipengele hiki (`%`) hurudisha mabaki ya mgawanyiko.
- **Kipengele cha nguvu**: Kipengele hiki (`**`) huinua nambari moja kwa nguvu ya nyingine.

## Mahesabu kwa kutumia nambari na mfuatano wa herufi

- **Maelezo**: Unapotumia kipengele cha `+` kwa nambari na mfuatano wa herufi, JavaScript itabadilisha nambari kuwa mfuatano wa herufi na kuunganisha thamani hizo mbili. Unapotumia vipengele vya `-`, `*` au `/` kwa mfuatano wa herufi na nambari, JavaScript itabadilisha mfuatano wa herufi kuwa nambari na matokeo yatakuwa nambari. Kwa `null` na `undefined`, JavaScript hutumia `null` kama 0 na `undefined` kama `NaN` katika hesabu za kihesabu.

:::interactive_editor

```js
const result = 5 + '10';

console.log(result); // "510"
console.log(typeof result); // string

const subtractionResult = '10' - 5;
console.log(subtractionResult); // 5
console.log(typeof subtractionResult); // number

const multiplicationResult = '10' * 2;
console.log(multiplicationResult); // 20
console.log(typeof multiplicationResult); // number

const divisionResult = '20' / 2;
console.log(divisionResult); // 10
console.log(typeof divisionResult); // number

const result1 = null + 5;
console.log(result1); // 5
console.log(typeof result1); // number

const result2 = undefined + 5;
console.log(result2); // NaN
console.log(typeof result2); // number
```

:::

## Kipaumbele cha opereta

- **Maelezo**: Kipaumbele cha opereta kinaamua mpangilio wa kutekelezwa kwa operesheni katika usemi wa kihesabu. Opereta zenye kipaumbele cha juu huchanganuliwa kabla ya zile zenye kipaumbele cha chini. Thamani zilizo ndani ya mabano zitachanganuliwa kwanza na kuzidisha/kugawanya kuna kipaumbele cha juu kuliko kuongeza/kutoa. Ikiwa opereta zina kipaumbele sawa, basi JavaScript itatumia mwelekeo wa chaguo.

:::interactive_editor

```js
const result = (2 + 3) * 4;

console.log(result); // 20

const result2 = 10 - 2 + 3;

console.log(result2); // 11

const result3 = 2 ** 3 ** 2;

console.log(result3); // 512
```

:::

- **Maelezo**: Mwelekeo wa chaguo hutueleza mwelekeo wa kutekeleza usemi wakati opereta nyingi za aina moja zipo. Huu unaamua kama usemi utachanganuliwa kutoka kushoto kwenda kulia (`left-associative`) au kulia kwenda kushoto (`right-associative`). Kwa mfano, opereta ya nguvu pia ni ya mwelekeo wa kulia kwenda kushoto:

:::interactive_editor

```js
const result4 = 5 ** 4 ** 1; 

console.log(result4); // 625
```

:::

## Opereta za kuongeza na kupunguza

- **Opereta ya kuongeza**: Opereta hii hutumika kuongeza thamani kwa moja. Uandishi wa kiambishi awali `++num` huongeza thamani ya kigezo kwanza, kisha kurudisha thamani mpya. Uandishi wa kiambishi tamati `num++` hurudisha thamani ya sasa ya kigezo kwanza, kisha huongeza.

:::interactive_editor

```js
let x = 5;

console.log(++x); // 6
console.log(x); // 6


let y = 5;

console.log(y++); // 5
console.log(y); // 6
```

:::

- **Opereta ya kupunguza**: Opereta hii hutumika kupunguza thamani kwa moja. Uandishi wa kiambishi awali na kiambishi tamati hufanya kazi sawa na ilivyoelezwa kwenye opereta ya kuongeza.

:::interactive_editor

```js
let num = 5;

console.log(--num); // 4
console.log(num--); // 4
console.log(num); // 3
```

:::

## Opereta za mgawo mchanganyiko

- **Opereta ya mgawo wa kuongeza (`+=`)**: Opereta hii hufanya kuongeza kwenye thamani na kuweka matokeo kwenye kigezo.
- **Opereta ya mgawo wa kupunguza (`-=`)**: Opereta hii hufanya kupunguza kwenye thamani na kuweka matokeo kwenye kigezo.
- **Opereta ya mgawo wa kuzidisha (`*=`)**: Opereta hii hufanya kuzidisha kwenye thamani na kuweka matokeo kwenye kigezo.
- **Opereta ya mgawo wa kugawa (`/=`)**: Opereta hii hufanya kugawa kwenye thamani na kuweka matokeo kwenye kigezo.
- **Opereta ya mgawo wa mabaki (`%=`)**: Opereta hii hugawa kigezo kwa nambari iliyobainishwa na kuweka mabaki kwenye kigezo.
- **Opereta ya mgawo wa exponentiation (`**=`)**: Opereta hii huinua kigezo kwa nguvu ya nambari iliyobainishwa na kuweka matokeo mapya kwenye kigezo.

## Thamani ya kweli au siyo kweli na usawa

- **Ufafanuzi wa Boolean**: Thamani ya kweli au siyo kweli ni aina ya data inayoweza kuwa na thamani mbili tu: `true` au `false`. 
- **Opereta ya usawa (`==`)**: Opereta hii hutumia kulazimisha aina kabla ya kuangalia kama thamani ni sawa.

:::interactive_editor

```js
console.log(5 == '5'); // true
```

:::

- **Opereta ya usawa mkali (`===`)**: Opereta hii haifanyi kulazimisha aina na huangalia kama aina na thamani zote ni sawa.

:::interactive_editor

```js
console.log(5 === '5'); // false
```

:::

- **Opereta ya kutokufanana (`!=`)**: Opereta hii hutumia kulazimisha aina kabla ya kuangalia kama thamani si sawa.
- **Opereta ya kutokufanana mkali (`!==`)**: Opereta hii haitumii kulazimisha aina na huangalia kama aina na thamani zote si sawa.

## Opereta za kulinganisha

- **Opereta ya kubwa kuliko (`>`)**: Opereta hii huangalia kama thamani upande wa kushoto ni kubwa kuliko ile ya kulia.
- **Opereta ya kubwa kuliko (`>=`) au sawa**: Opereta hii huangalia kama thamani upande wa kushoto ni kubwa kuliko au sawa na ile ya kulia.
- **Opereta ya ndogo kuliko (`<`)**: Opereta hii huangalia kama thamani upande wa kushoto ni ndogo kuliko ile ya kulia.
- **Opereta ya ndogo kuliko (`<=`) au sawa**: Opereta hii huangalia kama thamani upande wa kushoto ni ndogo kuliko au sawa na ile ya kulia.

## Opereta za upande mmoja

- **Opereta ya kuongeza upande mmoja**: Opereta hii hubadilisha operand kuwa nambari. Ikiwa operand tayari ni nambari, hubaki vilevile.

:::interactive_editor

```js
const str = '42';
const num = +str;

console.log(num); // 42
console.log(typeof num); // number
```

:::

- **Opereta ya kukanusha upande mmoja (`-`)**: Opereta hii hubadilisha operand kuwa hasi.

:::interactive_editor

```js
const num = 4;
console.log(-num); // -4
```

:::

- **Opereta ya NOT ya kimantiki (`!`)**: Opereta hii hubadilisha thamani ya kweli au siyo kweli ya operand. Kwa hiyo, ikiwa operand ni `true`, inakuwa `false`, na ikiwa ni `false`, inakuwa `true`.

## Opereta za bitwise

- **Opereta ya Bitwise AND (`&`)**: Opereta hii inarudisha 1 kwenye kila nafasi ya biti ambayo biti za operand zote mbili ni 1. 
- **Opereta ya Bitwise AND Assignment (`&=`)**: Opereta hii hufanya operesheni ya `bitwise AND` na nambari iliyobainishwa na kuweka matokeo kwenye kigezo.
- **Opereta ya Bitwise OR (`|`)**: Opereta hii inarudisha 1 kwenye kila nafasi ya biti ambayo biti za operand yoyote au zote mbili ni 1. 
- **Opereta ya Bitwise OR Assignment (`|=`)**: Opereta hii hufanya operesheni ya `bitwise OR` na nambari iliyobainishwa na kuweka matokeo kwenye kigezo.
- **Opereta ya Bitwise XOR (`^`)**: Opereta hii inarudisha 1 kwenye kila nafasi ya biti ambayo biti za operand yoyote, lakini si zote mbili, ni 1. 
- **Opereta ya Bitwise NOT (`~`)**: Opereta hii hubadilisha uwakilishi wa binari wa nambari.
- **Opereta ya Left Shift (`<<`)**: Opereta hii husogeza biti zote kushoto kwa idadi maalum ya nafasi. 
- **Opereta ya Right Shift (`>>`)**: Opereta hii husogeza biti zote kulia.

## Kauli za masharti, thamani ya kweli, thamani ya uongo na opereta ya ternary

- **`if/else if/else`**: Kauli ya `if` huchukua sharti na kuendesha sehemu ya msimbo ikiwa sharti hilo ni `truthy`. Ikiwa sharti ni `false`, basi huenda kwenye sehemu ya `else if`. Ikiwa hakuna kati ya masharti hayo yaliyokuwa `true`, basi itatekeleza kifungu cha `else`. Thamani za `Truthy` ni thamani zozote zinazotoa `true` zinapotathminiwa katika muktadha wa Boolean kama kwenye kauli ya `if`. Thamani za `Falsy` ni thamani zinazotathminiwa kuwa `false` katika muktadha wa Boolean.

:::interactive_editor

```js
const score = 87;

if (score >= 90) {
 console.log('You got an A'); 
} else if (score >= 80) {
 console.log('You got a B'); // You got a B
} else if (score >= 70) {
 console.log('You got a C');
} else {
 console.log('You failed! You need to study more!');
}
```

:::

- **Opereta ya Ternary**: Opereta hii mara nyingi hutumika kama njia fupi ya kuandika kauli za `if else`.

:::interactive_editor

```js
const temperature = 30;
const weather = temperature > 25 ? 'sunny' : 'cool';

console.log(`It's a ${weather} day!`); // It's a sunny day!
```

:::

## Opereta za kimantiki za pande mbili

- **Opereta ya kimantiki AND (`&&`)**: Opereta hii huangalia kama vigezo vyote ni `true`. Ikiwa vyote ni `true`, basi itarudisha thamani ya pili. Ikiwa kigezo chochote ni `falsy`, basi itarudisha thamani ya `falsy`. Ikiwa vigezo vyote ni `falsy`, itarudisha thamani ya kwanza ya `falsy`.

:::interactive_editor

```js
const result = true && 'hello';

console.log(result); // hello
```

:::

- **Opereta ya kimantiki OR (`||`)**: Opereta hii huangalia kama angalau kimoja cha vigezo ni thamani ya kweli. 
- **Opereta ya kuunganisha thamani zisizo na maana (`??`)**: Opereta hii itarudisha thamani tu ikiwa ya kwanza ni `null` au `undefined`.

:::interactive_editor

```js
const userSettings = {
 theme: null,
 volume: 0,
 notifications: false,
};

let theme = userSettings.theme ?? 'light';
console.log(theme); // light
```

:::

## Kitu cha `Math`

- **Njia ya `Math.random()`**: Njia hii huzalisha nambari ya desimali ya nasibu kati ya 0 (ikijumuishwa) na 1 (isipokuwa 1). Hii inamaanisha matokeo yanaweza kuwa 0, lakini hayatakuwa 1 kamwe. 
- **Njia ya `Math.max()`**: Njia hii huchukua seti ya nambari na kurudisha thamani ya juu zaidi.
- **Njia ya `Math.min()`**: Njia hii huchukua seti ya nambari na kurudisha thamani ya chini zaidi.
- **Njia ya `Math.ceil()`**: Njia hii huzungusha thamani juu hadi nambari kamili iliyo karibu zaidi. 
- **Njia ya `Math.floor()`**: Njia hii huzungusha thamani chini hadi nambari kamili iliyo karibu zaidi. 
- **Njia ya `Math.round()`**: Njia hii huzungusha thamani hadi nambari kamili iliyo karibu zaidi.

:::interactive_editor

```js
console.log(Math.round(2.3)); // 2
console.log(Math.round(4.5)); // 5
console.log(Math.round(4.8)); // 5
```

:::

- **Njia ya `Math.trunc()`**: Njia hii huondoa sehemu ya desimali ya nambari, na kurudisha sehemu ya nambari kamili pekee, bila kuzungusha.
- **Njia ya `Math.sqrt()`**: Njia hii itarudisha mzizi wa mraba wa nambari.
- **Njia ya `Math.cbrt()`**: Njia hii itarudisha mzizi wa tatu wa nambari.
- **Njia ya `Math.abs()`**: Njia hii itarudisha thamani kamili ya nambari.
- **Njia ya `Math.pow()`**: Njia hii huchukua nambari mbili na huinua ya kwanza kwa nguvu ya ya pili.

## Njia za kawaida za nambari

- **`isNaN()`**: `NaN` inasimama kwa "Not-a-Number" (Siyo Nambari). Hii ni thamani maalum inayowakilisha matokeo ya kihesabu yasiyowezekana au yasiyofafanuliwa. Sifa ya kitendakazi `isNaN()` hutumika kubaini kama thamani ni `NaN` au la. `Number.isNaN()` hutoa njia ya kuaminika zaidi ya kukagua thamani za `NaN`, hasa pale ambapo kulazimisha aina kunaweza kusababisha matokeo yasiyotegemewa na kitendakazi cha kimataifa `isNaN()`.

:::interactive_editor

```js
console.log(isNaN(NaN));       // true
console.log(isNaN(undefined)); // true
console.log(isNaN({}));        // true

console.log(isNaN(true));      // false
console.log(isNaN(null));      // false
console.log(isNaN(37));        // false


console.log(Number.isNaN(NaN));        // true
console.log(Number.isNaN(Number.NaN)); // true
console.log(Number.isNaN(0 / 0));      // true

console.log(Number.isNaN("NaN"));      // false
console.log(Number.isNaN(undefined));  // false
```

:::

- **Njia ya `parseFloat()`**: Njia hii inachanganua hoja ya mfuatano wa herufi na kurudisha nambari ya desimali. Imeundwa kuchukua nambari mwanzoni mwa mfuatano wa herufi, hata kama kuna herufi zisizo za nambari baadaye kwenye mfuatano huo.
- **Njia ya `parseInt()`**: Njia hii inachanganua hoja ya mfuatano wa herufi na kurudisha nambari kamili. `parseInt()` inasimamisha uchanganuzi inapokutana na herufi ya kwanza isiyo nambari. Kwa nambari za desimali, inarudisha sehemu ya nambari kamili pekee. Ikiwa haiwezi kupata nambari kamili halali mwanzoni mwa mfuatano, inarudisha `NaN`.
- **Njia ya `toFixed()`**: Njia hii inaitwa kwenye nambari na inachukua hoja moja ya hiari, ambayo ni idadi ya tarakimu zinazotakiwa kuonekana baada ya nukta ya desimali. Inarudisha mwakilishi wa mfuatano wa herufi wa nambari hiyo ukiwa na idadi maalum ya sehemu za desimali.
