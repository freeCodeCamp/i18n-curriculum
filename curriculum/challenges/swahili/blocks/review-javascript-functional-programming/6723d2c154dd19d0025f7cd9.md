# --interactive--

## Kitendakazi safi dhidi ya kisichosafi

- Kitendakazi safi ni kile kinachotoa matokeo sawa kila wakati kwa ingizo sawa na hakina athari za pembeni. Matokeo yake hutegemea tu ingizo lake, na hakibadilishi hali yoyote ya nje.
- Vitendakazi visivyo safi vina athari za pembeni, ambazo ni mabadiliko ya hali ya programu yanayoonekana nje ya kitendakazi.

## Programu utendaji

- Programu utendaji ni mbinu ya ukuzaji wa programu inayosisitiza matumizi ya vitendakazi kutatua matatizo, ikizingatia kile kinachotakiwa kufanywa badala ya jinsi ya kufanikisha.
- Programu utendaji inahimiza matumizi ya mbinu zinazosaidia kuepuka athari za pembeni, kama vile kutumia miundo ya data isiyobadilika na vitendakazi vya daraja la juu.
- Ikiwa itatumiwa kwa usahihi, kanuni za programu utendaji huleta msimbo safi zaidi na rahisi kudumishwa.

## Kuringa

- Kuringa ni mbinu ya programu utendaji inayobadilisha kitendakazi chenye hoja nyingi kuwa mfululizo wa vitendakazi, kila kikiweka hoja moja.

Hapa kuna mfano wa kitendakazi cha kawaida dhidi ya kitendakazi kilichokuringwa:

:::interactive_editor

```js
// Regular function
function average(a, b, c) {
  return (a + b + c) / 3;
}

console.log(average(2, 3, 4)); // 3

// Curried function
function curriedAverage(a) {
  return function(b) {
    return function(c) {
      return (a + b + c) / 3;
    };
  };
}

console.log(curriedAverage(2)(3)(4)); // 3
```

:::

- Kuringa inaweza kuwa na nguvu hasa unapotumia vitendakazi vinavyopokea hoja nyingi.
- Kuringa huifanya msimbo wako kuwa rahisi kubadilika na rahisi kutumika tena.
- Unaweza kutumia vitendakazi vya mshale kuunda vitendakazi vilivyokuringwa kwa ufupi zaidi:

```js
const curriedAverage = a => b => c => (a + b + c) / 3;
```

- Ingawa kuringa inaweza kusababisha msimbo kuwa rahisi kubadilika na kutumika tena, pia inaweza kufanya msimbo kuwa mgumu kusoma ikiwa itatumika kupita kiasi.
