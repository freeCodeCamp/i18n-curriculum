# --description--

在 Express 中，路由参数是嵌入在 URL 路径中的动态值。它们让你的应用能够捕捉每次请求中可能变化的 URL 部分。这对于创建动态页面非常有用，比如根据 URL 显示特定用户、博客文章或产品。

下面是一个示例：

```javascript
// Route with a parameter in the path
app.get("/post/:postId", (req, res) => {
  const postId = req.params.postId;
  res.send(`Viewing post with ID: ${postId}`);
});
```

这里，路由监听对 `/post/:postId` 的 GET 请求。

* `:postId` 是一个路由参数，是 URL 中的占位符，会被实际值替换。例如，对 `/post/42` 的请求匹配此路由，`42` 就成为 `postId` 的值。
    
* 你可以使用 `req.params` 访问路由参数。`req.params` 对象包含所有路由参数，在这个例子中，`req.params.postId` 包含来自 URL 的值 `42`。
    

最后，`res.send()` 将响应发送回客户端。所以如果 URL 是 `/post/42`，客户端会看到 `Viewing post with ID: 42`。

如你所见，路由参数让你的路由灵活且动态，无需为每个可能的值创建单独的路由。

现在你已经了解了路由参数的工作原理，让我们回到上一课的一个示例，看看如何为不同类型的菜单使用动态路由：

```javascript
app.get("/menu/:category", (req, res) => {
  const category = req.params.category;
  res.send(`You are viewing the ${category} menu!`);
});
```

下面是这个过程的解析：

1. 用户访问 [`http://localhost:3000/menu/drinks`](http://localhost:3000/menu/drinks)。
    
2. 服务器将此请求匹配到路由 `/menu/:category`，并将 `category` 参数捕捉为 `drinks`。
    
3. 服务器将此响应发送回浏览器：`You are viewing the drinks menu!`
    

类似地，如果用户访问 [`http://localhost:3000/menu/food`](http://localhost:3000/menu/food)，服务器会响应以下内容：`You are viewing the food menu!`

路由参数让你可以根据 URL 显示动态内容。你无需为每个类别创建单独路由，只需通过更改 URL，使用单一路由处理饮料、食物、甜点等多个类别。
