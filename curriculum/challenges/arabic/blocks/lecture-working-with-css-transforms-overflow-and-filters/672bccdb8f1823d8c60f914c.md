# --interactive--

CSS reset هو ورقة أنماط تزيل كل أو بعض التنسيقات الافتراضية التي يطبقها متصفحو الويب على عناصر HTML. على سبيل المثال، قد تكون لاحظت أن بعض العناصر، مثل الفقرات والعناوين، لها هوامش افتراضية حتى لو لم تحددها صراحة في أوراق الأنماط المخصصة الخاصة بك.

ستلاحظ هذا أيضًا مع خصائص مختلفة في مجموعة واسعة من عناصر HTML. قد يطبق المتصفحات المختلفة أنماطًا افتراضية مختلفة على عناصر HTML. الأنماط الافتراضية في Google Chrome قد لا تكون نفسها تمامًا كما في Mozilla Firefox أو Microsoft Edge. هذا قد يؤدي إلى أنماط غير متناسقة عبر المتصفحات، ويجب عليك تجنب ذلك قدر الإمكان.

للتعامل مع هذا، يزيل CSS reset الأنماط الافتراضية. بإزالة كل أو بعض الأنماط الافتراضية، يمكنك الحصول على قاعدة ثابتة متناسقة لتصميمك وتقليل التباينات المحتملة عبر المتصفحات والأجهزة. إزالة الأنماط الافتراضية تجعل عملية التنسيق أسهل أيضًا لأنك سترى فقط الأنماط التي طبقتها صراحة في أوراق الأنماط المخصصة. هناك نهجان رئيسيان لـ CSS resets: إما تعريف CSS resets مخصصة أو استخدام CSS resets من طرف ثالث.

CSS resets المخصصة هي أوراق أنماط تنشئها من الصفر لتناسب احتياجات مشروعك. بهذه الطريقة يمكنك التحكم في الأنماط المحددة التي ستتم إعادة تعيينها مع مرونة كبيرة. مع ذلك، تحتاج إلى استثمار وقت لتطوير وصيانة أوراق الأنماط. إليك مثالًا على قاعدة CSS شائعة جدًا لإعادة تعيين الهامش والحشو لجميع عناصر HTML:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<h1>Example Heading</h1>
<p>This is a paragraph.</p>
```

```css
* {
  margin: 0;
  padding: 0;
}
```

:::

عادةً ما تُكتب في أعلى ورقة أنماط CSS. محدد النجمة هو محدد شامل يطابق كل عناصر HTML، لذا سيكون لها هامش افتراضي صفر وحشو افتراضي صفر على جميع الجوانب الأربعة. هذا يمنحك نقطة بداية ثم يمكنك تخصيصها في محددات CSS أكثر تحديدًا في أسفل ورقة الأنماط.

يمكنك استخدام هذا النهج لاختيار أي عنصر HTML وإعادة تعيين أنماطه الافتراضية. فقط أنشئ ورقة أنماط مخصصة واستخدم محددات CSS المناسبة لمطابقة العناصر وتعيين الأنماط.

لكن هذا قد يكون عملية تستغرق وقتًا طويلاً. إذا أردت توفير الوقت، يمكنك أيضًا استخدام CSS reset من طرف ثالث. هذه أوراق أنماط مُعدة مسبقًا، لذا يمكنك فقط تنزيلها وإضافتها إلى مشروعك مباشرة. مثال رائع على CSS reset من طرف ثالث هو `Normalize.css`. هذه الورقة تطبع أنماطًا لمجموعة واسعة من عناصر HTML مع الحفاظ على بعض الأنماط الافتراضية المفيدة، خاصة تلك المهمة لإمكانية الوصول. كما تصحح الأخطاء الشائعة وتباينات الأنماط.

خيار آخر هو `sanitize.css`. هذه مكتبة CSS يمكنك استخدامها لضمان أن الأنماط الافتراضية ستكون متناسقة عبر جميع المتصفحات الحديثة الرئيسية. تم تطوير هذه المكتبة جنبًا إلى جنب مع `Normalize.css`، لذا تتطوران معًا. كما تحتوي على أوراق أنماط فردية يمكنك تنزيلها لأغراض محددة مثل تطبيع النماذج والطباعة.

هناك العديد من الخيارات المتاحة، لكن يجب أن تختار الأنسب لاحتياجات مشروعك. يمكنك أيضًا دمج النهجين باستخدام CSS resets من طرف ثالث مع resets مخصصة.

عند العمل مع CSS resets، من المهم أيضًا مراعاة إمكانية الوصول. يجب أن يكون تطبيق الويب الخاص بك متاحًا للجميع. لذا، لا يجب إعادة تعيين الأنماط التي قد تكون مفيدة لقارئ الشاشة أو تقنيات المساعدة الأخرى. يجب أيضًا أن تأخذ في الاعتبار تأثير هذه الأوراق الإضافية على أداء تطبيقاتك، لأنها يجب أن تُحمّل قبل تطبيق الأنماط المخصصة.

بإزالة الأنماط الافتراضية، تمنحك CSS resets نقطة بداية فارغة لتنفيذ تصميمك. هذا يؤدي إلى تجربة مستخدم أكثر اتساقًا وتناسقًا عبر المتصفحات والأجهزة.
