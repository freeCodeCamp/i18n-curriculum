# --description--

当你用 JavaScript 构建项目时，你经常需要额外的工具来帮助你完成任务。即使是简单的应用程序也可能需要诸如保存数据、登录、路由以在页面之间切换等特色。

你可以使用其他开发者已经构建并共享的开源包，而不是从头编写所有这些代码。

这就是 `npm` 派上用场的地方。`npm` 是一个庞大的开源软件在线注册表。把它想象成代码的应用商店——你可以查找你需要的内容，安装它，并开始在你的项目中使用它。

许多人认为 npm 代表 "Node Package Manager"，但这不是它的官方含义。在较早的 npm 文档中，它被描述为“`npm is not an acronym`”的递归的缩略语。换句话说，npm 官方上并不代表任何东西。

今天，npm 参考三个相关部分：

* npmjs.com 网站，你可以在注册表中查找 `package`。
    
* npm 注册表是一个大型公共数据库，保存所有的包。
    
* 随 Node.js 一起安装的 npm CLI 工具，允许你安装和管理包。
    

例如，如果你需要管理状态、处理日期、发送电子邮件或在你的项目中发出超文本传输协议请求，可能已经有一个 npm 包可以完成这些工作。通过一个简单的 `npm install` 命令，你可以开始使用这个现成的软件，而不必自己完成所有工作。

npm 还帮助你管理你的项目依赖。这意味着它会跟踪你在项目中使用的包，以及每个包的版本。它通过 `package.json` 和 `package-lock.json` 文件来实现这一点，你将在后续课程中学习这些内容。

Yarn、PNPM 和 Bun 是一些随着时间推移变得流行的 npm 备选。虽然它们各自在安装速度和存储效率等特定领域提供优势，但 npm 仍然是最广泛使用的。

在接下来的课程中，你将学习各种 npm 命令，如何安装包，关于 `package.json` 和 `package-lock.json` 文件，甚至如何将你自己的包发布到 npm 注册表。
