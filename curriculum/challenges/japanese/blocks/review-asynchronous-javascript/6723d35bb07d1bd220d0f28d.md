# --description--

- **同期的 JavaScript** は順番に実行され、前の操作が終わるまで次に進みません。
- **非同期的 JavaScript** はメインスレッドをブロックせずに複数の操作をバックグラウンドで実行できます。
- **スレッド** はメインプログラムの流れとは独立して実行できる命令の連続です。
- **コールバック関数** は他の関数に引数として渡され、操作完了後やイベント発生時に実行される関数です。

## JavaScript エンジンと JavaScript ランタイム

- **JavaScriptエンジン**はウェブブラウザでJavaScriptコードを実行するプログラムです。コードを受け取り、コンピュータが理解して動作できる命令に変換します。
- V8はGoogleが開発したJavaScriptエンジンの例です。
- **JavaScriptランタイム**はJavaScriptコードが実行される環境で、コードを処理・実行するJavaScriptエンジンと、ウェブブラウザやNode.jsのような追加機能を含みます。

## Fetch API

- Fetch API はウェブアプリがサーバーにネットワークリクエストを送るための仕組みで、データの取得や送信に使われます。`fetch()` メソッドを使ってリクエストを行います。
- テキスト、画像、音声、JSON など様々なデータを Fetch API で取得できます。

## Fetch API の HTTP メソッド

Fetch API はサーバーとやり取りするために様々な HTTP メソッドをサポートしています。一般的なメソッドは以下の通りです。

- **GET**: サーバーからデータを取得するために使います。デフォルトでは Fetch API は `GET` メソッドを使ってデータを取得します。

```js
fetch('https://api.example.com/data')
```

取得したデータを使うには `.json()` メソッドで JSON 形式に変換する必要があります。

```js
fetch('https://api.example.com/data')
  .then(response => response.json())
  .then(data => console.log(data))
```

このコードでは、`fetch()` はPromiseを返し、`.then()` ハンドラが解決されたレスポンスをJSON形式に変換しています。

- **POST**: サーバーにデータを送信し、新しいリソースを作成するために使います。`POST` メソッドを使います。

```js
fetch('https://api.example.com/users', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({
    name: 'John Doe',
    email: 'john@example.com'
  })
})
```

この例では新しいユーザーを作成するために `POST` リクエストを送っています。メソッドは `POST` に設定し、適切なヘッダーを指定し、送信したいデータをボディに含めています。ボディは文字列である必要があるため、`JSON.stringify()` でオブジェクトを JSON 文字列に変換しています。

- **PUT**: サーバー上の既存データを更新するために使います。`PUT` メソッドを使います。

```js
fetch('https://api.example.com/users/45', {
  method: 'PUT',
  headers: {
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({
    name: 'John Smith',
    email: 'john@example.com'
  })
})
```

この例では URL の末尾に指定された ID `45` のデータを更新しています。コードでは `PUT` メソッドを使い、更新するデータをボディに指定しています。

- **DELETE**: サーバー上のデータを削除するために使います。`DELETE` メソッドを使います。

```js
fetch('https://api.example.com/users/45', {
  method: 'DELETE'
})
```

この例では ID `45` のユーザーを削除するために `DELETE` リクエストを送っています。

## Promise と Promise チェーン

- **Promise**は非同期操作の最終的な完了または失敗と、その結果の値を表すオブジェクトです。Promiseの値は非同期操作が完了したときにのみわかります。
- シンプルなPromiseを作成する例はこちらです：

```js
const promise = new Promise((resolve, reject) => {
  setTimeout(() => {
    resolve('Data received successfully');
  }, 2000);
});
```

- Promise では `.then()` メソッドで成功時の処理を指定し、`.catch()` でエラー処理を行います。
- 以下は Promise で `.then()` と `.catch()` を使う例です。

```js
promise
  .then(data => {
    console.log(data);
  })
  .catch(error => {
    console.error(error);
  });
```

上の例では `.then()` メソッドで Promise から受け取ったデータをログに出力し、`.catch()` メソッドでエラーをログに出力しています。

- **Promise チェーン**: Promise の強力な機能の一つは複数の非同期操作を連鎖させられることです。各 `.then()` は新しい Promise を返せるため、一連の非同期操作を順番に実行できます。
- 以下は Promise チェーンの例です。

```js
fetch('https://api.example.com/data')
  .then(response => response.json())
  .then(data => {
    console.log(data);
    return fetch('https://api.example.com/other-data');
  })
  .then(response => response.json())
  .then(otherData => {
    console.log(otherData);
  })
  .catch(error => {
    console.error(error);
  });
```

上の例では最初にある URL からデータを取得し、次に最初のレスポンスに基づいて別の URL からデータを取得し、最後に2つ目のデータをログに出力しています。

`catch` メソッドは処理中に発生したエラーを処理します。これにより各ステップでエラー処理を書く必要がなくなり、コードが大幅に簡潔になります。

## `async/await` を使った Promise の扱い方

`async`/`await` はPromiseの上に構築されており、非同期コードの記述と読み取りを簡単にします。

- **async**: `async` キーワードは非同期関数を定義するために使います。`async` 関数は Promise を返し、その Promise は `async` 関数が返す値で解決されます。
- **await**: `await` キーワードは `async` 関数内で使い、Promise が解決されるまで関数の実行を一時停止します。`async` 関数内でのみ使えます。
- 以下は `async/await` を使った例です。

```js
async function delayedGreeting(name) {
  console.log("A Messenger entered the chat...");
  await new Promise(resolve => setTimeout(resolve, 2000));
  console.log(`Hello, ${name}!`);
}

delayedGreeting("Alice");
console.log("First Printed Message!");
```

上の例では `delayedGreeting` 関数が `async` 関数で、2秒間待ってから挨拶メッセージを出力します。`await` キーワードは `Promise` が解決されるまで関数の実行を一時停止します。

- `async/await` の最大の利点の一つは `try/catch` ブロックによるエラー処理です。以下は例です。

```js
async function fetchData() {
  try {
    const response = await fetch('https://api.example.com/data');
    const data = await response.json();
    console.log(data);
  } catch (error) {
    console.error(error);
  }
}

fetchData();
```

上の例では `try` ブロックにエラーが発生する可能性のあるコードがあり、`catch` ブロックがエラーを処理します。これによりエラー処理がより簡潔で読みやすくなります。

## `async` 属性

- `async` 属性はブラウザにスクリプトファイルを非同期にダウンロードしながらHTMLドキュメントの解析を続けるよう指示します。
- スクリプトがダウンロードされるとHTML解析は一時停止し、スクリプトが実行され、その後HTML解析が再開されます。
- 実行順序が重要でない独立したスクリプトには`async` を使うべきです。

## `defer` 属性

- `defer` 属性もスクリプトを非同期にダウンロードしますが、HTMLドキュメントの解析が完全に終わるまでスクリプトの実行を遅延させます。
- `defer` スクリプトはHTMLドキュメントに現れる順序で実行順序を維持します。

- `async` と `defer` 属性はインラインスクリプトには無視され、外部スクリプトファイルにのみ有効です。

- `async` と `defer` 属性が両方ある場合は `async` 属性が優先されます。

## Geolocation API

- Geolocation API はウェブサイトがユーザーの位置情報を取得する方法を提供します。

- 以下の例は、ユーザーの現在位置を取得するために使用されるAPIの`getCurrentPosition()`メソッドを示しています。

```js
navigator.geolocation.getCurrentPosition(
  (position) => {
    console.log("Latitude: " + position.coords.latitude);
    console.log("Longitude: " + position.coords.longitude);
  },
  (error) => {
    console.log("Error: " + error.message);
  }
);
```

このコードでは、`getCurrentPosition`を呼び出し、位置情報が正常に取得されたときに呼び出される関数を渡しています。

`position` オブジェクトには様々な情報が含まれていますが、ここでは `latitude` と `longitude` のみを選択しています。

`position` の取得に問題がある場合は、エラーがコンソールに出力されます。

- ユーザーのプライバシーを尊重し、必要な場合にのみ位置情報をリクエストすることが重要です。
