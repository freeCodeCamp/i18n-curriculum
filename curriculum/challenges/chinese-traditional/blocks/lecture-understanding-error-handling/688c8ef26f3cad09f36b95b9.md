# --description--

偵錯是任何 Python 開發者必備的技能。了解基礎技術可以幫助你高效地識別和修復問題。

偵錯是識別並解決你程式碼中錯誤或漏洞的處理程序。它涉及檢查程式碼、理解流程，並使用工具來定位問題的來源。

在本課程中，我們將介紹你可以在下一個 Python 專案中使用的常見偵錯技術。

## 使用列印函式（程式）和 f-strings

首先，在你的程式碼中於各個點使用 `print()` 函式和 f-string 可以幫助你了解流程和變數的狀態。例如：

```py
def add(a, b):
    result = a + b
    print(f'Adding {a} and {b} gives {result}')
    return result
```

透過列印 `a`、`b` 和 `result` 的值，你可以驗證該函式是否如預期般運作。

## 使用 `pdb` 模組進行互動式偵錯

接著，你可以利用 Python 內建的 `pdb` 模組進行互動式偵錯：

```py
import pdb

def divide(a, b):
    pdb.set_trace()
    return a / b

print(divide(10, 2))
```

透過使用 `.set_trace()` 方法設定追蹤點，你可以逐步執行程式碼、檢查變數，並了解程式的行為。

如果你執行上述程式碼，你會看到一些輸出，顯示你正在執行的檔案位置、你呼叫 `.set_trace()` 方法的行數以及其後的程式碼，還有一個互動式的 `pdb` 提示符：

```py
> /Users/fcc/Desktop/debugging.py(5)divide()
-> return a / b
(Pdb)
```

如果你在提示中輸入 help，你會看到你可以使用的指令列表：

```py
(Pdb) help

Documented commands (type help <topic>):
========================================
EOF    c          d        h         list      q        rv       undisplay
a      cl         debug    help      ll        quit     s        unt      
alias  clear      disable  ignore    longlist  r        source   until    
args   commands   display  interact  n         restart  step     up       
b      condition  down     j         next      return   tbreak   w        
break  cont       enable   jump      p         retval   u        whatis   
bt     continue   exit     l         pp        run      unalias  where    

Miscellaneous help topics:
==========================
exec  pdb
```

然後你可以使用這些指令來偵錯你的程式碼。

例如，如果你想查看當下整個程式碼中元素的型別，可以使用 `whatis` 指令：

```py
(Pdb) whatis a
<class 'int'>
(Pdb) whatis divide
Function divide
```

如你所見，當你執行 `.set_trace()` 時，參數 a 的型別是整數，而 divide 是一個函式（程式）。

然後，若要繼續執行你的程式碼，你可以使用 `continue` 指令，或其別名之一，`cont` 或 `c`：

```py
(Pdb) continue
5.0
```

## 整合開發環境偵錯工具

許多整合開發環境（`IDEs`）提供進階偵錯工具，例如中斷點、逐步執行和變數檢查。

## 使用 VS Code 除錯器

如果你使用 VS Code，你可以在你的程式碼中設定中斷點，並執行除錯器以在那些點暫停執行。以下是如何偵錯相同的 `divide` 函式（程式）：

**步驟 1：設定你的程式碼** 創建一個名為 `main.py` 的檔案，內容如下：

```py
def divide(a, b):
    result = a / b
    return result

print(divide(10, 2))
print(divide(15, 3))
```

**步驟 2：設定中斷點**

1. 在第 2 行（`result = a / b`）旁邊的邊距（左側邊緣）點擊以設定中斷點

2. 會出現一個紅點，表示已設定中斷點

**步驟 3：開始偵錯**

1. 按下 `F5` 或前往 Run > Start Debugging

2. 當系統提示時，選擇「Python 檔案」

3. 除錯器會在你的中斷點暫停執行

**步驟 4：檢查變數**

- 將滑鼠暫留在變數上以查看它們的當前值

- 使用左側的變數面板來查看所有區域的變數

- 使用底部的偵錯主控台來評估運算式

**步驟 5：逐步執行程式碼**

使用偵錯工具列來：

- **繼續 (F5)**：繼續執行直到下一個中斷點

- **Step Over (F10)**：執行目前行並移至下一行

- **Step Into (F11)**：進入函式（程式）呼叫

- **Step Out (Shift+F11)**：離開目前的函式（程式）

整合開發環境偵錯工具提供視覺化介面來檢查你的程式狀態，使你比單獨使用列印述語更容易識別和修正問題。

透過掌握這些基礎的偵錯技巧－使用 `print()` 述語、`pdb` 模組和整合開發環境工具－你可以有效地識別並決議你 Python 程式碼中的問題。每種技巧都有其適用場合：`print()` 述語用於快速檢查，`pdb` 用於互動式探索，整合開發環境除錯器用於視覺化檢查。
