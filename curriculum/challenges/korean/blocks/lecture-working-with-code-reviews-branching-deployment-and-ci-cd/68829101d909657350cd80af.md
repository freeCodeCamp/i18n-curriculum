# --description--

GitHub Pages는 정적 사이트나 백엔드 서버가 필요 없는 애플리케이션을 배포하는 데 정말 좋은 선택입니다. 즉, 클라이언트 측에서만 실행되거나 사용자의 브라우저에서 완전히 실행되는 애플리케이션을 이 플랫폼에서 완전히 배포할 수 있습니다.

또한, GitHub Pages는 오픈 소스 프로젝트에 대해 완전히 무료입니다. 프로젝트를 오픈 소스로 만드는 데는 여러 좋은 이유가 있으며, Pages는 그중 하나를 더해줍니다.

그런데 실제로 GitHub Pages에 어떻게 배포할까요? 음, 애플리케이션이 어떻게 작동하는지에 따라 다릅니다.

기본 HTML 페이지부터 시작해 보겠습니다. 원한다면 기존 저장소에 추가할 수 있는 예시가 있습니다:

```html
<!DOCTYPE html>
<html>
  <head>
    <title>freeCodeCamp</title>
    <link rel="stylesheet" href="./style.css" />
  </head>
  <body>
    <h1>Welcome to freeCodeCamp's Lessons!</h1>
    <p>Let's have some fun learning about Git and GitHub!</p>
  </body>
</html>
```

이제 저장소로 가서 "Settings" 탭을 클릭해 저장소 설정으로 이동하세요. 그런 다음 사이드바에서 "Pages"를 선택하세요.

처음에는 몇 가지 옵션만 있지만, 설정을 구성하면서 변경될 것입니다. 먼저 GitHub에 어디서 배포할지 알려야 합니다.

우리는 프레임워크를 사용하지 않고 GitHub 액션도 전혀 설정하지 않았기 때문에 "Deploy from a branch"로 두는 것이 좋습니다. 이 부분은 앞으로 수업에서 더 배우게 될 것입니다.

다음으로, `main` 브랜치를 배포 대상으로 설정하세요.

그다음 프로젝트의 루트 폴더에서 배포할지 `docs` 폴더에서 배포할지 선택할 수 있습니다. 이 기능은 freeCodeCamp 같은 프로젝트에서 사용됩니다 - 저희는 앱과 함께 문서를 묶어서 `docs` 폴더를 통해 문서만 GitHub Pages에 배포할 수 있었습니다. 지금은 `/` (루트) 폴더로 두고 이 드롭다운 옆의 "Save" 버튼을 눌러 변경 사항을 저장하세요.

저장하면 사용자 지정 도메인을 설정할 수 있는 새 입력란이 나타납니다. 이미 도메인이 있고 사용하고 싶다면 문서 링크를 따라 설정할 수 있습니다. 하지만 이번 수업에서는 이 부분을 비워 두겠습니다.

또한 새 GitHub Pages 사이트가 배포된 도메인을 볼 수 있습니다. 사용자 지정 도메인이 없으면 기본값은 `<username>.github.io`이며, 여기서 `<username>`는 계정 이름(또는 조직 소유 저장소에서 배포하는 경우 조직 계정 이름)입니다. 저장소는 저장소 이름과 일치하는 경로 `<username>.github.io/<repo-name>`에서 서비스됩니다. URL에 접속하면 사이트를 볼 수 있습니다!

이제 `main` 브랜치에 새 변경 사항을 푸시하면 GitHub Pages가 자동으로 최신 변경 사항으로 배포된 사이트를 업데이트합니다. 이 과정은 서버 부하에 따라 몇 분 걸릴 수 있으며, 캐시를 지우려면 강력 새로 고침이 필요할 수 있습니다.

이렇게 해서 기본 웹 애플리케이션을 GitHub Pages에 성공적으로 배포했습니다. Pages 배포에 CSS와 JavaScript 파일도 포함할 수 있어 아름답고 인터랙티브한 사이트를 만들 수 있습니다. React나 Angular 같은 프레임워크를 배우면 GitHub Pages에 그런 것도 배포할 수 있습니다!
