# --description--

CRUD — це акронім, який означає Create, Read, Update і Delete. Це чотири операції постійного збереження даних.

Постійне збереження означає збереження даних таким чином, щоб вони були доступні навіть після вимкнення живлення або перезавантаження пристрою.

Розуміння того, як працює базова операція CRUD, є ключовим для веб-розробки, оскільки це основа для роботи з базами даних і створення застосунків, де користувачі можуть додавати, переглядати, змінювати та видаляти дані.

Тепер давайте детальніше розглянемо кожну частину CRUD

- Create означає процес створення нових даних. Наприклад, у веб-застосунку це може бути, коли користувач додає новий допис у блог.

- Read — це операція, під час якої дані отримуються з бази даних. Наприклад, коли ви переглядаєте допис у блозі або свій профіль на сайті, ви виконуєте операцію читання, щоб отримати та відобразити дані, збережені в базі даних.

- Update передбачає зміну існуючих даних у базі даних. Прикладом може бути редагування допису в блозі або оновлення інформації у вашому профілі.

- Delete — це операція, яка видаляє дані з бази даних. Наприклад, коли ви видаляєте допис у блозі або акаунт, ви виконуєте операцію видалення.

CRUD використовується при роботі з базами даних, інтерфейсом користувача та RESTful API. RESTful API — це набір конвенцій для створення веб-сервісів, які дозволяють клієнту взаємодіяти з базою даних або бекенд-системою, виконуючи операції CRUD через стандартні HTTP-методи.

HTTP означає Hypertext Transfer Protocol і є основою для обміну даними в мережі. Існують HTTP-методи, які визначають дії, що можна виконувати над ресурсами в мережі. Загальні методи — це `GET`, `POST`, `PUT`, `PATCH`, `DELETE`.

Ви дізнаєтеся більше про RESTful API та HTTP у майбутніх уроках, а зараз коротко розглянемо, як CRUD відповідає різним HTTP-методам.

- `POST` використовується для створення нового ресурсу.

- `GET` використовується для отримання або читання даних.

- `PUT` використовується для оновлення ресурсу шляхом повної заміни.

- `PATCH` використовується для часткового оновлення ресурсу.

- `DELETE` використовується для видалення ресурсу.

Ось приклад того, як операції CRUD можуть бути представлені в коді за допомогою простого масиву в JavaScript:

```js
let items = [];

// Create
function createItem(item) {
  items.push(item);
}

// Read
function readItems() {
  return items;
}

// Update
function updateItem(index, newItem) {
  items[index] = newItem;
}

// Delete
function deleteItem(index) {
  items.splice(index, 1);
}

// Example Usage
createItem('Book');
console.log(readItems()); // ['Book']
updateItem(0, 'Magazine');
console.log(readItems()); // ['Magazine']
deleteItem(0);
console.log(readItems()); // []
```

У цьому прикладі:

- Ми створюємо елемент, додаючи його в масив.

- Ми читаємо елементи, повертаючи масив.

- Ми оновлюємо елемент, змінюючи елемент масиву за певним індексом.

- Ми видаляємо елемент, видаляючи його з масиву за допомогою `splice()`.

Це базове представлення того, як працюють операції CRUD на концептуальному рівні.
