# --interactive--

## Множини в JavaScript

- `Set` — це вбудований варіант для керування колекцією даних.
- Множини гарантують, що кожне значення в них з’являється лише один раз, що робить їх корисними для усунення дублікатів у масиві або роботи з колекціями унікальних значень.
- Ви можете створити `Set` за допомогою конструктора `Set()`:

:::interactive_editor

```js
const set = new Set([1, 2, 3, 4, 5]);
console.log(set); // Set { 1, 2, 3, 4, 5 }
```

:::

- Множинами можна керувати за допомогою таких методів:

  - `add()`: додає новий елемент до `Set`.
  - `delete()`: видаляє елемент із `Set`.
  - `has()`: перевіряє, чи існує елемент у `Set`.
  - `clear()`: видаляє всі елементи з `Set`.
  - `keys()` та `values()`: обидва повертають `SetIterator`, який містить значення `Set`. Вони однакові, бо `keys()` — це псевдонім для `values()`.
  - `forEach()`: для ітерації по значеннях `Set`.

## Weaksets у JavaScript

- `WeakSet` — це колекція об’єктів, яка дозволяє зберігати об’єкти з «слабкими» посиланнями.

## Множини проти WeakSets

- На відміну від множин, `WeakSet` не підтримує примітиви, як числа чи рядки.
- `WeakSet` зберігає лише об’єкти, а референси на ці об’єкти є «слабкими», тобто якщо об’єкт ніде більше не використовується у вашому коді, він автоматично видаляється для звільнення пам’яті.

## Карти в JavaScript

- `Map` — це вбудований об’єкт, який зберігає пари ключ-значення, як і об’єкт.
- Карти відрізняються від стандартних JavaScript об’єктів тим, що дозволяють ключі будь-якого типу, включно з об’єктами та функціями.
- `Map` забезпечує кращу продуктивність порівняно зі стандартним об’єктом при частому додаванні та видаленні пар ключ-значення.
- Ви можете створити `Map` за допомогою конструктора `Map()`:

:::interactive_editor

```js
const map = new Map([
  ['flower', 'rose'],
  ['fruit', 'apple'],
  ['vegetable', 'carrot']
]);
console.log(map); // Map(3) { 'flower' => 'rose', 'fruit' => 'apple', 'vegetable' => 'carrot' }
```

:::

- Картами можна керувати за допомогою таких методів:

  - `set()`: додає нову пару ключ-значення до `Map`.
  - `get()`: отримує значення за ключем із `Map`.
  - `delete()`: видаляє пару ключ-значення з `Map`.
  - `has()`: перевіряє, чи існує ключ у `Map`.
  - `clear()`: видаляє всі пари ключ-значення з `Map`.

Зверніть увагу, що і карти, і множини мають властивість `size`, яка повертає кількість унікальних елементів у них.

## WeakMaps у JavaScript

- `WeakMap` — це колекція пар ключ-значення, як `Map`, але з «слабкими» референсами на ключі. Ключі мають бути об’єктами, а значення можуть бути будь-якими.

## Карти проти WeakMaps

- WeakMaps схожі на WeakSets тим, що зберігають лише об’єкти, а референси на них є «слабкими».
