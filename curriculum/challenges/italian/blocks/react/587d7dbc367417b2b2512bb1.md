# --description--

React è una libreria di visualizzazione open source creata e mantenuta da Facebook. È uno strumento eccellente per renderizzare l'interfaccia utente (UI) delle moderne applicazioni web.

React usa un'estensione di sintassi di JavaScript chiamata JSX che ti permette di scrivere HTML direttamente all'interno di JavaScript. Questo offre diversi vantaggi. Ti consente di usare tutta la potenza programmatica di JavaScript dentro l'HTML e aiuta a mantenere il codice leggibile. Per la maggior parte, JSX è simile all'HTML che hai già imparato, tuttavia ci sono alcune differenze chiave che verranno trattate nel corso di queste sfide.

Per esempio, poiché JSX è un'estensione sintattica di JavaScript, puoi effettivamente scrivere JavaScript direttamente dentro JSX. Per farlo, includi semplicemente il codice che vuoi venga trattato come JavaScript all'interno di parentesi graffe: `{ 'this is treated as JavaScript code' }`. Tieni a mente questo, perché sarà usato in diverse sfide future.

Tuttavia, poiché JSX non è JavaScript valido, il codice JSX deve essere compilato in JavaScript. Il transpiler Babel è uno strumento popolare per questo processo. Per tua convenienza, è già aggiunto dietro le quinte per queste sfide. Se dovessi scrivere JSX sintatticamente non valido, vedrai il primo test di queste sfide fallire.

Vale la pena notare che sotto il cofano le sfide chiamano `ReactDOM.render(JSX, document.getElementById('root'))`. Questa chiamata di funzione è ciò che inserisce il tuo JSX nella rappresentazione leggera del DOM di React. React poi usa snapshot del proprio DOM per ottimizzare l'aggiornamento solo di parti specifiche del DOM reale.

# --instructions--

Il codice attuale usa JSX per assegnare un elemento `div` alla costante `JSX`. Sostituisci `div` con un elemento `h1` e aggiungi il testo `Hello JSX!` al suo interno.
