# --description--

React에서 이펙트는 컴포넌트 렌더링 과정 외부에서 발생하는 모든 일을 의미합니다. 즉, React가 UI 렌더링의 일부로 직접 처리하지 않는 모든 작업입니다.

일반적인 예로는 데이터 가져오기, 브라우저 탭 제목 업데이트, 브라우저 로컬 스토리지 읽기 또는 쓰기, 사용자 위치 가져오기 등이 있습니다. 이러한 작업은 외부 세계와 상호작용하며 부수 효과라고 불립니다.

React는 이러한 부수 효과를 처리할 수 있도록 `useEffect` 훅을 제공합니다. `useEffect` 훅은 컴포넌트가 렌더링되거나 업데이트된 후 함수를 실행할 수 있게 해줍니다.

`useEffect` 훅이 어떻게 작동하는지, 그리고 왜 현대 React 개발에 필수적인지 살펴보겠습니다.

`useEffect` 훅을 사용하려면 먼저 가져와야 합니다:

```js
import { useEffect } from "react";
```

그다음 함수처럼 사용하세요:

```js
useEffect(() => {
  // Your side effect logic (usually a function) here
}, [dependencies]);
```

이펙트 함수는 컴포넌트가 렌더링된 후 실행되며, 선택적인 `dependencies` 인수는 이펙트가 실행되는 시점을 제어합니다.

`dependencies`는 "반응 값"(state, props, 함수, 변수 등)의 배열일 수도 있고, 빈 배열이거나 완전히 생략할 수도 있습니다. 다음은 이 모든 옵션이 `useEffect` 작동 방식을 어떻게 제어하는지입니다:

- `dependencies`가 하나 이상의 반응 값을 포함하는 배열이라면, 해당 값들이 변경될 때마다 이펙트가 실행됩니다.

- `dependencies`가 빈 배열이라면, `useEffect`는 컴포넌트가 처음 렌더링될 때 한 번만 실행됩니다.

- `dependencies`를 생략하면, 컴포넌트가 렌더링되거나 업데이트될 때마다 이펙트가 실행됩니다.

예를 들어, 이 `Counter` 애플리케이션에서는 `dependencies` 인수를 전달하지 않아서, 컴포넌트가 렌더링될 때와 업데이트될 때마다 이펙트가 실행됩니다:

```jsx
import { useState, useEffect } from "react";

const Counter = () => {
  const [count, setCount] = useState(0);

  useEffect(() => {
    console.log("Component renders");
  });

  return (
    <div
      style={{
        display: "flex",
        alignItems: "center",
        flexDirection: "column",
      }}
    >
      <h2>{count}</h2>
      <div>
        <button onClick={() => setCount(count + 1)}>Increase</button>
        <button onClick={() => setCount(count - 1)}>Decrease</button>
      </div>
    </div>
  );
};

export default Counter;
```

하지만 의존성으로 빈 배열을 전달하면, 이펙트는 첫 렌더링 시에만 실행됩니다:

```js
useEffect(() => {
  console.log('Component renders');
}, []);
```

`count` state를 의존성으로 전달하면, 이펙트는 컴포넌트가 처음 렌더링될 때와 `count`가 변경될 때 실행됩니다:

```js
useEffect(() => {
  document.title = `The current count is ${count}`;
  console.log('component renders');
}, [count]);
```

설정한 이펙트가 컴포넌트 렌더링 수명 주기 이후에도 지속된다면, 컴포넌트가 렌더링되거나 업데이트된 후에 그 함수를 "정리"하는 또 다른 함수가 필요할 수 있습니다.

예를 들어, 이펙트 함수가 `setInterval()`를 사용하거나 `window.addEventListener()` 같은 이벤트 리스너를 설정하거나 서버에 연결한다면, 각각 `clearInterval()`, `window.removeEventListener()`를 실행하고 서버 연결을 끊는 정리 함수가 필요합니다.

`useEffect` 훅에서 정리 함수를 반환하는 구문은 다음과 같습니다:

```js
useEffect(() => {
  // Your side effect logic here
  return () => {
    // Cleanup logic here (optional)
  };
}, [dependencies]);
```

예를 들어, 스크롤 이벤트 리스너를 추가했다면, 정리 함수에서 이를 제거하여 정리할 수 있습니다:

```js
useEffect(() => {
  const handleScroll = () => {
    // Handle scroll logic
  };
  window.addEventListener("scroll", handleScroll);

  return () => {
    window.removeEventListener("scroll", handleScroll);
  };
}, []);
```
