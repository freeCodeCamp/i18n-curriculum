# --explanation--

Un `edge case` es una situación rara o extrema que podría causar problemas en un programa. Por ejemplo:

`We need to test the app with edge cases to make sure it works in all situations.` - Necesitas probar situaciones inusuales para asegurarte de que la app funcione correctamente.

El `Unexpected behavior` ocurre cuando un programa hace algo que el programador no esperaba. Por ejemplo:

`The app showed unexpected behavior when I tried to open it.` - La app no funcionó como esperabas cuando intentaste abrirla.
