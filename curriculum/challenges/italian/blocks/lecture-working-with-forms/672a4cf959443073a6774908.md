# --interactive--

In HTML, i controlli del modulo, come gli input, possono trovarsi in diverse fasi o condizioni come uno stato `focused`, uno stato `readonly` o uno stato `disabled`.

Il primo stato è considerato lo stato `default`. Lo stato predefinito di un input per l'indirizzo email è un input vuoto. Questo è l'aspetto dell'input email quando viene renderizzato per la prima volta nella pagina.

:::interactive_editor

```html
<input type="email" name="email" id="email" />
```

:::

Quando l'utente clicca su un controllo del modulo o lo seleziona con il tasto tab della tastiera, significa che si trova nello stato `focused`. Quando un input è nello stato `focused`, la maggior parte dei browser mostra un bordo evidenziato di colore blu intorno all'input. Ma puoi scegliere di definire ulteriori stili in CSS.

Clicca su qualsiasi parte dello spazio vuoto nella finestra di anteprima e poi premi il tasto `tab` per vedere lo stato di focus. Per vedere le anteprime, devi abilitare l'editor interattivo.

:::interactive_editor

```html
<input type="email" name="email" id="email" />
```

:::

Un altro stato del modulo è lo stato `disabled`. Questo stato indica agli utenti che un input non può essere messo a fuoco o attivato.

Abilita l'editor interattivo e prova a cliccare sull'input email: noterai che non sarà più possibile metterlo a fuoco.

:::interactive_editor

```html
<input disabled type="email" name="email" id="email" />
```

:::

Simile allo stato `focused`, puoi scegliere di definire ulteriori stili per lo stato `disabled` usando CSS.

Un altro tipo di stato del modulo è lo stato `readonly`. Questo è quando un controllo del modulo, come un input, non è modificabile dall'utente. Ecco un esempio di come impostare un input email in sola lettura. L'attributo `value` viene usato per impostare il valore mostrato all'interno del campo di input.

Abilita l'editor interattivo e prova a modificare il valore attuale di `example@email.com` nella finestra di anteprima: noterai che non è possibile.

:::interactive_editor

```html
<input
  readonly
  type="email"
  name="email"
  id="email"
  value="example@email.com"
/>
```

:::

Una differenza chiave tra lo stato `disabled` e lo stato `readonly` è che `readonly` può essere messo a fuoco mentre lo stato `disabled` no.

Capire i diversi stati di un modulo è importante perché garantiscono un'esperienza utente fluida fornendo feedback chiari e indicazioni durante la gestione degli errori.
