# --description--

Middleware ni dhana muhimu katika Express, na inatuwezesha kushughulikia maombi na majibu kati ya mteja na seva. Kitendakazi cha middleware kina upatikanaji wa kitu cha ombi, kitu cha jibu, na kitendakazi kinachofuata katika mzunguko wa ombi-jibu wa programu.

Tuchambue kwa undani:

## Middleware ni nini?

Middleware kwa msingi ni kitendakazi kinachotekelezwa wakati wa mzunguko wa ombi kwa seva. Hufanya kazi mbalimbali kama vile kurekodi maombi, kushughulikia makosa, kuchambua maombi yanayoingia, au kuhudumia mafaili ya takatifu.

Express hutoa njia rahisi ya kufafanua middleware inayoshughulikia maombi yanayoingia kabla hayajafikia washughulikiaji wa njia za data.

Jinsi middleware inavyofanya kazi:

Wakati ombi linapotumwa kwa programu ya Express, middleware hufanyika kwa mpangilio ilivyoongezwa. Mtiririko ni kama huu:

* Ombi hutumwa kwa seva.
* Vitendakazi vya middleware hushughulikia ombi.
* Ikiwa middleware inaita `next()`, inapeleka udhibiti kwa middleware inayofuata au mshughulikiaji wa njia ya data.
* Ikiwa middleware haipigi `next()`, mzunguko wa ombi-jibu unasimama.

```js
const express = require('express')
const app = express()

// Simple middleware that logs request info
app.use((req, res, next) => {
  console.log('Request URL:', req.url)
  console.log('Request Method:', req.method)
  next() // Passes control to the next middleware
})

app.get('/', (req, res) => {
  res.send('Hello World!')
})

app.listen(3000, () => {
  console.log('Server running on http://localhost:3000')
})
```

`app.use()`: Inaongeza middleware kwa ujumla katika programu yako.

Katika mfano huu, middleware inarekodi maelezo ya ombi na inaita `next()` kuendelea kwa middleware inayofuata au mshughulikiaji wa njia ya data.

Middleware ni kipengele chenye nguvu katika Express kinachokuwezesha kukamata na kubadilisha maombi katika vituo mbalimbali. Unaweza kutumia middleware kushughulikia mambo kama vile kurekodi, uthibitisho wa utambulisho, au hata kubadilisha vitu vya ombi na jibu kabla havijafikia washughulikiaji wa njia zako za data.
