# --description--

Node.js에는 내장 WebSocket 서버가 없습니다. 개발자들은 일반적으로 `ws` 같은 외부 패키지를 사용해 WebSocket 프로토콜을 구현하고 클라이언트와 서버 간 실시간 통신을 처리합니다.

`ws` 라이브러리는 가볍고 빠르며 사용하기 쉽습니다. WebSocket 프로토콜을 구현하여 WebSocket 연결을 생성하고 관리할 수 있습니다.

npm으로 설치할 수 있습니다:

```bash
npm install ws
```

설치 후, `ws` 라이브러리를 가져오고 `Server` 메서드를 호출하며 포트를 전달해 WebSocket 서버를 만드세요:

```js
const WebSocket = require("ws");

const server = new WebSocket.Server({
  port: 3000,
});
```

이렇게 하면 포트 `3000`에서 연결을 대기하는 WebSocket 서버가 시작됩니다.

클라이언트가 연결되면 서버는 `connection` 이벤트를 발생시킵니다.

```js
server.on("connection", (socket) => {
  console.log("Client connected");
});
```

이 이벤트는 서버가 "새 클라이언트가 연결되었습니다"라고 알리는 것과 같습니다.

`socket` 매개변수는 서버와 해당 클라이언트 간 연결을 나타냅니다. 이 연결을 사용해 메시지를 보내고 받을 수 있습니다.

연결된 클라이언트가 데이터를 보내면 서버는 `message` 이벤트를 발생시킵니다.

```js
server.on("connection", (socket) => {
  socket.on("message", (message) => {
    console.log(message.toString());
  });
});
```

이 예제에서 서버는 들어오는 메시지를 듣고 콘솔에 기록합니다.

`send()` 메서드를 사용해 클라이언트에 데이터를 다시 보낼 수 있습니다:

```js
socket.send("Hello from the server");
```

이것은 열린 WebSocket 연결을 통해 메시지를 보냅니다.

연결이 열린 상태인 한, 클라이언트와 서버는 실시간으로 메시지를 계속 주고받을 수 있습니다.

`connection`와 `message` 외에도 WebSocket 연결은 `close`과 `error` 이벤트도 발생시킵니다.

`close`은 연결이 종료될 때 발생합니다:

```js
socket.on("close", () => {
  console.log("Disconnected");
});
```

`error`은 문제가 발생했을 때 발생합니다:

```js
socket.on("error", (error) => {
  console.error(error);
});
```

이 이벤트들이 모여 대부분 WebSocket 애플리케이션의 기반을 이룹니다.

채팅 애플리케이션에서 흔한 요구사항은 브로드캐스팅입니다. 브로드캐스팅은 메시지를 한 명이 아닌 여러 연결된 클라이언트에게 보내는 것을 의미합니다.

세 명의 사용자가 있는 채팅방을 상상해 보세요: **John Doe**, **Jane Doe**, 그리고 **Wes Smith**.

John Doe가 메시지를 보내면 서버는 그 메시지를 Jane Doe와 Wes Smith에게 전달해야 할 수 있습니다.

이를 위해 서버는 연결된 클라이언트를 추적하고 메시지가 도착할 때마다 반복합니다.

메시지를 보내기 전에 서버는 클라이언트 연결이 아직 열려 있는지 확인합니다:

```js
if (client.readyState === WebSocket.OPEN) {
  // do something
}
```

모든 WebSocket 연결에는 현재 상태를 나타내는 `readyState` 속성이 있습니다.

`WebSocket.OPEN`는 연결이 활성 상태이며 데이터를 주고받을 준비가 되었음을 의미합니다.

이 상태를 확인함으로써 서버는 이미 연결이 끊겼거나 아직 연결이 설정 중인 클라이언트에게 메시지를 보내는 것을 피합니다:

```js
server.on("connection", (socket) => {
  socket.on("message", (message) => {
    server.clients.forEach((client) => {
      if (client.readyState === WebSocket.OPEN) {
        client.send(message.toString());
      }
    });
  });
});
```

이 예제에서는 모든 연결된 클라이언트가 들어오는 메시지를 받습니다. 이것을 브로드캐스팅이라고 합니다.

많은 실시간 애플리케이션이 채팅 메시지, 실시간 알림, 스포츠 업데이트, 멀티플레이어 게임 이벤트 같은 데이터를 전달하기 위해 브로드캐스팅을 사용합니다.

`ws` 라이브러리를 사용하면 Node.js에서 WebSocket 서버를 쉽게 구축할 수 있습니다.

서버를 만들고, 이벤트를 듣고, 메시지를 교환하며, 연결된 클라이언트에게 업데이트를 브로드캐스트하기만 하면 됩니다.

이 기본 요소들이 많은 실시간 애플리케이션의 토대가 됩니다.
