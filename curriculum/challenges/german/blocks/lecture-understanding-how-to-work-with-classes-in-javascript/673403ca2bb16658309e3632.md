# --interactive--

Sie haben gelernt, dass Klassen wie Baupläne sind, die Sie verwenden können, um Objekte zu erstellen.

Diese Baupläne benötigen eine allgemeine Möglichkeit, auf das spezifische Objekt zuzugreifen, das im Code gerade erstellt, aufgerufen oder verändert wird. Genau dafür wird das `this`-Schlüsselwort verwendet. Es bezieht sich auf den Kontext, in dem der Code ausgeführt werden soll. Wenn Sie es innerhalb einer Methode verwenden, ist der Wert von `this` eine Referenz auf das Objekt, das mit dieser Methode verbunden ist. So können Sie auf die Eigenschaften und Methoden des Objekts innerhalb der Klasse zugreifen und die Methode für verschiedene Objekte wiederverwenden.

Zum Beispiel haben wir hier eine `Dessert`-Klasse. Diesmal definieren wir eine `hasPeanuts`-Eigenschaft.

```js
class Dessert {
  constructor(hasPeanuts) {
    this.hasPeanuts = hasPeanuts;
  }
}
```

Beachten Sie, dass wir das `this`-Schlüsselwort innerhalb des Konstruktors verwenden. Denken Sie daran, dass der Konstruktor automatisch aufgerufen wird, wenn ein neues Objekt der Klasse erstellt wird.

Das `this`-Schlüsselwort sagt dem Programm, dass `hasPeanuts` eine Eigenschaft des Objekts sein soll, das gerade erstellt wird.

Die rechte Seite der Zuweisung weist den Wert des `hasPeanuts`-Parameters der `hasPeanuts`-Eigenschaft des neuen Objekts zu.

```js
this.hasPeanuts = hasPeanuts;
```

Das ist im Kontext des Konstruktors, aber Sie können das `this`-Schlüsselwort auch innerhalb anderer Methoden verwenden.

Dies ist unsere `Dessert`-Klasse mit einigen Änderungen. Jetzt haben wir zwei Eigenschaften: `name` und `price`.

```js
class Dessert {
  constructor(name, price) {
    this.name = name;
    this.price = price;
  }

  showPrice() {
    console.log(`The price of ${this.name} is $${this.price}.`);
  }
}
```

Wir haben auch eine neue `showPrice`-Methode. In dieser Methode geben wir eine beschreibende Nachricht mit dem Namen und dem Preis des Dessert-Objekts in der Konsole aus.

Beachten Sie, wie wir das `this`-Schlüsselwort verwenden, um innerhalb der `showPrice`-Methode auf die Eigenschaften des Objekts zuzugreifen.

Welchen Wert wird das `this`-Schlüsselwort im Kontext dieser Methode haben? Es bezieht sich auf das Objekt, auf dem die Methode aufgerufen wird. In diesem Fall erstellen wir eine neue Dessert-Instanz mit dem Namen `"Brownie"` und dem Preis `$5.99`:

```js
const brownie = new Dessert("Brownie", 5.99);
```

Wenn wir die `.showPrice()`-Methode für dieses Objekt aufrufen:

```js
brownie.showPrice();
```

Hier ist das vollständige Beispiel:

:::interactive_editor

```js
class Dessert {
  constructor(name, price) {
    this.name = name;
    this.price = price;
  }

  showPrice() {
    console.log(`The price of ${this.name} is $${this.price}.`);
  }
}

const brownie = new Dessert("Brownie", 5.99);
brownie.showPrice();
console.log(brownie);
```

:::

Der Name und der Preis des aktuellen Objekts (`brownie`) werden im String ersetzt.

Das ist alles dank des `this`-Schlüsselworts möglich, das Ihnen eine allgemeine Möglichkeit gibt, auf das aktuelle Objekt innerhalb der Klasse zuzugreifen. So können Sie die Methode für verschiedene Objekte wiederverwenden.

Das ist im Kontext von Objektmethoden, aber das `this`-Schlüsselwort hat auch in eigenständigen Funktionen und Pfeilfunktionen einen Wert.

Sein Wert innerhalb einer eigenständigen Funktion bezieht sich typischerweise auf das globale Objekt (im Non-Strict-Modus) oder `undefined` (im Strict-Modus).

Der Wert des `this`-Schlüsselworts ist bei Pfeilfunktionen etwas anders. Pfeilfunktionen erben den Wert von `this` aus dem umgebenden Gültigkeitsbereich, in dem sie definiert sind. Sie erzeugen keine eigene `this`-Bindung. Das bedeutet, dass der Wert von `this` vom Kontext abhängt, in dem die Pfeilfunktion definiert ist, nicht davon, wo oder wie die Pfeilfunktion aufgerufen wird. Das macht Pfeilfunktionen hilfreich für Callbacks oder um den Kontext zu bewahren.

Aufgrund dessen bezieht sich der Wert von `this` in Objektmethoden nicht auf das aktuelle Objekt, was Sie bei der Arbeit mit Pfeilfunktionen beachten sollten.

Das Gleiche gilt für Pfeilfunktionen, die innerhalb anderer Funktionen definiert sind. Sie erben den Wert von `this` aus ihrem übergeordneten Gültigkeitsbereich.

Das `this`-Schlüsselwort ist ein mächtiges Werkzeug, um auf Objekte zuzugreifen und sie zu manipulieren. Zu verstehen, wie es funktioniert, ist entscheidend, um objektorientierte Programmierung in JavaScript zu meistern.
