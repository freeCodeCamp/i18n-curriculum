# --description--

Umejifunza awali jinsi ya kuhifadhi mabadiliko na kuyasukuma hadi kwenye hifadhidata yako ya mbali, lakini yote hayo yamekuwa kwenye tawi lako la msingi la `main`. Hii inaweza kufanya kazi kwa mradi mdogo, lakini unapo changia miradi mingine ya chanzo huru, au mradi wako unapokua, matawi ni muhimu sana.

Kwa mfano, ukifanya fork ya hifadhidata na kufanya uhifadhi kwenye tawi la `main` la fork yako, unaweza kupata mabadiliko kwenye fork yako ambayo hifadhidata ya awali haikuyakubali. Hii inaweza kufanya kuwa vigumu sana kusasisha tawi lako na mabadiliko kutoka kwa fork yaliyokubaliwa. Au, unapo fanya kazi kwenye mradi wako mwenyewe, kubadilisha kati ya mazoezi tofauti kunakuwa vigumu zaidi kama hutumii matawi. Badala yake, unapaswa kufuatilia kwa mkono ni mabadiliko gani yanahusiana na kipengele gani au hitilafu gani.

Lakini tawi hasa ni nini? Fikiria tawi lako la `main` kama barabara kuu, ambapo kila uhifadhi ni gari linaloingia barabarani. Unaweza kuamua kwenda kuangalia kitu, kama kuunda kipengele kipya, kwa hivyo unahitaji kutoka barabarani kwa muda kidogo kwenda kuona.

Tawi ni tofauti ya kihistoria kutoka kwenye historia yako kuu, ambapo unaweza kucheza kwa uhuru na mabadiliko kwa ajili ya kipengele kipya au kurekebisha hitilafu. Ikiwa unaridhika na mabadiliko yako, unaweza kurudi barabarani kwa kuunganisha tawi hilo kwenye `main`. Ikiwa hutaridhika na mabadiliko yako, unaweza kufuta tawi na kurudi kwenye `main` bila kukubali mabadiliko hayo.

Kabla ya kuunda tawi jipya, unaangalia matawi yako kwa amri ya `git branch`. Matokeo yanaweza kuonekana kama haya:

```sh
* main
```

Kwa sasa tuna tawi moja tu, tawi letu la msingi la `main`. Nyota (`*`) inaonyesha kuwa hilo ndilo tawi ulilo "chagua" kwa sasa. Utajifunza zaidi kuhusu hilo hivi karibuni.

Unaweza kuunda tawi jipya kwa amri ya `git branch`, ukiweka hoja ya jina la tawi jipya:

```sh
git branch feature
```

Hii inaunda tawi jipya lenye jina `feature`. Tuchunguze tena matawi yetu kwa `git branch`:

```sh
  feature
* main
```

Sasa tunaweza kuona tawi letu jipya, lakini bado tuko kwenye tawi la `main`. Tunahitaji kubadilisha kwenda tawi letu jipya kwa "kulichagua". Unapochagua tawi jipya, unaambia Git kupakia mafaili katika hali halisi walivyokuwa wakati huo.

Tuchague tawi letu jipya kwa:

```sh
git checkout feature
```

au

```sh
git switch feature
```

Na tumia `git branch` tena kuona:

```sh
* feature
  main
```

Angalia kuwa nyota sasa imehamia kwenye tawi ulilo "chagua" kwa sasa.
 
Mara nyingi unapotengeneza tawi jipya, utataka kulichukua mara moja. Badala ya kutengeneza tawi kisha kulichukua kwa kutumia amri mbili, unaweza kutengeneza na kubadilisha tawi kwa amri moja kwa:

```sh
git checkout -b feature
```

au

```sh
git switch -c feature
```

Sasa tumetengeneza na kuchukua tawi jipya. Tawi hili ni nakala ya tawi la `main`. Lina msimbo wote ule ule ambao `main` lilikuwa nalo wakati ulipotengeneza tawi. Sasa, tunaweza kuongeza msimbo wowote kwenye tawi hili bila kuathiri tawi la `main`. Kwa maana nyingine, tumetoka kwenye barabara kuu kuunda kipengele kipya. Ukikagua `git status` yako, utaona kitu kama hiki:

```sh
On branch feature
nothing to commit, working tree clean
```

Unda faili mpya ya `feature.md` kwa kuendesha hii:

```sh
echo "This is our new feature" > feature.md
```

Na sasa hali itakuwa kama hii:

```sh
On branch feature
Untracked files:
  (use "git add <file>..." to include in what will be committed)
	feature.md

nothing added to commit but untracked files present (use "git add" to track)
```

Ifuatayo, ni wakati wa kuhifadhi kipengele chako kipya. Kwanza, ongeza faili kwenye eneo la kuandaa:

```sh
git add feature.md
```

Kisha, hifadhi mabadiliko:

```sh
git commit -m "my new feature"
```

Mwishowe, sukuma tawi lako la `feature` hadi kwenye hifadhidata yako:

```sh
git push -u origin feature
```

Bendera ya `-u` inaunganisha tawi lako la `feature` la ndani na tawi la mbali, hivyo kusukuma siku zijazo kunaweza kufanywa kwa amri ya `git push` tu.

Ifuatayo, tutajifunza jinsi ya kufanya ombi la kuvuta mabadiliko kwa kipengele chetu kipya!
