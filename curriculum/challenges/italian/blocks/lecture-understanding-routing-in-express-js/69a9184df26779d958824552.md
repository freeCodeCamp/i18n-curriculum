# --description--

Servire file statici è una necessità comune nelle applicazioni web. I file statici includono risorse come immagini, file CSS, JavaScript lato client, caratteri tipografici e altre risorse che non cambiano dinamicamente.

Express rende semplice servire questi file grazie al suo middleware integrato `express.static()`.

In Express, servire file statici significa consegnare i file direttamente dal file system del server al browser del client senza alcuna elaborazione aggiuntiva lato server.

Per esempio, quando un browser richiede file come `/styles.css`, `/script.js` o `/images/logo.png`, Express può trovare quei file in una cartella specificata e servirli direttamente. Se un file non viene trovato, Express risponde automaticamente con un 404. Poiché questi file non devono essere generati o modificati sul server, si chiamano *statici*.

Questo rende facile servire risorse come fogli di stile e immagini, eliminando la necessità di definire manualmente rotte per ogni file.

Ecco come puoi configurare il servizio di file statici in Express:

```javascript
const express = require("express");
const path = require("path");
const app = express();
const port = 3000;

// Serve static files from the "public" directory
app.use(express.static(path.join(__dirname, "public")));

app.get("/", (req, res) => {
  res.send("Welcome to the static file demo!");
});

app.listen(port, () => {
  console.log(`Server running at http://localhost:${port}`);
});
```

Per prima cosa, importiamo `express` per creare il nostro server e `path` per costruire in modo sicuro i percorsi dei file su diversi sistemi operativi.

Poi, `app.use(express.static(path.join(__dirname, "public")))` dice a Express di servire i file dalla cartella `public`. Il middleware `express.static()` dice a Express di servire i file dalla cartella `public`. Qualsiasi file all’interno di questa directory diventa accessibile al browser.

Definiamo anche una semplice rotta `GET` per `/` che invia un messaggio di benvenuto. Questo mostra che il servizio di file statici può coesistere con rotte dinamiche.

Infine, `app.listen()` avvia il server sulla porta 3000. Una volta che il server è in esecuzione, le richieste per le risorse statiche nella cartella `public` vengono gestite automaticamente.

Per far funzionare tutto ciò, ti serve una cartella chiamata `public` dove archiviare i tuoi file statici. Una struttura tipica del progetto potrebbe essere questa:

```md
my-app/
│
├── app.js
└── public/
    ├── index.html
    ├── styles.css
    └── image.png
```

Quando un utente fa una richiesta, per esempio visitando `http://localhost:3000/index.html`, Express verifica se il file richiesto esiste nella directory `public`. Se il file esiste, Express lo invia direttamente al browser. Se il file non esiste, Express passa al middleware successivo o restituisce una risposta 404.

Ricorda che non devi definire rotte per ogni file, quindi una richiesta a `http://localhost:3000/styles.css` significa che Express cerca `public/styles.css` e risponde automaticamente con quel file.

Puoi anche personalizzare come vengono serviti i file statici. Per esempio, potresti servire file da più directory o configurare il comportamento della cache.

Immagina che la tua app abbia questa struttura di file:

```md
my-app/
│
├── app.js
│
├── public/
│   ├── index.html
│   └── styles.css
│
└── images/
    └── image.png
```

Ecco come puoi registrare più directory per servire file statici da esse:

```javascript
app.use(express.static(path.join(__dirname, "public")));
app.use("/images", express.static(path.join(__dirname, "images")));
```

Qui usiamo `app.use(express.static(path.join(__dirname, "public")));` come prima, il che significa che i file dentro la directory `public` vengono serviti dal percorso root, proprio come prima.

Ma se guardi la riga successiva, vedrai che passiamo `"/images"` come primo argomento a `app.use()`, seguito da `express.static(path.join(__dirname, "images"))`. In questo caso, `"/images"` è un percorso di montaggio, che dice a Express di usare questo middleware solo quando riceve una richiesta a `/images`. Quando succede, `express.static()` si attiva e cerca i file dentro la directory `images`.

Quindi con questa configurazione, ecco come funzionerebbero le seguenti richieste:

*   GET `http://localhost:3000/styles.css`: Express cerca nella directory `public` il file `styles.css`, lo trova e lo invia al client.
    
*   GET `http://localhost:3000/images/image.png`: Poiché questa richiesta inizia con `/images`, Express cerca nella directory `images` il file `image.png`, lo trova e lo invia al client.
    

Servire file statici con Express è semplice e potente. Usando `express.static()`, puoi facilmente servire risorse come immagini, fogli di stile e JavaScript lato client.

Tenere i file statici in una cartella dedicata come `public`, o usare più directory di file e percorsi di montaggio, aiuta a mantenere la tua applicazione organizzata, manutenibile e facile da scalare.
