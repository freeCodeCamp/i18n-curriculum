### --question--

#### --text--

Como a comunicação HTTP difere da comunicação WebSocket?

#### --distractors--

HTTP é bidirecional enquanto WebSocket é unidirecional.

---

HTTP mantém uma conexão permanente, enquanto WebSocket fecha após cada troca.

---

HTTP e WebSocket seguem o mesmo padrão de requisição-resposta.

#### --answer--

HTTP segue `Client --> Request --> Server --> Response`, enquanto WebSocket é `Client <--> Server`.

Qual é o fluxo de mensagens no Pub/Sub?

#### --distractors--

Um publicador envia para um assinante --> o assinante encaminha para um tópico --> o tópico armazena a mensagem.

---

Um tópico envia uma mensagem --> publicadores a recebem --> publicadores encaminham para assinantes.

---

Um assinante conecta a um publicador --> o publicador envia dados --> o assinante confirma o recebimento.

#### --answer--

Um assinante se inscreve em um tópico --> um publicador envia uma mensagem para esse tópico --> o sistema de mensagens entrega para todos os assinantes desse tópico.
