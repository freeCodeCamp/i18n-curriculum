# --description--

HTML 是一種非常寬容的語言－即使你犯錯，例如忘記包含結束頁籤，元素仍然會呈現。

假設你有一個沒有結束標籤的 `h2` 元素：

```html
<h1>Article Topic</h1>
<h2>Subheading 1 </h2>
<h2>Subheading 2 </h2>

<!-- This h2 does not have a closing tag -->
<h2>Subheading 3
```

沒有結束標籤的 `h2` 仍然會正常呈現。這是因為瀏覽器使用一種解析演算法來控制代碼常見錯誤，並嘗試盡可能接近作者的意圖來呈現 HTML。

但這有時可能會適得其反。讓我們在程式碼中現有的標題 2 頁籤下添加幾段文字：

```html
<h1>Article Topic</h1>
<p>Lorem ipsum dolor sit amet consectetur adipisicing elit. Maiores, nisi.</p>

<h2>Subheading 1 </h2>
<p>Lorem, ipsum dolor sit amet consectetur adipisicing elit. At, doloremque.</p>

<h2>Subheading 2 </h2>
<p>Lorem ipsum dolor sit amet consectetur adipisicing elit. Unde, placeat.</p>

<!-- This h2 does not have a closing tag -->
<h2>Subheading 3
<p>Lorem ipsum dolor, sit amet consectetur adipisicing elit. Tempore, illum.</p>
```

因此，位於沒有結束 `h2` 標籤的 `h2` 下方的段落元素會呈現為標題 2。這就是為什麼你需要一個 HTML 驗證器。

HTML 驗證器是一種工具，用來檢查你的 HTML 程式碼是否符合標準 HTML 規格。它幫助你識別 HTML 程式碼中的錯誤和警告，確保你的網頁結構正確且符合網路標準。

使用 HTML 驗證器不僅對你和你未來的程式碼審查有利益，也對任何瀏覽你程式碼的人有利益，例如你的隊友和開源貢獻者。

有幾個你可以使用的 HTML 驗證工具。最廣泛被接受的是 `w3.org` 標記驗證服務。

當你造訪網站 [`validator.w3.org`](https://validator.w3.org/) 時，你可以點擊 `Validate by Direct Input` 按鈕並貼上你的 HTML 程式碼。

當你點擊 `Check` 按鈕時，將會顯示一個包含需要修正錯誤的結果列表。

你可以使用的另一個 HTML 驗證工具是 [`jsonformatter.org`](https://jsonformatter.org/)。

你可以將你的 HTML 程式碼複製並貼上到第一個編輯器中，當你點擊 `Validate` 按鈕時，它會顯示你程式碼中的任何錯誤。
