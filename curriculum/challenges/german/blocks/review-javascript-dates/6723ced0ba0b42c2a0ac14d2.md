## Das `date()`-Objekt und gängige Methoden

- **Definition**: Das `date()`-Objekt wird verwendet, um Daten und Zeiten in JavaScript zu erstellen, zu bearbeiten und zu formatieren. Im folgenden Beispiel wird das Schlüsselwort `new` verwendet, um eine neue Instanz des `Date`-Objekts zu erstellen, und das `Date`-Objekt wird dann der Variablen `now` zugewiesen. Wenn Sie den Wert von `now` in der Konsole ausgeben würden, sehen Sie das aktuelle Datum und die aktuelle Uhrzeit basierend auf der Systemuhr des Computers, der den Code ausführt.

:::interactive_editor

```js
const now = new Date();
console.log(now); // prints the current date and time
```

:::

- **`Date.now()`-Methode**: Diese Methode wird verwendet, um das aktuelle Datum und die aktuelle Uhrzeit zu erhalten. `Date.now()` gibt die Anzahl der Millisekunden seit dem 1. Januar 1970, 00:00:00 UTC zurück. Dies ist als Unix-Epoch-Zeit bekannt. Die Unix-Epoch-Zeit ist eine gängige Methode, um Daten und Zeiten in Computersystemen darzustellen, da sie eine Ganzzahl ist, die leicht gespeichert und bearbeitet werden kann. UTC steht für Coordinated Universal Time, den primären Zeitstandard, nach dem die Welt Uhren und Zeit reguliert.
- **`getDate()`-Methode**: Diese Methode wird verwendet, um den Tag des Monats basierend auf dem aktuellen Datum zu erhalten. `getDate()` gibt einen ganzzahligen Wert zwischen 1 und 31 zurück, abhängig vom Tag des Monats. Wenn das Datum ungültig ist, wird `NaN` (Not a Number) zurückgegeben.

:::interactive_editor

```js
const now = new Date("2014-10-15");
const date = now.getDate();
console.log(date); // 15
```

:::

- **`getMonth()`-Methode**: Diese Methode wird verwendet, um den Monat zu erhalten. Der Monat ist nullbasiert, daher ist Januar 0, Februar 1 und so weiter. In diesem Beispiel ist die Ausgabe 9, was Oktober entspricht. Wenn der Monat ungültig ist, wird `NaN` zurückgegeben.

:::interactive_editor

```js
const now = new Date("2014-10-15");
const month = now.getMonth();
console.log(month); // 9
```

:::

- **`getFullYear()`-Methode**: Diese Methode wird verwendet, um das volle Jahr zu erhalten. Wenn das Jahr ungültig ist, wird `NaN` zurückgegeben.

:::interactive_editor

```js
const now = new Date("2014-10-15");
const year = now.getFullYear();
console.log(year); // 2014
```

:::

## Verschiedene Möglichkeiten, Daten zu formatieren

- **`toISOString()`-Methode**: Diese Methode wird verwendet, um das Datum im erweiterten `ISO` (ISO 8601)-Format zu formatieren. ISO 8601 ist ein internationaler Standard zur Darstellung von Daten und Zeiten. Das Format ist `YYYY-MM-DDTHH:mm:ss.sssZ`.

:::interactive_editor

```js
const date = new Date("2014-10-15");
console.log(date.toISOString()); // "2014-10-15T00:00:00.000Z"
```

:::

- **`toLocaleDateString()`-Methode**: Diese Methode wird verwendet, um das Datum basierend auf der Gebietsschema-Einstellung des Benutzers zu formatieren.

:::interactive_editor

```js
const date = new Date("2014-10-15");
console.log(date.toLocaleDateString("en-US"));  // "10/15/2014"
```

:::

Die `toLocaleDateString()`-Methode akzeptiert zwei optionale Parameter: locales und options.

Der Parameter locales ist ein String, der das zu verwendende Gebietsschema darstellt. Zum Beispiel können Sie `"en-US"` für Englisch (Vereinigte Staaten) oder `"fr-FR"` für Französisch (Frankreich) übergeben. Wenn Sie keinen locales-Parameter übergeben, wird das Standardgebietsschema verwendet. Der zweite optionale Parameter ist der options-Parameter. Dieser Parameter ist ein Objekt, mit dem Sie das Format des Datumsstrings angeben können.

:::interactive_editor

```js
const date = new Date("2014-10-15");
const options = {
  weekday: "long",
  year: "numeric",
  month: "long",
  day: "numeric",
};
console.log(date.toLocaleDateString("en-GB", options)); // "Wednesday, 15 October 2014"
```

:::
