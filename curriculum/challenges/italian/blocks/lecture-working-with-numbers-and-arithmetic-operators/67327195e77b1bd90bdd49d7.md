# --interactive--

JavaScript è un linguaggio in cui a volte le cose funzionano in modi sorprendenti o addirittura strani. Una di queste sorprese si verifica quando mescoli numeri e stringhe nei calcoli. La prima cosa che probabilmente proverai è aggiungere un numero e una stringa. In JavaScript, l'operatore `+` ha una doppia funzione. Gestisce sia l'addizione sia la concatenazione di stringhe, che è un modo per unire due stringhe insieme.

Quando usi `+` con un numero e una stringa, JavaScript decide di trattarli entrambi come stringhe e li unisce. Se controlli il tipo del risultato con l'operatore `typeof`, vedrai che è effettivamente una stringa:

:::interactive_editor

```js
const result = 5 + '10';

console.log(result); // 510
console.log(typeof result); // string
```

:::

Cosa pensi succederà se inverti l'ordine di `5` e `'10'`?

:::interactive_editor

```js
const result = '10' + 5;

console.log(result); // 105
console.log(typeof result); // string
```

:::

Puoi vedere che è successo lo stesso. JavaScript vede una stringa in `'10'` e un numero in `5`, quindi converte il numero in stringa e li concatena. Questo è noto come conversione implicita. La conversione implicita è quando un valore di un tipo di dato viene convertito in un altro.

Le cose diventano più interessanti quando provi a eseguire altre operazioni aritmetiche come sottrazione, moltiplicazione o divisione con una stringa e un numero. In questi casi, JavaScript cerca di convertire la stringa in un numero prima di fare il calcolo – un’altra conversione implicita! Ecco cosa succede:

:::interactive_editor

```js
const subtractionResult = '10' - 5;
console.log(subtractionResult); // 5
console.log(typeof subtractionResult); // number

const multiplicationResult = '10' * 2;
console.log(multiplicationResult); // 20
console.log(typeof multiplicationResult); // number

const divisionResult = '20' / 2;
console.log(divisionResult); // 10
console.log(typeof divisionResult); // number
```

:::

Negli esempi sopra, JavaScript converte con successo la stringa `'10'` o `'20'` in un numero e poi esegue il calcolo. Di conseguenza, `'10' - 5` restituisce `5`, `'10' * 2` dà `20` e `'20' / 2` produce `10`.

Ma cosa succede se la stringa non sembra un numero? Vediamo cosa succede in quel caso:

:::interactive_editor

```js
const subtractionResult = 'abc' - 5;
console.log(subtractionResult); // NaN
console.log(typeof subtractionResult); // number

const multiplicationResult = 'abc' * 2;
console.log(multiplicationResult); // NaN
console.log(typeof multiplicationResult); // number

const divisionResult = 'abc' / 2;
console.log(divisionResult); // NaN
console.log(typeof divisionResult); // number
```

:::

Negli esempi sopra, la stringa `'abc'` non rappresenta un valore numerico valido, quindi JavaScript non può convertirla in un numero significativo. Quando questa conversione fallisce, JavaScript restituisce `NaN`, che sta per "Not a Number". `NaN` è un valore speciale del tipo `Number` che rappresenta un numero non valido o non rappresentabile.

Cosa succede se esegui operazioni aritmetiche con un booleano (`true` o `false`)? Vediamo cosa succede. JavaScript tratta i booleani come numeri nelle operazioni matematiche: `true` diventa `1` e `false` diventa `0`.

:::interactive_editor

```js
const result1 = true + 1;
console.log(result1); // 2
console.log(typeof result1); // number

const result2 = false + 1;
console.log(result2); // 1
console.log(typeof result2); // number

const result3 = 'Hello' + true;
console.log(result3); // "Hellotrue"
console.log(typeof result3); // string
```

:::

Nei primi due esempi, `true + 1` ha prodotto `2` e `false + 1` ha prodotto `1`. Nel terzo esempio, `'Hello' + true`, JavaScript ha convertito `true` in una stringa e l’ha concatenata con `'Hello'`, ottenendo `'Hellotrue'`, che è una stringa.

Per `null` e `undefined`, JavaScript tratta `null` come `0` e `undefined` come `NaN` nelle operazioni matematiche:

:::interactive_editor

```js
const result1 = null + 5;
console.log(result1); // 5
console.log(typeof result1); // number

const result2 = undefined + 5;
console.log(result2); // NaN
console.log(typeof result2); // number
```

:::

JavaScript spesso esegue conversioni implicite, convertendo automaticamente tipi di dati come numeri, stringhe e booleani in modi a volte inaspettati. Capire queste conversioni è fondamentale per evitare bug e scrivere codice robusto nei tuoi progetti.
