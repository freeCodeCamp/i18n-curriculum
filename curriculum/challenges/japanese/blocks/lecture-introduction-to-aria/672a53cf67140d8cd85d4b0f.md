# --interactive--

静的なコンテンツをアクセシブルにするのは比較的簡単ですが、動的なコンテンツはより難しい場合があります。ここでWAI-ARIAが役立ちます。

WAI-ARIAとは何か、その目的、仕組み、そしていくつかの例を見てみましょう。

WAI-ARIAはWeb Accessibility Initiative - Accessible Rich Internet Applicationsの略です。動的なコンテンツやUI（ユーザーインターフェース）コンポーネントのアクセシビリティを向上させる仕様です。

WCAGとWAI-ARIAは同じものではないことに注意してください。WCAGはウェブアクセシビリティの一般的なガイドラインを提供し、WAI-ARIAは支援技術のユーザーが動的でインタラクティブなコンテンツにアクセスできるようにするための具体的なルールを提供します。

つまり、WAI-ARIAの主な目的は、ネイティブのHTMLに対応する要素がない動的コンテンツやUIコンポーネントのアクセシビリティを改善することです。

WAI-ARIAは、HTML要素に追加の意味情報を提供する属性のセットを導入することで機能します。これらの属性は役割、状態、プロパティに分類されます。

ARIAの役割は、ウェブサイトやウェブアプリ内の要素の目的を定義します。以下は`button`要素に`div`の役割を設定する例です。

```html
<div role="button">Click Me</div>
```

これにより、その要素がボタンであることを支援技術に示しています。ただし、役割は機能を提供しません。この`div`に`role`の`button`を与えただけではボタンのように動作しません。見た目や動作をボタンのようにするには、CSSとJavaScriptを使って望む結果を得る必要があります。

以下はHTML、CSS、JavaScriptを使ってカスタム`button`要素を作成する例です。

**注意**：CSSとJavaScriptのコードを理解しようと心配しないでください。これらの言語は今後のモジュールで学びます。

:::interactive_editor

```html
<link href="styles.css" rel="stylesheet">
<div id="custom-btn" role="button">Click Me</div>
<script src="index.js"></script>
```

```css
#custom-btn {
  display: inline-block;
  padding: 0.4em 1em;
  font-size: 1rem;
  font-family: sans-serif;
  color: buttontext;
  background-color: buttonface;
  border: 1px solid #ccc;
  border-radius: 4px;
  cursor: pointer;
  user-select: none;
  text-align: center;
}

#custom-btn:focus {
  outline: 2px solid Highlight;
  outline-offset: 2px;
}

#custom-btn:active {
  background-color: #ddd;
}
```

```js
const button = document.getElementById("custom-btn");

button.addEventListener("click", () => {
  alert("Button clicked!");
});

button.addEventListener("keydown", (e) => {
  if (e.key === "Enter" || e.key === " ") {
    e.preventDefault(); 
    button.click();
  }
});
```

:::

ネイティブの`button`や`input`要素に`type="button"`を使うほうが常に望ましいです。

ARIAプロパティは要素に関する追加の詳細を提供します。例えば、`aria-labelledby`プロパティは要素を特定のラベルに関連付けることができます。

```html
<h2 id="header-id">About freeCodeCamp</h2>
<button id="button-id" aria-labelledby="header-id button-id">Learn More</button>
```

これにより、支援技術のユーザーにとって要素が理解しやすく、ナビゲートしやすくなります。

WAI-ARIAを最大限に活用するには、可能な限りネイティブのHTMLを使うことをお勧めします。ネイティブのHTMLは通常、最初からより多くのアクセシビリティを提供します。

HTMLだけでは不十分な場合にのみWAI-ARIAを使い、スクリーンリーダーなどの支援技術でテストしたり、障害のある方にテストしてもらうことを忘れないでください。また、WAI-ARIAの状態やプロパティがコンテンツの変化にリアルタイムで対応して更新されるようにしてください。ARIAの使いすぎは混乱を招くことが多いので避けましょう。
