# --interactive--

Списки описів ідеально підходять для подання термінів і визначень у впорядкованому та легкому для читання форматі, наприклад, у глосарії або справжньому словнику, де можна знайти слова з відповідними визначеннями.

Ось приклад списку описів у HTML з двома термінами та їхніми відповідними деталями.

Увімкніть інтерактивний редактор і спробуйте розкоментувати код, щоб побачити новий елемент деталей у вікні попереднього перегляду.

:::interactive_editor

```html
<dl>
  <dt>HTML</dt>
  <dd>HyperText Markup Language</dd>
  <dt>CSS</dt>
  <dd>Cascading Style Sheets</dd>
  <!-- <dt>JS</dt>
  <dd>JavaScript</dd> -->
</dl>
```

:::

У цьому випадку термінами є абревіатури HTML і CSS, а деталями — їхні розшифровки. Деталі також можуть бути визначеннями або іншою інформацією, пов’язаною з термінами.

Для визначення списку описів вам знадобляться три HTML-елементи. По-перше, елемент списку описів, `dl`, який є контейнером для всього списку. Ви можете побачити, що він охоплює всі інші елементи списку описів у прикладі.

Потім, по одному елементу терміна опису, `dt`, для кожного терміна. У цьому випадку список описів має два терміни, HTML і CSS, тому він містить два таких елементи.

І нарешті, після кожного терміна ви знайдете елемент деталей опису, `dd`, для опису або деталей, пов’язаних із цим терміном. У цьому прикладі це Hypertext Markup Language і Cascading Style Sheets.

У браузері ви побачите кожен термін із відповідним описом. За замовчуванням описи трохи відступають вправо, щоб їх було легше візуально відрізнити.

Але списки описів не обмежуються лише термінами та визначеннями. Вони набагато універсальніші. Тут у нас рецепт із двома інгредієнтами.

Увімкніть інтерактивний редактор і спробуйте розкоментувати код, щоб побачити новий елемент деталей у вікні попереднього перегляду.

:::interactive_editor

```html
<dl>
  <dt>Flour</dt>
  <dd>2 cups</dd>
  <dt>Sugar</dt>
  <dd>1/2 cup</dd>
  <!-- <dt>Vegetable Oil</dt>
  <dd>2 tablespoons</dd> -->
</dl>
```

:::

Весь список описів знаходиться всередині елемента списку описів. Перший інгредієнт, `Flour`, знаходиться в елементі терміна опису. Потім ви можете побачити, скільки цього інгредієнта вам знадобиться: `2 cups`. Це знаходиться в елементі деталей опису безпосередньо після відповідного інгредієнта.

І та сама структура повторюється для `Sugar`. У цьому випадку рецепт має лише два інгредієнти, але якщо б їх було більше, така сама структура могла б повторюватися по всьому списку описів.

У браузері ви побачите інгредієнти вирівняними ліворуч, а вимірювання відступленими, щоб візуально їх розділити.

Інші випадки використання списків описів включають технічні характеристики продуктів, часто задавані питання, контактну інформацію та метадані. По суті, коли у вас є дві пов’язані частини інформації у форматі пари ключ-значення, де одна виступає як мітка, ключ, а інша — як додаткова пов’язана інформація, значення, ви можете використовувати список описів.
