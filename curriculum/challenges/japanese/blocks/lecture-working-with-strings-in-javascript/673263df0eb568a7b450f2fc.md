# --interactive--

JavaScriptで文字列を扱うとき、JavaScriptエンジンが誤解する可能性のある特殊文字を含める必要がある場合があります。

よくある2つの作業は、文字列内に改行を作成することと、特定の文字（例えば引用符）を正しく表示させるためにエスケープすることです。

多くのプログラミング言語、JavaScriptも含めて、文字列内に改行を作成するにはエスケープシーケンスと呼ばれる特殊文字を使います。改行のための最も一般的なエスケープシーケンスは`\n`です。

例えば、文字列を複数行に分けたい場合は、改行を入れたい場所に`\n`を使います。

:::interactive_editor

```js
let poem = "Roses are red,\nViolets are blue,\nJavaScript is fun,\nAnd so are you.";
console.log(poem);
```

:::

`\n`のエスケープシーケンスは、その位置に改行を挿入するようJavaScriptに指示し、結果として文字列が複数行にわたって表示されます。

文字列を扱う際にもう一つ重要な概念は、文字のエスケープです。時には、JavaScriptが別の用途で使う文字（例えば引用符）を文字列に含める必要があります。

もしエスケープせずに文字列内で引用符を使うと、JavaScriptは文字列の終わりと誤解してエラーになります。

例えば、次のコードはエラーになります。

```js
let statement = "She said, "Hello!"";
```

JavaScriptは`"said,"`の後で文字列が終わったと思い混乱しますが、`"Hello!"`の周りの引用符は文字列の一部にしたいのです。

これを直すには、内側の引用符の前にバックスラッシュ（`\`）を置いてエスケープします。

:::interactive_editor

```js
let statement = "She said, \"Hello!\"";
console.log(statement); // She said, "Hello!"
```

:::

バックスラッシュはJavaScriptに引用符を文字通りの文字として扱うよう指示し、正しく表示されます。

他の特殊文字もエスケープできます。例えばバックスラッシュ自体（`\\`）、またはシングルクォートで囲まれた文字列内のシングルクォート（`\'`）などです。

シングルクォートを使った別の例を示します。

:::interactive_editor

```js
let quote = 'It\'s a beautiful day!';
console.log(quote); // It's a beautiful day!
```

:::

シングルクォートを`\'`でエスケープすることで、JavaScriptは文字列の途中で終わらせず、その一部として扱います。

改行の作成と文字のエスケープは、出力のフォーマットや文字列内の特殊文字の処理で不可欠です。これらの手法はエラーを防ぎ、テキストを意図した通りに表示させるのに役立ちます。
