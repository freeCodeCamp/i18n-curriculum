# --description--

Das Divide-and-Conquer-Paradigma in der Informatik ist eine Technik, um Probleme rekursiv in kleinere Teilprobleme zu zerlegen. Ein wesentlicher Aspekt dieser Technik ist die Rekursion, die auftritt, wenn eine Funktion sich selbst wiederholt aufruft, bis ein Basisfall erreicht ist. In dieser Lektion betrachten wir den Merge-Sort-Algorithmus, um besser zu verstehen, wie die Divide-and-Conquer-Technik funktioniert.

Angenommen, wir hätten diese Liste von Zahlen:

```md
42 37 53 17
```

Das Ziel ist es, diese Liste mit dem Merge-Sort-Algorithmus von klein nach groß zu sortieren. Der erste Schritt ist, die Liste in zwei Hälften zu teilen:

```md
42 37 | 53 17
```

Dann schauen wir uns die linke Seite der Liste an:

```md
42 37
```

Wir nehmen diese Teilliste und teilen sie erneut in der Mitte, bis jede Teilliste nur noch ein Element enthält:

```md
42 | 37
```

Eine Liste mit nur einem Element ist standardmäßig sortiert. Als Nächstes müssen wir jede dieser Ein-Element-Teillisten zu einer sortierten Liste zusammenführen:

```md
37 42
```

Dann folgen wir dem gleichen Prozess für die rechte Seite der ursprünglichen Liste:

```javascript
// right side of original list
53 17

// divide the list in half
53 | 17

// merge the lists in sorted order
17 53
```

Jetzt, da beide Hälften der ursprünglichen Liste sortiert sind, führen wir diese beiden Hälften zusammen und sortieren die Elemente:

```markdown
17 37 42 53
```

So sieht der Algorithmus im Code aus:

```javascript
function mergeSort(arr) {
    if (arr.length <= 1) {
        return arr;
    }

    const mid = Math.floor(arr.length / 2);
    const left = mergeSort(arr.slice(0, mid));
    const right = mergeSort(arr.slice(mid));

    const sorted = [];
    let i = 0;
    let j = 0;

    while (i < left.length && j < right.length) {
        if (left[i] <= right[j]) {
            sorted.push(left[i]);
            i += 1;
        } else {
            sorted.push(right[j]);
            j += 1;
        }
    }

    return sorted.concat(left.slice(i)).concat(right.slice(j));
}
```

Die Zeitkomplexität von Merge Sort wäre `O(n log n)`, weil die Liste kontinuierlich halbiert wird (`log n`) und dann zusammengeführt wird (`O(n)`). Im Gegensatz zu anderen Sortieralgorithmen wie Bubble Sort wird Merge Sort nicht in-place sortiert und hat eine Speicherkomplexität von `O(n)`.
