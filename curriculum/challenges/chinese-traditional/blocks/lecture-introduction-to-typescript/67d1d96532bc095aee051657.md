# --description--

JavaScript 被視為動態型別語言。這表示變數可以在執行時接收任何值－將變數宣告為數字型別並不會阻止你之後為它指定字串值，且函式的參數可以傳遞任何值。

動態型別語言的挑戰在於，如果你不小心，缺乏型別安全性可能會引入錯誤。TypeScript 擴充了 JavaScript 語言，加入靜態型別，這有助於在你甚至部署程式碼之前就捕捉那些錯誤。但它是如何運作的？

請考慮這個 JavaScript 函式（程式）：

```js
const getRandomValue = (array) => {
  return array[Math.floor(Math.random() * array.length)];
}
```

我們的 `getRandomValue` 函式（程式）接受一個陣列並從該陣列中傳回一個隨機值。但因為 JavaScript 不會驗證型別，所以沒有任何東西阻止你用數字來呼叫該函式（程式）：

```js
console.log(getRandomValue(10));
```

目前範例的主控台輸出將傳回 `undefined`，因為它預期的是陣列而不是數字。

但是使用 TypeScript 你可以為陣列參數定義型別：

```js
const getRandomValue = (array: string[]) => {
  return array[Math.floor(Math.random() * array.length)];
}
```

這個型別定義告訴 TypeScript 陣列引數必須是字串陣列。然後當你呼叫 `getRandomValue` 並傳入一個數字時，你會得到一種不同的錯誤，稱為編譯器錯誤。

大多數 JavaScript 執行期環境，例如瀏覽器或 Node.js，無法原生執行 `TypeScript`。你必須先將 `TypeScript` 程式碼編譯或轉譯成一般的 JavaScript。你可以使用隨 `TypeScript` 語言附帶的內建編譯器來完成這項工作。當你執行編譯器時，`TypeScript` 會評估你的程式碼，並在型別不匹配的任何問題上發出錯誤——例如將數字傳入期望陣列的函式。

你將在接下來的幾堂課中學習這些型別如何運作。但正是這額外的安全性，使得 TypeScript 成為許多開發者和組織喜愛的語言。
