# --interactive--

JavaScriptは、時に驚くような、あるいは奇妙な動作をする言語です。その一例が、数字と文字列を混ぜて計算するときに起こります。まず最初に試すのは、おそらく数字と文字列を足し合わせることです。JavaScriptでは、`+`演算子は二つの役割を持っています。加算と文字列の連結、つまり二つの文字列をつなげる処理を行います。

数字と文字列に`+`を使うと、JavaScriptは両方を文字列として扱い、連結します。結果の型を`typeof`演算子で確認すると、確かに文字列であることがわかります。

:::interactive_editor

```js
const result = 5 + '10';

console.log(result); // 510
console.log(typeof result); // string
```

:::

`5`と`'10'`の順番を入れ替えたらどうなると思いますか？

:::interactive_editor

```js
const result = '10' + 5;

console.log(result); // 105
console.log(typeof result); // string
```

:::

同じことが起こるのがわかります。JavaScriptは`'10'`を文字列、`5`を数字と見て、数字を文字列に変換して連結します。これを型変換と呼びます。型変換とは、あるデータ型の値を別の型に変換することです。

文字列と数字で引き算、掛け算、割り算などの他の算術演算をしようとすると、さらに面白いことが起こります。この場合、JavaScriptは計算をする前に文字列を数字に変換しようとします。これも型変換の一種です。以下のようになります。

:::interactive_editor

```js
const subtractionResult = '10' - 5;
console.log(subtractionResult); // 5
console.log(typeof subtractionResult); // number

const multiplicationResult = '10' * 2;
console.log(multiplicationResult); // 20
console.log(typeof multiplicationResult); // number

const divisionResult = '20' / 2;
console.log(divisionResult); // 10
console.log(typeof divisionResult); // number
```

:::

上の例では、JavaScriptは文字列`'10'`や`'20'`を数字に変換し、計算を行っています。その結果、`'10' - 5`は`5`を返し、`'10' * 2`は`20`を返し、`'20' / 2`は`10`になります。

では、文字列が数字の形をしていなかったらどうなるでしょうか？その場合を見てみましょう。

:::interactive_editor

```js
const subtractionResult = 'abc' - 5;
console.log(subtractionResult); // NaN
console.log(typeof subtractionResult); // number

const multiplicationResult = 'abc' * 2;
console.log(multiplicationResult); // NaN
console.log(typeof multiplicationResult); // number

const divisionResult = 'abc' / 2;
console.log(divisionResult); // NaN
console.log(typeof divisionResult); // number
```

:::

上の例では、文字列`'abc'`は有効な数値を表していないため、JavaScriptは意味のある数字に変換できません。変換に失敗すると、JavaScriptは`NaN`を返します。これは「Not a Number」を表します。`NaN`は無効または表現不可能な数値を表す`Number`型の特別な値です。

ブール値（`true`や`false`）で算術演算を行うとどうなるでしょうか？見てみましょう。JavaScriptは数学的な演算でブール値を数字として扱います。`true`は`1`になり、`false`は`0`になります。

:::interactive_editor

```js
const result1 = true + 1;
console.log(result1); // 2
console.log(typeof result1); // number

const result2 = false + 1;
console.log(result2); // 1
console.log(typeof result2); // number

const result3 = 'Hello' + true;
console.log(result3); // "Hellotrue"
console.log(typeof result3); // string
```

:::

最初の二つの例では、`true + 1`は`2`になり、`false + 1`は`1`になりました。三つ目の例、`'Hello' + true`では、JavaScriptは`true`を文字列に変換し、`'Hello'`と連結して`'Hellotrue'`という文字列になります。

`null`と`undefined`の場合、JavaScriptは数学的な演算で`null`を`0`として、`undefined`を`NaN`として扱います。

:::interactive_editor

```js
const result1 = null + 5;
console.log(result1); // 5
console.log(typeof result1); // number

const result2 = undefined + 5;
console.log(result2); // NaN
console.log(typeof result2); // number
```

:::

JavaScriptはしばしば型変換を行い、数字、文字列、ブール値などのデータ型を時に予想外の方法で自動的に変換します。これらの変換を理解することは、バグを避けてプロジェクトで堅牢なコードを書くために重要です。
