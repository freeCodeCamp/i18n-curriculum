# --description--

Ti è mai capitato che la tua app Express si blocchi senza un messaggio di errore chiaro? Il debugging e il logging possono aiutarti a vedere esattamente cosa succede dietro le quinte, così puoi trovare e risolvere i problemi più velocemente.

In questa lezione esploreremo come Express gestisce il debugging e il logging e come puoi usarli per migliorare il tuo flusso di sviluppo.

## Debugging integrato con Express

Express include già uno strumento di debugging chiamato `debug`. Non devi installare nulla per usarlo.

Per attivare il debugging, ti basta impostare la variabile d'ambiente `DEBUG` quando avvii la tua app.

Ecco un esempio:

```js
DEBUG=express:* node index.js
```

Questo comando abilita tutti i messaggi di debugging sotto lo spazio dei nomi `express:*`. Vedrai output come:

```js
express:router:route dispatching GET /
express:router:layer processing GET /
```

Questi messaggi ti aiutano a tracciare l'inoltro delle rotte, le chiamate ai middleware e altro ancora. È un ottimo modo per capire il flusso delle richieste nella tua app.

Vuoi essere più specifico? Puoi restringere lo spazio dei nomi:

```js
DEBUG=express:router node index.js
```

Questo mostrerà solo i log relativi alla logica del router di Express.

Su Windows, la sintassi è leggermente diversa per PowerShell:

```js
$env:DEBUG = "express:*"; node index.js
```

E per CMD, è così:

```js
set DEBUG=express:* && node index.js
```

Stessa idea, solo un modo diverso per impostare le variabili d'ambiente.

Ora vediamo il logging in Express.

Per il logging delle richieste HTTP, molti sviluppatori usano middleware come `morgan`.

Ecco come usarlo:

```bash
npm install morgan
```

Poi aggiungilo alla tua app:

```js
const express = require('express')
const morgan = require('morgan')
const app = express()

app.use(morgan('dev'))s
```

Ora, ogni richiesta in arrivo verrà registrata nel terminale con dettagli come metodo, URL, codice di stato e tempo di risposta.

E perché è importante?

Il debugging ti dà una visione di come Express gestisce richieste, middleware e routing senza intasare i log di produzione.

Il logging ti fornisce una registrazione di cosa succede durante le richieste, fondamentale per eseguire il debug dei bug, capire il traffico e individuare problemi precocemente.

Insieme ti danno controllo e chiarezza nel processo di sviluppo.
