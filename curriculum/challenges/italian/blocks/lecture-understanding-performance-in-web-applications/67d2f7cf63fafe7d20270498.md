# --description--

Anche se strumenti come Lighthouse e Chrome DevTools sono ottimi, a volte hai bisogno di approfondimenti personalizzati su misura per il tuo sito.

Ecco dove entrano in gioco le Performance Web API!

In questa lezione, spiegheremo come usare tre Web API fondamentali per misurare e analizzare la velocità del tuo sito web.

Ma prima, cosa sono le Performance Web API?

Le Performance Web API permettono agli sviluppatori di monitorare quanto efficientemente una pagina web si carica e risponde, direttamente dal codice. Queste API ti consentono di misurare i tempi di caricamento della pagina, tracciare i ritardi nel rendering e nelle interazioni e analizzare il tempo di esecuzione di JavaScript.

Con queste API puoi costruire i tuoi strumenti di monitoraggio delle prestazioni senza dipendere da software di terze parti!

Esploriamo tre potenti Web API che puoi usare fin da subito.

Per prima cosa, `performance.now()`.

Questa API ti fornisce timestamp ad alta precisione (in millisecondi) per misurare quanto tempo impiegano diverse parti del tuo sito a caricarsi.

Supponiamo che tu voglia misurare quanto velocemente viene eseguita una funzione:

```js
const start = performance.now();  
// Run some code here  
const end = performance.now();  

console.log(`Execution time: ${end - start}ms`);
```

Questo è più preciso rispetto a usare `Date.now()` perché misura il tempo in millisecondi con precisione al microsecondo, evitando problemi di deriva dell’orologio. Puoi usarlo per tracciare il tempo di esecuzione degli script, i ritardi nelle risposte agli eventi e le prestazioni delle animazioni.

Poi, la Performance Timing API.

Questa API ti dà un’analisi dettagliata di ogni singola fase del caricamento della pagina, dalla ricerca DNS fino a `DOMContentLoaded`.

Vuoi misurare quanto tempo impiega la tua pagina a caricarsi completamente?

```js
let [navigationTiming] = performance.getEntriesByType("navigation");

if (navigationTiming instanceof PerformanceNavigationTiming) {
  // Calculate time from navigation start to DOM content loaded
  const pageLoadTime =
    navigationTiming.domContentLoadedEventEnd - navigationTiming.startTime;

  console.log("DOM Content Loaded Time:", pageLoadTime, "ms");
}
```

Le metriche chiave che puoi tracciare con questa API sono il tempo di ricerca DNS — cioè la velocità di connessione, il Time to First Byte (TTFB) — cioè la velocità di risposta del server, e `DOMContentLoaded` — cioè quando la pagina è pronta per l’interazione.

Se i tempi di caricamento della tua pagina sono lenti, questa API individua esattamente dove si verifica il ritardo!

E infine, parliamo di `PerformanceObserver`.

Questa API ascolta gli eventi di performance come spostamenti di layout, attività lunghe e interazioni dell’utente.

Vuoi monitorare attività JavaScript che durano a lungo?

```js
const observer = new PerformanceObserver((list) => {  
  list.getEntries().forEach((entry) => {  
    console.log(`Long task detected: ${entry.duration}ms`);  
  });  
});  

observer.observe({ type: "longtask", buffered: true });
```

E cosa può tracciare questa API? Può tracciare attività lunghe — cioè JavaScript che blocca il rendering, spostamenti di layout per rilevare problemi nell’interfaccia utente, e il First Input Delay (FID) — cioè quanto velocemente una pagina risponde all’input dell’utente.

Se vuoi un monitoraggio delle prestazioni in tempo reale, questa API è una svolta.

Allora, quale API dovresti usare? Ecco un rapido confronto:

| Performance API        | Ideale per                                         |
| :--------------------- | :------------------------------------------------ |
| `performance.now()`    | Misurazioni precise di funzioni e script          |
| Performance Timing API | Misurare le prestazioni di caricamento completo della pagina |
| Performance Observer   | Monitoraggio in tempo reale di interazioni e rendering |

Combinando queste API, puoi costruire i tuoi strumenti di misurazione delle prestazioni e tracciare esattamente ciò che conta di più per il tuo sito.
