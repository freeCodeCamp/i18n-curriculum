# --description--

У браузері є вбудовані інструменти розробника, які ви можете використовувати для налагодження HTML, CSS і JavaScript.

Однак вони не дуже зручні для пошуку та виправлення помилок у React-застосунках. Тому команда React розробила інструмент під назвою "React Developer Tools" (або React DevTools), щоб ви могли інспектувати, налагоджувати та профілювати React-застосунки.

React DevTools доступний як розширення для браузерів Chrome, Edge і Firefox. Якщо ви користуєтеся Chrome або Edge, перейдіть до Chrome web store, знайдіть "React Developer Tools" і додайте його до свого браузера.

Якщо ви користуєтеся Firefox, перейдіть на сторінку розширень Firefox, знайдіть цей інструмент і додайте його до браузера.

Якщо ви користуєтеся Safari, ви можете встановити React DevTools через npm, виконавши `npm install -g react-devtools` або `yarn global add react-devtools`.

Після встановлення та увімкнення React DevTools, якщо ви відкриєте React-застосунок у браузері, а потім відкриєте інструменти розробника браузера, ви побачите дві додаткові вкладки: Components і Profiler.

Вкладка Components відображає кожен компонент у вигляді дерева. За допомогою неї ви можете:

- Переглядати ієрархію компонентів застосунку.
- Перевіряти та змінювати пропси, стани та значення контексту в реальному часі.
- Переглядати вихідний код для кожного вибраного компонента.
- Логувати дані компонента у консоль.
- Інспектувати DOM-елементи компонента.

Водночас вкладка Profiler допомагає записувати та аналізувати продуктивність компонентів, щоб ви могли виявити непотрібні повторні відтворення, переглянути тривалість комітів і те, що можна оптимізувати.

Ось простий застосунок, щоб показати, як ви можете інспектувати компоненти та їхні пропси й стан. Це схоже на код, який ми використовували в попередньому уроці про передачу пропсів:

```jsx
import { useState } from "react";

export default function App() {
  const greeting = "Hello, Prop Drilling!";
  const response = "I'm not here to play!";

  return <Parent greeting={greeting} response={response} />;
}

const Parent = ({ greeting, response }) => {
  return <Child greeting={greeting} response={response} />;
};

const Child = ({ greeting, response }) => {
  return <Grandchild greeting={greeting} response={response} />;
};

const Grandchild = ({ greeting, response }) => {
  const [count, setCount] = useState(0);

  return (
    <>
      <h1>{greeting}</h1>
      <h2>{response}</h2>

      <button onClick={() => setCount(count + 1)}>Increase Count</button>
      <h2>Count: {count}</h2>
      <button onClick={() => setCount(count - 1)}>Decrease Count</button>
    </>
  );
};
```

Якщо подивитися у вкладку Components у React DevTools, ви побачите дерево компонентів. Компонент `App` знаходиться на верху, за ним ідуть компоненти `Parent`, `Child` і `Grandchild`.

Якщо вибрати будь-який із цих компонентів, можна побачити їхні пропси та стан. Якщо вибрати компонент `Parent`, ви побачите пропси `greeting` і `response`, які відповідно є `Hello, Prop Drilling!` і `I'm not here to play!`.

Ви можете бачити пропси та оновлювати стан у реальному часі, а також змінювати їх за потреби. Наприклад, можна вибрати компонент `Grandchild` і змінити проп `greeting` з `Hello, Prop Drilling!` на `Hello, Welcome to Prop Drilling!`, і це одразу відобразиться на сторінці.

Щоб залогувати дані компонента у консоль, інспектуйте відповідні DOM-елементи та перегляньте вихідний код компонента. Іконки у верхньому правому куті дозволяють це зробити. Якщо вибрати компонент `Grandchild` і натиснути кнопку `Log the component data to the console`, у консолі буде виведено пропси, стан, хуки, вузли та інші дані.

Поширена помилка в React називається невідповідністю пропсів.

Наприклад, уявімо, що для компонента `Child` ви помилково передали проп `reply` замість `response`:

```jsx
const Child = ({ greeting, response }) => {
  return <Grandchild greeting={greeting} reply={response} />;
};
```

Пам’ятайте, що `Grandchild` очікує проп `response`. Оскільки компонент отримує інший проп, він не може відобразити цей текст на сторінці і просто додає порожній `h2` у DOM. Замість цього ви побачите елемент `h1` з текстом `Hello, Prop Drilling!`, а також інші кнопки й текст, які вже є на сторінці. Порожній елемент `h2` все ще присутній, але через те, що він порожній, ви не побачите його без інспектування DOM.

Щоб виправити це, ви можете простежити передачу пропса від компонента `Parent` до `Child` і відредагувати ім’я пропса безпосередньо. Якщо перейти у вкладку Components, вибрати компонент `Child` і змінити проп `reply` на `response`, ви побачите елемент `h2` на сторінці з текстом `I'm not here to play!`.
