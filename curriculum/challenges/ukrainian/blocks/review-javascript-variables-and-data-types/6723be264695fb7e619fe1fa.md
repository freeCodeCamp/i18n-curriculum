# --interactive--

## Робота з HTML, CSS і JavaScript

Поки HTML і CSS забезпечують структуру вебсайту, JavaScript додає інтерактивність, дозволяючи реалізовувати складний функціонал, такий як обробка введення користувача, анімація елементів і навіть створення повноцінних вебзастосунків.

## Типи даних у JavaScript

Типи даних допомагають програмі розуміти, з яким видом даних вона працює — числом, текстом чи чимось іншим.

- **Число**: Число представляє як цілі числа, так і числа з плаваючою комою. Прикладами цілих чисел є 7, 19 і 90. 
- **Число з плаваючою комою**: Число з плаваючою комою — це число з десятковою крапкою. Прикладами є 3.14, 0.5 і 0.0001.
- **Рядок**: Рядок — це послідовність символів або текст, укладений у лапки. `"I like coding"` і `'JavaScript is fun'` — приклади рядків.
- **Булевий**: Булевий тип представляє одне з двох можливих значень: `true` або `false`. Ви можете використовувати булевий тип, щоб представити умову, наприклад `isLoggedIn = true`.
- **Undefined і Null**: Значення `undefined` — це змінна, яка була оголошена, але їй не призначено значення. Значення `null` — це порожнє значення, або змінна, якій навмисно призначено значення `null`.
- **Об’єкт**: Об’єкт — це колекція пар ключ-значення. Ключ — це ім’я властивості, а значення — це значення властивості.

Тут об’єкт `pet` має три властивості або ключі: `name`, `age` і `type`. Відповідні значення — `Fluffy`, `3` і `dog`.

```js
let pet = {
  name: "Fluffy",
  age: 3,
  type: "dog"
};
```

- **Symbol**: Тип даних Symbol — це унікальне та незмінне значення, яке може використовуватися як ідентифікатор для властивостей об’єкта.

У наведеному нижче прикладі створено два символи з однаковим описом, але вони не рівні.

:::interactive_editor

```js
const crypticKey1= Symbol("saltNpepper");
const crypticKey2= Symbol("saltNpepper");
console.log(crypticKey1 === crypticKey2); // false
```

:::

- **BigInt**: Коли число занадто велике для типу даних `Number`, можна використати тип BigInt для представлення цілих чисел довільної довжини.

Додавши `n` у кінець числа, ви можете створити BigInt.

```js
const veryBigNumber = 1234567890123456789012345678901234567890n;
```

## Змінні у JavaScript

- Змінні можна оголосити за допомогою ключового слова `let`.

```js
let cityName;
```

- Щоб призначити значення змінній, можна використати оператор присвоєння `=`.

```js
cityName = "New York";
```

- Змінні, оголошені за допомогою `let`, можна переназначати новим значенням.

:::interactive_editor

```js
let cityName = "New York";
cityName = "Los Angeles";
console.log(cityName); // Los Angeles
```

:::

- Окрім `let`, ви також можете використати `const` для оголошення змінної. Однак змінну `const` не можна переназначати новим значенням.

```js
const cityName = "New York";
cityName = "Los Angeles"; // TypeError: Assignment to constant variable.
```

- Змінні, оголошені за допомогою `const`, використовуються для оголошення констант, які не можна змінювати протягом усього коду, наприклад `PI` або `MAX_SIZE`.

## Конвенції іменування змінних

- Імена змінних мають бути описовими та змістовними.
- Імена змінних мають бути у верблюжому регістрі, як `cityName`, `isLoggedIn` і `veryBigNumber`.
- Імена змінних не повинні починатися з числа. Вони мають починатися з літери, `_` або `$`.
- Імена змінних не повинні містити пробілів або спеціальних символів, окрім `_` і `$`.
- Імена змінних не повинні бути зарезервованими ключовими словами.
- Імена змінних чутливі до регістру. `age` і `Age` — це різні змінні.

## Рядки та їх незмінність у JavaScript

- Рядки — це послідовності символів, укладені в лапки. Їх можна створювати за допомогою одинарних і подвійних лапок.

```js
let correctWay = 'This is a string';
let alsoCorrect = "This is also a string";
```

- Рядки в JavaScript незмінні. Це означає, що після створення рядка ви не можете змінити символи в ньому. Проте ви все ще можете переназначити рядок новим значенням.

```js
let firstName = "John";
firstName = "Jane"; // Reassigning the string to a new value
```

## Конкатенація рядків у JavaScript

- Конкатенація — це процес об’єднання кількох рядків або поєднання рядків із змінними, які містять текст. Оператор `+` — один із найпростіших і найчастіше використовуваних методів конкатенації рядків.

:::interactive_editor

```js
let studentName = "Asad";
let studentAge = 25;
let studentInfo = studentName + " is " + studentAge + " years old.";
console.log(studentInfo); // Asad is 25 years old.
```

:::

- Якщо потрібно додати або приєднати до існуючого рядка, можна використати оператор `+=`. Це корисно, коли ви хочете поступово доповнювати рядок новим текстом.

:::interactive_editor

```js
let message = "Welcome to programming, ";
message += "Asad!";
console.log(message); // Welcome to programming, Asad!
```

:::

- Інший спосіб конкатенації рядків — використання методу `concat()`. Цей метод об’єднує два або більше рядків.

:::interactive_editor

```js
let firstName = "John";
let lastName = "Doe";
let fullName = firstName.concat(" ", lastName);
console.log(fullName); // John Doe
```

:::

## Виведення повідомлень за допомогою `console.log()`

- Метод `console.log()` використовується для виведення повідомлень у консоль. Це корисний інструмент для налагодження і тестування коду.

:::interactive_editor

```js
console.log("Hello, World!");
// Output: Hello, World!
```

:::

## Крапки з комою у JavaScript

- Крапки з комою головним чином використовуються для позначення кінця інструкції. Це допомагає движку JavaScript розуміти розділення окремих інструкцій, що є важливим для правильного виконання.

```js
let message = "Hello, World!"; // first statement ends here
let number = 42; // second statement starts here
```

- Крапки з комою допомагають уникнути неоднозначностей у виконанні коду і гарантують правильне завершення інструкцій.

## Коментарі у JavaScript

- Будь-який рядок коду, який закоментовано, ігнорується движком JavaScript. Коментарі використовують для пояснення коду, нотаток або тимчасового відключення коду.
- Однорядкові коментарі створюються за допомогою `//`.

```js
// This is a single-line comment and will be ignored by the JavaScript engine
```

- Багаторядкові коментарі починаються з `/*` і закінчуються `*/`.

```js
/*
This is a multi-line comment.
It can span multiple lines.
*/
```

## JavaScript як мова з динамічною типізацією

- JavaScript — це мова з динамічною типізацією, що означає, що вам не потрібно вказувати тип даних змінної під час її оголошення. Движок JavaScript автоматично визначає тип даних на основі призначеного значення.

```js
let error = 404; // JavaScript treats error as a number
error = "Not Found"; // JavaScript now treats error as a string
```

- Інші мови, як-от C#, які не є динамічно типізованими, викликали б помилку:

```csharp
int error = 404; // value must always be an integer
error = "Not Found"; // This would cause an error in C#
```

## Використання оператора `typeof`

- Оператор `typeof` використовується для перевірки типу даних змінної. Він повертає рядок, що вказує тип змінної.

:::interactive_editor

```js
let age = 25;
console.log(typeof age); // "number"

let isLoggedIn = true;
console.log(typeof isLoggedIn); // "boolean"
```

:::

- Однак у JavaScript є відома особливість щодо `null`. Оператор `typeof` повертає `"object"` для значень `null`.

:::interactive_editor

```js
let user = null;
console.log(typeof user); // "object"
```

:::
