# --description--

Bis auf die letzte Aufgabe haben Sie Props an zustandslose funktionale Komponenten übergeben. Diese Komponenten verhalten sich wie reine Funktionen. Sie akzeptieren Props als Eingabe und geben jedes Mal dieselbe Ansicht zurück, wenn sie dieselben Props erhalten. Vielleicht fragen Sie sich, was Zustand ist – die nächste Aufgabe behandelt das ausführlicher. Zuvor hier eine Wiederholung der Terminologie für Komponenten.

Eine *zustandslose funktionale Komponente* ist jede Funktion, die Sie schreiben, die Props akzeptiert und JSX zurückgibt. Eine *zustandslose Komponente* hingegen ist eine Klasse, die `React.Component` erweitert, aber keinen internen Zustand verwendet (dies wird in der nächsten Aufgabe behandelt). Schließlich ist eine *zustandsbehaftete Komponente* eine Klassenkomponente, die ihren eigenen internen Zustand verwaltet. Zustandsbehaftete Komponenten werden oft einfach als Komponenten oder React-Komponenten bezeichnet.

Ein gängiges Muster ist es, die Zustandsbehaftung zu minimieren und wann immer möglich zustandslose funktionale Komponenten zu erstellen. Das hilft, Ihr Zustandsmanagement auf einen bestimmten Bereich Ihrer Anwendung zu beschränken. Das wiederum verbessert die Entwicklung und Wartung Ihrer Anwendung, weil es leichter nachvollziehbar macht, wie Änderungen am Zustand das Verhalten beeinflussen.

# --instructions--

Der Code-Editor enthält eine `CampSite`-Komponente, die eine `Camper`-Komponente als Kind rendert. Definieren Sie die `Camper`-Komponente und weisen Sie ihr Standard-Props von `{ name: 'CamperBot' }` zu. Innerhalb der `Camper`-Komponente rendern Sie beliebigen Code, aber stellen Sie sicher, dass ein `p`-Element enthalten ist, das nur den `name`-Wert enthält, der als `prop` übergeben wird. Definieren Sie schließlich `propTypes` an der `Camper`-Komponente, um zu verlangen, dass `name` als Prop bereitgestellt wird, und überprüfen Sie, dass es vom Typ `string` ist.
