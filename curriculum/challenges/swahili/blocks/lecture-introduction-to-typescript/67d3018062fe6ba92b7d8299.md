# --description--

Katika funzo hili, utajifunza jinsi ya kufanya kazi na aina za msingi za msingi na jinsi maelezo ya aina zinavyofanya kazi kwa vigezo.

Kwa aina za data za msingi `string`, `null`, `undefined`, `number`, `boolean`, na `bigint`, TypeScript hutoa maneno muhimu ya aina zinazolingana.

Kwa funzo hili, tutazingatia aina za msingi zinazotumika mara kwa mara na hatutazingatia kwa undani `bigint`. Aina ya data ya `bigint` ni aina ya nambari katika JavaScript inayotumika kuwakilisha nambari kubwa zaidi kuliko aina ya kawaida ya nambari kamili.

Tuchunguze kila moja ya aina hizi za msingi kuanzia na `string`. Hapa ni jinsi unavyoweza kuandika mfuatano wa herufi katika TypeScript:

```ts
let firstName: string = "Angie";
```

Tofauti na JavaScript ya kawaida, kigezo `firstName` kinatarajiwa kuwa na aina ya `string`. Kwa hivyo ikiwa utajaribu kuweka thamani mpya ya nambari, basi utapata kosa kama hili:

```ts
let firstName: string = "Angie";
firstName = 9 // Type 'number' is not assignable to type 'string'.
```

Aina inayofuata tutakayochunguza ni aina ya `number`. Hapa kuna mfano:

```ts
let age: number = 16;
```

Unaweza kuweka nambari kamili na nambari za desimali kwa aina ya `number`. Lakini ikiwa utajaribu kuweka thamani ambayo ni aina ya `bigint`, basi utapata kosa kama hili:

```ts
let age: number = 16;

// Type 'bigint' is not assignable to type 'number'.
age = 123n;
```

Aina inayofuata tutakayochunguza ni aina ya `boolean`. Hapa kuna mifano kadhaa:

```ts
let isLoggedIn: boolean = true;
let isAdmin: boolean = false;
```

Aina mbili za mwisho ni aina za `null` na `undefined`.

Kama unavyokumbuka kutoka kwa mafundisho ya awali ya JavaScript, `null` inawakilisha kutokuwepo kwa makusudi kwa thamani. `undefined` ina maana kigezo kime tangazwa lakini hakijapewa thamani.

Utaona mifano ya jinsi ya kufanya kazi na aina hizi baadaye unapo jifunza kuhusu aina za kitendakazi na aina za muungano.

Sasa baada ya kuona jinsi maelezo ya aina zinavyofanya kazi kwa vigezo, swali linajitokeza kama unahitaji kila mara kuweka aina wazi kwa vigezo vyako.

Ikiwa una maelezo rahisi ya aina kama haya, basi huenda usihitaji kutumia aina wazi.

```ts
let firstName: string = "Angie";
```

Sababu ni kwamba TypeScript itajaribu kubaini aina moja kwa moja kulingana na thamani yake. Kwa hivyo ukitumia hii, TypeScript itafahamu ni mfuatano wa herufi.

```js
let firstName = "Angie";
```

Unapaswa kuweka aina wazi kwa vigezo vyako wakati TypeScript haiwezi kubaini aina kwa uwazi au unapotaka kulazimisha aina maalum kwa hali kama vile vigezo vya kitendakazi au vitu tata.
