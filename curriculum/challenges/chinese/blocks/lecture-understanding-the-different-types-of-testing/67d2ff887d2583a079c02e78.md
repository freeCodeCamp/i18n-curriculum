# --description--

网络是一个强大的地方，充满了丰富的信息和活动。但如果你不小心，它也可能是一个危险的地方。外面有许多黑客和网络犯罪分子在寻找新的方法来破解网站并危及数据。

你可能听说过一些影响数十亿美元公司的高调数据泄露事件，比如 Meta 和 Microsoft。在开发 web 应用时，你需要进行彻底的安全性测试，以帮助识别漏洞和弱点。

在执行安全性审核和测试时，需要注意以下几个原则：

- 保密性：这可防止敏感信息泄露给非预期接收者。

- 完整性：这涉及防止恶意用户修改用户信息。

- 认证：这涉及验证用户的同一性，以确保他们被允许使用该系统。

- 授权：这是确定经过身份验证的用户被允许执行哪些操作或被允许访问系统哪些部分的进程。

- 可用性：这确保授权用户在需要时能够获得信息和服务。

- 不可否认性：这确保发送方和接收方都拥有交付证明和发送方身份的验证。它防止发送方否认已发送该信息。

既然我们已经了解了安全性测试背后的原因，让我们来看几个常见的安全威胁：

- 跨站脚本攻击（XSS）：你在之前的课程中已经了解过这个内容。但作为复习，XSS 攻击发生在攻击者将恶意脚本注入到其他用户查看的网页中时。这些脚本随后可以在受害者浏览器的上下文中执行，可能窃取 cookie 和会话数据，或在用户不知情或未同意的情况下执行其他恶意操作。

- SQL 注入：SQL 是一种用于管理和存储关系数据库中信息的语言。SQL 注入允许恶意用户向数据库注入恶意代码。你将在未来的模块中学习更多关于 SQL、关系数据库和 SQL 注入的内容。

- 拒绝服务（DoS）攻击：这是指恶意用户向网站发送大量请求或流量，导致服务器变慢甚至崩溃，使网站对用户不可用。

安全威胁的类型还有很多，但你可以使用许多工具来帮助保护你的网络应用程序，使其更加安全。以下是安全性测试工具的大致分类：

- 静态应用安全性测试：这些工具用于评估应用的源代码，以识别安全漏洞。

- 动态应用安全性测试（DAST）：这些工具与应用的前端接口，以发现潜在的安全性弱点。与静态应用安全性测试不同，DAST 工具无法访问源代码。

另一种安全性测试类型是渗透测试（简称 pentest）。这涉及对应用进行模拟网络攻击，以识别系统中的任何漏洞。中大型公司可能拥有一支专门的网络安全专业团队，作为其常规安全审计的一部分，定期执行 pentest。

网络安全的世界非常广阔且细致。但每个开发者都应该了解一些针对 web 应用的常见威胁及其防护方法。
