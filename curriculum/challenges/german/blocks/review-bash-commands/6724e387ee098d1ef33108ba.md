## Grundlagen zu Terminal, Shell und Kommandozeile

- **Kommandozeile**: Eine Textschnittstelle, in der Benutzer Befehle eingeben.  
- **Terminal**: Die Anwendung, die Zugriff auf die Kommandozeile bietet.  
- **Terminal-Emulator**: Fügt einem Terminal zusätzliche Funktionen hinzu.  
- **Shell**: Interpretiert die in das Terminal eingegebenen Befehle (z. B. Bash).  
- **PowerShell / Eingabeaufforderung / Microsoft Terminal**: Optionen zum Zugriff auf die Kommandozeile unter Windows.  
- **Terminal (macOS)**: Eingebaute Option auf macOS, mit Drittanbieter-Alternativen wie iTerm oder Ghostty.  
- **Terminal (Linux)**: Optionen variieren je nach Distribution, mit vielen Drittanbieter-Emulatoren wie kitty.  
- **Terminologie**: Obwohl „Terminal“, „Shell“ und „Kommandozeile“ oft synonym verwendet werden, haben sie spezifische Bedeutungen.  


## Tastenkombinationen für die Kommandozeile

- **Pfeil nach oben/unten**: Durch vorherige/nächste Befehle in der Historie blättern.
- **Tab**: Befehle automatisch vervollständigen.
- **`Control+L`** (Linux/macOS) oder Eingabe von `cls` (Windows): Terminalbildschirm löschen.
- **`Control+C`**: Einen laufenden Befehl unterbrechen (wird in PowerShell auch zum Kopieren verwendet, wenn Text markiert ist).
- **`Control+Z`** (nur Linux/macOS): Eine Aufgabe in den Hintergrund schicken; mit `fg` wieder aufnehmen.
- **`!!`**: Den zuletzt ausgeführten Befehl sofort erneut ausführen.

## Grundlagen zu Bash

- **Bash** (Bourne Again Shell): Weit verbreitete Unix-ähnliche Shell.  
  Wichtige Befehle:

  - **`pwd`**: Das aktuelle Verzeichnis anzeigen.
  - **`cd`**: Verzeichnisse wechseln.
    - **`..`** bezeichnet das übergeordnete Verzeichnis (eine Ebene höher).
    - **`.`** bezeichnet das aktuelle Verzeichnis.
  - **`ls`**: Dateien und Ordner auflisten.
    - **`-a`**: Alle Dateien anzeigen, einschließlich versteckter Dateien.
    - **`-l`**: Detaillierte Informationen zu Dateien anzeigen.
  - **`less`**: Dateiinhalte seitenweise mit Navigationsmöglichkeiten anzeigen, inklusive Rückwärtsblättern und Suche.
  - **`more`**: Dateiinhalte bildschirmweise mit eingeschränktem Rückwärtsblättern und einfacher Navigation anzeigen.
  - **`cat`**: Den gesamten Dateiinhalt auf einmal ohne Scrollen oder Navigation anzeigen, nützlich für kleinere Dateien.
  - **`mkdir`**: Ein neues Verzeichnis erstellen.
  - **`rmdir`**: Ein leeres Verzeichnis löschen.
  - **`touch`**: Eine neue Datei erstellen.
  - **`mv`**: Dateien verschieben oder umbenennen.
    - Umbenennen: `mv oldname.txt newname.txt`
    - Verschieben: `mv filename.txt /path/to/target/`
  - **`cp`**: Dateien kopieren.
    - **`-r`**: Verzeichnisse und deren Inhalte rekursiv kopieren.
  - **`rm`**: Dateien löschen.
    - **`-r`**: Verzeichnisse und deren Inhalte rekursiv löschen.
  - **`echo`**: Eine Textzeile oder den Wert einer Variablen anzeigen.
    - Verwenden Sie `>`, um den bestehenden Inhalt einer Datei zu überschreiben (z. B. `echo "text" > file.txt`).
    - Verwenden Sie `>>`, um Ausgabe an eine Datei anzuhängen **ohne den bestehenden Inhalt zu überschreiben** (z. B. `echo "text" >> file.txt`).
  - **`exit`**: Die Terminal-Sitzung beenden.
  - **`clear`**: Den Terminalbildschirm löschen.
  - **`find`**: Nach Dateien und Verzeichnissen suchen.
    - **`-name`**: Nach Dateien anhand eines Namensmusters suchen (z. B. `find . -name "*.txt"`).
  - Verwenden Sie **`man`** gefolgt von einem Befehl (z. B. `man ls`), um detaillierte Handbuch-/Hilfeseiten aufzurufen.

## Befehlsoptionen und Flags

- **Optionen** oder **Flags**: Ändern das Verhalten eines Befehls und sind meist mit Bindestrichen versehen:
  - **Langform (zwei Bindestriche)**:  
    - Beispiel: `--help`, `--version`
    - Werte werden mit einem Gleichheitszeichen angehängt, z. B. `--width=50`.
  - **Kurzform (ein Bindestrich)**:  
    - Beispiel: `-a`, `-l`
    - Werte werden mit einem Leerzeichen übergeben, z. B. `-w 50`.
    - Mehrere Kurzoptionen können zusammengefasst werden, z. B. `ls -alh`.
- **`--help`**: Sie können einen Befehl immer mit diesem Flag verwenden, um die verfügbaren Optionen für jeden Befehl zu verstehen.
