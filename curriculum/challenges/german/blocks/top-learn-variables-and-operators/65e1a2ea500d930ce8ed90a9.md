# --description--

Lernen Sie die Funktionen von JavaScript-Operatoren kennen, die über die Schularithmetik hinausgehen.

Normalerweise summiert der Plus-Operator `+` Zahlen.

Wenn der binäre `+` jedoch auf Strings angewendet wird, führt er sie zusammen (verkettet sie):

```js
let s = "my" + "string";
console.log(s); // mystring
```

Beachten Sie, dass, wenn einer der Operanden ein String ist, der andere ebenfalls in einen String umgewandelt wird.

Zum Beispiel:

```js
console.log(1 + '2'); // '12'
console.log('1' + 2); // '12'
```

Sehen Sie, es spielt keine Rolle, ob der erste Operand ein String oder der zweite ist.

Hier ist ein komplexeres Beispiel:

```js
console.log(2 + 2 + '1') // "41" and not "221"
```

Hier arbeiten die Operatoren nacheinander. Der erste `+` summiert zwei Zahlen, sodass er `4` zurückgibt, dann fügt der nächste `+` den String `1` hinzu, also ist es wie `4 + '1' = '41'`.

```js
console.log('1' + 2 + 2) // "122" and not "14"
```

Hier verkettet der erste `+` den String und die Zahl, sodass er `'12'` zurückgibt, dann fügt der nächste `+` `2` hinzu, also ist es wie `'12' + 2 = '122'`.

Der binäre `+` ist der einzige Operator, der Strings auf diese Weise unterstützt. Andere arithmetische Operatoren arbeiten nur mit Zahlen und wandeln ihre Operanden immer in Zahlen um.

Hier ist die Demo für Subtraktion und Division:

```js
console.log( 6 - '2' ); // 4, converts '2' to a number
console.log( '6' / '2' ); // 3, converts both operands to numbers
```

Angesichts des Verhaltens des `+`-Operators bei gemischten Datentypen in JavaScript, was wird die Ausgabe des folgenden Code-Ausschnitts sein?

```js
console.log('3' + 2 - 1);
```

## --answers--

`31`, weil der `+`-Operator den String und die Zahl vor der Subtraktion verkettet.

---

`32`, weil `+` den String und die Zahl vor der Subtraktion verkettet.

---

`4`, weil alle Operanden zuerst in Zahlen umgewandelt werden.

---
