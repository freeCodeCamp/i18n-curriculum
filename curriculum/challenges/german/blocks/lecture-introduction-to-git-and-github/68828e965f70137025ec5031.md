# --description--

Wenn Sie schon einmal an etwas gearbeitet haben, zum Beispiel an einer Abschlussarbeit, bei der Sie nach größeren Änderungen Sicherungskopien gespeichert und die Dateien etwa `Thesis-Draft`, `Thesis-Final`, `Thesis-Final-For-Real` und `Thesis-Absolute-Final-DO-NOT-CHANGE` genannt haben, dann haben Sie eine sehr rudimentäre Methode der Versionskontrolle verwendet. Aber was, wenn ich Ihnen sage, dass es eine bessere Methode gibt?

Vielleicht haben Sie schon von Tools wie Git, SVN oder Mercurial gehört. Das sind Versionskontrollsysteme, und sie sind ein zentraler Bestandteil des Workflows jedes Entwicklers. Ein Versionskontrollsystem ermöglicht es Ihnen, Änderungen in Ihrem Projekt nachzuverfolgen und zu verwalten. Für die Zwecke dieser Lektionen konzentrieren wir uns auf Git.

Git bietet leistungsstarke Werkzeuge, mit denen Sie die Bearbeitungshistorie eines Projekts verwalten können. Sie werden in den kommenden Lektionen mehr darüber lernen, wie Sie Git und seine Funktionen nutzen. Für den Moment sollten Sie Folgendes wissen:

Git ermöglicht es Ihnen, sogenannte Commits zu erstellen, das sind Schnappschüsse eines bestimmten Zustands Ihrer Codebasis. Sie können dann Commits vergleichen, um zu sehen, welche Änderungen Sie vorgenommen haben, einen Commit mit fehlerhaften Änderungen zurücksetzen und vieles mehr.

Sie können auch „Branches“ erstellen, die Sie sich wie verschiedene Wege in Ihrem Code vorstellen können. Während ein Branch der Weg ist, den Sie gehen, sind Commits die Wegmarken entlang dieses Weges. Branches erlauben es Ihnen, an einer neuen Funktion isoliert zu arbeiten, sodass Sie den Branch löschen können, falls Ihre Arbeit schiefgeht, ohne die Kern-Codebasis zu beeinträchtigen.

Diese Funktionen machen Versionskontrolle zu einem unverzichtbaren Bestandteil der Zusammenarbeit mit anderen Entwicklern. Wenn mehrere Personen am selben Projekt arbeiten, ermöglichen Branches jedem, an eigenem Code zu arbeiten, ohne die Änderungen der anderen zu überschreiben.

Oft kombinieren Sie ein Versionskontrollsystem wie Git mit einem Versionskontrollanbieter wie GitHub oder Gitlab, der Ihnen zusätzliche Kollaborationswerkzeuge und die Möglichkeit bietet, Ihre Projekte in der Cloud zu sichern.

Aber selbst ohne eine cloudbasierte Speicherlösung können Versionskontrollsysteme Ihre Arbeitsabläufe revolutionieren. Keine `THIS-IS-THE-FINAL-COPY-FOREVER`-Dateien mehr.

Es gibt viele spannende Informationen über die Nutzung dieser Systeme, also tauchen wir in den nächsten Lektionen tief ein!
