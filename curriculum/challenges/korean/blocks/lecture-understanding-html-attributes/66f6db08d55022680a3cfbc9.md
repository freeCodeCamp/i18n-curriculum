# --interactive--

HTML은 Hypertext Markup Language의 약자로, 웹 페이지를 만드는 마크업 언어입니다. 웹사이트를 방문해서 단락, 제목, 링크, 이미지, 비디오 같은 콘텐츠를 볼 때, 그것이 바로 HTML입니다.

다음은 HTML 요소를 사용한 작은 예시입니다. 편집기에서 텍스트를 일부 수정해 보고 미리보기 창에서 변경 사항이 어떻게 반영되는지 확인하세요.

:::interactive_editor

```html
<h1>Main heading element</h1>

<p>I am a paragraph element.</p>
```

:::

HTML은 요소를 사용해 웹페이지의 콘텐츠와 구조를 나타냅니다. 대부분의 요소는 여는 태그와 닫는 태그를 가집니다. 때로는 이 태그들을 시작 태그와 끝 태그라고도 부릅니다. 이 두 태그 사이에는 콘텐츠가 들어갑니다. 이 콘텐츠는 텍스트이거나 다른 HTML 요소일 수 있습니다.

다음은 단락 요소의 또 다른 예시입니다. 편집기에서 텍스트를 `I love coding!`로 바꾸고 미리보기 창에서 결과를 확인하세요.

:::interactive_editor

```html
<p>I am a paragraph element.</p>
```

:::

여는 태그와 닫는 태그 모두 왼쪽 꺾쇠 괄호(`<`)로 시작하고 오른쪽 꺾쇠 괄호(`>`)로 끝나며, 태그 이름은 이 괄호 사이에 위치합니다. HTML 태그 이름은 대소문자를 구분하지 않지만, 소문자로 작성하는 것이 널리 받아들여진 관례이자 모범 사례입니다.

여기 여는 단락 태그와 닫는 단락 태그를 자세히 살펴보겠습니다:

```html
<p>
```

```html
</p>
```

여는 태그와 닫는 태그를 구분하는 점은 닫는 태그의 왼쪽 꺾쇠 괄호 바로 다음에 오는 슬래시(`/`)입니다. 일부 HTML 요소는 닫는 태그가 없습니다. 이런 요소를 void 요소라고 합니다.

다음은 void 요소인 이미지 요소의 예시입니다:

```html
<img>
```

이 이미지 요소에는 닫는 태그가 없고 콘텐츠도 없습니다. void 요소는 콘텐츠를 가질 수 없으며 시작 태그만 가집니다.

때때로 void 요소가 `/`를 `>` 앞에 사용하는 것을 볼 수 있습니다:

```html
<img />
```

많은 코드 포매터(예: Prettier)는 void 요소에 `/`를 포함시키는 것을 선택하지만, HTML 명세에서는 `/`의 존재가 "시작 태그를 self-closing으로 표시하지 않으며 불필요하고 아무런 효과도 없다"고 명시합니다.

실제 개발에서는 두 형태 모두 볼 수 있으니 둘 다 익숙해지는 것이 중요합니다.

이미지를 표시하려면 이미지 요소 안에 몇 가지 속성을 포함해야 합니다. 속성은 HTML 요소의 동작을 조정하는 데 사용하는 특별한 값입니다.

다음은 `src` 속성이 포함된 이미지 요소 예시입니다. `src` 속성의 값을 `"https://cdn.freecodecamp.org/curriculum/cat-photo-app/cats.jpg"`으로 바꾸면 두 마리 고양이가 평화롭게 자는 이미지로 바뀌는 것을 볼 수 있습니다.

:::interactive_editor

```html
<img src="https://cdn.freecodecamp.org/curriculum/cat-photo-app/running-cats.jpg" />
```

:::

`src` 속성은 이미지 위치를 지정하는 데 사용됩니다. 이미지 요소에는 `alt`라는 또 다른 속성을 포함하는 것이 좋은 모범 사례입니다. `alt` 속성은 이미지에 대한 짧고 설명적인 텍스트를 제공합니다.

다음은 `src`와 `alt` 속성이 포함된 이미지 요소 예시입니다. `src` 값을 `"https://.freecodecamp.org/curriculum/cat-photo-app/cats.jpg"`으로 바꿔 이미지를 깨뜨려 보세요. 이미지가 사라지고 `alt` 텍스트만 표시되는 것을 볼 수 있습니다.

:::interactive_editor

```html
<img src="https://cdn.freecodecamp.org/curriculum/cat-photo-app/cats.jpg" alt="Two tabby kittens sleeping together on a couch." />
```

:::

그렇다면 HTML만으로 웹사이트를 만들기에 충분할까요? 답은 상황에 따라 다릅니다. 텍스트와 이미지만 표시하는 작은 연습 프로젝트를 만든다면 HTML만으로도 충분할 수 있습니다. 하지만 현대적인 전문 웹사이트를 만든다면 HTML, CSS, JavaScript가 모두 필요합니다.

HTML은 콘텐츠와 구조를 담당합니다. CSS는 스타일링을 담당합니다. JavaScript는 웹 페이지에 상호작용을 추가합니다. 이를 완성된 건물에 비유할 수 있습니다.

HTML은 벽을 구성하는 블록, 콘크리트, 철근을 나타냅니다. 건물을 튼튼하게 만드는 기초입니다. CSS는 집을 아름답게 만드는 내부와 외부 디자인을 나타냅니다. JavaScript는 물과 전기가 끊김 없이 공급되도록 하는 전기 및 수도 시스템을 나타냅니다.
