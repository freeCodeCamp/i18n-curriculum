# --description--

In JavaScript ist ein `Map` ein eingebautes Objekt, das Schlüssel-Wert-Paare speichert, ähnlich wie ein Objekt. Es unterscheidet sich jedoch von Standard-JavaScript-Objekten dadurch, dass Schlüssel beliebigen Typs sein können, einschließlich Objekte und Funktionen.

Ein `WeakMap` ist eine Sammlung von Schlüssel-Wert-Paaren, ähnlich einem `Map`, verwendet jedoch schwache Referenzen für seine Schlüssel. Die Schlüssel müssen Objekte sein, während die Werte beliebigen Typs sein können.

Man kann die Beziehung zwischen einem `WeakMap` und einem `Map` ähnlich sehen wie die Beziehung zwischen einem `WeakSet` und einem `Set`, wie Sie in der vorherigen Lektion gelernt haben.

Um ein `Map` zu erstellen, verwenden Sie den `Map()`-Konstruktor, dem das Schlüsselwort `new` vorangestellt ist:

```js
const myFirstMap = new Map();
```

Sie können die Map mit Werten initialisieren:

```js
const myTreesMap = new Map([
 [{ type: 'deciduous' }, 'Maple tree'],
 [['forest', 'grove'], 'Pine tree'],
 [42, 'Oak tree'],
 [true, 'Birch tree'],
 [function() { return 'I am a function key'; }, 'Willow tree'],
]);
```

Wenn Sie die `Map` nicht mit Werten initialisiert haben, können Sie die `set()`-Methode verwenden, um sie hinzuzufügen:

```js
const myTreesMap = new Map();
myTreesMap.set({ type: 'deciduous' }, 'Maple tree');
myTreesMap.set([1, 2], 'Pine tree');
myTreesMap.set(42, 'Oak tree');
myTreesMap.set(true, 'Birch tree');
myTreesMap.set(function() { return "I'm a function key"; }, 'Willow tree');

console.log(myTreesMap);
```

Weitere Methoden, die Sie verwenden können, um mit einem `Map` zu arbeiten, sind:

- `get(key)`, um den Wert abzurufen, der mit dem angegebenen `key` verknüpft ist.
- `has(key)`, um zu prüfen, ob ein `key` im `Map` existiert.
- `delete(key)`, um ein Schlüssel-Wert-Paar aus dem `Map` zu entfernen.
- `clear()`, um alle Schlüssel-Wert-Paare zu entfernen.
- `entries()`, um die Einträge des `Map` zu prüfen (es gibt die Einträge in einem `MapIterator` zurück).
- `forEach()`, um durch die Einträge des `Map` zu iterieren.
- `size`, um die Anzahl der Schlüssel-Wert-Paare im `Map` anzuzeigen.
    

Es gibt einen `WeakMap()`-Konstruktor, den Sie verwenden können, um ein `WeakMap` zu erstellen:

```js
const myFirstWeakMap = new WeakMap();
```

Die `set()`-, `get()`-, `has()`- und `delete()`-Methoden sind alle auch für ein `WeakMap` verfügbar. Zum Beispiel so können Sie Elemente mit der `Map`-Methode einem `set()` zuweisen:

```js
const myTreeWeakMap = new WeakMap();

myTreeWeakMap.set({ id: 1 }, 'Maple tree');
myTreeWeakMap.set({ id: 2 }, 'Pine tree');
myTreeWeakMap.set({ id: 3 }, 'Oak tree');
myTreeWeakMap.set({ id: 4 }, 'Birch tree');
myTreeWeakMap.set({ id: 5 }, 'Willow tree');

console.log(myTreeWeakMap);
```

So sieht ein `WeakMap` in der Konsole aus:

```js
/*
WeakMap {{…} => 'Willow tree', {…} => 'Maple tree', {…} => 'Pine tree', {…} => 'Oak tree'}
  [[Entries]]
    No properties
  [[Prototype]]: WeakMap
*/
```

Hier sind die Unterschiede zwischen einem `Map` und einem `WeakMap`:

| **Funktion** | **Map** | **WeakMap** |
| --- | --- | --- |
| **Schlüsseltyp** | Schlüssel können jeden Datentyp haben, einschließlich Strings, Zahlen, Objekte oder sogar Funktionen. | Schlüssel müssen Objekte sein. |
| **Anwendungsfall** | Verwenden Sie eine `Map`, wenn Sie Daten mit beliebigen Schlüsseln verknüpfen müssen. | Verwenden Sie eine `WeakMap`, wenn Sie Daten nur mit Objekten verknüpfen müssen. |
| **Iteration** | Sie können mit `Map`, `forEach()`, `keys()` oder `values()` durch eine `entries()` iterieren. | Eine `WeakMap` ist nicht iterierbar. |
| **Größe-Eigenschaft** | `Map` hat eine `size`-Eigenschaft, um die Anzahl der Schlüssel-Wert-Paare zu erhalten. | `WeakMap` hat keine `size`-Eigenschaft. |

## --text--

Was unterscheidet in JavaScript eine `Map` von einer `WeakMap`?

## --answers--

Eine `Map` erlaubt nur Objekte als Schlüssel, während eine `WeakMap` Schlüssel beliebigen Typs erlaubt.
