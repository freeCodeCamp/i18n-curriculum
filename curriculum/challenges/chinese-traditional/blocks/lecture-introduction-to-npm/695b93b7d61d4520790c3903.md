# --description--

當你在 JavaScript 中建置專案時，你經常需要額外的工具來幫助你完成工作。即使是簡單的應用程式，也可能需要像是儲存資料、登入、路由以在頁面間切換等功能。

你可以使用其他開發者已經建立並分享的開源軟體包，而不是從頭撰寫所有這些程式碼。

這就是 npm 派上用場的地方。npm 是一個龐大的開放原始碼軟體線上登錄檔。把它想像成程式碼的應用程式商店——你可以搜尋你需要的東西，安裝它，並開始在你的專案中使用它。

許多人認為 npm 代表「Node Package Manager」，但那並不是它的官方含義。在較舊的 npm 文件中，它被描述為「npm 不是縮寫」的遞迴反向縮寫。換句話說，npm 並沒有官方代表任何意義。

今天，npm 取用三個相關的零件、部分：

* 網站 npmjs.com，你可以在登錄檔中查詢軟體包。
    
* npm 登錄檔是一個大型公共資料庫，儲存所有的軟體包。
    
* 隨 Node.js 安裝的 npm CLI 工具，可讓你安裝和管理軟體包。
    

例如，如果你需要管理狀態、控制代碼日期、發送電子郵件或在你的專案中進行 HTTP 請求，可能已經有一個 npm 軟體包可以做到這些。只要使用簡單的 `npm install` 指令，你就可以開始使用這個現成的軟體，而不必自己完成所有工作。

npm 也幫助你管理你的專案相依性。這表示它會追蹤你在專案中使用的軟體包，以及每個軟體包的版本。它透過 `package.json` 和 `package-lock.json` 檔案來做到這點，你會在未來的課程中學習這些檔案。

Yarn、PNPM 和 Bun 是一些隨著時間變得流行的 npm 替代方案。雖然它們各自在安裝速度和儲存效率等特定領域提供利益，但 npm 仍然是最廣泛使用的。

在接下來的課程中，你將學習各種 npm 指令、如何安裝軟體包、關於 `package.json` 和 `package-lock.json` 檔案，甚至如何將你自己的軟體包發佈到 npm 登錄檔。
