# --description--

配列はコンピュータサイエンスにおける基本的なデータ構造です。すべての配列は順序付けられたデータの集合を格納しますが、その種類によって内部の動作が異なる場合があります。

配列の基盤となる動作はプログラムの効率に重要な影響を与えるため、動的配列と静的配列の違いについて学び、プログラムに最も効率的なものを選べるようにしましょう。

まずは静的配列から始めます。

**静的配列**はサイズが固定されています。要素は隣接したメモリ領域に格納されます。

静的配列のサイズは配列が初期化されるときに決まります。一度特定のメモリ領域が割り当てられると、それは固定され、プログラムの実行中に変更できません。これが静的配列の重要な特徴です。

要素を隣接したメモリ領域に格納することで、プログラムは最初の要素の場所を記憶し、インデックスを使って簡単な計算を行い、他の要素をメモリ内で見つけることができるため、データの取得が効率的になります。

このため、静的配列の値にアクセスする操作は定数時間 `O(1)` で行われ、とても効率的です。

格納する要素の数があらかじめわかっている場合は静的配列を使うことができます。また、アクセス操作が非常に効率的なので、値に頻繁にアクセスする場合にも役立ちます。

しかし、このデータ構造はサイズを増減できないため、格納する要素の数が変動する可能性がある場合は動的配列を使うべきです。

静的配列のサイズを増やそうとすると、新しい配列を作成し、古い配列のすべての要素を新しい配列にコピーする必要があり、効率が悪くなります。その場合は、この処理を自動で行う動的配列のほうがずっと適しています。

Pythonには伝統的な静的配列は組み込みのデータ構造として含まれていません。

しかし、Javaのような他のプログラミング言語ではサポートされています。以下は3つの整数を格納できるJavaの静的配列の例です。

```java
int[] numbers = new int[3]; 
```

Pythonの配列は動的なので、そちらを見てみましょう。

**動的配列**はプログラムの実行中に自動でサイズを増減できるため、より柔軟です。

動的配列は自動リサイズ機構によって動作し、元の配列がいっぱいになると要素を新しい配列にコピーします。この処理は、新しい配列のサイズを効率的に選ぶことで、計算コストの高い操作が頻繁に起こらないようにしています。

動的配列の要素にアクセスする操作は定数時間 `O(1)` で行われるため、とても効率的です。

配列の途中に要素を挿入する操作は線形時間 `O(n)` かかります。なぜなら、その後の要素を移動させる必要があるからです。

配列の末尾に要素を挿入する操作は、動的配列にまだ空きがある場合は定数時間 `O(1)` で行われますが、配列がいっぱいでリサイズが必要な場合は `O(n)` の計算量になります。

格納する値の数があらかじめわからない場合は動的配列を使うべきです。また、頻繁に要素の挿入や削除を行う場合にも役立ちます。

Pythonの組み込み `list` データ構造は動的配列として機能します。リストは大括弧内に要素をカンマで区切って書くことで作成できます。

```python
numbers = [3, 4, 5, 6]
```

リストの要素には、リストを保持する変数名の後に大括弧を書き、その中に対応するインデックスを指定してアクセスできます。

インデックスは最初の要素が0から始まり、次の要素ごとに1ずつ増えます。

```python
numbers[0]  # 3
numbers[1]  # 4
numbers[2]  # 5
numbers[3]  # 6
```

値を更新するには、単に再代入すればよいです。

```python
numbers[2] = 16
```

`.append()` メソッドを使ってリストに要素を追加できます。

```python
numbers.append(7)
```

`.insert()` メソッドを使うと、特定のインデックスに要素を挿入できます。最初の引数にインデックス、2番目の引数に挿入する要素を渡します。

```python
numbers.insert(3, 15)
```

`.pop()` メソッドを使うと、特定のインデックスの要素を削除できます。

```python
numbers.pop(2)
```

インデックスを指定しない場合、`.pop()` は最後の要素を削除します。

他にも、要素の追加や削除を簡単に行える組み込みのリストメソッドがドキュメントにあります。

これが動的配列、つまりこの場合はリストの力です。

一般的に、要素数があらかじめわかっていて頻繁にアクセスする場合は静的配列を使い、要素数が不明または変動する場合は動的配列を使うべきです。

静的配列のシンプルさと動的配列の柔軟さのトレードオフを常に考慮してください。どちらも特定のユースケースやシナリオで役立ちます。問題に応じて最適なものを選べることは、練習を重ねることで徐々に身につく問題解決スキルの一部です。
