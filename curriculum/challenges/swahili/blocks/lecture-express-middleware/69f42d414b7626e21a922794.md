# --description--

Middleware ya kushughulikia makosa katika Express ni aina maalum ya programu ya kati inayotumika kushughulikia makosa yanayotokea wakati wa mzunguko wa ombi-jibu. Ni muhimu kwa kugundua makosa yasiyotegemewa, kuyarekodi, na kutoa majibu yanayofaa kwa mteja.

Kama ulivyoona katika funzo la awali kuhusu kushughulikia makosa, middleware ya kushughulikia makosa ya Express ina vigezo vinne: `err`, `req`, `res`, na `next`. Middleware hii huanzishwa wakati kosa linapotupwa kwa kitendakazi cha `next()`.

Hapa kuna ukumbusho wa sintaksia tena:

```js
app.use((err, req, res, next) => {
  console.error(err.message);
  res.status(500).send("Internal Server Error");
});
```

Na hivi ndivyo inavyoonekana katika programu rahisi ya Express:

```js
const express = require('express')
const app = express()

// A simple route that throws an error
app.get('/', (req, res) => {
  throw new Error('Something went wrong!')
})

// Error-handling middleware
app.use((err, req, res, next) => {
  console.error(err.stack); // Logs the error stack
  res.status(500).send('Something went wrong!')
})

app.listen(3000, () => {
  console.log('Server running on http://localhost:3000')
})
```

* Mshughulikiaji wa njia ya data: Mshughulikiaji wa njia ya data hutupa kosa kwa makusudi kuiga tatizo.
* Middleware ya kushughulikia makosa: Middleware hii imesajiliwa na `app.use()`. Inakamata makosa yanayotupwa kupitia `next(err)` na kutuma jibu la HTTP 500 kwa mteja lenye ujumbe wa kosa wa jumla.
* `err`: Huu ni kitu cha kosa kinachopitishwa kwa middleware.

Middleware ya kushughulikia makosa huhakikisha kwamba wakati kitu kinapokosea katika programu yako, watumiaji bado wanapata jibu sahihi. Ni muhimu kila mara kujumuisha middleware hii mwishoni mwa safu yako ya middleware ili kukamata makosa yoyote.
