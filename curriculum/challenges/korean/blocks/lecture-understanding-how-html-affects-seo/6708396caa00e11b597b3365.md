# --description--

오픈 그래프 프로토콜은 Facebook, LinkedIn 등 다양한 소셜 미디어 플랫폼에서 웹사이트 콘텐츠가 어떻게 보일지 제어할 수 있게 해줍니다. 이러한 오픈 그래프 속성을 설정하면 사용자가 콘텐츠를 클릭하고 참여하도록 유도할 수 있습니다. HTML `meta` 섹션 안에 있는 `head` 요소 모음을 통해 이러한 속성을 설정할 수 있습니다.

포함해야 할 첫 번째 중요한 OG 속성은 `title`입니다. 다음은 freeCodeCamp 홈페이지에 대해 OG `title`를 설정하는 예시입니다:

```html
<meta content="freeCodeCamp.org" property="og:title" />
```

`property` 속성에는 `og:title`임을 지정해야 합니다. `content` 속성에는 소셜 미디어 사이트에 표시할 제목을 작성합니다.

다음으로 중요한 OG 속성은 `type`입니다. 다음은 freeCodeCamp 홈페이지에 대해 OG `type`를 사용하는 예시입니다:

```html
<meta property="og:type" content="website" />
```

`type` 속성은 소셜 미디어에서 공유되는 콘텐츠 유형을 나타내는 데 사용됩니다. 이 콘텐츠 유형에는 기사, 웹사이트, 비디오 또는 음악 등이 포함됩니다.

세 번째 중요한 OG 속성은 `image`입니다. 다음은 freeCodeCamp 홈페이지에 대해 OG `image`를 설정하는 예시입니다:

```html
<meta
  content="https://cdn.freecodecamp.org/platform/universal/fcc_meta_1920X1080-indigo.png"
  property="og:image"
/>
```

이 예시에서 오픈 그래프 이미지는 freeCodeCamp 로고를 가리키고 있습니다. 모든 이미지는 고품질이어야 하며 적절한 크기와 비율을 가져야 합니다. 대부분의 소셜 미디어 플랫폼은 이미지 요구 사항에 대한 기준을 제공하여 콘텐츠가 사이트에서 잘 표시되도록 돕습니다. 예를 들어 developers.facebook.com 문서 페이지에서는 다음과 같이 명시합니다:

"고해상도 기기에서 최상의 표시를 위해 최소 1200 x 630 픽셀 크기의 이미지를 사용하세요. 최소한 600 x 315 픽셀 크기의 이미지를 사용하여 링크 페이지 게시물에 더 큰 이미지를 표시해야 합니다."

네 번째 중요한 OG 속성은 `url`입니다. 다음은 freeCodeCamp 홈페이지에 대해 OG `url`를 설정하는 예시입니다:

```html
<meta property="og:url" content="https://www.freecodecamp.org" />
```

`description`, `audio`, `video`, `locale` 같은 더 많은 OG 속성을 설정할 수 있습니다. 하지만 오픈 그래프 `url`, `image`, `type`, `title`가 가장 중요한 속성입니다.

그렇다면 이러한 오픈 그래프 속성은 검색 엔진 최적화에 어떻게 영향을 미칠까요? 콘텐츠가 소셜 미디어에서 공유될 때, 잘 만들어진 OG 속성은 사용자의 피드에서 콘텐츠의 외관을 향상시킵니다. 이는 클릭률 증가로 이어질 수 있으며, 검색 엔진에 콘텐츠가 관련성 있고 흥미롭다는 신호를 보낼 수 있습니다.
