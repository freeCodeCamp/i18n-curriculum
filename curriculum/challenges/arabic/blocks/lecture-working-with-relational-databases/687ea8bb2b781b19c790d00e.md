# --description--

في SQL، تُستخدم المفاتيح الأساسية والمفاتيح الخارجية لإنشاء علاقات بين الجداول.

لنبدأ بالمفاتيح الأساسية.

المفتاح الأساسي هو عمود أو مجموعة أعمدة تُميز كل سجل (صف) في جدول بشكل فريد. هذا القيد يضمن ألا يكون هناك سجلات في الجدول لها نفس القيمة للمفتاح الأساسي.

لهذا السبب يمكن أن يحتوي الجدول على مفتاح أساسي واحد فقط.

القيم في عمود المفتاح الأساسي لا يمكن أن تكون `NULL` أيضًا، لذا سيكون لها دائمًا قيمة صالحة.

في PostgreSQL، لجعل عمود هو المفتاح الأساسي لجدولك، فقط أضف `PRIMARY KEY` بعد نوع البيانات:

```sql
column_name data_type PRIMARY KEY
```

هذا مثال على جدول `students`:

```sql
CREATE TABLE students (
  student_id SERIAL PRIMARY KEY,
  name VARCHAR(100)
);
```

كل طالب سيكون له `student_id` خاص به، وهذه المعرف سيكون المفتاح الأساسي للجدول. نوع `SERIAL` مفيد هنا لأنه يضمن أن يكون لـ `student_id` قيمة فريدة دائمًا.

المفتاح الأساسي المركب يُستخدم عندما لا يحتوي الجدول على عمود فريد واحد لتحديد الصف. في هذه الحالة، يمكنك استخدام مجموعة من عمودين أو أكثر كمفتاح أساسي يكون فريدًا معًا.

للقيام بذلك، حدد الأعمدة التي تشكل المفتاح الأساسي المركب عند إنشاء الجدول هكذا:

```sql
CREATE TABLE table_name (
  column1 data_type column_constraint,
  column2 data_type column_constraint,
  column3 data_type column_constraint,
  ...
  PRIMARY KEY (column1, column2)
);
```

تخيل أن لديك جدولًا يحتوي على عمودين `student_id` و `course_id` يخبرك بالطلاب المسجلين في مختلف الدروس.

كل طالب سيكون مسجلاً في عدة دروس، وكل درس سيكون لديه عدة طلاب مسجلين فيه. لذا لا يكون أي من العمودين فريدًا - لكنك لن تجد الطالب نفسه مسجلاً في نفس الدرس أكثر من مرة. لذلك يمكنك استخدام تركيبة العمودين كمفتاح أساسي:

```sql
CREATE TABLE course_enrollments (
  student_id INT,
  course_id INT,
  ...
  PRIMARY KEY (student_id, course_id),
);
```

المفاتيح الأساسية ضرورية لضمان تفرد البيانات وسلامتها، لتحسين استرجاع البيانات، ولإنشاء علاقات مع جداول أخرى في قاعدة بيانات علائقية.

هذا يقودنا إلى نوع المفتاح التالي الذي سنغطيه في هذا الدرس: المفاتيح الخارجية.

المفتاح الخارجي هو عمود (أو مجموعة أعمدة) في جدول يشير إلى المفتاح الأساسي في جدول آخر. يمكن أن يحتوي الجدول على عدة مفاتيح خارجية.

في هذا المثال، لدينا جدولان، `customers` و `orders`:

```sql
CREATE TABLE customers (
  customer_id SERIAL PRIMARY KEY,
  first_name VARCHAR(100) NOT NULL,
  ...
);

CREATE TABLE orders (
  order_id SERIAL PRIMARY KEY,
  customer_id INTEGER,
  ...
  FOREIGN KEY (customer_id) REFERENCES customers(customer_id)
);
```

المفتاح الأساسي لجدول `customers` هو `customer_id` والمفتاح الأساسي لجدول `orders` هو `order_id`.

لكن لاحظ أن هناك أيضًا عمود `customer_id` في جدول `orders`.

لماذا نخزن معلومات العميل في جدول `orders`؟

نفعل ذلك لإنشاء علاقة بين جدولَي `customers` و `orders`. كل طلب سيخزن معرف العميل الذي أرسله.

لإنشاء العلاقة بين الجدولين، سنجعل عمود `customer_id` في جدول `orders` مفتاحًا خارجيًا يشير إلى عمود `customer_id` في جدول `customers`:

```sql
FOREIGN KEY (customer_id) REFERENCES customers(customer_id)
```

يجب أن تتطابق القيم في عمود المفتاح الخارجي إما مع قيم المفتاح الأساسي في الجدول المشار إليه أو تكون `NULL` (إذا سمح المفتاح الخارجي بـ `NULL`).

لماذا هذا مفيد؟

بالتأكد من تطابق هذه الأعمدة في كلا الجدولين تمامًا، تمنع المفاتيح الخارجية والأساسية إنشاء سجلات "يتيمة"، وهي سجلات تشير إلى سجل غير موجود في جدول آخر.

المفاتيح الأساسية والخارجية أساسية لقواعد البيانات العلائقية. فهي تدعم الجانب "العلائقي" لهذه القواعد، مما يسمح لك بنمذجة البيانات الواقعية بدقة.
