# --description--

當你使用 CSS 時，可能會遇到某些挑戰，例如保持樣式在各瀏覽器間的一致性和相容性、避免重複，以及處理專案中日益增加的 CSS 檔案的複雜度和數量。CSS 框架非常有助於從一開始就防止這些問題。

CSS 框架由預先撰寫的 CSS 程式碼組成，你可以用來為你的 HTML 元素設計樣式。

透過使用 CSS 框架，你可以加快你的工作流程，為整個網站創建統一的視覺風格，使你的設計在多個瀏覽器中看起來一致，並且整體上讓你的 CSS 程式碼更有條理。

你可以將 CSS 框架視為一個工具箱，在那裡你可以快速找到符合你需求的樣式或組件並立即使用，而不必自己使用 CSS 屬性來指定。這些框架通常包含預先定義的樣式和組件，你可以非常輕鬆地使用，包括用於建立響應式版面的樣式。

一些最受歡迎的 CSS 框架有 Tailwind CSS、Bootstrap、Materialize 和 Foundation。

讓我們稍微談談兩種 CSS 框架的型別：以工具為先的 CSS 框架，以及以組件為基礎的 CSS 框架。

Tailwind CSS 是一個以工具為先的 CSS 框架。這表示它有許多具有非常特定用途的小類別，例如設定邊距、內距或背景顏色。你通常可以直接將這些小類別指定到 HTML 元素。

例如：

```html
<button class="bg-blue-500 text-white font-bold py-2 px-4 rounded-full hover:bg-blue-700">
  Button
</button>
```

這個按鈕將會有藍色背景、白色且粗體的文字、垂直 2 rem 的內距、水平 4 rem 的內距，以及圓角邊框。當使用者用滑鼠暫留時，類別 `hover:bg-blue-700` 將會使背景變成深藍色。

如你所見，Tailwind CSS 類別非常細緻，因此可以根據需要套用到單一 HTML 元素。

相較之下，Bootstrap 是一個基於組件的 CSS 框架。它擁有預先建置且帶有預定義樣式的組件，你可以輕鬆地將這些組件添加到你的網站。這些組件通常會出現在 CSS 框架的官方文件中，你可以快速地將它們複製並貼上到你的專案中。

例如，如果你想在 Bootstrap 中創建一個列表群組，你會在 HTML 中寫類似這樣的內容：

```html
<div class="card" style="width: 25rem;">
  <ul class="list-group list-group-flush">
    <li class="list-group-item">HTML</li>
    <li class="list-group-item">CSS</li>
    <li class="list-group-item">JavaScript</li>
  </ul>
</div>
```

這些預定義的類別將自動套用標準樣式到卡片、列表群組和列表群組項目。

請注意，這裡我們不是添加小型且通用的類別，而是添加整個組件，包括 HTML 結構。這就是為什麼 Bootstrap 是基於組件的原因。

```html
<div class="card" style="width: 25rem;">
  <ul class="list-group list-group-flush">
    <li class="list-group-item">HTML</li>
    <li class="list-group-item">CSS</li>
    <li class="list-group-item">JavaScript</li>
  </ul>
</div>
```

雖然 CSS 框架有許多優點，但它們也有某些缺點。你必須了解兩者，才能做出明智的決定。

由於 CSS 框架依賴預先定義的樣式，如果你嘗試在其上添加訂製的 CSS，可能會產生特異性問題。框架使用的選擇器可能具有比你在訂製 CSS 中使用的選擇器更高的特異性，因此正確的樣式可能無法顯示。為了解決這個問題，你需要使用更具特異性的 CSS 選擇器，這可能會帶來一些額外的挑戰。

它們也可能在某種程度上限制獨特性，因為它們的標準樣式被廣泛用於許多網站。然而，這可以透過使用你獨特的樣式來訂製 CSS 來解決。 

最後，你應該考慮 CSS 框架可能帶來的潛在效率問題。有些框架相當龐大，可能需要一些時間來載入。

作為開發者，你將決定使用 CSS 框架是否對你的專案有幫助，使用完全訂製的 CSS 是否更好，或是兩者的混合。
