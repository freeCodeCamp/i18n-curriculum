# --description--

Ein Pfad ist ein String, der den Speicherort einer Datei oder eines Verzeichnisses in einem Dateisystem angibt. In der Webentwicklung ermöglichen Pfade Entwicklern, Ressourcen wie Bilder, Stylesheets, Skripte und andere Webseiten zu verknüpfen. Es gibt absolute und relative Pfade – beide sind wichtig, um Dateistandorte innerhalb eines Dateisystems anzugeben. Schauen wir uns die beiden an, damit Sie entscheiden können, welchen Sie wann verwenden sollten.

Ein absoluter Pfad ist ein vollständiger Link zu einer Ressource. Er beginnt beim Stammverzeichnis, enthält jedes weitere Verzeichnis und schließlich den Dateinamen und die Erweiterung. Das „Stammverzeichnis“ bezeichnet das oberste Verzeichnis oder den Ordner in einer Hierarchie.

Wenn Sie auf eine Ressource auf Ihrem lokalen Rechner verlinken, verwenden Sie einen absoluten Pfad, der den vollständigen Verzeichnispfad der Datei enthält. So verlinken Sie mit einem absoluten Pfad auf die `about.html`-Datei:

```html
<p>
  Read more on the
  <a
    href="/Users/user/Desktop/fCC/script-code/absolute-vs-relative-paths/pages/about.html"
    >About Page</a
    >
</p>
```

Es sieht so aus, weil wir vom Stammverzeichnis starten und in einen Ordner namens `Users` gehen, dann in einen Ordner namens `user`, dann in einen Ordner namens `Desktop`, dann in einen Ordner namens `fCC`, dann in einen Ordner namens `script-code`, dann in einen Ordner namens `absolute-vs-relative-paths`, dann in einen Ordner namens `pages`, um schließlich die `about.html`-Datei zu erreichen.

Eine absolute URL ist eine vollständige Adresse, die zum Zugriff auf eine Ressource verwendet wird. Sie enthält das Protokoll – das `http`, `https` oder `file` sein kann – und den Domainnamen, wenn sich die Ressource im Web befindet. Hier ist ein Beispiel für eine absolute URL, die auf das freeCodeCamp-Logo verlinkt:

```html
<a href="https://design-style-guide.freecodecamp.org/img/fcc_secondary_small.svg">
  View fCC Logo
</a>
```

In diesem Beispiel ist das Protokoll `https`, der Domainname `design-style-guide.freecodecamp.org` und der Dateiname `fcc_secondary_small.svg`.

So sieht die absolute URL in der Adressleiste des Browsers aus:

```sh
file:///Users/user/Desktop/fCC/script-code/absolute-vs-relative-paths/pages/about.html
```

Die URL enthält das Protokoll `file://`. Sie enthält auch den Pfad, der so aussieht: `/Users/user/Desktop/fCC/script-code/absolute-vs-relative-paths/pages/`, und die Reihe von Ordnern darstellt, die zur Datei führen. Und schließlich enthält sie auch den `about.html`, also den Dateinamen und die Erweiterung.

Ein absoluter Pfad zeigt den vollständigen Speicherort einer Datei innerhalb eines Dateisystems an und wird häufig für Ressourcen auf einem lokalen Rechner verwendet. Eine absolute URL enthält Zugriffsinformationen – wie das Protokoll und für Webressourcen den Domainnamen –, die dem Browser mitteilen, wie und wo die Ressource abgerufen wird.

Schauen wir uns nun den relativen Pfad an. Ein relativer Pfad gibt den Speicherort einer Datei relativ zum Verzeichnis der aktuellen Datei an. Er enthält nicht das Protokoll oder den Domainnamen, wodurch er kürzer und flexibler für interne Verknüpfungen innerhalb derselben Webseite ist. Hier ist ein Beispiel, wie Sie von der `about.html`-Seite auf die `contact.html`-Seite verlinken, die sich beide im selben Ordner befinden:

```html
<p>
  Read more on the
  <a href="about.html">About Page</a>
</p>
```

Stellen Sie sich vor, Sie befinden sich auf der `contact.html`-Seite, und da sich die `about.html`-Seite am selben Ort befindet, geben Sie einfach den Dateinamen an. Dies ist ein Beispiel für die Verwendung eines relativen Dateipfads.

Welche sollten Sie also wann verwenden: einen absoluten Pfad, eine absolute URL oder einen relativen Pfad? Hier sind die Regeln, die Sie befolgen sollten:

- Verwenden Sie absolute Pfade, wenn Sie auf eine Ressource von einem festen Speicherort aus verweisen möchten, etwa vom Stammverzeichnis Ihrer Seite oder einem bekannten Verzeichnis auf Ihrem lokalen Rechner.

- Verwenden Sie absolute URLs, wenn Sie auf eine Ressource verlinken, die auf einer externen Webseite gehostet wird.

- Verwenden Sie relative Pfade, wenn Sie auf Ressourcen innerhalb derselben Webseite verlinken.

- Verwenden Sie relative Pfade, wenn Sie Ihren Code während der Entwicklung sauberer und leichter wartbar halten möchten.

- Verwenden Sie relative Pfade bei lokalen Tests, um sicherzustellen, dass Verknüpfungen auch ohne Internetverbindung funktionieren.
