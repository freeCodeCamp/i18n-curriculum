# --description--

في الدرس السابق، تعلمت كيف تُشغّل سكريبتات Python محليًا. لكن قد تكون هناك أوقات لا تريد فيها إنشاء برامج Python كاملة وتحتاج فقط لاختبار بعض شفرة Python. هنا يأتي دور القشرة التفاعلية في Python.

لكن أولًا، دعنا نراجع ما هو الطرفية.

الطرفية هي واجهة نصية تتيح لك التفاعل مع الكمبيوتر عبر كتابة الأوامر. كل نظام تشغيل يأتي مع تطبيق طرفية افتراضي. على macOS، يمكنك استخدام تطبيق Terminal. على Windows، يمكنك استخدام Command Prompt أو PowerShell. على Linux، لكل بيئة سطح مكتب تطبيق طرفية افتراضي خاص بها، مثل GNOME Terminal أو Konsole.

افتح تطبيق الطرفية واكتب `python` ثم اضغط `Enter`. هذا سيبدأ القشرة التفاعلية لـ Python. القشرة التفاعلية هي برنامج يتيح لك كتابة الأوامر واحدة تلو الأخرى ورؤية النتائج.

عندما تبدأ جلسة جديدة، قد ترى هذا النوع من المخرجات في البداية:

```bash
Python 3.12.2 (main, Mar 21 2024, 22:48:26) [Clang 14.0.3 (clang-1403.0.22.14.1)] on darwin
Type "help", "copyright", "credits" or "license" for more information.
>>> 
```

رمز `>>>` يعني أن Python ينتظر منك كتابة أمر. جرّب طباعة `"Hello, world!"` في الطرفية:

```bash
print("Hello, world!")
```

يجب أن ترى النص يظهر هكذا:

```bash
>>> print("Hello, world!")
Hello, world!
```

بعد طباعة النص، يعود Python إلى الشكل التالي:

```bash
>>>
```

هذا يعني أنه يمكنك كتابة أمر آخر.

مفسر Python يتبع ما يُعرف بدورة القراءة، التقييم، الطباعة، التكرار. أو REPL اختصارًا. كلما كتبت أوامر، يقرأ المفسر المُدخَل، يُقيّمه، يطبع النتيجة، ثم يعود ليعرض `>>>` لتكتب أوامر أخرى.

ماذا يحدث إذا حاولت كتابة أمر غير صالح مثل هذا؟

```bash
>>> something random
```

حسنًا، مفسر Python سيظل يتبع نفس عملية REPL. في هذه الحالة، ستحصل على رسالة خطأ:

```bash
>>> something random
  File "<stdin>", line 1
    something random
              ^^^^^^
SyntaxError: invalid syntax
```

إذا أردت الخروج من القشرة التفاعلية، يمكنك كتابة `exit()` أو الضغط على `Ctrl + D` في (Mac/Linux) أو `Ctrl + Z + Enter` في (Windows).

استخدام القشرة التفاعلية في Python رائع للتجارب الصغيرة مع الشفرة أو الاستكشاف الأساسي. أثناء تقدمك في الدروس المتبقية في الدورة، يمكنك كتابة بعض الطرق الجديدة التي تعلمتها داخل القشرة التفاعلية.

إذا كنت تخطط للعمل على برامج أطول أو عبر ملفات متعددة، فمن الأفضل العمل في محرر شفرة أو بيئة تطوير متكاملة (IDE).
