¿Qué son los cuantificadores de regex y cómo funcionan?

Aprendamos sobre cuantificadores en expresiones regulares.

Considera un escenario donde desees coincidir un código de identificación de cuatro dígitos. Sabes que puedes usar la clase de caracteres `\d`, así que podrías escribir eso cuatro veces. Y para evitar cualquier carácter extraño, incluirías tanto el inicio como los anclajes de fin:

```js
const regex = /^\d\d\d\d$/;
```

Y esto funciona, coincidirá con cuatro caracteres numéricos. Pero en lugar de tener que escribir la misma clase múltiples veces, puedes darle un cuantificador.

Los cuantificadores están definidos por llaves que contienen uno o dos números. Usemos un cuantificador en nuestro patrón:

```js
const regex = /^\d{4}$/;
```

Observa cómo nuestro cuantificador solo contiene el número `4`. Esta sintaxis significa "coincidir el carácter anterior exactamente cuatro veces". Veamos cómo se comporta:

```js
const regex = /^\d{4}$/;
console.log(regex.test("123")); // false
console.log(regex.test("1234")); // true
console.log(regex.test("12345")); // false
console.log(regex.test("123456")); // false
console.log(regex.test("1234567")); // false
```

El patrón solo coincide con la cadena de exactamente cuatro dígitos, porque hemos usado los anclajes y nuestro cuantificador solo permite exactamente cuatro dígitos. Pero tal vez el código de identificación solo necesita ser un mínimo de cuatro dígitos.

Para permitir cuatro o más dígitos, agrega una coma después del número en tu cuantificador:

```js
const regex = /^\d{4,}$/;
```

Ahora nuestra sintaxis permite que el patrón coincida con cuatro o más dígitos. Probémoslo:

```js
const regex = /^\d{4,}$/;
console.log(regex.test("123")); // false
console.log(regex.test("1234")); // true
console.log(regex.test("12345")); // true
console.log(regex.test("123456")); // true
console.log(regex.test("1234567")); // true
```

Un identificador de siete dígitos es bastante largo. Estos identificadores deben tener un máximo de 6 dígitos y un mínimo de 4 dígitos. Para lograr esto, puedes agregar un segundo número a tu cuantificador después de la coma:

```js
const regex = /^\d{4,6}$/;
```

Y ahora nuestro patrón ya no coincide con el identificador de siete dígitos, porque es mayor que nuestro máximo de seis dígitos.

```js
const regex = /^\d{4,6}$/;
console.log(regex.test("123")); // false
console.log(regex.test("1234")); // true
console.log(regex.test("12345")); // true
console.log(regex.test("123456")); // true
console.log(regex.test("1234567")); // false
```

Nota que no puedes usar esta sintaxis para establecer solo un máximo, siempre debes establecer un mínimo. Pero si estableces el mínimo en `1`, puedes lograr efectivamente el mismo resultado.

Hemos recibido requisitos actualizados de nuestros usuarios. Ahora los identificadores pueden comenzar opcionalmente con una letra. Ya conocemos la clase de caracteres para esto, así que agreguémosla a nuestra expresión regular:

```js
const regex = /^[a-zA-z]\d{4,6}$/;
```

Pero ahora mandamos la presencia de una letra. ¿Cómo podemos hacerlo opcional?

Podrías usar la sintaxis del cuantificador con `0` como mínimo y `1` como máximo:

```js
const regex = /^[a-zA-Z]{0,1}\d{4,6}$/;
```

Pero en realidad hay un cuantificador abreviado especial para un único carácter opcional – el signo de interrogación (`?`). Reemplacemos nuestro cuantificador con el signo de interrogación:

```js
const regex = /^[a-zA-Z]?\d{4,6}$/;
```

Deberíamos validar el resultado:

```js
const regex = /^[a-zA-Z]?\d{4,6}$/;
console.log(regex.test("123")); // false
console.log(regex.test("a1234")); // true
console.log(regex.test("12345")); // true
console.log(regex.test("az12345")); // false
console.log(regex.test("X123456")); // true
console.log(regex.test("1234567")); // false
```

Nuestro patrón ahora permite una única letra opcional, seguida de cuatro a seis dígitos.

Desafortunadamente, acabamos de darnos cuenta de que leímos mal los requisitos. Debemos permitir cualquier cantidad de letras antes de los números. Podemos usar nuestro cuantificador con `0` como mínimo y sin un máximo definido:

```js
const regex = /^[a-zA-Z]{0,}\d{4,6}$/;
```

Pero nuestro patrón se está alargando de nuevo. Afortunadamente, hay otro abreviado para "coincidir el carácter anterior cero o más veces" – el símbolo asterisco (`*`). Reemplacemos nuestro cuantificador con eso en el patrón, y probémoslo:

```js
const regex = /^[a-zA-Z]*\d{4,6}$/;
console.log(regex.test("123")); // false
console.log(regex.test("a1234")); // true
console.log(regex.test("12345")); // true
console.log(regex.test("az12345")); // true
console.log(regex.test("X123456")); // true
console.log(regex.test("1234567")); // false
```

Ahora coincidimos correctamente con cualquier identificador con cero o más letras seguidas de cuatro a seis números. Pero resulta que esto está provocando un fallo en nuestro sistema – de hecho tenemos que requerir al menos una letra.

Nuevamente, podríamos usar un cuantificador con un mínimo de uno y sin un máximo definido, o podríamos usar otra sintaxis especial: el símbolo más (`+`):

```js
const regex = /^[a-zA-Z]+\d{4,6}$/;
console.log(regex.test("123")); // false
console.log(regex.test("a1234")); // true
console.log(regex.test("12345")); // false
console.log(regex.test("az12345")); // true
console.log(regex.test("X123456")); // true
console.log(regex.test("1234567")); // false
```

Ahora los identificadores que no empiezan con al menos una letra fallan, sin importar cuántos números haya.

Puedes usar cuantificadores para mejorar enormemente la concisión y legibilidad de tus expresiones regulares.
