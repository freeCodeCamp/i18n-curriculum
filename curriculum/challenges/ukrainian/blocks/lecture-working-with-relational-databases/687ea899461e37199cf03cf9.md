# --description--

Щоб визначити таблицю в SQL, потрібно вказати тип даних для кожного стовпця.

Пам’ятайте, що це базовий синтаксис для створення таблиці в SQL. Кожен стовпець має свій тип даних.

```sql
CREATE TABLE table_name(
  column1 data_type column_constraint,
  column2 data_type column_constraint,
  column3 data_type column_constraint,
  ... etc
);
```

Як розробник, ваша мета — вибрати ці типи даних якомога точніше.

Існує шість популярних категорій типів даних у SQL:

- Числові, такі як `INTEGER`, `FLOAT`, `SERIAL` і `DECIMAL`.
- Дата і час, такі як `TIMESTAMP`, `DATE` і `TIME`.
- Символьні та рядкові. До них належать `CHAR`, `VARCHAR` і `TEXT`.
- Юнікод, включно з `NTEXT` і `NVARCHAR`. Вони використовуються, щоб гарантувати правильне збереження та отримання тексту незалежно від походження символів.
- Двійкові, які використовують для збереження нетекстових даних, як-от зображення, аудіо та відеофайли.
- І інші різноманітні типи даних, такі як `XML` і `TABLE`.

У цьому уроці ми розглянемо деякі з найпоширеніших типів даних у PostgreSQL. Типи даних можуть відрізнятися в різних системах керування базами даних, але загалом вони досить схожі. Тож вам потрібно буде перевірити повний список у документації.

Почнемо з числових значень.

Спершу маємо тип даних `INTEGER`:

```sql
units_sold INTEGER
```

У цьому прикладі ми призначаємо цей тип даних стовпцю `units_sold`.

Типи `INTEGER` займають 4 байти в базі даних і можуть мати значення від -2,147,483,648 до 2,147,483,647. Офіційна документація PostgreSQL описує його як «типовий вибір для цілих чисел».

Також є `SMALLINT` і `BIGINT`, які фактично такі ж, як `INTEGER`, але мають менший і більший діапазон чисел відповідно через обсяг пам’яті, який їм виділено в базі даних.

Корисною функцією в PostgreSQL є створення стовпця за допомогою ключового слова `SERIAL`. Тут створюється стовпець `id` із використанням `SERIAL`:

```sql
id SERIAL
```

Хоч це й не справжній тип даних, він дуже корисний для створення стовпців унікальних ідентифікаторів, оскільки стовпець матиме тип `INTEGER`, не дозволятиме `NULL` значень і автоматично збільшуватиметься при додаванні рядків.

Наприклад, перший рядок автоматично матиме `id` зі значенням `1`, другий рядок матиме `id` зі значенням `2` і так далі, створюючи унікальний ідентифікатор для кожного запису.

У MySQL еквівалентом `SERIAL` буде атрибут `AUTO_INCREMENT`, який використовується для автоматичного генерування послідовних цілих чисел:

```sql
id INT AUTO_INCREMENT
```

Ви можете побачити, що різні системи керування базами даних можуть мати різні способи досягнення точно такої самої функціональності у вашій базі даних.

Це найпоширеніші числові типи даних, але іноді вам може знадобитися представити текст або послідовності символів.

Для цього існує тип даних `VARCHAR`. Цей тип даних використовується для рядків змінної довжини. Ви можете встановити максимальну довжину символів у круглих дужках:

```sql
name VARCHAR(50)
```

У цьому прикладі ми визначаємо стовпець `name`, де значення можуть бути довжиною до 50 символів.

Цей тип даних встановлює максимальну довжину рядків, але якщо потрібно зберігати рядки будь-якої довжини, можна замість цього використати `TEXT`:

```sql
name TEXT
```

Окрім чисел і рядків, також часто зберігають дати та час.

Наприклад, якщо ви створюєте таблицю для збереження подій, вам може знадобитися зберігати дату кожної події у певному форматі. Для цього є тип даних `DATE`:

```sql
event_date DATE
```

Для збереження часу також можна використати тип даних `TIME`. Наприклад, ви можете зберігати час початку події:

```sql
start_time TIME
```

Тип даних `TIMESTAMP` поєднує обидва — дату і час. Він також може включати часовий пояс, якщо ви напишете `TIMESTAMP WITH TIME ZONE`:

```sql
event_timestamp TIMESTAMP
```

```sql
event_timestamp TIMESTAMP WITH TIME ZONE
```

І нарешті, якщо потрібно зберегти булеве значення, `TRUE` або `FALSE`, можна використати тип даних `BOOLEAN`:

```sql
is_active BOOLEAN
```

Тепер ви знаєте деякі з найпоширеніших типів даних у SQL, але їх набагато більше. Ви можете перевірити документацію вашої системи керування базами даних, щоб дізнатися більше про типи даних, які можна використовувати.
