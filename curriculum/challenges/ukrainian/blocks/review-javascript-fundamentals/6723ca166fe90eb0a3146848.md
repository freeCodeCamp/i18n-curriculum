## Конструктор рядків і метод `toString()`

- **Визначення**: об’єкт рядка використовується для представлення послідовності символів. Об’єкти рядків створюються за допомогою функції конструктора `String`, яка обгортає примітивне значення в об’єкт.

:::interactive_editor

```js
const greetingObject = new String("Hello, world!");

console.log(typeof greetingObject); // "object"
```

:::

- **Метод `toString()`**: цей метод перетворює значення у його рядкове представлення. Це метод, який можна використовувати для чисел, булевих значень, масивів і об’єктів.

:::interactive_editor

```js
const num = 10;
console.log(num.toString()); // "10"

const arr = [1, 2, 3];
console.log(arr.toString()); // "1,2,3"
```

:::

Цей метод приймає необов’язковий префікс, який є числом від 2 до 36. Цей префікс представляє основу, наприклад, основу 2 для двійкової системи або основу 8 для вісімкової. Якщо префікс не вказано, за замовчуванням використовується основа 10, тобто десяткова.

:::interactive_editor

```js
const num = 10;
console.log(num.toString(2)); // "1010"(binary)
```

:::

## Конструктор числа

- **Визначення**: конструктор `Number` використовується для створення об’єкта числа. Об’єкт числа містить кілька корисних властивостей і методів, таких як методи `isNaN` і `toFixed`. Здебільшого ви будете використовувати конструктор `Number` для перетворення інших типів даних у тип числа.

:::interactive_editor

```js
const myNum = new Number("34");
console.log(typeof myNum); // "object"

const num = Number('100');
console.log(num); // 100

console.log(typeof num); // number
```

:::

## Найкращі практики іменування змінних і функцій

- **camelCasing**: за конвенцією розробники JavaScript використовують camel casing для іменування змінних і функцій. Camel casing — це коли перше слово повністю в нижньому регістрі, а наступні слова починаються з великої літери. Наприклад, `isLoading`.

- **Іменування булевих змінних**: для булевих змінних загальною практикою є використання префіксів, таких як "is", "has" або "can".

```js
let isLoading = true;
let hasPermission = false;
let canEdit = true;
```

- **Іменування функцій**: для функцій ім’я має чітко вказувати, що робить функція. Для функцій, які повертають булеве значення (часто їх називають предикатами), можна використовувати ті ж префікси "is", "has" або "can". Коли функції отримують дані, зазвичай починають ім’я з "get". Коли функції встановлюють дані, зазвичай починають ім’я з "set". Для обробників подій можна додавати префікс "handle" або суфікс "Handler".

```js
function getUserData() { /* ... */ }

function isValidEmail(email) { /* ... */ }

function getProductDetails(productId) { /* ... */ }

function setUserPreferences(preferences) { /* ... */ }

function handleClick() { /* ... */ }
```

- **Іменування змінних у циклах**: при іменуванні змінних-ітераторів у циклах зазвичай використовують поодинокі літери, такі як `i`, `j` або `k`.

```js
for (let i = 0; i < array.length; i++) { /* ... */ }
```

## Робота з розрідженими масивами

- **Визначення**: можливо мати масиви з порожніми слотами. Порожні слоти — це слоти, у яких нічого немає. Це відрізняється від слотів масиву зі значенням `undefined`. Такі масиви називають розрідженими масивами.

:::interactive_editor

```js
const sparseArray = [1, , , 4];
console.log(sparseArray.length); // 4
```

:::

## Лінтери та форматери

- **Лінтери**: лінтер — це інструмент статичного аналізу коду, який позначає помилки програмування, баги, стилістичні помилки та підозрілі конструкції. Прикладом загального лінтера є ESLint.
- **Форматери**: форматери — це інструменти, які автоматично форматують ваш код відповідно до певного стилю. Прикладом загального форматера є Prettier.

## Керування пам’яттю

- **Визначення**: керування пам’яттю — це процес контролю пам’яті, виділення її за потребою та звільнення, коли вона більше не потрібна. JavaScript використовує автоматичне керування пам’яттю. Це означає, що JavaScript (точніше, движок JavaScript у вашому веб-браузері) самостійно піклується про виділення та звільнення пам’яті. Вам не потрібно явно звільняти пам’ять у вашому коді. Цей автоматичний процес часто називають "збиранням сміття".

## Замикання

- **Визначення**: замикання — це функція, яка має доступ до змінних у зовнішній (обгортаючій) лексичній області видимості, навіть після того, як зовнішня функція повернула результат.

:::interactive_editor

```js
function outerFunction(x) {
  let y = 10;
  function innerFunction() {
    console.log(x + y);
  }
  return innerFunction;
}

let closure = outerFunction(5);
closure(); // 15
```

:::

## Ключове слово `var` і підняття

- **Визначення**: `var` був оригінальним способом оголошення змінних до 2015 року. Але з `var` виникли деякі проблеми, пов’язані з областю видимості, повторним оголошенням та іншими аспектами. Саме тому сучасне програмування на JavaScript використовує `let` і `const` замість нього.
- **Повторне оголошення змінних з `var`**: якщо ви спробуєте повторно оголосити змінну за допомогою `let`, отримаєте `SyntaxError`. Але з `var` повторне оголошення змінної дозволене.

```js
// Uncaught SyntaxError: Identifier 'num' has already been declared 
let num = 19;
let num = 18;

var myNum = 5;
var myNum = 10; // This is allowed and doesn't throw an error

console.log(myNum) // 10
```

- **`var` і область видимості**: змінні, оголошені з `var` всередині блоку (наприклад, інструкції `if` або циклу `for`), залишаються доступними поза межами цього блоку.

:::interactive_editor

```js
if (true) {
  var num = 5;
}
console.log(num); // 5
```

:::

- **Підняття**: це стандартна поведінка JavaScript, коли оголошення піднімаються на початок відповідної області видимості під час фази компіляції перед виконанням коду. Коли ви оголошуєте змінну за допомогою ключового слова `var`, JavaScript піднімає оголошення на початок її області видимості.

:::interactive_editor

```js
console.log(num); // undefined
var num = 5;
console.log(num); // 5
```

:::

Коли ви оголошуєте функцію за допомогою синтаксису оголошення функції, і ім’я функції, і тіло функції піднімаються. Це означає, що ви можете викликати функцію до того, як оголосите її у своєму коді.

:::interactive_editor

```js
sayHello(); // "Hello, World!"

function sayHello() {
  console.log("Hello, World!");
}
```

:::

Оголошення змінних за допомогою `let` або `const` піднімаються, але вони не ініціалізовані, і ви не можете отримати до них доступ до фактичного оголошення у вашому коді. Цю поведінку часто називають "тимчасовою мертвою зоною".

:::interactive_editor

```js
console.log(num); // Throws a ReferenceError
let num = 10;
```

:::

## Робота з імпортами, експортами та модулями

- **Модуль**: це автономна одиниця коду, яка інкапсулює пов’язані функції, класи або змінні. Щоб створити модуль, напишіть свій JavaScript-код у окремому файлі.
- **Експорти**: будь-які змінні, функції або класи, які ви хочете зробити доступними для інших частин вашого застосунку, потрібно явно експортувати за допомогою ключового слова `export`. Існують два типи експорту: іменований експорт і експорт за замовчуванням.
- **Імпорти**: щоб використовувати експортовані елементи в іншій частині вашого застосунку, потрібно імпортувати їх за допомогою ключового слова `import`. Типи імпортів можуть бути іменованим імпортом, імпортом за замовчуванням і імпортом простору імен.

```js
// Within a file called math.js, we export the following functions:

// Named export
export function add(num1, num2) {
  return num1 + num2;
}

// Default export
export default function subtract(num1, num2) {
  return num1 - num2;
}

// Within another file, we can import the functions from math.js.

// Named import - This line imports the add function.
// The name of the function must exactly match the one exported from math.js.
import { add } from './math.js';

// Default import - This line imports the subtract function.
// The name of the function can be anything.
import subtractFunc from './math.js';

// Namespace import - This line imports everything from the file.
import * as Math from './math.js';

console.log(add(5, 3)); // 8
console.log(subtractFunc(5, 3)); // 2
console.log(Math.add(5, 3)); // 8
console.log(Math.subtract(5, 3)); // 2
```
