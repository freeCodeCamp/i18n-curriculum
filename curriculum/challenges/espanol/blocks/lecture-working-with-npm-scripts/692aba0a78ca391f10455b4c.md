# --description--

Ahora que entiendes el sistema de módulos CommonJS, veamos el sistema de módulos ES.

Los módulos ES, o módulos ECMAScript, son el formato de módulo oficial y estandarizado para JavaScript definido por la especificación ECMAScript. Usan las declaraciones `import` y `export`, y en Node.js, se cargan de forma asíncrona, lo que es más eficiente.

Node.js trata los archivos como módulos ES en los siguientes casos:

* Archivos con una extensión `.mjs`.
    
* Y archivos con una extensión `.js` cuando el archivo `package.json` padre más cercano contiene un campo de nivel superior `"type"` con un valor de `"module"`.
    

Aquí tenemos la misma función `multiply` que has visto antes, pero ahora vamos a usar el sistema ESM. Para esto, estamos asignando una extensión `.mjs` a ambos módulos.

Para exportar la función `multiply` del módulo `math.mjs`, usa la palabra clave `export` seguida del nombre de la función, variable u objeto que quieres exportar dentro de llaves:

```javascript
function multiply(a, b) {
  return a * b;
}

export { multiply };
```

También puedes exportar la función directamente usando la palabra clave `export` antes de la palabra clave `function`. Esto se conoce como una exportación nombrada. Puedes tener múltiples exportaciones nombradas por módulo:

```javascript
export function multiply(a, b) {
  return a * b;
}
```

Luego, para importar este módulo, usa la palabra clave `import` seguida de llaves. Observa que estamos usando `import` en lugar de `require()`. Esa es una de las diferencias clave entre CommonJS y los módulos ES.

Dentro de las llaves, escribimos el nombre de la función, variable u objeto que queremos importar. En este caso, es `multiply`. Luego, usa la palabra clave `from` y la ruta relativa al módulo. Este código se ejecutará correctamente y la salida será `42`.

```javascript
import { multiply } from './math.mjs';

console.log(multiply(6, 7)); // 42
```

En el sistema ES, puedes establecer una exportación predeterminada por módulo. De esta manera, no necesitas usar llaves para importarla.

Aquí hay un ejemplo donde estamos estableciendo la función `multiply` como la exportación predeterminada para todo el módulo `math.mjs`:

```javascript
export default function multiply(a, b) {
  return a * b;
}
```

Ahora no necesitamos las llaves, ya que la función será importada y asignada por defecto:

```javascript
import multiply from './math.mjs';
```

Si no usas extensiones de archivo ni el archivo `package.json` para indicarle explícitamente a Node.js qué tipo de sistema de módulos quieres usar, inspeccionará el código fuente para buscar la sintaxis de módulos ES. Si se encuentra esa sintaxis, ejecutará el código como un módulo ES; de lo contrario, lo ejecutará como un módulo CommonJS.

Aunque CommonJS sirvió como el estándar predeterminado para Node.js en sus primeros días, a medida que los desarrolladores adoptan cada vez más ESM por su eficiencia y compatibilidad con el JavaScript moderno, es importante conocer sus diferencias y cómo usar ambos.
