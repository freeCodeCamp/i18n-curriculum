# --interactive--

静的プロパティとメソッドは、クラスの個々のインスタンスではなく、クラス自体に属します。クラスのインスタンスを作成せずに、クラス名で直接アクセスできます。関連する機能をカプセル化するためにクラス内で定義されます。

静的メソッドは、メソッド名の前に`static`キーワードを書くことで定義できます。

```js
class MyClass {
  static staticMethod() { ... }
}
```

その後、ドット記法を使ってクラス上で直接静的メソッドを呼び出し、必要な引数を渡せます。

```js
MyClass.staticMethod();
```

インスタンスを作成せずにメソッドを呼び出せることに注目してください。これが静的メソッドの重要な特徴の一つです。

例を見てみましょう。映画のアプリケーションを作成していて、評価に基づいて映画を比較できるようにしたいとします。

この比較メソッドは、特定の映画に限定されないより上位のメソッドと考えられます。

```js
if (movieA.rating < movieB.rating) {
  console.log(`${movieB.title} has a higher rating.`);
} 
```

これは`Movie`クラスに関連するより一般的なメソッドのようなものです。

可読性と保守性のために、関連するメソッドを比較的近くにまとめるために`Movie`クラス内に定義するのが望ましいです。

したがって、これは静的メソッドに最適な候補です。コンストラクタのすぐ下に定義されているのがわかります。

:::interactive_editor

```js
class Movie {

  constructor(title, rating) {
    this.title = title;
    this.rating = rating;
  }

  static compareMovies(movieA, movieB) {
    if (movieA.rating > movieB.rating) {
      console.log(`${movieA.title} has a higher rating.`);
    } else if (movieA.rating < movieB.rating) {
      console.log(`${movieB.title} has a higher rating.`);
    } else {
      console.log("These movies have the same rating.");
    }
  }

}

let movieA = new Movie("Movie A", 80);
let movieB = new Movie("Movie B", 45);
console.log(movieA);
```

:::

静的メソッドは`static`キーワードで定義され、名前は`compareMovies`です。パラメータは2つあり、`movieA`と`movieB`です。これらは`Movie`クラスのインスタンスになります。

評価は`0`から`100`の範囲で比較されます。このロジックは条件文で実装されており、どちらの映画の評価が高いかに基づいて適切なメッセージを出力します。

メソッドが定義されたら、クラス上で呼び出せます。ただし、まずプログラム内で引数を用意して定義する必要があります。

この場合、メソッドは2つの映画インスタンスを引数として受け取るので、ここでそれらのインスタンスを定義します。

```js
let movieA = new Movie("Movie A", 80);
let movieB = new Movie("Movie B", 45);
```

`movieA`の評価が`movieB`より高いことがわかります。このメソッドの出力を見てみましょう。

メソッドを呼び出すには、クラス自体にドット記法を使います。
クラス名（`Movie`）を書き、ドットを付けてから静的メソッド名（`compareMovies`）を書きます。

その後、丸括弧内に引数を渡します。この場合、メソッドが必要とする2つの映画インスタンスです。

```js
Movie.compareMovies(movieA, movieB);
```

更新された例はこちらです。

:::interactive_editor

```js

    this.title = title;

  }
```

静的メソッドは「ファクトリ」メソッドの実装にも役立ちます。ファクトリメソッドは、特定の条件に基づいてオブジェクトを作成するためにコンストラクタに加えて定義するメソッドです。

`Pizza`クラスの例を見てみましょう。静的メソッド`createMargherita`は、タイプと価格がすでに設定されたマルゲリータピザのインスタンスを作成するために呼び出せるファクトリメソッドです。

```js
class Pizza {
  constructor(type, price) {
    this.type = type;
    this.price = price;
  }

  static createMargherita() {
    return new this("Margherita", 6.99);
  }
}
```

ここで静的メソッドに関して非常に重要なことが出てきます。

静的メソッド内の`this`キーワードの値はクラス自体です。静的メソッドはクラスに属しているためです。

そのため、`this`を使って`Pizza`クラスの新しいインスタンスを作成できます。

このメソッドを`Pizza`クラス自体で呼び出し、返されたインスタンスを変数に代入すると、次のようになります。

```js
let myPizza = Pizza.createMargherita();
```

コード内の必要な場所で使えます。例えば、コンソールに出力することもできます。

```js
console.log(myPizza);
```

出力は次の通りです。

```js
Pizza { type: 'Margherita', price: 6.99 }
```

ドット記法を使ってメソッドを呼び出したり、プロパティにアクセスしたりもできます。

```js
console.log(myPizza.type);
```

完全な例はこちらです。

```js
class Pizza {
  constructor(type, price) {
    this.type = type;
    this.price = price;
  }

  static createMargherita() {
    return new this("Margherita", 6.99);
  }
}

let myPizza = Pizza.createMargherita();
console.log(myPizza);
console.log(myPizza.type);
```

メソッドに加えて、`static`キーワードで静的プロパティも定義できます。

この例では、静的な`numberOfPizzasSold`プロパティがあります。

```js
class Pizza {
  static numberOfPizzasSold = 0;

  constructor(type) {
    this.type = type;
    Pizza.numberOfPizzasSold++;
  }
}
```

特定のピザインスタンスに属しているのではなく、クラス自体に属しているため静的です。

初期値は`0`で、新しいインスタンスが作成されるたびに更新されます。

2つのピザインスタンスを作成すると、値は2回更新されます。

```js
let pizza1 = new Pizza("Margherita");
let pizza2 = new Pizza("Neapolitan");
```

静的プロパティの値にアクセスするには、プロパティがクラスに属しているため、クラス自体にドット記法を使うだけです。

```js
class Pizza {
  static numberOfPizzasSold = 0;

  constructor(type) {
    this.type = type;
    Pizza.numberOfPizzasSold++;
  }
}

let pizza1 = new Pizza("Margherita");
let pizza2 = new Pizza("Neapolitan");

console.log(Pizza.numberOfPizzasSold);
```

この場合、2つのピザが売れたので出力は`2`です。

これがJavaScriptにおける静的プロパティとメソッドの基本です。静的メンバーを理解することは、再利用可能で効率的なクラスを作成するために不可欠です。
