# --description--

Wie wir wissen, geht es bei der wahrgenommenen Leistung nicht nur darum, wie schnell eine Webseite tatsächlich ist – sondern darum, wie schnell sie sich für die Nutzer anfühlt. Haben Sie zum Beispiel schon einmal eine Webseite benutzt, die sich langsam anfühlte, obwohl sie es nicht war? Das ist wahrgenommene Leistung – wie schnell sich eine Webseite für den Nutzer anfühlt.

Kleine Anpassungen können einen großen Unterschied im Nutzererlebnis machen. Lassen Sie uns einige praktische Möglichkeiten erkunden, um die wahrgenommene Leistung zu verbessern!

Ladeanimationen können Nutzer das Gefühl geben, dass sie länger warten müssen, als sie tatsächlich tun. Versuchen Sie stattdessen Skeleton Screens – graue Platzhalter, die echten Inhalt nachahmen und nach und nach geladen werden:

```html
<div class="skeleton"></div>
<script>
  setTimeout(() => {
    document.querySelector(".skeleton").innerHTML = "✅ Content Loaded!";
  }, 2000);
</script>
```

Nutzer haben das Gefühl, dass die Seite schrittweise lädt, anstatt im Dunkeln zu warten.

Außerdem sollten Nutzer nicht warten müssen, bis alles geladen ist, bevor sie interagieren können. Laden Sie zuerst wichtige Inhalte und dann den Rest im Hintergrund. Wir haben uns das bereits mit Lazy Loading angesehen:

```html
<img src="placeholder.jpg" data-src="real-image.jpg" class="lazy">
<script>
  document.addEventListener("DOMContentLoaded", function () {
    const lazyImages = document.querySelectorAll("img.lazy");
    lazyImages.forEach(img => {
      img.src = img.dataset.src;
    });
  });
</script>
```

Das lässt die Seite responsiv wirken, auch wenn noch nicht alles geladen ist.

Und was, wenn Ihre Seite vorhersagen könnte, was der Nutzer als Nächstes anklicken wird? Mit Preloading können Sie die nächste Seite schon laden, bevor der Nutzer sie anfordert:

```html
<link rel="preload" href="next-page.html" as="document">
```

Das reduziert die wahrgenommene Ladezeit beim Navigieren auf nahezu null.

Und schließlich mögen Nutzer keine Unsicherheit. Wenn sie auf einen Button klicken, bestätigen Sie ihre Aktion sofort:

```html
<button onclick="this.innerText='⏳ Processing...'; setTimeout(() => this.innerText='✅ Done!', 2000);">
  Click Me
</button>
```

Das bedeutet, dass der Nutzer auch bei Verzögerungen das Gefühl hat, dass etwas passiert.

Zusammenfassend lässt sich sagen: Die Verbesserung der wahrgenommenen Leistung dreht sich nicht nur um rohe Geschwindigkeit, sondern darum, wie sich Geschwindigkeit für den Nutzer anfühlt. Verwenden Sie Skeleton Screens, priorisieren Sie wichtige Inhalte, preloaden Sie intelligent und geben Sie sofortiges Feedback, damit sich Ihre Seite schneller anfühlt.
