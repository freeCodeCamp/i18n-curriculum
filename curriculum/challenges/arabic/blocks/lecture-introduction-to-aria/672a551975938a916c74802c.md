# --interactive--

تُستخدم الخاصية `aria-describedby` لتوفير معلومات إضافية عن عنصر لمستخدمي قارئ الشاشة عبر الإشارة إلى محتوى موجود على الصفحة. تُنشئ ارتباطًا برمجيًا بين العنصر والمحتوى (يُشار إليه تقنيًا بالوصف القابل للوصول)، والذي يمكن لقارئات الشاشة استخدامه لإبلاغ المستخدمين بالمعلومات الإضافية عند التفاعل مع العنصر.

الاستخدام الأكثر شيوعًا لـ `aria-describedby` هو ربط التعليمات ورسائل الخطأ بحقول النماذج. بسبب الطرق المختلفة التي يستخدمها مستخدمو قارئ الشاشة للتنقل في الصفحة، قد يفوتون هذه الرسائل عند التنقل بين الحقول. يساعد استخدام `aria-describedby` على ضمان سماعهم لهذه الرسائل.

لنلقِ نظرة على بعض الأمثلة لفهم كيفية عملها. في هذا المثال الأول، لدينا عنصر `form` يقبل كلمة مرور.

فعّل محرر التفاعل واكتب بعض الأحرف في حقل إدخال كلمة المرور. سترى أن كلمة المرور مخفية في نافذة المعاينة. يجب أن تلاحظ أيضًا أن نص `password-help` يبقى باللون الأحمر حتى تكتب 8 أحرف أو أكثر في الحقل.

ملاحظة: يستخدم هذا المثال التفاعلي CSS وJavaScript لتحديث لون نص `password-help` ديناميكيًا. لا تقلق بشأن محاولة فهم شفرة JavaScript لأنك ستتعلم JavaScript في الوحدات القادمة.

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<form>
  <label for="password">Password:</label>
  <input type="password" id="password" aria-describedby="password-help" />
  <p id="password-help">Your password must be at least 8 characters long.</p>
</form>

<script src="index.js"></script>
```

```css
#password-help {
  color: red;
}
```

```js
const passwordEl = document.getElementById("password");
const passwordHelpText = document.getElementById("password-help");

passwordEl.addEventListener("input", (e) => { 
  const userInput = e.target.value;
  passwordHelpText.style.color = userInput.length >= 8 ? "green" : "red";
});
```

:::

نستخدم عنصر `label` لنص `Password` ونربطه بحقل إدخال كلمة المرور `input`.

لدينا أيضًا عنصر فقرة يوضح متطلبات كلمة المرور. نستخدم خاصية `aria-describedby` لربط حقل `input` الخاص بكلمة المرور بمتطلبات كلمة المرور الموجودة في عنصر الفقرة. عندما يتفاعل مستخدم قارئ الشاشة مع هذا الـ `input`، سيعلن قارئ الشاشة اسم الـ `input` (`Password`) وقد يعلن أيضًا متطلبات كلمة المرور. مع ذلك، هذا ليس مضمونًا دائمًا، إذ قد لا تعلن بعض برامج قراءة الشاشة المحتوى الإضافي تلقائيًا، أو قد تعلنه فقط في ظروف معينة. هذا أمر شائع. كل قارئ شاشة يتعامل مع خصائص ARIA بطريقة مختلفة. مع ذلك، يبقى استخدام `aria-describedby` مفيدًا لمستخدمي قارئ الشاشة بشكل عام.

حالة استخدام جيدة أخرى لخاصية `aria-describedby` هي عندما يكون لديك `button` للحذف. إليك مثالًا على `button` للحذف يتبعه رسالة تصف ما سيحدث عند نقر الزر:

:::interactive_editor

```html
<button aria-describedby="delete-message">Delete</button>

<p id="delete-message">Warning! All deletions are permanent.</p>
```

:::

تمامًا كما في المثال السابق، نربط زر الحذف بالرسالة باستخدام الخاصية `aria-describedby`. يجب أن تتطابق قيمة `id` مع قيمة الخاصية `aria-describedby`.

الخاصية `aria-describedby` هي خاصية قوية يمكن استخدامها لضمان توفير معلومات إضافية عن عنصر لمستخدمي قارئ الشاشة عند التفاعل مع العنصر. تُستخدم غالبًا لربط التعليمات ورسائل الخطأ بحقول النماذج لتقليل احتمال تفويت مستخدمي قارئ الشاشة لهذه الرسائل أثناء التنقل في النموذج.
