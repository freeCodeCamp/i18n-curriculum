# --description--

在本課程中，你將學習如何使用基本的原生型別以及型別註解如何作用於 `variable`。

對於原始資料型別 `string`、`null`、`undefined`、`number`、`boolean` 和 `bigint`，TypeScript 提供相應的型別關鍵字。 

在本課程中，我們將專注於常用的原始型別，而不會詳細說明 `bigint`。`bigint` 資料型別是 JavaScript 中用來表述比標準整數型別更大的數字的數值資料型別。 

讓我們從 `string` 開始，來看看這些原生型別中的每一個。以下是你如何在 TypeScript 中定義字串的方式：

```ts
let firstName: string = "Angie";
```

與純 JavaScript 不同，變數 `firstName` 預期是 `string` 型別。所以如果你嘗試用數字值重新指定它，則會得到像這樣的錯誤：

```ts
let firstName: string = "Angie";
firstName = 9 // Type 'number' is not assignable to type 'string'.
```

下一個我們將要看的型別是 `number` 型別。這裡有一個範例：

```ts
let age: number = 16;
```

你可以將整數和浮點數指定給 `number` 型別。但如果你嘗試重新指定一個 `bigint` 型別的值，則會得到像這樣的錯誤：

```ts
let age: number = 16;

// Type 'bigint' is not assignable to type 'number'.
age = 123n;
```

下一個我們將要看的型別是 `boolean` 型別。以下是一些範例：

```ts
let isLoggedIn: boolean = true;
let isAdmin: boolean = false;
```

最後兩個型別是 `null` 和 `undefined` 型別。 

如你從先前的 JavaScript 課程所回想，`null` 表現為有意的值缺失。`undefined` 意味著一個變數已被宣告但尚未被指定值。

當你學習 `function` 型別和 `union` 型別時，稍後你將看到如何使用這些型別的範例。

現在你已經看到型別註解如何與變數一起使用，問題就出現了：你是否需要總是明確地為你的變數標註型別。 

如果你有像這樣的簡單型別註解，那麼你可能不需要使用明確的型別。

```ts
let firstName: string = "Angie";
```

原因是 TypeScript 會嘗試根據其值自動推斷型別。所以如果你這樣寫，TypeScript 會理解它是字串。

```js
let firstName = "Angie";
```

當 TypeScript 無法明確推斷型別或你想為像是函式參數或複雜物件等情況強制指定特定型別時，你應該明確地為你的變數指定型別。
