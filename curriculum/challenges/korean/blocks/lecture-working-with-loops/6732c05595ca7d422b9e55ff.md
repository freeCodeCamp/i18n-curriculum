# --interactive--

`for...in` 루프는 객체의 속성을 순회해야 할 때 가장 적합합니다. 이 루프는 상속된 속성과 숫자가 아닌 속성을 포함하여 객체의 모든 열거 가능한 속성을 반복합니다.

상속된 속성은 객체의 프로토타입 체인에서 상속된 속성입니다. 숫자가 아닌 속성은 숫자가 아니거나 숫자로 변환할 수 없는 문자열 속성입니다.

다음은 `for...in` 루프의 기본 구문입니다:

```js
for (variable in object) {
  // code block to be executed
}
```

예제의 변수는 현재 순회 중인 객체의 속성을 나타냅니다.

`for...in` 루프가 어떻게 작동하는지 더 잘 이해할 수 있도록 몇 가지 예제를 살펴보겠습니다.

첫 번째 예제에서는 `fruit` 객체가 있고 각 속성을 순회하며 값을 콘솔에 출력하려고 합니다.

:::interactive_editor

```js
const fruit = {
  name: 'apple',
  color: 'red',
  price: 0.99
};

for (const prop in fruit) {
  console.log(fruit[prop]);
}
```

:::

`prop` 변수는 객체의 현재 속성을 나타냅니다. `fruit[prop]`는 각 속성의 값을 접근하는 데 사용됩니다.

첫 번째 반복에서는 `prop`가 `name`가 됩니다. 두 번째 반복에서는 `prop`가 `color`가 되고, 계속 이어집니다.

콘솔에 출력되는 결과는 `apple`, `red`, `0.99`입니다.

두 번째 예제에서는 중첩된 객체가 있고 각 속성을 순회하며 값을 콘솔에 출력하려고 합니다.

:::interactive_editor

```js
const person = {
  name: 'John',
  age: 30,
  address: {
    street: '123 Main St',
    city: 'Anytown',
    state: 'CA'
  }
};

for (const prop in person) {
  console.log(person[prop]);
}
```

:::

`address` 속성 자체가 객체입니다. `for...in` 루프는 `person` 객체의 속성도 순회하며 전체 `address` 객체를 콘솔에 출력합니다.

콘솔에서 결과는 다음과 같이 나타납니다:

```md
John
30
{ street: '123 Main St', city: 'Anytown', state: 'CA' }
```

`address` 객체의 속성을 순회하려면 첫 번째 루프 안에 또 다른 `for...in` 루프를 중첩할 수 있습니다.

:::interactive_editor

```js
const person = {
  name: 'John',
  age: 30,
  address: {
    street: '123 Main St',
    city: 'Anytown',
    state: 'CA'
  }
};

function isObject(obj) {
  return typeof obj === 'object' && !Array.isArray(obj) && obj !== null;
}

for (const prop in person) {
  if (isObject(person[prop])) {
    for (const nestedProp in person[prop]) {
      console.log(person[prop][nestedProp]);
    }
  } else {
    console.log(person[prop]);
  }
}
```

:::

이 예제에서는 값이 객체인지 확인하는 사용자 정의 함수 `isObject`가 있습니다.

`Array.isArray` 메서드는 값이 배열인지 확인하는 데 사용됩니다. 논리 부정 연산자(`!`)를 메서드 앞에 붙여 값이 배열이 아닌지 확인합니다.

단순히 `typeof`가 `'object'`와 같은지 확인할 수 없는 이유는 배열도 JavaScript에서 객체로 간주되기 때문입니다. 배열은 검사에서 제외하려고 합니다.

또한 JavaScript의 역사적인 버그 때문에 `typeof null`는 `'object'`를 반환합니다. 따라서 `null` 값도 검사에서 제외해야 합니다.

조건이 참이면 중첩된 객체의 속성을 순회하며 값을 콘솔에 출력하는 또 다른 `for...in` 루프를 중첩합니다.

`nestedProp` 변수는 중첩된 객체의 현재 속성을 나타냅니다.

수정된 결과는 콘솔에서 다음과 같이 나타납니다:

```md
"John"
30
"123 Main St"
"Anytown"
"CA"
```

`for...in` 루프는 객체의 속성을 순회해야 할 때 유용합니다.

배열의 요소를 순회할 때 `for...in` 루프를 사용하는 것은 권장하지 않습니다. 대신 `for...of` 루프나 앞으로 수업에서 배우게 될 `forEach`, `map`, `filter`, `reduce` 같은 배열 메서드를 사용하세요.
