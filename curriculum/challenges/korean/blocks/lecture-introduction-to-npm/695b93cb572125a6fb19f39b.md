# --description--

의미 있는 버전 관리, 또는 SemVer는 버전 번호에 명확한 의미를 부여하는 시스템으로, 개발자가 소프트웨어에 어떤 종류의 변경이 있었는지 빠르게 이해할 수 있게 합니다.

SemVer는 다음 형식을 사용합니다:

```bash
MAJOR.MINOR.PATCH
```

예를 들어, 인기 있는 라이브러리 `lodash`는 현재 `4.17.21` 버전입니다.

각 부분이 의미하는 바는 다음과 같습니다:

* MAJOR — 기존 코드를 깨뜨리는 큰 변경입니다. 예: 기존 코드가 의존할 수 있는 함수를 제거하거나 이름을 바꾸는 경우.
    
* MINOR — 기존 코드를 깨뜨리지 않는 새로운 기능입니다. 예: 새 함수나 선택적 매개변수를 추가하는 경우.
    
* PATCH — 작은 수정과 개선입니다. 예: 작동 방식을 바꾸지 않고 버그나 오타를 고치는 경우.
    

<img src="https://cdn.freecodecamp.org/curriculum/lecture-transcripts/what-is-semantic-versioning-1.png" alt="버전 번호 4.17.21에서 4는 Major Version으로, 깨뜨리는 변경에 따라 바뀌고, 17은 minor version으로, 깨뜨리지 않는 변경에 따라 바뀌며, 21은 patch로, 버그 수정과 작은 기능에 따라 바뀝니다">

`my-package`라는 패키지가 있다고 가정해 봅시다. 의미 있는 버전 관리를 사용해 시간이 지남에 따라 버전 번호가 어떻게 바뀔 수 있는지 예를 들어 보겠습니다:

| 버전 | 변경 내용 |
| --- | --- |
| 1.0.0 | 초기 안정 버전 출시 |
| 1.1.0 | 새로운 선택적 기능 추가 |
| 1.1.1 | 작은 버그 수정 또는 기능 개선 |
| 2.0.0 | 깨뜨리는 변경 도입 |

하지만 npm의 버전 번호에는 더 많은 의미가 있습니다. 패키지를 설치할 때 `package.json` 파일에 다음과 같은 버전이 있을 수 있습니다:

```json
"lodash": "^4.17.21" 
```

이미 `4`가 메이저 버전, `17`가 마이너 버전, `21`가 패치 버전이라는 것을 배웠습니다. 그렇다면 앞에 붙은 캐럿(`^`)은 무엇을 의미할까요?

캐럿 기호는 npm이 메이저 버전이 아닌 최신 마이너 및 패치 버전을 자동으로 설치하도록 허용합니다(`4.x.x`). 메이저 업데이트는 깨뜨리는 변경을 포함할 수 있기 때문에 최신 메이저 버전(`5.x.x`)은 설치하지 않습니다.

메이저 버전이 `0`일 때 캐럿은 더 제한적이 됩니다. 예를 들어 `^0.2.3`은 `0.2.4` 같은 업데이트를 허용하지만 `0.3.0`은 허용하지 않습니다. `0`로 시작하는 메이저 버전은 덜 안정적인 것으로 간주되기 때문입니다.

캐럿은 `npm`와 다른 JavaScript 패키지 관리자가 의존성 버전을 관리할 때 사용하는 유일한 기호가 아닙니다.

물결표 기호(`~`)는 패치 버전만 업데이트할 수 있게 허용합니다. 따라서 `~1.2.3`은 `1.2.4`과 `1.2.9`는 허용하지만 `1.3.0`은 허용하지 않습니다.

또한 크거나 같음(`>`), 작거나 같음(`<`)과 같은 기호를 사용해 버전 범위를 지정할 수 있습니다. 예를 들어 `>=1.2.3 <2.0.0`은 `1.2.3`부터 `2.0.0` 미만까지 모든 버전을 의미합니다.

별표 기호(`*`)는 모든 버전과 일치하므로 모든 버전을 허용합니다. 이는 테스트용으로만 적합하며 프로덕션에는 권장하지 않습니다.

마지막으로, 버전 번호 앞에 기호를 붙이지 않으면 정확한 버전만 고정할 수 있습니다. 예를 들어 `1.2.3` 버전은 `1.2.3`만 허용합니다.

간단한 요약 표는 다음과 같습니다:

| 기호 | 의미 | 예시 범위 |
| --- | --- | --- |
| `^` | 마이너 + 패치 업데이트 허용 | `^1.2.3` → `1.x.x` |
| `~` | 패치 업데이트만 허용 | `~1.2.3` → `1.2.x` |
| `>`, `<`, `>=`, `<=` | 사용자 지정 범위 | `>=1.2.3` 또는 `<2.0.0` |
| **\*** | 모든 버전 | 모든 버전 |
| 없음 | 정확한 버전 | `1.2.3` |
