# --description--

Bedingtes Rendern in React ermöglicht es Ihnen, dynamische Benutzeroberflächen zu erstellen. Es wird verwendet, um je nach bestimmten Bedingungen oder Zuständen innerhalb Ihrer Anwendung unterschiedliche Inhalte anzuzeigen.

Die gebräuchlichsten Ansätze für bedingtes Rendern umfassen die Verwendung von `if`-Anweisungen, dem ternären Operator (`?:`) und dem logischen UND-Operator (`&&`).

Die einfachste Form des bedingten Renderns verwendet eine `if`-Anweisung. Hier ist ein Beispiel:

```jsx
function Greeting({ isLoggedIn }) {
  if (isLoggedIn) {
    return <h1>Welcome back!</h1>;
  }
  return <h1>Please sign in</h1>;
}
```

In diesem Beispiel prüft die `Greeting`-Komponente die `isLoggedIn`-Prop. Wenn sie `true` ist, gibt sie eine Willkommensnachricht zurück, andernfalls fordert sie den Benutzer auf, sich anzumelden.

Hier ist ein Beispiel, das die `Greeting`-Komponente innerhalb der übergeordneten `App`-Komponente verwendet:

```jsx
function App() {
  return (
    <div className="App">
      <Greeting isLoggedIn={false} />
    </div>
  );
}
```

Für einfachere Bedingungen wird der ternäre Operator (`?:`) oft direkt innerhalb von JSX verwendet. Er ermöglicht ein inline bedingtes Rendern, was Ihren Code kompakter machen kann:

```jsx
function Greeting({ isLoggedIn }) {
  return <h1>{isLoggedIn ? "Welcome back!" : "Please sign in."}</h1>;
}
```

Dieser Code erzielt dasselbe Ergebnis wie das vorherige Beispiel, jedoch in einer kompakteren Form. Der ternäre Operator prüft `isLoggedIn` und rendert die passende Nachricht.

Ein weiteres häufiges Muster für bedingtes Rendern ist die Verwendung des logischen UND-Operators (`&&`). Dies ist besonders nützlich, wenn Sie etwas oder nichts basierend auf einer Bedingung rendern möchten:

```jsx
function Notification({ message }) {
  return (
    <div>
      {message && <p>{message}</p>}
    </div>
  );
}
```

In diesem Beispiel wird das Absatz-Element mit der Nachricht nur gerendert, wenn die `message`-Prop truthy ist. Wenn `message` falsy ist – also ein leerer String, `null` oder `undefined` – wird nichts auf dem Bildschirm angezeigt.

Wenn Sie diese Techniken des bedingten Renderns beherrschen, können Sie interaktivere und benutzerfreundlichere Anwendungen erstellen, die sich an veränderte Daten und Benutzerinteraktionen anpassen.
