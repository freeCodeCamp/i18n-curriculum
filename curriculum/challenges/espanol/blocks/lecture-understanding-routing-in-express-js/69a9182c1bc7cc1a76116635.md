# --description--

En Express, las **rutas** son los patrones de URL que determinan cómo responde el servidor a las solicitudes entrantes. Cuando un usuario solicita una URL específica, Express compara esa URL con una ruta y ejecuta el manejador correspondiente.

Las rutas se definen dentro de métodos de ruta como `app.get()`, `app.post()`, y así sucesivamente. Estas rutas pueden ser estáticas, como `/home`, o dinámicas, donde se usan parámetros para capturar valores de la URL, como `/user/:id`.

En esta lección nos enfocaremos en rutas estáticas. Aquí tienes un ejemplo:

```javascript
// Static route
app.get("/home", (req, res) => {
  res.send("Welcome to the homepage!");
});
```

En este ejemplo, el servidor escucha solicitudes GET a `/home` y responde con un mensaje simple.

El método `app.get()` recibe dos argumentos:

* El primer argumento, `"/home"`, es la ruta estática (o URL) que el servidor escucha.
    
* El segundo argumento es una función callback que se ejecuta cuando se hace una solicitud GET a esa ruta.
    

Dentro del callback, `res.send()` envía una respuesta al cliente. En este caso, envía el texto `Welcome to the homepage!`, que se muestra en el navegador.

Aquí tienes otro ejemplo de una ruta estática:

```javascript
// Static route
app.get("/menu/drinks", (req, res) => {
  res.send("Welcome to the drinks menu!");
});
```

En este caso, el servidor escucha solicitudes GET a la URL `/menu/drinks` y envía un mensaje cuando se solicita esa ruta.

Podrías usar rutas así cuando tu sitio tiene varias páginas relacionadas agrupadas bajo una ruta común. Por ejemplo, un sitio web de un restaurante podría organizar sus páginas de menú así:

* `/menu/food`
    
* `/menu/drinks`
    
* `/menu/desserts`
    

Este enfoque ayuda a mantener tus URLs claras y significativas, y también facilita organizar y mantener tus rutas a medida que tu aplicación crece.

En la próxima lección, veremos rutas dinámicas con parámetros.
