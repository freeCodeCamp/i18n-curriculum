# --description--

In Python arbeiten Klassen und Objekte Hand in Hand, um Daten zu organisieren und zu verwalten. Sie erstellen eine Klasse, um gemeinsames Verhalten zu definieren, und erzeugen dann Objekte, die dieses Verhalten nutzen.

Mit anderen Worten: Eine Klasse ist wie ein Bauplan oder eine Vorlage, die Sie verwenden, um Objekte zu erstellen.

Schauen wir uns an, was Klassen sind und wie Sie sie verwenden, um Objekte zu erstellen.

Um eine Klasse zu erstellen, verwenden Sie das Schlüsselwort `class` gefolgt vom Namen der Klasse und einem Doppelpunkt. Innerhalb der Klasse können Sie dann einen Initialisierer sowie beliebige Attribute und Methoden hinzufügen.

Attribute sind wie Variablen innerhalb einer Klasse und dienen dazu, Daten zu speichern. Methoden sind Funktionen, die innerhalb einer Klasse definiert sind und die Aktionen darstellen, die Objekte, die mit einer Klasse erstellt wurden, ausführen können.

Hier ist die grundlegende Syntax einer Klasse:

```python
class ClassName:
    def __init__(self, name, age):
        self.name = name
        self.age = age

    def sample_method(self):               
        print(self.name.upper())
```

* `class ClassName` besteht aus dem Schlüsselwort `class`, um eine Klasse zu erstellen, gefolgt vom Namen der Klasse, hier `ClassName` genannt. Es ist in Python üblich, bei der Benennung von Klassen die **PascalCase**-Konvention zu verwenden.
    
* `def __init__(self, name, age)` ist die spezielle Methode, die automatisch aufgerufen wird, wenn ein neues Objekt erstellt wird. Sie initialisiert die Attribute der Objekte, die mit der Klasse erstellt werden.
    
    Zusätzlich ist der erste Parameter von `__init__` immer eine Referenz auf das spezifische Objekt, das gerade erstellt oder verwendet wird. Üblicherweise wird dieser Parameter `self` genannt, aber technisch können Sie jeden Namen verwenden. `self` ermöglicht den Zugriff auf die eigenen Attribute und Methoden des Objekts.
    
* `self.name = name` und `self.age = age` sind die Attribute, die die Objekte haben werden.
    
* `def sample_method(self):` ist die Methode, die jedes erstellte Objekt aufrufen kann.
    
* `print(self.name.upper())` ist das, was die `sample_method`-Methode tun wird, in diesem Fall gibt sie den Namen in Großbuchstaben aus.
    

Wenn das alles viel klingt, keine Sorge. Schauen wir uns ein ähnliches Beispiel einer `Dog`-Klasse an und wie Sie daraus Objekte erstellen können:

```python
class Dog:
    def __init__(self, name, age):
        self.name = name
        self.age = age

    def bark(self):
        print(f"{self.name.upper()} says woof woof!")
```

Mit dieser `Dog`-Klasse können Sie ein Objekt erstellen. Hier ist die grundlegende Syntax, um Objekte aus einer Klasse zu erstellen:

```python
object_1 = ClassName(attribute_1, attribute_2)
object_2 = ClassName(attribute_1, attribute_2)
```

Sie können auch jede der in der Klasse definierten Methoden von jedem Objekt aus aufrufen:

```python
object_1.method_name()
object_2.method_name()
```

Jetzt erstellen wir zwei Hunde, indem wir die `Dog`-Klasse als Vorlage verwenden:

```python
class Dog:
    def __init__(self, name, age):
        self.name = name
        self.age = age

    def bark(self):
        print(f"{self.name.upper()} says woof woof! I'm {self.age} years old!")

dog_1 = Dog("Jack", 3)
dog_2 = Dog("Thatcher", 5)

# Call the bark method
dog_1.bark()  # JACK says woof woof! I'm 3 years old!
dog_2.bark()  # THATCHER says woof woof! I'm 5 years old!
```

Wie Sie sehen, erstellen wir zwei Hund-Objekte mit der `Dog`-Klasse. Beim Initialisieren von `dog_1` werden der String `Jack` und die Zahl `3` übergeben, die die Attribute `name` und `age` für diese Instanz festlegen. Und `dog_2` wird mit dem String `Thatcher` und der Zahl `5` als seine `name` und `age` initialisiert.

Wenn Sie dann die `.bark()`-Methode auf `dog_1` und `dog_2` aufrufen, sehen Sie, wie sich beide Ausgaben unterscheiden und die einzigartigen `name` und `age` Attribute verwenden, die Sie beim Erstellen jedes Objekts übergeben haben.

Zusammenfassend ist der Unterschied zwischen einer Klasse und einem Objekt, dass eine Klasse die Vorlage oder der Bauplan ist und ein Objekt das ist, was mit dieser Vorlage erstellt wird.

Außerdem definiert eine Klasse, welche Daten und welches Verhalten das Objekt haben soll, und ein Objekt hält die tatsächlichen Daten und nutzt dieses Verhalten. Sie schreiben eine Klasse einmal und können viele Objekte daraus erstellen, jedes mit unterschiedlichen Daten.
