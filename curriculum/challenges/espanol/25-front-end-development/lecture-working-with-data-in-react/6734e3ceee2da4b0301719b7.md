# --description--

En las conferencias anteriores, aprendimos cómo construir pequeños componentes en React como este:

```jsx
function Greeting() {
  const developerName = "Jessica";
  return <h1>Hi {developerName}!</h1>;
}
```

Podemos elegir anidar este componente dentro de otro componente padre o el componente raíz así:

```jsx
function App() {
  return <Greeting />;
}

function Greeting() {
  const developerName = "Jessica";
  return <h1>Hi {developerName}!</h1>;
}
```

Aunque este código se ejecutará bien y mostrará el resultado de `¡Hola Jessica!` en la pantalla, no es un componente tan flexible.

¿Qué pasa si queremos mostrar un nombre diferente como Naomi, Tom o Oliver? Aquí es donde entran en juego las propiedades de React. Las propiedades, que son abreviaturas de propiedades, son la forma en que los componentes principales pasan datos al componente hijo. Las propiedades pueden ser de cualquier tipo: cadenas, números, booleanos, objetos o matrices.

Actualicemos nuestro ejemplo anterior para ahora aceptar una propiedad `nombre`:

```jsx
function App() {
  return <Greeting name="Jessica" />;
}

export default App;

function Greeting(props) {
  console.log(props);
  return <h1>Hi {props.name}!</h1>;
}
```

Para el componente hijo llamado `Saludo` ahora estamos usando `props.nombre` en lugar de codificar el nombre `"Jessica"`. También estamos registrando `props` en la consola, que se muestra como un objeto.

Luego, dentro del componente principal `App`, estamos pasando el valor a la propiedad `nombre` para que pueda ser pasado al hijo. El resultado será el mismo en la pantalla que anteriormente, pero ahora hemos creado un componente más flexible.

Ahora tenemos la capacidad de reutilizar el componente hijo varias veces y pasar diferentes nombres cada vez:

```jsx
function App() {
  return (
    <>
      <Greeting name="Naomi" />
      <Greeting name="Tom" />
      <Greeting name="Jessica" />
      <Greeting name="Oliver" />
    </>
  );
}
```

También puedes optar por usar la desestructuración de objetos en las propiedades para hacerlo más legible. Aquí tienes cómo podrías reescribir el componente `Saludo`:

```jsx
function Greeting({ name }) {
  return <h1>Hi {name}!</h1>;
}
```

Este código logra el mismo resultado pero hace más claro qué propiedades espera recibir el componente.

A veces, puedes tener muchas propiedades que debes pasar como propiedades. En lugar de pasarlas una por una, puedes usar el operador de propagación (`...`), después de convertirlas en un objeto.

Aquí tienes un ejemplo de un nuevo componente hijo llamado `TarjetaDesarrollador`:

```jsx
function DeveloperCard({ name, age, country }) {
  return (
    <div className="developer-card">
      <h1>Developer: {name}</h1>
      <p>Age: {age}</p>
      <p>Country: {country}</p>
    </div>
  );
}
```

Este componente `TarjetaDesarrollador` acepta tres propiedades: `nombre`, `edad` y `país`.

En el componente principal `App`, podemos usar la sintaxis de propagación para pasar todas las propiedades de un objeto como propiedades individuales al componente hijo:

```jsx
function App() {
  const developerObj = {
    name: "Alice",
    age: 30,
    country: "USA",
  };

  return (
    <div className="App">
      <DeveloperCard {...developerObj} />
    </div>
  );
}
```

Esto es particularmente útil cuando se trabaja con matrices de objetos y se pasan múltiples conjuntos de propiedades a componentes hijos. Por ejemplo, podrías tener una lista de desarrolladores donde cada objeto en la matriz tiene la misma estructura, pero representa a una persona diferente.

Aprenderás más sobre cómo renderizar listas en matrices en futuros videos de conferencias.

Usar propiedades en React hace que tus componentes sean más flexibles y reutilizables, permitiéndote construir interfaces de usuario más complejas. Sin embargo, es importante notar que las propiedades son inmutables, lo que significa que no pueden cambiarse una vez pasadas a un componente. Si necesitas manejar la entrada del usuario y modificar datos, debes usar el estado en su lugar. Aprenderás más sobre la gestión del estado en futuros videos de conferencias.
