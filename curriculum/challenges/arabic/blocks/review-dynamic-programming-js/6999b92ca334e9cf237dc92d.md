## مقدمة إلى البرمجة الديناميكية

- **التعريف**: البرمجة الديناميكية هي تقنية خوارزمية تحل المشكلات المعقدة بتقسيمها إلى مشكلات فرعية أبسط وتخزين النتائج لتجنب الحسابات المتكررة.
- **المشكلات الفرعية المتداخلة**: تظهر نفس المشكلات الصغيرة عدة مرات عند حل المشكلة الأكبر. بدلاً من إعادة حساب هذه المشكلات الفرعية مرارًا، نخزن حلولها.
- **البنية الفرعية المثلى**: الحل الأمثل للمشكلة يحتوي على حلول مثلى لمشكلاتها الفرعية. هذا يعني أننا نستطيع بناء أفضل حل بدمج أفضل الحلول للأجزاء الأصغر.

## حلول البرمجة الديناميكية

- **التخزين المؤقت (النهج من الأعلى إلى الأسفل)**: التخزين المؤقت يخزن نتائج استدعاءات الدوال المكلفة ويُرجع النتيجة المخزنة عند تكرار نفس المُدخلات.

```js
function climbStairsMemo(n, memo = {}) {
  // If already calculated, return cached value
  if (memo[n] !== undefined) {
    return memo[n];
  }

  // Base cases: 1 way for 1 step, 2 ways for 2 steps
  if (n <= 2) {
    return n;
  }

  // Calculate once and store in memo for future use
  memo[n] =
    climbStairsMemo(n - 1, memo) +
    climbStairsMemo(n - 2, memo);

  return memo[n];
}
```

- **الجدولة (النهج من الأسفل إلى الأعلى)**: الجدولة تبني الحل من القاعدة للأعلى، بملء جدول بحلول المشكلات الفرعية.

```js
function climbStairsTabulation(n) {
  if (n <= 2) {
    return n;
  }

  // Create an array to store solutions from 0 to n
  const dp = new Array(n + 1).fill(0);

  // Base cases
  dp[1] = 1; // 1 way to reach step 1
  dp[2] = 2; // 2 ways to reach step 2

  // Build up the solution iteratively
  for (let i = 3; i <= n; i++) {
    // Ways to reach step i = ways to reach (i-1) + ways to reach (i-2)
    dp[i] = dp[i - 1] + dp[i - 2];
  }

  return dp[n];
}
```

## تطبيقات البرمجة الديناميكية في العالم الحقيقي

- **تحسين المسارات**: تستخدم أنظمة GPS خوارزميات البرمجة الديناميكية لإيجاد أقصر المسارات بين المواقع.
- **معالجة النصوص**: تعتمد مدققات الإملاء وميزات الإكمال التلقائي غالبًا على البرمجة الديناميكية لحساب مسافات التعديل بين الكلمات.
- **النمذجة المالية**: تستخدم استراتيجيات الاستثمار وتحسين معرض الأعمال تقنيات البرمجة الديناميكية بشكل متكرر.
- **تخصيص الموارد**: تظهر مشكلة الحقيبة وأنواعها في الجدولة والميزانية وإدارة الموارد.

## متى تستخدم البرمجة الديناميكية

يجب أن تفكر في استخدام البرمجة الديناميكية في الحالات التالية:

- يمكن تقسيم المشكلة إلى مشكلات فرعية متداخلة.
- تظهر المشكلة بنية فرعية مثلى.
- الحل التكراري البسيط يتطلب حسابات متكررة.
- تحتاج إلى تحسين تعقيد الزمن على حساب تعقيد المساحة.
