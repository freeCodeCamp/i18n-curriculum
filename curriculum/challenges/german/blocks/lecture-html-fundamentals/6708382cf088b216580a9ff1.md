# --interactive--

Das `id`-Attribut fügt einem HTML-Element eine eindeutige Kennung hinzu.

Hier ist ein Beispiel für ein `h1`-Element mit einem `id` von `title`.

Fügen Sie unter dem `h1`-Element ein `h2`-Element mit einem `id` hinzu, das auf `"subtitle"` gesetzt ist. Sie können für das `h2` beliebigen Text schreiben, und Sie werden die Änderungen im Vorschaufenster sehen. Um mit dem Beispiel zu interagieren, müssen Sie den interaktiven Editor aktivieren.

:::interactive_editor

```html
<h1 id="title">Movie Review Page</h1>

```

:::

Sie können den `id`-Namen von `title` in Ihrem JavaScript oder CSS referenzieren. Hier ist ein CSS-Beispiel, das den `id` von `title` referenziert, um die Textfarbe `color` in `red` zu ändern.

**HINWEIS**: Für dieses interaktive Beispiel wurde bereits etwas CSS bereitgestellt. Machen Sie sich keine Sorgen, wenn Sie den CSS-Code nicht verstehen, denn Sie werden in zukünftigen Lektionen mehr darüber lernen. Wenn Sie jedoch die Textfarbe in Blau ändern möchten, aktivieren Sie den interaktiven Editor, klicken Sie auf den `styles.css`-Tab und ändern Sie das `color: red;` in `color: blue;`.

:::interactive_editor

```html
<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="utf-8" />
    <meta
       name="viewport"
       content="width=device-width, initial-scale=1.0" />
    <title>Review page Example</title>
    <link rel="stylesheet" href="./styles.css" />
  </head>
  <body>
    <h1 id="title">Movie Review Page</h1>
  </body>
</html>
```

```css
#title {
  color: red;
}
```

:::

Das Rautezeichen (`#`) vor `title` teilt dem Computer mit, dass Sie ein `id` mit diesem Wert ansprechen möchten. `id`-Namen sollten nicht mehr als einmal verwendet werden und müssen immer eindeutig sein. Ein weiterer Hinweis zu `id`-Werten ist, dass sie keine Leerzeichen enthalten dürfen. Hier ist ein Beispiel, wie die Wörter `main` und `heading` auf einen `id`-Attributwert angewendet werden:

```html
<h1 id="main heading">Main heading</h1>
```

Browser betrachten dieses Leerzeichen als Teil des `id`, was zu unerwünschten Problemen bei der Gestaltung und Skripterstellung führt. `id`-Attributwerte sollten nur Buchstaben, Ziffern, Unterstriche und Bindestriche enthalten.

Im Gegensatz zum `id`-Attribut muss der `class`-Attributwert nicht eindeutig sein und kann Leerzeichen enthalten.

Hier ist ein Beispiel, wie eine Klasse namens `box` auf ein `div`-Element angewendet wird.

```html
<div class="box"></div>
```

Wenn Sie mehreren Elementen Klassennamen hinzufügen möchten, können Sie die Namen durch Leerzeichen trennen. Hier ist ein aktualisiertes Beispiel, das mehrere Klassen auf ein `div`-Element anwendet.

```html
<div class="box red-box"></div>
```

Hier ist ein weiteres Beispiel, wie mehrere Klassen auf mehrere `div`-Elemente angewendet werden.

**HINWEIS**: Für dieses interaktive Beispiel wurde bereits etwas CSS bereitgestellt. Machen Sie sich keine Sorgen, wenn Sie den CSS-Code nicht verstehen, denn Sie werden in zukünftigen Lektionen mehr darüber lernen. Wenn Sie jedoch die Farbe der ersten und dritten Box ändern möchten, aktivieren Sie den interaktiven Editor, klicken Sie auf den `styles.css`-Tab und ändern Sie das `background-color: red;` in `background-color: black;`.

:::interactive_editor

```html
<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="utf-8" />
    <meta
       name="viewport"
       content="width=device-width, initial-scale=1.0" />
    <title>Colored boxes example</title>
    <link rel="stylesheet" href="./styles.css" />
  </head>
  <body>
    <div class="box red-box"></div>
    <div class="box blue-box"></div>
    <div class="box red-box"></div>
    <div class="box blue-box"></div>
  </body>
</html>
```

```css
.box {
  width: 100px;
  height: 100px;
}

.red-box {
  background-color: red;
}

.blue-box {
  background-color: blue;
}
```

:::

Fassen wir also zusammen: Wann sollten Sie ein `id` anstelle eines `class` verwenden? Klassen eignen sich am besten, wenn Sie eine Reihe von Stilen auf viele Elemente anwenden möchten. Wenn Sie ein bestimmtes Element ansprechen möchten, ist es am besten, `id` zu verwenden, da diese Werte eindeutig sein müssen.

# --questions--

## --text--

Wann sollten Sie ein `id` anstelle eines `class` verwenden?

## --answers--

Verwenden Sie ein `class`, wenn Sie eine eindeutige Kennung benötigen, die nur auf ein Element angewendet werden soll.

### --feedback--

Denken Sie daran, dass ein `id`-Name eindeutig ist und nur einmal pro HTML-Dokument verwendet werden sollte.

---

Verwenden Sie ein `id`, wenn Sie eine eindeutige Kennung für ein bestimmtes Element benötigen, und verwenden Sie ein `class`, wenn Sie Stile oder Verhalten auf mehrere Elemente anwenden möchten.

---

Verwenden Sie ein `id`, wenn Sie mehrere Elemente auf Ihrer Webseite konsistent gestalten möchten.

### --feedback--

Denken Sie daran, dass ein `id`-Name eindeutig ist und nur einmal pro HTML-Dokument verwendet werden sollte.

---

Verwenden Sie ein `id`, wenn Sie Stile anwenden möchten, die leicht durch andere Stile in Ihrem CSS überschrieben werden können.

### --feedback--

Denken Sie daran, dass ein `id`-Name eindeutig ist und nur einmal pro HTML-Dokument verwendet werden sollte.
