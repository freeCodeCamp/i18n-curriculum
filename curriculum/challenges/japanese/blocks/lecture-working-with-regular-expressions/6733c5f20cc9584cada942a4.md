# --interactive--

キャプチャグループは、マッチした文字列の一部を「キャプチャ」して、必要に応じて使えるようにするものです。キャプチャグループは、先読みのような先頭の文字がなく、パターンを囲む丸括弧で定義されます。

`code` の正規表現から `freeCodeCamp` をキャプチャしてみましょう。そのために、`code` を丸括弧で囲み、キャプチャグループとして定義します。

```js
const regex = /free(code)camp/i;
```

動作を確認するために、`freecodecamp` の文字列に対してテストしてみましょう。

:::interactive_editor

```js
const regex = /free(code)camp/i;
console.log(regex.test("freecodecamp")); // true
```

:::

しかし、これは実際にはキャプチャしたグループを使っていません。代わりに、`match` を使った結果を見てみましょう。

```js
const regex = /free(code)camp/i;
console.log("freecodecamp".match(regex));
// [
//   'freecodecamp',
//   'code', <--
//   index: 0,
//   input: 'freecodecamp',
//   groups: undefined
// ]
```

ここで、`match` 配列の2番目の要素が、キャプチャグループによってキャプチャされた文字列の部分であることがわかります。

キャプチャグループは、文字クラスが `code`、`c`、`o`、`d` のリストから1文字をマッチするのに対し、`e` という正確なパターンにマッチすることに注意してください。

では、これをどう使うのでしょうか？キャプチャグループは、文字列の内容を置換するときによく使われます。これを実現するコードを用意しましょう。`freecodecamp` を `paidcodeworld` に変換します。

:::interactive_editor

```js
const regex = /free(code)camp/i;
console.log("freecodecamp".replace(regex, "paidcodeworld"));
```

:::

これは単独でも動作しますが、もし `o` に `code` がいくつあるかわからなかったらどうでしょう？1つ以上の `o` に対して量指定子が必要です。

:::interactive_editor

```js
const regex = /free(co+de)camp/i;
console.log("freecoooooooodecamp".replace(regex, "paidcodeworld"));
```

:::

結果として `paidcodeworld` が得られています。`o` の数を保持したいので、正規表現でキャプチャしたものを再利用する必要があります。

ここでバックリファレンスが登場します。置換文字列の `code` 部分をハードコードする代わりに、キャプチャしたグループを直接参照できます。

`replace` の呼び出しでは、ドル記号（`$`）に続けて使いたいキャプチャグループの番号を指定することでバックリファレンスを実現します。今回の場合、`$1` は最初のキャプチャグループでキャプチャされているので、`code` となります。

:::interactive_editor

```js
const regex = /free(co+de)camp/i;
console.log("freecoooooooodecamp".replace(regex, "paid$1world")); // paidcoooooooodeworld
```

:::

これで、`o` を `freecodecamp` に変換するときに不明な数の `paidcodeworld` 文字を正しく保持できました。しかし、バックリファレンスは replace 呼び出しに限らず、正規表現の中でも直接使えます。

これにより、正規表現の後半で以前にキャプチャしたパターンにマッチさせることができます。

例えば、同じ数の `freecodecamp` を持つ `o` を2回マッチさせたいとします。ただし、文字列のどこにあってもよいとします。

まず、ワイルドカード文字で区切り、そのワイルドカードに任意の数の文字がマッチするようにします。

```js
const regex = /free(co+de)camp.*free(co+de)camp/i;
```

ただし、この式では `o` 文字の数が同じであることは保証されません。これを実現するには、2番目のキャプチャグループを1番目の参照に置き換える必要があります。

正規表現の中でバックリファレンスは、バックスラッシュに続けてキャプチャグループの番号を付けて表します。

:::interactive_editor

```js
const regex = /free(co+de)camp.*free\1camp/i;
console.log(regex.test("freecooooodecamp is great i love freecooooodecamp")); // true
console.log(regex.test("freecooooodecamp is great i love freecodecamp")); // false
```

:::

これで、正しい数の `o` を持つ文字列はマッチし、異なる数の `o` を持つ文字列はマッチしないことがわかります。

この構文は便利ですが、複数のキャプチャグループを参照するときはすぐに混乱しやすいです。幸いなことに、番号の代わりにグループに名前を付けることができます。

名前付きキャプチャグループを定義するには、グループの先頭に疑問符（`?`）を付け、その後に名前を小なり・大なり記号で囲んで書きます。ここではキャプチャグループに `code` という名前を付けましょう。

```js
const regex = /free(?<code>co+de)camp.*free\1camp/i;
```

次に、正規表現内のバックリファレンスをこの名前付きグループに更新します。名前付きバックリファレンスは、JavaScriptではバックスラッシュに続けて文字 `k` を書き、その後に名前を小なり（`<`）と大なり（`>`）で囲んで書きます。見てみましょう。

```js
const regex = /free(?<code>co+de)camp.*free\k<code>camp/i;
```

`test()` の呼び出しを確認すると、依然として通過していることがわかります。

:::interactive_editor

```js
const regex = /free(?<code>co+de)camp.*free\k<code>camp/i;
console.log(regex.test("freecooooodecamp is freecooooodecamp")); // true
```

:::

名前付きキャプチャグループを `replace()` の呼び出しで使うには、文字列内にドル記号を入れ、その後に名前を小なり・大なり記号で囲んで書きます。

:::interactive_editor

```js
const regex = /free(?<code>co+de)camp/i;
console.log("freecooooodecamp".replace(regex, "paid$<code>camp")); // paidcooooodecamp
```

:::

最後に、文字のグループを作りたいけれど、キャプチャ結果は必要ない場合があります。

例えば、`freecodecamp` または `freecandycamp` のどちらかにマッチさせたいとします。OR アサーションで区切られた2つのパターンを作ることができます。

```js
const regex = /freecodecamp|freecandycamp/i;
```

しかし、大規模な正規表現ではこれがかなり長くなってしまいます。代わりに、OR が必要な文字の周りに非キャプチャグループを作ることができます。

```js
const regex = /free(?:code|candy)camp/i;
```

非キャプチャグループは、`code|candy` のマッチをメモリに別々に保存しません。しかし、可読性やパフォーマンスを損なわずに代替パターンを作るのに役立ちます。
