# --description--

CSS를 다루다 보면 스타일을 브라우저 간에 일관되고 호환 가능하게 유지하는 것, 반복을 피하는 것, 프로젝트 내 CSS 파일이 많아지고 복잡해지는 문제를 처리하는 것과 같은 여러 도전에 직면할 수 있습니다. CSS 프레임워크는 이러한 문제를 처음부터 예방하는 데 매우 유용합니다.

CSS 프레임워크는 HTML 요소에 스타일을 적용하는 데 사용할 수 있는 미리 작성된 CSS 코드 모음입니다.

CSS 프레임워크를 사용하면 작업 속도를 높이고, 웹사이트 전반에 걸쳐 균일한 시각적 스타일을 만들며, 여러 브라우저에서 디자인이 일관되게 보이도록 하고, 전반적으로 CSS 코드를 더 체계적으로 관리할 수 있습니다.

CSS 프레임워크를 도구 상자처럼 생각할 수 있습니다. 필요에 맞는 스타일이나 컴포넌트를 빠르게 찾아 바로 사용할 수 있으며, CSS 속성을 직접 지정할 필요가 없습니다. 이러한 프레임워크는 반응형 레이아웃을 만드는 스타일을 포함해 미리 정의된 스타일과 컴포넌트를 쉽게 사용할 수 있도록 제공합니다.

가장 인기 있는 CSS 프레임워크 중 일부는 Tailwind CSS, Bootstrap, Materialize, Foundation입니다.

이제 두 가지 유형의 CSS 프레임워크에 대해 이야기해 보겠습니다: 유틸리티 우선 CSS 프레임워크와 컴포넌트 기반 CSS 프레임워크입니다.

Tailwind CSS는 유틸리티 우선 CSS 프레임워크입니다. 이는 여백, 패딩, 배경색 설정과 같이 매우 구체적인 목적을 가진 작은 클래스들이 있다는 뜻입니다. 보통 이러한 작은 클래스들을 HTML 요소에 직접 할당할 수 있습니다.

예를 들어:

```html
<button class="bg-blue-500 text-white font-bold py-2 px-4 rounded-full hover:bg-blue-700">
  Button
</button>
```

이 버튼은 파란색 배경에 흰색 굵은 텍스트, 세로 패딩 2rem, 가로 패딩 4rem, 둥근 테두리를 갖습니다. 클래스 `hover:bg-blue-700`는 사용자가 마우스로 버튼 위에 올렸을 때 배경을 진한 파란색으로 바꿉니다.

보시다시피 Tailwind CSS 클래스는 매우 세분화되어 있어 필요에 따라 개별 HTML 요소에 적용할 수 있습니다.

반면에 Bootstrap은 컴포넌트 기반 CSS 프레임워크입니다. 미리 만들어진 컴포넌트와 미리 정의된 스타일을 제공하여 웹사이트에 쉽게 추가할 수 있습니다. 이러한 컴포넌트는 보통 CSS 프레임워크 공식 문서에 있으며, 프로젝트에 빠르게 복사해 붙여넣을 수 있습니다.

예를 들어, Bootstrap에서 리스트 그룹을 만들고 싶다면 HTML에서 다음과 같이 작성합니다:

```html
<div class="card" style="width: 25rem;">
  <ul class="list-group list-group-flush">
    <li class="list-group-item">HTML</li>
    <li class="list-group-item">CSS</li>
    <li class="list-group-item">JavaScript</li>
  </ul>
</div>
```

이 미리 정의된 클래스들은 카드, 리스트 그룹, 리스트 그룹 항목에 표준 스타일을 자동으로 적용합니다.

여기서는 작은 일반 클래스를 추가하는 대신 HTML 구조를 포함한 전체 컴포넌트를 추가하는 것을 볼 수 있습니다. 이것이 Bootstrap이 컴포넌트 기반인 이유입니다.

```html
<div class="card" style="width: 25rem;">
  <ul class="list-group list-group-flush">
    <li class="list-group-item">HTML</li>
    <li class="list-group-item">CSS</li>
    <li class="list-group-item">JavaScript</li>
  </ul>
</div>
```

CSS 프레임워크는 많은 장점이 있지만, 단점도 있습니다. 둘 다 알고 있어야 현명한 선택을 할 수 있습니다.

CSS 프레임워크는 미리 정의된 스타일에 의존하기 때문에, 그 위에 커스텀 CSS를 추가하려 하면 특이성 문제(specificity issue)가 발생할 수 있습니다. 프레임워크가 사용하는 선택자가 커스텀 CSS 선택자보다 특이성이 높을 수 있어 올바른 스타일이 표시되지 않을 수 있습니다. 이를 해결하려면 더 특이성이 높은 CSS 선택자를 사용해야 하므로 추가적인 어려움이 생길 수 있습니다.

또한 표준 스타일이 많은 웹사이트에서 널리 사용되기 때문에 독창성이 어느 정도 제한될 수 있습니다. 하지만 고유한 스타일로 CSS를 커스터마이징하면 이 문제를 해결할 수 있습니다.

마지막으로 CSS 프레임워크가 도입할 수 있는 잠재적인 성능 문제도 고려해야 합니다. 일부 프레임워크는 크기가 꽤 커서 로딩하는 데 시간이 걸릴 수 있습니다.

개발자로서 CSS 프레임워크를 사용하는 것이 프로젝트에 도움이 될지, 완전히 커스텀 CSS를 사용하는 것이 나을지, 아니면 둘을 혼합해서 사용할지 결정하게 될 것입니다.
