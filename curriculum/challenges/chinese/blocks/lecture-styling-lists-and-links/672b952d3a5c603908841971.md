# --interactive--

默认 `link` 样式在提升网页的可用性和无障碍性方面起着关键作用。

这些样式，通常未访问的链接为蓝色，已访问的链接为紫色，已经成为用户在浏览网站时期望并依赖的标准。

默认链接样式的主要目的是提供清晰的视觉提示，帮助用户区分网页上的交互元素和非交互元素。

这种区分对于创建直观且用户友好的浏览体验至关重要。

让我们考虑链接的基本默认样式：

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<a href="/">Example link</a>
```

```css
a:link {
  color: blue;
  text-decoration: underline;
}

a:visited {
  color: purple;
}
```

:::

这些样式具有几个重要的函数。

首先，未访问链接的蓝色在大多数背景色和文本中都很突出，使链接易于识别。这种对比对于用户快速扫描页面并查找导航元素或重要信息至关重要。

下划线进一步强调了文本是可点击的，提供了额外的视觉提示。这对于可能有色盲或难以区分颜色的用户特别有帮助。

已访问链接的颜色变化（通常为紫色）帮助用户跟踪他们访问过的位置。此特色对于浏览大型网站或进行研究非常宝贵，因为它防止用户无意中重复访问相同的页面。

考虑这个超文本标记语言 示例：

:::interactive_editor

```html
<p>Learn more about <a href="https://www.example.com/cats">cats</a> and <a href="https://www.example.com/dogs">dogs</a>.</p>
```

:::

在没有任何自定义 CSS 的情况下，大多数浏览器会将这些链接以蓝色并带下划线的形式渲染。点击其中一个链接后，其颜色会变为紫色，立即向用户提供有关其浏览历史的反馈。

虽然设计师通常会修改这些默认样式以匹配网站的美学，但维护其背后的核心原则至关重要。

如果你选择更改默认样式，请确保 `link` 仍然清晰可区分于普通 `text`，已访问和未访问的 `link` 之间有明显差异，并且所选颜色与背景有足够的对比度。

例如，你可能会使用像这样的自定义样式：

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<a href="/">Example link</a>
```

```css
a:link {
  color: blue;
  text-decoration: none;
  border-bottom: 1px solid blue;
}

a:visited {
  color: purple;
  border-bottom: 1px solid purple;
}
```

:::

这维护了蓝色和紫色的配色方案，同时用底部边框替换了下划线，以获得更现代的外观。

还需要考虑链接的不同状态。除了默认和已访问状态外，链接通常还有悬停和激活状态：

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<a href="/">Example link</a>
```

```css
a:hover {
  color: red;
}

a:active {
  color: darkorange;
}
```

:::

这些状态在用户与链接交互时提供即时反馈，提升网站的整体可用性。

总之，虽然可以自定义 `link` 样式，但应维护默认样式背后的原则。

它们在创建可用且可访问的网页体验中起着关键作用，帮助用户有效地导航和交互内容。

在为你的网站设计链接样式时，始终优先考虑清晰度和用户体验。
