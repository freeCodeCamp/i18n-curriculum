## Algoriti na noti ya Big O

- **Algoriti**: Seti ya maagizo yasiyo na utata kwa kutatua tatizo au kutekeleza zoezi. Algoriti lazima zikamilike kwa idadi ya hatua zisizo na mwisho na kila hatua lazima iwe sahihi na isiyo na utata.

- **Noti ya Big O**: Inaelezea utendaji mbaya zaidi, au kiwango cha ukuaji, cha algoriti kadri ukubwa wa ingizo unavyoongezeka. Inazingatia jinsi matumizi ya rasilimali yanavyokua kwa ukubwa wa ingizo, ikiepuka kuzingatia vipengele vya kudumu na vigezo vya daraja la chini.

### Ugumu wa kawaida wa muda

- **O(1) - Muda wa kudumu**: Algoriti huchukua muda sawa bila kujali ukubwa wa ingizo.

```python
def check_even_or_odd(number):
    if number % 2 == 0:
        return 'Even'
    else:
        return 'Odd'
```

- **O(log n) - Muda wa logaritmiki**: Muda huongezeka polepole kadri ingizo linavyokua. Mara nyingi hutumika katika algoriti zinazopunguza ukubwa wa tatizo kwa sehemu (kama Binary Search).

- **O(n) - Muda wa mstari**: Muda wa utekelezaji huongezeka kwa uwiano na ukubwa wa ingizo.

```python
for grade in grades:
    print(grade)
```

- **O(n log n) - Muda wa mstari-logaritmiki**: Ugumu wa kawaida wa muda wa algoriti za kupanga zenye ufanisi kama Merge Sort na Quick Sort.

- **O(n²) - Muda wa mraba**: Muda wa utekelezaji huongezeka kwa mraba. Mara nyingi huonekana katika mizunguko iliyopangwa ndani.

```python
for i in range(n):
    for j in range(n):
        print("Hello, World!")
```

### Ugumu wa nafasi

- **O(1) - Nafasi ya kudumu**: Algoriti hutumia kiasi sawa cha kumbukumbu bila kujali ukubwa wa ingizo.
- **O(n) - Nafasi ya mstari**: Matumizi ya kumbukumbu huongezeka kwa uwiano na ukubwa wa ingizo.
- **O(n²) - Nafasi ya mraba**: Matumizi ya kumbukumbu huongezeka kwa mraba kwa ukubwa wa ingizo.

## Mbinu za kutatua matatizo

- **Kuelewa tatizo**: Soma tamko la tatizo mara kadhaa. Tambua ingizo, matokeo yanayotarajiwa, na jinsi ya kubadilisha ingizo kuwa matokeo.

- **Msimbo wa bandia**: Maelezo ya kiwango cha juu ya mantiki ya algoriti ambayo haitegemei lugha yoyote. Inatumia lugha ya kawaida iliyochanganywa na miundo ya programu kama `IF`, `ELSE`, `FOR`, `WHILE`.

```md
GET original_string
SET reversed_string = ""
FOR EACH character IN original_string:
  ADD character TO THE BEGINNING OF reversed_string
DISPLAY reversed_string
```

- **Matukio ya pembe**: Ingizo maalum, halali yanayotokea katika mipaka ya kile algoriti inapaswa kushughulikia. Daima zingatia na jaribu matukio ya pembe.

## Safu za data

- **Safu za data za kudumu**: Zina ukubwa uliowekwa wakati wa uanzishaji. Vipengele huhifadhiwa katika maeneo ya kumbukumbu yanayopakana. Ukubwa hauwezi kubadilika wakati wa utekelezaji wa programu.

- **Safu za data zinazobadilika**: Zinaweza kukua au kupungua moja kwa moja wakati wa utekelezaji wa programu. Zinashughulikia mabadiliko ya ukubwa kwa kunakili moja kwa moja kwenye safu kubwa zaidi inapohitajika.

### Orodha za Python (safu za data zinazobadilika)

```python
numbers = [3, 4, 5, 6]

# Access elements
numbers[0]  # 3

# Update elements
numbers[2] = 16

# Add elements
numbers.append(7)
numbers.insert(3, 15)  # Insert at specific index

# Remove elements
numbers.pop(2)  # Remove at specific index
numbers.pop()   # Remove last element
```

### Ugumu wa muda kwa safu za data zinazobadilika

- **Upatikanaji**: O(1)
- **Ingiza mwishoni**: O(1) wastani, O(n) wakati wa mabadiliko ya ukubwa
- **Ingiza katikati**: O(n)
- **Futa**: O(n) kwa katikati, O(1) mwishoni

## Staki

- **Staki**: Muundo wa data wa Last-In, First-Out (LIFO). Vipengele huongezwa na kufutwa kutoka juu tu.

- **Hatua ya kusukuma (Push)**: Kuongeza kipengele juu ya staki. Ugumu wa muda: O(1).

- **Hatua ya kutoa (Pop)**: Kuondoa kipengele kutoka juu ya staki. Ugumu wa muda: O(1).

```python
# Using Python list as stack
stack = []

# Push operations
stack.append(1)
stack.append(2)
stack.append(3)

# Pop operations
top_element = stack.pop()  # Returns 3
```

## Foleni

- **Foleni**: Muundo wa data wa First-In, First-Out (FIFO). Vipengele huongezwa nyuma na kufutwa mbele.

- **Hatua ya kuingiza (Enqueue)**: Kuongeza kipengele nyuma ya foleni. Ugumu wa muda: O(1).

- **Hatua ya kuondoa (Dequeue)**: Kuondoa kipengele mbele ya foleni. Ugumu wa muda: O(1).

```python
from collections import deque

# Using deque for efficient queue operations
queue = deque()

# Enqueue operations
queue.append(1)
queue.append(2)
queue.append(3)

# Dequeue operations
first_element = queue.popleft()  # Returns 1
```

## Orodha zilizounganishwa

- **Orodha zilizounganishwa**: Muundo wa data wa mstari ambapo kila nodi ina data na rejea kwa nodi inayofuata. Nodi zimeunganishwa kama mnyororo.

### Orodha zilizounganishwa za peke yake

- **Muundo**: Kila nodi ina data na rejea moja kwa nodi inayofuata.
- **Kupitia**: Inaweza kusogea mbele tu kutoka kichwa hadi mkia.
- **Nodi ya kichwa**: Nodi ya kwanza katika orodha, kawaida ndiyo nodi pekee inayopatikana moja kwa moja.
- **Nodi ya mkia**: Nodi ya mwisho katika orodha, inaonyesha `None`.

### Hatua na ugumu wa muda

- **Ingiza mwanzoni**: O(1)
- **Ingiza mwishoni**: O(n) - lazima pitia hadi mwisho
- **Ingiza katikati**: O(n) - lazima pitia hadi nafasi
- **Futa mwanzoni**: O(1)
- **Futa mwishoni**: O(n) - lazima pitia kupata nodi ya awali
- **Futa katikati**: O(n) - lazima pitia kupata nodi

### Orodha zilizounganishwa mara mbili

- **Muundo**: Kila nodi ina data na rejea mbili: nodi inayofuata na nodi iliyopita.
- **Kupitia**: Inaweza kusogea katika mwelekeo yote miwili.
- **Kumbukumbu**: Inahitaji kumbukumbu zaidi kuliko orodha zilizounganishwa za peke yake kwa sababu ya rejea ya ziada.

## Ramani za hash na seti

### Ramani na ramani za hash

- **Ramani (Aina ya data ya muhtasari)**: Inasimamia makusanyo ya jozi za ufunguo-thamani. Kila ufunguo lazima uwe wa kipekee, lakini thamani zinaweza kurudiwa.

- **Ramani ya hash**: Utekelezaji halisi wa ramani ADT kwa kutumia mbinu ya hashing. Inatumia kitendakazi cha hash kuunda thamani za hash kwa funguo, ambazo huamua mahali pa kuhifadhi katika safu ya msingi.

### Kamusi za Python (ramani za hash)

```python
# Creating dictionaries
my_dictionary = {
    "A": 1,
    "B": 2, 
    "C": 3
}

# Alternative creation
my_dictionary = dict(A=1, B=2, C=3)

# Access and modify
value = my_dictionary["A"]  # 1
my_dictionary["A"] = 4      # Update value
del my_dictionary["A"]      # Remove key-value pair

# Check membership
"C" in my_dictionary

# Get keys, values, items
my_dictionary.keys()
my_dictionary.values()
my_dictionary.items()
```

### Ugumu wa muda kwa ramani za hash

- **Kesi ya wastani**: O(1) kwa kuingiza, kupata, kufuta
- **Kesi mbaya zaidi**: O(n) wakati kuna migongano mingi ya hash

### Seti

- **Seti**: Makusanyo yasiyo na mpangilio ya vipengele vya kipekee. Hakuna rudufu, hakuna mpangilio maalum unaohifadhiwa.

- **Vipengele visivyobadilika tu**: Seti zinaweza kuwa na aina za data zisizobadilika tu (nambari, mfuatano wa herufi, tuple) kwa sababu thamani za hash lazima zibaki thabiti.

```python
# Creating sets
numbers = {1, 2, 3, 4}
empty_set = set()  # Must use set(), not {}

# Add and remove elements
numbers.add(5)
numbers.remove(4)      # Raises KeyError if not found
numbers.discard(4)     # No error if not found

# Set operations
set_a = {1, 2, 3, 4}
set_b = {2, 3, 4, 5, 6}

# Union, intersection, difference, symmetric difference
set_a.union(set_b)                    # or set_a | set_b
set_a.intersection(set_b)             # or set_a & set_b
set_a.difference(set_b)               # or set_a - set_b
set_a.symmetric_difference(set_b)     # or set_a ^ set_b

# Subset and superset checks
set_a.issubset(set_b)
set_a.issuperset(set_b)
set_a.isdisjoint(set_b)

# Membership testing
5 in numbers
```

### Ugumu wa muda kwa seti

- **Kesi ya wastani**: O(1) kwa kuongeza, kuondoa, kujaribu uanachama
- **Kesi mbaya zaidi**: O(n) kutokana na migongano ya hash

## Migongano ya hash

- **Mgongano wa hash**: Hutokea wakati funguo mbili tofauti zinazalisha thamani sawa ya hash.

- **Mbinu za kutatua migongano**:
  - **Uunganishaji wa mfululizo**: Kielezo cha safu kinaonyesha orodha iliyounganishwa inayohifadhi vipengele vyote vyenye thamani sawa ya hash
  - **Anwani wazi**: Tafuta kielezo kinachopatikana kinachofuata kwa kutumia mfululizo uliowekwa awali

## Wakati wa kutumia kila muundo wa data

- **Orodha**: Unapohitaji upatikanaji uliopangwa, uliyo na kielezo na haujui ukubwa mapema
- **Staki**: Kwa shughuli za LIFO (kazi ya kufuta, tathmini ya usemi, kurudi nyuma)
- **Foleni**: Kwa shughuli za FIFO (ratiba ya kazi, utafutaji wa upana kwanza)
- **Orodha zilizounganishwa**: Unapohitaji kuingiza/kufuta mara kwa mara mwanzoni, ukubwa usiojulikana, hakuna upatikanaji wa nasibu unaohitajika
- **Ramani za hash**: Kwa upatikanaji wa haraka wa jozi za ufunguo-thamani, kuhesabu matukio, kuhifadhi kwa muda
- **Seti**: Kwa ukaguzi wa kipekee, shughuli za seti za kihesabu, kuondoa rudufu
