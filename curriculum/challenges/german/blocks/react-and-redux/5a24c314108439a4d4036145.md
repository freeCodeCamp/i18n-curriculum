# --description--

Die `Provider`-Komponente ermöglicht es Ihnen, `state` und `dispatch` an Ihre React-Komponenten zu übergeben, aber Sie müssen genau angeben, auf welchen Zustand und welche Aktionen Sie zugreifen möchten. So stellen Sie sicher, dass jede Komponente nur Zugriff auf den Zustand hat, den sie benötigt. Dies erreichen Sie, indem Sie zwei Funktionen erstellen: `mapStateToProps()` und `mapDispatchToProps()`.

In diesen Funktionen deklarieren Sie, auf welche Teile des Zustands Sie zugreifen möchten und welche Action Creators Sie ausführen können müssen. Sobald diese Funktionen vorhanden sind, sehen Sie in einer weiteren Aufgabe, wie Sie die React Redux `connect`-Methode verwenden, um sie mit Ihren Komponenten zu verbinden.

**Hinweis:** Im Hintergrund verwendet React Redux die `store.subscribe()`-Methode, um `mapStateToProps()` zu implementieren.

# --instructions--

Erstellen Sie eine Funktion `mapStateToProps()`. Diese Funktion soll `state` als Argument übernehmen und dann ein Objekt zurückgeben, das diesen Zustand auf bestimmte Eigenschaftsnamen abbildet. Diese Eigenschaften werden für Ihre Komponente über `props` zugänglich. Da in diesem Beispiel der gesamte Zustand der App in einem einzigen Array gehalten wird, können Sie diesen gesamten Zustand an Ihre Komponente übergeben. Erstellen Sie eine Eigenschaft `messages` in dem zurückgegebenen Objekt und setzen Sie sie auf `state`.

# --hints--

Das const `state` sollte ein leeres Array sein.

```js
assert(Array.isArray(state) && state.length === 0);
```

`mapStateToProps` sollte eine Funktion sein.

```js
assert(typeof mapStateToProps === 'function');
```

`mapStateToProps` sollte ein Objekt zurückgeben.

```js
assert(typeof mapStateToProps() === 'object');
```

Wenn Sie ein Array als Zustand an `mapStateToProps` übergeben, sollte dieses Array einem Schlüssel von `messages` zugewiesen zurückgegeben werden.

```js
assert(mapStateToProps(['messages']).messages.pop() === 'messages');
```
