# --description--

أدوات المطوِّر، وأدوات الفحص، ومدققات CSS هي موارد أساسية لتصحيح أخطاء CSS وضمان أن تكون أوراق الأنماط خالية من الأخطاء ومحسّنة.

تُوفر هذه الأدوات رؤى لا تقدر بثمن حول كيفية تطبيق CSS الخاص بك ويمكنها المساعدة في تحديد المشاكل المحتملة.

لنبدأ بأدوات مطوِّر المتصفح، المعروفة عادةً باسم DevTools. معظم المتصفحات الحديثة، بما في ذلك Chrome وFirefox وSafari، تأتي مزودة بأدوات DevTools مدمجة. للوصول إليها، يمكنك النقر بزر الفأرة الأيمن على عنصر في صفحة الويب الخاصة بك واختيار "Inspect" أو استخدام اختصارات لوحة المفاتيح مثل `F12` أو `Cmd+Option+I` (على macOS).

تتيح لك DevTools فحص وتعديل CSS الخاص بك في الوقت الحقيقي. يعرض جزء الأنماط جميع قواعد CSS المطبقة على العنصر المحدد، بما في ذلك الأنماط الموروثة.

يمكنك تشغيل وإيقاف الخصائص الفردية، تعديل القيم، وحتى إضافة قواعد جديدة مباشرة في المتصفح. هذه الملاحظات الفورية مفيدة جدًا لتجربة أنماط مختلفة دون تعديل الكود المصدري.

أداة الفحص، التي هي جزء من DevTools، تتيح لك تحريك المؤشر فوق العناصر في صفحتك ورؤية نموذج الصندوق الخاص بها، بما في ذلك الهوامش والحدود والحشو ومنطقة المحتوى. هذا مفيد بشكل خاص لتشخيص مشاكل التخطيط أو لفهم سبب تموضع العناصر بطريقة معينة.

مدققات CSS هي أداة مهمة أخرى لتصحيح الأخطاء. مدقق CSS الخاص بـ W3C هو خيار شائع. يتحقق من CSS الخاص بك مقابل المواصفات الرسمية ويبلغ عن أي أخطاء أو تحذيرات. لاستخدامه، يمكنك رفع ملف CSS الخاص بك، أو إدخال CSS مباشرة، أو تقديم عنوان URL للتحقق.

على سبيل المثال، لنفترض أنك تعمل مع CSS التالي:

```css
.container {
  width: 100%;
  height: 200px
  background-color: #F0F0F0;
}
```

سيشير المدقق إلى وجود فاصلة منقوطة مفقودة بعد خاصية `height`. هذا النوع من الأخطاء قد يكون من السهل تجاهله لكنه قد يسبب مشاكل كبيرة في ورقة الأنماط الخاصة بك.

عند تصحيح أخطاء التصميمات المتجاوبة، تكون ميزة محاكاة الأجهزة في DevTools لا تقدر بثمن. تتيح لك محاكاة كيف يبدو موقعك على أحجام شاشات وأجهزة مختلفة. هذا يمكن أن يساعدك في تحديد مشاكل نقاط التوقف أو الأنماط التي لا تتناسب جيدًا مع أحجام إطارات العرض المختلفة.

تذكر، تصحيح أخطاء CSS الفعال غالبًا ما يتطلب مزيجًا من هذه الأدوات. قد تبدأ باستخدام مدقق لالتقاط أي أخطاء صياغية، ثم تستخدم DevTools لفحص عناصر محددة وتجربة التعديلات. بعد ذلك، تساعدك أداة محاكاة الأجهزة على التأكد من أن أنماطك تعمل عبر أحجام الشاشات المختلفة.

بإتقان هذه الأدوات، يمكنك تسريع عملية تصحيح أخطاء CSS بشكل كبير وإنشاء أوراق أنماط أكثر متانة وخالية من الأخطاء.

الاستخدام المنتظم لهذه التقنيات في تصحيح الأخطاء لن يساعدك فقط في حل المشاكل الفورية، بل سيحسن أيضًا فهمك العام لـ CSS وكيفية تفاعله مع HTML.
