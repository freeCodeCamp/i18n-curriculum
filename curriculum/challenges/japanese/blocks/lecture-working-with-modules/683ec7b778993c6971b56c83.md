# --description--

ソフトウェア開発において、ライブラリは開発者のための道具箱のようなものです。

コードのすべての部分を最初から自分で実装する代わりに、ライブラリは関数、クラス、データ構造など、プロジェクトで使えるあらかじめ書かれた再利用可能なコードを提供してくれます。

Pythonには多くの組み込みモジュールを含む広範な標準ライブラリがあります。これらはすべて標準化され、よく検証されたソリューションで、プログラマーが日常的に直面する多くの問題やタスクに対応しています。例えば：

* オペレーティングシステムとのやり取り
    
* ファイル操作
    
* ネットワーキング
    
* 日付と時刻の操作
    
* 数学的演算の実行
    
* 正規表現の使用
    
* コードのテストとデバッグ
    
* その他多数
    

よく使われる組み込みモジュールの例として、`math`、`random`、`re`（「正規表現」の略）および`datetime`があります。

`math`モジュールには、より複雑な数学的演算を行うための便利な関数が含まれています。

`random`モジュールは乱数生成に役立ちます。

`re`モジュールは正規表現の操作に使われます。

そして`datetime`モジュールはPythonで日付や時刻を扱うのに便利です。

では、これらの組み込みモジュールで定義された変数、定数、関数、クラスにどうやってアクセスするのでしょうか？

インポート文を使います。インポート文はモジュールをPythonスクリプトに読み込むためのものです。通常、インポート文はファイルの先頭に書きます。また、必要に応じてカスタマイズもできます。まず、`import`文を使い、その後にモジュール名を書きます。

```python
import module_name
```

例えば、`math`モジュールをインポートしたい場合、ファイルの先頭に次のように書きます。

```python
import math
```

その後、スクリプト内でそのモジュールの関数を呼び出す必要がある場合は、ドット記法を使い、モジュール名の後に関数名を書きます。

```python
module_name.function_name() 
```

例えば、36の平方根を求めるには、`math`の後にドットを付けて`sqrt`（平方根の略）を書き、丸括弧内に必要な引数を渡します。この場合は平方根を求めたい数値だけを渡します。

```python
math.sqrt(36)
```

これは最も基本的なインポート文の形ですが、他にも方法があります。

モジュールを別名（「エイリアス」とも呼ばれます）でインポートしたい場合は、インポート文の最後に`as`の後にエイリアス名を書きます。

```python
import module_name as module_alias
```

これは長いモジュール名を短くしたり、名前の衝突を避けたりするためによく使われます。

例えば、`math`モジュールをコード内で`m`として参照したい場合、次のようにエイリアスを割り当てます。

```python
import math as m 
```

その後、エイリアスを使ってモジュールの要素にアクセスできます。

```python
m.sqrt(36)
```

しかし、モジュールのすべてをインポートする必要がない場合もあります。特定の関数やクラスだけが必要な場合です。Pythonにはそのための方法があります。

インポート文は`from`で始まり、モジュール名の後に`import`キーワードを書き、インポートしたい要素の名前を続けます。

```python
from module_name import name1, name2
```

これにより、Pythonスクリプト内でモジュール名の接頭辞なしでこれらの名前を使えます。

これらの名前にエイリアスを割り当てたい場合は、各名前の後に`as`キーワードを書き、使いたいエイリアスを指定します。

```python
from module_name import name1 as alias1, name2 as alias2
```

例えば、`math`モジュールからradians、sine、cosine関数だけをインポートしたい場合は次のように書きます。

```python
from math import radians, sin, cos
```

これで、`math`モジュールの接頭辞なしでこれらの関数を直接呼び出せます。

ここにより詳しい例があります。

特定の角度（最初は度数法で表現）について、`radians`関数を呼び出してラジアンに変換し、その後sineとcosine関数を呼び出してラジアン単位の角度を渡します。

```python
from math import radians, sin, cos

angle_degrees = 40
angle_radians = radians(angle_degrees)

sine_value = sin(angle_radians)
cos_value = cos(angle_radians)

print(sine_value) # 0.6427876096865393
print(cos_value)  # 0.766044443118978
```

モジュール名の接頭辞なしで関数を直接呼び出しているのがわかりますね。これはこの代替構文で関数をインポートしたためです。

これは便利ですが、同じ名前の関数や変数がスクリプト内にすでに定義されている場合、名前の衝突が起こる可能性があります。どのタイプのインポート文を使うか選ぶ際にはこの点に注意してください。

最後に、アスタリスクで終わるインポート文があります。アスタリスクは、そのモジュール内のすべてをインポートし、モジュール名の接頭辞なしで使いたいことをPythonに伝えます。

```python
from module_name import *
```

例えば、`math`モジュールをこのようにインポートすると、そのモジュール内で定義された関数をモジュール名を指定せずに呼び出せます。いくつか例を示します。

```python
from math import *
print(sqrt(36))  # 6.0
print(pow(5, 2)) # 25.0
print(exp(1))    # 2.718281828459045
```

ただし、これは名前空間の衝突を引き起こしやすく、どこから名前が来ているのか分かりにくくなるため、一般的には推奨されません。

インポート文は関数、クラス、定数、変数、その他モジュール内で定義された要素すべてに対して同じように機能します。

例えば、`math`モジュールの定数`pi`の例です。

```python
import math
print(math.pi)
```

そして、`datetime`モジュールのクラスの例です。1959年7月15日を表す日付オブジェクトを作成し、その`date`オブジェクトを変数に代入し、ドット記法で日、月、年を個別にアクセスしています。

```python
import datetime
birthday = datetime.date(1959, 7, 15)
print(birthday.day)    # 15
print(birthday.month)  # 7
print(birthday.year)   # 1959
```

モジュールの内容については、そのモジュールの公式Pythonドキュメントでさらに詳しく調べられます。

さて、モジュールについて理解が深まったところで、Pythonスクリプトで非常に重要な慣用句についても知っておくべきです。これらは密接に関連しています。

```python
if __name__ == '__main__': 
    # Code
```

`__name__`はPythonの特別な組み込み変数です。

Pythonファイルが直接実行されると、この変数の値は文字列`"__main__"`に設定されます。

しかし、そのPythonファイルが別のPythonスクリプトにモジュールとしてインポートされると、`__name__`変数の値はそのモジュールの名前（通常は`.py`拡張子を除いたファイル名）に設定されます。

だからこそ、Pythonスクリプトでこの条件文をよく見かけます。これはPythonスクリプトがメインプログラムとして実行されている場合にのみ実行したいコードを含んでいます。

```python
if __name__ == '__main__': 
    # Code
```

しかし、スクリプトがモジュールとしてインポートされた場合、そのブロック内のコードは実行されません。

これはPythonスクリプトに2つの目的を持たせるのに役立ちます。直接実行してメインのロジックを実行することも、別のモジュールにインポートしてメインのロジックを実行せずに使うこともできます。
