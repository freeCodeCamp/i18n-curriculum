# --description--

Impariamo come trovare o sostituire tutte le occorrenze di un pattern in una stringa.

Hai già imparato i metodi `replace()` e `match()`, così come il modificatore globale `g`. Ora puoi combinare queste conoscenze per gestire tutti i pattern in una stringa. Ricordiamo il nostro codice originale per il match:

```js
const regex = /freeCodeCamp/;
const match = "freeCodeCamp".match(regex);
console.log(match);
```

E il nostro oggetto match risultante:

```js
// [
//   'freeCodeCamp',
//   index: 0,
//   input: 'freeCodeCamp',
//   groups: undefined
// ]
```

Ma cosa succede se abbiamo una stringa con più occorrenze di `freecodecamp` da trovare? Vediamo come si comporta `match()` in questo caso. Inseriamo anche il nostro vecchio esempio `replace()`, giusto per confrontare:

```js
const regex = /freecodecamp/;
const str = "freecodecamp is the best we love freecodecamp";
const matched = str.match(regex);
const replaced = str.replace(regex, "freeCodeCamp");
console.log(matched);
console.log(replaced);
```

E il risultato è questo:

```js
// [
//   'freecodecamp',
//   index: 0,
//   input: 'freecodecamp is the best we love freecodecamp',
//   groups: undefined
// ]
// freeCodeCamp is the best we love freecodecamp
```

Oh no! `match()` ha restituito solo il primo match, e `replace()` ha sostituito solo il primo match. Questo perché, per impostazione predefinita, `match()` e `replace()` operano solo sulla prima occorrenza del pattern.

Per fortuna, puoi evitare questo usando il modificatore globale nella tua espressione regolare. Aggiungiamolo alla nostra:

```js
const regex = /freecodecamp/g;
const str = "freecodecamp is the best we love freecodecamp";
const matched = str.match(regex);
const replaced = str.replace(regex, "freeCodeCamp");
console.log(matched);
console.log(replaced);
```

E confermiamo il risultato:

```js
// [ 'freecodecamp', 'freecodecamp' ]
// freeCodeCamp is the best we love freeCodeCamp
```

Ha funzionato! La nostra chiamata a replace ha sostituito tutte le stringhe minuscole `freecodecamp`, e il nostro metodo `match()` ha trovato entrambe.

La cosa interessante è che quando usi il modificatore globale con `match()`, perdi le informazioni extra sui gruppi di cattura e sugli indici della stringa che normalmente sono presenti nell'array di match.

Per fortuna, l'aggiornamento ECMAScript del 2019 ci ha portato due nuovi metodi: `matchAll()` e `replaceAll()`. Come i loro equivalenti singolari, questi metodi accettano una stringa o un'espressione regolare, e `replaceAll()` accetta anche un secondo argomento come stringa di sostituzione.

Ma a differenza dei metodi precedenti, `replaceAll()` e `matchAll()` lanceranno un errore se gli passi un'espressione regolare senza il modificatore globale. Aggiorniamo il nostro codice per usare questi nuovi metodi:

```js
const pattern = "freecodecamp";
const str = "freecodecamp is the best we love freecodecamp";
const matched = str.matchAll(pattern);
const replaced = str.replaceAll(pattern, "freeCodeCamp");
console.log(matched);
console.log(replaced);
```

E il nostro risultato:

```js
// {}
// freeCodeCamp is the best we love freeCodeCamp
```

Buone notizie! Il nostro `replaceAll()` ha funzionato esattamente come volevamo – ha sostituito tutte le occorrenze della `freecodecamp` minuscola con la versione correttamente camelCase.

Ma cos'è quell'oggetto vuoto? Beh, `matchAll()` restituisce un tipo speciale di oggetto chiamato `Iterator`, che la console di freeCodeCamp non è pronta a gestire.

Se diamo un'occhiata nella console del browser, il `Iterator` ha un metodo `next()` che possiamo chiamare per ottenere il valore successivo:

```js
// RegExpStringIterator {  }
//  <prototype>: RegExp String Iterator {
//    next: ƒ next(),
//    Symbol(Symbol.toStringTag): "RegExp String Iterator"
//    <prototype>: Object { ... }
//  }
```

Procediamo e chiamiamo `matched.next()`, e registriamo il risultato:

```js
// {
//   "done": false,
//   "value": [
//     0: "freecodecamp"
//     groups: undefined
//     index: 0
//     input: "freecodecamp is the best we love freecodecamp"
//   ]
// }
```

Ecco il nostro array di match! `next()` ci dà un oggetto con due valori: `done`, che è `false` quando ci sono altri elementi disponibili nell'iteratore, e `value` che è il valore su cui abbiamo appena iterato. Quindi, se lo chiamiamo un'altra volta:

```js
const regex = /freecodecamp/g;
const str = "freecodecamp is the best we love freecodecamp";
const matched = str.matchAll(regex);
const replaced = str.replaceAll(regex, "freeCodeCamp");
console.log(matched);
console.log(replaced);
console.log(matched.next());
console.log(matched.next());
```

```js
// {
//   "done": false,
//   "value": Array [ "freecodecamp"]
// }
//
// {
//   "done": false,
//   "value": Array [ "freecodecamp"]
// }
```

Aspetta, perché dice che `done` è ancora `false`? Dovrebbero esserci solo due match nell'array, giusto? Chiamiamolo una terza volta e vediamo cosa otteniamo:

```js
// {
//   "done": false,
//   "value": Array [ "freecodecamp"]
// }
//
// {
//   "done": false,
//   "value": Array [ "freecodecamp"]
// }
//
// {
//   "done": true,
//   "value": undefined
// }
```

`done` è finalmente `true`, ma perché quel valore è `undefined`? Beh, a quanto pare, l'iteratore `matchAll()` è lazy. Non trova tutti i match in una volta. Trova un match solo quando gli dici di farlo chiamando `next()`.

Finché trova un match, non è `done`. Quando non riesce a trovare un match e restituisce `undefined`, allora è `done`. Questo può sembrare scomodo, ma può essere molto utile quando la tua espressione regolare è computazionalmente costosa.

Se il tuo esempio è meno impegnativo, come il nostro, puoi saltare questa funzionalità ed estrarre tutti i match in una volta convertendolo in un array. Questo si ottiene chiamando `Array.from()` e passando il tuo iteratore come argomento.

Aggiorniamo il nostro codice per usare questo – puliamo le nostre chiamate `replaceAll` dato che sappiamo che funziona:

```js
const regex = /freecodecamp/g;
const str = "freecodecamp is the best we love freecodecamp";
const matched = str.matchAll(regex);
console.log(Array.from(matched));
```

E finalmente otteniamo il nostro array di match:

```js
// [
//   'freecodecamp',
//   index: 0,
//   input: 'freecodecamp is the best we love freecodecamp',
//   groups: undefined
// ]
//
// [
//   'freecodecamp',
//   index: 33,
//   input: 'freecodecamp is the best we love freecodecamp',
//   groups: undefined
// ]
```

Questi potenti metodi possono aiutarti a manipolare ed estrarre dati dalle stringhe senza dover sacrificare prestazioni o leggibilità.
