# --instructions--

このチャレンジでは、**K-Nearest Neighbors** を使った書籍推薦アルゴリズムを作成します。

このプロジェクトでは、Book-Crossingsデータセットを使用します。このデータセットには、90,000人のユーザーによる270,000冊の書籍の評価（1～10のスケール）が110万件含まれています。データセットはノートブックにすでにインポートされているため、追加のダウンロードは不要です。

`NearestNeighbors` を`sklearn.neighbors`から使って、ある書籍に似ている書籍を表示するモデルを開発してください。Nearest Neighborsアルゴリズムは、インスタンスの「近さ」を測るために距離を計算します。

`get_recommends`という名前の関数を作成してください。この関数は書籍のタイトル（データセット内のもの）を引数に取り、その書籍に似ている5冊の書籍とそれぞれの距離のリストを返します。

このコード：

```py
get_recommends("The Queen of the Damned (Vampire Chronicles (Paperback))")
```

は次のように返すはずです：

```py
[
  'The Queen of the Damned (Vampire Chronicles (Paperback))',
  [
    ['Catch 22', 0.793983519077301], 
    ['The Witching Hour (Lives of the Mayfair Witches)', 0.7448656558990479], 
    ['Interview with the Vampire', 0.7345068454742432],
    ['The Tale of the Body Thief (Vampire Chronicles (Paperback))', 0.5376338362693787],
    ['The Vampire Lestat (Vampire Chronicles, Book II)', 0.5178412199020386]
  ]
]
```

`get_recommends()`から返されるデータはリストであることに注意してください。リストの最初の要素は関数に渡された書籍のタイトルです。リストの2番目の要素はさらに5つのリストを含むリストで、それぞれのリストには推薦された書籍とその書籍から関数に渡された書籍までの距離が含まれています。

（任意ですが）データセットをグラフ化すると、多くの書籍が頻繁に評価されていないことがわかります。統計的に有意な結果を得るために、評価数が200未満のユーザーと評価数が100未満の書籍はデータセットから除外してください。

最初の3つのセルは必要なライブラリとデータをインポートしています。最後のセルはテスト用です。コードはこれらのセルの間にすべて書いてください。
