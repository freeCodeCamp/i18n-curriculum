# --description--

Nachdem eine Aktion erstellt und ausgelöst wurde, muss der Redux-Store wissen, wie er auf diese Aktion reagieren soll. Dies ist die Aufgabe einer `reducer`-Funktion. Reducer in Redux sind verantwortlich für die Zustandsänderungen, die als Reaktion auf Aktionen stattfinden. Eine `reducer` nimmt `state` und `action` als Argumente und gibt immer einen neuen `state` zurück. Es ist wichtig zu verstehen, dass dies die **einzige** Rolle des Reducers ist. Er hat keine Nebeneffekte – er ruft niemals einen API-Endpunkt auf und enthält keine versteckten Überraschungen. Der Reducer ist einfach eine reine Funktion, die Zustand und Aktion nimmt und dann den neuen Zustand zurückgibt.

Ein weiteres wichtiges Prinzip in Redux ist, dass `state` schreibgeschützt ist. Mit anderen Worten, die `reducer`-Funktion muss **immer** eine neue Kopie von `state` zurückgeben und darf den Zustand niemals direkt verändern. Redux erzwingt keine Unveränderlichkeit des Zustands, aber Sie sind dafür verantwortlich, diese in Ihrem Reducer-Code durchzusetzen. Das werden Sie in späteren Aufgaben üben.

# --instructions--

Der Code-Editor enthält das vorherige Beispiel sowie den Anfang einer `reducer`-Funktion für Sie. Füllen Sie den Körper der `reducer`-Funktion so aus, dass sie, falls sie eine Aktion vom Typ `'LOGIN'` erhält, ein Zustandsobjekt mit `login` zurückgibt, das auf `true` gesetzt ist. Andernfalls gibt sie den aktuellen `state` zurück. Beachten Sie, dass der aktuelle `state` und die ausgelöste `action` an den Reducer übergeben werden, sodass Sie direkt mit `action.type` auf den Typ der Aktion zugreifen können.

assert(loginAction().type === 'LOGIN');

Der Store sollte mit einem Objekt initialisiert werden, dessen Eigenschaft `login` auf `false` gesetzt ist.

```js
assert(store.getState().login === false);
```

Das Auslösen von `loginAction` sollte die Eigenschaft `login` im Store-Zustand auf `true` aktualisieren.

```js
assert(
  (function () {
    const initialState = store.getState();
    store.dispatch(loginAction());
    const afterState = store.getState();
    return initialState.login === false && afterState.login === true;
  })()
);
```

Wenn die Aktion nicht vom Typ `LOGIN` ist, sollte der Store den aktuellen Zustand zurückgeben.

```js
assert(
  (function () {
    store.dispatch({ type: '__TEST__ACTION__' });
    let afterTest = store.getState();
    return typeof afterTest === 'object' && afterTest.hasOwnProperty('login');
  })()
);
```

const defaultState = {
  login: false
};

const reducer = (state = defaultState, action) => {

  if (action.type === 'LOGIN') {
    return {login: true}
  }

  else {
    return state
  }

};

const store = Redux.createStore(reducer);

const loginAction = () => {
  return {
    type: 'LOGIN'
  }
