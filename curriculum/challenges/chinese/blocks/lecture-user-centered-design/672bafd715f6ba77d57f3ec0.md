# --description--

用户研究是对使用你产品的人的系统性研究。目标是衡量用户需求、行为和痛点。

用户研究有多种形式。也许最常见的一种是净推荐值，或称 NPS。NPS 衡量你的用户向朋友推荐你的产品的可能性。NPS 通过在用户旅程中的关键节点提供的调查来测量，例如在 7 天、30 天和 90 天之后。NPS 的评分范围是 0 到 10，9 和 10 表示你的站点的积极推广者。

另一个研究向量是退出访谈。这是你在用户取消订阅或删除账户时向他们展示的调查。来自此调查的数据可以让你洞察导致用户流失的因素，从而加以解决。

用户测试，另一方面，是指在用户与您的应用接口时捕获数据的实践。例如，正在进行测试版测试的视频游戏就是一种用户测试。作为网页开发者，你可能会遇到 A/B 测试。A/B 测试涉及将新建特色发布给随机选定的用户子集。然后你可以杠杆分析数据来确定该特色是否有益。

Finally, user requirements refer to the stories or rubric that your application needs to follow.  This can inform the development process. 用户需求可以通过用户研究或行业标准来确定。 它们甚至可以由利益相关者的意见来定义。

这些需求可能是功能性的，意味着它们规定了你的 `application` 应该如何工作，或者是非功能性的，意味着它们定义了你的 `application` 应该如何表现。用户需求也不是静态的。来自用户测试和用户调研的信息都会影响需求，并且随着你的用户群体的变化而变化。

理解差异对于收集最准确的数据至关重要，这样你才能为你的最终用户提供最佳体验。
