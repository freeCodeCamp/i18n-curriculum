# --interactive--

JavaScriptの`typeof`演算子は、変数や値のデータ型を確認できるシンプルで強力なツールです。常に型を示す文字列を返します。

いくつか例を見てみましょう。

:::interactive_editor

```js
let num = 42;
console.log(typeof num); // "number"
```

:::

この最初の例では、`num`という変数を作成し、数値の`42`を代入しました。`typeof`という名前の変数に対して`num`演算子を使うと、文字列の`number`が返されます。

次に、`typeof`演算子を`isUserLoggedIn`という変数に使った例です。

:::interactive_editor

```js
let isUserLoggedIn = true;
console.log(typeof isUserLoggedIn); // "boolean"
```

:::

`typeof`変数に対して`isUserLoggedIn`演算子を使うと、ブール値の`boolean`が代入されているため、文字列の`true`が返されます。

`typeof`演算子は、デバッグ中やコードで扱っているデータの種類を理解しようとするときに特に役立ちます。

しかし、JavaScriptの`null`にはよく知られた特異な挙動があります。

例を見てみましょう。

:::interactive_editor

```js
let exampleVariable = null;
console.log(typeof exampleVariable); // "object"
```

:::

この例では、`exampleVariable`という変数に`null`の値を代入しています。しかし`typeof`演算子を使うと、文字列の`object`が返されます。

これはJavaScriptの初期から存在するバグとして広く認識されています。この挙動の理由は、JavaScriptが最初に設計されたときの仕組みに由来します。

言語が最初に実装された際、`null`のような値は特別な種類のオブジェクトとして表されていたため、この予期しない結果が生じました。

残念ながらこれは言語の一部となっており、混乱を招きますが知っておく必要があります。
