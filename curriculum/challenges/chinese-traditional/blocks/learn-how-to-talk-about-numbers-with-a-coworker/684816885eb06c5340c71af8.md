# --explanation--

`Initial unit tests` 是測試的第一步。開發者會檢查程式碼的小零件（`units`），以確保它們能執行預期的功能。 例如：

`We wrote initial unit tests to check the main functions.` — 團隊創建了早期測試，以確保程式碼中最重要的零件能正確運作。

`Quality assurance (QA) checks` 是由 QA 團隊進行的最終測試，以確保軟體可靠、使用者友善且準備好發布。例如：

`Regular QA checks help prevent bugs from reaching users.` — QA 的頻繁測試有助於及早發現錯誤，讓使用者不會遇到問題。
