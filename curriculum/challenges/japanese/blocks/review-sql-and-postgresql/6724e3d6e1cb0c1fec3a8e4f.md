## リレーショナルデータベースの紹介

- **リレーショナルデータベース**：データを行と列で構成された関連するテーブルに整理します。各行はレコードを表し、各列はデータの属性を表します。
- **リレーショナルデータベースの利点**：スケーラブルで、医療、ビジネス、ゲームなどさまざまな分野で広く使われており、信頼性の高いデータを維持するために構造化されています。
- **一般的な利用例**：ウェブ開発、在庫管理システム、電子商取引、医療、企業向けアプリケーションなど。

## 重要な概念

- **スキーマ**：リレーショナルデータベースは、テーブル、列、データ型、制約、関係性などの構造を定義するスキーマが必要です。
- **主キー**：テーブル内の各行を一意に識別するキーです。データの整合性に不可欠で、外部キーを介してテーブル間のレコードを関連付けるために使われます。
- **外部キー**：別のテーブルの主キーを参照し、テーブル間の関連データをリンクするために使われます。
- **リレーションシップ**：主キーと外部キーを使ってテーブルを接続し、正規化されたデータ構造を作り、有意義なクエリを実行できます。
- **エンティティリレーションシップ図（ERD）**：データベーススキーマ内でエンティティ（テーブル）がどのように関連しているかを視覚化します。
- **データの整合性**：キーやデータ型を使って強制され、保存されたデータの一貫性と正確性を保証します。

## SQLの基本

- **クエリ**：データベースから特定のデータを取得するリクエストです。

```sql
SELECT * FROM dogs WHERE age < 3;
```

- **WHERE節**：条件に基づいて結果を絞り込みます。`<`、`=`、`>`などの比較演算子を使います。
- **ORDER BYを使った選択**：列に基づいて結果を取得し、並べ替えます。

```sql
SELECT columns FROM table_name ORDER BY column_name;
```

## テーブル操作

- **`CREATE TABLE` 文**：データベースに新しいテーブルを作成するための文です。

```sql
CREATE TABLE first_table();
```

- **`ALTER TABLE ADD COLUMN` 文**：既存のテーブルに列を追加するための文です。

```sql
ALTER TABLE table_name ADD COLUMN column_name DATATYPE;
```

- **`ALTER TABLE DROP COLUMN` 文**：既存のテーブルから列を削除するための文です。

```sql
ALTER TABLE table_name DROP COLUMN column_name;
```

- **`ALTER TABLE RENAME COLUMN` 文**：テーブル内の列の名前を変更するための文です。

```sql
ALTER TABLE table_name RENAME COLUMN column_name TO new_name;
```

- **`DROP TABLE` 文**：データベースからテーブル全体を削除するための文です。

```sql
DROP TABLE table_name;
```

- **`ALTER DATABASE RENAME` 文**：データベースの名前を変更するための文です。

```sql
ALTER DATABASE database_name RENAME TO new_database_name;
```

- **`DROP DATABASE` 文**：データベース全体を削除するための文です。

```sql
DROP DATABASE database_name;
```

## 制約とデータの整合性

- **`ALTER TABLE ADD COLUMN` と制約**：既存のテーブルに制約付きの列を追加するための文です。

```sql
ALTER TABLE table_name ADD COLUMN column_name DATATYPE CONSTRAINT;
```

- **`NOT NULL` 制約**：列にNULL値を許可しないことを保証する制約です。

```sql
column_name VARCHAR(50) NOT NULL
```

- **`ALTER TABLE ADD PRIMARY KEY` 文**：テーブルに主キー制約を追加するための文です。

```sql
ALTER TABLE table_name ADD PRIMARY KEY(column_name);
```

- **`ALTER TABLE DROP CONSTRAINT` 文**：テーブルから制約を削除するための文です。

```sql
ALTER TABLE table_name DROP CONSTRAINT constraint_name;
```

- **`ALTER TABLE ADD COLUMN` と外部キー**：別のテーブルを参照する外部キー列を追加するための文です。

```sql
ALTER TABLE table_name ADD COLUMN column_name DATATYPE REFERENCES referenced_table_name(referenced_column_name);
```

- **`ALTER TABLE ADD UNIQUE` 文**：列にUNIQUE制約を追加するための文です。

```sql
ALTER TABLE table_name ADD UNIQUE(column_name);
```

- **`ALTER TABLE ALTER COLUMN SET NOT NULL` 文**：既存の列にNOT NULL制約を設定するための文です。

```sql
ALTER TABLE table_name ALTER COLUMN column_name SET NOT NULL;
```

- **`INSERT` 文とNULL値**：テーブルにNULL値を挿入する方法を示す文です。

```sql
INSERT INTO table_name(column_a) VALUES(NULL);
-- or
INSERT INTO table_name(column_b) VALUES('value'); -- if column_a allows nulls
```

- **複合主キー**：複数の列で構成される主キーを定義する制約です。

```sql
CREATE TABLE course_enrollments (
    student_id INT,
    course_id INT,
    PRIMARY KEY (student_id, course_id)
);
```

## データ操作（CRUD）

- **`INSERT` 文**：テーブルに単一の行を挿入するための文です。

```sql
INSERT INTO table_name(column_1, column_2) VALUES(value1, value2);
```

- **`INSERT` 文（列名省略）**：列名を明示的に指定せず、テーブルのデフォルトの列順に従って値を挿入する方法を示す文です。

```sql
INSERT INTO dogs VALUES ('Gino', 3);
```

- **`INSERT` 文（複数行挿入）**：一度の操作で複数の行をテーブルに挿入するための文です。

```sql
INSERT INTO dogs (name, age) VALUES 
('Gino', 3),
('Nora', 2);
```

- **`UPDATE` 文**：条件に基づいて既存のデータを更新するための文です。

```sql
UPDATE table_name SET column_name=new_value WHERE condition;
```

- **`DELETE` 文**：条件に基づいてテーブルから行を削除するための文です。

```sql
DELETE FROM table_name WHERE condition;
```

## データ型

- **`NUMERIC` データ型**：指定した精度とスケールで正確な小数を保存するためのデータ型です。

```sql
price NUMERIC(10, 2)
```

- **`TEXT` データ型**：特定の長さ制限なしで可変長の文字列を保存するためのデータ型です。

```sql
column_name TEXT
```

- **`INTEGER` データ型**：小数点なしの整数を保存するためのデータ型です。

```sql
units_sold INTEGER
```

- **`SMALLINT` と `BIGINT` データ型**：それぞれより小さい範囲とより大きい範囲のINTEGERのバリエーションです。

- **`SERIAL` データ型**：PostgreSQLで自動増分する整数列を作成するためのデータ型です。

```sql
id SERIAL
```

- **`AUTO_INCREMENT` 属性**：MySQLで自動増分する整数列を作成するために使われる属性です。

```sql
id INT AUTO_INCREMENT
```

- **`VARCHAR` データ型**：最大長を指定して可変長の文字列を保存するためのデータ型です。

```sql
name VARCHAR(50)
```

- **`DATE` データ型**：日付（年、月、日）を保存するためのデータ型です。

```sql
event_date DATE
```

- **`TIME` データ型**：時刻（時、分、秒）を保存するためのデータ型です。

```sql
start_time TIME
```

- **`TIMESTAMP` データ型**：日時を保存し、オプションでタイムゾーン情報も含められるデータ型です。

```sql
event_timestamp TIMESTAMP
event_timestamp TIMESTAMP WITH TIME ZONE
```

- **`BOOLEAN` データ型**：真偽値（true/false）を保存するためのデータ型です。

```sql
is_active BOOLEAN
```

## データベースのリレーションシップ

- **リレーションシップの種類**：リレーショナルデータベースでテーブルが関連付けられるさまざまな方法です。
  - 1対1
  - 1対多
  - 多対1
  - 多対多
  - 自己参照（再帰）

- **1対1リレーションシップ**：一方のテーブルの各レコードが、もう一方のテーブルのちょうど1つのレコードに対応する関係です。

```md
One employee is assigned exactly one vehicle.
Tables: employees, vehicles
```

- **1対多リレーションシップ**：一方のテーブルの1つのレコードが、もう一方のテーブルの複数のレコードに関連付けられる関係です。

```md
One customer can have many orders.
Tables: customers → orders
```

- **結合テーブルを使った多対多リレーションシップ**：両方の関連テーブルの外部キーを含む結合テーブルを使って実装されます。

```sql
CREATE TABLE books_authors (
    author_id INT REFERENCES authors(id),
    book_id INT REFERENCES books(id)
);
```

- **自己参照リレーションシップ**：テーブルが自分自身を参照し、階層構造を作る関係です。

```md
An employee table where each employee may report to another employee.
```

## 高度なSQL（結合）

- **`INNER JOIN` 文**：両方のテーブルに一致する値がある行のみを返す結合です。

```sql
SELECT *
FROM products
INNER JOIN sales ON products.product_id = sales.product_id;
```

- **`FULL OUTER JOIN` 文**：両方のテーブルのすべての行を返し、どちらかのテーブルに一致しない行も含みます。

```sql
SELECT *
FROM products
FULL OUTER JOIN sales ON products.product_id = sales.product_id;
```

- **`LEFT OUTER JOIN` 文**：左側のテーブルのすべての行と、右側のテーブルの一致する行を返します。

```sql
SELECT *
FROM products
LEFT JOIN sales ON products.product_id = sales.product_id;
```

- **`RIGHT OUTER JOIN` 文**：右側のテーブルのすべての行と、左側のテーブルの一致する行を返します。

```sql
SELECT *
FROM products
RIGHT JOIN sales ON products.product_id = sales.product_id;
```

- **`SELF JOIN` 文**：同じテーブル内の行を比較するためにテーブルを自己結合するための結合です。

```sql
SELECT A.column_name, B.column_name
FROM table_name A
JOIN table_name B ON A.related_column = B.related_column;
```

- **`CROSS JOIN` 文**：2つのテーブルの直積を返し、最初のテーブルのすべての行と2番目のテーブルのすべての行を組み合わせます。

```sql
SELECT *
FROM table1
CROSS JOIN table2;
```

## PostgreSQL固有のコマンド

- **`psql` ログインコマンド**：特定のユーザー名とデータベースでPostgreSQLにログインするためのコマンドです。

```bash
psql --username=freecodecamp --dbname=postgres
```

- **`\l` コマンド**：PostgreSQLインスタンス内のすべてのデータベースを一覧表示するコマンドです。

```sql
\l
```

- **`CREATE DATABASE` と `\c` コマンド**：新しいデータベースを作成し、それに接続するためのコマンドです。

```sql
CREATE DATABASE database_name;
\c database_name
```

- **`\d` コマンド**：現在のデータベース内のすべてのテーブルを一覧表示するコマンドです。

```sql
\d
```

- **`\d table_name` コマンド**：特定のテーブルのスキーマ／構造を表示するコマンドです。

```sql
\d table_name
```

- **`\q` コマンド**：PostgreSQLクライアントを終了するコマンドです。

```bash
\q
```

## リレーショナルと非リレーショナルの違い

- **非リレーショナル（NoSQL）データベース**：非構造化または半構造化データを保存します。厳密なスキーマを必要とせず、進化するデータモデルに柔軟に対応できます。
- **リレーショナルと非リレーショナルの選択**：データの性質やアプリケーションの要件によって決まります。
- **リレーショナル vs 非リレーショナル**：構造化データと一貫性が必要ならリレーショナルを、柔軟性と変化の速いデータにはNoSQLを選びます。

## 人気のあるRDBMSシステム

- **MySQL**：オープンソースで信頼性が高く、ウェブ開発で広く使われており、大きなコミュニティに支えられています。
- **PostgreSQL**：オープンソースで高度かつ拡張性があります。カスタムデータ型やサーバーサイドプログラミングをサポートします。
- **SQLite**：軽量でファイルベース、サーバーレス。小規模なアプリケーションに最適です。
- **Microsoft SQL Server**：商用のエンタープライズ向けデータベースです。
- **Oracle Database**：大規模なパフォーマンスとスケーラビリティで知られる商用RDBMSです。
