# --description--

Тепер, коли ви розумієте систему модулів CommonJS, давайте розглянемо систему ES-модулів.

ES-модулі, або ECMAScript-модулі, — це офіційний, стандартизований формат модулів для JavaScript, визначений у специфікації ECMAScript. Вони використовують інструкції `import` та `export`, а в Node.js їх завантажують асинхронно, що є більш ефективним.

Node.js розглядає файли як ES-модулі у таких випадках:

* Файли з розширенням `.mjs`.
    
* А також файли з розширенням `.js`, коли найближчий батьківський файл `package.json` містить на верхньому рівні поле `"type"` зі значенням `"module"`.
    

Тут у нас є та сама функція `multiply`, яку ви вже бачили, але тепер ми будемо використовувати систему ESM. Для цього ми призначаємо розширення `.mjs` обом модулям.

Щоб експортувати функцію `multiply` з модуля `math.mjs`, використовуйте ключове слово `export`, за яким слідує ім’я функції, змінної або об’єкта, який ви хочете експортувати, у фігурних дужках:

```javascript
function multiply(a, b) {
  return a * b;
}

export { multiply };
```

Ви також можете експортувати функцію безпосередньо, використовуючи ключове слово `export` перед ключовим словом `function`. Це називається іменованим експортом. У модулі може бути кілька іменованих експортів:

```javascript
export function multiply(a, b) {
  return a * b;
}
```

Потім, щоб імпортувати цей модуль, використовуйте ключове слово `import` з фігурними дужками. Зверніть увагу, що ми використовуємо `import` замість `require()`. Це одна з ключових відмінностей між CommonJS та ES-модулями.

У фігурних дужках ми пишемо ім’я функції, змінної або об’єкта, який хочемо імпортувати. У цьому випадку це `multiply`. Потім використовуйте ключове слово `from` і відносний шлях до модуля. Цей код успішно запуститься, і вихідні дані будуть `42`.

```javascript
import { multiply } from './math.mjs';

console.log(multiply(6, 7)); // 42
```

У системі ES можна призначити один експорт за замовчуванням на модуль. Таким чином, для імпорту не потрібно використовувати фігурні дужки.

Ось приклад, де ми призначаємо функцію `multiply` як експорт за замовчуванням для всього модуля `math.mjs`:

```javascript
export default function multiply(a, b) {
  return a * b;
}
```

Тепер нам не потрібні фігурні дужки, оскільки функція буде імпортована і призначена за замовчуванням:

```javascript
import multiply from './math.mjs';
```

Якщо ви не використовуєте розширення файлів або файл `package.json`, щоб явно вказати Node.js, яку систему модулів ви хочете використовувати, він перевірить вихідний код на наявність синтаксису ES-модулів. Якщо цей синтаксис знайдеться, код запуститься як ES-модуль; інакше — як модуль CommonJS.

Хоча CommonJS був стандартом за замовчуванням для Node.js на початку, оскільки розробники дедалі частіше використовують ESM через його ефективність і сумісність із сучасним JavaScript, важливо знати їхні відмінності та як користуватися обома.
