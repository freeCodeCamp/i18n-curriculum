# --interactive--

`DOMContentLoaded` イベントは、HTMLドキュメント内のすべてが読み込まれ解析されたときに発火します。外部のスタイルシートや画像があっても、`DOMContentLoaded` イベントはそれらの読み込みを待ちません。HTMLの読み込みだけを待ちます。

これは、外部のスタイルシートや画像などすべての読み込みを待つ `load` イベントとは異なります。

`DOMContentLoaded` イベントを使う例の構文は以下の通りです。

```js
document.addEventListener("DOMContentLoaded", function () {
  console.log("DOM is loaded.");
});
```

DOMが読み込まれると、関数が実行され、メッセージ `DOM is loaded.` がコンソールに記録されます。

次に、`DOMContentLoaded` イベントを使った別の例を見てみましょう。この例ではHTML内に画像があります。画像を更新するために、画像の `src` と `alt` 属性を変更する関数を作成できます。

:::interactive_editor

```html
<h1>Image Change on DOM Loaded</h1>
<img
  id="example-img"
  src="https://cdn.freecodecamp.org/curriculum/cat-photo-app/relaxing-cat.jpg"
  alt="Cat relaxing"
/>

<script src="index.js"></script>
```

```js
function changeImg() {
  const img = document.getElementById("example-img");
  img.src =
    "https://cdn.freecodecamp.org/curriculum/responsive-web-design-principles/FCCStickers-CamperBot200x200.jpg";
  img.alt = "CamperBot sticker";
  console.log("image was just changed");
}

changeImg();
```

:::

次に、DOMがまだ読み込み中かどうかを確認し、`DOMContentLoaded` イベントのイベントリスナーを追加します。もし `DOMContentLoaded` イベントがすでに発火していれば、`changeImg` 関数を直接呼び出せます。

```js
if (document.readyState === "loading") {
  document.addEventListener("DOMContentLoaded", changeImg);
} else {
  console.log("DOMContentLoaded has already fired");
  changeImg();
}
```

このコードを実行してページを更新すると、DOMが読み込まれた後に画像が新しいものに変わるのが見えます。かなり速く起こりますが、画像が変わる際にわずかなちらつきが見られます。

`DOMContentLoaded` イベントは、DOMが読み込まれた直後にJavaScriptコードを実行したいが、画像やスタイルシートなどの外部リソースの読み込みはまだ終わっていない場合に便利です。
