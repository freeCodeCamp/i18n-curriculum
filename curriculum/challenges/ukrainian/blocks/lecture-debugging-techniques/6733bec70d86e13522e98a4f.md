# --description--

Інструкція `throw` у JavaScript використовується для викидання користувацької виняткової ситуації. Виняток у програмуванні — це коли трапляється несподівана подія, яка порушує нормальний потік програми.

Як програмістам, важливо обробляти ці винятки, щоб ваші програми не аварійно завершували роботу при виникненні помилок. Ось базовий синтаксис інструкції `throw`:

```js
throw expression;
```

`expression` у цьому випадку — це об’єкт або значення, яке представляє виняток, який ви хочете згенерувати. Прикладами цього можуть бути вбудовані класи винятків, такі як клас `Error`, `TypeError` або `RangeError`. Ось приклад використання інструкції `throw` для генерації `TypeError`:

```js
function validateNumber(input) {
  if (typeof input !== "number") {
    throw new TypeError("Expected a number, but received " + typeof input);
  }
  return input * 2;
}
```

У цьому прикладі ми перевіряємо, чи тип `input` не є типом `number`. Якщо ні, то ми викидаємо `TypeError` з кастомним повідомленням. Інакше функція поверне результат множення `input` на `2`.

Якщо ви хочете викинути більш загальне повідомлення про помилку, то можете звернутися до конструктора `Error` ось так:

```js
function divide(numerator, denominator) {
  if (denominator === 0) {
    throw new Error("Cannot divide by zero");
  }
  return numerator / denominator;
}
```

Ось приклад функції, яка перевіряє, чи є знаменник `0`. Якщо так, то вона викидає кастомне повідомлення про помилку з текстом `Cannot divide by zero`.

У наступному уроці ми розглянемо, як викидати повідомлення про помилки в контексті блоку `try`/`catch`, який використовується для плавної обробки винятків у JavaScript.
