# --description--

لقد رأيت بالفعل نوعًا عامًا في درس سابق: `Array<string>`. لكنك عادةً ما تستخدم الأنواع العامة في الدوال. في الواقع، يمكن اعتبارها معلمة خاصة تقدمها إلى الدالة للتحكم في سلوك تعريف نوع الدالة.

لنعد إلى مثالنا السابق لدالة تحصل على قيمة عشوائية من مصفوفة:

```js
const getRandomValue = (array: string[]): string => {
  return array[Math.floor(Math.random() * array.length)];
}
```

دالتنا تقبل مصفوفة من السلاسل النصية وتُرجع سلسلة نصية. لكن ماذا لو أردنا تمرير مصفوفة من الأرقام وإرجاع رقم؟

يمكنك استخدام نوع اتحاد:

```js
const getRandomValue = (array: (string|number)[]): (string|number) => {
  return array[Math.floor(Math.random() * array.length)];
}
const val = getRandomValue([1, 2, 4])
```

لكن مع هذا الأسلوب، حتى لو مررنا مصفوفة تحتوي فقط على أرقام، يحدد TypeScript أن المتغير `val` قد يكون رقمًا أو سلسلة نصية.

بدلاً من ذلك، يمكنك تعريف نوع عام للدالة:

```js
const getRandomValue = <T>(array: T[]): T => {
  return array[Math.floor(Math.random() * array.length)];
}
const val = getRandomValue([1, 2, 4])
```

هناك بعض المكونات المهمة في مثالنا الجديد. أولًا، صيغة `<T>` تخبر TypeScript بأنك تعرف نوعًا عامًا اسمه `T` للدالة. `T` اسم شائع للأنواع العامة، لكن يمكنك تسميته بأي اسم تريده (ضمن اصطلاحات متغيرات JavaScript).

ثم نخبر TypeScript أن معلمة المصفوفة هي مصفوفة من القيم التي تطابق النوع العام، وأن القيمة المرجعة هي عنصر واحد من نفس النوع. هذا يسمح لـ TypeScript بالكشف الصحيح أن `val` سيكون رقمًا، لأن المصفوفة التي مررناها تتكون فقط من أرقام.

لكن ماذا عن الدوال التي لا تتحكم بها؟ لنلق نظرة على طريقة `document.querySelector()`:

```js
const email = document.querySelector("#email");
console.log(email.value);
```

بما أننا نستعلم باستخدام محدد معرف (ID)، لا يعرف TypeScript نوع العنصر الذي نستعلم عنه. يعرف أنه `Element` من نوع ما، لكن محاولتنا للوصول إلى خاصية `value` ستُسبب خطأ لأن واجهة العنصر العليا لا تحتوي على تلك الخاصية.

لحسن الحظ، يمكننا تمرير معلمة نوع إلى استدعاء الدالة:

```js
const email = document.querySelector<HTMLInputElement>("#email");
console.log(email.value);
```

لاحظ كيف نستخدم صيغة الأقواس الزاوية مرة أخرى (`<HTMLInputElement>`)، لكن هذه المرة في استدعاء دالة. هذا يخبر TypeScript أن العنصر الذي نتوقع العثور عليه سيكون عنصر `input`، ولم نعد نحصل على خطأ في خاصية value لأن عناصر الإدخال تحتوي على تلك الخاصية. بالطبع، يمكن أن تُرجع `querySelector` قيمة `null` إذا لم يُعثر على عنصر، وسيُصدر TypeScript خطأ في المترجم هنا لأننا لا نتعامل مع هذه الحالة.

لكنك ستتعلم عن ذلك في الدرس التالي.
