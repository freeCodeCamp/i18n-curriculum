**目標：** 構建一個功能類似於此的應用程序：<a href="https://markdown-previewer.freecodecamp.rocks/" target="_blank" rel="noopener noreferrer nofollow">https://markdown-previewer.freecodecamp.rocks/</a>。

實現以下的用戶故事並且通過所有的測試用例。 使用你需要的任何庫和第三方API。 使用你自己的個人風格樣式。

你可以使用 HTML、JavaScript、CSS、Bootstrap、SASS、React、Redux 和 jQuery 的任意組合來完成此項目。 你應該使用一個前端框架（比如像是React）因爲這個部分是有關於學習前端框架的。 不推薦使用以上沒有列出的其他技術，不然風險自擔。 我們正着手支持其他前端框架像是Angular和Vue，但是現在還不支持他們。 我們將會接受並且努力修復所有使用推薦的技術棧在這個項目中出現的問題。 編程愉快！

**用戶故事 #1：** 我可以看到 `textarea` 元素及其對應的 `id="editor"`。

**用戶故事 #2：** 我可以看到一個具有對應 `id="preview"` 的元素。

**用戶故事 #3：** 當我在 `#editor` 元素中輸入文本時，`#preview` 元素會隨着我的輸入而更新，以顯示文本區域的內容。

**用戶故事 #4：** 當我在 `#editor` 元素中輸入 GitHub 風格的 markdown 時，文本會在我輸入時在 `#preview` 元素中呈現爲 HTML（提示：你不需要自己解析 Markdown - 你可以爲此導入 Marked 庫：<https://cdnjs.com/libraries/marked>）。

**用戶故事 #5：** 當我的 markdown 預覽器首次加載時，`#editor` 字段中的默認文本應包含有效的 markdown，該 markdown 至少代表以下每個元素之一：標題元素（H1 大小）、子標題元素（H2 大小）、鏈接、內聯代碼、代碼塊、列表項、塊引用、圖像和粗體文本。

**用戶故事 #6：** 當我的 markdown 預覽器首次加載時，`#editor` 字段中的默認 markdown 應該在 `#preview` 元素中呈現爲 HTML。

**可選獎勵（你不需要通過此測試）：** 我的 markdown 預覽器解釋回車符並將其呈現爲 `br`（換行符）元素。
