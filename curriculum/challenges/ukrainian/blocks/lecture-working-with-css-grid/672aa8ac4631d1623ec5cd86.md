# --interactive--

CSS Grid — це потужна система макетів, яка дозволяє веброзробникам легко створювати складні та адаптивні макети вебсторінок.

Уявіть, що ви розставляєте меблі в кімнаті — CSS Grid схожий на невидиму сітку на підлозі, яка допомагає точно розмістити все, де ви хочете.

Коли ми створюємо вебсайти, часто потрібно розташувати різні елементи на сторінці.

До появи CSS Grid це іноді було складно, особливо для складних макетів. CSS Grid спрощує цей процес, розбиваючи вашу вебсторінку на рядки та стовпці, створюючи структуру у вигляді сітки.

Уявімо, що ви працюєте з контейнером `div`, у якому вкладено кілька елементів, як ось так:

:::interactive_editor

```html
<div class="container">
  <div class="item">Item 1</div>
  <div class="item">Item 2</div>
  <div class="item">Item 3</div>
  <div class="item">Item 4</div>
  <div class="item">Item 5</div>
  <div class="item">Item 6</div>
</div>
```

:::

Якщо ви хочете стилізувати ці елементи у форматі сітки, ви можете встановити `display` у `grid` і задати стовпці так:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<div class="container">
  <div class="item">Item 1</div>
  <div class="item">Item 2</div>
  <div class="item">Item 3</div>
  <div class="item">Item 4</div>
  <div class="item">Item 5</div>
  <div class="item">Item 6</div>
</div>
```

```css
.container {
  display: grid;
  grid-template-columns: 1fr 1fr 1fr;
  grid-gap: 20px;
}

.item {
  background-color: lightgray;
  padding: 20px;
  text-align: center;
  border: 1px solid #ccc;
}
```

:::

У цьому коді ми кажемо браузеру створити сітку з трьома стовпцями однакової ширини, саме це означає `1fr 1fr 1fr`, і додаємо горизонтальний розрив у 20 пікселів між кожним елементом сітки.

Тепер ви можете запитати: «А як щодо гнучкого контейнера? Хіба його не використовують для макетів?»

Ви праві! Гнучкий контейнер — це ще одна модель макетів CSS, і вона теж дуже корисна. Але є кілька ключових відмінностей.

Гнучкий контейнер — одновимірний, а Grid — двовимірний. Це означає, що гнучкий контейнер чудово підходить для розташування елементів у одному рядку або стовпці, тоді як Grid відмінно справляється зі створенням макетів із рядками та стовпцями.

Гнучкий контейнер орієнтований на вміст, тобто він налаштовує макет залежно від вмісту. Grid, навпаки, орієнтований на макет: ви спочатку створюєте макет, а потім розміщуєте в ньому елементи. Grid дає вам більш точний контроль над розташуванням. Ви можете вказати елементу, який саме рядок і стовпець він має займати.

Ось приклад гнучкого контейнера для порівняння:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<div class="container">
  <div class="item">Item 1</div>
  <div class="item">Item 2</div>
  <div class="item">Item 3</div>
  <div class="item">Item 4</div>
  <div class="item">Item 5</div>
  <div class="item">Item 6</div>
</div>
```

```css
.container {
  display: flex;
  justify-content: space-between;
}
```

:::

Це створює гнучкий контейнер, де елементи рівномірно розподілені вздовж головної осі.

І Grid, і гнучкий контейнер мають свої переваги, і часто найкращі макети використовують комбінацію обох. Ви можете застосувати Grid для загального макета сторінки, а потім гнучкий контейнер — для вирівнювання елементів у кожній області сітки.

Підсумовуючи, CSS Grid — це потужний інструмент, який дозволяє створювати точні двовимірні макети. Хоча спочатку він може здатися складним, з практикою він стає незамінним для створення адаптивних і складних вебмакетів.
