# --description--

Як ви дізналися в попередніх уроках, cookie — це прості дані, які вебсайти можуть зберігати на пристрої користувача.

Однак насправді ви можете зберігати у cookie більш складні структури даних. Один із загальних методів — використовувати JSON для зберігання об’єктів або масивів. Ось приклад:

```js
const userData = {
  name: "John Doe",
  age: 30,
  role: "admin"
};

document.cookie = "userInfo=" + JSON.stringify(userData) + "; path=/";
```

У цьому прикладі ми створюємо об’єкт із даними користувача, перетворюємо його на рядок JSON, а потім зберігаємо у cookie. Коли ми хочемо отримати ці дані, ми можемо проаналізувати рядок JSON у об’єкт за допомогою `JSON.parse()`.

Тепер, можливо, ви цікавитеся аспектом «HTTP-заголовків» у нашій темі. Зазвичай cookie встановлюються сервером за допомогою HTTP-заголовків.

Наприклад, сервер може надіслати такий заголовок:

```http
Set-Cookie: username=John Doe; expires=Thu, 31 Dec 2024 6:00:00 IST; path=/
```

Цей заголовок повідомляє браузеру встановити cookie.

Ми також можемо встановлювати cookie безпосередньо в браузері за допомогою JavaScript. Це корисно для зберігання даних, які не потрібно негайно надсилати на сервер.

Зверніть увагу, що cookie мають обмеження розміру приблизно 4 КБ, і зберігання занадто великої кількості даних у них може уповільнити ваш вебзастосунок.

Зберігання великих обсягів даних у cookie може збільшити мережевий трафік, оскільки cookie надсилаються з кожним HTTP-запитом.
