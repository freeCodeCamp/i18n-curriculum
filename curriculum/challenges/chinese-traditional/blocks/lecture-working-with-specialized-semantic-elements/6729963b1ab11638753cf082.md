# --interactive--

未明確標示的註解元素，簡稱為 `u` 元素，用於表述已套用非文字註解的行內文字。

以下是使用 `u` 元素來凸顯標示各種拼字錯誤的範例：

:::interactive_editor

```html
<p>
  You can use the unarticulated annotation element to highlight
  <u>inccccort</u> <u>spling</u> <u>issses</u>.
</p>
```

:::

在範例中，字 `incorrect`、`spelling` 和 `issues` 拼寫錯誤。`u` 元素的預設樣式是在文字下方加上黑色底線。

在 HTML4 中，`u` 元素用於樣式設計。但在 HTML5 中，`u` 元素應該只用來顯示文字已套用非文字註解。

如果你想為一段文字添加底線，應該使用 CSS 而不是 HTML。

刪除線元素，簡稱為 `s` 元素，應用於表述文字不再準確或相關的情況。以下是使用 `s` 元素來顯示活動取消的範例：

:::interactive_editor

```html
<p><s>Tomorrow's hike will be meeting at noon.</s></p>

<p>Due to unforeseen weather conditions, the hike has been canceled.</p>
```

:::

在這個範例中，第一句話被刪除線劃掉，因為健行因天氣原因被取消。

`s` 元素絕不應僅用於顯示對文件的變更。在這種情況下，更合適的元素是刪除文字元素和插入文字元素。

`ruby` 元素表現顯示在主要文字上方或下方的小文字。它通常用於顯示東亞字元的發音。以下是來自 MDN 網頁文件頁面的 `ruby` 元素範例：

:::interactive_editor

```html
<ruby> 明日 <rp>(</rp><rt>Ashita</rt><rp>)</rp> </ruby>
```

:::

`rp` 元素，或稱 ruby 備用括號元素，用於在瀏覽器不支援顯示 ruby 註解時作為備用。

`rt` 元素，或稱 ruby 文字元素，用於顯示 ruby 註解的文字。這些文字通常用於東亞排版中的發音或翻譯細節。

雖然 `ruby` 元素可以用於其他型別的註解，但最常見的使用情境是東亞排版。
