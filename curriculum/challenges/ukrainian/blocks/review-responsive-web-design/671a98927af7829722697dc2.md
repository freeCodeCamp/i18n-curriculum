# --interactive--

## Адаптивний вебдизайн

- **Визначення**: Основний принцип адаптивного дизайну — це здатність пристосовуватися, тобто можливість вебсайту змінювати свій макет і вміст залежно від розміру екрана та можливостей пристрою, на якому його переглядають.
- **Рідкі сітки**: Вони використовують відносні одиниці, такі як відсотки, замість фіксованих одиниць, наприклад пікселів, що дозволяє вмісту змінювати розмір і перетікати залежно від розміру екрана.
- **Гнучкі зображення**: Вони налаштовані так, щоб змінювати розмір у межах своїх контейнерів, забезпечуючи, що вони не виходять за межі контейнерів на менших екранах.

## Медіа-запити

- **Визначення**: Це дозволяє розробникам застосовувати різні стилі залежно від характеристик пристрою, головним чином ширини вікна перегляду.

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<p class="mq-example">The background of this paragraph will change when the viewport is at least 768px wide.</p>
```

```css
.mq-example {
  padding: 10px;
  border: 1px solid #ccc;
  background-color: lightyellow;
}

@media screen and (min-width: 768px) {
  /* Styles for screens at least 768px wide */
  .mq-example {
    background-color: lightblue;
  }
}
```

:::

- **`all` тип медіа**: Підходить для всіх пристроїв. Це значення за замовчуванням, якщо тип медіа не вказано.
- **`print` типи медіа**: Призначений для друкованих матеріалів і документів, переглянутих у режимі попереднього перегляду друку.
- **`screen` типи медіа**: Призначений переважно для екранів.
- **`aspect-ratio`**: Описує співвідношення між шириною і висотою вікна перегляду.

```css
@media screen and (aspect-ratio: 16/9) {
  /* Styles for screens with a 16:9 aspect ratio */
}
```

- **`orientation`**: Використовується для вказівки, чи пристрій у ландшафтній чи портретній орієнтації.

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<p class="orientation-example">This paragraph's background will change in landscape orientation.</p>
```

```css
.orientation-example {
  padding: 10px;
  border: 1px solid #ccc;
  background-color: lightyellow;
}

@media screen and (orientation: landscape) {
  /* Styles for landscape orientation */
  .orientation-example {
    background-color: lightgreen;
  }
}
```

:::

- **`resolution`**: Використовується для опису роздільності вихідного пристрою в точках на дюйм (dpi) або точках на сантиметр (dpcm).

```css
@media screen and (min-resolution: 300dpi) {
  /* Styles for high-resolution screens */
}
```

- **`hover`**: Використовується для перевірки, чи основний механізм введення може наводити курсор на елементи.

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<button>Hover over me</button>
```

```css
button {
  padding: 10px 20px;
  font-size: 1rem;
  background-color: lightgray;
  border: 1px solid #999;
  cursor: pointer;
}

@media (hover: hover) {
  /* Styles for devices that support hover */
  button:hover {
    background-color: yellow;
  }
}
```

:::

- **`prefers-color-scheme`**: Використовується для визначення, чи користувач обрав світлу чи темну кольорову тему.
- **Медіа-запити та логічні оператори**: Оператор `and` використовується для поєднання кількох медіа-функцій, тоді як `not` і `only` можна застосовувати для заперечення або ізоляції медіа-запитів.

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<p class="logical-example">This paragraph's background will change when the screen is at least 768px wide AND in landscape orientation.</p>
```

```css
.logical-example {
  padding: 10px;
  border: 1px solid #ccc;
  background-color: lightyellow;
}

@media screen and (min-width: 768px) and (orientation: landscape) {
  /* Styles for landscape screens at least 768px wide */
  .logical-example {
    background-color: lightcoral;
  }
}
```

:::

## Загальні точки зупинки медіа

- **Визначення**: Точки зупинки медіа — це конкретні моменти в дизайні вебсайту, коли макет і вміст змінюються, щоб пристосуватися до різних розмірів екрана. Існують загальні точки зупинки, які можна використовувати для націлювання на телефони, планшети та десктопні екрани. Але не варто намагатися охопити кожен можливий розмір екрана для різних пристроїв.

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<p class="breakpoint-example">This text will change size when the viewport width is at least 768px.</p>
```

```css
.breakpoint-example {
  font-size: 1rem;
  padding: 10px;
  border: 1px solid #ccc;
}

/* Styles for screens wider than 768px */
@media screen and (min-width: 768px) {
  .breakpoint-example {
    font-size: 1.125rem;
    background-color: lightblue;
  }
}
```

:::

- **Малі пристрої (смартфони)**: до 640px
- **Середні пристрої (планшети)**: від 641px до 1024px
- **Великі пристрої (десктопи)**: від 1025px і більше

## Підхід mobile first

- **Визначення**: Підхід mobile-first — це філософія дизайну та стратегія розробки в адаптивному вебдизайні, яка ставить пріоритет на створення вебсайтів для мобільних пристроїв перед розробкою для більших екранів.

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<div class="container">
  <p>The width of this container changes based on the viewport size.</p>
</div>
```

```css
/* Base styles for mobile */
.container {
  width: 100%;
  padding: 10px;
  background-color: lightblue;
}

/* Styles for larger screens */
@media screen and (min-width: 768px) {
  .container {
    width: 750px;
    margin: 0 auto;
    padding: 20px;
  }
}

@media screen and (min-width: 1024px) {
  .container {
    width: 960px;
  }
}
```
