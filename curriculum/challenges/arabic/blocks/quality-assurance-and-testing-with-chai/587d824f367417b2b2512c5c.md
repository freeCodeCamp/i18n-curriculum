# --description--

كتذكير، يُبنى هذا المشروع على مشروع قالب جاهز تم استنساخه من <a href="https://github.com/freeCodeCamp/boilerplate-mochachai/" target="_blank" rel="noopener noreferrer nofollow">GitHub</a>.

في التحديات التالية، ستحاكي التفاعل البشري مع صفحة باستخدام متصفح بدون واجهة.

المتصفحات بدون واجهة هي متصفحات ويب بدون واجهة مستخدم رسومية. يمكنها عرض وتفسير HTML وCSS وJavaScript بنفس طريقة المتصفح العادي، مما يجعلها مفيدة جدًا لاختبار صفحات الويب.

في التحديات التالية ستستخدم Zombie.js، وهو متصفح بدون واجهة خفيف الوزن لا يعتمد على تثبيت ملفات ثنائية إضافية. لكن هناك خيارات أخرى كثيرة لمتصفحات بدون واجهة أكثر قوة.

يسمح لك Mocha بتشغيل شفرة قبل بدء أي من الاختبارات الفعلية. هذا مفيد لإضافة مدخلات إلى قاعدة بيانات ستُستخدم في بقية الاختبارات.

مع المتصفح بدون واجهة، قبل تشغيل الاختبارات، يجب عليك **زيارة** الصفحة التي ستختبرها.

يُنفذ الخطاف `suiteSetup` مرة واحدة فقط في بداية مجموعة الاختبارات.

هناك عدة أنواع أخرى من الخطافات يمكنها تنفيذ شفرة قبل كل اختبار، أو بعد كل اختبار، أو في نهاية مجموعة الاختبارات. راجع وثائق Mocha لمزيد من المعلومات.

# --instructions--

داخل `tests/2_functional-tests.js`، مباشرة بعد إعلان `Browser`، أضف عنوان URL الخاص بمشروعك إلى خاصية `site` للمتغير:

```js
Browser.site = 'http://0.0.0.0:3000'; // Your URL here
```

ثم على المستوى الجذري لمجموعة `'Functional Tests with Zombie.js'`، أنشئ نسخة جديدة من كائن `Browser` باستخدام الشفرة التالية:

```js
const browser = new Browser();
```

واستخدم الخطاف `suiteSetup` لتوجيه `browser` إلى المسار `/` باستخدام الشفرة التالية. **ملاحظة**: يتم تمرير `done` كاستدعاء راجع إلى `browser.visit`، فلا يجب عليك استدعاؤه.

```js
suiteSetup(function(done) {
  return browser.visit('/', done);
});
```
