# --description--

Nello sviluppo software, una libreria è come una cassetta degli attrezzi per gli sviluppatori.

Invece di dover implementare ogni singola parte del codice da zero, una libreria ti offre codice già scritto e riutilizzabile, come funzioni, classi e strutture dati che puoi usare nei tuoi progetti.

Python ha una vasta libreria standard con molti moduli integrati diversi. Sono tutte soluzioni standardizzate e ben testate per molti dei problemi e delle attività che affronterai quotidianamente come programmatore, come:

* Interagire con il sistema operativo.
    
* Lavorare con i file.
    
* Networking.
    
* Gestire date e orari.
    
* Eseguire operazioni matematiche.
    
* Usare espressioni regolari.
    
* Testare ed eseguire il debug del tuo codice.
    
* E molto altro!
    

Alcuni esempi di moduli integrati popolari sono `math`, `random`, `re` (abbreviazione di "espressioni regolari") e `datetime`.

Il modulo `math` ha funzioni utili per eseguire operazioni matematiche più complesse.

Il modulo `random` è utile per generare numeri casuali.

Il modulo `re` serve per lavorare con le espressioni regolari.

E il modulo `datetime` è utile per lavorare con date e orari in Python.

Ma come puoi consultare le variabili, le costanti, le funzioni e le classi definite in questi moduli integrati?

Usi un'istruzione import. Queste istruzioni ti permettono di importare moduli nel tuo script Python. Le istruzioni import sono generalmente scritte all'inizio del file. Inoltre, puoi personalizzarle in base alle tue esigenze. Prima usi l'istruzione `import`, seguita dal nome del modulo:

```python
import module_name
```

Supponiamo che tu voglia importare il modulo `math`. In tal caso, scriveresti questo all'inizio del tuo file:

```python
import math
```

Poi, se hai bisogno di chiamare una funzione da quel modulo nel tuo script Python, useresti la notazione a punto, con il nome del modulo seguito dal nome della funzione:

```python
module_name.function_name() 
```

Per esempio, per ottenere la radice quadrata di 36, scriveresti `math` seguito da un punto e poi `sqrt`, un'abbreviazione di radice quadrata, e tra parentesi tonde passeresti gli argomenti necessari. In questo caso, dobbiamo passare solo il numero di cui vogliamo la radice quadrata:

```python
math.sqrt(36)
```

Questa è la versione più semplice di un'istruzione import, ma ci sono altre alternative.

Se hai bisogno di importare il modulo con un nome diverso (noto anche come "alias"), puoi usare questa sintassi, con `as` seguito dall'alias alla fine dell'istruzione import:

```python
import module_name as module_alias
```

Questo si usa spesso per abbreviare nomi di moduli lunghi o per evitare conflitti di nomi.

Per esempio, per riferirti al modulo `math` come `m` nel tuo codice, puoi assegnargli un alias così:

```python
import math as m 
```

Poi, puoi consultare gli elementi del modulo usando l'alias:

```python
m.sqrt(36)
```

Ma a volte non hai bisogno di importare tutto da un modulo. Forse ti servono solo una o due funzioni o classi specifiche. Python ha esattamente quello che ti serve in quel caso.

Ora l'istruzione import inizia con `from`, seguito dal nome del modulo, e poi la parola chiave `import` seguita dal nome degli elementi che vuoi importare:

```python
from module_name import name1, name2
```

Poi, puoi usare questi nomi senza il prefisso del modulo nel tuo script Python.

Se vuoi assegnare alias a questi nomi, puoi farlo usando la parola chiave `as` dopo ciascuno, seguita dall'alias che vuoi usare:

```python
from module_name import name1 as alias1, name2 as alias2
```

Supponiamo che tu voglia importare solo le funzioni radians, sine e cosine dal modulo `math`. Scriveresti:

```python
from math import radians, sin, cos
```

Ora puoi chiamare queste funzioni direttamente nel tuo codice, senza il modulo `math` come prefisso.

Ecco un esempio più dettagliato:

Per trovare il seno e il coseno di un angolo specifico espresso inizialmente in gradi, possiamo chiamare la funzione `radians` per convertirlo in radianti, e poi chiamare le funzioni seno e coseno, passando l'angolo in radianti:

```python
from math import radians, sin, cos

angle_degrees = 40
angle_radians = radians(angle_degrees)

sine_value = sin(angle_radians)
cos_value = cos(angle_radians)

print(sine_value) # 0.6427876096865393
print(cos_value)  # 0.766044443118978
```

Nota come stiamo chiamando le funzioni direttamente, senza il nome del modulo come prefisso. Questo perché abbiamo importato le funzioni con questa sintassi alternativa.

Questo è utile, ma può causare conflitti di nomi se hai già funzioni o variabili con lo stesso nome definite nello script Python stesso. Quindi è qualcosa da tenere a mente quando scegli quale tipo di istruzione import usare.

E infine, troviamo questa istruzione import che termina con un asterisco. L'asterisco dice a Python che vuoi importare tutto quel modulo, ma vuoi farlo in modo da non dover usare il nome del modulo come prefisso:

```python
from module_name import *
```

Per esempio, se fai questo importando il modulo `math`, potrai chiamare qualsiasi funzione definita in quel modulo senza specificare il nome del modulo come prefisso. Ecco alcuni esempi:

```python
from math import *
print(sqrt(36))  # 6.0
print(pow(5, 2)) # 25.0
print(exp(1))    # 2.718281828459045
```

Tuttavia, questo è generalmente sconsigliato perché può portare a collisioni di namespace e rendere più difficile sapere da dove provengono certi nomi.

Le istruzioni import funzionano esattamente allo stesso modo per funzioni, classi, costanti, variabili e qualsiasi altro elemento definito nel modulo.

Ecco un esempio di una costante dal modulo `math`, il numero `pi`:

```python
import math
print(math.pi)
```

Ecco un esempio di una classe dal modulo `datetime`. Creiamo un oggetto data che rappresenta il 15 luglio 1959. Poi assegniamo quell'oggetto `date` a una variabile e consultiamo giorno, mese e anno singolarmente usando la notazione a punto:

```python
import datetime
birthday = datetime.date(1959, 7, 15)
print(birthday.day)    # 15
print(birthday.month)  # 7
print(birthday.year)   # 1959
```

Puoi trovare più informazioni sul contenuto del modulo nella documentazione ufficiale di Python per quel modulo.

Ottimo. Ora che sai di più sui moduli, dovresti conoscere anche questo idiomatico molto importante negli script Python, perché sono strettamente correlati:

```python
if __name__ == '__main__': 
    # Code
```

`__name__` è una variabile integrata speciale in Python.

Quando un file Python viene eseguito direttamente, Python imposta il valore di questa variabile sulla stringa `"__main__"`.

Ma se il file Python viene importato come modulo in un altro script Python, il valore della variabile `__name__` viene impostato sul nome di quel modulo (di solito il nome del file senza l'estensione `.py`).

Ecco perché spesso trovi questa condizione negli script Python. Contiene il codice che vuoi eseguire **solo se** lo script Python viene eseguito come programma principale:

```python
if __name__ == '__main__': 
    # Code
```

Ma se lo script viene importato come modulo, il codice all'interno di quel blocco non viene eseguito.

Questo è utile perché permette agli script Python di avere due scopi. Possono essere eseguiti direttamente per eseguire la loro logica principale, oppure possono essere importati in un altro modulo senza eseguire la loro logica principale.
