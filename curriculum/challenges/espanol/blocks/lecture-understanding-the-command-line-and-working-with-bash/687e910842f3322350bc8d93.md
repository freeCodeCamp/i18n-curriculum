# --description--

Aprendamos sobre el terminal y cómo se diferencia de la línea de comandos.

La línea de comandos es una interfaz básica de entrada de texto que permite a un usuario ingresar "commands", usualmente en forma de una serie de caracteres, y enviarlos o ejecutarlos, generalmente presionando la tecla "Enter". Lo más común es que veas interfaces de línea de comandos dentro de un terminal.

Un terminal es una aplicación especial que ofrece una interfaz de línea de comandos para ejecutar comandos a nivel del sistema más allá de las operaciones básicas de lectura/escritura. También puedes escuchar sobre "emuladores de terminal", que son esencialmente aplicaciones que envuelven una interfaz básica de terminal para ofrecer funciones o características adicionales. Pero para la mayoría de los propósitos generales, probablemente estés seguro de referirte a estos también como "terminales".

Finalmente, también necesitaremos hablar sobre "shells". Un shell es el software que envuelve la línea de comandos e interpreta tus entradas como comandos, devolviendo la salida.

¿Pero cómo puedes acceder al terminal en tu sistema?

Bueno, si estás usando una versión moderna de Windows, probablemente tendrás dos opciones diferentes. Windows ofrece dos aplicaciones de shell distintas desde el principio: Powershell y Command Prompt.

Ambas shells vienen con sus propias aplicaciones de terminal (que comparten el nombre de la shell respectiva), así que puedes usar el menú de aplicaciones para iniciar un terminal de Powershell o un terminal de Command Prompt.

Windows también ofrece un emulador de terminal relativamente nuevo y modernizado llamado Microsoft Terminal.

Puedes instalar esta aplicación desde la Windows Store, y luego podrás iniciarla igual que iniciarías tus terminales Powershell o Command Prompt. La ventaja de Microsoft Terminal es que te permite acceder tanto a las shells de Powershell como de Command Prompt desde la misma aplicación, así como a tu shell de Linux respectivo si has configurado Windows Subsystems for Linux.

MacOS ofrece una terminal que se llama justamente "Terminal". Puedes acceder a esta a través de la búsqueda Spotlight o mediante el lanzador de aplicaciones.

MacOS también ofrece soporte para emuladores de terminal de terceros, como iTerm o Ghostty. Puedes instalar estas aplicaciones desde sus páginas principales.

En Linux, la terminal predeterminada (y cómo la inicias) depende mucho de la distro y el entorno de escritorio que uses. También hay mucho soporte para emuladores de terminal de terceros. Por ejemplo, podrías configurar kitty en Arch Linux, y puedes iniciarlo con un gestor de aplicaciones como wofi.

Linux probablemente tiene la mayor variedad de aplicaciones de terminal para elegir. La mayoría de estas se pueden instalar a través del gestor de paquetes de tu distro.

La distinción entre "terminal", "shell" y "línea de comandos" puede parecer bastante pedante a veces. Y en la mayoría de los casos, los términos generalmente pueden usarse de manera intercambiable. Pero es importante saber que tienen significados específicos, y cuáles son cada uno de esos significados.
