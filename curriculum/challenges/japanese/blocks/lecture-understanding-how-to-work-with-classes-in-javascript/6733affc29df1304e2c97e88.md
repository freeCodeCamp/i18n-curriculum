# --interactive--

現代のJavaScriptでは、クラスは同じ種類の複数のオブジェクトを作成するための設計図のようなものとしてコード内で定義できます。

たとえば、在庫管理システムをモデル化する場合、製品の種類ごとに複数のオブジェクトを作成する必要があります。

各個別の製品はそれぞれ独自のプロパティや動作を持ちますが、同じ種類の製品は同じ「設計図」から作成できます。これがクラスの使い道です。

クラスはクラス式またはクラス宣言として定義できます。

クラス宣言の基本的な構文は次の通りです。

```js
class MyClassName {
  // Class Methods
  constructor() { ... }
  method1() { ... }
  method2() { ... }
  ...
}
```

まず`class`キーワードを書き、その後にクラス名を続けます。クラス名を付ける際は`PascalCase`の命名慣例を使い、`camelCase`は使わないようにしてください。

波括弧の中身はクラス本体と呼ばれます。ここにコンストラクタやメソッドが入ります。

コンストラクタは、新しいオブジェクトがクラスから作成されるときに自動的に呼び出される特別なメソッドです。通常、オブジェクトのプロパティを初期化するために使われます。

この例では、`Dog`クラスにコンストラクタがあるのがわかります。

```js
class Dog {
  constructor(name) {
    this.name = name;
  }
}
```

ここで`this`キーワードは、`Dog`クラスの現在のインスタンスを指すために使われており、作成されるオブジェクトに`name`プロパティを代入しています。`this`キーワードについては今後のレッスンで詳しく学びます。

コンストラクタの下にはメソッドと呼ばれるものを置けます。メソッドはクラス内で定義された関数で、動作や処理を実行します。クラスのインスタンスが実行できる振る舞いを定義するために使われます。

ここでは`bark`クラスに`Dog`というメソッドを追加する例です。

```js
class Dog {
  constructor(name) {
    this.name = name;
  }

  bark() {
    console.log(`${this.name} says woof!`);
  }
}
```

`bark`メソッドが呼び出されると、渡された実際の名前とともにメッセージがコンソールに出力されます。もし`Dog`という名前で`"Buddy"`インスタンスを作成し、そのインスタンスで`bark()`メソッドを呼び出すと、コンソールに`"Buddy says woof!"`が表示されます。

クラスを定義したら、`new`キーワード、クラス名、引数を使ってクラスのインスタンスを作成できます。

この例では、`Dog`という名前で新しい`Gino`インスタンスを作成し、それを変数`dog`に代入しています。

```js
const dog = new Dog("Gino");
```

コード内で`new Dog("Gino")`と書くと、新しいオブジェクトが作成され、指定された引数でコンストラクタが自動的に呼び出されます。

その後、ドット記法を使ってこのプロパティにアクセスできます。次のコード行でそれが確認できます。

```js
console.log(dog.name);
```

こちらが完全な例です。

:::interactive_editor

```js
class Dog {
  constructor(name) {
    this.name = name;
  }

  bark() {
    console.log(`${this.name} says woof!`);
  }
}

const dog = new Dog("Gino");
console.log(dog.name);
```

:::

メソッドも呼び出せます。この例では`bark()`インスタンスの`dog`メソッドを呼び出しています。

```js
dog.bark();
```

メソッド呼び出しを含む完全な例はこちらです。

:::interactive_editor

```js
class Dog {
  constructor(name) {
    this.name = name;
  } 
  bark() {
    console.log(`${this.name} says woof!`);
  }
}

const dog = new Dog("Gino");
console.log(dog.name);
dog.bark();
```

:::

前述のように、クラスはクラス式としても定義できます。これはクラスが匿名で変数に代入される形です。

先ほどの例をクラス式で書くと次のようになります。

```js
const Dog = class {
  constructor(name) {
    this.name = name;
  }

  bark() {
    console.log(`${this.name} says woof!`);
  }
};
```

ほとんどの場合はクラス宣言を使いますが、クラス式も理解しておくと、特に動的なプログラミングの場面で柔軟に対応できるため役立ちます。

クラスを使い、その基本的な構文を理解することで、再利用可能なコードを作成し、論理的な単位に整理し、コードのモジュール化を促進できます。
