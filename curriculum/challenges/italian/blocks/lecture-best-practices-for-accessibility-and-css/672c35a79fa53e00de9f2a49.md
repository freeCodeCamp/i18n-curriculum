# --interactive--

Nascondere contenuti su una pagina web è una pratica comune nello sviluppo web, ma è fondamentale farlo in modo che non comprometta l'accessibilità. Diverse tecniche di nascondimento possono avere impatti diversi su come le tecnologie assistive interpretano e presentano il contenuto agli utenti. Esploriamo alcune buone pratiche per nascondere contenuti mantenendo l'accessibilità. Un metodo comune per nascondere contenuti è usare `display: none`. Ecco un esempio:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<p class="hidden">Hidden text</p>
<p>Visible text</p>
```

```css
.hidden {
  display: none;
}
```

:::

Anche se questo nasconde efficacemente il contenuto visivamente, lo rimuove anche dall'albero di accessibilità. L'albero di accessibilità è una struttura usata dalle tecnologie assistive, come i lettori di schermo, per interpretare e interagire con il contenuto di una pagina web. Rappresenta il contenuto e il suo significato semantico in un modo che la tecnologia assistiva può comprendere e presentare all'utente.

Usare `display: none` significa che i lettori di schermo e altre tecnologie assistive non potranno consultare questo contenuto, poiché non è incluso nell'albero di accessibilità. Perciò, è importante usare questo metodo solo quando vuoi rimuovere completamente il contenuto sia dalla presentazione visiva sia dall'accessibilità.

Un altro modo per nascondere contenuti è usare `visibility: hidden`:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<p class="hidden">Hidden text</p>
<p>Visible text</p>
```

```css
.hidden {
  visibility: hidden;
}
```

:::

`visibility: hidden` nasconde il contenuto visivamente ma lo mantiene nel flusso del documento, cioè occupa ancora spazio nella pagina. Come `display: none`, anche `visibility: hidden` rimuove il contenuto dall'albero di accessibilità. Questo significa che le tecnologie assistive, come i lettori di schermo, non potranno consultare il contenuto nascosto. Usa `visibility: hidden` solo quando vuoi nascondere il contenuto a tutti, comprese le persone che usano tecnologie assistive.

Per contenuti che devono essere nascosti visivamente ma rimanere accessibili ai lettori di schermo, puoi usare una tecnica spesso chiamata "visivamente nascosto" o "solo per lettori di schermo". Ecco un esempio che usa la classe CSS `.sr-only`, una tecnica comune per nascondere visivamente il contenuto mantenendolo accessibile ai lettori di schermo:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<p class="sr-only">Hidden text</p>
<p>Visible text</p>
```

```css
.sr-only {
  position: absolute;
  width: 1px;
  height: 1px;
  padding: 0;
  margin: -1px;
  overflow: hidden;
  clip: rect(0, 0, 0, 0);
  white-space: nowrap;
  border: 0;
}
```

:::

In questo esempio, usiamo proprietà come `position`, `clip` e `white-space`, che imparerai nelle lezioni future. Per ora, sappi solo che questa regola CSS nasconde efficacemente il contenuto visivamente mantenendolo accessibile ai lettori di schermo. È utile per fornire contesto aggiuntivo agli utenti di lettori di schermo senza influire sul layout visivo.

Per alternare la visibilità del contenuto, considera di usare l'attributo `hidden`:

:::interactive_editor

```html
<p hidden>This content is hidden</p>
<p>This content is visible</p>
```

:::

L'attributo `hidden` è supportato dalla maggior parte dei browser moderni e nasconde il contenuto sia visivamente sia dall'albero di accessibilità. Può essere facilmente alternato con JavaScript. Infine, fai attenzione a nascondere contenuti importanti. Se un'informazione è cruciale per comprendere o usare il sito, dovrebbe essere visibile e accessibile a tutti gli utenti. Nascondi contenuti solo quando questo migliora davvero l'esperienza utente. Seguendo queste buone pratiche, puoi assicurarti che il tuo contenuto rimanga accessibile a tutti, indipendentemente da come interagiscono con il tuo sito.
