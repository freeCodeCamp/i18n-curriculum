# --description--

セキュリティに関しては、最初の対策と予防はソフトウェアを展開する企業やチームにあります。なぜなら、セキュリティ問題は偶然に起こることはほとんどないからです。これらの企業がプロセスを急ぎ、基本的なソフトウェア設計の原則を無視したり、スタッフの適切な訓練を怠ったときにセキュリティ問題が発生します。

セキュリティ問題の大きな引き金は、**結果を急いで出すプレッシャーの急増**です。企業が新製品の発売を急ぐと、開発チームはしばしばセキュリティを後回しにします。厳しい締め切りを守るために、徹底したコードレビューを省略し、深いセキュリティテストを急いで通過します。

このスピードは、コーディングの見落としやアクセス制御の欠如、修正されていないバグが多い不安定なソフトウェアを生み出し、ハッカーが簡単に悪用できる状態になります。

もう一つよくある状況は、**複雑なシステム統合**です。現代のアプリケーションは外部ライブラリ、サードパーティのAPI、マイクロサービスに大きく依存しています。

開発者がこれらの異なる技術を組み合わせると、巨大なサプライチェーンのリスクが生じます。

オープンソースツールに脆弱性があったり、外部ベンダーがデータ漏洩を起こした場合、そのセキュリティの失敗は自動的にメインのアプリケーションに広がります。

**不適切な設定管理**も大きなセキュリティ問題を引き起こします。この状況は、技術チームが標準の工場出荷時設定でサーバーやクラウドデータベースを展開するときに起こります。

デフォルト設定にはしばしば空の管理者パスワード、開いたネットワークポート、公開されたデータアクセス権限が含まれています。攻撃者は自動化されたインターネットスキャナーを使ってこれらの未設定システムを探し出し、即座にアクセスを得ます。

**日常業務での人的ミス**もまた高リスクな状況です。従業員はフィッシングメール、詐欺的なテキストメッセージ、偽の電話などのソーシャルエンジニアリング攻撃に常にさらされています。

適切なセキュリティ意識の訓練を受けていないと、従業員は個人のログイン情報を明かしたり、悪意のあるダウンロードリンクをクリックしたり、会社のデータを詐欺アカウントに送信してしまうことがあります。

最後に、**組織の合併や企業再編の際にもセキュリティ問題が発生します**。異なる2つの企業がデジタルネットワークを統合するとき、ITスタッフは全く異なるソフトウェア環境を迅速に接続しなければなりません。

この移行はしばしば盲点を生み、セキュリティチームが古いサーバーの可視性を失ったり、ネットワークファイアウォールを誤設定したり、必要のないユーザーに過剰なアクセス権限を与えてしまうことがあります。
