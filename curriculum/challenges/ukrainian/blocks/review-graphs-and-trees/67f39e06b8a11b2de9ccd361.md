## Огляд графів

Граф — це множина вузлів (вершин), з’єднаних ребрами (зв’язками). Кожен вузол може бути з’єднаний з кількома іншими вузлами, утворюючи мережу. Різновиди графів включають:

- Орієнтований: ребра мають напрямок (від одного вузла до іншого), часто зображуються прямими лініями зі стрілками.
- Неорієнтований: ребра не мають напрямку, зображуються простими лініями.
- Вершинний: кожен вузол пов’язаний з міткою або ідентифікатором.
- Циклічний: містить цикли (шлях, що починається і закінчується в одному й тому самому вузлі).
- Ациклічний (DAG): не містить циклів.
- З мітками на ребрах: кожне ребро має мітку, зазвичай намальовану поруч із відповідним ребром.
- Зважений: ребра мають ваги (значення), які можна використовувати для арифметичних дій.
- Роз’єднаний: містить два або більше вузлів, які не з’єднані жодними ребрами.

Графи використовують у різних застосунках, таких як карти, мережі, системи рекомендацій, розв’язання залежностей.

## Обхід графів

Це процес відвідування всіх вузлів у графі. Два основні алгоритми:

- **Пошук у ширину (BFS)**
  - Використовує чергу.
  - Досліджує рівень за рівнем.
  - Знаходить найкоротший шлях у незважених графах.

- **Пошук у глибину (DFS)**
  - Використовує стек (або рекурсію).
  - Повністю досліджує гілку перед поверненням назад.
  - Корисний для виявлення циклів і пошуку шляхів.

## Представлення графів

Графи можна представити двома основними способами:

- **Список суміжності**
  - Кожен вузол має список своїх сусідів.
  - Економить простір для розріджених графів.
  - Легко ітерувати над сусідами.

- **Матриця суміжності**
  - Двовимірний масив, де рядки і стовпці представляють вузли.
  - Вимагає багато пам’яті для великих графів.
  - Швидко перевіряє наявність ребра між двома вузлами.

## Дерева

Дерево — це особливий тип графа, який є ациклічним і зв’язним. Основні властивості:

- Вони не мають циклів або петель (шляхів, де початок і кінець — один і той самий вузол).
- Вони мають бути зв’язними (кожен вузол можна досягти з будь-якого іншого).

### Загальні типи дерев

Найпоширеніші типи дерев:

- Двійкові дерева
  - Кожен вузол має не більше двох дочірніх елементів: лівий і правий.

- Двійкові дерева пошуку (BST)
  - Двійкове дерево, у якому кожен лівий дочірній елемент менший за батьківський, а кожен правий — більший за батьківський.

## Префіксні дерева (tries)

Також відомі як префіксні дерева, використовуються для зберігання множин рядків, де кожен вузол представляє символ.

Спільні префікси зберігаються лише один раз, що робить їх ефективними для таких завдань, як автозаповнення та перевірка орфографії.

Операції пошуку та вставки мають часову складність O(L), де L — довжина рядка.

## Пріоритетні черги

Пріоритетна черга — це абстрактний тип даних, у якому кожен елемент має пріоритет.

Черги та стеки враховують лише порядок вставки, тоді як пріоритетні черги враховують пріоритет елементів.

Стандартні черги працюють за принципом FIFO (перший прийшов — перший пішов), а стеки — за LIFO (останній прийшов — перший пішов). Проте в пріоритетній черзі елементи з вищим пріоритетом обслуговуються раніше за елементи з нижчим пріоритетом, незалежно від порядку вставки.

## Купи

Це спеціалізована дерево-подібна структура даних із дуже специфічною властивістю, що називається властивістю купи.

Властивість купи визначає відношення між батьківськими та дочірніми вузлами. Існують два типи куп:

- Макс-купа
  - Значення кожного батьківського вузла більше або дорівнює значенням його дочірніх вузлів.
  - Найбільший елемент знаходиться в корені.

- Мін-купа
  - Значення кожного батьківського вузла менше або дорівнює значенням його дочірніх вузлів.
  - Найменший елемент знаходиться в корені.

### Приклад модуля Python `heapq`

```py
import heapq

# Create empty heap
my_heap = []

# Insert elements
heapq.heappush(my_heap, 9)
heapq.heappush(my_heap, 3)
heapq.heappush(my_heap, 5)
print(my_heap) # [3, 9, 5]

# Remove smallest element
print(heapq.heappop(my_heap))  # 3
print(my_heap) # [5, 9]

# Push + Pop in one step
print(heapq.heappushpop(my_heap, 7)) # 5
print(my_heap) # [7, 9]

# Transform list into heap
nums = [5, 7, 3, 1]
heapq.heapify(nums)
```

### Використання пріоритетів

```py
my_heap = []
heapq.heappush(my_heap, (3, "A"))
heapq.heappush(my_heap, (2, "B"))
heapq.heappush(my_heap, (1, "C"))

# Removes lowest number = highest priority
print(heapq.heappop(my_heap))  # (1, "C")
```
