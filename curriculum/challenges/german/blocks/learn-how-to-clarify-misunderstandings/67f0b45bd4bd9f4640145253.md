`diagnostics`

### --feedback--

Dieses Wort im Plural bezeichnet Tests oder Werkzeuge, die verwendet werden, um herauszufinden, was mit einem System oder Programm nicht stimmt.

# --explanation--

`Dig into the code` bedeutet, den Code genau zu betrachten, um die Ursache eines Problems zu finden. Zum Beispiel:

`We'll need to dig into the code to find the bug.` – Das bedeutet, den Code sorgfältig zu lesen und auf Fehler zu überprüfen.

`Run diagnostics` bedeutet, Werkzeuge oder Befehle zu verwenden, um ein System zu testen und Probleme zu finden. Zum Beispiel:

`Let's run diagnostics to see what's causing the crash.` – Das bedeutet, Software zu verwenden, um das System auf Fehler zu überprüfen.
