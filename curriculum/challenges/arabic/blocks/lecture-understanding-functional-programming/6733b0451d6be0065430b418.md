# --interactive--

البرمجة الوظيفية تتعلق بكتابة شفرة نظيفة، متوقعة، وأسهل في الاختبار ومعالجة الأخطاء. سنتناول اليوم مفهومين رئيسيين هما الدوال النقية وتجنب التأثيرات الجانبية.

لنبدأ بالدوال النقية.

الدالة النقية هي دالة تعطي نفس الناتج دائمًا لنفس المُدخَل، ولا تُغير أي شيء خارج نفسها.

إليك مثالًا على دالة نقية:

```js
function add(a, b) {
  return a + b;
}
```

هذه الدالة نقية لأنها تعيد نفس النتيجة دائمًا لنفس المُدخَلات، ولا تُغير أي شيء خارج نفسها. مهما استدعيت `add(2, 3)`، ستُعيد دائمًا `5`، ولن تؤثر على أي جزء آخر من برنامجك.

الآن، لنقارن ذلك بدالة غير نقية:

```js
let total = 0;
function addToTotal(value) {
  total += value;
  return total;
}
```

هذه الدالة غير نقية لأنها تُغير متغيرًا خارج نفسها (`total`). نتيجة `addToTotal(5)` ستختلف حسب قيمة `total` قبل استدعاء الدالة. هذا عدم التوقع يجعل شفرتنا أصعب في الفهم ومعالجة الأخطاء.

هذا يقودنا إلى المفهوم المهم التالي وهو التأثيرات الجانبية.

التأثير الجانبي هو أي تغيير يحدث في حالة البرنامج عند استدعاء دالة. قد يشمل ذلك تعديل متغير عام، كتابة إلى ملف، أو إجراء استدعاء API.

الدوال النقية لا تحتوي على تأثيرات جانبية، أما الدوال غير النقية فتحتوي عليها.

تجنب التأثيرات الجانبية هو مبدأ أساسي في البرمجة الوظيفية. عندما نقلل التأثيرات الجانبية، تصبح الشفرة أكثر توقعًا وأسهل في الاختبار.

أثناء كتابة الشفرة، يجب أن يكون هدفنا الرئيسي أن تجعل الدوال تقوم بشيء واحد فقط وتؤديه بشكل جيد جدًا دون تغيير أجزاء أخرى من برنامجنا.

إليك مثالًا على دالة تحتوي على تأثير جانبي:

:::interactive_editor

```js
function greet(name) {
  console.log(`Hello, ${name}!`);
}

greet("Alice");
```

:::

هذه الدالة تحتوي على تأثير جانبي لأنها تتفاعل مع العالم الخارجي عن طريق تسجيل بيانات في وحدة التحكم.

في البرمجة الوظيفية، قد نعيد كتابة هذا كالتالي:

:::interactive_editor

```js
function greet(name) {
  return `Hello, ${name}!`;
}
console.log(greet("Alice"));
```

:::

الآن `greet` دالة نقية تعيد سلسلة نصية فقط، ونتعامل مع التأثير الجانبي (التسجيل في وحدة التحكم) بشكل منفصل.

تشجع البرمجة الوظيفية على تركيب برامجنا باستخدام دوال نقية عن طريق تقسيم الشفرة إلى أجزاء صغيرة ومتوقعة. وهكذا، يمكننا بناء برامج أسهل في الفهم والاختبار والصيانة.
