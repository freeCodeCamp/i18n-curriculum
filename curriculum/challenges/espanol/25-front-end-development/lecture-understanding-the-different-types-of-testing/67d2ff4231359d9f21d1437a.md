¿Qué es la prueba de extremo a extremo?

La prueba de extremo a extremo, o E2E para abreviar, prueba escenarios del mundo real desde la perspectiva del usuario. Los ejemplos de escenarios de prueba de E2E incluyen probar el proceso de registro y de inicio de sesión o de pago en un sitio de comercio electrónico.

Para comprender mejor cómo funciona la prueba de extremo a extremo, echemos un vistazo a un ejemplo del mundo real del código base de freeCodeCamp. Este ejemplo utiliza Playwright, un popular marco de pruebas de extremo a extremo desarrollado por Microsoft.

Dentro del código base de freeCodeCamp, hay un directorio llamado `e2e` que contiene pruebas para donaciones, certificaciones, exámenes y más. Dentro del directorio, hay un archivo llamado `donate-page-donor.spec.ts` que tiene pruebas para la página del donante autenticado.

El primer paso es navegar a la página de donaciones como un usuario de demostración que ya es colaborador:

```js
test.beforeEach(async ({ page }) => {
  execSync("node ./tools/scripts/seed/seed-demo-user --set-true isDonating");
  await page.goto("/donate");
});
```

Dado que hay múltiples pruebas en este archivo, el gancho `beforeEach` se ejecutará antes de cada una de las pruebas.

Una vez que el usuario esté en la página de donaciones, verá un mensaje de agradecimiento con sugerencias sobre cómo apoyar aún más a freeCodeCamp. Aquí hay solo algunas de las pruebas para verificar parte del texto que se encuentra en la página de donaciones:

```js
test("should render the donate page correctly", async ({ page }) => {
  await expect(
    page.getByText("Thank you for your continued support")
  ).toBeVisible();

  await expect(
    page.getByText(
      "Your contributions are crucial in creating resources that empower millions of people to learn new skills and support their families."
    )
  ).toBeVisible();

  ...
});
```

También hay pruebas para verificar que el donante tiene un enlace de colaborador en la barra de menú, así como un borde especial alrededor de su avatar para indicar que es colaborador:

```js
test("The menu should have a supporters link", async ({ page }) => {
  const menuButton = page.getByTestId("header-menu-button");
  const menu = page.getByTestId("header-menu");

  await expect(menuButton).toBeVisible();
  await menuButton.click();

  await expect(menu).toBeVisible();

  await expect(page.getByRole("link", { name: "Supporters" })).toBeVisible();
});

test("The Avatar should have a special border for donors", async ({ page }) => {
  const container = page.locator(".avatar-container");
  await expect(container).toHaveClass("avatar-container gold-border");
});
```

Al tener este tipo de pruebas detalladas, puedes aumentar la cobertura de pruebas para tu aplicación más allá de solo las pruebas unitarias. Las pruebas de extremo a extremo también ayudan a garantizar que tu aplicación se comporte correctamente y sea predecible para los usuarios. Los ejemplos de otras herramientas de prueba de extremo a extremo incluyen Cypress, Selenium y Puppeteer.

Si bien la prueba de extremo a extremo es importante, es costosa porque consume mucho tiempo configurarla, diseñarla y mantenerla. Las pruebas E2E son excelentes para flujos de usuario críticos, mientras que las pruebas unitarias son ideales para probar pequeñas unidades en la aplicación.
