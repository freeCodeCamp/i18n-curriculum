# --description--

前のレッスンでは、プロジェクトフォルダを作成し、Expressをインストールしました。

```bash
mkdir my-app
cd my-app
npm init -y
npm install express
```

おさらいすると、これらのコマンドは新しいプロジェクトフォルダを作成し、`cd`でプロジェクトフォルダに移動し、プロジェクトを初期化して基本的な`package.json`ファイルを作成し、Expressをインストールします。

次に、コードエディタを開いて`index.js`という名前のファイルを作成し、最初のExpressサーバーを書いてみましょう。

```javascript
const express = require("express");
const app = express();
const port = 3000;

app.get("/", (req, res) => {
  res.send("Hello World!");
});

app.listen(port, () => {
  console.log(`Example app listening on port ${port}`);
});
```

ターミナルで次のコマンドを使ってコードを実行できます。

```bash
node index.js
```

ブラウザで[http://localhost:3000](http://localhost:3000)を開くと、メッセージ**"Hello World!"**が表示されます。

これがExpressを始める速さです！では、コードをステップごとに見ていきましょう。

最初のコード行では、Expressライブラリを読み込み、`express`という変数に代入しています。

```javascript
const express = require("express");
```

`express`変数はファクトリー関数です。呼び出すと、新しいExpressアプリケーションインスタンスを作成して返します。

```javascript
const app = express();
```

`app`はExpressが提供する`.get()`、`.post()`、`.listen()`などのすべてのメソッドにアクセスできます。これがメインのサーバーオブジェクトです。

次の行では`port`変数を定義しています。ポートはアプリケーションがリッスンするネットワークのエンドポイントで、`3000`はローカル開発でよく使われるポートです。

```javascript
const port = 3000;
```

次の部分を見てみましょう。

```javascript
app.get("/", (req, res) => {
  res.send("Hello World!");
});
```

このコードブロックはルートを定義し、サーバーがリクエストにどう応答するかを指定しています。`app.get()`はHTTP GETリクエストのハンドラを登録します。GETリクエストはデータを取得するためによく使われるHTTPメソッドです。

`"/"`引数は`app.get()`内でルートパスを表します。この場合、アプリケーションのルートURLを指しています。

`(req, res) => { ... }`部分は、このルートにマッチするリクエストがあったときに実行されるコールバック関数です。`req`オブジェクトは受信したリクエストの情報を含み、`res`オブジェクトはクライアントにレスポンスを送るために使われます。

最後に`res.send("Hello World!")`がクライアントにレスポンスを送ります。この例ではレスポンスはプレーンテキストの`"Hello World!"`です。

最後のコード部分を見てみましょう。

```javascript
app.listen(port, () => {
  console.log(`Example app listening on port ${port}`);
});
```

`app.listen()`メソッドはExpressアプリケーションを起動し、Nodeに指定したポート、ここでは`3000`でHTTPリクエストの受信を待つよう指示します。

サーバーが起動するとコールバック関数が実行され、メッセージ`Example app listening on port 3000`がコンソールにログされ、Expressアプリが正常に起動しリクエストを受け付ける準備ができたことを確認できます。
