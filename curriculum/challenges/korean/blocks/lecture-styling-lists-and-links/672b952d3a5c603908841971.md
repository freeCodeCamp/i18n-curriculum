# --interactive--

기본 링크 스타일은 웹에서 사용성과 접근성을 향상하는 데 매우 중요한 역할을 합니다.

이 스타일은 일반적으로 방문하지 않은 링크는 파란색, 방문한 링크는 보라색으로 표시되며, 사용자가 웹사이트를 탐색할 때 기대하고 의존하는 표준이 되었습니다.

기본 링크 스타일의 주요 목적은 사용자가 웹페이지에서 상호작용 가능한 요소와 그렇지 않은 요소를 명확하게 구분할 수 있도록 시각적 신호를 제공하는 것입니다.

이 구분은 직관적이고 사용자 친화적인 탐색 경험을 만드는 데 필수적입니다.

기본 링크 스타일을 살펴보겠습니다:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<a href="/">Example link</a>
```

```css
a:link {
  color: blue;
  text-decoration: underline;
}

a:visited {
  color: purple;
}
```

:::

이 스타일은 여러 중요한 기능을 수행합니다.

첫째, 방문하지 않은 링크의 파란색은 대부분의 배경색과 텍스트에 대해 눈에 띄어 링크를 쉽게 식별할 수 있게 합니다. 이 대비는 사용자가 페이지를 빠르게 훑어보고 탐색 요소나 중요한 정보를 찾는 데 매우 중요합니다.

밑줄은 텍스트가 클릭 가능한 것임을 강조하는 추가 시각적 신호를 제공합니다. 이는 색맹인 사용자나 색 구분이 어려운 사용자에게 특히 도움이 됩니다.

방문한 링크의 색상 변화(일반적으로 보라색)는 사용자가 어디를 방문했는지 추적하는 데 도움을 줍니다. 이 기능은 대형 웹사이트를 탐색하거나 연구할 때 매우 유용하며, 사용자가 같은 페이지를 실수로 다시 방문하는 것을 방지합니다.

다음은 HTML 예시입니다:

:::interactive_editor

```html
<p>Learn more about <a href="https://www.example.com/cats">cats</a> and <a href="https://www.example.com/dogs">dogs</a>.</p>
```

:::

별도의 CSS 없이 대부분의 브라우저는 이 링크들을 파란색 밑줄로 렌더링합니다. 링크 중 하나를 클릭하면 색상이 보라색으로 바뀌어 사용자가 자신의 탐색 기록을 즉시 알 수 있습니다.

디자이너가 웹사이트의 미적 요소에 맞게 기본 스타일을 변경하는 경우가 많지만, 그 기본 원칙은 반드시 유지해야 합니다.

기본 스타일을 변경할 경우에도 링크가 일반 텍스트와 명확히 구분되고, 방문한 링크와 방문하지 않은 링크 사이에 눈에 띄는 차이가 있으며, 선택한 색상이 배경과 충분한 대비를 이루는지 확인하세요.

예를 들어, 다음과 같은 사용자 정의 스타일을 사용할 수 있습니다:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<a href="/">Example link</a>
```

```css
a:link {
  color: blue;
  text-decoration: none;
  border-bottom: 1px solid blue;
}

a:visited {
  color: purple;
  border-bottom: 1px solid purple;
}
```

:::

이 스타일은 파란색과 보라색 색상 체계를 유지하면서 밑줄 대신 더 현대적인 느낌의 아래쪽 테두리를 사용합니다.

링크의 다양한 상태도 고려하는 것이 중요합니다. 기본 상태와 방문 상태 외에도 링크는 일반적으로 호버 상태와 활성 상태를 가집니다:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<a href="/">Example link</a>
```

```css
a:hover {
  color: red;
}

a:active {
  color: darkorange;
}
```

:::

이 상태들은 사용자가 링크와 상호작용할 때 즉각적인 피드백을 제공하여 사이트의 전반적인 사용성을 향상합니다.

결론적으로, 링크 스타일을 사용자 정의할 수 있지만 기본 스타일의 원칙은 반드시 지켜야 합니다.

이 원칙들은 사용자가 콘텐츠를 효과적으로 탐색하고 상호작용할 수 있도록 하는 사용 가능하고 접근 가능한 웹 경험을 만드는 데 중요한 역할을 합니다.

웹사이트의 링크 스타일을 디자인할 때는 항상 명확성과 사용자 경험을 최우선으로 고려하세요.
