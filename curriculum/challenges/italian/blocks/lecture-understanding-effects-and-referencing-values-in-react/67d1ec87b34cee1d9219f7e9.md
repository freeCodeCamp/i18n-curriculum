# --description--

React offre molti hook integrati che ti permettono di implementare diverse funzionalità nei tuoi progetti. Questi includono `useState`, `useEffect`, `useContext` e altri.

Ma a volte, avrai bisogno di aggiungere una funzionalità che nessuno degli hook integrati può fornire. Fortunatamente, puoi creare i tuoi hook personalizzati in React.

Gli hook personalizzati non sono così complicati come potrebbero sembrare. Sono semplicemente funzioni riutilizzabili che ti permettono di condividere la logica tra più componenti. Questo significa che la riutilizzabilità è un altro motivo per cui potresti voler creare un tuo hook.

Con un hook personalizzato, puoi estrarre la logica da qualsiasi componente che li usa, come il recupero dati, la gestione dello stato, l’attivazione e disattivazione, effetti collaterali come il controllo dello stato online o offline degli utenti, e così via.

Puoi poi importare l’hook per usarlo in qualsiasi componente, così puoi concentrarti sul render e sulla presentazione all’interno di quei componenti. Questo significa meno ripetizioni e meno duplicazioni, quindi meno punti in cui dover fare modifiche quando vuoi aggiornare qualcosa.

Ora, vediamo come puoi creare il tuo hook personalizzato.

In React, tutti gli hook integrati iniziano con la parola `use`, quindi il tuo hook personalizzato dovrebbe seguire la stessa convenzione. Il nome del tuo hook personalizzato dovrebbe anche comunicare chiaramente cosa fa.

Quindi, se il tuo hook personalizzato…

- recupera dati, puoi chiamarlo `useFetch`

- attiva o disattiva qualcosa, puoi chiamarlo `useToggle`

- oppure se implementa il debouncing, `useDebounce` è un buon nome

Supponiamo che tu voglia creare un hook personalizzato per aggiungere il debouncing alla tua app.

Il debouncing è una tecnica di programmazione che limita la frequenza con cui una funzione viene eseguita. Funziona aspettando che un utente smetta di eseguire un’azione per un periodo di tempo specificato prima di eseguire la funzione. Per esempio, in un riquadro di ricerca, invece di fare una chiamata API per ogni battitura, il debouncing aspetta che l’utente smetta di digitare per, diciamo, 500 millisecondi.

Per creare un hook personalizzato per il debouncing, devi prima creare un file `useDebounce.jsx` o `useDebounce.js`. Convenzionalmente, i file per qualsiasi hook personalizzato che crei vengono salvati in una cartella `hooks`.

Puoi usare alcuni hook integrati all’interno del tuo hook personalizzato. Per il debouncing, ti servono gli hook `useState` e `useEffect`, quindi importali all’inizio del file:

```js
import { useState, useEffect } from "react";
```

Poi, crea una funzione `useDebounce` che prende `value` e `delay` come parametri. `value` è la risorsa per cui vuoi aspettare, e `delay` è il periodo di tempo per cui vuoi aspettare. Dal momento che vuoi aspettare un certo periodo di tempo, le funzioni `setTimeout` e `clearTimeout` ti saranno utili:

```js
function useDebounce(value, delay) {
  const [debouncedValue, setDebouncedValue] = useState(value);

  useEffect(() => {
    const handler = setTimeout(() => {
      setDebouncedValue(value);
    }, delay);

    return () => {
      clearTimeout(handler);
    };
  }, [value, delay]);

  return debouncedValue;
}

export { useDebounce };
```

Lo stato `debouncedValue` contiene e restituisce il valore ritardato, che si aggiorna solo dopo il periodo di timeout specificato.

`useEffect` è dove avviene la vera magia. Se ricordi dalla lezione precedente, tutto ciò che esiste al di fuori del ciclo di rendering di React, come impostare e cancellare un timer, è un effetto collaterale, e dovresti usare l’hook `useEffect` per gestirli.

All’interno dell’hook `useEffect` qui, usi `setTimeout` per impostare `debouncedValue`. Poi restituisci una funzione di pulizia che usa `clearTimeout` per cancellare il timeout precedente ogni volta che `value` o `delay` cambiano, o quando il componente viene smontato.

Ecco un componente `FootballerSearch` che usa l’hook `useDebounce` per ritardare la ricerca di 1 secondo dopo che l’utente smette di digitare:

```jsx
import { useState, useEffect } from "react";
import { useDebounce } from "./hooks/useDebounce";
import footballers from "./footballers";

const FootballerSearch = () => {
  const [query, setQuery] = useState("");
  const debouncedQuery = useDebounce(query, 1000); // Start searching 1 second after the user stops typing

  useEffect(() => {
    if (debouncedQuery) {
      const results = footballers.filter((footballer) =>
        footballer.toLowerCase().includes(debouncedQuery.toLowerCase()),
      );
      console.log("Search results:", results);
    } else {
      console.log("Search results: []");
    }
  }, [debouncedQuery]);

  return (
    <>
      <h1 style={{ textAlign: "center" }}>Footballer Search App</h1>
      <div style={{ textAlign: "center" }}>
        <input
          style={{ padding: "0.5rem", width: "30%" }}
          type="text"
          value={query}
          onChange={(e) => setQuery(e.target.value)}
          placeholder="Search for a footballer..."
        />
      </div>
    </>
  );
};

export default FootballerSearch;
```

Come puoi vedere, la variabile `debouncedQuery` è quella che inizializza l’hook `useDebounce` con lo stato della query (ciò che l’utente digita), e il ritardo di 1.000 millisecondi, cioè 1 secondo. La ricerca vera e propria è gestita all’interno dell’hook `useEffect`, e i risultati della ricerca vengono mostrati nella console.
