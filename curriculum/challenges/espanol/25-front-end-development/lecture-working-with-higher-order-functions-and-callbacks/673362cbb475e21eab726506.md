¿Qué es el encadenamiento de métodos y cómo funciona?

El encadenamiento de métodos es una técnica de programación que te permite llamar a múltiples métodos sobre el mismo objeto en una única línea de código. Esta técnica puede hacer que tu código sea más legible y conciso, especialmente al realizar una serie de operaciones sobre el mismo objeto.

Veamos un ejemplo usando diferentes métodos de strings:

```js
const result = "  Hello, World!  "
  .trim()
  .toLowerCase()
  .replace("world", "JavaScript");

console.log(result); // "hello, JavaScript!"
```

En este ejemplo, comenzamos con un string y realizamos tres operaciones en secuencia: recortamos espacios en blanco, convertimos en minúsculas y reemplazamos el string `world` con `JavaScript`. Cada método devuelve un nuevo string, que se convierte en el objetivo de la siguiente llamada al método.

El encadenamiento de métodos puede mejorar significativamente la legibilidad del código cuando se trabaja con operaciones complejas.

Por ejemplo, considera este ejemplo usando los métodos `filter`, `map` y `reduce`:

```js
const transactions = [
  { amount: 100, type: "credit" },
  { amount: 20, type: "cash" },
  { amount: 150, type: "credit" },
  { amount: 50, type: "cash" },
  { amount: 75, type: "credit" }
];

const totalCreditWithBonus = transactions
  .filter((transaction) => transaction.type === "credit")
  .map((transaction) => transaction.amount * 1.1)
  .reduce((sum, amount) => sum + amount, 0);

console.log(totalCreditWithBonus); // 357.5
```

En este ejemplo, tenemos un arreglo de transacciones donde cada objeto tiene un monto y un tipo de tarjeta de crédito o efectivo.

Primero filtramos a través de las transacciones y creamos un nuevo arreglo de solo transacciones con tarjeta de crédito. Luego, encadenamos el método `map` al resultado filtrado y para cada monto de transacción, lo multiplicamos por `1.1`, que representa un bono del `10%`.

Después, tomamos ese resultado y encadenamos el método `reduce` para sumar cada monto, lo que resulta en `357.5`.

Aunque el encadenamiento de métodos puede hacer el código más conciso y legible, es importante usarlo con prudencia.

Las cadenas muy largas pueden volverse difíciles de depurar, ya que no está claro de inmediato qué paso en la cadena podría estar causando un problema. A menudo es una buena práctica dividir cadenas muy largas en múltiples pasos para una mejor claridad y depuración más fácil.
