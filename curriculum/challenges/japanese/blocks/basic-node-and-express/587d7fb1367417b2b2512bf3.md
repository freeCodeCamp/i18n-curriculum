# --description--

以前、`express.static()` ミドルウェア関数を紹介しました。今回はミドルウェアが何かをより詳しく見ていきます。ミドルウェア関数は3つの引数を取る関数です：リクエストオブジェクト、レスポンスオブジェクト、そしてアプリケーションのリクエスト-レスポンスサイクル内の次の関数です。これらの関数はアプリに副作用をもたらすコードを実行し、通常はリクエストやレスポンスオブジェクトに情報を追加します。また、ある条件が満たされたときにレスポンスを送信してサイクルを終了させることもできます。レスポンスを送信しなければ、スタック内の次の関数の実行を開始します。これが3番目の引数、`next()` の呼び出しを引き起こします。

次の例を見てみましょう：

```js
function(req, res, next) {
  console.log("I'm a middleware...");
  next();
}
```

この関数をルートにマウントしたとします。リクエストがそのルートにマッチすると、「I’m a middleware…」という文字列を表示し、その後スタック内の次の関数を実行します。この演習ではルートレベルのミドルウェアを作成します。チャレンジ4で見たように、ルートレベルでミドルウェア関数をマウントするには `app.use(<mware-function>)` メソッドを使います。この場合、すべてのリクエストに対して関数が実行されますが、より具体的な条件を設定することもできます。例えば、POSTリクエストに対してのみ関数を実行したい場合は `app.post(<mware-function>)` を使うことができます。GET、DELETE、PUTなどすべてのHTTP動詞に対応する同様のメソッドがあります。

# --instructions--

シンプルなロガーを作成してください。すべてのリクエストに対して、次の形式の文字列をコンソールにログ出力する必要があります：`method path - ip`。例は次のようになります：`GET /json - ::ffff:127.0.0.1`。`method` と `path` の間にスペースがあり、`path` と `ip` を区切るダッシュの両側にもスペースがあることに注意してください。リクエストメソッド（HTTP動詞）、相対ルートパス、呼び出し元のIPはリクエストオブジェクトの `req.method`、`req.path`、`req.ip` から取得できます。処理が終わったら必ず `next()` を呼び出してください。そうしないとサーバーが永久に停止してしまいます。『Logs』を開いて、リクエストが来たときに何が起こるかを確認してください。

**注意：** Expressはコード内に現れる順に関数を評価します。ミドルウェアも同様です。すべてのルートで動作させたい場合は、それらの前にマウントしてください。
