# --description--

<a href="/learn/javascript-algorithms-and-data-structures/basic-javascript/replace-loops-using-recursion" target="_blank" rel="noopener noreferrer nofollow">前のチャレンジ</a>では、再帰を使って`for`ループを置き換える方法を学びました。今回は、`1`から関数に渡された数までの連続した整数の配列を返す、より複雑な関数を見ていきましょう。

前のチャレンジで述べたように、<dfn>基本ケース</dfn>があります。基本ケースは、再帰関数がこれ以上自分自身を呼び出す必要がないことを示します。これは戻り値がすでにわかっている単純なケースです。また、<dfn>再帰呼び出し</dfn>もあり、これは異なる引数で元の関数を実行します。関数が正しく書かれていれば、最終的に基本ケースに到達します。

例えば、`1`から`n`までの数を含む配列を返す再帰関数を書きたいとします。この関数は、最終の数を表す引数`n`を受け取る必要があります。そして、`n`の値を徐々に小さくしながら自分自身を呼び出し、`1`に到達するまで繰り返します。関数は次のように書けます。

```javascript
function countup(n) {
  if (n < 1) {
    return [];
  } else {
    const countArray = countup(n - 1);
    countArray.push(n);
    return countArray;
  }
}
console.log(countup(5));
```

値`[1, 2, 3, 4, 5]`がコンソールに表示されます。

最初は、`n`の値が*減少*しているのに、最終的な配列の値は*増加*しているので直感に反するように思えます。これは、pushが再帰呼び出しの後、最後に行われるためです。`n`が配列に追加される時点で、`countup(n - 1)`はすでに評価されて`[1, 2, ..., n - 1]`を返しています。

# --instructions--

`countdown`という名前の関数が1つのパラメータ（`n`）で定義されています。この関数は再帰を使って、`n`パラメータに基づき`1`から`n`までの整数を含む配列を返す必要があります。もし関数が1未満の数で呼び出された場合、空の配列を返してください。例えば、この関数を`n = 5`で呼び出すと、配列`[5, 4, 3, 2, 1]`を返すべきです。関数は必ず自分自身を呼び出す再帰を使い、ループは一切使わないでください。
