# --description--

Пошук у списку елементів — це загальна операція в інформатиці. Існує два ключові алгоритми, які варто знати для пошуку: лінійний пошук і бінарний пошук.

Лінійний пошук починається з початку списку і ітерує кожен елемент, доки не знайде цільове значення, яке шукає.

Якщо цільове значення знайдено, повертається індекс, де воно розташоване у списку. Якщо цільове значення не знайдено, повертається `-1`. Ми повертаємо `-1`, бо це невалідний індекс у більшості мов програмування.

Ось як виглядає код для лінійного пошуку:

```python
def linear_search(arr, target):
    for i in range(len(arr)):
        if arr[i] == target:
            return i
    return -1
```

Якщо список, у якому ми шукаємо, — це `[13, 4, 7, 9, 10]`, а цільове значення — `9`, функція поверне `3`, бо `9` знаходиться на індексі `3`.

Якщо змінити цільове значення на `5`, функція поверне `-1`, бо `5` відсутнє у списку.

Хоч це досить простий алгоритм, він не є найефективнішим. Якщо у вас великий список елементів, лінійний пошук може зайняти багато часу, щоб знайти цільове значення.

Часова складність лінійного пошуку — `O(n)`, бо час пошуку зростає лінійно зі збільшенням розміру списку.

Просторова складність лінійного пошуку — `O(1)`, бо він не потребує додаткового простору для пошуку у списку.

Бінарний пошук — це більш ефективний алгоритм для пошуку у великому списку елементів. Умова полягає в тому, що список має бути відсортований за зростанням.

Бінарний пошук працює, розділяючи список навпіл і перевіряючи, чи знаходиться цільове значення посередині списку. Якщо цільове значення посередині, повертається індекс цільового значення. Інакше алгоритм перевіряє, чи знаходиться цільове значення у лівій або правій половині списку.

Він продовжує ділити залишкові частини списку навпіл, доки не знайде цільове значення. Якщо цільове значення відсутнє у списку, повертається `-1`.

Ось як виглядає код для бінарного пошуку:

```python
def binary_search(arr, target):
    low = 0
    high = len(arr) - 1

    while low <= high:
        mid = (low + high) // 2  

        if arr[mid] == target:
            return mid
        elif arr[mid] < target:
            low = mid + 1
        else:
            high = mid - 1

    return -1
```

Починаємо з визначення індексів `low` і `high`. Це представляє діапазон списку, у якому ми шукаємо.

Потім перевіряємо умову, що `low` менше або дорівнює `high`. Якщо `low` більше за `high`, ми переглянули весь список і цільове значення не знайдено. У такому разі припиняємо пошук і повертаємо `-1`.

Якщо індекс `low` менший або дорівнює індексу `high`, обчислюємо середній індекс списку, `mid`. Потім перевіряємо, чи цільове значення знаходиться на середньому індексі. Якщо так, повертаємо середній індекс.

Інакше перевіряємо, чи значення на середньому індексі менше за цільове. Якщо так, оновлюємо нижній індекс, встановлюючи його рівним середньому індексу плюс один. Це означає, що шукатимемо у правій половині списку.

Нарешті, якщо жодна з інших умов не є `True`, оновлюємо індекс `high`, встановлюючи його рівним середньому індексу мінус один. Це означає, що шукатимемо у лівій половині списку.

Продовжуємо повторювати цей процес, доки не знайдемо ціль або не визначимо, що ціль відсутня у списку.

Часова складність бінарного пошуку — `O(log n)`, бо час пошуку зростає логарифмічно зі збільшенням розміру списку.

Просторова складність бінарного пошуку — `O(1)`, бо він не потребує додаткового простору для пошуку у списку.

Бінарний і лінійний пошук можна використовувати для різних завдань, з якими ви зіткнетесь в інформатиці. Важливо розуміти відмінності між цими двома алгоритмами і коли слід використовувати кожен із них.
