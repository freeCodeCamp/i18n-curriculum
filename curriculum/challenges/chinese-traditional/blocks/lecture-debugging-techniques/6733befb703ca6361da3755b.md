# --description--

偵錯你的 JavaScript 程式不僅僅是使用程式碼中的 `console.log()` 述語。還有更多進階技巧，一旦你習慣了，就能讓偵錯變得輕而易舉。讓我們一起來了解那些進階技巧，這樣當你遇到錯誤時，就會樂於偵錯你的 JavaScript 程式。

我們將首先探討的概念是使用中斷點。中斷點讓你可以在你選擇的特定程式碼行暫停執行。暫停後，你可以檢查變數、評估表達式，並檢視呼叫堆疊。

要在 Chrome 瀏覽器中為你的程式碼的任何列添加中斷點，請開啟開發者工具並切換到 Sources 分頁，打開你想要偵錯的 JavaScript 檔案，然後點擊列號以設定中斷點。

當執行到達中斷點並停止後，你可以使用右上角的圖示逐步執行程式碼。你也可以透過右鍵點擊某一行並選擇「Add conditional breakpoint…」來為你的中斷點添加條件。

現在，讓我們繼續了解監看表達式。監看表達式讓你在程式碼執行時監控變數或表達式的值，即使它們不在目前的作用域內。

要新增監看表達式，請前往開發者工具的 Sources 分頁，尋找右側的監看面板，然後點擊加號（`+`）圖示以新增。

接下來我們將探討的概念是分析。分析幫助你透過擷取螢幕截圖和記錄 CPU 使用率、函式呼叫及執行時間，來識別效率瓶頸。

為了對你的程式碼執行該操作，請打開 Performance 標籤，點擊 record，然後執行你想要評測的動作。當你完成該動作的執行後，停止記錄以分析結果。

如果你的應用程式在某些操作行為中出現延遲，這可能會很有用。使用效能(效率)評測器，你可以看到程式碼中哪個函式（程式）或其他資料消耗最多資源。

現在，讓我們繼續檢查網路請求。檢查網路請求可以幫助你偵錯與 API 請求相關的問題，例如參數錯誤、位址錯誤或伺服端錯誤。

要使用 Network 分頁進行偵錯，請開啟開發者工具並前往 Network 分頁，然後點擊各個請求以查看標頭、回應和有效載荷等詳細資訊。

在本課程的最後部分，我們將專注於 `console.table` 和 `console.dir`。

`console.table()` 會在主控台中以表格形式顯示表格式資料。它接受一個必填引數，該引數必須是陣列或物件，以及一個選用引數，用來指定要顯示哪些屬性或行。

另一方面，`console.dir()` 讓你顯示指定 JavaScript 物件的屬性作用中列表。它輸出一個可展開以查看所有巢狀屬性的階層式列表。

既然你已經擴充了偵錯技術的套件，請在你即將進行的 JavaScript 專案中測試這些新技能。透過應用這些策略，你將能更高效地識別和決議問題，從而產生更乾淨、更可靠的程式碼。
