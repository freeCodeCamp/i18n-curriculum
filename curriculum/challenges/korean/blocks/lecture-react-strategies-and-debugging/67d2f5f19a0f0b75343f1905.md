# --description--

React 서버 컴포넌트(RSC)는 React 개발자들이 접근 방식을 바꾸게 한 새로운 트렌드입니다. RSC를 사용하면 더 많은 작업이 서버로 이동하며, 이는 많은 이점을 가져옵니다.

서버 컴포넌트가 무엇인지, 어떻게 작동하는지, 그리고 서버 컴포넌트가 도입된 배경을 살펴보겠습니다.

React 서버 컴포넌트는 서버에서만 렌더링되는 React 컴포넌트로, 최종 HTML만 클라이언트에 전송합니다. 이는 해당 컴포넌트가 서버 측 리소스에 직접 접근할 수 있고, 브라우저로 전송되는 JavaScript 양을 크게 줄일 수 있음을 의미합니다.

React 앱은 전통적으로 렌더링, 상호작용, 부수 효과 등 모든 것을 처리하는 "클라이언트 컴포넌트" 시스템을 사용해왔습니다. "클라이언트 컴포넌트"라는 용어는 최근 React 서버 컴포넌트가 도입되기 전까지는 거의 사용되지 않았습니다.

하지만 클라이언트 컴포넌트 시스템에는 큰 JavaScript 번들 크기와 느린 초기 로드 시간 같은 단점이 있습니다.

Next.js와 Gatsby 같은 React 프레임워크는 이러한 문제를 해결하기 위해 일부 프로세스를 서버로 오프로드하는 우회 방법을 찾았지만, 표준화되지는 않았습니다. 두 프레임워크를 사용해봤다면 `getServerSideProps`와 `getServerData`에 대해 들어봤을 것입니다.

그다음에 React 서버 컴포넌트가 등장했는데, 이를 통해 일부 컴포넌트를 완전히 서버에서 실행하여 사용자의 브라우저에서 코드가 실행되기 전에 데이터 가져오기와 계산 같은 작업을 할 수 있습니다.

서버 컴포넌트는 Next.js에서 처음 대중화되었고 쉽게 사용할 수 있습니다. Remix와 Gatsby 같은 다른 프레임워크도 따라잡고 있으며, Vite용 실험적 플러그인인 `vite-plugin-react-server`도 서버 컴포넌트를 빌드할 수 있게 해줍니다.

그렇다면 서버 컴포넌트는 어떻게 작동할까요?

React 서버 컴포넌트를 설명하는 가장 좋은 방법 중 하나는 데이터 가져오기입니다.

전통적인 React 클라이언트 컴포넌트에서는 브라우저가 API 요청을 처리하게 합니다. 데이터 가져오기는 부수 효과이므로 `useEffect` 훅에서 API 호출을 합니다.

또한 로딩, 데이터, 오류 같은 상태 변수를 설정하여 데이터가 로딩 중임을 표시하거나, 데이터가 준비되면 보여주거나, 앱에서 오류를 표시하는 것이 좋은 관행입니다.

React 서버 컴포넌트를 사용하면 전체 컴포넌트를 서버로 옮겨 `useState`나 `useEffect`를 사용하지 않고도 데이터를 가져올 수 있습니다:

```jsx
const Users = async () => {
  const res = await fetch("https://jsonplaceholder.typicode.com/users");
  const users = await res.json();

  return (
    <>
      <h1 className="text-4xl text-center mt-6">Users</h1>
      <ul className="text-center mt-3">
        {users.map((user) => (
          <li key={user.id}>{user.name}</li>
        ))}
      </ul>
    </>
  );
};

export default Users;
```

React 서버 컴포넌트는 서버에서만 실행되므로 API에서 데이터를 가져와 한 번만 렌더링하면 됩니다. 또한 데이터 가져오기가 서버에서, 즉 데이터 출처에 더 가까운 곳에서 이루어지기 때문에 네트워크 속도가 느린 사용자에게 특히 앱 성능이 더 좋아질 수 있습니다.

한 가지 중요한 점은 서버 컴포넌트의 모든 코드는 서버에 남아 있고 브라우저로 전송되지 않는다는 것입니다. 따라서 React 훅을 사용할 수 없고, Web API나 브라우저 이벤트 리스너에도 접근할 수 없습니다. 그렇다면 상호작용은 어떻게 추가할 수 있을까요?

Next.js 앱 라우터에서는 모든 컴포넌트가 기본적으로 서버 컴포넌트입니다. 상호작용을 추가하려면 `"use client"` 지시어로 컴포넌트를 클라이언트 컴포넌트로 표시해야 합니다.

이전 예제를 클라이언트 컴포넌트로 만들고 싶다면 이렇게 하면 됩니다:

```jsx
"use client";

import { useState, useEffect } from "react";

const Users2 = () => {
  const [status, setStatus] = useState({
    users: [],
    loading: true,
    error: null,
  });

  async function fetchUsers() {
    try {
      const res = await fetch("https://jsonplaceholder.typicode.com/users");
      const data = await res.json();
      setStatus((prevStatus) => ({
        ...prevStatus,
        users: data,
        loading: false,
      }));
    } catch (err) {
      setStatus((prevStatus) => ({
        ...prevStatus,
        error: err.message,
        loading: false,
      }));
    }
  }

  useEffect(() => {
    fetchUsers();
  }, []);

  if (status.loading) {
    return <p>Loading Users...</p>;
  }
  if (status.error) {
    return <p>Error getting users: {status.error}</p>;
  }

  return (
    <>
      <h1 className="text-4xl text-center mt-6">Users</h1>
      <ul className="text-center mt-3">
        {status.users.map((user) => (
          <li key={user.id}>{user.name}</li>
        ))}
      </ul>
    </>
  );
};

export default Users2;
```

클릭 이벤트 같은 상호작용을 추가하려면 컴포넌트도 클라이언트 컴포넌트로 표시해야 합니다:

```jsx
"use client";

import { useState } from "react";

const Counter = () => {
  const [count, setCount] = useState(0);

  return (
    <>
      <h1>Counter</h1>
      <button onClick={() => setCount(count + 1)}>Increment</button>
      <h2>{count}</h2>
      <button onClick={() => setCount(count - 1)}>Decrement</button>
    </>
  );
};

export default Counter;
```

컴포넌트에 use client 지시어를 추가하지 않으면 "`useState`가 필요한 서버 컴포넌트를 가져오고 있습니다. 이 React 훅은 클라이언트 컴포넌트에서만 작동합니다. 해결하려면 파일(또는 상위 파일)에 `"use client"` 지시어를 표시하세요."라는 오류 메시지가 나타납니다.

React 서버 컴포넌트의 주요 장점은 데이터 가져오기가 더 간단해지고, 코드가 더 읽기 쉬워지며, 클라이언트 복잡성이 줄어든다는 점입니다.
