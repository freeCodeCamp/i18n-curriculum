# --description--

표현식에 연산자가 여러 개 있을 때 실행 순서는 우선순위, 즉 연산자의 기본 우선순위에 따라 정해집니다.

학교에서 배웠듯이 표현식 `1 + 2 * 2`에서 곱셈은 덧셈보다 먼저 계산되어야 합니다. 이것이 바로 우선순위 개념입니다. 곱셈이 덧셈보다 우선순위가 높다고 합니다.

소괄호는 모든 우선순위를 재정의하므로 기본 순서가 마음에 들지 않으면 소괄호를 사용해 변경할 수 있습니다. 예를 들어 `(1 + 2) * 2`을 작성하세요.

JavaScript에는 많은 연산자가 있습니다. 각 연산자는 해당하는 우선순위 숫자를 가지고 있습니다. 숫자가 클수록 먼저 실행됩니다. 우선순위가 같으면 실행 순서는 왼쪽에서 오른쪽입니다.

다음은 우선순위 표의 일부입니다(외우지 않아도 되지만 단항 연산자가 해당 이항 연산자보다 우선순위가 높다는 점을 참고하세요):

| 우선순위 | 이름             | 기호 |
|------------|------------------|------|
| ...        | ...              | ...  |
| 14         | 단항 덧셈        | +    |
| 14         | 단항 부정        | -    |
| 13         | 거듭제곱         | **   |
| 12         | 곱셈             | *    |
| 12         | 나눗셈           | /    |
| 11         | 덧셈             | +    |
| 11         | 뺄셈             | -    |
| ...        | ...              | ...  |
| 2          | 할당             | =    |
| ...        | ...              | ...  |



# --questions--

## --text--

JavaScript 표현식에서 여러 연산자가 있을 때 실행 순서에 대해 올바른 설명은 무엇인가요?

## --answers--

연산자는 우선순위와 상관없이 오른쪽에서 왼쪽으로 실행됩니다.

---

우선순위가 높은 연산자가 먼저 실행되며, 소괄호는 미리 정해진 우선순위를 재정의할 수 있습니다.

---

연산자 우선순위는 연산자의 실행 속도로 결정됩니다.

---

곱셈은 항상 덧셈보다 우선하며, 단항 연산자는 이항 연산자 다음에 실행됩니다.
