# --interactive--

CSS-Kombinatoren werden verwendet, um die Beziehung zwischen Selektoren in CSS zu definieren. Sie helfen dabei, Elemente basierend auf ihrer Beziehung zu anderen Elementen auszuwählen, was präziseres und effizienteres Styling ermöglicht.

Wir besprechen einige der wichtigsten CSS-Kombinatoren und ihre Anwendungsfälle, beginnend mit dem Nachfahren-Kombinator.

Ein Nachfahre-Kombinator wird verwendet, um Elemente anzusprechen, die vom zweiten Selektor ausgewählt werden, wenn sie innerhalb eines Vorfahren-Elements verschachtelt sind, das vom ersten Selektor ausgewählt wird. Ein Vorfahre kann ein Elternteil, Großelternteil oder ein beliebiger weiter oben im Dokumentbaum liegender Vorfahre sein.

Um besser zu verstehen, wie das funktioniert, schauen wir uns ein Beispiel an.

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<figure>
  <img src="https://cdn.freecodecamp.org/curriculum/cat-photo-app/relaxing-cat.jpg" alt="Relaxing Cat">
  <figcaption>A relaxing cat with a border</figcaption>
</figure>
```

```css
figure img {
  border: 4px solid red;
}
```

:::

Im obigen Beispiel verwenden wir den Nachfahren-Kombinator, um alle Bild-Elemente innerhalb von `figure`-Elementen auszuwählen und auf allen vier Seiten einen `solid red`-Rahmen anzuwenden.

Beachten Sie, dass zwischen dem Eltern- und dem Kind-Selektor ein `space` verwendet wird.

In diesem Fall wäre das `figure` das Elternelement und das `img` das Kindelement.

Wenn Sie mehrere Bilder innerhalb eines `figure`-Elements hätten, wäre die Verwendung des Nachfahren-Kombinators eine gute Möglichkeit, jedem dieser Bilder einen schwarzen Rahmen zu geben.

Eine andere Art von Kombinator ist der Kind-Kombinator.

Der Kind-Kombinator (`>`) in CSS wird verwendet, um Elemente auszuwählen, die direkte Kinder eines bestimmten Elternelements sind.

Dieser Kombinator zielt nur auf Elemente mit einem bestimmten Elternteil ab, wodurch Ihre CSS-Regeln präziser werden und unbeabsichtigtes Styling von tiefer verschachtelten Elementen verhindert wird.

Schauen wir uns das folgende HTML-Beispiel an:

:::interactive_editor

```html
<div class="container">
  <p>First</p>
  <div>
    <p>Second</p>
  </div>
  <div>
    <p>Third</p>
  </div>
</div>
```

:::

In der obigen HTML-Struktur wird die `container`-Klasse auf ein `div`-Element angewendet.

Innerhalb dieses Containers gibt es ein direktes Kind-`p`-Element („First“), gefolgt von zwei weiteren `div`-Elementen, die jeweils ein `p`-Element („Second“ und „Third“) enthalten.

Das erste `p`-Element ist ein direktes Kind des `.container`-Elements, während die anderen beiden `p`-Elemente innerhalb anderer `div`-Elemente verschachtelt sind und somit tiefere Nachfahren darstellen.

Um nur das direkte Kind der `container`-Klasse zu stylen, können Sie den Kind-Kombinator so verwenden:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<div class="container">
  <p>First</p>
  <div>
    <p>Second</p>
  </div>
  <div>
    <p>Third</p>
  </div>
</div>
```

```css
.container > p {
  color: blue;
}
```

:::

Im obigen Beispiel zielen wir nur auf das direkte Kind der `container`-Klasse ab. Dieses erhält die Textfarbe `blue`.

Da die anderen beiden Absatz-Elemente innerhalb von `div`-Elementen verschachtelt sind, gelten sie nicht als direkte Kinder der `container`-Klasse und erhalten daher nicht die blaue Textfarbe.

Ein weiterer häufiger Kombinator ist der Nachbar-Kombinator.

Der Nachbar-Kombinator (`+`) in CSS wählt ein Element aus, das unmittelbar auf ein bestimmtes Geschwisterelement folgt. Dieser Kombinator ist nützlich, wenn Sie einem Element, das direkt auf ein anderes folgt, ein Styling zuweisen möchten, sodass Sie gezieltes Styling basierend auf der Position des Elements relativ zu seinen Geschwistern vornehmen können.

Schauen wir uns das folgende HTML-Beispiel an:

:::interactive_editor

```html
<figure>
  <img
    src="https://cdn.freecodecamp.org/curriculum/cat-photo-app/relaxing-cat.jpg"
    alt="A cute orange cat lying on its back."
  />
  <figcaption>A cute orange cat lying on its back.</figcaption>
</figure>
```

:::

Hier haben wir ein `figure`-Element, das ein `img`-Element enthält, gefolgt von einem `figcaption`-Element. Das `figcaption`-Element ist das unmittelbare Geschwisterelement des `img`-Elements.

Wenn Sie dem `figcaption`-Element einen schwarzen Rahmen geben möchten, können Sie den Nachbar-Kombinator so verwenden:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<figure>
  <img
    src="https://cdn.freecodecamp.org/curriculum/cat-photo-app/relaxing-cat.jpg"
    alt="A cute orange cat lying on its back."
  />
  <figcaption>A cute orange cat lying on its back.</figcaption>
</figure>
```

```css
img + figcaption {
  border: 4px solid black;
}
```

:::

In diesem Beispiel wählt der Nachbar-Kombinator (`+`) das `figcaption`-Element aus, das unmittelbar auf das `img`-Element folgt. Die angewendete CSS-Regel fügt dem `4px solid black border` einen `figcaption` hinzu.

Ein weiterer häufiger Kombinator ist der allgemeine Geschwister-Kombinator.

Der allgemeine Geschwister-Kombinator (`~`) in CSS wählt alle Geschwister eines bestimmten Elements aus, die nach ihm kommen.

Im Gegensatz zum Nachbar-Kombinator, der nur das unmittelbar folgende Geschwisterelement auswählt, kann der allgemeine Geschwister-Kombinator (`~`) alle Geschwister ansprechen, die dem angegebenen Element folgen, und bietet so mehr Flexibilität beim Styling.

Schauen wir uns das folgende HTML-Beispiel an:

:::interactive_editor

```html
<div class="container">
  <h2>Subheading</h2>
  <p>First paragraph.</p>
  <p>Second paragraph.</p>
  <p>Third paragraph.</p>
  <p>Another paragraph element</p>
</div>
```

:::

In dieser HTML-Struktur haben wir ein `h2`-Element, gefolgt von vier Absatz-Elementen. Die Absatz-Elemente sind Geschwister des `h2`-Elements.

Wenn Sie alle Absatz-Elemente stylen möchten, die nach dem `h2`-Element kommen, können Sie den allgemeinen Geschwister-Kombinator so verwenden:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<div class="container">
  <h2>Subheading</h2>
  <p>First paragraph.</p>
  <p>Second paragraph.</p>
  <p>Third paragraph.</p>
  <p>Another paragraph element</p>
</div>
```

```css
h2 ~ p {
  color: green;
}
```

:::

In diesem Beispiel erhalten alle Absatz-Elemente, die dem `h2`-Element folgen, die Textfarbe Grün.

Der allgemeine Geschwister-Kombinator (`~`) zielt auf alle Absatz-Geschwister ab, die nach dem `h2`-Element erscheinen, unabhängig davon, ob sie unmittelbare Geschwister sind.

Zusammenfassend ermöglicht Ihnen das Verständnis und die Verwendung verschiedener CSS-Kombinatoren, präzise Styles auf Ihre HTML-Elemente anzuwenden und so die Kontrolle und Flexibilität Ihres Designs zu verbessern.

Wenn Sie diese Selektoren beherrschen, können Sie anspruchsvollere und gezieltere Styling-Regeln erstellen, die sowohl das Aussehen als auch die Funktionalität Ihrer Webseiten verbessern.
