# --description--

Come hai imparato nelle lezioni precedenti, i cookie sono dati semplici che i siti web possono archiviare sul dispositivo di un utente.

Tuttavia, puoi effettivamente archiviare strutture dati più complesse nei cookie. Un metodo comune è usare JSON per archiviare oggetti o array. Ecco un esempio:

```js
const userData = {
  name: "John Doe",
  age: 30,
  role: "admin"
};

document.cookie = "userInfo=" + JSON.stringify(userData) + "; path=/";
```

In questo esempio, creiamo un oggetto con dati utente, lo convertiamo in una stringa JSON e poi lo archiviamo in un cookie. Quando vogliamo recuperare questi dati, possiamo analizzare la stringa JSON in un oggetto usando `JSON.parse()`.

Ora, potresti chiederti dell'aspetto degli "header HTTP" nel nostro argomento. Tipicamente, i cookie vengono impostati dal server usando gli header HTTP.

Per esempio, un server potrebbe inviare un header come questo:

```http
Set-Cookie: username=John Doe; expires=Thu, 31 Dec 2024 6:00:00 IST; path=/
```

Questo header dice al browser di impostare un cookie.

Possiamo anche impostare i cookie direttamente nel browser usando JavaScript. Questo è utile per archiviare dati che non devono essere inviati immediatamente al server.

Nota che i cookie hanno un limite di dimensione di circa 4KB, e archiviare troppi dati in essi può rallentare la tua app web.

Archiviare grandi quantità di dati nei cookie può aumentare il traffico di rete poiché i cookie vengono inviati con ogni richiesta HTTP.
