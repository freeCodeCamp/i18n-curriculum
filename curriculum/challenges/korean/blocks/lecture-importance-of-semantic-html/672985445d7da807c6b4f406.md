# --interactive--

구조적 계층을 만드는 데 가장 중요한 점은 제목 요소를 올바르게 사용하는 것입니다. 제목 요소는 `h1`, `h2`, `h3` 등으로 번호가 매겨집니다. 이 번호들은 해당 요소의 제목 수준을 나타냅니다.

텍스트 문서와 마찬가지로 제목 수준을 잘못된 순서로 사용하지 않아야 합니다. `h1` 요소는 최상위 제목입니다. 페이지에 이 요소가 여러 개 있는 경우는 드물고, 일반적으로 모든 콘텐츠 앞에 와야 합니다.

`h2` 요소는 하위 제목입니다. 항상 `h1` 다음에 와야 하며, 일부 소개 텍스트 다음에 올 수도 있습니다. `h1` 요소와 달리, 페이지에 여러 개의 `h2` 요소가 있을 수 있습니다. 이는 종종 서로 다른 콘텐츠 섹션을 구분하기 위해 사용합니다.

이 패턴에 따라 `h3` 요소는 항상 `h2` 요소 다음에 와야 합니다. 즉, `h1`에서 `h3`로 바로 건너뛰면 안 됩니다. 하지만 같은 수준의 제목 요소가 여러 개 있을 수는 있습니다. 예를 들어, 이 코드는 올바릅니다:

```html
<section>
  <h1>freeCodeCamp</h1>
  <h2>Learn Front-End Development</h2>
  <h2>Learn Back-End Development</h2>
</section>
```

하지만 `h3`가 `h2`보다 먼저 나오면 이 코드는 올바르지 않습니다.

`h2` 요소를 `h3` 위로 옮겨 의미상 올바르게 만드세요. 코드를 편집하려면 인터랙티브 편집기를 활성화해야 합니다.

:::interactive_editor

```html
<section>
  <h1>freeCodeCamp</h1>
  <h3>Introduction to HTML</h3>
  <h2>Learn Front-End Development</h2>
</section>
```

:::

큰 텍스트 스타일 때문에 `h1` 같은 특정 제목 요소를 사용하고 싶을 수 있습니다:

:::interactive_editor

```html
<article>
  <p>
    Here is some
    <h1>Large Text</h1>
  </p>
</article>
```

:::

대신 문서 구조에 맞는 적절한 요소를 선택하고 CSS로 원하는 스타일을 적용하세요.

올바른 계층 구조를 사용하는 것은 접근성에 중요합니다. 화면 읽기 프로그램 같은 보조 기술은 웹 페이지의 구조를 기반으로 페이지를 해석하고 사용자에게 알립니다. `h3` 요소가 `h1` 다음에 오면 `h2` 요소가 없어서 화면 읽기 프로그램 사용자가 중요한 내용을 건너뛴 것으로 오해할 수 있습니다.

적절한 구조는 SEO에도 중요합니다. 검색 엔진은 자동화로 웹 페이지 내용을 분석해 결과에 언제 어디에 표시할지 결정합니다. 구조가 잘못되면 관련 검색 결과에서 좋은 순위를 받기 어려울 수 있습니다.

마지막으로, 구조가 너무 잘못되면 HTML이 기술적으로 유효하지 않을 수도 있습니다. 이 경우 웹 브라우저가 작성자가 의도한 바를 추측해야 하는데, 그 추측이 원하는 결과와 다를 수 있습니다.

오늘 배운 것처럼, 페이지에 올바른 구조적 계층을 사용하는 데는 여러 이유가 있습니다. 새 프로젝트를 만들 때 이 점을 기억하세요.
