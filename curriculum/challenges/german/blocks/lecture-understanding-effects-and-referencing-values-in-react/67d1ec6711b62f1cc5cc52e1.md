# --description--

In React ist ein Effekt alles, was außerhalb des Renderprozesses einer Komponente passiert. Das heißt, alles, was React nicht direkt als Teil des Renderns der Benutzeroberfläche steuert.

Typische Beispiele sind das Abrufen von Daten, das Aktualisieren des Titels des Browser-Tabs, das Lesen aus oder Schreiben in den lokalen Speicher des Browsers, das Ermitteln des Standorts des Benutzers und vieles mehr. Diese Vorgänge interagieren mit der Außenwelt und werden als Nebeneffekte bezeichnet.

React stellt den `useEffect`-Hook bereit, mit dem Sie diese Nebeneffekte handhaben können. `useEffect` ermöglicht es Ihnen, eine Funktion auszuführen, nachdem die Komponente gerendert oder aktualisiert wurde.

Sehen wir uns an, wie der `useEffect`-Hook funktioniert und warum er für die moderne React-Entwicklung so wichtig ist.

Um den `useEffect`-Hook zu verwenden, müssen Sie ihn zuerst importieren:

```js
import { useEffect } from "react";
```

Dann verwenden Sie ihn als Funktion, so:

```js
useEffect(() => {
  // Your side effect logic (usually a function) here
}, [dependencies]);
```

Die Effektfunktion wird ausgeführt, nachdem die Komponente gerendert wurde, während das optionale `dependencies`-Argument steuert, wann der Effekt ausgeführt wird.

Beachten Sie, dass `dependencies` ein Array von „reaktiven Werten“ (State, Props, Funktionen, Variablen usw.), ein leeres Array oder ganz weggelassen werden kann. So steuern all diese Optionen, wie `useEffect` funktioniert:

- Wenn `dependencies` ein Array ist, das einen oder mehrere reaktive Werte enthält, wird der Effekt immer dann ausgeführt, wenn sich diese ändern.

- Wenn `dependencies` ein leeres Array ist, wird `useEffect` nur einmal ausgeführt, wenn die Komponente zum ersten Mal gerendert wird.

- Wenn Sie `dependencies` weglassen, wird der Effekt bei jedem Rendern oder Aktualisieren der Komponente ausgeführt.

Zum Beispiel in dieser `Counter`-Anwendung übergeben wir kein `dependencies`-Argument, sodass der Effekt beim Rendern der Komponente und bei jeder Aktualisierung ausgeführt wird:

```jsx
import { useState, useEffect } from "react";

const Counter = () => {
  const [count, setCount] = useState(0);

  useEffect(() => {
    console.log("Component renders");
  });

  return (
    <div
      style={{
        display: "flex",
        alignItems: "center",
        flexDirection: "column",
      }}
    >
      <h2>{count}</h2>
      <div>
        <button onClick={() => setCount(count + 1)}>Increase</button>
        <button onClick={() => setCount(count - 1)}>Decrease</button>
      </div>
    </div>
  );
};

export default Counter;
```

Wenn wir jedoch ein leeres Array als Abhängigkeit übergeben, wird der Effekt nur beim ersten Rendern ausgeführt:

```js
useEffect(() => {
  console.log('Component renders');
}, []);
```

Wenn Sie den `count`-State als Abhängigkeit übergeben, wird der Effekt beim ersten Rendern der Komponente und bei jeder Änderung von `count` ausgeführt:

```js
useEffect(() => {
  document.title = `The current count is ${count}`;
  console.log('component renders');
}, [count]);
```

Beachten Sie, dass wenn der von Ihnen eingerichtete Effekt über den Renderzyklus der Komponente hinaus andauert, Sie möglicherweise eine weitere Funktion benötigen, um diese Funktion nach dem Rendern oder Aktualisieren der Komponente „aufzuräumen“.

Wenn Ihre Effektfunktion beispielsweise `setInterval()` verwendet, einen Ereignis-Listener wie `window.addEventListener()` setzt oder eine Verbindung zu einem Server herstellt, benötigen Sie eine Aufräumfunktion, die `clearInterval()`, `window.removeEventListener()` ausführt und die Verbindung zum Server trennt.

Hier ist die Syntax, um eine Aufräumfunktion aus dem `useEffect`-Hook zurückzugeben:

```js
useEffect(() => {
  // Your side effect logic here
  return () => {
    // Cleanup logic here (optional)
  };
}, [dependencies]);
```

Wenn Sie zum Beispiel einen Scroll-Ereignis-Listener hinzufügen, können Sie ihn in Ihrer Aufräumfunktion entfernen:

```js
useEffect(() => {
  const handleScroll = () => {
    // Handle scroll logic
  };
  window.addEventListener("scroll", handleScroll);

  return () => {
    window.removeEventListener("scroll", handleScroll);
  };
}, []);
```
