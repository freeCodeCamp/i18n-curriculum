# --description--

La inyección SQL es una vulnerabilidad de seguridad web en la que los atacantes modifican o aprovechan las consultas que la aplicación web realiza a la base de datos. Por ejemplo, los atacantes pueden insertar código SQL malicioso en la consulta para ejecutar el código malicioso en el backend.

Esto puede ocurrir cuando das a los usuarios la opción de ingresar datos a través de formularios de inicio de sesión, barras de búsqueda o parámetros de URL y luego usas esa entrada del usuario en interpolación o concatenación de cadenas para generar el comando SQL final.

Dependiendo del objetivo del atacante, el código SQL malicioso puede causar interrupciones significativas en tu base de datos y aplicación web, como evadir la autenticación, robar datos de la base de datos o modificar o eliminar registros de la base de datos. Por lo tanto, debes tomar esta amenaza muy en serio.

Por ejemplo, supongamos que tu aplicación web usa una consulta SQL como esta para autenticar a los usuarios, insertando directamente el nombre de usuario y la contraseña que el usuario ingresó:

```sql
'SELECT * FROM users WHERE username = ' + username_input + ' AND password = ' + password_input + ';'
```

Si un atacante ingresa algo así en el campo `username`:

```sql
'" " OR "1"="1" --'
```

Y cualquier otra cosa en el campo `password`, ya que `"1"="1"` siempre es `true` y `--` comenta el resto de la consulta, la cláusula `WHERE` se convierte en:

```sql
WHERE username = " " OR TRUE
```

Que siempre es `true`.

Dependiendo de la implementación de la base de datos, esto puede devolver los datos del primer usuario o de cualquier usuario en la base de datos y permitir que el atacante inicie sesión sin credenciales válidas.

Entonces, ¿cómo puedes prevenir esto? Hay dos formas principales de prevenir la inyección SQL:

- No escribas consultas dinámicas con concatenación de cadenas.
- Y evita que entradas SQL maliciosas se incluyan en consultas que se ejecutarán en el backend.

En la práctica, puedes usar sentencias parametrizadas, también conocidas como consultas parametrizadas. Estas separan la estructura SQL de los datos ingresados por el usuario, lo que evita que la base de datos interprete los datos de entrada como código ejecutable.

Otra técnica importante es la validación de entrada. Se recomienda como una medida de seguridad secundaria en todos los casos, pero es especialmente importante para las partes de las consultas SQL relacionadas con nombres de tablas, nombres de columnas y ordenamiento. Si es posible, se recomienda asignar los nombres de tablas y columnas directamente dentro de tu código, no a partir de la entrada del usuario.

Si, a pesar de todas las medidas preventivas, sufres un ataque de inyección SQL, puedes minimizar su impacto en la base de datos limitando los permisos de cada cuenta de base de datos. Otorga a cada usuario de la base de datos solo los permisos necesarios para sus tareas, pero nada más. Si una cuenta solo necesita leer datos de la base de datos, otórgale acceso de solo lectura. Y como medida preventiva, nunca asignes permisos de administrador a las cuentas de tu aplicación. Es muy peligroso hacerlo porque un atacante podría obtener acceso completo y control total sobre la base de datos.

La inyección SQL es un ataque muy común del que tú, como desarrollador, debes estar consciente y prevenir en la implementación de tu aplicación web. Con estas técnicas, puedes proteger tu base de datos y mantener seguros los datos de tus usuarios.
