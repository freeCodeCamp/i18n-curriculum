## العمل مع النماذج في React

- **المدخلات المُتحكم بها**: هذا عندما تخزن قيمة حقل الإدخال في الحالة وتُحدّثها عبر أحداث `onChange`. هذا يمنحك تحكمًا كاملاً في بيانات النموذج ويسمح بالتحقق الفوري والعرض الشرطي.

```jsx
import { useState } from "react";

function App() {
  const [name, setName] = useState("");

  const handleChange = (e) => {
    setName(e.target.value);
  };

  const handleSubmit = (e) => {
    e.preventDefault();
    console.log(name);
  };

  return (
    <>
      <form onSubmit={handleSubmit}>
        <label htmlFor="name">Your name</label> <br />
        <input value={name} id="name" onChange={handleChange} type="text" />
        <button type="submit">Submit</button>
      </form>
    </>
  );
}

export default App;
```

- **المدخلات غير المُتحكم بها**: بدلاً من التعامل مع المدخلات عبر الخطاف `useState`، تحتفظ المدخلات غير المُتحكم بها في HTML بحالتها الداخلية بمساعدة DOM. وبما أن DOM يتحكم في قيم الإدخال، تحتاج إلى جلب قيم حقول الإدخال باستخدام `ref`.

```jsx
import { useRef } from "react";

function App() {
 const nameRef = useRef();

 const handleSubmit = (e) => {
   e.preventDefault();
   console.log(nameRef.current.value);
 };

 return (
   <form onSubmit={handleSubmit}>
     <label htmlFor="name">Your</label>{" "}
     <input type="text" ref={nameRef} id="name" />
     <button type="submit">Submit</button>
   </form>
 );
}

export default App;
```

## العمل مع الخطاف `useActionState`

- **إجراءات الخادم**: هي دوال تُنفذ على الخادم للسماح بمعالجة النموذج مباشرة على الخادم دون الحاجة إلى نقاط نهاية API. إليك مثالًا من تطبيق Next.js:

```js
"use server";

async function submitForm(formData) {
 const name = formData.get("name");
 return { message: `Hello, ${name}!` };
}
```

تُشير التوجيهة `"user server"` إلى أن الدالة هي إجراء خادم.

- **الخطاف `useActionState`**: يُحدّث هذا الخطاف الحالة بناءً على نتيجة إرسال النموذج. إليك الصيغة الأساسية للخطاف `useActionState`:

```js
const [state, action, isPending] = useActionState(actionFunction, initialState, permalink);
```

- `state` هي الحالة الحالية التي تُرجعها الإجراء.
- `action` هي الدالة التي تُشغّل إجراء الخادم.
- `isPending` هي قيمة منطقية تشير إلى ما إذا كان الإجراء قيد التنفيذ أم لا.
- معلمة `actionFunction` هي إجراء الخادم نفسه.
- `initialState` هي المعلمة التي تمثل نقطة البداية للحالة قبل تشغيل الإجراء.
- `permalink` هي سلسلة اختيارية تحتوي على عنوان URL الفريد للصفحة التي يُعدّلها النموذج.

## جلب البيانات في React

- **خيارات جلب البيانات**: هناك طرق عديدة لجلب البيانات في React. يمكنك استخدام Fetch API الأصلي، أو أدوات طرف ثالث مثل Axios أو SWR.
- **متغيرات الحالة الشائعة عند جلب البيانات**: بغض النظر عن الطريقة التي تختارها لجلب البيانات في React، هناك بعض متغيرات الحالة التي تحتاج إلى تتبعها. أولها البيانات نفسها. الثاني يتتبع ما إذا كانت البيانات لا تزال تُجلب. الثالث هو متغير حالة يلتقط أي أخطاء قد تحدث أثناء عملية جلب البيانات.

```js
const [data, setData] = useState(null);
const [loading, setLoading] = useState(true);
const [error, setError] = useState(null);
```

بما أن جلب البيانات هو تأثير جانبي، من الأفضل استخدام `Fetch API` داخل الخطاف `useEffect`.

```js
useEffect(() => {
  const fetchData = async () => {
    try {
      const res = await fetch("https://jsonplaceholder.typicode.com/posts");
      
      if (!res.ok) {
        throw new Error("Network response was not ok");
      }

      const data = await res.json();
      setData(data);
    } catch (err) {
      setError(err);
    } finally {
      setLoading(false);
    }
  };

  fetchData();
}, []);
```

بعد ذلك يمكنك عرض رسالة تحميل إذا لم تكتمل عملية جلب البيانات، أو رسالة خطأ إذا حدث خطأ أثناء الجلب، أو عرض النتائج.

```jsx
if (loading) {
  return <p>Loading...</p>;
}

if (error) {
  return <p>{error.message}</p>;
}

return (
  <ul>
    {data.map((post) => (
      <li key={post.id}>{post.title}</li>
    ))}
  </ul>
);
```

إذا أردت استخدام Axios، تحتاج إلى تثبيته واستيراده:

```bash
npm i axios
```

```js
import axios from "axios";
```

ثم يمكنك جلب البيانات باستخدام `axios.get`:

```jsx
const [data, setData] = useState(null);
const [loading, setLoading] = useState(true);
const [error, setError] = useState(null);

useEffect(() => {
  const fetchData = async () => {
    try {
      const res = await axios.get(
        "https://jsonplaceholder.typicode.com/users"
      );
      setData(res.data);
    } catch (err) {
      setError(err);
    } finally {
      setLoading(false);
    }
  };

  fetchData();
}, []);
```

لاستخدام الخطاف `useSWR` لجلب البيانات، تحتاج أولاً إلى تثبيته واستيراده.

```bash
npm i swr
```

```js
import useSWR from "swr";
```

إليك كيفية استخدام الخطاف لجلب البيانات:

```jsx
import useSWR from "swr";

const fetcher = (url) => fetch(url).then((res) => res.json());

const FetchTodos = () => {
 const { data, error } = useSWR(
   "https://jsonplaceholder.typicode.com/todos",
   fetcher
 );

 if (!data) {
   return <h2>Loading...</h2>;
 }
 if (error) {
   return <h2>Error: {error.message}</h2>;
 }

 return (
   <>
     <h2>Todos</h2>
     <div>
       {data.map((todo) => (
         <h3 key={todo.id}>{todo.title}</h3>
       ))}
     </div>
   </>
 );
};

export default FetchTodos;
```

## العمل مع الخطاف `useOptimistic`

- **الخطاف `useOptimistic`**: يُستخدم هذا الخطاف للحفاظ على تجاوب واجهات المستخدم أثناء انتظار اكتمال إجراء غير متزامن في الخلفية. يساعد في إدارة "التحديثات المتفائلة" في الواجهة، وهي استراتيجية تقدم تحديثات فورية للواجهة بناءً على النتيجة المتوقعة لإجراء، مثل انتظار استجابة الخادم.

إليك الصيغة الأساسية:

```js
const [optimisticState, addOptimistic] = useOptimistic(actualState, updateFunction);
```

- `optimisticState` هي الحالة المؤقتة التي تُحدّث فورًا لتحسين تجربة المستخدم.
- `addOptimistic` هي الدالة التي تطبق التحديث المتفائل قبل تغيّر الحالة الفعلي.
- `actualState` هي القيمة الحقيقية للحالة الناتجة عن نتيجة إجراء، مثل جلب بيانات من خادم.
- `updateFunction` هي الدالة التي تحدد كيف يجب تحديث الحالة المتفائلة عند استدعائها.

إليك مثالًا على استخدام الخطاف `useOptimistic` في مكوّن `TaskList`:

```jsx
"use client";

import { useOptimistic } from "react";

export default function TaskList({ tasks, addTask }) {
  const [optimisticTasks, addOptimisticTask] = useOptimistic(
    tasks,
    (state, newTask) => [...state, { text: newTask, pending: true }]
  );

  async function handleSubmit(e) {
    e.preventDefault();
    const formData = new FormData(e.target);

    addOptimisticTask(formData.get("task"));

    addTask(formData);
    e.target.reset();
  }

  return <>{/* UI */}</>;
}
```

- **`startTransition`**: يُستخدم هذا لتقديم جزء من الواجهة وتحديد تحديث الحالة كتحول غير عاجل. يسمح هذا للواجهة بأن تبقى متجاوبة أثناء التحديثات المكلفة. إليك الصيغة الأساسية:

```js
startTransition(action);
```

يقوم `action` بتحديث الحالة أو تشغيل بعض المنطق المرتبط بالتحول. هذا يضمن عدم حجب تحديثات الواجهة العاجلة (مثل الكتابة أو النقر).

## العمل مع الخطاف `useMemo`

- **التخزين المؤقت (Memoization)**: هي تقنية تحسين تُخزّن نتائج استدعاءات الدوال المكلفة بناءً على معلمات محددة. عند تقديم نفس المعلمات مرة أخرى، تُعاد النتيجة المخزنة بدلاً من إعادة حساب الدالة.
- **الخطاف `useMemo`**: يُستخدم هذا الخطاف لتخزين القيم المحسوبة مؤقتًا. إليك مثالًا لتخزين نتيجة فرز مصفوفة كبيرة. ستُنفذ الدالة `expensiveSortFunction` فقط عندما تتغير `largeArray`:

```js
const memoizedSortedArray = useMemo(
  () => expensiveSortFunction(largeArray),
  [largeArray]
);
```

## العمل مع الخطاف `useCallback`

- **الخطاف `useCallback`**: يُستخدم هذا الخطاف لتخزين مراجع الدوال مؤقتًا.

```js
const handleClick = useCallback(() => {
  // code goes here
}, [dependency]);
``` 

- **`React.memo`**: يُستخدم هذا لتخزين مكوّن مؤقتًا لمنع إعادة عرضه غير الضرورية عندما لا تتغير خاصيته.

```jsx
const MemoizedComponent = React.memo(({ prop }) => {
 return (
   <>
     {/* Presentation */}
   </>
 )
});
``` 

## أدوات إدارة التبعيات

- **تعريف التبعية**: في البرمجيات، التبعية هي عندما يعتمد مكوّن أو وحدة في تطبيق على آخر ليعمل بشكل صحيح. التبعيات شائعة في التطبيقات لأنها تسمح للمطوّرين باستخدام دوال أو أدوات مُعدة مسبقًا من قبل الآخرين. التبعيتان الأساسيتان لمشروع React هما حزمتا `react` و `react-dom`:

```json
"dependencies": {
  "react": "^18.3.1",
  "react-dom": "^18.3.1"
}
```

- **تعريف مدير الحزم**: لإدارة تبعيات البرمجيات في مشروع، تحتاج إلى استخدام مدير حزم. مدير الحزم هو أداة تُستخدم للتثبيت، التحديث، وإزالة التبعيات. العديد من لغات البرمجة الشهيرة مثل JavaScript و Python و Ruby و Java تستخدم مديري حزم. من أشهر مديري الحزم لـ JavaScript: npm و Yarn و pnpm.
- **ملف `package.json`**: هو ملف تكوين رئيسي في المشاريع يحتوي على بيانات وصفية عن المشروع، مثل اسمه، إصداره، وتبعياته. كما يحدد السكربتات، معلومات الترخيص، وإعدادات أخرى تساعد في إدارة المشروع وتبعياته.
- **ملف `package-lock.json`**: هذا الملف يُثبت الإصدارات الدقيقة لكل الحزم التي يستخدمها مشروعك. عند تحديث حزمة، تُحدّث الإصدارات الجديدة في ملف القفل أيضًا.
- **مجلد `node_modules`**: يحتوي هذا المجلد على الشفرة الفعلية للتبعيات المدرجة في ملف `package.json`، بما في ذلك التبعيات المباشرة لمشروعك وأي تبعيات لتلك التبعيات.
- **تبعيات التطوير**: هي الحزم التي تُستخدم فقط أثناء التطوير وليس في الإنتاج. مثال على ذلك مكتبة اختبار مثل Jest. تثبت Jest كتبعيات تطوير لأنها ضرورية لاختبار التطبيق محليًا لكنها غير مطلوبة لتشغيل التطبيق في الإنتاج.

## React Router

- **مقدمة**: React Router هي مكتبة طرف ثالث تسمح لك بإضافة التوجيه إلى تطبيقات React. للبدء، تحتاج إلى تثبيت React Router في مشروع React موجود كما يلي:

```bash
npm i react-router
```

ثم داخل ملف `main.jsx` أو `index.jsx`، تحتاج إلى إعداد هيكل المسارات كما يلي:

```jsx
import { StrictMode } from "react";
import { createRoot } from "react-dom/client";
import { BrowserRouter, Routes, Route } from "react-router";
import App from "./App.jsx";

import "./index.css";

createRoot(document.getElementById("root")).render(
  <StrictMode>
    <BrowserRouter>
      <Routes>
        <Route path="/" element={<App />} />
      </Routes>
    </BrowserRouter>
  </StrictMode>
);
```

تُستخدم الخاصيتان `path` و `element` لربط عنوان URL بمكوّنات الواجهة. في هذه الحالة، نُعد مسارًا للصفحة الرئيسية يشير إلى المكوّن `App`.

- **إعداد عدة عروض ومسارات**: من الشائع في التطبيقات الكبيرة إعداد عدة عروض ومسارات كما يلي:

```jsx
<Routes>
  <Route index element={<Home />} />
  <Route path="about" element={<About />} />

  <Route path="products">
    <Route index element={<ProductsHome />} />
    <Route path=":category" element={<Category />} />
    <Route path=":category/:productId" element={<ProductDetail />} />
    <Route path="trending" element={<Trending />} />
  </Route>
</Routes>
```

الخاصية `index` في هذه الأمثلة تمثل المسار الافتراضي لجزء معين من المسار. لذا، سيُعرض مكوّن `Home` عند المسار الجذري `/` بينما يُعرض مكوّن `ProductsHome` عند المسار `/products`.

- **تداخل المسارات**: يمكنك تداخل المسارات داخل مسارات أخرى، مما يؤدي إلى إلحاق مسار الطفل بمسار الأب.

```jsx
<Route path="products">
  <Route path="trending" element={<Trending />} />
</Route>
```

في المثال أعلاه، سيكون مسار المنتجات الرائجة هو products/trending.

- **الأجزاء الديناميكية**: الجزء الديناميكي هو حيث يكون أي جزء من مسار URL ديناميكيًا.

```jsx
<Route path=":category" element={<Category />} />
```

في هذا المثال لدينا جزء ديناميكي يسمى `category`. عندما ينتقل المستخدم إلى عنوان URL مثل products/brass-instruments، يتغير العرض إلى مكوّن `Category` ويمكنك جلب البيانات المناسبة ديناميكيًا بناءً على الجزء.

- **الخطاف `useParams`**: يُستخدم هذا الخطاف للوصول إلى المعلمات الديناميكية من مسار URL.

```jsx
import { useParams } from "react-router";

export default function Category() {
  let params = useParams();
  {/* Accessing the category param: params.category */}
  {/* rest of code goes here */}
}
```

## أُطُر عمل React

- **مقدمة**: توفر أُطُر عمل React ميزات مثل التوجيه، تحسين الصور، جلب البيانات، التوثيق والمزيد. هذا يعني أنك قد لا تحتاج إلى إعداد تطبيقات منفصلة للواجهة الأمامية والخلفية لبعض الحالات. من أمثلة أُطُر عمل React: Next.js و Remix.
- **توجيه Next.js**: يشمل هذا النظام دعم المسارات الديناميكية، المسارات المتوازية، معالجات المسارات، إعادة التوجيه، التعريب والمزيد.

إليك مثالًا على إنشاء معالج طلب مخصص:

```js
export async function GET() {
  const res = await fetch("https://example-api.com");
  const data = await res.json();

  return Response.json({ data });
}
```

- **تحسين الصور في Next.js**: يوسع مكوّن `Image` عنصر HTML الأصلي `img` ويسمح بتحميل الصفحات بشكل أسرع وتحسين حجم الصور. هذا يعني أن الصور ستُحمّل فقط عند دخولها إطار العرض، وسيقدم مكوّن `Image` الصور بالحجم المناسب تلقائيًا لكل جهاز.

```jsx
import Image from "next/image";

export default function Page() {

  );
``` 

## تمرير الخصائص (Prop Drilling)

- **تعريف**: تمرير الخصائص هو عملية تمرير الخصائص من مكوّن أب إلى مكوّنات فرعية متداخلة بعمق، حتى عندما لا تحتاج بعض المكوّنات الفرعية إلى هذه الخصائص.

## إدارة الحالة

- **واجهة برمجة التطبيقات للسياق (Context API)**: يشير السياق إلى عندما يجعل مكوّن أب المعلومات متاحة للمكوّنات الفرعية دون الحاجة لتمريرها صراحة عبر الخصائص. تُستخدم `createContext` لإنشاء كائن سياق يمثل السياق الذي ستقرأه المكوّنات الأخرى. يُستخدم `Provider` لتوفير قيم السياق للمكوّنات الفرعية.

```jsx
import { useState, createContext } from "react";

const CounterContext = createContext();

const CounterProvider = ({ children }) => {
  const [count, setCount] = useState(0);

  return (
    <CounterContext.Provider value={{ count, setCount }}>
      {children}
    </CounterContext.Provider>
  );
};

export { CounterContext, CounterProvider };
```

- **Redux**: يدير Redux الحالة عبر توفير مخزن مركزي وتحكم صارم في تحديثات الحالة. يستخدم نمطًا متوقعًا مع الإجراءات، المخفضات، والبرمجيات الوسيطة. الإجراءات هي حمولة معلومات ترسل بيانات من تطبيقك إلى مخزن Redux، غالبًا ما تُشغّل بواسطة تفاعلات المستخدم. المخفضات هي دوال تحدد كيف يجب أن تتغير الحالة استجابة لتلك الإجراءات، مع ضمان تحديث الحالة بطريقة غير قابلة للتغيير. البرمجيات الوسيطة تعمل كجسر بين إرسال الإجراءات والمخفض، مما يسمح لك بتوسيع وظائف Redux (مثل التسجيل، التعامل مع العمليات غير المتزامنة) دون تعديل التدفق الأساسي.

- **Zustand**: حل إدارة الحالة هذا مثالي للتطبيقات الصغيرة إلى المتوسطة. يعمل باستخدام الخطاف `useStore` للوصول إلى الحالة مباشرة في المكوّنات والصفحات. هذا يتيح لك تعديل والوصول إلى البيانات دون الحاجة إلى إجراءات، مخفضات، أو مزود.

## تصحيح مكوّنات React باستخدام أدوات مطوّر React

- **أدوات مطوّر React**: هي امتداد للمتصفح يمكنك استخدامه في Chrome و Firefox و Edge لفحص مكوّنات React وتحديد مشاكل الأداء. بالنسبة لـ Safari، تحتاج إلى تثبيت حزمة npm `react-devtools`. بعد تثبيت أدوات مطوّر React وفتح تطبيق React في المتصفح، افتح أدوات مطوّر المتصفح للوصول إلى علامتي التبويب الإضافيتين المخصصتين لتصحيح React – المكوّنات وProfiler.
- **علامة تبويب المكوّنات**: تعرض هذه العلامة كل مكوّن في شكل شجرة. إليك بعض الأمور التي يمكنك القيام بها في هذه العلامة:
  - عرض تسلسل المكوّنات في التطبيق
  - فحص وتعديل الخصائص، الحالات، وقيم السياق في الوقت الحقيقي
  - فحص شفرة المصدر لكل مكوّن محدد
  - تسجيل بيانات المكوّن في وحدة التحكم
  - فحص عناصر DOM للمكوّن
- **علامة تبويب Profiler**: تساعدك هذه العلامة على تحليل أداء المكوّنات. يمكنك تسجيل أداء المكوّنات لتحديد عمليات إعادة العرض غير الضرورية، عرض مدة الالتزام، ومن ثم تحسين المكوّنات البطيئة.

## مكوّنات خادم React

- **تعريف**: مكوّنات خادم React هي مكوّنات React تُعرض حصريًا على الخادم، وترسل فقط HTML النهائي إلى العميل. هذا يعني أن تلك المكوّنات يمكنها الوصول مباشرة إلى موارد الخادم وتقليل كمية جافاسكريبت المرسلة إلى المتصفح بشكل كبير.
