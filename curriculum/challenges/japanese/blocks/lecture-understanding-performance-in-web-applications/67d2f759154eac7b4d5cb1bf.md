# --description--

ご存じの通り、体感パフォーマンスはサイトの実際の速さだけでなく、ユーザーにどれだけ速く感じられるかが重要です。例えば、実際には遅くないのに遅く感じたウェブサイトを使ったことはありませんか？それが体感パフォーマンス、つまりユーザーにとっての速さの感覚です。

小さな工夫がユーザー体験に大きな違いをもたらします。体感パフォーマンスを改善する実践的な方法を見ていきましょう！

スピナーはユーザーに実際よりも長く待たされている印象を与えがちです。代わりにスケルトンスクリーンを使ってみてください。これは実際のコンテンツを模したグレーのプレースホルダーで、徐々に読み込まれていきます：

```html
<div class="skeleton"></div>
<script>
  setTimeout(() => {
    document.querySelector(".skeleton").innerHTML = "✅ Content Loaded!";
  }, 2000);
</script>
```

ユーザーは暗闇で待つのではなく、ページが段階的に読み込まれていると感じます。

これに加えて、ユーザーはすべてが読み込まれるまで待つ必要はありません。重要なコンテンツを先に読み込み、残りはバックグラウンドで読み込みましょう。これは以前に遅延読み込みで見ましたね：

```html
<img src="placeholder.jpg" data-src="real-image.jpg" class="lazy">
<script>
  document.addEventListener("DOMContentLoaded", function () {
    const lazyImages = document.querySelectorAll("img.lazy");
    lazyImages.forEach(img => {
      img.src = img.dataset.src;
    });
  });
</script>
```

これにより、すべてが読み込まれていなくてもページがレスポンシブに感じられます。

では、もしサイトがユーザーが次にクリックするものを予測できたらどうでしょう？プリロードを使えば、ユーザーがリクエストする前に次のページを取得できます：

```html
<link rel="preload" href="next-page.html" as="document">
```

これにより、ユーザーが移動するときの体感読み込み時間がほぼゼロになります。

最後に、ユーザーは不確実さを嫌います。ボタンをクリックしたら、すぐにその操作を認識しましょう：

```html
<button onclick="this.innerText='⏳ Processing...'; setTimeout(() => this.innerText='✅ Done!', 2000);">
  Click Me
</button>
```

これにより、遅延があってもユーザーは何かが起きていると感じられます。

まとめると、体感パフォーマンスの改善は単なる速度の向上ではなく、ユーザーにとっての速さの感覚を高めることです。スケルトンスクリーンを使い、重要なコンテンツを優先し、賢くプリロードし、即時フィードバックを与えてサイトを速く感じさせましょう。
