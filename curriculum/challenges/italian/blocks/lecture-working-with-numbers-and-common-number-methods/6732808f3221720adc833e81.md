# --interactive--

`parseFloat()` e `parseInt()` sono due metodi essenziali in JavaScript per convertire stringhe in numeri. Questi metodi sono particolarmente utili quando si lavora con input dell'utente o si elaborano dati che arrivano in formato stringa ma devono essere trattati come valori numerici.

Iniziamo con `parseFloat()`. Questo metodo analizza un argomento stringa e restituisce un numero a virgola mobile. È progettato per estrarre un numero dall'inizio di una stringa, anche se la stringa contiene caratteri non numerici più avanti. Ricorda che i float sono numeri con la virgola decimale. Ecco come funziona `parseFloat()`:

:::interactive_editor

```js
console.log(parseFloat("3.14"));     // 3.14
console.log(parseFloat("3.14 abc")); // 3.14
console.log(parseFloat("3.14.5"));   // 3.14
console.log(parseFloat("abc 3.14")); // NaN
```

:::

Come puoi vedere, `parseFloat()` inizia ad analizzare dall'inizio della stringa e continua finché non incontra un carattere che non può far parte di un numero a virgola mobile. Se non trova un numero valido all'inizio della stringa, restituisce `NaN` (Not a Number).

`parseInt()`, invece, analizza un argomento stringa e restituisce un intero. Come `parseFloat()`, inizia dall'inizio della stringa, ma si ferma al primo carattere non numerico. Ecco come funziona `parseInt()`:

:::interactive_editor

```js
console.log(parseInt("42"));       // 42
console.log(parseInt("42px"));     // 42
console.log(parseInt("3.14"));     // 3
console.log(parseInt("abc123"));   // NaN
```

:::

`parseInt()` si ferma all'incontro del primo carattere non numerico. Per i numeri a virgola mobile, restituisce solo la parte intera. Se non trova un intero valido all'inizio della stringa, restituisce `NaN`.

Entrambi i metodi hanno alcuni comportamenti degni di nota. Ignorano gli spazi bianchi iniziali:

:::interactive_editor

```js
console.log(parseFloat("  3.14"));  // 3.14
console.log(parseInt("  42"));      // 42
```

:::

Gestiscono i segni più e meno all'inizio della stringa:

:::interactive_editor

```js
console.log(parseFloat("+3.14"));  // 3.14
console.log(parseInt("-42"));      // -42
```

:::

Vale la pena notare che, sebbene questi metodi siano potenti, hanno alcune limitazioni. Per esempio, non gestiscono direttamente tutti i formati numerici, come la notazione scientifica. Per esigenze di analisi più complesse, potresti dover usare tecniche o librerie aggiuntive.

In conclusione, `parseFloat()` e `parseInt()` sono strumenti preziosi per convertire stringhe in numeri in JavaScript. Capire come funzionano e i loro comportamenti specifici ti permette di gestire i dati numerici in modo più efficace nelle tue applicazioni, specialmente quando lavori con input utente o fonti di dati esterne.
