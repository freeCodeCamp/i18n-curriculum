# --description--

Reactは、プロジェクトでさまざまな機能を実装できる多くの組み込みフックを提供しています。これには`useState`、`useEffect`、`useContext`などがあります。

しかし時には、組み込みフックでは対応できない機能を追加したい場合があります。幸いなことに、Reactでは自分でカスタムフックを作成できます。

カスタムフックは見た目ほど複雑ではありません。複数のコンポーネント間でロジックを共有できる再利用可能な関数にすぎません。つまり、再利用性もカスタムフックを作る理由の一つです。

カスタムフックを使えば、データ取得、状態管理、トグル切り替え、ユーザーのオンライン・オフライン状態のチェックなどの副作用のようなロジックを、使用するコンポーネントから切り離せます。

そして、そのフックをどのコンポーネントでもインポートして使えるので、コンポーネント内ではレンダリングや表示に集中できます。つまり、繰り返しや重複が減り、更新したいときに変更箇所が少なくなります。

では、カスタムフックの作り方を見てみましょう。

Reactでは、すべての組み込みフックは`use`という単語で始まるので、カスタムフックも同じ慣例に従うべきです。カスタムフックの名前は、その機能が明確に伝わるものにしてください。

たとえば、カスタムフックが…

- データを取得するなら、`useFetch`と呼べます

- 何かをオン・オフ切り替えるなら、`useToggle`と呼べます

- あるいはデバウンスを実装するなら、`useDebounce`が良い名前です

ここでは、アプリにデバウンス機能を追加するカスタムフックを作るとしましょう。

デバウンスは、関数の実行頻度を制限するプログラミング手法です。ユーザーがある動作をやめてから指定した時間が経過するまで関数の実行を待ちます。たとえば検索ボックスでは、キー入力のたびにAPIを呼び出すのではなく、ユーザーが500ミリ秒ほど入力を止めるまで待ちます。

デバウンス用のカスタムフックを作るには、まず`useDebounce.jsx`または`useDebounce.js`ファイルを作成します。慣例として、作成したカスタムフックのファイルは`hooks`フォルダーに保存します。

カスタムフック内で組み込みフックを使うこともできます。デバウンスには`useState`と`useEffect`フックが必要なので、ファイルの先頭でインポートしてください。

```js
import { useState, useEffect } from "react";
```

次に、`useDebounce`関数を作成し、`value`と`delay`をパラメータとして受け取ります。`value`は待ちたいリソースで、`delay`は待つ時間です。一定時間待つので、`setTimeout`と`clearTimeout`関数が役立ちます。

```js
function useDebounce(value, delay) {
  const [debouncedValue, setDebouncedValue] = useState(value);

  useEffect(() => {
    const handler = setTimeout(() => {
      setDebouncedValue(value);
    }, delay);

    return () => {
      clearTimeout(handler);
    };
  }, [value, delay]);

  return debouncedValue;
}

export { useDebounce };
```

`debouncedValue`状態は遅延した値を保持し、指定したタイムアウト期間後にのみ更新されて返されます。

`useEffect`が本当の魔法が起きる場所です。前のレッスンで学んだように、タイマーの設定やクリアなどReactのレンダリングサイクル外で起こることは副作用なので、`useEffect`フックで処理すべきです。

ここでの`useEffect`フック内では、`setTimeout`を使って`debouncedValue`を設定します。そして、`clearTimeout`を使って前のタイムアウトをクリアするクリーンアップ関数を返します。これは`value`や`delay`が変わったとき、またはコンポーネントがアンマウントされたときに実行されます。

そしてこちらは、ユーザーが入力を止めてから1秒遅延して検索を実行するために`FootballerSearch`フックを使った`useDebounce`コンポーネントです。

```jsx
import { useState, useEffect } from "react";
import { useDebounce } from "./hooks/useDebounce";
import footballers from "./footballers";

const FootballerSearch = () => {
  const [query, setQuery] = useState("");
  const debouncedQuery = useDebounce(query, 1000); // Start searching 1 second after the user stops typing

  useEffect(() => {
    if (debouncedQuery) {
      const results = footballers.filter((footballer) =>
        footballer.toLowerCase().includes(debouncedQuery.toLowerCase()),
      );
      console.log("Search results:", results);
    } else {
      console.log("Search results: []");
    }
  }, [debouncedQuery]);

  return (
    <>
      <h1 style={{ textAlign: "center" }}>Footballer Search App</h1>
      <div style={{ textAlign: "center" }}>
        <input
          style={{ padding: "0.5rem", width: "30%" }}
          type="text"
          value={query}
          onChange={(e) => setQuery(e.target.value)}
          placeholder="Search for a footballer..."
        />
      </div>
    </>
  );
};

export default FootballerSearch;
```

ご覧の通り、`debouncedQuery`変数はクエリ状態（ユーザーの入力）と1,000ミリ秒、つまり1秒の遅延で`useDebounce`フックを初期化しています。検索自体は`useEffect`フック内で処理され、検索結果はコンソールにログ出力されます。
