# --description--

Der Browser verfügt über eingebaute Entwicklerwerkzeuge, mit denen Sie HTML, CSS und JavaScript debuggen können.

Diese sind jedoch nicht optimal, um Bugs in React-Apps zu finden und zu beheben. Deshalb hat das React-Team ein Tool namens „React Developer Tools“ (auch React DevTools genannt) entwickelt, mit dem Sie React-Apps inspizieren, debuggen und profilieren können.

React DevTools ist als Browsererweiterung für Chrome, Edge und Firefox verfügbar. Wenn Sie Chrome oder Edge verwenden, gehen Sie zum Chrome Web Store, suchen Sie nach „React Developer Tools“ und fügen Sie es Ihrem Browser hinzu.

Wenn Sie Firefox nutzen, gehen Sie zur Firefox-Add-ons-Seite, suchen Sie nach dem Tool und fügen Sie es Ihrem Browser hinzu.

Wenn Sie Safari verwenden, können Sie React DevTools über npm installieren, indem Sie `npm install -g react-devtools` oder `yarn global add react-devtools` ausführen.

Nachdem Sie React DevTools installiert und aktiviert haben, sollten Sie beim Öffnen einer React-App im Browser und anschließendem Öffnen der Entwicklerwerkzeuge Ihres Browsers zwei zusätzliche Tabs sehen: Komponenten und Profiler.

Der Tab Komponenten zeigt Ihnen jede Komponente in einer Baumansicht an. Damit können Sie:

- Die Komponenten-Hierarchie der App ansehen.
- Props, States und Kontextwerte in Echtzeit prüfen und ändern.
- Den Quellcode jeder ausgewählten Komponente ansehen.
- Die Komponentendaten in der Konsole protokollieren.
- Die DOM-Elemente der Komponente inspizieren.

Der Tab Profiler hingegen hilft Ihnen, die Leistung der Komponenten aufzuzeichnen und zu analysieren, damit Sie unnötige Re-Renders erkennen, Commit-Dauern einsehen und Optimierungsmöglichkeiten finden können.

Hier ist eine einfache App, die Ihnen zeigt, wie Sie Komponenten sowie deren Props und State inspizieren können. Das ist ähnlich wie der Code, den wir in einer vorherigen Lektion zum Prop Drilling verwendet haben:

```jsx
import { useState } from "react";

export default function App() {
  const greeting = "Hello, Prop Drilling!";
  const response = "I'm not here to play!";

  return <Parent greeting={greeting} response={response} />;
}

const Parent = ({ greeting, response }) => {
  return <Child greeting={greeting} response={response} />;
};

const Child = ({ greeting, response }) => {
  return <Grandchild greeting={greeting} response={response} />;
};

const Grandchild = ({ greeting, response }) => {
  const [count, setCount] = useState(0);

  return (
    <>
      <h1>{greeting}</h1>
      <h2>{response}</h2>

      <button onClick={() => setCount(count + 1)}>Increase Count</button>
      <h2>Count: {count}</h2>
      <button onClick={() => setCount(count - 1)}>Decrease Count</button>
    </>
  );
};
```

Wenn Sie im Komponenten-Tab von React DevTools nachsehen, sehen Sie die Baumansicht der Komponenten. Die `App`-Komponente steht ganz oben, gefolgt von den `Parent`-, `Child`- und `Grandchild`-Komponenten.

Wenn Sie eine dieser Komponenten auswählen, können Sie die Props und den State darin sehen. Wenn Sie die `Parent`-Komponente auswählen, sehen Sie die `greeting`- und `response`-Props, die `Hello, Prop Drilling!` bzw. `I'm not here to play!` sind.

Sie können die Props sehen und den State in Echtzeit aktualisieren und bei Bedarf ändern. Zum Beispiel können Sie die `Grandchild`-Komponente auswählen und die `greeting`-Prop von `Hello, Prop Drilling!` auf `Hello, Welcome to Prop Drilling!` ändern und die Änderung sofort auf der Seite sehen.

Um Daten einer Komponente in der Konsole zu protokollieren, inspizieren Sie die passenden DOM-Elemente und sehen Sie sich den Quellcode der Komponente an. Die Symbole oben rechts ermöglichen Ihnen das. Wenn Sie die `Grandchild`-Komponente auswählen und auf die `Log the component data to the console`-Schaltfläche klicken, werden die Props, der State, Hooks, Knoten und weitere Daten in der Konsole protokolliert.

Ein häufiger Bug, der in React auftreten kann, heißt Props-Mismatch.

Zum Beispiel geben Sie für die `Child`-Komponente versehentlich `reply` als Prop anstelle von `response`:

```jsx
const Child = ({ greeting, response }) => {
  return <Grandchild greeting={greeting} reply={response} />;
};
```

Denken Sie daran, dass `Grandchild` eine `response`-Prop erwartet. Da die Komponente eine andere Prop erhält, kann sie diesen Text nicht auf der Seite anzeigen und fügt stattdessen ein leeres `h2` zum DOM hinzu. Stattdessen sehen Sie nur das `h1`-Element mit dem Text `Hello, Prop Drilling!` zusammen mit den anderen Buttons und Texten auf der Seite. Das leere `h2`-Element ist zwar vorhanden, aber da es leer ist, sehen Sie es nur, wenn Sie das DOM inspizieren.

Um das zu beheben, können Sie die Prop-Weitergabe von der `Parent`-Komponente bis zur `Child`-Komponente inspizieren und den Prop-Namen direkt bearbeiten. Wenn Sie zum Komponenten-Tab gehen, die `Child`-Komponente auswählen und die `reply`-Prop in `response` ändern, sehen Sie das `h2`-Element auf der Seite mit dem Text `I'm not here to play!`.
