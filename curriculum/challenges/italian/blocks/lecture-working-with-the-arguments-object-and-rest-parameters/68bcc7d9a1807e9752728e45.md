# --interactive--

Nella lezione precedente, hai imparato a lavorare con l'oggetto `arguments` che è un oggetto simile a un array contenente i valori degli argomenti passati alla funzione.

:::interactive_editor

```js
function logArgs() {
  for (const arg of arguments) {
    console.log(arg);
  }
}

logArgs(1, 2, 3);
// result:
// 1
// 2
// 3
```

:::

Anche se questo è un modo valido per consultare e lavorare con un insieme variabile di argomenti di una funzione, le applicazioni JavaScript moderne useranno invece la sintassi del parametro rest.

Ecco un esempio aggiornato che usa la sintassi del parametro rest invece dell'oggetto `arguments`:

:::interactive_editor

```js
function logArgs(...args) {
  for (const arg of args) {
    console.log(arg);
  }
}

logArgs(1, 2, 3);
// result:
// 1
// 2
// 3
```

:::

Questo esempio aggiornato non fa più riferimento direttamente all'oggetto `arguments`. Invece, l'ultimo parametro della definizione della funzione ha tre puntini davanti. Questo fa sì che questo parametro rest venga inserito in un oggetto `Array`. Puoi chiamare questo parametro rest come vuoi. Assicurati solo che sia l'ultimo parametro nella definizione della funzione, così:

```js
function exampleFunction(a, b, ...restOfArgs) {
  // some code here
}

function anotherFunction(x, y, ...theArgs) {
  // some code here
}
```

Ci sono alcune restrizioni in più quando si usa la sintassi del parametro rest. Una restrizione è che le definizioni di funzione possono avere solo un parametro rest. Quindi l'esempio seguente sarebbe considerato non valido.

```js
// Won't work

function badFunction(...args, ...moreArgs) {
  // some code here
}
```

La restrizione successiva è che non sono ammessi virgole finali dopo il parametro rest: `function exampleFunction(a, b, ...restOfArgs, )`.

Un'altra restrizione è che il parametro rest non può avere un valore predefinito. Altrimenti verrà lanciato un `SyntaxError`.

```js
function badFunction(...args = [1,2]){
  // some code here
}
```

Quali sono quindi alcune differenze tra l'oggetto `arguments` e i parametri `rest`?

Una differenza chiave è che l'oggetto `arguments` non è un vero array quindi non supporta metodi come `includes`, `pop` e `push`. Ma il parametro rest è un'istanza di `Array`. Quindi puoi usare metodi validi di array incorporati senza doverlo prima convertire in un vero array.

:::interactive_editor

```js
function hasCat(...args) {
  return args.includes("cat");
}

console.log(hasCat("dog", "chicken", "cat")); // true
console.log(hasCat("dog", "chicken", "horse")); // false
```

:::

Qual è una differenza chiave tra l'oggetto `arguments` e i parametri rest?

## --answers--
