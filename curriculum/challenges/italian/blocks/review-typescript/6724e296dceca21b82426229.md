## Cos’è TypeScript

- **JavaScript**: JavaScript è un linguaggio a tipizzazione dinamica. Questo significa che le variabili possono ricevere qualsiasi valore a runtime. La sfida di un linguaggio a tipizzazione dinamica è che la mancanza di sicurezza dei tipi può introdurre errori.

Per esempio, anche se la tua funzione JavaScript si aspetta un array, puoi comunque chiamarla con un numero:

```javascript
const getRandomValue = (array) => {
  return array[Math.floor(Math.random() * array.length)];
}

console.log(getRandomValue(10));
```

L’output `console` per l’esempio sopra sarà `undefined`.

- **TypeScript**: TypeScript estende il linguaggio JavaScript includendo la tipizzazione statica, che aiuta a intercettare errori causati da incongruenze di tipo prima di eseguire il codice.

Per esempio, puoi definire un tipo per il parametro `array` come segue:

```typescript
const getRandomValue = (array: string[]) => {
  return array[Math.floor(Math.random() * array.length)];
}
```

Questa definizione di tipo dice a TypeScript che il parametro `array` deve essere un array di stringhe. Poi, quando chiami `getRandomValue` e gli passi un numero, ottieni un errore chiamato errore di compilazione.

- **Compilatore**: Prima devi compilare il codice TypeScript in JavaScript normale. Quando esegui il compilatore, TypeScript valuta il tuo codice e lancia un errore per qualsiasi problema in cui i tipi non corrispondono.

## Tipi di dati in TypeScript

- **Tipi di dati primitivi in TypeScript**: Per i tipi di dati primitivi `string`, `null`, `undefined`, `number`, `boolean` e `bigint`, TypeScript offre parole chiave di tipo corrispondenti.

```typescript
let str: string = "Naomi";
let num: number = 42;
let bool: boolean = true;
let nope: null = null;
let nada: undefined = undefined;
```

- **Array**: Puoi definire un array di tipo specifico con due sintassi diverse.

```typescript
const arrOne: string[] = ["Naomi"];
const arrTwo: Array<string> = ["Camperchan"];
```

- **Oggetti**: Puoi definire la struttura esatta di un oggetto.

```typescript
const obj: { a: string, b: number } = { a: "Naomi", b: 42 };
```

Se vuoi un oggetto con chiavi qualsiasi, ma dove tutti i valori devono essere stringhe, ci sono due modi per definirlo:

```typescript
const objOne: Record<string, string> = {};
const objTwo: { [key: string]: string } = {};
```

- **Altri tipi utili in TypeScript**:
  - **`any`**: `any` indica che un valore può avere qualsiasi tipo. Dice al compilatore di non preoccuparsi del tipo di quella variabile.
  - **`unknown`**: `unknown` dice a TypeScript che ti interessa il tipo del valore, ma in realtà non sai quale sia. `unknown` è generalmente preferito rispetto a `any`.
  - **`void`**: Questo è un tipo speciale che userai tipicamente solo quando definisci funzioni. Le funzioni che non hanno un valore di ritorno usano un tipo di ritorno `void`.
  - **`never`**: Rappresenta un tipo che non esisterà mai.

- **Parola chiave `type`**: Questa parola chiave è come `const`, ma invece di dichiarare una variabile, puoi dichiarare un tipo.

È utile per dichiarare tipi personalizzati come tipi unione o tipi che includono solo valori specifici:

```typescript
type stringOrNumber = string | number;
type bot = "camperchan" | "camperbot" | "naomi";
```

- **`interface`**: Le interfacce sono come classi per i tipi. Possono implementare o estendere altre interfacce, sono specificamente tipi oggetto e sono generalmente preferite a meno che non ti serva una funzionalità specifica offerta da una dichiarazione `type`.

```typescript
interface wowie {
  zowie: boolean;
  method: () => void;
}
```

- **Definire il tipo di ritorno**: Puoi anche definire il tipo di ritorno della funzione.

L’esempio sotto definisce il valore di ritorno come stringa. Se provi a restituire altro, TypeScript darà un errore di compilazione.

```typescript
const getRandomValue = (array: string[]): string => {
  return array[Math.floor(Math.random() * array.length)];
}
```

## Generici

- **Definire un tipo generico**: Puoi definire un tipo generico e farvi riferimento nella tua funzione. Puoi pensarlo come un parametro speciale che fornisci a una funzione per controllare il comportamento della definizione del tipo della funzione.

Ecco un esempio di definizione di un tipo generico per una funzione:

```typescript
const getRandomValue = <T>(array: T[]): T => {
  return array[Math.floor(Math.random() * array.length)];
}
const val = getRandomValue([1, 2, 4])
```

La sintassi `<T>` dice a TypeScript che stai definendo un tipo generico `T` per la funzione. `T` è un nome comune per i tipi generici, ma puoi usare qualsiasi nome.

Poi dici a TypeScript che il parametro `array` è un array di valori che corrispondono al tipo generico, e che il valore restituito è un singolo elemento di quel tipo.

- **Specificare l’argomento di tipo nella chiamata di funzione**: Puoi passare un argomento di tipo a una chiamata di funzione usando parentesi angolari tra il nome della funzione e i suoi parametri.

Ecco un esempio di passaggio di un argomento di tipo a una chiamata di funzione:

```typescript
const email = document.querySelector<HTMLInputElement>("#email");
console.log(email.value);
```

Questo dice a TypeScript che l’elemento che ti aspetti di trovare sarà un elemento input.

## Restringimento del tipo

- **Restringimento tramite veridicità**: Nell’esempio sotto, ottieni un errore di compilazione tentando di consultare la proprietà `value` di `email` perché `email` potrebbe essere `null`.

```typescript
const email = document.querySelector<HTMLInputElement>("#email");
console.log(email.value);
```

Puoi usare un’istruzione condizionale per confermare che `email` è veritiero prima di consultare la proprietà:

```typescript
const email = document.querySelector<HTMLInputElement>("#email");
if (email) {
  console.log(email.value);
}
```

I controlli di veridicità possono funzionare anche al contrario:

```typescript
const email = document.querySelector<HTMLInputElement>("#email");
if (!email) {
  throw new ReferenceError("Could not find email element!")
}
console.log(email.value);
```

Lanciare un errore termina l’esecuzione logica di questo codice, il che significa che quando arrivi alla chiamata `console.log()`, TypeScript sa che `email` non può essere `null`.

- **Concatenamento opzionale**: Il concatenamento opzionale `?.` è anche una forma di restringimento del tipo, sotto lo stesso presupposto che l’accesso alla proprietà non può avvenire se il valore `email` è `null`.

```typescript
const email = document.querySelector<HTMLInputElement>("#email");
console.log(email?.value);
```

- **Operatore `typeof`**: Puoi usare un condizionale per controllare il tipo della variabile usando l’operatore `typeof`.

```typescript
const myVal = Math.random() > 0.5 ? 222 : "222";
if (typeof myVal === "number") {
  console.log(myVal / 10);
}
```

- **Parola chiave `instanceof`**: Se l’oggetto proviene da una classe, puoi usare la parola chiave `instanceof` per restringere il tipo.

```typescript
const email = document.querySelector("#email");

if (email instanceof HTMLInputElement) {
    console.log(email.value);
}
```

- **Casting**: Quando TypeScript non riesce a determinare automaticamente il tipo di un valore, come il risultato del metodo `request.json()` nell’esempio sotto, ottieni un errore di compilazione. Un modo per risolvere è fare il casting del tipo, ma così si indebolisce la capacità di TypeScript di intercettare potenziali errori.

```typescript
interface User {
    name: string;
    age: number;
}

const printAge = (user: User) => 
  console.log(`${user.name} is ${user.age} years old!`)

const request = await fetch("url")
const myUser = await request.json() as User;
printAge(myUser);
```

- **Type guard**: Invece di fare il casting del tipo, puoi scrivere un type guard:

```typescript
interface User {
    name: string;
    age: number;
}

const isValidUser = (user: unknown): user is User => {
  return !!user && 
    typeof user === "object" &&
    "name" in user &&
    "age" in user;
}
```

La sintassi `user is User` indica che la tua funzione restituisce un valore booleano, che quando è true significa che il valore `user` soddisfa l’interfaccia `User`.

## File `tsconfig`

- **`tsconfig.json`**: Le impostazioni del compilatore di TypeScript vivono in un file `tsconfig.json` nella directory principale del tuo progetto.

```json
{
  "compilerOptions": {
    "rootDir": "./src",
    "outDir": "./prod",
    "lib": ["ES2023"],
    "target": "ES2023",
    "module": "ES2022",
    "moduleResolution": "Node",
    "esModuleInterop": true,
    "skipLibCheck": true,
    "strict": true
  },
  "exclude": ["test/"]
}
```

Ecco le descrizioni delle opzioni del compilatore usate nell’esempio sopra:

- **`compilerOptions`**: La proprietà `compilerOptions` è dove controlli come si comporta il compilatore TypeScript.
- **`rootDir` e `outDir`**: `rootDir` e `outDir` dicono a TypeScript quale directory contiene i tuoi file sorgente e quale directory deve contenere il codice JavaScript trascritto.
- **`lib`**: La proprietà `lib` determina quali definizioni di tipo usa il compilatore e ti permette di includere supporto per specifiche release ES, il DOM e altro.
- **`module` e `moduleResolution`**: `module` e `moduleResolution` lavorano insieme per gestire come il tuo pacchetto usa i moduli — sia CommonJS che ECMAScript.
- **`esModuleInterop`**: `esModuleInterop` permette una migliore interoperabilità tra moduli CommonJS e ES creando automaticamente oggetti namespace per gli import.
- **`skipLibCheck`**: L’opzione `skipLibCheck` salta la validazione dei file `.d.ts` che non sono referenziati da import nel tuo codice.
- **`strict`**: Il flag `strict` abilita diversi controlli, come assicurare la corretta gestione dei tipi nullable e avvisare quando TypeScript ricade su `any`.
- **`exclude`**: La proprietà di livello superiore `exclude` dice al compilatore di ignorare questi file TypeScript durante la compilazione.
