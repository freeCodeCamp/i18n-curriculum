# --interactive--

Побітові оператори в JavaScript — це спеціальні оператори, які працюють із двійковими поданнями чисел. Щоб зрозуміти побітові оператори, спочатку потрібно засвоїти поняття бітів і двійкових чисел. У комп’ютерній науці біт — це найпростіша одиниця інформації. Він може мати лише два значення: `0` або `1`. Двійкова система — це система числення, яка використовує лише ці дві цифри для представлення всіх чисел.

Наприклад, двійкове подання десяткового числа `10` — це `1010`. У цій системі кожна цифра представляє ступінь `2`, починаючи з правої цифри і збільшуючись при русі вліво.

У таблиці вище перший рядок показує двійкове число `1010`, другий рядок показує ступінь `2`, який представляє кожна двійкова позиція, а третій рядок показує результат кожного множення. Якщо додати всі значення в третьому рядку, отримаємо `10`.

Тепер перейдемо до побітових операторів. Ці оператори виконують операції над двійковими поданнями чисел. JavaScript надає кілька побітових операторів, зокрема AND (`&`), OR (`|`), XOR (`^`), NOT (`~`), зсув вліво (`<<`) і зсув вправо (`>>`).

Побітовий оператор AND (`&`) повертає `1` у кожній позиції біта, для якої відповідні біти обох операндів є `1`. Ось приклад:

:::interactive_editor

```js
let a = 5;  // Binary: 101
let b = 3;  // Binary: 011
console.log(a & b);  // 1 (Binary: 001)
```

:::

У цьому прикладі ми виконуємо побітову операцію AND над `5` (`101` у двійковому вигляді) і `3` (`011` у двійковому вигляді). Результат — `1` (`001` у двійковому вигляді), тому що лише найправіший біт є `1` в обох числах.

Побітовий оператор OR (`|`) повертає `1` у кожній позиції біта, для якої відповідні біти одного або обох операндів є `1`. Наприклад:

:::interactive_editor

```js
let a = 5;  // Binary: 101
let b = 3;  // Binary: 011
console.log(a | b);  // 7 (Binary: 111)
```

:::

Тут результат — `7` (`111` у двійковому вигляді), тому що принаймні один із бітів у кожній позиції є `1`.

Побітовий оператор XOR (`^`) повертає `1` у кожній позиції біта, для якої відповідні біти одного, але не обох, операндів є `1`. Наприклад:

:::interactive_editor

```js
let a = 5;  // Binary: 101
let b = 3;  // Binary: 011
console.log(a ^ b);  // 6 (Binary: 110)
```

:::

Результат — `6` (`110` у двійковому вигляді), тому що перший і другий біти справа відрізняються в обох числах.

Побітовий оператор NOT (`~`) інвертує всі біти свого операнда. Наприклад:

:::interactive_editor

```js
let a = 5;  // Binary: 101
console.log(~a);  // -6
```

:::

Це може здатися дивним, але це через те, як від’ємні числа представлені у двійковому вигляді за допомогою доповнення до двійки.

Оператор зсуву вліво (`<<`) зсуває всі біти вліво на вказану кількість позицій. Наприклад:

:::interactive_editor

```js
let a = 5;  // Binary: 101
console.log(a << 1);  // 10 (Binary: 1010)
```

:::

Тут усі біти зсунуті на одну позицію вліво, що фактично помножує число на `2`.

Оператор зсуву вправо (`>>`) зсуває всі біти вправо. Наприклад:

:::interactive_editor

```js
let a = 5;  // Binary: 101
console.log(a >> 1);  // 2 (Binary: 10)
```

:::

Тут усі біти зсунуті на одну позицію вправо, що фактично ділить число на `2` із округленням вниз.

Побітові оператори часто використовують у низькорівневому програмуванні та криптографії. Хоча вони можуть не бути такими поширеними у повсякденному програмуванні на JavaScript, розуміння їх може бути корисним для певних спеціалізованих завдань і поглибить ваше розуміння того, як комп’ютери працюють на фундаментальному рівні.
