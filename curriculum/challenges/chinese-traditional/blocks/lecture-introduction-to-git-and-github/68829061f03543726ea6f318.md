# --description--

我們已經稍微談過 Git 和 GitHub，所以現在我們可以開始將這些整合在一起。為了做到這一點，你需要一個 repository。但什麼是 repository 呢？你可以將 repository 視為一個專案的容器－如果你正在開發一個應用程式，你會將該應用程式的檔案集中保存在 repository 中。

儲存庫可以存在你電腦上的區域，也可以存在像 GitHub 這樣的遠端服務上。要創建儲存庫，你可以從區域或遠端開始——結果會是一樣的，因此採用最適合你工作流程的方法。若你的工作流程是從遠端儲存庫開始，你可以透過 GitHub 網站直接創建一個，請依照以下說明操作：

造訪 <a href="https://github.com" target="_blank">GitHub 的網站</a>，確保你已登入，然後點擊導覽列右上角的加號圖示。你會看到相當多的選項，可是你關心的是「New Repository」。選擇該選項以開啟建立新儲存庫的使用者介面。

你會看到的第一個選項是選擇一個模板。你可能還沒有設定這些模板，但模板是一種特殊類型的儲存庫，你可以用來作為新儲存庫的跳板。這些模板有助於在你所有的專案中包含文件或問題模板。

你的下一個選項是選擇一個擁有者，你可以選擇你的使用者帳戶或任何你擁有或有存取權的組織。然後你可以為儲存庫命名－一般來說，你會想使用一個簡潔描述你正在建置的專案的名稱。作為一般最佳實務，你應避免在儲存庫名稱中使用空格。GitHub 會自動將它們取代為連字號。

接著，你可以選用為你的描述添加一段文字，該文字將顯示在儲存庫的首頁。

你也可以選擇儲存庫是公開還是私有。公開儲存庫任何人都可以檢視和下載。私有儲存庫只有你和你明確授權存取的人才能存取。

最後，如果你沒有選擇模板，你將會看到自動在儲存庫中產生檔案的選項。README 檔案會出現在儲存庫的首頁，是提供你的專案文件和描述的絕佳方式。`.gitignore` 檔案允許你指定 Git 將「忽略」的檔案－這對於像是軟體包依賴、環境檔案以及其他你不想被存檔的項目非常有用。GitHub 會提供此檔案的模板，排除指定語言或執行期的常見項目。授權條款是一份法律文件，說明誰可以使用你的軟體，以及他們可以對軟體做什麼。

當你將設定組態完成到你喜歡的狀態時，你可以點擊「Create Repository」按鈕來完成此處理程序。你將自動被重新導向到你的新儲存庫的登陸頁面！

從這裡，你可以點擊「Code」按鈕以取得將你的儲存庫複製到你的電腦的選項。

你可以忽略本課程中的「Codespaces」和「Copilot」分頁。相反地，我們將專注於「Local」分頁中的三個選項。前兩個，HTTPS 和 SSH，使用方式相同。你使用 `git clone` 指令搭配方框中提供的 URL，將遠端儲存庫複製到你的電腦。這兩者之間的差異在於連線和驗證方法。

要複製公開的儲存庫，實際上你不需要進行驗證。但要複製私人儲存庫，或是推送到公開或私人儲存庫，你必須先通過驗證。使用 HTTPS 遠端 URL 可透過你的 GitHub 使用者名稱和密碼來驗證你。然而，因為安全性考量，這已不再被支援，你必須改用使用者名稱和存取語彙單元。即使如此，這種方法仍比使用 SSH 不安全，SSH 使用你的私鑰和公鑰。這還有一個額外利益，就是自動幫你登入，而不必每次都提供語彙單元。會有一個獨立課程深入探討 SSH 及其運作方式，但你也可以閱讀 GitHub 文件了解如何設定 SSH。

要複製這個儲存庫，你可以在終端機中執行這個指令：

```sh
git clone git@github.com:your-username/your-repository-name
```

它會在你的電腦上創建一個以你的儲存庫名稱命名的資料夾，並包含你儲存庫中的所有檔案。從那裡你可以使用 `cd`（切換目錄）到你複製的儲存庫：

```sh
cd name-of-cloned-repo
```

此外，你的 repo 的區域複本將會有一個遠端指向 GitHub 上的複本。若要檢查你的遠端，可以執行以下指令：

```sh
git remote -v
```

另一個複製你的儲存庫的選項是使用 GitHub CLI。這個工具用於在不離開命令列的情況下執行 GitHub 特定的任務。如果你尚未安裝它，可以從 GitHub 的文件中取得安裝說明－但你應該能在系統的軟體包管理員中找到它。

要使用 CLI 工具複製一個儲存庫，我會使用：

```sh
gh repo clone your-username/your-repository-name
```

這些指令的最終結果是一樣的：你將會有一份你的儲存庫的區域複本。

你也可以使用 GitHub CLI 來創建一個儲存庫：

```sh
gh repo create
```

如果你沒有傳入任何引數，CLI 將會提供你一個互動式精靈來設定你的儲存庫。這個精靈會提供你與網頁使用者介面相同的所有選項。使用這個工具，你還會得到立即複製新儲存庫的選項，這樣你就不必經歷先前說明的手動步驟。

你現在已經對儲存庫的運作方式有基本的了解，以及如何創建一個儲存庫以開始正確地版本控制你的工作。在未來的課程中，你將學習如何將現有的區域的儲存庫推送到 GitHub，以及如何將變更推送到遠端儲存庫。
