# --interactive--

في الدروس السابقة، تعلمت عن البرمجة غير المتزامنة التي تسمح بتشغيل شفرة أخرى أثناء انتظارنا لإكمال مهام تستغرق وقتًا طويلاً، مثل جلب البيانات من خادم، أو قراءة البيانات من ملف، وهكذا.

`async`/`await`، المبنيان على الوعود، يسهلان كتابة وقراءة الشفرة غير المتزامنة. عندما تضع الكلمة المفتاحية `async` قبل دالة، فهذا يعني أن الدالة سترجع دائمًا `Promise`. يمكن استخدام الكلمة المفتاحية `await`، التي تسمح لك بالانتظار حتى يُحل `Promise` قبل الانتقال إلى السطر التالي من الشفرة، فقط داخل الدوال غير المتزامنة أو في المستويات العليا من وحدات البرمجة. إليك مثالًا يوضح كيف يعمل `async`/`await`:

:::interactive_editor

```js
async function delayedGreeting(name) {
  console.log("A Messenger entered the chat...");
  await new Promise(resolve => setTimeout(resolve, 2000));
  console.log(`Hello, ${name}!`);
}

delayedGreeting("Alice");
console.log("First Printed Message!");
```

:::

في هذه الشفرة، نُعرّف دالة `async` تُسمى `delayedGreeting`. داخل هذه الدالة، نستخدم `await` لإيقاف التنفيذ لمدة ثانيتين. بعد التأخير، تطبع تحية.

عندما نستدعي هذه الدالة، سترى ظهور `First Printed Message!` قبل التحية. هذا لأن الدالة غير متزامنة - فهي لا تمنع بقية الشفرة من التنفيذ.

واحدة من أكبر مزايا `async`/`await` هي التعامل مع الأخطاء. مع الوعود، كنا غالبًا نستخدم طريقة `.catch()` لمعالجة الأخطاء. مع `async`/`await`، يمكننا استخدام كتل `try`/`catch`. إليك مثالًا:

```js
async function fetchUserData() {
  try {
    let response = await fetch(`https://api.example.com/users`);
    let userData = await response.json();
    console.log(userData);
  } catch (error) {
    console.log("Error fetching user data:", error);
  }
}

fetchUserData();
```

في هذا المثال، نستخدم `async`/`await` لجلب بيانات المستخدم من واجهة برمجة التطبيقات. تُستخدم الكلمة المفتاحية `await` مرتين: مرة للانتظار حتى تكتمل عملية الجلب، ومرة أخرى للانتظار حتى ينتهي تحليل JSON. إذا حدث أي خطأ أثناء هذه العملية، سيتم التقاطه في كتلة `catch`.
