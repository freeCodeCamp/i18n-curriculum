## 알고리즘과 빅오 표기법

- **알고리즘**: 문제를 해결하거나 작업을 수행하기 위한 명확한 지침 집합입니다. 알고리즘은 유한한 단계 내에 종료되어야 하며 각 단계는 정확하고 명확해야 합니다.

- **빅오 표기법**: 입력 크기가 증가할 때 알고리즘의 최악의 성능 또는 성장률을 설명합니다. 상수 요소와 하위 차수 항은 무시하고 입력 크기에 따른 자원 사용량의 증가에 집중합니다.

### 일반적인 시간 복잡도

- **O(1) - 상수 시간**: 입력 크기와 상관없이 알고리즘이 동일한 시간 내에 실행됩니다.

```python
def check_even_or_odd(number):
    if number % 2 == 0:
        return 'Even'
    else:
        return 'Odd'
```

- **O(log n) - 로그 시간**: 입력이 증가해도 시간이 천천히 증가합니다. 문제 크기를 일정 비율로 반복해서 줄이는 알고리즘(예: 이진 탐색)에서 흔히 나타납니다.

- **O(n) - 선형 시간**: 실행 시간이 입력 크기에 비례하여 증가합니다.

```python
for grade in grades:
    print(grade)
```

- **O(n log n) - 로그 선형 시간**: 병합 정렬과 퀵 정렬 같은 효율적인 정렬 알고리즘의 일반적인 시간 복잡도입니다.

- **O(n²) - 이차 시간**: 실행 시간이 제곱에 비례하여 증가합니다. 중첩된 반복문에서 자주 나타납니다.

```python
for i in range(n):
    for j in range(n):
        print("Hello, World!")
```

### 공간 복잡도

- **O(1) - 상수 공간**: 입력 크기와 상관없이 동일한 메모리를 사용합니다.
- **O(n) - 선형 공간**: 메모리 사용량이 입력 크기에 비례하여 증가합니다.
- **O(n²) - 이차 공간**: 메모리 사용량이 입력 크기의 제곱에 비례하여 증가합니다.

## 문제 해결 기법

- **문제 이해하기**: 문제 설명을 여러 번 읽으세요. 입력, 예상 출력, 입력을 출력으로 변환하는 방법을 파악하세요.

- **의사 코드**: 언어에 독립적인 알고리즘 논리의 고수준 설명입니다. 일반적인 문장과 `IF`, `ELSE`, `FOR`, `WHILE` 같은 프로그래밍 구성 요소를 혼합해 사용합니다.

```md
GET original_string
SET reversed_string = ""
FOR EACH character IN original_string:
  ADD character TO THE BEGINNING OF reversed_string
DISPLAY reversed_string
```

- **경계 사례**: 알고리즘이 처리해야 하는 범위의 경계에 위치한 특정하고 유효한 입력입니다. 항상 경계 사례를 고려하고 테스트하세요.

## 배열

- **정적 배열**: 초기화 시 크기가 고정됩니다. 요소들은 인접한 메모리 위치에 저장됩니다. 프로그램 실행 중 크기를 변경할 수 없습니다.

- **동적 배열**: 프로그램 실행 중 자동으로 크기가 늘어나거나 줄어들 수 있습니다. 필요할 때 더 큰 배열로 자동 복사를 통해 크기 조절을 처리합니다.

### 파이썬 리스트 (동적 배열)

```python
numbers = [3, 4, 5, 6]

# Access elements
numbers[0]  # 3

# Update elements
numbers[2] = 16

# Add elements
numbers.append(7)
numbers.insert(3, 15)  # Insert at specific index

# Remove elements
numbers.pop(2)  # Remove at specific index
numbers.pop()   # Remove last element
```

### 동적 배열의 시간 복잡도

- **접근**: O(1)
- **끝에 삽입**: 평균 O(1), 크기 조절 시 O(n)
- **중간에 삽입**: O(n)
- **삭제**: 중간에서 O(n), 끝에서 O(1)

## 스택

- **스택**: 후입선출(LIFO) 자료 구조입니다. 요소는 오직 맨 위에서 추가되고 제거됩니다.

- **푸시 연산**: 스택 맨 위에 요소를 추가합니다. 시간 복잡도: O(1).

- **팝 연산**: 스택 맨 위에서 요소를 제거합니다. 시간 복잡도: O(1).

```python
# Using Python list as stack
stack = []

# Push operations
stack.append(1)
stack.append(2)
stack.append(3)

# Pop operations
top_element = stack.pop()  # Returns 3
```

## 큐

- **큐**: 선입선출(FIFO) 자료 구조입니다. 요소는 뒤에서 추가되고 앞에서 제거됩니다.

- **인큐 연산**: 큐 뒤에 요소를 추가합니다. 시간 복잡도: O(1).

- **디큐 연산**: 큐 앞에서 요소를 제거합니다. 시간 복잡도: O(1).

```python
from collections import deque

# Using deque for efficient queue operations
queue = deque()

# Enqueue operations
queue.append(1)
queue.append(2)
queue.append(3)

# Dequeue operations
first_element = queue.popleft()  # Returns 1
```

## 연결 리스트

- **연결 리스트**: 각 노드가 데이터와 다음 노드에 대한 참조를 포함하는 선형 자료 구조입니다. 노드들이 체인처럼 연결되어 있습니다.

### 단일 연결 리스트

- **구조**: 각 노드는 데이터와 다음 노드에 대한 하나의 참조를 가집니다.
- **순회**: 머리에서 꼬리 방향으로만 이동할 수 있습니다.
- **머리 노드**: 리스트의 첫 번째 노드로 보통 유일하게 직접 접근 가능한 노드입니다.
- **꼬리 노드**: 리스트의 마지막 노드로 `None`를 가리킵니다.

### 연산과 시간 복잡도

- **처음에 삽입**: O(1)
- **끝에 삽입**: O(n) - 끝까지 순회해야 함
- **중간에 삽입**: O(n) - 위치까지 순회해야 함
- **처음에서 삭제**: O(1)
- **끝에서 삭제**: O(n) - 이전 노드를 찾기 위해 순회해야 함
- **중간에서 삭제**: O(n) - 노드를 찾기 위해 순회해야 함

### 이중 연결 리스트

- **구조**: 각 노드가 데이터와 두 개의 참조(다음 노드와 이전 노드)를 가집니다.
- **순회**: 양방향으로 이동할 수 있습니다.
- **메모리**: 추가 참조 때문에 단일 연결 리스트보다 더 많은 메모리를 사용합니다.

## 해시 맵과 세트

### 맵과 해시 맵

- **맵 (추상 자료형)**: 키-값 쌍 컬렉션을 관리합니다. 모든 키는 고유해야 하지만 값은 중복될 수 있습니다.

- **해시 맵**: 해싱 기법을 사용해 맵 ADT를 구체적으로 구현한 것입니다. 해시 함수로 키의 해시 값을 생성하고, 이 값이 기본 배열 내 저장 위치를 결정합니다.

### 파이썬 딕셔너리 (해시 맵)

```python
# Creating dictionaries
my_dictionary = {
    "A": 1,
    "B": 2, 
    "C": 3
}

# Alternative creation
my_dictionary = dict(A=1, B=2, C=3)

# Access and modify
value = my_dictionary["A"]  # 1
my_dictionary["A"] = 4      # Update value
del my_dictionary["A"]      # Remove key-value pair

# Check membership
"C" in my_dictionary

# Get keys, values, items
my_dictionary.keys()
my_dictionary.values()
my_dictionary.items()
```

### 해시 맵의 시간 복잡도

- **평균 경우**: 삽입, 조회, 삭제 모두 O(1)
- **최악 경우**: 해시 충돌이 많을 때 O(n)

### 세트

- **세트**: 중복 없는 고유 요소들의 무순서 컬렉션입니다. 중복을 허용하지 않고 특정 순서를 유지하지 않습니다.

- **불변 요소만 허용**: 세트는 숫자, 문자열, 튜플 같은 불변 데이터 타입만 포함할 수 있습니다. 해시 값이 변하지 않아야 하기 때문입니다.

```python
# Creating sets
numbers = {1, 2, 3, 4}
empty_set = set()  # Must use set(), not {}

# Add and remove elements
numbers.add(5)
numbers.remove(4)      # Raises KeyError if not found
numbers.discard(4)     # No error if not found

# Set operations
set_a = {1, 2, 3, 4}
set_b = {2, 3, 4, 5, 6}

# Union, intersection, difference, symmetric difference
set_a.union(set_b)                    # or set_a | set_b
set_a.intersection(set_b)             # or set_a & set_b
set_a.difference(set_b)               # or set_a - set_b
set_a.symmetric_difference(set_b)     # or set_a ^ set_b

# Subset and superset checks
set_a.issubset(set_b)
set_a.issuperset(set_b)
set_a.isdisjoint(set_b)

# Membership testing
5 in numbers
```

### 세트의 시간 복잡도

- **평균 경우**: 추가, 제거, 멤버십 테스트 모두 O(1)
- **최악 경우**: 해시 충돌로 인해 O(n)

## 해시 충돌

- **해시 충돌**: 서로 다른 두 키가 동일한 해시 값을 생성할 때 발생합니다.

- **충돌 해결 전략**:
  - **체이닝**: 각 배열 인덱스가 동일 해시 값을 가진 모든 요소를 저장하는 연결 리스트를 가리킵니다.
  - **오픈 어드레싱**: 미리 정의된 수열을 사용해 다음 사용 가능한 인덱스를 탐색합니다.

## 각 자료 구조 사용 시기

- **리스트**: 순서가 있고 인덱스로 접근해야 하며 크기를 미리 모를 때
- **스택**: 후입선출 작업(실행 취소 기능, 표현식 평가, 백트래킹)에 적합
- **큐**: 선입선출 작업(작업 스케줄링, 너비 우선 탐색)에 적합
- **연결 리스트**: 처음에 빈번한 삽입/삭제가 필요하고 크기를 모를 때, 임의 접근이 필요 없을 때
- **해시 맵**: 빠른 키-값 조회, 발생 횟수 세기, 캐싱에 적합
- **세트**: 고유성 검사, 수학적 집합 연산, 중복 제거에 적합
