# --description--

设置自定义认证需要编写大量手动逻辑。你必须处理读取请求头、提取 cookie、验证签名以及管理误差。

虽然自己构建这些对学习很有帮助，但为每个生产应用都这样做既耗时又容易出现安全疏漏。这就是开发者使用 Passport.js 的原因。

Passport.js 是 Node.js 的认证中间件。它作为一个有组织的框架，标准化了你认证请求的方式。它处理底层的认证细节，让你专注于应用的业务逻辑。

Passport.js 的核心理念是每个应用需要不同的登录方式，但底层的认证流程应保持一致。Passport 通过名为 **Strategies** 的模块化插件实现了这种灵活性。

你无需为每种认证方法编写自定义代码，只需安装所需的特定策略。目前有超过 500 种策略，涵盖了几乎所有现有的认证方法。其中一些是：

- **passport-local：** 处理针对你自己数据库的传统用户名和密码认证。

- **passport-jwt：** 通过从传入请求中提取和验证 JSON Web 令牌来保护 API 端点。

- **passport-google-oauth20：** 允许用户使用现有的 Google 账户即时登录。

- **passport-facebook：** 通过 OAuth 2.0 使用 Facebook 账户登录。

- **passport-github2：** 允许用户使用 GitHub 账户即时登录。

通过将认证拆分为不同的策略，Passport 让你的应用可以无缝扩展。如果你今天使用用户名/密码系统，决定下个月添加“使用 Google 登录”，你无需重写整个认证代码。只需安装 Google 策略并将其接入 Passport 即可。

要在 Express 应用中使用 Passport，必须安装核心 Passport 包和你计划使用的特定策略。对于标准的用户名和密码设置，在终端运行以下命令：

```bash
npm install passport passport-local
```

安装完成后，需要将 Passport 集成到你的 Express 应用设置中。这必须在定义路由之前完成：

```js
const express = require('express');
const passport = require('passport');
const LocalStrategy = require('passport-local').Strategy;

const app = express();

// initialize Passport middleware
app.use(passport.initialize());

// configure Passport to use the Local Strategy
passport.use(new LocalStrategy((username, password, done) => {
    // your database look-up and password verification logic goes here
}));
```

Passport 作为中间件顺利集成到标准 Express 路由系统中。当用户向你的登录路由提交凭据时，Passport 会拦截请求，将数据传递给你配置的策略，并评估结果。

```js
app.post('/login', passport.authenticate('local'), (req, res) => {
  // if this function runs, then authentication was successful
  res.send(`Welcome back, ${req.user.username}!`);
});
```

如果策略成功验证用户，Passport 会自动处理两个关键任务：

1. **会话管理：** 它触发辅助函数，将一小部分用户数据存储在会话 cookie 中。

2. **请求装饰：** 它将经过认证的用户对象直接附加到传入请求的 `req.user` 上。

一旦 `req.user` 被填充，应用中后续的任何中间件或路由处理程序都能立即知道谁已登录，并基于其同一性做出授权决策。

Passport 为你的 Node.js 应用管理用户同一性提供了一套一致、安全且适合生产环境的基础。
