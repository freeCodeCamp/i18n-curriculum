# --interactive--

في الدروس السابقة، تعلمت كيف تتعامل مع الأحداث باستخدام طريقة `addEventListener()`. لكن هناك طريقة أخرى، غير موصى بها، للتعامل مع الأحداث في JavaScript.

معالجات الأحداث المضمَّنة هي خصائص خاصة على عنصر HTML تُستخدم لتنفيذ شفرة JavaScript عند حدوث حدث.

إليك مثالًا على عنصر `button` مع معالج حدث نقر مضمَّن:

:::interactive_editor

```html
<button onclick="alert('Hello World!')">Show alert</button>
```

:::

عندما ينقر المستخدم على الزر، تُستدعى الدالة `alert` ويُعرض مربع تنبيه بالرسالة `Hello World!`.

طريقة أخرى لاستخدام معالجات الأحداث المضمَّنة هي استدعاء دالة معرفة داخل وسم `script` في مستند HTML.

إليك مثالًا لتعريف دالة باسم `changeBgColor` واستدعائها من معالج حدث نقر مضمَّن:

:::interactive_editor

```html
<script>
  function changeBgColor() {
    document.body.style.backgroundColor = "lightblue";
  }
</script>

<button onclick="changeBgColor()">Change background color</button>
```

:::

عندما ينقر المستخدم على الزر، تُستدعى الدالة `changeBgColor` ويتغير لون خلفية الصفحة إلى أزرق فاتح. رغم إمكانية استخدام معالجات الأحداث المضمَّنة بهذه الطريقة، إلا أنها ليست من أفضل الممارسات.

لسبب واحد، يمكن استخدام معالجات الأحداث المضمَّنة فقط لإرفاق مستمع حدث واحد إلى عنصر. إذا أردت إرفاق مستمعي أحداث متعددين إلى نفس العنصر، ستحتاج إلى استخدام `addEventListener()`. سبب آخر هو أن معالجات الأحداث المضمَّنة تخلط بين شفرة HTML وJavaScript، مما يصعّب قراءة الشفرة وصيانتها. من الأفضل فصل شفرة HTML عن شفرة JavaScript باستخدام `addEventListener()` لإرفاق مستمعي الأحداث إلى العناصر.

لا يُنصح باستخدام معالجات الأحداث المضمَّنة في JavaScript الحديثة. لذلك، من أفضل الممارسات الالتزام بطريقة `addEventListener()` عند التعامل مع الأحداث في JavaScript.
