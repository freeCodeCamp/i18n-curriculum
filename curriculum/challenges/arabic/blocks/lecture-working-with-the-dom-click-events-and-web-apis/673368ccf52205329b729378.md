# --interactive--

دعنا نتعلم عن `innerText` و `textContent` و `innerHTML`.

هذه خصائص يمكنك الوصول إليها في JavaScript للحصول على محتوى عنصر HTML أو تغييره. رغم أنها قد تبدو متشابهة جدًا في البداية، إلا أن هناك اختلافات رئيسية بينها. اختيار الخاصية المناسبة يعتمد على حالتك الخاصة، فلنبدأ.

لنبدأ بـ `innerText`.

`innerText` يمثل النص المرئي داخل عنصر HTML وعناصره التابعة. هذه الخاصية لا تشمل النص المخفي أو وسوم HTML، بل النص المعروض فقط.

على سبيل المثال، هنا ترى عنصر `div` يحتوي على فقرتين:

```html
<div id="container">
  <p>Hello, World!</p>
  <p>I'm learning JavaScript</p>
</div>
```

إذا حصلنا على مرجع لهذا العنصر في شفرة JavaScript باستخدام `getElementById()`، يمكننا الوصول إلى خاصية `innerText` لهذا العنصر:

:::interactive_editor

```html
<div id="container">
  <p>Hello, World!</p>
  <p>I'm learning JavaScript</p>
</div>
<script src="index.js"></script>
```

```js
const container = document.getElementById("container");
console.log(container.innerText);
```

:::

هذا هو النص الداخلي لهذا العنصر:

```md
Hello, World!
I'm learning JavaScript
```

تعيد الخاصية سلسلة نصية تحتوي على النص الموجود داخل العنصر، بما في ذلك نص العناصر التابعة له.

يجب أن تعرف أن `innerText` يعيد فقط النص المرئي في اللحظة التي يُطلب فيها النص. إذا كان عنصر فرعي مخفيًا، فلن يظهر نصه.

هذا مثال حيث الفقرة الثانية مخفية:

:::interactive_editor

```html
<div id="container">
  <p>Hello, World!</p>
  <p hidden>I'm learning JavaScript</p>
</div>
```

:::

إذا حاولنا تسجيل `innerText` مرة أخرى، فلن يحتوي الناتج على نص الفقرة الثانية:

:::interactive_editor

```html
<div id="container">
  <p>Hello, World!</p>
  <p hidden>I'm learning JavaScript</p>
</div>
<script src="index.js"></script>
```

```js
const container = document.getElementById("container");
console.log(container.innerText);
```

:::

يمكنك تعيين `innerText` لعنصر HTML بهذه الطريقة، لكن هذا سيستبدل النص الموجود ويضيف عنصر فاصل سطر (`br`) لكل فاصل سطر:

:::interactive_editor

```html
<div id="container">
  <p>Hello, World!</p>
  <p>I'm learning JavaScript</p>
</div>
<script src="index.js"></script>
```

```js
const container = document.getElementById("container");
container.innerText = "JavaScript is awesome!";
```

:::

بما أن `innerText` يأخذ مستوى الظهور في الاعتبار، فإن الحصول على قيمته يُشغل عملية تسمى "إعادة التدفق"، التي تعيد حساب موقع بعض العناصر في الموقع. هذه العملية قد تكون مكلفة حسابيًا، لذا يجب تجنب تشغيلها إذا أمكن.

جيد. الآن لنتحدث عن `textContent`.

`textContent` يعيد النص العادي داخل العنصر، بما في ذلك كل النص داخل العناصر التابعة.

الفرق الأهم بين `innerText` و `textContent` هو أن `textContent` يعيد دائمًا النص الكامل لعنصر HTML وعناصره التابعة، بغض النظر عن كونه مرئيًا أو مخفيًا.

هنا نفس المثال في HTML:

```html
<div id="container">
  <p>Hello, World!</p>
  <p>I'm learning JavaScript</p>
</div>
```

إذا حاولت الوصول إلى هذه الخاصية، سترى نص العنصر وعناصره التابعة في الناتج، مع الحفاظ على المسافات والتنسيق:

:::interactive_editor

```html
<div id="container">
  <p>Hello, World!</p>
  <p>I'm learning JavaScript</p>
</div>
<script src="index.js"></script>
```

```js
const container = document.getElementById("container");
console.log(container.textContent);
```

:::

إذا كان عنصر HTML غير مرئي، كما ترى هنا حيث أخفينا الفقرة الثانية، فإن نصه سيظل مدرجًا في هذه الخاصية:

:::interactive_editor

```html
<div id="container">
  <p>Hello, World!</p>
  <p hidden>I'm learning JavaScript</p>
</div>
<script src="index.js"></script>
```

```js
const container = document.getElementById("container");
console.log(container.textContent);
```

:::
  
`textContent` يشمل أيضًا محتوى عناصر مثل `script` و `style`.

إذا حاولت استبدال قيمة `textContent` على عقدة، فسيُزال كل العقد الفرعية ويُستبدل بعقدة نصية واحدة تحتوي على السلسلة الجديدة:

:::interactive_editor

```html
<div id="container">
  <p>Hello, World!</p>
  <p hidden>I'm learning JavaScript</p>
</div>
<script src="index.js"></script>
```

```js
const container = document.getElementById("container");
container.textContent = "New content";
```

:::

وأخيرًا، لنتحدث عن كيف يختلف `textContent` و `innerText` عن `innerHTML`.

تذكر أنه مع `innerHTML` يمكنك تعيين محتوى HTML الداخلي لعنصر. هذا مفيد لإدخال HTML جديد في DOM بشكل ديناميكي.

لكن تذكر أن هذا يشكل خطرًا أمنيًا إذا لم تتحكم في السلسلة، مثل السلاسل التي تحتوي على بيانات يدخلها المستخدم. إذا كانت هذه البيانات خبيثة، فقد تؤدي إلى مشاكل أمنية خطيرة.

لتجنب ذلك، يُنصح باستخدام خاصية `textContent` لإدخال نص عادي بدلاً من ذلك.

توفر خصائص `innerText` و `textContent` و `innerHTML` في JavaScript طرقًا مختلفة للوصول إلى محتوى عناصر HTML والتعامل معه. يجب أن تفهم الفروقات بينها إذا كان هدفك العمل بفعالية مع محتوى HTML في JavaScript.
