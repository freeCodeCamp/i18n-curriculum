# --description--

Durante la progettazione delle tue applicazioni, spesso avrai una serie di piccole funzioni responsabili di un solo compito. Questo è noto come il "principio di responsabilità singola". Quando hai una serie di piccole funzioni, è meglio testarle per assicurarti che tutto funzioni come previsto. In questa lezione, vedremo come creare un test unitario usando il popolare framework di test Jest.

In questo esempio, creeremo una funzione responsabile di restituire una stringa formattata di nuovo:

```js
export function getFormattedWord(str) {
  if (!str) return '';
  return str.charAt(0).toUpperCase() + str.slice(1);
}
```

Questa funzione `getFormattedWord` ha un parametro chiamato `str` e controllerà prima se `str` è vuoto. In tal caso, viene restituita una stringa vuota. Altrimenti, viene restituita una nuova stringa in cui la prima lettera è maiuscola. Esportiamo questa funzione per poterla usare in un file di test.

In un file `getFormattedWord.test.js` separato, possiamo scrivere alcuni test per verificare che la funzione faccia ciò che dovrebbe fare.

All’inizio del file `getFormattedWord.test.js`, dobbiamo prima importare la funzione in questo modo:

```js
import { getFormattedWord } from "./getFormattedWord.js";
```

Poi dobbiamo installare il pacchetto Jest usando `npm i jest`. Dopodiché possiamo aggiungere un test per verificare che una parola composta da caratteri minuscoli, `hello`, venga formattata correttamente in `Hello`, che inizia con un `H` maiuscolo:

```js
test('capitalizes the first letter of a word', () => {
  expect(getFormattedWord('hello')).toBe('Hello');
});
```

La funzione `expect` serve a testare un valore. È combinata con un matcher, che è una funzione che verifica se il valore si comporta come previsto. In questo caso il matcher è `toBe()`. Jest ha una varietà di matcher per aiutarti a testare veridicità, stringhe, numeri e altro.

Un modo per eseguire i tuoi test è aggiungere uno script npm al file `package.json`. Ecco un esempio di script per il framework Jest:

```js
"scripts": {
  "test": "jest"
},
```

Poi basta eseguire `npm run test` nel terminale per far partire i test. Se i test passano, dovresti vedere qualcosa nel terminale come questo:

```sh
PASS ./getFormattedWord.test.js
  ✓ capitalizes the first letter of a word (1 ms)

Test Suites: 1 passed, 1 total
Tests:       1 passed, 1 total
Snapshots:   0 total
Time:        0.103 s, estimated 1 s
Ran all test suites.
```

Per vedere come appare un test che fallisce, possiamo rompere intenzionalmente il test aggiornando la funzione come segue:

```js
export function getFormattedWord(str) {
  if (!str) return '';
  return "This is incorrect";
}
```

Ora, quando esegui il comando `npm run test`, ci sarà un messaggio di errore perché il test si aspettava un risultato diverso:

```sh
FAIL ./getFormattedWord.test.js
  ✕ capitalizes the first letter of a word (1 ms)

  ● capitalizes the first letter of a word

  expect(received).toBe(expected) // Object.is equality

  Expected: "Hello"
  Received: "This is incorrect"

    2 |
    3 | test('capitalizes the first letter of a word', () => {
    4 |   expect(getFormattedWord('hello')).toBe('Hello');
                                            ^
    5 | });
    6 |

    at Object.toBe (getFormattedWord.test.js:4:37)

Test Suites: 1 failed, 1 total
Tests:       1 failed, 1 total
Snapshots:   0 total
Time:        0.121 s, estimated 1 s
Ran all test suites.
```

Ora possiamo riportare la funzione all’originale qui:

```js
export function getFormattedWord(str) {
  if (!str) return '';
  return str.charAt(0).toUpperCase() + str.slice(1);
}
```

Aggiungi un altro test per verificare i casi in cui non viene fornito alcun argomento a `getFormattedWord`:

```js
test("returns an empty string when no argument is provided", () => {
  expect(getFormattedWord()).toBe("");
});
```

Ci sono altri test che puoi aggiungere per rendere tutto più robusto, ma questi primi due test sono una buona introduzione al test unitario.

Quando si tratta di testare applicazioni JavaScript, alcuni framework di test comuni includono Jest, Mocha e Vitest.

Il test unitario è importante perché ti aiuta a individuare più bug nei tuoi programmi e a garantire che tutto funzioni come previsto. Può anche servire come forma di documentazione per la tua applicazione perché rappresenta il comportamento atteso del tuo codice.
