## Основи CSS

- **Що таке CSS?**: Каскадні таблиці стилів (CSS) — це мова розмітки, яка використовується для застосування стилів до HTML-елементів. CSS використовується для кольорів, фонових зображень, макетів і багато чого іншого.
- **Основна структура правила CSS**: Правило CSS складається з двох основних частин: селектора та блоку оголошень. Селектор — це шаблон, який використовується в CSS для ідентифікації та цільового застосування стилів до конкретних HTML-елементів. Блок оголошень застосовує набір стилів для заданого селектора або селекторів.

Ось загальний синтаксис правила CSS:

```css
selector {
    property: value;
}
```

- **`meta name="viewport"` елемент**: Цей `meta` елемент дає браузеру інструкції щодо керування розмірами сторінки та масштабуванням на різних пристроях, особливо на мобільних телефонах і планшетах.
- **Стилі браузера за замовчуванням**: Кожен HTML-елемент матиме застосовані стилі браузера за замовчуванням. Зазвичай це включає такі речі, як стандартні поля та відступи.

## Рядковий, внутрішній та зовнішній CSS

- **Рядковий CSS**: Ці стилі записуються безпосередньо в HTML-елементі за допомогою атрибута `style`. Зазвичай ви не будете використовувати рядковий CSS через принцип розділення обов’язків.

Ось приклад рядкового CSS:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<p style="color: red;">This is a red paragraph.</p>
```

:::

- **Внутрішній CSS**: Ці стилі записуються всередині тегів `<style>` у розділі `head` HTML-документа. Це може бути корисно для створення коротких прикладів коду, але зазвичай вам не потрібно використовувати внутрішній CSS.
- **Зовнішній CSS**: Ці стилі записуються у окремому CSS-файлі та підключаються до HTML-документа за допомогою елемента `link` у розділі `head`. Для більшості проєктів ви використовуватимете зовнішній CSS-файл замість внутрішнього або рядкового CSS.

## Робота з властивостями `width` і `height`

- **Властивість `width`**: Ця властивість задає ширину елемента. Якщо ширина не вказана, за замовчуванням встановлюється `auto`. Це дозволяє браузеру визначати ширину елемента залежно від його вмісту, батьківського елемента та типу відображення.
- **Властивість `min-width`**: Ця властивість задає мінімальну ширину елемента.
- **Властивість `max-width`**: Ця властивість задає максимальну ширину елемента.
- **Властивість `height`**: Ця властивість задає висоту елемента. Аналогічно, висота за замовчуванням є `auto`, що означає, що вона підлаштовується під вміст.
- **Властивість `min-height`**: Ця властивість задає мінімальну висоту елемента.
- **Властивість `max-height`**: Ця властивість задає максимальну висоту елемента.

## Різні типи комбінацій CSS

- **Комбінатор нащадка**: Цей комбінатор використовується для цільового вибору елементів, які є нащадками заданого батьківського елемента. Наступний приклад націлюється на всі `li` елементи всередині `ul` елементів.

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<ul>
    <li>Example item one</li>
    <li>Example item two</li>
    <li>Example item three</li>
</ul>
```

```css
ul li {
    background-color: yellow;
}
```

:::

- **Комбінатор дочірнього елемента (`>`)**: Цей комбінатор використовується для вибору елементів, які є прямими дочірніми елементами заданого батьківського елемента. Наступний приклад націлюється на всі `p` елементи, які є прямими дочірніми елементами класу `container`.

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<div class="container">
  <p>This will get styled.</p>

  <div>
    <p>This will not get styled.</p>
  </div>
</div>
```

```css
.container > p {
  background-color: black;
  color: white;
}
```

:::

- **Комбінатор наступного братського елемента (`+`)**: Цей комбінатор вибирає елемент, який йде безпосередньо після заданого братського елемента. Наступний приклад вибирає абзац, який йде безпосередньо після елемента `h2`.

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<h2>I am a sub heading</h2>

<p>This paragraph element will get a red background.</p>
```

```css
h2 + p {
  background-color: red;
}
```

:::

- **Комбінатор наступних братських елементів (`~`)**: Цей комбінатор вибирає всі братські елементи, які йдуть після заданого елемента. Наступний приклад стилізує лише другий абзац, оскільки він є єдиним братським елементом `ul` і має того самого батька.

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<div class="container">
  <p>This will not get styled.</p>
  <ul>
    <li>Example item one</li>
    <li>Example item two</li>
    <li>Example item three</li>
  </ul>
  <p>This will get styled.</p>
</div>
<p>This will not get styled.</p>
```

```css
ul ~ p {
  background-color: green;
}
```

:::

## Рядкові, блочні та рядково-блочні елементи

- **Рядкові елементи**: Рядкові елементи займають лише ту ширину, яка їм потрібна, і не починаються з нового рядка. Вони розміщуються в потоці вмісту, дозволяючи тексту та іншим рядковим елементам розташовуватися поруч із ними. Загальними рядковими елементами є `span`, `anchor` і `img`.
- **Блочні елементи**: Блочні елементи починаються з нового рядка і за замовчуванням займають всю доступну ширину, розтягуючись на ширину свого контейнера. Деякі загальні блочні елементи — це `div`, `paragraph` і `section`.
- **Рядково-блочні елементи**: Ви можете встановити елемент як `inline-block`, використовуючи властивість `display`. Ці елементи поводяться як рядкові, але можуть мати встановлені `width` і `height`, як блочні елементи.

## Поля і відступи

- **Властивість `margin`**: Ця властивість використовується для застосування простору поза межами елемента, між кордоном елемента та навколишніми елементами.
- **Властивість `padding`**: Ця властивість використовується для застосування простору всередині елемента, між вмістом і його кордоном.
- **Скорочене записування `margin`**: Ви можете вказати від 1 до 4 значень, щоб встановити поля з усіх боків. Одне значення застосовується до всіх чотирьох сторін; два значення встановлюють `top` і `bottom`, потім `right` і `left`; три значення встановлюють `top`, горизонтальні (`right` і `left`), потім `bottom`; чотири значення встановлюють `top`, `right`, `bottom`, `left`.
- **Скорочене записування `padding`**: Ви можете вказати від 1 до 4 значень, щоб встановити відступи з усіх боків. Одне значення застосовується до всіх чотирьох сторін; два значення встановлюють `top` і `bottom`, потім `right` і `left`; три значення встановлюють `top`, горизонтальні (`right` і `left`), потім `bottom`; чотири значення встановлюють `top`, `right`, `bottom`, `left`.

## Специфічність CSS

- **Специфічність рядкового CSS**: Рядковий CSS має найвищу специфічність, оскільки застосовується безпосередньо до елемента. Він заміщує будь-який внутрішній або зовнішній CSS. Значення специфічності для рядкових стилів — (1, 0, 0, 0).
- **Специфічність внутрішнього CSS**: Внутрішній CSS визначається всередині елемента `style` у розділі `head` HTML-документа. Він має нижчу специфічність, ніж рядкові стилі. Внутрішній і зовнішній CSS мають однаковий рівень специфічності, який визначається їхніми селекторами, а не розташуванням.
- **Специфічність зовнішнього CSS**: Зовнішній CSS підключається через елемент `link` у розділі `head` і записується у окремих `.css` файлах. Як і внутрішній CSS, його специфічність визначається використаними селекторами. Коли два правила мають однакову специфічність, переможцем стає правило, яке йде пізніше в документі. Зовнішній CSS забезпечує найкращу підтримуваність для більших проєктів.
- **Універсальний селектор (`*`)**: спеціальний тип селектора CSS, який відповідає будь-якому елементу в документі. Його часто використовують для застосування стилю до всіх елементів на сторінці, що корисно для скидання або нормалізації стилів у різних браузерах. Універсальний селектор має найнижче значення специфічності серед усіх селекторів. Він додає 0 до всіх частин значення специфічності (0, 0, 0, 0).
- **Селектори типу**: ці селектори націлюються на елементи за їхнім тегом. Селектори типу мають відносно низьку специфічність порівняно з іншими селекторами. Значення специфічності для селектора типу — (0, 0, 0, 1).
- **Селектори класу**: ці селектори визначаються крапкою (`.`) перед назвою класу. Значення специфічності для селектора класу — (0, 0, 1, 0). Це означає, що селектори класу можуть заміщати селектори типу, але їх можуть заміщати селектори ідентифікатора та рядкові стилі.
- **Селектори ідентифікатора**: селектори ідентифікатора визначаються символом решітки (`#`) перед назвою ідентифікатора. Вони мають дуже високу специфічність, вищу за селектори типу та класу, але нижчу за рядкові стилі. Значення специфічності для селектора ідентифікатора — (0, 1, 0, 0).
- **Ключове слово `!important`**: використовується, щоб надати правилу стилю найвищий пріоритет, дозволяючи йому заміщати будь-які інші оголошення для властивості. При використанні воно змушує браузер застосувати вказаний стиль, незалежно від специфічності інших селекторів. Слід бути обережним із використанням `!important`, оскільки це може ускладнити підтримку та налагодження CSS.
- **Алгоритм каскаду**: алгоритм, який визначає, які правила CSS застосовувати, коли кілька стилів націлені на один і той самий елемент. Він гарантує, що використовуються найбільш відповідні стилі на основі чітко визначених правил.
- **Успадкування CSS**: процес, за допомогою якого стилі передаються від батьківських елементів до їхніх дочірніх. Успадкування дозволяє визначати стилі на вищому рівні дерева документа і застосовувати їх до кількох елементів без явного вказування для кожного.
