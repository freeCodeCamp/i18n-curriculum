# --description--


In generale, ci sono due tipi di link che creerai:

- Link a pagine su altri siti web su internet

- Link a pagine situate sul tuo stesso sito web


## Link assoluti
I link a pagine su altri siti web su internet si chiamano link assoluti. Un tipico link assoluto sarà composto dalle seguenti parti: `protocol://domain/path`. Un link assoluto conterrà sempre il protocollo e il dominio della destinazione.

Hai già visto un link assoluto in azione. Il link che hai creato in precedenza alla pagina About di The Odin Project era un link assoluto perché contiene il protocollo e il dominio.

`https://www.theodinproject.com/about`

## Link relativi
I link ad altre pagine all’interno del tuo stesso sito web si chiamano link relativi. I link relativi non includono il nome del dominio, dato che si tratta di un’altra pagina sullo stesso sito, si presume che il nome del dominio sia lo stesso della pagina su cui hai creato il link.

I link relativi includono solo il percorso del file verso l’altra pagina, relativo alla pagina su cui stai creando il link. Questo è piuttosto astratto, vediamo come funziona con un esempio.

All’interno della directory `odin-links-and-images`, crea un altro file HTML chiamato `about.html` e incolla il seguente codice al suo interno:

```html
<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8">
    <title>Odin Links and Images</title>
  </head>

  <body>
    <h1>About Page</h1>
  </body>
</html>
```

Torna alla pagina `index` e aggiungi il seguente elemento àncora per creare un link alla pagina `about`:

```html
<body>
  <h1>Homepage</h1>
	<a href="https://www.theodinproject.com/about">click me</a>

	<a href="about.html">About</a>
</body>
```

Apri il file `index.html` in un browser e clicca sul link about per assicurarti che tutto sia collegato correttamente. Cliccando sul link dovresti andare alla pagina `about` che hai appena creato.

Questo funziona perché le pagine `index` e `about` si trovano nella stessa directory. Questo significa che puoi semplicemente usare il suo nome (`about.html`) come valore `href` del link.

Ma di solito vorrai organizzare meglio le directory del tuo sito web. Normalmente avresti solo il `index.html` nella directory radice e tutti gli altri file HTML nelle loro directory.

Crea una directory chiamata `pages` all’interno della directory `odin-links-and-images` e sposta il file `about.html` in questa nuova directory.

Aggiorna la pagina `index` nel browser e poi clicca sul link `about`. Ora sarà rotto. Questo succede perché la posizione del file della pagina `about` è cambiata.

Per sistemarlo, devi solo aggiornare il valore `about` del link `href` includendo la directory `pages/` dato che quella è la nuova posizione del file `about.html` rispetto al file `index.html`.

```html
<body>
  <h1>Homepage</h1>
  <a href="pages/about.html">About</a>
</body>
```

Aggiorna la pagina `index` nel browser e prova a cliccare di nuovo sul link `about`, ora dovrebbe funzionare di nuovo.

In molti casi questo funzionerà bene; tuttavia, potresti comunque incontrare problemi imprevisti con questo approccio. Aggiungere `./` prima del link nella maggior parte dei casi eviterà questi problemi. Inserendo `./` stai specificando al tuo codice di iniziare a cercare il file/directory a partire dalla directory **corrente**.

```html
<body>
  <h1>Homepage</h1>
  <a href="./pages/about.html">About</a>
</body>
```

## --text--

Qual è la differenza tra un link assoluto e un link relativo?

## --answers--

Un link assoluto è un link a un’altra pagina sul sito web corrente. Un link relativo è un link a un altro sito web.
