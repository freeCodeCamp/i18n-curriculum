# --description--

В Express **обробники маршрутів** (їх також називають функціями зворотного виклику маршруту або функціями контролера) — це функції, які визначають, що відбувається, коли клієнт надсилає запит на певний маршрут.

Обробник маршруту обробляє вхідний запит, виконує необхідну логіку, наприклад, робить запит до бази даних або виконує обчислення, а потім надсилає відповідь клієнту.

Розглянемо це на простому прикладі (який ви фактично використовували весь час):

```javascript
app.get("/menu", (req, res) => {
  res.send("Welcome to the menu!");
});
```

Метод `.get()` — один із маршрутних методів Express. Він слухає **GET** запити на вказаному маршруті. У цьому прикладі він слухає запити до `/menu`.

Функція `(req, res) => { ... }` — це **обробник маршруту**. Вона виконується щоразу, коли клієнт надсилає GET-запит до `/menu`.

Всередині обробника:

* `req` (запит) — це об’єкт, який містить інформацію про вхідний запит, таку як параметри маршруту, рядки запиту, заголовки та дані тіла.
    
* `res` (відповідь) — це об’єкт, який використовується для надсилання відповіді назад клієнту.
    

У цьому випадку `res.send("Welcome to the menu!")` надсилає клієнту відповідь у вигляді простого тексту.

Коли користувач переходить у браузері за адресою `/menu` або надсилає GET-запит на цей маршрут, сервер виконує цей обробник і відповідає повідомленням `Welcome to the menu!`.

Обробники маршрутів — це основа застосунку Express. Вони дозволяють визначати, як різні маршрути вашого вебзастосунку реагуватимуть на запити. Ви також можете додавати логіку до цих обробників, наприклад, обробляти дані форм, взаємодіяти з базами даних або виконувати валідацію.
