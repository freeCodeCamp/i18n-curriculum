# --interactive--

JavaScript에서 원시형과 비원시형 데이터 타입의 차이를 이해하는 것은 효율적이고 버그 없는 코드를 작성하는 데 중요합니다.

이 두 가지 데이터 타입 범주는 메모리에 저장되는 방식과 프로그램에서 조작되는 방식이 다릅니다.

원시형 데이터 타입은 JavaScript에서 가장 단순한 형태의 데이터입니다. 여기에는 number, bigint, string, boolean, `null`, `undefined`, 그리고 symbol이 포함됩니다. 이 타입들은 단일 값을 나타내고 객체가 아니기 때문에 "원시형"이라고 부릅니다.

원시형 데이터 타입을 다룰 때는 값 자체를 직접 다루는 것입니다. 예를 들어, 원시형 값을 가진 변수를 만들면 그 값이 변수에 직접 저장됩니다.

원시형 값은 불변이며, 한 번 생성되면 그 값을 변경할 수 없습니다. 하지만 변수에 새로운 값을 다시 할당할 수는 있습니다. 다음은 원시형 데이터 타입을 다루는 예시입니다:

:::interactive_editor

```js
let num1 = 5;
let num2 = num1;
num1 = 10;

console.log(num2); // 5
```

:::

이 예시에서는 `5`에서 `num1`로 원시형 값(`num2`)을 할당하고 있습니다. 이는 값의 독립적인 복사본을 만듭니다. 따라서 원래 변수(`num1`)에 변경을 가해도 복사본(`num2`)에는 영향을 주지 않습니다.

반면 비원시형 데이터 타입은 더 복잡합니다. JavaScript에서 비원시형은 객체이며, 일반 객체, 배열, 함수가 포함됩니다. 원시형과 달리 비원시형은 여러 값을 속성이나 요소로 가질 수 있습니다.

비원시형 값을 가진 변수를 만들면, 변수에 저장되는 것은 객체 자체가 아니라 객체가 저장된 메모리 위치에 대한 참조입니다. 이로 인해 동작에 중요한 차이가 생깁니다. 다음은 비원시형 타입의 예시입니다:

:::interactive_editor

```js
const originalPerson = { name: "John", age: 30 };
const copiedPerson = originalPerson;

originalPerson.age = 31;

console.log(copiedPerson.age); // 31
```

:::

이 예시에서는 `originalPerson`라는 객체가 `name`와 `age` 두 개의 속성을 가지고 있습니다. 그리고 `originalPerson` 객체를 `copiedPerson`이라는 변수에 할당합니다.

그 후 `age` 객체의 `originalPerson` 값을 업데이트합니다. `age` 객체의 `copiedPerson` 속성을 출력하면 업데이트된 값이 나타납니다.

왜 이런 일이 일어날까요? 이는 `originalPerson`와 `copiedPerson`가 메모리 내 같은 객체를 참조하고 있기 때문입니다.

JavaScript에서 객체를 다른 변수에 할당하면 객체 자체가 아니라 객체에 대한 참조를 복사하는 것입니다. 이를 참조에 의한 얕은 복사라고 합니다. 따라서 한 참조를 통해 객체에 변경을 가하면 그 객체를 참조하는 모든 변수에 변경이 반영됩니다.

JavaScript를 계속 사용하다 보면 원시형과 비원시형 타입의 차이를 이해하는 것이 중요한 상황을 많이 만나게 됩니다. 이는 언어의 많은 부분에 근본적인 개념이며 효율적이고 올바른 코드를 작성하는 데 핵심입니다.
