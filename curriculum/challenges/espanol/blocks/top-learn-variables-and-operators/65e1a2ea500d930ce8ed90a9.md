# --description--

Conozcamos las características de los operadores de JavaScript que van más allá de la aritmética escolar.

Normalmente, el operador más `+` suma números.

Pero, si el binario `+` se aplica a cadenas, las une (concatena):

```js
let s = "my" + "string";
console.log(s); // mystring
```

Ten en cuenta que si alguno de los operandos es una cadena, entonces el otro también se convierte en una cadena.

Por ejemplo:

```js
console.log(1 + '2'); // '12'
console.log('1' + 2); // '12'
```

Mira, no importa si el primer operando es una cadena o el segundo.

Aquí tienes un ejemplo más complejo:

```js
console.log(2 + 2 + '1') // "41" and not "221"
```

Aquí, los operadores trabajan uno tras otro. El primer `+` suma dos números, por lo que devuelve `4`, luego el siguiente `+` añade la cadena `1` a esto, así que es como `4 + '1' = '41'`.

```js
console.log('1' + 2 + 2) // "122" and not "14"
```

Aquí, el primer `+` concatena la cadena y el número, por lo que devuelve `'12'`, luego el siguiente `+` le suma `2`, así que es como `'12' + 2 = '122'`.

El binario `+` es el único operador que soporta cadenas de esa manera. Otros operadores aritméticos funcionan solo con números y siempre convierten sus operandos a números.

Aquí está la demostración para la resta y la división:

```js
console.log( 6 - '2' ); // 4, converts '2' to a number
console.log( '6' / '2' ); // 3, converts both operands to numbers
```

## --text--
Considerando el comportamiento del operador `+` con tipos de datos mixtos en JavaScript, ¿cuál será la salida del siguiente fragmento de código?

```js
console.log('3' + 2 - 1);
```

## --answers--

`31`, porque el operador `+` concatena la cadena y el número antes de la resta.

---

`32`, porque `+` concatena la cadena y el número antes de la resta.

---

`4`, porque todos los operandos se convierten primero en números.

---

`321`, porque el operador `-` no es compatible con cadenas.
