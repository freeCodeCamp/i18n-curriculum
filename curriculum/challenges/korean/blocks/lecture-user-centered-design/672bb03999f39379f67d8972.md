# --interactive--

**참고**: 일부 인터랙티브 예제는 아직 배우지 않은 CSS를 사용할 수 있습니다. 모든 코드를 이해하려고 걱정하지 마세요. 이 예제들의 목적은 이러한 디자인 개념을 미리 보여줘서 작동 방식을 더 잘 이해하도록 돕는 것입니다. 미리 보기를 보려면 인터랙티브 편집기를 활성화해야 합니다.

점진적 공개는 사용자의 현재 활동에 따라 관련된 내용만 보여주고 나머지는 숨기는 디자인 패턴입니다. 이는 인지 부하를 줄이고 사용자 경험을 더 직관적으로 만들기 위해 사용합니다.

```js
const btn = document.getElementById('show-details-btn');
const content = document.getElementById('extra-content');

btn.addEventListener('click', () => {
  const isHidden = content.classList.toggle('hidden');
  btn.setAttribute('aria-expanded', !isHidden);
  if (!isHidden) {
    content.focus();
  }
});
```

예를 들어, 구글 검색 페이지를 사용할 때 대부분의 검색은 간단합니다. 그래서 옵션이 매우 적은 단순한 사용자 인터페이스가 합리적입니다. 하지만 가끔 더 고급 검색이 필요할 때는 구글이 더 많은 옵션을 보여주는 고급 검색 기능을 제공합니다.

구글의 고급 검색 옵션은 구글 홈페이지의 설정 메뉴에서 찾을 수 있습니다. 또는 `google.com/advanced_search`로 이동할 수도 있습니다.

또 다른 점진적 공개의 예는 전자상거래 사이트의 제품에 있는 "자세히 보기" 버튼입니다. 아마존 같은 사이트에 방문하면 다양한 카테고리의 제품 이미지 목록을 볼 수 있습니다. 이는 사용자가 쉽게 쇼핑할 수 있게 하며 너무 많은 정보에 압도되지 않도록 합니다.

사용자가 특정 제품에 관심이 있으면 제품 이미지를 클릭해 제품에 대한 자세한 정보를 볼 수 있습니다. 모든 정보를 한꺼번에 보여주면 사용자가 부담을 느낄 수 있습니다. 그래서 점진적 공개를 사용하면 사용자가 준비되었을 때 더 많은 정보를 선택해서 볼 수 있습니다.

```js
document.querySelectorAll('.details-btn').forEach(button => {
  button.addEventListener('click', () => {
    const product = button.closest('.product');
    const details = product.querySelector('.product-details');
    const isHidden = details.classList.toggle('hidden');
    product.setAttribute('aria-expanded', !isHidden);
    if (!isHidden) {
      details.focus();
      button.textContent = 'Hide details';
    } else {
      button.textContent = 'More details';
    }
  });
});
```
