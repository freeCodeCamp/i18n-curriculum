# --description--

웹 페이지에서 이미지 같은 미디어를 사용할 때 고려해야 할 세 가지 도구가 있습니다: 크기, 형식, 그리고 압축입니다.

먼저 이미지 크기에 대해 이야기해 보겠습니다. 웹사이트를 만들 때 이미지를 특정 크기로 스타일링하는 경우가 많습니다. 예를 들어, 640x480 해상도로 이미지를 표시할 수 있습니다. 640은 너비를, 480은 픽셀 단위의 높이를 나타냅니다. 이미지를 준비할 때는 스타일링과 일치하도록 640x480 크기로 조정해야 합니다. 만약 1920x1080 크기의 이미지를 제공하면서 스타일링은 훨씬 작게 한다면, 사용자가 불필요한 데이터를 다운로드하게 됩니다. 해상도가 작을수록 파일 크기도 작아집니다.

다음으로 고려할 것은 파일 형식입니다. 가장 흔한 파일 형식 두 가지는 PNG와 JPG이지만, 이들은 더 이상 이미지를 제공하는 데 가장 이상적인 형식이 아닙니다. 구형 브라우저 지원이 필요하지 않다면 WEBP나 AVIF 같은 더 최적화된 형식을 사용하는 것을 고려하세요.

마지막으로, 이미지 파일 크기를 줄이기 위해 압축 알고리즘을 실행할 수 있습니다. pngcrush 같은 도구는 PNG 같은 무손실 형식에 잘 작동하는데, 원본 데이터를 완벽하게 복원할 수 있기 때문에 품질 손실 없이 압축할 수 있습니다. 하지만 모든 형식이 무손실은 아닙니다. 예를 들어 JPG는 손실 압축을 사용합니다 — JPG를 다시 저장하거나 재압축할 때마다 일부 이미지 데이터가 영구적으로 버려져 품질이 저하됩니다. 웹 페이지용 이미지를 선택하고 준비할 때 이런 점들을 모두 염두에 두세요.
