# --description--

前のレッスンでは、Reactでこのような小さなコンポーネントを作る方法を学びました。

```jsx
function Greeting() {
  const developerName = "Jessica";
  return <h1>Hi {developerName}!</h1>;
}
```

このコンポーネントを別の親コンポーネントやルートコンポーネントの中にネストすることもできます。

```jsx
function App() {
  return <Greeting />;
}

function Greeting() {
  const developerName = "Jessica";
  return <h1>Hi {developerName}!</h1>;
}
```

このコードは問題なく動作し、`Hi Jessica!`の結果を画面に表示しますが、あまり柔軟なコンポーネントではありません。

もしNaomiやTom、Oliverなど別の名前を表示したい場合はどうでしょう？ここでReactのプロップスが役立ちます。プロップスはpropertiesの略で、親コンポーネントが子コンポーネントにデータを渡す方法です。プロップスは文字列、数値、ブール値、オブジェクト、配列など、どんな型でも使えます。

先ほどの例を更新して、`name`というプロップを受け取るようにしましょう。

```jsx
function App() {
  return <Greeting name="Jessica" />;
}

export default App;

function Greeting(props) {
  console.log(props);
  return <h1>Hi {props.name}!</h1>;
}
```

子コンポーネントの`Greeting`では、名前を`props.name`とハードコーディングする代わりに`"Jessica"`を使っています。また、`props`をコンソールにログ出力しており、オブジェクトとして表示されています。

親の`App`コンポーネントの中で、`name`プロップに値を渡して子に伝えています。画面上の結果は先ほどと同じですが、より柔軟なコンポーネントができました。

これで子コンポーネントを何度も再利用し、毎回違う名前を渡すことができます。

```jsx
function App() {
  return (
    <>
      <Greeting name="Naomi" />
      <Greeting name="Tom" />
      <Greeting name="Jessica" />
      <Greeting name="Oliver" />
    </>
  );
}
```

また、プロップスの分割代入を使うと読みやすくなります。`Greeting`コンポーネントを次のように書き換えられます。

```jsx
function Greeting({ name }) {
  return <h1>Hi {name}!</h1>;
}
```

このコードは同じ結果を出しますが、どのプロップを受け取るかがより明確になります。

Reactには`children`という特別なプロップもあります。コンポーネントの開始タグと終了タグの間に置いたJSXは、`children`としてコンポーネントに渡されます。

例えば：

```jsx
function Card({ children }) {
  return <div className="card">{children}</div>;
}

function App() {
  return (
    <Card>
      <h2>Hello</h2>
    </Card>
  );
}
```

この場合、`<Card>...</Card>`内のJSXが`children`プロップになります。DOMにレンダリングされる出力は次のようになります。

```html
<div class="card">
  <h2>Hello</h2>
</div>
```

このパターンは、コンポーネントが他のUI要素をラップするコンポーネント合成によく使われます。

時には、渡すプロパティがたくさんある場合があります。1つずつ渡す代わりに、オブジェクトに変換してからスプレッド演算子（`...`）を使うことができます。

新しい子コンポーネント`DeveloperCard`の例を示します。

```jsx
function DeveloperCard({ name, age, country }) {
  return (
    <div className="developer-card">
      <h1>Developer: {name}</h1>
      <p>Age: {age}</p>
      <p>Country: {country}</p>
    </div>
  );
}
```

この`DeveloperCard`コンポーネントは、`name`、`age`、`country`の3つのプロップを受け取ります。

親の`App`コンポーネントでは、スプレッド構文を使ってオブジェクトのすべてのプロパティを個別のプロップとして子コンポーネントに渡せます。

```jsx
function App() {
  const developerObj = {
    name: "Alice",
    age: 30,
    country: "USA",
  };

  return (
    <div className="App">
      <DeveloperCard {...developerObj} />
    </div>
  );
}
```

これは、オブジェクトの配列を扱い、複数のプロパティセットを子コンポーネントに渡す場合に特に便利です。例えば、開発者のリストがあり、配列内の各オブジェクトが同じ構造で異なる人物を表している場合などです。

配列のリストをレンダリングする方法は、今後のレッスンでさらに学びます。

Reactでプロップスを使うと、コンポーネントがより柔軟で再利用可能になり、複雑なUIを作りやすくなります。ただし、プロップスは不変で、一度渡すと変更できません。ユーザー入力を扱いデータを変更する必要がある場合は、代わりにstateを使います。stateの管理については今後のレッスンで学びます。
