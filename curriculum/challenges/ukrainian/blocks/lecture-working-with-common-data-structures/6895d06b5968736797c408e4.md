# --description--

Масиви — це фундаментальна структура даних у комп’ютерних науках. Всі масиви зберігають впорядковані колекції даних, але залежно від їх типу вони можуть працювати по-різному «під капотом».

Їхня базова поведінка може суттєво впливати на ефективність програми, тож давайте дізнаємося про динамічні та статичні масиви та їхні відмінності, щоб ви могли обрати найефективніший для свого застосунку.

Почнемо зі статичних масивів.

**Статичні масиви** мають фіксований розмір. Вони зберігають елементи у суміжних ділянках пам’яті.

Розмір статичного масиву визначається під час його ініціалізації. Після того, як певний блок пам’яті виділено, він фіксований і не може бути змінений під час виконання програми. Це ключова характеристика статичних масивів.

Зберігання елементів у суміжних ділянках пам’яті робить процес отримання даних більш ефективним, бо програма може зберегти адресу першого елемента, а потім використовувати індекси для простих обчислень і знаходження інших елементів у пам’яті.

Завдяки цьому доступ до значень статичного масиву займає сталий час `O(1)`, що дуже ефективно.

Ви можете використовувати статичний масив, коли заздалегідь знаєте кількість елементів, які будуть збережені. Це також корисно, коли значення будуть часто доступні, оскільки операція доступу дуже ефективна.

Однак ця структура даних не може розширюватися або звужуватися, тож якщо кількість елементів може змінюватися, слід використовувати динамічний масив.

Спроба збільшити розмір статичного масиву вимагатиме створення нового масиву та копіювання всіх елементів зі старого в новий, що є неефективним. У такому випадку динамічний масив буде набагато кращим, бо він автоматично обробляє цей процес.

Python не включає традиційні статичні масиви як вбудовані структури даних.

Але інші мови програмування, як Java, підтримують їх. Ось приклад статичного масиву в Java, який може зберігати три цілі числа:

```java
int[] numbers = new int[3]; 
```

Масиви в Python є динамічними, тож розглянемо їх.

**Динамічні масиви** більш гнучкі, бо можуть автоматично розширюватися або звужуватися під час виконання програми.

Вони працюють через механізм автоматичного зміни розміру, який копіює елементи у новий масив, коли оригінальний масив заповнений. Процес виконується ефективно, бо розмір нового масиву обирається так, щоб зробити ці обчислювально дорогі операції менш частими.

Доступ до елементів динамічного масиву займає сталий час `O(1)`, тож ця операція дуже ефективна.

Вставка елемента посередині масиву займає лінійний час `O(n)`, бо елементи після нього потрібно перемістити.

Вставка елемента в кінець масиву займає сталий час `O(1)`, якщо в динамічному масиві ще є вільне місце, але якщо масив заповнений і потребує зміни розміру, ця операція має складність `O(n)`.

Ви повинні використовувати динамічні масиви, коли не знаєте заздалегідь кількість значень, які потрібно зберегти в масиві. Вони також корисні, коли ви часто вставляєте та видаляєте елементи.

Вбудована структура даних `list` у Python працює як динамічний масив. Ви можете створити список, записуючи елементи у квадратних дужках, розділених комами.

```python
numbers = [3, 4, 5, 6]
```

Ви можете отримати доступ до елемента, записавши ім’я змінної, що містить список, після чого квадратні дужки з відповідним індексом.

Індекси починаються з 0 для першого елемента і збільшуються на 1 для кожного наступного:

```python
numbers[0]  # 3
numbers[1]  # 4
numbers[2]  # 5
numbers[3]  # 6
```

Щоб оновити значення, просто призначте його заново:

```python
numbers[2] = 16
```

Ви можете додати елементи до списку за допомогою методу `.append()`:

```python
numbers.append(7)
```

Ви можете вставити елементи на певний індекс за допомогою методу `.insert()`, передаючи індекс як перший аргумент, а сам елемент — як другий.

```python
numbers.insert(3, 15)
```

Ви можете видалити елемент за певним індексом за допомогою методу `.pop()`:

```python
numbers.pop(2)
```

Якщо індекс не вказано, `.pop()` видалить останній елемент.

Існують інші вбудовані методи списків, які ви можете переглянути в документації для легкого додавання та видалення елементів.

Ось така сила динамічних масивів, або списків у цьому випадку.

Загалом, слід використовувати статичні масиви, коли ви знаєте кількість елементів заздалегідь і вам потрібно часто отримувати до них доступ, а динамічні масиви — коли кількість елементів невідома або змінюється з часом.

Завжди враховуйте компроміс між простотою статичних масивів і гнучкістю динамічних. Обидва корисні для конкретних випадків і сценаріїв. Вміння обрати найкращий варіант для певної задачі — це частина навичок розв’язання проблем, які ви поступово розвиватимете з практикою.
