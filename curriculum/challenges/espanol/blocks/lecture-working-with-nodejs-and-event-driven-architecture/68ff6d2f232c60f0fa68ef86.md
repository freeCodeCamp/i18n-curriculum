# --description--

Hasta ahora, has estado escribiendo código del lado del cliente con JavaScript, código que se ejecuta en el lado del cliente de una aplicación web, usualmente un navegador web.

JavaScript del lado del cliente puede ser muy poderoso, pero tiene ciertas limitaciones para tareas más complejas.

Por ejemplo:

- Tiene acceso muy restringido a archivos locales.
- No está realmente diseñado para manejar lógica compleja de aplicaciones.
- Y tiene posibles problemas de seguridad. Por ejemplo, no querrías exponer información sensible, como las credenciales de la base de datos, al código del lado del cliente.
    
JavaScript del lado del cliente tiene estas limitaciones porque fue diseñado inicialmente para ejecutarse exclusivamente en navegadores web.

Los navegadores web proporcionan el entorno necesario para ejecutar código JavaScript, incluido el JavaScript Engine. También proporcionan acceso al Document Object Model (DOM), para que puedas acceder a los elementos HTML en tu código.

Antes, no podíamos ejecutar JavaScript fuera de un navegador, pero ahora podemos con Node.js.

Node.js es un entorno de ejecución de JavaScript. Está definido oficialmente como “un entorno de ejecución de JavaScript de código abierto y multiplataforma.”

Veamos qué significa esto:

- Node.js es de código abierto porque su código está disponible públicamente y es mantenido por una gran comunidad de desarrolladores.
- Es multiplataforma porque funciona en cualquier sistema operativo, incluyendo Windows, macOS y Linux.
- Y es un entorno de ejecución de JavaScript porque permite ejecutar código JavaScript fuera de un navegador.
    
Node.js se utiliza en una amplia variedad de industrias. Los desarrolladores lo usan para construir servidores web y APIs que manejan solicitudes HTTP, así como aplicaciones web y móviles.

Si estás usando o desarrollando un sitio web o una aplicación web que interactúa con una base de datos o maneja datos complejos, hay una alta probabilidad de que uses Node.js detrás de escena en algún momento.

Es eficiente, escalable y cuenta con una gran comunidad de desarrolladores y mantenedores alrededor del mundo.

Ahora que sabes qué es Node.js, veamos algunas de las diferencias entre el navegador y el entorno de ejecución de Node.

El entorno del navegador está diseñado principalmente para el desarrollo web front-end, por lo que ejecuta JavaScript del lado del cliente. En contraste, el entorno de ejecución Node está diseñado principalmente para el desarrollo web back-end, por lo que ejecuta JavaScript del lado del servidor. Esto también determina las API que ofrecen.

Por ejemplo, desde el navegador, puedes acceder a la API del DOM, pero hay restricciones para acceder al sistema de archivos local. En cambio, desde el entorno de ejecución Node, puedes acceder a casi todos los recursos del sistema, incluido el sistema de archivos, pero no al DOM.

Ambos entornos de JavaScript proporcionan un objeto global. En el entorno de ejecución de JavaScript del navegador, esto se llama `window`. Este objeto `window` proporciona acceso a funcionalidades relacionadas con el navegador, como acceso a métodos para manipular el DOM, gestionar cookies y manejar eventos del navegador. En el entorno de Node.js, este objeto global se llama `global`. El objeto `global` proporciona acceso a funcionalidades específicas de Node.js, como módulos integrados para trabajar con archivos locales, redes e interactuar con el sistema operativo.

Finalmente, otra diferencia importante es que puedes elegir la versión de Node.js que quieres usar en tu servidor. Sin embargo, no tienes control sobre la versión del entorno del navegador que tus usuarios usarán para visitar tu sitio web.

Node.js se ha vuelto muy importante para los desarrolladores en todo el mundo. Es una herramienta poderosa, escalable y versátil que ha transformado el desarrollo web al permitir que los desarrolladores usen JavaScript para construir tanto el frontend como el backend de aplicaciones full-stack. Esto ha hecho que el proceso de desarrollo sea más eficiente porque los desarrolladores no necesitan aprender un nuevo lenguaje de programación solo para desarrollar el backend.
