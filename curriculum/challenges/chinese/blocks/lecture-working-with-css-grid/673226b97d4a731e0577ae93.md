# --interactive--

`grid-template-areas` 属性允许你通过使用命名标签来设计可视化的网格布局。

然后使用 grid-area 属性将标签分配给特定的网格项。 换句话说，这些命名标签也被称为“网格区域名称”。

这是 `grid-template-areas` 属性的基本语法：

```css
grid-template-areas:
 'header header header'
 'left-sidebar main right-sidebar'
 'footer footer footer';
```

这是基本语法中需要注意的要点：

- 像 `header` 和 `main` 这样的值是网格区域的名称。

- 字符串中的每个空格分隔值对应于一个列。

- 每个字符串代表网格中的一行。

因此，在语法中，我们有一个 3*3 的网格容器。

定义模板之后，你必须使用 `grid-area` 属性和命名标签（或区域）作为值来告诉 CSS 特定元素属于网格的该区域。

`grid-area` 属性将网格项连接到你在 `grid-template-areas` 中定义的命名区域。

演示 `grid-template-areas` 属性功能的一种流行方法是创建经典的圣杯布局。

如果你不知道它是什么，圣杯布局是一种网页设计模式，有页眉、页脚、两个侧边栏和一个主内容区。 它确保主要内容优先，而侧边栏和其他部分在布局中进行响应性调整。

有许多实现圣杯布局的解决方案，但是使用 `grid-template-areas` 和 `grid-area` 属性是创建它的最直接的方法。

这是圣杯版型的一个示例：

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<div class="grid-container">
  <div class="header">
    <h2>Header</h2>
  </div>
  <div class="sidebar-left">
    <h2>Left Sidebar</h2>
  </div>
  <div class="main"><h2>Main Content</h2></div>
  <div class="sidebar-right">
    <h2>Right Sidebar</h2>
  </div>
  <div class="footer">
    <h2>Footer</h2>
  </div>
</div>
```

```css
.grid-container {
  display: grid;
  grid-template-areas:
    'header header header'
    'sidebar-left main sidebar-right'
    'footer footer footer';
  grid-gap: 10px;
  background-color: #2196F3;
  padding: 10px;
}

.header {
  grid-area: header;
  background-color: rgba(255, 255, 255, 0.8);
  padding: 20px;
  text-align: center;
}

.sidebar-left {
  grid-area: sidebar-left;
  background-color: rgba(255, 255, 255, 0.8);
  padding: 20px;
  text-align: center;
}

.main {
  grid-area: main;
  background-color: rgba(255, 255, 255, 0.8);
  padding: 20px;
  text-align: center;
}

.sidebar-right {
  grid-area: sidebar-right;
  background-color: rgba(255, 255, 255, 0.8);
  padding: 20px;
  text-align: center;
}

.footer {
  grid-area: footer;
  background-color: rgba(255, 255, 255, 0.8);
  padding: 20px;
  text-align: center;
}
```

:::

注意，`grid-template-areas` 和 `grid-area` 属性可以彼此独立使用。

`grid-template-areas` 属性专门用于通过在网格容器内映射命名的网格区域来定义可视化布局。

另一方面，`grid-area` 属性用于定位单个网格项，方法是指定它们的行和列位置，或者引用用 `grid-template-areas` 属性定义的命名区域。
