# --description--

JavaScriptの`throw`文は、ユーザー定義の例外をスローするために使われます。プログラミングにおける例外とは、予期しない出来事が発生し、プログラムの通常の流れを妨げることを指します。

プログラマーとして、これらの例外を適切に処理することが重要です。そうしないと、エラーが発生した際にプログラムが予期せずクラッシュしてしまいます。`throw`文の基本的な構文は以下の通りです。

```js
throw expression;
```

この場合の`expression`は、スローしたい例外を表すオブジェクトや値になります。例としては、組み込みの例外クラスである`Error`、`TypeError`、`RangeError`クラスなどがあります。以下は`throw`文を使って`TypeError`をスローする例です。

```js
function validateNumber(input) {
  if (typeof input !== "number") {
    throw new TypeError("Expected a number, but received " + typeof input);
  }
  return input * 2;
}
```

この例では、`input`の型が`number`でないかどうかをチェックしています。そうでなければ、カスタムメッセージ付きの`TypeError`をスローします。そうでなければ、関数は`input`と`2`の乗算結果を返します。

より一般的なエラーメッセージをスローしたい場合は、次のように`Error`コンストラクタを参照できます。

```js
function divide(numerator, denominator) {
  if (denominator === 0) {
    throw new Error("Cannot divide by zero");
  }
  return numerator / denominator;
}
```

これは、分母が`0`かどうかをチェックする関数の例です。もしそうなら、「`Cannot divide by zero`」というカスタムエラーメッセージをスローします。
