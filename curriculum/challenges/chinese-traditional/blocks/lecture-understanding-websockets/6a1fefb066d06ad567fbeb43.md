# --description--

WebSockets 讓客戶端和伺服端能夠即時通訊。

通常，像是網頁和行動應用程式這類的客戶端應用會使用 HTTP。使用 HTTP 時，客戶端會送出請求，伺服端則回傳回應。這就是你所學過的請求與回應循環。

連線建立並交換資料後，連線就會關閉。這對大多數情況來說運作良好，例如載入頁面、取得或送出資料。但對於即時應用來說，這種方式會變得緩慢且不可靠。

舉例來說，想像一個聊天應用。如果你決定用 HTTP 來實作，客戶端必須不斷地向伺服端詢問資料，每則訊息都要開啟連線、接收資料，然後再關閉連線。

這正是 WebSockets 解決的問題。它們在客戶端和伺服端之間建立一條永久連線。連線一旦開啟，雙方就能隨時在這條即時通道中交換資料，直到任一方送出關閉信號。

雖然 WebSockets 與 HTTP 不同，但每個 WebSocket 連線都是從 HTTP 開始的，這個過程稱為握手。

接著，客戶端請求將連線升級為 WebSocket。伺服端同意後，會送出 `101 Switching Protocols` 回應，將連線升級為 WebSocket。之後，連線保持開啟，即時通訊開始。

流程如下：

1. 客戶端透過 HTTP 連接伺服端
2. 伺服端接受並將連線升級為 WebSocket
3. 連線保持開啟
4. 雙方隨時交換資料直到連線關閉

這個流程是雙向的，不像 HTTP 是請求-回應模式。

```md
Client <---> Server
```

使用 HTTP 時，流程通常遵循請求-回應模型：

```md
Client ---> Request
Server ---> Response
```

關於 WebSockets 與 HTTP 的差異，HTTP 就像交換信件，你寫信、等待回信，對話結束。但 WebSocket 就像打電話，你和對方可以即時交談。

WebSockets 支援即時應用，例如聊天應用、線上多人遊戲、即時體育賽事更新、股市儀表板，以及像 Google Docs 這樣的協作工具。
