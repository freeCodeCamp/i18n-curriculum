# --description--

コンビネーターは、グループ化やチェーンとは異なる方法で複数のセレクターを組み合わせることができ、セレクター間の関係性を示します。コンビネーターには全部で4種類ありますが、今回はCSSでセレクター間に単一のスペースを入れて表す子孫コンビネーターだけを紹介します。子孫コンビネーターは、最後のセレクターにマッチする要素が、前のセレクターにマッチする祖先（親、祖父母など）を持つ場合にのみ、その要素を選択します。

例えば`.ancestor .child`は、`child`というクラスを持つ祖先がある場合に、`ancestor`というクラスを持つ要素を選択します。別の言い方をすると、子要素は`ancestor`の中にネストされている場合にのみ選択されます。以下の例を見て、指定されたCSSルールに基づいてどの要素が選択されるか考えてみてください。

```html
<!-- index.html -->

<div class="ancestor"> <!-- A -->
  <div class="contents"> <!-- B -->
    <div class="contents"> <!-- C -->
    </div>
  </div>
</div>

<div class="contents"></div> <!-- D -->
```

```css
/* styles.css */

.ancestor .contents {
  /* some declarations */
}
```

上の例では、`contents`クラスを持つ最初の2つの要素（`B`と`C`）は選択されますが、最後の要素（`D`）は選択されません。あなたの予想は当たっていましたか？

ルールに追加できるコンビネーターの数に制限はないので、`.one .two .three .four`のように書くことも完全に有効です。これは、`four`というクラスを持つ要素が、`three`というクラスを持つ祖先を持ち、その祖先がさらに`two`というクラスを持つ祖先を持つ場合に選択されることを意味します。ただし、これほど深くネストされた要素を選択しようとすると、非常に複雑で長くなり、特異性の問題も起きやすいため、避けることをお勧めします。
