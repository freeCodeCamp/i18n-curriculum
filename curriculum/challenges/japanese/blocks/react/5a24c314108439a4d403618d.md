# --description--

これまで、React コンポーネントはクライアントでレンダリングしてきました。通常はこれが一般的な方法です。しかし、サーバーで React コンポーネントをレンダリングすることが理にかなっている場合もあります。React は JavaScript のビューライブラリであり、Node を使えばサーバー上で JavaScript を実行できるため、これが可能です。実際、React にはこの目的で使える `renderToString()` メソッドが用意されています。

実際のアプリでサーバーでのレンダリングが使われる主な理由は2つあります。まず、これをしないと、React アプリは最初にブラウザに読み込まれるときに比較的空の HTML ファイルと大きな JavaScript バンドルで構成されます。これは、検索エンジンがページの内容をインデックス化して人々が見つけやすくするためには理想的ではない場合があります。サーバーで初期の HTML マークアップをレンダリングしてクライアントに送れば、最初のページ読み込み時にページのすべてのマークアップが含まれ、検索エンジンがクロールできます。次に、これにより初期のページ読み込みが速くなります。レンダリングされた HTML はアプリ全体の JavaScript コードよりも小さいためです。React は初期読み込み後もアプリを認識して管理できます。

# --instructions--

`renderToString()` メソッドは `ReactDOMServer` に用意されており、ここではグローバルオブジェクトとして利用できます。このメソッドは React 要素を引数に取ります。これを使って `App` を文字列にレンダリングしてください。

# --hints--

`App` コンポーネントは `ReactDOMServer.renderToString` を使って文字列にレンダリングされるべきです。

