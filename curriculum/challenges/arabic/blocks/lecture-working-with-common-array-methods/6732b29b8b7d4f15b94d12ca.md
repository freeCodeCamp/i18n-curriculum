# --interactive--

النسخة السطحية من المصفوفة هي مصفوفة جديدة تحتوي على نفس العناصر الموجودة في الأصل. إذا كانت المصفوفة تحتوي فقط على قيم بدائية مثل الأرقام أو السلاسل النصية، تكون المصفوفة الجديدة منفصلة تمامًا. لكن إذا كانت المصفوفة تحتوي على مصفوفات أخرى بداخلها، فإن الأصل والنسخة يشتركان في مراجع لنفس المصفوفات الداخلية. هذا يعني أنه إذا غيرت شيئًا داخل مصفوفة داخلية مشتركة، سترى هذا التغيير في كلتا المصفوفتين.

النسخ السطحية مفيدة عندما تحتاج إلى تعديل الهيكل على المستوى الأعلى، مثل إضافة أو إزالة أو إعادة ترتيب العناصر، دون تعديل المصفوفة الأصلية أو المصفوفات الداخلية.

هناك عدة طرق لإنشاء نسخ سطحية من المصفوفات، وسنستعرض بعض الطرق الأكثر شيوعًا: `concat()`، و`slice()`، ومعامل النشر.

لنبدأ بطريقة `concat()`. هذه الطريقة تنشئ مصفوفة جديدة بدمج مصفوفتين أو أكثر. عند استخدامها مع مصفوفة واحدة، فإنها تنشئ نسخة سطحية فعليًا. إليك مثالًا:

:::interactive_editor

```js
const originalArray = [1, 2, 3];
const copyArray = [].concat(originalArray);

console.log(copyArray); // [1, 2, 3]
console.log(copyArray === originalArray); // false
```

:::

في هذا المثال، نستخدم طريقة `concat()` لدمج مصفوفة فارغة مع `originalArray`. هذا سينشئ مصفوفة جديدة هي نسخة سطحية من `originalArray`.

تحتوي `copyArray` على نفس العناصر الموجودة في `originalArray`، لكنها كائن مصفوفة مختلف، ولهذا فإن فحص المساواة الصارمة (`===`) يرجع `false`.

طريقة أخرى لإنشاء نسخة سطحية هي طريقة `slice()`. عند استدعائها بدون معلمات، تعيد `slice()` نسخة سطحية من المصفوفة كاملة. إليك كيف تعمل:

:::interactive_editor

```js
const originalArray = [1, 2, 3];
const copyArray = originalArray.slice();

console.log(copyArray); // [1, 2, 3]
console.log(copyArray === originalArray); // false
```

:::

في هذه الحالة، تنشئ `originalArray.slice()` مصفوفة جديدة هي نسخة سطحية من `originalArray`. مرة أخرى، تحتوي `copyArray` على نفس العناصر لكنها كائن مصفوفة مختلف.

معامل النشر (`...`)، الذي تم تقديمه في ES6، يوفر طريقة مختصرة أخرى لإنشاء نسخ سطحية من المصفوفات. إليك مثالًا:

:::interactive_editor

```js
const originalArray = [1, 2, 3];
const copyArray = [...originalArray];

console.log(copyArray); // [1, 2, 3]
console.log(copyArray === originalArray); // false
```

:::

معامل النشر (`...`) ينشر عناصر `originalArray` داخل مصفوفة جديدة، مما ينشئ نسخة سطحية فعالة. من المهم ملاحظة أن كل هذه الطرق تنشئ كائنات مصفوفة جديدة، مما يعني أنه يمكنك تعديل النسخة دون التأثير على المصفوفة الأصلية. على سبيل المثال:

:::interactive_editor

```js
const originalArray = [1, 2, 3];
const copyArray = [...originalArray];

copyArray.push(4);
console.log(originalArray); // [1, 2, 3]
console.log(copyArray);     // [1, 2, 3, 4]
```

:::

في هذا المثال، إضافة عنصر إلى `copyArray` لا يؤثر على `originalArray`.

باختصار، يمكن إنشاء نسخ سطحية من المصفوفات بسهولة باستخدام طرق مثل `concat()`، و`slice()`، أو معامل النشر. هذه الطرق مفيدة لإنشاء مصفوفات جديدة يمكن تعديلها بشكل مستقل عن المصفوفة الأصلية.
