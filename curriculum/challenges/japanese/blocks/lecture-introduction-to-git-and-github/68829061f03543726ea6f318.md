# --description--

GitとGitHubについて少し話しましたので、ここからそれらを組み合わせていきましょう。そのためにはリポジトリが必要です。でもリポジトリとは何でしょうか？リポジトリはプロジェクトのコンテナのようなものだと考えてください。アプリを作っている場合、そのアプリのファイルをまとめてリポジトリに保管します。

リポジトリはコンピューターのローカルにある場合もあれば、GitHubのようなサービス上のリモートにある場合もあります。リポジトリを作成するには、ローカルかリモートのどちらかから始められます。結果は同じなので、自分のワークフローに合った方法を選んでください。リモートリポジトリから始めるワークフローの場合は、GitHubのウェブサイトから直接リポジトリを作成できます。以下の手順に従ってください。

<a href="https://github.com" target="_blank">GitHubのウェブサイト</a>にアクセスし、ログインしていることを確認したら、ナビゲーションバーの右上にあるプラスアイコンをクリックします。いくつかのオプションが表示されますが、関心があるのは「New Repository」です。これを選択すると、新しいリポジトリを作成するためのUIが開きます。

最初に表示されるオプションはテンプレートの選択です。まだ設定していないかもしれませんが、テンプレートは新しいリポジトリの出発点として使える特別な種類のリポジトリです。これにより、すべてのプロジェクトにドキュメントやイシューテンプレートを含めるのに役立ちます。

次のオプションはオーナーを選択することです。ユーザーアカウントや所有している、またはアクセス権のある組織を選べます。その後、リポジトリに名前を付けます。一般的には、作成しているプロジェクトを簡潔に説明する名前を使うことをお勧めします。ベストプラクティスとして、リポジトリ名にスペースを含めないようにしてください。GitHubはスペースを自動的にハイフンに置き換えます。

次に、リポジトリのホームページに表示される説明を任意で追加できます。

リポジトリを公開するか非公開にするかも選べます。公開リポジトリは誰でも閲覧およびダウンロードできます。非公開リポジトリは自分と明示的にアクセス権を与えた人だけがアクセスできます。

最後に、テンプレートを選択しなかった場合は、リポジトリ内でファイルを自動生成するオプションが表示されます。READMEファイルはリポジトリのホームページに表示され、プロジェクトのドキュメントや説明を提供するのに最適です。`.gitignore`ファイルはGitが「無視」するファイルを指定でき、パッケージの依存関係や環境ファイルなど、保存したくないものに便利です。GitHubは指定した言語やランタイムに共通する項目を除外するテンプレートを提供します。そしてライセンスは、誰がソフトウェアを使用できるか、何ができるかを定めた法的文書です。

設定がすべて完了したら、「Create Repository」ボタンをクリックして作成を完了してください。新しいリポジトリのランディングページに自動的にリダイレクトされます！

ここから「Code」ボタンをクリックすると、リポジトリをコンピューターにクローンするためのオプションが表示されます。

このレッスンでは「Codespaces」と「Copilot」タブは無視してください。代わりに「Local」タブの3つのオプションにフォーカスします。最初の2つ、HTTPSとSSHは同じ使い方です。`git clone`コマンドを使い、ボックスに表示されたURLでリモートリポジトリをコンピューターにクローンします。この2つの違いは接続方法と認証方法です。

公開リポジトリをクローンする場合は認証は不要です。しかし、非公開リポジトリをクローンしたり、公開・非公開リポジトリにプッシュしたりするには認証が必要です。HTTPSのリモートURLはGitHubのユーザー名とパスワードで認証していましたが、セキュリティ上の理由で現在はサポートされておらず、代わりにユーザー名とアクセストークンを使う必要があります。それでもSSHよりは安全性が低いです。SSHは秘密鍵と公開鍵を使い、自動的にサインインできる利点があります。SSHの仕組みや設定方法については別のレッスンやGitHubのドキュメントを参照してください。

リポジトリをクローンするには、ターミナルで次のコマンドを実行してください：

```sh
git clone git@github.com:your-username/your-repository-name
```

これにより、リポジトリ名のフォルダーがコンピューターに作成され、その中にリポジトリのすべてのファイルが含まれます。そこからは、`cd`（ディレクトリを変更する）コマンドでクローンしたリポジトリに移動できます：

```sh
cd name-of-cloned-repo
```

さらに、ローカルのコピーにはGitHub上のコピーを指すリモートが設定されています。リモートを確認するには以下のコマンドを実行します。

```sh
git remote -v
```

もう一つのクローン方法はGitHub CLIを使うことです。このツールはコマンドラインを離れずにGitHub固有の作業を行えます。インストールしていない場合はGitHubのドキュメントで手順を確認してください。多くのシステムのパッケージマネージャーで利用可能です。

CLIツールでリポジトリをクローンするには、以下を使います。

```sh
gh repo clone your-username/your-repository-name
```

これらのコマンドの結果は同じで、ローカルにリポジトリのコピーができます。

GitHub CLIはリポジトリの作成にも使えます。

```sh
gh repo create
```

引数を渡さない場合、CLIは対話式ウィザードを表示し、リポジトリの設定を行えます。このウィザードはウェブUIと同じオプションを提供します。このツールでは新しいリポジトリを作成後すぐにクローンするオプションもあり、先ほど説明した手動の手順を省けます。

これでリポジトリの基本的な仕組みと、作成して作業のバージョン管理を始める方法がわかりました。今後のレッスンでは、既存のローカルリポジトリをGitHubにプッシュする方法や、リモートリポジトリに変更をプッシュする方法を学びます。
