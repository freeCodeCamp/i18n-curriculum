# --description--

現在你已準備好在你的電腦上安裝 Node.js，踏出進入 Node.js 世界的第一步。但首先，在你安裝 Node.js 之前，我們建議安裝一個名為 NVM（Node Version Manager）的工具。顧名思義，NVM 是一個用於在同一台電腦上管理多個 Node.js 版本的工具。

從一開始設定這個工具將有助於你未來，因為你可能需要在不同的專案中切換不同版本的 Node.js。使用 NVM，你可以非常輕鬆地做到這一點。

讓我們來看看你如何在 macOS/Linux 和 Windows 上安裝 NVM，以及你如何使用它來安裝 Node.js。

## 如何在 macOS/Linux 上安裝 NVM

在 macOS 和 Linux 上，你可以直接透過 shell 腳本安裝 NVM。

但對於 macOS，請注意 NVM 的官方文件提到「你需要在執行安裝腳本之前手動安裝 Xcode 命令列工具，否則會失敗。」

在 macOS 上，你可以透過在你的終端機中執行此命令來安裝 Xcode 命令列工具：

```bash
xcode-select --install
```

安裝完成後，你可以使用此指令檢查版本號並驗證安裝。

```bash
xcode-select --version
```

接著，就是安裝 NVM 的時候了。你可以透過執行以下其中一個指令來下載並執行 NVM 安裝腳本：

```bash
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.3/install.sh | bash
```

```bash
wget -qO- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.3/install.sh | bash
```

若你在閱讀本文時想取得最新版本的 NVM，請參考官方 NVM 文件：`https://github.com/nvm-sh/nvm?tab=readme-ov-file#install--update-script`。在那裡，你會找到帶有最新版本號的這些指令。

當處理程序完成時，你可能需要重新啟動你的終端機以載入新的指令和新的設定。

然後，執行此命令以確認 NVM 是否已正確安裝，並檢查你目前的版本：

```bash
nvm -v
```

你應該會在輸出中看到你的 NVM 版本。

## 如何在 macOS/Linux 上使用 NVM 安裝 Node.js

既然你已經在 macOS/Linux 上安裝了 NVM，你就可以安裝 Node.js。

要安裝最新的長期支援（LTS）版本的 Node.js，請執行此指令：

```bash
nvm install --lts
```

請注意 `--lts` 旗標。此旗標顯示你想安裝目前的長期支援（LTS）版本的 Node.js。此版本通常建議使用，因為它優先考量穩定性、可靠性和安全性，而非新的或實驗性的特性。它經過徹底測試並準備好用於生產環境。它也保證有較長的支援期限，重點在於錯誤修正和安全性修補。

或者，你可以透過在指令中指定版本號來安裝特定版本的 Node.js：

```bash
nvm install <version>
```

例如，這是你如何安裝 Node.js v20：

```bash
nvm install 20
```

當你完成安裝某個版本的 Node.js 後，可以使用此指令切換到該版本：

```bash
nvm use <version>
```

例如，這是如何切換到版本 20：

```bash
nvm use 20
```

最後，你可以使用此指令檢查目前的 Node.js 版本：

```bash
node -v
```

## 如何在 Windows 上安裝 NVM

現在讓我們看看你如何在 Windows 上安裝和使用 NVM。

NVM 的正式版本僅在特定情況下支援 Windows。如果你的裝置已設定 Windows Subsystem for Linux（WSL），你可以依照上述 macOS/Linux 指南來安裝和使用 NVM。

然而，對於 Windows 本身，有幾個替代方案。其中之一是 `nvm-windows`，一個用於 Windows 的 Node.js 版本管理工具。這是與官方 NVM 專案完全獨立的專案。但這些工具的目的相同——幫助你在同一裝置上使用多個版本的 Node.js。

在安裝 `nvm-windows` 之前，你應該知道官方儲存庫建議在安裝 NVM for Windows 之前先卸載任何先前的 Node 安裝。如果這是你第一次安裝 Node.js，應該沒問題。如果你已經安裝了其他版本的 Node.js，建議先將它們卸載。

要安裝 `nvm-windows`：

- 前往官方儲存庫的 Releases 區段：`https://github.com/coreybutler/nvm-windows/releases`。
- 下載最新的安裝程式。
- 遵循指示。

完成安裝後，如果 `nvm` 指令無法立即使用，你可能需要重新啟動你的終端機。

你可以透過執行此指令來檢查安裝是否成功並查看你目前的 `nvm-windows` 版本：

```powershell
nvm -v
```

## 如何使用 NVM for Windows 安裝 Node.js

既然你已經在 Windows 上安裝了 `nvm-windows`，你就可以準備安裝 Node.js。

要安裝最新的長期支援（LTS）版本的 Node.js，請執行此指令：

```powershell
nvm install lts
```

通常建議使用長期支援（LTS）版本，但你可以透過在指令中指定其版本號來安裝不同版本：

```powershell
nvm install <version>
```

例如：

```powershell
nvm install 20
```

當安裝完成後，你可以使用此指令切換到該版本的 Node.js：

```powershell
nvm use <version>
```

例如：

```powershell
nvm use 20
```

你也可以使用此指令檢查你目前的 Node.js 版本：

```bash
node -v
```

## 基本指令

既然你已經知道如何安裝 NVM 和 Node.js，讓我們來看看一些在使用 Node.js 和 NVM 時會覺得有用的基本指令。

此外，雖然所有這些指令都適用於安裝在 macOS／Linux（包括 WSL）上的 NVM，但它們在 `nvm-windows` 上可能會有不同的執行方式，甚至可能完全無法使用。我們會盡可能提供 `nvm-windows` 的替代指令。

### 如何調整 Powershell 的執行政策

如果你正在執行 Windows 並且正在使用 Powershell 和 `nvm-windows`，你可能需要調整 Powershell 的執行政策。Powershell 的預設執行政策是 `Restricted`，這是一項防止腳本執行的安全措施。

首先，右鍵點擊 Powershell，然後點擊「以系統管理員身分執行」來以系統管理員身分執行 Powershell。

然後，若要查看目前 Powershell 工作階段的執行政策，請執行以下命令：

```powershell
Get-ExecutionPolicy
```

最後，若要將 Powershell 的執行政策設定為 `RemoteSigned`，以允許區域的腳本和 `npm` 執行，請使用以下指令：

```powershell
Set-ExecutionPolicy RemoteSigned
```

在你執行上述指令後，你可以關閉 Powershell 的管理員視窗，並開啟非管理員 Powershell 視窗。

### 列出所有版本

要列出所有已安裝的 Node.js 版本並檢查目前正在使用的版本，請執行此命令：

```bash
nvm ls
```

### 設定預設的 Node.js 版本（僅限 macOS/Linux 上的 NVM）

此指令會設定在新的終端機工作階段中將使用的 Node.js 預設版本：

```bash
nvm alias default <version>
```

例如：

```bash
nvm alias default 22.20.0
```

你也可以創建訂製別名，為版本指定更具體的名稱：

```bash
nvm alias <new_name> <version>
```

例如：

```bash
nvm alias new-api 18.20.8
```

設定別名後，你可以使用這個新名稱來切換到該版本：

```bash
nvm use <new_name>
```

例如：

```bash
nvm use new-api
```

### 移除 Node.js 的版本

要移除 Node.js 的某個版本，請執行此指令：

```bash
nvm uninstall <version>
```

例如：

```bash
nvm uninstall 20.19.5
```

使用 macOS/Linux 的 NVM，你可以透過執行帶有主版本號如 `22` 或 `20` 的 `uninstall` 指令來移除 Node.js 的版本。例如：

```bash
nvm uninstall 20
```

如果你安裝了該主要版本的多個發行版本，例如 `20.14.0` 和 `20.12.2`，將會先移除最新版本。在這種情況下，執行 `nvm uninstall 20` 將會先移除 `20.14.0`。

### 檢查你安裝的 Node.js 版本

要檢查你目前的 Node.js 版本，你可以使用以下指令：

```bash
node -v
```

### 檢查你使用的 npm 版本

`npm` 是 Node.js 的套件管理工具。你可以使用它來安裝、發佈和管理軟體套件。當你安裝 Node.js 時，它會自動安裝。

要檢查你目前的 `npm` 版本，請執行此指令：

```bash
npm -v
```

### 執行 JavaScript 檔案

此指令會在目前作用中的 Node.js 執行期中執行你指定的 JavaScript 檔案：

```bash
node <file.js>
```

例如：

```bash
node app.js
```

### 創建 Node.js 專案

此指令透過創建一個 `package.json` 檔案來創建一個新的 Node.js 專案，該檔案用於追蹤其詳細資訊和相依性。

```bash
npm init
```

當你執行 `npm init` 指令時，它會引導你透過互動式 CLI 指南來設定 `package.json` 檔案。它會詢問你專案名稱、描述、版本等問題。如果你想從預設的 `package.json` 檔案開始，你可以使用 `npm init -y` 或 `npm init --yes` 指令。

### 安裝軟體包

要安裝特定的軟體包，請執行：

```bash
npm install <package>
```

例如：

```bash
npm install express
```

### 安裝相依套件

要安裝專案中 `package.json` 檔案列出的所有相依性，請執行：

```bash
npm install
```
