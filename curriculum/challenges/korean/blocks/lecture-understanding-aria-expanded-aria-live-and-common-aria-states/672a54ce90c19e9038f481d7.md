# --interactive--

`aria-expanded` 속성은 접근성을 위해 사용되며, 컨트롤이 확장되었는지 축소되었는지를 나타냅니다. 메뉴, 아코디언, 그리고 콘텐츠 가시성을 제어하는 기타 공개 위젯과 같은 접을 수 있는 위젯과 함께 사용됩니다. `aria-expanded` 속성은 컨트롤이 확장되었으면 `true`로, 축소되었으면 `false`으로 설정됩니다.

`aria-expanded`가 제공하는 정보는 화면 읽기 프로그램을 사용하는 사람들이 컨트롤의 현재 상태(확장되었는지 축소되었는지)를 이해하는 데 도움을 줍니다.

`aria-expanded` 속성은 접을 수 있는 위젯의 가시성을 토글하는 인터랙티브 요소에 적용됩니다. 예를 들어, 버튼이 확장 가능한 메뉴를 토글한다면 `aria-expanded` 속성이 버튼에 적용됩니다.

메뉴가 확장되었을 때는 `aria-expanded` 속성이 이 예시처럼 `true`로 설정되어야 합니다:

```html
<button aria-expanded="true">Menu</button>
```

메뉴가 축소되었을 때는 대신 `false`로 설정되어야 합니다.

```html
<button aria-expanded="false">Menu</button>
```

컨트롤하는 요소에는 항상 `aria-expanded` 속성이 `true` 또는 `false` 중 하나로 설정되어 있어야 합니다. 예를 들어, 버튼이 메뉴의 가시성을 토글한다면 `aria-expanded`의 기본값은 메뉴의 기본 가시성에 따라 정해집니다.

메뉴가 기본적으로 확장되어 있다면 `aria-expanded`는 처음에 `true`로 설정되어야 합니다. 메뉴가 기본적으로 축소되어 있다면 `aria-expanded`는 처음에 `false`로 설정되어야 합니다.

`aria-expanded`의 값은 사용자가 요소와 상호작용할 때 JavaScript를 사용해 동적으로 업데이트되어야 합니다.

또한 `aria-controls`와 `aria-owns` 속성은 `aria-expanded`과 함께 사용되어 컨트롤하는 요소와 제어되는 요소 사이에 프로그래밍적 연결을 설정할 수 있습니다.

`aria-controls`부터 시작해 보겠습니다. `aria-expanded`과 함께 사용될 때 `aria-controls` 속성은 제어되는 요소를 지정하는 데 사용됩니다. 예를 들어, 버튼이 메뉴 역할을 하는 목록을 확장하거나 축소할 수 있습니다. `aria-controls`의 값은 제어되는 요소(이 예시에서는 메뉴 목록)의 `id`가 됩니다.

```html

<button aria-expanded="false" aria-controls="menu1">Menu</button>
<ul id="menu1" hidden>
  <li>...</li>
  <li>...</li>
</ul>

<script src="index.js"></script>
```

```js
const button = document.querySelector('button');
const menu = document.getElementById(button.getAttribute('aria-controls'));

button.addEventListener('click', () => {
  const expanded = button.getAttribute('aria-expanded') === 'true';
  button.setAttribute('aria-expanded', String(!expanded));
  menu.hidden = expanded;
});

```

목록이 컨트롤하는 버튼 바로 다음에 위치하는 것을 확인하세요. 이처럼 확장 가능한 컨트롤의 경우, 확장된 콘텐츠가 DOM에서 그것을 제어하는 요소 바로 뒤에 위치하는 것이 가장 좋습니다. 이렇게 하면 화면 읽기 프로그램 사용자가 확장된 콘텐츠를 찾기 위해 헤매지 않아도 되고, 키보드 사용자가 확장된 콘텐츠 내의 인터랙티브 컨트롤을 더 쉽게 탐색할 수 있습니다.

확장된 콘텐츠를 컨트롤하는 요소 바로 뒤에 배치할 수 없다면 `aria-owns` 속성을 사용해 접근성 트리에서 가상으로 그 위치를 옮길 수 있습니다. 이렇게 하면 화면 읽기 프로그램 같은 보조 기술이 확장된 콘텐츠가 DOM에서 컨트롤 바로 뒤에 있는 것처럼 인식할 수 있습니다.

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<button aria-owns="menu1" aria-expanded="true">Menu</button>
<main>
  <!-- an entire page worth of content --->
</main>
<ul id="menu1">
  <li>...</li>
  <li>...</li>
</ul>

<script src="index.js"></script>
```

```css
  border: none;
  border-radius: 4px;
  cursor: pointer;
  margin-bottom: 10px;

  background-color: #005ea2;
}

  list-style: none;
  padding: 0;
  margin: 8px 0;
  border: 1px solid #ccc;
  border-radius: 4px;
  width: 150px;
  background: #fff;

  z-index: 10;
}

li {
  padding: 8px;
  cursor: pointer;
}

}

```

```js
const button = document.querySelector('button');
const menu = document.getElementById(button.getAttribute('aria-owns'));

button.addEventListener('click', () => {
  const expanded = button.getAttribute('aria-expanded') === 'true';
  button.setAttribute('aria-expanded', String(!expanded));
  menu.hidden = expanded;
});

// Close the menu if user clicks outside
document.addEventListener('click', (e) => {
  if (!button.contains(e.target) && !menu.contains(e.target)) {
    button.setAttribute('aria-expanded', 'false');
    menu.hidden = true;
  }
});

```

`aria-owns` 속성을 사용하는 데는 단점이 있습니다. 대부분의 화면 읽기 프로그램은 확장 시 확장된 요소의 전체 내용을 자동으로 읽어주기 때문에, 이 속성은 화면 읽기 프로그램 사용자에게 불필요한 장황함을 유발합니다. 또한 탭 순서를 변경하지 않으므로, JavaScript로 탭 순서를 관리하지 않으면 키보드 사용자가 확장된 콘텐츠에 도달하기 전에 페이지 내 다른 모든 인터랙티브 컨트롤을 탭해야 할 수 있습니다.

이상적으로는 확장 가능한 콘텐츠를 컨트롤 요소 뒤에 배치하고, `aria-owns` 속성은 그럴 수 없는 최악의 경우에만 사용해야 합니다. 사용해야 한다면, 다양한 화면 읽기 프로그램과 브라우저에서 철저히 테스트하여 모든 사용자가 접근할 수 있도록 해야 합니다.

그리고 `aria-controls` 또는 `aria-owns`을 사용할 때는 컨트롤이 확장되고 축소될 때마다 `aria-expanded`의 값이 계속 업데이트되어야 한다는 점을 기억하세요.

`aria-expanded` 속성은 컨트롤이 확장되었는지 축소되었는지를 나타냅니다. 이 정보는 화면 읽기 프로그램 사용자에게 메뉴, 아코디언, 기타 유사한 공개 위젯과 같은 접을 수 있는 요소의 현재 상태를 이해하는 데 필수적입니다.

`aria-expanded`를 올바르게 사용하면 모두에게 직관적인 사용자 경험을 만들 수 있습니다.
