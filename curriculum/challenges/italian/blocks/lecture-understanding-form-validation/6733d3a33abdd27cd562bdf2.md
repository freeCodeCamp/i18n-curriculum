# --interactive--

Impariamo qual è lo scopo del metodo `preventDefault()` sugli eventi.

Ogni evento che si attiva nel DOM ha un certo comportamento predefinito. L'evento `click` su una casella di spunta cambia lo stato di quella casella, per impostazione predefinita. Premere la barra spaziatrice su un pulsante con focus attiva il pulsante. Il metodo `preventDefault()` su questi oggetti evento impedisce che quel comportamento avvenga.

Diamo un’occhiata a un esempio. Definiamo un elemento `input` in cui un utente può digitare:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<label>Enter some characters:
  <input type="text">
</label>
```

```css
label {
  display: flex;
  flex-direction: column;
  font-family: Arial, sans-serif;
  font-size: 14px;
  color: #333;
  margin-bottom: 12px;
}

input[type="text"] {
  padding: 8px 12px;
  border: 1px solid #ccc;
  border-radius: 4px;
  font-size: 14px;
  margin-top: 4px;
  width: 100%;
  box-sizing: border-box;
  transition: border-color 0.2s, box-shadow 0.2s;
}

input[type="text"]:focus {
  border-color: #0078d4;
  box-shadow: 0 0 3px rgba(0, 120, 212, 0.5);
  outline: none;
}

```

:::

E se guardiamo il risultato, possiamo digitare nel campo di input come previsto. Ma forse non vogliamo questo. Forse, invece, vogliamo mostrare il carattere che l’utente digita in un elemento separato. Prima, definiamo il nostro elemento per questo:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<label>Enter some characters:
  <input type="text">
</label>
<p id="output"></p>
```

```css

  align-items: center;
  justify-content: center;

  margin: 0;

  font-size: 1.2rem;
  color: #333;
  margin-bottom: 1rem;
  display: flex;
  flex-direction: column;
}

  padding: 0.5rem;
  font-size: 1rem;
  border: 2px solid #ccc;
  border-radius: 5px;
  margin-top: 0.5rem;
  outline: none;
}

input[type="text"]:focus {
  border-color: #007BFF;
}

#output {
  margin-top: 1rem;
  font-size: 1.1rem;
  color: #555;
}

```

Poi, dobbiamo agganciarci all’evento `keydown` per ascoltare quando viene digitato un carattere sulla tastiera. Nota che qui non vogliamo gli eventi `change` o `input`, perché abbiamo bisogno delle informazioni della tastiera.

```js
const input = document.querySelector("input");

input.addEventListener("keydown", (e) => {
    
})
```

L’evento `keydown` si attiva quando premi un tasto sulla tastiera. Quando succede, mostriamo il carattere nel nostro elemento `p`.

```js

input.addEventListener("keydown", (e) => {
  output.innerText = `You pressed the ${e.key} key`;
});
```

:::

`e.key` ti dà il valore del tasto premuto, come `a` per il tasto `a` o `Enter` per il tasto `Enter`.

Con il codice sopra, quando digiti nel `input`, il carattere che digiti verrà mostrato nell’elemento `p`.

Questo è ottimo, ma non vogliamo mostrare i caratteri anche nel `input`. Qui entra in gioco il nostro metodo `preventDefault()`. Il comportamento predefinito di un `keydown` è renderizzare il carattere nell’input. Evitiamo questo chiamando `e.preventDefault()`:

```html
  <input type="text">
</label>

```

```css
  margin: 0;

  color: #333;
  margin-bottom: 1rem;
  display: flex;
  flex-direction: column;
}

input[type="text"] {
  padding: 0.5rem;
  font-size: 1rem;
  border: 2px solid #ccc;
  border-radius: 5px;
  margin-top: 0.5rem;
  outline: none;
}

input[type="text"]:focus {
  border-color: #007BFF;
```

```js
const input = document.querySelector("input");
const output = document.getElementById("output");

input.addEventListener("keydown", (e) => {
  e.preventDefault();
  output.innerText = `You pressed the ${e.key} key`;
});
```

:::

E così, hai impedito il comportamento predefinito per poterti implementare una gestione personalizzata dell’evento.

Un altro esempio comune di quando usare il metodo `e.preventDefault` riguarda gli invii di modulo. Per impostazione predefinita, inviare un modulo manda i dati al server e ricarica la pagina. Usare `e.preventDefault()` impedisce che questo accada.

```js
const form = document.querySelector("form");

form.addEventListener("submit", (e) => {
  e.preventDefault();
  // rest of code goes here
});
```

Impedire il comportamento predefinito è ottimo quando hai bisogno di più controllo su come un utente interagisce con la pagina, ma è importante tenere a mente l’accessibilità – il tuo comportamento personalizzato dovrebbe offrire le stesse funzionalità di quello predefinito.
