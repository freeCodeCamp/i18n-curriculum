# --description--

在 Express 中，**路由路徑** 是決定伺服器如何回應進來請求的 URL 樣式。當使用者請求特定 URL 時，Express 會將該 URL 與路由路徑比對，並執行對應的處理常式。

路由路徑定義在像 `app.get()`、`app.post()` 等路由方法中。這些路徑可以是靜態的，例如 `/home`，也可以是動態的，使用參數從 URL 捕捉值，像 `/user/:id`。

本課程將專注於靜態路由。以下是一個範例：

```javascript
// Static route
app.get("/home", (req, res) => {
  res.send("Welcome to the homepage!");
});
```

在這個範例中，伺服器監聽對 `/home` 的 GET 請求，並回應一則簡單的訊息。

`app.get()` 方法接受兩個引數：

* 第一個引數 `"/home"` 是伺服器監聽的靜態路由路徑（或 URL）。
    
* 第二個引數是一個回呼函式，當對該路徑發出 GET 請求時會執行。
    

在回呼函式內，`res.send()` 傳送回應給客戶端。在此例中，它傳送文字 `Welcome to the homepage!`，會顯示在瀏覽器中。

這是另一個靜態路由的範例：

```javascript
// Static route
app.get("/menu/drinks", (req, res) => {
  res.send("Welcome to the drinks menu!");
});
```

在這個例子中，伺服器監聽對 `/menu/drinks` URL 的 GET 請求，並在該路由被請求時回傳訊息。

當你的網站有多個相關頁面歸類在共同路徑下時，可能會使用這種路由。例如，一個餐廳網站可以這樣組織它的選單頁面：

* `/menu/food`
    
* `/menu/drinks`
    
* `/menu/desserts`
    

這種做法有助於保持你的 URL 清晰且有意義，也讓你的路由在應用程式成長時更容易組織和維護。
