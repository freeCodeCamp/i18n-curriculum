# --description--

プロジェクト管理では、タスクの扱い方を説明するために特定のフレーズが使われます。

`break down`とは、何かをより小さく、管理しやすい部分に分割することを意味します。例えば、`When I work on a large project, I often break down the work into smaller sections`です。

`set deadlines`とは、タスクを完了する必要がある時間を決めることを意味します。例えば、`Setting deadlines for each task ensures that the project stays on schedule`です。

これらのフレーズを理解することで、プロジェクトの組織の仕方がわかりやすくなります。
