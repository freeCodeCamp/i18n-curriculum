# --explanation--

Significa planejar alcançar um objetivo ou meta ao `aim`. Por exemplo:

`We aim to release the app next month.` — Eles esperam ou planejam fazer isso.

`Beta testing` é quando usuários reais testam um produto quase finalizado para encontrar quaisquer bugs. Por exemplo:

`The app is now in beta testing.` — As pessoas estão testando o app para encontrar problemas.

`Critical issues` são problemas muito sérios que devem ser corrigidos antes do lançamento. Por exemplo:

`The app crashing on login is a critical issue.` — É um problema sério que bloqueia o uso.
