## 객체 지향 프로그래밍이란?

- **객체 지향 프로그래밍**: 개발자가 코드 내 모든 것을 실제 세계의 객체처럼 다루는 프로그래밍 스타일입니다. 흔히 OOP라고 부릅니다. 코드를 효과적으로 조직하고 관리하는 데 도움이 되는 네 가지 핵심 원칙은 **캡슐화**, **상속**, **다형성**, 그리고 **추상화**입니다.
- **클래스**: 객체를 생성하기 위한 청사진입니다. 클래스에서 생성된 모든 객체는 데이터를 정의하는 속성과 객체의 동작을 결정하는 메서드를 가집니다.

## 캡슐화란 무엇인가요?

- **캡슐화**: 객체의 속성과 메서드를 하나의 단위로 묶는 것입니다. 객체의 내부 상태를 단순한 공개 메서드와 속성 집합 뒤에 숨길 수 있게 해줍니다. 이 문 뒤에는 데이터 변경과 접근을 제어하는 비공개 속성과 메서드가 있습니다.
- **캡슐화 예시**: 지갑 잔액을 추적하려면 입금과 출금을 허용하지만, 지갑 잔액 자체를 누구나 조작하지 못하게 해야 합니다:

```py
class Wallet:
   def __init__(self, balance):
       self.__balance = balance # Private attribute

   def deposit(self, amount):
       if amount > 0:
           self.__balance += amount # Add to the balance safely

   def withdraw(self, amount):
       if 0 < amount <= self.__balance:
           self.__balance -= amount # Remove from the balance safely

account = Wallet(500)
print(account.__balance) # AttributeError: 'Wallet' object has no attribute '__balance'
```

- **단일 밑줄과 이중 밑줄 접두사의 차이**: 속성과 메서드에 단일 밑줄을 접두사로 붙이면 내부 용도임을 나타내는 관례입니다. 외부에서 속성에 접근하는 것을 강제하지는 않습니다. 이중 밑줄을 접두사로 붙이면 클래스 외부에서 접근하는 것을 효과적으로 막습니다.

## 게터와 세터란 무엇인가요?

- **게터와 세터**: 클래스의 속성에 어떻게 접근하고 수정할지 제어하는 메서드입니다. 게터로 값을 가져오고 세터로 값을 설정합니다.
- **속성(프로퍼티)**: 게터와 세터를 연결해 데이터를 접근할 수 있게 합니다. 값을 가져오거나 설정하거나 삭제할 때 추가 로직을 실행합니다.
- **메서드 대신 속성을 사용하는 이유**: 속성은 메서드보다 가독성이 좋고 코드를 깔끔하게 만듭니다. 소괄호 없이 점 표기법으로 일반 속성처럼 값을 접근할 수 있습니다.
- **게터 생성하기**: 게터를 만들려면 `@property` 데코레이터를 사용합니다. 다음은 원의 반지름을 가져오는 게터입니다:

```py
class Circle:
    def __init__(self, radius):
        self._radius = radius

    @property
    def radius(self): # A getter to get the radius
        return self._radius
  
    @property
    def area(self):  # A getter to calculate area
        return 3.14 * (self._radius ** 2)

my_circle = Circle(3)

print(my_circle.radius) # 3
print(my_circle.area) # 28.26
```

- **세터 생성하기**: 반지름을 설정하는 세터를 만들려면 같은 이름의 메서드를 정의하고 위에 `@<property_name>.setter`을 붙입니다:

```py
class Circle:
    def __init__(self, radius):
        self._radius = radius

    @property
    def radius(self):  # A getter to get the radius
        return self._radius

    @radius.setter
    def radius(self, value):  # A setter to set the radius
        if value <= 0:
            raise ValueError('Radius must be positive')
        self._radius = value

my_circle = Circle(3)
print('Initial radius:', my_circle.radius) # Initial radius: 3

my_circle.radius = 8
print('After modifying the radius:', my_circle.radius) # After modifying the radius: 8
```

- **파이썬이 게터와 세터를 처리하는 방법**: 게터와 세터를 정의하면, 파이썬은 일반 속성 구문을 사용할 때 자동으로 호출합니다:

```py
my_circle.radius # This will call the getter
my_circle.radius = 4 # This will call the setter
```

 값을 설정할 때는 속성 이름 자체에 할당하지 마세요. 그렇게 하면 `RecursionError`가 발생합니다. 보통 밑줄이 붙은 별도의 내부 이름에 값을 저장합니다.

- **삭제자(Deleter)**: 세터와 게터로 값을 설정하고 가져온 후, `deleter`로 삭제하는 방법도 제어할 수 있습니다. 삭제자는 속성에 `del` 문을 사용할 때 사용자 정의 로직을 실행합니다. 삭제자를 만들려면 `@<property_name>.deleter` 데코레이터를 사용합니다.

```py
  # Deleter
    @radius.deleter
    def radius(self):
        print("Deleting radius...")
        del self._radius
```

## 상속이란 무엇이며 코드 재사용에 어떻게 도움이 되나요?

- **상속**: 자식 클래스가 부모 클래스의 속성과 메서드를 사용하는 과정입니다. 상속은 코드 재사용을 촉진하고 명확한 계층 구조를 제공하며 모든 것을 다시 작성하지 않고 동작을 맞춤화할 수 있게 합니다. 상속을 구현하려면 자식 클래스가 부모 클래스 이름을 받습니다:

```py
class Parent:
    # Parent attributes and methods

class Child(Parent):
    # Child inherits, extends, and/or overrides where necessary
```

- **단일 상속과 다중 상속**: 자식 클래스가 단일 부모로부터 속성과 메서드를 상속받으면 **단일 상속**이라고 합니다. 여러 부모로부터 상속받으면 **다중 상속**입니다. 문법은 다음과 같습니다:

```py
class Parent:
    # Attributes and methods for Parent

class Child:
    # Attributes and methods for Child

class GrandChild(Parent, Child):
    # GrandChild inherits from both Parent and Child
    # GrandChild can combine or override behavior from each
```

- **`super()` 함수**: 클래스가 메서드를 다르게 구현하거나 확장할 때, 부모 클래스의 메서드를 중복 없이 **호출**할 수 있게 해주는 함수입니다.

## 다형성이란 무엇이며 코드 재사용에 어떻게 도움이 되나요?

- **다형성**: 서로 다른 클래스가 같은 메서드 이름을 사용하지만, 호출될 때 각 클래스가 다르게 구현하는 OOP 원칙입니다. 문법은 다음과 같습니다:

```py
class A:
   def action(self): ...

class B:
   def action(self): ...

class C:
   def action(self): ...

Class().method()  # Works for A, B, or C
```

- **상속 기반 다형성**: 부모가 메서드를 정의하고, 각 자식 클래스가 이를 자신에게 맞게 변형합니다.

## 이름 맹글링(Name Mangling)이란 무엇이며 어떻게 작동하나요?

- **이름 맹글링**: 파이썬이 이중 밑줄 접두사가 붙은 속성 이름을 내부적으로 클래스 이름과 밑줄을 접두사로 붙여 이름을 바꾸는 과정입니다. 예를 들어 `__attribute`가 `_ClassName__attribute`로 바뀝니다.
- **이름 맹글링의 목적**: 상속을 사용할 때 속성과 메서드가 실수로 덮어쓰이는 것을 방지하는 데 목적이 있습니다. 다음 코드를 보면 이해하기 쉽습니다:

```py
class Parent:
    def __init__(self):
        self.__data = 'Parent data'

class Child(Parent):
    def __init__(self):
        super().__init__()
        self.__data = 'Child data'

c = Child()
print(c.__dict__) # {'_Parent__data': 'Parent data', '_Child__data': 'Child data'}
```

## 추상화란 무엇이며 복잡한 시스템을 어떻게 정리하나요?

- **추상화**: 객체나 시스템의 복잡한 구현 세부사항을 숨기고 필수 기능만 보여주는 프로그래밍 개념입니다. 파이썬과 다른 언어에서 추상화는 재사용성을 높여 복잡한 시스템을 단순화합니다.
- **추상화 예시**: 일상생활에서 자동차는 엔진이나 브레이크 작동 방식을 몰라도 핸들, 페달, 변속기만 사용하게 해주는 좋은 예입니다.
- **파이썬에서 추상화 구현 방법**: 파이썬은 `abc` 모듈을 통해 추상화를 구현합니다. 이 모듈은 `ABC` 클래스(추상 기본 클래스)와 `@abstractmethod` 데코레이터를 제공합니다. 추상 기본 클래스(ABC)는 하위 클래스가 반드시 구현해야 하는 공통 메서드와 속성을 정의하며, 인스턴스화할 수 없습니다.
- **추상 메서드 정의 방법**: 추상 메서드는 `@abstractmethod`로 정의하며, 기본 구현이 있어도 하위 클래스에서 반드시 재정의해야 합니다. 추상화 기본 구문은 다음과 같습니다:

```py
from abc import ABC, abstractmethod

# Define an abstract base class
class AbstractClass(ABC):
    @abstractmethod
    def abstract_method(self):
        pass

# Concrete subclass that implements the abstract method
class ConcreteClassOne(AbstractClass):
    def abstract_method(self):
        print('Implementation in ConcreteClassOne')

# Another concrete subclass
class ConcreteClassTwo(AbstractClass):
    def abstract_method(self):
        print('Implementation in ConcreteClassTwo')
```
