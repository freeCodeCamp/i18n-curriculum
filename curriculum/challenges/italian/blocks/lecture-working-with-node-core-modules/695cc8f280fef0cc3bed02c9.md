# --description--

Il modulo OS è un altro modulo standard incluso in Node.js.

Ti permette di interagire con il sistema operativo attualmente in uso da Node, così puoi consultare informazioni vitali come il tipo di OS, i dettagli della CPU, la memoria disponibile, la memoria totale, le interfacce di rete e altro ancora.

Per usare il modulo OS, lo importi in questo modo:

```js
const os = require("os");
```

Il modulo OS fornisce diversi metodi che puoi usare per rendere la tua applicazione più consapevole del sistema. Vediamo alcuni dei più comuni.

Il metodo `platform()` recupera il sistema operativo su cui Node sta girando in quel momento:

```js
const os = require("os");
console.log(os.platform()); // darwin
```

Può essere utile per implementare script multipiattaforma:

```js
if (os.platform() === 'win32') {
 // Windows specific code
} else {
 // Non-Windows specific code
}
```

Il metodo `arch()` mostra una stringa che rappresenta l’architettura della CPU per cui Node.js è stato compilato:

```js
const os = require("os");
console.log(os.arch()); // arm64
```

Questo può essere utile se vuoi che gli utenti scarichino i binari e le dipendenze corrette per una data architettura.

Altri valori possibili sono `'x64'`, `'arm'`, `'arm64'`, `'ia32'`, `'mips'`, `'ppc'` e altri.

`type()` ottiene il nome ufficiale del sistema operativo, così puoi identificare programmaticamente i sistemi operativi:

```js
const os = require("os");
console.log(os.type()); // Darwin (the core OS for macOS, iOS, and other Apple products)
```

`release()` mostra la versione del kernel del sistema operativo, la parte centrale del sistema operativo che gestisce le risorse di sistema e la comunicazione tra hardware e software. Questo metodo può essere utile per monitorare la compatibilità tra versioni del kernel OS e requisiti del server.

Restituisce questo come stringa tipo `'20.6.0'`:

```js
const os = require("os");
console.log(os.release()); // 25.0.0
```

`version()` restituisce la versione specifica del sistema operativo con più dettagli rispetto al metodo `release()`:

```js
const os = require("os");
console.log(os.version());
// Darwin Kernel Version 25.0.0: Wed Sep 17 21:41:39 PDT 2025;
// root:xnu-12377.1.9~141/RELEASE_ARM64_T8103
```

`cpus()` restituisce un array di oggetti con dettagli su ogni core logico della CPU. Questo può aiutare a monitorare il carico della CPU:


`uptime()` mostra il tempo trascorso dall’avvio del sistema. Può aiutare a determinare da quanto tempo i server sono attivi:

```js
const os = require("os");
console.log(os.uptime()); // 23047
```

`totalmem()` e `freemem()` mostrano rispettivamente la quantità totale di memoria di sistema in byte e la memoria libera di sistema in byte:

```js
const os = require("os");
console.log(os.totalmem()); // 8589934592 (8 GB)
console.log(os.freemem()); // 93585408 (87 MB)
```

`userInfo()` restituisce un oggetto contenente informazioni sull’utente di sistema attuale:

```js
const os = require("os");
console.log(os.userInfo());

/*
[Object: null prototype] {
 uid: 502,
 gid: 20,
 username: 'user',
 homedir: '/Users/user',
 shell: '/bin/zsh'
}
*/
```
