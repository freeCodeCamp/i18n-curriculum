# --description--

GitHub Pages è un'opzione davvero solida per distribuire siti statici o applicazioni che non richiedono un server back-end per gestire la logica. Cioè, le applicazioni che girano interamente lato client, o nel browser dell'utente, possono essere completamente distribuite su questa piattaforma.

Inoltre, GitHub Pages è completamente gratuito per i progetti open source. Ci sono molte buone ragioni per rendere il tuo progetto open source, e Pages ne aggiunge un'altra.

Ma come si distribuisce effettivamente su GitHub Pages? Beh, dipende da come funziona la tua applicazione.

Iniziamo con una pagina HTML di base. Eccone una che puoi aggiungere al tuo repo esistente se vuoi:

```html
<!DOCTYPE html>
<html>
  <head>
    <title>freeCodeCamp</title>
    <link rel="stylesheet" href="./style.css" />
  </head>
  <body>
    <h1>Welcome to freeCodeCamp's Lessons!</h1>
    <p>Let's have some fun learning about Git and GitHub!</p>
  </body>
</html>
```

Ora dobbiamo visitare il nostro repository e cliccare sulla scheda "Settings" per andare alle impostazioni del repository. Poi dobbiamo selezionare "Pages" dalla barra laterale.

All'inizio ci sono solo poche opzioni, ma questo cambierà man mano che inizieremo a configurare le impostazioni. Prima di tutto, dobbiamo dire a GitHub cosa distribuire.

Vogliamo lasciare questa impostazione su "Deploy from a branch", perché non stiamo usando un framework e non abbiamo configurato alcuna GitHub action. Imparerai di più su cosa sono in una lezione futura.

Poi, imposta il branch da cui stiamo distribuendo su `main` per distribuire ciò che c'è su quel branch.

Ti verrà quindi chiesto di scegliere se distribuire dalla cartella radice del progetto o da una cartella `docs`. Questa funzionalità è usata da progetti come freeCodeCamp - usavamo includere la documentazione con la nostra app, così potevamo distribuire solo la documentazione su GitHub Pages tramite la cartella `docs`. Per ora, lasciala sulla cartella `/` (radice) e clicca sul pulsante "Save" accanto a questo menu a tendina per salvare le modifiche.

Una volta salvato, vedrai un nuovo input per configurare un dominio personalizzato. Se ne hai già uno e vuoi usarlo, puoi seguire il collegamento alla documentazione per farlo. Per questa lezione, però, lo lasceremo vuoto.

Puoi anche vedere il dominio a cui è stato distribuito il tuo nuovo sito GitHub Pages. Senza un dominio personalizzato, questo di default sarà `<username>.github.io`, dove `<username>` è il nome del tuo account (o dell'organizzazione se stai distribuendo da un repository di un'organizzazione). E il repository sarà servito su un percorso che corrisponde al nome del repository, `<username>.github.io/<repo-name>`, per esempio. Se visiti l'URL, vedrai il tuo sito!

Ora, quando fai push di nuove modifiche sul branch `main`, GitHub Pages aggiornerà automaticamente il sito distribuito con le ultime modifiche. Questo processo può richiedere qualche minuto, a seconda del carico del server - e potresti dover fare un aggiornamento forzato per cancellare la cache.

E con questo, abbiamo distribuito con successo una semplice applicazione web su GitHub Pages. Puoi anche includere file CSS e JavaScript nella tua distribuzione Pages, così puoi costruire siti belli e interattivi. E una volta che impari un framework come React o Angular, puoi distribuire anche quelli su GitHub Pages!
