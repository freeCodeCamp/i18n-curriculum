## JavaScript의 세트

- `Set`는 데이터 컬렉션을 관리하는 내장 옵션입니다.
- 세트는 그 안의 각 값이 한 번만 나타나도록 보장하여 배열에서 중복을 제거하거나 고유한 값들의 컬렉션을 다룰 때 유용합니다.
- `Set` 생성자를 사용해 `Set()`를 만들 수 있습니다:

:::interactive_editor

```js
const set = new Set([1, 2, 3, 4, 5]);
console.log(set); // Set { 1, 2, 3, 4, 5 }
```

:::

- 세트는 다음 메서드로 조작할 수 있습니다:

  - `add()`: `Set`에 새 요소를 추가합니다.
  - `delete()`: `Set`에서 요소를 제거합니다.
  - `has()`: `Set`에 요소가 존재하는지 확인합니다.
  - `clear()`: `Set`의 모든 요소를 제거합니다.
  - `keys()`와 `values()`: 둘 다 `SetIterator`의 값을 포함하는 `Set`를 반환합니다. `keys()`가 `values()`의 별칭이기 때문에 동일합니다.
  - `forEach()`: `Set`의 값을 반복하는 데 사용합니다.

## JavaScript의 Weakset

- `WeakSet`는 약한 참조로 객체를 저장할 수 있는 객체 컬렉션입니다.

## 세트와 Weakset 비교

- 세트와 달리 `WeakSet`는 숫자나 문자열 같은 원시 타입을 지원하지 않습니다.
- `WeakSet`는 객체만 저장하며, 그 객체에 대한 참조가 "약한" 참조이므로 코드 내 다른 곳에서 사용되지 않으면 자동으로 제거되어 메모리를 확보합니다.

## JavaScript의 맵

- `Map`는 객체처럼 키-값 쌍을 저장하는 내장 객체입니다.
- 맵은 키로 객체나 함수 등 어떤 타입도 허용할 수 있다는 점에서 표준 JavaScript 객체와 다릅니다.
- `Map`는 키-값 쌍을 자주 추가하거나 제거할 때 표준 객체보다 더 나은 성능을 제공합니다.
- `Map` 생성자를 사용해 `Map()`를 만들 수 있습니다:

:::interactive_editor

```js
const map = new Map([
  ['flower', 'rose'],
  ['fruit', 'apple'],
  ['vegetable', 'carrot']
]);
console.log(map); // Map(3) { 'flower' => 'rose', 'fruit' => 'apple', 'vegetable' => 'carrot' }
```

:::

- 맵은 다음 메서드로 조작할 수 있습니다:

  - `set()`: `Map`에 새 키-값 쌍을 추가합니다.
  - `get()`: `Map`에서 키의 값을 가져옵니다.
  - `delete()`: `Map`에서 키-값 쌍을 제거합니다.
  - `has()`: `Map`에 키가 존재하는지 확인합니다.
  - `clear()`: `Map`의 모든 키-값 쌍을 제거합니다.

맵과 세트 모두 고유한 요소 수를 반환하는 `size` 속성을 가지고 있다는 점에 유의하세요.

## JavaScript의 WeakMap

- `WeakMap`는 `Map`처럼 키-값 쌍 컬렉션이지만 키에 대해 약한 참조를 가집니다. 키는 객체여야 하며 값은 원하는 어떤 것이든 될 수 있습니다.

## 맵과 WeakMap 비교

- WeakMap은 WeakSet과 비슷하게 객체만 저장하며 그 객체에 대한 참조가 "약한" 참조입니다.
