# --description--

OS 模組是另一個內建於 Node.js 的標準模組。

它讓你與目前 Node 正在執行的作業系統互動，因此你可以存取重要的資訊，例如作業系統型別、CPU 詳細資料、可用記憶體、總記憶體、網路介面等等。

要使用 OS 模組，你可以這樣匯入它：

```js
const os = require("os");
```

`OS` 模組提供了幾個你可以使用的行為，讓你的應用程式更具系統感知能力。讓我們來看看一些常見的行為。

`platform()` 方法會擷取 Node 目前正在執行的作業系統：

```js
const os = require("os");
console.log(os.platform()); // darwin
```

它對於實作跨平台腳本可能很有用：

```js
if (os.platform() === 'win32') {
 // Windows specific code
} else {
 // Non-Windows specific code
}
```

`arch()` 方法會顯示一個表述 Node.js 編譯所用 CPU 架構的字串：

```js
const os = require("os");
console.log(os.arch()); // arm64
```

如果你想讓使用者下載特定架構的正確二進位檔和相依性，這會很有用。

其他可能的值有 `'x64'`、`'arm'`、`'arm64'`、`'ia32'`、`'mips'`、`'ppc'` 等。

`type()` 會取得官方作業系統名稱，因此你可以以程式方式識別作業系統：

```js
const os = require("os");
console.log(os.type()); // Darwin (the core OS for macOS, iOS, and other Apple products)
```

`release()` 顯示系統的 OS kernel 版本，作業系統中管理系統資源以及硬體和軟體組件之間通訊的核心部分。這個方法對於追蹤 OS kernel 版本與伺服器需求之間的相容性非常有用。

它會傳回這個字串，格式如 `'20.6.0'`：

```js
const os = require("os");
console.log(os.release()); // 25.0.0
```

`version()` 傳回比 `release()` 方法更詳細的特定作業系統版本：

```js
const os = require("os");
console.log(os.version());
// Darwin Kernel Version 25.0.0: Wed Sep 17 21:41:39 PDT 2025;
// root:xnu-12377.1.9~141/RELEASE_ARM64_T8103
```

`cpus()` 傳回一個包含每個邏輯 CPU 核心詳細資訊的物件陣列。這可以幫助監控 CPU 載入：


`uptime()` 顯示系統開機以來的時間。它可以幫助判斷伺服器已經執行了多久：

```js
const os = require("os");
console.log(os.uptime()); // 23047
```

`totalmem()` 和 `freemem()` 分別顯示系統記憶體的總位元組數和可用記憶體的位元組數：

```js
const os = require("os");
console.log(os.totalmem()); // 8589934592 (8 GB)
console.log(os.freemem()); // 93585408 (87 MB)
```

`userInfo()` 傳回一個包含目前系統使用者資訊的物件：

```js
const os = require("os");
console.log(os.userInfo());

/*
```
