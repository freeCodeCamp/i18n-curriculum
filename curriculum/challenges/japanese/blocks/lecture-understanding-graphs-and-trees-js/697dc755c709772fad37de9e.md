# --description--

グラフは、ノード（頂点とも呼ばれます）とそれらをつなぐエッジの集合で構成される非常に強力なデータ構造です。

グラフをコードで実装する一般的な方法は2つあります：

* 隣接行列
    
* 隣接リスト
    

これらについて詳しく見ていき、それぞれの利点と制限を説明します。

## 隣接行列

まずは隣接行列から始めましょう。

隣接行列は、行と列がグラフの頂点を表す二次元配列です。

行列の値はノード間のエッジや接続を表します。

例えば、`matrix`という変数に格納された行列がある場合、`matrix[i][j]`が行、`i`が列の`j`に格納された値は、ノード`i`とノード`j`の間のエッジや接続を表します。

値はグラフが重み付きか無重みかによって意味が異なります：

* グラフが無重みの場合、`1`の値はこれらのノード間にエッジがあることを意味し、`0`の値はエッジがないことを意味します。
    
* グラフが重み付きの場合、その値はノードをつなぐエッジの重みを表します。

隣接行列を使う大きな利点の一つは、2つのノード間にエッジがあるかどうかを定数時間`O(1)`で確認できることです。これはプログラムが2D配列の特定の値を見つけるだけで済むためです。

しかし、このエッジ検出の効率には代償があります。隣接行列はグラフのノード数`O(V²)`の二乗に比例する大きな二次的な空間を必要とします`V`。

これはエッジが少ないスパースグラフには非効率的です。なぜなら、スパースグラフではノード間にエッジがないことを表す多くの0を行列に格納する必要があり、これらの0もメモリ空間を消費するからです。

また、隣接行列はノードの隣接ノードを見つけるのにも非効率的です。プログラムはエッジを表す0や1を見つけるために行または列全体を繰り返す必要があり、最悪の場合、グラフのノード数`O(V)`に比例する`V`の時間がかかります。

この特定のグラフの隣接行列の例を見てみましょう：

<img src="https://cdn.freecodecamp.org/curriculum/lecture-transcripts/how-do-matrices-and-adjacency-lists-work-1.png" alt="A graph with four nodes labeled A, B, C, and D. Node A connects to B, C, and D. Node B connects to A and D. Node C connects to A. Node D connects to A and B.">

隣接行列では：

* 各行はノードを表します。最初の行はノードA、2行目はノードB、以下同様です。
    
* 各列もノードを表します。
    
* 行列の各値はノードのペア間にエッジがあるかどうかを表します。0はエッジがないこと、1はエッジがあることを意味します。
    
対角線上の値は各ノードが自己ループ（ノード自身に接続するエッジ）を持つかどうかを表します。例では自己ループがないため、すべて0です。

これは行と列が対応するノードをどのように表すかを示す隣接行列の視覚的な表現です。

例えば、最初の行は`[0, 1, 1, 1]`です。なぜならノードAはノードB、C、Dにエッジでつながっているからです：

```markdown
#      A  B  C  D
# A   [0, 1, 1, 1],
# B   [1, 0, 0, 1],
# C   [1, 0, 0, 0],
# D   [1, 1, 0, 0]
```

そしてこれは同じ隣接行列をJavaScriptコードで実装したものです：

```javascript
const adjacencyMatrix = [
  [0, 1, 1, 1], // The neighbors of A are B, C, and D
  [1, 0, 0, 1], // The neighbors of B are A and D
  [1, 0, 0, 0], // The only neighbor of C is A
  [1, 1, 0, 0]  // The neighbors of D are A and B
];
```

## 隣接リスト

グラフを表すもう一つの一般的な方法は隣接リストを使うことです。

隣接リストは、各ノードのすべての隣接ノードを格納する配列やオブジェクト（または`Map`）です。

隣接リストの実装方法は2つあります：

* 配列として、各インデックスがノードを表し、そのインデックスに格納された配列が隣接ノードを含む。
* オブジェクト（または`Map`）として、各キーがノードを表し、そのキーに対応する値（配列）が隣接ノードを含む。

隣接リストは隣接行列よりも空間効率が良く、空間計算量は`O(V + E)`で、`V`が頂点（ノード）数、`E`がエッジ数です。

また、隣接ノードをすべて見つける操作も効率的です。この操作はノードに対応するリストにアクセスするだけで済みます。

しかし、ここにもトレードオフがあります。

隣接リストは2つのノード間にエッジがあるかどうかを判定するのに隣接行列より非効率的です。

検索処理は通常`O(deg(v))`の時間がかかり（`deg(v)`は調べるノードの隣接ノード数）、最悪の場合、ノードがグラフ内のすべての他のノードに接続されていると`O(V)`になることがあります。

このグラフの隣接リストの例を示します：

<img src="https://cdn.freecodecamp.org/curriculum/lecture-transcripts/how-do-matrices-and-adjacency-lists-work-1.png" alt="A graph with four nodes labeled A, B, C, and D. Node A connects to B, C, and D. Node B connects to A and D. Node C connects to A. Node D connects to A and B. It's the same image as before.">

この隣接リストはオブジェクトとして実装されています。オブジェクトの各キーはノードを表し、そのキーに対応する値は対応するノードのすべての隣接ノードを含む配列です：

```javascript
const adjacencyList = {
  A: ["B", "C", "D"],
  B: ["A", "D"],
  C: ["A"],
  D: ["A", "B"]
};
```

または、2D配列として実装することもできます。各インデックスがノードを表します。例えば、インデックス0はノードA、インデックス1はノードB、以下同様です：

```javascript
const adjacencyList = [
  ["B", "C", "D"], // Neighbors of A (index 0)
  ["A", "D"],      // Neighbors of B (index 1)
  ["A"],           // Neighbors of C (index 2)
  ["A", "B"]       // Neighbors of D (index 3)
];
```

この2D配列は隣接行列に似ているように見えますが、実際はかなり異なります。

* 隣接行列はグラフのエッジやエッジの重みを表す0、1、または他の値を格納します。

* 隣接リストは各ノードの実際の隣接ノードのリストを格納します。

これは非常に重要な違いで、よく理解しておくべきです。

隣接行列と隣接リストはどちらもグラフを実装する上で非常に重要です。どちらを選ぶかはグラフのサイズやデータの使い方によります。隣接行列はエッジが多い密なグラフに適しており、隣接リストはスパースグラフが多い現実のシナリオで一般的に好まれます。
