# --interactive--

**注意**：有些互動範例可能會使用你還沒學過的 CSS。不要擔心試著理解所有的程式碼。這些範例的目標是為了展示這些設計概念的預覽，讓你更了解運作方式。要看到預覽，你需要啟用互動式編輯器。

在網頁設計方面，你可以使用許多種類的導覽輔助工具。範例包括頂部導覽列、側邊欄和頁腳。但如果你的網站較為複雜且具有較深的導覽層次，你可能會想考慮使用麵包屑。

麵包屑是一種導航輔助工具，顯示使用者在網站階層體系中的位置。以下是一個模擬電子產品網站的麵包屑範例：

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<div class="breadcrumbs">
  Home / Electronics / Phones / Smartphone XYZ
</div>
```

```css
.breadcrumbs {
  font-size: 0.9rem;
  color: #555;
  margin: 1rem 0;
}
```

:::

在大多數網站中，麵包屑會顯示在頁面頂部，向使用者展示他們到達當前頁面的路徑。從 `Homepage` 開始，使用者瀏覽到 `Electronics` 分類，接著到 `Phones` 分類，最後到 `Smartphone XYZ` 產品。當你在搜尋產品或特定資訊時，可能已經與網站上的麵包屑互動過。

使用麵包屑有助於使用者了解他們在網站的階層體系中的位置，以及如何返回先前的頁面。當使用者是從搜尋結果或外部連結進入，並且需要了解他們所在頁面的背景關係時，這尤其有用。

在設計麵包屑時，有幾點需要注意。首先是決定分隔符會是什麼。分隔符是用來分隔階層體系不同層次的字元。常見的分隔符包括大於號（`>`）、右角引號（`»`）和斜線（`/`）。

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<div class="breadcrumbs">
  Home &gt; Electronics &gt; Phones &gt; Smartphone XYZ
</div>
```

```css
.breadcrumbs {
  font-size: 0.9rem;
  color: #555;
  margin: 1rem 0;
}
```

:::

第二個考量是麵包屑的擺放位置。麵包屑通常放置在頁面頂部，位於主要導覽列的上方或下方。使用者不應該費力尋找麵包屑，因此請確保它們可見且容易找到。 

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<nav class="main-nav">
  <a href="#">Home</a>
  <a href="#">Electronics</a>
  <a href="#">Phones</a>
</nav>
<div class="breadcrumbs">
  Home &gt; Electronics &gt; Phones &gt; Smartphone XYZ
</div>
```

```css
.main-nav {
  background-color: #333;
  padding: 1rem;
}

.main-nav a {
  color: white;
  margin-right: 1rem;
  text-decoration: none;
}

.breadcrumbs {
  font-size: 0.9rem;
  color: #555;
  margin: 1rem 0;
}
```

:::

另一個考量是麵包屑的大小。你要確保麵包屑足夠大以便輕鬆閱讀，但又不能太大以致佔用頁面過多空間。請記住，麵包屑並非用來作為主要的導覽工具，而是作為次要的導覽輔助。在頁面上有大量資訊的網站中，使用者可以輕鬆看到自己在階層體系中的位置，並使用麵包屑返回先前的頁面。

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<div class="breadcrumbs">
  Home &gt; Electronics &gt; Phones &gt; Smartphone XYZ
</div>
<p>
  Lorem ipsum dolor sit amet, consectetur adipiscing elit. Sed do eiusmod tempor incididunt ut labore et dolore magna aliqua.
</p>
```

```css
.breadcrumbs {
  font-size: 1rem;
  color: #555;
  margin: 1rem 0;
}   

p {
  font-size: 1.2rem;
  line-height: 1.6;
}
```

:::

那麼你應該在什麼時候在你的網站上使用 breadcrumbs 呢？如果你的網站簡單且階層體系淺，breadcrumbs 可能不是必要的。階層體系淺表示網站只有少數幾個導覽層級，因此 breadcrumbs 可能不會帶來太多價值。通常，你會在電子商務網站、新聞網站以及其他具有深階層體系的網站上看到 breadcrumbs，例如技術文件網站。
