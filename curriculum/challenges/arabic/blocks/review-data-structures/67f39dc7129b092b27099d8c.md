## الخوارزميات وترميز Big O

- **الخوارزميات**: مجموعة من التعليمات الواضحة لحل مشكلة أو تنفيذ مهمة. يجب أن تنتهي الخوارزميات بعدد محدود من الخطوات، ويجب أن تكون كل خطوة دقيقة وواضحة.

- **ترميز Big O**: يصف أسوأ أداء محتمل أو معدل نمو الخوارزمية مع زيادة حجم المُدخَل. يركز على كيفية زيادة استخدام الموارد مع حجم المُدخَل، متجاهلاً العوامل الثابتة والحدود ذات الرتب الأقل.

### تعقيدات الوقت الشائعة

- **O(1) - وقت ثابت**: تستغرق الخوارزمية نفس الوقت بغض النظر عن حجم المُدخَل.

```python
def check_even_or_odd(number):
    if number % 2 == 0:
        return 'Even'
    else:
        return 'Odd'
```

- **O(log n) - وقت لوغاريتمي**: يزداد الوقت ببطء مع نمو المُدخَل. شائع في الخوارزميات التي تقلل حجم المشكلة بشكل متكرر بنسبة معينة (مثل البحث الثنائي).

- **O(n) - وقت خطي**: يزداد وقت التشغيل تناسبياً مع حجم المُدخَل.

```python
for grade in grades:
    print(grade)
```

- **O(n log n) - وقت لوغاريتمي خطي**: تعقيد زمني شائع لخوارزميات الفرز الفعالة مثل دمج الفرز والفرز السريع.

- **O(n²) - وقت تربيعي**: يزداد وقت التشغيل تربيعياً. يظهر غالبًا في الحلقات المتداخلة.

```python
for i in range(n):
    for j in range(n):
        print("Hello, World!")
```

### تعقيد المساحة

- **O(1) - مساحة ثابتة**: تستخدم الخوارزمية نفس كمية الذاكرة بغض النظر عن حجم المُدخَل.
- **O(n) - مساحة خطية**: يزداد استخدام الذاكرة تناسبياً مع حجم المُدخَل.
- **O(n²) - مساحة تربيعية**: يزداد استخدام الذاكرة تربيعياً مع حجم المُدخَل.

## تقنيات حل المشكلات

- **فهم المشكلة**: اقرأ بيان المشكلة عدة مرات. حدد المُدخَل، الناتج المتوقع، وكيفية تحويل المُدخَل إلى الناتج.

- **الرمز الزائف**: وصف عالي المستوى لمنطق الخوارزمية مستقل عن اللغة. يستخدم لغة مكتوبة شائعة ممزوجة ببنى برمجية مثل `IF`، `ELSE`، `FOR`، `WHILE`.

```md
GET original_string
SET reversed_string = ""
FOR EACH character IN original_string:
  ADD character TO THE BEGINNING OF reversed_string
DISPLAY reversed_string
```

- **حالات الحافة**: مُدخلات محددة وصحيحة تقع عند حدود ما يجب أن تتعامل معه الخوارزمية. اعتبرها دائمًا واختبرها.

## المصفوفات

- **المصفوفات الثابتة**: لها حجم ثابت يُحدد عند التهيئة. تُخزن العناصر في مواقع ذاكرة متجاورة. لا يمكن تغيير الحجم أثناء تنفيذ البرنامج.

- **المصفوفات الديناميكية**: يمكن أن تنمو أو تتقلص تلقائيًا أثناء تنفيذ البرنامج. تتعامل مع تغيير الحجم عبر النسخ التلقائي إلى مصفوفات أكبر عند الحاجة.

### قوائم بايثون (مصفوفات ديناميكية)

```python
numbers = [3, 4, 5, 6]

# Access elements
numbers[0]  # 3

# Update elements
numbers[2] = 16

# Add elements
numbers.append(7)
numbers.insert(3, 15)  # Insert at specific index

# Remove elements
numbers.pop(2)  # Remove at specific index
numbers.pop()   # Remove last element
```

### تعقيدات الوقت للمصفوفات الديناميكية

- **الوصول**: O(1)
- **الإدراج في النهاية**: O(1) في المتوسط، O(n) عند الحاجة لتغيير الحجم
- **الإدراج في الوسط**: O(n)
- **الحذف**: O(n) للوسط، O(1) للنهاية

## المكدسات

- **المكدسات**: هيكل بيانات من نوع آخر يدخل أولاً يخرج أخيراً (LIFO). تُضاف العناصر وتُزال من الأعلى فقط.

- **عملية الدفع (Push)**: إضافة عنصر إلى أعلى المكدس. تعقيد الوقت: O(1).

- **عملية السحب (Pop)**: إزالة عنصر من أعلى المكدس. تعقيد الوقت: O(1).

```python
# Using Python list as stack
stack = []

# Push operations
stack.append(1)
stack.append(2)
stack.append(3)

# Pop operations
top_element = stack.pop()  # Returns 3
```

## الطوابير

- **الطوابير**: هيكل بيانات من نوع أول من يدخل أول من يخرج (FIFO). تُضاف العناصر إلى الخلف وتُزال من الأمام.

- **عملية الإدخال في الطابور (Enqueue)**: إضافة عنصر إلى نهاية الطابور. تعقيد الوقت: O(1).

- **عملية الإخراج من الطابور (Dequeue)**: إزالة عنصر من بداية الطابور. تعقيد الوقت: O(1).

```python
from collections import deque

# Using deque for efficient queue operations
queue = deque()

# Enqueue operations
queue.append(1)
queue.append(2)
queue.append(3)

# Dequeue operations
first_element = queue.popleft()  # Returns 1
```

## القوائم المرتبطة

- **القوائم المرتبطة**: هيكل بيانات خطي حيث تحتوي كل عقدة على بيانات ومرجع للعقدة التالية. العقد متصلة كسلسلة.

### القوائم المرتبطة الأحادية

- **الهيكل**: كل عقدة تحتوي على بيانات ومرجع واحد للعقدة التالية.
- **التنقل**: يمكن التحرك فقط للأمام من الرأس إلى الذيل.
- **عقدة الرأس**: العقدة الأولى في القائمة، عادةً العقدة الوحيدة التي يمكن الوصول إليها مباشرة.
- **عقدة الذيل**: العقدة الأخيرة في القائمة، تشير إلى `None`.

### العمليات وتعقيدات الوقت

- **الإدراج في البداية**: O(1)
- **الإدراج في النهاية**: O(n) - يجب التنقل إلى النهاية
- **الإدراج في الوسط**: O(n) - يجب التنقل إلى الموضع
- **الحذف من البداية**: O(1)
- **الحذف من النهاية**: O(n) - يجب التنقل لإيجاد العقدة السابقة
- **الحذف من الوسط**: O(n) - يجب التنقل لإيجاد العقدة

### القوائم المرتبطة المزدوجة

- **الهيكل**: كل عقدة تحتوي على بيانات ومرجعين: العقدة التالية والعقدة السابقة.
- **التنقل**: يمكن التحرك في كلا الاتجاهين.
- **الذاكرة**: تتطلب ذاكرة أكثر من القوائم المرتبطة الأحادية بسبب المرجع الإضافي.

## جداول التجزئة والمجموعات

### الخرائط وجداول التجزئة

- **الخريطة (نوع بيانات مجرد)**: تدير مجموعات من أزواج المفتاح-القيمة. يجب أن يكون كل مفتاح فريدًا، لكن القيم يمكن تكرارها.

- **جدول التجزئة**: تنفيذ ملموس لنوع بيانات الخريطة باستخدام تقنية التجزئة. يستخدم دالة تجزئة لتوليد قيم تجزئة للمفاتيح، تحدد موقع التخزين في المصفوفة الأساسية.

### قواميس بايثون (جداول التجزئة)

```python
# Creating dictionaries
my_dictionary = {
    "A": 1,
    "B": 2, 
    "C": 3
}

# Alternative creation
my_dictionary = dict(A=1, B=2, C=3)

# Access and modify
value = my_dictionary["A"]  # 1
my_dictionary["A"] = 4      # Update value
del my_dictionary["A"]      # Remove key-value pair

# Check membership
"C" in my_dictionary

# Get keys, values, items
my_dictionary.keys()
my_dictionary.values()
my_dictionary.items()
```

### تعقيدات الوقت لجداول التجزئة

- **الحالة المتوسطة**: O(1) للإدراج، الاسترجاع، الحذف
- **أسوأ حالة**: O(n) عند حدوث العديد من تصادمات التجزئة

### المجموعات

- **المجموعات**: مجموعات غير مرتبة من عناصر فريدة. لا يسمح بالتكرار، ولا يتم الحفاظ على ترتيب معين.

- **العناصر غير القابلة للتغيير فقط**: يمكن أن تحتوي المجموعات فقط على أنواع بيانات غير قابلة للتغيير (أرقام، سلاسل نصية، صفوف مرتبة) لأن قيم التجزئة يجب أن تبقى ثابتة.

```python
# Creating sets
numbers = {1, 2, 3, 4}
empty_set = set()  # Must use set(), not {}

# Add and remove elements
numbers.add(5)
numbers.remove(4)      # Raises KeyError if not found
numbers.discard(4)     # No error if not found

# Set operations
set_a = {1, 2, 3, 4}
set_b = {2, 3, 4, 5, 6}

# Union, intersection, difference, symmetric difference
set_a.union(set_b)                    # or set_a | set_b
set_a.intersection(set_b)             # or set_a & set_b
set_a.difference(set_b)               # or set_a - set_b
set_a.symmetric_difference(set_b)     # or set_a ^ set_b

# Subset and superset checks
set_a.issubset(set_b)
set_a.issuperset(set_b)
set_a.isdisjoint(set_b)

# Membership testing
5 in numbers
```

### تعقيدات الوقت للمجموعات

- **الحالة المتوسطة**: O(1) للإضافة، الإزالة، اختبار العضوية
- **أسوأ حالة**: O(n) بسبب تصادمات التجزئة

## تصادمات التجزئة

- **تصادم التجزئة**: يحدث عندما تنتج مفتاحان مختلفان نفس قيمة التجزئة.

- **استراتيجيات حل التصادم**:
  - **التسلسل**: يشير كل فهرس في المصفوفة إلى قائمة مرتبطة تخزن كل العناصر ذات نفس قيمة التجزئة
  - **العنوان المفتوح**: البحث عن الفهرس التالي المتاح باستخدام متتالية محددة مسبقًا

## متى تستخدم كل هيكل بيانات

- **القوائم**: عندما تحتاج إلى وصول مرتب ومفهرس ولا تعرف الحجم مسبقًا
- **المكدسات**: لعمليات LIFO (وظيفة التراجع، تقييم التعبير، التراجع)
- **الطوابير**: لعمليات FIFO (جدولة المهام، البحث بالاتساع أولاً)
- **القوائم المرتبطة**: عند الإدراج/الحذف المتكرر في البداية، الحجم غير معروف، لا حاجة للوصول العشوائي
- **جداول التجزئة**: للبحث السريع عن أزواج المفتاح-القيمة، عد التكرارات، التخزين المؤقت
- **المجموعات**: للتحقق من التفرد، عمليات المجموعات الرياضية، إزالة التكرارات
