## 검색 알고리즘

검색 알고리즘은 특정 항목 목록 내에서 목표를 검색할 수 있게 합니다.

컴퓨터 과학에서 알아야 할 검색 알고리즘은 두 가지가 있습니다. 바로 **선형 검색**과 **이진 검색** 알고리즘입니다. 두 알고리즘의 차이점과 각각 언제 사용하는지 이해하는 것이 중요합니다.

### 선형 검색

- 선형 검색은 목록의 처음부터 끝까지 각 항목을 반복하며 목표 항목을 찾습니다.
- 목표 항목을 찾으면 목록에서 위치한 인덱스를 반환합니다.
- 목표를 찾지 못하면 `-1`를 반환하는데, 이는 대부분 프로그래밍 언어에서 **유효하지 않은 인덱스**를 의미합니다.
- 선형 검색은 목표를 찾을 때까지 각 항목을 확인하기 때문에, 항목이 많은 목록에서는 효율적이지 않습니다.
- 선형 검색의 시간 복잡도는 `O(n)`인데, 목록 크기에 따라 검색 시간이 선형적으로 증가하기 때문입니다.
- 선형 검색의 공간 복잡도는 `O(1)`인데, 목록을 검색하는 데 추가 공간이 필요하지 않기 때문입니다.

### 이진 검색

- 이진 검색은 목록을 반으로 나누고, 목표 값이 목록 중간에 있는지 확인하는 방식으로 작동합니다.
- 이진 검색이 작동하려면 목록의 항목들이 오름차순 또는 내림차순으로 정렬되어 있어야 합니다.
- 이진 검색은 목록을 반으로 나누고 목표가 없는 절반은 무시하기 때문에, 많은 항목이 있는 목록을 검색할 때 더 효율적인 알고리즘입니다.
- 목표 항목이 목록 중간에 있으면 해당 항목의 인덱스를 반환합니다.
- 항목을 찾지 못하면 알고리즘은 목표 항목이 목록의 왼쪽 절반에 있는지 오른쪽 절반에 있는지 확인합니다.
- 목표 항목을 찾을 때까지 남은 목록 부분을 계속 반으로 나눕니다.
- 최종적으로 목표 항목을 찾지 못하면 `-1`를 반환합니다.
- 이진 검색의 시간 복잡도는 `O(log n)`인데, 목록 크기에 따라 검색 시간이 로그 함수적으로 증가하기 때문입니다.
- 이진 검색의 공간 복잡도는 `O(1)`인데, 목록을 검색하는 데 추가 공간이 필요하지 않기 때문입니다.

### 선형 검색과 이진 검색의 차이점

- 이진 검색은 선형 검색에 비해 많은 항목이 있는 목록에 더 적합합니다.
- 선형 검색의 시간 복잡도는 `O(n)`인데, 목록 크기에 따라 검색 시간이 선형적으로 증가하기 때문입니다.
- 이진 검색의 시간 복잡도는 `O(log n)`인데, 목록 크기에 따라 검색 시간이 로그 함수적으로 증가하기 때문입니다.

## 정렬 알고리즘과 분할 정복

컴퓨터 과학에서 분할 정복은 문제를 더 작은 하위 문제로 나누어 해결하기 쉽게 만드는 기법입니다. 재귀는 분할 정복에서 자주 사용되는 기법이며, 분할 정복은 병합 정렬과 같은 효율적인 정렬 알고리즘을 구현하는 강력한 전략입니다.

### 병합 정렬

- 병합 정렬은 분할 정복 방식을 따르는 정렬 알고리즘입니다.
- 목록을 재귀적으로 더 작은 하위 목록으로 나누어 각 하위 목록이 하나의 요소만 포함할 때까지 나눕니다.
- 그런 다음 하위 목록들을 정렬된 순서로 반복해서 병합합니다.
- 병합 정렬의 시간 복잡도는 `O(n log n)`인데, 목록을 계속 반으로 나누고 `(log n)` 다시 병합하기 때문입니다 `(O(n))`.
- 병합 정렬의 공간 복잡도는 `O(n)`인데, 제자리 정렬 알고리즘이 아니기 때문입니다.
