# --description--

Hai imparato a conoscere le API `HTMLMediaElement` e MediaStreams, ma ci sono alcune altre API che possono essere molto utili per lavorare con video e audio.

La prima è la Screen Capture API. Come suggerisce il nome, questa API ti permette di registrare lo schermo di un utente. Questa API è esposta chiamando il metodo `getDisplayMedia()` dell’oggetto `mediaDevices` e utilizzando il media stream restituito.

Poi c’è la MediaStream Recording API. Questa API funziona in tandem con le API MediaStreams, permettendoti di registrare un MediaStream (o anche direttamente un `HTMLMediaElement`). Quindi genera eventi `dataavailable` con payload Blob che puoi scrivere nella memoria locale.

Alla base di tutta questa tecnologia c’è la Media Source Extensions API. La Media Source Extensions API è ciò che ti permette di passare direttamente il feed della webcam di un utente a un elemento video tramite la proprietà `srcObject`, per esempio.

E infine, la Web Audio API che alimenta tutto ciò che è udibile sul web. Questa API include oggetti importanti come un `AudioBuffer` (che rappresenta un Buffer contenente specificamente dati audio) o il `AudioContext`.

E con questo si conclude la nostra lezione su audio e video!
