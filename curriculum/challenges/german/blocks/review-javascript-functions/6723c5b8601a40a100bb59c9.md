# --interactive--

## JavaScript-Funktionen

- Funktionen sind wiederverwendbare Codeblöcke, die eine bestimmte Aufgabe ausführen.
- Funktionen können mit dem `function`-Schlüsselwort definiert werden, gefolgt von einem Namen, einer Liste von Parametern und einem Codeblock, der die Aufgabe ausführt.

:::interactive_editor

```js
function addNumbers(x, y, z) {
  return x + y + z;
}

console.log(addNumbers(5, 3, 8)); // Output: 16
```

:::

- Argumente sind Werte, die an eine Funktion übergeben werden, wenn sie aufgerufen wird.
- Ein Funktionsaufruf ist der Vorgang, eine Funktion in einem Programm auszuführen, indem der Name der Funktion gefolgt von runden Klammern angegeben wird, optional mit Argumenten innerhalb der Klammern.
- Wenn eine Funktion ihre Ausführung beendet, gibt sie immer einen Wert zurück.
- Standardmäßig ist der Rückgabewert einer Funktion `undefined`.
- Das `return`-Schlüsselwort wird verwendet, um den Wert anzugeben, der von der Funktion zurückgegeben wird, und beendet die Ausführung der Funktion.
- Standardparameter ermöglichen es Funktionen, vordefinierte Werte zu haben, die verwendet werden, wenn beim Aufruf der Funktion kein Argument übergeben wird. Das macht Funktionen flexibler und verhindert Fehler in Fällen, in denen bestimmte Argumente weggelassen werden könnten.

:::interactive_editor

```js
const calculateTotal = (amount, taxRate = 0.05) => {
  return amount + (amount * taxRate);
};

console.log(calculateTotal(100)); // Output: 105
```

:::

- Anonyme Funktionen sind Funktionen ohne Namen, die Variablen zugewiesen werden können. Durch die Zuweisung an Variablen können Sie sie überall dort wiederverwenden, wo die Variable zugänglich ist.

:::interactive_editor

```js
const multiplyNumbers = function(firstNumber, secondNumber) {
  return firstNumber * secondNumber;
};

console.log(multiplyNumbers(4, 5)); // Output: 20
```

:::

## Pfeilfunktionen

- Pfeilfunktionen sind eine prägnantere Art, Funktionen in JavaScript zu schreiben.

:::interactive_editor

```js
const calculateArea = (length, width) => {
  const area = length * width;
  return `The area of the rectangle is ${area} square units.`;
};

console.log(calculateArea(5, 10)); // Output: "The area of the rectangle is 50 square units."
```

:::

- Beim Definieren einer Pfeilfunktion benötigen Sie nicht das `function`-Schlüsselwort.
- Wenn Sie nur einen Parameter verwenden, können Sie die runden Klammern um die Parameterliste weglassen.

:::interactive_editor

```js
const cube = x => {
  return x * x * x;
};

console.log(cube(3)); // Output: 27
```

:::

- Wenn der Funktionskörper aus einem einzigen Ausdruck besteht, können Sie die geschweiften Klammern und das `return`-Schlüsselwort weglassen.

:::interactive_editor

```js
const square = number => number * number;

console.log(square(5)); // Output: 25
```

:::

## Gültigkeitsbereich in der Programmierung

- **Globaler Gültigkeitsbereich**: Dies ist der äußerste Gültigkeitsbereich in JavaScript. Variablen, die im globalen Gültigkeitsbereich deklariert sind, sind von überall im Code zugänglich und werden globale Variablen genannt.
- **Lokaler Gültigkeitsbereich**: Dies bezieht sich auf Variablen, die innerhalb einer Funktion deklariert sind. Diese Variablen sind nur innerhalb der Funktion zugänglich, in der sie deklariert wurden, und werden lokale Variablen genannt.
- **Block-Gültigkeitsbereich**: Ein Block ist eine Gruppe von Anweisungen, die in geschweifte Klammern `{}` eingeschlossen sind, wie zum Beispiel in `if`-Anweisungen oder Schleifen.
- Die Block-Gültigkeitsbereichssteuerung mit `let` und `const` bietet eine noch feinere Kontrolle über die Zugänglichkeit von Variablen, hilft Fehler zu vermeiden und macht Ihren Code vorhersehbarer.
