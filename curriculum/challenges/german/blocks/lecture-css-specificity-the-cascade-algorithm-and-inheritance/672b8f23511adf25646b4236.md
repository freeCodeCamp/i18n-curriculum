# --interactive--

Der Cascade-Algorithmus ist der Prozess, den der Browser verwendet, um zu entscheiden, welche CSS-Regeln angewendet werden, wenn mehrere Styles dasselbe Element ansprechen. Er stellt sicher, dass die passendsten Styles verwendet werden, basierend auf einem Satz klar definierter Regeln.

Der Prozess beginnt mit der **Relevanz**. Der Browser filtert zunächst alle CSS-Regeln, um diejenigen zu finden, die tatsächlich auf das betreffende Element zutreffen. Dazu gehört das Abgleichen von Selektoren und das Berücksichtigen von Media Queries, die möglicherweise aktiv sind.

Eine Media Query ist eine CSS-Technik, mit der Styles basierend auf den Eigenschaften des Geräts oder Viewports angewendet werden, wie etwa dessen Breite, Höhe oder Ausrichtung.

Als Nächstes betrachtet der Algorithmus **Herkunft und Wichtigkeit**. CSS kann aus verschiedenen Quellen stammen: den Standard-Styles des Browsers (User-Agent), vom Benutzer festgelegten Styles und vom Autor (Ihnen) geschriebenen Styles.

Nach der Berücksichtigung der Herkunft bewertet der Algorithmus dann die Wichtigkeit jeder Regel und gibt Regeln mit `!important` Priorität, die andere Regeln unabhängig von ihrer Quelle überschreiben.

Nachdem nach Herkunft und Wichtigkeit gefiltert wurde, betrachtet der Algorithmus die **Spezifität**. Wenn zwei Regeln aus derselben Herkunft und Wichtigkeitsstufe zutreffen, wird die Regel mit der höheren Spezifität angewendet.

Spezifität ist ein Maß dafür, wie gezielt ein Selektor ist, wobei spezifischere Selektoren Vorrang vor allgemeineren haben.

Schließlich, wenn alles andere gleich ist, kommt die **Reihenfolge des Auftretens** ins Spiel. Wenn zwei Regeln dieselbe Spezifität haben, wird die Regel angewendet, die zuletzt im CSS erscheint.

Deshalb kann die Reihenfolge, in der Sie Ihre Styles schreiben, manchmal das Ergebnis beeinflussen.

Schauen wir uns ein Beispiel an.

In der HTML-Datei gibt es ein einzelnes Absatz-Element. Im CSS haben wir zwei Regeln, die jeweils das Absatz-Element ansprechen.

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<p>example paragraph</p>
```

```css
p {
  color: blue;
}

p {
  color: green; 
}
```

:::

Die erste Regel setzt die Textfarbe aller Absatz-Elemente auf Blau, während die zweite Regel die Textfarbe aller Absatz-Elemente auf Grün setzt.

Welche Farbe wird also angewendet? Die Farbe Grün wird auf die Absatz-Elemente angewendet.

Indem der Cascade-Algorithmus Relevanz, Herkunft und Wichtigkeit, Spezifität, Gültigkeitsbereich und Reihenfolge des Auftretens berücksichtigt, sorgt er dafür, dass Ihr CSS vorhersehbar funktioniert und Sie komplexere und nuanciertere Webseiten gestalten können.
