# --description--

在 Express 中，`res`（响应）对象用于在处理完传入请求后向客户发送响应。Express 提供了多种内置方法来发送不同类型的响应。例如，你可以发送纯文本、JSON 数据、将用户重定向到另一个 URL，或者渲染模板。

最常用的响应方法之一是 `res.send()`：

```javascript
app.get("/text", (req, res) => {
  res.send("Hello, World!");
});
```

`res.send()` 方法向客户发送响应。它可以发送纯文本、HTML、JSON，甚至是二进制数据。在这个例子中，当客户访问 `/text` 时，服务器会响应文本 `Hello, World!`。

另一个常用的响应方法是 `res.json()`。

该方法向客户发送 JSON 响应。当你构建 API 时尤其有用，因为你经常需要返回结构化的数据，比如对象或数组。Express 会自动将 JavaScript 对象转换为 JSON 格式的字串：

```javascript
app.get("/user", (req, res) => {
  res.json({ name: "John Doe", age: 30 });
});
```

如果客户向 `/user` 发送请求，服务器会响应以下内容：

```json
{
  "name": "John Doe",
  "age": 30
}
```

当你需要控制响应的 HTTP 状态码时，可以使用 `res.status()`。该方法设置状态码，通常会与其他响应方法如 `res.send()` 或 `res.json()` 链式调用：

```javascript
app.get("/notfound", (req, res) => {
  res.status(404).send("Not Found");
});
```

这里，服务器将状态码设置为 `404`，并向客户发送消息 `Not Found`。

如果你想将用户发送到不同的 URL，可以使用 `res.redirect()`：

```javascript
app.get("/oldpage", (req, res) => {
  res.redirect("/newpage");
});
```

当客户访问 `/oldpage` 时，他们会被自动重定向到 `/newpage`。

最后，`res.render()` 用于与模板引擎（如 Pug、EJS 或 Handlebars）配合使用。它渲染视图模板并将生成的 HTML 发送给客户：

```javascript
app.get("/profile", (req, res) => {
  res.render("profile", { username: "john_doe" });
});
```

在这个例子中，服务器渲染了一个 `profile` 视图，并传递了一个 `username` 变量，变量可以在 HTML 中显示。

这些响应方法允许你根据请求的性质返回不同类型的数据。无论你是发送文本、JSON 数据，还是重定向客户，Express 都有一套灵活的工具来处理。
