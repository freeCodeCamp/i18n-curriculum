# --description--

JavaScript erlaubt die Verwendung von sowohl einfachen (`'`) als auch doppelten (`"`) Anführungszeichen, um einen String zu deklarieren. Die Entscheidung, welche man verwendet, beruht meist auf persönlicher Vorliebe, mit einigen Ausnahmen.

Zwei Möglichkeiten zu haben, ist praktisch, wenn ein String Kontraktionen oder einen anderen Text enthält, der in Anführungszeichen steht. Seien Sie nur vorsichtig, dass Sie den String nicht zu früh schließen, da dies einen Syntaxfehler verursacht.

Hier sind einige Beispiele für gemischte Anführungszeichen:

```js
const grouchoContraction = "I've had a perfectly wonderful evening, but this wasn't it.";
const quoteInString = "Groucho Marx once said 'Quote me as saying I was mis-quoted.'";
const uhOhGroucho = 'I've had a perfectly wonderful evening, but this wasn't it.';
```

Die ersten beiden sind korrekt, aber das dritte ist falsch.

Natürlich ist es auch in Ordnung, nur einen Stil von Anführungszeichen zu verwenden. Sie können die Anführungszeichen im String mit dem Backslash (`\`) als Escape-Zeichen maskieren:

```js
const allSameQuotes = 'I\'ve had a perfectly wonderful evening, but this wasn\'t it.';
```

# --instructions--

Korrigieren Sie den String so, dass entweder unterschiedliche Anführungszeichen für den `href` Wert verwendet werden oder die vorhandenen maskiert sind. Behalten Sie die doppelten Anführungszeichen um den gesamten String bei.

# --hints--

Ihr Code sollte die Anführungszeichen um den `href` Wert `#Home` entweder ändern oder maskieren.

```js
assert(__helpers.removeJSComments(code).match(/<a href=\s*?('|\\")#Home\1\s*?>/g));
```

Ihr Code sollte die doppelten Anführungszeichen um den gesamten String beibehalten.

```js
assert(__helpers.removeJSComments(code).match(/"<p>.*?<\/p>";/g));
```
