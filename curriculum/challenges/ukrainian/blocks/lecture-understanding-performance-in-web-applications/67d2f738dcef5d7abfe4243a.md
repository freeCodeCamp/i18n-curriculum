# --description--

Покращення «часу до готовності», або іншими словами інтервалу від моменту, коли користувач запитує сторінку, до того, як він може з нею змістовно взаємодіяти, є ключовим для підвищення користувацького досвіду.

Давайте розглянемо ефективні стратегії для досягнення швидшого часу до готовності.

Почніть із фокусу на вмісті, який користувач бачить першим. Це означає негайне завантаження необхідних елементів і відкладення некритичних компонентів.

Наприклад, ви можете спробувати реалізувати lazy loading.

Ми вже розглядали lazy loading, але ось приклад lazy load зображень для повторення:

```html
<img src="image.jpg" loading="lazy" alt="Description">
```

Все, що потрібно зробити — додати атрибут `loading="lazy"` до тегів зображень. Це відкладає завантаження зображень до моменту, коли вони потрібні, зазвичай коли вони потрапляють у вікно перегляду користувача. Завантажуючи зображення та відео лише тоді, коли вони з’являються у вікні перегляду, ми економимо пропускну здатність і прискорюємо початкове відтворення.

Зазвичай вебсторінка завантажує всі зображення одночасно, навіть ті, що не видимі на екрані. З lazy loading зображення, які знаходяться нижче видимої частини сторінки, або іншими словами поза екраном, завантажуються лише тоді, коли користувач прокручує до них.

А ось приклад lazy load для iframe:

```html
<iframe src="video.html" loading="lazy"></iframe>
```

Ви можете застосувати той самий атрибут, що й раніше, але цього разу до iframe, і той самий принцип буде працювати.

Ви також можете використати атрибут `defer` для відкладення некритичного JavaScript до моменту після початкового завантаження сторінки. Ось приклад, як ми це робимо:

```html
<script src="non-critical.js" defer></script>
```

Також варто розглянути мінімізацію ресурсів, що блокують відтворення.

Ресурси, що блокують відтворення, затримують можливість взаємодії зі сторінкою.

Щоб уникнути цього, можна спробувати асинхронне завантаження CSS. Для некритичного CSS використовуйте атрибут `media` для умовного завантаження стилів. Ось приклад, як ми це робимо:

```html
<link rel="stylesheet" href="print.css" media="print">
```

Вказуючи значення media як `print`, `print.css` буде завантажуватися лише під час друку сторінки або її перегляду на екрані. У наступному прикладі, передаючи `min-width: 800px` як значення атрибуту `media`, ми вказуємо, що `desktop.css` має завантажуватися лише для екранів ширше 800px:

```html
<!-- This stylesheet is only loaded on screens wider than 800px -->
<link rel="stylesheet" href="desktop.css" media="(min-width: 800px)">
```

Це економить пропускну здатність, пропускаючи непотрібні стилі.

Отже, підсумовуючи, реалізуючи ці стратегії, ви можете значно скоротити «час до готовності» вашого вебсайту, забезпечуючи користувачам швидку можливість взаємодії з вашим вмістом.
