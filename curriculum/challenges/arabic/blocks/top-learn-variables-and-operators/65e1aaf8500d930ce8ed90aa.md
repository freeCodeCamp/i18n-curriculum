# --description--

علامة الجمع `+` توجد في شكلين: الشكل الثنائي الذي استخدمته في التحدٍّ السابق والشكل الأحادي.

علامة الجمع الأحادية، أو بعبارة أخرى، عامل الجمع `+` المطبّق على قيمة واحدة، لا يفعل شيئًا مع الأعداد. لكن إذا لم تكن القيمة عددًا، فإن الجمع الأحادي يحولها إلى عدد.

على سبيل المثال:

```js
// No effect on numbers
let x = 1;
console.log( +x ); // 1

let y = -2;
console.log( +y ); // -2

// Converts non-numbers
console.log( +true ); // 1
console.log( +"" );   //
```

تظهر الحاجة لتحويل السلاسل النصية إلى أعداد كثيرًا. مثلاً، إذا كنت تحصل على قيم من حقول نموذج HTML، فهي عادةً سلاسل نصية. ماذا لو أردت جمعها؟

علامة الجمع الثنائية ستضيفها كسلاسل نصية:

```js
let apples = "2";
let oranges = "3";

alert( apples + oranges ); // "23", the binary plus concatenates strings
```

إذا أردت التعامل معها كأعداد، يجب تحويلها ثم جمعها:

```js
let apples = "2";
let oranges = "3";

// both values converted to numbers before the binary plus
alert( +apples + +oranges ); // 5

// the longer variant
// alert( Number(apples) + Number(oranges) ); // 5
```

من وجهة نظر رياضي، قد يبدو كثرة علامات الجمع غريبًا. لكن من وجهة نظر المبرمج، لا شيء خاص: تُطبّق علامات الجمع الأحادية أولًا، فتُحوّل السلاسل النصية إلى أعداد، ثم يجمعها الجمع الثنائي.

لماذا تُطبّق علامات الجمع الأحادية على القيم قبل الثنائية؟ كما سترى، ذلك بسبب أسبقيتها _الأعلى_.

# --questions--

## --text--
بالنظر إلى شرح عاملَي الجمع الأحادي والثنائي `+` في JavaScript، كيف يمكنك جمع القيم النصية `'2'` و `'3'` كأعداد بشكل صحيح باستخدام الجمع الأحادي `+`؟

## --answers--

`console.log('2' + '3'); // Outputs: '23'` 

---

`console.log(+2 + +3); // Outputs: 5`

---

`console.log(+'2' + +'3'); // Outputs: 5`

---

`console.log(Number('2') + Number('3')); // Outputs: '23'`


## --video-solution--

3
