# --description--

Expressでは、ルートパラメータはURLパスに埋め込まれた動的な値です。これにより、アプリケーションはリクエストごとに変わるURLの一部を取得できます。これは、特定のユーザー、ブログ投稿、商品などをURLに基づいて表示する動的なページを作成するのに役立ちます。

例を見てみましょう：

```javascript
// Route with a parameter in the path
app.get("/post/:postId", (req, res) => {
  const postId = req.params.postId;
  res.send(`Viewing post with ID: ${postId}`);
});
```

ここで、ルートは`/post/:postId`へのGETリクエストを待ち受けています。

* `:postId`はルートパラメータで、URL内のプレースホルダーです。実際の値に置き換えられます。例えば、`/post/42`へのリクエストはこのルートにマッチし、`42`が`postId`の値になります。
    
* ルートパラメータには`req.params`を使ってアクセスできます。`req.params`オブジェクトにはすべてのルートパラメータが含まれており、この場合は`req.params.postId`がURLからの値、`42`を持っています。
    

最後に、`res.send()`がクライアントにレスポンスを送ります。つまり、URLが`/post/42`の場合、クライアントは`Viewing post with ID: 42`を見ます。

このように、ルートパラメータを使うと、すべての可能な値に対して別々のルートを作成する必要なく、ルートを柔軟かつ動的にできます。

ルートパラメータの仕組みがわかったところで、前のレッスンの例に戻り、異なる種類のメニューに動的ルートを使う方法を見てみましょう：

```javascript
app.get("/menu/:category", (req, res) => {
  const category = req.params.category;
  res.send(`You are viewing the ${category} menu!`);
});
```

この動作の内訳は次の通りです：

1. ユーザーが[`http://localhost:3000/menu/drinks`](http://localhost:3000/menu/drinks)を訪れます。
    
2. サーバーはこのリクエストをルート`/menu/:category`にマッチさせ、`category`パラメータを`drinks`として取得します。
    
3. サーバーはブラウザにこのレスポンスを送ります：`You are viewing the drinks menu!`
    

同様に、ユーザーが[`http://localhost:3000/menu/food`](http://localhost:3000/menu/food)を訪れた場合、サーバーは次のレスポンスを返します：`You are viewing the food menu!`

ルートパラメータを使うと、URLに基づいて動的なコンテンツを表示できます。すべてのカテゴリに対して別々のルートを作る代わりに、URLを変えるだけで飲み物、食べ物、デザートなど複数のカテゴリを1つのルートで処理できます。
