# --description--

JavaScript 프로그램을 디버깅하는 것은 코드에 `console.log()` 문을 사용하는 것 이상입니다. 익숙해지면 디버깅을 훨씬 쉽게 만들어 주는 더 고급 기법들이 있습니다. 이러한 고급 기법들을 살펴보면서 오류가 발생했을 때 JavaScript 프로그램을 기쁘게 디버깅할 수 있도록 하겠습니다.

먼저 살펴볼 개념은 중단점 작업입니다. 중단점은 코드 실행을 사용자가 선택한 특정 줄에서 일시 중지할 수 있게 해줍니다. 일시 중지 후에는 변수들을 검사하고, 표현식을 평가하며, 호출 스택을 확인할 수 있습니다.

Chrome 브라우저에서 코드의 어느 줄에든 중단점을 추가하려면 개발자 도구를 열고 Sources 탭으로 이동한 다음 디버깅할 JavaScript 파일을 열고 줄 번호를 클릭하여 중단점을 설정하세요.

실행이 중단점에 도달해 정지하면 오른쪽 상단의 아이콘을 사용해 코드를 단계별로 실행할 수 있습니다. 또한 줄을 우클릭하고 "조건부 중단점 추가…"를 선택해 조건부 중단점을 추가할 수도 있습니다.

다음으로 표현식 감시에 대해 알아보겠습니다. 표현식 감시는 코드가 실행되는 동안 현재 범위 밖에 있더라도 변수나 표현식의 값을 모니터링할 수 있게 해줍니다.

표현식 감시를 추가하려면 개발자 도구의 Sources 탭으로 이동해 오른쪽의 감시 패널을 찾고 더하기(`+`) 아이콘을 클릭하세요.

다음 개념은 프로파일링입니다. 프로파일링은 스크린샷을 캡처하고 CPU 사용량, 함수 호출, 실행 시간을 기록해 성능 병목 현상을 파악하는 데 도움을 줍니다.

코드에 대해 프로파일링하려면 Performance 탭을 열고 기록을 클릭한 후 프로파일링할 작업을 수행하세요. 작업 실행이 끝나면 기록을 중지해 결과를 분석할 수 있습니다.

이 방법은 애플리케이션이 특정 작업 중에 느려질 때 유용합니다. 프로파일러를 사용하면 어떤 함수나 코드 내 데이터가 가장 많은 자원을 소비하는지 확인할 수 있습니다.

이제 네트워크 요청 검사로 넘어가겠습니다. 네트워크 요청 검사는 매개변수 오류, 주소 오류, 서버 오류 같은 API 요청 관련 문제를 디버깅하는 데 도움을 줍니다.

디버깅을 위해 Network 탭을 사용하려면 개발자 도구를 열고 Network 탭으로 이동한 다음 개별 요청을 클릭해 헤더, 응답, 페이로드 같은 세부 정보를 확인하세요.

수업의 마지막 부분에서는 `console.table`와 `console.dir`에 집중하겠습니다.

`console.table()`는 콘솔에 표 형식으로 데이터를 표시합니다. 필수 인수로 배열이나 객체를 받고, 표시할 속성이나 열을 지정하는 선택적 인수를 하나 더 받습니다.

반면 `console.dir()`는 지정한 JavaScript 객체의 속성 목록을 인터랙티브하게 표시합니다. 중첩된 모든 속성을 볼 수 있도록 확장 가능한 계층적 목록을 출력합니다.

이제 확장된 디버깅 기법 도구를 갖추었으니, 다가오는 JavaScript 프로젝트에서 이 새로운 기술들을 시험해 보세요. 이러한 전략을 적용하면 문제를 더 효율적으로 찾아내고 해결해 더 깔끔하고 신뢰할 수 있는 코드를 작성할 수 있습니다.
