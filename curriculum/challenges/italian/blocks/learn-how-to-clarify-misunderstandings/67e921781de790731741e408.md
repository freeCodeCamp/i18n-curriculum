# --explanation--

`Clarifying` significa spiegare qualcosa in modo da eliminare la confusione. Aiuta a comprendere meglio un concetto o una situazione. Per esempio:

`Thanks for clarifying how the API works.` - Il parlante è grato perché la spiegazione ha reso l'API più facile da capire.

`Abstraction` indica semplificare qualcosa mostrando solo le funzionalità essenziali e nascondendo il funzionamento interno complesso. Per esempio:

`Using a login function is an example of abstraction - you don't need to know how the password is checked, just that it works.` - Questo mostra come gli utenti interagiscono con interfacce semplici mentre la logica tecnica resta nascosta.
