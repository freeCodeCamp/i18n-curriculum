## Інструменти для перевірки контрастності кольорів

- **Перевірка контрастності кольорів WebAIM**: Цей онлайн-інструмент дозволяє ввести кольори переднього плану та фону вашого дизайну і миттєво побачити, чи відповідають вони стандартам Web Content Accessibility Guidelines (WCAG).
- **TPGi Colour Contrast Analyzer**: Це безкоштовний інструмент для перевірки контрастності кольорів, який дозволяє перевірити, чи відповідають ваші вебсайти та застосунки стандартам Web Content Accessibility Guidelines (WCAG). Цей інструмент також має функцію симулятора колірної сліпоти, що дозволяє побачити, як ваш сайт або застосунок виглядає для людей із проблемами кольорового сприйняття.

## Дерево доступності

Дерево доступності — це структура, яку використовують допоміжні технології, такі як читачі екрана, щоб інтерпретувати та взаємодіяти з вмістом на вебсторінці.

## `max()` функція

Функція `max()` повертає найбільше значення з множини значень, розділених комами:

:::interactive_editor

```html
<head>
  <link rel="stylesheet" href="styles.css">
</head>
<body>
  <img src="https://cdn.freecodecamp.org/curriculum/css-photo-gallery/9.jpg" alt="A cat sitting on a table">
</body>
```

```css
img {
  width: max(250px, 25vw);
}
```

:::

У наведеному вище прикладі ширина зображення буде 250px, якщо ширина вікна перегляду менша за 1000 пікселів. Якщо ширина вікна перегляду більша за 1000 пікселів, ширина зображення буде 25vw. Це тому, що 25vw дорівнює 25% ширини вікна перегляду.

## Найкращі практики з CSS і доступністю

- **`display: none;`**: Використання `display: none;` означає, що читачі екрана та інші допоміжні технології не зможуть отримати доступ до цього вмісту, оскільки він не включений до дерева доступності. Тому важливо використовувати цей метод лише тоді, коли ви хочете повністю видалити вміст як із візуального відображення, так і з доступності.
- **`visibility: hidden;`**: Ця властивість і значення приховують вміст візуально, але залишають його в потоці документа, тобто він все ще займає місце на сторінці. Ці елементи також більше не будуть читатися читачами екрана, оскільки вони будуть видалені з дерева доступності.
- **`.sr-only` CSS клас**: Це загальна техніка, яка використовується для візуального приховування вмісту, залишаючи його доступним для читачів екрана.

:::interactive_editor

```html
<head>
  <link rel="stylesheet" href="styles.css">
</head>
<body>
  <span>This span element can be read</span>
  <span class="sr-only">This span element can only be read by screen readers</span>
  <p style="display: none">
    This text is hidden with display: none and is NOT accessible to screen readers.
  </p>
  <p style="visibility: hidden">
    This text is hidden with visibility: hidden and is NOT read by screen readers,
    but it still takes up space.
  </p>
</body>

```

```css
.sr-only {
  position: absolute;
  width: 1px;
  height: 1px;
  padding: 0;
  margin: -1px;
  overflow: hidden;
  clip: rect(0, 0, 0, 0);
  white-space: nowrap;
  border: 0;
}

```

:::

- **`scroll-behavior: smooth;`**: Ця властивість і значення увімкнуть плавну поведінку прокручування.
- **`prefers-reduced-motion` функція**: Це медіафункція, яка може використовуватися для визначення переваг користувача щодо анімації.

## Приховування вмісту за допомогою HTML-атрибутів

- **`aria-hidden` атрибут**: Використовується для приховування елемента від користувачів допоміжних технологій, таких як читачі екрана. Наприклад, його можна застосувати для приховування декоративних зображень, які не несуть смислового вмісту.
- **`hidden` атрибут**: Цей атрибут підтримується більшістю сучасних браузерів і приховує вміст як візуально, так і з дерева доступності. Його можна легко вмикати та вимикати за допомогою JavaScript.

:::interactive_editor

```html
<p aria-hidden>This paragraph is visible for sighted users, but is hidden from assistive technology.</p>
<p hidden>This paragraph is hidden from both sighted users and assistive technology.</p>
<p>This is a normal paragraph</p>

```

:::

## Проблема доступності атрибута `placeholder`

Використання тексту-заповнювача не є хорошим для доступності. Надто часто користувачі плутають текст-заповнювач із фактичним значенням у полі введення — вони думають, що в полі вже є значення.
