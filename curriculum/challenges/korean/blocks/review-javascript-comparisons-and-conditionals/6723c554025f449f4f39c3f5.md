## 비교와 `null` 및 `undefined` 데이터 타입

- **비교와 `undefined`**: 변수가 선언되었지만 값이 할당되지 않은 상태일 때 `undefined`입니다. 초기화되지 않은 변수와 인수가 제공되지 않은 함수 매개변수의 기본값입니다. `undefined`는 숫자 문맥에서 `NaN`로 변환되어 `undefined`와의 모든 숫자 비교가 `false`를 반환하게 만듭니다.

:::interactive_editor

```js
console.log(undefined < 0); // false (NaN < 0 is false)
console.log(undefined >= 0); // false (NaN >= 0 is false)
```

:::

- **비교와 `null`**: `null` 타입은 의도적으로 값이 없음을 나타냅니다. `null`는 숫자 문맥에서 `0`로 변환되어 숫자 비교에서 예상치 못한 동작을 일으킬 수 있습니다:

:::interactive_editor

```js
console.log(null < 0); // false (0 < 0 is false)
console.log(null >= 0); // true (0 >= 0 is true)
```

:::

- 동등 연산자(`==`)를 사용할 때, `null`와 `undefined`는 서로와 자기 자신에게만 같습니다:

:::interactive_editor

```js
console.log(null == undefined); // true
console.log(null == 0); // false
console.log(undefined == NaN); // false
```

:::

- 그러나 타입 강제 변환 없이 값과 타입을 모두 검사하는 엄격한 동등 연산자(`===`)를 사용할 때는 `null`와 `undefined`가 같지 않습니다:

:::interactive_editor

```js
console.log(null === undefined); // false
```

:::

## `switch` 문

- **정의**: `switch` 문은 표현식을 평가하고 그 값을 일련의 `case` 절과 비교합니다. 일치하는 절이 발견되면 해당 케이스에 연결된 코드 블록이 실행됩니다. 각 케이스 끝에는 실행을 종료하고 다음으로 넘어가기 위해 `break` 문이 있어야 합니다. `default` 케이스는 선택적이며 다른 케이스가 모두 일치하지 않을 때만 실행됩니다. `default` 케이스는 `switch` 문 끝에 위치합니다.

:::interactive_editor

```js
const dayOfWeek = 3; 

switch (dayOfWeek) {
  case 1:
    console.log("It's Monday! Time to start the week strong.");
    break;
  case 2:
    console.log("It's Tuesday! Keep the momentum going.");
    break;
  case 3:
    console.log("It's Wednesday! We're halfway there.");
    break;
  case 4:
    console.log("It's Thursday! Almost the weekend.");
    break;
  case 5:
    console.log("It's Friday! The weekend is near.");
    break;
  case 6:
    console.log("It's Saturday! Enjoy your weekend.");
    break;
  case 7:
    console.log("It's Sunday! Rest and recharge.");
    break;
  default:
    console.log("Invalid day! Please enter a number between 1 and 7.");
}
```

:::
