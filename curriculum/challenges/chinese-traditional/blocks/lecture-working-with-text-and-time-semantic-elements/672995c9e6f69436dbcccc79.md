# --interactive--

在 HTML 中，引用元素用於區別引用的文字與周圍內容。這使引用的文字具有易於識別的格式。

你應該使用區塊引用元素來表述從其他來源引用的區段。它主要用於較長的引用。如果引用的來源有位址，你可以使用 `cite` 屬性來引用它。此屬性的值應該是有效的 URL。以下是一個在區塊引用元素中引用的範例：

:::interactive_editor

```html
<blockquote cite="https://www.freecodecamp.org/news/learn-to-code-book/">
  "Can you imagine what it would be like to be a successful developer? To have built software systems that people rely upon?"
</blockquote>
```

:::

此元素具有 `cite` 屬性。`cite` 屬性的值是來源的 URL。雖然此屬性不會改變區塊引用的呈現方式，但它對於提供螢幕閱讀器和搜尋引擎更多關於引用的資訊非常有幫助。在瀏覽器中，你會看到文字稍微有縮排。

如果你想用引號來開始和結束區塊引用，你可能需要在文字中明確地寫出它們。你可以像我剛才那樣，直接在區塊引用元素中撰寫文字，或者將文字包裹在一個或多個段落元素中。當文字有多個段落，但你想將它們保留在同一個區塊引用中時，這會很有幫助。以下是一個包含四個段落的範例：

:::interactive_editor

```html
<blockquote cite="https://www.freecodecamp.org/news/learn-to-code-book/">
  <p>Build your projects. Show them to your friends. Build projects for your friends.</p>
  <p>Build your network. Help the people you meet along the way. What goes around comes around. You'll get what's coming to you.</p>   
  <p>It is not too late. Life is long.</p>
  <p>You will look back on this moment years from now and be glad you made a move.</p>
</blockquote>
```

:::

所有段落都包含在同一個區塊引用元素中，因此它們是同一個引用的一部分。你可以看到該元素具有帶有來源 URL 的 `cite` 屬性。在瀏覽器中，你會看到這四個段落彼此相對齊，但它們相對於其容器有縮排。

到目前為止，我一直使用 `cite` 屬性來標註引用的原始碼，但該屬性並不會真正向使用者顯示來源。它只是在幕後運作。

如果你想要以視覺方式標示來源，你可以在區段引用元素外部添加一個引用元素 `cite`。這與 `cite` 屬性不同。引用元素是一個 HTML 元素，你可以用來標記被引用創作作品的標題，例如書籍文章、歌曲、電影、網站或研究論文。以下是一個範例：

:::interactive_editor

```html
<div>
  <blockquote cite="https://www.freecodecamp.org/news/learn-to-code-book/">
    Can you imagine what it would be like to be a successful developer? To have built software systems that people rely upon?
  </blockquote>
  <p>—Quincy Larson, <cite>How to Learn to Code and Get a Developer Job [Full Book].</cite></p>
</div>
```

:::

區塊引用元素包含被引用的 `text`。在該元素下方，你可以看到一個帶有作者名稱的段落元素，接著是一個引用元素。引用元素包含引用出處的書名。

如果你前往瀏覽器，現在原始碼將清楚可見，並且看到引文來自 Quincy Larson 所寫的一本書。這本書的標題是 `How to Learn to Code and Get a Developer Job`。

你應該使用像這樣的區塊引用來引用其他來源的長篇引文。但有時你只需要在較大的段落中引用幾個字。

這正是行內引用元素的用途。它用於來自其他來源的短行內引用。大多數現代瀏覽器在你使用此元素時會自動在行內引用周圍添加引號。這是一個範例：

:::interactive_editor

```html
<p>
  As Quincy Larson said,
  <q cite="https://www.freecodecamp.org/news/learn-to-code-book/">
    Momentum is everything.
  </q>
</p>
```

:::

你可以看到一個包含文字的段落元素。部分文字是行內引用，因為它位於行內引用元素中。你也可以為你的 `cite` 屬性添加來源。

這與區塊引用元素的運作方式完全相同。視覺上不會有任何變化，但它會為螢幕閱讀器和搜尋引擎提供更多關於引用的資訊。

在瀏覽器中，你會看到被引用的文字是段落的一部分，並且被引號包圍。大多數現代瀏覽器會自動添加這些引號。 

區塊引用和行內引用有什麼差別？你應該為來自其他來源的長篇引用使用區塊引用，並為應該成為現有段落一部分的短篇引用使用行內引用。
