# --description--

JavaScript gilt als dynamisch typisierte Sprache. Das bedeutet, dass Variablen zur Laufzeit beliebige Werte erhalten können – eine Variable als Zahl zu deklarieren, verhindert nicht, dass Sie ihr später einen String zuweisen, und Funktionsparameter können beliebige Werte übergeben bekommen.

Die Herausforderung einer dynamisch typisierten Sprache besteht darin, dass das Fehlen von Typensicherheit Fehler verursachen kann, wenn Sie nicht vorsichtig sind. TypeScript erweitert die JavaScript-Sprache um statische Typisierung, die dabei hilft, diese Fehler zu erkennen, bevor Sie Ihren Code überhaupt deployen. Aber wie funktioniert das?

Betrachten Sie diese JavaScript-Funktion:

```js
const getRandomValue = (array) => {
  return array[Math.floor(Math.random() * array.length)];
}
```

Unsere `getRandomValue`-Funktion nimmt ein Array und gibt einen zufälligen Wert aus diesem Array zurück. Da JavaScript jedoch keine Typen validiert, gibt es nichts, was Sie daran hindert, die Funktion mit einer Zahl aufzurufen:

```js
console.log(getRandomValue(10));
```

Die Ausgabe in der Konsole für dieses Beispiel wird `undefined` sein, weil ein Array erwartet wurde, aber eine Zahl übergeben wurde.

Mit TypeScript können Sie jedoch einen Typ für den Array-Parameter definieren:

```js
const getRandomValue = (array: string[]) => {
  return array[Math.floor(Math.random() * array.length)];
}
```

Diese Typdefinition sagt TypeScript, dass das Array-Argument ein Array von Strings sein muss. Wenn Sie dann `getRandomValue` aufrufen und eine Zahl übergeben, erhalten Sie einen anderen Fehlertyp, einen sogenannten Compiler-Fehler.

Die meisten JavaScript-Laufzeitumgebungen, wie ein Browser oder Node.js, können TypeScript nicht nativ ausführen. Stattdessen müssen Sie TypeScript-Code zuerst kompilieren oder übersetzen, um reguläres JavaScript zu erhalten. Das können Sie mit dem eingebauten Compiler machen, der mit der TypeScript-Sprache geliefert wird. Wenn Sie den Compiler ausführen, wertet TypeScript Ihren Code aus und wirft einen Fehler bei allen Problemen, bei denen die Typen nicht übereinstimmen – zum Beispiel wenn eine Zahl an eine Funktion übergeben wird, die ein Array erwartet.

Sie werden in den nächsten Lektionen mehr darüber lernen, wie diese Typen funktionieren. Aber gerade diese zusätzliche Sicherheit macht TypeScript für viele Entwickler und Organisationen zu einer attraktiven Sprache.
