# --interactive--

في البرمجة، تتيح لك الوراثة تعريف أصناف ترث الخصائص والطرق من أصناف أخرى.

على سبيل المثال، `Car` هو نوع من `Vehicle`، لذا يمكنك تعريف صنف `Vehicle` بأعم الخصائص والطرق المشتركة بين جميع أنواع المركبات، ثم تعريف صنف `Car` الذي يرث خصائص وطرق صنف `Vehicle`.

هذه العلاقة "هو نوع من" هي سمة مميزة للوراثة. في هذا المثال، `Car` هو شكل أكثر تخصصًا من `Vehicle`.

هناك مفهومان رئيسيان يجب أن تعرفهما لتبدأ العمل مع الوراثة: الصنف الأب والصنف الابن.

الصنف الأب هو صنف يعمل كخطة أساسية لأصناف أخرى. يعرّف خصائص وطرق ترثها الأصناف الأخرى.

الصنف الابن هو صنف يرث خصائص وطرق صنف آخر. يمكن للأصناف الابنة أيضًا توسيع وظائف أصنافها الأب بإضافة خصائص وطرق جديدة.

في مثالنا السابق، سيكون `Vehicle` هو الصنف الأب لصنف `Car`، بينما يكون صنف `Car` هو الصنف الابن لـ `Vehicle`.

يمكن أن يكون للصنف الأب عدة أصناف ابنة.

في JavaScript، نستخدم الكلمة المفتاحية `extends` لتنفيذ الوراثة. تشير هذه الكلمة إلى أن الصنف هو صنف ابن لصنف آخر.

في هذا المثال، حيث لدينا فقط الأسطر الأولى من تعريفات الأصناف، يمكنك أن ترى أن صنف `Car` يمتد من صنف `Vehicle`، لذا `Car` يرث من `Vehicle`.

```js
class Vehicle {
  // Implementation of Vehicle...
}

class Car extends Vehicle {
  // Implementation of Car...
}
```

لنفترض أن صنف `Vehicle` لديه خاصيتان: `brand` و `year`. يمكنك رؤية ذلك في الشفرة المحدثة.

لـ `Car` طريقة تسمى `honk`. ومع ذلك، لاحظ أن صنف `Car` لا يملك دالة بناء خاصة به. دعنا نرى إذا كان يرث الخصائص.

```js
class Vehicle {
  constructor(brand, year) {
    this.brand = brand;
    this.year = year;
  }
}

class Car extends Vehicle {
  honk() {
    console.log("Honk! Honk!");
  }
}
```

لتعريف نسخة من `Car`، ستحتاج إلى تمرير معلمتين: `brand` و `year`. في هذه الحالة، العلامة التجارية هي `freeCodeCamp Motors` والسنة هي `2019`.

```js
let myCar = new Car("freeCodeCamp Motors", 2019);
```

إذا حاولت الوصول إلى هذه الخصائص واستدعاء الطريقة باستخدام ترميز النقطة:

```js
console.log(myCar.brand);
console.log(myCar.year);
myCar.honk();
```

إليك المثال الكامل:

:::interactive_editor

```js
class Vehicle {
  constructor(brand, year) {
    this.brand = brand;
    this.year = year;
  }
}

class Car extends Vehicle {
  honk() {
    console.log("Honk! Honk!");
  }
}

let myCar = new Car("freeCodeCamp Motors", 2019);
console.log(myCar.brand);
console.log(myCar.year);
myCar.honk();
```

بالفعل، الناتج صحيح، لذا هذه الخصائص والطريقة معرفة في نسخة `Car` وقد وفرت على نفسك الكثير من تكرار الشفرة بوراثة هذه الخصائص من صنف `Vehicle`.

في هذا المثال، لم يكن للصنف الابن أي خصائص إضافية. لهذا السبب لم يكن للصنف دالة بناء، بل فقط طريقة.

إذا كنت بحاجة إلى إضافة خصائص إضافية، ستحتاج إلى تعريف دالة بناء.

لنر مثالًا.


```js
class Vehicle {
  constructor(brand, year) {
    this.brand = brand;
    this.year = year;
  }
}

class Car extends Vehicle {
  constructor(brand, year, numDoors) {
    super(brand, year);
    this.numDoors = numDoors;
  }
}
```

هذه المرة، سنضيف خاصية `numDoors` إلى صنف `Car` الفرعي.

يمكنك أن ترى أن هذا الصنف لديه الآن دالة بناء في الشفرة المحدثة. داخل دالة البناء، هناك استدعاء لـ `super()`، مع تمرير المعلمتين `brand` و `year`.

تستدعي `super()` دالة البناء للصنف الأم، لذا باستخدام `super(brand, year)` ستعرف خصائص الصنف الأم في الصنف الفرعي. هذه مصطلحات أخرى يمكنك استخدامها — الصنف الأم هو الصنف الأب والصنف الفرعي هو الصنف الابن.

ثم، السطر `this.numDoors = numDoors` يعرف خاصية جديدة تنتمي حصريًا إلى صنف `Car`، وهي عدد الأبواب (`numDoors`).

هذه طريقة لتوسيع صنف `Vehicle`، بإضافة خاصية حصرية لنسخ `Car`.

هذا مثال على قابلية التوسع، وهي واحدة من المزايا الأساسية للوراثة.

الآن، إذا أنشأت نسخة من صنف `Car`، سترى أنها تحتوي على الخصائص الثلاث: `brand` و `year` و `numDoors`. ستحتاج أيضًا إلى تمرير المعلمات اللازمة عند إنشاء النسخة.

```js
let myCar = new Car("freeCodeCamp Motors", 2019, 4);
```

إذا طبعت قيم هذه الخصائص باستخدام `console.log()`:

```js
console.log(myCar.brand);
console.log(myCar.year);
console.log(myCar.numDoors);
```

إليك المثال الكامل:

:::interactive_editor

```js
class Vehicle {
  constructor(brand, year) {
    this.brand = brand;
    this.year = year;
  }
}

class Car extends Vehicle {
  constructor(brand, year, numDoors) {
    super(brand, year);
    this.numDoors = numDoors;
  }
}

let myCar = new Car("freeCodeCamp Motors", 2019, 4);
console.log(myCar.brand);
console.log(myCar.year);
console.log(myCar.numDoors);
```

:::

المزايا الرئيسية للوراثة هي إعادة استخدام الشفرة، والتجزئة، وقابلية التوسع، وتحسين هيكل الشفرة.

من خلال تنفيذ التسلسل الهرمي، يمكنك إعادة استخدام الشفرة التي كتبتها بالفعل لصنف الأب في الصنف الابن، متجنبًا التكرار.

تعزز الوراثة أيضًا التجزئة بتقسيم الأنظمة المعقدة إلى مكونات أبسط في التسلسل الهرمي.

أيضًا، القدرة على توسيع وظائف الصنف الأب تجعل من السهل التكيف مع المتطلبات المتغيرة وإضافة ميزات جديدة لاحقًا في عملية التطوير.

أخيرًا، يمكن أن يجعل الهيكل الهرمي شفرتك أسهل في الفهم والصيانة.

هذه هي أساسيات الوراثة. بفهم كيفية عمل الوراثة، يمكنك تصميم برامج موجهة كائنات منظمة وقابلة للصيانة وقابلة للتوسع.
