# --interactive--

Оператор необов’язкового ланцюжка (`?.`) — це корисний інструмент у JavaScript, який дозволяє безпечно отримувати доступ до властивостей об’єкта або викликати методи, не турбуючись про те, чи існують вони. Це ніби страховка при роботі з об’єктами, які можуть мати відсутні частини.

:::interactive_editor

```js
const person = {
  name: "Alice",
  age: 30
};

console.log(person.name); // "Alice"
console.log(person.job); // undefined
```

:::

У цьому прикладі `person.name` існує, тому він виводить `Alice`. Але `person.job` не існує, тому ми отримуємо `undefined`.

Тепер уявімо, що ми хочемо отримати доступ до властивості об’єкта, яка може не існувати:

```js
const person = {
  name: "Alice",
  age: 30
};

console.log(person.address.street); // This will throw an error!
```

Цей приклад викличе `Uncaught TypeError`. Оскільки `person.address` є `undefined`, ми не можемо отримати доступ до властивості `street`. Саме тут оператор необов’язкового ланцюжка стає у пригоді. Ось приклад використання оператора необов’язкового ланцюжка:

:::interactive_editor

```ts
const user = {
  name: "John",
  profile: {
    email: "john@example.com",
    address: {
      street: "123 Main St",
      city: "Somewhere"
    }
  }
};

console.log(user?.profile?.address?.street); // "123 Main St"
console.log(user?.profile?.phone?.number);   // undefined
```

:::

Використовуючи оператор необов’язкового ланцюжка, ми кажемо JavaScript продовжувати операцію лише якщо об’єкт (або значення перед `?.`) існує і не є `null` або `undefined`.

Якщо значення перед `?.` є `null` або `undefined`, JavaScript повертає `undefined` замість того, щоб намагатися виконати операцію і викликати помилку.

Пам’ятайте, оператор необов’язкового ланцюжка найбільш корисний, коли ви не впевнені, чи існує властивість або метод. Він допомагає уникнути помилок і робить ваш код більш надійним.
