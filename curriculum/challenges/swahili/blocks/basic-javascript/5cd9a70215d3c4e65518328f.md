# --description--

Katika <a href="/learn/javascript-algorithms-and-data-structures/basic-javascript/replace-loops-using-recursion" target="_blank" rel="noopener noreferrer nofollow">changamoto iliyopita</a>, ulijifunza jinsi ya kutumia kurudia (recursion) kubadilisha mzunguko wa `for`. Sasa, tuchunguze kitendakazi kigumu zaidi kinachorudisha safu ya data ya nambari kamili mfululizo kuanzia `1` hadi nambari inayopitishwa kwa kitendakazi hicho.

Kama ilivyotajwa katika changamoto iliyopita, kutakuwa na <dfn>hali ya msingi</dfn>. Hali ya msingi inamwambia kitendakazi kinachojirudia lini hakiwezi kujiita tena. Ni hali rahisi ambapo thamani ya return tayari inajulikana. Pia kutakuwa na <dfn>wito la kurudia</dfn> ambalo hufanya kitendakazi asili kwa hoja tofauti. Ikiwa kitendakazi kimeandikwa kwa usahihi, hatimaye hali ya msingi itafikiwa.

Kwa mfano, sema unataka kuandika kitendakazi kinachojirudia kinachorudisha safu ya data yenye nambari kuanzia `1` hadi `n`. Kitendakazi hiki kitahitaji kupokea hoja, `n`, inayowakilisha nambari ya mwisho. Kisha kitahitaji kujiita mwenyewe kwa thamani ndogo ndogo za `n` hadi kifikie `1`. Unaweza kuandika kitendakazi kama ifuatavyo:

```javascript
function countup(n) {
  if (n < 1) {
    return [];
  } else {
    const countArray = countup(n - 1);
    countArray.push(n);
    return countArray;
  }
}
console.log(countup(5));
```

Thamani ya `[1, 2, 3, 4, 5]` itaonyeshwa kwenye konsoli.

Mwanzo, hii inaonekana isiyoeleweka kwa sababu thamani ya `n` *inapungua*, lakini thamani katika safu ya mwisho *inaongezeka*. Hii hutokea kwa sababu push hufanyika mwisho, baada ya wito la kurudia kurudisha thamani. Wakati `n` inasukumwa kwenye safu, `countup(n - 1)` tayari imethaminiwa na kurudisha `[1, 2, ..., n - 1]`.
