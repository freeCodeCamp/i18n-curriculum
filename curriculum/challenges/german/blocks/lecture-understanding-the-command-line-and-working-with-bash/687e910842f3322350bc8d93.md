# --description--

Lernen wir das Terminal kennen und wie es sich von der Kommandozeile unterscheidet.

Die Kommandozeile ist eine einfache Texteingabeschnittstelle, die es einem Benutzer ermöglicht, „Befehle“ einzugeben, meist in Form einer Zeichenfolge, und diese durch Drücken der „Enter“-Taste abzusenden oder auszuführen. Kommandozeilenschnittstellen sehen Sie am häufigsten innerhalb eines Terminals.

Ein Terminal ist eine spezielle Anwendung, die eine Kommandozeilenschnittstelle bietet, um systembezogene Befehle auszuführen, die über die grundlegenden Lese-/Schreiboperationen hinausgehen. Sie hören vielleicht auch von „Terminal-Emulatoren“, das sind im Grunde Anwendungen, die eine einfache Terminal-Schnittstelle einbetten, um zusätzliche Funktionen oder Features anzubieten. Für die meisten allgemeinen Zwecke können Sie diese jedoch einfach als „Terminals“ bezeichnen.

Schließlich müssen wir auch über „Shells“ sprechen. Eine Shell ist die Software, die die Kommandozeile umgibt und Ihre Eingaben als Befehle interpretiert und die Ausgabe zurückgibt.

Aber wie können Sie auf das Terminal Ihres Systems zugreifen?

Wenn Sie eine moderne Version von Windows verwenden, haben Sie wahrscheinlich zwei verschiedene Optionen. Windows bietet von Haus aus zwei verschiedene Shell-Anwendungen: Powershell und Eingabeaufforderung.

Beide Shells verfügen über eigene Terminal-Anwendungen (die den Namen der jeweiligen Shell tragen), sodass Sie über Ihr Anwendungsmenü entweder ein Powershell-Terminal oder ein Eingabeaufforderungs-Terminal starten können.

Windows bietet außerdem einen relativ neuen und modernisierten Terminal-Emulator mit dem passenden Namen Microsoft Terminal.

Sie können diese Anwendung aus dem Windows Store installieren und dann genauso starten wie Ihre Powershell- oder Eingabeaufforderungs-Terminals. Der Vorteil von Microsoft Terminal ist, dass Sie sowohl auf die Powershell- als auch auf die Eingabeaufforderungsshells aus derselben Anwendung zugreifen können, ebenso wie auf Ihre jeweilige Linux-Shell, wenn Sie Windows Subsystem for Linux konfiguriert haben.

MacOS bietet ein Terminal, das passend „Terminal“ heißt. Sie können darauf über die Spotlight-Suche oder den Anwendungsstarter zugreifen.

MacOS unterstützt auch Terminal-Emulatoren von Drittanbietern wie iTerm oder Ghostty. Diese Anwendungen können Sie von deren Webseiten installieren.

Unter Linux hängt das Standardterminal (und wie Sie es starten) stark von der verwendeten Distribution und Desktop-Umgebung ab. Es gibt auch viel Unterstützung für Terminal-Emulatoren von Drittanbietern. Zum Beispiel könnten Sie kitty unter Arch Linux einrichten – und es mit einem Anwendungsmanager wie wofi starten.

Linux hat vermutlich die größte Auswahl an Terminal-Anwendungen. Die meisten davon können Sie über den Paketmanager Ihrer Distribution installieren.

Die Unterscheidung zwischen „Terminal“, „Shell“ und „Kommandozeile“ mag manchmal etwas pedantisch erscheinen. Und meistens können die Begriffe auch austauschbar verwendet werden. Aber es ist wichtig zu wissen, dass sie spezifische Bedeutungen haben und was genau diese Bedeutungen sind.
