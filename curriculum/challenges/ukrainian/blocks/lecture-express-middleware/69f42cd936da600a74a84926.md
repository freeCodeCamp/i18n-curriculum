# --description--

Підпрограмне забезпечення — це ключове поняття в Express, яке дозволяє обробляти запити та відповіді між клієнтом і сервером. Функції підпрограмного забезпечення мають доступ до об’єкта запиту, об’єкта відповіді та функції next у циклі запит-відповідь застосунку.

## Що таке підпрограмне забезпечення?

Підпрограмне забезпечення — це, по суті, функція, яка виконується під час життєвого циклу запиту до сервера. Вона виконує різні завдання, такі як логування запитів, обробка помилок, розбір вхідних запитів або обслуговування статичних файлів.

Express надає зручний спосіб визначати підпрограмне забезпечення, яке обробляє вхідні запити перед тим, як вони потраплять до обробників маршрутів.

Як працює підпрограмне забезпечення:

Коли надходить запит до застосунку Express, підпрограмне забезпечення виконується в тому порядку, в якому його додано. Потік виглядає так:

* Запит надсилається на сервер.
* Функції підпрограмного забезпечення обробляють запит.
* Якщо підпрограмне забезпечення викликає `next()`, воно передає керування наступному підпрограмному забезпеченню або обробнику маршруту.
* Якщо підпрограмне забезпечення не викликає `next()`, цикл запит-відповідь зупиняється.

```js
const express = require('express')
const app = express()

// Simple middleware that logs request info
app.use((req, res, next) => {
  console.log('Request URL:', req.url)
  console.log('Request Method:', req.method)
  next() // Passes control to the next middleware
})

app.get('/', (req, res) => {
  res.send('Hello World!')
})

app.listen(3000, () => {
  console.log('Server running on http://localhost:3000')
})
```

`app.use()`: Додає підпрограмне забезпечення глобально до вашого застосунку.

У цьому прикладі підпрограмне забезпечення логуватиме деталі запиту і викликає `next()`, щоб перейти до наступного підпрограмного забезпечення або обробника маршруту.

Підпрограмне забезпечення — це потужна функція в Express, яка дозволяє перехоплювати та змінювати запити в різних точках. Ви можете використовувати підпрограмне забезпечення для обробки таких речей, як логування, автентифікація або навіть змінювати об’єкти запиту та відповіді перед тим, як вони потраплять до ваших обробників маршрутів.
