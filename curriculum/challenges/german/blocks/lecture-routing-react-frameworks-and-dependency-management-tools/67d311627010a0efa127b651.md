# --description--

In früheren Lektionen haben Sie gelernt, dass React eine Single-Page-Anwendung ist. Single-Page-Anwendungen sind Anwendungen, die eine HTML-Datei enthalten und JavaScript verwenden, um den Inhalt der Seite dynamisch zu aktualisieren.

Was passiert also, wenn Sie mehrere „Seiten“ zu Ihrer React-Anwendung hinzufügen müssen? Wie navigieren Sie zu diesen verschiedenen Ansichten?

Genau hier kommt React Router ins Spiel.

React Router ist eine Drittanbieter-Bibliothek, mit der Sie Routing zu Ihren React-Anwendungen hinzufügen können. Zunächst müssen Sie React Router in einem bestehenden React-Projekt so installieren:

```sh
npm i react-router
```

Wenn Sie die `package.json`-Datei überprüfen, sehen Sie, dass `react-router` zur Liste der Abhängigkeiten hinzugefügt wurde:

```json
"dependencies": {
  "react": "^18.3.1",
  "react-dom": "^18.3.1",
  "react-router": "^7.2.0"
}
```

Dann müssen Sie in Ihrer `main.jsx`- oder `index.jsx`-Datei `BrowserRouter` importieren und `BrowserRouter` um Ihre `App`-Komponente rendern:

```jsx
import { StrictMode } from "react";
import { createRoot } from "react-dom/client";
import { BrowserRouter } from "react-router";
import App from "./App.jsx";

import "./index.css";

createRoot(document.getElementById("root")).render(
  <StrictMode>
    <BrowserRouter>
      <App />
    </BrowserRouter>
  </StrictMode>
);
```

Um Routen in Ihrer Anwendung zu aktivieren, müssen Sie Ihre `import`-Anweisung aktualisieren, um die `Routes`- und `Route`-Komponenten wie folgt einzuschließen:

```js
import { BrowserRouter, Routes, Route } from "react-router";
```

Dann fügen Sie innerhalb von `BrowserRouter` die `Routes`- und `Route`-Komponenten hinzu:

```jsx
createRoot(document.getElementById("root")).render(
  <StrictMode>
    <BrowserRouter>
      <Routes>
        <Route path="/" element={<App />} />
      </Routes>
    </BrowserRouter>
  </StrictMode>
);
```

Die `path` und `element` werden verwendet, um die URL und die UI-Komponenten miteinander zu koppeln. In diesem Fall richten wir eine Route für die Startseite ein, die auf die `App`-Komponente zeigt.

Es ist üblich, in größeren Anwendungen mehrere Ansichten und Routen wie diese einzurichten:

```jsx
<Routes>
  <Route index element={<Home />} />
  <Route path="about" element={<About />} />

  <Route path="products">
    <Route index element={<ProductsHome />} />
    <Route path=":category" element={<Category />} />
    <Route path=":category/:productId" element={<ProductDetail />} />
    <Route path="trending" element={<Trending />} />
  </Route>
</Routes>
```

Die `index`-Prop in diesen Beispielen soll die Standardroute für ein bestimmtes Pfadsegment darstellen. Die `Home`-Komponente wird also am Stamm-Pfad (`/`) angezeigt, während die `ProductsHome`-Komponente am Pfad `products` angezeigt wird.

Sie haben vielleicht auch bemerkt, dass wir einige Routen innerhalb einer anderen Route verschachteln, so:

```jsx
<Route path="products">
  <Route path="trending" element={<Trending />} />
</Route>
```

Das bedeutet, dass der Pfad der Kindroute an den Pfad der Elternroute angehängt wird. In diesem Beispiel wäre der `path` für die angesagten Produkte `products/trending`.

Beginnt der `path` mit einem Doppelpunkt (`:`), dann stellt das ein dynamisches Segment in der Route dar:

```jsx
<Route path=":category" element={<Category />} />
```

In diesem Beispiel haben wir ein dynamisches Segment namens `category`. Wenn ein Benutzer zu einer URL wie `products/brass-instruments` navigiert, ändert sich die Ansicht zur `Category`-Komponente, und Sie können die passenden Daten basierend auf dem Segment dynamisch abrufen.

Sie können den Wert des dynamischen Segments mit dem `useParams`-Hook innerhalb der Kindkomponente so abrufen:

```jsx
import { useParams } from "react-router";

export default function Category() {
  let params = useParams();
   {/* Accessing the category param: params.category */}
   {/* rest of code goes here */}
}
```

Dynamische Routen sind hilfreich, weil sie es Ihnen ermöglichen, flexible und wiederverwendbare Komponenten zu erstellen, die je nach Parametern in der URL unterschiedliche Inhalte rendern. Anstatt eine feste Liste von Pfaden für jede mögliche Route zu definieren, können Sie dynamische Segmente verwenden, um verschiedene Inhalte basierend auf der Benutzeranfrage darzustellen.
