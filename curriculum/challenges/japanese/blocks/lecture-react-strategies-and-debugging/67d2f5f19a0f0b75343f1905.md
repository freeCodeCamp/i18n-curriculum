# --description--

React Server Components（RSC）は、React開発者のアプローチを変えた新しいトレンドです。RSCを使うと、より多くの処理がサーバー側に移り、多くの利点があります。

サーバーコンポーネントとは何か、どのように動作するのか、そしてサーバーコンポーネントが導入された背景を見ていきましょう。

React Server Componentsは、サーバー上でのみレンダリングされ、最終的なHTMLだけをクライアントに送信するReactコンポーネントです。これにより、これらのコンポーネントはサーバー側のリソースに直接アクセスでき、ブラウザに送信されるJavaScriptの量を大幅に減らせます。

Reactアプリは従来、「クライアントコンポーネント」システムを使って、レンダリング、インタラクティビティ、サイドエフェクトなどを処理してきました。「クライアントコンポーネント」という用語は、最近React Server Componentsが登場するまではあまり使われていませんでした。

しかし、クライアントコンポーネントシステムには、大きなJavaScriptバンドルや初期読み込みの遅さといった欠点があります。

Next.jsやGatsbyのようなReactフレームワークは、これらの問題を解決するために一部の処理をサーバーにオフロードする方法を見つけましたが、どれも標準化されていませんでした。もしこれらのフレームワークを使ったことがあれば、`getServerSideProps`や`getServerData`について聞いたことがあるかもしれません。

そしてReact Server Componentsが登場し、一部のコンポーネントを完全にサーバー上で実行できるようになりました。これにより、ユーザーのブラウザでコードが実行される前にデータ取得や計算が可能になりました。

サーバーコンポーネントはNext.jsで最初に普及し、すぐに利用可能になりました。RemixやGatsbyなど他のフレームワークも追随しており、Vite用の実験的なプラグイン`vite-plugin-react-server`もサーバーコンポーネントの構築を可能にしています。

では、サーバーコンポーネントはどのように動作するのでしょうか？

React Server Componentsを示す最良の方法の一つはデータ取得です。

従来のReactクライアントコンポーネントでは、ブラウザがAPIリクエストを処理します。データ取得はサイドエフェクトなので、`useEffect`フック内でAPIコールを行います。

また、loading、data、errorのような状態変数を設定して、データが読み込み中であることを示したり、データが準備できたら表示したり、エラーをアプリに表示したりするのが良い習慣です。

React Server Componentsでは、コンポーネント全体をサーバーに移し、`useState`や`useEffect`を使わずにデータを取得できます。

```jsx
const Users = async () => {
  const res = await fetch("https://jsonplaceholder.typicode.com/users");
  const users = await res.json();

  return (
    <>
      <h1 className="text-4xl text-center mt-6">Users</h1>
      <ul className="text-center mt-3">
        {users.map((user) => (
          <li key={user.id}>{user.name}</li>
        ))}
      </ul>
    </>
  );
};

export default Users;
```

React Server Componentsはサーバー上でのみ実行されるため、APIからデータを取得して一度だけレンダリングすれば済みます。また、データ取得がサーバー側で行われるため、ソースに近く、特にネットワークが遅いユーザーにとってアプリのパフォーマンスが向上する場合があります。

大きな注意点は、サーバーコンポーネントのコードはすべてサーバーに残り、ブラウザには送られないことです。つまり、Reactフックは使えず、Web APIやブラウザのイベントリスナーにもアクセスできません。では、どうやってインタラクティビティを追加するのでしょうか？

Next.jsのアプリルーターでは、すべてのコンポーネントがデフォルトでサーバーコンポーネントです。インタラクティビティを追加したい場合は、`"use client"`ディレクティブでコンポーネントをクライアントコンポーネントとしてマークする必要があります。

先ほどの例をクライアントコンポーネントにしたいとしましょう。方法は以下の通りです。

```jsx
"use client";

import { useState, useEffect } from "react";

const Users2 = () => {
  const [status, setStatus] = useState({
    users: [],
    loading: true,
    error: null,
  });

  async function fetchUsers() {
    try {
      const res = await fetch("https://jsonplaceholder.typicode.com/users");
      const data = await res.json();
      setStatus((prevStatus) => ({
        ...prevStatus,
        users: data,
        loading: false,
      }));
    } catch (err) {
      setStatus((prevStatus) => ({
        ...prevStatus,
        error: err.message,
        loading: false,
      }));
    }
  }

  useEffect(() => {
    fetchUsers();
  }, []);

  if (status.loading) {
    return <p>Loading Users...</p>;
  }
  if (status.error) {
    return <p>Error getting users: {status.error}</p>;
  }

  return (
    <>
      <h1 className="text-4xl text-center mt-6">Users</h1>
      <ul className="text-center mt-3">
        {status.users.map((user) => (
          <li key={user.id}>{user.name}</li>
        ))}
      </ul>
    </>
  );
};

export default Users2;
```

クリックイベントのようなインタラクティビティを追加したい場合も、コンポーネントをクライアントコンポーネントとしてマークする必要があります。

```jsx
"use client";

import { useState } from "react";

const Counter = () => {
  const [count, setCount] = useState(0);

  return (
    <>
      <h1>Counter</h1>
      <button onClick={() => setCount(count + 1)}>Increment</button>
      <h2>{count}</h2>
      <button onClick={() => setCount(count - 1)}>Decrement</button>
    </>
  );
};

export default Counter;
```

コンポーネントにuse clientディレクティブを追加しないと、「`useState`が必要なサーバーコンポーネントをインポートしています。このReactフックはクライアントコンポーネントでのみ動作します。修正するには、ファイル（または親）に`"use client"`ディレクティブをマークしてください」というエラーメッセージが表示されます。

React Server Componentsの主な利点は、データ取得がシンプルになり、コードが読みやすくなり、クライアントの複雑さが減ることです。
