# --description--

في React، قد تواجه مواقف تحتاج فيها إلى وصول مباشر إلى عنصر DOM. هنا تأتي فائدة "refs". يمكن للـ refs أيضًا تخزين قيم قابلة للتغيير، لكن الحالة (state) خيار أفضل لذلك.

في جافاسكريبت العادية، كنت تستخدم دالتي `getElementById()` و `querySelector()` للوصول إلى عناصر DOM. لكن في React، تستخدم refs للوصول إلى العناصر في DOM.

أحد الاختلافات الرئيسية هو أنه مع refs، لا تحتاج إلى معرفات مثل IDs أو أصناف (classes) للإشارة إلى العناصر.

فكيف يمكنك إنشاء refs واستخدامها؟ يوفر React هوك `useRef()` الذي يتيح لك ذلك.

الخطوة الأولى هي استيراد الهوك من React:

```js
import { useRef } from "react";
```

بعدها، تحتاج إلى إنشاء متغير يحمل الـ ref مع القيمة الابتدائية للـ ref داخل هوك `useRef`، مثلاً `sectionRef` مُهيأ إلى `null`:

```js
const sectionRef = useRef(null);
```

الخطوة الأخيرة هي ربط متغير الـ ref بالعنصر في JSX باستخدام الخاصية `ref`:

```jsx
<section ref={sectionRef}>
  {/* Section content */}
</section>
```

إذا سجلت الـ ref في وحدة التحكم، سترى أنه كائن يحتوي على القيمة الحالية، في هذه الحالة `null`:

```js
console.log(sectionRef); // { current: null }
```

يمكنك أيضًا تسجيل القيمة الحالية في وحدة التحكم باستخدام الخاصية `current` لترى القيمة مباشرة:

```js
console.log(sectionRef.current); // null
```

القيم اللاحقة للـ ref تعتمد على دورة حياة المكوِّن.

على سبيل المثال، القيمة الابتدائية لـ `sectionRef` ستكون دائمًا `null` لأنها القيمة التي تم تهيئتها بها. بعد تركيب المكوِّن، ستكون قيمة الـ ref هي عنصر `section` الذي تم ربط الـ ref به.

إذا تم إزالة المكوِّن، تعود قيمة الـ ref إلى القيمة الابتدائية `null`.

مثال نموذجي لعرض ref هو تركيز عنصر مُدخَل عند العرض، أو عند نقر زر.

إليك كيفية القيام بذلك عند نقر زر:

```jsx
import { useRef } from "react";

const Focus = () => {
  const inputRef = useRef(null);

  const handleFocus = () => {
    if (inputRef.current) {
      inputRef.current.focus();
    }
  };

  return (
    <div>
      <input ref={inputRef} type="text" placeholder="Enter text" />
      <button onClick={handleFocus}>Focus Input</button>
    </div>
  );
};

export default Focus;
```

في الشفرة أعلاه، تم إنشاء `inputRef` وربطه بعنصر `input`. هناك أيضًا زر مع حدث `onClick` يستدعي دالة `handleFocus`.

كل ما تفعله دالة `handleFocus` هو استدعاء طريقة `focus()` على عنصر `input`. لاحظ أنه، لأن `input` مكوِّن مدمج يأتي مع React، فإن عنصر DOM الفعلي لـ `input` يُخزن في الخاصية `current` للـ ref. لذا تستدعي طريقة `focus()` عبر `input.current.focus()`.

إليك بعض أفضل الممارسات التي يجب أن تعرفها أثناء العمل مع refs:

- استخدم refs بشكل رئيسي للتفاعل مع DOM. يمكنك أيضًا استخدامها للبيانات القابلة للتغيير، لكن الحالة خيار أفضل لذلك.

- لا تستخدم refs لإدارة الحالة الأساسية – لهذا الغرض يوجد `useState`.

- تأكد من التحقق من وجود `ref.current` قبل الوصول إلى خصائصه. إليك كيفية القيام بذلك مرة أخرى:

```js
const handleFocus = () => {
  if (inputRef.current) {
    inputRef.current.focus();
  }
};
```

هذا يمنع الأخطاء في حال تم الوصول إلى الـ ref قبل ربطه بـ DOM أو بعد إزالته.
