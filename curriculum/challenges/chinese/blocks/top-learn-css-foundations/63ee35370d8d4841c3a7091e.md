# --description--

组合器允许我们以不同于分组或链接选择器的方式组合多个选择器，因为它们表现出选择器之间的关系。组合器总共有四种类型，但现在我们只向你展示后代组合器，它在 CSS 中由选择器之间的单个空格表现。后代组合器只有在元素匹配最后一个选择器且其祖先（父元素、祖父元素等）匹配前一个选择器时，才会选择这些元素。

所以像 `.ancestor .child` 这样的选择器会选中具有 `child` 类的元素，前提是它有一个具有 `ancestor` 类的祖先。另一种理解方式是，只有当 `child` 嵌套在 `ancestor` 内部时，无论嵌套多深，才会被选中。快速看看下面的示例，看看你能否根据提供的 CSS 规则判断哪些元素会被选中：

```html
<!-- index.html -->

<div class="ancestor"> <!-- A -->
  <div class="contents"> <!-- B -->
    <div class="contents"> <!-- C -->
    </div>
  </div>
</div>

<div class="contents"></div> <!-- D -->
```

```css
/* styles.css */

.ancestor .contents {
  /* some declarations */
}
```

在上面的示例中，前两个具有 `contents` 类的元素（`B` 和 `C`）将被选中，但最后一个元素（`D`）不会。你的猜测正确吗？

你可以为一个规则添加的组合器数量实际上没有限制，所以 `.one .two .three .four` 是完全有效的。这将选择一个具有 `four` 类的元素，前提是它有一个具有 `three` 类的祖先，并且该祖先又有一个具有 `two` 类的祖先，依此类推。不过，你通常应避免选择需要这种层级嵌套的元素，因为这会变得相当混乱且冗长，并且在特异性方面可能会引发问题。

# --questions--    

## --text--

后代组合器的作用是什么？

## --answers--

它将共享相同声明的某些类分组在一起。

---

它提供了选择具有相同 `class` 和 `id` 的元素的能力。

---

它允许你根据元素与其祖先（父元素、祖父元素等）的关系来选择元素。
