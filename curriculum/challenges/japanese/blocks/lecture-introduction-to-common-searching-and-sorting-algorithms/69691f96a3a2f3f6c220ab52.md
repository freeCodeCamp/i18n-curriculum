# --description--

リストの中からアイテムを検索することは、コンピューターサイエンスでよくある操作です。検索に関して知っておくべき重要なアルゴリズムは、リニアサーチとバイナリサーチの2つです。

リニアサーチはリストの先頭から始めて、目的の値が見つかるまで各アイテムを反復します。

目的の値が見つかれば、その値がリストのどのインデックスにあるかを返します。目的の値が見つからなければ `-1` を返します。多くのプログラミング言語で有効なインデックスではないため、`-1` を返します。

リニアサーチのコードは以下のようになります。

```javascript
function linearSearch(arr, target) {
    for (let i = 0; i < arr.length; i++) {
        if (arr[i] === target) {
            return i;
        }
    }
    return -1;
}
```

検索対象のリストが `[13, 4, 7, 9, 10]` で、目的の値が `9` の場合、この関数は `3` を返します。なぜなら `9` はインデックス `3` にあるからです。

もし目的の値を `5` に変えた場合、この関数は `-1` を返します。なぜなら `5` はリストに存在しないからです。

このアルゴリズムは比較的単純ですが、最も効率的とは言えません。大きなリストの場合、リニアサーチは目的の値を見つけるのに時間がかかることがあります。

リニアサーチの時間計算量は `O(n)` です。なぜならリストのサイズに比例して検索にかかる時間が増えるからです。

リニアサーチの空間計算量は `O(1)` です。なぜならリストを検索するのに追加の空間を必要としないからです。

バイナリサーチは大きなリストを検索するのにより効率的なアルゴリズムです。ただし、リストは昇順にソートされている必要があります。

バイナリサーチはリストを半分に分割し、目的の値がリストの中央にあるかを確認します。目的の値が中央にあれば、そのインデックスを返します。そうでなければ、目的の値がリストの左半分か右半分かを調べます。

残りのリストを半分に分割する操作を繰り返し、目的の値が見つかるまで続けます。目的の値がリストにない場合は `-1` を返します。

バイナリサーチのコードは以下のようになります。

```javascript
function binarySearch(arr, target) {
    let low = 0;
    let high = arr.length - 1;

    while (low <= high) {
        const mid = Math.floor((low + high) / 2);

        if (arr[mid] === target) {
            return mid;
        } else if (arr[mid] < target) {
            low = mid + 1;
        } else {
            high = mid - 1;
        }
    }

    return -1;
}
```

まず `low` と `high` のインデックスを特定します。これは検索対象のリストの範囲を表します。

次に `low` が `high` 以下であるかを確認します。もし `low` が `high` より大きければ、リスト全体を検索し終えたことになり、目的の値は見つかりません。この場合、検索を停止して `-1` を返します。

`low` のインデックスが `high` のインデックス以下であれば、リストの中央のインデックス `mid` を計算します。次に目的の値が中央のインデックスにあるかを確認します。あれば中央のインデックスを返します。

そうでなければ、中央の値が目的の値より小さいかを確認します。小さければ、低いインデックスを中央のインデックスプラス1に更新します。これはリストの右半分を検索することを意味します。

最後に、他の条件がすべて `True` の場合、`high` のインデックスを中央のインデックスマイナス1に更新します。これはリストの左半分を検索することを意味します。

この処理を目的の値が見つかるか、リストにないと判断されるまで繰り返します。

バイナリサーチの時間計算量は `O(log n)` です。なぜならリストのサイズに対して対数的に検索時間が増えるからです。

バイナリサーチの空間計算量は `O(1)` です。なぜならリストを検索するのに追加の空間を必要としないからです。

バイナリサーチとリニアサーチは、コンピューターサイエンスで遭遇するさまざまな問題に使えます。両者の違いと使いどころを理解することが重要です。
