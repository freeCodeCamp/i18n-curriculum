## Reine vs. unreine Funktionen

- Eine reine Funktion ist eine Funktion, die für dieselbe Eingabe immer dieselbe Ausgabe liefert und keine Nebeneffekte hat. Ihre Ausgabe hängt nur von der Eingabe ab, und sie verändert keinen externen Zustand.
- Unreine Funktionen haben Nebeneffekte, das sind Änderungen am Zustand des Programms, die außerhalb der Funktion beobachtbar sind.

## Funktionale Programmierung

- Funktionale Programmierung ist ein Ansatz zur Softwareentwicklung, der die Verwendung von Funktionen zur Problemlösung betont und sich darauf konzentriert, was getan werden muss, statt wie es getan wird.
- Funktionale Programmierung fördert Techniken, die helfen, Nebeneffekte zu vermeiden, wie die Verwendung unveränderlicher Datenstrukturen und höherwertiger Funktionen.
- Wenn funktionale Programmiergrundsätze richtig angewendet werden, führt das zu saubererem und besser wartbarem Code.

## Currying

- Currying ist eine Technik der funktionalen Programmierung, die eine Funktion mit mehreren Argumenten in eine Folge von Funktionen umwandelt, von denen jede ein einzelnes Argument entgegennimmt.

Hier ist ein Beispiel für eine normale Funktion im Vergleich zu einer curried Funktion:

:::interactive_editor

```js
// Regular function
function average(a, b, c) {
  return (a + b + c) / 3;
}

console.log(average(2, 3, 4)); // 3

// Curried function
function curriedAverage(a) {
  return function(b) {
    return function(c) {
      return (a + b + c) / 3;
    };
  };
}

console.log(curriedAverage(2)(3)(4)); // 3
```

:::

- Currying kann besonders mächtig sein, wenn man mit Funktionen arbeitet, die viele Argumente erwarten.
- Currying macht Ihren Code flexibler und leichter wiederverwendbar.
- Sie können Pfeilfunktionen verwenden, um curried Funktionen prägnanter zu erstellen:

```js
const curriedAverage = a => b => c => (a + b + c) / 3;
```

- Obwohl Currying zu flexiblerem und wiederverwendbarem Code führt, kann es den Code schwerer lesbar machen, wenn es übermäßig eingesetzt wird.
