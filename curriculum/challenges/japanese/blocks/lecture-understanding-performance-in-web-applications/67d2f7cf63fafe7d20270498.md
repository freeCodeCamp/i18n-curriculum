# --description--

LighthouseやChrome開発者ツールのようなツールは便利ですが、時には自分のサイトに合わせたカスタムのインサイトが必要になることがあります。

そこで役立つのがパフォーマンスWeb APIです！

このレッスンでは、ウェブサイトの速度を計測・分析するために使える3つの重要なWeb APIを分かりやすく解説します。

まず、パフォーマンスWeb APIとは何でしょうか？

パフォーマンスWeb APIは、開発者がコードから直接ウェブページの読み込み効率やレスポンスを追跡できる仕組みです。これらのAPIを使うと、ページの読み込み時間を計測したり、レンダリングや操作の遅延を追跡したり、JavaScriptの実行時間を分析したりできます。

これらのAPIを使えば、サードパーティのソフトに頼らずに自分だけのパフォーマンス監視ツールを作れます！

それでは、今日から使える強力な3つのWeb APIを見ていきましょう。

まずは`performance.now()`です。

このAPIは高精度なタイムスタンプ（ミリ秒単位）を提供し、サイトのさまざまな部分がどれくらい時間をかけて読み込まれているかを測れます。

例えば、関数の実行速度を計測したいときは：

```js
const start = performance.now();  
// Run some code here  
const end = performance.now();  

console.log(`Execution time: ${end - start}ms`);
```

これは`Date.now()`を使うよりも正確です。なぜなら、ミリ秒単位でマイクロ秒の精度を持って時間を測り、クロックドリフトの問題を避けられるからです。スクリプトの実行時間、イベントのレスポンス遅延、アニメーションのパフォーマンスを追跡するのに使えます。

次に、Performance Timing APIです。

このAPIはDNSルックアップから`DOMContentLoaded`まで、ページ読み込みのあらゆる段階を詳細に分解して教えてくれます。

ページが完全に読み込まれるまでの時間を計測したいですか？

```js
let [navigationTiming] = performance.getEntriesByType("navigation");

if (navigationTiming instanceof PerformanceNavigationTiming) {
  // Calculate time from navigation start to DOM content loaded
  const pageLoadTime =
    navigationTiming.domContentLoadedEventEnd - navigationTiming.startTime;

  console.log("DOM Content Loaded Time:", pageLoadTime, "ms");
}
```

このAPIで追跡できる主な指標は、DNSルックアップ時間（つまり接続速度）、Time to First Byte（TTFB、サーバーレスポンス速度）、そして`DOMContentLoaded`（つまりページが操作可能になるタイミング）です。

もしページの読み込み時間が遅い場合、このAPIがどこで遅延が起きているかを正確に特定してくれます！

最後に、`PerformanceObserver`について説明します。

このAPIはレイアウトシフト、長時間タスク、ユーザー操作などのパフォーマンスイベントを監視します。

長時間実行されるJavaScriptタスクを監視したいですか？

```js
const observer = new PerformanceObserver((list) => {  
  list.getEntries().forEach((entry) => {  
    console.log(`Long task detected: ${entry.duration}ms`);  
  });  
});  

observer.observe({ type: "longtask", buffered: true });
```

このAPIが追跡できるのは、レンダリングをブロックする長時間タスク、UIのジャンクを検出するレイアウトシフト、そしてFirst Input Delay（FID、ページがユーザー入力にどれだけ速く反応するか）です。

リアルタイムのパフォーマンス追跡をしたいなら、このAPIは非常に役立ちます。

では、どのAPIを使うべきでしょうか？簡単に比較してみましょう：

| Performance API        | 最適な用途                                         |
| :--------------------- | :------------------------------------------------- |
| `performance.now()`    | 関数やスクリプトの正確なタイミング計測             |
| Performance Timing API | ページ全体の読み込みパフォーマンスの計測           |
| Performance Observer   | インタラクションやレンダリングのリアルタイム監視   |

これらのAPIを組み合わせることで、自分のサイトにとって最も重要なパフォーマンスを正確に追跡できる独自の計測ツールを作れます。
