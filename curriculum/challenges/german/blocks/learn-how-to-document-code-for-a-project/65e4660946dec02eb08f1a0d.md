# --description--

Eine `README`-Datei ist ein wichtiger Bestandteil vieler Projekte, insbesondere in der Softwareentwicklung. Sie ist oft das erste Dokument, das Menschen sehen, wenn sie sich ein Projekt ansehen. Eine `README`-Datei erklärt, worum es bei dem Projekt geht, wie man es benutzt und manchmal auch, wie man dazu beitragen kann. Zu verstehen, wofür eine `README`-Datei verwendet wird, ist im Kontext der Projektdokumentation entscheidend.

# --questions--

## --text--

Wofür wird eine `README`-Datei in einem Projekt üblicherweise verwendet?

## --answers--

Code speichern

### --feedback--

Obwohl Code ein wesentlicher Bestandteil eines Projekts ist, wird die `README`-Datei nicht zum Speichern von Code verwendet, sondern um Informationen über das Projekt bereitzustellen.

---

Anweisungen zum Projekt

---

Kontaktinformationen

### --feedback--

Auch wenn Kontaktinformationen enthalten sein können, ist der Hauptzweck einer `README`-Datei, Anweisungen und Informationen über das Projekt zu geben, nicht nur Kontaktdaten.

---

Änderungsprotokoll

### --feedback--

Ein Änderungsprotokoll, oft Changelog genannt, ist normalerweise getrennt von einer `README`-Datei. Die `README`-Datei konzentriert sich darauf, das Projekt vorzustellen und zu erklären.
