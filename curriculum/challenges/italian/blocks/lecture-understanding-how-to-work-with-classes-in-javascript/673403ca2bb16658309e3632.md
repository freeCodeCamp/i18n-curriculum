# --interactive--

Hai imparato che le classi sono come modelli che puoi usare per creare oggetti.

Questi modelli hanno bisogno di un modo generale per riferirsi all’oggetto specifico che viene creato, consultato o modificato all’interno del codice. Proprio per questo si usa la parola chiave `this`. Essa si riferisce al contesto in cui il codice deve essere eseguito. Se la usi all’interno di un metodo, il valore di `this` sarà un riferimento all’oggetto associato a quel metodo. In questo modo puoi consultare le proprietà e i metodi dell’oggetto all’interno della classe e riutilizzare il metodo su oggetti diversi.

Per esempio, qui abbiamo una classe `Dessert`. Questa volta stiamo definendo una proprietà `hasPeanuts`.

```js
class Dessert {
  constructor(hasPeanuts) {
    this.hasPeanuts = hasPeanuts;
  }
}
```

Nota che stiamo usando la parola chiave `this` all’interno del costruttore. Ricorda che il costruttore viene chiamato automaticamente quando si crea un nuovo oggetto della classe.

La parola chiave `this` dice al programma che `hasPeanuts` deve essere una proprietà dell’oggetto che si sta creando in quel momento.

Il lato destro dell’istruzione di assegnazione assegna il valore del parametro `hasPeanuts` alla proprietà `hasPeanuts` del nuovo oggetto.

```js
this.hasPeanuts = hasPeanuts;
```

Questo è nel contesto del costruttore, ma puoi usare la parola chiave `this` anche all’interno di altri metodi.

Questa è la nostra classe `Dessert` con qualche modifica. Ora abbiamo due proprietà: `name` e `price`.

```js
class Dessert {
  constructor(name, price) {
    this.name = name;
    this.price = price;
  }

  showPrice() {
    console.log(`The price of ${this.name} is $${this.price}.`);
  }
}
```

Abbiamo anche un nuovo metodo `showPrice`. In questo metodo stampiamo un messaggio descrittivo nella console con il nome e il prezzo dell’oggetto dessert.

Nota come stiamo usando la parola chiave `this` per consultare le proprietà dell’oggetto all’interno del metodo `showPrice`.

Quale valore avrà la parola chiave `this` nel contesto di questo metodo? Si riferirà all’oggetto su cui il metodo viene chiamato. In questo caso stiamo creando una nuova istanza Dessert con il nome `"Brownie"` e il prezzo `$5.99`:

```js
const brownie = new Dessert("Brownie", 5.99);
```

Se chiamiamo il metodo `.showPrice()` su questo oggetto:

```js
brownie.showPrice();
```

Ecco l’esempio completo:

:::interactive_editor

```js
class Dessert {
  constructor(name, price) {
    this.name = name;
    this.price = price;
  }

  showPrice() {
    console.log(`The price of ${this.name} is $${this.price}.`);
  }
}

const brownie = new Dessert("Brownie", 5.99);
brownie.showPrice();
console.log(brownie);
```

:::

Il nome e il prezzo dell’oggetto corrente (`brownie`) vengono sostituiti nella stringa.

Tutto questo è possibile grazie alla parola chiave `this`, che ti dà un modo generale per riferirti all’oggetto corrente all’interno della classe. Questo ti permette di riutilizzare il metodo su oggetti diversi.

Questo è nel contesto dei metodi degli oggetti, ma la parola chiave `this` avrà un valore anche nelle funzioni autonome e nelle funzioni freccia.

Il suo valore all’interno di una funzione autonoma di solito si riferisce all’oggetto globale (in modalità non strict) o a `undefined` (in modalità strict).

Il valore della parola chiave `this` è un po’ diverso nelle funzioni freccia. Le funzioni freccia ereditano il valore di `this` dall’ambito esterno in cui sono definite. Non creano un proprio binding di `this`. Questo significa che il valore di `this` dipende dal contesto in cui la funzione freccia è definita, non da dove o come viene chiamata. Questo rende le funzioni freccia utili per le callback o per preservare il contesto.

Tuttavia, proprio per questo, il valore di `this` non si riferirà all’oggetto corrente nei metodi degli oggetti, quindi è qualcosa da tenere a mente quando lavori con le funzioni freccia.

Lo stesso vale per le funzioni freccia definite all’interno di altre funzioni. Erediteranno il valore di `this` dal loro ambito genitore.

La parola chiave `this` è uno strumento potente per consultare e manipolare oggetti. Capire come funziona è essenziale per padroneggiare la programmazione orientata agli oggetti in JavaScript.
