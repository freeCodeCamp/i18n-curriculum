# --description--

컴포넌트는 React 애플리케이션의 기본 구성 요소로, 개발자가 복잡한 사용자 인터페이스를 더 작고 관리하기 쉬운 조각으로 나누어 대규모 애플리케이션을 더 쉽게 개발하고 유지 관리할 수 있게 합니다.

React에서 컴포넌트의 두 가지 유형은 함수형과 클래스 기반 컴포넌트입니다. 현대 React에서는 개발자가 함수형 컴포넌트를 사용하며, 오늘 살펴볼 모든 예제도 함수형 컴포넌트입니다.

더 높은 수준에서 컴포넌트를 UI를 설명하는 요소를 반환하는 JavaScript 함수처럼 생각할 수 있습니다.

이 UI는 JSX를 사용해 설명하는데, JSX는 JavaScript의 구문 확장으로 HTML과 비슷하게 보이지만 UI 요소를 더 선언적으로 작성할 수 있게 해줍니다.

React 컴포넌트 예제를 살펴보겠습니다:

```jsx
function Greeting() {
  const name = "John"
  {/* The result will be Hello John*/}
  return <h1 className="title">Hello {name}</h1>;
}
```

이 예제에서는 `Greeting`라는 컴포넌트를 정의했습니다. `{}` 태그 안의 중괄호 `h1`는 JavaScript 표현식을 임베드할 수 있게 해주어 `name` 요소 내에서 `h1` 변수를 접근할 수 있게 합니다.

또한 `className` 요소에 `title`라는 `h1`를 적용하고 있습니다.

그런데 왜 일반 HTML 요소처럼 `className` 대신 `class`를 사용하나요?

이는 JavaScript에서 `class`가 예약어이기 때문입니다. 그래서 `className`를 대신 사용해야 합니다.

JSX에서 결과를 보여주는 주석도 사용하고 있습니다. 일반 주석 구문을 이렇게 사용할 수 있지만, 작동하려면 중괄호로 감싸야 한다는 점이 중요합니다:

```jsx
{/* Block Comments */}
```

또 한 가지 눈치챘을 수도 있는 점은 컴포넌트 이름의 첫 글자를 대문자로 사용한다는 것입니다. 그런데 왜 모두 소문자를 사용할 수 없나요?

React는 대문자로 시작하는 컴포넌트를 사용자 정의 컴포넌트로 취급하고, 소문자로 시작하는 요소는 내장 HTML 요소로 간주하기 때문입니다.

React가 `<div>`나 `<span>` 같은 소문자 태그를 만나면 표준 HTML 요소로 간주합니다. 하지만 컴포넌트 이름이 대문자로 시작하면 React는 이를 사용자 정의 컴포넌트로 인식해 적절히 렌더링합니다. 이 구분은 React가 기본 HTML 태그와 사용자가 만든 컴포넌트를 구분하는 데 도움을 줍니다.

이 `Greeting` 컴포넌트를 애플리케이션에서 사용하려면 다음과 같이 작성합니다:

```jsx
<Greeting />
```

이렇게 하면 페이지에 `h1` 요소가 `Hello John` 텍스트와 함께 렌더링됩니다. 여기 구문을 자세히 보면, 컴포넌트를 사용할 때 끝에 슬래시와 그 다음에 부등호가 옵니다.

JSX를 사용할 때 모든 태그와 컴포넌트 사용은 명시적으로 닫혀야 합니다. 따라서 컴포넌트나 태그에 자식 요소가 없으면 여기처럼 명시적으로 닫아야 합니다:

```jsx
<Greeting /> {/* /> is required */}
```

지금까지는 단일 `h1` 요소를 렌더링하는 방법만 살펴봤습니다. 하지만 실제로 여러 요소를 렌더링할 수 있습니다.

다음 예제 코드를 살펴보겠습니다:

```jsx
function Greeting() {
  const name = "John";
  {/* This will throw an error */}
  return <h1>Hello {name}</h1> 
  <p>Nice to meet you.</p>
}
```

`Nice to meet you` 문장을 하나 더 추가하려 했지만 페이지에 제대로 렌더링되지 않습니다.

대신 오류 메시지가 나타납니다. 오류 메시지는 `Adjacent JSX elements must be wrapped in an enclosing tag.`라고 합니다.

이 오류가 발생하는 이유는 여러 형제 요소를 부모 요소로 감싸야 하기 때문입니다. `h1`와 `p` 요소를 단순한 `div`로 감쌀 수도 있지만, 오류를 없애는 다른 방법도 있습니다.

React 프래그먼트는 요소들을 그룹화하는 데 사용됩니다. 수정된 예제는 다음과 같습니다:

```jsx
function Greeting() {
  const name = "John";
  return (
    <Fragment>
      <h1>Hello {name}</h1>
      <p>Nice to meet you.</p>
    </Fragment>
  );
}
```

또한 프래그먼트의 축약형으로 빈 JSX 태그를 사용할 수도 있습니다:

```jsx
function Greeting() {
  const name = "John";
  return (
    <>
      <h1>Hello {name}</h1>
      <p>Nice to meet you.</p>
    </>
  );
}
```

앞으로 수업에서 컴포넌트와 JSX를 다루는 방법을 계속 배울 것입니다. 지금은 컴포넌트로 사용자 인터페이스를 구축하는 기본을 탄탄히 다졌습니다.
