## 시작하기 전에

플렉스박스 레이아웃은 조금 복잡할 수 있습니다. 이전 수업에서 브라우저 개발자 도구를 사용해 검사하고 디버그하는 방법을 배웠습니다. 이 도구들은 이후 수업에서 매우 중요합니다. 예상과 다르게 동작하는 부분이 있다면, 항상 먼저 개발자 도구로 검사하는 것이 좋습니다.

플렉스박스가 지금까지 다룬 다른 개념들보다 꼭 더 어렵지는 않지만, 조금 더 많은 요소들이 움직입니다. 처음 수업에서 배우는 내용을 끝까지 모두 이해하고 나서야 제대로 활용할 수 있을 것입니다. 진행하면서 모든 코드 예제를 꼭 직접 실습해 보세요.

섹션 마지막 과제를 할 때 이 수업이나 제공하는 몇몇 자료를 다시 참고해야 할 가능성이 큽니다. 하지만 시간을 들여 모든 코드 예제를 실험해 보면, 그때 어디를 봐야 할지 훨씬 잘 알게 될 것입니다.

## 플렉스 시작하기

플렉스박스는 아이템들을 행이나 열로 배치하는 방법입니다. 이 아이템들은 간단한 규칙에 따라 늘어나거나 줄어들며 유연하게 배치됩니다. 먼저 간단한 데모를 살펴보겠습니다.

> 이 수업에는 많은 인터랙티브 예제가 포함되어 있습니다. 개념을 확실히 익히려면 천천히 실험해 보세요!

곧 이 코드가 어떻게 동작하는지 자세히 살펴보겠습니다. 우선 위 CodePen에서 `/*`와 `*/` 태그를 제거해 플렉스 관련 CSS 선언 두 줄의 주석을 해제하고 결과를 확인해 보세요.

> 주석은 브라우저가 코드를 해석하지 못하게 막으며, 특정 태그로 감싸져 있습니다. CSS는 `/*`가 주석 시작 태그, `*/`가 종료 태그이고, HTML과 JavaScript는 각각 다른 구문을 사용합니다. 주석 처리된 코드는 주석 태그를 제거하면 다시 활성화됩니다.

이제 3개의 div가 모두 가로로 배치되어야 합니다. `1x`, `.5x`, `.25x` 버튼으로 결과 프레임 크기를 조절하면 `div` 요소들이 "플렉스"되어 가용 영역을 채우고 각각 같은 너비를 가지는 것을 볼 수 있습니다.

HTML에 `div`를 `.flex-container` 안에 추가하면 다른 요소들과 함께 나타나고, 모두 가용 영역에 맞게 유연하게 배치됩니다.

> 작은 임베드된 CodePen에서 동작이 잘 보이지 않는다면 "Edit on CodePen" 또는 "Fork on CodePen" 버튼을 클릭해 전체 화면 환경에서 실험해 보세요. 이후 예제들은 특히 이렇게 하는 것이 도움이 될 수 있습니다.
