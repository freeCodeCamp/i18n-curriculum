# --description--

Los WebSockets permiten que un cliente y un servidor se comuniquen en tiempo real.

Normalmente, las aplicaciones del lado del cliente, como las apps web y móviles, usan HTTP. Con HTTP, el cliente envía una solicitud y el servidor responde. Esto se llama ciclo de solicitud y respuesta, como ya aprendiste.

Después de que se establece la conexión y se intercambian datos, la conexión se cierra. Esto funciona bien para la mayoría de las cosas, como cargar una página, obtener o enviar datos. Pero se vuelve lento e inestable para aplicaciones en tiempo real.

Por ejemplo, piensa en una app de chat. Si decides usar HTTP para ella, el cliente debe estar preguntando al servidor por los datos de vez en cuando, con la conexión abriéndose, llegando los datos y cerrándose después para cada mensaje.

Este es el problema que resuelven los WebSockets. Crean una conexión permanente entre el cliente y el servidor. Una vez que la conexión se abre, ambos lados pueden intercambiar datos en cualquier momento en un canal en tiempo real hasta que uno de ellos envíe una señal para cerrar la conexión.

Aunque los WebSockets son diferentes de HTTP, cada conexión WebSocket comienza con HTTP, en un proceso llamado handshake.

Luego el cliente solicita que la conexión se actualice a WebSocket. Cuando el servidor acepta, envía una `101 Switching Protocols` respuesta que actualiza la conexión a WebSocket. Después de eso, la conexión permanece abierta y comienza la comunicación en tiempo real.

El flujo es así:

1. El cliente se conecta al servidor vía HTTP  
2. El servidor acepta y actualiza la conexión a WebSocket  
3. La conexión permanece abierta  
4. Ambos lados intercambian datos en cualquier momento hasta que la conexión se cierra  

Este flujo es bidireccional, no de solicitud-respuesta como HTTP.

```md
Client <---> Server
```

Con HTTP, el flujo normalmente sigue un modelo de solicitud-respuesta:

```md
Client ---> Request
Server ---> Response
```

En cuanto a cómo se diferencian los WebSockets de HTTP, HTTP es como intercambiar cartas y WebSocket es como una llamada telefónica. Escribes una carta, esperas una respuesta y la conversación termina. Pero con una llamada telefónica, tú y el destinatario hablan en tiempo real.

Los WebSockets impulsan aplicaciones en tiempo real como apps de chat, juegos multijugador en línea, actualizaciones deportivas en vivo, paneles de mercado bursátil y herramientas colaborativas como Google Docs.
