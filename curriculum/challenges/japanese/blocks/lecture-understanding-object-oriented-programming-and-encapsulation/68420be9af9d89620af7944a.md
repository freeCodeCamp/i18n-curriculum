# --description--

オブジェクト指向プログラミング（OOP）は、開発者がコード内のすべてを現実世界のオブジェクトのように扱うプログラミングスタイルです。

クラスはオブジェクトを作成するための設計図のようなものです。クラスから作成されたすべてのオブジェクトは、データを定義する属性とオブジェクトの振る舞いを定義するメソッドを持っています。

前のレッスンでクラスの作り方を学びました。構文の復習はこちらです：

```py
class ClassName:
   def __init__(self, parameters):
       attribute = value

   def method_name(self):
       # method logic
```

次に、クラスを使ってオブジェクトが作成されるたびにブランドと色の属性を初期化する`__init__`特別メソッドを使ったクラスの例を示します：

```py
class Car:
   def __init__(self, brand, color):
       self.brand = brand
       self.color = color

# create two objects from the Car class
car1 = Car('Toyota', 'red')
car2 = Car('Lambo', 'green')

print('Car 1 Brand:', car1.brand) # Car 1 Brand: Toyota
print('Car 1 Color:', car1.color) # Car 1 Color: red

print('Car 2 Brand:', car2.brand) # Car 2 Brand: Lambo
print('Car 2 Color:', car2.color) # Car 2 Color: green
```

オブジェクト指向プログラミングには、コードを効果的に整理・管理するための4つの重要な原則があります。それはカプセル化、継承、ポリモーフィズム、抽象化です。

このレッスンの残りはカプセル化の仕組みに焦点を当てます。

カプセル化とは、オブジェクトの属性とメソッドをクラスという単一の単位にまとめることです。

カプセル化を使うと、オブジェクトの内部状態を単純な公開されたメソッドと属性のセットの背後に隠すことができます。これらはまるで扉のように機能します。その扉の向こうには、データの変更方法や誰がそれを見られるかを制御するプライベートな属性とメソッドがあります。

例えば、財布の残高を管理したいとします。人々が財布に入金や出金できるようにしたいけれど、誰も残高を直接いじれないようにしたい場合です。

その場合、`deposit()`と`withdraw()`を公開メソッドにして、残高は`_balance`属性の下に隠すことができます：

```py
class Wallet:
   def __init__(self, balance):
       self._balance = balance # For internal use by convention

   def deposit(self, amount):
       if amount > 0:
           self._balance += amount # Add to the balance safely

   def withdraw(self, amount):
       if 0 < amount <= self._balance:
           self._balance -= amount # Remove from the balance safely
```

慣例として、属性やメソッドの名前の前に単一のアンダースコアを付けるのは内部用であることを意味します。クラスの外から直接アクセスすべきではありません。これはカプセル化の原則に反し、バグの原因になる可能性があります。

単一のアンダースコアはあくまで慣例ですが、属性やメソッドの名前の前に二重アンダースコアを付けると、クラスの外部からアクセスできなくなり、それらをプライベートにします。

```py
class Wallet:
   def __init__(self, balance):
       self.__balance = balance # Private attribute

   def deposit(self, amount):
       if amount > 0:
           self.__balance += amount # Add to the balance safely

   def withdraw(self, amount):
       if 0 < amount <= self.__balance:
           self.__balance -= amount # Remove from the balance safely

account = Wallet(500)
print(account.__balance) # AttributeError: 'Wallet' object has no attribute '__balance'
```

`__balance`の現在の値を取得するには、get_balanceメソッドを定義できます。例えば：

```py
class Wallet:
   def __init__(self, balance):
       self.__balance = balance

   def deposit(self, amount):
       if amount > 0:
           self.__balance += amount

   def withdraw(self, amount):
       if 0 < amount <= self.__balance:
           self.__balance -= amount
  
   def get_balance(self):
       return self.__balance


acct_one = Wallet(100)
acct_one.deposit(50)
print(acct_one.get_balance()) # 150

acct_two = Wallet(450)
acct_two.withdraw(28)
print(acct_two.get_balance()) # 422

acct_two.deposit(150)
print(acct_two.get_balance()) # 572
```

また、入金や出金の金額が正の数であるかをチェックするプライベートな`__validate`メソッドを定義することもできます：

```py
class Wallet:
   def __init__(self):
       self.__balance = 0

   def __validate(self, amount):
       if amount < 0:
           raise ValueError('Amount must be positive')

   def deposit(self, amount):
       self.__validate(amount)
       self.__balance += amount

   def withdraw(self, amount):
       self.__validate(amount)
       if amount > self.__balance:
           raise ValueError('Insufficient funds')
       self.__balance -= amount

   def get_balance(self):
       return self.__balance

acct_one = Wallet()
acct_one.deposit(3)
print(acct_one.get_balance()) # 3

acct_one.deposit(50)
print(acct_one.get_balance()) # 53

acct_one.deposit(-4)  # ValueError: Amount must be positive
acct_one.withdraw(-8) # ValueError: Amount must be positive
acct_one.withdraw(58) # ValueError: Insufficient funds
```

ご覧の通り、`__validate`メソッドはプライベートで、`deposit()`と`withdraw()`の公開メソッドの裏で実行され、常に金額が有効であることを確認しています。

今後のレッスンで、二重アンダースコアで始まる属性がどのように機能するかをさらに学びます。

まとめると、カプセル化は内部データを明確な公開メソッドの背後にロックします。これによりクラスを改ざんから守り、バリデーションを一箇所に集中させられます。公開したインターフェースだけが外部コードに触れられるので、コードの更新や拡張も安心して行えます。
