# --description--

このレッスンでは、オブジェクトと配列の型を追加する方法を学びます。

まずは配列を見てみましょう。

こちらはプログラミング言語の配列の例です：

```js
const programmingLanguages = ["Java", "C++", "Python"];
```

もしその配列を文字列の配列として明示的に型付けしたい場合は、`string[]`のように型を指定できます：

```ts
const programmingLanguages:string[] = ["Java", "C++", "Python"];
```

前に学んだように、TypeScriptはこれだけ書いても文字列の配列として型を推論してくれます：

```js
const programmingLanguages = ["Java", "C++", "Python"];
```

しかし、明示的に型を指定しておくと、次のように配列に別の型を追加しようとした場合に役立ちます：

```ts
// Type 'number' is not assignable to type 'string'.(2322)
const programmingLanguages:string[] = ["Java", 9, "Python"];
```

配列の型付けにはもう一つの方法として、次の構文を使うこともできます：

```ts
const programmingLanguages:Array<string> = ["Java", "C++", "Python"];
```

基本的にこれら二つの構文は同じ意味で、どちらを使うかは好みの問題です。

次にオブジェクトの型付けを見てみましょう。

こちらはオブジェクトリテラルの例です：

```js
const developer = {
  firstName: "Jessica",
  isEmployed: true
}
```

オブジェクトのプロパティを明示的に型付けするには、次の構文を使います：

```ts
const developer: { firstName: string, isEmployed: boolean } = {
  firstName: "Jessica",
  isEmployed: true
}
```

オブジェクトにプロパティを追加したり削除したりしようとすると、次のようなエラーが表示されます：

```ts
// Property 'isEmployed' is missing in type '{ firstName: string; }' but required in type '{ firstName: string; isEmployed: boolean; }'
const developer: { firstName: string, isEmployed: boolean } = {
  firstName: "Jessica",
}
```

プロパティ名の後に`?`を付けることで、オブジェクトのプロパティをオプショナルにできます：

```ts
const developer: { firstName: string, isEmployed?: boolean } = {
  firstName: "Jessica",
}
```

`isEmployed`プロパティがオプショナルとしてマークされたため、TypeScriptはもうエラーを表示しません。

これまでのオブジェクトの例はすべて制限的なプロパティを持っていました。つまり、オブジェクトが持つことが許されるプロパティをすべて明示的に列挙していました：

```ts
const developer: { firstName: string, isEmployed: boolean } = {
  firstName: "Jessica",
  isEmployed: true
}
```

しかし、もし事前にすべてのプロパティ名がわからない場合はどうでしょう？任意の数のプロパティを持てるけれど、すべての値が特定の型に従うオブジェクトが欲しい場合は？

それを実現する方法は二つあります。最初の方法は、TypeScriptの組み込みジェネリックユーティリティ型である`Record`を使うことです。

基本的な構文はこちらです：

```ts
Record<Keys, ValueType>
```

最初の型はキーの型を表し、二つ目の型は値の型を表します。

こちらは`Record`を使った`userRoles`オブジェクトの例です：

```ts
const userRoles: Record<string, string> = {
  admin: "full-access",
  editor: "limited-access",
  viewer: "read-only"
};
```

この`userRoles`オブジェクトは任意の文字列キーを持てて、すべての値は文字列でなければなりません。文字列以外の値を代入しようとするとエラーになります：

```ts
// Type 'number' is not assignable to type 'string'
const userRoles: Record<string, string> = {
  admin: 1
};
```

同じ考え方を表す二つ目の方法はインデックスシグネチャを使うことです：

```ts
const obj: { [key: string]: string } = {};
```

上記の構文は、文字列キーを持つ任意のプロパティは文字列の値を持つ必要があることを示しています。こちらは`settings`オブジェクトの例です：

```ts
const settings: { [key: string]: string } = {
  theme: "dark",
  language: "en",
  layout: "grid"
};
```

先ほどと同様に、値の型が間違っているとTypeScriptはエラーメッセージを表示します：

```ts
// Type 'boolean' is not assignable to type 'string'
const settings: { [key: string]: string } = {
  darkMode: true
};
```

`Record`とインデックスシグネチャは基本的に同じものです。どちらを使うかは主に好みの問題です。
