# --interactive--

JavaScript에서 함수와 객체 메서드는 모두 재사용 가능한 코드를 캡슐화하는 방법이지만, 정의 방식, 사용법, 그리고 작동하는 문맥에서 몇 가지 중요한 차이가 있습니다. 이러한 차이를 이해하는 것은 효과적이고 체계적인 JavaScript 코드를 작성하는 데 매우 중요합니다.

이전 모듈에서 배운 것처럼, 함수는 특정 작업을 수행하는 재사용 가능한 코드 블록입니다:

:::interactive_editor

```js
function greet(name) {
    return "Hello, " + name + "!";
}
console.log(greet("Alice")); // "Hello, Alice!"
```

:::

반면 객체 메서드는 객체와 연관된 함수입니다. 객체의 프로퍼티로 정의되며, 객체의 데이터를 접근하고 조작할 수 있습니다. 다음은 메서드를 가진 객체의 예입니다:

:::interactive_editor

```js
const person = {
    name: "Bob",
    age: 30,
    sayHello: function() {
        return "Hello, my name is " + this.name;
    }
};

console.log(person.sayHello()); // "Hello, my name is Bob"
```

:::

이 예제에서 `sayHello`는 `person` 객체의 메서드입니다. `this` 키워드는 `sayHello` 메서드가 `person`라는 이름의 객체 프로퍼티에 접근할 수 있게 해줍니다. `this` 키워드에 대해서는 앞으로의 수업에서 더 배우게 됩니다.

함수와 메서드의 차이점 중 하나는 호출 방식입니다. 함수는 이름으로 호출하지만, 메서드는 속한 객체에서 점 표기법을 사용해 호출합니다. 예를 들어, `greet` 함수는 `greet("Alice")`로 호출하지만, `sayHello` 메서드는 `person.sayHello()`로 호출합니다.

또 다른 중요한 차이점은 작동하는 문맥입니다. 일반 함수는 자체 범위를 가지지만 특정 객체에 대한 내장 참조는 없습니다. 반면 메서드는 객체에 바인딩되어 `this` 키워드를 사용해 해당 객체의 프로퍼티와 다른 메서드에 접근할 수 있습니다.

중요한 점은 메서드는 코드를 논리적인 객체로 조직하는 데 도움을 주고, 함수는 보다 일반적이고 재사용 가능한 코드를 위해 사용된다는 것입니다.

함수와 객체 메서드의 차이를 이해하는 것은 JavaScript 학습 과정에서 매우 중요합니다. 처음에는 비슷해 보일 수 있지만, 언제 각각을 사용해야 하는지 알면 더 체계적이고 효율적이며 더 나은 코드를 작성할 수 있습니다.
