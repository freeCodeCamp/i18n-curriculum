# --description--

Sehen wir uns einige der grundlegenden Befehle an, mit denen Sie Daten in SQL einfügen und anzeigen können.

Zuerst müssen Sie, um Daten anzuzeigen, diese einfügen. Deshalb beginnen wir mit dem Einfügen.

Die Beispiele, die wir in dieser Lektion behandeln, basieren auf dieser Tabelle mit Hundedatensätzen:

```sql
CREATE TABLE dogs(
  id SERIAL,
  name VARCHAR(100),
  age INTEGER
);
```

Die Tabelle speichert die Namen und das Alter verschiedener Hunde.

Wenn wir annehmen, dass wir mit PostgreSQL arbeiten, können wir den `SERIAL`-Datentyp für die `id` verwenden. Da `id` mit `SERIAL` erstellt wurde, ist sein Wert ein `INTEGER`, der bei `1` beginnt und automatisch erhöht wird, wenn ein neuer Datensatz eingefügt wird. Sie müssen also keinen Wert dafür angeben, wenn Sie einen Datensatz einfügen.

Beginnen wir damit, Datensätze in unsere `dogs`-Tabelle mit der `INSERT INTO`-Anweisung einzufügen. Es gibt viele Möglichkeiten, dies zu tun. Die erste wäre, eine einzelne Zeile einzufügen und die Spalten der Werte anzugeben, die eingefügt werden.

Sie schreiben `INSERT INTO`, gefolgt vom Namen der Tabelle (in diesem Fall `dogs`), dann die Spalten in runden Klammern, dann `VALUES` und deren entsprechende Werte ebenfalls in runden Klammern.

Dies ist ein Beispiel: Wir fügen einen Hundedatensatz mit dem `name` `'Gino'` ein, dessen `age` `3` ist. Denken Sie daran, dass der `id` automatisch zugewiesen wird:

```sql
INSERT INTO dogs (name, age)
VALUES ('Gino', 3);
```

Dies ist normalerweise die sicherste Option, da Sie die Spalten explizit angeben, sodass die Werte in der Reihenfolge diesen Spalten zugewiesen werden.

Beachten Sie, dass `'Gino'` in einfache Anführungszeichen gesetzt werden muss, weil es sich um einen `VARCHAR`-Typ handelt.

Eine andere Möglichkeit ist, den Datensatz einzufügen, ohne die Spalten anzugeben, so:

```sql
INSERT INTO dogs
VALUES ('Gino', 3);
```

Diese Syntax ist gültiges SQL, aber fehleranfällig.

Da die Werte basierend auf der Spaltenreihenfolge zugewiesen werden, versucht PostgreSQL, `'Gino'` in die `id`-Spalte einzufügen, die einen Integer erwartet. Das führt zu einem Fehler wie `invalid input syntax for type integer`.

Aus diesem Grund ist es sicherer, beim Einfügen von Daten die Spaltennamen explizit anzugeben.

Sie können auch mehrere Datensätze im selben SQL-Befehl einfügen, indem Sie sie mit einem Komma trennen.

Hier fügen wir zwei Hundedatensätze ein, einen für `'Gino'` und einen weiteren für `'Nora'`.

```sql
INSERT INTO dogs (name, age)
VALUES
  ('Gino', 3),
  ('Nora', 2);
```

Und genauso wie Sie Datensätze einfügen können, können Sie Informationen aus der Datenbank abfragen. Im Kontext von Datenbanken ist eine Abfrage eine Anfrage nach Daten.

In SQL wird die `SELECT`-Anweisung verwendet, um Daten aus einer oder mehreren Tabellen abzufragen. Sie können die Abfrage anpassen, um genau die Informationen zu erhalten, die Sie benötigen.

Wenn Sie alle Informationen aus der `dogs`-Tabelle abfragen möchten, müssen Sie nur `SELECT` schreiben, gefolgt von einem Sternchen (`*`), dann `FROM` und dem Namen der Tabelle, `dogs`:

```sql
SELECT *
FROM dogs;
```

In diesem Befehl ist das Sternchen ein Platzhalterzeichen, das „alle Spalten“ darstellt.

Sie erhalten alle Daten aller Datensätze in der Tabelle:

```sql
 id | name | age 
----+------+-----
  1 | Gino |   3
  2 | Nora |   2
```

Um bestimmte Spalten abzufragen, können Sie die Namen dieser Spalten direkt nach `SELECT`, getrennt durch ein Komma, schreiben. In diesem Beispiel fragen wir die `name`- und `age`-Spalten der `dogs`-Tabelle ab:

```sql
SELECT name, age
FROM dogs;
```

Das Ergebnis enthält nur die `name`- und `age`-Spalten:

```sql
 name | age 
------+-----
 Gino |   3
 Nora |   2
```

Manchmal müssen Sie Daten basierend auf einer bestimmten Bedingung abfragen. Zum Beispiel können Sie, um alle Hunde zu erhalten, die jünger als 3 Jahre sind, das Schlüsselwort `WHERE` und den Vergleichsoperator „kleiner als“ (`<`) verwenden:

```sql
SELECT *
FROM dogs
WHERE age < 3;
```

Das Ergebnis enthält nur Hunde, deren Alter kleiner als 3 ist:

```sql
 id | name | age 
----+------+-----
  2 | Nora |   2
```

Wenn Sie nur den `age` von `'Gino'` finden möchten, können Sie den Vergleichsoperator „gleich“ (`=`) verwenden:

```sql
SELECT age
FROM dogs
WHERE name = 'Gino';
```

Und hier ist das Ergebnis:

```sql
 age 
-----
  3
```

Dies sind gängige und einfache Möglichkeiten, Daten in SQL einzufügen und anzuzeigen, aber es gibt viele verschiedene Optionen, aus denen Sie wählen können, und einige davon sind fortgeschrittener. Sie werden sie in kommenden Lektionen kennenlernen.
