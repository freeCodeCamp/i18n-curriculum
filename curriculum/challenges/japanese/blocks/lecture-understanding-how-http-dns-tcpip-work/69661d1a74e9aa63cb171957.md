# --description--

ウェブサイトを訪問するとき、最初にすることはブラウザにウェブアドレスを入力することです。

例えば、freeCodeCampに行くには、ブラウザのアドレスバーに`freecodecamp.org`と入力します。

これはドメイン名と呼ばれます。インターネット上のそのリソースを識別するための、人間が読みやすい一意の識別子です。

ドメイン名はDNSの概念と非常に密接に関連しています。DNSはこのレッスンの主なテーマです。まず、DNSが裏でどのように機能しているか理解できるように、ドメイン名の階層構造について少し説明しましょう。

ドメイン名の階層は右から左へ読み、最も一般的なカテゴリから最も具体的なものへと進みます。

ドメイン名の主な構造は右から左へ次の通りです：

* **トップレベルドメイン（TLD）：** ドメイン名の最後の部分で、右端にあります。`freecodecamp.org`の場合、それは`.org`です。他の例としては`.com`や`.edu`があります。
    
* **セカンドレベルドメイン（SLD）：** 個人や組織が登録した名前です。`freecodecamp`の中の`freecodecamp.org`にあたります。ウェブアドレスを見るときに通常最初に目にする部分です。
    
* **サードレベルドメイン（サブドメイン）：** メインドメインの特定のセクションを識別するために使われます。例えば、`www`は「ワールドワイドウェブ」を意味します。現在のモダンなウェブサーバーはドメインルーティングを自動で処理するため、`www`を入力しなくてもウェブサイトにアクセスできます。その他のサブドメインの例には`blog.`、`shop.`、`support.`、`api.`があります。
    

ドメイン名は人間にとって覚えやすく、ブラウザに入力しやすいため非常に便利です。しかし、ご存知の通り、コンピューターは数字で動作します。ブラウザは入力されたドメイン名をIPアドレスと呼ばれるものに変換して、目的のウェブサイトを見つけます。

**IPアドレス**は、ネットワーク上のコンピューターを識別するための一意の数字の並びです。インターネットも含まれます。

IPアドレスはクライアントが正しいサーバーにリクエストを送り、適切なレスポンスを受け取るために不可欠です。IPv4アドレスの例は`127.0.0.1`です。新しい規格であるIPv6の同等のIPアドレスは`0:0:0:0:0:0:0:1`です。

IPアドレスについては学ぶことがたくさんあり、学習を進める中でさらに深く探求したくなるかもしれません。ですが今は、ネットワーク上のコンピューターを識別するための一意の数字のアドレスであることだけ理解していれば十分です。

ドメイン名とIPアドレスの概念から、DNSの概念に進みましょう。

DNSは**Domain Name System**の略です。これは通常「インターネットの電話帳」と呼ばれます。

DNSはブラウザに入力されたドメイン名を対応するIPアドレスに変換します。インターネットや特定のネットワークに接続されたコンピューターやデバイスのための階層的で分散型の命名システムです。

ユーザーがブラウザに`freecodecamp.org`のようなドメイン名を入力すると、DNSはその名前を対応するサーバーのIPアドレスにマッピングします。サーバーが特定されると、クライアントのリクエストがサーバーに送られます。サーバーはプログラムされた通りにそのリクエストを処理し、最終的にレスポンスをクライアントに送ります。これでリクエスト-レスポンスのサイクルが完了します。

このすべては、最初にDNSがターゲットのIPアドレスへの変換を行ったおかげで可能になっています。

DNSの解決プロセスの詳細を見ていきましょう。

## DNS解決プロセス

* DNS解決はDNSサーバーの連鎖を通じてクエリを送信することで機能します。

* ユーザーがブラウザにドメイン名を入力すると、**クライアント**はローカルDNSサーバーにリクエストを送ります。このローカルDNSサーバーは**再帰的リゾルバー**と呼ばれます。通常はインターネットサービスプロバイダー（ISP）が管理しています。
    
* 再帰的リゾルバーはそのリクエストを**ルートサーバー**に転送します。ルートサーバーはIPアドレスの直接情報を持たない一般的なサーバーです。再帰的リゾルバーをより具体的なサーバー、つまりリクエストのトップレベルドメイン（TLD）サーバーに案内します。例えば、すべての`.org`ドメインを担当するサーバーです。
    
* 再帰的リゾルバーは次に**トップレベルドメイン（TLD）サーバー**にクエリを送ります。このサーバーはリクエストされたドメインのレコードを持つ特定の権威ネームサーバーのアドレスを返します（例えば`freecodecamp.org`）。
    
* 再帰的リゾルバーは対応する**権威ネームサーバー**に最終クエリを開始します。**DNSレコード**が見つかれば、それが再帰的リゾルバーに返されます。
    
* 再帰的リゾルバーがIPアドレスを受け取ると、それをクライアントに届けます。これでウェブサイトの閲覧を開始できます。

権威ネームサーバーはIPアドレスだけでなく**DNSレコード**を返します。DNSレコードはサブドメイン情報、情報のキャッシュ期間、リクエストされたサービスを実行するために必要な情報など、ドメインに関する重要な情報を含む構造化されたテキストエントリーです。

プロセスを最適化するために、再帰的リゾルバーはDNSレコードを一時的にキャッシュして処理を高速化します。これにより、短時間に同じリクエストを繰り返すことがなくなり、全体の処理が効率的になります。

ご覧の通り、DNSサーバーはドメイン名をIPアドレスに変換する最初の重要なステップを提供するため不可欠です。この変換プロセスにより、すべてのクライアントが正しいサーバーを見つけて接続できます。このディレクトリシステムがなければ、現代のウェブの基盤であるクライアント-サーバーモデルは機能しません。
