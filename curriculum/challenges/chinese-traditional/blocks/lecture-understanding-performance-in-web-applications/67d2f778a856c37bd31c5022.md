# --description--

為了衡量和提升網頁效率，我們使用關鍵效率指標，如 First Contentful Paint（或 FCP）、Speed Index、Total Blocking Time（或 TBT）、跳出率和獨立使用者。讓我們逐一檢視這些指標。

首次內容繪製，或稱 FCP。這衡量第一個內容片段（例如文字或影像）在螢幕上出現的速度。良好的 FCP 通常是在 1.8 秒以下，而不佳的 FCP 通常超過 3 秒。你可以使用 Chrome DevTools 這樣檢查你的 FCP：

步驟 1：開啟 Chrome DevTools

- 打開 Google Chrome。

- 前往你想測試的網頁。

- 按下「F12」或右鍵點擊並選擇「檢查」。

步驟 2：在 DevTools 中找到並點擊「Lighthouse」標籤。

步驟 3：點擊「Analyze page load」按鈕並等待分析完成。

現在我們可以分析結果。我們透過向下捲動到「Metrics」區段並尋找「First Contentful Paint」標記來完成此操作。我們可以看到以毫秒或秒為單位的精確時間。

FCP 和 Interaction to Next Paint（或 INP），我們之前提過，都是效率、性能指標，但它們衡量使用者體驗的不同面向。重要的是不要將它們混淆。

提醒你，FCP 是關於使用者在載入後多快看到內容，而 INP 是關於使用者在點擊或輸入後多快看到回授。

接下來是 Speed Index。

Speed Index 衡量頁面可見零件繪製的速度。分數越低代表效率越好！你可以在 Lighthouse 報告中檢查你的 Speed Index。

我們也有 Total Blocking Time 或 TBT。這顯示主執行緒被大量 JavaScript 任務阻塞了多久。如果 TBT 很高，使用者會感受到互動延遲。若要改善 TBT，請將長時間任務拆分並延緩非必要的腳本。

接下來是跳出率。這是指未進行互動即離開的訪客百分比。如果你的網站跳出率很高，可能是因為你的頁面太慢。

最後，我們有獨立使用者。此指標追蹤有多少獨立訪客造訪你的網站。若要查看跳出率和獨立使用者，你可以使用 Google Analytics。它將允許你監控這些指標並提升互動率。
