# --description--

Der letzte Teil der Einrichtung Ihrer GitHub-Authentifizierung besteht darin, die Strategie selbst zu erstellen. `passport-github@~1.1.0` wurde bereits als Abhängigkeit hinzugefügt, also fordern Sie sie in Ihrer `auth.js`-Datei als `GitHubStrategy` wie folgt an: `const GitHubStrategy = require('passport-github').Strategy;`. Vergessen Sie nicht, `dotenv` anzufordern und zu konfigurieren, damit Ihre Umgebungsvariablen verwendet werden.

Um die GitHub-Strategie einzurichten, müssen Sie Passport mitteilen, eine instanziierte `GitHubStrategy` zu verwenden, die 2 Argumente akzeptiert: ein Objekt (das `clientID`, `clientSecret` und `callbackURL` enthält) und eine Funktion, die aufgerufen wird, wenn ein Benutzer erfolgreich authentifiziert wurde. Diese Funktion bestimmt, ob der Benutzer neu ist und welche Felder zunächst im Benutzer-Datenbankobjekt gespeichert werden sollen. Dies ist bei vielen Strategien üblich, aber einige benötigen weitere Informationen, wie in der jeweiligen GitHub-README der Strategie beschrieben. Zum Beispiel erfordert Google auch einen *scope*, der bestimmt, welche Art von Informationen Ihre Anfrage zurückgeben soll, und bittet den Benutzer, diesen Zugriff zu genehmigen.

Die aktuelle Strategie, die Sie implementieren, authentifiziert Benutzer mit einem GitHub-Konto und OAuth-2.0-Tokens. Die Client-ID und das Secret, die bei der Erstellung einer Anwendung erhalten wurden, werden als Optionen beim Erstellen der Strategie übergeben. Die Strategie benötigt außerdem einen `verify`-Callback, der das Zugriffstoken und optional ein Refresh-Token erhält, sowie `profile`, das das GitHub-Profil des authentifizierten Benutzers enthält. Der `verify`-Callback muss `cb` aufrufen und dabei einen Benutzer übergeben, um die Authentifizierung abzuschließen.

So sollte Ihre neue Strategie zu diesem Zeitpunkt aussehen:

```js
passport.use(new GitHubStrategy({
  clientID: process.env.GITHUB_CLIENT_ID,
  clientSecret: process.env.GITHUB_CLIENT_SECRET,
  callbackURL: /*INSERT CALLBACK URL ENTERED INTO GITHUB HERE*/
},
  function(accessToken, refreshToken, profile, cb) {
    console.log(profile);
    //Database logic here with callback containing your user object
  }
));
```

Ihre Authentifizierung wird noch nicht erfolgreich sein, und sie wird tatsächlich einen Fehler werfen, wenn die Datenbanklogik und der Callback fehlen. Aber wenn Sie es versuchen, sollte Ihr GitHub-Profil in Ihrer Konsole protokolliert werden!

Senden Sie Ihre Seite ab, wenn Sie denken, dass alles richtig ist. Wenn Sie auf Fehler stoßen, können Sie <a href="https://forum.freecodecamp.org/t/advanced-node-and-express/567135#implementation-of-social-authentication-ii-4" target="_blank" rel="noopener noreferrer nofollow">das bis zu diesem Punkt abgeschlossene Projekt ansehen</a>.
