# --interactive--

Мітки на полях введення можуть здаватися дрібницею, але вони відіграють ключову роль у тому, щоб форми були зручнішими для всіх.

Мітки допомагають людям з інвалідністю, роблять форми більш дружніми для користувачів і врешті-решт покращують конверсію форм.

Давайте розберемося, чому додавання міток до елементів введення важливе для доступності та зручності використання.

Читачі екрана часто покладаються на мітки, щоб описати призначення полів введення. Щоб це працювало правильно, мітка має бути програмно пов’язана з `input`. Існує кілька способів зробити це, але найпоширеніший — використовувати HTML-елемент `label`.

Ось як пов’язати `label` з `input`:

:::interactive_editor

```html
<form>
   <label for="name">Your Name</label>
   <input type="text" id="name" />
</form>
```

:::

У цьому прикладі атрибут `for` елемента `label` пов’язаний з `id` елемента `input`. Це з’єднання дозволяє читачам екрана оголошувати `label`, коли `input` перебуває у стані фокусу, що дає змогу користувачам читачів екрана зрозуміти призначення `input`.

Пов’язування міток з усіма полями введення також корисне для SEO. Завдяки правильним міткам пошукові системи краще розумітимуть вміст сторінки, що позитивно впливає на рейтинг сторінки у пошуку.

Але важливо розуміти, що ви створюєте вміст для людей, а не для пошукових систем. Тож слід переконатися, що мітки максимально чіткі та описові.
