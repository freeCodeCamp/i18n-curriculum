# --interactive--

Das `aria-expanded`-Attribut wird für Barrierefreiheitszwecke verwendet, um anzuzeigen, ob ein Steuerelement erweitert oder eingeklappt ist. Es wird in Verbindung mit einklappbaren Widgets wie Menüs, Akkordeons und anderen Offenlegungs-Widgets verwendet, die die Sichtbarkeit von Inhalten steuern. Das `aria-expanded`-Attribut wird auf `true` gesetzt, wenn das Steuerelement erweitert ist, oder auf `false`, wenn es eingeklappt ist.

Die durch `aria-expanded` bereitgestellten Informationen ermöglichen es Menschen, die Screenreader verwenden, den aktuellen Zustand des Steuerelements zu verstehen (ob es erweitert oder eingeklappt ist).

Das `aria-expanded`-Attribut wird auf das interaktive Element angewendet, das die Sichtbarkeit eines einklappbaren Widgets umschaltet. Zum Beispiel, wenn ein Button ein erweiterbares Menü umschaltet, wird das `aria-expanded`-Attribut auf den Button gesetzt.

Wenn das Menü erweitert ist, sollte das `aria-expanded`-Attribut auf `true` gesetzt werden, wie in diesem Beispiel:

```html
<button aria-expanded="true">Menu</button>
```

Wenn das Menü eingeklappt ist, sollte es stattdessen auf `false` gesetzt werden.

```html
<button aria-expanded="false">Menu</button>
```

Sie müssen immer das `aria-expanded`-Attribut entweder auf `true` oder `false` am steuernden Element gesetzt haben. Zum Beispiel, wenn ein Button die Sichtbarkeit eines Menüs umschaltet, sollte der Standardwert für `aria-expanded` basierend auf der standardmäßigen Sichtbarkeit des Menüs sein.

Wenn das Menü standardmäßig erweitert ist, sollte `aria-expanded` zunächst auf `true` gesetzt werden. Wenn das Menü standardmäßig eingeklappt ist, sollte `aria-expanded` zunächst auf `false` gesetzt werden.

Der Wert von `aria-expanded` sollte dynamisch mit JavaScript aktualisiert werden, während der Benutzer mit dem Element interagiert.

Zusätzlich können die Eigenschaften `aria-controls` und `aria-owns` in Kombination mit `aria-expanded` verwendet werden, um eine programmatische Verbindung zwischen dem steuernden Element und dem Element, das es steuert, herzustellen.

Beginnen wir mit `aria-controls`. Wenn es zusammen mit `aria-expanded` verwendet wird, wird das `aria-controls`-Attribut verwendet, um das gesteuerte Element anzugeben. Zum Beispiel könnte ein Button eine Liste erweitern oder einklappen, die als Menü fungiert. Der Wert von `aria-controls` wird die `id` des gesteuerten Elements sein (in diesem Beispiel die Menüliste).

Beachten Sie, dass die Liste unmittelbar dem steuernden Button folgt. Für erweiterbare Steuerelemente wie dieses ist es am besten, den erweiterten Inhalt unmittelbar nach dem Element zu platzieren, das ihn im DOM steuert. Dies verhindert, dass Screenreader-Nutzende den erweiterten Inhalt suchen müssen, und erleichtert Tastaturnutzenden die Navigation durch alle interaktiven Steuerelemente im erweiterten Inhalt.

Wenn es nicht möglich ist, den erweiterten Inhalt unmittelbar nach dem steuernden Element zu platzieren, erlaubt das `aria-owns`-Attribut, ihn virtuell nach dem Steuerelement im Barrierefreiheitsbaum zu verschieben. Dies ermöglicht Hilfstechnologien wie Screenreadern, so zu tun, als ob der erweiterte Inhalt direkt nach dem Steuerelement im DOM platziert wäre.

const button = document.querySelector('button');
const menu = document.getElementById(button.getAttribute('aria-owns'));

button.addEventListener('click', () => {
  const expanded = button.getAttribute('aria-expanded') === 'true';
  button.setAttribute('aria-expanded', String(!expanded));
  menu.hidden = expanded;
});

// Close the menu if user clicks outside
document.addEventListener('click', (e) => {
  if (!button.contains(e.target) && !menu.contains(e.target)) {
    button.setAttribute('aria-expanded', 'false');
    menu.hidden = true;
  }
});

Die Verwendung des `aria-owns`-Attributs hat Nachteile. Es erzeugt unnötige Wortfülle für Screenreader-Nutzende, da die meisten Screenreader automatisch den gesamten Inhalt des erweiterten Elements vorlesen, wenn es erstmals erweitert wird. Außerdem ändert es nicht die Tabulatorreihenfolge, was Tastaturnutzende möglicherweise dazu zwingt, durch alle anderen interaktiven Steuerelemente auf der Seite zu tabben, bevor sie den erweiterten Inhalt erreichen, es sei denn, Sie verwalten die Tabulatorreihenfolge mit JavaScript.

Idealerweise sollte der erweiterbare Inhalt nach dem Steuerelement platziert werden, und das `aria-owns`-Attribut sollte nur im schlimmsten Fall verwendet werden, wenn dies nicht möglich ist. Wenn es verwendet werden muss, sollten Sie gründlich mit einer breiten Palette von Screenreadern und Browsern testen, um sicherzustellen, dass Ihre Implementierung für alle zugänglich ist.

Und noch eine Erinnerung: Wenn Sie entweder `aria-controls` oder `aria-owns` verwenden, muss der Wert von `aria-expanded` weiterhin aktualisiert werden, während das Steuerelement erweitert und eingeklappt wird.

Das `aria-expanded`-Attribut zeigt an, ob ein Steuerelement erweitert oder eingeklappt ist. Diese Information ist für Screenreader-Nutzende essenziell, da sie ihnen hilft, den aktuellen Zustand einklappbarer Elemente wie Menüs, Akkordeons und ähnlicher Offenlegungs-Widgets zu verstehen.

Durch die korrekte Verwendung von `aria-expanded` können Sie eine intuitive Benutzererfahrung für alle schaffen.
