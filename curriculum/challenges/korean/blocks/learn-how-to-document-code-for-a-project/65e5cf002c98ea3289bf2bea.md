# --description--

`y`로 끝나는 명사가 복수형이 될 때, `y`이 종종 `ies.`로 바뀝니다. 이 규칙은 특정 용어가 `y.`로 끝나는 기술 분야에서 특히 유용합니다.

# --questions--

## --text--

`y`로 끝나는 명사의 복수형은 어떻게 만드나요?

## --answers--

`y`를 `ys`로 바꿔서 만듭니다.

### --feedback--

이것은 일반적인 규칙이 아닙니다. 보통은 `y`가 복수형에서 `ies`로 바뀌는데, 예를 들어 `query`가 `queries.`로 변합니다.

---

`s`를 추가해서 만듭니다.

### --feedback--

`y`로 끝나는 대부분 명사는 단순히 `s`를 추가하는 것이 올바르지 않습니다. 예를 들어 `directory`는 `directories`가 되며, `directorys.`가 아닙니다.

---

`y`를 `ies`로 바꿔서 만듭니다.

---

`y`를 제거해서 만듭니다.

### --feedback--

`y`를 제거하는 것은 복수형을 만드는 올바른 방법이 아닙니다. 보통 복수형에서는 `y`가 `ies`로 바뀌는데, 예를 들어 `binary`가 `binaries.`로 변합니다.
