# --interactive--

الخصائص والطرق الثابتة تنتمي إلى الصنف نفسه، وليس إلى كل نسخة فردية من الصنف. يمكنك الوصول إليها مباشرة عبر اسم الصنف دون إنشاء نسخة من الصنف. تُعرّف داخل الأصناف لتغليف الوظائف ذات الصلة.

يمكنك تعريف طريقة ثابتة بكتابة الكلمة المفتاحية `static` قبل اسم الطريقة.

```js
class MyClass {
  static staticMethod() { ... }
}
```

بعدها، يمكنك استدعاء الطريقة الثابتة مباشرة على الصنف، باستخدام ترميز النقطة وتمرير أي معلمات ضرورية:

```js
MyClass.staticMethod();
```

لاحظ أنك تستطيع استدعاء الطريقة دون إنشاء نسخة من الصنف. هذه إحدى الخصائص الرئيسية للطرق الثابتة.

إليك مثالًا. لنفترض أننا ننشئ تطبيق أفلام ونريد أن نتمكن من مقارنة الأفلام بناءً على تقييمها.

يمكننا اعتبار طريقة المقارنة هذه طريقة على مستوى أعلى ليست خاصة بأي فيلم معين:

```js
if (movieA.rating < movieB.rating) {
  console.log(`${movieB.title} has a higher rating.`);
} 
```

إنها مثل طريقة أكثر عمومية مرتبطة بـ `Movie` الصنف.

لأغراض القراءة والصيانة، سيكون من المفيد تعريفها داخل صنف `Movie` للحفاظ على قرب جميع الطرق ذات الصلة من بعضها البعض.

لذلك، هذه طريقة مثالية لتكون طريقة ثابتة. يمكنك رؤيتها هنا، أسفل دالة البناء مباشرة:

:::interactive_editor

```js
class Movie {

  constructor(title, rating) {
    this.title = title;
    this.rating = rating;
  }

  static compareMovies(movieA, movieB) {
    if (movieA.rating > movieB.rating) {
      console.log(`${movieA.title} has a higher rating.`);
    } else if (movieA.rating < movieB.rating) {
      console.log(`${movieB.title} has a higher rating.`);
    } else {
      console.log("These movies have the same rating.");
    }
  }

}

let movieA = new Movie("Movie A", 80);
let movieB = new Movie("Movie B", 45);
console.log(movieA);
```

:::

تعرف الطريقة الثابتة باستخدام الكلمة المفتاحية `static` وتسمى `compareMovies`. لديها معلمتان: `movieA` و `movieB`. ستكون هذه نسخًا من صنف `Movie`.

سنقارن بينهما بناءً على تقييمهما، ضمن نطاق من `0` إلى `100`. تُنفذ هذه المنطقية باستخدام شرط وستطبع رسالة مناسبة بناءً على أي فيلم لديه تقييم أعلى.

بمجرد تعريف الطريقة، يمكنك استدعاؤها على الصنف. لكن أولًا، تحتاج إلى تحضير المعلمات وتعريفها في برنامجك.

في هذه الحالة، تأخذ الطريقة نسختين من الأفلام كمعلمات، لذا نعرّف هذه النسخ هنا:

```js
let movieA = new Movie("Movie A", 80);
let movieB = new Movie("Movie B", 45);
```

يمكنك أن ترى أن `movieA` لديه تقييم أعلى من `movieB`. لنرَ ناتج هذه الطريقة.

لاستدعاء الطريقة، تحتاج فقط إلى استخدام ترميز النقطة على الصنف نفسه.
تكتب اسم الصنف (`Movie`)، متبوعًا بنقطة، ثم اسم الطريقة الثابتة (`compareMovies`).

بعدها، تمرر المعلمات داخل الأقواس العادية. في هذه الحالة، هما نسختا الفيلم اللتان تحتاجهما الطريقة.

```js
Movie.compareMovies(movieA, movieB);
```

إليك المثال المحدث:

:::interactive_editor

```js
class Movie {

  constructor(title, rating) {
    this.title = title;
    this.rating = rating;
  }

  static compareMovies(movieA, movieB) {
    if (movieA.rating > movieB.rating) {
      console.log(`${movieA.title} has a higher rating.`);
    } else if (movieA.rating < movieB.rating) {
      console.log(`${movieB.title} has a higher rating.`);
    } else {
      console.log("These movies have the same rating.");
    }
  }

}

let movieA = new Movie("Movie A", 80);
let movieB = new Movie("Movie B", 45);
Movie.compareMovies(movieA, movieB);
console.log(movieA);
```

:::

الطرق الثابتة مفيدة أيضًا لتنفيذ طرق "المصنع". طريقة المصنع هي طريقة تعرفها بالإضافة إلى دالة البناء لإنشاء كائنات بناءً على معايير محددة.

إليك مثالًا مع صنف `Pizza`. الطريقة الثابتة `createMargherita` هي طريقة مصنع يمكنك استدعاؤها لإنشاء نسخة بيتزا مارغريتا مع نوعها وسعرها محددين مسبقًا.

```js
class Pizza {
  constructor(type, price) {
    this.type = type;
    this.price = price;
  }

  static createMargherita() {
    return new this("Margherita", 6.99);
  }
}
```

هذا يثير نقطة مهمة جدًا حول الطرق الثابتة.

قيمة الكلمة المفتاحية `this` في الطرق الثابتة هي الصنف نفسه، لأن الطريقة الثابتة تنتمي إلى الصنف.

لهذا يمكننا استخدام `this` لإنشاء نسخة جديدة من صنف `Pizza`.

إذا استدعيت هذه الطريقة على صنف `Pizza` نفسه وعينت النسخة المرجعة إلى متغير، كما في هذا المثال:

```js
let myPizza = Pizza.createMargherita();
```

يمكنك استخدامها في أي مكان تحتاجه في شفرتك. على سبيل المثال، يمكنك طباعتها في وحدة التحكم:

```js
console.log(myPizza);
```

هذا هو الناتج:

```js
Pizza { type: 'Margherita', price: 6.99 }
```

يمكنك أيضًا استخدام ترميز النقطة لاستدعاء طرقها والوصول إلى خصائصها، مثل هذا:

```js
console.log(myPizza.type);
```

إليك المثال الكامل:

```js
class Pizza {
  constructor(type, price) {
    this.type = type;
    this.price = price;
  }

  static createMargherita() {
    return new this("Margherita", 6.99);
  }
}

let myPizza = Pizza.createMargherita();
console.log(myPizza);
console.log(myPizza.type);
```

بالإضافة إلى الطرق، يمكنك أيضًا تعريف خصائص ثابتة باستخدام الكلمة المفتاحية `static`.

في هذا المثال، لدينا خاصية ثابتة `numberOfPizzasSold`.

```js
class Pizza {
  static numberOfPizzasSold = 0;

  constructor(type) {
    this.type = type;
    Pizza.numberOfPizzasSold++;
  }
}
```

هي ثابتة لأنها لا تنتمي إلى أي نسخة بيتزا معينة، بل تنتمي إلى الصنف نفسه.

لها قيمة ابتدائية `0` ويتم تحديثها في كل مرة تُنشأ نسخة جديدة.

إذا أنشأت نسختين من البيتزا، ستُحدث القيمة مرتين:

```js
let pizza1 = new Pizza("Margherita");
let pizza2 = new Pizza("Neapolitan");
```

للوصول إلى قيمة خاصية ثابتة، تحتاج فقط إلى استخدام ترميز النقطة على الصنف نفسه، لأن الخاصية تنتمي إلى الصنف.

```js
class Pizza {
  static numberOfPizzasSold = 0;

  constructor(type) {
    this.type = type;
    Pizza.numberOfPizzasSold++;
  }
}

let pizza1 = new Pizza("Margherita");
let pizza2 = new Pizza("Neapolitan");

console.log(Pizza.numberOfPizzasSold);
```

في هذه الحالة، الناتج هو `2` لأن بيتزا تم بيعها مرتين.

هذه هي أساسيات الخصائص والطرق الثابتة في JavaScript. فهم الأعضاء الثابتة ضروري لإنشاء أصناف قابلة لإعادة الاستخدام وفعالة.
