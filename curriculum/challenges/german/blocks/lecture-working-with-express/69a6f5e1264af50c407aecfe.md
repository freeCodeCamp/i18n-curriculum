# --description--

In der vorherigen Lektion haben Sie einen Projektordner erstellt und Express installiert:

```bash
mkdir my-app
cd my-app
npm init -y
npm install express
```

Zur Erinnerung: Diese Befehle erstellen einen neuen Projektordner, `cd` in den Projektordner, initialisieren das Projekt und erstellen eine grundlegende `package.json`-Datei sowie installieren Express.

Öffnen Sie als Nächstes einen Code-Editor, erstellen Sie eine Datei namens `index.js` und schreiben Sie Ihren ersten Express-Server:

```javascript
const express = require("express");
const app = express();
const port = 3000;

app.get("/", (req, res) => {
  res.send("Hello World!");
});

app.listen(port, () => {
  console.log(`Example app listening on port ${port}`);
});
```

Sie können Ihren Code mit folgendem Befehl im Terminal ausführen:

```bash
node index.js
```

Öffnen Sie Ihren Browser unter [http://localhost:3000](http://localhost:3000) und Sie sehen die Nachricht **"Hello World!"**

So schnell können Sie mit Express loslegen! Gehen wir nun den Code Schritt für Schritt durch.

In der ersten Codezeile laden Sie die Express-Bibliothek und weisen sie einer Variablen namens `express` zu:

```javascript
const express = require("express");
```

Die `express`-Variable ist eine Factory-Funktion. Wenn Sie sie aufrufen, erstellt und gibt sie eine neue Express-Anwendungsinstanz zurück:

```javascript
const app = express();
```

`app` hat nun Zugriff auf alle von Express bereitgestellten Methoden, wie `.get()`, `.post()`, `.listen()` und so weiter. Dies ist Ihr Hauptserverobjekt.

Die nächste Zeile definiert die `port`-Variable. Ein Port ist ein Netzwerkendpunkt, auf dem Ihre Anwendung lauscht, und `3000` ist ein häufig verwendeter Port für die lokale Entwicklung:

```javascript
const port = 3000;
```

Sehen wir uns nun den nächsten Teil an:

```javascript
app.get("/", (req, res) => {
  res.send("Hello World!");
});
```

Dieser Codeblock definiert eine Route und sagt dem Server, wie er auf Anfragen reagieren soll. `app.get()` registriert einen Handler für HTTP-GET-Anfragen. Eine GET-Anfrage ist eine häufig verwendete HTTP-Methode zum Abrufen von Daten.

Das `"/"`-Argument innerhalb von `app.get()` stellt den Routenpfad dar. In diesem Fall zeigt es auf die Stamm-URL der Anwendung.

Der `(req, res) => { ... }`-Teil ist eine Callback-Funktion, die ausgeführt wird, wenn eine Anfrage mit dieser Route übereinstimmt. Das `req`-Objekt enthält Informationen über die eingehende Anfrage, und das `res`-Objekt wird verwendet, um eine Antwort an den Client zu senden.

Schließlich sendet `res.send("Hello World!")` eine Antwort an den Client. In diesem Beispiel ist die Antwort der Klartext `"Hello World!"`.

Sehen wir uns nun den letzten Codeabschnitt an:

```javascript
app.listen(port, () => {
  console.log(`Example app listening on port ${port}`);
});
```

Die `app.listen()`-Methode startet die Express-Anwendung und sagt Node, dass es auf eingehende HTTP-Anfragen am angegebenen Port, in diesem Fall `3000`, lauschen soll.

Sobald der Server gestartet ist, wird die Callback-Funktion ausgeführt und die Nachricht `Example app listening on port 3000` wird in der Konsole protokolliert, was bestätigt, dass die Express-App erfolgreich gestartet wurde und bereit ist, Anfragen zu empfangen.
