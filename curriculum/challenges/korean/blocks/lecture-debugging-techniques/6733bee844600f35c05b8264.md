# --interactive--

이전 수업에서 프로그램에서 예외를 발생시키는 방법을 배웠습니다. 이번 수업에서는 `try…catch…finally` 블록에서 이러한 오류를 우아하게 처리하는 방법을 살펴보겠습니다.

`try` 블록은 오류가 발생할 수 있는 코드를 감싸는 데 사용됩니다. 실패할 수 있는 작업을 시도하는 안전한 공간 역할을 합니다.

`catch` 블록은 `try` 블록에서 발생한 오류를 포착하고 처리합니다. `Error` 안에서 `catch` 객체를 사용해 무엇이 잘못되었는지 확인할 수 있습니다.

`finally` 블록은 오류 발생 여부와 상관없이 `try` 및 `catch` 블록 이후에 실행됩니다. 보통 파일 닫기나 자원 해제 같은 정리 작업에 사용됩니다.

다음은 `try…catch` 블록을 사용하는 예시입니다:

:::interactive_editor

```js
function processInput(input) {
  if (typeof input !== "string") {
    throw new TypeError("Input must be a string.");
  }

  return input.toUpperCase();
}

try {
  console.log("Starting to process input...");
  const result = processInput(9);
  console.log("Processed result:", result);
} catch (error) {
  console.error("Error occurred:", error.message);
} 
```

:::

이 예시에서 `processInput`라는 함수가 있습니다. 이 함수는 먼저 `input`이 `string` 타입이 아닌지 확인합니다. 그렇다면 오류를 발생시킵니다. 그렇지 않으면 `toUpperCase`에서 `input` 메서드를 사용한 결과를 반환합니다.

함수는 `try` 블록 안에서 호출됩니다. 함수 호출이 오류를 발생시키므로 `catch` 블록에서 잡히고 오류 메시지가 콘솔에 표시됩니다.

`catch` 블록에 전달된 오류는 그 오류에 관한 정보를 담은 `Error` 객체입니다. 이 경우 사용자에게 사람이 읽을 수 있는 정보를 보여주는 `message` 속성을 사용하고 있습니다.

오류를 기록하도록 특별히 설계된 `console.error`을 사용하고 있습니다. 최신 브라우저에서는 `console.error()` 출력이 콘솔에서 빨간색으로 표시됩니다.

`finally` 문장은 예외가 발생했든 안 했든 항상 실행됩니다.

```js
try {
  // Code that might throw an error
} catch (error) {
  // Code to handle the error
} finally {
  // Code that runs regardless of whether an error occurred or not
}
```

`finally` 문장의 좋은 사용 사례는 파일 작업입니다. JavaScript에서 파일을 열고 `try` 블록을 사용해 파일에 데이터를 씁니다. 오류가 있으면 `catch`를 사용해 잡을 수 있습니다. 그리고 `finally` 문장으로 파일을 닫습니다.
