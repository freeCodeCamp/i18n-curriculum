# --description--

事件處理是每個互動式網站的重要零件。

React 提供了一種強大且一致的方式來控制代碼事件，透過其合成事件系統，該系統是對你在先前課程中學到的 `click`、`keydown` 和 `submit` 等原生事件的外覆。

這個跨瀏覽器外覆確保 `events` 在所有瀏覽器中運作一致，避免不一致的情況。

讓我們來看看 `event` 在 React 中的運作方式，這樣你就可以開始在你的專案中使用它們。

在 React 中，事件處理常式的運作方式與原生的瀏覽器事件類似，但有一些調整。

React 使用駝峰式大小寫，例如 `onClick` 和 `onSubmit`，而不是使用小寫的事件屬性名稱如 `onclick` 和 `onsubmit`。

此外，React 預期事件處理常式使用函式，而非使用字串來指定事件種類。

事件處理常式函式作為 prop 傳遞給元素，而像 `onClick` 或 `onSubmit` 這樣的事件型別則在 JSX 中用作屬性。

這裡是如何在一般 HTML 中處理 click 事件的提醒：

```html
<button onclick="alert('Button clicked!')">Click Me</button>
```

以下是在 React 中執行相同操作的方法：

```jsx
function handleClick() {
  console.log("Button clicked!");
}

<button onClick={handleClick}>Click Me</button>;
```

在此範例中，當使用者點擊按鈕時，`handleClick` 會將訊息記錄到主控台。

請注意，在 `onClick` 屬性中你不需要在 `handleClick` 後面加上小括弧，因為你是在傳遞函式的引用，而不是呼叫它。

在 React 中，事件處理常式函式通常以前置式 `handle` 開頭，以顯示它們負責控制事件，例如 `handleClick` 或 `handleSubmit`。

當使用者操作觸發一個事件時，React 會將一個 Synthetic Event 物件傳遞給你的處理常式。這個物件的行為非常類似於原生的 JavaScript 事件物件，提供像是 `type`、`target` 和 `currentTarget` 等屬性。

你可以將 `event` 作為參數傳遞給處理常式函式，並將它記錄到主控台以查看其結構：

```js
function handleClick(event) {
  console.log(event);
}
```

為了防止像瀏覽器重新整理這類的預設行為，例如在 `onSubmit` 事件期間，你可以呼叫 `preventDefault()` 方法：

```jsx
function handleSubmit(event) {
  event.preventDefault();
  console.log("Form submitted!");
}

<form onSubmit={handleSubmit}>
  <input type="text" />
  <button>Submit</button>
</form>;
```

你也可以透過呼叫 `event.stopPropagation()` 來阻止事件向父元素冒泡。

有時候，在控制刪除和編輯等特殊功能時，你可能想要將額外的資料傳遞給事件處理常式。你可以透過將處理常式包裝在行內箭頭函式中來做到這一點：

```jsx
function handleDelete(id) {
  console.log("Deleting item:", id);
}

<button onClick={() => handleDelete(1)}>Delete Item</button>;
```

在 React 中使用行內事件處理常式是可以的，因為 React 高效地管理重新呈現並避免效率問題。 

在純 JavaScript 中，行內事件處理常式會在每次呈現時建立新的函式，因為沒有虛擬 DOM 來最佳化處理程序，這可能導致效率問題。
