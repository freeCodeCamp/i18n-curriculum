# --interactive--

级联算法是浏览器在多个样式针对同一元素时决定应用哪些 CSS 规则的进程。它确保基于一组明确定义的规则使用最合适的样式。

该进程从**相关性**开始。浏览器首先过滤所有 CSS 规则，以查找那些实际应用于相关元素的规则。这包括匹配选择器和考虑可能生效的媒体查询。

媒体查询是一种 CSS 技术，用于根据设备或视图的特性（例如其宽度、高度或方向）应用样式。

接下来，算法会考虑**来源和重要性**。CSS 可以来自不同的来源：浏览器的默认样式（user-agent）、用户设置的样式以及作者（你）编写的样式。

在考虑来源之后，算法会评估每条规则的重要性，优先处理标记为 `!important` 的规则，这些规则无论来源如何都将覆写其他规则。

在按来源和重要性过滤后，算法会查看**特异性**。当来自相同来源和重要性级别的两个规则都适用时，将应用特异性更高的规则。

特异性是衡量选择器针对性的指标，更具体的选择器优先于更通用的选择器。

最后，如果其他条件都相等，**出现顺序** 就会起作用。当两个规则具有相同的特异性时，CSS 中最后出现的规则将被应用。

这就是为什么你编写样式的顺序有时会影响结果的原因。

让我们来看一个例子。

在 `HTML` 文件中，有一个单独的段落元素。然后在 `CSS` 中，我们有两个规则，每个规则都针对该段落元素。

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<p>example paragraph</p>
```

```css
p {
  color: blue;
}

p {
  color: green; 
}
```

:::

第一条规则将所有段落元素的文本颜色设置为蓝色，而第二条规则将所有段落元素的文本颜色设置为绿色。

那么将应用哪种颜色？绿色将应用于段落元素。

通过考虑相关性、来源和重要性、特异性、作用域以及出现顺序，Cascade 算法确保你的 CSS 表现得可预测，从而让你设计出更复杂且细致的网页。
