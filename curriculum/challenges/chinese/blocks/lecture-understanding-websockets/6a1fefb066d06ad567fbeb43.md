# --description--

WebSockets 让客户和服务器能够实时通信。

通常，客户端应用程序如网页和移动应用使用超文本传输协议。使用超文本传输协议时，客户发送请求，服务器返回响应。这就是你所学过的请求和响应周期。

连接建立并交换数据后，连接会关闭。这对于大多数情况都很有效，比如加载页面、获取或提交数据。但对于实时应用来说，这种方式变得缓慢且不可靠。

举个例子，想象一个聊天应用。如果你决定用超文本传输协议来实现，客户必须不断地向服务器请求数据，每条消息都要打开连接、接收数据，然后关闭连接。

这正是 WebSockets 解决的问题。它们在客户和服务器之间创建一个永久连接。连接一旦打开，双方可以随时通过实时通道交换数据，直到任一方发送关闭信号。

虽然 WebSockets 与超文本传输协议不同，但每个 WebSocket 连接都是从超文本传输协议开始的，这个过程称为握手。

然后客户请求将连接升级为 WebSocket。一旦服务器同意，它会发送一个 `101 Switching Protocols` 响应，将连接升级为 WebSocket。之后，连接保持打开，实时通信开始。

流程如下：

1. 客户通过超文本传输协议连接服务器
2. 服务器接受并将连接升级为 WebSocket
3. 连接保持打开
4. 双方随时交换数据，直到连接关闭

这个流程是双向的，而不是像超文本传输协议那样的请求-响应模式。

```md
Client <---> Server
```

使用超文本传输协议时，流程通常遵循请求-响应模型：

```md
Client ---> Request
Server ---> Response
```

关于 WebSockets 与超文本传输协议的区别，超文本传输协议就像交换信件，你写信、等待回复，交流结束。但 WebSocket 就像打电话，你和对方实时对话。

WebSockets 支持实时应用，如聊天应用、在线多人游戏、体育赛事直播、股票市场仪表盘和协作工具如 Google Docs。
