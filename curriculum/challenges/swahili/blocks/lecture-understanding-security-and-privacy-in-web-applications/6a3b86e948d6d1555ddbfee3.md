# --description--

Sera ya Asili Imoja ni hatua ya usalama iliyojengwa ndani ya vivinjari vyako vya mtandao inayozuia tovuti moja kusoma data binafsi kutoka kwa tovuti nyingine. Sheria hii inalinda akaunti zako binafsi dhidi ya wadukuzi.

Asili ina sehemu tatu maalum: itifaki ya mtandao, jina la kikoa, na nambari ya port.

Kwa mfano, katika anwani `https://example.com:443`, itifaki ni HTTPS, jina la kikoa ni `example.com`, na nambari ya port ni `443`.

Vivinjari mara nyingi hujificha nambari ya port ikiwa ni ile ya kawaida. HTTPS hutumia port 443 kwa chaguo-msingi, na HTTP hutumia port 80. Hata ikiwa imefichwa, kivinjari bado hukikagua. Tovuti mbili zina asili moja tu wakati sehemu hizi tatu zote zinafanana kabisa.

Chini ya sheria hii kali, kivinjari huzuia tovuti moja kuingilia tovuti nyingine. Fikiria unafungua akaunti yako ya benki mtandaoni kwenye kichupo kimoja cha kivinjari na unafungua tovuti yenye madhara kwenye kichupo kingine. Sera ya asili imoja huzuia tovuti yenye madhara kuiba taarifa zako za benki.

Hata hivyo, tovuti za kisasa mara nyingi zinahitaji kushirikiana data kati ya asili tofauti. Kwa mfano, ukurasa wa mbele unaweza kuhitaji kupata data kutoka kwa seva ya nyuma tofauti. Sheria za asili imoja katika vivinjari huzuia maombi haya salama, hivyo wasanidi programu walihitaji njia ya kushirikiana data kwa usalama. Hapa ndipo CORS inapoingia.

CORS ni mfumo unaoruhusu seva kufungua upatikanaji kupitia sheria za kivinjari. Inatumia vichwa maalum kutoa ruhusa ya upatikanaji wa data kwa tovuti zinazotegemewa.

Hivi ndivyo inavyotokea wakati kivinjari chako kinaomba data kutoka tovuti nyingine:

* Inajumuisha kichwa cha `Origin` kinachotaja tovuti inayofanya ombi
* Seva inayopokea kisha hukagua jina hili
* Ikiwa seva inamtumaini tovuti hiyo, hurejea na kichwa maalum cha ruhusa kinachomwambia kivinjari kuwa tovuti hiyo inaruhusiwa kusoma data inayokuja.

Ikiwa seva haitumii ruhusa hii, kivinjari huzuia data mara moja. Tovuti ya mbele hupokea ujumbe wa kosa badala ya mafaili binafsi.

Katika programu za Node.js zinazotumia Express, CORS hutekelezwa kwa kifurushi cha `cors`:

```javascript
const express = require('express');
const cors = require('cors');
const app = express();

// this enables CORS for all origins and all routes for development
app.use(cors());
```

Katika uzalishaji, unaweza kuhitaji kupitisha kitu cha usanidi ili kuzuia upatikanaji kwa kikoa maalum:

```javascript
const corsOptions = {
  origin: ['https://yourapp.com', 'https://admin.yourapp.com'],
  credentials: true
};

app.use(cors(corsOptions));
```

Ikiwa unatumia Node.js ghafi, utalazimika kushughulikia hii kwa mikono kwa kuweka vichwa vya jibu vya CORS mwenyewe kwa kila ombi:

```javascript
res.setHeader('Access-Control-Allow-Origin', 'https://example.com');
res.setHeader('Access-Control-Allow-Methods', 'GET, POST, PUT, DELETE, OPTIONS');
res.setHeader('Access-Control-Allow-Headers', 'Content-Type, Authorization');
```
