## Überblick über Graphen

Ein Graph ist eine Menge von Knoten (Vertices), die durch Kanten (Verbindungen) verbunden sind. Jeder Knoten kann mit mehreren anderen Knoten verbunden sein und bildet so ein Netzwerk. Die verschiedenen Arten von Graphen umfassen:

- Gerichtet: Kanten haben eine Richtung (von einem Knoten zu einem anderen), oft dargestellt durch gerade Linien mit Pfeilen.
- Ungerichtet: Kanten haben keine Richtung, dargestellt durch einfache Linien.
- Vertex: Jeder Knoten ist mit einer Bezeichnung oder einem Identifikator verbunden.
- Zyklisch: Enthält Zyklen (ein Pfad, der am gleichen Knoten beginnt und endet).
- Azyklisch (DAG): Enthält keine Zyklen.
- Kantenbeschriftet: Jede Kante hat eine Bezeichnung, die üblicherweise neben der entsprechenden Kante gezeichnet wird.
- Gewichtet: Kanten haben Gewichte (Werte), die für arithmetische Operationen verwendet werden können.
- Getrennt: Enthält zwei oder mehr Knoten, die durch keine Kanten verbunden sind.

Graphen werden in verschiedenen Anwendungen verwendet, wie Karten, Netzwerke, Empfehlungssysteme und Auflösungen von Abhängigkeiten.

## Graphdurchläufe

Dabei werden alle Knoten in einem Graphen besucht. Die zwei Hauptalgorithmen sind:

- **Breitensuche (BFS)**
  - Verwendet eine Warteschlange.
  - Erkundet Ebene für Ebene.
  - Findet den kürzesten Pfad in ungewichteten Graphen.

- **Tiefensuche (DFS)**
  - Verwendet einen Stapel (oder Rekursion).
  - Erkundet einen Zweig vollständig, bevor zurückgegangen wird.
  - Nützlich für Zykluserkennung und Pfadsuche.

## Graphdarstellungen

Graphen können auf zwei Hauptarten dargestellt werden:

- **Adjazenzliste**
  - Jeder Knoten hat eine Liste seiner Nachbarn.
  - Speicherplatzsparend für dünn besetzte Graphen.
  - Einfach, um Nachbarn zu durchlaufen.

- **Adjazenzmatrix**
  - Ein 2D-Array, bei dem Zeilen und Spalten Knoten darstellen.
  - Speicherintensiv für große Graphen.
  - Schnell, um zu prüfen, ob eine Kante zwischen zwei Knoten existiert.

## Bäume

Ein Baum ist eine spezielle Art von Graph, der azyklisch und zusammenhängend ist. Wichtige Eigenschaften sind:

- Sie haben keine Schleifen oder Zyklen (Pfade, bei denen Start- und Endknoten gleich sind).
- Sie müssen zusammenhängend sein (jeder Knoten ist von jedem anderen Knoten erreichbar).

### Häufige Baumtypen

Die häufigsten Baumtypen sind:

- Binäre Bäume
  - Jeder Knoten hat höchstens zwei Kinder, ein linkes und ein rechtes Kind.

- Binäre Suchbäume (BST)
  - Ein binärer Baum, bei dem jedes linke Kind kleiner als sein Elternknoten und jedes rechte Kind größer als sein Elternknoten ist.


## Tries

Auch als Präfixbäume bekannt, werden sie verwendet, um Mengen von Strings zu speichern, wobei jeder Knoten ein Zeichen darstellt.

Gemeinsame Präfixe werden nur einmal gespeichert, was sie effizient für Aufgaben wie Autovervollständigung und Rechtschreibprüfung macht.

Such- und Einfügeoperationen haben eine Zeitkomplexität von O(L), wobei L die Länge des Strings ist.

## Prioritätswarteschlangen

Eine Prioritätswarteschlange ist ein abstrakter Datentyp, bei dem jedes Element eine Priorität hat.

Warteschlangen und Stapel berücksichtigen nur die Einfügereihenfolge, während Prioritätswarteschlangen die Priorität der Elemente berücksichtigen.

Standardwarteschlangen folgen FIFO (First In First Out) und Stapel folgen LIFO (Last In First Out). In einer Prioritätswarteschlange werden jedoch Elemente mit höherer Priorität vor denen mit niedrigerer Priorität bedient, unabhängig von ihrer Einfügereihenfolge.

## Heaps

Es handelt sich um eine spezialisierte baumbasierte Datenstruktur mit einer sehr spezifischen Eigenschaft, der Heap-Eigenschaft.

Die Heap-Eigenschaft bestimmt die Beziehung zwischen Eltern- und Kindknoten. Es gibt zwei Arten von Heaps:

- Max-Heap
  - Der Wert jedes Elternknotens ist größer oder gleich den Werten seiner Kinder.
  - Das größte Element befindet sich an der Wurzel.

- Min-Heap
  - Der Wert jedes Elternknotens ist kleiner oder gleich den Werten seiner Kinder.
  - Das kleinste Element befindet sich an der Wurzel.

### Python `heapq` Modulbeispiel

```py
import heapq

# Create empty heap
my_heap = []

# Insert elements
heapq.heappush(my_heap, 9)
heapq.heappush(my_heap, 3)
heapq.heappush(my_heap, 5)
print(my_heap) # [3, 9, 5]

# Remove smallest element
print(heapq.heappop(my_heap))  # 3
print(my_heap) # [5, 9]

# Push + Pop in one step
print(heapq.heappushpop(my_heap, 7)) # 5
print(my_heap) # [7, 9]

# Transform list into heap
nums = [5, 7, 3, 1]
heapq.heapify(nums)
```

### Verwendung von Prioritäten

```py
my_heap = []
heapq.heappush(my_heap, (3, "A"))
heapq.heappush(my_heap, (2, "B"))
heapq.heappush(my_heap, (1, "C"))

# Removes lowest number = highest priority
print(heapq.heappop(my_heap))  # (1, "C")
```
