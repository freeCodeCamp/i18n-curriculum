# --description--

開発者はワークフロー内のタスクのタイミングを示すために `once` をよく使います。これは `as soon as` または `when` を意味します。

例えば、`Once you receive the client's feedback, update the project documentation.` これは、フィードバックを受け取った直後にドキュメントの更新を開始すべきであることを示し、プロジェクトのライフサイクルにおけるアクションのシーケンスを表しています。

### --feedback--

`Once` は、次のステップ（開発）が前のアクション（承認）が完了した後に行われることを示しているため、クライアントはデザインを見ているはずです。
