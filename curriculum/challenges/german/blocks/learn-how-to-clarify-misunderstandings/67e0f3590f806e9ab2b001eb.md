`Sit in on` bedeutet, an etwas (wie einem Kurs oder einer Sitzung) teilzunehmen, ohne aktiv mitzuwirken. Zum Beispiel:

`She asked if she could sit in on our design session.` – Sie wollte die Sitzung beobachten, aber nicht leiten oder viel sprechen.

`Code review` ist ein Prozess, bei dem Entwickler den Code der anderen auf Fehler und Verbesserungen überprüfen. Zum Beispiel:

`We found a small bug during the code review.` – Das Team überprüfte den Code und entdeckte ein Problem.
