# --description--

Eseguire il debugging dei tuoi programmi JavaScript va oltre l'uso delle istruzioni `console.log()` nel codice. Esistono tecniche più avanzate che rendono il debugging un gioco da ragazzi una volta che ti ci abitui. Vediamo insieme queste tecniche più avanzate così sarai felice di eseguire il debug del tuo programma JavaScript quando incontrerai degli errori.

Il primo concetto che esamineremo è lavorare con i breakpoint. I breakpoint ti permettono di mettere in pausa l'esecuzione del codice su una riga specifica a tua scelta. Dopo la pausa, puoi ispezionare variabili, valutare espressioni ed esaminare lo stack delle chiamate.

Per aggiungere un breakpoint a qualsiasi riga del codice nel browser Chrome, apri gli strumenti per sviluppatori e vai alla scheda Sources, apri il file JavaScript che vuoi debuggare e clicca sul numero di riga per impostare un breakpoint.

Quando l'esecuzione raggiunge il breakpoint e si ferma, puoi procedere passo passo nel codice usando le icone nell'angolo in alto a destra. Puoi anche aggiungere un breakpoint condizionale cliccando con il tasto destro su una riga e selezionando "Add conditional breakpoint…"

Ora passiamo alle watch expressions. Le watch expressions ti permettono di monitorare i valori di variabili o espressioni mentre il codice viene eseguito, anche se sono fuori dall'ambito corrente.

Per aggiungere una watch expression, vai alla scheda Sources degli strumenti per sviluppatori, cerca il pannello watch a destra e clicca sull'icona più (`+`) per aggiungerla.

Il concetto successivo è il profiling. Il profiling ti aiuta a identificare i colli di bottiglia nelle prestazioni permettendoti di catturare screenshot e registrare l'utilizzo della CPU, le chiamate alle funzioni e i tempi di esecuzione.

Per farlo con il tuo codice, apri la scheda Performance, clicca su record e esegui l'azione che vuoi profilare. Quando hai finito, ferma la registrazione per analizzare i risultati.

Questo può essere utile se la tua applicazione rallenta durante certe operazioni. Con il profiler puoi vedere quale funzione o altri dati nel codice consumano più risorse.

Ora passiamo a ispezionare le richieste di rete. Ispezionare le richieste di rete può aiutarti a eseguire il debug di problemi legati alle richieste API come errori nei parametri, errori di indirizzo o errori del server.

Per usare la scheda Network per il debugging, apri gli strumenti per sviluppatori e vai alla scheda Network, poi clicca sulle singole richieste per vedere dettagli come intestazioni, risposte e payload.

Per l'ultima parte di questa lezione, ci concentreremo su `console.table` e `console.dir`.

`console.table()` mostra dati tabellari come una tabella nella console. Richiede un argomento obbligatorio, che deve essere un array o un oggetto, e un argomento opzionale per specificare quali proprietà o colonne mostrare.

`console.dir()`, invece, ti permette di mostrare una lista interattiva delle proprietà di un oggetto JavaScript specificato. Produce un elenco gerarchico che può essere espanso per vedere tutte le proprietà annidate.

Ora che hai un set ampliato di tecniche di debugging, metti alla prova queste nuove competenze nei tuoi prossimi progetti JavaScript. Applicando queste strategie, potrai identificare e risolvere i problemi in modo più efficiente, ottenendo codice più pulito e affidabile.
