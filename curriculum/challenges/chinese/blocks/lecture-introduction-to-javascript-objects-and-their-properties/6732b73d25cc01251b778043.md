# --interactive--

在 JavaScript 中，理解原语和非原语数据类型之间的区别对于编写高效且无错误的代码非常重要。 

这两类数据类型在它们如何保存在内存中以及如何在你的程序中操作方面表现不同。

原语数据类型是 JavaScript 中最简单的数据形式。它们包括数字、字串、布尔、`null`、`undefined` 和符号。这些类型被称为“原语”，因为它们表现单一值且不是对象。 

当你使用原语数据类型时，你直接处理它们的值。例如，当你用原语值创建一个变量时，该值会直接保存在变量中。 

原语值是不可变的，这意味着一旦它们被创建，它们的值就不能被更改。然而，你可以为变量赋值一个新值。以下是使用原语数据类型的示例：

:::interactive_editor

```js
let num1 = 5;
let num2 = num1;
num1 = 10;

console.log(num2); // 5
```

:::

在此示例中，我们将一个原语值（`5`）从 `num1` 赋值到 `num2`。这会创建该值的独立拷贝。因此，对原始变量（`num1`）所做的任何更改都不会影响拷贝（`num2`）。

非原生类型的数据类型则更复杂。在 JavaScript 中，这些是对象，包括普通对象、数组和函数。与原语不同，非原生类型可以作为属性或元素包含多值。 

当你创建一个非原生类型值的变量时，存储在变量中的实际上是对象存储的内存位置的引用，而不是对象本身。这导致了一些重要的行为差异。以下是一个关于非原生类型的示例：

:::interactive_editor

```js
const originalPerson = { name: "John", age: 30 };
const copiedPerson = originalPerson;

originalPerson.age = 31;

console.log(copiedPerson.age); // 31
```

:::

在此示例中，我们有一个名为 `originalPerson` 的对象，具有两个属性 `name` 和 `age`。然后我们将 `originalPerson` 对象赋值给一个名为 `copiedPerson` 的变量。

然后我们更新 `originalPerson` 对象的 `age` 值。当我们记录 `copiedPerson` 对象的 `age` 属性时，它显示的是更新后的值。

但是为什么会这样？这是因为 `originalPerson` 和 `copiedPerson` 都在引用内存中的同一个对象。

在 JavaScript 中，当你将一个对象赋值给另一个变量时，你是在拷贝对该对象的引用，而不是对象本身。这被称为通过引用的浅拷贝。因此，通过一个引用对对象所做的任何更改都会反映在所有对该对象的引用中。

随着你继续使用 JavaScript，你会遇到许多需要理解原生类型和非原生类型区别的情况。这是一个基础概念，贯穿语言的许多方面，是编写高效且正确代码的关键。
