## Nozioni di base su CSS

- **Cos’è CSS?**: Cascading Style Sheets (CSS) è un linguaggio di markup usato per definire lo stile degli elementi HTML. CSS si usa per colori, immagini di sfondo, layout e altro.
- **Anatomia di base di una regola CSS**: Una regola CSS è composta da due parti principali: un selettore e un blocco di dichiarazioni. Un selettore è un pattern usato in CSS per identificare e prendere di mira specifici elementi HTML da definire nello stile. Un blocco di dichiarazioni applica un insieme di stili a un dato selettore o a più selettori.
- **Elemento `meta name="viewport"`**: Questo elemento `meta` dà al browser istruzioni su come controllare le dimensioni e la scala della pagina su dispositivi diversi, in particolare su telefoni cellulari e tablet.
- **Stili predefiniti del browser**: Ogni elemento HTML avrà applicati gli stili predefiniti del browser. Questo di solito include elementi come margini e spaziature interne predefiniti.
## CSS in linea, interno ed esterno

- **CSS in linea**: Questi stili sono scritti direttamente all'interno di un elemento HTML usando l'attributo `style`. La maggior parte delle volte non userai CSS in linea per mantenere la separazione delle responsabilità.  
- **CSS interno**: Questi stili sono scritti all'interno dei tag `<style>` nella sezione `head` di un documento HTML. Può essere utile per creare esempi di codice brevi, ma di solito non userai CSS interno.  
- **CSS esterno**: Questi stili sono scritti in un file CSS separato e collegati al documento HTML usando l'elemento `link` nella sezione `head`. Per la maggior parte dei progetti, userai un file CSS esterno invece di CSS interno o in linea.
## Lavorare con le proprietà `width` e `height`

- **Proprietà `width`**: Questa proprietà specifica la larghezza di un elemento. Se non specifichi una larghezza, il valore predefinito è `auto`. Questo permette al browser di determinare la larghezza dell'elemento in base al suo contenuto, al genitore e al tipo di visualizzazione.
- **Proprietà `min-width`**: Questa proprietà specifica la larghezza minima per un elemento.
- **Proprietà `max-width`**: Questa proprietà specifica la larghezza massima per un elemento.
- **Proprietà `height`**: Questa proprietà specifica l'altezza di un elemento. Analogamente, l'altezza è `auto` per impostazione predefinita, il che significa che si adatterà al contenuto interno.
- **Proprietà `min-height`**: Questa proprietà specifica l'altezza minima per un elemento.
- **Proprietà `max-height`**: Questa proprietà specifica l'altezza massima per un elemento.
## Tipi diversi di combinatori CSS

- **Combinatore discendente (` `)&nbsp;\*uno spazio singolo tra i selettori**: questo combinatore si usa per selezionare elementi che sono discendenti di un elemento genitore specificato.
- **Combinatore figlio (`>`)**: questo combinatore si usa per selezionare elementi che sono figli diretti di un elemento genitore specificato.
- **Combinatore fratello immediato (`+`)**: questo combinatore seleziona un elemento che segue immediatamente un elemento fratello specificato.

- **Combinatore fratello successivo (`~`)**: questo combinatore seleziona tutti i fratelli di un elemento specificato che lo seguono.
## Elementi in linea, a blocchi e a blocchi in linea

- **Elementi in linea**: Gli elementi in linea occupano solo la larghezza necessaria e non iniziano su una nuova riga. Questi elementi scorrono all'interno del contenuto, permettendo al testo e ad altri elementi in linea di apparire accanto a loro. Gli elementi in linea comuni sono gli elementi `span`, `anchor` e `img`.
- **Elementi a blocchi**: Gli elementi a blocchi occupano per impostazione predefinita tutta la larghezza disponibile, estendendosi per tutta la larghezza del loro contenitore. Alcuni elementi a blocchi comuni sono gli elementi `div`, `p` e `section`.
- **Elementi a blocchi in linea**: Puoi impostare un elemento come `inline-block` usando la proprietà `display`. Questi elementi si comportano come elementi in linea ma possono avere una `width` e `height` impostate come gli elementi a blocchi.
## Margine e spaziatura interna

- **Proprietà `margin`**: Questa proprietà serve per applicare spazio esterno all'elemento, tra il bordo dell'elemento e gli elementi circostanti.
- **Forma abbreviata `margin`**: Puoi specificare da 1 a 4 valori per impostare i lati del margine. Un valore si applica a tutti e quattro i lati; due valori impostano `top` e `bottom`, poi `right` e `left`; tre valori impostano `top`, orizzontale (`right` e `left`), poi `bottom`; quattro valori impostano `top`, `right`, `bottom`, `left`.
- **Proprietà `padding`**: Questa proprietà serve per applicare spazio interno all'elemento, tra il contenuto e il suo bordo.
- **Forma abbreviata `padding`**: Puoi specificare da 1 a 4 valori per impostare i lati della spaziatura interna. Un valore si applica a tutti e quattro i lati; due valori impostano `top` e `bottom`, poi `right` e `left`; tre valori impostano `top`, orizzontale (`right` e `left`), poi `bottom`; quattro valori impostano `top`, `right`, `bottom`, `left`.
## Specificità CSS

- **Specificità CSS in linea**: Il CSS in linea ha la specificità più alta perché viene applicato direttamente all'elemento. Sovrascrive qualsiasi CSS interno o esterno. Il valore di specificità per gli stili in linea è (1, 0, 0, 0).
- **Specificità CSS interna**: Il CSS interno è definito all'interno di un elemento `style` nella sezione `head` del documento HTML. Ha una specificità inferiore rispetto agli stili in linea. Sia il CSS interno che quello esterno condividono lo stesso livello di specificità, che è determinato dai loro selettori, non dalla loro posizione.
- **Specificità CSS esterna**: Il CSS esterno è collegato tramite un elemento `link` nella sezione `head` ed è scritto in file `.css` separati. Come il CSS interno, la sua specificità è determinata dai selettori usati. Quando due regole hanno la stessa specificità, l'ordine di origine determina il vincitore — la regola che appare più tardi nel documento ha la precedenza. Il CSS esterno offre la migliore manutenibilità per progetti più grandi.
- **Selettore universale (`*`)**: Questo è un tipo speciale di selettore CSS che corrisponde a qualsiasi elemento nel documento. Viene spesso usato per applicare uno stile a tutti gli elementi della pagina, utile per resettare o normalizzare gli stili tra diversi browser. Il selettore universale ha il valore di specificità più basso di qualsiasi selettore. Contribuisce con 0 a tutte le parti del valore di specificità (0, 0, 0, 0).
- **Selettori di tipo**: Questi selettori mirano agli elementi in base al loro nome di tag. I selettori di tipo hanno una specificità relativamente bassa rispetto ad altri selettori. Il valore di specificità per un selettore di tipo è (0, 0, 0, 1).
- **Selettori di classe**: Questi selettori sono definiti da un punto (`.`) seguito dal nome della classe. Il valore di specificità per un selettore di classe è (0, 0, 1, 0). Ciò significa che i selettori di classe possono sovrascrivere i selettori di tipo, ma possono essere sovrascritti dai selettori ID e dagli stili in linea.
- **Selettori ID**: I selettori ID sono definiti da un simbolo hash (`#`) seguito dal nome ID. I selettori ID hanno una specificità molto alta, superiore ai selettori di tipo e di classe, ma inferiore agli stili in linea. Il valore di specificità per un selettore ID è (0, 1, 0, 0).
- **Parola chiave `!important`**: usata per dare a una regola di stile la priorità più alta, permettendole di sovrascrivere qualsiasi altra dichiarazione per una proprietà. Quando viene usata, forza il browser ad applicare lo stile specificato, indipendentemente dalla specificità degli altri selettori. Bisogna usare `!important` con cautela perché può rendere il CSS più difficile da mantenere e eseguire il debug.
- **Algoritmo di cascade**: Un algoritmo usato per decidere quali regole CSS applicare quando ci sono più stili che mirano allo stesso elemento. Garantisce che vengano usati gli stili più appropriati, basandosi su un insieme di regole ben definite.
- **Ereditarietà CSS**: Il processo tramite cui gli stili vengono trasmessi dagli elementi genitori ai loro figli. L'ereditarietà permette di definire stili a un livello superiore nell'albero del documento e farli applicare a più elementi senza specificarli esplicitamente per ciascun elemento.
## Terminologia del design

- **Layout**: È il modo in cui gli elementi visivi sono disposti su una pagina o uno schermo per comunicare un messaggio. Questi elementi possono includere testo, immagini e spazi bianchi.
- **Allineamento**: È il modo in cui gli elementi sono posizionati in relazione l’uno all’altro. Usare correttamente l’allineamento aiuta a rendere il design pulito e organizzato.
- **Composizione**: È l’atto di disporre gli elementi per creare un design armonioso. Determina come elementi come immagini, testo e forme si relazionano tra loro e contribuiscono al design in modo artistico.
- **Bilanciamento**: È come il peso visivo è distribuito all’interno di una composizione. I designer mirano a creare un equilibrio tramite disposizioni simmetriche o asimmetriche.
- **Scala**: Si riferisce al confronto delle dimensioni o della grandezza di un elemento rispetto a un altro.
- **Gerarchia**: Stabilisce l’ordine di importanza degli elementi in un design. Serve a garantire che l’informazione più importante venga notata per prima.
- **Contrasto**: È il processo di creare distinzioni chiare tra gli elementi. Puoi farlo tramite variazioni di colore, dimensione, forma, texture o qualsiasi altra caratteristica visiva. Un forte contrasto aiuta anche a migliorare la leggibilità.
- **Spazio bianco (spazio negativo)**: È lo spazio vuoto in un design. È l’area che circonda gli elementi.
- **UI (interfaccia utente)**: La UI include gli elementi visivi e interattivi che gli utenti vedono sullo schermo, come icone, immagini, testo, menu, collegamenti e pulsanti.
- **UX (esperienza utente)**: La UX riguarda come gli utenti si sentono usando un prodotto o servizio. Un’applicazione con una buona esperienza utente è intuitiva, facile da usare, efficiente, accessibile e piacevole.
- **Brief di design**: È un documento che delinea gli obiettivi, le finalità e i requisiti di un progetto. È una guida che orienta il processo di design e assicura che il prodotto finale soddisfi le esigenze del cliente.
- **Design vettoriale**: Consiste nel creare arte digitale usando formule matematiche per definire linee, forme e colori.
- **Prototipazione**: Si riferisce al processo di creare un modello interattivo di un prodotto o di un’interfaccia utente.
## Fondamenti di design dell’interfaccia utente

- **Buon contrasto tra colori di sfondo e primo piano**: è importante assicurarsi che i colori di sfondo e primo piano abbiano un buon contrasto per rendere il testo leggibile. Le Web Content Accessibility Guidelines (WCAG) raccomandano un rapporto di contrasto minimo di 4,5:1 per il testo normale e 3:1 per il testo grande.
- **Buona gerarchia visiva nel design**: una gerarchia visiva forte può fornire un percorso chiaro per l’occhio, garantendo che le informazioni che trasmetti vengano consumate nell’ordine che desideri.
- **Immagini responsive**: le immagini responsive sono immagini che si adattano alle dimensioni dello schermo su cui vengono visualizzate. Questo è importante perché assicura che le tue immagini appaiano bene su tutti i dispositivi, dai desktop ai telefoni cellulari.
- **Progressive enhancement**: è un approccio di design che garantisce a tutti gli utenti, indipendentemente dal browser o dal dispositivo, di consultare il contenuto essenziale e le funzionalità di un’applicazione.
- **Design centrato sull’utente**: è un approccio che dà priorità all’utente finale, dalle sue esigenze alle sue preferenze e limitazioni. L’obiettivo del design centrato sull’utente è creare una pagina web intuitiva, efficiente da usare e piacevole con cui interagire.
- **Ricerca sugli utenti**: è lo studio sistematico delle persone che usano il tuo prodotto. L’obiettivo è misurare le esigenze, i comportamenti e i punti critici degli utenti.
- **Test con gli utenti**: si riferisce alla pratica di raccogliere dati dagli utenti mentre interagiscono con la tua applicazione.
- **Test A/B**: è il processo di distribuire una nuova funzionalità a un sottoinsieme casuale della tua base utenti. Puoi poi sfruttare i dati analitici per determinare se la funzionalità è vantaggiosa.
- **Requisiti degli utenti**: si riferisce alle storie o ai criteri che la tua applicazione deve seguire. I requisiti degli utenti possono essere definiti dalla ricerca sugli utenti o dagli standard di settore. Possono anche essere definiti dal contributo degli stakeholder.
- **Progressive disclosure**: è un pattern di design usato per mostrare agli utenti solo il contenuto rilevante in base alla loro attività corrente e nascondere il resto. Questo serve a ridurre il carico cognitivo e rendere l’esperienza utente più intuitiva.
- **Registrazione differita/pigra**: è un pattern di design dell’interfaccia che permette agli utenti di navigare e interagire con la tua applicazione senza dover registrarsi.
## Buone pratiche di design

- **Modalità scura**: è una funzionalità speciale nelle applicazioni web che permette di cambiare lo schema di colori predefinito chiaro in uno scuro. Dovresti usare colori desaturati in modalità scura. I colori desaturati sono colori meno intensi e con un livello di saturazione più basso.
- **Breadcrumbs**: è un aiuto alla navigazione che mostra all’utente la posizione nella gerarchia del sito. È meglio posizionare i breadcrumbs in cima alla pagina così gli utenti li trovano facilmente. Inoltre, assicurati che i breadcrumbs siano abbastanza grandi da essere letti facilmente, ma non così grandi da occupare troppo spazio nella pagina.
- **Componente card**: il tuo componente card dovrebbe avere un design semplice, senza essere visivamente sovraccarico o mostrare troppe informazioni. Per i media, scegli immagini e video di alta qualità per migliorare l’esperienza utente.
- **Scroll infinito**: è un pattern di design che carica più contenuti mentre l’utente scorre verso il basso nella pagina. Dovresti considerare di usare un pulsante “carica altro” perché dà all’utente il controllo su quando vuole vedere più contenuti. Puoi anche aggiungere un pulsante “indietro” così gli utenti possono tornare alla pagina precedente senza dover scorrere tutta la pagina verso l’alto.
- **Finestra modale**: è un tipo di pop-up che appare sopra il contenuto esistente della pagina. Di solito il contenuto di sfondo ha una sovrapposizione di colore scuro per aiutare l’utente a concentrarsi meglio sul contenuto della modale. Inoltre, è sempre una buona idea permettere all’utente di cliccare all’esterno della modale per chiuderla. Quando usi l’elemento HTML `dialog`, ottieni molte funzionalità e benefici di accessibilità integrati.
- **Indicazione di progresso per la registrazione in un modulo**: è un modo per mostrare agli utenti quanto sono avanzati in un processo. Può essere usato in moduli, registrazioni e processi di configurazione. Il design dovrebbe essere semplice, facile da trovare e permettere di tornare ai passaggi precedenti.
- **Carrello della spesa**: i carrelli sono un luogo dove l’utente può vedere quali elementi ha già selezionato su una piattaforma e-commerce. I tuoi carrelli dovrebbero essere sempre visibili all’utente, usare un’icona comune come un carrello, una borsa o un cestino, e avere un pulsante chiaro per procedere al checkout.

## Unità assolute

- **`px` (Pixel)**: Questa unità assoluta è una misura a dimensione fissa in CSS, che offre un controllo preciso sulle dimensioni. Ciò significa che 1px è sempre uguale a 1/96 di pollice.
- **`in` (Pollice)**: Questa unità assoluta è uguale a 96px.
- **`cm` (Centimetri)**: Questa unità assoluta è uguale a 25,2/64 di pollice.
- **`mm` (Millimetri)**: Questa unità assoluta è uguale a 1/10 di centimetro.
- **`q` (Quartidi millimetro)**: Questa unità assoluta è uguale a 1/40 di centimetro.
- **`pc` (Pica)**: Questa unità assoluta è uguale a 1/6 di pollice.
- **`pt` (Punti)**: Questa unità assoluta è uguale a 1/72 di pollice.
## Unità relative

- **Percentuali**: Queste unità relative ti permettono di definire dimensioni, misure e altre proprietà come proporzione rispetto all'elemento genitore. Per esempio, se imposti `width: 50%;` su un elemento, occuperà metà della larghezza del suo contenitore genitore.
- **Unità `em`**: Queste unità sono relative alla dimensione del carattere tipografico dell'elemento. Se usi `ems` per la proprietà `font-size`, la dimensione del testo sarà relativa alla dimensione del carattere tipografico dell'elemento genitore.
- **Unità `rem`**: Queste unità sono relative alla dimensione del carattere tipografico dell'elemento radice, che è l'elemento `html`.
- **Unità `vh`**: `vh` indica `"viewport height"` e `1vh` corrisponde all'1% dell'altezza del viewport.
- **Unità `vw`**: `vw` indica `"viewport width"` e `1vw` corrisponde all'1% della larghezza del viewport.
## `calc` funzione

- **`calc()` funzione**: Con la funzione `calc()` puoi eseguire calcoli direttamente all'interno dei tuoi fogli di stile per determinare dinamicamente i valori delle proprietà. Questo significa che puoi creare interfacce utente flessibili e responsivi calcolando le dimensioni in base alla dimensione del viewport o ad altri elementi.
## Pseudo-class di azione dell’utente

- **Definizione di pseudo-classi**: sono parole chiave speciali di CSS che permettono di selezionare un elemento in base al suo stato o posizione specifica.
- **Pseudo-classi di azione dell’utente**: sono parole chiave speciali che consentono di modificare l’aspetto degli elementi in base alle interazioni dell’utente, migliorando l’esperienza complessiva.
- **Pseudo-classe `:active`**: questa pseudo-classe ti permette di selezionare lo stato attivo di un elemento, come quando si clicca su un pulsante.
- **Pseudo-classe `:hover`**: questa pseudo-classe definisce lo stato hover di un elemento.
- **Pseudo-classe `:focus`**: questa pseudo-classe applica gli stili quando un elemento ottiene il focus, tipicamente tramite la navigazione da tastiera o quando un utente clicca in un input di un modulo.
- **Pseudo-classe `:focus-within`**: questa pseudo-classe si usa per applicare stili a un elemento quando esso o uno qualsiasi dei suoi discendenti ha il focus.
## Pseudo-classi degli input

- **Pseudo-classi degli input**: Queste pseudo-classi si usano per definire lo stile degli elementi HTML `input` in base allo stato in cui si trovano prima e dopo l’interazione dell’utente.
- **Pseudo-classe `:enabled`**: Questa pseudo-classe si usa per definire lo stile dei pulsanti del modulo o di altri elementi attualmente abilitati.
- **Pseudo-classe `:disabled`**: Questa pseudo-classe permette di definire lo stile di un elemento interattivo in modalità disabilitata.
- **Pseudo-classe `:checked`**: Questa pseudo-classe indica all’utente che è selezionato.
- **Pseudo-classe `:valid`**: Questa pseudo-classe definisce lo stile dei campi di input che soddisfano i criteri di validazione.
- **Pseudo-classe `:invalid`**: Questa pseudo-classe definisce lo stile dei campi di input che non soddisfano i criteri di validazione.
- **Pseudo-classi `:in-range` e `:out-of-range`**: Queste pseudo-classi applicano stili agli elementi in base al fatto che i loro valori siano entro o fuori dai vincoli di intervallo specificati.
- **Pseudo-classe `:required`**: Questa pseudo-classe definisce lo stile degli elementi `input` che hanno l’attributo `required`. Indica all’utente che deve compilare il campo per inviare il modulo.
- **Pseudo-classe `:optional`**: Questa pseudo-classe applica stili agli elementi di input che non sono obbligatori e possono essere lasciati vuoti.
- **Pseudo-classe `:autofill`**: Questa pseudo-classe applica stili ai campi di input che il browser compila automaticamente con dati salvati.
## Pseudo-classi di posizione

- **Pseudo-classi di posizione**: Queste pseudo-classi si usano per definire lo stile di collegamenti ed elementi che sono obiettivi all’interno del documento corrente.
- **Pseudo-classe `:any-link`**: Questa pseudo-classe è una combinazione delle pseudo-classi `:link` e `:visited`. Quindi, corrisponde a qualsiasi elemento àncora con un attributo href, indipendentemente dal fatto che sia stato visitato o meno.
- **Pseudo-classe `:link`**: Questa pseudo-classe ti permette di selezionare tutti i collegamenti non visitati su una pagina web. Puoi usarla per definire uno stile diverso ai collegamenti prima che l’utente ci clicchi sopra.
- **Pseudo-classe `:local-link`**: Questa pseudo-classe seleziona i collegamenti che puntano allo stesso documento. Può essere utile quando vuoi distinguere i collegamenti interni da quelli esterni.
- **Pseudo-classe `:visited`**: Questa pseudo-classe seleziona un collegamento che l’utente ha già visitato.
- **Pseudo-classe `:target`**: Questa pseudo-classe si usa per applicare uno stile a un elemento che è l’obiettivo di un frammento URL.
## Pseudo-classi strutturali dell'albero

- **Pseudo-classi strutturali dell'albero**: Queste pseudo-classi ti permettono di selezionare e definire lo stile degli elementi in base alla loro posizione all'interno dell'albero del documento.  
- **Pseudo-classe `:root`**: Questa pseudo-classe è solitamente l'elemento radice `html`. Ti aiuta a selezionare il livello più alto nel documento così puoi applicare uno stile comune all'intero documento.  
- **Pseudo-classe `:empty`**: Gli elementi vuoti, cioè quelli senza figli diversi dagli spazi bianchi, sono inclusi nell'albero del documento. Per questo esiste una pseudo-classe `:empty` per selezionare gli elementi vuoti.  
- **Pseudo-classe `:nth-child(n)`**: Questa pseudo-classe ti permette di selezionare elementi in base alla loro posizione all'interno di un genitore.  
- **Pseudo-classe `:nth-last-child(n)`**: Questa pseudo-classe ti consente di selezionare elementi contando dalla fine.  
- **Pseudo-classe `:first-child`**: Questa pseudo-classe seleziona il primo elemento in un genitore o nel documento.  
- **Pseudo-classe `:last-child`**: Questa pseudo-classe seleziona l'ultimo elemento in un genitore o nel documento.  
- **Pseudo-classe `:only-child`**: Questa pseudo-classe seleziona l'unico elemento in un genitore o nel documento.  
- **Pseudo-classe `:first-of-type`**: Questa pseudo-classe seleziona la prima occorrenza di un tipo specifico di elemento all'interno del suo genitore.  
- **Pseudo-classe `:last-of-type`**: Questa pseudo-classe seleziona l'ultima occorrenza di un tipo specifico di elemento all'interno del suo genitore.  
- **Pseudo-classe `:nth-of-type(n)`**: Questa pseudo-classe ti permette di selezionare un elemento specifico all'interno del suo genitore in base alla sua posizione tra i fratelli dello stesso tipo.  
- **Pseudo-classe `:only-of-type`**: Questa pseudo-classe seleziona un elemento se è l'unico del suo tipo all'interno del suo genitore.
## Pseudo-classi funzionali

- **Pseudo-classi funzionali**: Le pseudo-classi funzionali ti permettono di selezionare elementi basandoti su condizioni o relazioni più complesse. A differenza delle pseudo-classi normali che mirano a elementi in base a uno stato (per esempio, `:hover`, `:focus`), le pseudo-classi funzionali accettano argomenti.
- **Pseudo-classe `:is()`**: Questa pseudo-classe prende una lista di selettori (es. `ol`, `ul`) e seleziona un elemento che corrisponde a uno dei selettori nella lista.
- **Pseudo-classe `:where()`**: Questa pseudo-classe prende una lista di selettori (es. `ol`, `ul`) e seleziona un elemento che corrisponde a uno dei selettori nella lista. La differenza tra `:is` e `:where` è che quest’ultima avrà una specificità pari a 0.
- **Pseudo-classe `:has()`**: Questa pseudo-classe è spesso chiamata selettore `"parent"` perché ti permette di definire lo stile degli elementi che contengono figli specificati nella lista dei selettori.
## Pseudo-elementi

- **`::before` pseudo-elemento**: Questo pseudo-elemento usa la proprietà `content` per inserire contenuto cosmetico come icone subito prima dell'elemento.
- **`::after` pseudo-elemento**: Questo pseudo-elemento usa la proprietà `content` per inserire contenuto cosmetico come icone subito dopo l'elemento.
- **`::first-letter` pseudo-elemento**: Questo pseudo-elemento prende di mira la prima lettera del contenuto di un elemento, permettendoti di definirne lo stile.
- **`::marker` pseudo-elemento**: Questo pseudo-elemento ti permette di selezionare il marcatore (punto elenco o numerazione) degli elementi di una lista per definirne lo stile.
## Teoria del colore

- **Definizione di teoria del colore**: È lo studio di come i colori interagiscono tra loro e di come influenzano la nostra percezione. Copre le relazioni tra i colori, l’armonia cromatica e l’impatto psicologico del colore.
- **Colori primari**: Questi colori, che sono giallo, blu e rosso, sono le tonalità fondamentali da cui derivano tutti gli altri colori.
- **Colori secondari**: Questi colori derivano dalla miscelazione di quantità uguali di due colori primari. Verde, arancione e viola sono esempi di colori secondari.
- **Colori terziari**: Questi colori derivano dalla combinazione di un colore primario con un colore secondario adiacente. Giallo-verde, blu-verde e blu-viola sono esempi di colori terziari.
- **Colori caldi**: Questi colori, che includono rossi, arancioni e gialli, evocano sensazioni di comfort, calore e accoglienza.
- **Colori freddi**: Questi colori, che includono blu, verde e viola, evocano sensazioni di calma, serenità e professionalità.
- **Ruota dei colori**: La ruota dei colori è un diagramma circolare che mostra come i colori sono collegati tra loro. È uno strumento essenziale per i designer perché li aiuta a scegliere combinazioni di colori.
- **Schemi di colori analoghi**: Questi schemi creano esperienze coerenti e rilassanti. Sono composti da colori analoghi, cioè adiacenti tra loro sulla ruota dei colori.
- **Schemi di colori complementari**: Questi schemi creano un alto contrasto e un forte impatto visivo. I loro colori si trovano agli estremi opposti della ruota dei colori, rispetto l’uno all’altro.
- **Schema di colori triadico**: Questo schema ha colori vivaci. Sono formati da colori che sono approssimativamente equidistanti tra loro. Se collegati, formano un triangolo equilatero sulla ruota dei colori.
- **Schema di colori monocromatico**: In questo schema tutti i colori derivano dalla stessa tonalità base, modificandone la luminosità, l’oscurità e la saturazione. Evoca un senso di unità e armonia pur creando contrasto.
## Modi diversi per lavorare con i colori in CSS

- **Colori con nome**: Questi colori sono nomi predefiniti riconosciuti dai browser. Esempi includono `blue`, `darkred`, `lightgreen`.
- **Funzione `rgb()`**: RGB sta per rosso, verde e blu — i colori primari della luce. Questi tre colori si combinano in diverse intensità per creare un'ampia gamma di colori. La funzione `rgb()` ti permette di definire i colori usando il modello di colore RGB.
- **Funzione `rgba()`**: Questa funzione aggiunge un quarto valore — alpha — che controlla la trasparenza del colore.
- **Funzione `hsl()`**: HSL sta per tonalità, saturazione e luminosità — tre componenti chiave che definiscono un colore.
- **Funzione `hsla()`**: Questa funzione aggiunge un quarto valore — alpha — che controlla l'opacità del colore.
- **Esadecimale**: Un codice esadecimale (abbreviazione di codice esadecimale) è una stringa di sei caratteri usata per rappresentare i colori nel modello di colore RGB. "Esadecimale" si riferisce al sistema di numerazione in base 16, che usa le cifre da 0 a 9 e le lettere da A a F.
## `box-shadow` proprietà

- **Definizione**: La proprietà `box-shadow` applica una o più ombre attorno a un elemento.
- **Valori di offset**: Devi specificare i valori orizzontali (`offset-x`) e verticali (`offset-y`). Valori `offset-x` positivi spostano l'ombra verso destra, mentre valori negativi la spostano verso sinistra. Valori `offset-y` positivi spostano l'ombra verso il basso, mentre valori negativi la spostano verso l'alto. Se un valore è `0`, non è necessario includere un'unità.
- **Raggio di sfocatura**: Questo valore opzionale controlla quanto l'ombra appare sfocata. Se non incluso, il valore predefinito è `0`, che crea bordi netti. Più alto è il valore, più morbida è l'ombra.
- **Raggio di diffusione**: Questo valore opzionale controlla quanto l'ombra si espande o si restringe. Se non incluso, il valore predefinito è `0`.
- **Colore dell'ombra**: Puoi specificare il colore usando colori nominati, valori esadecimali, `rgb()`, `rgba()`, `hsl()` o funzioni `hsla()`.
- **Parola chiave `inset`**: Aggiungendo la parola chiave `inset` l'ombra viene posizionata all'interno dell'elemento invece che all'esterno.
- **Applicare più ombre al riquadro**: Puoi applicare più ombre separandole con delle virgole. Le ombre si sovrappongono dall'interno verso l'esterno.

box-shadow: offset-x offset-y blur-radius spread-radius color;

## Buone pratiche per definire lo stile degli input

- **Definire lo stile degli input**: Come per tutti gli elementi di testo, devi assicurarti che gli stili applicati agli input di testo siano accessibili. Questo significa che il carattere tipografico deve avere una dimensione adeguata e il colore deve avere un contrasto sufficiente con lo sfondo. Gli elementi input sono anche focalizzabili. Quando modifichi i tuoi stili, devi fare attenzione a mantenere un indicatore ben visibile quando l’elemento ha il focus, come un bordo in grassetto.
## Usare `appearance: none` per gli input

- **`appearance: none`**: I browser applicano uno stile predefinito a molti elementi. La proprietà CSS `appearance: none` ti dà il controllo completo sullo stile, ma presenta alcune avvertenze. Quando crei stili personalizzati per gli elementi input, devi assicurarti che gli indicatori di focus e di errore siano ancora visibili.
## Problemi comuni nello stile delle proprietà `datetime-local` e `color`

- **Problemi comuni**: Questi tipi speciali di input si basano su pseudo-elementi complessi per creare elementi come i selettori di data e colore. Questo rappresenta una sfida significativa per lo stile di questi input. Una sfida è che lo stile predefinito dipende completamente dal browser, quindi il CSS che scrivi per far apparire il selettore come desideri potrebbe essere completamente diverso su un altro browser.
## Proprietà CSS overflow

- **Definizione**: L’overflow indica il modo in cui gli elementi gestiscono il contenuto che supera, o "esce", dalle dimensioni dell’elemento contenitore. L’overflow è bidimensionale.  
- **`overflow-x`**: L’asse x determina l’overflow orizzontale.  
- **`overflow-y`**: L’asse y determina l’overflow verticale.
## Proprietà CSS transform

- **Definizione**: Questa proprietà ti permette di applicare varie trasformazioni agli elementi, come ruotarli, scalarli, inclinarli o traslarli (spostarli) nello spazio 2D o 3D.
- **Funzione `translate()`**: Questa funzione serve a spostare un elemento dalla sua posizione attuale.
- **Funzione `scale()`**: Questa funzione ti consente di cambiare la dimensione di un elemento.
- **Trasformazioni e accessibilità**: Se usi transform per nascondere o mostrare contenuto, assicurati che il contenuto resti comunque consultabile da lettori di schermo e navigazione da tastiera. Il contenuto nascosto dovrebbe essere veramente nascosto, ad esempio usando `display: none` o `visibility: hidden`, e non semplicemente spostato visivamente fuori dallo schermo.
## Il modello a riquadro

- **Definizione**: Nel modello a riquadro CSS, ogni elemento è circondato da un riquadro. Questo riquadro è composto da quattro componenti: l’area del contenuto, `padding`, `border`, `margin`.
- **Area del contenuto**: L’area del contenuto è la parte più interna del riquadro. È lo spazio che contiene il contenuto effettivo di un elemento, come testo o immagini.
- **`padding`**: La spaziatura interna è l’area immediatamente dopo l’area del contenuto. È lo spazio tra l’area del contenuto e il bordo di un elemento.
- **`border`**: Il bordo è il margine esterno o il contorno di un elemento nel modello a riquadro CSS. È il confine visivo dell’elemento.
- **`margin`**: Il margine è lo spazio esterno al bordo di un elemento. Determina la distanza tra un elemento e gli altri elementi intorno a esso.
## Collasso del margine

- **Definizione**: Questo comportamento si verifica quando i margini verticali di elementi adiacenti si sovrappongono, risultando in un singolo margine uguale al più grande dei due. Questo comportamento si applica solo ai margini verticali (superiore e inferiore), non ai margini orizzontali (sinistro e destro).
## I valori delle proprietà `content-box` e `border-box`

- **Proprietà `box-sizing`**: Questa proprietà serve a determinare come vengono calcolati i valori finali di `width` e `height` per un elemento HTML.
- **Valore `content-box`**: Nel modello `content-box`, i valori di `width` e `height` che imposti per un elemento determinano le dimensioni dell'area contenuto ma non includono `padding`, `border` o `margin`.
- **Valore `border-box`**: Con `border-box`, i valori di `width` e `height` di un elemento includono l'area contenuto, `padding` e `border`, ma non includono `margin`.
## Reset CSS

- **Definizione**: Un reset CSS è un foglio di stile che rimuove tutta o parte della formattazione predefinita che i browser web applicano agli elementi HTML. Tra le opzioni di terze parti per i reset CSS ci sono `sanitize.css` e `normalize.css`.
## Proprietà filtro CSS

- **Definizione**: Questa proprietà può essere usata per creare vari effetti come sfocatura, variazione di colore e regolazione del contrasto.
- **Funzione `blur()`**: Questa funzione applica una sfocatura gaussiana all'elemento. La quantità è definita in pixel e rappresenta il raggio della sfocatura.
- **Funzione `brightness()`**: Questa funzione regola la luminosità dell'elemento. Un valore di 0% rende l'elemento completamente nero, mentre valori superiori al 100% aumentano la luminosità.
- **Funzione `grayscale()`**: Questa funzione converte l'elemento in scala di grigi. La quantità è definita come percentuale, dove 100% è completamente in scala di grigi e 0% lascia l'immagine invariata.
- **Funzione `sepia()`**: Questa funzione applica una tonalità seppia all'elemento. Come per la scala di grigi, usa un valore percentuale.
- **Funzione `hue-rotate()`**: Questa funzione applica una rotazione della tonalità all'elemento. Il valore è definito in gradi e rappresenta una rotazione intorno al cerchio dei colori.
## Introduzione a CSS flexbox e al modello flex

- **Definizione**: CSS flexbox è un modello di layout monodimensionale che ti permette di disporre gli elementi in righe e colonne all'interno di un contenitore.  
- **Modello flex**: questo modello definisce come gli elementi flex sono disposti all'interno di un contenitore flex. Ogni contenitore flex ha due assi: l'asse principale e l'asse trasversale.
## La proprietà `flex-direction`

- **Definizione**: Questa proprietà imposta la direzione dell'asse principale. Il valore predefinito di `flex-direction` è `row`, che posiziona tutti gli elementi flex sulla stessa riga, nella direzione della lingua predefinita del tuo browser (da sinistra a destra o da destra a sinistra).
- **`flex-direction: row-reverse;`**: Inverte gli elementi nella riga.
- **`flex-direction: column;`**: Allinea gli elementi flex verticalmente invece che orizzontalmente.
- **`flex-direction: column-reverse;`**: Inverte l'ordine degli elementi flex verticalmente.
## La proprietà `flex-wrap`

- **Definizione**: Questa proprietà determina come gli elementi flex vengono avvolti all'interno di un contenitore flex per adattarsi allo spazio disponibile. `flex-wrap` può assumere tre valori possibili: `nowrap`, `wrap` e `wrap-reverse`.
- **`flex-wrap: nowrap;`**: Questo è il valore predefinito. Gli elementi flex non verranno avvolti su una nuova riga, anche se la loro larghezza supera quella del contenitore.
- **`flex-wrap: wrap;`**: Questa proprietà avvolgerà gli elementi quando superano la larghezza del loro contenitore.
- **`flex-wrap: wrap-reverse;`**: Questa proprietà avvolgerà gli elementi flex in ordine inverso.
- **Proprietà `flex-flow`**: Questa proprietà è una proprietà abbreviata per `flex-direction` e `flex-wrap`.
## La proprietà `justify-content`

- **Definizione**: Questa proprietà allinea gli elementi figli lungo l'asse principale del contenitore flex.
- **`justify-content: flex-start;`**: In questo caso, gli elementi flex saranno allineati all'inizio dell'asse principale. Questo può essere orizzontale o verticale.
- **`justify-content: flex-end;`**: In questo caso, gli elementi flex sono allineati alla fine dell'asse principale, orizzontalmente o verticalmente.
- **`justify-content: center;`**: Questo centra gli elementi flex lungo l'asse principale.
- **`justify-content: space-between;`**: Questo distribuirà gli elementi in modo uniforme lungo l'asse principale.
- **`justify-content: space-around;`**: Questo distribuirà gli elementi flex in modo uniforme all'interno dell'asse principale, aggiungendo uno spazio prima del primo elemento e dopo l'ultimo.
- **`justify-content: space-evenly;`**: Questo distribuisce gli elementi in modo uniforme lungo l'asse principale.
## La proprietà `align-items`

- **Definizione**: Questa proprietà serve a distribuire gli elementi lungo l’asse trasversale. Ricorda che l’asse trasversale è perpendicolare all’asse principale.
- **`align-items: center;`**: Serve per centrare gli elementi lungo l’asse trasversale.
- **`align-items: flex-start;`**: Allinea gli elementi all’inizio dell’asse trasversale.
- **`align-items: stretch;`**: Serve per estendere gli elementi flex lungo l’asse trasversale.
## Introduzione alla tipografia

- **Definizione**: La tipografia è l'arte di scegliere i caratteri tipografici e il formato giusti per rendere il testo visivamente attraente e facile da leggere. "Tipo" si riferisce a come i singoli caratteri sono progettati e disposti.
- **Definizione di carattere tipografico**: Un carattere tipografico è il design e lo stile complessivo di un insieme di caratteri, numeri e simboli. È come un progetto per una famiglia di font.
- **Definizione di font**: Un font è una variazione specifica di un carattere tipografico con caratteristiche particolari, come dimensione, spessore, stile e larghezza.
- **Carattere serif**: Questo carattere ha uno stile classico con piccole linee alla fine dei caratteri. I caratteri serif sono comunemente usati per materiali stampati, come i libri.
- **Carattere sans serif**: Questo carattere ha un aspetto più moderno, senza le piccole linee alla fine dei caratteri. I caratteri sans serif sono comunemente usati nel design digitale perché sono facili da leggere sullo schermo. Alcuni esempi includono Helvetica, Arial e Roboto.
- **Spessore del font**: È lo spessore dei caratteri, inclusi light, regular, bold e black.
- **Stile del font**: È l'inclinazione e l'orientamento dei caratteri, come italic e oblique.
- **Larghezza del font**: È lo spazio orizzontale occupato dai caratteri, come condensed e expanded.
- **Linea di base**: È la linea immaginaria su cui si appoggiano la maggior parte dei caratteri.
- **Altezza delle maiuscole**: È l'altezza delle lettere maiuscole, misurata dalla linea di base alla sommità.
- **Altezza della x**: È l'altezza media delle lettere minuscole, escludendo ascendenti e discendenti.
- **Ascendenti**: Sono le parti delle lettere minuscole che si estendono sopra l'altezza della x, come le sommità delle lettere 'h', 'b', 'd' e 'f'.
- **Discendenti**: Sono le parti delle lettere minuscole che si estendono sotto la linea di base, come le code di 'y', 'g', 'p' e 'q'.
- **Kerning**: È l'aggiustamento dello spazio tra coppie specifiche di caratteri per migliorarne la leggibilità e l'estetica. Per esempio, ridurre lo spazio tra le lettere A e V.
- **Tracking**: È l'aggiustamento dello spazio tra tutti i caratteri in un blocco di testo. È essenzialmente la distanza tra i caratteri. Influisce su quanto il testo risulta denso o aperto.
- **Interlinea**: È lo spazio verticale tra le righe di testo. Si misura dalla linea di base di una riga a quella della riga successiva.
- **Buone pratiche con la tipografia**: Dovresti scegliere font chiari e semplici per rendere i tuoi design facili da comprendere. Questo è particolarmente importante per il testo principale del tuo sito web. Gli utenti sono più propensi a interagire con il tuo contenuto se il font è facile da leggere. Dovresti usare al massimo due o tre font per creare una coerenza visiva. Usare troppi font può rendere il testo più difficile da leggere e indebolire l'identità del tuo brand. Questo può anche influire sull'esperienza utente aumentando i tempi di caricamento del sito. Puoi usare la dimensione del font per creare una gerarchia visiva per intestazioni, sottotitoli, paragrafi e altri elementi. Per esempio, l'intestazione principale di una pagina web dovrebbe avere un font più grande, seguita da sottotitoli con dimensioni più piccole. Questo darà a ogni elemento nella gerarchia una dimensione specifica che aiuta gli utenti a navigare visivamente nella struttura.
## Proprietà CSS `font-family`

- **Definizione**: Una famiglia di caratteri è un gruppo di caratteri tipografici che condividono un design comune. Tutti i caratteri appartenenti alla stessa famiglia si basano sullo stesso tipo di carattere principale, ma presentano anche variazioni nello stile, nel peso e nella larghezza. Puoi specificare più famiglie di caratteri in ordine di priorità, dalla più alta alla più bassa, separandole con delle virgole. Questi caratteri alternativi fungeranno da opzioni fallback. Dovresti sempre includere una famiglia di caratteri generica alla fine dell'elenco font-family.  
- **Famiglie di caratteri comuni**: Ecco alcune famiglie di caratteri comuni usate in CSS: serif, sans-serif, monospace, cursive, fantasy
## Caratteri tipografici sicuri per il web

- **Definizione**: Questi caratteri tipografici sono un sottoinsieme di caratteri molto probabilmente installati su un computer o dispositivo. Quando il browser deve renderizzare un carattere, cerca di trovare il file del carattere sul sistema dell'utente. Ma se il carattere non viene trovato, di solito ricade su un carattere di sistema predefinito.

- **Caratteri tipografici sicuri per il web**: 
  - I caratteri sans-serif sono comunemente usati per lo sviluppo web perché non hanno piccole "zampe" o linee alla fine dei caratteri, quindi sono facili da leggere sullo schermo. Alcuni esempi di caratteri sans-serif sicuri per il web sono: Arial, Verdana e Trebuchet MS.
  - Al contrario, i caratteri serif hanno piccole "zampe" alla fine dei caratteri, quindi sono comunemente usati per la stampa tradizionale. I caratteri serif sicuri per il web includono: Times New Roman e Georgia.
## Regole at e la regola at `@font-face`

- **Definizione**: Le regole at sono istruzioni che forniscono indicazioni al browser. Puoi usarle per definire vari aspetti del foglio di stile, come media query, fotogrammi chiave, caratteri tipografici e altro.  
- **`@font-face`**: Questo ti permette di definire un carattere tipografico personalizzato specificando il file del carattere, il formato e altre proprietà importanti, come peso e stile. Perché la regola at `@font-face` sia valida, devi anche specificare la proprietà `src`. Questa proprietà contiene riferimenti alle risorse del carattere.
- **Formati dei caratteri**: Per ogni risorsa del carattere puoi anche specificare il formato. È opzionale. È un suggerimento per il browser sul formato del carattere. Se il formato viene omesso, la risorsa verrà scaricata e il formato sarà rilevato dopo il download. Se il formato è non valido, la risorsa non verrà scaricata. I formati di carattere possibili includono `collection`, `embedded-opentype`, `opentype`, `svg`, `truetype`, `woff` e `woff2`.
- **`woff` e `woff2`**: `woff` sta per "Web Open Font Format". È un formato di carattere molto usato sviluppato da Mozilla in collaborazione con Type Supply, LettError e altre organizzazioni. La differenza tra `woff` e `woff2` è l'algoritmo usato per comprimere i dati.
- **OpenType**: È un formato per caratteri scalabili per computer sviluppato da Microsoft e Adobe che permette agli utenti di accedere a funzionalità aggiuntive in un carattere. È ampiamente usato nei principali sistemi operativi.
- **`tech()`**: Questo serve a specificare la tecnologia della risorsa del carattere. È opzionale.
## Lavorare con i caratteri tipografici esterni

- **Definizione**: Un carattere tipografico esterno è un file di carattere che non è incluso direttamente nei file del tuo progetto. Di solito sono ospitati su un server separato. Per includere questi caratteri esterni all'interno del tuo progetto, puoi usare un elemento `link` o un'istruzione `@import` nel tuo CSS.  
- **Google Fonts**: Questo è un servizio di Google che offre una raccolta di caratteri, molti dei quali sono progettati specificamente per lo sviluppo web.  
- **Font Squirrel**: Questa è un'altra risorsa popolare che puoi usare per aggiungere caratteri esterni personalizzati ai tuoi progetti.
## `text-shadow` Proprietà

- **Definizione**: Questa proprietà si usa per applicare ombre al testo. Devi specificare l’offset X e l’offset Y, che rappresentano rispettivamente la distanza orizzontale e verticale dell’ombra rispetto al testo. Questi valori sono obbligatori. Valori positivi di offset X e offset Y sposteranno l’ombra verso destra e verso il basso, mentre valori negativi la sposteranno verso sinistra e verso l’alto.
- **Colore dell’ombra**: Puoi anche personalizzare il colore dell’ombra specificando questo valore prima o dopo l’offset. Se il colore non viene specificato, il browser lo determinerà automaticamente, quindi di solito è meglio impostarlo esplicitamente.
- **Raggio di sfocatura**: Il raggio di sfocatura è opzionale ma rende l’ombra molto più morbida e sottile. Il valore predefinito del raggio di sfocatura è zero. Più alto è il valore, maggiore sarà la sfocatura, il che significa che l’ombra diventa più chiara.
- **Applicare più ombre al testo**: Il testo può avere più di un’ombra. Le ombre verranno applicate a strati, dalla prima in primo piano fino all’ultima sullo sfondo.

## `max()` funzione

La funzione `max()` restituisce il valore più grande di un insieme di valori separati da virgola:

```css
img {
  width: max(250px, 25vw);
}
```

Nell'esempio sopra, la larghezza dell'immagine sarà di 250px se la larghezza del viewport è inferiore a 1000 pixel (dato che 250px è maggiore del 25% di quella larghezza del viewport). Se la larghezza del viewport è superiore a 1000 pixel, la larghezza dell'immagine sarà 25vw. Questo perché 25vw corrisponde al 25% della larghezza del viewport.
## Buone pratiche con CSS e accessibilità    

- **`display: none;`**: Usare `display: none;` significa che i lettori di schermo e altre tecnologie assistive non potranno consultare questo contenuto, perché non è incluso nell'albero di accessibilità. Perciò, è importante usare questo metodo solo quando vuoi rimuovere completamente il contenuto sia dalla presentazione visiva sia dall'accessibilità.
- **`visibility: hidden;`**: Questa proprietà e valore nascondono il contenuto visivamente ma lo mantengono nel flusso del documento, quindi occupa ancora spazio nella pagina. Questi elementi non saranno più letti dai lettori di schermo perché saranno stati rimossi dall'albero di accessibilità.
- **`.sr-only` classe CSS**: Questa è una tecnica comune usata per nascondere visivamente il contenuto mantenendolo accessibile ai lettori di schermo.

```css
.sr-only {
  position: absolute;
  width: 1px;
  height: 1px;
  padding: 0;
  margin: -1px;
  overflow: hidden;
  clip: rect(0, 0, 0, 0);
  white-space: nowrap;
  border: 0;
}
```

- **`scroll-behavior: smooth;`**: Questa proprietà e valore abilitano un comportamento di scorrimento fluido.
- **`prefers-reduced-motion` funzionalità**: Questa è una funzionalità media che può essere usata per rilevare la preferenza di animazione dell'utente.

```css
@media (prefers-reduced-motion: no-preference) {
  * {
    scroll-behavior: smooth;
  }
}
```

Nell'esempio sopra, lo scorrimento fluido è abilitato se l'utente non ha impostato una preferenza di animazione sul proprio dispositivo.
## Nascondere contenuti con attributi HTML

- **Attributo `aria-hidden`**: usato per nascondere un elemento alle persone che usano tecnologia assistiva come i lettori di schermo. Per esempio, può essere usato per nascondere immagini decorative che non forniscono contenuti significativi.
- **Attributo `hidden`**: questo attributo è supportato dalla maggior parte dei browser moderni e nasconde il contenuto sia visivamente sia dall'albero di accessibilità. Può essere facilmente attivato o disattivato con JavaScript.

:::interactive_editor

```html
<p aria-hidden>This paragraph is visible for sighted users, but is hidden from assistive technology.</p>
<p hidden>This paragraph is hidden from both sighted users and assistive technology.</p>
```

:::
## Problema di accessibilità dell'attributo `placeholder`  

Usare il testo segnaposto non è ideale per l'accessibilità. Spesso gli utenti confondono il testo segnaposto con un vero valore di input — pensano che ci sia già un valore nell'input.
## Lavorare con selettori di attributi diversi e collegamenti

- **Definizione**: Il selettore `attribute` ti permette di selezionare elementi HTML in base ai loro attributi come gli attributi `href` o `title`.  
- **Attributo `title`**: Questo attributo fornisce informazioni aggiuntive su un elemento.
## Selezionare elementi con l'attributo `lang` e `data-lang`

- **Attributo `lang`**: Questo attributo si usa in HTML per specificare la lingua del contenuto all'interno di un elemento. Potresti voler definire lo stile degli elementi in modo diverso in base alla lingua in cui sono scritti, soprattutto su un sito web multilingue.  
- **Attributo `data-lang`**: Gli attributi dati personalizzati come l'attributo `data-lang` sono comunemente usati per archiviare informazioni aggiuntive negli elementi, ad esempio per specificare la lingua usata in una sezione specifica di testo.
## Lavorare con il selettore di attributi, gli elementi di lista ordinata e l'attributo `type`

- **Attributo `type`**: Quando lavori con le liste ordinate in HTML, l'attributo `type` ti permette di specificare lo stile di numerazione usato, come numeri, lettere o numeri romani.
## Lavorare con i float

- **Definizione**: I float si usano per rimuovere un elemento dal suo flusso normale nella pagina e posizionarlo sul lato sinistro o destro del suo contenitore. Quando succede questo, il testo si avvolge attorno a quel contenuto flottante.
- **Cancellare i float**: La proprietà `clear` serve a stabilire se un elemento deve essere spostato sotto il contenuto flottante. Quando ci sono più elementi flottanti affiancati, possono sorgere problemi di sovrapposizione e collasso nei layout. Perciò è stato creato un trucco `clearfix` per risolvere questo problema.
## Posizionamento statico, relativo e assoluto

- **Posizionamento statico**: è il flusso normale del documento. Gli elementi sono posizionati dall'alto verso il basso e da sinistra a destra uno dopo l'altro.
- **Posizionamento relativo**: permette di usare le proprietà `top`, `left`, `right` e `bottom` per posizionare l'elemento all'interno del flusso normale del documento. Puoi anche usare il posizionamento relativo per far sovrapporre gli elementi ad altri elementi nella pagina.
- **Posizionamento assoluto**: permette di estrarre un elemento dal flusso normale del documento, facendolo comportare in modo indipendente dagli altri elementi.
## Posizionamento fisso e sticky

- **Posizionamento fisso**: Quando un elemento è posizionato con `position: fixed`, viene rimosso dal flusso normale del documento e posizionato rispetto alla viewport, quindi rimane nella stessa posizione anche quando l’utente scorre la pagina. Questo si usa spesso per elementi come intestazioni o barre di navigazione che devono restare sempre visibili.
- **Posizionamento sticky**: Questo tipo di posizionamento funziona come un ibrido tra il posizionamento relativo e quello fisso. All’inizio, l’elemento si comporta come se fosse posizionato relativamente, restando nel flusso del documento. Però, una volta che l’utente scorre oltre un certo punto, l’elemento “si attacca” alla viewport (di solito in alto) e si comporta come se fosse fisso. È ottimo per creare elementi come barre di navigazione sticky, che diventano fisse solo quando l’utente scorre fino a una certa posizione.
## Lavorare con la proprietà `z-index`

- **Definizione**: La proprietà `z-index` in CSS serve a controllare l'ordine di sovrapposizione verticale degli elementi posizionati che si sovrappongono sulla pagina.
## Design web responsivo

- **Definizione**: Il principio fondamentale del design responsivo è l’adattabilità – la capacità di un sito web di modificare il proprio layout e contenuto in base alla dimensione dello schermo e alle capacità del dispositivo su cui viene visualizzato.
- **Griglie fluide**: Usano unità relative come percentuali invece di unità fisse come i pixel, permettendo al contenuto di ridimensionarsi e riorganizzarsi in base alla dimensione dello schermo.
- **Immagini flessibili**: Sono impostate per ridimensionarsi all’interno degli elementi contenitori, assicurando che non escano dai loro contenitori su schermi più piccoli.
## Media query

- **Definizione**: permettono agli sviluppatori di applicare stili diversi in base alle caratteristiche del dispositivo, principalmente alla larghezza del viewport.  
- **`all` tipo di media**: è adatto a tutti i dispositivi. È il predefinito se non viene specificato alcun tipo di media.  
- **`print` tipi di media**: è pensato per materiale impaginato e documenti visualizzati su schermo in modalità anteprima di stampa.  
- **`screen` tipi di media**: è destinato principalmente agli schermi.  
- **`aspect-ratio`**: descrive il rapporto tra larghezza e altezza del viewport.  
- **`orientation`**: indica se il dispositivo è in orientamento orizzontale o verticale.  
- **`resolution`**: descrive la risoluzione del dispositivo di output in punti per pollice (dpi) o punti per centimetro (dpcm).  
- **`hover`**: serve a verificare se il meccanismo di input principale può passare sopra gli elementi (hover).  
- **`prefers-color-scheme`**: rileva se l’utente ha richiesto un tema chiaro o scuro.  
- **Media query e operatori logici**: l’operatore `and` serve a combinare più funzionalità media, mentre `not` e `only` possono essere usati per negare o isolare le media query.
## Punti di interruzione comuni per i media

- **Definizione**: I punti di interruzione per i media sono momenti specifici nel design di un sito web in cui il layout e il contenuto si adattano per accogliere diverse dimensioni dello schermo. Esistono alcuni punti di interruzione generali che puoi usare per indirizzare telefoni, tablet e schermi desktop. Ma non è consigliabile cercare di coprire ogni singola dimensione possibile per i diversi dispositivi.
- **Dispositivi piccoli (smartphone)**: fino a 640px
- **Dispositivi medi (tablet)**: da 641px a 1024px
- **Dispositivi grandi (desktop)**: 1025px e oltre
## Approccio mobile first

- **Definizione**: L'approccio mobile first è una filosofia di design e una strategia di sviluppo nel design web responsivo che dà priorità alla creazione di siti web per dispositivi mobili prima di progettare per schermi più grandi.
## Proprietà personalizzate CSS (variabili CSS)

- **Definizione**: Le proprietà personalizzate CSS, conosciute anche come variabili CSS, sono entità definite dagli autori CSS che contengono valori specifici da riutilizzare in tutto un documento. Sono una funzionalità potente che permette di creare fogli di stile più efficienti, manutenibili e flessibili. Le proprietà personalizzate sono particolarmente utili per creare design con temi personalizzabili. Puoi definire un insieme di proprietà per temi diversi.
## La regola `@property`

- **Definizione**: La regola `@property` è una potente funzionalità CSS che permette agli sviluppatori di definire proprietà personalizzate con un controllo maggiore sul loro comportamento, incluso come animarle e i loro valori iniziali.

```css
@property --property-name {
  syntax: '<type>';
  inherits: true | false;
  initial-value: <value>;
}
```

- **`--property-name`**: Questo è il nome della proprietà personalizzata che stai definendo. Come tutte le proprietà personalizzate, deve iniziare con due trattini.
- **`syntax`**: Questo definisce il tipo della proprietà, che può essere cose come `<color>`, `<length>`, `<number>`, `<percentage>` o tipi più complessi.
- **`inherits`**: Questo specifica se la proprietà dovrebbe ereditare il suo valore dall'elemento genitore.
- **`initial-value`**: Questo imposta il valore predefinito della proprietà.
- **Fallback**: Quando usi la proprietà personalizzata, puoi fornire un valore fallback usando la funzione `var()`, proprio come faresti con le proprietà personalizzate standard.
## Nozioni di base su CSS Grid

- **Definizione**: CSS Grid è un sistema di layout bidimensionale usato per creare layout complessi nelle pagine web. Le griglie hanno righe e colonne con spazi tra di esse. Per definire un layout a griglia, imposta la proprietà `display` su `grid`.
- **L'unità `fr` (frazionale)**: Questa unità rappresenta una frazione dello spazio all'interno del contenitore griglia. Puoi usare l'unità `fr` per creare griglie flessibili.
- **Creare spazi tra le tracce**: Ci sono tre modi per creare spazi tra le tracce in CSS grid. Puoi usare la proprietà `column-gap` per creare spazi tra le colonne. Puoi usare la proprietà `row-gap` per creare spazi tra le righe. Oppure puoi usare la proprietà abbreviata `gap` per creare spazi sia tra righe sia tra colonne.
- **Funzione `repeat()`**: Questa funzione serve per ripetere sezioni nell'elenco delle tracce. Invece di scrivere `grid-template-columns: 1fr 1fr 1fr;` puoi usare la funzione `repeat()`.
- **Griglia esplicita**: Puoi specificare il numero di righe o colonne usando le proprietà `grid-template-columns` o `grid-template-rows`.
- **Griglia implicita**: Quando gli elementi sono posizionati fuori dalla griglia, righe e colonne vengono create automaticamente per quegli elementi esterni.
- **Funzione `minmax()`**: Questa funzione serve per impostare le dimensioni minime e massime di una traccia.
- **Posizionamento automatico della griglia**: Il browser posiziona automaticamente gli elementi usando l'algoritmo di posizionamento automatico. La proprietà `grid-auto-flow` controlla questo comportamento, usando valori come `row`, `column` o `dense` per influenzare la direzione del posizionamento e se gli spazi vuoti vengono riempiti.
- **Posizionamento basato sulle righe**: Tutte le griglie hanno righe. Per specificare dove inizia un elemento su una riga, puoi usare le proprietà `grid-column-start` e `grid-row-start`. Per specificare dove finisce l'elemento sulla riga, puoi usare le proprietà `grid-column-end` e `grid-row-end`. Puoi anche scegliere di usare le proprietà abbreviate `grid-column` o `grid-row`.
- **`grid-template-areas`**: Questa proprietà serve per dare un nome agli elementi che vuoi posizionare sulla griglia.
- **Allineamento:** Usa `justify-items` e `align-items` per controllare come gli elementi sono allineati nelle loro aree di griglia. Usa `justify-self` e `align-self` sugli elementi griglia individuali per sovrascrivere l’allineamento impostato sul contenitore.
## Nozioni di base sulle animazioni CSS

- **Definizione**: Le animazioni CSS ti permettono di creare effetti dinamici e visivamente coinvolgenti sulle pagine web senza bisogno di JavaScript o programmazione complessa. Offrono un modo per far passare gli elementi in modo fluido tra stili diversi in un intervallo di tempo specificato.
- **La regola `@keyframes`**: Questa regola definisce le fasi e gli stili dell'animazione. Specifica quali stili l'elemento deve avere in vari momenti durante l'animazione.
- **Proprietà `animation`**: Questa è la proprietà abbreviata usata per applicare le animazioni.
- **`animation-name`**: Specifica il nome della regola `@keyframes` da usare.
- **`animation-duration`**: Imposta la durata necessaria per completare l'animazione.
- **`animation-timing-function`**: Definisce come l'animazione procede nel tempo (ad esempio ease, linear, ease-in-out).
- **`animation-delay`**: Specifica un ritardo prima che l'animazione inizi.
- **`animation-iteration-count`**: Imposta quante volte l'animazione deve ripetersi.
- **`animation-direction`**: Determina se l'animazione deve andare avanti, indietro o alternarsi.
- **`animation-fill-mode`**: Specifica come l'elemento deve essere stilizzato prima e dopo l'animazione.
- **`animation-play-state`**: Permette di mettere in pausa e riprendere l'animazione.
## Accessibilità e la media query `prefers-reduced-motion`

- **La media query `prefers-reduced-motion`**: Una delle principali preoccupazioni di accessibilità con le animazioni è che possono causare disagio o addirittura danni fisici ad alcuni utenti. Le persone con disturbi vestibolari o sensibilità al movimento possono provare vertigini, nausea o mal di testa quando sono esposte a certi tipi di movimento sullo schermo. La media query `prefers-reduced-motion` permette agli sviluppatori web di rilevare se l’utente ha richiesto animazioni o effetti di movimento ridotti a livello di sistema.
