# --description--

`README` 文件是许多项目的重要组成部分，尤其是在软件开发中。它通常是人们查看项目时看到的第一个文档。`README` 文件解释了项目的内容、如何使用它，有时还包括如何为其做出贡献。在项目文档的上下文中，理解 `README` 文件的用途至关重要。

# --questions--

## --text--

`README` 文件在项目中通常用于什么？

## --answers--

存储代码

### --feedback--

虽然代码是项目的组成部分，但 `README` 文件不是用来存储代码的，而是用来提供关于项目的信息。

---

关于该项目的说明

---

联系信息

### --feedback--

尽管可以包含联系信息，`README` 文件的主要目的是提供关于项目的说明和信息，而不仅仅是联系详情。

---

变更日志

### --feedback--

变更日志，通常称为 changelog，通常与 `README` 文件分开。`README` 文件侧重于介绍和解释项目。
