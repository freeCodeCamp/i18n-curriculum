# --description--

プロジェクトを作成していると、プログラムが期待通りに動作しない問題に頻繁に遭遇します。

プログラマーはこうした問題をバグと呼ぶことが多いです。バグを見つけて修正するプロセスはデバッグと呼ばれます。

コードをデバッグするには、ブラウザが提供するいくつかのツールを使う必要があります。

重要なツールの2つは、DOMインスペクターと開発者ツールです。

DOMインスペクターは、現在表示しているページのHTML構造を調べることができます。

DOMはDocument Object Modelの略です。ページ上の要素を表す木構造のようなものです。DOMについては後のモジュールでさらに学びます。

開発者ツールは、現在のページのHTML、CSS、JavaScriptを調べることができます。

アンカー要素に小さなバグがあるHTMLの例を見てみましょう。

```html
<a href="https://www.freecodecamp.org/larn/">freeCodeCamp curriculum</a>
```

リンクをクリックすると、404ページに移動します。404ページは、ユーザーがサーバー上に存在しないウェブページにアクセスしようとしたときに表示されるエラーページです。

このリンクはfreeCodeCampのカリキュラムにリンクすることが意図されています。

問題が何かを確認するために、開発者ツールを使うことができます。

ブラウザで開発者ツールを開くには、ページ上で右クリックして`Inspect`を選択してください。

また、PCのキーボードでは`Control Shift I`、Macでは`Command Option I`を使うこともできます。

Google Chromeで開発者ツールを開くと、いくつかのタブが表示されます。最初のタブは`Elements`タブと呼ばれます。このタブは現在のページのHTML構造を表示します。

2番目のタブは`Console`タブと呼ばれます。このタブはページ上で発生している可能性のあるエラーを表示します。

リンク切れがある場合は、コンソールでそのリンクのエラーメッセージを確認できます。リンク切れでよく表示されるメッセージは404エラーです。404エラーはページが見つからないことを示します。

これにより、アンカー要素のURLに問題があることがわかります。`href`の値を調べるとタイプミスがあることがわかります。

現在コンソールのメッセージには404に対して`/larn`が表示されていますが、正しいURLは`/learn`です。リンクを修正すれば、期待通りに動作します。

開発者ツールの使い方は認定資格の中でさらに学びますが、これはコードのデバッグに役立つ簡単な例です。
