## HTML 基礎知識

- **HTML 的角色**：HTML 表述網頁的內容和結構。
- **HTML 元素**：元素是 HTML 文件的建造區塊。它們表述標題、段落、連結、圖片等。大多數 HTML 元素由一個開啟頁籤（`<elementName>`）和一個關閉頁籤（`</elementName>`）組成。

這是基本語法：

```html
<elementName>Content goes here</elementName>
```

- **空元素**：空元素不能有任何內容，且只有開始標籤。範例包括 `img` 和 `meta` 元素。

```html
<img>
<meta>
```

在某些程式碼庫中看到在 void 元素內包含斜線 `/` 是很常見的。以下兩種寫法皆可接受：

```html
<img>
<img/>
```

- **屬性**：屬性是放置在 HTML 元素開啟標記內的值。屬性提供關於該元素的額外資訊或指定該元素應該如何運作。以下是屬性的基本語法：

```html
<element attribute="value"></element>
```

布林值屬性是在 HTML 頁籤中可以存在或不存在的屬性。如果存在，值為 true，否則為 false。布林值屬性的範例包括 `disabled`、`readonly` 和 `required`。 

- **註解**：註解用於程式設計中，在你的程式碼中為你自己和其他開發者留下備註。以下是 HTML 中註解的語法：

```html
<!--This is an HTML comment.-->
```

## 常見的 HTML 元素

- **標題元素**：HTML 中有六個標題元素。`h1` 到 `h6` 標題元素用來表示其下方內容的重要性。數字越小，重要性越高，因此 `h2` 元素的重要性低於 `h1` 元素。

:::interactive_editor

```html
<h1>most important heading element</h1>
<h2>second most important heading element</h2>
<h3>third most important heading element</h3>
<h4>fourth most important heading element</h4>
<h5>fifth most important heading element</h5>
<h6>least important heading element</h6>
```

:::

- **段落元素**：這用於網頁上的段落。

:::interactive_editor

```html
<p>This is a paragraph element.</p>
```

:::

- **`img` 元素**：`img` 元素用於將圖片添加到網頁中。`src` 屬性用來指定該圖片的位置。對於圖片元素，最好包含另一個稱為 `alt` 屬性的屬性。以下是一個帶有 `src` 和 `alt` 屬性的 `img` 元素範例：

:::interactive_editor

```html
<img src="https://cdn.freecodecamp.org/curriculum/cat-photo-app/lasagna.jpg" alt="A slice of lasagna on a plate.">
```

:::

- **`body` Element**：此元素用於表述 HTML 文件的內容。

:::interactive_editor

```html
<body>
  <h1>CatPhotoApp</h1>
  <p>This is a paragraph element.</p>
</body>    
```

:::

- **`section` 元素**：此元素用於將內容劃分為較小的區段。

:::interactive_editor

```html
<section>
  <h2>About Me</h2>
  <p>Hi, I am Jane Doe and I am a web developer.</p>
</section>
```

:::

- **`div` 元素**：此元素是一個泛型的 HTML 元素，沒有任何語意意義。它用作泛型容器來容納其他 HTML 元素。

:::interactive_editor

```html
<div>
  <h1>I am a heading</h1>
  <p>I am a paragraph</p>
</div>
```

:::

- **錨點（`a`）元素**：這些元素用於在網頁上應用聯結。`href` 屬性用於指定當使用者點擊時聯結應該導向的位置。

:::interactive_editor

```html
<a href="https://cdn.freecodecamp.org/curriculum/cat-photo-app/running-cats.jpg">cute cats</a>
```

:::

- **無序（`ul`）和有序（`ol`）列表元素**：要創建一個項目的項目符號列表，你應該使用 `ul` 元素，並在其中巢狀一個或多個 `li` 元素，如下所示：

:::interactive_editor

```html
<ul>
  <li>catnip</li>
  <li>laser pointers</li>
  <li>lasagna</li>
</ul>
```

:::

要創建有序的項目列表，你應該使用 `ol` 元素：

:::interactive_editor

```html
<ol>
  <li>flea treatment</li>
  <li>thunder</li>
  <li>other cats</li>
</ol>
```

:::

- **強調 (`em`) 元素**：這用於對一段文字進行強調。

:::interactive_editor

```html
<p>Cats <em>love</em> lasagna.</p>  
```

:::

- **強調重要性 (`strong`) 元素**：此元素用於對文字施加強烈強調，以顯示緊迫感和嚴肅性。

:::interactive_editor

```html
<p>
  <strong>Important:</strong> Before proceeding, make sure to wear your safety goggles. 
</p>
```

:::

- **`figure` 和 `figcaption` 元素**：`figure` 元素用於將影像、圖片和圖表等內容進行群組。`figcaption` 元素用於表述 `figure` 元素內該內容的標題。

:::interactive_editor

```html
<figure>
  <img src="https://cdn.freecodecamp.org/curriculum/cat-photo-app/cats.jpg" alt="Two tabby kittens sleeping together on a couch.">
  <figcaption>Cats <strong>hate</strong> other cats.</figcaption>  
</figure>
```

:::

- **`main` 元素**：此元素用於表述網頁的主要內容。
- **`footer` 元素**：此元素置於 HTML 文件的底部，通常包含版權資訊及其他重要頁面連結。

:::interactive_editor

```html
<footer>
  <p>
    No Copyright - <a href="https://www.freecodecamp.org">freeCodeCamp.org</a>
  </p>
</footer>
```

:::

## 識別字與群組

- **IDs**：HTML 元素的唯一元素識別符。ID 名稱在每個 HTML 文件中應該只使用一次。

```html
<h1 id="title">Movie Review Page</h1>
```

ID 名稱不能有空格。如果你的 ID 名稱包含多個字，則你可以在字與字之間使用連字號，如下所示：

```html
<div id="red-box"></div>
```

- **類別**：類別用於將元素群組以便進行樣式和行為設定。

```html
<div class="box"></div>
```

與 ID 不同，你可以在整個 HTML 文件中重複使用相同的類別名稱。`class` 值也可以像這樣包含空格：

```html
<div class="box red-box"></div>
<div class="box blue-box"></div>
```

## 特殊字元和聯結

- **HTML 物體**：HTML 物體，或稱字元引用，是用來表述 HTML 中保留字元的一組字元。範例包括和號 (`&amp;`) 符號以及小於號 (`&lt;`) 符號。

```html
<p>This is an &lt;img /&gt; element</p>
```

- **`link` 元素**：此元素用於連結到外部資源，如樣式表和網站圖示。以下是使用 `link` 元素連結外部 CSS 檔案的基本語法：

```html
<link rel="stylesheet" href="./styles.css" />
```

`rel` 屬性用於指定連結資源與 HTML 文件之間的關係。`href` 屬性用於指定外部資源的 URL 位置。

- **`script` 元素**：此元素用於嵌入可執行的程式碼。

```html
<body>
  <script>
    alert("Welcome to freeCodeCamp");
  </script>
</body>
```

雖然你技術上可以將所有 JavaScript 程式碼寫在 `script` 標記內，但最佳實務是將其聯結到外部 JavaScript 檔案。以下是使用 `script` 元素聯結到外部 JavaScript 檔案的範例：

```html
<script src="path-to-javascript-file.js"></script>
```

這裡使用 `src` 屬性來指定該外部 JavaScript 檔案的位置。

## 樣板程式碼和編碼

- **HTML boilerplate**：這是一個包含每個 HTML 文件所需基本結構和必要元素的樣板。

:::interactive_editor

```html
<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="utf-8" />
    <title>freeCodeCamp</title>
    <link rel="stylesheet" href="./styles.css" />
  </head>
  <body>
    <!--Headings, paragraphs, images, etc. go inside here-->
  </body>
</html>
```

:::

- **`DOCTYPE`**：這用來告訴瀏覽器你正在使用哪個版本的 HTML。
- **`html` 元素**：這代表 HTML 文件的最高層級元素或根元素。若要指定文件的語言，應該使用 `lang` 屬性。
- **`head` 元素**：`head` 區段包含重要的後台資訊，是瀏覽器和搜尋引擎所需的元資料。
- **`meta` Elements**：這些元素表述你網站的詮釋資料。這些元素包含關於字元編碼，以及像 Twitter 這類網站應如何預覽你頁面聯結等詳細資訊。
- **`title` 元素**：此元素用於設定出現在瀏覽器分頁或視窗中的文字。
- **UTF-8 字元編碼**：UTF-8，或稱 UCS 轉換格式 8，是網路上廣泛使用的標準化字元編碼。字元編碼是電腦用來將字元儲存為資料的方法。`charset` 屬性用於 `meta` 元素內，以將字元編碼設定為 UTF-8。

## SEO 和社群分享

- **SEO**：搜尋引擎最佳化是一種最佳化網頁，使其在搜尋引擎上更容易被看見並獲得更高等級的做法。
- **Meta (`description`) 元素**：用於提供網頁的簡短描述並影響 SEO。

```html
<meta
  name="description"
  content="Discover expert tips and techniques for gardening in small spaces, choosing the right plants, and maintaining a thriving garden."
/>
```

- **Open Graph 頁籤**：Open Graph 協定讓你能控制網站內容在各種社群媒體平台（例如 Facebook、LinkedIn 等）上的顯示方式。  
透過設定這些 Open Graph 屬性，你可以吸引使用者點擊並與你的內容互動。  
你可以透過 HTML `head` 區段內的一組 `meta` 元素來設定這些屬性。
- **`og:title` 屬性**：這用於設定在社群媒體貼文中顯示的標題。

```html
<meta content="freeCodeCamp.org" property="og:title" />
```

- **`og:type` 屬性**：`type` 屬性用於表述在社群媒體上分享的內容類型。此類內容的範例包括文章、網站、影片或音樂。

```html
<meta property="og:type" content="website" />
``` 

- **`og:image` 屬性**：這用於設定社群媒體貼文所顯示的影像。

```html
<meta
  content="https://cdn.freecodecamp.org/platform/universal/fcc_meta_1920X1080-indigo.png"
  property="og:image"
/>
```

- **`og:url` 屬性**：這用於設定使用者點擊社群媒體貼文時的 URL。

```html
<meta property="og:url" content="https://www.freecodecamp.org" />
```

## 媒體元素與優化

- **取代元素**：取代元素是指其內容由外部資源決定，而非由 CSS 本身決定的元素。範例為 `iframe` 元素。`iframe` 代表行內訊框。它是一個行內元素，用於直接在 HTML 頁面中嵌入其他 HTML 內容。

```html
<iframe src="https://www.example.com" title="Example Site"></iframe>
```

你可以包含 `allowfullscreen` 屬性，允許使用者將 iframe 顯示為全螢幕模式。 

```html
<iframe
  src="video-url"
  width="width-value"
  height="height-value"
  allowfullscreen
></iframe>
```

要在 `iframe` 中嵌入影片，你可以直接從像 YouTube 和 Vimeo 這類熱門影片服務複製，或是自行定義 `src` 屬性指向該影片的 URL。以下是一個從 YouTube 嵌入熱門 freeCodeCamp 課程的範例：

:::interactive_editor

```html
<h1>A freeCodeCamp YouTube Video Embedded with the iframe Element</h1>

<iframe
  width="560"
  height="315"
  src="https://www.youtube.com/embed/PkZNo7MFNFg?si=-UBVIUNM3csdeiWF"
  title="YouTube video player"
  allow="accelerometer; autoplay; clipboard-write; encrypted-media; gyroscope; picture-in-picture; web-share"
  referrerpolicy="strict-origin-when-cross-origin"
  allowfullscreen
></iframe>
```

:::

還有一些其他的取代元素，例如 `video` 和 `embed`。而且某些元素在特定情況下會表現為取代元素。以下是一個 `input` 元素，其 `type` 屬性設定為 `image`：

```html
<input type="image" alt="Descriptive text goes here" src="example-img-url">
```

- **優化媒體**：在你的網頁中使用媒體（例如影像）時，有三個工具需要考慮：大小、格式和壓縮。壓縮演算法用於減少檔案或資料的大小。
- **影像格式**：兩種最常見的檔案格式是 PNG 和 JPG，但這些已不再是提供影像的最佳格式。除非你需要支援較舊的瀏覽器，否則你應該考慮使用更最佳化的格式，例如 WEBP 或 AVIF。
- **影像授權**：屬於公有領域的影像沒有版權限制，可以自由使用且沒有任何限制。特別依據 Creative Commons 0 授權的影像被視為公有領域。有些影像可能會以寬鬆授權方式釋出，例如 Creative Commons 授權，或 freeCodeCamp 使用的 BSD 授權。
- **SVGs**：可縮放向量圖形根據路徑和方程式追蹤資料以繪製點、線和曲線。這真正的意思是，像 SVG 這樣的向量圖形可以縮放到任何大小而不影響品質。

## 多媒體整合

- **`audio` 和 `video` 元素**：`audio` 和 `video` 元素允許你為你的 HTML 文件添加音訊和影片內容。`audio` 元素支援像是 mp3、wav 和 ogg 等流行的音訊格式。`video` 元素支援 mp4、ogg 和 webm 格式。

```html
<audio src="CrystalizeThatInnerChild.mp3"></audio>
```

如果你想在頁面上看到音訊播放器，則可以將帶有 `controls` 屬性的 `audio` 元素添加進去：

:::interactive_editor

```html
<audio src="CrystalizeThatInnerChild.mp3" controls></audio>
```

:::

`controls` 屬性啟用使用者管理音訊播放，包括調整音量、暫停或繼續播放。`controls` 屬性是一個布林值屬性，可以添加到元素以啟用內建播放控制元件。如果省略，將不會顯示任何控制元件。

- **`loop` 屬性**：`loop` 屬性是一個布林值屬性，可讓音訊持續重播。

:::interactive_editor

```html
<audio
  src="https://cdn.freecodecamp.org/curriculum/js-music-player/can't-stay-down.mp3"
  loop
  controls
></audio>
```

:::

- **`muted` 屬性**：當出現在 `audio` 元素中時，`muted` 布林值屬性會讓音訊以靜音狀態開始。

:::interactive_editor

```html
<audio
  src="https://cdn.freecodecamp.org/curriculum/js-music-player/can't-stay-down.mp3"
  loop
  controls
  muted
></audio>
```

:::

- **`source` 元素**：當談到音訊檔案型別時，不同瀏覽器對於支援的型別有所差異。為了因應這點，你可以在 `audio` 元素內使用多個 `source` 元素，瀏覽器會選擇它能理解的第一個來源。以下是為 `audio` 元素使用多個 `source` 元素的範例：

```html
<audio controls>
  <source src="audio.ogg" type="audio/ogg" />
  <source src="audio.wav" type="audio/wav" />
  <source src="audio.mp3" type="audio/mpeg" />
</audio>
```

到目前為止我們所學的所有屬性也都支援在 `video` 元素中。以下是一個使用 `video` 元素並帶有 `loop`、`controls` 和 `muted` 屬性的範例：

:::interactive_editor

```html
<video
  src="https://archive.org/download/BigBuckBunny_124/Content/big_buck_bunny_720p_surround.mp4"
  loop
  controls
  muted
></video>
```

:::

- **`poster` 屬性**：如果你想在影片載入時顯示影像，可以使用 `poster` 屬性。此屬性不適用於 `audio` 元素，且僅限於 `video` 元素。

:::interactive_editor

```html
<video
  src="https://archive.org/download/BigBuckBunny_124/Content/big_buck_bunny_720p_surround.mp4"
  loop
  controls
  muted
  poster="https://peach.blender.org/wp-content/uploads/title_anouncement.jpg?x11217"
  width="620"
></video>
```

:::

## 標的屬性型別

- **`target` 屬性**：此屬性告訴瀏覽器在哪裡開啟錨點元素的 URL。此屬性有四個重要的可能值：`_self`、`_blank`、`_parent` 和 `_top`。還有第五個值，稱為 `_unfencedTop`，目前用於實驗性的 `FencedFrame` API。你可能還沒有理由使用這個值。
- **`_self` 值**：這是 `target` 屬性的預設值。這會在目前的瀏覽背景關係中開啟連結。在大多數情況下，這將是目前的分頁或視窗。

```html
<a href="https://freecodecamp.org" target="_self">Visit freeCodeCamp</a>
```

- **`_blank` 值**：這會在新的瀏覽背景關係中開啟連結。通常，這會在新的分頁中開啟。但有些使用者可能會組態他們的瀏覽器改為開啟新的視窗。

```html
<a href="https://freecodecamp.org" target="_blank">Visit freeCodeCamp</a>
```

- **`_parent` 值**：這會在目前背景關係的父層中開啟聯結。 例如，如果你的網站有一個 iframe，該 iframe 中的 `_parent` 值會在你網站的分頁/視窗中開啟，而不是在嵌入的訊框中。

```html
<a href="https://freecodecamp.org" target="_parent">Visit freeCodeCamp</a>
```

- **`_top` 值**：這會在最頂層的瀏覽背景關係中開啟聯結－想像成「父層的父層」。這與 `_parent` 類似，但聯結將始終在完整的瀏覽器分頁／視窗中開啟，即使是巢狀的嵌入式訊框。

```html
<a href="https://freecodecamp.org" target="_top">Visit freeCodeCamp</a>
```

## 絕對路徑與相對路徑

- **路徑定義**：路徑是指定檔案或目錄在檔案系統中位置的字串。在網頁開發中，路徑讓開發者能夠連結到像是影像、樣式表、腳本和其他網頁等資源。
- **路徑語法**：有三個關鍵語法需要知道。第一個是斜線，根據你的作業系統，可以是反斜線（`\`）或正斜線（`/`）。第二個是單點（`.`）。最後是雙點（`..`）。斜線被稱為「路徑分隔符」。它用來顯示目錄或檔案名稱之間的文字斷點。單點指向當前目錄，雙點指向上層目錄。

```md
public/index.html
./favicon.ico
../src/index.css
```

- **絕對路徑**：絕對路徑是指向資源的完整連結。它從根目錄開始，包含每個其他目錄，最後是檔名和擴充套件。「根目錄」指的是階層體系中的頂層目錄或資料夾。絕對路徑也包含協定－可能是 `http`、`https` 和 `file`，以及如果資源位於網路上則包含網域名稱。以下是一個指向 freeCodeCamp 標誌的絕對路徑範例：

:::interactive_editor

```html
<a href="https://design-style-guide.freecodecamp.org/img/fcc_secondary_small.svg">
  View fCC Logo
</a>
```

:::

- **相對路徑**：相對路徑指定檔案相對於目前檔案所在目錄的位置。它不包含協定或網域名稱，使其在同一網站內部聯結時更短且更具彈性。以下是從同一資料夾中的 `contact.html` 頁面連結到 `about.html` 頁面的範例：

```html
<p>
  Read more on the
  <a href="about.html">About Page</a>
</p>
```

## 聯結狀態

- **`:link`**：這是預設狀態。此狀態表述使用者尚未造訪、點擊或互動過的聯結。你可以將此狀態視為為你頁面上所有聯結提供基礎樣式。其他狀態則是在此基礎上構築。
- **`:visited`**：當使用者已經造訪過被連結的頁面時，會套用此樣式。預設值會將連結變成紫色－但你可以利用 CSS 為使用者提供不同的視覺提示。
- **`:hover`**：當使用者將游標暫留在聯結上時，會套用此狀態。此狀態有助於為聯結提供額外的注意力，以確保使用者確實打算點擊它。
- **`:focus`**：當我們聚焦於一個聯結時，會套用此狀態。
- **`:active`**：此狀態適用於使用者正在作用中的連結。這通常表示在大多數情況下，使用滑鼠左鍵點擊該連結。
