# --description--

Das FCC-Team hatte eine Stimmungsschwankung und möchte jetzt zwei Teesorten: grünen Tee und schwarzen Tee. Allgemeine Tatsache: Stimmungsschwankungen bei Clients sind ziemlich häufig.

Mit diesen Informationen müssen wir die `getTea`-Funktion aus der letzten Aufgabe noch einmal ansehen, um verschiedene Tee-Anfragen zu bearbeiten. Wir können `getTea` so ändern, dass sie eine Funktion als Parameter akzeptiert, um die Art des zubereiteten Tees ändern zu können. Das macht `getTea` flexibler und gibt der Programmierkraft mehr Kontrolle, wenn sich die Client-Anfragen ändern.

Aber zuerst behandeln wir einige Begriffe der funktionalen Programmierung:

<dfn>Callbacks</dfn> sind die Funktionen, die in eine andere Funktion eingeschoben oder übergeben werden, um den Aufruf dieser Funktion zu steuern. Sie haben sie vielleicht schon in anderen Methoden gesehen, zum Beispiel in `filter`, wo die Callback-Funktion JavaScript die Kriterien vorgibt, wie ein Array gefiltert wird.

Funktionen, die einer Variablen zugewiesen, in eine andere Funktion übergeben oder von einer anderen Funktion zurückgegeben werden können wie jeder andere normale Wert, nennt man <dfn>first class</dfn>-Funktionen. In JavaScript sind alle Funktionen first class-Funktionen.

Funktionen, die eine Funktion als Argument annehmen oder eine Funktion als Rückgabewert zurückgeben, nennt man <dfn>higher order</dfn>-Funktionen.

Wenn Funktionen in eine andere Funktion übergeben oder von ihr zurückgegeben werden, können diese übergebenen oder zurückgegebenen Funktionen als <dfn>Lambda</dfn> bezeichnet werden.
