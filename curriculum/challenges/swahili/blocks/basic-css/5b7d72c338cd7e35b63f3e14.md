# --description--

Unapofanya kazi na CSS, kuna uwezekano wa kukutana na matatizo ya ulinganifu wa vivinjari wakati fulani. Hii ndiyo sababu ni muhimu kutoa njia mbadala za vivinjari ili kuepuka matatizo yanayoweza kutokea.

Wakati kivinjari chako kinapochanganuzi CSS ya ukurasa wa mtandao, hukubali sifa zozote ambazo hakizitambui au hazikuungi mkono. Kwa mfano, ukitumia variable ya CSS kuweka rangi ya nyuma kwenye tovuti, Internet Explorer itapuuzia rangi ya nyuma kwa sababu haikuungi mkono variable za CSS. Katika hali hiyo, kivinjari kitaitumia thamani yoyote iliyo kwa sifa hiyo. Ikiwa hakiwezi kupata thamani nyingine yoyote iliyowekwa kwa sifa hiyo, kitarejea kwa thamani ya msingi, ambayo kwa kawaida si nzuri.

Hii inamaanisha kwamba ikiwa unataka kutoa njia mbadala ya kivinjari, ni rahisi kama kutoa thamani nyingine inayoungwa mkono zaidi mara moja kabla ya tamko lako lililopo. Kwa njia hiyo kivinjari cha zamani kitakuwa na kitu cha kutumia kama njia mbadala, wakati kivinjari kipya kitatafsiri tu tamko lolote linalokuja baadaye katika mfululizo.

# --instructions--

Inaonekana variable inatumiwa kuweka rangi ya nyuma ya darasa la `.red-box`. Hebu boresha ulinganifu wetu wa kivinjari kwa kuongeza tamko lingine la `background` mara moja kabla ya tamko lililopo na weka thamani yake kuwa `red`.

# --hints--

Sera yako ya `.red-box` inapaswa kujumuisha njia mbadala yenye `background` iliyowekwa kuwa `red` mara moja kabla ya tamko lililopo la `background`.

```js
const spacelessCode = __helpers.removeWhiteSpace(__helpers.removeCssComments(code));
assert.match(
  spacelessCode,
  /\.red-box{background:(red|#ff0000|#f00|rgb\(255,0,0\)|rgb\(100%,0%,0%\)|hsl\(0,100%,50%\));background:var\(--red-color\);height:200px;width:200px;}/gi
);
```
