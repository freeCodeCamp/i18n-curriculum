# --interactive--

## Nozioni di base sugli oggetti

- **Definizione**: Un oggetto è una struttura dati composta da proprietà. Una proprietà consiste in una chiave e un valore. Per consultare i dati di un oggetto puoi usare la notazione a punto oppure la notazione a parentesi.

:::interactive_editor

```js
const person = {
  name: "Alice",
  age: 30,
  city: "New York"
};

console.log(person.name);  // Alice
console.log(person["name"]); // Alice
```

:::

Per impostare una proprietà di un oggetto esistente puoi usare la notazione a punto oppure la notazione a parentesi insieme all’operatore di assegnazione.

:::interactive_editor

```js
const person = {
  name: "Alice",
  age: 30
};

person.job = "Engineer"
person["hobby"] = "Knitting"
console.log(person);  // {name: 'Alice', age: 30, job: 'Engineer', hobby: 'Knitting'}
```

:::

## Rimuovere proprietà da un oggetto

- **Operatore `delete`**: Questo operatore serve a rimuovere una proprietà da un oggetto.

:::interactive_editor

```js
const person = {
  name: "Alice",
  age: 30,
  job: "Engineer"
};

delete person.job;

console.log(person.job); // undefined
```

:::

## Verificare se un oggetto ha una proprietà

- **Metodo `hasOwnProperty()`**: Questo metodo restituisce un booleano che indica se l’oggetto possiede la proprietà specificata come propria proprietà.

:::interactive_editor

```js
const person = {
  name: "Alice",
  age: 30
};

console.log(person.hasOwnProperty("name")); // true
console.log(person.hasOwnProperty("job")); // false
```

:::

- **Metodo `Object.hasOwn()`**: Questo è il modo moderno e consigliato per verificare se un oggetto ha una proprietà come propria (non ereditata). La sintassi è `Object.hasOwn(object, propertyName)`. Restituisce `true` se la proprietà esiste sull’oggetto e `false` se non esiste — indipendentemente dal valore della proprietà. Questo lo rende più sicuro di `hasOwnProperty()` e più affidabile di `if (obj.prop)` quando i valori possono essere `0`, `false`, `null` o `undefined`.

:::interactive_editor

```js
const person = {
  name: "Alice",
  age: 30
};

console.log(Object.hasOwn(person, "name")); // true
console.log(Object.hasOwn(person, "job")); // false
```

:::

Ecco un esempio più dettagliato che mostra perché `Object.hasOwn()` è più affidabile rispetto a controllare direttamente il valore:

:::interactive_editor

```js
const settings = {
  darkMode: false,
  fontSize: 0,
  language: null
};

// Object.hasOwn() correctly sees these properties exist
console.log(Object.hasOwn(settings, "darkMode")); // true (value is false, but exists)
console.log(Object.hasOwn(settings, "fontSize")); // true (value is 0, but exists)
console.log(Object.hasOwn(settings, "theme"));    // false (property was never added)

// Using if() directly is unsafe for falsy values!
if (settings.darkMode) {
  console.log("Dark mode on"); // Does NOT print — misleading!
}

// Object.hasOwn() is the safe way
if (Object.hasOwn(settings, "darkMode")) {
  console.log("darkMode exists, value is:", settings.darkMode); // darkMode exists, value is: false
}
```

:::

- **Operatore `in`**: Questo operatore restituisce `true` se la proprietà esiste nell’oggetto.

:::interactive_editor

```js
const person = {
  name: "Bob",
  age: 25
};

console.log("name" in person);  // true
```

:::

## Consultare proprietà da oggetti annidati

- **Consultare dati**: Consultare proprietà da oggetti annidati implica usare la notazione a punto o la notazione a parentesi, proprio come per gli oggetti semplici. Tuttavia, devi concatenare questi accessi per scendere nella struttura annidata.

:::interactive_editor

```js
const person = {
  name: "Alice",
  age: 30,
  contact: {
    email: "alice@example.com",
    phone: {
      home: "123-456-7890",
      work: "098-765-4321"
    }
  }
};

console.log(person.contact.phone.work); // "098-765-4321"
```

:::

## Tipi di dati primitivi e non primitivi

- **Tipi di dati primitivi**: Questi tipi di dati includono numeri, stringhe, booleani, `null`, `undefined` e simboli. Questi tipi si chiamano "primitivi" perché rappresentano valori singoli e non sono oggetti. I valori primitivi sono immutabili, cioè una volta creati il loro valore non può essere modificato.
- **Tipi di dati non primitivi**: In JavaScript, questi sono oggetti, che includono oggetti regolari, array e funzioni. A differenza dei primitivi, i tipi non primitivi possono contenere valori multipli come proprietà o elementi.

## Metodi degli oggetti

- **Definizione**: I metodi degli oggetti sono funzioni associate a un oggetto. Sono definiti come proprietà di un oggetto e possono consultare e manipolare i dati dell’oggetto. La parola chiave `this` all’interno del metodo si riferisce all’oggetto stesso, permettendo di accedere alle sue proprietà.

:::interactive_editor

```js
const person = {
  name: "Bob",
  age: 30,
  sayHello: function() {
    return "Hello, my name is " + this.name;
  }
};

console.log(person.sayHello()); // "Hello, my name is Bob"
```

:::

## Costruttore di oggetti

- **Definizione**: In JavaScript, un costruttore è un tipo speciale di funzione usata per creare e inizializzare oggetti. Viene invocato con la parola chiave `new` e può inizializzare proprietà e metodi sull’oggetto appena creato. Il costruttore `Object()` crea un nuovo oggetto vuoto.

```js
new Object()
```

## Usare l’operatore di concatenamento opzionale (`?.`)

- **Definizione**: Questo operatore ti permette di consultare in sicurezza proprietà di un oggetto o chiamare metodi senza preoccuparti se esistono o meno.

:::interactive_editor

```ts
const user = {
  name: "John",
  profile: {
    email: "john@example.com",
    address: {
      street: "123 Main St",
      city: "Somewhere"
    }
  }
};

console.log(user.profile?.address?.street); // "123 Main St"
console.log(user.profile?.phone?.number);   // undefined
```

:::

## Destrutturazione degli oggetti

- **Definizione**: La destrutturazione degli oggetti ti permette di estrarre valori dagli oggetti e assegnarli a variabili in modo più conciso e leggibile.

:::interactive_editor

```js
const person = { name: "Alice", age: 30, city: "New York" };

const { name, age } = person;

console.log(name); // Alice
console.log(age);  // 30
```

:::

## Lavorare con JSON

- **Definizione**: JSON sta per JavaScript Object Notation. È un formato di dati leggero basato su testo, comunemente usato per scambiare dati tra un server e un’applicazione web.

```js
{
  "name": "Alice",
  "age": 30,
  "isStudent": false,
  "list of courses": ["Mathematics", "Physics", "Computer Science"]
}
```

- **Metodo `JSON.stringify()`**: Questo metodo serve a convertire un oggetto JavaScript in una stringa JSON. È utile quando vuoi archiviare o trasmettere dati in un formato facilmente condivisibile o trasferibile tra sistemi.

:::interactive_editor

```js
const user = {
  name: "John",
  age: 30,
  isAdmin: true
};

const jsonString = JSON.stringify(user);
console.log(jsonString); // '{"name":"John","age":30,"isAdmin":true}'
```

:::

- **Metodo `JSON.parse()`**: Questo metodo converte una stringa JSON di nuovo in un oggetto JavaScript. È utile quando recuperi dati JSON da un server web o da localStorage e devi manipolare i dati nella tua applicazione.

:::interactive_editor

```js
const jsonString = '{"name":"John","age":30,"isAdmin":true}';
const userObject = JSON.parse(jsonString);

// result: { name: 'John', age: 30, isAdmin: true }
console.log(userObject);
```

:::
