# --description--

React에서는 인라인 스타일을 사용해 별도의 CSS 파일에 정의하지 않고 JSX 코드 내 React 요소에 직접 CSS 스타일을 적용합니다.

React의 인라인 스타일 접근법은 전통적인 CSS 문자열 대신 JavaScript 객체를 사용해 스타일을 정의하는 것입니다. 즉, CSS 파일에서 스타일을 작성하는 대신, CSS 속성 이름을 카멜 케이스로 변환한 키와 CSS 값 문자열을 값으로 하는 JavaScript 객체를 만듭니다.

다음은 `Button` 컴포넌트에 인라인 스타일을 사용하는 예시입니다:

```jsx
function Button({ buttonText }) {
  const defaultStyles = {
    backgroundColor: "#007BFF",
    color: "white",
    border: "none",
    borderRadius: "4px",
    padding: "10px 20px",
    fontSize: "16px",
    fontWeight: "bold",
    cursor: "pointer",
    transition: "background-color 0.3s ease",
  };

  return <button style={defaultStyles}>{buttonText}</button>;
}
```

이 예시에서는 `defaultStyles`라는 스타일 객체를 정의했습니다. 그런 다음 `style` 속성을 사용해 버튼 요소에 이 스타일을 적용합니다. React는 렌더링할 때 이 스타일을 요소에 적용합니다.

또한 `style` 속성에 객체를 직접 전달할 수도 있습니다. 수정된 예시는 다음과 같습니다:

```jsx
function Button({ buttonText }) {
  return (
    <button
      style={{
        backgroundColor: "#007BFF",
        color: "white",
      }}
    >
      {buttonText}
    </button>
  );
}
```

`{{}}` 속성에 중괄호가 두 번 `style` 사용된 점을 주목하세요. 바깥 중괄호는 JSX 내 JavaScript 표현식을 나타내고, 안쪽 중괄호는 JavaScript 객체 리터럴을 정의합니다. 이 구문 덕분에 JavaScript 객체를 JSX 속성에 직접 임베드할 수 있습니다.

속성이 몇 개뿐이라면 직접 객체를 전달하는 것이 좋을 때도 있습니다. 그렇지 않으면 첫 번째 예시처럼 이름을 가진 객체를 전달하는 편이 낫습니다.

CSS 속성 이름은 보통 케밥 케이스(예: `font-size`)로 작성되지만, React 인라인 스타일에서는 카멜 케이스(예: `fontSize`)를 사용한다는 점이 중요합니다. 이는 스타일 객체가 JavaScript 객체이기 때문이며, 케밥 케이스 이름은 따옴표 없이 JavaScript 객체 키로 사용할 수 없습니다.

React 인라인 스타일의 큰 장점 중 하나는 컴포넌트 상태나 프로프스에 따라 동적으로 스타일을 적용할 수 있다는 점입니다. 예를 들어:

```jsx
function DynamicButton({ isActive }) {
  const buttonStyles = {
    backgroundColor: isActive ? "green" : "red",
    color: "white",
    padding: "10px 15px",
    border: "none",
    cursor: "pointer",
  };

  return <button style={buttonStyles}>Login</button>;
}
```

이 예시에서 버튼의 배경색은 `isActive` 프로프스에 따라 바뀝니다. 이런 동적 스타일링은 인터랙티브하고 반응형 사용자 인터페이스를 만드는 데 매우 유용합니다.

요약하자면, React의 인라인 스타일은 컴포넌트 내에서 스타일을 직접 적용하고 조작하는 강력한 방법입니다. CSS 문자열 대신 JavaScript 객체를 사용하고, 카멜 케이스 속성 이름을 요구하며, 동적 값을 쉽게 포함할 수 있습니다. 특히 고도로 맞춤화되고 인터랙티브한 UI를 만들 때 React 개발자에게 필수적인 도구입니다.
