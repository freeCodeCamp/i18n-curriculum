# --description--

Sie können sich Elemente als Container für Inhalte vorstellen. Die öffnenden und schließenden Tags sagen dem Browser, welche Inhalte das Element enthält. Der Browser kann diese Informationen dann nutzen, um zu bestimmen, wie er die Inhalte interpretieren und formatieren soll.

Es gibt einige HTML-Elemente, die kein schließendes Tag haben. Diese Elemente sehen oft so aus: `<br />` oder `<img/>`, aber einige können auch ohne den schließenden Schrägstrich verwendet werden, wie `<br>` oder `<img>`. Diese werden als Void-Elemente oder leere Elemente bezeichnet, weil sie keine Inhalte umschließen. Sie werden in späteren Lektionen auf einige davon stoßen, aber größtenteils haben Elemente sowohl öffnende als auch schließende Tags.

HTML verfügt über eine umfangreiche Liste vordefinierter Tags, mit denen Sie alle möglichen verschiedenen Elemente erstellen können. Es ist wichtig, die richtigen Tags für Inhalte zu verwenden. Die Verwendung der richtigen Tags kann sich stark auf zwei Aspekte Ihrer Websites auswirken: wie sie in Suchmaschinen eingestuft werden und wie zugänglich sie für Nutzer sind, die auf unterstützende Technologien wie Screenreader angewiesen sind, um das Web zu nutzen.

Die Verwendung der richtigen Elemente für Inhalte nennt man semantisches HTML. Dies werden Sie später im Curriculum noch viel ausführlicher erkunden.

# --questions--
    
## --text--

Warum sind Void-Elemente wie `<br/>` oder `<img>` in HTML so wichtig?

## --answers--

Sie verhindern Formatierungsprobleme in HTML-Dokumenten.

---

Sie kennzeichnen leere Elemente und benötigen kein schließendes Tag.

---

Sie verbessern die Website-Leistung, indem sie die Code-Komplexität reduzieren.
