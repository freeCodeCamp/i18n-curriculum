# --description--
  
Динамічне програмування — це алгоритмічна техніка, яка розв’язує складні задачі, розбиваючи їх на простіші підзадачі та зберігаючи результати, щоб уникнути повторних обчислень. Такий підхід перетворює задачі, які зазвичай потребують експоненційного часу, на ті, що можна розв’язати за поліноміальний час.

## Основні принципи динамічного програмування

Динамічне програмування працює, коли в задачі присутні дві ключові умови.

- **Перекривні підзадачі**: одні й ті самі менші задачі з’являються багаторазово під час розв’язання більшої задачі. Замість того, щоб повторно обчислювати ці підзадачі, ми зберігаємо їхні розв’язки.
    
- **Оптимальна підструктура**: оптимальний розв’язок задачі містить оптимальні розв’язки її підзадач. Це означає, що ми можемо побудувати найкращий розв’язок, комбінуючи найкращі розв’язки менших частин.
    

Розглянемо ці концепції на класичній задачі «підйом по сходах».

## Проблема наївної рекурсії

Розглянемо задачу підйому по сходах: ви піднімаєтеся сходами з `n` сходинок і можете підніматися на 1 або 2 сходинки за раз. Скільки різних способів дістатися до вершини?

```python
def climb_stairs_recursive(n):
    """Recursive approach"""
    if n <= 2:
        return n  # Base cases: 1 way for 1 step, 2 ways for 2 steps
    # To reach step n, we can come from step (n-1) or step (n-2)
    return climb_stairs_recursive(n-1) + climb_stairs_recursive(n-2)
```

Ця реалізація має експоненційну часову складність через величезну кількість повторних обчислень. При обчисленні `climb_stairs(5)` відбувається таке:

- `climb_stairs(5)` викликає `climb_stairs(4)` і `climb_stairs(3)`
    
- `climb_stairs(4)` викликає `climb_stairs(3)` і `climb_stairs(2)`
    
- `climb_stairs(3)` обчислюється **двічі**
    
- `climb_stairs(3)` викликає `climb_stairs(2)` і `climb_stairs(1)`
    
- `climb_stairs(2)` обчислюється **3 рази загалом**
    

Для `n=5` ми робимо 9 викликів функції, хоча потрібно лише 5 унікальних обчислень. Зі збільшенням `n` ця надмірність зростає експоненційно — `climb_stairs(30)` вимагатиме кілька мільйонів викликів функції. Часова складність стає `O(2^n)`, що робить цей підхід неефективним і непридатним для більших значень `n`.

## Рішення за допомогою динамічного програмування

Динамічне програмування усуває цю надмірність обчислень двома основними підходами:

### Мемоізація (підхід зверху вниз)

Мемоізація зберігає результати дорогих викликів функцій і повертає кешований результат, коли ті самі вхідні дані з’являються знову:

```python
def climb_stairs_memo(n, memo={}):
    """Dynamic programming with memoization"""
    # Check if we've already calculated this value
    if n in memo:
        return memo[n]  # Return cached result - O(1) lookup!
    
    # Base cases
    if n <= 2:
        return n
    
    # Calculate once and store in memo for future use
    memo[n] = climb_stairs_memo(n-1, memo) + climb_stairs_memo(n-2, memo)
    return memo[n]
```

Мемоізація значно ефективніша, бо кожне унікальне значення від `1` до `n` обчислюється рівно один раз. Коли нам знову потрібен `climb_stairs(3)`, замість повторного обчислення (що викликало б додаткові рекурсивні виклики) ми просто шукаємо його в нашому словнику memo за `O(1)` часу.

Пройдемося по виконанню `climb_stairs(5)` з підходом зверху вниз, щоб побачити, як мемоізація усуває надмірну роботу:

```md
Call: climb_stairs_memo(5)
  memo = {} (empty)
  
  Call: climb_stairs_memo(4) 
    memo = {} (empty)
    
    Call: climb_stairs_memo(3)
      memo = {} (empty)
      
      Call: climb_stairs_memo(2) → returns 2 (base case)
      Call: climb_stairs_memo(1) → returns 1 (base case)
      
      Result: 2 + 1 = 3
      memo = {3: 3} (stored!)
    
    Call: climb_stairs_memo(2) → returns 2 (base case)
    
    Result: 3 + 2 = 5
    memo = {3: 3, 4: 5} (stored!)
  
  Call: climb_stairs_memo(3) → returns 3 (FROM MEMO - no recursion!)
  
  Result: 5 + 3 = 8
  memo = {3: 3, 4: 5, 5: 8}
```

**Порівняння ефективності**

- **Наївна рекурсія**: 9 викликів функції з повторними обчисленнями
    
- **Мемоізація**: лише 5 унікальних обчислень, потім повторне використання збережених результатів
    
- **Часова складність**: скорочена з `O(2^n)` до `O(n)`, бо робиться лише `n` унікальних обчислень
    
- **Просторова складність**: `O(n)` для зберігання memo і стеку викликів
    
- **Реальний вплив**: `climb_stairs(30)` з мільйонів викликів функції зменшується до близько 30 унікальних підзадач
    

### Табуляція (підхід знизу вгору)

Табуляція будує розв’язок знизу вгору, заповнюючи таблицю розв’язками підзадач:

```python
def climb_stairs_tabulation(n):
    """Dynamic programming with tabulation"""
    if n <= 2:
        return n
    
    # Create array to store results for all steps from 0 to n
    dp = [0] * (n + 1)
    dp[1] = 1  # 1 way to reach step 1
    dp[2] = 2  # 2 ways to reach step 2
    
    # Build up the solution iteratively
    for i in range(3, n + 1):
        # Ways to reach step i = ways to reach (i-1) + ways to reach (i-2)
        dp[i] = dp[i-1] + dp[i-2]
    
    return dp[n]
```

Табуляція повністю усуває рекурсію, будуючи розв’язок ітеративно від найменших підзадач до цілі.

Погляньмо на підхід знизу вгору в дії, щоб побачити, як систематично будується розв’язок. Ось ітеративна побудова для `climb_stairs(5)`:

```md
Initial state:
dp = [0, 1, 2, 0, 0, 0]
     [0, 1, 2, 3, 4, 5] ← indices (step numbers)

Step by step construction:

i = 3:
  dp[3] = dp[2] + dp[1] = 2 + 1 = 3
  dp = [0, 1, 2, 3, 0, 0]
  
i = 4:
  dp[4] = dp[3] + dp[2] = 3 + 2 = 5
  dp = [0, 1, 2, 3, 5, 0]
  
i = 5:
  dp[5] = dp[4] + dp[3] = 5 + 3 = 8
  dp = [0, 1, 2, 3, 5, 8]

Final result: dp[5] = 8
```

**Ключові переваги табуляції**

- **Відсутність рекурсії**: на відміну від мемоізації, немає рекурсивного стеку викликів.
    
- **Передбачуване виконання**: обчислення значень у визначеному порядку (1, 2, 3, 4, 5...).
    
- **Оптимізація кешу**: послідовний доступ до масиву покращує використання пам’яті.
    
- **Легко оптимізувати**: можна скоротити просторову складність до `O(1)`, бо потрібні лише останні два значення.
    

```python
def climb_stairs_optimized(n):
    if n <= 2:
        return n
    
    prev2, prev1 = 1, 2  # Only store last two values
    for i in range(3, n + 1):
        current = prev1 + prev2
        prev2, prev1 = prev1, current
    return prev1
```

**Порівняння ефективності**

- **Наївна рекурсія**: 9 викликів функції для `n=5`, експоненційне зростання.
    
- **Табуляція**: 3 прості додавання для `n=5`, лінійне зростання.
    
- **Часова складність**: `O(n)` замість `O(2^n)`.
    
- **Просторова складність**: `O(n)` для масиву, або `O(1)` з оптимізацією.
    
- **Передбачувана продуктивність**: відсутність ризику переповнення стеку для великих вхідних даних.
    

Обидва підходи зменшують часову складність з експоненційної `O(2^n)` до лінійної `O(n)`, що є драматичним покращенням і визначає різницю між розв’язанням задачі за мілісекунди та очікуванням роками для більших вхідних даних.

## Реальні застосування

Динамічне програмування широко застосовується в інформатиці та інших сферах:

- **Оптимізація маршрутів**: GPS-системи використовують алгоритми динамічного програмування для пошуку найкоротших шляхів між локаціями.
    
- **Обробка тексту**: перевірка орфографії та функції автозаповнення часто покладаються на динамічне програмування для обчислення відстаней редагування між словами.
    
- **Фінансове моделювання**: інвестиційні стратегії та оптимізація портфоліо часто використовують техніки динамічного програмування.
    
- **Розподіл ресурсів**: задача рюкзака та її варіанти зустрічаються у плануванні, бюджетуванні та управлінні ресурсами.
    

## Практичний приклад: задача про зміну монет

Задача про зміну монет — класичне завдання програмування, яке при розв’язанні за допомогою динамічного програмування демонструє обидва ключові принципи ДП: оптимальну підструктуру та перекривні підзадачі.

Задача про зміну монет ставить питання: «Яка мінімальна кількість монет потрібна, щоб отримати цільову суму?»

Ось одне з рішень за допомогою динамічного програмування:

```python
def min_coins(amount, coins):
    """Find minimum number of coins needed to make the given amount"""
    # Initialize dp array with "infinity" - represents impossible to make
    dp = [float('inf')] * (amount + 1)
    dp[0] = 0  # Base case: 0 coins needed for amount 0
    
    # For each amount from 1 to target amount
    for i in range(1, amount + 1):
        # Try each coin denomination
        for coin in coins:
            if coin <= i:  # Can only use coin if it doesn't exceed current amount
                # Update minimum: current minimum vs (coins for remaining amount + 1)
                dp[i] = min(dp[i], dp[i - coin] + 1)
    
    # Return result if possible, -1 if impossible
    return dp[amount] if dp[amount] != float('inf') else -1

# Example usage:
# coins = [1, 3, 4], amount = 6
# dp[6] = min(dp[5]+1, dp[3]+1, dp[2]+1) = min(3+1, 1+1, 2+1) = 2
# Result: 2 coins (3 + 3)
```

А ось як алгоритм динамічного програмування для задачі про зміну монет працює крок за кроком для `coins = [1, 3, 4]`, `amount = 6`:

```md
Initial state:
dp = [0, ∞, ∞, ∞, ∞, ∞, ∞]
     [0, 1, 2, 3, 4, 5, 6] ← amounts

Building up the solution:

For amount = 1:
  Try coin 1: dp[1] = min(∞, dp[0] + 1) = min(∞, 0 + 1) = 1
  dp = [0, 1, ∞, ∞, ∞, ∞, ∞]

For amount = 2:
  Try coin 1: dp[2] = min(∞, dp[1] + 1) = min(∞, 1 + 1) = 2
  dp = [0, 1, 2, ∞, ∞, ∞, ∞]

For amount = 3:
  Try coin 1: dp[3] = min(∞, dp[2] + 1) = min(∞, 2 + 1) = 3
  Try coin 3: dp[3] = min(3, dp[0] + 1) = min(3, 0 + 1) = 1
  dp = [0, 1, 2, 1, ∞, ∞, ∞]

For amount = 4:
  Try coin 1: dp[4] = min(∞, dp[3] + 1) = min(∞, 1 + 1) = 2
  Try coin 3: dp[4] = min(2, dp[1] + 1) = min(2, 1 + 1) = 2
  Try coin 4: dp[4] = min(2, dp[0] + 1) = min(2, 0 + 1) = 1
  dp = [0, 1, 2, 1, 1, ∞, ∞]

For amount = 5:
  Try coin 1: dp[5] = min(∞, dp[4] + 1) = min(∞, 1 + 1) = 2
  Try coin 3: dp[5] = min(2, dp[2] + 1) = min(2, 2 + 1) = 2
  Try coin 4: dp[5] = min(2, dp[1] + 1) = min(2, 1 + 1) = 2
  dp = [0, 1, 2, 1, 1, 2, ∞]

For amount = 6:
  Try coin 1: dp[6] = min(∞, dp[5] + 1) = min(∞, 2 + 1) = 3
  Try coin 3: dp[6] = min(3, dp[3] + 1) = min(3, 1 + 1) = 2
  Try coin 4: dp[6] = min(2, dp[2] + 1) = min(2, 2 + 1) = 2
  dp = [0, 1, 2, 1, 1, 2, 2]

Final result: dp[6] = 2 (achieved with coins 3 + 3)
```

Це рішення демонструє обидва ключові принципи динамічного програмування. Тут є перекривні підзадачі, бо для знаходження мінімальної кількості монет для суми 6 потрібно знати розв’язки для сум 5, 3 і 2. Ці ж підзадачі з’являються при обчисленні інших сум. Є оптимальна підструктура, бо оптимальний розв’язок для будь-якої суми включає оптимальні розв’язки для менших сум. Якщо ми знаємо, що мінімальна кількість монет для суми 3 — це 1, то один зі способів отримати суму 6 — використати цей розв’язок плюс ще одну монету номіналом 3.

Без ДП довелося б перевіряти всі можливі комбінації монет — експоненційна кількість варіантів. З ДП ми систематично будуємо розв’язок:

- **Часова складність**: `O(amount × number of coins)` замість експоненційної.
    
- **Просторова складність**: `O(amount)` для масиву `dp`.
    
- **Відсутність надмірної роботи**: кожна підзадача (пошук мінімальної кількості монет для кожної суми) розв’язується рівно один раз.
    
- **Повторне використання результатів**: як тільки ми знаємо мінімальну кількість монет для суми 3, використовуємо це знання для всіх більших сум, які можуть від цього виграти.
    

## Коли використовувати динамічне програмування

Динамічне програмування ефективне, коли:

- Задачу можна розбити на перекривні підзадачі.
    
- Задача має оптимальну підструктуру.
    
- Наївне рекурсивне рішення передбачає повторні обчислення.
    
- Потрібно оптимізувати часову складність за рахунок просторової.
    

Поширені шаблони динамічного програмування включають задачі оптимізації (пошук мінімальних/максимальних значень), задачі підрахунку (кількість способів досягти чогось) та задачі прийняття рішень, які можна розбити на менші рішення.

Динамічне програмування перетворює складні задачі на керовані, систематично зберігаючи та повторно використовуючи розв’язки підзадач. Розуміння цієї техніки відкриває двері до ефективного розв’язання широкого спектра обчислювальних завдань.
