# --interactive--

L'attributo `id` aggiunge un identificatore unico a un elemento HTML.

Ecco un esempio di un elemento `h1` con un `id` di `title`.

Sotto l'elemento `h1`, aggiungi un elemento `h2` con un `id` impostato su `"subtitle"`. Puoi scrivere qualsiasi testo tu voglia per il `h2` e vedrai le modifiche nella finestra di anteprima. Per interagire con l'esempio, devi abilitare l'editor interattivo.

:::interactive_editor

```html
<h1 id="title">Movie Review Page</h1>

```

:::

Puoi fare riferimento al nome `id` di `title` all'interno del tuo JavaScript o CSS. Ecco un esempio CSS che fa riferimento al `id` di `title` per cambiare il `color` del testo in `red`.

**NOTA**: Alcuni CSS sono stati forniti per te in questo esempio interattivo. Non preoccuparti di cercare di capire il codice CSS perché ne imparerai di più nelle lezioni future. Ma se vuoi vedere il colore del testo cambiare in blu, abilita l'editor interattivo, clicca sulla scheda `styles.css` e cambia il `color: red;` in `color: blue;`.

:::interactive_editor

```html
<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="utf-8" />
    <meta
       name="viewport"
       content="width=device-width, initial-scale=1.0" />
    <title>Review page Example</title>
    <link rel="stylesheet" href="./styles.css" />
  </head>
  <body>
    <h1 id="title">Movie Review Page</h1>
  </body>
</html>
```

```css
#title {
  color: red;
}
```

:::

Il simbolo cancelletto (`#`) davanti a `title` indica al computer che vuoi selezionare un `id` con quel valore. I nomi `id` non dovrebbero essere usati più di una volta e devono essere sempre unici. Un'altra cosa da notare riguardo ai valori `id` è che non possono contenere spazi. Ecco un esempio di applicazione delle parole `main` e `heading` a un valore di attributo `id`:

```html
<h1 id="main heading">Main heading</h1>
```

I browser considereranno questo spazio come parte del `id`, il che porterà a problemi indesiderati quando si tratta di definire lo stile e scripting. I valori degli attributi `id` dovrebbero contenere solo lettere, cifre, underscore e trattini.

A differenza dell'attributo `id`, il valore dell'attributo `class` non deve essere unico e può contenere spazi.

Ecco un esempio di applicazione di una classe chiamata `box` a un elemento `div`.

```html
<div class="box"></div>
```

Se vuoi aggiungere più nomi di classi a un elemento, puoi farlo separando i nomi con uno spazio. Ecco un esempio aggiornato che applica più classi a un elemento `div`.

```html
<div class="box red-box"></div>
```

Ecco un altro esempio di applicazione di più classi a più elementi `div`.

**NOTA**: Alcuni CSS sono stati forniti per te in questo esempio interattivo. Non preoccuparti di cercare di capire il codice CSS perché ne imparerai di più nelle lezioni future. Ma se vuoi cambiare il colore del primo e del terzo riquadro, abilita l'editor interattivo, clicca sulla scheda `styles.css` e cambia il `background-color: red;` in `background-color: black;`.

:::interactive_editor

```html
<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="utf-8" />
    <meta
       name="viewport"
       content="width=device-width, initial-scale=1.0" />
    <title>Colored boxes example</title>
    <link rel="stylesheet" href="./styles.css" />
  </head>
  <body>
    <div class="box red-box"></div>
    <div class="box blue-box"></div>
    <div class="box red-box"></div>
    <div class="box blue-box"></div>
  </body>
</html>
```

```css
.box {
  width: 100px;
  height: 100px;
}

.red-box {
  background-color: red;
}

.blue-box {
  background-color: blue;
}
```

:::

Quindi, per ricapitolare, quando dovresti usare un `id` invece di un `class`? Le classi sono più adatte quando vuoi applicare un insieme di stili a molti elementi. Se vuoi selezionare un elemento specifico, è meglio usare `id` perché quei valori devono essere unici.

## --text--

Quando dovresti usare un `id` invece di un `class`?

## --answers--

Usa un `class` quando vuoi un identificatore unico che dovrebbe applicarsi solo a un elemento.

### --feedback--

Ricorda che un nome `id` è unico e dovrebbe essere usato solo una volta per documento HTML.

---

Usa un `id` quando hai bisogno di un identificatore unico per un elemento specifico, e usa un `class` quando vuoi applicare stili o comportamenti a più elementi.
