# --description--

في React، تُستخدم الأنماط المضمَّنة لتطبيق أنماط CSS مباشرة على عناصر React داخل شفرة JSX الخاصة بك بدلاً من تعريفها في ملفات CSS منفصلة.

تعتمد طريقة React في الأنماط المضمَّنة على استخدام كائنات JavaScript لتعريف الأنماط، بدلاً من سلاسل CSS التقليدية. هذا يعني أنه بدلاً من كتابة الأنماط كما تفعل في ملف CSS، تنشئ كائن JavaScript حيث تكون المفاتيح بصيغة camel case لأسماء خصائص CSS، والقيم هي سلاسل القيم الخاصة بـ CSS.

إليك مثالًا على كيفية استخدام الأنماط المضمَّنة لمكوِّن `Button`:

```jsx
function Button({ buttonText }) {
  const defaultStyles = {
    backgroundColor: "#007BFF",
    color: "white",
    border: "none",
    borderRadius: "4px",
    padding: "10px 20px",
    fontSize: "16px",
    fontWeight: "bold",
    cursor: "pointer",
    transition: "background-color 0.3s ease",
  };

  return <button style={defaultStyles}>{buttonText}</button>;
}
```

في هذا المثال، نُعرّف كائن نمط يسمى `defaultStyles`. ثم نطبق هذه الأنماط على عنصر زر باستخدام الخاصية `style`. تتولى React تطبيق هذه الأنماط على العنصر عند عرضه.

يمكنك أيضًا اختيار تمرير كائن مباشرة إلى الخاصية `style`. هذا ما سيبدو عليه المثال المعدل:

```jsx
function Button({ buttonText }) {
  return (
    <button
      style={{
        backgroundColor: "#007BFF",
        color: "white",
      }}
    >
      {buttonText}
    </button>
  );
}
```

لاحظ الأقواس المتعرجة المزدوجة `{{}}` في الخاصية `style`. الأقواس الخارجية تشير إلى تعبير JavaScript في JSX، بينما الأقواس الداخلية تُعرّف كائن JavaScript حرفي. تتيح هذه الصيغة تضمين كائنات JavaScript مباشرة في خصائص JSX.

أحيانًا قد ترغب في تمرير كائن مباشرة إذا كانت هناك بعض الخصائص فقط كما هو موضح هنا. وإلا، فتمرير اسم لكائن سيكون أفضل كما في المثال الأول.

من المهم ملاحظة أنه بينما تُكتب أسماء خصائص CSS عادةً بصيغة kebab case مثل `font-size`، في الأنماط المضمَّنة في React نستخدم camel case مثل `fontSize`. ذلك لأن كائن النمط هو كائن JavaScript، وأسماء kebab case غير صالحة كمفاتيح كائن في JavaScript بدون استخدام علامات اقتباس.

ميزة كبيرة للأنماط المضمَّنة في React هي أنها تدعم الأنماط الديناميكية بناءً على حالة المكوِّن أو الخصائص. على سبيل المثال:

```jsx
function DynamicButton({ isActive }) {
  const buttonStyles = {
    backgroundColor: isActive ? "green" : "red",
    color: "white",
    padding: "10px 15px",
    border: "none",
    cursor: "pointer",
  };

  return <button style={buttonStyles}>Login</button>;
}
```

في هذا المثال، يتغير لون خلفية الزر بناءً على الخاصية `isActive`. يمكن لهذا النوع من الأنماط الديناميكية أن يكون قويًا لإنشاء واجهات مستخدم تفاعلية ومتجاوبة.

باختصار، توفر الأنماط المضمَّنة في React طريقة قوية لتطبيق وتعديل الأنماط مباشرة داخل مكوناتك. تستخدم كائنات JavaScript بدلاً من سلاسل CSS، وتتطلب أسماء خصائص بصيغة camel case، ويمكنها دمج القيم الديناميكية بسهولة. إنها أداة أساسية في مجموعة أدوات مطوِّر React، خاصة لإنشاء واجهات مستخدم مخصصة وتفاعلية للغاية.
