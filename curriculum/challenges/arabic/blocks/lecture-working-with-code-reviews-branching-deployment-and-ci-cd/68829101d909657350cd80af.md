# --description--

GitHub Pages خيار قوي جدًا لنشر المواقع الثابتة، أو التطبيقات التي لا تحتاج إلى خادم خلفي لمعالجة المنطق. أي التطبيقات التي تعمل بالكامل على جانب العميل، أو في متصفح المستخدم، يمكن نشرها بالكامل على هذه المنصة.

بالإضافة إلى ذلك، GitHub Pages مجانية تمامًا للمشاريع مفتوحة المصدر. هناك العديد من الأسباب الجيدة لجعل مشروعك مفتوح المصدر، وPages تضيف سببًا آخر.

لكن كيف تنشر فعليًا على GitHub Pages؟ حسنًا، هذا يعتمد على كيفية عمل تطبيقك.

لنبدأ بصفحة HTML أساسية. إليك واحدة يمكنك إضافتها إلى مستودعك الحالي إذا أردت:

```html
<!DOCTYPE html>
<html>
  <head>
    <title>freeCodeCamp</title>
    <link rel="stylesheet" href="./style.css" />
  </head>
  <body>
    <h1>Welcome to freeCodeCamp's Lessons!</h1>
    <p>Let's have some fun learning about Git and GitHub!</p>
  </body>
</html>
```

الآن نحتاج إلى زيارة مستودعنا والنقر على تبويب "الإعدادات" للذهاب إلى إعدادات المستودع. ثم نحتاج إلى اختيار "Pages" من الشريط الجانبي.

هناك بعض الخيارات فقط في البداية، لكن هذا سيتغير مع بدء تكوين الإعدادات. أولًا، نحتاج إلى إخبار GitHub من أين ننشر.

سنترك هذا على "النشر من فرع"، لأننا لا نستخدم إطار عمل ولم نُعد أي إجراءات GitHub على الإطلاق. ستتعلم المزيد عن هذه في درس قادم.

بعد ذلك، عيّن الفرع الذي ننشر منه إلى `main` لنشر ما هو موجود عليه.

بعدها ستحصل على خيار للنشر من مجلد الجذر لمشروعك أو من مجلد `docs`. تُستخدم هذه الميزة في مشاريع مثل freeCodeCamp - كنا نضم التوثيق مع تطبيقنا، حتى نتمكن من نشر التوثيق فقط على GitHub Pages عبر مجلد `docs`. في الوقت الحالي، اتركه كما هو على مجلد `/` (الجذر) واضغط على زر "حفظ" بجانب هذا القائمة المنسدلة لحفظ التغييرات.

بمجرد الحفظ، سترى مُدخلًا جديدًا لتكوين نطاق مخصص. إذا كان لديك واحد بالفعل وترغب في استخدامه، يمكنك اتباع الرابط إلى الوثائق للقيام بذلك. لكن في هذا الدرس، سنترك هذا فارغًا.

يمكنك أيضًا رؤية النطاق الذي تم نشر موقع GitHub Pages الجديد عليه. بدون نطاق مخصص، يكون الافتراضي هو `<username>.github.io`، حيث `<username>` هو اسم حسابك (أو حساب المنظمة إذا كنت تنشر من مستودع مملوك للمنظمة). وسيتم تقديم المستودع على مسار مطابق لاسم المستودع، مثل `<username>.github.io/<repo-name>`. إذا زرت هذا الرابط، سترى موقعك!

الآن، عندما تدفع تغييرات جديدة إلى فرع `main`، سيقوم GitHub Pages تلقائيًا بتحديث الموقع المنشور بأحدث التغييرات. قد تستغرق هذه العملية بضع دقائق، حسب حمل الخادم - وقد تحتاج إلى تحديث قسري لمسح ذاكرة التخزين المؤقت.

وبذلك، نكون قد نشرنا بنجاح تطبيق ويب أساسي على GitHub Pages. يمكنك أيضًا تضمين ملفات CSS وJavaScript في نشر Pages، حتى تبني مواقع جميلة وتفاعلية. ومتى تعلمت إطار عمل مثل React أو Angular، يمكنك حتى نشرها على GitHub Pages أيضًا!
