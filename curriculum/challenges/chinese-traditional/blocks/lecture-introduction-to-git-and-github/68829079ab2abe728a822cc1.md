# --description--

在之前的課程中，你學會了如何在 GitHub 上創建一個新的儲存庫。但如果你有一個現有的區域專案想要添加到 GitHub 呢？在本課程中，你將學習如何將現有的區域儲存庫推送到 GitHub，並從遠端儲存庫拉取變更。

首先在 GitHub 上建立一個沒有任何檔案的新儲存庫。當操作正確時，你應該會被導向包含用於複製你的儲存庫的 URL 的頁面。

現在請前往你的終端機並選擇你想要添加到 GitHub 的現有專案。在本課程中，我選擇了一個名為 `super-awesome-game` 的區域專案，裡面包含以下檔案：

```sh
jessicawilkins super-awesome-game >> ls
README.md  index.html script.js  styles.css
```

要為這個專案設定版本控制並使用 Git，你需要在專案目錄中執行 `git init` 指令。這將初始化一個空的 Git 儲存庫，使 Git 可以開始追蹤這個專案的變更。正確完成後，你應該會在終端機中看到類似的輸出：

```sh
jessicawilkins super-awesome-game >> git init
Initialized empty Git repository in /Users/jessicawilkins/workspace/freeCodeCamp/super-awesome-game/.git/
```

當你為專案初始化一個空的 Git 倉庫時，將會新增一個 `.git` 隱藏目錄。若要查看該目錄的新增情況，你可以執行 `ls -a` 指令，該指令會列出所有檔案和目錄，包括隱藏的。

```sh
jessicawilkins super-awesome-game >> ls -a
.          ..         .git       README.md  index.html script.js  styles.css
```

重要的是不要刪除該 `.git` 目錄，否則你所有的 Git 歷史紀錄將會遺失。

既然已經初始化了空的 Git 儲存庫，你可以執行 `git status` 指令。此指令用於顯示你工作目錄的當前狀態－你會在工作流程中經常使用此指令。你應該會看到類似以下的輸出：

```sh
jessicawilkins super-awesome-game >> git status
On branch main

No commits yet

Untracked files:
  (use "git add <file>..." to include in what will be committed)
	README.md
	index.html
	script.js
	styles.css

nothing added to commit but untracked files present (use "git add" to track)
```

`On branch main` 意味著你目前在 `main` 分支。Git 中的分支是一個獨立的工作區，你可以在其中進行更改。`main` 分支通常會表現為實際應用中的主要或生產分支。開發團隊會為新功能和錯誤修正創建多個分支，然後將這些更改合併回 `main` 分支。你將在未來的課程中學習更多關於該處理程序的內容。

`Untracked files` 意味著 Git 偵測到目前未追蹤的檔案。

檔案 可以處於 五 種狀態：

1. 「Untracked」：這表示該檔案是倉庫中的新檔案，且 Git 之前未曾「看到」過它。
2. 「Modified」：此檔案存在於先前的提交中，且有尚未提交的變更。
3. 「Ignored」：你可能不會在 Git 中看到被忽略的檔案，但你的整合開發環境可能會有指示器顯示它們。被忽略的檔案會被排除在 Git 操作之外，通常是因為它們被包含在 `.gitignore` 檔案中。
4. 「Deleted」：已刪除的檔案是未追蹤檔案的相反——它是先前存在且已被移除的檔案。
5. 「重新命名」：重新命名的檔案是指內容未更改，但檔案的名稱或位置已被修改。在某些情況下，即使檔案有少量變更，也可以視為重新命名。

當你的 repo 中有未提交的變更時，你需要先將它們「暫存」。所謂的「暫存區」是 `.git` 目錄中一個特殊的檔案，裡面包含將會納入你下一次提交的資訊。提交實際上是你目錄當前狀態的螢幕快照。

如果你想一次暫存一個檔案，則可以使用以下指令：

```sh
git add name-of-file
```

如果你想要暫存所有未暫存的變更，則可以使用 `git add .`。句點（`.`）是你目前所在目錄的別名。有些開發者會警告不要使用 `git add .`，因為你可能會不小心暫存到你不打算暫存的檔案。如果發生這種情況，你可以使用以下指令來「取消暫存」它們：

```sh
git reset name-of-file
```

一般來說，只要你在暫存之前使用 `git status` 監控哪些檔案被新增，你就應該沒問題。

一旦你將檔案加入暫存區，接著你需要提交它們。你可以透過執行 `git commit` 指令來提交你的變更。這會開啟你在 Git 組態中設定的偏好編輯器。編輯器開啟後，你可以提供詳細的變更訊息。你也可以選擇使用 `git commit -m` 指令來提供較短的訊息，如下所示：

```sh
git commit -m "short message goes here"
```

如果你打算使用 `-m` 旗標，你仍然會想要提供一個描述性的訊息，以提供變更的背景關係。以下是一些你可能會使用的良好訊息範例：

```sh
git commit -m "feat(api): implement JWT-based authentication"
git commit -m "fix(auth): correct token expiration time"
git commit -m "refactor: simplify login flow logic"
```

`feat(api)` 和 `fix(auth)` 遵循 Conventional Commits 風格，用於提供該提交中變更型別的背景關係。

在你提交你的工作後，再次執行 `git status`，你應該會在終端機中看到以下輸出：

```sh
On branch main
nothing to commit, working tree clean
```

若要查看你在此專案中所有先前的提交，可以執行 `git log` 指令。這將列出所有先前的提交，並附有有用的資訊，例如作者、提交日期、提交訊息和提交雜湊值。提交雜湊值是一個長字串，作為提交的唯一識別符。

完成提交你的工作後，你現在可以將變更推送到 GitHub。你首先需要設定到遠端儲存庫的遠端連接。你在上一課中已經學過這個，但這裡用 SSH 提醒你：

```sh
git remote add origin git@github.com:your-github-username/name-of-repo.git
```

最後一步是使用此指令將你的變更推送到 GitHub：

```sh
git push -u origin main
```

`-u` 旗標是 `--set-upstream` 的縮寫。它將你的區域的 `main` 分支連接到 `origin` 上的遠端 `main` 分支。你每個新分支只需使用一次。之後，你就可以直接執行 `git push`，而不需要指定分支名稱。

當正確完成時，你應該會在終端機中看到類似的結果：

```sh
Enumerating objects: 3, done.
Counting objects: 100% (3/3), done.
Delta compression using up to 8 threads
Compressing objects: 100% (2/2), done.
Writing objects: 100% (3/3), 253 bytes | 253.00 KiB/s, done.
Total 3 (delta 0), reused 0 (delta 0), pack-reused 0 (from 0)
To https://github.com/jdwilkin4/super-awesome-game.git
 * [new branch]      main -> main
branch 'main' set up to track 'origin/main'.
```

當你重新整理頁面時，你也應該會在你的 GitHub 倉庫頁面看到新的變更。

如果你正在與他人合作一個專案，並且需要從儲存庫拉取新的變更，你可以使用 `git pull` 指令。你將在未來的課程中學習更多關於團隊合作與協作的內容。
