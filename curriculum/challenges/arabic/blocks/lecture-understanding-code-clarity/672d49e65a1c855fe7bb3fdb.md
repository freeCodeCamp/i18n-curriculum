# --description--

تُستخدم التعليقات في البرمجة لتوفير سياق إضافي للشفرة أو لترك ملاحظات لنفسك وللآخرين.

التعليقات هي أسطر أو كتل نصية يتجاهلها محرك JavaScript عند تنفيذ الشفرة. وجودها فقط لفائدة الأشخاص الذين يقرؤون الشفرة، سواء كنت أنت أو شخصًا آخر.

يوفر JavaScript طريقتين لإضافة التعليقات إلى الشفرة: تعليقات سطر واحد وتعليقات متعددة الأسطر.

يتم إنشاء تعليقات السطر الواحد باستخدام شرطتين مائلتين للأمام (`//`). إليك مثالًا:

```js
// I am a single line comment in JavaScript
```

هذا النوع من التعليقات مناسب للتفسيرات أو التوضيحات القصيرة.

إليك مثالًا من ملفات مشروع منهج freeCodeCamp في العالم الحقيقي:

```js
// This is to allow English to build without having to download the i18n files.
// It fails when trying to resolve the i18n-curriculum path if they don't exist.
const curriculumLocale = process.env.CURRICULUM_LOCALE ?? 'english';
const I18N_CURRICULUM_DIR = path.resolve(
  __dirname,
  curriculumLocale === 'english' ? '.' : 'i18n-curriculum/curriculum'
);
```

لا تقلق بشأن محاولة فهم ما تفعله الشفرة فعليًا لأنها أكثر تقدمًا مما تعلمته حتى الآن. بدلًا من ذلك، ركز على التعليق الذي تركه المطوِّر. هذا التعليق يوفر سياقًا مهمًا لسبب وجود هذه الشفرة.

التعليقات مثل هذه مهمة لمن يعملون ضمن فرق لسببين:

1. المطوِّرون الآخرون الذين يعملون على المشروع سيفهمون هدف هذه الشفرة.

1. تساعد على منع التغييرات أو الحذف غير الضروري دون استشارة الفريق، مما قد يؤدي إلى أخطاء برمجية أو مشاكل.

نوع آخر من التعليقات هو تعليقات متعددة الأسطر. إليك الصيغة النحوية الأساسية:

```js
/*
 I am a multiline comment.
 This is helpful for longer explanations.
*/
```

تعليقات متعددة الأسطر مفيدة عندما تحتاج إلى كتابة أوصاف أو تفسيرات أو ملاحظات أطول في الشفرة.

لنلق نظرة أخرى على ملفات مشروع منهج freeCodeCamp لنرى كيف يمكن استخدام التعليقات متعددة الأسطر في العالم الحقيقي.

```js
/* Since there can be more than one way to complete a certification (using the
legacy curriculum or the new one, for instance), we need a certification
field to track which certification this belongs to. */
const dupeCertifications = [
  {
    certification: 'responsive-web-design',
    dupe: '2022/responsive-web-design'
  }
];
const hasDupe = dupeCertifications.find(
  cert => cert.dupe === meta.superBlock
);
```

كما في السابق، تجاهل كل شفرة JavaScript لأنها تستخدم مفاهيم لم تُدرّس بعد. بدلًا من ذلك، ركز على التعليق الذي تركه المطوِّر.

يمكن لمطوِّر في الفريق، أو حتى مساهم جديد يعمل على المشروع، أن يفهم سبب وجود هذا الجزء من الشفرة وأن يحصل على السياق الكامل قبل العمل على هذه المنطقة من المشروع.

بينما التعليقات مفيدة في البرمجة، من المهم تجنب الإفراط في التعليق. لا تحتاج إلى التعليق على كل سطر من الشفرة، خاصة إذا كانت الشفرة واضحة ومفهومة بذاتها.

إليك مثالًا على استخدام التعليقات لشرح ما هو واضح:

```js
// This code uses the const keyword to create a new variable called price.
// We are assigning the number 10 to the price variable.
const price = 10;
```

في هذه الحالة، لا حاجة لإضافة أي تعليقات هنا لأن الشفرة واضحة بذاتها. الهدف هو تحسين قابلية القراءة، وليس إرباك الشفرة بتفسيرات غير ضرورية.

إذا أردت إضافة تعليقات إلى مشاريعك الشخصية أثناء تعلم البرمجة، فهذا جيد. لكن بمجرد أن تبدأ العمل على مشاريع حقيقية مع مطوِّرين آخرين، من المهم عدم استخدام التعليقات للشفرة الواضحة بذاتها.

من المهم أيضًا عدم استخدام التعليقات لتبرير الشفرة المربكة أو المعقدة جدًا أو المكتوبة بشكل سيئ. في تلك الحالات، من الأفضل إعادة هيكلة الشفرة أو تعديلها حتى يفهمها المطوِّرون الآخرون بشكل أفضل.

التعليقات أدوات قوية لتوثيق الشفرة وجعلها أسهل للفهم. يجب استخدام التعليقات لتوفير السياق أو ترك ملاحظات لنفسك وللآخرين.
