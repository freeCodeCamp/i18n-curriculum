# --interactive--

In una lezione precedente, hai conosciuto per la prima volta il concetto di booleani, ma in questa lezione approfondiremo come funzionano i booleani e come funzionano gli operatori di uguaglianza e disuguaglianza.

I booleani sono un tipo di dato con solo `true` e `false` come valori. Sono utili perché ti permettono di fare qualcosa in base a certe condizioni. I booleani sono essenziali quando vuoi valutare se qualcosa dovrebbe accadere o meno, come decidere se qualcuno può consultare una certa funzionalità nella tua app. Ecco un esempio di impostare il valore `true` in una variabile chiamata `isOldEnoughToDrive`:

:::interactive_editor

```js
let isOldEnoughToDrive = true;

console.log(isOldEnoughToDrive); // true
```

:::

Puoi usare questa variabile all’interno di un condizionale così:

:::interactive_editor

```js
let isOldEnoughToDrive = true;

if (isOldEnoughToDrive) {
 console.log("You're old enough to drive"); // You're old enough to drive
} else {
 console.log("Sorry, you are not old enough to drive");
}
```

:::

Un condizionale ti aiuta a prendere decisioni nel tuo codice basandoti su una condizione. Questo esempio usa quella che si chiama un’istruzione `if/else`.

Se `isOldEnoughToDrive` è `true`, allora la frase `You're old enough to drive` verrà mostrata nella console. Altrimenti, se `isOldEnoughToDrive` è `false`, allora la frase `Sorry, you are not old enough to drive` verrà mostrata nella console. Poiché la variabile `isOldEnoughToDrive` è impostata su `true`, la prima frase verrà mostrata nella console. Imparerai di più sulle istruzioni `if/else` in una lezione futura.

Per confrontare due valori, puoi usare l’operatore di uguaglianza o l’operatore di uguaglianza stretta. Il risultato del confronto sarà un booleano, o `true` o `false`. Ecco un esempio di uso dell’operatore di uguaglianza per confrontare una stringa e un numero. L’operatore di uguaglianza è rappresentato da un doppio segno di uguale (`==`).

:::interactive_editor

```js
console.log(5 == "5"); // true
```

:::

In questo esempio, JavaScript converte la stringa `"5"` nel numero `5` e poi verifica se sono uguali. Poiché entrambi i valori ora sono uguali, il risultato è true. L’operatore di uguaglianza usa la conversione implicita prima di verificare se ogni valore è uguale.

Questo è diverso dall’operatore di uguaglianza stretta, che non esegue conversioni implicite. L’operatore di uguaglianza stretta verifica se i tipi sono gli stessi e se i valori sono gli stessi. Ecco un esempio che usa l’operatore di uguaglianza stretta per confrontare un numero e una stringa. Questo operatore è rappresentato da un triplo segno di uguale (`===`).

:::interactive_editor

```js
console.log(5 === '5'); // false
```

:::

Il confronto seguente sarà `false`, perché un tipo di dato stringa non è uguale a un tipo di dato numero. Se hai bisogno di verificare se qualcosa non è uguale a un altro valore, puoi usare gli operatori di disuguaglianza o di disuguaglianza stretta. Ecco un esempio che usa l’operatore di disuguaglianza (`!=`) per confrontare un numero con una stringa.

:::interactive_editor

```js
console.log(5 != "5"); // false
```

:::

In questo esempio, il risultato sarebbe `false` perché l’operatore di disuguaglianza prima converte il valore stringa in un numero e poi confronta i valori. Poiché i valori sarebbero gli stessi, restituirà `false`. Se provassi a usare l’operatore di disuguaglianza stretta, otterresti un risultato diverso. L’operatore di disuguaglianza stretta è rappresentato da un punto esclamativo seguito da due segni di uguale (`!==`).

:::interactive_editor

```js
console.log(5 !== "5"); // true
```

:::

Il risultato sarebbe `true` perché l’operatore di disuguaglianza stretta non esegue alcuna conversione implicita. Poiché il numero `5` non è uguale alla stringa `"5"`, il risultato è `true`.

È considerata una buona pratica usare gli operatori di uguaglianza e disuguaglianza stretta ogni volta che è possibile, perché non eseguono conversioni implicite. Nella maggior parte dei progetti professionali, vedrai basi di codice che preferiscono solitamente questi due operatori rispetto agli operatori di uguaglianza e disuguaglianza.
