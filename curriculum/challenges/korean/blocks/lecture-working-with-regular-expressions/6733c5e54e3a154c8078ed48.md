# --interactive--

정규식에서 수량자에 대해 배워보겠습니다.

네 자리 숫자 식별 코드를 찾고 싶다고 가정해 보세요. `\d` 문자 클래스를 사용할 수 있다는 것을 알고 있으니, 네 번 반복해서 쓸 수 있습니다. 그리고 불필요한 문자가 포함되지 않도록 시작과 끝 앵커를 모두 포함하겠죠:

```js
const regex = /^\d\d\d\d$/;
```

이 방법도 작동합니다 – 네 개의 숫자 문자를 찾을 수 있습니다. 하지만 같은 클래스를 여러 번 쓰는 대신 수량자를 사용할 수 있습니다.

수량자는 중괄호 안에 한 개 또는 두 개의 숫자를 넣어 정의합니다. 패턴에 수량자를 사용해 보겠습니다:

```js
const regex = /^\d{4}$/;
```

수량자에 `4` 숫자만 들어 있는 것을 확인하세요. 이 구문은 "이전 문자를 정확히 네 번 일치시켜라"는 뜻입니다. 어떻게 작동하는지 살펴보겠습니다:

:::interactive_editor

```js
const regex = /^\d{4}$/;
console.log(regex.test("123")); // false
console.log(regex.test("1234")); // true
console.log(regex.test("12345")); // false
console.log(regex.test("123456")); // false
console.log(regex.test("1234567")); // false
```

:::

앵커를 사용했고 수량자가 정확히 네 개만 허용하므로, 이 패턴은 정확히 네 자리 숫자 문자열만 일치시킵니다. 하지만 식별 코드는 네 자리 이상이어야 할 수도 있습니다.

네 자리 이상을 허용하려면 수량자 숫자 뒤에 쉼표를 추가하세요:

```js
const regex = /^\d{4,}$/;
```

이제 구문이 네 자리 이상 숫자와 일치할 수 있습니다. 테스트해 보겠습니다:

:::interactive_editor

```js
const regex = /^\d{4,}$/;
console.log(regex.test("123")); // false
console.log(regex.test("1234")); // true
console.log(regex.test("12345")); // true
console.log(regex.test("123456")); // true
console.log(regex.test("1234567")); // true
```

:::

일곱 자리 식별자는 꽤 깁니다. 이 식별자는 최대 6자리, 최솟값 4자리여야 합니다. 이를 위해 쉼표 뒤에 두 번째 숫자를 수량자에 추가할 수 있습니다:

```js
const regex = /^\d{4,6}$/;
```

이제 일곱 자리 식별자는 최대 6자리를 초과하므로 패턴과 일치하지 않습니다.

:::interactive_editor

```js
const regex = /^\d{4,6}$/;
console.log(regex.test("123")); // false
console.log(regex.test("1234")); // true
console.log(regex.test("12345")); // true
console.log(regex.test("123456")); // true
console.log(regex.test("1234567")); // false
```

:::

최댓값만 설정하는 구문은 사용할 수 없다는 점에 유의하세요 – 항상 최솟값도 설정해야 합니다. 하지만 최솟값을 `1`로 설정하면 사실상 같은 결과를 얻을 수 있습니다.

사용자 요구사항이 업데이트되었습니다. 식별자는 이제 선택적으로 문자로 시작할 수 있습니다. 이미 해당 문자 클래스는 알고 있으니 정규식에 추가해 보겠습니다:

```js
const regex = /^[a-zA-Z]\d{4,6}$/;
```

하지만 이제 문자가 반드시 있어야 합니다. 어떻게 선택적으로 만들 수 있을까요?

수량자 구문에서 `0`을 최솟값, `1`를 최댓값으로 사용할 수 있습니다:

```js
const regex = /^[a-zA-Z]{0,1}\d{4,6}$/;
```

하지만 단일 선택적 문자를 위한 특별한 축약 수량자가 있습니다 – 물음표(`?`)입니다. 수량자를 물음표로 바꿔 보겠습니다:

```js
const regex = /^[a-zA-Z]?\d{4,6}$/;
```

결과를 검증해 보겠습니다:

:::interactive_editor

```js
const regex = /^[a-zA-Z]?\d{4,6}$/;
console.log(regex.test("123")); // false
console.log(regex.test("a1234")); // true
console.log(regex.test("12345")); // true
console.log(regex.test("az12345")); // false
console.log(regex.test("X123456")); // true
console.log(regex.test("1234567")); // false
```

:::

이제 패턴은 한 개의 선택적 문자 다음에 네 자리에서 여섯 자리 숫자를 허용합니다.

불행히도 요구사항을 잘못 읽은 것을 방금 알았습니다. 숫자 앞에 아무 개수의 문자가 올 수 있어야 합니다. 최솟값이 `0`이고 최댓값이 없는 수량자를 사용할 수 있습니다:

```js
const regex = /^[a-zA-Z]{0,}\d{4,6}$/;
```

하지만 패턴이 다시 길어지고 있습니다. 다행히 "이전 문자를 0번 이상 일치"시키는 또 다른 축약 수량자가 있습니다 – 별표(`*`)입니다. 패턴에서 수량자를 별표로 바꾸고 테스트해 보겠습니다:

:::interactive_editor

```js
const regex = /^[a-zA-Z]*\d{4,6}$/;
console.log(regex.test("123")); // false
console.log(regex.test("a1234")); // true
console.log(regex.test("12345")); // true
console.log(regex.test("az12345")); // true
console.log(regex.test("X123456")); // true
console.log(regex.test("1234567")); // false
```

:::

이제 0개 이상의 문자 다음에 4~6자리 숫자가 오는 모든 식별자와 성공적으로 일치합니다. 하지만 이 패턴이 시스템을 다운시키고 있습니다 – 최소한 한 개의 문자가 반드시 있어야 합니다.

다시, 최솟값 1에 최댓값이 없는 수량자를 쓰거나 또 다른 특별 구문인 더하기(`+`) 기호를 사용할 수 있습니다:

:::interactive_editor

```js
const regex = /^[a-zA-Z]+\d{4,6}$/;
console.log(regex.test("123")); // false
console.log(regex.test("a1234")); // true
console.log(regex.test("12345")); // false
console.log(regex.test("az12345")); // true
console.log(regex.test("X123456")); // true
console.log(regex.test("1234567")); // false
```

:::

이제 최소 한 개의 문자로 시작하지 않는 식별자는 숫자가 몇 개 있든 실패합니다.

수량자를 사용하면 정규식의 간결성과 가독성을 크게 높일 수 있습니다.
