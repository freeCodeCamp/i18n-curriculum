# --interactive--

In der vorherigen Lektion haben Sie gelernt, wie Sie mit dem `Date`-Objekt in JavaScript arbeiten. Es gibt jedoch verschiedene Möglichkeiten, Daten in JavaScript zu formatieren.

In dieser Lektion sehen wir uns an, wie Sie mit den `toISOString()`-, `toString()`- und `toLocaleDateString()`-Methoden Daten in JavaScript formatieren können.

Bevor wir uns die verschiedenen Methoden ansehen, überprüfen wir zunächst, wie das `Date`-Objekt aussieht:

:::interactive_editor

```js
const date = new Date();
console.log(date);
```

:::

Wenn Sie das `date`-Objekt in der Konsole ausgeben, sehen Sie das aktuelle Datum und die Uhrzeit basierend auf den Systemeinstellungen des Benutzers. Hier ist ein Beispiel für die Ausgabe, die Sie sehen könnten:

```js
Sun Sep 29 2024 19:45:37 GMT-0700 (Pacific Daylight Time)
```

Wenn Sie die `toString()`-Methode auf dem `date`-Objekt verwenden würden, würden Sie dieselbe Ausgabe wie oben sehen.

Um das Datum im erweiterten ISO-Format (ISO 8601) zu formatieren, können Sie die `toISOString()`-Methode wie folgt verwenden:

:::interactive_editor

```js
const date = new Date();
console.log(date.toISOString());
```

:::

ISO 8601 ist ein internationaler Standard zur Darstellung von Daten und Zeiten. Das Format ist `YYYY-MM-DDTHH:mm:ss.sssZ`.

Hier ist ein Beispiel für die Ausgabe, die Sie sehen könnten:

```js
2024-09-30T02:47:20.292Z
```

Eine weitere Möglichkeit, das Datum zu formatieren, besteht darin, die `toLocaleDateString()`-Methode zu verwenden. Diese Methode ermöglicht es Ihnen, das Datum basierend auf der Spracheinstellung des Benutzers zu formatieren. Hier ist die grundlegende Syntax für die Verwendung der `toLocaleDateString()`-Methode:

:::interactive_editor

```js
const date = new Date();
console.log(date.toLocaleDateString());
```

:::

Hier ist ein Beispiel für die Ausgabe, die Sie sehen könnten:

```js
9/29/2024
```

Die `toLocaleDateString()`-Methode akzeptiert zwei optionale Parameter: `locales` und `options`.

Der `locales`-Parameter ist ein String, der die zu verwendende Spracheinstellung darstellt. Zum Beispiel können Sie `en-US` für Englisch (Vereinigte Staaten) oder `fr-FR` für Französisch (Frankreich) übergeben. Wenn Sie keinen `locales`-Parameter übergeben, wird die Standardspracheinstellung verwendet. Hier ist ein Beispiel, wie Sie den `locales`-Parameter verwenden:

:::interactive_editor

```js
const date = new Date();
console.log(date.toLocaleDateString("fr-FR"));
```

:::

Der zweite optionale Parameter ist der `options`-Parameter. Dieser Parameter ist ein Objekt, mit dem Sie das Format des Datumsstrings angeben können. Hier ist ein Beispiel, wie Sie den `options`-Parameter verwenden:

:::interactive_editor

```js
const date = new Date();
const options = {
  weekday: "long",
  year: "numeric",
  month: "long",
  day: "numeric",
};
console.log(date.toLocaleDateString("en-GB", options));
```

:::

Im obigen Beispiel haben wir die Optionen so festgelegt, dass sie auf Englisch (Großbritannien) sind und den vollständigen Wochentag, das Jahr, den Monat und den Tag enthalten.

Hier ist ein Beispiel für die Ausgabe, die Sie sehen könnten:

```js
Sunday, September 29, 2024
```

In dieser Lektion haben wir nur einige der Möglichkeiten behandelt, Daten in JavaScript zu formatieren. Es gibt viele weitere Methoden und Bibliotheken, die Ihnen helfen, Daten in JavaScript zu formatieren. Aber die `toISOString()`- und `toLocaleDateString()`-Methoden sind ein guter Ausgangspunkt, um Daten in JavaScript zu formatieren.
