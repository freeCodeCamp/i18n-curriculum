# --interactive--

Подія `change` — це спеціальна подія, яка спрацьовує, коли користувач змінює значення певних вхідних елементів. Конкретніше:

- Коли прапорець позначається або знімається.

- Коли радіокнопка позначається.

- Коли користувач робить вибір у чомусь на кшталт вибору дати або випадаючого меню.

- Коли поле введення втрачає фокус (користувач переходить до наступного поля за допомогою табуляції або клікає поза формою) після того, як значення було змінено.

- Коли користувач інакше підтверджує значення, наприклад, натискаючи Enter після введення тексту.

Зверніть увагу, що подія `change` НЕ спрацьовує під час введення тексту користувачем у полі. Подія `change` спрацює лише після того, як користувач сфокусується на іншому елементі.

Ось приклад використання події `change` з випадаючим меню:

:::interactive_editor

```html
<select id="select-menu">
  <option value="option1">Option 1</option>
  <option value="option2">Option 2</option>
  <option value="option3">Option 3</option>
</select>
<script src="index.js"></script>
```

```js
const selectMenu = document.getElementById("select-menu");
selectMenu.addEventListener("change", (event) => {
  console.log(`You selected: ${event.target.value}`);
});
```

:::

Кожного разу, коли користувач вибирає інший варіант у випадаючому меню, спрацьовує подія `change`, і вибране значення виводиться в консоль.

Подія `change` все ще створює об’єкт `Event`, але на відміну від більшості інших подій, вона не створює кастомну реалізацію — є доступними лише ті властивості та методи, які є у базового об’єкта `Event`.

Це відрізняється від події `input`, яка створює спеціалізований об’єкт `InputEvent`. Подія change також відрізняється в кількох аспектах. Наприклад, подія `input` спрацює, коли користувач вводить текст у поле.

Ці відмінності важливо пам’ятати, щоб не заплутатися у часі спрацьовування цих подій.
