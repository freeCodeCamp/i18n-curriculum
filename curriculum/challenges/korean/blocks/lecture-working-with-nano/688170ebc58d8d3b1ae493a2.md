# --description--

터미널에서 작업할 때는 일반 텍스트 편집기를 항상 사용할 수 없지만, 터미널 내에서 직접 파일을 편집하도록 설계된 프로그램들이 있습니다.

리눅스 환경에는 일반적으로 텍스트 편집기가 포함되어 있습니다. Vi 또는 Vim이 가장 흔합니다. Emacs와 Nano는 수동으로 설치해야 할 수도 있지만 거의 모든 배포판에서 쉽게 구할 수 있습니다.

Vim의 문제는 학습 곡선이 상당히 가파르다는 점입니다. 이미 익숙하지 않다면 애플리케이션을 어떻게 종료하는지도 모를 수 있습니다! Emacs는 조금 더 사용자 친화적이라고 할 수 있지만, 여전히 많은 키보드 단축키를 익혀야 합니다.

Nano는 특히 초보자에게 훨씬 간단하고 사용하기 쉽습니다. Nano로 파일을 빠르게 열고 편집할 수 있으며, 기본 키보드 단축키에 대한 화면 내 도움말도 제공합니다.

하지만 그 대신 확장성이 부족합니다. Vim이나 Emacs는 플러그인으로 확장해 완전한 개발 환경으로 만들 수 있지만, Nano는 기본 기능만 갖추고 있습니다.

Nano가 할 수 있는 것을 살펴보겠습니다. 파일을 열려면 터미널에서 `nano <filename>`를 입력하세요.

파일이 열리면 화면 상단에 실행 중인 Nano 버전과 열린 파일 이름이 표시됩니다.

그 아래에는 파일 내용이 나타납니다. 화살표 키로 커서를 이동하고 키보드로 텍스트를 삽입하거나 삭제할 수 있습니다. 필요에 따라 파일이 스크롤됩니다.

그리고 하단에는 다음과 같은 사용 가능한 키보드 단축키 목록이 표시됩니다:

```bash
^G Help   ^O Write Out    ^F Where Is   ^K Cut      ^T Execute  ^C Location     M-U Undo    M-A Set Mark    M-] To Bracket    M-B Previous
^X Exit   ^R Read File    ^\ Replace    ^U Paste    ^J Justify  ^/ Go To Line   M-E Redo    M-6 Copy        ^B Where Was      M-F Next
```

캐럿(`^`) 기호로 시작하는 단축키는 Ctrl 키를 누른 상태에서 표시된 문자를 눌러야 한다는 뜻입니다. `M-`로 시작하는 단축키는 대부분 설정에서 Alt 키인 "메타 키"를 누른 상태에서 문자를 눌러야 합니다.

Nano에서 파일을 편집하려면 몇 가지 단축키만 알면 됩니다. 파일을 수정한 후에는 `Ctrl + O`를 눌러 "Write Out", 즉 파일을 저장할 수 있습니다. 명령을 입력하면 하단 메뉴가 다음과 같이 바뀝니다:

```bash
File Name to write : <filename>
^G Get Help                 ^T  To Files
^C Cancel                   TAB Complete
```

엔터를 눌러 파일을 저장하고 편집으로 돌아가거나, `Ctrl + C`를 눌러 취소하고 편집으로 돌아가세요.

편집 중에는 `Ctrl + X`를 눌러 Nano를 종료하고 터미널로 돌아갈 수 있습니다. 저장하지 않은 변경 사항이 있을 때 종료하려 하면 하단 메뉴에 다음이 표시됩니다:

```bash
Save modified buffer (ANSWERING "No" WILL DESTROY CHANGES) ?
                            Y Yes
^C Cancel                   N No
```

`Y`를 눌러 저장하지 않은 변경 사항을 저장하거나, `N`를 눌러 버리세요.

이 지식을 바탕으로 전체 그래픽 환경에 접근할 수 없는 원격 서버에서 빠르게 파일을 편집해야 할 때 더 잘 대비할 수 있을 것입니다.
