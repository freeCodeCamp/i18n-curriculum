# --interactive--

Стилі посилань за замовчуванням відіграють ключову роль у підвищенні зручності використання та доступності в інтернеті.

Ці стилі, зазвичай синій для непереглянутих посилань і фіолетовий для переглянутих, стали стандартом, якого користувачі очікують і на який покладаються під час навігації по сайтах.

Основна мета стилів посилань за замовчуванням — надати чіткі візуальні підказки, які допомагають користувачам розрізняти інтерактивні та неінтерактивні елементи на сторінці.

Це розмежування є фундаментальним для створення інтуїтивного та зручного для користувача досвіду перегляду.

Розглянемо базові стилі посилань за замовчуванням:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<a href="/">Example link</a>
```

```css
a:link {
  color: blue;
  text-decoration: underline;
}

a:visited {
  color: purple;
}
```

:::

Ці стилі виконують кілька важливих функцій.

По-перше, синій колір для непереглянутих посилань виділяється на більшості фонових кольорів і тексту, що робить посилання легко впізнаваними. Цей контраст дуже важливий, щоб користувачі могли швидко переглянути сторінку і знайти навігаційні елементи або важливу інформацію.

Підкреслення додатково підкреслює, що текст є активним, надаючи ще одну візуальну підказку. Це особливо корисно для користувачів з порушенням сприйняття кольору або тих, кому важко розрізняти кольори.

Зміна кольору для переглянутих посилань (зазвичай на фіолетовий) допомагає користувачам відстежувати, де вони вже були. Ця функція безцінна при навігації великими сайтами або під час досліджень, оскільки запобігає випадковому повторному відвідуванню тих самих сторінок.

Розглянемо цей HTML-приклад:

:::interactive_editor

```html
<p>Learn more about <a href="https://www.example.com/cats">cats</a> and <a href="https://www.example.com/dogs">dogs</a>.</p>
```

:::

Без будь-якого кастомного CSS більшість браузерів відтворять ці посилання синім кольором з підкресленням. Після натискання на одне з посилань його колір зміниться на фіолетовий, надаючи користувачу миттєвий зворотний зв’язок про історію перегляду.

Хоча дизайнери часто змінюють ці стилі за замовчуванням, щоб відповідати естетиці сайту, важливо зберігати основні принципи, які за ними стоять.

Якщо ви вирішите змінити стилі за замовчуванням, переконайтеся, що посилання все ще чітко відрізняються від звичайного тексту, існує помітна різниця між переглянутими та непереглянутими посиланнями, а обрані кольори забезпечують достатній контраст із фоном.

Наприклад, ви можете використати кастомний стиль, як цей:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<a href="/">Example link</a>
```

```css
a:link {
  color: blue;
  text-decoration: none;
  border-bottom: 1px solid blue;
}

a:visited {
  color: purple;
  border-bottom: 1px solid purple;
}
```

:::

Це зберігає синю та фіолетову кольорову схему, замінюючи підкреслення на нижній кордон для більш сучасного вигляду.

Також важливо враховувати різні стани посилань. Окрім станів за замовчуванням і переглянутих, посилання зазвичай мають стани наведення та активності:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<a href="/">Example link</a>
```

```css
a:hover {
  color: red;
}

a:active {
  color: darkorange;
}
```

:::

Ці стани надають користувачам миттєвий зворотний зв’язок під час взаємодії з посиланнями, покращуючи загальну зручність використання сайту.

На завершення, хоча можна кастомізувати стилі посилань, принципи, що лежать в основі стилів за замовчуванням, слід зберігати.

Вони відіграють ключову роль у створенні зручного та доступного веб-досвіду, допомагаючи користувачам ефективно орієнтуватися та взаємодіяти з вмістом.

Завжди ставте на перше місце чіткість і зручність користувача при розробці стилів посилань для своїх сайтів.
