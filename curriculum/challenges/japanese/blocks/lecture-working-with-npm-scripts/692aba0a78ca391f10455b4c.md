# --description--

CommonJSモジュールシステムを理解したので、次はESモジュールシステムを見てみましょう。

ESモジュール、またはECMAScriptモジュールは、ECMAScript仕様で定義されたJavaScriptの公式で標準化されたモジュール形式です。`import` と `export` ステートメントを使い、Node.jsでは非同期で読み込まれるため、より効率的です。

Node.jsがファイルをESモジュールとして扱うのは以下の場合です：

* `.mjs` 拡張子のファイル。
    
* そして、最も近い親の`.js` ファイルにトップレベルの`package.json` フィールドが`"type"` の値を持つ場合の`"module"` 拡張子のファイル。
    

ここでは、以前に見たことのある同じ`multiply` 関数がありますが、今回はESMシステムを使います。そのために、両方のモジュールに`.mjs` 拡張子を代入しています。

`multiply` モジュールから`math.mjs` 関数をエクスポートするには、`export` キーワードの後に、エクスポートしたい関数、変数、またはオブジェクトの名前を中括弧内に書きます。

```javascript
function multiply(a, b) {
  return a * b;
}

export { multiply };
```

また、`export` キーワードを`function` キーワードの前に使って関数を直接エクスポートすることもできます。これは名前付きエクスポートと呼ばれ、モジュールごとに複数の名前付きエクスポートを持つことができます。

```javascript
export function multiply(a, b) {
  return a * b;
}
```

次に、このモジュールをインポートするには、`import` キーワードの後に中括弧を使います。ここで`import` を使い、`require()` ではないことに注目してください。これがCommonJSとESモジュールの主な違いの一つです。

中括弧の中には、インポートしたい関数、変数、またはオブジェクトの名前を書きます。この場合は`multiply` です。そして、`from` キーワードとモジュールへの相対パスを使います。このコードは正常に実行され、出力は`42` になります。

```javascript
import { multiply } from './math.mjs';

console.log(multiply(6, 7)); // 42
```

ESシステムでは、モジュールごとに1つのデフォルトエクスポートを設定できます。こうすると、インポート時に中括弧を使う必要がありません。

以下は、`multiply` モジュール全体のデフォルトエクスポートとして`math.mjs` 関数を設定する例です。

```javascript
export default function multiply(a, b) {
  return a * b;
}
```

これで関数はデフォルトでインポートされ代入されるため、中括弧は不要です。

```javascript
import multiply from './math.mjs';
```

ファイル拡張子や`package.json` ファイルでNode.jsに明示的にどのモジュールシステムを使うか指示しない場合、ソースコードを調べてESモジュール構文があればESモジュールとして実行し、なければCommonJSモジュールとして実行します。

CommonJSはNode.jsの初期にデフォルトの標準として使われていましたが、開発者が効率性や最新のJavaScriptとの互換性のためにESMを採用することが増えているため、両者の違いと使い方を知ることが重要です。
