# --description--

<ol>
  <li>Prendi un numero intero <code>n₀</code>, maggiore di zero.</li>
  <li>Forma il numero successivo <code>n</code> della serie invertendo <code>n₀</code> e sommando il risultato a <code>n₀</code>.</li>
  <li>Termina quando <code>n</code> diventa palindromo — cioè le cifre di <code>n</code> in ordine inverso sono uguali a <code>n</code>.</li>
</ol>

La relazione di ricorrenza sopra, applicata alla maggior parte dei numeri iniziali `n` = 1, 2, ... termina abbastanza rapidamente in un palindromo.

Ad esempio, se `n₀` = 12 otteniamo:

```md
12
12 + 21 = 33,  a palindrome!
```

E se `n₀` = 55 otteniamo:

```md
55
55 + 55 = 110
110 + 011 = 121,  a palindrome!
```

Nota che il controllo per un palindromo avviene *dopo* una somma.

Alcuni numeri iniziali sembrano andare avanti all'infinito; la relazione di ricorrenza per 196 è stata calcolata per milioni di ripetizioni formando numeri con milioni di cifre, senza mai formare un palindromo. Questi numeri che non terminano in un palindromo sono chiamati **numeri lychrel**.

Per questa attività un numero lychrel è qualsiasi numero iniziale che non forma un palindromo entro 500 (o più) iterazioni.

**Numero seed e numeri lychrel correlati:**

Qualsiasi numero intero prodotto nella sequenza di un numero lychrel è anch'esso un numero lychrel.

In generale, qualsiasi sequenza da un numero lychrel *potrebbe* convergere per unirsi alla sequenza di un precedente candidato numero lychrel; ad esempio le sequenze per i numeri 196 e poi 689 iniziano così:

```md
    196
    196 + 691 = 887
    887 + 788 = 1675
    1675 + 5761 = 7436
    7436 + 6347 = 13783
    13783 + 38731 = 52514
    52514 + 41525 = 94039
    ...
    689
    689 + 986 = 1675
    1675 + 5761 = 7436
    ...
```

Quindi vediamo che la sequenza che parte da 689 converge e continua con gli stessi numeri di quella per 196.

Per questo motivo possiamo ulteriormente dividere i numeri lychrel in veri candidati **numero seed** lychrel, e numeri **correlati** che non producono palindromi ma hanno numeri interi nella loro sequenza già visti come parte della sequenza generata da un numero lychrel inferiore.

# --solutions--

```js
function isLychrel(n) {
  function reverse(num) {
    return parseInt(
      num
        .toString()
        .split('')
        .reverse()
        .join('')
    );
  }

  var i;
  for (i = 0; i < 500; i++) {
    n = n + reverse(n);
    if (n == reverse(n)) break;
  }

  return i == 500;
}
```
