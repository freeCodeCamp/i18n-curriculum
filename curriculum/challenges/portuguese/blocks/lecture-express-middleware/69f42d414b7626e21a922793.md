# --description--

Middleware no nível do roteador é semelhante ao middleware no nível da aplicação, mas está vinculado a uma instância de `express.Router()` em vez da instância principal do app. Ele pode ser usado para modularizar sua aplicação Express criando módulos de rota isolados.

Middleware no nível do roteador funciona da mesma forma que o middleware no nível da aplicação, mas é aplicado a instâncias específicas do roteador. Isso facilita aplicar middleware apenas a um conjunto de rotas em vez de globalmente.

```js
const express = require('express')
const app = express()
const router = express.Router()

// Router-level middleware to log requests to '/menu' routes
router.use((req, res, next) => {
  console.log('Request made to /menu route')
  next() // Pass control to the next handler
})

// Define routes
router.get('/drinks', (req, res) => {
  res.send('Welcome to the drinks menu!')
})

app.use('/menu', router) // Apply the router

app.listen(3000, () => {
  console.log('Server running on http://localhost:3000')
})
```

* `router.use()`: Este é middleware no nível do roteador, que é aplicado somente às rotas tratadas pelo roteador.
* `app.use('/menu', router)`: O middleware se aplica apenas às rotas que começam com `/menu` (por exemplo, `/menu/drinks`).

Middleware no nível do roteador permite dividir seu app em componentes modulares e aplicar middleware seletivamente a um grupo de rotas. Isso ajuda na organização e clareza do código.
