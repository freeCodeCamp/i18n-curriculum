# --description--

```css
/* rule 1 */
.class.second-class {
  font-size: 12px;
}

/* rule 2 */
.class .second-class {
  font-size: 24px;
}
```

Тут і правило 1, і правило 2 мають однакову специфічність. Правило 1 використовує ланцюговий селектор (без пробілу), а правило 2 — комбінацію нащадка (порожній пробіл). Але обидва правила містять по два класи, і сам символ комбінації не додає специфічності.

```css
/* rule 1 */
.class.second-class {
  font-size: 12px;
}
/* rule 2 */
.class > .second-class {
  font-size: 24px;
}
```

Цей приклад показує те саме. Навіть якщо правило 2 використовує комбінацію дочірнього елемента (`>`), це не змінює значення специфічності. Обидва правила все одно мають по два класи, тому їх специфічність однакова.

*Примітка:* не все додає специфічність

Порівнюючи селектори, ви можете зустріти спеціальні символи універсального селектора (`*`) та комбінації (`+`, `~`, `>` і порожній пробіл). Ці символи самі по собі не додають специфічності.
