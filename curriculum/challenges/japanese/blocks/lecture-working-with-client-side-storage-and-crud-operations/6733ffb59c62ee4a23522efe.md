# --interactive--

`IndexedDB` はブラウザ内で構造化データを保存するためのツールです。これは最新のウェブブラウザに組み込まれており、ウェブアプリがJavaScriptオブジェクトを効率的に保存および取得できるようにします。

`localStorage` のように文字列の保存に限定される他のストレージ機構とは異なり、`IndexedDB` はJavaScriptオブジェクト、ファイル、そしてほぼあらゆる種類のデータを保存できます。これにより、大規模で複雑なデータ構造を扱う必要があるウェブアプリケーションにとって便利です。

では、`IndexedDB` がどのように動作するか見てみましょう。

最初のステップはデータベースを開くことです。例を示します：

:::interactive_editor

```js
let request = indexedDB.open("Sample DB", 1);

request.onerror = function(event) {
  console.log("Error opening database");
};

request.onsuccess = function(event) {
  let db = event.target.result;
  console.log("Database opened successfully");
};
```

:::

このコードでは、バージョン1の`"Sample DB"` という名前のデータベースを開いています。エラー処理用のコールバック関数と、データベースが正常に開かれたときのコールバック関数の2つを提供しています。成功コールバックで得られる`db` オブジェクトを使ってデータベースとやり取りします。

ブラウザの開発者ツールのアプリケーションインターフェースを確認すると、`Sample DB` セクションに`IndexedDb` が開かれているのが見えます。データベースを開いたら、オブジェクトストアの操作を始められます。

IndexedDBのオブジェクトストアは、従来のデータベースのテーブルに似ています。保存したい実際のデータを保持します。オブジェクトストアを作成する方法は次の通りです：

```js
let request = indexedDB.open("Sample DB", 1);

request.onupgradeneeded = function(event) {
  let db = event.target.result;
  let objectStore = db.createObjectStore("customers", { keyPath: "id" });
};
```

このコードは、`"customers"` をキー・パスとして`"id"` という名前のオブジェクトストアを作成します。キー・パスは従来のデータベースの主キーのようなもので、各レコードを一意に識別するために使われます。

オブジェクトストアにデータを追加するには、次のようにします。この例の`db` は`IndexedDB` データベースのインスタンスを表します。

```js
let transaction = db.transaction(["customers"], "readwrite");
let objectStore = transaction.objectStore("customers");
let request = objectStore.add({ id: 1, name: "John Doe", email: "john@example.com" });

request.onerror = function(event) {
  console.log("Error adding data");
};

request.onsuccess = function(event) {
  console.log("Data added successfully");
};
```

このコードは、`"customers"` オブジェクトストアに新しい顧客を追加します。トランザクションを開始し（これはデータベース操作をまとめる方法です）、オブジェクトストアの参照を取得し、データを追加しています。

データの取得も同様の方法で行います。トランザクションを開始し、オブジェクトストアを取得してから、`get` のようなメソッドを使ってデータを取得します：

```js
let transaction = db.transaction(["customers"]);
let objectStore = transaction.objectStore("customers");
let request = objectStore.get(1);

request.onerror = function(event) {
  console.log("Error retrieving data");
};

request.onsuccess = function(event) {
  console.log("Customer:", request.result);
};
```

このコードは、`id` オブジェクトストアから`1` の`"customers"` を持つ顧客を取得します。

`IndexedDB` の重要な特徴の一つは非同期であることです。

これは、`IndexedDB` とやり取りするときに、操作がウェブアプリケーションのメインスレッドをブロックしないことを意味します。これにより、大量のデータを扱う場合でもウェブアプリケーションがレスポンシブなままでいられます。

`IndexedDB` は強力な機能を提供しますが、他のよりシンプルなストレージAPIオプションに比べて学習曲線が急であり、初心者には難しい場合があります。しかし、クライアント側で大量の構造化データを扱う必要があるアプリケーションにとって、`IndexedDB` は比類のない機能を提供します。
