## Was ist objektorientierte Programmierung?

- **Objektorientierte Programmierung**: Ein Programmierstil, bei dem Entwickler alles in ihrem Code wie ein reales Objekt behandeln. Es wird oft OOP genannt. Die vier Grundsätze, die Ihnen helfen, Code effektiv zu organisieren und zu verwalten, sind **Kapselung**, **Vererbung**, **Polymorphismus** und **Abstraktion**.
- **Klassen**: Die Vorlage zur Erstellung von Objekten. Jedes einzelne Objekt, das aus einer Klasse erstellt wird, hat Attribute, die Daten definieren, und Methoden, die das Verhalten der Objekte bestimmen.

## Was ist Kapselung?

- **Kapselung**: Das Bündeln der Attribute und Methoden eines Objekts zu einer einzigen Einheit. Sie ermöglicht es, den internen Zustand des Objekts hinter einer einfachen Menge öffentlicher Methoden und Attribute zu verbergen, die wie Türen funktionieren. Hinter diesen Türen befinden sich private Attribute und Methoden, die steuern, wie sich die Daten ändern und wer sie sehen kann.
- **Beispiel für Kapselung**: Wenn Sie den Kontostand einer Brieftasche verfolgen möchten, erlauben Sie Einzahlungen und Abhebungen, aber niemand soll den Kontostand selbst manipulieren können:

```py
class Wallet:
   def __init__(self, balance):
       self.__balance = balance # Private attribute

   def deposit(self, amount):
       if amount > 0:
           self.__balance += amount # Add to the balance safely

   def withdraw(self, amount):
       if 0 < amount <= self.__balance:
           self.__balance -= amount # Remove from the balance safely

account = Wallet(500)
print(account.__balance) # AttributeError: 'Wallet' object has no attribute '__balance'
```

- **Unterschied zwischen Präfix mit einfachem und doppeltem Unterstrich bei Attributen**: Ein Präfix mit einem einfachen Unterstrich bei Attributen und Methoden bedeutet, dass sie für den internen Gebrauch gedacht sind. Dies ist eine Konvention und verhindert nicht den Zugriff von außen. Ein Präfix mit doppeltem Unterstrich verhindert effektiv, dass sie von außerhalb ihrer Klasse aufgerufen werden können.

## Was sind Getter und Setter?

- **Getter und Setter**: Methoden, mit denen Sie steuern, wie auf die Attribute einer Klasse zugegriffen und wie sie geändert werden. Sie rufen Werte mit Gettern ab und setzen Werte mit Settern.
- **Eigenschaften**: Sie verbinden Getter und Setter und ermöglichen den Zugriff auf Daten. Sie führen hinter den Kulissen zusätzliche Logik aus, wenn Sie Werte abrufen, setzen oder löschen.
- **Warum Eigenschaften statt Methoden?** Eigenschaften werden anstelle von Methoden verwendet, um die Lesbarkeit zu verbessern und den Code sauberer zu machen. Sie ermöglichen den Zugriff auf Werte mit Punktnotation, wie bei regulären Attributen, jedoch ohne runde Klammern.
- **Einen Getter erstellen**: Um einen Getter zu erstellen, verwenden Sie den `@property`-Dekorator. Hier ist ein Getter, der den Radius eines Kreises abruft:

```py
class Circle:
    def __init__(self, radius):
        self._radius = radius

    @property
    def radius(self): # A getter to get the radius
        return self._radius
  
    @property
    def area(self):  # A getter to calculate area
        return 3.14 * (self._radius ** 2)

my_circle = Circle(3)

print(my_circle.radius) # 3
print(my_circle.area) # 28.26
```

- **Einen Setter erstellen**: Um den Setter zu erstellen, der den Radius setzt, müssen Sie eine weitere Methode mit demselben Namen definieren und `@<property_name>.setter` darüber verwenden:

```py
class Circle:
    def __init__(self, radius):
        self._radius = radius

    @property
    def radius(self):  # A getter to get the radius
        return self._radius

    @radius.setter
    def radius(self, value):  # A setter to set the radius
        if value <= 0:
            raise ValueError('Radius must be positive')
        self._radius = value

my_circle = Circle(3)
print('Initial radius:', my_circle.radius) # Initial radius: 3

my_circle.radius = 8
print('After modifying the radius:', my_circle.radius) # After modifying the radius: 8
```

- **Wie Python Getter und Setter behandelt**: Sobald Sie Getter und Setter definieren, ruft Python sie automatisch im Hintergrund auf, wann immer Sie die normale Attributsyntax so verwenden:

```py
my_circle.radius # This will call the getter
my_circle.radius = 4 # This will call the setter
```

 Beim Setzen eines Werts sollten Sie nicht dem Eigenschaftsnamen selbst einen Wert zuweisen, da dies zu einem `RecursionError` führt. Verwenden Sie einen separaten internen Namen, oft mit einem Unterstrich, um den Wert zu speichern.

- **Deleter**: Nach dem Setzen und Abrufen eines Werts mit Setter und Getter können Sie steuern, wie er mit einem `deleter` gelöscht wird. Ein Deleter führt benutzerdefinierte Logik aus, wenn Sie die `del`-Anweisung für eine Eigenschaft verwenden. Um einen Deleter zu erstellen, verwenden Sie den `@<property_name>.deleter`-Dekorator.

```py
  # Deleter
    @radius.deleter
    def radius(self):
        print("Deleting radius...")
        del self._radius
```

## Was ist Vererbung und wie fördert sie die Wiederverwendung von Code?

- **Vererbung**: Der Prozess, bei dem eine Kindklasse die Attribute und Methoden einer Elternklasse verwendet. Vererbung fördert die Wiederverwendung von Code, schafft klare Hierarchien und ermöglicht die Anpassung des Verhaltens, ohne alles neu schreiben zu müssen. Um Vererbung zu implementieren, nimmt eine Kindklasse den Namen einer Elternklasse an:

```py
class Parent:
    # Parent attributes and methods

class Child(Parent):
    # Child inherits, extends, and/or overrides where necessary
```

- **Einfache und Mehrfachvererbung**: Wenn eine Kindklasse Eigenschaften und Methoden von nur einer Elternklasse erbt, wie oben gezeigt, nennt man das **einfache Vererbung**. Wenn eine Kindklasse Eigenschaften und Methoden von mehr als einer Elternklasse erbt, ist das **Mehrfachvererbung**. Hier ist die Syntax dafür:

```py
class Parent:
    # Attributes and methods for Parent

class Child:
    # Attributes and methods for Child

class GrandChild(Parent, Child):
    # GrandChild inherits from both Parent and Child
    # GrandChild can combine or override behavior from each
```

- **`super()`-Funktion**: Eine Funktion, mit der Sie eine Methode einer Elternklasse **aufrufen** können, wenn eine Klasse eine andere Implementierung dieser Methode hat oder die Methode erweitert, ohne Code zu duplizieren.

## Was ist Polymorphismus und wie fördert er die Wiederverwendung von Code?

- **Polymorphismus**: Der OOP-Grundsatz, der es verschiedenen Klassen erlaubt, denselben Methodennamen zu verwenden, wobei jede Klasse die Methode unterschiedlich implementiert, wenn sie aufgerufen wird. Hier ist die Syntax dafür:

```py
class A:
   def action(self): ...

class B:
   def action(self): ...

class C:
   def action(self): ...

Class().method()  # Works for A, B, or C
```

- **Vererbungsbasierter Polymorphismus**: Eine Elternklasse definiert eine Methode, und jede Kindklasse passt sie für ihre Zwecke an.

## Was ist Name Mangling und wie funktioniert es?

- **Name Mangling**: Ein Prozess, bei dem Python intern ein Attribut mit doppeltem Unterstrich-Präfix umbenennt, indem es einen Unterstrich und den Klassennamen als Präfix hinzufügt, wodurch `__attribute` zu `_ClassName__attribute` wird.
- **Zweck des Name Mangling**: Der Hauptzweck des Name Mangling ist es, versehentliches Überschreiben von Attributen und Methoden bei Verwendung von Vererbung zu verhindern. Hier ist ein Code, der das verständlicher macht:

```py
class Parent:
    def __init__(self):
        self.__data = 'Parent data'

class Child(Parent):
    def __init__(self):
        super().__init__()
        self.__data = 'Child data'

c = Child()
print(c.__dict__) # {'_Parent__data': 'Parent data', '_Child__data': 'Child data'}
```

## Was ist Abstraktion und wie hilft sie, komplexe Systeme organisiert zu halten?

- **Abstraktion**: Ein Programmierkonzept, bei dem komplexe Implementierungsdetails eines Objekts oder Systems verborgen bleiben und nur die wesentlichen Funktionen gezeigt werden. In Python und anderen Programmiersprachen vereinfacht Abstraktion komplexe Systeme durch erhöhte Wiederverwendbarkeit.
- **Beispiel für Abstraktion**: Ein gutes Beispiel für Abstraktion im Alltag ist ein Auto, bei dem Sie nur das Lenkrad, die Pedale und den Schalthebel benutzen, ohne zu wissen, wie der Motor oder die Bremsen funktionieren.
- **Wie Python Abstraktion implementiert**: Python implementiert Abstraktion durch das `abc`-Modul. Das Modul stellt die `ABC`-Klasse (abstrakte Basisklasse) und den `@abstractmethod`-Dekorator bereit. Eine abstrakte Basisklasse (ABC) definiert die gemeinsamen Methoden und Eigenschaften, die Unterklassen implementieren müssen. Sie kann nicht instanziiert werden.
- **Wie eine abstrakte Methode definiert wird**: Eine abstrakte Methode wird mit `@abstractmethod` definiert und muss in Unterklassen überschrieben werden, auch wenn sie eine Standardimplementierung hat. Die grundlegende Syntax der Abstraktion sieht so aus:

```py
from abc import ABC, abstractmethod

# Define an abstract base class
class AbstractClass(ABC):
    @abstractmethod
    def abstract_method(self):
        pass

# Concrete subclass that implements the abstract method
class ConcreteClassOne(AbstractClass):
    def abstract_method(self):
        print('Implementation in ConcreteClassOne')

# Another concrete subclass
class ConcreteClassTwo(AbstractClass):
    def abstract_method(self):
        print('Implementation in ConcreteClassTwo')
```
