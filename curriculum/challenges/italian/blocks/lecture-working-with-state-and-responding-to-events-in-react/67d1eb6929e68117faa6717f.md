# --description--

In React, il rendering è il processo con cui i componenti appaiono nell’interfaccia utente (UI), di solito nel browser.

React prende tutto il tuo codice JavaScript, JSX e CSS, capisce come dovrebbe apparire e poi lo mostra nell’interfaccia utente.

Il processo di rendering si compone di tre fasi: trigger, render e commit. Vediamole più nel dettaglio.

La fase di trigger avviene quando React rileva che qualcosa è cambiato e che l’interfaccia utente potrebbe aver bisogno di essere aggiornata. Questo cambiamento è spesso dovuto a un aggiornamento dello stato o delle props.

Per esempio, accorgersi che è ora di cena può farti andare in cucina per iniziare a cucinare.

Nell’esempio `Counter` qui sotto, cliccare il pulsante di incremento o decremento fa scattare React per mostrare il nuovo valore `count`:

```jsx
import { useState } from "react";

function Counter() {
  const [count, setCount] = useState(0);

  return (
    <div>
      <h1>{count}</h1>
      <button onClick={() => setCount(count - 1)}>Decrement</button>
      <button onClick={() => setCount(count + 1)}>Increment</button>
    </div>
  );
}

export default Counter;
```

Una volta che il trigger avviene, React entra nella fase di render. Qui React rivaluta i tuoi componenti e decide cosa mostrare.

Per farlo, React usa una copia leggera del DOM “reale”. Questa si chiama virtual DOM. Con il virtual DOM, React può rapidamente controllare cosa deve cambiare nel componente.

Pensa a questa fase come al momento in cui sei in cucina, hai raccolto gli ingredienti e stai cucinando la cena.

Per il componente `Counter`, la fase di render è il momento in cui React esegue di nuovo le funzioni con il nuovo valore `count`. React ricalcola come dovrebbe apparire la parte `<h1>{count}</h1>` del componente basandosi sul valore `count` aggiornato, ma non vedrai alcun cambiamento sullo schermo fino alla fase successiva – commit.

La fase di commit è quella in cui React prende le modifiche preparate nel virtual DOM e le applica al DOM reale. In altre parole, è la fase in cui vedi il risultato finale sullo schermo.

Per farlo, React confronta il virtual DOM con il DOM reale, individua solo le parti che devono essere aggiornate e applica quei cambiamenti al DOM reale per aggiornare l’interfaccia utente.

Puoi pensare a questa fase come al momento in cui servi il cibo che hai cucinato, rendendolo visibile proprio come fa React quando applica gli aggiornamenti al DOM reale.

Per quanto riguarda il componente `Counter`, la fase di commit è il momento in cui il nuovo valore `count` viene applicato all’elemento `h1` e puoi vedere il cambiamento sulla pagina.

Questi tre processi sono estremamente veloci perché React minimizza la manipolazione diretta del DOM calcolando prima le modifiche nel virtual DOM, poi aggiorna solo le parti che devono cambiare nel DOM reale.
