# --interactive--

تسلسل الطرق هو تقنية تستدعي فيها عدة طرق واحدة تلو الأخرى. يمكنك استخدام تسلسل الطرق على أنواع كثيرة من القيم في JavaScript بما في ذلك السلاسل النصية، والمصفوفات، والكائنات. رغم أن السلاسل النصية قيم بدائية، إلا أن JavaScript تغلفها مؤقتًا في كائن `String` عند استخدام طريقة من طرق السلسلة النصية.

لننظر إلى مثال يستخدم طرقًا مختلفة للسلاسل النصية:

:::interactive_editor

```js
const result = "  Hello, World!  "
  .trim()
  .toLowerCase()
  .replace("world", "JavaScript");

console.log(result); // "hello, JavaScript!"
```

:::

في هذا المثال، نبدأ بسلسلة نصية وننفذ ثلاث عمليات متتالية: مسح الفراغات، تحويل الأحرف إلى أحرف صغيرة، واستبدال السلسلة النصية `world` بـ `JavaScript`. كل طريقة تُرجع سلسلة نصية جديدة تصبح هدف الاستدعاء التالي للطريقة.

يمكن لتسلسل الطرق أن يحسن بشكل كبير من قابلية قراءة الشفرة عند العمل مع عمليات معقدة.

على سبيل المثال، انظر هذا المثال الذي يستخدم طرق `filter` و `map` و `reduce`:

:::interactive_editor

```js
const transactions = [
  { amount: 100, type: "credit" },
  { amount: 20, type: "cash" },
  { amount: 150, type: "credit" },
  { amount: 50, type: "cash" },
  { amount: 75, type: "credit" }
];

const totalCreditWithBonus = transactions
  .filter((transaction) => transaction.type === "credit")
  .map((transaction) => transaction.amount * 1.1)
  .reduce((sum, amount) => sum + amount, 0);

console.log(totalCreditWithBonus); // 357.5
```

:::

في هذا المثال، لدينا مصفوفة من المعاملات حيث يحتوي كل كائن على مبلغ ونوع بطاقة ائتمان أو نقدي.

نبدأ بتصفية المعاملات لإنشاء مصفوفة جديدة تحتوي فقط على معاملات بطاقة الائتمان. ثم نُسلسل طريقة `map` على النتيجة المصفاة ولكل مبلغ معاملة نضربه بـ `1.1` الذي يمثل مكافأة بنسبة `10%`.

بعد ذلك، نأخذ تلك النتيجة ونسلسل طريقة `reduce` لجمع كل المبالغ، مما ينتج عنه `357.5`.

بينما يمكن لتسلسل الطرق أن يجعل الشفرة أكثر إيجازًا وقابلية للقراءة، من المهم استخدامه بحكمة.

يمكن أن تصبح السلاسل الطويلة جدًا صعبة في تصحيح الأخطاء، إذ لا يتضح فورًا أي خطوة في السلسلة قد تسبب المشكلة. من الممارسات الجيدة غالبًا تقسيم السلاسل الطويلة إلى خطوات متعددة لزيادة الوضوح وتسهيل تصحيح الأخطاء.

يمكنك أيضًا تسلسل الطرق على كائن. في هذه الحالة، تُرجع كل طريقة `this`، التي تشير إلى الكائن الحالي، مما يسمح للطريقة التالية في السلسلة بالتنفيذ عليه.

:::interactive_editor

```js
const calculator = {
  total: 0,
  add(n) {
    this.total += n;
    return this;
  },
  multiply(n) {
    this.total *= n;
    return this;
  },
  subtract(n) {
    this.total -= n;
    return this;
  },
  getResult() {
    return this.total;
  }
};

const result = calculator.add(5).multiply(2).subtract(3).getResult();
console.log(result); // 7
```

:::

بهذه الطريقة، تُرجع كل طريقة نفس الكائن، لذا يمكنك الاستمرار في تسلسل الاستدعاءات واحدة تلو الأخرى.
