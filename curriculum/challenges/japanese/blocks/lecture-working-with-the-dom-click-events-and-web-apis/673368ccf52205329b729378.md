# --interactive--

`innerText`、`textContent`、`innerHTML`について学びましょう。

これらはJavaScriptでアクセスできるプロパティで、HTML要素の内容を取得したり変更したりできます。最初は非常に似ているように見えますが、重要な違いがあります。どれを選ぶかは具体的な用途によるので、詳しく見ていきましょう。

まずは`innerText`から始めます。

`innerText`はHTML要素とその子孫要素の可視テキスト内容を表します。このプロパティは非表示のテキストやHTMLタグは含まず、レンダリングされたテキストのみを含みます。

例えば、ここに2つの段落を含む`div`要素があります：

```html
<div id="container">
  <p>Hello, World!</p>
  <p>I'm learning JavaScript</p>
</div>
```

JavaScriptコードでこのHTML要素の参照を`getElementById()`を使って取得すると、この要素の`innerText`プロパティにアクセスできます：

:::interactive_editor

```html
<div id="container">
  <p>Hello, World!</p>
  <p>I'm learning JavaScript</p>
</div>
<script src="index.js"></script>
```

```js
const container = document.getElementById("container");
console.log(container.innerText);
```

:::

これはこの要素の内部テキストです：

```md
Hello, World!
I'm learning JavaScript
```

このプロパティは、要素内に含まれるテキストを文字列として返します。子孫要素のテキストも含まれます。

`innerText`は文字列が要求された時点で表示されているテキストのみを返すことを知っておいてください。子要素が非表示の場合、そのテキストは表示されません。

これは2番目の段落が非表示になっている例です：

:::interactive_editor

```html
<div id="container">
  <p>Hello, World!</p>
  <p hidden>I'm learning JavaScript</p>
</div>
```

:::

`innerText`を再度ログに出力すると、今度は2番目の段落のテキストが含まれていません：

:::interactive_editor

```html
<div id="container">
  <p>Hello, World!</p>
  <p hidden>I'm learning JavaScript</p>
</div>
<script src="index.js"></script>
```

```js
const container = document.getElementById("container");
console.log(container.innerText);
```

:::

HTML要素の`innerText`をこのように設定できますが、これは既存のテキストを置き換え、改行ごとに改行要素（`br`）を追加します：

:::interactive_editor

```html
<div id="container">
  <p>Hello, World!</p>
  <p>I'm learning JavaScript</p>
</div>
<script src="index.js"></script>
```

```js
const container = document.getElementById("container");
container.innerText = "JavaScript is awesome!";
```

:::

`innerText`は可視性を考慮するため、その値を取得すると「リフロー」と呼ばれる処理が発生し、ウェブサイト上の特定の要素の位置が再計算されます。この処理は計算負荷が高いため、可能な限りトリガーしないようにしてください。

では、次に`textContent`について話しましょう。

`textContent`は要素のプレーンテキスト内容を返し、子孫要素内のすべてのテキストを含みます。

`innerText`と`textContent`の最も重要な違いは、`textContent`は要素とその子孫のテキスト内容を常に完全に返すことで、表示されているか非表示かは関係ありません。

ここに同じHTMLの例があります：

```html
<div id="container">
  <p>Hello, World!</p>
  <p>I'm learning JavaScript</p>
</div>
```

このプロパティにアクセスすると、インデントやスペースを保持したまま要素と子孫のテキストが出力されます：

:::interactive_editor

```html
<div id="container">
  <p>Hello, World!</p>
  <p>I'm learning JavaScript</p>
</div>
<script src="index.js"></script>
```

```js
const container = document.getElementById("container");
console.log(container.textContent);
```

:::

HTML要素が非表示の場合でも、ここで見られるように2番目の段落を非表示にしても、そのテキストはこのプロパティに含まれます：

:::interactive_editor

```html
<div id="container">
  <p>Hello, World!</p>
  <p hidden>I'm learning JavaScript</p>
</div>
<script src="index.js"></script>
```

```js
const container = document.getElementById("container");
console.log(container.textContent);
```

:::
  
`textContent`は`script`や`style`のような要素の内容も含みます。

ノードの`textContent`の値を置き換えようとすると、すべての子ノードが削除され、新しい文字列を含む単一のテキストノードに置き換えられます：

:::interactive_editor

```html
<div id="container">
  <p>Hello, World!</p>
  <p hidden>I'm learning JavaScript</p>
</div>
<script src="index.js"></script>
```

```js
const container = document.getElementById("container");
container.textContent = "New content";
```

:::

最後に、`textContent`と`innerText`が`innerHTML`とどう違うか話しましょう。

`innerHTML`を使うと要素の内部HTML内容を設定できます。これはDOMに新しいHTMLを動的に挿入するのに便利です。

ただし、文字列の内容を制御できない場合、例えばユーザーが入力したデータを含む文字列の場合はセキュリティリスクがあります。そのデータが悪意のあるものであれば、深刻なセキュリティ問題につながる可能性があります。

これを避けるために、プレーンテキストを挿入するには`textContent`プロパティを使うことをお勧めします。

JavaScriptの`innerText`、`textContent`、`innerHTML`プロパティは、HTML要素の内容にアクセスし操作する異なる方法を提供します。HTMLコンテンツをJavaScriptで効果的に扱うには、これらのプロパティの違いを理解することが重要です。
