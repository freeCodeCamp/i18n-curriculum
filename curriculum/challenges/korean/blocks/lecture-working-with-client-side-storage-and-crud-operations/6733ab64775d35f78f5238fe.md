# --description--

CRUD는 Create, Read, Update, Delete의 약자입니다. 이 네 가지는 영구 저장소의 기본 작업입니다.

영구 저장소는 전원이 꺼지거나 장치가 재시작된 후에도 데이터를 사용할 수 있도록 저장하는 방식을 말합니다.

CRUD의 기본 작업이 어떻게 작동하는지 이해하는 것은 데이터베이스를 다루고 사용자가 데이터를 추가, 조회, 수정, 삭제할 수 있는 애플리케이션을 만드는 데 중요한 웹 개발의 기초가 됩니다.

이제 CRUD의 각 부분을 자세히 살펴보겠습니다.

- Create는 새로운 데이터를 생성하는 과정을 의미합니다. 예를 들어, 웹 애플리케이션에서 사용자가 블로그에 새 게시물을 추가할 때입니다.

- Read는 데이터베이스에서 데이터를 조회하는 작업입니다. 예를 들어, 블로그 게시물을 방문하거나 웹사이트에서 프로필을 볼 때 데이터베이스에 저장된 데이터를 가져와 표시하는 작업입니다.

- Update는 데이터베이스에 있는 기존 데이터를 수정하는 작업입니다. 예를 들어, 블로그 게시물을 편집하거나 프로필 정보를 업데이트하는 경우입니다.

- Delete는 데이터베이스에서 데이터를 삭제하는 작업입니다. 예를 들어, 블로그 게시물이나 계정을 삭제할 때 수행하는 작업입니다.

CRUD는 데이터베이스, UI, RESTful API를 다룰 때 사용됩니다. RESTful API는 클라이언트가 표준 HTTP 메서드를 통해 데이터베이스나 백엔드 시스템과 CRUD 작업을 수행할 수 있도록 하는 웹 서비스 구축 관례입니다.

HTTP는 하이퍼텍스트 전송 프로토콜의 약자로, 웹에서 데이터 통신의 기반입니다. HTTP 메서드는 웹상의 자원에 대해 수행할 수 있는 작업을 정의합니다. 일반적인 메서드는 `GET`, `POST`, `PUT`, `PATCH`, `DELETE`입니다.

RESTful API와 HTTP에 대해서는 앞으로 수업에서 더 배우겠지만, 여기 CRUD가 각 HTTP 메서드에 어떻게 대응하는지 간단히 정리했습니다.

- `POST`는 새 자원을 생성할 때 사용합니다.

- `GET`는 데이터를 조회하거나 읽을 때 사용합니다.

- `PUT`는 자원을 완전히 교체하여 업데이트할 때 사용합니다.

- `PATCH`는 자원을 부분적으로 업데이트할 때 사용합니다.

- `DELETE`는 자원을 삭제할 때 사용합니다.

다음은 JavaScript의 간단한 배열을 사용해 CRUD 작업이 코드에서 어떻게 나타날 수 있는지 예시입니다.

```js
let items = [];

// Create
function createItem(item) {
  items.push(item);
}

// Read
function readItems() {
  return items;
}

// Update
function updateItem(index, newItem) {
  items[index] = newItem;
}

// Delete
function deleteItem(index) {
  items.splice(index, 1);
}

// Example Usage
createItem('Book');
console.log(readItems()); // ['Book']
updateItem(0, 'Magazine');
console.log(readItems()); // ['Magazine']
deleteItem(0);
console.log(readItems()); // []
```

이 예시에서:

- 배열에 항목을 추가하여 생성합니다.

- 배열을 반환하여 항목을 읽습니다.

- 주어진 인덱스의 배열 요소를 수정하여 업데이트합니다.

- `splice()`를 사용해 배열에서 항목을 삭제합니다.

이것은 CRUD 작업이 개념적으로 어떻게 작동하는지 기본적으로 나타낸 것입니다.
