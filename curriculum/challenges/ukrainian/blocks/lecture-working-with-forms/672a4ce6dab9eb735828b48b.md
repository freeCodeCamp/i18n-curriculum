# --interactive--

Коли користувач заповнює форму на вашому сайті, важливо, щоб він заповнив усю необхідну інформацію у правильному форматі. Елементи керування формою HTML, як-от поля введення, мають багато вбудованої валідації, яку ви можете використати для перевірки некоректних даних. Це допоможе переконатися, що користувач виправить ці помилки до того, як інформація буде відправлена та оброблена сервером.

Термін «на стороні клієнта» означає все, що відбувається на комп’ютері або пристрої користувача, наприклад, ту частину сайту чи застосунку, з якою ви взаємодієте безпосередньо. Це включає макет, дизайн і будь-які інтерактивні функції.

Термін «на стороні сервера» означає все, що відбувається на сервері, комп’ютері або системі, що хостить сайт чи застосунок. Це включає обробку даних, запуск застосунків і обробку запитів, які надходять із пристрою користувача.

Хоча валідація на стороні клієнта важлива, вам також потрібна валідація на стороні сервера для додаткової безпеки. Зловмисники можуть обійти перевірки на стороні клієнта, тому необхідні надійні заходи на стороні сервера. Ви дізнаєтеся більше про це в одному з наступних модулів. А поки давайте розглянемо кілька прикладів валідації форми на стороні клієнта.

Одним із загальних прикладів вбудованої валідації форми є використання атрибута `required` у полях введення. Атрибут `required` вказує, що користувач повинен заповнити цю частину форми перед відправленням. Ось приклад використання атрибута `required` у полі введення для е-пошти.

Натисніть на кнопку `Submit Form` без введення адреси е-пошти, і ви побачите повідомлення, яке підказує заповнити це поле.

:::interactive_editor

```html
<form action="">
  <label for="email">Email Address (Required field):</label>
  <input required type="email" name="email" id="email" />
  <button type="submit">Submit Form</button>
</form>
```

:::

Кожен браузер має власний набір стилів для відображення цього повідомлення-попередження.

Ще одна перевага використання поля для е-пошти полягає в тому, що такі поля мають базову валідацію для перевірки правильності формату адреси е-пошти. Наприклад, якщо ви введете випадкові слова і натиснете кнопку відправки, браузер покаже попередження, що відсутній знак `@`.

Введіть `abc` у поле для е-пошти і натисніть кнопку відправки. Ви побачите повідомлення, бо це не є дійсною адресою е-пошти.

:::interactive_editor

```html
<form action="">
  <label for="email">Email Address (Required field):</label>
  <input required type="email" name="email" id="email" />
  <button type="submit">Submit Form</button>
</form>
```

:::

Важливо зазначити, що браузери перевіряють лише базову валідацію стандартних адрес е-пошти. Додаткові рівні валідації потрібно додавати самостійно, про що ви дізнаєтеся в наступних модулях.

Інші способи валідації для полів е-пошти — це використання атрибутів `minlength` та `maxlength`. Ось приклад із додатковою валідацією.

Введіть `b@m` у поле і натисніть кнопку відправки. Ви побачите повідомлення, бо це не відповідає мінімальній необхідній довжині.

:::interactive_editor

```html
<form action="">
  <label for="email">Email Address (Required field):</label>
  <input
    required
    type="email"
    name="email"
    id="email"
    minlength="4"
    maxlength="64"
  />
  <button type="submit">Submit Form</button>
</form>
```

:::

Атрибути `minlength` і `maxlength` використовуються для встановлення мінімальної та максимальної довжини в символах для поля е-пошти. Якщо ви не вкажете мінімальну довжину або перевищите максимальну довжину символів, браузер покаже повідомлення-попередження.
