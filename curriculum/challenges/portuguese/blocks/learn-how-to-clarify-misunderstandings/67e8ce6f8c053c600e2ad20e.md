# --explanation--

`Not exactly` é uma forma educada de dizer que a ideia ou suposição de alguém está próxima mas não está totalmente correta.É frequentemente usado antes de fornecer uma explicação mais clara ou mais precisa.Por exemplo:

- Pessoa 1: `Is encapsulation just about hiding code?`

- Pessoa 2: `Not exactly, it's also about organizing and protecting it.` - Isso mostra como `not exactly` é usado para corrigir uma afirmação de forma suave enquanto adiciona mais detalhes.

`Inheritance` é um conceito chave em programação orientada a objetos.Ele permite que uma classe (a filha ou subclasse) receba as propriedades e métodos de outra classe (a pai ou superclasse).Isso suporta a reutilização de código, o que significa que você não precisa reescrever a mesma lógica em vários lugares e também facilita estender ou construir sobre a funcionalidade existente.Por exemplo:

`Inheritance allows the "Employee" class to reuse code from the "Person" class, like name and email methods.` - Isso mostra como a herança ajuda uma classe a obter propriedades e comportamentos de outra, evitando duplicação de código.
