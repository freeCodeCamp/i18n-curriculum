# --interactive--

クロージャはJavaScriptで最も強力でありながら、誤解されやすい機能の一つです。基本的に、クロージャとは、外側の関数が戻った後でも、その外側の囲むレキシカルスコープ内の変数にアクセスできる関数のことです。これは複雑に聞こえるかもしれませんが、JavaScriptで多くの高度なプログラミングパターンを可能にする基本的な概念です。

クロージャを理解するために、まず例を見てみましょう。

:::interactive_editor

```js
function outerFunction(x) {
    let y = 10;
    function innerFunction(){
        console.log(x + y);
    }
    return innerFunction;
}

let closure = outerFunction(5);
console.log(closure()); // 15
```

:::

この例では、`outerFunction`がパラメータ`x`を受け取り、ローカル変数`y`を定義しています。次に`innerFunction`と`x`の両方を使う`y`を定義し、最後に`innerFunction`を返します。`outerFunction(5)`を呼び出すと`innerFunction`を返し、それを変数`closure`に代入します。後で`closure()`を呼び出しても、`x`の実行が終了しているにもかかわらず、`y`の`outerFunction`と`outerFunction`にアクセスできます。これがクロージャの本質です。

内側の関数は外側のレキシカル環境への参照を保持し、外側の関数が完了した後でもその環境内の変数にアクセスし続けます。

クロージャは特にプライベートな変数や関数を作成するのに便利です。次の例を考えてみましょう。

:::interactive_editor

```js
function createCounter() {
    let count = 0;
    return function () {
        count++;
        return count;
    };
}

let counter = createCounter();
console.log(counter()); // 1
console.log(counter()); // 2
```

:::

この場合、`createCounter`は`count`変数をインクリメントして返す関数を返します。`count`変数は`createCounter`の外部から直接アクセスできませんが、返された関数（クロージャ）はそれにアクセスできます。`counter()`を呼び出すたびに、`count`がインクリメントされて返されます。

クロージャは外側のスコープから複数の変数をキャプチャすることもできます。例えば：

:::interactive_editor

```js
function multiply(x) {
    return function (y) {
        return x * y;
    };
}

let double = multiply(2);
console.log(double(5)); // 10
```

:::

ここで内側の関数は`x`のパラメータ`multiply`をキャプチャしています。`double`を呼び出して`multiply(2)`を作成すると、常に引数に`2`を掛ける関数を返します。

クロージャについて重要な点は、変数を値ではなく参照でキャプチャすることです。つまり、キャプチャされた変数の値が変わると、クロージャは新しい値を参照します。例えば：

:::interactive_editor

```js
function createIncrementer() {
    let count = 0;
    return function () {
        count++;
        console.log(count);
    };
}

let increment = createIncrementer();
increment(); // 1
increment(); // 2
```

:::

`increment`を呼び出すたびに、初期値のコピーではなく同じ`count`変数を操作しています。クロージャはJavaScriptで強力なツールです。JavaScriptを使い続ける中で、クロージャを理解し効果的に使うことが、クリーンで効率的かつ強力なコードを書く能力を大きく高めることがわかるでしょう。
