## CSS 기초

- **CSS란 무엇인가요?**: CSS(Cascading Style Sheets)는 HTML 요소에 스타일을 적용하는 데 사용하는 마크업 언어입니다. CSS는 색상, 배경 이미지, 레이아웃 등 다양한 스타일에 사용됩니다.
- **CSS 규칙의 기본 구조**: CSS 규칙은 선택자와 선언 블록 두 가지 주요 부분으로 구성됩니다. 선택자는 CSS에서 특정 HTML 요소를 식별하고 스타일을 적용하기 위해 사용하는 패턴입니다. 선언 블록은 주어진 선택자 또는 선택자들에 대해 스타일 집합을 적용합니다.
- **`meta name="viewport"` 요소**: 이 `meta` 요소는 브라우저에 페이지 크기와 다양한 기기, 특히 휴대폰과 태블릿에서의 확대/축소 방식을 제어하는 방법을 지시합니다.
- **기본 브라우저 스타일**: 각 HTML 요소에는 기본 브라우저 스타일이 적용됩니다. 보통 기본 여백과 패딩 같은 항목이 포함됩니다.

## 인라인, 내부, 외부 CSS

- **인라인 CSS**: 이 스타일은 HTML 요소 내에서 `style` 속성을 사용해 직접 작성합니다. 보통 관심사의 분리 때문에 인라인 CSS를 사용하지 않습니다.  
- **내부 CSS**: 이 스타일은 HTML 문서의 `<style>` 섹션 안에 있는 `head` 태그 내에 작성합니다. 짧은 코드 예제를 만들 때 유용하지만 보통 내부 CSS는 사용하지 않습니다.  
- **외부 CSS**: 이 스타일은 별도의 CSS 파일에 작성하고 `link` 섹션의 `head` 요소를 사용해 HTML 문서에 연결합니다. 대부분의 프로젝트에서는 내부나 인라인 CSS보다 외부 CSS 파일을 사용합니다.

## `width`와 `height` 속성 다루기

- **`width` 속성**: 이 속성은 요소의 너비를 지정합니다. 너비를 지정하지 않으면 기본적으로 `auto`로 설정됩니다. 이는 브라우저가 요소의 내용, 부모, 그리고 표시 유형에 따라 너비를 결정하도록 합니다.
- **`min-width` 속성**: 이 속성은 요소의 최솟값 너비를 지정합니다.
- **`max-width` 속성**: 이 속성은 요소의 최대 너비를 지정합니다.
- **`height` 속성**: 이 속성은 요소의 높이를 지정합니다. 마찬가지로, 높이는 기본적으로 `auto`로 설정되어 내부 내용에 맞춰 조정됩니다.
- **`min-height` 속성**: 이 속성은 요소의 최솟값 높이를 지정합니다.
- **`max-height` 속성**: 이 속성은 요소의 최대 높이를 지정합니다.

## CSS 결합자 유형

- **자손 결합자 (` `)&nbsp;\*선택자 사이에 공백 한 칸**: 이 결합자는 지정한 부모 요소의 자손 요소를 선택할 때 사용합니다.
- **자식 결합자 (`>`)**: 이 결합자는 지정한 부모 요소의 직접 자식 요소를 선택할 때 사용합니다.
- **다음 형제 결합자 (`+`)**: 이 결합자는 지정한 형제 요소 바로 다음에 오는 요소를 선택합니다.

- **이후 형제 결합자 (`~`)**: 이 결합자는 지정한 요소 이후에 나오는 모든 형제 요소를 선택합니다.

## 인라인, 블록, 인라인-블록 레벨 요소

- **인라인 레벨 요소**: 인라인 요소는 필요한 만큼의 너비만 차지하며 새 줄에서 시작하지 않습니다. 이 요소들은 콘텐츠 내에서 흐르며 텍스트나 다른 인라인 요소가 옆에 함께 나타날 수 있습니다. 일반적인 인라인 요소로는 `span`, `anchor`, `img` 요소가 있습니다.
- **블록 레벨 요소**: 블록 레벨 요소는 기본적으로 컨테이너의 전체 너비를 차지하며 가로 전체에 걸쳐 늘어납니다. 흔한 블록 레벨 요소로는 `div`, `p`, `section` 요소가 있습니다.
- **인라인-블록 레벨 요소**: `inline-block` 속성을 사용해 요소를 `display`로 설정할 수 있습니다. 이 요소들은 인라인 요소처럼 동작하지만 블록 레벨 요소처럼 `width`와 `height`을 설정할 수 있습니다.

## 여백과 패딩

- **`margin` 속성**: 이 속성은 요소의 테두리와 주변 요소 사이, 요소 바깥에 공간을 적용할 때 사용합니다.
- **`margin` 축약형**: 1~4개의 값을 지정해 여백의 각 면을 설정할 수 있습니다. 한 값은 네 면 모두에 적용되고, 두 값은 `top`와 `bottom`에, 그다음 `right`와 `left`에 적용됩니다. 세 값은 `top`, 수평(`right`와 `left`), 그리고 `bottom`에 적용됩니다. 네 값은 `top`, `right`, `bottom`, `left`에 각각 적용됩니다.
- **`padding` 속성**: 이 속성은 요소 안쪽, 콘텐츠와 테두리 사이에 공간을 적용할 때 사용합니다.
- **`padding` 축약형**: 1~4개의 값을 지정해 패딩의 각 면을 설정할 수 있습니다. 한 값은 네 면 모두에 적용되고, 두 값은 `top`와 `bottom`에, 그다음 `right`와 `left`에 적용됩니다. 세 값은 `top`, 수평(`right`와 `left`), 그리고 `bottom`에 적용됩니다. 네 값은 `top`, `right`, `bottom`, `left`에 각각 적용됩니다.

## CSS 특이성

- **인라인 CSS 특이성**: 인라인 CSS는 요소에 직접 적용되기 때문에 가장 높은 특이성을 가집니다. 내부 또는 외부 CSS를 재정의합니다. 인라인 스타일의 특이성 값은 (1, 0, 0, 0)입니다.
- **내부 CSS 특이성**: 내부 CSS는 HTML 문서의 `style` 섹션 내 `head` 요소 안에 정의됩니다. 인라인 스타일보다 낮은 특이성을 가집니다. 내부 CSS와 외부 CSS는 위치가 아니라 선택자에 의해 결정되는 동일한 특이성 수준을 공유합니다.
- **외부 CSS 특이성**: 외부 CSS는 `link` 섹션의 `head` 요소를 통해 연결되며 별도의 `.css` 파일에 작성됩니다. 내부 CSS와 마찬가지로 선택자에 의해 특이성이 결정됩니다. 두 규칙의 특이성이 같으면 소스 순서가 우선순위를 결정하며, 문서에서 나중에 나타나는 규칙이 우선합니다. 외부 CSS는 대규모 프로젝트에서 유지보수에 가장 적합합니다.
- **범용 선택자 (`*`)**: 문서 내 모든 요소와 일치하는 특별한 유형의 CSS 선택자입니다. 페이지의 모든 요소에 스타일을 적용할 때 자주 사용되며, 브라우저 간 스타일을 초기화하거나 표준화하는 데 유용합니다. 범용 선택자는 모든 선택자 중 가장 낮은 특이성 값을 가지며, 특이성 값의 모든 부분에 0을 기여합니다 (0, 0, 0, 0).
- **타입 선택자**: 태그 이름을 기준으로 요소를 선택합니다. 타입 선택자는 다른 선택자에 비해 상대적으로 낮은 특이성을 가집니다. 타입 선택자의 특이성 값은 (0, 0, 0, 1)입니다.
- **클래스 선택자**: 점(`.`) 뒤에 클래스 이름이 오는 선택자입니다. 클래스 선택자의 특이성 값은 (0, 0, 1, 0)입니다. 이는 클래스 선택자가 타입 선택자를 재정의할 수 있지만, ID 선택자와 인라인 스타일에 의해 재정의될 수 있음을 의미합니다.
- **ID 선택자**: 해시 기호(`#`) 뒤에 ID 이름이 오는 선택자입니다. ID 선택자는 타입 선택자와 클래스 선택자보다 훨씬 높은 특이성을 가지지만, 인라인 스타일보다는 낮습니다. ID 선택자의 특이성 값은 (0, 1, 0, 0)입니다.
- **`!important` 키워드**: 스타일 규칙에 가장 높은 우선순위를 부여하여 해당 속성에 대한 다른 선언을 모두 재정의하게 합니다. 사용 시, 다른 선택자의 특이성에 관계없이 브라우저가 지정된 스타일을 적용하도록 강제합니다. `!important` 사용은 CSS 유지보수와 디버그를 어렵게 만들 수 있으므로 주의해야 합니다.
- **캐스케이드 알고리즘**: 동일한 요소를 대상으로 하는 여러 스타일이 있을 때 어떤 CSS 규칙을 적용할지 결정하는 알고리즘입니다. 명확한 규칙 집합에 따라 가장 적절한 스타일이 사용되도록 보장합니다.
- **CSS 상속**: 스타일이 부모 요소에서 자식 요소로 전달되는 과정입니다. 상속을 통해 문서 트리의 상위 수준에서 스타일을 정의하고, 여러 요소에 명시적으로 지정하지 않아도 적용할 수 있습니다.

## 디자인 용어

- **레이아웃**: 메시지를 전달하기 위해 페이지나 화면에 시각적 요소를 배치하는 방식입니다. 이 요소에는 텍스트, 이미지, 여백이 포함될 수 있습니다.
- **정렬**: 요소들이 서로 어떻게 배치되는지를 의미합니다. 정렬을 올바르게 사용하면 디자인이 깔끔하고 정돈된 느낌을 줍니다.
- **구성**: 조화를 이루는 디자인을 만들기 위해 요소를 배치하는 행위입니다. 이미지, 텍스트, 도형 같은 요소들이 서로 어떻게 관계를 맺고 예술적으로 디자인에 기여하는지를 결정합니다.
- **균형**: 구성 내에서 시각적 무게가 어떻게 분배되는지를 뜻합니다. 디자이너는 대칭 또는 비대칭 배치를 통해 균형을 이루려고 합니다.
- **크기비교**: 한 요소의 크기나 치수를 다른 요소와 비교하는 것을 말합니다.
- **계층 구조**: 디자인 요소의 중요도 순서를 정하는 것입니다. 가장 중요한 정보가 먼저 눈에 띄도록 만드는 것이 목적입니다.
- **대비**: 요소들 사이에 명확한 구분을 만드는 과정입니다. 색상, 크기, 모양, 질감 또는 기타 시각적 특성의 변화를 통해 할 수 있습니다. 강한 대비는 가독성 향상에도 도움이 됩니다.
- **여백 (부정 공간)**: 디자인에서 비어 있는 공간입니다. 요소들을 둘러싼 영역을 말합니다.
- **UI (사용자 인터페이스)**: 아이콘, 이미지, 텍스트, 메뉴, 링크, 버튼 등 사용자가 화면에서 볼 수 있는 시각적이고 상호작용하는 요소를 포함합니다.
- **UX (사용자 경험)**: 사용자가 제품이나 서비스를 사용할 때 느끼는 경험입니다. 잘 설계된 사용자 경험을 가진 애플리케이션은 직관적이고, 사용하기 쉽고, 효율적이며, 접근 가능하고, 즐거움을 줍니다.
- **디자인 브리프**: 프로젝트의 목표, 목적, 요구사항을 개략적으로 정리한 문서입니다. 디자인 과정을 안내하고 최종 결과물이 클라이언트의 요구를 충족하도록 돕는 로드맵 역할을 합니다.
- **벡터 기반 디자인**: 선, 도형, 색상을 수학적 공식으로 정의하여 디지털 아트를 만드는 방식입니다.
- **프로토타이핑**: 제품이나 사용자 인터페이스의 인터랙티브 모델을 만드는 과정을 말합니다.

## UI 디자인 기초

- **배경색과 전경색의 좋은 명암 대비**: 텍스트를 읽기 쉽게 하려면 배경색과 전경색이 좋은 명암 대비를 가져야 합니다. 웹 콘텐츠 접근성 지침(WCAG)은 일반 텍스트에 대해 최소 명암 대비율 4.5:1, 큰 텍스트에 대해 3:1을 권장합니다.
- **디자인에서 좋은 시각적 계층 구조**: 강력한 시각적 계층 구조는 눈이 따라갈 명확한 경로를 제공하여 전달하려는 정보가 의도한 순서대로 소비되도록 합니다.
- **반응형 이미지**: 반응형 이미지는 화면 크기에 맞게 크기가 조절되는 이미지입니다. 이는 데스크톱부터 모바일폰까지 모든 기기에서 이미지가 잘 보이도록 보장하기 때문에 중요합니다.
- **점진적 향상**: 모든 사용자가 브라우저나 기기에 상관없이 애플리케이션의 필수 콘텐츠와 기능에 접근할 수 있도록 하는 디자인 접근법입니다.
- **사용자 중심 디자인**: 사용자의 요구, 선호, 한계를 우선시하는 접근법입니다. 사용자 중심 디자인의 목표는 직관적이고 효율적이며 사용자가 상호작용하기에 즐거운 웹 페이지를 만드는 것입니다.
- **사용자 조사**: 제품을 사용하는 사람들을 체계적으로 연구하는 것입니다. 목표는 사용자 요구, 행동, 문제점을 측정하는 것입니다.
- **사용자 테스트**: 사용자가 애플리케이션과 상호작용하는 동안 데이터를 수집하는 행위를 말합니다.
- **A/B 테스트**: 새로운 기능을 무작위로 선택된 사용자 부분집합에 배포하는 과정입니다. 그런 다음 분석 데이터를 활용해 기능이 유용한지 판단할 수 있습니다.
- **사용자 요구사항**: 애플리케이션이 따라야 하는 스토리나 기준을 말합니다. 사용자 요구사항은 사용자 조사, 업계 표준, 또는 이해관계자의 의견에 의해 정의될 수 있습니다.
- **점진적 공개**: 사용자의 현재 활동에 따라 관련 콘텐츠만 보여주고 나머지는 숨기는 디자인 패턴입니다. 인지 부하를 줄이고 사용자 경험을 더 직관적으로 만들기 위해 사용합니다.
- **지연/레이지 등록**: 사용자가 등록하지 않고도 애플리케이션을 탐색하고 상호작용할 수 있게 하는 UI 디자인 패턴입니다.

## 모범 사례 디자인

- **다크 모드**: 웹 애플리케이션에서 기본 밝은 색상 체계를 어두운 색상 체계로 변경할 수 있는 특별한 기능입니다. 다크 모드에서는 채도가 낮은 색상을 사용하세요. 채도가 낮은 색상은 강도가 약하고 채도 수준이 낮은 색상입니다.
- **브레드크럼**: 사용자가 사이트 계층 구조에서 현재 위치를 알 수 있도록 돕는 내비게이션 도구입니다. 브레드크럼은 사용자가 쉽게 찾을 수 있도록 페이지 상단에 배치하는 것이 가장 좋습니다. 또한, 브레드크럼은 읽기 쉽도록 충분히 크되, 페이지 공간을 너무 많이 차지하지 않도록 적당한 크기로 만드세요.
- **카드 컴포넌트**: 카드 컴포넌트는 디자인이 단순해야 하며, 시각적으로 복잡하거나 너무 많은 정보를 표시하지 않아야 합니다. 미디어의 경우, 사용자 경험을 향상시키기 위해 고품질 이미지와 비디오를 선택하세요.

- **모달 대화상자**: 기존 페이지 콘텐츠 위에 표시되는 팝업 유형입니다. 보통 배경 콘텐츠에는 어두운 색상 오버레이가 적용되어 사용자가 모달 콘텐츠에 더 집중할 수 있도록 돕습니다. 또한, 사용자가 모달 외부를 클릭해 모달을 닫을 수 있도록 하는 것이 항상 좋은 방법입니다. HTML `dialog` 요소를 사용하면 많은 기능과 접근성 이점을 기본으로 얻을 수 있습니다.
- **폼 등록 진행 표시**: 사용자가 프로세스에서 얼마나 진행했는지 보여주는 방법입니다. 폼, 등록, 설정 과정에서 사용할 수 있습니다. 디자인은 단순하고 찾기 쉬워야 하며, 이전 단계로 돌아갈 수 있도록 해야 합니다.
- **쇼핑 카트**: 전자상거래 플랫폼에서 사용자가 이미 선택한 항목을 볼 수 있는 공간입니다. 카트는 항상 사용자에게 보여야 하며, 카트, 가방, 바구니 같은 일반적인 아이콘을 사용하고, 사용자가 결제 단계로 진행할 수 있도록 명확한 행동 유도 버튼을 포함해야 합니다.

## 절대 단위

- **`px` (픽셀)**: 이 절대 단위는 CSS에서 고정 크기 단위로, 치수를 정확하게 제어할 수 있습니다. 즉, 1px는 항상 1/96인치와 같습니다.
- **`in` (인치)**: 이 절대 단위는 96px와 같습니다.
- **`cm` (센티미터)**: 이 절대 단위는 25.2/64인치와 같습니다.
- **`mm` (밀리미터)**: 이 절대 단위는 1/10센티미터와 같습니다.
- **`q` (쿼터밀리미터)**: 이 절대 단위는 1/40센티미터와 같습니다.
- **`pc` (파이카)**: 이 절대 단위는 1/6인치와 같습니다.
- **`pt` (포인트)**: 이 절대 단위는 1/72인치와 같습니다.

## 상대 단위

- **퍼센트(%)**: 이 상대 단위는 크기, 치수 및 기타 속성을 부모 요소의 비율로 정의할 수 있게 합니다. 예를 들어, 요소에 `width: 50%;`를 설정하면 부모 컨테이너 너비의 절반을 차지합니다.
- **`em` 단위**: 이 단위는 요소의 폰트 크기를 기준으로 합니다. `ems`를 `font-size` 속성에 사용하면 텍스트 크기가 부모 요소의 폰트 크기에 상대적입니다.
- **`rem` 단위**: 이 단위는 루트 요소인 `html` 요소의 폰트 크기를 기준으로 합니다.
- **`vh` 단위**: `vh`는 `"viewport height"`를 의미하며 `1vh`는 뷰포트 높이의 1%와 같습니다.
- **`vw` 단위**: `vw`는 `"viewport width"`를 의미하며 `1vw`는 뷰포트 너비의 1%와 같습니다.

## `calc` 함수

- **`calc()` 함수**: `calc()` 함수를 사용하면 스타일시트 내에서 직접 계산을 수행하여 속성 값을 동적으로 결정할 수 있습니다. 즉, 뷰포트 크기나 다른 요소를 기준으로 치수를 계산하여 유연하고 반응형 사용자 인터페이스를 만들 수 있습니다.

## 사용자 동작 가상 클래스

- **가상 클래스 정의**: 특정 상태나 위치에 따라 요소를 선택할 수 있게 해주는 특별한 CSS 키워드입니다.
- **사용자 동작 가상 클래스**: 사용자 상호작용에 따라 요소의 모양을 변경할 수 있게 해주는 특별한 키워드로, 전반적인 사용자 경험을 향상시킵니다.
- **`:active` 가상 클래스**: 버튼 클릭처럼 요소의 활성 상태를 선택할 수 있게 해줍니다.
- **`:hover` 가상 클래스**: 요소의 호버 상태를 정의합니다.
- **`:focus` 가상 클래스**: 키보드 탐색이나 사용자가 폼 입력에 클릭할 때처럼 요소가 포커스를 얻었을 때 스타일을 적용합니다.
- **`:focus-within` 가상 클래스**: 요소나 그 자손 요소 중 하나가 포커스를 가질 때 스타일을 적용하는 데 사용합니다.

## 입력 가상 클래스

- **입력 가상 클래스**: 이 가상 클래스들은 사용자 상호작용 전후 상태에 따라 HTML `input` 요소를 대상으로 지정하는 데 사용됩니다.
- **`:enabled` 가상 클래스**: 이 가상 클래스는 현재 활성화된 폼 버튼이나 다른 요소를 대상으로 지정하는 데 사용됩니다.
- **`:disabled` 가상 클래스**: 이 가상 클래스를 사용하면 비활성화된 상태의 인터랙티브 요소에 스타일을 적용할 수 있습니다.
- **`:checked` 가상 클래스**: 이 가상 클래스는 사용자에게 해당 요소가 선택되었음을 나타내는 데 사용됩니다.
- **`:valid` 가상 클래스**: 이 가상 클래스는 유효성 검사를 통과한 입력 필드를 대상으로 지정합니다.
- **`:invalid` 가상 클래스**: 이 가상 클래스는 유효성 검사를 통과하지 못한 입력 필드를 대상으로 지정합니다.
- **`:in-range` 및 `:out-of-range` 가상 클래스**: 이 가상 클래스들은 값이 지정된 범위 제약 조건 내에 있거나 벗어난 요소에 스타일을 적용합니다.
- **`:required` 가상 클래스**: 이 가상 클래스는 `input` 속성을 가진 `required` 요소를 대상으로 지정합니다. 사용자가 폼을 제출하려면 이 필드를 반드시 작성해야 함을 알립니다.
- **`:optional` 가상 클래스**: 이 가상 클래스는 필수가 아니며 비워둘 수 있는 입력 요소에 스타일을 적용합니다.
- **`:autofill` 가상 클래스**: 이 가상 클래스는 브라우저가 자동으로 저장된 데이터로 채우는 입력 필드에 스타일을 적용합니다.

## 위치 가상 클래스

- **위치 가상 클래스**: 이 가상 클래스들은 현재 문서 내에서 대상이 되는 링크와 요소를 스타일링할 때 사용합니다.
- **`:any-link` 가상 클래스**: 이 가상 클래스는 `:link`와 `:visited` 가상 클래스의 조합입니다. 따라서 방문 여부와 상관없이 href 속성이 있는 모든 앵커 요소와 일치합니다.
- **`:link` 가상 클래스**: 이 가상 클래스는 웹페이지의 모든 방문하지 않은 링크를 대상으로 지정할 수 있습니다. 사용자가 클릭하기 전 링크를 다르게 스타일링할 때 사용할 수 있습니다.
- **`:local-link` 가상 클래스**: 이 가상 클래스는 같은 문서를 가리키는 링크를 대상으로 지정합니다. 내부 링크와 외부 링크를 구분할 때 유용합니다.
- **`:visited` 가상 클래스**: 이 가상 클래스는 사용자가 방문한 링크를 대상으로 지정합니다.
- **`:target` 가상 클래스**: 이 가상 클래스는 URL 조각의 대상이 되는 요소에 스타일을 적용할 때 사용합니다.

## 트리 구조 가상 클래스

- **트리 구조 가상 클래스**: 이 가상 클래스들은 문서 트리 내에서 요소의 위치를 기준으로 요소를 선택하고 스타일을 지정할 수 있게 합니다.
- **`:root` 가상 클래스**: 이 가상 클래스는 보통 루트 `html` 요소입니다. 문서의 가장 상위 수준을 선택해 문서 전체에 공통 스타일을 적용할 때 유용합니다.
- **`:empty` 가상 클래스**: 자식이 공백 문자뿐인 빈 요소도 문서 트리에 포함됩니다. 그래서 빈 요소를 선택하기 위한 `:empty` 가상 클래스가 있습니다.
- **`:nth-child(n)` 가상 클래스**: 이 가상 클래스는 부모 내에서 요소의 위치를 기준으로 선택할 수 있게 합니다.
- **`:nth-last-child(n)` 가상 클래스**: 이 가상 클래스는 끝에서부터 요소를 세어 선택할 수 있게 합니다.
- **`:first-child` 가상 클래스**: 이 가상 클래스는 부모 요소나 문서 내에서 첫 번째 요소를 선택합니다.
- **`:last-child` 가상 클래스**: 이 가상 클래스는 부모 요소나 문서 내에서 마지막 요소를 선택합니다.
- **`:only-child` 가상 클래스**: 이 가상 클래스는 부모 요소나 문서 내에서 유일한 요소를 선택합니다.
- **`:first-of-type` 가상 클래스**: 이 가상 클래스는 부모 내에서 특정 요소 유형의 첫 번째 발생을 선택합니다.
- **`:last-of-type` 가상 클래스**: 이 가상 클래스는 부모 내에서 특정 요소 유형의 마지막 발생을 선택합니다.
- **`:nth-of-type(n)` 가상 클래스**: 이 가상 클래스는 같은 유형의 형제 요소 중 위치를 기준으로 특정 요소를 선택할 수 있게 합니다.
- **`:only-of-type` 가상 클래스**: 이 가상 클래스는 부모 내에서 해당 유형이 유일한 요소일 때 선택합니다.

## 함수형 가상 클래스

- **함수형 가상 클래스**: 함수형 가상 클래스는 더 복잡한 조건이나 관계에 따라 요소를 선택할 수 있게 합니다. 상태에 따라 요소를 선택하는 일반 가상 클래스와 달리(예: `:hover`, `:focus`), 함수형 가상 클래스는 인수를 받습니다.
- **`:is()` 가상 클래스**: 이 가상 클래스는 선택자 목록(예: `ol`, `ul`)을 인수로 받아 목록 중 하나와 일치하는 요소를 선택합니다.
- **`:where()` 가상 클래스**: 이 가상 클래스는 선택자 목록(예: `ol`, `ul`)을 인수로 받아 목록 중 하나와 일치하는 요소를 선택합니다. `:is`와 `:where`의 차이는 후자가 특이성이 0이라는 점입니다.
- **`:has()` 가상 클래스**: 이 가상 클래스는 자식 요소가 선택자 목록에 지정된 요소를 포함하는 경우 스타일을 적용할 수 있어 `"parent"` 선택자라고도 불립니다.

## 가상 요소

- **`::before` 가상 요소**: 이 가상 요소는 `content` 속성을 사용해 요소 바로 앞에 아이콘 같은 장식용 콘텐츠를 삽입합니다.
- **`::after` 가상 요소**: 이 가상 요소는 `content` 속성을 사용해 요소 바로 뒤에 아이콘 같은 장식용 콘텐츠를 삽입합니다.
- **`::first-letter` 가상 요소**: 이 가상 요소는 요소 내용의 첫 글자를 선택해 스타일을 지정할 수 있게 합니다.
- **`::marker` 가상 요소**: 이 가상 요소는 목록 항목의 마커(글머리 기호나 번호)를 선택해 스타일을 지정할 수 있게 합니다.

## 색채 이론

- **색채 이론 정의**: 색이 서로 어떻게 상호작용하는지, 그리고 우리의 인식에 어떤 영향을 미치는지 연구하는 학문입니다. 색 관계, 색 조화, 색의 심리적 영향을 다룹니다.
- **기본 색상**: 노랑, 파랑, 빨강으로 이루어진 이 색상들은 모든 다른 색상이 파생되는 기본 색조입니다.
- **보조 색상**: 두 기본 색상을 같은 비율로 섞어 만든 색상입니다. 초록, 주황, 보라가 보조 색상의 예입니다.
- **삼차 색상**: 기본 색상과 인접한 보조 색상을 결합해 만든 색상입니다. 노랑-초록, 파랑-초록, 파랑-보라가 삼차 색상의 예입니다.
- **따뜻한 색상**: 빨강, 주황, 노랑을 포함하며, 편안함, 따뜻함, 아늑함을 불러일으킵니다.
- **차가운 색상**: 파랑, 초록, 보라를 포함하며, 차분함, 평온함, 전문성을 느끼게 합니다.
- **색상환**: 색상환은 색이 서로 어떻게 관련되는지 보여주는 원형 도표입니다. 디자이너에게 필수 도구로, 색 조합을 선택하는 데 도움을 줍니다.
- **유사 색상 조합**: 색상환에서 서로 인접한 유사 색상으로 구성되어 일관되고 편안한 느낌을 만듭니다.
- **보색 조합**: 색상환에서 서로 반대편에 위치한 색상으로 구성되어 강한 대비와 시각적 임팩트를 만듭니다.
- **삼원색 조합**: 대략 같은 간격으로 떨어진 세 가지 색상으로 구성되어 생동감 있는 색상을 만듭니다. 이 색상을 연결하면 색상환에서 정삼각형을 이룹니다.
- **단색 조합**: 같은 기본 색상에서 밝기, 어둡기, 채도를 조절해 파생된 색상들로 구성되어 통일감과 조화를 느끼게 하면서도 대비를 만듭니다.

## CSS에서 색상을 다루는 다양한 방법

- **이름 있는 색상**: 브라우저가 인식하는 미리 정의된 색상 이름입니다. 예로 `blue`, `darkred`, `lightgreen`이 있습니다.
- **`rgb()` 함수**: RGB는 빛의 기본 색상인 빨강, 초록, 파랑을 의미합니다. 이 세 가지 색상을 다양한 강도로 조합해 다양한 색상을 만듭니다. `rgb()` 함수는 RGB 색상 모델을 사용해 색상을 정의할 수 있게 합니다.
- **`rgba()` 함수**: 이 함수는 네 번째 값인 알파를 추가해 색상의 투명도를 조절합니다.
- **`hsl()` 함수**: HSL은 색상, 채도, 밝기라는 세 가지 주요 요소로 색상을 정의합니다.
- **`hsla()` 함수**: 이 함수는 네 번째 값인 알파를 추가해 색상의 불투명도를 조절합니다.
- **16진수**: 16진수 코드는 RGB 색상 모델에서 색상을 나타내는 데 쓰이는 6자리 문자열입니다. "16진수"는 0부터 9까지 숫자와 A부터 F까지 문자를 사용하는 16진법 숫자 체계를 뜻합니다.

## `box-shadow` 속성

- **정의**: `box-shadow` 속성은 요소 주위에 하나 이상의 그림자를 적용합니다.
- **오프셋 값**: 수평(`offset-x`)과 수직(`offset-y`) 값을 지정해야 합니다. 양수 `offset-x`는 그림자를 오른쪽으로 이동시키고, 음수 값은 왼쪽으로 이동시킵니다. 양수 `offset-y`는 그림자를 아래로 이동시키고, 음수 값은 위로 이동시킵니다. 값이 `0`이면 단위를 포함하지 않아도 됩니다.
- **블러 반경**: 이 선택적 값은 그림자의 흐림 정도를 조절합니다. 포함하지 않으면 기본값은 `0`로, 선명한 가장자리를 만듭니다. 값이 클수록 그림자가 더 부드러워집니다.
- **확산 반경**: 이 선택적 값은 그림자가 얼마나 확장되거나 축소되는지 조절합니다. 포함하지 않으면 기본값은 `0`입니다.
- **그림자 색상**: 이름이 지정된 색상, 16진수 값, `rgb()`, `rgba()`, `hsl()`, 또는 `hsla()` 함수를 사용해 색상을 지정할 수 있습니다.
- **`inset` 키워드**: `inset` 키워드를 추가하면 그림자가 요소 내부에 위치합니다.
- **여러 개의 박스 그림자 적용**: 쉼표로 구분하여 여러 그림자를 적용할 수 있습니다. 그림자는 앞에서 뒤로 겹쳐집니다.

- **선형 그라디언트**: 이 그라디언트는 직선을 따라 색상이 점차적으로 섞이도록 만듭니다. 이 선의 방향과 사용되는 색상을 조절할 수 있습니다.  
- **방사형 그라디언트**: 이 그라디언트는 중심점에서 원형 또는 타원형으로 퍼지는 그라디언트를 만듭니다.

## 입력 스타일링을 위한 모범 사례

- **입력 스타일링**: 모든 텍스트 요소와 마찬가지로, 텍스트 입력에 적용하는 스타일이 접근 가능하도록 해야 합니다. 즉, 폰트 크기가 적절해야 하고, 색상은 배경과 충분한 대비를 가져야 합니다. 입력 요소는 포커스가 가능하기도 합니다. 스타일을 편집할 때는 요소가 포커스를 받을 때 굵은 테두리와 같은 눈에 띄는 표시가 유지되도록 주의하세요.

## 입력에 `appearance: none` 사용하기

- **`appearance: none`**: 브라우저는 많은 요소에 기본 스타일을 적용합니다. `appearance: none` CSS 속성은 스타일을 완전히 제어할 수 있게 해주지만 몇 가지 주의할 점이 있습니다. 입력 요소에 맞춤 스타일을 만들 때는 포커스와 오류 표시가 여전히 잘 보이도록 해야 합니다.

## `datetime-local`와 `color` 프로프 스타일링 시 흔한 문제

- **흔한 문제**: 이 특별한 유형의 입력은 날짜 및 색상 선택기 같은 것을 만들기 위해 복잡한 가상 요소에 의존합니다. 이로 인해 이러한 입력 스타일링에 큰 도전 과제가 생깁니다. 한 가지 문제는 기본 스타일이 완전히 브라우저에 따라 달라서, 선택기를 원하는 모양으로 만들기 위해 작성한 CSS가 다른 브라우저에서는 전혀 다르게 보일 수 있다는 점입니다.

## CSS 오버플로우 속성

- **정의**: 오버플로우는 요소가 포함하는 요소의 크기를 초과하거나 "넘치는" 콘텐츠를 처리하는 방식을 말합니다. 오버플로우는 2차원입니다.
- **`overflow-x`**: x축은 수평 오버플로우를 결정합니다.
- **`overflow-y`**: y축은 수직 오버플로우를 결정합니다.

## CSS 변환 속성

- **정의**: 이 속성은 요소에 회전, 크기 조절, 기울이기 또는 2D나 3D 공간에서 이동(변환) 같은 다양한 변형을 적용할 수 있게 합니다.
- **`translate()` 함수**: 이 함수는 요소를 현재 위치에서 이동시키는 데 사용됩니다.
- **`scale()` 함수**: 이 함수는 요소의 크기를 변경할 수 있게 합니다.
- **변환과 접근성**: 변환을 사용해 콘텐츠를 숨기거나 표시할 때는 화면 읽기 프로그램과 키보드 탐색에서 콘텐츠에 여전히 접근할 수 있도록 하세요. 숨긴 콘텐츠는 단순히 시각적으로 화면 밖으로 이동시키는 대신 `display: none` 또는 `visibility: hidden` 같은 방법으로 실제로 숨겨야 합니다.

## 박스 모델

- **정의**: CSS 박스 모델에서 모든 요소는 박스로 둘러싸여 있습니다. 이 박스는 네 가지 컴포넌트로 구성됩니다: 콘텐츠 영역, `padding`, `border`, `margin`.
- **콘텐츠 영역**: 콘텐츠 영역은 박스의 가장 안쪽 부분입니다. 텍스트나 이미지 같은 요소의 실제 콘텐츠가 들어 있는 공간입니다.
- **`padding`**: 패딩은 콘텐츠 영역 바로 다음에 위치한 영역입니다. 콘텐츠 영역과 요소의 테두리 사이의 공간입니다.
- **`border`**: 테두리는 CSS 박스 모델에서 요소의 바깥 가장자리 또는 윤곽선입니다. 요소의 시각적 경계입니다.
- **`margin`**: 여백은 요소의 테두리 바깥에 있는 공간입니다. 요소와 주변 다른 요소 사이의 거리를 결정합니다.

## 여백 겹침

- **정의**: 인접한 요소들의 수직 여백이 겹쳐서 두 여백 중 더 큰 값과 같은 하나의 여백으로 합쳐지는 현상입니다. 이 현상은 수직 여백(위쪽과 아래쪽)에만 적용되며, 수평 여백(왼쪽과 오른쪽)에는 적용되지 않습니다.

## `content-box`와 `border-box` 속성 값

- **`box-sizing` 속성**: 이 속성은 HTML 요소의 최종 `width`와 `height`이 어떻게 계산되는지 결정하는 데 사용됩니다.
- **`content-box` 값**: `content-box` 모델에서는 요소에 설정한 `width`와 `height`이 콘텐츠 영역의 크기를 결정하지만 `padding`, `border`, 또는 `margin`은 포함하지 않습니다.
- **`border-box` 값**: `border-box`에서는 요소의 `width`와 `height`가 콘텐츠 영역, `padding`, `border`을 포함하지만 `margin`은 포함하지 않습니다.

## CSS 초기화

- **정의**: CSS 초기화는 웹 브라우저가 HTML 요소에 기본적으로 적용하는 모든 또는 일부 기본 서식을 제거하는 스타일시트입니다. CSS 초기화를 위한 서드파티 옵션에는 `sanitize.css`과 `normalize.css`가 있습니다.

## CSS 필터 속성

- **정의**: 이 속성은 블러 처리, 색상 변화, 대비 조정과 같은 다양한 효과를 만드는 데 사용할 수 있습니다.
- **`blur()` 함수**: 이 함수는 요소에 가우시안 블러를 적용합니다. 양은 픽셀 단위로 정의되며 블러의 반경을 나타냅니다.
- **`brightness()` 함수**: 이 함수는 요소의 밝기를 조정합니다. 0% 값은 요소를 완전히 검게 만들고, 100%를 초과하는 값은 밝기를 증가시킵니다.
- **`grayscale()` 함수**: 이 함수는 요소를 그레이스케일로 변환합니다. 양은 백분율로 정의되며, 100%는 완전한 그레이스케일, 0%는 이미지를 변경하지 않습니다.
- **`sepia()` 함수**: 이 함수는 요소에 세피아 톤을 적용합니다. 그레이스케일과 마찬가지로 백분율 값을 사용합니다.
- **`hue-rotate()` 함수**: 이 함수는 요소에 색상 회전을 적용합니다. 값은 도 단위로 정의되며 색상 원을 중심으로 회전하는 것을 나타냅니다.

## CSS 플렉스박스와 플렉스 모델 소개

- **정의**: CSS 플렉스박스는 컨테이너 내에서 요소를 행과 열로 배치할 수 있는 1차원 레이아웃 모델입니다.  
- **플렉스 모델**: 이 모델은 플렉스 컨테이너 안에서 플렉스 항목이 어떻게 배치되는지를 정의합니다. 모든 플렉스 컨테이너는 주축과 교차 축, 두 개의 축을 가집니다.

## `flex-direction` 속성

- **정의**: 이 속성은 주축의 방향을 설정합니다. `flex-direction`의 기본값은 `row`로, 모든 플렉스 항목을 브라우저 기본 언어 방향(왼쪽에서 오른쪽 또는 오른쪽에서 왼쪽)으로 같은 행에 배치합니다.
- **`flex-direction: row-reverse;`**: 이 값은 행 내 항목들의 순서를 반대로 바꿉니다.
- **`flex-direction: column;`**: 이 값은 플렉스 항목을 수평이 아닌 수직으로 정렬합니다.
- **`flex-direction: column-reverse;`**: 이 값은 플렉스 항목의 수직 순서를 반대로 바꿉니다.

## `flex-wrap` 속성

- **정의**: 이 속성은 플렉스 항목들이 플렉스 컨테이너 내에서 사용 가능한 공간에 맞게 어떻게 감싸지는지 결정합니다. `flex-wrap`는 세 가지 값, `nowrap`, `wrap`, `wrap-reverse`을 가질 수 있습니다.
- **`flex-wrap: nowrap;`**: 기본값입니다. 플렉스 항목들은 컨테이너 너비를 초과해도 새 줄로 감싸지지 않습니다.
- **`flex-wrap: wrap;`**: 이 속성은 항목들이 컨테이너 너비를 초과할 때 감싸도록 합니다.
- **`flex-wrap: wrap-reverse;`**: 이 속성은 플렉스 항목들을 역순으로 감쌉니다.
- **`flex-flow` 속성**: 이 속성은 `flex-direction`과 `flex-wrap`의 단축 속성입니다.

## `justify-content` 속성

- **정의**: 이 속성은 플렉스 컨테이너의 주축을 따라 자식 요소들을 정렬합니다.
- **`justify-content: flex-start;`**: 이 경우 플렉스 항목들은 주축의 시작 부분에 정렬됩니다. 주축은 가로 또는 세로일 수 있습니다.
- **`justify-content: flex-end;`**: 이 경우 플렉스 항목들은 주축의 끝 부분에 정렬됩니다. 가로 또는 세로일 수 있습니다.
- **`justify-content: center;`**: 이 속성은 플렉스 항목들을 주축을 따라 가운데로 정렬합니다.
- **`justify-content: space-between;`**: 이 속성은 요소들을 주축을 따라 고르게 분배합니다.
- **`justify-content: space-around;`**: 이 속성은 플렉스 항목들을 주축 내에서 고르게 분배하며, 첫 번째 항목 앞과 마지막 항목 뒤에 공간을 추가합니다.
- **`justify-content: space-evenly;`**: 이 속성은 항목들을 주축을 따라 고르게 분배합니다.

## `align-items` 속성

- **정의**: 이 속성은 교차 축을 따라 항목을 배치하는 데 사용됩니다. 교차 축은 주축에 수직임을 기억하세요.
- **`align-items: center;`**: 교차 축을 따라 항목을 가운데 정렬하는 데 사용됩니다.
- **`align-items: flex-start;`**: 교차 축의 시작 부분에 항목을 정렬합니다.
- **`align-items: stretch;`**: 교차 축을 따라 플렉스 항목을 늘리는 데 사용됩니다.

## 타이포그래피 소개

- **정의**: 타이포그래피는 텍스트를 시각적으로 매력적이고 읽기 쉽게 만들기 위해 적절한 폰트와 형식을 선택하는 예술입니다. "타입"은 개별 문자가 어떻게 디자인되고 배열되는지를 의미합니다.
- **서체 정의**: 서체는 문자, 숫자, 기호 세트의 전체적인 디자인과 스타일입니다. 폰트 가족의 청사진과 같습니다.
- **폰트 정의**: 폰트는 크기, 굵기, 스타일, 너비 등 특정 특성을 가진 서체의 구체적인 변형입니다.
- **세리프 서체**: 이 서체는 문자 끝에 작은 선이 있는 고전적인 스타일입니다. 세리프 서체는 책과 같은 인쇄물에 주로 사용됩니다.
- **산세리프 서체**: 이 서체는 문자 끝에 작은 선이 없는 보다 현대적인 모습입니다. 산세리프 서체는 화면에서 읽기 쉽기 때문에 디지털 디자인에 자주 사용됩니다. 예로는 Helvetica, Arial, Roboto가 있습니다.
- **폰트 굵기**: 문자의 두께를 의미하며, 라이트, 레귤러, 굵게, 블랙 등이 있습니다.
- **폰트 스타일**: 문자의 기울기와 방향을 의미하며, 이탤릭과 오블리크가 있습니다.
- **폰트 너비**: 문자가 차지하는 가로 공간을 의미하며, 압축형과 확장형이 있습니다.
- **기준선**: 대부분 문자가 놓이는 가상의 선입니다.
- **대문자 높이**: 기준선에서 대문자의 꼭대기까지의 높이입니다.
- **x-높이**: 올림자와 내림자를 제외한 소문자의 평균 높이입니다.
- **올림자**: 소문자 중 x-높이 위로 올라가는 부분으로, 'h', 'b', 'd', 'f'의 꼭대기 부분이 해당합니다.
- **내림자**: 소문자 중 기준선 아래로 내려가는 부분으로, 'y', 'g', 'p', 'q'의 꼬리 부분이 해당합니다.
- **커닝**: 특정 문자 쌍 사이의 간격을 조정하여 가독성과 미적 감각을 개선하는 것입니다. 예를 들어, A와 V 사이의 간격을 줄이는 경우입니다.
- **트래킹**: 텍스트 블록 내 모든 문자 사이의 간격을 조정하는 것으로, 문자 간 거리입니다. 텍스트의 밀도와 여백에 영향을 줍니다.
- **리딩**: 텍스트 줄 사이의 수직 간격으로, 한 줄의 기준선에서 다음 줄의 기준선까지의 거리입니다.
- **타이포그래피 모범 사례**: 디자인을 이해하기 쉽게 만들려면 명확하고 단순한 폰트를 선택하세요. 특히 웹사이트 본문에 중요합니다. 폰트가 읽기 쉬우면 사용자가 콘텐츠에 더 많이 참여합니다. 시각적 일관성을 위해 최대 두세 개의 폰트만 사용하세요. 너무 많은 폰트를 사용하면 텍스트가 읽기 어려워지고 브랜드 정체성이 약해질 수 있습니다. 또한 웹사이트 로드 시간이 길어져 사용자 경험에 영향을 줄 수 있습니다. 제목, 부제목, 문단 등 요소별로 시각적 계층 구조를 만들기 위해 폰트 크기를 활용하세요. 예를 들어, 웹페이지의 주요 제목은 더 큰 폰트를 사용하고, 부제목은 더 작은 폰트 크기를 사용합니다. 이렇게 하면 계층 구조의 각 요소에 특정 폰트 크기를 부여해 사용자가 시각적으로 구조를 쉽게 파악할 수 있습니다.

## CSS `font-family` 속성

- **정의**: 폰트 패밀리는 공통 디자인을 공유하는 폰트 그룹입니다. 같은 패밀리에 속한 모든 폰트는 동일한 기본 서체를 기반으로 하지만 스타일, 굵기, 너비에 변형이 있습니다. 우선순위가 높은 것부터 낮은 순서로 여러 폰트 패밀리를 쉼표로 구분해 지정할 수 있습니다. 이러한 대체 폰트들은 대체 수단 역할을 합니다. 폰트 패밀리 목록 끝에는 항상 일반 폰트 패밀리를 포함해야 합니다.
- **일반적인 폰트 패밀리**: CSS에서 자주 사용하는 폰트 패밀리에는 serif, sans-serif, monospace, cursive, fantasy가 있습니다.

## 웹 안전 폰트

- **정의**: 이 폰트들은 컴퓨터나 기기에 설치되어 있을 가능성이 매우 높은 폰트의 부분집합입니다. 브라우저가 폰트를 렌더링해야 할 때, 사용자의 시스템에서 폰트 파일을 찾으려고 시도합니다. 하지만 폰트를 찾지 못하면 보통 기본 시스템 폰트로 대체합니다.

- **웹 안전 폰트**: 
  - 산세리프 폰트는 문자 끝에 작은 "발"이나 선이 없어서 화면에서 읽기 쉽기 때문에 웹 개발에 자주 사용됩니다. 웹 안전 산세리프 폰트의 예로는 Arial, Verdana, Trebuchet MS가 있습니다.
  - 반면에 세리프 폰트는 문자 끝에 작은 "발"이 있어서 전통적인 인쇄물에 주로 사용됩니다. 웹 안전 세리프 폰트에는 Times New Roman과 Georgia가 포함됩니다.

## @규칙과 `@font-face` @규칙

- **정의**: @규칙은 브라우저에 지시를 제공하는 문장입니다. 미디어 쿼리, 키프레임, 폰트 페이스 등 스타일시트의 다양한 측면을 정의하는 데 사용할 수 있습니다.  
- **`@font-face`**: 폰트 파일, 형식, 그리고 굵기와 스타일 같은 중요한 속성을 지정하여 사용자 정의 폰트를 정의할 수 있게 해줍니다. `@font-face` @규칙이 유효하려면 `src` 속성도 지정해야 합니다. 이 속성은 폰트 리소스에 대한 참조를 포함합니다.
- **폰트 형식**: 각 폰트 리소스에 대해 형식을 지정할 수도 있습니다. 이는 선택 사항이며, 브라우저에 폰트 형식에 대한 힌트를 제공합니다. 형식을 생략하면 리소스가 다운로드된 후 형식이 감지됩니다. 형식이 유효하지 않으면 리소스가 다운로드되지 않습니다. 가능한 폰트 형식에는 `collection`, `embedded-opentype`, `opentype`, `svg`, `truetype`, `woff`, `woff2`가 있습니다.
- **`woff`와 `woff2`**: `woff`는 "웹 오픈 폰트 형식"을 의미합니다. Mozilla가 Type Supply, LettError 및 기타 조직과 협력해 개발한 널리 사용되는 폰트 형식입니다. `woff`와 `woff2`의 차이는 데이터를 압축하는 알고리즘에 있습니다.
- **OpenType**: Microsoft와 Adobe가 개발한 확장 가능한 컴퓨터 폰트 형식으로, 사용자가 폰트의 추가 기능에 접근할 수 있게 해줍니다. 주요 운영체제에서 널리 사용됩니다.
- **`tech()`**: 폰트 리소스의 기술을 지정하는 데 사용됩니다. 선택 사항입니다.

## 외부 폰트 사용하기

- **정의**: 외부 폰트는 프로젝트 파일에 직접 포함되지 않은 폰트 파일입니다. 보통 별도의 서버에 호스트되어 있습니다. 이러한 외부 폰트를 프로젝트에 포함하려면 CSS 안에 `link` 요소나 `@import` 구문을 사용할 수 있습니다.  
- **Google Fonts**: 웹 개발을 위해 특별히 디자인된 폰트를 많이 제공하는 구글 서비스입니다.  
- **Font Squirrel**: 프로젝트에 맞춤 외부 폰트를 추가할 때 사용할 수 있는 또 다른 인기 있는 리소스입니다.

## `text-shadow` 속성

- **정의**: 이 속성은 텍스트에 그림자를 적용하는 데 사용됩니다. 그림자의 수평 및 수직 거리를 나타내는 X 오프셋과 Y 오프셋을 지정해야 합니다. 이 값들은 필수입니다. X 오프셋과 Y 오프셋의 양수 값은 각각 그림자를 오른쪽과 아래로 이동시키고, 음수 값은 그림자를 왼쪽과 위로 이동시킵니다.
- **그림자 색상**: 오프셋 앞이나 뒤에 이 값을 지정하여 그림자 색상을 사용자 지정할 수 있습니다. 색상을 지정하지 않으면 브라우저가 자동으로 색상을 결정하므로 보통 명시적으로 설정하는 것이 좋습니다.
- **블러 반경**: 블러 반경은 선택 사항이지만 그림자를 훨씬 부드럽고 은은하게 만듭니다. 블러 반경의 기본값은 0입니다. 값이 클수록 블러가 커져 그림자가 더 연해집니다.
- **여러 텍스트 그림자 적용**: 텍스트에 여러 개의 그림자를 적용할 수 있습니다. 그림자는 앞에서 뒤로, 즉 첫 번째 그림자가 가장 위에 오도록 층별로 적용됩니다.

## 색상 대비 도구

- **WebAIM의 Color Contrast Checker**: 이 온라인 도구는 디자인의 전경색과 배경색을 입력하면 웹 콘텐츠 접근성 지침(WCAG) 기준을 충족하는지 즉시 확인할 수 있습니다.  
- **TPGi Colour Contrast Analyzer**: 이 무료 색상 대비 검사기는 웹사이트와 앱이 웹 콘텐츠 접근성 지침(WCAG) 기준을 충족하는지 확인할 수 있습니다. 이 도구에는 색각 이상 시뮬레이터 기능도 포함되어 있어 색각 이상이 있는 사용자가 웹사이트나 앱을 어떻게 보는지 확인할 수 있습니다.

## 접근성 트리

접근성 트리는 화면 읽기 프로그램과 같은 보조 기술이 웹페이지의 내용을 해석하고 상호작용하는 데 사용하는 구조입니다.

## `max()` 함수

`max()` 함수는 쉼표로 구분된 값들 중 가장 큰 값을 반환합니다:

```css
img {
  width: max(250px, 25vw);
}
```

위 예제에서 뷰포트 너비가 1000픽셀보다 작으면 이미지 너비는 250px가 됩니다(250px가 뷰포트 너비의 25%보다 크기 때문입니다). 뷰포트 너비가 1000픽셀보다 크면 이미지 너비는 25vw가 됩니다. 이는 25vw가 뷰포트 너비의 25%와 같기 때문입니다.

## CSS와 접근성 모범 사례    

- **`display: none;`**: `display: none;`를 사용하면 화면 읽기 프로그램과 기타 보조 기술이 이 콘텐츠에 접근할 수 없는데, 이는 접근성 트리에 포함되지 않기 때문입니다. 따라서 시각적 표현과 접근성 모두에서 콘텐츠를 완전히 제거하려는 경우에만 이 방법을 사용하는 것이 중요합니다.
- **`visibility: hidden;`**: 이 속성과 값은 콘텐츠를 시각적으로 숨기지만 문서 흐름에는 남겨두어 페이지에서 공간을 차지하게 합니다. 이 요소들은 접근성 트리에서 제거되어 화면 읽기 프로그램이 더 이상 읽지 않습니다.
- **`.sr-only` CSS 클래스**: 화면 읽기 프로그램에는 접근 가능하지만 시각적으로는 숨기는 데 자주 사용하는 기법입니다.

```css
.sr-only {
  position: absolute;
  width: 1px;
  height: 1px;
  padding: 0;
  margin: -1px;
  overflow: hidden;
  clip: rect(0, 0, 0, 0);
  white-space: nowrap;
  border: 0;
}
```

- **`scroll-behavior: smooth;`**: 이 속성과 값은 부드러운 스크롤 동작을 활성화합니다.
- **`prefers-reduced-motion` 기능**: 사용자의 애니메이션 선호도를 감지하는 데 사용할 수 있는 미디어 기능입니다.

```css
@media (prefers-reduced-motion: no-preference) {
  * {
    scroll-behavior: smooth;
  }
}
```

위 예제에서는 사용자가 기기에서 애니메이션 선호도를 설정하지 않은 경우 부드러운 스크롤이 활성화됩니다.

## HTML 속성으로 콘텐츠 숨기기

- **`aria-hidden` 속성**: 화면 읽기 프로그램 같은 보조 기술을 사용하는 사용자가 요소에 접근하지 못하도록 숨길 때 사용합니다. 예를 들어, 의미 있는 내용을 제공하지 않는 장식용 이미지를 숨길 때 쓸 수 있습니다.
- **`hidden` 속성**: 대부분 최신 브라우저에서 지원하며, 시각적으로뿐만 아니라 접근성 트리에서도 콘텐츠를 숨깁니다. JavaScript로 쉽게 켜고 끌 수 있습니다.

:::interactive_editor

```html
<p aria-hidden>This paragraph is visible for sighted users, but is hidden from assistive technology.</p>
<p hidden>This paragraph is hidden from both sighted users and assistive technology.</p>
```

:::

## `placeholder` 속성의 접근성 문제

자리 표시자 텍스트를 사용하는 것은 접근성에 좋지 않습니다. 사용자들이 자리 표시자 텍스트를 실제 입력 값과 혼동하는 경우가 많아, 입력란에 이미 값이 있는 것으로 생각합니다.

## 다양한 속성 선택자와 링크 다루기

- **정의**: `attribute` 선택자는 `href` 또는 `title` 속성과 같은 속성을 기준으로 HTML 요소를 선택할 수 있게 합니다.  
- **`title` 속성**: 이 속성은 요소에 대한 추가 정보를 제공합니다.

## `lang`와 `data-lang` 속성으로 요소 선택하기

- **`lang` 속성**: 이 속성은 HTML에서 요소 내 콘텐츠의 언어를 지정할 때 사용합니다. 다국어 웹사이트에서는 작성된 언어에 따라 요소를 다르게 스타일링하고 싶을 때가 있습니다.  
- **`data-lang` 속성**: `data-lang` 속성과 같은 사용자 정의 데이터 속성은 특정 텍스트 섹션 내에서 사용된 언어를 지정하는 등 요소에 추가 정보를 저장할 때 흔히 사용합니다.

## 속성 선택자, 순서 있는 목록 요소 및 `type` 속성 다루기

- **`type` 속성**: HTML에서 순서 있는 목록을 다룰 때, `type` 속성은 숫자, 알파벳, 로마 숫자 등 사용되는 번호 매기기 스타일을 지정할 수 있게 해줍니다.

## 플로트 다루기

- **정의**: 플로트는 요소를 페이지의 일반적인 흐름에서 제거하고 컨테이너의 왼쪽이나 오른쪽에 위치시키는 데 사용합니다. 이렇게 하면 텍스트가 플로트된 콘텐츠 주위를 감싸게 됩니다.
- **플로트 지우기**: `clear` 속성은 요소가 플로트된 콘텐츠 아래로 이동해야 하는지 결정하는 데 사용합니다. 여러 개의 플로트된 요소가 나란히 쌓여 있을 때 레이아웃에서 겹침이나 붕괴 문제가 발생할 수 있습니다. 그래서 이 문제를 해결하기 위해 `clearfix` 해킹 기법이 만들어졌습니다.

## 고정, 상대 및 절대 위치 지정

- **고정 위치 지정**: 문서의 기본 흐름입니다. 요소들은 위에서 아래로, 왼쪽에서 오른쪽으로 차례대로 배치됩니다.
- **상대 위치 지정**: `top`, `left`, `right` 및 `bottom` 프로퍼티를 사용해 요소를 기본 문서 흐름 내에서 위치시킬 수 있습니다. 상대 위치 지정을 사용하면 요소들이 페이지 내 다른 요소와 겹치도록 만들 수도 있습니다.
- **절대 위치 지정**: 요소를 기본 문서 흐름에서 분리해 다른 요소와 독립적으로 동작하게 만듭니다.

## 고정 및 스티키 위치 지정

- **고정 위치 지정**: 요소가 `position: fixed`로 위치 지정되면 일반 문서 흐름에서 제거되고 뷰포트를 기준으로 배치되어 사용자가 스크롤해도 같은 위치에 유지됩니다. 이는 항상 보여야 하는 헤더나 내비게이션 바 같은 요소에 자주 사용됩니다.
- **스티키 위치 지정**: 이 위치 지정 방식은 상대 위치와 고정 위치의 혼합형으로 작동합니다. 처음에는 요소가 상대적으로 위치한 것처럼 문서 흐름 내에 머무르지만, 사용자가 특정 지점까지 스크롤하면 요소가 뷰포트(보통 상단)에 "붙어서" 고정된 것처럼 동작합니다. 이는 사용자가 특정 위치까지 스크롤할 때만 고정되는 스티키 내비게이션 바 같은 요소를 만들기에 적합합니다.

## `z-index` 속성 다루기

- **정의**: CSS에서 `z-index` 속성은 페이지에서 겹치는 위치 지정 요소들의 수직 쌓임 순서를 제어하는 데 사용됩니다.

## 반응형 웹 디자인

- **정의**: 반응형 디자인의 핵심 원리는 적응성입니다. 즉, 웹사이트가 보는 기기의 화면 크기와 기능에 따라 레이아웃과 콘텐츠를 조정하는 능력입니다.
- **유동 그리드**: 픽셀과 같은 고정 단위 대신 백분율과 같은 상대 단위를 사용하여 화면 크기에 따라 콘텐츠가 크기 조절되고 재배치되도록 합니다.
- **유연한 이미지**: 컨테이너 내에서 크기가 조절되도록 설정하여 작은 화면에서 이미지가 컨테이너를 넘치지 않도록 합니다.

## 미디어 쿼리

- **정의**: 개발자가 주로 뷰포트 너비 같은 장치 특성에 따라 다른 스타일을 적용할 수 있게 합니다.
- **`all` 미디어 유형**: 모든 장치에 적합합니다. 미디어 유형이 지정되지 않으면 기본값입니다.
- **`print` 미디어 유형**: 페이지 매김된 자료와 인쇄 미리보기 모드에서 화면에 표시되는 문서에 적합합니다.
- **`screen` 미디어 유형**: 주로 화면용입니다.
- **`aspect-ratio`**: 뷰포트의 너비와 높이 비율을 설명합니다.
- **`orientation`**: 장치가 가로 방향인지 세로 방향인지 나타내는 데 사용합니다.
- **`resolution`**: 출력 장치의 해상도를 인치당 도트(dpi) 또는 센티미터당 도트(dpcm)로 설명하는 데 사용합니다.
- **`hover`**: 기본 입력 장치가 요소 위에 마우스를 올릴 수 있는지 테스트하는 데 사용합니다.
- **`prefers-color-scheme`**: 사용자가 밝은 색상 테마 또는 어두운 색상 테마를 요청했는지 감지하는 데 사용합니다.
- **미디어 쿼리와 논리 연산자**: `and` 연산자는 여러 미디어 기능을 결합하는 데 사용하며, `not`와 `only`는 미디어 쿼리를 부정하거나 분리하는 데 사용할 수 있습니다.

## 일반적인 미디어 중간점

- **정의**: 미디어 중간점은 웹사이트 디자인에서 레이아웃과 콘텐츠가 다양한 화면 크기에 맞게 조정되는 특정 지점을 말합니다. 휴대폰, 태블릿, 데스크톱 화면을 대상으로 할 수 있는 일반적인 중간점이 있습니다. 하지만 모든 기기의 가능한 모든 화면 크기를 일일이 맞추려 하는 것은 현명하지 않습니다.
- **소형 기기(스마트폰)**: 최대 640px
- **중간 기기(태블릿)**: 641px에서 1024px까지
- **대형 기기(데스크톱)**: 1025px 이상

## 모바일 우선 접근법

- **정의**: 모바일 우선 접근법은 반응형 웹 디자인에서 모바일 기기를 위한 웹사이트를 먼저 만들고, 그다음에 더 큰 화면을 위한 디자인을 하는 철학이자 개발 전략입니다.

## CSS 사용자 정의 속성(CSS 변수)

- **정의**: CSS 사용자 정의 속성, 즉 CSS 변수는 CSS 작성자가 문서 전체에서 재사용할 특정 값을 담기 위해 정의하는 항목입니다. 이는 더 효율적이고 유지 관리가 쉬우며 유연한 스타일시트를 만들 수 있게 하는 강력한 기능입니다. 사용자 정의 속성은 특히 테마 디자인을 만들 때 유용합니다. 서로 다른 테마에 맞는 속성 집합을 정의할 수 있습니다.

## `@property` 규칙

- **정의**: `@property` 규칙은 개발자가 동작 방식, 애니메이션 방법, 초기값 등을 더 세밀하게 제어할 수 있는 사용자 정의 속성을 정의할 수 있게 하는 강력한 CSS 기능입니다.

```css
@property --property-name {
  syntax: '<type>';
  inherits: true | false;
  initial-value: <value>;
}
```

- **`--property-name`**: 정의하는 사용자 정의 속성의 이름입니다. 모든 사용자 정의 속성처럼 두 개의 대시로 시작해야 합니다.
- **`syntax`**: 속성의 타입을 정의하며, `<color>`, `<length>`, `<number>`, `<percentage>` 또는 더 복잡한 타입이 될 수 있습니다.
- **`inherits`**: 속성이 부모 요소로부터 값을 상속받을지 여부를 지정합니다.
- **`initial-value`**: 속성의 기본값을 설정합니다.
- **대체 수단**: 사용자 정의 속성을 사용할 때, 표준 사용자 정의 속성과 마찬가지로 `var()` 함수를 사용해 대체 수단 값을 제공할 수 있습니다.

## CSS 그리드 기초

- **정의**: CSS 그리드는 웹 페이지에서 복잡한 레이아웃을 만들기 위해 사용하는 2차원 레이아웃 시스템입니다. 그리드는 행과 열로 구성되며 그 사이에 간격이 있습니다. 그리드 레이아웃을 정의하려면 `display` 속성을 `grid`로 설정하세요.
- **`fr` (분수) 단위**: 이 단위는 그리드 컨테이너 내 공간의 분수를 나타냅니다. 유연한 그리드를 만들 때 `fr` 단위를 사용할 수 있습니다.
- **트랙 사이 간격 만들기**: CSS 그리드에서 트랙 사이 간격을 만드는 방법은 세 가지가 있습니다. 열 사이 간격을 만들려면 `column-gap` 속성을 사용하세요. 행 사이 간격을 만들려면 `row-gap` 속성을 사용하세요. 또는 행과 열 모두 사이 간격을 만들려면 `gap` 단축 속성을 사용하세요.
- **`repeat()` 함수**: 이 함수는 트랙 목록에서 섹션을 반복할 때 사용합니다. `grid-template-columns: 1fr 1fr 1fr;`를 직접 쓰는 대신 `repeat()` 함수를 사용할 수 있습니다.
- **명시적 그리드**: `grid-template-columns` 또는 `grid-template-rows` 속성을 사용해 선이나 트랙 수를 지정할 수 있습니다.
- **암시적 그리드**: 항목이 그리드 밖에 배치되면, 그리드 밖 요소를 위해 행과 열이 자동으로 생성됩니다.
- **`minmax()` 함수**: 이 함수는 트랙의 최솟값과 최댓값 크기를 설정할 때 사용합니다.
- **그리드 자동 배치**: 브라우저가 자동 배치 알고리즘을 사용해 항목을 자동으로 배치합니다. `grid-auto-flow` 속성은 이 동작을 제어하며, `row`, `column`, `dense` 같은 값을 사용해 배치 방향과 빈 공간 채우기 여부에 영향을 줍니다.
- **선 기반 배치**: 모든 그리드에는 선이 있습니다. 항목이 어느 선에서 시작하는지 지정하려면 `grid-column-start`와 `grid-row-start` 속성을 사용하세요. 항목이 어느 선에서 끝나는지 지정하려면 `grid-column-end`와 `grid-row-end` 속성을 사용하세요. 또는 `grid-column`나 `grid-row` 단축 속성을 사용할 수도 있습니다.
- **`grid-template-areas`**: 이 속성은 그리드에서 위치를 지정할 항목에 이름을 부여할 때 사용합니다.
- **정렬**: `justify-items`와 `align-items`를 사용해 항목이 그리드 영역 내에서 어떻게 정렬되는지 제어하세요. 개별 그리드 항목에서는 `justify-self`와 `align-self`를 사용해 컨테이너에 설정된 정렬을 재정의할 수 있습니다.

## CSS 애니메이션 기초

- **정의**: CSS 애니메이션은 JavaScript나 복잡한 프로그래밍 없이 웹 페이지에서 동적이고 시각적으로 매력적인 효과를 만들 수 있게 합니다. 지정한 기간 동안 요소가 서로 다른 스타일로 부드럽게 전환되도록 합니다.
- **`@keyframes` 규칙**: 이 규칙은 애니메이션의 단계와 스타일을 정의합니다. 애니메이션 중 요소가 각 시점에 가져야 할 스타일을 지정합니다.
- **`animation` 속성**: 애니메이션을 적용할 때 사용하는 단축 속성입니다.
- **`animation-name`**: 사용할 `@keyframes` 규칙의 이름을 지정합니다.
- **`animation-duration`**: 애니메이션이 완료되는 데 걸리는 시간을 설정합니다.
- **`animation-timing-function`**: 애니메이션이 시간에 따라 어떻게 진행되는지 정의합니다(예: ease, linear, ease-in-out).
- **`animation-delay`**: 애니메이션이 시작되기 전 지연 시간을 지정합니다.
- **`animation-iteration-count`**: 애니메이션이 몇 번 반복될지 설정합니다.
- **`animation-direction`**: 애니메이션이 앞으로 재생될지, 뒤로 재생될지, 또는 번갈아 재생될지 결정합니다.
- **`animation-fill-mode`**: 애니메이션 전후에 요소가 어떻게 스타일링될지 지정합니다.
- **`animation-play-state`**: 애니메이션을 일시 정지하고 다시 시작할 수 있게 합니다.

## 접근성과 `prefers-reduced-motion` 미디어 쿼리

- **`prefers-reduced-motion` 미디어 쿼리**: 애니메이션과 관련된 주요 접근성 문제 중 하나는 일부 사용자에게 불편함이나 신체적 피해를 줄 수 있다는 점입니다. 전정 장애나 움직임에 민감한 사람들은 화면에서 특정 유형의 움직임에 노출될 때 어지러움, 메스꺼움, 두통을 경험할 수 있습니다. `prefers-reduced-motion` 미디어 쿼리는 웹 개발자가 사용자가 시스템 수준에서 최소한의 애니메이션이나 움직임 효과를 요청했는지 감지할 수 있게 해줍니다.
