# --interactive--

تعيين بالتفكيك للمصفوفة هي ميزة في JavaScript تسمح لك باستخراج القيم من المصفوفات وتعيينها إلى متغيرات بطريقة أكثر إيجازًا وسهولة في القراءة. توفر صيغة نحوية مريحة لفك عناصر المصفوفة إلى متغيرات منفصلة.

تُعد هذه التقنية مفيدة بشكل خاص عند العمل مع المصفوفات والدوال التي تُرجع قيمًا متعددة. إليك مثالًا على استخدام تعيين بالتفكيك للمصفوفة:

:::interactive_editor

```js
let fruits = ["apple", "banana", "orange"];

let [first, second, third] = fruits;

console.log(first);  // "apple"
console.log(second); // "banana"
console.log(third);  // "orange"
```

:::

في هذا المثال، لدينا مصفوفة تسمى `fruits` تحتوي على ثلاثة عناصر. باستخدام تعيين بالتفكيك للمصفوفة، نُعيّن العنصر الأول إلى المتغير `first`، والعنصر الثاني إلى `second`، والعنصر الثالث إلى `third`. هذا يسمح لنا بالوصول بسهولة إلى عناصر المصفوفة الفردية دون استخدام تدوين الفهرس.

إليك كيف سيكون شكل الشفرة إذا وصلت إلى كل من تلك العناصر باستخدام فهرسها بدلًا من استخدام تعيين بالتفكيك للمصفوفة:

:::interactive_editor

```js
const fruits = ["apple", "banana", "orange"];

const first = fruits[0];
const second = fruits[1];
const third = fruits[2];

console.log(first); // "apple"
console.log(second); // "banana"
console.log(third); // "orange"
```

:::

يسمح تعيين بالتفكيك للمصفوفة أيضًا بتخطي العناصر التي لا تهتم بها باستخدام الفواصل. على سبيل المثال:

:::interactive_editor

```js
let colors = ["red", "green", "blue", "yellow"];
let [firstColor, , thirdColor] = colors;

console.log(firstColor); // "red"
console.log(thirdColor); // "blue"
```

:::

في هذا المثال، نتخطى العنصر الثاني من مصفوفة `colors` باستخدام فاصلة إضافية. هذا يعين `red` إلى `firstColor` و`blue` إلى `thirdColor`، متجاهلين بذلك `green` بفعالية.

ميزة قوية أخرى في تعيين بالتفكيك للمصفوفة هي القدرة على استخدام القيم الافتراضية. إذا كانت المصفوفة تحتوي على عناصر أقل من المتغيرات التي تحاول تعيينها، يمكنك توفير قيم افتراضية:

:::interactive_editor

```js
let numbers = [1, 2];
let [a, b, c = 3] = numbers;

console.log(a); // 1
console.log(b); // 2
console.log(c); // 3
```

:::

هنا، نُعيّن القيمة الافتراضية `3` إلى `c` لأن مصفوفة `numbers` لا تحتوي على عنصر ثالث.

الآن، دعنا نناقش صيغة rest، التي تُرمز إليها بثلاث نقاط (`...`). تسمح لك بالتقاط العناصر المتبقية من المصفوفة التي لم تُفكك إلى مصفوفة جديدة. إليك كيف تعمل:

:::interactive_editor

```js
let fruits = ["apple", "banana", "orange", "mango", "kiwi"];
let [first, second, ...rest] = fruits;

console.log(first);  // "apple"
console.log(second); // "banana"
console.log(rest);   // ["orange", "mango", "kiwi"]
```

:::

في هذا المثال، يلتقط `first` و`second` أول عنصرين من مصفوفة `fruits`، ويجمع `rest` كل العناصر المتبقية كمصفوفة جديدة. يجب أن تكون صيغة rest العنصر الأخير في نمط التفكيك.

تعيين بالتفكيك للمصفوفة ميزة قوية يمكن أن تجعل شفرتك أكثر إيجازًا وأسهل في القراءة. هي مفيدة بشكل خاص عند العمل مع المصفوفات، وعندما تحتاج إلى استخراج عناصر محددة من مصفوفة.
