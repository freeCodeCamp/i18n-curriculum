# --description--

「使えるまでの時間」、つまりユーザーがページをリクエストしてから実際に意味のある操作ができるまでの間隔を短くすることは、ユーザー体験を向上させるために非常に重要です。

より速い「使えるまでの時間」を実現するための効果的な戦略を見ていきましょう。

まずはユーザーが最初に目にするコンテンツに注目してください。これは、重要な要素をすぐに読み込み、重要でないコンポーネントは後回しにすることを意味します。

例えば、遅延読み込みを実装してみるのも良いでしょう。

遅延読み込みについてはすでに学びましたが、復習として画像の遅延読み込みの例を示します：

```html
<img src="image.jpg" loading="lazy" alt="Description">
```

画像タグに`loading="lazy"`属性を追加するだけでOKです。これにより、画像は必要になるまで、通常はユーザーのビューポートに入るまで読み込みが遅延されます。画像や動画をビューポートに入ったときだけ読み込むことで、帯域幅を節約し、初期レンダリングの速度も向上します。

通常、ウェブページは画面に表示されていない画像も含めてすべての画像を一度に読み込みます。遅延読み込みを使うと、画面外にある画像はユーザーがスクロールして初めて読み込まれます。

次に、iframeの遅延読み込みの例です：

```html
<iframe src="video.html" loading="lazy"></iframe>
```

先ほどと同じ属性をiframeに適用すれば、同じ考え方が使えます。

また、`defer`属性を使って、重要でないJavaScriptを初期ページ読み込み後まで遅延させることもできます。以下はその例です：

```html
<script src="non-critical.js" defer></script>
```

レンダーブロックするリソースを最小化することも検討してください。

レンダーブロックするリソースは、ページが操作可能になるのを遅らせます。

これを避けるために、CSSを非同期で読み込む方法があります。重要でないCSSには`media`属性を使って条件付きでスタイルシートを読み込むことができます。以下はその例です：

```html
<link rel="stylesheet" href="print.css" media="print">
```

media属性に`print`を指定すると、`print.css`はページが印刷されるか画面でプレビューされるときだけ読み込まれます。次の例では、`min-width: 800px`属性に`media`を渡すことで、画面幅が800px以上のときだけ`desktop.css`を読み込むようにしています：

```html
<!-- This stylesheet is only loaded on screens wider than 800px -->
<link rel="stylesheet" href="desktop.css" media="(min-width: 800px)">
```

これにより不要なスタイルの読み込みを省き、帯域幅を節約できます。

まとめると、これらの戦略を実装することで、ウェブサイトの「使えるまでの時間」を大幅に短縮し、ユーザーがすぐにコンテンツとやり取りできるようにできます。
