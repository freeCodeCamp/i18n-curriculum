# --description--

<ol>
  <li>Nehmen Sie eine ganze Zahl <code>n₀</code>, größer als null.</li>
  <li>Bildung der nächsten Zahl <code>n</code> der Folge, indem Sie <code>n₀</code> umkehren und zu <code>n₀</code> addieren.</li>
  <li>Stoppen Sie, wenn <code>n</code> palindromisch wird – das heißt, die Ziffern von <code>n</code> in umgekehrter Reihenfolge sind gleich <code>n</code>.</li>
</ol>

Die obige Rekursionsvorschrift führt bei den meisten Startzahlen `n` = 1, 2, ... recht schnell zu einem Palindrom.

Zum Beispiel, wenn `n₀` = 12 ist, erhalten wir:

```md
12
12 + 21 = 33,  a palindrome!
```

Und wenn `n₀` = 55 ist, erhalten wir:

```md
55
55 + 55 = 110
110 + 011 = 121,  a palindrome!
```

Beachten Sie, dass die Prüfung auf ein Palindrom *nach* einer Addition erfolgt.

Einige Startzahlen scheinen unendlich weiterzulaufen; die Rekursionsvorschrift für 196 wurde über Millionen von Wiederholungen berechnet und erzeugte Zahlen mit Millionen von Ziffern, ohne ein Palindrom zu bilden. Diese Zahlen, die nicht in einem Palindrom enden, werden **Lychrel-Zahlen** genannt.

Für diese Aufgabe ist eine Lychrel-Zahl jede Startzahl, die innerhalb von 500 (oder mehr) Iterationen kein Palindrom bildet.

**Seed- und verwandte Lychrel-Zahlen:**

Jede ganze Zahl, die in der Folge einer Lychrel-Zahl erzeugt wird, ist ebenfalls eine Lychrel-Zahl.

Im Allgemeinen *kann* jede Folge von einer Lychrel-Zahl konvergieren und sich der Folge eines vorherigen Lychrel-Zahlen-Kandidaten anschließen; zum Beispiel beginnen die Folgen für die Zahlen 196 und dann 689 mit:

```md
    196
    196 + 691 = 887
    887 + 788 = 1675
    1675 + 5761 = 7436
    7436 + 6347 = 13783
    13783 + 38731 = 52514
    52514 + 41525 = 94039
    ...
    689
    689 + 986 = 1675
    1675 + 5761 = 7436
    ...
```

Wir sehen also, dass die Folge, die mit 689 beginnt, konvergiert und mit denselben Zahlen wie die für 196 fortgesetzt wird.

Deshalb können wir die Lychrel-Zahlen weiter in echte **Seed**-Lychrel-Zahlen-Kandidaten und **verwandte** Zahlen aufteilen, die keine Palindrome erzeugen, aber ganze Zahlen in ihrer Folge haben, die auch in der Folge eines niedrigeren Lychrel-Zahlen-Kandidaten vorkommen.

# --solutions--

```js
function isLychrel(n) {
  function reverse(num) {
    return parseInt(
      num
        .toString()
        .split('')
        .reverse()
        .join('')
    );
  }

  var i;
  for (i = 0; i < 500; i++) {
    n = n + reverse(n);
    if (n == reverse(n)) break;
  }

  return i == 500;
}
```
