# --interactive--

`change` 事件是一種特殊事件，當使用者修改某些輸入元素的值時會觸發。更具體來說：

- 當勾選框被勾選或取消勾選時。

- 當圓鈕、單選按鈕被勾選時。

- 當使用者從類似日期選擇器或下拉選單中做出選擇時。

- 當輸入欄位失去焦點（使用者按 Tab 鍵切換到下一個欄位，或點擊表單外部）且使用者已更改值時。

- 當使用者以其他方式確認該值，例如在輸入一些文字後按下 enter。

請注意，當你的使用者在輸入框中輸入時，`change` 事件不會觸發。`change` 事件只會在他們將焦點移到另一個元素後觸發。

以下是使用帶有下拉選單的 `change` 事件的範例：

:::interactive_editor

```html
<select id="select-menu">
  <option value="option1">Option 1</option>
  <option value="option2">Option 2</option>
  <option value="option3">Option 3</option>
</select>
<script src="index.js"></script>
```

```js
const selectMenu = document.getElementById("select-menu");
selectMenu.addEventListener("change", (event) => {
  console.log(`You selected: ${event.target.value}`);
});
```

:::

每次使用者從下拉選單中選擇不同的選項時，`change` 事件將會觸發，且所選的值會被記錄到主控台。

`change` 事件仍然會產生一個 `Event` 物件，但與大多數其他事件不同的是，它不會產生訂製的實作－你唯一能存取的屬性和方法是基礎 `Event` 物件上的那些。
