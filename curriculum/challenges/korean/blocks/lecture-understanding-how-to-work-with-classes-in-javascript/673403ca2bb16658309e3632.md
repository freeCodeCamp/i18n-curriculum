# --interactive--

클래스가 객체를 생성하는 데 사용할 수 있는 청사진과 같다는 것을 배웠습니다.

이 청사진들은 코드 내에서 생성되거나 접근되거나 수정되는 특정 객체를 일반적으로 가리키는 방법이 필요합니다. 바로 `this` 키워드가 그런 역할을 합니다. 이 키워드는 코드가 실행되어야 하는 문맥을 가리킵니다. 메서드 내에서 사용하면 `this`의 값은 해당 메서드와 연결된 객체에 대한 참조가 됩니다. 이렇게 하면 클래스 내 객체의 속성과 메서드에 접근할 수 있고, 메서드를 여러 객체에서 재사용할 수 있습니다.

예를 들어, 여기 `Dessert` 클래스가 있습니다. 이번에는 `hasPeanuts` 속성을 정의하고 있습니다.

```js
class Dessert {
  constructor(hasPeanuts) {
    this.hasPeanuts = hasPeanuts;
  }
}
```

생성자 내에서 `this` 키워드를 사용하고 있다는 점에 주목하세요. 생성자는 클래스의 새 객체가 생성될 때 자동으로 호출된다는 것을 기억하세요.

`this` 키워드는 현재 생성 중인 객체의 속성으로 `hasPeanuts`이 되어야 한다고 프로그램에 알려줍니다.

할당문의 오른쪽은 `hasPeanuts` 매개변수의 값을 새 객체의 `hasPeanuts` 속성에 할당하고 있습니다.

```js
this.hasPeanuts = hasPeanuts;
```

이것은 생성자의 문맥에서의 예지만, `this` 키워드는 다른 메서드 내에서도 사용할 수 있습니다.

여기 약간 변경된 `Dessert` 클래스가 있습니다. 이제 두 개의 속성 `name`과 `price`가 있습니다.

```js
class Dessert {
  constructor(name, price) {
    this.name = name;
    this.price = price;
  }

  showPrice() {
    console.log(`The price of ${this.name} is $${this.price}.`);
  }
}
```

또한 새로운 `showPrice` 메서드가 있습니다. 이 메서드에서는 디저트 객체의 이름과 가격을 포함한 설명 메시지를 콘솔에 출력합니다.

`this` 키워드를 사용해 `showPrice` 메서드 내에서 객체의 속성에 접근하는 방식을 주목하세요.

이 메서드 문맥에서 `this` 키워드는 어떤 값을 가질까요? 메서드가 호출되는 객체를 가리킵니다. 이 경우, 이름이 `"Brownie"`이고 가격이 `$5.99`인 새로운 Dessert 인스턴스를 생성하고 있습니다:

```js
const brownie = new Dessert("Brownie", 5.99);
```

이 객체에서 `.showPrice()` 메서드를 호출하면:

```js
brownie.showPrice();
```

전체 예제는 다음과 같습니다:

:::interactive_editor

```js
class Dessert {
  constructor(name, price) {
    this.name = name;
    this.price = price;
  }

  showPrice() {
    console.log(`The price of ${this.name} is $${this.price}.`);
  }
}

const brownie = new Dessert("Brownie", 5.99);
brownie.showPrice();
console.log(brownie);
```

:::

현재 객체(`brownie`)의 이름과 가격이 문자열에 교체되어 들어갑니다.

이 모든 것은 `this` 키워드 덕분입니다. 이 키워드는 클래스 내에서 현재 객체를 일반적으로 가리키는 방법을 제공합니다. 덕분에 메서드를 여러 객체에서 재사용할 수 있습니다.

이것은 객체 메서드 문맥에서의 예지만, `this` 키워드는 독립 함수와 화살표 함수 내에서도 값을 가집니다.

독립 함수 내에서의 값은 일반적으로 전역 객체(비엄격 모드) 또는 `undefined`(엄격 모드)를 가리킵니다.

화살표 함수 내에서 `this` 키워드의 값은 조금 다릅니다. 화살표 함수는 자신이 정의된 외부 범위에서 `this`의 값을 상속받습니다. 자체 `this` 바인딩을 만들지 않습니다. 즉, `this`의 값은 화살표 함수가 호출되는 위치나 방식이 아니라, 화살표 함수가 정의된 문맥에 따라 달라집니다. 이 때문에 화살표 함수는 콜백이나 문맥 보존에 유용합니다.

하지만 이 때문에 `this` 값은 객체 메서드 내에서 현재 객체를 가리키지 않으므로, 화살표 함수를 사용할 때 이 점을 유념해야 합니다.

다른 함수 내에 정의된 화살표 함수도 마찬가지입니다. 부모 범위에서 `this` 값을 상속받습니다.

`this` 키워드는 객체에 접근하고 조작하는 데 강력한 도구입니다. 이 키워드가 어떻게 작동하는지 이해하는 것은 JavaScript에서 객체지향 프로그래밍을 마스터하는 데 필수적입니다.
