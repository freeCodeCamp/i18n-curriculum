# --description--

有時候會有你比 TypeScript 更了解某個型別的情況。這時候，型別斷言就非常有用。

這裡是一個使用 `querySelector()` 方法存取 `id` 為 `submit` 的元素的範例：

```ts
const submitBtn = document.querySelector("#submit");
```

目前，TypeScript 只知道這個 `submitBtn` 是某種 `Element` 或可能是 `null`。TypeScript 不會查看對應的 HTML 來判斷。

這是使用型別斷言告訴 TypeScript 這是什麼型別元素的好地方。以下是一個範例：

```ts
const submitBtn = document.querySelector("#submit") as HTMLButtonElement;
```

現在，TypeScript 會因為那個型別斷言而將此視為 `button` 元素。 

使用 `as` 關鍵字有其限制。 例如，若沒有 TypeScript 發出錯誤，你就無法執行像這樣的操作：

```ts
// Conversion of type 'string' to type 'number' may be a mistake because neither type sufficiently overlaps with the other. If this was intentional, convert the expression to 'unknown' first.
const age = "awesome" as number;
```

在此範例中，嘗試斷言字串 `"awesome"` 應被視為數字型別並沒有太大意義。TypeScript 僅允許在相容或重疊的型別之間進行型別斷言。

一種消除該錯誤的方法是使用像這樣的雙重斷言：

```ts
const age = "awesome" as unknown as number;
```

即使這在 TypeScript 中可行，但仍不建議這麼做，因為從邏輯上來說這並不合理。

另一種撰寫型別斷言的方法是使用尖括號語法，如下所示：

```ts
const submitBtn = <HTMLButtonElement>document.querySelector("#submit");
```

當使用 `tsx` 檔案時，你需要小心這個語法，因為 TypeScript 會認為你正在嘗試呈現一個組件，並發出像這樣的錯誤：

```ts
// file: index.tsx
// This JSX tag requires 'React' to be in scope, but it could not be found.
// JSX element 'HTMLButtonElement' has no corresponding closing tag.
const submitBtn =  <HTMLButtonElement>document.querySelector("#submit");
```

在大多數情況下，你會看到第一種使用 `as` 關鍵字的語法被使用，而非第二種選項。
