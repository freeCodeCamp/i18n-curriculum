# --description--

Express 앱을 안정적이고 프로덕션 환경에 적합하게 유지하는 방법, 특히 부하가 많거나 종료할 때 어떻게 해야 하는지 궁금했나요?

바로 그때 헬스 체크와 그레이스풀 셧다운이 필요합니다. 이 두 가지 패턴은 앱을 계속 사용할 수 있게 유지하고, 요청을 놓치거나 데이터가 손상되지 않도록 깔끔하게 종료되도록 도와줍니다.

자, 자세히 살펴보겠습니다.

먼저, Express에서 "헬스 체크"가 무엇인지부터 알아볼까요?

헬스 체크는 앱이 정상적으로 작동하는지 알려주는 간단한 라우트입니다.

예를 들어:

```js
app.get('/health', (req, res) => {
  res.status(200).send('OK')
})
```

로드 밸런서나 컨테이너 오케스트레이터—예를 들어 Kubernetes나 AWS Elastic Beanstalk 같은—가 이 라우트를 주기적으로 호출해서 앱이 건강한지 판단할 수 있습니다.

이 라우트가 200 상태 코드를 반환하면 서비스가 정상으로 간주됩니다. 참고로 200 HTTP 상태 코드는 모든 것이 정상임을 의미합니다!

응답하지 않으면 오케스트레이터가 자동으로 서비스를 재시작할 수 있습니다.

그렇다면 "그레이스풀 셧다운"은 무엇일까요?

그레이스풀 셧다운은 서버가 재시작되거나 배포가 진행될 때처럼 앱이 종료되려 할 때 일어나는 과정입니다.

즉시 연결을 끊는 대신, 앱이 현재 작업을 마칠 시간을 줍니다.

Express에서는 `SIGTERM` 신호를 감지할 수 있는데, 이 신호는 프로세스가 종료되어야 할 때 시스템에서 보내는 신호입니다.

예시는 다음과 같습니다:

```js
process.on('SIGTERM', () => {
  server.close(() => {
    console.log('Server closed. Cleaning up...')
    // Close DB connections or other cleanup here
  })
})
```

이 코드는 Express에 새 연결을 받지 말고, 기존 연결이 끝날 때까지 기다렸다가 종료하라고 알려줍니다.

또한 `SIGINT` 신호도 처리해야 하는데, 이 신호는 로컬 개발 중에 Ctrl+C를 눌러 앱을 중지할 때 보내집니다.

`SIGTERM`는 보통 Kubernetes 같은 오케스트레이터가 프로덕션 환경에서 사용하지만, `SIGINT`는 일상적인 개발자 작업 흐름을 위한 신호입니다. 이를 처리하지 않으면 Ctrl+C를 누를 때 프로세스가 즉시 종료되어 설정한 정리 작업이 실행되지 않습니다.

두 신호 모두 같은 그레이스풀 셧다운 동작을 트리거해야 하므로, 개발자들은 보통 두 신호를 모두 감지합니다.

이제 모든 내용을 종합해 봅시다.

앱이 컨테이너에서 실행 중이라고 가정해 보세요. Kubernetes가 업데이트를 위해 `SIGTERM` 신호를 보내 종료를 요청합니다.

기본적인 그레이스풀 셧다운만 있어도 앱은 새 연결을 받지 않고 현재 요청을 마무리합니다. 모든 작업이 끝나면 데이터베이스 연결을 끊고 깔끔하게 종료합니다. 데이터 손실이나 상태 손상 없이요.

하지만 문제가 하나 있습니다 — 로드 밸런서는 앱이 종료 중인 것을 알지 못해 계속 새 트래픽을 보낼 수 있습니다. 이때 아래 보너스 패턴이 도움이 됩니다.

이제 보너스를 하나 드리겠습니다. "종료 전에 앱을 비정상 상태로 표시하기"를 살펴보겠습니다.

종료 전에 헬스 체크가 503 상태 코드를 반환하도록 업데이트할 수 있습니다. 이렇게 하면 로드 밸런서가 비정상 응답을 보고 종료 기간 동안 앱으로 새 트래픽을 보내지 않게 됩니다. 위 코드에서 보여준 것처럼요.

참고: `503` 상태 코드는 "서비스 이용 불가"를 의미합니다. 서버가 일시적으로 요청을 처리할 수 없음을 알리는 신호로, 그레이스풀 셧다운 중에 정확히 발생하는 상황입니다.

```js
let isShuttingDown = false

app.get('/health', (req, res) => {
  if (isShuttingDown) {
    res.status(500).send('Shutting down')
  } else {
    res.status(200).send('OK')
  }
})

process.on('SIGTERM', () => {
  isShuttingDown = true

  server.close(() => {
    console.log('Graceful shutdown complete')
    // Perform cleanup
  })
})
```
