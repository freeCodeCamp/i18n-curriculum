# --description--

변수와 함수 이름 짓기는 깔끔하고 읽기 쉽고 유지보수하기 좋은 코드를 작성하는 데 중요한 부분입니다. 좋은 이름 짓기 관례는 코드를 스스로 문서화하게 만들어 주석을 많이 달 필요를 줄이고, 미래의 자신을 포함한 다른 개발자가 코드를 이해하기 쉽게 만듭니다.

먼저 JavaScript에서 일반적인 이름 짓기 관례부터 시작해 보겠습니다. 이전 수업에서 변수 이름에 카멜 케이스를 사용하는 법을 배웠습니다. 불리언 변수에는 `is`, `has`, 또는 `can` 같은 접두사를 사용하는 것이 흔한 관례입니다. 이렇게 하면 변수가 불리언임을 바로 알 수 있습니다.

```js
let isLoading = true;
let hasPermission = false;
let canEdit = true;
```

함수 이름은 함수가 무엇을 하는지 명확히 나타내야 합니다. 동사로 시작하는 것이 도움이 되는 경우가 많습니다.

```js
function getUserData(){
  /* ... */
}

function calculateTotal(){
  /* ... */
}

function validateInput(){
  /* ... */
}
```

불리언을 반환하는 함수, 즉 술어 함수에는 `is`, `has`, 또는 `can` 같은 접두사를 사용할 수 있습니다.

```js
function isValidEmail(email) {
  /* ... */
}

function hasRequiredFields(form) {
  /* ... */
}
```

데이터를 가져오는 함수에는 보통 `get`라는 단어로 시작합니다.

```js
function getProductDetails(productId) {
  /* ... */
}

function getUserProfile(userId) {
  /* ... */
}
```

데이터를 설정하는 함수에는 보통 `set`라는 단어로 시작합니다.

```js
function setUserPreferences(preferences) {
  /* ... */
}

function setPageTitle(title) {
  /* ... */
}
```

이벤트 핸들러 함수에는 `handle`를 접두사로 붙이거나 `handler`를 접미사로 붙일 수 있습니다.

```js
function handleClick(){
  /* ... */
}

function onSubmit(){
  /* ... */
}

function keyPressHandler(){
  /* ... */
}
```

이벤트 핸들러는 버튼 클릭 같은 이벤트가 발생한 후에 실행되는 동작입니다. 이 부분은 앞으로 수업에서 배우게 됩니다.

반복자 변수와 반복문 이름을 지을 때는 `i`, `j`, 또는 `k` 같은 한 글자 이름을 사용하는 것이 흔하지만, 중첩 반복문이나 더 복잡한 반복에서는 더 설명적인 이름이 도움이 될 수 있습니다.

```js
for (let i = 0; i < array.length; i++) {
  /* ... */
}

for (let studentIndex = 0; studentIndex < students.length; studentIndex++) {
  /* ... */
}
```

배열 이름에는 변수에 여러 항목이 들어 있음을 나타내기 위해 복수형 명사를 사용하는 것을 고려하세요.

```js
const colors = ['red', 'green', 'blue'];
const userNames = ['Alice', 'Bob', 'Charlie'];
```

목표는 코드를 가능한 한 스스로 설명할 수 있게 만드는 것입니다. 변수나 함수가 무엇을 하는지 설명하기 위해 주석을 달아야 한다면, 더 설명적인 이름으로 바꾸는 것을 고려해 보세요.

마지막으로, 코드베이스나 팀 내에서 일관성을 유지하세요. 팀에서 이미 이름 짓기 관례를 정해 두었다면 그것을 따르세요. 일관성은 모든 사람이 코드를 더 읽기 쉽고 유지보수하기 쉽게 만듭니다.
