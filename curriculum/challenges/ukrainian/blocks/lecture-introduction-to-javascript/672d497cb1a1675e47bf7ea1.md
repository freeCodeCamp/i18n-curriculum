# --interactive--

У JavaScript змінні виконують роль контейнерів для зберігання даних, до яких ви можете отримувати доступ і змінювати їх протягом виконання програми.

Ви можете уявити змінні як блоки, які тримають значення. За допомогою змінних ви можете відстежувати такі речі, як числа або текст, і звертатися до цих значень, коли вони потрібні у вашій програмі.

Один зі способів оголосити змінну в JavaScript — використати ключове слово `let`. Ви також дізнаєтеся більше про ключове слово `let` та інші способи оголошення змінних у майбутніх уроках.

Ось приклад використання `let` для оголошення змінної з ім’ям `age`:

```js
let age;
```

Наразі змінна `age` не має призначеного значення. Якщо ви спробуєте її використати, вона поверне `undefined`, що означає, що вона не має значення.

Ось приклад.

**ПРИМІТКА**: `console.log()` — це функція, яка виводить інформацію у консоль, що є частиною вашого веб-браузера, призначеною для налагодження коду. Ви дізнаєтеся більше про `console.log()` у майбутніх уроках. Також символи `//` використовуються для додавання коментарів у ваш код. Коментарі — це нотатки для себе або інших програмістів, які ігноруються під час виконання коду.

:::interactive_editor
 
```js
let age;
console.log(age); // undefined
```

:::

Щоб призначити значення змінній, потрібно використати оператор присвоєння ось так:

```js
let age = 25;
```

Тепер, коли ви використовуєте змінну `age`, вона поверне значення `25`.

:::interactive_editor

```js
let age = 25;
console.log(age); // 25
```

:::

Оператор присвоєння виглядає як знак рівності (`=`), але він не перевіряє рівність. Ви дізнаєтеся про правильні оператори для перевірки рівності у майбутніх уроках.

Оператор присвоєння використовується для призначення значення змінній. Цей процес призначення значення змінній називається ініціалізацією.

Однією з переваг використання ключового слова `let` для оголошення змінних є те, що ви можете повторно призначати їм значення. У програмуванні повторне присвоєння означає надання нового значення змінній, яка вже має значення.

Ось приклад повторного присвоєння значення змінній `age`.

:::interactive_editor

```js
let age = 25;
console.log(age); // 25
age = 30;
console.log(age); // 30
```

:::

Тепер змінна `age` містить значення `30`. Зверніть увагу, що ключове слово `let` більше не було потрібне, бо змінна `age` вже була оголошена, тому немає потреби оголошувати її вдруге.

При повторному присвоєнні потрібно лише посилатися на ім’я змінної. Повторне присвоєння корисне, бо дозволяє оновлювати і змінювати значення, що зберігається у змінній, під час виконання програми. Хорошим прикладом цього є оновлення балів у грі.

Іменування змінних може здаватися простим, але існують певні правила та найкращі практики, щоб ваш код був читабельним і функціональним.

Імена ваших змінних мають описувати, що саме представляють дані. Наприклад, замість імені `x` більш описове ім’я, таке як `age` або `points`, робить ваш код легшим для розуміння.

```js
// Bad variable names
let x = 10;
let y = "John";

// Good variable names
let age = 10;
```

Змінні в JavaScript мають починатися з літери, підкреслення (`_`) або знака долара (`$`). Вони не можуть починатися з числа.

```js
// Valid variable names
let age;
let _score;
let $total;

// Invalid variable names
let 1stPlace; // starts with a number
```

Імена змінних чутливі до регістру, тобто слово `age` у нижньому регістрі і слово `Age` з великою `A` вважаються різними змінними.

:::interactive_editor

```js
let age = 25;
let Age = 30;
console.log(age); // 25
console.log(Age); // 30
```

:::

Ось чому важливо дотримуватися послідовної конвенції іменування, наприклад верблюжого регістру (camelCase). Верблюжий регістр — це коли перше слово повністю у нижньому регістрі, а кожне наступне слово починається з великої літери.

Ось приклад використання конвенції верблюжого регістру для змінної:

```js
let thisIsCamelCase;
let anotherExampleVariable;
let freeCodeCampStudents;
```

У JavaScript є певні ключові слова, які не можна використовувати як імена змінних, наприклад `let`, `const`, `function` або `return`, оскільки вони зарезервовані для самої мови.

Також слід уникати використання спеціальних символів, таких як знак оклику (`!`) або символ собачки (`@`) у іменах змінних. Найкраще робити імена змінних читабельними, використовуючи літери, числа, підкреслення або знаки долара.

Дотримуючись цих рекомендацій, ваш код буде чистішим і легшим у підтримці, коли він ускладнюватиметься.
