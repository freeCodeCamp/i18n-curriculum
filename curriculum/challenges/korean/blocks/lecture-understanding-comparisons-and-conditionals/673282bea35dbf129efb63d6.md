# --interactive--

`switch` 문과 `if/else if/else` 체인은 모두 프로그래밍에서 특정 조건에 따라 다른 코드 블록을 실행할 수 있게 하는 제어 흐름 구조입니다. 하지만 각각 고유한 특징과 사용 사례가 있습니다.

`switch` 문은 표현식을 평가하고 그 값을 일련의 `case` 절과 비교합니다. 일치하는 값이 발견되면 해당 `case`와 연결된 코드 블록이 실행됩니다. `switch` 문의 기본 구조는 다음과 같습니다.

```js
switch (expression) {
  case value1:
    // code to be executed if expression === value1
    break;
  case value2:
    // code to be executed if expression === value2
    break;
  default:
    // code to be executed if expression doesn't match any case
}
```

각 `break` 끝에 있는 `case` 문은 매우 중요합니다. 일치하는 `case`가 실행된 후 switch 블록을 종료하도록 프로그램에 지시합니다. 이것이 없으면 프로그램은 이후의 케이스들을 계속 실행하는데, 이를 "fall-through"라고 합니다.

`switch` 문은 일반적으로 하나의 변수를 여러 가능한 값과 비교할 때 사용합니다. 특히 하나의 변수에 대해 많은 조건을 확인해야 할 때 유용합니다. 다음은 요일에 대해 `switch` 문을 사용한 예입니다.

:::interactive_editor

```js
let dayOfWeek = 3; 

switch (dayOfWeek) {
    case 1:
        console.log("It's Monday! Time to start the week strong.");
        break;
    case 2:
        console.log("It's Tuesday! Keep the momentum going.");
        break;
    case 3:
        console.log("It's Wednesday! We're halfway there.");
        break;
    case 4:
        console.log("It's Thursday! Almost the weekend.");
        break;
    case 5:
        console.log("It's Friday! The weekend is near.");
        break;
    case 6:
        console.log("It's Saturday! Enjoy your weekend.");
        break;
    case 7:
        console.log("It's Sunday! Rest and recharge.");
        break;
    default:
        console.log("Invalid day! Please enter a number between 1 and 7.");
}
```

:::

`switch` 문은 하나의 변수에 대해 많은 가능한 값을 다룰 때 더 읽기 쉽고 간결할 수 있습니다.

반면 `if/else if` 문은 더 유연합니다. 각 절에서 복잡한 조건과 서로 다른 변수를 평가할 수 있어 더 다양한 상황에 적합합니다. 다음은 `if/else` 문을 `switch` 문 대신 사용할 수 있는 예입니다.

:::interactive_editor

```js
let creditScore = 720; 
let annualIncome = 60000; 
let loanAmount = 200000; 

let eligibilityStatus;

if (creditScore >= 750 && annualIncome >= 80000) {
    eligibilityStatus = "Eligible for premium loan rates.";
} else if (creditScore >= 700 && annualIncome >= 50000) {
    eligibilityStatus = "Eligible for standard loan rates.";
} else if (creditScore >= 650 && annualIncome >= 40000) {
    eligibilityStatus = "Eligible for subprime loan rates.";
} else if (creditScore < 650) {
    eligibilityStatus = "Not eligible due to low credit score.";
} else {
    eligibilityStatus = "Not eligible due to insufficient income.";
}

console.log(eligibilityStatus);
```

:::

이 예에서는 한 사람의 연간 소득과 신용 점수를 가지고 어떤 종류의 대출 자격이 되는지 확인합니다. 복잡한 논리 평가와 여러 변수를 다루기 때문에 `if/else` 문을 `switch` 문보다 사용하는 것이 더 낫습니다.

JavaScript의 `switch` 문은 엄격한 비교(`===`)를 사용한다는 점도 주목할 만합니다. 즉, 타입 강제 변환을 수행하지 않습니다. 이는 예측 가능성과 미묘한 버그를 피하는 데 장점이 될 수 있습니다.

요약하자면, `switch` 문과 `if/else if` 체인 모두 코드에서 다중 분기 로직을 허용하지만 각각 강점이 다릅니다. `switch` 문은 하나의 변수에 대해 여러 가능한 값을 처리하는 데 뛰어나고, `if/else if` 체인은 복잡한 조건에 더 유연합니다. 둘 중 선택은 코드의 구체적인 요구 사항과 개인 또는 팀의 코딩 스타일 선호도에 따라 달라집니다.
