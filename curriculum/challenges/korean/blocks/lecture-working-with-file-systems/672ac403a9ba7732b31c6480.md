# --description--

파일은 내용과 구조에 따라 분류됩니다. 파일 유형은 컴퓨터 프로그램이 파일을 어떻게 열고 처리할지 결정합니다.

파일 확장자를 통해 파일 유형을 식별합니다. 파일 확장자는 파일 이름 끝에 붙는 세 글자 또는 네 글자의 접미사입니다. 웹 애플리케이션에서 주로 사용하는 세 가지 주요 파일 유형은 HTML, CSS, 그리고 JavaScript입니다.

HTML 파일은 웹 애플리케이션의 구조와 내용을 담고 있습니다. 확장자는 `.html`입니다.

CSS 파일은 스타일을 정의합니다. 확장자는 `.css`입니다.

JavaScript 파일은 더 고급 기능과 상호작용을 추가합니다. 확장자는 `.js`입니다.

웹 애플리케이션을 개발할 때 이미지를 포함해야 할 때도 있습니다. 다음은 가장 널리 사용되는 이미지 파일 유형입니다.

JPEG는 "Joint Photographic Experts Group"의 약자로, 효율적인 압축으로 유명합니다. 사진과 이미지에 적합합니다. 하지만 압축이 손실 압축이어서 파일 크기를 줄이기 위해 일부 이미지 품질이 손실됩니다.

PNG는 "Portable Network Graphics"의 약자로, 압축 후에도 이미지 품질을 유지합니다. 이로 인해 파일 크기가 더 큽니다. PNG는 투명도를 지원하며, 로고나 아이콘처럼 선명한 가장자리를 가진 이미지에 적합합니다.

GIF 이미지 형식은 애니메이션과 투명도를 지원하지만 색상 팔레트가 제한적입니다. GIF는 "Graphics Interchange Format"의 약자입니다.

마지막으로 SVG는 "Scalable Vector Graphics"의 약자로, 벡터 이미지 형식입니다. 이 이미지는 품질 손실 없이 원하는 만큼 확대할 수 있습니다.

좋습니다. 이미지 형식을 알았으니 이제 비디오와 오디오 형식을 살펴보겠습니다.

MP3는 효율적인 압축으로 알려진 오디오 형식입니다. 손실 압축 형식이라 파일 크기를 줄이기 위해 일부 오디오 데이터가 손실됩니다.

MP4는 가장 일반적인 비디오 형식 중 하나입니다. 좋은 압축률을 제공하며 여러 오디오 및 비디오 코덱, 자막, 메타데이터를 지원합니다.

MOV는 Apple이 개발한 QuickTime 멀티미디어 파일 형식으로, 주로 QuickTime Player와 연관됩니다.

다른 인기 있는 오디오 및 비디오 파일 유형으로는 원본 오디오 품질을 유지하는 무손실 오디오 형식인 WAV와 고품질 오픈 소스 비디오 형식인 WebM이 있습니다.

웹 애플리케이션의 폰트를 사용자 지정해야 할 경우 폰트 형식도 다루게 됩니다. 가장 널리 사용되는 세 가지입니다.

TTF는 "TrueType Font"의 약자로, 다양한 운영체제와 호환 가능한 폰트 형식입니다.

WOFF는 웹 개발을 위해 특별히 설계된 현대적인 폰트 형식입니다. "Web Open Font Format"의 약자입니다. 더 나은 압축으로 파일 크기가 작으며, 라이선스 정보를 포함한 메타데이터도 저장할 수 있습니다.

WOFF2는 WOFF의 후속 형식으로, 더 효율적인 압축과 성능을 제공합니다.

여러 폴더와 파일을 묶어야 할 경우 아카이브 파일을 만들 수도 있습니다. 가장 널리 사용되는 아카이브 형식은 ZIP입니다. ZIP은 무손실 압축을 제공하여 데이터 손실 없이 압축합니다. 운영체제와 소프트웨어 애플리케이션에서 널리 지원됩니다.

마지막으로 문서에 대해 이야기해 보겠습니다. 웹 애플리케이션에서는 보통 `README`라는 파일을 찾아볼 수 있는데, 이 파일에는 애플리케이션 사용법, 설치 방법, 라이선스, 기여 방법 등의 정보가 담겨 있습니다.

이 문서들은 보통 마크다운이라는 파일 형식으로 작성됩니다. 마크다운을 사용하면 제목, 부제목, 링크, 이미지, 목록 등으로 구조화된 문서를 만들 수 있습니다. 마크다운 파일은 `.md` 또는 `.markdown` 확장자를 가집니다.

여기 freeCodeCamp GitHub 저장소의 `README` 파일이 있습니다: https://github.com/freeCodeCamp/freeCodeCamp/blob/main/README.md

마크다운을 사용해 이렇게 상세한 구조와 형식을 만들 수 있습니다.

경력 동안 많은 `README` 파일을 읽고 쓸 것입니다. 이러한 일반적인 파일 유형을 아는 것은 웹 개발에 필수적입니다.

HTML, CSS, JavaScript라는 핵심 구성 요소부터 이미지, 비디오, 오디오 형식까지, 모든 파일 유형은 상호작용하는 웹 애플리케이션을 만드는 데 중요한 역할을 합니다.
