# --description--

تحتاج المواقع والتطبيقات إلى جمع البيانات ومعالجتها، أو توليد شيء يحتاجه المستخدم مثل نتائج البحث. لذلك، تمامًا مثل النصوص، تُعد النماذج جزءًا أساسيًا من الويب.

من خلال النماذج، يمكن للمستخدمين التسجيل وتسجيل الدخول إلى موقع، إرسال رسائل، والبحث عن عناصر. على سبيل المثال، تحصل على وصول إلى صفحة نتائج بحث Google بكتابة استعلامك في أكثر النماذج شعبية على الويب، وهو حقل إدخال بحث Google.

لننظر إلى مكونات النموذج، مثل حقول الإدخال، خاصية `action`، طرق النموذج، والأهم من ذلك، ماذا يحدث عند إرسال النموذج.

تُضمّن النماذج في صفحة ويب باستخدام عنصر `form`، ولكن لعرض المدخلات تحتاج أيضًا إلى عناصر `input` حسب حاجتك. بعض هذه العناصر هي نص، بريد إلكتروني، كلمة مرور، اختيار، ملف، مدخلات مخفية، والمزيد.

لاستخدام هذه الحقول، يجب تغليفها بعنصر `form`. يحتاج كل مدخل إلى خاصية `name` ليعرف الخادم البيانات التي تتعلق بكل حقل.

إليك مثالاً:

```html
<form>
   <input type="text" name="username" />
   <input type="email" name="email" />
   <input type="file" name="profilePicture" />
   <input type="radio" name="gender" />
   <input type="checkbox" name="subscribe" />
   <input type="hidden" name="userId" />
</form>
```

تحتاج حقول الإدخال أيضًا إلى التحقق لضمان إدخال المستخدم للمعلومات الصحيحة.

يمكن أن يحدث هذا التحقق على جانب العميل وجانب الخادم. يتيح لك HTML إجراء تحقق أساسي على جانب العميل باستخدام خاصيتي `required` و`pattern`. ومع ذلك، يجب أيضًا التحقق على جانب الخادم لأمان إضافي، إذ أن تحقق HTML موجه أساسًا لتحسين تجربة المستخدم والتغذية الراجعة الفورية.

تحدد خاصية `required` أن حقل الإدخال يجب أن يُملأ، بينما تتيح خاصية `pattern` إدخال تعبيرات نمطية تجبر المستخدم على كتابة المعلومات بالطريقة التي تحتاجها.

إليك مثالًا لذلك لحقل اسم المستخدم:

```html
<input type="text" name="username" required pattern="[A-Za-z0-9]{3,}" />
```

بعد التحقق من حقول الإدخال، يجب أن يتم الإرسال. فماذا يحدث عند إرسال النموذج؟

أولًا، لكي يحدث الإرسال، يجب أن يحتوي النموذج على خاصية `action` تحدد إلى أين يرسل المتصفح قيم حقول الإدخال، وخاصية `method` تحدد كيفية إرسال بيانات النموذج. سنتحدث أكثر عن خاصية `method` لاحقًا.

إليك كيف يمكن أن تبدو خاصية action:

```html
<form action="/register">
	<!-- input fields -->
</form>
```

يمكن أن تكون خاصية `method` إما `GET`، `POST`، `PUT`، `PATCH`، أو `DELETE`:

* `GET` يسترجع شيئًا من الخادم، مثل نتائج البحث.
    
* `POST` يرسل شيئًا إلى الخادم، مثل معلومات تسجيل الدخول أو التسجيل المدخلة عبر حقول الإدخال.
    
* `PUT` يعدل شيئًا على الخادم.
    
* `PATCH` يعدل شيئًا جزئيًا على الخادم.
    
* `DELETE` يحذف شيئًا من الخادم.
    
لاحظ أن HTML لا يدعم طرق `PUT` و`PATCH` و`DELETE` مباشرة، لكن هناك حلول بديلة. إذا كنت تستخدم JavaScript عادي، يمكنك استخدام Fetch API لتحديد الطريقة، وفي Node.js وExpress يمكنك استخدام حزمة `method-override`.

إليك كيف يمكن أن يبدو نموذج يقوم بطلب `GET`:

```html
<form action="/search" method="GET">
   <input type="text" name="q">
   <button type="submit">Search</button>
</form>
```

وهنا كيف يمكن أن يبدو نموذج يقوم بطلب `POST`:

```html
<form action="/submit" method="POST">
   <input type="text" name="username" required pattern="[A-Za-z0-9]{3,}" />
   <input type="password" name="password" />
   <button type="submit">Submit</button>
</form>
```

بعد طلب `GET` أو `POST`، يتحقق الخادم من التحقق على جانب الخادم ويعالج أو يرفض قيم الإدخال.

إذا نجح التحقق، يرسل الخادم المعلومات المطلوبة أو يرمي خطأ إذا فشل التحقق.

إذا نجح التحقق، قد تكون الاستجابة من الخادم إعادة توجيه في حالة التوثيق أو البحث، أو رسالة نجاح تؤكد الإجراء.

إليك ملخص لما يحدث عند إرسال نموذج:

* يملأ المستخدم حقول النموذج.
    
* يتحقق المتصفح من التحقق على جانب العميل (إن وجد) ويستمر إذا نجح التحقق.
    
* يرسل المتصفح بيانات النموذج إلى الخادم بناءً على خاصيتي action وmethod.
    
* يستقبل الخادم البيانات ويعالجها (تخزينها في قاعدة بيانات، تسجيل مستخدم، تسجيل دخول مستخدم، إرسال استعلام بحث، وهكذا).
    
* إذا كان هناك تحقق على جانب الخادم، يعيد الخادم فحص البيانات قبل المعالجة.
    
* يرد الخادم برسالة نجاح، خطأ، أو يعيد توجيه المستخدم بناءً على حالة التحقق (نجاح أو فشل).
    
* يرى المستخدم تغذية راجعة بناءً على استجابة الخادم.
