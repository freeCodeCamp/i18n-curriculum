# --interactive--

讓我們來了解嘗試為像 `datetime-local` 和 `color` 輸入這類特殊輸入元素設計樣式時的一些常見問題。

這些特殊型別的輸入依賴複雜的偽元素來創建像是日期和顏色選擇器之類的東西。這對於這些輸入的樣式設計帶來了重大挑戰。一個挑戰是，由於預設樣式完全依賴瀏覽器，讓選擇器在一個瀏覽器中看起來正確的 CSS，可能在另一個瀏覽器中產生非常不同的結果。

這裡是一個顏色輸入的範例：

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<form>
  <label for="favorite-color">Pick your favorite color:</label>
  <input type="color" id="favorite-color" name="favorite-color">
</form>
```

```css
input {
  padding: 8px 12px;
  margin: 8px 0;
  border-radius: 6px;
  border: 1px solid #ccc;
}

input[type="color"] {
  width: 60px;
  height: 40px;
  padding: 0;
  border: 2px solid #555;
  border-radius: 4px;
  cursor: pointer;
}

```

:::

另一個可能是偽元素的複雜性。以日期選擇器為例；這裡有許多活動的零件，而偽元素的複雜結構可能會對在正確區域套用樣式造成重大挑戰。

這裡是一個日期輸入的範例：

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<form>
  <label for="birthdate">Select your birthdate:</label>
  <input type="date" id="birthdate" name="birthdate">
</form>
```

```css
input {
  padding: 8px 12px;
  margin: 8px 0;
  border-radius: 6px;
  border: 1px solid #ccc;
}

input[type="date"] {
  padding: 6px 10px;
  border: 2px solid #555;
  border-radius: 4px;
  font-size: 14px;
  cursor: pointer;
}

input[type="date"]::-webkit-calendar-picker-indicator {
  background-color: #4CAF50;
  color: white;
  border-radius: 4px;
  cursor: pointer;
}

```

:::

當然，使用這些複雜的元素時，你也有可能在手動設定樣式時意外遺失重要的功能。不僅可能遺失像是焦點狀態或已選擇項目的重要指示，還可能完全破壞選擇器。  

基於這些原因，許多開發者完全依賴 JavaScript 函式庫或訂製組件，而非使用瀏覽器內建的組件。
