# --interactive--

이전 수업에서 불리언 개념을 처음 접했지만, 이번 수업에서는 불리언이 어떻게 작동하는지 그리고 동등 연산자와 부등 연산자가 어떻게 작동하는지 더 깊이 알아보겠습니다.

불리언은 `true`와 `false` 값만 가지는 데이터 타입입니다. 조건에 따라 무언가를 실행할 수 있게 해주기 때문에 유용합니다. 불리언은 앱에서 특정 기능에 접근할 수 있는지 결정하는 것처럼 어떤 일이 일어나야 하는지 평가할 때 필수적입니다. 다음은 `true` 값을 `isOldEnoughToDrive`라는 변수에 설정하는 예입니다:

:::interactive_editor

```js
let isOldEnoughToDrive = true;

console.log(isOldEnoughToDrive); // true
```

:::

이 변수를 조건문 안에서 이렇게 사용할 수 있습니다:

:::interactive_editor

```js
let isOldEnoughToDrive = true;

if (isOldEnoughToDrive) {
 console.log("You're old enough to drive"); // You're old enough to drive
} else {
 console.log("Sorry, you are not old enough to drive");
}
```

:::

조건문은 조건에 따라 코드에서 결정을 내리는 데 도움을 줍니다. 이 예제는 `if/else` 문이라고 부르는 것을 사용합니다.

만약 `isOldEnoughToDrive`가 `true`라면, 문장 `You're old enough to drive`이 콘솔에 기록됩니다. 그렇지 않고 `isOldEnoughToDrive`가 `false`라면, 문장 `Sorry, you are not old enough to drive`이 콘솔에 기록됩니다. `isOldEnoughToDrive` 변수가 `true`로 설정되어 있으므로 첫 번째 문장이 콘솔에 기록됩니다. `if/else` 문에 대해서는 이후 수업에서 더 배우게 됩니다.

두 값을 비교하려면 동등 연산자나 엄격한 동등 연산자 중 하나를 사용할 수 있습니다. 비교 결과는 `true` 또는 `false`인 불리언이 됩니다. 다음은 문자열과 숫자를 비교하기 위해 동등 연산자를 사용하는 예입니다. 동등 연산자는 등호 두 개(`==`)로 나타냅니다.

:::interactive_editor

```js
console.log(5 == "5"); // true
```

:::

이 예제에서 JavaScript는 문자열 `"5"`을 숫자 `5`로 변환한 후 두 값이 같은지 확인합니다. 두 값이 같으므로 결과는 true입니다. 동등 연산자는 값을 비교하기 전에 타입 강제 변환을 사용합니다.

이는 타입 강제 변환을 수행하지 않는 엄격한 동등 연산자와 다릅니다. 엄격한 동등 연산자는 타입이 같은지 그리고 값이 같은지 확인합니다. 다음은 숫자와 문자열을 비교하기 위해 엄격한 동등 연산자를 사용하는 예입니다. 이 연산자는 등호 세 개(`===`)로 나타냅니다.

:::interactive_editor

```js
console.log(5 === '5'); // false
```

:::

다음 비교는 `false`가 될 것입니다. 문자열 데이터 타입은 숫자 데이터 타입과 같지 않기 때문입니다. 만약 어떤 값이 다른 값과 같지 않은지 확인해야 한다면, 부등 연산자나 엄격한 부등 연산자를 사용할 수 있습니다. 다음은 숫자와 문자열을 비교하기 위해 부등 연산자(`!=`)를 사용하는 예입니다.

:::interactive_editor

```js
console.log(5 != "5"); // false
```

:::

이 예제에서 결과는 `false`가 됩니다. 부등 연산자는 먼저 문자열 값을 숫자로 변환한 후 값을 비교하기 때문입니다. 값이 같으므로 `false`를 반환합니다. 만약 엄격한 부등 연산자를 사용했다면 다른 결과가 나왔을 것입니다. 엄격한 부등 연산자는 느낌표와 등호 두 개(`!==`)로 나타냅니다.

:::interactive_editor

```js
console.log(5 !== "5"); // true
```

:::

결과는 `true`가 됩니다. 엄격한 부등 연산자는 타입 강제 변환을 수행하지 않기 때문입니다. 숫자 `5`는 문자열 `"5"`와 같지 않으므로 결과는 `true`입니다.

타입 강제 변환을 수행하지 않으므로 가능한 한 엄격한 부등 연산자와 엄격한 동등 연산자를 사용하는 것이 모범 사례로 간주됩니다. 대부분의 전문 프로젝트에서는 부등 연산자와 동등 연산자보다 이 두 연산자를 선호하는 코드베이스를 볼 수 있습니다.
