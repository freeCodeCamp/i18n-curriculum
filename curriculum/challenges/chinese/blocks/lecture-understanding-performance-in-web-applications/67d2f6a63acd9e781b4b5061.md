# --description--

为什么有些网站感觉快速且响应灵敏，而有些则感觉迟缓？答案在于影响页面装载和渲染的关键性能概念。理解关键的网页性能概念对于创建快速、流畅且用户友好的网站至关重要。 

让我们分解 source order、关键渲染路径、延迟等内容。

我们先来看源顺序。

源顺序指的是 HTML 元素在文档中的结构方式。这决定了什么内容先装载，并且会显著影响性能和无障碍。

一些最佳实践包括：

- 将关键内容（例如标题、导航或主要文本）放置在 `HTML` 结构的更高位置。

- 推迟非必要的脚本，例如用于分析的脚本或第三方小部件，以免它们阻塞渲染。

- 使用渐进增强，确保核心体验即使在样式和脚本装载之前也能正常工作。渐进增强是一种基于先使页面使用超文本标记语言正常工作的理念来创建网站和应用的方法。

这是一个良好源顺序的示例，使用了我们刚刚讲解的最佳实践。

```html
<!-- Good source order: Essential content first -->
<h1>Welcome to FastSite!</h1>
<p>Critical information loads first.</p>
<script src="slow-script.js" defer></script>
```

通过优化源顺序，我们确保用户尽快看到重要内容。

现在让我们来看关键渲染路径。

关键渲染路径是浏览器将代码转换为屏幕上像素所遵循的步骤序列。

以下是我们稍后将详细介绍的关键步骤：

1. 解析超文本标记语言：构建 DOM（文档对象模型）

2. 解析 CSS：构建 CSSOM（CSS 对象模型）

3. JavaScript 执行：可以修改 DOM 和 CSSOM

4. 渲染树构造：结合 DOM 和 CSSOM

5. 版型与绘制：确定元素大小并绘制像素

6. 优化：

    - 最小化阻塞 `Render` 的资源（例如，大型 CSS 文件、未使用的 JS）。

    - 为脚本使用 async 和 defer 属性：

    - 首先只装载必要的样式；推迟非关键的 CSS。

总体而言，更短的关键渲染路径等于更快的感知性能。我们将在后面更详细地讨论这一点。

最后，让我们来看一下延迟。

延迟是指一个 `request` 在浏览器和 `server` 之间传输所花费的时间。换句话说，高延迟等于页面加载缓慢。

减少延迟的一些方法包括：

- 使用 CDN，换句话说，内容分发网络，从更近的位置提供文件。

- 使用诸如 Gzip 等工具启用压缩以减小文件大小。

- 优化图像和使用惰性装载——我们稍后也会详细讲解。

通过减少延迟，我们使交互感觉即时。

总之，通过优化资源顺序、减少关键路径和降低延迟，你可以让你的网站感觉更快。
