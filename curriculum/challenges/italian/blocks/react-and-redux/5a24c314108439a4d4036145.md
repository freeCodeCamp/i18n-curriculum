# --description--

Il componente `Provider` ti permette di fornire `state` e `dispatch` ai tuoi componenti React, ma devi specificare esattamente quale stato e quali azioni vuoi. In questo modo, ti assicuri che ogni componente abbia accesso solo allo stato di cui ha bisogno. Lo ottieni creando due funzioni: `mapStateToProps()` e `mapDispatchToProps()`.

In queste funzioni dichiari quali parti dello stato vuoi consultare e quali action creator devi poter dispatchare. Una volta che queste funzioni sono pronte, vedrai come usare il metodo `connect` di React Redux per collegarle ai tuoi componenti in un'altra sfida.

**Nota:** Dietro le quinte, React Redux usa il metodo `store.subscribe()` per implementare `mapStateToProps()`.

# --instructions--

Crea una funzione `mapStateToProps()`. Questa funzione deve prendere `state` come argomento, quindi restituire un oggetto che mappa quello stato a nomi di proprietà specifici. Queste proprietà diventeranno accessibili al tuo componente tramite `props`. Poiché in questo esempio l'intero stato dell'app è contenuto in un singolo array, puoi passare quell'intero stato al tuo componente. Crea una proprietà `messages` nell'oggetto che viene restituito e impostala su `state`.

# --hints--

La const `state` dovrebbe essere un array vuoto.

```js
assert(Array.isArray(state) && state.length === 0);
```

`mapStateToProps` dovrebbe essere una funzione.

```js
assert(typeof mapStateToProps === 'function');
```

`mapStateToProps` dovrebbe restituire un oggetto.

```js
assert(typeof mapStateToProps() === 'object');
```

Passare un array come stato a `mapStateToProps` dovrebbe restituire questo array assegnato a una chiave `messages`.

```js
assert(mapStateToProps(['messages']).messages.pop() === 'messages');
```
