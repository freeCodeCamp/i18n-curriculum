# --description--

Qui passeremo dagli alberi binari di ricerca a un altro tipo di struttura ad albero chiamata trie. Un trie è un albero di ricerca ordinato comunemente usato per contenere stringhe, o più genericamente array associativi o dataset dinamici in cui le chiavi sono stringhe. Sono molto efficaci nell’archiviare insiemi di dati quando molte chiavi hanno prefissi sovrapposti, per esempio tutte le parole in un dizionario. A differenza di un albero binario, i nodi non sono associati a valori effettivi. Invece, il percorso verso un nodo rappresenta una chiave specifica. Per esempio, se volessimo archiviare la stringa code in un trie, avremmo quattro nodi, uno per ogni lettera: c — o — d — e. Seguire quel percorso attraverso tutti questi nodi creerà la stringa code — quel percorso è la chiave che abbiamo archiviato. Poi, se volessimo aggiungere la stringa coding, condivideremmo i primi tre nodi di code prima di diramare dopo la d. In questo modo, grandi dataset possono essere archiviati in modo molto compatto. Inoltre, la ricerca può essere molto veloce perché è effettivamente limitata alla lunghezza della stringa che stai archiviando. Inoltre, a differenza degli alberi binari un nodo può contenere un numero qualsiasi di nodi figli. Come avrai intuito dall’esempio sopra, alcuni metadati sono comunemente archiviati nei nodi che rappresentano la fine di una chiave in modo che in successive traversate quella chiave possa ancora essere recuperata. Per esempio, se aggiungessimo codes nel nostro esempio sopra, avremmo bisogno di un modo per sapere che la e in code rappresenta la fine di una chiave precedentemente inserita. Altrimenti, questa informazione sarebbe effettivamente persa quando aggiungiamo codes.

# --instructions--

Creiamo un trie per archiviare parole. Accetterà parole tramite un metodo `add` e le archivierà in una struttura dati trie. Permetterà anche di verificare se una data stringa è una parola con un metodo `isWord` e di recuperare tutte le parole inserite nel trie con un metodo `print`. `isWord` dovrebbe restituire un valore booleano e `print` dovrebbe restituire un array di tutte queste parole come valori stringa. Per permetterci di verificare che questa struttura dati sia implementata correttamente, abbiamo fornito una struttura `Node` per ogni nodo nell’albero. Ogni nodo sarà un oggetto con una proprietà `keys` che è un oggetto Map di JavaScript. Questo conterrà le singole lettere che sono chiavi valide di ogni nodo. Abbiamo anche creato una proprietà `end` sui nodi che può essere impostata su `true` se il nodo rappresenta la terminazione di una parola.
