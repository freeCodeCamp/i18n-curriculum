# --description--

Робота над цими завданнями передбачає написання коду одним із таких способів:

- Клонуйте <a href="https://github.com/freeCodeCamp/boilerplate-express/" target="_blank" rel="noopener noreferrer nofollow">цей репозиторій GitHub</a> і виконайте завдання локально.
- Використайте будь-який конструктор сайтів на свій вибір для виконання проєкту. Обов’язково включіть усі файли з нашого репозиторію GitHub.

Під час розробки важливо мати можливість перевіряти, що відбувається у вашому коді.

Node — це просто середовище JavaScript. Як і JavaScript на стороні клієнта, ви можете використовувати консоль для відображення корисної інформації для налагодження. На вашому локальному комп’ютері ви бачитимете вихідні дані консолі у терміналі.

Рекомендуємо тримати термінал відкритим під час роботи над цими завданнями. Читаючи вихідні дані в терміналі, ви можете бачити будь-які помилки, які можуть виникнути.

Після внесення змін у файли сервера його потрібно перезапустити.

Ви можете зупинити сервер із терміналу за допомогою `Ctrl + C` і запустити його безпосередньо через Node (`node mainEntryFile.js`) або використовуючи скрипт запуску у файлі `package.json` з `npm run`.

Наприклад, скрипт `"start": "node server.js"` запускається з терміналу за допомогою `npm run start`.

Щоб реалізувати автоматичний перезапуск сервера при збереженні файлу, Node надає прапорець `--watch`, який можна додати до скрипта запуску `"start": "node --watch server.js"`, або ви можете встановити пакет npm, наприклад `nodemon`. Ми залишаємо це вам як вправу.
