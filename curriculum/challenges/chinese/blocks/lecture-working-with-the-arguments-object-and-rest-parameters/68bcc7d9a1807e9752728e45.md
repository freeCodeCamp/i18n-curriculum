# --interactive--

在上一课中，你学习了如何使用 `arguments` 对象，它是一个类数组对象，包含传递给函数的参数值。

:::interactive_editor

```js
function logArgs() {
  for (const arg of arguments) {
    console.log(arg);
  }
}

logArgs(1, 2, 3);
// result:
// 1
// 2
// 3
```

:::

虽然这是访问和处理函数中可变参数的有效方法，但现代 JavaScript 应用将改用剩余参数语法。

这是一个使用剩余参数语法替代 `arguments` 对象的更新示例：

:::interactive_editor

```js
function logArgs(...args) {
  for (const arg of args) {
    console.log(arg);
  }
}

logArgs(1, 2, 3);
// result:
// 1
// 2
// 3
```

:::

此更新示例不再直接引用 `arguments` 对象。相反，函数定义的最后一个参数前有三个点。这会导致该剩余参数被放置在一个 `Array` 对象中。你可以随意命名这个剩余参数。只需确保它是函数定义中的最后一个参数，如下所示：

```js
function exampleFunction(a, b, ...restOfArgs) {
  // some code here
}

function anotherFunction(x, y, ...theArgs) {
  // some code here
}
```

在处理剩余参数语法时还有一些限制。一个限制是函数定义中只能有一个剩余参数。所以下面的示例将被视为无效。

```js
// Won't work

function badFunction(...args, ...moreArgs) {
  // some code here
}
```

下一个限制是剩余参数后不允许有尾随逗号：`function exampleFunction(a, b, ...restOfArgs, )`。

另一个限制是剩余参数不能有默认值。否则会抛出 `SyntaxError`。

```js
function badFunction(...args = [1,2]){
  // some code here
}
```

那么 `arguments` 对象和 `rest` 参数之间有哪些区别呢？

一个关键区别是 `arguments` 对象不是真正的数组，因此它不支持像 `includes`、`pop` 和 `push` 这样的方法。但剩余参数是一个 `Array` 实例。所以你可以直接使用有效的内置数组方法，而无需先将其转换为真正的数组。

:::interactive_editor

```js
function hasCat(...args) {
  return args.includes("cat");
}

console.log(hasCat("dog", "chicken", "cat")); // true
console.log(hasCat("dog", "chicken", "horse")); // false
```

:::
