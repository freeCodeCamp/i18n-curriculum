# --interactive--

تسمح تحقق النظر للأمام والنظر للخلف بمطابقة أنماط محددة بناءً على وجود أو غياب أنماط محيطة. هناك أربعة أنواع من هذه التحققات.

أولها تحقق النظر للأمام الإيجابي. هذا التحقق يطابق نمطًا عندما يتبعه نمط آخر.

لبناء تحقق نظر للأمام إيجابي، ابدأ بالنمط الذي تريد مطابقته. ثم استخدم الأقواس العادية لتغليف النمط الذي تريد استخدامه كشرط. بعد القوس الافتتاحي، استخدم `?=` لتعريف هذا النمط كتحقق نظر للأمام إيجابي.

هذا النمط سيطابق كلمة `free` فقط إذا تبعها `code`:

```js
const regex = /free(?=code)/i;
```

دعنا نختبر سلوك نمطنا:

:::interactive_editor

```js
const regex = /free(?=code)/i;
console.log(regex.test("freeCodeCamp")); // true
console.log(regex.test("free code camp")); // false
console.log(
  regex.test("I need someone for free to write code for me")
); // false
```

:::

لاحظ كيف أن السلسلة النصية التي يتبع فيها `free` مباشرةً `code` فقط هي التي تجتاز الاختبار.

لكن ماذا لو أردت مطابقة وجود `free` عندما لا يتبعها `code`؟ يمكنك تحويل تحقق النظر للأمام الإيجابي إلى تحقق نظر للأمام سلبي لعكس السلوك. للقيام بذلك، غيّر `?=` إلى `?!`:

```js
const regex = /free(?!code)/i;
```

دعنا نختبر هذا على نفس السلاسل النصية:

:::interactive_editor

```js
const regex = /free(?!code)/i;
console.log(regex.test("freeCodeCamp")); // false
console.log(regex.test("free code camp")); // true
console.log(
  regex.test("I need someone for free to write code for me")
); // true
```

:::

كما هو متوقع، النتائج معكوسة. السلسلة الوحيدة التي تفشل هي الأولى، حيث يتبع `free` مباشرةً `code`.

تعمل تحقق النظر للخلف بطريقة مشابهة لتحقق النظر للأمام، إلا أنه بدلاً من المطابقة بناءً على نمط يتبع، تطابق بناءً على نمط يسبق. لنلق نظرة على تحقق نظر للخلف إيجابي.

يُرمز إلى تحقق النظر للخلف الإيجابي بـ `?<=` بدلاً من `?=`. لنصنع تعبيرًا نمطيًا يطابق `code` عندما يسبقه `free`:

```js
const regex = /(?<=free)code/i;
```

تمامًا كما في تحقق النظر للأمام الإيجابي، يطابق تحقق النظر للخلف الإيجابي السلسلة الأولى لأن `code` يسبقه مباشرةً `free`:

:::interactive_editor

```js
const regex = /(?<=free)code/i;
console.log(regex.test("freeCodeCamp")); // true
console.log(regex.test("free code camp")); // false
console.log(
  regex.test("I need someone for free to write code for me")
); // false
```

:::

لمطابقة `code` عندما لا يسبقه `free`، يمكننا استخدام تحقق نظر للخلف سلبي. يُعرف التحقق السلبي باستبدال `?<=` بـ `?<!`:

```js
const regex = /(?<!free)code/i;
```

هذا سيطابق أي ظهور لـ `code` لا يسبقه مباشرةً `free`:

:::interactive_editor

```js
const regex = /(?<!free)code/i;
console.log(regex.test("freeCodeCamp")); // false
console.log(regex.test("free code camp")); // true
console.log(
  regex.test("I need someone for free to write code for me")
); // true
```

:::

تذكر أن `RegExp.prototype.test` يؤكد فقط ما إذا كانت السلسلة تطابق التعبير النمطي. لنستخدم تحقق النظر للخلف السلبي مع `String.prototype.match` لنرى كيف تؤثر التحققات على ذلك:

```js
const regex = /(?<!free)code/i;
console.log("freeCodeCamp".match(regex)); // null
console.log("free code camp".match(regex)); // ['code', index: 5, input: 'free code camp', groups: undefined]
console.log(
  "I need someone for free to write code for me".match(regex)
); // ['code', index: 33, input: 'I need someone for free to write code for me', groups: undefined]
```

لاحظ كيف أن التعبير النمطي يستخدم تحقق نظر للخلف للتحقق من وجود `free`، لكنه لا يطابق `free`. النص الوحيد المشمول في المطابقة هو `code`.

تُعد تحقق النظر للأمام والنظر للخلف أدوات مفيدة جدًا لمطابقة النصوص شرطياً دون التأثير على القيمة المرجعة للمطابقة.
