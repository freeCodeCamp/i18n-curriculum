# --description--

JavaScript에서 `Set`는 데이터 컬렉션을 관리하기 위한 내장 객체입니다. 원시값이든 객체 참조든 상관없이 고유한 값을 저장할 수 있습니다. `Set`는 그 안의 각 값이 한 번만 나타나도록 보장하여 배열에서 중복을 제거하거나 고유한 값들의 컬렉션을 다룰 때 유용합니다.

`WeakSet`는 기능이 적은 특별한 유형의 `Set`로, 약한 참조로 객체와 심볼을 저장할 수 있습니다. `Set`와 달리 `WeakSet`는 숫자나 문자열 같은 원시값을 지원하지 않습니다.

일반 `Set`와 달리 `WeakSet`는 객체만 저장하며, 그 객체에 대한 참조가 "약한" 참조입니다. 즉, WeakSet은 저장된 객체가 다른 참조가 없으면 가비지 컬렉션 대상이 되는 것을 막지 않습니다. 쉽게 말해, 코드 어디에서도 사용되지 않는 객체는 메모리 확보를 위해 자동으로 제거됩니다.

`Set`를 생성하려면 `Set` 생성자를 사용해 변수에 할당하세요:

```js
const myFirstSet = new Set();
```

값으로 `Set`를 초기화할 수도 있습니다:

```js
const treeSet = new Set(["Baobab", "Jackalberry", "Mopane Tree", "Breadfruit"]);
```

`Set`를 콘솔에 출력하면 다음과 같은 결과가 나타납니다:

```js
/*
Set(4) {"Baobab", "Jackalberry", "Mopane Tree", "Breadfruit"}
  [[Entries]]
  0: "Baobab"
    value: "Baobab"
  1: "Jackalberry"
    value: "Jackalberry"
  2: "Mopane Tree"
    value: "Mopane Tree"
  3: "Breadfruit"
    value: "Breadfruit"
  size: 4
  [[Prototype]]: Set
*/
```

값으로 `Set`를 초기화하지 않았다면 `add()` 메서드를 사용해 `Set`에 항목을 추가할 수 있습니다:

```js
const treeSet = new Set();

// Add items to the treeSet
treeSet.add("Baobab");
treeSet.add("Jackalberry");
treeSet.add("Mopane Tree");
treeSet.add("Breadfruit");
```

결과와 콘솔에 나타나는 모습은 동일합니다.

중복된 항목은 `Set`에서 무시된다는 점을 잊지 마세요:

```js
const treeSet = new Set();

// Add items to the treeSet
treeSet.add("Baobab");
treeSet.add("Jackalberry");
treeSet.add("Mopane Tree");
treeSet.add("Breadfruit");
treeSet.add("Baobab"); //duplicate item will be ignored

console.log(treeSet);
// Set(4) {"Baobab", "Jackalberry", "Mopane Tree", "Breadfruit"}
```

`Set`를 조작할 때 사용할 수 있는 다른 메서드는 다음과 같습니다:

- `delete()`  
- `clear()`
- `has()`
- `entries()`
- `forEach()`
- `keys()`
- `values()`

이 메서드들이 어떻게 작동하는지 하나씩 살펴보겠습니다. `delete()`는 `Set`에서 지정한 항목을 제거합니다:

```js
const treeSet = new Set();

// Add items to the treeSet
treeSet.add("Baobab");
treeSet.add("Jackalberry");
treeSet.add("Mopane Tree");
treeSet.add("Breadfruit");

treeSet.delete("Breadfruit");

console.log(treeSet); // Set(3) {"Baobab", "Jackalberry", "Mopane Tree"}
```

`has()`는 `Set`에 지정한 값이 존재하는지 확인합니다:

```js
const treeSet = new Set();

// Add items to the treeSet
treeSet.add("Baobab");
treeSet.add("Jackalberry");
treeSet.add("Mopane Tree");
treeSet.add("Breadfruit");

treeSet.delete("Breadfruit");

console.log(treeSet.has("Breadfruit")); // false
```

`entries()`는 `Set` 반복자를 반환하며, `[value, value]` 형식으로 `[value, value]` 안의 값을 배열로 포함합니다:

```js
const treeSet = new Set();

// Add items to the treeSet
treeSet.add("Baobab");
treeSet.add("Jackalberry");
treeSet.add("Mopane Tree");
treeSet.add("Breadfruit");

treeSet.delete("Breadfruit");

console.log(treeSet.entries());
// SetIterator {"Baobab" => "Baobab", "Jackalberry" => "Jackalberry", "Mopane Tree" => "Mopane Tree"}
```

`keys()`와 `values()`는 `Set` 안의 값을 보여줍니다. `keys()`는 `values()` 메서드의 별칭일 뿐입니다:

```js
const treeSet = new Set();

// Add items to the treeSet
treeSet.add("Baobab");
treeSet.add("Jackalberry");
treeSet.add("Mopane Tree");
treeSet.add("Breadfruit");

treeSet.delete("Breadfruit");

console.log("Keys: ", treeSet.keys());
console.log("Values: ", treeSet.values());
// Keys: SetIterator {"Baobab", "Jackalberry", "Mopane Tree"}
// Values: SetIterator {"Baobab", "Jackalberry", "Mopane Tree"}
```

`forEach()`는 `Set`를 순차적으로 반복할 수 있게 해줍니다:

```js
const treeSet = new Set();

// Add items to the treeSet
treeSet.add("Baobab");
treeSet.add("Jackalberry");
treeSet.add("Mopane Tree");
treeSet.add("Breadfruit");

treeSet.delete("Breadfruit");

treeSet.forEach((tree) => console.log(tree));
/*
Baobab
Jackalberry
Mopane Tree
*/
```

`clear()`는 배열의 모든 항목을 제거합니다:

```js
const treeSet = new Set();

// Add items to the treeSet
treeSet.add("Baobab");
treeSet.add("Jackalberry");
treeSet.add("Mopane Tree");
treeSet.add("Breadfruit");

treeSet.delete("Breadfruit");

treeSet.clear();

console.log(treeSet); // Set(0) {size: 0}
```

또한 `size` 속성이 있어 `Set` 안의 항목 수를 반환한다는 점도 언급할 만합니다:

```js
const treeSet = new Set();

// Add items to the treeSet
treeSet.add("Baobab");
treeSet.add("Jackalberry");
treeSet.add("Mopane Tree");
treeSet.add("Breadfruit");

treeSet.delete("Breadfruit");

console.log(treeSet.size); // 3
```

`Set`처럼 `WeakSet` 생성자를 사용해 `WeakSet`를 생성할 수도 있습니다:

```javascript
const treeWeakSet = new WeakSet();
```

`WeakSet`에는 `add()`, `delete()`, `has()` 메서드도 있습니다:

```javascript
const treeWeakSet = new WeakSet();

const baobab = { name: "Baobab" };
const jackalberry = { name: "Jackalberry" };
const mopaneTree = { name: "Mopane Tree" };
const breadfruit = { name: "Breadfruit" };

treeWeakSet.add(baobab);
treeWeakSet.add(jackalberry);
treeWeakSet.add(mopaneTree);
treeWeakSet.add(breadfruit);

treeWeakSet.delete(jackalberry);
console.log(treeWeakSet.has(jackalberry)); // false

console.log(treeWeakSet);
```

출력 결과에서 `WeakSet`의 내용은 다음과 같이 나타납니다:

```javascript
/*
WeakSet {{…}, {…}, {…}}
  [[Entries]]
    0: value: {name: "Mopane Tree"}
    1: value: {name: "Baobab"}
    2: value: {name: "Breadfruit"}
  [[Prototype]]: WeakSet
*/
```

객체 변수가 여전히 강한 참조를 유지해 메모리에 객체가 남아 있기 때문에 내용이 보입니다. WeakSet은 반복 가능하지 않아 항목을 순회하거나 프로그래밍적으로 접근할 수 없다는 점을 기억하세요. 출력에 보이는 것은 디버깅을 위한 보조 도구일 뿐입니다.

심볼과 키와 값이 명확히 정의된 객체만 지원된다는 점도 잊지 마세요. 숫자나 문자열 같은 원시값을 추가하면 오류가 발생합니다:

```js
treeWeakSet.add("Alan Smith");

console.log(treeWeakSet); // Invalid value used in weak set
//    at WeakSet.add (<anonymous>)
```

`Set`와 `WeakSet`의 주요 차이는 `Set`는 모든 값을 저장할 수 있지만 `WeakSet`는 객체만 저장할 수 있다는 점입니다.

다음은 `Set`와 `WeakSet` 사이의 다른 눈에 띄는 차이점입니다:

| 기능 | Set | WeakSet |
| --- | --- | --- |
| 저장하는 값의 유형 | 모든 데이터 유형 저장 | 객체만 저장 |
| 참조 방식 | 강한 참조 | 약한 참조 |
| 반복 지원 | `forEach`와 반복문 지원 | 반복 지원 안 함 |
| 메서드 및 속성 | `add()`, `delete()`, `has()`, `keys()`, `values()`, `size` 등 다양 | `add()`, `delete()`, `has()`만 지원 |
| 사용 사례 | 고유 값의 일반 컬렉션 및 배열 중복 제거 | 객체 참조의 효율적인 메모리 추적 |

두 종류의 Set이 저장할 수 있는 값 유형, 저장된 객체를 반복하는 지원 여부, 이상적인 사용 사례에서 차이를 확인할 수 있습니다. 이 표 내용을 잠시 읽어보세요.
