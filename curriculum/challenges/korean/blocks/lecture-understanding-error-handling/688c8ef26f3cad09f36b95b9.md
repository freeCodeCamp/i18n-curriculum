# --description--

디버깅은 모든 Python 개발자에게 필수적인 기술입니다. 기본적인 기법을 이해하면 문제를 효율적으로 찾아내고 고칠 수 있습니다.

디버깅은 코드에서 오류나 버그를 찾아내고 해결하는 과정입니다. 코드를 살펴보고 흐름을 이해하며 문제의 원인을 정확히 찾아내기 위해 도구를 사용하는 것을 포함합니다.

이번 수업에서는 다음 Python 프로젝트에서 사용할 수 있는 일반적인 디버깅 기법을 살펴보겠습니다.

## print 함수와 f-문자열 사용하기

먼저, 코드의 여러 지점에서 `print()` 함수와 f-문자열을 사용하면 흐름과 변수 상태를 이해하는 데 도움이 됩니다. 예를 들어:

```py
def add(a, b):
    result = a + b
    print(f'Adding {a} and {b} gives {result}')
    return result
```

`a`, `b`, `result`의 값을 출력함으로써 함수가 예상대로 동작하는지 확인할 수 있습니다.

## 내장 `pdb` 모듈을 이용한 대화형 디버깅

다음으로, Python의 내장 `pdb` 모듈을 사용해 대화형 디버깅을 할 수 있습니다:

```py
import pdb

def divide(a, b):
    pdb.set_trace()
    return a / b

print(divide(10, 2))
```

`set_trace()` 함수를 사용해 추적을 설정하면 코드를 한 단계씩 실행하며 변수들을 검사하고 프로그램의 동작을 이해할 수 있습니다.

위 코드를 실행하면, 실행 중인 파일 위치, `set_trace()` 함수를 호출한 줄과 그 직후 코드를 보여주는 출력과 함께 대화형 `pdb` 프롬프트가 나타납니다:

```py
> /Users/fcc/Desktop/debugging.py(5)divide()
-> return a / b
(Pdb)
```

프롬프트에 help를 입력하면 사용할 수 있는 명령어 목록이 표시됩니다:

```py
(Pdb) help

Documented commands (type help <topic>):
========================================
EOF    c          d        h         list      q        rv       undisplay
a      cl         debug    help      ll        quit     s        unt      
alias  clear      disable  ignore    longlist  r        source   until    
args   commands   display  interact  n         restart  step     up       
b      condition  down     j         next      return   tbreak   w        
break  cont       enable   jump      p         retval   u        whatis   
bt     continue   exit     l         pp        run      unalias  where    

Miscellaneous help topics:
==========================
exec  pdb
```

이후 명령어를 사용해 코드를 디버깅할 수 있습니다.

예를 들어, 그 순간 코드 내 요소들의 타입을 확인하고 싶다면 `whatis` 명령어를 사용할 수 있습니다:

```py
(Pdb) whatis a
<class 'int'>
(Pdb) whatis divide
Function divide
```

보시다시피, `.set_trace()`를 실행할 때 매개변수 a의 타입은 정수이고 divide는 함수입니다.

그다음 코드를 계속 실행하려면 `continue` 명령어 또는 그 별칭인 `cont`, `c` 중 하나를 사용하세요:

```py
(Pdb) continue
5.0
```

## IDE 디버깅 도구

많은 통합 개발 환경(IDE)은 중단점 설정, 단계별 실행, 변수 검사 같은 고급 디버깅 도구를 제공합니다.

## VS Code 디버거 사용하기

VS Code를 사용한다면 코드에 중단점을 설정하고 디버거를 실행해 해당 지점에서 실행을 일시 중지할 수 있습니다. 다음은 같은 `divide` 함수를 디버깅하는 방법입니다:

**1단계: 코드 준비하기** `main.py`라는 파일을 만들고 다음 내용을 입력하세요:

```py
def divide(a, b):
    result = a / b
    return result

print(divide(10, 2))
print(divide(15, 3))
```

**2단계: 중단점 설정하기**

1. 2번째 줄(`result = a / b`) 왼쪽 여백(거터)을 클릭해 중단점을 설정하세요

2. 빨간 점이 나타나 중단점이 설정되었음을 표시합니다

**3단계: 디버깅 시작하기**

1. `F5` 키를 누르거나 Run > Start Debugging 메뉴로 이동하세요

2. 프롬프트가 뜨면 "Python File"을 선택하세요

3. 디버거가 중단점에서 실행을 일시 중지합니다

**4단계: 변수 검사하기**

- 변수 위에 마우스를 올려 현재 값을 확인하세요

- 왼쪽의 Variables 패널에서 모든 지역 변수를 볼 수 있습니다

- 하단의 Debug Console에서 표현식을 평가할 수 있습니다

**5단계: 코드 단계별 실행하기**

디버그 툴바를 사용해 다음을 할 수 있습니다:

- **Continue (F5)**: 다음 중단점까지 실행을 계속합니다

- **Step Over (F10)**: 현재 줄을 실행하고 다음 줄로 이동합니다

- **Step Into (F11)**: 함수 호출 내부로 들어갑니다

- **Step Out (Shift+F11)**: 현재 함수에서 나옵니다

IDE 디버깅 도구는 프로그램 상태를 시각적으로 검사할 수 있는 인터페이스를 제공해 print 문만 사용하는 것보다 문제를 찾고 고치기 쉽습니다.

이러한 기본 디버깅 기법 — `print()` 문장, pdb 모듈, IDE 도구 사용 — 을 익히면 Python 코드의 문제를 효과적으로 찾아내고 해결할 수 있습니다. 각 기법은 용도가 다릅니다: 빠른 확인을 위한 `print()` 문장, 대화형 탐색을 위한 pdb, 시각적 검사를 위한 IDE 디버거입니다.
