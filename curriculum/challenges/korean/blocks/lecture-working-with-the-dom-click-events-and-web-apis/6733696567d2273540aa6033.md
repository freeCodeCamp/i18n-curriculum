# --interactive--

`Web Animations` API(WAAPI)는 JavaScript 내에서 직접 애니메이션을 만들고 제어할 수 있게 해줍니다. WAAPI를 사용하면 애니메이션을 더 동적으로 다룰 수 있어 조작이 훨씬 쉬워집니다.

WAAPI를 좀 더 자세히 살펴보면서 JavaScript 코드 안에서 직접 애니메이션을 다루는 방법을 배워보세요. WAAPI의 핵심은 `Animation` 생성자로, 여러 인스턴스 메서드와 속성을 제공해 요소를 동적으로 애니메이션할 수 있게 합니다.

`Animation` 생성자에서 중요한 메서드는 `animate()`입니다. 이 메서드는 키프레임과 지속 시간, 방향, 이징, 반복 횟수 같은 옵션을 지정해 애니메이션을 만들 수 있게 해줍니다.

`animate()` 메서드의 기본 구문은 다음과 같습니다:

```js
element.animate(keyframes, options);
```

예제를 살펴보겠습니다.

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css" />
<div class="square" id="square"></div>
<script src="index.js"></script>
```

```css
body {
  background: #f1f1f1;
  display: flex;
  align-items: center;
  justify-content: center;
  height: 100vh;
}

.square {
  background: #1b1b32;
  width: 10rem;
  aspect-ratio: 1/1;
}
```

```js
const square = document.querySelector("#square");

const animation = square.animate(
  [{ transform: "translateX(0px)" }, { transform: "translateX(100px)" }],
  {
    duration: 2000, // makes animation lasts 2 seconds
    iterations: Infinity, // loops indefinitely
    direction: "alternate", // moves back and forth
    easing: "ease-in-out" // smooth easing
  }
);
```

`Animation` 생성자의 인스턴스 메서드는 다음과 같습니다:

```js
play()
pause()
reverse()
finish()
cancel()
```

인스턴스 속성은 다음과 같습니다:

```md
playbackRate
currentTime
startTime
effect
timeline
playState
finished
onfinish
oncancel
```

예제를 `play()`, `pause()` 메서드와 `onfinish` 속성을 사용하도록 수정해보겠습니다. 새 버전은 다음과 같습니다:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css" />
<div class="square" id="square"></div>

<button id="playBtn">Play</button>
<button id="pauseBtn">Pause</button>
<script src="index.js"></script>
```

```css
body {
  background: #f1f1f1;
  display: flex;
  flex-direction: column;
  align-items: center;
  justify-content: center;
  height: 100vh;
}
.square {
  background: #1b1b32;
  width: 10rem;
  aspect-ratio: 1/1;
  margin-bottom: 20px;
}
button {
  margin: 10px;
  padding: 10px 20px;
  font-size: 16px;
}
```

```js
const square = document.querySelector("#square");
const playBtn = document.querySelector("#playBtn");
const pauseBtn = document.querySelector("#pauseBtn");

const animation = square.animate(
  [{ transform: "translateX(0px)" }, { transform: "translateX(200px)" }],
  {
    duration: 5000, // Animation lasts 5 seconds
    // iterations: Infinity, // Loops indefinitely
    direction: "alternate", // Moves back and forth
    easing: "ease-in-out" // Smooth easing function
  }
);

// Set the onfinish property to log a message when the animation ends
animation.onfinish = () => {
  console.log("Animation finished!");
};

// Play the animation when the "Play" button is clicked
playBtn.addEventListener("click", () => {
  animation.play();
  console.log("You start the animation");
});

// Pause the animation when the "Pause" button is clicked
pauseBtn.addEventListener("click", () => {
  animation.pause();
  console.log("You pause the animation");
});
```

:::

브라우저 결과는 `play` 버튼을 클릭하면 파란 상자가 왼쪽에서 오른쪽으로 움직이는 모습입니다.

WAAPI는 CSS 애니메이션의 기능을 확장해 JavaScript 내에서 애니메이션을 더 동적으로 제어할 수 있게 합니다.

CSS 애니메이션에서는 `animation-name`, `animation-duration`, `animation-timing-function` 같은 속성을 사용해 선언적으로 애니메이션을 정의합니다. WAAPI의 `animate()` 메서드를 사용해 같은 작업을 할 수도 있습니다.

차이점은 `animate()` 메서드로 만든 애니메이션을 더 직접적이고 동적으로 제어할 수 있다는 점입니다. 반면 CSS 애니메이션은 사용자 정의 스타일을 정의하고 JavaScript 파일 내에서 이를 트리거하는 작업이 훨씬 더 필요합니다.

CSS 애니메이션은 자동으로 실행되는 간단하고 선언적인 애니메이션에 적합합니다. 여기에는 호버 효과, 전환, 또는 한 번 트리거된 후 많은 상호작용이 필요 없는 애니메이션이 포함됩니다. 만약 애니메이션이 클릭, 스크롤 같은 사용자 상호작용에 반응해야 하거나, 사용자가 일시 정지, 역방향 재생, 속도 변경 같은 동적 제어를 할 수 있어야 한다면 WAAPI가 더 좋은 선택입니다.

기본 설정에는 CSS의 단순함을 사용하고 실행 시간에는 애니메이션을 제어해야 할 때 WAAPI와 CSS 애니메이션을 함께 사용하는 것도 고려할 수 있습니다.
