# --description--

每個專案都是獨特的，你應該仔細考慮這些技術中哪些對你的網頁應用程式最有幫助。

首先，由於所有樣式都會被解析，你應該從你的 CSS 樣式表中移除任何不必要的樣式。

你也應該將你的 CSS 樣式拆分成多個檔案。這樣，如果某個檔案不需要用來呈現特定的網頁，它就不會被載入，也不會阻塞呈現過程。

預設情況下，瀏覽器假設所有樣式表都應該阻擋呈現程序。但你可以使用 `media` 屬性和媒體查詢告訴瀏覽器何時應該載入樣式表。 

在此範例中，只有當文件正在列印時，才會套用 `print.css` 樣式表，因為它具有值為 `print` 的 `media` 屬性：

```html
<link rel="stylesheet" href="print.css" media="print" />
```

你也可以在建置處理程序中自動縮小你的 CSS 檔案以減少其大小，並在你託管網頁應用程式的伺服端壓縮這些檔案。

盡量保持你的 CSS 選擇器儘可能簡單，且不要嘗試選擇超過必要數量的元素。

盡可能利用 CSS 的層疊特性。某些樣式將會被繼承。

使用 `rel="preload"` 預先載入資源是另一個重要的技術。關鍵資產應該先載入。

請記住，CSS 動畫可能會影響效率、性能。

動畫化某些 CSS 屬性，例如尺寸、位置和佈局，會觸發一個稱為「reflow」的處理程序，在此期間瀏覽器會重新計算頁面上某些元素的位置和幾何形狀。這需要重新繪製，且計算成本很高。

因此，建議盡可能減少 CSS 動畫的數量，或至少為使用者提供一個切換開關來開啟或關閉它們的選項。

即使字型可以增強你的網頁應用程式的視覺呈現，但它們可能會影響效率。有些字型檔案很大，最高可達數百萬位元組。盡量最多使用兩到三種字型，並盡可能使用網頁安全字型。

你也可以考慮預先載入最重要的字型，以便快速使用。

這些是在網頁開發中最佳化 CSS 的一些最重要且常用的技術。透過將它們納入，你可以最佳化你網站的效率，創造流暢的使用者體驗。
