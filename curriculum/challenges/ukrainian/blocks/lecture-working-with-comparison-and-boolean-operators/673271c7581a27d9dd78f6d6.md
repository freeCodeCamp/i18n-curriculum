# --interactive--

У попередньому уроці ви вперше познайомилися з поняттям булевих значень, а в цьому уроці ми глибше розглянемо, як працюють булеві значення та як працюють оператори рівності й нерівності.

Булеві значення — це тип даних, який має лише `true` і `false` значення. Вони корисні, бо дозволяють виконувати дії залежно від певних умов. Булеві значення є необхідними, коли ви хочете оцінити, чи має щось відбутися, наприклад, вирішити, чи може хтось отримати доступ до певної функції у вашому застосунку. Ось приклад присвоєння значення `true` змінній з ім’ям `isOldEnoughToDrive`:

:::interactive_editor

```js
let isOldEnoughToDrive = true;

console.log(isOldEnoughToDrive); // true
```

:::

Ви можете використати цю змінну всередині умовної інструкції так:

:::interactive_editor

```js
let isOldEnoughToDrive = true;

if (isOldEnoughToDrive) {
 console.log("You're old enough to drive"); // You're old enough to drive
} else {
 console.log("Sorry, you are not old enough to drive");
}
```

:::

Умова допомагає вам приймати рішення у вашому коді на основі певної умови. У цьому прикладі використовується те, що називається `if/else` інструкцією.

Якщо `isOldEnoughToDrive` дорівнює `true`, тоді у консолі буде виведено речення `You're old enough to drive`. Інакше, якщо `isOldEnoughToDrive` дорівнює `false`, тоді у консолі буде виведено речення `Sorry, you are not old enough to drive`. Оскільки змінна `isOldEnoughToDrive` встановлена у `true`, у консолі буде виведено перше речення. Ви дізнаєтеся більше про `if/else` інструкції в одному з наступних уроків.

Щоб порівняти два значення, ви можете використати оператор рівності або оператор строгої рівності. Результатом порівняння буде булеве значення — або `true`, або `false`. Ось приклад використання оператора рівності для порівняння рядка і числа. Оператор рівності позначається подвійним знаком «дорівнює» (`==`).

:::interactive_editor

```js
console.log(5 == "5"); // true
```

:::

У цьому прикладі JavaScript перетворює рядок `"5"` у число `5`, а потім перевіряє, чи вони рівні. Оскільки обидва значення тепер однакові, результат — true. Оператор рівності використовує перетворення типів перед перевіркою рівності значень.

Це відрізняється від оператора строгої рівності, який не виконує перетворення типів. Оператор строгої рівності перевіряє, чи однакові типи і чи однакові значення. Ось приклад використання оператора строгої рівності для порівняння числа і рядка. Цей оператор позначається потрійним знаком «дорівнює» (`===`).

:::interactive_editor

```js
console.log(5 === '5'); // false
```

:::

Наступне порівняння поверне `false`, бо тип даних рядок не є тим самим, що тип даних число. Якщо потрібно перевірити, чи щось не дорівнює іншому значенню, можна використати оператор нерівності або оператор строгої нерівності. Ось приклад використання оператора нерівності (`!=`) для порівняння числа з рядком.

:::interactive_editor

```js
console.log(5 != "5"); // false
```

:::

У цьому прикладі результат буде `false`, бо оператор нерівності спочатку перетворює рядок у число, а потім порівнює значення. Оскільки значення будуть однакові, він поверне `false`. Якщо ви спробуєте використати оператор строгої нерівності, отримаєте інший результат. Оператор строгої нерівності позначається знаком оклику, за яким ідуть два знаки «дорівнює» (`!==`).

:::interactive_editor

```js
console.log(5 !== "5"); // true
```

:::

Результат буде `true`, бо оператор строгої нерівності не виконує перетворення типів. Оскільки число `5` не дорівнює рядку `"5"`, результат — `true`.

Вважається найкращою практикою використовувати оператори строгої нерівності та строгої рівності, коли це можливо, оскільки вони не виконують перетворення типів. У більшості професійних проєктів код зазвичай віддає перевагу саме цим двом операторам замість операторів нерівності та рівності.
