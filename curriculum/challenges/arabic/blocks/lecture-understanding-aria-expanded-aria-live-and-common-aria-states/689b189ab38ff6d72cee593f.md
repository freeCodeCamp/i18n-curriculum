# --interactive--

تعلمت سابقًا كيف يجعل الترميز الدلالي وخصائص ARIA المحتوى قابلًا للوصول. في التطبيقات الواقعية، نادرًا ما يكون المحتوى ثابتًا، وغالبًا ما تحدث تحديثات ديناميكية للصفحات عبر JavaScript. عند حدوث ذلك، من المهم التأكد من أن هذه التغييرات تنعكس أيضًا في HTML. هذا يسمح لقارئات الشاشة بنقل الحالة المحدثة بدقة للمستخدمين. بدون ذلك، قد لا يعرف الأشخاص الذين يعتمدون على أدوات المساعدة أن شيئًا ما قد تغير، أو قد يحصلون على معلومات قديمة أو مضللة.

لننظر إلى سيناريو شائع حيث يغير JavaScript حالة عنصر، وكيف يمكنك الحفاظ على إمكانية وصول HTML الخاص بك من خلال تحديث خصائص ARIA.

افترض أن لديك زرًا يفتح ويغلق قائمة منسدلة:

:::interactive_editor

```html
<button id="menuButton" aria-expanded="false" aria-controls="menuList">
  Menu
</button>

<ul id="menuList" hidden>
  <li>List item 1</li>
  <li>List item 2</li>
</ul>
<script src="index.js"></script>
```

```js
const button = document.getElementById("menuButton");
const menu = document.getElementById("menuList");

button.addEventListener("click", () => {
  const expanded = button.getAttribute("aria-expanded") === "true";
  button.setAttribute("aria-expanded", String(!expanded));
  menu.hidden = expanded;
});
```

:::

يقوم HTML بإعداد زر يتحكم في قائمة منسدلة. يستخدم الزر `aria-expanded` للإشارة إلى ما إذا كانت القائمة مفتوحة، و`aria-controls` لربطه بالقائمة من خلال الإشارة إلى `id` الخاص بالقائمة.

يستمع JavaScript للنقرات على زر القائمة. عند النقر، يبدل قيمة `aria-expanded` ويحدث خاصية `hidden` على القائمة. هذا يحافظ على تزامن مستوى ظهور القائمة ومعلومات إمكانية الوصول، بحيث يعرف كل من المستخدمين المبصرين ومستخدمي قارئ الشاشة ما إذا كانت القائمة مفتوحة أو مغلقة.

سوف تتعلم المزيد عن تلك الخصائص ARIA في الدروس القادمة. الآن، تذكر فقط أنه عندما يغير JavaScript حالة عنصر، يجب عليك أيضًا تحديث خصائص ARIA ذات الصلة حتى تعكس تقنيات المساعدة بدقة ما هو معروض على الشاشة.
