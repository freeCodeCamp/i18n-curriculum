# --description--

У перших двох лініях файлу `myApp.js` ви можете побачити, як легко створити об’єкт застосунку Express. Цей об’єкт має кілька методів, і ви дізнаєтеся про багато з них у цих завданнях. Одним із основних методів є `app.listen(port)`. Він повідомляє вашому серверу слухати на вказаному порті, переводячи його в робочий стан. Для тестування нам потрібно, щоб застосунок працював у фоновому режимі, тому ми додали цей метод у файл `server.js` для вас.

Давайте обслуговуватимемо наш перший рядок! У Express маршрути мають таку структуру: `app.METHOD(PATH, HANDLER)`. METHOD — це http-метод у нижньому регістрі. PATH — це відносний шлях на сервері (це може бути рядок або навіть регулярний вираз). HANDLER — це функція, яку Express викликає, коли маршрут співпадає. Обробники мають форму `function(req, res) {...}`, де req — це об’єкт запиту, а res — об’єкт відповіді. Наприклад, обробник

```js
function(req, res) {
  res.send('Response String');
}
```

обслуговуватиме рядок 'Response String'.

# --instructions--

Використайте метод `app.get()`, щоб обслуговувати рядок "Hello Express" для GET-запитів, що відповідають шляху `/` (кореневий шлях). Переконайтеся, що ваш код працює, переглянувши логи.

**Примітка:** Увесь код для цих уроків слід додавати між кількома рядками коду, з якими ми вас почали.
