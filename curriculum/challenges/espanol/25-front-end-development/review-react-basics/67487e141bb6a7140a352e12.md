## Bibliotecas y frameworks de JavaScript

- Las bibliotecas y frameworks de JavaScript ofrecen soluciones rápidas a problemas comunes y aceleran el desarrollo al proporcionar código predefinido.
- Las bibliotecas suelen centrarse más en proporcionar soluciones para tareas específicas, como manipular el DOM, manejar eventos o gestionar solicitudes AJAX.
- Algunos ejemplos de bibliotecas de JavaScript son jQuery y React.
- Los frameworks, por otro lado, proporcionan una estructura más definida para construir aplicaciones. A menudo vienen con un conjunto de reglas y convenciones que los desarrolladores deben seguir.
- Ejemplos de frameworks incluyen Angular y Next.js, un meta-framework para React.
- **Aplicaciones de una sola página** (SPAs) son aplicaciones web que cargan una sola página de HTML y actualizan dinámicamente esa página a medida que el usuario interactúa con la aplicación sin recargar toda la página.
- Los SPAs utilizan JavaScript para gestionar el estado de la aplicación y renderizar contenido. Esto se hace a menudo usando frameworks que proporcionan excelentes herramientas para construir interfaces de usuario complejas.
- Algunos problemas relacionados con los SPAs incluyen:

  - Los lectores de pantalla tienen dificultades con el contenido actualizado dinámicamente.
  - La URL no cambia cuando el usuario navega dentro de la aplicación, lo que puede dificultar el uso de marcadores, retroceder o compartir páginas específicas.
  - El tiempo de carga inicial puede ser lento si la aplicación es grande, ya que todos los recursos necesitan ser cargados por adelantado.

## React

- React es una biblioteca popular de JavaScript para construir interfaces de usuario y aplicaciones web.
- Un concepto central de React es la creación de componentes de interfaz de usuario reutilizables que pueden actualizarse y renderizarse de manera independiente cuando los datos cambian.
- React permite a los desarrolladores describir cómo debe verse la interfaz de usuario según el estado de la aplicación. React luego actualiza y renderiza los componentes correctos cuando los datos o el estado cambian.

## Componentes de React

- Los componentes son los bloques de construcción de las aplicaciones React que permiten a los desarrolladores descomponer interfaces de usuario complejas en piezas más pequeñas y manejables.
- La interfaz de usuario se describe usando JSX, una extensión de la sintaxis de JavaScript que permite a los desarrolladores escribir código similar a HTML dentro de JavaScript.
- Los componentes son básicamente funciones o clases de JS que devuelven una pieza de UI.

Aquí tienes un ejemplo de un componente simple de React que renderiza un mensaje de saludo:

```jsx
function Greeting() {
  const name = 'Anna';
  return <h1>Welcome, {name}!</h1>;
}
```

Para usar el componente, simplemente puedes llamar:

```jsx
  <Greeting />
```

## Importación y exportación de componentes de React

- Los componentes de React pueden exportarse desde un archivo e importarse en otro archivo.
- Supongamos que tienes un componente llamado `City` en un archivo llamado `City.js`. Puedes exportar el componente usando la palabra clave `export`:

```jsx
// City.js
function City() {
  return <p>New York</p>;
}

export default City;
```

- Para importar el componente `City` en otro archivo, puedes usar la palabra clave `import`:

```jsx
// App.js
import City from './City';

function App() {
  return (
    <div>
      <h1>My favorite city is:</h1>
      <City />
    </div>
  );
}
```

- La palabra clave `default` se utiliza ya que es la exportación predeterminada del archivo `City.js`.

- También puedes elegir exportar el componente en la misma línea que la definición del componente de esta manera:

```jsx
export default function City() {
  return <p>New York</p>;
}
```

## Configurar un proyecto de React usando Vite

- Las herramientas de configuración de proyectos y CLIs proporcionan una forma rápida y sencilla de empezar nuevos proyectos, permitiendo a los desarrolladores centrarse en escribir código en lugar de lidiar con la configuración.
- Vite, una herramienta popular para la configuración de proyectos, puede usarse con React.
- Para crear un nuevo proyecto con Vite, puedes usar el siguiente comando en tu terminal:

```bash
npm create vite@latest my-react-app -- --template react
```

Este comando crea un nuevo proyecto de React llamado `my-react-app` usando la plantilla de React de Vite. En el directorio del proyecto, verás un archivo `package.json` con las dependencias y comandos del proyecto listados en él.

- Para ejecutar el proyecto, navega al directorio del proyecto y ejecuta los siguientes comandos:

```bash
cd my-react-app # path to the project directory
npm install # installs the dependencies listed in the package.json file
```

- Una vez instaladas las dependencias, deberías notar una nueva carpeta en tu proyecto llamada `node_modules`.

- La carpeta `node_modules` es donde se almacenan todos los paquetes y bibliotecas requeridos por tu proyecto.

- Para ejecutar tu proyecto, usa el siguiente comando:

```bash
npm run dev
```

- Después de eso, abre tu navegador y navega a `http://localhost:5173` para ver tu aplicación de React en funcionamiento.

- Para ver realmente el código del modelo de inicio, puedes entrar en tu proyecto dentro de la carpeta `src` y deberías ver el archivo `App.jsx`.

## Pasando props en componentes de React

- En React, los props (abreviatura de propiedades) son una forma de pasar datos de un componente padre a un componente hijo. Este mecanismo es necesario para crear elementos de interfaz de usuario reutilizables y dinámicos.
- Los props pueden ser cualquier valor de JavaScript. Para pasar props de un componente padre a un componente hijo, agrega los props como atributos cuando uses el componente hijo en el JSX del padre. Aquí hay un ejemplo sencillo:

```jsx
// Parent component
function Parent() {
  const name = 'Anna';
  return <Child name={name} />;
}

// Child component
function Child(props) {
  return <h1>Hello, {props.name}!</h1>;
}
```

Puedes pasar múltiples props utilizando el operador de propagación `(...)`, después de convertirlos en un objeto. Aquí tienes un ejemplo:

```jsx
// Parent component
function Parent() {
  const person = {
    name: 'Anna',
    age: 25,
    city: 'New York'
  };
  return <Child {...person} />;
}
```

En este código, el operador de propagación `{...person}` convierte el objeto persona en props individuales que son pasados al componente Hijo.

## Renderizado condicional en React

- El renderizado condicional en React te permite crear interfaces de usuario dinámicas. Se utiliza para mostrar diferentes contenidos según ciertas condiciones o estados dentro de tu aplicación.
- Hay varias maneras de renderizar condicionalmente contenido en React. Un enfoque común es usar el operador ternario. Aquí tienes un ejemplo:

```jsx
function Greeting({ isLoggedIn }) {
  return (
    <div>
      {isLoggedIn ? <h1>Welcome back!</h1> : <h1>Please log in</h1>}
    </div>
  );
}
```

- Otra forma de renderizar condicionalmente contenido es usar el operador lógico Y (`&&`). Esto es útil cuando quieres renderizar contenido solo si se cumple una cierta condición. Aquí tienes un ejemplo:

```jsx
function Greeting({ user }) {
  return (
    <div>
      {user && <h1>Welcome, {user.name}!</h1>}
    </div>
  );
}
```

En el código anterior, el elemento `h1` solo se renderiza si el objeto usuario es verdadero.

También puedes usar una declaración `if` directa de esta manera:

```jsx
function Greeting({ isLoggedIn }) {
  if (isLoggedIn) {
    return <h1>Welcome back!</h1>;
  }
  return <h1>Please sign in</h1>;
}
```

## Renderizado de listas en React

- El renderizado de listas en React es una tarea común al construir interfaces de usuario.
- Las listas pueden renderizarse usando el método `map()` del array de JavaScript para iterar sobre un array de elementos y devolver un nuevo array de elementos JSX.
- Por ejemplo, si tienes un array de nombres que quieres renderizar como una lista, puedes hacerlo de la siguiente manera:

```jsx
function NameList({ names }) {
  return (
    <ul>
      {names.map((name, index) => (
        <li key={`${name}-${index}`}>{name}</li>
      ))}
    </ul>
  );
}
```

- Recuerda siempre proporcionar una clave única para cada elemento de la lista para ayudar a React a gestionar las funciones de actualización y renderización. Con estas técnicas, puedes crear listas flexibles, eficientes y dinámicas en tus aplicaciones React.

## Estilos en línea en React

- Los estilos en línea en React te permiten aplicar estilos CSS directamente a los elementos JSX utilizando objetos de JavaScript.
- Para aplicar estilos en línea en React, puedes usar el atributo style en los elementos JSX. El atributo style toma un objeto donde las claves son propiedades CSS en camelCase y los valores son los correspondientes. Aquí tienes un ejemplo:

```jsx
function Greeting() {
  return (
    <h1
      style={{ color: 'blue', fontSize: '24px', backgroundColor: 'lightgray' }}
    >
      Hello, world!
    </h1>
  );
}

export default Greeting;
```

También puedes extraer los estilos en un objeto separado y referenciarlo en el atributo `style` de esta manera:

```jsx
function Greeting() {
  const styles = {
    color: 'blue',
    fontSize: '24px',
    backgroundColor: 'lightgray'
  };

  return <h1 style={styles}>Hello, world!</h1>;
}

export default Greeting;
```

- Los estilos en línea soportan estilo dinámico al permitirte aplicar estilos condicionalmente basados en props o estado. Aquí tienes un ejemplo de cómo puedes aplicar estilos condicionalmente basados en un prop:

```jsx
function Greeting({ isImportant }) {
  const styles = {
    color: isImportant ? 'red' : 'black',
    fontSize: isImportant ? '24px' : '16px'
  };

  return <h1 style={styles}>Hello, world!</h1>;
}

export default Greeting;
```

- En el código anterior, los estilos de `color` y `fontSize` se establecen condicionalmente basándose en el prop `isImportant`.
