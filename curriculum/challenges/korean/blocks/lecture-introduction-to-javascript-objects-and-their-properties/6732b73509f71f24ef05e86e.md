# --interactive--

JavaScript를 다룰 때, 중첩 객체와 객체 내 배열을 포함하는 복잡한 자료 구조를 자주 만나게 됩니다. 이러한 구조는 풍부하고 계층적인 데이터를 나타내지만, 그 안의 데이터를 접근하고 조작하는 방법을 명확히 이해해야 합니다. 중첩된 구조를 효과적으로 탐색하는 방법을 살펴보겠습니다.

중첩 객체의 속성에 접근하려면, 단순 객체의 속성에 접근할 때처럼 점 표기법이나 대괄호 표기법을 사용합니다. 다만, 중첩 구조를 깊이 파고들기 위해 이러한 접근자를 연쇄적으로 연결해야 합니다.

예를 들어, 연락처 정보를 가진 사람을 나타내는 중첩 객체를 생각해 봅시다:

```js
const person = {
  name: "Alice",
  age: 30,
  contact: {
    email: "alice@example.com",
    phone: {
      home: "123-456-7890",
      work: "098-765-4321"
    }
  }
};
```

`Alice`의 직장 전화번호에 접근하려면 다음과 같이 속성 접근자를 연결합니다:

:::interactive_editor

```js
const person = {
  name: "Alice",
  age: 30,
  contact: {
    email: "alice@example.com",
    phone: {
      home: "123-456-7890",
      work: "098-765-4321"
    }
  }
};

console.log(person.contact.phone.work); // "098-765-4321"
```

:::

속성 이름에 공백이나 특수 문자가 포함되어 있거나 변수를 사용해 속성에 접근할 때 특히 유용한 대괄호 표기법도 사용할 수 있습니다:

:::interactive_editor

```js
const person = {
  name: "Alice",
  age: 30,
  contact: {
    email: "alice@example.com",
    phone: {
      home: "123-456-7890",
      work: "098-765-4321"
    }
  }
};

console.log(person['contact']['phone']['work']); // "098-765-4321"
```

:::

이제 객체 속성 중 하나가 배열 값을 가진 데이터를 어떻게 접근하는지 살펴보겠습니다. 주소 배열을 포함한 수정된 `person` 객체가 있습니다:

```js
const person = {
  name: "Alice",
  age: 30,
  addresses: [
    { type: "home", street: "123 Main St", city: "Anytown" },
    { type: "work", street: "456 Market St", city: "Workville" }
  ]
};
```

`Alice`의 직장 주소 도시를 접근하는 예시는 다음과 같습니다:

:::interactive_editor

```js
const person = {
  name: "Alice",
  age: 30,
  addresses: [
    { type: "home", street: "123 Main St", city: "Anytown" },
    { type: "work", street: "456 Market St", city: "Workville" }
  ]
};

console.log(person.addresses[1].city); // "Workville"
```

:::

이 예시에서 `person.addresses`은 주소 배열을 나타냅니다. 배열에서 두 번째 주소에 접근하려면 대괄호 표기법과 인덱스 `1`를 사용합니다. 그런 다음 점 표기법으로 해당 주소 객체의 `city`에 접근합니다.

중첩 객체와 배열에서 속성에 접근하는 방법을 이해하는 것은 복잡한 자료 구조를 다룰 때 필수적입니다. 앞으로의 워크숍과 실습에서 이러한 유형의 자료 구조를 다루는 연습을 할 기회가 있을 것입니다.
