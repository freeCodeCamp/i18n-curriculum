# --description--

El CSS interno (o CSS embebido) consiste en agregar el CSS dentro del archivo HTML en lugar de crear un archivo completamente separado. Con el método interno, colocas todas las reglas dentro de un par de etiquetas `<style>` de apertura y cierre, que luego se colocan dentro de las etiquetas `<head>` de apertura y cierre de tu archivo HTML. Dado que los estilos se colocan directamente dentro de las etiquetas `<head>`, ya no necesitas un elemento `<link>` que requiere el método externo.

Además de estas diferencias, la sintaxis es exactamente la misma que la del método externo (selector, llaves, declaraciones):

```html
<head>
  <style>
    div {
      color: white;
      background-color: black;
    }

    p {
      color: red;
    }
  </style>
</head>
<body>...</body>
```

Este método puede ser útil para agregar estilos únicos a una sola página de un sitio web, pero no mantiene las cosas separadas como el método externo, y dependiendo de cuántas reglas y declaraciones haya, puede hacer que el archivo HTML sea bastante grande.

# --questions--    

## --text--

¿Cuál de las siguientes es una diferencia entre los métodos CSS internos y externos?

## --answers--

El método externo coloca las reglas CSS en un archivo separado, mientras que el método interno coloca las reglas CSS dentro del mismo archivo HTML.

---

El método interno mantiene el CSS separado del HTML, mientras que el método externo incrusta el CSS directamente en el HTML.

---

El método interno usa el elemento `<link>` para enlazar CSS con HTML, mientras que el método externo inserta CSS directamente en HTML.
