# --description--

Die `async` und `defer` Attribute in HTML `script` Elementen spielen eine entscheidende Rolle dabei, wie JavaScript-Dateien in Webseiten geladen und ausgeführt werden. Ihr Verständnis kann die Leistung Ihrer Webseite und die Benutzererfahrung verbessern. Wenn Sie ein `script` in Ihre HTML-Datei einfügen, sieht das so aus:

```js
<script src="example.js"></script>
```

Wenn der Browser dieses `script`-Tag findet, stoppt er das Parsen des HTML, lädt das Script herunter, führt es aus und setzt dann das Parsen des HTML fort. Das kann das Laden Ihrer Webseite verlangsamen, besonders wenn Sie große Scripts haben.

Hier kommen die `async` und `defer` Attribute ins Spiel. Sie bieten Möglichkeiten, Scripts effizienter zu laden.

Beginnen wir mit dem `async`-Attribut:

```js
<script src="example.js" async></script>
```

Indem Sie das `async`-Attribut zu einem `script`-Tag hinzufügen, wird der Browser das HTML weiterhin parsen, während das Script heruntergeladen wird. Sobald das Script vollständig heruntergeladen ist, pausiert der Browser das HTML-Parsen, führt das Script aus und setzt dann das Parsen des HTML fort. Das kann das Laden der Seite deutlich beschleunigen.

Es ist wichtig zu beachten, dass async-Scripts ausgeführt werden, sobald sie heruntergeladen sind, was bedeutet, dass sie möglicherweise nicht in der gewünschten Reihenfolge ausgeführt werden. Hier kommt das `defer`-Attribut zur Rettung. Schauen wir uns an, wie das `defer`-Attribut aussieht:

```js
<script src="example.js" defer></script>
```

Das `defer`-Attribut ähnelt dem `async`-Attribut. Allerdings werden defer-Scripts nicht sofort nach dem Herunterladen ausgeführt. Stattdessen warten sie, bis das Parsen des HTML abgeschlossen ist. Außerdem werden defer-Scripts in der Reihenfolge ausgeführt, in der sie im HTML-Code erscheinen.

Kurz gesagt: Verwenden Sie `async` für Scripts, bei denen die Ausführungsreihenfolge keine Rolle spielt, und `defer`, wenn Sie sicherstellen müssen, dass Scripts in der richtigen Reihenfolge ausgeführt werden. Beide Attribute können die Ladezeiten der Seite erheblich verbessern, indem sie dem Browser erlauben, das HTML weiter zu parsen, während Scripts im Hintergrund heruntergeladen werden.
