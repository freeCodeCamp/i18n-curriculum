# --description--

Für diese Aufgabe erstellen Sie ein Programm, das Schere, Stein, Papier spielt. Ein Programm, das zufällig auswählt, gewinnt normalerweise 50 % der Zeit. Um diese Aufgabe zu bestehen, muss Ihr Programm gegen vier verschiedene Bots antreten und in jedem Match mindestens 60 % der Spiele gewinnen.

Sie werden <a href="https://app.ona.com/?autostart=true#https://github.com/freeCodeCamp/boilerplate-rock-paper-scissors/" target="_blank" rel="noopener noreferrer nofollow">mit unserem Ona-Starter-Code an diesem Projekt arbeiten</a>.

# --instructions--

In der Datei `RPS.py` steht Ihnen eine Funktion namens `player` zur Verfügung. Die Funktion nimmt ein Argument entgegen, das ein String ist und den letzten Zug des Gegners beschreibt („R“, „P“ oder „S“). Die Funktion soll einen String zurückgeben, der den nächsten Zug darstellt, den sie spielen soll („R“, „P“ oder „S“).

Eine Spielerfunktion erhält für das erste Spiel eines Matches ein leeres String-Argument, da es keinen vorherigen Zug gibt.

Die Datei `RPS.py` zeigt eine Beispiel-Funktion, die Sie aktualisieren müssen. Die Beispiel-Funktion ist mit zwei Argumenten definiert (`player(prev_play, opponent_history = [])`). Die Funktion wird nie mit einem zweiten Argument aufgerufen, daher ist dieses komplett optional. Der Grund, warum die Beispiel-Funktion ein zweites Argument (`opponent_history = []`) enthält, ist, dass dies die einzige Möglichkeit ist, einen Zustand zwischen aufeinanderfolgenden Aufrufen der `player`-Funktion zu speichern. Sie benötigen das `opponent_history`-Argument nur, wenn Sie die Historie des Gegners verfolgen möchten.

*Hinweis: Um alle vier Gegner zu schlagen, muss Ihr Programm möglicherweise mehrere Strategien haben, die sich je nach den Zügen des Gegners ändern.*

## Entwicklung

Ändern Sie `RPS_game.py` nicht. Schreiben Sie Ihren gesamten Code in `RPS.py`. Für die Entwicklung können Sie `main.py` verwenden, um Ihren Code zu testen.

`main.py` importiert die Spiel-Funktion und die Bots aus `RPS_game.py`.

Um Ihren Code zu testen, spielen Sie ein Spiel mit der `play`-Funktion. Die `play`-Funktion nimmt vier Argumente entgegen:

- zwei Spieler, die gegeneinander spielen (die Spieler sind tatsächlich Funktionen)
- die Anzahl der Spiele, die im Match gespielt werden sollen
- ein optionales Argument, um ein Protokoll jedes Spiels zu sehen. Setzen Sie es auf `True`, um diese Nachrichten zu sehen.

```py
play(player1, player2, num_games[, verbose])
```

Zum Beispiel so würden Sie die Funktion aufrufen, wenn `player` und `quincy` 1000 Spiele gegeneinander spielen sollen und Sie die Ergebnisse jedes Spiels sehen möchten:

```py
play(player, quincy, 1000, verbose=True)
```
