# --description--

<dfn>Emit</dfn> は、あなたが使う最も一般的な通信方法です。サーバーから 'io' に何かを emit すると、イベント名とデータが接続されているすべてのソケットに送信されます。この概念の良い例は、新しいユーザーが接続するたびに接続中のユーザー数を emit することです！

まず、接続をリッスンしている場所の直前に、ユーザー数を追跡する変数を追加してください。

```js
let currentUsers = 0;
```

次に、誰かが接続したときに、emit する前にカウントをインクリメントする必要があります。したがって、接続リスナー内にインクリメント処理を追加してください。

```js
++currentUsers;
```

最後に、カウントをインクリメントした後、イベントを emit してください（これも接続リスナー内です）。イベント名は 'user count' とし、データは `currentUsers` のみとしてください。

```js
io.emit('user count', currentUsers);
```

次に、クライアントがこのイベントをリッスンする方法を実装しましょう！サーバーで接続をリッスンするのと同様に、`on` キーワードを使います。

```js
socket.on('user count', function(data) {
  console.log(data);
});
```

アプリを起動して認証すると、クライアントのコンソールに現在のユーザー数を表す '1' が表示されるはずです！さらにクライアントを起動して認証し、数字が増えるのを確認してください。

正しくできたと思ったらページを提出してください。エラーが出る場合は、<a href="https://forum.freecodecamp.org/t/advanced-node-and-express/567135#communicate-by-emitting-7" target="_blank" rel="noopener noreferrer nofollow">ここまで完成したプロジェクト</a>を確認できます。
