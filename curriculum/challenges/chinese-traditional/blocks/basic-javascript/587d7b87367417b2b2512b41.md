# --description--

關鍵字 `let` 並不是宣告變數的唯一新方法。在 ES6 中，你也可以使用關鍵字 `const` 來宣告變數。

`const` 擁有 `let` 的所有優秀特性，並且額外的好處是使用 `const` 宣告的變數是唯讀的。它們是常數值，這表示一旦變數被 `const` 指定，便無法重新指定：

```js
const FAV_PET = "Cats";
FAV_PET = "Dogs";
```

主控台會顯示錯誤，因為重新指定 `FAV_PET` 的值。

你應該總是使用 `const` 關鍵字來命名你不想重新指定的變數。這有助於當你不小心嘗試重新指定本應保持不變的變數時。

**注意：** 開發者通常會使用大寫變數識別字來表示不可變的值，並使用小寫或駝峰式命名來表示可變的值（物件和陣列）。你將在後續的挑戰中學習更多關於物件、陣列，以及不可變和可變值的內容。在後續的挑戰中，你也會看到大寫、小寫或駝峰式變數識別字的範例。

# --instructions--

將程式碼更改為所有變數皆使用 `let` 或 `const` 宣告。當你希望變數會改變時，使用 `let`；當你希望變數保持不變時，使用 `const`。此外，將使用 `const` 宣告的變數重新命名，以符合常見慣例。不要更改指定給變數的字串。

# --hints--

`var` 不應該存在於你的程式碼中。

```js
assert.notMatch(code, /var/g);
```

你應該將 `fCC` 改成全大寫。

```js
assert.match(code, /(FCC)/);
assert.notMatch(code, /(fCC)/);
```

`FCC` 應該是用 `const` 宣告的常數變數。

```js
assert.match(code, /const\s+FCC/);
```

指定給 `FCC` 的字串不應該被更改。

```js
assert.equal(FCC, 'freeCodeCamp');
```

`fact` 應該用 `let` 宣告。

```js
assert.match(code, /(let\s+fact)/g);
```

應將 `console.log` 更改為列印 `FCC` 和 `fact` 變數。

```js
assert.match(code, /console\.log\(\s*FCC\s*\,\s*fact\s*\)\s*;?/g);
```
