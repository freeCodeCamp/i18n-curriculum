# --description--

Verzweigende Programme, also solche, die unterschiedliche Dinge ausführen, wenn bestimmte Bedingungen erfüllt sind, basieren in JavaScript auf `if`, `else if` und `else`-Anweisungen. Die Bedingung nimmt manchmal die Form an, zu testen, ob ein Ergebnis gleich einem Wert ist.

Diese Logik wird (zumindest auf Englisch) so ausgedrückt: „if x equals y, then ...“, was im Code wörtlich mit dem `=` oder Zuweisungsoperator übersetzt werden kann. Das führt zu unerwartetem Programmablauf.

Wie in vorherigen Aufgaben behandelt, weist der Zuweisungsoperator (`=`) in JavaScript einem Variablennamen einen Wert zu. Und die `==` und `===` Operatoren prüfen auf Gleichheit (der dreifache `===` testet auf strikte Gleichheit, das heißt, sowohl Wert als auch Typ sind gleich).

Der untenstehende Code weist `x` den Wert 2 zu, was als `true` ausgewertet wird. Fast jeder Wert für sich allein in JavaScript wird als `true` ausgewertet, außer den sogenannten „falsy“-Werten: `false`, `0`, `""` (ein leerer String), `NaN`, `undefined` und `null`.

```js
let x = 1;
let y = 2;
if (x = y) {

} else {

}
```

In diesem Beispiel wird der Codeblock innerhalb der `if`-Anweisung für jeden Wert von `y` ausgeführt, es sei denn, `y` ist falsy. Der `else`-Block, von dem wir erwarten, dass er hier ausgeführt wird, wird tatsächlich nicht ausgeführt.

# --instructions--

Beheben Sie die Bedingung, damit das Programm den richtigen Zweig ausführt und der passende Wert `result` zugewiesen wird.

# --hints--

Ihr Code sollte die Bedingung so korrigieren, dass sie auf Gleichheit prüft, anstatt eine Zuweisung zu verwenden.

```js
assert(result == 'Not equal!');
```

Die Bedingung sollte entweder `==` oder `===` verwenden, um auf Gleichheit zu testen.

```js
assert(__helpers.removeJSComments(code).match(/x\s*?===?\s*?y/g));
```
