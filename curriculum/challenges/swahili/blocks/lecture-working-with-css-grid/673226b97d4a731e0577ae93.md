# --interactive--

Sifa ya `grid-template-areas` inakuwezesha kubuni mpangilio wa gridi wa kuona kwa kutumia lebo zilizopewa majina.

Kisha unaweka lebo hizo kwa vipande maalum vya gridi kwa kutumia sifa ya grid-area. Kwa maneno mengine, lebo hizo zilizopewa majina pia huitwa "majina ya eneo la gridi".

Hapa kuna sintaksia ya msingi ya sifa ya `grid-template-areas`:

```css
grid-template-areas:
 'header header header'
 'left-sidebar main right-sidebar'
 'footer footer footer';
```

Hapa kuna mambo ya kukumbuka kutoka kwa sintaksia ya msingi:

- Thamani kama `header` na `main` ni majina ya maeneo ya gridi.

- Kila thamani inayotenganishwa kwa nafasi ndani ya mfuatano wa herufi inahusiana na safu moja.

- Kila mfuatano wa herufi unawakilisha safu moja katika gridi.

Kwa hivyo, katika sintaksia, tuna kontena la gridi la 3 kwa 3.

Baada ya kufafanua kiolezo, unapaswa kutumia sifa ya `grid-area` na lebo zilizopewa majina (au maeneo) kama thamani ili kusema kwa CSS kuwa kipengele fulani kinahusiana na eneo hilo la gridi.

Sifa ya `grid-area` inaunganisha kipande cha gridi na eneo lililopewa jina unalofafanua katika `grid-template-areas`.

Njia maarufu ya kuonyesha uwezo wa sifa ya `grid-template-areas` ni kwa kuunda mpangilio wa Holy Grail wa kawaida.

Kama hujui ni nini, mpangilio wa Holy Grail ni muundo wa mtandao wenye kichwa, kijachini, pande mbili za pembeni, na eneo kuu la maudhui. Huhakikisha maudhui kuu yanapata kipaumbele, wakati pande za pembeni na sehemu nyingine zinajibadilisha kulingana na kifaa ndani ya mpangilio.

Kuna suluhisho nyingi za kutekeleza mpangilio wa holy grail, lakini kutumia `grid-template-areas` na sifa ya `grid-area` ni njia rahisi zaidi ya kuutengeneza.

Hapa kuna mfano wa mpangilio wa holy grail:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<div class="grid-container">
  <div class="header">
    <h2>Header</h2>
  </div>
  <div class="sidebar-left">
    <h2>Left Sidebar</h2>
  </div>
  <div class="main"><h2>Main Content</h2></div>
  <div class="sidebar-right">
    <h2>Right Sidebar</h2>
  </div>
  <div class="footer">
    <h2>Footer</h2>
  </div>
</div>
```

```css
.grid-container {
  display: grid;
  grid-template-areas:
    'header header header'
    'sidebar-left main sidebar-right'
    'footer footer footer';
  grid-gap: 10px;
  background-color: #2196F3;
  padding: 10px;
}

.header {
  grid-area: header;
  background-color: rgba(255, 255, 255, 0.8);
  padding: 20px;
  text-align: center;
}

.sidebar-left {
  grid-area: sidebar-left;
  background-color: rgba(255, 255, 255, 0.8);
  padding: 20px;
  text-align: center;
}

.main {
  grid-area: main;
  background-color: rgba(255, 255, 255, 0.8);
  padding: 20px;
  text-align: center;
}

.sidebar-right {
  grid-area: sidebar-right;
  background-color: rgba(255, 255, 255, 0.8);
  padding: 20px;
  text-align: center;
}

.footer {
  grid-area: footer;
  background-color: rgba(255, 255, 255, 0.8);
  padding: 20px;
  text-align: center;
}
```

:::

Tafadhali kumbuka kuwa sifa za `grid-template-areas` na `grid-area` zinaweza kutumika kwa uhuru bila kuhusiana moja kwa moja.

Sifa ya `grid-template-areas` hutumika mahsusi kufafanua mpangilio wa kuona kwa kuweka ramani maeneo ya gridi yaliyopewa majina ndani ya kontena la gridi.

Kwa upande mwingine, sifa ya `grid-area` hutumika kuweka nafasi vipande vya gridi binafsi, iwe kwa kubainisha nafasi zao za safu na safu wima au kwa kurejelea maeneo yaliyopewa majina yaliyofafanuliwa na sifa ya `grid-template-areas`.
