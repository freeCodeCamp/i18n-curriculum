# --description--

Масиви — це фундаментальна структура даних у комп’ютерних науках. Всі масиви зберігають впорядковані колекції даних, але залежно від їх типу вони можуть працювати по-різному «під капотом».

Їхня внутрішня поведінка може суттєво впливати на ефективність програми, тому давайте дізнаємося про динамічні та статичні масиви та їхні відмінності, щоб ви могли обрати найефективніший для своєї програми.

Почнемо зі статичних масивів.

**Статичні масиви** мають фіксований розмір. Вони зберігають елементи у суміжних ділянках пам’яті.

Розмір статичного масиву визначається під час його ініціалізації. Після того, як певний блок пам’яті виділено, він фіксований і не може бути змінений під час виконання програми. Це ключова характеристика статичних масивів.

Зберігання елементів у суміжних ділянках пам’яті робить процес отримання даних більш ефективним, оскільки програма може зберегти розташування першого елемента, а потім використовувати індекси для простих обчислень і знаходження інших елементів у пам’яті.

Завдяки цьому доступ до значень статичного масиву займає сталий час `O(1)`, що дуже ефективно.

Ви можете використовувати статичний масив, коли заздалегідь знаєте кількість елементів, які будуть збережені. Це також корисно, коли значення будуть доступні дуже часто, оскільки операція доступу дуже ефективна.

Однак ця структура даних не може розширюватися або звужуватися, тому якщо кількість елементів може змінюватися, слід використовувати динамічний масив.

Спроба збільшити розмір статичного масиву означала б створення нового масиву та копіювання всіх елементів зі старого в новий, що є неефективним. У такому випадку динамічний масив буде набагато кращим, бо він автоматично обробляє цей процес.

JavaScript не має традиційних низькорівневих статичних масивів, як C або Java, але ви можете симулювати масив фіксованого розміру, використовуючи вбудований об’єкт `Array`:

```js
let numbers = new Array(3); // fixed size of 3
numbers[0] = 10;
numbers[1] = 20;
numbers[2] = 30;
```

Хоча JavaScript дозволяє змінювати розмір, концептуально це представляє масив фіксованої довжини.

Масиви в JavaScript за замовчуванням є динамічними, тож розглянемо їх.

**Динамічні масиви** більш гнучкі, бо можуть автоматично розширюватися або звужуватися під час виконання програми.

Вони працюють через механізм автоматичного зміни розміру, який копіює елементи у новий масив, коли оригінальний масив заповнений. Цей процес виконується ефективно, оскільки розмір нового масиву обирається так, щоб зробити ці обчислювально дорогі операції менш частими.

Доступ до елементів динамічного масиву займає сталий час `O(1)`, тому ця операція дуже ефективна.

Вставка елемента посередині масиву займає лінійний час `O(n)`, бо елементи після нього потрібно перемістити.

Вставка елемента в кінець масиву займає сталий час `O(1)`, якщо в динамічному масиві ще є вільне місце, але якщо масив заповнений і потребує зміни розміру, ця операція має складність `O(n)`.

Ви повинні використовувати динамічні масиви, коли не знаєте заздалегідь кількість значень, які потрібно зберегти в масиві. Вони також корисні, коли ви часто вставляєте та видаляєте елементи.

Щоб створити масив у JavaScript, напишіть елементи всередині квадратних дужок, розділених комами:

```js
let numbers = [3, 4, 5, 6];
```

Це створює динамічний масив із чотирьох елементів.

Ви можете отримати доступ до елемента масиву, написавши ім’я змінної, а потім квадратні дужки з індексом.

Індекси починаються з 0 для першого елемента і збільшуються на 1 для кожного наступного.

```js
numbers[0]; // 3
numbers[1]; // 4
numbers[2]; // 5
numbers[3]; // 6
```

Щоб оновити значення, просто призначте його знову:

```js
numbers[2] = 16
```

Тепер масив стає `[3, 4, 16, 6]`.

Щоб додати елемент у кінець масиву, використовуйте метод `.push()`:

```js
numbers.push(7)
```

Тепер масив стає `[3, 4, 16, 6, 7]`:

Щоб вставити елементи на певний індекс, у JavaScript немає `.insert()`, як у Python, але ви можете використати метод `.splice()`. Синтаксис виглядає так:

```js
array.splice(index, deleteCount, newElement)
```

Ось приклад:

```js
numbers.splice(3, 0, 15);
```

Цей приклад означає:

- Почати з індексу 3
- Видалити 0 елементів
- Вставити 15

Якщо масив був `[3, 4, 16, 6, 7]`, тепер він стає `[3, 4, 16, 15, 6, 7]`.

Ви можете видалити елемент за певним індексом тим самим методом `.splice`:

```js
numbers.splice(2, 1);
```

Це означає почати з індексу 2 і видалити 1 елемент.

Щоб видалити останній елемент, використовуйте метод `.pop()`:

```js
numbers.pop();
```

Існує безліч інших вбудованих методів для роботи з масивами, які ви можете переглянути в документації з роботи з масивами в JavaScript.

Ось сила динамічних масивів.

Загалом, слід використовувати статичні масиви, коли ви знаєте кількість елементів заздалегідь і вам потрібно часто отримувати до них доступ, а динамічні масиви — коли кількість елементів невідома або змінюється з часом.

Завжди враховуйте компроміс між простотою статичних масивів і гнучкістю динамічних. Обидва корисні для певних випадків і сценаріїв. Вміння обрати найкращий варіант для конкретної задачі — це частина навичок розв’язання проблем, які ви поступово розвиватимете на практиці.
