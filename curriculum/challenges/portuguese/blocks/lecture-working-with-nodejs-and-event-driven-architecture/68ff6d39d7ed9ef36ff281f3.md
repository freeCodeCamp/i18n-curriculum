# --description--

Agora você está pronto para dar seus primeiros passos no mundo do Node.js instalando-o no seu computador. Mas primeiro, antes de instalar o Node.js, recomendamos instalar uma ferramenta chamada NVM (Node Version Manager). Como o nome indica, o NVM é uma ferramenta para gerenciar múltiplas versões do Node.js no mesmo computador.

Configurar esta ferramenta desde o início vai te ajudar no futuro porque você pode precisar alternar entre diferentes versões do Node.js para diferentes projetos. Com o NVM, você pode fazer isso muito facilmente.

Vamos ver como você pode instalar NVM no macOS/Linux e Windows e como você pode usá-lo para instalar Node.js.

## Como Instalar o NVM no macOS/Linux

No macOS e Linux, você pode instalar o NVM diretamente através de um script shell.

Mas para macOS, note que a documentação oficial do NVM menciona que “você precisa instalar manualmente as ferramentas de linha de comando do Xcode antes de executar o script de instalação, caso contrário, ele falhará.”

No macOS, você pode instalar as ferramentas de linha de comando do Xcode executando este comando no seu terminal:

```bash
xcode-select --install
```

Uma vez que a instalação esteja concluída, você pode verificar o número da versão e confirmar a instalação com este comando:

```bash
xcode-select --version
```

Então, é hora de instalar o NVM. Você pode baixar e executar o script de instalação do NVM executando um destes comandos:

```bash
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.3/install.sh | bash
```

```bash
wget -qO- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.3/install.sh | bash
```

Para obter a versão mais recente do NVM no momento em que você está lendo este artigo, consulte a documentação oficial do NVM: `https://github.com/nvm-sh/nvm?tab=readme-ov-file#install--update-script`. Lá, você encontrará esses comandos com o número da versão mais recente.

Pode ser necessário reiniciar seu terminal quando o processo for concluído para carregar o novo comando e a nova configuração.

Então, execute este comando para confirmar que o NVM foi instalado corretamente e para verificar sua versão atual:

```bash
nvm -v
```

Você deve ver sua versão do NVM na saída.

## Como instalar Node.js com NVM no macOS/Linux

Agora que você tem o NVM instalado no macOS/Linux, você pode instalar o Node.js.

Para instalar a versão mais recente de Long-Term Support (LTS) do Node.js, execute este comando:

```bash
nvm install --lts
```

Observe a flag `--lts`. Esta flag indica que você deseja instalar a versão atual de Long-Term Support (LTS) do Node.js. Esta versão geralmente é recomendada porque prioriza estabilidade, confiabilidade e segurança em vez de recursos novos ou experimentais. Ela é rigorosamente testada e pronta para produção. Também é garantido que tenha períodos de suporte mais longos com foco em correções de bugs e patches de segurança.

Alternativamente, você pode instalar uma versão específica do Node.js especificando o número da versão no comando:

```bash
nvm install <version>
```

Por exemplo, aqui está como você pode instalar o Node.js v20:

```bash
nvm install 20
```

Depois de terminar de instalar uma versão do Node.js, você pode mudar para essa versão com este comando:

```bash
nvm use <version>
```

Por exemplo, aqui está como mudar para a versão 20:

```bash
nvm use 20
```

Finalmente, você pode verificar a versão atual do Node.js com este comando:

```bash
node -v
```

## Como instalar o NVM no Windows

Agora vamos ver como você pode instalar e usar o NVM no Windows.

As versões oficiais do NVM suportam Windows apenas em casos específicos. Se você tiver o Windows Subsystem for Linux (WSL) configurado no seu dispositivo, pode seguir o guia para macOS/Linux acima para instalar e usar o NVM.

No entanto, para o próprio Windows, existem algumas alternativas. Uma delas é `nvm-windows`, uma ferramenta de gerenciamento de versões do Node.js para Windows. Este é um projeto completamente separado do projeto oficial do NVM. Mas o propósito dessas ferramentas é o mesmo — ajudar você a trabalhar com múltiplas versões do Node.js no mesmo dispositivo.

Antes de instalar o `nvm-windows`, você deve saber que o repositório oficial recomenda desinstalar qualquer instalação anterior do Node antes de instalar o NVM para Windows. Se esta for a sua primeira vez instalando o Node.js, você deve estar bem. Se você já tem outras versões do Node.js instaladas, é recomendado desinstalá-las primeiro.

Para instalar o `nvm-windows`:

- Vá para a seção Releases do repositório oficial: `https://github.com/coreybutler/nvm-windows/releases`.
- Baixe o instalador mais recente.
- Siga as instruções.

Após concluir a instalação, pode ser necessário reiniciar seu terminal se o comando `nvm` não estiver disponível imediatamente.

Você pode verificar se a instalação foi bem-sucedida e ver sua versão atual do `nvm-windows` executando este comando:

```powershell
nvm -v
```

## Como Instalar Node.js usando NVM para Windows

Agora que você tem o `nvm-windows` instalado no Windows, você está pronto para instalar o Node.js.

Para instalar a versão mais recente de Long-Term Support (LTS) do Node.js, execute este comando:

```powershell
nvm install lts
```

A versão Long-Term Support (LTS) geralmente é recomendada, mas você pode instalar uma versão diferente especificando seu número de versão no comando:

```powershell
nvm install <version>
```

Por exemplo:

```powershell
nvm install 20
```

Quando a instalação for concluída, você pode mudar para essa versão do Node.js com este comando:

```powershell
nvm use <version>
```

Por exemplo:

```powershell
nvm use 20
```

Você também pode verificar sua versão atual do Node.js com este comando:

```bash
node -v
```

## Comandos Básicos

Agora que você sabe como instalar o NVM e o Node.js, vamos conferir alguns comandos básicos que serão úteis ao trabalhar com o Node.js e o NVM.

Além disso, embora todos esses comandos funcionem para NVM instalado no macOS/Linux, incluindo WSL, eles podem funcionar de forma diferente ou podem não funcionar para `nvm-windows`. Deixaremos comandos alternativos para `nvm-windows` sempre que possível.

### Como Ajustar a Política de Execução do Powershell

Se você está usando Windows e acompanhando com Powershell e `nvm-windows`, pode ser necessário ajustar a política de execução do Powershell. A política de execução padrão do Powershell é `Restricted`, que é uma medida de segurança para impedir a execução de scripts.

Primeiro, execute o Powershell como administrador clicando com o botão direito no Powershell e depois clicando em “Executar como Administrador“.

Então, para ver a política de execução atual da sessão atual do Powershell, execute o seguinte comando:

```powershell
Get-ExecutionPolicy
```

Finalmente, para definir a política de execução do Powershell para `RemoteSigned`, que permite que scripts locais e `npm` sejam executados, use o seguinte comando:

```powershell
Set-ExecutionPolicy RemoteSigned
```

Depois de executar o comando acima, você pode fechar a janela do administrador do Powershell e abrir uma janela do Powershell sem privilégios de administrador.

### Listar Todas as Versões

Para listar todas as versões instaladas do Node.js e verificar qual está sendo usada atualmente, execute este comando:

```bash
nvm ls
```

### Definir Versão Padrão do Node.js (NVM apenas no macOS/Linux)

Este comando define a versão padrão do Node.js que será usada para novas sessões do terminal:

```bash
nvm alias default <version>
```

Por exemplo:

```bash
nvm alias default 22.20.0
```

Você também pode criar aliases personalizados para atribuir um nome mais específico a uma versão:

```bash
nvm alias <new_name> <version>
```

Por exemplo:

```bash
nvm alias new-api 18.20.8
```

Após configurar o alias, você pode usar esse novo nome para alternar para essa versão:

```bash
nvm use <new_name>
```

Por exemplo:

```bash
nvm use new-api
```

### Remover uma Versão do Node.js

Para remover uma versão do Node.js, execute este comando:

```bash
nvm uninstall <version>
```

Por exemplo:

```bash
nvm uninstall 20.19.5
```

Com o NVM para macOS/Linux, você pode remover uma versão do Node.js executando o comando `uninstall` com uma versão principal como `22` ou `20`. Por exemplo:

```bash
nvm uninstall 20
```

Se você tiver várias versões para essa versão principal instaladas como `20.14.0` e `20.12.2`, a versão mais recente será removida primeiro. Neste caso, executar `nvm uninstall 20` removerá `20.14.0` primeiro.

### Verifique Sua Versão do Node.js

Para verificar sua versão atual do Node.js, você pode usar o seguinte:

```bash
node -v
```

### Verifique Sua Versão do npm

`npm` é um gerenciador de pacotes para Node.js. Você pode usá-lo para instalar, publicar e gerenciar pacotes de software. Ele é instalado automaticamente quando você instala o Node.js.

Para verificar sua versão atual do `npm`, execute este comando:

```bash
npm -v
```

### Executar um arquivo JavaScript

Este comando executa o arquivo JavaScript que você especificar no runtime Node.js atualmente ativo:

```bash
node <file.js>
```

Por exemplo:

```bash
node app.js
```

### Criar um Projeto Node.js

Este comando cria um novo projeto Node.js criando um arquivo `package.json` que rastreia seus detalhes e dependências.

```bash
npm init
```

Quando você executa o comando `npm init`, ele irá guiá-lo por um assistente interativo de CLI para configurar o arquivo `package.json`. Ele fará perguntas sobre o nome do projeto, descrição, versão, etc. Se você quiser começar com um arquivo `package.json` padrão, pode usar o comando `npm init -y` ou `npm init --yes`.

### Instalar um Pacote

Para instalar um pacote específico, execute:

```bash
npm install <package>
```

Por exemplo:

```bash
npm install express
```

### Instalar Dependências

Para instalar todas as dependências listadas no arquivo `package.json` de um projeto, execute:

```bash
npm install
```
