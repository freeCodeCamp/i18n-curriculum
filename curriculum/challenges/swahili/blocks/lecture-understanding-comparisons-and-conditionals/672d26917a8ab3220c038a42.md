# --interactive--

Katika JavaScript, `null` na `undefined` ni aina mbili tofauti za data zinazowakilisha kutokuwepo kwa thamani, lakini hufanya kazi tofauti katika mlinganisho. Kuelewa jinsi aina hizi zinavyoshirikiana katika hali mbalimbali za mlinganisho ni muhimu kwa kuandika msimbo thabiti na usio na hitilafu.

Tuanze na aina ya `undefined`. Kigezo kinachoitwa `undefined` kinapotangazwa lakini hakijapewa thamani. Hii ni thamani ya msingi ya vigezo visivyotangazwa na vigezo vya kitendakazi ambavyo havikupatiwa hoja.

Aina ya `null`, kwa upande mwingine, ni thamani ya mgawo inayowakilisha kutokuwepo kwa thamani kwa makusudi. Mara nyingi hutumika kuonyesha kuwa kigezo hakina thamani kwa makusudi.

Unapolinganisha `null` na `undefined` kwa kutumia opereta ya usawa (`==`), JavaScript hufanya kulazimisha aina. Hii ina maana inajaribu kubadilisha vigezo kuwa aina moja kabla ya kufanya mlinganisho. Katika kesi hii, `null` na `undefined` zinachukuliwa kuwa sawa:

:::interactive_editor

```js
console.log(null == undefined); // true
```

:::

Hata hivyo, unapotumia opereta ya usawa mkali (`===`), ambayo hukagua thamani na aina bila kulazimisha aina, `null` na `undefined` hazilingani:

:::interactive_editor

```js
console.log(null === undefined); // false
```

:::

Tofauti hii ni muhimu kuizingatia unapoandika sentensi za masharti au unapotekeleza ukaguzi wa usawa katika msimbo wako. Unapolinganisha `null` au `undefined` na thamani nyingine kwa kutumia opereta ya usawa (`==`), tabia inaweza kuwa isiyotegemewa. Kwa mfano:

:::interactive_editor

```js
console.log(null == 0);  // false
console.log(null == ''); // false
console.log(undefined == 0); // false
console.log(undefined == ''); // false
```

:::

Mlinganisho huu hurudisha `false` kwa sababu `null` na `undefined` ni sawa kwa kila mmoja (na kwa nafsi zao) tu wakati wa kutumia opereta ya usawa. Tabia ya `null` katika mlinganisho mingine ni ngumu hasa:

:::interactive_editor

```js
console.log(null > 0);  // false
console.log(null == 0); // false
console.log(null >= 0); // true
```

:::

`undefined`, kwa upande mwingine, hubadilishwa kila mara kuwa `NaN` katika muktadha wa nambari, jambo linalofanya mlinganisho wote wa nambari na `undefined` urudishe `false`:

:::interactive_editor

```js
console.log(undefined > 0);  // false
console.log(undefined < 0);  // false
console.log(undefined == 0); // false
```

:::

Kwa kuzingatia tofauti hizi ndogo, kwa ujumla inashauriwa kutumia opereta ya usawa mkali unapolinganisha thamani, hasa unapoangalia `null` na `undefined`. Njia hii husaidia kuepuka kulazimisha aina isiyotegemewa na kufanya tabia ya msimbo wako iwe rahisi kutabirika.

Kwa muhtasari, ingawa `null` na `undefined` zote hutumika kuwakilisha kutokuwepo kwa thamani, hufanya kazi tofauti katika mlinganisho. Kuelewa tofauti hizi ni muhimu kwa kuandika msimbo wa JavaScript ulio wazi na usio na hitilafu.
