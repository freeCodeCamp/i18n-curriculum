# --interactive--

`for...of`ループは、イテラブルからの値を繰り返し処理する必要があるときに使います。イテラブルの例としては配列や文字列があります。

`for...of`ループの基本的な構文は以下の通りです。

```js
for (variable of iterable) {
  // code block to be executed
}
```

この例の変数は、繰り返し処理されているイテラブルの現在の値を表します。

もし数値の配列があれば、その変数は配列内の現在の数値になります。文字列があれば、その変数は文字列内の現在の文字になります。

`for...of`ループの動作をよりよく理解できるように、いくつか例を見てみましょう。

最初の例では、数値の配列があり、それぞれの数値を繰り返し処理してコンソールに出力したいと思います。

:::interactive_editor

```js
const numbers = [1, 2, 3, 4, 5];

for (const num of numbers) {
  console.log(num);
}
```

:::

`num`という変数を作成し、配列内の現在の数値を表すようにしました。1回目の繰り返しでは`num`が`1`になり、2回目の繰り返しでは`num`が`2`になります。

ループの中では、現在の数値をコンソールに出力しています。

次の例では、文字列があり、それぞれの文字を繰り返し処理してコンソールに出力したいと思います。

:::interactive_editor

```js
const str = 'freeCodeCamp';

for (let char of str) {
  console.log(char);
}
```

:::

この例では、`char`という変数を作成し、文字列内の現在の文字を表すようにしました。

繰り返しごとに、ループは現在の文字をコンソールに出力します。

`let`や`const`を使って`for...of`ループ内の変数を宣言することができる点は重要です。

ただし、`const`を使う場合は、ループ内で変数の値が変わらないようにしてください。もし変わるとエラーになります。

`const`を使った場合のエラー例を見てみましょう。

```js
const numbers = [1, 2, 3, 4, 5];

for (const num of numbers) {
  console.log(num);
  num = num + 1; // This will cause an error
}
```

この例では、ループ内で`num`の値を変更しようとしています。`num`を`const`で宣言しているため、エラーになります。ループ内で変数の値を変更する必要がある場合は、代わりに`let`を使ってください。

最後に、オブジェクトの配列を扱う例を見てみましょう。

:::interactive_editor

```js
const people = [
  { name: 'John', age: 30 },
  { name: 'Jane', age: 25 },
  { name: 'Jim', age: 40 }
];

for (const person of people) {
  console.log(`${person.name} is ${person.age} years old`);
}
```

:::

この例では、`people`というオブジェクトの配列があります。各オブジェクトは`name`と`age`のプロパティを持っています。

配列をループするとき、`person`という変数を作成し、配列内の現在のオブジェクトを表します。

ループ内では、コンソールにメッセージを出力しています。

最初のメッセージは`John is 30 years old`、2番目は`Jane is 25 years old`、3番目は`Jim is 40 years old`になります。

`for...of`ループは、配列や文字列のようなイテラブルの値を繰り返し処理する必要があるときに非常に便利です。また、読みやすくコードを簡潔にできます。
