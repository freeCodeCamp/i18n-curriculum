# --interactive--

У попередньому уроці ви дізналися, як використовувати HTML, щоб обмежити значення, які користувачі можуть відправити у вашій формі. Але іноді цього недостатньо. Якщо ви хочете зробити щось складніше, наприклад показувати власні повідомлення про помилки користувачу, вам потрібно буде використати JavaScript.

Деякі HTML-елементи, такі як `textarea` і `input`, відкривають доступ до Constraint Validation API. Цей API дозволяє перевірити, чи проходить значення, яке користувач ввів для цього елемента, будь-яку валідацію на рівні HTML, яку ви задали, наприклад мінімальну довжину або відповідність шаблону.

Але як це насправді використовувати? Припустимо, ви хочете, щоб співробітники компанії надсилали відгуки через форму, схожу на цю:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css" />
<form>
  <label>Enter your email: </label>
  <input required type="email" />

  <label>Enter your feedback: </label>
  <textarea required placeholder="Your feedback here..."></textarea>

  <button type="submit">Submit Feedback</button>
</form>
```

Ми використовуємо поле введення типу email `input`, яке має вбудовану валідацію для перевірки базових речей, наприклад, чи містить введене значення знак @ (`@`).

Але що, якщо користувач введе адресу е-пошти, наприклад `example@email.com`? Це пройде базову валідацію, але ми хочемо бути більш конкретними і приймати лише адреси е-пошти компанії.

Ось тут ми можемо використати атрибут `pattern`, щоб вказати, що адреса е-пошти має закінчуватися на домен компанії. Ось як виглядатиме оновлений приклад:

У наведеному прикладі ми отримали наше поле введення з DOM і додали слухача події input.

Ми знаємо, що `e.target` посилається на елемент, який викликав подію. У цьому випадку це наше `input`. Але що таке метод `checkValidity()`?

Це частина Constraint Validation API. Метод `checkValidity()` повертає `true`, якщо елемент проходить усі HTML-валидації (на основі його атрибутів), і `false`, якщо ні.

Коли ми вводимо некоректне значення, у консолі з’являється `false`. Тепер, коли ми знаємо, що введення некоректне, давайте повідомимо про це:

Властивість `validity` є екземпляром об’єкта `ValidityState`. Ось приклад того, як цей об’єкт може виглядати у браузері:

```js
ValidityState {
  badInput: false,
  customError: false,
  patternMismatch: true,
  rangeOverflow: false,
  rangeUnderflow: false,
  stepMismatch: false,
  tooLong: false,
  tooShort: false,
  typeMismatch: true,
  valueMissing: false,
  valid: true
}
```

Існує кілька корисних властивостей, які містять булеві значення `true` або `false`.

Деякі з цих корисних властивостей, які ви можете дослідити самостійно, це властивість `valueMissing`, яка є `true`, коли обов’язкове поле введення залишено порожнім. Або `patternMismatch`, яка є `true`, якщо значення не відповідає заданому регулярному виразу.
