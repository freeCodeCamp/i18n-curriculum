# --interactive--

繼承是 CSS 中一個關鍵的概念，用來決定樣式如何從父元素傳遞到子元素。

就像在現實世界中，孩子經常從父母繼承特徵一樣，在 CSS 中，某些屬性可以由子元素從父元素繼承。

這允許在整個文件中以更高效的方式套用一致的樣式。

在 CSS 中，並非所有屬性都會被預設繼承。<dfn>例如</dfn>，像是 `color`、`font-family` 和 `line-height` 這些屬性會被繼承。這表示如果你在父元素上設定了文字顏色，所有子元素都會繼承該顏色，除非你特別改寫它。

例如，考慮以下範例，其中父層 `div` 元素使用行內樣式設定其顏色，而子層 `p` 元素繼承該顏色：

:::interactive_editor

```html
<div style="color: blue;">
  This is the parent element.
  <p>This is the child element inheriting the color.</p>
</div>
```

:::

在這種情況下，父層的 `div` 和子層的 `p` 都會以藍色顯示它們的文字，因為顏色是被繼承的。

另一方面，像是 `margin`、`padding`、`border` 和 `background` 這些屬性預設不會被繼承。如果你想讓子元素繼承這些樣式，你需要明確地設定它們，無論是直接在子元素上設定，或是使用 `inherit` 關鍵字。

`inherit` 關鍵字可用來強制從父元素繼承一個屬性，即使該屬性通常不會被繼承。

例如，如果你想讓特定的子元素擁有與其父元素相同的 `padding`，你可以在子元素上設定 `padding: inherit`：

:::interactive_editor

```html
<div style="padding: 20px;">
  This is the parent element with padding.
  <p style="padding: inherit;">This is the child element inheriting the padding.</p>
</div>
```

:::

在這種情況下，子元素 `p` 將會繼承其父元素 `div` 的 `20px` 內邊距。

繼承對於維護樣式表的一致性和減少冗餘特別有用。

你可以在父元素上定義一次相同的樣式規則，子元素將會繼承它，取代為多個元素重複撰寫相同的樣式規則。這可以讓你的 CSS 更簡潔且更容易管理。

但是，重要的是要記住繼承只會單向運作－從父元素到子元素。如果你在子元素上改寫樣式，不會影響父元素。
