# --description--

现在你已经准备好通过在你的计算机上安装 Node.js 来迈出进入 Node.js 世界的第一步。但首先，在你安装 Node.js 之前，我们建议安装一个名为 NVM（Node Version Manager）的工具。顾名思义，NVM 是一个用于在同一台计算机上管理多个 Node.js 版本的工具。

从一开始设置这个工具将有助于你将来，因为你可能需要在不同的项目中切换不同版本的 Node.js。使用 NVM，你可以非常轻松地做到这一点。

让我们看看你如何在 macOS/Linux 和 Windows 上安装 NVM，以及如何使用它来安装 Node.js。

## 如何在 macOS/Linux 上安装 NVM

在 macOS 和 Linux 上，你可以通过一个 shell 脚本直接安装 NVM。

但对于 macOS，请注意 NVM 的官方文档提到“你需要在运行安装脚本之前手动安装 Xcode 命令行工具，否则安装将失败。”

在 macOS 上，你可以通过在终端中运行此命令来安装 Xcode 命令行工具：

```bash
xcode-select --install
```

安装完成后，你可以使用此 `Command` 查看版本号并验证安装：

```bash
xcode-select --version
```

然后，是时候安装 NVM 了。你可以通过运行以下命令之一来下载并执行 NVM 安装脚本：

```bash
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.3/install.sh | bash
```

```bash
wget -qO- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.3/install.sh | bash
```

要获取你阅读本文时的最新版本的 NVM，请参阅官方 NVM 文档：`https://github.com/nvm-sh/nvm?tab=readme-ov-file#install--update-script`。在那里，你将找到带有最新版本号的这些命令。

当进程完成后，你可能需要重启终端以装载新建的命令和新建的配置。

然后，运行此 `Command` 以确认已正确安装 `NVM` 并查看你当前的版本：

```bash
nvm -v
```

你应该在输出中看到你的 NVM 版本。

## 如何在 macOS/Linux 上使用 NVM 安装 Node.js

既然你已经在 macOS/Linux 上安装了 NVM，你就可以安装 Node.js。

要安装最新的长期支持（LTS）版本的 Node.js，请运行此命令：

```bash
nvm install --lts
```

注意 `--lts` 标记。该标记表示你想安装当前的长期支持（LTS）版本的 Node.js。通常推荐使用此版本，因为它优先考虑稳定性、可靠性和安全性，而非新建或实验性特色。它经过彻底测试，适合生产环境使用。它还保证有更长的支持周期，重点是错误修复和安全性补丁。

备选方案是，你可以通过在命令中指定版本号来安装特定版本的 Node.js：

```bash
nvm install <version>
```

例如，下面是你如何安装 Node.js v20：

```bash
nvm install 20
```

安装完某个版本的 Node.js 后，你可以使用此命令切换到该版本：

```bash
nvm use <version>
```

例如，以下是如何切换到版本 20：

```bash
nvm use 20
```

最后，你可以使用此 `Command` 查看当前的 Node.js 版本：

```bash
node -v
```

## 如何在 Windows 上安装 NVM

现在让我们看看你如何在 Windows 上安装和使用 NVM。

NVM 的官方版本仅在特定分支中支持 Windows。如果你的设备上设置了 Windows Subsystem for Linux（WSL），你可以按照上面的 macOS/Linux 指南安装和使用 NVM。

但是，对于 Windows 本身，有一些备选方案。其中之一是 `nvm-windows`，这是一个用于 Windows 的 Node.js 版本管理工具。这是一个与官方 NVM 项目完全独立的项目。但这些工具的目的相同——帮助你在同一设备上使用多个版本的 Node.js。

在安装 `nvm-windows` 之前，你应该知道官方仓库建议在安装 NVM for Windows 之前卸载任何先前的 Node 安装。如果这是你第一次安装 Node.js，应该没问题。如果你已经安装了其他版本的 Node.js，建议先卸载它们。

要安装 `nvm-windows`：

- 前往官方仓库的版本部分：`https://github.com/coreybutler/nvm-windows/releases`。
- 下载最新的安装程序。
- 遵循说明。

完成安装后，如果 `nvm` 命令未立即可用，你可能需要重启你的终端。

你可以通过运行此 `Command` 来查看安装是否成功以及你当前的 `nvm-windows` 版本：

```powershell
nvm -v
```

## 如何使用 NVM 为 Windows 安装 Node.js

现在你已经在 Windows 上安装了 `nvm-windows`，你可以准备安装 Node.js。

要安装最新的长期支持（LTS）版本的 Node.js，请运行此命令：

```powershell
nvm install lts
```

通常推荐使用长期支持（LTS）版本，但你可以通过在命令中指定其版本号来安装不同的版本：

```powershell
nvm install <version>
```

例如：

```powershell
nvm install 20
```

安装完成后，你可以使用此命令切换到该版本的 Node.js：

```powershell
nvm use <version>
```

例如：

```powershell
nvm use 20
```

你也可以使用此 `Command` 查看你当前的 Node.js 版本：

```bash
node -v
```

## 基本命令

既然你已经知道如何安装 NVM 和 Node.js，接下来让我们查看一些在使用 Node.js 和 NVM 时你会觉得有用的基本命令。

此外，虽然所有这些命令都适用于安装在 macOS/Linux（包括 WSL）上的 NVM，但它们在 `nvm-windows` 上可能表现不同或根本无法使用。我们将在可能的情况下为 `nvm-windows` 提供备选命令。

### 如何调整 Powershell 的执行策略

如果你正在运行 Windows 并且正在使用 Powershell 和 `nvm-windows`，你可能需要调整 Powershell 的执行策略。Powershell 的默认执行策略是 `Restricted`，这是一项防止脚本运行的安全性措施。

首先，右键点击 Powershell，然后点击“Run as Administrator”以管理员身份运行 Powershell。

然后，要查看当前 Powershell 会话的当前执行策略，请运行以下命令：

```powershell
Get-ExecutionPolicy
```

最后，要将 Powershell 的执行策略设置为 `RemoteSigned`，允许局部的脚本和 `npm` 运行，请使用以下命令：

```powershell
Set-ExecutionPolicy RemoteSigned
```

一旦你运行了上面的命令，你可以关闭 Powershell 的管理员窗口，然后打开一个非管理员 Powershell 窗口。

### 列出所有版本

要列出所有已安装的 Node.js 版本并查看当前正在使用的版本，运行此 `Command`：

```bash
nvm ls
```

### 设置默认 Node.js 版本（仅限 macOS/Linux 上的 NVM）

此命令设置将在新终端会话中使用的 Node.js 默认版本：

```bash
nvm alias default <version>
```

例如：

```bash
nvm alias default 22.20.0
```

你也可以创建自定义别名，为版本赋值一个更具体的名称：

```bash
nvm alias <new_name> <version>
```

例如：

```bash
nvm alias new-api 18.20.8
```

设置别名后，你可以使用这个新建名称切换到该版本：

```bash
nvm use <new_name>
```

例如：

```bash
nvm use new-api
```

### 删除 Node.js 的某个版本

要删除某个版本的 Node.js，请运行此命令：

```bash
nvm uninstall <version>
```

例如：

```bash
nvm uninstall 20.19.5
```

使用适用于 macOS/Linux 的 NVM，你可以通过运行带有主版本号（如 `22` 或 `20`）的 `uninstall` 命令来移除某个版本的 Node.js。例如：

```bash
nvm uninstall 20
```

如果你为该主版本安装了多个版本，例如 `20.14.0` 和 `20.12.2`，将首先移除最新的版本。在这种情况下，运行 `nvm uninstall 20` 将首先移除 `20.14.0`。

### 查看你的 Node.js 版本

要查看你当前的 Node.js 版本，你可以使用以下命令：

```bash
node -v
```

### 查看你 的 `npm` 版本

`npm` 是 Node.js 的包管理器。你可以使用它来安装、发布和管理软件包。安装 Node.js 时会自动安装它。

要查看你当前的 `npm` 版本，运行此命令：

```bash
npm -v
```

### 运行一个 JavaScript 文件

此命令运行你在当前活动的 Node.js 运行期中指定的 JavaScript 文件：

```bash
node <file.js>
```

例如：

```bash
node app.js
```

### 创建一个 Node.js 项目

此命令通过创建一个 `package.json` 文件来创建一个新的 Node.js 项目，该文件跟踪其详细信息和依赖项。

```bash
npm init
```

当你运行 `npm init` 命令时，它会通过交互式 CLI 指南引导你设置 `package.json` 文件。它会询问你项目名称、描述、版本等问题。如果你想从默认的 `package.json` 文件开始，可以使用 `npm init -y` 或 `npm init --yes` 命令。

### 安装一个包

要安装特定的包，运行：

```bash
npm install <package>
```

例如：

```bash
npm install express
```

### 安装依赖项

要安装项目的 `package.json` 文件中列出的所有依赖项，请运行：

```bash
npm install
```
