# --interactive--

JavaScript 中的 `typeof` 運算子是一個簡單但強大的工具，讓你可以查看變數或值的資料型別。它總是傳回一個表示型別的字串。

讓我們來看看幾個範例：

:::interactive_editor

```js
let num = 42;
console.log(typeof num); // "number"
```

:::

在這個第一個範例中，我們已經創建了一個名為 `num` 的變數並指定其值為數字 `42`。當你對名為 `num` 的變數使用 `typeof` 運算子時，它會傳回字串 `number`。

這裡是另一個使用 `typeof` 運算子對名為 `isUserLoggedIn` 的變數進行操作的範例：

:::interactive_editor

```js
let isUserLoggedIn = true;
console.log(typeof isUserLoggedIn); // "boolean"
```

:::

當你對 `isUserLoggedIn` 變數使用 `typeof` 運算子時，它會傳回字串 `boolean`，因為布林值 `true` 已指定給該變數。

在你偵錯或嘗試了解你在程式碼中處理的是什麼型別資料時，使用 `typeof` 運算子特別有用。

然而，JavaScript 在處理 `null` 時有一個眾所周知的怪癖。

讓我們來看一個範例：

:::interactive_editor

```js
let exampleVariable = null;
console.log(typeof exampleVariable); // "object"
```

:::

在這個範例中，我們有一個名為 `exampleVariable` 的變數，並且已將它指定為 `null`。但是當我們使用 `typeof` 運算子時，它會傳回字串 `object`。

這被廣泛認為是 JavaScript 的一個錯誤，可追溯到其早期階段。這種行為的原因根源於 JavaScript 最初的設計方式。

當語言首次被實作時，像 `null` 這樣的值是以一種特殊型別的物件表現，導致這個意料之外的結果。

不幸的是，這已成為語言的一部分，雖然令人困惑，但你需要注意這一點。
