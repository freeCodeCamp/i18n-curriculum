# --description--

تسمية المتغيرات والدوال جانب مهم من كتابة شفرة نظيفة، قابلة للقراءة وسهلة الصيانة. الممارسات الجيدة في التسمية تجعل شفرتك توثق نفسها، مما يقلل الحاجة إلى تعليقات مطولة ويسهل على المطورين الآخرين، بما في ذلك أنت في المستقبل، فهم شفرتك.

لنبدأ بالاصطلاحات العامة في التسمية في JavaScript. في الدروس السابقة تعلمت استخدام camel case لأسماء المتغيرات. بالنسبة للمتغيرات التي تحمل قيمة منطقية، من الممارسات الشائعة استخدام بادئات مثل `is`، `has`، أو `can`. هذا يخبر القارئ فورًا أن المتغير يحمل قيمة منطقية:

```js
let isLoading = true;
let hasPermission = false;
let canEdit = true;
```

بالنسبة للدوال، يجب أن يشير الاسم بوضوح إلى ما تفعله الدالة. غالبًا ما يكون من المفيد أن تبدأ بفعل:

```js
function getUserData(){
  /* ... */
}

function calculateTotal(){
  /* ... */
}

function validateInput(){
  /* ... */
}
```

بالنسبة للدوال التي تُرجع قيمة منطقية وغالبًا ما تُسمى predicates، يمكنك استخدام نفس البادئات `is`، `has`، أو `can`:

```js
function isValidEmail(email) {
  /* ... */
}

function hasRequiredFields(form) {
  /* ... */
}
```

عندما تكون لديك دوال تسترجع بيانات، من الشائع أن تبدأ بالكلمة `get`:

```js
function getProductDetails(productId) {
  /* ... */
}

function getUserProfile(userId) {
  /* ... */
}
```

عندما تكون لديك دوال تضبط بيانات، من الشائع أن تبدأ بالكلمة `set`:

```js
function setUserPreferences(preferences) {
  /* ... */
}

function setPageTitle(title) {
  /* ... */
}
```

بالنسبة لدوال معالِجات الأحداث، قد تسبق الاسم بـ `handle` أو تلحقه بـ `handler`:

```js
function handleClick(){
  /* ... */
}

function onSubmit(){
  /* ... */
}

function keyPressHandler(){
  /* ... */
}
```

معالِج الحدث هو إجراء يحدث بعد وقوع حدث مثل نقر زر. ستتعلم عن ذلك في دروس لاحقة.

عند تسمية متغيرات التكرار والحلقات، من الشائع استخدام أحرف مفردة مثل `i`، `j`، أو `k`، لكن للحلقات المسكنة أو التكرارات المعقدة قد تكون الأسماء الوصفية أكثر فائدة:

```js
for (let i = 0; i < array.length; i++) {
  /* ... */
}

for (let studentIndex = 0; studentIndex < students.length; studentIndex++) {
  /* ... */
}
```

لأسماء المصفوفات، فكر في استخدام أسماء جمع للدلالة على أن المتغير يحتوي على عناصر متعددة:

```js
const colors = ['red', 'green', 'blue'];
const userNames = ['Alice', 'Bob', 'Charlie'];
```

تذكر أن الهدف هو جعل شفرتك واضحة قدر الإمكان. قاعدة جيدة هي أنه إذا احتجت إلى إضافة تعليق لشرح ما يفعله متغير أو دالة، قد ترغب في إعادة تسميته إلى شيء أكثر وصفًا.

أخيرًا، كن متسقًا مع قاعدة الشفرة أو الفريق. إذا كان لفريقك اصطلاحات تسمية محددة، التزم بها. الاتساق يجعل الشفرة أكثر قابلية للقراءة والصيانة للجميع.
