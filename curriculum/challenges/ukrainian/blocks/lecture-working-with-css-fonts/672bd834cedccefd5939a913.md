# --description--

Перед тим як почати, вам слід знати, що таке директива в CSS. Директиви — це інструкції для браузера. Їх можна використовувати, щоб визначати різні аспекти таблиці стилів, такі як медіазапити, ключові кадри, шрифти та інше. Ви дізнаєтеся більше про такі поняття, як медіазапити та ключові кадри, у майбутніх уроках. Цього разу ви вивчите директиву `@font-face`.

За допомогою `@font-face` ви можете визначити кастомний шрифт, вказавши файл шрифту, формат і інші важливі властивості, як-от вагу та стиль. Ось базовий синтаксис. Ви бачите `@font-face`, за яким іде блок у фігурних дужках:

```css
@font-face {
  /* Descriptors */
}
```

Усередині фігурних дужок потрібно додати дескриптори, щоб налаштувати свій шрифт. Давайте розглянемо деякі з найпоширеніших. Дескриптор font-family вказує ім’я, яке ви будете використовувати в таблиці стилів для посилання на цей шрифт. Наприклад, уявімо, що ви визначили цей `@font-face rule`. Він має дескриптор `font-family` зі значенням `MyCustomFont`:

```css
@font-face {
  font-family: "MyCustomFont";
}
```

У такому разі вам потрібно використовувати це ім’я у таблиці стилів там, де ви хочете призначити це сімейство шрифтів усім елементам, які відповідають селектору CSS. Наприклад, ви б використали його як значення `font-family`:

```css
body { 
  font-family: "MyCustomFont"; 
}
```

Але щоб директива `@font-face` була дійсною, потрібно також вказати `src`. Це містить референси на ресурси шрифтів. Фактично це список зовнішніх посилань або локально встановлених імен шрифтів, розділених комами. Тут також можуть бути підказки про формат і технологію ресурсів шрифту.

У коді нижче визначено `src`. Значення — це список URL-адрес, розділених комами і розміщених на окремих рядках для кращої читабельності:

```css
@font-face {
  font-family: "MyCustomFont"; 
  src: url("path/to/font.woff2"),
    url("path/to/font.woff"),
    url("path/to/font.otf");
}
```

Ви можете викликати функцію `url()`, щоб включити файл у таблицю стилів. У цьому випадку ми імпортуємо файли шрифтів. Потрібно написати шлях до файлу в круглих дужках і лапках, включно з розширенням файлу. Для кращої читабельності кожен ресурс можна писати на окремому рядку, але останній має закінчуватися крапкою з комою.

Для кожного ресурсу шрифту можна також вказати формат. Це необов’язково. Це підказка для браузера про формат шрифту. Якщо формат не вказано, ресурс буде завантажено, а формат визначать після завантаження. Якщо формат недійсний, ресурс не завантажуватиметься.

Можливі формати шрифтів включають `collection`, `embedded-opentype`, `opentype`, `svg`, `truetype`, `woff` і `woff2`.

Ось приклад із форматами шрифтів. Зверніть увагу, як ми пишемо конкретний формат у круглих дужках і лапках:

```css
@font-face {
  font-family: "MyCustomFont"; 
  src: url("path/to/font.woff2") format("woff2"),
    url("path/to/font.otf") format("opentype"),
    url("path/to/font.woff") format("woff");
}
```

У цьому прикладі видно, що ми вказуємо формат WOFF2, формат OpenType і формат WOFF.

woff означає «Web Open Font Format». Різниця між WOFF і WOFF2 — це алгоритм стиснення даних. OpenType — це формат масштабованих комп’ютерних шрифтів, розроблений Microsoft і Adobe, який дозволяє користувачам отримувати додаткові функції шрифту. Він широко використовується в основних операційних системах.

Окрім вказання формату, можна також вказати технологію ресурсу шрифту. Це теж необов’язково. Ось приклад, де ми вказуємо технологію другого ресурсу шрифту.

```css
@font-face {
  font-family: "MyCustomFont"; 
  src: url("path/to/font.woff2") format("woff2"),
    url("path/to/font.otf") format("opentype") tech(color-COLRv1),
    url("path/to/font.woff") format("woff");
}
```

Це основи директиви `@font-face`. За допомогою цієї директиви ви можете вказати файл шрифту, щоб визначити кастомні шрифти для своїх унікальних дизайнів.
