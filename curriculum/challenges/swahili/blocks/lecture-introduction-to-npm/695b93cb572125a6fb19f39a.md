# --description--

Katika funzo la awali, ulijifunza ni nini utegemezi. Hapa, utajifunza zaidi kuhusu utegemezi na mambo ya kuzingatia unapotafuta kifurushi cha nje cha kutumia kwa mradi wako.

Utegemezi ni kipande kingine cha msimbo ambacho mradi wako unahitaji ili ufanye kazi. Hivi kawaida ni vifurushi vya nje ambavyo wasanidi programu wengine wameunda na kushiriki katika rejista ya npm.

Kutumia utegemezi kunakuokoa muda mwingi. Badala ya kujenga kila kitu mwenyewe, unaweza kusakinisha kifurushi ambacho tayari kinatatua tatizo la kawaida — mambo kama kuunda seva ya mtandao, kushughulikia njia za maelekezo (routing), kupata data, kusimamia uthibitisho wa utambulisho, na zaidi.

Katika faili yako ya `package.json`, vifurushi vyote ambavyo mradi unategemea vimeorodheshwa chini ya ufunguo wa `dependencies`:

```json
{
  "name": "package-dependencies",
  "version": "1.0.0",
  "description": "a demo on how package dependencies work",
  "main": "index.js",
  "scripts": {
    "test": "echo \"Error: no test specified\" && exit 1"
  },
  "keywords": [
    "Node",
    "NPM"
  ],
  "author": "Kolade Chris",
  "license": "ISC",
  "dependencies": {
    "chalk": "^5.6.2",
    "lodash": "^4.17.21"
  }
}
```

Katika mfano wa faili ya `package.json` hapo juu, `chalk` na `lodash` vimeorodheshwa kama utegemezi wa mradi. Hii ina maana mradi unategemea vifurushi hivi kufanya kazi. Ikiwa havijasakinishwa—na havijaorodheshwa katika faili ya `package.json`—vipengele vyovyote vinavyovitumia vitavunjika.

Kuna pia aina nyingine ya utegemezi inayoitwa `devDependencies`. Hivi ni vifurushi ambavyo unahitaji tu wakati wa kujenga au kupima mradi wako, si wakati unapoendesha mradi katika mazingira ya uzalishaji. Mifano ya kawaida ni:

* Nodemon kwa kuanzisha mradi upya moja kwa moja wakati kuna masasisho
    
* Jest au Mocha kwa ajili ya upimaji
    
* ESLint kwa ukaguzi wa msimbo
    

Kama unavyotarajia, `devDependencies` huonekana chini ya sehemu ya `devDependencies` katika faili ya `package.json`:

```json
{
  "name": "package-dependencies",
  "version": "1.0.0",
  "description": "a demo on how package dependencies work",
  "main": "index.js",
  "scripts": {
    "test": "echo \"Error: no test specified\" && exit 1"
  },
  "keywords": [
    "Node",
    "NPM"
  ],
  "author": "Kolade Chris",
  "license": "ISC",
  "dependencies": {
    "chalk": "^5.6.2",
    "lodash": "^4.17.21"
  },
  "devDependencies": {
    "jest": "^30.2.0",
    "nodemon": "^3.1.10"
  }
}
```

Unapochagua vifurushi vya kutumia katika mradi wako, kuna mambo machache muhimu ya kuzingatia.

Usalama daima unapaswa kuja kwanza. Hata kifurushi kimoja kisicho salama kinaweza kuleta matatizo kwa mradi mzima. Pia unapaswa kuzingatia utendaji, kwa sababu kifurushi kikubwa au kisichoboreshwa vizuri kinaweza kuchelewesha mradi wako.

Pia ni msaada kuchagua vifurushi vinavyopendwa na vinavyopata msaada mzuri. Vifurushi vinavyopendwa kawaida vina jamii zenye shughuli nyingi, nyaraka nzuri, na masasisho mara kwa mara. Njia rahisi ya kuangalia umaarufu ni kuangalia idadi ya upakuaji wa kila wiki kwenye npm. Kwa mfano, kifurushi cha `chalk` hupakuliwa zaidi ya mara 300,000 kila wiki:

Matengenezo ni muhimu pia. Vifurushi vinavyotunzwa vizuri husasishwa mara kwa mara na hufanya kazi vizuri zaidi na matoleo mapya ya programu nyingine.

Mwishowe, usisahau nyaraka. Nyaraka zilizo wazi na za kina hufanya iwe rahisi kuelewa jinsi ya kutumia kifurushi na kunakili muda mwingi unapojumuisha katika mradi wako.

## --answers--
