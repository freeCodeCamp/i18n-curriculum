# --interactive--

在上一课中，我们学习了像素等绝对长度单位。虽然绝对单位在某些情况下很有用，但有时你会想使用相对单位。

在本课中，我们将学习两种相对单位：`em`s 和 `rem`s。

`em` 单位相对于元素的字体大小。如果你为 `font-size` 属性使用 em，则文本的大小将相对于父元素的字体大小。

为了更好地理解其工作原理，让我们来看一个例子：

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css" />
<p class="para">I am a paragraph element</p>

<div class="blue-box"></div>
```

```css
.para {
  font-size: 20px;
  margin-bottom: 1.5em;
  border: 2px solid red;
}

.blue-box {
  background-color: blue;
  color: white;
  padding: 10px;
  width: 100px;
  height: 100px;
}
```

:::

对于超文本标记语言，我们有一个段落和一个 `div` 元素。段落元素的类是 `para`，而 `div` 元素的类是 `blue-box`。

对于 `para` 类，我们将 `font-size` 设置为 `20px`，将 `margin-bottom` 设置为 `1.5em`。这意味着边距将是段落元素字体大小的 1.5 倍。`1.5em` 导致段落底部的边距为 30 像素。我们还设置了一个 `2px solid red` 的 `border`，以便你更好地看到边距。 

对于 `blue-box` 类，我们将背景颜色设置为 `blue`，文本颜色设置为 `white`，并将四个边的 `padding` 设置为 10px。

从示例中，可以看到段落元素底部和蓝色框之间会有一个明显的空隙。

那么如果我们从 `para` 类中移除 `font-size` 属性会发生什么？

底部边距将相对于父元素的字体大小。在这种情况下，父元素是 `body` 元素，其默认字体大小为 `16px`。

`em` 的良好使用场景是在你处理像按钮或卡片这样的模块化组件时。通过使用 `em` 单位，你可以确保组件的所有方面（例如内边距、外边距和边框）随着字体大小成比例缩放，保持一致的比例。

所以，到目前为止，我们一直在使用像素为元素设置字体大小。但这确实给用户带来了一些挑战。

在你的浏览器 `Settings` 中，你可以控制默认 `font` 大小。

对于视力受损的人，他们可能会增大字体大小以便更容易读取。但如果你在网页设计中为字体大小设置了像素，文本将不会与其他内容按比例缩放。

解决此问题的一种方法是为排版使用 `rem` 单位。`rem` 单位相对于根元素的字体大小，该根元素是 `html` 元素。 

默认情况下，`html` 元素的字体大小是 `16px`。如果用户在他们的浏览器设置中增加字体大小，`html` 元素的字体大小将增加，所有 rem 单位将按比例缩放。

下面是使用 `rem` 单位作为字体大小而非像素的示例：

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css" />
<p>This is regular text.</p>
<p class="para">This text is slightly larger.</p>
```

```css
.para {
  font-size: 1.2rem;
  margin-bottom: 1.5em;
  border: 2px solid red;
}
```

:::

通过将字体大小设置为 `1.2rem`，段落元素的字体大小将是根元素字体大小的 1.2 倍。如果用户没有更改默认字体大小，段落元素的字体大小将是 `19.2px`，因为它是 `16px` 的 1.2 倍。

那么你什么时候应该使用 `rem` 单位？`rem` 单位优于像素用于排版，因为它们会根据用户的浏览器设置按比例缩放。这使你的内容对视力障碍用户更具可访问性。

`rem` 单位也有助于维护不同元素之间的一致间距和版型。
