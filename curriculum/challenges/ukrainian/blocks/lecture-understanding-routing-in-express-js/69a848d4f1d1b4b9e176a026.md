# --description--

Тепер, коли ви створили базовий застосунок Express, настав час детальніше розглянути маршрутизацію та як вона працює.

У Express маршрутизація — це спосіб визначення кінцевих точок веб-застосунку. Можна уявити маршрути як правила, які кажуть застосунку, як реагувати, коли користувач заходить на певний URL.

Давайте знову подивимося на наш простий застосунок Express:

```javascript
const express = require("express");
const app = express();
const port = 3000;

app.get("/", (req, res) => {
  res.send("Hello World!");
});

app.listen(port, () => {
  console.log(`Example app listening on port ${port}`);
});
```

У цьому прикладі застосунок визначає маршрут для кореневого URL (`/`). Коли клієнт надсилає GET-запит на цей шлях, сервер відповідає текстом `Hello World!`. Потім застосунок починає слухати вхідні запити на порті `3000`.

Але є й інші методи маршрутів.

Express дозволяє обробляти різні типи HTTP-запитів, використовуючи різні методи маршрутизації. Найпоширеніші з них:

* `app.get()` – отримує дані
    
* `app.post()` – створює нові дані
    
* `app.put()` – оновлює існуючі дані
    
* `app.delete()` – видаляє дані
    

Кожен метод визначає, як ваш застосунок має реагувати на конкретний HTTP-запит за певним маршрутом.

Ось кілька простих прикладів:

```javascript
app.post("/submit", (req, res) => {
  res.send("Form submitted!");
});

app.put("/update", (req, res) => {
  res.send("Item updated!");
});

app.delete("/delete", (req, res) => {
  res.send("Item deleted!");
});
```

У кожному випадку Express слухає запит із певним HTTP-методом і URL-шляхом, а потім виконує відповідну функцію-обробник, щоб надіслати відповідь клієнту.

Ви можете протестувати ці маршрути за допомогою інструментів, як Postman або `curl`, які дозволяють надсилати різні типи HTTP-запитів на ваш сервер.

Ось і все про маршрутизацію в Express — це просто визначення того, як ваш застосунок реагує на різні запити.
