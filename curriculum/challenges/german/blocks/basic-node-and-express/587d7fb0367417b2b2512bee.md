# --description--

In den ersten beiden Zeilen der Datei `myApp.js` sehen Sie, wie einfach es ist, ein Express-App-Objekt zu erstellen. Dieses Objekt verfügt über mehrere Methoden, von denen Sie viele in diesen Aufgaben kennenlernen werden. Eine grundlegende Methode ist `app.listen(port)`. Sie weist Ihren Server an, auf einem bestimmten Port zu lauschen und versetzt ihn so in den laufenden Zustand. Aus Testgründen muss die App im Hintergrund laufen, daher haben wir diese Methode für Sie in der `server.js`-Datei hinzugefügt.

Lassen Sie uns unseren ersten String ausliefern! In Express haben Routen die folgende Struktur: `app.METHOD(PATH, HANDLER)`. METHOD ist eine HTTP-Methode in Kleinbuchstaben. PATH ist ein relativer Pfad auf dem Server (er kann ein String oder sogar ein regulärer Ausdruck sein). HANDLER ist eine Funktion, die Express aufruft, wenn die Route übereinstimmt. Handler haben die Form `function(req, res) {...}`, wobei req das Anfrageobjekt und res das Antwortobjekt ist. Zum Beispiel liefert der Handler

```js
function(req, res) {
  res.send('Response String');
}
```

den String 'Response String' aus.

# --instructions--

Verwenden Sie die `app.get()`-Methode, um den String „Hello Express“ für GET-Anfragen zu bedienen, die dem `/` (Root-)Pfad entsprechen. Stellen Sie sicher, dass Ihr Code funktioniert, indem Sie die Protokolle überprüfen.

**Hinweis:** Der gesamte Code für diese Lektionen sollte zwischen den wenigen Zeilen Code hinzugefügt werden, mit denen wir Sie gestartet haben.
