# --description--

你有没有想过，当你输入一个 URL 并按下回车键时，幕后发生了什么？浏览器如何将代码转换成我们看到的交互式页面？

理解浏览器如何 `Render` 网页对于旨在优化 `performance` 和提升 `user` 体验的开发者来说至关重要。

在本课中，我们将揭开渲染进程的神秘面纱，将其分解为清晰、易于理解的步骤。

首先浏览器解析 `HTML` 并构建 `DOM`。

该进程从浏览器获取页面的超文本标记语言内容开始。它解析该超文本标记语言以构造文档对象模型，或称 DOM —— 一个表现页面内容的树状结构。

考虑这个简单的超文本标记语言截图：

```html
<html>
  <body>
    <h1>Hello, World!</h1>
    <p>Welcome to our website.</p>
  </body>
</html>
```

浏览器解析此内容以创建以 `html` 为根的 DOM 树，包含 `body`，而 `body` 又包含 `h1` 和 `p` 元素。

接下来，浏览器处理 CSS，构造 CSS 对象模型，即 CSSOM。这是另一个树结构，决定元素应如何被样式化。

给定此 CSS：

```css
h1 {
  color: blue;
}
p {
  font-size: 16px;
}
```

CSS 对象模型规定 `h1` 元素为蓝色，`p` 元素的字体大小为 16 像素。

浏览器随后将 DOM 和 CSS 对象模型结合起来形成渲染树。该树仅包含渲染页面所需的节点，每个节点都与其对应的样式点对。

对于我们的示例，渲染树由 `h1` 和 `p` 元素组成，并进行了适当的样式设置。

有了渲染树后，浏览器会在一个称为版型或重排的进程中计算每个元素的确切位置和大小。这确保元素出现在屏幕上的正确位置和维度。

最后，浏览器将像素绘制到屏幕上，根据计算出的样式和版型渲染每个元素。在复杂的页面中，这可能涉及多个图层的合成，以形成最终的视觉输出。

总之，从解析 `HTML` 和 `CSS` 到在屏幕上绘制像素，浏览器的渲染进程是一系列复杂的步骤。
