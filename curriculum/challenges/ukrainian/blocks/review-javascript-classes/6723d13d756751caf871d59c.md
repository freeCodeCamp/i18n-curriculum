# --interactive--

## Основи роботи з класами

- **Визначення**: Класи в JavaScript використовуються для визначення шаблонів створення об’єктів і інкапсуляції даних. Класи містять конструктор — спеціальний метод, який викликається автоматично під час створення нового об’єкта з класу. Він використовується для ініціалізації властивостей об’єкта. Ключове слово `this` тут використовується для посилання на поточний екземпляр класу. Під конструктором можуть бути методи. Методи — це функції, визначені всередині класу, які виконують дії або операції над даними чи станом класу. Вони визначають поведінку, яку можуть виконувати екземпляри класу.

:::interactive_editor

```js
class Dog {
  constructor(name) {
    this.name = name;
  }

  bark() {
    console.log(`${this.name} says woof!`);
  }
}

const dog = new Dog("Gino");
console.log(dog.name);  // Gino
```

:::

Щоб створити новий екземпляр класу, використовуйте ключове слово `new` з іменем класу:

```js
const dog = new Dog("Gino");
```

Також можна створювати класи як вирази класу. Це коли клас анонімний і призначений змінній.

:::interactive_editor

```js
const Dog = class {
  constructor(name) {
    this.name = name;
  }

  bark() {
    console.log(`${this.name} says woof!`);
  }
};

const dog = new Dog("Gino");
console.log(dog.name);  // Gino
dog.bark();  // Gino says woof!
```

:::

## Успадкування класів

- **Визначення**: В програмуванні успадкування дозволяє визначати класи, які успадковують властивості та методи від батьківських класів. Це сприяє повторному використанню коду і встановлює ієрархічні відносини між класами. Батьківський клас — це клас, який слугує шаблоном для інших класів. Він визначає властивості та методи, які успадковуються іншими класами. Дочірній клас — це клас, який успадковує властивості та методи іншого класу. Дочірні класи також можуть розширювати функціональність батьківських класів, додаючи нові властивості та методи. У JavaScript для реалізації успадкування використовується ключове слово `extends`. Воно вказує, що клас є дочірнім класом іншого класу.

:::interactive_editor

```js
class Vehicle {
  constructor(brand, year) {
    this.brand = brand;
    this.year = year;
  }
}

class Car extends Vehicle {
  honk() {
    console.log("Honk! Honk!");
  }
}

const myCar = new Car("freeCodeCamp Motors", 2019);
console.log(myCar.brand);  // freeCodeCamp Motors
console.log(myCar.year);  // 2019
myCar.honk();  // Honk! Honk!
```

:::

Ключове слово `super` використовується для доступу до методів, конструкторів і полів батьківського класу.

:::interactive_editor

```js
class Vehicle {
  constructor(brand, year) {
    this.brand = brand;
    this.year = year;
  }
}

class Car extends Vehicle {
  constructor(brand, year, numDoors) {
    super(brand, year);
    this.numDoors = numDoors;
  }
}

const myCar = new Car("freeCodeCamp Motors", 2019, 4);
console.log(myCar.brand);  // freeCodeCamp Motors
console.log(myCar.year);  // 2019
console.log(myCar.numDoors);  // 4
```

:::

## Робота зі статичними методами та статичними властивостями

- **Статичні методи**: Ці методи часто використовують для утилітних функцій, які не потребують доступу до конкретного стану об’єкта. Вони визначені всередині класів для інкапсуляції пов’язаної функціональності.

:::interactive_editor

```js
class Movie {
  constructor(title, rating) {
    this.title = title;
    this.rating = rating;
  }

  static compareMovies(movieA, movieB) {
    if (movieA.rating > movieB.rating) {
      console.log(`${movieA.title} has a higher rating.`);
    } else if (movieA.rating < movieB.rating) {
      console.log(`${movieB.title} has a higher rating.`);
    } else {
      console.log("These movies have the same rating.");
    }
  }
}

let movieA = new Movie("Movie A", 80);
let movieB = new Movie("Movie B", 45);

Movie.compareMovies(movieA, movieB);  // Movie A has a higher rating.
```

:::

Статичні методи також корисні для реалізації "фабричних" методів. Фабричний метод — це метод, який ви визначаєте додатково до конструктора для створення об’єктів на основі певних критеріїв.

```js
class Pizza {
  constructor(type, price) {
    this.type = type;
    this.price = price;
  }

  static createMargherita() {
    return new this("Margherita", 6.99);
  }
}

let myPizza = Pizza.createMargherita();
console.log(myPizza);  // Pizza { type: "Margherita", price: 6.99 }
console.log(myPizza.type);  // Margherita
```

- **Статичні властивості**: Ці властивості використовуються для визначення значень або атрибутів, які пов’язані з самим класом, а не з екземплярами класу. Статичні властивості спільні для всіх екземплярів класу і до них можна отримати доступ без створення екземпляра класу.

:::interactive_editor

```ts
class Car {
  // Static property
  static numberOfWheels = 4;

  constructor(make, model) {
    this.make = make;
    this.model = model;
  }

  // Instance method
  getCarInfo() {
    return `${this.make} ${this.model}`;
  }

  // Static method
  static getNumberOfWheels() {
    return Car.numberOfWheels;
  }
}

// Accessing static property directly from the class
console.log(Car.numberOfWheels);  // 4
```

:::
