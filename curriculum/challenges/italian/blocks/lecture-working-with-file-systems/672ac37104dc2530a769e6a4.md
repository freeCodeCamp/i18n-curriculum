# --description--

Quando inizi a costruire le tue applicazioni web, devi fare attenzione a come nomini i tuoi file.

Vuoi nominare i file in modo che siano facili da capire e da mantenere.

Diamo un’occhiata ad alcuni esempi di nomi di file buoni e cattivi per file HTML. Nota che l’estensione `.html` in questi esempi ci indica che si tratta di un file HTML.

Ecco un esempio di un nome di file cattivo:

```md
index1234.html
```

Dal solo nome non è chiaro di cosa tratta il file. Questo file potrebbe essere per una pagina prodotti, una pagina blog o qualsiasi altro tipo di pagina, ma non lo sappiamo dal nome del file.

Ecco un esempio di un nome di file migliore per un file HTML:

```md
about-us.html
```

Usando un nome più descrittivo come `about-us.html`, è chiaro di cosa tratta il file.

Noterai anche che in questo nome di file non ci sono spazi. Invece, usi un trattino, `-`, per separare le parole.

A volte potresti vedere un nome di file chiamato `index.html`. Questo è un nome di file speciale usato per rappresentare la pagina principale di un sito web. Quando visiti un sito, il file `index.html` è il primo file che viene caricato.

Un’altra considerazione importante quando nomini i file è l’uso di caratteri speciali.

Usare una combinazione di caratteri speciali, numeri e lettere può rendere difficile capire di cosa tratta il file.

Ecco un esempio di un nome di file cattivo:

```md
file-1!@.html
```

Questo nome sembra inutilmente complicato e non ci dà alcuna informazione su cosa tratta il file.

Quindi è meglio attenersi all’uso solo di lettere e trattini nei nomi dei file.

Quando inizi a imparare altre lingue, vedrai che ci sono convenzioni comuni per nominare i file anche in quelle lingue.

In questi casi, è sempre meglio consultare la documentazione ufficiale o altre risorse per capire le buone pratiche per nominare i file in quella lingua.

Inoltre ti troverai in situazioni in cui lavori in un team e dovrai seguire le convenzioni che il team ha stabilito per nominare i file.

Consulta sempre la guida di stile del team per assicurarti che i tuoi nomi di file siano coerenti con il resto del team.
