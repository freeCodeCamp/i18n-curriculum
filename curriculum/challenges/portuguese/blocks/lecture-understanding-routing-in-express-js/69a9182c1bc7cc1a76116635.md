# --description--

No Express, **caminhos de rota** são os padrões de URL que determinam como o servidor responde às requisições recebidas. Quando um usuário solicita uma URL específica, o Express associa essa URL a um caminho de rota e executa o manipulador correspondente.

Os caminhos de rota são definidos dentro de métodos de rota como `app.get()`, `app.post()` e assim por diante. Esses caminhos podem ser estáticos, como `/home`, ou dinâmicos, onde parâmetros são usados para capturar valores da URL, como `/user/:id`.

Nesta lição, vamos focar em rotas estáticas. Aqui está um exemplo:

```javascript
// Static route
app.get("/home", (req, res) => {
  res.send("Welcome to the homepage!");
});
```

Neste exemplo, o servidor escuta requisições GET para `/home` e responde com uma mensagem simples.

O método `app.get()` recebe dois argumentos:

* O primeiro argumento, `"/home"`, é o caminho de rota estático (ou URL) que o servidor escuta.
    
* O segundo argumento é uma função de callback que é executada quando uma requisição GET é feita para esse caminho.
    

Dentro do callback, `res.send()` envia uma resposta de volta para o cliente. Neste caso, ele envia o texto `Welcome to the homepage!`, que é exibido no navegador.

Aqui está outro exemplo de uma rota estática:

```javascript
// Static route
app.get("/menu/drinks", (req, res) => {
  res.send("Welcome to the drinks menu!");
});
```

Neste caso, o servidor escuta requisições GET para a URL `/menu/drinks` e envia uma mensagem quando essa rota é solicitada.

Você pode usar rotas assim quando seu site tem várias páginas relacionadas agrupadas sob um caminho comum. Por exemplo, um site de restaurante poderia organizar suas páginas de menu assim:

* `/menu/food`
    
* `/menu/drinks`
    
* `/menu/desserts`
    

Essa abordagem ajuda a manter suas URLs claras e significativas, além de facilitar a organização e manutenção das rotas conforme sua aplicação cresce.
