# --description--

npm 레지스트리에 대해 알아보고, JavaScript 패키지를 전 세계와 공유하는 방법을 배워보세요.

**npm 레지스트리**는 공개 및 비공개 패키지의 데이터베이스입니다.

개발자와 조직은 이 레지스트리에 패키지를 게시하여 도구를 커뮤니티, 팀원, 그리고 많은 경우 모든 Node.js 개발자와 공유할 수 있습니다.

패키지는 공개 또는 비공개일 수 있습니다. 공개 패키지는 누구나 접근할 수 있고, 비공개 패키지는 특정 사용자나 조직 구성원만 접근할 수 있습니다.

npm 레지스트리에서는 몇 번의 클릭만으로 만든 것을 전 세계와 공유할 수 있습니다. 멋지죠?

패키지를 단계별로 게시하는 방법을 살펴보겠습니다.

## Step 1: 계정 만들기

첫 번째 단계는 무료 npm 계정을 만드는 것입니다. 계정을 만들려면:

* [https://www.npmjs.com/signup](https://www.npmjs.com/signup)로 이동하세요.
    
* 양식을 작성하세요. 사용자 이름, 이메일, 비밀번호를 입력해야 합니다.
    

가입 페이지에 따르면 "이메일 주소는 게시하는 패키지의 메타데이터에 추가되어 공개될 수 있습니다"라는 점을 유의하세요.

## Step 2: **로그인**

npm 계정을 만들었으면 터미널에서 다음 명령어를 실행해 로그인하세요:

```bash
npm login
```

그런 다음 계정 정보를 사용해 인증해야 합니다.

## **Step 3: `package.json` 파일 만들기**

패키지를 게시하기 전에 `package.json` 파일을 다음 명령어로 만들어야 합니다:

```bash
npm init
```

(프로젝트에 이미 있다면 **4단계**로 바로 넘어가세요).

`package.json` 파일에 다음 정보를 입력해야 합니다:

* **name**: 보통 프로젝트나 디렉터리 이름입니다. 패키지를 공개할 계획이라면, 이 이름은 고유해야 하며, 범위(scope)를 사용하지 않는 한 중복될 수 없습니다(곧 설명합니다).
    
* **version**: 기본값인 `"1.0.0"`부터 시작하세요. 이는 패키지의 초기 버전을 나타내며, 변경할 때마다 업데이트할 수 있습니다.
    
* **main**: 프로젝트의 진입점입니다. 기본값은 `index.js`입니다.
    
* **author**: 이름이나 선호하는 닉네임을 적으세요.
    
* **license**: 특정 라이선스를 선택하세요. 예를 들어 MIT, ISC 등이 있습니다.
    
* **description**: 패키지에 대한 간단한 요약입니다.
    

패키지 이름을 정할 때 npm 레지스트리에 이미 사용 중인지 확인하세요. 다음 URL에 접속해 `<your-package-name>`를 패키지 이름으로 바꾸면 됩니다:

```bash
https://www.npmjs.com/package/<your-package-name>
```

해당 URL에 패키지가 없다면 이름을 사용할 수 있습니다.

하지만 이미 사용 중인 이름을 꼭 쓰고 싶다면, **scope**를 사용해 게시할 수 있습니다.

### **스코프(Scope)**

npm 레지스트리에 사용자나 조직 계정을 만들면 사용자 이름이나 조직 이름으로 스코프가 부여됩니다.

스코프는 이름 충돌을 피하는 데 유용합니다. 같은 이름의 패키지가 있어도 스코프 아래에 게시할 수 있기 때문입니다.

스코프가 있는 패키지 이름은 스코프 이름으로 시작합니다. 예를 들어 `package.json`에서는 다음과 같은 키-값 쌍을 찾을 수 있습니다:

```bash
"name": "@<scope-name>/<your-package-name>"
```

필요하면 파일에서 직접 이 부분을 수정하거나, `npm init` 명령어에 `--scope=@<scope-name>`를 추가해 처음부터 스코프를 포함한 `package.json` 파일을 만들 수 있습니다.

```bash
npm init --scope=@<scope-name>
```

스코프 이름은 개인 계정이라면 사용자 이름, 조직 계정이라면 조직 이름입니다.

## Step 4: README.md 만들기

패키지의 README는 마크다운 파일로, 패키지가 하는 일과 사용법을 설명합니다.

포함하면 좋은 내용 예시는 다음과 같습니다:

* 목적
    
* 설치 방법
    
* 중요한 의존성 및 주의사항
    
* 시작하는 방법
    
* 예제 코드 스니펫
    
* 기여 가이드라인(기여를 받는 경우)
    
* 버전별 변경 사항을 기록하는 변경 로그 파일 링크
    

README 구조는 유연하므로 필요에 맞게 조정하세요.

## **Step 5:** `.npmignore`

`.npmignore` 파일은 npm 레지스트리에 게시할 때 어떤 파일을 포함하고 어떤 파일을 무시할지 알려줍니다. 비밀 키, 비밀번호, 개인정보 같은 민감한 정보를 공유하지 않도록 매우 중요합니다.

이 파일에는 무시할 파일, 디렉터리, 파일 유형을 지정하는 패턴을 작성할 수 있습니다. `.gitignore` 파일과 같은 패턴 매칭 규칙을 따릅니다.

Git을 사용해봤다면 이 파일이 `.gitignore`와 매우 비슷하다는 생각이 들 겁니다. 실제로 npm 문서에 따르면 "`.npmignore` 파일이 없고 `.gitignore` 파일이 있으면, npm은 `.gitignore` 파일에 매칭되는 항목을 무시합니다."

또한 npm은 패키지의 루트 디렉터리뿐 아니라 모든 하위 디렉터리에서 `.npmignore`와 `.gitignore` 파일을 찾습니다.

기본적으로 다음 경로들은 자동으로 무시되므로 `.npmignore`에 추가할 필요가 없습니다:

* `.*.swp`
    
* `._*`
    
* `.DS_Store`
    
* `.git`
    
* `.gitignore`
    
* `.hg`
    
* `.npmignore`
    
* `.npmrc`
    
* `.lock-wscript`
    
* `.svn`
    
* `.wafpickle-*`
    
* `config.gypi`
    
* `CVS`
    
* `npm-debug.log`
    

`node_modules` 디렉터리도 무시되지만, 번들된 의존성은 제외됩니다.

## **Step 6: 패키지 게시하기**

npm 레지스트리에 패키지를 게시하기 전에 다음 명령어로 테스트하세요:

```bash
npm install <full-path-to-package-directory>
```

게시할 준비가 되면, 패키지 유형(스코프 여부)에 따라 명령어가 달라집니다.

스코프가 없는 패키지는 다음 명령어를 실행하세요:

```bash
npm publish
```

스코프가 있는 패키지는 기본적으로 비공개 패키지로 게시됩니다(계정에 이 기능이 있으면). 공개하려면 명령어에 `--access public`를 추가해야 합니다.

```bash
npm publish --access public
```

공개 패키지를 보려면 다음 URL로 이동하세요:

```bash
https://npmjs.com/package/<your-package-name>
```

패키지 이름 아래에 가시성(공개 또는 비공개) 표시가 있을 것입니다.

## **Step 7 (선택 사항): 패키지 설치하기**

준비가 완료되었으면, Node.js 프로젝트에서 다음 명령어로 패키지를 설치할 수 있습니다:

```bash
npm install <your-package-name>
```

## 패키지 업데이트하기

게시 후 패키지에 변경 사항이 필요하면 `package.json`에서 버전을 업데이트하세요.

의미 있는 버전 관리의 표준 관례는 다음과 같습니다:

* **첫 번째 릴리스:** `1.0.0`.
    
* **패치 릴리스:** 세 번째 숫자를 증가시킵니다. 예: `1.0.0`에서 `1.0.1`로 업데이트.
    
* **마이너 릴리스:** 가운데 숫자를 증가시키고 마지막 숫자를 0으로 초기화합니다. 예: `1.0.0`에서 `1.1.0`로 업데이트.
    
* **메이저 릴리스:** 첫 번째 숫자를 증가시키고 가운데와 마지막 숫자를 0으로 초기화합니다. 예: `1.1.0`에서 `2.0.0`로 업데이트.
    
잘하셨습니다. 이제 npm 레지스트리에 패키지를 게시하고, 버그 수정과 기능 추가를 위해 업데이트하는 방법을 알게 되었습니다.
