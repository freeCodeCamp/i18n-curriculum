## 클래스 작업의 기본

- **정의**: JavaScript의 클래스는 객체를 생성하고 데이터를 캡슐화하는 청사진을 정의하는 데 사용됩니다. 클래스에는 생성자가 포함되어 있는데, 생성자는 클래스에서 새 객체가 만들어질 때 자동으로 호출되는 특별한 메서드입니다. 생성자는 객체의 속성을 초기화하는 데 사용됩니다. `this` 키워드는 클래스의 현재 인스턴스를 가리키는 데 사용됩니다. 생성자 아래에는 메서드라고 하는 것들이 있을 수 있습니다. 메서드는 클래스 내부에 정의된 함수로, 클래스의 데이터나 상태에 대해 작업이나 동작을 수행합니다. 메서드는 클래스 인스턴스가 수행할 수 있는 동작을 정의하는 데 사용됩니다.

:::interactive_editor

```js
class Dog {
  constructor(name) {
    this.name = name;
  }

  bark() {
    console.log(`${this.name} says woof!`);
  }
}

const dog = new Dog("Gino");
console.log(dog.name);  // Gino
```

:::

클래스의 새 인스턴스를 만들려면 `new` 키워드 다음에 클래스 이름을 사용하세요:

```js
const dog = new Dog("Gino");
```

클래스를 클래스 표현식으로 만들 수도 있습니다. 이 경우 클래스는 익명이며 변수에 할당됩니다.

:::interactive_editor

```js
const Dog = class {
  constructor(name) {
    this.name = name;
  }

  bark() {
    console.log(`${this.name} says woof!`);
  }
};

const dog = new Dog("Gino");
console.log(dog.name);  // Gino
dog.bark();  // Gino says woof!
```

:::

## 클래스 상속

- **정의**: 프로그래밍에서 상속은 부모 클래스의 속성과 메서드를 상속받는 클래스를 정의할 수 있게 해줍니다. 이는 코드 재사용을 촉진하고 클래스 간 계층적 관계를 설정합니다. 부모 클래스는 다른 클래스의 청사진 역할을 하는 클래스입니다. 부모 클래스는 다른 클래스가 상속받는 속성과 메서드를 정의합니다. 자식 클래스는 다른 클래스의 속성과 메서드를 상속받는 클래스입니다. 자식 클래스는 새로운 속성과 메서드를 추가하여 부모 클래스의 기능을 확장할 수도 있습니다. JavaScript에서는 `extends` 키워드를 사용해 상속을 구현합니다. 이 키워드는 클래스가 다른 클래스의 자식 클래스임을 나타냅니다.

:::interactive_editor

```js
class Vehicle {
  constructor(brand, year) {
    this.brand = brand;
    this.year = year;
  }
}

class Car extends Vehicle {
  honk() {
    console.log("Honk! Honk!");
  }
}

const myCar = new Car("freeCodeCamp Motors", 2019);
console.log(myCar.brand);  // freeCodeCamp Motors
console.log(myCar.year);  // 2019
myCar.honk();  // Honk! Honk!
```

:::

`super` 키워드는 부모 클래스의 메서드, 생성자, 필드에 접근하는 데 사용됩니다.

:::interactive_editor

```js
class Vehicle {
  constructor(brand, year) {
    this.brand = brand;
    this.year = year;
  }
}

class Car extends Vehicle {
  constructor(brand, year, numDoors) {
    super(brand, year);
    this.numDoors = numDoors;
  }
}

const myCar = new Car("freeCodeCamp Motors", 2019, 4);
console.log(myCar.brand);  // freeCodeCamp Motors
console.log(myCar.year);  // 2019
console.log(myCar.numDoors);  // 4
```

:::

## 정적 메서드와 정적 속성 작업

- **정적 메서드**: 이 메서드는 객체의 특정 상태에 접근할 필요가 없는 유틸리티 함수에 자주 사용됩니다. 관련 기능을 캡슐화하기 위해 클래스 내에 정의됩니다.

:::interactive_editor

```js
class Movie {
  constructor(title, rating) {
    this.title = title;
    this.rating = rating;
  }

  static compareMovies(movieA, movieB) {
    if (movieA.rating > movieB.rating) {
      console.log(`${movieA.title} has a higher rating.`);
    } else if (movieA.rating < movieB.rating) {
      console.log(`${movieB.title} has a higher rating.`);
    } else {
      console.log("These movies have the same rating.");
    }
  }
}

let movieA = new Movie("Movie A", 80);
let movieB = new Movie("Movie B", 45);

Movie.compareMovies(movieA, movieB);  // Movie A has a higher rating.
```

:::

정적 메서드는 "팩토리" 메서드를 구현하는 데도 유용합니다. 팩토리 메서드는 생성자 외에 특정 기준에 따라 객체를 생성하기 위해 정의하는 메서드입니다.

```js
class Pizza {
  constructor(type, price) {
    this.type = type;
    this.price = price;
  }

  static createMargherita() {
    return new this("Margherita", 6.99);
  }
}

let myPizza = Pizza.createMargherita();
console.log(myPizza);  // Pizza { type: "Margherita", price: 6.99 }
console.log(myPizza.type);  // Margherita
```

- **정적 속성**: 이 속성은 클래스 자체와 연관된 값이나 속성을 정의하는 데 사용되며, 클래스 인스턴스와는 관련이 없습니다. 정적 속성은 클래스의 모든 인스턴스에서 공유되며, 클래스 인스턴스를 생성하지 않고도 접근할 수 있습니다.

:::interactive_editor

```ts
class Car {
  // Static property
  static numberOfWheels = 4;

  constructor(make, model) {
    this.make = make;
    this.model = model;
  }

  // Instance method
  getCarInfo() {
    return `${this.make} ${this.model}`;
  }

  // Static method
  static getNumberOfWheels() {
    return Car.numberOfWheels;
  }
}

// Accessing static property directly from the class
console.log(Car.numberOfWheels);  // 4
```

:::
