## CSSの基本

- **CSSとは何ですか？**: Cascading Style Sheets（CSS）は、HTML要素にスタイルを適用するためのマークアップ言語です。CSSは色、背景画像、レイアウトなどに使われます。
- **CSSルールの基本構造**: CSSルールは主に2つの部分で構成されます：セレクターと宣言ブロックです。セレクターはCSSで特定のHTML要素を識別し、スタイルを適用するためのパターンです。宣言ブロックは、指定されたセレクターに対して一連のスタイルを適用します。

CSSルールの一般的な構文は以下の通りです：

```css
selector {
    property: value;
}
```

- **`meta name="viewport"` 要素**: この`meta`要素は、特にスマートフォンやタブレットなどの異なるデバイスでページの寸法やスケーリングを制御する方法をブラウザーに指示します。
- **ブラウザーのデフォルトスタイル**: 各HTML要素にはデフォルトでブラウザーのスタイルが適用されます。通常、デフォルトのマージンやパディングなどが含まれます。

## インライン、内部、外部CSS

- **インラインCSS**: これらのスタイルは、HTML要素内で`style`属性を使って直接記述されます。関心の分離のため、通常はインラインCSSを使うことはほとんどありません。

インラインCSSの例は以下の通りです：

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<p style="color: red;">This is a red paragraph.</p>
```

:::

- **内部CSS**: これらのスタイルは、HTMLドキュメントの`<style>`セクション内の`head`タグ内に記述されます。短いコード例を作成する際に便利ですが、通常は内部CSSを使う必要はありません。
- **外部CSS**: これらのスタイルは別のCSSファイルに記述され、HTMLドキュメントの`link`セクション内の`head`要素を使ってリンクされます。ほとんどのプロジェクトでは、内部やインラインCSSよりも外部CSSファイルを使います。

## `width`および`height`プロパティの操作

- **`width`プロパティ**: このプロパティは要素の幅を指定します。幅を指定しない場合、デフォルトは`auto`に設定されます。これにより、ブラウザーは内容、親要素、表示タイプに基づいて要素の幅を決定します。
- **`min-width`プロパティ**: このプロパティは要素の最小幅を指定します。
- **`max-width`プロパティ**: このプロパティは要素の最大幅を指定します。
- **`height`プロパティ**: このプロパティは要素の高さを指定します。同様に、高さはデフォルトで`auto`に設定されており、内部の内容に合わせて調整されます。
- **`min-height`プロパティ**: このプロパティは要素の最小高さを指定します。
- **`max-height`プロパティ**: このプロパティは要素の最大高さを指定します。

## CSSの異なる結合子の種類

- **子孫結合子**: この結合子は、指定された親要素の子孫である要素をターゲットにします。以下の例は、`li`要素内のすべての`ul`アイテムをターゲットにします。

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<ul>
    <li>Example item one</li>
    <li>Example item two</li>
    <li>Example item three</li>
</ul>
```

```css
ul li {
    background-color: yellow;
}
```

:::

- **子結合子（`>`）**: この結合子は、指定された親要素の直接の子要素を選択します。以下の例は、`p`クラスの直接の子であるすべての`container`要素をターゲットにします。

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<div class="container">
  <p>This will get styled.</p>

  <div>
    <p>This will not get styled.</p>
  </div>
</div>
```

```css
.container > p {
  background-color: black;
  color: white;
}
```

:::

- **隣接兄弟結合子（`+`）**: この結合子は、指定された兄弟要素に直後に続く要素を選択します。以下の例は、`h2`要素に直後に続く段落要素を選択します。

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<h2>I am a sub heading</h2>

<p>This paragraph element will get a red background.</p>
```

```css
h2 + p {
  background-color: red;
}
```

:::

- **後続兄弟結合子（`~`）**: この結合子は、指定された要素の後に続くすべての兄弟要素を選択します。以下の例は、`ul`要素の兄弟であり、同じ親を共有する唯一の2番目の段落要素だけにスタイルを適用します。

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<div class="container">
  <p>This will not get styled.</p>
  <ul>
    <li>Example item one</li>
    <li>Example item two</li>
    <li>Example item three</li>
  </ul>
  <p>This will get styled.</p>
</div>
<p>This will not get styled.</p>
```

```css
ul ~ p {
  background-color: green;
}
```

:::

## インライン、ブロック、インラインブロックレベル要素

- **インラインレベル要素**: インライン要素は必要な幅だけを取り、新しい行で始まりません。これらの要素はコンテンツ内で流れ、テキストや他のインライン要素と並んで表示されます。一般的なインライン要素は`span`、`anchor`、`img`要素です。
- **ブロックレベル要素**: ブロックレベル要素は新しい行で始まり、デフォルトで利用可能な幅全体を占め、コンテナの幅いっぱいに広がります。一般的なブロックレベル要素は`div`、`paragraph`、`section`要素です。
- **インラインブロックレベル要素**: `inline-block`プロパティを使って要素を`display`に設定できます。これらの要素はインライン要素のように振る舞いますが、ブロックレベル要素のように`width`や`height`を設定できます。

## マージンとパディング

- **`margin`プロパティ**: このプロパティは要素の外側、要素のボーダーと周囲の要素の間にスペースを適用するために使います。
- **`padding`プロパティ**: このプロパティは要素の内側、コンテンツとボーダーの間にスペースを適用するために使います。
- **`margin`省略形**: 1～4つの値を指定してマージンの各辺を設定できます。1つの値は4辺すべてに適用されます。2つの値は`top`と`bottom`、次に`right`と`left`に設定します。3つの値は`top`、水平（`right`と`left`）、次に`bottom`に設定します。4つの値は`top`、`right`、`bottom`、`left`に設定します。
- **`padding`省略形**: 1～4つの値を指定してパディングの各辺を設定できます。1つの値は4辺すべてに適用されます。2つの値は`top`と`bottom`、次に`right`と`left`に設定します。3つの値は`top`、水平（`right`と`left`）、次に`bottom`に設定します。4つの値は`top`、`right`、`bottom`、`left`に設定します。

## CSSの特異性

- **インラインCSSの特異性**: インラインCSSは要素に直接適用されるため、最も高い特異性を持ちます。内部や外部CSSをオーバーライドします。インラインスタイルの特異性の値は(1, 0, 0, 0)です。
- **内部CSSの特異性**: 内部CSSはHTMLドキュメントの`style`セクション内の`head`要素で定義されます。インラインスタイルより特異性は低いです。内部と外部CSSは同じ特異性レベルを共有し、場所ではなくセレクターによって決まります。
- **外部CSSの特異性**: 外部CSSは`link`セクションの`head`要素を介してリンクされ、別の`.css`ファイルに記述されます。内部CSSと同様に、特異性は使用されるセレクターによって決まります。2つのルールが同じ特異性の場合、ソースの順序が勝者を決めます。ドキュメントで後に現れるルールが優先されます。外部CSSは大規模なプロジェクトでの保守性に優れています。
- **ユニバーサルセレクター（`*`）**: ドキュメント内のすべての要素にマッチする特別なタイプのCSSセレクターです。ページ上のすべての要素にスタイルを適用するためによく使われ、異なるブラウザー間でのスタイルのリセットや正規化に便利です。ユニバーサルセレクターは最も低い特異性値を持ちます。特異性値はすべての部分に0を寄与します(0, 0, 0, 0)。
- **型セレクター**: これらのセレクターはタグ名に基づいて要素をターゲットにします。型セレクターは他のセレクターに比べて比較的低い特異性を持ちます。型セレクターの特異性値は(0, 0, 0, 1)です。
- **クラスセレクター**: これらのセレクターはピリオド(`.`)に続けてクラス名を指定して定義されます。クラスセレクターの特異性値は(0, 0, 1, 0)です。これはクラスセレクターが型セレクターをオーバーライドできることを意味しますが、IDセレクターやインラインスタイルにはオーバーライドされます。
- **IDセレクター**: IDセレクターはハッシュ記号(`#`)に続けてID名を指定して定義されます。IDセレクターは非常に高い特異性を持ち、型セレクターやクラスセレクターより高いですが、インラインスタイルよりは低いです。IDセレクターの特異性値は(0, 1, 0, 0)です。
- **`!important`キーワード**: スタイルルールに最高の優先度を与え、他の宣言をオーバーライドできるようにします。使用すると、ブラウザーは他のセレクターの特異性に関係なく指定されたスタイルを適用します。`!important`の使用は、CSSの保守やデバッグを難しくするため注意が必要です。
- **カスケードアルゴリズム**: 複数のスタイルが同じ要素をターゲットにする場合に、どのCSSルールを適用するかを決定するアルゴリズムです。明確に定義されたルールに基づいて最適なスタイルが使われるようにします。
- **CSSの継承**: スタイルが親要素から子要素に受け継がれるプロセスです。継承により、ドキュメントツリーの上位でスタイルを定義し、複数の要素に明示的に指定しなくても適用できます。
