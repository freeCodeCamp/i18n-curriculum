# --description--

Селектори класу вибирають усі елементи з заданим `class`, який є атрибутом, що ви додаєте до HTML-елемента. Ось як додати клас до HTML-тега і вибрати його в CSS:

```html
<!-- index.html -->

<div class="alert-text">
  Please agree to our terms of service.
</div>
```

```css
/* styles.css */

.alert-text {
  color: red;
}
```

Зверніть увагу на синтаксис селекторів `class`: крапка безпосередньо перед регістрозалежним значенням атрибута класу. Класи не обов’язково мають бути унікальними, тому ви можете використовувати той самий `class` на скільки завгодно елементів.

Ще одна річ, яку можна зробити з атрибутом `class`, — додати кілька класів до одного елемента у вигляді списку, розділеного пробілами, наприклад `class="alert-text severe-alert"`. Оскільки пробіли використовуються для розділення імен `class`, ніколи не слід використовувати пробіли для багатослівних імен, натомість слід використовувати дефіс.

## ID селектори
ID селектори схожі на селектори `class`. Вони вибирають елемент із заданим `id`, який є іншим атрибутом, що ви додаєте до HTML-елемента:

```html
<!-- index.html -->

<div id="title">My Awesome 90's Page</div>
```

```css
/* styles.css */

#title {
  background-color: red;
}
```

Замість крапки використовується хештег безпосередньо перед регістрозалежним значенням атрибута `id`. Поширена помилка — надмірне використання атрибута `id`, коли це не обов’язково, і коли класи цілком підходять. Хоча є випадки, коли використання `id` має сенс або необхідне, наприклад, щоб скористатися специфічністю або щоб посилання перенаправляло на розділ на поточній сторінці, слід використовувати `id` помірковано (якщо взагалі).

Головна відмінність між класами та ID полягає в тому, що елемент може мати лише один `id`. `id` не можна повторювати на одній сторінці, а атрибут `id` не повинен містити жодних пробілів.
