# --interactive--

CSS kann auf einer Webseite auf drei Hauptarten angewendet werden: inline, intern oder extern.

Jede Methode hat ihren eigenen Anwendungsfall, Vorteile und Einschränkungen. Zu wissen, wann Sie welche verwenden sollten, ist entscheidend, um sauberen, effizienten und wartbaren Code zu schreiben.

Lassen Sie uns die drei Arten von CSS und deren Einsatzmöglichkeiten genauer betrachten.

Inline-CSS wird direkt innerhalb eines HTML-Elements mit dem `style` Attribut geschrieben. Es wendet Stile auf ein bestimmtes Element an.

Hier ist ein Beispiel für inline-CSS:

:::interactive_editor

```html
<p style="color: green;">This is an inline-styled paragraph.</p>
```

:::

In diesem Beispiel verwenden wir das `style` Attribut, um den Text des Absatzes auf `green` zu setzen.

Inline-CSS wird in der Regel für schnelle, einmalige Stile oder zum Überschreiben anderer Stile für ein bestimmtes Element verwendet.

Es sollte jedoch in den meisten Fällen vermieden werden, da es das HTML unübersichtlich machen und den Code schwerer wartbar machen kann.

Meistens ist es besser, internes oder externes CSS zu verwenden, um Ihre Stile organisiert und wartbar zu halten.

Internes CSS wird innerhalb der `style` Tags im `head` Abschnitt eines HTML-Dokuments geschrieben. Es wendet Stile auf die gesamte Seite an und ist nützlich, wenn Sie ein einzelnes Dokument gestalten müssen.

Hier ist ein Beispiel für internes CSS:

:::interactive_editor

```html
<head>
  <style>
    p {
      color: blue;
    }
  </style>
</head>
<body>
  <p>This paragraph is styled using internal CSS.</p>
</body>
```

:::

Im obigen Beispiel wendet das interne CSS blauen Text auf alle `p` Elemente auf der Seite an.

Internes CSS eignet sich am besten, wenn Sie Stile nur auf eine bestimmte Seite anwenden möchten und nicht auf mehrere Seiten. Es ist nützlich für einseitige Websites oder wenn die Stile nicht an anderer Stelle wiederverwendet werden müssen.

Es gibt jedoch einige Nachteile, wie die fehlende Wiederverwendbarkeit über mehrere Seiten hinweg. Außerdem vermischt es, ähnlich wie inline-CSS, HTML und CSS, was den Code in größeren Projekten schwerer wartbar macht.

Externes CSS wird in einer separaten `.css` Datei geschrieben und mit dem HTML-Dokument über das `link` Element im `head` Abschnitt verknüpft.

Es ermöglicht Ihnen, mehrere Seiten einheitlich zu gestalten und ist die bevorzugte Methode in der professionellen Webentwicklung.

Hier ist ein Beispiel, wie alle Absatz-Elemente gestaltet werden:

:::interactive_editor

```html
<head>
  <link rel="stylesheet" href="styles.css">
</head>
<body>
  <p>This paragraph is styled using external CSS.</p>
</body>
```

```css
p {
  color: red;
}
```

:::

In einer früheren Lektion haben Sie gelernt, dass das `link` Element ein `rel` Attribut besitzt, das die Beziehung zwischen dem aktuellen Dokument und der verknüpften Ressource angibt, z. B. das Verknüpfen mit einem Stylesheet oder einer externen Ressource.

Das `href` Attribut gibt hingegen die URL der verknüpften Ressource an und zeigt, woher die Ressource abgerufen werden soll.

Externes CSS ist ideal für große Projekte, bei denen Sie einen konsistenten Stil über mehrere Seiten hinweg beibehalten möchten.

Es fördert die Trennung der Verantwortlichkeiten, indem HTML die Struktur übernimmt und CSS die Gestaltung, was den Code wartbarer und skalierbarer macht.

Zu verstehen, wann Sie welche Art von CSS verwenden sollten, ist entscheidend für eine effiziente und effektive Webentwicklung.

In den meisten Fällen sollte externes CSS Ihre bevorzugte Methode sein, besonders bei größeren und komplexeren Projekten.
