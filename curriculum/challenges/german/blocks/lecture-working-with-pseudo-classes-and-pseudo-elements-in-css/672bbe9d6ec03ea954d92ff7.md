# --interactive--

Das Aussehen und Verhalten von Eingabeelementen ist wichtig beim Erstellen von Webformularen. Ein Formular mit Eingaben, die auf Benutzeraktionen reagieren, verbessert die Benutzererfahrung erheblich und sollte keine Verwirrung oder Frustration verursachen.

CSS bietet mehrere Eingabe-Pseudoklassen, die Ihre Formulare intuitiver und benutzerfreundlicher machen können. Schauen wir uns diese Pseudoklassen im Detail an.

Die `:focus` Pseudoklasse ermöglicht es Ihnen, ein Eingabeelement anzusprechen, wenn ein Benutzer es auswählt oder anklickt, und lenkt so die Aufmerksamkeit auf das aktive Eingabefeld. Das hilft den Nutzenden, leicht zu erkennen, wo sie gerade tippen:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css" />
<form>
  <input type="text" />
</form>
```

```css
input:focus {
  border: 2px solid crimson;
  outline: none;
}
```

:::

Wie der Name schon sagt, wird die `:hover` Pseudoklasse ausgelöst, wenn der Benutzer den Cursor über ein Element bewegt. Sie gibt visuelles Feedback, bevor der Benutzer mit der Eingabe interagiert, und signalisiert so effektiv, dass die Eingabe bereit für eine Aktion ist.

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css" />
<form>
  <input type="text" />
</form>
```

```css
input:hover {
  background-color: orange;
}
```

:::

Im obigen Beispiel ändert sich der Hintergrund des Eingabeelements zu einer orangen Farbe, wenn der Benutzer mit der Maus darüberfährt, was ihm signalisiert, dass das Feld zur Interaktion bereit ist.

Die `:checked` Pseudoklasse erlaubt es Ihnen, eine Optionsschaltfläche oder ein Kontrollkästchen anders zu gestalten, wenn ein Benutzer es auswählt. So kann der Benutzer leicht sehen, welche Option er gewählt hat.

Hier ist ein Beispiel mit einem Kontrollkästchen, um den Nutzungsbedingungen auf einer Website zuzustimmen.

**HINWEIS**: Einige CSS-Eigenschaften in diesem Beispiel wurden noch nicht behandelt. Dies soll Ihnen nur eine Vorstellung davon geben, wie man ein benutzerdefiniertes Kontrollkästchen erstellt. Sie lernen, wie das alles funktioniert, in zukünftigen Lektionen und Workshops.

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css" />
<form>
  <label>
  Agree <input class="checkbox" type="checkbox" />
  </label>
</form>
```

```css
.checkbox {
  appearance: none;
  width: 18px;
  height: 18px;
  border: 2px solid #ccc;
  border-radius: 4px;
  display: inline-block;
  position: relative;
  cursor: pointer;
  transition: all 0.25s ease;
  vertical-align: middle; 
}

.checkbox:hover {
  border-color: #888;
}

.checkbox:checked {
  background-color: #4caf50;
  border-color: #4caf50;
}

.checkbox:checked::after {
  content: "";
  position: absolute;
  left: 4px;
  top: 0px;
  width: 5px;
  height: 10px;
  border: solid white;
  border-width: 0 2px 2px 0;
  transform: rotate(45deg);
}

.checkbox:focus {
  outline: 2px solid #90caf9;
  outline-offset: 2px;
}

```

:::

`:required` spricht Eingabeelemente an, die das `required` Attribut haben. Es signalisiert dem Benutzer, dass er das Feld ausfüllen muss, um das Formular absenden zu können.

Hier ist ein Beispiel mit einem Formular, das einige Pflicht-Eingabefelder enthält:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css" />
<form action="#">
  <label for="name">Name:</label>
  <input type="text" id="name" name="name" required />

  <label for="email">Email:</label>
  <input type="email" id="email" name="email" required />

  <label for="phone">Phone Number:</label>
  <input type="tel" id="phone" name="phone" />

  <button type="submit">Submit</button>
</form>
```

```css
input:required {
  border: 2px solid orange;
}
```

:::

Bei der Validierung von Formularen können Sie die `:valid` Pseudoklasse verwenden, um Eingabefelder zu gestalten, die die Validierungskriterien erfüllen, und `:invalid`, um Eingabefelder zu gestalten, die die Kriterien nicht erfüllen. Typischerweise verwenden Sie Grün für eine gültige Eingabe und Rot für eine ungültige Eingabe.

Hier ist ein Beispiel mit der `:valid` Pseudoklasse:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css" />
<form>
  <label for="email">Email:</label>
  <input type="email" id="email" name="email" />
</form> 
```

```css
input:valid {
  border-color: green;
}
```

:::

Hier ist ein Beispiel für die Verwendung der `:invalid` Pseudoklasse:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css" />
<form>  
  <label for="email">Email:</label>
  <input type="email" id="email" name="email" />
</form>
```

```css
input:invalid {
  border-color: crimson;
}
```

:::

Die `:disabled` Pseudoklasse erlaubt es Ihnen, Eingabeelemente auszuwählen und zu gestalten, die deaktiviert sind. Diese Elemente haben das `disabled` Attribut, das verhindert, dass Benutzer mit ihnen interagieren. Wenn eine Eingabe deaktiviert ist, kann sie nicht angeklickt, fokussiert oder bearbeitet werden.

Hier ist ein Beispiel mit einer Bezeichnung und einem deaktivierten Eingabeelement.

Im CSS sprechen wir diese deaktivierte Eingabe an und geben ihr eine Hintergrundfarbe von `lightgray` und ändern den Mauszeiger von Pointer zu `not-allowed`.

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css" />
<form>
  <label for="name">Name:</label>
  <input class="text-input" type="text" id="name" name="name" disabled />
</form>
```

```css
.text-input:disabled {
  background-color: lightgray;
  cursor: not-allowed;
}
```

:::

Der CSS-Wert `cursor: not-allowed;` ändert das Aussehen des Mauszeigers, um anzuzeigen, dass eine Aktion nicht erlaubt ist.

Wenn er auf ein Element angewendet wird, zeigt er einen Cursor mit einem Kreis-Strich-Symbol (normalerweise ein Kreis mit einer diagonalen Linie), um den Nutzenden zu signalisieren, dass das Element nicht interaktiv ist oder nicht angeklickt bzw. bearbeitet werden kann.

Hier sind einige weitere Beispiele für Eingabe-Pseudoklassen und ihre Funktionen:

- `:autofill`: wendet Stile auf Eingabefelder an, die der Browser automatisch mit gespeicherten Daten füllt.
- `:optional`: gestaltet Eingabeelemente, die nicht erforderlich sind und leer bleiben können.
- `:in-range` und `:out-of-range`: gestalten Elemente basierend darauf, ob ihre Werte innerhalb oder außerhalb vorgegebener Bereichsbeschränkungen liegen.
