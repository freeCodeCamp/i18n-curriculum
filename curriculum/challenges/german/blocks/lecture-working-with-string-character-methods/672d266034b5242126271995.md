# --interactive--

Im Programmieren ist es grundlegend zu verstehen, wie Zeichen als Zahlen dargestellt werden. Hier kommt ASCII ins Spiel. ASCII, kurz für American Standard Code for Information Interchange, ist ein Zeichencodierungsstandard, der in Computern verwendet wird, um Text darzustellen. Er weist jedem Zeichen einen numerischen Wert zu, der von Maschinen weltweit erkannt wird.

In dieser Lektion werden wir erkunden, was ASCII ist, wie es funktioniert und wie JavaScript-Methoden wie `charCodeAt()` und `fromCharCode()` mit Zeichencodierung zusammenhängen. Obwohl JavaScript-Strings intern Unicode (UTF-16) verwenden, stimmen die ASCII-Werte mit den ersten 128 Unicode-Zeichen überein, weshalb ASCII-basierte Beispiele in JavaScript funktionieren.

ASCII ist ein System zur Codierung von Zeichen wie Buchstaben, Ziffern und Symbolen in numerische Werte. Jedem Zeichen wird eine bestimmte Zahl zugeordnet.

Zum Beispiel wird der Großbuchstabe `A` in ASCII durch die Zahl `65` dargestellt, während der Kleinbuchstabe `a` durch `97` dargestellt wird. Diese Codierung ermöglicht es Computern, Text zu speichern und zu verarbeiten.

Der ASCII-Standard umfasst 128 Zeichen, darunter:

- Groß- und Kleinbuchstaben des englischen Alphabets (A-Z, a-z).
- Zahlen (0-9).  
- Häufige Satzzeichen und Symbole (!, @, # und so weiter).
- Steuerzeichen (wie Zeilenumbruch und Tabulator).  

In JavaScript können Sie mit der Methode `charCodeAt()` auf den numerischen Code eines Zeichens zugreifen. Diese Methode gibt die UTF-16-Codeeinheit des Zeichens an einem bestimmten Index zurück. Für die ersten 128 Zeichen entspricht dieser Wert dem ASCII-Code.

Schauen wir uns ein Beispiel an:

:::interactive_editor

```js
let letter = "A";
console.log(letter.charCodeAt(0));  // 65
```

:::

In diesem Beispiel ist `A` das erste Zeichen des Strings, und das Aufrufen von `charCodeAt(0)` gibt seinen numerischen Code zurück (der für grundlegende lateinische Zeichen mit dem ASCII-Wert übereinstimmt), `65`.

Sie können diese Methode auch mit anderen Zeichen verwenden, um deren numerische Codewerte zu ermitteln:

:::interactive_editor

```js
let symbol = "!";
console.log(symbol.charCodeAt(0));  // 33
```

:::

Hier wird der numerische Code für das Ausrufezeichen `!` als `33` zurückgegeben (was seinem ASCII-Wert entspricht).

Während `charCodeAt()` Ihnen hilft, den numerischen Code eines Zeichens abzurufen, ermöglicht die Methode `fromCharCode()` das Gegenteil: eine UTF-16-Codeeinheit (die für grundlegende Zeichen mit ASCII übereinstimmt) in das entsprechende Zeichen umzuwandeln.

Sehen wir uns das in der Praxis an:

:::interactive_editor

```js
let char = String.fromCharCode(65);
console.log(char);  //  A
```

:::

In diesem Beispiel wandelt `fromCharCode(65)` den numerischen Code `65` (der dem ASCII-Wert für `A` entspricht) zurück in das Zeichen `A` um.

Ein weiteres Beispiel wäre, die Zahl `97` in den entsprechenden Kleinbuchstaben umzuwandeln:

:::interactive_editor

```js
let char = String.fromCharCode(97);
console.log(char);  // a
```

:::

Diese Methoden sind besonders nützlich, wenn Sie Zeichen basierend auf ihren numerischen Codewerten manipulieren oder vergleichen müssen.

Zum Beispiel könnten Sie `charCodeAt()` verwenden, um zu prüfen, ob ein Zeichen ein Großbuchstabe, Kleinbuchstabe oder eine Ziffer ist, indem Sie seinen ASCII-Wert vergleichen.

Andererseits kann `fromCharCode()` verwendet werden, um Zeichen dynamisch aus ihren ASCII-Codes zu generieren.
