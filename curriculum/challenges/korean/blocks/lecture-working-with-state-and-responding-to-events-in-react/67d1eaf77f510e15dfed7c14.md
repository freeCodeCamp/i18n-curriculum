# --description--

상태는 React와 다른 프런트엔드 프레임워크에서 가장 중요한 기본 개념 중 하나입니다. 컴포넌트의 두뇌와 같아서 시간이 지나면서 변할 수 있는 정보를 담고 컴포넌트가 어떻게 동작하고 보일지 제어합니다.

상태가 무엇인지, 그리고 `useState` 훅이 상태를 어떻게 다루게 해주는지 살펴보겠습니다.

상태는 React 컴포넌트 내에서 사용자 입력 값, API에서 가져온 데이터, 할 일 목록의 항목 같은 동적인 데이터를 나타냅니다.

상태가 변경될 때마다 React는 페이지를 새로 고치지 않고도 컴포넌트를 다시 렌더링하여 사용자 인터페이스에 그 변화를 반영합니다. 이러한 반응성 덕분에 앱이 인터랙티브해집니다.

`useState` 훅은 함수형 컴포넌트에서 상태 변수를 선언할 수 있게 해주는 함수입니다.

훅이 도입되기 전에는 상태를 클래스 컴포넌트에서만 사용할 수 있었습니다. 하지만 React 16.8부터 훅이 도입되면서 `useState` 훅을 사용해 함수형 컴포넌트에서도 상태를 사용할 수 있게 되었습니다.

`useState` 훅을 사용하려면 React에서 가져와야 합니다:

```js
import { useState } from "react";
```

React 자체를 가져와서 `useState` 훅을 프로퍼티로 접근할 수도 있습니다:

```js
import React from "react";
```

`useState`를 가져올 때 상태 변수를 선언하는 방법은 다음과 같습니다:

```js
const [stateVariable, setStateFunction] = useState(initialValue);
```

React를 가져올 때 상태 변수를 선언하는 방법은 다음과 같습니다:

```js
const [stateVariable, setStateFunction] = React.useState(initialValue);
```

상태 변수에는 다음이 포함됩니다:

- `stateVariable`는 현재 상태 값을 담고 있습니다
- `setStateFunction` (설정 함수)은 상태 변수를 업데이트합니다
- `initialValue`는 초기 상태를 설정합니다

React 컴포넌트의 상태는 비공개이며 각 컴포넌트 인스턴스에 고유하게 격리되어 있다는 점에 유의하세요. 즉, 같은 컴포넌트를 두 번 렌더링해도 한 컴포넌트의 상태가 다른 컴포넌트에 영향을 주지 않습니다. 또한 컴포넌트 간에 상태를 공유하려면 상태를 공통 부모 컴포넌트로 끌어올려 프로프스로 전달해야 합니다.

또한 훅은 컴포넌트 최상위, `return` 키워드 바로 전에 호출되어야 하며, 그래야 렌더링 간 상태와 효과가 일관성을 유지합니다. 따라서 상태를 루프, 조건문, 중첩 함수 안에서 사용할 수 없습니다.

다음은 `useState` 훅으로 `Counter` 컴포넌트에서 상태를 관리하는 예시입니다:

```jsx
// Importing the useState hook
import { useState } from "react";

function Counter() {
  const initialValue = 0;

  // The state variable and setter function
  const [count, setCount] = useState(initialValue);

  return (
    <div>
      {/* Display current state value */}
      <h2>{count}</h2>

      <button onClick={() => setCount(count - 1)}>Decrement</button>
      <button onClick={() => setCount(count + 1)}>Increment</button>
    </div>
  );
}

export default Counter;
```

위 코드에서 React에서 `useState` 훅을 가져왔습니다. `Counter` 컴포넌트에서 `count`는 현재 상태를 나타내고 `setCount`는 상태를 업데이트하는 설정 함수입니다. 현재 상태 값은 `0`입니다. `return` 문장은 count와 두 개의 버튼을 포함하며, 버튼을 눌러 count를 `1`만큼 감소 또는 증가시킵니다.

여러 상태를 관리하려면 `useState` 훅을 여러 번 호출할 수 있습니다. 이는 서로 관련 없는 상태 변수가 있을 때 특히 중요합니다:

```jsx
function UserProfile() {
  const [isOnline, setIsOnline] = useState(false);
  const [notifications, setNotifications] = useState(0);

  // The rest of the component logic
}
```

또한 여러 개의 상태가 각각 따로 업데이트되는 경우, 예를 들어 폼 필드처럼 `useState` 훅을 여러 번 호출할 수 있습니다:

```jsx
function SignUpForm() {
  const [name, setName] = useState("");
  const [username, setUsername] = useState("");
  const [email, setEmail] = useState("");

  // The rest of the component logic
}
```

하지만 이 경우 모두 같은 폼의 일부이므로 상태를 합치는 것이 더 좋습니다:

```jsx
function SignUpForm() {
  const [formData, setFormData] = useState({
    name: "",
    username: "",
    email: "",
  });

  // The rest of the component logic
}
```

이것이 상태가 무엇이며 `useState` 훅을 어떻게 사용할 수 있는지에 대한 설명입니다.
