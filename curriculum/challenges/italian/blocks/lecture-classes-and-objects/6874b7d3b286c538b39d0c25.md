# --description--

I metodi speciali in Python, noti anche come "metodi magici" o "metodi dunder", sono metodi Python speciali che iniziano e finiscono con doppi underscore (`__`). La parola "dunder" deriva proprio da double underscores (**d** per double, **under** per underscores).

Probabilmente hai già usato i metodi speciali senza saperlo. Ogni volta che scrivi qualcosa come `3 + 4`, Python esegue silenziosamente `3.__add__(4)` sotto il cofano. Quello è un metodo speciale in azione. Quindi, anche se *puoi* chiamare direttamente i metodi speciali, raramente lo fai. Qualcosa come `3 + 4` è molto più chiaro e facile da leggere rispetto a chiamare `3.__add__(4)` da solo.

A parte `__add__`, `__init__()` è un altro metodo speciale che vedrai e userai di più, perché è un inizializzatore di classe. Ce ne sono anche altri come `__len__()` e `__str__()`.

Pensa ai metodi speciali come ai registi delle attività tra chi programma e l'interprete del linguaggio Python stesso.

Ricorda, non devi chiamare direttamente i metodi speciali. Python li chiama automaticamente quando succedono certe azioni. Queste operazioni includono:

- **Operazioni aritmetiche come addizione, sottrazione, moltiplicazione, divisione e altre**. In particolare, viene chiamato `__add__()` per l'addizione, `__sub__()` per la sottrazione, `__mul__()` per la moltiplicazione e `__truediv__()` per la divisione.
    
- **Operazioni sulle stringhe come concatenazione, ripetizione, formattazione e conversione in testo**. Viene chiamato `__add__()` per la concatenazione, `__mul__()` per la ripetizione, `__format__()` per la formattazione, `__str__()` e `__repr__()` per la conversione in testo, e così via.
    
- **Operazioni di confronto come uguaglianza, minore di, maggiore di e altre**. Viene chiamato `__eq__()` per i controlli di uguaglianza, `__lt__()` per minore di, `__gt__()` per maggiore di, e così via.
    
- **Operazioni di iterazione come rendere un oggetto iterabile e avanzare tra gli elementi**. Viene chiamato `__iter__()` per restituire un iteratore e `__next__()` per ottenere l'elemento successivo.
    

Normalmente, i tipi di dati Python come stringhe e numeri già sanno come aggiungere cose, fare concatenazione, confrontare per uguaglianza, essere usati nei loop e altro.

Ma quando crei la tua classe, Python non saprà come gestire queste cose automaticamente.

Qui entrano in gioco i metodi speciali — ti permettono di personalizzare il comportamento integrato di Python.

Supponiamo che tu voglia ottenere il numero di pagine negli oggetti libro creati con la classe qui sotto, o confrontarli e ottenere una stringa leggibile degli oggetti. Ecco cosa succede senza metodi speciali:

```python
class Book:
   def __init__(self, title, pages):
       self.title = title
       self.pages = pages

book1 = Book("Built Wealth Like a Boss", 420)
book2 = Book("Be Your Own Start", 420)

print(len(book1)) # TypeError: object of type 'Book' has no len()
print(str(book1)) # <__main__.Book object at 0x102ed2900>
print(book1 == book2) # False even though they have the same number of pages
```

Nell'esempio:

- `len(book1)` fallisce perché Python non sa come ottenere la lunghezza del tuo oggetto libro senza `__len__()`
    
- `str(book1)` stampa qualcosa come `<__main__.Book object at 0x102ed2900>` perché quella è la rappresentazione predefinita quando non usi `__str__()`
    
- `book1 == book2` restituisce `False` perché Python controlla solo se entrambi gli oggetti sono gli stessi in memoria, non per contenuto.
    

Ecco come puoi definire i tuoi metodi speciali `__len__()`, `__str__()` e `__eq__()` per rendere più facile lavorare con gli oggetti creati dalla classe `Book`:

```python
class Book:
   def __init__(self, title, pages):
       self.title = title
       self.pages = pages

   def __len__(self):
       return self.pages

   def __str__(self):
       return f"'{self.title}' has {self.pages} pages"

   def __eq__(self, other):
       return self.pages == other.pages
  
book1 = Book("Built Wealth Like a Boss", 420)
book2 = Book("Be Your Own Start", 420)

print(len(book1)) # 420
print(len(book2)) # 420
print(str(book1)) # 'Built Wealth Like a Boss' has 420 pages
print(str(book2)) # 'Be Your Own Start' has 420 pages
print(book1 == book2) # True
```

Un altro esempio è un carrello della spesa dove puoi fare quanto segue:

- Aggiungere elementi al carrello
    
- Rimuovere elementi dal carrello
    
- Ottenere il numero di elementi nel carrello
    
- Controllare quali elementi ci sono nel carrello
    
- Verificare se un elemento specifico è nel carrello
    
- Restituire o mostrare un elemento a un indice specifico nel carrello
    

Anche se potresti avere un metodo che aggiunge elementi al carrello e rimuove certi elementi, puoi creare metodi speciali per tutte le altre funzionalità:

- `__len__()` per ottenere la lunghezza degli elementi nel carrello
    
- `__iter__()` per iterare sugli elementi nel carrello così puoi vederli
    
- `__contains__()` per controllare se un elemento specifico è nel carrello
    
- `__getitem__()` per restituire o mostrare un elemento a un indice specifico nel carrello
    

Ecco un esempio di una classe `Cart` con questi metodi definiti dall’utente e metodi speciali:

```python
class Cart:
   def __init__(self):
       self.items = []

   def add(self, item):
       self.items.append(item)

   def remove(self, item):
       if item in self.items:
           self.items.remove(item)
       else:
           print(f'{item} is not in cart')

   def list_items(self):
       return self.items

   def __len__(self):
       return len(self.items)

   def __getitem__(self, index):
       return self.items[index]

   def __contains__(self, item):
       return item in self.items

   def __iter__(self):
       return iter(self.items)
```

Ecco come puoi usarli:

```python
cart = Cart()
cart.add('Laptop')
cart.add('Wireless mouse')
cart.add('Ergo keyboard')
cart.add('Monitor')

for item in cart:
   print(item, end=' ') # Laptop Wireless mouse Ergo keyboard Monitor

print(len(cart)) # 4
print(cart[3]) # Monitor

print('Monitor' in cart) # True
print('banana' in cart) # False

cart.remove('Ergo keyboard')

print(cart.list_items()) # ['Laptop', 'Wireless mouse', 'Monitor']

cart.remove('banana') # banana is not in cart
```

Questi sono alcuni modi in cui userai i metodi speciali in Python nel mondo reale.
