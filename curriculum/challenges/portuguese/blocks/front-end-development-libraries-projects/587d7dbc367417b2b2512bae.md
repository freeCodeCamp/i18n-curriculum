**Nota:** **React 18 tem incompatibilidades conhecidas com os testes deste projeto (veja [issue](https://github.com/freeCodeCamp/freeCodeCamp/issues/45922))**

**Objetivo:** Construir um aplicativo que seja funcionalmente semelhante a este: <a href="https://drum-machine.freecodecamp.rocks/" target="_blank" rel="noopener noreferrer nofollow">https://drum-machine.freecodecamp.rocks/</a>.

Atenda às histórias de usuário abaixo e faça com que todos os testes passem. Use quaisquer bibliotecas ou APIs de que você precisar. Dê ao projeto o seu próprio estilo pessoal.

Você pode usar qualquer mistura de HTML, JavaScript, CSS, Bootstrap, SASS, React, Redux e JQuery para completar este projeto. Você deve usar um framework de front-end (como React por exemplo) porque essa seção trata de aprender frameworks de front-end. Tecnologias adicionais não listadas acima não são recomendadas e usá-las é por sua conta e risco. Estamos buscando apoiar outros frameworks de front-end, como Angular e Vue, mas eles não são atualmente suportados. Vamos aceitar e tentar corrigir todos os relatórios de problemas que usem o conjunto de tecnologias sugeridas para esse projeto.  Happy coding!

**História de Usuário #1:** Eu devo ser capaz de ver um contêiner externo com um `id="drum-machine"` correspondente que contenha todos os outros elementos.

**História de Usuário #2:** Dentro de `#drum-machine` eu posso ver um elemento com um `id="display"` correspondente.

**User Story #3:** Dentro de `#drum-machine` posso ver 9 elementos clicáveis de drum pad, cada um com um nome de classe `drum-pad`, um id único que descreve o clipe de áudio que o drum pad estará configurado para acionar, e um texto interno que corresponde a uma das seguintes teclas do teclado: `Q`, `W`, `E`, `A`, `S`, `D`, `Z`, `X`, `C`. Os tambores DEVEM estar nesta ordem.

**User Story #4:** Dentro de cada `.drum-pad`, deve haver um elemento HTML5 `audio` que possua um atributo `src` apontando para um clipe de áudio, um nome de classe `clip` e um id correspondente ao texto interno de seu elemento pai `.drum-pad` (ex.: `id="Q"`, `id="W"`, `id="E"`, etc.).

**História de Usuário #5:** Quando eu clicar em um elemento `.drum-pad`, o clipe de áudio contido em seu elemento filho `audio` deve ser acionado.

**User Story #6:** Quando eu pressionar a tecla de disparo associada a cada `.drum-pad`, o clipe de áudio contido em seu elemento filho `audio` deve ser acionado (por exemplo, pressionar a tecla `Q` deve acionar o drum pad que contém a string `Q`, pressionar a tecla `W` deve acionar o drum pad que contém a string `W`, etc.).

**User Story #7:** Quando um `.drum-pad` é acionado, uma string descrevendo o clipe de áudio associado é exibida como o texto interno do elemento `#display` (cada string deve ser única).
