# --description--

Sehemu nyingine muhimu ya hitilafu ni stack trace. Hii inakusaidia kuelewa lini hitilafu ilitupwa katika programu yako, na ni vitendakazi gani vilivyorudiwa ambavyo vilisababisha hitilafu hiyo. Kwa mfano, ikiwa una msimbo ufuatao:

```javascript
const a = 5;
const b = 10;

function add() {
  return c;
}

function print() {
  add();
}

print();
```

Kitendakazi chetu `print()` kinapaswa kuita `add()`, ambayo inarudisha kigezo kinachoitwa `c`, ambacho kwa sasa hakijatangazwa. Hitilafu inayolingana nayo ni kama ifuatavyo:

<img src="https://cdn.statically.io/gh/TheOdinProject/curriculum/284f0cdc998be7e4751e29e8458323ad5d320303/foundations/javascript_basics/understanding_errors/imgs/01.png" width="100%" alt="Hitilafu ikionyesha stacktrace katika konsoli ya msanidi programu">

Stack trace inatuambia kwamba:

1. `c is not defined` iko katika wigo wa `add()`, ambayo imetangazwa kwenye mstari wa 5.
1. `add()` ilitumiwa na `print()`, ambayo ilitangazwa kwenye mstari wa 9.
1. `print()` yenyewe ilitumiwa kwenye mstari wa 12.

Hivyo basi stack trace inakuwezesha kufuatilia mabadiliko ya hitilafu hadi chanzo chake, ambacho hapa ni tamko la `add()`.

# --questions--

## --text--

Stack trace husaidiaje katika kurekebisha hitilafu katika msimbo wa JavaScript uliotolewa?

## --answers--  

Stack trace inaonyesha kwamba kitendakazi `add()` kinarudisha thamani ya `c` kwa usahihi.

---

Stack trace inaonyesha mstari gani katika msimbo unahitaji kuhaririwa ili kurekebisha kosa la sintaksia.

---

Stack trace hutoa njia ya kina ya wito za vitendakazi zinazosababisha hitilafu, kusaidia kubaini mahali ambapo `c` imetajwa vibaya.

---

Stack trace inaonyesha tu kwamba `print()` ilitumiwa, lakini haionyeshi maelezo kuhusu hitilafu katika `add()`.
