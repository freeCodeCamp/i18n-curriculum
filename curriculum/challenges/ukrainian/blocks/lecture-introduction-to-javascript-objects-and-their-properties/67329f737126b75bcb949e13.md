# --interactive--

У JavaScript об’єкт — це базова структура даних, яка дозволяє зберігати та організовувати пов’язані дані й функціональність.

Ви можете уявити об’єкт як контейнер, що містить різні фрагменти інформації, подібно до того, як картотечна шафа зберігає різні папки та документи.

Ці фрагменти інформації називаються властивостями, і вони складаються з імені (або ключа) та значення.

```js
const exampleObject = {
  propertyName: value,
}
```

Об’єкти надзвичайно універсальні і становлять основу JavaScript. Насправді майже все в JavaScript є об’єктом або може розглядатися як об’єкт. Це включає масиви, функції і навіть примітивні типи даних, такі як рядки та числа, коли їх використовують у певних контекстах.

Ця орієнтація JavaScript на об’єкти — одна з причин, чому ця мова така гнучка і потужна. Давайте подивимося, як можна створити об’єкт:

```js
const person = {
  name: "Alice",
  age: 30,
  city: "New York"
};
```

У цьому прикладі ми створили об’єкт з назвою `person` з трьома властивостями: `name`, `age` і `city`. Кожна властивість має ім’я та значення, розділені двокрапкою.

Тепер розглянемо, як отримати доступ до цих властивостей. Існує два основні способи доступу до властивостей об’єкта в JavaScript: точкова нотація та дужкова нотація.

Точкова нотація — це найпоширеніший і найпростіший спосіб доступу до властивостей об’єкта. Ось базовий синтаксис для точкової нотації:

```js
objectName.propertyName
```

Ось як ви б використали точкову нотацію з нашим об’єктом `person`:

:::interactive_editor

```js
const person = {
  name: "Alice",
  age: 30,
  city: "New York"
};

console.log(person.name);  // Alice
console.log(person.age);   // 30
```

:::

Точкова нотація лаконічна і легко читається, тому її зазвичай обирають, коли ви точно знаєте ім’я властивості, до якої хочете отримати доступ, і це ім’я є дійсним ідентифікатором JavaScript (тобто не починається з числа і не містить спеціальних символів або пробілів).

Дужкова нотація, навпаки, дозволяє отримати доступ до властивостей об’єкта, використовуючи рядок у квадратних дужках. Ось як ви б використали дужкову нотацію:

:::interactive_editor

```js
const person = {
  name: "Alice",
  age: 30,
  city: "New York"
};

console.log(person["name"]); // Alice
console.log(person["age"]); //  30
```

:::

Дужкова нотація є гнучкішою за точкову, оскільки дозволяє використовувати імена властивостей, які не є дійсними ідентифікаторами JavaScript. Наприклад, якщо ім’я властивості містить пробіли або починається з числа, потрібно використовувати дужкову нотацію:

:::interactive_editor

```js
const oddObject = {
  "1stProperty": "Hello",
  "property with spaces": "World"
};

console.log(oddObject["1stProperty"]);  // Hello
console.log(oddObject["property with spaces"]);  // World
```

:::

Ще одна перевага дужкової нотації — можливість використовувати змінні для динамічного доступу до властивостей:

:::interactive_editor

```js
const person = {
  name: "Alice",
  age: 30,
  city: "Wonderland"
};

let propertyName = "city";
console.log(person[propertyName]); // Wonderland
```

:::

Ця гнучкість робить дужкову нотацію особливо корисною, коли ви не знаєте точне ім’я властивості під час написання коду або коли працюєте з іменами властивостей, які надходять від користувача чи з іншого динамічного джерела.

Варто зазначити, що об’єкти в JavaScript надзвичайно потужні і універсальні. Вони можуть містити не лише прості значення, як рядки та числа, а й масиви або інші об’єкти.

Розуміння об’єктів і вміння працювати з ними є ключовим у JavaScript, оскільки вони широко використовуються по всій мові та в багатьох бібліотеках і фреймворках JavaScript.

Продовжуючи вивчати і працювати з JavaScript, ви побачите, що освоєння об’єктів відкриває світ можливостей для створення складних і потужних застосунків.
