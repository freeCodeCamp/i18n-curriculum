# --description--

JavaScript는 동적 타입 언어로 간주됩니다. 이는 변수에 실행 시간에 어떤 값이든 할당할 수 있다는 뜻입니다. 변수를 숫자로 선언해도 나중에 문자열 값을 할당하는 것을 막지 않으며, 함수 매개변수에도 어떤 값이든 전달할 수 있습니다.

동적 타입 언어의 문제는 타입 안전성이 부족해 주의하지 않으면 오류가 발생할 수 있다는 점입니다. TypeScript는 JavaScript 언어를 확장해 정적 타입을 포함시켜, 코드를 배포하기 전에 이러한 오류를 잡는 데 도움을 줍니다. 그렇다면 어떻게 작동할까요?

다음 JavaScript 함수를 생각해 보세요:

```js
const getRandomValue = (array) => {
  return array[Math.floor(Math.random() * array.length)];
}
```

우리의 `getRandomValue` 함수는 배열을 받아 그 배열에서 무작위 값을 반환합니다. 하지만 JavaScript는 타입을 검증하지 않기 때문에, 숫자를 인수로 함수에 전달하는 것을 막지 않습니다:

```js
console.log(getRandomValue(10));
```

현재 예제의 콘솔 출력은 `undefined`이 될 텐데, 배열 대신 숫자를 전달했기 때문입니다.

하지만 TypeScript에서는 배열 매개변수의 타입을 정의할 수 있습니다:

```js
const getRandomValue = (array: string[]) => {
  return array[Math.floor(Math.random() * array.length)];
}
```

이 타입 정의는 TypeScript에 배열 인수가 문자열 배열이어야 한다고 알려줍니다. 그래서 `getRandomValue`를 호출하면서 숫자를 전달하면 컴파일러 오류라는 다른 종류의 오류가 발생합니다.

브라우저나 Node.js 같은 대부분의 JavaScript 런타임 환경은 TypeScript를 직접 실행할 수 없습니다. 대신 TypeScript 코드를 일반 JavaScript로 컴파일하거나 변환해야 합니다. TypeScript 언어에 내장된 컴파일러를 사용해 이 작업을 할 수 있습니다. 컴파일러를 실행하면 TypeScript가 코드를 평가해 타입이 맞지 않는 부분, 예를 들어 배열을 기대하는 함수에 숫자를 전달하는 경우 오류를 발생시킵니다.

다음 수업들에서 이러한 타입이 어떻게 작동하는지 더 배우게 될 것입니다. 하지만 바로 이 추가적인 안전성 때문에 TypeScript는 많은 개발자와 조직에 매력적인 언어가 되었습니다.
