# --description--

Працюючи в терміналі, стандартні текстові редактори не завжди доступні, але існують програми, призначені для редагування файлів безпосередньо з терміналу.

У середовищах Linux зазвичай є текстовий редактор. Vi або Vim є найпоширенішими. Emacs і Nano можуть потребувати ручної установки, але вони доступні майже для кожного дистрибутива.

Проблема з Vim у тому, що він має значну криву навчання. Якщо ви з ним не знайомі, ви навіть можете не знати, як закрити застосунок! Emacs, можливо, трохи дружніший до користувача, але все одно має свій великий набір комбінацій клавіш.

Nano значно простіший і зручніший, особливо для новачків. З Nano можна швидше почати редагувати файл, а застосунок навіть пропонує на екрані підказки для базових комбінацій клавіш.

Однак за це доводиться платити відсутністю розширюваності. На відміну від Vim або Emacs, які можна розширювати плагінами, щоб отримати повноцінне середовище розробки, Nano досить базовий.

Давайте подивимося, що може Nano. Щоб відкрити файл, у терміналі можна ввести `nano <filename>`.

Після відкриття файлу у верхній частині екрана ви побачите версію Nano, яку ви використовуєте, та ім’я відкритого файлу.

Нижче знаходиться вміст файлу. Ви можете використовувати стрілки для навігації курсором і вставляти або видаляти текст за допомогою клавіатури. Файл прокручуватиметься за потреби.

Внизу ви побачите список доступних комбінацій клавіш, який виглядає приблизно так:

```bash
^G Help   ^O Write Out    ^F Where Is   ^K Cut      ^T Execute  ^C Location     M-U Undo    M-A Set Mark    M-] To Bracket    M-B Previous
^X Exit   ^R Read File    ^\ Replace    ^U Paste    ^J Justify  ^/ Go To Line   M-E Redo    M-6 Copy        ^B Where Was      M-F Next
```

Комбінації, що починаються з символу карету (`^`), означають, що потрібно утримувати клавішу Ctrl і натиснути вказану літеру. Комбінації, що починаються з `M-`, вимагають утримувати «мета-клавішу», яка в більшості випадків — це клавіша Alt, і натиснути літеру.

Щоб редагувати файл у Nano, потрібно знати лише кілька команд. Після внесення змін у файл можна натиснути `Ctrl + O`, щоб «Записати» (зберегти) файл. Після цього внизу з’явиться таке меню:

```bash
File Name to write : <filename>
^G Get Help                 ^T  To Files
^C Cancel                   TAB Complete
```

Натисніть Enter, щоб зберегти файл і повернутися до редагування, або `Ctrl + C`, щоб скасувати і повернутися до редагування.

Під час роботи в редакторі можна натиснути `Ctrl + X`, щоб вийти з Nano і повернутися до терміналу. Якщо при виході є незбережені зміни, внизу з’явиться таке повідомлення:

```bash
Save modified buffer (ANSWERING "No" WILL DESTROY CHANGES) ?
                            Y Yes
^C Cancel                   N No
```

Натисніть `Y`, щоб зберегти незбережені зміни, або `N`, щоб їх відхилити.

З цими знаннями ви будете краще підготовлені наступного разу, коли потрібно буде швидко відредагувати файл на віддаленому сервері, де може не бути доступу до повноцінного графічного середовища.
