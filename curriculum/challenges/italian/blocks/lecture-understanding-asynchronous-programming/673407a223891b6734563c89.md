# --description--

La Fetch API permette alle app web di effettuare richieste di rete, solitamente per recuperare o inviare dati al server. Questa API fornisce un metodo `fetch()` che puoi usare per fare queste richieste. Vediamo un esempio base di come usare `fetch`:

```js
fetch('https://api.example.com/data')
```

In questo esempio, stiamo facendo una richiesta `GET` a `https://api.example.com/data`. Questo ci restituirà dei dati che dobbiamo convertire in formato JSON e che possiamo usare ovunque vogliamo.

Per impostazione predefinita, la Fetch API usa il metodo `GET` per recuperare i dati. Questo sarà trattato nella prossima lezione, insieme ad altri metodi comuni di richiesta HTTP.

Ora, parliamo di alcuni tipi comuni di risorse che vengono recuperate dalla rete.

Nelle nostre app web, abbiamo bisogno di dati comuni come dati meteorologici, liste di professioni, nomi di paesi, codici paese o icone/immagini delle bandiere nazionali. Usando questi dati possiamo rendere la nostra app più informativa e interattiva. Grazie alla Fetch API, possiamo ottenere queste risorse dalla rete.

Le immagini sono alcune delle risorse più frequentemente recuperate. Potresti usare fetch per caricare immagini in modo statico o dinamico in base alle azioni dell’utente e mostrarle nella tua app web.

I file di testo sono un altro tipo di risorsa spesso recuperata. Questo può includere file di configurazione, file di log o anche interi documenti che vuoi mostrare nella tua pagina web.

In alcuni casi, potresti recuperare file audio o video. La Fetch API può gestire anche questi tipi di risorse, permettendoti di lavorare con una vasta gamma di tipi di dati.
