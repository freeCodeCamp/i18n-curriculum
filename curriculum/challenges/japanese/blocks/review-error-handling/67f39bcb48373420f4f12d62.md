## Pythonでよくあるエラー

- **SyntaxError**: Pythonの構文ルールに従っていないコードに対して発生するエラーです。例えば、コード`print("Hello there"`は閉じ括弧が欠けているため、メッセージ`SyntaxError: '(' was never closed`の構文エラーになります。
- **NameError**: 定義していない変数や関数にアクセスしようとするとPythonが`NameError`を発生させます。例えば、コードに`print(username)`があり、事前に`username`変数が定義されていなければ、メッセージ`NameError: name 'username' is not defined`の名前エラーになります。
- **TypeError**: 互換性のないデータ型同士で操作を行った場合にPythonがスローするエラーです。例えば、文字列と数値を足そうとすると、エラー`TypeError: can only concatenate str (not "int") to str`が発生します。
- **IndexError**: リストやタプル、文字列などのシーケンスで存在しないインデックスにアクセスすると`IndexError`が発生します。例えば、`Hello world`文字列の最後の文字のインデックスは`11`です。`greet = "hello world"; print(greet[12])`のようにアクセスすると、メッセージ`IndexError: string index out of range`のエラーになります。
- **AttributeError**: その型のオブジェクトに存在しないメソッドやプロパティを使おうとするとPythonがこのエラーを発生させます。例えば、文字列`.append()`に対して`"hello".append("!")`を呼び出すと、メッセージ`AttributeError: 'str' object has no attribute 'append'`のエラーになります。

## Pythonでの良いデバッグ手法

- **`print`関数の使用**: デバッグ中にコードのさまざまな箇所に`print()`文を挿入すると、変数の値やコードの流れを確認できます。
- **Pythonの組み込みデバッガー（`pdb`）の利用**: Pythonには`pdb`モジュールがあり、標準ライブラリの一部なのでいつでも使えます。`pdb`を使うと、`set_trace()`関数でトレースを設定し、対話的にコードをステップ実行しながら変数を調べられます。
- **IDEのデバッグツールの活用**: PycharmやVS Codeなど多くの統合開発環境やコードエディターには、ブレークポイント、ステップ実行、変数の検査などのデバッグ機能が備わっています。

## 例外処理

- **`try...except`**: 例外が発生する可能性のあるコードブロックを実行するために使います。`try`ブロックはエラーが起こるかもしれない部分で、`except`ブロックは指定した例外が発生した場合に実行されます。例を示します。

  ```py
  try:
    print(22 / 0)
  except ZeroDivisionError:
    print('You can\'t divide by zero!')
    # You can't divide by zero!
  ```

  複数の`except`ブロックを連結して、より多くの例外タイプを処理することもできます。

  ```py
  try:
    number = int(input('Enter a number: '))
    print(22 / number)
  except ZeroDivisionError:
    print('You cannot divide by zero!')
    # You cannot divide by zero! prints when you enter 0
  except ValueError:
    print('Please enter a valid number!')
    # Please enter a valid number! prints when you enter a string  
  ```

- **`else`と`finally`**: これらのブロックは`try...except`を拡張します。例外が発生しなければ`else`ブロックが実行されます。`finally`ブロックはエラーの有無にかかわらず必ず実行されます。

  ```py
  try:
    result = 100 / 4
  except ZeroDivisionError:
    print('You cannot divide by zero!') # This will not run
  else:
    print(f'Result is {result}') # Result is 25.0
  finally:
    print('Execution complete!') # Execution complete!
  ```

- **例外オブジェクト**: 例外そのものにアクセスして、より良いデバッグや直接のエラーメッセージの出力ができます。例外オブジェクトにアクセスするには`as`キーワードを使います。例を示します。

  ```py
  try:
      value = int('This will raise an error')
  except ValueError as e:
      print(f'Caught an error: {e}')
      # Caught an error: invalid literal for int() with base 10: 'This will raise an error'
  ```

- **`raise`文**: これは例外を手動でスローするために使います。特定の条件が満たされたときに例外をスローできます。例を示します。

  ```py
  def divide(a, b):
      if b == 0:
          raise ZeroDivisionError('You cannot divide by zero')
      return a / b
  ```
  
## 例外の通知
  
  `raise`文はカスタム例外を作成するときにも便利で、カスタムメッセージ付きで例外をスローできます。例を示します。

  ```py
  class InvalidCredentialsError(Exception):
      def __init__(self, message="Invalid username or password"):
          self.message = message
          super().__init__(self.message)

  def login(username, password):
      stored_username = "admin"
      stored_password = "password123"
      
      if username != stored_username or password != stored_password:
          raise InvalidCredentialsError()
      
      return f"Welcome, {username}!"
  ```  

  `login`関数を`InvalidCredentialsError`例外と一緒に使う方法の例です。

  ```py
  # failed login attempt
  try:
      message = login("user", "wrongpassword")
  except InvalidCredentialsError as e:
      print(f"Login failed: {e}")
  else:
      print(message)

  # successful login attempt
  try:
      message = login("admin", "password123")
  except InvalidCredentialsError as e:
      # This block is not executed because the login was successful
      print(f"Login failed: {e}")
  else:
      # The else block runs if the 'try' block completes without an exception
      print(message)
  ```

  `raise`文は`from`キーワードと一緒に使って例外を連鎖させ、異なるエラー間の関係を示すこともできます。

  ```py
  def parse_config(filename):
    try:
        with open(filename, 'r') as file:
            data = file.read()
            return int(data)
    except FileNotFoundError:
        raise ValueError('Configuration file is missing') from None
    except ValueError as e:
        raise ValueError('Invalid configuration format') from e

  config = parse_config('config.txt')
  ```
