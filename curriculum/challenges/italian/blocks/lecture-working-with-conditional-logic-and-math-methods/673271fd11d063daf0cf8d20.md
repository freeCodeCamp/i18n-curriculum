# --interactive--

Le istruzioni condizionali ti permettono di prendere decisioni nel tuo codice JavaScript. Consentono al tuo programma di seguire un flusso particolare in base a certe condizioni. Vediamo come funzionano `if`, `else if`, `else` e l'operatore ternario per controllare il flusso del tuo codice.

Un'istruzione `if` prende una condizione ed esegue un blocco di codice se quella condizione è truthy. I valori truthy sono tutti quei valori che risultano `true` quando valutati in un contesto booleano come un'istruzione `if`. Ecco alcuni esempi di valori truthy:

- stringhe non vuote, per esempio `hello`

- qualsiasi numero diverso da `0` e `-0`, per esempio `4`, `-5` e altri

- array

- oggetti

- il booleano `true`

D'altra parte, i valori falsy sono valori che valutano a `false` in un contesto booleano. JavaScript ha pochi valori falsy, quindi sono facili da ricordare. Ecco alcuni valori falsy:

- booleano `false`

- `0` (zero)

- `""` (stringa vuota)

- `null`

- `undefined`

- `NaN` (Not a Number)

Ora che abbiamo una comprensione di base dei valori truthy e falsy, vediamo come funzionano con le istruzioni `if`. In questo primo esempio, usiamo un paio di istruzioni `if` per verificare valori truthy e falsy:

:::interactive_editor

```js
if (null) {
  console.log("This will not run.");
}

if ("freeCodeCamp") {
  console.log("This will run.");
}
```

:::

Poiché `null` è un valore falsy, il messaggio all'interno del blocco non verrà mai registrato nella console. Ma per la seconda istruzione `if`, la stringa `freeCodeCamp` è un valore truthy e sarà considerata `true` in questo contesto booleano dell'istruzione `if`. Di conseguenza, il messaggio `This will run.` verrà registrato nella console.

Vediamo qualche altro esempio su come funzionano le istruzioni `if` con diversi operatori di confronto. Ecco un esempio di utilizzo di un'istruzione `if` per verificare se l'utente ha diritto di voto:

:::interactive_editor

```js
const age = 22;

if (age >= 18) {
 console.log("You're eligible to vote"); // You're eligible to vote
}
```

:::

In questo esempio, poiché `age` è attualmente `22`, la condizione valuterà a `true` perché `22` è maggiore o uguale a `18`. Quindi il messaggio `You're eligible to vote` verrà registrato nella console. Se modifichiamo l'esempio in modo che `age` sia ora `15`, la condizione valuterà a `false` e il messaggio non verrà registrato nella console:

:::interactive_editor

```js
const age = 15;

if (age >= 18) {
 console.log("You're eligible to vote"); // Code not running because age is less than 18
}
```

:::

Quando una condizione è `false`, puoi usare una clausola `else`:

:::interactive_editor

```js
const age = 15;

if (age >= 18) {
 console.log("You're eligible to vote");
} else {
 console.log("You're not eligible to vote"); // You're not eligible to vote
}
```

:::

In questo esempio, `15` non è maggiore o uguale a `18`, quindi la condizione sarebbe `false`. Il codice all'interno del blocco `else` verrà eseguito in questo caso.

Se vuoi verificare più condizioni, puoi usare un blocco `else if`. Questo permette al tuo programma di scegliere tra più di due percorsi.

:::interactive_editor

```js
const score = 87;

if (score >= 90) {
 console.log('You got an A'); 
} else if (score >= 80) {
 console.log('You got a B'); // You got a B
} else if (score >= 70) {
 console.log('You got a C');
} else {
 console.log('You failed! You need to study more!');
}
```

:::

Poiché `score` è attualmente `87`, il messaggio `You got a B` verrà registrato nella console.

L'operatore ternario è un modo compatto per scrivere semplici istruzioni `if/else`. Ha tre parti: una condizione, un risultato se la condizione è vera e un risultato se è falsa. Ecco la sintassi base:

```js
condition ? expressionIfTrue : expressionIfFalse;
```

Ecco un esempio che riguarda le temperature meteorologiche in gradi Celsius:

:::interactive_editor

```js
const temperature = 30;
const weather = temperature > 25 ? 'sunny' : 'cool';

console.log(`It's a ${weather} day!`);
```

:::

Se `temperature` è maggiore di `25`, il codice sopra registra `It's a sunny day!`. Se `temperature` è mai minore o uguale a `25`, registra `It's a cool day!`.

Quindi, quale dovresti usare tra un'istruzione if e un ternario? Usa un ternario quando hai a che fare con una singola condizione o espressioni singole, o quando vuoi una sintassi compatta per una logica semplice. Usa le istruzioni `if/else` quando gestisci condizioni complesse e istruzioni multiple, perché diventa illeggibile se annidi troppi ternari.
