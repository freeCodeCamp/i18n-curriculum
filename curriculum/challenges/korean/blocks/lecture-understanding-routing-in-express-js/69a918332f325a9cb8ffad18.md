# --description--

Express에서 **라우트 핸들러**(라우트 콜백 함수 또는 컨트롤러 함수라고도 함)는 클라이언트가 특정 라우트에 요청을 보낼 때 어떤 일이 일어나는지 정의하는 함수입니다.

라우트 핸들러는 들어오는 요청을 처리하고, 데이터베이스 쿼리나 계산 실행 같은 필요한 로직을 수행한 다음 클라이언트에 응답을 보냅니다.

간단한 예제로 살펴보겠습니다(사실 지금까지 계속 사용해 온 예제입니다):

```javascript
app.get("/menu", (req, res) => {
  res.send("Welcome to the menu!");
});
```

`.get()` 메서드는 Express의 라우트 메서드 중 하나입니다. 지정된 라우트에서 **GET** 요청을 기다립니다. 이 예제에서는 `/menu`에 대한 요청을 기다립니다.

함수 `(req, res) => { ... }`가 **라우트 핸들러**입니다. 클라이언트가 `/menu`에 GET 요청을 보낼 때마다 실행됩니다.

핸들러 내부에서는:

* `req` (요청)은 라우트 매개변수, 쿼리 문자열, 헤더, 본문 데이터 등 들어오는 요청에 관한 정보를 담고 있는 객체입니다.
    
* `res` (응답)은 클라이언트에 응답을 보내는 데 사용하는 객체입니다.
    

이 경우 `res.send("Welcome to the menu!")`는 클라이언트에 일반 텍스트 응답을 보냅니다.

사용자가 브라우저에서 `/menu`로 이동하거나 해당 라우트에 GET 요청을 보내면 서버가 이 핸들러를 실행하고 `Welcome to the menu!` 메시지로 응답합니다.

라우트 핸들러는 Express 애플리케이션의 핵심입니다. 웹 애플리케이션의 다양한 라우트가 요청에 어떻게 응답할지 정의할 수 있게 해줍니다. 또한 폼 데이터 처리, 데이터베이스 상호작용, 유효성 검사 같은 로직을 이 핸들러에 추가할 수 있습니다.
