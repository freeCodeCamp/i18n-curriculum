# --description--

漸進式增強是一種設計方法，確保所有使用者，不論瀏覽器或裝置，皆能存取應用程式的基本內容和功能。 

它專注於提供適用於所有人的核心體驗，同時為使用更進階瀏覽器或更佳網際網路連線的使用者提供額外的特性和改進。

漸進式增強方法遵循這些核心原則：

- 所有核心內容和基本功能應該能在所有瀏覽器上存取。
- 所有進階版面配置應透過外部 CSS 樣式表提供
- 所有進階功能都應該透過外部 JavaScript 檔案提供
- 應尊重使用者的瀏覽器偏好設定

使用漸進式增強方法使你的應用程式更具可存取性，因為所有核心內容和功能不應在不支援的環境中被區塊。 

在速度方面，漸進式增強方法也能幫助提升你的應用程式的效率、性能。 

使用較慢網際網路連線速度的使用者仍然能夠存取內容，因為瀏覽器會先下載必要的資源。

在 SEO 方面，漸進式增強也能幫助提升你的應用程式的能見度。 

搜尋引擎將能夠爬取你的應用程式內容，因為核心內容會包含在初始的 HTML 回應中。

雖然有些人批評這種方法，認為對於嚴重依賴 JavaScript 來實現其功能的應用程式來說並不總是現實的，但在建置應用程式時仍然是值得遵循的良好做法。
