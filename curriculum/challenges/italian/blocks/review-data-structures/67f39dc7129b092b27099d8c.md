## Algoritmi e notazione Big O

- **Algoritmi**: Un insieme di istruzioni non ambigue per risolvere un problema o svolgere un’attività. Gli algoritmi devono terminare in un numero finito di passaggi e ogni passaggio deve essere preciso e non ambiguo.

- **Notazione Big O**: Descrive la prestazione nel caso peggiore, o il tasso di crescita, di un algoritmo all’aumentare della dimensione dell’input. Si concentra su come l’utilizzo delle risorse cresce con la dimensione dell’input, ignorando fattori costanti e termini di ordine inferiore.

### Complessità temporali comuni

- **O(1) - Tempo costante**: L’algoritmo impiega lo stesso tempo indipendentemente dalla dimensione dell’input.

```python
def check_even_or_odd(number):
    if number % 2 == 0:
        return 'Even'
    else:
        return 'Odd'
```

- **O(log n) - Tempo logaritmico**: Il tempo aumenta lentamente all’aumentare dell’input. Comune negli algoritmi che riducono ripetutamente la dimensione del problema di una frazione (come la ricerca binaria).

- **O(n) - Tempo lineare**: Il tempo di esecuzione cresce proporzionalmente alla dimensione dell’input.

```python
for grade in grades:
    print(grade)
```

- **O(n log n) - Tempo log-lineare**: Complessità temporale comune degli algoritmi di ordinamento efficienti come Merge Sort e Quick Sort.

- **O(n²) - Tempo quadratico**: Il tempo di esecuzione cresce quadraticamente. Spesso si vede nei loop annidati.

```python
for i in range(n):
    for j in range(n):
        print("Hello, World!")
```

### Complessità spaziale

- **O(1) - Spazio costante**: L’algoritmo usa la stessa quantità di memoria indipendentemente dalla dimensione dell’input.
- **O(n) - Spazio lineare**: L’utilizzo della memoria cresce proporzionalmente alla dimensione dell’input.
- **O(n²) - Spazio quadratico**: L’utilizzo della memoria cresce quadraticamente con la dimensione dell’input.

## Tecniche di risoluzione dei problemi

- **Comprendere il problema**: Leggi più volte la descrizione del problema. Identifica l’input, l’output atteso e come trasformare l’input in output.

- **Pseudocodice**: Descrizione ad alto livello della logica dell’algoritmo indipendente dal linguaggio. Usa un linguaggio scritto comune mescolato a costrutti di programmazione come `IF`, `ELSE`, `FOR`, `WHILE`.

```md
GET original_string
SET reversed_string = ""
FOR EACH character IN original_string:
  ADD character TO THE BEGINNING OF reversed_string
DISPLAY reversed_string
```

- **Casi limite**: Input specifici e validi che si trovano ai confini di ciò che un algoritmo dovrebbe gestire. Considera e testa sempre i casi limite.

## Array

- **Array statici**: Hanno una dimensione fissa determinata all’inizializzazione. Gli elementi sono memorizzati in posizioni di memoria adiacenti. La dimensione non può essere modificata durante l’esecuzione del programma.

- **Array dinamici**: Possono crescere o ridursi automaticamente durante l’esecuzione del programma. Gestiscono il ridimensionamento tramite copia automatica in array più grandi quando necessario.

### Liste Python (array dinamici)

```python
numbers = [3, 4, 5, 6]

# Access elements
numbers[0]  # 3

# Update elements
numbers[2] = 16

# Add elements
numbers.append(7)
numbers.insert(3, 15)  # Insert at specific index

# Remove elements
numbers.pop(2)  # Remove at specific index
numbers.pop()   # Remove last element
```

### Complessità temporali per array dinamici

- **Consultare**: O(1)
- **Inserire alla fine**: O(1) in media, O(n) quando serve ridimensionare
- **Inserire in mezzo**: O(n)
- **Eliminare**: O(n) per il mezzo, O(1) per la fine

## Stack

- **Stack**: Struttura dati Last-In, First-Out (LIFO). Gli elementi si aggiungono e si rimuovono solo dalla cima.

- **Operazione push**: Aggiungere un elemento in cima allo stack. Complessità temporale: O(1).

- **Operazione pop**: Rimuovere un elemento dalla cima dello stack. Complessità temporale: O(1).

```python
# Using Python list as stack
stack = []

# Push operations
stack.append(1)
stack.append(2)
stack.append(3)

# Pop operations
top_element = stack.pop()  # Returns 3
```

## Code

- **Code**: Struttura dati First-In, First-Out (FIFO). Gli elementi si aggiungono in coda e si rimuovono dalla testa.

- **Operazione enqueue**: Aggiungere un elemento in coda alla coda. Complessità temporale: O(1).

- **Operazione dequeue**: Rimuovere un elemento dalla testa della coda. Complessità temporale: O(1).

```python
from collections import deque

# Using deque for efficient queue operations
queue = deque()

# Enqueue operations
queue.append(1)
queue.append(2)
queue.append(3)

# Dequeue operations
first_element = queue.popleft()  # Returns 1
```

## Liste concatenate

- **Liste concatenate**: Struttura dati lineare in cui ogni nodo contiene dati e un riferimento al nodo successivo. I nodi sono collegati come una catena.

### Liste concatenate singole

- **Struttura**: Ogni nodo ha dati e un riferimento al nodo successivo.
- **Attraversamento**: Si può muovere solo in avanti dalla testa alla coda.
- **Nodo testa**: Primo nodo della lista, di solito l’unico nodo accessibile direttamente.
- **Nodo coda**: Ultimo nodo della lista, punta a `None`.

### Operazioni e complessità temporali

- **Inserire all’inizio**: O(1)
- **Inserire alla fine**: O(n) - serve attraversare fino alla fine
- **Inserire in mezzo**: O(n) - serve attraversare fino alla posizione
- **Eliminare dall’inizio**: O(1)
- **Eliminare dalla fine**: O(n) - serve attraversare per trovare il nodo precedente
- **Eliminare dal mezzo**: O(n) - serve attraversare per trovare il nodo

### Liste concatenate doppie

- **Struttura**: Ogni nodo ha dati e due riferimenti: al nodo successivo e al nodo precedente.
- **Attraversamento**: Si può muovere in entrambe le direzioni.
- **Memoria**: Richiede più memoria rispetto alle liste concatenate singole a causa del riferimento extra.

## Mappe hash e insiemi

### Mappe e mappe hash

- **Mappa (tipo di dato astratto)**: Gestisce collezioni di coppie chiave-valore. Ogni chiave deve essere unica, ma i valori possono ripetersi.

- **Mappa hash**: Implementazione concreta della mappa ADT che usa la tecnica di hashing. Usa una funzione hash per generare valori hash per le chiavi, che determinano la posizione di archiviazione nell’array sottostante.

### Dizionari Python (mappe hash)

```python
# Creating dictionaries
my_dictionary = {
    "A": 1,
    "B": 2, 
    "C": 3
}

# Alternative creation
my_dictionary = dict(A=1, B=2, C=3)

# Access and modify
value = my_dictionary["A"]  # 1
my_dictionary["A"] = 4      # Update value
del my_dictionary["A"]      # Remove key-value pair

# Check membership
"C" in my_dictionary

# Get keys, values, items
my_dictionary.keys()
my_dictionary.values()
my_dictionary.items()
```

### Complessità temporali per mappe hash

- **Caso medio**: O(1) per inserire, ottenere, eliminare
- **Caso peggiore**: O(n) quando si verificano molte collisioni hash

### Insiemi

- **Insiemi**: Collezioni non ordinate di elementi unici. Non sono ammessi duplicati, non si mantiene un ordine specifico.

- **Solo elementi immutabili**: Gli insiemi possono contenere solo tipi di dati immutabili (numeri, stringhe, tuple) perché i valori hash devono rimanere costanti.

```python
# Creating sets
numbers = {1, 2, 3, 4}
empty_set = set()  # Must use set(), not {}

# Add and remove elements
numbers.add(5)
numbers.remove(4)      # Raises KeyError if not found
numbers.discard(4)     # No error if not found

# Set operations
set_a = {1, 2, 3, 4}
set_b = {2, 3, 4, 5, 6}

# Union, intersection, difference, symmetric difference
set_a.union(set_b)                    # or set_a | set_b
set_a.intersection(set_b)             # or set_a & set_b
set_a.difference(set_b)               # or set_a - set_b
set_a.symmetric_difference(set_b)     # or set_a ^ set_b

# Subset and superset checks
set_a.issubset(set_b)
set_a.issuperset(set_b)
set_a.isdisjoint(set_b)

# Membership testing
5 in numbers
```

### Complessità temporali per insiemi

- **Caso medio**: O(1) per aggiungere, rimuovere, testare appartenenza
- **Caso peggiore**: O(n) a causa di collisioni hash

## Collisioni hash

- **Collisione hash**: Si verifica quando due chiavi diverse producono lo stesso valore hash.

- **Strategie di risoluzione delle collisioni**:
  - **Concatenamento**: Ogni indice dell’array punta a una lista concatenata che memorizza tutti gli elementi con lo stesso valore hash
  - **Indirizzamento aperto**: Cerca il prossimo indice disponibile usando una sequenza predefinita

## Quando usare ogni struttura dati

- **Liste**: Quando serve un accesso ordinato e indicizzato e non si conosce la dimensione in anticipo
- **Stack**: Per operazioni LIFO (funzionalità annulla, valutazione di espressioni, backtracking)
- **Code**: Per operazioni FIFO (pianificazione attività, ricerca in ampiezza)
- **Liste concatenate**: Quando servono inserimenti/eliminazioni frequenti all’inizio, dimensione sconosciuta, nessun accesso casuale necessario
- **Mappe hash**: Per ricerche rapide chiave-valore, conteggio occorrenze, caching
- **Insiemi**: Per verificare unicità, operazioni matematiche sugli insiemi, rimuovere duplicati
