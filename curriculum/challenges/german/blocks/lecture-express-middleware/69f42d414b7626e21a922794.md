# --description--

Error-handling-Middleware in Express ist eine spezielle Art von Middleware, die verwendet wird, um Fehler zu behandeln, die während des Anfrage-Antwort-Zyklus auftreten. Sie ist unerlässlich, um unerwartete Fehler abzufangen, zu protokollieren und dem Client angemessene Antworten zu geben.

Wie Sie in einer vorherigen Lektion zum Thema Error-handling gesehen haben, hat die Express-Error-handling-Middleware vier Parameter: `err`, `req`, `res` und `next`. Diese Middleware wird aufgerufen, wenn ein Fehler an die `next()`-Funktion übergeben wird.

Hier noch einmal eine Erinnerung an die Syntax:

```js
app.use((err, req, res, next) => {
  console.error(err.message);
  res.status(500).send("Internal Server Error");
});
```

Und so sieht das in einer einfachen Express-Anwendung aus:

```js
const express = require('express')
const app = express()

// A simple route that throws an error
app.get('/', (req, res) => {
  throw new Error('Something went wrong!')
})

// Error-handling middleware
app.use((err, req, res, next) => {
  console.error(err.stack); // Logs the error stack
  res.status(500).send('Something went wrong!')
})

app.listen(3000, () => {
  console.log('Server running on http://localhost:3000')
})
```

* Route-Handler: Der Route-Handler wirft absichtlich einen Fehler, um ein Problem zu simulieren.
* Error-handling-Middleware: Diese Middleware wird mit `app.use()` registriert. Sie fängt Fehler ab, die über `next(err)` weitergegeben werden, und sendet eine HTTP-Antwort 500 an den Client mit einer allgemeinen Fehlermeldung.
* `err`: Dies ist das Fehlerobjekt, das an die Middleware übergeben wird.

Error-handling-Middleware stellt sicher, dass Benutzer eine ordnungsgemäße Antwort erhalten, wenn in Ihrer Anwendung etwas schiefgeht. Es ist wichtig, diese Middleware immer am Ende Ihres Middleware-Stacks einzufügen, um alle Fehler abzufangen.
