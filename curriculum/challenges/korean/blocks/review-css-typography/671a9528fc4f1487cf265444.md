## 타이포그래피 소개

- **정의**: 타이포그래피는 텍스트를 시각적으로 매력적이고 읽기 쉽게 만들기 위해 적절한 폰트와 형식을 선택하는 예술입니다. "타입"은 개별 문자가 디자인되고 배열되는 방식을 의미합니다.
- **서체 정의**: 서체는 문자, 숫자, 기호 세트의 전체적인 디자인과 스타일입니다. 폰트 패밀리의 청사진과 같습니다.
- **폰트 정의**: 폰트는 크기, 두께, 스타일, 너비와 같은 특정 특성을 가진 서체의 구체적인 변형입니다.
- **세리프 서체**: 이 서체는 문자 끝에 작은 선이 있는 고전적인 스타일입니다. 세리프 서체는 책과 같은 인쇄물에 주로 사용됩니다.
- **산세리프 서체**: 이 서체는 문자 끝에 작은 선이 없는 보다 현대적인 모습입니다. 산세리프 서체는 화면에서 읽기 쉽기 때문에 디지털 디자인에 자주 사용됩니다. 예로는 Helvetica, Arial, Roboto가 있습니다.
- **폰트 두께**: 문자의 굵기를 나타내며, 라이트, 레귤러, 굵게, 블랙 등이 있습니다.
- **폰트 스타일**: 문자의 기울기와 방향으로, 이탤릭과 오블리크가 있습니다.
- **폰트 너비**: 문자가 차지하는 수평 공간으로, 압축형과 확장형이 있습니다.
- **기준선**: 대부분 문자가 놓이는 가상의 선입니다.
- **대문자 높이**: 기준선에서 대문자의 꼭대기까지의 높이입니다.
- **x-높이**: 올림자와 내림자를 제외한 소문자의 평균 높이입니다.
- **올림자**: 소문자 중에서 x-높이 위로 올라가는 부분으로, 'h', 'b', 'd', 'f' 등의 꼭대기입니다.
- **내림자**: 소문자 중에서 기준선 아래로 내려가는 부분으로, 'y', 'g', 'p', 'q'의 꼬리입니다.
- **커닝**: 특정 문자 쌍 사이의 간격을 조정하여 가독성과 미관을 개선하는 방법입니다. 예를 들어, A와 V 사이의 간격을 줄이는 것입니다.
- **트래킹**: 텍스트 블록 내 모든 문자 사이의 간격을 조정하는 것으로, 문자 간 거리입니다. 텍스트의 밀도와 여백에 영향을 줍니다.
- **리딩**: 텍스트 줄 사이의 수직 간격으로, 한 줄의 기준선에서 다음 줄의 기준선까지의 거리입니다.
- **타이포그래피 모범 사례**: 디자인을 이해하기 쉽게 만들려면 명확하고 단순한 폰트를 선택하세요. 특히 웹사이트 본문에 중요합니다. 폰트가 읽기 쉬우면 사용자가 콘텐츠에 더 많이 참여합니다. 시각적 일관성을 위해 최대 두세 개의 폰트만 사용하세요. 너무 많은 폰트를 사용하면 텍스트가 읽기 어려워지고 브랜드 정체성이 약해질 수 있습니다. 또한 웹사이트 로드 시간이 길어져 사용자 경험에 영향을 줄 수 있습니다. 제목, 부제목, 문단 등 요소별로 폰트 크기를 달리하여 시각적 계층 구조를 만드세요. 예를 들어, 웹페이지의 주요 제목은 더 큰 폰트를 사용하고, 부제목은 더 작은 폰트 크기를 사용합니다. 이렇게 하면 계층 구조의 각 요소가 특정 폰트 크기를 갖게 되어 사용자가 시각적으로 구조를 쉽게 파악할 수 있습니다.

  
## CSS `font-family` 속성

- **정의**: 폰트 패밀리는 공통 디자인을 공유하는 폰트 그룹입니다. 같은 패밀리에 속한 모든 폰트는 동일한 핵심 서체를 기반으로 하지만 스타일, 두께, 너비에 변형이 있습니다. 우선순위에 따라 여러 폰트 패밀리를 쉼표로 구분해 지정할 수 있습니다. 이러한 대체 폰트는 대체 수단으로 작동합니다. 폰트 패밀리 목록 끝에는 항상 일반 폰트 패밀리를 포함해야 합니다.

```css
font-family: Arial, Lato;
```

- **일반적인 폰트 패밀리**: CSS에서 자주 사용하는 폰트 패밀리는 다음과 같습니다: serif, sans-serif, monospace, cursive, fantasy 

## 웹 안전 폰트

- **정의**: 이 폰트들은 컴퓨터나 장치에 설치되어 있을 가능성이 매우 높은 폰트 부분집합입니다. 브라우저가 폰트를 렌더링할 때 사용자의 시스템에서 폰트 파일을 찾으려 시도합니다. 폰트를 찾지 못하면 보통 기본 시스템 폰트로 대체합니다.

- **웹 안전 폰트**:
  - 산세리프 폰트는 문자 끝에 작은 "발"이나 선이 없어 화면에서 읽기 쉽기 때문에 웹 개발에 자주 사용됩니다. 웹 안전 산세리프 폰트 예시는 Arial, Verdana, Trebuchet MS입니다.
  - 반면 세리프 폰트는 문자 끝에 작은 "발"이 있어 전통적인 인쇄에 주로 사용됩니다. 웹 안전 세리프 폰트에는 Times New Roman과 Georgia가 있습니다.

## @규칙과 `@font-face` @규칙

- **정의**: @규칙은 브라우저에 지침을 제공하는 문장입니다. 미디어 쿼리, 키프레임, 폰트 페이스 등 스타일시트의 다양한 측면을 정의하는 데 사용할 수 있습니다.  
- **`@font-face`**: 이 @규칙은 폰트 파일, 형식, 두께와 스타일 같은 중요한 속성을 지정하여 사용자 정의 폰트를 정의할 수 있게 합니다. `@font-face` @규칙이 유효하려면 `src` 속성도 지정해야 합니다. 이 속성은 폰트 리소스에 대한 참조를 포함합니다.

```css
@font-face {
  font-family: "MyCustomFont"; 
  src: url("path/to/font.woff2"),
       url("path/to/font.woff"),
       url("path/to/font.otf");
}
```

- **폰트 형식**: 각 폰트 리소스에 대해 형식을 지정할 수 있습니다. 이는 선택 사항이며, 브라우저에 폰트 형식에 대한 힌트를 제공합니다. 형식을 생략하면 리소스를 다운로드한 후 형식을 감지합니다. 형식이 유효하지 않으면 리소스를 다운로드하지 않습니다. 가능한 폰트 형식에는 `collection`, `embedded-opentype`, `opentype`, `svg`, `truetype`, `woff`, `woff2`이 있습니다.

```css
@font-face {
  font-family: "MyCustomFont"; 
  src: url("path/to/font.woff2") format("woff2"),
       url("path/to/font.otf") format("opentype"),
       url("path/to/font.woff") format("woff");
}
```

- **`woff`와 `woff2`**: `woff`는 "웹 오픈 폰트 형식"을 의미합니다. Mozilla가 Type Supply, LettError 등과 협력해 개발한 널리 사용되는 폰트 형식입니다. `woff`와 `woff2`의 차이는 데이터를 압축하는 알고리즘입니다.
- **OpenType**: Microsoft와 Adobe가 개발한 확장 가능한 컴퓨터 폰트 형식으로, 사용자가 폰트의 추가 기능에 접근할 수 있게 합니다. 주요 운영체제에서 널리 사용됩니다.
- **`tech()`**: 폰트 리소스의 기술을 지정하는 데 사용합니다. 선택 사항입니다.

```css
@font-face {
  font-family: "MyCustomFont"; 
  src: url("path/to/font.woff2") format("woff2"),
       url("path/to/font.otf") format("opentype") tech(color-COLRv1),
       url("path/to/font.woff") format("woff");
}
```

## 외부 폰트 사용하기

- **정의**: 외부 폰트는 프로젝트 파일에 직접 포함되지 않은 폰트 파일입니다. 보통 별도의 서버에 호스팅됩니다. 프로젝트에 외부 폰트를 포함하려면 CSS 내에서 `link` 요소나 `@import` 문장을 사용할 수 있습니다. 
- **Google Fonts**: 웹 개발을 위해 특별히 디자인된 폰트를 많이 제공하는 Google 서비스입니다.

다음은 `link` 요소를 사용하는 예시입니다:

```html
<link rel="preconnect" href="https://fonts.googleapis.com">
<link rel="preconnect" href="https://fonts.gstatic.com" crossorigin>
<link href="https://fonts.googleapis.com/css2?family=Roboto:ital,wght@0,100;0,300;0,400;0,500;0,700;0,900;1,100;1,300;1,400;1,500;1,700;1,900&display=swap" rel="stylesheet">
```

```css
.roboto-thin {
  font-family: "Roboto", sans-serif;
  font-weight: 100;
  font-style: normal;
}
```

다음은 `@import` 문장을 사용하는 예시입니다:

```css
@import url('https://fonts.googleapis.com/css2?family=Roboto:ital,wght@0,100;0,300;0,400;0,500;0,700;0,900;1,100;1,300;1,400;1,500;1,700;1,900&display=swap');
```

- **Font Squirrel**: 프로젝트에 사용자 정의 외부 폰트를 추가할 때 사용할 수 있는 또 다른 인기 있는 리소스입니다.

## `text-shadow` 속성

- **정의**: 이 속성은 텍스트에 그림자를 적용하는 데 사용합니다. 그림자의 수평 및 수직 거리인 X 오프셋과 Y 오프셋을 지정해야 합니다. 이 값들은 필수입니다. X 오프셋과 Y 오프셋의 양수 값은 각각 그림자를 오른쪽과 아래로 이동시키고, 음수 값은 왼쪽과 위로 이동시킵니다.

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<p>Typography Shadow</p>
```

```css
  p {
    text-shadow: 3px 2px;
  }
```

:::

- **그림자 색상**: 오프셋 앞이나 뒤에 이 값을 지정해 그림자 색상을 사용자 정의할 수 있습니다. 색상을 지정하지 않으면 브라우저가 자동으로 색상을 결정하므로 보통 명시적으로 설정하는 것이 좋습니다.

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<p>Colored Shadow</p>
```

```css
p {
  text-shadow: 3px 2px #00ffc3;
}
```

:::

- **블러 반경**: 블러 반경은 선택 사항이지만 그림자를 훨씬 부드럽고 은은하게 만듭니다. 기본값은 0입니다. 값이 클수록 블러가 커져 그림자가 더 연해집니다.

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<p>Blurred Shadow</p>
```

```css
p {
  text-shadow: 3px 2px 3px #00ffc3;
}
```

:::

- **여러 텍스트 그림자 적용하기**: 텍스트는 여러 개의 그림자를 가질 수 있습니다. 그림자는 앞에서 뒤로, 첫 번째 그림자가 가장 위에 오도록 층별로 적용됩니다.

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<p>Multiple Shadows</p>
```

```css
p {
  text-shadow:
    3px 2px 3px #00ffc3,
    -3px -2px 3px #0077ff,
    5px 4px 3px #dee7e5;
}
```

:::
