# --description--

ターミナルで作業するとき、標準のテキストエディターが常に使えるわけではありませんが、ターミナル内から直接ファイルを編集するためのプログラムがあります。

Linux環境には通常テキストエディターが含まれています。ViやVimが最も一般的です。EmacsやNanoは手動でインストールする必要がある場合がありますが、ほぼすべてのディストリビューションで簡単に利用可能です。

Vimの課題は学習曲線がかなり急であることです。もしすでに使い慣れていなければ、アプリケーションの終了方法すらわからないかもしれません！Emacsは多少ユーザーフレンドリーですが、それでも独自の膨大なキーボードショートカットがあります。

Nanoは特に初心者にとってかなりシンプルで使いやすいです。Nanoを使えばファイルの編集にすぐに取りかかれて、基本的なキーボードショートカットの画面上のヘルプも表示されます。

ただし、その代わりに拡張性がありません。VimやEmacsのようにプラグインで拡張して完全な開発環境にすることはできず、Nanoはかなりシンプルな作りです。

では、Nanoが何ができるか見てみましょう。ファイルを開くには、ターミナルで`nano <filename>`と入力します。

ファイルを開くと、画面上部に実行中のNanoのバージョンと開いているファイル名が表示されます。

その下にファイルの内容が表示されます。矢印キーでキャレットを移動し、キーボードでテキストを挿入または削除できます。必要に応じてファイルはスクロールします。

そして画面下部には、次のような利用可能なキーボードショートカットの一覧が表示されます。

```bash
^G Help   ^O Write Out    ^F Where Is   ^K Cut      ^T Execute  ^C Location     M-U Undo    M-A Set Mark    M-] To Bracket    M-B Previous
^X Exit   ^R Read File    ^\ Replace    ^U Paste    ^J Justify  ^/ Go To Line   M-E Redo    M-6 Copy        ^B Where Was      M-F Next
```

キャレット（`^`）記号で始まるショートカットは、Ctrlキーを押しながら指定された文字を押す必要があることを示しています。`M-`で始まるショートカットは「メタキー」、ほとんどの環境ではAltキーを押しながら文字を押す必要があります。

Nanoでファイルを編集するには、いくつかのショートカットコマンドだけを知っていれば十分です。ファイルを変更したら、`Ctrl + O`を押して「Write Out」、つまりファイルを保存します。コマンドを入力すると、画面下部のメニューが次のように変わります。

```bash
File Name to write : <filename>
^G Get Help                 ^T  To Files
^C Cancel                   TAB Complete
```

Enterキーを押すとファイルが保存されて編集画面に戻り、`Ctrl + C`を押すとキャンセルして編集画面に戻ります。

編集中に`Ctrl + X`を押すとNanoを終了してターミナルに戻ります。保存していない変更がある場合、画面下部のメニューに次のように表示されます。

```bash
Save modified buffer (ANSWERING "No" WILL DESTROY CHANGES) ?
                            Y Yes
^C Cancel                   N No
```

`Y`を押すと未保存の変更を保存し、`N`を押すと破棄します。

この知識があれば、フルのグラフィカル環境にアクセスできないリモートサーバーでファイルを素早く編集する必要があるときに役立ちます。
