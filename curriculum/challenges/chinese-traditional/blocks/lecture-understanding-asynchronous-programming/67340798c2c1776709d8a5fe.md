# --description--

HTML `script` 元素中的 `async` 和 `defer` 屬性在 JavaScript 檔案如何在網頁中載入和執行方面扮演著關鍵角色。了解它們可以提升你網站的效率和使用者體驗。當你在 HTML 檔案中包含 `script` 時，它看起來像這樣：

```js
<script src="example.js"></script>
```

當瀏覽器找到這個 `script` 頁籤時，它會停止解析 HTML，下載該腳本，執行它，然後繼續解析 HTML。這可能會減慢你的網頁載入速度，特別是當你有大型腳本時。

這就是 `async` 和 `defer` 屬性發揮作用的地方。它們提供了更高效載入腳本的方法。

讓我們從 `async` 屬性開始：

```js
<script src="example.js" async></script>
```

透過為 `script` 頁籤添加 `async` 屬性，瀏覽器會在腳本下載時繼續解析 HTML。當腳本完全下載後，瀏覽器會暫停 HTML 解析，執行腳本，然後恢復解析 HTML。這可以大幅加快頁面載入速度。

重要的是要注意，非同步腳本會在下載完成後立即執行，這表示它們可能不會以我們期望的正確順序執行。這時候 `defer` 屬性就派上用場了。讓我們來看看 `defer` 屬性長什麼樣子：

```js
<script src="example.js" defer></script>
```

`defer` 屬性類似於 `async` 屬性。然而，延緩的腳本不會在下載後立即執行。相反地，它們會等到 HTML 解析完成後才執行。此外，延緩的腳本會依照它們在 HTML 程式碼中出現的順序執行。

簡而言之，當執行順序不重要時，使用 `async`，而當你需要確保腳本以正確順序執行時，使用 `defer`。這兩個屬性都能顯著提升頁面載入時間，因為瀏覽器可以在腳本於背景下載時繼續解析 HTML。
