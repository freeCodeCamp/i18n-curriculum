# --description--

ここに100万人のユーザーがいる忙しい電話システムの記録があります：

| RecNr | 発信者 | 着信者 |
|-------|--------|--------|
|   1   | 200007 | 100053 |
|   2   | 600183 | 500439 |
|   3   | 600863 | 701497 |
|  ...  |  ...   |  ...   |

記録$n$における発信者の電話番号と着信者の番号はそれぞれ $Caller(n) = S_{2n - 1}$ と $Called(n) = S_{2n}$ であり、${S}_{1,2,3,\ldots}$ は「遅延フィボナッチ生成器」から得られます：

$1 ≤ k ≤ 55$ のとき、$S_k = [100003 - 200003k + 300007{k}^3]\\;(\text{モジュラス}\\;1000000)$

$56 ≤ k$ のとき、$S_k = [S_{k - 24} + S_{k - 55}]\\;(\text{モジュラス}\\;1000000)$

もし $Caller(n) = Called(n)$ なら、ユーザーが誤ってダイヤルしたとみなされ、その通話は失敗します。そうでなければ通話は成功です。

記録の開始時点から、任意のユーザーのペア $X$ と $Y$ は、$X$ が $Y$ に電話をかけるかその逆であれば友達とみなします。同様に、$X$ は $Z$ の友達の友達であるとは、$X$ が $Y$ の友達で、$Y$ が $Z$ の友達である場合を指します。これがより長い連鎖でも同様です。

首相の電話番号は524287です。誤ダイヤルを除く成功した通話が何回あった後に、ユーザーの99％（首相を含む）が首相の友達、または友達の友達などになるでしょうか？
