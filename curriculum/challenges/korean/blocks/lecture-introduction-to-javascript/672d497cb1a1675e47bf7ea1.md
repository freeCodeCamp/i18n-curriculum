# --interactive--

JavaScript에서 변수는 프로그램 전반에 걸쳐 접근하고 수정할 수 있는 데이터를 저장하는 컨테이너 역할을 합니다.

변수를 값들을 담는 상자로 생각할 수 있습니다. 변수를 사용하면 숫자나 텍스트 같은 것들을 추적하고 프로그램에서 필요할 때마다 이 값들을 참조할 수 있습니다.

JavaScript에서 변수를 선언하는 한 가지 방법은 `let` 키워드를 사용하는 것입니다. `let` 키워드와 다른 변수 선언 방법에 대해서는 앞으로 수업에서 더 배우게 됩니다.

다음은 `let`를 사용해 `age`라는 변수를 선언하는 예입니다:

```js
let age;
```

지금은 `age` 변수에 값이 할당되어 있지 않습니다. 만약 사용하려고 하면 값이 없다는 의미인 `undefined`를 반환합니다.

예를 들어 보겠습니다.

**참고**: `console.log()`는 코드 디버깅에 사용되는 웹 브라우저의 일부인 콘솔에 정보를 출력하는 함수입니다. `console.log()`에 대해서는 앞으로 수업에서 더 배우게 됩니다. 또한 `//` 기호는 코드에 주석을 추가하는 데 사용됩니다. 주석은 코드 실행 시 무시되는 자신이나 다른 프로그래머를 위한 메모입니다.

:::interactive_editor
 
```js
let age;
console.log(age); // undefined
```

:::

변수에 값을 할당하려면 다음과 같이 대입 연산자를 사용해야 합니다:

```js
let age = 25;
```

이제 `age` 변수를 사용하면 `25` 값을 반환합니다.

:::interactive_editor

```js
let age = 25;
console.log(age); // 25
```

:::

대입 연산자는 등호(`=`)처럼 보이지만, 동등성을 검사하지는 않습니다. 동등성 검사를 위한 올바른 연산자에 대해서는 앞으로 수업에서 배우게 됩니다.

대입 연산자는 변수에 값을 할당하는 데 사용됩니다. 변수를 값으로 초기화하는 과정을 초기화라고 합니다.

`let` 키워드를 사용해 변수를 선언하는 장점 중 하나는 값을 다시 할당할 수 있다는 점입니다. 프로그래밍에서 재할당은 이미 값이 있는 변수에 새 값을 주는 것을 의미합니다.

다음은 `age` 변수의 값을 재할당하는 예입니다.

:::interactive_editor

```js
let age = 25;
console.log(age); // 25
age = 30;
console.log(age); // 30
```

:::

이제 `age` 변수는 `30` 값을 갖습니다. `let` 키워드는 다시 필요하지 않았는데, `age` 변수가 이미 선언되어 있어서 두 번째로 선언할 필요가 없기 때문입니다.

재할당할 때는 변수 이름만 참조하면 됩니다. 재할당은 프로그램 실행 중에 변수에 저장된 값을 업데이트하고 변경할 수 있게 해주므로 유용합니다. 게임에서 점수를 업데이트하는 좋은 예가 될 수 있습니다.

변수 이름 짓기는 간단해 보일 수 있지만, 코드가 읽기 쉽고 기능적으로 작동하도록 하기 위한 규칙과 모범 사례가 있습니다.

변수 이름은 데이터가 무엇을 나타내는지 설명해야 합니다. 예를 들어 `x` 같은 이름 대신 `age`나 `points` 같은 더 설명적인 이름을 사용하면 코드를 이해하기 쉬워집니다.

```js
// Bad variable names
let x = 10;
let y = "John";

// Good variable names
let age = 10;
```

JavaScript 변수는 문자, 밑줄(`_`), 또는 달러 기호(`$`)로 시작해야 합니다. 숫자로 시작할 수 없습니다.

```js
// Valid variable names
let age;
let _score;
let $total;

// Invalid variable names
let 1stPlace; // starts with a number
```

변수 이름은 대소문자를 구분합니다. 즉, 모두 소문자인 `age`와 첫 글자가 대문자인 `Age`는 서로 다른 변수로 간주됩니다.

:::interactive_editor

```js
let age = 25;
let Age = 30;
console.log(age); // 25
console.log(Age); // 30
```

:::

그래서 일관된 이름 짓기 관례인 카멜 케이스(camelCase)를 지키는 것이 중요합니다. 카멜 케이스는 첫 단어는 모두 소문자이고, 그 다음 단어부터는 첫 글자가 대문자인 방식입니다.

다음은 카멜 케이스 이름 짓기 관례를 사용한 변수 예입니다:

```js
let thisIsCamelCase;
let anotherExampleVariable;
let freeCodeCampStudents;
```

JavaScript에는 `let`, `const`, `function`, `return` 같은 변수 이름으로 사용할 수 없는 예약어가 있습니다.

또한 변수 이름에 느낌표(`!`)나 골뱅이(`@`) 같은 특수 문자를 사용하는 것도 피해야 합니다. 변수 이름은 문자, 숫자, 밑줄, 달러 기호를 사용해 읽기 쉽게 유지하는 것이 가장 좋습니다.

이 지침을 따르면 코드가 더 깔끔하고 복잡해져도 관리하기 쉬워집니다.
