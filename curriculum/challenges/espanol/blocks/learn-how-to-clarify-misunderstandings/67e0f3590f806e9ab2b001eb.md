# --explanation--

`Sit in on` significa asistir a algo (como una clase o reunión) sin participar activamente. Por ejemplo:

`She asked if she could sit in on our design session.` – Ella quería observar la sesión pero no liderar ni hablar mucho.

`Code review` es un proceso donde los desarrolladores revisan el código de otros para encontrar errores y mejoras. Por ejemplo:

`We found a small bug during the code review.` – El equipo revisó el código y notó un problema.
