# --description--

En la lección anterior, aprendiste cómo trabajar con tipos unión. Una forma común de trabajar con tipos unión es usar un alias de tipo. 

Un alias de tipo es un nombre para un tipo. Aquí hay un ejemplo de cómo proporcionar un nombre para un tipo unión:

```ts
type ID = number | string;
```

A menudo en tus proyectos, crearás alias de tipo y los usarás en todo tu programa así:

```ts
type ID = number | string;

type User = {
  id: ID;
  name: string;
};

function getUserId(user: User): ID {
  return user.id;
}
```

Este ejemplo usa dos tipos llamados `ID` y `User`. `ID` sería un tipo unión para `number` o `string` mientras que `User` es un tipo objeto con las propiedades `id` y `name`. El `id` se especifica con el tipo `ID` mientras que el `name` tiene el tipo `string`. 

La función `getUserId` tiene un parámetro llamado `user` y su tipo es `User`. El tipo de retorno para la función `getUserId` sería `ID`. 

Los alias de tipo son una excelente manera de describir la forma de un objeto. Es común crear tipos de objetos y usarlos en arreglos así:

```ts
type User = {
  id: number | string;
  name: string;
};

const users: User[] = [
  { id: 101, name: "Alice" },
  { id: "A102", name: "Bob" },
  { id: 103, name: "Charlie" }
];
```

En este ejemplo, el arreglo `users` debe llenarse solo con objetos `User`. Si olvidas una propiedad en uno de los objetos, entonces TypeScript generará un error como este:

```ts
type User = {
  id: number | string;
  name: string;
};

const users: User[] = [
  // Property 'name' is missing in type '{ id: number; }' but required in type 'User'.
  { id: 101 },
  { id: "A102", name: "Bob" },
  { id: 103, name: "Charlie" }
];
```

Es posible que hayas notado el uso de PascalCase al nombrar los alias de tipo. Esto se debe a que, por convención, usar PascalCase ayuda a distinguir los tipos de las variables. También mejora la legibilidad. Cuando veas `User[]`, sabrás que es un arreglo de objetos del tipo `User`.
