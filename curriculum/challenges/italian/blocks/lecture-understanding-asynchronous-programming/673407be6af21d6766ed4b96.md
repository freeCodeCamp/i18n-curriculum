# --description--

Nella lezione precedente, abbiamo visto cos’è la Fetch API e come usarla. In questa lezione parleremo dei metodi HTTP `GET`, `POST`, `PUT` e `DELETE` della Fetch API.

Iniziamo con il metodo HTTP più comune, che è il metodo `GET`. Questo serve per recuperare dati da un server. Quando usi `fetch()` senza specificare un metodo, il valore predefinito è `GET`.

```js
fetch('https://api.example.com/data')
```

In questo codice, stiamo facendo una richiesta `GET` a `https://api.example.com/data`. Nota che non puoi usare direttamente questi dati, devi prima convertire la risposta in formato JSON. Solo così puoi usarla dove vuoi nel tuo progetto. Ecco un esempio di come farlo:

```js
fetch('https://api.example.com/data')
  .then(response => response.json())
  .then(data => console.log(data))
```

In questo codice, la risposta che arriva dalla Fetch API è una promise, e stiamo usando un gestore `.then` per convertire la risposta in formato JSON. In una lezione precedente, hai imparato che una `Promise` è un oggetto che rappresenta il completamento (o il fallimento) futuro di un processo asincrono e il suo valore.

Il valore di una promise non è noto quando la promise viene creata. Lo si conosce solo quando il processo asincrono è completato. Quando concatenamo i due gestori `.then` alla chiamata fetch, si tratta di un concatenamento di promise che verrà spiegato nella prossima lezione.

Finora abbiamo recuperato risorse da un server. Ma sapevi che possiamo anche inviare dati al server? Il metodo `POST` serve per inviare dati a un server per creare una risorsa. Ecco un esempio di metodo `POST` usato per creare dati nel server:

```js
fetch('https://api.example.com/users', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({
    name: 'John Doe',
    email: 'john@example.com'
  })
})
```

In questo esempio, stiamo inviando una richiesta `POST` per creare un nuovo utente. Specifichiamo il metodo come `POST`, impostiamo le intestazioni appropriate e includiamo un `body` con i dati che vogliamo inviare. Il corpo deve essere una stringa, quindi usiamo `JSON.stringify()` per convertire il nostro oggetto in una stringa JSON.

Il metodo `PUT` serve per aggiornare risorse esistenti di un server. Ecco un esempio:

```js
fetch('https://api.example.com/users/45', {
  method: 'PUT',
  headers: {
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({
    name: 'John Smith',
    email: 'john@example.com'
  })
})
```

In questo esempio, guarda attentamente l’URL, puoi vedere un `45` alla fine. Questo viene tipicamente usato come ID univoco per identificare i dati che vogliamo aggiornare. Nel codice abbiamo usato il metodo `PUT` e abbiamo anche specificato i dati come `body` che saranno usati per aggiornare i dati identificati.

Il metodo `DELETE` serve per eliminare una risorsa dal server. Ecco un esempio:

```js
fetch('https://api.example.com/users/45', {
  method: 'DELETE',
})
```

In questo codice, includiamo il metodo `DELETE` e un ID alla fine dell’URL per identificare i dati da eliminare.
