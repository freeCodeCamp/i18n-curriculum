# --description--

В Express ви можете створювати **ланцюжкові обробники маршрутів** за допомогою методу `app.route()`. Це корисно, коли потрібно обробляти кілька HTTP-методів, таких як GET, POST або PUT, для одного й того самого шляху маршруту без повторення цього шляху щоразу.

Замість того, щоб писати окремі визначення маршрутів, як `app.get("/user")`, `app.post("/user")` і `app.put("/user")`, `app.route()` дозволяє згрупувати їх разом у одному організованому блоці коду.

Метод `app.route()` виступає як скорочення для визначення кількох обробників для однієї кінцевої точки. Це особливо корисно в RESTful API, де один ресурс (наприклад, `/user`) часто підтримує кілька операцій.

Ось приклад, який імітує RESTful API для керування користувачами:

```javascript
const express = require("express");
const app = express();
const port = 3000;

app.use(express.json()); // Parse JSON request bodies

// Use app.route() to group handlers for the same path
app
  .route("/user")
  .get((req, res) => {
    res.send("Fetching user data");
  })
  .post((req, res) => {
    const { name } = req.body;
    res.send(`Creating a user with name: ${name}`);
  })
  .put((req, res) => {
    const { name } = req.body;
    res.send(`Updating the user to: ${name}`);
  })
  .delete((req, res) => {
    res.send("Deleting the user");
  });

app.listen(port, () => {
  console.log(`Server running at http://localhost:${port}`);
});
```

У наведеному вище прикладі `app.route("/user")` визначає один маршрут для шляху `/user`. Потім ми ланцюжком додаємо кілька HTTP-методів: `.get()`, `.post()`, `.put()` і `.delete()`, щоб обробляти різні типи запитів для того самого маршруту.

Ось розбір різних маршрутів і як вони можуть працювати у простому RESTful застосунку:

* GET `/user`: отримує та повертає дані користувача.
    
* POST `/user`: створює нового користувача з даними у тілі запиту.
    
* PUT `/user`: оновлює існуючого користувача з даними у тілі запиту.
    
* DELETE `/user`: видаляє користувача.
    

Використання `app.route()` допомагає тримати ваш код організованим при обробці кількох HTTP-методів для одного шляху. Замість повторення маршруту щоразу, ви групуєте пов’язані обробники в одному місці.

Це зменшує надлишковість і робить код легшим для читання та підтримки. Коли всі методи для маршруту визначені в одному блоці, набагато зрозуміліше, як поводиться ця кінцева точка.

Коли ваш застосунок зростає і маршрути стають складнішими, `app.route()` забезпечує просту структуру, яка тримає пов’язану логіку разом. Це особливо корисно при проєктуванні RESTful API, де один ресурс часто підтримує кілька операцій, таких як GET, POST, PUT і DELETE.
