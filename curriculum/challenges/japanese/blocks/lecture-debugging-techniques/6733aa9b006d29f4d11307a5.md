# --description--

JavaScriptでプログラミングをしていると、必ずエラーメッセージに遭遇します。よくあるエラーメッセージを理解することで、より効果的にデバッグでき、より強い開発者に成長できます。よくあるエラーメッセージの種類は`SyntaxError`、`ReferenceError`、`TypeError`、`RangeError`の4つです。

`SyntaxError`は、コード内で誤って書いてしまったときに発生します。例えば、括弧やブラケットの閉じ忘れなどです。これは文法ミスのようなものだと考えてください。配列を作成するときに開発者がよく犯すミスの例を示します。

```js
const arr = ["Beau", "Quincy" "Tom"]
```

各配列要素はカンマで区切る必要があります。そうしないとエラーメッセージが発生します。

2つ目によくあるJavaScriptのエラーは`ReferenceError`です。`ReferenceError`にはいくつかの種類があり、発生する原因もさまざまです。最初のタイプの`ReferenceError`は未定義の変数です。

```js
console.log(price);
```

この例では、`price` 変数をコンソールに出力しようとしていますが、まだ定義されていません。これにより `ReferenceError` が発生します。

`ReferenceError`のもう一つの例は、`let`や`const`で宣言された変数に、定義される前にアクセスしようとすることです。

```js
console.log(b);
const b = 50;
```

上の例は`Cannot access 'b' before initialization`エラーになります。

3つ目によくあるエラーは`TypeError`です。これは間違った型に対して操作をしようとしたときに発生します。オブジェクトに対して`map`メソッドを使おうとした例を示します。

```js
const developerObj = {
  name: "Jessica",
  country: "USA",
  isEmployed: true
};

developerObj.map()
```

この例は`developerObj.map is not a function`エラーになります。なぜなら`map`メソッドは配列に使うものであって、オブジェクトには使えないからです。

最後に見ていくよくあるエラーは`RangeError`です。

`RangeError` は、コードが JavaScript が扱える範囲外の値を使おうとしたときに発生します。ここでは配列の length に無効な値を代入する例を示します。

```js
const arr = [];
arr.length = -1; 
```

配列の`length`は0以上の整数でなければならないため、`-1`を設定すると`RangeError`が発生します。

JavaScriptでプログラミングを続けるうえで、これらの異なるタイプのエラーがなぜ起きるのかを意識しておくとよいでしょう。
