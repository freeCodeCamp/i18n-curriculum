# --description--

SQLでデータを挿入して表示するために使える基本的なコマンドを見ていきましょう。

まず、データを表示するには、データを挿入する必要があります。なので、挿入から始めます。

このレッスンで扱う例は、犬の記録を保存するこのテーブルに基づいています：

```sql
CREATE TABLE dogs(
  id SERIAL,
  name VARCHAR(100),
  age INTEGER
);
```

このテーブルは、さまざまな犬の名前と年齢を保存します。

PostgreSQLを使っていると仮定すると、`SERIAL` データ型を`id` に使えます。`id` は`SERIAL` を使って作成されているため、その値は`INTEGER` で、`1` から始まり、新しいレコードが挿入されると自動的に増加します。つまり、レコードを挿入するときに値を渡す必要はありません。

`dogs` 文を使って`INSERT INTO` テーブルにレコードを挿入するところから始めましょう。これにはいくつか方法があります。最初の方法は、1行だけ挿入し、挿入する値の列を指定することです。

`INSERT INTO` と書き、その後にテーブル名（この場合は`dogs`）、丸括弧内に列名、次に`VALUES`、そして対応する値を丸括弧内に書きます。

これは例です。`name` が`'Gino'` で、`age` が`3` の犬のレコードを挿入しています。`id` は自動的に代入されることを覚えておいてください：

```sql
INSERT INTO dogs (name, age)
VALUES ('Gino', 3);
```

これは通常最も安全な方法です。列を明示的に指定するため、値がその列に順番に代入されます。

`'Gino'` は`VARCHAR` 型なので、シングルクォートで囲む必要があることに注意してください。

もう一つの方法は、列を指定せずにレコードを挿入することです。例えば：

```sql
INSERT INTO dogs
VALUES ('Gino', 3);
```

この構文は有効なSQLですが、エラーが起きやすいです。

値は列の順番に基づいて代入されるため、PostgreSQLは`'Gino'` を整数を期待する`id` 列に挿入しようとします。これにより、`invalid input syntax for type integer` のようなエラーが発生します。

このため、データを挿入するときは列名を明示的に指定する方が安全です。

また、複数のレコードを同じSQLコマンドで挿入することもできます。カンマで区切ってください。

ここでは、`'Gino'` と`'Nora'` の2つの犬のレコードを挿入しています。

```sql
INSERT INTO dogs (name, age)
VALUES
  ('Gino', 3),
  ('Nora', 2);
```

レコードを挿入できるように、データベースから情報をクエリすることもできます。データベースの文脈では、クエリはデータのリクエストを意味します。

SQLでは、`SELECT` 文が1つまたは複数のテーブルからデータをクエリするために使われます。必要な情報を正確に取得できるようにクエリをカスタマイズできます。

`dogs` テーブルのすべての情報をクエリしたい場合は、`SELECT` と書き、その後にアスタリスク（`*`）、次に`FROM` とテーブル名`dogs` を書くだけです：

```sql
SELECT *
FROM dogs;
```

このコマンドでは、アスタリスクは「すべての列」を表すワイルドカード文字です。

テーブル内のすべてのレコードのすべてのデータが取得されます：

```sql
 id | name | age 
----+------+-----
  1 | Gino |   3
  2 | Nora |   2
```

特定の列だけをクエリしたい場合は、`SELECT` の直後にカンマで区切って列名を書きます。この例では、`name` と`age` 列を`dogs` テーブルからクエリしています：

```sql
SELECT name, age
FROM dogs;
```

結果には`name` と`age` 列だけが含まれます：

```sql
 name | age 
------+-----
 Gino |   3
 Nora |   2
```

時には、特定の条件に基づいてデータをクエリする必要があります。例えば、3歳未満の犬をすべて取得したい場合は、`WHERE` キーワードと「より小さい」(`<`) 比較演算子を使います：

```sql
SELECT *
FROM dogs
WHERE age < 3;
```

結果には年齢が3未満の犬だけが含まれます：

```sql
 id | name | age 
----+------+-----
  2 | Nora |   2
```

`age` が`'Gino'` のものを探したいだけなら、等しい(`=`) 比較演算子を使えます：

```sql
SELECT age
FROM dogs
WHERE name = 'Gino';
```

結果は以下の通りです：

```sql
 age 
-----
  3
```

これらはSQLでデータを挿入して表示する一般的で簡単な方法ですが、他にもさまざまな選択肢があり、その中にはより高度なものもあります。今後のレッスンで学んでいきましょう。
