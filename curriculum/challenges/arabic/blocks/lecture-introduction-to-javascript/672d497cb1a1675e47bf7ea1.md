# --interactive--

في JavaScript، تعمل المتغيرات كحاويات لتخزين البيانات التي يمكنك الوصول إليها وتعديلها طوال برنامجك.

يمكنك التفكير في المتغيرات كصناديق تحمل القيم. باستخدام المتغيرات، يمكنك تتبع أشياء مثل الأرقام أو النصوص والرجوع إلى هذه القيم متى احتجت إليها في برنامجك.

إحدى طرق إعلان متغير في JavaScript هي استخدام الكلمة المفتاحية `let`. ستتعلم المزيد عن الكلمة المفتاحية `let` وكذلك طرق أخرى لإعلان المتغيرات في الدروس القادمة.

إليك مثالًا على استخدام `let` لإعلان متغير يسمى `age`:

```js
let age;
```

الآن، المتغير `age` لا يحتوي على قيمة مُعيَّنة له. إذا حاولت استخدامه، سيُرجع `undefined`، مما يعني أنه لا يحتوي على قيمة.

إليك مثالًا.

**ملاحظة**: الدالة `console.log()` تُخرج معلومات إلى وحدة التحكم، وهي جزء من متصفح الويب تُستخدم في تصحيح الأخطاء. ستتعلم المزيد عن `console.log()` في الدروس القادمة. أيضًا، تُستخدم الرموز `//` لإضافة تعليقات في شفرتك. التعليقات هي ملاحظات لنفسك أو للمبرمجين الآخرين تُتجاهل عند تنفيذ الشفرة.

:::interactive_editor
 
```js
let age;
console.log(age); // undefined
```

:::

لتعيين قيمة لمتغير، ستحتاج إلى استخدام معامل التعيين بهذا الشكل:

```js
let age = 25;
```

الآن عند استخدام المتغير `age`، سيُرجع القيمة `25`.

:::interactive_editor

```js
let age = 25;
console.log(age); // 25
```

:::

معامل التعيين يشبه علامة يساوي (`=`) لكنه لا يتحقق من المساواة. ستتعلم عن المعاملات الصحيحة للتحقق من المساواة في الدروس القادمة.

يُستخدم معامل التعيين لتعيين قيمة لمتغير. تُعرف هذه العملية بتهيئة المتغير.

أحد مزايا استخدام الكلمة المفتاحية `let` لإعلان المتغيرات هو إمكانية إعادة تعيين القيم لها. في البرمجة، إعادة التعيين تعني إعطاء قيمة جديدة لمتغير لديه قيمة بالفعل.

إليك مثالًا على إعادة تعيين القيمة للمتغير `age`.

:::interactive_editor

```js
let age = 25;
console.log(age); // 25
age = 30;
console.log(age); // 30
```

:::

الآن المتغير `age` يحمل القيمة `30`. لاحظ أن الكلمة المفتاحية `let` لم تكن ضرورية مرة أخرى لأن المتغير `age` مُعلن بالفعل، فلا حاجة لإعلانه مرة ثانية.

عند استخدام إعادة التعيين، تحتاج فقط إلى الإشارة إلى اسم المتغير. إعادة التعيين مفيدة لأنها تسمح لك بتحديث وتغيير القيمة المخزنة في المتغير أثناء تشغيل برنامجك. مثال جيد على ذلك هو تحديث النقاط في لعبة.

قد يبدو تسمية المتغيرات أمرًا بسيطًا، لكن هناك بعض القواعد وأفضل الممارسات لضمان أن شفرتك قابلة للقراءة والوظيفية.

يجب أن تصف أسماء متغيراتك ما تمثله البيانات. على سبيل المثال، بدلاً من استخدام اسم مثل `x`، يجعل اسم أكثر وصفًا مثل `age` أو `points` شفرتك أسهل للفهم.

```js
// Bad variable names
let x = 10;
let y = "John";

// Good variable names
let age = 10;
```

يجب أن تبدأ أسماء المتغيرات في JavaScript بحرف أو شرطة سفلية (`_`) أو علامة دولار (`$`). لا يمكن أن تبدأ برقم.

```js
// Valid variable names
let age;
let _score;
let $total;

// Invalid variable names
let 1stPlace; // starts with a number
```

أسماء المتغيرات حساسة لحالة الأحرف، مما يعني أن الكلمة `age` بأحرف صغيرة كلها والكلمة `Age` بحرف `A` كبير تُعتبر متغيرات مختلفة.

:::interactive_editor

```js
let age = 25;
let Age = 30;
console.log(age); // 25
console.log(Age); // 30
```

:::

لهذا من المهم الالتزام باصطلاح تسمية ثابت مثل تنسيق سنام الجمل (camelCase). تنسيق سنام الجمل يعني أن الكلمة الأولى كلها أحرف صغيرة وكل كلمة تالية تبدأ بحرف كبير.

إليك مثالًا على استخدام اصطلاح تسمية تنسيق سنام الجمل لمتغير:

```js
let thisIsCamelCase;
let anotherExampleVariable;
let freeCodeCampStudents;
```

هناك كلمات مفتاحية معينة في JavaScript لا يمكنك استخدامها كأسماء متغيرات، مثل `let`، `const`، `function`، أو `return`، لأنها محجوزة للغة نفسها.

يجب أيضًا تجنب استخدام أحرف خاصة مثل علامات التعجب (`!`) أو الرموز @ (`@`) في أسماء المتغيرات. من الأفضل الحفاظ على قابلية قراءة أسماء المتغيرات باستخدام الحروف والأرقام والشرطات السفلية أو علامات الدولار.

باتباع هذه الإرشادات، ستكون شفرتك أنظف وأسهل في الإدارة مع زيادة تعقيدها.
