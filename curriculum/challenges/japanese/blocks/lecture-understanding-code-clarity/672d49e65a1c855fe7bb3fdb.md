# --description--

プログラミングにおけるコメントは、コードの追加の説明を提供したり、自分や他の人のためにメモを残したりするために使われます。

コメントは、コードが実行される際にJavaScriptエンジンによって無視されるテキストの行やブロックです。これはコードを読む人、つまり自分自身や他の誰かのためだけに存在します。

JavaScriptでは、コードにコメントを追加する方法が2つあります：一行コメントと複数行コメントです。

一行コメントは、スラッシュ2つ（`//`）を使って作成します。例を示します：

```js
// I am a single line comment in JavaScript
```

このタイプのコメントは、簡単な説明や補足に最適です。

freeCodeCampのカリキュラムのプロジェクトファイルからの実例を見てみましょう：

```js
// This is to allow English to build without having to download the i18n files.
// It fails when trying to resolve the i18n-curriculum path if they don't exist.
const curriculumLocale = process.env.CURRICULUM_LOCALE ?? 'english';
const I18N_CURRICULUM_DIR = path.resolve(
  __dirname,
  curriculumLocale === 'english' ? '.' : 'i18n-curriculum/curriculum'
);
```

コードが実際に何をしているか理解しようと心配しないでください。これはこれまでに学んだ内容よりも高度です。代わりに、開発者が残したコメントに注目してください。このコメントは、このコードが存在する理由について重要な背景を提供しています。

このようなコメントは、チームで作業する際に2つの理由で重要です：

1. プロジェクトに関わる他の開発者がこのコードの目的を理解できること。

1. チームに相談せずに不必要な変更や削除を防ぎ、バグや問題の発生を防ぐこと。

もう一つのコメントの種類は複数行コメントです。基本的な構文は以下の通りです：

```js
/*
 I am a multiline comment.
 This is helpful for longer explanations.
*/
```

複数行コメントは、コード内で長い説明や解説、メモを書く必要がある場合に便利です。

freeCodeCampのカリキュラムのプロジェクトファイルをもう一度見て、複数行コメントが実際にどのように使われているか確認しましょう。

```js
/* Since there can be more than one way to complete a certification (using the
legacy curriculum or the new one, for instance), we need a certification
field to track which certification this belongs to. */
const dupeCertifications = [
  {
    certification: 'responsive-web-design',
    dupe: '2022/responsive-web-design'
  }
];
const hasDupe = dupeCertifications.find(
  cert => cert.dupe === meta.superBlock
);
```

前回と同様に、JavaScriptのコードはまだ教わっていない概念を使っているので無視してください。代わりに、開発者が残したコメントに注目してください。

チームの開発者や新しい貢献者が、このコードがなぜここにあるのか理解し、プロジェクトのこのエリアで作業する前に全体の背景を把握できます。

コメントはプログラミングで役立ちますが、コメントを入れすぎないことも重要です。コードが明快で自明な場合は、すべての行にコメントを付ける必要はありません。

明らかなことを説明するためにコメントを使った例を示します：

```js
// This code uses the const keyword to create a new variable called price.
// We are assigning the number 10 to the price variable.
const price = 10;
```

この場合、コードが自明なのでコメントを追加する必要はありません。目的は読みやすさを高めることであり、不必要な説明でコードを散らかすことではありません。

学習中に個人のプロジェクトにコメントを追加するのは問題ありません。しかし、他の開発者と一緒に実際のプロジェクトで作業を始めたら、自明なコードにコメントを使わないことが重要です。

また、混乱を招く複雑すぎるコードや、書き方が悪いコードを説明するためにコメントを使うのも避けてください。そのような場合は、コードをリファクタリング（書き換え）して、他の開発者が何が起きているか理解しやすくするのが最善です。

コメントはコードを文書化し、理解しやすくするための強力なツールです。コメントは、背景情報を提供したり、自分や他の人のためにメモを残したりするために使いましょう。
