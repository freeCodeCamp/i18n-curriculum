# --description--

HTMLは非常に寛容な言語です。終了タグを忘れるなどのミスをしても、要素はレンダリングされます。

例えば、終了タグがない`h2`要素があるとします。

```html
<h1>Article Topic</h1>
<h2>Subheading 1 </h2>
<h2>Subheading 2 </h2>

<!-- This h2 does not have a closing tag -->
<h2>Subheading 3
```

終了タグがない`h2`でも問題なくレンダリングされます。これはブラウザが一般的なエラーを処理し、作者の意図にできるだけ近い形でHTMLをレンダリングしようとするパースアルゴリズムを使っているためです。

しかし、これが時には裏目に出ることもあります。コード内の既存の見出し`h2`タグの下にいくつかの段落を追加してみましょう。

```html
<h1>Article Topic</h1>
<p>Lorem ipsum dolor sit amet consectetur adipisicing elit. Maiores, nisi.</p>

<h2>Subheading 1 </h2>
<p>Lorem, ipsum dolor sit amet consectetur adipisicing elit. At, doloremque.</p>

<h2>Subheading 2 </h2>
<p>Lorem ipsum dolor sit amet consectetur adipisicing elit. Unde, placeat.</p>

<!-- This h2 does not have a closing tag -->
<h2>Subheading 3
<p>Lorem ipsum dolor, sit amet consectetur adipisicing elit. Tempore, illum.</p>
```

その結果、終了`h2`タグがない`h2`の下の段落要素が見出し2としてレンダリングされてしまいます。これがHTMLバリデーターが必要な理由です。

HTMLバリデーターは、HTMLコードが標準のHTML仕様に準拠しているかどうかをチェックするツールです。HTMLコードのエラーや警告を特定し、ウェブページが正しく構造化され、ウェブ標準に準拠していることを確認するのに役立ちます。

HTMLバリデーターを使うことは、あなた自身や将来のコードレビューだけでなく、チームメイトやオープンソースの貢献者など、コードを扱う他の人にもメリットがあります。

利用できるHTMLバリデーターはいくつかありますが、最も広く受け入れられているのは`w3.org`マークアップバリデーションサービスです。

サイト[`validator.w3.org`](https://validator.w3.org/)にアクセスすると、`Validate by Direct Input`ボタンをクリックしてHTMLコードを貼り付けることができます。

`Check`ボタンをクリックすると、修正が必要なエラーのリストが表示されます。

もう一つ使えるHTMLバリデーターは[`jsonformatter.org`](https://jsonformatter.org/)です。

最初のエディターにHTMLコードをコピー＆ペーストし、`Validate`ボタンをクリックすると、コード内のエラーが表示されます。
