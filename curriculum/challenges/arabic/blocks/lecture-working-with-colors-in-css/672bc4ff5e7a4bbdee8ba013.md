# --interactive--

تلعب الألوان في CSS دورًا حيويًا في تصميم صفحات الويب، وتعزيز قابلية القراءة، وتحديد المزاج، وتحسين تجربة المستخدم. واحدة من أبسط الطرق لتعريف الألوان في CSS هي استخدام الألوان المسماة. الألوان المسماة هي أسماء ألوان معرفة مسبقًا ومعترف بها من المتصفحات. إليك مثالًا على استخدام لون مسمى لعنصر فقرة:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css" />
<p>This is a paragraph.</p>
```

```css
p {
  color: red;
}
```

:::

في هذا المثال، نستخدم اللون المسمى `red` لتنسيق النص في فقرة.

الألوان المسماة في CSS هي مجموعة من 140 اسم لون قياسي مثل `red`، `blue`، `yellow`، `aqua`، `fuchsia`، `black`، وهكذا. هذه الأسماء سهلة الاستخدام وتجعل شفرتك أكثر قابلية للقراءة، وهي ذاتية الوصف.

الألوان المسماة مفيدة للنماذج الأولية السريعة، والتصاميم البسيطة، وتحسين قابلية قراءة الشفرة. إليك مثالًا آخر لاستخدام الألوان المسماة لمحدد `h1`:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css" />
<h1>This is a heading</h1>
```

```css
h1 {
  color: navy;
  background-color: lightgray;
}
```

:::

في هذا المثال، سيتم تنسيق نص العنوان باللون البحري، مع خلفية رمادية فاتحة. تتحسن قابلية قراءة الشفرة لأن الألوان المسماة توفر فهمًا بصريًا فوريًا للأسلوب المقصود.

الألوان المسماة في CSS مريحة لكنها محدودة، إذ تتوفر فقط 140 خيارًا. قد لا توفر الدرجة الدقيقة المطلوبة للتصاميم الأكثر تفصيلاً.

الألوان المسماة في CSS طريقة رائعة لتطبيق الألوان بسرعة وبشكل وصفي. بينما هي مفيدة للتصاميم الأساسية، والنماذج الأولية، وتحسين قراءة الشفرة، فإن نطاقها المحدود يجعلها أقل ملاءمة للتصاميم المعقدة التي تتطلب تحكمًا دقيقًا في الألوان.

من خلال فهم نقاط القوة والقيود في الألوان المسماة، يمكنك تحديد متى يكون من الأفضل استخدامها بدلاً من نماذج الألوان الأكثر تفصيلاً مثل RGB أو HSL، والتي ستتعلم عنها في الدروس القادمة.
