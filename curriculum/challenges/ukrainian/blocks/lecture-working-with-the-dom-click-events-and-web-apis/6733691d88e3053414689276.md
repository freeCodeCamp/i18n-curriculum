# --interactive--

У попередніх уроках ви дізналися, як працювати з подіями, використовуючи метод `addEventListener()`. Але існує ще один, не рекомендований спосіб роботи з подіями в JavaScript.

Рядкові обробники подій — це спеціальні атрибути на HTML-елементі, які використовуються для виконання JavaScript-коду, коли відбувається подія.

Ось приклад елемента `button` з рядковим обробником події click:

:::interactive_editor

```html
<button onclick="alert('Hello World!')">Show alert</button>
```

:::

Коли користувач натискає на кнопку, викликається функція `alert` і відображається діалогове вікно alert з повідомленням `Hello World!`.

Інший спосіб використання рядкових обробників подій — виклик функції, визначеної в тегу `script` у HTML-документі.

Ось приклад визначення функції з назвою `changeBgColor` і виклику її з рядкового обробника події click:

:::interactive_editor

```html
<script>
  function changeBgColor() {
    document.body.style.backgroundColor = "lightblue";
  }
</script>

<button onclick="changeBgColor()">Change background color</button>
```

:::

Коли користувач натискає на кнопку, викликається функція `changeBgColor`, і колір фону сторінки змінюється на світло-синій. Хоча можна використовувати рядкові обробники подій таким чином, це не вважається найкращою практикою.

З однієї причини: рядкові обробники подій дозволяють приєднати лише одного слухача подій до елемента. Якщо ви хочете приєднати кілька слухачів подій до одного елемента, вам потрібно використовувати `addEventListener()`. Інша причина — рядкові обробники подій змішують HTML і JavaScript-код, що ускладнює читання та підтримку коду. Краще тримати HTML і JavaScript окремо, використовуючи `addEventListener()` для приєднання слухачів подій до елементів.

Рядкові обробники подій не рекомендуються для використання в сучасному JavaScript. Тому найкраща практика — дотримуватися методу `addEventListener()` при роботі з подіями в JavaScript.
