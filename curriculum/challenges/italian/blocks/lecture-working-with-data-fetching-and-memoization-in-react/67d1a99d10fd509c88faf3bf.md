# --description--

Le app React spesso si basano su API esterne e database per contenuti dinamici. Per consultare i dati da queste API e database, devi usare alcune tecniche di recupero dati.

Vediamo come funziona il recupero dati in React e le diverse opzioni disponibili per recuperare i dati.

React non impone un modo specifico per recuperare i dati, questo significa che a un livello base puoi usare l'API Fetch integrata, supportata da tutti i browser moderni.

Puoi anche usare Axios e SWR. Axios è una libreria per richieste HTTP basata su promise costruita sopra l'oggetto XMLHttpRequest, e SWR è un hook React per il recupero dati creato dal team di Vercel.

Iniziamo con un esempio. Prima devi importare gli hook `useState` e `useEffect`:

```js
import { useState, useEffect } from "react";
```

Poi dovrai creare tre variabili di stato chiamate `loading`, `data` e `error`:

```js
const [data, setData] = useState(null);
const [loading, setLoading] = useState(true);
const [error, setError] = useState(null);
```

La variabile `loading` terrà traccia se i dati sono ancora in fase di recupero. La variabile `data` rappresenta i dati stessi, e la variabile `error` catturerà eventuali errori che potrebbero verificarsi durante il processo di recupero dati.

Poiché il recupero dati è un effetto collaterale, è meglio usare l'API Fetch all'interno di un hook `useEffect`.

Ecco un esempio:

```js
useEffect(() => {
  fetch("https://jsonplaceholder.typicode.com/posts")
    .then((res) => res.json())
    .then((data) => {
      setData(data);
      setLoading(false);
    })
    .catch((err) => {
      setError(err);
      setLoading(false);
    });
}, []);
```

Questo `useEffect` recupera i dati con l'API Fetch e imposta tutti gli stati.

Puoi migliorare le cose usando `async`/`await` invece della sintassi `.then()`. Questo significa che devi avere una funzione separata dentro `useEffect` perché non puoi anteporre `useEffect` con la parola chiave `async`:

```js
useEffect(() => {
  const fetchData = async () => {
    try {
      const res = await fetch("https://jsonplaceholder.typicode.com/posts");
       
      if (!res.ok) {
        throw new Error("Network response was not ok");
      }

      const data = await res.json();
      setData(data);
    } catch (err) {
      setError(err);
    } finally {
      setLoading(false);
    }
  };

  fetchData();
}, []);
```

Poi puoi usare tutti quegli stati per renderizzare i dati dall'API.

Ecco il codice completo:

```jsx
import { useState, useEffect } from "react";

const FetchPosts = () => {
  const [data, setData] = useState(null);
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState(null);

  useEffect(() => {
    const fetchData = async () => {
      try {
        const res = await fetch("https://jsonplaceholder.typicode.com/posts");
       
        if (!res.ok) {
          throw new Error("Network response was not ok");
        }

        const data = await res.json();
        setData(data);
      } catch (err) {
        setError(err);
      } finally {
        setLoading(false);
      }
    };

    fetchData();
  }, []);

  if (loading) {
    return <p>Loading...</p>;
  }

  if (error) {
    return <p>{error.message}</p>;
  }

  return (
    <ul>
      {data.map((post) => (
        <li key={post.id}>{post.title}</li>
      ))}
    </ul>
  );
};

export default FetchPosts;
```

Nell'interfaccia utente vedresti `Loading...` sullo schermo mentre i dati vengono recuperati, e poi i dati o l'errore a seconda che il recupero sia andato a buon fine.

Ricorda che abbiamo parlato anche del recupero dati con Axios e SWR. Vediamo un esempio usando Axios.

Prima devi installare Axios dalla riga di comando così:

```sh
npm i axios
```

Poi devi importare Axios così:

```js
import axios from "axios";
```

Poi puoi usare le stesse variabili di stato di prima e recuperare dati dall'API usando `axios.get`:

```js
const [data, setData] = useState(null);
const [loading, setLoading] = useState(true);
const [error, setError] = useState(null);

useEffect(() => {
  const fetchData = async () => {
    try {
      const res = await axios.get(
        "https://jsonplaceholder.typicode.com/users"
      );
      setData(res.data);
    } catch (err) {
      setError(err);
    } finally {
      setLoading(false);
    }
  };

  fetchData();
}, []);
```

Avrai notato che in questo esempio non c'è la riga `await res.json()`. Questo perché Axios analizza automaticamente il JSON, quindi non serve.

L'ultimo esempio che vedremo è usare l'hook `useSWR` per recuperare dati.

Proprio come con Axios, devi installare SWR così:

```sh
npm install swr
```

Poi devi importare l'hook `useSWR` nel file così:

```js
import useSWR from "swr";
```

Rispetto agli esempi precedenti, la sintassi di SWR è molto più breve. Quello che devi fare è creare una funzione fetcher e passarla all'hook `useSWR` come secondo parametro (l'endpoint è il primo parametro).

Puoi anche destrutturare sia i dati che gli stati di errore dall'hook `useSWR`, quindi non ti serve l'hook `useState`.

Ecco la sintassi:

```js
const fetcher = (url) => fetch(url).then((res) => res.json());
const { data, error } = useSWR(endpoint, fetcher);
```

Nota che il nome "fetcher" qui è solo una convenzione, quindi puoi chiamare la variabile come vuoi.

Ecco un componente che recupera i todo dall'API JSON Placeholder:

```jsx
import useSWR from "swr";

const fetcher = (url) => fetch(url).then((res) => res.json());

const FetchTodos = () => {
  const { data, error } = useSWR(
    "https://jsonplaceholder.typicode.com/todos",
    fetcher
  );

  if (!data) {
    return <h2>Loading...</h2>;
  }
  if (error) {
    return <h2>Error: {error.message}</h2>;
  }

  return (
    <>
      <h2>Todos</h2>
      <div>
        {data.map((todo) => (
          <h3 key={todo.id}>{todo.title}</h3>
        ))}
      </div>
    </>
  );
};

export default FetchTodos;
```

Come hai imparato in una lezione precedente sugli hook personalizzati, il recupero dati è una logica che puoi estrarre in un hook personalizzato. Quindi, se stai recuperando dati in più componenti e pagine, è meglio creare un hook `useFetch`.

Ecco un hook `useFetch` che usa SWR per il recupero dati:

```jsx
import useSWR from "swr";

const fetcher = (url) => fetch(url).then((res) => res.json());

const useFetch = (url) => {
  const { data, error } = useSWR(url, fetcher);

  return {
    data,
    loading: !data && !error,
    error,
  };
};

export default useFetch;
```

Ecco come usare l'hook `useFetch` per riscrivere il primo esempio che recupera post dall'API JSON Placeholder:

```jsx
import useFetch from "./useFetch";

const FetchPosts = () => {
  const { data, loading, error } = useFetch(
    "https://jsonplaceholder.typicode.com/posts"
  );

  if (loading) {
    return <h2>Loading...</h2>;
  }

  if (error) {
    return <h2>{error.message}</h2>;
  }

  return (
    <>
      <h2>Posts</h2>
      <ul>
        {data.map((post) => (
          <li key={post.id}>{post.title}</li>
        ))}
      </ul>
    </>
  );
};

export default FetchPosts;
```
