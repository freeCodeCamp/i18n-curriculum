# --description--

Якщо ви коли-небудь працювали над чимось, наприклад, дипломною роботою, де зберігали копії після значних змін і називали файли щось на кшталт `Thesis-Draft`, `Thesis-Final`, `Thesis-Final-For-Real` і `Thesis-Absolute-Final-DO-NOT-CHANGE`, ви використовували дуже примітивний підхід до контролю версій. Але що якби я сказав, що є кращий спосіб?

Ви, можливо, чули про такі інструменти, як Git, SVN або Mercurial. Це системи контролю версій, і вони є ключовим компонентом у робочому процесі кожного розробника. Система контролю версій дозволяє відстежувати та керувати змінами у вашому проєкті. Для цілей цих уроків ми зосередимося на Git.

Git пропонує потужні інструменти, які дозволяють керувати історією редагувань проєкту. Ви дізнаєтеся більше про те, як використовувати Git і які функції він пропонує в наступних уроках. А поки ось що вам справді потрібно знати:

Git дозволяє робити такі речі, як створення комітів — це знімок конкретного стану вашої кодової бази. Потім ви можете порівнювати коміти, щоб побачити, які зміни ви внесли, відкотити коміт із неправильними змінами і багато іншого.

Ви також можете створювати «гілки», які можна уявити як різні шляхи у вашому коді. Якщо гілка — це дорога, якою ви йдете, то коміти — це орієнтири на цьому шляху. Гілки дозволяють працювати над новою функцією окремо, так що якщо ваша робота піде не так, ви можете видалити гілку, не впливаючи на основний код.

Ці функції роблять контроль версій необхідним компонентом для співпраці з іншими розробниками. Коли над одним проєктом працює кілька людей, гілки дозволяють кожному працювати зі своїм кодом, не заважаючи змінам інших.

Ви часто поєднуєте систему контролю версій, як-от Git, із провайдером контролю версій, наприклад GitHub або Gitlab, які пропонують додаткові інструменти для співпраці та можливість зберігати проєкти у хмарі.

Але навіть без хмарного сховища системи контролю версій можуть кардинально змінити ваш робочий процес. Більше ніяких файлів `THIS-IS-THE-FINAL-COPY-FOREVER`.

Попереду багато цікавої інформації про використання цих систем, тож давайте зануримося в це в наступних уроках!
