# --description--

JavaScript로 프로그래밍하다 보면 필연적으로 오류 메시지를 만나게 됩니다. 자주 발생하는 오류 메시지를 이해하면 디버그를 더 효과적으로 할 수 있고 더 강력한 개발자로 성장할 수 있습니다. 네 가지 흔한 오류 유형은 `SyntaxError`, `ReferenceError`, `TypeError`, 그리고 `RangeError`입니다.

`SyntaxError`는 코드에서 괄호나 대괄호를 빠뜨리는 등 잘못 작성했을 때 발생합니다. 문장의 문법 오류와 비슷하다고 생각하세요. 개발자가 배열을 만들 때 흔히 하는 실수 예시는 다음과 같습니다.

```js
const arr = ["Beau", "Quincy" "Tom"]
```

각 배열 요소는 쉼표로 구분해야 하며, 그렇지 않으면 오류 메시지가 발생합니다.

두 번째 흔한 JavaScript 오류는 `ReferenceError`입니다. `ReferenceError`에는 여러 유형이 있으며, 다양한 방식으로 발생합니다. 첫 번째 `ReferenceError` 유형은 선언되지 않은 변수를 사용하는 경우입니다.

```js
console.log(price);
```

이 예시에서는 `price` 변수를 콘솔에 출력하려 하지만 선언되지 않아 `ReferenceError`가 발생합니다.

또 다른 `ReferenceError` 예시는 `let` 또는 `const`로 선언한 변수를 정의되기 전에 접근하려 할 때입니다:

```js
console.log(b);
const b = 50;
```

위 예시는 `Cannot access 'b' before initialization` 오류를 발생시킵니다.

세 번째 흔한 오류는 `TypeError`입니다. 이 오류는 잘못된 타입에 연산을 시도할 때 발생합니다. 다음은 객체에 `map` 메서드를 사용하려는 예시입니다:

```js
const developerObj = {
  name: "Jessica",
  country: "USA",
  isEmployed: true
};

developerObj.map()
```

이 예시는 `developerObj.map is not a function` 메서드가 배열에만 사용되므로 `map` 오류가 발생합니다.

마지막으로 살펴볼 흔한 오류는 `RangeError`입니다.

`RangeError`는 JavaScript가 처리할 수 있는 범위를 벗어난 값을 사용하려 할 때 발생합니다. 배열의 길이에 잘못된 값을 할당하는 예시는 다음과 같습니다:

```js
const arr = [];
arr.length = -1; 
```

배열의 `length`은 0 이상의 정수여야 하므로 `-1`로 설정하면 `RangeError`가 발생합니다.

JavaScript로 계속 프로그래밍하면서 이러한 다양한 오류 유형과 그 원인을 인지하세요.
