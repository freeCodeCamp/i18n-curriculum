# --interactive--

In der vorherigen Lektion haben Sie gelernt, wie Sie Ausnahmen in Ihren Programmen werfen. In dieser Lektion sehen wir uns an, wie Sie diese Fehler in einem `try...catch...finally`-Block elegant behandeln können.

Der `try`-Block wird verwendet, um Code einzuschließen, der einen Fehler werfen könnte. Er dient als sicherer Bereich, um etwas auszuprobieren, das fehlschlagen kann.

Der `catch`-Block fängt Fehler ab und behandelt sie, die im `try`-Block auftreten. Sie können das `Error`-Objekt innerhalb von `catch` verwenden, um zu prüfen, was schiefgelaufen ist.

Der `finally`-Block wird nach den `try`- und `catch`-Blöcken ausgeführt, unabhängig davon, ob ein Fehler aufgetreten ist. Er wird häufig für Aufräumarbeiten verwendet, wie zum Beispiel zum Schließen von Dateien oder Freigeben von Ressourcen.

Hier ist ein Beispiel für die Verwendung eines `try...catch`-Blocks:

:::interactive_editor

```js
function processInput(input) {
  if (typeof input !== "string") {
    throw new TypeError("Input must be a string.");
  }

  return input.toUpperCase();
}

try {
  console.log("Starting to process input...");
  const result = processInput(9);
  console.log("Processed result:", result);
} catch (error) {
  console.error("Error occurred:", error.message);
} 
```

:::

In diesem Beispiel haben wir eine Funktion namens `processInput`, die zuerst überprüft, ob das `input` nicht vom Typ `string` ist. Falls das zutrifft, werfen wir einen Fehler. Andernfalls geben wir das Ergebnis der Verwendung der `toUpperCase`-Methode auf dem `input` zurück.

Wir rufen die Funktion innerhalb eines `try`-Blocks auf. Da der Funktionsaufruf einen Fehler wirft, wird dieser im `catch`-Block abgefangen und die Fehlermeldung wird in der Konsole angezeigt.

Der Fehler, der in den `catch`-Block übergeben wird, ist ein `Error`-Objekt, das Informationen über diesen Fehler enthält. In diesem Fall verwenden wir die `message`-Eigenschaft, die dem Benutzer menschenlesbare Informationen anzeigt.

Wir verwenden `console.error`, weil es speziell dafür ausgelegt ist, Fehler zu protokollieren. In vielen modernen Browsern erscheint die Ausgabe von `console.error()` in der Konsole rot.

Die `finally`-Anweisung wird ausgeführt, unabhängig davon, ob eine Ausnahme geworfen wurde oder nicht.

```js
try {
  // Code that might throw an error
} catch (error) {
  // Code to handle the error
} finally {
  // Code that runs regardless of whether an error occurred or not
}
```

Ein guter Anwendungsfall für die `finally`-Anweisung ist, wenn Sie mit Dateien arbeiten. In JavaScript können Sie eine Datei öffnen, einen `try`-Block verwenden, um Daten in die Datei zu schreiben. Wenn Fehler auftreten, können Sie `catch` verwenden, um diese Fehler abzufangen. Anschließend verwenden Sie die `finally`-Anweisung, um die Datei zu schließen.
