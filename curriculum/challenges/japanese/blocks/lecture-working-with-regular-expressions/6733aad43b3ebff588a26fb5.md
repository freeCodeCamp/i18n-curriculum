# --interactive--

正規表現、またはregexは、多くの異なるプログラミング言語でサポートされている機能です。

正規表現は「パターン」を作成するための特別な構文で、それを使って文字列をチェックしたり、テキストを抽出したりできます。

基本的な正規表現を見てみましょう：

```js
const regex = /freeCodeCamp/;
```

JavaScriptでは、正規表現を2つのスラッシュ（`/`）の間にパターンを作成して定義することに注意してください。これは、両方のスラッシュの後にテキストが続くコメントとは混同しないでください。

この特定の正規表現は、文字列のどこかに大文字の`C`を含むテキスト`freeCodeCamp`にマッチします。でも、実際にどうやってそれを行うのでしょうか？

ここで最初のメソッド、`test()`メソッドの登場です。`test()`メソッドは`RegExp`オブジェクトに存在します。`RegExp`オブジェクトは、先ほど定義したような正規表現を表すオブジェクトです。

`test()`メソッドは文字列を受け取り、その文字列が正規表現にマッチするかどうかをテストします。例えば、文字列`e`をテストしてみましょう：

:::interactive_editor

```js
const regex = /freeCodeCamp/;
const test = regex.test("e");
console.log(test);
```

:::

新しいregexで`test()`メソッドを呼び出し、文字列`e`を引数として渡したことがわかります。

`test()`メソッドは`false`を返しました。なぜなら文字列`e`はパターン`freeCodeCamp`にマッチしないからです。パターン`freeCodeCamp`には文字`e`が含まれていますが、それは正規表現の動作とは逆方向です。

いくつか例を見てみましょう。これらを考えてみてください：

:::interactive_editor

```js
const regex = /freeCodeCamp/;
console.log(regex.test("freeCodeCamp"));
console.log(regex.test("freeCodeCamp is great"));
console.log(regex.test("I love freeCodeCamp"));
console.log(regex.test("freecodecamp"));
console.log(regex.test("FREECODECAMP"));
console.log(regex.test("free"));
console.log(regex.test("code"));
console.log(regex.test("camp"));
```

:::

最初の3つの文字列は`true`を返しました。これらの文字列はすべて、文字列のどこかに正確にテキスト`freeCodeCamp`を含んでいます。

5行目と6行目は`false`を返します。`freecodecamp`のテキストは含まれていますが、大文字・小文字が一致しません。正規表現はデフォルトでは大文字・小文字を区別します。

最後の3つはパターンの一部を含んでいますが、文字列はパターン全体を含んでいません。

`test()`メソッドは`boolean`を返し、文字列が正規表現にマッチするかどうかを示します。

しかし、それ以上の情報が欲しい場合はどうでしょう？文字列には`match()`メソッドがあります。このメソッドは正規表現を受け取りますが、文字列を渡すと正規表現に変換されます。

`match()`は文字列のマッチ配列を返します。マッチ配列とは何でしょう？見てみましょう：

```js
const regex = /freeCodeCamp/;
const match = "freeCodeCamp".match(regex);
console.log(match);
```

これを実行すると配列が返ってきます！でもちょっと変わった配列です。いくつか追加のプロパティがあります：

```js
// [
//   'freeCodeCamp',
//   index: 0,
//   input: 'freeCodeCamp',
//   groups: undefined
// ]
```

`groups`プロパティはキャプチャグループを表示します。これは今後のレッスンで学びます。

`index`プロパティは文字列のどの文字でマッチが見つかったかを教えてくれます。今回の場合は文字列の先頭で見つかりました。

`input`プロパティは`match()`メソッドが呼ばれた文字列を示します。

もう一度いくつか試して、結果がどう変わるか見てみましょう：

```js
const regex = /freeCodeCamp/;
console.log("freeCodeCamp".match(regex)); // ['freeCodeCamp', index: 0, input: 'freeCodeCamp', groups: undefined]
console.log("freeCodeCamp is great".match(regex)); // ['freeCodeCamp', index: 0, input: 'freeCodeCamp is great', groups: undefined]
console.log("I love freeCodeCamp".match(regex)); // ['freeCodeCamp', index: 7, input: 'I love freeCodeCamp', groups: undefined]
console.log("freecodecamp".match(regex)); // null
console.log("FREECODECAMP".match(regex)); // null
console.log("free".match(regex)); // null
```

最初の3つの文字列はマッチを返すはずなので、見てみましょう：

```js
// ['freeCodeCamp', index: 0, input: 'freeCodeCamp', groups: undefined]
// ['freeCodeCamp', index: 0, input: 'freeCodeCamp is great', groups: undefined]
// ['freeCodeCamp', index: 7, input: 'I love freeCodeCamp', groups: undefined]
```

期待通りでしたか？文字列やマッチの位置によって`input`と`index`が変わっているのがわかります。

マッチしない他の3行は配列ではなく`null`を返します。

正規表現で文字列をテストしマッチさせることができましたが、文字列の内容を置き換えたい場合はどうでしょう？例えば、誰かが`freecodecamp`をすべて小文字で書いてしまった場合、自動的に大文字・小文字を直したいかもしれません。

まず、`freecodecamp`の小文字にマッチするように正規表現を更新し、テスト文字列を作成します：

```js
const regex = /freecodecamp/;
const str = "freecodecamp is rly kewl";
```

文字列には`replace()`メソッドがあり、2つの引数を受け取ります。1つ目はマッチさせる正規表現（または正規表現の機能が不要なら文字列）、2つ目はマッチした部分を置き換える文字列（またはマッチごとに実行する関数）です。

では、`freecodecamp`を正しい大文字・小文字に置き換えたい場合：

:::interactive_editor

```js
const regex = /freecodecamp/;
const str = "freecodecamp is rly kewl";
const replaced = str.replace(regex, "freeCodeCamp");
console.log(replaced);
```

:::

`replace()`がマッチしたパターン`replaced`を含む更新された文字列を返しているのがわかります。

正規表現とそれに関連するメソッドは複雑で圧倒されるかもしれませんが、この次のレッスンでさらに詳しく学ぶ機会があります。
