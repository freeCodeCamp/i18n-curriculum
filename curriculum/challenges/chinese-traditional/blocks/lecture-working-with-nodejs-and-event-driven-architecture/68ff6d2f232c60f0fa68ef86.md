# --description--

到目前為止，你一直在使用 JavaScript 撰寫客戶端程式碼，這些程式碼會在網頁應用程式的客戶端執行，通常是在網頁瀏覽器中。

客戶端 JavaScript 可以非常強大，但對於較複雜的任務確實有某些限制。

例如：

- 它對區域的檔案有非常受限的存取。
- 它並非真正設計用來處理複雜的應用程式邏輯。
- 而且它具有潛在的安全性問題。<dfn>例如</dfn>，你不會想將敏感的資訊，例如資料庫憑證，暴露給客戶端的程式碼。
    
客戶端 JavaScript 有這些限制，因為它最初是設計為專門在網頁瀏覽器上執行。

網頁瀏覽器提供執行 JavaScript 程式碼所需的環境，包括 JavaScript 引擎。它們也提供對 Document Object Model（DOM）的存取，讓你可以在程式碼中存取 HTML 元素。

過去，我們無法在瀏覽器外執行 JavaScript，但現在我們可以使用 Node.js。

Node.js 是一個 JavaScript 執行期環境。它被官方定義為「一個開放原始碼且跨平台的 JavaScript 執行期環境」。

讓我們來看看這是什麼意思：

- Node.js 是開放原始碼，因為它的程式碼公開可用，並由大量開發者社群維護。
- 它是跨平台的，因為它能在任何作業系統上運作，包括 Windows、macOS 和 Linux。
- 它是一個 JavaScript 執行期環境，因為它允許你在瀏覽器外執行 JavaScript 程式碼。
    
Node.js 被廣泛應用於各種產業。開發者使用它來建置處理 HTTP 請求的網頁伺服器和 API，以及網頁和行動應用程式。

如果你正在使用或開發一個與資料庫交談或控制複雜資料的網站或網頁應用程式，那麼你很可能在某個時候會在幕後使用 `Node.js`。

它高效、可擴充，並且擁有遍布全球的大量開發者和維護者社群。

既然你知道什麼是 Node.js，讓我們來看看瀏覽器和 Node 執行期環境之間的一些差異。

瀏覽器環境主要是為前端網頁開發設計，因此它執行客戶端的 JavaScript。相較之下，Node 執行期環境主要是為後端網頁開發設計，因此它執行伺服端的 JavaScript。這也決定了它們所提供的 API。

例如，從瀏覽器中，你可以存取 `DOM API`，但存取區域的檔案系統有侷限。相較之下，從 `Node` 執行期環境中，你可以存取幾乎所有系統資源，包括檔案系統，但無法存取 `DOM`。

兩個 JavaScript 環境都提供一個全域物件。在瀏覽器的 JavaScript 執行期中，這個物件稱為 `window`。這個 `window` 物件提供存取與瀏覽器相關的功能，例如存取操作 DOM、管理 cookie 和處理瀏覽器事件的方法。在 Node.js 環境中，這個全域物件稱為 `global`。`global` 物件提供存取 Node.js 特定的功能，例如用於操作區域檔案、網路和與作業系統互動的內建模組。

最後，另一個重要的差異是你可以選擇在你的伺服端使用的 Node.js 版本。然而，你無法控制使用者用來造訪你網站的瀏覽器環境版本。

Node.js 已經成為全球開發者非常重要的工具。它是一個強大、可擴充且多功能的工具，透過讓開發者使用 JavaScript 建造全端應用程式的前端和後端，改變了網頁開發。這使得開發流程更加高效，因為開發者不需要為了開發後端而學習新的程式語言。
