## Що таке підпрограмне забезпечення?

- **Підпрограмне забезпечення** — це функція, яка виконується під час життєвого циклу запиту до сервера.
- Підпрограмне забезпечення має доступ до **об’єкта запиту** (`req`), **об’єкта відповіді** (`res`) і функції **`next`**.
- Загальні випадки використання: логування запитів, обробка помилок, аналіз вхідних даних і обслуговування статичних файлів.
- Підпрограмне забезпечення виконується в порядку, у якому його додано до застосунку.

### Як працює підпрограмне забезпечення

1. Запит надсилається на сервер.
2. Функції підпрограмного забезпечення обробляють запит.
3. Якщо підпрограмне забезпечення викликає `next()`, керування передається наступному підпрограмному забезпеченню або обробнику маршруту.
4. Якщо підпрограмне забезпечення не викликає `next()`, цикл запит-відповідь зупиняється.

```javascript
app.use((req, res, next) => {
  console.log('Request URL:', req.url);
  console.log('Request Method:', req.method);
  next();
});
```

- `app.use()` додає підпрограмне забезпечення глобально до застосунку.

## Підпрограмне забезпечення рівня застосунку

- **Підпрограмне забезпечення рівня застосунку** прив’язане до екземпляра застосунку Express.
- Реєструється за допомогою `app.use()` або методів, специфічних для маршруту, таких як `app.get()`, `app.post()` тощо.
- Виконується для кожного вхідного запиту, якщо не обмежено конкретним маршрутом.
- Випадки використання: логування, автентифікація та обробка помилок на кількох маршрутах.

```javascript
app.use((req, res, next) => {
  console.log('Request received at:', new Date());
  next();
});
```

## Підпрограмне забезпечення рівня маршрутизатора

- **Підпрограмне забезпечення рівня маршрутизатора** прив’язане до екземпляра `express.Router()` замість головного застосунку.
- Застосовується за допомогою `router.use()` і впливає лише на маршрути, які обробляє цей маршрутизатор.
- Допомагає модульно організувати застосунок у ізольовані модулі маршрутів.
- Підключіть маршрутизатор за допомогою `app.use('/path', router)`, щоб обмежити підпрограмне забезпечення конкретними маршрутами.

```javascript
const router = express.Router();

router.use((req, res, next) => {
  console.log('Request made to /menu route');
  next();
});

router.get('/drinks', (req, res) => {
  res.send('Welcome to the drinks menu!');
});

app.use('/menu', router);
```

## Підпрограмне забезпечення для обробки помилок

- **Підпрограмне забезпечення для обробки помилок** — це спеціальне підпрограмне забезпечення, яке використовується для перехоплення помилок під час циклу запит-відповідь.
- Воно має **чотири параметри**: `err`, `req`, `res` і `next`. Підпис із чотирма параметрами вказує Express, що це обробник помилок.
- Викликається, коли помилка передається до `next(err)`.
- Має розміщуватися **в кінці** стеку підпрограмного забезпечення, після всіх інших підпрограм і маршрутів.

```javascript
app.use((err, req, res, next) => {
  console.error(err.stack);
  res.status(500).send('Something went wrong!');
});
```

## Вбудоване та стороннє підпрограмне забезпечення

### Вбудоване підпрограмне забезпечення

- **`express.json()`**: аналізує вхідні запити з JSON-корисним навантаженням.

```javascript
app.use(express.json());
```

- **`express.static()`**: обслуговує статичні файли, такі як зображення, CSS і JavaScript.

```javascript
app.use(express.static('public'));
```

- **`express.urlencoded()`**: аналізує вхідні запити з URL-кодованим корисним навантаженням (формат, який надсилають HTML-форми); робить проаналізовані дані доступними на `req.body`.

```javascript
app.use(express.urlencoded({ extended: true }));
```

### Стороннє підпрограмне забезпечення

- **`cors`**: увімкнення спільного використання ресурсів між різними джерелами (CORS).

```javascript
app.use(cors());
```

- **`morgan`**: логування HTTP-запитів.

```javascript
app.use(morgan('tiny'));
```

- І вбудовані, і сторонні функції підпрограмного забезпечення додаються в порядку, у якому їх викликають за допомогою `app.use()`.
