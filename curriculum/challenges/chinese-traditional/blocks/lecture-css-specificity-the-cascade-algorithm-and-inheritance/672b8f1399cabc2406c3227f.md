# --interactive--

CSS 中的 `!important` 關鍵字用來賦予樣式規則最高的優先權，使其能覆寫該屬性的任何其他宣告。

使用時，會強制瀏覽器套用指定的樣式，無論其他選擇器的特異性為何。

假設你有一個帶有行內樣式的 HTML 段落元素，如下所示：

:::interactive_editor

```html
<p class="para" style="background-color: lightblue; color: black;">
  This is a paragraph.
</p>
```

:::

在此範例中，段落元素的背景顏色設定為 `lightblue`，文字顏色設定為 `black`。

CSS 檔案對段落元素套用以下樣式：

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<p class="para" style="background-color: lightblue; color: black;">
  This is a paragraph.
</p>
```

```css
.para {
  background-color: black;
  color: white;
}
```

:::

由於行內樣式的優先權高於類別、ID 和型別選擇器，因此該段落元素不會套用黑色背景色和白色文字色。

若要改寫那些行內樣式，你可以像這樣使用 `!important` 關鍵字：

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<p class="para" style="background-color: lightblue; color: black;">
  This is a paragraph.
</p>
```

```css
.para {
  background-color: black !important;
  color: white !important;
}
```

:::

`!important` 關鍵字用於 CSS 值之後及分號之前。

現在段落元素將會套用那些黑白色彩，因為行內樣式被使用 `!important` 關鍵字所覆寫。

CSS 中的 `!important` 關鍵字用於賦予樣式規則最高的優先權，有效地覆蓋其他宣告，包括具有較高特異性的宣告和行內樣式。

然而，`!important` 關鍵字不會改變 CSS 選擇器本身的特異性。它只是確保帶有 `!important` 的規則被套用，即使有其他具有較高特異性的衝突規則。

另一個適當使用 `!important` 關鍵字的情境是當你無法控制原始 CSS 時，用來覆寫第三方函式庫或框架的樣式。

然而，過度使用 `!important` 關鍵字會導致維護和偵錯你的 CSS 變得困難，因為它破壞了樣式的自然層疊，並可能導致意想不到的後果。

因此，最好謹慎使用 `!important` 關鍵字。
