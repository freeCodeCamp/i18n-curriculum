# --interactive--

`IndexedDB`는 브라우저에서 구조화된 데이터를 저장하는 도구입니다. 이는 최신 웹 브라우저에 내장되어 있어 웹 애플리케이션이 JavaScript 객체를 효율적으로 저장하고 가져올 수 있게 합니다.

문자열 저장에 제한이 있는 `localStorage`과 달리, `IndexedDB`는 JavaScript 객체, 파일, 그리고 거의 모든 유형의 데이터를 저장할 수 있습니다. 이로 인해 크고 복잡한 자료 구조를 다뤄야 하는 웹 애플리케이션에 적합합니다.

이제 `IndexedDB`가 어떻게 작동하는지 살펴보겠습니다.

첫 번째 단계는 데이터베이스를 여는 것입니다. 예시는 다음과 같습니다:

:::interactive_editor

```js
let request = indexedDB.open("Sample DB", 1);

request.onerror = function(event) {
  console.log("Error opening database");
};

request.onsuccess = function(event) {
  let db = event.target.result;
  console.log("Database opened successfully");
};
```

:::

이 코드에서는 `"Sample DB"`라는 이름의 데이터베이스를 버전 1로 엽니다. 오류를 처리하는 콜백 함수와 데이터베이스가 성공적으로 열렸을 때 호출되는 콜백 함수 두 개를 제공합니다. 성공 콜백에서 받는 `db` 객체가 데이터베이스와 상호작용하는 데 사용됩니다.

브라우저 개발자 도구의 애플리케이션 인터페이스를 확인하면 `Sample DB` 섹션에 `IndexedDb`가 열려 있는 것을 볼 수 있습니다. 데이터베이스를 열었으면 이제 객체 저장소 작업을 시작할 수 있습니다.

IndexedDB의 객체 저장소는 전통적인 데이터베이스의 표와 비슷합니다. 저장하려는 실제 데이터를 보관합니다. 객체 저장소를 만드는 방법은 다음과 같습니다:

```js
let request = indexedDB.open("Sample DB", 1);

request.onupgradeneeded = function(event) {
  let db = event.target.result;
  let objectStore = db.createObjectStore("customers", { keyPath: "id" });
};
```

이 코드는 `"customers"`라는 이름의 객체 저장소를 `"id"`를 키 경로로 하여 만듭니다. 키 경로는 전통적인 데이터베이스의 기본 키와 같아서 각 레코드를 고유하게 식별하는 데 사용됩니다.

객체 저장소에 데이터를 추가하려면 다음과 같이 합니다. 이 예제에서 `db`는 `IndexedDB` 데이터베이스 인스턴스를 나타냅니다.

```js
let transaction = db.transaction(["customers"], "readwrite");
let objectStore = transaction.objectStore("customers");
let request = objectStore.add({ id: 1, name: "John Doe", email: "john@example.com" });

request.onerror = function(event) {
  console.log("Error adding data");
};

request.onsuccess = function(event) {
  console.log("Data added successfully");
};
```

이 코드는 `"customers"` 객체 저장소에 새 고객을 추가합니다. 트랜잭션을 시작하고(데이터베이스 작업을 그룹화하는 방법), 객체 저장소 참조를 얻은 다음 데이터를 추가합니다.

데이터를 가져오는 것도 비슷한 방식으로 작동합니다. 트랜잭션을 시작하고 객체 저장소를 얻은 후 `get` 같은 메서드를 사용해 데이터를 가져옵니다:

```js
let transaction = db.transaction(["customers"]);
let objectStore = transaction.objectStore("customers");
let request = objectStore.get(1);

request.onerror = function(event) {
  console.log("Error retrieving data");
};

request.onsuccess = function(event) {
  console.log("Customer:", request.result);
};
```

이 코드는 `id` 객체 저장소에서 `1`의 `"customers"` 고객을 가져옵니다.

`IndexedDB`의 주요 기능 중 하나는 비동기라는 점입니다.

즉, `IndexedDB`와 상호작용할 때 작업이 웹 애플리케이션의 메인 스레드를 차단하지 않습니다. 이로 인해 대량의 데이터를 다룰 때도 웹 애플리케이션이 반응형으로 유지됩니다.

`IndexedDB`는 강력한 기능을 제공하지만, 다른 더 간단한 저장 API 옵션에 비해 학습 곡선이 가파르기 때문에 초보자에게는 어려울 수 있습니다. 그러나 클라이언트 측에서 대량의 구조화된 데이터를 처리해야 하는 애플리케이션에는 `IndexedDB`가 뛰어난 기능을 제공합니다.
