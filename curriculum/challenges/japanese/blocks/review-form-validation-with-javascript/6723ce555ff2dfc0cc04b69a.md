# --interactive--

## JavaScriptでのフォームバリデーション

- **Constraint Validation API**: `textarea`や`input`のような特定のHTML要素は、制約バリデーションAPIを公開しています。このAPIを使うと、ユーザーが入力した値が最小長やパターンマッチングなど、HTMLレベルで設定したバリデーションを通過しているかどうかを確認できます。
- **`checkValidity()`メソッド**: このメソッドは、要素がすべてのHTMLバリデーション（属性に基づく）に合格すると`true`を返し、失敗すると`false`を返します。

:::interactive_editor

```html
<form>
  <label>
    Email:
    <input
      id="email"
      type="email"
      required
      pattern=".+\.com$"
      placeholder="example@site.com"
    />
  </label>
</form>

<script>
  const input = document.getElementById("email");

  input.addEventListener("input", (e) => {
    if (!e.target.checkValidity()) {
      e.target.setCustomValidity("You must use a .com email.");
    } else {
      e.target.setCustomValidity("");
    }
  });
</script>
```

:::

- **`reportValidity()`メソッド**: このメソッドは、ブラウザに対して`input`が無効であることを伝えます。

:::interactive_editor

```html
<form>
  <label>
    Email:
    <input
      id="email2"
      type="email"
      required
      pattern=".+\.com$"
      placeholder="example@site.com"
    />
  </label>
</form>

<script>
  const input = document.getElementById("email2");

  input.addEventListener("input", (e) => {
    if (!e.target.checkValidity()) {
      e.target.reportValidity();
    }
  });
</script>
```

:::

- **`validity`プロパティ**: このプロパティは、`<input>`や`<select>`などのフォームコントロールの有効状態を取得または設定するために使われ、ユーザー入力がその要素に定義された制約（例：`required`フィールド、パターン制約、最大長など）を満たしているかどうかの情報を提供します。

:::interactive_editor

```html
<input
  id="age"
  type="number"
  min="18"
  placeholder="Enter age (18+)"
/>

<script>
  const input = document.getElementById("age");

  input.addEventListener("input", (e) => {
    console.log(e.target.validity);
  });
</script>
```

:::

- **`patternMismatch`プロパティ**: この値が指定された正規表現パターンに一致しない場合、`true`になります。

## `preventDefault()`メソッド

- **定義**: DOMで発生するすべてのイベントには何らかのデフォルトの動作があります。例えば、チェックボックスのclickイベントはデフォルトでそのチェックボックスの状態を切り替えます。フォーカスされたボタンでスペースキーを押すと、そのボタンがアクティブになります。これらの`preventDefault()`オブジェクトの`Event`メソッドは、その動作が起こるのを止めます。

:::interactive_editor

```html
<form id="form">
  <input type="text" placeholder="Try to submit" />
  <button type="submit">Submit</button>
</form>

<p id="status"></p>

<script>
  const form = document.getElementById("form");
  const status = document.getElementById("status");

  form.addEventListener("submit", (event) => {
    event.preventDefault();
    status.textContent = "Form submission prevented.";
  });
</script>
```

:::

## フォームの送信

- **定義**: フォームが送信される方法は3つあります。1つ目は、フォーム内の`type`属性が`submit`に設定されたボタンをユーザーがクリックした場合です。2つ目は、フォーム内の編集可能な`Enter`フィールドでユーザーが`input`キーを押した場合です。3つ目は、JavaScriptで`requestSubmit()`要素の`submit()`または`form`メソッドを呼び出した場合です。
- **`action`属性**: `action`属性にはURLまたは現在のドメインに対する相対パスを指定します。この値はフォームがデータを送信しようとする場所を決定します。`action`属性を設定しない場合、フォームは現在のページのURLにデータを送信します。

```html
<form action="https://freecodecamp.org" method="GET">
  <input
    type="number"
    name="number"
    placeholder="Enter a number"
  />
  <button type="submit">Submit</button>
</form>
```

- **`method`属性**: この属性は`HTTP`メソッド（`GET`や`POST`など）を受け入れ、アクションURLへのリクエストにそのメソッドを使います。メソッドが設定されていない場合、フォームはデフォルトで`GET`リクエストになります。フォーム内のデータは`name=value`ペアとしてURLエンコードされ、クエリパラメータとしてアクションURLに追加されます。

```html
<form action="/data" method="POST">
  <input
    type="number"
    id="input"
    placeholder="Enter a number"
    name="number"
  />
  <button type="submit">Submit</button>
</form>
```

- **`enctype`属性**: `form`要素は`enctype`属性を受け入れ、データのエンコードタイプを表します。この属性は3つの値のみを受け入れます：`application/x-www-form-urlencoded`（デフォルトで、データをURLエンコードされたフォームボディとして送信）、`text/plain`（データをプレーンテキスト形式で、`name=value`ペアを改行で区切って送信）、および`multipart/form-data`で、ファイルアップロードを含むフォームの処理に特化しています。
