# --description--

Agora que você já criou um app básico com Express, é hora de entender melhor o roteamento e como ele funciona.

No Express, roteamento é como você define os endpoints de uma aplicação web. Você pode pensar nas rotas como regras que dizem ao app como responder quando um usuário acessa uma URL específica.

Vamos olhar novamente para nosso app simples em Express:

```javascript
const express = require("express");
const app = express();
const port = 3000;

app.get("/", (req, res) => {
  res.send("Hello World!");
});

app.listen(port, () => {
  console.log(`Example app listening on port ${port}`);
});
```

Neste exemplo, o app define uma rota para a URL raiz (`/`). Quando um cliente envia uma requisição GET para esse caminho, o servidor responde com o texto `Hello World!`. O app então começa a escutar requisições na porta `3000`.

Mas existem outros métodos de rota.

O Express permite que você trate diferentes tipos de requisições HTTP usando métodos de roteamento diferentes. Os mais comuns são:

* `app.get()` – recupera dados
    
* `app.post()` – cria novos dados
    
* `app.put()` – atualiza dados existentes
    
* `app.delete()` – exclui dados
    

Cada método define como sua aplicação deve responder a uma requisição HTTP específica em uma rota específica.

Aqui estão alguns exemplos simples:

```javascript
app.post("/submit", (req, res) => {
  res.send("Form submitted!");
});

app.put("/update", (req, res) => {
  res.send("Item updated!");
});

app.delete("/delete", (req, res) => {
  res.send("Item deleted!");
});
```

Em cada caso, o Express escuta uma requisição com um método HTTP e caminho de URL específicos, e então executa a função manipuladora correspondente para enviar uma resposta de volta ao cliente.

Você pode testar essas rotas usando ferramentas como Postman ou `curl`, que permitem enviar diferentes tipos de requisições HTTP para seu servidor.

E isso é roteamento no Express — é simplesmente definir como sua aplicação responde a diferentes requisições.
