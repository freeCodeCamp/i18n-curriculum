# --description--

CSP(Content Security Policy)와 Permissions-Policy는 웹 애플리케이션을 보호하는 데 도움을 주는 강력한 HTTP 보안 헤더로, 어떤 리소스를 로드할 수 있고 페이지가 어떤 브라우저 기능에 접근할 수 있는지를 제한합니다.

Content Security Policy는 주로 리소스 로드에 집중합니다. 프론트엔드 스크립트, 스타일, 이미지에 대한 안전 필터 역할을 합니다.

악의적인 공격자는 종종 취약한 웹사이트에 해로운 스크립트를 주입하려고 합니다. 강력한 CSP는 신뢰할 수 있는 출처 도메인의 엄격한 허용 목록을 설정하여 이러한 교차 사이트 스크립팅(XSS) 공격을 막습니다.

서버가 CSP 헤더를 보내면 브라우저는 특정 지시문을 읽습니다. 예를 들어, `script-src` 지시문은 브라우저에 어떤 도메인에서 안전하게 JavaScript를 실행할 수 있는지 알려줍니다.

해커가 승인되지 않은 외부 서버에서 악성 스크립트를 주입하려 하면, 브라우저는 정책 위반을 감지하고 즉시 스크립트를 차단하며 보안 오류를 기록합니다.

CSP는 이미지 소스, 폼 전송 대상, 스타일시트 출처도 제한할 수 있습니다.

Permissions-Policy는 브라우저 기능과 장치 하드웨어에 집중합니다. 이전의 Feature-Policy 헤더를 대체했습니다.

이 보안 메커니즘은 사이트 소유자가 자신의 페이지와 임베드된 iframe에 대해 특정 브라우저 기능을 명시적으로 활성화하거나 비활성화할 수 있게 하며, 타사 코드가 사용자 프라이버시를 악용하는 것을 방지합니다.

Permissions-Policy 헤더를 사용하면 웹캠, 마이크, 위치 추적 같은 하드웨어 접근을 제어할 수 있습니다.

예를 들어, 웹사이트가 사용자 위치를 알 필요가 없다면 정책을 설정해 위치 API를 완전히 차단할 수 있습니다. 페이지의 스크립트가 공격당해도 브라우저가 추적 기능을 활성화하지 않습니다.

Permissions-Policy 구현은 기능 이름과 허용 출처를 정의하는 명확하고 구조화된 헤더 형식을 사용합니다.

하드웨어 기능을 자신의 도메인으로 엄격히 제한하려면 소괄호로 감싼 `self` 키워드(`camera=(self)`)를 사용합니다. 모든 사용자에 대해 기능을 완전히 비활성화하려면 빈 리스트(`geolocation=()`)를 사용합니다.

두 헤더를 결합하면 웹 애플리케이션에 강력한 방어막을 제공합니다. CSP는 데이터 전달을 잠그고 코드 주입을 막으며, Permissions-Policy는 물리적 장치를 보호하고 프라이버시 준수를 보장합니다.

현대 웹 개발자는 두 도구를 모두 사용해 매우 견고한 브라우징 환경을 만듭니다.
