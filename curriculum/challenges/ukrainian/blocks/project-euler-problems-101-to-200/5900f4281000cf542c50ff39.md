# --description--

Ось записи з навантаженої телефонної системи з мільйоном користувачів:

| RecNr | Caller | Called |
|-------|--------|--------|
|   1   | 200007 | 100053 |
|   2   | 600183 | 500439 |
|   3   | 600863 | 701497 |
|  ...  |  ...   |  ...   |

Телефонний номер того, хто дзвонить, і номер того, кому дзвонять у записі $n$ — це $Caller(n) = S_{2n - 1}$ та $Called(n) = S_{2n}$, де ${S}_{1,2,3,\ldots}$ генерує "Генератор із затримкою Фібоначчі":

Для $1 ≤ k ≤ 55$, $S_k = [100003 - 200003k + 300007{k}^3]\\;(\text{за модулем}\\;1000000)$

Для $56 ≤ k$, $S_k = [S_{k - 24} + S_{k - 55}]\\;(\text{за модулем}\\;1000000)$

Якщо $Caller(n) = Called(n)$, вважається, що користувач помилився номером і виклик не відбувся; інакше виклик успішний.

Від початку записів вважаємо, що будь-які два користувачі $X$ та $Y$ — друзі, якщо $X$ дзвонить $Y$ або навпаки. Аналогічно, $X$ — друг друга друга $Z$, якщо $X$ — друг $Y$, а $Y$ — друг $Z$; і так далі для довших ланцюгів.

Телефонний номер прем’єр-міністра — 524287. Після скількох успішних викликів, не рахуючи помилкових наборів, 99% користувачів (включно з прем’єр-міністром) будуть другом, або другом друга тощо, прем’єр-міністра?
