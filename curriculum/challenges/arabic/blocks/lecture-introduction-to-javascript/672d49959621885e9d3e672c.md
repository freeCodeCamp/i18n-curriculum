# --interactive--

عند العمل مع JavaScript، غالبًا ما تعلن عن متغيرات لتخزين بيانات تخطط لاستخدامها طوال برنامجك.

في JavaScript الحديثة، تُعتبر `let` و `const` الطرق المفضلة للإعلان عن المتغيرات، لكنهما يختلفان في كيفية التعامل مع تعيين القيم وإعادة تعيينها.

في هذا الدرس، سنستكشف كيف يختلف `let` و `const` في الإعلان عن المتغيرات، التعيين، وإعادة التعيين.

تسمح الكلمة المفتاحية `let` بإعلان متغيرات يمكن تحديثها أو إعادة تعيينها لاحقًا. يمكنك اعتبار `let` كحاوية مرنة: بمجرد تخزين قيمة فيها، يمكنك تغيير تلك القيمة حسب الحاجة طوال برنامجك.

إليك مثالًا على الإعلان عن متغير وتعيينه باستخدام `let`:

```js
let score = 10;
```

في هذه الحالة، تم إعلان المتغير `score` وتعيين القيمة `10` له. إذا أردت تحديث القيمة لاحقًا، يمكنك فعل ذلك بسهولة:

:::interactive_editor

```js
let score = 10;
console.log(score); // 10
score = 20;
console.log(score); // 20
```

:::

الآن، يحمل `score` القيمة `20`. هذا يجعل `let` مفيدًا بشكل خاص عندما تعرف أن قيمة المتغير ستتغير أثناء تشغيل برنامجك.

من ناحية أخرى، يُستخدم `const` لإعلان المتغيرات الثابتة. بمجرد تعيين قيمة لمتغير مُعلن بـ `const`، لا يمكنك إعادة تعيينه.

هذا يجعل `const` مثاليًا للقيم التي لا تريد تغييرها عن طريق الخطأ أثناء تنفيذ برنامجك.

إليك مثالًا على الإعلان عن متغير وتعيينه باستخدام `const`:

:::interactive_editor

```js
const maxScore = 100;
console.log(maxScore); // 100
```

:::

بمجرد تعيين القيمة `100` للمتغير `maxScore`، لا يمكن تغييره:

```js
maxScore = 200; // This will result in an error
```

محاولة إعادة تعيين قيمة لمتغير `const` ستُصدر خطأ في وحدة التحكم الخاصة بـ JavaScript، لأن متغيرات `const` غير قابلة للتغيير بمجرد تعيينها.

يمكنك إعلان متغير `let` بدون تعيين قيمة له فورًا، ويمكنك تعيين قيمة لاحقًا:

:::interactive_editor

```js
let age;
console.log(age); // undefined
age = 25;
console.log(age); // 25
```

:::

يجب تعيين قيمة للمتغيرات المُعلنة بـ `const` عند وقت الإعلان. إذا حاولت إعلان متغير `const` بدون تعيين قيمة، ستحصل على خطأ:

```js
const age; // Error: Missing initializer in const declaration
```

يجب استخدام `let` عندما تحتاج إلى إعلان متغيرات ستُعاد تعيينها لاحقًا. على سبيل المثال، تتبع نتيجة متغيرة أو تحديث قيمة مع مرور الوقت في برنامجك.

استخدم `const` عندما تريد إعلان متغيرات يجب أن تبقى ثابتة، مثل قيم التهيئة أو الإعدادات التي لا يجب تغييرها عن طريق الخطأ.

يمكنك أيضًا استخدام الكلمة المفتاحية `var`، لكنها لم تعد موصى بها. `var` تشبه `let` إلى حد ما، لكنها تمتلك نطاقًا أوسع، مما قد يسبب مشاكل أكثر في برنامجك.
