# --interactive--

Твердження випередження і випередження ззаду дозволяють співставляти конкретні шаблони залежно від наявності або відсутності навколишніх шаблонів. Існує чотири варіанти цих тверджень.

Перший — позитивне твердження випередження. Це твердження співставить шаблон, якщо за ним слідує інший шаблон.

Щоб створити позитивне твердження випередження, потрібно почати з шаблону, який ви хочете співставити. Потім використайте круглі дужки, щоб обгорнути шаблон, який ви хочете використати як умову. Після відкриваючої дужки використайте `?=`, щоб визначити цей шаблон як позитивне твердження випередження.

Цей шаблон співставить слово `free` лише якщо за ним слідує `code`:

```js
const regex = /free(?=code)/i;
```

Давайте перевіримо поведінку нашого шаблону:

:::interactive_editor

```js
const regex = /free(?=code)/i;
console.log(regex.test("freeCodeCamp")); // true
console.log(regex.test("free code camp")); // false
console.log(
  regex.test("I need someone for free to write code for me")
); // false
```

:::

Зверніть увагу, що тест проходить лише рядок, де `free` безпосередньо слідує за `code`.

Але що, якщо ви хочете співставити наявність `free`, коли за ним НЕ слідує `code`? Ви можете перетворити позитивне твердження випередження на від’ємне, щоб інвертувати поведінку. Для цього замініть `?=` на `?!`:

```js
const regex = /free(?!code)/i;
```

Перевіримо це на тих самих рядках:

:::interactive_editor

```js
const regex = /free(?!code)/i;
console.log(regex.test("freeCodeCamp")); // false
console.log(regex.test("free code camp")); // true
console.log(
  regex.test("I need someone for free to write code for me")
); // true
```

:::

Як і очікувалося, результати інвертовані. Єдиний рядок, який не проходить, — це перший, де `free` безпосередньо слідує за `code`.

Твердження випередження ззаду працюють подібно до тверджень випередження, але замість співставлення залежно від наступного шаблону, вони співставляють залежно від попереднього шаблону. Розглянемо позитивне твердження випередження ззаду.

Позитивне твердження випередження ззаду позначається `?<=` замість `?=`. Зробимо так, щоб наш регулярний вираз співставляв `code`, коли він передує `free`:

```js
const regex = /(?<=free)code/i;
```

Як і з позитивним твердженням випередження, позитивне твердження випередження ззаду співставляє перший рядок, бо `code` безпосередньо передує `free`:

:::interactive_editor

```js
const regex = /(?<=free)code/i;
console.log(regex.test("freeCodeCamp")); // true
console.log(regex.test("free code camp")); // false
console.log(
  regex.test("I need someone for free to write code for me")
); // false
```

:::

Щоб співставити `code`, коли він НЕ передує `free`, можна використати від’ємне твердження випередження ззаду. Від’ємне твердження випередження ззаду визначається заміною `?<=` на `?<!`:

```js
const regex = /(?<!free)code/i;
```

Це співставить будь-яке входження `code`, яке НЕ безпосередньо передує `free`:

:::interactive_editor

```js
const regex = /(?<!free)code/i;
console.log(regex.test("freeCodeCamp")); // false
console.log(regex.test("free code camp")); // true
console.log(
  regex.test("I need someone for free to write code for me")
); // true
```

:::

Пам’ятайте, що `Regex.prototype.test` лише підтверджує, чи співпадає рядок із регулярним виразом. Використаємо наше від’ємне твердження випередження ззаду з `String.prototype.match`, щоб побачити, як твердження впливають на це:

```js
const regex = /(?<!free)code/i;
console.log("freeCodeCamp".match(regex)); // null
console.log("free code camp".match(regex)); // ['code', index: 5, input: 'free code camp', groups: undefined]
console.log(
  "I need someone for free to write code for me".match(regex)
); // ['code', index: 33, input: 'I need someone for free to write code for me', groups: undefined]
```

Зверніть увагу, що хоча наш регулярний вираз використовує твердження випередження ззаду для перевірки наявності `free`, він не співпадає з `free`. Єдиний текст, включений у співпадіння, — це `code`.

Твердження випередження і випередження ззаду надзвичайно корисні для умовного співставлення тексту без впливу на повернуте значення співпадіння.
