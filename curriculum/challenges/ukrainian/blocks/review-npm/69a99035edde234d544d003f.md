## Що таке npm?

- **Визначення**: npm — це онлайн-реєстр для відкритого програмного забезпечення, який функціонує як «магазин застосунків» для коду.
- **Офіційне значення**: npm офіційно не розшифровується як «Node Package Manager»; це рекурсивна абревіатура, що означає «npm не є акронімом».
- **Три пов’язані частини**:

  - **Вебсайт (npmjs.com)**: Для пошуку пакетів у реєстрі.
  - **Реєстр**: Велика публічна база даних, що зберігає всі пакети.
  - **Інтерфейс командного рядка (CLI)**: Інструмент, який встановлюється разом із Node.js для управління залежностями проєкту.

- **Альтернативи**: Хоча npm є найпоширенішим, популярними альтернативами є **Yarn**, **PNPM** та **Bun**.

## Файл package.json

- **Призначення**: Конфігураційний файл, який зберігає метадані, залежності та скрипти для проєкту Node.js.
- **Ключові поля**:

  - `name` та `version`: Ідентифікують проєкт.
  - `main`: Вказує файл вхідної точки, часто `index.js`.
  - `license`: Визначає юридичні умови (за замовчуванням `ISC`).
  - `dependencies`: Пакети, необхідні для запуску застосунку в продакшені.
  - `devDependencies`: Інструменти, потрібні лише під час розробки, наприклад тестові фреймворки.

- **Створення**:

  - `npm init`: Запускає інтерактивний процес налаштування.
  - `npm init -y`: Миттєво встановлює `package.json` з типовими значеннями.

## Семантичне версіонування (SemVer)

- **Формат**: `MAJOR.MINOR.PATCH`.
- **Правила**:

  - **MAJOR**: Зміни, що порушують сумісність і можуть вплинути на існуючий код.
  - **MINOR**: Нові функції, сумісні з попередніми версіями.
  - **PATCH**: Невеликі виправлення помилок і покращення.

- **Символи діапазону**:

  - `^` (карет): Дозволяє оновлення мінорних і патч-версій (наприклад, `^1.2.3` дозволяє `1.x.x`).
  - `~` (тильда): Дозволяє лише патч-оновлення (наприклад, `~1.2.3` дозволяє `1.2.x`).
  - `*` (зірочка): Відповідає будь-якій версії; використовується переважно для тестування.

## Управління залежностями

- **Встановлення**:

  - `npm install <name>`: Додає залежність для продакшену.
  - `npm install <name> -D`: Додає залежність для розробки.
  - `npm install <name>@<version>`: Встановлює конкретну версію.

- **Node Modules і Lock-файли**:

  - `node_modules/`: Містить фактичний код встановлених пакетів.
  - `package-lock.json`: Фіксує точні версії всіх залежностей і дочірніх залежностей для забезпечення консистентності в різних середовищах.

- **Видалення**: `npm uninstall <name>` видаляє пакет і оновлює `package.json`.
- **Обслуговування**:

  - `npm outdated`: Визначає, які пакети мають новіші версії.
  - `npm update`: Оновлює пакети в межах дозволеного діапазону семантичного версіонування.

## Скрипти npm

- **Функція**: Кастомні команди, визначені в розділі `scripts` файлу `package.json`, щоб автоматизувати завдання.
- **Виконання**: Запускаються через `npm run <script_name>`.
- Існують скорочення для `start`, `test`, `stop` та `restart` (наприклад, `npm start`).
- **Послідовне виконання**:
- `&&`: Запускає другу команду лише якщо перша успішна.
- `;`: Виконує команди одна за одною незалежно від успіху.
- **Паралельне виконання**: Використовуйте оператор `&` для запуску процесів у фоновому режимі.
- **Аргументи**: Передавайте аргументи скриптам за допомогою роздільника `--` (наприклад, `npm start -- 8000`).

## Публікація в реєстр npm

- **Доступ до реєстру**: База даних публічних і приватних пакетів.
- **Кроки публікації**:

1. Створіть безкоштовний акаунт на `npmjs.com`.
2. Увійдіть через термінал за допомогою `npm login`.
3. Переконайтеся, що у файлі `name` є унікальний `version` та дійсний `package.json`.
4. Створіть `README.md` та `.npmignore` (щоб виключити конфіденційні файли).
5. Запустіть `npm publish` (або `npm publish --access public` для пакетів зі скоупом).

- **Скоупи**: Використовуються, щоб уникнути конфліктів імен, публікуючи під `@username/package-name`.

## Системи модулів у Node.js

### CommonJS (CJS)

- **Система за замовчуванням**: Початковий спосіб роботи Node.js з модулями; завантажує синхронно.
- **Синтаксис**:
- **Імпорт**: `const module = require('./path')`.
- **Експорт**: `module.exports = { functionName }`.
- **Ідентифікація**: Використовується для файлів `.cjs` або `.js`, якщо в `type` не вказано `package.json`.

### ES Modules (ESM)

- **Стандартизований формат**: Сучасний стандарт JavaScript; завантажується асинхронно для підвищення ефективності.
- **Синтаксис**:

  - **Експорт**: `export { name }` або `export default function`.
  - **Імпорт**: `import { name } from './path.mjs'`.
  - **Ідентифікація**: Використовується для файлів `.mjs` або `.js`, якщо `package.json` містить `"type": "module"`.
