# --description--

**REST**는 표현 상태 전송을 의미합니다. 웹 클라이언트와 서버가 인터넷을 통해 어떻게 소통해야 하는지 설명하는 일련의 지침으로 생각하세요.

이 수업에서는 REST가 무엇인지, 주요 개념, 그리고 어떻게 작동하는지 배우게 됩니다.

REST는 표준 웹 기술을 사용해 서로 다른 시스템이 상호작용하는 방식을 정의합니다. REST를 통해 웹 브라우저, 모바일 앱, API 같은 클라이언트가 이전 수업에서 다룬 `GET`, `POST`, `PUT`, `PATCH`, `DELETE` 같은 일반적인 HTTP 메서드를 사용해 서버와 통신합니다.

REST의 핵심 개념에는 **무상태성**, **자원**, **HTTP 메서드**, 그리고 **응답 또는 상태 코드**가 포함됩니다.

**무상태성**은 클라이언트가 서버에 보내는 각 요청이 독립적으로 처리된다는 뜻입니다. 서버는 이전 요청을 기억하지 않습니다. 따라서 모든 요청에는 처리에 필요한 모든 정보가 포함되어야 합니다. 예를 들어, 사용자가 인증된 경우 인증 토큰을 각 요청에 포함해야 합니다.

REST에서는 모든 것을 자원으로 취급합니다. 자원은 접근하거나 수정할 수 있는 데이터 조각입니다. 각 자원은 `/products` (모든 제품) 또는 `/products/1` (특정 제품) 같은 URL로 식별됩니다.

아래는 HTTP 메서드, 상태 코드, 그리고 REST 아키텍처가 작동하는 단계별 예시 개요입니다.

## HTTP 메서드

HTTP 메서드는 자원에 수행할 작업을 정의합니다:

* `GET`: 자원 조회
    
* `POST`: 새 자원 생성
    
* `PUT / PATCH`: 기존 자원 업데이트
    
* `DELETE`: 자원 삭제
    

## HTTP 상태 코드

요청을 처리한 후 서버는 결과를 나타내는 **상태 코드**로 응답합니다:

* `200` **(OK)**: 요청이 성공함
    
* `201` **(Created)**: 새 자원이 생성됨
    
* `400` **(Bad Request)**: 클라이언트가 잘못된 데이터를 보냄
    
* `404` **(Not Found)**: 요청한 자원이 존재하지 않음
    
* `500` **(Internal Server Error)**: 서버에서 오류 발생
    

## REST 작동 방식: 단계별 예시

### 1\. 클라이언트가 요청을 보냄

* 클라이언트가 `GET` 또는 `POST` 같은 메서드를 사용해 HTTP 요청을 보냄
    
* 요청에는 자원을 식별하는 URL이 포함됨
    
* 헤더, 메타데이터, 요청 본문이 포함될 수도 있음
    

### 2\. 서버가 요청을 받음

* 서버가 요청 형식과 인증(필요한 경우)을 검증함
    
* URL에서 대상 자원을 식별함
    
* HTTP 메서드에 따라 요청된 작업을 결정함
    
* 권한과 접근 제어를 확인함
    

### 3\. 서버가 작업을 수행함

* `GET`: 자원 조회
    
* `POST`: 새 자원 생성
    
* `PUT / PATCH`: 자원 업데이트
    
* `DELETE`: 자원 삭제
    
* 필요한 다른 로직을 실행함
    

### 4\. 서버가 응답을 만듦

* 서버가 적절한 상태 코드(`200`, `201`, 또는 `404` 등)를 선택함
    
* 보통 **JSON** 형식으로 응답 데이터를 포맷함
    
* `Content-Type` 또는 `Cache-Control` 같은 관련 헤더를 추가함
    

### 5\. 서버가 응답을 보냄

* 응답에는 상태 코드, 헤더, 선택적 본문이 포함됨
    
* 서버에는 클라이언트 상태가 저장되지 않음(무상태성)
    

### 6\. 클라이언트가 응답을 처리함

* 클라이언트가 상태 코드를 확인해 성공 여부를 판단함
    
* 서버가 반환한 데이터를 처리함
