# --description--

HTMLの`async`属性と`defer`属性は、`script`要素内でJavaScriptファイルがウェブページに読み込まれ実行される方法において重要な役割を果たします。これらを理解することで、ウェブサイトのパフォーマンスとユーザー体験を向上させることができます。HTMLファイルに`script`を含めると、次のようになります。

```js
<script src="example.js"></script>
```

ブラウザがこの`script`タグを見つけると、HTMLの解析を停止し、スクリプトをダウンロードして実行し、その後HTMLの解析を再開します。大きなスクリプトがある場合、これによりウェブページの読み込みが遅くなることがあります。

ここで`async`属性と`defer`属性の出番です。これらはスクリプトをより効率的に読み込む方法を提供します。

まずは`async`属性から見ていきましょう。

```js
<script src="example.js" async></script>
```

`async`タグに`script`属性を追加すると、ブラウザはスクリプトをダウンロードしている間もHTMLの解析を続けます。スクリプトのダウンロードが完了すると、ブラウザはHTMLの解析を一時停止してスクリプトを実行し、その後HTMLの解析を再開します。これによりページの読み込み速度が大幅に向上します。

ただし、asyncスクリプトはダウンロードが完了したらすぐに実行されるため、望む順序で実行されない場合があります。ここで`defer`属性が役立ちます。`defer`属性は次のようになります。

```js
<script src="example.js" defer></script>
```

`defer`属性は`async`属性に似ています。しかし、deferスクリプトはダウンロード直後に実行されるのではなく、HTMLの解析が完了するまで待機します。さらに、deferスクリプトはHTMLコードに現れる順序で実行されます。

まとめると、実行順序が重要でないスクリプトには`async`を使い、スクリプトを正しい順序で実行する必要がある場合は`defer`を使います。どちらの属性も、スクリプトがバックグラウンドでダウンロードされている間にブラウザがHTMLの解析を続けられるため、ページの読み込み時間を大幅に短縮できます。
