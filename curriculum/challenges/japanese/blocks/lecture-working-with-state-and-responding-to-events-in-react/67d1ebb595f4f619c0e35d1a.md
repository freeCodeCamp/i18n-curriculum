# --description--

Reactで状態のオブジェクトを更新するのは、オブジェクトのプロパティ値を直接変更することに慣れていると難しい場合があります。

Reactは状態を不変として扱うため、直接変更してはいけません。

Reactの状態のオブジェクトを直接変更しようとするとどうなるかを見てから、正しい方法を説明します。

コンポーネントの状態にユーザーのプロフィールを表すオブジェクトがあり、ユーザーが年齢を更新したいとします。

```jsx
import { useState } from "react";

function Profile() {
  const [user, setUser] = useState({
    name: "John Doe",
    age: 31,
    city: "LA",
  });

  // Change user age directly
  const handleAgeChange = (e) => {
    user.age = e.target.value;
    console.log(user);
  };

  return (
    <div>
      <h1>User Profile</h1>
      <p>Name: {user.name}</p>
      <p>Age: {user.age}</p>
      <p>City: {user.city}</p>

      <h2>Update User Age </h2>
      <input type="number" value={user.age} onChange={handleAgeChange} />
    </div>
  );
}

export default Profile;
```

このコードは動作しません。なぜなら`user.age`プロパティを直接変更しているからです。

`console.log(user)`はコンソールに新しい年齢を表示しますが、状態のセッター関数である`setUser`を使っていないため、Reactはコンポーネントを再レンダリングしません。

状態のオブジェクトを直接更新するには、セッター関数を使って更新された値を持つ新しいオブジェクトを作成する必要があります。例えば：

```js
const handleAgeChange = (e) => {
  setUser({
    age: e.target.value,
  });
};
```

これは動作します。しかしページを見ると、ユーザーの年齢は更新されますが、名前と都市の値が失われています。

これは、セッター関数に渡した新しいオブジェクトが`age`のキーと値のペアだけを含んでいるためです。

これを防ぐには、まず既存のオブジェクトをコピーしてから、更新したいプロパティだけを変更します。ここでは`age`です。

これを行うには、セッター関数`setUser`にアップデータ関数という特別な関数を渡します。アップデータ関数は保留中の状態を引数として受け取り、ここでは`prevUser`と呼ばれ、次の状態を返す必要があります。

```js
const handleAgeChange = (e) => {
  setUser((prevUser) => {
    const updatedUser = { ...prevUser, age: e.target.value };
    console.log('Previous State:', prevUser);
    console.log('Updated State:', updatedUser);
    return updatedUser;
  });
};
```

ご覧の通り、スプレッド構文を使って保留中のユーザーオブジェクト`updatedUser`をコピーし、`...prevUser`という新しいユーザーオブジェクトを作成しています。その後、フォームの入力に基づいて年齢を更新し、関数の最後で`updatedUser`を次の状態として返しています。

これでプロジェクトは期待通りに動作し、年齢の入力更新がユーザーの名前や都市名に影響を与えません。コンソールには以前の状態と更新後の状態も表示されます。

これは、特にすべてのプロパティを更新しない場合に、状態のオブジェクトを更新する理想的な方法です。

残りの`name`と`city`のプロパティを更新するには、それぞれ別のセッター関数を書いて対応する入力に接続できます。

```js
const handleNameChange = (e) => {
  setUser((prevUser) => ({
    ...prevUser,
    name: e.target.value,
  }));
};

const handleCityChange = (e) => {
  setUser((prevUser) => ({
    ...prevUser,
    city: e.target.value,
  }));
};
```

または、次のように1つのセッター関数にまとめることもできます。

```js
const handleChange = (e) => {
  const { name, value } = e.target;
  setUser((prevUser) => ({
    ...prevUser,
    [name]: value,
  }));
};
```

これを機能させるには、各入力フィールドに`name`属性が必要です。

現在の完全なコードはこちらです。

```jsx
import { useState } from "react";

function Profile() {
  const [user, setUser] = useState({ name: "John Doe", age: 31, city: "LA" });

  const handleChange = (e) => {
    const { name, value } = e.target;

    setUser((prevUser) => ({...prevUser, [name]: value}));
  };

  return (
    <div>
      <h1>User Profile</h1>
      <p>Name: {user.name}</p>
      <p>Age: {user.age}</p>
      <p>City: {user.city}</p>

      <h2>Update User Age </h2>
      <input type="number" name="age" value={user.age} onChange={handleChange} />

      <h2>Update User Name </h2>
      <input type="text" name="name" value={user.name} onChange={handleChange} />

      <h2>Update User City </h2>
      <input type="text" name="city" value={user.city} onChange={handleChange} />
    </div>
  );
}

export default Profile;
```
