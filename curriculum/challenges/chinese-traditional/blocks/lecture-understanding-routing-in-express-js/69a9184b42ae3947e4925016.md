# --description--

在 Express 中，`express.Router()` 是一個強大的功能，幫助你將路由組織成較小且模組化的部分。當你的應用程式規模變大時，這尤其有用。

你不需要將所有路由都定義在一個大型檔案中，而是可以使用路由器將它們拆分成多個檔案。這樣能讓你的程式碼更容易閱讀、維護和擴展。

`express.Router()` 方法會創建一個新的路由器物件。

路由器就像一個迷你版的 Express 應用程式。它可以處理自己的路由和中介層，但它並不是完整的 Express 應用程式。一旦你在路由器上定義了路由，就可以使用 `app.use()` 將它掛載到你的主應用程式中。

換句話說，路由器讓你能將相關的路由群組在一起，然後再整合到你的主應用程式中。

使用路由器有幾個好處：

**關注點分離**

你可以在不同檔案中定義路由，這樣能提升應用程式的組織性和可讀性。

**程式碼重複使用**  
路由器可以在應用程式的不同部分，甚至跨專案重複使用。

**主檔案更乾淨**  
你不需要在主 `app.js` 或 `server.js` 檔案中塞滿許多路由定義，可以將它們移到路由器中，讓主檔案專注於設定和初始化。

讓我們看一個簡單範例，使用 `express.Router()` 定義 `/users` 路徑的路由。

首先，這是 `app.js` 檔案：

```javascript
const express = require("express");
const app = express();
const port = 3000;

// Importing the user routes
const userRoutes = require("./userRoutes");

// Using the userRoutes router for /users path
app.use("/users", userRoutes);

app.listen(port, () => {
  console.log(`Server running at http://localhost:${port}`);
});
```

接著是 `userRoutes.js`：

```javascript
const express = require("express");
const router = express.Router();

// GET request for /users
router.get("/", (req, res) => {
  res.send("List of users");
});

// POST request for /users
router.post("/", (req, res) => {
  res.send("Creating a new user");
});

// GET request for /users/:id
router.get("/:id", (req, res) => {
  const userId = req.params.id;
  res.send(`User details for user with id: ${userId}`);
});

module.exports = router;
```

現在讓我們仔細看看整個運作流程。

在 `app.js` 中，我們匯入 `express`，並創建一個 Express 應用程式實體，並指定給名為 `app` 的變數。

接著，我們從 `userRoutes` 匯入 `userRoutes.js` 路由器，並使用 `app.use("/users", userRoutes)` 將路由器掛載。這告訴 Express，如果請求的路徑以 `/users` 開頭，就將它傳給 `userRoutes` 路由器。

然後在 `userRoutes.js` 中，我們使用 `express.Router()` 創建一個新的路由器，並使用 `router.get()`、`router.post()` 和 `router.get("/:id")` 定義路由。

請注意，這裡定義的路由是相對於 `/users`，因為路由器就是掛載在那裡。所以：

*   `router.get("/")` → `GET /users`：回應使用者列表
    
*   `router.post("/")` → `POST /users`：回應有關建立新使用者的訊息
    
*   `router.get("/:id")` → `GET /users/:id`：回應指定使用者 ID 的使用者詳細資料
    

使用 `express.Router()` 讓你的主檔案保持乾淨，專注於設定。所有與使用者相關的路由都集中在 `userRoutes.js`。

隨著應用程式成長，你可以為其他區塊創建額外的路由器，例如 `/products`、`/orders` 或 `/auth`。

每個區塊都維持在自己的檔案中，讓應用程式更容易維護和擴展。
