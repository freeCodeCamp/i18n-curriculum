# --description--

React 19에는 서버 컴포넌트와 서버 액션이라는 두 가지 주목할 만한 새로운 기능이 도입되었습니다.

그 버전부터 서버 컴포넌트는 Next.js 같은 프레임워크에서 기본값이 되었습니다.

반면 서버 액션은 API 엔드포인트 없이도 서버에서 바로 폼 처리를 할 수 있도록 서버에서 실행되는 함수입니다.

서버 액션은 다음과 같이 생겼습니다:

```js
"use server";

async function submitForm(formData) {
  const name = formData.get("name");
  return { message: `Hello, ${name}!` };
}
```

이 서버 액션은 폼에서 `name` 필드를 추출하고 그 이름을 인사하는 문자열을 반환합니다.

서버 액션의 상태 관리를 단순화하고 간단한 폼에 클라이언트 측 JavaScript가 필요 없도록 하기 위해 React 팀은 19버전에서 `useActionState` 훅을 도입했습니다.

이 훅을 자세히 살펴보고 어떻게 작동하는지 알아봅시다.

React 문서에서는 `useActionState` 훅을 "폼 액션 결과를 기반으로 상태를 업데이트할 수 있게 해주는 훅"이라고 설명합니다.

하지만 이것이 `useActionState` 훅을 폼에만 사용할 수 있다는 뜻은 아닙니다. 액션이 있다면 버튼 클릭이나 다른 이벤트에도 사용할 수 있습니다.

그리고 `useActionState`가 훅이기 때문에 서버 컴포넌트 내부에서는 사용할 수 없다는 점을 기억하세요.

다음은 `useActionState` 훅의 기본 구문입니다:

```js
const [state, action, isPending] = useActionState(actionFunction, initialState, permalink);
```

- `state`는 액션이 반환하는 현재 상태입니다.

- `action`는 서버 액션을 실행하는 함수입니다.

- `isPending`는 액션이 현재 실행 중인지 여부를 나타내는 불리언입니다.

- `actionFunction` 매개변수는 서버 액션 자체입니다.

- `initialState`는 액션 실행 전 상태의 시작점을 나타내는 매개변수입니다.

- `permalink`는 폼이 수정하는 고유한 페이지 URL을 담은 선택적 문자열입니다.

`useActionState` 훅을 사용하려면 먼저 액션이 있어야 합니다. 이전 예제의 액션을 약간 변형해서 사용해 봅시다:

```js
"use server";

export async function submitForm(_, formData) {
  const name = formData.get("name");

  const hour = new Date().getHours();
  let greeting;

  if (hour < 12) {
    greeting = "Good morning";
  } else if (hour < 18) {
    greeting = "Good afternoon";
  } else {
    greeting = "Good evening";
  }

  return { message: `${greeting}, ${name}` };
}
```

컴포넌트에서는 `useActionState` 훅을 가져와서 컴포넌트 본문 최상단(리턴문 전에)에서 다른 훅처럼 호출해야 합니다. 액션도 가져와야 합니다:

```jsx
"use client";

// Import the useActionState hook
import { useActionState } from "react";

// Import the submitForm action
import { submitForm } from "./actions/submitForm";

const Greeter = () => {

 // Initialize the hook
 const [state, submit, isPending] = useActionState(submitForm, {
   message: "",
 });

  return (
    <div className="flex flex-col items-center justify-center min-h-screen bg-gray-100 p-6">
      {/* Rest of component */}
    </div>
  );
};

export default Greeter;
```

약간의 스타일을 적용한 전체 코드는 다음과 같습니다:

```jsx
"use client";

import { useActionState } from "react";
import { submitForm } from "./actions/submitForm";

const Greeter = () => {
  const [state, submit, isPending] = useActionState(submitForm, {
    message: "",
  });

  return (
    <div className="flex flex-col items-center justify-center min-h-screen bg-gray-100 p-6">
      <form
        action={submit}
        className="bg-white p-6 rounded-2xl shadow-md w-full max-w-md"
      >
        <h2 className="text-2xl text-center font-semibold text-gray-700 mb-4">
          Greet Someone
        </h2>

        <input
          type="text"
          name="name"
          placeholder="Enter your name"
          required
          className="w-full p-3 border border-gray-300 rounded-lg focus:outline-none focus:ring-2 focus:ring-green-400"
        />

        <button
          type="submit"
          disabled={isPending}
          className="w-full mt-4 p-3 bg-green-500 text-white font-semibold rounded-lg hover:bg-green-600 disabled:bg-gray-400 transition-all"
        >
          {isPending ? "Greeting..." : "Greet"}
        </button>

        {state.message && (
          <p className="mt-4 text-green-600 text-center font-medium">
            {state.message}
          </p>
        )}
      </form>
    </div>
  );
};

export default Greeter;
```

브라우저에서는 폼 버튼이 `Greet`에서 `Greeting...`로 바뀌고, 액션이 `isPending`되는 동안 인사가 `Good morning, {name}`, `Good afternoon, {name}`, `Good evening, {name}` 중 하나로 표시됩니다. 이는 폼 제출 시각에 따라 달라집니다.

`useActionState` 훅을 폼 외부에서도 사용할 수 있다고 했던 점 기억하시나요?

이번 예제에서는 버튼 클릭으로 JSONPlaceholder에서 사용자 다섯 명을 가져옵니다:

```js
"use server";

export async function getUsers() {
  const res = await fetch(
    "https://jsonplaceholder.typicode.com/users?_start=0&_limit=5/"
  );
  return await res.json();
}
```

스타일이 적용된 UI는 다음과 같습니다:

```jsx
"use client";

import { useActionState } from "react";
import { getUsers } from "./actions/getUsers";

export default function FetchUsers() {
  const [users, fetchAction, isPending] = useActionState(getUsers, []);

  return (
    <div className="p-6 max-w-lg mx-auto">
      <button
        onClick={fetchAction}
        disabled={isPending}
        className="px-4 py-2 cursor-pointer bg-green-500 text-white rounded-lg hover:bg-green-600 disabled:bg-gray-400 font-bold"
      >
        {isPending ? "Fetching Users..." : "Fetch Users"}
      </button>

      <ul className="mt-4 space-y-2">
        {users.map((user) => (
          <li key={user.id} className="p-3 bg-gray-100 rounded-lg">
            <p className="font-semibold">{user.name}</p>
            <p className="text-sm text-gray-600">{user.email}</p>
          </li>
        ))}
      </ul>
    </div>
  );
}
```

브라우저에서는 버튼 텍스트가 클릭 후에도 `Fetching Users...`로 업데이트되지 않는 것을 볼 수 있습니다.

이것은 React가 데이터 가져오기와 렌더링을 `isPending` 상태보다 우선순위가 높게 처리해서 `isPending`가 차단되고 오류가 발생하기 때문입니다.

이 문제를 해결하려면 액션을 `startTransition`로 감싸야 합니다:

```jsx
"use client";

// import startTransition from React
import { useActionState, startTransition } from "react";
import { getUsers } from "./actions/getUsers";

export default function FetchUsers() {
  const [users, fetchAction, isPending] = useActionState(getUsers, []);

  return (
    <div className="p-6 max-w-lg mx-auto">
      <button
        {/* wrap fetchAction in startTransition */}
        onClick={() => startTransition(() => fetchAction())}
        disabled={isPending}
        className="px-4 py-2 bg-green-500 font-bold cursor-pointer text-white rounded-lg hover:bg-green-600 disabled:bg-gray-400"
      >
        {isPending ? 'Fetching Users...' : 'Fetch Users'}
      </button>

      <ul className="mt-4 space-y-2">
        {users.map((user) => (
          <li key={user.id} className="p-3 bg-gray-100 rounded-lg">
            <p className="font-semibold">{user.name}</p>
            <p className="text-sm text-gray-600">{user.email}</p>
          </li>
        ))}
      </ul>
    </div>
  );
}
```

`startTransition`가 무엇인지 궁금하다면, 이는 React에 상태 업데이트가 낮은 우선순위이며 중단될 수 있음을 알려주는 함수입니다. 이렇게 하면 서버 액션 같은 비동기 업데이트를 처리하는 동안 UI가 반응형으로 유지됩니다.

이것이 폼 안팎에서 `useActionState` 훅을 사용하는 방법입니다.

React에서 `useActionState` 훅의 목적은 무엇인가요?

## --answers--

액션 결과를 기반으로 상태를 관리하며, 폼 제출, 버튼 클릭, 기타 이벤트 기반 업데이트를 포함합니다.

---

폼 제출 시에만 상태를 업데이트하며 버튼 클릭이나 사용자 정의 이벤트 핸들러 같은 다른 액션은 지원하지 않습니다.

왜 서버 컴포넌트 내부에서 `useActionState` 훅을 사용할 수 없나요?

## --answers--

서버 컴포넌트는 상태 관리를 지원하지 않아 `useActionState`와 호환되지 않기 때문입니다.

### --feedback--

훅은 특정 유형의 컴포넌트에서만 작동합니다.

---

`useActionState`가 데이터베이스 연결을 필요로 하는데, 이는 클라이언트 컴포넌트에서만 가능하기 때문입니다.

### --feedback--

훅은 특정 유형의 컴포넌트에서만 작동합니다.

---

`useActionState`는 React 훅이며, 훅은 클라이언트 컴포넌트 내부에서만 사용할 수 있기 때문입니다.

---

`useActionState`가 서버에서 사용할 수 없는 브라우저 API에 의존하기 때문입니다.
