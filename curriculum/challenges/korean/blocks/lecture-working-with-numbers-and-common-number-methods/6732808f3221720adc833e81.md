# --interactive--

`parseFloat()`와 `parseInt()`는 문자열을 숫자로 변환할 때 JavaScript에서 꼭 필요한 두 가지 메서드입니다. 이 메서드들은 사용자 입력을 처리하거나 문자열 형식으로 들어오는 데이터를 숫자 값으로 다뤄야 할 때 특히 유용합니다.

먼저 `parseFloat()`부터 살펴보겠습니다. 이 메서드는 문자열 인수를 파싱하여 부동 소수점 숫자를 반환합니다. 문자열에 숫자가 아닌 문자가 뒤에 있더라도 문자열의 시작 부분에서 숫자를 추출하도록 설계되었습니다. 부동 소수점 숫자는 소수점이 있는 숫자라는 점을 기억하세요. `parseFloat()`가 작동하는 방식은 다음과 같습니다.

:::interactive_editor

```js
console.log(parseFloat("3.14"));     // 3.14
console.log(parseFloat("3.14 abc")); // 3.14
console.log(parseFloat("3.14.5"));   // 3.14
console.log(parseFloat("abc 3.14")); // NaN
```

:::

보시다시피 `parseFloat()`는 문자열의 시작부터 파싱을 시작해 부동 소수점 숫자의 일부가 될 수 없는 문자를 만나면 멈춥니다. 만약 문자열 시작 부분에서 유효한 숫자를 찾지 못하면 `NaN` (숫자가 아님)를 반환합니다.

반면 `parseInt()`는 문자열 인수를 파싱하여 정수를 반환합니다. `parseFloat()`처럼 문자열의 시작부터 파싱하지만, 첫 번째 숫자가 아닌 문자를 만나면 정지를 합니다. `parseInt()`가 작동하는 방식은 다음과 같습니다.

:::interactive_editor

```js
console.log(parseInt("42"));       // 42
console.log(parseInt("42px"));     // 42
console.log(parseInt("3.14"));     // 3
console.log(parseInt("abc123"));   // NaN
```

:::

`parseInt()`는 첫 번째 숫자가 아닌 문자를 만나면 파싱을 멈춥니다. 부동 소수점 숫자의 경우 정수 부분만 반환합니다. 만약 문자열 시작 부분에서 유효한 정수를 찾지 못하면 `NaN`을 반환합니다.

두 메서드 모두 주목할 만한 동작이 있습니다. 앞쪽 공백은 무시합니다.

:::interactive_editor

```js
console.log(parseFloat("  3.14"));  // 3.14
console.log(parseInt("  42"));      // 42
```

:::

문자열 시작 부분의 플러스와 마이너스 부호도 처리합니다.

:::interactive_editor

```js
console.log(parseFloat("+3.14"));  // 3.14
console.log(parseInt("-42"));      // -42
```

:::

이 메서드들이 강력하지만 몇 가지 한계가 있다는 점도 알아두세요. 예를 들어, 과학적 표기법 같은 모든 숫자 형식을 직접 처리하지는 못합니다. 더 복잡한 파싱이 필요할 때는 추가 기법이나 라이브러리를 사용해야 할 수도 있습니다.

결론적으로 `parseFloat()`와 `parseInt()`는 JavaScript에서 문자열을 숫자로 변환할 때 매우 유용한 도구입니다. 이들이 어떻게 작동하는지, 그리고 각각의 구체적인 동작 방식을 이해하면 사용자 입력이나 외부 데이터 소스를 다룰 때 숫자 데이터를 더 효과적으로 처리할 수 있습니다.
