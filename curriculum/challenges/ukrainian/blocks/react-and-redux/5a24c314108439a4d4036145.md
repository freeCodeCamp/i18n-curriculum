# --description--

Компонент `Provider` дозволяє надавати `state` і `dispatch` вашим React-компонентам, але ви маєте точно вказати, до якого стану і яких дій ви хочете мати доступ. Таким чином ви гарантуєте, що кожен компонент матиме доступ лише до того стану, який йому потрібен. Ви досягаєте цього, створюючи дві функції: `mapStateToProps()` і `mapDispatchToProps()`.

У цих функціях ви оголошуєте, до яких частин стану хочете мати доступ і які генератори дій вам потрібно мати змогу викликати. Коли ці функції будуть готові, ви побачите, як використати метод `connect` React Redux для підключення їх до ваших компонентів у іншому завданні.

**Примітка:** Під капотом React Redux використовує метод `store.subscribe()` для реалізації `mapStateToProps()`.

# --instructions--

Створіть функцію `mapStateToProps()`. Ця функція має приймати `state` як аргумент, а потім повертати об’єкт, який спроєктує цей стан на конкретні імена властивостей. Ці властивості стануть доступними вашому компоненту через `props`. Оскільки в цьому прикладі весь стан застосунку зберігається в одному масиві, ви можете передати цей увесь стан вашому компоненту. Створіть властивість `messages` в об’єкті, що повертається, і призначте їй значення `state`.

# --hints--

Константа `state` має бути порожнім масивом.

```js
assert(Array.isArray(state) && state.length === 0);
```

`mapStateToProps` має бути функцією.

```js
assert(typeof mapStateToProps === 'function');
```

`mapStateToProps` має повертати об’єкт.

```js
assert(typeof mapStateToProps() === 'object');
```

Передача масиву як стану до `mapStateToProps` має повертати цей масив, призначений ключу `messages`.

```js
assert(mapStateToProps(['messages']).messages.pop() === 'messages');
```
