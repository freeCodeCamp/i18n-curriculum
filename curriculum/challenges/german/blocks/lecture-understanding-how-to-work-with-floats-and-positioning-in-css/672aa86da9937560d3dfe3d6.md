# --interactive--

Floats in CSS sind eine Technik, die ursprünglich entwickelt wurde, um Text um ein Element, wie zum Beispiel ein Bild, herumfließen zu lassen. Im Laufe der Zeit haben Entwickler jedoch neue Einsatzmöglichkeiten für Floats gefunden und sie kreativ im Layout-Design angewendet. Obwohl moderne Layout-Methoden wie Flexbox und Grid heute häufiger verwendet werden, ist das Verständnis von Floats weiterhin wichtig, insbesondere beim Arbeiten mit älterem Code oder wenn bestimmte Layout-Effekte erzielt werden sollen.

Wenn ein Element gefloatet wird, wird es aus dem normalen Dokumentenfluss genommen und an die linke oder rechte Seite seines Containers geschoben. Der folgende Inhalt fließt um das gefloatete Element herum und füllt den verbleibenden Platz aus. Ein klassischer Anwendungsfall ist das Umfließen von Text um Bilder, bei dem ein Bild an eine Seite gefloatet wird und der Text sich darum legt. Diese Technik wird immer noch häufig verwendet, besonders in Artikeln und Blogs, wo Bilder neben Text platziert werden müssen. Hier ist das Codebeispiel:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<div class="container">
  <img src="https://placehold.co/150x150" alt="Placeholder Image">
</div>
```

```css
.container {
  border: 1px solid black;
  padding: 10px;
}

img {
  float: left;
  margin-right: 20px;
}
```

:::

Im obigen Beispiel umschließt der Container das gefloatete Bild nicht. Das Bild befindet sich außerhalb des normalen Dokumentenflusses, und der Container kollabiert auf eine Höhe von null, weil er die gefloateten Kindelemente nicht „sieht“.

Floats waren auch beliebt, um mehrspaltige Layouts zu erstellen, bevor Flexbox und Grid verbreitet waren. Indem Elemente nebeneinander gefloatet wurden, konnten Entwickler Spalten erzeugen, die horizontal ausgerichtet sind. Beim Einsatz von Floats ist es jedoch wichtig, das Problem des Kollabierens von Elternelementen zu behandeln, wenn deren Kindelemente gefloatet sind. Die clearfix-Technik wird auf das Element der Klasse `container` angewendet, um dieses Problem zu beheben.

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<div class="container">
  <img src="https://placehold.co/150x150" alt="Placeholder Image">
  <p> This is an example of text flowing around a floated image.</p>
</div>
```

```css
.container {
  border: 1px solid black;
}

/* Clearfix CSS */
.container::after {
  content: "";  
  display: block;
  clear: both;
}

img {
  float: left;
  margin-right: 20px;
}
```

:::

- `::after` ist ein Pseudoelement, das nach dem Inhalt des Containers einen unsichtbaren Block hinzufügt.
- `content: ""` stellt sicher, dass das Pseudoelement vorhanden ist, aber keinen Inhalt anzeigt.
- `display: block` macht das Pseudoelement zu einem Block-Element.
- `clear: both` sorgt dafür, dass das Pseudoelement beide Seiten von gefloateten Elementen darüber freiräumt.

Im obigen Beispiel haben wir ein neues Absatz-Element hinzugefügt, um den Kollaps deutlicher zu machen. Da der Absatz im normalen Dokumentenfluss bleibt, erweitert sich der Container gerade so weit, dass er ihn umschließt. Anschließend wenden wir die clearfix-Technik an, um den Kollaps zu beheben und den Rahmen des Containers korrekt anzuzeigen. Sie können versuchen, die clearfix-Regel zu entfernen, um den Kollaps erneut zu sehen.

Die clearfix-Technik sorgt dafür, dass das Elternelement seine gefloateten Kindelemente richtig umschließt. Clearfix zwingt den übergeordneten Container, die gefloateten Kindelemente „zu sehen“, indem nach dem gefloateten Inhalt eine clear-Eigenschaft hinzugefügt wird.

Obwohl Floats heute nicht mehr die bevorzugte Methode für komplexe Layouts sind, da modernere Techniken wie Flexbox und Grid zur Verfügung stehen, spielen sie in bestimmten Szenarien weiterhin eine wichtige Rolle. Ob Sie Text um Bilder fließen lassen oder an veralteten Projekten arbeiten – das Verständnis, wie Floats funktionieren und wie man sie richtig cleared, ist entscheidend für gut strukturierte und responsive Layouts.
