# --description--

Das Plus `+` existiert in zwei Formen: die binäre Form, die Sie in der letzten Aufgabe verwendet haben, und die unäre Form.

Das unäre Plus oder anders gesagt, der Plusoperator `+`, der auf einen einzelnen Wert angewendet wird, bewirkt bei Zahlen nichts. Aber wenn der Operand keine Zahl ist, wandelt das unäre Plus ihn in eine Zahl um.

Zum Beispiel:

```js
// No effect on numbers
let x = 1;
console.log( +x ); // 1

let y = -2;
console.log( +y ); // -2

// Converts non-numbers
console.log( +true ); // 1
console.log( +"" );   //
```

Die Notwendigkeit, Strings in Zahlen umzuwandeln, tritt sehr häufig auf. Zum Beispiel, wenn Sie Werte aus HTML-Formularfeldern erhalten, sind diese normalerweise Strings. Was ist, wenn Sie sie summieren möchten?

Das binäre Plus würde sie als Strings addieren:

```js
let apples = "2";
let oranges = "3";

alert( apples + oranges ); // "23", the binary plus concatenates strings
```

Wenn Sie sie als Zahlen behandeln möchten, müssen Sie sie zuerst umwandeln und dann summieren:

```js
let apples = "2";
let oranges = "3";

// both values converted to numbers before the binary plus
alert( +apples + +oranges ); // 5

// the longer variant
// alert( Number(apples) + Number(oranges) ); // 5
```

Aus Sicht eines Mathematikers mag die Vielzahl der Pluszeichen seltsam erscheinen. Aus Sicht eines Programmierers ist das jedoch nichts Besonderes: Unäre Pluszeichen werden zuerst angewendet, sie wandeln Strings in Zahlen um, und dann summiert das binäre Plus sie.

Warum werden unäre Pluszeichen vor den binären auf Werte angewendet? Wie Sie sehen werden, liegt das an ihrer _höheren_ Priorität.

# --questions--

## --text--
Angesichts der Erklärung der unären und binären `+`-Operatoren in JavaScript, wie können Sie die String-Werte `'2'` und `'3'` korrekt als Zahlen mit unärem `+` summieren?

## --answers--

`console.log('2' + '3'); // Outputs: '23'` 

---

`console.log(+2 + +3); // Outputs: 5`

---

`console.log(+'2' + +'3'); // Outputs: 5`

---

`console.log(Number('2') + Number('3')); // Outputs: '23'`


## --video-solution--

3
