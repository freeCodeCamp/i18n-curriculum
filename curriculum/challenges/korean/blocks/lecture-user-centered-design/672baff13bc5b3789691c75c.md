# --interactive--

**참고**: 일부 인터랙티브 예제는 아직 배우지 않은 CSS를 사용할 수 있습니다. 모든 코드를 이해하려고 걱정하지 마세요. 예제의 목적은 이러한 디자인 개념을 미리 보여줘서 작동 방식을 더 잘 이해하도록 돕는 것입니다. 미리 보기를 보려면 인터랙티브 편집기를 활성화하세요.

웹 디자인에서는 다양한 유형의 내비게이션 도구를 사용할 수 있습니다. 예를 들어 상단 내비게이션 바, 사이드바, 푸터 등이 있습니다. 하지만 사이트가 더 복잡하고 내비게이션 레벨이 깊다면 빵 부스러기 사용을 고려해 보세요.

빵 부스러기는 사용자가 사이트 계층 구조에서 현재 위치를 보여주는 내비게이션 도구입니다. 다음은 가상의 전자제품 웹사이트에서 빵 부스러기가 어떻게 보이는지 예시입니다:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<div class="breadcrumbs">
  Home / Electronics / Phones / Smartphone XYZ
</div>
```

```css
.breadcrumbs {
  font-size: 0.9rem;
  color: #555;
  margin: 1rem 0;
}
```

:::

대부분의 웹사이트에서 빵 부스러기는 페이지 상단에 표시되어 사용자가 현재 페이지에 도달하기까지 거친 경로를 보여줍니다. `Homepage`에서 시작해 사용자가 `Electronics` 카테고리, 그다음 `Phones` 카테고리, 마지막으로 `Smartphone XYZ` 제품으로 이동했습니다. 제품이나 특정 정보를 찾으면서 웹사이트에서 빵 부스러기를 사용해 본 적이 있을 것입니다.

빵 부스러기 사용은 사용자가 사이트 계층 구조에서 자신의 위치를 이해하고 이전 페이지로 쉽게 돌아갈 수 있게 도와주기 때문에 유용합니다. 특히 사용자가 검색 결과나 외부 링크에서 왔을 때 현재 페이지의 맥락을 이해하는 데 도움이 됩니다.

빵 부스러기를 디자인할 때 고려할 점이 몇 가지 있습니다. 첫 번째는 구분자를 무엇으로 할지 결정하는 것입니다. 구분자는 계층의 각 레벨을 구분하는 문자입니다. 일반적인 구분자는 부등호(`>`), 오른쪽 꺾쇠 괄호(`»`), 슬래시(`/`) 등이 있습니다.

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<div class="breadcrumbs">
  Home &gt; Electronics &gt; Phones &gt; Smartphone XYZ
</div>
```

```css
.breadcrumbs {
  font-size: 0.9rem;
  color: #555;
  margin: 1rem 0;
}
```

:::

두 번째 고려 사항은 빵 부스러기의 위치입니다. 빵 부스러기는 일반적으로 페이지 상단, 메인 내비게이션 바 위나 아래에 배치합니다. 사용자가 빵 부스러기를 찾느라 애쓰지 않도록 눈에 잘 띄고 쉽게 찾을 수 있게 하세요.

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<nav class="main-nav">
  <a href="#">Home</a>
  <a href="#">Electronics</a>
  <a href="#">Phones</a>
</nav>
<div class="breadcrumbs">
  Home &gt; Electronics &gt; Phones &gt; Smartphone XYZ
</div>
```

```css
.main-nav {
  background-color: #333;
  padding: 1rem;
}

.main-nav a {
  color: white;
  margin-right: 1rem;
  text-decoration: none;
}

.breadcrumbs {
  font-size: 0.9rem;
  color: #555;
  margin: 1rem 0;
}
```

:::

또 다른 고려 사항은 빵 부스러기의 크기입니다. 빵 부스러기가 읽기 쉽도록 충분히 커야 하지만, 페이지에서 너무 많은 공간을 차지하지 않도록 해야 합니다. 빵 부스러기는 기본 내비게이션 도구가 아니라 보조 내비게이션 도구임을 기억하세요. 페이지에 정보가 많을 때 사용자는 빵 부스러기를 통해 자신의 위치를 쉽게 파악하고 이전 페이지로 돌아갈 수 있습니다.

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<div class="breadcrumbs">
  Home &gt; Electronics &gt; Phones &gt; Smartphone XYZ
</div>
<p>
  Lorem ipsum dolor sit amet, consectetur adipiscing elit. Sed do eiusmod tempor incididunt ut labore et dolore magna aliqua.
</p>
```

```css
.breadcrumbs {
  font-size: 1rem;
  color: #555;
  margin: 1rem 0;
}   

p {
  font-size: 1.2rem;
  line-height: 1.6;
}
```

:::

그렇다면 언제 사이트에 빵 부스러기를 사용해야 할까요? 사이트가 단순하고 계층 구조가 얕다면 빵 부스러기가 필요하지 않을 수 있습니다. 얕은 계층 구조란 내비게이션 레벨이 몇 개뿐인 경우를 말하며, 이때 빵 부스러기는 큰 도움이 되지 않을 수 있습니다. 보통 전자상거래 사이트, 뉴스 사이트, 기술 문서 사이트처럼 계층 구조가 깊은 사이트에서 빵 부스러기를 볼 수 있습니다.
