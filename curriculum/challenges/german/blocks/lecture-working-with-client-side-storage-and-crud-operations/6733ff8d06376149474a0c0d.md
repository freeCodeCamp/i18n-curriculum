# --description--

Die `Cache`-API wird verwendet, um Netzwerk-Anfragen und -Antworten zu speichern, wodurch Webanwendungen effizienter arbeiten und sogar offline funktionieren können. Sie ist Teil der umfassenderen Service-Worker-API und entscheidend für die Erstellung von Progressive Web Apps (PWAs), die unter unzuverlässigen oder langsamen Netzwerkbedingungen funktionieren können.

Bevor wir mit der `Cache`-API fortfahren, müssen wir zunächst verstehen, wie Service Worker, PWAs und Netzwerk-Anfragen auf grundlegender Ebene funktionieren.

Eine Netzwerk-Anfrage ist eine Anfrage, die von einem Webbrowser oder einer Anwendung an einen Server gesendet wird, um Daten oder Ressourcen über das Internet abzurufen. Zum Beispiel sendet Ihr Browser beim Besuch einer Webseite eine Netzwerk-Anfrage an den Webserver, um die Dateien (wie HTML, Bilder oder Videos) zu erhalten, die zum Anzeigen der Seite benötigt werden.

Ein Service Worker ist eine spezielle Art von JavaScript-Datei, die im Hintergrund einer Webanwendung läuft, getrennt vom Haupt-Thread des Browsers. Er fungiert als Vermittler zwischen der Webseite und dem Netzwerk, wodurch Entwickler Netzwerk-Anfragen abfangen, Ressourcen zwischenspeichern und Dinge wie Push-Benachrichtigungen oder Hintergrund-Synchronisation verwalten können.

Eine PWA, oder Progressive Web App, ist eine Art von Webanwendung, die moderne Webtechnologien nutzt, um ein natives App-ähnliches Erlebnis im Web zu bieten. PWAs sind so konzipiert, dass sie auf jedem Gerät zuverlässig funktionieren, unabhängig von den Netzwerkbedingungen, und können über einen Browser aufgerufen oder wie eine traditionelle App auf dem Gerät eines Nutzers installiert werden.

Jetzt, da wir ein besseres Verständnis von Service Workern, PWAs und Netzwerk-Anfragen haben, können wir tiefer in die Cache-API eintauchen.

Die `Cache`-API ist ein Speichermedium, das `Request`- und `Response`-Objekte speichert. Wenn eine Anfrage an einen Server gestellt wird, kann die Anwendung die Antwort speichern und später aus dem Cache abrufen, anstatt eine neue Netzwerk-Anfrage zu stellen. Das reduziert Ladezeiten, spart Bandbreite und verbessert das gesamte Benutzererlebnis.

Der Browser stellt einen Speicherbereich bereit, der als `CacheStorage` bekannt ist, in dem Entwickler Schlüssel-Wert-Paare von Netzwerk-Anfragen und deren entsprechenden Antworten speichern können.

Mit dem `Cache-Control`-Header können Entwickler angeben, wie lange eine zwischengespeicherte Ressource gespeichert bleiben soll und ob sie neu validiert oder direkt aus dem Cache ausgeliefert werden soll.

Durch die Verwendung der `Cache`-API können Entwickler Offline-zuerst-Webanwendungen erstellen, die es einer PWA beispielsweise ermöglichen, zwischengespeicherte Ressourcen bereitzustellen, wenn der Nutzer keine Netzwerkverbindung hat.
