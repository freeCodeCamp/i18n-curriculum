# --description--

你已經了解了 `HTMLMediaElement` 和 MediaStreams API，但還有一些 API 在處理影片和音訊時相當有用。

第一個是螢幕擷取 API。如其名稱所示，這個 API 允許你記錄使用者的螢幕。這個 API 是透過呼叫 `mediaDevices` 物件的 `getDisplayMedia()` 方法並使用傳回的媒體資料流來暴露的。

接著是 MediaStream Recording API。這個 API 與 MediaStreams API 協同運作，允許你錄製 MediaStream（甚至直接錄製 `HTMLMediaElement`）。然後它會觸發帶有 Blob 載荷的 `dataavailable` 事件，你可以將這些載荷寫入區域的檔案儲存。

這些技術的基礎是 Media Source Extensions API。Media Source Extensions API 允許你直接將使用者的網路攝影機串流透過 `srcObject` 屬性傳遞給影片元素，例如。 

最後，是驅動網頁上所有可聽見內容的 Web Audio API。這個 API 包含重要的物件，例如 `AudioBuffer`（表述專門包含音訊資料的 Buffer）或 `AudioContext`。 

這就結束我們關於音訊和影片、視訊的課程！
