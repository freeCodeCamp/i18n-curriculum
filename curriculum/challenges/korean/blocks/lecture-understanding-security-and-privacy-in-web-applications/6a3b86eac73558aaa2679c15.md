# --description--

쿠키는 웹사이트가 사용자의 기기에 저장하는 작은 텍스트 파일로, 사용자의 정보를 기억하기 위해 사용됩니다.

쿠키를 받고 저장하는 과정은 다음과 같습니다:

- 웹사이트를 방문합니다
- 웹사이트 서버가 응답에 `Set-Cookie` 헤더를 보냅니다
- 브라우저가 헤더를 받아 쿠키 파일을 컴퓨터 하드 드라이브에 저장합니다
- 이후 해당 웹사이트를 방문할 때마다 브라우저가 자동으로 쿠키를 서버에 다시 보냅니다

이 과정 덕분에 웹사이트는 로그인 세션, 장바구니 항목, 환경설정 및 기타 중요한 정보를 기억할 수 있습니다.

이 저장 방식은 심각한 보안 위험을 초래합니다. 해커가 기기에 물리적으로 접근하거나 악성코드로 로컬 파일을 읽으면 세션 쿠키를 훔칠 수 있습니다.

도난당한 세션 쿠키로 공격자는 비밀번호를 몰라도 완전히 다른 컴퓨터에서 계정에 로그인할 수 있습니다.

사용자를 보호하고 쿠키 도난을 막기 위해 개발자는 쿠키 설정 시 특정 보안 플래그를 사용합니다:

`HttpOnly` 플래그는 JavaScript가 쿠키 파일을 읽지 못하게 차단합니다. 이 제한은 크로스 사이트 스크립팅 공격으로 세션 토큰을 훔치는 것을 막습니다.

`Secure` 플래그는 브라우저가 쿠키를 암호화된 HTTPS 연결에서만 전송하도록 보장합니다. 이는 공용 Wi-Fi 등에서 쿠키를 도청하는 악성 행위를 방지합니다.

`SameSite` 플래그는 브라우저가 교차 사이트 요청 시 쿠키 전송을 제한해 크로스 사이트 요청 위조 공격을 차단합니다. 두 가지 일반적인 값이 있습니다:

- `SameSite=Strict`는 모든 교차 사이트 요청에서 쿠키 전송을 차단합니다.
- `SameSite=Lax`는 링크 클릭 같은 안전한 최상위 탐색에서는 쿠키를 허용하지만, 교차 사이트 폼 제출과 API 요청에서는 차단합니다.

쿠키는 개인정보에도 큰 영향을 미칩니다. 1자 쿠키는 사용자가 방문한 특정 웹사이트에서만 행동을 추적하지만, 3자 쿠키는 완전히 다른 여러 웹사이트에서 사용자를 추적합니다.

광고 네트워크는 수천 개 인기 사이트에 이 쿠키를 배치합니다. 인터넷을 탐색할 때 광고 네트워크는 같은 3자 쿠키를 반복해서 읽어 사용자의 관심사, 검색 기록, 온라인 습관에 대한 상세 프로필을 만듭니다.

이 보이지 않는 추적은 사용자의 적극적인 인지 없이 기업이 디지털 생활을 감시할 수 있게 해 큰 개인정보 문제를 만듭니다.

이러한 위협 때문에 GDPR 같은 개인정보 보호법은 웹사이트가 쿠키 동의 배너를 표시해 쿠키 사용 내역을 알리도록 요구합니다. 이 법은 추적 쿠키를 거부할 법적 권리를 제공합니다.

또한 최신 브라우저는 기본적으로 3자 추적 쿠키를 차단해 사용자의 자동 개인정보 보호를 지원합니다.
