# --description--

SQL 是結構化查詢語言的縮寫。它於 1970 年代開發，當時最初稱為結構化英文查詢語言（SEQUEL）。這個術語後來縮短為 SQL。

SQL 是用於在關聯式資料庫中儲存和管理資料的程式設計語言。

如你在先前的課程中所學，關聯式資料庫將資料組織成表格。每個表格都有一組列和行，其中表述資料屬性及其關係。每列表現一筆記錄，而每行表現該記錄的特定屬性或欄。

使用 SQL，你可以為你的關聯式資料庫添加、變更、刪除、尋找和擷取資料。它提供一整套用於查詢、篩選、排序和彙總資料的指令。

SQL 可以透過只查詢必要的資料來最佳化資料庫效率，並且可以輕鬆地與各種程式設計語言整合，讓你能夠直接從你的應用程式與資料庫互動。

它也已被採用為產業標準。它強制執行資料完整性，並包含許多安全特性，例如支援使用者驗證和資料加密。

它同時具備可擴充性、可攜性，並且與廣泛範圍的資料庫管理系統相容。

SQL 是基於指令，通常稱為 SQL 述語或 SQL 查詢。

作為開發者，你將使用特定的 SQL 語言元素或關鍵字來撰寫這些述語。它們允許你對你的資料庫執行必要的操作。

在你可以輸入 SQL 指令之前，你需要開啟 `psql` shell，這樣你才能直接與 PostgreSQL 資料庫互動。在命令提示字元或終端機中，輸入：

```bash
psql -U <username> -d <database_name>
```

將 `username` 取代為你的使用者名稱，將 `database_name` 取代為你想連接的資料庫。

如果你還沒有創建自己的資料庫，PostgreSQL 在安裝時會自動包含一個名為 `postgres` 的預設資料庫。你可以使用這個 `postgres` 資料庫來跟著操作。

連接後，你會看到提示變更為：

```bash
postgres=#
```

它顯示你所連接的資料庫名稱並等待 SQL 指令。

讓我們來看看你應該熟悉的一些最基本的 SQL 指令。你會注意到，依照慣例，SQL 指令以分號（`;`）結尾。

首先，你可以使用此命令來創建名為 `my_database` 的資料庫：

```sql
CREATE DATABASE my_database;
```

你可以輸入 `CREATE`，接著是 `DATABASE`，以及你想要創建的資料庫名稱。請注意，該指令的 SQL 關鍵字是以大寫字母書寫。

然後，一旦你有了你的資料庫，你就可以連接到它。

你如何連接到資料庫，實際上取決於你使用的環境和工具。

在命令提示字元或終端機中，你可以使用先前顯示的方法。在互動式 `psql` 殼層中，你可以使用 `\c` 快捷命令，後面接資料庫名稱，這樣切換資料庫：

```bash
\c my_database
```

請注意，` \c` 指令和其他快捷指令（前面帶有反斜線的指令）是 psql shell 的零件，而不是 SQL 語言本身的零件，因此它們不需要以分號來結束指令。

既然你已經連接到你的新資料庫，你將會看到提示已經改變：

```bash
my_database=#
```

一旦你已經創建並連接到資料庫，你就可以使用 `CREATE TABLE` 關鍵字來創建資料表：

```sql
CREATE TABLE products (
  id SERIAL,
  name VARCHAR(255)
);
```

在指令中，你撰寫 `CREATE TABLE`，接著是你想要創建的資料表名稱。在這個例子中，它將創建一個名為 `products` 的資料表。

然後，在小括弧內，你寫出資料表應該擁有的行名稱，並指定將儲存在每個行中的值的資料型別。

在範例中，表格將有兩個行，一個是產品 ID 的行，另一個是產品名稱的行。

資料表和行名稱的標準命名慣例是使用蛇形命名法，將字、單字以小寫書寫並用底線（`_`）分隔。例如：`delivery_orders`

這些是你應該知道以開始使用的幾個基本 SQL 指令。

使用 SQL，你可以輕鬆查詢、操作和分析資料，以做出明智的決策並解決現實世界的問題。
