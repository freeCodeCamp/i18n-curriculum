# --description--

エラーのもう一つの重要な部分はスタックトレースです。これは、アプリケーション内でエラーがスローされたタイミングや、そのエラーに至るまでにどの関数が呼び出されたかを理解するのに役立ちます。例えば、次のコードがあるとします。

```javascript
const a = 5;
const b = 10;

function add() {
  return c;
}

function print() {
  add();
}

print();
```

私たちの関数`print()`は`add()`を呼び出すべきですが、`add()`は`c`という変数を返します。この変数は現在宣言されていません。対応するエラーは次の通りです。

<img src="https://cdn.statically.io/gh/TheOdinProject/curriculum/284f0cdc998be7e4751e29e8458323ad5d320303/foundations/javascript_basics/understanding_errors/imgs/01.png" width="100%" alt="開発者コンソールに表示されたスタックトレース付きのエラー">

スタックトレースは次のことを教えてくれます：

1. `c is not defined`は`add()`のスコープ内にあり、これは5行目で宣言されています。
1. `add()`は`print()`によって呼び出され、これは9行目で宣言されています。
1. `print()`自体は12行目で呼び出されました。

このようにスタックトレースはエラーの発生源までの経緯をたどることができ、ここでは`add()`の宣言が起点となっています。
