# --interactive--

JavaScript에서 여러 연산자가 포함된 표현식을 작성하면, JavaScript는 어떤 연산자를 먼저 평가할지 어떻게 결정할까요? 바로 이때 연산자 우선순위가 작동합니다. 코드 예제와 함께 연산자 우선순위를 자세히 살펴보고, 우선순위가 같은 연산자가 있을 때는 어떻게 되는지도 알아보겠습니다.

연산자 우선순위는 표현식에서 연산이 평가되는 순서를 결정합니다. 우선순위가 높은 연산자가 낮은 연산자보다 먼저 평가됩니다. 수학에서 나누기와 곱셈이 덧셈과 뺄셈보다 먼저 일어나는 것과 비슷하게 생각하세요.

이 규칙을 따르지 않으면 기본적인 방정식도 잘못된 답을 내게 됩니다. JavaScript도 마찬가지입니다. 어떤 연산자가 먼저, 두 번째로, 그다음으로 오는지에 대한 자체 규칙이 있습니다. 예를 들어, 아래 표현식을 보세요:

:::interactive_editor

```js
const result = 2 + 3 * 4;

console.log(result); // 14
```

:::

JavaScript가 이 표현식을 왼쪽에서 오른쪽으로 평가한다면 `2 + 3 = 5`가 먼저, 그다음 `5 * 4 = 20`가 평가될 것으로 예상할 수 있습니다. 하지만 곱셈이 덧셈보다 우선순위가 높기 때문에 JavaScript는 `3 * 4` 부분을 먼저 평가하여 `2 + 12 = 14` 결과를 얻습니다.

때로는 우선순위 규칙과 상관없이 표현식의 특정 부분을 먼저 실행하고 싶을 수 있습니다. 이럴 때는 소괄호(`()`)를 사용하세요. 소괄호 안에 있는 내용은 무엇이든 우선적으로 평가됩니다. 이전 예제에서 `2 + 3` 부분을 먼저 평가하도록 만들어 보겠습니다:

:::interactive_editor

```js
const result = (2 + 3) * 4;

console.log(result); // 20
```

:::

위 예제에서 소괄호는 JavaScript가 `2 + 3`를 먼저 평가하도록 강제하고, 그 결과에 `4`를 곱하게 만듭니다. 그래서 `20` 대신 `14` 결과가 나옵니다.

나누기 연산자도 덧셈이나 뺄셈보다 우선순위가 높습니다:

:::interactive_editor

```js
const result = 2 + 6 / 3;

console.log(result); // 4
```

:::

JavaScript가 이 표현식을 왼쪽에서 오른쪽으로 평가한다면 `2 + 6 = 8`가 먼저, 그다음 `8 / 3 = 2.67`가 평가될 것으로 예상할 수 있습니다. 하지만 나누기가 덧셈보다 우선순위가 높기 때문에 JavaScript는 먼저 나누기를 평가하여 `6 / 3 = 2`를 계산하고, 그다음 `2 + 2`를 더해 `4` 결과를 얻습니다.

따라서 곱셈과 나누기는 소괄호로 순서를 바꾸지 않는 한 항상 덧셈과 뺄셈보다 먼저 실행됩니다. 그렇다면 연산자들의 우선순위가 같으면 어떻게 될까요? JavaScript는 평가 순서를 결정하기 위해 결합법칙을 사용합니다.

결합법칙은 JavaScript가 연산자를 왼쪽에서 오른쪽으로 평가할지, 오른쪽에서 왼쪽으로 평가할지 알려줍니다. 덧셈과 곱셈 같은 대부분 연산자는 왼쪽에서 오른쪽으로 결합합니다. 그래서 JavaScript는 표현식의 가장 왼쪽부터 오른쪽으로 처리합니다:

:::interactive_editor

```js
const result = 10 - 2 + 3;

console.log(result); // 11
```

:::

먼저 `10 - 2 = 8`가 평가되고, 그다음 `8 + 3 = 11`가 평가됩니다. 이 경우 JavaScript는 왼쪽에서 오른쪽으로 진행합니다. 할당 연산자(`=`) 같은 일부 연산자는 오른쪽에서 왼쪽으로 결합합니다. 즉, 표현식의 오른쪽 부분이 먼저 평가됩니다:

:::interactive_editor

```js
let a, b;
a = b = 5;

console.log(a); // 5
console.log(b); // 5
console.log(a + b); // 10
```

:::

위 예제에서 JavaScript는 먼저 `5`를 `b`에 할당하고, 그다음 `b` (현재 `5`)를 `a`에 할당합니다.

거듭제곱 연산자도 오른쪽에서 왼쪽으로 결합합니다:

:::interactive_editor

```js
const result = 2 ** 3 ** 2;

console.log(result); // 512
```

:::

먼저 JavaScript는 `3 ** 2`를 평가하여 `9`를 얻고, 그다음 `2 ** 9`를 평가하여 `512`를 얻습니다. 만약 거듭제곱 연산자가 왼쪽에서 오른쪽으로 결합했다면, JavaScript는 먼저 `2 ** 3`를 계산해 `8`를 얻고, 그다음 `8 ** 2`를 계산해 `64`를 얻었을 것입니다.
