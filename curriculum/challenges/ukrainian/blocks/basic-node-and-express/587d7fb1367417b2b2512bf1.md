# --description--

Поки HTML-сервер обслуговує HTML, API обслуговує дані. <dfn>REST</dfn> (передача репрезентативного стану) API дозволяє обмінюватися даними просто, без необхідності, щоб клієнти знали будь-які деталі про сервер. Клієнту потрібно лише знати, де знаходиться ресурс (URL), і яку дію він хоче виконати над ним (дієслово). Дієслово GET використовується, коли ви отримуєте інформацію, не змінюючи нічого. Нині переважним форматом даних для передачі інформації в інтернеті є JSON. Просто кажучи, JSON — це зручний спосіб представити JavaScript-об’єкт у вигляді рядка, щоб його можна було легко передати.

Давайте створимо простий API, створивши маршрут, який відповідає JSON за шляхом `/json`. Ви можете зробити це, як зазвичай, за допомогою методу `app.get()`. Усередині обробника маршруту використайте метод `res.json()`, передаючи об’єкт як аргумент. Цей метод завершує цикл запит-відповідь, повертаючи дані. Під капотом він перетворює дійсний JavaScript-об’єкт у рядок, потім встановлює відповідні заголовки, щоб повідомити вашому браузеру, що ви обслуговуєте JSON, і відправляє дані назад. Дійсний об’єкт має звичайну структуру `{key: data}`. `data` може бути числом, рядком, вкладеним об’єктом або масивом. `data` також може бути змінною або результатом виклику функції, у такому разі він буде обчислений перед перетворенням у рядок.

# --instructions--

Обслуговуйте об’єкт `{"message": "Hello json"}` як відповідь у форматі JSON на GET-запити до маршруту `/json`. Потім відкрийте у браузері `your-app-url/json`, і ви побачите повідомлення на екрані.

# --hints--

Кінцева точка `/json` має обслуговувати JSON-об’єкт `{"message": "Hello json"}`

```js
  const response = await fetch(code + '/json');
  if (!response.ok) {
    throw new Error(await response.text());
  }
  const data = await response.json();
  assert.equal(
    data.message,
    'Hello json',
    "The '/json' endpoint does not serve the right data"
  );
```
