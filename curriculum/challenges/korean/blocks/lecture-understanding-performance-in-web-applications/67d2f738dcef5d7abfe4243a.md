# --description--

‘사용 가능 시간’은 사용자가 페이지를 요청한 시점부터 의미 있게 상호작용할 수 있을 때까지 걸리는 시간을 말하며, 사용자 경험을 향상하는 데 매우 중요합니다.

더 빠른 사용 가능 시간을 달성하기 위한 효과적인 전략을 살펴보겠습니다.

먼저 사용자가 가장 먼저 보는 콘텐츠에 집중하세요. 즉, 필수 요소를 즉시 로드하고 중요하지 않은 컴포넌트는 지연시키는 것입니다.

예를 들어, 지연 로딩(lazy loading)을 구현해 볼 수 있습니다.

이미 지연 로딩에 대해 다뤘지만, 복습을 위해 이미지 지연 로딩 예제를 보여드리겠습니다:

```html
<img src="image.jpg" loading="lazy" alt="Description">
```

이미지 태그에 `loading="lazy"` 속성만 추가하면 됩니다. 이 속성은 이미지가 필요할 때, 보통 사용자의 뷰포트에 들어올 때까지 로딩을 미룹니다. 이미지와 비디오를 뷰포트에 들어올 때만 로드함으로써 대역폭을 절약하고 초기 렌더링 속도를 높일 수 있습니다.

보통 웹 페이지는 화면에 보이지 않는 이미지도 한꺼번에 모두 로드합니다. 지연 로딩을 사용하면 화면 아래쪽, 즉 오프스크린에 있는 이미지는 사용자가 스크롤할 때만 로드됩니다.

다음은 iframe 지연 로딩 예제입니다:

```html
<iframe src="video.html" loading="lazy"></iframe>
```

앞서 사용한 것과 같은 속성을 iframe에 적용하면 동일한 개념이 적용됩니다.

또한 `defer` 속성을 사용해 초기 페이지 로드 후에 중요하지 않은 JavaScript를 지연시킬 수 있습니다. 다음은 그 예입니다:

```html
<script src="non-critical.js" defer></script>
```

렌더링을 차단하는 리소스를 최소화하는 것도 고려해 보세요.

렌더링 차단 리소스는 페이지가 상호작용 가능해지는 것을 지연시킵니다.

이를 피하려면 CSS를 비동기적으로 로드해 보세요. 중요하지 않은 CSS는 `media` 속성을 사용해 조건부로 스타일시트를 로드할 수 있습니다. 다음은 그 예입니다:

```html
<link rel="stylesheet" href="print.css" media="print">
```

media 값을 `print`로 지정하면 `print.css`는 페이지가 인쇄되거나 화면에서 미리보기될 때만 로드됩니다. 다음 예에서는 `min-width: 800px` 속성에 `media` 값을 전달해 화면 너비가 800px 이상일 때만 `desktop.css`가 로드되도록 했습니다:

```html
<!-- This stylesheet is only loaded on screens wider than 800px -->
<link rel="stylesheet" href="desktop.css" media="(min-width: 800px)">
```

이렇게 하면 불필요한 스타일을 건너뛰어 대역폭을 절약할 수 있습니다.

결론적으로, 이러한 전략을 구현하면 웹사이트의 ‘사용 가능 시간’을 크게 줄여 사용자가 콘텐츠와 신속하게 상호작용할 수 있게 됩니다.
