# --description--

고보안 사무실 건물에 들어간다고 상상해 보세요. 안내 데스크에서 경비원이 신분증을 보여 달라고 요청해 본인임을 증명합니다. 경비원이 얼굴과 신분증이 일치하는지 확인하면 로비를 지나갑니다. 하지만 같은 카드를 사용해 임원 서버실에 들어가려고 할 때, 스캐너가 빨간 불을 켜며 출입을 거부합니다.

이 일상적인 상황은 웹 보안의 두 축인 **인증**과 **권한 부여**를 완벽하게 보여줍니다. 비슷하게 들리고 밀접하게 작동하지만, 백엔드 시스템에서는 완전히 다른 역할을 수행합니다.

인증은 당신이 누구인지 결정합니다. 사용자의 신원을 확인하는 과정입니다. 기본적인 질문 하나에 답합니다. *"이 사람이 주장하는 그 사람인가?"*

사용자가 계정을 등록할 때, 사용자 이름, 이메일, 비밀번호 및 기타 자격 증명과 같은 고유 식별자를 제공합니다. 이 정보는 안전하게 데이터베이스에 저장됩니다. 다음에 사용자가 방문해 로그인하려고 하면, 백엔드 시스템은 입력한 자격 증명을 데이터베이스에 저장된 기록과 비교합니다.

백엔드 개발자는 사용자를 인증하기 위해 세 가지 주요 범주의 요소를 사용합니다:

- **알고 있는 것:** 비밀번호, PIN, 또는 보안 질문에 대한 답변.

- **가지고 있는 것:** SMS 인증 코드를 받는 스마트폰이나 인증 앱에서 생성된 임시 토큰.

- **자신인 것:** 지문이나 얼굴 인식 스캔 같은 생체 정보.

시스템이 사용자를 성공적으로 인증하면, **권한 부여**가 이어서 사용자가 무엇을 할 수 있고 접근할 수 있는지를 결정합니다. 다른 질문에 답합니다. *"이 특정 신원에 어떤 권한이 있는가?"*

사용자가 성공적으로 로그인했다고 해서 애플리케이션을 완전히 제어할 수 있는 것은 아닙니다. 권한 부여는 그 사용자에 대한 규칙, 역할, 경계를 정의합니다. 다음은 일반적인 애플리케이션 역할입니다:

- **표준 사용자:** 자신의 프로필은 수정할 수 있지만, 다른 사용자의 개인 청구 정보를 볼 수 없습니다.

- **편집자:** 블로그 게시물을 작성하고 게시할 수 있지만, 전체 웹사이트 코드베이스를 삭제할 수는 없습니다.

- **관리자:** 데이터베이스 수정, 사용자 삭제, 시스템 전체 설정 변경에 대한 완전한 권한을 가집니다.

백엔드 코드에서는 권한 부여가 보통 특정 페이지나 데이터에 접근하기 전에 사용자 역할이나 특정 권한을 확인하는 미들웨어 함수로 이루어집니다.

이 둘을 명확히 하려면, 이 간단한 규칙을 기억하세요: **항상 먼저 인증이 이루어지고, 그 다음에 권한 부여가 이루어집니다.** 누군지 정확히 알기 전에는 어떤 권한을 줄지 결정할 수 없습니다.

인증은 현관문을 여는 열쇠이고, 권한 부여는 안에 들어간 후 어떤 방에 들어갈 수 있는지를 결정합니다.
