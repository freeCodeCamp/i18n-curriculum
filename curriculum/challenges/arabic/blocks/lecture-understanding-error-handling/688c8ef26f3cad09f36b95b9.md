# --description--

تصحيح الأخطاء مهارة أساسية لأي مطوِّر Python. فهم التقنيات الأساسية يساعدك على تحديد المشكلات وإصلاحها بكفاءة.

تصحيح الأخطاء هو عملية تحديد الأخطاء أو العيوب في الشفرة وحلها. يتضمن فحص الشفرة، وفهم سير التنفيذ، واستخدام أدوات لتحديد مصدر المشاكل.

في هذا الدرس، سنستعرض تقنيات تصحيح الأخطاء الشائعة التي يمكنك استخدامها في مشروع Python التالي.

## استخدام الدالة `print()` وسلاسل f النصية

أولًا، استخدام الدالة `print()` وسلاسل f النصية في نقاط مختلفة من الشفرة يساعدك على فهم سير التنفيذ وحالة المتغيرات. على سبيل المثال:

```py
def add(a, b):
    result = a + b
    print(f'Adding {a} and {b} gives {result}')
    return result
```

عن طريق طباعة قيم `a` و `b` و `result`، يمكنك التحقق من أن الدالة تعمل كما هو متوقع.

## التصحيح التفاعلي باستخدام وحدة `pdb`

بعد ذلك، يمكنك استخدام وحدة `pdb` المدمجة في Python للتصحيح التفاعلي:

```py
import pdb

def divide(a, b):
    pdb.set_trace()
    return a / b

print(divide(10, 2))
```

بإعداد تتبع باستخدام الدالة `set_trace()`، يمكنك التنقل خطوة بخطوة في الشفرة، وفحص المتغيرات، وفهم سلوك البرنامج.

إذا شغلت الشفرة أعلاه، سترى بعض المخرجات التي تعرض موقع الملف الذي تشغله، والسطر الذي استدعيت فيه الدالة `set_trace()` والشفرة التي تليه مباشرة، بالإضافة إلى موجه `pdb` التفاعلي:

```py
> /Users/fcc/Desktop/debugging.py(5)divide()
-> return a / b
(Pdb)
```

إذا أدخلت `help` في الموجه، سترى قائمة بالأوامر التي يمكنك استخدامها:

```py
(Pdb) help

Documented commands (type help <topic>):
========================================
EOF    c          d        h         list      q        rv       undisplay
a      cl         debug    help      ll        quit     s        unt      
alias  clear      disable  ignore    longlist  r        source   until    
args   commands   display  interact  n         restart  step     up       
b      condition  down     j         next      return   tbreak   w        
break  cont       enable   jump      p         retval   u        whatis   
bt     continue   exit     l         pp        run      unalias  where    

Miscellaneous help topics:
==========================
exec  pdb
```

بعدها يمكنك استخدام هذه الأوامر لتصحيح الشفرة.

على سبيل المثال، إذا أردت معرفة نوع العناصر في الشفرة في تلك اللحظة، يمكنك استخدام الأمر `whatis`:

```py
(Pdb) whatis a
<class 'int'>
(Pdb) whatis divide
Function divide
```

كما ترى، بحلول الوقت الذي تشغل فيه `set_trace()`، نوع المعلمة `a` هو عدد صحيح، و`divide` هي دالة.

ثم لمتابعة تنفيذ الشفرة، يمكنك استخدام الأمر `continue`، أو أحد أسمائه المستعارة، `cont` أو `c`:

```py
(Pdb) continue
5.0
```

## أدوات تصحيح الأخطاء في بيئات التطوير المتكاملة

تقدم العديد من بيئات التطوير المتكاملة (IDEs) أدوات متقدمة لتصحيح الأخطاء، مثل نقاط التوقف، والتنفيذ خطوة بخطوة، وفحص المتغيرات.

## استخدام مصحح الأخطاء في VS Code

إذا كنت تستخدم VS Code، يمكنك تعيين نقاط توقف في الشفرة وتشغيل المصحح لإيقاف التنفيذ عند تلك النقاط. إليك كيفية تصحيح نفس دالة `divide`:

**الخطوة 1: إعداد الشفرة**

أنشئ ملفًا باسم `main.py` بالمحتوى التالي:

```py
def divide(a, b):
    result = a / b
    return result

print(divide(10, 2))
print(divide(15, 3))
```

**الخطوة 2: تعيين نقطة توقف**

1. انقر في مسافة الفراغ (الهامش الأيسر) بجانب السطر 2 (`result = a / b`) لتعيين نقطة توقف

2. ستظهر نقطة حمراء تشير إلى تعيين نقطة التوقف

**الخطوة 3: بدء التصحيح**

1. اضغط `F5` أو اذهب إلى تشغيل > بدء التصحيح

2. اختر "ملف Python" عند الطلب

3. سيوقف المصحح التنفيذ عند نقطة التوقف

**الخطوة 4: فحص المتغيرات**

- مرر فوق المتغيرات لرؤية قيمها الحالية

- استخدم لوحة المتغيرات على اليسار لرؤية كل المتغيرات المحلية

- استخدم وحدة التحكم في التصحيح في الأسفل لتقييم التعبيرات

**الخطوة 5: التنقل في الشفرة خطوة بخطوة**

استخدم شريط أدوات التصحيح لـ:

- **استمرار (F5)**: استئناف التنفيذ حتى نقطة التوقف التالية

- **تخطي (F10)**: تنفيذ السطر الحالي والانتقال إلى التالي

- **الدخول (F11)**: الدخول إلى استدعاءات الدوال

- **الخروج (Shift+F11)**: الخروج من الدالة الحالية

توفر أدوات تصحيح الأخطاء في بيئات التطوير المتكاملة واجهة بصرية لفحص حالة البرنامج، مما يسهل تحديد المشكلات وإصلاحها مقارنة باستخدام أوامر الطباعة فقط.

بإتقان هذه التقنيات الأساسية لتصحيح الأخطاء (أوامر `print()`، وحدة `pdb`، وأدوات IDE)، يمكنك تحديد المشكلات وحلها بفعالية في شفرة Python الخاصة بك. لكل تقنية مكانها: أوامر `print()` للفحوصات السريعة، و`pdb` للاستكشاف التفاعلي، ومصححات IDE للفحص البصري.
