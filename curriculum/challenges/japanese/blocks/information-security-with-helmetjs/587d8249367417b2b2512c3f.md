# --description--

このプロジェクトは、<a href="https://github.com/freeCodeCamp/boilerplate-infosec/" target="_blank" rel="noopener noreferrer nofollow">GitHub</a> からクローンした以下のスタータープロジェクトを基に構築されています。

このチャレンジでは、現代のブラウザで多くの種類の攻撃のリスクと影響を大幅に減らせる有望な新しい防御策を紹介します。コンテンツセキュリティポリシーを設定・構成することで、ページに意図しないものが注入されるのを防げます。これにより、XSSの脆弱性、望ましくないトラッキング、悪意のあるフレームなどからアプリを保護できます。CSPは信頼できるコンテンツソースの許可リストを定義することで機能します。ウェブページが必要とする各種リソース（スクリプト、スタイルシート、フォント、フレーム、メディアなど）ごとに設定可能です。複数のディレクティブが用意されているため、ウェブサイトの所有者は細かく制御できます。詳細はHTML 5 RocksやKeyCDNをご覧ください。残念ながら、古いブラウザではCSPはサポートされていません。

デフォルトではディレクティブは広く開放されているため、フォールバックとして defaultSrc ディレクティブを設定することが重要です。Helmetは defaultSrc と default-src の両方の命名スタイルをサポートしています。フォールバックは指定されていないほとんどのディレクティブに適用されます。

# --instructions--

この演習では `helmet.contentSecurityPolicy()` を使います。`directives` オブジェクトを追加して設定してください。そのオブジェクト内で、`defaultSrc` を `["'self'"]` に設定してください（許可されたソースのリストは配列である必要があります）。これにより、デフォルトで自分のウェブサイトのアドレスのみを信頼します。また、`scriptSrc` ディレクティブを設定して、スクリプトは自分のウェブサイト（`'self'`）とドメイン `'trusted-cdn.com'` からのみダウンロードを許可してください。

ヒント：`'self'` キーワードでは、シングルクォートはキーワードの一部なので、動作させるには二重引用符で囲む必要があります。

# --hints--

helmet.contentSecurityPolicy() ミドルウェアが正しくマウントされていることを確認してください

```js
const response = await fetch(code + '/_api/app-info');
if (!response.ok) {
  throw Error(await response.text());
}
const data = await response.json();
assert.include(data.appStack, 'csp');
```

CSPの設定が正しくありません。defaultSrc は ["'self'"]、scriptSrc は ["'self'", 'trusted-cdn.com'] にしてください

```js
const response = await fetch(code + '/_api/app-info');
if (!response.ok) {
  throw Error(await response.text());
}
const data = await response.json();
var cspHeader = Object.keys(data.headers).filter(function (k) {
  return (
    k === 'content-security-policy' ||
    k === 'x-webkit-csp' ||
    k === 'x-content-security-policy'
  );
})[0];
assert.equal(
  data.headers[cspHeader],
  "default-src 'self'; script-src 'self' trusted-cdn.com"
);
```
