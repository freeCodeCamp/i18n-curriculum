# --description--

У попередніх уроках ви дізналися, що React — це застосунок з однією сторінкою. Застосунки з однією сторінкою містять один HTML-файл і використовують JavaScript для динамічного оновлення будь-якого вмісту на сторінці.

Отже, що відбувається, коли потрібно додати кілька «сторінок» до вашого React-застосунку? Як ви будете переходити до цих різних виглядів?

Саме тут на допомогу приходить React Router.

React Router — це стороння бібліотека, яка дозволяє додати маршрутизацію до ваших React-застосунків. Щоб почати, вам потрібно встановити React Router у наявний React-проєкт ось так:

```sh
npm i react-router
```

Якщо ви перевірите файл `package.json`, то побачите, що `react-router` було додано до списку залежностей:

```json
"dependencies": {
  "react": "^18.3.1",
  "react-dom": "^18.3.1",
  "react-router": "^7.2.0"
}
```

Потім у вашому файлі `main.jsx` або `index.jsx` потрібно імпортувати `BrowserRouter` і відтворити `BrowserRouter` навколо компонента `App`:

```jsx
import { StrictMode } from "react";
import { createRoot } from "react-dom/client";
import { BrowserRouter } from "react-router";
import App from "./App.jsx";

import "./index.css";

createRoot(document.getElementById("root")).render(
  <StrictMode>
    <BrowserRouter>
      <App />
    </BrowserRouter>
  </StrictMode>
);
```

Щоб увімкнути маршрути у вашому застосунку, потрібно оновити інструкцію `import`, додавши компоненти `Routes` і `Route` ось так:

```js
import { BrowserRouter, Routes, Route } from "react-router";
```

Потім всередині `BrowserRouter` додайте компоненти `Routes` і `Route`:

```jsx
createRoot(document.getElementById("root")).render(
  <StrictMode>
    <BrowserRouter>
      <Routes>
        <Route path="/" element={<App />} />
      </Routes>
    </BrowserRouter>
  </StrictMode>
);
```

`path` і `element` використовуються для зв’язування URL і UI-компонентів. У цьому випадку ми налаштовуємо маршрут для головної сторінки, який вказує на компонент `App`.

У більших застосунках часто налаштовують кілька виглядів і маршрутів ось так:

```jsx
<Routes>
  <Route index element={<Home />} />
  <Route path="about" element={<About />} />

  <Route path="products">
    <Route index element={<ProductsHome />} />
    <Route path=":category" element={<Category />} />
    <Route path=":category/:productId" element={<ProductDetail />} />
    <Route path="trending" element={<Trending />} />
  </Route>
</Routes>
```

Проп `index` у цих прикладах призначений для представлення маршруту за замовчуванням для певного сегмента шляху. Тож компонент `Home` буде показаний за кореневим шляхом (`/`), а компонент `ProductsHome` — за шляхом `products`.

Ви також могли помітити, що ми вкладаємо кілька маршрутів всередину іншого маршруту ось так:

```jsx
<Route path="products">
  <Route path="trending" element={<Trending />} />
</Route>
```

Це означає, що шлях дочірнього маршруту буде доданий до шляху батьківського маршруту. Тож у цьому прикладі шлях `path` для популярних товарів буде `products/trending`.

Якщо `path` починається з двокрапки (`:`), це означає динамічний сегмент у маршруті:

```jsx
<Route path=":category" element={<Category />} />
```

У цьому прикладі ми маємо динамічний сегмент з назвою `category`. Коли користувач переходить за URL на кшталт `products/brass-instruments`, вигляд змінюється на компонент `Category`, і ви можете динамічно отримувати відповідні дані на основі цього сегмента.

Ви можете отримати доступ до значення динамічного сегмента, використовуючи хук `useParams` всередині дочірнього компонента ось так:

```jsx
import { useParams } from "react-router";

export default function Category() {
  let params = useParams();
   {/* Accessing the category param: params.category */}
   {/* rest of code goes here */}
}
```

Динамічні маршрути корисні, бо дозволяють створювати гнучкі та багаторазові компоненти, які можуть відтворювати різний вміст залежно від параметрів у URL. Замість того, щоб визначати фіксований список шляхів для кожного можливого маршруту, ви можете використовувати динамічні сегменти, щоб відтворювати різний вміст залежно від того, що запросив користувач.
