# --description--

소프트웨어 개발에서 라이브러리는 개발자를 위한 도구 상자와 같습니다.

코드를 처음부터 전부 직접 구현하는 대신, 라이브러리는 함수, 클래스, 자료 구조 같은 미리 작성되고 재사용 가능한 코드를 제공하여 프로젝트에서 사용할 수 있게 합니다.

파이썬은 다양한 내장 모듈을 포함하는 방대한 표준 라이브러리를 가지고 있습니다. 이들은 모두 표준화되고 잘 검증된 솔루션으로, 프로그래머가 매일 마주하는 여러 문제와 작업을 해결하는 데 도움을 줍니다. 예를 들면:

* 운영 체제와 상호작용하기
    
* 파일 작업하기
    
* 네트워킹
    
* 날짜와 시간 다루기
    
* 수학 연산 수행하기
    
* 정규 표현식 사용하기
    
* 코드 테스트 및 디버깅
    
* 그리고 더 많은 기능들!
    

인기 있는 내장 모듈의 예로는 `math`, `random`, `re` (정규 표현식의 약자), 그리고 `datetime`이 있습니다.

`math` 모듈은 더 복잡한 수학 연산을 수행하는 데 유용한 함수들을 제공합니다.

`random` 모듈은 난수 생성에 도움을 줍니다.

`re` 모듈은 정규 표현식을 다루는 데 사용됩니다.

그리고 `datetime` 모듈은 파이썬에서 날짜와 시간을 다루는 데 유용합니다.

그런데 이 내장 모듈에 정의된 변수, 상수, 함수, 클래스에 어떻게 접근할 수 있을까요?

바로 import 문을 사용합니다. 이 문장은 모듈을 파이썬 스크립트로 가져오게 해줍니다. 일반적으로 import 문은 파일 상단에 작성하며, 필요에 따라 맞춤 설정할 수도 있습니다. 먼저 `import` 문을 쓰고, 그 뒤에 모듈 이름을 적습니다:

```python
import module_name
```

예를 들어 `math` 모듈을 가져오고 싶다면, 파일 상단에 이렇게 작성합니다:

```python
import math
```

그다음, 파이썬 스크립트에서 해당 모듈의 함수를 호출할 때는 점 표기법을 사용하여 모듈 이름 다음에 함수 이름을 적습니다:

```python
module_name.function_name() 
```

예를 들어 36의 제곱근을 구하려면 `math` 다음에 점을 찍고 `sqrt` (제곱근의 약어)을 적은 뒤 소괄호 안에 필요한 인수를 전달합니다. 이 경우 제곱근을 구할 숫자 하나만 전달하면 됩니다:

```python
math.sqrt(36)
```

이것이 가장 기본적인 import 문이지만, 다른 방법도 있습니다.

모듈을 다른 이름(별칭)으로 가져와야 할 때는 import 문 끝에 `as` 다음에 별칭을 적는 구문을 사용합니다:

```python
import module_name as module_alias
```

이 방법은 긴 모듈 이름을 줄이거나 이름 충돌을 피할 때 자주 사용됩니다.

예를 들어 `math` 모듈을 코드에서 `m`로 부르고 싶다면, 이렇게 별칭을 지정할 수 있습니다:

```python
import math as m 
```

그다음 별칭을 사용해 모듈의 요소에 접근할 수 있습니다:

```python
m.sqrt(36)
```

하지만 때로는 모듈 전체를 가져올 필요 없이 특정 함수나 클래스 몇 개만 필요할 때가 있습니다. 파이썬은 이런 경우에 딱 맞는 방법을 제공합니다.

이제 import 문은 `from`로 시작하고, 모듈 이름 다음에 `import` 키워드와 가져올 요소 이름들을 적습니다:

```python
from module_name import name1, name2
```

그다음 파이썬 스크립트에서 모듈 접두사 없이 이 이름들을 바로 사용할 수 있습니다.

이 이름들에 별칭을 붙이고 싶다면, 각 이름 뒤에 `as` 키워드와 원하는 별칭을 적으면 됩니다:

```python
from module_name import name1 as alias1, name2 as alias2
```

예를 들어 `math` 모듈에서 radians, sine, cosine 함수만 가져오고 싶다면 이렇게 작성합니다:

```python
from math import radians, sin, cos
```

이제 코드에서 `math` 모듈 접두사 없이 이 함수들을 직접 호출할 수 있습니다.

더 자세한 예를 보겠습니다:

처음에 도 단위로 표현된 특정 각도의 사인과 코사인을 구하려면, `radians` 함수를 호출해 라디안으로 변환한 뒤, 사인과 코사인 함수를 호출하며 라디안 단위의 각도를 전달합니다:

```python
from math import radians, sin, cos

angle_degrees = 40
angle_radians = radians(angle_degrees)

sine_value = sin(angle_radians)
cos_value = cos(angle_radians)

print(sine_value) # 0.6427876096865393
print(cos_value)  # 0.766044443118978
```

함수들을 모듈 이름 접두사 없이 직접 호출하는 것을 볼 수 있습니다. 이는 이 대체 구문으로 함수를 가져왔기 때문입니다.

이 방법은 편리하지만, 이미 스크립트 내에 같은 이름의 함수나 변수가 있을 경우 이름 충돌이 발생할 수 있으니 주의해야 합니다.

마지막으로, import 문이 별표(*)로 끝나는 경우가 있습니다. 별표는 해당 모듈의 모든 내용을 가져오되, 모듈 이름 접두사 없이 사용하겠다는 의미입니다:

```python
from module_name import *
```

예를 들어 `math` 모듈을 이렇게 가져오면, 그 모듈에 정의된 모든 함수를 모듈 이름 없이 호출할 수 있습니다. 몇 가지 예시는 다음과 같습니다:

```python
from math import *
print(sqrt(36))  # 6.0
print(pow(5, 2)) # 25.0
print(exp(1))    # 2.718281828459045
```

하지만 이 방법은 네임스페이스 충돌을 일으키기 쉽고, 특정 이름이 어디서 왔는지 알기 어려워지므로 일반적으로 권장되지 않습니다.

import 문은 함수, 클래스, 상수, 변수, 그리고 모듈 내 정의된 다른 모든 요소에 대해 똑같이 작동합니다.

다음은 `math` 모듈의 상수 예시로, 숫자 `pi`입니다:

```python
import math
print(math.pi)
```

그리고 `datetime` 모듈의 클래스 예시입니다. 1959년 7월 15일을 나타내는 날짜 객체를 만들고, 그 `date` 객체를 변수에 할당한 뒤 점 표기법으로 일, 월, 연도를 각각 접근합니다:

```python
import datetime
birthday = datetime.date(1959, 7, 15)
print(birthday.day)    # 15
print(birthday.month)  # 7
print(birthday.year)   # 1959
```

해당 모듈 내용에 관한 자세한 정보는 공식 파이썬 문서에서 확인할 수 있습니다.

좋습니다. 이제 모듈에 대해 더 알게 되었으니, 파이썬 스크립트에서 매우 중요한 관용구도 알아야 합니다. 이 둘은 매우 밀접하게 관련되어 있습니다:

```python
if __name__ == '__main__': 
    # Code
```

`__name__`는 파이썬 내장 특별 변수입니다.

파이썬 파일이 직접 실행될 때, 파이썬은 이 변수의 값을 문자열 `"__main__"`로 설정합니다.

하지만 파이썬 파일이 다른 파이썬 스크립트에 모듈로 가져와질 경우, `__name__` 변수의 값은 그 모듈 이름(보통 `.py` 확장자를 뺀 파일 이름)으로 설정됩니다.

그래서 파이썬 스크립트에서 이 조건문을 자주 볼 수 있습니다. 이 조건문 안에는 파이썬 스크립트가 메인 프로그램으로 실행될 때만 실행하고 싶은 코드가 들어 있습니다:

```python
if __name__ == '__main__': 
    # Code
```

하지만 스크립트가 모듈로 가져와지면, 이 블록 안의 코드는 실행되지 않습니다.

이것은 파이썬 스크립트가 두 가지 용도로 사용될 수 있게 해줍니다. 직접 실행되어 주요 로직을 수행하거나, 다른 모듈에 가져와져 주요 로직을 실행하지 않고 사용할 수 있습니다.
