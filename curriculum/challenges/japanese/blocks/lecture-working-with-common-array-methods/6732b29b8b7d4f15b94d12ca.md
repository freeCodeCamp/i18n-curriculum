# --interactive--

配列のシャローコピーとは、元の配列と同じ要素を持つ新しい配列のことです。配列が数値や文字列のようなプリミティブ値だけを含む場合、新しい配列は完全に独立しています。しかし、配列の中に他の配列が含まれている場合、元の配列とコピーの両方が同じ内部配列を参照しています。つまり、共有されている内部配列の中身を変更すると、その変更は両方の配列に反映されます。

シャローコピーは、元の配列や内部配列を変更せずに、要素の追加、削除、並べ替えなどトップレベルの構造を変更したい場合に役立ちます。

配列のシャローコピーを作成する方法はいくつかあり、ここでは最も一般的な`concat()`、`slice()`、およびスプレッド演算子を見ていきます。

まずは`concat()`メソッドから始めましょう。このメソッドは2つ以上の配列を結合して新しい配列を作成します。単一の配列に使うと、実質的にシャローコピーを作成します。例を示します：

:::interactive_editor

```js
const originalArray = [1, 2, 3];
const copyArray = [].concat(originalArray);

console.log(copyArray); // [1, 2, 3]
console.log(copyArray === originalArray); // false
```

:::

この例では、`concat()`メソッドを使って空の配列を`originalArray`に連結しています。これにより、`originalArray`のシャローコピーとなる新しい配列が作成されます。

`copyArray`は`originalArray`と同じ要素を含みますが、異なる配列オブジェクトなので、厳密な等価性チェック（`===`）は`false`を返します。

シャローコピーを作成する別の方法は`slice()`メソッドです。引数なしで呼び出すと、`slice()`は配列全体のシャローコピーを返します。動作は以下の通りです：

:::interactive_editor

```js
const originalArray = [1, 2, 3];
const copyArray = originalArray.slice();

console.log(copyArray); // [1, 2, 3]
console.log(copyArray === originalArray); // false
```

:::

この場合、`originalArray.slice()`は`originalArray`のシャローコピーとなる新しい配列を作成します。やはり`copyArray`は同じ要素を含みますが異なる配列オブジェクトです。

ES6で導入されたスプレッド演算子（`...`）は、配列のシャローコピーを作成するもう一つの簡潔な方法です。例を示します：

:::interactive_editor

```js
const originalArray = [1, 2, 3];
const copyArray = [...originalArray];

console.log(copyArray); // [1, 2, 3]
console.log(copyArray === originalArray); // false
```

:::

スプレッド演算子（`...`）は`originalArray`の要素を新しい配列に展開し、実質的にシャローコピーを作成します。これらの方法はすべて新しい配列オブジェクトを作成するため、コピーを変更しても元の配列には影響しません。例えば：

:::interactive_editor

```js
const originalArray = [1, 2, 3];
const copyArray = [...originalArray];

copyArray.push(4);
console.log(originalArray); // [1, 2, 3]
console.log(copyArray);     // [1, 2, 3, 4]
```

:::

この例では、`copyArray`に要素を追加しても`originalArray`には影響しません。

まとめると、配列のシャローコピーは`concat()`、`slice()`、またはスプレッド演算子のようなメソッドを使って簡単に作成できます。これらの方法は元の配列とは独立して操作できる新しい配列を作るのに便利です。
