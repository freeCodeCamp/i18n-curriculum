# --description--

Manter suas dependências atualizadas e saber como instalar uma versão específica é importante para manter seu projeto estável. Isso ajuda a evitar bugs inesperados e dá mais controle quando novas versões são lançadas.

Para instalar uma versão específica de um pacote, use este formato:

```bash
npm install package@version-number
```

Por exemplo, se o seu projeto precisar de uma versão mais antiga do Express, digamos `4.21.2`, você pode instalá-la assim:

```bash
npm install express@4.21.2
```

Se você não incluir um número de versão, o npm instalará automaticamente a versão mais recente disponível.

Se você não tem certeza de quais versões existem, pode verificá-las com este comando:

```bash
npm view package-name versions
```

Por exemplo:

```bash
npm view express versions
```

Para atualizar pacotes, você pode atualizá-los individualmente ou todos de uma vez. Antes de fazer isso, é útil primeiro ver quais estão desatualizados:

```bash
npm outdated
```

Isso mostrará quais dependências têm versões mais recentes disponíveis e você deverá ver uma saída semelhante a esta:

```bash
user@Kolade ~/Desktop/fCC/script-code/node/npm/install-spec-version  % npm outdated
Package  Current  Wanted  Latest  Location              Depended by
chalk      5.5.0   5.5.0   5.6.2  node_modules/chalk    install-spec-version
cleave     1.0.1   1.0.1   1.0.2  node_modules/cleave   install-spec-version
express   4.21.2  4.21.2   5.2.1  node_modules/express  install-spec-version
```

Depois de verificar quais dependências estão desatualizadas, você pode começar a atualizá-las.

Para atualizar uma única dependência, use o seguinte:

```bash
npm update package-name
```

Isso atualiza o pacote apenas dentro do intervalo de versões permitido pelo seu arquivo `package.json`. Por exemplo, para atualizar o Express, você pode usar o seguinte:

```bash
npm update express
```

Se você quiser pular o intervalo de versões e instalar a versão mais recente de um pacote, use o seguinte comando:

```bash
npm install package-name@latest
```

Para atualizar todas as dependências que estão dentro das faixas de versão permitidas, execute:

```bash
npm update
```

No entanto, note que `npm update` não atualiza um pacote para uma nova versão major se o seu `package.json` não permitir. Versões major frequentemente contêm mudanças incompatíveis, então o npm não as aplicará automaticamente.

Se você quiser especificamente a versão principal mais recente, você deve executar:

```bash
npm install package-name@latest
```

Isso força o npm a instalar a versão mais recente, não importando a faixa de versão que está no seu `package.json`.
