# --description--

أنت الآن جاهز لاتخاذ خطواتك الأولى في عالم Node.js بتثبيته على جهاز الكمبيوتر. لكن أولًا، قبل تثبيت Node.js، نوصي بتثبيت أداة تسمى NVM (مدير إصدارات Node). كما يشير اسمها، NVM هي أداة لإدارة إصدارات متعددة من Node.js على نفس الجهاز.

إعداد هذه الأداة من البداية سيساعدك في المستقبل لأنك قد تحتاج إلى التبديل بين إصدارات مختلفة من Node.js لمشاريع مختلفة. مع NVM، يمكنك فعل ذلك بسهولة كبيرة.

دعنا نرى كيف يمكنك تثبيت NVM على macOS/Linux وWindows، وكيف يمكنك استخدامه لتثبيت Node.js.

## كيفية تثبيت NVM على macOS/Linux

على macOS وLinux، يمكنك تثبيت NVM مباشرة عبر سكربت شل.

لكن بالنسبة لـ macOS، لاحظ أن الوثائق الرسمية لـ NVM تشير إلى أنه "يجب تثبيت أدوات سطر أوامر Xcode يدويًا قبل تشغيل سكربت التثبيت، وإلا سيفشل التثبيت."

على macOS، يمكنك تثبيت أدوات سطر أوامر Xcode بتشغيل هذا الأمر في الطرفية:

```bash
xcode-select --install
```

بعد الانتهاء من التثبيت، يمكنك التحقق من رقم الإصدار والتأكد من التثبيت بهذا الأمر:

```bash
xcode-select --version
```

بعدها، حان وقت تثبيت NVM. يمكنك تنزيل وتنفيذ سكربت تثبيت NVM بتشغيل أحد هذه الأوامر:

```bash
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.3/install.sh | bash
```

```bash
wget -qO- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.3/install.sh | bash
```

للحصول على أحدث إصدار من NVM في الوقت الذي تقرأ فيه هذا المقال، يرجى الرجوع إلى الوثائق الرسمية لـ NVM: `https://github.com/nvm-sh/nvm?tab=readme-ov-file#install--update-script`. هناك ستجد هذه الأوامر مع أحدث رقم إصدار.

قد تحتاج إلى إعادة تشغيل الطرفية بعد اكتمال العملية لتحميل الأمر الجديد والإعدادات الجديدة.

ثم، شغّل هذا الأمر لتأكيد تثبيت NVM بشكل صحيح وللتحقق من الإصدار الحالي:

```bash
nvm -v
```

يجب أن ترى إصدار NVM الخاص بك في المخرجات.

## كيفية تثبيت Node.js باستخدام NVM على macOS/Linux

الآن بعد أن ثبتت NVM على macOS/Linux، يمكنك تثبيت Node.js.

لتثبيت أحدث إصدار دعم طويل الأمد (LTS) من Node.js، شغّل هذا الأمر:

```bash
nvm install --lts
```

لاحظ العلامة `--lts`. هذه العلامة تشير إلى رغبتك في تثبيت الإصدار الحالي من دعم طويل الأمد (LTS) لـ Node.js. يُنصح عادة بهذا الإصدار لأنه يعطي الأولوية للاستقرار والموثوقية والأمان على الميزات الجديدة أو التجريبية. تم اختباره جيدًا وجاهز للإنتاج. كما يضمن فترات دعم أطول مع التركيز على إصلاح الأخطاء وتحديثات الأمان.

بدلاً من ذلك، يمكنك تثبيت إصدار محدد من Node.js بتحديد رقم الإصدار في الأمر:

```bash
nvm install <version>
```

على سبيل المثال، هكذا يمكنك تثبيت Node.js الإصدار 20:

```bash
nvm install 20
```

بعد الانتهاء من تثبيت إصدار من Node.js، يمكنك التبديل إلى هذا الإصدار بهذا الأمر:

```bash
nvm use <version>
```

على سبيل المثال، هكذا يمكنك التبديل إلى الإصدار 20:

```bash
nvm use 20
```

أخيرًا، يمكنك التحقق من الإصدار الحالي لـ Node.js بهذا الأمر:

```bash
node -v
```

## كيفية تثبيت NVM على Windows

الآن دعنا نرى كيف يمكنك تثبيت واستخدام NVM على Windows.

الإصدارات الرسمية من NVM تدعم Windows فقط في حالات محددة. إذا كان لديك Windows Subsystem for Linux (WSL) مُثبت على جهازك، يمكنك اتباع دليل macOS/Linux أعلاه لتثبيت واستخدام NVM.

لكن بالنسبة لـ Windows نفسه، هناك بعض البدائل. أحدها هو `nvm-windows`، أداة لإدارة إصدارات Node.js على Windows. هذا مشروع منفصل تمامًا عن مشروع NVM الرسمي. لكن هدف هذه الأدوات هو نفسه — مساعدتك على العمل مع إصدارات متعددة من Node.js على نفس الجهاز.

قبل تثبيت `nvm-windows`، يجب أن تعرف أن المستودع الرسمي يوصي بإلغاء تثبيت أي تثبيت سابق لـ Node قبل تثبيت NVM على Windows. إذا كانت هذه هي المرة الأولى التي تثبت فيها Node.js، فستكون الأمور على ما يرام. أما إذا كان لديك إصدارات أخرى مثبتة، فمن الأفضل إلغاء تثبيتها أولًا.

لتثبيت `nvm-windows`:

- اذهب إلى قسم الإصدارات في المستودع الرسمي: `https://github.com/coreybutler/nvm-windows/releases`.
- حمّل أحدث مثبت.
- اتبع التعليمات.

بعد إكمال التثبيت، قد تحتاج إلى إعادة تشغيل الطرفية إذا لم يكن أمر `nvm` متاحًا فورًا.

يمكنك التحقق من نجاح التثبيت ورؤية إصدار `nvm-windows` الحالي بتشغيل هذا الأمر:

```powershell
nvm -v
```

## كيفية تثبيت Node.js باستخدام NVM على Windows

الآن بعد تثبيت `nvm-windows` على Windows، أنت جاهز لتثبيت Node.js.

لتثبيت أحدث إصدار دعم طويل الأمد (LTS) من Node.js، شغّل هذا الأمر:

```powershell
nvm install lts
```

عادةً ما يُنصح بإصدار دعم طويل الأمد (LTS)، لكن يمكنك تثبيت إصدار مختلف بتحديد رقم الإصدار في الأمر:

```powershell
nvm install <version>
```

على سبيل المثال:

```powershell
nvm install 20
```

عند اكتمال التثبيت، يمكنك التبديل إلى هذا الإصدار من Node.js بهذا الأمر:

```powershell
nvm use <version>
```

على سبيل المثال:

```powershell
nvm use 20
```

يمكنك أيضًا التحقق من الإصدار الحالي لـ Node.js بهذا الأمر:

```bash
node -v
```

## الأوامر الأساسية

الآن بعد أن عرفت كيف تثبت NVM وNode.js، دعنا نراجع بعض الأوامر الأساسية التي ستجدها مفيدة عند العمل مع Node.js وNVM.

أيضًا، بينما تعمل كل هذه الأوامر مع NVM المثبت على macOS/Linux، بما في ذلك WSL، قد تعمل بشكل مختلف أو قد لا تعمل على الإطلاق مع `nvm-windows`. سنوفر أوامر بديلة لـ `nvm-windows` متى أمكن.

### كيفية تعديل سياسة تنفيذ Powershell

إذا كنت تستخدم Windows وتتابع مع Powershell و`nvm-windows`، قد تحتاج إلى تعديل سياسة تنفيذ Powershell. السياسة الافتراضية لـ Powershell هي `Restricted`، وهي إجراء أمني لمنع تشغيل السكربتات.

أولًا، شغّل Powershell كمسؤول بالنقر بزر الفأرة الأيمن على Powershell، ثم النقر على “تشغيل كمسؤول“.

ثم، لرؤية سياسة التنفيذ الحالية لجلسة Powershell الحالية، شغّل الأمر التالي:

```powershell
Get-ExecutionPolicy
```

أخيرًا، لتعيين سياسة تنفيذ Powershell إلى `RemoteSigned`، التي تسمح بتشغيل السكربتات المحلية و`npm`، استخدم الأمر التالي:

```powershell
Set-ExecutionPolicy RemoteSigned
```

بعد تشغيل الأمر أعلاه، يمكنك إغلاق نافذة Powershell الخاصة بالمسؤول، وفتح نافذة Powershell عادية.

### عرض جميع الإصدارات

لعرض جميع إصدارات Node.js المثبتة والتحقق من الإصدار المستخدم حاليًا، شغّل هذا الأمر:

```bash
nvm ls
```

### تعيين إصدار Node.js الافتراضي (NVM على macOS/Linux فقط)

يحدد هذا الأمر الإصدار الافتراضي من Node.js الذي سيُستخدم في جلسات الطرفية الجديدة:

```bash
nvm alias default <version>
```

على سبيل المثال:

```bash
nvm alias default 22.20.0
```

يمكنك أيضًا إنشاء أسماء مستعارة مخصصة لتعيين اسم أكثر تحديدًا لإصدار معين:

```bash
nvm alias <new_name> <version>
```

على سبيل المثال:

```bash
nvm alias new-api 18.20.8
```

بعد إعداد الاسم المستعار، يمكنك استخدام هذا الاسم الجديد للتبديل إلى ذلك الإصدار:

```bash
nvm use <new_name>
```

على سبيل المثال:

```bash
nvm use new-api
```

### إزالة إصدار من Node.js

لإزالة إصدار من Node.js، شغّل هذا الأمر:

```bash
nvm uninstall <version>
```

على سبيل المثال:

```bash
nvm uninstall 20.19.5
```

مع NVM على macOS/Linux، يمكنك إزالة إصدار من Node.js بتشغيل أمر `uninstall` مع رقم الإصدار الرئيسي مثل `22` أو `20`. على سبيل المثال:

```bash
nvm uninstall 20
```

إذا كان لديك عدة إصدارات لذلك الرقم الرئيسي مثل `20.14.0` و`20.12.2`، فسيتم إزالة أحدث إصدار أولًا. في هذه الحالة، تشغيل `nvm uninstall 20` سيزيل `20.14.0` أولًا.

### التحقق من إصدار Node.js الخاص بك

للتحقق من إصدار Node.js الحالي، يمكنك استخدام التالي:

```bash
node -v
```

### التحقق من إصدار npm الخاص بك

`npm` هو مدير حزم لـ Node.js. يمكنك استخدامه لتثبيت ونشر وإدارة حزم البرمجيات. يتم تثبيته تلقائيًا عند تثبيت Node.js.

للتحقق من إصدار `npm` الحالي، شغّل هذا الأمر:

```bash
npm -v
```

### تشغيل ملف JavaScript

يشغل هذا الأمر ملف JavaScript الذي تحدده في بيئة تشغيل Node.js النشطة حاليًا:

```bash
node <file.js>
```

على سبيل المثال:

```bash
node app.js
```

### إنشاء مشروع Node.js

ينشئ هذا الأمر مشروع Node.js جديدًا بإنشاء ملف `package.json` الذي يتتبع تفاصيله واعتمادياته.

```bash
npm init
```

عند تشغيل أمر `npm init`، سيرشدك عبر دليل تفاعلي في واجهة سطر الأوامر لإعداد ملف `package.json`. سيطرح عليك أسئلة عن اسم المشروع، الوصف، الإصدار، وغيرها. إذا أردت البدء بملف `package.json` افتراضي، يمكنك استخدام الأمر `npm init -y` أو `npm init --yes`.

### تثبيت حزمة

لتثبيت حزمة محددة، شغّل:

```bash
npm install <package>
```

على سبيل المثال:

```bash
npm install express
```

### تثبيت الاعتماديات

لتثبيت كل الاعتماديات المدرجة في ملف `package.json` لمشروع، شغّل:

```bash
npm install
```
