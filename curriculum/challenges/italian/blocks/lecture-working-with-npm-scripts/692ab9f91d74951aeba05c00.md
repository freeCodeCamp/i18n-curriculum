# --description--

Ora imparerai cosa sono gli script npm.

Prima di iniziare, rivediamo alcuni concetti importanti:

* **npm** è uno strumento da riga di comando che puoi usare per installare e gestire pacchetti JavaScript.
    
* Un **pacchetto** è una directory che raggruppa codice correlato in una struttura logica.
    
* Uno **script** è una sequenza di istruzioni o comandi scritti in un linguaggio di scripting.
    

Gli **script npm** sono comandi personalizzati che puoi definire nel file `package.json` della tua applicazione.

Sono utili per automatizzare attività e flussi di lavoro che userai ripetutamente durante le fasi di sviluppo, test e distribuzione del tuo progetto.

Ad esempio, un caso d’uso per gli script npm è personalizzare `npm start` per avviare l’intero ambiente di sviluppo. Invece di scrivere manualmente tutti i comandi uno per uno ogni volta che devi avviare l’ambiente di sviluppo, puoi semplicemente eseguire `npm start` e l’intero processo verrà eseguito.

Puoi anche usare gli script npm per automatizzare il linting in tutto il codice e per eseguire suite di test, tra gli altri usi.

Sono molto potenti, quindi vediamo i loro fondamenti.

## Come creare un file package.json

Per poter creare ed eseguire script npm personalizzati, ti serve un file `package.json`.

Il file `package.json` è essenziale per i progetti Node.js. Questo file contiene i metadati, le dipendenze e gli script del progetto.

Per ricordarlo, per creare un file `package.json` devi eseguire il comando `npm init` nella directory principale del tuo progetto. Poi puoi inserire manualmente il nome del pacchetto, la versione, la descrizione e altri dettagli, oppure lasciare quei campi vuoti per usare i valori predefiniti.

Una volta completato questo processo, avrai un file `package.json` nella directory principale del tuo progetto.

Se hai mantenuto tutti i valori predefiniti, dovresti vedere queste coppie chiave-valore quando apri `package.json` nel tuo editor di codice:

```json
{
  "name": "<project_name>",
  "version": "1.0.0",
  "main": "index.js",
  "scripts": {
    "test": "echo \"Error: no test specified\" && exit 1"
  },
  "author": "",
  "license": "ISC",
  "description": ""
}
```

Nota questa coppia chiave-valore qui:

```json
"scripts": {
  "test": "echo \"Error: no test specified\" && exit 1"
}
```

Qui specificherai i tuoi script npm, sotto la chiave `scripts`.

Vedrai uno script lì per impostazione predefinita, lo script test:

```json
"test": "echo \"Error: no test specified\" && exit 1"
```

Questo è un esempio di uno script npm definito in `package.json`.

Ecco la sintassi base per definire uno script npm:

```json
"<script_name>": "<commands>"
```

A sinistra dei due punti trovi il nome dello script. I più comuni includono `run`, `start`, `test`, `build`, `dev` e `lint`. Questi nomi fungono da "scorciatoie" per i comandi sottostanti a destra.

A destra dei due punti trovi i comandi sottostanti stessi.

Ad esempio:

```json
"scripts": {
  "start": "node app.js"
}
```

Per eseguire lo script, digita questo nella riga di comando:

```bash
npm run <script_name>
```

Ecco come puoi eseguire qualsiasi script personalizzato definito in `package.json`.

A volte puoi usare questa versione abbreviata senza la parola `run` per eseguire gli script:

```bash
npm <script_name>
```

Ma questo funziona solo per specifici script npm chiamati `start`, `test`, `stop` e `restart`.

Tornando al nostro esempio, se abbiamo questo in `package.json`:

```json
"scripts": {
  "start": "node app.js"
}
```

Per eseguire lo script npm, devi digitare `npm run start` nella riga di comando. Quando lo fai, il comando sottostante effettivamente eseguito è `node app.js`.

Poiché questo script npm ha una versione abbreviata, potresti anche usare `npm start` e il risultato sarebbe lo stesso.

In questo caso, stai solo sostituendo un comando semplice, `node app.js`, con un altro, `npm start`. Ma immagina se avessi una sequenza di comandi complessi che devono essere eseguiti in sequenza o in parallelo, e potessi semplicemente usare `npm start` per eseguirli tutti. Ti farebbe risparmiare molto tempo, giusto? Qui sta la vera potenza degli script npm.

Ecco una panoramica rapida di alcuni degli script npm più comunemente usati:

* `npm run`: usato per eseguire uno script npm personalizzato.
    
* `npm start`: usato per avviare l’applicazione principale o il server di sviluppo.
    
* `npm lint`: usato per eseguire un controllo dello stile del codice per verificare se il codice rispetta gli standard di stile e le buone pratiche.
    
* `npm test`: usato per eseguire la suite di test del progetto.
    
Ci sono molti altri script npm per scopi diversi, ma questi sono alcuni dei più comuni che vedrai.

Inoltre, una convenzione comune è usare i due punti per raggruppare script correlati sotto lo stesso namespace. Per esempio, `start:server` e `start:ui`.

## Come eseguire comandi in sequenza

Se devi eseguire più comandi in sequenza, hai due opzioni. Puoi usare l’operatore logico AND (`&&`) oppure il punto e virgola (`;`). Vediamo le loro differenze.

L’operatore logico AND esegue il comando successivo solo se il primo viene eseguito con successo.

Ad esempio:

```json
"scripts": {
  "build": "webpack --mode production",
  "start": "npm run build && node server.js"
}
```

Nota che c’è un operatore logico AND nel comando `start`:

```json
"start": "npm run build && node server.js"
```

Qui, `npm start` proverà a eseguire prima `npm run build`, e solo se questo script viene eseguito con successo, `node server.js` verrà eseguito.

Questo è un approccio molto comune perché garantisce affidabilità e previene di sprecare tempo e risorse su comandi successivi che potrebbero fallire o lanciare errori se il precedente fallisce.

In alternativa, puoi usare l’operatore punto e virgola per eseguire i comandi in sequenza, indipendentemente dal fatto che il comando precedente sia stato eseguito con successo o meno.

Nel nostro esempio, questa semplice modifica avrebbe un impatto importante su come i comandi verranno eseguiti:

```json
"scripts": {
  "build": "webpack --mode production",
  "start": "npm run build; node server.js"
}
```

Concentrati di nuovo sul comando `start`:

```json
"start": "npm run build; node server.js"
```

Ora, dato che stiamo usando un punto e virgola invece dell’operatore `&&`, il comando `node server.js` verrà sempre eseguito dopo `npm run build`, anche se `node server.js` non è stato eseguito con successo.

```json
"start": "npm run build; node server.js"
```

## Come eseguire comandi contemporaneamente

Se devi eseguire comandi contemporaneamente (in parallelo), puoi usare l’operatore `&`.

```json
"scripts": {
  "start:server": "node server/index.js",
  "start:client": "npm run build:watch",
  "dev": "npm run start:server & npm run start:client"
}
```

Tuttavia, devi sapere che questa opzione esegue i processi in background.

Questo significa che, se li esegui nella stessa sessione del terminale, potresti vedere il loro output combinato, che potrebbe essere difficile da comprendere.

Altre opzioni per eseguire comandi contemporaneamente includono strumenti come `concurrently` e `npm-run-all`.

## Come passare argomenti agli script npm

Se devi passare argomenti direttamente al comando che viene eseguito all’interno dello script npm, puoi specificarli nel comando scrivendo due trattini (`--`) seguiti dagli argomenti.

Il `--` dice a npm che tutto ciò che segue sarà argomenti per i comandi sottostanti che lo script npm eseguirà.

Questa è la sintassi generale:

```bash
npm run <script_name> -- <argument_1> <argument_2> <argument_3>
```

Ad esempio, supponiamo che tu definisca questo script `npm start`:

```json
{
  "name": "project-example",
  "scripts": {
    "start": "node app.js"
  }
}
```

Come potresti specificare la porta su cui vuoi eseguire il server se usi `npm start`?

Supponiamo che vogliamo passare la porta direttamente come argomento da riga di comando. Scriveresti così:

```bash
npm start -- 8000
```

Nel tuo codice, puoi consultare questi argomenti nell’array `process.argv` usando gli indici:

```bash
process.argv[0] # The path to the node executable.
process.argv[1] # The script file being executed.
process.argv[2] # The first argument (8000).
```

Quindi, potresti consultare e usare il numero di porta ovunque ti serva con `process.argv[2]`. Questa è la potenza degli argomenti da riga di comando combinati con gli script npm.

Come puoi vedere, gli script npm sono strumenti essenziali per semplificare il flusso di lavoro di sviluppo. Ti aiutano ad automatizzare e trasformare sequenze di comandi complesse e riutilizzabili in scorciatoie semplici e coerenti che puoi usare nel tuo lavoro quotidiano.
