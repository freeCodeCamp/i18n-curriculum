# --description--

웹 안전 폰트는 컴퓨터나 기기에 설치되어 있을 가능성이 매우 높은 폰트의 부분집합입니다. 이들은 다양한 운영체제와 웹 브라우저에서 널리 지원되므로, 일관되게 렌더링되고 표시될 가능성이 큽니다. 어떻게 작동하는지 살펴보겠습니다.

브라우저는 웹사이트에서 폰트를 해석하고 표시하는 역할을 합니다. 브라우저가 폰트를 렌더링해야 할 때, 사용자의 시스템에서 폰트 파일을 찾으려고 시도합니다. 하지만 폰트를 찾지 못하면 보통 기본 시스템 폰트로 대체 수단을 사용합니다. 이렇게 하면 웹사이트에서 렌더링되어야 할 특정 폰트가 없더라도 내용이 읽을 수 있게 유지됩니다.

그러나 브라우저가 선택한 대체 수단 폰트는 원래 렌더링하려던 폰트와 매우 다르게 보일 수 있습니다. 이는 전체 디자인과 사용자 경험에 치명적인 영향을 줄 수 있습니다. 이를 피하려면 가능한 한 웹 안전 폰트를 사용해야 합니다. 두 가지 선택지가 있습니다. 웹 안전 폰트를 기본 폰트로 사용하거나, 사용자 지정 폰트를 사용할 때 웹 안전 폰트를 대체 수단으로 지정하는 방법입니다. 이렇게 하면 사용자 지정 폰트를 찾지 못할 경우 웹사이트가 어떻게 보일지 제어할 수 있습니다.

웹 안전 폰트의 몇 가지 예를 살펴보겠습니다. 이전 수업에서 산세리프 폰트는 문자 끝에 작은 "발"이나 선이 없어 화면에서 읽기 쉽기 때문에 웹 개발에 일반적으로 사용된다는 것을 배웠습니다. 웹 안전 산세리프 폰트의 예는 다음과 같습니다:

- Arial
- Verdana
- Trebuchet MS

반면에 세리프 폰트는 문자 끝에 작은 "발"이 있어 전통적인 인쇄에 주로 사용됩니다. 하지만 웹 개발에 세리프 폰트를 사용해야 할 경우, 웹 안전 세리프 폰트에는 다음이 포함됩니다:

- Times New Roman
- Georgia

웹 안전 폰트를 사용하면 기기와 플랫폼 전반에서 디자인이 일관되게 보이도록 할 수 있습니다. 또한 단순하고 읽기 쉬워 시각 장애가 있는 사용자의 접근성을 향상할 수 있습니다. 마지막으로, 이미 설치되어 있으면 다운로드할 필요가 없기 때문에 페이지 로드 시간을 줄일 수 있습니다.

이제 웹 안전 폰트가 무엇인지 알았으니, 이를 사용하고 대체 수단으로 할당하여 브라우저와 플랫폼 전반에서 신뢰할 수 있는 사용자 경험을 만들어 보세요.
