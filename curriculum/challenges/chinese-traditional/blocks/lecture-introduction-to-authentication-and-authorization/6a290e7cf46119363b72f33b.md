# --description--

當你建置網頁應用程式時，後端伺服器不僅僅是傳送資料，它還會以 HTTP 回應標頭的形式，將隱藏的資訊回傳給瀏覽器。

預設情況下，Express 的組態會暴露你的伺服器技術細節。攻擊者可以利用這些預設標頭來尋找你應用程式的漏洞。這就是為什麼開發者會使用 Helmet.js。

Helmet.js 是一套用於 Node.js 應用程式的安全中介層集合。它透過自動設定、修改或移除特定的 HTTP 標頭，扮演保護盾牌的角色。

透過管理這些標頭，Helmet 幫助保護你的應用程式免受常見的網路漏洞、惡意腳本和資料竊聽攻擊。

Helmet 不需要你手動組態數十種安全協定，它會預設組態一組較小的中介層函式。安裝只需一條終端機指令：

```bash
npm install helmet
```

要將這層安全防護套用到整個 Express 應用程式，你只需要一行程式碼：

```js
const express = require('express');
const helmet = require('helmet');

const app = express();

// apply Helmet security middleware
app.use(helmet());
```

將 `app.use(helmet())` 放在中介層堆疊的最上方，Helmet 就會攔截每個外送的回應，並安全地組態你的 HTTP 標頭。

當你初始化 Helmet 時，它會自動處理幾項關鍵的安全工作。以下是它設定的最重要組態：

**隱藏伺服器技術（X-Powered-By）**：預設情況下，Express 會透過標頭 `X-Powered-By: Express` 自我宣傳。這會告訴駭客你的後端使用什麼技術，讓他們能針對特定版本的漏洞發動攻擊。Helmet 完全移除此標頭，迫使攻擊者只能盲猜你的底層架構。

**減輕跨站指令碼攻擊（Content-Security-Policy）**：內容安全政策（CSP）是防禦跨站指令碼（XSS）攻擊最強大的防線之一。Helmet 設定嚴格規則，定義瀏覽器允許從哪裡載入腳本、圖片和樣式。這防止惡意注入者在使用者瀏覽器中執行未授權的腳本。

**防止點擊劫持（X-Frame-Options）**：點擊劫持發生在惡意網站將你的網站嵌入隱形的 iframe 中，誘騙使用者點擊他們不想點擊的按鈕。Helmet 將 `X-Frame-Options` 標頭設為 `SAMEORIGIN`，確保你的網站無法被外部不受信任的網域嵌入。

**強制安全連線（Strict-Transport-Security）**：Helmet 強制瀏覽器只透過安全的 HTTPS 連線與你的伺服器通訊，而非不安全的 HTTP。這保護資料不被同一網路上的攻擊者攔截。

簡言之，Helmet 為生產環境應用程式提供了重要的防禦層。雖然它不會修正你自訂程式碼中的漏洞，但它能立即強化你的伺服器組態，抵禦自動掃描器和常見的攻擊手法。
