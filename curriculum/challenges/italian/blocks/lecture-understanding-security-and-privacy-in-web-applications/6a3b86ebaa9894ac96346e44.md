# --description--

CSP (Content Security Policy) e Permissions-Policy sono due potenti header di sicurezza HTTP che aiutano a proteggere le applicazioni web limitando quali risorse possono essere caricate e quali funzionalità del browser una pagina può consultare.

Content Security Policy si concentra principalmente sul caricamento delle risorse. Funziona come un filtro di sicurezza per i tuoi script frontend, stili e immagini.

Gli attori malevoli spesso cercano di iniettare script dannosi in siti vulnerabili. Un CSP forte blocca questi attacchi Cross-Site Scripting (XSS) stabilendo una whitelist rigorosa di domini di origine affidabili.

Quando un server invia l'header CSP, il browser legge le direttive specifiche. Per esempio, la direttiva `script-src` indica al browser quali domini possono eseguire JavaScript in modo sicuro sulla pagina.

Se un hacker tenta di iniettare uno script dannoso da un server esterno non approvato, il browser rileva la violazione della policy, elimina immediatamente lo script e registra un errore di sicurezza.

CSP può anche limitare le origini delle immagini, le destinazioni dei moduli e le origini dei fogli di stile.

Permissions-Policy si concentra sulle funzionalità del browser e sull'hardware del dispositivo. Ha sostituito il vecchio header Feature-Policy.

Questo meccanismo di sicurezza permette ai proprietari dei siti di abilitare o disabilitare esplicitamente specifiche funzionalità del browser per la loro pagina e per qualsiasi iframe incorporato, impedendo agli snippet di terze parti di abusare della privacy degli utenti.

Usando un header Permissions-Policy, puoi controllare l'accesso all'hardware come webcam, microfono e tracciamento della geolocalizzazione.

Per esempio, se il tuo sito non ha bisogno di conoscere la posizione di un utente, puoi impostare la policy per bloccare completamente l'API di geolocalizzazione. Anche se un attaccante compromette uno script sulla tua pagina, il browser rifiuterà di attivare la funzionalità di tracciamento.

L'implementazione di Permissions-Policy usa un formato di header chiaro e strutturato che nomina la funzionalità e definisce le origini consentite.

Per limitare una funzionalità hardware strettamente al tuo dominio, usi la parola chiave `self` racchiusa tra parentesi, come in `camera=(self)`. E se vuoi disabilitare una funzionalità su tutta la pagina per ogni utente, usi invece una lista vuota, che appare come `geolocation=()`.

Combinare entrambi gli header offre una difesa solida per le applicazioni web. CSP blocca la consegna dei dati e previene l'iniezione di codice, mentre Permissions-Policy protegge il dispositivo fisico e garantisce la conformità alla privacy.

Gli sviluppatori web moderni usano entrambi gli strumenti per creare un ambiente di navigazione fortemente protetto.
