# --description--

タプルは、Pythonのデータ型で、順序付けられた値のシーケンスを作成するために使われます。タプルはこのように異なるデータ型を混在させることができます。

```py
developer = ('Alice', 34, 'Rust Developer')
```

タプルはリストに似ていますが、リストが変更可能なデータ型であるのに対し、タプルは変更不可能です。つまり、一度作成されたタプルの要素は変更できません。

もしタプルの中のアイテムを更新しようとすると、`TypeError`が発生します。

```py
programming_languages = ('Python', 'Java', 'C++', 'Rust')
programming_languages[0] = 'JavaScript'

"""
Traceback (most recent call last):
  File "<stdin>", line 2, in <module>
TypeError: 'tuple' object does not support item assignment
"""
```

タプルの要素にアクセスするには、ブラケット記法とインデックス番号を使います。

```py
developer = ('Alice', 34, 'Rust Developer')
developer[1] # 34
```

タプルの末尾から要素にアクセスする必要がある場合は、負のインデックスを使うことができます。以下は、タプルの後ろから2番目の要素にアクセスする例です。

```py
numbers = (1, 2, 3, 4, 5)
numbers[-2] # 4
```

もしタプルの長さを超えるか等しいインデックス番号を渡そうとすると、次のような`IndexError`が発生します。

```py
numbers = (1, 2, 3, 4, 5)
numbers[7]

"""
Traceback (most recent call last):
  File "<stdin>", line 2, in <module>
IndexError: list index out of range
"""
```

タプルを作成するもう一つの方法は、`tuple()`コンストラクタを使うことです。

```py
developer = 'Jessica'
tuple(developer) # ('J', 'e', 's', 's', 'i', 'c', 'a')
```

`tuple()`コンストラクタには、文字列、リスト、さらには他のタプルなど、さまざまなイテラブルを渡すことができます。

タプルにアイテムが含まれているかどうかを確認するには、`in`キーワードを使います。

```py
programming_languages = ('Python', 'Java', 'C++', 'Rust')

'Rust' in programming_languages # True
'JavaScript' in programming_languages # False
```

リストと同様に、タプルからアイテムをアンパックすることもできます。

```py
developer = ('Alice', 34, 'Rust Developer')
name, age, job = developer

print(name) # 'Alice'
print(age) # 34
print(job) # 'Rust Developer'
```

この例では、`name`は`'Alice'`の値を持ち、`age`は`34`の値を持ち、`job`は`'Rust Developer'`の値を持ちます。

タプルから残りの要素をまとめて取得したい場合は、アスタリスク（`*`）演算子を使うことができます。

```py
developer = ('Alice', 34, 'Rust Developer')
name, *rest = developer

print(name) # 'Alice'
print(rest) # [34, 'Rust Developer']
```

ここで、`name`は`'Alice'`の値を持ち、`rest`は数値`34`と文字列`'Rust Developer'`からなるリストです。

リストと同様に、スライス演算子を使ってタプルの一部を抽出することができます。以下は、`'pie'`と`'cookies'`のアイテムを別のタプルに抽出する例です。

```py
desserts = ('cake', 'pie', 'cookies', 'ice cream')
desserts[1:3] # ('pie', 'cookies')
```

最初の数字は抽出の開始インデックスを表し、2番目の数字は終了インデックスを表します。ただし、終了インデックスのアイテムは抽出されたタプルに含まれません。

タプルからアイテムを削除することはできません。なぜならタプルは変更不可能だからです。この例はエラーになります。

```py
developer = ('Jane Doe', 23, 'Python Developer')
del developer[1]

"""
Traceback (most recent call last):
  File "<stdin>", line 2, in <module>
TypeError: "tuple" object doesn't support item deletion
"""
```

では、リストではなくタプルを使うのはどんな場合でしょうか？

要素を追加、削除、更新できる動的なコレクションが必要な場合はリストを使うべきです。固定されていて変更されないデータのコレクションを扱う場合はタプルを使うべきです。
