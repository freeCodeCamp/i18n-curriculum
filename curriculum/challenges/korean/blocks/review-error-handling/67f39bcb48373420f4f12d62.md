## 파이썬에서 자주 발생하는 오류

- **SyntaxError**: 코드가 파이썬 구문 규칙을 따르지 않을 때 발생하는 오류입니다. 예를 들어, `print("Hello there"` 코드는 닫는 괄호가 없기 때문에 `SyntaxError: '(' was never closed` 메시지와 함께 구문 오류가 발생합니다.
- **NameError**: 정의하지 않은 변수나 함수를 접근하려 할 때 파이썬이 `NameError`를 발생시킵니다. 예를 들어, `print(username)` 코드가 있고 `username` 변수가 먼저 정의되어 있지 않으면 `NameError: name 'username' is not defined` 메시지와 함께 이름 오류가 발생합니다.
- **TypeError**: 서로 호환되지 않는 데이터 타입에 대해 연산을 수행할 때 파이썬이 발생시키는 오류입니다. 예를 들어, 문자열과 숫자를 더하려 하면 `TypeError: can only concatenate str (not "int") to str` 오류가 발생합니다.
- **IndexError**: 리스트나 튜플, 문자열 같은 시퀀스에서 존재하지 않는 인덱스를 접근하면 `IndexError`가 발생합니다. 예를 들어, `Hello world` 문자열에서 마지막 문자의 인덱스는 `11`입니다. `greet = "hello world"; print(greet[12])` 방식으로 문자를 접근하면 `IndexError: string index out of range` 메시지와 함께 오류가 발생합니다.
- **AttributeError**: 해당 타입 객체에 존재하지 않는 메서드나 속성을 사용하려 할 때 발생하는 오류입니다. 예를 들어, `.append()` 같은 문자열에서 `"hello".append("!")`를 호출하면 `AttributeError: 'str' object has no attribute 'append'` 메시지와 함께 오류가 발생합니다.

## 파이썬에서 좋은 디버깅 기법

- **`print` 함수 사용하기**: 디버깅 중 코드 여러 지점에 `print()` 문을 삽입하면 변수 값과 코드 흐름을 확인하는 데 도움이 됩니다.
- **파이썬 내장 디버거(`pdb`) 사용하기**: 파이썬은 `pdb` 모듈을 제공하며, 표준 라이브러리의 일부라 항상 사용할 수 있습니다. `pdb`를 사용하면 `set_trace()` 함수로 추적을 설정해 대화형으로 코드를 단계별 실행하고 변수를 검사할 수 있습니다.
- **IDE 디버깅 도구 활용하기**: Pycharm, VS Code 같은 통합 개발 환경과 코드 편집기는 중단점, 단계 실행, 변수 검사 등 다양한 디버깅 기능을 제공합니다.

## 예외 처리

- **`try...except`**: 예외가 발생할 수 있는 코드 블록을 실행할 때 사용합니다. `try` 블록은 오류가 발생할 가능성이 있는 부분이고, `except` 블록은 특정 예외가 발생했을 때 실행됩니다. 예시는 다음과 같습니다:

  ```py
  try:
    print(22 / 0)
  except ZeroDivisionError:
    print('You can\'t divide by zero!')
    # You can't divide by zero!
  ```

  여러 `except` 블록을 연결해 다양한 예외를 처리할 수도 있습니다:

  ```py
  try:
    number = int(input('Enter a number: '))
    print(22 / number)
  except ZeroDivisionError:
    print('You cannot divide by zero!')
    # You cannot divide by zero! prints when you enter 0
  except ValueError:
    print('Please enter a valid number!')
    # Please enter a valid number! prints when you enter a string  
  ```

- **`else`와 `finally`**: 이 블록들은 `try...except`를 확장합니다. 예외가 발생하지 않으면 `else` 블록이 실행되고, `finally` 블록은 오류 여부와 상관없이 항상 실행됩니다.

  ```py
  try:
    result = 100 / 4
  except ZeroDivisionError:
    print('You cannot divide by zero!') # This will not run
  else:
    print(f'Result is {result}') # Result is 25.0
  finally:
    print('Execution complete!') # Execution complete!
  ```

- **예외 객체**: 예외 자체에 접근해 디버깅과 직접적인 오류 메시지 출력을 할 수 있습니다. 예외 객체에 접근하려면 `as` 키워드를 사용해야 합니다. 예시는 다음과 같습니다:

  ```py
  try:
      value = int('This will raise an error')
  except ValueError as e:
      print(f'Caught an error: {e}')
      # Caught an error: invalid literal for int() with base 10: 'This will raise an error'
  ```

- **`raise` 문**: 수동으로 예외를 발생시킬 때 사용합니다. 특정 조건이 충족되면 예외를 발생시키는 데 쓸 수 있습니다. 예시는 다음과 같습니다:

  ```py
  def divide(a, b):
      if b == 0:
          raise ZeroDivisionError('You cannot divide by zero')
      return a / b
  ```
  
## 예외 신호 보내기
  
  `raise` 문은 사용자 정의 예외를 만들 때도 유용하며, 사용자 정의 메시지와 함께 예외를 발생시킬 수 있습니다. 예시는 다음과 같습니다:

  ```py
  class InvalidCredentialsError(Exception):
      def __init__(self, message="Invalid username or password"):
          self.message = message
          super().__init__(self.message)

  def login(username, password):
      stored_username = "admin"
      stored_password = "password123"
      
      if username != stored_username or password != stored_password:
          raise InvalidCredentialsError()
      
      return f"Welcome, {username}!"
  ```  

  `login` 함수와 `InvalidCredentialsError` 예외를 함께 사용하는 방법은 다음과 같습니다:

  ```py
  # failed login attempt
  try:
      message = login("user", "wrongpassword")
  except InvalidCredentialsError as e:
      print(f"Login failed: {e}")
  else:
      print(message)

  # successful login attempt
  try:
      message = login("admin", "password123")
  except InvalidCredentialsError as e:
      # This block is not executed because the login was successful
      print(f"Login failed: {e}")
  else:
      # The else block runs if the 'try' block completes without an exception
      print(message)
  ```

  `raise` 문은 `from` 키워드와 함께 예외를 연결해 서로 다른 오류 간 관계를 보여줄 때도 사용할 수 있습니다:

  ```py
  def parse_config(filename):
    try:
        with open(filename, 'r') as file:
            data = file.read()
            return int(data)
    except FileNotFoundError:
        raise ValueError('Configuration file is missing') from None
    except ValueError as e:
        raise ValueError('Invalid configuration format') from e

  config = parse_config('config.txt')
  ```
