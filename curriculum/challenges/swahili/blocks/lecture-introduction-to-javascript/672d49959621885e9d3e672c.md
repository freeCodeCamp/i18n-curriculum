# --interactive--

Unapofanya kazi na JavaScript, mara nyingi utatangaza vigezo kuhifadhi data unayopanga kuitumia katika programu yako yote.

Katika JavaScript ya kisasa, `let` na `const` ni njia zinazopendekezwa zaidi za kutangaza vigezo, lakini zinatofautiana jinsi zinavyoshughulikia mgawo wa thamani na mgawo upya wa thamani.

Katika funzo hili, tutaangalia jinsi `let` na `const` zinavyotofautiana katika tamko, mgawo wa thamani, na mgawo upya wa vigezo.

Neno kuu `let` linakuwezesha kutangaza vigezo ambavyo vinaweza kusasishwa au kupewa thamani tena baadaye. Unaweza kufikiria `let` kama kontena lenye kubadilika: mara tu unapohifadhi thamani ndani yake, unaweza kubadilisha thamani hiyo kadri inavyohitajika katika programu yako yote.

Hapa kuna mfano wa kutangaza na kuweka thamani kwa kigezo kwa kutumia `let`:

```js
let score = 10;
```

Katika kesi hii, kigezo `score` kime tangazwa na kimepewa thamani `10`. Ikiwa unataka kusasisha thamani baadaye, unaweza kufanya hivyo kwa urahisi:

:::interactive_editor

```js
let score = 10;
console.log(score); // 10
score = 20;
console.log(score); // 20
```

:::

Sasa, `score` inashikilia thamani `20`. Hii inafanya `let` kuwa na manufaa hasa wakati unajua thamani ya kigezo itabadilika wakati programu yako inapo endeshwa.

Kwa upande mwingine, `const` hutumika kutangaza vigezo ambavyo ni thabiti. Mara tu unapoweka thamani kwa kigezo kilichotangazwa kwa `const`, huwezi kugawa upya thamani hiyo.

Hii inafanya `const` kuwa bora kwa thamani ambazo hutaki kubadilishwa kwa bahati mbaya wakati wa utekelezaji wa programu yako.

Hapa kuna mfano wa kutangaza na kuweka thamani kwa kigezo kwa kutumia `const`:

:::interactive_editor

```js
const maxScore = 100;
console.log(maxScore); // 100
```

:::

Mara `maxScore` imepewa thamani `100`, haiwezi kubadilishwa:

```js
maxScore = 200; // This will result in an error
```

Kujaribu kugawa upya thamani kwa kigezo cha `const` kuta tupa kosa katika konsoli yako ya JavaScript, kwani vigezo vya `const` haviwezi kubadilishwa mara baada ya kupewa thamani.

Unaweza kutangaza kigezo cha `let` bila mara moja kupewa thamani, na unaweza kukiwekea thamani baadaye:

:::interactive_editor

```js
let age;
console.log(age); // undefined
age = 25;
console.log(age); // 25
```

:::

Vigezo vilivyotangazwa kwa `const` lazima viwe na thamani wakati wa tamko. Ikiwa utajaribu kutangaza kigezo cha `const` bila kupewa thamani, utapata kosa:

```js
const age; // Error: Missing initializer in const declaration
```

Unapaswa kutumia `let` unapohitaji kutangaza vigezo ambavyo vita gawiwa upya baadaye. Kwa mfano, kufuatilia alama zinazo badilika au kusasisha thamani kwa muda katika programu yako.

Tumia `const` unapotaka kutangaza vigezo ambavyo vinapaswa kubaki thabiti, kama vile thamani za usanidi au mipangilio ambayo haipaswi kubadilishwa kwa bahati mbaya.

Unaweza pia kutumia neno kuu la `var`, lakini haipendekezwi tena sana. `var` ni kama `let`, isipokuwa lina wigo mpana zaidi, jambo ambalo lina uwezekano mkubwa wa kusababisha matatizo katika programu yako.
