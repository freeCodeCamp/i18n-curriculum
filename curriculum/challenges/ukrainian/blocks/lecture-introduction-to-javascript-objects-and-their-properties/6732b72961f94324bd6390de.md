# --interactive--

У JavaScript існує кілька способів перевірити, чи має об’єкт певну властивість. Розуміння цих методів важливе для ефективної роботи з об’єктами, особливо коли ви працюєте з даними з зовнішніх джерел або коли потрібно переконатися, що певні властивості існують перед їх використанням.

Ми розглянемо кілька загальних підходів: метод `hasOwnProperty()`, метод `Object.hasOwn()`, оператор `in` та перевірку через `undefined`.

Почнемо з методу `hasOwnProperty()`. Цей метод повертає булеве значення, яке вказує, чи має об’єкт вказану властивість як власну властивість. Ось приклад:

:::interactive_editor

```js
const person = {
  name: "Alice",
  age: 30
};

console.log(person.hasOwnProperty("name")); // true
console.log(person.hasOwnProperty("job")); // false
```

:::

У цьому прикладі у нас є об’єкт `person` з двома властивостями: `name` і `age`. Щоб перевірити, чи є `name` властивістю об’єкта `person`, ми використовуємо метод `hasOwnProperty()`. Оскільки `name` є властивістю, він поверне `true`. Але коли ми використовуємо `hasOwnProperty()` для перевірки, чи є `job` властивістю, він поверне `false`, бо такої властивості в об’єкті немає.

`Object.hasOwn()` — це сучасний, рекомендований спосіб перевірки, чи має об’єкт власну (не успадковану) властивість. Можна уявити його як оновлену, безпечнішу версію `hasOwnProperty()`. Синтаксис такий: `Object.hasOwn(object, propertyName)` — ви передаєте об’єкт як перший аргумент і назву властивості як другий.

Ось базовий приклад:

:::interactive_editor

```js
const person = {
  name: "Alice",
  age: 30
};

console.log(Object.hasOwn(person, "name")); // true
console.log(Object.hasOwn(person, "job")); // false
```

:::

У цьому прикладі `Object.hasOwn(person, "name")` повертає `true`, бо `name` існує безпосередньо в об’єкті `person`. `Object.hasOwn(person, "job")` повертає `false`, бо `job` ніколи не додавали до об’єкта.

Дуже важливо розуміти, що `Object.hasOwn()` перевіряє лише, чи **існує** властивість — йому байдуже до значення властивості. Це означає, що він все одно поверне `true`, навіть якщо значення є `0`, `false`, `null` або `undefined`:

:::interactive_editor

```js
const user = {
  username: "coder123",
  score: 0,
  isActive: false,
  nickname: null
};

// Object.hasOwn() correctly reports these all exist
console.log(Object.hasOwn(user, "score"));    // true  (value is 0, but property exists)
console.log(Object.hasOwn(user, "isActive")); // true  (value is false, but property exists)
console.log(Object.hasOwn(user, "nickname")); // true  (value is null, but property exists)
console.log(Object.hasOwn(user, "email"));   // false (property was never added)

// Danger! Using if() directly gives wrong results for falsy values
if (user.score) {
  console.log("Has score"); // This will NOT print even though score exists!
}

// Safe! Object.hasOwn() gives correct result
if (Object.hasOwn(user, "score")) {
  console.log("Has score:", user.score); // Has score: 0
}
```

:::

Ще один спосіб перевірити наявність властивості в об’єкті — використати оператор `in`. Як і `hasOwnProperty()`, оператор `in` поверне `true`, якщо властивість існує в об’єкті. Ось як це можна зробити:

:::interactive_editor

```js
const person = {
  name: "Bob",
  age: 25
};
console.log("name" in person);  // true
```

:::

У цьому прикладі `"name" in person` повертає `true`, бо `name` є властивістю об’єкта `person`.

Третій метод полягає в перевірці, чи властивість є `undefined`. Цей підхід може бути корисним, але має деякі обмеження. Ось приклад:

:::interactive_editor

```js
const car = {
  brand: "Toyota",
  model: "Corolla",
  year: 2020
};

console.log(car.brand !== undefined); // true
console.log(car.color !== undefined); // false
```

:::

У цьому коді ми перевіряємо, чи `car.brand` і `car.color` не є `undefined`. Це працює, бо доступ до неіснуючої властивості об’єкта повертає `undefined`. Однак цей метод може давати хибні негативні результати, якщо властивість явно має значення `undefined`.

На практиці вибір між цими методами часто залежить від конкретних вимог вашого коду. Розуміння відмінностей між ними допоможе зробити правильний вибір у різних ситуаціях.
