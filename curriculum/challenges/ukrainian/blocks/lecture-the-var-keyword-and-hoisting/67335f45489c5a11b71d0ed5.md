# --interactive--

Підняття — це поведінка в JavaScript, яка часто плутає початківців, але розуміння її допоможе уникнути прихованих помилок у вашому коді. Простими словами, підняття — це стандартна поведінка JavaScript, яка полягає в тому, що оголошення переміщуються на початок відповідних областей видимості під час фази компіляції перед виконанням коду.

Щоб зрозуміти підняття, важливо знати, що JavaScript працює у дві фази: фаза компіляції та фаза виконання. Під час фази компіляції движок JavaScript проходить по вашому коду і виділяє пам’ять для змінних і функцій. Саме тут і відбувається підняття.

Почнемо з підняття змінних: коли ви оголошуєте змінну за допомогою ключового слова `var`, JavaScript піднімає оголошення на початок її області видимості. Однак важливо зазначити, що піднімається лише оголошення, а не ініціалізація. Це означає, що ви можете використовувати змінну у коді до того, як її оголосили, але її значення буде `undefined`, доки ви фактично не призначите їй значення.

:::interactive_editor

```js
console.log(x); // undefined
var x = 5;
console.log(x); // 5
```

:::

У цьому коді, хоча ми використовуємо `x` до його оголошення, помилки не виникає, натомість ми отримуємо `undefined`. Це тому, що JavaScript піднімає оголошення `var x` на початок області видимості, але не ініціалізацію `x = 5`. Можна уявити, що код переписаний так:

:::interactive_editor

```js
var x;
console.log(x); // undefined
x = 5;
console.log(x); // 5
```

:::

Підняття функцій працює трохи інакше. Коли ви оголошуєте функцію за допомогою синтаксису оголошення `function`, і ім’я функції, і тіло функції піднімаються. Це означає, що ви можете викликати функцію до того, як оголосили її у коді. Ось приклад підняття функції:

:::interactive_editor

```js
sayHello(); // "Hello, World!"

function sayHello(){
  console.log("Hello, World!");
}
```

:::

У цьому випадку ми можемо викликати `sayHello()` до її оголошення, бо вся функція піднята на початок області видимості.

Важливо зазначити, що підняття працює інакше з оголошеннями `let` і `const`, які з’явилися в ES6.

```js
console.log(y); // Throws a ReferenceError
let y = 10;
```

Ці оголошення піднімаються, але не ініціалізуються, і ви не можете отримати до них доступ до фактичного оголошення у коді. Це часто називають тимчасовою зоною мертвих.

Розуміння підняття допоможе писати чистіший і передбачуваніший код. Однак покладатися на підняття може ускладнити читання і підтримку коду. Як найкращу практику рекомендується оголошувати змінні на початку їхньої області видимості, а функції — перед їх використанням, незалежно від підняття. Це робить поведінку коду більш явною і зрозумілою для інших, включно з вашим майбутнім «я».
