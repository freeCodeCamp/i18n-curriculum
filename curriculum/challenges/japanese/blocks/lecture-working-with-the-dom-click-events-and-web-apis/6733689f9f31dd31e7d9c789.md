# --description--

DOMノードとDOMツリー内でのそれらの関係について学びましょう。

実際の木が大きな枝や小さな枝が階層構造でつながっているのと同じように、DOMノードも互いに直接的または間接的な関係を持っています。この例を使ってこれらの関係を説明します。

```html
<!DOCTYPE html>
<html>
  <head>
    <title>DOM Tree Example</title>
  </head>
  <body>
    <h1>Heading 1</h1>
    <p>Paragraph 1</p>
    <ul>
      <li>List item 1</li>
      <li>List item 2</li>
    </ul>
  </body>
</html>
```

まずは一番上から始めましょう。DOMツリーのルートは`html`要素です。これはHTMLドキュメントのすべての内容を含む最上位のコンテナです。ほかのすべてのノードはこのルートノードの子孫要素です。

次に、ルートノードの下には階層内の他のノードがあります。親ノードは他の要素を含む要素です。子ノードは他の要素に含まれている要素です。

この例では、`body`要素が段落要素の親ノードであり、段落要素は`body`要素の子ノードです。これらの要素はDOMツリー内で同じ関係を持つノードとして表されています。

親ノードと子ノードがあるように、兄弟ノードもあります。兄弟ノードは同じ親を共有する要素です。この例では、2つのリストアイテム要素が同じ番号なしリストの親を共有しているので兄弟ノードです。

同様に、`h1`要素と段落要素は同じ親である`body`要素を共有しているため兄弟ノードです。

また、階層の異なるレベル間での間接的な関係もあります。子孫要素は、直接的または間接的に他の要素に含まれている要素です。ある要素が別の要素の子孫要素とみなされるのは、その祖先からDOMツリーを下ってたどり着ける場合です。

この例では、リストアイテムの`li`要素は`body`要素の子孫要素です。なぜなら間接的にその中に含まれているからです。

祖先ノードは、DOMツリーの階層で他の要素よりも上位にある要素です。この例では、`body`要素がリストアイテムの`li`要素の祖先ノードです。

これらの関係を理解することは、JavaScriptでDOMツリーを操作したりナビゲートしたりするために不可欠です。
