# --description--

Як ми знаємо, сприймана швидкодія — це не лише про те, наскільки швидко сайт працює насправді, а про те, наскільки швидко він здається користувачам. Наприклад, чи користувалися ви коли-небудь сайтом, який здавався повільним, хоча насправді таким не був? Це і є сприймана швидкодія — наскільки швидко сайт здається користувачу.

Невеликі зміни можуть суттєво покращити досвід користувача. Давайте розглянемо кілька практичних способів покращити сприйману швидкодію!

Індикатори завантаження можуть створювати враження, що користувачі чекають довше, ніж насправді. Замість цього спробуйте скелетон-екрани — сірі заповнювачі, які імітують реальний вміст і поступово завантажуються:

```html
<div class="skeleton"></div>
<script>
  setTimeout(() => {
    document.querySelector(".skeleton").innerHTML = "✅ Content Loaded!";
  }, 2000);
</script>
```

Користувачі відчувають, що сторінка завантажується поступово, а не чекають у темряві.

Крім того, користувачам не потрібно чекати, поки все завантажиться, щоб почати взаємодіяти. Спочатку завантажуйте важливий вміст, а решту — у фоновому режимі. Ми вже розглядали це раніше з лінивим завантаженням:

```html
<img src="placeholder.jpg" data-src="real-image.jpg" class="lazy">
<script>
  document.addEventListener("DOMContentLoaded", function () {
    const lazyImages = document.querySelectorAll("img.lazy");
    lazyImages.forEach(img => {
      img.src = img.dataset.src;
    });
  });
</script>
```

Це робить сторінку адаптивною, навіть якщо не все ще завантажено.

А тепер — що якби ваш сайт міг передбачити, що користувач натисне далі? За допомогою передзавантаження ви можете отримати наступну сторінку ще до того, як користувач її запросить:

```html
<link rel="preload" href="next-page.html" as="document">
```

Це зменшує сприйманий час завантаження майже до нуля під час навігації.

І нарешті, користувачі не люблять невизначеність. Коли вони натискають кнопку, миттєво підтверджуйте їхню дію:

```html
<button onclick="this.innerText='⏳ Processing...'; setTimeout(() => this.innerText='✅ Done!', 2000);">
  Click Me
</button>
```

Це означає, що навіть якщо є затримка, користувач відчуває, що щось відбувається.

Отже, підсумовуючи, покращення сприйманої швидкодії — це не лише про сиру швидкість, а про те, як швидкість відчувається користувачем. Використовуйте скелетон-екрани, надавайте пріоритет важливому вмісту, розумно передзавантажуйте і давайте миттєвий зворотний зв’язок, щоб ваш сайт здавався швидшим.
