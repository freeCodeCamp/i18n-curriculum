# --description--

如果一个表达式中有多个操作符，执行顺序由它们的优先级定义，换句话说，就是操作符的默认优先级顺序。

从学校你就知道表达式 `1 + 2 * 2` 中的乘法应该在加法之前计算。这正是优先级的含义。乘法被认为比加法具有更高的优先级。

括号覆写任何优先级，所以如果你对默认顺序不满意，可以使用它们来更改。例如，写 `(1 + 2) * 2`。

JavaScript 中有许多操作符。每个操作符都有一个对应的优先级数字。数字较大的先执行。如果优先级相同，执行顺序从左到右。

以下是优先级表的摘录（你不需要记住它，但请注意一元操作符的优先级高于相应的二元操作符）：

| Precedence | Name             | Sign |
|------------|------------------|------|
| ...        | ...              | ...  |
| 14         | unary plus       | +    |
| 14         | unary negation   | -    |
| 13         | exponentiation   | **   |
| 12         | multiplication   | *    |
| 12         | division         | /    |
| 11         | addition         | +    |
| 11         | subtraction      | -    |
| ...        | ...              | ...  |
| 2          | assignment       | =    |
| ...        | ...              | ...  |



# --questions--

## --text--

在处理 JavaScript 表达式中的多个操作符时，关于执行顺序哪个语句是正确的？

## --answers--

操作符从右向左执行，无论它们的优先级如何。

---

优先级更高的操作符先执行，括号可以覆写预定义的优先级。

---

操作符的优先级由操作符的执行速度决定。

---

乘法总是优先于加法执行，一元操作符在二元操作符之后执行。
