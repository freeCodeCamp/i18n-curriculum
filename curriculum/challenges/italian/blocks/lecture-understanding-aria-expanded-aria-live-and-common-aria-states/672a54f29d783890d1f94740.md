# --interactive--

Gli elementi di controllo del modulo semantici come `input`, `select`, `textarea`, `button` e `fieldset` hanno stati integrati che vengono comunicati alle tecnologie assistive.

Ad esempio, potresti usare l'attributo `disabled` per disabilitare un pulsante o l'attributo `checked` per indicare che una casella di spunta è selezionata.

Ma se stai creando un elemento di controllo personalizzato, devi usare gli attributi ARIA per comunicare lo stato del controllo alle tecnologie assistive.

In questa lezione, discuteremo alcuni stati ARIA comuni che puoi usare sugli elementi di controllo personalizzati.

Il primo stato ARIA di cui parleremo è `aria-selected`. Questo stato serve a indicare che un elemento è selezionato. Puoi usare questo stato su controlli personalizzati come un'interfaccia a schede, una listbox o una griglia.

Ecco un esempio di come puoi usare `aria-selected` su un controllo a schede personalizzato:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<div role="tablist">
  <button role="tab" aria-selected="true">Tab 1</button>
  <button role="tab" aria-selected="false">Tab 2</button>
  <button role="tab" aria-selected="false">Tab 3</button>
</div>
<script src="index.js"></script>
```

```css
[role="tablist"] {
  display: flex;
  border-bottom: 2px solid #ddd;
  gap: 0.25rem;
  font-family: system-ui, sans-serif;
}

[role="tab"] {
  appearance: none;
  border: none;
  background: none;
  padding: 0.5rem 1rem;
  cursor: pointer;
  font-size: 1rem;
  color: #444;
  border-radius: 4px 4px 0 0;
  transition: background-color 0.2s, color 0.2s;
}

[role="tab"]:hover {
  background-color: #f3f3f3;
}

[role="tab"][aria-selected="true"] {
  background-color: #fff;
  color: #0078d4;
  border: 2px solid #0078d4;
  border-bottom: 2px solid #fff; 
  font-weight: 600;
  position: relative;
  z-index: 1;
}

[role="tab"]:focus {
  outline: 2px solid #0078d4;
  outline-offset: 2px;
}

```

```js
document.addEventListener("click", (event) => {
  const clickedTab = event.target.closest('[role="tab"]');
  if (!clickedTab) return;

  const tablist = clickedTab.closest('[role="tablist"]');
  const tabs = tablist.querySelectorAll('[role="tab"]');

  tabs.forEach((tab) => {
    const isSelected = tab === clickedTab;
    tab.setAttribute("aria-selected", isSelected);
    tab.tabIndex = isSelected ? 0 : -1;
  });
});

document.addEventListener("keydown", (event) => {
  const activeTab = document.activeElement;
  if (activeTab.getAttribute("role") !== "tab") return;

  const tablist = activeTab.closest('[role="tablist"]');
  const tabs = Array.from(tablist.querySelectorAll('[role="tab"]'));
  const index = tabs.indexOf(activeTab);

  let newIndex = index;
  if (event.key === "ArrowRight") newIndex = (index + 1) % tabs.length;
  if (event.key === "ArrowLeft") newIndex = (index - 1 + tabs.length) % tabs.length;

  if (newIndex !== index) {
    tabs[newIndex].focus();
    tabs[newIndex].click();
  }
});

```

:::

Le schede sono usate per mostrare più pannelli di contenuto in uno spazio limitato. Lo stato `aria-selected` serve a indicare quale scheda è attualmente selezionata.

Quando l'utente seleziona una scheda, lo stato `aria-selected` della scheda selezionata viene impostato su `true`, e lo stato `aria-selected` delle altre schede viene impostato su `false`.

Un altro stato ARIA comune è `aria-disabled`. Questo stato serve a indicare che un elemento è disabilitato solo per le persone che usano tecnologie assistive, come i lettori di schermo. È importante notare che `aria-disabled` non disabilita effettivamente l'elemento. Spetta a te, lo sviluppatore, farlo sembrare e comportarsi come un elemento disabilitato. Questo attributo è anche comunemente usato sugli elementi HTML nativi al posto dell'attributo `disabled`. Quale scegliere dipenderà dal contesto in cui il pulsante viene usato.

Ecco un esempio di come puoi usare `aria-disabled` su un pulsante di modifica personalizzato:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<div role="button" tabindex="-1" aria-disabled="true">Edit</div>
```

```css
[role="button"] {
  display: inline-block;
  background-color: #0078d4;
  color: #fff;
  padding: 0.5rem 1rem;
  border-radius: 6px;
  font-family: system-ui, sans-serif;
  font-size: 1rem;
  text-align: center;
  cursor: pointer;
  user-select: none;
  transition: background-color 0.2s, transform 0.1s, opacity 0.2s;
}

[role="button"]:not([aria-disabled="true"]):hover {
  background-color: #005fa3;
}

[role="button"]:not([aria-disabled="true"]):focus {
  outline: 2px solid #005fa3;
  outline-offset: 2px;
}

[role="button"]:not([aria-disabled="true"]):active {
  transform: scale(0.97);
}

[role="button"][aria-disabled="true"] {
  opacity: 0.5;
  pointer-events: none;
  cursor: not-allowed;
  background-color: #b0b0b0;
  color: #f2f2f2;
}
```

:::

L'attributo `aria-disabled` serve a comunicare agli utenti di lettori di schermo che il pulsante di modifica è disabilitato e non può essere interagito. Ancora una volta, non disabilita effettivamente il pulsante. Quando usi `aria-disabled`, dovrai applicare stili e JavaScript per far sembrare e comportare il controllo come un pulsante disabilitato.

Nella maggior parte dei casi probabilmente userai l'elemento pulsante nativo, ma ci sono casi in cui potresti dover usare un controllo personalizzato. Quindi è essenziale sapere come comunicare lo stato del controllo alle tecnologie assistive.

Il prossimo stato ARIA di cui parleremo è `aria-haspopup`. Questo stato serve a indicare che un elemento interattivo attiverà un elemento popup quando viene attivato. Puoi usare l'attributo `aria-haspopup` solo quando il popup ha uno dei seguenti ruoli: `menu`, `listbox`, `tree`, `grid` o `dialog`. Il valore di `aria-haspopup` deve essere uno di questi ruoli o `true`, che per impostazione predefinita corrisponde al ruolo `menu`.

Ecco un esempio di un menu editor di file che usa `aria-haspopup`:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<button id="menubutton" aria-haspopup="menu" aria-controls="filemenu" aria-expanded="false">File</button>
<ul id="filemenu" role="menu" aria-labelledby="menubutton" hidden>
  <li role="menuitem" tabindex="-1">Open</li>
  <li role="menuitem" tabindex="-1">New</li>
  <li role="menuitem" tabindex="-1">Save</li>
  <li role="menuitem" tabindex="-1">Delete</li>
</ul>
```

```css
#menubutton {
  background-color: #0078d4;
  color: #fff;
  border: none;
  padding: 8px 14px;
  border-radius: 4px;
  cursor: pointer;
  font-size: 14px;
  position: relative;
}

#menubutton:hover,
#menubutton:focus {
  background-color: #005ea2;
  outline: none;
}

#filemenu {
  list-style: none;
  padding: 4px 0;
  margin: 4px 0 0;
  border: 1px solid #ccc;
  border-radius: 4px;
  background-color: #fff;
  width: 160px;
  position: absolute;
  z-index: 1000;
  box-shadow: 0 4px 6px rgba(0,0,0,0.1);
}

#filemenu[hidden] {
  display: none;
}

#filemenu [role="menuitem"] {
  display: block;
  padding: 8px 12px;
  font-size: 14px;
  color: #333;
  cursor: pointer;
}

#filemenu [role="menuitem"]:hover,
#filemenu [role="menuitem"]:focus {
  background-color: #e5f1fb;
  outline: none;
}

#filemenu [role="menuitem"]:focus-visible {
  box-shadow: inset 0 0 0 2px #0078d4;
}

```

:::

Lo stato `aria-haspopup` serve a indicare che il pulsante menu `File` aprirà un menu popup quando attivato. Gli utenti di lettori di schermo potrebbero sentire questa informazione aggiuntiva quando navigano sul pulsante.

Dovrai usare JavaScript per mostrare e nascondere il menu popup e per implementare un supporto adeguato alla tastiera per interagire con il menu. Inoltre, tieni presente che il ruolo ARIA `menu` si riferisce a un tipo molto specifico di menu. Generalmente indica una lista di azioni che l'utente può invocare, simile a un menu in un'applicazione desktop. Non include gli usi più comuni di ciò che normalmente chiamiamo "menu", come i menu di navigazione. Realisticamente, la maggior parte dei "menu" che crei sul web non saranno menu ARIA e non userai `aria-haspopup` con essi.

Il prossimo stato ARIA di cui parleremo è `aria-required`. L'attributo `aria-required` serve a indicare che un campo deve essere compilato prima che il modulo venga inviato.

Ecco un esempio di come lavorare con l'attributo `aria-required` per un controllo modulo personalizzato.

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<div id="name-label">Full Name*</div>
<div role="textbox" contenteditable aria-labelledby="name-label" aria-required="true" id="name"></div>
```

```css
#name-label {
  font-family: system-ui, sans-serif;
  font-size: 0.95rem;
  font-weight: 600;
  margin-bottom: 0.25rem;
  color: #333;
}

[role="textbox"] {
  display: block;
  width: 100%;
  min-height: 2rem;
  padding: 0.5rem 0.75rem;
  border: 1.5px solid #ccc;
  border-radius: 4px;
  font-family: system-ui, sans-serif;
  font-size: 1rem;
  color: #222;
  background-color: #fff;
  line-height: 1.4;
  transition: border-color 0.2s, box-shadow 0.2s;
}

[role="textbox"]:hover {
  border-color: #999;
}

[role="textbox"]:focus {
  outline: none;
  border-color: #0078d4;
  box-shadow: 0 0 0 3px rgba(0, 120, 212, 0.25);
}

[role="textbox"]:empty::before {
  content: attr(data-placeholder);
  color: #aaa;
  pointer-events: none;
}

[role="textbox"][aria-required="true"] {
  border-left: 3px solid #e81123;
  padding-left: calc(0.75rem - 3px);
}
```

:::

Dobbiamo usare l'attributo `contenteditable` così gli utenti possono digitare il loro input. Stiamo anche usando l'attributo `aria-required` impostato su `true` per indicare che questo controllo modulo personalizzato è obbligatorio.

Per far sembrare il controllo modulo come un controllo modulo normale, dovresti aggiungere CSS. Dovresti anche aggiungere JavaScript per impedire che il modulo venga inviato senza contenuto.

Se l'etichetta contiene già la parola `required`, allora dovresti omettere l'attributo `aria-required`. Questo assicura che i lettori di schermo annuncino la parola obbligatorio solo una volta.

Nella maggior parte dei casi probabilmente userai gli elementi nativi `label` e `form` con l'attributo `required`. Ma se devi creare un controllo modulo personalizzato, allora è importante aggiungere l'attributo `aria-required` quando necessario.

Inoltre, l'attributo `aria-required` può essere usato anche sugli input modulo nativi, come gli elementi `input`, `textarea` e `select`. Questo è spesso preferito all'attributo required nativo, poiché l'attributo required può avere potenziali problemi di usabilità e accessibilità, in particolare con la gestione degli errori predefinita del browser. Alla fine, dovrai testare per determinare quale attributo è migliore per la tua situazione.

L'ultimo stato ARIA di cui parleremo è `aria-checked`. Questo attributo serve a indicare se un elemento è nello stato selezionato. È usato più comunemente quando si creano caselle di spunta, pulsanti di opzione, interruttori e listbox personalizzati.

Ecco un esempio di come puoi usare `aria-checked` su un controllo casella di spunta personalizzato:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<div role="checkbox" aria-checked="true" tabindex="0">Checkbox</div>
```

```css
[role="checkbox"] {
  display: inline-flex;
  align-items: center;
  gap: 0.5rem;
  font-family: system-ui, sans-serif;
  font-size: 1rem;
  cursor: pointer;
  user-select: none;
  color: #222;
}

[role="checkbox"]::before {
  content: "";
  display: inline-block;
  width: 1rem;
  height: 1rem;
  border: 2px solid #666;
  border-radius: 4px;
  background-color: #fff;
  transition: all 0.2s ease;
  box-sizing: border-box;
}

[role="checkbox"]:hover::before {
  border-color: #0078d4;
}

[role="checkbox"]:focus::before {
  outline: 2px solid #0078d4;
  outline-offset: 2px;
}

[role="checkbox"][aria-checked="true"]::before {
  background-color: #0078d4;
  border-color: #0078d4;
  background-image: url("data:image/svg+xml;utf8,<svg xmlns='http://www.w3.org/2000/svg' width='14' height='14' viewBox='0 0 14 14'><path fill='white' d='M5.2 10.4L2 7.2l1.1-1.1 2.1 2.1L10.9 2.5 12 3.6z'/></svg>");
  background-repeat: no-repeat;
  background-position: center;
}

[role="checkbox"][aria-disabled="true"] {
  opacity: 0.5;
  pointer-events: none;
  cursor: not-allowed;
}

```

:::

Gli elementi casella di spunta nativi hanno uno stato `checked` integrato che viene comunicato alle tecnologie assistive. Ma se stai creando un controllo casella di spunta personalizzato, dovrai usare l'attributo `aria-checked` per indicarne lo stato.

Quando l'utente interagisce con il controllo casella di spunta personalizzato, dovrai usare lo stato `aria-checked` per riflettere il nuovo stato della casella di spunta. Quando la casella è selezionata, l'attributo `aria-checked` viene impostato su `true`. Quando non è selezionata, viene impostato su `false`.

Gli elementi nativi generalmente hanno un supporto migliore e funzionalità di accessibilità integrate.

Tuttavia, se devi creare controlli personalizzati, usare gli attributi ARIA è essenziale per comunicare efficacemente lo stato di questi controlli alle tecnologie assistive.

Come sempre, testa il tuo lavoro per assicurarti che gli attributi ARIA siano applicati correttamente e che il controllo personalizzato funzioni in modo accessibile e facile da usare.
