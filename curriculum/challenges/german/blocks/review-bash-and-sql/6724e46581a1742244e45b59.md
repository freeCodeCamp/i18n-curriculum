## Datenbanknormalisierung

Dies ist der Prozess, eine relationale Datenbank so zu organisieren, dass Datenredundanz reduziert und Integrität verbessert wird.

Die Vorteile sind:

- Minimierung doppelter Daten, was Speicher spart und Inkonsistenzen reduziert.
- Durchsetzung der Datenintegrität durch Verwendung von Primär- und Fremdschlüsseln.
- Datenbanken werden leichter wartbar und verständlich.

### Normalformen

- **1NF (erste Normalform)**
  - Jede Zelle enthält einen einzelnen (atomaren) Wert.
  - Jeder Datensatz ist eindeutig (durch Primärschlüssel erzwungen).
  - Reihenfolge von Zeilen/Spalten ist irrelevant.
  - Beispiel: Mehrere Telefonnummern aus einer `students`-Tabelle in eine separate `student_phones`-Tabelle verschieben.

- **2NF (zweite Normalform)**
  - Erfüllt Anforderungen der 1NF.
  - Keine **partiellen Abhängigkeiten**: Jedes Nicht-Schlüssel-Attribut muss vom gesamten zusammengesetzten Primärschlüssel abhängen.
  - Beispiel: `orders`-Tabelle in `order_header` und `order_items` aufteilen, um Attribute zu vermeiden, die nur von einem Teil des Schlüssels abhängen.

- **3NF (dritte Normalform)**
  - Erfüllt Anforderungen der 2NF.
  - Keine **transitiven Abhängigkeiten**: Nicht-Schlüssel-Attribute dürfen nicht von anderen Nicht-Schlüssel-Attributen abhängen.
  - Beispiel: `city_postal_code` in eine `cities`-Tabelle verschieben, anstatt es mit jeder Bestellung zu speichern.

- **BCNF (Boyce-Codd-Normalform)**
  - Erfüllt Anforderungen der 3NF.
  - Jeder Determinant (linke Seite einer funktionalen Abhängigkeit) muss ein Superschlüssel sein.

**Tipp**: Streben Sie in den meisten Designs die 3NF an, um ein gutes Gleichgewicht zwischen Integrität und Performance zu erreichen.

## Wichtige SQL-Konzepte

- SQL steht für Structured Query Language, eine Sprache zur Kommunikation mit relationalen Datenbanken.
- **Grundlegende Befehle** → `SELECT`, `INSERT`, `UPDATE`, `DELETE`, `CREATE TABLE`, `ALTER TABLE` usw.
- **Joins** → Kombiniert Daten aus mehreren Tabellen (`INNER JOIN`, `LEFT JOIN`, `RIGHT JOIN`, `FULL JOIN`).

## Ausführen von SQL-Befehlen in Bash

Sie können SQL-Befehle direkt über die Befehlszeile mit dem `psql`-Kommandozeilen-Client für PostgreSQL oder ähnlichen Tools für andere Datenbanken ausführen.

Zum Beispiel, um eine SQL-Datei in PostgreSQL auszuführen:

```bash
psql -U username -d database_name -c "SELECT * FROM students;"
```

Sie können auch MySQL-Befehle direkt ausführen:

```bash
mysql -u username -p database_name -e "SELECT * FROM students;"
```

### SQL aus einer Datei ausführen

```bash
# PostgreSQL
psql -U username -d database_name -f script.sql

# MySQL
mysql -u username -p database_name < script.sql
```

### SQL in ein Bash-Skript einbetten

```bash
#!/bin/bash
DB_USER="school_admin"
DB_NAME="school"

# Insert student data
psql -U "$DB_USER" -d "$DB_NAME" -c \
"INSERT INTO students (name, age, major) VALUES ('Alice', 20, 'CS');"
```

### Verwendung von Variablen in SQL

```bash
#!/bin/bash
DB_USER="school_admin"
DB_NAME="school"
STUDENT_NAME="Bob"
AGE=21

psql -U "$DB_USER" -d "$DB_NAME" -c \
"INSERT INTO students (name, age) VALUES ('$STUDENT_NAME', $AGE);"
```

**Tipp**: Variablen bereinigen, um SQL-Injection zu vermeiden.

## Abrufen und Verwenden von SQL-Abfrageergebnissen in Bash

Wenn Sie SQL-Abfragen über `psql` ausführen, können Sie die zurückgegebenen Werte in Ihren Bash-Skripten **erfassen** und **verarbeiten**.

### Einzelnen Wert erfassen

```bash
#!/bin/bash
DB_USER="school_admin"
DB_NAME="school"

# Get total student count
STUDENT_COUNT=$(psql -U "$DB_USER" -d "$DB_NAME" -t -A -c \
"SELECT COUNT(*) FROM students;")

echo "Total students: $STUDENT_COUNT"
```

Ausgabe → 42

### Mehrere Spalten abrufen

```bash
#!/bin/bash
DB_USER="school_admin"
DB_NAME="school"

# Get top 3 students' names and ages
RESULTS=$(psql -U "$DB_USER" -d "$DB_NAME" -t -A -F"," -c \
"SELECT name, age FROM students LIMIT 3;")

echo "Top 3 students:"
echo "$RESULTS"
```

Ausgabe

```bash
Alice,20
Bob,21
Charlie,22
```

### Durch Abfrageergebnisse iterieren

```bash
#!/bin/bash
DB_USER="school_admin"
DB_NAME="school"

# Get student names and majors
psql -U "$DB_USER" -d "$DB_NAME" -t -A -F"," -c \
"SELECT name, major FROM students;" | while IFS="," read -r name major
do
    echo "Student: $name | Major: $major"
done
```

Form der Ausgabe

```bash
Student: Alice | Major: CS
Student: Bob   | Major: Math
Student: Carol | Major: Physics
```

## SQL-Injection

Es handelt sich um eine Sicherheitslücke im Web, bei der Angreifer bösartigen SQL-Code in Eingabefelder einschleusen, um die Datenbank zu manipulieren.

Dies kann zu riskanten Aktionen führen wie:

- Umgehen der Authentifizierung.
- Diebstahl sensibler Daten.
- Ändern oder Löschen von Datensätzen.

Ein Beispiel für einen SQL-Injection-Angriff:

```sql
SELECT * FROM users WHERE username = '' OR '1'='1' --' AND password = 'anything';
```

Diese Abfrage würde alle Benutzer zurückgeben, weil die Bedingung `OR '1'='1'` immer wahr ist, wodurch Angreifer die Anmeldeprüfungen umgehen können.

### Vermeidung von SQL-Injection

1. **Prepared Statements verwenden**: Diese trennen SQL-Code von Daten und verhindern Injection. Beispiel (Node.js mit pg):

    ```js
    client.query('SELECT * FROM users WHERE username = $1 AND password = $2', [username, password]);
    ```

2. **Eingabevalidierung**: Säubern und validieren Sie alle Benutzereingaben, um sicherzustellen, dass sie erwarteten Formaten entsprechen.

3. **Minimalrechteprinzip**: Verwenden Sie Datenbankkonten mit den minimal notwendigen Berechtigungen für die Anwendung.

**Hinweis**: Gewähren Sie niemals Admin-Rechte an Anwendungskonten.

## N+1-Problem

Das N+1-Problem tritt auf, wenn eine Anwendung eine Abfrage macht, um eine Liste von Elementen (N) abzurufen, und dann für jedes Element eine zusätzliche Abfrage ausführt, um verwandte Daten abzurufen, was zu N+1 Abfragen führt.

**Warum das schlecht ist**

- Jede Abfrage verursacht Netzwerk- und Verarbeitungsaufwand.
- Viele kleine Abfragen sind langsamer als eine optimierte Abfrage.

### Beispiel für das N+1-Muster

```sql
-- 1: Get list of orders
SELECT * FROM orders LIMIT 50;

-- N: For each order, get customer
SELECT * FROM customers WHERE customer_id = ...;
```

**Lösung**: Verwenden Sie `JOINs` oder andere set-basierte Operationen.

```sql
SELECT 
  orders.order_id,
  orders.product,
  orders.quantity,
  customers.customer_id,
  customers.name,
  customers.email,
  customers.address
FROM orders
JOIN customers 
  ON orders.customer_id = customers.customer_id
WHERE orders.order_id IN (SELECT order_id FROM orders LIMIT 50);
```

Suchen Sie immer nach Möglichkeiten, verwandte Daten in einer einzigen Abfrage zusammenzufassen.
