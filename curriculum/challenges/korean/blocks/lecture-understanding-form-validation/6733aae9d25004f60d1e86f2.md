# --interactive--

이전 수업에서 HTML을 사용해 사용자가 폼에 제출할 수 있는 값을 제한하는 방법을 배웠습니다. 하지만 때로는 그것만으로는 부족할 때가 있습니다. 사용자에게 직접 오류 메시지를 보여주는 등 더 복잡한 작업을 하려면 JavaScript를 사용해야 합니다.

`textarea`와 `input` 같은 특정 HTML 요소들은 Constraint Validation API를 노출합니다. 이 API를 사용하면 사용자가 입력한 값이 최솟값이나 패턴 일치 같은 HTML 수준의 유효성 검사를 통과하는지 확인할 수 있습니다.

그렇다면 실제로 어떻게 사용할 수 있을까요? 예를 들어 회사 직원들이 다음과 같은 폼을 통해 피드백 메시지를 보내길 원한다고 가정해 봅시다:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css" />
<form>
  <label>Enter your email: </label>
  <input required type="email" />

  <label>Enter your feedback: </label>
  <textarea required placeholder="Your feedback here..."></textarea>

  <button type="submit">Submit Feedback</button>
</form>
```


우리는 기본적인 유효성 검사, 예를 들어 입력에 `input` 기호가 포함되어 있는지 확인하는 내장 유효성 검사를 가진 이메일 `@`를 사용하고 있습니다.

하지만 사용자가 `example@email.com` 같은 이메일 주소를 입력하면 어떻게 될까요? 기본 유효성 검사는 통과하지만, 회사 이메일 주소만 받도록 더 구체적으로 제한하고 싶습니다.

이럴 때 `pattern` 속성을 사용해 이메일 주소가 회사 이메일 주소로 끝나야 한다고 지정할 수 있습니다. 업데이트된 예시는 다음과 같습니다:

위 예제에서는 DOM에서 입력을 쿼리하고 input 이벤트 리스너를 추가했습니다.

`e.target`는 이벤트를 발생시킨 요소를 가리킵니다. 이 경우 우리의 `input`입니다. 그런데 `checkValidity()` 메서드는 무엇일까요?

이것은 Constraint Validation API의 일부입니다. `checkValidity()` 메서드는 요소가 모든 HTML 유효성 검사(속성 기준)를 통과하면 `true`를, 실패하면 `false`를 반환합니다.

잘못된 입력으로 시도하면 `false`가 콘솔에 기록되는 것을 볼 수 있습니다. 입력이 유효하지 않다는 것을 알았으니, 이제 유효하지 않음을 보고해 봅시다:

:::interactive_editor


`validity` 속성은 `ValidityState` 객체의 인스턴스입니다. 브라우저에서 객체가 어떻게 보일지 예시는 다음과 같습니다:

```js
ValidityState {
  badInput: false,
  customError: false,
  patternMismatch: true,
  rangeOverflow: false,
  rangeUnderflow: false,
  stepMismatch: false,
  tooLong: false,
  tooShort: false,
  typeMismatch: true,
  valueMissing: false,
  valid: true
}
```

여러 유용한 속성이 있으며, 모두 `true` 또는 `false` 불리언 값을 가집니다.

직접 더 탐구해 볼 만한 유용한 속성 중 하나는 필수 입력 필드가 비어 있을 때 `valueMissing`인 `true` 속성입니다. 또는 값이 지정된 정규 표현식 패턴과 일치하지 않을 때 `patternMismatch`인 `true`도 있습니다.

이 수업을 마친 후에는 수업에서 제공된 예제를 직접 실습하며 다양한 유효성 속성을 더 탐구해 보길 권장합니다.
