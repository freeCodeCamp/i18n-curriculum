# --description--

Хоча такі інструменти, як Lighthouse і Chrome DevTools, чудові, іноді потрібні кастомні дані, адаптовані саме під ваш сайт.

Ось тут і приходять на допомогу Performance Web API!

У цьому уроці ми розберемо, як використовувати три ключові Web API для вимірювання та аналізу швидкості вашого вебсайту.

Але спершу, що таке Performance Web API?

Performance Web API дозволяють розробникам відстежувати, наскільки ефективно завантажується та реагує вебсторінка, безпосередньо з коду. Ці API дають змогу вимірювати час завантаження сторінки, відстежувати затримки відтворення та взаємодії, а також аналізувати час виконання JavaScript.

За допомогою цих API ви можете створювати власні інструменти моніторингу продуктивності без залежності від стороннього програмного забезпечення!

Давайте розглянемо три потужні Web API, які ви можете використовувати вже сьогодні.

Перший — `performance.now()`.

Цей API надає високоточні часові позначки (у мілісекундах) для вимірювання, скільки часу займає завантаження різних частин вашого сайту.

Припустимо, ви хочете виміряти, як швидко виконується функція:

```js
const start = performance.now();  
// Run some code here  
const end = performance.now();  

console.log(`Execution time: ${end - start}ms`);
```

Це точніше, ніж використання `Date.now()`, бо вимірює час у мілісекундах з мікросекундною точністю, уникаючи проблем з дрейфом годинника. Ви можете використовувати його для відстеження часу виконання скриптів, затримок відповіді на події та продуктивності анімації.

Далі — Performance Timing API.

Цей API дає розбивку по кожному етапу завантаження сторінки — від DNS-запиту до `DOMContentLoaded`.

Хочете виміряти, скільки часу потрібно, щоб сторінка повністю завантажилася?

```js
let [navigationTiming] = performance.getEntriesByType("navigation");

if (navigationTiming instanceof PerformanceNavigationTiming) {
  // Calculate time from navigation start to DOM content loaded
  const pageLoadTime =
    navigationTiming.domContentLoadedEventEnd - navigationTiming.startTime;

  console.log("DOM Content Loaded Time:", pageLoadTime, "ms");
}
```

Ключові метрики, які можна відстежувати за допомогою цього API: час DNS-запиту — або іншими словами швидкість з’єднання, Time to First Byte (TTFB) — швидкість відповіді сервера, і `DOMContentLoaded` — або коли сторінка готова до взаємодії.

Якщо час завантаження вашої сторінки повільний, цей API точно покаже, де саме виникає затримка!

І нарешті, поговоримо про `PerformanceObserver`.

Цей API слухає події продуктивності, такі як зсуви макету, довгі завдання та взаємодії користувача.

Хочете відстежувати довгі JavaScript-завдання?

```js
const observer = new PerformanceObserver((list) => {  
  list.getEntries().forEach((entry) => {  
    console.log(`Long task detected: ${entry.duration}ms`);  
  });  
});  

observer.observe({ type: "longtask", buffered: true });
```

Що саме може відстежувати цей API? Він може відстежувати довгі завдання — тобто JavaScript, який блокує відтворення, зсуви макету для виявлення проблем з інтерфейсом, а також First Input Delay (FID) — наскільки швидко сторінка реагує на введення користувача.

Якщо вам потрібне відстеження продуктивності в реальному часі, цей API — справжня знахідка.

Отже, який API варто використовувати? Ось коротке порівняння:

| Performance API        | Найкраще підходить для                              |
| :--------------------- | :------------------------------------------------- |
| `performance.now()`    | Точне вимірювання часу функцій і скриптів          |
| Performance Timing API | Вимірювання повної продуктивності завантаження сторінки |
| Performance Observer   | Моніторинг у реальному часі взаємодій і відтворення |

Поєднуючи ці API, ви можете створити власні інструменти вимірювання продуктивності і відстежувати саме те, що найважливіше для вашого сайту.
