# --description--

Prima di poter davvero iniziare a usare il controllo di versione e siti come GitHub, devi avere Git installato e pronto all’uso.

Per verificare se Git è già installato sulla tua macchina, puoi eseguire il seguente comando nel terminale:

```sh
git --version
```

Se vedi un numero di versione, significa che Git è installato. Altrimenti, dovrai installarlo.

Per i sistemi Linux, Git spesso è preinstallato nella maggior parte delle distribuzioni. Se non hai Git preinstallato, dovresti riuscire a installarlo con i comandi del tuo gestore di pacchetti come `sudo apt-get install git` o `sudo pacman -S git`.

Per gli utenti Mac, puoi installare Git tramite Homebrew con `brew install git`, oppure puoi scaricare l’installer eseguibile dal sito di Git.

Per Windows, puoi scaricare l’installer eseguibile dal sito di Git. Oppure, se hai configurato Chocolatey, puoi eseguire `choco install git.install` in PowerShell. Nota che su Windows potresti anche voler scaricare GitBash per avere un ambiente shell simile a Unix.

Per assicurarti che l’installazione abbia funzionato, esegui di nuovo il comando `git --version` nel terminale.

Una volta installato Git, dovrai fare alcune modifiche alla configurazione. Dovrai farle solo una volta sul tuo computer e non sarà necessario ripeterle ogni volta che aggiorni Git.

`git config` serve per impostare variabili di configurazione che determinano come Git funziona sulla tua macchina. Per vedere le variabili di configurazione attuali e dove sono archiviate nel sistema, puoi eseguire il seguente comando:

```sh
git config --list --show-origin
```

In questo momento dovresti vedere solo le impostazioni di configurazione a livello di sistema se hai appena installato Git per la prima volta. Se invece hai già installato Git prima di questa lezione, potresti vedere configurazioni utente per nome e email.

Per impostare il tuo nome utente, puoi eseguire il seguente comando:

```sh
git config --global user.name "Jane Doe"
```

La flag `--global` viene usata qui per impostare il nome utente per tutti i progetti sul tuo sistema che usano Git. Se hai bisogno di sovrascrivere il nome utente per un progetto specifico, puoi eseguire il comando nella directory di quel progetto senza la flag `--global`.

Per impostare l’indirizzo email dell’utente, puoi eseguire il seguente comando:

```sh
git config --global user.email janedoe@example.com
```

Configurare il nome utente e l’email è importante perché Git userà queste informazioni per ogni commit che fai in un progetto. Imparerai di più sui commit in una lezione futura.

Un’altra configurazione che puoi impostare è l’editor preferito che vuoi che Git usi. Ecco un esempio di come impostare l’editor preferito su Emacs:

```sh
git config --global core.editor emacs
```

Altre opzioni includono Vim, Nano, VS Code e altro. Per esempio, se usi VS Code come editor, puoi configurare Git così:

```sh
git config --global core.editor "code --wait"
```

Se scegli di non impostare un editor preferito, Git userà l’editor predefinito del sistema.

Se usi una macchina Windows e vuoi cambiare editor, dovrai fornire il percorso completo al file eseguibile così:

```sh
git config --global core.editor "'C:/Program Files/Notepad++/notepad++.exe' -multiInst -notabbar -nosession -noPlugin"
```

Per vedere la lista delle tue configurazioni, puoi eseguire il seguente comando:

```sh
git config --list
```

Ora che hai Git installato e configurato, sei pronto per iniziare ad aggiungere il controllo di versione ai tuoi progetti.
