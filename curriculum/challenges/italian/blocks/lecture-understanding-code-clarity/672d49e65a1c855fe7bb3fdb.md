# --description--

I commenti nella programmazione servono a fornire un contesto aggiuntivo per il codice o a lasciare note per te stesso e per gli altri.

I commenti sono righe o blocchi di testo ignorati dal motore JavaScript quando il tuo codice viene eseguito. Esistono solo per il beneficio di chi legge il codice, che sia tu o qualcun altro.

JavaScript offre due modi per aggiungere commenti al tuo codice: commenti su singola riga e commenti su più righe.

I commenti su singola riga si creano usando due barre oblique (`//`). Ecco un esempio:

```js
// I am a single line comment in JavaScript
```

Questo tipo di commento è ottimo per spiegazioni o chiarimenti brevi.

Ecco un esempio reale tratto dai file del progetto del curriculum freeCodeCamp:

```js
// This is to allow English to build without having to download the i18n files.
// It fails when trying to resolve the i18n-curriculum path if they don't exist.
const curriculumLocale = process.env.CURRICULUM_LOCALE ?? 'english';
const I18N_CURRICULUM_DIR = path.resolve(
  __dirname,
  curriculumLocale === 'english' ? '.' : 'i18n-curriculum/curriculum'
);
```

Non preoccuparti di cercare di capire cosa fa effettivamente il codice perché è più avanzato di quello che hai imparato finora. Concentrati invece sul commento lasciato dallo sviluppatore. Questo commento fornisce un contesto importante sul motivo per cui questo codice esiste.

Commenti come questo sono importanti per chi lavora in team per due motivi:

1. Altri sviluppatori che lavorano al progetto capiranno lo scopo di questo codice.

1. Aiuta a evitare modifiche o cancellazioni non necessarie senza consultare il team, cosa che potrebbe causare bug o problemi.

Un altro tipo di commento è quello su più righe. Ecco la sintassi base:

```js
/*
 I am a multiline comment.
 This is helpful for longer explanations.
*/
```

I commenti su più righe sono utili quando devi scrivere descrizioni, spiegazioni o note più lunghe nel tuo codice.

Diamo un’altra occhiata ai file del progetto del curriculum freeCodeCamp per vedere come i commenti su più righe possono essere usati nel mondo reale.

```js
/* Since there can be more than one way to complete a certification (using the
legacy curriculum or the new one, for instance), we need a certification
field to track which certification this belongs to. */
const dupeCertifications = [
  {
    certification: 'responsive-web-design',
    dupe: '2022/responsive-web-design'
  }
];
const hasDupe = dupeCertifications.find(
  cert => cert.dupe === meta.superBlock
);
```

Come prima, ignora tutto il codice JavaScript perché usa concetti che non sono ancora stati insegnati. Concentrati invece sul commento lasciato dallo sviluppatore.

Uno sviluppatore del team, o anche un nuovo contributore che lavora al progetto, può capire perché questo pezzo di codice è qui e avere il contesto completo prima di lavorare su quest’area del progetto.

Anche se i commenti sono utili nella programmazione, è importante evitare di commentare troppo. Non serve commentare ogni singola riga di codice, specialmente se il codice è semplice e si spiega da solo.

Ecco un esempio di commenti usati per spiegare l’ovvio:

```js
// This code uses the const keyword to create a new variable called price.
// We are assigning the number 10 to the price variable.
const price = 10;
```

In questa situazione non serve aggiungere commenti perché il codice si spiega da solo. L’obiettivo è migliorare la leggibilità, non appesantire il codice con spiegazioni inutili.

Se vuoi aggiungere commenti ai tuoi progetti personali mentre impari a programmare, va bene. Ma quando inizi a lavorare su progetti reali con altri sviluppatori, è importante non usare commenti per codice che si spiega da solo.

È anche importante non usare i commenti per giustificare codice confuso, troppo complicato o scritto male. In questi casi è meglio rifattorizzare, cioè modificare, il codice così che altri sviluppatori capiscano meglio cosa sta succedendo.

I commenti sono strumenti potenti per documentare il tuo codice e renderlo più facile da capire. Dovresti usarli per fornire contesto o lasciare note per te stesso e per gli altri.
