# --interactive--

未指定の注釈要素、または `u` 要素は、非テキストの注釈が適用されたインラインテキストを表すために使われます。

以下は、さまざまなスペルミスを強調表示するために `u` 要素を使った例です：

:::interactive_editor

```html
<p>
  You can use the unarticulated annotation element to highlight
  <u>inccccort</u> <u>spling</u> <u>issses</u>.
</p>
```

:::

この例では、単語 `incorrect`、`spelling`、および `issues` が誤って綴られています。`u` 要素のデフォルトのスタイルは、テキストの下に黒い下線が引かれます。

HTML4 では、`u` 要素はスタイリング目的で使われていました。しかし HTML5 では、`u` 要素はテキストに非テキストの注釈が適用されていることを示すためだけに使うべきです。

テキストに下線を引きたい場合は、HTML ではなく CSS を使うことをお勧めします。

取り消し線要素、または `s` 要素は、テキストがもはや正確でないか関連性がないことを表すために使われます。以下は、活動の中止を示すために `s` 要素を使った例です：

:::interactive_editor

```html
<p><s>Tomorrow's hike will be meeting at noon.</s></p>

<p>Due to unforeseen weather conditions, the hike has been canceled.</p>
```

:::

この例では、天候の理由でハイキングが中止されたため、最初の文が取り消し線で消されています。

`s` 要素は、文書の変更を示すだけの目的で使うべきではありません。その場合は、削除されたテキスト要素や挿入されたテキスト要素のほうが適切です。

`ruby` 要素は、メインテキストの上または下に表示される小さなテキストを表します。通常、東アジアの文字の発音を示すために使われます。以下は MDN ウェブドックスのページからの `ruby` 要素の例です：

:::interactive_editor

```html
<ruby> 明日 <rp>(</rp><rt>Ashita</rt><rp>)</rp> </ruby>
```

:::

`rp` 要素、またはルビフォールバック括弧要素は、ルビ注釈の表示をサポートしていないブラウザのフォールバックとして使われます。

`rt` 要素、またはルビテキスト要素は、ルビ注釈のテキストを示すために使われます。このテキストは通常、東アジアのタイポグラフィで発音や翻訳の詳細を示すために使われます。

`ruby` 要素は他の種類の注釈にも使えますが、最も一般的な用途は東アジアのタイポグラフィです。
