# --description--

정적 파일 제공은 웹 애플리케이션에서 흔히 필요한 기능입니다. 정적 파일에는 이미지, CSS 파일, 클라이언트 측 JavaScript, 폰트 및 동적으로 변경되지 않는 기타 리소스가 포함됩니다.

Express는 내장 `express.static()` 미들웨어로 이러한 파일 제공을 간단하게 만듭니다.

Express에서 정적 파일 제공이란 서버의 파일 시스템에서 클라이언트 브라우저로 추가 서버 측 처리 없이 파일을 직접 전달하는 것을 의미합니다.

예를 들어, 브라우저가 `/styles.css`, `/script.js`, 또는 `/images/logo.png` 같은 파일을 요청하면 Express는 지정된 폴더에서 해당 파일을 찾아 직접 제공합니다. 파일을 찾지 못하면 Express는 자동으로 404 응답을 보냅니다. 이 파일들은 서버에서 생성되거나 수정될 필요가 없기 때문에 *정적*이라고 부릅니다.

이렇게 하면 스타일시트나 이미지 같은 자산 제공이 쉬워지고, 각 파일마다 라우트를 수동으로 정의할 필요가 없어집니다.

Express에서 정적 파일 제공을 설정하는 방법은 다음과 같습니다:

```javascript
const express = require("express");
const path = require("path");
const app = express();
const port = 3000;

// Serve static files from the "public" directory
app.use(express.static(path.join(__dirname, "public")));

app.get("/", (req, res) => {
  res.send("Welcome to the static file demo!");
});

app.listen(port, () => {
  console.log(`Server running at http://localhost:${port}`);
});
```

먼저, 서버를 생성하기 위해 `express`를 가져오고, 다양한 운영체제에서 안전하게 파일 경로를 구성하기 위해 `path`를 가져옵니다.

다음으로, `app.use(express.static(path.join(__dirname, "public")))`는 Express에게 `public` 폴더에서 파일을 제공하라고 지시합니다. `express.static()` 미들웨어는 Express에게 `public` 폴더에서 파일을 제공하라고 알려줍니다. 이 디렉터리 내의 모든 파일은 브라우저에서 접근할 수 있게 됩니다.

또한, `GET`에 대한 간단한 `/` 라우트를 정의하여 환영 메시지를 보냅니다. 이는 정적 파일 제공이 동적 라우트와 공존할 수 있음을 보여줍니다.

마지막으로, `app.listen()`는 포트 3000에서 서버를 시작합니다. 서버가 실행되면 `public` 폴더 내의 정적 자산 요청이 자동으로 처리됩니다.

이 기능을 사용하려면 정적 파일을 저장할 `public`라는 폴더가 필요합니다. 일반적인 프로젝트 구조는 다음과 같습니다:

```md
my-app/
│
├── app.js
└── public/
    ├── index.html
    ├── styles.css
    └── image.png
```

사용자가 예를 들어 `http://localhost:3000/index.html`를 방문해 요청하면, Express는 `public` 디렉터리 내에 요청한 파일이 있는지 확인합니다. 파일이 있으면 Express는 이를 브라우저로 직접 보냅니다. 파일이 없으면 Express는 다음 미들웨어로 넘어가거나 404 응답을 반환합니다.

각 파일마다 라우트를 정의할 필요가 없으므로, `http://localhost:3000/styles.css`에 대한 요청은 Express가 `public/styles.css`를 찾아 자동으로 응답한다는 뜻입니다.

정적 파일 제공 방식을 사용자 정의할 수도 있습니다. 예를 들어, 여러 디렉터리에서 파일을 제공하거나 캐싱 동작을 설정할 수 있습니다.

앱에 다음과 같은 파일 구조가 있다고 가정해 보세요:

```md
my-app/
│
├── app.js
│
├── public/
│   ├── index.html
│   └── styles.css
│
└── images/
    └── image.png
```

여러 디렉터리에서 정적 파일을 제공하도록 등록하는 방법은 다음과 같습니다:

```javascript
app.use(express.static(path.join(__dirname, "public")));
app.use("/images", express.static(path.join(__dirname, "images")));
```

여기서는 이전과 마찬가지로 `app.use(express.static(path.join(__dirname, "public")));`를 사용하여 `public` 디렉터리 내 파일을 루트 경로에서 제공합니다.

하지만 다음 줄을 보면 `"/images"`를 `app.use()`의 첫 번째 인수로 전달하고 `express.static(path.join(__dirname, "images"))`를 뒤따르게 합니다. 이 경우 `"/images"`는 마운트 경로로, Express가 `/images`에 대한 요청을 받을 때만 이 미들웨어를 사용하도록 지시합니다. 요청이 들어오면 `express.static()`가 실행되어 `images` 디렉터리 내 파일을 찾습니다.

따라서 이 구성에서 다음 요청은 이렇게 작동합니다:

*   GET `http://localhost:3000/styles.css`: Express는 `public` 디렉터리에서 `styles.css`를 찾아 클라이언트에 보냅니다.
    
*   GET `http://localhost:3000/images/image.png`: 이 요청은 `/images`로 시작하므로 Express는 `images` 디렉터리에서 `image.png`를 찾아 클라이언트에 보냅니다.
    

Express로 정적 파일을 제공하는 것은 간단하면서도 강력합니다. `express.static()`를 사용하면 이미지, 스타일시트, 클라이언트 측 JavaScript 같은 자산을 쉽게 제공할 수 있습니다.

정적 파일을 `public` 같은 전용 폴더에 보관하거나 여러 파일 디렉터리와 마운트 경로를 사용하면 애플리케이션을 체계적이고 유지보수하기 쉬우며 확장하기 편하게 관리할 수 있습니다.
