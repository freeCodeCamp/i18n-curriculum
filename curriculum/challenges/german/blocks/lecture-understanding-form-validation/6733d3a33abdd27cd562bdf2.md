# --interactive--

Lernen wir den Zweck der `preventDefault()`-Methode bei Ereignissen kennen.

Jedes Ereignis, das im DOM ausgelöst wird, hat eine Art Standardverhalten. Das `click`-Ereignis bei einem Kontrollkästchen schaltet standardmäßig den Zustand dieses Kontrollkästchens um. Das Drücken der Leertaste bei einem fokussierten Button aktiviert den Button. Die `preventDefault()`-Methode bei diesen Ereignisobjekten verhindert, dass dieses Verhalten eintritt.

Schauen wir uns ein Beispiel an. Definieren wir ein `input`-Element, in das ein Benutzer tippen kann:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<label>Enter some characters:
  <input type="text">
</label>
```

```css

  flex-direction: column;

  font-size: 14px;
  color: #333;
  margin-bottom: 12px;
}

  padding: 8px 12px;
  border: 1px solid #ccc;
  border-radius: 4px;
  font-size: 14px;
  margin-top: 4px;
  width: 100%;
  box-sizing: border-box;

input[type="text"]:focus {
  border-color: #0078d4;
  box-shadow: 0 0 3px rgba(0, 120, 212, 0.5);
  outline: none;

```

:::

Wenn wir uns das Ergebnis ansehen, können wir wie erwartet in das Eingabefeld tippen. Aber vielleicht wollen wir das nicht. Vielleicht möchten wir stattdessen das Zeichen, das der Benutzer tippt, in einem separaten Element anzeigen. Definieren wir zuerst unser Element dafür:

:::interactive_editor

Dann müssen wir uns in das `keydown`-Ereignis einklinken, um auf das Tippen eines Zeichens auf der Tastatur zu hören. Beachten Sie, dass wir hier nicht die `change`- oder `input`-Ereignisse wollen, weil wir die Tastaturinformationen benötigen.

```js
const input = document.querySelector("input");

input.addEventListener("keydown", (e) => {
    
})
```


`e.key` gibt Ihnen den Wert der gedrückten Taste, zum Beispiel `a` für die `a`-Taste oder `Enter` für die `Enter`-Taste.

Mit dem obigen Code wird beim Tippen in das `input` das Zeichen, das Sie tippen, im `p`-Element angezeigt.

Das ist großartig, aber wir wollen nicht, dass die Zeichen auch im `input` angezeigt werden. Hier kommt unsere `preventDefault()`-Methode ins Spiel. Das Standardverhalten eines `keydown` ist es, das Zeichen im Eingabefeld anzuzeigen. Verhindern wir das, indem wir `e.preventDefault()` aufrufen:

```js
const input = document.querySelector("input");
const output = document.getElementById("output");

input.addEventListener("keydown", (e) => {
  e.preventDefault();
  output.innerText = `You pressed the ${e.key} key`;
});
```

:::

Und so haben Sie das Standardverhalten verhindert, um Ihre eigene benutzerdefinierte Ereignisbehandlung zu implementieren.

Ein weiteres häufiges Beispiel für die Verwendung der `e.preventDefault()`-Methode betrifft Formularübermittlungen. Standardmäßig sendet das Absenden eines Formulars Daten an den Server und lädt die Seite neu. Die Verwendung von `e.preventDefault()` verhindert, dass dies geschieht.

```js
const form = document.querySelector("form");

form.addEventListener("submit", (e) => {
  e.preventDefault();
  // rest of code goes here
});
```

Das Verhindern des Standardverhaltens ist großartig, wenn Sie mehr Kontrolle darüber benötigen, wie ein Benutzer mit der Seite interagiert. Dabei ist es jedoch wichtig, Aspekte wie Barrierefreiheit zu berücksichtigen: Ihr benutzerdefiniertes Verhalten sollte dieselben Funktionen wie das Standardverhalten bieten.
