# --interactive--

Рекурсія — це складна функція, яка дозволяє викликати функцію повторно, доки не буде досягнуто базового випадку. На відміну від традиційного циклу, рекурсія дає змогу обробляти структури з невідомою глибиною, наприклад, глибоко вкладені об’єкти/масиви або дерево файлів. Але її також можна використати для більш базових завдань, наприклад, відліку від заданого числа.

Давайте створимо функцію, яка робить саме це. Назвемо нашу функцію `recursiveCountdown`, і вона має приймати число. Ми виведемо це число в консоль:

:::interactive_editor

```js
const recursiveCountdown = (number) => {
  console.log(number);
};

recursiveCountdown(5);
```

:::

Якщо викликати цю функцію з числом 5, ми побачимо це число у консолі. Але нічого більше не відбувається — і число 5 точно не є відліком.

Перш ніж почати створювати рекурсивну частину функції, нам потрібно спочатку визначити базовий випадок. Якщо базовий випадок не встановлено, ваш код працюватиме доти, доки не перевищить виділену пам’ять і не аварійно завершиться.

:::interactive_editor

```js
const recursiveCountdown = (number) => {
  if (number < 1) {
    return;
  }
  console.log(number);
};

recursiveCountdown(5);
```

:::

Для базового випадку ми хочемо, щоб відлік зупинився, якщо число менше за 1. Коли ми досягнемо цього базового випадку, ми можемо повернутися, щоб припинити виконання функції.

Тепер, коли базовий випадок безпечно підготовлено, можна налаштувати рекурсію. Ключовий момент, що робить функцію рекурсивною, — це те, що вона викликає сама себе під час виконання. У нашому випадку ми хочемо викликати функцію після того, як вивели число. Але щоб відлік йшов вниз, наше нове число має бути на одиницю меншим:

:::interactive_editor

```js
const recursiveCountdown = (number) => {
  if (number < 1) {
    return;
  }
  console.log(number);
  recursiveCountdown(number - 1);
};

recursiveCountdown(5); 
```

:::

Це виведе числа 5, 4, 3, 2 і 1 у консоль.

Ми отримали наші п’ять чисел! Але що, якби ми хотіли відлічувати вгору? Замість того, щоб писати зовсім нову функцію, ми можемо поміняти місцями порядок виведення і рекурсивного виклику:

:::interactive_editor

```js
const recursiveCountdown = (number) => {
    if (number < 1) {
        return;
    }
    recursiveCountdown(number - 1);
    console.log(number);
  };

recursiveCountdown(5);
```

:::

Це виведе числа 1, 2, 3, 4 і 5 у консоль.

Але чому це працює? Щоб це зрозуміти, потрібно розібратися зі стеком викликів. Стек викликів — це те, як JavaScript відстежує і вирішує виклики функцій. Стек працює як черга з принципом останній увійшов — перший вийшов. Щоб краще це зрозуміти, додамо трохи логування у нашу функцію:

:::interactive_editor

```js
const recursiveCountdown = (number) => {
  console.log(`Function execution started for number: ${number}`);
  if (number < 1) {
    console.log(`Base case reached, begin resolving stack`);
    return;
  }
  console.log(`Calling recursiveCountdown with number: ${number - 1}`);
  recursiveCountdown(number - 1);
  console.log(`Function execution completed for number: ${number}`);
};

recursiveCountdown(5);
```

:::

Ми додали тут чотири ключові інструкції. Перший лог виконується, коли починається виконання виклику функції. Третій лог виконується безпосередньо перед рекурсивним викликом функції. А четвертий лог виконується, коли виконання функції завершено. Результат:

```md
Function execution started for number: 5
Calling recursiveCountdown with number: 4
Function execution started for number: 4
Calling recursiveCountdown with number: 3
Function execution started for number: 3
Calling recursiveCountdown with number: 2
Function execution started for number: 2
Calling recursiveCountdown with number: 1
Function execution started for number: 1
Calling recursiveCountdown with number: 0
Function execution started for number: 0
Base case reached, begin resolving stack
Function execution completed for number: 1
Function execution completed for number: 2
Function execution completed for number: 3
Function execution completed for number: 4
Function execution completed for number: 5
```

Але як це відбувається? Ось тут і вступає в гру стек викликів. Коли ми викликаємо `recursiveCountdown(5)`, цей виклик функції додається до стека викликів.

Коли цей виклик функції доходить до моменту, коли потрібно викликати `recursiveCountdown(4)`, він має зупинитися і чекати на результат. Тим часом наш `recursiveCountdown(4)` додається до стека викликів поверх `recursiveCountdown(5)`.

Коли цей виклик функції доходить до моменту, коли потрібно викликати `recursiveCountdown(3)`, він має зупинитися і чекати на результат. Тим часом наш `recursiveCountdown(3)` додається до стека викликів поверх `recursiveCountdown(4)`.

Коли цей виклик функції доходить до моменту, коли потрібно викликати `recursiveCountdown(2)`, він має зупинитися і чекати на результат. Тим часом наш `recursiveCountdown(2)` додається до стека викликів поверх `recursiveCountdown(3)`.

Коли цей виклик функції доходить до моменту, коли потрібно викликати `recursiveCountdown(1)`, він має зупинитися і чекати на результат. Тим часом наш `recursiveCountdown(1)` додається до стека викликів поверх `recursiveCountdown(2)`.

І нарешті, коли цей виклик функції доходить до моменту, коли потрібно викликати `recursiveCountdown(0)`, він має зупинитися і чекати на результат. Тим часом наш `recursiveCountdown(0)` додається до стека викликів поверх `recursiveCountdown(1)`.

Але `recursiveCountdown(0)` не викликає іншу функцію — він досягає нашого базового випадку, де повертається раніше. Оскільки виконання цієї функції завершено, виклик функції вважається «вирішеним». Коли виклик вирішено, його видаляють зі стека.

Тепер наш `recursiveCountdown(1)` більше не чекає на цей виклик — він на вершині стека і може продовжити виконання.

`recursiveCountdown(1)` вирішується, видаляється зі стека і дозволяє `recursiveCountdown(2)` продовжити виконання.

`recursiveCountdown(2)` вирішується, видаляється зі стека і дозволяє `recursiveCountdown(3)` продовжити виконання.

`recursiveCountdown(3)` вирішується, видаляється зі стека і дозволяє `recursiveCountdown(4)` продовжити виконання.

`recursiveCountdown(4)` вирішується, видаляється зі стека і дозволяє `recursiveCountdown(5)` продовжити виконання.

І `recursiveCountdown(5)` вирішується і видаляється зі стека. Наш стек викликів тепер порожній, отже рекурсія завершена!

Це базовий огляд того, як працює рекурсія в JavaScript. Це складна концепція, і вам варто поекспериментувати з кодом і логами, доки ви не відчуєте себе впевнено з поведінкою стека викликів.

Для цікавинок: ми говорили про те, що відсутність базового випадку може спричинити аварійне завершення коду через нестачу пам’яті. Це тому, що рекурсія продовжує додавати все більше викликів функцій у стек викликів, доки стек не переповниться. Саме так називається популярна спільнота програмістів.
