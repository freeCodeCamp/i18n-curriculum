# --description--

In dieser Lektion lernen Sie, wie Sie mit Funktionstypen arbeiten. Beginnen wir mit Typannotationen für Parameter.

Hier ist ein Beispiel für eine einfache Funktion:

```ts
function circleArea(radius) {
  return Math.PI * radius * radius;
}
```

Im Moment würde TypeScript eine Meldung von `Parameter 'radius' implicitly has an 'any' type` anzeigen. Der Typ `any` zeigt an, dass ein Wert jeden Typ haben kann. Mehr über diesen Typ lernen Sie in einer späteren Lektion.

Um den Fehler zu beseitigen und den Parameter explizit zu typisieren, können Sie ihn so typisieren:

```ts
function circleArea(radius: number) {
  return Math.PI * radius * radius;
}
```

Wenn Sie versuchen, die `circleArea`-Funktion mit einem anderen Typ als einer Zahl aufzurufen, zeigt TypeScript einen Fehler wie diesen an:

```ts
function circleArea(radius: number) {
  return Math.PI * radius * radius;
}

// Argument of type 'string' is not assignable to parameter of type 'number'
circleArea("3");
```

Wenn Sie mehr Argumente angeben, als die Funktion erwartet, zeigt TypeScript einen Fehler an:

```ts
function circleArea(radius: number) {
  return Math.PI * radius * radius;
}

// Expected 1 arguments, but got 2.
circleArea(3, 4);
```

Wenn Sie zu wenige Argumente angeben, zeigt TypeScript ebenfalls einen Fehler an:

```ts
function circleArea(radius: number) {
  return Math.PI * radius * radius;
}

// Expected 1 arguments, but got 0.
// An argument for 'radius' was not provided.
circleArea();
```

Sie können Parameter optional machen, indem Sie das `?`-Symbol neben dem Parameter verwenden, so:

```ts
function area(radius: number, height?: number): number {
  const baseArea = Math.PI * radius * radius;

  if (height !== undefined) {
    const sideArea = 2 * Math.PI * radius * height;
    return baseArea + sideArea;
  }

  return baseArea;
}
```

Beachten Sie, dass optionale Parameter immer nach den erforderlichen Parametern kommen müssen. Wenn Sie einen erforderlichen Parameter nach einem optionalen platzieren, führt das zu einem TypeScript-Fehler.

Die bedingte Prüfung innerhalb der Funktion ist vorhanden, falls das `height`-Argument nicht übergeben wird.

Wenn Sie Ihre Rückgabewerte typisieren möchten, können Sie Rückgabetypannotationen so hinzufügen:

```ts
function circleArea(radius: number): number {
  return Math.PI * radius * radius;
}
```

Die Rückgabetypannotation steht nach der Parameterliste. Ähnlich wie bei Variablentypannotationen ist TypeScript meistens intelligent genug, den Rückgabetyp zu ermitteln. Verwenden Sie also eine explizite Rückgabetypannotation, wenn Sie die Absicht der Funktion klarer machen oder unbeabsichtigte Änderungen am Rückgabewert verhindern möchten.

Das nächste Beispiel, das wir uns ansehen, behandelt asynchrone Funktionen und Promises. Wenn Ihre Funktion ein Promise zurückgeben soll, können Sie den Typ `Promise` so verwenden:

```ts
async function getCityTemperature(city: string): Promise<number> {
  const response = await fetch(`https://api.example.com/weather?city=${city}`);
  const data = await response.json(); 
  return data.temperature;
}
```

In diesem Fall könnte TypeScript, wenn Sie den Rückgabetyp entfernen, den Rückgabetyp als `Promise<any>` ableiten. Daher wäre dies eine gute Situation, um den Rückgabewert explizit zu typisieren.

Bis zu diesem Punkt haben wir nur Beispiele für benannte Funktionen betrachtet. Aber wie funktionieren Typen bei anonymen Funktionen?

Hier ist ein Beispiel, das eine anonyme Funktion im Callback verwendet, das an ein `forEach` übergeben wird:

```ts
const languages = ["JavaScript", "Python", "TypeScript"];

languages.forEach((lang) => {
  console.log(`${lang.toUpperCase()} has ${lang.length} characters`);
});
```

Der `lang`-Parameter hat keine Typannotation. Stattdessen verwendet TypeScript kontextbezogenes Typisieren, was bedeutet, dass der Typ aus dem Kontext abgeleitet wird, in dem er erscheint.
