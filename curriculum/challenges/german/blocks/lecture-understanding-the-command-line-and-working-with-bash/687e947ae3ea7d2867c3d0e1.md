# --description--

Lernen wir etwas über Befehlsoptionen und Flags.

In einer vorherigen Lektion haben Sie gelernt, wie man Argumente an Befehle wie `touch readme.md` übergibt. Optionen oder Flags sind spezielle Argumente, die Sie an einen Befehl übergeben können und die das Verhalten des Befehls beeinflussen. Die beiden Begriffe werden austauschbar verwendet, obwohl „Flags“ eher für Optionen verwendet wird, die als Ein-/Ausschalter dienen.

Optionen sind typischerweise mit einem oder zwei Bindestrichen (`-`) versehen, was eine visuelle Unterscheidung zwischen einer Option und einem Argument ermöglicht. Schauen wir uns zuerst die Syntax mit zwei Bindestrichen, die sogenannte „Langform“, an.

Viele Anwendungen, wie `ls`, akzeptieren ein `--version`-Flag, das die aktuelle Version der Anwendung im Terminal ausgibt, anstatt die Befehle der Anwendung auszuführen.

Viele Anwendungen bieten auch ein `--help`-Flag an, das Anweisungen zur Verwendung der Anwendung ausgibt.

Die Syntax mit einem Bindestrich, die sogenannte „Kurzform“, verwendet typischerweise Optionen, die aus einem einzelnen Buchstaben bestehen. Zum Beispiel listet das `-a`-Flag mit `ls` alle Dateien auf, einschließlich versteckter Dateien, die mit einem Punkt (`.`) beginnen, wie `.env`.

Der Vorteil dieser Kurzoptionen ist, dass Sie mehrere Flags zusammenketten können. Anstatt `ls --all --human-readable --size` zu verwenden, können Sie die einzelnen Buchstaben alle auf einmal mit `ls -ahs` verwenden.

Manche Optionen erwarten, dass ihnen ein Wert übergeben wird.

Bei der Verwendung von Langform-Optionen müssen Sie typischerweise ein Gleichheitszeichen verwenden. In dieser Syntax wird der Wert direkt mit dem Gleichheitszeichen (`=`) an die Option angehängt. Hier ist ein Beispiel, wie das Verhalten von `ls` so geändert wird, dass Farben ein- oder ausgeschlossen werden:

```bash
ls --color=never
```

Bei der Verwendung von Kurzform-Optionen trennen Sie den Wert typischerweise mit einem Leerzeichen. Zum Beispiel hier die Lang- und Kurzform-Optionen, um die `width` des `ls`-Ergebnisses festzulegen:

```bash
ls -w 50
```

```bash
ls --width=50
```

Beachten Sie, dass die Kurzform `-w 50` verwendet, während die Langform `--width=50` nutzt. Diese Unterscheidung ist wichtig, um zu vermeiden, dass ein Optionswert stattdessen als Positionsargument übergeben wird.

Wenn Sie sich jedoch unsicher sind, denken Sie daran, dass Sie normalerweise das `--help`-Flag verwenden können, um die erwartete Syntax für Optionen zu sehen!

## --text--

Was ist der Unterschied zwischen der „Langform“- und der „Kurzform“-Syntax für Befehlsoptionen?

## --answers--

Die Langform verwendet zwei Bindestriche, während die Kurzform einen einzelnen Bindestrich verwendet.
