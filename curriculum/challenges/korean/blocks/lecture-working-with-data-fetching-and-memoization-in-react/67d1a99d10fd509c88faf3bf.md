# --description--

React 앱은 종종 동적 콘텐츠를 위해 외부 API와 데이터베이스에 의존합니다. 이러한 API와 데이터베이스에서 데이터를 접근하려면 몇 가지 데이터 가져오기 기법을 사용해야 합니다.

React에서 데이터 가져기가 어떻게 작동하는지, 그리고 데이터를 가져오기 위해 사용할 수 있는 다양한 옵션을 살펴보겠습니다.

React는 데이터를 어떻게 가져올지에 대해 특정한 방식을 강요하지 않습니다. 기본적으로 모든 최신 브라우저가 지원하는 내장 Fetch API를 사용할 수 있습니다.

또한 Axios와 SWR을 사용할 수도 있습니다. Axios는 XMLHttpRequest 객체 위에 구축된 프로미스 기반 HTTP 요청 라이브러리이고, SWR은 Vercel 팀이 만든 데이터 가져기를 위한 React 훅입니다.

예제부터 시작해 보겠습니다. 먼저 `useState`와 `useEffect` 훅을 가져와야 합니다:

```js
import { useState, useEffect } from "react";
```

그다음 `loading`, `data`, `error`라는 세 개의 상태 변수를 만들어야 합니다:

```js
const [data, setData] = useState(null);
const [loading, setLoading] = useState(true);
const [error, setError] = useState(null);
```

`loading` 변수는 데이터가 아직 가져오는 중인지 추적합니다. `data` 변수는 데이터 자체를 나타내고, `error` 변수는 데이터 가져오기 과정에서 발생할 수 있는 오류를 포착합니다.

데이터 가져오기는 부수 효과이므로, Fetch API를 `useEffect` 훅 안에서 사용하는 것이 가장 좋습니다.

예시는 다음과 같습니다:

```js
useEffect(() => {
  fetch("https://jsonplaceholder.typicode.com/posts")
    .then((res) => res.json())
    .then((data) => {
      setData(data);
      setLoading(false);
    })
    .catch((err) => {
      setError(err);
      setLoading(false);
    });
}, []);
```

이 `useEffect`는 Fetch API로 데이터를 가져오고 모든 상태를 설정합니다.

`async` 구문 대신 `await`/`.then()`를 사용하면 더 나아집니다. 즉, `useEffect` 키워드로 `useEffect`를 접두사할 수 없기 때문에 `async` 안에 별도의 함수를 만들어야 합니다:

```js
useEffect(() => {
  const fetchData = async () => {
    try {
      const res = await fetch("https://jsonplaceholder.typicode.com/posts");
       
      if (!res.ok) {
        throw new Error("Network response was not ok");
      }

      const data = await res.json();
      setData(data);
    } catch (err) {
      setError(err);
    } finally {
      setLoading(false);
    }
  };

  fetchData();
}, []);
```

그다음 API에서 가져온 데이터를 렌더링하기 위해 이 모든 상태를 사용할 수 있습니다.

전체 코드는 다음과 같습니다:

```jsx
import { useState, useEffect } from "react";

const FetchPosts = () => {
  const [data, setData] = useState(null);
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState(null);

  useEffect(() => {
    const fetchData = async () => {
      try {
        const res = await fetch("https://jsonplaceholder.typicode.com/posts");
       
        if (!res.ok) {
          throw new Error("Network response was not ok");
        }

        const data = await res.json();
        setData(data);
      } catch (err) {
        setError(err);
      } finally {
        setLoading(false);
      }
    };

    fetchData();
  }, []);

  if (loading) {
    return <p>Loading...</p>;
  }

  if (error) {
    return <p>{error.message}</p>;
  }

  return (
    <ul>
      {data.map((post) => (
        <li key={post.id}>{post.title}</li>
      ))}
    </ul>
  );
};

export default FetchPosts;
```

UI에서는 데이터가 가져오는 중일 때 `Loading...`가 화면에 표시되고, 데이터 가져오기가 성공했는지에 따라 데이터나 오류가 표시됩니다.

Axios와 SWR로 데이터 가져오기에 대해서도 이야기했었죠. Axios를 사용하는 예제를 살펴보겠습니다.

먼저 명령줄에서 Axios를 이렇게 설치해야 합니다:

```sh
npm i axios
```

그다음 Axios를 이렇게 가져와야 합니다:

```js
import axios from "axios";
```

이전과 같은 상태 변수를 사용하고 `axios.get`를 사용해 API에서 데이터를 가져올 수 있습니다:

```js
const [data, setData] = useState(null);
const [loading, setLoading] = useState(true);
const [error, setError] = useState(null);

useEffect(() => {
  const fetchData = async () => {
    try {
      const res = await axios.get(
        "https://jsonplaceholder.typicode.com/users"
      );
      setData(res.data);
    } catch (err) {
      setError(err);
    } finally {
      setLoading(false);
    }
  };

  fetchData();
}, []);
```

이 예제에는 `await res.json()` 줄이 없다는 점을 눈치챘을 겁니다. Axios는 JSON을 자동으로 파싱하기 때문에 그럴 필요가 없습니다.

마지막 예제는 `useSWR` 훅을 사용해 데이터를 가져오는 방법입니다.

Axios와 마찬가지로 SWR도 이렇게 설치해야 합니다:

```sh
npm install swr
```

그다음 `useSWR` 훅을 파일에 이렇게 가져와야 합니다:

```js
import useSWR from "swr";
```

이전 예제들과 비교해 보면 SWR 구문이 훨씬 짧습니다. 해야 할 일은 fetcher 함수를 만들고 이를 `useSWR` 훅의 두 번째 매개변수로 전달하는 것입니다(엔드포인트가 첫 번째 매개변수입니다).

또한 `useSWR` 훅에서 데이터와 오류 상태를 모두 구조 분해할당할 수 있으므로 `useState` 훅이 필요 없습니다.

구문은 다음과 같습니다:

```js
const fetcher = (url) => fetch(url).then((res) => res.json());
const { data, error } = useSWR(endpoint, fetcher);
```

여기서 "fetcher"라는 이름은 단지 관례일 뿐이므로 원하는 변수명으로 자유롭게 바꿀 수 있습니다.

다음은 JSON Placeholder API에서 할 일을 가져오는 컴포넌트입니다:

```jsx
import useSWR from "swr";

const fetcher = (url) => fetch(url).then((res) => res.json());

const FetchTodos = () => {
  const { data, error } = useSWR(
    "https://jsonplaceholder.typicode.com/todos",
    fetcher
  );

  if (!data) {
    return <h2>Loading...</h2>;
  }
  if (error) {
    return <h2>Error: {error.message}</h2>;
  }

  return (
    <>
      <h2>Todos</h2>
      <div>
        {data.map((todo) => (
          <h3 key={todo.id}>{todo.title}</h3>
        ))}
      </div>
    </>
  );
};

export default FetchTodos;
```

이전에 커스텀 훅에 대해 배운 것처럼, 데이터 가져오기는 커스텀 훅으로 추출할 수 있는 로직입니다. 여러 컴포넌트와 페이지에서 데이터를 가져온다면 `useFetch` 훅을 만드는 것이 가장 좋습니다.

다음은 SWR을 사용해 데이터 가져오기를 하는 `useFetch` 훅입니다:

```jsx
import useSWR from "swr";

const fetcher = (url) => fetch(url).then((res) => res.json());

const useFetch = (url) => {
  const { data, error } = useSWR(url, fetcher);

  return {
    data,
    loading: !data && !error,
    error,
  };
};

export default useFetch;
```

그리고 JSON Placeholder API에서 게시물을 가져오는 첫 번째 예제를 `useFetch` 훅으로 다시 작성하는 방법은 다음과 같습니다:

```jsx
import useFetch from "./useFetch";

const FetchPosts = () => {
  const { data, loading, error } = useFetch(
    "https://jsonplaceholder.typicode.com/posts"
  );

  if (loading) {
    return <h2>Loading...</h2>;
  }

  if (error) {
    return <h2>{error.message}</h2>;
  }

  return (
    <>
      <h2>Posts</h2>
      <ul>
        {data.map((post) => (
          <li key={post.id}>{post.title}</li>
        ))}
      </ul>
    </>
  );
};

export default FetchPosts;
```
