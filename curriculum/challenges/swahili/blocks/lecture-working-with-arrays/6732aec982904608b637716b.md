# --interactive--

Safu za data katika JavaScript ni za mabadiliko, ambayo inamaanisha unaweza kwa urahisi kuongeza au kuondoa vipengele kutoka kwao. Kuna njia kuu nne za kuongeza na kuondoa vipengele mwanzoni na mwishoni mwa safu ya data: `push()`, `pop()`, `shift()`, na `unshift()`. Hebu tuchunguze kila njia hizi kwa undani.

Njia ya `push()` hutumika kuongeza kipengele kimoja au zaidi mwishoni mwa safu ya data. Thamani ya return ya njia ya `push()` ni urefu mpya wa safu ya data. Hapa kuna mfano wa kuongeza tunda jipya kwenye safu ya data iliyopo ya `fruits`:

:::interactive_editor

```js
const fruits = ["apple", "banana"];
const newLength = fruits.push("orange");
console.log(newLength); // 3
console.log(fruits); // ["apple", "banana", "orange"]
```

:::

Katika mfano huu, tunaanza na safu ya data iitwayo `fruits` ambayo ina vipengele viwili. Kisha tunatumia njia ya `push()` kuongeza mfuatano wa herufi `orange` mwishoni mwa safu ya data.

Huenda umeona kwamba tunatumia `const` wakati tunatangaza safu ya data `fruits`. Lakini kwa nini inawezekana kuongeza vipengele zaidi kwenye safu ya data hii ya `fruits` wakati `fruits` ni thabiti? Hii inawezekana kwa sababu kutangaza safu ya data kwa neno kuu la `const` huunda rejea kwa safu ya data. Wakati safu ya data yenyewe ni ya kubadilika na inaweza kubadilishwa, huwezi kuweka thamani mpya kwa thabiti ya `fruits`, kama ifuatavyo:

```js
const fruits = ["apple", "banana"];
fruits = ["This", "will", "not", "work"];
console.log(fruits); // Uncaught TypeError: Assignment to constant variable. 
```

Njia inayofuata tutakayochunguza ni njia ya `pop()`. Njia ya `pop()` huondoa kipengele cha mwisho kutoka kwa safu ya data na kurudisha kipengele hicho. Pia hubadilisha safu ya data ya awali. Hivi ndivyo inavyofanya kazi:

:::interactive_editor

```js
let fruits = ["apple", "banana", "orange"];
let lastFruit = fruits.pop();
console.log(fruits); // ["apple", "banana"]
console.log(lastFruit); // "orange"
```

:::

Katika mfano huu, tunaanza na safu ya matunda matatu. Njia ya `pop()` huondoa kipengele cha mwisho (`orange`) kutoka kwa safu ya data na kurudisha. Safu ya data ya awali ya `fruits` imerekebishwa na sasa ina vipengele viwili tu.

Njia ya `unshift()` huongeza kipengele kimoja au zaidi mwanzoni mwa safu ya data na kurudisha urefu wake mpya. Inafanya kazi kama `push()`, lakini hubadilisha mwanzo wa safu ya data badala ya mwisho. Hapa kuna mfano:

:::interactive_editor

```js
let numbers = [2, 3];
let newLength = numbers.unshift(1);
console.log(numbers); // [1, 2, 3]
console.log(newLength); // 3
```

:::

Katika mfano huu, tunatumia `unshift()` kuongeza nambari `1` mwanzoni mwa safu ya data ya `numbers`. Njia hii inarudisha urefu mpya wa safu ya data, ambao ni `3`.

Mwishowe, njia ya `shift()` huondoa kipengele cha kwanza kutoka kwa safu ya data na kurudisha kipengele hicho. Ni sawa na `pop()`, lakini hufanya kazi mwanzoni mwa safu ya data badala ya mwisho. Hivi ndivyo inavyofanya kazi:

:::interactive_editor

```js
let colors = ["red", "green", "blue"];
let firstColor = colors.shift();
console.log(colors); // ["green", "blue"]
console.log(firstColor); // "red"
```

:::

Katika mfano huu, tunaanza na safu ya rangi tatu. Njia ya `shift()` huondoa kipengele cha kwanza (`red`) kutoka kwa safu ya data na kurudisha. Safu ya data ya awali ya `colors` imerekebishwa na sasa ina vipengele viwili tu.

Kumbuka kwamba wakati `push()` na `unshift()` zinaweza kuongeza vipengele vingi mara moja, `pop()` na `shift()` huondoa kipengele kimoja tu kwa wakati.
