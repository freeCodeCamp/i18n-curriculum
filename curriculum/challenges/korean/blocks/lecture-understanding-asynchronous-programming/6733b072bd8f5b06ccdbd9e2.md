# --description--

비동기 JavaScript에 대해 배워봅시다. 비동기 이벤트는 서로 독립적으로, 서로 다른 시간에 발생합니다.

소프트웨어 개발 맥락에서 "비동기"는 프로그램의 주요 흐름과 독립적으로 백그라운드에서 실행되는 작업을 의미합니다. 비동기 프로세스의 가장 큰 장점은 메인 프로그램의 실행을 막지 않는다는 점입니다.

이는 원격 서버에서 데이터를 가져오거나, 큰 파일을 처리하거나, 사용자 입력을 다루거나, 복잡한 계산을 수행하는 등 실행하는 데 시간이 오래 걸릴 수 있는 작업에 특히 유용합니다. 이것이 바로 비동기 프로그래밍입니다.

이 접근법은 지금까지 작업해온 전통적인 동기 프로그래밍 기법과 대조됩니다. 여러 줄의 코드를 작성할 때, 보통 어떤 일이 언제 일어날지 예측할 수 있습니다. 첫 번째 줄이 실행되고, 그 다음 두 번째 줄이 실행되는 식으로 작성한 순서대로 진행됩니다.

이 예시에서는 변수를 정의하는 첫 번째 줄이 먼저 실행되고, 그다음 두 번째 줄, 마지막으로 문자열이 콘솔에 기록되는 세 번째 줄이 실행됩니다:

```js
const topic = "JavaScript";
const learning = `I'm learning ${topic}!`;
console.log(learning);
```

모든 것이 순차적이고 예측 가능합니다. 각 줄이 완료된 후에 다음 줄이 시작됩니다. 이런 유형의 JavaScript 프로그램을 단일 스레드라고 합니다.

스레드 개념은 매우 중요합니다. 스레드는 프로세스 내에서 실행 단위입니다. 프로그램 내에서 별도의 제어 흐름과 같습니다.

동기 프로그래밍에서는 스레드가 순차적으로 하나씩 실행됩니다. 만약 스레드가 사용자 입력을 기다리는 등 차단되면, 해당 스레드가 완료될 때까지 전체 프로세스가 멈춥니다.

비동기 프로그래밍에서는 여러 스레드를 동시에 실행할 수 있습니다. 이렇게 하면 한 스레드가 차단되어도 메인 프로그램이 멈추지 않고 여러 작업을 동시에 계속 실행할 수 있습니다.

비동기 프로그래밍은 종종 콜백, 프로미스, 또는 async/await를 사용해 논블로킹 작업을 처리합니다.

앞서 수업에서 배운 것처럼, 콜백은 나중에 호출하기 위해 다른 함수에 전달하는 함수입니다. 이벤트 핸들러는 이전에 다뤘던 콜백의 특별한 유형입니다. 과거에는 JavaScript에서 비동기 함수를 구현하는 가장 일반적인 방법이었습니다. 하지만 콜백 함수가 또 다른 콜백 함수를 받으면, 코드와 로직이 매우 복잡해질 수 있습니다.

현재 JavaScript에서 비동기 프로그래밍을 구현하는 가장 일반적인 기법은 프로미스입니다. `Promise`는 비동기 프로세스의 최종 완료(또는 실패)와 그 값을 나타내는 객체입니다.

프로미스의 값은 프로미스가 생성될 때 알 수 없습니다. 비동기 프로세스가 완료될 때만 알 수 있습니다. 예를 들어, 웹 애플리케이션을 위한 원격 API에서 데이터를 가져오는 과정은 시간이 걸릴 수 있습니다. 그동안 사용자에게 좋은 사용자 경험을 제공하고 싶겠죠?

이를 구현하려면, 비동기 프로세스가 실행되는 동안 사용자 인터페이스를 활성 상태로 유지하는 프로미스를 만들 수 있습니다. 프로세스가 완료되면 프로미스는 API가 반환한 데이터를 포함하므로, 애플리케이션이 데이터를 적절히 처리하거나 렌더링할 수 있습니다.

비동기 프로그래밍은 효율적인 JavaScript 애플리케이션을 만드는 강력한 도구입니다. 동기 프로그래밍과 비동기 프로그래밍의 차이를 이해하면 애플리케이션에 맞는 접근법을 선택하고 더 효율적인 코드를 작성할 수 있습니다.
