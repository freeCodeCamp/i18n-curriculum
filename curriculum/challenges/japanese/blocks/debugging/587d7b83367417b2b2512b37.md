# --description--

freeCodeCampのいくつかのチャレンジには独自のコンソールが含まれていることに気づいたかもしれません。このコンソールはブラウザコンソールとは少し動作が異なります。

`console`でメッセージを出力するために使えるメソッドはたくさんあります。例えば`log`、`warn`、`clear`などです。freeCodeCampコンソールは`log`メッセージのみを出力しますが、ブラウザコンソールはすべてのメッセージを出力します。コードを変更すると自動的に実行されてログが表示されます。freeCodeCampコンソールはコードが実行されるたびにクリアされます。

# --instructions--

まず、ログを確認できるようにブラウザコンソールを開いてください。そのためには、ほとんどのブラウザでfreeCodeCampのナビゲーションバーを右クリックし、`inspect`をクリックします。次に開いたウィンドウで`console`タブを探してください。

その後、`console.log`を使って`output`変数をログに出力してください。2つのコンソールを見てログを確認しましょう。最後に、ログの後に`console.clear`を使ってブラウザコンソールをクリアしてください。2つのコンソールの違いを確認しましょう。

# --hints--

```js
    .match(/console\.log\(output\)[\s\S]*console.clear\(\)/)
);
```
