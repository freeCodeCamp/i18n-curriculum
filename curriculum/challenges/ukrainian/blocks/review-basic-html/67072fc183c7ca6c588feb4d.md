# --interactive--

## Основи HTML

- **Роль HTML**: HTML представляє вміст і структуру вебсторінки. 
- **HTML-елементи**: Елементи — це будівельні блоки HTML-документа. Вони представляють заголовки, абзаци, посилання, зображення та інше. Більшість HTML-елементів складаються з початкового тегу (`<elementName>`) і кінцевого тегу (`</elementName>`).

Ось базовий синтаксис:

```html
<elementName>Content goes here</elementName>
```

- **Порожні елементи**: Порожні елементи не можуть мати вміст і мають лише початковий тег. Прикладами є елементи `img` і `meta`.

```html
<img>
<meta>
```

Часто можна побачити код, де всередині порожнього елемента є коса риска `/`. Обидва варіанти є прийнятними:

```html
<img>
<img/>
```

- **Атрибути**: Атрибут — це значення, розміщене всередині початкового тегу HTML-елемента. Атрибути надають додаткову інформацію про елемент або визначають, як елемент має поводитися. Ось базовий синтаксис атрибута:

```html
<element attribute="value"></element>
```

Булевий атрибут — це атрибут, який може бути присутнім або відсутнім у HTML-тезі. Якщо він присутній, його значення true, інакше — хиба. Прикладами булевих атрибутів є `disabled`, `readonly` і `required`. 

- **Коментарі**: Коментарі використовуються в програмуванні, щоб залишати нотатки для себе та інших розробників у коді. Ось синтаксис коментаря в HTML:

```html
<!--This is an HTML comment.-->
```

## Загальні HTML-елементи

- **Елементи заголовків**: В HTML є шість елементів заголовків. Елементи заголовків від `h1` до `h6` використовуються, щоб позначити важливість вмісту, що йде після них. Чим менше число, тим вища важливість, тому елементи `h2` мають меншу важливість, ніж `h1`.

:::interactive_editor

```html
<h1>most important heading element</h1>
<h2>second most important heading element</h2>
<h3>third most important heading element</h3>
<h4>fourth most important heading element</h4>
<h5>fifth most important heading element</h5>
<h6>least important heading element</h6>
```

:::

- **Елементи абзаців**: Використовуються для абзаців на вебсторінці.

:::interactive_editor

```html
<p>This is a paragraph element.</p>
```

:::

- **Елементи `img`**: Елемент `img` використовується для додавання зображень на вебсторінку. Атрибут `src` визначає розташування цього зображення. Для елементів зображень рекомендується додавати ще один атрибут — `alt`. Ось приклад елемента `img` з атрибутами `src` і `alt`:

:::interactive_editor

```html
<img src="https://cdn.freecodecamp.org/curriculum/cat-photo-app/lasagna.jpg" alt="A slice of lasagna on a plate.">
```

:::

- **Елемент `body`**: Цей елемент використовується для представлення вмісту HTML-документа.

:::interactive_editor

```html
<body>
  <h1>CatPhotoApp</h1>
  <p>This is a paragraph element.</p>
</body>    
```

:::

- **Елементи `section`**: Цей елемент використовується для поділу вмісту на менші розділи.

:::interactive_editor

```html
<section>
  <h2>About Me</h2>
  <p>Hi, I am Jane Doe and I am a web developer.</p>
</section>
```

:::

- **Елементи `div`**: Це загальний HTML-елемент, який не має семантичного значення. Використовується як загальний контейнер для інших HTML-елементів.

:::interactive_editor

```html
<div>
  <h1>I am a heading</h1>
  <p>I am a paragraph</p>
</div>
```

:::

- **Елементи посилань (`a`)**: Ці елементи використовуються для додавання посилань на вебсторінку. Атрибут `href` визначає, куди має вести посилання, коли користувач натискає на нього.

:::interactive_editor

```html
<a href="https://cdn.freecodecamp.org/curriculum/cat-photo-app/running-cats.jpg">cute cats</a>
```

:::

- **Елементи невпорядкованого (`ul`) та впорядкованого (`ol`) списків**: Щоб створити маркований список, слід використовувати елемент `ul` з одним або кількома вкладеними елементами `li`, як у цьому прикладі:

:::interactive_editor

```html
<ul>
  <li>catnip</li>
  <li>laser pointers</li>
  <li>lasagna</li>
</ul>
```

:::

Щоб створити впорядкований список, слід використовувати елемент `ol`:

:::interactive_editor

```html
<ol>
  <li>flea treatment</li>
  <li>thunder</li>
  <li>other cats</li>
</ol>
```

:::

- **Елемент виділення (`em`)**: Використовується для виділення частини тексту.

:::interactive_editor

```html
<p>Cats <em>love</em> lasagna.</p>  
```

:::

- **Елемент сильної важливості (`strong`)**: Використовується для сильного виділення тексту, щоб підкреслити терміновість і серйозність.

:::interactive_editor

```html
<p>
  <strong>Important:</strong> Before proceeding, make sure to wear your safety goggles. 
</p>
```

:::

- **Елементи `figure` і `figcaption`**: Елемент `figure` використовується для групування вмісту, наприклад, зображень і діаграм. Елемент `figcaption` використовується для підпису цього вмісту всередині елемента `figure`.

:::interactive_editor

```html
<figure>
  <img src="https://cdn.freecodecamp.org/curriculum/cat-photo-app/cats.jpg" alt="Two tabby kittens sleeping together on a couch.">
  <figcaption>Cats <strong>hate</strong> other cats.</figcaption>  
</figure>
```

:::

- **Елемент `main`**: Використовується для представлення основного вмісту вебсторінки.
- **Елемент `footer`**: Розміщується внизу HTML-документа і зазвичай містить інформацію про авторські права та інші важливі посилання сторінки.

:::interactive_editor

```html
<footer>
  <p>
    No Copyright - <a href="https://www.freecodecamp.org">freeCodeCamp.org</a>
  </p>
</footer>
```

:::

## Ідентифікатори та групування

- **Ідентифікатори (ID)**: Унікальні ідентифікатори елементів HTML. Імена ID слід використовувати лише один раз у HTML-документі.

```html
<h1 id="title">Movie Review Page</h1>
```

Імена ID не можуть містити пробілів. Якщо ім’я ID складається з кількох слів, між ними можна використовувати дефіси, як у цьому прикладі:

```html
<div id="red-box"></div>
```

- **Класи**: Використовуються для групування елементів для стилізації та поведінки.

```html
<div class="box"></div>
```

На відміну від ID, однакове ім’я класу можна використовувати багаторазово в HTML-документі. Значення `class` також може містити пробіли, як у цьому прикладі:

```html
<div class="box red-box"></div>
<div class="box blue-box"></div>
```

## Спеціальні символи та посилання

- **HTML-сутності**: HTML-сутність або посилання на символ — це набір символів, який використовується для представлення зарезервованого символу в HTML. Прикладами є символ амперсанда (`&amp;`) і символ меншого за (`&lt;`).

```html
<p>This is an &lt;img /&gt; element</p>
```

- **Елемент `link`**: Використовується для підключення зовнішніх ресурсів, таких як таблиці стилів і іконки сайту. Ось базовий синтаксис використання елемента `link` для зовнішнього CSS-файлу:

```html
<link rel="stylesheet" href="./styles.css" />
```

Атрибут `rel` використовується для визначення зв’язку між підключеним ресурсом і HTML-документом. Атрибут `href` визначає розташування URL зовнішнього ресурсу.

- **Елемент `script`**: Використовується для вставки виконуваного коду.

```html
<body>
  <script>
    alert("Welcome to freeCodeCamp");
  </script>
</body>
```

Хоча технічно весь JavaScript-код можна писати всередині тегів `script`, найкращою практикою вважається підключення зовнішнього JavaScript-файлу. Ось приклад використання елемента `script` для підключення зовнішнього JavaScript-файлу:

```html
<script src="path-to-javascript-file.js"></script>
```

Атрибут `src` тут використовується для визначення розташування зовнішнього JavaScript-файлу.

## Шаблонний код і кодування

- **HTML-шаблонний код**: Це шаблон, який включає базову структуру та необхідні елементи, які потрібні кожному HTML-документу.

:::interactive_editor

```html
<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="utf-8" />
    <title>freeCodeCamp</title>
    <link rel="stylesheet" href="./styles.css" />
  </head>
  <body>
    <!--Headings, paragraphs, images, etc. go inside here-->
  </body>
</html>
```

:::

- **`DOCTYPE`**: Використовується, щоб повідомити браузерам, яку версію HTML ви використовуєте.
- **Елемент `html`**: Представляє елемент найвищого рівня або корінь HTML-документа. Щоб вказати мову документа, слід використовувати атрибут `lang`.
- **Елемент `head`**: Розділ `head` містить важливі метадані — інформацію, яка потрібна браузерам і пошуковим системам.
- **Елементи `meta`**: Представляють метадані вашого сайту. Ці елементи містять деталі, наприклад, про кодування символів, і як сайти на кшталт Twitter мають попередньо переглядати посилання на вашу сторінку тощо.
- **Елемент `title`**: Використовується для встановлення тексту, який відображається на вкладці або вікні браузера.
- **Кодування символів UTF-8**: UTF-8, або UCS Transformation Format 8, — це стандартизоване кодування символів, широко використовуване в інтернеті. Кодування символів — це спосіб, яким комп’ютери зберігають символи як дані. Атрибут `charset` використовується всередині елемента `meta` для встановлення кодування символів у UTF-8.

## SEO та соціальний обмін

- **SEO**: Оптимізація для пошукових систем — це практика, яка оптимізує вебсторінки, щоб вони ставали більш помітними і займали вищі позиції у пошукових системах.  
- **Елемент мета (`description`)**: Використовується для надання короткого опису вебсторінки, що впливає на SEO.

```html
<meta
  name="description"
  content="Discover expert tips and techniques for gardening in small spaces, choosing the right plants, and maintaining a thriving garden."
/>
```

- **Теги Open Graph**: Протокол Open Graph дозволяє контролювати, як вміст вашого сайту відображається на різних соціальних платформах, таких як Facebook, LinkedIn та інші.  
Встановлюючи ці властивості Open Graph, ви можете заохотити користувачів натиснути і взаємодіяти з вашим вмістом.  
Ви можете встановити ці властивості через набір елементів `meta` всередині розділу `head` вашого HTML-документа.
- **Властивість `og:title`**: Використовується для встановлення заголовка, який відображається у соціальних мережах.

```html
<meta content="freeCodeCamp.org" property="og:title" />
```

- **Властивість `og:type`**: Властивість `type` використовується для представлення типу вмісту, який поширюється у соціальних мережах. Прикладами такого вмісту є статті, вебсайти, відео або музика.

```html
<meta property="og:type" content="website" />
``` 

- **Властивість `og:image`**: Використовується для встановлення зображення, яке показується у соціальних мережах.

```html
<meta
  content="https://cdn.freecodecamp.org/platform/universal/fcc_meta_1920X1080-indigo.png"
  property="og:image"
/>
```

- **Властивість `og:url`**: Використовується для встановлення URL, за яким користувачі натискають у соціальних мережах.

```html
<meta property="og:url" content="https://www.freecodecamp.org" />
```

## Медіаелементи та оптимізація

- **Замінені елементи**: Замінений елемент — це елемент, вміст якого визначається зовнішнім ресурсом, а не CSS. Прикладом є елемент `iframe`. `iframe` означає рядковий фрейм. Це рядковий елемент, який використовується для вставки іншого HTML-вмісту безпосередньо в HTML-сторінку.

```html
<iframe src="https://www.example.com" title="Example Site"></iframe>
```

Ви можете додати атрибут `allowfullscreen`, який дозволяє користувачу відображати iframe у повноекранному режимі.

```html
<iframe
  src="video-url"
  width="width-value"
  height="height-value"
  allowfullscreen
></iframe>
```

Щоб вставити відео всередину `iframe`, ви можете скопіювати його безпосередньо з популярних відеосервісів, таких як YouTube і Vimeo, або визначити його самостійно за допомогою атрибута `src`, який вказує URL цього відео. Ось приклад вставлення популярного курсу freeCodeCamp з YouTube:

:::interactive_editor

```html
<h1>A freeCodeCamp YouTube Video Embedded with the iframe Element</h1>

<iframe
  width="560"
  height="315"
  src="https://www.youtube.com/embed/PkZNo7MFNFg?si=-UBVIUNM3csdeiWF"
  title="YouTube video player"
  allow="accelerometer; autoplay; clipboard-write; encrypted-media; gyroscope; picture-in-picture; web-share"
  referrerpolicy="strict-origin-when-cross-origin"
  allowfullscreen
></iframe>
```

:::

Існують інші замінені елементи, такі як `video` і `embed`. Деякі елементи поводяться як замінені за певних обставин. Ось приклад елемента `input` з атрибутом `type`, встановленим у `image`:

```html
<input type="image" alt="Descriptive text goes here" src="example-img-url">
```

- **Оптимізація медіа**: Існують три інструменти, які слід враховувати при використанні медіа, наприклад, зображень, на ваших вебсторінках: розмір, формат і стиснення. Алгоритм стиснення використовується для зменшення розміру файлів або даних.
- **Формати зображень**: Два з найпоширеніших форматів файлів — PNG і JPG, але вони вже не є найкращими для подачі зображень. Якщо вам не потрібна підтримка старих браузерів, варто розглянути більш оптимізовані формати, такі як WEBP або AVIF.
- **Ліцензії на зображення**: Зображення у публічному надбанні не мають авторських прав і можуть використовуватися без обмежень. Зображення, ліцензовані за Creative Commons 0, вважаються публічним надбанням. Деякі зображення можуть бути випущені під дозволяючою ліцензією, наприклад, Creative Commons або BSD, яку використовує freeCodeCamp.
- **SVG**: Масштабовані векторні графіки базуються на шляхах і рівняннях для побудови точок, ліній і кривих. Це означає, що векторна графіка, як SVG, може масштабуватися до будь-якого розміру без втрати якості.

## Інтеграція мультимедіа

- **Елементи `audio` і `video`**: Елементи `audio` і `video` дозволяють додавати звуковий і відеоконтент у ваші HTML-документи. Елемент `audio` підтримує популярні аудіоформати, такі як mp3, wav і ogg. Елемент `video` підтримує формати mp4, ogg і webm.

```html
<audio src="CrystalizeThatInnerChild.mp3"></audio>
```

Якщо ви хочете бачити аудіоплеєр на сторінці, додайте елемент `audio` з атрибутом `controls`:

:::interactive_editor

```html
<audio src="CrystalizeThatInnerChild.mp3" controls></audio>
```

:::

Атрибут `controls` дозволяє користувачам керувати відтворенням аудіо, включно з регулюванням гучності, паузою та відновленням відтворення. Атрибут `controls` — булевий атрибут, який можна додати до елемента, щоб увімкнути вбудовані елементи керування відтворенням. Якщо його опустити, керування не відображатиметься.

- **Атрибут `loop`**: Атрибут `loop` — булевий атрибут, який робить відтворення аудіо безперервним.

:::interactive_editor

```html
<audio
  src="https://cdn.freecodecamp.org/curriculum/js-music-player/can't-stay-down.mp3"
  loop
  controls
></audio>
```

:::

- **Атрибут `muted`**: Якщо він присутній у елементі `audio`, булевий атрибут `muted` запускає аудіо в приглушеному стані.

:::interactive_editor

```html
<audio
  src="https://cdn.freecodecamp.org/curriculum/js-music-player/can't-stay-down.mp3"
  loop
  controls
  muted
></audio>
```

:::

- **Елемент `source`**: Щодо типів аудіофайлів, браузери підтримують різні формати. Щоб врахувати це, можна використовувати елементи `source` всередині елемента `audio`, і браузер вибере перше джерело, яке він розуміє. Ось приклад використання кількох елементів `source` для елемента `audio`:

```html
<audio controls>
  <source src="audio.ogg" type="audio/ogg" />
  <source src="audio.wav" type="audio/wav" />
  <source src="audio.mp3" type="audio/mpeg" />
</audio>
```

Всі атрибути, які ми вивчили досі, також підтримуються в елементі `video`. Ось приклад використання елемента `video` з атрибутами `loop`, `controls` і `muted`:

:::interactive_editor

```html
<video
  src="https://archive.org/download/BigBuckBunny_124/Content/big_buck_bunny_720p_surround.mp4"
  loop
  controls
  muted
></video>
```

:::

- **Атрибут `poster`**: Якщо ви хочете показати зображення під час завантаження відео, можна використати атрибут `poster`. Цей атрибут недоступний для елементів `audio` і є унікальним для елемента `video`.

:::interactive_editor

```html
<video
  src="https://archive.org/download/BigBuckBunny_124/Content/big_buck_bunny_720p_surround.mp4"
  loop
  controls
  muted
  poster="https://peach.blender.org/wp-content/uploads/title_anouncement.jpg?x11217"
  width="620"
></video>
```

:::

## Типи атрибутів target

- **Атрибут `target`**: Цей атрибут повідомляє браузеру, де відкривати URL для елемента посилання. Існує чотири важливі можливі значення цього атрибута: `_self`, `_blank`, `_parent` і `_top`. Існує п’яте значення, `_unfencedTop`, яке наразі використовується для експериментального API `FencedFrame`. Ймовірно, вам поки що не знадобиться це значення.
- **Значення `_self`**: Це значення за замовчуванням для атрибута `target`. Відкриває посилання в поточному контексті перегляду. У більшості випадків це поточна вкладка або вікно.

```html
<a href="https://freecodecamp.org" target="_self">Visit freeCodeCamp</a>
```

- **Значення `_blank`**: Відкриває посилання в новому контексті перегляду. Зазвичай це нова вкладка, але деякі користувачі можуть налаштувати браузери на відкриття нового вікна.

```html
<a href="https://freecodecamp.org" target="_blank">Visit freeCodeCamp</a>
```

- **Значення `_parent`**: Відкриває посилання у батьківському контексті поточного. Наприклад, якщо ваш сайт має iframe, значення `_parent` у цьому iframe відкриється у вкладці/вікні вашого сайту, а не в самому вбудованому фреймі.

```html
<a href="https://freecodecamp.org" target="_parent">Visit freeCodeCamp</a>
```

- **Значення `_top`**: Відкриває посилання у найвищому контексті перегляду — подумайте «батько батька». Це схоже на `_parent`, але посилання завжди відкривається у повній вкладці/вікні браузера, навіть для вкладених вбудованих фреймів.

```html
<a href="https://freecodecamp.org" target="_top">Visit freeCodeCamp</a>
```

## Абсолютні та відносні шляхи

- **Визначення шляху**: Шлях — це рядок, який визначає розташування файлу або каталогу в файловій системі. У веброзробці шляхи дозволяють розробникам посилатися на ресурси, такі як зображення, таблиці стилів, скрипти та інші вебсторінки.
- **Синтаксис шляху**: Існує три ключові синтаксиси. Перший — скісна риска, яка може бути зворотною (`\`) або прямою (`/`) залежно від операційної системи. Другий — одна крапка (`.`). І нарешті, подвійна крапка (`..`). Скісна риска — це «роздільник шляху». Вона використовується, щоб позначити розрив між іменами папок або файлів. Одна крапка вказує на поточний каталог, а дві крапки — на батьківський каталог.

```md
public/index.html
./favicon.ico
../src/index.css
```

- **Абсолютний шлях**: Абсолютний шлях — це повне посилання на ресурс. Він починається з кореневого каталогу, включає всі інші каталоги і нарешті ім’я файлу з розширенням. «Кореневий каталог» — це каталог найвищого рівня в ієрархії. Абсолютний шлях також включає протокол — це може бути `http`, `https` або `file`, а також доменне ім’я, якщо ресурс знаходиться в інтернеті. Ось приклад абсолютного шляху, який посилається на логотип freeCodeCamp:

:::interactive_editor

```html
<a href="https://design-style-guide.freecodecamp.org/img/fcc_secondary_small.svg">
  View fCC Logo
</a>
```

:::

- **Відносний шлях**: Відносний шлях визначає розташування файлу відносно каталогу поточного файлу. Він не включає протокол або доменне ім’я, що робить його коротшим і гнучкішим для внутрішніх посилань на тому ж сайті. Ось приклад посилання на сторінку `about.html` зі сторінки `contact.html`, які знаходяться в одній папці:

```html
<p>
  Read more on the
  <a href="about.html">About Page</a>
</p>
```

## Стан посилань

- **`:link`**: Це стан за замовчуванням. Він означає посилання, яке користувач ще не відвідував, не натискав і не взаємодіяв з ним. Можна вважати цей стан базовим стилем для всіх посилань на сторінці. Інші стани будуються поверх нього.
- **`:visited`**: Застосовується, коли користувач уже відвідав сторінку, на яку веде посилання. За замовчуванням це робить посилання фіолетовим, але ви можете використовувати CSS, щоб надати інший візуальний сигнал користувачу.
- **`:hover`**: Цей стан застосовується, коли користувач наводить курсор на посилання. Він допомагає привернути додаткову увагу до посилання, щоб переконатися, що користувач дійсно хоче натиснути на нього.
- **`:focus`**: Цей стан застосовується, коли посилання отримує фокус.
- **`:active`**: Цей стан застосовується, коли користувач активує посилання. Зазвичай це означає натискання на посилання основною кнопкою миші (лівим кліком).
