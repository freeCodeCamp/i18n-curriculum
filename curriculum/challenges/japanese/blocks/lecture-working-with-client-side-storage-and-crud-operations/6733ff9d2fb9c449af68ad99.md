# --interactive--

クライアントサイドストレージは、ウェブサイトがユーザーのデバイスにデータを保存できるようにします。しかし、多くの技術と同様に、誤用されることがあります。

ここでは、クッキーとその追跡やフィンガープリンティングの誤用に焦点を当てて、クライアントサイドストレージに関連するネガティブなパターンを見ていきましょう。

まずはクッキーから始めましょう。

前のレッスンで、クッキーはユーザーがウェブアプリを訪れたときにユーザーのデバイスに保存されるデータであることを学びました。クッキーの一般的な誤用は過剰な追跡です。

ウェブサイトはクッキーを使ってユーザーのウェブアプリでの操作を追跡し、デジタル活動の履歴を作成します。これはターゲット広告のために行われますが、重大なプライバシーの懸念を引き起こす場合があります。

例えば、ショッピングサイトが次のような追跡クッキーを使うことがあります。

```js
document.cookie = "userID=123; path=/; expires=Thu, 18 Dec 2024 6:00:00 UTC";
```

このコードは`userID`という名前のクッキーを`123`の値で設定します。このクッキーはウェブサイトへのすべてのリクエストに送信され、ユーザーの行動を追跡できます。一見無害に思えますが、何百ものウェブサイトがこの情報を共有すると、あなたのオンラインでの生活や選択の非常に詳細な情報が作られてしまいます。

もう一つ懸念されるのはブラウザフィンガープリンティングです。

この手法はクライアントサイドの情報を使ってユーザーのブラウザのユニークな「フィンガープリント」を作成します。ウェブサイトはブラウザのバージョン、インストールされているプラグイン、画面の解像度などの情報を収集して、ユーザーを特定できます。

ウェブサイトがあなたのフィンガープリントを作成する簡単な例を示します。

:::interactive_editor

```js
let fingerprintExample = navigator.userAgent + screen.width + screen.height;
console.log(fingerprintExample);
```

:::

このコードはブラウザのユーザーエージェントと画面の寸法を組み合わせています。これは基本的な例ですが、実際のフィンガープリンティング手法はもっと複雑で、ユーザーを非常に正確に特定できます。

`localStorage`も誤用されることがあります。いくつかのウェブサイトは機密情報を安全でない方法で保存するために使っています。例えば：

```js
localStorage.setItem('userPassword', 'someonesPasswordHere');
```

このコードはユーザーのパスワードを`localStorage`に保存しています。これは重大なセキュリティリスクです。`localStorage`内のデータは暗号化されておらず、簡単にアクセスされてしまいます。

まとめると、クライアントサイドストレージは多くの利点がありますが、責任を持って使うことが重要です。ウェブ開発の旅を続ける中で、常にユーザーのデータのプライバシーとセキュリティを考慮してください。
