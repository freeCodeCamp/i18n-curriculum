## JavaScript 库和框架

- JavaScript 库和框架为常见问题提供快速解决方案，通过提供预构建代码加快开发速度。
- 库通常更专注于为特定任务提供解决方案，例如操作 DOM、处理事件或管理 AJAX 请求。
- JavaScript 库的几个例子是 jQuery 和 React。
- 另一方面，框架为构建应用提供更明确的结构。它们通常带有一套开发者需要遵循的规则和约定。
- 框架的例子包括 Angular 和 Next.js（React 的元框架）。
- **单页应用程序**（SPA）是加载单个 HTML 页面的网络应用，用户与应用交互时动态更新该页面，而无需重新加载整个页面。
- SPA 使用 JavaScript 管理应用状态和渲染内容。通常使用框架来构建复杂的用户界面，这些框架提供了强大的工具。
- 关于 SPA 的一些问题包括：

  - 屏幕阅读器难以处理动态更新的内容。
  - 用户在应用内导航时 URL 不会改变，这会导致难以书签、回退或分享特定页面。
  - 如果应用较大，初始加载时间可能较慢，因为所有资源都需要预先加载。

## React

- React 是一个流行的 JavaScript 库，用于构建用户界面和网络应用。
- React 的核心概念是创建可复用的 UI 组件，这些组件可以在数据变化时独立更新和渲染。
- React 允许开发者根据应用状态描述 UI 应该是什么样子。当数据或状态变化时，React 会更新并渲染正确的组件。

## React 组件

- 组件是 React 应用的构建块，允许开发者将复杂的用户界面拆分成更小、更易管理的部分。
- UI 使用 JSX 描述，JSX 是 JavaScript 语法的扩展，允许开发者在 JavaScript 中编写类似 HTML 的代码。
- 组件基本上是返回一段 UI 的 JS 函数或类。

下面是一个简单的 React 组件示例，它渲染一个问候消息：

```jsx
function Greeting() {
  const name = 'Anna';
  return <h1>Welcome, {name}!</h1>;
}
```

要使用该组件，你只需调用：
    
```jsx
  <Greeting />
```

## 导入和导出 React 组件

- 导出使组件可供其他文件使用。导入允许文件使用其他地方导出的组件。
- React 中有两种导出类型：默认导出和命名导出。

**默认导出**

- 一个文件只能有一个默认导出。适合主要包含单个组件的文件。
- 你可以这样导出一个组件作为默认导出：

```jsx
// City.js
function City() {
  return <p>New York</p>;
}

export default City;
```

- 也可以在组件定义的同一行导出：

```jsx
export default function City() {
  return <p>New York</p>;
}
```

- 导入默认导出时，使用 `import` 关键字且不带花括号：

```jsx
// App.js
import City from './City';

function App() {
  return (
    <div>
      <h1>My favorite city is:</h1>
      <City />
    </div>
  );
}
```

**命名导出**

- 命名导出允许文件共享多个组件。与默认导出不同，必须使用导出时的确切名称导入。
- 你可以这样从同一个文件导出多个组件：

```jsx
// Animals.jsx
export function Cat() {
  return <h2>Mr. Whiskers</h2>;
}

export function Dog() {
  return <h2>Fido</h2>;
}
```

- 导入命名导出时，使用花括号包裹确切的导出名称：

```jsx
import { Cat, Dog } from './Animals';
```

- 你可以在导入时使用 `as` 关键字重命名命名导出：

```jsx
import { Cat as Kitty } from './Animals';
```

**混合默认和命名导出**

- 一个文件可以同时有一个默认导出和多个命名导出：

```jsx
// Animals.jsx
export default function Cat() {
  return <h2>Mr. Whiskers</h2>;
}

export function Dog() {
  return <h2>Fido</h2>;
}
```

- 导入混合导出时，默认导出放在前面（不带花括号），后面跟命名导出（带花括号）：

```jsx
import Cat, { Dog } from './Animals';
```

## 使用 Vite 设置 React 项目

- 项目设置工具和 CLI 提供快速简便的方式启动新项目，让开发者专注于编写代码而非配置。

该命令使用 Vite 的 React 模板创建一个名为 `my-react-app` 的新 React 项目。在项目目录中，你会看到一个 `package.json` 文件，列出了项目依赖和命令。

```bash

npm install # installs the dependencies listed in the package.json file
```

- 依赖安装完成后，你会在项目中看到一个名为 `node_modules` 的新文件夹。

- 运行项目的命令如下：

```bash

```

- 然后，打开浏览器并访问 `http://localhost:5173`，即可看到你的 React 应用运行。

- 若想查看起始模板的代码，可以进入项目中的 `src` 文件夹，找到 `App.jsx` 文件。

## 在 React 组件中传递 props

- 在 React 中，props（属性的缩写）是从父组件向子组件传递数据的方式。这种机制用于创建可复用和动态的 UI 元素。
- props 可以是任何 JavaScript 值。要从父组件向子组件传递 props，在父组件的 JSX 中使用子组件时，将 props 作为属性添加。示例如下：

```jsx
// Parent component
function Parent() {
  const name = 'Anna';
  return <Child name={name} />;
}

// Child component
function Child(props) {
  return <h1>Hello, {props.name}!</h1>;
}
```

你可以使用扩展运算符 `(...)` 传递多个 props，前提是先将它们转换为对象。示例如下：
    
```jsx
// Parent component
function Parent() {
  const person = {
    name: 'Anna',
    age: 25,
    city: 'New York'
  };
  return <Child {...person} />;
}
```

在这段代码中，扩展运算符 `{...person}` 将 person 对象转换为单独的 props，传递给 Child 组件。

## React 中的条件渲染

- React 中的条件渲染允许你创建动态用户界面。它用于根据应用中的某些条件或状态显示不同内容。
- React 中有多种条件渲染方式。一种常见方法是使用三元运算符。示例如下：

```jsx
function Greeting({ isLoggedIn }) {
  return (
    <div>
      {isLoggedIn ? <h1>Welcome back!</h1> : <h1>Please log in</h1>}
    </div>
  );
}
```

- 另一种条件渲染方式是使用逻辑与（`&&`）运算符。当你只想在某个条件满足时渲染内容时，这种方式很有用。示例如下：

```jsx
function Greeting({ user }) {
  return (
    <div>
      {user && <h1>Welcome, {user.name}!</h1>}
    </div>
  );
}
```

在上述代码中，只有当 user 对象为真值时，`h1` 元素才会被渲染。

你也可以这样使用直接的 `if` 语句：

```js
function Greeting({ isLoggedIn }) {
  if (isLoggedIn) {
    return <h1>Welcome back!</h1>;
  }
  return <h1>Please sign in</h1>;
}
```

## React 中的列表渲染

- 在构建用户界面时，渲染列表是常见任务。
- 列表可以使用 JS 数组的 `map()` 方法迭代数组项，并返回新的 JSX 元素数组。
- 例如，如果你有一个名字数组想渲染成列表，可以这样做：

```jsx
function NameList({ names }) {
  return (
    <ul>
      {names.map((name, index) => (
        <li key={`${name}-${index}`}>{name}</li>
      ))}
    </ul>
  );
}
```

- 始终为每个列表项提供唯一的 key，帮助 React 管理更新和渲染。使用这些技巧，你可以在 React 应用中创建灵活、高效且动态的列表。

## React 中的内联样式

- React 中的内联样式允许你使用 JavaScript 对象直接为 JSX 元素应用 CSS 样式。
- 在 React 中应用内联样式，可以在 JSX 元素上使用 style 属性。style 属性接受一个对象，键是驼峰式大小写的 CSS 属性，值是对应的值。示例如下：

```js
function Greeting() {
  return (
    <h1
      style={{ color: 'blue', fontSize: '24px', backgroundColor: 'lightgray' }}
    >
      Hello, world!
    </h1>
  );
}

export default Greeting;
```

你也可以将样式提取到单独对象中，并通过 `style` 属性引用，如下：

```jsx
function Greeting() {

  const styles = {
    color: 'blue',
    fontSize: '24px',
    backgroundColor: 'lightgray'
  };

  return <h1 style={styles}>Hello, world!</h1>;
}

export default Greeting;
```

- 内联样式支持动态样式，可以根据 props 或状态有条件地应用样式。下面是根据 prop 条件应用样式的示例：

```jsx
function Greeting({ isImportant }) {

  const styles = {
    color: isImportant ? 'red' : 'black',
    fontSize: isImportant ? '24px' : '16px'
  };

  return <h1 style={styles}>Hello, world!</h1>;
}

export default Greeting;
```

- 在上述代码中，`color` 和 `fontSize` 样式根据 `isImportant` prop 条件设置。

## React 中的事件处理

- **合成事件系统**：这是 React 处理事件的方式。它作为原生事件（如 `click`、`keydown` 和 `submit` 事件）的包装器。React 中事件处理函数使用驼峰式命名（例如 `onClick`、`onSubmit` 等）。

下面是 React 中为 `onClick` 元素使用 `button` 属性的示例：

```jsx
function handleClick() {
  console.log("Button clicked!");
}

<button onClick={handleClick}>Click Me</button>;
```

在 React 中，事件处理函数通常以 `handle` 前缀开头，表示它们负责处理事件，如 `handleClick` 或 `handleSubmit`。

当用户操作触发事件时，React 会将一个合成事件对象传递给你的处理函数。该对象行为类似于原生 JavaScript 的 Event 对象，提供 `type`、`target` 和 `currentTarget` 等属性。

例如，为防止 `onSubmit` 事件触发浏览器刷新等默认行为，可以调用 `preventDefault()` 方法：

```jsx
function handleSubmit(event) {
  event.preventDefault();
  console.log("Form submitted!");
}

<form onSubmit={handleSubmit}>
  <input type="text" />
  <button>Submit</button>
</form>;
```

你也可以用箭头函数包裹处理函数，如下：

```jsx
function handleDelete(id) {
  console.log("Deleting item:", id);
}

<button onClick={() => handleDelete(1)}>Delete Item</button>;
```

## 使用状态和 `useState` Hook

- **状态定义**：在 React 中，状态是包含组件数据的对象。当状态更新时，组件会重新渲染。React 将状态视为不可变，意味着你不应直接修改它。
- **`useState()` Hook**：`useState` hook 是一个函数，允许你在函数组件中声明状态变量。基本语法如下：

```js
const [stateVariable, setStateFunction] = useState(initialValue);
```

状态变量包含：

- `stateVariable` 保存当前状态值
- `setStateFunction`（设置函数）更新状态变量
- `initialValue` 设置初始状态

下面是一个完整的 `Counter` 组件示例：

```jsx
import { useState } from "react";

function Counter() {
  const [count, setCount] = useState(0);

  return (
    <div>
      <h2>{count}</h2>

      <button onClick={() => setCount(count - 1)}>Decrement</button>
      <button onClick={() => setCount(count + 1)}>Increment</button>
    </div>
  );
}

export default Counter;
```

## 渲染和 React 组件

- **定义**：在 React 中，渲染是组件出现在用户界面（通常是浏览器）中的过程。渲染过程包括三个阶段：触发、渲染和提交。

触发阶段发生在 React 发现某些内容变化，用户界面可能需要更新时。通常是状态或 props 更新导致。

触发后，React 进入渲染阶段。此时 React 重新评估组件，确定显示内容。为此，React 使用一个轻量级的“真实”DOM 副本，称为虚拟 DOM。通过虚拟 DOM，React 能快速检查组件中需要更改的部分。

提交阶段是 React 将虚拟 DOM 中准备好的更改应用到真实 DOM 的过程。换句话说，这是你在屏幕上看到最终结果的阶段。

## 更新状态中的对象和数组

- **更新状态中的对象**：如果需要更新状态中的对象，应先创建新对象或复制现有对象，然后设置该新对象为状态。任何放入状态的对象都应视为只读。下面是设置用户姓名、年龄和城市的示例。`handleChange` 函数用于处理用户信息更新：

```jsx
import { useState } from "react";

function Profile() {
  const [user, setUser] = useState({ name: "John Doe", age: 31, city: "LA" });

  const handleChange = (e) => {
    const { name, value } = e.target;

    setUser((prevUser) => ({...prevUser, [name]: value}));
  };

  return (
    <div>
      <h1>User Profile</h1>
      <p>Name: {user.name}</p>
      <p>Age: {user.age}</p>
      <p>City: {user.city}</p>

      <h2>Update User Age </h2>
      <input type="number" name="age" value={user.age} onChange={handleChange} />

      <h2>Update User Name </h2>
      <input type="text" name="name" value={user.name} onChange={handleChange} />

      <h2>Update User City </h2>
      <input type="text" name="city" value={user.city} onChange={handleChange} />
    </div>
  );
}

export default Profile;
```

- **更新状态中的数组**：更新状态中的数组时，重要的是不要直接使用 `push()` 或 `pop()` 等方法修改数组。应在更新状态时创建新数组：

```jsx
const addItem = () => {
  const newItem = {
    id: items.length + 1,
    name: `Item ${items.length + 1}`,
  };

  // Creates a new array
  setItems((prevItems) => [...prevItems, newItem]);
};
```

如果想从数组中移除项，应使用 `filter()` 方法，该方法返回过滤后新的数组：

```jsx
const removeItem = (id) => {
  setItems((prevItems) => prevItems.filter((item) => item.id !== id));
};
```

## 使用 Refs 引用值

- **`ref` 属性**：你可以通过 React 的 `ref` 属性访问 DOM 节点。下面示例展示了使用 `ref` 来聚焦 `input` 元素。`current` 属性用于访问该 `ref` 的当前值：

```jsx
import { useRef } from "react";

const Focus = () => {
  const inputRef = useRef(null);

  const handleFocus = () => {
    if (inputRef.current) {
      inputRef.current.focus();
    }
  };

  return (
    <div>
      <input ref={inputRef} type="text" placeholder="Enter text" />
      <button onClick={handleFocus}>Focus Input</button>
    </div>
  );
};

export default Focus;
```

## 使用 `useEffect` Hook

- **`useEffect()` Hook**：在 React 中，副作用是指组件渲染过程之外发生的任何操作。即 React 不直接处理的 UI 渲染之外的操作。常见例子包括获取数据、更新浏览器标签标题、读写本地存储、获取用户位置等。这些操作与外部世界交互，称为副作用。React 提供 `useEffect` hook 让你处理这些副作用。`useEffect` 允许你在组件渲染或更新后运行函数。

```js
import { useEffect } from "react";

useEffect(() => {
 // Your side effect logic (usually a function) goes here
}, [dependencies]);
```

副作用函数在组件渲染后运行，可选的 `dependencies` 参数控制副作用运行时机。

注意，`dependencies` 可以是“响应值”数组（状态、props、函数、变量等）、空数组或完全省略。以下是这些选项如何控制 `useEffect` 的工作方式：

- 如果 `dependencies` 是包含一个或多个响应值的数组，副作用将在它们变化时运行。

- 如果 `dependencies` 是空数组，`useEffect` 只在组件首次渲染时运行一次。

- 如果省略 `dependencies`，副作用将在每次组件渲染或更新时运行。

## 如何创建自定义 Hooks

- **自定义 Hooks**：自定义 hook 允许你从组件中提取可复用逻辑，如数据获取、状态管理、切换和副作用（例如跟踪在线状态）。React 中所有内置 hook 都以 `use` 开头，因此你的自定义 hook 也应遵循此命名约定。

下面是创建 `useDebounce` hook 的示例：

```jsx
function useDebounce(value, delay) {
  const [debouncedValue, setDebouncedValue] = useState(value);

  useEffect(() => {
    const handler = setTimeout(() => {
      setDebouncedValue(value);
    }, delay);

    return () => {
      clearTimeout(handler);
    };
  }, [value, delay]);

  return debouncedValue;
}

export { useDebounce };
```

## React 中的表单处理

- **受控输入**：指将输入字段的值存储在状态中，并通过 `onChange` 事件更新。这让你完全控制表单数据，并允许即时验证和条件渲染。

```jsx
import { useState } from "react";

function App() {
  const [name, setName] = useState("");

  const handleChange = (e) => {
    setName(e.target.value);
  };

  const handleSubmit = (e) => {
    e.preventDefault();
    console.log(name);
  };

  return (
    <>
      <form onSubmit={handleSubmit}>
        <label htmlFor="name">Your name</label> <br />
        <input value={name} id="name" onChange={handleChange} type="text" />
        <button type="submit">Submit</button>
      </form>
    </>
  );
}

export default App;
```

- **非受控输入**：非受控输入不通过 `useState` hook 处理，而是由 HTML 的 DOM 维护其内部状态。由于 DOM 控制输入值，你需要使用 `ref` 来获取输入字段的值。

```jsx
import { useRef } from "react";

function App() {
 const nameRef = useRef();

 const handleSubmit = (e) => {
   e.preventDefault();
   console.log(nameRef.current.value);
 };

 return (
   <form onSubmit={handleSubmit}>
     <label htmlFor="name">Your</label>{" "}
     <input type="text" ref={nameRef} id="name" />
     <button type="submit">Submit</button>
   </form>
 );
}

export default App;
```

## 使用 `useActionState` Hook

- **服务器动作**：这些函数在服务器上运行，允许直接在服务器处理表单，无需 API 端点。下面是 Next.js 应用的示例：

```js
"use server";

async function submitForm(formData) {
 const name = formData.get("name");
 return { message: `Hello, ${name}!` };
}
```

`"use server"` 指令将函数标记为服务器动作。

- **`useActionState` Hook**：该 hook 根据表单提交结果更新状态。`useActionState` hook 的基本语法如下：

```js
const [state, action, isPending] = useActionState(actionFunction, initialState, permalink);
```

- `state` 是动作返回的当前状态。
- `action` 是触发服务器动作的函数。
- `isPending` 是布尔值，表示动作是否正在运行。
- `actionFunction` 参数是服务器动作本身。
- `initialState` 是动作运行前状态的起始点参数。
- `permalink` 是可选字符串，包含表单修改的唯一页面 URL。

## React 中的数据获取

- **获取数据的选项**：React 中有多种获取数据的方式。你可以使用原生 Fetch API，或第三方工具如 Axios 或 SWR。
- **获取数据时常用的状态变量**：无论选择哪种方式获取数据，都需要跟踪几个状态。第一个是数据本身。第二个跟踪数据是否仍在获取中。第三个是捕获获取过程中可能发生的任何错误的状态变量。

```js
const [data, setData] = useState(null);
const [loading, setLoading] = useState(true);
const [error, setError] = useState(null);
```

由于数据获取是副作用，最好在 `Fetch API` hook 中使用 `useEffect`。

```js
useEffect(() => {
  const fetchData = async () => {
    try {
      const res = await fetch("https://jsonplaceholder.typicode.com/posts");
      
      if (!res.ok) {
        throw new Error("Network response was not ok");
      }

      const data = await res.json();
      setData(data);
    } catch (err) {
      setError(err);
    } finally {
      setLoading(false);
    }
  };

  fetchData();
}, []);
```

然后你可以在数据未获取完成时渲染加载信息，获取出错时渲染错误信息，或渲染结果。

```jsx
if (loading) {
  return <p>Loading...</p>;
}

if (error) {
  return <p>{error.message}</p>;
}

return (
  <ul>
    {data.map((post) => (
```

## 使用 `useOptimistic` Hook

- **`useOptimistic` Hook**：该 hook 用于在后台等待异步操作完成时保持 UI 响应。它帮助管理 UI 中的“乐观更新”，即基于预期结果立即更新 UI 的策略，比如等待服务器响应。

基本语法如下：

```js
const [optimisticState, addOptimistic] = useOptimistic(actualState, updateFunction);
```

- `optimisticState` 是临时状态，立即更新以提升用户体验。
- `addOptimistic` 是在实际状态变化前应用乐观更新的函数。
- `actualState` 是来自操作结果（如服务器数据）的真实状态值。
- `updateFunction` 是调用时决定乐观状态如何更新的函数。

下面是 `useOptimistic` hook 在 `TaskList` 组件中的示例：

```js
"use client";

import { useOptimistic } from "react";

export default function TaskList({ tasks, addTask }) {
  const [optimisticTasks, addOptimisticTask] = useOptimistic(
    tasks,
    (state, newTask) => [...state, { text: newTask, pending: true }]
  );

  async function handleSubmit(e) {
    e.preventDefault();
    const formData = new FormData(e.target);

    addOptimisticTask(formData.get("task"));

    addTask(formData);
    e.target.reset();
  }

  return <>{/* UI */}</>;
}
```

## React Router

- **介绍**：React Router 是一个第三方库，允许你为 React 应用添加路由。首先，你需要在已有 React 项目中安装 React Router，如下：

```bash
npm i react-router
```

然后在 `main.jsx` 或 `index.jsx` 文件中设置路由结构，如下：

```jsx
import { StrictMode } from "react";
import { createRoot } from "react-dom/client";
import { BrowserRouter, Routes, Route } from "react-router";
import App from "./App.jsx";

import "./index.css";

createRoot(document.getElementById("root")).render(
  <StrictMode>
    <BrowserRouter>
      <Routes>
        <Route path="/" element={<App />} />
      </Routes>
    </BrowserRouter>
  </StrictMode>
);
```

`path` 和 `element` 用于将 URL 和 UI 组件绑定在一起。在此示例中，我们为首页设置了指向 `App` 组件的路由。

- **多视图和路由设置**：在大型应用中，常见多视图和路由设置如下：

```jsx
<Routes>
  <Route index element={<Home />} />
  <Route path="about" element={<About />} />

  <Route path="products">
    <Route index element={<ProductsHome />} />
    <Route path=":category" element={<Category />} />
    <Route path=":category/:productId" element={<ProductDetail />} />
    <Route path="trending" element={<Trending />} />
  </Route>
</Routes>
```

这些示例中的 `index` prop 表示给定路径段的默认路由。因此，`Home` 组件将在根路径 `/` 显示，而 `ProductsHome` 组件将在 `/products` 路径显示。

- **嵌套路由**：你可以在路由内嵌套其他路由，子路由路径会追加到父路由路径后面。

```jsx
<Route path="products">
  <Route path="trending" element={<Trending />} />
</Route>
```

上述示例中，热门产品的路径为 products/trending。

- **动态段**：动态段指 URL 路径的任意部分是动态的。

```jsx
<Route path=":category" element={<Category />} />
```

此示例中有一个名为 `category` 的动态段。当用户访问如 products/brass-instruments 的 URL 时，视图会切换到 `Category` 组件，你可以基于该段动态获取相应数据。

- **`useParams` Hook**：该 hook 用于访问 URL 路径中的动态参数。

```js
import { useParams } from "react-router";

export default function Category() {
  let params = useParams();
   {/* Accessing the category param: params.category */}
   {/* rest of code goes here */}
}
```

## 状态管理

- **Context API**：Context 指父组件无需通过 props 显式传递信息即可让子组件访问。`createContext` 用于创建表示其他组件将读取的上下文对象。`Provider` 用于向子组件提供上下文值。

```js

  );
```

- **Redux**：Redux 通过提供中央存储和严格控制状态更新来管理状态。它使用包含动作、reducers 和中间件的可预测模式。动作是将数据从应用发送到 Redux 存储的载荷，通常由用户交互触发。Reducers 是指定状态如何响应动作变化的函数，确保状态以不可变方式更新。中间件则作为动作分派和 reducer 之间的桥梁，允许扩展 Redux 功能（如日志记录、处理异步操作），而无需修改核心流程。

## 真实性能与感知性能的区别

- **感知性能**：指用户对网站性能的感受，包括响应速度和可靠性。这是主观测量，难以量化，但非常重要，因为用户体验决定网站的成功与否。
- **真实性能**：指网站的客观可测性能。通过页面加载时间、服务器响应时间和渲染时间等指标衡量。这些指标受网络和代码本身多种因素影响。

## 提升感知性能的技巧

- **惰性加载**：通过后台加载非关键资源，尽可能减少初始加载时间。
- **最小化字体延迟**：如果网站使用自定义字体，应尽量减少字体加载延迟，避免字体闪烁或显示回退字体。建议使用与自定义字体相似的回退字体，使变化更自然。
- **使用加载指示器**：用户点击元素后立即显示加载指示器，有助于用户感知连接和参与，缩短等待感。

## 核心性能概念

- **源代码顺序**：指 HTML 元素在文档中的结构顺序，决定了加载优先级，显著影响性能和无障碍。

源代码顺序的最佳实践包括：

- 将关键内容（如标题、导航或主要文本）放在 HTML 结构的较高位置。
- 推迟非关键脚本（如分析或第三方小部件），避免阻塞渲染。
- 使用渐进增强，确保核心体验在样式和脚本加载前可用。渐进增强是基于先用 HTML 使页面可用的理念构建网站和应用。

下面是符合上述最佳实践的良好源代码顺序示例：

```html
<h1>Welcome to FastSite!</h1>
<p>Critical information loads first.</p>
<script src="slow-script.js" defer></script>
```

- **关键渲染路径**：浏览器将代码转换为屏幕像素的步骤序列。
- **延迟**：请求在浏览器和服务器之间传输所需时间。高延迟意味着页面加载慢。

减少延迟的方法包括：

- 使用 CDN（内容分发网络）从更近位置提供文件。
- 启用压缩，如 Gzip，减少文件大小。
- 优化图像并使用惰性加载。

```html
<img src="placeholder.jpg" data-src="real-image.jpg" loading="lazy">
```

## 提升 INP

- **定义**：INP（Interaction to Next Paint）通过测量用户交互（如点击或按键）到浏览器下一次更新显示的时间，评估页面整体响应性。INP 越低，页面响应越快。

提升 INP 的方法：

- 通过拆分长时间 JavaScript 任务减少主线程工作。
- 对非关键脚本使用 `requestIdleCallback()`，该方法会在浏览器空闲时调用函数。
- 推迟或惰性加载前面提到的重资产。
- 优化事件处理器。如果处理器运行过于频繁或执行重操作，会降低页面速度并增加 INP。解决方案是防抖。防抖确保函数仅在用户停止输入短暂延迟后运行（例如 300 毫秒），避免不必要计算，提升性能。

## 浏览器中的渲染工作原理

- **渲染工作原理**：浏览器首先解析 HTML 并构建 DOM。接着处理 CSS，构建 CSS 对象模型（CSSOM），这是另一个树结构，决定元素样式。最后，浏览器将像素绘制到屏幕，根据计算的样式和布局渲染每个元素。在复杂页面中，可能涉及多个图层合成形成最终视觉输出。

## 性能对可持续性的影响

- **背景信息**：互联网约占全球碳排放的 2%，与航空业相当！每传输一个字节都需要电力，从数据中心到用户设备。较大的文件和低效脚本意味着更多能耗。高性能网站不仅更快，还减少不必要的处理和能源使用。

## 减少页面加载时间的方法

- **优化媒体资源**：大图像和视频是加载缓慢的常见原因。通过优化这些资源，可以显著加快网站速度，包括压缩图像、使用现代格式如 WebP 和惰性加载资源。
- **利用浏览器缓存**：缓存允许浏览器本地存储网站部分内容，减少回访者加载时间。
- **压缩和缩小文件**：减小文件大小可加快下载速度，包括压缩传输文件和缩小 CSS、JavaScript 文件。

## 提升“可用时间”

- **定义**：“可用时间”是用户请求页面到能有效交互之间的时间间隔。提升“可用时间”可以通过惰性加载资源或最小化阻塞渲染的资源实现。

## 性能测量关键指标

- **首次内容绘制（FCP）**：衡量屏幕上第一个内容（文本或图像）出现的速度。良好 FCP 通常低于 1.8 秒，较差 FCP 超过 3 秒。可使用 Chrome DevTools 的性能标签页检查 FCP。
- **总阻塞时间**：显示主线程被重 JavaScript 任务阻塞的时长。总阻塞时间（TBT）高时，用户体验卡顿。提升 TBT 可拆分长任务并推迟非关键脚本。
- **跳出率**：指未进行交互即离开的访问者比例。跳出率高可能因页面加载过慢。
- **独立用户数**：跟踪访问网站的独立访客数量。可用 Google Analytics 查看跳出率和独立用户数，帮助监控指标并提升参与度。

## 常用性能测量工具

- **Chrome DevTools**：Chrome 内置工具，实时分析和调试性能。显示加载时间、CPU 使用率和渲染延迟。特别适合测量首次内容绘制（FCP），帮助发现性能瓶颈。
- **Lighthouse**：自动化工具，检查性能、SEO 和无障碍。
- **WebPageTest**：测试网站从不同地点和设备的加载情况，详细分解速度指数、总阻塞时间等关键指标。想了解全球真实用户体验，WebPageTest 是理想工具。
- **PageSpeed Insights**：分析网站并给出移动端和桌面端的快速改进建议。指出性能瓶颈，提供优化图像、移除阻塞渲染脚本、减少服务器响应时间等具体建议。PageSpeed Insights 是快速检查 Google 视角下网站性能的便捷工具。
- **真实用户监控（RUM）**：跟踪真实用户行为，展示真实访客体验。流行 RUM 工具包括 Google Analytics（跟踪页面加载时间和跳出率）、New Relic 和 Datadog（监控实时性能问题）。想获取真实用户数据，RUM 工具必不可少。

## 使用性能 Web API

- **定义**：性能 Web API 允许开发者在代码中直接跟踪网页加载和响应效率。它们可测量页面加载时间、渲染和交互延迟，以及 JavaScript 执行时间。
- **`performance.now()`**：该 API 提供高精度时间戳（毫秒级），用于测量网站不同部分加载时长。

```js
const start = performance.now();  
// Run some code here  
const end = performance.now();  

console.log(`Execution time: ${end - start}ms`);
```

- **性能计时 API**：该 API 细分页面加载的每个阶段，从 DNS 查询到 `DOMContentLoaded`。

```js
const timing = performance.timing;  

const pageLoadTime = timing.loadEventEnd - timing.navigationStart;  
console.log(`Page load time: ${pageLoadTime}ms`);
```

- **`PerformanceObserver`**：该 API 监听性能事件，如布局偏移、长任务和用户交互。

```js
const observer = new PerformanceObserver((list) => {  
  list.getEntries().forEach((entry) => {  
    console.log(`Long task detected: ${entry.duration}ms`);  
  });  
});  

observer.observe({ type: "longtask", buffered: true });
```

## 提升 CSS 性能的技巧

- **CSS 动画**：动画某些 CSS 属性（如尺寸、位置和布局）会触发“重排”过程，浏览器重新计算元素位置和几何形状。这需要重绘，计算开销大。因此，建议尽量减少 CSS 动画数量，或至少提供开关让用户选择开启或关闭。

## 提升 JavaScript 性能的技巧

- **代码拆分**：将 JavaScript 代码拆分为执行关键和非关键任务的模块，有助于尽快预加载关键模块，推迟非关键模块渲染页面。
- **DOM 操作**：DOM 操作指通过 JavaScript 动态修改页面内容，操作文档对象模型（DOM）。DOM 操作计算开销大。减少 JavaScript 代码中的 DOM 操作量可提升性能。

## CSS 框架

- **CSS 框架**：CSS 框架能加快工作流程，创建网站统一视觉风格，确保设计在多个浏览器间一致，并保持 CSS 代码更有序。
- **流行 CSS 框架**：一些流行的 CSS 框架包括 Tailwind CSS、Bootstrap、Materialize 和 Foundation。
- **潜在缺点**：
  - 框架提供的 CSS 可能与自定义 CSS 冲突。
  - 你的网站可能看起来与使用相同框架的其他网站相似。
  - 大型框架可能导致性能问题。

## 两种类型的 CSS 框架

- **实用优先 CSS 框架**：这类框架包含具有特定用途的小类，如设置边距、内边距或背景色。你可以根据需要直接将这些小类分配给 HTML 元素。Tailwind CSS 属于实用优先 CSS 框架。

下面是使用 Tailwind CSS 样式按钮的示例。

```html
<button class="bg-blue-500 text-white font-bold py-2 px-4 rounded-full hover:bg-blue-700">
  Button
</button>
```

- **基于组件的 CSS 框架**：这类框架提供预构建组件和预定义样式，你可以添加到网站中。组件在 CSS 框架官方文档中提供，你可以复制粘贴到项目中。Bootstrap 属于基于组件的 CSS 框架。

下面是使用 Bootstrap 创建列表组的示例。你将添加整个组件，包括 HTML 结构，而非给 HTML 元素应用小类。

```html
<div class="card" style="width: 25rem;">
  <ul class="list-group list-group-flush">
    <li class="list-group-item">HTML</li>
    <li class="list-group-item">CSS</li>
    <li class="list-group-item">JavaScript</li>
  </ul>
</div>
```

## Tailwind CSS

Tailwind 是实用优先 CSS 框架。你无需编写自定义 CSS 规则，而是通过在 HTML 中组合小实用类构建设计。

### 响应式设计工具类

Tailwind 使用前缀如 `sm:`、`md:` 和 `lg:`，在不同屏幕尺寸应用样式。

```html
  Responsive layout
</div>
```

### Flexbox 工具类

类如 `flex`、`flex-col`、`justify-around` 和 `items-center` 让创建灵活布局变得简单。

```html

  <p>Row on medium and larger screens</p>
```

### 网格工具类

Tailwind 包含 CSS 网格工具类，如 `grid`、`grid-cols-1` 和 `md:grid-cols-3`。

```html
  <div class="bg-gray-100 p-4">Column 3</div>
```

### 间距工具类

工具类如 `mt-8`、`mx-auto`、`p-4` 和 `gap-4` 帮助创建一致间距，无需编写 CSS。

```html

</div>
```

### 排版工具类

工具类如 `uppercase`、`font-bold`、`font-semibold` 和 `text-4xl` 控制文本外观。

### 颜色和悬停状态

Tailwind 提供丰富颜色调色板，如 `text-red-700`、`bg-indigo-600` 和 `bg-gray-100`。

类如 `hover:bg-pink-600` 让交互效果简单实现。

### 边框、环和效果

- **边框**：`border-2 border-red-300` 添加指定厚度和颜色的边框。
- **环**：`ring-1 ring-gray-300` 创建类似轮廓的效果，常用于聚焦或卡片。
- **圆角和缩放**：类如 `rounded-md`、`rounded-xl` 和 `scale-105` 增加细节润色。

### 渐变

Tailwind 支持渐变工具类，如 `bg-gradient-to-r from-fuchsia-500 to-indigo-600`。

```html

  Gradient background

```

## CSS 预处理器

- **CSS 预处理器**：CSS 预处理器是扩展标准 CSS 的工具。它将带有扩展语法的代码编译成原生 CSS 文件。对复杂项目编写更简洁、可复用、减少重复且可扩展的 CSS 很有帮助。
- **功能**：CSS 预处理器提供变量、混入、嵌套和选择器继承等功能。
- **流行 CSS 预处理器**：流行的 CSS 预处理器包括 Sass、Less 和 Stylus。
- **潜在缺点**：
  - 编译 CSS 规则为标准 CSS 可能带来额外开销。
  - 编译后的代码可能难以调试。

- **Sass**：最流行的 CSS 预处理器之一。Sass 代表“语法优美的样式表”。
- **Sass 支持的功能**：支持变量、嵌套 CSS 规则、模块、混入、继承和基本数学运算符。

```sass
$primary-color: #3498eb
```

### 混入

- **混入**：混入允许你将多个 CSS 属性及其值组合在一个名称下，并在整个样式表中复用该 CSS 代码块。

下面是 SCSS 语法中定义混入的示例。此混入名为 `center-flex`，包含三个使用 flexbox 居中元素的 CSS 属性。

## 手动和自动测试

- **手动测试**：测试人员手动检查应用的每个部分，测试不同功能确保其正常工作。如果发现错误，会反馈给软件团队修复。
- **自动测试**：通过编写单独程序自动检查应用是否按预期运行。

## 单元测试

- **单元测试**：测试每个函数，确保其按预期工作。单元测试也可作为应用文档，表示代码的预期行为。
- **单一职责原则**：建议保持每个函数小且只负责一件事。
- **常见 JavaScript 测试框架**：包括 Jest、Mocha 和 Vitest。Jest 是流行的单元测试框架。

下面是使用 Jest 的单元测试示例。

首先，创建一个负责返回新格式字符串的函数：

```javascript
export function getFormattedWord(str) {
  if (!str) return '';
  return str.charAt(0).toUpperCase() + str.slice(1);
}
```

在单独的 `getFormattedWord.test.js` 文件中，编写测试验证函数是否按预期工作。`getFormattedWord.test.js` 文件内容如下：

```javascript
import { getFormattedWord } from "./getFormattedWord.js";
import { test, expect } from "jest";

test('capitalizes the first letter of a word', () => {
  expect(getFormattedWord('hello')).toBe('Hello');
});
```

- **`expect` 函数**：`expect` 函数用于测试一个值。
- **匹配器**：匹配器是检查值是否符合预期的函数。上例中的匹配器是 `toBe()`。Jest 提供多种匹配器。

使用 Jest 前，需通过 `jest` 安装 `npm i jest` 包。还需在 `package.json` 文件中添加脚本，如下：

```js
"scripts": {
  "test": "jest"
},
```

然后运行命令 `npm run test` 执行测试。

## 软件开发生命周期

- **软件开发生命周期的不同阶段**：
  - **规划阶段**：开发团队从利益相关者收集需求。
  - **设计阶段**：软件团队拆解需求，决定最佳解决方案。
  - **实现阶段**：软件团队将需求拆分为可管理任务并实现。
  - **测试阶段**：包括手动和自动测试新功能。有时团队在整个开发阶段测试应用，及时发现并修复问题。
  - **部署阶段**：团队将新变更部署到构建或测试环境。
  - **维护阶段**：修复生产应用中客户反馈的问题。

- **软件开发生命周期的不同模型**：
  - **瀑布模型**：生命周期每个阶段需完成后才能进入下一阶段。
  - **敏捷模型**：关注迭代开发，将工作拆分为冲刺。

## BDD 和 TDD

- **TDD**：测试驱动开发强调先写测试。先写测试再开发功能，能在开发过程中实时反馈。
- **BDD**：行为驱动开发将测试场景与业务目标对齐。BDD 测试场景应使用技术和非技术人员都能理解的语言。Gherkin 是此类语法示例。
- **BDD 测试框架**：包括 Cucumber、JBehave 和 SpecFlow。

## 单元测试中的断言

- **断言**：用于测试代码是否按预期运行。
- **断言库**：Chai 是常用断言库。其他常见 JavaScript 断言库有 `should.js` 和 `expect.js`。

下面是使用 Chai 的断言示例，检查 `addThreeAndFour` 函数返回值是否等于数字 7：

```js
assert.equal(addThreeAndFour(), 7);
```

- **最佳实践**：无论使用哪个断言库，都应编写清晰的断言和失败信息，帮助理解测试失败原因。

## 模拟、伪造和存根

- **模拟**：用假数据替代真实数据，模拟真实组件行为。例如，在测试中模拟 API 响应，避免持续调用 API 获取数据。
- **存根**：存根是返回预定义响应或虚拟数据的对象，用于模拟应用中的预期行为。例如，在测试中存根数据库连接行为，无需依赖真实数据库。
- **伪造**：伪造是简化的真实组件版本，无复杂性或副作用。例如，伪造数据库通过内存存储数据，模拟数据库操作，比真实数据库操作更快。

## 功能测试

- **功能测试**：检查应用功能是否按预期工作。目标是针对多种场景测试系统整体。
- **非功能测试**：关注性能和可靠性等方面。
- **冒烟测试**：在开始更广泛测试前，对系统进行基本或关键问题的初步检查。

## 端到端测试

- **端到端测试**：简称 E2E，从用户视角测试真实场景。确保应用行为正确且用户可预测。但设置、设计和维护耗时。

## 可用性测试

- **可用性测试**：让真实用户与应用交互，发现设计、用户体验或功能问题。关注用户对应用的直观感受。
- **四种常见可用性测试类型**：
  - **探索性**：用户交互应用不同功能，了解其工作方式。
  - **比较性**：将应用用户体验与市场上类似应用比较。
  - **评估性**：研究应用使用的直观性。
  - **验证性**：识别阻碍用户有效使用应用的重大问题。
- **可用性测试工具**：包括 Loop11、Maze、Userbrain、UserTesting 和 UXTweak。

## 兼容性测试

- **兼容性测试**：确保应用在不同计算环境中正常工作。
- **不同类型兼容性测试**：
  - **向后兼容**：软件兼容早期版本。
  - **向前兼容**：软件和系统能兼容未来版本。
  - **硬件兼容**：软件能在不同硬件配置下正常工作。
  - **操作系统兼容**：软件能在不同操作系统（如 macOS、Windows、Ubuntu 和 Fedora 等 Linux 发行版）上运行。
  - **网络兼容**：软件能适应不同网络条件，如不同速度、协议、安全设置等。
  - **浏览器兼容**：网络应用能在不同浏览器（如 Chrome、Safari、Firefox 等）中一致运行。
  - **移动兼容**：确保软件能在各种 Android 和 iOS 设备（手机和平板）上运行。

## 性能测试

- **性能测试**：测试应用在不同负载下的速度、响应性、可扩展性和稳定性。目标是解决性能瓶颈。
- **不同类型性能测试**：
  - **负载测试**：确定系统在正常和峰值负载下的表现。
  - **压力测试**：在极端负载下测试应用，观察系统响应。
  - **浸泡测试（耐力测试）**：长时间高负载测试系统。
  - **突发测试**：剧烈增加和减少负载，分析系统反应。
  - **断点测试（容量测试）**：逐步增加负载，直到系统开始失败或性能下降。

## 安全测试

- **安全测试**：识别漏洞和弱点。
- **安全原则**：
  - **机密性**：防止敏感信息泄露给非预期接收者。
  - **完整性**：防止恶意用户篡改用户信息。
  - **认证**：验证用户身份，确保其有权使用系统。
  - **授权**：确定认证用户允许执行的操作或访问的系统部分。
  - **可用性**：确保授权用户在需要时能访问信息和服务。
  - **不可否认性**：确保发送者和接收者都有交付和身份验证的证据，防止发送者否认发送信息。
- **常见安全威胁**：
  - **跨站脚本攻击（XSS）**：攻击者向网页注入恶意脚本，在受害者浏览器上下文中执行。
  - **SQL 注入**：恶意用户向数据库注入恶意代码。
  - **拒绝服务攻击（DoS）**：恶意用户通过大量请求或流量淹没网站，导致服务器变慢或崩溃，使网站无法访问。
- **安全测试工具类别**：
  - **静态应用安全测试**：评估应用源代码，识别安全漏洞。
  - **动态应用安全测试**：与应用前端交互，发现潜在安全弱点。DAST 工具无法访问源代码。
- **渗透测试（pentest）**：模拟网络攻击，识别系统漏洞的安全测试。

## A/B 测试

- **A/B 测试**：比较页面或应用的两个版本，研究哪个表现更好。也称为桶测试或分割测试。A/B 测试帮助你做出更多数据驱动的决策，持续改进用户体验。
- **A/B 测试工具**：包括 GrowthBook 和 LaunchDarkly。

## Alpha 和 Beta 测试

初步开发和软件测试完成后，重要的是让测试人员和真实用户测试应用。这就是 Alpha 和 Beta 测试的作用。

- **Alpha 测试**：由选定测试人员进行，确保发布前无错误。Alpha 测试属于验收测试，使用白盒和黑盒测试技术。
- **Beta 测试**：应用向真实用户开放，用户可交互并提供反馈。Beta 测试也是用户验收测试的一种。
- **验收测试**：确保软件满足业务需求和用户需求，方可发布。
- **黑盒测试**：只关注应用的预期行为。
- **白盒测试**：测试人员了解内部组件并对其进行测试。

## 回归测试

- **回归**：指新变更无意中破坏了现有功能。
- **回归测试**：帮助发现回归问题。通过重新运行功能测试，确保应用各部分仍按预期工作。
- **回归测试工具**：包括 Puppeteer、Playwright、Selenium 和 Cypress。
- **回归测试技术**：
  - **单元回归测试**：每次重大变更或修复后，测试需覆盖的项目列表。
  - **部分回归测试**：针对特定方面的有针对性测试，确保新变更未破坏。
  - **完整回归测试**：对代码库所有功能运行测试，最耗时且详尽。
- **重测**：检查已知问题是否解决。与回归测试不同，回归测试寻找因代码变更引入的未知问题。

## 什么是 TypeScript

- **JavaScript**：JavaScript 是动态类型语言，变量运行时可接收任意值。动态类型语言的挑战是缺乏类型安全，可能引入错误。

例如，即使你的 JavaScript 函数期望一个数组，也可以传入数字：

```javascript
const getRandomValue = (array) => {
  return array[Math.floor(Math.random() * array.length)];
}

console.log(getRandomValue(10));
```

上述示例的 `console` 输出将是 `undefined`。

- **TypeScript**：TypeScript 扩展了 JavaScript 语言，加入静态类型，有助于在运行代码前捕获类型不匹配错误。

例如，你可以为 `array` 参数定义类型：

```typescript
const getRandomValue = (array: string[]) => {
  return array[Math.floor(Math.random() * array.length)];
}
```

该类型定义告诉 TypeScript，`array` 参数必须是字符串数组。然后，当你调用 `getRandomValue` 并传入数字时，会出现编译错误。

- **编译器**：你需要先将 TypeScript 代码编译成普通 JavaScript。编译器会评估代码，发现类型不匹配时抛出错误。

## TypeScript 中的数据类型

- **TypeScript 中的原始数据类型**：对于原始数据类型 `string`、`null`、`undefined`、`number`、`boolean` 和 `bigint`，TypeScript 提供对应的类型关键字。

```typescript
let str: string = "Naomi";
let num: number = 42;
let bool: boolean = true;
let nope: null = null;
let nada: undefined = undefined;
```

- **数组**：你可以用两种语法定义特定类型的数组。

```typescript
const arrOne: string[] = ["Naomi"];
const arrTwo: Array<string> = ["Camperchan"];
```

- **对象**：你可以定义对象的确切结构。

```typescript
const obj: { a: string, b: number } = { a: "Naomi", b: 42 };
```

如果你想定义一个键为任意字符串，但所有值必须是字符串的对象，有两种定义方式：

```typescript
const objOne: Record<string, string> = {};
const objTwo: { [key: string]: string } = {};
```

- **TypeScript 中的其他有用类型**：
  - **`any`**：`any` 表示值可以是任意类型。告诉编译器停止关心该变量的类型。
  - **`unknown`**：`unknown` 告诉 TypeScript 你确实关心值的类型，但实际上不知道它是什么。`unknown` 通常优于 `any`。
  - **`void`**：特殊类型，通常用于定义函数。无返回值的函数使用 `void` 作为返回类型。
  - **`never`**：表示永远不存在的类型。

- **`type` 关键字**：类似 `const`，但用于声明类型而非变量。

用于声明自定义类型，如联合类型或只包含特定值的类型：

```typescript
type stringOrNumber = string | number;
type bot = "camperchan" | "camperbot" | "naomi";
```

- **`interface`**：接口类似于类型的类。它们可以实现或扩展其他接口，专门用于对象类型，通常优先使用，除非需要 `type` 声明提供的特定功能。

```typescript
interface wowie {
  zowie: boolean;
  method: () => void;
}
```

- **定义返回类型**：你也可以定义函数的返回类型。

下面示例定义返回值为字符串。如果尝试返回其他类型，TypeScript 会报编译错误。

```typescript
const getRandomValue = (array: string[]): string => {
  return array[Math.floor(Math.random() * array.length)];
}
```

## 泛型

- **定义泛型类型**：你可以定义泛型类型，并在函数中引用。它相当于为函数提供的特殊参数，用于控制函数类型定义的行为。

下面是定义泛型函数的示例：

```typescript
const getRandomValue = <T>(array: T[]): T => {
  return array[Math.floor(Math.random() * array.length)];
}
const val = getRandomValue([1, 2, 4])
```

`<T>` 语法告诉 TypeScript 你正在为函数定义泛型类型 `T`。`T` 是泛型类型的常用名称，但你可以使用任意名称。

然后告诉 TypeScript，`array` 参数是泛型类型数组，返回值是该类型的单个元素。

- **在函数调用中指定类型参数**：你可以在函数名和参数之间使用尖括号传递类型参数。

下面是传递类型参数的示例：

```typescript
const email = document.querySelector<HTMLInputElement>("#email");
console.log(email.value);
```

这告诉 TypeScript 你期望找到的元素是输入元素。

## 类型缩小

- **通过真值缩小**：下面示例中，尝试访问 `value` 属性的 `email` 会报编译错误，因为 `email` 可能是 `null`。

```typescript
const email = document.querySelector<HTMLInputElement>("#email");
console.log(email.value);
```

你可以使用条件语句确认 `email` 为真值后再访问属性：

```typescript
const email = document.querySelector<HTMLInputElement>("#email");
if (email) {
  console.log(email.value);
}
```

真值检查也可反向使用：

```typescript
const email = document.querySelector<HTMLInputElement>("#email");
if (!email) {
  throw new ReferenceError("Could not find email element!")
}
console.log(email.value);
```

抛出错误会结束代码逻辑执行，这意味着当你到达 `console.log()` 调用时，TypeScript 知道 `email` 不可能是 `null`。

- **可选链**：可选链 `?.` 也是类型缩小的一种形式，前提是如果 `email` 值是 `null`，则不会访问属性。

```typescript
const email = document.querySelector<HTMLInputElement>("#email");
console.log(email?.value);
```

- **`typeof` 操作符**：你可以使用 `typeof` 操作符的条件判断变量类型。

```typescript
const myVal = Math.random() > 0.5 ? 222 : "222";
if (typeof myVal === "number") {
  console.log(myVal / 10);
}
```

- **`instanceof` 关键字**：如果对象来自类，可以使用 `instanceof` 关键字缩小类型。

```typescript
const email = document.querySelector("#email");

if (email instanceof HTMLInputElement) {
    console.log(email.value);
}
```

- **类型断言**：当 TypeScript 无法自动确定值类型（如下面示例中的 `request.json()` 方法结果）时，会报编译错误。解决方法之一是类型断言，但这会削弱 TypeScript 捕获潜在错误的能力。

```typescript
interface User {
    name: string;
    age: number;
}

const printAge = (user: User) => 
  console.log(`${user.name} is ${user.age} years old!`)

const request = await fetch("url")
const myUser = await request.json() as User;
printAge(myUser);
```

- **类型保护**：你也可以编写类型保护：

```typescript
interface User {
    name: string;
    age: number;
}

const isValidUser = (user: unknown): user is User => {
  return !!user && 
    typeof user === "object" &&
    "name" in user &&
    "age" in user;
}
```

`user is User` 语法表示函数返回布尔值，返回 true 时表示 `user` 值满足 `User` 接口。

## `tsconfig` 文件

- **`tsconfig.json`**：TypeScript 的编译器设置存放在项目根目录的 `tsconfig.json` 文件中。

```json
{
  "compilerOptions": {
    "rootDir": "./src",
    "outDir": "./prod",
    "lib": ["ES2023"],
    "target": "ES2023",
    "module": "ES2022",
    "moduleResolution": "Node",
    "esModuleInterop": true,
    "skipLibCheck": true,
    "strict": true
  },
  "exclude": ["test/"]
}
```

以下是示例中使用的编译器选项说明：

- **`compilerOptions`**：`compilerOptions` 属性控制 TypeScript 编译器行为。
- **`rootDir` 和 `outDir`**：`rootDir` 和 `outDir` 指定源代码目录和转译后 JavaScript 代码目录。
- **`lib`**：`lib` 属性决定编译器使用哪些类型定义，支持特定 ES 版本、DOM 等。
- **`module` 和 `moduleResolution`**：`module` 和 `moduleResolution` 协同管理包使用的模块类型——CommonJS 或 ECMAScript。
- **`esModuleInterop`**：`esModuleInterop` 允许 CommonJS 和 ES 模块更顺畅互操作，自动为导入创建命名空间对象。
- **`skipLibCheck`**：`skipLibCheck` 选项跳过验证未被代码导入引用的 `.d.ts` 文件。
- **`strict`**：`strict` 标志启用多项检查，如确保正确处理可空类型，警告 TypeScript 回退到 `any`。
- **`exclude`**：顶层 `exclude` 属性告诉编译器在编译时忽略这些 TypeScript 文件。
