# --explanation--

`Initial unit tests` são o primeiro passo nos testes. Desenvolvedores verificam pequenas partes do código (`units`) para garantir que elas façam o que deveriam. Por exemplo:

`We wrote initial unit tests to check the main functions.` — A equipe criou testes iniciais para garantir que as partes mais importantes do código funcionassem corretamente.

`Quality assurance (QA) checks` são testes finais realizados pelas equipes de QA para garantir que o software seja confiável, fácil de usar e pronto para o lançamento. Por exemplo:

`Regular QA checks help prevent bugs from reaching users.` — Testes frequentes pela equipe de QA ajudam a detectar erros cedo, para que os usuários não enfrentem problemas.

Alice está descrevendo como o orçamento de testes cobre todas as etapas, desde as verificações iniciais de codificação até os testes finais antes do lançamento.
