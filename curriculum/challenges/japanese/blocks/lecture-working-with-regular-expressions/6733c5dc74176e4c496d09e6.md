# --interactive--

先読みアサーションと後読みアサーションは、周囲のパターンの有無に基づいて特定のパターンをマッチさせることができます。これらのアサーションには4つのバリエーションがあります。

まずは正の先読みアサーションです。このアサーションは、あるパターンの後に別のパターンが続く場合にマッチします。

正の先読みを作成するには、まずマッチさせたいパターンから始めます。次に、条件として使いたいパターンを丸括弧で囲みます。開き丸括弧の後に`?=`を使って、そのパターンを正の先読みとして定義します。

このパターンは、`free`が続く場合にのみ`code`という単語にマッチします：

```js
const regex = /free(?=code)/i;
```

パターンの動作をテストしてみましょう：

:::interactive_editor

```js
const regex = /free(?=code)/i;
console.log(regex.test("freeCodeCamp")); // true
console.log(regex.test("free code camp")); // false
console.log(
  regex.test("I need someone for free to write code for me")
); // false
```

:::

`free`の直後に`code`が続く文字列だけがテストに合格することに注目してください。

では、`free`が`code`に続かない場合にマッチさせたい場合はどうでしょうか？正の先読みを否定先読みへ変えることで動作を反転できます。これを行うには、`?=`を`?!`に変更してください：

```js
const regex = /free(?!code)/i;
```

同じ文字列に対してこれをテストしてみましょう：

:::interactive_editor

```js
const regex = /free(?!code)/i;
console.log(regex.test("freeCodeCamp")); // false
console.log(regex.test("free code camp")); // true
console.log(
  regex.test("I need someone for free to write code for me")
); // true
```

:::

予想通り、結果は逆になります。唯一失敗するのは、`free`の直後に`code`が続く最初の文字列です。

後読みアサーションは先読みアサーションと似ていますが、後読みは後に続くパターンではなく、前にあるパターンに基づいて条件付きでマッチします。正の後読みを見てみましょう。

正の後読みは`?<=`で示され、`?=`とは異なります。`code`に続く場合に`free`にマッチする正規表現を作りましょう：

```js
const regex = /(?<=free)code/i;
```

正の先読みと同様に、`code`の直前に`free`があるため、最初の文字列にマッチします：

:::interactive_editor

```js
const regex = /(?<=free)code/i;
console.log(regex.test("freeCodeCamp")); // true
console.log(regex.test("free code camp")); // false
console.log(
  regex.test("I need someone for free to write code for me")
); // false
```

:::

`code`に続かない場合に`free`にマッチさせたい場合は、否定後読みを使います。否定後読みは`?<=`を`?<!`に置き換えて定義します：

```js
const regex = /(?<!free)code/i;
```

これは、`code`に直前に続かない`free`のすべての出現にマッチします：

:::interactive_editor

```js
const regex = /(?<!free)code/i;
console.log(regex.test("freeCodeCamp")); // false
console.log(regex.test("free code camp")); // true
console.log(
  regex.test("I need someone for free to write code for me")
); // true
```

:::

`Regex.prototype.test`は文字列が正規表現にマッチするかどうかを確認するだけであることを覚えておいてください。否定後読みを`String.prototype.match`で使って、アサーションがどのように影響するか見てみましょう：

```js
const regex = /(?<!free)code/i;
console.log("freeCodeCamp".match(regex)); // null
console.log("free code camp".match(regex)); // ['code', index: 5, input: 'free code camp', groups: undefined]
console.log(
  "I need someone for free to write code for me".match(regex)
); // ['code', index: 33, input: 'I need someone for free to write code for me', groups: undefined]
```

正規表現が`free`の存在を後読みでチェックしていても、`free`にはマッチしないことに注目してください。マッチに含まれるのは`code`だけです。

先読みと後読みは、マッチの返り値に影響を与えずに条件付きでテキストをマッチさせるのに非常に便利です。
