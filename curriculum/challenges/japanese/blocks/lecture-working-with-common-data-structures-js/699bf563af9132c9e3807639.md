# --description--

配列はコンピュータサイエンスにおける基本的なデータ構造です。すべての配列は順序付けられたデータの集合をストアしますが、その種類によって内部の動作が異なる場合があります。

その基盤となる動作はプログラムの効率に重要な影響を与えるため、動的配列と静的配列、それぞれの違いを学び、プログラムに最も効率的なものを選べるようにしましょう。

まずは静的配列から始めます。

**静的配列**はサイズが固定されています。要素は隣接したメモリ領域にストアされます。

静的配列のサイズは配列が初期化されるときに決まります。一度特定のメモリ領域が割り当てられると、それは固定され、プログラムの実行中に変更できません。これが静的配列の重要な特徴です。

要素を隣接したメモリ領域にストアすることで、プログラムは最初の要素の場所を記憶し、インデックスを使って簡単な計算を行い、他の要素をメモリ内で見つけられるため、データの取得が効率的になります。

このため、静的配列の値にアクセスする操作は定数時間 `O(1)` で行われ、とても効率的です。

格納する要素の数があらかじめわかっている場合は静的配列を使うことができます。また、アクセス操作が非常に効率的なので、値に頻繁にアクセスする場合にも役立ちます。

しかし、このデータ構造はサイズを増減できないため、格納する要素数が変動する可能性がある場合は動的配列を使うべきです。

静的配列のサイズを増やそうとすると、新しい配列を作成し、古い配列のすべての要素を新しい配列にコピーする必要があり、これは非効率です。その場合は、この処理を自動で行う動的配列のほうがずっと適しています。

JavaScriptにはCやJavaのような伝統的な低レベルの静的配列はありませんが、組み込みの `Array` オブジェクトを使って固定サイズの配列をシミュレートできます。

```js
let numbers = new Array(3); // fixed size of 3
numbers[0] = 10;
numbers[1] = 20;
numbers[2] = 30;
```

JavaScriptはサイズ変更が可能ですが、概念的にはこれは固定長配列を表しています。

JavaScriptの配列はデフォルトでは動的なので、次にそちらを見ていきましょう。

**動的配列**はプログラムの実行中に自動でサイズが増減できるため、より柔軟です。

動的配列は自動リサイズ機構を通じて動作し、元の配列がいっぱいになると要素を新しい配列にコピーします。この処理は、新しい配列のサイズが効率的に選ばれるため、計算コストの高い操作が頻繁にならないように効率的に行われます。

動的配列の要素にアクセスする操作は定数時間 `O(1)` で行われるため、とても効率的です。

配列の途中に要素を挿入する操作は線形時間 `O(n)` かかります。なぜなら、その後の要素を移動させる必要があるからです。

配列の末尾に要素を追加する操作は、動的配列にまだ空きがあれば定数時間 `O(1)` で行われますが、配列がいっぱいでリサイズが必要な場合は `O(n)` の計算量になります。

配列に格納する値の数があらかじめわからない場合は動的配列を使うべきです。また、頻繁に要素の挿入や削除を行う場合にも役立ちます。

JavaScriptで配列を作成するには、要素を大括弧の中にカンマで区切って書きます。

```js
let numbers = [3, 4, 5, 6];
```

これで4つの要素を持つ動的配列が作成されます。

配列の要素にアクセスするには、変数名の後にインデックスを含む大括弧を書きます。

インデックスは最初の要素が0から始まり、次の要素ごとに1ずつ増えます。

```js
numbers[0]; // 3
numbers[1]; // 4
numbers[2]; // 5
numbers[3]; // 6
```

値を更新するには、単に再代入すればいいです。

```js
numbers[2] = 16
```

これで配列は `[3, 4, 16, 6]` になります。

配列の末尾に要素を追加するには、`.push()` メソッドを使います。

```js
numbers.push(7)
```

これで配列は `[3, 4, 16, 6, 7]` になります。

特定のインデックスに要素を挿入する場合、JavaScriptにはPythonのような`.insert()`はありませんが、`.splice()` メソッドを使えます。構文は次のようになります。

```js
array.splice(index, deleteCount, newElement)
```

例を見てみましょう。

```js
numbers.splice(3, 0, 15);
```

この例は次の意味です。

- インデックス3から始める
- 0個の要素を削除する
- 15を挿入する

もし配列が `[3, 4, 16, 6, 7]` だった場合、今は `[3, 4, 16, 15, 6, 7]` になります。

同じ`.splice` メソッドで特定のインデックスの要素を削除できます。

```js
numbers.splice(2, 1);
```

これはインデックス2から始めて1つの要素を削除することを意味します。

最後の要素を削除するには、`.pop()` メソッドを使います。

```js
numbers.pop();
```

JavaScriptの配列には他にもたくさんの組み込みメソッドがあり、ドキュメントで確認できます。

これが動的配列の強みです。

一般的に、要素数があらかじめわかっていて頻繁にアクセスする場合は静的配列を使い、要素数が不明または変動する場合は動的配列を使うべきです。

静的配列のシンプルさと動的配列の柔軟性のトレードオフを常に考慮してください。どちらも特定のユースケースやシナリオで役立ちます。問題に応じて最適なものを選べることは、練習を重ねて身につく問題解決能力の一部です。
