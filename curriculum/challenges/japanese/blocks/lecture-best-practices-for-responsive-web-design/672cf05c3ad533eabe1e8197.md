# --interactive--

メディアクエリはレスポンシブウェブデザインの基本的なコンポーネントであり、開発者がユーザーのデバイスやブラウザの特性に基づいて異なるスタイルを適用できるようにします。コンテンツ自体を変更することなく、さまざまなデバイスに合わせて表示を調整する方法を提供します。

メディアクエリの基本は、メディアタイプと1つ以上の特定の条件をチェックする式で構成されます。これらの条件が真の場合、対応するスタイルが適用されます。この仕組みにより、異なる画面サイズ、解像度、デバイスの機能に適応するレスポンシブデザインを作成できます。

CSSにおけるメディアクエリの基本構文は次のようになります。

```css
@media mediatype and (feature: value) {
  /* CSS rules go here */
}
```

この構造では、mediatypeがクエリが適用されるメディアの種類を指定し、feature: valueの組み合わせがスタイルを適用するために満たす必要のある条件を定義します。

メディアタイプはデバイスの一般的なカテゴリを表します。最もよく使われるメディアタイプについて説明します：

- `all` はすべてのデバイスに適しています。メディアタイプが指定されていない場合のデフォルトです。

- `print` はページ分割された資料や印刷プレビューモードで画面に表示される文書向けです。

- `screen` は主にスクリーン向けです。

かつては`handheld`や`tv`のようなメディアタイプもありましたが、これらの多くは廃止され、より正確にデバイスをターゲットにするために機能を使うことが推奨されています。

メディア機能はユーザーエージェント、出力デバイス、または環境の特定の特性を表します。よく使われるメディア機能について説明します。`width`と`height`はビューポートの幅と高さを表し、範囲クエリには`min-`や`max-`の接頭辞とよく組み合わせて使われます。例を示します：

```css
@media screen and (min-width: 768px) {
  /* Styles for screens at least 768px wide */
}
```

`aspect-ratio`はビューポートの幅と高さの比率を表します。例を示します：

```css
@media screen and (aspect-ratio: 16/9) {
  /* Styles for screens with a 16:9 aspect ratio */
}
```

`orientation`機能はデバイスがランドスケープ（横向き）かポートレート（縦向き）かを示します。例を示します：

```css
@media screen and (orientation: landscape) {
  /* Styles for landscape orientation */
}
```

`resolution`機能は出力デバイスの解像度をdpi（ドット毎インチ）またはdpcm（ドット毎センチメートル）で表します。例を示します：

```css
@media screen and (min-resolution: 300dpi) {
  /* Styles for high-resolution screens */
}
```

`hover`機能は主要な入力機構が要素上にホバーできるかどうかをテストします。例を示します：

```css
@media (hover: hover) {
  /* Styles for devices that support hover */
}
```

`prefers-color-scheme`機能はユーザーがライトテーマかダークテーマを要求しているかを検出します。例を示します：

```css
@media (prefers-color-scheme: dark) {
  /* Styles for dark mode */
}
```

メディアクエリは論理演算子を使って複数の条件を組み合わせることもできます。`and`演算子は複数のメディア機能を組み合わせるために使われ、`not`や`only`はメディアクエリを否定したり単独にしたりするために使われます。複数の機能を組み合わせた例を示します：

```css
@media screen and (min-width: 768px) and (orientation: landscape) {
  /* Styles for landscape screens at least 768px wide */
}
```

また、カンマ区切りのリストで複数のクエリを指定することもでき、これは「または」演算子のように機能します：

```css
@media screen and (min-width: 768px), print {
  /* Styles for screens at least 768px wide OR for print */
}
```

メディアクエリを使う際はカスケードを考慮することが重要です。メディアクエリは特異性を高めるものではなく、条件付きルールをグループ化するだけです。CSSの通常のカスケードルールは各メディアクエリ内でも適用されます。

実際には、メディアクエリはレスポンシブレイアウトを作成するためによく使われます。一般的なパターンはモバイルデバイス向けの基本スタイルを定義し、メディアクエリで大きな画面向けにレイアウトを強化する方法です：

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<div class="container">
  <p>Lorem ipsum dolor sit amet, consectetur adipiscing elit. Integer nec odio. Praesent libero. Sed cursus ante dapibus diam.</p>
  <p>Sed nisi. Nulla quis sem at nibh elementum imperdiet. Duis sagittis ipsum. Praesent mauris. Fusce nec tellus sed augue semper porta.</p>
  <p>Mauris massa. Vestibulum lacinia arcu eget nulla. Class aptent taciti sociosqu ad litora torquent per conubia nostra, per inceptos himenaeos.</p>
</div>
```

```css
/* Base styles for mobile */
.container {
  width: 100%;
  padding: 15px;
}

/* Styles for tablets */
@media screen and (min-width: 768px) {
  .container {
    width: 750px;
    margin: 0 auto;
  }
}

/* Styles for desktops */
@media screen and (min-width: 1024px) {
  .container {
    width: 960px;
  }
}
```

:::

この方法は「モバイルファースト」レスポンシブデザインとして知られ、基本スタイルがモバイルデバイスに適していることを保証し、大きな画面向けに強化を加えます。

まとめると、メディアクエリはCSSでレスポンシブで適応性のあるウェブデザインを作成するための強力なツールです。異なるメディアタイプや機能の使い方を理解することで、開発者は幅広いデバイスやユーザーの好みに最適なユーザー体験を提供するウェブサイトを作成できます。ウェブ技術が進化し続ける中で、新しいメディア機能を把握しておくことは、より細やかで使いやすいレスポンシブデザインを作るのに役立ちます。
