# --description--

У Node.js немає вбудованого WebSocket-сервера. Розробники зазвичай використовують зовнішні пакети, такі як `ws`, щоб реалізувати протокол WebSocket і забезпечити обмін даними в реальному часі між клієнтами та серверами.

Бібліотека `ws` є легкою, швидкою та простою у використанні. Вона реалізує протокол WebSocket, тож ви можете створювати й керувати WebSocket-з’єднаннями.

Ви можете встановити її за допомогою npm:

```bash
npm install ws
```

Після встановлення створіть WebSocket-сервер, імпортувавши бібліотеку `ws`, викликавши її метод `Server` і передавши порт:

```js
const WebSocket = require("ws");

const server = new WebSocket.Server({
  port: 3000,
});
```

Це запускає WebSocket-сервер, який слухає підключення на порті `3000`.

Коли клієнт підключається, сервер генерує подію `connection`.

```js
server.on("connection", (socket) => {
  console.log("Client connected");
});
```

Уявіть цю подію як спосіб сервера сказати: «новий клієнт приєднався».

Параметр `socket` представляє з’єднання між сервером і цим клієнтом. Ви використовуєте його, щоб надсилати й отримувати повідомлення.

Коли підключений клієнт надсилає дані, сервер генерує подію `message`.

```js
server.on("connection", (socket) => {
  socket.on("message", (message) => {
    console.log(message.toString());
  });
});
```

У цьому прикладі сервер слухає вхідні повідомлення й виводить їх у консоль.

Ви можете надіслати дані назад клієнту, використовуючи метод `send()`:

```js
socket.send("Hello from the server");
```

Це надсилає повідомлення через відкритий WebSocket-зв’язок.

Поки з’єднання залишається відкритим, і клієнт, і сервер можуть продовжувати обмінюватися повідомленнями в реальному часі.

Окрім `connection` і `message`, WebSocket-з’єднання також генерують події `close` і `error`.

`close` спрацьовує, коли з’єднання закривається:

```js
socket.on("close", () => {
  console.log("Disconnected");
});
```

`error` спрацьовує, коли щось іде не так:

```js
socket.on("error", (error) => {
  console.error(error);
});
```

Разом ці події становлять основу більшості WebSocket-застосунків.

Поширеною вимогою в чат-застосунках є трансляція повідомлень. Трансляція означає надсилання повідомлення кільком підключеним клієнтам замість одного.

Уявіть чат із трьома користувачами: **John Doe**, **Jane Doe** та **Wes Smith**.

Якщо John Doe надсилає повідомлення, сервер може потребувати переслати це повідомлення Jane Doe та Wes Smith.

Для цього сервер відстежує підключених клієнтів і проходить циклом по них, коли надходить повідомлення.

Перед надсиланням повідомлення сервер перевіряє, чи з’єднання з клієнтом ще відкрите:

```js
if (client.readyState === WebSocket.OPEN) {
  // do something
}
```

Кожне WebSocket-з’єднання має властивість `readyState`, яка описує його поточний стан.

`WebSocket.OPEN` означає, що з’єднання активне і готове надсилати та отримувати дані.

Перевіряючи це, сервер уникає надсилання повідомлень клієнтам, які вже відключилися або чиї з’єднання ще встановлюються:

```js
server.on("connection", (socket) => {
  socket.on("message", (message) => {
    server.clients.forEach((client) => {
      if (client.readyState === WebSocket.OPEN) {
        client.send(message.toString());
      }
    });
  });
});
```

У цьому прикладі кожен підключений клієнт отримує вхідне повідомлення. Це називається трансляцією.

Багато застосунків у реальному часі використовують трансляцію для доставки даних, таких як чат-повідомлення, живі сповіщення, спортивні оновлення та події багатокористувацьких ігор.

З бібліотекою `ws` створювати WebSocket-сервери в Node.js нескладно.

Вам потрібно лише створити сервер, слухати події, обмінюватися повідомленнями та транслювати оновлення підключеним клієнтам.

Ці будівельні блоки становлять основу багатьох застосунків у реальному часі.
