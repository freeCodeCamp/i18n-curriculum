# --description--

Спеціальні методи в Python, також відомі як «магічні методи» або «dunder-методи», — це спеціальні методи Python, які починаються і закінчуються подвійним підкресленням (`__`). Саме слово «dunder» походить від double underscores (**d** — подвійний, **under** — підкреслення).

Ви, ймовірно, вже використовували спеціальні методи, навіть не усвідомлюючи цього. Кожного разу, коли ви пишете щось на кшталт `3 + 4`, Python тихо виконує `3.__add__(4)` «під капотом». Це і є дія спеціального методу. Тож хоча ви *можете* викликати спеціальні методи напряму, ви робите це рідко. Щось на кшталт `3 + 4` набагато зрозуміліше і легше читається, ніж виклик `3.__add__(4)` власноруч.

Окрім `__add__`, `__init__()` — це ще один спеціальний метод, який ви найчастіше побачите і використовуватимете, адже це ініціалізатор класу. Є також інші, як `__len__()` і `__str__()`.

Уявіть спеціальні методи як режисерів взаємодії між людиною, що програмує, і самим інтерпретатором мови Python.

Пам’ятайте, вам не потрібно викликати спеціальні методи напряму. Замість цього Python автоматично викликає їх, коли відбуваються певні дії. Ці операції включають:

- **Арифметичні дії, як додавання, віднімання, множення, ділення та інші**. Зокрема, викликається `__add__()` для додавання, `__sub__()` для віднімання, `__mul__()` для множення і `__truediv__()` для ділення.
    
- **Операції з рядками, як конкатенація, повторення, форматування та перетворення у текст**. Викликається `__add__()` для конкатенації, `__mul__()` для повторення, `__format__()` для форматування, `__str__()` і `__repr__()` для перетворення у текст тощо.
    
- **Операції порівняння, як перевірка на рівність, менше, більше та інші**. Викликається `__eq__()` для перевірки рівності, `__lt__()` для менше, `__gt__()` для більше тощо.
    
- **Операції ітерації, як зробити об’єкт ітерованим і переходити між елементами**. Викликається `__iter__()` для повернення ітератора і `__next__()` для отримання наступного елемента.
    

Зазвичай типи даних Python, як рядки і числа, вже знають, як додавати, конкатенувати, порівнювати на рівність, використовуватися в циклах і так далі.

Але коли ви створюєте власний клас, Python не знатиме, як автоматично обробляти ці дії.

Ось тут і приходять на допомогу спеціальні методи — вони дозволяють кастомізувати вбудовану поведінку Python.

Припустимо, ви хочете отримати кількість сторінок у об’єктах книги, створених класом нижче, або порівняти їх і отримати читабельний рядок об’єктів. Ось що відбувається без спеціальних методів:

```python
class Book:
   def __init__(self, title, pages):
       self.title = title
       self.pages = pages

book1 = Book("Built Wealth Like a Boss", 420)
book2 = Book("Be Your Own Start", 420)

print(len(book1)) # TypeError: object of type 'Book' has no len()
print(str(book1)) # <__main__.Book object at 0x102ed2900>
print(book1 == book2) # False even though they have the same number of pages
```

У прикладі:

- `len(book1)` не спрацював, бо Python не знає, як отримати довжину вашого об’єкта книги без `__len__()`
    
- `str(book1)` вивів щось на кшталт `<__main__.Book object at 0x102ed2900>`, бо це стандартне представлення, коли не використовується `__str__()`
    
- `book1 == book2` дав результат `False`, бо Python просто перевіряє, чи обидва об’єкти однакові в пам’яті, а не за вмістом.
    

Ось як ви можете визначити власні спеціальні методи `__len__()`, `__str__()` і `__eq__()`, щоб полегшити роботу з об’єктами, створеними з класу `Book`:

```python
class Book:
   def __init__(self, title, pages):
       self.title = title
       self.pages = pages

   def __len__(self):
       return self.pages

   def __str__(self):
       return f"'{self.title}' has {self.pages} pages"

   def __eq__(self, other):
       return self.pages == other.pages
  
book1 = Book("Built Wealth Like a Boss", 420)
book2 = Book("Be Your Own Start", 420)

print(len(book1)) # 420
print(len(book2)) # 420
print(str(book1)) # 'Built Wealth Like a Boss' has 420 pages
print(str(book2)) # 'Be Your Own Start' has 420 pages
print(book1 == book2) # True
```

Інший приклад — кошик для покупок, де ви робите таке:

- Додаєте елементи до кошика
    
- Видаляєте елементи з кошика
    
- Отримуєте кількість елементів у кошику
    
- Перевіряєте, які елементи є в кошику
    
- Перевіряєте, чи є певний елемент у кошику
    
- Повертаєте або виводите елемент за певним індексом у кошику
    

Хоча у вас може бути метод, який додає елементи до кошика і видаляє певні елементи, ви можете створити спеціальні методи для всіх інших функцій:

- `__len__()` для отримання довжини елементів у кошику
    
- `__iter__()` для циклу по елементах кошика, щоб ви могли їх побачити
    
- `__contains__()` для перевірки, чи є певний елемент у кошику
    
- `__getitem__()` для повернення або виведення елемента за певним індексом у кошику
    

Ось приклад класу `Cart` з цими користувацькими методами і спеціальними методами:

```python
class Cart:
   def __init__(self):
       self.items = []

   def add(self, item):
       self.items.append(item)

   def remove(self, item):
       if item in self.items:
           self.items.remove(item)
       else:
           print(f'{item} is not in cart')

   def list_items(self):
       return self.items

   def __len__(self):
       return len(self.items)

   def __getitem__(self, index):
       return self.items[index]

   def __contains__(self, item):
       return item in self.items

   def __iter__(self):
       return iter(self.items)
```

І ось як ви можете їх використовувати:

```python
cart = Cart()
cart.add('Laptop')
cart.add('Wireless mouse')
cart.add('Ergo keyboard')
cart.add('Monitor')

for item in cart:
   print(item, end=' ') # Laptop Wireless mouse Ergo keyboard Monitor

print(len(cart)) # 4
print(cart[3]) # Monitor

print('Monitor' in cart) # True
print('banana' in cart) # False

cart.remove('Ergo keyboard')

print(cart.list_items()) # ['Laptop', 'Wireless mouse', 'Monitor']

cart.remove('banana') # banana is not in cart
```

Ось кілька способів, як ви будете використовувати спеціальні методи в Python у реальному житті.
