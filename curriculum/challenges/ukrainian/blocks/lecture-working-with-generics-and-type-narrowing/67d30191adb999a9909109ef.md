# --description--

Ви вже бачили дженериковий тип у попередньому уроці: `Array<string>`. Але зазвичай ви використовуєте дженерикові типи у функціях. Насправді їх можна розглядати як спеціальний параметр, який ви передаєте функції, щоб контролювати поведінку визначення типу функції.

Повернемося до нашого попереднього прикладу функції для отримання випадкового значення з масиву:

```js
const getRandomValue = (array: string[]): string => {
  return array[Math.floor(Math.random() * array.length)];
}
```

Наша функція приймає масив рядків і повертає рядок. Але що, якби ми хотіли передати масив чисел і повернути число?

Ви могли б використати об’єднаний тип:

```js
const getRandomValue = (array: (string|number)[]): (string|number) => {
  return array[Math.floor(Math.random() * array.length)];
}
const val = getRandomValue([1, 2, 4])
```

Але з таким підходом, навіть якщо ми передали масив лише чисел, TypeScript визначає, що змінна `val` може бути числом АБО рядком.

Замість цього ви можете визначити дженериковий тип для функції:

```js
const getRandomValue = <T>(array: T[]): T => {
  return array[Math.floor(Math.random() * array.length)];
}
const val = getRandomValue([1, 2, 4])
```

У нашому новому прикладі є кілька важливих компонентів. По-перше, синтаксис `<T>` повідомляє TypeScript, що ви визначаєте дженериковий тип `T` для функції. `T` — це загальна назва для дженерикових типів, але ви можете назвати його як завгодно (відповідно до конвенцій JavaScript для змінних).

Потім ми повідомляємо TypeScript, що параметр масиву — це масив значень, які відповідають дженериковому типу, а повернене значення — це один елемент того самого типу. Це дозволяє TypeScript правильно визначити, що `val` буде числом, бо масив, який ми передали, складається виключно з чисел.

А як щодо функцій, які ви не контролюєте? Давайте подивимося на метод `document.querySelector()`:

```js
const email = document.querySelector("#email");
console.log(email.value);
```

Оскільки ми робимо запит за селектором ідентифікатора, TypeScript не знає, який саме елемент ми шукаємо. Він знає, що це якийсь `Element`, але наша спроба отримати доступ до властивості `value` викличе помилку, бо інтерфейс верхнього рівня елемента не має такої властивості.

На щастя, ми можемо передати типовий аргумент у виклик функції:

```js
const email = document.querySelector<HTMLInputElement>("#email");
console.log(email.value);
```

Зверніть увагу, що ми знову використовуємо синтаксис кутових дужок (`<HTMLInputElement>`), але цього разу у виклику функції. Це повідомляє TypeScript, що елемент, який ми очікуємо знайти, буде елементом `input`, і ми більше не отримуємо помилку на властивості value, бо елементи input мають цю властивість. Звісно, `querySelector` може повернути `null`, якщо елемент не знайдено, і TypeScript видасть помилку компіляції, бо ми не обробляємо цей випадок.

Але про це ви дізнаєтеся в наступному уроці.
