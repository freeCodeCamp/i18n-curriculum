## Importancia del HTML semántico

- **Jerarquía estructural para elementos de encabezado**: Es importante usar el elemento de encabezado correcto para mantener la jerarquía estructural del contenido. El elemento `h1` es el nivel más alto de encabezado y el elemento `h6` es el nivel más bajo de encabezado.
- **Elementos HTML de presentación**: Elementos que definen la apariencia del contenido. Por ejemplo, los elementos obsoletos `center`, `big` y `font`.
- **Elementos HTML semánticos**: Elementos que contienen significado y estructura. Ej. `header`, `nav`, `figure`.

## Elementos HTML semánticos

- **Elemento de encabezado**: se utiliza para definir el encabezado de un documento o sección.

- **Elemento principal**: se usa para contener el contenido principal de la página web.

- **Elemento de sección**: se utiliza para dividir el contenido en secciones más pequeñas.

- **Elemento de sección de navegación (`nav`)**: representa una sección con enlaces de navegación.

- **Elemento figura**: se utiliza para contener ilustraciones y diagramas.

- **Elemento de énfasis (`em`)**: marca texto que tiene énfasis de estrés.

- **Elemento de texto idiomático (`i`)**: se usa para resaltar voces o estados de ánimo alternativos, términos idiomáticos de otro idioma, términos técnicos y pensamientos.

El atributo `lang` dentro de la etiqueta de apertura `i` se utiliza para especificar el idioma del contenido. En este caso, el idioma sería francés. El elemento `i` no indica si el texto es importante o no, solo muestra que de alguna manera es diferente del texto circundante.

- **Elemento de importancia fuerte (`strong`)**: marca texto que tiene gran importancia.

- **Elemento de atraer atención (`b`)**: se usa para llamar la atención sobre texto que no es importante para el significado del contenido. Se utiliza comúnmente para resaltar palabras clave en resúmenes o nombres de productos en reseñas.

- **Elemento de lista de descripciones (`dl`)**: se usa para representar una lista de agrupaciones término-descripción.
- **Elemento de término de descripción (`dt`)**: se usa para representar el término que se está definiendo.
- **Elemento de detalles de descripción (`dd`)**: se usa para representar la descripción del término.

- **Elemento de cita en bloque (`blockquote`)**: se usa para representar una sección que se cita de otra fuente. Este elemento tiene un atributo `cite`. El valor del atributo `cite` es la URL de la fuente.

- **Elemento de citación (`cite`)**: se usa para atribuir la fuente del trabajo referenciado visualmente. Marca el título de la referencia.

- **Elemento de cita en línea (`q`)**: se usa para representar una cita en línea breve.

- **Elemento de abreviatura (`abbr`)**: se usa para representar una abreviatura o acrónimo. Para ayudar a los usuarios a entender lo que es la abreviatura o el acrónimo, se puede mostrar su forma completa, una descripción legible por humanos, con el atributo `title`.

- **Elemento de dirección de contacto (`address`)**: se usa para representar la información de contacto.
- **Elemento de (Fecha) Tiempo (`time`)**: se usa para representar una fecha y/o hora. El atributo `datetime` se utiliza para traducir fechas y horas a un formato legible por máquina.

- **Atributo de fecha ISO 8601 (`datetime`)**: se usa para representar fechas y horas en un formato legible por máquina. El formato estándar es `YYYY-MM-DDThh:mm:ss`, con la `T` mayúscula como separador entre la fecha y la hora.
- **Elemento de superíndice (`sup`)**: se usa para representar texto en superíndice. Los casos de uso comunes para el elemento `sup` incluirían exponentes, letras superiores y números ordinales.

- **Elemento de subíndice (`sub`)**: se usa para representar texto en subíndice. Los casos de uso comunes para el elemento subíndice incluyen fórmulas químicas, notas al pie y subíndices de variables.

- **Elemento de código en línea (`code`)**: se usa para representar un fragmento de código de computadora.
- **Elemento de texto preformateado (`pre`)**: representa texto preformateado.

- **Elemento de anotación no articulada (`u`)**: se usa para representar un fragmento de texto en línea que debe renderizarse de manera que indique que tiene una anotación no textual.

- **Elemento de anotación en ruby (`ruby`)**: se usa para anotar texto con pronunciaciones o explicaciones de significado. Un uso de ejemplo es para tipografía de Asia oriental.
- **Elemento de paréntesis de caída de ruby (`rp`)**: se usa como sustituto para navegadores que no tienen soporte para mostrar anotaciones de ruby.
- **Elemento de texto ruby (`rt`)**: se usa para indicar texto para la anotación ruby. Normalmente se utiliza para pronunciación, o detalles de traducción en la tipografía de Asia oriental.

- **Elemento de tachado (`s`)**: se usa para representar contenido que ya no es preciso o relevante.

- **Enlaces internos**: se usan para enlazar a otra sección de la página usando `href="#id"` en un elemento `a` y asignando al elemento de destino el mismo `id`. Esto se usa comúnmente para enlaces de salto, tablas de contenido o páginas largas con múltiples secciones.

```html
<a href="#about-section">Go to About Section</a>

<section id="about-section">
  <h2>About</h2>
  <p>This is the about section of the page.</p>
</section>
```
