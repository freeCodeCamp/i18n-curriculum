# --description--

Prima di lavorare con le variabili in Python, è importante capire i tipi di dati. Un tipo di dato descrive il tipo di valore che una variabile contiene. Per esempio, un numero, un testo o una lista di elementi. I linguaggi di programmazione usano i tipi di dati per sapere come archiviare e lavorare con diversi tipi di informazioni.

Python è un linguaggio a tipizzazione dinamica come JavaScript, il che significa che non devi dichiarare esplicitamente i tipi per le variabili. Il linguaggio sa quale tipo di dato ha una variabile in base a ciò che le assegni.

Ecco alcuni esempi:

```python
name = 'John Doe' # Python knows this is a string
age = 25 # Python knows this is an integer
```

Questo è in contrasto con alcuni linguaggi a tipizzazione statica come C#, Java e C++, dove devi dichiarare i tipi con le variabili, così:

```csharp
string name = 'John Doe'
int age = 25
```

La natura a tipizzazione dinamica di Python rende la scrittura del codice molto veloce e più flessibile, ma può portare a bug imprevisti perché gli errori di tipo vengono rilevati solo quando il programma viene eseguito, non quando viene compilato.

Poiché Python determina i tipi di dati mentre il programma è **in esecuzione**, gli errori legati ai tipi vengono scoperti solo in quel momento. Quando un programma viene eseguito, Python esegue il codice riga per riga. Se arriva a una riga in cui ci si aspetta che un certo oggetto si comporti in un modo che non può, Python si ferma e mostra un errore.

Al contrario, alcuni linguaggi **compilano** il programma prima che venga eseguito. Compilare significa che il computer controlla il codice in anticipo e lo prepara per l'esecuzione. Durante questo passaggio, quei linguaggi possono intercettare errori di tipo prima che il programma inizi.

Non devi ancora conoscere quei linguaggi. L’idea importante è semplicemente:

- **In Python gli errori di tipo possono manifestarsi durante l’esecuzione**, quando il programma sta effettivamente girando e usando il tuo codice.

- **I linguaggi compilati intercettano gli errori di tipo durante la compilazione**, prima che il programma possa partire.

Per questo motivo, potresti non scoprire un errore di tipo in Python finché il programma non raggiunge quella specifica riga di codice durante l’esecuzione.

Ecco i tipi di dati più comuni che userai in Python:

- Integer: un numero intero senza decimali, per esempio, `10` o `-5`.

```python
my_integer_var = 10
print('Integer:', my_integer_var) # Integer: 10
```

- Float: un numero con la virgola decimale, come `4.41` o `-0.4`.

```python
my_float_var = 4.50
print('Float:', my_float_var) # Float: 4.5
```

- String: una sequenza di caratteri racchiusa tra virgolette singole o doppie come `'Hello world!'`.

```python
my_string_var = 'hello'
print('String:', my_string_var) # String: hello
```

- Boolean: un tipo vero o falso, scritto come `True` o `False`.

```python
my_boolean_var = True
print('Boolean:', my_boolean_var) # Boolean: True
```

- Set: una raccolta non ordinata di elementi unici, come `{0.5, 4, 'apple'}`.
 
```python
my_set_var = {7, 'hello', 8.5}
print('Set:', my_set_var) # Set: {7, 'hello', 8.5}
```

- Dictionary: una raccolta di coppie chiave-valore racchiuse tra parentesi graffe, come `{'name': 'John Doe', 'age': 28}`. 

```python
my_dictionary_var = {'name': 'Alice', 'age': 25}
print('Dictionary:', my_dictionary_var) # Dictionary: {'name': 'Alice', 'age': 25}
```

- Tuple: una raccolta ordinata e immutabile, racchiusa tra parentesi tonde, come `('apple', 4.5, 7)`.

```python
my_tuple_var = (7, 'hello', 8.5)
print('Tuple:', my_tuple_var) # Tuple: (7, 'hello', 8.5)
```

- Range: una sequenza di numeri, spesso usata nei loop, per esempio, `range(5)`.

```python
my_range_var = range(5)
print('Range:', my_range_var) # Range: range(0, 5)
```

- List: una raccolta ordinata di elementi che supporta tipi di dati diversi.
    
```python
my_list = [22, 'Hello world', 3.14, True]
print(my_list) # [22, 'Hello world', 3.14, True]
```

- None: un valore speciale che rappresenta l’assenza di un valore.

```python
my_none_var = None
print('None:', my_none_var) # None: None
```
