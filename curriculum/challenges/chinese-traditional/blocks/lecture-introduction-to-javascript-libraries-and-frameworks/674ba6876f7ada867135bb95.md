# --description--

在之前的課程中，你學會了如何在 JavaScript 中使用匯入和匯出。在本課程中，我們將探討如何在 React 中匯入和匯出組件。

匯出使組件可供其他檔案使用。匯入允許檔案使用在其他地方匯出的組件。在 React 專案中，這是組件被重複使用並結合以建造使用者介面的方式。

在本課程中，你將學習如何使用預設匯出和具名匯出來匯出 React 組件，以及如何在需要的地方匯入它們。

在這個範例中，我們有一個 `Cat` 組件，它屬於一個名為 `Cat.jsx` 的檔案。至於副檔名，我們使用 `.jsx` 副檔名，因為這個檔案主要處理 JSX。

這個 `Cat` 組件傳回一個 JSX 標記，其中包含一個標題和一張名為 Mr. Whiskers 的貓的影像。

```jsx
function Cat() {
  return (
    <div className="card">
      <h2>Mr. Whiskers</h2>
      <img
        src="https://cdn.freecodecamp.org/curriculum/cat-photo-app/running-cats.jpg"
        alt="Tuxedo cats running on dirt ground."
      />
    </div>
  );
}
```

如果我們想在另一個檔案中使用我們的 `Cat` 組件，我們需要先像這樣匯出它：

```jsx
function Cat() {
  return (
    <div className="card">
      <h2>Mr. Whiskers</h2>
      <img
        src="https://cdn.freecodecamp.org/curriculum/cat-photo-app/running-cats.jpg"
        alt="Tuxedo cats running on dirt ground."
      />
    </div>
  );
}

export default Cat;
```

我們使用 `default` 關鍵字，因為這將會是該模組的預設匯出。你也可以選擇在組件定義的同一行匯出該組件，如下所示：

```jsx
export default function Cat() {
  return (
    <div className="card">
      <h2>Mr. Whiskers</h2>
      <img
        src="https://cdn.freecodecamp.org/curriculum/cat-photo-app/running-cats.jpg"
        alt="Tuxedo cats running on dirt ground."
      />
    </div>
  );
}
```

你可以選擇在其他父組件檔案中匯入子組件，或是在根組件檔案中匯入它們。以此範例，我們將在根組件檔案中匯入 `Cat` 組件。

每個 React 專案都會有一個頂層組件，通常稱為 `App.jsx`：

```jsx
export default function App() {
  return (
    // return component here
  );
}
```

此檔案通常位於你專案的 `src` 目錄中。你將在未來的課程中學習更多關於常見專案佈局的內容。

要在根 `App` 組件中使用 `Cat` 組件，你需要這樣匯入它：

```jsx
import Cat from "./Cat";

export default function App() {
  return (
    // return component here
  );
}
```

現在，你可以像這樣在 `App` 組件內傳回 `Cat` 組件：

```jsx
import Cat from "./Cat";

export default function App() {
  return (
    <Cat />
  );
}
```

當一個檔案負責匯出單一主要組件時，這種方法運作良好，能保持匯入語法簡單且易於閱讀。請記住，一個檔案只能有一個預設匯出，這使得它非常適合主要包含單一組件的檔案。


現在你已經知道如何使用預設匯出，讓我們來看看命名匯出。命名匯出允許一個檔案共享多個組件或函式。與預設匯出不同，這些必須使用它們匯出時的精確名稱來匯入（除非你使用 `as` 來重新命名）。 

當一個檔案作為組件的函式庫時，這非常有用。 例如，這裡有一個包含兩個組件 Cat 和 Dog 的檔案：

```jsx
// Animals.jsx
export function Cat() {
  return <h2>Mr. Whiskers</h2>;
}

export function Dog() {
  return <h2>Fido</h2>;
}
```

我們在每個組件上單獨使用 `export` 關鍵字。要在另一個檔案中使用這些組件，我們使用大括號將它們 `import` 進來：


```jsx
import { Cat, Dog } from "./Animals";

export default function App() {
  return (
    <div>
      <Cat />
      <Dog />
    </div>
  );
}
```

`{ Cat, Dog }` 語法告訴 JavaScript 從 `Animals.jsx` 檔案匯入特定的命名匯出。大括號內的名稱必須與匯出的名稱完全相符。如果你需要避免命名衝突，或是你希望在目前檔案中使用不同的名稱，你可以在匯入時使用 as 關鍵字重新命名組件。

```jsx
import { Cat as Kitty } from "./Animals";

export default function App() {
  return <Kitty />;
}
```

最後，一個檔案可以有一個預設匯出和多個具名匯出。

```jsx
// Animals.jsx
export default function Cat() {
  return <h2>Mr. Whiskers</h2>;
}

export function Dog() {
  return <h2>Fido</h2>;
}
```

當匯入混合匯出時，預設匯出先出現（不帶大括號），接著是具名匯出（在大括號內）：

```jsx
// App.jsx
import Cat, { Dog } from "./Animals";

export default function App() {
  return (
    <div>
      <Cat />
      <Dog />
    </div>
  );
}
```

在此範例中，`Cat` 是預設匯出，因此不使用大括號。`Dog` 是具名匯出，因此需要使用大括號。

透過了解預設與命名匯出的運作方式，你可以將 React 組件組織在多個檔案中，並在需要時重複使用。當你建置較大型的應用程式時，選擇正確的匯出方式將有助於保持你的程式碼清晰且易於維護，並且更容易與他人創建的組件協作。
