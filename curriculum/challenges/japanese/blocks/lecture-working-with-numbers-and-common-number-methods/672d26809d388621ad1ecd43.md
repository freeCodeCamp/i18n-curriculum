# --interactive--

JavaScriptでは、`NaN`は「Not a Number」を表します。これは、表現できないか未定義の数値結果を表す特別な値です。`NaN`はグローバルオブジェクトのプロパティであり、JavaScriptでは数値の一種と見なされるため、一見すると直感に反するかもしれません。

`NaN`は通常、数値を返すはずの操作が意味のある数値を生成できなかった場合の結果です。例えば：

:::interactive_editor

```js
let result = 0 / 0;
console.log(result); // NaN
```

:::

この場合、ゼロをゼロで割ることは数学的に未定義なので、JavaScriptは`NaN`を返します。`NaN`の奇妙な性質の一つは、それ自身を含め何とも等しくないことです：

:::interactive_editor

```js
console.log(NaN === NaN); // false
```

:::

この独特の動作により、標準の比較演算子で値が`NaN`かどうかをチェックするのは難しくなります。これに対処するために、JavaScriptは`isNaN()`関数を提供しています。`isNaN()`関数プロパティは、値が`NaN`かどうかを判定するために使われます。ただし、`isNaN()`の動作を理解することが重要で、時に予期しない結果を生むことがあります。`isNaN()`の動作は以下の通りです：

:::interactive_editor

```js
console.log(isNaN(NaN));       // true
console.log(isNaN(undefined)); // true
console.log(isNaN({}));        // true

console.log(isNaN(true));      // false
console.log(isNaN(null));      // false
console.log(isNaN(37));        // false

console.log(isNaN("37"));      // false: "37" is converted to 37
console.log(isNaN("37.37"));   // false: "37.37" is converted to 37.37
console.log(isNaN(""));        // false: empty string is converted to 0
console.log(isNaN(" "));       // false: string with a space is converted to 0

console.log(isNaN("blabla"));  // true: "blabla" is not a number
```

:::

ご覧の通り、`isNaN()`はまずパラメータを数値に変換しようとします。変換できなければ`true`を返します。この動作は、特に数値に型変換できる文字列を扱う場合に驚くべき結果をもたらすことがあります。

これらの潜在的な不整合を受けて、ES6（2015年にリリースされたJavaScriptの第6版）では`Number.isNaN()`が導入されました。このメソッドはテスト前にパラメータを数値に変換しようとはせず、値が正確に`true`の場合にのみ`NaN`を返します：

:::interactive_editor

```js
console.log(Number.isNaN(NaN));        // true
console.log(Number.isNaN(Number.NaN)); // true
console.log(Number.isNaN(0 / 0));      // true

console.log(Number.isNaN("NaN"));      // false
console.log(Number.isNaN(undefined));  // false
console.log(Number.isNaN({}));         // false
console.log(Number.isNaN("blabla"));   // false
```

:::

`Number.isNaN()`は、特に型変換によってグローバル`NaN`関数で予期しない結果が生じる場合に、`isNaN()`値をチェックするより信頼できる方法を提供します。実際には、数値演算や数値であるべきユーザー入力を扱う際に、エラーや予期しない入力を適切に処理するために`NaN`をチェックする必要があります。例えば：

:::interactive_editor

```js
let a = 0;
let b = 0;
let result = a / b;

if (Number.isNaN(result)) {
  result = "Error: Division resulted in NaN";
}

console.log(result); // "Error: Division resulted in NaN"
```

:::

この例では、`Number.isNaN()`を使って割り算の結果が`NaN`になる場合を検出し、この状況を適切に処理しています。`NaN`を理解し、正しくチェックすることは、特に数学的演算やユーザー入力の解析を扱う際に堅牢なJavaScriptコードを書くために重要です。
