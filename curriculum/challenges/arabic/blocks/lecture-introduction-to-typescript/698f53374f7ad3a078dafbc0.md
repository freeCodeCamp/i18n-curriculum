# --description--

في هذا الدرس، ستتعلم كيفية إضافة أنواع للكائنات والمصفوفات.

لنلقِ نظرة على المصفوفات أولاً.

إليك مثال على مصفوفة من لغات البرمجة:

```js
const programmingLanguages = ["Java", "C++", "Python"];
```

إذا أردت تعيين نوع تلك المصفوفة صراحة كمصفوفة من السلاسل النصية، يمكنك كتابتها كـ `string[]`:

```ts
const programmingLanguages:string[] = ["Java", "C++", "Python"];
```

كما تعلمت سابقًا، TypeScript ذكي بما يكفي ليستنتج النوع كمصفوفة من السلاسل النصية إذا كتبت هذا فقط:

```js
const programmingLanguages = ["Java", "C++", "Python"];
```

لكن التعيين الصريح سيساعد في الحالات التي تحاول فيها إضافة نوع آخر إلى المصفوفة مثل هذا:

```ts
// Type 'number' is not assignable to type 'string'.(2322)
const programmingLanguages:string[] = ["Java", 9, "Python"];
```

طريقة أخرى لتعيين أنواع المصفوفات هي استخدام هذه الصيغة النحوية:

```ts
const programmingLanguages:Array<string> = ["Java", "C++", "Python"];
```

في الأساس، هاتان الصيغتان النحويتان متطابقتان، والاختيار بينهما عادةً مسألة تفضيل.

الآن، لنلقِ نظرة على تعيين أنواع الكائنات.

إليك مثال على كائن حرفي:

```js
const developer = {
  firstName: "Jessica",
  isEmployed: true
}
```

لتعيين أنواع خصائص الكائن صراحة، يمكنك استخدام هذه الصيغة النحوية:

```ts
const developer: { firstName: string, isEmployed: boolean } = {
  firstName: "Jessica",
  isEmployed: true
}
```

إذا حاولت إضافة أو إزالة خصائص من الكائن، ستحصل على خطأ مثل هذا:

```ts
// Property 'isEmployed' is missing in type '{ firstName: string; }' but required in type '{ firstName: string; isEmployed: boolean; }'
const developer: { firstName: string, isEmployed: boolean } = {
  firstName: "Jessica",
}
```

يمكنك جعل خصائص الكائن اختيارية باستخدام `?` بعد اسم الخاصية هكذا:

```ts
const developer: { firstName: string, isEmployed?: boolean } = {
  firstName: "Jessica",
}
```

لن يعرض TypeScript خطأ بعد الآن لأن الخاصية `isEmployed` تم تعيينها كخيارية.

حتى الآن، جميع أمثلة الكائنات كانت ذات خصائص مقيدة. هذا يعني أننا سردنا صراحة كل خاصية مسموح للكائن أن يحتويها:

```ts
const developer: { firstName: string, isEmployed: boolean } = {
  firstName: "Jessica",
  isEmployed: true
}
```

لكن ماذا لو لم تكن تعرف جميع أسماء الخصائص مسبقًا؟ ماذا لو أردت كائنًا يمكن أن يحتوي على أي عدد من الخصائص لكن يجب أن تتبع جميع القيم نوعًا محددًا؟

هناك طريقتان لتحقيق ذلك. الطريقة الأولى هي استخدام `Record` وهو نوع مساعد عام مدمج في TypeScript.

إليك الصيغة الأساسية:

```ts
Record<Keys, ValueType>
```

النوع الأول يمثل نوع المفاتيح بينما النوع الثاني يمثل نوع القيم.

إليك مثالًا على استخدام `Record` لكائن `userRoles`:

```ts
const userRoles: Record<string, string> = {
  admin: "full-access",
  editor: "limited-access",
  viewer: "read-only"
};
```

يمكن لهذا الكائن `userRoles` أن يحتوي على أي مفاتيح نصية، ويجب أن تكون جميع القيم سلاسل نصية. إذا حاولت تعيين قيمة غير نصية، ستحصل على خطأ:

```ts
// Type 'number' is not assignable to type 'string'
const userRoles: Record<string, string> = {
  admin: 1
};
```

الطريقة الثانية لكتابة نفس الفكرة هي باستخدام توقيع الفهرس:

```ts
const obj: { [key: string]: string } = {};
```

الصيغة أعلاه تعني أن أي خاصية بمفتاح نصي يجب أن تكون قيمتها نصية. إليك مثالًا لكائن `settings`:

```ts
const settings: { [key: string]: string } = {
  theme: "dark",
  language: "en",
  layout: "grid"
};
```

تمامًا كما في السابق، إذا كانت أنواع القيم غير صحيحة، سيعرض TypeScript رسالة خطأ:

```ts
// Type 'boolean' is not assignable to type 'string'
const settings: { [key: string]: string } = {
  darkMode: true
};
```

كل من `Record` وتوقيعات الفهرس متطابقان في الأساس. لذا الاختيار بينهما مسألة تفضيل في الغالب.
