# --description--

Você já teve seu app Express travando sem uma mensagem de erro clara? A depuração e o registro em log podem ajudar você a ver exatamente o que está acontecendo nos bastidores, para que encontre e corrija problemas mais rápido.

Nesta lição, vamos explorar como o Express lida com depuração e registro em log — e como você pode usar isso para melhorar seu fluxo de desenvolvimento.

## Depuração embutida no Express

O Express já inclui uma ferramenta de depuração chamada `debug`. Você não precisa instalar nada para usá-la.

Para habilitar a depuração, tudo que você precisa fazer é definir a variável de ambiente `DEBUG` quando executar seu app.

Aqui está um exemplo:

```js
DEBUG=express:* node index.js
```

Esse comando habilita todas as mensagens de depuração sob o namespace `express:*`. Você começará a ver saídas como:

```js
express:router:route dispatching GET /
express:router:layer processing GET /
```

Essas mensagens ajudam você a rastrear o despacho de rotas, chamadas de middleware e mais. É uma ótima forma de entender o fluxo das requisições no seu app.

Quer ser mais específico? Você pode restringir o namespace:

```js
DEBUG=express:router node index.js
```

Isso mostrará apenas os logs relacionados à lógica do roteador do Express.

No Windows, a sintaxe é um pouco diferente para o PowerShell:

```js
$env:DEBUG = "express:*"; node index.js
```

E para o CMD, fica assim:

```js
set DEBUG=express:* && node index.js
```

A ideia é a mesma, só muda a forma de definir variáveis de ambiente.

Agora vamos ver o registro em log no Express.

Para registrar requisições HTTP, muitos desenvolvedores usam middleware como o `morgan`.

Veja como usá-lo:

```bash
npm install morgan
```

Depois adicione ao seu app:

```js
const express = require('express')
const morgan = require('morgan')
const app = express()

app.use(morgan('dev'))s
```

Agora, toda requisição recebida será registrada no terminal com detalhes como método, URL, código de status e tempo de resposta.

E por que isso importa?

A depuração dá a você uma visão de como o Express está lidando com requisições, middleware e roteamento sem poluir seus logs de produção.

O registro em log fornece um histórico do que está acontecendo nas requisições, o que é crucial para depurar bugs, entender o tráfego e identificar problemas cedo.

Juntos, eles dão controle e clareza no seu processo de desenvolvimento.
