# --interactive--

テーブルを見ると、私たちはすぐにデータとヘッダーの間に視覚的な関連付けを始めます。

例えば、ペットを表すテーブルがあるとしましょう。犬が2匹、猫が2匹いて、そのテーブルには名前と年齢が表示されています。視覚的に見る人はこのテーブルの関係性を理解できるかもしれませんが、スクリーンリーダーを使ってテーブルを操作する人にとっては、値とヘッダーの関連付けを理解するのはずっと難しくなります。

ウェブ開発者として、これらの関連付けを作成し、HTMLマークアップをスクリーンリーダーユーザーにもわかりやすい形で構造化する責任があります。

では、誰でも理解できるアクセシブルなテーブルをどのように作成できるか見ていきましょう。最初のベストプラクティスはテーブルキャプションの使用です。`caption` 要素を使うと、テーブルのキャプション（またはタイトル）を書けるので、特に支援技術を使うユーザーがテーブルの目的や内容をすぐに理解できます。`caption` 要素は`table` 要素の開始タグの直後に置くべきです。こうすることで、スクリーンリーダーや他の支援技術が内容を読む前にキャプションを読み上げて、より多くの文脈を提供できます。

```html
<table>
  <caption>Our Pets</caption>
  <!-- Table Rows and Columns -->
</table>
```

次に行ヘッダーと列ヘッダーについて話しましょう。ヘッダーは特別なセルで、通常は行や列の先頭にあり、その行や列に格納されているデータの種類を説明します。行ヘッダーや列ヘッダーはテーブルヘッダー要素、`th` で定義できます。

例えば、以下のコードは2匹のペットのテーブルを作成しています。各行には行ヘッダー（ペットの名前）があり、各列には列ヘッダーがあり、その列のデータが何を表しているか（年齢や種類）を説明しています。

:::interactive_editor

```html
<table>
  <caption>Our Pets</caption>
  <thead>
    <tr>
      <!-- Column Headers -->
      <th>Name</th>
      <th>Age</th>
      <th>Type</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <th>Nora</th> <!-- Row Header -->
      <td>5</td>
      <td>Dog</td>
    </tr>
    <tr>
      <th>Gino</th> <!-- Row Header -->
      <td>2</td>
      <td>Cat</td>
    </tr>
  </tbody>
</table>
```

:::

上記のコードでは、`caption` 要素の開始タグの直後に`table` 要素があります。そして、テーブルヘッド（`thead`）要素の中に列ヘッダー（`Name`、`Age`、`Type`）があります。2行目と3行目では、テーブルボディ（`tbody`）要素の中にペットのデータが入っています。ペットの名前はテーブルヘッダー要素（`th`）の中にあるため、行ヘッダーです。

データセルと対応するヘッダーを関連付けることもスクリーンリーダーにとって非常に重要です。`scope` 属性はヘッダーが行ヘッダーか列ヘッダーかを決定します。
スクリーンリーダーはテーブルの構造からこれを推測できる場合もありますが、明確にするために`scope` を明示的に示すことが一般的に推奨されます。

`scope` 属性には4つの可能な値があります。最もよく使うのは列用の`col` と行用の`row` です。
以下のコードでは、列ヘッダーと行ヘッダーに`scope` 属性を追加しています。3つの列ヘッダー（`Name`、`Age`、`Type`）には`scope` が`col`、つまり列です。

2つの行ヘッダー（`Nora` と `Gino`）には`scope` が`row` となっています。

:::interactive_editor

```html
<table>
  <caption>Our Pets</caption>
  <thead>
    <tr>
      <!-- Now they have scope -->
      <th scope="col">Name</th>
      <th scope="col">Age</th>
      <th scope="col">Type</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <th scope="row">Nora</th>
      <td>5</td>
      <td>Dog</td>
    </tr>
    <tr>
      <th scope="row">Gino</th>
      <td>2</td>
      <td>Cat</td>
    </tr>
  </tbody>
</table>
```

:::

列や行のヘッダーが複数のセルにまたがる場合、`scope` はそれぞれのセルに個別に適用されます。以下はその例です。

:::interactive_editor

```html
<table>
  <tbody>
    <tr>
      <td></td>
      <th scope="col">Name</th>
      <th scope="col">Age</th>
    </tr>
    <tr>
      <th rowspan="2" scope="row">Dogs</th>
      <th scope="row">Nora</th>
      <td>5</td>
    </tr>
    <tr>
      <th scope="row">Gino</th>
      <td>2</td>
    </tr>
    <tr>
      <th rowspan="2" scope="row">Cats</th>
      <th scope="row">Lulu</th>
      <td>10</td>
    </tr>
    <tr>
      <th scope="row">Elizabeth</th>
      <td>6</td>
    </tr>
  </tbody>
</table>
```

:::

このテーブルでは、`Nora` の年齢（`5`）には1つの列ヘッダー（`Age`）と2つの行ヘッダー（`Dogs` と `Nora`）があります。`Gino` の年齢（`2`）にも1つの列ヘッダー（`Age`）と2つの行ヘッダー（`Dogs` と `Gino`）があります。

ただし、複雑な構造のテーブルを解釈できないスクリーンリーダーもあるため、できるだけテーブルを平坦化して、複数のセルにまたがる行や列のヘッダーを避けることも心がけてください。

ユーザーがこの情報にアクセスできるようにすることが常に目標です。たとえスクリーンリーダーが複雑なテーブル構造を扱える場合でもです。

次にセルの幅についてですが、固定値の使用は避けることをお勧めします。代わりにパーセンテージなどの相対値を使いましょう。また、セルの高さを定義するのも避けてください。これによりユーザーがテキストサイズを自分のニーズに合わせて調整しやすくなります。

最後に、可能な限りブラウザーにテーブル幅を決定させるべきです。これにより横スクロールの必要性を減らせます。

HTMLテーブルは構造化されたデータをアクセシブルで理解しやすい形式で提示するために不可欠です。これらのアクセシビリティガイドラインに従うことで、誰にとってもわかりやすいテーブルを作成できます。
