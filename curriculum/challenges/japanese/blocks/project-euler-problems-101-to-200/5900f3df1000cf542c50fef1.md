# --description--

長さが `n` 単位の列に、最小長さが `m` 単位の赤いブロックが配置されます。赤いブロックは長さが異なってもよく、任意の2つの赤いブロックの間には少なくとも1つの黒いマスが挟まれています。

塗りつぶし数を表す関数 $F(m, n)$ は、その列を塗りつぶす方法の数を表します。

例えば、$F(3, 29) = 673135$、$F(3, 30) = 1089155$ です。

つまり、m = 3 の場合、塗りつぶし数関数が初めて100万を超える最小の n は30であることがわかります。

同様に、m = 10 の場合、$F(10, 56) = 880711$、$F(10, 57) = 1148904$ であることが確認できるので、塗りつぶし数関数が初めて100万を超える最小の n は57です。

m = 50 のとき、塗りつぶし数関数が初めて100万を超える最小の `n` の値を求めてください。

**注意：** これは問題114のより難しいバージョンです。
