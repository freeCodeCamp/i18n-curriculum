# --description--

`/public/logo.png`、`./script.js`、または`../styles.css`のようなリンクを見たことがあるかもしれません。しかし、これらの特別な種類のリンクは何を意味しているのでしょうか？これらはファイルパスと呼ばれます。知っておくべき重要な構文が3つあります。まずスラッシュで、これはお使いのオペレーティングシステムによってバックスラッシュ（`\`）またはフォワードスラッシュ（`/`）になります。次に単一ドット（`.`）です。そして最後に二重ドット（`..`）があります。

スラッシュは「パス区切り文字」として知られています。これはフォルダー名やファイル名の間の区切りを示すために使われます。これにより、`naomis-files/`が同じ名前のディレクトリを指し、`naomis/files/`が`files`ディレクトリ内の`naomis`ディレクトリを指していることがコンピューターにわかります。

単一ドットは現在のディレクトリを指し、二重ドットは親ディレクトリを指します。単一ドットは通常、パスが相対パスとして認識されるように使われます。以前のレッスンで相対パスと絶対パスの違いを学んだことを思い出してください。

しかし、二重ドットは現在の作業ディレクトリの外にあるファイルにアクセスするためにより一般的に使われます。

例えば、次のファイルツリーがあるとします：

```sh
my-app/
├─ public/
│  ├─ favicon.ico
│  ├─ index.html
├─ src/
│  ├─ index.css
│  ├─ index.js
```

もし`public/index.html`ファイルが`favicon.ico`ファイルを読み込む必要がある場合、現在のディレクトリにアクセスするために単一ドットを使った相対パスを使います：`./favicon.ico`。もし`public/index.html`ファイルが`index.css`ファイルを読み込む必要がある場合、まず親の`my-app`ディレクトリに移動し、次に`src`ディレクトリに移動し、最後にファイルにアクセスするために二重ドットを使った相対パスを使います：`../src/index.css`。
