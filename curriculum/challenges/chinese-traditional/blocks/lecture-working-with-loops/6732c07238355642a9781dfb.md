# --interactive--

`break` 述語用於提前離開迴圈，而 `continue` 述語用於跳過迴圈的當前迭代並移至下一次迭代。

以下是使用 `break` 述語於 `for` 迴圈中的範例：

:::interactive_editor

```js
for (let i = 0; i < 10; i++) {
  if (i === 5) {
    break;
  }
  console.log(i);
}
```

:::

在上述範例中，迴圈從 `0` 開始計數，並且當 `i` 小於 `10` 時，迴圈將繼續執行。

在迴圈中，我們檢查 `i` 是否相等於 `5`。如果是，我們使用 `break` 述語提前離開迴圈。如果不是，我們將 `i` 的值記錄到主控台。因此，程式碼的輸出將列印數字 `0`、`1`、`2`、`3` 和 `4`。

當你想根據某個條件提早離開迴圈時，`break` 述語非常有用。 例如，如果你正在搜尋陣列中的特定值，當你找到該值時，可以使用 `break` 述語離開迴圈。

有時你可能想要跳過迴圈的特定一次執行，而不完全離開迴圈。這時候就會用到 `continue` 述語。以下是一個在 `for` 迴圈中使用 `continue` 述語的範例：

:::interactive_editor

```js
for (let i = 0; i < 10; i++) {
  if (i === 5) {
    continue;
  }
  console.log(i);
}
```

:::

就像之前一樣，我們已將 `i` 初始化為 `0`，並且有一個條件，只要 `i` 小於 `10` 就會執行迴圈。

在迴圈內，當 `i` 相等於 `5` 時，我們使用 `continue` 述語來跳過當前的迭代並移動到下一個。

這段程式碼的輸出將列印數字 `0`、`1`、`2`、`3`、`4`、`6`、`7`、`8` 和 `9`。因為 `continue` 述語，數字 `5` 被跳過。

你還可以對 `break` 和 `continue` 述語使用標籤，以指定你想要中斷或繼續的迴圈。

當你有巢狀迴圈並且想從內層迴圈控制外層迴圈的流程時，這非常有用。

這裡是一個使用標籤與 `break` 述語的範例：

:::interactive_editor

```js
outerLoop: for (let i = 0; i < 3; i++) {
  innerLoop: for (let j = 0; j < 3; j++) {
    if (i === 1 && j === 1) {
      break outerLoop;
    }
    console.log(`i: ${i}, j: ${j}`);
  }
}
```

:::

在此範例中，我們有一個外層的 `for` 迴圈，標記為 `outerLoop`，以及一個內層的 `for` 迴圈，標記為 `innerLoop`。

當 `i` 相等於 `1` 且 `j` 相等於 `1` 時，我們使用帶有 `outerLoop` 標籤的 `break` 述語來提前離開外層迴圈。這將離開內層和外層迴圈。

這段程式碼的輸出將會在主控台記錄以下內容：

```md
"i: 0, j: 0"
"i: 0, j: 1"
"i: 0, j: 2"
"i: 1, j: 0"
```

大多數時候你不會發現需要在 `break` 和 `continue` 述語中使用標籤，但知道你有這個選項以備不時之需是很好的。
