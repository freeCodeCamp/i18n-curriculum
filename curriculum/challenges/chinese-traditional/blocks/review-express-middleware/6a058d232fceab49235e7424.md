## 什麼是中介層？

- **中介層** 是在伺服器請求生命週期中執行的函式（程式）。
- 中介層可以存取 **請求物件** (`req`)、**回應物件** (`res`) 和 **`next` 函式**。
- 常見用途：記錄請求、處理錯誤、解析傳入資料，以及提供靜態檔案。
- 中介層會依照加入應用程式的順序執行。

### 中介層的運作方式

1. 請求送到伺服器。
2. 中介層函式處理該請求。
3. 如果中介層呼叫 `next()`，控制權會傳遞給下一個中介層或路由處理常式。
4. 如果中介層沒有呼叫 `next()`，請求-回應週期就會停止。

```javascript
app.use((req, res, next) => {
  console.log('Request URL:', req.url);
  console.log('Request Method:', req.method);
  next();
});
```

- `app.use()` 將中介層全域地添加到應用程式中。

## 應用程式層級中介層

- **應用程式層級中介層** 綁定在 Express 應用程式的實體上。
- 使用 `app.use()` 或路由專用方法如 `app.get()`、`app.post()` 等註冊。
- 對每個傳入請求執行，除非限制在特定路由。
- 使用情境：跨多個路由的記錄、驗證和錯誤處理。

```javascript
app.use((req, res, next) => {
  console.log('Request received at:', new Date());
  next();
});
```

## 路由器層級中介層

- **路由器層級中介層** 綁定在 `express.Router()` 的實體上，而非主應用程式。
- 使用 `router.use()` 套用，且只影響該路由器所處理的路由。
- 有助於將應用程式模組化成獨立的路由模組。
- 使用 `app.use('/path', router)` 掛載路由器，將中介層範圍限定在特定路由。

```javascript
const router = express.Router();

router.use((req, res, next) => {
  console.log('Request made to /menu route');
  next();
});

router.get('/drinks', (req, res) => {
  res.send('Welcome to the drinks menu!');
});

app.use('/menu', router);
```

## 錯誤處理中介層

- **錯誤處理中介層** 是用來捕捉請求-回應週期中錯誤的特殊中介層。
- 它有 **四個參數**：`err`、`req`、`res` 和 `next`。四參數的標記是 Express 判斷這是錯誤處理常式的依據。
- 當錯誤被傳給 `next(err)` 時會被喚起。
- 應該放在中介層堆疊的 **最後面**，在所有其他中介層和路由之後。

```javascript
app.use((err, req, res, next) => {
  console.error(err.stack);
  res.status(500).send('Something went wrong!');
});
```

## 內建與第三方中介層

### 內建中介層

- **`express.json()`**：解析帶有 JSON 載荷的傳入請求。

```javascript
app.use(express.json());
```

- **`express.static()`**：提供靜態檔案，如圖片、CSS 和 JavaScript。

```javascript
app.use(express.static('public'));
```

- **`express.urlencoded()`**：解析帶有 URL 編碼載荷的傳入請求（HTML 表單送出的格式）；解析後的資料可在 `req.body` 上取得。

```javascript
app.use(express.urlencoded({ extended: true }));
```

### 第三方中介層

- **`cors`**：啟用跨來源資源共享。

```javascript
app.use(cors());
```

- **`morgan`**：記錄 HTTP 請求。

```javascript
app.use(morgan('tiny'));
```

- 內建和第三方中介層函式皆依照呼叫 `app.use()` 的順序加入。
