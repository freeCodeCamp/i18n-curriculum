# --description--

渐进增强是一种设计方法，它确保所有用户，无论使用何种浏览器或设备，都能访问应用程序的核心内容和功能。 

它侧重于为所有人提供一个基础体验，同时为使用更高级浏览器或网络连接更好的用户提供额外功能和改进。

渐进增强方法遵循以下核心原则：

- 所有核心内容和基本功能都应在所有浏览器上可访问
- 所有高级布局都应通过外部 CSS 样式表提供
- 所有高级功能都应通过外部 JavaScript 文件提供
- 应尊重用户的浏览器偏好设置

采用渐进增强方法可以使你的应用程序更具可访问性，因为在不受支持的环境中，所有核心内容和功能都不会被阻塞。 

在速度方面，渐进增强方法也有助于提高应用程序的性能。 

使用较慢网络连接速度的用户仍然能够访问内容，因为浏览器会首先下载必要的资源。

在 SEO 方面，渐进增强也有助于提高应用程序的可见性。 

搜索引擎将能够抓取应用程序的内容，因为核心内容在初始 HTML 响应中已可用。

尽管有人批评这种方法，认为对于严重依赖 JavaScript 实现功能的应用程序来说，它并不总是现实的，但在构建应用程序时，这仍然是一个值得遵循的良好实践。
