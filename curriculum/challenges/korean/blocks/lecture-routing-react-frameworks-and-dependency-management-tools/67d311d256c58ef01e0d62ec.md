# --description--

지금까지 React를 사용해 사용자 인터페이스를 구축해왔습니다. 라우팅 같은 추가 기능이 필요했다면 React Router 같은 서드파티 라이브러리를 가져와서 서로 다른 뷰 간 전환을 할 수 있었을 겁니다.

그런데 전체 스택 웹 애플리케이션을 만들어야 한다면 어떻게 할까요? 프론트엔드에는 React를, 백엔드 로직에는 JavaScript만 사용하고 싶다면 Node와 Express를 사용할 수 있습니다. 또는 백엔드에 Go, Python, Java 같은 다른 언어를 사용할 수도 있습니다.

이 모든 방법이 가능하지만, 때로는 React 프레임워크를 사용하는 것이 좋을 때가 있습니다. React 프레임워크는 라우팅, 이미지 최적화, 데이터 가져오기, 인증 등 다양한 기능을 제공합니다. 이는 특정 용도에 따라 별도의 프론트엔드와 백엔드 애플리케이션을 따로 설정하지 않아도 된다는 뜻입니다.

이제 인기 있는 React 프레임워크인 Next.js를 좀 더 자세히 살펴보겠습니다. Next.js의 주요 기능 중 하나는 파일 시스템 기반 라우터입니다. 이 라우팅 시스템은 동적 라우트, 병렬 라우트, 라우트 핸들러, 리다이렉트, 국제화 등을 지원합니다.

다음은 커스텀 요청 핸들러를 만드는 예시입니다:

```js
export async function GET() {
  const res = await fetch("https://example-api.com");
  const data = await res.json();

  return Response.json({ data });
}
```

`route.js` 디렉터리 안에 `app/api`라는 파일에서 GET 또는 POST 요청 같은 라우트 핸들러를 정의할 수 있습니다.

Next.js의 또 다른 기능은 자동 이미지 및 폰트 최적화입니다.

다음은 `Image` 파일 안에서 `page.js` 컴포넌트를 사용하는 예시입니다:

```jsx
import Image from "next/image";

export default function Page() {
  return (
    <Image src="link-to-image-goes-here" alt="descriptive-title-goes-here" />
  );
}
```

`Image` 컴포넌트는 기본 HTML `img` 요소를 확장하여 페이지 로드 속도와 크기 최적화를 빠르게 합니다. 즉, 이미지는 뷰포트에 들어올 때만 로드되고 `Image` 컴포넌트는 각 기기에 맞는 크기의 이미지를 자동으로 제공합니다.

이것들은 Next.js의 일부 기능에 불과하며, 견고한 전체 스택 웹 애플리케이션을 구축하는 데 사용할 수 있는 더 많은 기능이 있습니다. Remix 같은 다른 React 프레임워크도 현대적인 전체 스택 웹 애플리케이션을 구축할 수 있는 동일한 기능을 제공합니다.

JavaScript 라이브러리와 프레임워크 환경은 끊임없이 변하지만, 사용 가능한 옵션을 알고 각 프레임워크의 장단점을 배우는 것이 중요합니다.
