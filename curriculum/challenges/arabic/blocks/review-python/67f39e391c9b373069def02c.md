## ما هي بايثون؟

- **مقدمة**: بايثون هي لغة برمجة عامة الغرض معروفة ببساطتها وسهولة استخدامها. تُستخدم بايثون في مجالات عديدة مثل علم البيانات وتعلم الآلة، تطوير الويب، البرمجة النصية والأتمتة، الأنظمة المدمجة وإنترنت الأشياء، وأكثر من ذلك.
- **حالات الاستخدام الشائعة**: تُستخدم بايثون في علم البيانات، تعلم الآلة، تطوير الويب، الأمن السيبراني، الأتمتة، وأجهزة الحاسوب الصغيرة مثل Raspberry Pi واللوحات المتوافقة مع MicroPython.

## بايثون في بيئة العمل المحلية الخاصة بك

- **التثبيت**: أفضل طريقة لتثبيت بايثون على ويندوز، ماك، ولينكس هي تحميل المثبت من الموقع الرسمي لبايثون (`https://www.python.org/`).
- **الطرفية**: واجهة نصية تتيح لك التفاعل مع الكمبيوتر عبر كتابة الأوامر. كل نظام تشغيل يأتي مع تطبيق طرفية افتراضي. على macOS، يمكنك استخدام تطبيق Terminal. على ويندوز، يمكنك استخدام Command Prompt أو PowerShell. على لينكس، لكل بيئة سطح مكتب تطبيق طرفية افتراضي خاص بها، مثل GNOME Terminal أو Konsole.
- **بيئة تطوير متكاملة (IDE)**: تعني بيئة التطوير المتكاملة وتحتوي على ميزات تشمل محرر شفرة، أدوات اختبار، طرفية، والمزيد.
- **محررات الشفرة وبيئات التطوير المتكاملة الشهيرة لبايثون**: VS Code، PyCharm، وSpyder
- **تشغيل الشفرة محليًا**: لتشغيل شفرة بايثون، لديك عدة خيارات. الخيار الأول هو النقر على زر التشغيل في VS Code الذي يشبه زر التشغيل الموجود في الزاوية العلوية اليمنى. هذا سيفتح طرفية ويشغل سكريبت بايثون هناك.

الخيار الآخر هو فتح طرفية وتشغيل البرنامج يدويًا. إليك مثال لتشغيل برنامج يسمى `main.py`:

```bash
python main.py
```

يجب تشغيل هذا الأمر من المجلد الذي تم حفظ `main.py` فيه. على سبيل المثال، إذا كان `main.py` داخل مجلد يسمى `python-projects`، استخدم أولاً الأمر التالي في الطرفية:

```bash
cd python-projects
```

ثم شغّل:

```bash
python main.py
```

## استخدام القشرة التفاعلية لبايثون

- **التعريف**: القشرة التفاعلية هي برنامج يتيح لك كتابة الأوامر واحدة تلو الأخرى ورؤية النتائج. افتح تطبيق طرفية واكتب `python` واضغط `Enter`. سيبدأ هذا قشرة تفاعلية لبايثون. الرمز `>>>` يعني أن بايثون ينتظر منك كتابة أمر. جرّب طباعة `"Hello, world!"` في الطرفية:

```bash
print("Hello, world!")
```

يجب أن ترى النص يظهر هكذا:

```bash
>>> print("Hello, world!")
Hello, world!
```

بعد طباعة النص، تعود بايثون إلى التالي:

```bash
>>>
```

هذا يعني أنه يمكنك كتابة أمر آخر.

- **دورة القراءة، التقييم، الطباعة، التكرار (REPL)**: كلما كتبت أوامر، سيقرأ المفسر المُدخل، يُقيّمه، يطبع النتيجة، ثم يعود ليعرض `>>>` لتتمكن من كتابة المزيد من الأوامر.

## المتغيرات

- **إعلان المتغيرات**: لإعلان متغير، تبدأ باسم المتغير متبوعًا بمعامل التعيين (`=`) ثم القيمة. يمكن أن تكون القيمة رقمًا، سلسلة نصية، قيمة منطقية، إلخ. إليك بعض الأمثلة:

```py
name = 'John Doe'
age = 25
```

- **اصطلاحات تسمية المتغيرات**: إليك الاصطلاحات التي يجب استخدامها لتسمية المتغيرات:

- أسماء المتغيرات يمكن أن تبدأ فقط بحرف أو شرطة سفلية (_) وليس برقم.
  - أسماء المتغيرات يمكن أن تحتوي فقط على أحرف أبجدية رقمية (a-z, A-Z, 0-9) والشرطة السفلية (_).
  - أسماء المتغيرات حساسة لحالة الأحرف — `age`، `Age`، و`AGE` تعتبر أسماء مختلفة.
  - أسماء المتغيرات لا يمكن أن تكون من الكلمات المفتاحية المحجوزة في بايثون مثل `if`، `class`، أو `def`.
  - أسماء المتغيرات متعددة الكلمات تُفصل بالشرطة السفلية. مثلًا، `snake_case`.


## التعليقات

- **تعليقات السطر الواحد**: تستخدم هذه التعليقات للملاحظات القصيرة التي تريد تركها في شفرتك.

```py
# This is a single line comment
```

- **سلاسل نصية متعددة الأسطر**: يمكن استخدام هذه السلاسل لترك ملاحظات أكبر أو لإضافة تعليق على أجزاء من الشفرة.

```py
"""
This is a multi-line string.
Here is some code commented out.

name = 'John Doe'
age = 25
"""
```

- **دالة `print()`**: لطباعة البيانات في وحدة التحكم، يمكنك استخدام الدالة `print()` هكذا:

```py
print('Hello world!') # Hello world!
```

## أنواع البيانات الشائعة في بايثون

- **مقدمة**: بايثون لغة ذات نوع ديناميكي مثل جافاسكريبت، مما يعني أنك لا تحتاج إلى إعلان أنواع المتغيرات صراحة. اللغة تعرف نوع المتغير بناءً على القيمة التي تعينها له.
- **عدد صحيح**: رقم كامل بدون كسور عشرية:

```py
my_integer_var = 10
print('Integer:', my_integer_var) # Integer: 10
```

- **عدد عشري (float)**: رقم يحتوي على كسور عشرية:

```py
my_float_var = 4.50
print('Float:', my_float_var) # Float: 4.5
```

- **سلسلة نصية**: متتالية من الأحرف محاطة بعلامات اقتباس:

```py
my_string_var = 'hello'
print('String:', my_string_var) # String: hello
```

- **قيمة منطقية**: قيمة تمثل إما `True` أو `False`:

```py
my_boolean_var = True
print('Boolean:', my_boolean_var) # Boolean: True
```

- **مجموعة (Set)**: مجموعة غير مرتبة من العناصر الفريدة:

```py
my_set_var = {7, 5, 8}
print('Set:', my_set_var) # Set: {7, 5, 8}
```

- **قاموس (Dictionary)**: مجموعة من أزواج المفتاح-القيمة، محاطة بأقواس متعرجة:

```py
my_dictionary_var = {"name": "Alice", "age": 25}
print('Dictionary:', my_dictionary_var) # Dictionary: {'name': 'Alice', 'age': 25}
```

- **صف مرتب (Tuple)**: مجموعة مرتبة غير قابلة للتغيير، محاطة بأقواس عادية:

```py
my_tuple_var = (7, 5, 8)
print('Tuple:', my_tuple_var) # Tuple: (7, 5, 8)
```

- **نطاق (Range)**: متتالية من الأعداد، تُستخدم غالبًا في الحلقات:

```py
my_range_var = range(5)
print(my_range_var) # range(0, 5)
```

- **قائمة (List)**: مجموعة مرتبة من العناصر تدعم أنواع بيانات مختلفة:

```py
my_list = [22, 'Hello world', 3.14, True]
print(my_list) # [22, 'Hello world', 3.14, True]
```

- **None**: قيمة خاصة تمثل غياب القيمة:

```py
my_none_var = None
print('None:', my_none_var) # None: None
```

## الأنواع الثابتة والمتغيرة

- **الأنواع الثابتة**: هذه الأنواع لا تتغير بعد إعلانها، رغم أنه يمكنك إعادة تعيين المتغير ليشير إلى شيء جديد. تشمل الأعداد الصحيحة، العشرية، المركبة، القيمة المنطقية، السلسلة النصية، الصف المرتب، النطاق، و`None`.
- **الأنواع المتغيرة**: هذه الأنواع يمكن تغييرها بعد إعلانها. يمكنك إضافة، إزالة، أو تحديث عناصرها. تشمل أنواع المجموعات مثل القائمة، المجموعة، والقاموس.
- **دالة `type()`**: لرؤية نوع متغير، يمكنك استخدام الدالة `type()` هكذا:

```py
greeting = 'Hello there!'
age = 21

print(type(greeting)) # <class 'str'>
print(type(age)) # <class 'int'>
```

- **دالة `isinstance()`**: تُستخدم للتحقق مما إذا كان المتغير يطابق نوع بيانات معين:

```py
greeting = 'Hello world'
name = 'John Doe'

print(isinstance(greeting, str)) # True
print(isinstance(name, int)) # False
```

## العمل مع السلاسل النصية

- **التعريف**: كما تتذكر من جافاسكريبت، السلاسل النصية ثابتة، مما يعني أنك لا تستطيع تغييرها بعد إنشائها. في بايثون، يمكنك استخدام علامات اقتباس مفردة أو مزدوجة. يُنصح باختيار قاعدة والالتزام بها:

```py
developer = 'Jessica'
city = "Los Angeles"
```

- **الوصول إلى الأحرف في السلاسل النصية**: يمكنك الوصول إلى الأحرف باستخدام ترميز الأقواس هكذا:

```py
my_str = 'Hello world'

print(my_str[0])  # H
print(my_str[6])  # w

print(my_str[-1])  # d
print(my_str[-2]) # l
```

- **الهروب في السلاسل النصية**: يمكنك استخدام الشرطة المائلة العكسية (`\`) إذا كانت السلسلة تحتوي على علامات اقتباس هكذا:

```py
msg = 'It\'s a sunny day'
quote = "She said, \"Hello!\""
```

- **ربط السلاسل النصية**: لربط السلاسل النصية، يمكنك استخدام معامل `+` هكذا:

```py
developer = 'Jessica'
print('My name is ' + developer + '.') # My name is Jessica.
```

طريقة أخرى لربط السلاسل النصية هي باستخدام معامل `+=`. يُستخدم هذا لأداء الربط والتعيين في خطوة واحدة هكذا:

```py
greeting = 'My name is '
developer = 'Jessica.'

greeting += developer
print(greeting) # My name is Jessica.
```

- **`f-strings`**: اختصار لـ formatted string literals. تتيح لك التعامل مع الاستيفاء وأيضًا إجراء بعض الربط بصيغة مختصرة وقابلة للقراءة:

```py
developer = 'Jessica'
greeting = f'My name is {developer}.'
print(greeting) # My name is Jessica.
```

- **تقطيع السلاسل النصية**: هو استخراج أجزاء من السلسلة النصية. الصيغة الأساسية هي:

```py
str[start:stop:step]
```

تمثل نقطة البداية الفهرس الذي يبدأ منه الاستخراج. نقطة النهاية هي حيث ينتهي التقطيع، وهذه النقطة غير شاملة. تمثل خطوة التقطيع مقدار الزيادة بين الفهارس. إليك بعض الأمثلة:

```py
message = 'Python is fun!'

print(message[0:6])  # Python
print(message[7:])  # is fun!
print(message[::2])  # Pto sfn
```

- **الحصول على طول السلسلة النصية**: تُستخدم الدالة `len()` لإرجاع عدد الأحرف في السلسلة:

```py
developer = 'Jessica'

print(len(developer)) # 7
```

## العمل مع معامل `in`

- **معامل `in`**: يُرجع قيمة منطقية تحدد ما إذا كان الحرف أو الأحرف موجودة في السلسلة النصية أم لا:

```py
my_str = 'Hello world'

print('Hello' in my_str)  # True
print('hey' in my_str)    # False
print('hi' in my_str)    # False
print('e' in my_str)  # True
print('f' in my_str)  # False
```

## طرق السلاسل النصية الشائعة

- **`upper()`**: تُرجع نسخة جديدة من السلسلة النصية مع تحويل جميع الأحرف إلى أحرف كبيرة:

```py
developer = 'Jessica'

print(developer.upper()) # JESSICA
```

- **`lower()`**: تُرجع نسخة جديدة من السلسلة النصية مع تحويل جميع الأحرف إلى أحرف صغيرة:

```py
developer = 'Jessica'

print(developer.lower()) # jessica
```

- **`strip()`**: تُرجع نسخة من السلسلة النصية مع إزالة الأحرف المحددة من البداية والنهاية (إذا لم يُمرر أي معطى، تزيل الفراغات من البداية والنهاية).

```py
greeting = '  hello world  '

trimmed_my_str = greeting.strip()
print(trimmed_my_str)  # 'hello world'
```

- **`replace()`**: تُرجع سلسلة جديدة مع استبدال كل تكرارات السلسلة القديمة بسلسلة جديدة.

```py
greeting = 'hello world'

replaced_my_str = greeting.replace('hello', 'hi')
print(replaced_my_str)  # 'hi world'
```

- **`split()`**: تُستخدم لتقسيم سلسلة نصية إلى قائمة باستخدام فاصل محدد. الفاصل هو سلسلة تحدد مكان التقسيم.

```py
dashed_name = 'example-dashed-name'

split_words = dashed_name.split('-')
print(split_words)  # ['example', 'dashed', 'name']
```

- **`join()`**: تُستخدم لربط عناصر قابلة للتكرار في سلسلة نصية باستخدام فاصل. القابل للتكرار هو مجموعة من العناصر يمكن التكرار عليها مثل القائمة، السلسلة النصية، أو الصف المرتب.

```py
example_list = ['example', 'dashed', 'name']

joined_str = ' '.join(example_list)
print(joined_str)  # example dashed name
```

- **`startswith(prefix)`**: تُرجع قيمة منطقية تشير إلى ما إذا كانت السلسلة تبدأ بالبادئة المحددة:

```py
developer = 'Naomi'

result = developer.startswith('N')
print(result)  # True
```

- **`endswith(suffix)`**: تُرجع قيمة منطقية تشير إلى ما إذا كانت السلسلة تنتهي باللاحقة المحددة:

```py
developer = 'Naomi'

result = developer.endswith('N')
print(result)  # False
```

- **`find()`**: تُرجع الفهرس لأول ظهور لسلسلة فرعية. إذا لم تُعثر عليها، تُرجع `-1`:

```py
developer = 'Naomi'

result = developer.find('N')
print(result)  # 0

city = 'Los Angeles'
print(city.find('New')) # -1
```

- **`count(substring)`**: تحصي عدد مرات ظهور سلسلة فرعية في السلسلة النصية:

```py
city = 'Los Angeles'
print(city.count('e')) # 2
```

- **`capitalize()`**: تُرجع سلسلة جديدة مع تحويل الحرف الأول إلى كبير وباقي الأحرف إلى صغيرة:

```py
dessert = 'chocolate cake'
print(dessert.capitalize()) # Chocolate cake
```

- **`isupper()`**: تُرجع `True` إذا كانت كل الأحرف في السلسلة كبيرة و`False` خلاف ذلك:

```py
dessert = 'chocolate cake'
print(dessert.isupper()) # False
```

- **`islower()`**: تُرجع `True` إذا كانت كل الأحرف في السلسلة صغيرة و`False` خلاف ذلك:

```py
dessert = 'chocolate cake'
print(dessert.islower()) # True
```

- **`title()`**: تُرجع سلسلة جديدة مع تحويل أول حرف من كل كلمة إلى كبير:

```py
city = 'los angeles'
print(city.title()) # Los Angeles
```

- **`maketrans()`**: تُستخدم لإنشاء جدول تحويل حرف إلى حرف. غالبًا ما تُستخدم مع طريقة `translate()` التي تطبق هذا الجدول على سلسلة نصية وتُرجع النتيجة المترجمة.

```py
trans_table = str.maketrans('abc', '123')
print(trans_table) # {97: 49, 98: 50, 99: 51}

result = 'abcabc'.translate(trans_table)
print(result)  # 123123
```

## العمليات الشائعة مع الأعداد الصحيحة والعشرية

- **العمليات الحسابية الأساسية**: في بايثون، يمكنك إجراء عمليات حسابية أساسية مع الأعداد الصحيحة والعشرية مثل الجمع، الطرح، الضرب، والقسمة:

```py
int_1 = 56
int_2 = 12
float_1 = 5.4
float_2 = 12.0

# Addition

print('Integer Addition:', int_1 + int_2) # Integer Addition: 68
print('Float Addition:', float_1 + float_2) # Float Addition: 17.4

# Subtraction

print('Int Subtraction:', int_1 - int_2) # Int Subtraction: 44
print('Float Subtraction:',  float_2 - float_1) # Float Subtraction: 6.6

# Multiplication

print('Int Multiplication:', int_1 * int_2) # Int Multiplication: 672
print('Float Multiplication:', float_2 * float_1) # Float Multiplication: 64.80000000000001

# Division

print('Division:', int_1 / int_2) # Division: 4.666666666666667
print('Float Division:', float_2 / float_1) # Float Division: 2.222222222222222
```

عند جمع عدد عشري وعدد صحيح، تتحول النتيجة إلى عدد عشري هكذا:

```py
int_1 = 56
float_1 = 5.4

print(int_1 + float_1) # 61.4
```

- **معامل باقي القسمة (`%`)**: يُرجع الباقي عند قسمة رقم على رقم آخر:

```py
int_1 = 56
int_2 = 12

print(int_1 % int_2) # 8
```

- **القسمة الأرضية (`//`)**: يُستخدم لقسمة رقمين وتقريب النتيجة إلى أقرب عدد صحيح أقل أو يساوي الناتج:

```py
int_1 = 56
int_2 = 12

print(int_1 // int_2) # 4
```

- **معامل الأس (`**`)**: يُستخدم لرفع رقم إلى قوة رقم آخر:

```py
int_1 = 4
int_2 = 2

print(int_1 ** int_2) # 16
```

- **دالة `float()`**: يمكنك استخدام هذه الدالة لتحويل عدد صحيح إلى عدد عشري.

```py
num = 4

print(float(num)) # 4.0
```

- **دالة `int()`**: يمكنك استخدام هذه الدالة لتحويل عدد عشري إلى عدد صحيح.

```py
num = 4.0

print(int(num)) # 4
```

- **دالة `round()`**: تُستخدم لتقريب رقم إلى أقرب عدد صحيح:

```py
num_1 = 3.4
num_2 = 7.7

print(round(num_1)) # 3
print(round(num_2)) # 8
```

- **دالة `abs()`**: تُستخدم لإرجاع القيمة المطلقة لرقم:

```py
num = -13

print(abs(num)) # 13
```

- **دالة `pow()`**: تُستخدم لرفع رقم إلى قوة رقم آخر:

```py
result = pow(2, 3) 
print(result)  # 8
```

## التعيينات المعززة

- **التعريف**: التعيين المعزز يجمع بين عملية ثنائية مع التعيين في خطوة واحدة. يأخذ متغيرًا، يطبق عليه عملية مع قيمة أخرى، ويخزن النتيجة في نفس المتغير.

```py
# Addition assignment 
my_var = 10
my_var += 5

print(my_var) # 15

# Subtraction assignment
count = 14
count -= 3

print(count) # 11

# Multiplication assignment 
product = 65
product *= 7

print(product) # 455

# Division assignment 
price = 100
price /= 4

print(price) # 25.0

# Floor Division assignment 
total_pages = 23
total_pages //= 5

print(total_pages) # 4

# Modulo assignment 
bits = 35
bits %= 2

print(bits) # 1

# Exponentiation assignment 
power = 2
power **= 3

print(power) # 8
```

هناك معاملات تعيين معززة أخرى، مثل تلك الخاصة بالعمليات على مستوى البت. تشمل `&=`, `^=`, `>>=`, و`<<=`.

## العمل مع الدوال

- **التعريف**: الدوال هي قطع قابلة لإعادة الاستخدام من الشفرة تأخذ مدخلات (معلمات) وتُرجع مخرجًا. لاستدعاء دالة، تحتاج إلى الإشارة إلى اسم الدالة متبوعًا بمجموعة من الأقواس العادية:

```py
# Defining a function

def get_sum(num_1, num_2):
    return num_1 + num_2

result = get_sum(3, 4) # function call
print(result) # 7
```

إذا لم تُرجع الدالة قيمة صراحة، فإن قيمة الإرجاع الافتراضية هي `None`:

```py
def greet():
    print('hello') 

result = greet() # hello
print(result) # None
```

يمكنك أيضًا تزويد المعلمات بقيم افتراضية هكذا:

```py
def get_sum(num_1, num_2=2):
    return num_1 + num_2

result = get_sum(3) 
print(result) # 5
```

إذا استدعيت الدالة بدون العدد الصحيح من المعلمات، ستحصل على `TypeError`:

```py
def calculate_sum(a, b):
    print(a + b)

calculate_sum()

# TypeError: calculate_sum() missing 2 required positional arguments: 'a' and 'b'
```

## الدوال المدمجة الشائعة

- **دالة `input()`**: تُستخدم لطلب مُدخَل من المستخدم:

```py
name = input('What is your name?') # User types 'Kolade' and presses Enter  
print('Hello', name) # Hello Kolade
```

- **دالة `int()`**: تُستخدم لتحويل رقم، قيمة منطقية، أو سلسلة رقمية إلى عدد صحيح:

```py
print(int(3.14)) # 3
print(int('42')) # 42
print(int(True)) # 1
print(int(False)) # 0 
```

## النطاق في بايثون

- **النطاق المحلي**: هو عندما يُعلن متغير داخل دالة أو صنف ولا يمكن الوصول إليه إلا داخل تلك الدالة أو الصنف.

```py
def my_func():
    num = 10
    print(num)
```

- **النطاق المحيط**: هو عندما تكون دالة متداخلة داخل دالة أخرى ويمكنها الوصول إلى متغيرات الدالة التي تحتويها.

```py
def outer_func():
    msg = 'Hello there!'
    
    def inner_func():
        print(msg)
    inner_func()

print(outer_func()) # Hello there!
```

- **النطاق العالمي**: يشير إلى المتغيرات المُعلنة خارج أي دوال أو أصناف ويمكن الوصول إليها من أي مكان في البرنامج.

```py
tax = 0.70 

def get_total(subtotal):
    total = subtotal + (subtotal * tax)
    return total

print(get_total(100))  # 170.0
```

- **النطاق المدمج**: أسماء محجوزة في بايثون للدوال، الوحدات، الكلمات المفتاحية، والكائنات المعرفة مسبقًا.

```py
print(str(45)) # '45'
print(type(3.14)) # <class 'float'>
print(isinstance(3, str)) # False
```

## معاملات المقارنة

- **يساوي (`==`)**: يتحقق مما إذا كانت قيمتان متساويتين:

```py
print(3 == 4) # False
```

- **لا يساوي (`!=`)**: يتحقق مما إذا كانت قيمتان غير متساويتين:

```py
print(3 != 4) # True
```

- **أكبر من بشكل صارم (`>`)**: يتحقق مما إذا كانت قيمة أكبر من أخرى:

```py
print(3 > 4) # False
```

- **أصغر من بشكل صارم (`<`)**: يتحقق مما إذا كانت قيمة أصغر من أخرى:

```py
print(3 < 4) # True
```

- **أكبر من أو يساوي (`>=`)**: يتحقق مما إذا كانت قيمة أكبر من أو تساوي أخرى:

```py
print(3 >= 4) # False
```

- **أصغر من أو يساوي (`<=`)**: يتحقق مما إذا كانت قيمة أصغر من أو تساوي أخرى:

```py
print(3 <= 4) # True
```

## العمل مع عبارات `if`، `elif` و`else`

- **عبارات `if`**: هي شروط تُستخدم لتحديد ما إذا كان شيء ما صحيحًا أم لا. إذا كانت النتيجة `True`، يتم تنفيذ كتلة الشفرة تلك.

```py
age = 18

if age >= 18:
    print('You are an adult') # You are an adult
```

- **بند `elif`**: هذه شروط تأتي بعد جملة `if`. إذا كانت شرط `elif` تُقيّم إلى `True`، فسيتم تنفيذ ذلك الجزء من الشفرة.

```py
age = 16

if age >= 18:
    print('You are an adult')
elif age >= 13:
    print('You are a teenager')  # You are a teenager
```

- **بند `else`**: يتم تنفيذ هذا إذا لم تُقيّم أي شروط أخرى إلى `True`.

```py
age = 12

if age >= 18:
    print('You are an adult')
elif age >= 13:
    print('You are a teenager')
else:
    print('You are a child')  # You are a child
```

يمكنك أيضًا استخدام جمل `if` متداخلة بهذا الشكل:

```py
is_citizen = True
age = 25

if is_citizen:
    if age >= 18:
        print('You are eligible to vote') # You are eligible to vote
else:
    print('You are not eligible to vote')
```

## القيم الصحيحة والخاطئة (Truthy and Falsy)

- **التعريف**: في بايثون، لكل قيمة قيمة منطقية ضمنية، أو إحساس مدمج بها إذا كان يجب التعامل معها كـ `True` أو `False` في سياق منطقي. العديد من القيم تُعتبر صحيحة (truthy)، أي تُقيّم إلى `True` في السياق المنطقي. والبعض الآخر يُعتبر خاطئًا (falsy)، أي يُقيّم إلى `False`. إليك بعض الأمثلة على القيم الخاطئة:

```md
None
False
Integer 0
Float 0.0
Empty strings ''
```

القيم الأخرى مثل الأعداد غير الصفرية والسلاسل النصية غير الفارغة تُعتبر صحيحة (truthy).

## العمل مع الدالة `bool()`

- **التعريف**: إذا أردت التحقق مما إذا كانت القيمة صحيحة أو خاطئة، يمكنك استخدام الدالة المدمجة `bool()`. تقوم بتحويل القيمة صراحةً إلى مكافئها المنطقي وتُرجع `True` للقيم الصحيحة و`False` للقيم الخاطئة. إليك بعض الأمثلة:

```py
print(bool(False)) # False
print(bool(0))  # False
print(bool('')) # False

print(bool(True)) # True
print(bool(1)) # True
print(bool('Hello')) # True
```

## عوامل القيم المنطقية والقطع المبكر (Short-circuiting)

- **التعريف**: هذه عوامل خاصة تسمح لك بدمج عدة تعبيرات لإنشاء منطق قرار أكثر تعقيدًا في شفرتك. هناك ثلاثة عوامل منطقية في بايثون: `and`، `or`، و`not`.
- **عامل `and`**: يأخذ عاملين ويُرجع العامل الأول إذا كان خاطئًا، وإلا يُرجع العامل الثاني. يجب أن يكون كلا العاملين صحيحين لكي تكون النتيجة صحيحة.

```py
is_citizen = True
age = 25

print(is_citizen and age) # 25
```

يمكنك أيضًا استخدام عامل `and` في الشروط بهذا الشكل:

```py
is_citizen = True
age = 25

if is_citizen and age >= 18:
    print('You are eligible to vote') # You are eligible to vote
else:
    print('You are not eligible to vote')
```

- **عامل `or`**: يُرجع العامل الأول إذا كان صحيحًا، وإلا يُرجع العامل الثاني. تعبير `or` يُنتج قيمة صحيحة إذا كان أحد العاملين على الأقل صحيحًا. إليك مثالًا:

```py
age = 19
is_employed = False

print(age or is_employed) # 19
```

تمامًا كما في عامل `and`، يمكنك استخدام عامل `or` في الشروط بهذا الشكل:

```py
age = 19
is_student = True

if age < 18 or is_student:
    print('You are eligible for a student discount') # You are eligible for a student discount
else:
    print('You are not eligible for a student discount')
```

- **القطع المبكر**: يُعرف عاملَي `and` و`or` بعوامل القطع المبكر. القطع المبكر يعني أن بايثون يتحقق من القيم من اليسار إلى اليمين ويتوقف بمجرد تحديد النتيجة النهائية.
- **عامل `not`**: يأخذ عاملًا واحدًا ويعكس قيمته المنطقية. يحول القيم الصحيحة إلى `False` والقيم الخاطئة إلى `True`. على عكس العوامل السابقة، `not` دائمًا يُرجع `True` أو `False`. إليك بعض الأمثلة:

```py
print(not '') # True, because empty string is falsy
print(not 'Hello') # False, because non-empty string is truthy
print(not 0) # True, because 0 is falsy
print(not 1) # False, because 1 is truthy
print(not False) # True, because False is falsy
print(not True) # False, because True is truthy
```

إليك مثالًا على استخدام عامل `not` في شرط:

```py
is_admin = False

if not is_admin:
    print('Access denied for non-administrators.') # Access denied for non-administrators.
else:
    print('Welcome, Administrator!')
```


## القوائم في بايثون

- **مقدمة**: في بايثون، نوع البيانات القائمة هو متتالية مرتبة من العناصر التي يمكن أن تتكون من سلاسل نصية أو أعداد أو حتى قوائم أخرى. القوائم قابلة للتغيير ومُرقمة بفهرس يبدأ من الصفر.

```python
cities = ['Los Angeles', 'London', 'Tokyo']
```

- **الوصول إلى عناصر القائمة**: للوصول إلى عنصر من قائمة `cities`، يمكنك الإشارة إلى رقم الفهرس الخاص به في المتتالية:

```python
cities = ['Los Angeles', 'London', 'Tokyo']
cities[0] # Los Angeles
```

- **الوصول إلى العناصر باستخدام الفهرسة السالبة**: للوصول إلى العنصر الأخير في أي قائمة، يمكنك استخدام `-1` كرقم الفهرس:

```python
cities = ['Los Angeles', 'London', 'Tokyo']
cities[-1] # Tokyo
```

- تُستخدم الفهرسة السالبة للوصول إلى العناصر بدءًا من نهاية القائمة بدلًا من البداية عند الفهرس `0`.

- **إنشاء القوائم باستخدام دالة البناء `list()`**: يمكن أيضًا إنشاء القوائم باستخدام دالة البناء `list()`. تُستخدم دالة البناء `list()` لتحويل كائن قابل للتكرار إلى قائمة:

```python
developer = 'Jessica'

print(list(developer)) 
# Result: ['J', 'e', 's', 's', 'i', 'c', 'a']
```

- **إيجاد طول القائمة**: يمكنك استخدام الدالة `len()` للحصول على طول القائمة:

```python
numbers = [1, 2, 3, 4, 5]
len(numbers) # 5
```

- **قابلية تعديل القوائم**: القوائم قابلة للتغيير، مما يعني أنه يمكنك تحديث أي عنصر في القائمة طالما أنك تمرر رقم فهرس صالح. لتحديث عنصر في فهرس معين، يمكنك تعيين قيمة جديدة لذلك الفهرس:

```python
programming_languages = ['Python', 'Java', 'C++', 'Rust']
programming_languages[0] = 'JavaScript'
print(programming_languages) # ['JavaScript', 'Java', 'C++', 'Rust']
```

- **خطأ تجاوز الفهرس**: إذا مررت برقم فهرس (موجب أو سالب) خارج حدود القائمة، فستتلقى خطأ `IndexError`:

```python
programming_languages = ['Python', 'Java', 'C++', 'Rust']
programming_languages[10] = 'JavaScript'

"""
Traceback (most recent call last):
  File "<stdin>", line 1, in <module>
IndexError: list assignment index out of range
"""
```

- **إزالة عناصر من القائمة**: يمكن إزالة العناصر من القائمة باستخدام الكلمة المفتاحية `del`:

```python
developer = ['Jane Doe', 23, 'Python Developer']
del developer[1]
print(developer) # ['Jane Doe', 'Python Developer']
```

- **التحقق من وجود عنصر في القائمة**: يمكن استخدام الكلمة المفتاحية `in` للتحقق مما إذا كان عنصر موجودًا في القائمة:

```py
programming_languages = ['Python', 'Java', 'C++', 'Rust']

'Rust' in programming_languages # True
'JavaScript' in programming_languages # False
```

- **تداخل القوائم**: يمكن أن تحتوي القوائم على قوائم أخرى بداخلها:

```python
developer = ['Alice', 25, ['Python', 'Rust', 'C++']]
```

- للوصول إلى القائمة المتداخلة، تحتاج إلى الوصول إليها باستخدام الفهرس `2` لأن القوائم مُرقمة بفهرس يبدأ من الصفر.

```python
developer = ['Alice', 25, ['Python', 'Rust', 'C++']]
developer[2] # ['Python', 'Rust', 'C++']
```

- للوصول إلى اللغة الثانية من تلك القائمة المتداخلة، تحتاج إلى الوصول إليها باستخدام الفهرس `1`:

```python
developer = ['Alice', 25, ['Python', 'Rust', 'C++']]
developer[2][1] # Rust
```

- **فك تجميع القيم من قائمة:** فك التجميع هو تقنية تُستخدم لتعيين قيم من قائمة إلى متغيرات جديدة. إليك مثالًا لفك تجميع قائمة `developer` إلى متغيرات جديدة تسمى `name` و`age` و`job` بهذا الشكل:

```py
developer = ['Alice', 34, 'Rust Developer']
name, age, job = developer
```

- إذا لم يتطابق عدد المتغيرات على الجانب الأيسر من معامل التعيين مع عدد العناصر في القائمة، فستتلقى خطأ `ValueError`.

- **جمع العناصر المتبقية من القائمة**: لجمع أي عناصر متبقية من القائمة، يمكنك استخدام عامل النجمة (`*`) بهذا الشكل:

```py
developer = ['Alice', 34, 'Rust Developer']
name, *rest = developer
```

- **تقطيع القوائم**: التقطيع هو مفهوم الوصول إلى جزء من القائمة باستخدام عامل التقطيع `:`. لتقطيع قائمة تبدأ عند الفهرس `1` وتنتهي قبل الفهرس `3`، يمكنك استخدام الصيغة التالية:

```python
desserts = ['Cake', 'Cookies', 'Ice Cream', 'Pie']
desserts[1:3] # ['Cookies', 'Ice Cream']
```

- **فواصل الخطوات**: من الممكن أيضًا تحديد فاصل خطوة يحدد مقدار الزيادة بين الفهارس. إليك مثالًا إذا أردت استخراج قائمة من الأعداد الزوجية فقط باستخدام التقطيع:

```python
numbers = [1, 2, 3, 4, 5, 6]
numbers[1::2] # [2, 4, 6]
```

## طرق القوائم

- **append()**: تُستخدم لإضافة عنصر إلى نهاية القائمة. إليك مثالًا على استخدام طريقة `append()` لإضافة الرقم `6` إلى قائمة `numbers` هذه:

```py
numbers = [1, 2, 3, 4, 5]
numbers.append(6)
print(numbers) # [1, 2, 3, 4, 5, 6]
```

- **إضافة قوائم**: يمكن أيضًا استخدام طريقة `append()` لإضافة قائمة واحدة إلى نهاية أخرى:

```py
numbers = [1, 2, 3, 4, 5]
even_numbers = [6, 8, 10]

numbers.append(even_numbers)
print(numbers) # [1, 2, 3, 4, 5, [6, 8, 10]]
```

- **extend()**: تُستخدم لإضافة عدة عناصر إلى نهاية القائمة. إليك مثالًا لإضافة الأرقام `6` و`8` و`10` إلى نهاية قائمة `numbers`:

```py
numbers = [1, 2, 3, 4, 5]
even_numbers = [6, 8, 10]

numbers.extend(even_numbers)
print(numbers) # [1, 2, 3, 4, 5, 6, 8, 10]
```

- **insert()**: تُستخدم لإدخال عنصر في فهرس محدد في القائمة. إليك مثالًا على استخدام طريقة `insert()`:

```py
numbers = [1, 2, 3, 4, 5]
numbers.insert(2, 2.5)

print(numbers) # [1, 2, 2.5, 3, 4, 5]
```

- **remove():** تُستخدم لإزالة عنصر من القائمة. طريقة `remove()` تزيل فقط أول ظهور للعنصر في القائمة:

```py
numbers = [1, 2, 3, 4, 5, 5, 5]
numbers.remove(5)

print(numbers) # [1, 2, 3, 4, 5, 5]

```

- **pop()**: تُستخدم لإزالة عنصر محدد من القائمة وإرجاعه:

```py
numbers = [1, 2, 3, 4, 5]
numbers.pop(1) # The number 2 is returned
```

- إذا لم تحدد عنصرًا لطريقة `pop`، فسيتم إزالة العنصر الأخير.

```py
numbers = [1, 2, 3, 4, 5]
numbers.pop() # The number 5 is returned
```

- **clear()**: تُستخدم لإزالة كل العناصر من القائمة:

```py
numbers = [1, 2, 3, 4, 5]
numbers.clear()

print(numbers) # []
```

- **sort()**: تُستخدم طريقة `sort()` لترتيب العناصر في مكانها. إليك مثالًا لترتيب قائمة عشوائية من `numbers` في مكانها:

```py
numbers = [19, 2, 35, 1, 67, 41]
numbers.sort()

print(numbers) # [1, 2, 19, 35, 41, 67]
```

- **sorted()**: دالة مدمجة تُرجع قائمة مرتبة جديدة بدلًا من تعديل القائمة الأصلية:

```py
numbers = [19, 2, 35, 1, 67, 41]
sorted_numbers = sorted(numbers)

print(sorted_numbers) # [1, 2, 19, 35, 41, 67]
print(numbers) # [19, 2, 35, 1, 67, 41]
```

- **reverse()**: تُستخدم لعكس ترتيب العناصر في القائمة:

```py
numbers = [6, 5, 4, 3, 2, 1]
numbers.reverse()

print(numbers) # [1, 2, 3, 4, 5, 6]
```

- **index()**: تُستخدم لإيجاد أول فهرس يمكن العثور فيه على عنصر في القائمة:

```python
programming_languages = ['Rust', 'Java', 'Python', 'C++']
programming_languages.index('Java') # 1
```

- إذا لم يُعثر على العنصر باستخدام طريقة `index()`، فستكون النتيجة خطأ `ValueError`.

## الصفوف المرتبة (Tuples) في بايثون

- **التعريف**: الصف المرتب هو نوع بيانات في بايثون يُستخدم لإنشاء متتالية مرتبة من القيم. يمكن أن تحتوي الصفوف المرتبة على مجموعة مختلطة من أنواع البيانات:

```py
developer = ('Alice', 34, 'Rust Developer')
```

- الصفوف المرتبة غير قابلة للتغيير، مما يعني أن العناصر داخل الصف لا يمكن تعديلها بعد إنشائها. إذا حاولت تحديث أحد العناصر في الصف، ستحصل على خطأ `TypeError`:

```py
programming_languages = ('Python', 'Java', 'C++', 'Rust')
programming_languages[0] = 'JavaScript'

"""
Traceback (most recent call last):
  File "<stdin>", line 1, in <module>
TypeError: "tuple" object does not support item assignment
"""
```

- **الوصول إلى عناصر من صف مرتب**: للوصول إلى عنصر من صف مرتب، استخدم ترميز الأقواس ورقم الفهرس:

```py
developer = ('Alice', 34, 'Rust Developer')
developer[1] # 34
```

- يمكن استخدام الفهرسة السالبة للوصول إلى العناصر بدءًا من نهاية الصف المرتب:

```py
numbers = (1, 2, 3, 4, 5)
numbers[-2] # 4
```

- إذا حاولت تمرير رقم فهرس يتجاوز أو يساوي طول الصف المرتب، فستتلقى خطأ `IndexError`:

```py
numbers = (1, 2, 3, 4, 5)
numbers[7]

"""
Traceback (most recent call last):
  File "<stdin>", line 1, in <module>
IndexError: tuple index out of range
"""
```

- يمكن أيضًا إنشاء صف مرتب باستخدام دالة البناء `tuple()`. داخل دالة البناء، يمكنك تمرير كائنات قابلة للتكرار مختلفة مثل السلاسل النصية والقوائم وحتى صفوف مرتبة أخرى.

```python
developer = 'Jessica'

print(tuple(developer)) 
# Result: ('J', 'e', 's', 's', 'i', 'c', 'a')
```

- **التحقق من وجود عناصر في صف مرتب**: للتحقق مما إذا كان عنصر موجودًا في صف مرتب، يمكنك استخدام الكلمة المفتاحية `in` بهذا الشكل:

```py
programming_languages = ('Python', 'Java', 'C++', 'Rust')

'Rust' in programming_languages # True
'JavaScript' in programming_languages # False
```

- **فك تجميع الصفوف المرتبة**: يمكن فك تجميع العناصر من صف مرتب بهذا الشكل:

```py
developer = ('Alice', 34, 'Rust Developer')
name, age, job = developer
```

- إذا كنت بحاجة إلى جمع أي عناصر متبقية من صف مرتب، يمكنك استخدام عامل النجمة (`*`) بهذا الشكل:

```py
developer = ('Alice', 34, 'Rust Developer')
name, *rest = developer
```

- **تقطيع الصفوف المرتبة**: يمكن استخدام التقطيع لاستخراج جزء من صف مرتب. على سبيل المثال، يمكن تقطيع العناصر `pie` و`cookies` إلى صف مرتب منفصل:

```py
desserts = ('cake', 'pie', 'cookies', 'ice cream')
desserts[1:3] # ('pie', 'cookies')
```

- **إزالة عناصر من الصفوف المرتبة**: محاولة إزالة عنصر من صف مرتب ستؤدي إلى رفع خطأ `TypeError` لأن الصفوف المرتبة غير قابلة للتغيير:

```py
developer = ('Jane Doe', 23, 'Python Developer')
del developer[1]

"""
Traceback (most recent call last):
  File "<stdin>", line 1, in <module>
TypeError: "tuple" object doesn't support item deletion
"""
```

- **متى تستخدم صف مرتب مقابل قائمة؟**: إذا كنت تحتاج إلى مجموعة ديناميكية من العناصر حيث يمكنك الإضافة والإزالة والتحديث، فاستخدم قائمة. إذا كنت تعرف أنك تعمل مع مجموعة ثابتة وغير قابلة للتغيير من البيانات، فاستخدم صف مرتب.

## طرق الصفوف المرتبة الشائعة

- **`count()`**: تُستخدم لمعرفة عدد مرات ظهور عنصر في صف مرتب. على سبيل المثال، يمكنك التحقق من عدد مرات ظهور اللغة `'Rust'` في الصف المرتب:

```py
programming_languages = ('Rust', 'Java', 'Python', 'C++', 'Rust')
programming_languages.count('Rust') # 2
```

- إذا لم يكن العنصر المحدد في طريقة `count()` موجودًا على الإطلاق في الصف المرتب، فستكون قيمة الإرجاع `0`:

```py
programming_languages = ('Rust', 'Java', 'Python', 'C++', 'Rust')
programming_languages.count('JavaScript') # 0
```

- إذا لم يتم تمرير أي معلمات إلى طريقة `count()`، فستُرجع بايثون خطأ `TypeError`.

- **index()**: تُستخدم لإيجاد الفهرس الذي يوجد فيه عنصر معين في الصف المرتب. إليك مثالًا على استخدام طريقة `index()` لإيجاد الفهرس للغة `'Java'`:

```py
programming_languages = ('Rust', 'Java', 'Python', 'C++', 'Rust')
programming_languages.index('Java') # 1
```

- إذا لم يُعثر على العنصر المحدد، فستُرجع بايثون خطأ `ValueError`.

- يمكنك تمرير فهرس بداية اختياري إلى طريقة `index()` لتحديد مكان بدء البحث عن العنصر في الصف المرتب:

```py
programming_languages = ('Rust', 'Java', 'Python', 'C++', 'Rust', 'Python')
programming_languages.index('Python', 3) # 5
```

- يمكنك أيضًا تمرير فهرس نهاية اختياري إلى طريقة `index()` لتحديد مكان إيقاف البحث عن العنصر في الصف المرتب:

```py
programming_languages = ('Rust', 'Java', 'Python', 'C++', 'Rust', 'Python', 'JavaScript', 'Python')
programming_languages.index('Python', 2, 5) # 2
```

- **`sorted()`**: تُستخدم لترتيب العناصر في أي كائن قابل للتكرار وإرجاع قائمة مرتبة جديدة. إليك مثالًا على إنشاء قائمة جديدة من الأرقام باستخدام دالة `sorted()`:

```py
numbers = (13, 2, 78, 3, 45, 67, 18, 7)
sorted(numbers) # [2, 3, 7, 13, 18, 45, 67, 78]
```

- **تعديل سلوك الترتيب**: يمكنك تخصيص سلوك الترتيب لكائن قابل للتكرار باستخدام المعاملات الاختيارية `reverse` و`key`. إليك مثالًا على استخدام معامل `key` لترتيب عناصر في صف مرتب حسب الطول:

```py
programming_languages = ('Rust', 'Java', 'Python', 'C++', 'Rust', 'Python')
sorted(programming_languages, key=len)

# Result
# ['C++', 'Rust', 'Java', 'Rust', 'Python', 'Python']
```

- يمكنك إنشاء قائمة جديدة من القيم بترتيب عكسي باستخدام معامل `reverse` بهذا الشكل:

```py
programming_languages = ('Rust', 'Java', 'Python', 'C++', 'Rust', 'Python')

print(sorted(programming_languages, reverse=True))

# Result
# ['Rust', 'Rust', 'Python', 'Python', 'Java', 'C++']
```

## الحلقات في بايثون

- **التعريف**: تُستخدم الحلقات لتكرار كتلة من الشفرة لعدد معين من المرات.

- **حلقة `for`**: تُستخدم للتكرار على متتالية (مثل قائمة، صف مرتب، أو سلسلة نصية) وتنفيذ كتلة من الشفرة لكل عنصر في تلك المتتالية. إليك مثالًا على استخدام حلقة `for` للتكرار خلال قائمة وطباعة كل لغة إلى وحدة التحكم:

```py
programming_languages = ['Rust', 'Java', 'Python', 'C++']

for language in programming_languages:
    print(language)

"""
Result 

Rust
Java
Python
C++
"""
```

- إليك مثالًا على استخدام حلقة `for` للتكرار خلال السلسلة النصية `code` وطباعة كل حرف:

```py
for char in 'code':
    print(char)

"""
Result 

c
o
d
e
"""
```

- يمكن تداخل حلقات `for`. إليك مثالًا على استخدام حلقة `for` متداخلة:

```py
categories = ['Fruit', 'Vegetable']
foods = ['Apple', 'Carrot', 'Banana']

for category in categories:
    for food in foods:
        print(category, food)

"""
Result

Fruit Apple
Fruit Carrot
Fruit Banana
Vegetable Apple
Vegetable Carrot
Vegetable Banana
"""

```

- **حلقة `while`**: تكرر كتلة من الشفرة حتى يصبح الشرط `False`. إليك مثالًا على استخدام حلقة `while` للعبة تخمين:

```py
secret_number = 3
guess = 0

while guess != secret_number:
    guess = int(input('Guess the number (1-5): '))
    if guess != secret_number:
        print('Wrong! Try again.')

print('You got it!')

"""
Result

Guess the number (1-5): 2
Wrong! Try again.
Guess the number (1-5): 1
Wrong! Try again.
Guess the number (1-5): 3
You got it!
"""

```

- **جملتي `break` و`continue`**: تُستخدمان في الحلقات لتعديل تنفيذ الحلقة.

- تُستخدم جملة `break` للخروج من الحلقة فورًا عند تحقق شرط معين. إليك مثالًا على استخدام جملة `break` لقائمة `developer_names`:

```py
developer_names = ['Jess', 'Naomi', 'Tom']

for developer in developer_names:
    if developer == 'Naomi':
        break
    print(developer)
```

- تُستخدم جملة `continue` لتخطي التكرار الحالي والانتقال إلى التكرار التالي من الحلقة. إليك مثالًا على استخدام جملة `continue` بدلًا من `break`:

```py
developer_names = ['Jess', 'Naomi', 'Tom']

for developer in developer_names:
    if developer == 'Naomi':
        continue
    print(developer)
```

- يمكن دمج كل من حلقات `for` و`while` مع بند `else`، الذي يُنفذ فقط عندما لا يتم إنهاء الحلقة بواسطة `break`:

```python
words = ['sky', 'apple', 'rhythm', 'fly', 'orange']

for word in words:
    for letter in word:
        if letter.lower() in 'aeiou':
            print(f"'{word}' contains the vowel '{letter}'")
            break
    else:
        print(f"'{word}' has no vowels")
```

## الدوال النطاقية (Ranges) واستخدامها في الحلقات

- **دالة `range()`**: تُستخدم لتوليد متتالية من الأعداد الصحيحة.

```py
range(start, stop, step)
```

- المعلمة المطلوبة `stop` هي عدد صحيح (غير شامل) يمثل نقطة النهاية لمتتالية الأعداد التي يتم توليدها. إليك مثالًا على استخدام دالة `range()`:

```py
for num in range(3):
    print(num)
```

- إذا لم يتم تحديد معلمة `start`، فالقيمة الافتراضية ستكون `0`. بشكل افتراضي، ستزداد متتالية الأعداد الصحيحة بمقدار `1`. يمكنك استخدام المعلمة الاختيارية `step` لتغيير قيمة الزيادة الافتراضية. إليك مثالًا لتوليد متتالية من الأعداد الزوجية من 2 حتى 10 (شاملة):

```py
for num in range(2, 11, 2):
    print(num)
```

- إذا لم تقدم أي معلمات إلى دالة `range()`، فستتلقى خطأ `TypeError`.

- تقبل دالة `range()` الأعداد الصحيحة فقط كمعلمات وليس الأعداد العشرية. استخدام الأعداد العشرية يؤدي أيضًا إلى خطأ `TypeError`:

```py
ERROR!
Traceback (most recent call last):
  File "<main.py>", line 1, in <module>
TypeError: 'float' object cannot be interpreted as an integer
```

- يمكنك استخدام عدد صحيح سالب للمعلمة `step` لتوليد متتالية من الأعداد تناقصيًا:

```py
for num in range(40, 0, -10):
    print(num)
```

- يمكن أيضًا استخدام الدالة `range()` لإنشاء قائمة من الأعداد الصحيحة باستخدام دالة البناء `list`. تُستخدم دالة البناء `list` لتحويل كائن قابل للتكرار إلى قائمة. إليك مثالًا على إنشاء قائمة من الأعداد الزوجية بين 2 و10 شاملة:

```py
numbers = list(range(2, 11, 2))
print(numbers) # [2, 4, 6, 8, 10]
```

## دالتي `enumerate()` و`zip()` في بايثون

- **`enumerate()`**: تُستخدم للتكرار على متتالية مع تتبع الفهرس لكل عنصر في تلك المتتالية. تأخذ دالة `enumerate()` كائنًا قابلًا للتكرار كمعلمة وتُرجع كائن `enumerate` يتكون من الفهرس والقيمة لكل عنصر في الكائن القابل للتكرار.

```python
languages = ['Spanish', 'English', 'Russian', 'Chinese']

for index, language in enumerate(languages):
    print(f'Index {index} and language {language}')

# Result
# Index 0 and language Spanish
# Index 1 and language English
# Index 2 and language Russian
# Index 3 and language Chinese

```

- يمكن أيضًا استخدام دالة `enumerate()` خارج حلقة `for`:

```python
languages = ['Spanish', 'English', 'Russian', 'Chinese']

print(list(enumerate(languages)))
# [(0, 'Spanish'), (1, 'English'), (2, 'Russian'), (3, 'Chinese')]
```

- تقبل دالة `enumerate()` أيضًا معلمة اختيارية `start` تحدد القيمة الابتدائية للعد. إذا تم حذف هذه المعلمة، يبدأ العد من `0`.

-  **`zip()`** : تُستخدم للتكرار على عدة كائنات قابلة للتكرار بالتوازي. إليك مثالًا على استخدام دالة `zip()` للتكرار على `developers` و`ids`:

```python
developers = ['Naomi', 'Dario', 'Jessica', 'Tom']
ids = [1, 2, 3, 4]

for name, id in zip(developers, ids):
    print(f'Name: {name}')
    print(f'ID: {id}')


"""
Result

Name: Naomi
ID: 1
Name: Dario
ID: 2
Name: Jessica
ID: 3
Name: Tom
ID: 4
"""

```

## تعبيرات القوائم في بايثون

- **التعريف**: تعبير القائمة يسمح لك بإنشاء قائمة جديدة في سطر واحد بدمج الحلقة والشرط مباشرة داخل الأقواس المربعة. هذا يجعل الشفرة أقصر وغالبًا أسهل للقراءة.

```py
even_numbers = [num for num in range(21) if num % 2 == 0]
print(even_numbers)
```

## دوال الكائنات القابلة للتكرار

- **`filter()`**: تُستخدم لتصفية العناصر من كائن قابل للتكرار بناءً على شرط. تُرجع مُكرّرًا يحتوي فقط على العناصر التي تحقق الشرط. إليك مثالًا لإنشاء قائمة جديدة من الكلمات التي يزيد طولها عن أربعة أحرف:

```py
words = ['tree', 'sky', 'mountain', 'river', 'cloud', 'sun']

def is_long_word(word):
    return len(word) > 4

long_words = list(filter(is_long_word, words))
print(long_words) # ['mountain', 'river', 'cloud']
```

- **`map()`**: تُستخدم لتطبيق دالة على كل عنصر في كائن قابل للتكرار وإرجاع كائن قابل للتكرار جديد بالنتائج. إليك مثالًا على استخدام دالة `map()` لتحويل قائمة درجات الحرارة من سلسيوس إلى فهرنهايت:

```py
celsius = [0, 10, 20, 30, 40]

def to_fahrenheit(temp):
    return (temp * 9/5) + 32

fahrenheit = list(map(to_fahrenheit, celsius))
print(fahrenheit) # [32.0, 50.0, 68.0, 86.0, 104.0]
```

- **`sum()`**: تُستخدم للحصول على مجموع عناصر قابلة للتكرار مثل القائمة أو الصف المرتب. إليك مثالًا على استخدام الدالة `sum()`:

```py
numbers = [5, 10, 15, 20]
total = sum(numbers)
print(total) # Result: 50
```

- يمكنك أيضًا تمرير معلمة اختيارية `start` التي تحدد القيمة الابتدائية للمجموع. إليك مثالًا محدثًا يستخدم المعلمة `start` كمعلمة موضعية:

```py
numbers = [5, 10, 15, 20]
total = sum(numbers, 10) # positional argument
print(total) # 60
```

- يمكنك أيضًا اختيار استخدام المعلمة `start` كمعلمة مسماة هكذا بدلاً من ذلك:

```py
numbers = [5, 10, 15, 20]
total = sum(numbers, start=10) # keyword argument
print(total) # 60
```

## دوال لامبدا

- **التعريف**: دالة لامبدا في بايثون هي طريقة مختصرة لإنشاء دالة بدون اسم (دالة مجهولة). 
- غالبًا ما تُستخدم دوال لامبدا كمعلمة لدالة أخرى. إليك مثالًا على دالة لامبدا:

```py
numbers = [1, 2, 3, 4, 5]

even_numbers = list(filter(lambda x: x % 2 == 0, numbers))
print(even_numbers)  # [2, 4]
```

- أفضل الممارسات لاستخدام دوال لامبدا تشمل عدم تعيينها لمتغير، والحفاظ عليها بسيطة وقابلة للقراءة، واستخدامها للدوال القصيرة والمؤقتة.

## القواميس

- **القواميس**: القواميس هي هياكل بيانات مدمجة تخزن مجموعات من أزواج المفتاح-القيمة. يجب أن تكون المفاتيح من أنواع بيانات غير قابلة للتغيير. هذا هو الصياغة العامة لقاموس بايثون:

```python
dictionary = {
    key1: value1,
    key2: value2
}
```

- **دالة البناء `dict()`**: دالة البناء `dict()` هي طريقة بديلة لبناء القاموس. تمرر قائمة من الصفوف المرتبة كمعامل إلى دالة البناء `dict()`. تحتوي هذه الصفوف على المفتاح كالعنصر الأول والقيمة كالعنصر الثاني.

```python
pizza = dict([('name', 'Margherita Pizza'), ('price', 8.9), ('calories_per_slice', 250), ('toppings', ['mozzarella', 'basil'])])
```

- **ترميز الأقواس**: للوصول إلى قيمة زوج المفتاح-القيمة، يمكنك استخدام الصياغة المعروفة بترميز الأقواس.

```python
dictionary[key]
```

## طرق القاموس الشائعة

- **طريقة `get()`**: تسترجع طريقة `get()` القيمة المرتبطة بمفتاح. تشبه ترميز الأقواس، لكنها تتيح لك تعيين قيمة افتراضية لتجنب الأخطاء إذا لم يكن المفتاح موجودًا.

```python
dictionary.get(key, default)
```

- **طرائق `keys()` و `values()`**: تُرجع طرائق `keys()` و `values()` كائن عرض يحتوي على جميع المفاتيح والقيم في القاموس، على التوالي. كائن العرض هو طريقة لرؤية محتوى القاموس دون إنشاء نسخة منفصلة من البيانات.

```python
pizza = {
    'name': 'Margherita Pizza',
    'price': 8.9,
    'calories_per_slice': 250
}

pizza.keys()
# dict_keys(['name', 'price', 'calories_per_slice'])

pizza.values()
# dict_values(['Margherita Pizza', 8.9, 250])
```

- **طريقة `items()`**: تُرجع طريقة `items()` كائن عرض يحتوي على جميع أزواج المفتاح-القيمة في القاموس، بما في ذلك المفاتيح والقيم.

```python
pizza.items()
# dict_items([('name', 'Margherita Pizza'), ('price', 8.9), ('calories_per_slice', 250)])
```

- **طريقة `clear()`**: تزيل طريقة `clear()` جميع أزواج المفتاح-القيمة من القاموس.

```python
pizza.clear()
```

- **طريقة `pop()`**: تزيل طريقة `pop()` زوج المفتاح-القيمة الذي يحدد مفتاحه كالمعامل الأول وتُرجع قيمته. إذا لم يكن المفتاح موجودًا، تُرجع القيمة الافتراضية المحددة كالمعامل الثاني. إذا لم يكن المفتاح موجودًا ولم تُحدد قيمة افتراضية، يُثار خطأ `KeyError`.

```python
pizza.pop('price', 10)
pizza.pop('total_price') # KeyError
```

- **طريقة `popitem()`**: في بايثون 3.7 وما بعدها، تزيل طريقة `popitem()` آخر عنصر تم إدخاله.

```python
pizza.popitem()
```

- **طريقة `update()`**: تُحدّث طريقة `update()` أزواج المفتاح-القيمة بأزواج قاموس آخر. إذا كان لديهم مفاتيح مشتركة، تُستبدل قيمها. تُضاف المفاتيح الجديدة إلى القاموس كأزواج جديدة.

```python
pizza.update({ 'price': 15, 'total_time': 25 })
```

## التكرار على قاموس

- **التكرار على القيم**: إذا كنت بحاجة إلى التكرار على القيم في قاموس، يمكنك كتابة حلقة `for` مع `values()` للحصول على جميع قيم القاموس.

```python
products = {
    'Laptop': 990,
    'Smartphone': 600,
    'Tablet': 250,
    'Headphones': 70,
}

for price in products.values():
    print(price)
```

- **التكرار على المفاتيح**: إذا كنت بحاجة إلى التكرار على المفاتيح في قاموس `products` أعلاه، يمكنك كتابة `products.keys()` أو `products` مباشرة.

for product in products.keys():

for product in products.items():

for product, price in products.items():

- **دالة `enumerate()`**: إذا كنت بحاجة إلى التكرار على قاموس مع تتبع عداد، يمكنك استدعاء دالة `enumerate()`. تُرجع الدالة كائن `enumerate` يعين عددًا صحيحًا لكل عنصر، مثل عداد. يمكنك بدء العداد من أي رقم، لكن افتراضيًا يبدأ من 0.

تعيين الفهرس والعنصر إلى متغيرات حلقة منفصلة هو الطريقة الشائعة لاستخدام `enumerate()`. على سبيل المثال، مع `products.items()`، يمكنك الحصول على زوج المفتاح-القيمة بالكامل بالإضافة إلى الفهرس:

for index, product in enumerate(products.items()):

for index, product in enumerate(products.items(), 1):

## المجموعات

- **المجموعات**: المجموعات هي هياكل بيانات مدمجة في بايثون لا تسمح بالقيم المكررة. المجموعات قابلة للتغيير وغير مرتبة، مما يعني أن عناصرها لا تُخزن بترتيب محدد، لذلك لا يمكنك استخدام الفهارس أو المفاتيح للوصول إليها. أيضًا، يمكن للمجموعات أن تحتوي فقط على قيم من أنواع بيانات غير قابلة للتغيير، مثل الأعداد، والسلاسل النصية، والصفوف المرتبة.

- **تعريف مجموعة**: لتعريف مجموعة، تحتاج إلى كتابة عناصرها داخل أقواس متعرجة وفصلها بفواصل.

```python
my_set = {1, 2, 3, 4, 5}
```

- **تعريف مجموعة فارغة**: إذا كنت بحاجة إلى تعريف مجموعة فارغة، يجب عليك استخدام الدالة `set()`. كتابة أقواس متعرجة فارغة فقط ستنشئ قاموسًا تلقائيًا.

```python
set() # Set
{}    # Dictionary
```

## طرق المجموعات الشائعة

- **طريقة `add()`**: يمكنك إضافة عنصر إلى مجموعة باستخدام طريقة `add()`، مع تمرير العنصر الجديد كمعامل.

```python
my_set.add(6)
```

- **طرائق `remove()` و `discard()`**: لإزالة عنصر من مجموعة، يمكنك استخدام طريقة `remove()` أو طريقة `discard()`، مع تمرير العنصر الذي تريد إزالته كمعامل. سترمي طريقة `remove()` خطأ `KeyError` إذا لم يُعثر على العنصر، بينما طريقة `discard()` لن تفعل ذلك.

```python
my_set.remove(4)
my_set.discard(4)
```

- **طريقة `clear()`**: تزيل طريقة `clear()` جميع العناصر من المجموعة.

```python
my_set.clear()
```

## العمليات الرياضية على المجموعات

- **طرائق `issubset()` و `issuperset()`**: تتحقق طرائق `issubset()` و `issuperset()` مما إذا كانت مجموعة ما مجموعة جزئية أو مجموعة شاملة لمجموعة أخرى، على التوالي.

```python
my_set = {1, 2, 3, 4, 5}
your_set = {2, 3, 4, 5}

print(your_set.issubset(my_set)) # True
print(my_set.issuperset(your_set)) # True
```

- **طريقة `isdisjoint()`**: تتحقق طريقة `isdisjoint()` مما إذا كانت مجموعتان منفصلتين، أي لا تشتركان في عناصر.

```python
my_set = {1, 2, 3}
your_set = {4, 5, 6}

print(my_set.isdisjoint(your_set)) # True
```

- **عامل الاتحاد (`|`)**: يعيد عامل الاتحاد `|` مجموعة جديدة تحتوي على جميع العناصر من المجموعتين.

```python
my_set = {1, 2, 3}
your_set = {4, 5, 6}

my_set | your_set # {1, 2, 3, 4, 5, 6}
```

- **عامل التقاطع (`&`)**: يعيد عامل التقاطع `&` مجموعة جديدة تحتوي فقط على العناصر المشتركة بين المجموعتين.

```python
my_set = {1, 2, 3, 4, 5}
your_set = {2, 3, 4, 6}

my_set & your_set # {2, 3, 4}
```

- **عامل الفرق (`-`)**: يعيد عامل الفرق `-` مجموعة جديدة تحتوي على عناصر المجموعة الأولى التي ليست في المجموعة الأخرى.

```python
my_set = {1, 2, 3, 4, 5}
your_set = {2, 3, 4, 6}

my_set - your_set # {1, 5}
```

- **عامل الفرق المتماثل (`^`)**: يعيد عامل الفرق المتماثل `^` مجموعة جديدة تحتوي على العناصر الموجودة في المجموعة الأولى أو الثانية، ولكن ليس في كلتيهما.

```python
my_set = {1, 2, 3, 4, 5}
your_set = {2, 3, 4, 6}

my_set ^ your_set # {1, 5, 6}
```

- **عامل `in`**: يمكنك التحقق مما إذا كان عنصر موجودًا في مجموعة باستخدام عامل `in`.

```python
print(5 in my_set)
```

## مكتبة بايثون القياسية

- **مكتبة بايثون القياسية**: المكتبة توفر لك شفرة مكتوبة مسبقًا وقابلة لإعادة الاستخدام، مثل الدوال، الأصناف، وهياكل البيانات، التي يمكنك إعادة استخدامها في مشاريعك. تحتوي بايثون على مكتبة قياسية واسعة مع وحدات مدمجة تنفذ حلولًا معيارية للعديد من المشاكل والمهام. بعض الأمثلة على الوحدات المدمجة الشهيرة هي `math`، `random`، `re` (اختصار لـ "التعبيرات النمطية"), و `datetime`.

## جملة الاستيراد

- **جملة الاستيراد**: للوصول إلى العناصر المعرفة في الوحدات المدمجة، تستخدم جملة استيراد. تُكتب جمل الاستيراد عادة في أعلى الملف. تعمل جمل الاستيراد بنفس الطريقة للدوال، الأصناف، الثوابت، المتغيرات، وأي عناصر أخرى معرفة في الوحدة.

- **جملة الاستيراد الأساسية**: يمكنك استخدام الكلمة المفتاحية `import` متبوعة باسم الوحدة:

```python
import module_name
```

ثم، إذا كنت بحاجة إلى استدعاء دالة من تلك الوحدة، تستخدم ترميز النقطة، مع اسم الوحدة متبوعًا باسم الدالة.

```python
module_name.function_name()
```

على سبيل المثال، تكتب التالي في شفرتك لاستيراد وحدة `math` والحصول على الجذر التربيعي للعدد 36:

```python
import math

math.sqrt(36)
```

- **استيراد وحدة باسم مختلف**: إذا كنت بحاجة إلى استيراد الوحدة باسم مختلف (المعروف أيضًا باسم "الاسم المستعار"), يمكنك استخدام `as` متبوعًا بالاسم المستعار في نهاية جملة الاستيراد. يُستخدم هذا غالبًا لأسماء الوحدات الطويلة أو لتجنب تعارض الأسماء.

```python
import module_name as module_alias
```

على سبيل المثال، للإشارة إلى وحدة `math` باسم `m` في شفرتك، يمكنك تعيين اسم مستعار هكذا:

```python
import math as m
```

ثم، يمكنك الوصول إلى عناصر الوحدة باستخدام الاسم المستعار:

```python
m.sqrt(36)
```

- **استيراد عناصر محددة**: إذا لم تكن بحاجة إلى كل شيء من الوحدة، يمكنك استيراد عناصر محددة باستخدام `from`. في هذه الحالة، تبدأ جملة الاستيراد بـ `from`، متبوعة باسم الوحدة، ثم الكلمة المفتاحية `import`، وأخيرًا أسماء العناصر التي تريد استيرادها.

```python
from module_name import name1, name2
```

ثم، يمكنك استخدام هذه الأسماء بدون بادئة الوحدة في شفرتك. على سبيل المثال:

```python
from math import radians, sin, cos

angle_degrees = 40
angle_radians = radians(angle_degrees)

sine_value = sin(angle_radians)
cos_value = cos(angle_radians)

print(sine_value) # 0.6427876096865393
print(cos_value)  # 0.766044443118978
```
 
هذا مفيد، لكنه قد يؤدي إلى تعارضات في الأسماء إذا كان لديك دوال أو متغيرات بنفس الاسم. ضع ذلك في اعتبارك عند اختيار نوع جملة الاستيراد التي تريد استخدامها.

إذا كنت بحاجة إلى تعيين أسماء مستعارة لهذه الأسماء، يمكنك فعل ذلك أيضًا باستخدام الكلمة المفتاحية `as` متبوعة بالاسم المستعار.

```python
from module_name import name1 as alias1, name2 as alias2
```

- **جملة الاستيراد مع النجمة (`*`)**: النجمة تخبر بايثون أنك تريد استيراد كل شيء في تلك الوحدة، لكنك تريد استيراده بحيث لا تحتاج إلى استخدام اسم الوحدة كبادئة.

```python
from module_name import *
```

على سبيل المثال، إذا استخدمت هذا لاستيراد وحدة `math`، ستتمكن من استدعاء أي دالة معرفة في تلك الوحدة بدون تحديد اسم الوحدة كبادئة.

```python
from math import *
print(sqrt(36))  # 6.0
```

ومع ذلك، يُنصح عمومًا بعدم استخدام هذا لأنه قد يؤدي إلى تصادم في فضاء الأسماء ويجعل من الصعب معرفة مصدر الأسماء.

## `if __name__ == '__main__'`

- **المتغير `__name__`**: `__name__` هو متغير مدمج خاص في بايثون. عندما يُنفذ ملف بايثون مباشرة، يعيّن بايثون قيمة هذا المتغير إلى السلسلة `"__main__"`. لكن إذا تم استيراد ملف بايثون كوحدة في سكربت بايثون آخر، تُعيّن قيمة المتغير `__name__` إلى اسم تلك الوحدة.

لهذا السبب غالبًا ما تجد هذا الشرط في سكربتات بايثون. يحتوي على الشفرة التي تريد تشغيلها **فقط** إذا كان سكربت بايثون يعمل كبرنامج رئيسي.

```python
if __name__ == '__main__': 
    # Code
```

## الأخطاء الشائعة في بايثون

- **SyntaxError**: الخطأ الذي تطلقه Python عندما لا تتبع شفرتك قواعد صياغتها. على سبيل المثال، ستؤدي الشفرة `print("Hello there"` إلى خطأ صياغة بالرسالة `SyntaxError: '(' was never closed`، لأن الشفرة تفتقد قوسًا ختاميًا.
- **NameError**: تطلق Python خطأ `NameError` عندما تحاول الوصول إلى متغير أو دالة لم تعرّفها. على سبيل المثال، إذا كان لديك السطر `print(username)` في شفرتك دون تعريف متغير `username` أولًا، فستحصل على خطأ اسم بالرسالة `NameError: name 'username' is not defined`.
- **TypeError**: هذا هو الخطأ الذي تطلقه Python عندما تُجري عملية على نوعي بيانات غير متوافقين أو أكثر. على سبيل المثال، إذا حاولت إضافة سلسلة نصية إلى رقم، فستحصل على الخطأ `TypeError: can only concatenate str (not "int") to str`.
- **IndexError**: ستحصل على خطأ `IndexError` إذا وصلت إلى فهرس غير موجود في قائمة أو تسلسلات أخرى مثل الصف المرتب والسلسلة النصية. على سبيل المثال، في السلسلة النصية `Hello world`، يكون فهرس آخر حرف `10`. وإذا تابعت ووصلت إلى حرف بهذه الطريقة، `greet = "hello world"; print(greet[11])`، فستحصل على خطأ بالرسالة `IndexError: string index out of range`.
- **AttributeError**: تطلق Python هذا الخطأ عندما تحاول استخدام طريقة أو خاصية غير موجودة في كائن من ذلك النوع. على سبيل المثال، استدعاء `.append()` على سلسلة نصية مثل `"hello".append("!")` سيؤدي إلى خطأ بالرسالة `AttributeError: 'str' object has no attribute 'append'`.

## تقنيات تصحيح الأخطاء الجيدة في بايثون

- **استخدام دالة `print`**: إدراج عبارات `print()` في نقاط مختلفة من شفرتك أثناء تصحيح الأخطاء يساعدك على رؤية قيم المتغيرات وكيفية تدفق الشفرة.
- **استخدام مصحح الأخطاء المدمج في بايثون (`pdb`)**: توفر بايثون وحدة `pdb` لتصحيح الأخطاء. هي جزء من مكتبة بايثون القياسية، لذا فهي متاحة دائمًا للاستخدام. مع `pdb`، يمكنك تعيين نقطة توقف باستخدام الدالة `set_trace()` لتبدأ في تنفيذ الشفرة خطوة بخطوة وفحص المتغيرات بطريقة تفاعلية.
- **الاستفادة من أدوات تصحيح الأخطاء في بيئات التطوير المتكاملة**: العديد من بيئات التطوير المتكاملة (IDEs) ومحررات الشفرة مثل Pycharm و VS Code توفر أدوات تصحيح أخطاء مع نقاط توقف، تنفيذ خطوة بخطوة، فحص المتغيرات، وميزات أخرى لتصحيح الأخطاء.

## معالجة الاستثناءات

- **`try...except`**: تُستخدم لتنفيذ كتلة شفرة قد تثير استثناء. كتلة `try` هي حيث تتوقع حدوث خطأ، بينما كتلة `except` تتعامل مع استثناء محدد وتنفذ إذا حدث ذلك الخطأ. إليك مثالًا:

  ```py
  try:
    print(22 / 0)
  except ZeroDivisionError:
    print('You can\'t divide by zero!')
    # You can't divide by zero!
  ```

يمكنك أيضًا ربط عدة كتل `except` لتتعامل مع أنواع متعددة من الاستثناءات:

  ```py
  try:
    number = int(input('Enter a number: '))
    print(22 / number)
  except ZeroDivisionError:
    print('You cannot divide by zero!')
    # You cannot divide by zero! prints when you enter 0
  except ValueError:
    print('Please enter a valid number!')
    # Please enter a valid number! prints when you don't enter an integer
  ```

- **`else` و `finally`**: هذه الكتل توسع `try...except`. إذا لم يحدث استثناء، تُنفذ كتلة `else`. كتلة `finally` تُنفذ دائمًا بغض النظر عن الأخطاء.

  ```py
  try:
    result = 100 / 4
  except ZeroDivisionError:
    print('You cannot divide by zero!') # This will not run
  else:
    print(f'Result is {result}') # Result is 25.0
  finally:
    print('Execution complete!') # Execution complete!
  ```

- **كائن الاستثناء**: يتيح لك الوصول إلى الاستثناء نفسه لتحسين تصحيح الأخطاء وطباعة رسالة الخطأ المباشرة. للوصول إلى كائن الاستثناء، تحتاج إلى استخدام الكلمة المفتاحية `as`. إليك مثالًا:

  ```py
  try:
      value = int('This will raise an error')
  except ValueError as e:
      print(f'Caught an error: {e}')
      # Caught an error: invalid literal for int() with base 10: 'This will raise an error'
  ```

- **جملة `raise`**: تتيح لك رفع استثناء يدويًا. يمكنك استخدامها لرمي استثناء عند تحقق شرط معين. إليك مثالًا:

  ```py
  def divide(a, b):
      if b == 0:
          raise ZeroDivisionError('You cannot divide by zero')
      return a / b
  ```
  
## الإشارة إلى الاستثناءات
  
جملة `raise` مفيدة أيضًا عند إنشاء استثناءات مخصصة، حيث يمكنك استخدامها لرمي استثناء مع رسالة مخصصة. إليك مثالًا على ذلك:

  ```py
  class InvalidCredentialsError(Exception):
      def __init__(self, message="Invalid username or password"):
          self.message = message
          super().__init__(self.message)

  def login(username, password):
      stored_username = "admin"
      stored_password = "password123"
      
      if username != stored_username or password != stored_password:
          raise InvalidCredentialsError()
      
      return f"Welcome, {username}!"
  ```  

إليك كيفية استخدام دالة `login` مع استثناء `InvalidCredentialsError`:

  ```py
  # failed login attempt
  try:
      message = login("user", "wrongpassword")
  except InvalidCredentialsError as e:
      print(f"Login failed: {e}")
  else:
      print(message)

  # successful login attempt
  try:
      message = login("admin", "password123")
  except InvalidCredentialsError as e:
      # This block is not executed because the login was successful
      print(f"Login failed: {e}")
  else:
      # The else block runs if the 'try' block completes without an exception
      print(message)
  ```

يمكن أيضًا استخدام جملة `raise` مع الكلمة المفتاحية `from` لربط الاستثناءات، مما يظهر العلاقة بين الأخطاء المختلفة:

  ```py
  def parse_config(filename):
    try:
        with open(filename, 'r') as file:
            data = file.read()
            return int(data)
    except FileNotFoundError:
        raise ValueError('Configuration file is missing') from None
    except ValueError as e:
        raise ValueError('Invalid configuration format') from e

  config = parse_config('config.txt')
  ```

## أصناف وكائنات بايثون

- **تعريف الصنف**: الصنف هو مخطط لإنشاء الكائنات. يحدد السلوك الذي سيكون للكائن من خلال خصائصه وطرقه. إليك مثالًا أساسيًا لتعريف صنف في بايثون:

```python
class Dog:
    def __init__(self, name, age):
        self.name = name
        self.age = age

    def bark(self):
        print(f'{self.name.upper()} says woof woof!')
```

- **إنشاء الكائنات**: الكائنات هي نسخ من الصنف. تُنشأ باستدعاء الصنف مع المعاملات اللازمة.

```python
dog1 = Dog('Jack', 3)
dog2 = Dog('Thatcher', 5)

dog1.bark()  # JACK says woof woof!
dog2.bark()  # THATCHER says woof woof!
```

- **استدعاء الطرق على الكائنات**: يمكنك استدعاء الطرق على الكائنات لأداء إجراءات أو استرجاع معلومات.

```python
object_name1.method_name()
object_name2.method_name()
```

- **الفرق بين الصنف والكائن**: الصنف هو قالب قابل لإعادة الاستخدام، بينما الكائن هو نسخة محددة من ذلك الصنف تحتوي على بيانات فعلية.

## الخصائص

- **خصائص المثيل**: تُعرف في `__init__()` باستخدام `self`، وهي فريدة لكل كائن.
- **خصائص الصنف**: تُعرف مباشرة داخل الصنف وتُشترك بها جميع النسخ.

```python
class Dog:
    species = 'French Bulldog'  # Class attribute

    def __init__(self, name):
        self.name = name  # Instance attribute

print(Dog.species) # French Bulldog

jack = Dog('Jack')
print(jack.name)     # Jack
print(jack.species)  # French Bulldog
```

## الطرق

- **الطرق**: دوال معرفة داخل الصنف تعمل على خصائص الكائن.

```python
class Car:
    def __init__(self, color, model):
        self.color = color
        self.model = model

    def describe(self):
        return f'This car is a {self.color} {self.model}'

my_car_1 = Car('red', 'Tesla Model S')
print(my_car_1.describe())  # This car is a red Tesla Model S
```

- **الوصول إلى الطرق**: استدعِ الطرق على الكائنات باستخدام ترميز النقطة. إليك مثالًا لاستدعاء طريقة `describe` على كائنين مختلفين من صنف السيارة:

```python
class Car:
    def __init__(self, color, model):
        self.color = color  
        self.model = model  

    def describe(self):
        return f'This car is a {self.color} {self.model}'

my_car_1 = Car('red', 'Tesla Model S')
my_car_2 = Car('green', 'Lamborghini Revuelto')

print(my_car_1.describe()) # Calling method using the dot notation

print(my_car_2.describe()) # Calling method using the dot notation
```

## الطرق الخاصة (Dunder)

- **التعريف**: طرق خاصة تبدأ وتنتهي بشرطتين سفليتين مزدوجتين (مثل `__init__`, `__len__`, `__str__`, `__eq__`). يستخدمها بايثون داخليًا للعمليات المدمجة.

```python
class Book:
    def __init__(self, title, pages):
        self.title = title
        self.pages = pages

    def __len__(self):
        return self.pages

    def __str__(self):
        return f"'{self.title}' has {self.pages} pages"

    def __eq__(self, other):
        return self.pages == other.pages

book1 = Book('Built Wealth Like a Boss', 420)
print(len(book1))        # 420
print(str(book1))        # 'Built Wealth Like a Boss' has 420 pages
```

- **استدعاء الطرق الخاصة بشكل غير مباشر**: لا تحتاج إلى استدعاء الطرق الخاصة مباشرة. بدلاً من ذلك، يستدعيها بايثون تلقائيًا عند حدوث إجراءات معينة. تشمل هذه العمليات:

- **العمليات الحسابية مثل الجمع، الطرح، الضرب، القسمة، وغيرها**. تُستدعى `__add__()` للجمع، `__sub__()` للطرح، `__mul__()` للضرب، و `__truediv__()` للقسمة.
        
- **عمليات السلاسل النصية مثل الربط، التكرار، التنسيق، والتحويل إلى نص**. تُستدعى `__add__()` للربط، `__mul__()` للتكرار، `__format__()` للتنسيق، و `__str__()` و `__repr__()` للتحويل إلى نص، وهكذا.
        
- **عمليات المقارنة مثل المساواة، الأصغر من، الأكبر من، وغيرها**. تُستدعى `__eq__()` لفحص المساواة، `__lt__()` للأصغر من، `__gt__()` للأكبر من، وهكذا.
        
- **عمليات التكرار مثل جعل الكائن قابلًا للتكرار والتقدم عبر العناصر**. تُستدعى `__iter__()` لإرجاع مكرر و `__next__()` لجلب العنصر التالي.

## مثال واقعي: عربة التسوق

- **صنف العربة مع الطرق الخاصة**: يسمح بالإضافة، الإزالة، التكرار، والتحقق من المحتويات بسلوك مدمج.

```python
class Cart:
    def __init__(self):
        self.items = []

    def add(self, item):
        self.items.append(item)

    def remove(self, item):
        if item in self.items:
            self.items.remove(item)
        else:
            print(f'{item} is not in cart')

    def list_items(self):
        return self.items

    def __len__(self):
        return len(self.items)

    def __getitem__(self, index):
        return self.items[index]

    def __contains__(self, item):
        return item in self.items

    def __iter__(self):
        return iter(self.items)

cart = Cart()
cart.add('Laptop')
print(len(cart))        # 1
print('Laptop' in cart) # True
```

## ما هو البرمجة كائنية التوجه؟

- **البرمجة كائنية التوجه**: أسلوب برمجي يعامل فيه المطوّرون كل شيء في شفرتهم ككائنات من العالم الحقيقي. يُعرف هذا الأسلوب اختصارًا بـ OOP. المبادئ الأربعة الأساسية التي تساعدك على تنظيم وإدارة الشفرة بفعالية هي **التغليف**، **الوراثة**، **تعدد الأشكال**، و**التجريد**.
- **الأصناف**: هي المخطط لإنشاء الكائنات. كل كائن يتم إنشاؤه من صنف يحتوي على خصائص تحدد البيانات وطرق تحدد سلوكيات الكائن.

## ما هو التغليف؟

- **التغليف**: تجميع الخصائص والطرق الخاصة بكائن في وحدة واحدة. يسمح لك بإخفاء الحالة الداخلية للكائن خلف مجموعة بسيطة من الطرق والخصائص العامة التي تعمل كبوابات. خلف هذه البوابات توجد خصائص وطرق خاصة تتحكم في كيفية تغير البيانات ومن يمكنه رؤيتها.
- **مثال على التغليف**: إذا أردت تتبع رصيد محفظة، ستسمح بالإيداع والسحب، لكنك لن تسمح لأي شخص بالتلاعب برصيد المحفظة نفسه:

```py
class Wallet:
   def __init__(self, balance):
       self.__balance = balance # Private attribute

   def deposit(self, amount):
       if amount > 0:
           self.__balance += amount # Add to the balance safely

   def withdraw(self, amount):
       if 0 < amount <= self.__balance:
           self.__balance -= amount # Remove from the balance safely

account = Wallet(500)
print(account.__balance) # AttributeError: 'Wallet' object has no attribute '__balance'
```

- **الفرق بين بادئة السمة بشرطة سفلية واحدة وشرطتين سفليتين**: بادئة الخصائص والطرق بشرطة سفلية واحدة تعني أنها مخصصة للاستخدام الداخلي فقط. هذا مجرد اصطلاح ولا يمنع الوصول إلى الخصائص من الخارج. أما بادئة الخصائص والطرق بشرطتين سفليتين فتمنع فعليًا الوصول إليها من خارج الصنف.

## ما هي الدوال الجالبة والمضبوطة (Getters and Setters)؟

- **الدوال الجالبة والمضبوطة**: طرق تسمح لك بالتحكم في كيفية الوصول إلى خصائص الصنف وتعديلها. تسترجع القيم باستخدام الدوال الجالبة وتعيّن القيم باستخدام الدوال المضبوطة.
- **الخصائص**: تربط بين الدوال الجالبة والمضبوطة، وتسمح بالوصول إلى البيانات. تنفذ منطقًا إضافيًا خلف الكواليس عند الحصول على القيم أو تعيينها أو حذفها.
- **لماذا الخصائص بدلًا من الطرق؟**: تُستخدم الخصائص بدلاً من الطرق لزيادة وضوح الشفرة ونظافتها. تسمح لك بالوصول إلى القيم باستخدام ترميز النقطة، مثل الخصائص العادية، بدون أقواس.
- **إنشاء دالة جالبة**: لإنشاء دالة جالبة، تستخدم المزخرف `@property`. إليك دالة جالبة تسترجع نصف قطر دائرة:

```py
class Circle:
    def __init__(self, radius):
        self.radius = radius # Calling the setter

    @property
    def radius(self): # A getter to get the radius
        return self._radius
  
    @property
    def area(self):  # A getter to calculate area
        return 3.14 * (self._radius ** 2)

my_circle = Circle(3)

print(my_circle.radius) # 3
print(my_circle.area) # 28.26
```

استخدام self.radius داخل `__init__()` يضمن تشغيل الدالة المضبوطة أثناء إنشاء الكائن، بحيث يتم التقاط قيم نصف القطر غير الصالحة فورًا.

- **إنشاء دالة مضبوطة**: لإنشاء الدالة المضبوطة التي تعيّن نصف القطر، يجب تعريف دالة أخرى بنفس الاسم واستخدام المزخرف `@<property_name>.setter` فوقها:

```py
class Circle:
    def __init__(self, radius):
        self.radius = radius # Calling the setter

    @property
    def radius(self):  # A getter to get the radius
        return self._radius

    @radius.setter
    def radius(self, value):  # A setter to set the radius
        if value <= 0:
            raise ValueError('Radius must be positive')
        self._radius = value

my_circle = Circle(3)
print('Initial radius:', my_circle.radius) # Initial radius: 3

my_circle.radius = 8
print('After modifying the radius:', my_circle.radius) # After modifying the radius: 8
```

- **كيفية تعامل بايثون مع الدوال الجالبة والمضبوطة**: بمجرد تعريف الدوال الجالبة والمضبوطة، يستدعيها بايثون تلقائيًا تحت الكواليس كلما استخدمت صيغة الوصول العادية للخاصية بهذه الطريقة:

```py
my_circle.radius # This will call the getter
my_circle.radius = 4 # This will call the setter
```

داخل الدالة المضبوطة، يجب ألا تعيّن إلى اسم الخاصية نفسه لأن ذلك سيسبب `RecursionError`.

- **المزيل (Deleter)**: بعد تعيين القيمة واسترجاعها باستخدام setter وgetter، يمكنك التحكم في كيفية حذفها باستخدام `deleter`. ينفذ المزيل منطقًا مخصصًا عند استخدام عبارة `del` على خاصية. لإنشاء مزيل، استخدم الـ `@<property_name>.deleter` decorator.

```py
  # Deleter
    @radius.deleter
    def radius(self):
        print("Deleting radius...")
        del self._radius
```

## ما هي الوراثة وكيف تعزز إعادة استخدام الشفرة؟

- **الوراثة**: العملية التي يستخدم فيها صنف فرعي خصائص وطرق صنف أبوي. تعزز الوراثة إعادة استخدام الشفرة، وتوفر تسلسلات هرمية واضحة، وتخصص السلوك دون الحاجة لإعادة كتابة كل شيء. لتنفيذ الوراثة، يأخذ الصنف الفرعي اسم الصنف الأبوي:

```py
class Parent:
    # Parent attributes and methods

class Child(Parent):
    # Child inherits, extends, and/or overrides where necessary
```

- **الوراثة المفردة والمتعددة**: عندما يرث صنف فرعي الخصائص والطرق من صنف أبوي واحد، كما في المثال أعلاه، تسمى هذه **الوراثة المفردة**. أما عندما يرث صنف فرعي من أكثر من صنف أبوي، فتسمى **الوراثة المتعددة**. إليك الصيغة لذلك:

```py
class Parent:
    # Attributes and methods for Parent

class Child:
    # Attributes and methods for Child

class GrandChild(Parent, Child):
    # GrandChild inherits from both Parent and Child
    # GrandChild can combine or override behavior from each
```

- **دالة `super()`**: دالة تسمح لك **بالاستدعاء** لطريقة من الصنف الأبوي، عندما يكون للصنف طريقة مختلفة أو يوسع الطريقة، دون تكرار الشفرة.

## ما هو تعدد الأشكال وكيف يعزز إعادة استخدام الشفرة؟

- **تعدد الأشكال**: مبدأ في البرمجة كائنية التوجه يسمح لأصناف مختلفة باستخدام نفس اسم الطريقة، لكن كل صنف ينفذها بطريقة مختلفة عند الاستدعاء. إليك الصيغة لذلك:

```py
class A:
   def action(self): ...

class B:
   def action(self): ...

class C:
   def action(self): ...

Class().method()  # Works for A, B, or C
```

- **تعدد الأشكال القائم على الوراثة**: يحدد الصنف الأبوي طريقة، ويعدل كل صنف فرعي هذه الطريقة حسب استخدامه.

## ما هو تشويش الأسماء وكيف يعمل؟

- **تشويش الأسماء**: عملية يعيد فيها بايثون داخليًا تسمية خاصية تبدأ بشرطتين سفليتين بإضافة شرطة سفلية واسم الصنف كبادئة، محولًا `__attribute` إلى `_ClassName__attribute`.
- **هدف تشويش الأسماء**: الهدف الرئيسي من تشويش الأسماء هو منع تجاوز الخصائص والطرق عن طريق الخطأ عند استخدام الوراثة. إليك شفرة توضح ذلك بشكل أفضل:

```py
class Parent:
    def __init__(self):
        self.__data = 'Parent data'

class Child(Parent):
    def __init__(self):
        super().__init__()
        self.__data = 'Child data'

c = Child()
print(c.__dict__) # {'_Parent__data': 'Parent data', '_Child__data': 'Child data'}
```

## ما هو التجريد وكيف يساعد في تنظيم الأنظمة المعقدة؟

- **التجريد**: مفهوم برمجي يخفي تفاصيل التنفيذ المعقدة لكائن أو نظام ويعرض فقط الميزات الأساسية. في بايثون ولغات برمجة أخرى، يبسط التجريد الأنظمة المعقدة بزيادة إعادة الاستخدام.
- **مثال على التجريد**: مثال جيد على التجريد في الحياة اليومية هو السيارة التي تتيح لك استخدام المقود والدواسات وذراع التغيير دون معرفة كيفية عمل المحرك أو المكابح.
- **كيفية تنفيذ بايثون للتجريد**: ينفذ بايثون التجريد عبر وحدة `abc`. توفر الوحدة الصنف `ABC` (الصنف الأساسي المجرد) والمزخرف `@abstractmethod`. يحدد الصنف الأساسي المجرد الطرق والخصائص المشتركة التي يجب على الأصناف الفرعية تنفيذها. لا يمكن إنشاء مثيل منه.
- **كيفية تعريف الطريقة المجردة**: تُعرف الطريقة المجردة باستخدام `@abstractmethod` ويجب تجاوزها في الأصناف الفرعية، حتى لو كان لها تنفيذ افتراضي. الصيغة الأساسية للتجريد تبدو هكذا:

```py
from abc import ABC, abstractmethod

# Define an abstract base class
class AbstractClass(ABC):
    @abstractmethod
    def abstract_method(self):
        pass

# Concrete subclass that implements the abstract method
class ConcreteClassOne(AbstractClass):
    def abstract_method(self):
        print('Implementation in ConcreteClassOne')

# Another concrete subclass
class ConcreteClassTwo(AbstractClass):
    def abstract_method(self):
        print('Implementation in ConcreteClassTwo')
```

## الخوارزميات وترميز Big O

- **الخوارزميات**: مجموعة من التعليمات غير الغامضة لحل مشكلة أو تنفيذ مهمة. يجب أن تنتهي الخوارزميات بعدد محدود من الخطوات، ويجب أن تكون كل خطوة دقيقة وغير غامضة.

- **ترميز Big O**: يصف أداء الخوارزمية في أسوأ الحالات، أو معدل النمو، مع زيادة حجم الإدخال. يركز على كيفية نمو استخدام الموارد مع حجم الإدخال، متجاهلًا العوامل الثابتة والحدود ذات الرتب الأقل.

### التعقيدات الزمنية الشائعة

- **O(1) - زمن ثابت**: تستغرق الخوارزمية نفس الوقت بغض النظر عن حجم الإدخال.

```python
def check_even_or_odd(number):
    if number % 2 == 0:
        return 'Even'
    else:
        return 'Odd'
```

- **O(log n) - زمن لوغاريتمي**: يزداد الوقت ببطء مع نمو الإدخال. شائع في الخوارزميات التي تقلل حجم المشكلة بشكل متكرر (مثل البحث الثنائي).

- **O(n) - زمن خطي**: يزداد وقت التشغيل تناسبًا مع حجم الإدخال.

```python
for grade in grades:
    print(grade)
```

- **O(n log n) - زمن لوغاريتمي خطي**: تعقيد زمني شائع للخوارزميات الفعالة للترتيب مثل دمج الترتيب والفرز السريع.

- **O(n²) - زمن تربيعي**: يزداد وقت التشغيل تربيعيًا. غالبًا ما يظهر في الحلقات المتداخلة.

```python
for i in range(n):
    for j in range(n):
        print("Hello, World!")
```

### التعقيد المكاني

- **O(1) - مساحة ثابتة**: تستخدم الخوارزمية نفس كمية الذاكرة بغض النظر عن حجم الإدخال.
- **O(n) - مساحة خطية**: يزداد استخدام الذاكرة تناسبًا مع حجم الإدخال.
- **O(n²) - مساحة تربيعية**: يزداد استخدام الذاكرة تربيعيًا مع حجم الإدخال.

## تقنيات حل المشكلات

- **فهم المشكلة**: اقرأ بيان المشكلة عدة مرات. حدد الإدخال، الناتج المتوقع، وكيفية تحويل الإدخال إلى الناتج.

- **الرمز الزائف**: وصف عالي المستوى لمنطق الخوارزمية مستقل عن اللغة. يستخدم لغة مكتوبة مشتركة مختلطة مع تراكيب برمجية مثل `IF`، `ELSE`، `FOR`، `WHILE`.

```md
GET original_string
SET reversed_string = ""
FOR EACH character IN original_string:
  ADD character TO THE BEGINNING OF reversed_string
DISPLAY reversed_string
```

- **حالات الحافة**: إدخالات محددة وصحيحة تحدث عند حدود ما يجب أن تتعامل معه الخوارزمية. اعتبرها دائمًا واختبرها.

## المصفوفات

- **المصفوفات الثابتة**: لها حجم ثابت يحدد عند التهيئة. تُخزن العناصر في مواقع ذاكرة متجاورة. لا يمكن تغيير الحجم أثناء تنفيذ البرنامج.

- **المصفوفات الديناميكية**: يمكن أن تنمو أو تتقلص تلقائيًا أثناء تنفيذ البرنامج. تتعامل مع تغيير الحجم عبر النسخ التلقائي إلى مصفوفات أكبر عند الحاجة.

### قوائم بايثون (مصفوفات ديناميكية)

```python
numbers = [3, 4, 5, 6]

# Access elements
numbers[0]  # 3

# Update elements
numbers[2] = 16

# Add elements
numbers.append(7)
numbers.insert(3, 15)  # Insert at specific index

# Remove elements
numbers.pop(2)  # Remove at specific index
numbers.pop()   # Remove last element
```

### التعقيدات الزمنية للمصفوفات الديناميكية

- **الوصول**: O(1)
- **الإدراج في النهاية**: O(1) في المتوسط، O(n) عند الحاجة لتغيير الحجم
- **الإدراج في الوسط**: O(n)
- **الحذف**: O(n) للوسط، O(1) للنهاية

## المكدسات

- **المكدسات**: هيكل بيانات يعتمد مبدأ آخر داخل، أول خارج (LIFO). تُضاف العناصر وتُزال من الأعلى فقط.

- **عملية الدفع (Push)**: إضافة عنصر إلى أعلى المكدس. التعقيد الزمني: O(1).

- **عملية السحب (Pop)**: إزالة عنصر من أعلى المكدس. التعقيد الزمني: O(1).

```python
# Using Python list as stack
stack = []

# Push operations
stack.append(1)
stack.append(2)
stack.append(3)

# Pop operations
top_element = stack.pop()  # Returns 3
```

## الطوابير

- **الطوابير**: هيكل بيانات يعتمد مبدأ أول داخل، أول خارج (FIFO). تُضاف العناصر إلى الخلف وتُزال من الأمام.

- **عملية الإدخال (Enqueue)**: إضافة عنصر إلى نهاية الطابور. التعقيد الزمني: O(1).

- **عملية الإخراج (Dequeue)**: إزالة عنصر من بداية الطابور. التعقيد الزمني: O(1).

```python
from collections import deque

# Using deque for efficient queue operations
queue = deque()

# Enqueue operations
queue.append(1)
queue.append(2)
queue.append(3)

# Dequeue operations
first_element = queue.popleft()  # Returns 1
```

## القوائم المرتبطة

- **القوائم المرتبطة**: هيكل بيانات خطي حيث تحتوي كل عقدة على بيانات ومرجع للعقدة التالية. العقد متصلة كسلسلة.

### القوائم المرتبطة المفردة

- **الهيكل**: كل عقدة تحتوي على بيانات ومرجع واحد للعقدة التالية.
- **التنقل**: يمكن التحرك فقط من الرأس إلى الذيل.
- **عقدة الرأس**: أول عقدة في القائمة، عادة العقدة الوحيدة التي يمكن الوصول إليها مباشرة.
- **عقدة الذيل**: آخر عقدة في القائمة، تشير إلى `None`.

### العمليات والتعقيدات الزمنية

- **الإدراج في البداية**: O(1)
- **الإدراج في النهاية**: O(n) - يجب التنقل إلى النهاية
- **الإدراج في الوسط**: O(n) - يجب التنقل إلى الموضع
- **الحذف من البداية**: O(1)
- **الحذف من النهاية**: O(n) - يجب التنقل لإيجاد العقدة السابقة
- **الحذف من الوسط**: O(n) - يجب التنقل لإيجاد العقدة

### القوائم المرتبطة المزدوجة

- **الهيكل**: كل عقدة تحتوي على بيانات ومرجعين: العقدة التالية والعقدة السابقة.
- **التنقل**: يمكن التحرك في كلا الاتجاهين.
- **الذاكرة**: تتطلب ذاكرة أكثر من القوائم المرتبطة المفردة بسبب المرجع الإضافي.

## جداول التجزئة والمجموعات

### الخرائط وجداول التجزئة

- **الخريطة (نوع بيانات مجرد)**: تدير مجموعات من أزواج المفتاح-القيمة. يجب أن يكون كل مفتاح فريدًا، لكن القيم يمكن تكرارها.

- **جدول التجزئة**: تنفيذ ملموس لنوع بيانات الخريطة باستخدام تقنية التجزئة. يستخدم دالة تجزئة لتوليد قيم تجزئة للمفاتيح، تحدد موقع التخزين في المصفوفة الأساسية.

### قواميس بايثون (جداول التجزئة)

```python
# Creating dictionaries
my_dictionary = {
    "A": 1,
    "B": 2, 
    "C": 3
}

# Alternative creation
my_dictionary = dict(A=1, B=2, C=3)

# Access and modify
value = my_dictionary["A"]  # 1
my_dictionary["A"] = 4      # Update value
del my_dictionary["A"]      # Remove key-value pair

# Check membership
"C" in my_dictionary

# Get keys, values, items
my_dictionary.keys()
my_dictionary.values()
my_dictionary.items()
```

### التعقيدات الزمنية لجداول التجزئة

- **الحالة المتوسطة**: O(1) للإدراج، الاسترجاع، الحذف
- **أسوأ حالة**: O(n) عند حدوث العديد من تصادمات التجزئة

### المجموعات

- **المجموعات**: مجموعات غير مرتبة من عناصر فريدة. لا يسمح بالتكرار، ولا يتم الحفاظ على ترتيب محدد.

- **العناصر غير القابلة للتغيير فقط**: يمكن للمجموعات أن تحتوي فقط على أنواع بيانات غير قابلة للتغيير (أرقام، سلاسل نصية، صفوف مرتبة) لأن قيم التجزئة يجب أن تبقى ثابتة.

```python
# Creating sets
numbers = {1, 2, 3, 4}
empty_set = set()  # Must use set(), not {}

# Add and remove elements
numbers.add(5)
numbers.remove(4)      # Raises KeyError if not found
numbers.discard(4)     # No error if not found

# Set operations
set_a = {1, 2, 3, 4}
set_b = {2, 3, 4, 5, 6}

# Union, intersection, difference, symmetric difference
set_a.union(set_b)                    # or set_a | set_b
set_a.intersection(set_b)             # or set_a & set_b
set_a.difference(set_b)               # or set_a - set_b
set_a.symmetric_difference(set_b)     # or set_a ^ set_b

# Subset and superset checks
set_a.issubset(set_b)
set_a.issuperset(set_b)
set_a.isdisjoint(set_b)

# Membership testing
5 in numbers
```

### التعقيدات الزمنية للمجموعات

- **الحالة المتوسطة**: O(1) للإضافة، الإزالة، اختبار العضوية
- **أسوأ حالة**: O(n) بسبب تصادمات التجزئة

## تصادمات التجزئة

- **تصادم التجزئة**: يحدث عندما تنتج مفتاحان مختلفان نفس قيمة التجزئة.

- **استراتيجيات حل التصادم**:
  - **التسلسل (Chaining)**: يشير كل فهرس في المصفوفة إلى قائمة مرتبطة تخزن كل العناصر ذات قيمة التجزئة نفسها
  - **العنوان المفتوح (Open Addressing)**: البحث عن الفهرس التالي المتاح باستخدام تسلسل محدد مسبقًا

## متى تستخدم كل هيكل بيانات

- **القوائم**: عندما تحتاج إلى وصول مرتب ومفهرس ولا تعرف الحجم مسبقًا
- **المكدسات**: لعمليات LIFO (وظائف التراجع، تقييم التعبيرات، التراجع)
- **الطوابير**: لعمليات FIFO (جدولة المهام، البحث بالاتساع أولاً)
- **القوائم المرتبطة**: عند الإدراج/الحذف المتكرر في البداية، الحجم غير معروف، لا حاجة للوصول العشوائي
- **جداول التجزئة**: للبحث السريع عن أزواج المفتاح-القيمة، عد التكرارات، التخزين المؤقت
- **المجموعات**: لفحص التفرد، العمليات الرياضية على المجموعات، إزالة التكرارات

## خوارزميات البحث

تسمح خوارزميات البحث بالبحث عن هدف ضمن قائمة معينة من العناصر.

في علوم الحاسوب، هناك خوارزميتان للبحث يجب أن تعرفهما: هما **البحث الخطي** و**البحث الثنائي**. من المهم فهم الفرق بينهما ومتى تستخدم كل واحدة.

### البحث الخطي

- يبحث البحث الخطي عبر قائمة من العناصر، ويفحص كل عنصر من البداية حتى يتم العثور على العنصر الهدف.
- إذا تم العثور على العنصر الهدف، يتم إرجاع الفهرس الذي يقع فيه في القائمة.
- إذا لم يتم العثور على الهدف، يرجع `-1`، مما يعني **فهرس غير صالح** في معظم لغات البرمجة.
- لأن البحث الخطي يفحص كل عنصر حتى يجد الهدف، فهو غير فعال لقوائم كبيرة.
- التعقيد الزمني للبحث الخطي هو `O(n)` لأن الوقت اللازم للبحث ينمو خطيًا مع حجم القائمة.
- التعقيد المكاني للبحث الخطي هو `O(1)` لأنه لا يتطلب مساحة إضافية للبحث في القائمة.

### البحث الثنائي

- يعمل البحث الثنائي عن طريق تقسيم قائمة العناصر إلى نصفين، وفحص ما إذا كانت القيمة الهدف في منتصف القائمة.
- الشرط لعمل البحث الثنائي هو أن تكون العناصر في القائمة مرتبة تصاعديًا.
- البحث الثنائي هو خوارزمية أكثر كفاءة للبحث في قائمة كبيرة لأنه يقسم القائمة إلى نصفين ويتجاهل أي نصف لا يحتوي على الهدف.
- إذا تم العثور على العنصر الهدف في منتصف القائمة، يتم إرجاع فهرس العنصر الهدف.
- إذا لم يتم العثور على العنصر، تتحقق الخوارزمية مما إذا كان العنصر الهدف في النصف الأيسر أو الأيمن من القائمة.
- تستمر في تقسيم الأجزاء المتبقية من القائمة إلى أنصاف حتى يتم العثور على العنصر الهدف.
- إذا لم يتم العثور على العنصر الهدف في النهاية، يرجع `-1`.
- التعقيد الزمني للبحث الثنائي هو `O(log n)` لأن الوقت اللازم للبحث ينمو لوغاريتميًا مع حجم القائمة.
- التعقيد المكاني للبحث الثنائي هو `O(1)` لأنه لا يتطلب مساحة إضافية للبحث في القائمة.

### كيف يختلف البحث الخطي عن البحث الثنائي

- البحث الثنائي أكثر ملاءمة لقائمة كبيرة من العناصر مقارنة بالبحث الخطي.
- التعقيد الزمني للبحث الخطي هو `O(n)` لأن الوقت اللازم للبحث ينمو خطيًا مع حجم القائمة.
- التعقيد الزمني للبحث الثنائي هو `O(log n)` لأن الوقت اللازم للبحث ينمو لوغاريتميًا مع حجم القائمة.

## خوارزميات الترتيب وتقنية القسمة والتغلب

في علوم الحاسوب، القسمة والتغلب هي تقنية تستخدم لتقسيم المشكلة إلى مشاكل فرعية أصغر لتسهيل حلها. غالبًا ما تستخدم العودية في القسمة والتغلب، وهي استراتيجية قوية تُستخدم لتنفيذ العديد من خوارزميات الترتيب الفعالة مثل دمج الترتيب.

### دمج الترتيب

- دمج الترتيب هو خوارزمية ترتيب تتبع نهج القسمة والتغلب.
- تعمل عن طريق تقسيم القائمة بشكل متكرر إلى قوائم فرعية أصغر حتى تحتوي كل قائمة فرعية على عنصر واحد فقط.
- ثم تدمج القوائم الفرعية معًا بشكل متكرر بترتيب مرتب.
- التعقيد الزمني لدمج الترتيب هو `O(n log n)` لأن القائمة تُقسم باستمرار إلى نصفين `(log n)` ثم تُدمج معًا `(O(n))`.
- التعقيد المكاني لدمج الترتيب هو `O(n)` لأنه ليس خوارزمية ترتيب في المكان.

## نظرة عامة على الرسوم البيانية

الرسم البياني هو مجموعة من العقد (الرؤوس) متصلة بحواف (الاتصالات). يمكن لكل عقدة الاتصال بعدة عقد أخرى، مكونة شبكة. أنواع الرسوم البيانية المختلفة تشمل:

- موجه: الحواف لها اتجاه (من عقدة إلى أخرى)، غالبًا ما تمثل بخطوط مستقيمة وأسهم.
- غير موجه: الحواف ليس لها اتجاه، تمثل بخطوط بسيطة.
- رأس: كل عقدة مرتبطة بمسمى أو معرف.
- دوري: يحتوي على دورات (مسار يبدأ وينتهي عند نفس العقدة).
- غير دوري (DAG): لا يحتوي على دورات.
- ملصق الحافة: كل حافة لها ملصق يُرسم عادة بجانب الحافة المقابلة.
- موزون: الحواف لها أوزان (قيم) مرتبطة بها، يمكن استخدامها لإجراء العمليات الحسابية.
- غير متصل: يحتوي على عقدتين أو أكثر غير متصلتين بأي حواف.

تُستخدم الرسوم البيانية في تطبيقات مختلفة مثل الخرائط، الشبكات، أنظمة التوصية، وحل التبعيات.

## استعراضات الرسوم البيانية

تشمل زيارة جميع العقد في الرسم البياني. الخوارزميات الرئيسية هي:

- **البحث بالاتساع أولاً (BFS)**
  - يستخدم طابور.
  - يستكشف مستوى بمستوى.
  - يجد أقصر مسار في الرسوم غير الموزونة.

- **البحث بالعمق أولاً (DFS)**
  - يستخدم مكدس (أو العودية).
  - يستكشف فرعًا بالكامل قبل التراجع.
  - مفيد لاكتشاف الدورات والعثور على المسارات.

## تمثيلات الرسوم البيانية

يمكن تمثيل الرسوم البيانية بطريقتين رئيسيتين:

- **قائمة المجاورة**
  - تحتوي كل عقدة على قائمة بجيرانها.
  - فعالة من حيث المساحة للرسوم البيانية المتفرقة.
  - سهلة للتكرار على الجيران.

- **المصفوفة المجاورة**
  - مصفوفة ثنائية الأبعاد حيث تمثل الصفوف والأعمدة العقد.
  - تستهلك مساحة كبيرة للرسوم البيانية الكبيرة.
  - سريعة للتحقق مما إذا كان هناك حافة بين عقدتين.

## الأشجار

الشجرة نوع خاص من الرسوم البيانية التي لا تحتوي على دورات ومتصلة. الخصائص الرئيسية تشمل:

- لا تحتوي على حلقات أو دورات (مسارات يبدأ وينتهي عند نفس العقدة).
- يجب أن تكون متصلة (يمكن الوصول إلى كل عقدة من أي عقدة أخرى).

### الأنواع الشائعة من الأشجار

الأنواع الأكثر شيوعًا من الأشجار هي:

- الأشجار الثنائية
  - تحتوي كل عقدة على طفلين كحد أقصى، طفل أيسر وطفل أيمن.

- شجرة البحث الثنائي (BST)
  - شجرة ثنائية حيث كل طفل أيسر أقل من عقدة الأصل، وكل طفل أيمن أكبر من عقدة الأصل.


## شجرة Trie

تُعرف أيضًا بأشجار البادئة، تُستخدم لتخزين مجموعات من السلاسل النصية، حيث تمثل كل عقدة حرفًا.

يتم تخزين البادئات المشتركة مرة واحدة فقط، مما يجعلها فعالة لمهام مثل الإكمال التلقائي وفحص الإملاء.

عمليات البحث والإدخال لها تعقيد زمني قدره O(L)، حيث L هو طول السلسلة النصية.

## طوابير الأولوية

طابور الأولوية هو نوع مجرد من هياكل البيانات حيث لكل عنصر أولوية.

تأخذ الطوابير والمكدسات في الاعتبار فقط ترتيب الإدخال، بينما تأخذ طوابير الأولوية في الاعتبار أولوية العناصر.

تتبع الطوابير القياسية مبدأ FIFO (الأول في الدخول، الأول في الخروج) وتتبع المكدسات مبدأ LIFO (الأخير في الدخول، الأول في الخروج). لكن في طابور الأولوية، تُخدم العناصر ذات الأولوية الأعلى قبل تلك ذات الأولوية الأقل، بغض النظر عن ترتيب إدخالها.

## الحزم

هيكل بيانات شجري متخصص له خاصية محددة جدًا تسمى خاصية الحزمة.

تحدد خاصية الحزمة العلاقة بين عقدة الأصل والعقد الفرعية. هناك نوعان من الحزم:

- الحزمة العظمى
  - قيمة كل عقدة أصل أكبر من أو تساوي قيم عقدها الفرعية.
  - أكبر عنصر يكون في الجذر.

- الحزمة الصغرى
  - قيمة كل عقدة أصل أقل من أو تساوي قيم عقدها الفرعية.
  - أصغر عنصر يكون في الجذر.

### مثال على وحدة `heapq` في بايثون

```py
import heapq

# Create empty heap
my_heap = []

# Insert elements
heapq.heappush(my_heap, 9)
heapq.heappush(my_heap, 3)
heapq.heappush(my_heap, 5)
print(my_heap) # [3, 9, 5]

# Remove smallest element
print(heapq.heappop(my_heap))  # 3
print(my_heap) # [5, 9]

# Push + Pop in one step
print(heapq.heappushpop(my_heap, 7)) # 5
print(my_heap) # [7, 9]

# Transform list into heap
nums = [5, 7, 3, 1]
heapq.heapify(nums)
```

### استخدام الأولويات

```py
my_heap = []
heapq.heappush(my_heap, (3, "A"))
heapq.heappush(my_heap, (2, "B"))
heapq.heappush(my_heap, (1, "C"))

# Removes lowest number = highest priority
print(heapq.heappop(my_heap))  # (1, "C")
```

## مقدمة في البرمجة الديناميكية

- **التعريف**: البرمجة الديناميكية هي تقنية خوارزمية تحل المشكلات المعقدة بتقسيمها إلى مشكلات فرعية أبسط وتخزين النتائج لتجنب الحسابات المتكررة.
- **المشكلات الفرعية المتداخلة**: تظهر نفس المشكلات الفرعية الصغيرة عدة مرات عند حل المشكلة الأكبر. بدلاً من إعادة حساب هذه المشكلات الفرعية مرارًا، نخزن حلولها.
- **البنية المثلى الفرعية**: الحل الأمثل للمشكلة يحتوي على حلول مثلى لمشكلاتها الفرعية. هذا يعني أنه يمكننا بناء أفضل حل من خلال دمج أفضل الحلول للأجزاء الأصغر.

## حلول البرمجة الديناميكية

- **التخزين المؤقت (النهج من الأعلى إلى الأسفل)**: يخزن التخزين المؤقت نتائج استدعاءات الدوال المكلفة ويعيد النتيجة المخزنة عند تكرار نفس المُدخلات.

```py
def climb_stairs_memo(n, memo={}):
    """Dynamic programming with memoization"""
    # Check if we've already calculated this value
    if n in memo:
        return memo[n]  # Return cached result - O(1) lookup!
    
    # Base cases
    if n <= 2:
        return n
    
    # Calculate once and store in memo for future use
    memo[n] = climb_stairs_memo(n-1, memo) + climb_stairs_memo(n-2, memo)
    return memo[n]
```

- **الجدولة (النهج من الأسفل إلى الأعلى)**: يبني الحل من القاعدة إلى الأعلى، مع ملء جدول بحلول المشكلات الفرعية.

```py
def climb_stairs_tabulation(n):
    """Dynamic programming with tabulation"""
    if n <= 2:
        return n
    
    # Create array to store results for all steps from 0 to n
    dp = [0] * (n + 1)
    dp[1] = 1  # 1 way to reach step 1
    dp[2] = 2  # 2 ways to reach step 2
    
    # Build up the solution iteratively
    for i in range(3, n + 1):
        # Ways to reach step i = ways to reach (i-1) + ways to reach (i-2)
        dp[i] = dp[i-1] + dp[i-2]
    
    return dp[n]
```

## تطبيقات العالم الحقيقي باستخدام البرمجة الديناميكية

- **تحسين المسارات**: تستخدم أنظمة GPS خوارزميات البرمجة الديناميكية لإيجاد أقصر المسارات بين المواقع.
- **معالجة النصوص**: تعتمد أدوات التدقيق الإملائي والإكمال التلقائي غالبًا على البرمجة الديناميكية لحساب مسافات التحرير بين الكلمات.
- **النمذجة المالية**: تستخدم استراتيجيات الاستثمار وتحسين المحافظ تقنيات البرمجة الديناميكية بشكل متكرر.
- **تخصيص الموارد**: تظهر مشكلة حقيبة الظهر ومتغيراتها في الجدولة والميزانية وإدارة الموارد.

## متى تستخدم البرمجة الديناميكية

يجب أن تفكر في استخدام البرمجة الديناميكية في السيناريوهات التالية:

- يمكن تقسيم المشكلة إلى مشكلات فرعية متداخلة.
- تظهر المشكلة بنية مثلى فرعية.
- الحل التكراري البسيط يتضمن حسابات متكررة.
- تحتاج إلى تحسين تعقيد الوقت على حساب تعقيد المساحة.
