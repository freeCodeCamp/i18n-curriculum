# --interactive--

Das `aria-controls`-Attribut wird verwendet, um eine programmatische Beziehung zwischen einem Element herzustellen, das die Anwesenheit eines anderen Elements auf der Seite steuert, und dem Element, das es steuert. Diese Beziehung kann Screenreader-Nutzenden helfen, besser zu verstehen, wie die Steuerung funktioniert. Häufige Anwendungsfälle sind eine Reihe von Tabs, die die Sichtbarkeit verschiedener Inhaltsabschnitte steuern, oder ein Button, der die Sichtbarkeit eines Menüs umschaltet.

Schauen wir uns ein Beispiel an, um zu sehen, wie das funktioniert. In diesem Beispiel einer tab-basierten Benutzeroberfläche haben wir ein `div`-Element mit einer Gruppe von drei Buttons:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<div role="tablist">
  <button role="tab" id="tab1" aria-controls="section1" aria-selected="true">
    Tab 1
  </button>
  <button role="tab" id="tab2" aria-controls="section2" aria-selected="false">
    Tab 2
  </button>
  <button role="tab" id="tab3" aria-controls="section3" aria-selected="false">
    Tab 3
  </button>
</div>
```

Das `div` verwendet `role="tablist"`, um anzuzeigen, dass es als Container-Element für eine Gruppe von Tabs dient.

Jeder Button stellt einen Tab dar und hat ein `role`-Attribut mit dem Wert `tab`. In den meisten tab-basierten Benutzeroberflächen ist das erste Tab-Panel standardmäßig sichtbar, daher hat der erste Tab-Button ein `aria-selected`-Attribut mit dem Wert `true`, um anzuzeigen, dass der zugehörige Inhaltsabschnitt aktuell sichtbar ist. Das `aria-controls`-Attribut wird verwendet, um jeden Button mit dem Inhaltsabschnitt zu verknüpfen, den er steuert.

Hier sind die drei Inhaltsabschnitte, die den Tabs entsprechen:

```js
const tabs = document.querySelectorAll('[role="tab"]');
const tabList = document.querySelector('[role="tablist"]');

tabs.forEach(tab => {
  tab.addEventListener('click', () => {
    activateTab(tab);
  });

  tab.addEventListener('keydown', (e) => {
    const key = e.key;
    const currentIndex = Array.from(tabs).indexOf(tab);
    let newIndex = null;

    if (key === 'ArrowRight') {
      newIndex = (currentIndex + 1) % tabs.length;
    } else if (key === 'ArrowLeft') {
      newIndex = (currentIndex - 1 + tabs.length) % tabs.length;
    } else if (key === 'Home') {
      newIndex = 0;
    } else if (key === 'End') {
      newIndex = tabs.length - 1;
    }

    if (newIndex !== null) {
      tabs[newIndex].focus();
      activateTab(tabs[newIndex]);
    }
  });
});

function activateTab(tab) {
  const tabPanels = document.querySelectorAll('[role="tabpanel"]');

  tabs.forEach(t => {
    t.setAttribute('aria-selected', 'false');
    t.setAttribute('tabindex', '-1');
  });

  tabPanels.forEach(panel => panel.hidden = true);

  tab.setAttribute('aria-selected', 'true');
  tab.removeAttribute('tabindex');

  const panelId = tab.getAttribute('aria-controls');
  const panel = document.getElementById(panelId);
  panel.hidden = false;
  tab.focus();
}

```

:::

Jeder Inhaltsabschnitt hat ein `role`-Attribut mit dem Wert `tabpanel` und ein `aria-labelledby`-Attribut, das auf den entsprechenden Tab zeigt, um dem Panel einen zugänglichen Namen zu geben. Das `hidden`-Attribut wird verwendet, um die Inhaltsabschnitte auszublenden, die aktuell nicht ausgewählt sind. Jede Abschnitts-ID entspricht dem Wert des `aria-controls`-Attributs am entsprechenden Button. Die `section1`-ID entspricht dem `aria-controls`-Attribut am ersten Button, `section2` entspricht dem `aria-controls`-Attribut am zweiten Button, und `section3` entspricht dem `aria-controls`-Attribut am dritten Button. So wird die Verknüpfung zwischen den Tabs und ihren Inhaltsabschnitten hergestellt.

Um zwischen den verschiedenen Abschnitten zu wechseln, müssen Sie JavaScript verwenden, um das `hidden`-Attribut am Abschnitt und das `aria-selected`-Attribut am Button basierend darauf zu aktualisieren, welcher Abschnitt aktuell sichtbar ist. Es kann immer nur ein Abschnitt sichtbar sein, und dieser sollte nicht das `hidden`-Attribut haben, während `aria-selected` auf seinem Button auf `true` gesetzt sein sollte. Die übrigen ausgeblendeten Abschnitte sollten alle das `hidden`-Attribut haben, und `aria-selected` sollte auf ihren Buttons auf `false` gesetzt sein.

Tabs sind ein häufiger Anwendungsfall für das `aria-controls`-Attribut, aber es kann auch in anderen Szenarien verwendet werden, in denen ein Element die Sichtbarkeit oder das Verhalten eines anderen Elements steuert. Weitere Beispiele sind Akkordeons, Dropdown-Menüs und Modals.

Wenn Sie das nächste Mal eine Benutzeroberfläche entwerfen, die die Sichtbarkeit von Elementen steuert, sollten Sie das `aria-controls`-Attribut verwenden, um die Beziehung zwischen dem steuernden Element und dem gesteuerten Element herzustellen.
