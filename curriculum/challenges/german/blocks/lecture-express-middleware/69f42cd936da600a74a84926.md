# --description--

Middleware ist ein grundlegendes Konzept in Express und ermöglicht es uns, Anfragen und Antworten zwischen Client und Server zu verarbeiten. Middleware-Funktionen haben Zugriff auf das Anfrageobjekt, das Antwortobjekt und die next-Funktion im Anfrage-Antwort-Zyklus der Anwendung.

Lassen Sie uns das aufschlüsseln:

## Was ist Middleware?

Middleware ist im Wesentlichen eine Funktion, die während des Lebenszyklus einer Anfrage an den Server ausgeführt wird. Sie übernimmt verschiedene Aufgaben wie das Protokollieren von Anfragen, das Behandeln von Fehlern, das Parsen eingehender Anfragen oder das Bereitstellen statischer Dateien.

Express bietet eine bequeme Möglichkeit, Middleware zu definieren, die eingehende Anfragen verarbeitet, bevor sie die Route-Handler erreichen.

Wie Middleware funktioniert:

Wenn eine Anfrage an eine Express-Anwendung gestellt wird, wird Middleware in der Reihenfolge ausgeführt, in der sie hinzugefügt wurde. Der Ablauf sieht folgendermaßen aus:

* Die Anfrage wird an den Server gesendet.
* Middleware-Funktionen verarbeiten die Anfrage.
* Wenn die Middleware `next()` aufruft, übergibt sie die Kontrolle an die nächste Middleware oder den nächsten Route-Handler.
* Wenn die Middleware `next()` nicht aufruft, stoppt der Anfrage-Antwort-Zyklus.

```js
const express = require('express')
const app = express()

// Simple middleware that logs request info
app.use((req, res, next) => {
  console.log('Request URL:', req.url)
  console.log('Request Method:', req.method)
  next() // Passes control to the next middleware
})

app.get('/', (req, res) => {
  res.send('Hello World!')
})

app.listen(3000, () => {
  console.log('Server running on http://localhost:3000')
})
```

`app.use()`: Fügt Middleware global zu Ihrer Anwendung hinzu.

In diesem Beispiel protokolliert die Middleware Details der Anfrage und ruft `next()` auf, um zur nächsten Middleware oder zum nächsten Route-Handler weiterzugehen.

Middleware ist eine mächtige Funktion in Express, mit der Sie Anfragen an verschiedenen Stellen abfangen und verändern können. Sie können Middleware verwenden, um Dinge wie Protokollierung, Authentifizierung oder sogar das Ändern von Anfrage- und Antwortobjekten zu behandeln, bevor diese Ihre Route-Handler erreichen.
