# --interactive--

Il concatenamento di metodi è una tecnica in cui chiami diversi metodi uno dopo l’altro. Puoi usare il concatenamento di metodi su molti tipi di valori in JavaScript, inclusi stringhe, array e oggetti. Anche se le stringhe sono valori primitivi, JavaScript le avvolge temporaneamente in un oggetto `String` quando usi un metodo su una stringa.

Vediamo un esempio che usa diversi metodi per le stringhe:

:::interactive_editor

```js
const result = "  Hello, World!  "
  .trim()
  .toLowerCase()
  .replace("world", "JavaScript");

console.log(result); // "hello, JavaScript!"
```

:::

In questo esempio, partiamo da una stringa e eseguiamo tre operazioni in sequenza: rimuovere gli spazi bianchi, convertire in minuscolo e sostituire la stringa `world` con `JavaScript`. Ogni metodo restituisce una nuova stringa, che diventa l’obiettivo della chiamata al metodo successivo.

Il concatenamento di metodi può migliorare molto la leggibilità del codice quando lavori con operazioni complesse.

Per esempio, considera questo esempio che usa i metodi `filter`, `map` e `reduce`:

:::interactive_editor

```js
const transactions = [
  { amount: 100, type: "credit" },
  { amount: 20, type: "cash" },
  { amount: 150, type: "credit" },
  { amount: 50, type: "cash" },
  { amount: 75, type: "credit" }
];

const totalCreditWithBonus = transactions
  .filter((transaction) => transaction.type === "credit")
  .map((transaction) => transaction.amount * 1.1)
  .reduce((sum, amount) => sum + amount, 0);

console.log(totalCreditWithBonus); // 357.5
```

:::

In questo esempio, abbiamo un array di transazioni dove ogni oggetto ha un importo e un tipo di pagamento con carta di credito o contanti.

Prima filtriamo le transazioni e creiamo un nuovo array contenente solo quelle con carta di credito. Poi concatenamo il metodo `map` al risultato filtrato e per ogni importo della transazione lo moltiplichiamo per `1.1`, che rappresenta un bonus `10%`.

Infine, prendiamo quel risultato e concatenamo il metodo `reduce` per sommare tutti gli importi, ottenendo `357.5`.

Anche se il concatenamento di metodi può rendere il codice più conciso e leggibile, è importante usarlo con giudizio.

Catene molto lunghe possono diventare difficili da eseguire il debug, perché non è subito chiaro quale passaggio della catena potrebbe causare un problema. Spesso è buona pratica spezzare catene molto lunghe in più passaggi per avere più chiarezza e facilitare il debugging.
