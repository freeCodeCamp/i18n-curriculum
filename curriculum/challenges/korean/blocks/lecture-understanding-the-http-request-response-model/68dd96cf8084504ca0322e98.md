# --description--

HTTP 요청-응답 모델은 월드 와이드 웹(WWW)을 구동하는 기본 통신 패턴입니다. 이 모델은 효율성을 높이는 여러 구성 요소를 포함하여 모두가 웹에서 자료에 접근할 수 있게 합니다.

HTTP 요청-응답 모델이 무엇인지, 이를 구성하는 요소들, 그리고 이 요소들이 어떻게 협력하여 웹을 구동하는 데 효율성을 가져오는지 살펴보겠습니다.

HTTP 요청-응답 모델은 웹 상호작용에서 클라이언트, 보통 웹 브라우저가 서버에 **HTTP 요청**을 보내고, 서버가 요청을 처리한 후 **HTTP 응답**을 다시 보내는 통신 패턴입니다.

HTTP 요청-응답 모델을 구성하는 요소는 다음과 같습니다:

* **클라이언트**: 서버에 HTTP 요청을 보내고 받은 HTTP 응답을 해석하는 브라우저, 모바일 앱 또는 기타 소프트웨어입니다.
    
* **요청**: 클라이언트가 서버에 보내는 메시지로, 메서드, URL, 헤더, 그리고 선택적 본문을 포함합니다.
    
* **HTTP 메서드**: 수행할 작업으로, `GET`는 데이터 조회, `POST`는 데이터 전송, `PATCH`와 `PUT`는 데이터 수정, `DELETE`는 데이터 삭제를 의미합니다.
    
* **URL (Uniform Resource Locator)**: 서버에서 요청하는 자원입니다.
    
* **헤더**: 요청 또는 응답과 함께 전송되는 메타데이터입니다. 예를 들어 `Content-Type`와 `Authorization`가 있습니다.
    
* **본문**: `POST`, `PUT`, `PATCH`, `DELETE` 요청과 함께 전송되는 데이터로, 보통 JSON 또는 폼 데이터 형식입니다.
    
* **서버**: 요청을 처리하고 응답을 생성하는 백엔드 시스템입니다. 서버 사이드 프로그래밍을 지원하는 어떤 프로그래밍 언어로도 작성할 수 있습니다.
    
* **응답**: 서버가 다시 보내는 메시지로, 상태 코드, 헤더, 그리고 선택적 본문을 포함합니다.
    
* **상태 코드 또는 응답 코드**: 성공, 오류, 또는 리다이렉트를 나타내는 숫자 코드입니다.
    
* **응답 본문**: 실제로 전송되는 데이터로, 보통 HTML, JSON, 또는 XML 형식입니다.

그렇다면 이 모든 요소가 HTTP 요청-응답 주기에서 어떻게 함께 작동할까요?

1. **클라이언트가 요청을 시작합니다**
    
    * 브라우저, 모바일 앱, 또는 API 클라이언트가 서버에 도달할 HTTP 요청을 보냅니다.
        
    * 요청에는 `GET`, `POST`, `PUT`, `PATCH`, `DELETE` 같은 메서드, URL, 헤더, 그리고 때로는 본문이 포함됩니다.
        
2. **요청이 서버로 전송됩니다**
    
    * 요청은 인터넷을 통해 프로그래밍 언어로 작성된 백엔드 시스템인 서버로 전달됩니다.
        
    * 서버는 요청을 받고 관련 정보를 추출합니다.
        
3. **서버가 요청을 처리합니다**
    
    * 서버는 요청 메서드와 URL을 확인합니다.
        
    * 필요하면 인증(authorization 같은 헤더를 통해)을 검증합니다.
        
    * 필요 시 데이터베이스와 상호작용하여 데이터를 조회하거나 수정합니다.
        
4. **서버가 응답을 준비합니다**
    
    * 서버는 200, 404, 500 등의 상태 코드가 포함된 응답을 생성합니다.
        
    * 응답에는 예를 들어 `Content-Type: application/json` 같은 헤더가 포함됩니다.
        
    * 해당하는 경우 JSON, HTML, 또는 XML 형식의 응답 본문이 추가됩니다.
        
5. **응답이 클라이언트로 전송됩니다**
    
    * 서버는 인터넷을 통해 응답을 다시 보냅니다.
        
    * 클라이언트는 응답을 받고 데이터를 처리합니다.
        
6. **클라이언트가 응답을 처리합니다**
    
    * 응답이 성공적일 경우(상태 코드 200), 클라이언트는 데이터를 표시하거나 웹페이지에 렌더링합니다.
        
    * 오류가 발생하면(`404 Not Found`, `500 Internal Server Error` 등), 클라이언트는 오류 메시지를 표시할 수 있습니다.

이 주기는 클라이언트가 새 요청을 할 때마다 반복됩니다.
