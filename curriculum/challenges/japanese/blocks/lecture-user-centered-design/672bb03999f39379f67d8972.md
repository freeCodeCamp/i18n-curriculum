# --interactive--

**注意**：いくつかのインタラクティブな例では、まだ学んでいないCSSが使われている場合があります。コードのすべてを理解しようと心配しないでください。これらの例の目的は、デザインコンセプトのプレビューを見せて、仕組みをよりよく理解してもらうことです。プレビューを見るには、インタラクティブエディターを有効にしてください。

プログレッシブ・ディスクロージャーは、ユーザーの現在の活動に基づいて関連するコンテンツだけを表示し、それ以外は隠すデザインパターンです。これは認知負荷を減らし、ユーザー体験をより直感的にするために行われます。

:::interactive_editor

<div class="progressive-disclosure">

    Show More Details
  </button>
  <div id="extra-content" class="hidden" tabindex="-1">
    <p>
      Here are additional details that are revealed only when you want to see them.
      This keeps the interface clean and reduces cognitive load.

  </div>
</div>

    content.focus();
  }

例えば、Googleの検索ページを使うとき、ほとんどの検索はシンプルです。だから、オプションが非常に少ないシンプルなユーザーインターフェースが理にかなっています。しかし、より高度な検索が必要な場合に備えて、Googleは追加のオプションを表示する高度な検索オプションを提供しています。

Googleの高度な検索オプションは、Googleのホームページの設定メニューから見つけることができます。または`google.com/advanced_search`にアクセスしてください。

もう一つのプログレッシブ・ディスクロージャーの例は、eコマースサイトの製品にある「詳細を見る」ボタンです。Amazonのようなサイトにアクセスすると、さまざまなカテゴリーの製品画像のリストが表示されます。これにより、ユーザーは簡単に買い物ができ、情報過多で圧倒されることがありません。

ユーザーが特定の製品に興味を持った場合、その製品画像をクリックして詳細情報を表示できます。すべての情報を一度に表示するとユーザーが圧倒される可能性があります。プログレッシブ・ディスクロージャーを使うことで、ユーザーは準備ができたときに詳細情報を見ることを選べます。

document.querySelectorAll('.details-btn').forEach(button => {
  button.addEventListener('click', () => {
    const product = button.closest('.product');
    const details = product.querySelector('.product-details');
    const isHidden = details.classList.toggle('hidden');
    product.setAttribute('aria-expanded', !isHidden);
    if (!isHidden) {
      details.focus();
      button.textContent = 'Hide details';
    } else {
      button.textContent = 'More details';
    }
  });
});

もう一つのポイントは、ユーザーが追加の機能や情報にアクセスできる単一のアクセスポイントを提供することです。これはページ上で常に見えるボタンやリンクであることが望ましいです。複数のアクセスポイントを設けるとユーザーが混乱し、ユーザー体験が悪くなる場合があります。
