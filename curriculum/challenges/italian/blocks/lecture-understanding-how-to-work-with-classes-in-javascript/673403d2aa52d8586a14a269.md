# --interactive--

Nella programmazione, l'ereditarietà ti permette di definire classi che ereditano proprietà e metodi da altre classi.

Per esempio, un `Car` è un tipo di `Vehicle`, quindi potresti definire una classe `Vehicle` con le proprietà e i metodi più generali comuni a tutti i tipi di veicoli e poi definire una classe `Car` che eredita le proprietà e i metodi della classe `Vehicle`.

Questa relazione "è un" è caratteristica dell'ereditarietà. In questo esempio, `Car` è una forma più specializzata di `Vehicle`.

Ci sono due concetti principali che dovresti conoscere per iniziare a lavorare con l'ereditarietà: classe genitore e classe figlia.

Una classe genitore è una classe che funge da modello per altre classi. Definisce proprietà e metodi che vengono ereditati da altre classi.

Una classe figlia è una classe che eredita le proprietà e i metodi di un'altra classe. Le classi figlie possono anche estendere la funzionalità delle loro classi genitore aggiungendo nuove proprietà e metodi.

Nel nostro esempio precedente, `Vehicle` sarebbe la classe genitore della classe `Car` mentre la classe `Car` sarebbe la classe figlia di `Vehicle`.

Una classe genitore può avere più classi figlie.

In JavaScript, usiamo la parola chiave `extends` per implementare l'ereditarietà. Questa parola chiave indica che una classe è la classe figlia di un'altra classe.

In questo esempio, in cui abbiamo solo le prime righe delle definizioni delle classi, puoi vedere che la classe `Car` estende la classe `Vehicle`, quindi `Car` eredita da `Vehicle`.

```js
class Vehicle {
  // Implementation of Vehicle...
}

class Car extends Vehicle {
  // Implementation of Car...
}
```

Supponiamo che la classe `Vehicle` abbia due proprietà: `brand` e `year`. Puoi vedere questo nel codice aggiornato.

`Car` ha un metodo chiamato `honk`. Tuttavia, nota che la classe `Car` non ha un proprio costruttore. Vediamo se eredita le proprietà.

```js
class Vehicle {
  constructor(brand, year) {
    this.brand = brand;
    this.year = year;
  }
}

class Car extends Vehicle {
  honk() {
    console.log("Honk! Honk!");
  }
}
```

Per definire un'istanza di `Car`, dovrai passare due argomenti: `brand` e `year`. In questo caso, la marca è `freeCodeCamp Motors` e l'anno è `2019`.

```js
let myCar = new Car("freeCodeCamp Motors", 2019);
```

Se provi a consultare queste proprietà e chiamare il metodo usando la notazione a punto:

```js
console.log(myCar.brand);
console.log(myCar.year);
myCar.honk();
```

Ecco l'esempio completo:

Infatti, l'output è corretto, quindi queste proprietà e metodo sono definiti nell'istanza `Car` e ti sei appena risparmiato molta ripetizione di codice ereditando queste proprietà dalla classe `Vehicle`.

In questo esempio, la classe figlia non aveva proprietà aggiuntive. Per questo motivo la classe non aveva un costruttore, solo un metodo.

Se invece hai bisogno di aggiungere proprietà aggiuntive, dovrai definire un costruttore.

Vediamo un esempio.


```js
class Vehicle {
  constructor(brand, year) {
    this.brand = brand;
    this.year = year;
  }
}

class Car extends Vehicle {
  constructor(brand, year, numDoors) {
    super(brand, year);
    this.numDoors = numDoors;
  }
}
```

Questa volta, aggiungeremo una proprietà `numDoors` alla sottoclasse `Car`.

Puoi vedere che ora questa classe ha un costruttore nel codice aggiornato. All'interno del costruttore, c'è una chiamata a `super()`, passando gli argomenti `brand` e `year`.

`super()` invoca il costruttore della superclasse, quindi usando `super(brand, year)` definirai essenzialmente le proprietà della superclasse nella sottoclasse. Questa è un'altra terminologia che puoi usare: superclasse è la classe genitore e sottoclasse è la classe figlia.

Poi, la riga `this.numDoors = numDoors` definisce una nuova proprietà che appartiene esclusivamente alla classe `Car`, il numero di porte (`numDoors`).

Questo è un modo per estendere la classe `Vehicle`, aggiungendo una proprietà che è esclusiva delle istanze `Car`.

Questo è un esempio di estendibilità, uno dei vantaggi fondamentali dell'ereditarietà.

Ora, se crei un'istanza della classe `Car`, vedrai che ha le tre proprietà: `brand`, `year` e `numDoors`. Dovrai anche passare gli argomenti necessari quando crei l'istanza.

```js
let myCar = new Car("freeCodeCamp Motors", 2019, 4);
```

Se stampi i valori di queste proprietà con `console.log()`:

```js
console.log(myCar.brand);
console.log(myCar.year);
console.log(myCar.numDoors);
```

Ecco l'esempio completo:

I principali vantaggi dell'ereditarietà sono la riutilizzabilità del codice, la modularità, l'estendibilità e una struttura del codice migliorata.

Implementando una gerarchia, puoi riutilizzare il codice che hai già scritto per una classe genitore nella classe figlia, evitando ripetizioni.

L'ereditarietà promuove anche la modularità suddividendo sistemi complessi in componenti più semplici nella gerarchia.

Inoltre, poter estendere la funzionalità della classe genitore rende più facile adattarsi a requisiti che cambiano e aggiungere nuove funzionalità in seguito nel processo di sviluppo.

Infine, la struttura gerarchica può rendere il tuo codice più facile da capire e mantenere.

Questi sono i fondamenti dell'ereditarietà. Capendo come funziona l'ereditarietà, puoi progettare programmi orientati agli oggetti ben strutturati, manutenibili ed estendibili.
