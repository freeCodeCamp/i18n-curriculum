# --description--

Die <dfn>Arity</dfn> einer Funktion ist die Anzahl der Argumente, die sie benötigt. <dfn>Currying</dfn> einer Funktion bedeutet, eine Funktion mit N Argumenten in N Funktionen mit jeweils einem Argument umzuwandeln.

Mit anderen Worten: Es wird eine Funktion so umstrukturiert, dass sie ein Argument entgegennimmt und dann eine weitere Funktion zurückgibt, die das nächste Argument entgegennimmt, und so weiter.

Hier ist ein Beispiel:

```js
function unCurried(x, y) {
  return x + y;
}

function curried(x) {
  return function(y) {
    return x + y;
  }
}

const curried = x => y => x + y

curried(1)(2)
```

`curried(1)(2)` würde `3` zurückgeben.

Das ist in Ihrem Programm nützlich, wenn Sie nicht alle Argumente einer Funktion auf einmal übergeben können. Sie können jeden Funktionsaufruf in einer Variablen speichern, die die zurückgegebene Funktionsreferenz hält, welche das nächste Argument entgegennimmt, sobald es verfügbar ist. Hier ist ein Beispiel mit der curried Funktion aus dem obigen Beispiel:

```js
const funcForY = curried(1);
console.log(funcForY(2)); // 3
```

Ähnlich kann man <dfn>partielle Anwendung</dfn> als das Anwenden einiger Argumente auf eine Funktion zu einem Zeitpunkt beschreiben, wobei eine weitere Funktion zurückgegeben wird, die auf weitere Argumente angewendet wird. Hier ist ein Beispiel:

```js
function impartial(x, y, z) {
  return x + y + z;
}

const partialFn = impartial.bind(this, 1, 2);
partialFn(10); // 13
```
