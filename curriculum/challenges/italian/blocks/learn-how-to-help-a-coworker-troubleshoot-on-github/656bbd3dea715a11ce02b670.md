# --description--

Nel coding, soprattutto quando si usano sistemi di controllo versione come Git, `push` significa caricare le modifiche da un repository locale a uno remoto.

Il passato `pushed` indica che questa azione è già stata completata.

Per esempio, `He pushed his code to the GitHub repository` significa che ha caricato le sue recenti modifiche al codice su GitHub.

# --questions--

## --text--

Cosa significa `push` nel contesto del controllo versione?

## --answers--

Scaricare le modifiche da un repository remoto

### --feedback--

`Push` in realtà significa caricare le modifiche su un repository remoto, non scaricare.

---

Eliminare le modifiche da un repository

### --feedback--

`Push` non riguarda l'eliminazione delle modifiche da un repository.

---

Revisionare le modifiche in un repository

### --feedback--

`push` non riguarda la revisione delle modifiche in un repository.
