# --description--

Quando costruisci applicazioni web, è importante pensare all'organizzazione dei tuoi file e cartelle. Questo aiuta a mantenere il codice ordinato e più facile da gestire.

Diamo un'occhiata a un esempio di struttura di cartelle per un progetto HTML e CSS:

```bash
.
├── /assets
│   ├── /images
│   │   ├── logo.png
│   │   ├── banner.jpg
│   │   └── icons.svg
│   ├── /fonts
│   │   ├── custom-font.woff
│   │   └── custom-font.woff2
├── /css
│   ├── main.css
│   ├── about.css
│   └── contact.css
├── index.html
├── about.html
├── contact.html
└── README.md
```

In cima all'esempio, c'è un singolo punto, che rappresenta la directory radice.

La directory radice è la directory di livello più alto in un file system, che funge da punto di partenza per tutti i percorsi dei file e contiene tutte le altre directory e file. Una directory è un altro nome per cartella.

All'interno della directory radice, la prima directory mostrata è la directory `assets`. Il termine `assets` si riferisce a tutti i file usati nel progetto, come immagini, caratteri tipografici o altre risorse.

In questo esempio, ci sono due directory all'interno della directory `assets` chiamate `images` e `fonts`.

La directory successiva sarebbe la directory `css`. A volte questa viene chiamata anche directory degli stylesheet. Qui è dove conserveresti tutti i tuoi file CSS.

Sotto la directory `css` ci sono i file HTML. Il file `index.html` rappresenta la homepage, mentre `about.html` e `contact.html` rappresentano rispettivamente le pagine About e Contact.

Infine, c'è un file `README.md`. I file `README` sono comunemente usati per fornire informazioni utili sul progetto, come cosa fa, come usarlo e altri dettagli rilevanti.

L'estensione `.md` indica Markdown, un linguaggio di markup leggero spesso usato per scrivere documentazione.

Organizzando il progetto in risorse, CSS e file HTML, diventa facile trovare ciò che cerchi. Questo rende anche semplice per altri sviluppatori contribuire al tuo progetto.

È importante notare che questo è solo un esempio di come puoi organizzare i tuoi file e cartelle. Esistono molti modi diversi per strutturare un progetto.
