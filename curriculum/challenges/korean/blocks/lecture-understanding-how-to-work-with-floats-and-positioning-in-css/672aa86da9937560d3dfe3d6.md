# --interactive--

CSS에서 플로트는 원래 텍스트가 이미지 같은 요소 주위를 감싸도록 하기 위해 고안된 기법입니다. 하지만 시간이 지나면서 개발자들은 플로트를 창의적으로 레이아웃 디자인에 활용하는 새로운 방법을 발견했습니다. Flexbox와 Grid 같은 최신 레이아웃 방법이 더 일반적으로 사용되지만, 특히 오래된 코드 작업이나 특정 레이아웃 효과를 내야 할 때 플로트를 이해하는 것은 여전히 중요합니다.

요소가 플로트되면, 일반 문서 흐름에서 벗어나 컨테이너의 왼쪽이나 오른쪽으로 밀려납니다. 그 다음에 오는 내용은 플로트된 요소 주위를 감싸며 남은 공간을 채웁니다. 대표적인 용도는 이미지 주위에 텍스트를 감싸는 것으로, 이미지를 한쪽으로 플로트하고 텍스트가 그 주위를 감싸게 합니다. 이 기법은 특히 기사나 블로그에서 이미지와 텍스트를 나란히 배치할 때 여전히 널리 사용됩니다. 다음은 코드 예시입니다:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<div class="container">
  <img src="https://placehold.co/150x150" alt="Placeholder Image">
</div>
```

```css
.container {
  border: 1px solid black;
  padding: 10px;
}

img {
  float: left;
  margin-right: 20px;
}
```

:::

위 예시에서 컨테이너는 플로트된 이미지를 감싸지 않습니다. 이미지는 일반 문서 흐름에서 벗어나 있으며, 컨테이너는 플로트된 자식 요소를 "보지" 못해 높이가 0으로 축소되었습니다.

플로트는 Flexbox와 Grid가 널리 퍼지기 전 다중 열 레이아웃을 만들 때도 인기가 있었습니다. 요소들을 나란히 플로트시켜 수평으로 정렬된 열을 만들 수 있었습니다. 하지만 플로트를 사용할 때는 자식 요소가 플로트되면 부모 요소가 축소되는 문제를 처리하는 것이 중요합니다. 이 문제를 해결하기 위해 `container` 클래스에 clearfix 기법을 적용했습니다.

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<div class="container">
  <img src="https://placehold.co/150x150" alt="Placeholder Image">
  <p> This is an example of text flowing around a floated image.</p>
</div>
```

```css
.container {
  border: 1px solid black;
}

/* Clearfix CSS */
.container::after {
  content: "";  
  display: block;
  clear: both;
}

img {
  float: left;
  margin-right: 20px;
}
```

:::

- `::after`은 컨테이너 내용 뒤에 보이지 않는 블록을 추가하는 가상 요소입니다.
- `content: ""`는 가상 요소가 존재하지만 내용을 표시하지 않도록 합니다.
- `display: block`는 가상 요소를 블록 레벨 요소로 만듭니다.
- `clear: both`는 가상 요소가 위에 있는 플로트된 요소 양쪽을 모두 지우도록 합니다.

위 예시에서는 축소 현상을 더 잘 보이게 하기 위해 새 단락 요소를 추가했습니다. 단락은 일반 문서 흐름에 남아 있어 컨테이너가 단락을 감쌀 만큼 확장됩니다. 그런 다음 clearfix 기법을 적용해 축소 문제를 고치고 컨테이너의 테두리가 제대로 표시되도록 했습니다. clearfix 규칙을 제거해 컨테이너가 다시 축소되는 것을 확인할 수 있습니다.

clearfix 기법은 부모 요소가 플로트된 자식 요소를 제대로 감싸도록 보장합니다. clearfix는 플로트된 내용 뒤에 clear 속성을 추가해 부모 컨테이너가 플로트된 자식 요소를 "보게" 만듭니다.

플로트는 Flexbox와 Grid 같은 최신 기법 때문에 복잡한 레이아웃에 더 이상 기본 방법은 아니지만, 특정 상황에서는 여전히 중요한 역할을 합니다. 이미지 주위에 텍스트를 감싸거나 레거시 프로젝트 작업 시 플로트가 어떻게 작동하고 어떻게 제대로 지우는지 이해하는 것이 잘 구조화되고 반응형 레이아웃을 유지하는 데 핵심입니다.
