# --description--

キーワード `let` は変数を宣言する新しい方法のひとつにすぎません。ES6では、`const` キーワードを使って変数を宣言することもできます。

`const` は `let` と同じ素晴らしい機能をすべて備えており、さらに `const` を使って宣言された変数は読み取り専用であるという特典があります。これは定数の値であり、`const` で変数に代入されたら再代入できません。

```js
const FAV_PET = "Cats";
FAV_PET = "Dogs";
```

`FAV_PET` の値を再代入したため、コンソールにエラーが表示されます。

再代入したくない変数は常に `const` キーワードを使って名前を付けるべきです。これにより、定数として扱うべき変数を誤って再代入しようとした場合に役立ちます。

**注意:** 開発者は不変の値には大文字の変数識別子を使い、変更可能な値（オブジェクトや配列）には小文字やキャメルケースを使うことが一般的です。オブジェクト、配列、不変値と可変値については後のチャレンジで詳しく学びます。また、後のチャレンジでは大文字、小文字、キャメルケースの変数識別子の例も見られます。

# --instructions--

すべての変数が `let` または `const` を使って宣言されるようにコードを変更してください。変数を変更したい場合は `let` を使い、変数を定数にしたい場合は `const` を使ってください。また、`const` で宣言された変数の名前を一般的な慣習に合わせて変更してください。変数に代入されている文字列は変更しないでください。

# --hints--

`var` はコードに存在してはいけません。

```js
assert.notMatch(code, /var/g);
```

`fCC` はすべて大文字に変更してください。

```js
assert.match(code, /(FCC)/);
assert.notMatch(code, /(fCC)/);
```

`FCC` は `const` で宣言された定数変数であるべきです。

```js
assert.match(code, /const\s+FCC/);
```

`FCC` に代入されている文字列は変更しないでください。

```js
assert.equal(FCC, 'freeCodeCamp');
```

`fact` は `let` で宣言されるべきです。

```js
assert.match(code, /(let\s+fact)/g);
```

`console.log` は `FCC` と `fact` の変数を出力するように変更してください。

```js
assert.match(code, /console\.log\(\s*FCC\s*\,\s*fact\s*\)\s*;?/g);
```
