# --description--

- **同步（Synchronous）JavaScript** 會依序執行，每個操作都會等前一個完成後才繼續。
- **非同步（Asynchronous）JavaScript** 允許多個操作在背景同時執行，不會阻塞主執行緒。
- **執行緒（Thread）** 是一組可以獨立於主程式流程執行的指令序列。
- **回呼函式（Callback functions）** 是作為引數傳遞給其他函式，並在操作完成或事件發生時執行的函式。

## JavaScript 引擎與 JavaScript 執行期

- **JavaScript 引擎** 是一個在網頁瀏覽器中執行 JavaScript 程式碼的程式。它像一個轉換器，將你的程式碼轉換成電腦能理解並依此運作的指令。
- V8 是 Google 開發的 JavaScript 引擎範例。
- **JavaScript runtime** 是執行 JavaScript 程式碼的環境。它包含處理並執行程式碼的 JavaScript 引擎，以及像是網頁瀏覽器或 Node.js 等額外的特性。

## Fetch API

- Fetch API 讓網頁應用程式可以發送網路請求，通常用來從伺服器擷取或傳送資料。它提供一個 `fetch()` 方法，你可以用來發送這些請求。
- 你可以用 Fetch API 擷取文字、圖片、音訊、JSON 及其他型態的資料。

## Fetch API 的 HTTP 方法

Fetch API 支援多種 HTTP 方法與伺服器互動。最常用的方法有：

- **GET**：用來從伺服器擷取資料。預設情況下，Fetch API 會使用 `GET` 方法來擷取資料。

```js
fetch('https://api.example.com/data')
```

若要使用擷取到的資料，必須用 `.json()` 方法將其轉換為 JSON 格式：

```js
fetch('https://api.example.com/data')
  .then(response => response.json())
  .then(data => console.log(data))
```

在這段程式碼中，`fetch()` 傳回一個 Promise，而 `.then()` 處理常式將決議的回應轉換為 JSON 格式。

- **POST**：用來傳送資料到伺服器。`POST` 方法用來在伺服器上建立新資源。

```js
fetch('https://api.example.com/users', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({
    name: 'John Doe',
    email: 'john@example.com'
  })
})
```

在這個範例中，我們發送一個 `POST` 請求來建立新使用者。我們指定方法為 `POST`，設置適當的標頭，並在 body 中包含要傳送的資料。body 必須是字串，因此我們用 `JSON.stringify()` 將物件轉換為 JSON 字串。

- **PUT**：用來在伺服器上更新資料。`PUT` 方法用來更新伺服器上的現有資源。

```js
fetch('https://api.example.com/users/45', {
  method: 'PUT',
  headers: {
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({
    name: 'John Smith',
    email: 'john@example.com'
  })
})
```

在這個範例中，我們正在更新 URL 結尾指定 ID 的 `45`。我們在程式碼中使用了 `PUT` 方法，並指定要更新的資料作為 body，這些資料會用來更新指定的資料。

- **DELETE**：用來在伺服器上刪除資料。`DELETE` 方法用來刪除伺服器上的資源。

```js
fetch('https://api.example.com/users/45', {
  method: 'DELETE'
})
```

在這個範例中，我們發送一個 `DELETE` 請求來移除 ID 為 `45` 的使用者。

## Promise 與 Promise 串鏈

- **Promises** 是表述非同步操作最終完成或失敗及其結果值的物件。只有在非同步操作完成後，才會知道 promise 的值。
- 這裡有一個創建簡單 promise 的範例：

```js
const promise = new Promise((resolve, reject) => {
  setTimeout(() => {
    resolve('Data received successfully');
  }, 2000);
});
```

- `.then()` 方法用於指定 Promise 完成時要執行的動作，而 `.catch()` 則用來處理發生的錯誤。
- 以下是使用 `.then()` 和 `.catch()` 的 Promise 範例：

```js
promise
  .then(data => {
    console.log(data);
  })
  .catch(error => {
    console.error(error);
  });
```

在上面的範例中，`.then()` 方法用來記錄 Promise 傳回的資料，而 `.catch()` 方法則用來記錄發生的錯誤。

- **Promise 串鏈**：Promise 的強大之處在於可以將多個非同步操作串接在一起。每個 `.then()` 都可以傳回新的 Promise，讓你能依序執行一連串非同步操作。
- 以下是 Promise 串鏈的範例：

```js
fetch('https://api.example.com/data')
  .then(response => response.json())
  .then(data => {
    console.log(data);
    return fetch('https://api.example.com/other-data');
  })
  .then(response => response.json())
  .then(otherData => {
    console.log(otherData);
  })
  .catch(error => {
    console.error(error);
  });
```

在上面的範例中，我們先從一個 URL 擷取資料，再根據第一次回應擷取另一個 URL 的資料，最後記錄第二次取得的資料。

`catch` 方法會處理過程中發生的任何錯誤。這表示你不需要在每個步驟都加上錯誤處理，能大幅簡化程式碼。

## 使用 `async/await` 處理 Promise

`async`/`await` 使撰寫和閱讀非同步程式碼更容易，它是建立在 Promises 之上的。

- **async**：`async` 關鍵字用來定義非同步函式。`async` 函式會傳回一個 Promise，該 Promise 會以 `async` 函式傳回的值解析。
- **await**：`await` 關鍵字用在 `async` 函式內，會暫停函式執行直到 Promise 被解析。只能在 `async` 函式內使用。
- 以下是使用 `async/await` 的範例：

```js
async function delayedGreeting(name) {
  console.log("A Messenger entered the chat...");
  await new Promise(resolve => setTimeout(resolve, 2000));
  console.log(`Hello, ${name}!`);
}

delayedGreeting("Alice");
console.log("First Printed Message!");
```

在上面的範例中，`delayedGreeting` 函式是一個 `async` 函式，會暫停 2 秒後列印問候訊息。`await` 關鍵字用來暫停函式執行直到 `Promise` 被解析。

- `async/await` 最大的優點之一是可以透過 `try/catch` 區塊進行錯誤處理。以下是範例：

```js
async function fetchData() {
  try {
    const response = await fetch('https://api.example.com/data');
    const data = await response.json();
    console.log(data);
  } catch (error) {
    console.error(error);
  }
}

fetchData();
```

在上面的範例中，`try` 區塊包含可能丟出錯誤的程式碼，而 `catch` 區塊則在發生錯誤時處理錯誤。這讓錯誤處理更直接且易讀。

## `async` 屬性

- `async` 屬性告訴瀏覽器在繼續解析 HTML 文件的同時非同步下載腳本檔案。
- 腳本下載完成後，HTML 解析會暫停，腳本會執行，然後 HTML 解析會繼續。
- 你應該為執行順序不重要的獨立腳本使用 `async`。

## `defer` 屬性

- `defer` 屬性也會非同步下載腳本，但會延緩腳本的執行直到 HTML 文件完全解析之後。
- `defer` 腳本維護它們在 HTML 文件中出現的執行順序。

- 請注意，`async` 和 `defer` 屬性對行內腳本無效，只適用於外部腳本檔案。

- 當 `async` 和 `defer` 屬性同時存在時，`async` 屬性優先。

## 地理位置 API

- Geolocation API 提供網站請求使用者位置的方法。

- 以下範例示範了 API 的 `getCurrentPosition()` 方法，用來取得使用者的當前位置。

```js
navigator.geolocation.getCurrentPosition(
  (position) => {
    console.log("Latitude: " + position.coords.latitude);
    console.log("Longitude: " + position.coords.longitude);
  },
  (error) => {
    console.log("Error: " + error.message);
  }
);
```

在這段程式碼中，我們呼叫了 `getCurrentPosition` 並傳入一個函式，當成功取得位置時該函式會被呼叫。

`position` 物件包含多種資訊，但這裡我們只取用 `latitude` 和 `longitude`。

如果取得 `position` 時發生問題，錯誤將會記錄到主控台。

- 尊重使用者的隱私非常重要，且僅在必要時請求他們的位置。
