## Kijenzi cha mfuatano wa herufi na njia ya `toString()`

- **Ufafanuzi**: Kitu cha mfuatano wa herufi hutumika kuwakilisha mfululizo wa herufi. Vitu vya mfuatano wa herufi huundwa kwa kutumia kitendakazi cha kijenzi cha `String`, ambacho hufunika thamani ya msingi ndani ya kitu.

:::interactive_editor

```js
const greetingObject = new String("Hello, world!");

console.log(typeof greetingObject); // "object"
```

:::

- **Njia ya `toString()`**: Njia hii hubadilisha thamani kuwa mwakilishi wake wa mfuatano wa herufi. Ni njia unayoweza kutumia kwa nambari, thamani za kweli au siyo kweli, safu za data, na vitu.

:::interactive_editor

```js
const num = 10;
console.log(num.toString()); // "10"

const arr = [1, 2, 3];
console.log(arr.toString()); // "1,2,3"
```

:::

Njia hii inakubali msingi wa nambari wa hiari ambao ni nambari kutoka 2 hadi 36. Msingi huu unawakilisha msingi wa nambari, kama vile msingi 2 kwa binary au msingi 8 kwa octal. Ikiwa msingi haujatolewa, hutumika msingi 10, yaani desimali.


:::interactive_editor

```js
const num = 10;
console.log(num.toString(2)); // "1010"(binary)
```

:::

## Kijenzi cha nambari

- **Ufafanuzi**: Kijenzi cha `Number` hutumika kuunda kitu cha nambari. Kitu cha nambari kina mali na njia kadhaa za msaada kama njia za `isNaN` na `toFixed`. Mara nyingi, utatumia kijenzi cha `Number` kubadilisha aina nyingine za data kuwa aina ya nambari.

:::interactive_editor

```js
const myNum = new Number("34");
console.log(typeof myNum); // "object"

const num = Number('100');
console.log(num); // 100

console.log(typeof num); // number
```

:::

## Mbinu bora za kuitia majina vigezo na vitendakazi

- **camelCasing**: Kwa mkataba, wasanidi programu wa JavaScript hutumia camel casing kwa kuitia majina vigezo na vitendakazi. Camel casing ni pale neno la kwanza likiwa herufi ndogo zote na maneno yanayofuata kuanza na herufi kubwa. Mfano `isLoading`.

- **Kuitia majina thamani za kweli au siyo kweli**: Kwa vigezo vya thamani za kweli au siyo kweli, ni mbinu ya kawaida kutumia viambishi awali kama "is", "has", au "can". 

```js
let isLoading = true;
let hasPermission = false;
let canEdit = true;
```

- **Kuitia majina vitendakazi**: Kwa vitendakazi, jina linapaswa kuonyesha wazi kinachofanywa na kitendakazi. Kwa vitendakazi vinavyorejesha thamani ya kweli au siyo kweli (vinavyoitwa mara nyingi predicates), unaweza kutumia viambishi awali "is", "has", au "can". Kwa vitendakazi vinavyopata data, ni kawaida kuanza na neno "get". Kwa vitendakazi vinavyoweka data, ni kawaida kuanza na neno "set". Kwa vitendakazi vya mshughulikiaji wa tukio, unaweza kuweka kiambishi awali "handle" au kiambishi tamati "Handler". 

```js
function getUserData() { /* ... */ }

function isValidEmail(email) { /* ... */ }

function getProductDetails(productId) { /* ... */ }

function setUserPreferences(preferences) { /* ... */ }

function handleClick() { /* ... */ }
```

- **Kuitia majina vigezo ndani ya mizunguko**: Unapotia majina vigezo vinavyotumika kama mizunguko, ni kawaida kutumia herufi moja kama `i`, `j`, au `k`.

```js
for (let i = 0; i < array.length; i++) { /* ... */ }
```

## Kufanya kazi na safu za data zenye mapengo

- **Ufafanuzi**: Inawezekana kuwa na safu za data zenye mapengo tupu. Mapengo haya yanatafsiriwa kama sehemu ambazo hazina kitu ndani yake. Hii ni tofauti na sehemu za safu za data zenye thamani ya `undefined`. Aina hizi za safu za data zinajulikana kama safu za data zenye mapengo.

:::interactive_editor

```js
const sparseArray = [1, , , 4];
console.log(sparseArray.length); // 4
```

## Linters na Formatters

- **Linters**: Linter ni chombo cha uchambuzi wa msimbo wa programu kinachobaini makosa ya programu, hitilafu, makosa ya mtindo, na miundo inayoshukiwa. Mfano wa linter maarufu ni ESLint.
- **Formatters**: Formatters ni zana zinazopanga msimbo wako kiotomatiki ili ufuate mwongozo maalum wa mtindo. Mfano wa formatter maarufu ni Prettier.

## Usimamizi wa kumbukumbu

- **Ufafanuzi**: Usimamizi wa kumbukumbu ni mchakato wa kudhibiti kumbukumbu, kuigawa inapohitajika na kuiacha huru inapokuwa haidingiki tena. JavaScript hutumia usimamizi wa kumbukumbu kiotomatiki. Hii ina maana kwamba JavaScript (hasa injini ya JavaScript katika kivinjari chako cha mtandao) inashughulikia ugawaji na kuachilia kumbukumbu kwa niaba yako. Huna haja ya kuachilia kumbukumbu waziwazi katika msimbo wako. Mchakato huu wa kiotomatiki mara nyingi huitwa "ukusanyaji wa takataka."

## Ufungaji

- **Ufafanuzi**: Ufungaji ni kitendakazi chenye upatikanaji wa vigezo katika wigo wa nje (wa muktadha wa lugha), hata baada ya kitendakazi cha nje kurudi.

:::interactive_editor

```js
function outerFunction(x) {
  let y = 10;
  function innerFunction() {
    console.log(x + y);
  }
  return innerFunction;
}

let closure = outerFunction(5);
closure(); // 15
```

## Neno kuu la `var` na Hoisting

- **Ufafanuzi**: `var` ilikuwa njia ya awali ya kutangaza vigezo kabla ya mwaka 2015. Lakini kulikuwa na matatizo fulani yanayohusiana na `var` kuhusu wigo, kutangaza tena na mengine. Ndiyo maana programu za kisasa za JavaScript hutumia `let` na `const` badala yake.
- **Kutangaza tena vigezo kwa `var`**: Ukijaribu kutangaza tena kigezo kwa kutumia `let`, basi utapata `SyntaxError`. Lakini kwa `var`, unaruhusiwa kutangaza tena kigezo.

```js
// Uncaught SyntaxError: Identifier 'num' has already been declared 
let num = 19;
let num = 18;

var myNum = 5;
var myNum = 10; // This is allowed and doesn't throw an error

console.log(myNum) // 10
```

- **`var` na wigo**: Vigezo vilivyotangazwa kwa `var` ndani ya block (kama tamko la `if` au mzunguko wa `for`) bado vinaweza kupatikana nje ya block hiyo.

:::interactive_editor

```js
if (true) {
  var num = 5;
}
console.log(num); // 5
```

:::

- **Hoisting**: Hii ni tabia ya kawaida ya JavaScript ya kusogeza matangazo ya vigezo juu ya wigo wake wakati wa awamu ya kukusanya kabla ya msimbo kutekelezwa. Unapotangaza kigezo kwa kutumia neno kuu la `var`, JavaScript husogeza tamko hilo juu ya wigo wake.

:::interactive_editor

```js
console.log(num); // undefined
var num = 5;
console.log(num); // 5
```

:::

Unapotangaza kitendakazi kwa sintaksia ya tamko la kitendakazi, jina la kitendakazi na mwili wa kitendakazi vinahamisishwa juu. Hii ina maana unaweza kuita kitendakazi kabla hujatangaza katika msimbo wako.

:::interactive_editor

```js
sayHello(); // "Hello, World!"

function sayHello() {
  console.log("Hello, World!");
}
```

:::

Matangazo ya vigezo yaliyotengenezwa kwa `let` au `const` husogezwa juu, lakini hayajaanzishwa, na huwezi kuyapata kabla ya tamko halisi katika msimbo wako. Tabia hii mara nyingi huitwa "eneo la kifo la muda".

:::interactive_editor

```js
console.log(num); // Throws a ReferenceError
let num = 10;
```

:::

## Kufanya kazi na Imports, Exports na Moduli

- **Moduli**: Hii ni kitengo cha msimbo kinachojumuisha vitendakazi, madarasa ya vitu, au vigezo vinavyohusiana. Ili kuunda moduli, andika msimbo wako wa JavaScript katika faili tofauti.
- **Exports**: Vigezo vyovyote, vitendakazi, au madarasa unayotaka kuyafanya kupatikana kwa sehemu nyingine za programu yako lazima yaondolewe wazi kwa kutumia neno kuu la `export`. Kuna aina mbili za export: export iliyopewa jina na export ya default.
- **Imports**: Ili kutumia vitu vilivyotolewa sehemu nyingine ya programu yako, unahitaji kuleta (import) kwa kutumia neno kuu la `import`. Aina za import ni import iliyopewa jina, import ya default, na import ya namespace.

```js
// Within a file called math.js, we export the following functions:

// Named export
export function add(num1, num2) {
  return num1 + num2;
}

// Default export
export default function subtract(num1, num2) {
  return num1 - num2;
}

// Within another file, we can import the functions from math.js.

// Named import - This line imports the add function.
// The name of the function must exactly match the one exported from math.js.
import { add } from './math.js';

// Default import - This line imports the subtract function.
// The name of the function can be anything.
import subtractFunc from './math.js';

// Namespace import - This line imports everything from the file.
import * as Math from './math.js';

console.log(add(5, 3)); // 8
console.log(subtractFunc(5, 3)); // 2
console.log(Math.add(5, 3)); // 8
console.log(Math.subtract(5, 3)); // 2
```
