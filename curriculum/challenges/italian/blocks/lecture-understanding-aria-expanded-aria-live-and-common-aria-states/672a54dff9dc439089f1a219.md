# --interactive--

L'attributo `aria-live` crea una regione live nella tua pagina che può essere usata per notificare agli utenti di lettori di schermo i cambiamenti dinamici del contenuto nella regione live man mano che avvengono.

Gli usi comuni delle regioni live includono messaggi visualizzati dopo che è stata eseguita un’azione (come un messaggio di errore o di conferma), contenuti che si aggiornano periodicamente (come un ticker, un marquee o un conto alla rovescia) o messaggi di stato generali (come aggiornamenti su un processo).

Poiché il focus di lettura per i lettori di schermo può essere solo in un punto alla volta, gli utenti di lettori di schermo non noteranno un cambiamento di contenuto se il loro focus è su un’altra parte della pagina. Le regioni live permettono agli utenti di lettori di schermo di essere notificati automaticamente dei cambiamenti che avvengono sulla pagina in tempo reale. Senza una regione live, gli utenti di lettori di schermo potrebbero perdere aggiornamenti importanti del contenuto disponibili agli utenti vedenti, dato che un utente vedente può scansionare l’intera pagina.

Ci sono tre valori possibili per questo attributo, basati sulla priorità dell’informazione.

Se imposti `aria-live` al valore `assertive` significa che l’aggiornamento è molto importante. Ha la priorità più alta, quindi l’utente dovrebbe essere notificato immediatamente.

Questo significa che il lettore di schermo interromperà qualsiasi annuncio stia facendo per annunciare il cambiamento di contenuto nella regione live. Tali interruzioni possono essere estremamente fastidiose, quindi il valore `assertive` dovrebbe essere usato solo per notifiche critiche o sensibili al tempo.

```html
<link rel="stylesheet" href="styles.css">

<div class="session-warning" aria-live="assertive">
  <p>Your session will expire in 30 seconds.</p>
</div>

<script src="index.js"></script>

```

```css
.session-warning {
  background-color: #ffcc00;
  color: #000;
  font-family: system-ui, sans-serif;
  font-weight: 500;
  padding: 1em 1.5em;
  border-radius: 6px;

  opacity: 0;

  transition: opacity 0.4s ease, transform 0.4s ease;
}

.session-warning.visible {
  opacity: 1;
  transform: translateY(0);
}

.session-warning.fade-out {
  opacity: 0;
  transform: translateY(10px);
}

```

```js
document.addEventListener("DOMContentLoaded", () => {
  const warning = document.querySelector(".session-warning");

  setTimeout(() => {
    warning.classList.add("visible");
  }, 100);

  setTimeout(() => {
    warning.classList.add("fade-out");
  }, 8000);

  warning.addEventListener("transitionend", () => {
    if (warning.classList.contains("fade-out")) {
      warning.remove();
    }
  });
});

```

Il valore successivo in ordine di priorità è `polite`.

Questo valore significa che l’aggiornamento non è urgente, quindi il lettore di schermo può aspettare che qualsiasi annuncio in corso sia terminato o che l’utente smetta di digitare prima di annunciare l’aggiornamento. La maggior parte delle regioni live userà il valore `polite`.

Il valore di priorità più basso per `aria-live` è `off` che significa che l’aggiornamento non sarà annunciato a meno che il contenuto non sia in un elemento che ha attualmente il focus da tastiera. Realisticamente, questo valore è quasi mai usato perché il caso d’uso è molto ristretto e non è implementato in modo coerente (se non per niente) tra i lettori di schermo. Se hai bisogno di regioni live, pianifica di usare `polite` per tutto tranne che per messaggi critici che richiedono `assertive`.

È anche importante notare che l’attributo `aria-live` non è richiesto se l’informazione aggiornata è contenuta in un elemento con un ruolo ARIA di `alert`, `log`, `marquee`, `status` o `timer`, poiché questi ruoli hanno già valori `aria-live` predefiniti. Ma il valore predefinito può essere cambiato impostando esplicitamente `aria-live` sull’elemento.

Scegliere il valore `aria-live` giusto dipende dalla priorità dell’informazione aggiornata.

Se gli aggiornamenti sono urgenti, dovresti notificare l’utente immediatamente con `assertive`. Ma dovresti usarlo solo se gli aggiornamenti sono davvero urgenti perché interruzioni improvvise possono potenzialmente disorientare gli utenti e influenzare l’esperienza d’uso.

Se l’aggiornamento può aspettare che l’attività corrente sia terminata, dovresti usare invece `polite`.

L’attributo `aria-live` permette alle tecnologie assistive di sapere quando il contenuto sta cambiando dinamicamente su una pagina web. Questo aiuta gli utenti con disabilità a rimanere aggiornati su annunci e aggiornamenti importanti.

Usando `aria-live` in modo appropriato, puoi assicurarti che gli utenti siano consapevoli di questi aggiornamenti in base alla loro priorità.
