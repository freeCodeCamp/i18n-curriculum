# --description--

一般的なHTTPレスポンスコードについて学びましょう。

HTTPレスポンスコードは、ウェブサーバーがクライアントに対してリクエストの結果を伝えるために使う3桁の数字コードです。

これらのコードは、リクエストが正常に処理されたか、処理中にエラーが発生したかを示します。

この講義では、よく使われるレスポンスコードのいくつかを紹介します。

## 成功コード

まずは**成功コード**から始めましょう。これらのコードは数字の2で始まります。

コードの**200 OK**は最も一般的なHTTPレスポンスコードです。これはリクエストが成功し、サーバーがクライアントの要求した情報を返したことを意味します。

コードの**201 Created**は、クライアントがリソースの作成をリクエストし、その処理が正常に完了したときにサーバーから返されます。例えば、新しいユーザーアカウントが正常に作成された場合、サーバーはこのレスポンスコードをクライアントに返します。

コードの**204 No Content**は、リクエストが正常に処理されたものの、サーバーが返すコンテンツがない場合に返されます。これは、データベース内のリソースを削除または更新する処理で、返すデータがない場合に使われます。

## **リダイレクトコード**

次に**リダイレクトコード**について説明します。

これらのコードは、クライアントがリクエストしたリソースが別の場所に移動した場合や、クライアントがリクエストを完了するために追加の操作を行う必要がある場合にサーバーから返されます。

これらのコードは数字の3で始まります。

コードの**301 Moved Permanently**は、クライアントがリクエストしたリソースが恒久的に別の場所に移動した場合に返されます。この場合、サーバーはレスポンスに新しい場所のURLを含めるべきで、クライアントがレスポンスを受け取ったらユーザーをそのURLにリダイレクトする必要があります。

コードの**302 Found**は、クライアントがリクエストしたリソースが一時的に別の場所に移動したことを示します。サーバーはレスポンスに新しいURLを提供し、クライアントはユーザーをそのURLにリダイレクトする必要があります。ただし、この変更は一時的なため、元のURLが将来的に利用可能になる場合があります。これが301と302の主な違いです。

コードの**304 Not Modified**は、クライアントに対してリクエストされたリソースが前回のリクエスト以降変更されていないことを伝えるためにサーバーから返されます。サーバーは、コンテンツが変わっていない場合に既に送信したデータを再送しないようにこのコードを送ります。これはパフォーマンス最適化の手法です。

## クライアントエラーコード

次に**エラーコード**について掘り下げます。

エラーはクライアント側とサーバー側の両方で発生する可能性があります。まずはクライアントエラーコードについて説明します。

クライアントエラーコードは、クライアント側の問題によりリクエストが完了できなかった場合に返されます。

これらのエラーコードは数字の4で始まります。

**400 Bad Request**コードは、サーバーがクライアントから送られたリクエストを理解できなかった場合に返されます。これは、クライアントが誤った構文、不正なパラメータ、または無効なデータを含むリクエストを送信した場合に発生します。

**401 Unauthorized**コードは認証に関連しています。これは、クライアントがリクエストされたリソースにアクセスするために特定の認証情報を提供する必要があることを示します。これらの認証情報にはユーザー名とパスワードが含まれる場合があります。

**403 Forbidden**コードは、クライアントがリクエストしたリソースにアクセスする権限を持っていない場合に返されます。401と403の違いは、403の場合はクライアントが有効な認証情報を持っていても、その特定のリソースに対する必要な権限がないことです。

**404 Not Found**コードは、サーバー上でリソースが見つからなかった場合に返されます。リソースが削除されたか、URLに誤字がありサーバーがリソースを見つけられない可能性があります。また、サーバーが一時的な問題を抱えている場合にもこのエラーコードが返されることがあります。

## サーバーエラーコード

次に**サーバーエラーコード**です。これらのコードは、サーバー側の問題によりリクエストが処理できなかった場合に返されます。

**500 Internal Server Error**エラーコードは、サーバーで予期しないエラーが発生したことを示します。このエラーは、設定の問題から一時的な問題まで幅広い原因で発生する可能性があります。

**502 Bad Gateway**エラーコードは、サーバーが別のサーバーと通信しようとした際に、そのサーバーから無効なレスポンスを受け取った場合に返されます。この種のエラーには多くの原因があります。

**503 Service Unavailable**エラーコードは、サーバーが一時的にリクエストを処理できないことを示し、クライアントは後で再試行する必要があります。このエラーの一般的な原因にはメンテナンス作業や、サーバーが現在処理できない大量のリクエストがあります。

HTTPレスポンスコードは、クライアントとウェブサーバー間の通信を理解するために不可欠です。これらのコードを理解することで、ウェブアプリケーションのトラブルシューティングやデバッグがしやすくなります。
