# --description--

在 Express 中，你可以使用 `app.route()` 方法創建**可串鏈的路由處理常式**。當你想要為相同的路由路徑處理多個 HTTP 方法，例如 GET、POST 或 PUT，而不想每次都重複該路徑時，這非常有用。

與其分別撰寫像 `app.get("/user")`、`app.post("/user")` 和 `app.put("/user")` 這樣的獨立路由定義，`app.route()` 讓你能將它們組合在一個有組織的程式碼區塊中。

`app.route()` 方法是為同一個端點定義多個處理常式的捷徑。這在 RESTful API 中尤其實用，因為單一資源（例如 `/user`）通常支援多種操作。

以下是一個模擬管理使用者的 RESTful API 範例：

```javascript
const express = require("express");
const app = express();
const port = 3000;

app.use(express.json()); // Parse JSON request bodies

// Use app.route() to group handlers for the same path
app
  .route("/user")
  .get((req, res) => {
    res.send("Fetching user data");
  })
  .post((req, res) => {
    const { name } = req.body;
    res.send(`Creating a user with name: ${name}`);
  })
  .put((req, res) => {
    const { name } = req.body;
    res.send(`Updating the user to: ${name}`);
  })
  .delete((req, res) => {
    res.send("Deleting the user");
  });

app.listen(port, () => {
  console.log(`Server running at http://localhost:${port}`);
});
```

在上述範例中，`app.route("/user")` 為 `/user` 路徑定義了一個單一路由。接著，我們串鏈多個 HTTP 方法 `.get()`、`.post()`、`.put()` 和 `.delete()`，用來處理該路由的不同請求類型。

以下是不同路由的說明，以及它們在簡單 RESTful 應用中的可能運作方式：

* GET `/user`：擷取並傳回使用者資料。
    
* POST `/user`：使用請求主體中的資料創建新使用者。
    
* PUT `/user`：使用請求主體中的資料更新現有使用者。
    
* DELETE `/user`：刪除使用者。
    

使用 `app.route()` 有助於在處理相同路徑的多個 HTTP 方法時保持程式碼的組織性。你不必每次都重複路由，而是將相關的處理常式集中在同一處。

這樣可以減少冗餘，讓程式碼更易讀且易於維護。當一個路由的所有方法都定義在同一區塊時，更清楚地呈現該端點的行為。

隨著應用程式成長且路由變得更複雜，`app.route()` 提供一個簡單的結構，將相關邏輯集中在一起。這在設計 RESTful API 時特別有用，因為單一資源通常支援多種操作，例如 GET、POST、PUT 和 DELETE。
