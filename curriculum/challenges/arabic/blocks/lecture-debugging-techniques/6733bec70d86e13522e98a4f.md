# --description--

تُستخدم جملة `throw` في JavaScript لرمي استثناء معرف من قبل المستخدم. الاستثناء في البرمجة هو عندما يحدث حدث غير متوقع ويعطل التدفق الطبيعي للبرنامج.

كمبرمجين، من المهم التعامل مع هذه الاستثناءات حتى لا تتعطل برامجك بشكل غير متوقع عند حدوث أخطاء. هذه هي الصيغة النحوية الأساسية لجملة `throw`:

```js
throw expression;
```

في هذه الحالة، يكون الـ `expression` هو الكائن أو القيمة التي تمثل الاستثناء الذي تريد رميه. من أمثلة ذلك أصناف الاستثناءات المدمجة مثل `Error`، `TypeError`، أو `RangeError`. إليك مثالًا على استخدام جملة `throw` لرمي `TypeError`:

```js
function validateNumber(input) {
  if (typeof input !== "number") {
    throw new TypeError("Expected a number, but received " + typeof input);
  }
  return input * 2;
}
```

في هذا المثال، نتحقق مما إذا كان نوع الـ `input` ليس من نوع `number`. إذا لم يكن كذلك، فنحن نرمي `TypeError` مع رسالة مخصصة. وإلا، فستُعيد الدالة نتيجة ضرب الـ `input` في `2`.

إذا أردت رمي رسالة خطأ أكثر عمومية، يمكنك الرجوع إلى دالة البناء `Error` بهذا الشكل:

```js
function divide(numerator, denominator) {
  if (denominator === 0) {
    throw new Error("Cannot divide by zero");
  }
  return numerator / denominator;
}
```

إليك مثالًا لدالة تتحقق مما إذا كان المقسوم عليه `0`. إذا كان الأمر كذلك، فسترمي رسالة خطأ مخصصة تقول `Cannot divide by zero`.

في الدرس التالي، سننظر في كيفية رمي رسائل الخطأ ضمن سياق كتلة `try`/`catch` التي تُستخدم للتعامل مع الاستثناءات بسلاسة في JavaScript.
