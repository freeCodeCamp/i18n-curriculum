# --interactive--

## العمل مع HTML وCSS وJavaScript

بينما يوفر HTML وCSS هيكل الموقع، يجلب JavaScript التفاعل إلى المواقع بتمكين وظائف معقدة، مثل التعامل مع مُدخَلات المستخدم، وتحريك العناصر، وحتى بناء تطبيقات ويب كاملة.

## أنواع البيانات في JavaScript

تساعد أنواع البيانات البرنامج على فهم نوع البيانات التي يتعامل معها، سواء كانت رقمًا أو نصًا أو غير ذلك.

- **Number**: الرقم يمثل كلًا من الأعداد الصحيحة والقيم العشرية. أمثلة على الأعداد الصحيحة تشمل 7، 19، و90.
- **Floating point**: الرقم العشري هو رقم يحتوي على فاصلة عشرية. أمثلة تشمل 3.14، 0.5، و0.0001.
- **String**: السلسلة النصية هي متتالية من الأحرف أو النص محاطة بعلامات اقتباس. `"I like coding"` و `'JavaScript is fun'` أمثلة على السلاسل النصية.
- **Boolean**: القيمة المنطقية تمثل إحدى قيمتين ممكنتين: `true` أو `false`. يمكنك استخدام القيمة المنطقية لتمثيل حالة، مثل `isLoggedIn = true`.
- **Undefined and Null**: القيمة `undefined` هي متغير تم إعلانه ولكن لم يُعيَّن له قيمة. القيمة `null` هي قيمة فارغة، أو متغير تم تعيينه عمدًا إلى `null`.
- **Object**: الكائن هو مجموعة من أزواج المفتاح-القيمة. المفتاح هو اسم الخاصية، والقيمة هي قيمة الخاصية.

هنا، يحتوي كائن `pet` على ثلاث خصائص أو مفاتيح: `name`، `age`، و`type`. القيم هي على التوالي `Fluffy`، `3`، و`dog`.

```js
let pet = {
  name: "Fluffy",
  age: 3,
  type: "dog"
};
```

- **Symbol**: نوع بيانات Symbol هو قيمة فريدة وثابتة يمكن استخدامها كمُعرّف لخصائص الكائنات.

في المثال أدناه، تم إنشاء رمزين لهما نفس الوصف، لكنهما غير متساويين.

:::interactive_editor

```js
const crypticKey1= Symbol("saltNpepper");
const crypticKey2= Symbol("saltNpepper");
console.log(crypticKey1 === crypticKey2); // false
```

:::

- **BigInt**: عندما يكون الرقم كبيرًا جدًا بالنسبة لنوع بيانات `Number`، يمكنك استخدام نوع بيانات BigInt لتمثيل أعداد صحيحة بطول غير محدود.

بإضافة `n` إلى نهاية الرقم، يمكنك إنشاء BigInt.

```js
const veryBigNumber = 1234567890123456789012345678901234567890n;
```

## المتغيرات في JavaScript

- يمكن إعلان المتغيرات باستخدام الكلمة المفتاحية `let`.

```js
let cityName;
```

- لتعيين قيمة إلى متغير، يمكنك استخدام معامل التعيين `=`.

```js
cityName = "New York";
```

- يمكن إعادة تعيين قيمة جديدة للمتغيرات المعلنة باستخدام `let`.

:::interactive_editor

```js
let cityName = "New York";
cityName = "Los Angeles";
console.log(cityName); // Los Angeles
```

:::

- بالإضافة إلى `let`، يمكنك أيضًا استخدام `const` لإعلان متغير. مع ذلك، لا يمكن إعادة تعيين قيمة جديدة لمتغير `const`.

```js
const cityName = "New York";
cityName = "Los Angeles"; // TypeError: Assignment to constant variable.
```

- تُستخدم المتغيرات المعلنة بـ `const` لإعلان الثوابت التي لا يُسمح بتغييرها طوال الشفرة، مثل `PI` أو `MAX_SIZE`.

## اصطلاحات تسمية المتغيرات

- يجب أن تكون أسماء المتغيرات وصفية وذات معنى.
- يجب أن تكون أسماء المتغيرات بتنسيق سنام الجمل مثل `cityName`، `isLoggedIn`، و`veryBigNumber`.
- يجب ألا تبدأ أسماء المتغيرات برقم. يجب أن تبدأ بحرف أو `_` أو `$`.
- يجب ألا تحتوي أسماء المتغيرات على فراغات أو أحرف خاصة، باستثناء `_` و`$`.
- يجب ألا تكون أسماء المتغيرات كلمات مفتاحية محجوزة.
- أسماء المتغيرات حساسة لحالة الأحرف. `age` و`Age` متغيران مختلفان.

## السلاسل النصية وثبات السلسلة في JavaScript

- السلاسل النصية هي متتاليات من الأحرف محاطة بعلامات اقتباس. يمكن إنشاؤها باستخدام علامات اقتباس مفردة أو مزدوجة.

```js
let correctWay = 'This is a string';
let alsoCorrect = "This is also a string";
```

- السلاسل النصية ثابتة في JavaScript. هذا يعني أنه بمجرد إنشاء سلسلة نصية، لا يمكنك تغيير الأحرف داخلها. مع ذلك، يمكنك إعادة تعيين السلاسل النصية إلى قيمة جديدة.

```js
let firstName = "John";
firstName = "Jane"; // Reassigning the string to a new value
```

## دمج السلاسل النصية في JavaScript

- الدمج هو عملية ربط عدة سلاسل نصية أو دمج سلاسل نصية مع متغيرات تحتوي على نص. معامل `+` هو أحد أبسط وأكثر الطرق استخدامًا لدمج السلاسل النصية.

:::interactive_editor

```js
let studentName = "Asad";
let studentAge = 25;
let studentInfo = studentName + " is " + studentAge + " years old.";
console.log(studentInfo); // Asad is 25 years old.
```

:::

- إذا كنت بحاجة إلى إضافة أو إلحاق نص إلى سلسلة نصية موجودة، يمكنك استخدام معامل `+=`. هذا مفيد عندما تريد بناء سلسلة نصية بإضافة المزيد من النصوص إليها مع الوقت.

:::interactive_editor

```js
let message = "Welcome to programming, ";
message += "Asad!";
console.log(message); // Welcome to programming, Asad!
```

:::

- طريقة أخرى لدمج السلاسل النصية هي استخدام الدالة `concat()`. هذه الدالة تربط سلسلتين نصيتين أو أكثر معًا.

:::interactive_editor

```js
let firstName = "John";
let lastName = "Doe";
let fullName = firstName.concat(" ", lastName);
console.log(fullName); // John Doe
```

:::

## تسجيل الرسائل باستخدام `console.log()`

- تُستخدم الدالة `console.log()` لتسجيل الرسائل في وحدة التحكم. إنها أداة مفيدة لتصحيح الأخطاء واختبار الشفرة.

:::interactive_editor

```js
console.log("Hello, World!");
// Output: Hello, World!
```

:::

## الفواصل المنقوطة في JavaScript

- تُستخدم الفواصل المنقوطة بشكل أساسي لتحديد نهاية العبارة. هذا يساعد محرك JavaScript على فهم فصل التعليمات الفردية، وهو أمر ضروري للتنفيذ الصحيح.

```js
let message = "Hello, World!"; // first statement ends here
let number = 42; // second statement starts here
```

- تساعد الفواصل المنقوطة على منع الالتباسات في تنفيذ الشفرة وتضمن إنهاء العبارات بشكل صحيح.

## التعليقات في JavaScript

- أي سطر من الشفرة يتم التعليق عليه يتجاهله محرك JavaScript. تُستخدم التعليقات لشرح الشفرة، تدوين الملاحظات، أو تعطيل الشفرة مؤقتًا.
- تُنشأ التعليقات ذات السطر الواحد باستخدام `//`.

```js
// This is a single-line comment and will be ignored by the JavaScript engine
```

- تُنشأ التعليقات متعددة الأسطر باستخدام `/*` لبدء التعليق و`*/` لإنهائه.

```js
/*
This is a multi-line comment.
It can span multiple lines.
*/
```

## JavaScript كلغة ذات نوع ديناميكي

- JavaScript هي لغة ذات نوع ديناميكي، مما يعني أنك لا تحتاج إلى تحديد نوع البيانات للمتغير عند إعلانه. يحدد محرك JavaScript نوع البيانات تلقائيًا بناءً على القيمة المعينة للمتغير.

```js
let error = 404; // JavaScript treats error as a number
error = "Not Found"; // JavaScript now treats error as a string
```

- لغات أخرى، مثل C#، التي ليست ذات نوع ديناميكي ستؤدي إلى ظهور خطأ:

```csharp
int error = 404; // value must always be an integer
error = "Not Found"; // This would cause an error in C#
```

## استخدام معامل `typeof`

- يُستخدم معامل `typeof` للتحقق من نوع بيانات المتغير. يعيد سلسلة نصية تشير إلى نوع المتغير.

:::interactive_editor

```js
let age = 25;
console.log(typeof age); // "number"

let isLoggedIn = true;
console.log(typeof isLoggedIn); // "boolean"
```

:::

- مع ذلك، هناك سلوك معروف في JavaScript يتعلق بـ `null`. يعيد معامل `typeof` القيمة `"object"` لقيم `null`.

:::interactive_editor

```js
let user = null;
console.log(typeof user); // "object"
```

:::
