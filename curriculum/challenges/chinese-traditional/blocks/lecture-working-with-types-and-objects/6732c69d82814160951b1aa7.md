# --interactive--

`Number()` 建構式用於創建數字物件。數字物件包含一些有用的屬性和方法，例如 `isNaN` 和 `toFixed` 方法。以下是使用 `new` 關鍵字搭配 `Number()` 建構式的範例：

:::interactive_editor

```js
const myNum = new Number("34");
console.log(typeof myNum); // "object" 
```

:::

在此範例中，我們將字串字面值傳入 `Number()` 建構式，且返回類型為 `object` 型別，而非 `string`。 

當 `Number()` 建構式在沒有 `new` 關鍵字的情況下被當作函式呼叫時，返回值將會是原始數字型別。大多數時候，你會使用 `Number()` 建構式將其他資料型別轉換為數字資料型別。以下是一個將字串轉換為數字的範例：

:::interactive_editor

```js
const myNum = Number("100");
console.log(myNum); // 100

console.log(typeof myNum); // number
```

:::

當你從使用者取得輸入並且需要將該字串輸入轉換為數字以便進行數學計算時，這會很有幫助。 

如果你嘗試透過空字串呼叫 `Number()` 建構式，結果將會是數字 `0`： 

:::interactive_editor

```js
const num = Number("");
console.log(num); // 0
```

:::

這是因為 JavaScript 會嘗試解析該字串，且由於它不包含任何數字，結果將會是零。

如果你嘗試傳入包含隨機字元的字串，結果將會是 `NaN` 或「不是數字」。 

:::interactive_editor

```js
const num = Number("random");
console.log(num); // NaN
```

:::

當使用布林值時，`true` 傳回 `1`，因為 `true` 會被轉換成一，而 `false` 傳回 `0`，因為 `false` 會被轉換成零。 

:::interactive_editor

```js
const boolTrue = Number(true);
const boolFalse = Number(false);

console.log(boolTrue); // 1
console.log(boolFalse); // 0
```

:::

如果你傳入 `null`，結果將會是 `0`，如果你傳入 `undefined`，結果將會是 `NaN`。 

:::interactive_editor

```js
const undefinedNum = Number(undefined);
const nullNum = Number(null);

console.log(undefinedNum); // NaN
console.log(nullNum); // 0
```

:::

在處理陣列時，有幾件事需要考慮。

空的陣列會傳回 `0`。只有一個數字的陣列會傳回該數字。多個數字的陣列會傳回 `NaN`。而包含字串的陣列也會傳回 `NaN`。

:::interactive_editor

```js
const emptyArr = Number([]);
const arrOneNum = Number([7]);
const arrMultiNum = Number([7, 36, 12]);
const arrStr = Number(["str1"]);
const arrMultiStr = Number(["str1", "str2"]);

console.log(emptyArr); // 0
console.log(arrOneNum); // 7
console.log(arrMultiNum); // NaN
console.log(arrStr); // NaN
console.log(arrMultiStr); // NaN
```

:::

當處理物件時，結果總是 `NaN`。

:::interactive_editor

```js
const obj1 = Number({});
const obj2 = Number({2: 2});
const obj3 = Number({key: "val"});
const obj4 = Number({key: true});

console.log(obj1); // NaN
console.log(obj2); // NaN
console.log(obj3); // NaN
console.log(obj4); // NaN
```

:::

總結來說，你大多會使用 `Number()` 建構式來進行型別轉換，而非建立數字或數字物件。
