# --explanation--

`Build tools` — це програмні продукти, які допомагають автоматизувати та керувати повторюваними завданнями, такими як компіляція, тестування та пакування програмних проєктів. Наприклад:

`Maven is a popular build tool for Java projects, helping automate the compilation process.` — це означає, що Maven спрощує процес створення Java-застосунків, автоматизуючи завдання, як-от компіляція коду та створення пакетів.

`Built-in` позначає функції, які вже є частиною програмного забезпечення і не потребують додаткової інсталяції чи налаштування. Наприклад:

`Visual Studio has built-in tools for debugging code.` — це означає, що інструменти налагодження вже включені у Visual Studio, і користувачам не потрібно встановлювати додаткові інструменти, щоб почати налагодження.

`Version control` — це система, яка відстежує та керує змінами у коді, дозволяючи командам працювати над одним проєктом без втрати роботи чи виникнення конфліктів. Наприклад:

`Git is a widely-used version control system for tracking code changes and collaborating with teammates.` — це означає, що Git допомагає розробникам керувати різними версіями коду та працювати разом над одним проєктом.
