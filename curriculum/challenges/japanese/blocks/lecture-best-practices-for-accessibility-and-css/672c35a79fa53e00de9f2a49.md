# --interactive--

ウェブページでコンテンツを非表示にすることはウェブ開発でよくある手法ですが、アクセシビリティを損なわない方法で行うことが重要です。異なる非表示の手法は、支援技術がコンテンツをどのように解釈しユーザーに提示するかにさまざまな影響を与えます。アクセシビリティを維持しながらコンテンツを非表示にするベストプラクティスを見ていきましょう。コンテンツを非表示にする一般的な方法の一つに`display: none`があります。例を示します。

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<p class="hidden">Hidden text</p>
<p>Visible text</p>
```

```css
.hidden {
  display: none;
}
```

:::

これは視覚的にはコンテンツを非表示にしますが、アクセシビリティツリーからも削除します。アクセシビリティツリーはスクリーンリーダーなどの支援技術がウェブページ上のコンテンツを解釈し操作するための構造です。支援技術が理解しユーザーに提示できるように、コンテンツとその意味論的な意味を表しています。

`display: none`を使うと、スクリーンリーダーなどの支援技術はこのコンテンツにアクセスできなくなります。つまりアクセシビリティツリーに含まれないためです。したがって、この方法は視覚的な表示とアクセシビリティの両方から完全にコンテンツを除外したい場合にのみ使うべきです。

別の非表示の方法として`visibility: hidden`があります。

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<p class="hidden">Hidden text</p>
<p>Visible text</p>
```

```css
.hidden {
  visibility: hidden;
}
```

:::

`visibility: hidden`は視覚的にコンテンツを非表示にしますが、文書の流れには残り、ページ上のスペースは占有し続けます。`display: none`と同様に、`visibility: hidden`もアクセシビリティツリーからコンテンツを除外します。つまりスクリーンリーダーなどの支援技術は非表示のコンテンツにアクセスできません。`visibility: hidden`は支援技術を使う人も含めて全員からコンテンツを隠したい場合にのみ使ってください。

視覚的には非表示にしつつスクリーンリーダーにはアクセス可能にしたい場合は、「visually hidden」や「screen reader only」と呼ばれる手法を使えます。以下は`.sr-only`というCSSクラスを使った例で、視覚的に非表示にしつつスクリーンリーダーにはアクセス可能にする一般的な手法です。

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<p class="sr-only">Hidden text</p>
<p>Visible text</p>
```

```css
.sr-only {
  position: absolute;
  width: 1px;
  height: 1px;
  padding: 0;
  margin: -1px;
  overflow: hidden;
  clip: rect(0, 0, 0, 0);
  white-space: nowrap;
  border: 0;
}
```

:::

この例では`position`、`clip`、`white-space`のようなプロパティを使っています。これらは今後のレッスンで学びますが、今はこのCSSルールが視覚的にコンテンツを非表示にしつつスクリーンリーダーにはアクセス可能にすることを理解してください。視覚的なレイアウトに影響を与えずにスクリーンリーダーユーザーに追加の文脈を提供するのに役立ちます。

コンテンツの可視性を切り替える場合は、`hidden`属性の使用を検討してください。

:::interactive_editor

```html
<p hidden>This content is hidden</p>
<p>This content is visible</p>
```

:::

`hidden`属性はほとんどの最新ブラウザでサポートされており、視覚的にもアクセシビリティツリーからもコンテンツを非表示にします。JavaScriptで簡単に切り替えられます。最後に、重要なコンテンツを隠す際は注意してください。ウェブサイトの理解や利用に不可欠な情報は、すべてのユーザーに見えるようにしアクセス可能にする必要があります。コンテンツを隠すのは、本当にユーザー体験を向上させる場合に限ってください。これらのベストプラクティスに従うことで、どのようにウェブサイトとやり取りしても、すべてのユーザーにコンテンツがアクセス可能な状態を保てます。
