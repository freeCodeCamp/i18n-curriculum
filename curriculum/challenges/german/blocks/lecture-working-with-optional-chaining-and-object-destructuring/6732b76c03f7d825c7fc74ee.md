# --interactive--

Der optionale Verkettungsoperator (`?.`) ist ein nützliches Werkzeug in JavaScript, mit dem Sie sicher auf Objekteigenschaften zugreifen oder Methoden aufrufen können, ohne sich Sorgen machen zu müssen, ob sie existieren. Er ist wie ein Sicherheitsnetz beim Arbeiten mit Objekten, die fehlende Teile haben könnten.

:::interactive_editor

```js
const person = {
  name: "Alice",
  age: 30
};

console.log(person.name); // "Alice"
console.log(person.job); // undefined
```

:::

In diesem Beispiel existiert `person.name`, daher wird `Alice` ausgegeben. Aber `person.job` existiert nicht, deshalb erhalten wir `undefined`.

Nun nehmen wir an, wir möchten auf eine Eigenschaft eines Objekts zugreifen, das möglicherweise nicht existiert:

```js
const person = {
  name: "Alice",
  age: 30
};

console.log(person.address.street); // This will throw an error!
```

Dieses Beispiel wirft einen `Uncaught TypeError`. Da `person.address` `undefined` ist, können wir nicht auf die `street`-Eigenschaft zugreifen. Hier kommt der optionale Verkettungsoperator ins Spiel. Hier ist ein Beispiel für die Verwendung des optionalen Verkettungsoperators:

:::interactive_editor

```ts
const user = {
  name: "John",
  profile: {
    email: "john@example.com",
    address: {
      street: "123 Main St",
      city: "Somewhere"
    }
  }
};

console.log(user?.profile?.address?.street); // "123 Main St"
console.log(user?.profile?.phone?.number);   // undefined
```

:::

Durch die Verwendung des optionalen Verkettungsoperators sagen wir JavaScript, dass es mit der Operation nur fortfahren soll, wenn das Objekt (oder der Wert vor dem `?.`) existiert und nicht `null` oder `undefined` ist.

Wenn der Wert vor dem `?.` `null` oder `undefined` ist, gibt JavaScript `undefined` zurück, anstatt zu versuchen, die Operation fortzusetzen und einen Fehler zu werfen.

Denken Sie daran, dass der optionale Verkettungsoperator am nützlichsten ist, wenn Sie sich nicht sicher sind, ob eine Eigenschaft oder Methode existiert. Er hilft, Fehler zu vermeiden und macht Ihren Code robuster.
