## Responsives Webdesign

- **Definition**: Das Kernprinzip des responsiven Designs ist Anpassungsfähigkeit – die Fähigkeit einer Website, ihr Layout und ihren Inhalt basierend auf der Bildschirmgröße und den Fähigkeiten des Geräts, auf dem sie angezeigt wird, anzupassen.
- **Flüssige Raster**: Diese verwenden relative Einheiten wie Prozentsätze statt feste Einheiten wie Pixel, wodurch sich der Inhalt basierend auf der Bildschirmgröße anpassen und umfließen kann.
- **Flexible Bilder**: Diese sind so eingestellt, dass sie sich innerhalb ihrer Container-Elemente anpassen, um sicherzustellen, dass sie auf kleineren Bildschirmen nicht aus ihren Containern herausragen.

## Media Queries

- **Definition**: Damit können Entwickler unterschiedliche Stile basierend auf den Eigenschaften des Geräts anwenden, hauptsächlich der Breite des Viewports.

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<p class="mq-example">The background of this paragraph will change when the viewport is at least 768px wide.</p>
```

```css
.mq-example {
  padding: 10px;
  border: 1px solid #ccc;
  background-color: lightyellow;
}

@media screen and (min-width: 768px) {
  /* Styles for screens at least 768px wide */
  .mq-example {
    background-color: lightblue;
  }
}
```

:::

- **`all` Medientyp**: Dieser ist für alle Geräte geeignet. Dies ist der Standard, wenn kein Medientyp angegeben ist.
- **`print` Medientypen**: Dieser ist für paginierte Materialien und Dokumente gedacht, die im Druckvorschau-Modus auf einem Bildschirm angezeigt werden.
- **`screen` Medientypen**: Dieser ist hauptsächlich für Bildschirme vorgesehen.
- **`aspect-ratio`**: Dies beschreibt das Verhältnis zwischen der Breite und Höhe des Viewports.

```css
@media screen and (aspect-ratio: 16/9) {
  /* Styles for screens with a 16:9 aspect ratio */
}
```

- **`orientation`**: Dies wird verwendet, um anzugeben, ob sich das Gerät im Quer- oder Hochformat befindet.

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<p class="orientation-example">This paragraph's background will change in landscape orientation.</p>
```

```css
.orientation-example {
  padding: 10px;
  border: 1px solid #ccc;
  background-color: lightyellow;
}

@media screen and (orientation: landscape) {
  /* Styles for landscape orientation */
  .orientation-example {
    background-color: lightgreen;
  }
}
```

:::

- **`resolution`**: Dies wird verwendet, um die Auflösung des Ausgabegeräts in Punkten pro Zoll (dpi) oder Punkten pro Zentimeter (dpcm) zu beschreiben.

```css
@media screen and (min-resolution: 300dpi) {
  /* Styles for high-resolution screens */
}
```

- **`hover`**: Dies wird verwendet, um zu testen, ob der primäre Eingabemechanismus über Elemente hovern kann.

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<button>Hover over me</button>
```

```css
button {
  padding: 10px 20px;
  font-size: 1rem;
  background-color: lightgray;
  border: 1px solid #999;
  cursor: pointer;
}

@media (hover: hover) {
  /* Styles for devices that support hover */
  button:hover {
    background-color: yellow;
  }
}
```

:::

- **`prefers-color-scheme`**: Dies wird verwendet, um zu erkennen, ob der Nutzer ein helles oder dunkles Farbschema angefordert hat.
- **Media Queries und logische Operatoren**: Der `and`-Operator wird verwendet, um mehrere Media Features zu kombinieren, während `not` und `only` verwendet werden können, um Media Queries zu negieren oder zu isolieren.

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<p class="logical-example">This paragraph's background will change when the screen is at least 768px wide AND in landscape orientation.</p>
```

```css
.logical-example {
  padding: 10px;
  border: 1px solid #ccc;
  background-color: lightyellow;
}

@media screen and (min-width: 768px) and (orientation: landscape) {
  /* Styles for landscape screens at least 768px wide */
  .logical-example {
    background-color: lightcoral;
  }
}
```

:::

## Übliche Media Breakpoints

- **Definition**: Media Breakpoints sind spezifische Punkte im Design einer Website, an denen sich Layout und Inhalt an verschiedene Bildschirmgrößen anpassen. Es gibt einige allgemeine Breakpoints, mit denen Sie Telefone, Tablets und Desktop-Bildschirme ansprechen können. Es ist jedoch nicht ratsam, jede einzelne mögliche Bildschirmgröße für verschiedene Geräte zu verfolgen.

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<p class="breakpoint-example">This text will change size when the viewport width is at least 768px.</p>
```

```css
.breakpoint-example {
  font-size: 1rem;
  padding: 10px;
  border: 1px solid #ccc;
}

/* Styles for screens wider than 768px */
@media screen and (min-width: 768px) {
  .breakpoint-example {
    font-size: 1.125rem;
    background-color: lightblue;
  }
}
```

:::

- **Kleine Geräte (Smartphones)**: bis zu 640px
- **Mittlere Geräte (Tablets)**: 641px bis 1024px
- **Große Geräte (Desktops)**: 1025px und größer

## Mobile-first-Ansatz

- **Definition**: Der Mobile-first-Ansatz ist eine Designphilosophie und Entwicklungsstrategie im responsiven Webdesign, die es priorisiert, Websites zuerst für mobile Geräte zu erstellen, bevor sie für größere Bildschirme gestaltet werden.

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<div class="container">
  <p>The width of this container changes based on the viewport size.</p>
</div>
```

```css
/* Base styles for mobile */
.container {
  width: 100%;
  padding: 10px;
  background-color: lightblue;
}

/* Styles for larger screens */
@media screen and (min-width: 768px) {
  .container {
    width: 750px;
    margin: 0 auto;
    padding: 20px;
  }
}

@media screen and (min-width: 1024px) {
  .container {
    width: 960px;
  }
}
```

:::
