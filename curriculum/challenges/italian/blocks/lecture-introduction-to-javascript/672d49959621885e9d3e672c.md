# --interactive--

Quando lavori con JavaScript, dichiari spesso variabili per archiviare dati che prevedi di usare in tutto il programma.

Nel JavaScript moderno, `let` e `const` sono i modi preferiti per dichiarare variabili, ma differiscono nel modo in cui gestiscono l’assegnazione e la riassegnazione dei valori.

In questa lezione, esploreremo come `let` e `const` differiscono nella dichiarazione, assegnazione e riassegnazione delle variabili.

La parola chiave `let` ti permette di dichiarare variabili che possono essere aggiornate o riassegnate in seguito. Puoi pensare a `let` come a un contenitore flessibile: una volta che hai archiviato un valore al suo interno, puoi cambiarlo secondo necessità durante l’esecuzione del programma.

Ecco un esempio di dichiarazione e assegnazione di una variabile con `let`:

```js
let score = 10;
```

In questo caso, la variabile `score` è dichiarata e le viene assegnato il valore `10`. Se vuoi aggiornare il valore in seguito, puoi farlo facilmente:

:::interactive_editor

```js
let score = 10;
console.log(score); // 10
score = 20;
console.log(score); // 20
```

:::

Ora `score` contiene il valore `20`. Questo rende `let` particolarmente utile quando sai che il valore di una variabile cambierà durante l’esecuzione del programma.

D’altra parte, `const` si usa per dichiarare variabili costanti. Una volta assegnato un valore a una variabile dichiarata con `const`, non puoi riassegnarlo.

Questo rende `const` ideale per valori che non vuoi cambiare accidentalmente durante l’esecuzione del programma.

Ecco un esempio di dichiarazione e assegnazione di una variabile con `const`:

:::interactive_editor

```js
const maxScore = 100;
console.log(maxScore); // 100
```

:::

Una volta che `maxScore` ha il valore `100`, non può essere cambiato:

```js
maxScore = 200; // This will result in an error
```

Provare a riassegnare un valore a una variabile `const` lancerà un errore nella console JavaScript, perché le variabili `const` sono immutabili una volta assegnate.

Puoi dichiarare una variabile `let` senza assegnarle subito un valore, e assegnarglielo in seguito:

:::interactive_editor

```js
let age;
console.log(age); // undefined
age = 25;
console.log(age); // 25
```

:::

Le variabili dichiarate con `const` devono avere un valore assegnato al momento della dichiarazione. Se provi a dichiarare una variabile `const` senza assegnarle un valore, otterrai un errore:

```js
const age; // Error: Missing initializer in const declaration
```

Dovresti usare `let` quando ti serve dichiarare variabili che saranno riassegnate in seguito. Per esempio, per tenere traccia di un punteggio che cambia o aggiornare un valore nel tempo nel tuo programma.

Usa `const` quando vuoi dichiarare variabili che devono rimanere costanti, come valori di configurazione o impostazioni che non devono essere modificate accidentalmente.

Puoi anche usare la parola chiave `var`, ma non è più molto consigliata. `var` è un po’ come `let`, solo che ha un ambito più ampio, il che può causare più problemi nel programma.
