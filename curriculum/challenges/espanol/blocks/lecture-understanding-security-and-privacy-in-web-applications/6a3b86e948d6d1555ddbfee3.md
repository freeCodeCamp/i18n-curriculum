# --description--

La política de mismo origen es una medida de seguridad integrada en tus navegadores web que impide que un sitio web lea datos privados de otro sitio web. Esta regla protege tus cuentas personales de los hackers.

Un origen tiene tres partes específicas: el protocolo web, el nombre de dominio y el número de puerto.

Por ejemplo, en la dirección `https://example.com:443`, el protocolo es HTTPS, el nombre de dominio es `example.com` y el número de puerto es `443`.

Los navegadores suelen ocultar el número de puerto si es el estándar. HTTPS usa por defecto el puerto 443 y HTTP el puerto 80. Aunque esté oculto, el navegador aún lo verifica. Dos sitios web tienen el mismo origen solo cuando las tres partes coinciden exactamente.

Bajo esta regla estricta, un navegador bloquea que un sitio web espíe a otro diferente. Imagina que abres tu cuenta bancaria en línea en una pestaña y en otra pestaña abres un sitio malicioso. La política de mismo origen bloquea que el sitio malicioso robe tu información bancaria.

Sin embargo, los sitios web modernos a menudo necesitan compartir datos entre diferentes orígenes. Por ejemplo, una página frontend puede necesitar obtener datos de un servidor backend separado. Las reglas de mismo origen en los navegadores bloquean estas solicitudes seguras, por lo que los desarrolladores necesitaban una forma de compartir datos de manera segura. Aquí es donde entra CORS.

CORS es un sistema que permite a los servidores abrir acceso a través de las reglas del navegador. Usa encabezados especiales para otorgar acceso a datos a sitios confiables.

Esto es lo que sucede cuando tu navegador solicita datos a otro sitio:

* Incluye un encabezado `Origin` que nombra el sitio web que hace la solicitud
* El servidor receptor verifica este nombre
* Si el servidor confía en el sitio, responde con un encabezado especial de permiso que indica al navegador que el sitio puede leer los datos entrantes.

Si el servidor no envía este permiso, el navegador bloquea los datos al instante. Entonces, el sitio frontend recibe un mensaje de error en lugar de los archivos privados.

En aplicaciones Node.js que usan Express, CORS se implementa con el paquete `cors`:

```javascript
const express = require('express');
const cors = require('cors');
const app = express();

// this enables CORS for all origins and all routes for development
app.use(cors());
```

En producción, puede que tengas que pasar un objeto de configuración para restringir el acceso a un dominio específico:

```javascript
const corsOptions = {
  origin: ['https://yourapp.com', 'https://admin.yourapp.com'],
  credentials: true
};

app.use(cors(corsOptions));
```

Si usas Node.js puro, tendrías que manejarlo manualmente configurando los encabezados de respuesta CORS tú mismo en cada solicitud:

```javascript
res.setHeader('Access-Control-Allow-Origin', 'https://example.com');
res.setHeader('Access-Control-Allow-Methods', 'GET, POST, PUT, DELETE, OPTIONS');
res.setHeader('Access-Control-Allow-Headers', 'Content-Type, Authorization');
```
