# --description--

У Express **маршрути** — це шаблони URL, які визначають, як сервер відповідає на вхідні запити. Коли користувач запитує певний URL, Express порівнює цей URL із маршрутом і виконує відповідний обробник.

Маршрути визначаються всередині методів маршрутизації, таких як `app.get()`, `app.post()` тощо. Ці маршрути можуть бути статичними, наприклад `/home`, або динамічними, де параметри використовуються для отримання значень із URL, як `/user/:id`.

У цьому уроці ми зосередимося на статичних маршрутах. Ось приклад одного з них:

```javascript
// Static route
app.get("/home", (req, res) => {
  res.send("Welcome to the homepage!");
});
```

У цьому прикладі сервер слухає GET-запити на `/home` і відповідає простим повідомленням.

Метод `app.get()` приймає два аргументи:

* Перший аргумент, `"/home"`, — це статичний маршрут (або URL), на який сервер слухає.
    
* Другий аргумент — це функція зворотного виклику, яка виконується, коли надходить GET-запит на цей маршрут.
    

Всередині функції зворотного виклику `res.send()` надсилає відповідь клієнту. У цьому випадку він надсилає текст `Welcome to the homepage!`, який відображається у браузері.

Ось ще один приклад статичного маршруту:

```javascript
// Static route
app.get("/menu/drinks", (req, res) => {
  res.send("Welcome to the drinks menu!");
});
```

У цьому випадку сервер слухає GET-запити на URL `/menu/drinks` і надсилає повідомлення у відповідь, коли цей маршрут запитується.

Ви можете використовувати такі маршрути, коли ваш сайт має кілька пов’язаних сторінок, згрупованих під спільним шляхом. Наприклад, сайт ресторану може організувати свої сторінки меню так:

* `/menu/food`
    
* `/menu/drinks`
    
* `/menu/desserts`
    

Такий підхід допомагає зберігати ваші URL зрозумілими та змістовними, а також полегшує організацію та підтримку маршрутів у міру зростання застосунку.

У наступному уроці ми розглянемо динамічні маршрути з параметрами.
