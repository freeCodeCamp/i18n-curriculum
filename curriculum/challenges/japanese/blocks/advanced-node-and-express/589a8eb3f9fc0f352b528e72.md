# --description--

戦略の最後の部分は、GitHubから返されるプロフィールの処理です。ユーザーのデータベースオブジェクトが存在すればそれを読み込み、存在しなければ新しく作成し、プロフィールからフィールドを埋めてからユーザーのオブジェクトを返す必要があります。GitHubは各プロフィール内に一意の*id*を提供しており、これを使ってユーザーをシリアライズするために検索できます（これはすでに実装済みです）。以下はプロジェクトで使える実装例で、新しい戦略の2番目の引数である関数内、現在`console.log(profile);`があるすぐ下に配置します。

```js
myDataBase.findOneAndUpdate(
  { id: profile.id },
  {
    $setOnInsert: {
      id: profile.id,
      username: profile.username,
      name: profile.displayName || 'John Doe',
      photo: profile.photos[0].value || '',
      email: Array.isArray(profile.emails)
        ? profile.emails[0].value
        : 'No public email',
      created_on: new Date(),
      provider: profile.provider || ''
    },
    $set: {
      last_login: new Date()
    },
    $inc: {
      login_count: 1
    }
  },
  { upsert: true, new: true },
  (err, doc) => {
    return cb(null, doc.value);
  }
);
```

`findOneAndUpdate`はオブジェクトを検索して更新できます。オブジェクトが存在しなければ挿入され、コールバック関数で利用可能になります。この例では、常に`last_login`を設定し、`login_count`を`1`だけ増やし、新しいオブジェクト（新規ユーザー）が挿入された場合にのみ大部分のフィールドを埋めています。デフォルト値の使い方に注目してください。返されるプロフィールにすべての情報が入っていなかったり、ユーザーが非公開にしている場合があります。その場合はエラーを防ぐために適切に処理します。
