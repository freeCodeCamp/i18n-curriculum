# --description--

Diese Reihe von Aufgaben führt Sie in die Verwendung von Redux mit React ein. Zuerst folgt eine Überprüfung einiger der wichtigsten Grundsätze jeder Technologie. React ist eine View-Bibliothek, der Sie Daten bereitstellen, und die dann die Ansicht effizient und vorhersehbar rendert. Redux ist ein Framework zur Zustandsverwaltung, das Sie verwenden können, um die Verwaltung des Zustands Ihrer Anwendung zu vereinfachen. Typischerweise erstellen Sie in einer React-Redux-Anwendung einen einzigen Redux-Store, der den Zustand der gesamten Anwendung verwaltet. Ihre React-Komponenten abonnieren nur die Datenstücke im Store, die für ihre Rolle relevant sind. Dann dispatchen Sie Aktionen direkt aus React-Komponenten, die anschließend Store-Updates auslösen.

Obwohl React-Komponenten ihren eigenen Zustand lokal verwalten können, ist es bei komplexen Anwendungen in der Regel besser, den Anwendungszustand an einem einzigen Ort mit Redux zu speichern. Es gibt Ausnahmen, wenn einzelne Komponenten einen nur für sie spezifischen lokalen Zustand haben. Schließlich ist Redux nicht dafür ausgelegt, direkt mit React zu arbeiten, daher müssen Sie das `react-redux`-Paket verwenden. Es bietet eine Möglichkeit, Redux-`state` und `dispatch` als `props` an Ihre React-Komponenten weiterzugeben.

In den nächsten Aufgaben erstellen Sie zunächst eine einfache React-Komponente, die es Ihnen erlaubt, neue Textnachrichten einzugeben. Diese werden zu einem Array hinzugefügt, das in der Ansicht angezeigt wird. Dies sollte eine gute Wiederholung dessen sein, was Sie in den React-Lektionen gelernt haben. Anschließend erstellen Sie einen Redux-Store und Aktionen, die den Zustand des Nachrichten-Arrays verwalten. Schließlich verwenden Sie `react-redux`, um den Redux-Store mit Ihrer Komponente zu verbinden und so den lokalen Zustand in den Redux-Store auszulagern.

# --instructions--

Beginnen Sie mit einer `DisplayMessages`-Komponente. Fügen Sie dieser Komponente einen Konstruktor hinzu und initialisieren Sie ihn mit einem Zustand, der zwei Eigenschaften hat: `input`, das auf einen leeren String gesetzt ist, und `messages`, das auf ein leeres Array gesetzt ist.
