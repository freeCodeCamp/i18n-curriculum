## Nozioni di base sul lavoro con le classi

- **Definizione**: Le classi in JavaScript servono a definire modelli per creare oggetti e incapsulare dati. Le classi includono un costruttore, un metodo speciale che viene chiamato automaticamente quando si crea un nuovo oggetto dalla classe. Serve a inizializzare le proprietà dell'oggetto. La parola chiave `this` viene usata qui per riferirsi all'istanza corrente della classe. Sotto il costruttore, puoi avere quelli che si chiamano metodi. I metodi sono funzioni definite all'interno di una classe che eseguono azioni o operazioni sui dati o sullo stato della classe. Servono a definire comportamenti che le istanze della classe possono eseguire.

:::interactive_editor

```js
class Dog {
  constructor(name) {
    this.name = name;
  }

  bark() {
    console.log(`${this.name} says woof!`);
  }
}

const dog = new Dog("Gino");
console.log(dog.name);  // Gino
```

:::

Per creare una nuova istanza della classe, userai la parola chiave `new` seguita dal nome della classe:

```js
const dog = new Dog("Gino");
```

Puoi anche creare classi come espressioni di classe. Qui la classe è anonima e assegnata a una variabile.

:::interactive_editor

```js
const Dog = class {
  constructor(name) {
    this.name = name;
  }

  bark() {
    console.log(`${this.name} says woof!`);
  }
};

const dog = new Dog("Gino");
console.log(dog.name);  // Gino
dog.bark();  // Gino says woof!
```

:::

## Ereditarietà delle classi

- **Definizione**: In programmazione, l'ereditarietà permette di definire classi che ereditano proprietà e metodi da classi genitrici. Questo favorisce il riuso del codice e stabilisce una relazione gerarchica tra le classi. Una classe genitore è una classe che funge da modello per altre classi. Definisce proprietà e metodi che vengono ereditati da altre classi. Una classe figlia è una classe che eredita le proprietà e i metodi di un'altra classe. Le classi figlie possono anche estendere la funzionalità delle loro classi genitrici aggiungendo nuove proprietà e metodi. In JavaScript, usiamo la parola chiave `extends` per implementare l'ereditarietà. Questa parola chiave indica che una classe è figlia di un'altra classe.

:::interactive_editor

```js
class Vehicle {
  constructor(brand, year) {
    this.brand = brand;
    this.year = year;
  }
}

class Car extends Vehicle {
  honk() {
    console.log("Honk! Honk!");
  }
}

const myCar = new Car("freeCodeCamp Motors", 2019);
console.log(myCar.brand);  // freeCodeCamp Motors
console.log(myCar.year);  // 2019
myCar.honk();  // Honk! Honk!
```

:::

La parola chiave `super` viene usata per consultare i metodi, i costruttori e i campi della classe genitore.

:::interactive_editor

```js
class Vehicle {
  constructor(brand, year) {
    this.brand = brand;
    this.year = year;
  }
}

class Car extends Vehicle {
  constructor(brand, year, numDoors) {
    super(brand, year);
    this.numDoors = numDoors;
  }
}

const myCar = new Car("freeCodeCamp Motors", 2019, 4);
console.log(myCar.brand);  // freeCodeCamp Motors
console.log(myCar.year);  // 2019
console.log(myCar.numDoors);  // 4
```

:::

## Lavorare con metodi statici e proprietà statiche

- **Metodi statici**: Questi metodi sono spesso usati per funzioni di utilità che non hanno bisogno di consultare lo stato specifico di un oggetto. Sono definiti all'interno delle classi per incapsulare funzionalità correlate.

:::interactive_editor

```js
class Movie {
  constructor(title, rating) {
    this.title = title;
    this.rating = rating;
  }

  static compareMovies(movieA, movieB) {
    if (movieA.rating > movieB.rating) {
      console.log(`${movieA.title} has a higher rating.`);
    } else if (movieA.rating < movieB.rating) {
      console.log(`${movieB.title} has a higher rating.`);
    } else {
      console.log("These movies have the same rating.");
    }
  }
}

let movieA = new Movie("Movie A", 80);
let movieB = new Movie("Movie B", 45);

Movie.compareMovies(movieA, movieB);  // Movie A has a higher rating.
```

:::

I metodi statici sono utili anche per implementare metodi "factory". Un metodo factory è un metodo che definisci oltre al costruttore per creare oggetti basati su criteri specifici.

```js
class Pizza {
  constructor(type, price) {
    this.type = type;
    this.price = price;
  }

  static createMargherita() {
    return new this("Margherita", 6.99);
  }
}

let myPizza = Pizza.createMargherita();
console.log(myPizza);  // Pizza { type: "Margherita", price: 6.99 }
console.log(myPizza.type);  // Margherita
```

- **Proprietà statiche**: Queste proprietà servono a definire valori o attributi associati alla classe stessa, non alle istanze della classe. Le proprietà statiche sono condivise da tutte le istanze della classe e possono essere consultate senza creare un'istanza della classe.

:::interactive_editor

```ts
class Car {
  // Static property
  static numberOfWheels = 4;

  constructor(make, model) {
    this.make = make;
    this.model = model;
  }

  // Instance method
  getCarInfo() {
    return `${this.make} ${this.model}`;
  }

  // Static method
  static getNumberOfWheels() {
    return Car.numberOfWheels;
  }
}

// Accessing static property directly from the class
console.log(Car.numberOfWheels);  // 4
```

:::
