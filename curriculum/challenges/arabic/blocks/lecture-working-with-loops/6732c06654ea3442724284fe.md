# --interactive--

في الدروس السابقة، تعلمت كيفية العمل مع حلقات `for`، و`for...in`، و`for...of`. في هذا الدرس، ستتعلم عن حلقة `while` وحلقة `do...while`.

تُشغّل حلقة `while` كتلة من الشفرة طالما أن الشرط صحيح. إليك الصيغة النحوية الأساسية لحلقة `while`:

```js
while (condition) {
  // code block to be executed
}
```

يُفحص الشرط قبل تنفيذ كتلة الشفرة. إذا كان الشرط خاطئًا، فلن تُنفذ كتلة الشفرة.

تُفيد حلقات `while` عندما لا تعرف عدد المرات التي تحتاج فيها لتشغيل كتلة الشفرة. إليك مثالًا على استخدام حلقة `while`:

:::interactive_editor

```js
let counter = 0;
while(counter < 5) {
  console.log(counter);
  counter++;
}
```

:::

في هذا المثال، لدينا متغير اسمه `counter` مُهيأ إلى `0`. ستستمر حلقة `while` في التشغيل طالما أن قيمة `counter` أقل من `5`. داخل الحلقة، نسجل قيمة `counter` في وحدة التحكم ثم نزيد `counter` بمقدار `1`.

حلقة أخرى مشابهة لحلقة `while` هي حلقة `do...while`. إليك الصيغة النحوية الأساسية:

```js
do {
  // code block to be executed
} while (condition);
```

الفرق الرئيسي بين حلقة `do...while` وحلقة `while` هو أن حلقة `do...while` تنفذ كتلة الشفرة مرة واحدة على الأقل قبل فحص الشرط.

إذا كان الشرط صحيحًا، ستستمر كتلة الشفرة في التنفيذ. إذا كان الشرط خاطئًا، ستتوقف كتلة الشفرة عن التنفيذ.

إليك مثالًا على استخدام حلقة `do...while`:

:::interactive_editor

```js
let counter = 0;
do {
  console.log(counter);
  counter++;
} while (counter < 5);  
```

:::

في هذا المثال، لدينا متغير اسمه `counter` مُهيأ إلى `0`. ستسجل حلقة `do...while` قيمة `counter` في وحدة التحكم ثم تزيد `counter` بمقدار `1`. بعد تنفيذ كتلة الشفرة، تتحقق مما إذا كانت قيمة `counter` أقل من `5`. إذا كانت كذلك، ستستمر الحلقة في التشغيل. إذا لم تكن، ستتوقف الحلقة.

في معظم الحالات، ستستخدم حلقة `while` أكثر من حلقة `do...while`. مع ذلك، من الجيد معرفة كلا النوعين من الحلقات ومتى تستخدم كلًا منهما.
