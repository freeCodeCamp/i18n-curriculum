# --description--

Python에서 `raise` 문은 코드에서 예외를 수동으로 발생시킬 수 있는 강력한 도구입니다. 언제 어떻게 오류가 발생할지 제어할 수 있어, 사용자 정의 오류 조건을 만들고 특정 프로그램 동작을 강제할 수 있습니다.

`raise` 문은 프로그램의 어느 지점에서든 명시적으로 예외를 발생시키는 데 사용되며, 오류 조건이 발생했거나 특정 요구 사항이 충족되지 않았음을 알릴 수 있습니다.

Python의 `raise` 문은 여러 방식으로 예외를 발생시키는 데 사용할 수 있습니다. 가장 기본적으로는 내장 예외를 발생시키거나 사용자 정의 오류 메시지를 만들 수 있습니다. 간단한 예는 다음과 같습니다:

```py
def check_age(age):
    if age < 0:
        raise ValueError('Age cannot be negative')
    return age

try:
    check_age(-5)
except ValueError as e:
    print(f'Error: {e}') # Error: Age cannot be negative
```

여기서 `raise`가 예외를 발생시키는 키워드임을 알 수 있습니다.

이 예에서는 잘못된 나이가 입력되었을 때 사용자 정의 메시지와 함께 `ValueError`를 발생시키고 있습니다.

`raise` 문은 현재 예외를 다시 발생시키는 데도 사용할 수 있는데, 이는 예외 처리에서 특히 유용합니다:

```py
def process_data(data):
    try:
        result = int(data)
        return result * 2
    except ValueError:
        print('Logging: Invalid data received')
        raise  # Re-raises the same ValueError

try:
    process_data('abc')
except ValueError:
    print('Handled at higher level')
```

여기서 인수 없이 `raise` 키워드는 처리 중인 현재 예외를 다시 발생시킵니다.

이렇게 하면 오류를 호출 스택 상위로 전파하면서 로그를 남기거나 정리 작업을 수행할 수 있습니다.

사용자 정의 예외 클래스를 정의하여 직접 예외를 만들고 발생시킬 수도 있습니다:

```py
class InsufficientFundsError(Exception):
    def __init__(self, balance, amount):
        self.balance = balance
        self.amount = amount
        super().__init__(f'Insufficient funds: ${balance} available, ${amount} requested')

def withdraw(balance, amount):
    if amount > balance:
        raise InsufficientFundsError(balance, amount)
    return balance - amount

try:
    new_balance = withdraw(100, 150)
except InsufficientFundsError as e:
    print(f'Transaction failed: {e}')
```

여기서 사용자 정의 예외 클래스가 `Exception` 또는 그 하위 클래스를 상속받는 것을 볼 수 있습니다.

클래스와 상속에 대해서는 앞으로 수업에서 더 배우게 됩니다. 지금은 이것이 사용자 정의 로직을 가진 예외를 만드는 방법임을 알아두세요.

`raise` 문은 `from` 키워드와 함께 사용하여 예외를 연결할 수도 있는데, 이는 서로 다른 오류 간의 관계를 보여줍니다:

```py
def parse_config(filename):
    try:
        with open(filename, 'r') as file:
            data = file.read()
            return int(data)
    except FileNotFoundError:
        raise ValueError('Configuration file is missing') from None
    except ValueError as e:
        raise ValueError('Invalid configuration format') from e

config = parse_config('config.txt')
```

여기서 `raise ... from None`가 원래 예외 컨텍스트를 억제하는 것을 볼 수 있습니다:

```bash
Traceback (most recent call last):
  File "main.py", line 12, in <module>
    config = parse_config('config.txt')
             ^^^^^^^^^^^^^^^^^^^^^^^^^^
  File "main.py", line 7, in parse_config
    raise ValueError('Configuration file is missing') from None
ValueError: Configuration file is missing
```

그리고 `raise ... from e`는 새 예외를 원래 예외에 연결하여 오류 추적을 보존합니다.

```bash
Traceback (most recent call last):
  File "main.py", line 5, in parse_config
    return int(data)
           ^^^^^^^^^
ValueError: invalid literal for int() with base 10: ''

The above exception was the direct cause of the following exception:

Traceback (most recent call last):
  File "main.py", line 12, in <module>
    config = parse_config('config.txt')
             ^^^^^^^^^^^^^^^^^^^^^^^^^^
  File "main.py", line 9, in parse_config
    raise ValueError('Invalid configuration format') from e
ValueError: Invalid configuration format
```

`assert` 문을 사용하여 조건부로 예외를 발생시킬 수도 있는데, 이는 본질적으로 `raise`와 `AssertionError`의 축약형입니다:

```py
def calculate_square_root(number):
    assert number >= 0, 'Cannot calculate square root of negative number'
    return number ** 0.5

try:
    result = calculate_square_root(-4)
except AssertionError as e:
    print(f'Assertion failed: {e}')
```

`raise` 문은 비즈니스 규칙을 강제하고 입력을 검증하며 의미 있는 오류 메시지를 제공해야 하는 견고한 애플리케이션을 만드는 데 필수적입니다. 전략적으로 `raise`를 사용하면 코드를 더 예측 가능하고 디버그하기 쉽게 만들면서 사용자에게 무엇이 잘못되었는지 명확한 피드백을 줄 수 있습니다.
