# --interactive--

I modificatori, spesso chiamati "flag", modificano il comportamento di un'espressione regolare. Ricordiamo il nostro esempio da una lezione precedente:

:::interactive_editor

```js
const regex = /freeCodeCamp/;
console.log(regex.test("freeCodeCamp")); // true
console.log(regex.test("freeCodeCamp is great")); // true
console.log(regex.test("I love freeCodeCamp")); // true
console.log(regex.test("freecodecamp")); // false
console.log(regex.test("FREECODECAMP")); // false
console.log(regex.test("free")); // false
console.log(regex.test("code")); // false
console.log(regex.test("camp")); // false
```

:::

Se ricordi, le stringhe `freeCodeCamp` tutte minuscole e tutte maiuscole non corrispondevano al pattern. Questo perché, per impostazione predefinita, le espressioni regolari sono sensibili al maiuscolo/minuscolo.

Ma se potessimo dire all'espressione regolare di non fare distinzione tra maiuscole e minuscole? Beh, esiste un modificatore per questo. Il flag `i` fa sì che una regex ignori il maiuscolo/minuscolo. Come si usa? I flag vanno dopo la barra finale in un'espressione regolare:

```js
const regex = /freeCodeCamp/i;
```

Nota il cambiamento nell'espressione regolare sulla prima riga. Ora possiamo vedere come cambia il risultato:

:::interactive_editor

```js
const regex = /freeCodeCamp/i;

console.log(regex.test("freeCodeCamp")); // true
console.log(regex.test("freeCodeCamp is great")); // true
console.log(regex.test("I love freeCodeCamp")); // true
console.log(regex.test("freecodecamp")); // true
console.log(regex.test("FREECODECAMP")); // true
console.log(regex.test("free")); // false
console.log(regex.test("code")); // false
console.log(regex.test("camp")); // false
```

:::

Poiché la nostra espressione regolare ora non fa distinzione tra maiuscole e minuscole, le stringhe tutte minuscole e tutte maiuscole "superano" il test. Questo funziona anche per una stringa con una combinazione casuale di lettere maiuscole e minuscole:

:::interactive_editor

```js
const regex = /freeCodeCamp/i;

console.log(regex.test("dO yOu LoVe fReEcOdEcAmP?")); // true
```

:::

Ci sono molti altri flag che puoi usare. Il flag `g`, o modificatore globale, permette alla tua espressione regolare di trovare più corrispondenze di un pattern.

Vediamo come questo influisce sul nostro codice. Noterai che abbiamo mantenuto il flag `i` – un'espressione regolare può usare più flag (tanti quanti servono) per ottenere il comportamento desiderato:

```js
const regex = /freeCodeCamp/gi;
```

Aspetta un attimo... cos'è questo? Sembra che il modificatore globale stia facendo fallire alcune delle nostre stringhe che invece dovrebbero passare:

:::interactive_editor

```js
const regex = /freeCodeCamp/gi;

console.log(regex.test("freeCodeCamp")); // true
console.log(regex.test("freeCodeCamp is great")); // false
console.log(regex.test("I love freeCodeCamp")); // true
console.log(regex.test("freecodecamp")); // false
console.log(regex.test("FREECODECAMP")); // true
console.log(regex.test("free")); // false
console.log(regex.test("code")); // false
console.log(regex.test("camp")); // false
```

:::

Perché? Il modificatore globale rende la tua espressione regolare con stato. Questo significa che tiene traccia di dove ha già trovato una corrispondenza. Quindi, quando trova la prima stringa `freeCodeCamp`, ricorda che ha trovato una corrispondenza a partire dall'indice `0`.

Poi la testiamo con `freeCodeCamp is great`, ma non inizia all'indice `0`. L'espressione regolare "sa" di aver già trovato una corrispondenza all'indice `0`, quindi anche se questa è una stringa diversa, parte dall'indice finale della corrispondenza.

`freeCodeCamp` è lunga `12` caratteri, quindi una corrispondenza a `0` termina all'indice `11`. La ricerca riprende all'indice `12`. E poiché `is great` non corrisponde a `freeCodeCamp`, restituisce `false`.

Poi, poiché non trova una corrispondenza, "perde" il suo stato e ricomincia la ricerca successiva dall'indice `0`.

Se invertiamo i nostri log in modo che una stringa con la corrispondenza a `0` sia seguita immediatamente da una stringa che ha una corrispondenza più avanti dell'indice `11`:

:::interactive_editor

```js
const regex = /freeCodeCamp/gi;

console.log(regex.test("freeCodeCamp")); // true
console.log(regex.test("I loooooooove freeCodeCamp")); // true
```

:::

Quando un'espressione regolare è globale, ottiene una nuova proprietà chiamata `lastIndex`. Riprendendo il nostro codice precedente, vediamo come funziona questa proprietà:

:::interactive_editor

```js
const regex = /freeCodeCamp/gi;

console.log(regex.lastIndex); // 0
console.log(regex.test("freeCodeCamp")); // true
console.log(regex.lastIndex); // 12
console.log(regex.test("freeCodeCamp is great")); // false
console.log(regex.lastIndex); // 0
console.log(regex.test("I love freeCodeCamp")); // true
console.log(regex.lastIndex); // 19
console.log(regex.test("freecodecamp")); // false
console.log(regex.lastIndex); // 0
console.log(regex.test("FREECODECAMP")); // true
console.log(regex.lastIndex); // 12
console.log(regex.test("free")); // false
console.log(regex.lastIndex); // 0
console.log(regex.test("code")); // false
console.log(regex.lastIndex); // 0
console.log(regex.test("camp")); // false
```

:::

Guardando questo esempio, puoi vedere come lo stato dell'espressione regolare cambia ad ogni chiamata di test usando `lastIndex` per tenere traccia delle corrispondenze precedenti.

Il flag globale è ottimo quando hai bisogno di trovare più corrispondenze in una singola stringa. Ma se stai testando più stringhe con la stessa espressione regolare, è meglio non usare il flag `g`.

Prima di imparare il prossimo flag, devi conoscere le àncore. L'àncora caret (`^`) all'inizio dell'espressione regolare indica "corrispondi all'inizio della stringa":

```js
const start = /^freecodecamp/i;
```

L'àncora del simbolo del dollaro (`$`) alla fine dell'espressione regolare indica "corrispondi alla fine della stringa":

```js
const end = /freecodecamp$/i;
```

Prenditi un momento per confrontare gli output a destra:

:::interactive_editor

```js
const start = /^freecodecamp/i;
const end = /freecodecamp$/i;
console.log(start.test("freecodecamp")); // true
console.log(end.test("freecodecamp")); // true
console.log(start.test("freecodecamp is great")); // true
console.log(end.test("freecodecamp is great")); // false
console.log(start.test("i love freecodecamp")); // false
console.log(end.test("i love freecodecamp")); // true
console.log(start.test("have met freecodecamp's founder")); // false
console.log(end.test("have met freecodecamp's founder")); // false
```

:::

Vedi come l'àncora di inizio corrisponde solo all'inizio della stringa, e l'àncora di fine solo alla fine della stringa? Ma cosa succede se vogliamo corrispondenze su più righe? Vediamolo:

:::interactive_editor

```js
const start = /^freecodecamp/i;
const end = /freecodecamp$/i;
const string = `I really love
freecodecamp
it's my favorite`;
console.log(start.test(string)); // false
console.log(end.test(string)); // false
```

:::

Anche se `freecodecamp` è lì su una riga a sé, fallisce entrambi i test. Questo perché, per impostazione predefinita, le àncore cercano l'inizio e la fine dell'intera stringa.

Ma puoi far sì che una regex gestisca più righe con il flag `m`, o modificatore multi-linea. Aggiungiamolo alle nostre espressioni regolari per vedere cosa otteniamo:

:::interactive_editor

```js
const start = /^freecodecamp/im;
const end = /freecodecamp$/im;
const string = `I really love
freecodecamp
it's my favorite`;
console.log(start.test(string)); // true
console.log(end.test(string)); // true
```

:::

Ora entrambe corrispondono! Poiché `freecodecamp` è completamente su una riga a sé, l'àncora di inizio corrisponde all'inizio di quella riga, e l'àncora di fine corrisponde alla fine di quella riga.

Infine, c'è il flag `d`, o modificatore degli indici. Ricorda che il flag `i` serve per ignorare il maiuscolo/minuscolo, quindi il modificatore degli indici ha bisogno di un flag diverso.

Il flag `d` espande le informazioni che ottieni in un oggetto di corrispondenza. Aggiungiamolo alla nostra espressione regolare:

```js
const regex = /freecodecamp/di;
const string = "we love freecodecamp isn't freecodecamp great?";
console.log(string.match(regex));
```

E il risultato è:

```js
// [
//   'freecodecamp',
//   index: 8,
//   input: "we love freecodecamp isn't freecodecamp great?",
//   groups: undefined,
//   indices: [
//     0: [8, 20],
//     groups: undefined
//   ]
// ]
```

Il nostro oggetto di corrispondenza ottiene una nuova proprietà `indices`! Questa proprietà è un array di due numeri, il primo è l'indice nella stringa originale dove inizia la corrispondenza, e il secondo è l'indice subito dopo la fine della corrispondenza. Questo array ha anche una proprietà extra `groups`, che serve per i gruppi di cattura nominati.

Ci sono altri flag che dovresti sapere essere disponibili, ma sono meno comuni nel codice tipico.

Il primo è il modificatore unicode, o flag `u`. Questo espande la funzionalità di un'espressione regolare per permettere di corrispondere caratteri unicode speciali.

Imparerai di più sulle classi di caratteri in una lezione futura, ma il flag `u` ti dà accesso a classi speciali come `Extended_Pictographic` per corrispondere alla maggior parte delle emoji:

:::interactive_editor

```js
const regex = /🍎/u;

const str = "I have an apple 🍎";
console.log(regex.test(str)); // true
```

:::

C'è anche un flag `v`, che espande ulteriormente la funzionalità della corrispondenza unicode.

Il secondo è il modificatore sticky, o flag `y`. Il modificatore sticky si comporta in modo molto simile al modificatore globale, ma con alcune eccezioni.

La più importante è che un'espressione regolare globale inizierà da `lastIndex` e cercherà in tutto il resto della stringa un'altra corrispondenza, mentre un'espressione regolare sticky restituirà `null` e resetterà `lastIndex` a `0` se non c'è subito una corrispondenza all'indice precedente `lastIndex`.

E l'ultimo è il modificatore single-line, o flag `s`. Ricorda che il modificatore multi-linea permette alle àncore di inizio e fine di corrispondere all'inizio e alla fine di una riga, invece che dell'intera stringa.

Il modificatore single-line permette a un carattere jolly, rappresentato da un punto (`.`) nella regex, di corrispondere anche ai ritorni a capo – trattando effettivamente la stringa come una singola riga di testo.

Ci sono molti di questi modificatori, ma i flag `i` e `g` sono quelli che userai più frequentemente e sono i più importanti da ricordare.
