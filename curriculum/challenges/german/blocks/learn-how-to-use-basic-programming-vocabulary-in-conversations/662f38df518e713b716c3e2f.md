## --text--

Welche Aussage beschreibt `object-oriented programming` am besten, wie Sarah es erklärt hat?

## --answers--

`Object-oriented programming` handelt davon, eine Folge von Befehlen zu erstellen, denen der Computer nacheinander folgt.

### --feedback--

Diese Aussage beschreibt prozedurale Programmierung, nicht `object-oriented programming`, das den Fokus auf Klassen und Objekte legt.

---

`Object-oriented programming` beinhaltet hauptsächlich die Verwendung von Standardvorlagen für die Webentwicklung.

### --feedback--

Diese Option beschreibt `object-oriented programming` falsch, da sie die Erstellung und Interaktion von Objekten, die im Zentrum von `OOP` stehen, nicht erwähnt.

---

`Object-oriented programming` verwendet Klassen als Blaupausen zur Erstellung von Objekten, und Methoden innerhalb dieser Klassen definieren, welche Aktionen die Objekte ausführen können.

---

`Object-oriented programming` ist eine einfache Methode, bei der bestehender Code in neue Programme kopiert wird.

### --feedback--

Diese Beschreibung ist falsch, da `object-oriented programming` nicht das Kopieren von Code bedeutet, sondern das Entwerfen von Systemen durch Objekte und Klassen.
