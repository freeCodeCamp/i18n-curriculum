# --interactive--

Ви могли бачити атрибут `target` на елементах посилання, або посиланнях. Цей важливий атрибут повідомляє браузеру, де відкривати URL для елемента посилання.

Увімкніть інтерактивний редактор, натисніть на посилання, і ви перейдете на головну сторінку freeCodeCamp у новій вкладці браузера.

:::interactive_editor

```html
<a href="https://freecodecamp.org" target="_blank">Visit freeCodeCamp</a>
```

:::

Існує чотири важливі можливі значення для цього атрибута. Зверніть увагу, що кожне значення передує підкреслення.

Перше значення — `_self`, яке є значенням за замовчуванням. Воно відкриває посилання в поточному контексті перегляду. У більшості випадків це буде поточна вкладка або вікно.

Друге значення — `_blank`, яке відкриває посилання в новому контексті перегляду. Зазвичай це відкривається у новій вкладці. Але деякі користувачі можуть налаштувати свої браузери так, щоб відкривати нове вікно замість цього.

Третє значення — `_parent`, яке відкриває посилання у батьківському елементі поточного контексту. Наприклад, якщо ваш сайт має `iframe`, значення `_parent` у цьому `iframe` відкривалося б у вкладці/вікні вашого сайту, а не у вкладеному фреймі.

Четверте значення — `_top`, яке відкриває посилання у найвищому контексті перегляду — подумайте "батько батька". Це схоже на `_parent`, але посилання завжди відкриватиметься у повній вкладці/вікні браузера, навіть для вкладених вставлених фреймів.

Існує п’яте значення, яке називається `_unfencedTop`, і наразі використовується для експериментального API FencedFrame. На момент цього уроку, ймовірно, у вас ще не буде причин його використовувати.

Вибір правильного значення `target` для контролю того, куди потрапляють ваші користувачі, є важливим аспектом при створенні сайту.
