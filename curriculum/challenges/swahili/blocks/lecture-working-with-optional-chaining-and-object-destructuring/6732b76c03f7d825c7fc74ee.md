# --interactive--

Operator ya uunganishaji wa mfululizo (`?.`) ni chombo muhimu katika JavaScript kinachokuwezesha kupata kwa usalama sifa za kitu au kuita njia bila kuwa na wasiwasi kama zipo au la. Ni kama mtandao wa usalama wa kufanya kazi na vitu ambavyo huenda vikawa na sehemu zilizokosekana.

:::interactive_editor

```js
const person = {
  name: "Alice",
  age: 30
};

console.log(person.name); // "Alice"
console.log(person.job); // undefined
```

:::

Katika mfano huu, `person.name` ipo, hivyo inarekodi `Alice`. Lakini `person.job` haipo, hivyo inatupa `undefined`.

Sasa, tuseme tunataka kupata sifa ya kitu ambacho huenda hakipo:

```js
const person = {
  name: "Alice",
  age: 30
};

console.log(person.address.street); // This will throw an error!
```

Mfano huu utatupa `Uncaught TypeError`. Kwa kuwa `person.address` ni `undefined`, hatuwezi kupata sifa ya `street`. Hapa ndipo operator ya uunganishaji wa mfululizo inakuwa na msaada. Hapa kuna mfano wa kutumia operator ya uunganishaji wa mfululizo:

:::interactive_editor

```ts
const user = {
  name: "John",
  profile: {
    email: "john@example.com",
    address: {
      street: "123 Main St",
      city: "Somewhere"
    }
  }
};

console.log(user?.profile?.address?.street); // "123 Main St"
console.log(user?.profile?.phone?.number);   // undefined
```

:::

Kwa kutumia operator ya uunganishaji wa mfululizo, tunaambia JavaScript kuendelea na operesheni tu ikiwa kitu (au thamani kabla ya `?.`) kipo na si `null` au `undefined`. 

Ikiwa thamani kabla ya `?.` ni `null` au `undefined`, JavaScript inarudisha `undefined` badala ya kujaribu kuendelea na operesheni na kutoa kosa.

Kumbuka, operator ya uunganishaji wa mfululizo ni muhimu zaidi wakati hujui kama sifa au njia ipo. Husaidia kuzuia makosa na kufanya msimbo wako kuwa imara zaidi.
