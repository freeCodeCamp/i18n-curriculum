# --description--

Mocking هو عملية استبدال البيانات الحقيقية ببيانات زائفة تحاكي سلوك المكوِّنات الحقيقية. هذا يسمح للمختبرين والمطوِّرين باختبار مكوِّنات محددة في التطبيق دون الاعتماد على التبعيات الخارجية.

حالة استخدام شائعة لـ mocking البيانات تكون عند التعامل مع استدعاءات API. تخيل أن لديك استدعاء fetch مثل هذا في تطبيقك:

```js
export const postService = {
  async getPosts() {
    const response = await fetch("https://api.example.com/posts");
    return response.json();
  },
};
```

إذا كنت بحاجة لاختبار الدالة `postService`، يمكنك محاكاة استجابة API بدلاً من إجراء استدعاءات API مستمرة لجلب البيانات. جلب البيانات من API يستغرق وقتًا ويعتمد على عوامل خارجية مثل توفر الشبكة واستجابات الخادم. محاكاة استجابة API تزيل تلك العوامل الخارجية وتسمح للمختبرين بالعمل مع مجموعة بيانات متوقعة ومتحكم بها.

إليك مثال على استجابة API محاكاة:

```json
[
  {
    "id": 1,
    "title": "Understanding Async/Await in JavaScript",
    "content": "Async/Await makes asynchronous code look and behave like synchronous code...",
    "author": "Jane Doe",

  },
  {
    "id": 2,
    "title": "10 Tips for Writing Clean Code",
    "content": "Clean code is not just about code formatting. It involves naming, architecture...",
    "author": "John Smith",
    "created_at": "2025-04-08T09:21:00Z"
  },
  {
    "id": 3,
    "title": "Exploring React 19 Features",
    "content": "React 19 comes with exciting features like new hooks and better performance...",
    "author": "Alex Lee",
    "created_at": "2025-04-07T18:47:00Z"
  }

```

الآن بعد أن فهمنا ما هو mocking، دعنا نقارنه بـ stubbing. الـ stubs هي كائنات تُرجع استجابات محددة مسبقًا أو بيانات وهمية لسلوك متوقع في التطبيق. على سبيل المثال، يمكنك عمل stub لسلوك اتصال قاعدة بيانات في اختباراتك دون الاعتماد على اتصال قاعدة بيانات حقيقي.

إليك مثال على stub في Jest:

```js
import { postService } from "./postService";
import { db } from "./db";

jest.mock("./db", () => ({
  db: {
    fetchPosts: jest.fn(),
  },
}));

test("should return stubbed posts", async () => {
  const fakePosts = [
    { id: 1, title: "Stubbed Post", content: "This is a stubbed post." },
  ];

  db.fetchPosts.mockResolvedValue(fakePosts);

  const posts = await postService.getPosts();

  expect(posts).toEqual(fakePosts);
});
```

`db.fetchPosts.mockResolvedValue(fakePosts);` هو الـ stub في هذا المثال.

المكوِّن الأخير الذي سنتحدث عنه هو fakes. الـ fakes هي نسخ مبسطة من المكوِّنات الحقيقية بدون التعقيد أو التأثيرات الجانبية للأصلية. على سبيل المثال، يمكنك تقليد قاعدة بيانات بتخزين البيانات في الذاكرة بدلاً من التفاعل مع قاعدة البيانات الحقيقية. هذا يسمح لك بمحاكاة عمليات قاعدة البيانات في الذاكرة، وهو أسرع بكثير من التعامل مع قاعدة البيانات الحقيقية. مثال شائع آخر هو استخدام fakes عند العمل مع أنظمة الملفات. يمكنك إنشاء نظام ملفات مزيف بدلاً من النظام الحقيقي عند الاختبار.

بينما تستمر في بناء تطبيقات برمجية أكثر تعقيدًا، فكر في استخدام mocks أو stubs أو fakes حيثما كان ذلك مناسبًا في اختباراتك.
