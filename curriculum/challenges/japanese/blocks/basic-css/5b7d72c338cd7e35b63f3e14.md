# --description--

CSSを扱うとき、いずれブラウザの互換性の問題に直面することがあります。だからこそ、潜在的な問題を避けるためにブラウザのフォールバックを用意することが重要です。

ブラウザがウェブページのCSSをパースするとき、認識しないまたはサポートしていないプロパティは無視されます。たとえば、CSS変数を使ってサイトの背景色を代入した場合、Internet ExplorerはCSS変数をサポートしていないため背景色を無視します。この場合、ブラウザはそのプロパティに設定されている他の値を使います。もし他に設定された値がなければ、通常は望ましくないデフォルト値に戻ります。

つまり、ブラウザのフォールバックを提供したい場合は、既存の宣言の直前により広くサポートされている別の値を用意するだけで簡単にできます。そうすれば、古いブラウザはフォールバックとしてその値を使い、新しいブラウザはカスケードで後にある宣言を解釈します。

# --instructions--

`.red-box` クラスの背景色を設定するために変数が使われているようです。既存の宣言の直前に別の `background` 宣言を追加し、その値を `red` に設定してブラウザの互換性を向上させましょう。

# --hints--

`.red-box` ルールには、既存の `background` 宣言の直前に `red` を `background` に設定したフォールバックを含める必要があります。

```js
const spacelessCode = __helpers.removeWhiteSpace(__helpers.removeCssComments(code));
assert.match(
  spacelessCode,
  /\.red-box{background:(red|#ff0000|#f00|rgb\(255,0,0\)|rgb\(100%,0%,0%\)|hsl\(0,100%,50%\));background:var\(--red-color\);height:200px;width:200px;}/gi
);
```

## --seed-contents--

```html
<style>
  :root {
    --red-color: red;
  }
  .red-box {

    background: var(--red-color);
    height: 200px;
    width:200px;
  }
</style>
<div class="red-box"></div>
```
