# --interactive--

L’hoisting è un comportamento di JavaScript che spesso confonde chi inizia, ma capirlo può aiutarti a evitare bug sottili nel tuo codice. In termini semplici, l’hoisting è il comportamento predefinito di JavaScript di spostare le dichiarazioni in cima ai rispettivi ambiti durante la fase di compilazione, prima che il codice venga eseguito.

Per capire l’hoisting, è importante sapere che JavaScript funziona in due fasi: la fase di compilazione e la fase di esecuzione. Durante la fase di compilazione, il motore JavaScript analizza il tuo codice e prepara lo spazio di memoria per variabili e funzioni. Qui entra in gioco l’hoisting.

Iniziamo con l’hoisting delle variabili: quando dichiari una variabile usando la parola chiave `var`, JavaScript sposta la dichiarazione in cima al suo ambito. Tuttavia è fondamentale notare che viene spostata solo la dichiarazione, non l’inizializzazione. Questo significa che puoi usare una variabile nel tuo codice prima di averla dichiarata, ma il suo valore sarà `undefined` finché non le assegni effettivamente un valore.

:::interactive_editor

```js
console.log(x); // undefined
var x = 5;
console.log(x); // 5
```

:::

In questo codice, anche se usiamo `x` prima di dichiararla, non otteniamo un errore, invece otteniamo `undefined`. Questo perché JavaScript sposta la dichiarazione `var x` in cima al suo ambito ma non l’inizializzazione `x = 5`. È come se il codice fosse riscritto così:

:::interactive_editor

```js
var x;
console.log(x); // undefined
x = 5;
console.log(x); // 5
```

:::

L’hoisting delle funzioni funziona in modo un po’ diverso. Quando dichiari una funzione usando la sintassi di dichiarazione `function`, sia il nome della funzione sia il corpo della funzione vengono spostati. Questo significa che puoi chiamare una funzione prima di averla dichiarata nel codice. Ecco un esempio di hoisting delle funzioni:

:::interactive_editor

```js
sayHello(); // "Hello, World!"

function sayHello(){
  console.log("Hello, World!");
}
```

:::

In questo caso, possiamo chiamare `sayHello()` prima della sua dichiarazione perché l’intera funzione viene spostata in cima al suo ambito.

È importante notare che l’hoisting funziona diversamente con le dichiarazioni `let` e `const` introdotte in ES6.

```js
console.log(y); // Throws a ReferenceError
let y = 10;
```

Queste dichiarazioni vengono spostate ma non sono inizializzate e non puoi consultarle prima della dichiarazione effettiva nel codice. Questo è spesso chiamato zona morta temporale.

Capire l’hoisting può aiutarti a scrivere codice più pulito e prevedibile. Tuttavia, affidarsi all’hoisting può rendere il codice più difficile da leggere e mantenere. Come buona pratica, è consigliato dichiarare le variabili in cima al loro ambito e le funzioni prima di usarle, indipendentemente dall’hoisting. Questo rende il comportamento del codice più esplicito e più facile da capire per gli altri, incluso il te stesso del futuro.
