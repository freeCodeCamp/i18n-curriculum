# --interactive--

在 JavaScript 中，了解原始型別與非原始型別資料之間的差異，對於撰寫高效且無錯誤的程式碼非常重要。 

這兩類資料型別在記憶體中的儲存方式以及在你的程式中如何操作行為不同。

原始資料型別是 JavaScript 中最簡單的資料形式。它們包括數字、字串、布林值、`null`、`undefined` 和符號。這些型別被稱為「原始」，因為它們表述單一值且不是物件。 

當你處理原始資料型別時，你是直接操作它們的值。例如，當你用原始值創建一個變數時，該值會直接儲存在變數中。 

原始值是不可變的，這表示一旦它們被創建，其值就無法更改。然而，你可以將新值指定給變數。以下是操作原始資料型別的範例：

:::interactive_editor

```js
let num1 = 5;
let num2 = num1;
num1 = 10;

console.log(num2); // 5
```

:::

在此範例中，我們將原始值（`5`）從 `num1` 指定到 `num2`。這會產生該值的獨立副本。因此，對原始變數（`num1`）所做的任何更改都不會影響副本（`num2`）。

非原生類型的資料則較為複雜。在 JavaScript 中，這些是物件，包括一般物件、陣列和函式（程式）。與原生類型不同，非原生類型可以將多個值作為屬性或元素來儲存。 

當你用非原生類型的值創建變數時，實際儲存在變數中的，是指向物件所在記憶體位置的引用，而不是物件本身。這會導致一些行為上的重要差異。以下是非原生類型的範例：

:::interactive_editor

```js
const originalPerson = { name: "John", age: 30 };
const copiedPerson = originalPerson;

originalPerson.age = 31;

console.log(copiedPerson.age); // 31
```

:::

在此範例中，我們有一個名為 `originalPerson` 的物件，具有兩個屬性 `name` 和 `age`。接著，我們將 `originalPerson` 物件指定給一個名為 `copiedPerson` 的變數。

然後我們更新 `originalPerson` 物件的 `age` 值。當我們記錄 `copiedPerson` 物件的 `age` 屬性時，它會顯示更新後的值。

但是為什麼會發生這種情況？這是因為 `originalPerson` 和 `copiedPerson` 都取用同一個記憶體中的物件。

在 JavaScript 中，當你將一個物件指定給另一個變數時，你是在複製對該物件的引用，而不是物件本身。這稱為透過引用的淺層複製。因此，透過其中一個引用對物件所做的任何更改，都會反映在所有對該物件的引用中。

當你繼續使用 JavaScript 時，你會遇到許多情況，理解原生型別與非原生型別之間的差異非常重要。這是一個基本的概念，支撐語言的許多面向，並且是撰寫高效且正確程式碼的關鍵。
