# --description--

У попередньому завданні `bird` мав публічну властивість `name`. Вона вважається публічною, бо до неї можна отримати доступ і змінити її поза межами визначення `bird`.

```js
bird.name = "Duffy";
```

Отже, будь-яка частина вашого коду може легко змінити ім’я `bird` на будь-яке значення. Уявіть собі, що паролі чи банківські рахунки можна легко змінити будь-якою частиною вашої кодової бази. Це може спричинити багато проблем.

Найпростіший спосіб зробити цю публічну властивість приватною — створити змінну всередині функції конструктора. Це змінює область видимості цієї змінної так, що вона існує лише всередині функції конструктора, а не глобально. Таким чином, до змінної можна отримати доступ і змінити її лише за допомогою методів, які також знаходяться всередині функції конструктора.

```js
function Bird() {
  let hatchedEgg = 10;

  this.getHatchedEggCount = function() { 
    return hatchedEgg;
  };
}
let ducky = new Bird();
ducky.getHatchedEggCount();
```

Тут `getHatchedEggCount` є привілейованим методом, бо має доступ до приватної змінної `hatchedEgg`. Це можливо, тому що `hatchedEgg` оголошена в тому самому контексті, що й `getHatchedEggCount`. У JavaScript функція завжди має доступ до контексту, в якому вона була створена. Це називається `closure`.
