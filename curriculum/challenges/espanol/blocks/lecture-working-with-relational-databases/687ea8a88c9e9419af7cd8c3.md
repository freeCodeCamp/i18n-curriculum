# --description--

Veamos algunos de los comandos fundamentales que puedes usar para insertar y ver datos en SQL.

Primero que nada, para ver datos, necesitas insertarlos. Así que comenzaremos con las inserciones.

Los ejemplos que cubriremos en esta lección estarán basados en esta tabla de registros de perros:

```sql
CREATE TABLE dogs(
  id SERIAL,
  name VARCHAR(100),
  age INTEGER
);
```

La tabla almacenará los nombres y edades de varios perros.

Si asumimos que estamos trabajando con PostgreSQL, podemos usar el tipo de dato `SERIAL` para el `id`. Dado que `id` fue creado usando `SERIAL`, su valor será un `INTEGER`, comenzando desde `1` e incrementándose automáticamente cuando se inserte un nuevo registro. Por lo tanto, no necesitarás pasar un valor para él cuando insertes un registro.

Comencemos insertando registros en nuestra tabla `dogs` con la sentencia `INSERT INTO`. Hay muchas formas de hacerlo. La primera sería insertar una sola fila y especificar las columnas de los valores que se insertarán.

Escribes `INSERT INTO`, seguido del nombre de la tabla (en este caso, `dogs`), luego las columnas entre paréntesis, después `VALUES` y sus valores correspondientes entre paréntesis.

Este es un ejemplo, estamos insertando un registro de perro con el `name` de `'Gino'`, cuyo `age` es `3`, recuerda que el `id` se asignará automáticamente:

```sql
INSERT INTO dogs (name, age)
VALUES ('Gino', 3);
```

Esta es usualmente la opción más segura porque especificas las columnas explícitamente, así que los valores se asignarán a esas columnas en orden.

Nota que `'Gino'` debe estar entre comillas simples porque es un tipo `VARCHAR`.

Otra alternativa es insertar el registro sin especificar las columnas, así:

```sql
INSERT INTO dogs
VALUES ('Gino', 3);
```

Esta sintaxis es SQL válido, pero propenso a errores.

Dado que los valores se asignan según el orden de las columnas, PostgreSQL intentará insertar `'Gino'` en la columna `id`, que espera un entero. Esto resulta en un error como `invalid input syntax for type integer`.

Por esta razón, es más seguro especificar explícitamente los nombres de las columnas al insertar datos.

También puedes insertar múltiples registros en el mismo comando SQL separándolos con una coma.

Aquí, estamos insertando dos registros de perros, uno para `'Gino'` y otro para `'Nora'`.

```sql
INSERT INTO dogs (name, age)
VALUES
  ('Gino', 3),
  ('Nora', 2);
```

Y así como puedes insertar registros, puedes consultar información de la base de datos. En el contexto de las bases de datos, una consulta es una solicitud de datos.

En SQL, la sentencia `SELECT` se usa para consultar datos de una o más tablas. Puedes personalizar la consulta para obtener la información exacta que necesitas.

Si necesitas consultar toda la información de la tabla `dogs`, solo tienes que escribir `SELECT`, seguido de un asterisco (`*`), luego `FROM` y el nombre de la tabla, `dogs`:

```sql
SELECT *
FROM dogs;
```

En este comando, el asterisco es un carácter comodín que representa "todas las columnas."

Obtendrás todos los datos de todos los registros en la tabla:

```sql
 id | name | age 
----+------+-----
  1 | Gino |   3
  2 | Nora |   2
```

Para consultar columnas específicas, puedes escribir los nombres de esas columnas en el comando, justo después de `SELECT`, separados por una coma. En este ejemplo, consultamos las columnas `name` y `age` de la tabla `dogs`:

```sql
SELECT name, age
FROM dogs;
```

El resultado solo incluirá las columnas `name` y `age`:

```sql
 name | age 
------+-----
 Gino |   3
 Nora |   2
```

A veces, podrías necesitar consultar datos basándote en una condición específica. Por ejemplo, para obtener todos los perros que tienen menos de 3 años puedes usar la palabra clave `WHERE` y el operador de comparación menor que (`<`):

```sql
SELECT *
FROM dogs
WHERE age < 3;
```

El resultado solo incluirá perros cuya edad sea menor a 3:

```sql
 id | name | age 
----+------+-----
  2 | Nora |   2
```

Si solo estás tratando de encontrar el `age` de `'Gino'`, puedes usar el operador de comparación igual (`=`):

```sql
SELECT age
FROM dogs
WHERE name = 'Gino';
```

Y aquí está el resultado:

```sql
 age 
-----
  3
```

Estas son formas comunes y sencillas de insertar y ver datos en SQL, pero hay muchas opciones diferentes que puedes elegir y algunas de ellas son más avanzadas. Las aprenderás en las próximas lecciones.
