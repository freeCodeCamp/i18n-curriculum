# --explanation--

`Clarifying` bedeutet, etwas so zu erklären, dass Verwirrung beseitigt wird. Es hilft jemandem, ein Konzept oder eine Situation besser zu verstehen. Zum Beispiel:

`Thanks for clarifying how the API works.` – Der Sprecher ist dankbar, weil die Erklärung die API leichter verständlich gemacht hat.

`Abstraction` bezieht sich darauf, etwas zu vereinfachen, indem nur die wesentlichen Funktionen gezeigt und die komplexen internen Abläufe verborgen werden. Zum Beispiel:

`Using a login function is an example of abstraction - you don't need to know how the password is checked, just that it works.` – Dies zeigt, wie Nutzer mit einfachen Oberflächen interagieren, während die technische Logik verborgen bleibt.
