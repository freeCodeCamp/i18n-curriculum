# --description--

SQL에서 데이터를 삽입하고 조회하는 데 사용할 수 있는 기본 명령어들을 살펴보겠습니다.

먼저 데이터를 조회하려면 데이터를 삽입해야 합니다. 그래서 삽입부터 시작하겠습니다.

이번 수업에서 다룰 예제는 개 기록을 담은 이 표를 기반으로 합니다:

```sql
CREATE TABLE dogs(
  id SERIAL,
  name VARCHAR(100),
  age INTEGER
);
```

이 표는 여러 개의 개 이름과 나이를 저장합니다.

PostgreSQL을 사용한다고 가정하면, `SERIAL`의 데이터 타입으로 `id`를 사용할 수 있습니다. `id`는 `SERIAL`를 사용해 생성되었기 때문에, 그 값은 `INTEGER`이며 `1`부터 시작하고 새 레코드가 삽입될 때 자동으로 증가합니다. 따라서 레코드를 삽입할 때 이 값은 전달할 필요가 없습니다.

`dogs` 문으로 `INSERT INTO` 테이블에 레코드를 삽입하는 것부터 시작하겠습니다. 이를 수행하는 방법은 여러 가지가 있습니다. 첫 번째는 한 행을 삽입하고 삽입할 값의 열을 지정하는 방법입니다.

`INSERT INTO`를 작성한 뒤 테이블 이름(이 경우 `dogs`)을 쓰고, 소괄호 안에 열 이름들을 적고, `VALUES`를 쓴 다음 대응하는 값을 소괄호 안에 적습니다.

예를 들어, `name`이 `'Gino'`이고 `age`이 `3`인 개 기록을 삽입하는 예입니다. `id`는 자동으로 할당된다는 점을 기억하세요:

```sql
INSERT INTO dogs (name, age)
VALUES ('Gino', 3);
```

이 방법이 보통 가장 안전한데, 열을 명확히 지정하기 때문에 값이 그 열에 순서대로 할당되기 때문입니다.

`'Gino'`는 `VARCHAR` 타입이므로 작은따옴표로 감싸야 한다는 점에 주의하세요.

또 다른 방법은 열을 지정하지 않고 레코드를 삽입하는 것입니다:

```sql
INSERT INTO dogs
VALUES ('Gino', 3);
```

이 구문은 유효한 SQL이지만 오류가 발생하기 쉽습니다.

값이 열 순서에 따라 할당되기 때문에, PostgreSQL은 `'Gino'`를 정수를 기대하는 `id` 열에 삽입하려고 시도합니다. 이로 인해 `invalid input syntax for type integer` 같은 오류가 발생합니다.

이런 이유로 데이터를 삽입할 때는 열 이름을 명확히 지정하는 것이 더 안전합니다.

또한 여러 레코드를 한 SQL 명령어에서 쉼표로 구분해 삽입할 수도 있습니다.

여기서는 `'Gino'`과 `'Nora'` 두 개의 개 기록을 삽입하고 있습니다.

```sql
INSERT INTO dogs (name, age)
VALUES
  ('Gino', 3),
  ('Nora', 2);
```

레코드를 삽입할 수 있듯이 데이터베이스에서 정보를 조회할 수도 있습니다. 데이터베이스 맥락에서 쿼리는 데이터를 요청하는 것입니다.

SQL에서는 `SELECT` 문을 사용해 하나 이상의 테이블에서 데이터를 조회합니다. 필요한 정확한 정보를 얻도록 쿼리를 맞춤 설정할 수 있습니다.

`dogs` 테이블의 모든 정보를 조회하려면 `SELECT`를 쓰고 별표(`*`)를 쓴 뒤 `FROM`와 테이블 이름 `dogs`을 적으면 됩니다:

```sql
SELECT *
FROM dogs;
```

이 명령어에서 별표는 "모든 열"을 나타내는 와일드카드 문자입니다.

테이블의 모든 레코드에 대한 모든 데이터를 얻을 수 있습니다:

```sql
 id | name | age 
----+------+-----
  1 | Gino |   3
  2 | Nora |   2
```

특정 열만 조회하려면 `SELECT` 바로 뒤에 쉼표로 구분해 열 이름을 적으면 됩니다. 이 예에서는 `name` 테이블의 `age`과 `dogs` 열을 조회합니다:

```sql
SELECT name, age
FROM dogs;
```

결과에는 `name`과 `age` 열만 포함됩니다:

```sql
 name | age 
------+-----
 Gino |   3
 Nora |   2
```

때로는 특정 조건에 따라 데이터를 조회해야 할 때가 있습니다. 예를 들어, 3살 미만인 모든 개를 조회하려면 `WHERE` 키워드와 미만(`<`) 비교 연산자를 사용할 수 있습니다:

```sql
SELECT *
FROM dogs
WHERE age < 3;
```

결과에는 나이가 3 미만인 개만 포함됩니다:

```sql
 id | name | age 
----+------+-----
  2 | Nora |   2
```

`age`의 `'Gino'`을 찾으려면 등호(`=`) 비교 연산자를 사용할 수 있습니다:

```sql
SELECT age
FROM dogs
WHERE name = 'Gino';
```

결과는 다음과 같습니다:

```sql
 age 
-----
  3
```

이것들은 SQL에서 데이터를 삽입하고 조회하는 일반적이고 간단한 방법이지만, 선택할 수 있는 다양한 옵션이 있고 그중 일부는 더 고급입니다. 앞으로 수업에서 배우게 될 것입니다.
