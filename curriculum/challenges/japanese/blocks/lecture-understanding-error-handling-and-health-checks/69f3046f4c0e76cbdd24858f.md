# --description--

Expressアプリがエラーメッセージなしに動かなくなったことはありませんか？デバッグとログ記録を使えば、裏側で何が起きているかを正確に把握できるので、問題をより早く見つけて修正できます。

このレッスンでは、Expressがどのようにデバッグとログ記録を扱っているか、そしてそれを使って開発の流れを改善する方法を探ります。

## Expressの組み込みデバッグ

Expressにはすでに`debug`というデバッグツールが組み込まれています。使うために何かをインストールする必要はありません。

デバッグを有効にするには、アプリを実行するときに`DEBUG`環境変数を設定するだけです。

例を見てみましょう：

```js
DEBUG=express:* node index.js
```

このコマンドは`express:*`名前空間のすべてのデバッグメッセージを有効にします。次のような出力が表示され始めます：

```js
express:router:route dispatching GET /
express:router:layer processing GET /
```

これらのメッセージはルートのディスパッチやミドルウェアの呼び出しなどを追跡するのに役立ちます。アプリのリクエストの流れを理解するのに最適です。

もっと特定の範囲に絞りたいですか？名前空間を絞ることもできます：

```js
DEBUG=express:router node index.js
```

これはExpressのルーターのロジックに関連するログだけを表示します。

Windowsの場合、PowerShellでは構文が少し異なります：

```js
$env:DEBUG = "express:*"; node index.js
```

CMDでは次のようになります：

```js
set DEBUG=express:* && node index.js
```

同じ考え方ですが、環境変数の設定方法が違うだけです。

次にExpressのログ記録を見てみましょう。

HTTPリクエストのログ記録には、多くの開発者が`morgan`のようなミドルウェアを使います。

使い方は次の通りです：

```bash
npm install morgan
```

そしてアプリに追加します：

```js
const express = require('express')
const morgan = require('morgan')
const app = express()

app.use(morgan('dev'))s
```

これで、すべての受信リクエストがメソッド、URL、ステータスコード、レスポンスタイムなどの詳細とともにターミナルに記録されます。

なぜこれが重要なのでしょうか？

デバッグは、Expressがリクエスト、ミドルウェア、ルーティングをどのように処理しているかを理解する手助けをし、本番ログを煩雑にしません。

ログ記録はリクエスト全体の記録を残し、バグのデバッグ、トラフィックの理解、問題の早期発見に不可欠です。

これらを組み合わせることで、開発プロセスにおける制御と明確さが得られます。
