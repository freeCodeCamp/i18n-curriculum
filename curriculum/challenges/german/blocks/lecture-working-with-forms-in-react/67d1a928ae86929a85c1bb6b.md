# --description--

Formulare sind grundlegend für jede Webanwendung, weil sie Ihnen erlauben, Benutzereingaben zu verarbeiten, Daten zu sammeln und Aktionen auszulösen.

In React werden Formulare mit State oder Refs verwaltet, was Ihnen die volle Kontrolle über ihr Verhalten und die Validierung gibt. Diese zwei Arten, Formulare zu verwalten, nennt man „controlled“ und „uncontrolled“ Input.

Schauen wir uns an, was controlled und uncontrolled Inputs sind.

Controlled Input ist die „React-typischste“ Methode, Formulareingaben zu handhaben. Bei controlled Inputs speichern Sie den Wert des Eingabefelds im State und aktualisieren ihn über `onChange`-Ereignisse. Das gibt Ihnen vollständige Kontrolle über die Formulardaten und ermöglicht sofortige Validierung und bedingtes Rendern.

Der Ablauf funktioniert so: React verwaltet den Formular-State mit dem `useState`-Hook, und Sie aktualisieren ihn bei jeder Änderung. Wenn ein Benutzer in ein Eingabefeld tippt, wird das `onChange`-Ereignis ausgelöst, aktualisiert den State, und React rendert die Komponente mit dem neuen Wert neu.

```jsx
import { useState } from "react";

function App() {
  const [name, setName] = useState("");

  const handleChange = (e) => {
    setName(e.target.value);
  };

  const handleSubmit = (e) => {
    e.preventDefault();
    console.log(name);
  };

  return (
    <>
      <form onSubmit={handleSubmit}>
        <label htmlFor="name">Your name</label> <br />
        <input value={name} id="name" onChange={handleChange} type="text" />
        <button type="submit">Submit</button>
      </form> 
    </>
  );
}

export default App;
```

Die Vorteile von controlled Inputs sind unter anderem:

- Sofortiger Zugriff auf die Formulardaten.

- Sie können eine sofortige Validierung implementieren.

- Sie können die Absenden-Schaltfläche bedingt deaktivieren.

- Sie können den Eingabewert programmatisch steuern.

Uncontrolled Inputs hingegen sieht man eher bei traditionellen HTML-Formularen. Statt die Eingaben über den `useState`-Hook zu verwalten, behalten uncontrolled Inputs in HTML ihren eigenen internen Zustand mithilfe des DOM.

Da das DOM die Eingabewerte steuert, müssen Sie die Werte der Eingabefelder mit ref abrufen. Dieser Ansatz benötigt weniger Code und ist performanter, weil refs kein erneutes Rendern von React auslösen.

Hier ist ein Beispiel für uncontrolled Inputs:

```jsx
import { useRef } from "react";

function App() {
  const nameRef = useRef();

  const handleSubmit = (e) => {
    e.preventDefault();
    console.log(nameRef.current.value);
  };

  return (
    <form onSubmit={handleSubmit}>
      <label htmlFor="name">Your</label>{" "}
      <input type="text" ref={nameRef} id="name" />
      <button type="submit">Submit</button>
    </form>
  );
}

export default App;
```

Ein sehr auffälliger Vorteil von uncontrolled Inputs ist, dass sie weniger Code erfordern. Sie sind auch performanter und wirken für React-Anfänger, die mit HTML vertraut sind, natürlicher.

Welche sollten Sie also verwenden: controlled oder uncontrolled Inputs?

Verwenden Sie controlled Inputs, wenn Sie dynamische Formularaktualisierungen, Echtzeit-Validierung oder die Synchronisation von Eingabewerten mit dem State benötigen. Sie bieten bessere Kontrolle, erfordern aber mehr Re-Renders.

Verwenden Sie uncontrolled Inputs, wenn Sie einfachere Formulare brauchen, Werte nur bei der Einsendung abrufen wollen oder mit nicht-React-Code arbeiten.

Unabhängig davon, ob Sie controlled oder uncontrolled Inputs verwenden, sollten Sie beim Erstellen von Formularen in React diese bewährten Vorgehensweisen beachten:

- Verhindern Sie immer die Standard-Formularabsendung.

- Stellen Sie sicher, dass Sie Eingaben vor der Einsendung validieren.

- Geben Sie den Nutzenden stets klare Rückmeldungen mit Ladezuständen, Validierungsfehlern oder anderen relevanten Zuständen.
