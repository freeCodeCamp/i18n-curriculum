## Trabajando con HTML, CSS y JavaScript

Mientras que HTML y CSS proporcionan la estructura del sitio web, JavaScript aporta interactividad a los sitios web, habilitando funciones complejas como el manejo de entrada de usuario, animación de elementos y construcción de aplicaciones web completas.

## Tipos de Datos en JavaScript

Los tipos de datos ayudan al programa a entender el tipo de dato con el que está trabajando, ya sea un número, texto o cualquier otra cosa.

- **Número**: Un número representa tanto enteros como valores de punto flotante. Ejemplos de enteros incluyen 7, 19 y 90.
- **Punto Flotante**: Un número de punto flotante es un número con un punto decimal. Ejemplos incluyen 3.14, 0.5 y 0.0001.
- **Cadena**: Una cadena es una secuencia de caracteres o texto, encerrada en comillas. `"Me gusta programar"` y `'JavaScript es divertido'` son ejemplos de cadenas.
- **Booleano**: Un booleano representa uno de dos valores posibles: `true` o `false`. Puedes usar un booleano para representar una condición, como `isLoggedIn = true`.
- **Indefinido y Nulo**: Un valor `undefined` es una variable que ha sido declarada pero no asignada un valor. Un valor `null` es un valor vacío, o una variable que intencionalmente ha sido asignada un valor de `null`.
- **Objeto**: Un objeto es una colección de pares de clave-valor. La clave es el nombre de la propiedad y el valor es el valor de la propiedad.

Aquí, el objeto `pet` tiene tres propiedades o claves: `name`, `age` y `type`. Los valores son `Fluffy`, `3` y `dog`, respectivamente.

```js
let pet = {
  name: "Fluffy",
  age: 3,
  type: "dog"
};
```

- **Símbolo**: El tipo de dato Symbol es un valor único e inmutable que se puede utilizar como un identificador para propiedades de objeto.

En el ejemplo a continuación, se crean dos símbolos con la misma descripción, pero no son iguales.

```js
const crypticKey1= Symbol("saltNpepper");
const crypticKey2= Symbol("saltNpepper");
console.log(crypticKey1 === crypticKey2); // false
```

- **BigInt**: Cuando el número es demasiado grande para el tipo de dato `Number`, puedes usar el tipo de dato BigInt para representar enteros de longitud arbitraria.

Al agregar una `n` al final del número, puedes crear un BigInt.

```js
const veryBigNumber = 1234567890123456789012345678901234567890n;
```

## Variables en JavaScript

- Las variables se pueden declarar usando la palabra clave `let`.

```js
let cityName;
```

- Para asignar un valor a una variable, puedes usar el operador de asignación `=`.

```js
cityName = "New York";
```

- Las variables declaradas usando `let` pueden ser reasignadas a un nuevo valor.

```js
cityName = "Los Angeles";
console.log(cityName); // Los Angeles
```

- Aparte de `let`, también puedes usar `const` para declarar una variable. Sin embargo, una variable `const` no puede ser reasignada a un nuevo valor.

```js
const cityName = "New York";
cityName = "Los Angeles"; // TypeError: Assignment to constant variable.
```

- Las variables declaradas usando `const` se utilizan para declarar constantes que no se permiten cambiar a lo largo del código, como `PI` o `MAX_SIZE`.

## Convenciones de Nomenclatura de Variables

- Los nombres de variables deben ser descriptivos y significativos.
- Los nombres de variables deben ser camelCase como `cityName`, `isLoggedIn` y `veryBigNumber`.
- Los nombres de variables no deben comenzar con un número. Deben comenzar con una letra, `_`, o `$`.
- Los nombres de variables no deben contener espacios ni caracteres especiales, excepto `_` y `$`.
- Los nombres de variables no deben ser palabras clave reservadas.
- Los nombres de variables son sensibles a mayúsculas. `age` y `Age` son variables diferentes.

## Cadenas de texto y su inmutabilidad en JavaScript

- Las cadenas de texto son secuencias de caracteres encerradas en comillas. Pueden ser creadas usando comillas simples o dobles.

```js
let correctWay = 'This is a string';
let alsoCorrect = "This is also a string";
```

- Las cadenas de texto son inmutables en JavaScript. Esto significa que una vez que se crea una cadena de texto, no puedes cambiar sus caracteres. Sin embargo, aún puedes reasignar cadenas de texto a un nuevo valor.

```js
let firstName = "John";
firstName = "Jane"; // Reassigning the string to a new value
```

## Concatenación de Cadenas de Texto en JavaScript

- La concatenación es el proceso de unir múltiples cadenas de texto o combinar cadenas de texto con variables que contienen texto. El operador `+` es uno de los métodos más simples y frecuentemente usados para concatenar cadenas de texto.

```js
let studentName = "Asad";
let studentAge = 25;
let studentInfo = studentName + " is " + studentAge + " years old.";
console.log(studentInfo); // Asad is 25 years old.
```

- Si necesitas añadir o agregar a una cadena de texto existente, puedes usar el operador `+=`. Esto es útil cuando deseas construir sobre una cadena de texto agregando más texto con el tiempo.

```js
let message = "Welcome to programming, ";
message += "Asad!";
console.log(message); // Welcome to programming, Asad!
```

- Otra manera de concatenar cadenas de texto es usar el método `concat()`. Este método une dos o más cadenas de texto.

```js
let firstName = "John";
let lastName = "Doe";
let fullName = firstName.concat(" ", lastName);
console.log(fullName); // John Doe
```

## Registro de Mensajes con `console.log()`

- El método `console.log()` se utiliza para registrar mensajes en la consola. Es una herramienta útil para depurar y probar tu código.

```js
console.log("Hello, World!");
// Output: Hello, World!
```

## Punto y Coma en JavaScript

- Los puntos y comas se utilizan principalmente para marcar el final de una sentencia. Esto ayuda al motor de JavaScript a entender la separación de instrucciones individuales, lo cual es crucial para una ejecución correcta.

```js
let message = "Hello, World!"; // first statement ends here
let number = 42; // second statement starts here
```

- Los puntos y comas ayudan a prevenir ambigüedades en la ejecución del código y aseguran que las sentencias se terminen correctamente.

## Comentarios en JavaScript

- Cualquier línea de código que esté comentada es ignorada por el motor de JavaScript. Los comentarios se utilizan para explicar código, hacer anotaciones o desactivar temporalmente código.
- Los comentarios de una sola línea se crean utilizando `//`.

```js
// This is a single-line comment and will be ignored by the JavaScript engine
```

- Los comentarios de múltiples líneas se crean utilizando `/*` para iniciar el comentario y `*/` para terminar el comentario.

```js
/*
This is a multi-line comment.
It can span multiple lines.
*/
```

## JavaScript como un Lenguaje de Tipo Dinámico

- JavaScript es un lenguaje de tipo dinámico, lo que significa que no tienes que especificar el tipo de dato de una variable cuando la declaras. El motor de JavaScript determina automáticamente el tipo de dato basado en el valor asignado a la variable.

```js
let error = 404; // JavaScript treats error as a number
error = "Not Found"; // JavaScript now treats error as a string
```

- Otros lenguajes, como Java, que no son de tipo dinámico resultarían en un error:

```js
int error = 404; // value must always be an integer
error = "Not Found"; // This would cause an error in Java
```

## Usando el Operador `typeof`

- El operador `typeof` se utiliza para comprobar el tipo de dato de una variable. Devuelve una cadena indicando el tipo de la variable.

```js
let age = 25;
console.log(typeof age); // "number"

let isLoggedIn = true;
console.log(typeof isLoggedIn); // "boolean"
```

- Sin embargo, hay una peculiaridad bien conocida en JavaScript cuando se trata de `null`. El operador `typeof` devuelve `"object"` para valores `null`.

```js
let user = null;
console.log(typeof user); // "object"
```
