## `Audio` 建構式和常用方法

- **定義**：`Audio` 建構式，像其他建構式一樣，是一個使用 `new` 關鍵字呼叫的特殊函式（程式）。它會傳回一個 `HTMLAudioElement`，你可以用來為使用者播放音訊，或附加到 DOM 讓使用者自行控制。當你呼叫建構式時，可以選用傳入一個 URL 作為（唯一）引數。這個 URL 應該指向你想播放的音訊檔案來源。或者，如果你需要動態變更來源，可以將 URL 指定給傳回的音訊元素的 `src` 屬性。
- **`play()` 方法**：此方法用於 `audio` 或 `video` 元素，以開始播放媒體。

:::interactive_editor

```html
<audio id="audio" controls>
  <source src="https://www.w3schools.com/html/horse.mp3" type="audio/mpeg">
</audio>

<button id="playBtn">Play Audio</button>

<script>
  const audio = document.getElementById("audio");
  const playBtn = document.getElementById("playBtn");

  playBtn.addEventListener("click", () => {
    audio.play();
  });
</script>
```

:::

- **`pause()` 方法**：此方法用於 `audio` 或 `video` 元素，以暫停媒體播放。它會維持目前播放位置，因此如果呼叫 `play()`，會從該位置繼續播放。

:::interactive_editor

```html
<audio id="myAudio" controls>
  <source src="https://www.w3schools.com/html/horse.mp3" type="audio/mpeg">
</audio>

<br>

<button id="pauseBtn">Pause Audio</button>

<script>
  document.addEventListener("DOMContentLoaded", () => {
    const audio = document.getElementById("myAudio");
    const pauseBtn = document.getElementById("pauseBtn");

    pauseBtn.addEventListener("click", () => {
      audio.pause();
    });
  });
</script>
```

:::

- **`addTextTrack()` 方法**：此方法允許你指定一個文字軌道與媒體元素相關聯——這對於為影片添加字幕特別有用。
- **`fastSeek()` 方法**：此方法允許你將播放位置移動到媒體中的特定時間。

`Audio()` 建構式除了有方法，也有屬性。這些屬性包括：

- `currentTime`：取得音訊目前的播放時間
- `loop`：讓音訊持續播放，當播放結束時自動重新開始
- `muted`：靜音媒體元素的音訊輸出，不論音量設定為何

## 不同的音訊與影片格式

- **MIME 類型**：MIME 類型，全名為多用途網際郵件擴充（Multipurpose Internet Mail Extensions），是一種以程式方式標示檔案類型的標準方式。MIME 類型可以告訴應用程式（例如你的瀏覽器）如何處理特定檔案。對於音訊和影片，MIME 類型表示它是一種可嵌入網頁的多媒體格式。
- **`source` 元素**：用於指定檔案類型和來源——可以透過多個 `source` 元素包含多種不同類型。瀏覽器會根據使用者當前環境判斷最佳格式。
- **MP3**：一種用於儲存音樂、音訊或聲音的數位檔案格式。它是聲音錄音的壓縮版本，使檔案大小更小，更容易儲存和分享。MP3 擁有最廣泛的瀏覽器支援，MIME 類型為 `audio/mpeg`。
- **MP4**：一種用於儲存影片和音訊的數位檔案格式。它作為一個容器，同時包含影片（影像）和聲音（音樂或語音）於一個檔案中。MP4 的 MIME 類型可為 `audio/mp4` 或 `video/mp4`，視是否為影片檔或僅音訊而定。
- 其他格式還有與 Windows Media Player 相關的 `WMV`、`OGG`、`MKV`、`AVI` 等。

## 編解碼器（codecs）

- **定義**：編解碼器（codec，encoder/decoder 的縮寫）是一種演算法或軟體，可以在類比和數位格式之間轉換音訊和影片。編解碼器可以作為 MIME 類型的一部分指定。定義編解碼器的基本語法是在媒體類型後加分號，接著是 `codecs=` 和編解碼器名稱。

## HTMLMediaElement API

- **定義**：`HTMLMediaElement` API 用於控制頁面上的音訊和影片元素行為。它擴充了基底的 `HTMLElement` 介面，因此你可以存取基底屬性、方法，以及像 `waiting`、`ended`、`canplay`、`canplaythrough` 等有用事件。方法範例包括 `play()`、`fastSeek()`、`pause()`，以及用於檢查瀏覽器是否可能播放音訊檔的 `canPlayType()`。

## Media Capture and Streams API

- **定義**：Media Capture and Streams API，或稱 MediaStream API，用於從你的裝置擷取音訊和影片。要使用此 API，你需要建立 `MediaStream` 物件。雖然可以用建構式建立，但它不會綁定使用者的硬體。相反地，`global` 的 `navigator` 物件的 `mediaDevices` 屬性有一個 `getUserMedia()` 方法供你使用。

```js
window.navigator.mediaDevices.getUserMedia({
  audio: true,
  video: {
    width: {
      min: 1280,
      ideal: 1920,
      max: 3840
    },
    height: {
      min: 720,
      ideal: 1080,
      max: 2160
    }
  }
});
```

## 螢幕擷取 API

- **定義**：螢幕擷取 API 用於記錄使用者的螢幕。此 API 是透過呼叫 `mediaDevices` 物件的 `getDisplayMedia()` 方法並使用傳回的媒體資料流來暴露。

## MediaStream 記錄 API

- **定義**：MediaStream Recording API 與 MediaStreams API 協同運作，讓你能夠錄製 MediaStream（甚至直接錄製 `HTMLMediaElement`）。

## 媒體原始碼擴充套件 API

- **主題**：Media Source Extensions API 讓你能夠直接將使用者的網路攝影機串流傳遞給具有 `srcObject` 屬性的影片元素。

## Web Audio API

- **定義**：Web Audio API 是驅動網頁上所有可聽見內容的技術。此 API 包含重要的物件，如 `AudioBuffer`（表述專門包含音訊資料的緩衝區）或用於表述音訊處理圖的 `AudioContext`。
