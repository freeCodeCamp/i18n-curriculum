# --interactive--

在 JavaScript 中，函式（程式）和物件方法都是封裝可重複使用程式碼的方式，但它們在定義、使用及運作的背景關係上有一些關鍵差異。理解這些差異對於撰寫有效且有組織的 JavaScript 程式碼至關重要。

正如你在先前的模組中學到的，函式（程式）是可重複使用的程式碼區塊，執行特定任務：

:::interactive_editor

```js
function greet(name) {
    return "Hello, " + name + "!";
}
console.log(greet("Alice")); // "Hello, Alice!"
```

:::

物件的方法則是與物件相關的函式。它們被定義為物件的屬性，並且可以存取及操作該物件的資料。以下是一個具有方法的物件範例：

:::interactive_editor

```js
const person = {
    name: "Bob",
    age: 30,
    sayHello: function() {
        return "Hello, my name is " + this.name;
    }
};

console.log(person.sayHello()); // "Hello, my name is Bob"
```

:::

在這個範例中，`sayHello` 是 `person` 物件的一個方法。`this` 關鍵字允許 `sayHello` 方法存取名為 `person` 的物件屬性。你將在未來的課程中學習更多關於 `this` 關鍵字的內容。

函式與方法之間的差異在於它們的呼叫方式。函式是透過名稱來呼叫，而方法則是使用點記法在其所屬的物件上呼叫。例如，我們以 `greet("Alice")` 呼叫 `greet` 函式，但以 `person.sayHello()` 呼叫 `sayHello` 方法。

另一個重要的差異是它們運作的背景關係。一般函式（程式）有自己的作用域，但它們沒有內建的引用任何特定物件。方法（行為）則是綁定到它們的物件，並且可以使用 `this` 關鍵字存取該物件的屬性和其他方法（行為）。

一個重點是，方法有助於將程式碼組織成邏輯物件，而函式則用於更通用且可重複使用的程式碼。

理解函式與物件方法之間的差異對你在 JavaScript 旅程中的進展非常重要。雖然它們乍看之下可能相似，但識別何時使用各自將幫助你撰寫更有組織、高效且更好的程式碼。
