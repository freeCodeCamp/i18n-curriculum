# --description--

RSA 暗号化は次の手順に基づいています：

異なる2つの素数 `p` と `q` を生成します。
`n=p*q` と `φ=(p-1)(q-1)` を計算します。
整数 `e`、`1 < e < φ` を見つけます。ただし `gcd(e,φ) = 1` となるようにします。

このシステムのメッセージは区間 `[0,n-1]` 内の数値です。
暗号化するテキストは何らかの方法でメッセージ（区間 `[0,n-1]` 内の数値）に変換されます。
テキストを暗号化するには、各メッセージについて `m`、c=m<sup>e</sup> mod n を計算します。

テキストを復号するには次の手順が必要です：`d` を計算し、`ed=1 mod φ` となるようにし、次に各暗号化されたメッセージについて `c`、m=c<sup>d</sup> mod n を計算します。

m<sup>e</sup> mod n = m となる `e` と `m` の値が存在します。
m<sup>e</sup> mod n=m となるメッセージ `m` を非隠蔽メッセージと呼びます。

`e` を選ぶ際の問題は、非隠蔽メッセージが多すぎてはいけないことです。
例えば、`p=19` と `q=37` とします。
すると `n=19*37=703` と `φ=18*36=648` となります。
`e=181` を選ぶと、`gcd(181,648)=1` にもかかわらず、すべての可能なメッセージ m `(0≤m≤n-1)` は m<sup>e</sup> mod n を計算すると非隠蔽になります。
`e` の有効な選択肢には必ずいくつかの非隠蔽メッセージが存在します。
非隠蔽メッセージの数が最小になることが重要です。

与えられた `p` と `q` に対して、この `e` の値で非隠蔽メッセージの数が最小になるような `1 < e < φ(p,q)`、`gcd(e,φ)=1`、`e` のすべての値の合計を求めてください。

# --solutions--

```js
function gcd(a, b) {
    if (b)
        return gcd(b, a % b);
    else
        return a;
}

function RSAEncryption(p, q) {
    let phi = (p - 1) * (q - 1);

    let best = Number.MAX_SAFE_INTEGER;
    let sum = 0;

    for (let e = 0; e < phi; ++e) {
        if (!(gcd(e, phi) == 1))
            continue;

        let msg = (gcd(p - 1, e - 1) + 1) * (gcd(q - 1, e - 1) + 1);

        if (best == msg) {
            sum += e;
        } else if (best > msg) {
            best = msg;
            sum = e;
        }
    }

    return sum;
}
```
