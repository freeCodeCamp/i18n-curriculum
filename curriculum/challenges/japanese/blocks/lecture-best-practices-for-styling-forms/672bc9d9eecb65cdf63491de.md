# --interactive--

検索入力、チェックボックス、ラジオボタンのスタイリング問題に対処するために `appearance: none` を使うべきタイミングを学びましょう。

ブラウザは多くの要素にデフォルトのスタイリングを適用します。入力要素の場合、CSSでスタイリングできる範囲がかなり制限されていると感じることがあります。そこで、`appearance: none` を使ってデフォルト要素の一部を隠し、自分でカスタムのスタイルを作ることができます。例えば、チェックボックス入力のデフォルトのチェックボックスを隠して、状態を示す緑のチェックマークや赤いバツ印のようなカスタムインジケーターを使うことができます。検索入力の場合、WebKitベースのブラウザはデフォルトで検索アイコンとキャンセルボタンを表示します。これらを隠すことで、すべてのブラウザで表示される独自のインジケーターを作成できます。

```html

    <input class="checkbox" type="checkbox" /> Agree
```

```css
.checkbox {
  appearance: none;
  width: 18px;
  height: 18px;
  border: 2px solid #ccc;
  border-radius: 4px;
  display: inline-block;
  position: relative;
  cursor: pointer;
  transition: all 0.25s ease;
  vertical-align: middle; 
}

.checkbox:hover {
  border-color: #888;
}

.checkbox:checked {
  background-color: #4caf50;
  border-color: #4caf50;
}

.checkbox:checked::after {
  content: "";
  position: absolute;
  left: 4px;
  top: 0px;
  width: 5px;
  height: 10px;
  border: solid white;
  border-width: 0 2px 2px 0;
  transform: rotate(45deg);
}

.checkbox:focus {
  outline: 2px solid #90caf9;
  outline-offset: 2px;
}
```

:::

WebKitはウェブブラウザがウェブサイトを表示するのを助けるソフトウェアエンジンです。SafariのようなブラウザはWebKitを使ってウェブページが正しく見え、動作するようにしています。この `appearance: none` CSSプロパティはスタイリングを完全にコントロールできますが、注意すべき点もあります。入力要素のデフォルトのインタラクティブなコンポーネントには、フォーカスやエラーのインジケーターなどの機能が含まれており、これらが失われないように注意が必要です。

プラットフォーム間で一貫したスタイリングを作るのにこのプロパティを使うのはとても良い理由です。また、モバイルデバイスでタップターゲットを十分な大きさにしたり、チェックボックスの色のコントラストを確保したりするためにも使えます。
