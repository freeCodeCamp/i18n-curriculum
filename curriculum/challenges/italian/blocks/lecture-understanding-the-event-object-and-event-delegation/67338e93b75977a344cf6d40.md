# --description--

Il bubbling degli eventi, o propagazione, si riferisce a come un evento "risale" agli oggetti genitori quando viene attivato. Per esempio, considera questo codice:

```html
<p>
  <span>Click me~!</span>
</p>
```

L'elemento `p` qui sarebbe considerato il genitore dell'elemento `span`. 

Quando clicchi sull'elemento `span`, come indicato, l'elemento `span` diventa l'obiettivo di un evento `click`. Tuttavia, quell'evento risale anche al genitore – l'elemento `p` può ricevere e gestire quell'evento secondo necessità.

Ma cosa significa realmente? Beh, potresti allegare un listener per l'evento all'elemento `p`:

```js
const p = document.querySelector("p");
p.addEventListener("click", (event) => {
  console.log(event.target);
});
```

Poi, quando clicchi sull'elemento `span`, vedrai il testo `Click me~!` stampato nella console.

L'evento si propaga all'elemento genitore `p`, che lo gestisce in un listener per mostrare l'obiettivo dell'evento. 

Nota come l'obiettivo sia ancora l'elemento `span`. Questo perché l'elemento `span` ha ricevuto il clic iniziale.

Per essere sicuri di come funziona, espandiamo il nostro codice:

```js
const p = document.querySelector("p");
const span = document.querySelector("span");
p.addEventListener("click", (event) => {
  console.log("P listener: ");
  console.log(event.target);
});
span.addEventListener("click", (event) => {
  console.log("Span listener: ");
  console.log(event.target);
});
```

Per darti un'idea di come l'evento risale, ecco cosa vedrai nella console dopo aver cliccato sull'elemento `span`:

```html
"Span listener: "
<span>Click me~!</span>
"P listener: "
<span>Click me~!</span>
```

Ora vediamo cosa succede quando impedisci la propagazione di un evento con `stopPropagation()`. Lo chiameremo nel listener dell'evento di `span`:

```js
const p = document.querySelector("p");
const span = document.querySelector("span");
p.addEventListener("click", (event) => {
  console.log("P listener: ");
  console.log(event.target);
});
span.addEventListener("click", (event) => {
  console.log("Span listener: ");
  console.log(event.target);
  event.stopPropagation();
});
```

E poi clicchiamo di nuovo su `span`:

```html
"Span listener: "
<span>Click me~!</span>
```
 
Questa volta, non vediamo scattare il listener `p`. L'evento non viene mai attivato per l'elemento `p`, perché abbiamo detto di fermare la propagazione mentre veniva processato per l'elemento figlio `span`.

La delega degli eventi può essere vista come l'opposto. È il processo di prendere un evento catturato e delegarlo a un altro elemento. 

Tornando al nostro codice, aggiorniamolo così che cliccare su un elemento `span` lo cambi in rosso:

```js
const p = document.querySelector("p");
const span = document.querySelector("span");
p.addEventListener("click", (event) => {});
span.addEventListener("click", (event) => {
  event.target.style.color = "red";
});
```

Ma cosa succede se hai venti elementi `span`? O magari usi JavaScript per creare altri elementi `span` al volo?

Invece di dover allegare un listener a ogni singolo elemento `span`, puoi usare il listener sull'elemento `p` per tutti loro. In altre parole, puoi delegare la gestione dei clic `span` all'elemento genitore `p`.

Il nostro codice potrebbe ora assomigliare a questo:

```js
const p = document.querySelector("p");
p.addEventListener("click", (event) => {
  event.target.style.color = "red";
});
```

Nota come non abbiamo più alcun listener allegato all'elemento `span`. Hai delegato correttamente la gestione degli eventi all'elemento `p`. Ma funziona?

Generiamo qualche elemento `span` in più e vediamo:

```html
<p>
  <span>Click me~!</span>
  <span>Click me~!</span>
  <span>Click me~!</span>
  <span>Click me~!</span>
</p>
```

Ora, ogni volta che clicchiamo su un `span`, il testo di quell'elemento diventerà rosso. 

E proprio così, con un singolo listener abbiamo permesso correttamente a un evento `click` di risalire dagli elementi `span` al genitore `p`, e delegato la logica per quell'evento `click` all'elemento `p`.

La propagazione e la delega degli eventi possono essere argomenti complessi, specialmente quando si lavora con elementi molto annidati come le tabelle. Ti incoraggiamo a esplorare ulteriormente e sperimentare con parte del codice che abbiamo scritto qui.
