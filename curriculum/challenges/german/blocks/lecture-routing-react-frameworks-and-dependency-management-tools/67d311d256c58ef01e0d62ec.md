# --description--

Bis zu diesem Punkt haben Sie React verwendet, um Benutzeroberflächen zu erstellen. Wenn Sie zusätzliche Funktionen wie Routing benötigten, mussten Sie eine Drittanbieter-Bibliothek wie React Router importieren, um zwischen den verschiedenen Ansichten wechseln zu können.

Aber was passiert, wenn Sie eine Full-Stack-Webanwendung erstellen müssen? Nun, Sie könnten React für das Frontend verwenden und Node und Express für die Backend-Logik, wenn Sie nur bei JavaScript bleiben möchten. Oder Sie könnten andere Sprachen wie Go, Python oder Java für Ihr Backend verwenden.

Obwohl all diese Optionen möglich sind, gibt es Situationen, in denen Sie stattdessen ein React-Framework verwenden möchten. React-Frameworks bieten Funktionen wie Routing, Bildoptimierungen, Datenabruf, Authentifizierung und mehr. Das bedeutet, dass Sie für bestimmte Anwendungsfälle möglicherweise keine separaten Frontend- und Backend-Anwendungen einrichten müssen.

Werfen wir einen genaueren Blick auf Next.js, ein beliebtes React-Framework. Eine der Hauptfunktionen von Next.js ist der dateisystembasierte Router. Dieses Routing-System unterstützt dynamische Routen, parallele Routen, Route-Handler, Umleitungen, Internationalisierung und mehr.

Hier ist ein Beispiel für das Erstellen eines benutzerdefinierten Request-Handlers:

```js
export async function GET() {
  const res = await fetch("https://example-api.com");
  const data = await res.json();

  return Response.json({ data });
}
```

Sie können Route-Handler wie GET- oder POST-Anfragen in einer Datei namens `route.js` im Verzeichnis `app/api` definieren.

Eine weitere Funktion von Next.js sind die automatischen Bild- und Schriftartoptimierungen.

Hier ist ein Beispiel für die Arbeit mit der `Image`-Komponente in einer `page.js`-Datei:

```jsx
import Image from "next/image";

export default function Page() {
  return (
    <Image src="link-to-image-goes-here" alt="descriptive-title-goes-here" />
  );
}
```

Die `Image`-Komponente erweitert das native HTML-`img`-Element und ermöglicht schnellere Seitenladezeiten und Größenoptimierungen. Das bedeutet, dass Bilder nur geladen werden, wenn sie in den Viewport gelangen, und die `Image`-Komponente automatisch korrekt dimensionierte Bilder für jedes Gerät bereitstellt.

Dies sind nur einige Funktionen von Next.js, es gibt viele weitere Funktionen, mit denen Sie robuste Full-Stack-Webanwendungen erstellen können. Es gibt auch andere React-Frameworks wie Remix, die dieselbe Möglichkeit bieten, moderne Full-Stack-Webanwendungen zu erstellen.

Obwohl sich die Landschaft der JavaScript-Bibliotheken und -Frameworks ständig ändert, ist es wichtig, die verfügbaren Optionen zu kennen und die Vor- und Nachteile jedes Frameworks zu lernen.
