# --interactive--

在先前的模組中，你已經習慣使用像這樣的字串字面值：

```js
const greeting = "Hello, World!";
```

但是 JavaScript 也有字串物件。字串物件和字串原始值都用來控制代碼文字，但它們在底層的運作方式不同。字串物件是使用字串建構式函式建立的，它會將原始值包裝在物件中。以下是你如何建立字串物件：

:::interactive_editor

```js
const greetingObject = new String("Hello, World!");

console.log(typeof greetingObject); // "object"
```

:::

當我們使用 `typeof` 運算子時，我們可以看到結果是物件型別，而不是字串型別。你可能一直在想的一件事是，如何在字串原始值上使用像 `.length` 這樣的屬性。

當你在字串原始值上使用 `length` 屬性時，JavaScript 會暫時將字串原始值包裝在字串物件中，以執行該操作。這就是為什麼你可以使用 `length` 屬性以及像 `repeat()`、`concat()` 和 `slice()` 等不同的方法。這類方法和屬性稱為實體方法、實體屬性和靜態方法。你將在未來的模組中詳細學習這些運作方式。

字串物件和字串原始值之間的一個主要差異在於它們與記憶體和效率的關係。字串原始值通常比字串物件更高效且更快。雖然你主要會在你的程式碼中使用字串原始值，但了解字串物件的運作方式仍然很有用。
