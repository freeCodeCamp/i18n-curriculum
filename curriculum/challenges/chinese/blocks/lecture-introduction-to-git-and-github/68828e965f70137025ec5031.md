# --description--

如果你曾经处理过某些东西，比如论文，在做了重大修改后储存了多个副本，并将文件命名为 `Thesis-Draft`、`Thesis-Final`、`Thesis-Final-For-Real` 和 `Thesis-Absolute-Final-DO-NOT-CHANGE`，那么你使用的是一种非常原始的版本控件方法。但如果我告诉你有一种更好的方法呢？

你可能听说过 Git、SVN 或 Mercurial 这样的工具。它们是版本控制系统，是每个开发者工作流程中的关键组件。版本控制系统允许你跟踪和管理你项目中的更改。为了这些课程的目的，我们将重点关注 Git。

Git 提供强大的工具，允许你管理一个项目的编辑历史。你将在接下来的课程中了解更多关于如何使用 Git 及其提供的特色。现在，你真正需要知道的是：

Git 允许你执行诸如创建提交的操作，提交是你代码库特定状态的屏幕截图。然后你可以比较提交以查看你所做的更改，回滚包含不当更改的提交，等等。

你也可以创建“分支”，你可以将其视为代码中的不同路径。虽然分支可能是你走的路，但提交是沿途的地标。分支允许你独立开发新特色，这样如果你的工作出现问题，你可以删除该分支而不会影响核心代码库。

这些特色使版本控制成为与其他开发者协作的关键组件。当有多人在同一个项目中工作时，分支允许他们各自在自己的代码上工作，而不会覆盖彼此的更改。

你经常会将像 Git 这样的版本控制系统与像 GitHub 或 Gitlab 这样的版本控制提供商配对，后者为你提供额外的协作工具以及将你的项目备份到云端的能力。

但即使没有基于云的存储方案，版本控制系统也能为你的工作流程带来革命性的变化。不再有 `THIS-IS-THE-FINAL-COPY-FOREVER` 文件。

关于使用这些系统有很多很酷的信息要介绍，所以让我们在接下来的几节课中深入探讨！
