# --description--

Man mano che la tua app cresce, gestire il flusso dei dati tra i componenti può diventare complesso.

All'inizio, il hook `useState` di React potrebbe essere sufficiente, ma aggiungendo funzionalità potresti incontrare problemi come:

- Passare props attraverso componenti che non ne hanno bisogno, noto anche come prop drilling
- Mantenere i dati sincronizzati in diverse parti della tua app
- Gestire aggiornamenti complessi che coinvolgono più componenti contemporaneamente

Queste e altre sfide possono emergere, portando a un codice più difficile da mantenere, eseguire il debug e testare. È qui che entrano in gioco le librerie per la gestione dello stato: offrono un luogo centralizzato dove i componenti possono ottenere o aggiornare i dati di cui hanno bisogno.

Diamo un’occhiata ad alcune opzioni diverse per la gestione dello stato e a quando usarle.

La Context API è un gestore di stato integrato in React che ti permette di condividere lo stato tra componenti senza usare una libreria di terze parti. È un miglioramento consolidato rispetto al hook `useState`, quindi è perfetta per casi come il cambio tema o lo stato di autenticazione dell’utente.

Tuttavia, la Context API non gestisce bene aggiornamenti frequenti e può causare render inutili, rendendola meno adatta per esigenze di stato complesse in applicazioni come eCommerce e piattaforme social.

Ecco un componente contatore che dimostra l’uso base della Context API:

```jsx
import { useState, createContext } from 'react';

const CounterContext = createContext();

const CounterProvider = ({ children }) => {
  const [count, setCount] = useState(0);

  return (
    <CounterContext.Provider value={{ count, setCount }}>
      {children}
    </CounterContext.Provider>
  );
};

export { CounterContext, CounterProvider };
```

Questo codice crea un contesto e un provider per condividere uno stato `count` in tutta l’applicazione.

`CounterProvider` usa il hook `useState` per inizializzare e gestire lo stato `count` e la sua funzione setter. Entrambi vengono poi passati ai componenti figli tramite il `Provider`.

Quindi, quando avvolgi tutta la tua app con il `CounterProvider`, lo stato `count` è disponibile ovunque nella tua applicazione.

Ecco come puoi avvolgere `CounterProvider` attorno alla tua applicazione:

```jsx
import { CounterProvider } from './context/CounterContext';

function App() {
  return (
    <CounterProvider>
        {/* App components */}
    </CounterProvider>
  );
}

export default App;
```

Ecco come puoi usare lo stato `count`:

```jsx
import React, { useContext } from 'react';
import { CounterContext } from '../context/CounterContext';

const Counter = () => {
  const { count, setCount } = useContext(CounterContext);

  return (
    <>
      <div style={{ textAlign: 'center' }}>
        <h1>Context API Counter</h1>
        <button style={{ marginRight: '5px' }} onClick={() => setCount(count - 1)}>
          Decrease
        </button>
        <span>{count}</span>
        <button style={{ marginLeft: '5px' }} onClick={() => setCount(count + 1)}>
          Increase
        </button>
      </div>
    </>
  );
};

export default Counter;
```

Come vedi, lo stato `count` e la sua funzione setter, `setCount`, sono inizializzati tramite la funzione `useContext`.

Lo stato `count` corrente viene poi mostrato, e `setCount` viene usato per aumentare e diminuire lo stato `count` quando l’utente clicca rispettivamente i pulsanti di decremento e incremento.

Un’altra libreria popolare per la gestione dello stato è Redux, una delle più usate con React. Esiste da molto tempo ed è ideale per applicazioni più grandi come eCommerce, piattaforme social, forum e così via.

Redux gestisce lo stato fornendo un archivio centrale e un controllo rigoroso sugli aggiornamenti dello stato. Usa un pattern prevedibile con azioni, reducer e middleware.

Le azioni sono payload di informazioni che inviano dati dalla tua applicazione all’archivio Redux, spesso attivate da interazioni dell’utente.

I reducer sono funzioni che specificano come lo stato deve cambiare in risposta a quelle azioni, assicurando che lo stato venga aggiornato in modo immutabile.

Il middleware, invece, agisce come ponte tra la dispatch delle azioni e il reducer, permettendoti di estendere le funzionalità di Redux (per esempio, logging, gestione di operazioni asincrone) senza modificare il flusso principale.

Il problema più comune con Redux è tutto il codice boilerplate necessario per iniziare. Per questo, il team di Redux ha introdotto "Redux Toolkit" e "RTK Query", che semplificano molto il processo di configurazione.

Di solito definisci sia azioni che reducer in un unico file usando la funzione `createSlice()`. È comune chiamare il file in modo che finisca con la parola `Slice`, per esempio `productSlice`, `userSlice`, `counterSlice` e così via.

Ecco un file `counterSlice` per mostrarti le basi:

```jsx
import { createSlice } from '@reduxjs/toolkit';

const counterSlice = createSlice({
  name: 'counter',

  initialState: { count: 0 },

  reducers: {
    increment: (state) => {
      state.count += 1;
    },
    decrement: (state) => {
      state.count -= 1;
    },
  },
});

export const { increment, decrement } = counterSlice.actions;

export default counterSlice.reducer;
```

Da qui, devi poi avvolgere tutta l’app con il `Provider`, selezionare una parte dello stato dal slice con `useSelector()` e poi usare `useDispatch()` per rendere attivo lo stato.

Un’altra opzione da considerare è Zustand.

Zustand è una libreria leggera per la gestione dello stato con un’API semplice. Si basa su hook, quindi c’è meno codice boilerplate rispetto a Redux, rendendo più facile e veloce la configurazione.

Zustand è ideale per applicazioni di piccole e medie dimensioni. Funziona usando un hook `useStore` per consultare lo stato direttamente in componenti e pagine. Questo ti permette di modificare e consultare i dati senza bisogno di azioni, reducer o provider.

Ecco un `useCounterStore` che implementa un’altra funzionalità di contatore:

```jsx
import { create } from 'zustand';

const useCounterStore = create((set) => ({
  count: 0,
  increment: () => set((state) => ({ count: state.count + 1 })),
  decrement: () => set((state) => ({ count: state.count - 1 })),
}));

export default useCounterStore;
```

Ecco come inizializzare e usare gli stati nella tua app:

```jsx
// Import the useCounterStore (it's just a hook)
import useCounterStore from '../useCounterStore';

const Counter = () => {
  // Initialize the states with the useCounterStore hook
  const { count, increment, decrement } = useCounterStore();

  return (
    <>
      <div style={{ textAlign: 'center' }}>
        <h1>Zustand Counter</h1>
        <button style={{ marginRight: '5px' }} onClick={() => decrement()}>
          Decrease
        </button>
        <span>{count}</span>
        <button style={{ marginLeft: '5px' }} onClick={() => increment()}>
          Increase
        </button>
      </div>
    </>
  );
};

export default Counter;
```

Anche se l’ecosistema front-end è in continua evoluzione e nuove librerie per la gestione dello stato emergono regolarmente, quelle di cui abbiamo parlato sono ampiamente usate nel settore.
