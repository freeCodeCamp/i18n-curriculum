# --description--

Quando você constrói projetos em JavaScript, frequentemente precisa de ferramentas extras para ajudar a realizar as tarefas. Mesmo aplicativos simples podem precisar de recursos como salvar dados, fazer login, roteamento para navegar entre páginas e assim por diante.

Em vez de escrever todo esse código do zero, você pode usar pacotes open-source que outros desenvolvedores já criaram e compartilharam.

É aqui que o npm entra. O npm é um enorme registro online de software open-source. Pense nele como uma loja de aplicativos para código — você pode procurar o que precisa, instalar e começar a usar no seu projeto.

Muitas pessoas pensam que npm significa "Node Package Manager", mas esse não é seu significado oficial. Em documentações antigas do npm, ele foi descrito como uma "abreviação bacrônica recursiva para 'npm is not an acronym'." Em outras palavras, npm não significa oficialmente nada.

Hoje, npm se refere a três partes relacionadas:

* O site npmjs.com, onde você pode pesquisar pacotes no registro.
    
* O registro npm, que é um grande banco de dados público que armazena todos os pacotes.
    
* A ferramenta CLI do npm que é instalada com o Node.js e permite que você instale e gerencie pacotes.
    

Por exemplo, se você precisa gerenciar estado, manipular datas, enviar e-mails ou fazer requisições HTTP no seu projeto, provavelmente já existe um pacote npm que faz isso. Com um simples comando `npm install`, você pode começar a usar esse software pronto em vez de fazer todo o trabalho você mesmo.

O npm também ajuda você a gerenciar as dependências do seu projeto. Isso significa que ele acompanha os pacotes que você usa em seus projetos e a versão de cada um desses pacotes. Ele faz isso com os arquivos `package.json` e `package-lock.json`, que você aprenderá nas próximas lições.

Yarn, PNPM e Bun são algumas alternativas ao npm que se tornaram populares ao longo do tempo. Embora cada uma delas ofereça benefícios em áreas específicas como velocidade de instalação e eficiência de armazenamento, o npm ainda é o mais utilizado.

Nas próximas aulas, você aprenderá vários comandos npm, como instalar pacotes, sobre os arquivos `package.json` e `package-lock.json` e até como publicar seu próprio pacote no registro npm.
