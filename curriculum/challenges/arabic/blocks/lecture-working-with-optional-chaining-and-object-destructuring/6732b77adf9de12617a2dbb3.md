# --interactive--

التفكيك في الكائنات هو ميزة قوية في JavaScript تتيح لك استخراج القيم من الكائنات وتعيينها إلى متغيرات بطريقة أكثر إيجازًا وقابلية للقراءة.

إنها جزء من مواصفة ES6 (ECMAScript 2015) وأصبحت أداة أساسية للعديد من مطوّري JavaScript.

يمكن للتفكيك تبسيط شفرتك، خاصة عند العمل مع كائنات معقدة أو عندما تحتاج إلى استخراج قيم متعددة دفعة واحدة.

في جوهره، التفكيك في الكائنات يعني تفكيك القيم من الكائنات إلى متغيرات منفصلة. بدلًا من الوصول إلى خصائص الكائن واحدة تلو الأخرى، يمكنك استخراج خصائص متعددة في بيان واحد. هذا يجعل شفرتك أنظف وأكثر كفاءة.

لنبدأ بمثال لتوضيح كيفية عمل التفكيك في الكائنات:

:::interactive_editor

```js
const person = { name: "Alice", age: 30, city: "New York" };

const { name, age } = person;

console.log(name); // Alice
console.log(age);  // 30
```

:::

في هذا المثال، نستخرج الخاصيتين `name` و `age` من كائن `person` ونعينهما إلى متغيرات تحمل نفس الأسماء.

من الجوانب القوية في التفكيك في الكائنات أنه يمكنك تعيين القيم المستخرجة إلى متغيرات بأسماء مختلفة. هذا مفيد خصوصًا عند العمل مع كائنات تحتوي على أسماء خصائص قد تتعارض مع متغيرات موجودة أو عندما تريد استخدام اسم مختلف:

:::interactive_editor

```js
let person = { name: "Alice", age: 30, city: "New York" };

let { name: personName, age: personAge } = person;

console.log(personName); // Alice
console.log(personAge); //  30
```

:::

في هذه الحالة، نستخرج الخاصية `name` ونعينها إلى متغير يسمى `personName`، ونفعل الشيء نفسه مع `age` و `personAge`.

يسمح التفكيك في الكائنات أيضًا بتعيين قيم افتراضية. إذا لم توجد خاصية في الكائن الذي تفككه، يمكنك تحديد قيمة نظام بديل:

:::interactive_editor

```js
let person = { name: "Alice", age: 30, city: "New York" };
let { name, age, country = "Unknown" } = person;

console.log(country); // Unknown
```

:::

هنا، بما أن `country` غير موجود في كائن `person`، فإنه يحصل على القيمة الافتراضية `Unknown`.

حالة شائعة أخرى هي التفكيك في الكائنات المسكنة. يمكنك تفكيك خصائص متداخلة داخل كائنات أخرى باستخدام مجموعة أخرى من الأقواس:

:::interactive_editor

```js
const recipe = {
  name: "Chocolate Cake",
  ingredients: {
    flour: "2 cups",
    sugar: "1 cup"
  }
};

// Extract `flour` from `ingredients`
const { ingredients: { flour } } = recipe;

console.log(flour); // "2 cups"
```

:::

هذا يعادل الوصول إلى الخاصية مباشرة:

```js
const flour = recipe.ingredients.flour;
console.log(flour); // "2 cups"
```

الآن، لنتحدث عن الصيغة المختصرة في التفكيك في الكائنات. عند إنشاء كائنات، خاصة عندما تتطابق أسماء الخصائص مع أسماء المتغيرات، يمكنك استخدام صيغة مختصرة:

:::interactive_editor

```js
let name = "Bob";
let age = 25;

let person = { name, age };

console.log(person); // { name: "Bob", age: 25 }
```

:::

الشفرة أعلاه تأخذ الخصائص التي تحمل نفس أسماء متغيراتنا وتعيّنها إلى قيم تلك المتغيرات.

هذه الصيغة المختصرة مفيدة بشكل خاص عند إرجاع كائنات من دوال أو إنشاء كائنات تحتوي على خصائص متعددة:

:::interactive_editor

```js
function createPerson(name, age) {
  return { name, age };
}

let person = createPerson("Charlie", 35);
console.log(person); // { name: "Charlie", age: 35 }
```

:::

التفكيك في الكائنات والصيغة المختصرة للكائنات ميزتان قويتان يمكن أن تجعلا شفرتك أكثر إيجازًا وأسهل في القراءة.

هما مفيدان خصوصًا عند العمل مع هياكل بيانات معقدة، أو عندما تحتاج إلى تمرير معلمات متعددة إلى دوال.

كلما واصلت العمل مع JavaScript، ستجد العديد من الحالات التي يمكن لهذه التقنيات تبسيط شفرتك وجعلها أكثر تعبيرًا.
