# --description--

Hasta ahora, has visto cómo JSX difiere de HTML de una manera clave con el uso de `className` vs. `class` para definir clases HTML.

Otra forma importante en la que JSX difiere de HTML está en la idea de la etiqueta de auto-cierre.

En HTML, casi todas las etiquetas tienen tanto una etiqueta de apertura como de cierre: `<div></div>`; la etiqueta de cierre siempre tiene una barra diagonal antes del nombre de la etiqueta que estás cerrando. Sin embargo, hay casos especiales en HTML llamados <dfn>elementos vacíos</dfn>, o elementos que no requieren tanto una etiqueta de apertura como de cierre antes de que pueda comenzar otro elemento.

Por ejemplo, la etiqueta de salto de línea puede escribirse como `<br>` o como `<br />`, pero nunca debe escribirse como `<br></br>`, ya que no contiene ningún contenido.

En JSX, las reglas son un poco diferentes. Cualquier elemento JSX se puede escribir con una etiqueta de auto-cierre, y cada elemento debe ser cerrado.  The line-break tag, for example, must always be written as `<br />` in order to be valid JSX that can be transpiled.  A `<div>`, on the other hand, can be written as `<div />` or `<div></div>`.  The difference is that in the first syntax version there is no way to include anything in the `<div />`. Verás en desafíos posteriores que esta sintaxis es útil al renderizar componentes de React.

# --instructions--

Corrige los errores en el editor de código para que sea JSX válido y se transpile exitosamente. Asegúrate de no cambiar nada del contenido: sólo tienes que cerrar las etiquetas donde se necesite.

