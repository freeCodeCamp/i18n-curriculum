# --description--

У цьому уроці ви навчитеся працювати з типами функцій. Почнемо з анотацій типів параметрів.

Ось приклад базової функції:

```ts
function circleArea(radius) {
  return Math.PI * radius * radius;
}
```

Зараз TypeScript відобразить повідомлення `Parameter 'radius' implicitly has an 'any' type`. Тип `any` означає, що значення може мати будь-який тип. Ви дізнаєтеся більше про цей тип у майбутньому уроці.

Щоб позбутися помилки і явно вказати тип параметра, ви можете зробити це так:

```ts
function circleArea(radius: number) {
  return Math.PI * radius * radius;
}
```

Якщо ви спробуєте викликати функцію `circleArea` з типом, який не є числом, TypeScript відобразить таку помилку:

```ts
function circleArea(radius: number) {
  return Math.PI * radius * radius;
}

// Argument of type 'string' is not assignable to parameter of type 'number'
circleArea("3");
```

Якщо ви спробуєте передати більше аргументів, ніж очікує функція, TypeScript відобразить помилку:

```ts
function circleArea(radius: number) {
  return Math.PI * radius * radius;
}

// Expected 1 arguments, but got 2.
circleArea(3, 4);
```

Якщо ви передасте замало аргументів, TypeScript також відобразить помилку:

```ts
function circleArea(radius: number) {
  return Math.PI * radius * radius;
}

// Expected 1 arguments, but got 0.
// An argument for 'radius' was not provided.
circleArea();
```

Ви можете зробити параметри необов’язковими, використовуючи символ `?` поруч із параметром, ось так:

```ts
function area(radius: number, height?: number): number {
  const baseArea = Math.PI * radius * radius;

  if (height !== undefined) {
    const sideArea = 2 * Math.PI * radius * height;
    return baseArea + sideArea;
  }

  return baseArea;
}
```

Умова всередині функції потрібна на випадок, якщо аргумент `height` не буде передано.

Якщо ви хочете вказати тип поверненого значення, можете додати анотацію типу повернення ось так:

```ts
function circleArea(radius: number): number {
  return Math.PI * radius * radius;
}
```

Анотація типу повернення розміщується після списку параметрів. Подібно до анотацій типів змінних, більшість часу TypeScript буде достатньо розумним, щоб вивести тип повернення. Тож використовуйте явну анотацію типу повернення, коли хочете зробити намір функції зрозумілішим або запобігти небажаним змінам поверненого значення.

Наступний приклад стосується асинхронних функцій і промісів. Якщо ваша функція має повертати проміс, ви можете використати тип `Promise` ось так:

```ts
async function getCityTemperature(city: string): Promise<number> {
  const response = await fetch(`https://api.example.com/weather?city=${city}`);
  const data = await response.json(); 
  return data.temperature;
}
```

У цій ситуації, якщо ви спробуєте видалити тип повернення, TypeScript може вивести тип повернення як `Promise<any>`. Тож це хороший випадок, щоб явно вказати тип поверненого значення.

До цього моменту ми розглядали лише приклади іменованих функцій. Але як працюють типи з анонімними функціями?

Ось приклад використання анонімної функції у функції зворотного виклику, переданій у `forEach`:

```ts
const languages = ["JavaScript", "Python", "TypeScript"];

languages.forEach((lang) => {
  console.log(`${lang.toUpperCase()} has ${lang.length} characters`);
});
```

Параметр `lang` не має анотації типу. Замість цього TypeScript використовує контекстне типізування, що означає, що він виводить тип із контексту, у якому цей параметр з’являється.
