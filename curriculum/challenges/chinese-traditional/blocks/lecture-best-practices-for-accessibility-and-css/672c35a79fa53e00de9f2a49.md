# --interactive--

在網頁上隱藏內容是網頁開發中常見的做法，但以不影響存取性的方法來進行非常重要。不同的隱藏技術會對輔助技術如何解讀並呈現內容給使用者產生不同的影響。讓我們探討一些在維護存取性的同時隱藏內容的最佳做法。一個常見的隱藏內容方法是使用 `display: none`。以下是一個範例：

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<p class="hidden">Hidden text</p>
<p>Visible text</p>
```

```css
.hidden {
  display: none;
}
```

:::

雖然這有效地在視覺上隱藏內容，但也會將其從無障礙樹中移除。無障礙樹是一種結構，用於輔助技術（例如螢幕閱讀器）來解讀和互動網頁上的內容。它以輔助技術能理解並呈現給使用者的方式表述內容及其語意意義。

使用 `display: none` 意味著螢幕閱讀器和其他輔助技術將無法存取此內容，因為它不包含在無障礙樹中。因此，只有當你想要完全從視覺呈現和無障礙中移除內容時，才重要使用此方法。 

另一種隱藏內容的方法是使用 `visibility: hidden`：

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<p class="hidden">Hidden text</p>
<p>Visible text</p>
```

```css
.hidden {
  visibility: hidden;
}
```

:::

`visibility: hidden` 在視覺上隱藏內容，但仍保留在文件流程中，這表示它仍然佔據頁面上的空間。像 `display: none` 一樣，`visibility: hidden` 也會將內容從可存取性樹中移除。這表示輔助技術，例如螢幕閱讀器，將無法存取被隱藏的內容。只有當你想要對所有人隱藏內容，包括使用輔助技術的人時，才使用 `visibility: hidden`。

對於應該在視覺上隱藏但仍可被螢幕閱讀器存取的內容，你可以使用一種通常稱為「視覺隱藏」或「僅限螢幕閱讀器」的技術。以下是一個使用 `.sr-only` CSS 類別的範例，這是一種常見的技術，用於在視覺上隱藏內容，同時保持其可被螢幕閱讀器存取：

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<p class="sr-only">Hidden text</p>
<p>Visible text</p>
```

```css
.sr-only {
  position: absolute;
  width: 1px;
  height: 1px;
  padding: 0;
  margin: -1px;
  overflow: hidden;
  clip: rect(0, 0, 0, 0);
  white-space: nowrap;
  border: 0;
}
```

:::

在此範例中，我們使用了像是 `position`、`clip` 和 `white-space` 這些屬性，你將在未來的課程中學習到。現在只需知道這條 CSS 規則能有效地在視覺上隱藏內容，同時保持其對螢幕閱讀器的可存取性。這對於在不影響視覺版面配置的情況下，為螢幕閱讀器使用者提供額外的背景關係非常有用。 

若要切換內容可見性，請考慮使用 `hidden` 屬性：

:::interactive_editor

```html
<p hidden>This content is hidden</p>
<p>This content is visible</p>
```

:::

大多數現代瀏覽器都支援 `hidden` 屬性，並且會在視覺上以及無障礙樹中隱藏內容。它可以透過 JavaScript 輕鬆切換。最後，請謹慎隱藏重要內容。如果資訊對於理解或使用網站至關重要，則應該對所有使用者可見且可存取。只有在確實能提升使用者體驗時，才隱藏內容。遵循這些最佳實務，你可以確保你的內容對所有使用者保持可存取，無論他們如何與你的網站互動。
