# --interactive--

`change` 이벤트는 사용자가 특정 입력 요소의 값을 수정할 때 발생하는 특별한 이벤트입니다. 구체적으로는:

- 체크박스가 선택되거나 선택 해제될 때.

- 라디오 버튼이 선택될 때.

- 사용자가 날짜 선택기나 드롭다운 메뉴 같은 것에서 선택할 때.

- 사용자가 값을 변경한 후 입력 필드가 포커스를 잃을 때(탭으로 다음 필드로 이동하거나 폼 밖을 클릭할 때).

- 사용자가 텍스트를 입력한 후 엔터 키를 눌러 값을 확인할 때.

`change` 이벤트는 사용자가 입력 필드에 타이핑할 때 발생하지 않는다는 점에 주의하세요. `change` 이벤트는 사용자가 다른 요소에 포커스를 옮긴 후에만 발생합니다.

다음은 드롭다운 메뉴에서 `change` 이벤트를 사용하는 예입니다:

:::interactive_editor

```html
<select id="select-menu">
  <option value="option1">Option 1</option>
  <option value="option2">Option 2</option>
  <option value="option3">Option 3</option>
</select>
<script src="index.js"></script>
```

```js
const selectMenu = document.getElementById("select-menu");
selectMenu.addEventListener("change", (event) => {
  console.log(`You selected: ${event.target.value}`);
});
```

:::

사용자가 드롭다운 메뉴에서 다른 옵션을 선택할 때마다 `change` 이벤트가 발생하고, 선택된 값이 콘솔에 기록됩니다.

`change` 이벤트도 `Event` 객체를 생성하지만, 대부분 다른 이벤트와 달리 커스텀 구현을 생성하지 않습니다 – 접근할 수 있는 프로퍼티와 메서드는 기본 `Event` 객체에 있는 것뿐입니다.

이는 전용 `input` 객체를 생성하는 `InputEvent` 이벤트와 다릅니다. Change 이벤트도 몇 가지 면에서 다릅니다. 예를 들어, `input` 이벤트는 사용자가 필드에 내용을 입력할 때 발생합니다.

이러한 차이점은 이벤트 발생 시점 때문에 혼동할 수 있으므로 기억하는 것이 중요합니다.
