# --description--

Mocking ni mchakato wa kubadilisha data halisi na data bandia inayofanana na tabia za sehemu halisi. Hii inawawezesha watatizaji na wasanidi programu kupima sehemu maalum katika programu bila kutegemea vitu vya nje.

Mfano wa kawaida wa kutumia mocking data ni pale unaposhughulika na wito za API. Fikiria una wito wa fetch kama huu katika programu yako:

```js
export const postService = {
  async getPosts() {
    const response = await fetch("https://api.example.com/posts");
    return response.json();
  },
};
```

Ikiwa unahitaji kupima kitendakazi cha `postService`, unaweza kutumia mock kwa jibu la API badala ya kufanya wito wa API mara kwa mara kupata data. Kupata data kutoka API kunachukua muda na kunategemea mambo ya nje kama upatikanaji wa mtandao na majibu ya seva. Mocking jibu la API huondoa mambo hayo ya nje na kuruhusu watatizaji kufanya kazi na seti ya data inayoweza kutabirika na kudhibitiwa.

Sasa tunapoelewa ni nini mocking, tulinganishe na stubbing. Stubs ni vitu vinavyorudisha majibu yaliyopangwa awali au data bandia kwa tabia inayotarajiwa katika programu. Kwa mfano, unaweza kutumia stub kwa tabia ya muunganisho wa hifadhidata katika majaribio yako bila kutegemea muunganisho halisi wa hifadhidata.

Hapa kuna mfano wa stub katika Jest:

```js
import { postService } from "./postService";
import { db } from "./db";

jest.mock("./db", () => ({
  db: {
    fetchPosts: jest.fn(),
  },
}));

test("should return stubbed posts", async () => {
  const fakePosts = [
    { id: 1, title: "Stubbed Post", content: "This is a stubbed post." },
  ];

  db.fetchPosts.mockResolvedValue(fakePosts);

  const posts = await postService.getPosts();

  expect(posts).toEqual(fakePosts);
});
```

`db.fetchPosts.mockResolvedValue(fakePosts);` ni stub katika mfano huu.

Sehemu ya mwisho ya kujadili ni fakes. Fakes ni matoleo rahisi ya sehemu halisi bila ugumu au athari za upande wa kitu halisi. Kwa mfano, unaweza kuiga hifadhidata kwa kuhifadhi data katika kumbukumbu badala ya kuingiliana na hifadhidata halisi. Hii itakuwezesha kuiga shughuli za hifadhidata katika kumbukumbu, ambayo itakuwa haraka zaidi kuliko kushughulika na hifadhidata halisi. Mfano mwingine wa kawaida ni kutumia fakes unaposhughulika na mifumo ya faili. Unaweza kuunda mfumo wa faili bandia badala ya halisi wakati wa kufanya majaribio.

Unapoendelea kujenga programu ngumu zaidi, fikiria kutumia mocks, stubs, au fakes pale inapofaa katika majaribio yako.
