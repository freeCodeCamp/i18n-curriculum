# --description--

Para demostrar una boilerplate HTML, primero necesita un archivo HTML para trabajar.

Crea una nueva carpeta en tu ordenador y nombrala `html-boilerplate`. Dentro de esa carpeta crear un nuevo archivo y nombralo `index.html`.

Probablemente ya estés familiarizado con muchos tipos diferentes de archivos, por ejemplo archivos doc, pdf e imagen.

Para que el equipo sepa que desea crear un archivo HTML, necesita añadir la extensión del archivo con el `.html` como se ha hecho al crear el archivo `index.html`.

It is worth noting that you named your HTML file index. Siempre debes nombrar el archivo HTML que contendrá la página principal de sus sitios web `index.html`. Esto se debe a que los servidores web buscarán por defecto una página `index.html` cuando los usuarios carguen tus sitios web - y no tener uno causará grandes problemas.

## El DOCTYPE

Toda página HTML comienza con una declaración doctype. El propósito de doctype es decirle al navegador qué versión de HTML debe utilizar para renderizar el documento.  The latest version of HTML is HTML5, and the doctype for that version is simply `<!DOCTYPE html>`.

Los tipos de documentación para versiones antiguas de HTML eran un poco más complicados. Por ejemplo, esta es la declaración doctype para HTML4:

```html
<!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.01 Transitional//EN" "http://www.w3.org/TR/html4/loose.dtd">
```

Sin embargo, probablemente nunca querrás usar una versión antigua de HTML, por lo que siempre usarás `<!DOCTYPE html>`.

Abre el archivo `index.html` creado anteriormente en tu editor de texto y agrega `<!DOCTYPE html>` en la primera línea.
