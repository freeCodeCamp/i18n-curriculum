# --interactive--

Le closure sono una delle funzionalità più potenti e spesso fraintese in JavaScript. In sostanza, una closure è una funzione che ha accesso alle variabili nel suo ambito lessicale esterno, anche dopo che la funzione esterna ha terminato l'esecuzione. Può sembrare complesso, ma è un concetto fondamentale che permette molti pattern di programmazione avanzati in JavaScript.

Per capire le closure, iniziamo con un esempio:

:::interactive_editor

```js
function outerFunction(x) {
    let y = 10;
    function innerFunction(){
        console.log(x + y);
    }
    return innerFunction;
}

let closure = outerFunction(5);
console.log(closure()); // 15
```

:::

In questo esempio, `outerFunction` prende un parametro `x` e definisce una variabile locale `y`. Poi definisce una `innerFunction` che usa sia `x` sia `y`. Infine restituisce `innerFunction`. Quando chiamiamo `outerFunction(5)` restituisce `innerFunction` che assegniamo alla variabile `closure`. Quando poi chiamiamo `closure()` ha ancora accesso a `x` e `y` da `outerFunction` anche se `outerFunction` ha già terminato l'esecuzione. Questa è l’essenza di una closure.

La funzione interna mantiene un riferimento al suo ambiente lessicale esterno, preservando l’accesso alle variabili in quell’ambito anche dopo che la funzione esterna ha completato.

Le closure sono particolarmente utili per creare variabili e funzioni private. Considera questo esempio:

:::interactive_editor

```js
function createCounter() {
    let count = 0;
    return function () {
        count++;
        return count;
    };
}

let counter = createCounter();
console.log(counter()); // 1
console.log(counter()); // 2
```

:::

In questo caso, `createCounter` restituisce una funzione che incrementa e restituisce una variabile `count`. La variabile `count` non è direttamente accessibile dall’esterno di `createCounter`, ma la funzione restituita (la nostra closure) vi ha accesso. Ogni volta che chiamiamo `counter()`, incrementa e restituisce `count`.

Le closure possono anche catturare più variabili dal loro ambito esterno. Per esempio:

:::interactive_editor

```js
function multiply(x) {
    return function (y) {
        return x * y;
    };
}

let double = multiply(2);
console.log(double(5)); // 10
```

:::

Qui la funzione interna cattura il parametro `x` da `multiply`. Quando creiamo `double` chiamando `multiply(2)` restituisce una funzione che moltiplica sempre il suo argomento per `2`.

Una cosa importante da notare sulle closure è che catturano le variabili per riferimento, non per valore. Questo significa che se il valore di una variabile catturata cambia, la closure vedrà il nuovo valore. Per esempio:

:::interactive_editor

```js
function createIncrementer() {
    let count = 0;
    return function () {
        count++;
        console.log(count);
    };
}

let increment = createIncrementer();
increment(); // 1
increment(); // 2
```

:::

Ogni volta che chiamiamo `increment` lavora con la stessa variabile `count`, non con una copia del suo valore iniziale. Le closure sono uno strumento potente in JavaScript. Man mano che continui a lavorare con JavaScript scoprirai che capire e usare efficacemente le closure può migliorare molto la tua capacità di scrivere codice pulito, efficiente e potente.
