# --description--

Die Web Storage API bietet einen Mechanismus, mit dem Browser Schlüssel-Wert-Paare direkt im Browser speichern können. So können Entwickler Informationen speichern, die über verschiedene Seiten-Neuladungen und Sitzungen hinweg verwendet werden können.

Die zwei Hauptkomponenten der Web Storage API sind `localStorage` und `sessionStorage`.

`localStorage` ist der Teil der Web Storage API, der es ermöglicht, dass Daten auch nach dem Schließen des Browserfensters oder dem Aktualisieren der Seite erhalten bleiben. Diese Daten bleiben verfügbar, bis sie explizit von der Anwendung oder dem Benutzer gelöscht werden.

`sessionStorage` ist ein weiterer Teil der Web Storage API, der Daten nur für die Dauer der aktuellen Sitzung speichert, das heißt, die Daten sind verfügbar, solange der Browser-Tab oder das Fenster geöffnet ist. Im Gegensatz zu `localStorage` werden die Daten in `sessionStorage` gelöscht, wenn der Tab oder das Fenster geschlossen wird. Mehr über `sessionStorage` lernen Sie in der nächsten Lektion.

Gängige Anwendungsfälle für `localStorage` sind das Speichern von Benutzereinstellungen wie Themen oder Spracheinstellungen, das Merken von Formularinhalten über Browser-Sitzungen hinweg und das Zwischenspeichern kleiner Datenmengen, um die Leistung von Webanwendungen zu verbessern.

Caching bedeutet, häufig genutzte Daten an einem temporären Speicherort, dem sogenannten Cache, zu speichern, damit nachfolgende Anfragen schneller bedient werden können, ohne die Daten erneut berechnen oder von einer langsameren Datenquelle wie einer Datenbank oder einem externen Server abrufen zu müssen.

Einige gängige `localStorage`-Methoden sind die `setItem`-, `getItem`-, `removeItem`- und `clear`-Methoden.

Hier ist ein Beispiel für die Verwendung der `setItem()`-Methode, mit der ein Schlüssel-Wert-Paar in `localStorage` gespeichert wird.

```js
localStorage.setItem('username', 'JaneDoe');
```

Wenn wir dann den Wert eines bestimmten Schlüssels aus `localStorage` abrufen wollen, können wir die `getItem()`-Methode verwenden.

```js
let username = localStorage.getItem('username');
console.log(username); // JaneDoe
```

Um ein Element aus `localStorage` anhand seines Schlüssels zu entfernen, können Sie die `removeItem()`-Methode verwenden.

```js
localStorage.removeItem('username');
```

Um alle Daten in `localStorage` zu löschen, können Sie die `clear()`-Methode verwenden.

```js
localStorage.clear();
```

Schauen wir uns nun ein Beispiel an, in dem wir `localStorage` verwenden, um das bevorzugte Thema eines Benutzers zu speichern.

```js
// Store the user's theme preference
localStorage.setItem('theme', 'dark');

// Retrieve the stored theme preference
const userTheme = localStorage.getItem('theme');
console.log(userTheme); // 'dark'

// Remove the theme preference
localStorage.removeItem('theme');

// Clear all localStorage data
localStorage.clear();
```

In diesem Beispiel:

- Speichern wir zuerst eine Themenauswahl (`dark`) für den Benutzer.

- Rufen wir dann dieses Thema ab und geben es in der Konsole aus.

- Zeigen wir schließlich, wie man ein bestimmtes Element entfernt oder alle gespeicherten Daten löscht.

`localStorage` ist sehr nützlich, um kleine Datenmengen zu speichern, die zwischen Sitzungen erhalten bleiben müssen. Es ist jedoch wichtig zu beachten, dass `localStorage` nicht zum Speichern sensibler Informationen wie Passwörter verwendet werden sollte, da dies Sicherheitsrisiken bergen kann.
