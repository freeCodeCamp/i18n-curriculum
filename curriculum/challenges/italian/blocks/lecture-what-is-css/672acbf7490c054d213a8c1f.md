# --interactive--

I CSS possono essere applicati a una pagina web in tre modi principali: inline, interno o esterno.

Ogni metodo ha il suo caso d’uso, vantaggi e limiti, e sapere quando usare ciascuno è essenziale per scrivere codice pulito, efficiente e facile da mantenere.

Analizziamo i tre tipi di CSS e quando dovresti usarli.

I CSS inline sono scritti direttamente all’interno di un elemento HTML usando l’attributo `style`. Applicano gli stili a un elemento specifico.

Ecco un esempio che usa i CSS inline:

:::interactive_editor

```html
<p style="color: green;">This is an inline-styled paragraph.</p>
```

:::

In questo esempio, stiamo usando l’attributo `style` per impostare il testo del paragrafo su `green`.

I CSS inline si usano generalmente per stili rapidi e occasionali o per sovrascrivere altri stili su un elemento specifico.

Tuttavia, dovrebbero essere evitati nella maggior parte dei casi perché possono appesantire l’HTML e rendere il codice più difficile da mantenere.

Quasi sempre è meglio usare CSS interni o esterni per mantenere gli stili organizzati e facili da gestire.

I CSS interni sono scritti all’interno dei tag `style` nella sezione `head` di un documento HTML. Applicano gli stili all’intera pagina e sono utili quando devi stilizzare un singolo documento.

Ecco un esempio di CSS interno:

:::interactive_editor

```html
<head>
  <style>
    p {
      color: blue;
    }
  </style>
</head>
<body>
  <p>This paragraph is styled using internal CSS.</p>
</body>
```

:::

Nell’esempio sopra, i CSS interni applicano il testo blu a tutti gli elementi `p` della pagina.

I CSS interni sono più indicati quando devi applicare stili a una pagina specifica e non a più pagine. Sono utili per siti a pagina singola o quando gli stili non devono essere riutilizzati altrove.

Tuttavia, ci sono alcuni svantaggi, come la mancata promozione della riusabilità su più pagine. Inoltre, come i CSS inline, mescolano HTML e CSS, rendendo il codice più difficile da mantenere in progetti più grandi.

I CSS esterni sono scritti in un file `.css` separato e collegati al documento HTML usando l’elemento `link` nella sezione `head`.

Permettono di stilizzare più pagine in modo coerente e sono il metodo preferito nello sviluppo web professionale.

Ecco un esempio di stile applicato a tutti gli elementi paragrafo:

:::interactive_editor

```html
<head>
  <link rel="stylesheet" href="styles.css">
</head>
<body>
  <p>This paragraph is styled using external CSS.</p>
</body>
```

```css
p {
  color: red;
}
```

:::

In una lezione precedente, hai imparato che l’elemento `link` ha un attributo `rel` che specifica la relazione tra il documento corrente e la risorsa collegata, come collegare un foglio di stile o una risorsa esterna.

D’altra parte, l’attributo `href` specifica l’URL della risorsa collegata, indicando da dove recuperare la risorsa.

I CSS esterni sono ideali per progetti grandi in cui vuoi mantenere uno stile coerente su più pagine.

Promuovono la separazione delle responsabilità, con l’HTML che gestisce la struttura e i CSS che gestiscono lo stile, rendendo il codice più facile da mantenere e scalare.

Capire quando usare ciascun tipo di CSS è fondamentale per uno sviluppo web efficiente ed efficace.

Nella maggior parte dei casi, i CSS esterni dovrebbero essere la tua scelta principale, soprattutto per progetti più grandi e complessi.
