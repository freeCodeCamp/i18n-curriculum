# --description--

In früheren Lektionen haben Sie gelernt, wie man kleine Codeeinheiten durch Unit-Testing testet. Unit-Testing ist ein Teil des funktionalen Testens. Funktionales Testen überprüft, ob die Funktionen und Features der Anwendung wie erwartet funktionieren.

In dieser Lektion betrachten wir die Unterschiede zwischen funktionalem und nicht-funktionalem Testen.

Das Ziel des funktionalen Testens ist es, das System als Ganzes anhand verschiedener Szenarien zu testen, um sicherzustellen, dass das System wie erwartet funktioniert. Ein Beispiel für funktionales Testen ist das Smoke Testing. Beim Smoke Testing führen Sie eine erste Überprüfung des Systems auf grundlegende oder kritische Probleme durch, bevor umfangreichere Tests erfolgen.

Im Gegensatz dazu konzentriert sich das nicht-funktionale Testen auf Aspekte wie Leistung und Zuverlässigkeit.

Ein realistisches Testszenario für funktionales Testen wäre die Benutzeranmeldung. Ein Benutzer sollte sich mit seinem Benutzernamen und Passwort in der Anwendung anmelden können. Wenn dies erfolgreich ist, sollte er auf seine Kontoinformationen zugreifen können. Wenn ein Problem auftritt, muss dieser Fehler ordnungsgemäß behandelt werden. Beim Schreiben von Tests müssen Sie alle Arten von Erfolgs- und Fehlschlagsszenarien berücksichtigen.

Ein Beispiel für nicht-funktionales Testen wäre das Testen der Leistung Ihrer Anwendung unter verschiedenen Netzwerkbedingungen. In einer zukünftigen Lektion lernen Sie mehr über Leistungstests.

Der Grund, warum funktionales Testen so wichtig ist, liegt darin, dass Sie Bugs frühzeitig in Ihrer Anwendung erkennen möchten. Außerdem wollen Sie sicherstellen, dass Ihre Anwendung die Anforderungen erfüllt, die zu Beginn der Planungsphase festgelegt wurden. Schließlich möchten Sie gewährleisten, dass Ihre Nutzer eine gute Benutzererfahrung haben. Nutzer sollten keine Schwierigkeiten haben, Ihre Anwendung zu verwenden, oder durch fehlerhafte Funktionen und Systemausfälle blockiert werden.
