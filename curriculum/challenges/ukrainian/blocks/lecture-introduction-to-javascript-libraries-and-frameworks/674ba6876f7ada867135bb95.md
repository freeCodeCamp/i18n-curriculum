# --description--

У попередніх уроках ви дізналися, як працювати з імпортами та експортами в JavaScript. У цьому уроці ми розглянемо, як імпортувати та експортувати компоненти в React.

Експорт робить компонент доступним для інших файлів. Імпорт дозволяє файлу використовувати компонент, який був експортований в іншому місці. У проєктах React саме так компоненти повторно використовуються та комбінуються для створення інтерфейсів користувача.

У цьому уроці ви навчитеся експортувати компоненти React за допомогою експорту за замовчуванням і іменованого експорту, а також імпортувати їх там, де це потрібно.

У цьому прикладі у нас є компонент `Cat`, який знаходиться у файлі з назвою `Cat.jsx`. Для розширення файлу ми використовуємо розширення `.jsx`, оскільки цей файл переважно працює з JSX.

Цей компонент `Cat` повертає JSX-розмітку з заголовком і зображенням кота на ім’я містер Віскерс:

```jsx
function Cat() {
  return (
    <div className="card">
      <h2>Mr. Whiskers</h2>
      <img
        src="https://cdn.freecodecamp.org/curriculum/cat-photo-app/running-cats.jpg"
        alt="Tuxedo cats running on dirt ground."
      />
    </div>
  );
}
```

Якщо ми хочемо використати наш компонент `Cat` в іншому файлі, спочатку потрібно його експортувати так:

```jsx
function Cat() {
  return (
    <div className="card">
      <h2>Mr. Whiskers</h2>
      <img
        src="https://cdn.freecodecamp.org/curriculum/cat-photo-app/running-cats.jpg"
        alt="Tuxedo cats running on dirt ground."
      />
    </div>
  );
}

export default Cat;
```

Ми використовуємо ключове слово `default`, тому що це буде експорт за замовчуванням із модуля. Ви також можете обрати експорт компонента на тій же лінії, що й його визначення, ось так:

```jsx
export default function Cat() {
  return (
    <div className="card">
      <h2>Mr. Whiskers</h2>
      <img
        src="https://cdn.freecodecamp.org/curriculum/cat-photo-app/running-cats.jpg"
        alt="Tuxedo cats running on dirt ground."
      />
    </div>
  );
}
```

Ви можете імпортувати дочірні компоненти в інших файлах батьківських компонентів або імпортувати їх у файлі кореневого компонента. У цьому прикладі ми імпортуємо компонент `Cat` всередині файлу кореневого компонента.

У кожному проєкті React є компонент найвищого рівня, зазвичай званий `App.jsx`:

```jsx
export default function App() {
  return (
    // return component here
  );
}
```

Цей файл зазвичай розташований у папці `src` вашого проєкту. Ви дізнаєтеся більше про загальні макети проєктів у майбутньому уроці.

Щоб використати компонент `Cat` всередині кореневого компонента `App`, потрібно імпортувати його так:

```jsx
import Cat from "./Cat";

export default function App() {
  return (
    // return component here
  );
}
```

Тепер ви можете повернути компонент `Cat` всередині компонента `App` ось так:

```jsx
import Cat from "./Cat";

export default function App() {
  return (
    <Cat />
  );
}
```

Такий підхід добре працює, коли файл відповідає за експорт одного основного компонента, що робить синтаксис імпорту простим і легким для читання. Пам’ятайте, що файл може мати лише один експорт за замовчуванням, що робить його ідеальним для файлу, який переважно містить один компонент.

Тепер, коли ви знаєте, як використовувати експорт за замовчуванням, давайте розглянемо іменований експорт. Іменовані експорти дозволяють файлу ділитися кількома компонентами або функціями. На відміну від експорту за замовчуванням, їх потрібно імпортувати з точною назвою, з якою вони були експортовані (якщо ви не перейменуєте їх за допомогою ключового слова `as`).

Це корисно, коли файл слугує бібліотекою компонентів. Наприклад, ось файл, що містить два компоненти, Cat і Dog:

```jsx
// Animals.jsx
export function Cat() {
  return <h2>Mr. Whiskers</h2>;
}

export function Dog() {
  return <h2>Fido</h2>;
}
```

Ми використовуємо ключове слово export окремо для кожного компонента. Щоб використати їх в іншому файлі, імпортуємо їх у фігурних дужках:

```jsx
import { Cat, Dog } from "./Animals";

export default function App() {
  return (
    <div>
      <Cat />
      <Dog />
    </div>
  );
}
```

Синтаксис `{ Cat, Dog }` каже JavaScript імпортувати конкретні іменовані експорти з файлу `Animals.jsx`. Імена всередині фігурних дужок мають точно відповідати експортованим іменам. Якщо потрібно уникнути конфліктів імен або ви хочете іншу назву у поточному файлі, можна перейменувати компонент під час імпорту за допомогою ключового слова as.

```jsx
import { Cat as Kitty } from "./Animals";

export default function App() {
  return <Kitty />;
}
```

Нарешті, файл може мати один експорт за замовчуванням і кілька іменованих експортів.

```jsx
// Animals.jsx
export default function Cat() {
  return <h2>Mr. Whiskers</h2>;
}

export function Dog() {
  return <h2>Fido</h2>;
}
```

При імпорті змішаних експортів експорт за замовчуванням іде першим (без дужок), за ним ідуть іменовані експорти (всередині дужок):

```jsx
// App.jsx
import Cat, { Dog } from "./Animals";

export default function App() {
  return (
    <div>
      <Cat />
      <Dog />
    </div>
  );
}
```

У цьому прикладі `Cat` — це експорт за замовчуванням, тому він не використовує фігурні дужки. `Dog` — іменований експорт, тому він їх вимагає.

Розуміючи, як працюють експорт за замовчуванням і іменований експорт, ви зможете організовувати компоненти React у кількох файлах і повторно використовувати їх там, де потрібно. Коли ви створюєте більші застосунки, правильний вибір стилю експорту допоможе зберегти ваш код чистим і підтримуваним, а також полегшить роботу з компонентами, створеними іншими.
