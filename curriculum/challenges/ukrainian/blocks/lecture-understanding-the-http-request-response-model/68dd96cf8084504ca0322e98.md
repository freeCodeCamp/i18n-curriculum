# --description--

Модель HTTP запит-відповідь — це базовий шаблон комунікації, який забезпечує роботу Всесвітньої павутини (WWW). Вона має багато компонентів, що роблять її ефективною, щоб ми всі могли отримувати доступ до матеріалів з вебу.

Давайте розглянемо, що таке модель HTTP запит-відповідь, які компоненти її складають і як ці компоненти працюють разом, щоб забезпечити ефективність роботи вебу.

Модель HTTP запит-відповідь — це шаблон комунікації у веб-взаємодіях, у якому клієнт, зазвичай веб-браузер, надсилає **HTTP-запит** на сервер, а сервер обробляє цей запит і надсилає назад **HTTP-відповідь**.

Ось компоненти, які складають модель HTTP запит-відповідь:

* **Клієнт**: браузер, мобільний застосунок або будь-яке інше програмне забезпечення, що надсилає HTTP-запити на сервер і інтерпретує отримані HTTP-відповіді.
    
* **Запит**: повідомлення, яке клієнт надсилає серверу. Воно містить метод, URL, заголовки та необов’язкове тіло.
    
* **HTTP-метод**: дія, яку потрібно виконати — `GET` для отримання даних, `POST` для надсилання даних, `PATCH` і `PUT` для зміни даних, а `DELETE` для видалення даних.
    
* **URL (уніфікований локатор ресурсу)**: ресурс, який запитується у серверу.
    
* **Заголовки**: метадані, що надсилаються разом із запитом або відповіддю. Наприклад, `Content-Type` і `Authorization`.
    
* **Тіло**: дані, що надсилаються з `POST`, `PUT`, `PATCH`, `DELETE` запитами, часто у форматі JSON або у вигляді даних форми.
    
* **Сервер**: бекенд-система, яка обробляє запит і формує відповідь. Він може бути написаний будь-якою мовою програмування, що підтримує серверне програмування.
    
* **Відповідь**: повідомлення, яке сервер надсилає назад. Воно містить код стану, заголовки та необов’язкове тіло.
    
* **Код стану або код відповіді**: числовий код, що вказує на успіх, помилки або перенаправлення.
    
* **Тіло відповіді**: фактичні дані, які надсилаються назад, зазвичай у форматі HTML, JSON або XML.

Отже, як усі ці компоненти працюють разом у циклі HTTP запит-відповідь?

1. **Клієнт ініціює запит**
    
    * Браузер, мобільний застосунок або API-клієнт надсилає HTTP-запит, який має дійти до серверу.
        
    * Запит містить метод, наприклад `GET`, `POST`, `PUT`, `PATCH`, `DELETE`, URL, заголовки і іноді тіло.
        
2. **Запит надсилається на сервер**
    
    * Запит передається через інтернет до серверу — бекенд-системи, написаної мовою програмування.
        
    * Сервер отримує запит і витягує з нього релевантну інформацію.
        
3. **Сервер обробляє запит**
    
    * Сервер перевіряє метод запиту і URL.
        
    * Можливо, потрібно перевірити автентифікацію (через заголовки, наприклад authorization).
        
    * За потреби він взаємодіє з базою даних, щоб отримати або змінити дані.
        
4. **Сервер готує відповідь**
    
    * Сервер формує відповідь із кодом стану, наприклад 200, 404, 500 тощо.
        
    * Відповідь містить заголовки, наприклад `Content-Type: application/json`.
        
    * Якщо потрібно, додається тіло відповіді у форматі JSON, HTML або XML.
        
5. **Відповідь надсилається клієнту**
    
    * Сервер надсилає відповідь назад через інтернет.
        
    * Клієнт отримує відповідь і обробляє дані.
        
6. **Клієнт обробляє відповідь**
    
    * Якщо відповідь успішна (код стану 200), клієнт відображає дані, наприклад, відтворює їх на вебсторінці.
        
    * Якщо сталася помилка (`404 Not Found`, `500 Internal Server Error` тощо), клієнт може показати повідомлення про помилку.

Цей цикл повторюється для кожного нового запиту, який робить клієнт.
