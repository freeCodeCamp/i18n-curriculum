# --description--

Якщо ви не знайомі з `let`, перегляньте <a href="/learn/javascript-algorithms-and-data-structures/basic-javascript/explore-differences-between-the-var-and-let-keywords" target="_blank" rel="noopener noreferrer nofollow">це завдання про відмінності між <code>let</code> і <code>var</code></a>.

Коли ви оголошуєте змінну за допомогою ключового слова `var`, вона оголошується глобально або локально, якщо оголошена всередині функції.

Ключове слово `let` поводиться подібно, але з деякими додатковими функціями. Коли ви оголошуєте змінну за допомогою ключового слова `let` всередині блоку, інструкції або виразу, її область видимості обмежена цим блоком, інструкцією або виразом.

Наприклад:

```js
var numArray = [];
for (var i = 0; i < 3; i++) {
  numArray.push(i);
}
console.log(numArray);
console.log(i);
```

Тут у консолі буде виведено значення `[0, 1, 2]` і `3`.

За допомогою ключового слова `var` `i` оголошується глобально. Тож коли виконується `i++`, оновлюється глобальна змінна. Цей код схожий на наступний:

```js
var numArray = [];
var i;
for (i = 0; i < 3; i++) {
  numArray.push(i);
}
console.log(numArray);
console.log(i);
```

Тут у консолі буде виведено значення `[0, 1, 2]` і `3`.

Ця поведінка спричинить проблеми, якщо ви створите функцію і збережете її для подальшого використання всередині циклу `for`, який використовує змінну `i`. Це тому, що збережена функція завжди посилатиметься на значення оновленої глобальної змінної `i`.

```js
var printNumTwo;
for (var i = 0; i < 3; i++) {
  if (i === 2) {
    printNumTwo = function() {
      return i;
    };
  }
}
console.log(printNumTwo());
```

Тут у консолі буде виведено значення `3`.

Як бачите, `printNumTwo()` виводить 3, а не 2. Це тому, що значення, призначене `i`, було оновлено, і `printNumTwo()` повертає глобальну `i`, а не значення, яке `i` мала, коли функція була створена в циклі for. Ключове слово `let` не має такої поведінки:

```js
let printNumTwo;
for (let i = 0; i < 3; i++) {
  if (i === 2) {
    printNumTwo = function() {
      return i;
    };
  }
}
console.log(printNumTwo());
console.log(i);
```

Тут у консолі буде виведено значення `2` і помилку, що `i is not defined`.

`i` не визначена, бо вона не оголошена в глобальній області видимості. Вона оголошена лише всередині інструкції циклу `for`. `printNumTwo()` повернула правильне значення, бо три різні змінні `i` з унікальними значеннями (0, 1 і 2) були створені ключовим словом `let` всередині інструкції циклу.

# --instructions--

Виправте код так, щоб `i`, оголошена в інструкції `if`, була окремою змінною від `i`, оголошеної в першій лінії функції. Обов’язково не використовуйте ключове слово `var` ніде у вашому коді.

Ця вправа покликана показати різницю в тому, як ключові слова `var` і `let` призначають область видимості оголошеній змінній. При програмуванні функції, подібної до тієї, що використовується в цій вправі, часто краще використовувати різні імена змінних, щоб уникнути плутанини.
