# --description--

Vediamo alcuni dei comandi fondamentali che puoi usare per inserire e visualizzare dati in SQL.

Prima di tutto, per visualizzare i dati, devi inserirli. Quindi inizieremo con le inserzioni.

Gli esempi che tratteremo in questa lezione si baseranno su questa tabella di record di cani:

```sql
CREATE TABLE dogs(
  id SERIAL,
  name VARCHAR(100),
  age INTEGER
);
```

La tabella conterrà i nomi e le età di vari cani.

Se assumiamo di lavorare con PostgreSQL, possiamo usare il tipo di dato `SERIAL` per il `id`. Poiché `id` è stato creato usando `SERIAL`, il suo valore sarà un `INTEGER`, che parte da `1` e viene incrementato automaticamente quando viene inserito un nuovo record. Quindi non dovrai passare un valore per esso quando inserisci un record.

Iniziamo inserendo record nella nostra tabella `dogs` con l’istruzione `INSERT INTO`. Ci sono molti modi per farlo. Il primo consiste nell’inserire una singola riga specificando le colonne dei valori che verranno inseriti.

Scrivi `INSERT INTO`, seguito dal nome della tabella (in questo caso `dogs`), poi le colonne tra parentesi, quindi `VALUES` e i valori corrispondenti tra parentesi.

Ecco un esempio: stiamo inserendo un record di un cane con il `name` `'Gino'`, la cui `age` è `3`; ricorda che il `id` verrà assegnato automaticamente:

```sql
INSERT INTO dogs (name, age)
VALUES ('Gino', 3);
```

Questa è di solito l’opzione più sicura perché specifichi esplicitamente le colonne, quindi i valori saranno assegnati a quelle colonne in ordine.

Nota che `'Gino'` deve essere tra virgolette singole perché è di tipo `VARCHAR`.

Un’altra alternativa è inserire il record senza specificare le colonne, così:

```sql
INSERT INTO dogs
VALUES ('Gino', 3);
```

Questa sintassi è SQL valida, ma è soggetta a errori.

Poiché i valori sono assegnati in base all’ordine delle colonne, PostgreSQL cercherà di inserire `'Gino'` nella colonna `id`, che si aspetta un intero. Questo genera un errore come `invalid input syntax for type integer`.

Per questo motivo, è più sicuro specificare esplicitamente i nomi delle colonne quando inserisci dati.

Puoi anche inserire più record nella stessa istruzione SQL separandoli con una virgola.

Qui stiamo inserendo due record di cani, uno per `'Gino'` e un altro per `'Nora'`.

```sql
INSERT INTO dogs (name, age)
VALUES
  ('Gino', 3),
  ('Nora', 2);
```

E proprio come puoi inserire record, puoi interrogare informazioni dal database. Nel contesto dei database, una query è una richiesta di dati.

In SQL, l’istruzione `SELECT` viene usata per interrogare dati da una o più tabelle. Puoi personalizzare la query per ottenere esattamente le informazioni di cui hai bisogno.

Se devi interrogare tutte le informazioni dalla tabella `dogs`, devi solo scrivere `SELECT`, seguito da un asterisco (`*`), poi `FROM` e il nome della tabella, `dogs`:

```sql
SELECT *
FROM dogs;
```

In questo comando, l’asterisco è un carattere jolly che rappresenta “tutte le colonne”.

Otterrai tutti i dati di tutti i record nella tabella:

```sql
 id | name | age 
----+------+-----
  1 | Gino |   3
  2 | Nora |   2
```

Per interrogare colonne specifiche, puoi scrivere i nomi di quelle colonne nel comando, subito dopo `SELECT`, separati da una virgola. In questo esempio, interroghiamo le colonne `name` e `age` della tabella `dogs`:

```sql
SELECT name, age
FROM dogs;
```

Il risultato includerà solo le colonne `name` e `age`:

```sql
 name | age 
------+-----
 Gino |   3
 Nora |   2
```

A volte potresti aver bisogno di interrogare dati basandoti su una condizione specifica. Per esempio, per ottenere tutti i cani che hanno meno di 3 anni puoi usare la parola chiave `WHERE` e l’operatore di confronto “minore di” (`<`):

```sql
SELECT *
FROM dogs
WHERE age < 3;
```

Il risultato includerà solo i cani la cui età è inferiore a 3:

```sql
 id | name | age 
----+------+-----
  2 | Nora |   2
```

Se stai cercando solo il `age` di `'Gino'`, puoi usare l’operatore di confronto “uguale” (`=`):

```sql
SELECT age
FROM dogs
WHERE name = 'Gino';
```

Ecco il risultato:

```sql
 age 
-----
  3
```

Questi sono modi comuni e semplici per inserire e visualizzare dati in SQL, ma ci sono molte opzioni diverse tra cui scegliere e alcune sono più avanzate. Le imparerai nelle prossime lezioni.
