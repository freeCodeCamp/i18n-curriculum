# --interactive--

У програмуванні успадкування дозволяє визначати класи, які успадковують властивості та методи від інших класів.

Наприклад, `Car` є типом `Vehicle`, тому ви можете визначити клас `Vehicle` з найзагальнішими властивостями та методами, які є спільними для всіх типів транспортних засобів, а потім визначити клас `Car`, який успадковує властивості та методи класу `Vehicle`.

Це відношення "є" є характерним для успадкування. У цьому прикладі `Car` є більш спеціалізованою формою `Vehicle`.

Існують два основні поняття, які потрібно знати, щоб почати працювати з успадкуванням: батьківський клас і дочірній клас.

Батьківський клас — це клас, який слугує як шаблон для інших класів. Він визначає властивості та методи, які успадковуються іншими класами.

Дочірній клас — це клас, який успадковує властивості та методи іншого класу. Дочірні класи також можуть розширювати функціональність своїх батьківських класів, додаючи нові властивості та методи.

У нашому попередньому прикладі `Vehicle` був би батьківським класом для класу `Car`, тоді як клас `Car` був би дочірнім класом для `Vehicle`.

Батьківський клас може мати кілька дочірніх класів.

У JavaScript ми використовуємо ключове слово `extends` для реалізації успадкування. Це ключове слово вказує, що клас є дочірнім класом іншого класу.

У цьому прикладі, де ми бачимо лише перші рядки визначень класів, ви можете побачити, що клас `Car` розширює клас `Vehicle`, тому `Car` успадковує від `Vehicle`.

```js
class Vehicle {
  // Implementation of Vehicle...
}

class Car extends Vehicle {
  // Implementation of Car...
}
```

Припустимо, що клас `Vehicle` має дві властивості: `brand` і `year`. Ви можете побачити це в оновленому коді.

`Car` має метод під назвою `honk`. Однак зверніть увагу, що клас `Car` не має власного конструктора. Подивимося, чи він успадковує властивості.

```js
class Vehicle {
  constructor(brand, year) {
    this.brand = brand;
    this.year = year;
  }
}

class Car extends Vehicle {
  honk() {
    console.log("Honk! Honk!");
  }
}
```

Щоб визначити екземпляр `Car`, вам потрібно передати два аргументи: `brand` і `year`. У цьому випадку бренд — `freeCodeCamp Motors`, а рік — `2019`.

```js
let myCar = new Car("freeCodeCamp Motors", 2019);
```

Якщо ви спробуєте отримати доступ до цих властивостей і викликати метод, використовуючи точкову нотацію:

```js
console.log(myCar.brand);
console.log(myCar.year);
myCar.honk();
```

Ось повний приклад:

:::interactive_editor

```js
class Vehicle {
  constructor(brand, year) {
    this.brand = brand;
    this.year = year;
  }
}

class Car extends Vehicle {
  honk() {
    console.log("Honk! Honk!");
  }
}

let myCar = new Car("freeCodeCamp Motors", 2019);
console.log(myCar.brand);
console.log(myCar.year);
myCar.honk();
```

:::

Дійсно, вихідні дані правильні, отже ці властивості та метод визначені в екземплярі `Car`, і ви щойно заощадили багато коду, успадкувавши ці властивості від класу `Vehicle`.

У цьому прикладі дочірній клас не мав додаткових властивостей. Ось чому клас не мав конструктора, лише метод.

Якщо вам потрібно додати додаткові властивості, потрібно визначити конструктор.

Давайте подивимося приклад.


```js
class Vehicle {
  constructor(brand, year) {
    this.brand = brand;
    this.year = year;
  }
}

class Car extends Vehicle {
  constructor(brand, year, numDoors) {
    super(brand, year);
    this.numDoors = numDoors;
  }
}
```

Цього разу ми додамо властивість `numDoors` до підкласу `Car`.

В оновленому коді ви бачите, що тепер цей клас має конструктор. Усередині конструктора є виклик `super()`, який передає аргументи `brand` і `year`.

`super()` викликає конструктор надкласу, тому, використовуючи `super(brand, year)`, ви фактично визначаєте властивості надкласу в підкласі. Це ще одна термінологія, яку можна використовувати — надклас це батьківський клас, а підклас — дочірній клас.

Потім рядок `this.numDoors = numDoors` визначає нову властивість, яка належить виключно класу `Car`, кількість дверей (`numDoors`).

Це спосіб розширити клас `Vehicle`, додавши властивість, яка є виключною для екземплярів `Car`.

Це приклад розширюваності, однієї з фундаментальних переваг успадкування.

Тепер, якщо ви створите екземпляр класу `Car`, ви побачите, що він має три властивості: `brand`, `year` і `numDoors`. Вам також потрібно передати необхідні аргументи при створенні екземпляра.

```js
let myCar = new Car("freeCodeCamp Motors", 2019, 4);
```

Якщо ви виведете значення цих властивостей за допомогою `console.log()`:

```js
console.log(myCar.brand);
console.log(myCar.year);
console.log(myCar.numDoors);
```

Ось повний приклад:

:::interactive_editor

```js
class Vehicle {
  constructor(brand, year) {
    this.brand = brand;
    this.year = year;
  }
}

class Car extends Vehicle {
  constructor(brand, year, numDoors) {
    super(brand, year);
    this.numDoors = numDoors;
  }
}

let myCar = new Car("freeCodeCamp Motors", 2019, 4);
console.log(myCar.brand);
console.log(myCar.year);
console.log(myCar.numDoors);
```

:::

Основні переваги успадкування — це повторне використання коду, модульність, розширюваність і покращена структура коду.

Реалізуючи ієрархію, ви можете повторно використовувати код, який уже написали для батьківського класу, у дочірньому класі, уникаючи повторень.

Успадкування також сприяє модульності, розбиваючи складні системи на простіші компоненти в ієрархії.

Крім того, можливість розширювати функціональність батьківського класу полегшує адаптацію до змінних вимог і додавання нових функцій пізніше в процесі розробки.

Нарешті, ієрархічна структура може зробити ваш код легшим для розуміння та підтримки.

Ось основи успадкування. Розуміючи, як працює успадкування, ви зможете проектувати добре структуровані, підтримувані та розширювані об’єктно-орієнтовані програми.
