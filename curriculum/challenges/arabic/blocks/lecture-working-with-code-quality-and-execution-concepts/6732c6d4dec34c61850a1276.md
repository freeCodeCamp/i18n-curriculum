# --description--

إدارة الذاكرة مفهوم أساسي في البرمجة، لكنها قد تكون مربكة قليلاً للمبتدئين، فلنشرحها ببساطة.

عندما تشغل برنامجًا، بما في ذلك شفرة JavaScript في متصفح الويب، يحتاج إلى ذاكرة لتخزين كل المعلومات التي يعمل بها. هذا يشمل المتغيرات، والدوال، والكائنات، وكل شيء ينشئه ويستخدمه الكود الخاص بك. إدارة الذاكرة هي عملية التحكم في هذه الذاكرة، تخصيصها عند الحاجة، وتحريرها عندما لا تعود ضرورية. في بعض لغات البرمجة، يجب على المطوّرين إدارة الذاكرة يدويًا. عليهم أن يخبروا الكمبيوتر صراحةً متى يخصص الذاكرة لأشياء جديدة ومتى يحرر الذاكرة التي لم تعد ضرورية. هذا قد يكون قويًا لكنه أيضًا معقد، لأن نسيان تحرير الذاكرة قد يؤدي إلى تسربات في الذاكرة.

لكن JavaScript تستخدم إدارة ذاكرة تلقائية. هذا يعني أن JavaScript (وبشكل أكثر تحديدًا محرك JavaScript في متصفح الويب) يتولى تخصيص الذاكرة وتحريرها نيابة عنك. لا تحتاج إلى تحرير الذاكرة صراحةً في شفرتك. تُسمى هذه العملية التلقائية غالبًا بجمع القمامة.

إليك كيف تعمل ببساطة. يحدث التخصيص أولًا عندما تنشئ متغيرًا أو كائنات أو دوال في شفرة JavaScript الخاصة بك، حيث تُخصص الذاكرة تلقائيًا لتخزينها. ثم تستخدم هذه الذاكرة المخصصة عندما تتعامل مع هذه المتغيرات أو الكائنات أو الدوال في شفرتك.

يمتلك محرك JavaScript طرقًا ذكية ليحدد متى لا يعود شيء في الذاكرة ضروريًا. عمومًا، إذا لم يعد بإمكان برنامجك الوصول إلى قطعة بيانات أو استخدامها، تُعتبر غير ضرورية. دوريًا، يعمل جامع القمامة، فيجد الذاكرة التي لم تعد ضرورية ويحررها، مما يجعلها متاحة للاستخدام مستقبلاً. تحدث هذه العملية تلقائيًا، وهذا رائع لأنه يعني أنك لا تحتاج للقلق بشأن إدارة الذاكرة بنفسك.

مع ذلك، من المهم فهم هذا المفهوم لأنك قد تحتفظ أحيانًا عن طريق الخطأ بمراجع لأشياء لم تعد تحتاجها، مما يمنع جامع القمامة من تحرير تلك الذاكرة. على سبيل المثال:

```js
function createLargeArray() {
  let largeArray = new Array(1000000);
  return function() {
    console.log(largeArray.length);
  };
}

let printArrayLength = createLargeArray();
printArrayLength();
```

في هذا الكود، حتى بعد انتهاء `createLargeArray` من التنفيذ، لا يمكن جمع `largeArray` من القمامة لأن الدالة المعادة لا تزال تصل إليها. هذا إغلاق، وعلى الرغم من أن الإغلاقات مفيدة، إلا أنها قد تؤدي أحيانًا إلى استخدام ذاكرة أكثر مما تتوقع. ستتعلم المزيد عن الإغلاقات في دروس لاحقة.

كمبتدئ، لا تحتاج للقلق كثيرًا بشأن تفاصيل إدارة الذاكرة. يجمع جامع القمامة التلقائي في JavaScript معظم الأمور نيابة عنك. لكن مع تقدمك في تعلم JavaScript، فهم هذه المفاهيم يساعدك على كتابة شفرة أكثر كفاءة، خاصةً للتطبيقات الكبيرة أو عند العمل بموارد محدودة.

تذكر، ممارسات البرمجة الجيدة، مثل تجنب المتغيرات العامة قدر الإمكان والانتباه لما تغلق عليه دوالك، تساعد محرك JavaScript على إدارة الذاكرة بكفاءة أكبر.
