# --description--

Форми є фундаментальними для кожного веб-застосунку, оскільки вони дозволяють обробляти вхідні дані користувача, збирати інформацію та запускати дії.

У React форми керуються за допомогою стану або refs, що дає вам повний контроль над їхньою поведінкою та валідацією. Ці два способи керування формами називаються «керованим» і «некерованим» введенням.

Давайте розглянемо, що таке кероване та некероване введення.

Кероване введення — це найбільш «React-подібний» спосіб обробки полів форми. При керованому введенні ви зберігаєте значення поля введення в стані і оновлюєте його через `onChange` події. Це дає вам повний контроль над даними форми та дозволяє миттєво виконувати валідацію і умовне відтворення.

Процес працює так: React підтримує стан форми за допомогою хука `useState`, і ви оновлюєте його при кожній зміні. Коли користувач вводить текст у поле введення, спрацьовує подія `onChange`, оновлює стан, і React повторно відтворює компонент із новим значенням.

```jsx
import { useState } from "react";

function App() {
  const [name, setName] = useState("");

  const handleChange = (e) => {
    setName(e.target.value);
  };

  const handleSubmit = (e) => {
    e.preventDefault();
    console.log(name);
  };

  return (
    <>
      <form onSubmit={handleSubmit}>
        <label htmlFor="name">Your name</label> <br />
        <input value={name} id="name" onChange={handleChange} type="text" />
        <button type="submit">Submit</button>
      </form> 
    </>
  );
}

export default App;
```

Переваги керованого введення включають:

- Миттєвий доступ до даних форми.

- Можливість реалізувати миттєву валідацію.

- Можливість умовно відключати кнопку відправки.

- Можливість програмно контролювати значення введення.

Некероване введення, натомість, частіше зустрічається у традиційних HTML-формах. Замість того, щоб керувати введенням через хук `useState`, некеровані поля в HTML підтримують власний внутрішній стан за допомогою DOM.

Оскільки DOM контролює значення введення, вам потрібно отримувати значення полів введення через ref. Цей підхід вимагає менше коду і працює швидше, бо refs не викликають повторного відтворення React.

Ось приклад некерованого введення:

```jsx
import { useRef } from "react";

function App() {
  const nameRef = useRef();

  const handleSubmit = (e) => {
    e.preventDefault();
    console.log(nameRef.current.value);
  };

  return (
    <form onSubmit={handleSubmit}>
      <label htmlFor="name">Your</label>{" "}
      <input type="text" ref={nameRef} id="name" />
      <button type="submit">Submit</button>
    </form>
  );
}

export default App;
```

Велика перевага некерованого введення — це менша кількість коду. Вони також працюють швидше і здаються більш природними для початківців у React, які знайомі з HTML.

Отже, яке введення обрати — кероване чи некероване?

Використовуйте кероване введення, коли потрібні динамічні оновлення форми, валідація в реальному часі або коли потрібно синхронізувати значення введення зі станом. Воно дає кращий контроль, але вимагає більше повторних відтворень.

Використовуйте некероване введення, коли потрібні простіші форми, доступ до значень лише при відправці або коли ви працюєте з кодом, що не належить React.

Незалежно від того, яке введення ви оберете, дотримуйтеся таких найкращих практик при створенні форм у React:

- Завжди запобігайте стандартній відправці форми.

- Переконайтеся, що виконуєте валідацію перед відправкою.

- Завжди надавайте користувачам чіткий зворотний зв’язок про завантаження, помилки валідації чи інші стани.
