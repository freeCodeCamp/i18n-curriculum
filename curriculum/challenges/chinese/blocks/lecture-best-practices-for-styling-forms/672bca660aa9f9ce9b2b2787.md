# --interactive--

让我们来了解一下在尝试设置特殊输入元素样式时的一些常见问题， 如 `datetime-local` 和 `color` 输入。

这些特殊类型的输入依赖于复杂的伪元素来创建日期和颜色选择器等内容。 这给设置这些输入的样式带来了巨大挑战。  One challenge is that, because the default styling depends entirely on the browser, CSS that makes the picker look right in one browser may produce a very different result in another.

这是一个颜色 `input` 的示例：

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<form>
  <label for="favorite-color">Pick your favorite color:</label>
  <input type="color" id="favorite-color" name="favorite-color">
</form>
```

```css
input {
  padding: 8px 12px;
  margin: 8px 0;
  border-radius: 6px;
  border: 1px solid #ccc;
}

input[type="color"] {
  width: 60px;
  height: 40px;
  padding: 0;
  border: 2px solid #555;
  border-radius: 4px;
  cursor: pointer;
}

```

:::

另一个原因可能是伪元素的复杂性。 考虑一下日期选择器；这里有很多移动部件，伪元素的复杂结构可能会给在正确区域应用样式带来巨大挑战。

这是一个日期输入的示例：

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<form>
  <label for="birthdate">Select your birthdate:</label>
  <input type="date" id="birthdate" name="birthdate">
</form>
```

```css
input {
  padding: 8px 12px;
  margin: 8px 0;
  border-radius: 6px;
  border: 1px solid #ccc;
}

input[type="date"] {
  padding: 6px 10px;
  border: 2px solid #555;
  border-radius: 4px;
  font-size: 14px;
  cursor: pointer;
}

input[type="date"]::-webkit-calendar-picker-indicator {
  background-color: #4CAF50;
  color: white;
  border-radius: 4px;
  cursor: pointer;
}

```

:::

当然，对于这些复杂的元素，手动调整样式时也有可能不小心丢失重要功能。 你可能不仅会丢失焦点状态或选中项等重要指示标识，还有可能完全破坏选择器。  

因此，许多开发人员完全依赖 JavaScript 库或自定义组件，而不是使用浏览器的内置组件。
