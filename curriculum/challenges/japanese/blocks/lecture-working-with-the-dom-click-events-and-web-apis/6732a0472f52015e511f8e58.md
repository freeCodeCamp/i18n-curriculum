# --description--

APIとその重要性について学びましょう。APIはアプリケーションプログラミングインターフェースの略です。APIはソフトウェアアプリケーション同士が効率的に通信し、データを交換するためのルールとプロトコルのセットを確立します。

APIは、開発者がすでに実装されているより単純な構成要素を基に、より複雑な機能を作り出せる仕組みと考えられます。

APIにはさまざまな種類があります。ウェブAPIは特にウェブアプリケーション向けに設計されています。クライアントサイドのJavaScript開発で使われるウェブAPIにはいくつかの種類がありますが、JavaScript自体の一部ではありません。

これらのAPIは主に2つのカテゴリーに分けられます：ブラウザAPIとサードパーティAPIです。

ブラウザAPIはブラウザからデータを公開します。ウェブ開発者はJavaScriptを使ってこのデータにアクセスし、操作できます。

また、ウェブサイトの構造を操作したり、イベントを処理したり、ストレージを扱ったり、ネットワークと通信したりするさまざまな機能へのアクセスも提供します。

よく使われるブラウザAPIの例には以下があります：

HTML要素、そのスタイルや属性を操作するためのDOM API。今後のレッスンでDOMについてさらに詳しく学びます。ウェブ開発の重要な概念です。

ユーザーのデバイスにデータをローカルに保存するためのストレージAPI。

その他にもあります。

ユーザーの現在位置など、ユーザーの操作を扱ったりブラウザからより多くの情報を取得したりするためのさまざまなブラウザAPIがあります。

動画や音声などの他の要素を扱うこともできます。

ブラウザAPIは強力なウェブアプリケーションを構築するための便利なツールです。ブラウザに組み込まれています。

しかし、デフォルトではブラウザに組み込まれていないサードパーティAPIも利用できます。これらを使うには、何らかの方法でそのコードを取得する必要があります。通常、サービスの使い方を説明した詳細なドキュメントが付属しています。

例として、ウェブサイトにインタラクティブな地図を表示できるGoogle Maps APIがあります。

サードパーティのウェブAPIには、天気API、ソーシャルメディアAPI、決済API、データAPI、翻訳APIなどさまざまな種類があります。自分のニーズに合ったAPIを見つけるだけです。

ウェブAPIは、さまざまなサービスやデータソースと連携できるモダンなウェブアプリケーションを構築するために欠かせないツールです。その力を活用する方法を学ぶことで、動的で魅力的なウェブサイトを作るためのツールセットが広がります。
