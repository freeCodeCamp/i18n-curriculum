# --description--

前のレッスンでは、`string`や`boolean`のようなプリミティブ型の扱い方を学びました。しかし、TypeScriptでは既存の型から新しい型を作ることができます。その一例がユニオン型です。

ユニオン型とは、2つ以上の型を組み合わせた型のことです。

基本的な構文は以下の通りです：

```ts
type1 | type2 | type3
```

この概念をよりよく理解するために、いくつか例を見てみましょう。

`id`または`string`のユニオン型を持つ`number`という変数があります：

```ts
let id: string | number = "user_783";
```

値は数値に更新することもでき、これはユニオン型として有効です：

```ts
let id: string | number = 1024;
```

もし`string`や`number`でない別の値を代入しようとすると、TypeScriptはエラーメッセージを表示します：

```ts
// Type 'string[]' is not assignable to type 'string | number'.
let id: string | number = ["Java", "Python"];
```

ユニオン型を扱う際は、すべての型で使えるメソッドを使うことが重要です。そうでないと、TypeScriptは次のようなエラーをスローします：

```ts
function getId(id: string | number) {
  // Property 'toUpperCase' does not exist on type 'string | number'.
  return id.toUpperCase();
}
```

この問題を解決する方法としてユニオンの絞り込みがありますが、それについてやユニオン型が型エイリアスとどのように連携するかは今後のレッスンで学びます。

ユニオン型は`null`や`undefined`の値を扱うときにも役立ちます。

例えば、変数が`string`の値を持つか、まだ値が設定されていなければ`null`である場合の例です：

```ts
let username: string | null;

// Type 'number' is not assignable to type 'string | null'.
username = 42;
```

この場合、`username`は`string`か`null`のどちらかですが、それ以外は許されません。値が存在しないことを意図的に表すときに`null`を使うことがあります。

`undefined`を使った別の例もあります：

```ts
let score: number | undefined;

score = undefined; 

// Type 'string' is not assignable to type 'number | undefined'.
score = "high";
```

この例では、`score`は`number`か`undefined`のどちらかです。値がまだ代入されていない場合に`undefined`を使うことがあります。

`null`や`undefined`とユニオン型を使うことで、値が欠けている可能性を明確にしつつ、TypeScriptが無効な代入を検出できるようになります。
