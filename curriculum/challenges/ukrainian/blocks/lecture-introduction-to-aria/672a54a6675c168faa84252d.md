# --interactive--

Важливо переконатися, що всі користувачі, включно з тими, хто має інвалідність, можуть без проблем отримувати доступ до вебсайтів.

Для людей, які користуються читачами екрана, атрибути `aria-label` та `aria-labelledby` надають важливу інформацію про елементи сторінки, які можуть бути незрозумілими або невидимими.

Давайте розглянемо, що таке атрибути `aria-label` та `aria-labelledby` і яку роль вони відіграють у забезпеченні доступності вебу для людей з порушеннями зору та суміжними інвалідностями.

Ви помітите, що обидва `aria-label` і `aria-labelledby` мають префікс aria. Що це означає? ARIA означає Accessible Rich Internet Applications — набір атрибутів із префіксом `aria-`, який дозволяє розробникам повідомляти призначення елементів допоміжним технологіям. Атрибут `aria-label` — це невидима мітка для інтерактивних елементів. Він додає текстову мітку до елемента, яку можуть прочитати читачі екрана.

`aria-label` особливо корисний для елементів, які не мають видимого тексту, але все одно потребують опису для читачів екрана. Наприклад, кнопки, що містять лише іконки, часто потребують `aria-label`, щоб передати їх призначення.

Ось приклад:

```html
<button aria-label="Search">
  <i class="fa-solid fa-magnifying-glass"></i>
</button>
```

У цьому випадку читач екрана може оголосити цю кнопку як `Search, button`, навіть якщо кнопка містить лише іконку. Атрибут `aria-label` повідомляє читачам екрана, який текст використовувати замість іконки.

Якщо б кнопка містила текст "Search" замість іконки, тоді атрибут `aria-label` не був би потрібен, оскільки текст забезпечував би мітку для кнопки.

Для елементів введення атрибут `aria-label` безпосередньо надає мітку, якщо немає видимої мітки, пов’язаної з введенням.

Атрибут `aria-labelledby` робить те саме, що й атрибут `aria-label`, але замість того, щоб визначати текст безпосередньо в атрибуті, ви використовуєте референс на текст, який уже існує на сторінці. Існуючий текст повинен мати атрибут `id`, який буде використаний як значення референсу в атрибуті `aria-labelledby`.

Ось приклад:

:::interactive_editor

```html
<input type="text" aria-labelledby="search-btn">
<button type="button" id="search-btn">Search</button>
```

:::

У цьому випадку текст кнопки використовується як мітка для поля пошуку. Читачі екрана оголосять введення як щось на кшталт `Search, edit`. Якщо пізніше ви вирішите змінити текст кнопки на `Find`, мітка для введення автоматично оновиться до нового тексту, оскільки вона посилається на текст кнопки.
Також можливо об’єднати кілька значень `id` в одне значення атрибута `aria-labelledby`. Ось як це працює:

:::interactive_editor

```html
<div>
  <span id="volume-label">Volume</span>
  <span id="volume-details">Adjust the volume level</span>
  <input
    type="range"
    min="0"
    max="100"
    value="30"
    aria-labelledby="volume-label volume-details">
</div>
```

:::

Для слайдера читач екрана звертатиме увагу на вміст елементів `volume-label` та `volume-details` і оголосить `Volume Adjust the volume level`.

Ви бачили, що обидва атрибути `aria-label` та `aria-labelledby` допомагають читачам екрана зрозуміти, що роблять елементи. Отже, який з них слід використовувати? Оскільки вони надають однакову функціональність, можна використовувати будь-який, але є кілька переваг використання `aria-labelledby` замість `aria-label`:

- Якщо хтось користується сервісом перекладу для перекладу вмісту вашої сторінки, текст у атрибуті `aria-label` може не завжди перекладатися.
- Використання `aria-labelledby` також допомагає уникнути невідповідності між видимим текстом мітки та невидимою міткою для користувачів читачів екрана, оскільки оновлення видимого тексту автоматично оновить невидиму мітку.
- `aria-labelledby` може значно полегшити програмне створення складних невидимих міток, що складаються з кількох джерел тексту.

Останнє зауваження: не використовуйте `aria-label` і `aria-labelledby` одночасно на одному елементі. У такому випадку невидима мітка для читачів екрана завжди визначатиметься `aria-labelledby`, а `aria-label` буде повністю ігноруватися.
