# --interactive--

`parseFloat()` і `parseInt()` — це два основні методи в JavaScript для перетворення рядків у числа. Ці методи особливо корисні при роботі з вхідними даними користувача або обробці даних, які надходять у форматі рядка, але мають розглядатися як числові значення.

Почнемо з `parseFloat()`. Цей метод аналізує рядковий аргумент і повертає число з плаваючою комою. Він призначений для вилучення числа з початку рядка, навіть якщо рядок містить нечислові символи далі. Пам’ятайте, що числа з плаваючою комою — це числа з десятковою крапкою. Ось як працює `parseFloat()`:

:::interactive_editor

```js
console.log(parseFloat("3.14"));     // 3.14
console.log(parseFloat("3.14 abc")); // 3.14
console.log(parseFloat("3.14.5"));   // 3.14
console.log(parseFloat("abc 3.14")); // NaN
```

:::

Як бачите, `parseFloat()` починає аналіз з початку рядка і продовжує, доки не зустріне символ, який не може бути частиною числа з плаваючою комою. Якщо він не знайде дійсного числа на початку рядка, поверне `NaN` (Not a Number).

`parseInt()`, з іншого боку, аналізує рядковий аргумент і повертає ціле число. Як і `parseFloat()`, він починає з початку рядка, але зупиняється на першому символі, що не є цифрою. Ось як працює `parseInt()`:

:::interactive_editor

```js
console.log(parseInt("42"));       // 42
console.log(parseInt("42px"));     // 42
console.log(parseInt("3.14"));     // 3
console.log(parseInt("abc123"));   // NaN
```

:::

`parseInt()` зупиняє аналіз на першому нецифровому символі, який зустріне. Для чисел з плаваючою комою він повертає лише цілу частину. Якщо не знайде дійсного цілого числа на початку рядка, поверне `NaN`.

Обидва методи мають деякі особливості. Вони ігнорують провідні пробіли:

:::interactive_editor

```js
console.log(parseFloat("  3.14"));  // 3.14
console.log(parseInt("  42"));      // 42
```

:::

Вони обробляють знаки плюс і мінус на початку рядка:

:::interactive_editor

```js
console.log(parseFloat("+3.14"));  // 3.14
console.log(parseInt("-42"));      // -42
```

:::

Варто зазначити, що хоча ці методи потужні, у них є деякі обмеження. Наприклад, вони не обробляють усі формати чисел, такі як наукова нотація, безпосередньо. Для складнішого аналізу може знадобитися використання додаткових технік або бібліотек.

Підсумовуючи, `parseFloat()` і `parseInt()` — це цінні інструменти для перетворення рядків у числа в JavaScript. Розуміння їх роботи та особливостей дозволяє ефективніше працювати з числовими даними у ваших застосунках, особливо при обробці вхідних даних користувачів або зовнішніх джерел.
