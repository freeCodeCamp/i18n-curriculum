# --description--

Rivediamo alcune delle tecniche più importanti per migliorare le prestazioni di JavaScript in modo specifico.

Potrebbe sorprenderti, ma la prima tecnica è usare meno JavaScript possibile.

Se stai sviluppando un sito web statico semplice, potresti riuscire a fare esattamente la stessa cosa senza framework che si basano su JavaScript.

Dovresti anche cercare di rimuovere qualsiasi codice JavaScript inutilizzato e usare soluzioni più semplici, se possibile.

Ad esempio, potresti considerare di usare funzionalità integrate del browser invece di quelle personalizzate. Queste funzionalità integrate includono la validazione dei moduli, il lettore video integrato del browser e le animazioni CSS invece di animazioni JavaScript personalizzate.

Usa il minor numero possibile di animazioni.

Riduci la quantità di manipolazione del DOM nel tuo codice JavaScript, poiché è computazionalmente costosa.

Dividere il tuo codice JavaScript in moduli che eseguono attività critiche e non critiche è utile. In questo modo, potrai precaricare quelli critici il prima possibile e rimandare quelli non critici per renderizzare la pagina il più velocemente possibile.

Precaricare i file non garantisce che saranno disponibili quando li usi, ma il download inizierà prima, quindi il tempo di attesa complessivo dovrebbe essere più breve.

Una volta che il tuo codice è diviso in più file, puoi minimizzarli automaticamente durante il processo di build per ridurne la dimensione e ottimizzare i tempi di caricamento.

Se un listener di eventi non è più necessario, dovresti rimuoverlo:

```js
element.removeEventListener("mousemove", handleMouseMove);
```

In questo esempio, rimuoviamo il listener per l'evento `mousemove`, quindi la funzione `handleMouseMove` non viene più chiamata quando l'evento si attiva. Se mantieni il listener, il programma continuerà a eseguire calcoli non più necessari, influenzando le prestazioni complessive. Questo effetto sarà ancora più evidente se hai più listener attivi contemporaneamente.

E dato che siamo in tema di eventi, è consigliato usare la delega degli eventi quando possibile.

Questo significa che non devi impostare i listener sugli elementi figli individuali. Puoi impostarli direttamente sul genitore e l'evento risalirà fino a trovare un genitore che può gestirlo.

Ottimizzare JavaScript è fondamentale per sviluppare applicazioni web veloci e responsive. In un progetto reale, analizzerai costantemente le prestazioni del tuo sito per trovare aree di miglioramento.
