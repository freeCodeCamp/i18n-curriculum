# --description--

В React ефект — це все, що відбувається поза процесом відтворення компонента. Тобто все, що React не обробляє безпосередньо як частину відтворення інтерфейсу користувача.

Загальні приклади включають отримання даних, оновлення назви вкладки браузера, читання або запис у локальне сховище браузера, визначення місцезнаходження користувача та багато іншого. Ці операції взаємодіють із зовнішнім світом і відомі як побічні ефекти.

React надає хук `useEffect`, щоб ви могли керувати цими побічними ефектами. `useEffect` дозволяє запускати функцію після відтворення або оновлення компонента.

Давайте подивимося, як працює хук `useEffect` і чому він важливий для сучасної розробки на React.

Щоб використати хук `useEffect`, спочатку потрібно його імпортувати:

```js
import { useEffect } from "react";
```

Потім використовуйте його як функцію, ось так:

```js
useEffect(() => {
  // Your side effect logic (usually a function) here
}, [dependencies]);
```

Функція ефекту запускається після відтворення компонента, а необов’язковий аргумент `dependencies` контролює, коли саме запускається ефект.

Зверніть увагу, що `dependencies` може бути масивом «реактивних значень» (стан, пропси, функції, змінні тощо), порожнім масивом або зовсім відсутнім. Ось як усі ці варіанти контролюють роботу `useEffect`:

- Якщо `dependencies` — це масив, що містить одне або кілька реактивних значень, ефект запускається щоразу, коли вони змінюються.

- Якщо `dependencies` — порожній масив, `useEffect` запускається лише один раз при першому відтворенні компонента.

- Якщо ви пропускаєте `dependencies`, ефект запускається щоразу, коли компонент відтворюється або оновлюється.

Наприклад, у цьому застосунку `Counter` ми не передаємо аргумент `dependencies`, тому ефект запускається при відтворенні компонента і щоразу при його оновленні:

```jsx
import { useState, useEffect } from "react";

const Counter = () => {
  const [count, setCount] = useState(0);

  useEffect(() => {
    console.log("Component renders");
  });

  return (
    <div
      style={{
        display: "flex",
        alignItems: "center",
        flexDirection: "column",
      }}
    >
      <h2>{count}</h2>
      <div>
        <button onClick={() => setCount(count + 1)}>Increase</button>
        <button onClick={() => setCount(count - 1)}>Decrease</button>
      </div>
    </div>
  );
};

export default Counter;
```

Але якщо передати порожній масив як залежність, ефект запускається лише при першому відтворенні:

```js
useEffect(() => {
  console.log('Component renders');
}, []);
```

Якщо передати стан `count` як залежність, ефект запускається при першому відтворенні компонента і щоразу, коли `count` змінюється:

```js
useEffect(() => {
  document.title = `The current count is ${count}`;
  console.log('component renders');
}, [count]);
```

Зверніть увагу, якщо налаштований ефект триває поза життєвим циклом відтворення компонента, вам може знадобитися інша функція для «очищення» цього ефекту після відтворення або оновлення компонента.

Наприклад, якщо ваша функція ефекту використовує `setInterval()`, встановлює слухача подій, як `window.addEventListener()`, або підключається до сервера, вам знадобиться функція очищення, щоб відповідно виконати `clearInterval()`, `window.removeEventListener()`, та відключитися від сервера.

Ось синтаксис для повернення функції очищення з хуку `useEffect`:

```js
useEffect(() => {
  // Your side effect logic here
  return () => {
    // Cleanup logic here (optional)
  };
}, [dependencies]);
```

Наприклад, якщо ви додаєте слухача події прокручування, ви можете очистити його, видаливши у функції очищення:

```js
useEffect(() => {
  const handleScroll = () => {
    // Handle scroll logic
  };
  window.addEventListener("scroll", handleScroll);

  return () => {
    window.removeEventListener("scroll", handleScroll);
  };
}, []);
```
