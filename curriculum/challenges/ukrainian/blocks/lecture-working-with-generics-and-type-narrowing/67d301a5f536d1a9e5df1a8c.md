# --description--

Іноді у вас буде значення з широким типом, і потрібно звузити його до більш конкретного типу. Наприклад, можливо, потрібно переконатися, що об’єкт відповідає інтерфейсу, який ви визначили. Або що рядок належить до певного списку значень. Існує кілька способів досягти цього.

Перший — звуження за правдивістю. Розглянемо наш приклад з попереднього уроку:

```js
const email = document.querySelector<HTMLInputElement>("#email");
console.log(email.value);
```

Ми отримуємо помилку компілятора, коли намагаємося отримати доступ до властивості `value` об’єкта `email`, бо `email` може бути `null`. Однак ми можемо використати умовну інструкцію, щоб переконатися, що `email` є правдивим, перш ніж звертатися до властивості:

```js
const email = document.querySelector<HTMLInputElement>("#email");
if (email) {
  console.log(email.value);
}
```

У цьому оновленому прикладі, оскільки `null` не є правдивим значенням, TypeScript може вивести, що `email` ОБОВ’ЯЗКОВО є елементом `input` у межах умовного блоку. Тож помилка компілятора більше не виникає.

Перевірки правдивості також можуть працювати у зворотному напрямку:

```js
const email = document.querySelector<HTMLInputElement>("#email");
if (!email) {
  throw new ReferenceError("Could not find email element!")
}
console.log(email.value);
```

У цьому підході ми викидаємо помилку, якщо `email` є хибним значенням. `null` — це хибне значення. Викидання помилки припиняє логічне виконання цього коду, що означає, що коли ми дійдемо до виклику `console.log()`, TypeScript знає, що `email` не може бути `null`.

Опціональний ланцюжок також є формою звуження типів, за тією ж логікою, що доступ до властивості не може відбутися, якщо значення `email` є `null`.

```js
const email = document.querySelector<HTMLInputElement>("#email");
console.log(email?.value);
```

А як щодо інших типів? Ви також можете звужувати типи за допомогою оператора `typeof`. Розглянемо приклад змінної, яку ми позначили як рядок АБО число:

```js
const myVal = Math.random() > 0.5 ? 222 : "222";
console.log(myVal / 10)
```

У цьому прикладі ми бачимо помилку компілятора, бо не можна виконувати арифметичні операції над рядком. Але ми можемо використати умовну інструкцію, щоб перевірити `typeof` тип змінної `myVal`:

```js
const myVal = Math.random() > 0.5 ? 222 : "222";
if (typeof myVal === "number") {
  console.log(myVal / 10);
}
```

Оскільки ми використали ключове слово `typeof`, TypeScript тепер знає, що `myVal` має бути числом, і ми можемо безпечно виконувати арифметичні операції з ним.

А як щодо складніших типів об’єктів? Якщо об’єкт походить із класу, ви можете використати ключове слово `instanceof` для звуження типу. Повернемося до нашого прикладу `querySelector()`:

```js
const email = document.querySelector("#email");
```

Замість того, щоб передавати загальний тип і казати TypeScript, що це за елемент, ми можемо використати `instanceof` для звуження типу і написати безпечніший код:

```js
const email = document.querySelector("#email");

if (email instanceof HTMLInputElement) {
    console.log(email.value);
}
```

Цей підхід може здатися схожим на попередній, але `instanceof` — це валідація під час виконання, що означає: якщо ми якось помилилися з типом у TypeScript, наш JavaScript-код все одно підтвердить, що `email` є елементом `input`.

Далі розглянемо приклад, де ми отримуємо об’єкт `User` з API і намагаємося вивести інформацію:

```js
interface User {
    name: string;
    age: number;
}

const printAge = (user: User) => 
  console.log(`${user.name} is ${user.age} years old!`)

const request = await fetch("url")
const myUser = await request.json();
printAge(myUser);
```

Ми отримаємо помилку компілятора, коли спробуємо передати `myUser` у функцію, бо, хоча ми знаємо, що API повертає правильний об’єкт, TypeScript цього не знає. І метод `.json()` не приймає загальний тип.

«Простий» спосіб вирішити цю проблему — привести тип:

```js
interface User {
    name: string;
    age: number;
}

const printAge = (user: User) => 
  console.log(`${user.name} is ${user.age} years old!`)

const request = await fetch("url")
const myUser = await request.json() as User;
printAge(myUser);
```

Але щоразу, коли ви приводите тип, ви фактично послаблюєте здатність TypeScript виявляти потенційні помилки. Тож замість приведення типу можна написати захисник типу:

```js
interface User {
    name: string;
    age: number;
}

const isValidUser = (user: unknown): user is User => {
  return !!user && 
    typeof user === "object" &&
    "name" in user &&
    "age" in user;
}
```

Тип повернення тут — ключовий компонент цього визначення функції. Синтаксис `user is User` вказує, що наша функція повертає булеве значення, яке, коли `true`, означає, що значення `user` задовольняє інтерфейс `User`. Потім ми робимо базові перевірки, щоб переконатися, що структура об’єкта `user` відповідає — зверніть увагу на використання звуження за правдивістю (`!!user`) і звуження `typeof`. Це потрібно, бо `typeof null` повертає `"object"`:

```js
interface User {
    name: string;
    age: number;
}

const isValidUser = (user: unknown): user is User => {
  return !!user && 
    typeof user === "object" &&
    "name" in user &&
    "age" in user;
}

const printAge = (user: User) => 
  console.log(`${user.name} is ${user.age} years old!`)

const request = await fetch("url")
const myUser = await request.json() as User;
if (isValidUser(myUser)) {
    printAge(myUser);
}
```

Тепер, якщо ми поєднаємо всю нашу логіку, помилки компілятора більше не виникають, і ми успішно збираємо наш код.

Звуження типів — це потужна функція, яка допомагає писати безпечніший, менш схильний до помилок код — але пам’ятайте, що типи TypeScript не є абсолютно жорсткими, тож уникайте практик, як-от приведення типу значення без його звуження.
