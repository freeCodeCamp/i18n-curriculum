# --description--

Нагадаємо, цей проєкт будується на основі шаблонного коду, скопійованого з <a href="https://github.com/freeCodeCamp/boilerplate-infosec/" target="_blank" rel="noopener noreferrer nofollow">GitHub</a>.

Міжсайтовий скриптинг (XSS) — це поширений тип атаки, коли шкідливі скрипти впроваджуються у вразливі сторінки з метою викрадення конфіденційних даних, таких як сесійні куки або паролі.

Основне правило для зменшення ризику XSS-атаки просте: «Ніколи не довіряйте вхідним даним користувача». Як розробник, ви завжди повинні очищувати всі вхідні дані, що надходять ззовні. Це включає дані з форм, GET-запитів у URL, а також з тіл POST-запитів. Очищення означає, що ви повинні знайти та закодувати символи, які можуть бути небезпечними, наприклад &lt;, >.

Сучасні браузери можуть допомогти зменшувати ризик, застосовуючи кращі програмні стратегії. Часто їх можна налаштовувати через HTTP-заголовки.

HTTP-заголовок X-XSS-Protection — це базовий захист. Браузер виявляє потенційно впроваджений скрипт за допомогою евристичного фільтра. Якщо заголовок увімкнено, браузер змінює код скрипта, нейтралізуючи його. Підтримка цього заголовка все ще обмежена.

# --instructions--

Використайте `helmet.xssFilter()` для очищення вхідних даних, що надходять на ваш сервер.

# --hints--

Підпрограмне забезпечення helmet.xssFilter() має бути правильно підключене

```js
const response = await fetch(code + '/_api/app-info');
if (!response.ok) {
  throw Error(await response.text());
}
const data = await response.json();
assert.include(data.appStack, 'xXssProtection');
assert.property(data.headers, 'x-xss-protection');
```
