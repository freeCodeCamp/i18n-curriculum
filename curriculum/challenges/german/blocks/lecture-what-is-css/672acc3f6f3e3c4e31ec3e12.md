# --interactive--

Wenn Sie mit CSS arbeiten, begegnen Ihnen oft drei verschiedene Arten von Anzeigeverhalten für Elemente: `inline`, `block` und `inline-block`.

Jede dieser Eigenschaften beeinflusst, wie Elemente positioniert werden und wie sie mit anderen Elementen auf der Seite interagieren.

In dieser Lektion konzentrieren wir uns darauf, wie die `inline-block`-Eigenschaft funktioniert und wie sie sich sowohl von inline- als auch von Blockelementen unterscheidet.

Blockelemente verhalten sich wie große Container oder „Blöcke“, die die volle Breite ihres übergeordneten Containers einnehmen. Sie beginnen immer in einer neuen Zeile, und ihre Höhe und Breite können angepasst werden.

Inline-Elemente nehmen nur so viel Platz ein, wie sie benötigen. Sie fließen innerhalb des umgebenden Inhalts und brechen nicht in eine neue Zeile um.

Die `inline-block`-Eigenschaft ist eine Mischung aus diesen beiden Verhaltensweisen. Wie inline-Elemente bleiben `inline-block`-Elemente im Textfluss, ohne eine neue Zeile zu beginnen.

Im Gegensatz zu inline-Elementen können Sie jedoch die Breite und Höhe eines `inline-block`-Elements anpassen, genau wie bei Blockelementen.

Kurz gesagt, der entscheidende Unterschied zwischen `inline` und `inline-block` besteht darin, dass `inline`-Elemente ihre Größe nicht steuern lassen, während `inline-block`-Elemente volle Kontrolle über die Abmessungen ermöglichen und dennoch inline mit anderem Inhalt bleiben.

Schauen wir uns ein Beispiel an.

:::interactive_editor

```html
<link href="styles.css" rel="stylesheet">

<div class="container">
  <span class="inline-block-element element1">Inline-Block</span>
  <span class="inline-block-element element2">Inline-Block</span>
</div>
```

```css
.inline-block-element {
  display: inline-block;
  width: 150px;
  height: 100px;
}

.element1 {
  background-color: lightblue;
}

.element2 {
  background-color: lightgreen;
}
```

:::

Im obigen Beispiel haben wir ein `div` mit der Klasse `container`. Innerhalb dieses `div`-Elements befinden sich zwei `span`-Elemente.

Jedes der span-Elemente ist auf `display:inline-block` gesetzt und hat ebenfalls eine definierte Breite und Höhe.

Die inline-block-Elemente erscheinen nebeneinander, weil sie sich wie inline-Elemente verhalten, aber auch eine festgelegte Breite und Höhe haben, was ihnen blockähnliche Eigenschaften verleiht.

Wenn Sie jedoch die `display: inline-block`-Eigenschaft entfernen, werden weder die `height` noch die `width` angewendet, obwohl Sie sie klar definiert haben.

Hier ist das überarbeitete CSS:

:::interactive_editor

```html
<link href="styles.css" rel="stylesheet">

<div class="container">
  <span class="inline-block-element element1">Span element</span>
  <span class="inline-block-element element2">Span element</span>
</div>
```

```css
.inline-block-element {
  width: 150px;
  height: 100px;
}

.element1 {
  background-color: lightblue;
}

.element2 {
  background-color: lightgreen;
}
```

:::

In diesem Code haben wir die `display: inline-block;`-Eigenschaft entfernt, aber alles andere beibehalten. Hier verhalten sich die `span`-Elemente wieder wie standardmäßige inline-Elemente.

Dadurch werden die angegebenen Breite und Höhe ignoriert, und die Elemente nehmen nur den Platz ein, den ihr Inhalt benötigt.

Zu verstehen, wie `inline-block` funktioniert, ist nützlich, weil Sie es für Layouts verwenden können, die sowohl Ausrichtung als auch Größenkontrolle innerhalb eines durchgehenden Textflusses erfordern.
