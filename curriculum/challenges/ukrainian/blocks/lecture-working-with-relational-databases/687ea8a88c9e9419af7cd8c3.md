# --description--

Давайте розглянемо деякі базові команди, які ви можете використовувати для додавання та перегляду даних у SQL.

Перш за все, щоб переглянути дані, їх потрібно додати. Тож почнемо з додавання.

Приклади, які ми розглянемо в цьому уроці, будуть базуватися на цій таблиці з записами про собак:

```sql
CREATE TABLE dogs(
  id SERIAL,
  name VARCHAR(100),
  age INTEGER
);
```

Таблиця зберігатиме імена та вік різних собак.

Якщо припустити, що ми працюємо з PostgreSQL, ми можемо використати тип даних `SERIAL` для `id`. Оскільки `id` було створено за допомогою `SERIAL`, його значення буде `INTEGER`, починаючи з `1`, і автоматично збільшуватиметься при додаванні нового запису. Тож вам не потрібно передавати значення для нього під час додавання запису.

Почнемо з додавання записів у нашу таблицю `dogs` за допомогою інструкції `INSERT INTO`. Існує багато способів це зробити. Перший — додати один рядок, вказавши стовпці для значень, які будуть додані.

Ви пишете `INSERT INTO`, потім ім’я таблиці (у цьому випадку `dogs`), далі стовпці в круглих дужках, потім `VALUES` і відповідні їм значення в круглих дужках.

Ось приклад: ми додаємо запис про собаку з `name` `'Gino'`, вік якої `age` `3`, пам’ятайте, що `id` буде призначений автоматично:

```sql
INSERT INTO dogs (name, age)
VALUES ('Gino', 3);
```

Це зазвичай найнадійніший варіант, бо ви явно вказуєте стовпці, тож значення будуть призначені цим стовпцям у правильному порядку.

Зверніть увагу, що `'Gino'` потрібно брати в одинарні лапки, бо це тип `VARCHAR`.

Інший варіант — додати запис без вказування стовпців, ось так:

```sql
INSERT INTO dogs
VALUES ('Gino', 3);
```

Цей синтаксис є дійсним SQL, але він схильний до помилок.

Оскільки значення призначаються за порядком стовпців, PostgreSQL спробує додати `'Gino'` у стовпець `id`, який очікує ціле число. Це призведе до помилки, наприклад `invalid input syntax for type integer`.

З цієї причини безпечніше явно вказувати імена стовпців при додаванні даних.

Ви також можете додати кілька записів в одній SQL-команді, розділяючи їх комою.

Тут ми додаємо два записи про собак: один для `'Gino'`, інший для `'Nora'`.

```sql
INSERT INTO dogs (name, age)
VALUES
  ('Gino', 3),
  ('Nora', 2);
```

І так само, як ви можете додавати записи, ви можете робити запити до бази даних. У контексті баз даних запит — це запит на отримання даних.

У SQL інструкція `SELECT` використовується для запиту даних з однієї або кількох таблиць. Ви можете налаштувати запит, щоб отримати саме ту інформацію, яка вам потрібна.

Якщо потрібно отримати всю інформацію з таблиці `dogs`, достатньо написати `SELECT`, потім зірочку (`*`), далі `FROM` і ім’я таблиці `dogs`:

```sql
SELECT *
FROM dogs;
```

У цій команді зірочка — це символ підстановки, який означає «всі стовпці».

Ви отримаєте всі дані всіх записів у таблиці:

```sql
 id | name | age 
----+------+-----
  1 | Gino |   3
  2 | Nora |   2
```

Щоб запросити конкретні стовпці, ви можете написати їхні імена в команді одразу після `SELECT`, розділяючи комою. У цьому прикладі ми запитуємо стовпці `name` і `age` таблиці `dogs`:

```sql
SELECT name, age
FROM dogs;
```

Результат включатиме лише стовпці `name` і `age`:

```sql
 name | age 
------+-----
 Gino |   3
 Nora |   2
```

Іноді потрібно зробити запит даних за певною умовою. Наприклад, щоб отримати всіх собак молодших за 3 роки, можна використати ключове слово `WHERE` і оператор порівняння «менше ніж» (`<`):

```sql
SELECT *
FROM dogs
WHERE age < 3;
```

Результат включатиме лише собак, вік яких менший за 3:

```sql
 id | name | age 
----+------+-----
  2 | Nora |   2
```

Якщо ви просто хочете знайти `age` `'Gino'`, можна використати оператор порівняння «дорівнює» (`=`):

```sql
SELECT age
FROM dogs
WHERE name = 'Gino';
```

Ось результат:

```sql
 age 
-----
  3
```

Це загальні та прості способи додавання та перегляду даних у SQL, але існує багато інших варіантів, деякі з них більш просунуті. Ви дізнаєтеся про них у наступних уроках.
