# --description--

Для цього завдання ви створите програму для гри в камінь, ножиці, папір. Програма, яка вибирає випадковий хід, зазвичай виграє 50% часу. Щоб пройти це завдання, ваша програма має грати матчі проти чотирьох різних ботів, виграючи принаймні 60% ігор у кожному матчі.

Ви будете <a href="https://app.ona.com/?autostart=true#https://github.com/freeCodeCamp/boilerplate-rock-paper-scissors/" target="_blank" rel="noopener noreferrer nofollow">працювати над цим проєктом із нашим стартовим кодом Ona</a>.

Ми все ще розробляємо інтерактивну навчальну частину навчальної програми з машинного навчання. Наразі вам доведеться використовувати інші ресурси, щоб навчитися проходити це завдання.

# --instructions--

У файлі `RPS.py` вам надано функцію з назвою `player`. Функція приймає аргумент, який є рядком, що описує останній хід опонента ("R", "P" або "S"). Функція має повертати рядок, що представляє наступний хід для гри ("R", "P" або "S").

Функція гравця отримає порожній рядок як аргумент для першої гри в матчі, оскільки попереднього ходу немає.

Файл `RPS.py` показує приклад функції, яку вам потрібно оновити. Приклад функції визначено з двома аргументами (`player(prev_play, opponent_history = [])`). Функція ніколи не викликається з другим аргументом, тому він є повністю необов’язковим. Причина, чому приклад функції містить другий аргумент (`opponent_history = []`), полягає в тому, що це єдиний спосіб зберегти стан між послідовними викликами функції `player`. Вам потрібен аргумент `opponent_history` лише якщо ви хочете відстежувати історію ходів опонента.

*Підказка: щоб перемогти всіх чотирьох опонентів, вашій програмі може знадобитися кілька стратегій, які змінюються залежно від ходів опонента.*

## Розробка

Не змінюйте `RPS_game.py`. Пишіть увесь свій код у `RPS.py`. Для розробки ви можете використовувати `main.py` для тестування свого коду.

`main.py` імпортує функцію гри та ботів із `RPS_game.py`.

Щоб протестувати свій код, зіграйте гру за допомогою функції `play`. Функція `play` приймає чотири аргументи:

- двох гравців, які грають один проти одного (гравці насправді є функціями)
- кількість ігор у матчі
- необов’язковий аргумент для перегляду журналу кожної гри. Встановіть його в `True`, щоб бачити ці повідомлення.

```py
play(player1, player2, num_games[, verbose])
```

Наприклад, ось як ви викликали б функцію, якщо хочете, щоб `player` і `quincy` зіграли 1000 ігор один проти одного і ви хочете бачити результати кожної гри:

```py
play(player, quincy, 1000, verbose=True)
```
