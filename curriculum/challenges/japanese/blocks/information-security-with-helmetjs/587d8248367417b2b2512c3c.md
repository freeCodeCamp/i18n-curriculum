# --description--

このプロジェクトは、<a href="https://github.com/freeCodeCamp/boilerplate-infosec/" target="_blank" rel="noopener noreferrer nofollow">GitHub</a> からクローンした以下のスタータープロジェクトを基に構築されています。

HTTP Strict Transport Security (HSTS) は、プロトコルのダウングレード攻撃やクッキーのハイジャックからウェブサイトを保護するウェブセキュリティポリシーです。ウェブサイトが HTTPS 経由でアクセス可能な場合、ユーザーのブラウザーに安全でない HTTP の使用を避けるよう指示できます。Strict-Transport-Security ヘッダーを設定することで、指定した期間内の将来のリクエストに対してブラウザーに HTTPS を使うよう伝えます。これは最初のリクエスト以降のリクエストに対して有効です。

# --instructions--

`helmet.hsts()` を次の 90 日間 HTTPS を使うよう設定してください。設定オブジェクト `{maxAge: timeInSeconds, force: true}` を渡してください。`ninetyDaysInSeconds = 90*24*60*60;` という変数を作成し、`timeInSeconds` に使うことができます。

注意：カスタムウェブサイトで HTTPS を設定するには、ドメインの取得と SSL/TLS 証明書が必要です。

# --hints--

helmet.hsts() ミドルウェアが正しくマウントされていること

```js
const response = await fetch(code + '/_api/app-info');
if (!response.ok) {
  throw Error(await response.text());
}
const data = await response.json();
assert.include(data.appStack, 'hsts');
assert.property(data.headers, 'strict-transport-security');
```

maxAge は 7776000 秒（90 日）に設定してください

```js
const response = await fetch(code + '/_api/app-info');
if (!response.ok) {
  throw Error(await response.text());
}
const data = await response.json();
assert.match(
  data.headers['strict-transport-security'],
  /^max-age=7776000;?/
);
```
