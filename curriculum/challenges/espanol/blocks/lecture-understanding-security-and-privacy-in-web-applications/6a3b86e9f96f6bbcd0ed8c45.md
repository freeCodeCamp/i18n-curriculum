# --description--

HTTPS significa Hypertext Transfer Protocol Secure. Es la versión segura de HTTP.

Como has aprendido en algunas clases anteriores, HTTP transfiere datos entre tu navegador y una página web. El HTTP estándar envía estos datos en texto plano, por lo que cualquiera puede interceptar esos datos en texto plano.

Por esta razón, HTTPS usa un protocolo criptográfico llamado Transport Layer Security (TLS) para asegurar esta transferencia de datos. TLS encripta el canal de comunicación entre dos puntos. Los sistemas antiguos usaban Secure Sockets Layer (SSL), que muchas plataformas de hosting web todavía usan hoy en día.

La encriptación convierte esos datos en un código ilegible que solo el navegador del usuario y el servidor web tienen las llaves para descifrar.

Por ejemplo, cuando escribes una contraseña en una página web, HTTPS la encripta al instante y protege la información de tarjetas de crédito, direcciones personales, credenciales de inicio de sesión y otra información sensible. Los hackers que interceptan los datos solo ven caracteres aleatorios, por lo que no pueden usar la información robada.

Esta protección es vital en redes Wi-Fi públicas. Las redes Wi-Fi abiertas en cafeterías, aeropuertos y otros lugares públicos carecen de seguridad básica, lo que da a los ciberdelincuentes la capacidad de monitorear el tráfico no encriptado en estas redes. HTTPS crea un túnel privado a través de la red pública para mantener tu historial de navegación seguro frente a actores malintencionados cercanos.

HTTPS también proporciona autenticación, que prueba que una página web es legítima. Para usar HTTPS, el dueño de la página debe obtener un certificado digital que una organización confiable emite. El certificado verifica la verdadera identidad del dueño de la página.

Cuando visitas una página web y la dirección comienza solo con "http", es una forma de saber que el dueño de la página no ha obtenido ese certificado, haciendo que la página sea insegura. Los navegadores modernos también te advierten sobre esas páginas.

Finalmente, la integridad de los datos es otro beneficio clave de HTTPS. Esto significa que nadie puede alterar tus datos durante la transmisión. Un hacker no puede inyectar código malicioso en los archivos de la página mientras viajan a tu dispositivo. Recibes exactamente los datos que el dueño de la página envió.

Motores de búsqueda como Google priorizan las páginas HTTPS, ya que las páginas seguras tienen mejor posición en los resultados de búsqueda. Los navegadores modernos también requieren HTTPS para funcionalidades avanzadas como geolocalización, notificaciones push y cámaras web.
