# --description--

En lecciones anteriores, aprendiste sobre Bash y algunos de los comandos comunes que podrías usar. Pero Bash soporta un lenguaje de scripting completo, que puedes usar para realizar tareas automatizadas más complejas.

La escritura de scripts en Bash implica escribir una secuencia de comandos Bash en un archivo, que luego puedes ejecutar con Bash para correr el contenido del archivo.

Veamos cómo puede ser un script de bash:

```bash
#!/bin/bash

servers=("prod" "dev")

for server in "${servers[@]}"
do
  echo "Pulling $server"
  rsync --archive --verbose $server:/etc/nginx/conf.d/server.conf configs/$server.conf
done
```

Este script extrae archivos de configuración de NGINX de una lista de servidores remotos, lo cual puede ser útil para respaldar esas configuraciones en un repositorio local. NGINX es un servidor web popular y aprenderás más sobre cómo trabajar con servidores en un módulo futuro. 

La primera línea se llama "shebang" (`#!`) y le indica al sistema qué intérprete debe usarse para ejecutar el script. Los valores comunes aquí son `/usr/bin/bash`, `/bin/bash` o `/bin/sh`.

La primera línea, `servers=("prod" "dev")`, crea una lista de cadenas que representan los nombres de los servidores. `"prod"` se usa a menudo para referirse al sitio de producción, mientras que `"dev"` se refiere al sitio de desarrollo, que normalmente se utiliza para probar nuevas funciones y cambios antes de que se implementen en el entorno de producción en vivo.

Luego, la lista se recorre con un bucle `for`: la sintaxis `servers[@]` expande la lista en elementos individuales. Cada elemento se asigna a server en cada iteración.

La sentencia `do` indica el inicio del bloque lógico del ciclo, o el código que debe ejecutarse en cada iteración.

`echo "Pulling $server"` interpola el valor actual de la variable server, que en este caso sería prod o dev, y lo muestra en el terminal.

Luego se usa `rsync` para obtener el archivo real del servidor y guardarlo localmente, usando más interpolación para gestionar ese comando.

`done` indica el final del bloque lógico del ciclo.

Hay varias ventajas significativas al entender y aprovechar la escritura de scripts en Bash.

La primera es que casi todos los entornos Unix vienen con Bash disponible, lo que significa que puedes ejecutar tu script tal cual sin tener que configurar ningún entorno de ejecución de lenguaje como Node.js o Python.

La segunda es que tienes acceso a todas las aplicaciones binarias disponibles en el sistema con un esfuerzo mínimo. Si tu sistema tiene instalado doctl, tu script puede ejecutar ese binario. Un binario es un programa compilado que la computadora ejecuta. Cuando tienes un script bash, puedes ejecutar el binario que lanzará el programa.

Y finalmente, la sintaxis que usas para tu script también es ejecutable directamente en el terminal. Esto significa que probar una pequeña porción de tu script se vuelve mucho menos complicado: ¡puedes copiar y pegar esa porción directamente en tu terminal y ver el resultado!

Entender Bash no solo mejorará tu habilidad en scripting, ¡sino que te convertirá en un mago de la línea de comandos!
