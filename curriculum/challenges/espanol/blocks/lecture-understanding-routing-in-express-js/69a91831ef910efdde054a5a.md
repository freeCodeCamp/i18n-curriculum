# --description--

En Express, los parámetros de ruta son valores dinámicos incrustados en la ruta URL. Permiten que tu aplicación capture partes de la URL que pueden cambiar con cada solicitud. Esto es útil para construir páginas dinámicas, como mostrar un usuario específico, una publicación de blog o un producto según la URL.

Aquí tienes un ejemplo:

```javascript
// Route with a parameter in the path
app.get("/post/:postId", (req, res) => {
  const postId = req.params.postId;
  res.send(`Viewing post with ID: ${postId}`);
});
```

Aquí, la ruta escucha solicitudes GET a `/post/:postId`.

* `:postId` es un parámetro de ruta, un marcador de posición en la URL que será reemplazado por un valor real. Por ejemplo, una solicitud a `/post/42` coincide con esta ruta, y `42` se convierte en el valor de `postId`.
    
* Puedes acceder a los parámetros de ruta usando `req.params`. El objeto `req.params` contiene todos los parámetros de ruta, y en este caso, `req.params.postId` contiene el valor de la URL, `42`.
    

Finalmente, `res.send()` envía una respuesta de vuelta al cliente. Así que si la URL es `/post/42`, el cliente ve `Viewing post with ID: 42`.

Como ves, los parámetros de ruta hacen que tus rutas sean flexibles y dinámicas sin necesidad de una ruta separada para cada valor posible.

Ahora que entiendes cómo funcionan los parámetros de ruta, volvamos a un ejemplo de la lección anterior y veamos cómo puedes usar rutas dinámicas para diferentes tipos de menús:

```javascript
app.get("/menu/:category", (req, res) => {
  const category = req.params.category;
  res.send(`You are viewing the ${category} menu!`);
});
```

Y aquí tienes un desglose de cómo funciona esto:

1. Un usuario visita [`http://localhost:3000/menu/drinks`](http://localhost:3000/menu/drinks).
    
2. El servidor coincide esta solicitud con la ruta `/menu/:category` y captura el parámetro `category` como `drinks`.
    
3. El servidor envía esta respuesta de vuelta al navegador: `You are viewing the drinks menu!`
    

De manera similar, si el usuario visita [`http://localhost:3000/menu/food`](http://localhost:3000/menu/food), el servidor responde con lo siguiente: `You are viewing the food menu!`

Los parámetros de ruta te permiten mostrar contenido dinámico basado en la URL. En lugar de crear una ruta separada para cada categoría, puedes usar una sola ruta para manejar múltiples categorías como bebidas, comida, postres y más, simplemente cambiando la URL.

## --text--

¿Cuál es el papel de `:postId` en la ruta `/post/:postId`?

## --answers--

Es una parte estática de la URL.

### --feedback--

`:` indica un parámetro, no un segmento fijo.

---

Captura un valor dinámico de la URL.

---

Es un parámetro de consulta.

### --feedback--

Los parámetros de consulta aparecen después de `?`, no dentro de la ruta.

---

Define un encabezado personalizado.

### --feedback--

Los parámetros de ruta están en la ruta URL, no en los encabezados.

## --text--

¿Cómo puedes usar los parámetros de ruta en tu aplicación?

## --answers--

Para almacenar valores estáticos en la URL.

### --feedback--

Los valores estáticos no se benefician de los marcadores de posición de parámetros.

---

Para capturar dinámicamente partes de la URL para su procesamiento.

---

Para validar la entrada del usuario.

### --feedback--

La validación puede ocurrir después de capturar el valor, pero el parámetro en sí captura datos.

---

Para definir encabezados para las solicitudes.

### --feedback--

Los parámetros de ruta pertenecen a la ruta y no definen encabezados.
