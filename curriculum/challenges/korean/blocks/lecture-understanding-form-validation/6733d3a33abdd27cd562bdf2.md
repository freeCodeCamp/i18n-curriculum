# --interactive--

이벤트에서 `preventDefault()` 메서드의 목적에 대해 배워보겠습니다.

DOM에서 발생하는 모든 이벤트는 기본 동작을 가지고 있습니다. 체크박스의 `click` 이벤트는 기본적으로 해당 체크박스의 상태를 전환합니다. 포커스된 버튼에서 스페이스 바를 누르면 버튼이 활성화됩니다. 이러한 이벤트 객체의 `preventDefault()` 메서드는 그 동작이 일어나지 않도록 정지시킵니다.

예제를 살펴보겠습니다. 사용자가 입력할 수 있는 `input` 요소를 정의해봅시다:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<label>Enter some characters:
  <input type="text">
</label>
```

```css
label {
  display: flex;
  flex-direction: column;
  font-family: Arial, sans-serif;
  font-size: 14px;
  color: #333;
  margin-bottom: 12px;
}

input[type="text"] {
  padding: 8px 12px;
  border: 1px solid #ccc;
  border-radius: 4px;
  font-size: 14px;
  margin-top: 4px;
  width: 100%;
  box-sizing: border-box;
  transition: border-color 0.2s, box-shadow 0.2s;
}

input[type="text"]:focus {
  border-color: #0078d4;
  box-shadow: 0 0 3px rgba(0, 120, 212, 0.5);
  outline: none;
}

```

:::

결과를 보면, 예상대로 입력 필드에 입력할 수 있습니다. 하지만 아마도 그렇게 하고 싶지 않을 수도 있습니다. 대신 사용자가 입력한 문자를 별도의 요소에 표시하고 싶을 수도 있습니다. 먼저, 그 요소를 정의해봅시다:

그다음, 키보드에서 문자가 입력되는 것을 듣기 위해 `keydown` 이벤트에 연결해야 합니다. 여기서는 키보드 정보를 얻어야 하므로 `change`나 `input` 이벤트는 원하지 않는다는 점에 주의하세요.

```js
const input = document.querySelector("input");

input.addEventListener("keydown", (e) => {
    
})
```

`keydown` 이벤트는 키보드 키를 누를 때 발생합니다. 이때, `p` 요소에 문자를 표시해봅시다.

```js
const input = document.querySelector("input");
const output = document.getElementById("output");

input.addEventListener("keydown", (e) => {
  output.innerText = `You pressed the ${e.key} key`;
});
```

:::

`e.key`는 눌린 키의 값을 제공합니다. 예를 들어 `a` 키에 대해 `a` 또는 `Enter` 키에 대해 `Enter`가 있습니다.

위 코드를 사용하면 `input`에 입력할 때 입력한 문자가 `p` 요소에 표시됩니다.

좋지만, `input`에도 문자가 표시되는 것은 원하지 않습니다. 여기서 `preventDefault()` 메서드가 필요합니다. `keydown`의 기본 동작은 입력 필드에 문자를 렌더링하는 것입니다. `e.preventDefault()`를 호출하여 이를 방지합시다:

:::interactive_editor

```html

  <input type="text">
</label>
<p id="output"></p>
<script src="index.js"></script>
```

```css

  align-items: center;
  justify-content: center;
  height: 100vh;
  margin: 0;
}

label {
  font-size: 1.2rem;
  color: #333;
  margin-bottom: 1rem;
  display: flex;
  flex-direction: column;
}

input[type="text"] {
  padding: 0.5rem;
  font-size: 1rem;
  border: 2px solid #ccc;
  border-radius: 5px;
  margin-top: 0.5rem;
  outline: none;
}

input[type="text"]:focus {
  border-color: #007BFF;
}

#output {
  margin-top: 1rem;
  font-size: 1.1rem;
  color: #555;
}

```

```js
const input = document.querySelector("input");
const output = document.getElementById("output");

input.addEventListener("keydown", (e) => {
  e.preventDefault();
  output.innerText = `You pressed the ${e.key} key`;
});
```

:::

이렇게 하면 기본 동작을 막아 자신만의 맞춤 이벤트 처리를 구현할 수 있습니다.

`e.preventDefault` 메서드를 사용해야 하는 또 다른 흔한 예는 폼 제출과 관련이 있습니다. 기본적으로 폼을 제출하면 데이터가 서버로 전송되고 페이지가 새로고침됩니다. `e.preventDefault()`를 사용하면 이것을 막을 수 있습니다.

```js
const form = document.querySelector("form");

form.addEventListener("submit", (e) => {
  e.preventDefault();
  // rest of code goes here
});
```

기본 동작을 막는 것은 사용자가 페이지와 상호작용하는 방식을 더 세밀하게 제어해야 할 때 유용하지만, 접근성과 같은 점도 고려하는 것이 중요합니다 — 맞춤 동작도 기본 동작과 동일한 기능을 제공해야 합니다.
