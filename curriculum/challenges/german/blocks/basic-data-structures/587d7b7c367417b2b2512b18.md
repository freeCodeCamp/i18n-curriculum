# --description--

Objekte sind im Grunde genommen einfach Sammlungen von <dfn>Schlüssel-Wert-Paaren</dfn>. Anders gesagt, sie sind Datenstücke (<dfn>Werte</dfn>), die eindeutigen Bezeichnern namens <dfn>Eigenschaften</dfn> (<dfn>Schlüsseln</dfn>) zugeordnet sind. Schauen Sie sich ein Beispiel an:

```js
const tekkenCharacter = {
  player: 'Hwoarang',
  fightingStyle: 'Tae Kwon Doe',
  human: true
};
```

Der obige Code definiert ein Tekken-Videospielcharakter-Objekt namens `tekkenCharacter`. Es hat drei Eigenschaften, von denen jede einem bestimmten Wert zugeordnet ist. Wenn Sie eine zusätzliche Eigenschaft hinzufügen möchten, wie zum Beispiel „origin“, können Sie dies tun, indem Sie `origin` dem Objekt zuweisen:

```js
tekkenCharacter.origin = 'South Korea';
```

Dies verwendet die Punktnotation. Wenn Sie das `tekkenCharacter`-Objekt betrachten würden, enthält es jetzt die Eigenschaft `origin`. Hwoarang hatte auch markant orangefarbenes Haar. Sie können diese Eigenschaft mit der Klammernotation hinzufügen, indem Sie Folgendes tun:

```js
tekkenCharacter['hair color'] = 'dyed orange';
```

Die Klammernotation ist erforderlich, wenn Ihre Eigenschaft ein Leerzeichen enthält oder wenn Sie eine Variable verwenden möchten, um die Eigenschaft zu benennen. Im obigen Fall ist die Eigenschaft in Anführungszeichen eingeschlossen, um sie als String zu kennzeichnen, und wird genau so hinzugefügt, wie gezeigt. Ohne Anführungszeichen wird sie als Variable ausgewertet, und der Name der Eigenschaft entspricht dem Wert der Variablen. Hier ist ein Beispiel mit einer Variablen:

```js
const eyes = 'eye color';

tekkenCharacter[eyes] = 'brown';
```

Nachdem alle Beispiele hinzugefügt wurden, sieht das Objekt so aus:

```js
{
  player: 'Hwoarang',
  fightingStyle: 'Tae Kwon Doe',
  human: true,
  origin: 'South Korea',
  'hair color': 'dyed orange',
  'eye color': 'brown'
};
```
