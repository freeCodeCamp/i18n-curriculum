# --description--

Hemos hablado un poco sobre Git y GitHub, así que ahora podemos empezar a juntar todo. Para hacer esto, necesitas un repositorio. Pero, ¿qué es un repositorio? Puedes pensar en un repositorio como un contenedor para un proyecto: si estás trabajando en una app, guardarías los archivos de esa app juntos en un repositorio.

Los repositorios pueden estar locales en tu computadora o remotos en un servicio como GitHub. Para crear un repositorio, puedes comenzar con uno local o remoto; el resultado será el mismo, así que elige el enfoque que mejor se adapte a tu flujo de trabajo. Para un flujo de trabajo donde comienzas con un repositorio remoto, puedes crear uno directamente a través del sitio web de GitHub siguiendo estas instrucciones:

Visita <a href="https://github.com" target="_blank">el sitio web de GitHub</a>, asegúrate de haber iniciado sesión y luego haz clic en el ícono de más en la parte superior derecha de la barra de navegación. Verás varias opciones, pero la que te interesa es "New Repository". Selecciónala para abrir la interfaz para crear un nuevo repositorio.

La primera opción que verás allí es la capacidad de elegir una plantilla. Probablemente aún no tengas una de estas configuradas, pero una plantilla es un tipo especial de repositorio que puedes usar como punto de partida para tus nuevos repositorios. Estas son útiles para incluir documentación o plantillas de problemas en todos tus proyectos.

Tu siguiente opción es seleccionar un propietario, puedes elegir tu cuenta de usuario o cualquier organización que poseas o a la que tengas acceso. Luego puedes nombrar el repositorio; en general, querrás usar un nombre que describa de manera concisa el proyecto que estás construyendo. Como práctica recomendada general, debes evitar espacios en el nombre de tu repo. GitHub los reemplazará automáticamente con guiones.

A continuación, puedes opcionalmente agregar una descripción que aparecerá en la página principal del repositorio.

También puedes elegir si el repositorio será público o privado. Un repositorio público puede ser visto y descargado por cualquiera. Un repositorio privado solo puede ser accedido por ti y por quienes les otorgues acceso explícito.

Finalmente, si no seleccionas una plantilla, verás opciones para generar archivos en el repositorio automáticamente. Un archivo README aparecerá en la página principal del repositorio y es una excelente manera de proporcionar documentación y descripciones de tu proyecto. Un archivo `.gitignore` te permite especificar archivos que Git "ignora"; esto es ideal para cosas como dependencias de paquetes, archivos de entorno y otras cosas que no quieres que se guarden. GitHub ofrecerá plantillas para este archivo, que excluyen los elementos comunes para el lenguaje o entorno especificado. Y una licencia es un documento legal que describe quién puede usar tu software y qué pueden hacer con él.

Cuando tengas todas tus configuraciones como te gustan, puedes hacer clic en el botón "Create Repository" para terminar el proceso. ¡Serás redirigido automáticamente a la página principal de tu nuevo repositorio!

Desde aquí, puedes hacer clic en el botón "Code" para obtener opciones para clonar tu repositorio en tu computadora.

Puedes ignorar las pestañas "Codespaces" y "Copilot" para esta lección. En cambio, nos vamos a centrar en las tres opciones de la pestaña "Local". Las dos primeras, HTTPS y SSH, se usan de la misma manera. Usas el comando `git clone` con la URL proporcionada en el cuadro para clonar el repositorio remoto en tu computadora. La diferencia entre estas dos es el método de conexión y autenticación.

Para clonar un repositorio público, en realidad no necesitarás autenticarte. Pero para clonar un repositorio privado, o para hacer push a repositorios públicos o privados, tendrás que estar autenticado. Una URL remota HTTPS se usa para autenticarte mediante tu nombre de usuario y contraseña de GitHub. Sin embargo, debido a preocupaciones de seguridad, esto ya no se soporta, y en su lugar se requiere usar un nombre de usuario y un token de acceso. Aun así, es un método mucho menos seguro que SSH, que utiliza tus claves privadas y públicas. Esto tiene el beneficio adicional de iniciar sesión automáticamente, en lugar de tener que proporcionar un token cada vez. Habrá una lección separada que profundizará en SSH y cómo funciona, pero también puedes leer la documentación de GitHub sobre cómo configurar SSH.

Para clonar el repositorio, puedes ejecutar esto en la terminal:

```sh
git clone git@github.com:your-username/your-repository-name
```

Creará una carpeta en tu computadora con el nombre de tu repositorio y contendrá todos los archivos de tu repositorio dentro de ella. Desde ahí puedes `cd` (cambiar directorio) a tu repositorio clonado con:

```sh
cd name-of-cloned-repo
```

Además, la copia local de tu repo tendrá un remoto apuntando a la copia en GitHub. Para verificar tus remotos puedes ejecutar el siguiente comando:

```sh
git remote -v
```

Otra opción para clonar tu repositorio es usar la CLI de GitHub. Esta herramienta se utiliza para realizar tareas específicas de GitHub sin salir de la línea de comandos. Si no la tienes instalada, puedes obtener instrucciones para hacerlo en la documentación de GitHub, pero deberías tenerla disponible en el gestor de paquetes de tu sistema.

Para clonar un repositorio con la herramienta CLI, usaría:

```sh
gh repo clone your-username/your-repository-name
```

El resultado final de estos comandos es el mismo: tendrás una copia de tu repositorio localmente.

También puedes usar la CLI de GitHub para crear un repositorio:

```sh
gh repo create
```

Si no pasas ningún argumento, el CLI te ofrecerá un asistente interactivo para configurar tu repositorio. Este asistente te dará todas las mismas opciones que la interfaz web. Con esta herramienta, también tendrás la opción de clonar el nuevo repositorio de inmediato, para que no tengas que seguir los pasos manuales descritos anteriormente.

Ahora tienes una comprensión básica de cómo funcionan los repositorios y cómo puedes crear uno para comenzar a versionar tu trabajo correctamente. En lecciones futuras, aprenderás cómo hacer push de un repositorio local existente a GitHub y cómo hacer push de cambios a repositorios remotos.
