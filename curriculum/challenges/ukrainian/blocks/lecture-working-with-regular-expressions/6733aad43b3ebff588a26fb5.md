# --interactive--

Регулярні вирази, або regex, — це функція, яку підтримують багато різних мов програмування.

Регулярний вираз — це спеціальний синтаксис для створення «шаблону», який ви потім можете використовувати для перевірки рядка, вилучення тексту та іншого.

Давайте розглянемо базовий регулярний вираз:

```js
const regex = /freeCodeCamp/;
```

Зверніть увагу, що в JavaScript регулярний вираз визначається шляхом створення шаблону між двома косими рисками (`/`). Намагайтеся не плутати це з коментарем, де текст йде після обох косих рисок.

Цей конкретний регулярний вираз співпаде з текстом `freeCodeCamp` з великою `C` будь-де в рядку. Але як це зробити насправді?

Це приводить нас до першого методу — методу `test()`. Метод `test()` присутній у `RegExp` об’єктів, які представляють регулярний вираз (наприклад, той, що ми щойно визначили).

Метод `test()` приймає рядок, який потрібно перевірити на відповідність регулярному виразу. Наприклад, давайте спробуємо перевірити рядок `e`:

:::interactive_editor

```js
const regex = /freeCodeCamp/;
const test = regex.test("e");
console.log(test);
```

:::

Ви бачите, що ми викликали метод `test()` на нашому новому regex і передали рядок `e` як аргумент.

Метод `test()` повернув `false`, тому що рядок `e` не відповідає шаблону `freeCodeCamp`. Навіть якщо шаблон `freeCodeCamp` містить літеру `e`, це протилежний напрямок від того, як працюють регулярні вирази.

Давайте розглянемо ще кілька прикладів. Приділіть хвилину, щоб подумати над ними:

:::interactive_editor

```js
const regex = /freeCodeCamp/;
console.log(regex.test("freeCodeCamp"));
console.log(regex.test("freeCodeCamp is great"));
console.log(regex.test("I love freeCodeCamp"));
console.log(regex.test("freecodecamp"));
console.log(regex.test("FREECODECAMP"));
console.log(regex.test("free"));
console.log(regex.test("code"));
console.log(regex.test("camp"));
```

:::

Зверніть увагу, що перші три рядки повернули `true`. Ці рядки всі містять текст `freeCodeCamp` точно, десь у рядку.

Лінії 5 і 6 повертають `false`. Хоч вони і містять текст `freecodecamp`, регістр не співпадає. Регулярні вирази за замовчуванням чутливі до регістру.

Нарешті, хоча останні три містять частину шаблону, рядки не містять весь шаблон повністю.

Метод `test()` повертає `boolean`, що вказує, чи співпадає рядок з регулярним виразом взагалі.

Але що, якщо вам потрібна детальніша інформація? Рядки мають метод `match()`. Цей метод приймає регулярний вираз, хоча ви також можете передати рядок, який буде перетворений у регулярний вираз.

`match()` повертає масив співпадінь для рядка. Що таке масив співпадінь? Давайте подивимося:

```js
const regex = /freeCodeCamp/;
const match = "freeCodeCamp".match(regex);
console.log(match);
```

Якщо ми запустимо це, отримаємо масив! Але це дивний масив. У нього є додаткові властивості:

```js
// [
//   'freeCodeCamp',
//   index: 0,
//   input: 'freeCodeCamp',
//   groups: undefined
// ]
```

Властивість `groups` показує будь-які захоплені групи. Ви дізнаєтеся, що це означає, у майбутньому уроці.

Властивість `index` повідомляє, на якому символі в рядку було знайдено співпадіння. У нашому випадку — на початку рядка.

Властивість `input` показує рядок, на якому був викликаний метод `match()`.

Давайте спробуємо ще кілька разів і подивимося, як змінюється результат:

```js
const regex = /freeCodeCamp/;
console.log("freeCodeCamp".match(regex)); // ['freeCodeCamp', index: 0, input: 'freeCodeCamp', groups: undefined]
console.log("freeCodeCamp is great".match(regex)); // ['freeCodeCamp', index: 0, input: 'freeCodeCamp is great', groups: undefined]
console.log("I love freeCodeCamp".match(regex)); // ['freeCodeCamp', index: 7, input: 'I love freeCodeCamp', groups: undefined]
console.log("freecodecamp".match(regex)); // null
console.log("FREECODECAMP".match(regex)); // null
console.log("free".match(regex)); // null
```

Ми вже знаємо, що перші три рядки мають дати співпадіння, тож подивимося на них:

```js
// ['freeCodeCamp', index: 0, input: 'freeCodeCamp', groups: undefined]
// ['freeCodeCamp', index: 0, input: 'freeCodeCamp is great', groups: undefined]
// ['freeCodeCamp', index: 7, input: 'I love freeCodeCamp', groups: undefined]
```

Чи це те, що ви очікували? Ви бачите, як `input` і `index` змінилися залежно від наданого рядка і позиції співпадіння в рядку.

Інші три рядки, які не співпадають, повертають `null` замість масиву.

Тепер, коли ми можемо перевіряти і знаходити співпадіння в рядках за допомогою регулярного виразу, що, якщо ми хочемо замінити вміст рядка? Можливо, хтось написав `freecodecamp` у нижньому регістрі, і ми хочемо автоматично виправити регістр.

Спочатку нам потрібно оновити регулярний вираз, щоб він співпадав із нижнім регістром `freecodecamp`, і створити тестовий рядок:

```js
const regex = /freecodecamp/;
const str = "freecodecamp is rly kewl";
```

Тепер рядки мають метод `replace()`, який приймає два аргументи: регулярний вираз для співпадіння (або рядок, якщо вам не потрібні всі можливості regex) і рядок, яким замінити співпадіння (або функцію, яку виконати для кожного співпадіння).

Отже, якщо ми хочемо замінити наш `freecodecamp` на правильний регістр:

:::interactive_editor

```js
const regex = /freecodecamp/;
const str = "freecodecamp is rly kewl";
const replaced = str.replace(regex, "freeCodeCamp");
console.log(replaced);
```

:::

Ви бачите, що `replace()` повертає оновлений рядок із співпадаючим шаблоном `replaced`.

Регулярні вирази і всі методи, пов’язані з ними, можуть здаватися складними і заплутаними. Але у вас буде можливість дослідити їх детальніше в наступних уроках.
