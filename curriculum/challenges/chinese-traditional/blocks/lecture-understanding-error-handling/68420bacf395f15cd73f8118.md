# --description--

當撰寫 Python 程式碼時，常常會遇到錯誤。理解這些錯誤是快速且高效偵錯你程式碼的關鍵。這些訊息會告訴你到底出了什麼問題，前提是你知道如何閱讀它們。

常見的 Python 錯誤包括 `SyntaxError`、`NameError`、`TypeError`、`IndexError` 和 `AttributeError`。當 Python 無法理解你的程式碼，或當你的邏輯與你正在處理的資料不符時，就會發生這些錯誤。

這裡是一個 `SyntaxError` 的範例：

```py
print("Hello, world!"
# SyntaxError: unexpected EOF while parsing
```

這一行缺少一個閉合括號。Python 引發 `SyntaxError`，因為程式碼不符合正確的語法規則。

這裡是一個 `NameError` 的範例：

```py
print(name)
# NameError: name 'name' is not defined
```

你正在嘗試列印尚未定義的變數。當 Python 找不到該名稱的變數時，會引發 `NameError`。

這裡是一個 `TypeError` 的範例：

```py
5 + "5"
# TypeError: unsupported operand type(s) for +: 'int' and 'str'
```

你不能將整數和字串相加。當你嘗試對不相容的資料型別執行操作時，Python 會引發 `TypeError`。

這裡是一個 `IndexError` 的範例：

```py
my_list = [1, 2, 3]
print(my_list[5])
# IndexError: list index out of range
```

你正在嘗試存取列表中不存在的索引。當你超出範圍時，Python 會引發 `IndexError`。

這裡是一個 `AttributeError` 的範例：

```py
num = 42
num.append(5)
# AttributeError: 'int' object has no attribute 'append'
```

`int` 物件沒有 `append()` 方法。當你嘗試使用該資料型別不存在的方法或屬性時，Python 會引發 `AttributeError`。

識別常見的 Python 錯誤訊息可以幫助你更快地修正問題。與其猜測，不如仔細閱讀錯誤訊息，它通常會告訴你到底出了什麼問題以及該從哪裡著手。
