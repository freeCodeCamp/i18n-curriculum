## JavaScriptのライブラリとフレームワーク

- JavaScriptのライブラリとフレームワークは、よくある問題に対する迅速な解決策を提供し、あらかじめ作られたコードを使うことで開発を加速させます。
- ライブラリは一般的に、DOM操作、イベント処理、AJAXリクエストの管理など、特定のタスクに対する解決策を提供することに重点を置いています。
- JavaScriptのライブラリの例としては、jQueryやReactがあります。
- 一方でフレームワークは、アプリケーション構築のためのより明確な構造を提供します。開発者が従うべき一連のルールや慣例が含まれていることが多いです。
- フレームワークの例にはAngularやReactのメタフレームワークであるNext.jsがあります。
- **シングルページアプリケーション**（SPA）は、単一のHTMLページを読み込み、ユーザーが操作するたびにページ全体をリロードせずにそのページを動的に更新するウェブアプリケーションです。
- SPAはJavaScriptを使ってアプリケーションの状態を管理し、コンテンツをレンダリングします。これは複雑なユーザーインターフェースを構築するための優れたツールを提供するフレームワークを使って行われることが多いです。
- SPAに関するいくつかの問題点は以下の通りです：

  - スクリーンリーダーが動的に更新されるコンテンツに対応しにくい。
  - ユーザーがアプリ内を移動してもURLが変わらないため、特定のページをブックマークしたり戻ったり共有したりするのが難しい。
  - アプリが大きい場合、すべてのアセットを最初に読み込む必要があるため初回の読み込み時間が遅くなることがある。

## React

- Reactはユーザーインターフェースやウェブアプリケーションを構築するための人気のJavaScriptライブラリです。
- Reactのコアコンセプトは、データが変わると独立して更新・レンダリングできる再利用可能なUIコンポーネントを作成することです。
- Reactはアプリケーションの状態に基づいてUIがどのように見えるべきかを記述でき、データや状態が変わると適切なコンポーネントを更新・レンダリングします。

## Reactコンポーネント

- コンポーネントはReactアプリケーションの構成要素で、複雑なユーザーインターフェースを小さく管理しやすい部分に分割できます。
- UIはJSXを使って記述されます。JSXはJavaScriptの構文拡張で、JavaScript内にHTMLのようなコードを書けるようにします。
- コンポーネントは基本的にUIの一部を返すJSの関数またはクラスです。

以下は挨拶メッセージをレンダリングするシンプルなReactコンポーネントの例です：

```jsx
function Greeting() {
  const name = 'Anna';
  return <h1>Welcome, {name}!</h1>;
}
```

コンポーネントを使うには、単に次のように呼び出します：
    
```jsx
  <Greeting />
```

## Reactコンポーネントのインポートとエクスポート

- Reactコンポーネントはあるファイルからエクスポートして、別のファイルにインポートできます。
- 例えば、`City`というファイルに`City.js`というコンポーネントがあるとします。`export`キーワードを使ってコンポーネントをエクスポートできます：

```jsx
// City.js
function City() {
  return <p>New York</p>;
}

export default City;
```

- 別のファイルに`City`コンポーネントをインポートするには、`import`キーワードを使います：

```jsx
// App.js
import City from './City';

function App() {
  return (
    <div>
      <h1>My favorite city is:</h1>
      <City />
    </div>
  );
}
```

- `default`キーワードは`City.js`ファイルからのデフォルトエクスポートとして使われます。

- コンポーネント定義と同じ行でエクスポートすることもできます：

```jsx
export default function City() {
  return <p>New York</p>;
}
```

## Viteを使ったReactプロジェクトのセットアップ

- プロジェクトセットアップツールやCLIは、新しいプロジェクトを素早く簡単に始められるようにし、開発者が設定に悩むことなくコードを書くことに集中できるようにします。
- Viteは人気のあるプロジェクトセットアップツールで、Reactと一緒に使えます。
- ターミナルで次のコマンドを使ってViteで新しいプロジェクトを作成できます：

```bash
npm create vite@latest my-react-app -- --template react
```

このコマンドはViteのReactテンプレートを使って`my-react-app`という名前の新しいReactプロジェクトを作成します。プロジェクトディレクトリには依存関係やコマンドが記載された`package.json`ファイルがあります。

- プロジェクトを実行するには、プロジェクトディレクトリに移動して次のコマンドを実行してください：

```bash
cd my-react-app # path to the project directory
npm install # installs the dependencies listed in the package.json file
```

- 依存関係がインストールされると、プロジェクト内に`node_modules`という新しいフォルダーができます。

- `node_modules`フォルダーは、プロジェクトで必要なすべてのパッケージやライブラリが保存される場所です。

- プロジェクトを実行するには、次のコマンドを使います：

```bash
npm run dev
```

- その後、ブラウザを開いて`http://localhost:5173`にアクセスするとReactアプリケーションが動作しているのが見られます。

- スターターテンプレートのコードを実際に見るには、プロジェクト内の`src`フォルダーに入り、`App.jsx`ファイルを確認してください。

## Reactコンポーネントへのpropsの渡し方

- Reactでは、props（プロパティの略）は親コンポーネントから子コンポーネントへデータを渡す方法です。この仕組みは再利用可能で動的なUI要素を作るために必要です。
- Propsは任意のJavaScriptの値を取れます。親から子にpropsを渡すには、親のJSX内で子コンポーネントを使う際に属性としてpropsを追加します。簡単な例を示します：

```jsx
// Parent component
function Parent() {
  const name = 'Anna';
  return <Child name={name} />;
}

// Child component
function Child(props) {
  return <h1>Hello, {props.name}!</h1>;
}
```

- 複数のpropsをスプレッド演算子`(...)`を使ってオブジェクトから渡すこともできます。例はこちら：
    
```jsx
// Parent component
function Parent() {
  const person = {
    name: 'Anna',
    age: 25,
    city: 'New York'
  };
  return <Child {...person} />;
}
```

このコードでは、スプレッド演算子`{...person}`がpersonオブジェクトを個別のpropsに変換し、Childコンポーネントに渡しています。

## Reactの条件付きレンダリング

- Reactの条件付きレンダリングは、動的なユーザーインターフェースを作るために使います。アプリ内の特定の条件や状態に基づいて異なるコンテンツを表示できます。
- 条件付きレンダリングにはいくつかの方法があります。よく使われるのは三項演算子です。例を示します：

```jsx
function Greeting({ isLoggedIn }) {
  return (
    <div>
      {isLoggedIn ? <h1>Welcome back!</h1> : <h1>Please log in</h1>}
    </div>
  );
}
```

- もう一つの方法は論理AND（`&&`）演算子を使うことです。これは特定の条件が満たされた場合のみコンテンツをレンダリングしたいときに便利です。例はこちら：

```jsx
function Greeting({ user }) {
  return (
    <div>
      {user && <h1>Welcome, {user.name}!</h1>}
    </div>
  );
}
```

上のコードでは、`h1`要素はuserオブジェクトが真の場合にのみレンダリングされます。

直接`if`文を使うこともできます：

```jsx
function Greeting({ isLoggedIn }) {
  if (isLoggedIn) {
    return <h1>Welcome back!</h1>;
  }
  return <h1>Please sign in</h1>;
}
```

## Reactでのリストのレンダリング

- Reactでリストをレンダリングするのはユーザーインターフェース構築でよくある作業です。
- JSの配列`map()`メソッドを使ってアイテムの配列を繰り返し処理し、新しいJSX要素の配列を返すことでリストをレンダリングできます。
- 例えば、名前の配列をリストとしてレンダリングしたい場合は次のようにします：

```jsx
function NameList({ names }) {
  return (
    <ul>
      {names.map((name, index) => (
        <li key={`${name}-${index}`}>{name}</li>
      ))}
    </ul>
  );
}
```

- 各リストアイテムに一意のキーを必ず指定して、Reactが更新やレンダリングを適切に管理できるようにしてください。これらの手法を使えば、柔軟で効率的かつ動的なリストをReactアプリケーションで作成できます。

## Reactのインラインスタイル

- Reactのインラインスタイルは、JavaScriptオブジェクトを使ってJSX要素に直接CSSスタイルを適用できます。
- Reactでインラインスタイルを適用するには、JSX要素のstyle属性を使います。style属性はキャメルケースのCSSプロパティをキーに持つオブジェクトを受け取り、値は対応するCSSの値です。例を示します：

```jsx
function Greeting() {
  return (
    <h1
      style={{ color: 'blue', fontSize: '24px', backgroundColor: 'lightgray' }}
    >
      Hello, world!
    </h1>
  );
}

export default Greeting;
```

- スタイルを別のオブジェクトに抽出して、`style`属性で参照することもできます：

```jsx
function Greeting() {
  const styles = {
    color: 'blue',
    fontSize: '24px',
    backgroundColor: 'lightgray'
  };

  return <h1 style={styles}>Hello, world!</h1>;
}

export default Greeting;
```

- インラインスタイルはpropsや状態に基づいて条件付きでスタイルを適用する動的なスタイリングもサポートします。以下はpropsに基づいて条件付きでスタイルを適用する例です：

```jsx
function Greeting({ isImportant }) {
  const styles = {
    color: isImportant ? 'red' : 'black',
    fontSize: isImportant ? '24px' : '16px'
  };

  return <h1 style={styles}>Hello, world!</h1>;
}

export default Greeting;
```

- 上のコードでは、`color`と`fontSize`のスタイルが`isImportant`プロップに基づいて条件付きで設定されています。
