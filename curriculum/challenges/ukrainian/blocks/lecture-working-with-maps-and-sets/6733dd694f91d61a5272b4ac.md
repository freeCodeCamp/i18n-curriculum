# --description--

У JavaScript `Map` — це вбудований об’єкт, який зберігає пари ключ-значення, подібно до об’єкта. Однак він відрізняється від стандартних об’єктів JavaScript тим, що дозволяє використовувати ключі будь-якого типу, включно з об’єктами та функціями.

`WeakMap` — це колекція пар ключ-значення, подібна до `Map`, але вона використовує слабкі референси для своїх ключів. Ключі мають бути об’єктами, тоді як значення можуть бути будь-якого типу.

Подумайте про відношення між `WeakMap` і `Map` як про відношення між `WeakSet` і `Set`, як ви вивчали на попередньому уроці.

Щоб створити `Map`, використовуйте конструктор `Map()`, перед яким стоїть ключове слово `new`:

```js
const myFirstMap = new Map();
```

Ви можете ініціалізувати Map значеннями:

```js
const myTreesMap = new Map([
 [{ type: 'deciduous' }, 'Maple tree'],
 [['forest', 'grove'], 'Pine tree'],
 [42, 'Oak tree'],
 [true, 'Birch tree'],
 [function() { return 'I am a function key'; }, 'Willow tree'],
]);
```

Якщо ви не ініціалізували `Map` значеннями, ви можете використати метод `set()`, щоб додати їх:

```js
const myTreesMap = new Map();
myTreesMap.set({ type: 'deciduous' }, 'Maple tree');
myTreesMap.set([1, 2], 'Pine tree');
myTreesMap.set(42, 'Oak tree');
myTreesMap.set(true, 'Birch tree');
myTreesMap.set(function() { return "I'm a function key"; }, 'Willow tree');

console.log(myTreesMap);
```

Інші методи, які ви можете використовувати для роботи з `Map`:

- `get(key)` для отримання значення, пов’язаного з вказаним `key`.
- `has(key)` для перевірки, чи існує `key` у `Map`.
- `delete(key)` для видалення пари ключ-значення з `Map`.
- `clear()` для видалення всіх пар ключ-значення.
- `entries()` для перевірки записів `Map` (повертає записи у вигляді `MapIterator`).
- `forEach()` для перебору записів `Map`.
- `size` для визначення кількості пар ключ-значення в `Map`.
    

Існує конструктор `WeakMap()`, який ви можете використати для створення `WeakMap`:

```js
const myFirstWeakMap = new WeakMap();
```

Методи `set()`, `get()`, `has()` і `delete()` також доступні для використання з `WeakMap`. Наприклад, ось як можна призначити елементи `Map` за допомогою методу `set()`:

```js
const myTreeWeakMap = new WeakMap();

myTreeWeakMap.set({ id: 1 }, 'Maple tree');
myTreeWeakMap.set({ id: 2 }, 'Pine tree');
myTreeWeakMap.set({ id: 3 }, 'Oak tree');
myTreeWeakMap.set({ id: 4 }, 'Birch tree');
myTreeWeakMap.set({ id: 5 }, 'Willow tree');

console.log(myTreeWeakMap);
```

Ось як виглядає `WeakMap` у консолі:

```js
/*
WeakMap {{…} => 'Willow tree', {…} => 'Maple tree', {…} => 'Pine tree', {…} => 'Oak tree'}
  [[Entries]]
    No properties
  [[Prototype]]: WeakMap
*/
```

Ось відмінності між `Map` і `WeakMap`:

| **Функція** | **Map** | **WeakMap** |
| --- | --- | --- |
| **Тип ключа** | Ключі можуть бути будь-якого типу даних, включно з рядками, числами, об’єктами або навіть функціями. | Ключі мають бути об’єктами. |
| **Використання** | Використовуйте `Map`, коли потрібно асоціювати дані з будь-яким типом ключа. | Використовуйте `WeakMap`, коли потрібно асоціювати дані лише з об’єктами. |
| **Ітерація** | Ви можете перебирати `Map` за допомогою `forEach()`, `keys()`, `values()` або `entries()`. | `WeakMap` не є ітерованим. |
| **Властивість розміру** | `Map` має властивість `size`, щоб отримати кількість пар ключ-значення. | `WeakMap` не має властивості `size`. |

У таблиці вище наведено узагальнені відмінності, включно з типом ключа, використанням, ітерацією та властивістю розміру. Приділіть хвилину, щоб ознайомитися з вмістом цієї таблиці, щоб дізнатися більше про їхні відмінності.

## --text--

Що з цього відрізняє `Map` від `WeakMap` у JavaScript?

## --answers--

`Map` дозволяє лише об’єкти як ключі, тоді як `WeakMap` дозволяє ключі будь-якого типу.
