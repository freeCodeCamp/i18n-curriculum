# --description--

Das Aktualisieren von Objekten im State in React kann schwierig sein, wenn Sie es gewohnt sind, Objekt-Eigenschaftswerte direkt zu ändern.

React behandelt den State als unveränderlich, das heißt, Sie sollten ihn nicht direkt modifizieren.

Schauen wir uns an, was passiert, wenn Sie versuchen, Objekte im React-State direkt zu ändern, und dann gehen wir auf die korrekte Vorgehensweise ein.

Angenommen, Sie haben ein Objekt im State Ihrer Komponente, das das Profil eines Benutzers darstellt, und Sie möchten, dass der Benutzer sein Alter aktualisiert:

```jsx
import { useState } from "react";

function Profile() {
  const [user, setUser] = useState({
    name: "John Doe",
    age: 31,
    city: "LA",
  });

  // Change user age directly
  const handleAgeChange = (e) => {
    user.age = e.target.value;
    console.log(user);
  };

  return (
    <div>
      <h1>User Profile</h1>
      <p>Name: {user.name}</p>
      <p>Age: {user.age}</p>
      <p>City: {user.city}</p>

      <h2>Update User Age </h2>
      <input type="number" value={user.age} onChange={handleAgeChange} />
    </div>
  );
}

export default Profile;
```

Dieser Code funktioniert nicht, weil Sie die `user.age`-Eigenschaft direkt ändern.

Obwohl `console.log(user)` das neue Alter in der Konsole anzeigt, wird React die Komponente nicht neu rendern, um es in der Benutzeroberfläche anzuzeigen, weil Sie nicht die Setter-Funktion `setUser` verwendet haben.

Um ein Objekt direkt im State zu aktualisieren, müssen Sie die Setter-Funktion verwenden, um ein neues Objekt mit dem aktualisierten Wert zu erstellen. Zum Beispiel:

```js
const handleAgeChange = (e) => {
  setUser({
    age: e.target.value,
  });
};
```

Das funktioniert. Aber wenn Sie sich die Seite jetzt ansehen, wird das Alter des Benutzers aktualisiert, aber die Werte für den Namen und die Stadt gehen verloren.

Das liegt daran, dass das neue Objekt, das Sie an die Setter-Funktion übergeben haben, nur ein Schlüssel/Wert-Paar für `age` enthielt.

Um dies zu verhindern, können Sie zuerst das bestehende Objekt kopieren und dann nur die Eigenschaft aktualisieren, die Sie ändern möchten, in diesem Fall `age`.

Dazu können Sie eine spezielle Funktion, die sogenannte Updater-Funktion, an Ihre Setter-Funktion `setUser` übergeben. Die Updater-Funktion nimmt den ausstehenden State als Argument, hier `prevUser` genannt, und sollte den nächsten State zurückgeben:

```js
const handleAgeChange = (e) => {
  setUser((prevUser) => {
    const updatedUser = { ...prevUser, age: e.target.value };
    console.log('Previous State:', prevUser);
    console.log('Updated State:', updatedUser);
    return updatedUser;
  });
};
```

Wie Sie sehen, erstellen wir ein neues Benutzerobjekt namens `updatedUser`, indem wir mit der Spread-Syntax das ausstehende Benutzerobjekt `...prevUser` kopieren. Dann aktualisieren wir das Alter basierend auf der Formulareingabe und geben `updatedUser` am Ende der Funktion als nächsten State zurück.

Jetzt funktioniert Ihr Projekt wie erwartet, und Aktualisierungen des Alterseingabefelds beeinflussen nicht den Namen oder die Stadt des Benutzers. Sie können auch den vorherigen und den aktualisierten State in der Konsole sehen.

Dies ist die ideale Methode, um ein Objekt im State zu aktualisieren, besonders wenn Sie nicht alle Eigenschaften ändern.

Um die verbleibenden `name` und `city` Eigenschaften zu aktualisieren, können Sie separate Setter-Funktionen schreiben und sie mit den jeweiligen Eingabefeldern verbinden:

```js
const handleNameChange = (e) => {
  setUser((prevUser) => ({
    ...prevUser,
    name: e.target.value,
  }));
};

const handleCityChange = (e) => {
  setUser((prevUser) => ({
    ...prevUser,
    city: e.target.value,
  }));
};
```

Oder Sie können sie in einer einzigen Setter-Funktion zusammenfassen, so:

```js
const handleChange = (e) => {
  const { name, value } = e.target;
  setUser((prevUser) => ({
    ...prevUser,
    [name]: value,
  }));
};
```

Damit das funktioniert, muss jedes Eingabefeld ein `name`-Attribut haben.

Hier ist der vollständige Code jetzt:

```jsx
import { useState } from "react";

function Profile() {
  const [user, setUser] = useState({ name: "John Doe", age: 31, city: "LA" });

  const handleChange = (e) => {
    const { name, value } = e.target;

    setUser((prevUser) => ({...prevUser, [name]: value}));
  };

  return (
    <div>
      <h1>User Profile</h1>
      <p>Name: {user.name}</p>
      <p>Age: {user.age}</p>
      <p>City: {user.city}</p>

      <h2>Update User Age </h2>
      <input type="number" name="age" value={user.age} onChange={handleChange} />

      <h2>Update User Name </h2>
      <input type="text" name="name" value={user.name} onChange={handleChange} />

      <h2>Update User City </h2>
      <input type="text" name="city" value={user.city} onChange={handleChange} />
    </div>
  );
}

export default Profile;
```
