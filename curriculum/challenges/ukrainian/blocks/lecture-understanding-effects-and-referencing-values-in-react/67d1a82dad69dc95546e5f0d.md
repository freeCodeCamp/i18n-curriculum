# --description--

У React можуть бути ситуації, коли потрібен прямий доступ до DOM-елемента. Саме тут стають у пригоді "refs". Refs також можуть зберігати змінні значення, але для цього краще підходить стан.

У чистому JavaScript ви використовували методи `getElementById()` і `querySelector()` для доступу до DOM-елементів. Але в React для доступу до елементів у DOM використовують refs.

Одна з головних відмінностей у тому, що з refs вам не потрібні ідентифікатори, як-от id чи класи, щоб посилатися на елементи.

Отже, як створити і використовувати refs? React надає хук `useRef()`, який дозволяє це робити.

Перший крок — імпортувати цей хук з React:

```js
import { useRef } from "react";
```

Далі потрібно створити змінну, яка триматиме ref із початковим значенням всередині хука `useRef`, наприклад, `sectionRef`, ініціалізовану `null`:

```js
const sectionRef = useRef(null);
```

Останнє, що потрібно зробити — приєднати змінну ref до елемента у вашому JSX, використовуючи атрибут `ref`:

```jsx
<section ref={sectionRef}>
  {/* Section content */}
</section>
```

Якщо вивести ref у консоль, ви побачите, що це об’єкт із властивістю current, у цьому випадку `null`:

```js
console.log(sectionRef); // { current: null }
```

Також можна вивести поточне значення в консоль через властивість `current`, щоб побачити значення безпосередньо:

```js
console.log(sectionRef.current); // null
```

Подальші значення ref залежать від життєвого циклу компонента.

Наприклад, початкове значення `sectionRef` завжди буде `null`, бо саме так його ініціалізували. Після монтування компонента значення ref стане DOM-елементом `section`, до якого ref приєднаний.

Якщо компонент буде демонтуваний, значення ref повернеться до початкового `null`.

Типовий приклад використання ref — встановити фокус на елемент введення під час відтворення або після натискання кнопки.

Ось як це зробити при натисканні кнопки:

```jsx
import { useRef } from "react";

const Focus = () => {
  const inputRef = useRef(null);

  const handleFocus = () => {
    if (inputRef.current) {
      inputRef.current.focus();
    }
  };

  return (
    <div>
      <input ref={inputRef} type="text" placeholder="Enter text" />
      <button onClick={handleFocus}>Focus Input</button>
    </div>
  );
};

export default Focus;
```

У наведеному коді створено `inputRef` і приєднано до елемента `input`. Також є кнопка з подією `onClick`, яка викликає функцію `handleFocus`.

Уся функція `handleFocus` робить виклик методу `focus()` на елементі `input`. Зверніть увагу, що оскільки `input` — це вбудований компонент React, фактичний DOM-елемент `input` встановлюється у властивість `current` ref. Тож викликаєте метод `focus()` з `input.current.focus()`.

Ось кілька найкращих практик, які варто пам’ятати при роботі з refs:

- Використовуйте refs переважно для взаємодії з DOM. Їх також можна застосовувати для змінних даних, але для цього краще підходить стан.

- Не використовуйте refs для базового керування станом — для цього існує `useState`.

- Переконайтеся, що `ref.current` існує перед доступом до його властивостей. Ось як це зробити знову:

```js
const handleFocus = () => {
  if (inputRef.current) {
    inputRef.current.focus();
  }
};
```

Це запобігає помилкам, якщо ref доступний до того, як він приєднаний до DOM, або після його видалення.
