# --interactive--

Wie bei allen Textelementen müssen Sie sicherstellen, dass die Styles, die Sie auf die Texteingabe anwenden, barrierefrei sind. Das bedeutet, die Schriftart muss ausreichend groß sein und die Farbe muss einen ausreichenden Kontrast zum Hintergrund haben.  

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<form class="accessible-form">
  <label for="username">Username</label>
  <input type="text" id="username" name="username" placeholder="Enter your username">
  <button type="submit">Submit</button>
</form>

```

```css
body {
  background-color: #f9fafb;
  color: #222;
  padding: 2rem;
}

.accessible-form {
  max-width: 320px;
  margin: 0 auto;
}

label {
  display: block;
  font-weight: 600;
  margin-bottom: 0.5rem;
}

input[type="text"] {
  width: 100%;
  padding: 0.6rem 0.8rem;
  font-size: 1rem;
  border: 2px solid #555;
  border-radius: 4px;
  background-color: #fff;
  color: #111;
}

input[type="text"]:focus {
  outline: 3px solid #1e90ff;
  border-color: #1e90ff;
}

button {
  margin-top: 1rem;
  padding: 0.6rem 1rem;
  font-size: 1rem;
  background-color: #1e90ff;
  color: #fff;
  border: none;
  border-radius: 4px;
  cursor: pointer;
}

button:hover,
button:focus {
  background-color: #187bcd;
}

```

:::

Der Platzhalter wird jedoch oft vergessen. Es ist wichtig zu bedenken, dass auch dies Text ist und Sie wahrscheinlich das Styling ändern müssen, damit er gut lesbar ist.

Ein weiterer Punkt, den Sie beachten sollten, ist, dass Sie dem Benutzer erlauben sollten, die Eingabe zu verändern. Wenn es sich zum Beispiel um ein Textfeld handelt, sollten Sie nicht die Möglichkeit entfernen, es zu vergrößern oder zu verkleinern. Die Eingabe sollte sich auch entsprechend anpassen, wenn der Benutzer die Seite zoomt.

:::interactive_editor

```html
<form class="accessible-form">
```

Eingabeelemente sind außerdem fokussierbar. Wenn Sie Ihre Styles bearbeiten, sollten Sie darauf achten, dass ein gut sichtbarer Hinweis erhalten bleibt, wenn das Element den Fokus hat, zum Beispiel ein fetter Rahmen.

:::interactive_editor


Außerdem gibt es den Fehlerzustand zu berücksichtigen. Wenn der Text des Benutzers die Eingabevalidierung nicht besteht, gibt es einen visuellen Hinweis, der zeigt, dass etwas nicht stimmt.

Um die Fehlermeldung dynamisch zu aktualisieren, müssten Sie JavaScript verwenden, das Sie in zukünftigen Lektionen kennenlernen werden.

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<form class="accessible-form">
  <label for="email">Email address</label>
  <input 
    type="email" 
    id="email" 
    name="email" 
    placeholder="you@example.com" 
    aria-describedby="email-error"
  >
  <p id="email-error" class="error-message">
    Please enter a valid email address.
  </p>

  <button type="submit">Submit</button>
</form>

<script src="index.js"></script>
```

```css
body {
```

Wenn Sie Ihrem Element Ihre persönliche Note geben, sollten Sie sicherstellen, dass der Fehlerzustand weiterhin gut erkennbar ist und sich nicht mit dem Fokusindikator ähnelt. Mit all dem im Hinterkopf können Sie nun ansprechende Benutzerformulare erstellen.
