## Нормалізація бази даних

Це процес організації реляційної бази даних для зменшення надмірності даних і покращення цілісності.

Її переваги включають:

- Мінімізацію дублювання даних, що економить місце для зберігання і зменшує невідповідності.
- Забезпечення цілісності даних через використання первинних і зовнішніх ключів.
- Полегшення підтримки та розуміння баз даних.

### Нормальні форми

- **1NF (перша нормальна форма)**
  - Кожна клітинка містить одне (атомарне) значення.
  - Кожен запис унікальний (забезпечується первинним ключем).
  - Порядок рядків/стовпців не має значення.
  - Приклад: перемістити кілька номерів телефонів з таблиці `students` у окрему таблицю `student_phones`.

- **2NF (друга нормальна форма)**
  - Відповідає вимогам 1NF.
  - Відсутні **часткові залежності**: кожен неключовий атрибут має залежати від усього складеного первинного ключа.
  - Приклад: розбити таблицю `orders` на `order_header` і `order_items`, щоб уникнути залежності атрибутів лише від частини ключа.

- **3NF (третя нормальна форма)**
  - Відповідає вимогам 2NF.
  - Відсутні **транзитивні залежності**: неключові атрибути не можуть залежати від інших неключових атрибутів.
  - Приклад: перемістити `city_postal_code` у таблицю `cities` замість зберігання його з кожним замовленням.

- **BCNF (нормальна форма Бойса-Кодда)**
  - Відповідає вимогам 3NF.
  - Кожен детермінант (ліва частина функціональної залежності) має бути суперключем.

**Порада**: прагніть до 3NF у більшості проєктів для оптимального балансу між цілісністю та продуктивністю.

## Основні поняття SQL

- SQL — це структурована мова запитів для взаємодії з реляційними базами даних.
- **Основні команди** → `SELECT`, `INSERT`, `UPDATE`, `DELETE`, `CREATE TABLE`, `ALTER TABLE` тощо.
- `Joins` → об’єднує дані з кількох таблиць (`INNER JOIN`, `LEFT JOIN`, `RIGHT JOIN`, `FULL JOIN`).

## Запуск SQL-команд у Bash

Ви можете запускати SQL-команди безпосередньо з командного рядка, використовуючи клієнт інтерфейсу командного рядка `psql` для PostgreSQL або подібні інструменти для інших баз даних.

Наприклад, щоб запустити SQL-файл у PostgreSQL:

```bash
psql -U username -d database_name -c "SELECT * FROM students;"
```

Також можна виконувати MySQL-команди напряму:

```bash
mysql -u username -p database_name -e "SELECT * FROM students;"
```

### Запуск SQL із файлу

```bash
# PostgreSQL
psql -U username -d database_name -f script.sql

# MySQL
mysql -u username -p database_name < script.sql
```

### Вставлення SQL у Bash-скрипт

```bash
#!/bin/bash
DB_USER="school_admin"
DB_NAME="school"

# Insert student data
psql -U "$DB_USER" -d "$DB_NAME" -c \
"INSERT INTO students (name, age, major) VALUES ('Alice', 20, 'CS');"
```

### Використання змінних у SQL

```bash
#!/bin/bash
DB_USER="school_admin"
DB_NAME="school"
STUDENT_NAME="Bob"
AGE=21

psql -U "$DB_USER" -d "$DB_NAME" -c \
"INSERT INTO students (name, age) VALUES ('$STUDENT_NAME', $AGE);"
```

**Порада**: очищуйте змінні, щоб уникнути SQL-ін’єкцій.

## Отримання та використання результатів SQL-запитів у Bash

Коли ви виконуєте SQL-запити через `psql`, ви можете **захоплювати** та **обробляти** отримані значення у своїх Bash-скриптах.

### Захоплення одного значення

```bash
#!/bin/bash
DB_USER="school_admin"
DB_NAME="school"

# Get total student count
STUDENT_COUNT=$(psql -U "$DB_USER" -d "$DB_NAME" -t -A -c \
"SELECT COUNT(*) FROM students;")

echo "Total students: $STUDENT_COUNT"
```

Вихідні дані → 42

### Отримання кількох стовпців

```bash
#!/bin/bash
DB_USER="school_admin"
DB_NAME="school"

# Get top 3 students' names and ages
RESULTS=$(psql -U "$DB_USER" -d "$DB_NAME" -t -A -F"," -c \
"SELECT name, age FROM students LIMIT 3;")

echo "Top 3 students:"
echo "$RESULTS"
```

Вихідні дані

```bash
Alice,20
Bob,21
Charlie,22
```

### Цикл по результатах запиту

```bash
#!/bin/bash
DB_USER="school_admin"
DB_NAME="school"

# Get student names and majors
psql -U "$DB_USER" -d "$DB_NAME" -t -A -F"," -c \
"SELECT name, major FROM students;" | while IFS="," read -r name major
do
    echo "Student: $name | Major: $major"
done
```

Форма вихідних даних

```bash
Student: Alice | Major: CS
Student: Bob   | Major: Math
Student: Carol | Major: Physics
```

## SQL-ін’єкція

Це вразливість веб-безпеки, коли зловмисники вставляють шкідливий SQL-код у поля введення, щоб маніпулювати базою даних.

Це може призвести до ризикованих дій, таких як:

- Обхід автентифікації.
- Крадіжка конфіденційних даних.
- Зміна або видалення записів.

Приклад атаки SQL-ін’єкції:

```sql
SELECT * FROM users WHERE username = ' " " OR "1"="1" -- ' AND password = 'anything';
```

Цей запит поверне всіх користувачів, бо умова `OR "1"="1"` завжди істинна, що дозволяє зловмисникам обійти перевірку входу.

### Запобігання SQL-ін’єкції

1. **Використовуйте підготовлені інструкції**: вони відокремлюють SQL-код від даних, запобігаючи ін’єкціям. Ось приклад (Node.js з pg):

    ```sql
    client.query('SELECT * FROM users WHERE username = $1 AND password = $2', [username, password]);
    ```

2. **Валідація введення**: очищуйте та перевіряйте всі вхідні дані, щоб вони відповідали очікуваним форматам.

3. **Принцип найменших привілеїв**: використовуйте облікові записи бази даних з мінімальними правами, необхідними для застосунку.

**Примітка**: ніколи не надавайте адміністративні права обліковим записам застосунку.

## Проблема N+1

Проблема N+1 виникає, коли застосунок виконує один запит для отримання списку елементів (N), а потім додатковий запит для кожного елемента, щоб отримати пов’язані дані, в результаті чого виконується N+1 запитів.

**Чому це погано**

- Кожен запит додає мережеве та обчислювальне навантаження.
- Багато маленьких запитів повільніші за один оптимізований.

### Приклад шаблону N+1

```sql
-- 1: Get list of orders
SELECT * FROM orders LIMIT 50;

-- N: For each order, get customer
SELECT * FROM customers WHERE customer_id = ...;
```

**Рішення**: використовуйте `JOINs` або інші операції над множинами.

```sql
SELECT 
  orders.order_id,
  orders.product,
  orders.quantity,
  customers.customer_id,
  customers.name,
  customers.email,
  customers.address
FROM orders
JOIN customers 
  ON orders.customer_id = customers.customer_id
WHERE orders.order_id IN (SELECT order_id FROM orders LIMIT 50);
```

Завжди шукайте можливості об’єднати пов’язані дані в один запит.
