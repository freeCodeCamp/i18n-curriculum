## Постійне зберігання

- **Визначення**: Постійне зберігання — це спосіб збереження даних таким чином, щоб вони залишалися доступними навіть після вимкнення живлення або перезавантаження пристрою.

## Створення, читання, оновлення, видалення (CRUD)

- **Створення**: Це процес створення нових даних. Наприклад, у вебзастосунку це може бути, коли користувач додає новий допис у блог.
- **Читання**: Це операція, під час якої дані отримуються з бази даних. Наприклад, коли ви переглядаєте допис у блозі або свій профіль на сайті, ви виконуєте операцію читання, щоб отримати й показати дані, збережені в базі даних.
- **Оновлення**: Це зміна існуючих даних у базі даних. Приклад — редагування допису в блозі або оновлення інформації у вашому профілі.
- **Видалення**: Це операція, яка видаляє дані з бази даних. Наприклад, коли ви видаляєте допис у блозі або акаунт, ви виконуєте операцію видалення.

## HTTP-методи

- **Визначення**: HTTP означає Hypertext Transfer Protocol і є основою для обміну даними в мережі. Існують HTTP-методи, які визначають дії, що можна виконувати над ресурсами в інтернеті. Загальні методи — GET, POST, PUT, PATCH, DELETE.
- **Метод `GET`**: Використовується для отримання даних із сервера.
- **Метод `POST`**: Використовується для відправлення даних на сервер, що створює новий ресурс.
- **Метод `PUT`**: Використовується для оновлення ресурсу шляхом повної заміни.
- **Метод `PATCH`**: Використовується для часткового оновлення ресурсу.
- **Метод `DELETE`**: Використовується для видалення записів із бази даних.

## Властивості `localStorage` і `sessionStorage`

- **Web Storage API**: Цей API надає механізм для браузерів зберігати пари ключ-значення безпосередньо в браузері, що дозволяє розробникам зберігати інформацію, яка може використовуватися між різними перезавантаженнями сторінок і сесіями. Два основні компоненти Web Storage API — це властивості `localStorage` і `sessionStorage`.
- **Властивість `localStorage`**: `localStorage` — це частина Web Storage API, яка дозволяє зберігати дані навіть після закриття вікна браузера або оновлення сторінки. Ці дані залишаються доступними, доки їх явно не видалить застосунок або користувач.
- **Метод `localStorage.setItem()`**: Цей метод використовується для збереження пари ключ-значення в `localStorage`.

```js
localStorage.setItem('username', 'Jessica');
```

- **Метод `localStorage.getItem()`**: Цей метод використовується для отримання значення за заданим ключем із `localStorage`.

```js
localStorage.setItem('username', 'codingRules');

let username = localStorage.getItem('username');
console.log(username); // codingRules
```

- **Метод `localStorage.removeItem()`**: Цей метод використовується для видалення конкретного елемента з `localStorage` за його ключем.

```js
localStorage.removeItem('username');
```

- **Метод `localStorage.clear()`**: Цей метод використовується для очищення всіх збережених даних у `localStorage`.

```js
localStorage.clear();
```

- **Властивість `sessionStorage`**: Зберігає дані, які існують лише під час поточної сесії і видаляються при закритті вкладки або вікна браузера.
- **Метод `sessionStorage.setItem()`**: Цей метод використовується для збереження пари ключ-значення в `sessionStorage`.

```js
sessionStorage.setItem('cart', '3 items');
```

- **Метод `sessionStorage.getItem()`**: Цей метод використовується для отримання значення за заданим ключем із `sessionStorage`.

```js
sessionStorage.setItem('cart', '3 items');

let cart = sessionStorage.getItem('cart');
console.log(cart); // '3 items'
```

- **Метод `sessionStorage.removeItem()`**: Цей метод використовується для видалення конкретного елемента з `sessionStorage` за його ключем.

```js
sessionStorage.removeItem('cart');
```

- **Метод `sessionStorage.clear()`**: Цей метод використовується для очищення всіх даних, збережених у `sessionStorage`.

```js
sessionStorage.clear();
```

## Робота з cookie

- **Визначення**: Cookie, також відомі як веб-куки або браузерні куки, — це невеликі фрагменти даних, які сервер надсилає веббраузеру користувача. Ці куки зберігаються на пристрої користувача і надсилаються назад на сервер з наступними запитами. Куки є важливими для того, щоб вебзастосунки могли підтримувати стан і запам’ятовувати інформацію про користувача, що особливо важливо, оскільки HTTP — це протокол без стану.
- **Сесійні куки**: Ці куки існують лише під час сесії користувача на сайті. Після закриття браузера або вкладки сесійні куки видаляються. Зазвичай їх використовують для таких завдань, як підтримка користувача в системі під час відвідування.
- **Безпечні куки**: Ці куки надсилаються лише через HTTPS, що гарантує, що їх не зможе перехопити зловмисник під час передачі.
- **HttpOnly куки**: Ці куки не можуть бути доступні або змінені JavaScript у браузері, що робить їх більш захищеними від атак типу XSS.
- **Заголовок Set-Cookie**: Коли ви відвідуєте сайт, сервер може надіслати заголовок Set-Cookie у відповіді HTTP. Цей заголовок наказує браузеру зберегти куку з певною інформацією. Наприклад, він може зберігати унікальний ідентифікатор, який допомагає сайту впізнати вас при наступному відвідуванні.
Ви можете вручну встановити куку в JavaScript за допомогою `document.cookie`:

```js
document.cookie = "organization=freeCodeCamp; expires=Fri, 31 Dec 2021 23:59:59 GMT; path=/";
```

Щоб видалити куку, можна встановити її дату закінчення на час у минулому.

```js
document.cookie = "username=; expires=Thu, 01 Jan 1970 00:00:00 GMT; path=/";
```

## Cache API

- **Визначення**: Кешування — це процес збереження копій файлів у тимчасовому сховищі, щоб отримувати до них доступ швидше. Cache API використовується для збереження мережевих запитів і відповідей, що робить вебзастосунки більш ефективними і навіть здатними працювати офлайн. Це частина ширшого Service Worker API і є ключовим для створення прогресивних вебзастосунків (PWA), які можуть працювати в умовах ненадійного або повільного інтернет-з’єднання.
Cache API — це механізм зберігання, який зберігає об’єкти Request і Response. Коли робиться запит до сервера, застосунок може зберегти відповідь і пізніше отримати її з кешу замість нового мережевого запиту. Це зменшує час завантаження, економить трафік і покращує загальний досвід користувача.
- **Сховище кешу**: Використовується для збереження пар ключ-значення HTTP-запитів і відповідних відповідей. Це дозволяє ефективно отримувати раніше запитані ресурси, зменшуючи потребу завантажувати їх з мережі при наступних відвідуваннях і покращуючи продуктивність.
- **Cache-Control**: Розробники можуть визначати, як довго зберігати кешований ресурс і чи потрібно його перевіряти або віддавати безпосередньо з кешу.
- **Підтримка офлайн**: Використовуючи Cache API, можна створювати вебзастосунки з пріоритетом офлайн. Наприклад, PWA може віддавати кешовані ресурси, коли користувач не має підключення до мережі.

## Від’ємні шаблони і клієнтське зберігання

- **Надмірне відстеження**: Це практика збору і зберігання надмірної кількості даних користувача в клієнтському сховищі (наприклад, куки, local storage або session storage) без чіткої, усвідомленої згоди або законної потреби. Часто це включає відстеження поведінки, вподобань і взаємодій користувача на різних сайтах або сесіях, що може порушувати приватність користувача.
- **Браузерний фингерпринтинг**: Техніка відстеження і ідентифікації окремих користувачів на основі унікальних характеристик їхнього пристрою і браузера, а не за допомогою куків чи інших традиційних методів відстеження. На відміну від куків, які зберігаються локально на пристрої користувача, фингерпринтинг збирає різноманітну інформацію, щоб створити унікальний «відбиток» сесії браузера користувача.
- **Зберігання паролів у LocalStorage**: Це очевидний від’ємний шаблон, але зберігання будь-яких чутливих даних, як-от паролі, у local storage становить ризик безпеки. Local Storage не шифрується і до нього легко отримати доступ. Тому ніколи не слід зберігати там чутливі дані.

## IndexedDB

- **Визначення**: IndexedDB використовується для зберігання структурованих даних у браузері. Це вбудовано в сучасні веббраузери, що дозволяє вебзастосункам ефективно зберігати і отримувати JavaScript-об’єкти.

## Cache/Service Workers

- **Визначення**: Service Worker — це скрипт, який працює у фоновому режимі, окремо від вашої вебсторінки. Він може перехоплювати мережеві запити, отримувати доступ до кешу і забезпечувати роботу вебзастосунку офлайн. Це ключовий компонент прогресивних вебзастосунків.
