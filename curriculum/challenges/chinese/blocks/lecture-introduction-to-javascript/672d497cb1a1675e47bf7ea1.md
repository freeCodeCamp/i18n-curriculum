# --interactive--

在 JavaScript 中，变量是存储数据的容器，你可以在整个程序中访问和修改这些数据。

你可以把变量想象成存放值的盒子。 有了变量，你就可以跟踪数值或文本等内容，并在程序中需要时随时调用这些值。

在 JavaScript 中声明变量的一种方法是使用 `let` 关键字。你将在后续课程中学习更多关于 `let` 关键字以及声明变量的其他方法。

下面是一个使用 `let` 声明名为 `age` 的变量的示例：

```js
let age;
```

目前，`age` 变量没有被赋值。如果你尝试使用它，它将返回 `undefined`，这意味着它没有值。

这是一个示例。 

**注意**：`console.log()` 是一个将信息输出到控制台的函数，控制台是你网页浏览器中用于调试代码的部分。你将在后续课程中学习更多关于 `console.log()` 的内容。此外，`//` 符号用于在你的代码中添加评论。评论是给你自己或其他程序员的注释，代码运行时会被忽略。

:::interactive_editor
 
```js
let age;
console.log(age); // undefined
```

:::

要给变量赋值，需要使用赋值运算符，如下所示：

```js
let age = 25;
```

现在当你使用 `age` 变量时，它将返回 `25` 的值。

:::interactive_editor

```js
let age = 25;
console.log(age); // 25
```

:::

赋值操作符看起来像等号（`=`），但它不检查相等性。你将在后续课程中学习用于检查相等性的正确操作符。

赋值操作符用于为变量赋值。 为变量赋值的过程称为初始化。

使用 `let` 关键字声明变量的一个好处是，你可以为变量重新赋值。 在程序设计中，重新赋值是指给一个已有值的变量赋予一个新值。

这是重新赋值 `age` 变量的示例。

:::interactive_editor

```js
let age = 25;
console.log(age); // 25
age = 30;
console.log(age); // 30
```

:::

现在 `age` 变量的值为 `30`。 请注意，`let` 关键字无需再次使用，因为 `age` 变量已被声明，所以无需再次声明。

使用重新赋值时，只需引用变量名即可。 重新赋值非常有用，因为它允许你在程序运行时更新和更改变量中存储的值。 游戏中的积分更新就是一个很好的例子。

变量命名看似简单，但要确保代码的可读性和功能性，还需要一些规则和最佳实践。

变量名应说明数据所代表的含义。 例如，与其使用 `x` 这样的名称，就不如使用 `age` 或 `points` 这样描述性更强的名称，这样代码会更容易理解。

```js
// Bad variable names
let x = 10;
let y = "John";

// Good variable names
let age = 10;
```

JavaScript 中的变量必须以字母、下划线（`_`）或美元符号开头（`$`）。 它们不能以数字开头。

```js
// Valid variable names
let age;
let _score;
let $total;

// Invalid variable names
let 1stPlace; // starts with a number
```

变量名是区分大小写的，也就是说，全部是小写字母的 `age` 和带有大写字母 `A` 的 `Age` 是不同的变量。

:::interactive_editor

```js
let age = 25;
let Age = 30;
console.log(age); // 25
console.log(Age); // 30
```

:::

这就是为什么坚持使用像驼峰式大小写这样一致的命名约定很重要。驼峰式大小写是指第一个单词全部小写，后续每个单词首字母大写。

下面是一个使用小驼峰命名法命名变量的示例：

```js
let thisIsCamelCase;
let anotherExampleVariable;
let freeCodeCampStudents;
```

JavaScript 中有些关键字不能用作变量名，例如 `let`、`const`、`function` 或 `return`，因为它们是为语言本身保留的。

你还应避免在变量名中使用感叹号（`!`）或 @（`@`）符号等特殊字符。 变量名最好使用字母、数字、下划线或美元符号，以保持可读性。

遵循这些指导原则，你的代码就会更加整洁，随着复杂性的增加也更易于管理。
