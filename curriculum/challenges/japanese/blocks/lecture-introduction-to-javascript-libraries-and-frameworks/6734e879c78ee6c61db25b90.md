# --description--

コンポーネントはReactアプリケーションの構成要素であり、開発者が複雑なユーザーインターフェースをより小さく管理しやすい部分に分割できるようにし、大規模なアプリケーションの開発と保守を容易にします。

Reactのコンポーネントには、関数コンポーネントとクラスコンポーネントの2種類があります。現代のReactでは、開発者は関数コンポーネントを使用し、今日見ていくすべての例も関数コンポーネントです。

より高いレベルで考えると、コンポーネントはUIを記述する要素を返すJavaScript関数のようなものと考えられます。

このUIはJSXを使って記述されます。JSXはJavaScriptの構文拡張で、HTMLに似ていますが、より宣言的な方法でUI要素を書くことができます。

Reactコンポーネントの例を見てみましょう：

```jsx
function Greeting() {
  const name = "John"
  {/* The result will be Hello John*/}
  return <h1 className="title">Hello {name}</h1>;
}
```

この例では、`Greeting`というコンポーネントを定義しています。`{}`タグ内の波括弧`h1`はJavaScriptの式を埋め込むことを可能にし、`name`要素内で`h1`変数にアクセスできます。

また、`className`要素に`title`という`h1`を適用しています。

しかし、なぜ通常のHTML要素のように`className`ではなく`class`を使っているのでしょうか？

これはJavaScriptで`class`が予約語だからです。そのため、代わりに`className`を使う必要があります。

JSX内で結果を示すコメントも使っています。通常のコメント構文はこのように使えますが、機能させるには波括弧で囲む必要があることに注意してください：

```jsx
{/* Block Comments */}
```

もう一つ気づいたかもしれませんが、コンポーネント名の最初の文字は大文字を使っています。なぜすべて小文字にできないのでしょうか？

これはReactが大文字で始まるコンポーネントをカスタムコンポーネントとして扱い、小文字の要素は組み込みのHTML要素とみなすためです。

Reactが`<div>`や`<span>`のような小文字のタグに出会うと、それを標準のHTML要素とみなします。しかし、コンポーネント名が大文字で始まる場合、Reactはそれをユーザー定義のコンポーネントとして扱い、適切にレンダリングします。この区別により、ReactはネイティブのHTMLタグと自分で作成したコンポーネントを区別できます。

この`Greeting`コンポーネントをアプリケーションで使うには、次のように書きます：

```jsx
<Greeting />
```

これにより、ページに`h1`というテキストを持つ`Hello John`要素がレンダリングされます。しかし、ここで構文をよく見てください。コンポーネントを使うときは、スラッシュで閉じてから大なり記号で終わっています。

JSXでは、すべてのタグやコンポーネントの使用は明示的に閉じる必要があります。子要素がない場合は、このように明示的に閉じる必要があります：

```jsx
<Greeting /> {/* /> is required */}
```

これまで単一の`h1`要素のレンダリング方法だけを見てきましたが、実際には複数の要素をレンダリングできます。

次の例のコードを見てみましょう：

```jsx
function Greeting() {
  const name = "John";
  {/* This will throw an error */}
  return <h1>Hello {name}</h1> 
  <p>Nice to meet you.</p>
}
```

`Nice to meet you`の別の文を追加しようとしていますが、ページに正しくレンダリングされません。

代わりにエラーメッセージが表示されます。エラーメッセージは`Adjacent JSX elements must be wrapped in an enclosing tag.`と言っています。

このエラーが出る理由は、複数の兄弟要素は親要素でラップする必要があるためです。`h1`と`p`要素を単純な`div`でラップすることもできますが、エラーを抑制する別の方法もあります。

Reactフラグメントは要素をグループ化するために使われます。修正した例は次のようになります：

```jsx
function Greeting() {
  const name = "John";
  return (
    <Fragment>
      <h1>Hello {name}</h1>
      <p>Nice to meet you.</p>
    </Fragment>
  );
}
```

空のJSXタグを使うこともでき、これはフラグメントの省略形として機能します：

```jsx
function Greeting() {
  const name = "John";
  return (
    <>
      <h1>Hello {name}</h1>
      <p>Nice to meet you.</p>
    </>
  );
}
```
