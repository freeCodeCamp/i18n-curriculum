# --description--

Die letzte Aufgabe war ein Schritt näher an den Grundsätzen der funktionalen Programmierung, aber es fehlt noch etwas.

Wir haben den Wert der globalen Variable nicht verändert, aber die Funktion `incrementer` würde ohne die globale Variable `fixedValue` nicht funktionieren.

Ein weiterer Grundsatz der funktionalen Programmierung ist, immer Ihre Abhängigkeiten explizit zu deklarieren. Das bedeutet, wenn eine Funktion davon abhängt, dass eine Variable oder ein Objekt vorhanden ist, dann übergeben Sie diese Variable oder dieses Objekt direkt als Argument an die Funktion.

Dieser Grundsatz hat mehrere gute Folgen. Die Funktion ist leichter zu testen, Sie wissen genau, welche Eingabe sie erwartet, und sie hängt von nichts anderem in Ihrem Programm ab.

Das gibt Ihnen mehr Sicherheit, wenn Sie Code ändern, entfernen oder neuen Code hinzufügen. Sie wissen, was Sie ändern können oder nicht, und Sie sehen, wo mögliche Fallen lauern.

Schließlich würde die Funktion immer dasselbe Ergebnis für denselben Satz von Eingaben liefern, egal welcher Teil des Codes sie ausführt.

# --instructions--

Lassen Sie uns die Funktion `incrementer` aktualisieren, um ihre Abhängigkeiten klar zu deklarieren.

Schreiben Sie die Funktion `incrementer` so, dass sie ein Argument annimmt und dann ein Ergebnis zurückgibt, nachdem sie den Wert um eins erhöht hat.

# --hints--

Ihre Funktion `incrementer` sollte den Wert von `fixedValue` (das ist `4`) nicht verändern.

```js
assert(fixedValue === 4);
```

Ihre Funktion `incrementer` sollte ein Argument annehmen.

```js
assert(incrementer.length === 1);
```

Ihre Funktion `incrementer` sollte einen Wert zurückgeben, der um eins größer ist als der Wert von `fixedValue`.

```js
const __newValue = incrementer(fixedValue);
assert(__newValue === 5);
```
