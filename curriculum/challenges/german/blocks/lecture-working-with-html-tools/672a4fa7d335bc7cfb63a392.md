# --description--

HTML ist eine sehr nachsichtige Sprache – Elemente werden weiterhin gerendert, selbst wenn Sie Fehler machen, wie zum Beispiel das Vergessen eines schließenden Tags.

Angenommen, Sie haben ein `h2`-Element ohne schließendes Tag:

```html
<h1>Article Topic</h1>
<h2>Subheading 1 </h2>
<h2>Subheading 2 </h2>

<!-- This h2 does not have a closing tag -->
<h2>Subheading 3
```

Das `h2` ohne schließendes Tag wird trotzdem korrekt gerendert. Das liegt daran, dass Browser einen Parsing-Algorithmus verwenden, der häufige Fehler behandelt und versucht, HTML so genau wie möglich nach der Absicht des Autors zu rendern.

Das kann aber manchmal nach hinten losgehen. Fügen wir ein paar Absätze unter den vorhandenen Überschriften der Ebene 2 im Code hinzu:

```html
<h1>Article Topic</h1>
<p>Lorem ipsum dolor sit amet consectetur adipisicing elit. Maiores, nisi.</p>

<h2>Subheading 1 </h2>
<p>Lorem, ipsum dolor sit amet consectetur adipisicing elit. At, doloremque.</p>

<h2>Subheading 2 </h2>
<p>Lorem ipsum dolor sit amet consectetur adipisicing elit. Unde, placeat.</p>

<!-- This h2 does not have a closing tag -->
<h2>Subheading 3
<p>Lorem ipsum dolor, sit amet consectetur adipisicing elit. Tempore, illum.</p>
```

Dadurch wird das Absatz-Element unter dem `h2` ohne schließendes `h2`-Tag als Überschrift der Ebene 2 gerendert. Deshalb brauchen Sie einen HTML-Validator.

Ein HTML-Validator ist ein Werkzeug, das die Gültigkeit Ihres HTML-Codes anhand der Standard-HTML-Spezifikationen überprüft. Er hilft Ihnen, Fehler und Warnungen in Ihrem HTML-Code zu erkennen, sodass Ihre Webseiten korrekt strukturiert und webstandardkonform sind.

Die Verwendung eines HTML-Validators bringt nicht nur Ihnen und Ihren zukünftigen Code-Reviews Vorteile, sondern auch allen anderen, die Ihren Code durchsehen, wie Ihren Teamkolleginnen und Teamkollegen oder Open-Source-Beiträgern.

Es gibt mehrere HTML-Validatoren, die Sie verwenden können. Der am weitesten verbreitete ist der `w3.org`-Markup-Validierungsdienst.

Wenn Sie die Seite [`validator.w3.org`](https://validator.w3.org/) besuchen, können Sie auf die `Validate by Direct Input`-Schaltfläche klicken und Ihren HTML-Code einfügen.

Wenn Sie auf die `Check`-Schaltfläche klicken, wird eine Ergebnisliste mit den zu behebenden Fehlern angezeigt.

Ein weiterer HTML-Validator, den Sie verwenden können, ist [`jsonformatter.org`](https://jsonformatter.org/).

Sie können Ihren HTML-Code in den ersten Editor kopieren und einfügen, und wenn Sie auf die `Validate`-Schaltfläche klicken, werden Ihnen alle Fehler in Ihrem Code angezeigt.
