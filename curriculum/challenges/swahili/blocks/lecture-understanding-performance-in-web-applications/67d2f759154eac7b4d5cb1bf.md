# --description--

Kama tunavyojua, utendaji unaoonekana si tu kuhusu kasi halisi ya tovuti — ni kuhusu jinsi inavyohisi kwa watumiaji. Kwa mfano, je, umewahi kutumia tovuti ambayo ilihisi polepole, ingawa haikuwa? Hiyo ni utendaji unaoonekana — jinsi tovuti inavyohisi kwa mtumiaji.

Marekebisho madogo yanaweza kuleta tofauti kubwa katika uzoefu wa mtumiaji. Hebu tuchunguze njia za vitendo za kuboresha utendaji unaoonekana!

Spinners zinaweza kufanya watumiaji wahisi kama wanangojea kwa muda mrefu zaidi kuliko walivyo kweli. Badala yake, jaribu skrini za mfupa (skeleton screens) — viwakilishi vya kijivu vinavyofanana na maudhui halisi na hujaza taratibu:

```html
<div class="skeleton"></div>
<script>
  setTimeout(() => {
    document.querySelector(".skeleton").innerHTML = "✅ Content Loaded!";
  }, 2000);
</script>
```

Watumiaji wanahisi kama ukurasa unajazwa hatua kwa hatua, badala ya kusubiri gizani.

Mbali na hili, watumiaji hawapaswi kusubiri kila kitu kijazwe kabla ya kuanza kuingiliana. Jaza maudhui muhimu kwanza, kisha jaza mengine nyuma ya pazia. Tuliziona hizi awali kwa kutumia lazy loading:

```html
<img src="placeholder.jpg" data-src="real-image.jpg" class="lazy">
<script>
  document.addEventListener("DOMContentLoaded", function () {
    const lazyImages = document.querySelectorAll("img.lazy");
    lazyImages.forEach(img => {
      img.src = img.dataset.src;
    });
  });
</script>
```

Hii hufanya ukurasa uhisi kuwa unajibu, hata kama si kila kitu kimejazwa bado.

Sasa - je, tovuti yako ingeweza kutabiri kitakachobofyiwa na mtumiaji baadaye? Kwa preloading, unaweza kupata ukurasa unaofuata kabla ya mtumiaji kuomba:

```html
<link rel="preload" href="next-page.html" as="document">
```

Hii hupunguza muda wa kuonekana wa mzigo karibu sifuri wakati watumiaji wanapotembea.

Na mwisho, watumiaji hawapendi kutokuwa na uhakika. Wanapobofya kitufe, thibitisha kitendo chao mara moja:

```html
<button onclick="this.innerText='⏳ Processing...'; setTimeout(() => this.innerText='✅ Done!', 2000);">
  Click Me
</button>
```

Hii ina maana kwamba, hata kama kuna kuchelewa, mtumiaji anahisi kama kuna kitu kinaendelea.

Kwa hiyo, kwa hitimisho, kuboresha utendaji unaoonekana si tu kuhusu kasi halisi, ni kuhusu jinsi kasi inavyohisi kwa mtumiaji. Tumia skrini za mfupa, panga maudhui muhimu kwanza, fanya preloading kwa busara, na toa mrejesho wa papo hapo ili kufanya tovuti yako ihisi kuwa ya kasi zaidi.
