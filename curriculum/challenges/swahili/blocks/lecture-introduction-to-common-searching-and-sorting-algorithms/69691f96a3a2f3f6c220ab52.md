# --description--

Kutatua orodha ya vitu ni jambo la kawaida katika sayansi ya kompyuta. Kuna algoriti kuu mbili unazopaswa kujua kuhusu utaftaji: utaftaji wa linear na utaftaji wa binary.

Utaftaji wa linear huanza mwanzoni mwa orodha na hurudia kila kipengee hadi upate thamani unayotafuta.

Ikiwa thamani unayotafuta inapatikana, kielezo ambacho iko kwenye orodha hurudishwa. Ikiwa thamani haipatikani, `-1` hurudishwa. Tunarudisha `-1` kwa sababu si kielezo halali katika lugha nyingi za programu.

Hapa ni jinsi msimbo wa utaftaji wa linear unavyoonekana:

```javascript
function linearSearch(arr, target) {
    for (let i = 0; i < arr.length; i++) {
        if (arr[i] === target) {
            return i;
        }
    }
    return -1;
}
```

Ikiwa orodha tutakayotafuta ni `[13, 4, 7, 9, 10]` na thamani unayotafuta ni `9`, kitendakazi kingerudisha `3` kwa sababu `9` iko kwenye kielezo `3`.

Ikiwa tutabadilisha thamani unayotafuta kuwa `5`, kitendakazi kingerudisha `-1` kwa sababu `5` haipo kwenye orodha.

Ingawa hii ni algoriti rahisi, siyo yenye ufanisi zaidi. Ikiwa una orodha kubwa ya vitu, utaftaji wa linear unaweza kuchukua muda mrefu kupata thamani unayotafuta.

Ugumu wa muda wa utaftaji wa linear ni `O(n)` kwa sababu muda unaochukua kutafuta kwenye orodha huongezeka kwa mstari kulingana na ukubwa wa orodha.

Ugumu wa nafasi wa utaftaji wa linear ni `O(1)` kwa sababu hauhitaji nafasi ya ziada kutafuta kwenye orodha.

Utaftaji wa binary ni algoriti yenye ufanisi zaidi kwa kutafuta kwenye orodha kubwa ya vitu. Sharti hapa ni kwamba orodha lazima iwe imepangwa kwa mpangilio wa kuongezeka.

Utaftaji wa binary hufanya kazi kwa kugawanya orodha nusu na kuangalia kama thamani unayotafuta iko katikati ya orodha. Ikiwa thamani iko katikati ya orodha, kielezo cha thamani hiyo hurudishwa. Vinginevyo, algoriti huangalia kama thamani iko nusu ya kushoto au nusu ya kulia ya orodha.

Inaendelea kugawanya sehemu zilizobaki za orodha kuwa nusu hadi thamani lengwa ipatikane. Ikiwa thamani lengwa haipo katika orodha, inarudisha `-1`.

Hapa ni jinsi msimbo wa utaftaji wa binary unavyoonekana:

```javascript
function binarySearch(arr, target) {
    let low = 0;
    let high = arr.length - 1;

    while (low <= high) {
        const mid = Math.floor((low + high) / 2);

        if (arr[mid] === target) {
            return mid;
        } else if (arr[mid] < target) {
            low = mid + 1;
        } else {
            high = mid - 1;
        }
    }

    return -1;
}
```

Huanza kwa kutambua kielezo cha `low` na `high`. Hii inawakilisha safu ya orodha tunayotafuta ndani yake.

Kisha tunakagua hali ya `low` kuwa ndogo au sawa na `high`. Ikiwa `low` ni kubwa kuliko `high`, tumetafuta orodha yote na thamani haijapatikana. Katika hali hiyo tunasimamisha utaftaji na kurudisha `-1`.

Ikiwa kielezo cha `low` ni kidogo au sawa na kielezo cha `high`, tunahesabu kielezo cha katikati cha orodha, `mid`. Kisha tunakagua kama thamani unayotafuta iko kwenye kielezo cha katikati. Ikiwa iko, tunarudisha kielezo cha katikati.

Vinginevyo, tunakagua kama thamani iliyoko katikati ni ndogo kuliko lengwa. Ikiwa ni ndogo, tunasasisha kielezo cha `low` kuwa kielezo cha katikati kiongezwe moja. Hii ina maana tutatafuta nusu ya kulia ya orodha.

Mwishowe, ikiwa hakuna masharti mengine yoyote ambayo ni `true`, tunasasisha kielezo cha `high` kuwa kielezo cha katikati kisha tukiondoe moja. Hii inamaanisha tutatafuta nusu ya kushoto ya orodha.

Tunaendelea kurudia mchakato huu hadi tupate thamani au kuamua kuwa thamani haipo kwenye orodha.

Ugumu wa muda wa utaftaji wa binary ni `O(log n)` kwa sababu muda unaochukua kutafuta kwenye orodha huongezeka kwa tarakimu za logari kulingana na ukubwa wa orodha.

Ugumu wa nafasi wa utaftaji wa binary ni `O(1)` kwa sababu hauhitaji nafasi ya ziada kutafuta kwenye orodha.

Utaftaji wa binary na utaftaji wa linear vinaweza kutumika kwa changamoto mbalimbali utakazokutana nazo katika sayansi ya kompyuta. Ni muhimu kuelewa tofauti kati ya algoriti hizi mbili na lini kutumia kila moja.
