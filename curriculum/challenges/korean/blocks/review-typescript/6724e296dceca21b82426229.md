## 타입스크립트란 무엇인가

- **JavaScript**: JavaScript는 동적 타입 언어입니다. 이는 변수에 실행 시간에 어떤 값이든 할당할 수 있다는 뜻입니다. 동적 타입 언어의 도전 과제는 타입 안전성이 부족해 오류가 발생할 수 있다는 점입니다.

예를 들어, JavaScript 함수가 배열을 기대하더라도 숫자를 인수로 호출할 수 있습니다:

```javascript
const getRandomValue = (array) => {
  return array[Math.floor(Math.random() * array.length)];
}

console.log(getRandomValue(10));
```

위 예제의 `console` 출력은 `undefined`입니다.

- **TypeScript**: TypeScript는 JavaScript 언어에 정적 타입을 추가하여, 코드 실행 전에 타입 불일치로 인한 오류를 잡는 데 도움을 줍니다.

예를 들어, `array` 매개변수에 대한 타입을 다음과 같이 정의할 수 있습니다:

```typescript
const getRandomValue = (array: string[]) => {
  return array[Math.floor(Math.random() * array.length)];
}
```

이 타입 정의는 TypeScript에 `array` 매개변수가 문자열 배열이어야 한다고 알려줍니다. 그런 다음 `getRandomValue`를 호출하면서 숫자를 전달하면 컴파일러 오류가 발생합니다.

- **컴파일러**: TypeScript 코드를 일반 JavaScript로 먼저 컴파일해야 합니다. 컴파일러를 실행하면 TypeScript가 코드를 평가하고 타입이 맞지 않는 문제에 대해 오류를 발생시킵니다.

## 타입스크립트의 데이터 타입

- **타입스크립트의 원시 데이터 타입**: 원시 데이터 타입인 `string`, `null`, `undefined`, `number`, `boolean`, 그리고 `bigint`에 대해 TypeScript는 대응하는 타입 키워드를 제공합니다.

```typescript
let str: string = "Naomi";
let num: number = 42;
let bool: boolean = true;
let nope: null = null;
let nada: undefined = undefined;
```

- **배열**: 특정 타입의 배열을 두 가지 구문으로 정의할 수 있습니다.

```typescript
const arrOne: string[] = ["Naomi"];
const arrTwo: Array<string> = ["Camperchan"];
```

- **객체**: 객체의 정확한 구조를 정의할 수 있습니다.

```typescript
const obj: { a: string, b: number } = { a: "Naomi", b: 42 };
```

모든 키는 임의지만 모든 값이 문자열이어야 하는 객체를 원한다면, 두 가지 정의 방법이 있습니다:

```typescript
const objOne: Record<string, string> = {};
const objTwo: { [key: string]: string } = {};
```

- **타입스크립트의 기타 유용한 타입**:
  - **`any`**: `any`는 값이 어떤 타입이든 될 수 있음을 나타냅니다. 컴파일러에게 해당 변수의 타입을 신경 쓰지 말라고 알려줍니다.
  - **`unknown`**: `unknown`는 값의 타입을 신경 쓰지만 실제로 무엇인지 모른다는 뜻입니다. `unknown`가 일반적으로 `any`보다 선호됩니다.
  - **`void`**: 주로 함수 정의 시 사용하는 특별한 타입입니다. 반환값이 없는 함수는 `void` 반환 타입을 사용합니다.
  - **`never`**: 절대 존재하지 않는 타입을 나타냅니다.

- **`type` 키워드**: 이 키워드는 `const`와 비슷하지만 변수를 선언하는 대신 타입을 선언할 수 있습니다.

이는 유니언 타입이나 특정 값만 포함하는 타입 같은 사용자 정의 타입 선언에 유용합니다:

```typescript
type stringOrNumber = string | number;
type bot = "camperchan" | "camperbot" | "naomi";
```

- **`interface`**: 인터페이스는 타입을 위한 클래스와 같습니다. 다른 인터페이스를 구현하거나 확장할 수 있고, 객체 타입에 특화되어 있으며, `type` 선언에서 제공하는 특정 기능이 필요하지 않으면 일반적으로 선호됩니다.

```typescript
interface wowie {
  zowie: boolean;
  method: () => void;
}
```

- **반환 타입 정의**: 함수의 _반환 타입_도 정의할 수 있습니다.

아래 예제는 반환값을 문자열로 정의합니다. 다른 값을 반환하려 하면 TypeScript가 컴파일러 오류를 발생시킵니다.

```typescript
const getRandomValue = (array: string[]): string => {
  return array[Math.floor(Math.random() * array.length)];
}
```

## 제네릭

- **제네릭 타입 정의**: 제네릭 타입을 정의하고 함수에서 참조할 수 있습니다. 이는 함수의 타입 정의 동작을 제어하기 위해 함수에 제공하는 특별한 매개변수로 생각할 수 있습니다.

함수에 대한 제네릭 타입 정의 예는 다음과 같습니다:

```typescript
const getRandomValue = <T>(array: T[]): T => {
  return array[Math.floor(Math.random() * array.length)];
}
const val = getRandomValue([1, 2, 4])
```

`<T>` 구문은 TypeScript에 함수에 대한 제네릭 타입 `T`를 정의한다고 알려줍니다. `T`는 제네릭 타입의 일반적인 이름이지만 어떤 이름이든 사용할 수 있습니다.

그런 다음 `array` 매개변수가 제네릭 타입에 맞는 값들의 배열이며, 반환값은 같은 타입의 단일 요소임을 TypeScript에 알립니다.

- **함수 호출 시 타입 인수 지정**: 함수 이름과 매개변수 사이에 꺾쇠 괄호를 사용해 함수 호출에 타입 인수를 전달할 수 있습니다.

함수 호출에 타입 인수를 전달하는 예는 다음과 같습니다:

```typescript
const email = document.querySelector<HTMLInputElement>("#email");
console.log(email.value);
```

이는 TypeScript에 찾으려는 요소가 입력 요소임을 알려줍니다.

## 타입 좁히기

- **진릿값에 의한 좁히기**: 아래 예제에서 `value`이 `email`일 _수_ 있기 때문에 `email`의 `null` 속성에 접근하려 하면 컴파일러 오류가 발생합니다.

```typescript
const email = document.querySelector<HTMLInputElement>("#email");
console.log(email.value);
```

조건문을 사용해 `email`가 _진릿값_임을 확인한 후 속성에 접근할 수 있습니다:

```typescript
const email = document.querySelector<HTMLInputElement>("#email");
if (email) {
  console.log(email.value);
}
```

진릿값 검사는 반대 방향으로도 작동합니다:

```typescript
const email = document.querySelector<HTMLInputElement>("#email");
if (!email) {
  throw new ReferenceError("Could not find email element!")
}
console.log(email.value);
```

오류를 발생시키면 이 코드의 논리 실행이 종료되어 `console.log()` 호출에 도달할 때 TypeScript는 `email`가 `null`일 _수 없다_고 알게 됩니다.

- **선택적 체이닝**: 선택적 체이닝 `?.`도 타입 좁히기의 한 형태로, `email` 값이 `null`일 경우 속성 접근이 불가능하다는 전제 하에 작동합니다.

```typescript
const email = document.querySelector<HTMLInputElement>("#email");
console.log(email?.value);
```

- **`typeof` 연산자**: `typeof` 연산자를 사용해 변수의 타입을 조건문으로 확인할 수 있습니다.

```typescript
const myVal = Math.random() > 0.5 ? 222 : "222";
if (typeof myVal === "number") {
  console.log(myVal / 10);
}
```

- **`instanceof` 키워드**: 객체가 클래스에서 온 경우 `instanceof` 키워드를 사용해 타입을 좁힐 수 있습니다.

```typescript
const email = document.querySelector("#email");

if (email instanceof HTMLInputElement) {
    console.log(email.value);
}
```

- **캐스팅**: TypeScript가 아래 예제의 `request.json()` 메서드 결과 같은 값의 타입을 자동으로 판단하지 못하면 컴파일러 오류가 발생합니다. 이를 해결하는 한 방법은 타입을 캐스팅하는 것이지만, 이렇게 하면 TypeScript가 잠재적 오류를 잡는 능력이 약해집니다.

```typescript
interface User {
    name: string;
    age: number;
}

const printAge = (user: User) => 
  console.log(`${user.name} is ${user.age} years old!`)

const request = await fetch("url")
const myUser = await request.json() as User;
printAge(myUser);
```

- **타입 가드**: 타입을 캐스팅하는 대신 타입 가드를 작성할 수 있습니다:

```typescript
interface User {
    name: string;
    age: number;
}

const isValidUser = (user: unknown): user is User => {
  return !!user && 
    typeof user === "object" &&
    "name" in user &&
    "age" in user;
}
```

`user is User` 구문은 함수가 불리언 값을 반환하며, true일 때 `user` 값이 `User` 인터페이스를 만족함을 나타냅니다.

## `tsconfig` 파일

- **`tsconfig.json`**: TypeScript 컴파일러 설정은 프로젝트 루트 디렉터리의 `tsconfig.json` 파일에 있습니다.

```json
{
  "compilerOptions": {
    "rootDir": "./src",
    "outDir": "./prod",
    "lib": ["ES2023"],
    "target": "ES2023",
    "module": "ES2022",
    "moduleResolution": "Node",
    "esModuleInterop": true,
    "skipLibCheck": true,
    "strict": true
  },
  "exclude": ["test/"]
}
```

아래는 위 예제에서 사용된 컴파일러 옵션 설명입니다:

- **`compilerOptions`**: `compilerOptions` 속성은 TypeScript 컴파일러 동작 방식을 제어합니다.
- **`rootDir` 와 `outDir`**: `rootDir` 와 `outDir` 는 TypeScript에 소스 파일이 있는 디렉터리와 트랜스파일된 JavaScript 코드가 들어갈 디렉터리를 알려줍니다.
- **`lib`**: `lib` 속성은 컴파일러가 사용할 타입 정의를 결정하며, 특정 ES 릴리스, DOM 지원 등을 포함할 수 있게 합니다.
- **`module` 와 `moduleResolution`**: `module` 와 `moduleResolution` 는 패키지의 모듈 사용 방식을 CommonJS 또는 ECMAScript로 관리하는 데 함께 작동합니다.
- **`esModuleInterop`**: `esModuleInterop` 는 CommonJS와 ES 모듈 간 원활한 상호 운용성을 위해 가져오기 시 네임스페이스 객체를 자동 생성합니다.
- **`skipLibCheck`**: `skipLibCheck` 옵션은 코드에서 가져오기로 참조되지 않는 `.d.ts` 파일 검증을 건너뜁니다.
- **`strict`**: `strict` 플래그는 널 가능 타입 처리 보장, TypeScript가 `any`로 대체될 때 경고 등 여러 검사를 활성화합니다.
- **`exclude`**: 최상위 `exclude` 속성은 컴파일 시 이 TypeScript 파일들을 무시하도록 컴파일러에 지시합니다.
