Progressive Offenlegung ist ein Designmuster, das verwendet wird, um Nutzern nur relevante Inhalte basierend auf ihrer aktuellen Aktivität anzuzeigen und den Rest zu verbergen. Dies geschieht, um die kognitive Belastung zu reduzieren und die Benutzererfahrung intuitiver zu gestalten.

:::interactive_editor

    Show More Details
  </button>
  <div id="extra-content" class="hidden" tabindex="-1">
    <p>
      Here are additional details that are revealed only when you want to see them.
      This keeps the interface clean and reduces cognitive load.

Zum Beispiel, wenn Sie die Google-Suchseite verwenden, sind die meisten Ihrer Suchanfragen einfach. Daher macht eine einfache Benutzeroberfläche mit sehr wenigen Optionen Sinn. Aber für die wenigen Male, in denen Sie eine erweiterte Suche benötigen, bietet Google eine erweiterte Suchoption an, die mehr Optionen anzeigt.

Sie finden die erweiterte Suchoption von Google im Einstellungsmenü auf der Google-Startseite. Oder Sie können zu `google.com/advanced_search` gehen.

Ein weiteres Beispiel für progressive Offenlegung ist der Button „Mehr Details“ bei Produkten auf einer E-Commerce-Website. Wenn Sie eine Seite wie Amazon besuchen, sehen Sie eine Liste von Produktbildern in verschiedenen Kategorien. Dies ermöglicht es den Nutzern, bequem einzukaufen, ohne von zu vielen Informationen überwältigt zu werden.

Wenn der Nutzer an einem bestimmten Produkt interessiert ist, kann er auf das Produktbild klicken, um mehr Details über das Produkt zu sehen. Alle Informationen sofort anzuzeigen, könnte den Nutzer überfordern. Durch progressive Offenlegung kann der Nutzer also selbst entscheiden, wann er mehr Informationen sehen möchte.

document.querySelectorAll('.details-btn').forEach(button => {
  button.addEventListener('click', () => {
    const product = button.closest('.product');
    const details = product.querySelector('.product-details');
    const isHidden = details.classList.toggle('hidden');
    product.setAttribute('aria-expanded', !isHidden);
    if (!isHidden) {
      details.focus();
      button.textContent = 'Hide details';
    } else {
      button.textContent = 'More details';
    }
  });
});

Welche bewährten Vorgehensweisen gibt es also beim Design mit der Technik der progressiven Offenlegung? Die erste Überlegung ist, alle wichtigen Informationen jederzeit sichtbar zu halten. Nutzer sollten nicht danach suchen müssen, weil sie durch zusätzliche erweiterte Funktionen verborgen sind.

Eine weitere Überlegung ist, einen einzigen Zugangspunkt bereitzustellen, über den Nutzer zusätzliche Funktionen oder Informationen erreichen können. Das kann ein Button oder Link sein, der immer auf der Seite sichtbar ist. Mehrere Zugangspunkte können Nutzer verwirren und zu einer schlechten Benutzererfahrung führen.

:::interactive_editor
