# --interactive--

如果你需要在顯示內容的同時，將其隱藏給使用輔助技術（例如螢幕閱讀器）的人員，你可以使用 `aria-hidden` 屬性。

你只需要將它添加到你想隱藏的 HTML 元素，並將其值設為 `true`，就像你在這裡看到的：`aria-hidden="true"`。

此屬性會將元素及其所有子元素從無障礙樹中隱藏，同時保持它們在頁面上的可見性。常見的使用情境包括：

- 僅具有裝飾用途的圖示和影像。
- 重複內容。

重要的是要記住，`aria-hidden` 僅會將內容從輔助技術（例如螢幕閱讀器）中隱藏。如果內容應該對所有人隱藏，則不應使用 `aria-hidden` 來隱藏它。例如，目前已摺疊的漢堡選單必須對所有鍵盤使用者隱藏，而不僅僅是螢幕閱讀器使用者。在這種情況下，你可以將選單的 CSS `display` 屬性設為 `none`，以在摺疊時將其從 DOM 中移除。

你絕對不應該使用 `aria-hidden` 來隱藏可以用鍵盤聚焦的元素。`aria-hidden` 屬性只會將元素從無障礙樹中移除。它不會將元素從 DOM 中移除。因此，螢幕閱讀器使用者仍然可以透過 Tab 鍵聚焦該元素，但他們的螢幕閱讀器不會宣告該元素，實際上會導致他們聚焦在「空白」上。

以下是一個範例，我們透過為 `aria-hidden` 屬性添加值為 `true`，將圖示從無障礙樹中隱藏。 

我們只保留暴露給輔助技術的文字，以避免因同時擁有相同用途的圖示和文字而產生的混淆。

  <i class="fa-solid fa-gear" aria-hidden="true"></i>
  <span class="label">Settings</span>

當你符合以下條件時，不需要使用 `aria-hidden`：

- 該 HTML 元素已經有 `hidden` 屬性。
- 該元素或該元素的祖先已經使用 `display: none` 隱藏。
- 該元素或該元素的祖先已經使用 `visibility: hidden` 隱藏。

在這三種情況下，元素已經從 DOM 中移除，因此在無障礙樹中被隱藏，所以不需要 `aria-hidden` 屬性。

如同使用任何 ARIA 屬性一樣，你應該始終使用輔助技術進行測試，並且最好讓有障礙的人士測試你的作品，以確保即使有這些隱藏元素，也能容易理解。

你也應該知道，如果任何父元素的 `aria-hidden` 屬性設為 `true`，即使將 `aria-hidden` 設為 `false`，該元素仍不會被輔助技術揭露。

`aria-hidden` 屬性用於將元素隱藏起來，讓使用輔助技術（例如螢幕閱讀器使用者）的用戶看不到。

雖然它對隱藏純裝飾元素和重複內容有幫助，但應該謹慎使用以避免妨礙存取。

一般來說，頁面上所有可用的內容和功能也應該對使用輔助技術的人可用。`aria-hidden` 的使用情境非常狹窄，應主要限於透過移除純裝飾性或重複的資訊，使螢幕閱讀器使用者的體驗更乾淨。不要使用 `aria-hidden` 來隱藏你認為螢幕閱讀器使用者不會感興趣的內容。螢幕閱讀器使用者應該有權存取頁面上的所有資訊。

透過遵循這些最佳實踐並測試使用者體驗，你可以為所有人創建包容性的線上體驗。
