# --description--

في بايثون، القواميس هي هياكل بيانات مدمجة تخزن مجموعات من أزواج المفتاح-القيمة. تعمل بطريقة مشابهة جدًا للقواميس الحقيقية، حيث تبحث عن كلمة لتجد معناها المقابل.

مع قواميس بايثون، تستخدم مفتاحًا لتجد القيمة المقابلة له. يجب استخدام القواميس عندما تحتاج إلى ربط قيم بمفاتيح فريدة. هذا مفيد عندما تحتاج إلى إيجاد قيمة بسرعة بناءً على المفتاح، وعندما تحتاج إلى تمثيل بيانات منظمة.

هذه هي الصيغة النحوية العامة لقاموس في بايثون:

```python
dictionary = {
    key1: value1,
    key2: value2
}
```

أولًا، نجد المتغير الذي يحمل القاموس. ليس من الضروري تعيين القاموس إلى متغير، لكن من الشائع جدًا فعل ذلك للاحتفاظ به في الذاكرة واستخدامه لاحقًا في الشفرة.

بعد ذلك تأتي الأقواس المتعرجة، التي تُسمى أحيانًا أقواس متعرجة. وداخل هذه الأقواس توجد أزواج المفتاح-القيمة.

يرتبط كل مفتاح بقيمة، لذا يمكنك استخدام المفتاح للوصول إلى تلك القيمة. بعد كل قيمة، ما عدا الأخيرة، توجد فاصلة لفصل أزواج المفتاح-القيمة المختلفة. يجب أن تكون المفاتيح فريدة في القاموس، ويجب أن تكون من نوع بيانات غير قابل للتغيير. أما القيم فيمكن تكرارها، ويمكن أن تكون من أي نوع بيانات.

هنا لدينا مثال على قاموس يخزن معلومات عن وصفة بيتزا مارغريتا:

```python
pizza = {
    'name': 'Margherita Pizza',
    'price': 8.9,
    'calories_per_slice': 250,
    'toppings': ['mozzarella', 'basil']
}
```

تم تعيين القاموس إلى المتغير `pizza`. يحتوي على أربعة أزواج مفتاح-قيمة: `name`، و`price`، و`calories_per_slice`، و`toppings`.

بديل آخر هو استخدام دالة البناء `dict()`، التي تبني القاموس من متتالية من أزواج المفتاح-القيمة.

هذه هي الصيغة النحوية المكافئة لمثال البيتزا لدينا. نمرر قائمة من الصفوف المرتبة كمعلمة إلى دالة البناء `dict()`. تحتوي هذه الصفوف على المفتاح كالعنصر الأول والقيمة كالعنصر الثاني.

```python
pizza = dict([('name', 'Margherita Pizza'), ('price', 8.9), ('calories_per_slice', 250), ('toppings', ['mozzarella', 'basil'])])
```

للوصول إلى قيمة زوج مفتاح-قيمة، يمكنك استخدام هذه الصيغة المعروفة بترميز الأقواس. هي اسم المتغير الذي يحمل القاموس، متبوعًا بأقواس مربعة، والمفتاح الذي تريد الوصول إليه داخل الأقواس المربعة:

```python
dictionary[key]
```

في مثال البيتزا لدينا، إذا أردت الوصول إلى قيمة `name`، تكتب اسم المتغير `pizza`، متبوعًا بأقواس مربعة، والمفتاح `name` بين علامات اقتباس:

```python
pizza['name']
```

هذا سيُقيَّم إلى:

```python
'Margherita Pizza'
```

لتحديث قيمة، تحتاج فقط إلى إضافة معامل التعيين، متبوعًا بالقيمة الجديدة.

إذا لم يكن المفتاح موجودًا في القاموس، سيتم إنشاء زوج مفتاح-قيمة جديد. في الإصدارات الحديثة من بايثون، تحافظ القواميس على ترتيب الإدخال. هذا مفيد عندما تحتاج إلى التكرار على القاموس:

```python
pizza['name'] = 'Margherita'
```

الآن قيمة المفتاح `name` هي `'Margherita'`:

```python
print(pizza['name']) # 'Margherita'
```

للقواميس أيضًا طرق مفيدة لأداء العمليات الشائعة.

طريقة `.get()` تسترجع القيمة المرتبطة بمفتاح. هي مشابهة لترميز الأقواس الذي استخدمناه للتو، لكن ميزتها أنك تستطيع تعيين قيمة افتراضية، فلا تحصل على خطأ إذا لم يكن المفتاح موجودًا:

```python
dictionary.get(key, default)
```

في هذا المثال، إذا لم يكن المفتاح `toppings` موجودًا، سترجع قائمة فارغة، وهي القيمة الافتراضية التي نمررها هنا كالمعلمة الثانية. لكن إذا كان `toppings` موجودًا، سترجع تلك القيمة:

```python
pizza.get('toppings', []) # ['mozzarella', 'basil']
```

طرقا `.keys()` و`.values()` تعيدان كائن عرض يحتوي على كل المفاتيح والقيم في القاموس، على التوالي:

```python
pizza.keys()
# dict_keys(['name', 'price', 'calories_per_slice'])

pizza.values()
# dict_values(['Margherita Pizza', 8.9, 250])
```

كائن العرض هو مجرد طريقة لرؤية محتوى القاموس دون إنشاء نسخة منفصلة من البيانات.

طريقة `.items()` تعيد كائن عرض يحتوي على كل أزواج المفتاح-القيمة في القاموس، بما في ذلك المفاتيح والقيم:

```python
pizza.items()
# dict_items([('name', 'Margherita Pizza'), ('price', 8.9), ('calories_per_slice', 250)])
```

طريقة `.clear()` تزيل كل أزواج المفتاح-القيمة من القاموس:

```python
pizza.clear()
```

طريقة `.pop()` تزيل زوج المفتاح-القيمة الذي تحدده كالمعلمة الأولى وتُرجع قيمته. إذا لم يكن المفتاح موجودًا، تُرجع القيمة الافتراضية التي تحددها كالمعلمة الثانية. إذا لم يكن المفتاح موجودًا ولم تمرر قيمة افتراضية، يُثار الخطأ `KeyError`:

```python
pizza.pop('price', 10)
pizza.pop('total_price') # KeyError
```

في بايثون 3.7 والإصدارات الأحدث، طريقة `.popitem()` تزيل آخر عنصر تم إدخاله:

```python
pizza.popitem()
```

وأخيرًا، طريقة `.update()` تحدث أزواج المفتاح-القيمة بأزواج قاموس آخر. إذا كان لديهما مفاتيح مشتركة، تُستبدل قيمها.

في هذا المثال، نحن نحدث قاموس `pizza`. مفتاح `price` موجود في كلاهما، لذا ستُستبدل قيمته بـ `15`.

لكن `total_time` جديد، لذا سيُضاف إلى قاموس `pizza` كزوج مفتاح-قيمة جديد:

```python
pizza.update({ 'price': 15, 'total_time': 25 })
```

هذا هو القاموس الجديد مع تحديث `price` و`total_time` الجديد. لاحظ كيف أن السعر الآن `15` و`total_time` زوج مفتاح-قيمة جديد:

```python
{
    'name': 'Margherita Pizza', 
    'price': 15, 
    'calories_per_slice': 250, 
    'toppings': ['mozzarella', 'basil'], 
    'total_time': 25
}
```

هذه بعض أكثر طرق القواميس استخدامًا، لكن هناك الكثير غيرها. اختيار الطريقة المناسبة يمكن أن يساعد في أداء العمليات المعقدة بكفاءة.
