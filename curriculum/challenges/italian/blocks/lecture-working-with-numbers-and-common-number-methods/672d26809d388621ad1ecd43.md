# --interactive--

In JavaScript, `NaN` indica "Not a Number". È un valore speciale che rappresenta un risultato numerico non rappresentabile o indefinito. `NaN` è una proprietà dell'oggetto globale, ed è anche considerato un tipo di numero in JavaScript, il che può sembrare controintuitivo all'inizio.

`NaN` è tipicamente il risultato di operazioni che dovrebbero restituire un numero ma non possono produrre un valore numerico significativo. Per esempio:

:::interactive_editor

```js
let result = 0 / 0;
console.log(result); // NaN
```

:::

In questo caso, dividere zero per zero è matematicamente indefinito, quindi JavaScript restituisce `NaN`. Una proprietà particolare di `NaN` è che non è uguale a nulla, incluso se stesso:

:::interactive_editor

```js
console.log(NaN === NaN); // false
```

:::

Questo comportamento unico rende difficile verificare se un valore è `NaN` usando gli operatori di confronto standard. Per risolvere questo, JavaScript fornisce la funzione `isNaN()`. La funzione `isNaN()` viene usata per determinare se un valore è `NaN` o no. Tuttavia, è importante capire come funziona `isNaN()`, perché a volte può produrre risultati inaspettati. Ecco come si comporta `isNaN()`:

:::interactive_editor

```js
console.log(isNaN(NaN));       // true
console.log(isNaN(undefined)); // true
console.log(isNaN({}));        // true

console.log(isNaN(true));      // false
console.log(isNaN(null));      // false
console.log(isNaN(37));        // false

console.log(isNaN("37"));      // false: "37" is converted to 37
console.log(isNaN("37.37"));   // false: "37.37" is converted to 37.37
console.log(isNaN(""));        // false: empty string is converted to 0
console.log(isNaN(" "));       // false: string with a space is converted to 0

console.log(isNaN("blabla"));  // true: "blabla" is not a number
```

:::

Come puoi vedere, `isNaN()` tenta prima di convertire il parametro in un numero. Se non può essere convertito, restituisce `true`. Questo comportamento può portare a risultati sorprendenti, specialmente quando si trattano stringhe che possono essere convertite in numeri.

A causa di queste potenziali incoerenze, ES6 (la sesta edizione di JavaScript, rilasciata nel 2015) ha introdotto `Number.isNaN()`. Questo metodo non tenta di convertire il parametro in un numero prima del test. Restituisce `true` solo se il valore è esattamente `NaN`:

:::interactive_editor

```js
console.log(Number.isNaN(NaN));        // true
console.log(Number.isNaN(Number.NaN)); // true
console.log(Number.isNaN(0 / 0));      // true

console.log(Number.isNaN("NaN"));      // false
console.log(Number.isNaN(undefined));  // false
console.log(Number.isNaN({}));         // false
console.log(Number.isNaN("blabla"));   // false
```

:::

`Number.isNaN()` offre un modo più affidabile per verificare valori `NaN`, specialmente nei casi in cui la conversione implicita potrebbe portare a risultati inaspettati con la funzione globale `isNaN()`. In pratica, quando si lavora con operazioni numeriche o input utente che dovrebbero essere numeri, spesso è necessario verificare `NaN` per gestire errori o input imprevisti in modo elegante. Per esempio:

:::interactive_editor

```js
let a = 0;
let b = 0;
let result = a / b;

if (Number.isNaN(result)) {
  result = "Error: Division resulted in NaN";
}

console.log(result); // "Error: Division resulted in NaN"
```

:::

In questo esempio, usiamo `Number.isNaN()` per intercettare i casi in cui l'operazione di divisione produce `NaN`, permettendoci di gestire correttamente questo scenario. Capire `NaN` e come verificarlo correttamente è fondamentale per scrivere codice JavaScript robusto, specialmente quando si trattano operazioni matematiche o si analizzano input utente.
