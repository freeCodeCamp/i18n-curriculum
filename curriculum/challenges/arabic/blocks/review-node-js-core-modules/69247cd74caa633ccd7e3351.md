## مقدمة إلى وحدات Node.js الأساسية

- **وحدات Node.js الأساسية**: وحدات مدمجة تأتي مثبتة مسبقًا مع Node.js، توفر وظائف أساسية لعمليات الملفات، التشفير، التفاعل مع نظام التشغيل، والمزيد.
- **الوحدات القياسية**: لا تحتاج إلى تثبيت منفصل؛ متاحة طالما Node.js مثبت.
- **صيغة الاستيراد**: استخدم `require()` لاستيراد الوحدات الأساسية إلى تطبيقك.

```javascript
const fs = require("fs");
const crypto = require("crypto");
const os = require("os");
```

## وحدة نظام الملفات (fs)

- **الغرض**: توفر طرقًا للعمل مع الملفات والمجلدات، تشمل فتح، إغلاق، قراءة، كتابة، وحذف الملفات.
- **الاستيراد**: `const fs = require("fs");`
- **المرونة**: الطرق متاحة بثلاثة أشكال: استدعاءات راجعة، وعود، ومتزامنة.

### أنماط استخدام الطرق

- **غير متزامن مع استدعاءات راجعة**: السلوك الافتراضي، غير محجوز.

```javascript
fs.writeFile("filePath", "content", "utf8", (err) => {
  if (err) throw err;
  console.log("File written to!");
});
```

- **معتمد على الوعود**: مفضل للتطبيقات الحديثة، يستخدم `async/await`.

```javascript
const fs = require("fs/promises");

async function writeToFile() {
  try {
    await fs.writeFile("article.md", "## Node fs Module");
    console.log("File written to!");
  } catch (err) {
    console.error(err);
  }
}
```

- **متزامن**: عملية حجز، مناسبة للسكريبتات البسيطة.

```javascript
try {
  fs.writeFileSync("article.md", "## Node fs Module", "utf8");
  console.log("File written to!");
} catch (err) {
  console.error(err);
}
```

### العمليات الأساسية على الملفات

- **طريقة `writeFile()`**: تنشئ ملفًا أو تكتب فوق ملف موجود بمحتوى محدد.

```javascript
await fs.writeFile("article.md", "## Node fs Module: The Complete Guide");
```

- **طريقة `appendFile()`**: تضيف محتوى إلى نهاية ملف موجود.

```javascript
await fs.appendFile("article.md", "\n\nIn this article, you will learn...");
```

- **طريقة `readFile()`**: تقرأ محتويات ملف.

```javascript
const content = await fs.readFile("article.md", "utf8");
console.log("File content:", content);
```

- **طريقة `unlink()`**: تحذف ملفًا من نظام الملفات.

```javascript
await fs.unlink("article.md");
console.log("File deleted successfully");
```

### ترميز الأحرف

- **ترميز UTF-8**: حدد `"utf8"` لقراءة الملفات كسلاسل نصية.
- **صيغة المخزن المؤقت**: بدون تحديد الترميز، تُقرأ الملفات ككائنات مخزن مؤقت ثنائية.

```javascript
// Returns string
const textContent = await fs.readFile("file.txt", "utf8");

// Returns Buffer
const bufferContent = await fs.readFile("file.txt");
```

## وحدة المخزن المؤقت (Buffer)

- **الغرض**: تتعامل مع البيانات الثنائية مباشرة في الذاكرة للملفات، الصور، وتدفقات الشبكة.
- **الاستيراد**: `const { Buffer } = require("buffer");`
- **حالات الاستخدام**: إدخال/إخراج الملفات، تدفقات TCP، معالجة الصور، والتعامل مع البيانات الثنائية.

### إنشاء المخزن المؤقت

- **طريقة `Buffer.from()`**: تنشئ مخزن مؤقت من سلاسل نصية، مصفوفات، أو بيانات أخرى.

```javascript
// From string
const myStrBuffer = Buffer.from("freeCodeCamp");
console.log(myStrBuffer); // <Buffer 66 72 65 65 43 6f 64 65 43 61 6d 70>

// From array of numbers
const myNumBuffer = Buffer.from([70, 82, 69, 69, 67, 79, 68, 69, 67, 65, 77, 80]);
```

- **طريقة `Buffer.alloc()`**: تنشئ مخزن مؤقت بحجم محدد مملوء بالأصفار.

```javascript
const someBuffer = Buffer.alloc(10);
console.log(someBuffer); // <Buffer 00 00 00 00 00 00 00 00 00 00>
```

### عمليات المخزن المؤقت

- **الوصول مثل المصفوفة**: الوصول إلى عناصر المخزن المؤقت الفردية باستخدام ترقيم الفهرس.

```javascript
console.log(myStrBuffer[0]); // 102 (byte value for 'f')
```

- **طريقة `toString()`**: تحول المخزن المؤقت إلى سلسلة نصية قابلة للقراءة.

```javascript
console.log(myStrBuffer.toString()); // "freeCodeCamp"
```

- **طريقة `Buffer.write()`**: تكتب بيانات إلى مخزن مؤقت مخصص.

```javascript
someBuffer.write("Hello fCC");
console.log(someBuffer.toString()); // "Hello fCC"
```

- **طريقة `Buffer.byteLength()`**: تعيد عدد البايتات اللازمة لتخزين سلسلة نصية.

```javascript
console.log(Buffer.byteLength("Hello freeCodeCamp")); // 18
```

### طرق إضافية للمخزن المؤقت

- **`Buffer.isBuffer()`**: تتحقق مما إذا كان الكائن مخزن مؤقت.
- **`Buffer.compare()`**: تقارن بين مخزنين مؤقتين وتعيد ترتيب الفرز.
- **`Buffer.concat()`**: تجمع عدة مخازن مؤقتة في واحد.

## وحدة التشفير (Crypto)

- **الغرض**: توفر وظائف التشفير، تشمل التجزئة، التشفير، فك التشفير، والتوقيعات الرقمية.
- **الاستيراد**: `const crypto = require("crypto");`
- **ملاحظة أمنية**: استخدم مكتبات مجربة مثل `bcrypt` أو `jsonwebtoken` للتوثيق في الإنتاج.

### طرق التجزئة

- **طريقة `createHash()`**: تنشئ تجزئة أحادية الاتجاه باستخدام خوارزميات مثل SHA256، SHA512، أو MD5.

```javascript
const hashedPassword = crypto
  .createHash("sha256")
  .update("myPassword123")
  .digest("hex");

console.log("createHash result:", hashedPassword);
```

- **طريقة `createHmac()`**: تنشئ تجزئة بمفتاح سري للتوثيق وسلامة البيانات.

```javascript
const hashedMessage = crypto
  .createHmac("sha256", "secret-key")
  .update("Hello World")
  .digest("hex");
```

### التشفير وفك التشفير

- **طرقا `createCipheriv()` و `createDecipheriv()`**: تشفر وتفك تشفير البيانات باستخدام خوارزميات، مفاتيح، ومتجهات التهيئة.

```javascript
const key = Buffer.from("12345678901234567890123456789012"); // 32 bytes for AES-256
const iv = Buffer.from("1234567890123456"); // 16 bytes for AES

// Encryption
const cipher = crypto.createCipheriv("aes-256-cbc", key, iv);
let encrypted = cipher.update("Hello campers!", "utf8", "hex");
encrypted += cipher.final("hex");

// Decryption
const decipher = crypto.createDecipheriv("aes-256-cbc", key, iv);
let decrypted = decipher.update(encrypted, "hex", "utf8");
decrypted += decipher.final("utf8");
```

### توليد القيم العشوائية

- **طريقة `randomBytes()`**: تولد بايتات عشوائية آمنة تشفيرياً.

```javascript
console.log("Random Bytes:", crypto.randomBytes(16).toString("hex"));
// Output: a6154ef5a296fa176ad0f332bd94d712
```

- **طريقة `randomInt()`**: تولد أعداد صحيحة عشوائية آمنة ضمن نطاق محدد.

```javascript
console.log("Random Int:", crypto.randomInt(0, 100)); // Random number 0-99
```

### إدارة المفاتيح

- **طريقة `createSecretKey()`**: تنشئ كائنات مفاتيح تشفيرية من بايتات خام.

```javascript
const secret = crypto.createSecretKey(crypto.randomBytes(32));
console.log(secret.export().toString('hex'));
```

### التوقيعات الرقمية

- **طرقا `sign()` و `verify()`**: تنشئ وتتحقق من التوقيعات الرقمية باستخدام أزواج المفاتيح الخاصة/العامة.
- **حالة الاستخدام**: تثبت صحة وسلامة البيانات.

### طرق تشفير إضافية

- **`createPublicKey()` و `createPrivateKey()`**: تعمل مع مفاتيح مولدة خارجيًا.
- **`createDiffieHellman()`**: تمكّن من توليد سر مشترك آمن.
- **`Certificate()`**: تتعامل مع عمليات شهادات HTTPS.

## وحدة نظام التشغيل (OS)

- **الغرض**: توفر أدوات متعلقة بنظام التشغيل لمعلومات النظام والتوافق عبر الأنظمة.
- **الاستيراد**: `const os = require("os");`
- **حالات الاستخدام**: مراقبة النظام، السكريبتات متعددة الأنظمة، معلومات الأجهزة.

### طرق معلومات النظام

- **طريقة `platform()`**: تعيد منصة نظام التشغيل.

```javascript
console.log(os.platform()); // 'darwin', 'win32', 'linux', etc.

// Cross-platform usage
if (os.platform() === 'win32') {
  // Windows-specific code
} else {
  // Unix-like systems
}
```

- **طريقة `arch()`**: تعيد بنية وحدة المعالجة المركزية.

```javascript
console.log(os.arch()); // 'arm64', 'x64', 'arm', 'ia32', etc.
```

- **طريقة `type()`**: تعيد الاسم الرسمي لنظام التشغيل.

```javascript
console.log(os.type()); // 'Darwin', 'Linux', 'Windows_NT'
```

### معلومات الإصدار

- **طريقة `release()`**: تعرض إصدار نواة نظام التشغيل.

```javascript
console.log(os.release()); // '25.0.0'
```

- **طريقة `version()`**: تعيد معلومات تفصيلية عن إصدار نظام التشغيل.

```javascript
console.log(os.version());
// Darwin Kernel Version 25.0.0: Wed Sep 17 21:41:39 PDT 2025
```

### معلومات الأجهزة

- **طريقة `cpus()`**: تعيد مصفوفة بمعلومات أنوية وحدة المعالجة المركزية.

```javascript
console.log(os.cpus()); // Array of CPU core objects with model, speed, times
```

- **طريقة `uptime()`**: تعيد مدة تشغيل النظام بالثواني.

```javascript
console.log(os.uptime()); // 23047 (seconds since system boot)
```

### معلومات الذاكرة

- **طريقة `totalmem()`**: تعيد إجمالي ذاكرة النظام بالبايت.
- **طريقة `freemem()`**: تعيد الذاكرة المتاحة في النظام بالبايت.

## أفضل الممارسات

### اعتبارات الأداء

- **تجنب الطرق المتزامنة**: استخدم الطرق غير المتزامنة في تطبيقات الإنتاج لتجنب الحجز.
- **فضل الوعود**: استخدم `async/await` مع واجهة برمجة التطبيقات المعتمدة على الوعود لشفرة أنظف وأسهل صيانة.
- **معالجة الأخطاء**: طبق دائمًا معالجة صحيحة للأخطاء باستخدام كتل try-catch أو استدعاءات راجعة للأخطاء.

### إرشادات الأمان

- **وحدة التشفير**: استخدم مكتبات معتمدة للتوثيق بدلاً من بناء حلول تشفير مخصصة.
- **التحقق من المدخلات**: تحقق من مسارات الملفات ومدخلات المستخدم لمنع الثغرات الأمنية.
- **إدارة المفاتيح**: خزّن مفاتيح التشفير بأمان، ولا تدرجها مباشرة في الكود المصدري.

### التطوير متعدد الأنظمة

- **استخدام وحدة OS**: استخدم طرق وحدة OS لضمان التوافق عبر الأنظمة.
- **معالجة المسارات**: استخدم وحدة `path` لعمليات مسارات الملفات متعددة الأنظمة.
- **متغيرات البيئة**: تحقق من متغيرات البيئة والتكوينات الخاصة بالمنصة.
