## Reactでのイベントの扱い方

- **Synthetic Event System**: これはReactのイベント処理の仕組みです。ネイティブの`click`、`keydown`、`submit`イベントのラッパーとして機能します。Reactのイベントハンドラはキャメルケースの命名慣例を使います。（例：`onClick`、`onSubmit`など）

Reactで`onClick`要素に`button`属性を使う例はこちらです：

```jsx
function handleClick() {
  console.log("Button clicked!");
}

<button onClick={handleClick}>Click Me</button>;
```

Reactでは、イベントハンドラ関数は通常、イベント処理を担当することを示すために`handle`という接頭辞で始まります。例えば`handleClick`や`handleSubmit`のようにです。

ユーザーの操作でイベントが発生すると、ReactはSynthetic Eventオブジェクトをハンドラに渡します。このオブジェクトは、バニラJavaScriptのネイティブEventオブジェクトとよく似ており、`type`、`target`、`currentTarget`などのプロパティを提供します。

例えば`onSubmit`イベント中にブラウザの更新などのデフォルト動作を防ぐには、`preventDefault()`メソッドを呼び出します：

```jsx
function handleSubmit(event) {
  event.preventDefault();
  console.log("Form submitted!");
}

<form onSubmit={handleSubmit}>
  <input type="text" />
  <button>Submit</button>
</form>;
```

また、ハンドラ関数をアロー関数でラップすることもできます：

```jsx
function handleDelete(id) {
  console.log("Deleting item:", id);
}

<button onClick={() => handleDelete(1)}>Delete Item</button>;
```

## Stateと`useState`フックの扱い方

- **stateの定義**: Reactではstateはコンポーネントのデータを含むオブジェクトです。stateが更新されるとコンポーネントは再レンダリングされます。Reactはstateを不変とみなし、直接変更しないようにします。  
- **`useState()`フック**: `useState`フックは関数コンポーネントでstate変数を宣言できる関数です。基本的な構文は以下の通りです：

```js
const [stateVariable, setStateFunction] = useState(initialValue);
```

state変数には以下があります：

- `stateVariable`は現在のstateの値を保持します
- `setStateFunction`（セッター関数）はstate変数を更新します
- `initialValue`は初期stateを設定します

`Counter`コンポーネントの完全な例はこちらです：

```jsx
import { useState } from "react";

function Counter() {
  const [count, setCount] = useState(0);

  return (
    <div>
      <h2>{count}</h2>

      <button onClick={() => setCount(count - 1)}>Decrement</button>
      <button onClick={() => setCount(count + 1)}>Increment</button>
    </div>
  );
}

export default Counter;
```

## レンダリングとReactコンポーネント

- **定義**: Reactではレンダリングはコンポーネントがユーザーインターフェース（UI）、通常はブラウザに表示されるプロセスです。レンダリングはトリガー、レンダー、コミットの3段階で構成されます。

トリガー段階はReactが何かが変わったことを検知し、UIの更新が必要かもしれないと判断したときに起こります。この変化は多くの場合、stateやpropsの更新によるものです。

トリガーが起こるとReactはレンダー段階に入ります。ここでReactはコンポーネントを再評価し、何を表示するかを決めます。そのためにReactは「本物の」DOMの軽量コピーである仮想DOMを使います。仮想DOMを使うことで、Reactはどこを変更すべきかを素早くチェックできます。

コミット段階はReactが仮想DOMで準備した変更を本物のDOMに適用する段階です。つまり、画面に最終結果が表示される段階です。

## State内のオブジェクトと配列の更新

- **State内のオブジェクトの更新**: state内のオブジェクトを更新する必要がある場合は、新しいオブジェクトを作成するか既存のオブジェクトをコピーしてから、その新しいオブジェクトでstateを設定してください。stateに入れるオブジェクトは読み取り専用とみなすべきです。以下はユーザーの名前、年齢、都市を設定する例です。`handleChange`関数はユーザー情報の更新を扱います：

```jsx
import { useState } from "react";

function Profile() {
  const [user, setUser] = useState({ name: "John Doe", age: 31, city: "LA" });

  const handleChange = (e) => {
    const { name, value } = e.target;

    setUser((prevUser) => ({...prevUser, [name]: value}));
  };

  return (
    <div>
      <h1>User Profile</h1>
      <p>Name: {user.name}</p>
      <p>Age: {user.age}</p>
      <p>City: {user.city}</p>

      <h2>Update User Age </h2>
      <input type="number" name="age" value={user.age} onChange={handleChange} />

      <h2>Update User Name </h2>
      <input type="text" name="name" value={user.name} onChange={handleChange} />

      <h2>Update User City </h2>
      <input type="text" name="city" value={user.city} onChange={handleChange} />
    </div>
  );
}

export default Profile;
```

- **State内の配列の更新**: state内の配列を更新するときは、`push()`や`pop()`のようなメソッドで配列を直接変更しないことが重要です。代わりにstateを更新するときは新しい配列を作成してください：

```js
const addItem = () => {
  const newItem = {
    id: items.length + 1,
    name: `Item ${items.length + 1}`,
  };

  // Creates a new array
  setItems((prevItems) => [...prevItems, newItem]);
};
```

配列からアイテムを削除したい場合は、削除したいものを除外した新しい配列を返す`filter()`メソッドを使うべきです：

```js
const removeItem = (id) => {
  setItems((prevItems) => prevItems.filter((item) => item.id !== id));
};
```

## Refsを使った値の参照

- **`ref`属性**: Reactでは`ref`属性を使ってDOMノードにアクセスできます。以下は`ref`を使って`input`要素にフォーカスを当てる例です。`current`プロパティはその`ref`の現在の値にアクセスするために使います：

```jsx
import { useRef } from "react";

const Focus = () => {
  const inputRef = useRef(null);

  const handleFocus = () => {
    if (inputRef.current) {
      inputRef.current.focus();
    }
  };

  return (
    <div>
      <input ref={inputRef} type="text" placeholder="Enter text" />
      <button onClick={handleFocus}>Focus Input</button>
    </div>
  );
};

export default Focus;
```

## `useEffect`フックの使い方

- **`useEffect()`フック**: Reactではエフェクトとはコンポーネントのレンダリングプロセスの外で起こる何かのことです。つまり、ReactがUIのレンダリングの一部として直接扱わないものです。よくある例はデータの取得、ブラウザタブのタイトル更新、ブラウザのローカルストレージの読み書き、ユーザーの位置情報取得などです。これらの操作は外部とやり取りし、副作用と呼ばれます。Reactはこれらの副作用を扱うために`useEffect`フックを提供しています。`useEffect`はコンポーネントのレンダリングや更新後に関数を実行できます。

```js
import { useEffect } from "react";

useEffect(() => {
 // Your side effect logic (usually a function) goes here
}, [dependencies]);
```

エフェクト関数はコンポーネントのレンダリング後に実行され、オプションの`dependencies`引数でエフェクトの実行タイミングを制御します。

`dependencies`は「リアクティブな値」（state、props、関数、変数など）の配列、空配列、または省略可能です。これらのオプションが`useEffect`の動作を次のように制御します：

- `dependencies`が1つ以上のリアクティブな値を含む配列の場合、それらが変わるたびにエフェクトが実行されます。

- `dependencies`が空配列の場合、`useEffect`はコンポーネントが最初にレンダリングされたときだけ実行されます。

- `dependencies`を省略すると、コンポーネントがレンダリングまたは更新されるたびにエフェクトが実行されます。

## カスタムフックの作り方

- **カスタムフック**: カスタムフックはデータ取得、state管理、トグル、オンライン状態の追跡などの副作用のような再利用可能なロジックをコンポーネントから切り出すことができます。Reactの組み込みフックはすべて`use`で始まるので、カスタムフックも同じ慣例に従うべきです。

`useDebounce`フックを作成する例はこちらです：

```js
function useDebounce(value, delay) {
  const [debouncedValue, setDebouncedValue] = useState(value);

  useEffect(() => {
    const handler = setTimeout(() => {
      setDebouncedValue(value);
    }, delay);

    return () => {
      clearTimeout(handler);
    };
  }, [value, delay]);

  return debouncedValue;
}

export { useDebounce };
```
