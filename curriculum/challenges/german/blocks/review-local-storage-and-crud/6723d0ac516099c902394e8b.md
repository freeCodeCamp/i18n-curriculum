## Persistenter Speicher

- **Definition**: Persistenter Speicher bezeichnet eine Möglichkeit, Daten so zu speichern, dass sie auch nach dem Ausschalten des Geräts oder einem Neustart weiterhin verfügbar bleiben.

## Erstellen, Lesen, Aktualisieren, Löschen (CRUD)

- **Erstellen**: Dies bezeichnet den Prozess, neue Daten anzulegen. Zum Beispiel kann ein Nutzer in einer Webanwendung einen neuen Beitrag in einem Blog hinzufügen.
- **Lesen**: Dies ist die Operation, bei der Daten aus einer Datenbank abgerufen werden. Wenn Sie beispielsweise einen Blogbeitrag besuchen oder Ihr Profil auf einer Webseite ansehen, führen Sie eine Leseoperation aus, um die gespeicherten Daten abzurufen und anzuzeigen.
- **Aktualisieren**: Dies umfasst das Ändern vorhandener Daten in der Datenbank. Ein Beispiel wäre das Bearbeiten eines Blogbeitrags oder das Aktualisieren Ihrer Profilinformationen.
- **Löschen**: Dies ist die Operation, die Daten aus einer Datenbank entfernt. Wenn Sie beispielsweise einen Blogbeitrag oder ein Konto löschen, führen Sie eine Löschoperation aus.

## HTTP-Methoden

- **Definition**: HTTP steht für Hypertext Transfer Protocol und ist die Grundlage für die Datenkommunikation im Web. Es gibt HTTP-Methoden, die definieren, welche Aktionen auf Ressourcen im Web ausgeführt werden können. Die gängigen Methoden sind GET, POST, PUT, PATCH, DELETE.
- **`GET` Methode**: Diese wird verwendet, um Daten von einem Server abzurufen.
- **`POST` Methode**: Diese wird verwendet, um Daten an einen Server zu senden, der dadurch eine neue Ressource erstellt.
- **`PUT` Methode**: Diese wird verwendet, um eine Ressource vollständig zu ersetzen und zu aktualisieren.
- **`PATCH` Methode**: Diese wird verwendet, um eine Ressource teilweise zu aktualisieren.
- **`DELETE` Methode**: Diese wird verwendet, um Datensätze aus einer Datenbank zu entfernen.

## `localStorage` und `sessionStorage` Eigenschaften

- **Web Storage API**: Diese API bietet eine Möglichkeit für Browser, Schlüssel-Wert-Paare direkt im Browser zu speichern, sodass Entwickler Informationen speichern können, die über verschiedene Seitenaktualisierungen und Sitzungen hinweg verwendet werden können. Die beiden Hauptkomponenten der Web Storage API sind die `localStorage` und `sessionStorage` Eigenschaften.
- **`localStorage` Eigenschaft**: `localStorage` ist der Teil der Web Storage API, der es ermöglicht, Daten zu speichern, die auch nach dem Schließen des Browserfensters oder dem Aktualisieren der Seite erhalten bleiben. Diese Daten bleiben verfügbar, bis sie explizit von der Anwendung oder dem Nutzer entfernt werden.
- **`localStorage.setItem()` Methode**: Diese Methode wird verwendet, um ein Schlüssel-Wert-Paar in `localStorage` zu speichern.

```js
localStorage.setItem('username', 'Jessica');
```

- **`localStorage.getItem()` Methode**: Diese Methode wird verwendet, um den Wert eines bestimmten Schlüssels aus `localStorage` abzurufen.

```js
localStorage.setItem('username', 'codingRules');

let username = localStorage.getItem('username');
console.log(username); // codingRules
```

- **`localStorage.removeItem()` Methode**: Diese Methode wird verwendet, um einen bestimmten Eintrag aus `localStorage` anhand seines Schlüssels zu entfernen.

```js
localStorage.removeItem('username');
```

- **`localStorage.clear()` Methode**: Diese Methode wird verwendet, um alle gespeicherten Daten in `localStorage` zu löschen.

```js
localStorage.clear();
```

- **`sessionStorage` Eigenschaft**: Speichert Daten, die nur für die aktuelle Sitzung gelten und gelöscht werden, wenn der Browser-Tab oder das Fenster geschlossen wird.
- **`sessionStorage.setItem()` Methode**: Diese Methode wird verwendet, um ein Schlüssel-Wert-Paar in `sessionStorage` zu speichern.

```js
sessionStorage.setItem('cart', '3 items');
```

- **`sessionStorage.getItem()` Methode**: Diese Methode wird verwendet, um den Wert eines bestimmten Schlüssels aus `sessionStorage` abzurufen.

```js
sessionStorage.setItem('cart', '3 items');

let cart = sessionStorage.getItem('cart');
console.log(cart); // '3 items'
```

- **`sessionStorage.removeItem()` Methode**: Diese Methode wird verwendet, um einen bestimmten Eintrag aus `sessionStorage` anhand seines Schlüssels zu entfernen.

```js
sessionStorage.removeItem('cart');
```

- **`sessionStorage.clear()` Methode**: Diese Methode wird verwendet, um alle gespeicherten Daten in `sessionStorage` zu löschen.

```js
sessionStorage.clear();
```

## Arbeiten mit Cookies

- **Definition**: Cookies, auch Web-Cookies oder Browser-Cookies genannt, sind kleine Datenstücke, die ein Server an den Webbrowser eines Nutzers sendet. Diese Cookies werden auf dem Gerät des Nutzers gespeichert und bei nachfolgenden Anfragen an den Server zurückgesendet. Cookies sind wichtig, damit Webanwendungen den Zustand aufrechterhalten und Nutzerinformationen speichern können, was besonders wichtig ist, da HTTP ein zustandsloses Protokoll ist.
- **Session-Cookies**: Diese Cookies gelten nur für die Dauer der Sitzung des Nutzers auf der Webseite. Sobald der Nutzer den Browser oder Tab schließt, wird das Session-Cookie gelöscht. Diese Cookies werden typischerweise verwendet, um einen Nutzer während seines Besuchs eingeloggt zu halten.
- **Sichere Cookies**: Diese Cookies werden nur über HTTPS gesendet, wodurch sichergestellt wird, dass sie unterwegs nicht von Angreifern abgefangen werden können.
- **HttpOnly-Cookies**: Diese Cookies können von JavaScript im Browser nicht aufgerufen oder verändert werden, was sie sicherer gegen Cross-Site-Scripting-(XSS-)Angriffe macht.
- **Set-Cookie-Kopfzeile**: Wenn Sie eine Webseite besuchen, kann der Server in der HTTP-Antwort eine Set-Cookie-Kopfzeile senden. Diese weist Ihren Browser an, ein Cookie mit bestimmten Informationen zu speichern. Zum Beispiel könnte eine eindeutige ID gespeichert werden, die der Seite hilft, Sie beim nächsten Besuch wiederzuerkennen.
Sie können ein Cookie manuell in JavaScript mit `document.cookie` setzen:

```js
document.cookie = "organization=freeCodeCamp; expires=Fri, 31 Dec 2021 23:59:59 GMT; path=/";
```

Um ein Cookie zu löschen, können Sie sein Ablaufdatum auf eine Zeit in der Vergangenheit setzen.

```js
document.cookie = "username=; expires=Thu, 01 Jan 1970 00:00:00 GMT; path=/";
```

## Cache API

- **Definition**: Caching ist der Prozess, Kopien von Dateien an einem temporären Speicherort abzulegen, damit sie schneller abgerufen werden können. Die Cache API wird verwendet, um Netzwerk-Anfragen und -Antworten zu speichern, wodurch Webanwendungen effizienter arbeiten und sogar offline funktionieren können. Sie ist Teil der umfassenderen Service Worker API und entscheidend für die Erstellung von Progressive Web Apps (PWAs), die auch bei unzuverlässigen oder langsamen Netzwerkverbindungen funktionieren.
Die Cache API ist ein Speichersystem, das Request- und Response-Objekte speichert. Wenn eine Anfrage an einen Server gesendet wird, kann die Anwendung die Antwort speichern und später aus dem Cache abrufen, anstatt eine neue Netzwerk-Anfrage zu stellen. Das reduziert Ladezeiten, spart Bandbreite und verbessert die Nutzererfahrung.
- **Cache-Speicher**: Dieser wird verwendet, um Schlüssel-Wert-Paare von HTTP-Anfragen und deren zugehörigen Antworten zu speichern. Dadurch können zuvor angeforderte Ressourcen effizient abgerufen werden, ohne sie bei späteren Besuchen erneut aus dem Netzwerk laden zu müssen, was die Leistung verbessert.
- **Cache-Control**: Entwickler können definieren, wie lange eine zwischengespeicherte Ressource aufbewahrt werden soll und ob sie neu validiert oder direkt aus dem Cache ausgeliefert wird.
- **Offline-Unterstützung**: Durch die Verwendung der Cache API können Sie Offline-First-Webanwendungen erstellen. Zum Beispiel kann eine PWA zwischengespeicherte Ressourcen bereitstellen, wenn der Nutzer keine Netzwerkverbindung hat.

## Negative Muster und clientseitiger Speicher

- **Übermäßiges Tracking**: Dies bezeichnet die Praxis, eine übermäßige Menge an Nutzerdaten im clientseitigen Speicher (wie Cookies, Local Storage oder Session Storage) ohne klare, informierte Zustimmung oder legitimen Bedarf zu sammeln und zu speichern. Dabei wird oft das Verhalten, die Vorlieben und Interaktionen der Nutzer über mehrere Seiten oder Sitzungen hinweg verfolgt, was die Privatsphäre der Nutzer beeinträchtigen kann.
- **Browser-Fingerprinting**: Eine Technik, mit der einzelne Nutzer anhand einzigartiger Merkmale ihres Geräts und Browsers identifiziert und verfolgt werden, ohne auf Cookies oder andere traditionelle Tracking-Methoden zurückzugreifen. Im Gegensatz zu Cookies, die lokal auf dem Gerät gespeichert werden, sammelt Fingerprinting eine Vielzahl von Informationen, um einen unverwechselbaren „Fingerabdruck“ der Browsersitzung zu erstellen.
- **Passwörter im LocalStorage speichern**: Dies mag wie ein offensichtliches negatives Muster erscheinen, aber das Speichern sensibler Daten wie Passwörter im Local Storage stellt ein Sicherheitsrisiko dar. Local Storage ist nicht verschlüsselt und kann leicht ausgelesen werden. Daher sollten Sie niemals sensible Daten dort speichern.

## IndexedDB

- **Definition**: IndexedDB wird verwendet, um strukturierte Daten im Browser zu speichern. Es ist in modernen Webbrowsern integriert und ermöglicht Webanwendungen, JavaScript-Objekte effizient zu speichern und abzurufen.

## Cache-/Service Worker

- **Definition**: Ein Service Worker ist ein Skript, das im Hintergrund läuft und von Ihrer Webseite getrennt ist. Er kann Netzwerk-Anfragen abfangen, auf den Cache zugreifen und die Webanwendung offline funktionsfähig machen. Dies ist eine Schlüsselkomponente von Progressive Web Apps.
