# --description--

Заголовки (елементи `h1` до `h6`) — це основні теги, які допомагають структурувати та маркувати ваш вміст. Читачі екрана можуть бути налаштовані так, щоб читати лише заголовки на сторінці, щоб користувач отримав короткий огляд. Це означає, що важливо, щоб теги заголовків у вашій розмітці мали семантичне значення і були пов’язані один з одним, а не вибиралися лише за розміром.

*Семантичне значення* означає, що тег, який ви використовуєте навколо вмісту, вказує на тип інформації, яку він містить.

Якщо ви писали б роботу з вступом, основною частиною та висновком, було б нелогічно робити висновок підрозділом основної частини в вашому плані. Він має бути окремим розділом. Аналогічно, теги заголовків на вебсторінці повинні йти за порядком і показувати ієрархічні зв’язки вашого вмісту.

Заголовки з однаковим (або вищим) рівнем починають нові умовні розділи, заголовки з нижчим рівнем починають підрозділи попереднього.

Наприклад, сторінка з елементом `h2`, за яким ідуть кілька підрозділів, позначених елементами `h4`, може збити з пантелику користувача читача екрана. При шести варіантах спокуса використати тег, який виглядає краще в браузері, велика, але ви можете використовувати CSS для редагування відносних розмірів.

І наостанок, кожна сторінка завжди має містити один (і лише один) елемент `h1`, який є головною темою вашого вмісту. Цей та інші заголовки частково використовуються пошуковими системами для розуміння теми сторінки.

# --instructions--

Кіт Кемпер хоче сторінку на своєму сайті, присвячену тому, як стати ніндзя. Допоможіть йому виправити заголовки, щоб його розмітка мала семантичне значення для вмісту і показувала правильні батьківсько-дочірні зв’язки розділів. Змініть усі теги `h5` на правильний рівень заголовка, щоб показати, що вони є підрозділами тегів `h2`. Використовуйте теги `h3` для цієї мети.

# --hints--

Ваш код має містити 6 елементів `h3`.

```js
assert.lengthOf(document.querySelectorAll('h3') , 6);
```

Ваш код має містити 6 кінцевих тегів `h3`.

```js
assert.lengthOf((code.match(/\/h3/g) || []) ,6);
```

Ваш код не повинен містити жодних елементів `h5`.

```js
assert.isEmpty(document.querySelectorAll('h5'));
```

Ваш код не повинен містити жодних кінцевих тегів `h5`.

```js
assert.notMatch(code, /\/h5/);
```
