# --interactive--

In der vorherigen Lektion haben Sie gelernt, wie man mit Funktionen arbeitet, das sind wiederverwendbare Codeabschnitte, die Ihren Code modularer, leichter wartbar und effizienter machen. Alle bisherigen Beispiele verwendeten die reguläre Funktionssyntax, so wie hier:

```js
function greetings(name) {
  console.log("Hello, " + name + "!");
}
```

Eine andere Möglichkeit, Funktionen in JavaScript zu schreiben, ist die Erstellung eines Pfeilfunktions-Ausdrucks. So können Sie das vorherige Beispiel umschreiben, um stattdessen die Pfeilfunktions-Syntax zu verwenden:

```js
const greetings = (name) => {
  console.log("Hello, " + name + "!");
};
```

In diesem überarbeiteten Beispiel erstellen wir eine `const`-Variable namens `greetings` und weisen ihr eine anonyme Funktion zu. Die meisten Teile der Syntax werden Ihnen vertraut vorkommen, außer dass das `function`-Schlüsselwort fehlt und stattdessen ein Pfeil (`=>`) zwischen dem `name`-Parameter und dem Funktionskörper steht. Wenn Ihre Parameterliste nur einen Parameter enthält, können Sie die runden Klammern weglassen, so:

```js
const greetings = name => {
  console.log("Hello, " + name + "!");
};
```

Hat Ihre Pfeilfunktion keine Parameter, müssen Sie die runden Klammern verwenden, so:

```js
const greetings = () => {
  console.log("Hello");
};
```

Beim ersten Lernen über Funktionen mussten Sie den Funktionskörper in geschweifte Klammern einschließen. Wenn Ihr Funktionskörper jedoch nur eine einzelne Codezeile enthält, können Sie die geschweiften Klammern weglassen, so:

```js
const greetings = name => console.log("Hello, " + name + "!");
```

Es ist wichtig zu beachten, dass das Weglassen der runden und geschweiften Klammern bei der regulären Funktionssyntax nicht funktioniert. Sie erhalten Fehler, wenn Sie so etwas versuchen:

```js
// This will produce syntax errors 
function greetings name console.log("Hello, " + name + "!");
```

Diese Art von Einzeilern funktioniert nur mit der Pfeilfunktions-Syntax. Ein weiterer wichtiger Begriff ist die `return`-Anweisung. Hier ein Beispiel, wie man mit der Pfeilfunktions-Syntax die Fläche berechnet:

:::interactive_editor

```js
const calculateArea = (width, height) => {
  const area = width * height;
  return area;
};

console.log(calculateArea(5, 3)); // 15
```

:::

Wir erstellen eine Variable innerhalb der Funktion namens `area` und geben diese dann zurück. Aber wir könnten unseren Code etwas aufräumen und direkt die Berechnung zurückgeben:

:::interactive_editor

```js
const calculateArea = (width, height) => {
  return width * height;
}; 

console.log(calculateArea(5, 3)); // 15
```

:::

Wenn Sie versuchen, die geschweiften Klammern zu entfernen und die Berechnung in derselben Zeile zu platzieren, erhalten Sie eine `Uncaught SyntaxError: Unexpected token 'return'`-Meldung:

```js
const calculateArea = (width, height) => return width * height;
```

Der Grund für diesen Fehler ist, dass Sie die `return`-Anweisung entfernen müssen. Wenn Sie diese `return`-Anweisung entfernen, verschwindet der Fehler und die Funktion gibt die Berechnung weiterhin implizit zurück.

```js
const calculateArea = (width, height) => width * height;
```

Wann sollten Sie also die Pfeilfunktions-Syntax verwenden? Das hängt davon ab. Viele Entwickler verwenden sie konsequent in ihren persönlichen Projekten. Wenn Sie jedoch im Team arbeiten, hängt die Wahl meist davon ab, ob der bestehende Code reguläre Funktionen oder Pfeilfunktionen verwendet. In zukünftigen Lektionen werden wir behandeln, wann Sie Pfeilfunktionen verwenden sollten und wann nicht.
