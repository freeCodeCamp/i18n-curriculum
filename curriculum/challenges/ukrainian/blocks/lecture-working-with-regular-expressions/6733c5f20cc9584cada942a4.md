# --interactive--

Захоплююча група дозволяє "захопити" частину співпадаючого рядка, щоб використати її так, як вам потрібно. Захоплюючі групи визначаються круглими дужками, які містять шаблон для захоплення, без провідних символів, як-от випередження.

Давайте захопимо `code` з нашого регулярного виразу `freeCodeCamp`. Для цього ми помістимо `code` у круглі дужки і визначимо як захоплюючу групу:

```js
const regex = /free(code)camp/i;
```

Щоб підтвердити поведінку, ми можемо протестувати це на рядку `freecodecamp`:

:::interactive_editor

```js
const regex = /free(code)camp/i;
console.log(regex.test("freecodecamp")); // true
```

:::

Але це насправді не використовує нашу захоплену групу. Замість цього подивімося на результат використання `match`:

```js
const regex = /free(code)camp/i;
console.log("freecodecamp".match(regex));
// [
//   'freecodecamp',
//   'code', <--
//   index: 0,
//   input: 'freecodecamp',
//   groups: undefined
// ]
```

Тут ми бачимо, що наш масив `match` має другий елемент, який є частиною рядка, захопленою нашою захоплюючою групою.

Зверніть увагу, як захоплююча група співпадає з точним шаблоном `code`, де клас символів співпадає з одним символом зі списку `c`, `o`, `d` і `e`.

Але як ми можемо це фактично використати? Захоплюючі групи часто використовують при заміні вмісту рядка. Давайте налаштуємо код для цього. Ми перетворимо `freecodecamp` у `paidcodeworld`:

:::interactive_editor

```js
const regex = /free(code)camp/i;
console.log("freecodecamp".replace(regex, "paidcodeworld"));
```

:::

Це працює само по собі, але що, якби ми не знали, скільки `o` у `code`? Якщо нам потрібен квантор для одного або більше `o`:

:::interactive_editor

```js
const regex = /free(co+de)camp/i;
console.log("freecoooooooodecamp".replace(regex, "paidcodeworld"));
```

:::

Ми отримуємо `paidcodeworld` як результат. Ми хочемо зберегти кількість `o`, тому нам потрібно повторно використати те, що було захоплено регулярним виразом.

Ось тут і з’являється референс. Замість того, щоб жорстко кодувати `code` частину нашого рядка заміни, ми можемо посилатися безпосередньо на захоплену групу.

У виклику `replace` референс до захопленої групи досягається за допомогою знака долара (`$`), за яким йде номер захоплюючої групи, яку потрібно використати. У нашому випадку це буде `$1`, оскільки `code` захоплено в першій захоплюючій групі:

:::interactive_editor

```js
const regex = /free(co+de)camp/i;
console.log("freecoooooooodecamp".replace(regex, "paid$1world")); // paidcoooooooodeworld
```

:::

Тепер ми успішно зберегли невідому кількість символів `o` при перетворенні `freecodecamp` у `paidcodeworld`. Але референси не обмежуються лише викликом replace. Ви можете фактично використовувати їх безпосередньо в регулярному виразі.

Це дозволить вам співпадати з раніше захопленим шаблоном пізніше в регулярному виразі.

Припустимо, ми хочемо співпасти `freecodecamp` двічі, з однаковою кількістю `o`, але в будь-якому місці рядка.

Спочатку нам потрібно розділити їх нашим символом підстановки і дозволити будь-яку кількість символів, щоб співпасти з цим символом:

```js
const regex = /free(co+de)camp.*free(co+de)camp/i;
```

Цей поточний вираз не гарантує, що кількість символів `o` буде однаковою. Щоб досягти цього, нам потрібно замінити другу захоплюючу групу посиланням на першу.

У регулярному виразі референс позначається зворотною скісною рискою, за якою йде номер захоплюючої групи:

:::interactive_editor

```js
const regex = /free(co+de)camp.*free\1camp/i;
console.log(regex.test("freecooooodecamp is great i love freecooooodecamp")); // true
console.log(regex.test("freecooooodecamp is great i love freecodecamp")); // false
```

:::

І з цим ми бачимо, що рядок з правильною кількістю `o` співпадає, тоді як рядок з двома різними кількостями `o` не співпадає.

Цей синтаксис чудовий, але може швидко заплутати, коли ви посилаєтеся на кілька захоплюючих груп. На щастя, замість використання номерів, ви можете дати своїм групам імена.

Щоб визначити іменовану захоплюючу групу, додайте знак питання (`?`) після якого йде ім’я в дужках меншого та більшого знаків на початку групи. Назвемо нашу захоплюючу групу `code`:

```js
const regex = /free(?<code>co+de)camp.*free\1camp/i;
```

Тепер ми можемо оновити наш референс у регулярному виразі, щоб посилатися на цю групу. Іменований референс починається зі зворотної скісної риски, за якою в JavaScript йде літера `k`. Потім додається ім’я, знову в дужках меншого (`<`) та більшого (`>`) знаків. Подивімося на це:

```js
const regex = /free(?<code>co+de)camp.*free\k<code>camp/i;
```

Якщо ми перевіримо наш виклик `test()`, ми побачимо, що він все ще проходить:

:::interactive_editor

```js
const regex = /free(?<code>co+de)camp.*free\k<code>camp/i;
console.log(regex.test("freecooooodecamp is freecooooodecamp")); // true
```

:::

Щоб використати нашу іменовану захоплюючу групу у виклику `replace()`, ми вставляємо знак долара у рядок, за яким йде ім’я в дужках меншого та більшого знаків:

:::interactive_editor

```js
const regex = /free(?<code>co+de)camp/i;
console.log("freecooooodecamp".replace(regex, "paid$<code>camp")); // paidcooooodecamp
```

:::

Нарешті, іноді ви хочете створити групу символів, але не потребуєте збереження захопленого результату.

Припустимо, ми хочемо співпасти або `freecodecamp`, або `freecandycamp`. Ви могли б створити два шаблони, розділені твердженням OR:

```js
const regex = /freecodecamp|freecandycamp/i;
```

Але це може стати досить довгим для великих регулярних виразів. Замість цього ви можете створити негрупуючу захоплюючу групу навколо символів, які потрібно об’єднати через OR:

```js
const regex = /free(?:code|candy)camp/i;
```

Негрупуюча захоплююча група не зберігає співпадіння `code|candy` окремо в пам’яті. Але вона може бути корисною для створення альтернативних шаблонів без втрати читабельності чи продуктивності.
