## `Audio` costruttore e metodi comuni

- **Definizione**: Il costruttore `Audio`, come altri costruttori, è una funzione speciale chiamata con la parola chiave `new`. Restituisce un `HTMLAudioElement`, che puoi poi usare per riprodurre audio per l’utente o aggiungere al DOM affinché l’utente possa controllarlo da solo. Quando chiami il costruttore, puoi opzionalmente passare un URL come (unico) argomento. Questo URL dovrebbe puntare alla sorgente del file audio che vuoi riprodurre. Oppure, se hai bisogno di cambiare la sorgente dinamicamente, puoi assegnare l’URL alla proprietà `src` dell’elemento audio restituito.
- **Metodo `play()`**: Questo metodo si usa con gli elementi `audio` o `video` per iniziare la riproduzione del media.

:::interactive_editor

```html
<audio id="audio" controls>
  <source src="https://www.w3schools.com/html/horse.mp3" type="audio/mpeg">
</audio>

<button id="playBtn">Play Audio</button>

<script>
  const audio = document.getElementById("audio");
  const playBtn = document.getElementById("playBtn");

  playBtn.addEventListener("click", () => {
    audio.play();
  });
</script>

:::

- **`pause()` Method**: This method is used with the `audio` or `video` elements to pause playback for the media. It maintains the current position, so if `play()` is called, it starts from that position.

:::interactive_editor

```html
<audio id="myAudio" controls>
  <source src="https://www.w3schools.com/html/horse.mp3" type="audio/mpeg">
</audio>

<br>

<button id="pauseBtn">Pausa audio</button>

<script>
  document.addEventListener("DOMContentLoaded", () => {
    const audio = document.getElementById("myAudio");
    const pauseBtn = document.getElementById("pauseBtn");

    pauseBtn.addEventListener("click", () => {
      audio.pause();
    });
  });
</script>

:::

- **Metodo `addTextTrack()`**: Questo metodo ti permette di specificare una traccia testuale da associare all’elemento media — molto utile per aggiungere sottotitoli a un video.
- **Metodo `fastSeek()`**: Questo metodo ti permette di spostare la posizione di riproduzione a un momento specifico all’interno del media.

Il costruttore `Audio()` ha proprietà proprio come ha metodi. Tra queste proprietà ci sono:

- `currentTime`: per ottenere il tempo corrente di riproduzione di un audio
- `loop`: per far riprodurre l’audio in modo continuo riavviandolo automaticamente quando arriva alla fine
- `muted`: per silenziare l’output audio di un elemento media indipendentemente dal volume impostato

## Formati audio e video diversi

- **Tipo MIME**: Un tipo MIME, che sta per Multipurpose Internet Mail Extensions, è un modo standardizzato per indicare programmaticamente un tipo di file. Il tipo MIME può dire a un’applicazione, come il tuo browser, come gestire un file specifico. Nel caso di audio e video, il tipo MIME indica che si tratta di un formato multimediale che può essere incorporato nella pagina web.
- **Elemento `source`**: Questo si usa per specificare un tipo di file e una sorgente — e può includere più tipi diversi usando più elementi source. Quando fai così, il browser determina il formato migliore da usare per l’ambiente attuale dell’utente.
- **MP3**: È un tipo di formato digitale usato per archiviare musica, audio o suoni. È una versione compressa di una registrazione sonora che rende il file più piccolo, quindi più facile da archiviare e condividere. MP3 ha il supporto più ampio nei browser e il tipo MIME `audio/mpeg`.
- **MP4**: Un MP4 è un tipo di formato digitale usato per archiviare video e audio. Funziona come un contenitore che contiene sia il video (immagini) sia il suono (musica o voce) in un unico file. Un MP4 può avere il tipo MIME `audio/mp4` O `video/mp4`, a seconda che sia un file video o solo audio.
- Altri formati sono `WMV` associato all’app Windows Media Player, `OGG`, `MKV`, `AVI` e altri.

## codec

- **Definizione**: Un codec, abbreviazione di "encoder/decoder", è un algoritmo o software che può convertire audio e video tra formati analogici e digitali. I codec possono essere specificati come parte del tipo MIME. La sintassi base per definire un codec è aggiungere un punto e virgola dopo il tipo di media, poi `codecs=` e il codec.

## API HTMLMediaElement

- **Definizione**: L’API `HTMLMediaElement` si usa per controllare il comportamento degli elementi audio e video nella tua pagina. Estende l’interfaccia base `HTMLElement`, quindi hai accesso alle proprietà base, ai metodi e anche a eventi utili come `waiting`, `ended`, `canplay`, `canplaythrough` e altri. Esempi di metodi includono `play()`, `fastSeek()`, `pause()` e `canPlayType()` per verificare se un browser probabilmente può riprodurre un file audio.

## API Media Capture and Streams

- **Definizione**: L’API Media Capture and Streams, o MediaStream API, si usa per catturare audio e video dal tuo dispositivo. Per usare l’API, devi creare l’oggetto `MediaStream`. Potresti farlo con il costruttore, ma non sarebbe collegato all’hardware dell’utente. Invece, la proprietà `mediaDevices` dell’oggetto navigator `global` ha un metodo `getUserMedia()` che puoi usare.

```js
window.navigator.mediaDevices.getUserMedia({
  audio: true,
  video: {
    width: {
      min: 1280,
      ideal: 1920,
      max: 3840
    },
    height: {
      min: 720,
      ideal: 1080,
      max: 2160
    }
  }
});
```

## API Screen Capture

- **Definizione**: L’API Screen Capture si usa per registrare lo schermo di un utente. Questa API si espone chiamando il metodo `getDisplayMedia()` dell’oggetto `mediaDevices` e consumando il media stream restituito.

## API MediaStream Recording

- **Definizione**: L’API MediaStream Recording funziona insieme alle API MediaStreams, permettendoti di registrare un MediaStream (o anche un `HTMLMediaElement` direttamente).

## API Media Source Extensions

- **argomento**: L’API Media Source Extensions permette di passare direttamente il feed della webcam di un utente a un elemento video con la proprietà `srcObject`.

## API Web Audio

- **Definizione**: L’API Web Audio alimenta tutto ciò che è udibile sul web. Questa API include oggetti importanti come un `AudioBuffer` (che rappresenta un buffer contenente specificamente dati audio) o il `AudioContext`, usato per rappresentare un grafo di elaborazione audio.
