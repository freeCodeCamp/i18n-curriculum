# --description--

Da Redux ein Framework zur Zustandsverwaltung ist, gehört das Aktualisieren des Zustands zu seinen Kernaufgaben. In Redux werden alle Zustandsaktualisierungen durch das Versenden von Actions ausgelöst. Eine Action ist einfach ein JavaScript-Objekt, das Informationen über ein aufgetretenes Ereignis enthält. Das Redux-Speichern empfängt diese Action-Objekte und aktualisiert dann entsprechend seinen Zustand. Manchmal enthält eine Redux-Action auch einige Daten. Zum Beispiel enthält die Action einen Benutzernamen, nachdem sich ein Benutzer angemeldet hat. Während die Daten optional sind, müssen Actions eine `type`-Eigenschaft enthalten, die den 'type' der aufgetretenen Action angibt.

Betrachten Sie Redux-Actions als Boten, die Informationen über Ereignisse in Ihrer App an das Redux-Speichern übermitteln. Das Speichern führt dann die Aufgabe aus, den Zustand basierend auf der aufgetretenen Action zu aktualisieren.

# --instructions--

Eine Redux-Action zu schreiben ist so einfach wie das Deklarieren eines Objekts mit einer type-Eigenschaft. Deklarieren Sie ein Objekt `action` und geben Sie ihm eine Eigenschaft `type`, die auf den String `'LOGIN'` gesetzt ist.

assert(

Das `action`-Objekt sollte eine Schlüssel-Eigenschaft `type` mit dem Wert `LOGIN` haben.

    return action.type === 'LOGIN';
  })()
