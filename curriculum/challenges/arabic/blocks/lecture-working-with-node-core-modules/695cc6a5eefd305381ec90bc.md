# --description--

توفر العديد من اللغات وأُطُر العمل طرقًا للعمل مع الملفات والمجلدات. يتيح لك Node.js القيام بذلك باستخدام وحدة `fs` أو نظام الملفات.

سيركز هذا الدرس على وحدة Node `fs`، كيف تعمل، والثلاث طرق المختلفة التي يمكنك استخدامها مع الطرق التي توفرها.

توفر وحدة Node `fs` طرقًا للعمل مع الملفات والمجلدات، بما في ذلك فتحها وإغلاقها، وعمليات القراءة والكتابة، والحذف.

وحدة `fs` هي وحدة قياسية في Node.js، لذا فهي متاحة للاستخدام طالما أن Node.js مثبت في بيئة العمل لديك. لاستخدام الوحدة، استوردها بهذه الطريقة:

```js
const fs = require("fs");
```

المرونة التي تقدمها وحدة `fs` هي أنها تسمح لك باستخدام طرقها بشكل متزامن وغير متزامن. طرق وحدة `fs` غير متزامنة افتراضيًا، لكن لكل طريقة هناك شكل متزامن:

```js
fs.writeFile() // Asynchronous file writing
fs.writeFileSync() // Synchronous file writing

fs.readFile() // Asynchronous file reading
fs.readFileSync() // Synchronous file reading

fs.open() // Opens a file
fs.openAsBlob() // Opens as blob
fs.openSync() // Synchronous open
fs.opendir() // Opens directory
fs.opendirSync() // Synchronous directory open
```

لا يتوقف الأمر عند هذا الحد. حسب حاجتك، يمكنك استخدام الطرق بثلاث طرق:

- مع استدعاءات راجعة، مثل `fs.writeFile()` (غير متزامن)
- مع الوعود إذا فضلت صيغة `async/await`، مثل `fs.promises.writeFile()`
- بشكل متزامن، مثل `fs.writeFileSync()`

إليك الصيغة الأساسية لاستخدام الطرق بشكل غير متزامن:

```js
fs.writeFile("filePath", "content", "utf8", (err) => {
  if (err) {
    throw err;
  }
  console.log("File written to!");
});
```

لإصدار الوعود، يمكنك استيراده من `fs/promises` أو ربط `promises` مع استيراد `fs` الخاص بك. إليك الصيغة:

```js
async function promisesExample() {
  try {
    await fs.promises.writeFile("filePath", "content", "utf8");
    console.log("File written to!");
  } catch (err) {
    console.error("Error:", err);
  }
}

promisesExample();
```

وهنا الإصدار المتزامن:

```js
try {
  fs.writeFileSync("filePath", "content", "utf8");
  console.log("File written to!");
} catch (err) {
  console.error("Error:", err);
}
```

لاحظ أن استخدام أي من الطرق بشكل متزامن يكون حجبًا. بمعنى آخر، سيتوقف برنامجك عن العمل وينتظر حتى تنتهي العملية قبل الانتقال إلى السطر التالي من الشفرة.

للسكربتات السريعة، المشاريع الصغيرة، أو المهام لمرة واحدة، الطرق المتزامنة ربما تكون مناسبة. لكن في التطبيقات الحقيقية، يصبح حجب الإدخال/الإخراج مشكلة لأنه قد يجمد أجزاء أخرى من التطبيق بينما ينتظر Node انتهاء عمليات نظام الملفات. هذا يبطئ الأمور ويؤثر سلبًا على الأداء وتجربة المستخدم.

لهذا السبب توجد الطرق غير المتزامنة. تسمح لبقية برنامجك بالاستمرار في العمل بينما يتولى Node عمليات الملفات في الخلفية. تقليديًا، كان هذا يتم باستخدام الاستدعاءات الراجعة، لكنها قد تصبح فوضوية بسرعة مع نمو التطبيق.

تحل الوعود و`async/await` هذه المشكلة. فهي لا تزال غير حابسة، لكن الشفرة تُقرأ كأنها شفرة متزامنة عادية وأسهل بكثير في الصيانة.

لهذا السبب، يُفضل اليوم استخدام النهج القائم على الوعود، وهذا ما سنستخدمه في بقية هذا الدرس.

تسمح لك الطريقة `writeFile()` بالكتابة إلى ملف موجود. إذا لم يكن الملف موجودًا، تنشئه في الدليل الحالي، ثم تكتب المحتوى المحدد فيه.

إليك كيفية استخدام الطريقة `writeFile()`:

```js
const fs = require("fs/promises");

async function writeToFile() {
  try {
    await fs.writeFile(
      "article.md",
      "## Node `fs` Module: The Complete Guide",
      "utf8",
    );
    console.log("File written to!");
  } catch (err) {
    console.error("Error writing to file:", err);
  }
}

writeToFile(); // File written to!
```

إليك محتوى ملف article.md بعد ذلك:

```md
## Node `fs` Module: The Complete Guide
```

تسمح لك الطريقة `appendFile()` بإضافة محتوى إلى ملف موجود. إليك كيفية استخدامها:

```js
const fs = require("fs/promises");

async function appendToFile() {
  try {
    await fs.appendFile(
      "article.md",
      "\nIn this article, you will learn all there is to know about the Node fs module...",
      "utf8",
    );
    console.log("File appended to!");
  } catch (err) {
    console.log("Error appending to file:", err);
  }
}

appendToFile(); // File appended to!
```

تسمح لك الطريقة `readFile()` برؤية محتوى الملف:

```js
const fs = require("fs/promises");

async function readFileContent() {
  try {
    const fileContent = await fs.readFile("article.md", "utf8");
    console.log("File content:", fileContent);
  } catch (err) {
    console.log("File read successfully");
  }
}

readFileContent();

/*
File content: ## Node `fs` Module: The Complete Guide

In this article, you will learn all there is to know about the Node fs module...
*/
```

إذا لم تحدد ترميز الأحرف `utf8`، ستحصل على محتوى الملف كمخزن مؤقت:

```js
async function readFileContent() {
  try {
    const fileContent = await fs.readFile("article.md");
    console.log("File content:", fileContent);
  } catch (err) {
    console.log("File read successfully");
  }
}

readFileContent();

/*
File content: <Buffer 23 23 20 4e 6f 64 65 20 60 66 73 60 20 4c 69 62 72 61 72 79 3a 20
 54 68 65 20 43 6f 6d 70 6c 65 74 65 20 47 75 69 64 65 0a 0a 49 6e 20 74 68 69 73 20 ... 
72 more bytes>
*/
```

في الدرس التالي، ستتعلم كيفية التعامل مع مخزن مؤقت مثل هذا باستخدام وحدة Buffer في Node.js.

أخيرًا، تسمح لك الطريقة `unlink()` بحذف ملف:

```js
const fs = require("fs/promises");

async function deleteFile() {
  try {
    await fs.unlink("article.ts");
    console.log("File deleted successfully!");
  } catch (err) {
    console.error("Error deleting file:", err);
  }
}

deleteFile();
```

إذا لم يكن الملف موجودًا، ستحصل على رسالة خطأ مشابهة لهذه:

```bash
Error deleting file: Error: ENOENT: no such file or directory, unlink 'article.ts'
```

إذا كان موجودًا، يُحذف الملف، وتحصل على رسالة النجاح التي حددتها.
