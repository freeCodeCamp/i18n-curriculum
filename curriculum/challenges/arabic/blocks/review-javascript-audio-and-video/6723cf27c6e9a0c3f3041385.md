# --interactive--

## دالة البناء `Audio` والطرق الشائعة

- **التعريف**: دالة البناء `Audio`، مثل دوال البناء الأخرى، هي دالة خاصة تُستدعى باستخدام الكلمة المفتاحية `new`. تُعيد عنصر `HTMLAudioElement`، يمكنك استخدامه لتشغيل الصوت للمستخدم أو إضافته إلى DOM ليتمكن المستخدم من التحكم به بنفسه. عند استدعاء دالة البناء، يمكنك اختيارياً تمرير عنوان URL كالمعلمة الوحيدة. يجب أن يشير هذا العنوان إلى مصدر ملف الصوت الذي تريد تشغيله. أو، إذا احتجت لتغيير المصدر ديناميكيًا، يمكنك تعيين العنوان إلى خاصية `src` لعنصر الصوت المعاد.
- **طريقة `play()`**: تُستخدم هذه الطريقة مع عناصر `audio` أو `video` لبدء تشغيل الوسائط.

:::interactive_editor

```html
<audio id="audio" controls>
  <source src="https://www.w3schools.com/html/horse.mp3" type="audio/mpeg">
</audio>

<button id="playBtn">Play Audio</button>

<script>
  const audio = document.getElementById("audio");
  const playBtn = document.getElementById("playBtn");

  playBtn.addEventListener("click", () => {
    audio.play();
  });
</script>
```

:::

- **طريقة `pause()`**: تُستخدم هذه الطريقة مع عناصر `audio` أو `video` لإيقاف تشغيل الوسائط مؤقتًا. تحافظ على الموضع الحالي، فإذا استُدعي `play()` يبدأ التشغيل من ذلك الموضع.

:::interactive_editor

```html
<audio id="myAudio" controls>
  <source src="https://www.w3schools.com/html/horse.mp3" type="audio/mpeg">
</audio>

<br>

<button id="pauseBtn">Pause Audio</button>

<script>
  document.addEventListener("DOMContentLoaded", () => {
    const audio = document.getElementById("myAudio");
    const pauseBtn = document.getElementById("pauseBtn");

    pauseBtn.addEventListener("click", () => {
      audio.pause();
    });
  });
</script>
```

:::

- **طريقة `addTextTrack()`**: تتيح لك هذه الطريقة تحديد مسار نصي مرتبط بعنصر الوسائط - وهو مفيد بشكل خاص لإضافة ترجمات إلى الفيديو.
- **طريقة `fastSeek()`**: تتيح لك هذه الطريقة تحريك موضع التشغيل إلى وقت محدد داخل الوسائط.

دالة البناء `Audio()` لها خصائص تمامًا كما لها طرق. تشمل هذه الخصائص:

- `currentTime`: للحصول على وقت التشغيل الحالي للصوت
- `loop`: لجعل الصوت يعاد تشغيله تلقائيًا باستمرار عند الوصول للنهاية
- `muted`: لكتم صوت عنصر الوسائط بغض النظر عن إعداد مستوى الصوت

## صيغ الصوت والفيديو المختلفة

- **نوع MIME**: نوع MIME، اختصار Multipurpose Internet Mail Extensions، هو طريقة معيارية للإشارة برمجيًا إلى نوع ملف. يمكن لنوع MIME أن يخبر التطبيق، مثل متصفحك، بكيفية التعامل مع ملف معين. في حالة الصوت والفيديو، يشير نوع MIME إلى أنه صيغة وسائط متعددة يمكن تضمينها في صفحة الويب.
- **عنصر `source`**: يُستخدم لتحديد نوع الملف والمصدر - ويمكن أن يشمل أنواعًا متعددة باستخدام عدة عناصر مصدر. عند القيام بذلك، يحدد المتصفح أفضل صيغة تناسب بيئة المستخدم الحالية.
- **MP3**: هو نوع من صيغ الملفات الرقمية المستخدمة لتخزين الموسيقى أو الصوت أو الصوتيات. هو نسخة مضغوطة من تسجيل صوتي تجعل حجم الملف أصغر، مما يسهل تخزينه ومشاركته. MP3 يحظى بأوسع دعم في المتصفحات ونوع MIME الخاص به هو `audio/mpeg`.
- **MP4**: MP4 هو نوع من صيغ الملفات الرقمية المستخدمة لتخزين الفيديو والصوت. يعمل كحاوية تحتوي على الفيديو (الصور) والصوت (الموسيقى أو الكلام) في ملف واحد. يمكن أن يكون لنوع MP4 نوع MIME إما `audio/mp4` أو `video/mp4`، حسب ما إذا كان ملف فيديو أو صوت فقط.
- الصيغ الأخرى تشمل `WMV` المرتبط بتطبيق Windows Media Player، و`OGG`، و`MKV`، و`AVI`، والمزيد.

## الترميزات (codecs)

- **التعريف**: الترميز، اختصار "المشفر/المفكك"، هو خوارزمية أو برنامج يمكنه تحويل الصوت والفيديو بين الصيغ التناظرية والرقمية. يمكن تحديد الترميزات كجزء من نوع MIME. الصيغة الأساسية لتعريف الترميز هي إضافة فاصلة منقوطة بعد نوع الوسائط، ثم `codecs=` متبوعة بالترميز.

## واجهة برمجة تطبيقات HTMLMediaElement

- **التعريف**: تُستخدم واجهة برمجة التطبيقات `HTMLMediaElement` للتحكم في سلوك عناصر الصوت والفيديو في صفحتك. تمتد من واجهة `HTMLElement` الأساسية، لذا لديك وصول إلى الخصائص والطرق الأساسية بالإضافة إلى أحداث مفيدة مثل `waiting`، و`ended`، و`canplay`، و`canplaythrough`، والمزيد. من أمثلة الطرق: `play()`، و`fastSeek()`، و`pause()`، و`canPlayType()` للتحقق مما إذا كان المتصفح قادرًا على تشغيل ملف صوتي.

## واجهة برمجة تطبيقات التقاط الوسائط وتدفقات البيانات

- **التعريف**: تُستخدم واجهة برمجة التطبيقات Media Capture and Streams، أو MediaStream API، لالتقاط الصوت والفيديو من جهازك. لاستخدام هذه الواجهة، تحتاج إلى إنشاء كائن `MediaStream`. يمكنك فعل ذلك باستخدام دالة البناء، لكن هذا لن يكون مرتبطًا بأجهزة المستخدم. بدلاً من ذلك، تحتوي خاصية `mediaDevices` في كائن `global` الخاص بالمستعرض على طريقة `getUserMedia()` التي يمكنك استخدامها.

```js
window.navigator.mediaDevices.getUserMedia({
  audio: true,
  video: {
    width: {
      min: 1280,
      ideal: 1920,
      max: 3840
    },
    height: {
      min: 720,
      ideal: 1080,
      max: 2160
    }
  }
});
```

## واجهة برمجة تطبيقات التقاط الشاشة

- **التعريف**: تُستخدم واجهة برمجة التطبيقات Screen Capture API لتسجيل شاشة المستخدم. تُتاح هذه الواجهة باستدعاء طريقة `getDisplayMedia()` لكائن `mediaDevices` واستهلاك تدفق الوسائط المعاد.

## واجهة برمجة تطبيقات تسجيل MediaStream

- **التعريف**: تعمل واجهة برمجة التطبيقات MediaStream Recording API بالتكامل مع واجهات MediaStreams، مما يسمح لك بتسجيل MediaStream (أو حتى عنصر `HTMLMediaElement` مباشرة).

## واجهة برمجة تطبيقات امتدادات مصدر الوسائط

- **الموضوع**: تتيح لك واجهة برمجة التطبيقات Media Source Extensions API تمرير بث كاميرا المستخدم مباشرة إلى عنصر فيديو باستخدام خاصية `srcObject`.

## واجهة برمجة تطبيقات الصوت على الويب

- **التعريف**: واجهة برمجة التطبيقات Web Audio API هي التي تشغل كل ما يُسمع على الويب. تشمل هذه الواجهة كائنات مهمة مثل `AudioBuffer` (يمثل مخزنًا مؤقتًا يحتوي على بيانات صوتية) أو `AudioContext`، المستخدم لتمثيل رسم بياني لمعالجة الصوت.
