# --description--

У Python класи та об’єкти працюють разом, щоб організувати й керувати даними. Ви створюєте клас, щоб визначити спільну поведінку, а потім створюєте об’єкти, які використовують цю поведінку.

Інакше кажучи, клас — це як креслення або шаблон, який ви використовуєте для створення об’єктів.

Давайте розглянемо, що таке класи і як їх використовувати для створення об’єктів.

Щоб створити клас, ви використовуєте ключове слово `class`, за яким іде назва класу та двокрапка. Потім всередині класу можна додати ініціалізатор, а також будь-які атрибути і методи.

Атрибути — це як змінні всередині класу, які використовуються для збереження даних. Методи — це функції, визначені всередині класу, і це дії, які можуть виконувати об’єкти, створені за допомогою класу.

Ось базовий синтаксис класу:

```python
class ClassName:
    def __init__(self, name, age):
        self.name = name
        self.age = age

    def sample_method(self):               
        print(self.name.upper())
```

* `class ClassName` складається з ключового слова `class` для створення класу, за яким іде назва класу, тут вона `ClassName`. У Python загальною практикою є використання конвенції **PascalCase** для іменування класів.
    
* `def __init__(self, name, age)` — це спеціальний метод, який автоматично викликається, коли створюється новий об’єкт. Він ініціалізує атрибути об’єктів, які будуть створені за допомогою класу.
    
    Крім того, перший параметр `__init__` завжди є референсом на конкретний об’єкт, який створюється або використовується. За конвенцією цей параметр називають `self`, але технічно можна використовувати будь-яке ім’я. `self` дозволяє отримати доступ до власних атрибутів і методів об’єкта.
    
* `self.name = name` і `self.age = age` — це атрибути, які матимуть об’єкти.
    
* `def sample_method(self):` — це метод, який може викликати кожен створений об’єкт.
    
* `print(self.name.upper())` — це те, що робитиме метод `sample_method`, у цьому випадку він виводить ім’я у верхньому регістрі.
    

Якщо це звучить складно, не хвилюйтеся. Давайте розглянемо схожий приклад класу `Dog` і як можна створювати об’єкти з нього:

```python
class Dog:
    def __init__(self, name, age):
        self.name = name
        self.age = age

    def bark(self):
        print(f"{self.name.upper()} says woof woof!")
```

За допомогою цього класу `Dog` ви можете створити об’єкт. Ось базовий синтаксис створення об’єктів із класу:

```python
object_1 = ClassName(attribute_1, attribute_2)
object_2 = ClassName(attribute_1, attribute_2)
```

Ви також можете викликати будь-які методи, визначені в класі, з кожного об’єкта:

```python
object_1.method_name()
object_2.method_name()
```

Тепер створимо двох собак, використовуючи клас `Dog` як шаблон:

```python
class Dog:
    def __init__(self, name, age):
        self.name = name
        self.age = age

    def bark(self):
        print(f"{self.name.upper()} says woof woof! I'm {self.age} years old!")

dog_1 = Dog("Jack", 3)
dog_2 = Dog("Thatcher", 5)

# Call the bark method
dog_1.bark()  # JACK says woof woof! I'm 3 years old!
dog_2.bark()  # THATCHER says woof woof! I'm 5 years old!
```

Як бачите, ми створюємо два об’єкти собак за допомогою класу `Dog`. При ініціалізації `dog_1` передаються рядок `Jack` і число `3`, які встановлюють атрибути `name` і `age` для цього екземпляра. А `dog_2` ініціалізується рядком `Thatcher` і числом `5` як відповідні атрибути `name` і `age`.

Потім, коли ви викликаєте метод `.bark()` у `dog_1` і `dog_2`, ви бачите, як відрізняються обидва вихідні дані, і використовуються унікальні атрибути `name` і `age`, які ви передали під час створення кожного об’єкта.

Підсумовуючи, різниця між класом і об’єктом полягає в тому, що клас — це шаблон або креслення, а об’єкт — це те, що створюється за допомогою цього шаблону.

Також клас визначає, які дані та поведінку має мати об’єкт, а об’єкт містить фактичні дані і використовує цю поведінку. Ви пишете клас один раз, а потім можете створювати багато об’єктів із різними даними.
