# --interactive--

預設連結樣式在提升網頁的可用性和可存取性方面扮演關鍵角色。

這些樣式，通常未造訪的聯結為藍色，已造訪的聯結為紫色，已成為使用者在瀏覽網站時所期望並依賴的標準。

預設聯結樣式的主要目的是提供清晰的視覺提示，幫助使用者區分網頁上作用中與非作用中的元素。

這種區別對於建立直覺且使用者友善的瀏覽體驗是基本的。

讓我們來看看聯結的基本預設樣式：

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<a href="/">Example link</a>
```

```css
a:link {
  color: blue;
  text-decoration: underline;
}

a:visited {
  color: purple;
}
```

:::

這些樣式具有多項重要功能。

首先，未造訪的聯結使用藍色，在大多數背景色和文字中都很突出，使聯結容易辨識。這種對比對使用者快速瀏覽頁面並找到導覽元素或重要資訊至關重要。

底線進一步強調該文字是可點擊的，提供額外的視覺提示。這對於可能有色盲或難以分辨顏色的使用者特別有幫助。

已造訪聯結的顏色變化（通常為紫色）有助於使用者追蹤他們曾經造訪過的位置。此特性對於瀏覽大型網站或進行研究非常寶貴，因為它能防止使用者不小心重複造訪相同的頁面。

請考慮此 HTML 範例：

:::interactive_editor

```html
<p>Learn more about <a href="https://www.example.com/cats">cats</a> and <a href="https://www.example.com/dogs">dogs</a>.</p>
```

:::

在沒有任何訂製 CSS 的情況下，大多數瀏覽器會將這些聯結呈現為藍色並帶有底線。點擊其中一個聯結後，其顏色會變為紫色，立即向使用者提供他們瀏覽歷史的回授。

雖然設計師常會修改這些預設樣式以符合網站的美學，但維護其背後的核心原則是至關重要的。

如果你選擇更改預設樣式，請確保聯結仍然能夠清楚區分於一般文字，已造訪與未造訪的聯結之間有明顯差異，且所選顏色與背景有足夠對比。

例如，你可能會使用像這樣的訂製樣式：

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<a href="/">Example link</a>
```

```css
a:link {
  color: blue;
  text-decoration: none;
  border-bottom: 1px solid blue;
}

a:visited {
  color: purple;
  border-bottom: 1px solid purple;
}
```

:::

這維護藍色和紫色的色彩結構綱目，同時將底線替換為底部邊框，以呈現更現代的外觀。

考慮連結的不同狀態也很重要。除了預設值和已造訪狀態外，連結通常還有暫留和作用中狀態：

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<a href="/">Example link</a>
```

```css
a:hover {
  color: red;
}

a:active {
  color: darkorange;
}
```

:::

這些狀態在使用者與聯結互動時提供即時回授，提升網站的整體可用性。

總結來說，雖然可以自訂聯結樣式，但應該維護預設樣式背後的原則。

它們在創造可用且可存取的網頁體驗中扮演關鍵角色，幫助使用者有效地瀏覽和互動內容。

在為你的網站設計聯結樣式時，始終優先考量清晰度和使用者體驗。
