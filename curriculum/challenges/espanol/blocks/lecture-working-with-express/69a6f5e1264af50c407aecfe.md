# --description--

En la lección anterior, creaste una carpeta para el proyecto e instalaste Express:

```bash
mkdir my-app
cd my-app
npm init -y
npm install express
```

Como recordatorio, estos comandos crean una nueva carpeta para el proyecto, `cd` dentro de la carpeta del proyecto, inicializan el proyecto y crean un archivo `package.json` básico, e instalan Express.

A continuación, abre un editor de código, crea un archivo llamado `index.js` y escribe tu primer servidor Express:

```javascript
const express = require("express");
const app = express();
const port = 3000;

app.get("/", (req, res) => {
  res.send("Hello World!");
});

app.listen(port, () => {
  console.log(`Example app listening on port ${port}`);
});
```

Puedes ejecutar tu código con el siguiente comando en la terminal:

```bash
node index.js
```

Abre tu navegador en [http://localhost:3000](http://localhost:3000) y verás el mensaje **"Hello World!"**

¡Así de rápido es empezar con Express! Ahora vamos a repasar el código paso a paso.

En la primera línea de código, cargas la biblioteca Express y la asignas a una variable llamada `express`:

```javascript
const express = require("express");
```

La variable `express` es una función fábrica. Cuando la llamas, crea y devuelve una nueva instancia de aplicación Express:

```javascript
const app = express();
```

`app` ahora tiene acceso a todos los métodos que ofrece Express, como `.get()`, `.post()`, `.listen()`, y así sucesivamente. Este es tu objeto principal del servidor.

La siguiente línea define la variable `port`. Un puerto es un punto final de red en el que tu aplicación escucha, y `3000` es un puerto comúnmente usado para desarrollo local:

```javascript
const port = 3000;
```

Ahora, veamos la siguiente parte aquí:

```javascript
app.get("/", (req, res) => {
  res.send("Hello World!");
});
```

Este bloque de código define una ruta y le indica al servidor cómo responder a las solicitudes. `app.get()` registra un manejador para solicitudes HTTP GET. Una solicitud GET es un método HTTP comúnmente usado para obtener datos.

El argumento `"/"` dentro de `app.get()` representa la ruta. En este caso, apunta a la URL raíz de la aplicación.

La parte `(req, res) => { ... }` es una función callback que se ejecuta cuando una solicitud coincide con esta ruta. El objeto `req` contiene información sobre la solicitud entrante, y el objeto `res` se usa para enviar una respuesta al cliente.

Finalmente, `res.send("Hello World!")` envía una respuesta al cliente. En este ejemplo, la respuesta es el texto plano `"Hello World!"`.

Ahora, veamos la última parte del código aquí:

```javascript
app.listen(port, () => {
  console.log(`Example app listening on port ${port}`);
});
```

El método `app.listen()` inicia la aplicación Express y le indica a Node que escuche las solicitudes HTTP entrantes en el puerto especificado, en este caso, `3000`.

Una vez que el servidor inicia, se ejecuta la función callback y el mensaje `Example app listening on port 3000` se muestra en la consola, confirmando que la app Express se inició correctamente y está lista para recibir solicitudes.
