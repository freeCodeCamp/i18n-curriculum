# --interactive--

تُستخدم الخاصية `aria-expanded` لأغراض إمكانية الوصول لتوضيح ما إذا كان عنصر التحكم موسعًا أو مطويًا. تُستخدم مع عناصر واجهة قابلة للطي مثل القوائم، والأكورديونات، وعناصر الكشف الأخرى التي تتحكم في مستوى ظهور المحتوى. تُعيّن الخاصية `aria-expanded` إلى `true` إذا كان عنصر التحكم موسعًا، أو إلى `false` إذا كان مطويًا.

تسمح المعلومات التي توفرها `aria-expanded` للأشخاص الذين يستخدمون قارئ الشاشة بفهم الحالة الحالية لعنصر التحكم (سواء كان موسعًا أو مطويًا).

تُطبّق الخاصية `aria-expanded` على العنصر التفاعلي الذي يبدل مستوى ظهور عنصر واجهة قابل للطي. على سبيل المثال، إذا كان زر يبدل قائمة قابلة للتوسيع، توضع الخاصية `aria-expanded` على الزر.

عندما تكون القائمة موسعة، يجب تعيين الخاصية `aria-expanded` إلى `true` كما في هذا المثال:

```html
<button aria-expanded="true">Menu</button>
```

عندما تكون القائمة مطوية، يجب تعيينها إلى `false` بدلاً من ذلك.

```html
<button aria-expanded="false">Menu</button>
```

يجب أن تكون الخاصية `aria-expanded` مضبوطة دائمًا على إما `true` أو `false` على العنصر المتحكم. على سبيل المثال، إذا كان زر يبدل مستوى ظهور قائمة، يجب أن تكون القيمة الافتراضية لـ `aria-expanded` مبنية على مستوى ظهور القائمة الافتراضي.

إذا كانت القائمة موسعة افتراضيًا، يجب تعيين `aria-expanded` في البداية إلى `true`. وإذا كانت القائمة مطوية افتراضيًا، يجب تعيين `aria-expanded` في البداية إلى `false`.

يجب تحديث قيمة `aria-expanded` ديناميكيًا باستخدام JavaScript أثناء تفاعل المستخدم مع العنصر.

بالإضافة إلى ذلك، يمكن استخدام الخاصيتين `aria-controls` و `aria-owns` مع `aria-expanded` لإنشاء ارتباط برمجي بين العنصر المتحكم والعنصر الذي يتحكم به.

لنبدأ بـ `aria-controls`. عند استخدامها مع `aria-expanded`، تُستخدم الخاصية `aria-controls` لتحديد العنصر الذي يتم التحكم به. على سبيل المثال، قد يوسع زر أو يطوي قائمة تعمل كقائمة. ستكون قيمة `aria-controls` هي `id` للعنصر المتحكم به (قائمة القائمة في هذا المثال).

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<button aria-expanded="false" aria-controls="menu1">Menu</button>
<ul id="menu1" hidden>
  <li>...</li>
  <li>...</li>
</ul>

<script src="index.js"></script>
```

```css
button {

  cursor: pointer;

button:hover {

  padding: 8px;

```

```js
const button = document.querySelector('button');
const menu = document.getElementById(button.getAttribute('aria-controls'));

button.addEventListener('click', () => {
  const expanded = button.getAttribute('aria-expanded') === 'true';
  button.setAttribute('aria-expanded', String(!expanded));
  menu.hidden = expanded;
});

```

لاحظ أن القائمة تتبع مباشرة الزر المتحكم. بالنسبة لعناصر التحكم القابلة للتوسيع مثل هذه، من الأفضل أن يتبع المحتوى الموسع العنصر الذي يتحكم به مباشرة في شجرة DOM. هذا يمنع مستخدمي قارئ الشاشة من البحث عن المحتوى الموسع، ويسهل على مستخدمي لوحة المفاتيح التنقل عبر أي عناصر تفاعلية في المحتوى الموسع.

إذا لم يكن من الممكن وضع المحتوى الموسع مباشرة بعد العنصر المتحكم، تسمح الخاصية `aria-owns` بنقله افتراضيًا بعد عنصر التحكم في شجرة إمكانية الوصول. هذا يسمح لتقنيات المساعدة مثل قارئات الشاشة بالتظاهر بأن المحتوى الموسع موضوع مباشرة بعد عنصر التحكم في DOM.

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<button aria-owns="menu1" aria-expanded="true">Menu</button>
<main>
  <!-- an entire page worth of content --->
</main>
<ul id="menu1">
  <li>...</li>
  <li>...</li>
</ul>

<script src="index.js"></script>
```

```css
button {

  color: white;
  padding: 8px 12px;
  border: none;
  border-radius: 4px;
  cursor: pointer;
  margin-bottom: 10px;
}

  background-color: #005ea2;
}

ul {
  list-style: none;
  padding: 0;
  margin: 8px 0;
  border: 1px solid #ccc;
  border-radius: 4px;
  width: 150px;
  background: #fff;
  position: absolute;
  z-index: 10;
}

li {
  padding: 8px;
  cursor: pointer;
}

```

```js
const button = document.querySelector('button');
const menu = document.getElementById(button.getAttribute('aria-owns'));

button.addEventListener('click', () => {
  const expanded = button.getAttribute('aria-expanded') === 'true';
  button.setAttribute('aria-expanded', String(!expanded));
  menu.hidden = expanded;
});

// Close the menu if user clicks outside
document.addEventListener('click', (e) => {
  if (!button.contains(e.target) && !menu.contains(e.target)) {
    button.setAttribute('aria-expanded', 'false');
    menu.hidden = true;
  }
});

```

هناك سلبيات لاستخدام الخاصية `aria-owns`. فهي تخلق إطنابًا غير ضروري لمستخدمي قارئ الشاشة لأن معظم قارئات الشاشة تقرأ تلقائيًا كامل محتويات العنصر الموسع عند توسيعه لأول مرة. كما أنها لا تغير ترتيب التبويب، مما قد يجبر مستخدمي لوحة المفاتيح على التنقل عبر جميع عناصر التحكم التفاعلية الأخرى في الصفحة قبل الوصول إلى المحتوى الموسع، ما لم تدير ترتيب التبويب باستخدام JavaScript.

من الأفضل أن يوضع المحتوى القابل للتوسيع بعد عنصر التحكم، ويجب استخدام الخاصية `aria-owns` فقط في أسوأ الحالات عندما لا يكون ذلك ممكنًا. وإذا اضطررت لاستخدامها، يجب أن تختبر بدقة مع مجموعة واسعة من قارئات الشاشة والمتصفحات لضمان أن تنفيذك متاح للجميع.

وتذكير فقط، عند استخدامك إما `aria-controls` أو `aria-owns`، يجب الاستمرار في تحديث قيمة `aria-expanded` أثناء توسيع وطوي عنصر التحكم.

تشير الخاصية `aria-expanded` إلى ما إذا كان عنصر التحكم موسعًا أو مطويًا. هذه المعلومة ضرورية لمستخدمي قارئ الشاشة، إذ تساعدهم على فهم الحالة الحالية للعناصر القابلة للطي مثل القوائم، والأكورديونات، وعناصر الكشف المشابهة.

باستخدام `aria-expanded` بشكل صحيح، يمكنك إنشاء تجربة مستخدم بديهية للجميع.
