# --description--

في درس سابق، تعلمت كيفية العمل مع الأسماء المستعارة للأنواع ورأيت أمثلة على كيفية تسمية أنواع الكائنات.

كما تتذكر، إحدى الطرق لتسمية نوع كائن هي استخدام الكلمة المفتاحية `type` بهذا الشكل:

```ts
type ID = number | string;

type User = {
  id: ID;
  name: string;
};
```

طريقة أخرى لتسمية نوع كائن هي استخدام إعلان `interface` بهذا الشكل:

```ts
type ID = number | string;

interface User {
  id: ID;
  name: string;
}
```

`type` و `interface` متشابهان لكن الفرق الرئيسي هو أن الواجهات تستخدم الكلمة المفتاحية `extends` للبناء على واجهات أو أنواع أخرى، بينما تستخدم ألقاب `type` أنواع التقاطع (`&`) لدمج الأنواع.

لننظر إلى بعض الأمثلة:

```ts
interface User {
  id: number | string;
  name: string;
}

interface Admin extends User {
  role: string;
}

const adminUser: Admin = {
  id: 101,
  name: "Alice",
  role: "superadmin"
};
```

في هذا المثال، يبدأ `User` بـ `id` و`name`. واجهة `Admin` تمتد من `User`، مما يعني أنها تشمل `id` و`name` وتضيف خاصية جديدة تسمى `role`.

إذا أردت توسيع `type`، ستستخدم نوع التقاطع (`&`) بهذا الشكل:

```ts
type User = {
  id: number | string;
  name: string;
};

type Admin = User & {
  role: string;
};

const adminUser: Admin = {
  id: 101,
  name: "Alice",
  role: "superadmin"
};
```

`Admin` يجمع نوع `User` مع نوع كائن جديد يتضمن `role`. النتيجة هي نفس شكل المثال الخاص بالواجهة، وهو كائن يحتوي على `id` و`name` و`role`.

كلا من `type` و`interface` يمكنهما وصف شكل الكائن. إذا لم تكن متأكدًا أيهما تستخدم، تذكر أن `interface` مصممة خصيصًا لأشكال الكائنات، بينما `type` أكثر مرونة ويمكنه أيضًا تمثيل أنواع الاتحاد، والأنواع الأولية، وتركيبات الأنواع المعقدة.
