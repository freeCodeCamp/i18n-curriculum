# --description--

I cookie, noti anche come cookie web o cookie del browser, sono piccoli pezzi di dati che un server invia al browser web di un utente. Questi cookie vengono memorizzati sul dispositivo dell'utente e inviati di nuovo al server con le richieste successive.

I cookie sono essenziali per aiutare le applicazioni web a mantenere lo stato e ricordare le informazioni dell'utente, cosa particolarmente importante dato che HTTP è un protocollo senza stato.

I cookie possono memorizzare una varietà di informazioni come preferenze dell'utente, dati di sessione o informazioni di tracciamento.

I cookie sono sempre memorizzati come coppie nome-valore. Questo significa che ogni cookie ha un nome (chiave) e un valore associato.

Ad esempio, un cookie potrebbe memorizzare l'id di sessione di un utente così: `sessionId=abc123`. La chiave è `sessionId` e il valore è `abc123`. Ogni volta che il browser comunica con il server, invia questi cookie sotto forma di coppie nome-valore.

Quando un utente visita un sito web, il server può inviare uno o più cookie al browser dell'utente includendo un'intestazione Set-Cookie nella risposta HTTP. Un'intestazione è una coppia chiave-valore che fornisce informazioni aggiuntive sulla richiesta o risposta HTTP. Imparerai di più sulle richieste e risposte HTTP nelle lezioni future.

Una volta impostati i cookie, il browser li memorizza e li include automaticamente nell'intestazione Cookie con ogni richiesta successiva allo stesso dominio. Questo permette al server di consultare i cookie memorizzati e usarli per cose come mantenere le sessioni utente o tracciare le preferenze.

Ecco un esempio di come un cookie viene impostato in una risposta HTTP:

```http
Set-Cookie: sessionId=abc123; Expires=Wed, 21 Oct 2021 07:28:00 GMT; Secure; HttpOnly
```

Il browser memorizzerà il cookie e nelle richieste future allo stesso server includerà il cookie nell'intestazione `Cookie`:

```http
Cookie: sessionId=abc123
```

Il server può quindi leggere il cookie e usare l'id di sessione memorizzato per recuperare informazioni sull'utente, come se è autenticato.

Ecco una panoramica dei diversi tipi di cookie.

- I cookie di sessione durano solo per la durata della sessione dell'utente sul sito web. Quando l'utente chiude il browser o la scheda, il cookie di sessione viene eliminato. Questi cookie sono tipicamente usati per attività come mantenere l'utente connesso durante la visita.

- I cookie persistenti hanno una data di scadenza e rimangono memorizzati sul dispositivo dell'utente fino a quella data. I cookie persistenti sono spesso usati per ricordare le preferenze dell'utente o i dettagli di accesso tra le sessioni.

- I cookie sicuri vengono inviati solo tramite HTTPS, garantendo che non possano essere intercettati da un attaccante durante il trasferimento.

- I cookie HttpOnly non possono essere consultati o modificati da JavaScript in esecuzione nel browser, rendendoli più sicuri contro attacchi di cross-site scripting (XSS). Gli attacchi di cross-site scripting (XSS) avvengono quando un attaccante inietta script dannosi in una pagina web visualizzata da altri utenti. Questi script possono poi eseguirsi nel contesto del browser della vittima, potenzialmente rubando cookie, dati di sessione o compiendo altre azioni dannose senza la conoscenza o il consenso dell'utente. Contrassegnando i cookie come HttpOnly, sono protetti dall'essere consultati tramite JavaScript, riducendo il rischio di tali attacchi.

Puoi creare cookie tramite risposte del server usando l'intestazione `Set-Cookie` o tramite JavaScript usando `document.cookie`.

Ecco un esempio di impostazione di un cookie usando JavaScript:

```js
document.cookie = "username=JohnDoe; expires=Fri, 31 Dec 2021 23:59:59 GMT; path=/";
```

Questo comando imposta un cookie chiamato `username` con il valore `"JohnDoe"` che scadrà alla fine del 2021. Puoi aggiornare un cookie esistente semplicemente impostandolo di nuovo con un nuovo valore.

Per eliminare un cookie, imposta la sua data di scadenza nel passato:

```js
document.cookie = "username=; expires=Thu, 01 Jan 1970 00:00:00 GMT; path=/";
```

Questo rimuoverà il cookie `username` dal browser.
