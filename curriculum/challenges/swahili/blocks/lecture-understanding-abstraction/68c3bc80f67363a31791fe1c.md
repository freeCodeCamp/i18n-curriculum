# --description--

Sasa tumeshazungumzia encapsulation, urithi, na polymorphism, hebu tujadili dhana kuu inayofuata katika programu za vitu – abstraction.

Abstraction ni mchakato wa kuficha maelezo magumu ya utekelezaji na kuonyesha tu vipengele muhimu vya kitu au mfumo. Fikiria kama makini na kile kitu kinachofanya badala ya jinsi kinavyokifanya.

Abstraction haijafungiwa kwa Python tu. Ni dhana ya programu inayoweza kutekelezwa katika lugha nyingi zinazounga mkono programu za vitu.

Ili kuonyesha abstraction, fikiria unaendesha gari. Unapokuwa kwenye kiti cha dereva, unashirikiana na sehemu muhimu kama usukani, shifter, na pedal za accelerator na breki. Huhitaji kujua maelezo magumu ya jinsi injini inavyofanya kazi, jinsi transmission inavyobadilisha gia, au fizikia nyuma ya mfumo wa breki, kwa kuwa yote hayo ni maelezo magumu ya utekelezaji.

Hiyo ni abstraction ikifanya kazi! Inakupa kiolesura kilichorahisishwa cha kushirikiana na mfumo mgumu.

Katika mfano wa gari tena, kiolesura kilichorahisishwa ni usukani, breki, na accelerator, wakati mfumo mgumu ni gari lenyewe.

Kuhusu jinsi Python inavyotekeleza abstraction, hufanya hivyo kupitia moduli ya `abc`.

Moduli hii hutoa darasa la `ABC` (linalomaanisha “abstract base class”) na dekoreta ya `@abstractmethod`.

`ABC` ni darasa linalokusudiwa kurithiwa, lakini huwezi kuunda vitu vya moja kwa moja kutoka kwake. Ndiyo inayobainisha kiolesura cha pamoja cha njia na vigezo ambavyo madarasa yake ya chini lazima yatekeleze.

Kwa upande mwingine, njia ya abstract ni njia inayotangazwa katika Darasa la Msingi la Abstract (ABC) kwa kutumia dekoreta ya `@abstractmethod`. Inaweza isiwe na utekelezaji wowote au uwe na utekelezaji wa msingi wa chaguo. Hata hivyo, darasa lolote la chini lazima lighairi ili lichukuliwe kuwa halisi na liweze kuanzishwa, hata kama utekelezaji wa msingi umetolewa.

Hapa kuna sintaksia ya msingi ya darasa la abstract katika Python:

```py
from abc import ABC, abstractmethod

# Define an abstract base class
class AbstractClass(ABC):
    @abstractmethod
    def abstract_method(self):
        pass

# Concrete subclass that implements the abstract method
class ConcreteClassOne(AbstractClass):
    def abstract_method(self):
        print('Implementation in ConcreteClassOne')

# Another concrete subclass
class ConcreteClassTwo(AbstractClass):
    def abstract_method(self):
        print('Implementation in ConcreteClassTwo')
```

Hapa kuna mfano wa msingi:

```py
from abc import ABC, abstractmethod

class Animal(ABC): # Inherits from abstract base class
   @abstractmethod # Abstract method decorator
   def make_sound(self):  # The method subclasses must override
       pass

# Concrete class that will override the abstract method
class Dog(Animal):
   def make_sound(self):
       print('Woof!')

# Another concrete class that will override the abstract method
class Cat(Animal):
   def make_sound(self):
       print('Meow!')

# Another concrete class that will override the abstract method
class Monkey(Animal):
   def make_sound(self):
       print('Ooh ooh aah aah!')

# Create instances of each concrete class
animals = [Dog(), Cat(), Monkey()]

# Loop through the instances to call the make_sound method
for animal in animals:
   animal.make_sound()

# Output:
# Woof!
# Meow!
# Ooh ooh aah aah!
```

Katika mfano huu:

- Tunaleta darasa la `ABC` na `abstractmethod` kutoka moduli ya `abc`.
- Kisha tunaunda darasa la `Animal` linalorithi kutoka `ABC`, na kuunda njia ya abstract `make_sound` ndani yake ambayo kila darasa la chini la `Animal` lazima lighairi.
- Tunaunda madarasa halisi `Dog`, `Cat`, na `Monkey`, ambayo lazima yaghairi njia ya abstract `make_sound`.
- Tunatengeneza vitu vya madarasa halisi na kuita njia yao ya `make_sound` kuonyesha jinsi kila moja inavyotekeleza njia ya abstract ya `make_sound` kwa njia yake mwenyewe.

Kumbuka huwezi kuunda mfano wa darasa la Animal. Hapa ni kile kinachotokea ukijaribu kufanya hivyo:

```py
dog = Animal() 
# TypeError: Can't instantiate abstract class Animal 
# without an implementation for abstract method 'make_sound'
```

Suala hilo hilo linahusu madarasa ya chini ambayo hayatoi utekelezaji wa njia ya abstract. Hata ukitangaza darasa la chini la Animal, huwezi kuanzisha hadi lighairi make_sound. Hapa kuna mfano unaoonyesha hilo:

```py
class Bird(Animal):
    pass

bird = Bird()
# TypeError: Can't instantiate abstract class Bird 
# without an implementation for abstract method 'make_sound'
```

Hapa kuna mfano mwingine, wakati huu na sifa ya mfano ambayo unaweza kuipatia vitu vya njia halisi:

```py
from abc import ABC, abstractmethod

# The blueprint for any toy that can speak
class TalkingToy(ABC):
   def __init__(self, name):
       self.name = name
   @abstractmethod
   def speak(self):
       pass

class RobotToy(TalkingToy):
   def speak(self):
       print(f'{self.name} says beep boop! I am a robot!')

class TeddyBearToy(TalkingToy):
   def speak(self):
       print(f"{self.name} says hug me! I'm cuddly!")

class DinosaurToy(TalkingToy):
   def speak(self):
       print(f'{self.name} says ROOOOAR!')

# Create toys
rusty = RobotToy('Rusty')
fluffy = TeddyBearToy('Fluffy')
rex = DinosaurToy('Rex')

toys = [rusty, fluffy, rex]
for toy in toys:
   toy.speak()

# Output:
# Rusty says beep boop! I am a robot!
# Fluffy says hug me! I'm cuddly!
# Rex says ROOOOAR!
```

Katika mfano huu:

- Tuna darasa la msingi la abstract `TalkingToy` linalobainisha ramani kwa ajili ya toy yoyote inayoweza kuzungumza.
- Madarasa ya chini `RobotToy`, `TeddyBearToy`, na `DinosaurToy` yatekeleza njia ya `speak` kwa njia zao wenyewe.
- Tunapotengeneza vitu vya madarasa haya ya chini na kuita njia ya `speak`, kila toy huzungumza kwa njia yake ya kipekee.

Kwa kumalizia, abstraction katika Python hurahisisha mifumo ngumu kwa kuongeza matumizi tena.

Umeona jinsi unavyoweza kutumia tena njia moja kutoka darasa la abstract katika madarasa mengi ya chini huku ukilazimisha kila darasa la chini kutoa tabia yake maalum.

Njia hii inahakikisha msimbo wako uko na mpangilio, una ufanisi, na ni rahisi kudumishwa, hasa programu yako inapokua.
