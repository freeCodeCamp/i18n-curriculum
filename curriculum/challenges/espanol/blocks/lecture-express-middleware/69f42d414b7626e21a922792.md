# --description--

El middleware a nivel de aplicación es middleware que está vinculado a una instancia de una app Express. Se usa típicamente para manejar solicitudes que requieren procesamiento a través de múltiples rutas. Puede aplicarse de forma global o a rutas específicas.

Este middleware se registra usando `app.use()` o métodos específicos de ruta como `app.get()`, `app.post()`, etc. Se ejecuta para cada solicitud entrante a menos que esté restringido a rutas específicas.

```js
const express = require('express')
const app = express()

// Application-level middleware that logs all incoming requests
app.use((req, res, next) => {
  console.log('Request received at:', new Date())
  next(); // Passes control to the next middleware
})

app.get('/', (req, res) => {
  res.send('Hello World!')
})

app.listen(3000, () => {
  console.log('Server running on http://localhost:3000')
})
```

* `app.use()`: Este es middleware a nivel de aplicación. Registra cada solicitud hecha al servidor, sin importar la ruta.
* El middleware se ejecuta para cada solicitud entrante, proporcionando al servidor detalles como la marca de tiempo actual.

El middleware a nivel de aplicación es útil para tareas que deben aplicarse en muchas o todas las rutas de la app. Esto puede incluir registro, autenticación y manejo de errores.
