# --description--

ウェブサイトやアプリはデータを収集して処理したり、ユーザーが必要とする検索結果のようなものを生成したりする必要があります。ですから、テキストと同様に、フォームもウェブの基本的な要素です。

フォームを使うことで、ユーザーはサイトに登録したりログインしたり、メッセージを送ったり、アイテムを検索したりできます。例えば、ウェブで最も人気のあるフォームであるGoogleの検索入力にクエリを入力すると、Googleの検索結果ページにアクセスできます。

フォームを構成するもの、例えば入力フィールド、`action`属性、フォームのメソッド、そして最も重要なフォームを提出したときに何が起こるのかを見ていきましょう。

フォームはウェブページに`form`要素で埋め込まれますが、入力を表示させるには必要に応じて`input`要素も必要です。これらの入力要素にはテキスト、メール、パスワード、選択、ファイル、隠し入力などがあります。

これらの入力フィールドを使うには、`form`要素で囲む必要があります。各入力には`name`属性が必要で、サーバーがどのデータがどのフィールドに対応するかを認識します。

例を示します：

```html
<form>
   <input type="text" name="username" />
   <input type="email" name="email" />
   <input type="file" name="profilePicture" />
   <input type="radio" name="gender" />
   <input type="checkbox" name="subscribe" />
   <input type="hidden" name="userId" />
</form>
```

入力フィールドはユーザーが正しい情報を入力しているかを確認するためにバリデーションも必要です。

このバリデーションはクライアント側とサーバー側の両方で行うことができます。HTMLでは`required`属性と`pattern`属性を使って基本的なクライアント側バリデーションが可能です。しかし、HTMLのバリデーションは主にユーザー体験と即時フィードバックのためなので、追加のセキュリティとしてサーバー側でもバリデーションを行うことをお勧めします。

`required`属性は入力フィールドの必須入力を指定し、`pattern`属性は正規表現を入力でき、ユーザーに必要な形式で情報を入力させることができます。

ユーザー名フィールドの例を示します：

```html
<input type="text" name="username" required pattern="[A-Za-z0-9]{3,}" />
```

入力フィールドがバリデーションを通過した後、提出が行われます。では、フォームが提出されると何が起こるのでしょうか？

まず、提出が行われるには、フォームにブラウザが入力フィールドの値を送信する先を指定する`action`属性と、フォームデータの送信方法を指定する`method`属性が必要です。`method`属性については後ほど説明します。

action属性の例は以下のようになります：

```html
<form action="/register">
	<!-- input fields -->
</form>
```

`method`属性は`GET`、`POST`、`PUT`、`PATCH`、または`DELETE`のいずれかになります：

* `GET`はサーバーから何かを取得します。例えば検索結果です。
    
* `POST`はサーバーに何かを送信します。例えば入力フィールドを通じて入力されたログインや登録情報です。
    
* `PUT`はサーバー上の何かを変更します。
    
* `PATCH`はサーバー上の何かを部分的に変更します。
    
* `DELETE`はサーバー上の何かを削除します。
    
HTMLは`PUT`、`PATCH`、`DELETE`のメソッドを直接サポートしていませんが、回避策があります。バニラJavaScriptを使う場合はFetch APIでメソッドを指定でき、Node.jsとExpressでは`method-override`パッケージを使えます。

`GET`リクエストを行うフォームの例は以下のようになります：

```html
<form action="/search" method="GET">
   <input type="text" name="q">
   <button type="submit">Search</button>
</form>
```

`POST`リクエストを行うフォームの例は以下のようになります：

```html
<form action="/submit" method="POST">
   <input type="text" name="username" required pattern="[A-Za-z0-9]{3,}" />
   <input type="password" name="password" />
   <button type="submit">Submit</button>
</form>
```

`GET`または`POST`リクエストの後、サーバーはサーバー側バリデーションをチェックし、入力値を処理するか拒否します。

バリデーションが成功した場合、サーバーは要求された情報を送信し、失敗した場合はエラーをスローします。

バリデーションが成功した場合、認証や検索の場合はリダイレクト、または操作を確認する成功メッセージがサーバーから返されます。

フォームが提出されたときに起こることをまとめると：

* ユーザーがフォームのフィールドに入力します。
    
* ブラウザがクライアント側バリデーション（あれば）をチェックし、成功すれば次に進みます。
    
* ブラウザがaction属性とmethod属性に基づいてフォームデータをサーバーに送信します。
    
* サーバーがデータを受け取り処理します（データベースに保存、ユーザー登録、ログイン、検索クエリの送信など）。
    
* サーバー側バリデーションがある場合は、処理前にデータを再チェックします。
    
* サーバーはバリデーションの状態（成功または失敗）に応じて成功メッセージ、エラー、またはユーザーのリダイレクトを返します。
    
* ユーザーはサーバーのレスポンスに基づいたフィードバックを受け取ります。
