# --interactive--

المعدِّلات، التي يُشار إليها غالبًا بـ"الأعلام"، تُعدِّل سلوك التعبير النمطي. دعنا نستذكر مثالنا من درس سابق:

:::محرر_تفاعلي

```js
const regex = /freeCodeCamp/;
console.log(regex.test("freeCodeCamp")); // true
console.log(regex.test("freeCodeCamp is great")); // true
console.log(regex.test("I love freeCodeCamp")); // true
console.log(regex.test("freecodecamp")); // false
console.log(regex.test("FREECODECAMP")); // false
console.log(regex.test("free")); // false
console.log(regex.test("code")); // false
console.log(regex.test("camp")); // false
```

:::

إذا تذكرت، فإن سلاسل `freeCodeCamp` المكتوبة بأحرف صغيرة فقط أو كبيرة فقط لم تطابق النمط. هذا لأن التعبيرات النمطية تكون حساسة لحالة الأحرف افتراضيًا.

لكن ماذا لو أخبرنا التعبير النمطي أن يتجاهل حالة الأحرف؟ هناك معدِّل لذلك. العلم `i` يجعل التعبير النمطي يتجاهل حالة الأحرف. كيف نستخدمه؟ توضع الأعلام بعد الشرطة المائلة النهائية في التعبير النمطي:

```js
const regex = /freeCodeCamp/i;
```

لاحظ التغيير في التعبير النمطي في السطر الأول. الآن يمكننا التحقق من كيفية تأثير هذا التغيير:

:::محرر_تفاعلي

```js
const regex = /freeCodeCamp/i;

console.log(regex.test("freeCodeCamp")); // true
console.log(regex.test("freeCodeCamp is great")); // true
console.log(regex.test("I love freeCodeCamp")); // true
console.log(regex.test("freecodecamp")); // true
console.log(regex.test("FREECODECAMP")); // true
console.log(regex.test("free")); // false
console.log(regex.test("code")); // false
console.log(regex.test("camp")); // false
```

:::

بما أن التعبير النمطي أصبح الآن غير حساس لحالة الأحرف، فقد "نجحت" سلاسل الأحرف الصغيرة والكبيرة فقط في الاختبار. يمكن أن يعمل هذا أيضًا مع سلسلة تحتوي على مزيج عشوائي من الأحرف الكبيرة والصغيرة:

:::محرر_تفاعلي

```js
const regex = /freeCodeCamp/i;

console.log(regex.test("dO yOu LoVe fReEcOdEcAmP?")); // true
```

:::

هناك العديد من الأعلام الأخرى التي يمكنك استخدامها. العلم `g`، أو المعدِّل العام، يسمح لتعبيرك النمطي بمطابقة النمط أكثر من مرة.

لنرَ كيف يؤثر ذلك على شفرتنا. ستلاحظ أننا احتفظنا بعلم `i`: يمكن لتعبير نمطي استخدام عدة أعلام (بقدر ما تحتاج) لتحقيق السلوك الذي تريده:

```js
const regex = /freeCodeCamp/gi;
```

انتظر لحظة... ما هذا؟ يبدو أن المعدِّل العام يجعل بعض سلاسلنا التي يجب أن تنجح تفشل بدلاً من ذلك:

:::محرر_تفاعلي

```js
const regex = /freeCodeCamp/gi;

console.log(regex.test("freeCodeCamp")); // true
console.log(regex.test("freeCodeCamp is great")); // false
console.log(regex.test("I love freeCodeCamp")); // true
console.log(regex.test("freecodecamp")); // false
console.log(regex.test("FREECODECAMP")); // true
console.log(regex.test("free")); // false
console.log(regex.test("code")); // false
console.log(regex.test("camp")); // false
```

:::

لماذا؟ لأن المعدِّل العام يجعل التعبير النمطي ذو حالة. هذا يعني أنه يتتبع مكان تطابق النمط سابقًا. فعندما يطابق أول سلسلة `freeCodeCamp`، يتذكر أنه وجد تطابقًا يبدأ عند الفهرس `0`.

ثم نختبره على `freeCodeCamp is great`، لكنه لا يبدأ عند الفهرس `0`. التعبير النمطي "يعلم" أنه وجد تطابقًا عند الفهرس `0` بالفعل، لذا حتى لو كانت هذه سلسلة مختلفة، يبدأ من نهاية التطابق السابق.

طول `freeCodeCamp` هو `12` حرفًا، لذا فإن التطابق عند `0` ينتهي عند الفهرس `11`. سيستأنف التطابق عند الفهرس `12`. وبما أن `is great` لا تطابق `freeCodeCamp`، فإن النتيجة تكون `false`.

ثم، لأنه فشل في إيجاد تطابق، "يفقد" حالته ويبدأ التطابق التالي من جديد عند `0`.

إذا بدلنا سجلاتنا بحيث تتبع سلسلة تحتوي على تطابق عند `0` مباشرة سلسلة أخرى تحتوي على تطابق بعد الفهرس `11`:

:::محرر_تفاعلي

```js
const regex = /freeCodeCamp/gi;

console.log(regex.test("freeCodeCamp")); // true
console.log(regex.test("I loooooooove freeCodeCamp")); // true
```

:::

عندما يكون التعبير النمطي عامًا، يحصل على خاصية جديدة تسمى `lastIndex`. لنأخذ شفرتنا السابقة ونرى كيف تعمل هذه الخاصية:

:::محرر_تفاعلي

```js
const regex = /freeCodeCamp/gi;

console.log(regex.lastIndex); // 0
console.log(regex.test("freeCodeCamp")); // true
console.log(regex.lastIndex); // 12
console.log(regex.test("freeCodeCamp is great")); // false
console.log(regex.lastIndex); // 0
console.log(regex.test("I love freeCodeCamp")); // true
console.log(regex.lastIndex); // 19
console.log(regex.test("freecodecamp")); // false
console.log(regex.lastIndex); // 0
console.log(regex.test("FREECODECAMP")); // true
console.log(regex.lastIndex); // 12
console.log(regex.test("free")); // false
console.log(regex.lastIndex); // 0
console.log(regex.test("code")); // false
console.log(regex.lastIndex); // 0
console.log(regex.test("camp")); // false
```

:::

من خلال هذا المثال، ترى كيف تتغير حالة التعبير النمطي مع كل استدعاء اختبار باستخدام `lastIndex` لتتبع التطابقات السابقة.

العلم العام مفيد عندما تحتاج إلى الحصول على تطابقات متعددة من سلسلة واحدة. لكن إذا كنت تختبر عدة سلاسل بنفس التعبير النمطي، فمن الأفضل عدم استخدام العلم `g`.

قبل أن تتعلم عن العلم التالي، يجب أن تتعرف على الأوسمة. وسم المؤشر (`^`) في بداية التعبير النمطي يعني "مطابقة بداية السلسلة":

```js
const start = /^freecodecamp/i;
```

وسم الدولار (`$`) في نهاية التعبير النمطي يعني "مطابقة نهاية السلسلة":

```js
const end = /freecodecamp$/i;
```

خذ لحظة لمقارنة النتائج على اليمين:

:::محرر_تفاعلي

```js
const start = /^freecodecamp/i;
const end = /freecodecamp$/i;
console.log(start.test("freecodecamp")); // true
console.log(end.test("freecodecamp")); // true
console.log(start.test("freecodecamp is great")); // true
console.log(end.test("freecodecamp is great")); // false
console.log(start.test("i love freecodecamp")); // false
console.log(end.test("i love freecodecamp")); // true
console.log(start.test("have met freecodecamp's founder")); // false
console.log(end.test("have met freecodecamp's founder")); // false
```

:::

هل ترى كيف يطابق وسم البداية فقط بداية السلسلة، ووسم النهاية فقط نهاية السلسلة؟ لكن ماذا عن المطابقة عبر عدة أسطر؟ لنلق نظرة على ذلك:

:::محرر_تفاعلي

```js
const start = /^freecodecamp/i;
const end = /freecodecamp$/i;
const string = `I really love
freecodecamp
it's my favorite`;
console.log(start.test(string)); // false
console.log(end.test(string)); // false
```

:::

رغم وجود `freecodecamp` في سطر منفصل، فإنه يفشل في كلا الاختبارين. هذا لأن الأوسمة تبحث افتراضيًا عن بداية ونهاية السلسلة بأكملها.

لكن يمكنك جعل التعبير النمطي يتعامل مع عدة أسطر باستخدام العلم `m`، أو المعدِّل متعدد الأسطر. دعنا نضيفه إلى تعبيراتنا النمطية لنرى النتيجة:

:::محرر_تفاعلي

```js
const start = /^freecodecamp/im;
const end = /freecodecamp$/im;
const string = `I really love
freecodecamp
it's my favorite`;
console.log(start.test(string)); // true
console.log(end.test(string)); // true
```

:::

الآن كلاهما يطابق! لأن `freecodecamp` موجودة بالكامل في سطر منفصل، يطابق وسم البداية بداية ذلك السطر، ووسم النهاية نهاية ذلك السطر.

أخيرًا، لديك العلم `d`، أو معدِّل الفهارس. تذكر أن العلم `i` خاص بعدم حساسية حالة الأحرف، لذا احتاج معدِّل الفهارس إلى علم مختلف.

العلم `d` يوسع المعلومات التي تحصل عليها في كائن التطابق. دعنا نضيفه إلى تعبيرنا النمطي:

```js
const regex = /freecodecamp/di;
const string = "we love freecodecamp isn't freecodecamp great?";
console.log(string.match(regex));
```

والنتيجة هي:

```js
// [
//   'freecodecamp',
//   index: 8,
//   input: "we love freecodecamp isn't freecodecamp great?",
//   groups: undefined,
//   indices: [
//     0: [8, 20],
//     groups: undefined
//   ]
// ]
```

يحصل كائن التطابق على خاصية جديدة `indices`! هذه الخاصية مصفوفة من رقمين، الأول هو الفهرس في السلسلة الأصلية حيث يبدأ التطابق، والثاني هو الفهرس بعد نهاية التطابق. تحتوي هذه المصفوفة أيضًا على خاصية إضافية `groups`، وهي للمجموعات المسماة في الالتقاط.

هناك بعض الأعلام الأخرى التي يجب أن تعرف أنها متاحة، لكنها أقل شيوعًا في الشفرة النموذجية.

الأول هو معدِّل اليونيكود، أو العلم `u`. هذا يوسع وظيفة التعبير النمطي للسماح له بمطابقة أحرف يونيكود الخاصة.

ستتعلم المزيد عن فئات الأحرف في درس لاحق، لكن العلم `u` يمنحك وصولًا إلى فئات خاصة مثل `Extended_Pictographic` لمطابقة معظم الرموز التعبيرية:

:::محرر_تفاعلي

```js
const regex = /🍎/u;

const str = "I have an apple 🍎";
console.log(regex.test(str)); // true
```

:::

هناك أيضًا العلم `v`، الذي يوسع أكثر وظيفة مطابقة اليونيكود.

الثاني هو المعدِّل اللاصق، أو العلم `y`. يتصرف المعدِّل اللاصق بشكل مشابه جدًا للمعدِّل العام، لكن مع بعض الاستثناءات.

الأكبر هو أن التعبير النمطي العام يبدأ من `lastIndex` ويبحث في بقية السلسلة كلها عن تطابق آخر، لكن التعبير اللاصق يرجع `null` ويعيد تعيين `lastIndex` إلى `0` إذا لم يكن هناك تطابق فوري عند `lastIndex` السابق.

وأخيرًا، هناك معدل السطر الواحد، أو العلم `s`. تذكر أن معدل الأسطر المتعددة يسمح لأوسمة البداية والنهاية بأن تطابق بداية ونهاية السطر، بدلًا من السلسلة النصية بأكملها.

يتيح معدل السطر الواحد لرمز البدل، الممثل بنقطة (`.`) في التعبير النمطي، أن يطابق فواصل الأسطر، مما يعامل السلسلة النصية كسطر واحد من النص.

هناك العديد من هذه المعدِّلات، لكن العلمين `i` و `g` هما الأكثر استخدامًا والأهم لتتذكرهما.
