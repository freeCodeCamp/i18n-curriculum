# --interactive--

カスケードアルゴリズムは、複数のスタイルが同じ要素を対象にしている場合に、どのCSSルールを適用するかをブラウザが決定するプロセスです。これは、一連の明確なルールに基づいて最も適切なスタイルが使われるようにします。

このプロセスは、まず**関連性**から始まります。ブラウザは最初に、実際に対象の要素に適用されるCSSルールだけを絞り込みます。これには、セレクターの一致や適用されている可能性のあるメディアクエリの考慮が含まれます。

メディアクエリとは、デバイスやビューポートの幅、高さ、向きなどの特性に基づいてスタイルを適用するCSSの手法です。

次に、アルゴリズムは**起源と重要度**を考慮します。CSSは異なるソースから来ることがあります：ブラウザのデフォルトスタイル（ユーザーエージェント）、ユーザーが設定したスタイル、そして作者（あなた）が書いたスタイルです。

起源を考慮した後、アルゴリズムは各ルールの重要度を評価し、`!important`でマークされたルールに優先度を与え、ソースに関係なく他のルールをオーバーライドします。

起源と重要度で絞り込んだ後、アルゴリズムは**特異性**を見ます。同じ起源と重要度のルールが複数適用される場合、より特異性の高いルールが適用されます。

特異性とは、セレクターがどれだけターゲットを絞っているかの指標で、より具体的なセレクターがより一般的なものより優先されます。

最後に、すべてが同じ場合は**出現順**が影響します。特異性が同じルールが2つある場合、CSS内で後に書かれた方が適用されます。

これが、スタイルを書く順番が結果に影響を与えることがある理由です。

例を見てみましょう。

HTMLファイルには1つの段落要素があります。CSS内には段落要素を対象にした2つのルールがあります。

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<p>example paragraph</p>
```

```css
p {
  color: blue;
}

p {
  color: green; 
}
```

:::

最初のルールはすべての段落要素のテキスト色を青に設定し、2つ目のルールはすべての段落要素のテキスト色を緑に設定しています。

では、どの色が適用されるでしょうか？段落要素には緑色が適用されます。

関連性、起源と重要度、特異性、スコープ、出現順を考慮することで、カスケードアルゴリズムはCSSが予測可能に動作するようにし、より複雑で微妙なウェブページのデザインを可能にします。
