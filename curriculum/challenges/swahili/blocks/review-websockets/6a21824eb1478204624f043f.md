## WebSockets dhidi ya HTTP

- **HTTP** hufuata mfano wa ombi-jibu: mteja anatumia ombi, seva inatuma jibu, kisha muunganisho unafungwa.
- **WebSockets** huunda muunganisho wa kudumu, wa pande zote mbili kati ya mteja na seva ambao unabaki wazi hadi upande wowote utumie ishara ya kufunga.
- HTTP ni mzuri kwa kupakia kurasa au kupata data, lakini huwa polepole na hauna ufanisi kwa programu za wakati halisi.
- Kila muunganisho wa WebSocket huanza na **mikono ya HTTP**: mteja huomba kuboreshwa, na seva hutuma jibu la `101 Switching Protocols`.

### Mtiririko wa muunganisho wa WebSocket

1. Mteja anajiunga na seva kupitia HTTP na kuomba kuboreshwa.
2. Seva inakubali na kutuma jibu la `101 Switching Protocols`.
3. Muunganisho unabaki wazi.
4. Pande zote mbili hubadilishana data wakati wowote hadi muunganisho ufungwe.

### Mawasiliano ya HTTP dhidi ya WebSocket

- **HTTP**: `Client → Request → Server → Response` (mzunguko mmoja, kisha unafungwa)
- **WebSocket**: `Client ↔ Server` (udumu, wa pande zote mbili)

### Mfano wa kueleweka

- **HTTP** ni kama kubadilishana barua: unaandika moja, unasubiri jibu, na mazungumzo yanakamilika.
- **WebSockets** ni kama simu: pande zote mbili huzungumza kwa wakati halisi kwa muda unaohitajika.

### Matumizi halisi ya WebSockets

- Programu za mazungumzo
- Michezo ya mtandaoni ya wachezaji wengi
- Sasisho za alama za michezo ya moja kwa moja
- Dashibodi za soko la hisa
- Zana za ushirikiano (mfano Google Docs)

## Muundo wa Pub/Sub

- **Pub/Sub** (Chapisha/Jisajili) ni muundo wa ujumbe unaoruhusu sehemu za mfumo kuwasiliana bila kujua moja kwa moja kuhusu kila mmoja.
- Kuna sehemu kuu tatu: **wachapishaji**, **mada**, na **wajisajili**.
  - **Mchapishaji** hutuma ujumbe kwa mada.
  - **Mada** (pia huitwa chaneli au kundi) hufanya kazi kama mpatanishi.
  - **Mjisajili** hupokea ujumbe kutoka kwa mada alizojisajili.

### Mtiririko wa ujumbe wa Pub/Sub

1. Mjisajili anajisajili kwa mada.
2. Mchapishaji hutuma ujumbe kwa mada hiyo.
3. Mfumo wa ujumbe hutuma ujumbe kwa kila mjisajili wa mada hiyo.

### Mawasiliano ya moja kwa moja dhidi ya Pub/Sub

- **Bila Pub/Sub**: `Application A → Application B` — uhusiano wa karibu; mabadiliko B yanaweza kuvunja A.
- **Kwa Pub/Sub**: `Application A → Topic → Subscribers` — A huchapisha tu; haitaji kujua nani anapokea.

### Matumizi halisi ya Pub/Sub

- Programu za mazungumzo zenye vyumba vingi
- Michezo ya mtandaoni ya wachezaji wengi
- Mifumo ya arifa
- Dashibodi za moja kwa moja
- Programu zinazoendeshwa na matukio

### WebSockets dhidi ya Pub/Sub

- **WebSockets** hutoa chaneli ya mawasiliano (barabara).
- **Pub/Sub** hutoa muundo wa ujumbe unaoamua nani anapokea ujumbe maalum (mfumo wa trafiki).
- Programu nyingi za wakati halisi hutumia zote mbili pamoja: WebSockets huweka muunganisho wazi, wakati Pub/Sub huongoza ujumbe kwa wajisajili sahihi.

## Kufanya kazi na WebSockets katika Node.js

- Node.js haina seva ya WebSocket iliyojengwa ndani. Maktaba ya `ws` ni kifurushi cha kawaida cha nje kinachotumika kwa hili.
- Sakinisha kwa: `npm install ws`

### Kuunda seva ya WebSocket

```js
const WebSocket = require("ws");

const server = new WebSocket.Server({ port: 3000 });
```

### Tukio za WebSocket

| Tukio | Kichocheo |
|---|---|
| `connection` | Mteja mpya anajiunga na seva |
| `message` | Mteja aliyeunganishwa anatumia data |
| `close` | Muunganisho unafungwa |
| `error` | Kitu kinakwenda vibaya kwenye muunganisho |

### Kutuma ujumbe kwa mteja

```js
socket.send("Hello from the server");
```

### Kutuma ujumbe kwa wateja wote waliounganishwa

- `server.clients` ina wateja wote waliounganishwa.
- Angalia `readyState === WebSocket.OPEN` kabla ya kutuma ili kuepuka makosa kwa wateja waliotengana.

```js
server.on("connection", (socket) => {
  socket.on("message", (message) => {
    server.clients.forEach((client) => {
      if (client.readyState === WebSocket.OPEN) {
        client.send(message.toString());
      }
    });
  });
});
```

- **Kutuma ujumbe kwa wateja wengi kwa wakati mmoja** ni muundo wa kutuma ujumbe kwa wateja wengi waliounganishwa kwa pamoja — hutumika katika programu za mazungumzo, arifa za moja kwa moja, sasisho za michezo, na matukio ya michezo ya wachezaji wengi.
