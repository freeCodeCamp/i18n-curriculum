# --interactive--

Статичні властивості та методи належать самому класу, а не окремим екземплярам класу. Ви можете отримати до них доступ безпосередньо через ім’я класу, не створюючи екземпляр класу. Вони визначені всередині класів, щоб інкапсулювати пов’язану функціональність.

Ви можете визначити статичний метод, написавши ключове слово `static` перед ім’ям методу.

```js
class MyClass {
  static staticMethod() { ... }
}
```

Потім ви можете викликати статичний метод безпосередньо на класі, використовуючи точкову нотацію та передаючи необхідні аргументи:

```js
MyClass.staticMethod();
```

Зверніть увагу, що ви можете викликати метод без створення екземпляра класу. Це одна з ключових характеристик статичних методів.

Ось приклад. Припустимо, що ми створюємо застосунок для фільмів і хочемо мати можливість порівнювати фільми за рейтингом.

Ми могли б розглядати цей метод порівняння як метод вищого рівня, який не належить жодному конкретному фільму:

```js
if (movieA.rating < movieB.rating) {
  console.log(`${movieB.title} has a higher rating.`);
} 
```

Це як більш загальний метод, пов’язаний із класом `Movie`.

Для зручності читання та підтримки коду було б корисно визначити його всередині класу `Movie`, щоб усі пов’язані методи були розташовані поруч.

Отже, це ідеальний кандидат на статичний метод. Ви можете побачити його тут, одразу під конструктором:

:::interactive_editor

```js
class Movie {

  constructor(title, rating) {
    this.title = title;
    this.rating = rating;
  }

  static compareMovies(movieA, movieB) {
    if (movieA.rating > movieB.rating) {
      console.log(`${movieA.title} has a higher rating.`);
    } else if (movieA.rating < movieB.rating) {
      console.log(`${movieB.title} has a higher rating.`);
    } else {
      console.log("These movies have the same rating.");
    }
  }

}

let movieA = new Movie("Movie A", 80);
let movieB = new Movie("Movie B", 45);
console.log(movieA);
```

:::

Статичний метод визначений за допомогою ключового слова `static` і називається `compareMovies`. Він має два параметри: `movieA` і `movieB`. Це будуть екземпляри класу `Movie`.

Ми порівнюватимемо їх за рейтингом у діапазоні від `0` до `100`. Ця логіка реалізована за допомогою умовного оператора, і вона виведе відповідне повідомлення залежно від того, який фільм має вищий рейтинг.

Після визначення методу ви можете викликати його на класі. Але спочатку потрібно мати готові аргументи, визначені у вашій програмі.

У цьому випадку метод приймає два екземпляри фільмів як аргументи, тому ми визначаємо ці екземпляри тут:

```js
let movieA = new Movie("Movie A", 80);
let movieB = new Movie("Movie B", 45);
```

Ви бачите, що `movieA` має вищий рейтинг, ніж `movieB`. Давайте подивимося на вихідні дані цього методу.

Щоб викликати метод, потрібно просто використати точкову нотацію на самому класі.
Ви пишете ім’я класу (`Movie`), потім крапку, а потім ім’я статичного методу (`compareMovies`).

Потім передаєте аргументи в круглих дужках. У цьому випадку це два екземпляри фільмів, які потрібні методу.

```js
Movie.compareMovies(movieA, movieB);
```

Ось оновлений приклад:

:::interactive_editor

```js
class Movie {

  constructor(title, rating) {
    this.title = title;
    this.rating = rating;
  }

  static compareMovies(movieA, movieB) {
    if (movieA.rating > movieB.rating) {
      console.log(`${movieA.title} has a higher rating.`);
    } else if (movieA.rating < movieB.rating) {
      console.log(`${movieB.title} has a higher rating.`);
    } else {
      console.log("These movies have the same rating.");
    }
  }

}

let movieA = new Movie("Movie A", 80);
let movieB = new Movie("Movie B", 45);
Movie.compareMovies(movieA, movieB);
console.log(movieA);
```

:::

Статичні методи також корисні для реалізації «фабричних» методів. Фабричний метод — це метод, який ви визначаєте додатково до конструктора, щоб створювати об’єкти на основі певних критеріїв.

Ось приклад із класом `Pizza`. Статичний метод `createMargherita` — це фабричний метод, який можна викликати, щоб створити екземпляр піци Маргарита з уже встановленим типом і ціною.

```js
class Pizza {
  constructor(type, price) {
    this.type = type;
    this.price = price;
  }

  static createMargherita() {
    return new this("Margherita", 6.99);
  }
}
```

Це також підкреслює дуже важливий момент про статичні методи.

Значення ключового слова `this` у статичних методах — це сам клас, оскільки статичний метод належить класу.

Ось чому ми можемо використовувати `this` для створення нового екземпляра класу `Pizza`.

Якщо викликати цей метод на самому класі `Pizza` і призначити повернений екземпляр змінній, як у цьому прикладі:

```js
let myPizza = Pizza.createMargherita();
```

Ви можете використовувати його будь-де у своєму коді. Наприклад, вивести в консоль:

```js
console.log(myPizza);
```

Ось вихідні дані:

```js
Pizza { type: 'Margherita', price: 6.99 }
```

Ви також можете використовувати точкову нотацію, щоб викликати його методи та отримувати доступ до властивостей, ось так:

```js
console.log(myPizza.type);
```

Ось повний приклад:

```js
class Pizza {
  constructor(type, price) {
    this.type = type;
    this.price = price;
  }

  static createMargherita() {
    return new this("Margherita", 6.99);
  }
}

let myPizza = Pizza.createMargherita();
console.log(myPizza);
console.log(myPizza.type);
```

Окрім методів, ви також можете визначати статичні властивості за допомогою ключового слова `static`.

У цьому прикладі ми маємо статичну властивість `numberOfPizzasSold`.

```js
class Pizza {
  static numberOfPizzasSold = 0;

  constructor(type) {
    this.type = type;
    Pizza.numberOfPizzasSold++;
  }
}
```

Вона статична, бо не належить жодному конкретному екземпляру піци, а належить самому класу.

Вона має ініціальне значення `0` і оновлюється щоразу, коли створюється новий екземпляр.

Якщо створити два екземпляри піци, значення оновиться двічі:

```js
let pizza1 = new Pizza("Margherita");
let pizza2 = new Pizza("Neapolitan");
```

Щоб отримати доступ до значення статичної властивості, потрібно просто використати точкову нотацію на самому класі, оскільки властивість належить класу.

```js
class Pizza {
  static numberOfPizzasSold = 0;

  constructor(type) {
    this.type = type;
    Pizza.numberOfPizzasSold++;
  }
}

let pizza1 = new Pizza("Margherita");
let pizza2 = new Pizza("Neapolitan");

console.log(Pizza.numberOfPizzasSold);
```

У цьому випадку вихідні дані — `2`, бо було продано дві піци.

Це основи статичних властивостей і методів у JavaScript. Розуміння статичних членів є важливим для створення багаторазових і ефективних класів.
