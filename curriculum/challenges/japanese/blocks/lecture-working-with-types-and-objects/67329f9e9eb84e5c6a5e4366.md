# --interactive--

これまでのモジュールでは、次のような文字列リテラルを使ってきました。

```js
const greeting = "Hello, World!";
```

しかし、JavaScriptには文字列オブジェクトもあります。文字列オブジェクトと文字列プリミティブはどちらもテキストを扱うために使われますが、内部の動作は異なります。文字列オブジェクトは文字列コンストラクタ関数を使って作成され、プリミティブ値をオブジェクトでラップします。文字列オブジェクトを作成する方法は次の通りです。

:::interactive_editor

```js
const greetingObject = new String("Hello, World!");

console.log(typeof greetingObject); // "object"
```

:::

`typeof`演算子を使うと、結果が文字列型ではなくオブジェクト型であることがわかります。文字列プリミティブで`.length`のようなプロパティをどうやって使えるのか疑問に思ったかもしれません。

文字列プリミティブで`length`プロパティを使うと、JavaScriptは一時的に文字列プリミティブを文字列オブジェクトでラップして操作を行います。だからこそ、`length`プロパティや`repeat()`、`concat()`、`slice()`のようなさまざまなメソッドを使うことができます。これらのメソッドやプロパティはインスタンスメソッド、インスタンスプロパティ、静的メソッドと呼ばれます。これらの仕組みについては今後のモジュールで詳しく学びます。

文字列オブジェクトと文字列プリミティブの大きな違いの一つは、メモリとパフォーマンスに関する点です。文字列プリミティブは通常、文字列オブジェクトよりもメモリ効率が良く高速です。コードでは主に文字列プリミティブを使いますが、文字列オブジェクトの仕組みを理解しておくことは役に立ちます。
