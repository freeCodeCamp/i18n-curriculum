# --description--

Un preprocessore CSS è uno strumento che estende il CSS standard con funzionalità potenti come variabili, mixin, annidamento e ereditarietà dei selettori.

Alcune di queste funzionalità, come le variabili e l’annidamento, sono ora supportate o stanno ricevendo più supporto nel CSS nativo, e questa tendenza probabilmente continuerà. Tuttavia, le versioni precedenti del CSS nativo non supportavano queste funzionalità.

Ecco perché i preprocessori CSS sono diventati così diffusi in primo luogo. Prendono le regole e gli stili CSS che scrivi, insieme alla loro sintassi estesa, e li compilano in un file CSS nativo che i browser possono interpretare.

Usando un preprocessore CSS, puoi strutturare il tuo codice CSS in modo più riutilizzabile e logico. Avrai anche accesso a funzionalità potenti, come i mixin, che non sono supportate direttamente dal CSS.

Alcuni dei preprocessori CSS più popolari sono Sass, Less e Stylus.

Parliamo di Sass.

Sass sta per “Syntactically Awesome Style Sheets”. È compatibile con tutte le versioni di CSS ed è mantenuto da una grande comunità di sviluppatori.

Sass supporta funzionalità come:

- Variabili, così puoi archiviare e riutilizzare valori in tutto il foglio di stile.

- Regole CSS annidate, così puoi creare una gerarchia visiva nel tuo file.

- Moduli, così puoi dividere i tuoi stili in più fogli di stile.

- Mixin, così puoi riutilizzare dichiarazioni CSS in tutto il sito.

- Ereditarietà, così più selettori CSS possono condividere proprietà.

- E operatori, per operazioni matematiche di base.

Sass supporta anche due sintassi.

La sintassi SCSS sta per Sassy CSS, che è un superset del CSS. Questo significa che SCSS espande la sintassi base del CSS.

SCSS è la sintassi più comune che userai e troverai lavorando con Sass. Richiede l’uso delle parentesi graffe (`{` e `}`) intorno alle proprietà CSS e dei punti e virgola (`;`) alla fine delle dichiarazioni CSS, proprio come il CSS nativo.

Questi file hanno un’estensione `.scss`. Ecco un esempio:

```scss
$primary-color: #3498eb;

header {
  background-color: $primary-color;
}
```

Quello che vedi qui, in alto, è una variabile definita in SCSS. Questa variabile viene usata nella regola CSS sottostante.

Confrontiamo SCSS con una sintassi meno usata, la sintassi indentata. Questa è anche conosciuta come “sintassi Sass” dato che era la sintassi originale di Sass.

Questa sintassi si basa sull’indentazione per definire le regole. Ecco un esempio:

```scss
$primary-color: #3498eb

header
  background-color: $primary-color
```

Nota che non ci sono parentesi graffe intorno alla regola CSS, né punti e virgola alla fine delle dichiarazioni CSS. In alto puoi vedere anche una variabile. Nota che il suo nome inizia con un segno di dollaro (`$`) in questa sintassi, che è diverso dalle variabili native del CSS, ma funziona in modo molto simile.

Sass supporta anche una funzionalità potente, chiamata mixin.

I mixin ti permettono di raggruppare più proprietà CSS e i loro valori sotto un nome e riutilizzare quel blocco di codice CSS in tutto il foglio di stile.

Questo rende il tuo codice CSS meno ripetitivo e più facile da mantenere perché, se cambi qualcosa in quel blocco, la modifica viene applicata ovunque usi il mixin.

Poiché i mixin possono avere nomi personalizzati e descrittivi, possono aiutarti a capire cosa fa ogni blocco di codice CSS, rendendo i tuoi fogli di stile più facili da comprendere.

E i mixin promuovono anche la coerenza negli stili applicando stili uniformi.

Questo è un esempio di mixin in sintassi SCSS:

```scss
@mixin center-flex {
  display: flex;
  justify-content: center;
  align-items: center;
}
```

Per definire un mixin, inizi scrivendo la regola at `@mixin`, seguita dal nome del mixin. In questo caso, il mixin si chiama `center-flex`. Ha tre proprietà CSS per centrare gli elementi usando Flexbox.

Poi, una volta definito il mixin, useresti la regola at `@include` per includere quelle proprietà in una regola CSS. Devi solo scrivere `@include` seguito dal nome del mixin. In questo caso, il nome è `center-flex`:

```scss
section {
  @include center-flex;
  height: 500px;
  background-color: #3289a8;
}
```

Questo è il codice completo con il mixin e la regola CSS:

```scss
@mixin center-flex {
  display: flex;
  justify-content: center;
  align-items: center;
}

section {
  @include center-flex;
  height: 500px;
  background-color: #3289a8;
}
```

Puoi includere il mixin in quante regole CSS vuoi. Se devi fare modifiche, devi cambiare solo il mixin stesso e le modifiche verranno applicate automaticamente ovunque. Questo esempio è in sintassi SCSS. Nota che ha le parentesi graffe e i punti e virgola.

Ecco l’equivalente in sintassi indentata, anche conosciuta come sintassi Sass, senza parentesi graffe o punti e virgola:

```scss
@mixin center-flex
  display: flex
  justify-content: center
  align-items: center

section
  @include center-flex
  height: 500px
  background-color: #3289a8
```

E questo è il codice CSS compilato, quello che il browser interpreterà effettivamente dopo che il file è stato compilato in CSS nativo:

```css
section {
  display: flex;
  justify-content: center;
  align-items: center;
  height: 500px;
  background-color: #3289a8;
}
```

Nota che include le tre proprietà del mixin in alto: `display`, `justify-content` e `align-items`.

Questo è solo un esempio semplice di cosa sono capaci i preprocessori CSS. Hanno molte funzionalità potenti che vale la pena imparare e applicare nel tuo flusso di lavoro quotidiano.

Tuttavia, hanno anche potenziali svantaggi di cui dovresti essere consapevole.

Innanzitutto, i preprocessori CSS devono compilare le regole CSS in CSS standard. Anche se il sovraccarico potenziale di questo processo di compilazione è spesso minimo in termini di tempo e risorse, è qualcosa da considerare per progetti reali.

Inoltre, possono creare sfide nel debugging, dato che i browser usano direttamente il CSS compilato. Scoprire cosa sta generando uno stile problematico dalla sintassi estesa può richiedere qualche passaggio in più rispetto al CSS standard.

Tuttavia, i vantaggi dei preprocessori CSS di solito superano gli svantaggi, specialmente per progetti complessi. Possono essere molto utili per scrivere CSS più pulito, riutilizzabile, meno ripetitivo e scalabile.
