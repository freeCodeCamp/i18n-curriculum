## JavaScript 函式庫與框架

- JavaScript 函式庫和框架提供快速解決常見問題的方案，並透過預先建置的程式碼加速開發。
- 函式庫通常更專注於提供特定任務的解決方案，例如操作 DOM、處理事件或管理 AJAX 請求。
- JavaScript 函式庫的例子有 jQuery 和 React。
- 框架則提供更明確的應用程式建構結構，通常附帶一套開發者必須遵循的規則和慣例。
- 框架的例子包括 Angular 和 Next.js（React 的元框架）。
- **單頁應用程式**（SPA）是載入單一 HTML 頁面，並在使用者與應用程式互動時動態更新該頁面，而不重新載入整個頁面。
- SPA 使用 JavaScript 管理應用程式狀態並呈現內容，通常透過框架來建構複雜的使用者介面。
- SPA 相關問題包括：

  - 螢幕閱讀器難以處理動態更新的內容。
  - 使用者在應用程式內導航時 URL 不會改變，導致難以書籤、回退或分享特定頁面。
  - 若應用程式龐大，初次載入時間可能較慢，因為所有資源需一次載入。

## React

- React 是一個流行的 JavaScript 函式庫，用於建構使用者介面和網頁應用程式。
- React 的核心概念是創建可重複使用的 UI 組件，這些組件能在資料變更時獨立更新和呈現。
- React 允許開發者根據應用程式狀態描述 UI 應該呈現的樣貌，當資料或狀態改變時，React 會更新並呈現正確的組件。

## React 組件

- 組件是 React 應用程式的基石，讓開發者能將複雜的使用者介面拆解成較小且易於管理的部分。
- UI 使用 JSX 描述，JSX 是 JavaScript 語法的擴充，允許開發者在 JavaScript 中撰寫類似 HTML 的程式碼。
- 組件基本上是回傳 UI 片段的 JS 函式或類別。

以下是一個簡單的 React 組件範例，呈現問候訊息：

```jsx
function Greeting() {
  const name = 'Anna';
  return <h1>Welcome, {name}!</h1>;
}
```

要使用該組件，只需呼叫：

```jsx
  <Greeting />
```

## 匯入與匯出 React 組件

- 匯出讓組件可供其他檔案使用，匯入則允許檔案使用其他地方匯出的組件。
- React 有兩種匯出方式：預設匯出和具名匯出。

**預設匯出**

- 一個檔案只能有一個預設匯出，適合主要包含單一組件的檔案。
- 你可以這樣匯出組件作為預設匯出：

```jsx
// City.js
function City() {
  return <p>New York</p>;
}

export default City;
```

- 也可以在組件定義同一行匯出：

```jsx
export default function City() {
  return <p>New York</p>;
}
```

- 匯入預設匯出時，使用 `import` 關鍵字且不加大括號：

```jsx
// App.js
import City from './City';

function App() {
  return (
    <div>
      <h1>My favorite city is:</h1>
      <City />
    </div>
  );
}
```

**具名匯出**

- 具名匯出允許檔案分享多個組件，匯入時必須使用與匯出時相同的名稱。
- 你可以這樣從同一檔案匯出多個組件：

```jsx
// Animals.jsx
export function Cat() {
  return <h2>Mr. Whiskers</h2>;
}

export function Dog() {
  return <h2>Fido</h2>;
}
```

- 匯入具名匯出時，使用大括號包住正確的匯出名稱：

```jsx
import { Cat, Dog } from './Animals';
```

- 匯入時可用 `as` 關鍵字重新命名具名匯出：

```jsx
import { Cat as Kitty } from './Animals';
```

**混合預設與具名匯出**

- 一個檔案可同時有一個預設匯出和多個具名匯出：

```jsx
// Animals.jsx
export default function Cat() {
  return <h2>Mr. Whiskers</h2>;
}

export function Dog() {
  return <h2>Fido</h2>;
}
```

- 匯入混合匯出時，預設匯出放前面（不加大括號），具名匯出放後面（加大括號）：

```jsx
import Cat, { Dog } from './Animals';
```

## 使用 Vite 設定 React 專案

- 專案設定工具和 CLI 提供快速且簡易的方式啟動新專案，讓開發者專注於撰寫程式碼而非設定。
- Vite 是流行的專案設定工具，可用於 React。
- 在終端機使用以下指令建立新專案：

```bash
npm create vite@latest my-react-app -- --template react
```

此指令會使用 Vite 的 React 範本建立名為 `my-react-app` 的新 React 專案。在專案目錄中，你會看到 `package.json` 檔案，列出專案相依性和指令。

- 執行專案時，切換至專案目錄並執行以下指令：

```bash
cd my-react-app # path to the project directory
npm install # installs the dependencies listed in the package.json file
```

- 安裝完相依性後，你會看到專案中新出現 `node_modules` 資料夾。

- `node_modules` 資料夾存放專案所需的所有套件和函式庫。

- 執行專案指令如下：

```bash
npm run dev
```

- 接著，開啟瀏覽器並前往 `http://localhost:5173`，即可看到 React 應用程式運行。

- 若要查看起始範本的程式碼，可進入專案內的 `src` 資料夾，會看到 `App.jsx` 檔案。

## 在 React 組件中傳遞 props

- 在 React 中，props（屬性）是從父組件傳遞資料到子組件的方式。此機制用於創建可重複使用且動態的 UI 元素。
- props 可以是任何 JavaScript 值。要從父組件傳遞 props 給子組件，需在父組件的 JSX 中使用子組件時，將 props 作為屬性加入。以下是簡單範例：

```jsx
// Parent component
function Parent() {
  const name = 'Anna';
  return <Child name={name} />;
}

// Child component
function Child(props) {
  return <h1>Hello, {props.name}!</h1>;
}
```

你也可以使用展開運算子 `(...)` 傳遞多個 props，前提是先將它們轉成物件。範例如下：

```jsx
// Parent component
function Parent() {
  const person = {
    name: 'Anna',
    age: 25,
    city: 'New York'
  };
  return <Child {...person} />;
}
```

此程式碼中，展開運算子 `{...person}` 將 person 物件轉成個別的 props 傳給 Child 組件。

## React 中的條件渲染

- React 的條件渲染讓你能根據應用程式內的特定條件或狀態，呈現不同內容，打造動態使用者介面。
- React 中有多種條件渲染方式，常見的是使用三元運算子。範例如下：

```jsx
function Greeting({ isLoggedIn }) {
  return (
    <div>
      {isLoggedIn ? <h1>Welcome back!</h1> : <h1>Please log in</h1>}
    </div>
  );
}
```

- 另一種條件渲染方式是使用邏輯 AND（`&&`）運算子，適用於只在條件成立時才呈現內容。範例如下：

```jsx
function Greeting({ user }) {
  return (
    <div>
      {user && <h1>Welcome, {user.name}!</h1>}
    </div>
  );
}
```

上述程式碼中，只有當 user 物件為真值時，`h1` 元素才會被呈現。

你也可以直接使用 `if` 陳述式，如下：

```js
function Greeting({ isLoggedIn }) {
  if (isLoggedIn) {
    return <h1>Welcome back!</h1>;
  }
  return <h1>Please sign in</h1>;
}
```

## React 中的列表渲染

- 在 React 中渲染列表是常見任務。
- 可使用 JS 陣列的 `map()` 方法迭代項目，回傳新的 JSX 元素陣列。
- 例如，若有一個名字陣列想渲染成列表，可這樣做：

```jsx
function NameList({ names }) {
  return (
    <ul>
      {names.map((name, index) => (
        <li key={`${name}-${index}`}>{name}</li>
      ))}
    </ul>
  );
}
```

- 記得為每個列表項目提供唯一的 key，幫助 React 管理更新和渲染。透過這些技巧，你可以在 React 應用中建立彈性、高效且動態的列表。

## React 中的行內樣式

- React 允許你使用 JavaScript 物件直接對 JSX 元素套用 CSS 行內樣式。
- 使用 style 屬性套用行內樣式，style 屬性接受一個物件，鍵為駝峰式 CSS 屬性名稱，值為對應的屬性值。範例如下：

```js
function Greeting() {
  return (
    <h1
      style={{ color: 'blue', fontSize: '24px', backgroundColor: 'lightgray' }}
    >
      Hello, world!
    </h1>
  );
}

export default Greeting;
```

你也可以將樣式抽出成獨立物件，並在 `style` 屬性中引用，如下：

```jsx
function Greeting() {

  const styles = {
    color: 'blue',
    fontSize: '24px',
    backgroundColor: 'lightgray'
  };

  return <h1 style={styles}>Hello, world!</h1>;
}

export default Greeting;
```

- 行內樣式支援動態樣式，可根據 props 或狀態條件套用。以下範例示範如何根據 prop 條件套用樣式：

```jsx
function Greeting({ isImportant }) {

  const styles = {
    color: isImportant ? 'red' : 'black',
    fontSize: isImportant ? '24px' : '16px'
  };

  return <h1 style={styles}>Hello, world!</h1>;
}

export default Greeting;
```

- 上述程式碼中，`color` 和 `fontSize` 樣式會根據 `isImportant` prop 有條件地設定。

## React 中的事件處理

- **合成事件系統**：React 使用合成事件系統包裝原生事件，如 `click`、`keydown` 和 `submit` 事件。React 事件處理函式使用駝峰式命名（例如 `onClick`、`onSubmit` 等）。

以下是 React 中使用 `onClick` 屬性於 `button` 元素的範例：

```jsx
function handleClick() {
  console.log("Button clicked!");
}

<button onClick={handleClick}>Click Me</button>;
```

在 React 中，事件處理函式通常以 `handle` 為前綴，表示負責處理事件，例如 `handleClick` 或 `handleSubmit`。

當使用者觸發事件時，React 會將合成事件物件傳給處理函式。此物件行為類似原生 JavaScript 的 Event 物件，提供 `type`、`target` 和 `currentTarget` 等屬性。

若要阻止預設行為（例如 `onSubmit` 事件中瀏覽器重新整理），可呼叫 `preventDefault()` 方法：

```jsx
function handleSubmit(event) {
  event.preventDefault();
  console.log("Form submitted!");
}

<form onSubmit={handleSubmit}>
  <input type="text" />
  <button>Submit</button>
</form>;
```

你也可以用箭頭函式包裹處理函式，如下：

```jsx
function handleDelete(id) {
  console.log("Deleting item:", id);
}

<button onClick={() => handleDelete(1)}>Delete Item</button>;
```

## 使用 State 和 `useState` Hook

- **State 定義**：在 React 中，state 是包含組件資料的物件。當 state 更新時，組件會重新渲染。React 將 state 視為不可變，應避免直接修改。
- **`useState()` Hook**：`useState` hook 讓你在函式組件中宣告 state 變數。基本語法如下：

```js
const [stateVariable, setStateFunction] = useState(initialValue);
```

state 變數包含：

- `stateVariable`：目前的 state 值
- `setStateFunction`（設值函式）：更新 state 變數
- `initialValue`：設定初始 state

以下是 `Counter` 組件的完整範例：

```jsx
import { useState } from "react";

function Counter() {
  const [count, setCount] = useState(0);

  return (
    <div>
      <h2>{count}</h2>

      <button onClick={() => setCount(count - 1)}>Decrement</button>
      <button onClick={() => setCount(count + 1)}>Increment</button>
    </div>
  );
}

export default Counter;
```

## React 的渲染與組件

- **定義**：React 中的渲染是組件出現在使用者介面（通常是瀏覽器）的過程。渲染包含三個階段：觸發、渲染和提交。

觸發階段發生在 React 偵測到狀態或 props 變更，可能需要更新 UI。

觸發後進入渲染階段，React 重新評估組件並決定要呈現什麼。此時 React 使用虛擬 DOM（virtual DOM）輕量複本快速判斷組件需變更的部分。

提交階段則是 React 將虛擬 DOM 的變更套用到真實 DOM，最終呈現在螢幕上。

## 更新 State 中的物件與陣列

- **更新 State 中的物件**：更新物件時，應先建立新物件或複製現有物件，再設定新的 state。放入 state 的物件應視為唯讀。以下範例示範設定使用者姓名、年齡和城市，`handleChange` 函式用於處理使用者資訊更新：

```jsx
import { useState } from "react";

function Profile() {
  const [user, setUser] = useState({ name: "John Doe", age: 31, city: "LA" });

  const handleChange = (e) => {
    const { name, value } = e.target;

    setUser((prevUser) => ({...prevUser, [name]: value}));
  };

  return (
    <div>
      <h1>User Profile</h1>
      <p>Name: {user.name}</p>
      <p>Age: {user.age}</p>
      <p>City: {user.city}</p>

      <h2>Update User Age </h2>
      <input type="number" name="age" value={user.age} onChange={handleChange} />

      <h2>Update User Name </h2>
      <input type="text" name="name" value={user.name} onChange={handleChange} />

      <h2>Update User City </h2>
      <input type="text" name="city" value={user.city} onChange={handleChange} />
    </div>
  );
}

export default Profile;
```

- **更新 State 中的陣列**：更新陣列時，避免直接使用 `push()` 或 `pop()` 等方法修改陣列，應建立新陣列更新 state：

```jsx
const addItem = () => {
  const newItem = {
    id: items.length + 1,
    name: `Item ${items.length + 1}`,
  };

  // Creates a new array
  setItems((prevItems) => [...prevItems, newItem]);
};
```

若要從陣列移除項目，應使用 `filter()` 方法，該方法會回傳過濾後的新陣列：

```jsx
const removeItem = (id) => {
  setItems((prevItems) => prevItems.filter((item) => item.id !== id));
};
```

## 使用 Refs 參考值

- **`ref` 屬性**：可透過 `ref` 屬性存取 React 中的 DOM 節點。以下範例展示如何使用 `ref` 來聚焦 `input` 元素，`current` 屬性用於存取該 `ref` 的當前值：

```jsx
import { useRef } from "react";

const Focus = () => {
  const inputRef = useRef(null);

  const handleFocus = () => {
    if (inputRef.current) {
      inputRef.current.focus();
    }
  };

  return (
    <div>
      <input ref={inputRef} type="text" placeholder="Enter text" />
      <button onClick={handleFocus}>Focus Input</button>
    </div>
  );
};

export default Focus;
```

## 使用 `useEffect` Hook

- **`useEffect()` Hook**：React 中的 effect 是組件渲染流程外發生的任何事，例如抓取資料、更新瀏覽器標籤標題、讀寫本地儲存、取得使用者位置等，稱為副作用。React 提供 `useEffect` hook 讓你處理這些副作用。`useEffect` 讓你在組件渲染或更新後執行函式。

```js
import { useEffect } from "react";

useEffect(() => {
 // Your side effect logic (usually a function) goes here
}, [dependencies]);
```

effect 函式在組件渲染後執行，選用的 `dependencies` 參數控制 effect 執行時機。

`dependencies` 可是包含「反應值」（state、props、函式、變數等）的陣列、空陣列或完全省略。以下說明各種選項如何控制 `useEffect`：

- 若 `dependencies` 是包含一個或多個反應值的陣列，當它們改變時 effect 會執行。
- 若 `dependencies` 是空陣列，`useEffect` 僅在組件首次渲染時執行一次。
- 若省略 `dependencies`，effect 每次組件渲染或更新時都會執行。

## 如何創建自訂 Hooks

- **自訂 Hooks**：自訂 hook 讓你從組件中抽取可重複使用的邏輯，例如資料抓取、狀態管理、切換功能和追蹤線上狀態等副作用。React 內建的 hooks 都以 `use` 開頭，自訂 hook 也應遵循此慣例。

以下是創建 `useDebounce` hook 的範例：

```jsx
function useDebounce(value, delay) {
  const [debouncedValue, setDebouncedValue] = useState(value);

  useEffect(() => {
    const handler = setTimeout(() => {
      setDebouncedValue(value);
    }, delay);

    return () => {
      clearTimeout(handler);
    };
  }, [value, delay]);

  return debouncedValue;
}

export { useDebounce };
```

## React 中的表單操作

- **受控輸入**：將輸入欄位值存放在 state，並透過 `onChange` 事件更新。這讓你完全控制表單資料，並可即時驗證與條件渲染。

```jsx
import { useState } from "react";

function App() {
  const [name, setName] = useState("");

  const handleChange = (e) => {
    setName(e.target.value);
  };

  const handleSubmit = (e) => {
    e.preventDefault();
    console.log(name);
  };

  return (
    <>
      <form onSubmit={handleSubmit}>
        <label htmlFor="name">Your name</label> <br />
        <input value={name} id="name" onChange={handleChange} type="text" />
        <button type="submit">Submit</button>
      </form>
    </>
  );
}

export default App;
```

- **非受控輸入**：非受控輸入不透過 `useState` hook 管理，而是由 DOM 維護內部狀態。因為 DOM 控制輸入值，你需用 `ref` 取得輸入欄位的值。

```jsx
import { useRef } from "react";

function App() {
 const nameRef = useRef();

 const handleSubmit = (e) => {
   e.preventDefault();
   console.log(nameRef.current.value);
 };

 return (
   <form onSubmit={handleSubmit}>
     <label htmlFor="name">Your</label>{" "}
     <input type="text" ref={nameRef} id="name" />
     <button type="submit">Submit</button>
   </form>
 );
}

export default App;
```

## 使用 `useActionState` Hook

- **伺服器動作**：這些函式在伺服器端執行，允許直接在伺服器處理表單，無需 API 端點。以下為 Next.js 應用範例：

```js
"use server";

async function submitForm(formData) {
 const name = formData.get("name");
 return { message: `Hello, ${name}!` };
}
```

`"use server"` 指令標記函式為伺服器動作。

- **`useActionState` Hook**：此 hook 根據表單提交結果更新狀態。`useActionState` hook 的基本語法如下：

```js
const [state, action, isPending] = useActionState(actionFunction, initialState, permalink);
```

- `state` 是動作返回的當前狀態。
- `action` 是觸發伺服器動作的函式。
- `isPending` 是布林值，表示動作是否正在執行。
- `actionFunction` 參數是伺服器動作本身。
- `initialState` 是動作執行前狀態的起始值參數。
- `permalink` 是可選字串，包含表單修改的唯一頁面 URL。

## React 中的資料抓取

- **資料抓取選項**：React 中有多種抓取資料方式，可使用原生 Fetch API，或第三方工具如 Axios 或 SWR。
- **抓取資料時常用的狀態變數**：無論使用哪種方式抓取資料，通常需追蹤三種狀態：資料本身、資料是否仍在抓取中，以及抓取過程中可能發生的錯誤。

```js
const [data, setData] = useState(null);
const [loading, setLoading] = useState(true);
const [error, setError] = useState(null);
```

因為資料抓取是副作用，最好在 `Fetch API` hook 中使用 `useEffect`。

```js
useEffect(() => {
  const fetchData = async () => {
    try {
      const res = await fetch("https://jsonplaceholder.typicode.com/posts");
      
      if (!res.ok) {
        throw new Error("Network response was not ok");
      }

      const data = await res.json();
      setData(data);
    } catch (err) {
      setError(err);
    } finally {
      setLoading(false);
    }
  };

  fetchData();
}, []);
```

```jsx
    ))}
  </ul>
);
```

## 使用 `useOptimistic` Hook

- **`useOptimistic` Hook**：此 hook 用於在背景等待非同步動作完成時保持 UI 響應，協助管理 UI 中的「樂觀更新」策略，即根據預期結果立即更新 UI，例如等待伺服器回應。

基本語法如下：

```js
const [optimisticState, addOptimistic] = useOptimistic(actualState, updateFunction);
```

- `optimisticState` 是立即更新的暫時狀態，提升使用者體驗。
- `addOptimistic` 是在實際狀態變更前套用樂觀更新的函式。

```js
"use client";

    tasks,

  );

    e.preventDefault();
    const formData = new FormData(e.target);

    addOptimisticTask(formData.get("task"));

    addTask(formData);
    e.target.reset();
  }

  return <>{/* UI */}</>;
}
```

## React Router

- **介紹**：React Router 是第三方函式庫，讓你為 React 應用程式新增路由功能。首先需在現有 React 專案安裝 React Router：

```bash

```

接著在 `main.jsx` 或 `index.jsx` 檔案中設定路由結構：

```jsx
import { StrictMode } from "react";
import { createRoot } from "react-dom/client";
import { BrowserRouter, Routes, Route } from "react-router";
import App from "./App.jsx";

import "./index.css";

createRoot(document.getElementById("root")).render(
  <StrictMode>
    <BrowserRouter>
      <Routes>
        <Route path="/" element={<App />} />
      </Routes>
    </BrowserRouter>
  </StrictMode>
);
```

`path` 和 `element` 用於將 URL 與 UI 組件綁定。此例中設定首頁路由指向 `App` 組件。

- **多視圖與路由設定**：大型應用常有多個視圖與路由設定，如下：

```jsx
<Routes>
  <Route index element={<Home />} />
  <Route path="about" element={<About />} />

  <Route path="products">
    <Route index element={<ProductsHome />} />
    <Route path=":category" element={<Category />} />
    <Route path=":category/:productId" element={<ProductDetail />} />
    <Route path="trending" element={<Trending />} />
  </Route>
</Routes>
```

範例中 `index` prop 代表該路徑段的預設路由。`Home` 組件會在根路徑 `/` 顯示，`ProductsHome` 組件會在 `/products` 路徑顯示。

- **巢狀路由**：可在路由內巢狀其他路由，子路由路徑會附加在父路由路徑後。

```jsx
<Route path="products">
  <Route path="trending" element={<Trending />} />
</Route>
```

上述例子中，熱門商品路徑為 products/trending。

- **動態路由段**：動態路由段指 URL 路徑中任意部分為動態。

```jsx
<Route path=":category" element={<Category />} />
```

此例中有名為 `category` 的動態段。使用者瀏覽 products/brass-instruments 時，視圖會切換到 `Category` 組件，並可根據該段動態抓取資料。

```js
import { useParams } from "react-router";

export default function Category() {

   {/* rest of code goes here */}
}
```

## 狀態管理

- **Context API**：Context 指父組件提供資訊給子組件，無需透過 props 明確傳遞。`createContext` 用於建立 context 物件，代表其他組件將讀取的上下文。`Provider` 用於供應 context 值給子組件。

## 真實效能與感知效能的差異

- **感知效能**：指使用者對網站效能的感受，評估其響應速度和可靠性。這是主觀衡量，難以量化，但非常重要，因為使用者體驗決定網站成敗。
- **真實效能**：網站的客觀且可測量效能，透過頁面載入時間、伺服器回應時間和渲染時間等指標衡量。這些指標受網路和程式碼多種因素影響。

## 改善感知效能的技巧

- **延遲載入**：盡可能減少初次載入時間，將非必要資源在背景載入。
- **減少字型延遲**：若網站使用自訂字型，應盡量減少字型載入延遲，避免閃爍或顯示備用字型。建議使用與自訂字型相似的備用字型，使變化更細微。
- **使用載入指示器**：當使用者點擊元素觸發長時間處理時，顯示載入指示器，讓使用者感覺連結且投入，縮短等待感。

## 核心效能概念

- **原始碼順序**：指 HTML 元素在文件中的結構順序，決定先載入什麼，對效能和可及性有重大影響。

原始碼順序的最佳實踐包括：

- 將重要內容如標題、導覽或主要文字置於 HTML 結構較高位置。
- 延遲非必要腳本，如分析或第三方小工具，避免阻塞渲染。
- 採用漸進式增強，確保核心體驗在樣式和腳本載入前可正常運作。漸進式增強是基於先用 HTML 讓頁面可用的理念建構網站和應用。

以下為良好原始碼順序範例，採用上述最佳實踐：

```html
<h1>Welcome to FastSite!</h1>
<p>Critical information loads first.</p>
<script src="slow-script.js" defer></script>
```

- **關鍵渲染路徑**：瀏覽器將程式碼轉換成螢幕像素的步驟序列。
- **延遲**：請求在瀏覽器與伺服器間往返所需時間。延遲高即頁面載入慢。

降低延遲的方法包括：

- 使用 CDN（內容傳遞網路）從較近位置提供檔案。
- 啟用壓縮（如 Gzip）減少檔案大小。
- 優化影像並使用延遲載入。

```html
<img src="placeholder.jpg" data-src="real-image.jpg" loading="lazy">
```

## 改善 INP

- **定義**：INP（Interaction to Next Paint）評估頁面整體響應速度，測量使用者互動（點擊或按鍵）到瀏覽器下一次更新畫面的時間。INP 越低表示頁面越響應迅速。

改善 INP 的方法：

- 拆分長時間執行的 JavaScript 任務，減少主執行緒工作量。
- 對非關鍵腳本使用 `requestIdleCallback()`，該方法會在瀏覽器閒置時執行函式。
- 延遲或延遲載入先前提過的重資產。
- 優化事件處理器。若處理器執行頻繁或耗時，會拖慢頁面並增加 INP。解決方案是防抖（debouncing），確保函式只在使用者停止輸入短暫延遲（例如 300ms）後執行，避免不必要計算並提升效能。

## 瀏覽器中的渲染運作方式

- **渲染運作**：瀏覽器先解析 HTML 並建立 DOM，接著處理 CSS，建構 CSS 物件模型（CSSOM），這是另一棵樹狀結構，決定元素樣式。最後瀏覽器將像素繪製到螢幕，根據計算的樣式和版面配置渲染每個元素。複雜頁面可能包含多層合成，形成最終視覺輸出。

## 效能對永續性的影響

- **背景資訊**：網際網路約佔全球碳排放 2％，與航空業相當！每個位元組傳輸都需耗電，從資料中心到使用者裝置。檔案越大、腳本效率越差，耗電越多。高效能網站不僅更快，也減少不必要的處理和能源使用。

## 降低頁面載入時間的方法

- **優化媒體資產**：大型影像和影片是常見的載入瓶頸。優化這些資產可大幅加速網站，包括壓縮影像、使用現代格式如 WebP，以及延遲載入資產。
- **利用瀏覽器快取**：快取讓瀏覽器在本地儲存網站部分內容，減少回訪者載入時間。
- **縮小與壓縮檔案**：減少檔案大小可加快下載速度，包括縮小傳輸檔案大小和縮減 CSS、JavaScript 檔案。

## 改善「可用時間」

- **定義**：「可用時間」是使用者請求頁面到能有意義互動的時間間隔。改善方法包括延遲載入資產或減少阻塞渲染的資源。

## 衡量效能的關鍵指標

- **首次內容繪製（FCP）**：衡量第一個內容（文字或影像）出現在螢幕上的速度。良好 FCP 為 1.8 秒以下，差則超過 3 秒。可用 Chrome DevTools 的效能分頁檢查。
- **總阻塞時間**：顯示主執行緒被重度 JavaScript 任務阻塞的時間。若總阻塞時間（TBT）高，使用者互動會遲緩。改善方法是拆分長任務並延遲非必要腳本。
- **跳出率**：離開網站且未互動的訪客比例。跳出率高可能因頁面過慢。
- **獨立使用者數**：追蹤訪問網站的獨立訪客數。可用 Google Analytics 查看跳出率和獨立使用者數，監控並提升使用者參與度。

## 常用效能測量工具

- **Chrome DevTools**：Google Chrome 內建工具，可即時分析和偵錯效能。顯示載入時間、CPU 使用率和渲染延遲。特別適合測量首次內容繪製（FCP），幫助找出瓶頸。
- **Lighthouse**：自動化工具，檢查效能、SEO 和可及性。
- **WebPageTest**：測試網站從不同地點和裝置載入情況，提供速度指數、總阻塞時間等詳細效能指標。想了解全球真實使用者體驗，WebPageTest 是好工具。
- **PageSpeed Insights**：分析網站並提供行動和桌面端快速改進建議，指出網站瓶頸並給出優化影像、移除阻塞渲染腳本、減少伺服器回應時間等具體建議。
- **真實使用者監控（RUM）**：追蹤實際使用者行為，顯示真實訪客體驗。常見工具有 Google Analytics（追蹤頁面載入時間和跳出率）、New Relic 和 Datadog（監控即時效能問題）。想要真實使用者資料，RUM 工具不可或缺。

## 使用效能 Web API

- **定義**：效能 Web API 讓開發者直接在程式碼中追蹤網頁載入和回應效率。這些 API 可測量頁面載入時間、追蹤渲染和互動延遲，分析 JavaScript 執行時間。
- **`performance.now()`**：提供高精度時間戳（毫秒），用於測量網站不同部分載入所需時間。

```js

console.log(`Execution time: ${end - start}ms`);
```

- **效能計時 API**：提供頁面載入各階段細項，如 DNS 查詢到 `DOMContentLoaded`。

```js
const timing = performance.timing;  

const pageLoadTime = timing.loadEventEnd - timing.navigationStart;  
console.log(`Page load time: ${pageLoadTime}ms`);
```

- **`PerformanceObserver`**：監聽效能事件，如版面移位、長時間任務和使用者互動。

```js
const observer = new PerformanceObserver((list) => {  
  list.getEntries().forEach((entry) => {  
    console.log(`Long task detected: ${entry.duration}ms`);  
  });  
});  

observer.observe({ type: "longtask", buffered: true });
```

## 改善 CSS 效能技巧

- **CSS 動畫**：動畫某些 CSS 屬性（尺寸、位置、版面）會觸發「重排」（reflow），瀏覽器需重新計算元素位置和幾何形狀，接著重繪，計算量大。建議盡量減少 CSS 動畫數量，或至少提供使用者切換開關。

## 改善 JavaScript 效能技巧

- **程式碼拆分**：將 JavaScript 程式碼拆分成執行關鍵任務和非關鍵任務的模組，能優先預載關鍵模組，延遲非關鍵模組，讓頁面盡快渲染。
- **DOM 操作**：DOM 操作指用 JavaScript 動態改變頁面內容，與文件物件模型（DOM）互動。DOM 操作計算量大，減少 DOM 操作可提升效能。

## CSS 框架

- **CSS 框架**：加速工作流程，創造網站統一視覺風格，讓設計在多瀏覽器間保持一致，並使 CSS 程式碼更有組織。
- **流行 CSS 框架**：Tailwind CSS、Bootstrap、Materialize 和 Foundation。
- **潛在缺點**：
  - 框架提供的 CSS 可能與自訂 CSS 衝突。
  - 網站可能與使用相同框架的其他網站外觀相似。
  - 大型框架可能造成效能問題。

## 兩種 CSS 框架類型

- **工具優先 CSS 框架**：擁有小型且具特定用途的類別，如設定邊距、內距或背景色。可直接將這些小類別指派給 HTML 元素。Tailwind CSS 屬於工具優先 CSS 框架。

以下為使用 Tailwind CSS 樣式化按鈕範例。

```html
  Button
</button>
```

- **元件式 CSS 框架**：擁有預建且預設樣式的元件，可直接加入網站。元件可在 CSS 框架官方文件中找到，複製貼上即可。Bootstrap 屬於元件式 CSS 框架。

```html
  <ul class="list-group list-group-flush">
    <li class="list-group-item">HTML</li>

    <li class="list-group-item">JavaScript</li>
```

## CSS 預處理器

- **CSS 預處理器**：擴充標準 CSS 的工具，將擴充語法編譯成原生 CSS 檔案。對於複雜專案，能幫助撰寫更乾淨、可重複、減少重複且可擴充的 CSS。
- **功能**：變數、混入（mixins）、巢狀、選擇器繼承等。
- **流行 CSS 預處理器**：Sass、Less 和 Stylus。

- **Sass**：最流行的 CSS 預處理器之一，意指「語法超棒的樣式表」。

## Sass 支援的兩種語法

- **SCSS 語法**：SCSS（Sassy CSS）擴充 CSS 基本語法，是最廣泛使用的 Sass 語法。SCSS 檔案副檔名為 `.scss`。


- **縮排語法**：縮排語法是 Sass 的原始語法，也稱為「Sass 語法」。

```sass
$primary-color: #3498eb
```

### 混入（Mixins）

- **混入**：允許你將多個 CSS 屬性及其值群組化，並以名稱重複使用該 CSS 區塊。

以下為 SCSS 語法中定義名為 `center-flex` 的混入範例，包含三個使用 flexbox 置中元素的 CSS 屬性。

```scss
@mixin center-flex {
```

```scss
section {
```

## 手動與自動化測試

- **手動測試**：測試人員會手動檢查應用程式各部分，測試不同功能是否正常。若發現錯誤，會回報給軟體團隊修正。
- **自動化測試**：透過撰寫獨立程式自動檢查應用程式是否符合預期行為。

## 單元測試

- **單元測試**：測試每個函式確保功能正常。單元測試也可作為應用程式的文件，因為它們代表程式碼的預期行為。
- **單一職責原則**：建議每個函式保持小且只負責一件事。
- **常見 JavaScript 測試框架**：Jest、Mocha 和 Vitest。Jest 是流行的單元測試框架。

以下為使用 Jest 的單元測試範例。

首先，建立一個負責回傳新格式字串的函式：

```javascript
export function getFormattedWord(str) {
  if (!str) return '';
  return str.charAt(0).toUpperCase() + str.slice(1);
}
```

在獨立 `getFormattedWord.test.js` 檔案中，撰寫測試以驗證函式是否正常。`getFormattedWord.test.js` 檔案如下：

```javascript
import { getFormattedWord } from "./getFormattedWord.js";
import { test, expect } from "jest";

test('capitalizes the first letter of a word', () => {
  expect(getFormattedWord('hello')).toBe('Hello');
});
```

- **`expect` 函式**：`expect` 函式用於測試值。
- **匹配器**：匹配器是檢查值是否符合預期行為的函式。上述範例中匹配器為 `toBe()`。Jest 提供多種匹配器。

使用 Jest 前，需先安裝 `jest` 套件，使用 `npm i jest`。並在 `package.json` 檔案新增指令：

```js
"scripts": {
  "test": "jest"
},
```

接著執行 `npm run test` 指令執行測試。

## 軟體開發生命週期

- **軟體開發生命週期不同階段**：
  - **規劃階段**：開發團隊從利害關係人收集需求。
  - **設計階段**：軟體團隊拆解需求，決定最佳解決方案。
  - **實作階段**：軟體團隊將需求拆成可管理任務並實作。
  - **測試階段**：包含手動和自動測試新功能，有時團隊會在整個開發階段持續測試應用程式，及早發現並修正問題。
  - **部署階段**：將新變更部署到建置或測試環境。
  - **維護階段**：修正生產環境中客戶回報的問題。

- **軟體開發生命週期不同模型**：
  - **瀑布模型**：每個階段需完成後，下一階段才能開始。
  - **敏捷模型**：聚焦迭代開發，將工作拆成多個短衝刺。

## BDD 與 TDD

- **TDD**：測試驅動開發，強調先撰寫測試。先寫測試再開發功能，能在開發過程中即時回饋。
- **BDD**：行為驅動開發，將一系列測試與商業目標對齊。BDD 測試場景應使用技術與非技術人員皆能理解的語言，例如 Gherkin。
- **BDD 測試框架**：例子有 Cucumber、JBehave 和 SpecFlow。

## 單元測試中的斷言

- **斷言**：用於測試程式碼是否符合預期行為。
- **斷言函式庫**：Chai 是常用斷言函式庫。其他常見 JavaScript 斷言函式庫有 `should.js` 和 `expect.js`。

以下為使用 Chai 的斷言範例，檢查 `addThreeAndFour` 函式的返回值是否等於數字 7：

```js
assert.equal(addThreeAndFour(), 7);
```

- **最佳實踐**：無論使用哪種斷言函式庫，應撰寫清楚的斷言和失敗訊息，幫助理解測試失敗原因。

## 模擬、偽造與 Stub

- **模擬**：用假資料取代真實資料，模擬真實組件行為。例如在測試中模擬 API 回應，避免持續呼叫 API 抓取資料。
- **Stub**：Stub 是回傳預設回應或假資料的物件，用於模擬應用程式中預期行為。例如在測試中 Stub 資料庫連線行為，無需依賴真實資料庫。
- **偽造**：偽造是簡化版的真實組件，無複雜度或副作用。例如用記憶體存資料偽造資料庫，模擬資料庫操作，速度比真實資料庫快。

## 功能測試

- **功能測試**：檢查應用程式功能是否符合預期。目標是針對多種場景測試系統整體。
- **非功能測試**：聚焦效能和可靠性等方面。
- **冒煙測試**：在進行更廣泛測試前，先檢查系統是否有基本或關鍵問題。

## 端對端測試

- **端對端測試**：又稱 E2E，從使用者角度測試真實場景。確保應用程式行為正確且可預期，但設置、設計和維護耗時。
- **端對端測試框架**：Playwright 是微軟開發的流行框架。其他工具有 Cypress、Selenium 和 Puppeteer。

以下為 freeCodeCamp 程式碼庫中使用 Playwright 的 E2E 測試範例。`beforeEach` hook 會在每個測試前執行。測試檢查捐贈者在選單列有支持者連結，且頭像周圍有特殊樣式邊框：

```js
test.beforeEach(async ({ page }) => {
  execSync("node ./tools/scripts/seed/seed-demo-user --set-true isDonating");
  await page.goto("/donate");
});

...

test("The menu should have a supporters link", async ({ page }) => {
  const menuButton = page.getByTestId("header-menu-button");
  const menu = page.getByTestId("header-menu");

  await expect(menuButton).toBeVisible();
  await menuButton.click();

  await expect(menu).toBeVisible();

  await expect(page.getByRole("link", { name: "Supporters" })).toBeVisible();

test("The Avatar should have a special border for donors", async ({ page }) => {
  const container = page.locator(".avatar-container");
  await expect(container).toHaveClass("avatar-container gold-border");

```

## 可用性測試

- **可用性測試**：讓真實使用者與應用程式互動，發現設計、使用者體驗或功能問題。聚焦使用者的直覺性。
- **四種常見可用性測試類型**：
  - **探索性**：使用者互動應用程式不同功能，了解其運作。
  - **比較性**：比較應用程式與市場上類似應用的使用者體驗。
  - **評估性**：研究應用程式的直覺易用性。
  - **驗證性**：找出阻礙使用者有效使用應用程式的重大問題。
- **可用性測試工具**：例子有 Loop11、Maze、Userbrain、UserTesting 和 UXTweak。

## 相容性測試

- **相容性測試**：確保應用程式在不同運算環境中正常運作。
- **不同相容性測試類型**：
  - **向後相容性**：軟體與舊版本相容。
  - **向前相容性**：軟體與未來版本相容。
  - **硬體相容性**：軟體能在不同硬體配置正常運作。
  - **作業系統相容性**：軟體能在不同作業系統（如 macOS、Windows、Ubuntu、Fedora）運作。
  - **網路相容性**：軟體能在不同網路條件（速度、協定、安全設定等）下運作。
  - **瀏覽器相容性**：網頁應用能在不同瀏覽器（Chrome、Safari、Firefox 等）一致運作。
  - **行動裝置相容性**：確保軟體能在多種 Android 和 iOS 裝置（手機和平板）上運作。

## 效能測試

- **效能測試**：測試應用程式在不同工作負載下的速度、響應性、擴展性和穩定性。目標是解決效能瓶頸。
- **不同效能測試類型**：
  - **負載測試**：測試系統在正常和高峰負載下的行為。
  - **壓力測試**：在極端負載下測試系統反應。
  - **浸泡測試（耐力測試）**：長時間高負載測試系統。
  - **突發測試**：劇烈增加和減少負載，分析系統反應。
  - **中斷點測試（容量測試）**：逐步增加負載，直到系統開始失效或性能下降。

## 安全測試

- **安全測試**：識別漏洞和弱點。
- **安全原則**：
  - **機密性**：防止敏感資訊洩漏給非預期接收者。
  - **完整性**：防止惡意使用者修改使用者資訊。
  - **身份驗證**：驗證使用者身份，確保其有權使用系統。
  - **授權**：決定已驗證使用者可執行哪些操作或存取系統哪些部分。
  - **可用性**：確保授權使用者在需要時能取得資訊和服務。
  - **不可否認性**：確保寄件人和收件人都有送達證明和寄件人身份驗證，防止寄件人否認發送資訊。
- **常見安全威脅**：
  - **跨站腳本攻擊（XSS）**：攻擊者注入惡意腳本並在受害者瀏覽器執行。
  - **SQL 注入**：惡意使用者注入惡意程式碼到資料庫。
  - **阻斷服務攻擊（DoS）**：惡意使用者大量請求網站，導致伺服器變慢或當機，使用者無法使用網站。
- **安全測試工具類別**：
  - **靜態應用安全測試**：評估應用程式原始碼以找出安全漏洞。
  - **動態應用安全測試**：介面測試應用程式前端，找出潛在安全弱點，無法存取原始碼。
- **滲透測試（pentest）**：模擬網路攻擊以找出系統漏洞的安全測試。

## A/B 測試

- **A/B 測試**：比較兩個版本的頁面或應用程式，研究哪個版本表現較佳。又稱桶測試或分割測試。A/B 測試讓你做出更具數據依據的決策，持續改善使用者體驗。
- **A/B 測試工具**：例子有 GrowthBook 和 LaunchDarkly。

## Alpha 與 Beta 測試

初期開發和軟體測試完成後，需由測試人員和真實使用者進行測試，即 Alpha 和 Beta 測試。

- **Alpha 測試**：由特定測試人員執行，確保上市前無錯誤。屬於驗收測試，使用白箱和黑箱測試技術。
- **Beta 測試**：讓真實使用者使用應用程式並提供回饋。也是一種使用者驗收測試。
- **驗收測試**：確保軟體符合商業需求和使用者需求，方可發布。
- **黑箱測試**：只關注應用程式的預期行為。
- **白箱測試**：測試人員了解內部組件並對其進行測試。

## 回歸測試

- **回歸**：指新變更意外破壞既有功能。
- **回歸測試**：重新執行功能測試，確保應用程式仍正常運作。
- **回歸測試工具**：可用 Puppeteer、Playwright、Selenium 和 Cypress。
- **回歸測試技巧**：
  - **單元回歸測試**：每次重大變更或修正時，測試清單中的項目。
  - **部分回歸測試**：針對特定功能進行測試，確保未破壞。
  - **完整回歸測試**：測試整個程式碼庫所有功能，耗時最長且最詳細。
- **重新測試**：檢查已知問題是否已解決。與回歸測試不同，回歸測試尋找因新變更產生的未知問題。

## 什麼是 TypeScript

- **JavaScript**：動態型別語言，變數可在執行時接收任意值。動態型別的挑戰是缺乏型別安全，易引入錯誤。

例如，即使函式預期陣列，也能用數字呼叫：

```javascript
const getRandomValue = (array) => {
  return array[Math.floor(Math.random() * array.length)];
}

console.log(getRandomValue(10));
```

上述範例的 `console` 輸出為 `undefined`。

- **TypeScript**：擴充 JavaScript，加入靜態型別，幫助在執行前捕捉型別不匹配錯誤。

例如，定義 `array` 參數的型別如下：

```typescript
const getRandomValue = (array: string[]) => {
  return array[Math.floor(Math.random() * array.length)];
}
```

此型別定義告訴 TypeScript，`array` 參數必須是字串陣列。若呼叫 `getRandomValue` 並傳入數字，會出現編譯錯誤。

- **編譯器**：需先將 TypeScript 編譯成一般 JavaScript。編譯器會評估程式碼，對型別不符處丟出錯誤。

## TypeScript 中的資料型別

- **TypeScript 原始資料型別**：對 `string`、`null`、`undefined`、`number`、`boolean` 和 `bigint`，TypeScript 提供對應型別關鍵字。

```typescript
let str: string = "Naomi";
let num: number = 42;
let bool: boolean = true;
let nope: null = null;
let nada: undefined = undefined;
```

- **陣列**：可用兩種語法定義特定型別陣列。

```typescript
const arrOne: string[] = ["Naomi"];
const arrTwo: Array<string> = ["Camperchan"];
```

- **物件**：可定義物件的精確結構。

```typescript
const obj: { a: string, b: number } = { a: "Naomi", b: 42 };
```

若想定義任意鍵，但所有值必須是字串的物件，有兩種寫法：

```typescript
const objOne: Record<string, string> = {};
const objTwo: { [key: string]: string } = {};
```

- **TypeScript 其他有用型別**：
  - **`any`**：`any` 表示值可為任意型別，告訴編譯器停止關注該變數型別。
  - **`unknown`**：`unknown` 告訴 TypeScript 你關心值的型別，但實際型別未知。`unknown` 通常優於 `any`。
  - **`void`**：特殊型別，通常用於定義函式。無返回值函式使用 `void` 作為返回型別。
  - **`never`**：代表永遠不會存在的型別。

- **`type` 關鍵字**：類似 `const`，但用於宣告型別。

用於宣告自訂型別，如聯集型別或只包含特定值的型別：

```typescript
type stringOrNumber = string | number;
type bot = "camperchan" | "camperbot" | "naomi";
```

- **`interface`**：介面類似型別的類別，可實作或擴充其他介面，專指物件型別，通常優先使用，除非需要 `type` 宣告提供的特定功能。

```typescript
interface wowie {
  zowie: boolean;
  method: () => void;
}
```

- **定義返回型別**：也可定義函式的返回型別。

以下範例定義返回值為字串，若返回其他型別，TypeScript 會報編譯錯誤。

```typescript
const getRandomValue = (array: string[]): string => {
  return array[Math.floor(Math.random() * array.length)];
}
```

## 泛型

- **定義泛型型別**：可定義泛型型別並在函式中引用。可視為提供給函式的特殊參數，用於控制函式型別定義行為。

以下為定義泛型函式的範例：

```typescript
const getRandomValue = <T>(array: T[]): T => {
  return array[Math.floor(Math.random() * array.length)];
}
const val = getRandomValue([1, 2, 4])
```

`<T>` 語法告訴 TypeScript 你正在定義泛型型別 `T`。`T` 是泛型型別常用名稱，但可用任意名稱。

接著告訴 TypeScript，`array` 參數是泛型型別陣列，返回值是該型別的單一元素。

- **在函式呼叫中指定型別引數**：可在函式名稱與參數間使用尖括號傳入型別引數。

以下為傳入型別引數的範例：

```typescript
const email = document.querySelector<HTMLInputElement>("#email");
console.log(email.value);
```

此範例告訴 TypeScript 預期找到的元素是輸入元素。

## 型別縮小

- **以真值縮小**：以下範例嘗試存取 `value` 的 `email` 屬性會出現編譯錯誤，因為 `email` 可能是 `null`。

```typescript
const email = document.querySelector<HTMLInputElement>("#email");
console.log(email.value);
```

可用條件判斷確認 `email` 為真值後再存取屬性：

```typescript
const email = document.querySelector<HTMLInputElement>("#email");
if (email) {
  console.log(email.value);
}
```

真值檢查也可反向使用：

```typescript
const email = document.querySelector<HTMLInputElement>("#email");
if (!email) {
  throw new ReferenceError("Could not find email element!")
}
console.log(email.value);
```

拋出錯誤會終止程式邏輯，當執行到 `console.log()` 呼叫時，TypeScript 知道 `email` 不可能是 `null`。

- **可選串接**：可選串接 `?.` 也是型別縮小的一種，前提是若 `email` 為 `null`，則不會存取該屬性。

```typescript
const email = document.querySelector<HTMLInputElement>("#email");
console.log(email?.value);
```

- **`typeof` 運算子**：可用 `typeof` 運算子條件判斷變數型別。

```typescript
const myVal = Math.random() > 0.5 ? 222 : "222";
if (typeof myVal === "number") {
  console.log(myVal / 10);
}
```

- **`instanceof` 關鍵字**：若物件來自類別，可用 `instanceof` 關鍵字縮小型別。

```typescript
const email = document.querySelector("#email");

if (email instanceof HTMLInputElement) {
    console.log(email.value);
}
```

- **類型轉換**：當 TypeScript 無法自動判斷值型別（如下例 `request.json()` 方法結果），會出現編譯錯誤。可用類型轉換解決，但會削弱 TypeScript 捕捉錯誤能力。

```typescript
interface User {
    name: string;
    age: number;
}

const printAge = (user: User) => 
  console.log(`${user.name} is ${user.age} years old!`)

const request = await fetch("url")
const myUser = await request.json() as User;
printAge(myUser);
```

- **型別守衛**：可撰寫型別守衛函式：

```typescript
interface User {
    name: string;
    age: number;
}

const isValidUser = (user: unknown): user is User => {
  return !!user && 
    typeof user === "object" &&
    "name" in user &&
    "age" in user;
}
```

`user is User` 語法表示函式回傳布林值，為 true 時表示 `user` 值符合 `User` 介面。

## `tsconfig` 檔案

- **`tsconfig.json`**：TypeScript 編譯器設定存放於專案根目錄的 `tsconfig.json` 檔案。

```json
{
  "compilerOptions": {
    "rootDir": "./src",
    "outDir": "./prod",
    "lib": ["ES2023"],
    "target": "ES2023",
    "module": "ES2022",
    "moduleResolution": "Node",
    "esModuleInterop": true,
    "skipLibCheck": true,
    "strict": true
  },
  "exclude": ["test/"]
}
```

以下為上述範例中編譯器選項說明：

- **`compilerOptions`**：`compilerOptions` 屬性控制 TypeScript 編譯器行為。
- **`rootDir` 和 `outDir`**：`rootDir` 和 `outDir` 指定原始碼所在目錄及轉譯後 JavaScript 代碼目錄。
- **`lib`**：`lib` 屬性決定編譯器使用哪些型別定義，並允許包含特定 ES 版本、DOM 等支援。
- **`module` 和 `moduleResolution`**：`module` 和 `moduleResolution` 配合管理套件使用模組方式，CommonJS 或 ECMAScript。
- **`esModuleInterop`**：`esModuleInterop` 允許 CommonJS 與 ES 模組更順暢互通，自動為匯入建立命名空間物件。
- **`skipLibCheck`**：`skipLibCheck` 選項跳過驗證未被匯入參考的 `.d.ts` 檔案。
- **`strict`**：`strict` 標記啟用多項檢查，如確保正確處理可為 null 型別，並警告 TypeScript 回退到 `any`。
- **`exclude`**：頂層 `exclude` 屬性告訴編譯器編譯時忽略這些 TypeScript 檔案。
