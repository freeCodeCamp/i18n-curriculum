# --interactive--

## HTML 기초

- **HTML의 역할**: HTML은 웹 페이지의 내용과 구조를 나타냅니다. 
- **HTML 요소**: 요소는 HTML 문서의 기본 구성 요소입니다. 제목, 단락, 링크, 이미지 등 다양한 내용을 나타냅니다. 대부분의 HTML 요소는 여는 태그(`<elementName>`)와 닫는 태그(`</elementName>`)로 구성됩니다.

기본 구문은 다음과 같습니다:

```html
<elementName>Content goes here</elementName>
```

- **빈 요소**: 빈 요소는 내용을 가질 수 없으며 시작 태그만 있습니다. 예로는 `img`와 `meta` 요소가 있습니다.

```html
<img>
<meta>
```

일부 코드베이스에서는 빈 요소 안에 슬래시 `/`를 포함하는 경우가 흔합니다. 두 가지 모두 허용됩니다:

```html
<img>
<img/>
```

- **속성**: 속성은 HTML 요소의 여는 태그 안에 위치하는 값입니다. 속성은 요소에 대한 추가 정보를 제공하거나 요소가 어떻게 동작해야 하는지 지정합니다. 속성의 기본 구문은 다음과 같습니다:

```html
<element attribute="value"></element>
```

불리언 속성은 HTML 태그에 존재하거나 존재하지 않을 수 있는 속성입니다. 존재하면 값은 true이고, 없으면 false입니다. 불리언 속성의 예로는 `disabled`, `readonly`, `required`가 있습니다.

- **주석**: 주석은 프로그래밍에서 자신과 다른 개발자를 위해 코드에 메모를 남길 때 사용합니다. HTML에서 주석 구문은 다음과 같습니다:

```html
<!--This is an HTML comment.-->
```

## 자주 사용하는 HTML 요소

- **제목 요소**: HTML에는 여섯 개의 제목 요소가 있습니다. `h1`부터 `h6`까지의 제목 요소는 그 아래 내용의 중요도를 나타냅니다. 숫자가 낮을수록 중요도가 높아 `h2` 요소는 `h1` 요소보다 중요도가 낮습니다.

:::interactive_editor

```html
<h1>most important heading element</h1>
<h2>second most important heading element</h2>
<h3>third most important heading element</h3>
<h4>fourth most important heading element</h4>
<h5>fifth most important heading element</h5>
<h6>least important heading element</h6>
```

:::

- **단락 요소**: 웹 페이지에서 단락을 나타낼 때 사용합니다.

:::interactive_editor

```html
<p>This is a paragraph element.</p>
```

:::

- **`img` 요소**: `img` 요소는 웹 페이지에 이미지를 추가할 때 사용합니다. `src` 속성은 이미지 위치를 지정하는 데 사용됩니다. 이미지 요소에는 `alt` 속성이라는 또 다른 속성을 포함하는 것이 모범 사례입니다. `img` 요소에 `src`과 `alt` 속성을 포함한 예는 다음과 같습니다:

:::interactive_editor

```html
<img src="https://cdn.freecodecamp.org/curriculum/cat-photo-app/lasagna.jpg" alt="A slice of lasagna on a plate.">
```

:::

- **`body` 요소**: 이 요소는 HTML 문서의 내용을 나타내는 데 사용됩니다.

:::interactive_editor

```html
<body>
  <h1>CatPhotoApp</h1>
  <p>This is a paragraph element.</p>
</body>    
```

:::

- **`section` 요소**: 이 요소는 내용을 더 작은 섹션으로 나누는 데 사용됩니다.

:::interactive_editor

```html
<section>
  <h2>About Me</h2>
  <p>Hi, I am Jane Doe and I am a web developer.</p>
</section>
```

:::

- **`div` 요소**: 이 요소는 의미론적 의미가 없는 일반 HTML 요소입니다. 다른 HTML 요소를 담는 일반 컨테이너로 사용됩니다.

:::interactive_editor

```html
<div>
  <h1>I am a heading</h1>
  <p>I am a paragraph</p>
</div>
```

:::

- **앵커 (`a`) 요소**: 이 요소들은 웹 페이지에 링크를 적용하는 데 사용됩니다. `href` 속성은 사용자가 클릭할 때 링크가 이동할 위치를 지정합니다.

:::interactive_editor

```html
<a href="https://cdn.freecodecamp.org/curriculum/cat-photo-app/running-cats.jpg">cute cats</a>
```

:::

- **순서 없는 (`ul`) 및 순서 있는 (`ol`) 목록 요소**: 항목의 글머리 기호 목록을 만들려면 `ul` 요소를 사용하고 그 안에 하나 이상의 `li` 요소를 중첩하세요:

:::interactive_editor

```html
<ul>
  <li>catnip</li>
  <li>laser pointers</li>
  <li>lasagna</li>
</ul>
```

:::

순서 있는 목록을 만들려면 `ol` 요소를 사용하세요:

:::interactive_editor

```html
<ol>
  <li>flea treatment</li>
  <li>thunder</li>
  <li>other cats</li>
</ol>
```

:::

- **강조 (`em`) 요소**: 텍스트의 일부에 강조를 줄 때 사용합니다.

:::interactive_editor

```html
<p>Cats <em>love</em> lasagna.</p>  
```

:::

- **강한 중요도 (`strong`) 요소**: 이 요소는 긴급성과 심각성을 나타내기 위해 텍스트에 강한 강조를 줄 때 사용합니다.

:::interactive_editor

```html
<p>
  <strong>Important:</strong> Before proceeding, make sure to wear your safety goggles. 
</p>
```

:::

- **`figure` 및 `figcaption` 요소**: `figure` 요소는 이미지나 도표 같은 콘텐츠를 그룹화하는 데 사용됩니다. `figcaption` 요소는 `figure` 요소 안의 콘텐츠에 대한 캡션을 나타냅니다.

:::interactive_editor

```html
<figure>
  <img src="https://cdn.freecodecamp.org/curriculum/cat-photo-app/cats.jpg" alt="Two tabby kittens sleeping together on a couch.">
  <figcaption>Cats <strong>hate</strong> other cats.</figcaption>  
</figure>
```

:::

- **`main` 요소**: 이 요소는 웹 페이지의 주요 콘텐츠를 나타냅니다.
- **`footer` 요소**: 이 요소는 HTML 문서 하단에 위치하며 보통 저작권 정보와 중요한 페이지 링크를 포함합니다.

:::interactive_editor

```html
<footer>
  <p>
    No Copyright - <a href="https://www.freecodecamp.org">freeCodeCamp.org</a>
  </p>
</footer>
```

:::

## 식별자와 그룹화

- **ID**: HTML 요소의 고유 식별자입니다. ID 이름은 HTML 문서 내에서 한 번만 사용해야 합니다.

```html
<h1 id="title">Movie Review Page</h1>
```

ID 이름에는 공백을 포함할 수 없습니다. 여러 단어로 구성된 ID 이름은 단어 사이에 대시를 사용할 수 있습니다:

```html
<div id="red-box"></div>
```

- **클래스**: 클래스는 스타일링과 동작을 위해 요소를 그룹화하는 데 사용됩니다.

```html
<div class="box"></div>
```

ID와 달리 클래스 이름은 HTML 문서 전체에서 재사용할 수 있습니다. `class` 값에는 공백도 포함할 수 있습니다:

```html
<div class="box red-box"></div>
<div class="box blue-box"></div>
```

## 특수 문자와 링크

- **HTML 엔티티**: HTML 엔티티 또는 문자 참조는 HTML에서 예약된 문자를 나타내기 위해 사용하는 문자 집합입니다. 예로는 앰퍼샌드(`&amp;`) 기호와 미만 기호(`&lt;`)가 있습니다.

```html
<p>This is an &lt;img /&gt; element</p>
```

- **`link` 요소**: 이 요소는 스타일시트나 사이트 아이콘 같은 외부 리소스에 링크할 때 사용됩니다. 외부 CSS 파일에 `link` 요소를 사용하는 기본 구문은 다음과 같습니다:

```html
<link rel="stylesheet" href="./styles.css" />
```

`rel` 속성은 링크된 리소스와 HTML 문서 간의 관계를 지정하는 데 사용됩니다. `href` 속성은 외부 리소스의 URL 위치를 지정합니다.

- **`script` 요소**: 이 요소는 실행 가능한 코드를 임베드할 때 사용됩니다.

```html
<body>
  <script>
    alert("Welcome to freeCodeCamp");
  </script>
</body>
```

기술적으로 모든 JavaScript 코드를 `script` 태그 안에 작성할 수 있지만, 외부 JavaScript 파일에 링크하는 것이 모범 사례로 간주됩니다. 외부 JavaScript 파일에 링크하기 위해 `script` 요소를 사용하는 예는 다음과 같습니다:

```html
<script src="path-to-javascript-file.js"></script>
```

`src` 속성은 외부 JavaScript 파일의 위치를 지정하는 데 사용됩니다.

## 보일러플레이트와 인코딩

- **HTML 보일러플레이트**: 모든 HTML 문서에 필요한 기본 구조와 필수 요소를 포함하는 보일러플레이트입니다.

:::interactive_editor

```html
<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="utf-8" />
    <title>freeCodeCamp</title>
    <link rel="stylesheet" href="./styles.css" />
  </head>
  <body>
    <!--Headings, paragraphs, images, etc. go inside here-->
  </body>
</html>
```

:::

- **`DOCTYPE`**: 브라우저에 사용 중인 HTML 버전을 알려주는 데 사용됩니다.
- **`html` 요소**: HTML 문서의 최상위 요소 또는 루트 요소를 나타냅니다. 문서의 언어를 지정하려면 `lang` 속성을 사용하세요.
- **`head` 요소**: `head` 섹션은 브라우저와 검색 엔진에 필요한 백그라운드 정보를 포함하는 중요한 메타데이터를 담고 있습니다.
- **`meta` 요소**: 이 요소들은 사이트의 메타데이터를 나타냅니다. 문자 인코딩, Twitter 같은 사이트가 페이지 링크를 미리보기하는 방법 등 세부 정보를 포함합니다.
- **`title` 요소**: 이 요소는 브라우저 탭이나 창에 표시되는 텍스트를 설정하는 데 사용됩니다.
- **UTF-8 문자 인코딩**: UTF-8은 웹에서 널리 사용되는 표준 문자 인코딩 방식입니다. 문자 인코딩은 컴퓨터가 문자를 데이터로 저장하는 방법입니다. `charset` 속성은 `meta` 요소 안에서 문자 인코딩을 UTF-8로 설정하는 데 사용됩니다.

## SEO와 소셜 공유

- **SEO**: 검색 엔진 최적화는 웹 페이지가 검색 엔진에서 더 잘 보이고 높은 순위를 차지하도록 최적화하는 방법입니다.  
- **메타 (`description`) 요소**: 이 요소는 웹 페이지에 대한 간단한 설명을 제공하며 SEO에 영향을 줍니다.

```html
<meta
  name="description"
  content="Discover expert tips and techniques for gardening in small spaces, choosing the right plants, and maintaining a thriving garden."
/>
```

- **오픈 그래프 태그**: 오픈 그래프 프로토콜은 Facebook, LinkedIn 등 다양한 소셜 미디어 플랫폼에서 웹사이트 콘텐츠가 어떻게 표시될지 제어할 수 있게 합니다. 
이 오픈 그래프 속성을 설정하면 사용자가 콘텐츠를 클릭하고 참여하도록 유도할 수 있습니다. 
HTML `meta` 섹션 안에 `head` 요소 모음으로 이 속성들을 설정할 수 있습니다.
- **`og:title` 속성**: 소셜 미디어 게시물에 표시될 제목을 설정하는 데 사용됩니다.

```html
<meta content="freeCodeCamp.org" property="og:title" />
```

- **`og:type` 속성**: `type` 속성은 소셜 미디어에서 공유되는 콘텐츠 유형을 나타냅니다. 예로는 기사, 웹사이트, 비디오, 음악 등이 있습니다.

```html
<meta property="og:type" content="website" />
``` 

- **`og:image` 속성**: 소셜 미디어 게시물에 표시될 이미지를 설정하는 데 사용됩니다.

```html
<meta
  content="https://cdn.freecodecamp.org/platform/universal/fcc_meta_1920X1080-indigo.png"
  property="og:image"
/>
```

- **`og:url` 속성**: 소셜 미디어 게시물에서 사용자가 클릭할 URL을 설정하는 데 사용됩니다.

```html
<meta property="og:url" content="https://www.freecodecamp.org" />
```

## 미디어 요소와 최적화

- **교체된 요소**: 교체된 요소는 CSS 자체가 아니라 외부 리소스에 의해 내용이 결정되는 요소입니다. 예로 `iframe` 요소가 있습니다. `iframe`는 인라인 프레임을 의미합니다. HTML 페이지 내에 다른 HTML 콘텐츠를 직접 임베드하는 인라인 요소입니다.

```html
<iframe src="https://www.example.com" title="Example Site"></iframe>
```

사용자가 iframe을 전체 화면 모드로 표시할 수 있도록 `allowfullscreen` 속성을 포함할 수 있습니다.

```html
<iframe
  src="video-url"
  width="width-value"
  height="height-value"
  allowfullscreen
></iframe>
```

`iframe` 안에 비디오를 임베드하려면 YouTube, Vimeo 같은 인기 비디오 서비스에서 직접 복사하거나 `src` 속성으로 비디오 URL을 지정할 수 있습니다. 다음은 YouTube에서 인기 있는 freeCodeCamp 강의를 임베드한 예입니다:

:::interactive_editor

```html
<h1>A freeCodeCamp YouTube Video Embedded with the iframe Element</h1>

<iframe
  width="560"
  height="315"
  src="https://www.youtube.com/embed/PkZNo7MFNFg?si=-UBVIUNM3csdeiWF"
  title="YouTube video player"
  allow="accelerometer; autoplay; clipboard-write; encrypted-media; gyroscope; picture-in-picture; web-share"
  referrerpolicy="strict-origin-when-cross-origin"
  allowfullscreen
></iframe>
```

:::

다른 교체된 요소로는 `video`, `embed`가 있습니다. 특정 상황에서 교체된 요소처럼 동작하는 요소도 있습니다. 다음은 `input` 요소에 `type` 속성이 `image`로 설정된 예입니다:

```html
<input type="image" alt="Descriptive text goes here" src="example-img-url">
```

- **미디어 최적화**: 웹 페이지에서 이미지 같은 미디어를 사용할 때 고려해야 할 세 가지 도구는 크기, 형식, 압축입니다. 압축 알고리즘은 파일이나 데이터 크기를 줄이는 데 사용됩니다.
- **이미지 형식**: 가장 일반적인 파일 형식 두 가지는 PNG와 JPG지만, 더 이상 이미지 제공에 가장 이상적인 형식은 아닙니다. 구형 브라우저 지원이 필요하지 않다면 WEBP나 AVIF 같은 더 최적화된 형식을 고려하세요.
- **이미지 라이선스**: 퍼블릭 도메인에 속한 이미지는 저작권이 없으며 제한 없이 자유롭게 사용할 수 있습니다. Creative Commons 0 라이선스 하에 명시된 이미지는 퍼블릭 도메인으로 간주됩니다. 일부 이미지는 Creative Commons 라이선스나 freeCodeCamp가 사용하는 BSD 라이선스 같은 관대한 라이선스 하에 배포될 수 있습니다.
- **SVG**: 스케일러블 벡터 그래픽은 경로와 방정식을 기반으로 점, 선, 곡선을 그립니다. 즉, SVG 같은 벡터 그래픽은 크기를 아무리 키워도 품질에 영향을 주지 않습니다.

## 멀티미디어 통합

- **`audio` 및 `video` 요소**: `audio`과 `video` 요소는 HTML 문서에 오디오와 비디오 콘텐츠를 추가할 수 있게 합니다. `audio` 요소는 mp3, wav, ogg 같은 인기 오디오 형식을 지원합니다. `video` 요소는 mp4, ogg, webm 형식을 지원합니다.

```html
<audio src="CrystalizeThatInnerChild.mp3"></audio>
```

페이지에 오디오 플레이어를 표시하려면 `audio` 요소에 `controls` 속성을 추가하세요:

:::interactive_editor

```html
<audio src="CrystalizeThatInnerChild.mp3" controls></audio>
```

:::

`controls` 속성은 사용자가 볼륨 조절, 일시 정지, 재생 등을 관리할 수 있게 합니다. `controls` 속성은 내장 재생 컨트롤을 활성화하는 불리언 속성입니다. 생략하면 컨트롤이 표시되지 않습니다.

- **`loop` 속성**: `loop` 속성은 오디오를 계속 반복 재생하게 하는 불리언 속성입니다.

:::interactive_editor

```html
<audio
  src="https://cdn.freecodecamp.org/curriculum/js-music-player/can't-stay-down.mp3"
  loop
  controls
></audio>
```

:::

- **`muted` 속성**: `audio` 요소에 존재하면 `muted` 불리언 속성이 오디오를 음소거 상태로 시작하게 합니다.

:::interactive_editor

```html
<audio
  src="https://cdn.freecodecamp.org/curriculum/js-music-player/can't-stay-down.mp3"
  loop
  controls
  muted
></audio>
```

:::

- **`source` 요소**: 오디오 파일 형식별로 브라우저 지원이 다릅니다. 이를 위해 `source` 요소 안에 여러 `audio` 요소를 넣으면 브라우저가 이해하는 첫 번째 소스를 선택합니다. 다음은 `source` 요소에 여러 `audio` 요소를 사용하는 예입니다:

```html
<audio controls>
  <source src="audio.ogg" type="audio/ogg" />
  <source src="audio.wav" type="audio/wav" />
  <source src="audio.mp3" type="audio/mpeg" />
</audio>
```

지금까지 배운 모든 속성은 `video` 요소에서도 지원됩니다. 다음은 `video` 요소에 `loop`, `controls`, `muted` 속성을 사용하는 예입니다:

:::interactive_editor

```html
<video
  src="https://archive.org/download/BigBuckBunny_124/Content/big_buck_bunny_720p_surround.mp4"
  loop
  controls
  muted
></video>
```

:::

- **`poster` 속성**: 비디오가 다운로드되는 동안 이미지를 표시하려면 `poster` 속성을 사용할 수 있습니다. 이 속성은 `audio` 요소에는 없고 `video` 요소에만 고유합니다.

:::interactive_editor

```html
<video
  src="https://archive.org/download/BigBuckBunny_124/Content/big_buck_bunny_720p_surround.mp4"
  loop
  controls
  muted
  poster="https://peach.blender.org/wp-content/uploads/title_anouncement.jpg?x11217"
  width="620"
></video>
```

:::

## target 속성 유형

- **`target` 속성**: 이 속성은 앵커 요소의 URL을 어디에서 열지 브라우저에 알려줍니다. 이 속성에 중요한 네 가지 값이 있습니다: `_self`, `_blank`, `_parent`, `_top`. 다섯 번째 값인 `_unfencedTop`은 현재 실험 중인 `FencedFrame` API에 사용됩니다. 아직 사용할 이유는 거의 없을 것입니다.
- **`_self` 값**: `target` 속성의 기본값입니다. 현재 브라우징 컨텍스트에서 링크를 엽니다. 대부분 현재 탭이나 창이 됩니다.

```html
<a href="https://freecodecamp.org" target="_self">Visit freeCodeCamp</a>
```

- **`_blank` 값**: 새 브라우징 컨텍스트에서 링크를 엽니다. 보통 새 탭에서 열리지만, 일부 사용자는 새 창에서 열리도록 브라우저를 설정할 수 있습니다.

```html
<a href="https://freecodecamp.org" target="_blank">Visit freeCodeCamp</a>
```

- **`_parent` 값**: 현재 컨텍스트의 부모에서 링크를 엽니다. 예를 들어, 웹사이트에 iframe이 있고 그 iframe에서 `_parent` 값을 사용하면 웹사이트 탭/창에서 열리고 임베드된 프레임에서는 열리지 않습니다.

```html
<a href="https://freecodecamp.org" target="_parent">Visit freeCodeCamp</a>
```

- **`_top` 값**: 최상위 브라우징 컨텍스트에서 링크를 엽니다 — 즉, "부모의 부모"라고 생각하세요. `_parent`과 비슷하지만, 중첩된 임베드 프레임에서도 항상 전체 브라우저 탭/창에서 링크가 열립니다.

```html
<a href="https://freecodecamp.org" target="_top">Visit freeCodeCamp</a>
```

## 절대 경로와 상대 경로

- **경로 정의**: 경로는 파일 시스템에서 파일이나 디렉터리 위치를 지정하는 문자열입니다. 웹 개발에서는 이미지, 스타일시트, 스크립트, 다른 웹 페이지 같은 리소스에 링크할 때 경로를 사용합니다.
- **경로 구문**: 알아야 할 세 가지 주요 구문이 있습니다. 첫째는 슬래시로, 운영체제에 따라 역슬래시(`\`) 또는 정방향 슬래시(`/`)일 수 있습니다. 둘째는 단일 점(`.`)입니다. 마지막으로 이중 점(`..`)이 있습니다. 슬래시는 "경로 구분자"로 폴더나 파일 이름 사이의 구분을 나타냅니다. 단일 점은 현재 디렉터리를, 두 점은 상위 디렉터리를 가리킵니다.

```md
public/index.html
./favicon.ico
../src/index.css
```

- **절대 경로**: 절대 경로는 리소스에 대한 완전한 링크입니다. 루트 디렉터리부터 시작해 모든 하위 디렉터리, 마지막으로 파일 이름과 확장자를 포함합니다. "루트 디렉터리"는 계층 구조에서 최상위 디렉터리 또는 폴더를 의미합니다. 절대 경로에는 프로토콜도 포함되며, `http`, `https`, `file` 등이 있습니다. 리소스가 웹에 있다면 도메인 이름도 포함됩니다. 다음은 freeCodeCamp 로고에 링크하는 절대 경로 예입니다:

:::interactive_editor

```html
<a href="https://design-style-guide.freecodecamp.org/img/fcc_secondary_small.svg">
  View fCC Logo
</a>
```

:::

- **상대 경로**: 상대 경로는 현재 파일의 디렉터리를 기준으로 파일 위치를 지정합니다. 프로토콜이나 도메인 이름을 포함하지 않아 같은 웹사이트 내 내부 링크에 더 짧고 유연합니다. 다음은 같은 폴더에 있는 `about.html` 페이지에서 `contact.html` 페이지로 링크하는 예입니다:

```html
<p>
  Read more on the
  <a href="about.html">About Page</a>
</p>
```

## 링크 상태

- **`:link`**: 기본 상태입니다. 사용자가 아직 방문하거나 클릭하거나 상호작용하지 않은 링크를 나타냅니다. 이 상태는 페이지의 모든 링크에 기본 스타일을 제공합니다. 다른 상태는 이 위에 쌓입니다.
- **`:visited`**: 사용자가 이미 링크된 페이지를 방문했을 때 적용됩니다. 기본적으로 링크를 보라색으로 바꾸지만 CSS를 활용해 다른 시각적 표시를 할 수 있습니다.
- **`:hover`**: 사용자가 링크 위에 마우스 커서를 올렸을 때 적용됩니다. 사용자가 실제로 클릭하려는 의도를 확실히 하기 위해 링크에 추가 주의를 주는 데 유용합니다.
- **`:focus`**: 링크에 포커스가 있을 때 적용됩니다.
- **`:active`**: 사용자가 링크를 활성화할 때 적용됩니다. 보통 기본 마우스 버튼으로 클릭하는 것을 의미합니다.
