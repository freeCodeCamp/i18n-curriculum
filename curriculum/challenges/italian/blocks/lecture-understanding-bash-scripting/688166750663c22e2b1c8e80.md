# --description--

Nelle lezioni precedenti, hai imparato cos’è Bash e alcuni dei comandi comuni che potresti usare. Ma Bash supporta un linguaggio di scripting completo, che puoi usare per eseguire attività automatizzate più complesse.

Lo scripting Bash consiste nello scrivere una sequenza di comandi Bash in un file, che poi puoi eseguire con Bash per far partire il contenuto del file.

Vediamo come potrebbe apparire uno script bash:

```bash
#!/bin/bash

servers=("prod" "dev")

for server in "${servers[@]}"
do
  echo "Pulling $server"
  rsync --archive --verbose $server:/etc/nginx/conf.d/server.conf configs/$server.conf
done
```

Questo script preleva i file di configurazione di NGINX da una lista di server remoti, cosa utile per fare il backup di quelle configurazioni in un repository locale. NGINX è un server web molto diffuso e imparerai di più su come lavorare con i server in un modulo futuro.

La prima riga si chiama "shebang" (`#!`) e indica al sistema quale interprete deve essere usato per eseguire lo script. I valori comuni qui sono `/usr/bin/bash`, `/bin/bash` o `/bin/sh`.

La prima riga, `servers=("prod" "dev")`, crea una lista di stringhe che rappresentano i nomi dei server. `"prod"` è spesso usato per indicare il sito di produzione, mentre `"dev"` indica il sito di sviluppo, che di solito serve per testare nuove funzionalità e modifiche prima che vengano distribuite nell’ambiente di produzione live.

Poi, la lista viene iterata con un loop `for` — la sintassi `servers[@]` espande la lista in singoli elementi. Ogni elemento viene assegnato a server a ogni iterazione.

L’istruzione `do` indica l’inizio del blocco logico del loop, cioè il codice che deve essere eseguito a ogni iterazione.

`echo "Pulling $server"` fa l’interpolazione del valore corrente della variabile server, che in questo caso sarebbe prod o dev, e lo stampa nel terminale.

Poi `rsync` viene usato per prelevare il file vero e proprio dal server e salvarlo localmente, usando un’altra interpolazione per gestire quel comando.

`done` indica la fine del blocco logico del loop.

Ci sono diversi vantaggi importanti nel comprendere e sfruttare lo scripting Bash.

Il primo è che quasi ogni ambiente Unix ha Bash già disponibile, il che significa che puoi eseguire il tuo script così com’è senza dover configurare runtime di linguaggi come Node.js o Python.

Il secondo è che hai accesso a tutte le applicazioni binarie disponibili sul sistema con uno sforzo minimo. Se il tuo sistema ha installato doctl, il tuo script può eseguire quel binario. Un binario è un programma compilato che il computer esegue. Quando hai uno script bash, puoi eseguire il binario che avvierà il programma.

Infine, la sintassi che usi per il tuo script è eseguibile direttamente anche nel terminale. Questo significa che testare una piccola parte del tuo script diventa molto meno complicato — puoi copiare e incollare quella parte direttamente nel terminale e vedere il risultato!

Capire Bash non solo migliorerà le tue capacità di scripting, ma ti renderà un mago della riga di comando!
