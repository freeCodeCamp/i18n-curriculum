# --interactive--

在 JavaScript 中，資料型別是你儲存的值的種類，例如數字或一段文字。

變數是一個具名的容器，用來儲存特定資料型別的值，讓你能在整個程式碼中取用並操作它。 

你可能還記得在數學類別中這樣使用變數：

```md
x = 2
y = 4

x + y
```

你能夠創建像 `x` 和 `y` 這樣的變數，然後在整個程式中取用它們並執行像加法這樣的數學操作。這在程式設計中是一個類似的概念。你可以創建自己的變數名稱並為它們指定值。這些值會是不同的資料型別。

資料型別幫助程式了解它正在處理的資料種類，無論是數字、文字或其他類型。

JavaScript 有幾種基本的資料型別，你會在你的程式中使用。我們會在未來的課程中更詳細地探討每種資料型別。現在，這裡是 JavaScript 中不同資料型別的簡要介紹。

我們將首先檢視的資料型別是 `Number` 型別。

`Number` 表述整數和浮點數值。整數的範例包括 `7`、`19` 和 `90`。 

**注意**：`console.log()` 是一個將資訊輸出到主控台的函式（程式），主控台是你的網頁瀏覽器中用於偵錯程式碼的零件。你將在未來的課程中學習更多關於 `console.log()` 的內容。此外，`//` 符號用於在你的程式碼中添加註解。註解是給你自己或其他程式設計師的備註，當程式執行時會被忽略。

啟用互動式編輯器並嘗試更改一些整數，以查看它如何在主控台中更新。

:::interactive_editor

```js
// Examples of integers
console.log(3);
console.log(5);
console.log(-67);
```

:::

浮點數是帶有小數點的數字。浮點數的範例包括 `3.14` 和 `5.2`。 

:::interactive_editor

```js
// Examples of floating point numbers
console.log(3.14);
console.log(7.2);
console.log(-14.5);
```

:::

下一個資料型別是 `String`。

`String` 是一連串被引號包圍的字元或文字。以下是一個使用雙引號的範例字串： 

:::interactive_editor

```js
console.log("I love to code!");
```

:::

這裡是一個使用單引號的範例：

:::interactive_editor

```js
console.log('I love to code!');
```

:::

你經常會使用字串來表述名稱、標籤、警示訊息等。

JavaScript 中使用的另一種資料型別是 `Boolean` 型別。

`Boolean` 表述兩個值之一：`true` 或 `false`。例如，程式可能會檢查使用者是否已登入（`true`）或未登入（`false`），並根據此更改頁面。如果使用者已登入，你可能想要顯示他們的儀表板頁面。否則，你會想要顯示登入頁面。

JavaScript 中接下來使用的兩個資料型別是 `undefined` 和 `null`。

`undefined` 意味著變數已被宣告但尚未被賦值。你將在下一課中學到更多關於這個的內容。

`null` 意味著該變數已被刻意設為「無」，且不包含任何值。我們將在未來的課程中進一步探討這是如何運作的。

最後三種資料型別本質上較為複雜。這些是物件、符號和 `BigInt`。 

`Object` 是一個鍵值對組的群集。 

```js
{
  name: "Alice",
  age: 30
};
```

物件非常適合將相關資訊群組在一起。你將在未來的模組中學習更多如何使用物件。

`Symbol` 是 JavaScript 中一種特殊的值型別，它永遠是唯一且不可更改的。它通常用於為屬性創建唯一的標籤或識別符。

```js
Symbol('mySymbol');
```

`BigInt` 用於超出 `Number` 型別限制的非常大數字：

```js
1234567890123456789012345678901234567890n;
```

在此範例中，我們透過在非常大的數字末尾添加 `n` 來創建 `BigInt`。

`Symbol` 和 `BigInt` 是兩種較少使用的型別，但你仍然需要了解它們。

了解這些資料型別有助於你在程式中控制代碼並處理各種資料，因為每種型別都有其自身的特性和行為。
