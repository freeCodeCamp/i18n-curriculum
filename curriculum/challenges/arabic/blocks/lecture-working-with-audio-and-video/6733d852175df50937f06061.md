# --description--

بدلًا من تشغيل الصوت والفيديو، قد ترغب أحيانًا في التقاط الصوت أو الفيديو. تتيح لك واجهة برمجة تطبيقات التقاط الوسائط والتدفقات، أو واجهة برمجة تطبيقات MediaStream، القيام بذلك.

لكي تستخدم الواجهة، تحتاج إلى إنشاء كائن `MediaStream`. يمكنك فعل ذلك باستخدام دالة البناء، لكنها لن تكون مرتبطة بأجهزة المستخدم. بدلًا من ذلك، تحتوي الخاصية `mediaDevices` في الكائن العالمي `navigator` على طريقة `getUserMedia()` لاستخدامها.

تقبل هذه الطريقة كائنًا واحدًا `constraints` يحدد نوع الوسائط التي تريد استقبالها. يحتوي هذا الكائن على خاصيتي `audio` و`video`، تمثلان تدفقات الصوت والفيديو. يمكن أن تكون هذه الخصائص `false` إذا لم ترغب في استقبال ذلك النوع من التدفق، أو `true` إذا أردت ذلك، أو كائنات تحدد قيودًا إضافية.

على سبيل المثال، يمكنك طلب دقة معينة لإخراج الفيديو:

```js
window.navigator.mediaDevices.getUserMedia({
  audio: true,
  video: {
    width: {
      min: 1280,
      ideal: 1920,
      max: 3840
    },
    height: {
      min: 720,
      ideal: 1080,
      max: 2160
    }
  }
});
```

يحدد كائن القيود هذا الحد الأدنى والحد الأقصى لدقة تدفق الفيديو. تحدد الخاصية `ideal` الدقة التي تفضلها أكثر – وسيقدم التدفق الدقة الأقرب إلى المثالية التي حددتها.

بمجرد إنشاء MediaStream الخاص بك (بافتراض موافقة المستخدم على طلب الوصول التلقائي إلى أجهزته)، يمكنك استخدام بيانات التدفق كما تحتاج.

لاحظ أن `getUserMedia()` تُرجع وعدًا، مما يعني أنك ستحتاج إما إلى دالة استدعاء راجع لاستهلاك التدفق، أو استخدام صيغة async/await. ستتعلم المزيد عن الوعود والبرمجة غير المتزامنة في الدروس القادمة.

إليك مثالًا أساسيًا يعرض بث كاميرا الويب الخاصة بالمستخدم على الصفحة.

```js
const video = document.querySelector("video");
const stream =
  await window.navigator.mediaDevices.getUserMedia({ video: true });
video.srcObject = stream;
await video.play();
```

يجدر بالذكر أن هذه الواجهة لا توفر وصولًا لالتقاط الشاشة. ستتعلم عن تلك الواجهة في الدرس التالي.
