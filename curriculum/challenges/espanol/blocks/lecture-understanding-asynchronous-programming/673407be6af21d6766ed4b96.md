# --description--

En la lección anterior, vimos qué es la API Fetch y cómo usarla. En esta lección, hablaremos de los métodos HTTP `GET`, `POST`, `PUT` y `DELETE` de la API Fetch.

Comencemos con el método HTTP más común, que es el método `GET`. Este se usa para obtener datos de un servidor. Cuando usas `fetch()` sin especificar un método, por defecto es `GET`.

```js
fetch('https://api.example.com/data')
```

En este código, estamos haciendo una solicitud `GET` a `https://api.example.com/data`. Ahora, ten en cuenta que no puedes usar estos datos directamente; tienes que convertir la respuesta a formato JSON. Solo entonces podrás usarla donde quieras en tu proyecto. Aquí tienes un ejemplo de cómo hacerlo:

```js
fetch('https://api.example.com/data')
  .then(response => response.json())
  .then(data => console.log(data))
```

En este código, la respuesta que viene de la API Fetch es una promesa, y estamos usando un manejador `.then` para convertir la respuesta a formato JSON. En una lección anterior, aprendiste que una `Promise` es un objeto que representa la eventual finalización (o fallo) de un proceso asíncrono y su valor.

El valor de una promesa no se conoce cuando se crea la promesa. Solo se conoce cuando el proceso asíncrono termina. Cuando encadenamos los dos manejadores `.then` a la llamada fetch, esto se llama encadenamiento de promesas, que se enseñará en la siguiente lección.

Hasta ahora hemos estado obteniendo recursos de un servidor. Pero, ¿sabías que también podemos enviar datos al servidor? El método `POST` se usa para enviar datos a un servidor para crear un recurso. Aquí tienes un ejemplo de un método `POST` que se usa para crear un recurso en el servidor:

```js
fetch('https://api.example.com/users', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({
    name: 'John Doe',
    email: 'john@example.com'
  })
})
```

En este ejemplo, estamos enviando una solicitud `POST` para crear un nuevo usuario. Especificamos el método como `POST`, configuramos las cabeceras apropiadas e incluimos un `body` con los datos que queremos enviar. El cuerpo debe ser una cadena, así que usamos `JSON.stringify()` para convertir nuestro objeto a una cadena JSON.

El método `PUT` se usa para actualizar recursos existentes en un servidor. Aquí tienes un ejemplo:

```js
fetch('https://api.example.com/users/45', {
  method: 'PUT',
  headers: {
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({
    name: 'John Smith',
    email: 'john@example.com'
  })
})
```

En este ejemplo, observa bien la URL, puedes ver un `45` al final. Esto se usa típicamente como un ID único para identificar los datos que queremos actualizar. Usamos el método `PUT` en el código y también especificamos los datos como el `body` que se usará para actualizar los datos identificados.

El método `DELETE` se usa para eliminar un recurso del servidor. Aquí tienes un ejemplo:

```js
fetch('https://api.example.com/users/45', {
  method: 'DELETE',
})
```

En este código, incluimos el método `DELETE` y un ID al final de la URL para identificar los datos que deben eliminarse.
