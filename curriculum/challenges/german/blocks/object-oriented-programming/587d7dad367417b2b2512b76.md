# --description--

Die letzte Aufgabe hat eine Methode zum `duck`-Objekt eingeführt. Sie verwendete `duck.name`-Punktnotation, um innerhalb der return-Anweisung auf den Wert der `name`-Eigenschaft zuzugreifen:

```js
sayName: function() {return "The name of this duck is " + duck.name + ".";}
```

Obwohl dies eine gültige Möglichkeit ist, auf die Eigenschaft des Objekts zuzugreifen, gibt es hier eine Falle. Wenn sich der Variablenname ändert, müsste jeder Code, der auf den ursprünglichen Namen verweist, ebenfalls aktualisiert werden. Bei einer kurzen Objektdefinition ist das kein Problem, aber wenn ein Objekt viele Verweise auf seine Eigenschaften hat, steigt die Fehleranfälligkeit.

Eine Möglichkeit, diese Probleme zu vermeiden, ist das `this`-Schlüsselwort:

```js
let duck = {
  name: "Aflac",
  numLegs: 2,
  sayName: function() {return "The name of this duck is " + this.name + ".";}
};
```

`this` ist ein tiefgehendes Thema, und das obige Beispiel ist nur eine Möglichkeit, es zu verwenden. Im aktuellen Kontext bezieht sich `this` auf das Objekt, mit dem die Methode verknüpft ist: `duck`. Wenn der Name des Objekts in `mallard` geändert wird, ist es nicht notwendig, alle Verweise auf `duck` im Code zu finden. Das macht den Code wiederverwendbar und leichter lesbar.

# --instructions--

Ändern Sie die `dog.sayLegs`-Methode so, dass alle Verweise auf `dog` entfernt werden. Verwenden Sie das `duck`-Beispiel als Orientierung.

```js
assert(dog.sayLegs() === 'This dog has 4 legs.');
```
