# --description--

Nelle lezioni precedenti, abbiamo imparato a creare piccoli componenti in React come questo:

```jsx
function Greeting() {
  const developerName = "Jessica";
  return <h1>Hi {developerName}!</h1>;
}
```

Possiamo scegliere di annidare questo componente all'interno di un altro componente genitore o del componente root così:

```jsx
function App() {
  return <Greeting />;
}

function Greeting() {
  const developerName = "Jessica";
  return <h1>Hi {developerName}!</h1>;
}
```

Anche se questo codice funziona e mostra il risultato di `Hi Jessica!` sullo schermo, non è un componente molto flessibile.

E se volessimo mostrare un nome diverso come Naomi, Tom o Oliver? Qui entrano in gioco le props di React. Le props, abbreviazione di proprietà, sono il modo in cui i componenti genitori passano dati ai componenti figli. Le props possono essere di qualsiasi tipo: stringhe, numeri, booleani, oggetti o array.

Aggiorniamo il nostro esempio precedente per accettare ora una prop `name`:

```jsx
function App() {
  return <Greeting name="Jessica" />;
}

export default App;

function Greeting(props) {
  console.log(props);
  return <h1>Hi {props.name}!</h1>;
}
```

Per il componente figlio chiamato `Greeting` ora usiamo `props.name` invece di scrivere direttamente il nome `"Jessica"`. Inoltre, stiamo registrando `props` nella console che appare come un oggetto.

Poi, all'interno del componente genitore `App`, stiamo passando il valore alla prop `name` così può essere trasmesso al figlio. Il risultato sullo schermo sarà lo stesso di prima, ma ora abbiamo creato un componente più flessibile.

Ora possiamo riutilizzare il componente figlio più volte e passare nomi diversi ogni volta:

```jsx
function App() {
  return (
    <>
      <Greeting name="Naomi" />
      <Greeting name="Tom" />
      <Greeting name="Jessica" />
      <Greeting name="Oliver" />
    </>
  );
}
```

Puoi anche scegliere di usare la destrutturazione dell'oggetto nelle props per renderlo più leggibile. Ecco come potresti riscrivere il componente `Greeting`:

```jsx
function Greeting({ name }) {
  return <h1>Hi {name}!</h1>;
}
```

Questo codice ottiene lo stesso risultato ma rende più chiaro quali props il componente si aspetta di ricevere.

React fornisce anche una prop speciale chiamata `children`. Qualsiasi JSX che metti tra il tag di apertura e il tag di chiusura di un componente viene passato al componente come `children`.

Per esempio:

```jsx
function Card({ children }) {
  return <div className="card">{children}</div>;
}

function App() {
  return (
    <Card>
      <h2>Hello</h2>
    </Card>
  );
}
```

In questo caso, il JSX dentro `<Card>...</Card>` diventa la prop `children`. L'output renderizzato nel DOM sarà:

```html
<div class="card">
  <h2>Hello</h2>
</div>
```

Questo pattern è spesso usato per la composizione dei componenti, dove un componente avvolge altri elementi dell'interfaccia utente.

A volte, puoi avere molte proprietà da passare come props. Invece di passarle una per una, puoi usare l'operatore di espansione (`...`), dopo averle convertite in un oggetto.

Ecco un esempio di un nuovo componente figlio chiamato `DeveloperCard`:

```jsx
function DeveloperCard({ name, age, country }) {
  return (
    <div className="developer-card">
      <h1>Developer: {name}</h1>
      <p>Age: {age}</p>
      <p>Country: {country}</p>
    </div>
  );
}
```

Questo componente `DeveloperCard` accetta tre props: `name`, `age` e `country`.

Nel componente genitore `App`, possiamo usare la sintassi di espansione per passare tutte le proprietà di un oggetto come props individuali al componente figlio:

```jsx
function App() {
  const developerObj = {
    name: "Alice",
    age: 30,
    country: "USA",
  };

  return (
    <div className="App">
      <DeveloperCard {...developerObj} />
    </div>
  );
}
```

Questo è particolarmente utile quando si lavora con array di oggetti e si passano più insiemi di proprietà ai componenti figli. Per esempio, potresti avere una lista di sviluppatori dove ogni oggetto nell'array ha la stessa struttura ma rappresenta una persona diversa.

Imparerai di più su come renderizzare liste in array nelle lezioni future.

Usare le props in React rende i tuoi componenti più flessibili e riutilizzabili, permettendoti di costruire interfacce utente più complesse. Tuttavia, è importante notare che le props sono immutabili, cioè non possono essere modificate una volta passate a un componente. Se hai bisogno di gestire input dell'utente e modificare dati, dovresti usare lo stato. Imparerai di più sulla gestione dello stato nelle lezioni future.
