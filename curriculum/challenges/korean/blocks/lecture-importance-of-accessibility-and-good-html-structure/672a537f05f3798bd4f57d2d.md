# --description--

접근성은 디지털 콘텐츠에서 매우 중요하지만 종종 간과되는 부분입니다. 디지털 콘텐츠를 접근 가능하게 만들 때는 접근성 기준을 충족하는지 확인하는 것이 중요합니다.

접근성 감사 도구는 자동화된 테스트를 통해 쉽게 발견할 수 있는 접근성 문제를 보고하여 디지털 콘텐츠의 접근성을 개선하는 데 도움을 주는 애플리케이션입니다. 이 콘텐츠에는 웹사이트, 웹 애플리케이션, 모바일 앱이 포함됩니다.

자동화된 접근성 도구가 접근성 향상에 역할을 하지만, 일반적으로 모든 가능한 접근성 문제의 약 3분의 1만 발견할 수 있다는 점을 유념하세요. 따라서 콘텐츠의 접근성을 평가할 때 이 도구들에만 전적으로 의존해서는 안 됩니다. 장애가 있는 사람이 직접 수행하는 수동 테스트가 항상 필요하며, 이를 통해 콘텐츠가 최대한 접근 가능하도록 보장할 수 있습니다.

디지털 콘텐츠의 접근성을 개선하는 데 도움이 되는 몇 가지 무료 도구를 살펴보겠습니다.

Google Lighthouse는 Chrome 개발자 도구 내에서 직접 또는 온라인으로 사용할 수 있는 인기 있는 웹 지표 검사 도구입니다. 이를 통해 라이브 웹사이트뿐 아니라 로컬에서 개발 중인 웹사이트도 검사할 수 있습니다.

검사할 수 있는 지표에는 접근성, SEO, 모범 사례, 성능이 포함됩니다.

Lighthouse를 사용하려면 `F12`를 눌러 개발자 도구를 열고 Lighthouse 탭으로 전환하세요.
검사할 지표를 선택하고 테스트할 기기를 고른 다음 "Analyze page load" 버튼을 클릭하세요.

검사가 완료되면 접근성 점수와 함께 수정이 필요한 문제 목록이 표시됩니다.

더 신뢰할 수 있는 지표를 원한다면 웹 버전을 사용하는 것을 고려하세요. 단점은 로컬 웹사이트 테스트를 지원하지 않는다는 점입니다. 웹 버전은 `pagespeed.web.dev`에서 접속할 수 있습니다.

WAVE는 Chrome 확장 프로그램이나 웹에서 사용할 수 있는 또 다른 신뢰할 수 있는 접근성 검사 도구입니다. 웹사이트 URL만 입력하면 포괄적인 접근성 보고서가 생성됩니다. 이 보고서에는 구현된 접근성 기능, ARIA, 대비 등이 포함됩니다.

IBM Equal Accessibility Checker는 디지털 콘텐츠 접근성 향상을 위한 또 다른 강력한 도구입니다. 이를 사용하면 웹사이트의 접근성 문제를 스캔하고 상세한 보고서를 생성할 수 있습니다.

Chrome 확장 프로그램이나 Firefox 애드온으로 사용할 수 있습니다.

IBM Accessibility Checker를 Chrome 확장 프로그램으로 사용하려면 Chrome 웹 스토어에서 다운로드하세요. `F12`를 눌러 개발자 도구를 열고 Elements 패널에 있는 "Accessibility Checker" 탭을 선택하세요. 스캔 버튼을 클릭하면 검사가 시작되고 보고서가 생성됩니다. "Export XLS" 버튼을 클릭하면 보고서를 스프레드시트와 HTML 파일로 내보낼 수 있습니다.

이 자동화 도구들이 콘텐츠를 더 접근 가능하게 만드는 데 도움을 주지만, 이들 중 어느 하나에서 완벽한 점수를 받았다고 해서 콘텐츠가 완전히 접근 가능하다는 의미는 아닙니다. 이 도구들이 검사하는 문제 범위가 제한적이며, 모두가 더 접근 가능한 경험을 할 수 있도록 수동 테스트가 항상 필요합니다.
