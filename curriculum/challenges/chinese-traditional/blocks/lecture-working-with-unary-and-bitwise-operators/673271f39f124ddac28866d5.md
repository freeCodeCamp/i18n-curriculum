# --interactive--

JavaScript 中的以 bit 為單元逐一運算子是作用於數字的二進位表述的特殊運算子。要理解以 bit 為單元逐一運算子，我們首先需要掌握位元和二進位數字的概念。在計算中，位元是最基本的資訊單位。它只能有兩個值：`0` 或 `1`。二進位是一種只使用這兩個數字來表述所有數字的數字系統。

例如，十進位數字 `10` 的二進位表現是 `1010`。在這個系統中，每個數字表述一個 `2` 的冪次方，從最右邊的數字開始，向左遞增。

現在，讓我們深入探討以 bit 為單元逐一運算子。這些運算子會對數字的二進位表述進行操作。JavaScript 提供了幾種以 bit 為單元逐一運算子，包括 AND（`&`）、OR（`|`）、XOR（`^`）、NOT（`~`）、左移位（`<<`）和右移位（`>>`）。

以 bit 為單元逐一 AND (`&`) 運算子會在兩個運算元的對應位元皆為 `1` 時，在該位元位置傳回 `1`。以下是一個範例：

:::interactive_editor

```js
let a = 5;  // Binary: 101
let b = 3;  // Binary: 011
console.log(a & b);  // 1 (Binary: 001)
```

:::

在此範例中，我們對 `5`（二進位為 `101`）和 `3`（二進位為 `011`）執行以 bit 為單元逐一 AND 操作。結果是 `1`（二進位為 `001`），因為只有最右邊的位元在兩個數字中都是 `1`。

以 bit 為單元逐一 OR (`|`) 運算子會在任一或兩個運算元的對應位元為 `1` 時，於該位元位置傳回 `1`。例如：

:::interactive_editor

```js
let a = 5;  // Binary: 101
let b = 3;  // Binary: 011
console.log(a | b);  // 7 (Binary: 111)
```

:::

這裡，結果是 `7`（二進位為 `111`），因為在每個位置上至少有一個位元是 `1`。

以 bit 為單元逐一 XOR (`^`) 運算子會在任一但非兩個運算元對應的位元為 `1` 時，在該位元位置傳回 `1`。例如：

:::interactive_editor

```js
let a = 5;  // Binary: 101
let b = 3;  // Binary: 011
console.log(a ^ b);  // 6 (Binary: 110)
```

:::

結果是 `6`（二進位為 `110`），因為從右邊數來的第一和第二位元在這兩個數字中不同。

以 bit 為單元逐一反轉運算子 `~` 會反轉其運算元的所有位元。例如：

:::interactive_editor

```js
let a = 5;  // Binary: 101
console.log(~a);  // -6
```

:::

這可能看起來令人驚訝，但這是因為負數在二進位中使用二補數的表述方式。

左移 (`<<`) 運算子會將所有位元向左移動指定的位元數。例如：

:::interactive_editor

```js
let a = 5;  // Binary: 101
console.log(a << 1);  // 10 (Binary: 1010)
```

:::

在這裡，所有位元都向左移動一個位置，有效地將數字乘以 `2`。

右移 (`>>`) 運算子將所有位元向右移動。 例如：

:::interactive_editor

```js
let a = 5;  // Binary: 101
console.log(a >> 1);  // 2 (Binary: 10)
```

:::

在這裡，所有位元都向右移動一個位置，有效地將數字除以 `2` 並向下取整。

以 bit 為單元逐一運算子常用於低層級程式設計和密碼學。雖然它們在日常的 JavaScript 程式設計中可能不那麼常用，但理解它們對於某些專門任務是有益的，並且可以加深你對電腦如何在基本層次運作的理解。
