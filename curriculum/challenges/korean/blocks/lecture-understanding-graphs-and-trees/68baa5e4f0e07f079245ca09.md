# --description--

그래프는 노드(정점) 집합과 이들을 연결하는 간선으로 이루어진 매우 강력한 자료 구조입니다.

코드에서 그래프를 구현하는 일반적인 방법 두 가지가 있습니다:

* 인접 행렬
    
* 인접 리스트
    

이 두 가지를 자세히 살펴보고 각각의 장점과 한계를 알아보겠습니다.

## 인접 행렬

먼저 인접 행렬부터 시작하겠습니다.

인접 행렬은 행과 열이 그래프의 정점을 나타내는 2차원 리스트입니다.

행렬의 값은 노드 간의 간선 또는 연결을 나타냅니다.

예를 들어, `matrix`라는 변수에 저장된 행렬이 있다면, `matrix[i][j]`가 행이고 `i`가 열인 `j`에 저장된 값은 노드 `i`와 노드 `j` 사이의 간선 또는 연결을 나타냅니다.

값은 그래프가 가중치가 있느냐 없느냐에 따라 의미가 달라질 수 있습니다:

* 그래프가 가중치가 없다면, `1` 값은 이 노드들 사이에 간선이 있음을 의미하고, `0` 값은 간선이 없음을 의미합니다.
    
* 그래프가 가중치가 있다면, 값은 노드들을 연결하는 간선의 가중치를 나타냅니다.
    

인접 행렬을 사용할 때 큰 장점 중 하나는 두 노드 사이에 간선이 있는지 확인하는 데 상수 시간 `O(1)`이 걸린다는 점입니다. 이는 프로그램이 2차원 리스트에서 해당 값을 바로 찾기 때문입니다.

하지만 이 간선 탐색의 효율성에는 대가가 따릅니다. 인접 행렬은 그래프의 노드 수 `O(V²)`에 대해 큰 이차 공간 복잡도 `V`를 요구합니다.

이는 간선이 적은 희소 그래프에는 비효율적입니다. 왜냐하면 희소 그래프라면 노드 사이에 간선이 없음을 나타내는 0이 행렬에 많이 저장되어야 하고, 이 0들도 메모리 공간을 차지하기 때문입니다.

또한 인접 행렬은 노드의 이웃을 찾는 데 비효율적입니다. 프로그램이 간선을 나타내는 0과 1을 찾기 위해 전체 행이나 열을 반복해야 하기 때문입니다. 최악의 경우 이 과정은 그래프의 노드 수 V에 대해 O(V) 시간이 걸릴 수 있습니다.

다음은 이 특정 그래프에 대한 인접 행렬 예시입니다:

<img src="https://cdn.freecodecamp.org/curriculum/lecture-transcripts/how-do-matrices-and-adjacency-lists-work-1.png" alt="A graph with four nodes labeled A, B, C, and D. Node A connects to B, C, and D. Node B connects to A and D. Node C connects to A. Node D connects to A and B.">

인접 행렬에서:

* 각 행은 노드를 나타냅니다. 첫 번째 행은 노드 A, 두 번째 행은 노드 B를 나타내고, 그 다음도 마찬가지입니다.
    
* 각 열도 노드를 나타냅니다.
    
* 행렬의 각 값은 노드 쌍 사이에 간선이 있는지 나타냅니다. 0은 간선이 없음을, 1은 간선이 있음을 의미합니다.
    

대각선의 값들은 각 노드가 자기 자신과 연결된 자기 루프가 있는지 여부를 나타냅니다. 예시에서는 그래프에 자기 루프가 없으므로 모두 0입니다.

이것은 행과 열이 해당 노드를 어떻게 나타내는지 보여주는 인접 행렬의 시각적 표현입니다.

예를 들어, 첫 번째 행은 `[0, 1, 1, 1]`입니다. 노드 A가 노드 B, C, D와 연결되어 있기 때문입니다:

```markdown
#      A  B  C  D
# A   [0, 1, 1, 1],
# B   [1, 0, 0, 1],
# C   [1, 0, 0, 0],
# D   [1, 1, 0, 0]
```

그리고 이것은 같은 인접 행렬을 Python 코드로 구현한 것입니다:

```python
adjacency_matrix = [
    [0, 1, 1, 1],  # The neighbors of A are B, C, and D
    [1, 0, 0, 1],  # The neighbors of B are A and D
    [1, 0, 0, 0],  # The only neighbor of C is A
    [1, 1, 0, 0]   # The neighbors of D are A and B
]
```

## 인접 리스트

그래프를 나타내는 또 다른 일반적인 방법은 인접 리스트를 사용하는 것입니다.

인접 리스트는 각 노드의 모든 이웃을 저장하는 배열 또는 사전입니다.

인접 리스트를 구현하는 두 가지 방법이 있습니다:

* 각 인덱스가 노드를 나타내고, 그 인덱스에 저장된 리스트가 이웃 노드를 포함하는 배열로 구현
    
* 각 키가 노드를 나타내고, 그 키에 연결된 값(리스트)이 해당 노드의 이웃을 포함하는 사전으로 구현
    

인접 리스트는 인접 행렬에 비해 공간 요구량이 더 효율적입니다. 공간 복잡도는 `O(V + E)`이며, 여기서 `V`은 정점(노드) 수, `E`는 간선 수입니다.

또한 모든 이웃 노드를 찾는 데도 효율적입니다. 이 작업은 해당 노드에 연결된 리스트에 접근하는 것만 필요하기 때문입니다.

하지만 단점도 있습니다.

인접 리스트는 두 노드 사이에 간선이 있는지 확인하는 데 인접 행렬보다 덜 효율적입니다.

최악의 경우, 노드가 그래프 내 모든 다른 노드와 연결되어 있다면 매우 긴 이웃 리스트를 검색해야 하므로 탐색에 `O(V)` 시간이 걸릴 수 있습니다.

다음은 이 그래프에 대한 인접 리스트 예시입니다:

<img src="https://cdn.freecodecamp.org/curriculum/lecture-transcripts/how-do-matrices-and-adjacency-lists-work-1.png" alt="A graph with four nodes labeled A, B, C, and D. Node A connects to B, C, and D. Node B connects to A and D. Node C connects to A. Node D connects to A and B. It's the same image as before.">

이 인접 리스트는 사전으로 구현되었습니다. 사전의 각 키는 노드를 나타내고, 그 키에 연결된 값은 해당 노드의 모든 이웃을 포함하는 리스트입니다:

```python
adjacency_list = {
    'A': ['B', 'C', 'D'],
    'B': ['A', 'D'],
    'C': ['A'],
    'D': ['A', 'B']
}
```

또는 2차원 리스트로 구현할 수도 있습니다. 각 인덱스가 노드를 나타냅니다. 예를 들어, 인덱스 0은 노드 A, 인덱스 1은 노드 B를 나타냅니다:

```python
adjacency_list = [
    ['B', 'C', 'D'],  # Neighbors of A (index 0)
    ['A', 'D'],       # Neighbors of B (index 1)
    ['A'],            # Neighbors of C (index 2)
    ['A', 'B']        # Neighbors of D (index 3)
]
```

이 2차원 리스트가 인접 행렬과 비슷해 보일 수 있지만, 둘은 꽤 다릅니다.

* 인접 행렬은 그래프 내 간선이나 간선의 가중치를 나타내는 0, 1 또는 다른 값을 저장합니다.
    
* 인접 리스트는 각 노드의 실제 이웃 리스트를 저장합니다.
    

이 차이는 매우 중요하므로 반드시 익숙해지세요.

인접 행렬과 인접 리스트 모두 그래프를 구현하는 데 매우 중요합니다. 둘 중 어떤 것을 선택할지는 그래프 크기와 데이터를 어떻게 사용할지에 따라 다릅니다. 인접 행렬은 간선이 많은 밀집 그래프에 유용하며, 인접 리스트는 희소 그래프가 더 흔한 실제 상황에서 보통 선호됩니다.
