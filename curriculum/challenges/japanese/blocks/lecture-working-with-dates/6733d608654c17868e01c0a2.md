# --interactive--

前のレッスンでは、JavaScriptで`Date`オブジェクトを扱う方法を学びました。しかし、JavaScriptで日付をフォーマットする方法はいくつかあります。

このレッスンでは、JavaScriptで日付をフォーマットするための`toISOString()`、`toString()`、`toLocaleDateString()`メソッドの使い方を見ていきます。

さまざまなメソッドを見る前に、まず`Date`オブジェクトがどのようなものかを復習しましょう。

:::interactive_editor

```js
const date = new Date();
console.log(date);
```

:::

`date`オブジェクトをコンソールに出力すると、ユーザーのシステム設定に基づいた現在の日付と時刻が表示されます。以下はその出力例です。

```js
Sun Sep 29 2024 19:45:37 GMT-0700 (Pacific Daylight Time)
```

`toString()`オブジェクトに`date`メソッドを使うと、上記と同じ出力が得られます。

拡張ISO形式（ISO 8601）で日付をフォーマットするには、次のように`toISOString()`メソッドを使います。

:::interactive_editor

```js
const date = new Date();
console.log(date.toISOString());
```

:::

ISO 8601は日付と時刻を表す国際標準です。フォーマットは`YYYY-MM-DDTHH:mm:ss.sssZ`です。

以下はその出力例です。

```js
2024-09-30T02:47:20.292Z
```

別の方法として、`toLocaleDateString()`メソッドを使うこともできます。このメソッドはユーザーのロケールに基づいて日付をフォーマットできます。`toLocaleDateString()`メソッドの基本的な構文は以下の通りです。

:::interactive_editor

```js
const date = new Date();
console.log(date.toLocaleDateString());
```

:::

以下はその出力例です。

```js
9/29/2024
```

`toLocaleDateString()`メソッドは2つのオプションのパラメータを受け取ります：`locales`と`options`です。

`locales`パラメータは使用するロケールを表す文字列です。例えば、英語（アメリカ合衆国）には`en-US`、フランス語（フランス）には`fr-FR`を渡せます。`locales`パラメータを渡さない場合は、デフォルトのロケールが使われます。以下は`locales`パラメータの使い方の例です。

:::interactive_editor

```js
const date = new Date();
console.log(date.toLocaleDateString("fr-FR"));
```

:::

2つ目のオプションパラメータは`options`パラメータです。このパラメータは日付文字列のフォーマットを指定できるオブジェクトです。以下は`options`パラメータの使い方の例です。

:::interactive_editor

```js
const date = new Date();
const options = {
  weekday: "long",
  year: "numeric",
  month: "long",
  day: "numeric",
};
console.log(date.toLocaleDateString("en-GB", options));
```

:::

上の例では、オプションとして英語（イギリス）を指定し、曜日の完全表記、年、月、日を含めるようにしました。

以下はその出力例です。

```js
Sunday, September 29, 2024
```

このレッスンでは、JavaScriptで日付をフォーマットする方法のうちいくつかだけを扱いました。JavaScriptで日付をフォーマットするための他の多くのメソッドやライブラリもありますが、`toISOString()`と`toLocaleDateString()`メソッドは日付フォーマットの良い出発点です。
