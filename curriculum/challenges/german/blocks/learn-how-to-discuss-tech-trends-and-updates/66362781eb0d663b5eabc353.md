# --description--

`Serverless computing` ist ein `cloud computing`-Ausführungsmodell, bei dem der Cloud-Anbieter den Server betreibt und die Zuweisung der Maschinenressourcen dynamisch verwaltet. Die Preisgestaltung basiert auf der tatsächlichen Menge der von einer Anwendung verbrauchten Ressourcen und nicht auf vorab gekauften Kapazitätseinheiten. Es wird `serverless` genannt, weil die Entwickler sich nicht um Server kümmern müssen – sie laden einfach ihren Code hoch, und der Cloud-Anbieter übernimmt den Rest.

Zum Beispiel könnten Sie in `serverless computing` eine Funktion zu einem Cloud-Dienst hochladen, die nur bei Bedarf ausgeführt wird.
