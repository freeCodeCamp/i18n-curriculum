# --description--

Komponenten sind die Bausteine von React-Anwendungen, die es Entwicklerinnen und Entwicklern ermöglichen, komplexe Benutzeroberflächen in kleinere, überschaubare Teile zu zerlegen. Das erleichtert die Entwicklung und Wartung von groß angelegten Anwendungen.

Die zwei Arten von Komponenten in React sind funktionale und klassenbasierte Komponenten. In modernem React verwenden Entwickler funktionale Komponenten, und alle Beispiele, die wir heute betrachten, sind funktionale Komponenten.

Auf einer höheren Ebene können Sie sich Komponenten wie JavaScript-Funktionen vorstellen, die Elemente zurückgeben, die die Benutzeroberfläche beschreiben.

Diese Benutzeroberfläche wird mit JSX beschrieben, einer Syntaxerweiterung für JavaScript, die HTML ähnlich sieht, Ihnen aber erlaubt, UI-Elemente auf eine deklarativere Weise zu schreiben.

Schauen wir uns ein Beispiel für eine React-Komponente an:

```jsx
function Greeting() {
  const name = "John"
  {/* The result will be Hello John*/}
  return <h1 className="title">Hello {name}</h1>;
}
```

In diesem Beispiel haben wir eine Komponente namens `Greeting` definiert. Die geschweiften Klammern `{}` innerhalb der `h1`-Tags ermöglichen es uns, JavaScript-Ausdrücke einzubetten, sodass wir auf die `name`-Variable innerhalb des `h1`-Elements zugreifen können.

Wir wenden außerdem eine `className` namens `title` auf das `h1`-Element an.

Aber warum verwenden wir `className` anstelle von `class`, wie bei regulären HTML-Elementen?

Das liegt daran, dass `class` in JavaScript ein reservierter Name ist. Deshalb müssen wir stattdessen `className` verwenden.

Wir verwenden auch einen Kommentar in JSX, der zeigt, wie das Ergebnis aussehen wird. Es ist wichtig zu beachten, dass Sie reguläre Kommentar-Syntax wie diese verwenden können, aber sie muss in geschweifte Klammern eingeschlossen sein, damit sie funktioniert:

```jsx
{/* Block Comments */}
```

Eine weitere Sache, die Ihnen vielleicht aufgefallen ist, ist, dass wir am Anfang des Komponentennamens einen Großbuchstaben verwenden. Aber warum können wir nicht nur Kleinbuchstaben verwenden?

Das liegt daran, dass React Komponenten mit einem Großbuchstaben als benutzerdefinierte Komponenten behandelt, während Elemente mit Kleinbuchstaben als eingebaute HTML-Elemente angesehen werden.

Wenn React auf ein Tag mit Kleinbuchstaben trifft, wie `<div>` oder `<span>`, geht es davon aus, dass es sich um ein Standard-HTML-Element handelt. Beginnt der Komponentenname jedoch mit einem Großbuchstaben, behandelt React es als benutzerdefinierte Komponente und rendert sie entsprechend. Diese Unterscheidung hilft React, zwischen nativen HTML-Tags und von Ihnen erstellten Komponenten zu unterscheiden.

Um diese `Greeting`-Komponente in unserer Anwendung zu verwenden, würden wir so etwas schreiben:

```jsx
<Greeting />
```

Dies würde ein `h1`-Element mit dem Text `Hello John` auf der Seite rendern. Schauen Sie sich aber die Syntax genau an. Wenn wir die Komponente verwenden, endet sie mit einem Schrägstrich und dann dem Größer-als-Zeichen.

Beim Arbeiten mit JSX müssen alle Tags und Verwendungen von Komponenten explizit geschlossen werden. Wenn die Komponente oder das Tag keine Kinder hat, müssen Sie sie also explizit so schließen, wie hier gezeigt:

```jsx
<Greeting /> {/* /> is required */}
```

Bisher haben wir nur betrachtet, wie man ein einzelnes `h1`-Element rendert. Aber Sie können tatsächlich mehrere Elemente rendern.

Schauen wir uns den folgenden Beispielcode an:

```jsx
function Greeting() {
  const name = "John";
  {/* This will throw an error */}
  return <h1>Hello {name}</h1> 
  <p>Nice to meet you.</p>
}
```

Wir versuchen, einen weiteren Satz von `Nice to meet you` hinzuzufügen, aber er wird nicht korrekt auf der Seite angezeigt.

Stattdessen erscheint eine Fehlermeldung. Die Fehlermeldung lautet `Adjacent JSX elements must be wrapped in an enclosing tag.`

Der Grund für diese Fehlermeldung ist, dass mehrere Geschwisterelemente in einem übergeordneten Element eingeschlossen sein müssen. Während Sie die `h1`- und `p`-Elemente in einem einfachen `div` einschließen könnten, gibt es eine andere Möglichkeit, den Fehler zu beheben.

React-Fragmente werden verwendet, um Elemente zusammenzufassen. So sieht das überarbeitete Beispiel aus:

```jsx
function Greeting() {
  const name = "John";
  return (
    <Fragment>
      <h1>Hello {name}</h1>
      <p>Nice to meet you.</p>
    </Fragment>
  );
}
```

Sie können auch leere JSX-Tags verwenden, die als Kurzform für Fragmente dienen können:

```jsx
function Greeting() {
  const name = "John";
  return (
    <>
      <h1>Hello {name}</h1>
      <p>Nice to meet you.</p>
    </>
  );
}
```

In zukünftigen Lektionen werden wir weiterhin lernen, wie man mit Komponenten und JSX arbeitet. Aber für den Moment haben Sie eine solide Einführung in den Aufbau von Benutzeroberflächen mit Komponenten erhalten und damit eine starke Grundlage für das, was noch kommt.
