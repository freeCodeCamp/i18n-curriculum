## 什麼是 npm？

- **定義**：npm 是一個線上開源軟體的登錄檔，功能類似程式碼的「應用程式商店」。
- **官方含義**：npm 並非正式代表「Node Package Manager」；它是一個遞歸的反向縮寫，意指「npm 不是一個縮寫」。
- **三個相關部分**：

  - **網站 (npmjs.com)**：用於查詢登錄檔中的套件。
  - **登錄檔**：一個大型公開資料庫，儲存所有套件。
  - **CLI（命令列介面）**：隨 Node.js 安裝的工具，用來管理專案相依性。

- **替代方案**：雖然 npm 是最廣泛使用的，但流行的替代方案包括 **Yarn**、**PNPM** 和 **Bun**。

## package.json 檔案

- **用途**：一個設定檔，儲存 Node.js 專案的詮釋資料、相依性和腳本。
- **主要欄位**：

  - `name` 和 `version`：識別專案。
  - `main`：指定進入點檔案，通常是 `index.js`。
  - `license`：定義法律條款（預設為 `ISC`）。
  - `dependencies`：執行應用程式時所需的套件。
  - `devDependencies`：只在開發期間需要的工具，例如測試框架。

- **產生**：

  - `npm init`：啟動互動式設定流程。
  - `npm init -y`：立即安裝一個帶有預設值的 `package.json`。

## 語義版本控制（SemVer）

- **格式**：`MAJOR.MINOR.PATCH`。
- **規則**：

  - **MAJOR**：破壞性變更，可能影響現有程式碼。
  - **MINOR**：向下相容的新功能。
  - **PATCH**：小型錯誤修正和改進。

- **範圍符號**：

  - `^`（插入符號）：允許次要和修補更新（例如 `^1.2.3` 允許 `1.x.x`）。
  - `~`（波浪號）：只允許修補更新（例如 `~1.2.3` 允許 `1.2.x`）。
  - `*`（星號）：匹配任何版本；主要用於測試。

## 相依性管理

- **安裝**：

  - `npm install <name>`：新增生產相依性。
  - `npm install <name> -D`：新增開發相依性。
  - `npm install <name>@<version>`：安裝特定版本。

- **Node Modules 與鎖定**：

  - `node_modules/`：包含已安裝套件的實際程式碼。
  - `package-lock.json`：鎖定所有相依性和「子相依性」的確切版本，確保環境一致性。

- **移除**：`npm uninstall <name>` 移除套件並更新 `package.json`。
- **維護**：

  - `npm outdated`：識別哪些套件有較新版本可用。
  - `npm update`：更新在允許的 SemVer 範圍內的套件。

## npm 腳本

- **功能**：在 `scripts` 區段的 `package.json` 中定義的自定命令，用來自動化任務。
- **執行**：透過 `npm run <script_name>` 執行。
- `start`、`test`、`stop` 和 `restart` 有簡寫（例如 `npm start`）。
- **序列執行**：
- `&&`：只有第一個命令成功時才執行第二個命令。
- `;`：不論成功與否，依序執行命令。
- **並行執行**：使用 `&` 運算子在背景執行多個處理程序。
- **引數**：使用 `--` 分隔符傳遞引數給腳本（例如 `npm start -- 8000`）。

## 發佈到 npm 登錄檔

- **登錄檔存取**：公開和私有套件的資料庫。
- **發佈步驟**：

1. 在 `npmjs.com` 創建免費帳號。
2. 使用 `npm login` 透過終端機登入。
3. 確保 `name` 唯一且 `version` 有效，並存在 `package.json`。
4. 創建 `README.md` 和 `.npmignore`（用於排除敏感檔案）。
5. 執行 `npm publish`（或針對作用域套件使用 `npm publish --access public`）。

- **作用域**：用於避免名稱衝突，透過 `@username/package-name` 發佈。

## Node.js 的模組系統

### CommonJS (CJS)

- **預設系統**：Node.js 處理模組的原始方式；同步載入。
- **語法**：
- **匯入**：`const module = require('./path')`。
- **匯出**：`module.exports = { functionName }`。
- **識別**：用於 `.cjs` 檔案，或當 `.js` 未在 `type` 中指定時的 `package.json` 檔案。

### ES 模組 (ESM)

- **標準格式**：現代 JavaScript 標準；為了效率而非同步載入。
- **語法**：

  - **匯出**：`export { name }` 或 `export default function`。
  - **匯入**：`import { name } from './path.mjs'`。
  - **識別**：用於 `.mjs` 檔案，或當 `.js` 包含 `package.json` 時的 `"type": "module"` 檔案。
