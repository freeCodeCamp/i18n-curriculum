# --description--

في بايثون، تعبير `raise` هو أداة قوية تتيح لك استدعاء الاستثناءات يدويًا في شفرتك. يمنحك التحكم في متى وكيف تُولد الأخطاء، مما يمكنك من إنشاء شروط خطأ مخصصة وفرض سلوك معين للبرنامج.

يُستخدم تعبير `raise` لرمي استثناء صريح في أي نقطة من برنامجك، مما يسمح لك بالإشارة إلى حدوث حالة خطأ أو عدم استيفاء متطلبات معينة.

يمكن استخدام تعبير `raise` في بايثون بعدة طرق لتحفيز الاستثناءات. في أبسط أشكاله، يمكنك رفع استثناءات مدمجة أو إنشاء رسائل خطأ مخصصة. إليك مثالًا بسيطًا:

```py
def check_age(age):
    if age < 0:
        raise ValueError('Age cannot be negative')
    return age

try:
    check_age(-5)
except ValueError as e:
    print(f'Error: {e}') # Error: Age cannot be negative
```

يمكنك هنا أن ترى أن `raise` هي الكلمة المفتاحية التي تحفز الاستثناء.

في هذا المثال، نرفع استثناء `ValueError` مع رسالة مخصصة عند تقديم عمر غير صالح.

يمكن أيضًا استخدام تعبير `raise` لإعادة رفع الاستثناء الحالي، وهو أمر مفيد بشكل خاص في معالجة الاستثناءات:

```py
def process_data(data):
    try:
        result = int(data)
        return result * 2
    except ValueError:
        print('Logging: Invalid data received')
        raise  # Re-raises the same ValueError

try:
    process_data('abc')
except ValueError:
    print('Handled at higher level')
```

هنا الكلمة المفتاحية `raise` (بدون معلمات) تعيد رفع الاستثناء الحالي الذي يتم التعامل معه.

هذا يسمح لك بتسجيل الأحداث أو إجراء التنظيف مع استمرار تمرير الخطأ عبر سلسلة الاستدعاءات.

يمكنك إنشاء ورفع استثناءات مخصصة بتعريف أصناف استثناء خاصة بك:

```py
class InsufficientFundsError(Exception):
    def __init__(self, balance, amount):
        self.balance = balance
        self.amount = amount
        super().__init__(f'Insufficient funds: ${balance} available, ${amount} requested')

def withdraw(balance, amount):
    if amount > balance:
        raise InsufficientFundsError(balance, amount)
    return balance - amount

try:
    new_balance = withdraw(100, 150)
except InsufficientFundsError as e:
    print(f'Transaction failed: {e}')
```

هنا ترى أن أصناف الاستثناء المخصصة ترث من `Exception` أو من أصنافها الفرعية.

ستتعلم المزيد عن الأصناف والوراثة في دروس لاحقة. الآن، اعرف أن هذه طريقة لإنشاء استثناءات خاصة بك بمنطق مخصص.

يمكن أيضًا استخدام تعبير `raise` مع الكلمة المفتاحية `from` لربط الاستثناءات، مما يوضح العلاقة بين الأخطاء المختلفة:

```py
def parse_config(filename):
    try:
        with open(filename, 'r') as file:
            data = file.read()
            return int(data)
    except FileNotFoundError:
        raise ValueError('Configuration file is missing') from None
    except ValueError as e:
        raise ValueError('Invalid configuration format') from e

config = parse_config('config.txt')
```

هنا ترى أن `raise ... from None` يكتم سياق الاستثناء الأصلي:

```bash
Traceback (most recent call last):
  File "main.py", line 12, in <module>
    config = parse_config('config.txt')
             ^^^^^^^^^^^^^^^^^^^^^^^^^^
  File "main.py", line 7, in parse_config
    raise ValueError('Configuration file is missing') from None
ValueError: Configuration file is missing
```

و`raise ... from e` يربط الاستثناء الجديد بالأصلي، محافظًا على أثر الخطأ:

```bash
Traceback (most recent call last):
  File "main.py", line 5, in parse_config
    return int(data)
           ^^^^^^^^^
ValueError: invalid literal for int() with base 10: ''

The above exception was the direct cause of the following exception:

Traceback (most recent call last):
  File "main.py", line 12, in <module>
    config = parse_config('config.txt')
             ^^^^^^^^^^^^^^^^^^^^^^^^^^
  File "main.py", line 9, in parse_config
    raise ValueError('Invalid configuration format') from e
ValueError: Invalid configuration format
```

يمكنك أيضًا رفع الاستثناءات بشكل شرطي باستخدام تعبيرات `assert`، التي هي اختصار فعليًا لـ `raise` مع `AssertionError`:

```py
def calculate_square_root(number):
    assert number >= 0, 'Cannot calculate square root of negative number'
    return number ** 0.5

try:
    result = calculate_square_root(-4)
except AssertionError as e:
    print(f'Assertion failed: {e}')
```

تعبير `raise` ضروري لإنشاء تطبيقات قوية تحتاج إلى فرض قواعد العمل، والتحقق من المُدخلات، وتوفير رسائل خطأ واضحة. باستخدام `raise` بشكل استراتيجي، تجعل شفرتك أكثر توقعًا وأسهل في معالجة الأخطاء، مع تقديم ملاحظات واضحة للمستخدمين حول ما حدث من خطأ.
