# --description--

`mapDispatchToProps()` 関数は、Reactコンポーネントに特定のアクションクリエーターを提供して、Reduxストアに対してアクションをディスパッチできるようにするために使います。これは、前のチャレンジで書いた `mapStateToProps()` 関数と構造が似ています。ディスパッチアクションをプロパティ名にマップするオブジェクトを返し、それがコンポーネントの `props` になります。ただし、各プロパティは `state` の一部を返す代わりに、アクションクリエーターと関連するアクションデータを使って `dispatch` を呼び出す関数を返します。この `dispatch` は、関数を定義するときに `mapDispatchToProps()` のパラメータとして渡されるため利用できます。これは `state` を `mapStateToProps()` に渡したのと同じです。裏側では、React Redux は Redux の `store.dispatch()` を使って `mapDispatchToProps()` でこれらのディスパッチを行っています。これは、`store.subscribe()` を `state` にマップされたコンポーネントに使うのと似ています。

例えば、`loginUser()` というアクションクリエーターがあり、`username` をアクションペイロードとして受け取るとします。このアクションクリエーターに対して `mapDispatchToProps()` から返されるオブジェクトは次のようになります。

```jsx
{
  submitLoginUser: function(username) {
    dispatch(loginUser(username));
  }
}
```

# --instructions--

コードエディターには `addMessage()` というアクションクリエーターが用意されています。`mapDispatchToProps()` を引数に取り、オブジェクトを返す `dispatch` 関数を書いてください。そのオブジェクトは、ディスパッチ関数に設定された `submitNewMessage` というプロパティを持ち、ディスパッチ時に追加する新しいメッセージのパラメータを受け取ります。`addMessage()` をディスパッチします。

# --hints--

`addMessage` は `type` と `message` というキーを持つオブジェクトを返すべきです。

```js
assert(
  (function () {
    const addMessageTest = addMessage();
    return (
      addMessageTest.hasOwnProperty('type') &&
      addMessageTest.hasOwnProperty('message')
    );
  })()
);
```

`mapDispatchToProps` は関数であるべきです。

```js
assert(typeof mapDispatchToProps === 'function');
```

`mapDispatchToProps` はオブジェクトを返すべきです。

```js
assert(typeof mapDispatchToProps() === 'object');
```

`addMessage` を `submitNewMessage` と共に `mapDispatchToProps` からディスパッチすると、ディスパッチ関数にメッセージが返されるはずです。

```js
assert(
  (function () {
    let testAction;
    const dispatch = (fn) => {
      testAction = fn;
    };
    let dispatchFn = mapDispatchToProps(dispatch);
    dispatchFn.submitNewMessage('__TEST__MESSAGE__');
    return (
      testAction.type === 'ADD' && testAction.message === '__TEST__MESSAGE__'
    );
  })()
);
```
