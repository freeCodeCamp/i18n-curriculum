# --description--

Express 앱이 명확한 오류 메시지 없이 갑자기 멈춘 적이 있나요? 디버깅과 로깅은 내부에서 무슨 일이 일어나고 있는지 정확히 볼 수 있게 도와주어 문제를 더 빠르게 찾고 고칠 수 있습니다.

이번 수업에서는 Express가 디버깅과 로깅을 어떻게 처리하는지, 그리고 이를 통해 개발 작업 흐름을 어떻게 개선할 수 있는지 살펴보겠습니다.

## Express의 내장 디버깅

Express에는 이미 `debug`라는 디버깅 도구가 내장되어 있습니다. 별도로 설치할 필요가 없습니다.

디버깅을 켜려면 앱을 실행할 때 `DEBUG` 환경 변수를 설정하기만 하면 됩니다.

예를 들어 다음과 같습니다:

```js
DEBUG=express:* node index.js
```

이 명령은 `express:*` 네임스페이스 아래의 모든 디버깅 메시지를 활성화합니다. 다음과 같은 출력이 나타나기 시작할 것입니다:

```js
express:router:route dispatching GET /
express:router:layer processing GET /
```

이 메시지들은 라우트 디스패치, 미들웨어 호출 등을 추적하는 데 도움을 줍니다. 앱에서 요청 흐름을 이해하는 데 아주 유용합니다.

좀 더 구체적으로 하고 싶나요? 네임스페이스를 좁힐 수 있습니다:

```js
DEBUG=express:router node index.js
```

이렇게 하면 Express의 라우터 로직과 관련된 로그만 표시됩니다.

Windows에서는 PowerShell에서 구문이 약간 다릅니다:

```js
$env:DEBUG = "express:*"; node index.js
```

CMD에서는 이렇게 보입니다:

```js
set DEBUG=express:* && node index.js
```

같은 개념이지만 환경 변수를 설정하는 방법이 다를 뿐입니다.

이제 Express에서 로깅을 살펴보겠습니다.

HTTP 요청을 로깅할 때 많은 개발자가 `morgan` 같은 미들웨어를 사용합니다.

사용 방법은 다음과 같습니다:

```bash
npm install morgan
```

그다음 앱에 추가하세요:

```js
const express = require('express')
const morgan = require('morgan')
const app = express()

app.use(morgan('dev'))s
```

이제 모든 들어오는 요청이 메서드, URL, 상태 코드, 응답 시간 같은 세부 정보와 함께 터미널에 기록됩니다.

왜 이것이 중요한가요?

디버깅은 프로덕션 로그를 복잡하게 만들지 않으면서 Express가 요청, 미들웨어, 라우팅을 어떻게 처리하는지 통찰을 제공합니다.

로깅은 요청 전반에서 무슨 일이 일어나는지 기록을 남겨, 버그를 디버깅하고 트래픽을 이해하며 문제를 조기에 발견하는 데 필수적입니다.

이 둘을 함께 사용하면 개발 과정에서 제어력과 명확성을 얻을 수 있습니다.
