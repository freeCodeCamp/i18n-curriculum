# --description--

In der vorherigen Lektion haben wir gesehen, was die Fetch-API ist und wie man sie verwendet. In dieser Lektion besprechen wir die `GET`, `POST`, `PUT` und `DELETE` HTTP-Methoden der Fetch-API.

Beginnen wir mit der gebräuchlichsten HTTP-Methode, der `GET`-Methode. Diese wird verwendet, um Daten von einem Server abzurufen. Wenn Sie `fetch()` ohne Angabe einer Methode verwenden, ist die Standardeinstellung `GET`.

```js
fetch('https://api.example.com/data')
```

In diesem Code machen wir eine `GET`-Anfrage an `https://api.example.com/data`. Beachten Sie bitte, dass Sie diese Daten nicht direkt verwenden können; Sie müssen die Antwort zuerst in ein JSON-Format umwandeln. Erst dann können Sie sie überall in Ihrem Projekt verwenden. Hier ist ein Beispiel, wie das funktioniert:

```js
fetch('https://api.example.com/data')
  .then(response => response.json())
  .then(data => console.log(data))
```

In diesem Code ist die Antwort, die von der Fetch-API kommt, ein Promise, und wir verwenden einen `.then`-Handler, um die Antwort in ein JSON-Format zu konvertieren. In einer früheren Lektion haben Sie gelernt, dass ein `Promise` ein Objekt ist, das die endgültige Fertigstellung (oder das Scheitern) eines asynchronen Prozesses und dessen Wert darstellt.

Der Wert eines Promise ist nicht bekannt, wenn das Promise erstellt wird. Er ist erst bekannt, wenn der asynchrone Prozess abgeschlossen ist. Wenn wir die beiden `.then`-Handler an den fetch-Aufruf verketten, nennt man das Promise-Verkettung, die in der nächsten Lektion behandelt wird.

Bisher haben wir Ressourcen von einem Server abgerufen. Aber wussten Sie, dass wir auch Daten an den Server senden können? Die `POST`-Methode wird verwendet, um Daten an einen Server zu senden, um eine Ressource zu erstellen. Hier ist ein Beispiel für eine `POST`-Methode, die verwendet wird, um eine Ressource auf dem Server zu erstellen:

```js
fetch('https://api.example.com/users', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({
    name: 'John Doe',
    email: 'john@example.com'
  })
})
```

In diesem Beispiel senden wir eine `POST`-Anfrage, um einen neuen Benutzer zu erstellen. Wir geben die Methode als `POST` an, setzen die passenden Kopfzeilen und fügen einen `body` mit den Daten hinzu, die wir senden möchten. Der Body muss ein String sein, deshalb verwenden wir `JSON.stringify()`, um unser Objekt in einen JSON-String umzuwandeln.

Die `PUT`-Methode wird verwendet, um bestehende Ressourcen eines Servers zu aktualisieren. Hier ist ein Beispiel:

```js
fetch('https://api.example.com/users/45', {
  method: 'PUT',
  headers: {
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({
    name: 'John Smith',
    email: 'john@example.com'
  })
})
```

In diesem Beispiel schauen Sie genau auf die URL, am Ende sehen Sie eine `45`. Diese wird typischerweise als eindeutige ID verwendet, um die Daten zu identifizieren, die wir aktualisieren möchten. Im Code haben wir die `PUT`-Methode verwendet und auch die Daten als `body` angegeben, die zur Aktualisierung der identifizierten Daten verwendet werden.

Die `DELETE`-Methode wird verwendet, um eine Ressource vom Server zu löschen. Hier ist ein Beispiel:

```js
fetch('https://api.example.com/users/45', {
  method: 'DELETE',
})
```

In diesem Code fügen wir die `DELETE`-Methode ein und eine ID am Ende der URL, um die Daten zu identifizieren, die gelöscht werden sollen.
