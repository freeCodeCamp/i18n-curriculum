## 在開始之前

Flexbox 佈局可能會有點複雜。在之前的課程中，你已經學會如何使用瀏覽器的開發者工具來檢查和偵錯。這些工具在接下來的課程中對你來說將非常重要。如果某些東西的行為不如你預期，每次都應該先在開發者工具中檢查它。

Flexbox 並不一定比我們迄今涵蓋的其他概念更困難，但它確實有更多的零件。直到你學完最後並能將所有內容整合起來之前，要運用這些第一課中學到的任何東西會有些困難。進行過程中，請善待自己，<strong>多多操作所有的程式碼範例。</strong>

你幾乎肯定會需要回來取用這些課程（或我們與你分享的幾個資源），當你進入本節結尾的指派時，但如果你花時間並嘗試我們提供的所有程式碼範例，當那個時候來臨時，你會更清楚該去哪裡尋找。

## 讓我們展現實力

Flexbox 是一種將項目排列成列或行的方式。這些項目會根據你定義的一些簡單規則進行彈性調整（即增長或縮小）。要開始，讓我們來看一個簡單的示範。

我們很快就會詳細說明這裡到底發生了什麼。但現在，讓我們透過移除包圍在上方 Codepen 中兩個與 flex 相關的 CSS 宣告式的 `/*` 和 `*/` 標記來取消註解，然後查看結果。

> 註解可防止瀏覽器將行解讀為程式碼，並包裹在特定的標記之間。CSS 使用 `/*` 作為開啟註解標記，`*/` 作為關閉註解標記，而 HTML 和 JavaScript 則有各自的語法。被註解掉的程式碼行，只要移除包圍程式碼的註解標記即可重新啟用。

現在所有 3 個 div 都應該水平排列。如果你使用 `1x`、`.5x` 和 `.25x` 按鈕調整結果訊框的大小，你也會看到 `div` 元素會「彈性調整」。它們會填滿可用區域，且每個寬度都相等。

如果你在 `.flex-container` 內的 HTML 中添加另一個 `div`，它將會與其他元素並排顯示，且所有元素會彈性調整以適應可用區域。

## --text--

當 CodePen 中與 flex 相關的兩個 CSS 宣告式被取消註解時會發生什麼？

## --answers--

`div` 元素會從視圖中隱藏，失去在容器中的可見性。

---

所有 3 個 `div` 元素水平排列，並且它們會彈性調整以相等寬度填滿可用區域。

---

每個 `div` 元素會放大並涵蓋整個容器，彼此重疊。

---

`div` 元素垂直排列在單一行中，忽略與 flex 相關的 CSS 宣告式。
