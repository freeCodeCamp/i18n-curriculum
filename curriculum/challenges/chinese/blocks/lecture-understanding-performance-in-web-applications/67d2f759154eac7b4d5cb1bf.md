# --description--

众所周知，感知性能不仅仅是网站实际有多快——更关乎用户感受到的速度。例如，你是否曾使用过一个感觉很慢的网站，尽管它实际上并不慢？这就是感知性能——网站给用户的速度感受。

小的调整可以极大地改善用户体验。让我们探索一些提升感知性能的实用方法！

加载动画可能会让用户感觉等待时间比实际更长。相反，尝试使用骨架屏——灰色占位符，模拟真实内容并逐渐装载：

```html
<div class="skeleton"></div>
<script>
  setTimeout(() => {
    document.querySelector(".skeleton").innerHTML = "✅ Content Loaded!";
  }, 2000);
</script>
```

用户感觉页面是在逐步装载，而不是在黑暗中等待。

除此之外，用户不应该在所有内容装载完成之前就开始等待交互。先装载重要内容，然后在后台装载其余内容。我们之前已经看过惰性装载：

```html
<img src="placeholder.jpg" data-src="real-image.jpg" class="lazy">
<script>
  document.addEventListener("DOMContentLoaded", function () {
    const lazyImages = document.querySelectorAll("img.lazy");
    lazyImages.forEach(img => {
      img.src = img.dataset.src;
    });
  });
</script>
```

这使页面感觉响应迅速，即使不是所有内容都已装载。

现在——如果你的网站能够预测用户接下来会点击什么呢？通过预装载，你可以在他们发出请求之前获取下一页：

```html
<link rel="preload" href="next-page.html" as="document">
```

当用户导航时，这会将感知的装载时间减少到接近零。

最后，用户不喜欢不确定性。当他们点击按钮时，立即确认他们的操作：

```html
<button onclick="this.innerText='⏳ Processing...'; setTimeout(() => this.innerText='✅ Done!', 2000);">
  Click Me
</button>
```

这意味着，即使有延迟，用户也会感觉到有事情正在发生。

总之，提升感知性能不仅仅是关于未经处理的速度，而是关于速度给用户的感觉。使用骨架屏，优先显示重要内容，智能预加载，并提供即时反馈，使你的网站感觉更快。
