# --interactive--

تسمح لك مجموعة الالتقاط بـ"التقاط" جزء من السلسلة المطابقة لاستخدامه كما تحتاج. تُعرّف مجموعات الالتقاط بواسطة أقواس عادية تحتوي على النمط المراد التقاطه، دون وجود أحرف بادئة مثل النظر للأمام.

لنلتقط `code` من التعبير النمطي الخاص بـ`freeCodeCamp`. للقيام بذلك، سنضع `code` بين أقواس عادية ونعرّفها كمجموعة التقاط:

```js
const regex = /free(code)camp/i;
```

لتأكيد السلوك، يمكننا اختباره على سلسلة `freecodecamp`:

:::interactive_editor

```js
const regex = /free(code)camp/i;
console.log(regex.test("freecodecamp")); // true
```

:::

لكن هذا لا يستخدم مجموعتنا الملتقطة فعليًا. بدلاً من ذلك، لنلق نظرة على نتيجة استخدام `match`:

```js
const regex = /free(code)camp/i;
console.log("freecodecamp".match(regex));
// [
//   'freecodecamp',
//   'code', <--
//   index: 0,
//   input: 'freecodecamp',
//   groups: undefined
// ]
```

هنا نرى أن مصفوفة `match` تحتوي على عنصر ثاني، وهو الجزء من السلسلة الذي التقطته مجموعة الالتقاط.

لاحظ كيف تطابق مجموعة الالتقاط النمط الدقيق `code`، بينما كانت فئة الأحرف ستطابق حرفًا واحدًا من القائمة `c`، `o`، `d`، و`e`.

لكن كيف نستخدم هذا فعليًا؟ تُستخدم مجموعات الالتقاط غالبًا عند استبدال محتويات سلسلة نصية. لنعد شفرة لذلك. سنحوّل `freecodecamp` إلى `paidcodeworld`:

:::interactive_editor

```js
const regex = /free(code)camp/i;
console.log("freecodecamp".replace(regex, "paidcodeworld"));
```

:::

هذا يعمل بمفرده، لكن ماذا لو لم نكن نعرف عدد أحرف `o` في `code`؟ إذا احتجنا إلى محدد كمي لواحد أو أكثر من `o`s:

:::interactive_editor

```js
const regex = /free(co+de)camp/i;
console.log("freecoooooooodecamp".replace(regex, "paidcodeworld"));
```

:::

نحصل على `paidcodeworld` كنتيجة. نريد الحفاظ على عدد أحرف `o`، لذا نحتاج لإعادة استخدام ما التقطه التعبير النمطي.

هنا يأتي دور المرجعية الخلفية. بدلًا من ترميز جزء `code` في سلسلة الاستبدال، يمكننا الإشارة مباشرة إلى مجموعة الالتقاط.

في استدعاء `replace`، تحقق المرجعية الخلفية باستخدام علامة الدولار (`$`) متبوعة برقم مجموعة الالتقاط التي تريد استخدامها. في حالتنا، سيكون `$1`، لأن `code` التقطت في مجموعة الالتقاط الأولى:

:::interactive_editor

```js
const regex = /free(co+de)camp/i;
console.log("freecoooooooodecamp".replace(regex, "paid$1world")); // paidcoooooooodeworld
```

:::

لقد حافظنا بنجاح على عدد غير معروف من أحرف `o` عند تحويل `freecodecamp` إلى `paidcodeworld`. لكن المرجعيات الخلفية ليست محصورة فقط على استدعاء الاستبدال. يمكنك استخدامها مباشرة في التعبير النمطي.

هذا يسمح لك بمطابقة نمط تم التقاطه سابقًا لاحقًا في التعبير النمطي.

لنفترض أننا نريد مطابقة `freecodecamp` مرتين، بنفس عدد أحرف `o`، ولكن في أي مكان في السلسلة.

أولًا، نحتاج لفصلهم باستخدام حرف البدل، والسماح لأي عدد من الأحرف بمطابقة هذا البدل:

```js
const regex = /free(co+de)camp.*free(co+de)camp/i;
```

لكن هذا التعبير الحالي لن يضمن أن يكون عدد أحرف `o` هو نفسه. لتحقيق ذلك، نحتاج لاستبدال مجموعة الالتقاط الثانية بمرجع إلى الأولى.

داخل التعبير النمطي، تُرمز المرجعية الخلفية بشرطة مائلة عكسية متبوعة برقم مجموعة الالتقاط:

:::interactive_editor

```js
const regex = /free(co+de)camp.*free\1camp/i;
console.log(regex.test("freecooooodecamp is great i love freecooooodecamp")); // true
console.log(regex.test("freecooooodecamp is great i love freecodecamp")); // false
```

:::

وبذلك، نرى أن السلسلة التي تحتوي على العدد الصحيح من أحرف `o` تطابق، بينما السلسلة التي تحتوي على عددين مختلفين من `o` لا تطابق.

هذه الصيغة رائعة، لكنها قد تصبح مربكة بسرعة عند الإشارة إلى مجموعات التقاط متعددة. لحسن الحظ، بدلًا من استخدام الأرقام، يمكنك إعطاء مجموعاتك أسماء.

لتعريف مجموعة التقاط مسماة، تضيف علامة استفهام (`?`) متبوعة بالاسم محاطًا بعلامتي أصغر وأكبر في بداية المجموعة. دعنا نسمي مجموعة الالتقاط `code`:

```js
const regex = /free(?<code>co+de)camp.*free\1camp/i;
```

الآن يمكننا تحديث المرجعية الخلفية في التعبير النمطي للإشارة إلى هذه المجموعة. تبدأ المرجعية الخلفية المسماة بشرطة مائلة عكسية متبوعة بالحرف `k` في جافاسكريبت. ثم تضيف الاسم، مرة أخرى محاطًا بعلامتي أصغر (`<`) وأكبر (`>`). لنلق نظرة على ذلك:

```js
const regex = /free(?<code>co+de)camp.*free\k<code>camp/i;
```

الآن إذا تحققنا من استدعاء `test()`، نرى أننا ما زلنا ننجح:

:::interactive_editor

```js
const regex = /free(?<code>co+de)camp.*free\k<code>camp/i;
console.log(regex.test("freecooooodecamp is freecooooodecamp")); // true
```

:::

لاستخدام مجموعة الالتقاط المسماة في استدعاء `replace()`، ندرج علامة الدولار في السلسلة، متبوعة بالاسم محاطًا بعلامتي أصغر وأكبر:

:::interactive_editor

```js
const regex = /free(?<code>co+de)camp/i;
console.log("freecooooodecamp".replace(regex, "paid$<code>camp")); // paidcooooodecamp
```

:::

أخيرًا، أحيانًا تريد إنشاء مجموعة من الأحرف، لكن لا تحتاج إلى النتيجة الملتقطة.

لنفترض أننا نريد مطابقة إما `freecodecamp` أو `freecandycamp`. يمكنك إنشاء نمطين مفصولين بواسطة تحقق OR:

```js
const regex = /freecodecamp|freecandycamp/i;
```

لكن هذا قد يصبح طويلاً جدًا للتعبيرات النمطية الأكبر. بدلًا من ذلك، يمكنك إنشاء مجموعة غير ملتقطة حول الأحرف التي تريد OR بينها:

```js
const regex = /free(?:code|candy)camp/i;
```

المجموعة غير الملتقطة لا تخزن تطابق `code|candy` بشكل منفصل في الذاكرة. لكنها مفيدة لإنشاء أنماط بديلة دون التضحية بالقراءة أو الأداء.
