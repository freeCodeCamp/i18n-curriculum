# --description--

保持你的依赖项最新——并且知道如何安装特定版本——对于保持你的项目稳定非常重要。它帮助你避免意外的错误，并在新版本发布时给予你更多的控件。

要安装特定版本的包，请使用此格式：

```bash
npm install package@version-number
```

例如，如果你的项目需要较旧版本的 Express，比如 `4.21.2`，你可以这样安装：

```bash
npm install express@4.21.2
```

如果你不包含版本号，npm 会自动安装可用的最新版本。

如果你不确定存在哪些版本，可以使用此 `Command` 查看它们：

```bash
npm view package-name versions
```

例如：

```bash
npm view express versions
```

要更新包，你可以单独更新它们或一次性全部更新。在此之前，先查看哪些包已过期会很有帮助：

```bash
npm outdated
```

这将显示哪些依赖项有可用的新版本，你应该看到类似如下的输出：

```bash
user@Kolade ~/Desktop/fCC/script-code/node/npm/install-spec-version  % npm outdated
Package  Current  Wanted  Latest  Location              Depended by
chalk      5.5.0   5.5.0   5.6.2  node_modules/chalk    install-spec-version
cleave     1.0.1   1.0.1   1.0.2  node_modules/cleave   install-spec-version
express   4.21.2  4.21.2   5.2.1  node_modules/express  install-spec-version
```

一旦你查看了哪些依赖项已过时，你就可以开始更新它们。

要更新单个依赖，请使用以下命令：

```bash
npm update package-name
```

这会仅在你的 `package.json` 文件允许的版本范围内更新包。例如，要更新 Express，你可以使用以下命令：

```bash
npm update express
```

如果你想跳过版本范围并安装一个包的最新版本，请使用以下命令：

```bash
npm install package-name@latest
```

要更新所有在允许版本范围内的依赖项，请运行：

```bash
npm update
```

但是，请注意，如果你的 `package.json` 不允许，`npm update` 不会将包升级到新的主版本。主版本通常包含破坏性更改，因此 npm 不会自动应用它们。

如果你特别想要最新的大版本，你必须运行：

```bash
npm install package-name@latest
```

这迫使 npm 安装最新的版本，无论你的 `package.json` 中的版本范围是什么。
