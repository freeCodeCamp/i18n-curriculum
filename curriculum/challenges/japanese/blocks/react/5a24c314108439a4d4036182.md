# --description--

前のチャレンジで、`style`属性にJavaScriptオブジェクトを設定する以外にも、HTMLのインラインスタイルとはいくつか構文の違いがあることに気づいたかもしれません。まず、特定のCSSスタイルプロパティの名前はキャメルケースを使います。例えば、前のチャレンジではフォントのサイズを`fontSize`で設定しましたが、`font-size`ではありません。`font-size`のようなハイフン付きの単語はJavaScriptオブジェクトのプロパティとしては無効な構文なので、Reactではキャメルケースを使います。ルールとして、ハイフン付きのスタイルプロパティはJSXではキャメルケースで書かれます。

すべてのプロパティ値の長さの単位（`height`、`width`、`fontSize`など）は、特に指定がなければ`px`とみなされます。例えば`em`を使いたい場合は、値と単位を引用符で囲み、`{fontSize: "4em"}`のようにします。長さの値で`px`がデフォルトのもの以外は、すべてのプロパティ値は引用符で囲む必要があります。

# --instructions--

大量のスタイルがある場合は、スタイル`object`を定数に代入してコードを整理できます。ファイルの先頭でスタイル定数をグローバル変数として宣言してください。`styles`定数を初期化し、3つのスタイルプロパティとその値を持つ`object`を代入してください。`div`には`purple`の色、`40`のフォントサイズ、`2px solid purple`のボーダーを設定します。次に`style`属性を`styles`定数に等しく設定してください。

# --hints--

`styles`変数は3つのプロパティを持つ`object`であるべきです。

```js
assert(Object.keys(styles).length === 3);
```

`styles`変数は`color`プロパティを`purple`の値に設定するべきです。

```js
assert(styles.color === 'purple');
```

`styles`変数は`fontSize`プロパティを`40`の値に設定するべきです。

```js
assert(styles.fontSize == 40);
```

`styles`変数は`border`プロパティを`2px solid purple`の値に設定するべきです。

```js
assert(styles.border === '2px solid purple');
```

コンポーネントは`div`要素をレンダリングするべきです。

```js
assert(
  (function () {
    const mockedComponent = Enzyme.shallow(React.createElement(Colorful));
    return mockedComponent.type() === 'div';
  })()

```

`div`要素は`styles`オブジェクトで定義されたスタイルを持つべきです。

```js
assert(
  (function () {
    const mockedComponent = Enzyme.shallow(React.createElement(Colorful));
    return (
      mockedComponent.props().style.color === 'purple' &&
      mockedComponent.props().style.fontSize == 40 &&
      mockedComponent.props().style.border === '2px solid purple'
    );
  })()
);
```
