# --description--

ここでは二分探索木から進んで、トライ木と呼ばれる別の種類の木構造を見ていきます。トライ木は、文字列やより一般的にはキーが文字列である連想配列や動的データセットを保持するためによく使われる順序付き探索木です。多くのキーが重複する接頭辞を持つ場合、例えば辞書のすべての単語のような場合に、非常に効率よくデータセットを保存できます。二分木とは異なり、ノードは実際の値に関連付けられていません。代わりに、ノードへの経路が特定のキーを表します。例えば、文字列 code をトライ木に保存したい場合、4つのノードがあり、それぞれの文字に対応します：c — o — d — e。この経路をたどることで code という文字列が作られ、その経路が保存したキーになります。次に、文字列 coding を追加したい場合、code の最初の3つのノードを共有し、d の後で分岐します。このようにして、大きなデータセットを非常にコンパクトに保存できます。さらに、検索も保存する文字列の長さにほぼ制限されるため非常に高速です。加えて、二分木とは異なり、ノードは任意の数の子ノードを持つことができます。上記の例から推測できるように、キーの終端を表すノードにはメタデータがよく保存され、後の探索でそのキーを取得できるようにします。例えば、上の例で codes を追加した場合、code の e が以前に入力されたキーの終端であることを知る方法が必要です。そうしないと、codes を追加したときにこの情報が失われてしまいます。

# --instructions--

単語を保存するトライ木を作成しましょう。`add` メソッドで単語を受け取り、トライ木のデータ構造に保存します。また、`isWord` メソッドで与えられた文字列が単語かどうかをクエリでき、`print` メソッドでトライ木に入力されたすべての単語を取得できます。`isWord` はブール値を返し、`print` はこれらすべての単語を文字列として配列で返す必要があります。このデータ構造が正しく実装されているか確認するために、ツリーの各ノードに `Node` 構造を用意しました。各ノードは `keys` プロパティを持つオブジェクトで、これは JavaScript の Map オブジェクトです。これは各ノードの有効なキーである個々の文字を保持します。また、ノードが単語の終端を表す場合に `end` に設定できる `true` プロパティもノードに作成しました。

```js
assert(

    var test = false;
    if (typeof Trie !== 'undefined') {
      test = new Trie();
    } else {
      return false;
    }
    test.add('hop');
    test.add('hops');
    test.add('hopped');
    test.add('hoppy');
    test.add('hope');
    return (
      test.isWord('hop') &&
      !test.isWord('ho') &&
      test.isWord('hopped') &&
      !test.isWord('hopp') &&
      test.isWord('hoppy') &&
      !test.isWord('hoping')
    );
  })()
);
```
