# --description--

コマンドのオプションとフラグについて学びましょう。

前のレッスンでは、`touch readme.md` のようなコマンドに引数を渡す方法を学びました。オプション、またはフラグは、コマンドの動作に影響を与える特別な引数です。この2つの用語はほぼ同じ意味で使われますが、「フラグ」はオン/オフの切り替えとして機能するオプションを指すことが多いです。

オプションは通常、1つまたは2つのハイフン（`-`）が接頭辞として付けられ、引数とオプションを視覚的に区別します。まずは2つのハイフン、つまり「ロングフォーム」の構文を見てみましょう。

`ls` のような多くのアプリケーションは、`--version` フラグを受け付けます。これはアプリケーションのコマンドを実行する代わりに、現在のバージョンをターミナルに出力します。

多くのアプリケーションはまた、アプリケーションの使い方を表示する `--help` フラグも提供しています。

1つのハイフン、つまり「ショートフォーム」の構文は、通常1文字のオプションを使います。例えば、`-a` フラグと `ls` は、ドット（`.`）で始まる隠しファイルも含めてすべてのファイルを一覧表示します。例として `.env` があります。

これらのショートオプションの利点は、複数のフラグをまとめて連結できることです。`ls --all --human-readable --size` の代わりに、単一の文字をまとめて `ls -ahs` のように使えます。

一部のオプションは値を渡すことを期待しています。

ロングフォームのオプションを使う場合、通常は等号を使う必要があります。この構文では、値が等号（`=`）でオプションに直接連結されます。以下は、`ls` の動作を色を含めるか除外するかで変更する例です。

```bash
ls --color=never
```

ショートフォームのオプションを使う場合、通常は値をスペースで区切ります。例えば、以下は `width` を `ls` の結果に設定するロングフォームとショートフォームの例です。

```bash
ls -w 50
```

```bash
ls --width=50
```

ショートフォームが `-w 50` を使い、ロングフォームが `--width=50` を使う点に注目してください。この違いを知っておくことは重要で、オプションの値を位置引数として渡してしまうのを避けられます。

もし迷ったら、通常は `--help` フラグを使ってオプションの期待される構文を確認できます！
