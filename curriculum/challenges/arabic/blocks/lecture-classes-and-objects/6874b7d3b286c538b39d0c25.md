# --description--

الطرق الخاصة في Python، المعروفة أيضًا باسم "الطرق السحرية" أو "طرق dunder"، هي طرق Python خاصة تبدأ وتنتهي بشرطتين سفليتين مزدوجتين (`__`). كلمة "dunder" نفسها مشتقة من الشرطتين السفليتين المزدوجتين (**d** من double، **under** من underscores).

ربما استخدمت الطرق الخاصة بالفعل دون أن تعرف ذلك. في كل مرة تكتب شيئًا مثل `3 + 4`، يقوم Python بهدوء بتنفيذ `3.__add__(4)` خلف الكواليس. هذه طريقة خاصة تعمل. لذلك، رغم أنه يمكنك *استدعاء* الطرق الخاصة مباشرة، إلا أنك نادرًا ما تفعل ذلك. شيء مثل `3 + 4` أوضح وأسهل في القراءة من استدعاء `3.__add__(4)` بنفسك.

بجانب `__add__`، هناك طريقة خاصة أخرى سترى وتستخدم كثيرًا وهي `__init__()`، لأنها مهيئ للصنف. وهناك أيضًا طرق أخرى مثل `__len__()` و `__str__()`.

فكر في الطرق الخاصة كمخرجين للأنشطة بين المبرمج ومفسر لغة Python نفسه.

تذكر، لا تحتاج إلى استدعاء الطرق الخاصة مباشرة. بدلاً من ذلك، يستدعيها Python تلقائيًا عند حدوث إجراءات معينة. تشمل هذه العمليات:

- **العمليات الحسابية مثل الجمع والطرح والضرب والقسمة وغيرها**. للجمع، تُستدعى `__add__()`، وللطرح `__sub__()`، وللضرب `__mul__()`، وللقسمة `__truediv__()`.
    
- **عمليات السلاسل النصية مثل الربط والتكرار والتنسيق والتحويل إلى نص**. تُستدعى `__add__()` للربط، و`__mul__()` للتكرار، و`__format__()` للتنسيق، و`__str__()` و`__repr__()` للتحويل إلى نص، وهكذا.
    
- **عمليات المقارنة مثل التساوي، والأقل من، والأكبر من، وغيرها**. تُستدعى `__eq__()` لفحص التساوي، و`__lt__()` للأقل من، و`__gt__()` للأكبر من، وهكذا.
    
- **عمليات التكرار مثل جعل الكائن قابلًا للتكرار والتقدم عبر العناصر**. تُستدعى `__iter__()` لإرجاع مكرر و `__next__()` لجلب العنصر التالي.
    

عادةً، أنواع بيانات Python مثل السلاسل النصية والأرقام تعرف بالفعل كيف تضيف الأشياء، وتربط السلاسل، وتقارن للتساوي، وتُستخدم في الحلقات، وغيرها.

لكن عندما تنشئ صنفك الخاص، لن يعرف Python كيف يتعامل مع الأشياء تلقائيًا.

هنا تأتي الطرق الخاصة — فهي تتيح لك تخصيص سلوك Python المدمج.

لنفترض أنك تريد الحصول على عدد الصفحات في كائنات الكتب التي تُنشأ باستخدام الصنف أدناه، أو مقارنتها والحصول على سلسلة نصية قابلة للقراءة للكائنات. هذا ما يحدث بدون الطرق الخاصة:

```python
class Book:
   def __init__(self, title, pages):
       self.title = title
       self.pages = pages

book1 = Book("Built Wealth Like a Boss", 420)
book2 = Book("Be Your Own Start", 420)

print(len(book1)) # TypeError: object of type 'Book' has no len()
print(str(book1)) # <__main__.Book object at 0x102ed2900>
print(book1 == book2) # False even though they have the same number of pages
```

في المثال:

- فشل `len(book1)` لأن Python لا يعرف كيف يحصل على طول كائن الكتاب الخاص بك بدون `__len__()`
    
- طبع `str(book1)` شيئًا مثل `<__main__.Book object at 0x102ed2900>` لأن هذا هو التمثيل الافتراضي عندما لا تستخدم `__str__()`
    
- كانت نتيجة `book1 == book2` هي `False` لأن Python فقط يتحقق مما إذا كان الكائنان نفسهما في الذاكرة، وليس بالمحتوى.
    

إليك كيف يمكنك تعريف طرق خاصة خاصة بك مثل `__len__()` و `__str__()` و `__eq__()` لتسهيل العمل مع الكائنات التي تُنشأ من صنف `Book`:

```python
class Book:
   def __init__(self, title, pages):
       self.title = title
       self.pages = pages

   def __len__(self):
       return self.pages

   def __str__(self):
       return f"'{self.title}' has {self.pages} pages"

   def __eq__(self, other):
       return self.pages == other.pages
  
book1 = Book("Built Wealth Like a Boss", 420)
book2 = Book("Be Your Own Start", 420)

print(len(book1)) # 420
print(len(book2)) # 420
print(str(book1)) # 'Built Wealth Like a Boss' has 420 pages
print(str(book2)) # 'Be Your Own Start' has 420 pages
print(book1 == book2) # True
```

مثال آخر هو عربة التسوق حيث تقوم بما يلي:

- إضافة عناصر إلى العربة
    
- إزالة عناصر من العربة
    
- الحصول على عدد العناصر في العربة
    
- التحقق من العناصر الموجودة في العربة
    
- التحقق مما إذا كان عنصر معين موجودًا في العربة
    
- إرجاع أو عرض عنصر في فهرس معين داخل العربة
    

بينما قد يكون لديك طريقة تضيف عناصر إلى العربة وتزيل بعض العناصر منها، يمكنك إنشاء طرق خاصة لكل الوظائف الأخرى:

- `__len__()` للحصول على طول العناصر في العربة
    
- `__iter__()` للتكرار عبر العناصر في العربة حتى تراها
    
- `__contains__()` للتحقق مما إذا كان عنصر معين موجودًا في العربة
    
- `__getitem__()` لإرجاع أو عرض عنصر في فهرس معين داخل العربة
    

إليك مثال على صنف `Cart` يحتوي على هذه الطرق المعرفة من المستخدم والطرق الخاصة:

```python
class Cart:
   def __init__(self):
       self.items = []

   def add(self, item):
       self.items.append(item)

   def remove(self, item):
       if item in self.items:
           self.items.remove(item)
       else:
           print(f'{item} is not in cart')

   def list_items(self):
       return self.items

   def __len__(self):
       return len(self.items)

   def __getitem__(self, index):
       return self.items[index]

   def __contains__(self, item):
       return item in self.items

   def __iter__(self):
       return iter(self.items)
```

وهنا كيف يمكنك استخدامها:

```python
cart = Cart()
cart.add('Laptop')
cart.add('Wireless mouse')
cart.add('Ergo keyboard')
cart.add('Monitor')

for item in cart:
   print(item, end=' ') # Laptop Wireless mouse Ergo keyboard Monitor

print(len(cart)) # 4
print(cart[3]) # Monitor

print('Monitor' in cart) # True
print('banana' in cart) # False

cart.remove('Ergo keyboard')

print(cart.list_items()) # ['Laptop', 'Wireless mouse', 'Monitor']

cart.remove('banana') # banana is not in cart
```
