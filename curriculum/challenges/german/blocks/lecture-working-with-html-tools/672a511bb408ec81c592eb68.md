# --description--

Wenn Sie Ihre Projekte erstellen, stoßen Sie häufig auf Probleme, bei denen Ihre Programme nicht wie erwartet funktionieren.

Programmierer bezeichnen solche Probleme oft als Bugs. Der Prozess, diese Bugs zu finden und zu beheben, wird Debugging genannt.

Um Ihren Code zu debuggen, müssen Sie einige Werkzeuge verwenden, die Ihr Browser bereitstellt.

Zwei wichtige Werkzeuge sind der DOM-Inspector und die Entwicklerwerkzeuge.

Der DOM-Inspector ermöglicht es Ihnen, die HTML-Struktur der Seite, auf der Sie sich befinden, zu untersuchen.

DOM steht für Document Object Model. Es ist eine baumartige Struktur, die die Elemente auf einer Seite darstellt. Mehr über den DOM lernen Sie in späteren Modulen.

Die Entwicklerwerkzeuge erlauben es Ihnen, das HTML, CSS und JavaScript der Seite, auf der Sie sich befinden, zu untersuchen.

Schauen wir uns ein HTML-Beispiel an, das einen kleinen Bug im Ankerelement enthält:

```html
<a href="https://www.freecodecamp.org/larn/">freeCodeCamp curriculum</a>
```

Wenn Sie auf den Link klicken, führt er zu einer 404-Seite. Eine 404-Seite ist eine Fehlerseite, die erscheint, wenn ein Benutzer versucht, auf eine Webseite zuzugreifen, die auf dem Server nicht existiert.

Der Link soll eigentlich zum freeCodeCamp-Curriculum führen.

Um herauszufinden, was das Problem sein könnte, können Sie die Entwicklerwerkzeuge verwenden.

Um die Entwicklerwerkzeuge in Ihrem Browser zu öffnen, können Sie mit der rechten Maustaste auf die Seite klicken und `Inspect` auswählen.

Sie können auch `Control Shift I` auf Ihrer PC-Tastatur oder `Command Option I` auf Ihrem Mac verwenden.

Wenn Sie die Entwicklerwerkzeuge in Google Chrome öffnen, sehen Sie mehrere Tabs. Der erste Tab heißt `Elements`. Dieser Tab zeigt Ihnen die HTML-Struktur der Seite, auf der Sie sich befinden.

Der zweite Tab heißt `Console`. Dieser Tab zeigt Ihnen alle Fehler, die auf der Seite auftreten könnten.

Wenn Sie einen defekten Link haben, können Sie in der Konsole die Fehlermeldungen für diesen defekten Link überprüfen. Die häufigste Meldung, die für den defekten Link angezeigt wird, ist der 404-Fehler. Der 404-Fehler zeigt an, dass die Seite nicht gefunden wurde.

Das zeigt uns, dass das Problem bei der URL im Ankerelement liegt. Wenn Sie den `href`-Wert untersuchen, sehen Sie, dass dort ein Tippfehler vorliegt.

Im Moment zeigt die Konsolenmeldung `/larn` mit einem 404-Fehler an, aber die korrekte URL sollte `/learn` sein. Wenn der Link korrigiert wird, funktioniert er wie erwartet.

Sie werden im Verlauf der Zertifizierung mehr über die Arbeit mit den Entwicklerwerkzeugen lernen, aber dies ist nur ein kurzes Beispiel, wie sie Ihnen beim Debuggen Ihres Codes helfen können.
