# --interactive--

## Валідація форм за допомогою JavaScript

- **Constraint Validation API**: Певні HTML-елементи, такі як `textarea` і `input`, відкривають доступ до API валідації обмежень. Цей API дозволяє переконатися, що значення, яке користувач ввів для цього елемента, проходить будь-яку HTML-валідацію, яку ви задали, наприклад мінімальну довжину або відповідність шаблону.
- **`checkValidity()` метод**: Цей метод повертає `true`, якщо елемент відповідає всім HTML-валідаціям (на основі його атрибутів), і `false`, якщо ні.

:::interactive_editor

```html
<form>
  <label>
    Email:
    <input
      id="email"
      type="email"
      required
      pattern=".+\.com$"
      placeholder="example@site.com"
    />
  </label>
</form>

<script>
  const input = document.getElementById("email");

  input.addEventListener("input", (e) => {
    if (!e.target.checkValidity()) {
      e.target.setCustomValidity("You must use a .com email.");
    } else {
      e.target.setCustomValidity("");
    }
  });
</script>
```

:::

- **`reportValidity()` метод**: Цей метод повідомляє браузеру, що `input` є недійсним.

:::interactive_editor

```html
<form>
  <label>
    Email:
    <input
      id="email2"
      type="email"
      required
      pattern=".+\.com$"
      placeholder="example@site.com"
    />
  </label>
</form>

<script>
  const input = document.getElementById("email2");

  input.addEventListener("input", (e) => {
    if (!e.target.checkValidity()) {
      e.target.reportValidity();
    }
  });
</script>
```

:::

- **`validity` властивість**: Ця властивість використовується для отримання або встановлення стану валідності елементів керування формою (наприклад, `<input>`, `<select>` тощо) і надає інформацію про те, чи відповідають вхідні дані користувача обмеженням, визначеним для цього елемента (наприклад, поля `required`, обмеження шаблону, максимальна довжина тощо).

:::interactive_editor

```html
<input
  id="age"
  type="number"
  min="18"
  placeholder="Enter age (18+)"
/>

<script>
  const input = document.getElementById("age");

  input.addEventListener("input", (e) => {
    console.log(e.target.validity);
  });
</script>
```

:::

- **`patternMismatch` властивість**: Вона буде `true`, якщо значення не відповідає заданому регулярному виразу.

## `preventDefault()` метод

- **Визначення**: Кожна подія, що відбувається в DOM, має певну поведінку за замовчуванням. Наприклад, подія click на прапорці за замовчуванням змінює його стан. Натискання пробілу на сфокусованій кнопці активує цю кнопку. Метод `preventDefault()` для цих об’єктів `Event` зупиняє цю поведінку.

:::interactive_editor

```html
<form id="form">
  <input type="text" placeholder="Try to submit" />
  <button type="submit">Submit</button>
</form>

<p id="status"></p>

<script>
  const form = document.getElementById("form");
  const status = document.getElementById("status");

  form.addEventListener("submit", (event) => {
    event.preventDefault();
    status.textContent = "Form submission prevented.";
  });
</script>
```

:::

## Відправлення форм

- **Визначення**: Існує три способи відправлення форми. Перший — коли користувач натискає кнопку у формі, яка має атрибут `type` зі значенням `submit`. Другий — коли користувач натискає клавішу `Enter` у будь-якому редагованому полі `input` у формі. Третій — через виклик JavaScript методів `requestSubmit()` або `submit()` елемента `form`.
- **Атрибут `action`**: Атрибут `action` повинен містити або URL, або відносний шлях для поточного домену. Це значення визначає, куди форма намагається надіслати дані — якщо атрибут `action` не встановлено, форма надішле дані на URL поточної сторінки.

```html
<form action="https://freecodecamp.org" method="GET">
  <input
    type="number"
    name="number"
    placeholder="Enter a number"
  />
  <button type="submit">Submit</button>
</form>
```

- **Атрибут `method`**: Цей атрибут приймає стандартний метод `HTTP`, наприклад `GET` або `POST`, і використовує цей метод при надсиланні запиту на URL дії. Якщо метод не встановлено, форма за замовчуванням використовує запит `GET`. Дані форми будуть URL-кодовані у вигляді пар `name=value` і додані до URL дії як параметри запиту.

```html
<form action="/data" method="POST">
  <input
    type="number"
    id="input"
    placeholder="Enter a number"
    name="number"
  />
  <button type="submit">Submit</button>
</form>
```

- **Атрибут `enctype`**: Елемент `form` приймає атрибут `enctype`, який визначає тип кодування для даних. Цей атрибут приймає лише три значення: `application/x-www-form-urlencoded` (за замовчуванням, надсилає дані як URL-кодоване тіло форми), `text/plain` (надсилає дані у вигляді простого тексту у парах `name=value`, розділених новими рядками) або `multipart/form-data`, який спеціально призначений для обробки форм із завантаженням файлів.
