# --description--

アクションが作成されてディスパッチされた後、Reduxストアはそのアクションにどう対応するかを知る必要があります。これは`reducer`関数の役割です。Reduxのリデューサーは、アクションに応じて状態を変更する責任を持ちます。`reducer`は`state`と`action`を引数に取り、常に新しい`state`を返します。これがリデューサーの**唯一の**役割であることが重要です。副作用はなく、APIエンドポイントを呼び出すことも隠れた動作もありません。リデューサーは単に状態とアクションを受け取り、新しい状態を返す純粋な関数です。

Reduxのもう一つの重要な原則は、`state`が読み取り専用であることです。つまり、`reducer`関数は**常に**`state`の新しいコピーを返し、状態を直接変更してはいけません。Reduxは状態の不変性を強制しませんが、リデューサー関数のコード内でそれを守る責任があります。これは後のチャレンジで練習します。

# --instructions--

コードエディターには前の例と`reducer`関数の開始部分が用意されています。`reducer`関数の本体を完成させてください。もし受け取ったアクションのタイプが`'LOGIN'`なら、`login`が`true`に設定された状態オブジェクトを返してください。そうでなければ、現在の`state`を返します。現在の`state`とディスパッチされた`action`はリデューサーに渡されるので、アクションのタイプは`action.type`で直接アクセスできます。

# --hints--

`loginAction`関数を呼び出すと、typeプロパティが文字列`LOGIN`に設定されたオブジェクトが返されるはずです。

```js
assert(loginAction().type === 'LOGIN');
```

ストアは`login`プロパティが`false`に設定されたオブジェクトで初期化する必要があります。

```js
assert(store.getState().login === false);
```

`loginAction`をディスパッチすると、ストアの状態の`login`プロパティが`true`に更新されます。

```js
assert(
  (function () {
    const initialState = store.getState();
    store.dispatch(loginAction());
    const afterState = store.getState();
    return initialState.login === false && afterState.login === true;
  })()
);
```

アクションのタイプが`LOGIN`でない場合、ストアは現在の状態を返すべきです。

```js
assert(
  (function () {
    store.dispatch({ type: '__TEST__ACTION__' });
    let afterTest = store.getState();
    return typeof afterTest === 'object' && afterTest.hasOwnProperty('login');
  })()
);
```
