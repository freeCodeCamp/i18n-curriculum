# --interactive--

在前一課中，你學會了如何使用 `arguments` 物件，該物件是一個類陣列物件，包含傳入函式的引數值。

:::interactive_editor

```js
function logArgs() {
  for (const arg of arguments) {
    console.log(arg);
  }
}

logArgs(1, 2, 3);
// result:
// 1
// 2
// 3
```

:::

雖然這是存取和操作來自函式的可變引數集合的有效方式，但現代 JavaScript 應用程式會改用剩餘參數語法。

這裡是一個更新的範例，使用剩餘參數語法取代 `arguments` 物件：

:::interactive_editor

```js
function logArgs(...args) {
  for (const arg of args) {
    console.log(arg);
  }
}

logArgs(1, 2, 3);
// result:
// 1
// 2
// 3
```

:::

此更新範例不再直接取用 `arguments` 物件。取而代之的是，函式定義的最後一個參數前面有三個點。這會使此剩餘參數被放置在一個 `Array` 物件中。你可以將此剩餘參數命名為任何你想要的名稱。只要確保它是函式定義中的最後一個參數，如下所示：

```js
function exampleFunction(a, b, ...restOfArgs) {
  // some code here
}

function anotherFunction(x, y, ...theArgs) {
  // some code here
}
```

在處理剩餘參數語法時還有一些限制。其中一個限制是函式定義只能有一個剩餘參數。因此，以下範例會被視為無效。

```js
// Won't work

function badFunction(...args, ...moreArgs) {
  // some code here
}
```

下一個限制是，在剩餘參數 `function exampleFunction(a, b, ...restOfArgs, )` 之後不允許有尾隨逗號。

另一個限制是 rest 參數不能有預設值。否則會發出 `SyntaxError`。

```js
function badFunction(...args = [1,2]){
  // some code here
}
```

那麼 `arguments` 物件和 `rest` 參數之間有什麼差異呢？

一個主要的差異是 `arguments` 物件不是一個真正的陣列，因此它不支援像 `includes`、`pop` 和 `push` 這些方法。但剩餘參數是 `Array` 實體，所以你可以直接使用有效的內建陣列方法，而不需要先將它轉換成真正的陣列。

:::interactive_editor

```js
function hasCat(...args) {
  return args.includes("cat");
}

console.log(hasCat("dog", "chicken", "cat")); // true
console.log(hasCat("dog", "chicken", "horse")); // false
```

:::
