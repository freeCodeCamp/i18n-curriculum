# --description--

버전 관리를 실제로 사용하고 GitHub 같은 사이트를 이용하려면 먼저 Git이 설치되어 있어야 합니다.

터미널에서 다음 명령어를 실행해 Git이 이미 설치되어 있는지 확인할 수 있습니다:

```sh
git --version
```

버전 번호가 보이면 Git이 설치된 것입니다. 그렇지 않으면 설치해야 합니다.

리눅스 시스템에서는 대부분의 배포판에 Git이 기본으로 설치되어 있습니다. 만약 Git이 설치되어 있지 않다면, `sudo apt-get install git` 또는 `sudo pacman -S git` 같은 패키지 관리자 명령어로 설치할 수 있습니다.

Mac 사용자는 Homebrew를 통해 `brew install git` 명령어로 Git을 설치하거나 Git 공식 웹사이트에서 실행 파일 설치 프로그램을 다운로드할 수 있습니다.

Windows에서는 Git 공식 웹사이트에서 실행 파일 설치 프로그램을 다운로드할 수 있습니다. 또는 Chocolatey를 설정했다면 PowerShell에서 `choco install git.install` 명령어를 실행할 수 있습니다. Windows에서는 Unix와 비슷한 셸 환경을 사용하려면 GitBash도 다운로드하는 것이 좋습니다.

설치가 제대로 되었는지 확인하려면 터미널에서 `git --version` 명령어를 다시 실행하세요.

Git을 설치한 후에는 몇 가지 설정을 변경해야 합니다. 이 설정은 컴퓨터에서 한 번만 하면 되고, 업그레이드할 때마다 계속 변경할 필요는 없습니다.

`git config` 명령어는 Git이 컴퓨터에서 어떻게 작동할지 결정하는 설정 변수를 지정하는 데 사용됩니다. 현재 설정 변수와 저장 위치를 보려면 다음 명령어를 실행하세요:

```sh
git config --list --show-origin
```

지금은 Git을 처음 설치했다면 시스템 수준 설정만 보일 것입니다. 이전에 Git을 설치했다면 사용자 이름과 이메일에 대한 사용자 설정이 보일 수도 있습니다.

사용자 이름을 설정하려면 다음 명령어를 실행하세요:

```sh
git config --global user.name "Jane Doe"
```

여기서 `--global` 플래그는 Git을 사용하는 시스템 내 모든 프로젝트에 대해 사용자 이름을 설정하는 데 사용됩니다. 특정 프로젝트에서 사용자 이름을 재정의하려면 `--global` 플래그 없이 해당 프로젝트 디렉터리에서 명령어를 실행하면 됩니다.

사용자 이메일 주소를 설정하려면 다음 명령어를 실행하세요:

```sh
git config --global user.email janedoe@example.com
```

사용자 이름과 이메일을 설정하는 것은 중요합니다. Git이 프로젝트에서 커밋할 때마다 이 정보를 사용하기 때문입니다. 커밋에 대해서는 이후 수업에서 더 자세히 배우게 됩니다.

또한 Git이 사용할 선호하는 편집기를 설정할 수도 있습니다. 예를 들어 Emacs를 선호 편집기로 설정하는 방법은 다음과 같습니다:

```sh
git config --global core.editor emacs
```

다른 옵션으로는 Vim, Nano, VS Code 등이 있습니다. 예를 들어 VS Code를 편집기로 사용한다면 다음과 같이 Git 설정을 할 수 있습니다:

```sh
git config --global core.editor "code --wait"
```

선호 편집기를 설정하지 않으면 Git은 시스템 기본 편집기를 사용합니다.

Windows를 사용 중이고 편집기를 변경하려면 실행 파일의 전체 경로를 다음과 같이 지정해야 합니다:

```sh
git config --global core.editor "'C:/Program Files/Notepad++/notepad++.exe' -multiInst -notabbar -nosession -noPlugin"
```

설정 목록을 보려면 다음 명령어를 실행하세요:

```sh
git config --list
```

이제 Git을 설치하고 설정했으니 프로젝트에 버전 관리를 추가할 준비가 되었습니다.
