# --description--

Potresti aver visto prima collegamenti come `/public/logo.png`, `./script.js` o `../styles.css`. Ma cosa significano questi tipi speciali di collegamenti? Si chiamano percorsi di file. Ci sono tre sintassi chiave da conoscere. La prima è lo slash, che può essere una barra rovesciata (`\`) o uno slash normale (`/`) a seconda del sistema operativo. La seconda è il punto singolo (`.`). Infine, abbiamo il doppio punto (`..`).

Lo slash è noto come "separatore di percorso". Serve a indicare una separazione nel testo tra nomi di cartelle o file. È così che il computer sa che `naomis-files/` indica una directory con quel nome, mentre `naomis/files/` indica una directory `files` dentro la directory `naomis`.

Un punto singolo indica la directory corrente, mentre due punti indicano la directory genitore. Di solito vedrai un punto singolo usato per assicurarti che il percorso venga riconosciuto come percorso relativo. Ricorda che in una lezione precedente hai imparato la differenza tra percorsi relativi e assoluti.

I doppi punti, invece, sono molto più comuni per consultare file esterni alla directory di lavoro corrente.

Per esempio, dato questo albero dei file:

```sh
my-app/
├─ public/
│  ├─ favicon.ico
│  ├─ index.html
├─ src/
│  ├─ index.css
│  ├─ index.js
```

Se il tuo file `public/index.html` dovesse caricare il file `favicon.ico`, useresti un percorso relativo con un punto singolo per consultare la directory corrente: `./favicon.ico`. Se il tuo file `public/index.html` dovesse caricare il file `index.css`, useresti un percorso relativo con doppi punti per navigare prima nella directory genitore `my-app`, poi nella directory `src` e infine al tuo file: `../src/index.css`.
