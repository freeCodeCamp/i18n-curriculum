# --description--

이벤트 버블링 또는 전파는 이벤트가 발생했을 때 어떻게 부모 객체로 "버블처럼 올라가는지"를 의미합니다. 예를 들어, 다음 코드를 살펴보세요:

```html
<p>
  <span>Click me~!</span>
</p>
```

여기서 `p` 요소는 `span` 요소의 부모로 간주됩니다.

`span` 요소를 클릭하면, 지시대로 하면 `span` 요소가 `click` 이벤트의 대상이 됩니다. 하지만 그 이벤트는 부모에게도 버블링되어 `p` 요소가 필요에 따라 그 이벤트를 받고 처리할 수 있습니다.

그럼 이게 실제로 무슨 뜻일까요? `p` 요소에 이벤트 리스너를 붙일 수 있습니다:

```js
const p = document.querySelector("p");
p.addEventListener("click", (event) => {
  console.log(event.target);
});
```

그다음 `span` 요소를 클릭하면 콘솔에 `Click me~!` 텍스트가 기록되는 것을 볼 수 있습니다.

이벤트가 부모인 `p` 요소로 전파되어, 이벤트 리스너에서 이벤트의 대상을 표시하기 위해 처리합니다.

대상이 여전히 `span` 요소인 점에 주목하세요. 이는 `span` 요소가 최초 클릭을 받았기 때문입니다.

작동 방식을 확실히 이해하기 위해 코드를 확장해 봅시다:

```js
const p = document.querySelector("p");
const span = document.querySelector("span");
p.addEventListener("click", (event) => {
  console.log("P listener: ");
  console.log(event.target);
});
span.addEventListener("click", (event) => {
  console.log("Span listener: ");
  console.log(event.target);
});
```

이벤트가 어떻게 버블링되는지 감을 잡으라고, `span` 요소를 클릭한 후 콘솔에 나타나는 내용은 다음과 같습니다:

```html
"Span listener: "
<span>Click me~!</span>
"P listener: "
<span>Click me~!</span>
```

이번에는 `stopPropagation()`으로 이벤트 전파를 막으면 어떻게 되는지 봅시다. `span`의 이벤트 리스너에서 호출할 것입니다:

```js
const p = document.querySelector("p");
const span = document.querySelector("span");
p.addEventListener("click", (event) => {
  console.log("P listener: ");
  console.log(event.target);
});
span.addEventListener("click", (event) => {
  console.log("Span listener: ");
  console.log(event.target);
  event.stopPropagation();
});
```

그리고 다시 `span`를 클릭하세요:

```html
"Span listener: "
<span>Click me~!</span>
```

이번에는 `p` 리스너가 작동하지 않습니다. 이벤트가 `p` 요소에 도달하지 못했는데, 자식인 `span` 요소에서 처리되는 동안 전파를 멈추라고 했기 때문입니다.

이벤트 위임은 반대 개념으로 생각할 수 있습니다. 캡처된 이벤트를 받아 다른 요소에 위임하는 과정입니다.

코드로 돌아가서 `span` 요소를 클릭하면 빨간색으로 바뀌도록 업데이트해 봅시다:

```js
const p = document.querySelector("p");
const span = document.querySelector("span");
p.addEventListener("click", (event) => {});
span.addEventListener("click", (event) => {
  event.target.style.color = "red";
});
```

그런데 `span` 요소가 스무 개나 있다면 어떨까요? 아니면 JavaScript로 동적으로 `span` 요소를 더 만든다면요?

모든 `span` 요소에 이벤트 리스너를 붙이는 대신, 부모인 `p` 요소에 리스너를 붙여서 모두 처리할 수 있습니다. 즉, `span` 클릭 처리를 부모인 `p` 요소에 위임하는 것입니다.

코드는 이제 다음과 비슷할 수 있습니다:

```js
const p = document.querySelector("p");
p.addEventListener("click", (event) => {
  event.target.style.color = "red";
});
```

`span` 요소에는 더 이상 리스너가 전혀 붙어 있지 않은 점에 주목하세요. 이벤트 처리를 `p` 요소에 제대로 위임한 것입니다. 그런데 제대로 작동할까요?

몇 개의 `span` 요소를 더 만들어서 확인해 봅시다:

```html
<p>
  <span>Click me~!</span>
  <span>Click me~!</span>
  <span>Click me~!</span>
  <span>Click me~!</span>
</p>
```

이제 `span`를 클릭할 때마다 그 요소의 텍스트가 빨간색으로 바뀝니다.

이렇게 단 하나의 이벤트 리스너로 `click` 이벤트가 `span` 요소에서 부모인 `p`로 버블링되고, 그 `click` 이벤트의 로직을 `p` 요소에 위임한 것입니다.

이벤트 전파와 위임은 특히 표 같은 중첩된 요소가 많을 때 복잡할 수 있습니다. 여기서 작성한 코드를 더 탐색하고 실험해 보길 권장합니다.
