# --description--

지금까지 HTML, CSS, JavaScript 등 여러 언어로 많은 작은 프로젝트를 만들어 왔습니다. 그런데 어떻게 프로젝트가 제대로 작동하는지 알 수 있을까요?

한 가지 방법은 수동 테스트라고 하는 것입니다. 수동 테스트는 테스터가 애플리케이션의 각 부분을 직접 살펴보고 모든 기능이 제대로 작동하는지 확인하는 과정입니다. 테스트 과정에서 버그가 발견되면 테스터가 소프트웨어 팀에 보고하여 수정할 수 있게 합니다.

애플리케이션을 수동으로 테스트하는 것이 중요하지만, 종종 이것만으로는 충분하지 않습니다. 실제 애플리케이션은 여러 컴포넌트와 서비스로 구성되어 복잡한 경우가 많기 때문입니다. 그래서 수동 테스트에만 의존하면 많은 버그가 표면 아래에 숨겨져 있어 애플리케이션의 사용성과 유지보수성에 악영향을 줄 수 있습니다.

이때 자동화 테스트가 필요합니다. 자동화 테스트는 애플리케이션을 직접 클릭하지 않고도 예상대로 작동하는지 확인하는 별도의 테스트 코드를 작성하는 것입니다. 수동 테스트와 자동화 테스트 모두 테스트 대상에 따라 여러 유형으로 나눌 수 있습니다.

앞으로 몇 개의 수업에서 단위 테스트, 종단 간 테스트, 기능 테스트 등 업계에서 사용하는 다양한 테스트 유형을 다룰 예정입니다.

애플리케이션에 대한 테스트를 작성하면 코드가 의도한 대로 작동하는지 확인할 수 있습니다. 개발자 직무에서 꼭 필요하므로 테스트의 기본을 익히는 것이 중요합니다.
