# --description--

No Express, `express.Router()` é um recurso poderoso que ajuda a organizar suas rotas em partes menores e modulares. Isso é especialmente útil conforme sua aplicação cresce.

Em vez de definir todas as suas rotas em um único arquivo grande, você pode dividi-las em arquivos separados usando roteadores. Isso mantém seu código mais fácil de ler, manter e escalar.

O método `express.Router()` cria um novo objeto roteador.

Um roteador funciona como uma mini aplicação Express. Ele pode lidar com suas próprias rotas e middleware, mas não é uma aplicação Express completa. Depois de definir rotas em um roteador, você pode montá-lo na sua aplicação principal usando `app.use()`.

Em outras palavras, um roteador permite agrupar rotas relacionadas e depois integrá-las na sua aplicação principal.

Usar roteadores tem várias vantagens:

**Separação de responsabilidades**

Você pode definir rotas em arquivos separados, o que melhora a organização e a legibilidade da sua aplicação.

**Reutilização de código**  
Roteadores podem ser reutilizados em diferentes partes de uma aplicação ou até mesmo em projetos diferentes.

**Arquivo principal mais limpo**  
Em vez de encher seu arquivo principal `app.js` ou `server.js` com muitas definições de rota, você pode movê-las para roteadores e manter seu arquivo principal focado em configuração e inicialização.

Vamos ver um exemplo simples onde usamos `express.Router()` para definir rotas para o caminho `/users`.

Primeiro, aqui está o arquivo `app.js`:

```javascript
const express = require("express");
const app = express();
const port = 3000;

// Importing the user routes
const userRoutes = require("./userRoutes");

// Using the userRoutes router for /users path
app.use("/users", userRoutes);

app.listen(port, () => {
  console.log(`Server running at http://localhost:${port}`);
});
```

E aqui está `userRoutes.js`:

```javascript
const express = require("express");
const router = express.Router();

// GET request for /users
router.get("/", (req, res) => {
  res.send("List of users");
});

// POST request for /users
router.post("/", (req, res) => {
  res.send("Creating a new user");
});

// GET request for /users/:id
router.get("/:id", (req, res) => {
  const userId = req.params.id;
  res.send(`User details for user with id: ${userId}`);
});

module.exports = router;
```

Agora vamos analisar mais de perto como tudo funciona.

No `app.js`, importamos `express` e criamos uma instância da aplicação Express, atribuindo-a a uma variável chamada `app`.

Em seguida, importamos o roteador `userRoutes` do `userRoutes.js` e usamos `app.use("/users", userRoutes)` para montar o roteador. Isso informa ao Express que, se uma requisição começar com `/users`, ela deve ser passada para o roteador `userRoutes`.

Depois, no `userRoutes.js`, criamos um novo roteador usando `express.Router()` e definimos rotas usando `router.get()`, `router.post()` e `router.get("/:id")`.

Note que as rotas definidas aqui são relativas a `/users` porque é onde o roteador está montado. Então:

*   `router.get("/")` → `GET /users`: responde com uma lista de usuários
    
*   `router.post("/")` → `POST /users`: responde com uma mensagem sobre criar um novo usuário
    
*   `router.get("/:id")` → `GET /users/:id`: responde com detalhes do usuário para um dado id
    

Usar `express.Router()` mantém seu arquivo principal limpo e focado em configuração. Todas as rotas relacionadas a usuários ficam agrupadas em `userRoutes.js`.

Conforme sua aplicação cresce, você pode criar roteadores adicionais para outras seções, como `/products`, `/orders` ou `/auth`.

Cada seção fica organizada em seu próprio arquivo, facilitando a manutenção e a escalabilidade da aplicação.
