# --description--

見出し（`h1` から `h6` 要素）は、コンテンツに構造とラベル付けを提供する重要なタグです。スクリーンリーダーはページ上の見出しだけを読み上げる設定ができるため、ユーザーは要約を得られます。つまり、マークアップ内の見出しタグは意味論的な意味を持ち、互いに関連していることが重要であり、単にサイズの値で選ばれるべきではありません。

*意味論的な意味* とは、コンテンツの周りに使うタグがその情報の種類を示すことを意味します。

もし、導入、本文、結論からなる論文を書いているとしたら、結論を本文のサブセクションとしてアウトラインに入れるのはあまり意味がありません。結論は独立したセクションであるべきです。同様に、ウェブページの見出しタグも順序通りに配置され、コンテンツの階層関係を示す必要があります。

同じ（またはより高い）ランクの見出しは新しい暗黙のセクションを開始し、より低いランクの見出しは前の見出しのサブセクションを開始します。

例として、`h2` 要素の後に `h4` 要素でラベル付けされた複数のサブセクションが続くページは、スクリーンリーダーユーザーを混乱させます。6つの選択肢があるため、ブラウザで見た目が良いからといってタグを使いたくなりますが、相対的なサイズはCSSで調整できます。

最後に、各ページには必ず1つ（かつ1つだけ）の `h1` 要素があり、これはコンテンツの主題を示します。この見出しと他の見出しは、検索エンジンがページのトピックを理解するためにも部分的に使われます。

# --instructions--

Camper Cat は忍者になることに特化したページをサイトに作りたいと思っています。彼のマークアップがコンテンツに意味論的な意味を与え、セクションの親子関係を正しく示すように見出しを修正してください。すべての `h5` タグを、`h2` のサブセクションであることを示す適切な見出しレベルに変更してください。`h3` タグを使ってください。

# --hints--

コードには6つの `h3` 要素があるはずです。

```js
assert.lengthOf(document.querySelectorAll('h3') , 6);
```

コードには6つの `h3` 終了タグがあるはずです。

```js
assert.lengthOf((code.match(/\/h3/g) || []) ,6);
```

コードには `h5` 要素があってはいけません。

```js
assert.isEmpty(document.querySelectorAll('h5'));
```

コードには `h5` 終了タグがあってはいけません。

```js
assert.notMatch(code, /\/h5/);
```
