## `Audio` конструктор і загальні методи

- **Визначення**: конструктор `Audio`, як і інші конструктори, є спеціальною функцією, яку викликають за допомогою ключового слова `new`. Він повертає `HTMLAudioElement`, який ви можете використовувати для відтворення аудіо для користувача або додати до DOM, щоб користувач міг керувати ним самостійно. Коли ви викликаєте конструктор, можна необов’язково передати URL як (єдиний) аргумент. Цей URL має вказувати на джерело аудіофайлу, який ви хочете відтворити. Або, якщо потрібно динамічно змінити джерело, можна призначити URL властивості `src` повернутого аудіоелемента.
- **Метод `play()`**: цей метод використовується з елементами `audio` або `video` для початку відтворення медіа.

:::interactive_editor

```html
<audio id="audio" controls>
  <source src="https://www.w3schools.com/html/horse.mp3" type="audio/mpeg">
</audio>

<button id="playBtn">Play Audio</button>

<script>
  const audio = document.getElementById("audio");
  const playBtn = document.getElementById("playBtn");

  playBtn.addEventListener("click", () => {
    audio.play();
  });
</script>
```

:::

- **Метод `pause()`**: цей метод використовується з елементами `audio` або `video` для призупинення відтворення медіа. Він зберігає поточну позицію, тому якщо викликати `play()`, відтворення почнеться з цієї позиції.

:::interactive_editor

```html
<audio id="myAudio" controls>
  <source src="https://www.w3schools.com/html/horse.mp3" type="audio/mpeg">
</audio>

<br>

<button id="pauseBtn">Pause Audio</button>

<script>
  document.addEventListener("DOMContentLoaded", () => {
    const audio = document.getElementById("myAudio");
    const pauseBtn = document.getElementById("pauseBtn");

    pauseBtn.addEventListener("click", () => {
      audio.pause();
    });
  });
</script>
```

:::

- **Метод `addTextTrack()`**: цей метод дозволяє вказати текстову доріжку, пов’язану з медіаелементом — це особливо корисно для додавання субтитрів до відео.
- **Метод `fastSeek()`**: цей метод дозволяє перемістити позицію відтворення на певний час у межах медіа.

Конструктор `Audio()` має властивості так само, як і методи. До цих властивостей належать:

- `currentTime`: для отримання поточного часу відтворення аудіо
- `loop`: для безперервного відтворення аудіо шляхом автоматичного перезапуску після досягнення кінця
- `muted`: для вимкнення звуку медіаелемента незалежно від налаштувань гучності

## Різні формати аудіо та відео

- **MIME-тип**: MIME-тип, що означає Multipurpose Internet Mail Extensions, — це стандартизований спосіб програмно вказати тип файлу. MIME-тип може повідомити застосунку, наприклад вашому браузеру, як обробляти конкретний файл. У випадку аудіо та відео MIME-тип вказує, що це мультимедійний формат, який можна вставити у вебсторінку.
- **Елемент `source`**: використовується для вказання типу файлу та джерела — і може містити кілька різних типів за допомогою кількох елементів source. Коли ви це робите, браузер визначає найкращий формат для поточного середовища користувача.
- **MP3**: це тип цифрового формату файлу, який використовується для зберігання музики, аудіо або звуку. Це стисла версія звукового запису, що зменшує розмір файлу, тому його легше зберігати та передавати. MP3 має найширшу підтримку в браузерах і MIME-тип `audio/mpeg`.
- **MP4**: MP4 — це тип цифрового формату файлу, який використовується для зберігання відео та аудіо. Він слугує контейнером, що містить і відео (зображення), і звук (музику або мову) в одному файлі. MP4 може мати MIME-тип `audio/mp4` АБО `video/mp4`, залежно від того, чи це відеофайл, чи лише аудіо.
- Інші формати — це `WMV`, який асоціюється із застосунком Windows Media Player, `OGG`, `MKV`, `AVI` та інші.

## кодеки

- **Визначення**: кодек, скорочено від "кодувальник/декодувальник", — це алгоритм або програмне забезпечення, яке може конвертувати аудіо та відео між аналоговими та цифровими форматами. Кодеки можна вказувати як частину MIME-типу. Основний синтаксис для визначення кодека — додати крапку з комою після типу медіа, потім `codecs=` і кодек.

## API HTMLMediaElement

- **Визначення**: API `HTMLMediaElement` використовується для керування поведінкою аудіо- та відеоелементів на вашій сторінці. Він розширює базовий інтерфейс `HTMLElement`, тому ви маєте доступ до базових властивостей, методів, а також до корисних подій, таких як `waiting`, `ended`, `canplay`, `canplaythrough` та інших. Прикладами методів є `play()`, `fastSeek()`, `pause()` і `canPlayType()`, які перевіряють, чи браузер, ймовірно, зможе відтворити аудіофайл.

## API Media Capture and Streams

- **Визначення**: API Media Capture and Streams, або MediaStream API, використовується для захоплення аудіо та відео з вашого пристрою. Щоб використати цей API, потрібно створити об’єкт `MediaStream`. Ви могли б зробити це за допомогою конструктора, але він не був би пов’язаний із апаратним забезпеченням користувача. Натомість властивість `mediaDevices` об’єкта `global` navigator має метод `getUserMedia()`, який ви можете використати.

```js
window.navigator.mediaDevices.getUserMedia({
  audio: true,
  video: {
    width: {
      min: 1280,
      ideal: 1920,
      max: 3840
    },
    height: {
      min: 720,
      ideal: 1080,
      max: 2160
    }
  }
});
```

## API Screen Capture

- **Визначення**: API Screen Capture використовується для запису екрану користувача. Цей API відкривається викликом методу `getDisplayMedia()` об’єкта `mediaDevices` і використанням повернутого медіапотоку.

## API MediaStream Recording

- **Визначення**: API MediaStream Recording працює разом із API MediaStreams, дозволяючи записувати MediaStream (або навіть `HTMLMediaElement` безпосередньо).

## API Media Source Extensions

- **тема**: API Media Source Extensions дозволяє безпосередньо передавати відеопотік із вебкамери користувача до відеоелемента за допомогою властивості `srcObject`.

## API Web Audio

- **Визначення**: API Web Audio забезпечує все, що звучить у вебі. Цей API включає важливі об’єкти, як `AudioBuffer` (представляє буфер, що містить аудіодані) або `AudioContext`, який використовується для представлення графа обробки аудіо.
