## 永続的ストレージ

- **定義**: 永続的ストレージとは、電源が切れたりデバイスが再起動された後でもデータが利用可能な状態で保存される方法を指します。

## 作成、読み取り、更新、削除（CRUD）

- **作成**: 新しいデータを作成するプロセスを指します。例えば、ウェブアプリでユーザーがブログに新しい投稿を追加する場合です。
- **読み取り**: データベースからデータを取得する操作です。例えば、ブログ投稿を閲覧したりウェブサイトでプロフィールを表示するときに、データベースに保存されたデータを取得して表示しています。
- **更新**: データベース内の既存データを変更することです。例えば、ブログ投稿を編集したりプロフィール情報を更新する場合です。
- **削除**: データベースからデータを削除する操作です。例えば、ブログ投稿やアカウントを削除するときに行われます。

## HTTPメソッド

- **定義**: HTTPはHypertext Transfer Protocolの略で、ウェブ上のデータ通信の基盤です。HTTPメソッドはウェブ上のリソースに対して実行できる操作を定義しています。一般的なメソッドにはGET、POST、PUT、PATCH、DELETEがあります。
- **`GET` メソッド**: サーバーからデータを取得するために使われます。
- **`POST` メソッド**: 新しいリソースを作成するためにサーバーにデータを送信する際に使われます。
- **`PUT` メソッド**: リソースを完全に置き換えて更新するために使われます。
- **`PATCH` メソッド**: リソースを部分的に更新するために使われます。
- **`DELETE` メソッド**: データベースからレコードを削除するために使われます。

## `localStorage` と `sessionStorage` プロパティ

- **Web Storage API**: このAPIはブラウザ内にキーと値のペアを保存する仕組みを提供し、開発者が異なるページのリロードやセッションをまたいで情報を保存できるようにします。Web Storage APIの主なコンポーネントは`localStorage` と `sessionStorage` プロパティです。
- **`localStorage` プロパティ**: `localStorage` はブラウザのウィンドウが閉じられたりページが更新された後でもデータを保持できるWeb Storage APIの部分です。このデータはアプリケーションやユーザーによって明示的に削除されるまで利用可能です。
- **`localStorage.setItem()` メソッド**: このメソッドは`localStorage` にキーと値のペアを保存するために使われます。

```js
localStorage.setItem('username', 'Jessica');
```

- **`localStorage.getItem()` メソッド**: このメソッドは`localStorage` から指定されたキーの値を取得するために使われます。

```js
localStorage.setItem('username', 'codingRules');

let username = localStorage.getItem('username');
console.log(username); // codingRules
```

- **`localStorage.removeItem()` メソッド**: このメソッドはキーを使って`localStorage` から特定のアイテムを削除するために使われます。

```js
localStorage.removeItem('username');
```

- **`localStorage.clear()` メソッド**: このメソッドは`localStorage` に保存されたすべてのデータをクリアするために使われます。

```js
localStorage.clear();
```

- **`sessionStorage` プロパティ**: 現在のセッションのみ持続し、ブラウザのタブやウィンドウが閉じられるとクリアされるデータを保存します。
- **`sessionStorage.setItem()` メソッド**: このメソッドは`sessionStorage` にキーと値のペアを保存するために使われます。

```js
sessionStorage.setItem('cart', '3 items');
```

- **`sessionStorage.getItem()` メソッド**: このメソッドは`sessionStorage` から指定されたキーの値を取得するために使われます。

```js
sessionStorage.setItem('cart', '3 items');

let cart = sessionStorage.getItem('cart');
console.log(cart); // '3 items'
```

- **`sessionStorage.removeItem()` メソッド**: このメソッドはキーを使って`sessionStorage` から特定のアイテムを削除するために使われます。

```js
sessionStorage.removeItem('cart');
```

- **`sessionStorage.clear()` メソッド**: このメソッドは`sessionStorage` に保存されたすべてのデータをクリアするために使われます。

```js
sessionStorage.clear();
```

## クッキーの扱い方

- **定義**: クッキーはウェブクッキーやブラウザクッキーとも呼ばれ、サーバーがユーザーのウェブブラウザに送信する小さなデータ片です。これらのクッキーはユーザーのデバイスに保存され、後続のリクエストとともにサーバーに送信されます。HTTPがステートレスなプロトコルであるため、クッキーはウェブアプリが状態を維持しユーザー情報を記憶するのに不可欠です。
- **セッションクッキー**: これらのクッキーはユーザーのウェブサイトでのセッションの間だけ持続します。ユーザーがブラウザやタブを閉じるとセッションクッキーは削除されます。通常、訪問中のログイン状態を維持するために使われます。
- **セキュアクッキー**: これらのクッキーはHTTPS経由でのみ送信され、通信途中で攻撃者に傍受されることを防ぎます。
- **HttpOnlyクッキー**: これらのクッキーはブラウザで実行されるJavaScriptからアクセスや変更ができず、クロスサイトスクリプティング（XSS）攻撃に対してより安全です。
- **Set-Cookieヘッダー**: ウェブサイトを訪問すると、サーバーはHTTPレスポンスでSet-Cookieヘッダーを送信できます。このヘッダーはブラウザに特定の情報を持つクッキーを保存するよう指示します。例えば、次回訪問時にユーザーを識別するためのユニークIDを保存することがあります。
JavaScriptでクッキーを手動で設定するには`document.cookie` を使います。

```js
document.cookie = "organization=freeCodeCamp; expires=Fri, 31 Dec 2021 23:59:59 GMT; path=/";
```

クッキーを削除するには、有効期限を過去の日時に設定してください。

```js
document.cookie = "username=; expires=Thu, 01 Jan 1970 00:00:00 GMT; path=/";
```

## Cache API

- **定義**: キャッシュはファイルのコピーを一時的な保存場所に保存し、より速くアクセスできるようにするプロセスです。Cache APIはネットワークリクエストとレスポンスを保存し、ウェブアプリの効率を高め、オフラインでも動作させるために使われます。これはService Worker APIの一部であり、不安定または遅いネットワーク環境でも動作するプログレッシブウェブアプリ（PWA）を作るのに重要です。
Cache APIはRequestとResponseオブジェクトを保存するストレージ機構です。サーバーへのリクエストが行われると、アプリはレスポンスを保存し、後で新たなネットワークリクエストを行わずにキャッシュから取得できます。これにより読み込み時間が短縮され、帯域幅が節約され、ユーザー体験が向上します。
- **キャッシュストレージ**: HTTPリクエストとそれに対応するレスポンスのキーと値のペアを保存します。これにより、以前にリクエストしたリソースを効率的に取得でき、再訪時にネットワークから取得する必要が減りパフォーマンスが向上します。
- **Cache-Control**: 開発者はキャッシュされたリソースをどのくらいの期間保持するか、再検証するか、直接キャッシュから提供するかを定義できます。
- **オフライン対応**: Cache APIを使うことでオフラインファーストのウェブアプリを作成できます。例えば、PWAはユーザーがネットワークから切断されている場合でもキャッシュされた資産を提供できます。

## ネガティブパターンとクライアントサイドストレージ

- **過剰なトラッキング**: これはクッキー、ローカルストレージ、セッションストレージなどのクライアントサイドストレージに、明確な同意や正当な理由なしに過剰なユーザーデータを収集・保存する行為を指します。複数のサイトやセッションにわたってユーザーの行動や好み、インタラクションを追跡し、ユーザーのプライバシーを侵害する場合があります。
- **ブラウザフィンガープリンティング**: クッキーや従来のトラッキング方法に頼らず、デバイスやブラウザの固有の特徴に基づいて個々のユーザーを追跡・識別する手法です。クッキーはユーザーのデバイスにローカル保存されますが、フィンガープリンティングはユーザーのブラウザセッションの特徴を収集して独自の「指紋」を作成します。
- **LocalStorageにパスワードを設定すること**: これは明らかに悪いパターンの一例ですが、パスワードなどの機密データをローカルストレージに保存することはセキュリティリスクがあります。ローカルストレージは暗号化されておらず簡単にアクセスできるため、機密データを保存してはいけません。

## IndexedDB

- **定義**: IndexedDBはブラウザ内で構造化データを保存するために使われます。これは最新のウェブブラウザに組み込まれており、ウェブアプリがJavaScriptオブジェクトを効率的に保存・取得できます。

## キャッシュ／サービスワーカー

- **定義**: サービスワーカーはウェブページとは別にバックグラウンドで動作するスクリプトです。ネットワークリクエストを傍受し、キャッシュにアクセスし、ウェブアプリをオフラインで動作させることができます。これはプログレッシブウェブアプリの重要なコンポーネントです。
