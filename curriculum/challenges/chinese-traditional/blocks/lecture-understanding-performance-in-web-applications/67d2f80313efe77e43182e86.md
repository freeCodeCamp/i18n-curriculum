# --description--

讓我們回顧一些專門用於提升 JavaScript 效率的最重要技術。

你可能會覺得這很令人驚訝，但第一個技巧是在可能的情況下使用較少的 JavaScript。

如果你正在開發簡單的靜態網站，你可能能夠在不依賴 JavaScript 的框架下做到完全相同的事。

如果可能，你也應該嘗試移除任何未使用的 JavaScript 程式碼並使用較簡單的解決方案。

例如，你可以考慮使用內建瀏覽器特性，而非訂製特性。這些內建瀏覽器特性包括表單驗證、瀏覽器內建的影片播放器，以及 CSS 動畫，而非訂製的 JavaScript 動畫。

盡可能少使用動畫。

減少在你的 JavaScript 程式碼中對 DOM 的操作量，因為這在計算上是昂貴的。

將你的 JavaScript 程式碼拆分成執行關鍵任務和非關鍵任務的模組也很有幫助。這樣，你就能盡快預先載入關鍵模組，並延緩非關鍵模組，以盡可能快速地呈現頁面。

預先載入檔案並不保證你使用時它們會可用，但下載會較早開始，因此整體等待時間應該會較短。

當你的程式碼拆分成多個檔案後，你可以在建置處理程序中自動將它們縮小，以減少大小並最佳化載入時間。

如果不再需要某個事件監聽器，你應該將它移除：

```js
element.removeEventListener("mousemove", handleMouseMove);
```

在此範例中，我們移除 `mousemove` 事件的監聽器，因此當事件被觸發時，`handleMouseMove` 函式（程式）將不再被呼叫。如果你保留該事件監聽器，程式將繼續執行不再必要的計算，影響整體效率。如果你同時有多個作用中的事件監聽器，這種影響將更加明顯。

既然我們正在討論事件，建議在可能的情況下使用事件委派。

這表示你不需要在個別子元素上設定事件監聽器。你可以直接將它設定在父元素上，事件會向上冒泡直到找到可以控制的父元素。

優化 JavaScript 對於開發快速且回應靈敏的網頁應用程式至關重要。在真實的專案中，你會不斷分析網站的效率、性能，以找出可改進的地方。
