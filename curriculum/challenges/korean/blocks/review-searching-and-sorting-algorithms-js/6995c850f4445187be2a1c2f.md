## 검색 알고리즘

검색 알고리즘은 특정 항목 목록 내에서 목표를 찾을 수 있게 해줍니다.

컴퓨터 과학에서 알아야 할 두 가지 검색 알고리즘이 있습니다. 그것은 **선형 검색**과 **이진 검색** 알고리즘입니다. 두 알고리즘의 차이점과 각각을 언제 사용해야 하는지 이해하는 것이 중요합니다.

### 선형 검색

- 선형 검색은 배열의 항목을 처음부터 끝까지 반복하며 목표 항목을 찾을 때까지 확인합니다.
- 목표 항목을 찾으면 배열 내 위치한 인덱스를 반환합니다.
- 목표를 찾지 못하면 `-1`를 반환하는데, 이는 JavaScript에서 **유효하지 않은 인덱스**를 의미합니다.
- 선형 검색은 목표를 찾을 때까지 모든 항목을 확인하기 때문에 큰 배열에서는 효율적이지 않습니다.
- 선형 검색의 시간 복잡도는 `O(n)`인데, 배열 크기에 따라 검색 시간이 선형적으로 증가하기 때문입니다.
- 선형 검색의 공간 복잡도는 `O(1)`인데, 배열을 검색하는 데 추가 공간이 필요하지 않기 때문입니다.

### 이진 검색

- 이진 검색은 배열을 반으로 나누고, 배열 중간에 목표 값이 있는지 확인하는 방식으로 작동합니다.
- 이진 검색이 작동하려면 배열의 항목이 오름차순 또는 내림차순으로 정렬되어 있어야 합니다.
- 이진 검색은 배열을 반으로 나누고 목표가 없는 절반은 무시하기 때문에 큰 배열을 검색할 때 더 효율적인 알고리즘입니다.
- 목표 항목이 배열 중간에 있으면 목표 항목의 인덱스를 반환합니다.
- 항목을 찾지 못하면 알고리즘은 목표 항목이 배열의 왼쪽 절반에 있는지 오른쪽 절반에 있는지 확인합니다.
- 목표 항목을 찾을 때까지 배열의 남은 부분을 계속 반으로 나눕니다.
- 배열에서 목표 항목을 찾지 못하면 `-1`를 반환합니다.
- 이진 검색의 시간 복잡도는 `O(log n)`인데, 배열 크기에 따라 검색 시간이 로그 함수적으로 증가하기 때문입니다.
- 이진 검색의 공간 복잡도는 `O(1)`인데, 배열을 검색하는 데 추가 공간이 필요하지 않기 때문입니다.

### 선형 검색과 이진 검색의 차이점

- 이진 검색은 선형 검색에 비해 큰 배열에 더 적합합니다.
- 선형 검색의 시간 복잡도는 `O(n)`인데, 배열 크기에 따라 검색 시간이 선형적으로 증가하기 때문입니다.
- 이진 검색의 시간 복잡도는 `O(log n)`인데, 배열 크기에 따라 검색 시간이 로그 함수적으로 증가하기 때문입니다.

## 정렬 알고리즘과 분할 정복

컴퓨터 과학에서 분할 정복은 문제를 더 작은 하위 문제로 나누어 해결하기 쉽게 만드는 기법입니다. 재귀는 분할 정복에서 자주 사용되는 기법이며, 분할 정복은 병합 정렬 같은 효율적인 정렬 알고리즘을 구현하는 강력한 전략입니다.

### 병합 정렬

- 병합 정렬은 분할 정복 방식을 따르는 정렬 알고리즘입니다.
- 배열을 재귀적으로 더 작은 하위 배열로 나누어 각 하위 배열이 하나의 요소만 포함할 때까지 나눕니다.
- 그런 다음 하위 배열들을 정렬된 순서로 반복해서 병합합니다.
- 병합 정렬의 시간 복잡도는 `O(n log n)`인데, 배열을 계속 반으로 나누고 `(log n)` 그다음 병합하기 때문입니다 `(O(n))`.
- 병합 정렬의 공간 복잡도는 `O(n)`인데, 제자리 정렬 알고리즘이 아니기 때문입니다.
