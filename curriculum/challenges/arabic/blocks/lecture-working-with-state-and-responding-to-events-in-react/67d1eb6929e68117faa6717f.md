# --description--

في React، العرض هو العملية التي تظهر بها المكوِّنات في واجهة المستخدم (UI)، عادةً في المتصفح.

تأخذ React كل شفرتك من JavaScript وJSX وCSS، وتحدد كيف يجب أن تبدو، ثم تعرضها في واجهة المستخدم.

تتكون عملية العرض من ثلاث مراحل: التفعيل، العرض، والتثبيت. دعنا نلقي نظرة على هذه المراحل بمزيد من التفصيل.

تحدث مرحلة التفعيل عندما تكتشف React أن شيئًا ما قد تغير وأن واجهة المستخدم قد تحتاج إلى تحديث. غالبًا ما يكون هذا التغيير بسبب تحديث في الحالة أو الخصائص.

على سبيل المثال، ملاحظة أن وقت العشاء قد حان يمكن أن تحفزك على الذهاب إلى المطبخ لبدء الطهي.

في مثال `Counter` أدناه، يؤدي النقر على زر الزيادة أو النقصان إلى تحفيز React لعرض القيمة الجديدة لـ `count`:

```jsx
import { useState } from "react";

function Counter() {
  const [count, setCount] = useState(0);

  return (
    <div>
      <h1>{count}</h1>
      <button onClick={() => setCount(count - 1)}>Decrement</button>
      <button onClick={() => setCount(count + 1)}>Increment</button>
    </div>
  );
}

export default Counter;
```

بمجرد حدوث التفعيل، تدخل React مرحلة العرض. هنا، تعيد React تقييم مكوِّناتك وتحدد ما يجب عرضه.

للقيام بذلك، تستخدم React نسخة خفيفة من DOM "الحقيقي". تُسمى هذه النسخة DOM الافتراضي. باستخدام DOM الافتراضي، يمكن لـ React بسرعة التحقق مما يحتاج إلى تغيير في المكوِّن.

فكر في هذه المرحلة كنقطة تكون فيها في المطبخ، جمعت مكوناتك، وتطبخ عشاءك.

بالنسبة لمكوِّن `Counter`، مرحلة العرض هي النقطة التي تعيد فيها React تشغيل الدوال مع القيمة الجديدة لـ `count`. تعيد React حساب كيف يجب أن يبدو جزء `<h1>{count}</h1>` من المكوِّن بناءً على القيمة المحدثة لـ `count`، لكنك لن ترى أي تغييرات على الشاشة حتى المرحلة التالية – التثبيت.

مرحلة التثبيت هي حيث تأخذ React التغييرات المُعدة من DOM الافتراضي وتطبقها على DOM الحقيقي. بعبارة أخرى، هذه هي المرحلة التي ترى فيها النتيجة النهائية على الشاشة.

لتحقيق ذلك، تقارن React بين DOM الافتراضي وDOM الفعلي، وتحدد فقط الأجزاء التي تحتاج إلى تحديث، وتطبق تلك التغييرات على DOM الحقيقي لتحديث واجهة المستخدم.

يمكنك التفكير في هذه المرحلة كنقطة تقدم فيها الطعام الذي طبخته، مما يجعله مرئيًا تمامًا كما تفعل React عند تثبيت التحديثات على DOM الفعلي.

أما بالنسبة لمكوِّن `Counter`، فإن مرحلة التثبيت هي النقطة التي تُطبق فيها القيمة الجديدة لـ `count` على عنصر `h1`، ويمكنك رؤية التغيير على الصفحة.

هذه العمليات الثلاث سريعة جدًا لأن React تقلل من التلاعب المباشر بـ DOM عن طريق حساب التغييرات أولاً في DOM الافتراضي، ثم تحدث فقط الأجزاء التي تحتاج إلى تغيير في DOM الحقيقي.
