# --description--

ウェブアプリケーションを構築するときは、ファイルやフォルダーの整理について考えることが重要です。これによりコードが整理され、メンテナンスがしやすくなります。

HTMLとCSSのプロジェクトの例として、フォルダー構成を見てみましょう。

```bash
.
├── /assets
│   ├── /images
│   │   ├── logo.png
│   │   ├── banner.jpg
│   │   └── icons.svg
│   ├── /fonts
│   │   ├── custom-font.woff
│   │   └── custom-font.woff2
├── /css
│   ├── main.css
│   ├── about.css
│   └── contact.css
├── index.html
├── about.html
├── contact.html
└── README.md
```

例の一番上にある単一のドットは、ルートディレクトリを表しています。

ルートディレクトリはファイルシステムの最上位ディレクトリで、すべてのファイルパスの起点となり、他のすべてのディレクトリやファイルを含みます。ディレクトリはフォルダーの別名です。

ルートディレクトリの中で最初に示されているのは`assets`ディレクトリです。`assets`は、画像やフォント、その他のリソースなど、プロジェクトで使われるファイルを指します。

この例では、`assets`ディレクトリの中に`images`と`fonts`という2つのディレクトリがあります。

次のディレクトリは`css`ディレクトリです。これはスタイルシートディレクトリとも呼ばれることがあります。ここにすべてのCSSファイルを保存します。

`css`ディレクトリの下にはHTMLファイルがあります。`index.html`ファイルはホームページを表し、`about.html`と`contact.html`はそれぞれAboutページとContactページを表します。

最後に`README.md`ファイルがあります。`README`ファイルは、プロジェクトの内容や使い方、その他の関連情報など、有用な情報を提供するためによく使われます。

`.md`拡張子はMarkdownを表し、ドキュメントを書くのによく使われる軽量のマークアップ言語です。

プロジェクトをassets、CSS、HTMLファイルに整理することで、探しているものが見つけやすくなります。また、他の開発者がプロジェクトに貢献しやすくなります。

これはファイルやフォルダーを整理する一例に過ぎないことに注意してください。プロジェクトの構成方法は他にもたくさんあります。
