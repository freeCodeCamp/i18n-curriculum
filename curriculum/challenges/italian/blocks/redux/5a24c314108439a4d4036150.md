# --description--

Dopo che un'azione viene creata e inviata, l'archivio Redux deve sapere come rispondere a quell'azione. Questo è il compito di una funzione `reducer`. I reducer in Redux sono responsabili delle modifiche allo stato che avvengono in risposta alle azioni. Una funzione `reducer` prende `state` e `action` come argomenti e restituisce sempre un nuovo `state`. È importante capire che questo è l'**unico** ruolo del reducer. Non ha effetti collaterali — non chiama mai un endpoint API e non nasconde mai sorprese. Il reducer è semplicemente una funzione pura che prende stato e azione, poi restituisce un nuovo stato.

Un altro principio chiave in Redux è che `state` è di sola lettura. In altre parole, la funzione `reducer` deve **sempre** restituire una nuova copia di `state` e non modificare mai direttamente lo stato. Redux non impone l'immutabilità dello stato, tuttavia sei tu a doverla garantire nel codice delle tue funzioni reducer. Lo praticherai nelle sfide successive.

# --instructions--

L'editor del codice contiene l'esempio precedente e l'inizio di una funzione `reducer` per te. Completa il corpo della funzione `reducer` in modo che, se riceve un'azione di tipo `'LOGIN'`, restituisca un oggetto stato con `login` impostato su `true`. Altrimenti, restituisce l'attuale `state`. Nota che l'attuale `state` e l'azione inviata `action` sono passate al reducer, quindi puoi consultare direttamente il tipo dell'azione con `action.type`.

# --hints--

Chiamare la funzione `loginAction` dovrebbe restituire un oggetto con la proprietà type impostata sulla stringa `LOGIN`.

```js
assert(loginAction().type === 'LOGIN');
```

L'archivio dovrebbe essere inizializzato con un oggetto con la proprietà `login` impostata su `false`.

```js
assert(store.getState().login === false);
```

Inviare `loginAction` dovrebbe aggiornare la proprietà `login` nello stato dell'archivio a `true`.

```js
assert(
  (function () {
    const initialState = store.getState();
    store.dispatch(loginAction());
    const afterState = store.getState();
    return initialState.login === false && afterState.login === true;
  })()
);
```

Se l'azione non è di tipo `LOGIN`, l'archivio dovrebbe restituire lo stato corrente.

```js
assert(
  (function () {
    store.dispatch({ type: '__TEST__ACTION__' });
    let afterTest = store.getState();
    return typeof afterTest === 'object' && afterTest.hasOwnProperty('login');
  })()
);
```

const defaultState = {
  login: false
};

const reducer = (state = defaultState, action) => {

  if (action.type === 'LOGIN') {
    return {login: true}
  }

  else {
    return state
  }

};

const store = Redux.createStore(reducer);

const loginAction = () => {
  return {
    type: 'LOGIN'
  }
};
