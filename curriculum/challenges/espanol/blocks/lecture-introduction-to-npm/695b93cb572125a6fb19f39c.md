# --description--

En una lección anterior, aprendiste qué son las dependencias y cómo elegir las adecuadas para tu proyecto.

En esta lección, aprenderás cómo instalar y eliminar dependencias usando npm.

Para instalar una dependencia, usa el comando `npm install` así:

```bash
npm install dependencyName
```

También puedes abreviar `install` a `i` así:

```bash
npm i dependencyName
```

Después de instalar un paquete, npm añade una nueva sección `dependencies` a tu archivo `package.json` (si no está ya allí) y lista el paquete que instalaste. Por ejemplo, así es como se ve cuando instalas el paquete `chalk`:

```json
{
  "name": "install-uninstall-deps",
  "version": "1.0.0",
  "description": "a simple demo on how to install and uninstall dependencies",
  "main": "index.js",
  "scripts": {
    "test": "echo \"Error: no test specified\" && exit 1"
  },
  "keywords": ["npm", "deps"],
  "author": "Kolade Chris",
  "license": "ISC",
  "dependencies": {
    "chalk": "^5.6.2"
  }
}
```

Cuando instalas una dependencia, dos cosas nuevas aparecen en la carpeta raíz de tu proyecto:

* `node_modules/`: una carpeta que contiene el código real de los paquetes que instalaste.
    
* `package-lock.json`: un archivo que almacena información detallada sobre las versiones exactas de esos paquetes, incluyendo sus propias dependencias. (Aprenderás más sobre este archivo en una lección posterior.)
    

<img src="https://cdn.freecodecamp.org/curriculum/lecture-transcripts/how-can-you-install-and-remove-dependencies-1.png" alt="">

La mayoría de los proyectos necesitan más de una dependencia. En lugar de instalarlas una por una, npm te permite instalar varios paquetes a la vez ingresando una lista de nombres de paquetes separados por espacios:

```bash
npm install express lodash bcrypt
```

Al igual que antes, cuando instalas múltiples dependencias a la vez, npm actualiza tus archivos `package.json` y `package-lock.json` y tus `node_modules` para incluirlas. Así es como se vería el `package.json`:

```json
{
  "name": "install-uninstall-deps",
  "version": "1.0.0",
  "description": "a simple demo on how to install and uninstall dependencies",
  "main": "index.js",
  "scripts": {
    "test": "echo \"Error: no test specified\" && exit 1"
  },
  "keywords": ["npm", "deps"],
  "author": "Kolade Chris",
  "license": "ISC",
  "dependencies": {
    "bcrypt": "^6.0.0",
    "chalk": "^5.6.2",
    "express": "^5.1.0",
    "lodash": "^4.17.21"
  }
}
```

Para instalar una `devDependency`, agrega la bandera `--save-dev` al comando `install`. También puedes usar la versión más corta `-D`:

```bash
npm i nodemon -D
```

Después de ejecutar este comando, npm actualiza tu archivo `package.json` y agrega el paquete en la sección `devDependencies`. Aquí tienes un ejemplo de cómo se ve eso:

```json
{
  "name": "install-uninstall-deps",
  "version": "1.0.0",
  "description": "a simple demo on how to install and uninstall dependencies",
  "main": "index.js",
  "scripts": {
    "test": "echo \"Error: no test specified\" && exit 1"
  },
  "keywords": ["npm", "deps"],
  "author": "Kolade Chris",
  "license": "ISC",
  "dependencies": {
    "bcrypt": "^6.0.0",
    "chalk": "^5.6.2",
    "express": "^5.1.0",
    "lodash": "^4.17.21"
  },
  "devDependencies": {
    "nodemon": "^3.1.10"
  }
}
```

Cada vez que ejecutes un comando `npm install`, npm mostrará un mensaje indicándote si la instalación fue exitosa o no. Un mensaje de éxito generalmente se ve algo así:

```bash
user@Kolade ~/Desktop/fCC/script-code/node/npm/install-uninstall-deps  % npm install express lodash bcrypt

added 72 packages, and audited 74 in 8s

17 packages are looking for funding
  run `npm fund` for details

found 0 vulnerabilities

user@Kolade ~/Desktop/fCC/script-code/node/npm/install-uninstall-deps  % npm install nodemon -D

added 27 packages, and audited 101 in 16s

21 packages are looking for funding
  run `npm fund` for details

found 0 vulnerabilities
```

Para desinstalar o eliminar un paquete, usa el comando `npm uninstall`:

```bash
npm uninstall dependencyName
```

Por ejemplo, para eliminar `chalk` de tu proyecto, ejecutarías este comando:

```bash
npm uninstall chalk
```

También puedes desinstalar múltiples paquetes a la vez listándolos con espacios entre cada nombre de paquete:

```bash
npm uninstall express lodash bcrypt
```

También recibirás un mensaje que muestra si el comando de desinstalación tuvo éxito o falló:

```bash
user@Kolade ~/Desktop/fCC/script-code/node/npm/install-uninstall-deps  % npm uninstall chalk

added 1 package, removed 1 package, and audited 101 packages in 1s

20 packages are looking for funding
  run `npm fund` for details

found 0 vulnerabilities

user@Kolade ~/Desktop/fCC/script-code/node/npm/install-uninstall-deps  % npm uninstall express lodash bcrypt

removed 70 packages, and audited 31 in 1s

4 packages are looking for funding
  run `npm fund` for details

found 0 vulnerabilities
```

Y tu archivo `package.json` se actualizará automáticamente:

```json
{
  "name": "install-uninstall-deps",
  "version": "1.0.0",
  "description": "a simple demo on how to install and uninstall dependencies",
  "main": "index.js",
  "scripts": {
    "test": "echo \"Error: no test specified\" && exit 1"
  },
  "keywords": ["npm", "deps"],
  "author": "Kolade Chris",
  "license": "ISC",
  "devDependencies": {
    "nodemon": "^3.1.10"
  }
}
```

Para desinstalar una `devDependency`, simplemente puedes ejecutar:

```bash
npm uninstall nodemon
```

Ten en cuenta que no necesitas la bandera `-D` al desinstalar. npm eliminará automáticamente el paquete de la sección correcta, `devDependencies` en este caso.

Después de desinstalar el `devDependency`, tu `package.json` se actualizará nuevamente:

```json
{
  "name": "install-uninstall-deps",
  "version": "1.0.0",
  "description": "a simple demo on how to install and uninstall dependencies",
  "main": "index.js",
  "scripts": {
    "test": "echo \"Error: no test specified\" && exit 1"
  },
  "keywords": ["npm", "deps"],
  "author": "Kolade Chris",
  "license": "ISC"
}
```
