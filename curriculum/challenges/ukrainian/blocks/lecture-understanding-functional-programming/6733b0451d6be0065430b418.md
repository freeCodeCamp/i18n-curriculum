# --interactive--

Функціональне програмування полягає у написанні коду, який є чистим, передбачуваним і легшим для тестування та налагодження. Два ключові поняття, які ми сьогодні розглянемо, — це чисті функції та уникнення побічних ефектів.

Почнемо з чистих функцій.

Чиста функція — це функція, яка, отримуючи однакові вхідні дані, завжди повертає однаковий результат і не змінює нічого поза собою.

Ось приклад чистої функції:

```js
function add(a, b) {
  return a + b;
}
```

Ця функція є чистою, бо завжди повертає однаковий результат для однакових вхідних даних і не змінює нічого поза собою. Скільки б разів ви не викликали `add(2, 3)`, вона завжди повертатиме `5` і не впливатиме на інші частини вашої програми.

Тепер порівняємо це з нечистою функцією:

```js
let total = 0;
function addToTotal(value) {
  total += value;
  return total;
}
```

Ця функція нечиста, бо змінює змінну поза собою (`total`). Результат `addToTotal(5)` буде різним залежно від того, яке значення було у `total` до виклику функції. Така непередбачуваність ускладнює розуміння та налагодження коду.

Це приводить нас до наступного важливого поняття — побічних ефектів.

Побічний ефект — це будь-яка зміна стану програми під час виклику функції. Це може бути зміна глобальної змінної, запис у файл або виклик API.

Чисті функції не мають побічних ефектів, а нечисті — мають.

Уникнення побічних ефектів — ключовий принцип функціонального програмування. Коли ми мінімізуємо побічні ефекти, наш код стає більш передбачуваним і легшим для тестування.

Пишучи код, наша головна мета — щоб функції виконували одну задачу і робили це дуже добре, не змінюючи інші частини програми.

Ось приклад функції з побічним ефектом:

:::interactive_editor

```js
function greet(name) {
  console.log(`Hello, ${name}!`);
}

greet("Alice");
```

:::

Ця функція має побічний ефект, бо взаємодіє з зовнішнім світом, виводячи повідомлення в консоль.

У функціональному програмуванні ми могли б переписати це так:

:::interactive_editor

```js
function greet(name) {
  return `Hello, ${name}!`;
}
console.log(greet("Alice"));
```

:::

Тепер `greet` — це чиста функція, яка просто повертає рядок, а побічний ефект (вивід у консоль) обробляється окремо.

Функціональне програмування заохочує нас складати програми з чистих функцій, розбиваючи код на невеликі, передбачувані частини. Таким чином ми створюємо програми, які легше розуміти, тестувати та підтримувати.
