# --interactive--

Hoisting ni tabia katika JavaScript ambayo mara nyingi huwatatiza wanaoanza, lakini kuelewa inaweza kusaidia kuepuka hitilafu ndogo katika msimbo wako. Kwa maneno rahisi, hoisting ni tabia ya kawaida ya JavaScript ya kusogeza tamko za vitu juu ya wigo wake wakati wa awamu ya uandikishaji kabla msimbo haujatekelezwa.

Ili kuelewa hoisting, ni muhimu kujua kuwa JavaScript inaendeshwa katika awamu mbili: awamu ya uandikishaji na awamu ya utekelezaji. Wakati wa awamu ya uandikishaji, injini ya JavaScript hupitia msimbo wako na kuandaa nafasi ya kumbukumbu kwa ajili ya vigezo na vitendakazi. Hapa ndipo hoisting huanza kufanya kazi.

Tuanze na hoisting ya vigezo, unapotangaza kigezo kwa kutumia neno `var`, JavaScript husogeza tamko la kigezo hicho juu ya wigo wake. Hata hivyo ni muhimu kuelewa kuwa tamko pekee ndilo husogezwa, si uanzishaji. Hii inamaanisha unaweza kutumia kigezo katika msimbo wako kabla hujatangaza, lakini thamani yake itakuwa `undefined` hadi utakapoweka thamani halisi.

:::interactive_editor

```js
console.log(x); // undefined
var x = 5;
console.log(x); // 5
```

:::

Katika msimbo huu, ingawa tunatumia `x` kabla ya kutangaza, hatupati hitilafu, badala yake tunapata `undefined`. Hii ni kwa sababu JavaScript husogeza tamko la `var x` juu ya wigo wake lakini si uanzishaji wa `x = 5`. Ni kama msimbo umeandikwa upya hivi:

:::interactive_editor

```js
var x;
console.log(x); // undefined
x = 5;
console.log(x); // 5
```

:::

Hoisting ya vitendakazi hufanya kazi tofauti kidogo. Unapotangaza kitendakazi kwa kutumia sintaksia ya tamko la `function`, jina la kitendakazi na mwili wake vyote husogezwa. Hii inamaanisha unaweza kuita kitendakazi kabla hujatangaza katika msimbo wako. Hapa kuna mfano wa hoisting ya kitendakazi:

:::interactive_editor

```js
sayHello(); // "Hello, World!"

function sayHello(){
  console.log("Hello, World!");
}
```

:::

Katika kesi hii, tunaweza kuita `sayHello()` kabla ya tamko lake kwa sababu kitendakazi kizima kimesogezwa juu ya wigo wake.

Ni muhimu kuelewa kuwa hoisting hufanya kazi tofauti na tamko la `let` na `const` lililotangazwa katika ES6.

```js
console.log(y); // Throws a ReferenceError
let y = 10;
```

Tamko hizi husogezwa lakini hazijaanzishwa na huwezi kuzipata kabla ya tamko halisi katika msimbo wako. Hii mara nyingi huitwa eneo la kifo la muda (temporal dead zone).

Kuelewa hoisting kunaweza kusaidia kuandika msimbo safi, unaotabirika zaidi. Hata hivyo, kutegemea hoisting kunaweza kufanya msimbo wako usomeke kwa ugumu na kuhitaji matengenezo zaidi. Kama mbinu bora, inashauriwa kutangaza vigezo vyako juu ya wigo wao na vitendakazi kabla ya kuvitumia bila kujali hoisting. Hii hufanya tabia ya msimbo wako iwe wazi zaidi na rahisi kwa wengine, ikiwa ni pamoja na wewe mwenyewe baadaye, kuelewa.
