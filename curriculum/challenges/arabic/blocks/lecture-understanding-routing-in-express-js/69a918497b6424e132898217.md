# --description--

في Express، يمكنك إنشاء **معالجات مسارات قابلة للربط** باستخدام الطريقة `app.route()`. هذا مفيد عندما تريد التعامل مع عدة طرق HTTP مثل GET وPOST وPUT لمسار واحد دون تكرار المسار في كل مرة.

بدلاً من كتابة تعريفات مسارات منفصلة مثل `app.get("/user")`، و`app.post("/user")`، و`app.put("/user")`، تتيح لك `app.route()` تجميعها معًا في كتلة شفرة واحدة ومنظمة.

تعمل طريقة `app.route()` كاختصار لتعريف عدة معالجات لنفس نقطة النهاية. هذا مفيد بشكل خاص في واجهات برمجة التطبيقات RESTful، حيث يدعم مورد واحد (مثل `/user`) عدة عمليات.

إليك مثالًا يحاكي واجهة برمجة تطبيقات RESTful لإدارة المستخدمين:

```javascript
const express = require("express");
const app = express();
const port = 3000;

app.use(express.json()); // Parse JSON request bodies

// Use app.route() to group handlers for the same path
app
  .route("/user")
  .get((req, res) => {
    res.send("Fetching user data");
  })
  .post((req, res) => {
    const { name } = req.body;
    res.send(`Creating a user with name: ${name}`);
  })
  .put((req, res) => {
    const { name } = req.body;
    res.send(`Updating the user to: ${name}`);
  })
  .delete((req, res) => {
    res.send("Deleting the user");
  });

app.listen(port, () => {
  console.log(`Server running at http://localhost:${port}`);
});
```

في المثال أعلاه، تعرف `app.route("/user")` مسارًا واحدًا لمسار `/user`. ثم نربط عدة طرق HTTP، `.get()` و`.post()` و`.put()` و`.delete()`، للتعامل مع أنواع مختلفة من الطلبات لنفس المسار.

إليك تفصيل المسارات المختلفة وكيف يمكن أن تعمل في تطبيق RESTful بسيط:

* GET `/user`: يسترجع ويُرجع بيانات المستخدم.
    
* POST `/user`: ينشئ مستخدمًا جديدًا بالبيانات الموجودة في جسم الطلب.
    
* PUT `/user`: يحدث مستخدمًا موجودًا بالبيانات الموجودة في جسم الطلب.
    
* DELETE `/user`: يحذف مستخدمًا.
    

يساعد استخدام `app.route()` في تنظيم شفرتك عند التعامل مع عدة طرق HTTP لنفس المسار. بدلاً من تكرار المسار في كل مرة، تجمع المعالجات المرتبطة في مكان واحد.

هذا يقلل التكرار ويجعل شفرتك أسهل في القراءة والصيانة. عندما تُعرف كل الطرق لمسار في كتلة واحدة، يصبح من الواضح كيف يتصرف ذلك النقطة النهاية.

مع نمو التطبيق وتعقيد المسارات، توفر `app.route()` هيكلًا بسيطًا يحافظ على منطق مرتبط معًا. هذا مفيد بشكل خاص عند تصميم واجهات برمجة التطبيقات RESTful، حيث يدعم مورد واحد عدة عمليات مثل GET وPOST وPUT وDELETE.
