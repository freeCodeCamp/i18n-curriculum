# --description--

Node.js não possui um servidor WebSocket embutido. Desenvolvedores normalmente usam pacotes externos como `ws` para implementar o protocolo WebSocket e gerenciar a comunicação em tempo real entre clientes e servidores.

A biblioteca `ws` é leve, rápida e fácil de usar. Ela implementa o protocolo WebSocket para que você possa criar e gerenciar conexões WebSocket.

Você pode instalá-la com npm:

```bash
npm install ws
```

Depois de instalada, crie um servidor WebSocket importando a biblioteca `ws`, invocando seu método `Server` e passando uma porta:

```js
const WebSocket = require("ws");

const server = new WebSocket.Server({
  port: 3000,
});
```

Isso inicia um servidor WebSocket que escuta conexões na porta `3000`.

Quando um cliente se conecta, o servidor emite um evento `connection`.

```js
server.on("connection", (socket) => {
  console.log("Client connected");
});
```

Pense nesse evento como a forma do servidor dizer "um novo cliente entrou".

O parâmetro `socket` representa a conexão entre o servidor e esse cliente. Você o usa para enviar e receber mensagens.

Quando um cliente conectado envia dados, o servidor emite um evento `message`.

```js
server.on("connection", (socket) => {
  socket.on("message", (message) => {
    console.log(message.toString());
  });
});
```

Neste exemplo, o servidor escuta mensagens recebidas e as registra no console.

Você pode enviar dados de volta para um cliente usando o método `send()`:

```js
socket.send("Hello from the server");
```

Isso envia uma mensagem pela conexão WebSocket aberta.

Enquanto a conexão permanecer aberta, tanto o cliente quanto o servidor podem continuar trocando mensagens em tempo real.

Além de `connection` e `message`, conexões WebSocket também emitem os eventos `close` e `error`.

`close` é disparado quando uma conexão é fechada:

```js
socket.on("close", () => {
  console.log("Disconnected");
});
```

`error` é disparado quando algo dá errado:

```js
socket.on("error", (error) => {
  console.error(error);
});
```

Juntos, esses eventos formam a base da maioria das aplicações WebSocket.

Um requisito comum em aplicações de chat é o broadcast. Broadcast significa enviar uma mensagem para vários clientes conectados em vez de apenas um.

Imagine uma sala de chat com três usuários chamados **John Doe**, **Jane Doe** e **Wes Smith**.

Se John Doe enviar uma mensagem, o servidor pode precisar encaminhar essa mensagem para Jane Doe e Wes Smith.

Para isso, o servidor mantém o controle dos clientes conectados e faz um loop por eles quando uma mensagem chega.

Antes de enviar uma mensagem, o servidor verifica se a conexão do cliente ainda está aberta:

```js
if (client.readyState === WebSocket.OPEN) {
  // do something
}
```

Cada conexão WebSocket tem uma propriedade `readyState` que descreve seu estado atual.

`WebSocket.OPEN` significa que a conexão está ativa e pronta para enviar e receber dados.

Ao verificar isso, o servidor evita enviar mensagens para clientes que já desconectaram ou cujas conexões ainda estão sendo estabelecidas:

```js
server.on("connection", (socket) => {
  socket.on("message", (message) => {
    server.clients.forEach((client) => {
      if (client.readyState === WebSocket.OPEN) {
        client.send(message.toString());
      }
    });
  });
});
```

Neste exemplo, todo cliente conectado recebe a mensagem recebida. Isso é chamado de broadcast.

Muitas aplicações em tempo real usam broadcast para entregar dados como mensagens de chat, notificações ao vivo, atualizações esportivas e eventos de jogos multiplayer.

Com a biblioteca `ws`, é simples construir servidores WebSocket em Node.js.

Tudo que você precisa fazer é criar um servidor, escutar eventos, trocar mensagens e broadcastar atualizações para os clientes conectados.

Esses blocos básicos formam a base de muitas aplicações em tempo real.
