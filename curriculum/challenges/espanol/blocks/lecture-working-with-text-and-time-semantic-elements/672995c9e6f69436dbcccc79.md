# --interactive--

En HTML, los elementos citados se usan para distinguir el texto citado del contenido circundante. Esto da al texto citado un formato que es fácil de identificar.

Debes usar el elemento de cita en bloque para representar una sección citada de otra fuente. Se usa principalmente para citas extensas. Si la fuente de la cita tiene una dirección, puedes citarla con el atributo `cite`. El valor de este atributo debe ser una URL válida. Este es un ejemplo de una cita dentro de un elemento de cita en bloque:

:::interactive_editor

```html
<blockquote cite="https://www.freecodecamp.org/news/learn-to-code-book/">
  "Can you imagine what it would be like to be a successful developer? To have built software systems that people rely upon?"
</blockquote>
```

:::

Este elemento tiene un atributo `cite`. El valor del atributo `cite` es la URL de la fuente. Aunque este atributo no cambia la presentación de la cita en bloque, es muy útil para dar a los lectores de pantalla y motores de búsqueda más información sobre la cita. En el navegador, verás que el texto está ligeramente indentado.

Si quieres comenzar y terminar la cita en bloque con comillas, puede que necesites escribirlas explícitamente dentro del texto. Puedes escribir el texto directamente dentro del elemento de cita en bloque, como acabo de hacer, o envolverlo dentro de uno o más elementos de párrafo. Esto es útil cuando el texto tiene múltiples párrafos, pero quieres mantenerlos dentro de la misma cita en bloque. Aquí tienes un ejemplo con cuatro párrafos:

:::interactive_editor

```html
<blockquote cite="https://www.freecodecamp.org/news/learn-to-code-book/">
  <p>Build your projects. Show them to your friends. Build projects for your friends.</p>
  <p>Build your network. Help the people you meet along the way. What goes around comes around. You'll get what's coming to you.</p>   
  <p>It is not too late. Life is long.</p>
  <p>You will look back on this moment years from now and be glad you made a move.</p>
</blockquote>
```

:::

Todos los párrafos están contenidos dentro del mismo elemento de cita en bloque, por lo que son parte de la misma cita. Puedes ver que el elemento tiene un atributo `cite` con la URL de la fuente. En el navegador, verás que los cuatro párrafos están alineados entre sí, pero están indentados con respecto a su contenedor.

Hasta ahora he estado usando el atributo `cite` para atribuir la fuente de la cita, pero el atributo realmente no muestra la fuente al usuario. Solo funciona detrás de escena.

Si quieres atribuir la fuente visualmente, puedes agregar un elemento de cita, `cite`, fuera del elemento de cita en bloque. Esto es diferente del atributo `cite`. El elemento de cita es un elemento HTML que puedes usar para marcar el título de una obra creativa referenciada como un libro, artículo, canción, película, sitio web o trabajo de investigación. Aquí tienes un ejemplo:

:::interactive_editor

```html
<div>
  <blockquote cite="https://www.freecodecamp.org/news/learn-to-code-book/">
    Can you imagine what it would be like to be a successful developer? To have built software systems that people rely upon?
  </blockquote>
  <p>—Quincy Larson, <cite>How to Learn to Code and Get a Developer Job [Full Book].</cite></p>
</div>
```

:::

El elemento de cita en bloque contiene texto citado. Debajo del elemento puedes ver un elemento de párrafo con el nombre del autor, seguido por un elemento de cita. El elemento de cita contiene el título del libro de donde provino la cita.

Si vas al navegador, ahora la fuente será claramente visible y verás que la cita proviene de un libro escrito por Quincy Larson. El título de este libro es `How to Learn to Code and Get a Developer Job`.

Debes usar citas en bloque como estas para citas largas de otras fuentes. Pero a veces solo necesitarás citar unas pocas palabras dentro de un párrafo más grande.

Eso es exactamente para lo que sirve el elemento de cita en línea. Es para citas breves en línea de otras fuentes. La mayoría de los navegadores modernos agregarán comillas alrededor de la cita en línea automáticamente cuando uses este elemento. Este es un ejemplo:

:::interactive_editor

```html
<p>
  As Quincy Larson said,
  <q cite="https://www.freecodecamp.org/news/learn-to-code-book/">
    Momentum is everything.
  </q>
</p>
```

:::

Puedes ver un elemento de párrafo que contiene texto. Parte del texto es una cita en línea, porque está dentro del elemento de cita en línea. También puedes agregar un atributo `cite` para atribuir la fuente.

Esto funciona exactamente igual que con el elemento de cita en bloque. No habrá cambios visuales, pero proporcionará a los lectores de pantalla y motores de búsqueda más información sobre la cita.

En el navegador, verás que el texto citado es parte del párrafo y está rodeado por comillas. La mayoría de los navegadores modernos agregará estas comillas automáticamente. 

¿Cuál es la diferencia entre citas en bloque y en línea? Debes usar citas en bloque para citas extensas de otras fuentes y citas en línea para citas breves de otras fuentes que deberían formar parte de párrafos existentes.
