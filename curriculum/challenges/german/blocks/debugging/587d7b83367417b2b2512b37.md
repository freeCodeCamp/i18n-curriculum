# --description--

Sie haben vielleicht bemerkt, dass einige freeCodeCamp-Aufgaben ihre eigene Konsole enthalten. Diese Konsole verhält sich etwas anders als die Browserkonsole.

Es gibt viele Methoden, die Sie mit `console` verwenden können, um Nachrichten auszugeben. `log`, `warn` und `clear` sind nur einige davon. Die freeCodeCamp-Konsole gibt nur `log`-Nachrichten aus, während die Browserkonsole alle Nachrichten anzeigt. Wenn Sie Änderungen an Ihrem Code vornehmen, wird dieser automatisch ausgeführt und die Protokolle werden angezeigt. Die freeCodeCamp-Konsole wird jedes Mal gelöscht, wenn Ihr Code ausgeführt wird.

# --instructions--

Öffnen Sie zuerst Ihre Browserkonsole, damit Sie die Protokolle sehen können. Klicken Sie dazu mit der rechten Maustaste auf die freeCodeCamp-Navigationsleiste oben und klicken Sie in den meisten Browsern auf `inspect`. Suchen Sie dann im sich öffnenden Fenster den Tab `console`.

Verwenden Sie danach `console.log`, um die Variable `output` zu protokollieren. Sehen Sie sich die beiden Konsolen an, um das Protokoll zu sehen. Verwenden Sie schließlich `console.clear` nach Ihrem Protokoll, um die Browserkonsole zu löschen. Beobachten Sie den Unterschied zwischen den beiden Konsolen.
