# --interactive--

Замикання — це одна з найпотужніших і часто неправильно зрозумілих функцій у JavaScript. По суті, замикання — це функція, яка має доступ до змінних у своїй зовнішній лексичній області видимості, навіть після того, як зовнішня функція вже повернула результат. Це може звучати складно, але це фундаментальна концепція, яка дозволяє реалізовувати багато просунутих шаблонів програмування в JavaScript.

Щоб зрозуміти замикання, почнемо з прикладу:

:::interactive_editor

```js
function outerFunction(x) {
    let y = 10;
    function innerFunction(){
        console.log(x + y);
    }
    return innerFunction;
}

let closure = outerFunction(5);
console.log(closure()); // 15
```

:::

У цьому прикладі `outerFunction` приймає параметр `x` і визначає локальну змінну `y`. Потім він визначає `innerFunction`, яка використовує і `x`, і `y`. Нарешті, він повертає `innerFunction`. Коли ми викликаємо `outerFunction(5)`, вона повертає `innerFunction`, яке ми призначаємо змінній `closure`. Коли пізніше ми викликаємо `closure()`, вона все ще має доступ до `x` і `y` з `outerFunction`, хоча `outerFunction` вже завершила виконання. Ось суть замикання.

Внутрішня функція зберігає референс на своє зовнішнє лексичне оточення, зберігаючи доступ до змінних у цьому оточенні навіть після завершення зовнішньої функції.

Замикання особливо корисні для створення приватних змінних і функцій. Розглянемо цей приклад:

:::interactive_editor

```js
function createCounter() {
    let count = 0;
    return function () {
        count++;
        return count;
    };
}

let counter = createCounter();
console.log(counter()); // 1
console.log(counter()); // 2
```

:::

У цьому випадку `createCounter` повертає функцію, яка збільшує і повертає змінну `count`. Змінна `count` недоступна безпосередньо поза `createCounter`, але повернена функція (наше замикання) має до неї доступ. Кожного разу, коли ми викликаємо `counter()`, вона збільшує і повертає `count`.

Замикання також можуть захоплювати кілька змінних зі свого зовнішнього оточення. Наприклад:

:::interactive_editor

```js
function multiply(x) {
    return function (y) {
        return x * y;
    };
}

let double = multiply(2);
console.log(double(5)); // 10
```

:::

Тут внутрішня функція захоплює параметр `x` із `multiply`. Коли ми створюємо `double`, викликаючи `multiply(2)`, вона повертає функцію, яка завжди множить свій аргумент на `2`.

Важливо пам’ятати, що замикання захоплюють змінні за референсом, а не за значенням. Це означає, що якщо значення захопленої змінної змінюється, замикання побачить нове значення. Наприклад:

:::interactive_editor

```js
function createIncrementer() {
    let count = 0;
    return function () {
        count++;
        console.log(count);
    };
}

let increment = createIncrementer();
increment(); // 1
increment(); // 2
```

:::

Кожного разу, коли ми викликаємо `increment`, вона працює з тією ж змінною `count`, а не з копією її початкового значення. Замикання — це потужний інструмент у JavaScript. Продовжуючи працювати з JavaScript, ви побачите, що розуміння і ефективне використання замикань значно покращує вашу здатність писати чистий, ефективний і потужний код.
