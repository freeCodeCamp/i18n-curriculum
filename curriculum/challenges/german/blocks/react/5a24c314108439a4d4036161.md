# --description--

Bisher haben Sie gesehen, wie sich JSX in einem wichtigen Punkt von HTML unterscheidet: bei der Verwendung von `className` gegenüber `class` zur Definition von HTML-Klassen.

Ein weiterer wichtiger Unterschied zwischen JSX und HTML ist das Konzept des selbstschließenden Tags.

In HTML haben fast alle Tags sowohl ein öffnendes als auch ein schließendes Tag: `<div></div>`; das schließende Tag hat immer einen Schrägstrich vor dem Tag-Namen, den Sie schließen. Es gibt jedoch spezielle Fälle in HTML, sogenannte <dfn>void elements</dfn>, also Elemente, die kein öffnendes und schließendes Tag benötigen, bevor ein anderes Element beginnen kann.

Zum Beispiel kann das Zeilenumbruch-Tag entweder als `<br>` oder als `<br />` geschrieben werden, sollte aber niemals als `<br></br>` geschrieben werden, da es keinen Inhalt enthält.

In JSX sind die Regeln etwas anders. Jedes JSX-Element kann als selbstschließendes Tag geschrieben werden, und jedes Element muss geschlossen sein. Das Zeilenumbruch-Tag muss zum Beispiel immer als `<br />` geschrieben werden, damit es gültiges JSX ist, das transpiliert werden kann. Ein `<div>` hingegen kann entweder als `<div />` oder als `<div></div>` geschrieben werden. Der Unterschied besteht darin, dass es in der ersten Syntax-Version keine Möglichkeit gibt, etwas in das `<div />` einzufügen. Sie werden in späteren Aufgaben sehen, dass diese Syntax beim Rendern von React-Komponenten nützlich ist.

# --instructions--

Beheben Sie die Fehler im Code-Editor, sodass gültiges JSX entsteht und erfolgreich transpiliert wird. Achten Sie darauf, keinen Inhalt zu ändern – Sie müssen nur dort Tags schließen, wo es nötig ist.

# --hints--

Die Konstante `JSX` sollte ein `div`-Element zurückgeben.

```js
assert.strictEqual(JSX.type, 'div');
```

Das `div` sollte ein `br`-Tag enthalten.

```js
assert(Enzyme.shallow(JSX).find('br').length === 1);
```

Das `div` sollte ein `hr`-Tag enthalten.

```js
assert(Enzyme.shallow(JSX).find('hr').length === 1);
```
