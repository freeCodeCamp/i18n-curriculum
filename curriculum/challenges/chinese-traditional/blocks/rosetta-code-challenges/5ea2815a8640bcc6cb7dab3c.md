# --description--

<ol>
  <li>Take an integer <code>n₀</code>, greater than zero.</li>
  <li>Form the next number <code>n</code> of the series by reversing <code>n₀</code> and adding it to <code>n₀</code></li>
  <li>Stop when <code>n</code> becomes palindromic - i.e. the digits of <code>n</code> in reverse order == <code>n</code>.</li>
</ol>

上述遞迴關係式當套用到大多數起始數字 `n` = 1、2、… 時，會相當快速地終止於一個回文。

例如如果 `n₀` = 12 我們得到：

```md
12
12 + 21 = 33,  a palindrome!
```

如果 `n₀` = 55，我們得到：

```md
55
55 + 55 = 110
110 + 011 = 121,  a palindrome!
```

請注意，回文檢查發生在*加法*之後。

有些起始數字似乎無限延續；196 的遞迴關係已計算數百萬次重複，形成數百萬位數的數字，卻未形成回文。這些不以回文結尾的數字稱為 **Lychrel numbers**。

在此任務中，Lychrel 數是指任何在 500 次（或更多）迭代內未形成回文的起始數字。

**種子數與相關的 Lychrel 數字：**

在 Lychrel 數列中產生的任何整數也是 Lychrel 數。

一般來說，任何來自一個 Lychrel 數的序列*可能*會收斂並加入先前 Lychrel 數候選者的序列；例如數字 196 和 689 的序列開始於：

```md
    196
    196 + 691 = 887
    887 + 788 = 1675
    1675 + 5761 = 7436
    7436 + 6347 = 13783
    13783 + 38731 = 52514
    52514 + 41525 = 94039
    ...
    689
    689 + 986 = 1675
    1675 + 5761 = 7436
    ...
```

所以我們看到以 689 開始的序列會收斂到，並且繼續與 196 的序列相同的數字。

因此，我們可以進一步將 Lychrel 數分為真正的 **Seed** Lychrel 數候選者，以及不產生回文但其序列中出現的整數被視為從較低 Lychrel 數產生的序列一部分的 **Related** 數。

# --solutions--

```js
function isLychrel(n) {
  function reverse(num) {
    return parseInt(
      num
        .toString()
        .split('')
        .reverse()
        .join('')
    );
  }

  var i;
  for (i = 0; i < 500; i++) {
    n = n + reverse(n);
    if (n == reverse(n)) break;
  }

  return i == 500;
}
```
