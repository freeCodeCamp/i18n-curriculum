# --description--

讓我們來了解終端機，以及它與命令列的不同之處。

命令列是一種基本的文字輸入介面，允許使用者輸入「commands」，通常以一連串字元的形式，並透過按下「Enter」鍵來提交或執行它們。你最常在終端機中看到命令列介面。

終端機是一種特殊的應用程式，提供命令列介面以執行超出基本讀寫操作的系統層級指令。你也可能聽過「終端機模擬器」，它們本質上是包裝基本終端機介面以提供額外特性或功能的應用程式。但在大多數一般用途下，你大概可以安全地將這些稱為「終端機」。

最後，我們還需要討論「shell」。shell 是包裹命令列並將你的輸入解釋為指令，然後傳回輸出的軟體。

但是你如何在你的系統上存取 `terminal`？

好吧，如果你正在執行現代版本的 Windows，你很可能會有兩種不同的選項。Windows 預設提供兩個不同的 shell 應用程式：Powershell 和命令提示字元。

這兩個 shell 都附帶各自的終端機應用程式（名稱與各自的 shell 相同），所以你可以使用你的應用程式選單來啟動 Powershell 終端機或命令提示字元終端機。

Windows 也提供一個相對較新且現代化的終端機模擬器，恰如其分地命名為 Microsoft Terminal。

你可以從 Windows Store 安裝此應用程式，然後像啟動你的 Powershell 或 Command Prompt 終端機一樣啟動它。Microsoft Terminal 的優點是它允許你從同一個應用程式存取 Powershell 和 Command Prompt shell，以及如果你已組態 Windows Subsystems for Linux，還能存取相應的 Linux shell。

MacOS 提供一個恰如其名的終端機，名為 "Terminal"。你可以透過你的 spotlight 搜尋或應用程式啟動器來存取它。

MacOS 也支援第三方終端機模擬器，例如 iTerm 或 Ghostty。你可以從它們的首頁安裝這些應用程式。

在 Linux 上，預設的終端機（以及你如何啟動它）高度依賴你使用的發行版和桌面環境。也有很多對第三方終端機模擬器的支援。例如，你可以在 Arch Linux 中設定 kitty，並且可以使用像 wofi 這樣的應用程式管理員來啟動它。

Linux 可說是擁有最多樣化的終端機應用程式可供選擇。這些大多數都可以透過你的發行版的軟體包管理員安裝。

「終端機」、「shell」和「命令列」之間的區別有時看起來相當吹毛求疵。而且在大多數情況下，這些術語通常可以互換使用。但重要的是要知道它們有特定的意義，以及這些意義分別是什麼。
