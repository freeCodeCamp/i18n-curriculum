# --interactive--

في JavaScript، هناك عدة طرق للتحقق مما إذا كان الكائن يحتوي على خاصية معينة. فهم هذه الطرق مهم للعمل بفعالية مع الكائنات، خاصة عند التعامل مع بيانات من مصادر خارجية أو عندما تحتاج إلى التأكد من وجود خصائص معينة قبل استخدامها.

سنستعرض بعض الطرق الشائعة: طريقة `hasOwnProperty()`، وطريقة `Object.hasOwn()`، وعامل `in`، والتحقق مقابل `undefined`.

لنبدأ بطريقة `hasOwnProperty()`. تعيد هذه الطريقة قيمة منطقية تشير إلى ما إذا كان الكائن يحتوي على الخاصية المحددة كخاصية خاصة به. إليك مثالاً:

:::interactive_editor

```js
const person = {
  name: "Alice",
  age: 30
};

console.log(person.hasOwnProperty("name")); // true
console.log(person.hasOwnProperty("job")); // false
```

:::

في هذا المثال، لدينا كائن يسمى `person` يحتوي على خاصيتين: `name` و `age`. للتحقق مما إذا كانت `name` خاصية في كائن `person`، نستخدم طريقة `hasOwnProperty()`. بما أن `name` خاصية، فستُعيد القيمة `true`. ولكن عند استخدام `hasOwnProperty()` للتحقق مما إذا كانت `job` خاصية، ستُعيد `false` لأنها غير موجودة في الكائن.

`Object.hasOwn()` هو الطريقة الحديثة والمُوصى بها للتحقق مما إذا كان الكائن يحتوي على خاصية خاصة به (ليست موروثة). فكر فيها كنسخة مطورة وأكثر أمانًا من `hasOwnProperty()`. الصيغة هي `Object.hasOwn(object, propertyName)` — تمرر الكائن كالمعلمة الأولى واسم الخاصية كالمعلمة الثانية.

إليك مثالًا بسيطًا:

:::interactive_editor

```js
const person = {
  name: "Alice",
  age: 30
};

console.log(Object.hasOwn(person, "name")); // true
console.log(Object.hasOwn(person, "job")); // false
```

:::

في هذا المثال، تُعيد `Object.hasOwn(person, "name")` القيمة `true` لأن `name` موجودة مباشرة على كائن `person`. وتُعيد `Object.hasOwn(person, "job")` القيمة `false` لأن `job` لم تُضاف أبدًا إلى الكائن.

أمر مهم جدًا يجب فهمه هو أن `Object.hasOwn()` يتحقق فقط مما إذا كانت الخاصية **موجودة** — ولا يهتم بقيمة الخاصية. هذا يعني أنه يعيد `true` حتى عندما تكون القيمة `0` أو `false` أو `null` أو `undefined`:

:::interactive_editor

```js
const user = {
  username: "coder123",
  score: 0,
  isActive: false,
  nickname: null
};

// Object.hasOwn() correctly reports these all exist
console.log(Object.hasOwn(user, "score"));    // true  (value is 0, but property exists)
console.log(Object.hasOwn(user, "isActive")); // true  (value is false, but property exists)
console.log(Object.hasOwn(user, "nickname")); // true  (value is null, but property exists)
console.log(Object.hasOwn(user, "email"));   // false (property was never added)

// Danger! Using if() directly gives wrong results for falsy values
if (user.score) {
  console.log("Has score"); // This will NOT print even though score exists!
}

// Safe! Object.hasOwn() gives correct result
if (Object.hasOwn(user, "score")) {
  console.log("Has score:", user.score); // Has score: 0
}
```

:::

طريقة أخرى للتحقق من وجود خاصية في كائن هي استخدام عامل `in`. مثل `hasOwnProperty()`، يعيد عامل `in` القيمة `true` إذا كانت الخاصية موجودة في الكائن. إليك كيفية استخدامه:

:::interactive_editor

```js
const person = {
  name: "Bob",
  age: 25
};
console.log("name" in person);  // true
```

:::

في هذا المثال، تُعيد القيمة `true` عند استخدام التعبير `"name" in person` لأن `name` خاصية في `person`.

الطريقة الثالثة تتضمن التحقق مما إذا كانت الخاصية `undefined`. يمكن أن تكون هذه الطريقة مفيدة، لكنها لها بعض القيود. إليك مثالاً:

:::interactive_editor

```js
const car = {
  brand: "Toyota",
  model: "Corolla",
  year: 2020
};

console.log(car.brand !== undefined); // true
console.log(car.color !== undefined); // false
```

:::

في هذا الكود، نتحقق مما إذا كانت `car.brand` و `car.color` ليستا `undefined`. هذا يعمل لأن الوصول إلى خاصية غير موجودة في كائن يُعيد `undefined`. مع ذلك، قد تعطي هذه الطريقة نتائج خاطئة إذا كانت الخاصية تحمل القيمة `undefined` صراحةً.

في الممارسة العملية، يعتمد اختيار الطريقة بين هذه الطرق غالبًا على متطلبات شفرتك الخاصة. فهم الفروقات بينها سيساعدك على اتخاذ القرار الصحيح في سيناريوهات مختلفة.
