# --description--

في درس سابق، تعلمت عن بادئة الخصائص بشرطة سفلية واحدة وشرطتين سفليتين.

لتذكيرك بالفرق بينهما، الشرط السفلية الواحدة هو اصطلاح يعني أن الخاصية مخصصة للاستخدام الداخلي داخل الصنف ولا يجب الوصول إليها مباشرة من خارج الصنف. أما الشرطتان السفليتان فتمنعان الوصول المباشر إلى تلك الخاصية من خارج الصنف.

إليك مثال يوضح كيف يعمل كلاهما:

```py
class Example:
    def __init__(self):
        self._internal = 'I can be accessed from outside the class, but should not'
        self.__private = 'You cannot access me directly from outside the class'

obj = Example()

print(obj._internal) # I can be accessed from outside the class, but should not
print(obj.__private)  # AttributeError: 'Example' object has no attribute '__private'
```

إضافة بادئة من شرطتين سفليتين لخاصية تُفعّل عملية تشويش الأسماء في بايثون، حيث يعيد بايثون تسمية الخاصية داخليًا بإضافة شرطة سفلية واسم الصنف كبادئة، محولًا `__attribute` إلى `_ClassName__attribute`.

لرؤية ذلك عمليًا، أنشئ نسخة من الصنف واستخدم الخاصية الخاصة `__dict__` لتلك النسخة، وهي قاموس يحتوي على خصائص الكائن:

```py
class Example:
    def __init__(self, internal, private):
        self._internal = internal
        self.__private = private

example1 = Example(
    'I can be accessed from outside the class, but should not',
    'I cannot be accessed directly from outside the class'
)

print(example1.__dict__)
```

النتيجة ستكون:

```py
{
  '_internal': 'I can be accessed from outside the class, but should not',
  '_Example__private': 'I cannot be accessed directly from outside the class'
}
```

كما ترى، الخاصية `__private` مخزنة كـ `_Example__private`. هذا يعني أنه لا يزال بإمكانك الوصول إلى تلك الخاصية من خارج الصنف بهذه الطريقة:

```py
class Example:
    def __init__(self, internal, private):
        self._internal = internal
        self.__private = private

example1 = Example(
    'I can be accessed from outside the class, but should not',
    'I cannot be accessed directly from outside the class'
)
example2 = Example(
    'I should not be accessed from outside the class',
    'But I can be accessed from outside the class with name mangling'
)

print(example1._Example__private) # I cannot be accessed directly from outside the class
print(example2._Example__private) # But I can be accessed from outside the class with name mangling
```

لماذا يقوم بايثون بتشويش الأسماء؟

الغرض الرئيسي من تشويش الأسماء هو منع التجاوز العرضي للخصائص والطرق عند استخدام الوراثة. إليك مثال يوضح ذلك:

```py
class Parent:
    def __init__(self):
        self.__data = 'Parent data'

class Child(Parent):
    def __init__(self):
        super().__init__()
        self.__data = 'Child data'

c = Child()
print(c.__dict__) # {'_Parent__data': 'Parent data', '_Child__data': 'Child data'}
```

يمكنك أن ترى أن كلا الصنفين `Parent` و`Child` الذي يرث منه يمتلكان خاصيتي `_class__data` منفصلتين. هذا ممكن بفضل عملية تشويش الأسماء. وإلا، كان `Child` سيُعيد كتابة بيانات Parent عن طريق الخطأ.

هذا ما كان سيحدث لو لم يسمح بايثون بتشويش الأسماء، أي إذا لم تضف بادئة الشرطتين السفليتين للخصائص في كلا الصنفين:

```py
class Parent:
   def __init__(self):
       self.data = 'Parent data'

class Child(Parent):
   def __init__(self):
       super().__init__()
       self.data = 'Child data'

c = Child()
print(c.__dict__)  # {'data': 'Child data'}
```

فأي بادئة يجب أن تستخدم للخصائص بين الشرط السفلية الواحدة (`_`) والشرطتين السفليتين (`__`)? الأمر يعتمد. إذا كانت الخاصية مخصصة فقط للاستخدام الداخلي داخل الصنف، فالتزم بالشرط السفلية الواحدة.

لكن إذا كنت تعمل مع صنف سيتم وراثته، يجب أن تستخدم الشرطتين السفليتين حتى لا يتم تجاوز الخاصية من الصنف الأصل.
