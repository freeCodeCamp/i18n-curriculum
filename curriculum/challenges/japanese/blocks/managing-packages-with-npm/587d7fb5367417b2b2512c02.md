# --description--

前のチャレンジでは、npmに特定のバージョンのパッケージだけを含めるように指示しました。これは、プロジェクトの異なる部分が互換性を保つ必要がある場合に依存関係を固定する便利な方法です。しかし、ほとんどのケースでは、重要なセキュリティパッチを含むことが多いバグ修正を見逃したくありませんし、（うまくいけば）それによって問題が起きることもありません。

npmの依存関係を最新のパッチバージョンに更新できるようにするには、依存関係のバージョンの前にチルダ（`~`）文字を付けます。以下は、任意の`1.3.x`バージョンへの更新を許可する例です。

```json
"package": "~1.3.8"
```

# --instructions--

package.jsonファイルでは、現在のnpmが`@freecodecamp/example`をアップグレードするルールは特定のバージョン（`1.2.13`）を使うようになっています。しかし今は、最新の`1.2.x`バージョンを許可したいです。

依存関係の`~`のバージョンの前にチルダ（`@freecodecamp/example`）文字を付けて、npmが新しいパッチリリースに更新できるようにしてください。

**注意:** バージョン番号自体は変更しないでください。

# --hints--

`"dependencies"`には`"@freecodecamp/example"`を含めるべきです。

```js
fetch(code + '/_api/package.json')
  .then(response => response.json())
  .then(
    data => {
      assert.property(
        data.dependencies,
        '@freecodecamp/example',
        '"dependencies" does not include "@freecodecamp/example"'
      );
    },
    error => {
      throw new Error(error.message || error.responseText);
    }
  );
```

`"@freecodecamp/example"`のバージョンは`"~1.2.13"`と一致する必要があります。

```js
fetch(code + '/_api/package.json')
  .then(response => response.json())
  .then(
    data => {
      assert.match(
        data.dependencies['@freecodecamp/example'],
        /^\~1\.2\.13/,
        'Wrong version of "@freecodecamp/example". It should be ~1.2.13'
      );
    },
    error => {
      throw new Error(error.message || error.responseText);
    }
  );
```
