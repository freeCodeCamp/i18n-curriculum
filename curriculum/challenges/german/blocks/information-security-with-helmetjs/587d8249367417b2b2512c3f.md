# --description--

Zur Erinnerung: Dieses Projekt basiert auf dem folgenden Starter-Projekt, das von <a href="https://github.com/freeCodeCamp/boilerplate-infosec/" target="_blank" rel="noopener noreferrer nofollow">GitHub</a> geklont wurde.

Diese Aufgabe hebt eine vielversprechende neue Schutzmaßnahme hervor, die das Risiko und die Auswirkungen vieler Arten von Angriffen in modernen Browsern erheblich reduzieren kann. Durch das Festlegen und Konfigurieren einer Content-Security-Policy können Sie verhindern, dass unerwünschte Inhalte in Ihre Webseite eingeschleust werden. Dies schützt Ihre App vor XSS-Sicherheitslücken, unerwünschtem Tracking, bösartigen Frames und vielem mehr. CSP funktioniert, indem eine erlaubte Liste vertrauenswürdiger Inhaltsquellen definiert wird. Sie können diese für jede Art von Ressource konfigurieren, die eine Webseite benötigt (Skripte, Stylesheets, Schriftarten, Frames, Medien und so weiter). Es gibt mehrere Direktiven, sodass Website-Besitzer eine feingranulare Kontrolle haben. Weitere Details finden Sie bei HTML 5 Rocks und KeyCDN. Leider wird CSP von älteren Browsern nicht unterstützt.

Standardmäßig sind die Direktiven sehr offen, daher ist es wichtig, die Direktive defaultSrc als Fallback festzulegen. Helmet unterstützt sowohl die Schreibweisen defaultSrc als auch default-src. Der Fallback gilt für die meisten nicht spezifizierten Direktiven.

# --instructions--

Verwenden Sie in dieser Übung `helmet.contentSecurityPolicy()`. Konfigurieren Sie es, indem Sie ein `directives`-Objekt hinzufügen. Legen Sie in dem Objekt `defaultSrc` auf `["'self'"]` fest (die Liste der erlaubten Quellen muss in einem Array stehen), um standardmäßig nur Ihrer eigenen Webseite zu vertrauen. Setzen Sie außerdem die Direktive `scriptSrc`, sodass Skripte nur von Ihrer Webseite (`'self'`) und von der Domain `'trusted-cdn.com'` heruntergeladen werden dürfen.

Hinweis: Im Schlüsselwort `'self'` sind die einfachen Anführungszeichen Teil des Schlüsselworts selbst, daher muss es in doppelte Anführungszeichen eingeschlossen werden, damit es funktioniert.

# --hints--

Die Middleware helmet.contentSecurityPolicy() sollte korrekt eingebunden sein

```js
const response = await fetch(code + '/_api/app-info');
if (!response.ok) {
  throw Error(await response.text());
}
const data = await response.json();
assert.include(data.appStack, 'csp');
```

Ihre CSP-Konfiguration ist nicht korrekt. defaultSrc sollte ["'self'"] sein und scriptSrc sollte ["'self'", 'trusted-cdn.com'] sein

```js
const response = await fetch(code + '/_api/app-info');
if (!response.ok) {
  throw Error(await response.text());
}
const data = await response.json();
var cspHeader = Object.keys(data.headers).filter(function (k) {
  return (
    k === 'content-security-policy' ||
    k === 'x-webkit-csp' ||
    k === 'x-content-security-policy'
  );
})[0];
assert.equal(
  data.headers[cspHeader],
  "default-src 'self'; script-src 'self' trusted-cdn.com"
);
```
