# --description--

Node.js non ha un server WebSocket integrato. Gli sviluppatori di solito usano pacchetti esterni come `ws` per implementare il protocollo WebSocket e gestire la comunicazione in tempo reale tra client e server.

La libreria `ws` è leggera, veloce e facile da usare. Implementa il protocollo WebSocket così puoi creare e gestire connessioni WebSocket.

Puoi installarla con npm:

```bash
npm install ws
```

Una volta installata, crea un server WebSocket importando la libreria `ws`, invocando il suo metodo `Server` e passando una porta:

```js
const WebSocket = require("ws");

const server = new WebSocket.Server({
  port: 3000,
});
```

Questo avvia un server WebSocket che ascolta le connessioni sulla porta `3000`.

Quando un client si connette, il server emette un evento `connection`.

```js
server.on("connection", (socket) => {
  console.log("Client connected");
});
```

Pensa a questo evento come al modo in cui il server dice "un nuovo client si è unito".

Il parametro `socket` rappresenta la connessione tra il server e quel client. Lo usi per inviare e ricevere messaggi.

Quando un client connesso invia dati, il server emette un evento `message`.

```js
server.on("connection", (socket) => {
  socket.on("message", (message) => {
    console.log(message.toString());
  });
});
```

In questo esempio, il server ascolta i messaggi in arrivo e li registra nella console.

Puoi inviare dati a un client usando il metodo `send()`:

```js
socket.send("Hello from the server");
```

Questo invia un messaggio attraverso la connessione WebSocket aperta.

Finché la connessione rimane aperta, sia il client sia il server possono continuare a scambiarsi messaggi in tempo reale.

Oltre a `connection` e `message`, le connessioni WebSocket emettono anche gli eventi `close` e `error`.

`close` si attiva quando una connessione si chiude:

```js
socket.on("close", () => {
  console.log("Disconnected");
});
```

`error` si attiva quando qualcosa va storto:

```js
socket.on("error", (error) => {
  console.error(error);
});
```

Insieme, questi eventi formano la base della maggior parte delle applicazioni WebSocket.

Un requisito comune nelle app di chat è la trasmissione a più destinatari. Trasmettere significa inviare un messaggio a più client connessi invece che a uno solo.

Immagina una chat con tre utenti chiamati **John Doe**, **Jane Doe** e **Wes Smith**.

Se John Doe invia un messaggio, il server potrebbe dover inoltrare quel messaggio a Jane Doe e Wes Smith.

Per farlo, il server tiene traccia dei client connessi e li itera quando arriva un messaggio.

Prima di inviare un messaggio, il server verifica se la connessione del client è ancora aperta:

```js
if (client.readyState === WebSocket.OPEN) {
  // do something
}
```

Ogni connessione WebSocket ha una proprietà `readyState` che descrive il suo stato attuale.

`WebSocket.OPEN` significa che la connessione è attiva e pronta per inviare e ricevere dati.

Controllando questo, il server evita di inviare messaggi a client che si sono già disconnessi o le cui connessioni sono ancora in fase di apertura:

```js
server.on("connection", (socket) => {
  socket.on("message", (message) => {
    server.clients.forEach((client) => {
      if (client.readyState === WebSocket.OPEN) {
        client.send(message.toString());
      }
    });
  });
});
```

In questo esempio, ogni client connesso riceve il messaggio in arrivo. Questo si chiama trasmissione a più destinatari.

Molte applicazioni in tempo reale usano la trasmissione per consegnare dati come messaggi di chat, notifiche live, aggiornamenti sportivi ed eventi di giochi multiplayer.

Con la libreria `ws` è semplice costruire server WebSocket in Node.js.

Tutto ciò che devi fare è creare un server, ascoltare gli eventi, scambiare messaggi e trasmettere aggiornamenti ai client connessi.

Questi elementi costituiscono la base di molte applicazioni in tempo reale.
