## npm ni nini?

- **Maana**: npm ni rejista mtandaoni ya programu za chanzo huria, inayofanya kazi kama "duka la programu" kwa msimbo.
- **Maana Rasmi**: npm haimaanishi rasmi "Node Package Manager"; ni kifupisho cha mzunguko kinachomaanisha "npm si kifupi cha neno lolote".
- **Sehemu Tatu Zinazohusiana**:

  - **Tovuti (npmjs.com)**: Kwa kutafuta vifurushi katika rejista.
  - **Rejista**: Hifadhidata kubwa ya umma inayohifadhi vifurushi vyote.
  - **CLI (Kiolesura cha mstari wa amri)**: Zana inayosakinishwa pamoja na Node.js kusimamia utegemezi wa mradi.

- **Mbadala**: Ingawa npm ni inayotumika zaidi, mbadala maarufu ni **Yarn**, **PNPM**, na **Bun**.

## Faili ya package.json

- **Madhumuni**: Faili ya usanidi inayohifadhi metadata, utegemezi, na maandishi ya kuendesha kwa mradi wa Node.js.
- **Sehemu Muhimu**:

  - `name` na `version`: Hutatua mradi.
  - `main`: Inaeleza faili la kuingia, mara nyingi `index.js`.
  - `license`: Hufafanua masharti ya kisheria (chaguo-msingi ni `ISC`).
  - `dependencies`: Vifurushi vinavyohitajika kuendesha programu katika uzalishaji.
  - `devDependencies`: Zana zinazotumika tu wakati wa ukuzaji, kama mifumo ya majaribio.

- **Uundaji**:

  - `npm init`: Huanza mchakato wa usanidi wa kuingiliana.
  - `npm init -y`: Husakinisha `package.json` kwa thamani za chaguo-msingi mara moja.

## Utoaji wa toleo la kimaelezo (Semantic Versioning - SemVer)

- **Muundo**: `MAJOR.MINOR.PATCH`.
- **Sheria**:

  - **MAJOR**: Mabadiliko yanayovunja msimbo uliopo.
  - **MINOR**: Vipengele vipya vinavyolingana na toleo la awali.
  - **PATCH**: Marekebisho madogo ya hitilafu na maboresho.

- **Alama za wigo**:

  - `^` (Ishara ya caret): Inaruhusu masasisho madogo na ya marekebisho (mfano, `^1.2.3` huruhusu `1.x.x`).
  - `~` (Tilde): Inaruhusu masasisho ya marekebisho pekee (mfano, `~1.2.3` huruhusu `1.2.x`).
  - `*` (Nyota): Inalingana na toleo lolote; hutumika hasa kwa majaribio.

## Usimamizi wa utegemezi

- **Usakinishaji**:

  - `npm install <name>`: Huongeza utegemezi wa uzalishaji.
  - `npm install <name> -D`: Huongeza utegemezi wa ukuzaji.
  - `npm install <name>@<version>`: Husakinisha toleo maalum.

- **Moduli za Node na Locks**:

  - `node_modules/`: Inahifadhi msimbo halisi wa vifurushi vilivyosakinishwa.
  - `package-lock.json`: Hufunga toleo halisi la utegemezi wote na "utegemezi wa watoto" kuhakikisha uthabiti katika mazingira tofauti.

- **Uondoaji**: `npm uninstall <name>` huondoa kifurushi na kusasisha `package.json`.
- **Matengenezo**:

  - `npm outdated`: Hutatua vifurushi vyenye matoleo mapya yanayopatikana.
  - `npm update`: Husasisha vifurushi ndani ya wigo wa SemVer unaoruhusiwa.

## Maandishi ya npm

- **Kazi**: Amri maalum zilizofafanuliwa katika sehemu ya `scripts` ya `package.json` ili kuendesha kazi kwa otomatiki.
- **Uendeshaji**: Endeshwa kupitia `npm run <script_name>`.
- Kuna vifupisho kwa `start`, `test`, `stop`, na `restart` (mfano, `npm start`).
- **Uendeshaji wa mfululizo**:
- `&&`: Huendesha amri ya pili tu ikiwa ya kwanza itafanikiwa.
- `;`: Huendesha amri moja baada ya nyingine bila kujali mafanikio.
- **Uendeshaji wa sambamba**: Tumia kiendeshaji `&` kuendesha michakato kwa wakati mmoja.
- **Hoja**: Pitia hoja kwa maandishi kwa kutumia kitenzi `--` (mfano, `npm start -- 8000`).

## Kuchapisha kwenye rejista ya npm

- **Upatikanaji wa rejista**: Hifadhidata ya vifurushi vya umma na binafsi.
- **Hatua za kuchapisha**:

1. Unda akaunti ya bure kwenye `npmjs.com`.
2. Ingia kupitia terminal kwa kutumia `npm login`.
3. Hakikisha `name` ni ya kipekee na `version` halali ziko katika `package.json`.
4. Unda `README.md` na `.npmignore` (kuepuka faili nyeti).
5. Endesha `npm publish` (au `npm publish --access public` kwa vifurushi vilivyo na wigo).

- **Wigo**: Hutumika kuepuka migongano ya majina kwa kuchapisha chini ya `@username/package-name`.

## Mifumo ya moduli katika Node.js

### CommonJS (CJS)

- **Mfumo wa chaguo-msingi**: Njia ya asili Node.js inavyoshughulikia moduli; huingiza kwa wakati mmoja.
- **Sintaksia**:
- **Import**: `const module = require('./path')`.
- **Export**: `module.exports = { functionName }`.
- **Utambulisho**: Inatumika kwa faili za `.cjs` au faili za `.js` wakati hakuna `type` iliyoainishwa katika `package.json`.

### ES Modules (ESM)

- **Muundo uliosawazishwa**: Kiwango cha kisasa cha JavaScript; huingiza isiyo ya wakati mmoja kwa ufanisi.
- **Sintaksia**:

  - **Export**: `export { name }` au `export default function`.
  - **Import**: `import { name } from './path.mjs'`.
  - **Utambulisho**: Inatumika kwa faili za `.mjs` au faili za `.js` wakati `package.json` ina `"type": "module"`.
