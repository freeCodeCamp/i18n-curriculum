# --description--

في الدروس السابقة، تعلمت كيفية العمل مع الأنواع الأولية بالإضافة إلى أنواع الاتحاد والواجهات.

لكن TypeScript يحتوي على بعض الأنواع الخاصة التي يجب أن تكون على دراية بها. لنبدأ بالنظر إلى النوع `any`.

يُستخدم النوع `any` لتمثيل أي نوع من القيم.

```ts
let randomValue: any;

randomValue = 42;           

randomValue = "Hello";     
 
randomValue = true;        

randomValue = { name: "Alice" }; 
```

مع هذا النوع، يجب أن تكون حذرًا لأنه يمكن استخدامه بشكل مفرط وسوء استخدامه بسهولة. لا تريد كتابة كل شيء بنوع `any` فقط لإسكات رسائل الخطأ في TypeScript. هذا يُفقد الهدف من استخدام TypeScript وإضافة أمان النوع في المقام الأول.

النظير الأكثر أمانًا لنوع `any` هو النوع `unknown`. النوع `unknown` مشابه لـ `any` لكن مع النوع `unknown`، يجب عليك إجراء فحص نوع للمتغير قبل استخدامه.

إليك مثالًا:

```ts
function doubleValue(value: unknown) {
  if (typeof value === "number") {
    console.log(value * 2);
  } else if (typeof value === "string") {
    console.log(value + value);
  }
}

doubleValue(10);    
doubleValue("Hi "); 
doubleValue(true);   
```

في هذا المثال، تضمن فحوصات النوع أن العمليات تُنفذ فقط على القيم من النوع الصحيح. القيم التي لا تتطابق مع الأنواع المتوقعة تُتجاهل.

النوع التالي الذي سننظر إليه هو النوع `never`. هذا النوع يمثل شيئًا لن يحدث أبدًا.

في معظم الحالات، لن تكتب تعليقات نوع باستخدام `never`. بدلاً من ذلك، سترى غالبًا نوع `never` يظهر في رسائل الخطأ مثل هذه:

```ts
function processValue(value: string | number) {
  if (typeof value === "string") {
    console.log("String value:", value.toUpperCase());
  } else if (typeof value === "number") {
    console.log("Number value:", value * 2);
  } else {
    console.log(value); 
  }
}
```

داخل بند `else` في المثال أعلاه، يكون للمتغير `value` نوع `never`. هذا لأن فرع `else` أصبح من المستحيل الوصول إليه ولا يوجد نوع متبقٍ. إذا حاولت استدعاء الدالة بقيمة غير `string` أو `number`، فسيظهر خطأ مثل هذا:

```ts
function processValue(value: string | number) {
  if (typeof value === "string") {
    console.log("String value:", value.toUpperCase());
  } else if (typeof value === "number") {
    console.log("Number value:", value * 2);
  } else {
    console.log(value); 
  }
}

// Argument of type 'boolean' is not assignable to parameter of type 'string | number'.
processValue(true); 
```

آخر نوع سننظر إليه هو النوع `void`. عندما تكون لديك دالة لا تُرجع قيمة، يكون ذلك من النوع `void`.

إليك مثالًا:

```ts
type Status = "loading" | "success" | "error";

type Handler = {
  status: Status;
  onChange: (newStatus: Status) => void; 
};
```

في هذا المثال، الخاصية `onChange` لنوع `Handler` لا تُرجع أي شيء، ولهذا السبب يُستخدم النوع `void`.
