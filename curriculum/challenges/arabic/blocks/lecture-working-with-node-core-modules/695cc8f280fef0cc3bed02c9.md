# --description--

وحدة OS هي وحدة قياسية أخرى مدمجة في Node.js.

تتيح لك التفاعل مع نظام التشغيل الحالي الذي يعمل عليه Node لتتمكن من الوصول إلى معلومات حيوية مثل نوع نظام التشغيل، تفاصيل وحدة المعالجة المركزية، الذاكرة المتاحة، إجمالي الذاكرة، واجهات الشبكة، والمزيد.

لاستخدام وحدة OS، استوردها بهذه الطريقة:

```js
const os = require("os");
```

توفر وحدة OS عدة طرق يمكنك استخدامها لجعل تطبيقك أكثر وعيًا بالنظام. لنلقِ نظرة على بعض الطرق الشائعة.

تسترجع الطريقة `platform()` نظام التشغيل الذي يعمل عليه Node حاليًا:

```js
const os = require("os");
console.log(os.platform()); // darwin
```

يمكن أن تكون مفيدة لتنفيذ السكريبتات عبر الأنظمة المختلفة:

```js
if (os.platform() === 'win32') {
 // Windows specific code
} else {
 // Non-Windows specific code
}
```

تعرض الطريقة `arch()` سلسلة نصية تمثل بنية وحدة المعالجة المركزية التي تم تجميع Node.js من أجلها:

```js
const os = require("os");
console.log(os.arch()); // arm64
```

هذا مفيد إذا أردت أن يقوم المستخدمون بتنزيل الملفات الثنائية والاعتمادات الصحيحة لبنية معينة.

القيم المحتملة الأخرى هي `'x64'`، `'arm'`، `'arm64'`، `'ia32'`، `'mips'`، `'ppc'`، وغيرها.

تحصل `type()` على الاسم الرسمي لنظام التشغيل، لتتمكن من التعرف على أنظمة التشغيل برمجيًا:

```js
const os = require("os");
console.log(os.type()); // Darwin (the core OS for macOS, iOS, and other Apple products)
```

تعرض `release()` إصدار نواة نظام التشغيل، وهو الجزء الأساسي من نظام التشغيل الذي يدير موارد النظام والتواصل بين مكونات الأجهزة والبرمجيات. يمكن أن تكون هذه الطريقة مفيدة لتتبع التوافق بين إصدارات نواة النظام ومتطلبات الخادم.

تعيد هذه القيمة كسلسلة نصية مثل `'20.6.0'`:

```js
const os = require("os");
console.log(os.release()); // 25.0.0
```

تعيد `version()` إصدار نظام التشغيل المحدد مع تفاصيل أكثر من طريقة `release()`:

```js
const os = require("os");
console.log(os.version());
// Darwin Kernel Version 25.0.0: Wed Sep 17 21:41:39 PDT 2025;
// root:xnu-12377.1.9~141/RELEASE_ARM64_T8103
```

تعيد `cpus()` مصفوفة من الكائنات التي تحتوي على تفاصيل عن كل نواة من أنوية وحدة المعالجة المركزية المنطقية. يمكن أن تساعد في مراقبة حمل وحدة المعالجة المركزية:

تعرض `uptime()` الوقت منذ تشغيل النظام. يمكن أن تساعد في تحديد مدة تشغيل الخوادم:

```js
const os = require("os");
console.log(os.uptime()); // 23047
```

تعرض `totalmem()` و `freemem()` إجمالي كمية ذاكرة النظام بالبايت والذاكرة الحرة بالبايت، على التوالي:

```js
const os = require("os");
console.log(os.totalmem()); // 8589934592 (8 GB)
console.log(os.freemem()); // 93585408 (87 MB)
```

تعيد `userInfo()` كائنًا يحتوي على معلومات عن المستخدم الحالي للنظام:

```js
const os = require("os");
console.log(os.userInfo());

/*

}
*/
```
