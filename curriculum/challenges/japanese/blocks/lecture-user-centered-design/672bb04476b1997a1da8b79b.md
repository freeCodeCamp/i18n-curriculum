# --description--

レイジー登録は、ユーザーが登録せずにアプリケーションを閲覧し操作できるようにするUIデザインパターンです。良い例としては、eコマースサイトがあります。ユーザーは商品を閲覧し、いくつかのアイテムをカートに追加できます。そして、購入に興味があれば登録が必要になります。

その理由は、ユーザーが情報を提供して登録する前に、サイトが提供する価値を実感する必要があるからです。アプリケーションを設計する際には、ユーザーが価値を感じ、情報を提供しても安全だと感じられるようにする必要があります。そうでなければ、登録をためらい、潜在的な顧客を失うことになります。

ユーザーの機密データが保護され安全であることを伝えることが重要です。後のモジュールで、アプリケーションのセキュリティ強化やユーザーデータの保護方法について説明します。

もう一つの良い例はYouTubeです。YouTubeは、テクノロジー、ポップカルチャー、ゲームなどあらゆるジャンルの動画が数百万本ある動画共有プラットフォームです。YouTubeを訪れると、ユーザーはサインインや登録なしで好きなだけ動画を視聴できます。しかし、動画に「いいね」したり、コメントしたり、チャンネル登録したりするには登録が必要です。

ユーザーが視聴しているコンテンツを気に入ったり、会話に参加したい場合は、登録する意欲が高まります。レイジー登録は、ユーザーが情報を提供する前にアプリケーションの価値を実感できる便利なデザインパターンです。

次にアプリケーションを設計するときは、ユーザーのエンゲージメントと定着率を高めるためにレイジー登録の利用を検討してください。
