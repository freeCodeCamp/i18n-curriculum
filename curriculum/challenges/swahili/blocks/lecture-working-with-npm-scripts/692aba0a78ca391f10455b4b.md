# --description--

Katika ukuzaji wa mtandao wa kisasa, unaweza kugawanya msimbo wako katika mafaili mengi yanayoitwa "moduli."

Unaweza kufikiria moduli kama kipande cha msimbo kilicho huru na kinachoweza kutumika tena.

Kwa kuleta moduli katika moduli nyingine, unaweza kutumia tena utendaji wake huku ukidumisha msimbo katika mafaili tofauti, jambo ambalo ni rahisi kudumisha, hasa kwa programu kubwa na ngumu za mtandao.

Katika Node.js, kila faili huchukuliwa kama moduli.

Node ina mifumo miwili tofauti ya kufanya kazi na moduli: CommonJS na ES.

CommonJS ni njia ya asili na kwa chaguo-msingi ya kufanya kazi na vifurushi katika Node.js. Inapakia moduli kwa wakati mmoja, ambayo ina maana kwamba inapaswa kusubiri moduli ipakie kikamilifu kabla ya kuanza kupakia nyingine au kuendelea na utekelezaji wa msimbo.

Bado inatumiwa sana, lakini polepole inabadilishwa na sintaksia mpya ya ES.

CommonJS hutumia kitendakazi cha `require()` kuleta moduli, kikichukua njia yake ya jamaa kama hoja.

Kwa mfano, tuseme tuna moduli inayoitwa `math.js`, ambapo tunaeleza shughuli za msingi za kihesabu.

Tutatumia kitendakazi cha kuzidisha hapa tu ili kuweka mfano huu rahisi na makini:

```js
function multiply(a, b) {
  return a * b;
}
```

Tunawezaje kutumia tena kitendakazi hiki nje ya moduli yetu ya sasa? Kwa chaguo-msingi, vitendakazi na vigezo vilivyotangazwa katika moduli ya Node.js ni **binafsi** — moduli nyingine haziwezi kuvipata isipokuwa tukaweka wazi kwa makusudi. Kuna njia kadhaa za kufanya hivyo.

Njia moja ya kawaida ni kuweka thamani kwa `module.exports`. Kitu hiki kinawakilisha kile moduli itakachorudisha inapoletawa mahali pengine, na kuwapa moduli nyingine upatikanaji wa vitendakazi, vigezo, au vitu ulivyochagua kuweka wazi.

Katika mfano huu, tunaweka kitu chenye sifa ya `multiply` kwa `module.exports`. Thamani ya sifa hiyo ni rejea kwa kitendakazi cha `multiply` tulichokitangaza awali. Hii itasafirisha kitendakazi, hivyo tunaweza kukiita kutoka moduli nyingine ikiwa tutahitaji.

```javascript
function multiply(a, b) {
  return a * b;
}

module.exports = {
  multiply: multiply
};
```

Tuseme tunajaribu kuleta moduli ya `math.js` katika moduli nyingine. Tunaita kitendakazi cha `require()`, na kupitisha njia ya jamaa kama hoja. Kisha, tunaweka kitu kilichorudiwa na `require()` katika kigezo cha `math`.

```javascript
const math = require('./math');
```

Tuchunguze thamani ya `math` kwenye konsoli kwa kuichapisha kwa `console.log()`:

```javascript
console.log(math);
```

Hii ndiyo matokeo. Ni kitu chenye sifa ya `multiply` na kitendakazi kama thamani:

```javascript
{ multiply: [Function: multiply] }
```

Ikiwa tuitaje kitendakazi hiki na `math.multiply()`, tukipitisha nambari `6` na `7` kama hoja, matokeo ni `42`. Kitendakazi kinafanya kazi kwa mafanikio na bado, kiko katika faili tofauti! Hivyo kauli yetu ya kuleta ilifanya kazi vizuri.

```javascript
const math = require('./math');

console.log(math.multiply(6, 7)); // 42
```

Lakini kuna njia nyingine za kusafirisha kitendakazi, kigezo, au kitu.

Kwa mfano, ikiwa unahitaji kusafirisha kitendakazi kimoja tu, kigezo, au kitu, unaweza kukiweka moja kwa moja kama thamani ya `module.exports`:

```javascript
function multiply(a, b) {
  return a * b;
}

module.exports = multiply;
```

Ikiwa unahitaji kusafirisha vitendakazi vingi, unahitaji tu kuviweka kwenye kitu cha `module.exports`, kimegawanywa kwa koma.

Hapa, unaweza kuona kwamba kitu kilichosafirishwa kina sifa mbili: `subtract` na `multiply`. Thamani zao ni rejea kwa vitendakazi vinavyolingana, hivyo utakuwa na upatikanaji wa vyote viwili katika moduli yoyote inayolleta `math.js`:

```javascript
function subtract(a, b) {
  return a - b;
}

function multiply(a, b) {
  return a * b;
}

module.exports = {
  subtract: subtract,
  multiply: multiply
};
```

Wakati kitu cha `module.exports` kina sifa nyingi, utakuwa na chaguo la kuchagua kama unataka kuleta kila kitu au kuleta sifa moja moja.

Katika mfano huu, tunaleta kitu kizima kilichosafirishwa na `math.js`, hivyo tutakuwa na upatikanaji wa `subtract` na `multiply`:

```javascript
const math = require('./math');

console.log(math.subtract(6, 7));  // -1
console.log(math.multiply(6, 7));  // 42
```

Ikiwa tunataka kuleta `multiply` tu badala yake, tunahitaji tu kuandika jina lake ndani ya mabano ya mviringo, baada ya neno kuu la `const`. Kimsingi, unatumia sintaksia ya mgawo wa ufumbuzi wa muundo kupata thamani yake kutoka kwa kitu kilichosafirishwa. Kisha, tunaweza kurejea moja kwa moja kwa jina lake katika msimbo wetu:

```javascript
const { multiply } = require('./math');

console.log(multiply(6, 7));
```

Hii inafanya kazi kwa vitu vingi pia. Unahitaji tu kuvitenganisha kwa koma ndani ya mabano ya mviringo. Hapa, tunaleta vitendakazi vya `subtract` na `multiply`:

```javascript
const { subtract, multiply } = require('./math');
```

Kwa chaguo-msingi, Node.js itachukulia yafuatayo kama moduli za CommonJS:

* Mafaili yenye nyongeza ya `.cjs`.
    
* Mafaili yenye nyongeza ya `.js` wakati faili la `package.json` la mzazi wa karibu lina sehemu ya juu ya ngazi `"type"` yenye thamani ya `"commonjs"`.
    
* Mafaili yenye nyongeza ya `.js` au bila nyongeza, wakati faili la `package.json` la mzazi wa karibu halina sehemu ya juu ya ngazi `"type"`, au hakuna `package.json` katika folda yoyote ya mzazi.
    
* Mafaili yenye nyongeza isiyo `.mjs`, `.cjs`, `.json`, `.node`, au `.js`.
    

Wakati Moduli za ES zinabadilisha polepole moduli za CommonJS, kuelewa hizi bado ni muhimu kwa wasanidi programu wanaofanya kazi na misimbo ya Node.js iliyopo na vifurushi vingi vya npm.
