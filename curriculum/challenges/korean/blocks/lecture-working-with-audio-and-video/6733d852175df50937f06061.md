# --description--

오디오와 비디오를 재생하는 대신, 때때로 오디오나 비디오를 캡처하고 싶을 수 있습니다. Media Capture and Streams API, 즉 MediaStream API가 이를 가능하게 합니다.

하지만 이 API를 사용하려면 `MediaStream` 객체를 생성해야 합니다. 생성자를 사용해 만들 수도 있지만, 그러면 사용자의 하드웨어와 연결되지 않습니다. 대신, 전역 `mediaDevices` 객체의 `navigator` 속성에는 사용할 수 있는 `getUserMedia()` 메서드가 있습니다.

이 메서드는 수신하려는 미디어 유형을 정의하는 단일 `constraints` 객체를 받습니다. 이 객체는 오디오와 비디오 스트림을 나타내는 `audio`와 `video` 속성을 가집니다. 이 속성들은 해당 유형의 스트림을 받고 싶지 않으면 `false`, 받고 싶으면 `true`이거나 추가 제약 조건을 정의하는 객체일 수 있습니다.

예를 들어, 특정 해상도의 비디오 출력을 요구할 수 있습니다:

```js
window.navigator.mediaDevices.getUserMedia({
  audio: true,
  video: {
    width: {
      min: 1280,
      ideal: 1920,
      max: 3840
    },
    height: {
      min: 720,
      ideal: 1080,
      max: 2160
    }
  }
});
```

이 제약 조건 객체는 비디오 피드의 최솟값과 최댓값 해상도를 지정합니다. `ideal` 속성은 가장 선호하는 해상도를 나타내며, 스트림은 이상적인 해상도에 가장 가까운 해상도를 제공합니다.

MediaStream을 생성한 후(사용자가 하드웨어 접근 자동 요청을 승인했다고 가정하면), 스트림 데이터를 필요에 따라 사용할 수 있습니다.

`getUserMedia()`는 프로미스를 반환한다는 점에 유의하세요. 따라서 스트림을 처리하려면 콜백 함수가 필요하거나 async/await 구문을 사용해야 합니다. 프로미스와 비동기 프로그래밍에 대해서는 이후 수업에서 더 배우게 됩니다.

다음은 사용자의 웹캠 피드를 페이지에 렌더링하는 기본 예제입니다.

```js
const video = document.querySelector("video");
const stream =
  await window.navigator.mediaDevices.getUserMedia({ video: true });
video.srcObject = stream;
await video.play();
```

이 API는 화면 캡처 접근 권한을 제공하지 않는다는 점도 알아두세요. 화면 캡처 API는 다음 수업에서 배우게 됩니다.
