# --interactive--

JavaScript — це мова, де іноді все працює несподівано або навіть дивно. Одне з таких несподіванок трапляється, коли ви змішуєте числа та рядки в обчисленнях. Перше, що ви, ймовірно, спробуєте — це додати число і рядок. В JavaScript оператор `+` виконує подвійне призначення. Він обробляє і додавання, і конкатенацію рядків, що є способом об’єднати два рядки разом.

Коли ви використовуєте `+` з числом і рядком, JavaScript вирішує трактувати їх обох як рядки і об’єднує їх. Якщо перевірити тип результату оператором `typeof`, ви побачите, що це дійсно рядок:

:::interactive_editor

```js
const result = 5 + '10';

console.log(result); // 510
console.log(typeof result); // string
```

:::

Як ви думаєте, що станеться, якщо поміняти місцями `5` і `'10'`?

:::interactive_editor

```js
const result = '10' + 5;

console.log(result); // 105
console.log(typeof result); // string
```

:::

Ви бачите, що сталося те саме. JavaScript бачить рядок у `'10'` і число в `5`, тому він перетворює число на рядок і конкатенує їх. Це називається перетворенням типів. Перетворення типів — це коли значення одного типу даних конвертується в інший.

Ситуація стає цікавішою, коли ви намагаєтеся виконати інші арифметичні дії, як віднімання, множення або ділення з рядком і числом. У таких випадках JavaScript намагається перетворити рядок у число перед виконанням обчислення — ще одне перетворення типів! Ось що відбувається:

:::interactive_editor

```js
const subtractionResult = '10' - 5;
console.log(subtractionResult); // 5
console.log(typeof subtractionResult); // number

const multiplicationResult = '10' * 2;
console.log(multiplicationResult); // 20
console.log(typeof multiplicationResult); // number

const divisionResult = '20' / 2;
console.log(divisionResult); // 10
console.log(typeof divisionResult); // number
```

:::

У наведених прикладах JavaScript успішно перетворює рядок `'10'` або `'20'` у число, а потім виконує обчислення. В результаті `'10' - 5` дає `5`, `'10' * 2` дає `20`, а `'20' / 2` дає `10`.

Але що, якщо рядок не схожий на число? Давайте подивимося, що станеться в такому випадку:

:::interactive_editor

```js
const subtractionResult = 'abc' - 5;
console.log(subtractionResult); // NaN
console.log(typeof subtractionResult); // number

const multiplicationResult = 'abc' * 2;
console.log(multiplicationResult); // NaN
console.log(typeof multiplicationResult); // number

const divisionResult = 'abc' / 2;
console.log(divisionResult); // NaN
console.log(typeof divisionResult); // number
```

:::

У наведених прикладах рядок `'abc'` не представляє дійсне числове значення, тому JavaScript не може перетворити його у корисне число. Коли таке перетворення не вдається, JavaScript повертає `NaN`, що означає "Не число". `NaN` — це спеціальне значення типу `Number`, яке позначає недійсне або непредставлене число.

Що, якщо ви виконуєте арифметичні дії з булевим значенням (`true` або `false`)? Давайте подивимося, що станеться. JavaScript трактує булеві значення як числа в математичних операціях: `true` стає `1`, а `false` стає `0`.

:::interactive_editor

```js
const result1 = true + 1;
console.log(result1); // 2
console.log(typeof result1); // number

const result2 = false + 1;
console.log(result2); // 1
console.log(typeof result2); // number

const result3 = 'Hello' + true;
console.log(result3); // "Hellotrue"
console.log(typeof result3); // string
```

:::

У перших двох прикладах `true + 1` дав `2`, а `false + 1` дав `1`. У третьому прикладі, `'Hello' + true`, JavaScript перетворив `true` на рядок і конкатенував його з `'Hello'`, отримавши `'Hellotrue'`, що є рядком.

Для `null` і `undefined` JavaScript трактує `null` як `0` і `undefined` як `NaN` у математичних операціях:

:::interactive_editor

```js
const result1 = null + 5;
console.log(result1); // 5
console.log(typeof result1); // number

const result2 = undefined + 5;
console.log(result2); // NaN
console.log(typeof result2); // number
```

:::

JavaScript часто виконує перетворення типів, автоматично конвертуючи типи даних, такі як числа, рядки та булеві значення, іноді несподіваними способами. Розуміння цих перетворень є важливим для уникнення помилок і написання надійного коду у ваших проєктах.
