Для чого використовують і не використовують HTML-таблиці?

Сьогодні HTML-таблиці не такі популярні, як колись. Але, як фронтенд розробник, ви повинні знати про них. Таблиці — це один з перших способів відтворення даних у браузері, яким користувались ще в 90-х.

Ось приклад коду, який генерує таблицю з даних Бюро статистики праці США:

```html
<table id="quickfacts">
  <thead>
    <tr>
      <th colspan="2">Quick Facts: Software Developers, Quality Assurance Analysts, and Testers</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <th>2023 Median Pay</th>
      <td>
        $130,160 per year
        <br>$62.58 per hour
      </td>
    </tr>
    <tr>
      <th>Typical Entry-Level Education</th>
      <td>Bachelor's degree</td>
    </tr>
    <tr>
      <th>Work Experience in a Related Occupation</th>
      <td>None</td>
    </tr>
    <tr>
      <th>On-the-job Training</th>
      <td>None</td>
    </tr>
    <tr>
      <th>Number of Jobs, 2022</th>
      <td>1,795,300</td>
    </tr>
    <tr>
      <th>Job Outlook, 2022-32</th>
      <td>25% (Much faster than average)</td>
    </tr>
    <tr>
      <th>Employment Change, 2022-32</th>
      <td>451,200</td>
    </tr>
  </tbody>
  <tfoot>
    <tr>
      <th>If this table had a footer it would be here.</th>
    </tr>
  </tfoot>
</table>
```

Як бачите, головним є елемент `table` з `id` зі значенням `quickfacts`. Він містить елементи `thead` (від англ. table head — заголовок таблиці), `tbody` (від англ. table body — тіло таблиці) та `tfoot` (від англ. table foot — підсумкова частина таблиці).

Ці елементи можуть містити певну кількість елементів `tr` (від англ. table row — рядок таблиці). Кожен рядок може містити елемент `th` (від англ. table header — заголовок таблиці), який позначає дані в цьому рядку. Рядок також може містити елементи `td` (від англ. table data — дані таблиці).

Так, елементів багато. Але не лякайтесь, а просто дотримуйтесь ієрархії.

Ось найпростіша таблиця, яка містить всі елементи:

```html
<table>
  <thead>
    <tr>
      <th>The title of this table</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <th>First Row</th>
      <td>
        First Data Cell
      </td>
    </tr>
    <tr>
      <th>Second Row</th>
      <td>
        Second Data Cell
      </td>
    </tr>
  </tbody>
  <tfoot>
    <tr>
      <th>The footer of this table, which might contain date of publication, author credits, or other meta information.</th>
    </tr>
  </tfoot>
</table>
```

Як бачите, дані завжди розташовані в елементі `tr`, який містить елемент `th` із заголовком та елемент `td` з даними.

Щоб створити таблиці, деякі вебсайти використовують елементи `div`, а не `table`.

Хоча табличні дані можливо відтворити завдяки поширеним елементам `div`, краще використовувати елемент `table`.

Багато років тому розробники використовували елемент `table`, щоб розташувати елементи, які не містили даних, на вебсторінці. Це ніколи не вважалось хорошою практикою. Але навіть зараз можна зустріти кодові бази, де елементи `table` використовують так.

Сьогодні розробники використовують CSS flexbox та сітку, щоб розпланувати макет дизайну. freeCodeCamp ознайомить вас з цими інструментами пізніше.

Наразі використовуйте HTML-таблиці для їхньої початкової мети: відтворення табличних даних.
