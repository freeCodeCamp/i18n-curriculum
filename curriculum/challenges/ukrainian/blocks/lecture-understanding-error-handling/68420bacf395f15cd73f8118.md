# --description--

Під час написання коду на Python часто виникають помилки. Розуміння цих помилок є ключем до швидкого та ефективного налагодження вашого коду. Ці повідомлення точно вказують, що пішло не так, якщо ви знаєте, як їх читати.

Загальні помилки Python включають `SyntaxError`, `NameError`, `TypeError`, `IndexError` та `AttributeError`. Вони виникають, коли Python не розуміє ваш код або коли ваша логіка не відповідає даним, з якими ви працюєте.

Ось приклад `SyntaxError`:

```py
print("Hello, world!"
# SyntaxError: unexpected EOF while parsing
```

У цьому рядку бракує закриваючої дужки. Python викликає `SyntaxError`, оскільки код не відповідає правилам правильного синтаксису.

Ось приклад `NameError`:

```py
print(name)
# NameError: name 'name' is not defined
```

Ви намагаєтеся вивести змінну, яка ще не була визначена. Python викликає `NameError`, коли не може знайти змінну з таким іменем.

Ось приклад `TypeError`:

```py
5 + "5"
# TypeError: unsupported operand type(s) for +: 'int' and 'str'
```

Ви не можете додати ціле число і рядок разом. Python викликає `TypeError`, коли ви намагаєтеся виконати операцію над несумісними типами даних.

Ось приклад `IndexError`:

```py
my_list = [1, 2, 3]
print(my_list[5])
# IndexError: list index out of range
```

Ви намагаєтеся отримати доступ до індексу, якого немає в списку. Python викликає `IndexError`, коли ви виходите за межі.

Ось приклад `AttributeError`:

```py
num = 42
num.append(5)
# AttributeError: 'int' object has no attribute 'append'
```

Об’єкт `int` не має методу `append()`. Python викликає `AttributeError`, коли ви намагаєтеся використати метод або властивість, які не існують для цього типу даних.

Розпізнавання загальних повідомлень про помилки Python допомагає швидше виправляти проблеми. Замість того, щоб гадати, уважно читайте повідомлення про помилку — воно часто точно вказує, що пішло не так і де шукати.
