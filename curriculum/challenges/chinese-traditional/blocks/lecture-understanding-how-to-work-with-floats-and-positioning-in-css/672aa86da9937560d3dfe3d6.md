# --interactive--

CSS 中的浮動是一種最初設計用來讓文字環繞元素（例如影像）的技術。然而，隨著時間推移，開發者發現了使用浮動的新方法，將其創意地應用於版面設計。雖然像 Flexbox 和 Grid 這類現代版面方法現在更常被使用，但理解浮動仍然很重要，尤其是在處理舊有程式碼或需要達成特定版面效果時。

當一個元素被設定為浮動時，它會被從正常的文件流程中移出，並推向其容器的左側或右側。接續的內容會環繞浮動元素，填滿剩餘的空間。一個經典的用法是讓文字環繞圖片，其中圖片被浮動到一側，文字則環繞其周圍。這種技術仍然被廣泛使用，特別是在需要將圖片與文字並排放置的文章和部落格中。以下是程式碼範例：

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<div class="container">
  <img src="https://placehold.co/150x150" alt="Placeholder Image">
</div>
```

```css
.container {
  border: 1px solid black;
  padding: 10px;
}

img {
  float: left;
  margin-right: 20px;
}
```

:::

在上述範例中，容器不會包裹浮動的影像。影像脫離了正常的文件流程，而容器因為無法「看到」浮動的子元素而塌陷為零高度。 

在 Flexbox 和 Grid 廣泛使用之前，浮動也很受歡迎，用於創建多行佈局。透過將元素浮動在彼此旁邊，開發者可以創建水平對齊的行。然而，使用浮動時，重要的是要控制子元素浮動時父元素塌陷的問題。clearfix 技術解決方案被應用於類別 `container` 元素以修正此問題。

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<div class="container">
  <img src="https://placehold.co/150x150" alt="Placeholder Image">
  <p> This is an example of text flowing around a floated image.</p>
</div>
```

```css
.container {
  border: 1px solid black;
}

/* Clearfix CSS */
.container::after {
  content: "";  
  display: block;
  clear: both;
}

img {
  float: left;
  margin-right: 20px;
}
```

:::

- `::after` 是一個偽元素，會在容器內容後面添加一個不可見的區塊。
- `content: ""` 確保偽元素存在但不顯示任何內容。
- `display: block` 使偽元素成為區塊層級元素。
- `clear: both` 確保偽元素清除其上方任何浮動元素的兩側。

在上述範例中，我們為了讓摺疊更明顯，添加了一個新的段落元素。因為段落仍然保持在正常的 `document` 流中，`container` 會擴展到剛好包住它。接著，我們套用 `clearfix` 技巧來修正摺疊，並使 `container` 的邊框正確顯示。你可以嘗試移除 `clearfix` 規則，看看 `container` 再次摺疊的情況。

clearfix 技巧確保父元素能正確包裹其浮動的子元素。clearfix 透過在浮動內容後添加 clear 屬性，強制父容器「看到」浮動的子元素。

雖然由於像 Flexbox 和 Grid 這類較現代的技術，浮動已不再是複雜版面配置的首選方法，但它們在某些情況下仍扮演著重要角色。無論你是在讓文字環繞影像，還是在處理舊有專案，了解浮動的運作方式以及如何正確清除浮動，對於維護良好結構且具響應性的版面配置至關重要。
