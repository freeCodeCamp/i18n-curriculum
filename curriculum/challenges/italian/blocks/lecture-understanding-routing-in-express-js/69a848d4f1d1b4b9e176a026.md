# --description--

Ora che hai creato una app Express di base, è il momento di esaminare più da vicino il routing e come funziona.

In Express, il routing è il modo in cui definisci gli endpoint di un'applicazione web. Puoi pensare alle rotte come a regole che dicono all'app come rispondere quando un utente visita un URL specifico.

Rivediamo la nostra semplice app Express:

```javascript
const express = require("express");
const app = express();
const port = 3000;

app.get("/", (req, res) => {
  res.send("Hello World!");
});

app.listen(port, () => {
  console.log(`Example app listening on port ${port}`);
});
```

In questo esempio, l'app definisce una rotta per l'URL radice (`/`). Quando un client invia una richiesta GET a questo percorso, il server risponde con il testo `Hello World!`. L'app poi inizia ad ascoltare le richieste in arrivo sulla porta `3000`.

Ma ci sono altri metodi di rotta.

Express ti permette di gestire diversi tipi di richieste HTTP usando metodi di routing diversi. I più comuni sono:

* `app.get()` – recupera dati
    
* `app.post()` – crea nuovi dati
    
* `app.put()` – aggiorna dati esistenti
    
* `app.delete()` – elimina dati
    

Ogni metodo definisce come la tua applicazione dovrebbe rispondere a una specifica richiesta HTTP su una rotta specifica.

Ecco alcuni esempi semplici:

```javascript
app.post("/submit", (req, res) => {
  res.send("Form submitted!");
});

app.put("/update", (req, res) => {
  res.send("Item updated!");
});

app.delete("/delete", (req, res) => {
  res.send("Item deleted!");
});
```

In ogni caso, Express ascolta una richiesta con un particolare metodo HTTP e percorso URL, quindi esegue la funzione gestore corrispondente per inviare una risposta al client.

Puoi testare queste rotte usando strumenti come Postman o `curl`, che ti permettono di inviare diversi tipi di richieste HTTP al tuo server.

E questo è il routing in Express: si tratta semplicemente di definire come la tua applicazione risponde a richieste diverse.
