# --description--

في الدرس السابق، رأينا ما هي واجهة برمجة التطبيقات Fetch وكيفية استخدامها. في هذا الدرس، سنناقش طرق HTTP `GET` و `POST` و `PUT` و `DELETE` الخاصة بواجهة Fetch.

لنبدأ بأكثر طرق HTTP شيوعًا وهي طريقة `GET`. تُستخدم هذه الطريقة لاسترجاع البيانات من خادم. عندما تستخدم `fetch()` دون تحديد طريقة، فإنها تفترض افتراضيًا `GET`.

```js
fetch('https://api.example.com/data')
```

في هذه الشفرة، نقوم بعمل طلب `GET` إلى `https://api.example.com/data`. لاحظ أنه لا يمكنك استخدام هذه البيانات مباشرة؛ يجب تحويل الاستجابة إلى صيغة JSON. فقط بعد ذلك يمكنك استخدامها في أي مكان تريد في مشروعك. إليك مثالًا على كيفية القيام بذلك:

```js
fetch('https://api.example.com/data')
  .then(response => response.json())
  .then(data => console.log(data))
```

في هذه الشفرة، الاستجابة القادمة من واجهة Fetch هي وعد، ونحن نستخدم معالِج `.then` لتحويل الاستجابة إلى صيغة JSON. في درس سابق، تعلمت أن `Promise` هو كائن يمثل الانتهاء النهائي (أو الفشل) لعملية غير متزامنة وقيمتها.

قيمة الوعد غير معروفة عند إنشائه. تُعرف فقط عند اكتمال العملية غير المتزامنة. عندما نسلسل معالِجي `.then` مع استدعاء fetch، يُسمى هذا تسلسل الوعود وسيُشرح في الدرس التالي.

حتى الآن كنا نسترجع الموارد من الخادم. لكن، هل تعلم أنه يمكننا أيضًا إرسال بيانات إلى الخادم؟ تُستخدم طريقة `POST` لإرسال بيانات إلى الخادم لإنشاء مورد. إليك مثالًا على طريقة `POST` تُستخدم لإنشاء مورد على الخادم:

```js
fetch('https://api.example.com/users', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({
    name: 'John Doe',
    email: 'john@example.com'
  })
})
```

في هذا المثال، نرسل طلب `POST` لإنشاء مستخدم جديد. نحدد الطريقة كـ `POST`، ونضبط الرؤوس المناسبة، ونُدرج `body` بالبيانات التي نريد إرسالها. يجب أن يكون الجسم سلسلة نصية، لذا نستخدم `JSON.stringify()` لتحويل كائننا إلى سلسلة JSON.

تُستخدم طريقة `PUT` لتحديث الموارد الموجودة على الخادم. إليك مثالًا:

```js
fetch('https://api.example.com/users/45', {
  method: 'PUT',
  headers: {
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({
    name: 'John Smith',
    email: 'john@example.com'
  })
})
```

في هذا المثال، انظر جيدًا إلى عنوان URL، سترى `45` في النهاية. يُستخدم هذا عادةً كمعرف فريد لتحديد البيانات التي نحاول تحديثها. استخدمنا طريقة `PUT` في الشفرة وحددنا البيانات كـ `body` التي ستُستخدم لتحديث البيانات المحددة.

تُستخدم طريقة `DELETE` لحذف مورد من الخادم. إليك مثالًا:

```js
fetch('https://api.example.com/users/45', {
  method: 'DELETE',
})
```

في هذه الشفرة، نُدرج طريقة `DELETE` ومعرفًا في نهاية عنوان URL لتحديد البيانات التي يجب حذفها.
