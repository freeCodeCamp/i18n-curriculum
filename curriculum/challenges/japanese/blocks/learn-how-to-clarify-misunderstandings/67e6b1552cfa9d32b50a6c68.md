# --explanation--

`Breaking into`は、何かをより小さな部分に分割することを意味します。例えば：

`We are breaking the project into smaller tasks.` - これはプロジェクトを管理しやすい部分に分割することを意味します。

`Encapsulating`は、何か（データや機能など）を1つの単位や構造にまとめ、詳細を隠して必要な部分だけを公開することを指します。例えば：

`Encapsulating the data in an object means users can only access the data through specific methods, not directly.` - これはデータが制御され、意図しない変更や誤用から安全であることを保証します。
