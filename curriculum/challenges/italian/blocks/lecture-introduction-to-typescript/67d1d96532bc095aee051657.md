# --description--

JavaScript è considerato un linguaggio a tipizzazione dinamica. Questo significa che le variabili possono ricevere qualsiasi valore a runtime: dichiarare una variabile come numero non ti impedisce di assegnarle in seguito un valore stringa, e ai parametri di una funzione possono essere passati valori di qualsiasi tipo.

La sfida di un linguaggio a tipizzazione dinamica è che la mancanza di sicurezza sui tipi può introdurre errori se non stai attento. TypeScript estende il linguaggio JavaScript includendo la tipizzazione statica, che aiuta a intercettare questi errori prima ancora di distribuire il tuo codice. Ma come funziona?

Considera questa funzione JavaScript:

```js
const getRandomValue = (array) => {
  return array[Math.floor(Math.random() * array.length)];
}
```

La nostra funzione `getRandomValue` prende un array e restituisce un valore casuale da quell’array. Ma poiché JavaScript non verifica i tipi, nulla ti impedisce di chiamare la funzione con un numero:

```js
console.log(getRandomValue(10));
```

L’output sulla console per l’esempio attuale restituirà `undefined` perché si aspettava un array invece di un numero.

Ma con TypeScript puoi definire un tipo per il parametro array:

```js
const getRandomValue = (array: string[]) => {
  return array[Math.floor(Math.random() * array.length)];
}
```

Questa definizione di tipo dice a TypeScript che l’argomento array deve essere un array di stringhe. Quindi, quando chiami `getRandomValue` e gli passi un numero, ottieni un tipo diverso di errore chiamato errore di compilazione.

La maggior parte degli ambienti di runtime JavaScript, come un browser o Node.js, non può eseguire TypeScript nativamente. Invece, devi prima compilare, o tradurre, il codice TypeScript in JavaScript normale. Puoi farlo con il compilatore integrato che viene con il linguaggio TypeScript. Quando esegui il compilatore, TypeScript valuta il tuo codice e lancia un errore per qualsiasi problema in cui i tipi non corrispondono — come passare un numero a una funzione che si aspetta un array.

Imparerai di più su come funzionano questi tipi nelle prossime lezioni. Ma è proprio questa sicurezza aggiuntiva che rende TypeScript un linguaggio interessante per molti sviluppatori e organizzazioni.
