# --description--
  
Programu inayobadilika ni mbinu ya algoriti inayotatua matatizo magumu kwa kuyavunja kuwa matatizo madogo madogo na kuhifadhi matokeo ili kuepuka mahesabu yanayojirudia. Njia hii hubadilisha matatizo ambayo kawaida yangechukua muda wa mabadiliko ya mlinganyo kuwa matatizo yanayoweza kutatuliwa kwa muda wa polinomiali.

## Kanuni kuu za programu inayobadilika

Programu inayobadilika hufanya kazi pale ambapo kuna masharti mawili muhimu katika tatizo.

- **Matatizo madogo yanayojirudia**: Matatizo madogo madogo yanayofanana hujitokeza mara nyingi wakati wa kutatua tatizo kubwa. Badala ya kuhesabu matatizo haya madogo mara kwa mara, tunahifadhi suluhisho lao.
    
- **Muundo bora wa tatizo**: Suluhisho bora la tatizo lina suluhisho bora za matatizo madogo madogo. Hii ina maana tunaweza kujenga suluhisho bora kwa kuunganisha suluhisho bora za sehemu ndogo ndogo.
    

Tuchunguze dhana hizi kwa kutumia tatizo la kawaida la "kupanda ngazi".

## Tatizo la kutumia rekursi ya kawaida

Fikiria tatizo la kupanda ngazi: unapanda ngazi yenye hatua `n` na unaweza kupanda hatua 1 au 2 kwa wakati mmoja. Ni njia ngapi tofauti unaweza kufikia kilele?

```python
def climb_stairs_recursive(n):
    """Recursive approach"""
    if n <= 2:
        return n  # Base cases: 1 way for 1 step, 2 ways for 2 steps
    # To reach step n, we can come from step (n-1) or step (n-2)
    return climb_stairs_recursive(n-1) + climb_stairs_recursive(n-2)
```

Utekelezaji huu una ugumu wa muda wa mabadiliko ya mlinganyo kwa sababu ya mahesabu mengi yanayojirudia. Wakati wa kuhesabu `climb_stairs(5)`, hii ndicho kinachotokea:

- `climb_stairs(5)` inaita `climb_stairs(4)` na `climb_stairs(3)`
    
- `climb_stairs(4)` inaita `climb_stairs(3)` na `climb_stairs(2)`
    
- Sasa `climb_stairs(3)` inahesabiwa **mara mbili**
    
- `climb_stairs(3)` inaita `climb_stairs(2)` na `climb_stairs(1)`
    
- `climb_stairs(2)` inahesabiwa **mara 3 kwa jumla**
    

Kwa `n=5` tu, tunafanya wito 9 za kitendakazi wakati tunahitaji mahesabu 5 tu ya kipekee. Kadiri `n` inavyoongezeka, kurudia hii kunazidi kwa mabadiliko ya mlinganyo - `climb_stairs(30)` ingehitaji mamilioni ya wito wa kitendakazi. Ugumu wa muda unakuwa `O(2^n)`, na kufanya kuwa hauna ufanisi na hauwezi kutumika kwa thamani kubwa za `n`.

## Suluhisho za programu inayobadilika

Programu inayobadilika huondoa mahesabu yanayojirudia kupitia mbinu kuu mbili:

### Kumbukumbu (Mbinu ya juu-chini)

Kumbukumbu huhifadhi matokeo ya wito za kitendakazi ghali na hurudisha matokeo yaliyohifadhiwa wakati ingizo sawa linapotokea tena:

```python
def climb_stairs_memo(n, memo={}):
    """Dynamic programming with memoization"""
    # Check if we've already calculated this value
    if n in memo:
        return memo[n]  # Return cached result - O(1) lookup!
    
    # Base cases
    if n <= 2:
        return n
    
    # Calculate once and store in memo for future use
    memo[n] = climb_stairs_memo(n-1, memo) + climb_stairs_memo(n-2, memo)
    return memo[n]
```

Kumbukumbu ni ya ufanisi zaidi kwa sababu kila thamani ya kipekee kutoka `1` hadi `n` inahesabiwa mara moja tu. Tunapohitaji `climb_stairs(3)` tena, badala ya kuhesabu tena (ambayo ingehamasisha wito zaidi wa rekursi), tunatafuta tu katika kamusi yetu ya kumbukumbu kwa muda wa `O(1)`.

Tufuate utekelezaji wa `climb_stairs(5)` kwa mbinu ya juu-chini kuona jinsi kumbukumbu inavyotenganisha kazi inayojirudia:

```md
Call: climb_stairs_memo(5)
  memo = {} (empty)
  
  Call: climb_stairs_memo(4) 
    memo = {} (empty)
    
    Call: climb_stairs_memo(3)
      memo = {} (empty)
      
      Call: climb_stairs_memo(2) → returns 2 (base case)
      Call: climb_stairs_memo(1) → returns 1 (base case)
      
      Result: 2 + 1 = 3
      memo = {3: 3} (stored!)
    
    Call: climb_stairs_memo(2) → returns 2 (base case)
    
    Result: 3 + 2 = 5
    memo = {3: 3, 4: 5} (stored!)
  
  Call: climb_stairs_memo(3) → returns 3 (FROM MEMO - no recursion!)
  
  Result: 5 + 3 = 8
  memo = {3: 3, 4: 5, 5: 8}
```

**Ulinganisho wa ufanisi**

- **Rekursi ya kawaida**: Inafanya wito 9 za kitendakazi na mahesabu yanayojirudia
    
- **Kumbukumbu**: Inafanya mahesabu 5 ya kipekee tu, kisha inatumia tena matokeo yaliyohifadhiwa
    
- **Ugumu wa muda**: Umepunguzwa kutoka `O(2^n)` hadi `O(n)` kwa sababu tunafanya mahesabu `n` ya kipekee tu
    
- **Ugumu wa nafasi**: `O(n)` kwa ajili ya hifadhi ya kumbukumbu na stack ya wito
    
- **Athari halisi**: `climb_stairs(30)` hupungua kutoka mamilioni ya wito wa kitendakazi hadi takriban mahesabu 30 ya matatizo madogo madogo ya kipekee
    

### Jedwali (Mbinu ya chini-juu)

Jedwali hujenga suluhisho kutoka chini juu, likijaza jedwali na suluhisho za matatizo madogo madogo:

```python
def climb_stairs_tabulation(n):
    """Dynamic programming with tabulation"""
    if n <= 2:
        return n
    
    # Create array to store results for all steps from 0 to n
    dp = [0] * (n + 1)
    dp[1] = 1  # 1 way to reach step 1
    dp[2] = 2  # 2 ways to reach step 2
    
    # Build up the solution iteratively
    for i in range(3, n + 1):
        # Ways to reach step i = ways to reach (i-1) + ways to reach (i-2)
        dp[i] = dp[i-1] + dp[i-2]
    
    return dp[n]
```

Jedwali huondoa kabisa rekursi kwa kujenga suluhisho kwa njia ya mfululizo kutoka matatizo madogo kabisa hadi lengo.

Tuchunguze mbinu ya chini-juu kwa vitendo kuona jinsi tunavyojenga suluhisho kwa mpangilio. Huu ni ujenzi wa mfululizo kwa `climb_stairs(5)`:

```md
Initial state:
dp = [0, 1, 2, 0, 0, 0]
     [0, 1, 2, 3, 4, 5] ← indices (step numbers)

Step by step construction:

i = 3:
  dp[3] = dp[2] + dp[1] = 2 + 1 = 3
  dp = [0, 1, 2, 3, 0, 0]
  
i = 4:
  dp[4] = dp[3] + dp[2] = 3 + 2 = 5
  dp = [0, 1, 2, 3, 5, 0]
  
i = 5:
  dp[5] = dp[4] + dp[3] = 5 + 3 = 8
  dp = [0, 1, 2, 3, 5, 8]

Final result: dp[5] = 8
```

**Faida kuu za jedwali**

- **Hakuna mzigo wa rekursi**: Tofauti na kumbukumbu, hakuna stack ya wito za rekursi.
    
- **Utekelezaji unaotabirika**: Tunahesabu thamani kwa mpangilio uliowekwa (1, 2, 3, 4, 5...).
    
- **Rafiki kwa hifadhi**: Upatikanaji wa safu ya data kwa mfululizo huongeza ufanisi wa matumizi ya kumbukumbu.
    
- **Rahisi kuboresha**: Inaweza kupunguza ugumu wa nafasi hadi `O(1)` kwa sababu tunahitaji thamani mbili za mwisho tu.
    

```python
def climb_stairs_optimized(n):
    if n <= 2:
        return n
    
    prev2, prev1 = 1, 2  # Only store last two values
    for i in range(3, n + 1):
        current = prev1 + prev2
        prev2, prev1 = prev1, current
    return prev1
```

**Ulinganisho wa ufanisi**

- **Rekursi ya kawaida**: Wito 9 za kitendakazi kwa `n=5`, ukuaji wa mabadiliko ya mlinganyo.
    
- **Jedwali**: Jumlisha rahisi 3 kwa `n=5`, ukuaji wa mstari.
    
- **Ugumu wa muda**: `O(n)` badala ya `O(2^n)`.
    
- **Ugumu wa nafasi**: `O(n)` kwa safu ya data, au `O(1)` kwa uboreshaji.
    
- **Utendaji unaotabirika**: Hakuna hatari ya kuzidi stack kwa ingizo kubwa.
    

Mbinu zote mbili hupunguza ugumu wa muda kutoka mabadiliko ya mlinganyo `O(2^n)` hadi mstari `O(n)`, kuboresha kwa kiasi kikubwa ambacho kinatofautisha kutatua tatizo kwa millisekunde badala ya kusubiri miaka kwa ingizo kubwa.

## Matumizi halisi

Programu inayobadilika ina matumizi mengi katika sayansi ya kompyuta na zaidi:

- **Uboreshaji wa njia ya data**: Mifumo ya GPS hutumia algoriti za programu inayobadilika kupata njia fupi kati ya maeneo.
    
- **Usindikaji wa maandishi**: Vifaa vya ukaguzi wa tahajia na kipengele cha kukamilisha maandishi mara nyingi hutegemea programu inayobadilika kuhesabu umbali wa mabadiliko kati ya maneno.
    
- **Uundaji wa mifano ya kifedha**: Mikakati ya uwekezaji na uboreshaji wa portfolio mara nyingi hutumia mbinu za programu inayobadilika.
    
- **Ugawaji wa rasilimali**: Tatizo la knapsack na aina zake hujitokeza katika upangaji ratiba, bajeti, na usimamizi wa rasilimali.
    

## Mfano wa vitendo: Tatizo la kubadilisha sarafu

Tatizo la kubadilisha sarafu ni changamoto ya kawaida ya uandishi wa programu ambayo, inapotatuliwa kwa kutumia programu inayobadilika, inaonyesha kanuni kuu mbili za DP: muundo bora wa tatizo na matatizo madogo yanayojirudia.

Tatizo la kubadilisha sarafu linauliza, "Ni sarafu ngapi za kiwango cha chini zinazohitajika kutengeneza kiasi lengwa?"

Hapa kuna suluhisho moja kwa kutumia programu inayobadilika:

```python
def min_coins(amount, coins):
    """Find minimum number of coins needed to make the given amount"""
    # Initialize dp array with "infinity" - represents impossible to make
    dp = [float('inf')] * (amount + 1)
    dp[0] = 0  # Base case: 0 coins needed for amount 0
    
    # For each amount from 1 to target amount
    for i in range(1, amount + 1):
        # Try each coin denomination
        for coin in coins:
            if coin <= i:  # Can only use coin if it doesn't exceed current amount
                # Update minimum: current minimum vs (coins for remaining amount + 1)
                dp[i] = min(dp[i], dp[i - coin] + 1)
    
    # Return result if possible, -1 if impossible
    return dp[amount] if dp[amount] != float('inf') else -1

# Example usage:
# coins = [1, 3, 4], amount = 6
# dp[6] = min(dp[5]+1, dp[3]+1, dp[2]+1) = min(3+1, 1+1, 2+1) = 2
# Result: 2 coins (3 + 3)
```

Na hapa ni jinsi algoriti ya kubadilisha sarafu ya programu inayobadilika inavyofanya kazi hatua kwa hatua kwa `coins = [1, 3, 4]`, `amount = 6`:

```md
Initial state:
dp = [0, ∞, ∞, ∞, ∞, ∞, ∞]
     [0, 1, 2, 3, 4, 5, 6] ← amounts

Building up the solution:

For amount = 1:
  Try coin 1: dp[1] = min(∞, dp[0] + 1) = min(∞, 0 + 1) = 1
  dp = [0, 1, ∞, ∞, ∞, ∞, ∞]

For amount = 2:
  Try coin 1: dp[2] = min(∞, dp[1] + 1) = min(∞, 1 + 1) = 2
  dp = [0, 1, 2, ∞, ∞, ∞, ∞]

For amount = 3:
  Try coin 1: dp[3] = min(∞, dp[2] + 1) = min(∞, 2 + 1) = 3
  Try coin 3: dp[3] = min(3, dp[0] + 1) = min(3, 0 + 1) = 1
  dp = [0, 1, 2, 1, ∞, ∞, ∞]

For amount = 4:
  Try coin 1: dp[4] = min(∞, dp[3] + 1) = min(∞, 1 + 1) = 2
  Try coin 3: dp[4] = min(2, dp[1] + 1) = min(2, 1 + 1) = 2
  Try coin 4: dp[4] = min(2, dp[0] + 1) = min(2, 0 + 1) = 1
  dp = [0, 1, 2, 1, 1, ∞, ∞]

For amount = 5:
  Try coin 1: dp[5] = min(∞, dp[4] + 1) = min(∞, 1 + 1) = 2
  Try coin 3: dp[5] = min(2, dp[2] + 1) = min(2, 2 + 1) = 2
  Try coin 4: dp[5] = min(2, dp[1] + 1) = min(2, 1 + 1) = 2
  dp = [0, 1, 2, 1, 1, 2, ∞]

For amount = 6:
  Try coin 1: dp[6] = min(∞, dp[5] + 1) = min(∞, 2 + 1) = 3
  Try coin 3: dp[6] = min(3, dp[3] + 1) = min(3, 1 + 1) = 2
  Try coin 4: dp[6] = min(2, dp[2] + 1) = min(2, 2 + 1) = 2
  dp = [0, 1, 2, 1, 1, 2, 2]

Final result: dp[6] = 2 (achieved with coins 3 + 3)
```

Suluhisho hili linaonyesha kanuni kuu mbili za programu inayobadilika. Lina matatizo madogo yanayojirudia kwa sababu kupata sarafu chache kwa kiasi 6 kunahitaji kujua suluhisho za kiasi 5, 3, na 2. Matatizo madogo madogo haya yanajitokeza pia wakati wa kuhesabu kiasi kingine. Lina muundo bora wa tatizo kwa sababu suluhisho bora kwa kiasi chochote linajumuisha suluhisho bora za kiasi kidogo. Ikiwa tunajua sarafu chache kwa kiasi 3 ni 1, basi njia moja ya kutengeneza kiasi 6 ni kutumia suluhisho hilo pamoja na sarafu moja ya thamani 3 zaidi.

Bila DP, tungehitaji kujaribu mchanganyiko wote wa sarafu - idadi ya mabadiliko ya mlinganyo. Kwa DP, tunajenga suluhisho kwa mpangilio:

- **Ugumu wa muda**: `O(amount × number of coins)` badala ya mabadiliko ya mlinganyo.
    
- **Ugumu wa nafasi**: `O(amount)` kwa safu ya `dp`.
    
- **Hakuna kazi inayojirudia**: Kila tatizo dogo (kupata sarafu chache kwa kila kiasi) linatatuliwa mara moja tu.
    
- **Matokeo yanayoweza kutumika tena**: Mara tunapojua sarafu chache kwa kiasi 3, tunatumia maarifa haya kwa kiasi kikubwa zaidi kinachoweza kufaidika.
    

## Lini utumie programu inayobadilika

Programu inayobadilika ni yenye ufanisi pale:

- Tatizo linaweza kugawanywa kuwa matatizo madogo madogo yanayojirudia.
    
- Tatizo lina muundo bora wa tatizo.
    
- Suluhisho la rekursi la kawaida lingehitaji mahesabu yanayojirudia.
    
- Unahitaji kuboresha ugumu wa muda kwa gharama ya ugumu wa nafasi.
    

Mifumo ya kawaida ya programu inayobadilika ni pamoja na matatizo ya uboreshaji (kupata thamani ya kiwango cha chini/juu), matatizo ya kuhesabu (idadi ya njia za kufanikisha jambo), na matatizo ya maamuzi yanayoweza kugawanywa kuwa maamuzi madogo madogo.

Programu inayobadilika hubadilisha matatizo magumu kuwa rahisi kwa kuhifadhi na kutumia tena suluhisho za matatizo madogo madogo kwa mpangilio. Kuelewa mbinu hii hufungua mlango wa kutatua changamoto nyingi za kihesabu kwa ufanisi.
