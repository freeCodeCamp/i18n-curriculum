# --description--

JavaScriptでは、文字列を宣言する際にシングルクォート（`'`）とダブルクォート（`"`）の両方を使うことができます。どちらを使うかは基本的に個人の好みによりますが、例外もあります。

文字列に短縮形や引用符で囲まれた別のテキストが含まれる場合、2つの選択肢があるのは便利です。ただし、文字列を早く閉じすぎてしまうと構文エラーになるので注意してください。

以下はクォートを混ぜて使った例です：

```js
const grouchoContraction = "I've had a perfectly wonderful evening, but this wasn't it.";
const quoteInString = "Groucho Marx once said 'Quote me as saying I was mis-quoted.'";
const uhOhGroucho = 'I've had a perfectly wonderful evening, but this wasn't it.';
```

最初の2つは正しいですが、3つ目は間違っています。

もちろん、どちらか一方のクォートだけを使うのも問題ありません。文字列内のクォートをバックスラッシュ（`\`）のエスケープ文字でエスケープすることもできます：

```js
const allSameQuotes = 'I\'ve had a perfectly wonderful evening, but this wasn\'t it.';
```

# --instructions--

`href`の値に対して、異なるクォートを使うか、既存のクォートをエスケープするように文字列を修正してください。文字列全体はダブルクォートで囲んだままにしてください。

# --hints--

`href`の値`#Home`の周りのクォートは、変更するかエスケープして修正してください。

```js
assert(__helpers.removeJSComments(code).match(/<a href=\s*?('|\\")#Home\1\s*?>/g));
```

文字列全体はダブルクォートで囲んだままにしてください。

```js
assert(__helpers.removeJSComments(code).match(/"<p>.*?<\/p>";/g));
```
