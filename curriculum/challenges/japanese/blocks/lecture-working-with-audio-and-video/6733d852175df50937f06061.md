# --description--

音声や動画を再生する代わりに、音声や動画をキャプチャしたい場合があります。Media Capture and Streams API、またはMediaStream APIを使うとこれが可能です。

ただし、このAPIを使うには`MediaStream`オブジェクトを作成する必要があります。コンストラクタで作成することもできますが、それはユーザーのハードウェアに紐づきません。代わりに、グローバル`mediaDevices`オブジェクトの`navigator`プロパティには使える`getUserMedia()`メソッドがあります。

このメソッドは、受け取りたいメディアの種類を定義する単一の`constraints`オブジェクトを受け取ります。このオブジェクトには音声と動画のストリームを表す`audio`と`video`プロパティがあります。これらのプロパティは、そのタイプのストリームを受け取りたくない場合は`false`、受け取りたい場合は`true`、または追加の制約を定義するオブジェクトにできます。

例えば、特定の解像度の動画出力を要求できます。

```js
window.navigator.mediaDevices.getUserMedia({
  audio: true,
  video: {
    width: {
      min: 1280,
      ideal: 1920,
      max: 3840
    },
    height: {
      min: 720,
      ideal: 1080,
      max: 2160
    }
  }
});
```

この制約オブジェクトは動画フィードの最小値と最大値の解像度を指定します。`ideal`プロパティは最も望ましい解像度を表し、ストリームは理想に最も近い解像度を提供します。

MediaStreamを作成したら（ユーザーがハードウェアへのアクセスを自動リクエストで承認した場合）、ストリームデータを必要に応じて使えます。

`getUserMedia()`はPromiseを返すため、ストリームを扱うにはコールバック関数を使うか、async/await構文を使う必要があります。Promiseや非同期プログラミングについては今後のレッスンで詳しく学びます。

以下はユーザーのウェブカメラ映像をページにレンダリングする基本例です。

```js
const video = document.querySelector("video");
const stream =
  await window.navigator.mediaDevices.getUserMedia({ video: true });
video.srcObject = stream;
await video.play();
```

このAPIは画面キャプチャへのアクセスは提供しないことに注意してください。画面キャプチャAPIについては次のレッスンで学びます。
