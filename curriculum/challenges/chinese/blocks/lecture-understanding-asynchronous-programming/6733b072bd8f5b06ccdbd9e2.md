# --description--

让我们学习异步的 JavaScript。异步的 `event` 在不同时间发生，彼此独立。

在软件开发的上下文中，"asynchronous" 指的是在后台运行、独立于程序主流程的任务。异步进程的主要优点是它们不会阻塞主程序的执行。

这对于可能需要较长时间运行的任务特别有帮助，例如从远程 `server` 获取 `data`、处理大型 `file`、处理 `user` `input` 以及执行复杂计算。这就是我们所说的异步的 `programming`。

这种方法与迄今为止你一直在使用的传统同步的编程技术形成对比。当你编写多行代码时，通常可以预测会发生什么以及何时发生。第一行将被执行，然后是第二行，依此类推，按照你编写的顺序。

在此示例中，第一行你定义 `variable` 的代码将首先执行，然后是第二行，最后是第三行将 `string` 记录到控制台：

```js
const topic = "JavaScript";
const learning = `I'm learning ${topic}!`;
console.log(learning);
```

一切都是循序的且可预测的。每一行在下一行开始之前完成。这种类型的 JavaScript 程序被称为单线程。

线程的概念非常重要。线程是进程内的一个执行单元。它就像程序内的一个独立控件流。

在同步的编程中，线程依次循序执行。如果线程被阻塞，比如等待用户输入，整个进程将被阻塞，直到该线程完成。

在异步的编程中，线程可以并发执行，同时运行多个线程。这样，程序可以继续同时运行多个任务，即使其中一个线程被阻塞，也不会使主程序无响应。

异步的编程通常涉及回调、promise 或 async/await 来处理非阻塞操作。

正如你在之前的课程中学到的，回调是你传递给另一个函数以便稍后调用的函数。事件处理程序是一种你之前使用过的特定类型的回调。它们曾经是实现 JavaScript 异步函数的最常见方式。然而，如果回调函数还接受其他回调函数，代码和逻辑可能会非常快地变得相当复杂。 

目前，在 JavaScript 中实现异步的编程最常用的技术是 promise。`Promise` 是一个对象，表现为异步进程最终完成（或失败）及其值。

当创建 `promise` 时，其值是未知的。只有当异步进程完成时，值才会被知道。例如，从远程 API 获取数据以供你的网页应用使用的过程可能需要一些时间。与此同时，你希望你的用户拥有良好的用户体验，对吧？

要实现这一点，你可以创建一个 promise，以保持用户界面在异步进程运行时保持活跃和交互。当进程完成时，promise 将包含由 API 返回的数据，因此应用可以在数据可用时适当地处理或渲染它。

异步的编程是创建高效 JavaScript 应用的强大工具。通过理解同步的和异步的编程之间的区别，你可以为你的应用选择正确的方法并编写更高效的代码。
