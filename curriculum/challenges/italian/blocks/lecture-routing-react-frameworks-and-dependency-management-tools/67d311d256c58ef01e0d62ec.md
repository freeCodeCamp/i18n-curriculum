# --description--

Fino a questo punto, hai usato React per costruire interfacce utente. Se avevi bisogno di funzionalità aggiuntive come il routing, dovevi importare una libreria di terze parti come React Router per poter passare tra le diverse viste.

Ma cosa succede quando devi costruire un’applicazione web full-stack? Potresti usare React per il front-end e Node e Express per la logica back-end se vuoi restare solo con JavaScript. Oppure potresti usare altri linguaggi come Go, Python o Java per il back-end.

Anche se tutte queste sono opzioni valide, ci sono casi in cui potresti voler usare un framework React invece. I framework React offrono funzionalità come il routing, l’ottimizzazione delle immagini, il recupero dati, l’autenticazione e altro ancora. Questo significa che potresti non dover configurare applicazioni front-end e back-end separate per certi casi d’uso.

Diamo un’occhiata più da vicino a Next.js, un framework React molto popolare. Una delle funzionalità principali di Next.js è il router basato sul file system. Questo sistema di routing supporta rotte dinamiche, rotte parallele, gestori di rotta, reindirizzamenti, internazionalizzazione e altro.

Ecco un esempio di creazione di un gestore di richieste personalizzato:

```js
export async function GET() {
  const res = await fetch("https://example-api.com");
  const data = await res.json();

  return Response.json({ data });
}
```

Puoi definire gestori di rotta come richieste GET o POST in un file chiamato `route.js` dentro la directory `app/api`.

Un’altra funzionalità di Next.js sono le ottimizzazioni automatiche di immagini e caratteri tipografici.

Ecco un esempio di utilizzo del componente `Image` dentro un file `page.js`:

```jsx
import Image from "next/image";

export default function Page() {
  return (
    <Image src="link-to-image-goes-here" alt="descriptive-title-goes-here" />
  );
}
```

Il componente `Image` estende l’elemento HTML nativo `img` e permette caricamenti di pagina più veloci e ottimizzazioni delle dimensioni. Questo significa che le immagini si caricheranno solo quando entrano nel viewport e il componente `Image` servirà automaticamente immagini della dimensione corretta per ogni dispositivo.

Anche se queste sono solo alcune funzionalità di Next.js, ce ne sono molte altre che puoi usare per costruire applicazioni web full-stack robuste. Ci sono anche altri framework React come Remix che offrono la stessa possibilità di costruire applicazioni web full-stack moderne.

Anche se il panorama delle librerie e dei framework JavaScript cambia costantemente, è importante conoscere le opzioni disponibili e imparare i pro e i contro di ogni framework.
