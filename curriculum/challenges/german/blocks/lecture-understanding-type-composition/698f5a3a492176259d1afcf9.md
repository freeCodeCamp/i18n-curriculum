# --description--

In einer vorherigen Lektion haben Sie gelernt, wie man Typ-Aliase verwendet, und Beispiele gesehen, wie man Objekttypen benennt.

Wie Sie sich erinnern, ist eine Möglichkeit, einen Objekttyp zu benennen, die Verwendung des Schlüsselworts `type`, so wie hier:

```ts
type ID = number | string;

type User = {
  id: ID;
  name: string;
};
```

Eine andere Möglichkeit, einen Objekttyp zu benennen, ist die Verwendung einer `interface`-Deklaration, so wie hier:

```ts
type ID = number | string;

interface User {
  id: ID;
  name: string;
}
```

`type` und `interface` sind ähnlich, aber ein wesentlicher Unterschied besteht darin, dass Interfaces das Schlüsselwort `extends` verwenden, um auf anderen Interfaces oder Typen aufzubauen, während `type`-Aliase Schnittmengentypen (`&`) verwenden, um Typen zu kombinieren.

Schauen wir uns einige Beispiele an:

```ts
interface User {
  id: number | string;
  name: string;
}

interface Admin extends User {
  role: string;
}

const adminUser: Admin = {
  id: 101,
  name: "Alice",
  role: "superadmin"
};
```

In diesem Beispiel beginnt `User` mit `id` und `name`. Das `Admin`-Interface erweitert `User`, was bedeutet, dass es `id` und `name` einschließt und eine neue Eigenschaft namens `role` hinzufügt.

Wenn Sie einen `type` erweitern wollten, würden Sie einen Schnittmengentyp (`&`) wie folgt verwenden:

```ts
type User = {
  id: number | string;
  name: string;
};

type Admin = User & {
  role: string;
};

const adminUser: Admin = {
  id: 101,
  name: "Alice",
  role: "superadmin"
};
```

`Admin` kombiniert den `User`-Typ mit einem neuen Objekttyp, der `role` enthält. Das Ergebnis hat dieselbe Form wie das Interface-Beispiel, nämlich ein Objekt mit `id`, `name` und `role`.

Sowohl `type` als auch `interface` können die Form eines Objekts beschreiben. Wenn Sie unsicher sind, welchen Sie verwenden sollen, denken Sie daran, dass `interface` speziell für Objektformen entwickelt wurde, während `type` flexibler ist und auch Unionstypen, Primitive und komplexere Typkombinationen darstellen kann.
