# --interactive--

`prompt()` メソッドは、JavaScriptがユーザーとやり取りする際に重要な役割を果たします。これは、小さなポップアップダイアログボックスを通じてユーザーから入力を得る最も簡単な方法の一つです。

ウェブページがユーザーから名前やその他のテキスト入力などの情報を必要とする場合によく使われます。

では、`prompt()` メソッドは具体的に何をするのでしょうか？これは、ユーザーに入力を求めるダイアログボックスを開き、ユーザーが入力した文字列を返します。

`prompt()` メソッドは2つの引数を取ります。1つ目はダイアログボックス内に表示されるメッセージで、通常はユーザーに情報の入力を促すものです。2つ目は省略可能なデフォルト値で、入力フィールドに最初から表示されます。

```js
prompt(message, default);
```

動作例を示します。

**注意**：この例にはまだ学んでいないコードが含まれています。コードのすべてを理解しようとしなくて大丈夫です。これは`prompt()` メソッドの動作を示すためのもので、ページが読み込まれた直後にpromptが表示されてしまうのを防ぐための工夫です。インタラクティブエディターが有効なら、自分で試してみてください。

:::interactive_editor

```html
<button id="prompt-btn">Show Prompt</button>
<p id="output"></p>
<script src="index.js"></script>
```

```js
const btn = document.getElementById("prompt-btn");
const output = document.getElementById("output");
btn.addEventListener("click", () => {
  const userName = prompt("What is your name?", "Guest");
  output.textContent = "Hello, " + userName + "!";
});
```

:::

この例では、ユーザーがボタンをクリックすると、`prompt()` メソッドがメッセージ`What is your name?`と、最初に`Guest`という値が入った入力フィールドを持つダイアログボックスを表示します。

ユーザーが名前を入力して「OK」を押すと、`userName` 変数に入力された値が保存されます。ユーザーが「キャンセル」を押すと、`userName` 変数には`null` が設定されます。`null` はユーザーが入力を提供しなかったことを示します。出力の段落には、入力された名前か、ユーザーがキャンセルした場合は`null` を使った挨拶メッセージが表示されます。将来のレッスンで、ユーザーがキャンセルしたときに`null` を表示しないようにする手法を学びます。

`prompt()` メソッドは、ユーザーがダイアログボックスとやり取りするまでスクリプトの実行を停止することに注意してください。

つまり、ユーザーが入力して「OK」をクリックするか、キャンセルするまで、JavaScriptの残りのコードは実行されません。

もう一つ考慮すべき点は、`prompt()` は簡単なテストや小規模なアプリケーションには便利ですが、その中断的な性質やブラウザ間での動作の違いから、現代の複雑なウェブアプリケーションでは一般的に避けられていることです。

`prompt()` メソッドを理解することで、ブラウザを通じてユーザーと直接やり取りし情報を取得するシンプルな方法を得られますが、現代のウェブアプリケーションではあまり使われない場合が多いです。
