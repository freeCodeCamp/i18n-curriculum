# --description--

Ein CSS-Preprocessor ist ein Werkzeug, das standardmäßiges CSS mit leistungsstarken Funktionen wie Variablen, Mixins, Verschachtelung und Selektor-Vererbung erweitert.

Einige dieser Funktionen, wie Variablen und Verschachtelung, werden inzwischen von nativem CSS unterstützt oder erhalten mehr Unterstützung, und dieser Trend wird sehr wahrscheinlich anhalten. Frühere Versionen von nativem CSS unterstützten diese Funktionen jedoch nicht.

Deshalb wurden CSS-Preprocessor überhaupt so weit verbreitet. Sie nehmen die CSS-Regeln und -Stile, die Sie schreiben, zusammen mit ihrer erweiterten Syntax, und kompilieren sie in eine native CSS-Datei, die Browser verstehen können.

Durch die Verwendung eines CSS-Preprocessors können Sie Ihren CSS-Code auf eine wiederverwendbare und logischere Weise strukturieren. Außerdem haben Sie Zugriff auf leistungsstarke Funktionen wie Mixins, die von CSS nicht direkt unterstützt werden.

Einige der beliebtesten CSS-Preprocessor sind Sass, Less und Stylus.

Sprechen wir über Sass.

Sass steht für „Syntactically Awesome Style Sheets“. Es ist mit allen CSS-Versionen kompatibel und wird von einer großen Entwickler-Community gepflegt.

Sass unterstützt Funktionen wie:

- Variablen, mit denen Sie Werte im gesamten Stylesheet speichern und wiederverwenden können.

- Verschachtelte CSS-Regeln, mit denen Sie eine visuelle Hierarchie in Ihrer Datei erstellen können.

- Module, mit denen Sie Ihre Stile in mehrere Stylesheets aufteilen können.

- Mixins, mit denen Sie CSS-Deklarationen auf Ihrer gesamten Webseite wiederverwenden können.

- Vererbung, damit mehrere CSS-Selektoren Eigenschaften teilen können.

- Und Operatoren für grundlegende mathematische Operationen.

Sass unterstützt außerdem zwei Syntaxen.

Die SCSS-Syntax steht für Sassy CSS und ist eine Obermenge von CSS. Das bedeutet, dass SCSS die grundlegende Syntax von CSS erweitert.

SCSS ist die gebräuchlichste Syntax, die Sie beim Arbeiten mit Sass verwenden und antreffen werden. Sie erfordert die Verwendung von geschweiften Klammern (`{` und `}`) um CSS-Eigenschaften und Semikolons (`;`) am Ende von CSS-Deklarationen, genau wie natives CSS.

Diese Dateien haben die `.scss`-Dateiendung. Hier ist ein Beispiel:

```scss
$primary-color: #3498eb;

header {
  background-color: $primary-color;
}
```

Was Sie hier oben sehen, ist eine in SCSS definierte Variable. Diese Variable wird in der CSS-Regel darunter verwendet.

Vergleichen wir SCSS mit einer seltener verwendeten Syntax, der eingerückten Syntax. Diese ist auch als „Sass-Syntax“ bekannt, da sie die ursprüngliche Syntax von Sass war.

Diese Syntax basiert auf Einrückungen, um die Regeln zu definieren. Hier ist ein Beispiel:

```scss
$primary-color: #3498eb

header
  background-color: $primary-color
```

Beachten Sie, dass es keine geschweiften Klammern um die CSS-Regel gibt und keine Semikolons am Ende der CSS-Deklarationen. Oben sehen Sie ebenfalls eine Variable. Beachten Sie, dass ihr Name in dieser Syntax mit einem Dollarzeichen (`$`) beginnt, was sich von nativen CSS-Variablen unterscheidet, aber sehr ähnlich funktioniert.

Sass unterstützt auch eine leistungsstarke Funktion namens Mixins.

Mixins erlauben es Ihnen, mehrere CSS-Eigenschaften und deren Werte unter einem Namen zu gruppieren und diesen CSS-Codeblock im gesamten Stylesheet wiederzuverwenden.

Das macht Ihren CSS-Code weniger repetitiv und leichter wartbar, denn wenn Sie etwas in diesem Block ändern, wird die Änderung überall dort angewendet, wo Sie das Mixin verwenden.

Da Mixins benutzerdefinierte und beschreibende Namen haben können, helfen sie dabei zu verstehen, was jeder CSS-Codeblock bewirkt, was Ihre Stylesheets leichter verständlich macht.

Mixins fördern außerdem die Konsistenz in Stylesheets, indem sie einheitliche Stile anwenden.

Dies ist ein Beispiel für ein Mixin in SCSS-Syntax:

```scss
@mixin center-flex {
  display: flex;
  justify-content: center;
  align-items: center;
}
```

Um ein Mixin zu definieren, beginnen Sie mit der `@mixin`-At-Regel, gefolgt vom Namen des Mixins. In diesem Fall heißt das Mixin `center-flex`. Es enthält drei CSS-Eigenschaften, um Elemente mit Flexbox zu zentrieren.

Sobald Sie Ihr Mixin definiert haben, verwenden Sie die `@include`-At-Regel, um diese Eigenschaften in einer CSS-Regel einzufügen. Sie müssen nur `@include` gefolgt vom Namen des Mixins schreiben. In diesem Fall ist der Name `center-flex`:

```scss
section {
  @include center-flex;
  height: 500px;
  background-color: #3289a8;
}
```

Dies ist der vollständige Code mit dem Mixin und der CSS-Regel:

```scss
@mixin center-flex {
  display: flex;
  justify-content: center;
  align-items: center;
}

section {
  @include center-flex;
  height: 500px;
  background-color: #3289a8;
}
```

Sie können das Mixin in so vielen CSS-Regeln verwenden, wie Sie möchten. Wenn Sie Änderungen vornehmen müssen, ändern Sie nur das Mixin selbst, und die Änderungen werden überall automatisch angewendet. Dieses Beispiel ist in SCSS-Syntax. Beachten Sie, dass es die geschweiften Klammern und Semikolons enthält.

Hier ist das Äquivalent in eingerückter Syntax, auch bekannt als Sass-Syntax, ohne geschweifte Klammern oder Semikolons:

```scss
@mixin center-flex
  display: flex
  justify-content: center
  align-items: center

section
  @include center-flex
  height: 500px
  background-color: #3289a8
```

Und dies ist der kompilierte CSS-Code, den der Browser tatsächlich interpretiert, nachdem die Datei in natives CSS kompiliert wurde:

```css
section {
  display: flex;
  justify-content: center;
  align-items: center;
  height: 500px;
  background-color: #3289a8;
}
```

Beachten Sie, dass er die drei Mixin-Eigenschaften oben enthält: `display`, `justify-content` und `align-items`.

Dies ist nur ein einfaches Beispiel dafür, was CSS-Preprocessor leisten können. Sie haben viele leistungsstarke Funktionen, die es wert sind, gelernt und im täglichen Workflow angewendet zu werden.

Allerdings haben sie auch potenzielle Nachteile, deren Sie sich bewusst sein sollten.

Erstens müssen CSS-Preprocessor die CSS-Regeln in standardmäßiges CSS kompilieren. Obwohl der potenzielle Mehraufwand dieses Kompilierungsprozesses oft minimal in Bezug auf Zeit und Ressourcen ist, sollten Sie dies bei realen Projekten berücksichtigen.

Außerdem können sie Debugging-Herausforderungen erzeugen, da Browser das kompilierte CSS direkt verwenden. Herauszufinden, was einen problematischen Stil aus der erweiterten Syntax erzeugt, kann im Vergleich zu standardmäßigem CSS einige zusätzliche Schritte erfordern.

Die Vorteile von CSS-Preprocessor überwiegen jedoch meist ihre Nachteile, besonders bei komplexen Projekten. Sie können sehr hilfreich sein, um saubereren, wiederverwendbaren, weniger repetitiven und skalierbaren CSS-Code zu schreiben.
