# --description--

現在你已經建立了一個基本的 Express 應用程式，是時候更深入了解 routing 以及它的運作方式。

在 Express 中，routing 是你定義網頁應用程式端點的方式。你可以把路由想像成一組規則，告訴應用程式當使用者造訪特定 URL 時該如何回應。

讓我們再看看這個簡單的 Express 應用程式：

```javascript
const express = require("express");
const app = express();
const port = 3000;

app.get("/", (req, res) => {
  res.send("Hello World!");
});

app.listen(port, () => {
  console.log(`Example app listening on port ${port}`);
});
```

在這個範例中，應用程式定義了根目錄 URL（`/`）的路由。當客戶端對這個路徑發送 GET 請求時，伺服端會回應文字 `Hello World!`。接著應用程式開始監聽埠 `3000` 上的進來請求。

但還有其他路由方法。

Express 讓你可以使用不同的路由方法來控制不同類型的 HTTP 請求。最常見的有：

* `app.get()` – 取得資料
    
* `app.post()` – 創建新資料
    
* `app.put()` – 更新既有資料
    
* `app.delete()` – 刪除資料
    

每個方法定義了你的應用程式應該如何回應特定路由上的特定 HTTP 請求。

以下是幾個簡單的範例：

```javascript
app.post("/submit", (req, res) => {
  res.send("Form submitted!");
});

app.put("/update", (req, res) => {
  res.send("Item updated!");
});

app.delete("/delete", (req, res) => {
  res.send("Item deleted!");
});
```

在每個案例中，Express 監聽特定 HTTP 方法和 URL 路徑的請求，然後執行對應的處理常式來回應客戶端。

你可以使用像 Postman 或 `curl` 這類工具來測試這些路由，這些工具讓你能發送不同類型的 HTTP 請求到你的伺服器。

這就是 Express 中的 routing——簡單來說，就是定義你的應用程式如何回應不同的請求。
