# --interactive--

## `Audio` コンストラクタとよく使うメソッド

- **定義**: `Audio` コンストラクタは他のコンストラクタと同様に、`new` キーワードで呼び出される特別な関数です。これは`HTMLAudioElement` を返し、それを使ってユーザーにオーディオを再生したり、ユーザーが自分で操作できるようにDOMに追加したりできます。コンストラクタを呼び出す際に、オプションでURLを（唯一の）引数として渡せます。このURLは再生したいオーディオファイルのソースを指すべきです。あるいは、ソースを動的に変更する必要がある場合は、返されたオーディオ要素の`src` プロパティにURLを代入できます。
- **`play()` メソッド**: このメソッドは`audio` または`video` 要素で使い、メディアの再生を開始します。

:::interactive_editor

```html
<audio id="audio" controls>
  <source src="https://www.w3schools.com/html/horse.mp3" type="audio/mpeg">
</audio>

<button id="playBtn">Play Audio</button>

<script>
  const audio = document.getElementById("audio");
  const playBtn = document.getElementById("playBtn");

  playBtn.addEventListener("click", () => {
    audio.play();
  });
</script>
```

:::

- **`pause()` メソッド**: このメソッドは`audio` または`video` 要素で使い、メディアの再生を一時停止します。現在の位置を維持するため、`play()` が呼ばれるとその位置から再生が始まります。

:::interactive_editor

```html
<audio id="myAudio" controls>
  <source src="https://www.w3schools.com/html/horse.mp3" type="audio/mpeg">
</audio>

<br>

<button id="pauseBtn">Pause Audio</button>

<script>
  document.addEventListener("DOMContentLoaded", () => {
    const audio = document.getElementById("myAudio");
    const pauseBtn = document.getElementById("pauseBtn");

    pauseBtn.addEventListener("click", () => {
      audio.pause();
    });
  });
</script>
```

:::

- **`addTextTrack()` メソッド**: このメソッドはメディア要素にテキストトラックを指定でき、特にビデオに字幕を追加するのに役立ちます。
- **`fastSeek()` メソッド**: このメソッドは再生位置をメディア内の特定の時間に移動できます。

`Audio()` コンストラクタにはメソッドだけでなくプロパティもあります。主なプロパティは以下の通りです：

- `currentTime`: オーディオの現在の再生時間を取得するためのもの
- `loop`: 再生が終わると自動的に再スタートして連続再生するためのもの
- `muted`: 音量設定に関係なくメディア要素の音声出力を消音するためのもの

## さまざまなオーディオとビデオのフォーマット

- **MIMEタイプ**: MIMEタイプはMultipurpose Internet Mail Extensionsの略で、ファイルタイプをプログラム的に示す標準化された方法です。MIMEタイプはブラウザなどのアプリケーションに特定のファイルの扱い方を伝えます。オーディオやビデオの場合、MIMEタイプはウェブページに埋め込めるマルチメディア形式であることを示します。
- **`source` 要素**: これはファイルタイプとソースを指定するために使われ、複数の異なるタイプを複数のsource要素で指定できます。こうするとブラウザはユーザーの環境に最適なフォーマットを選択します。
- **MP3**: 音楽や音声、サウンドを保存するためのデジタルファイル形式の一種です。圧縮された音声録音でファイルサイズが小さくなり、保存や共有がしやすくなっています。MP3は最も広くブラウザでサポートされており、MIMEタイプは`audio/mpeg`です。
- **MP4**: 動画と音声を保存するためのデジタルファイル形式です。映像（画像）と音声（音楽や音声）を1つのファイルにまとめるコンテナとして機能します。MP4は動画ファイルか音声のみかによって、MIMEタイプが`audio/mp4` または`video/mp4` になります。
- その他のフォーマットには、Windows Media Playerアプリに関連する`WMV`、`OGG`、`MKV`、`AVI`などがあります。

## コーデック

- **定義**: コーデックは「エンコーダー/デコーダー」の略で、アナログとデジタルの間でオーディオやビデオを変換するアルゴリズムやソフトウェアです。コーデックはMIMEタイプの一部として指定できます。基本的な構文はメディアタイプの後にセミコロンを付けて、`codecs=` とコーデック名を追加します。

## HTMLMediaElement API

- **定義**: `HTMLMediaElement` APIはページ上のオーディオやビデオ要素の動作を制御するために使われます。これは基本の`HTMLElement` インターフェースを拡張しており、基本的なプロパティやメソッドに加え、`waiting`、`ended`、`canplay`、`canplaythrough` などの便利なイベントも利用できます。メソッドの例には、ブラウザがオーディオファイルを再生可能かどうかを確認するための`play()`、`fastSeek()`、`pause()`、`canPlayType()` があります。

## Media Capture and Streams API

- **定義**: Media Capture and Streams API、またはMediaStream APIはデバイスからオーディオやビデオをキャプチャするために使われます。このAPIを使うには`MediaStream` オブジェクトを作成する必要があります。コンストラクタで作成することもできますが、それはユーザーのハードウェアに紐づきません。代わりに、`mediaDevices` navigatorオブジェクトの`global` プロパティには使える`getUserMedia()` メソッドがあります。

```js
window.navigator.mediaDevices.getUserMedia({
  audio: true,
  video: {
    width: {
      min: 1280,
      ideal: 1920,
      max: 3840
    },
    height: {
      min: 720,
      ideal: 1080,
      max: 2160
    }
  }
});
```

## Screen Capture API

- **定義**: Screen Capture APIはユーザーの画面を録画するために使われます。このAPIは`getDisplayMedia()` オブジェクトの`mediaDevices` メソッドを呼び出して公開され、返されたメディアストリームを利用します。

## MediaStream Recording API

- **定義**: MediaStream Recording APIはMediaStreams APIと連携して動作し、MediaStream（または`HTMLMediaElement` を直接）録画できるようにします。

## Media Source Extensions API

- **トピック**: Media Source Extensions APIはユーザーのウェブカメラ映像を`srcObject` プロパティを使ってビデオ要素に直接渡せるようにします。

## Web Audio API

- **定義**: Web Audio APIはウェブ上のすべての音声を支えています。このAPIには、オーディオデータを特に含むバッファを表す`AudioBuffer` や、オーディオ処理グラフを表す`AudioContext` など重要なオブジェクトが含まれています。
