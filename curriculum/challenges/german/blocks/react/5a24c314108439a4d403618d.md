# --description--

Bisher haben Sie React-Komponenten auf dem Client gerendert. Normalerweise ist das auch das Vorgehen, das Sie immer wählen werden. Es gibt jedoch einige Anwendungsfälle, in denen es sinnvoll ist, eine React-Komponente auf dem Server zu rendern. Da React eine JavaScript-View-Bibliothek ist und Sie JavaScript mit Node auf dem Server ausführen können, ist dies möglich. Tatsächlich stellt React eine `renderToString()`-Methode bereit, die Sie zu diesem Zweck verwenden können.

Es gibt zwei Hauptgründe, warum das Rendern auf dem Server in einer realen Anwendung genutzt werden kann. Erstens würde Ihre React-App ohne diese Methode beim ersten Laden im Browser aus einer relativ leeren HTML-Datei und einem großen JavaScript-Bündel bestehen. Das ist für Suchmaschinen, die versuchen, den Inhalt Ihrer Seiten zu indexieren, damit Menschen Sie finden können, nicht ideal. Wenn Sie das initiale HTML-Markup auf dem Server rendern und an den Client senden, enthält der erste Seitenaufruf das gesamte Markup der Seite, das von Suchmaschinen durchsucht werden kann. Zweitens sorgt dies für ein schnelleres erstes Laden der Seite, weil das gerenderte HTML kleiner ist als der JavaScript-Code der gesamten App. React kann Ihre App nach dem ersten Laden weiterhin erkennen und verwalten.

# --instructions--

Die `renderToString()`-Methode wird auf `ReactDOMServer` bereitgestellt, das hier als globales Objekt verfügbar ist. Die Methode nimmt ein Argument entgegen, das ein React-Element ist. Verwenden Sie dies, um `App` in einen String zu rendern.

# --hints--
