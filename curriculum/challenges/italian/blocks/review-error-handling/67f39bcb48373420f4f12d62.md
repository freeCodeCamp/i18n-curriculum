## Errori comuni in Python

- **SyntaxError**: l'errore che Python genera quando il tuo codice non rispetta le sue regole di sintassi. Per esempio, il codice `print("Hello there"` porterà a un errore di sintassi con il messaggio `SyntaxError: '(' was never closed`, perché manca una parentesi di chiusura.
- **NameError**: Python genera un `NameError` quando provi a consultare una variabile o funzione che non hai definito. Per esempio, se hai la riga `print(username)` nel tuo codice senza aver prima definito una variabile `username`, otterrai un errore di nome con il messaggio `NameError: name 'username' is not defined`.
- **TypeError**: questo è l'errore che Python lancia quando esegui un'operazione su due o più tipi di dati incompatibili. Per esempio, se provi ad aggiungere una stringa a un numero, otterrai l'errore `TypeError: can only concatenate str (not "int") to str`.
- **IndexError**: otterrai un `IndexError` se consulti un indice che non esiste in una lista o in altre sequenze come tuple e stringhe. Per esempio, in una stringa `Hello world`, l'indice dell'ultimo carattere è `11`. Se provi a consultare un carattere in questo modo, `greet = "hello world"; print(greet[12])`, otterrai un errore con il messaggio `IndexError: string index out of range`.
- **AttributeError**: Python genera questo errore quando provi a usare un metodo o una proprietà che non esiste in un oggetto di quel tipo. Per esempio, chiamare `.append()` su una stringa come `"hello".append("!")` porterà a un errore con il messaggio `AttributeError: 'str' object has no attribute 'append'`.

## Buone tecniche di debugging in Python

- **Usare la funzione `print`**: inserire istruzioni `print()` in vari punti del codice durante il debugging ti aiuta a vedere i valori delle variabili e come scorre il codice.
- **Usare il debugger integrato di Python (`pdb`)**: Python fornisce un modulo `pdb` per il debugging. Fa parte della libreria standard di Python, quindi è sempre disponibile. Con `pdb` puoi impostare un trace con la funzione `set_trace()` per iniziare a eseguire il codice passo passo e ispezionare le variabili in modo interattivo.
- **Sfruttare gli strumenti di debugging degli IDE**: molti ambienti di sviluppo integrati (IDE) e editor di codice come Pycharm e VS Code offrono strumenti di debugging con breakpoint, esecuzione passo passo, ispezione delle variabili e altre funzionalità di debugging.

## Gestione delle eccezioni

- **`try...except`**: serve per eseguire un blocco di codice che potrebbe generare un'eccezione. Il blocco `try` è dove prevedi che possa verificarsi un errore, mentre il blocco `except` cattura un'eccezione specifica e viene eseguito se quell'errore si verifica. Ecco un esempio:

  ```py
  try:
    print(22 / 0)
  except ZeroDivisionError:
    print('You can\'t divide by zero!')
    # You can't divide by zero!
  ```

  Puoi anche concatenare più blocchi `except` per gestire diversi tipi di eccezioni:

  ```py
  try:
    number = int(input('Enter a number: '))
    print(22 / number)
  except ZeroDivisionError:
    print('You cannot divide by zero!')
    # You cannot divide by zero! prints when you enter 0
  except ValueError:
    print('Please enter a valid number!')
    # Please enter a valid number! prints when you enter a string  
  ```

- **`else` e `finally`**: questi blocchi estendono `try...except`. Se non si verifica nessuna eccezione, viene eseguito il blocco `else`. Il blocco `finally` viene eseguito sempre, indipendentemente dagli errori.

  ```py
  try:
    result = 100 / 4
  except ZeroDivisionError:
    print('You cannot divide by zero!') # This will not run
  else:
    print(f'Result is {result}') # Result is 25.0
  finally:
    print('Execution complete!') # Execution complete!
  ```

- **Oggetto eccezione**: ti permette di consultare l'eccezione stessa per un debugging migliore e per stampare direttamente il messaggio di errore. Per accedere all'oggetto eccezione, devi usare la parola chiave `as`. Ecco un esempio:

  ```py
  try:
      value = int('This will raise an error')
  except ValueError as e:
      print(f'Caught an error: {e}')
      # Caught an error: invalid literal for int() with base 10: 'This will raise an error'
  ```

- **L'istruzione `raise`**: ti permette di lanciare manualmente un'eccezione. Puoi usarla per generare un'eccezione quando si verifica una certa condizione. Ecco un esempio:

  ```py
  def divide(a, b):
      if b == 0:
          raise ZeroDivisionError('You cannot divide by zero')
      return a / b
  ```
  
## Segnalazione delle eccezioni
  
  L'istruzione `raise` è utile anche quando crei le tue eccezioni personalizzate, perché puoi usarla per lanciare un'eccezione con un messaggio personalizzato. Ecco un esempio:

  ```py
  class InvalidCredentialsError(Exception):
      def __init__(self, message="Invalid username or password"):
          self.message = message
          super().__init__(self.message)

  def login(username, password):
      stored_username = "admin"
      stored_password = "password123"
      
      if username != stored_username or password != stored_password:
          raise InvalidCredentialsError()
      
      return f"Welcome, {username}!"
  ```  

  Ecco come puoi usare la funzione `login` con l'eccezione `InvalidCredentialsError`:

  ```py
  # failed login attempt
  try:
      message = login("user", "wrongpassword")
  except InvalidCredentialsError as e:
      print(f"Login failed: {e}")
  else:
      print(message)

  # successful login attempt
  try:
      message = login("admin", "password123")
  except InvalidCredentialsError as e:
      # This block is not executed because the login was successful
      print(f"Login failed: {e}")
  else:
      # The else block runs if the 'try' block completes without an exception
      print(message)
  ```

  L'istruzione `raise` può essere usata anche con la parola chiave `from` per concatenare eccezioni, mostrando la relazione tra errori diversi:

  ```py
  def parse_config(filename):
    try:
        with open(filename, 'r') as file:
            data = file.read()
            return int(data)
    except FileNotFoundError:
        raise ValueError('Configuration file is missing') from None
    except ValueError as e:
        raise ValueError('Invalid configuration format') from e

  config = parse_config('config.txt')
  ```
