# --description--

Вебсайти та застосунки потребують збору даних і їх обробки або створення чогось, що потрібно користувачу, наприклад, результатів пошуку. Тож, як і текст, форми є фундаментальною частиною вебу.

За допомогою форм користувачі можуть реєструватися та входити на сайт, надсилати повідомлення та шукати елементи. Наприклад, ви отримуєте доступ до сторінки результатів пошуку Google, вводячи свій запит у найпопулярнішому на вебі полі введення пошуку Google.

Давайте розглянемо, з чого складається форма: поля введення, атрибут `action`, методи форми і, найголовніше, що відбувається, коли ви відправляєте форму.

Форми вставляються на вебсторінку за допомогою елемента `form`, але щоб поля введення відображалися, вам також потрібні елементи `input` залежно від вашої потреби. Деякі з цих елементів введення — текстові, е-пошта, пароль, вибір, файл, приховані поля та інші.

Щоб використовувати ці поля введення, їх потрібно обгорнути елементом `form`. Кожне поле введення має мати атрибут `name`, щоб сервер знав, які дані відповідають кожному полю.

Ось приклад:

```html
<form>
   <input type="text" name="username" />
   <input type="email" name="email" />
   <input type="file" name="profilePicture" />
   <input type="radio" name="gender" />
   <input type="checkbox" name="subscribe" />
   <input type="hidden" name="userId" />
</form>
```

Поля введення також потрібно валідовувати, щоб переконатися, що користувач вводить правильну інформацію.

Ця валідація може відбуватися як на стороні клієнта, так і на стороні сервера. HTML дозволяє робити базову клієнтську валідацію за допомогою атрибутів `required` і `pattern`. Проте також слід робити валідацію на сервері для додаткової безпеки, адже HTML-валідація здебільшого покращує користувацький досвід і дає миттєвий зворотний зв’язок.

Атрибут `required` вказує, що поле введення має бути заповнене, а атрибут `pattern` дозволяє вводити регулярні вирази, які змушують користувача вводити інформацію у потрібному форматі.

Ось приклад для поля імені користувача:

```html
<input type="text" name="username" required pattern="[A-Za-z0-9]{3,}" />
```

Після валідації полів введення має відбутися відправлення. То що ж відбувається, коли форму відправляють?

По-перше, для відправлення форма має мати атрибут `action`, який вказує, куди браузер має надіслати значення полів введення, і атрибут `method`, який визначає, як відправляються дані форми. Про атрибут `method` поговоримо пізніше.

Ось як може виглядати атрибут action:

```html
<form action="/register">
	<!-- input fields -->
</form>
```

Атрибут `method` може мати значення `GET`, `POST`, `PUT`, `PATCH` або `DELETE`:

* `GET` отримує щось із сервера, наприклад, результати пошуку.
    
* `POST` надсилає щось на сервер, наприклад, інформацію для входу або реєстрації, введену через поля введення.
    
* `PUT` змінює щось на сервері.
    
* `PATCH` частково змінює щось на сервері.
    
* `DELETE` видаляє щось із сервера.
    
Зверніть увагу, що HTML безпосередньо не підтримує методи `PUT`, `PATCH` і `DELETE`, але існують обхідні шляхи. Якщо ви працюєте з чистим JavaScript, можна використовувати Fetch API для вказання методу, а в Node.js і Express — пакет `method-override`.

Ось як може виглядати форма, що робить запит `GET`:

```html
<form action="/search" method="GET">
   <input type="text" name="q">
   <button type="submit">Search</button>
</form>
```

А ось як може виглядати форма, що робить запит `POST`:

```html
<form action="/submit" method="POST">
   <input type="text" name="username" required pattern="[A-Za-z0-9]{3,}" />
   <input type="password" name="password" />
   <button type="submit">Submit</button>
</form>
```

Після запиту `GET` або `POST` сервер перевіряє валідацію на стороні сервера і або обробляє, або відхиляє значення введення.

Якщо валідація успішна, сервер надсилає запитувану інформацію або видає помилку, якщо валідація не пройдена.

Якщо валідація успішна, відповідь від сервера може бути перенаправленням у випадку автентифікації або пошуку, або повідомленням про успіх, що підтверджує дію.

Ось короткий підсумок того, що відбувається, коли форму відправляють:

* Користувач заповнює поля форми.
    
* Браузер перевіряє клієнтську валідацію (якщо вона є) і продовжує, якщо валідація успішна.
    
* Браузер надсилає дані форми на сервер відповідно до атрибутів action і method.
    
* Сервер отримує дані і обробляє їх (зберігає у базі даних, реєструє користувача, виконує вхід, надсилає запит пошуку тощо).
    
* Якщо є валідація на стороні сервера, сервер повторно перевіряє дані перед обробкою.
    
* Сервер відповідає повідомленням про успіх, помилкою або перенаправляє користувача залежно від стану валідації (успіх чи помилка).
    
* Користувач бачить зворотний зв’язок на основі відповіді сервера.
