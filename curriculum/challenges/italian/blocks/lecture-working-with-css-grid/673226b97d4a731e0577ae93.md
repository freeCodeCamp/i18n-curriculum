# --interactive--

La proprietà `grid-template-areas` ti permette di progettare un layout visivo a griglia usando etichette nominate.

Poi assegni le etichette a specifici elementi griglia usando la proprietà grid-area. In altre parole, queste etichette nominate sono anche chiamate "nomi delle aree griglia".

Ecco la sintassi base della proprietà `grid-template-areas`:

```css
grid-template-areas:
 'header header header'
 'left-sidebar main right-sidebar'
 'footer footer footer';
```

Ecco i punti da ricordare della sintassi base:

- Valori come `header` e `main` sono i nomi delle aree griglia.

- Ogni valore separato da spazi all’interno di una stringa corrisponde a una colonna.

- Ogni stringa rappresenta una riga nella griglia.

Quindi, nella sintassi, abbiamo un contenitore griglia 3 per 3.

Dopo aver definito il template, devi usare la proprietà `grid-area` e le etichette nominate (o aree) come valori per dire a CSS che un elemento specifico appartiene a quell’area della griglia.

La proprietà `grid-area` collega l’elemento griglia alla regione nominata che definisci nella `grid-template-areas`.

Un modo popolare per mostrare le capacità della proprietà `grid-template-areas` è creare il classico layout Holy Grail.

Se non lo conosci, il layout Holy Grail è un pattern di design web con un’intestazione, un piè di pagina, due barre laterali e un’area contenuto principale. Garantisce che il contenuto principale abbia priorità, mentre le barre laterali e le altre sezioni si adattano in modo responsivo all’interno del layout.

Esistono molte soluzioni per implementare il layout Holy Grail, ma usare `grid-template-areas` e la proprietà `grid-area` è il modo più semplice per crearlo.

Ecco un esempio per il layout Holy Grail:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<div class="grid-container">
  <div class="header">
    <h2>Header</h2>
  </div>
  <div class="sidebar-left">
    <h2>Left Sidebar</h2>
  </div>
  <div class="main"><h2>Main Content</h2></div>
  <div class="sidebar-right">
    <h2>Right Sidebar</h2>
  </div>
  <div class="footer">
    <h2>Footer</h2>
  </div>
</div>
```

```css
.grid-container {
  display: grid;
  grid-template-areas:
    'header header header'
    'sidebar-left main sidebar-right'
    'footer footer footer';
  grid-gap: 10px;
  background-color: #2196F3;
  padding: 10px;
}

.header {
  grid-area: header;
  background-color: rgba(255, 255, 255, 0.8);
  padding: 20px;
  text-align: center;
}

.sidebar-left {
  grid-area: sidebar-left;
  background-color: rgba(255, 255, 255, 0.8);
  padding: 20px;
  text-align: center;
}

.main {
  grid-area: main;
  background-color: rgba(255, 255, 255, 0.8);
  padding: 20px;
  text-align: center;
}

.sidebar-right {
  grid-area: sidebar-right;
  background-color: rgba(255, 255, 255, 0.8);
  padding: 20px;
  text-align: center;
}

.footer {
  grid-area: footer;
  background-color: rgba(255, 255, 255, 0.8);
  padding: 20px;
  text-align: center;
}
```

:::

Nota che sia la proprietà `grid-template-areas` sia la `grid-area` possono essere usate indipendentemente l’una dall’altra.

La proprietà `grid-template-areas` serve specificamente a definire un layout visivo mappando le aree griglia nominate all’interno del contenitore griglia.

D’altra parte, la proprietà `grid-area` serve a posizionare singoli elementi griglia, specificandone la posizione di riga e colonna oppure facendo riferimento alle aree nominate definite con la proprietà `grid-template-areas`.
