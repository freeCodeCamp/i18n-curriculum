# --interactive--

웹 페이지에서 콘텐츠를 숨기는 것은 웹 개발에서 흔한 작업이지만, 접근성을 해치지 않는 방식으로 하는 것이 매우 중요합니다. 다양한 숨김 기법은 보조 기술이 콘텐츠를 해석하고 사용자에게 제공하는 방식에 서로 다른 영향을 줄 수 있습니다. 접근성을 유지하면서 콘텐츠를 숨기는 모범 사례를 살펴보겠습니다. 콘텐츠를 숨기는 일반적인 방법 중 하나는 `display: none`를 사용하는 것입니다. 예를 들어 다음과 같습니다:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<p class="hidden">Hidden text</p>
<p>Visible text</p>
```

```css
.hidden {
  display: none;
}
```

:::

이 방법은 시각적으로 콘텐츠를 숨기지만, 접근성 트리에서도 제거합니다. 접근성 트리는 화면 읽기 프로그램과 같은 보조 기술이 웹 페이지의 콘텐츠를 해석하고 상호작용하는 데 사용하는 구조입니다. 이 트리는 보조 기술이 이해하고 사용자에게 제공할 수 있도록 콘텐츠와 그 의미론적 의미를 나타냅니다.

`display: none`를 사용하면 화면 읽기 프로그램과 다른 보조 기술이 이 콘텐츠에 접근할 수 없는데, 이는 접근성 트리에 포함되지 않기 때문입니다. 따라서 이 방법은 시각적 표시와 접근성 모두에서 콘텐츠를 완전히 제거하려는 경우에만 사용해야 합니다.

또 다른 숨김 방법은 `visibility: hidden`를 사용하는 것입니다:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<p class="hidden">Hidden text</p>
<p>Visible text</p>
```

```css
.hidden {
  visibility: hidden;
}
```

:::

`visibility: hidden`는 콘텐츠를 시각적으로 숨기지만 문서 흐름에는 남아 있어 페이지에서 공간을 차지합니다. `display: none`처럼 `visibility: hidden`도 접근성 트리에서 콘텐츠를 제거합니다. 즉, 화면 읽기 프로그램과 같은 보조 기술이 숨겨진 콘텐츠에 접근할 수 없습니다. `visibility: hidden`는 보조 기술 사용자까지 포함해 모두에게 콘텐츠를 숨기려는 경우에만 사용하세요.

시각적으로 숨기되 화면 읽기 프로그램에는 접근 가능하도록 하려면, 흔히 "visually hidden" 또는 "screen reader only"라고 부르는 기법을 사용할 수 있습니다. 다음은 화면 읽기 프로그램에 접근 가능하면서 시각적으로 숨기는 데 자주 쓰이는 `.sr-only` CSS 클래스를 사용한 예입니다:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<p class="sr-only">Hidden text</p>
<p>Visible text</p>
```

```css
.sr-only {
  position: absolute;
  width: 1px;
  height: 1px;
  padding: 0;
  margin: -1px;
  overflow: hidden;
  clip: rect(0, 0, 0, 0);
  white-space: nowrap;
  border: 0;
}
```

:::

이 예에서는 `position`, `clip`, `white-space` 같은 속성을 사용하고 있는데, 이들은 앞으로 수업에서 배우게 될 내용입니다. 지금은 이 CSS 규칙이 시각적으로 콘텐츠를 숨기면서도 화면 읽기 프로그램에는 접근 가능하게 한다는 점만 기억하세요. 이는 화면 읽기 프로그램 사용자에게 추가 문맥을 제공하면서 시각적 레이아웃에는 영향을 주지 않아 유용합니다.

콘텐츠 가시성을 전환할 때는 `hidden` 속성을 고려하세요:

:::interactive_editor

```html
<p hidden>This content is hidden</p>
<p>This content is visible</p>
```

:::

`hidden` 속성은 대부분 최신 브라우저에서 지원하며, 콘텐츠를 시각적으로 그리고 접근성 트리에서도 숨깁니다. JavaScript로 쉽게 전환할 수 있습니다. 마지막으로, 중요한 콘텐츠를 숨길 때는 주의하세요. 웹사이트를 이해하거나 사용하는 데 중요한 정보라면 모든 사용자에게 보이고 접근 가능해야 합니다. 콘텐츠를 숨기는 것은 진정으로 사용자 경험을 향상시킬 때만 하세요. 이러한 모범 사례를 따르면, 웹사이트와 상호작용하는 방식에 관계없이 모든 사용자가 콘텐츠에 접근할 수 있도록 할 수 있습니다.
