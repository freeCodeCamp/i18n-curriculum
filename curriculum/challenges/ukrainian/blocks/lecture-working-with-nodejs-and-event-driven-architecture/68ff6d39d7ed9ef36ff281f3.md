# --description--

Тепер ви готові зробити перші кроки у світі Node.js, встановивши його на свій комп’ютер. Але спершу, перед тим як встановити Node.js, ми рекомендуємо встановити інструмент під назвою NVM (Node Version Manager). Як випливає з назви, NVM — це інструмент для керування кількома версіями Node.js на одному комп’ютері.

Налаштування цього інструменту з самого початку допоможе вам у майбутньому, адже іноді потрібно буде перемикатися між різними версіями Node.js для різних проєктів. З NVM це можна зробити дуже легко.

Давайте подивимося, як встановити NVM на macOS/Linux і Windows, а також як ним користуватися для встановлення Node.js.

## Як встановити NVM на macOS/Linux

На macOS і Linux ви можете встановити NVM безпосередньо через shell-скрипт.

Але для macOS зверніть увагу, що офіційна документація NVM зазначає: «потрібно вручну встановити інструменти командного рядка Xcode перед запуском скрипта встановлення, інакше він не спрацює».

На macOS ви можете встановити інструменти командного рядка Xcode, виконавши цю команду у терміналі:

```bash
xcode-select --install
```

Після завершення встановлення ви можете перевірити номер версії та підтвердити встановлення цією командою:

```bash
xcode-select --version
```

Потім настав час встановити NVM. Ви можете скачати та виконати скрипт встановлення NVM, виконавши одну з цих команд:

```bash
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.3/install.sh | bash
```

```bash
wget -qO- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.3/install.sh | bash
```

Щоб отримати останню версію NVM на момент читання цієї статті, зверніться до офіційної документації NVM: `https://github.com/nvm-sh/nvm?tab=readme-ov-file#install--update-script`. Там ви знайдете ці команди з актуальним номером версії.

Можливо, вам доведеться перезапустити термінал після завершення процесу, щоб завантажити нову команду та конфігурацію.

Потім виконайте цю команду, щоб підтвердити, що NVM встановлено правильно, і перевірити вашу поточну версію:

```bash
nvm -v
```

У вихідних даних ви побачите версію NVM.

## Як встановити Node.js за допомогою NVM на macOS/Linux

Тепер, коли у вас встановлено NVM на macOS/Linux, ви можете встановити Node.js.

Щоб встановити останню версію з довготривалою підтримкою (LTS) Node.js, виконайте цю команду:

```bash
nvm install --lts
```

Зверніть увагу на прапорець `--lts`. Цей прапорець означає, що ви хочете встановити поточну версію Node.js з довготривалою підтримкою (LTS). Цю версію зазвичай рекомендують, бо вона надає перевагу стабільності, надійності та безпеці над новими або експериментальними функціями. Вона ретельно протестована і готова для продакшену. Також гарантується довший період підтримки з акцентом на виправлення помилок і патчі безпеки.

Альтернативно, ви можете встановити конкретну версію Node.js, вказавши номер версії в команді:

```bash
nvm install <version>
```

Наприклад, ось як встановити Node.js версії 20:

```bash
nvm install 20
```

Після встановлення версії Node.js ви можете переключитися на неї за допомогою цієї команди:

```bash
nvm use <version>
```

Наприклад, ось як переключитися на версію 20:

```bash
nvm use 20
```

Нарешті, ви можете перевірити поточну версію Node.js цією командою:

```bash
node -v
```

## Як встановити NVM на Windows

Тепер подивимося, як встановити та використовувати NVM на Windows.

Офіційні версії NVM підтримують Windows лише в певних випадках. Якщо у вас налаштовано Windows Subsystem for Linux (WSL) на вашому пристрої, ви можете слідувати інструкціям для macOS/Linux вище, щоб встановити та використовувати NVM.

Однак для самої Windows існує кілька альтернатив. Одна з них — `nvm-windows`, утиліта для керування версіями Node.js на Windows. Це абсолютно окремий проєкт від офіційного NVM. Але мета цих інструментів одна — допомогти вам працювати з кількома версіями Node.js на одному пристрої.

Перед встановленням `nvm-windows` слід знати, що офіційний репозиторій рекомендує видалити будь-які попередні встановлення Node перед встановленням NVM для Windows. Якщо ви вперше встановлюєте Node.js, це не проблема. Якщо у вас вже є інші версії Node.js, рекомендується спочатку їх видалити.

Щоб встановити `nvm-windows`:

- Перейдіть до розділу Releases офіційного репозиторію: `https://github.com/coreybutler/nvm-windows/releases`.
- Скачайте останній інсталятор.
- Дотримуйтесь інструкцій.

Після завершення встановлення можливо доведеться перезапустити термінал, якщо команда `nvm` не доступна одразу.

Ви можете перевірити, чи встановлення пройшло успішно, і побачити вашу поточну версію `nvm-windows`, виконавши цю команду:

```powershell
nvm -v
```

## Як встановити Node.js за допомогою NVM для Windows

Тепер, коли у вас встановлено `nvm-windows` на Windows, ви готові встановити Node.js.

Щоб встановити останню версію Node.js з довготривалою підтримкою (LTS), виконайте цю команду:

```powershell
nvm install lts
```

Версію з довготривалою підтримкою (LTS) зазвичай рекомендують, але ви можете встановити іншу версію, вказавши її номер у команді:

```powershell
nvm install <version>
```

Наприклад:

```powershell
nvm install 20
```

Після завершення встановлення ви можете переключитися на цю версію Node.js за допомогою цієї команди:

```powershell
nvm use <version>
```

Наприклад:

```powershell
nvm use 20
```

Також ви можете перевірити поточну версію Node.js цією командою:

```bash
node -v
```

## Основні команди

Тепер, коли ви знаєте, як встановити NVM і Node.js, давайте розглянемо кілька основних команд, які будуть корисні при роботі з Node.js і NVM.

Також, хоча всі ці команди працюватимуть для NVM, встановленого на macOS/Linux, включно з WSL, вони можуть працювати інакше або не працювати зовсім для `nvm-windows`. Ми надамо альтернативні команди для `nvm-windows` там, де це можливо.

### Як налаштувати політику виконання Powershell

Якщо ви працюєте на Windows і користуєтеся Powershell та `nvm-windows`, можливо, вам доведеться налаштувати політику виконання Powershell. Політика виконання за замовчуванням для Powershell — `Restricted`, що є заходом безпеки для запобігання запуску скриптів.

Спершу запустіть Powershell від імені адміністратора, клацнувши правою кнопкою на Powershell, а потім вибравши «Запустити від імені адміністратора».

Потім, щоб побачити поточну політику виконання для поточної сесії Powershell, виконайте таку команду:

```powershell
Get-ExecutionPolicy
```

Нарешті, щоб встановити політику виконання Powershell на `RemoteSigned`, що дозволяє запускати локальні скрипти та `npm`, використайте цю команду:

```powershell
Set-ExecutionPolicy RemoteSigned
```

Після виконання цієї команди ви можете закрити вікно Powershell адміністратора і відкрити звичайне вікно Powershell.

### Переглянути всі версії

Щоб переглянути всі встановлені версії Node.js і перевірити, яка зараз використовується, виконайте цю команду:

```bash
nvm ls
```

### Встановити версію Node.js за замовчуванням (тільки NVM на macOS/Linux)

Ця команда встановлює версію Node.js за замовчуванням, яка буде використовуватися у нових сесіях терміналу:

```bash
nvm alias default <version>
```

Наприклад:

```bash
nvm alias default 22.20.0
```

Ви також можете створювати кастомні псевдоніми, щоб призначити більш конкретну назву версії:

```bash
nvm alias <new_name> <version>
```

Наприклад:

```bash
nvm alias new-api 18.20.8
```

Після створення псевдоніма ви можете використовувати цю нову назву для перемикання на цю версію:

```bash
nvm use <new_name>
```

Наприклад:

```bash
nvm use new-api
```

### Видалити версію Node.js

Щоб видалити версію Node.js, виконайте цю команду:

```bash
nvm uninstall <version>
```

Наприклад:

```bash
nvm uninstall 20.19.5
```

За допомогою NVM для macOS/Linux ви можете видалити версію Node.js, виконавши команду `uninstall` з основною версією, наприклад `22` або `20`. Наприклад:

```bash
nvm uninstall 20
```

Якщо у вас встановлено кілька релізів для цієї основної версії, наприклад `20.14.0` і `20.12.2`, спочатку буде видалено останню версію. У цьому випадку виконання `nvm uninstall 20` спочатку видалить `20.14.0`.

### Перевірити версію Node.js

Щоб перевірити поточну версію Node.js, використайте таку команду:

```bash
node -v
```

### Перевірити версію npm

`npm` — це менеджер пакетів для Node.js. Ви можете використовувати його для встановлення, публікації та керування пакетами програмного забезпечення. Він автоматично встановлюється разом із Node.js.

Щоб перевірити поточну версію `npm`, виконайте цю команду:

```bash
npm -v
```

### Запустити JavaScript-файл

Ця команда запускає JavaScript-файл, який ви вкажете, у поточному активному середовищі виконання Node.js:

```bash
node <file.js>
```

Наприклад:

```bash
node app.js
```

### Створити проєкт Node.js

Ця команда створює новий проєкт Node.js, створюючи файл `package.json`, який відстежує його деталі та залежності.

```bash
npm init
```

Коли ви виконуєте команду `npm init`, вона проведе вас через інтерактивний інтерфейс командного рядка для налаштування файлу `package.json`. Вас запитають про назву проєкту, опис, версію тощо. Якщо хочете почати з файлу `package.json` за замовчуванням, можете використати команду `npm init -y` або `npm init --yes`.

### Встановити пакет

Щоб встановити конкретний пакет, виконайте:

```bash
npm install <package>
```

Наприклад:

```bash
npm install express
```

### Встановити залежності

Щоб встановити всі залежності, перелічені у файлі `package.json` проєкту, виконайте:

```bash
npm install
```
