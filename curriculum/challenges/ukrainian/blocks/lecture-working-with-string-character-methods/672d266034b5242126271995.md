# --interactive--

У програмуванні розуміння того, як символи представлені у вигляді чисел, є фундаментальним. Саме тут і з’являється ASCII. ASCII, скорочення від American Standard Code for Information Interchange, — це стандарт кодування символів, який використовується в комп’ютерах для представлення тексту. Він призначає числове значення кожному символу, яке універсально розпізнається машинами.

У цьому уроці ми розглянемо, що таке ASCII, як він працює і як методи JavaScript, такі як `charCodeAt()` та `fromCharCode()`, пов’язані з кодуванням символів. Хоча рядки JavaScript внутрішньо використовують Unicode (UTF-16), значення ASCII збігаються з першими 128 символами Unicode, тому приклади на основі ASCII працюють у JavaScript.

ASCII — це система кодування символів, таких як літери, цифри та символи, у числові значення. Кожен символ зіставляється з певним числом.

Наприклад, велика літера `A` представлена числом `65` в ASCII, тоді як мала `a` — `97`. Це кодування дозволяє комп’ютерам зберігати та обробляти текст.

Стандарт ASCII охоплює 128 символів, включно з:

- Великими та малими літерами англійського алфавіту (A-Z, a-z).
- Цифрами (0-9).  
- Загальними знаками пунктуації та символами (!, @, # тощо).
- Керуючими символами (наприклад, новий рядок і табуляція).  

У JavaScript ви можете отримати числовий код символу за допомогою методу `charCodeAt()`. Цей метод повертає кодову одиницю UTF-16 символу за вказаним індексом. Для перших 128 символів це значення збігається з кодом ASCII.

Давайте подивимося на приклад:

:::interactive_editor

```js
let letter = "A";
console.log(letter.charCodeAt(0));  // 65
```

:::

У цьому прикладі `A` — це перший символ рядка, а виклик `charCodeAt(0)` повертає його числовий код (який збігається з ASCII для базових латинських символів), `65`.

Ви також можете використати цей метод з іншими символами, щоб знайти їх числові коди:

:::interactive_editor

```js
let symbol = "!";
console.log(symbol.charCodeAt(0));  // 33
```

:::

Тут числовий код знака оклику `!` повертається як `33` (що збігається з його значенням ASCII).

Хоча `charCodeAt()` допомагає отримати числовий код символу, метод `fromCharCode()` дозволяє зробити протилежне: перетворити кодову одиницю UTF-16 (яка збігається з ASCII для базових символів) у відповідний символ.

Погляньмо на це в дії:

:::interactive_editor

```js
let char = String.fromCharCode(65);
console.log(char);  //  A
```

:::

У цьому прикладі `fromCharCode(65)` перетворює числовий код `65` (який збігається з ASCII для `A`) назад у символ `A`.

Ще один приклад — перетворення числа `97` у відповідну малу літеру:

:::interactive_editor

```js
let char = String.fromCharCode(97);
console.log(char);  // a
```

:::

Ці методи особливо корисні, коли потрібно маніпулювати або порівнювати символи на основі їх числових кодів.

Наприклад, ви можете використати `charCodeAt()`, щоб перевірити, чи символ є великим, малим або цифрою, порівнюючи його значення ASCII.

З іншого боку, `fromCharCode()` можна використати для динамічного створення символів із їх ASCII кодів.
