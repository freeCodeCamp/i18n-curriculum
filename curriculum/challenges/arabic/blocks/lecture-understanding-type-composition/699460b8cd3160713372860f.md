# --description--

أحيانًا ستواجه مواقف تعرف فيها عن نوع ما أكثر مما يعرفه TypeScript. هنا تأتي فائدة تحقق الأنواع.

إليك مثالًا يستخدم الطريقة `querySelector()` للوصول إلى عنصر له `id` يساوي `submit`:

```ts
const submitBtn = document.querySelector("#submit");
```

الآن، يعرف TypeScript فقط أن هذا `submitBtn` هو نوع من `Element` أو ربما `null`. TypeScript لن ينظر إلى HTML المقابل ليعرف ذلك.

هذا مكان مناسب لاستخدام تحقق النوع لإخبار TypeScript بنوع العنصر هذا. إليك مثالًا:

```ts
const submitBtn = document.querySelector("#submit") as HTMLButtonElement;
```

الآن، سيعامل TypeScript هذا كعنصر `button` بسبب تحقق النوع هذا.

هناك حدود لاستخدام الكلمة المفتاحية `as`. على سبيل المثال، لا يمكنك فعل أشياء كهذه دون أن يرمي TypeScript خطأ:

```ts
// Conversion of type 'string' to type 'number' may be a mistake because neither type sufficiently overlaps with the other. If this was intentional, convert the expression to 'unknown' first.
const age = "awesome" as number;
```

في هذا المثال، لا معنى كبير لمحاولة التحقق من أن السلسلة النصية `"awesome"` يجب أن تُعتبر نوع رقم. يسمح TypeScript فقط بتحقق الأنواع بين الأنواع المتوافقة أو المتداخلة.

طريقة لإزالة هذا الخطأ هي استخدام تحقق مزدوج مثل هذا:

```ts
const age = "awesome" as unknown as number;
```

رغم أن هذا يعمل في TypeScript، إلا أنه لا يُنصح به لأنه منطقيًا لا معنى له كثيرًا.

طريقة أخرى لكتابة تحقق الأنواع هي استخدام صيغة الأقواس الزاوية مثل هذا:

```ts
const submitBtn = <HTMLButtonElement>document.querySelector("#submit");
```

يجب أن تكون حذرًا مع هذه الصيغة عند استخدام ملفات `tsx` لأن TypeScript قد يظن أنك تحاول عرض مكوِّن ويرمي خطأ مثل هذا:

```ts
// file: index.tsx
// This JSX tag requires 'React' to be in scope, but it could not be found.
// JSX element 'HTMLButtonElement' has no corresponding closing tag.
const submitBtn =  <HTMLButtonElement>document.querySelector("#submit");
```

في معظم الحالات، سترى استخدام الصيغة الأولى مع الكلمة المفتاحية `as` أكثر من الخيار الثاني.
