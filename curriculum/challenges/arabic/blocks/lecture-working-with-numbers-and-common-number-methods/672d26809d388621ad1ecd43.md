# --interactive--

في JavaScript، ترمز `NaN` إلى "ليس رقمًا". إنها قيمة خاصة تمثل نتيجة عددية غير قابلة للتمثيل أو غير معرفة. `NaN` هي خاصية من خصائص الكائن العالمي، وتُعتبر أيضًا نوعًا من الأعداد في JavaScript، وهو ما قد يبدو غير بديهي في البداية.

عادةً ما تكون `NaN` نتيجة عمليات من المفترض أن تُرجع رقمًا لكنها لا تستطيع إنتاج قيمة عددية ذات معنى. على سبيل المثال:

:::interactive_editor

```js
let result = 0 / 0;
console.log(result); // NaN
```

:::

في هذه الحالة، قسمة الصفر على الصفر غير معرفة رياضيًا، لذا تُرجع JavaScript القيمة `NaN`. إحدى الخصائص الغريبة لـ `NaN` هي أنه لا يساوي أي شيء، بما في ذلك نفسه:

:::interactive_editor

```js
console.log(NaN === NaN); // false
```

:::

هذا السلوك الفريد يجعل من الصعب التحقق مما إذا كانت القيمة `NaN` باستخدام عوامل المقارنة العادية. لمعالجة هذا، توفر JavaScript الدالة `isNaN()`. تُستخدم دالة `isNaN()` لتحديد ما إذا كانت القيمة `NaN` أم لا. مع ذلك، من المهم فهم كيفية عمل `isNaN()` لأنها قد تُنتج أحيانًا نتائج غير متوقعة. إليك كيف تتصرف `isNaN()`:

:::interactive_editor

```js
console.log(isNaN(NaN));       // true
console.log(isNaN(undefined)); // true
console.log(isNaN({}));        // true

console.log(isNaN(true));      // false
console.log(isNaN(null));      // false
console.log(isNaN(37));        // false

console.log(isNaN("37"));      // false: "37" is converted to 37
console.log(isNaN("37.37"));   // false: "37.37" is converted to 37.37
console.log(isNaN(""));        // false: empty string is converted to 0
console.log(isNaN(" "));       // false: string with a space is converted to 0

console.log(isNaN("blabla"));  // true: "blabla" is not a number
```

:::

كما ترى، تحاول `isNaN()` أولًا تحويل المعلمة إلى رقم. إذا لم تستطع التحويل، تُرجع `true`. هذا السلوك قد يؤدي إلى نتائج مفاجئة، خاصة عند التعامل مع سلاسل نصية يمكن إجبارها على أن تصبح أرقامًا.

نظرًا لهذه التناقضات المحتملة، قدمت ES6 (الإصدار السادس من JavaScript، الصادر عام 2015) الدالة `Number.isNaN()`. هذه الطريقة لا تحاول تحويل المعلمة إلى رقم قبل الاختبار. تُرجع فقط `true` إذا كانت القيمة هي بالضبط `NaN`:

:::interactive_editor

```js
console.log(Number.isNaN(NaN));        // true
console.log(Number.isNaN(Number.NaN)); // true
console.log(Number.isNaN(0 / 0));      // true

console.log(Number.isNaN("NaN"));      // false
console.log(Number.isNaN(undefined));  // false
console.log(Number.isNaN({}));         // false
console.log(Number.isNaN("blabla"));   // false
```

:::

تُوفر `Number.isNaN()` طريقة أكثر موثوقية للتحقق من قيم `NaN`، خاصة في الحالات التي قد يؤدي فيها إجبار النوع إلى نتائج غير متوقعة مع دالة `isNaN()` العالمية. عمليًا، عند التعامل مع العمليات العددية أو مُدخلات المستخدم التي يجب أن تكون أرقامًا، غالبًا ما يكون من الضروري التحقق من `NaN` للتعامل مع الأخطاء أو المُدخلات غير المتوقعة بسلاسة. على سبيل المثال:

:::interactive_editor

```js
let a = 0;
let b = 0;
let result = a / b;

if (Number.isNaN(result)) {
  result = "Error: Division resulted in NaN";
}

console.log(result); // "Error: Division resulted in NaN"
```

:::

في هذا المثال، نستخدم `Number.isNaN()` لالتقاط الحالات التي تؤدي فيها عملية القسمة إلى `NaN`، مما يسمح لنا بالتعامل مع هذا السيناريو بشكل مناسب. فهم `NaN` وكيفية التحقق منه بشكل صحيح أمر ضروري لكتابة شفرة JavaScript قوية، خاصة عند التعامل مع العمليات الرياضية أو تحليل مُدخلات المستخدم.
