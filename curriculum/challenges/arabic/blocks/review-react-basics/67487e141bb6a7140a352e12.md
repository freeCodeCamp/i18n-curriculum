## مكتبات وأُطُر جافاسكريبت

- توفر مكتبات وأُطُر جافاسكريبت حلولًا سريعة للمشاكل الشائعة وتسريع التطوير عبر توفير شفرة جاهزة.
- تركز المكتبات عادةً على تقديم حلول لمهام محددة، مثل التلاعب بـ DOM، التعامل مع الأحداث، أو إدارة طلبات AJAX.
- من أمثلة مكتبات جافاسكريبت jQuery و React.
- أما الأُطُر فتوفر هيكلًا أكثر تحديدًا لبناء التطبيقات. غالبًا ما تأتي مع مجموعة قواعد واصطلاحات يجب على المطورين اتباعها.
- من أمثلة الأُطُر Angular و Next.js، وهو إطار عمل ميتا لـ React.
- **التطبيقات ذات الصفحة الواحدة** (SPAs) هي تطبيقات ويب تحمل صفحة HTML واحدة وتُحدّث تلك الصفحة ديناميكيًا أثناء تفاعل المستخدم مع التطبيق دون إعادة تحميل الصفحة كاملة.
- تستخدم SPAs جافاسكريبت لإدارة حالة التطبيق وعرض المحتوى. غالبًا ما يتم ذلك باستخدام أُطُر توفر أدوات رائعة لبناء واجهات مستخدم معقدة.
- بعض المشاكل المتعلقة بـ SPAs تشمل:

- تواجه قارئات الشاشة صعوبة مع المحتوى الذي يُحدّث ديناميكيًا.
  - لا يتغير عنوان URL عند تنقل المستخدم داخل التطبيق، مما يصعب حفظ الصفحة، التراجع، أو مشاركة صفحات محددة.
  - قد يكون وقت التحميل الأولي بطيئًا إذا كان التطبيق كبيرًا لأن جميع الموارد يجب تحميلها مقدمًا.

## React

- React هي مكتبة جافاسكريبت شهيرة لبناء واجهات المستخدم وتطبيقات الويب.
- المفهوم الأساسي في React هو إنشاء مكونات واجهة مستخدم قابلة لإعادة الاستخدام يمكنها التحديث والعرض بشكل مستقل عند تغير البيانات.
- تسمح React للمطورين بوصف كيف يجب أن تبدو واجهة المستخدم بناءً على حالة التطبيق. ثم تقوم React بتحديث وعرض المكونات المناسبة عند تغير البيانات أو الحالة.

## مكونات React

- المكونات هي لبنات بناء تطبيقات React التي تسمح للمطورين بتقسيم واجهات المستخدم المعقدة إلى أجزاء أصغر يمكن إدارتها.
- تُوصف واجهة المستخدم باستخدام JSX، وهو امتداد لصيغة جافاسكريبت يسمح بكتابة شفرة تشبه HTML داخل جافاسكريبت.
- المكونات هي في الأساس دوال أو أصناف JS تُرجع جزءًا من واجهة المستخدم.

إليك مثال على مكون React بسيط يعرض رسالة ترحيب:

```jsx
function Greeting() {
  const name = 'Anna';
  return <h1>Welcome, {name}!</h1>;
}
```

لاستخدام المكون، يمكنك ببساطة استدعاؤه:
    
```jsx
  <Greeting />
```

## استيراد وتصدير مكونات React

- التصدير يجعل المكون متاحًا لملفات أخرى. الاستيراد يسمح لملف باستخدام مكون تم تصديره من مكان آخر.
- هناك نوعان من التصدير في React: التصدير الافتراضي والتصدير المسمي.

**التصدير الافتراضي**

- يمكن أن يحتوي الملف على تصدير افتراضي واحد فقط. هذا مثالي للملفات التي تحتوي بشكل رئيسي على مكون واحد.
- يمكنك تصدير مكون كتصدير افتراضي هكذا:

```jsx
// City.js
function City() {
  return <p>New York</p>;
}

export default City;
```

- يمكنك أيضًا التصدير في نفس سطر تعريف المكون:

```jsx
export default function City() {
  return <p>New York</p>;
}
```

- لاستيراد تصدير افتراضي، استخدم الكلمة المفتاحية `import` بدون أقواس متعرجة:

```jsx
// App.js
import City from './City';

function App() {
  return (
    <div>
      <h1>My favorite city is:</h1>
      <City />
    </div>
  );
}
```

**التصدير المسمي**

- يسمح التصدير المسمي للملف بمشاركة عدة مكونات. على عكس التصدير الافتراضي، يجب استيرادها باستخدام الاسم الدقيق الذي صدرت به.
- يمكنك تصدير عدة مكونات من نفس الملف هكذا:

```jsx
// Animals.jsx
export function Cat() {
  return <h2>Mr. Whiskers</h2>;
}

export function Dog() {
  return <h2>Fido</h2>;
}
```

- لاستيراد التصديرات المسماة، استخدم الأقواس المتعرجة مع الأسماء المصدرة بالضبط:

```jsx
import { Cat, Dog } from './Animals';
```

- يمكنك إعادة تسمية التصدير المسمي أثناء الاستيراد باستخدام الكلمة المفتاحية `as`:

```jsx
import { Cat as Kitty } from './Animals';
```

**التصدير المختلط بين الافتراضي والمسمي**

- يمكن أن يحتوي الملف على تصدير افتراضي واحد وعدة تصديرات مسماة في نفس الوقت:

```jsx
// Animals.jsx
export default function Cat() {
  return <h2>Mr. Whiskers</h2>;
}

export function Dog() {
  return <h2>Fido</h2>;
}
```

- عند استيراد التصديرات المختلطة، يأتي التصدير الافتراضي أولًا (بدون أقواس)، يليه التصديرات المسماة (داخل الأقواس):

```jsx
import Cat, { Dog } from './Animals';
```

## إعداد مشروع React باستخدام Vite

- توفر أدوات إعداد المشاريع وواجهات سطر الأوامر طريقة سريعة وسهلة لبدء مشاريع جديدة، مما يسمح للمطورين بالتركيز على كتابة الشفرة بدلًا من التعامل مع الإعداد.
- Vite هي أداة إعداد مشاريع شهيرة ويمكن استخدامها مع React.
- لإنشاء مشروع جديد باستخدام Vite، يمكنك استخدام الأمر التالي في الطرفية:

```bash
npm create vite@latest my-react-app -- --template react
```

ينشئ هذا الأمر مشروع React جديد باسم `my-react-app` باستخدام قالب React الخاص بـ Vite. في مجلد المشروع، سترى ملف `package.json` يحتوي على تبعيات المشروع والأوامر المدرجة فيه.

- لتشغيل المشروع، انتقل إلى مجلد المشروع وشغّل الأوامر التالية:

```bash
cd my-react-app # path to the project directory
npm install # installs the dependencies listed in the package.json file
```

- بعد تثبيت التبعيات، يجب أن تلاحظ مجلدًا جديدًا في مشروعك يسمى `node_modules`.

- مجلد `node_modules` هو المكان الذي تُخزن فيه جميع الحزم والمكتبات التي يحتاجها مشروعك.

- لتشغيل مشروعك، استخدم الأمر التالي:

```bash
npm run dev
```

- بعد ذلك، افتح متصفحك وانتقل إلى `http://localhost:5173` لرؤية تطبيق React يعمل.

- لرؤية شفرة قالب البداية فعليًا، يمكنك الدخول إلى مشروعك داخل مجلد `src` وستجد ملف `App.jsx`.

## تمرير الخصائص (props) في مكونات React

- في React، الخصائص (props) هي طريقة لتمرير البيانات من مكون أب إلى مكون ابن. هذه الآلية ضرورية لإنشاء عناصر واجهة مستخدم قابلة لإعادة الاستخدام وديناميكية.
- يمكن أن تكون الخصائص أي قيمة جافاسكريبت. لتمرير الخصائص من الأب إلى الابن، تضيف الخصائص كسمات عند استخدام المكون الابن في JSX الخاص بالأب. إليك مثالًا بسيطًا:

```jsx
// Parent component
function Parent() {
  const name = 'Anna';
  return <Child name={name} />;
}

// Child component
function Child(props) {
  return <h1>Hello, {props.name}!</h1>;
}
```

يمكنك تمرير عدة خصائص باستخدام معامل النشر `(...)`، بعد تحويلها إلى كائن. إليك مثالًا:
    
```jsx
// Parent component
function Parent() {
  const person = {
    name: 'Anna',
    age: 25,
    city: 'New York'
  };
  return <Child {...person} />;
}
```

في هذا الكود، يقوم معامل النشر `{...person}` بتحويل كائن person إلى خصائص فردية تُمرر إلى مكون Child.

## العرض الشرطي في React

- يسمح العرض الشرطي في React بإنشاء واجهات مستخدم ديناميكية. يُستخدم لعرض محتوى مختلف بناءً على شروط أو حالات معينة داخل التطبيق.
- هناك عدة طرق لعرض المحتوى بشكل شرطي في React. إحدى الطرق الشائعة هي استخدام المعامل الثلاثي. إليك مثالًا:

```jsx
function Greeting({ isLoggedIn }) {
  return (
    <div>
      {isLoggedIn ? <h1>Welcome back!</h1> : <h1>Please log in</h1>}
    </div>
  );
}
```

- طريقة أخرى للعرض الشرطي هي استخدام معامل AND المنطقي (`&&`). هذا مفيد عندما تريد عرض المحتوى فقط إذا تحقق شرط معين. إليك مثالًا:

```jsx
function Greeting({ user }) {
  return (
    <div>
      {user && <h1>Welcome, {user.name}!</h1>}
    </div>
  );
}
```

في الكود أعلاه، يتم عرض عنصر `h1` فقط إذا كان كائن المستخدم صحيحًا.

يمكنك أيضًا استخدام جملة `if` مباشرة بهذه الطريقة:

```jsx
function Greeting({ isLoggedIn }) {
  if (isLoggedIn) {
    return <h1>Welcome back!</h1>;
  }
  return <h1>Please sign in</h1>;
}
```

## عرض القوائم في React

- عرض القوائم في React مهمة شائعة عند بناء واجهات المستخدم.
- يمكن عرض القوائم باستخدام دالة `map()` لمصفوفة JS للتكرار على عناصر المصفوفة وإرجاع مصفوفة جديدة من عناصر JSX.
- على سبيل المثال، إذا كان لديك مصفوفة أسماء تريد عرضها كقائمة، يمكنك القيام بما يلي:

```jsx
function NameList({ names }) {
  return (
    <ul>
      {names.map((name, index) => (
        <li key={`${name}-${index}`}>{name}</li>
      ))}
    </ul>
  );
}
```

- تذكر دائمًا توفير مفتاح فريد لكل عنصر في القائمة لمساعدة React في إدارة التحديثات وأدوار العرض. باستخدام هذه التقنيات، يمكنك إنشاء قوائم مرنة وفعالة وديناميكية في تطبيقات React.

## الأنماط المضمّنة في React

- تسمح الأنماط المضمّنة في React بتطبيق أنماط CSS مباشرة على عناصر JSX باستخدام كائنات جافاسكريبت.
- لتطبيق الأنماط المضمّنة في React، يمكنك استخدام الخاصية style على عناصر JSX. تأخذ الخاصية style كائنًا حيث المفاتيح هي خصائص CSS بصيغة تنسيق سنام الجمل والقيم هي القيم المقابلة. إليك مثالًا:

```jsx
function Greeting() {
  return (
    <h1
      style={{ color: 'blue', fontSize: '24px', backgroundColor: 'lightgray' }}
    >
      Hello, world!
    </h1>
  );
}

export default Greeting;
```

يمكنك أيضًا استخراج الأنماط إلى كائن منفصل والرجوع إليه في الخاصية `style` بهذه الطريقة:

```jsx
function Greeting() {
  const styles = {
    color: 'blue',
    fontSize: '24px',
    backgroundColor: 'lightgray'
  };

  return <h1 style={styles}>Hello, world!</h1>;
}

export default Greeting;
```

- تدعم الأنماط المضمّنة التخصيص الديناميكي عبر السماح بتطبيق الأنماط بشكل شرطي بناءً على الخصائص أو الحالة. إليك مثالًا على كيفية تطبيق الأنماط بشكل شرطي بناءً على خاصية:

```jsx
function Greeting({ isImportant }) {
  const styles = {
    color: isImportant ? 'red' : 'black',
    fontSize: isImportant ? '24px' : '16px'
  };

  return <h1 style={styles}>Hello, world!</h1>;
}

export default Greeting;
```

- في الكود أعلاه، يتم تعيين أنماط `color` و `fontSize` بشكل شرطي بناءً على خاصية `isImportant`.
