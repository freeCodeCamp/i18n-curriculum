## String-Konstruktor und `toString()`-Methode

- **Definition**: Ein String-Objekt wird verwendet, um eine Folge von Zeichen darzustellen. String-Objekte werden mit der `String`-Konstruktorfunktion erstellt, die den primitiven Wert in einem Objekt kapselt.

:::interactive_editor

```js
const greetingObject = new String("Hello, world!");

console.log(typeof greetingObject); // "object"
```

:::

- **`toString()`-Methode**: Diese Methode wandelt einen Wert in seine String-Darstellung um. Sie ist eine Methode, die Sie für Zahlen, Booleans, Arrays und Objekte verwenden können.

:::interactive_editor

```js
const num = 10;
console.log(num.toString()); // "10"

const arr = [1, 2, 3];
console.log(arr.toString()); // "1,2,3"
```

:::

Diese Methode akzeptiert eine optionale Basis, die eine Zahl von 2 bis 36 ist. Diese Basis stellt das Zahlensystem dar, zum Beispiel Basis 2 für binär oder Basis 8 für oktal. Wenn die Basis nicht angegeben wird, ist der Standard Basis 10, also dezimal.


:::interactive_editor

```js
const num = 10;
console.log(num.toString(2)); // "1010"(binary)
```

:::

## Number-Konstruktor

- **Definition**: Der `Number`-Konstruktor wird verwendet, um ein Number-Objekt zu erstellen. Das Number-Objekt enthält einige hilfreiche Eigenschaften und Methoden wie die `isNaN`- und `toFixed`-Methode. Meistens verwenden Sie den `Number`-Konstruktor, um andere Datentypen in den Number-Datentyp zu konvertieren.

:::interactive_editor

```js
const myNum = new Number("34");
console.log(typeof myNum); // "object"

const num = Number('100');
console.log(num); // 100

console.log(typeof num); // number
```

:::

## Bewährte Vorgehensweisen für die Benennung von Variablen und Funktionen

- **camelCasing**: Nach Konvention verwenden JavaScript-Entwickler camel casing zur Benennung von Variablen und Funktionen. Camel casing bedeutet, dass das erste Wort komplett in Kleinbuchstaben geschrieben wird und die folgenden Wörter mit einem Großbuchstaben beginnen. Zum Beispiel `isLoading`.

- **Benennung von Booleans**: Für Boolean-Variablen ist es üblich, Präfixe wie „is“, „has“ oder „can“ zu verwenden.

```js
let isLoading = true;
let hasPermission = false;
let canEdit = true;
```

- **Benennung von Funktionen**: Bei Funktionen sollte der Name klar angeben, was die Funktion tut. Für Funktionen, die einen Boolean zurückgeben (oft Prädikate genannt), können Sie dieselben Präfixe „is“, „has“ oder „can“ verwenden. Wenn Sie Funktionen haben, die Daten abrufen, beginnt der Name oft mit „get“. Wenn Sie Funktionen haben, die Daten setzen, beginnt der Name oft mit „set“. Für Event-Handler-Funktionen können Sie mit „handle“ voranstellen oder mit „Handler“ enden.

```js
function getUserData() { /* ... */ }

function isValidEmail(email) { /* ... */ }

function getProductDetails(productId) { /* ... */ }

function setUserPreferences(preferences) { /* ... */ }

function handleClick() { /* ... */ }
```

- **Benennung von Variablen in Schleifen**: Bei der Benennung von Iterator-Variablen in Schleifen ist es üblich, einzelne Buchstaben wie `i`, `j` oder `k` zu verwenden.

```js
for (let i = 0; i < array.length; i++) { /* ... */ }
```

## Arbeiten mit Sparse Arrays

- **Definition**: Es ist möglich, Arrays mit leeren Slots zu haben. Leere Slots sind Slots, in denen nichts enthalten ist. Das unterscheidet sich von Array-Slots mit dem Wert `undefined`. Diese Arten von Arrays werden als Sparse Arrays bezeichnet.

:::interactive_editor

```js
const sparseArray = [1, , , 4];
console.log(sparseArray.length); // 4
```

:::

## Linters und Formatierer

- **Linters**: Ein Linter ist ein statisches Code-Analyse-Tool, das Programmierfehler, Bugs, stilistische Fehler und verdächtige Konstrukte markiert. Ein Beispiel für einen gängigen Linter ist ESLint.
- **Formatierer**: Formatierer sind Werkzeuge, die Ihren Code automatisch formatieren, um einer bestimmten Stilrichtlinie zu entsprechen. Ein Beispiel für einen gängigen Formatierer ist Prettier.

## Speicherverwaltung

- **Definition**: Speicherverwaltung ist der Prozess der Kontrolle des Speichers, indem er bei Bedarf zugewiesen und freigegeben wird, wenn er nicht mehr benötigt wird. JavaScript verwendet automatische Speicherverwaltung. Das bedeutet, dass JavaScript (genauer gesagt die JavaScript-Engine in Ihrem Webbrowser) die Speicherzuweisung und -freigabe für Sie übernimmt. Sie müssen den Speicher in Ihrem Code nicht explizit freigeben. Dieser automatische Prozess wird oft als „Garbage Collection“ bezeichnet.

## Closures

- **Definition**: Ein Closure ist eine Funktion, die Zugriff auf Variablen in ihrem äußeren (einschließenden) lexikalischen Gültigkeitsbereich hat, selbst nachdem die äußere Funktion zurückgegeben wurde.

:::interactive_editor

```js
function outerFunction(x) {
  let y = 10;
  function innerFunction() {
    console.log(x + y);
  }
  return innerFunction;
}

let closure = outerFunction(5);
closure(); // 15
```

:::

## `var`-Schlüsselwort und Hoisting

- **Definition**: `var` war die ursprüngliche Möglichkeit, Variablen vor 2015 zu deklarieren. Aber es gab einige Probleme mit `var` bezüglich Gültigkeitsbereich, Neudeklaration und mehr. Deshalb verwendet die moderne JavaScript-Programmierung stattdessen `let` und `const`.
- **Neudeklaration von Variablen mit `var`**: Wenn Sie versuchen, eine Variable mit `let` neu zu deklarieren, erhalten Sie einen `SyntaxError`. Mit `var` ist die Neudeklaration einer Variable erlaubt.

```js
// Uncaught SyntaxError: Identifier 'num' has already been declared 
let num = 19;
let num = 18;

var myNum = 5;
var myNum = 10; // This is allowed and doesn't throw an error

console.log(myNum) // 10
```

- **`var` und Gültigkeitsbereich**: Variablen, die mit `var` innerhalb eines Blocks (wie einer `if`-Anweisung oder einer `for`-Schleife) deklariert werden, sind außerhalb dieses Blocks weiterhin zugänglich.

:::interactive_editor

```js
if (true) {
  var num = 5;
}
console.log(num); // 5
```

:::

- **Hoisting**: Dies ist das Standardverhalten von JavaScript, Deklarationen während der Kompilierungsphase vor der Ausführung des Codes an den Anfang ihres jeweiligen Gültigkeitsbereichs zu verschieben. Wenn Sie eine Variable mit dem `var`-Schlüsselwort deklarieren, hebt JavaScript die Deklaration an den Anfang ihres Gültigkeitsbereichs.

:::interactive_editor

```js
console.log(num); // undefined
var num = 5;
console.log(num); // 5
```

:::

Wenn Sie eine Funktion mit der Funktionsdeklarationssyntax deklarieren, werden sowohl der Funktionsname als auch der Funktionskörper gehoben. Das bedeutet, Sie können eine Funktion aufrufen, bevor Sie sie in Ihrem Code deklariert haben.

:::interactive_editor

```js
sayHello(); // "Hello, World!"

function sayHello() {
  console.log("Hello, World!");
}
```

:::

Variablendeklarationen mit `let` oder `const` werden gehoben, aber sie sind nicht initialisiert, und Sie können nicht auf sie zugreifen, bevor die eigentliche Deklaration in Ihrem Code steht. Dieses Verhalten wird oft als „temporäre Todzone“ bezeichnet.

:::interactive_editor

```js
console.log(num); // Throws a ReferenceError
let num = 10;
```

:::

## Arbeiten mit Imports, Exports und Modulen

- **Modul**: Dies ist eine in sich geschlossene Codeeinheit, die verwandte Funktionen, Klassen oder Variablen kapselt. Um ein Modul zu erstellen, schreiben Sie Ihren JavaScript-Code in eine separate Datei.
- **Exports**: Alle Variablen, Funktionen oder Klassen, die Sie in anderen Teilen Ihrer Anwendung verfügbar machen möchten, müssen explizit mit dem `export`-Schlüsselwort exportiert werden. Es gibt zwei Arten von Exporten: named export und default export.
- **Imports**: Um die exportierten Elemente in einem anderen Teil Ihrer Anwendung zu verwenden, müssen Sie sie mit dem `import`-Schlüsselwort importieren. Die Typen können named import, default import und namespace import sein.

```js
// Within a file called math.js, we export the following functions:

// Named export
export function add(num1, num2) {
  return num1 + num2;
}

// Default export
export default function subtract(num1, num2) {
  return num1 - num2;
}

// Within another file, we can import the functions from math.js.

// Named import - This line imports the add function.
// The name of the function must exactly match the one exported from math.js.
import { add } from './math.js';

// Default import - This line imports the subtract function.
// The name of the function can be anything.
import subtractFunc from './math.js';

// Namespace import - This line imports everything from the file.
import * as Math from './math.js';

console.log(add(5, 3)); // 8
console.log(subtractFunc(5, 3)); // 2
console.log(Math.add(5, 3)); // 8
console.log(Math.subtract(5, 3)); // 2
```
