# --description--

Supongamos que tienes dos cadenas que te gustaría combinar para crear un mensaje, como el siguiente:

```js
const str1 = "Hello";
const str2 = "World!";
const message = str1.push(str2);
```

<img src="https://cdn.statically.io/gh/TheOdinProject/curriculum/4ed59981b4ce2c60b5b83bf7415d3127b61821f5/foundations/javascript_basics/understanding_errors/imgs/03.png" style="width:100%" alt="an example of a type error in the Chrome developer console">

Aquí, obtendrás un `TypeError` con un mensaje que indica que `str1.push is not a function`. Este es un mensaje de error común que confunde a los aprendices porque podrías saber que `.push()` ciertamente es una función (por ejemplo, si la has usado para agregar elementos a arrays antes).

Pero esa es la clave: `.push()` no es un método de String, es un método de Array. Por lo tanto, “no es una función” que puedas encontrar como método de String. Si cambias `.push()` por `.concat()`, un método adecuado de String, ¡nuestro código se ejecuta como se espera!

Una buena nota para tener en cuenta cuando te enfrentas a un `TypeError` es considerar el tipo de dato contra el que intentas ejecutar un método u operación. Probablemente descubrirás que no es lo que piensas, o que la operación o método no es compatible con ese tipo.

# --questions-- 

## --text--

¿Por qué el siguiente código JavaScript resulta en un `TypeError`?

```js
const str1 = "Hello";
const str2 = "World!";
const message = str1.push(str2);
```

## --answers--

El método `.push()` solo se puede usar en objetos, no en cadenas.

---

El método `.push()` no está disponible para cadenas porque es un método de Array.

---

El método `.push()` está mal escrito y debería ser .pusch().

---

El `TypeError` es el resultado de un error de sintaxis en el código JavaScript.
