# --description--

HTTPS 代表超文字傳輸安全協定。它是 HTTP 的安全版本。

正如你在之前幾堂課中學到的，HTTP 在你的瀏覽器和網站之間傳輸資料。標準的 HTTP 以純文字形式傳送這些資料，因此任何人都可以攔截這些純文字資料。

因此，HTTPS 使用一種稱為傳輸層安全性（TLS）的密碼協定來保護這種資料傳輸。TLS 加密兩點之間的通訊通道。舊系統使用安全插口層（SSL），許多網頁主機平台至今仍在使用。

加密會將資料混淆成無法閱讀的程式碼，只有使用者的瀏覽器和網頁伺服器持有解密的金鑰。

例如，當你在網站輸入密碼時，HTTPS 會立即將其混淆，並保護信用卡資訊、個人地址、登入憑證和其他敏感資訊。攔截資料的駭客只會看到隨機字元，因此無法使用被竊取的資訊。

這種保護在公共 Wi-Fi 網路上至關重要。咖啡廳、機場和其他公共場所的開放 Wi-Fi 網路缺乏基本安全性，讓網路犯罪分子能監控這些網路上的未加密流量。HTTPS 在公共網路中建立一條私人通道，保護你的瀏覽紀錄免受附近壞人的窺視。

HTTPS 也提供身份驗證，證明網站的合法性。要使用 HTTPS，網站擁有者必須取得由受信任組織頒發的數位憑證。該憑證驗證網站擁有者的真實身份。

當你造訪一個網址只以「http」開頭的網站時，這是判斷該網站擁有者未取得憑證、網站不安全的一種方式。現代瀏覽器也會對這類網站發出警告。

最後，資料完整性是 HTTPS 的另一個重要利益。這表示沒有人能在資料傳輸過程中更改你的資料。駭客無法在網站檔案傳送到你的裝置時注入惡意程式碼。你收到的正是網站擁有者所傳送的資料。

像 Google 這樣的搜尋引擎會優先顯示 HTTPS 網站，因為安全網站在搜尋結果中排名較高。現代瀏覽器也要求 HTTPS 以使用進階功能，例如地理定位、推播通知和網路攝影機。
