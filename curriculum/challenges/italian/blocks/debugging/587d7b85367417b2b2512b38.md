# --description--

I programmi con ramificazioni, cioè quelli che fanno cose diverse se certe condizioni sono soddisfatte, si basano su istruzioni `if`, `else if` e `else` in JavaScript. La condizione a volte assume la forma di un test per verificare se un risultato è uguale a un valore.

Questa logica si esprime (almeno in inglese) come "se x è uguale a y, allora ..." che può tradursi letteralmente in codice usando l'operatore `=`, cioè l'operatore di assegnazione. Questo porta a un flusso di controllo inatteso nel tuo programma.

Come visto nelle sfide precedenti, l'operatore di assegnazione (`=`) in JavaScript assegna un valore a un nome di variabile. E gli operatori `==` e `===` verificano l'uguaglianza (il triplo `===` testa l'uguaglianza stretta, cioè sia valore che tipo sono uguali).

Il codice qui sotto assegna `x` a 2, che viene valutato come `true`. Quasi tutti i valori da soli in JavaScript vengono valutati come `true`, tranne quelli noti come valori "falsy": `false`, `0`, `""` (una stringa vuota), `NaN`, `undefined` e `null`.

```js
let x = 1;
let y = 2;
if (x = y) {

} else {

}
```

In questo esempio, il blocco di codice all'interno dell'istruzione `if` verrà eseguito per qualsiasi valore di `y`, a meno che `y` non sia falsy. Il blocco `else`, che ci aspettiamo venga eseguito qui, in realtà non verrà eseguito.

# --instructions--

Correggi la condizione in modo che il programma esegua il ramo giusto e che il valore appropriato venga assegnato a `result`.

# --hints--

Il tuo codice dovrebbe correggere la condizione in modo che verifichi l'uguaglianza, invece di usare l'assegnazione.

```js
assert(result == 'Not equal!');
```

La condizione dovrebbe usare `==` o `===` per testare l'uguaglianza.

```js
assert(__helpers.removeJSComments(code).match(/x\s*?===?\s*?y/g));
```
