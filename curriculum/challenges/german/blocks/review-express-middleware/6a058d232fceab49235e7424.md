## Was ist Middleware?

- **Middleware** ist eine Funktion, die während des Lebenszyklus einer Anfrage an den Server ausgeführt wird.
- Middleware hat Zugriff auf das **Request-Objekt** (`req`), das **Response-Objekt** (`res`) und die **`next`-Funktion**.
- Häufige Anwendungsfälle: Protokollierung von Anfragen, Fehlerbehandlung, Parsen eingehender Daten und Bereitstellung statischer Dateien.
- Middleware wird in der Reihenfolge ausgeführt, in der sie der Anwendung hinzugefügt wird.

### Wie Middleware funktioniert

1. Eine Anfrage wird an den Server gesendet.
2. Middleware-Funktionen verarbeiten die Anfrage.
3. Wenn Middleware `next()` aufruft, wird die Kontrolle an die nächste Middleware oder den nächsten Route-Handler übergeben.
4. Wenn Middleware `next()` nicht aufruft, stoppt der Anfrage-Antwort-Zyklus.

```javascript
app.use((req, res, next) => {
  console.log('Request URL:', req.url);
  console.log('Request Method:', req.method);
  next();
});
```

- `app.use()` fügt Middleware global zur Anwendung hinzu.

## Middleware auf Anwendungsebene

- **Middleware auf Anwendungsebene** ist an eine Instanz der Express-App gebunden.
- Registriert mit `app.use()` oder routenspezifischen Methoden wie `app.get()`, `app.post()` usw.
- Wird für jede eingehende Anfrage ausgeführt, sofern sie nicht auf eine bestimmte Route beschränkt ist.
- Anwendungsfälle: Protokollierung, Authentifizierung und Fehlerbehandlung über mehrere Routen hinweg.

```javascript
app.use((req, res, next) => {
  console.log('Request received at:', new Date());
  next();
});
```

## Middleware auf Router-Ebene

- **Middleware auf Router-Ebene** ist an eine Instanz von `express.Router()` gebunden, nicht an die Haupt-App.
- Wird mit `router.use()` angewendet und betrifft nur Routen, die von diesem Router behandelt werden.
- Hilft dabei, die Anwendung in isolierte Routemodule zu modularisieren.
- Ein Router wird mit `app.use('/path', router)` eingebunden, um die Middleware auf bestimmte Routen zu beschränken.

```javascript
const router = express.Router();

router.use((req, res, next) => {
  console.log('Request made to /menu route');
  next();
});

router.get('/drinks', (req, res) => {
  res.send('Welcome to the drinks menu!');
});

app.use('/menu', router);
```

## Fehlerbehandlungs-Middleware

- **Fehlerbehandlungs-Middleware** ist eine spezielle Middleware, die Fehler während des Anfrage-Antwort-Zyklus abfängt.
- Sie hat **vier Parameter**: `err`, `req`, `res` und `next`. Die Signatur mit vier Parametern zeigt Express, dass es sich um einen Fehler-Handler handelt.
- Wird aufgerufen, wenn ein Fehler an `next(err)` übergeben wird.
- Sollte **am Ende** des Middleware-Stacks stehen, nach allen anderen Middleware-Funktionen und Routen.

```javascript
app.use((err, req, res, next) => {
  console.error(err.stack);
  res.status(500).send('Something went wrong!');
});
```

## Eingebaute und Drittanbieter-Middleware

### Eingebaute Middleware

- **`express.json()`**: parst eingehende Anfragen mit JSON-Nutzlasten.

```javascript
app.use(express.json());
```

- **`express.static()`**: stellt statische Dateien wie Bilder, CSS und JavaScript bereit.

```javascript
app.use(express.static('public'));
```

- **`express.urlencoded()`**: parst eingehende Anfragen mit URL-kodierten Nutzlasten (das Format, das von HTML-Formularen gesendet wird); macht die geparsten Daten auf `req.body` verfügbar.

```javascript
app.use(express.urlencoded({ extended: true }));
```

### Drittanbieter-Middleware

- **`cors`**: ermöglicht Cross-Origin Resource Sharing.

```javascript
app.use(cors());
```

- **`morgan`**: protokolliert HTTP-Anfragen.

```javascript
app.use(morgan('tiny'));
```

- Sowohl eingebaute als auch Drittanbieter-Middleware-Funktionen werden in der Reihenfolge hinzugefügt, in der sie mit `app.use()` aufgerufen werden.
