# --explanation--

`Not exactly`는 누군가의 생각이나 추측이 거의 맞지만 완전히 정확하지 않다는 점을 정중하게 표현하는 방법입니다. 더 명확하거나 정확한 설명을 하기 전에 자주 사용합니다. 예를 들어:

- **학생:** `Is encapsulation just about hiding code?`

- **멘토:** `Not exactly, it's also about organizing and protecting it.` - 이 예시는 `not exactly`가 어떻게 부드럽게 정정하면서 더 자세한 설명을 덧붙이는지 보여줍니다.

`Inheritance`는 객체지향 프로그래밍의 핵심 개념입니다. 한 클래스(자식 또는 하위 클래스)가 다른 클래스(부모 또는 상위 클래스)의 프로퍼티와 메서드를 물려받을 수 있게 합니다. 이는 코드 재사용을 지원하여 같은 로직을 여러 곳에 다시 작성할 필요가 없게 하고, 기존 기능을 확장하거나 기반으로 삼기 쉽게 만듭니다. 예를 들어:

`Inheritance allows the "Employee" class to reuse code from the "Person" class, like name and email methods.` - 이 예시는 상속이 어떻게 한 클래스가 다른 클래스의 프로퍼티와 동작을 얻어 코드 중복을 피하는지 보여줍니다.
