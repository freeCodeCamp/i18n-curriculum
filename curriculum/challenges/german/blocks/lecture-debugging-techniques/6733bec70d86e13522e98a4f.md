# --description--

Die `throw`-Anweisung in JavaScript wird verwendet, um eine benutzerdefinierte Ausnahme zu werfen. Eine Ausnahme in der Programmierung ist ein unerwartetes Ereignis, das den normalen Ablauf des Programms unterbricht.

Als Programmierende ist es wichtig, diese Ausnahmen zu behandeln, damit Ihre Programme nicht unerwartet abstürzen, wenn Fehler auftreten. Hier ist die grundlegende Syntax für die `throw`-Anweisung:

```js
throw expression;
```

Der `expression` ist in diesem Fall das Objekt oder der Wert, der die Ausnahme darstellt, die Sie werfen möchten. Beispiele dafür sind die eingebauten Ausnahme-Klassen wie die `Error`-, `TypeError`- oder `RangeError`-Klasse. Hier ist ein Beispiel für die Verwendung der `throw`-Anweisung, um einen `TypeError` zu werfen:

```js
function validateNumber(input) {
  if (typeof input !== "number") {
    throw new TypeError("Expected a number, but received " + typeof input);
  }
  return input * 2;
}
```

In diesem Beispiel prüfen wir, ob der Typ von `input` nicht vom Typ `number` ist. Wenn nicht, werfen wir eine `TypeError` mit einer benutzerdefinierten Nachricht. Andernfalls gibt die Funktion das Ergebnis der Multiplikation von `input` mit `2` zurück.

Wenn Sie eine allgemeinere Fehlermeldung werfen möchten, können Sie den `Error`-Konstruktor so referenzieren:

```js
function divide(numerator, denominator) {
  if (denominator === 0) {
    throw new Error("Cannot divide by zero");
  }
  return numerator / denominator;
}
```

Hier ist ein Beispiel für eine Funktion, die überprüft, ob der Nenner `0` ist. Wenn das der Fall ist, wird eine benutzerdefinierte Fehlermeldung geworfen, die `Cannot divide by zero` sagt.

In der nächsten Lektion sehen wir uns an, wie man Fehlermeldungen im Kontext des `try`/`catch`-Blocks wirft, der verwendet wird, um Ausnahmen in JavaScript elegant zu behandeln.
