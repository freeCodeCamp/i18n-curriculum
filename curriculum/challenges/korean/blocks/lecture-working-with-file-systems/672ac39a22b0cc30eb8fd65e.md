# --description--

웹 애플리케이션을 구축할 때 파일과 폴더의 조직 방식을 생각하는 것이 중요합니다. 이렇게 하면 코드가 정리되고 유지 관리가 더 쉬워집니다.

HTML과 CSS 프로젝트의 예시 폴더 구조를 살펴보겠습니다:

```bash
.
├── /assets
│   ├── /images
│   │   ├── logo.png
│   │   ├── banner.jpg
│   │   └── icons.svg
│   ├── /fonts
│   │   ├── custom-font.woff
│   │   └── custom-font.woff2
├── /css
│   ├── main.css
│   ├── about.css
│   └── contact.css
├── index.html
├── about.html
├── contact.html
└── README.md
```

예시 상단에는 루트 디렉터리를 나타내는 단일 점이 있습니다.

루트 디렉터리는 파일 시스템에서 최상위 디렉터리로, 모든 파일 경로의 시작점이며 다른 모든 디렉터리와 파일을 포함합니다. 디렉터리는 폴더의 다른 이름입니다.

루트 디렉터리 안에는 `assets` 디렉터리가 처음으로 표시됩니다. `assets` 용어는 이미지, 폰트 또는 기타 리소스와 같이 프로젝트에서 사용되는 모든 파일을 가리킵니다.

이 예시에서는 `assets` 디렉터리 안에 `images`과 `fonts`라는 두 개의 디렉터리가 있습니다.

다음 디렉터리는 `css` 디렉터리입니다. 때때로 스타일시트 디렉터리라고도 합니다. 이곳에 모든 CSS 파일을 저장합니다.

`css` 디렉터리 아래에는 HTML 파일들이 있습니다. `index.html` 파일은 홈페이지를 나타내고, `about.html`과 `contact.html`는 각각 About 페이지와 Contact 페이지를 나타냅니다.

마지막으로 `README.md` 파일이 있습니다. `README` 파일은 프로젝트가 무엇인지, 어떻게 사용하는지, 기타 관련 세부 정보를 제공하는 데 자주 사용됩니다.

`.md` 확장자는 마크다운을 의미하며, 문서 작성을 위해 자주 사용되는 가벼운 마크업 언어입니다.

프로젝트를 assets, CSS, HTML 파일로 조직하면 원하는 것을 쉽게 찾을 수 있습니다. 또한 다른 개발자가 프로젝트에 기여하기도 쉽습니다.

이것은 파일과 폴더를 조직하는 한 가지 예시일 뿐이며, 프로젝트를 구조화하는 다양한 방법이 있다는 점을 기억하세요.
