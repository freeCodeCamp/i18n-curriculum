# --interactive--

Un'istruzione `break` serve per uscire anticipatamente da un loop, mentre un'istruzione `continue` serve per saltare l'iterazione corrente di un loop e passare a quella successiva.

Ecco un esempio di utilizzo di un'istruzione `break` in un loop `for`:

:::interactive_editor

```js
for (let i = 0; i < 10; i++) {
  if (i === 5) {
    break;
  }
  console.log(i);
}
```

:::

Nell'esempio sopra, il loop inizia a contare da `0` e continua a eseguire finché `i` è minore di `10`.

All'interno del loop, controlliamo se `i` è uguale a `5`. Se lo è, usiamo l'istruzione `break` per uscire anticipatamente dal loop. Altrimenti, stampiamo il valore di `i` nella console. Quindi l'output del codice stamperà i numeri `0`, `1`, `2`, `3` e `4`.

L'istruzione `break` è utile quando vuoi uscire anticipatamente da un loop in base a una certa condizione. Per esempio, se stai cercando un valore specifico in un array, puoi usare un'istruzione `break` per uscire dal loop appena trovi il valore.

A volte potresti voler saltare un'iterazione particolare di un loop senza uscire completamente dal loop. Qui entra in gioco l'istruzione `continue`. Ecco un esempio di utilizzo di un'istruzione `continue` in un loop `for`:

:::interactive_editor

```js
for (let i = 0; i < 10; i++) {
  if (i === 5) {
    continue;
  }
  console.log(i);
}
```

:::

Come prima, abbiamo inizializzato `i` a `0` e abbiamo una condizione che fa eseguire il loop finché `i` è minore di `10`.

All'interno del loop, quando `i` è uguale a `5`, usiamo l'istruzione `continue` per saltare l'iterazione corrente e passare a quella successiva.

L'output di questo codice stamperà i numeri `0`, `1`, `2`, `3`, `4`, `6`, `7`, `8` e `9`. Il numero `5` viene saltato a causa dell'istruzione `continue`.

Un'altra cosa che puoi fare con entrambe le istruzioni `break` e `continue` è usare le etichette per specificare quale loop vuoi interrompere o continuare.

Questo è utile quando hai loop annidati e vuoi controllare il flusso del loop esterno dall'interno del loop interno.

Ecco un esempio di utilizzo delle etichette con l'istruzione `break`:

:::interactive_editor

```js
outerLoop: for (let i = 0; i < 3; i++) {
  innerLoop: for (let j = 0; j < 3; j++) {
    if (i === 1 && j === 1) {
      break outerLoop;
    }
    console.log(`i: ${i}, j: ${j}`);
  }
}
```

:::

In questo esempio, abbiamo un loop esterno `for` etichettato `outerLoop` e un loop interno `for` etichettato `innerLoop`.

Quando `i` è uguale a `1` e `j` è uguale a `1`, usiamo l'istruzione `break` con l'etichetta `outerLoop` per uscire anticipatamente dal loop esterno. Questo farà uscire sia dal loop interno sia da quello esterno.

L'output di questo codice stamperà quanto segue nella console:

```md
"i: 0, j: 0"
"i: 0, j: 1"
"i: 0, j: 2"
"i: 1, j: 0"
```

La maggior parte delle volte non avrai bisogno di usare le etichette con le istruzioni `break` e `continue`, ma è bene sapere che hai questa opzione se mai ti servisse.
