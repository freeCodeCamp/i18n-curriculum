# --description--

Подача статичних файлів — це поширена потреба у веб-застосунках. Статичні файли включають ресурси, такі як зображення, CSS-файли, JavaScript на стороні клієнта, шрифти та інші ресурси, які не змінюються динамічно.

Express робить подачу цих файлів простою завдяки своєму вбудованому `express.static()` підпрограмному забезпеченню.

У Express подача статичних файлів означає доставку файлів безпосередньо з файлової системи сервера до браузера клієнта без додаткової обробки на сервері.

Наприклад, коли браузер запитує файли, такі як `/styles.css`, `/script.js` або `/images/logo.png`, Express може знайти ці файли у вказаній папці і подати їх напряму. Якщо файл не знайдено, Express автоматично відповідає кодом 404. Оскільки ці файли не потрібно генерувати або змінювати на сервері, їх називають *статичними*.

Це робить подачу ресурсів, таких як стилі та зображення, легкою і усуває необхідність вручну визначати маршрути для кожного файлу.

Ось як ви можете налаштувати подачу статичних файлів у Express:

```javascript
const express = require("express");
const path = require("path");
const app = express();
const port = 3000;

// Serve static files from the "public" directory
app.use(express.static(path.join(__dirname, "public")));

app.get("/", (req, res) => {
  res.send("Welcome to the static file demo!");
});

app.listen(port, () => {
  console.log(`Server running at http://localhost:${port}`);
});
```

Спершу ми імпортуємо `express` для створення нашого сервера та `path` для безпечного формування шляхів до файлів на різних операційних системах.

Далі `app.use(express.static(path.join(__dirname, "public")))` каже Express подавати файли з папки `public`. Підпрограмне забезпечення `express.static()` повідомляє Express подавати файли з папки `public`. Будь-який файл у цій директорії стає доступним для браузера.

Ми також визначаємо простий маршрут `GET` для `/`, який відправляє привітальне повідомлення. Це показує, що подача статичних файлів може співіснувати з динамічними маршрутами.

Нарешті, `app.listen()` запускає сервер на порті 3000. Коли сервер працює, запити на статичні ресурси у папці `public` обробляються автоматично.

Для цього потрібно мати папку з назвою `public`, де зберігатимуться ваші статичні файли. Типова структура проєкту може виглядати так:

```md
my-app/
│
├── app.js
└── public/
    ├── index.html
    ├── styles.css
    └── image.png
```

Коли користувач робить запит, наприклад, відвідуючи `http://localhost:3000/index.html`, Express перевіряє, чи існує запитуваний файл у директорії `public`. Якщо файл існує, Express відправляє його напряму до браузера. Якщо файл не існує, Express переходить до наступного підпрограмного забезпечення або повертає відповідь 404.

Пам’ятайте, що вам не потрібно визначати маршрути для кожного файлу, тому запит до `http://localhost:3000/styles.css` означає, що Express шукає `public/styles.css` і автоматично відповідає ним.

Ви також можете налаштувати, як подавати статичні файли. Наприклад, можна подавати файли з кількох директорій або налаштувати поведінку кешування.

Уявіть, що у вашого застосунку така структура файлів:

```md
my-app/
│
├── app.js
│
├── public/
│   ├── index.html
│   └── styles.css
│
└── images/
    └── image.png
```

Ось як ви можете зареєструвати кілька директорій для подачі статичних файлів:

```javascript
app.use(express.static(path.join(__dirname, "public")));
app.use("/images", express.static(path.join(__dirname, "images")));
```

Тут ми використовуємо `app.use(express.static(path.join(__dirname, "public")));`, як і раніше, що означає, що файли всередині директорії `public` подаються з кореневого шляху, як і раніше.

Але якщо подивитися на наступний рядок, ви побачите, що ми передаємо `"/images"` як перший аргумент `app.use()`, а потім `express.static(path.join(__dirname, "images"))`. У цьому випадку `"/images"` — це шлях монтування, який каже Express використовувати це підпрограмне забезпечення лише тоді, коли він отримує запит до `/images`. Коли це відбувається, `express.static()` запускається і шукає файли всередині директорії `images`.

Отже, з такою конфігурацією запити працюватимуть так:

*   GET `http://localhost:3000/styles.css`: Express шукає у директорії `public` файл `styles.css`, знаходить його і відправляє клієнту.
    
*   GET `http://localhost:3000/images/image.png`: Оскільки цей запит починається з `/images`, Express шукає у директорії `images` файл `image.png`, знаходить його і відправляє клієнту.
    

Подача статичних файлів у Express проста і потужна. Використовуючи `express.static()`, ви легко можете подавати ресурси, такі як зображення, стилі та JavaScript на стороні клієнта.

Зберігання статичних файлів у спеціальній папці, наприклад `public`, або використання кількох директорій і шляхів монтування допомагає тримати ваш застосунок організованим, підтримуваним і легким для масштабування.
