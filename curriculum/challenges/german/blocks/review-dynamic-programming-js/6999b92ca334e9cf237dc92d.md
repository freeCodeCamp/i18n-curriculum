## Einführung in die dynamische Programmierung

- **Definition**: Dynamische Programmierung ist eine algorithmische Technik, die komplexe Probleme löst, indem sie diese in einfachere Teilprobleme zerlegt und die Ergebnisse speichert, um redundante Berechnungen zu vermeiden.  
- **Überlappende Teilprobleme**: Dieselben kleineren Probleme treten mehrfach auf, wenn das größere Problem gelöst wird. Statt diese Teilprobleme wiederholt neu zu berechnen, speichern wir ihre Lösungen.  
- **Optimale Teilstruktur**: Die optimale Lösung des Problems enthält optimale Lösungen seiner Teilprobleme. Das bedeutet, wir können die beste Lösung aufbauen, indem wir die besten Lösungen der kleineren Teile kombinieren.

## Lösungen mit dynamischer Programmierung

- **Memoisierung (Top-Down-Ansatz)**: Memoisierung speichert die Ergebnisse aufwändiger Funktionsaufrufe und gibt das zwischengespeicherte Ergebnis zurück, wenn dieselben Eingaben erneut auftreten.  

```js
function climbStairsMemo(n, memo = {}) {
  // If already calculated, return cached value
  if (memo[n] !== undefined) {
    return memo[n];
  }

  // Base cases: 1 way for 1 step, 2 ways for 2 steps
  if (n <= 2) {
    return n;
  }

  // Calculate once and store in memo for future use
  memo[n] =
    climbStairsMemo(n - 1, memo) +
    climbStairsMemo(n - 2, memo);

  return memo[n];
}
```

- **Tabulation (Bottom-Up-Ansatz)**: Tabulation baut die Lösung von unten nach oben auf, indem eine Tabelle mit Lösungen der Teilprobleme gefüllt wird.

```js
function climbStairsTabulation(n) {
  if (n <= 2) {
    return n;
  }

  // Create an array to store solutions from 0 to n
  const dp = new Array(n + 1).fill(0);

  // Base cases
  dp[1] = 1; // 1 way to reach step 1
  dp[2] = 2; // 2 ways to reach step 2

  // Build up the solution iteratively
  for (let i = 3; i <= n; i++) {
    // Ways to reach step i = ways to reach (i-1) + ways to reach (i-2)
    dp[i] = dp[i - 1] + dp[i - 2];
  }

  return dp[n];
}
```

## Anwendungen der dynamischen Programmierung in der Praxis

- **Routenoptimierung**: GPS-Systeme verwenden Algorithmen der dynamischen Programmierung, um kürzeste Wege zwischen Orten zu finden.  
- **Textverarbeitung**: Rechtschreibprüfungen und Autovervollständigungsfunktionen basieren oft auf dynamischer Programmierung, um Editierabstände zwischen Wörtern zu berechnen.  
- **Finanzmodellierung**: Anlagestrategien und Portfolio-Optimierung nutzen häufig Techniken der dynamischen Programmierung.  
- **Ressourcenzuweisung**: Das Rucksackproblem und seine Varianten treten bei der Terminplanung, Budgetierung und Ressourcenverwaltung auf.

## Wann Sie dynamische Programmierung verwenden sollten

Sie sollten dynamische Programmierung in folgenden Fällen in Betracht ziehen:

- Das Problem lässt sich in überlappende Teilprobleme zerlegen.  
- Das Problem zeigt eine optimale Teilstruktur.  
- Eine naive rekursive Lösung würde wiederholte Berechnungen erfordern.  
- Sie müssen die Zeitkomplexität optimieren und sind bereit, dafür mehr Speicherplatz zu verwenden.
