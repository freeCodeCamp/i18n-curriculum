# --description--

In JavaScript müssen wir unseren Funktionen oft keinen Namen geben, besonders wenn wir eine Funktion als Argument an eine andere Funktion übergeben. Stattdessen erstellen wir inline Funktionen. Diese Funktionen müssen nicht benannt werden, weil wir sie sonst nirgendwo wiederverwenden.

Um das zu erreichen, verwenden wir oft folgende Syntax:

```js
const myFunc = function() {
  const myVar = "value";
  return myVar;
}
```

ES6 bietet uns syntaktischen Zucker, sodass wir anonyme Funktionen nicht mehr auf diese Weise schreiben müssen. Stattdessen können Sie die **Pfeilfunktions-Syntax** verwenden:

```js
const myFunc = () => {
  const myVar = "value";
  return myVar;
}
```

Wenn es keinen Funktionskörper gibt und nur ein Rückgabewert, erlaubt die Pfeilfunktions-Syntax, das Schlüsselwort `return` sowie die Klammern um den Code wegzulassen. Das hilft, kleinere Funktionen in Einzeiler zu vereinfachen:

```js
const myFunc = () => "value";
```

Dieser Code gibt standardmäßig weiterhin den String `value` zurück.

# --instructions--

Schreiben Sie die Funktion, die der Variablen `magic` zugewiesen ist und einen `new Date()` zurückgibt, mit Pfeilfunktions-Syntax um. Stellen Sie außerdem sicher, dass nichts mit dem Schlüsselwort `var` definiert wird.

# --hints--

Sie sollten das Schlüsselwort `var` ersetzen.

```js
assert.notMatch(code, /var/g)
```

`magic` sollte eine Konstante sein (durch Verwendung von `const`).

```js
assert.match(code, /const\s+magic/g)
```

`magic` sollte ein `function` sein.

```js
assert(typeof magic === 'function');
```

`magic()` sollte das korrekte Datum zurückgeben.

```js
assert(magic().setHours(0, 0, 0, 0) === new Date().setHours(0, 0, 0, 0));
```

Das Schlüsselwort `function` sollte nicht verwendet werden.

```js
assert.notMatch(code, /function/g)
```
