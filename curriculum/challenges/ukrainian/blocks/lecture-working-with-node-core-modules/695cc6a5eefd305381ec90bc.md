# --description--

Багато мов і фреймворків надають способи роботи з файлами та папками. Node.js дозволяє робити це за допомогою модуля `fs` або модуля файлової системи.

Цей урок зосередиться на модулі Node `fs`, як він працює та на трьох різних способах використання методів, які він надає.

Модуль Node `fs` надає вам методи для роботи з файлами та папками, включно з операціями відкриття та закриття, читання та запису, а також видалення.

Модуль `fs` є стандартним модулем Node.js, тому він доступний для використання, доки у вашому середовищі встановлено Node.js. Щоб використати модуль, імпортуйте його таким чином:

```js
const fs = require("fs");
```

Гнучкість модуля `fs` полягає в тому, що він дозволяє використовувати свої методи як синхронно, так і асинхронно. Методи модуля `fs` за замовчуванням асинхронні, але для кожного методу існує синхронна форма:

```js
fs.writeFile() // Asynchronous file writing
fs.writeFileSync() // Synchronous file writing

fs.readFile() // Asynchronous file reading
fs.readFileSync() // Synchronous file reading

fs.open() // Opens a file
fs.openAsBlob() // Opens as blob
fs.openSync() // Synchronous open
fs.opendir() // Opens directory
fs.opendirSync() // Synchronous directory open
```

На цьому не все. Залежно від ваших потреб, ви можете використовувати методи трьома способами:

- з функціями зворотного виклику, наприклад `fs.writeFile()` (асинхронно)
- з промісами, якщо ви віддаєте перевагу синтаксису `async/await`, наприклад `fs.promises.writeFile()`
- синхронно, наприклад `fs.writeFileSync()`

Ось базовий синтаксис для асинхронного використання методів:

```js
fs.writeFile("filePath", "content", "utf8", (err) => {
  if (err) {
    throw err;
  }
  console.log("File written to!");
});
```

Для версії з промісами ви можете імпортувати її з `fs/promises` або додати `promises` до вашого імпорту `fs`. Ось синтаксис:

```js
async function promisesExample() {
  try {
    await fs.promises.writeFile("filePath", "content", "utf8");
    console.log("File written to!");
  } catch (err) {
    console.error("Error:", err);
  }
}

promisesExample();
```

А ось синхронна версія:

```js
try {
  fs.writeFileSync("filePath", "content", "utf8");
  console.log("File written to!");
} catch (err) {
  console.error("Error:", err);
}
```

Зверніть увагу, що використання будь-якого з методів синхронно є блокуючим. Іншими словами, ваша програма зупиниться і чекатиме, поки операція завершиться, перш ніж перейти до наступного рядка коду.

Для швидких скриптів, невеликих проєктів або одноразових завдань синхронні методи, ймовірно, підходять. Але в реальних застосунках блокуючий ввід/вивід стає проблемою, бо він може заморожувати інші частини вашого застосунку, поки Node чекає на завершення операцій файлової системи. Це уповільнює роботу і погіршує продуктивність та досвід користувача.

Ось чому існують асинхронні методи. Вони дозволяють решті вашої програми продовжувати працювати, поки Node обробляє файлові операції у фоновому режимі. Традиційно для цього використовували функції зворотного виклику, але вони можуть швидко ускладнюватися, коли ваш застосунок зростає.

Проміси та `async/await` вирішують цю проблему. Вони все ще неблокуючі, але код читається як звичайний синхронний і значно легший у підтримці.

З цієї причини підхід на основі промісів сьогодні зазвичай вважається кращим, і саме його ми використовуватимемо в решті цього уроку.

Метод `writeFile()` дозволяє записувати у існуючий файл. Якщо файл не існує, він створює його у поточній папці, а потім записує вказаний вміст.

Ось як використовувати метод `writeFile()`:

```js
const fs = require("fs/promises");

async function writeToFile() {
  try {
    await fs.writeFile(
      "article.md",
      "## Node `fs` Module: The Complete Guide",
      "utf8",
    );
    console.log("File written to!");
  } catch (err) {
    console.error("Error writing to file:", err);
  }
}

writeToFile(); // File written to!
```

Ось вміст файлу article.md після цього:

```md
## Node `fs` Module: The Complete Guide
```

Метод `appendFile()` дозволяє додати до вмісту існуючого файлу. Ось як його використовувати:

```js
const fs = require("fs/promises");

async function appendToFile() {
  try {
    await fs.appendFile(
      "article.md",
      "\nIn this article, you will learn all there is to know about the Node fs module...",
      "utf8",
    );
    console.log("File appended to!");
  } catch (err) {
    console.log("Error appending to file:", err);
  }
}

appendToFile(); // File appended to!
```

Метод `readFile()` дозволяє побачити, який вміст файлу:

```js
const fs = require("fs/promises");

async function readFileContent() {
  try {
    const fileContent = await fs.readFile("article.md", "utf8");
    console.log("File content:", fileContent);
  } catch (err) {
    console.log("File read successfully");
  }
}

readFileContent();

/*
File content: ## Node `fs` Module: The Complete Guide

In this article, you will learn all there is to know about the Node fs module...
*/
```

Якщо ви не вкажете кодування символів `utf8`, ви отримаєте вміст файлу у вигляді буфера:

```js
async function readFileContent() {
  try {
    const fileContent = await fs.readFile("article.md");
    console.log("File content:", fileContent);
  } catch (err) {
    console.log("File read successfully");
  }
}

readFileContent();

/*
File content: <Buffer 23 23 20 4e 6f 64 65 20 60 66 73 60 20 4c 69 62 72 61 72 79 3a 20
 54 68 65 20 43 6f 6d 70 6c 65 74 65 20 47 75 69 64 65 0a 0a 49 6e 20 74 68 69 73 20 ... 
72 more bytes>
*/
```

У наступному уроці ви навчитеся працювати з буфером за допомогою модуля Buffer у Node.js.

Нарешті, метод `unlink()` дозволяє видалити файл:

```js
const fs = require("fs/promises");

async function deleteFile() {
  try {
    await fs.unlink("article.ts");
    console.log("File deleted successfully!");
  } catch (err) {
    console.error("Error deleting file:", err);
  }
}

deleteFile();
```

Якщо файл не існує, ви отримаєте повідомлення про помилку, схоже на це:

```bash
Error deleting file: Error: ENOENT: no such file or directory, unlink 'article.ts'
```

Якщо файл існує, він буде видалений, і ви отримаєте повідомлення про успішне видалення, яке ви задали.
