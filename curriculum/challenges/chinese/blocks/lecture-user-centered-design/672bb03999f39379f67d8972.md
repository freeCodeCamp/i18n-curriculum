# --interactive--

**注意**：某些交互示例可能使用你尚未学习的 CSS。不要担心尝试理解所有的代码。示例的目的是向你展示这些设计概念的预览，以便你更好地理解其工作原理。要查看预览，你需要启用交互式编辑器。

渐进式披露是一种设计模式，用于仅根据用户当前的活动显示相关内容并隐藏其余部分。这样做是为了减少认知负担，使用户体验更加直观。

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<div class="progressive-disclosure">
  <button id="show-details-btn" aria-expanded="false" aria-controls="extra-content">
    Show More Details
  </button>
  <div id="extra-content" class="hidden" tabindex="-1">
    <p>
      Here are additional details that are revealed only when you want to see them.
      This keeps the interface clean and reduces cognitive load.
    </p>
  </div>
</div>
```

```js
const btn = document.getElementById('show-details-btn');
const content = document.getElementById('extra-content');

btn.addEventListener('click', () => {
  const isHidden = content.classList.toggle('hidden');
  btn.setAttribute('aria-expanded', !isHidden);
  if (!isHidden) {
    content.focus();
  }
});

```

例如，当你使用 Google 的 `search` 页面时，你的大多数搜索将是简单的。因此，拥有一个选项非常少的简单用户界面是合理的。但在你需要更高级的 `search` 的少数情况下，Google 提供了一个高级的 `search` 选项，能够显示更多选项。

你可以在 Google 首页的设置菜单中查找 Google 的高级查找选项。或者你可以访问 `google.com/advanced_search`。

渐进式披露的另一个例子是在电子商务网站上的“More details”按钮。当你访问像 Amazon 这样的网站时，你会看到不同类别的产品图像列表。这使用户能够轻松购物，而不会被过多的信息淹没。

如果用户对某个特定产品感兴趣，他们可以点击产品 `Image` 以显示有关该产品的更多详细信息。立即显示所有信息可能会让用户感到不知所措。因此，通过使用渐进式披露，用户可以在准备好时选择查看更多信息。

```js
document.querySelectorAll('.details-btn').forEach(button => {
  button.addEventListener('click', () => {
    const product = button.closest('.product');
    const details = product.querySelector('.product-details');
    const isHidden = details.classList.toggle('hidden');
    product.setAttribute('aria-expanded', !isHidden);
    if (!isHidden) {
      details.focus();
      button.textContent = 'Hide details';
    } else {
      button.textContent = 'More details';
    }
  });
});
```

那么，使用渐进式披露技术设计时有哪些最佳实践？第一个考虑是始终保持所有重要信息可见。用户不应该因为额外的高级特色而难以查找关键信息。

另一个考虑是为用户提供一个单一的访问点，以访问额外的特色或信息。这个访问点可以是页面上始终可见的按钮或链接。添加多个访问点可能会让用户感到困惑，并导致糟糕的用户体验。
