# --interactive--

Ви дізналися, що класи — це як шаблони, які можна використовувати для створення об’єктів.

Ці шаблони потребують загального способу посилатися на конкретний об’єкт, який створюється, до якого здійснюється доступ або який змінюється в коді. Саме для цього використовується ключове слово `this`. Воно посилається на контекст, у якому має виконуватися код. Якщо ви використовуєте його всередині методу, значення `this` буде референсом на об’єкт, пов’язаний із цим методом. Таким чином ви можете отримати доступ до властивостей і методів об’єкта всередині класу та повторно використовувати метод для різних об’єктів.

Наприклад, тут у нас є клас `Dessert`. Цього разу ми визначаємо властивість `hasPeanuts`.

```js
class Dessert {
  constructor(hasPeanuts) {
    this.hasPeanuts = hasPeanuts;
  }
}
```

Зверніть увагу, що ми використовуємо ключове слово `this` всередині конструктора. Пам’ятайте, що конструктор викликається автоматично, коли створюється новий об’єкт класу.

Ключове слово `this` повідомляє програмі, що `hasPeanuts` має бути властивістю об’єкта, який наразі створюється.

Праворуч від інструкції присвоєння значення параметра `hasPeanuts` призначається властивості `hasPeanuts` нового об’єкта.

```js
this.hasPeanuts = hasPeanuts;
```

Це в контексті конструктора, але ви також можете використовувати ключове слово `this` у інших методах.

Ось наш клас `Dessert` з кількома змінами. Тепер у нас є дві властивості: `name` і `price`.

```js
class Dessert {
  constructor(name, price) {
    this.name = name;
    this.price = price;
  }

  showPrice() {
    console.log(`The price of ${this.name} is $${this.price}.`);
  }
}
```

Також у нас є новий метод `showPrice`. У цьому методі ми виводимо описове повідомлення в консоль із назвою та ціною об’єкта десерту.

Зверніть увагу, як ми використовуємо ключове слово `this` для доступу до властивостей об’єкта всередині методу `showPrice`.

Яке значення матиме ключове слово `this` у контексті цього методу? Воно посилатиметься на об’єкт, для якого викликається метод. У цьому випадку ми створюємо новий екземпляр Dessert із назвою `"Brownie"` і ціною `$5.99`:

```js
const brownie = new Dessert("Brownie", 5.99);
```

Якщо ми викликаємо метод `.showPrice()` для цього об’єкта:

```js
brownie.showPrice();
```

Ось повний приклад:

:::interactive_editor

```js
class Dessert {
  constructor(name, price) {
    this.name = name;
    this.price = price;
  }

  showPrice() {
    console.log(`The price of ${this.name} is $${this.price}.`);
  }
}

const brownie = new Dessert("Brownie", 5.99);
brownie.showPrice();
console.log(brownie);
```

:::

Назва та ціна поточного об’єкта (`brownie`) замінюються в рядку.

Все це завдяки ключовому слову `this`, яке дає загальний спосіб посилатися на поточний об’єкт всередині класу. Це дозволяє повторно використовувати метод для різних об’єктів.

Це в контексті методів об’єктів, але ключове слово `this` також матиме значення у звичайних функціях і стрілкових функціях.

Його значення у звичайній функції зазвичай посилається на глобальний об’єкт (у нестрогому режимі) або `undefined` (у строгому режимі).

Значення ключового слова `this` трохи відрізняється у стрілкових функціях. Стрілкові функції успадковують значення `this` із зовнішньої області видимості, де вони визначені. Вони не створюють власного зв’язку `this`. Це означає, що значення `this` залежить від контексту, у якому визначена стрілкова функція, а не від того, де або як вона викликається. Це робить стрілкові функції корисними для функцій зворотного виклику або для збереження контексту.

Однак через це значення `this` не посилатиметься на поточний об’єкт у методах об’єктів, тож це слід враховувати при роботі зі стрілковими функціями.

Те саме стосується стрілкових функцій, визначених у інших функціях. Вони успадковують значення `this` із батьківської області видимості.

Ключове слово `this` — це потужний інструмент для доступу та маніпулювання об’єктами. Розуміння того, як воно працює, є необхідним для опанування об’єктно-орієнтованого програмування в JavaScript.
