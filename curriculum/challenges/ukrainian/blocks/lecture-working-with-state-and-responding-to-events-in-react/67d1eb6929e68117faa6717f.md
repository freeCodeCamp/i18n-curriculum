# --description--

У React відтворення — це процес, під час якого компоненти з’являються в інтерфейсі користувача (UI), зазвичай у браузері.

React бере весь ваш JavaScript, JSX і CSS код, визначає, як це має виглядати, а потім відображає це в інтерфейсі користувача.

Процес відтворення складається з трьох етапів: тригер, відтворення та коміт. Давайте розглянемо їх детальніше.

Етап тригера відбувається, коли React виявляє, що щось змінилося і інтерфейс користувача може потребувати оновлення. Ця зміна часто пов’язана з оновленням стану або пропсів.

Наприклад, усвідомлення, що настав час обіду, може спонукати вас піти на кухню і почати готувати.

У наведеному нижче прикладі `Counter` натискання кнопки збільшення або зменшення викликає React показати нове значення `count`:

```jsx
import { useState } from "react";

function Counter() {
  const [count, setCount] = useState(0);

  return (
    <div>
      <h1>{count}</h1>
      <button onClick={() => setCount(count - 1)}>Decrement</button>
      <button onClick={() => setCount(count + 1)}>Increment</button>
    </div>
  );
}

export default Counter;
```

Після спрацьовування тригера React переходить до етапу відтворення. Тут React повторно оцінює ваші компоненти і визначає, що потрібно відобразити.

Для цього React використовує легку копію «справжнього» DOM. Це називається віртуальним DOM. За допомогою віртуального DOM React може швидко перевірити, що потрібно змінити в компоненті.

Уявіть цей етап як момент, коли ви на кухні, зібрали інгредієнти і готуєте обід.

Для компонента `Counter` етап відтворення — це момент, коли React знову запускає функції з новим значенням `count`. React перераховує, як має виглядати частина `<h1>{count}</h1>` компонента на основі оновленого значення `count`, але ви не побачите змін на екрані до наступного етапу — коміту.

Етап коміту — це момент, коли React бере підготовлені зміни з віртуального DOM і застосовує їх до справжнього DOM. Іншими словами, це етап, коли ви бачите фінальний результат на екрані.

Щоб це зробити, React порівнює віртуальний DOM зі справжнім DOM, визначає лише ті частини, які потребують оновлення, і застосовує ці зміни до справжнього DOM, оновлюючи інтерфейс користувача.

Уявіть цей етап як момент, коли ви подаєте приготовану їжу, роблячи її видимою, так само як React робить видимими оновлення, які він комітить у справжній DOM.

Щодо компонента `Counter`, етап коміту — це момент, коли нове значення `count` застосовується до елемента `h1`, і ви бачите зміну на сторінці.

Ці три процеси відбуваються дуже швидко, бо React мінімізує прямі маніпуляції з DOM, спочатку обчислюючи зміни у віртуальному DOM, а потім оновлюючи лише ті частини справжнього DOM, які потрібно змінити.
