# --description--

每个 `project` 都是独特的，你应该仔细考虑这些技术中哪些对你的网页 `application` 最有帮助。

首先，由于所有样式都会被解析，你应该从你的 CSS 样式表中移除任何不必要的样式。

你还应该将你的 CSS 样式拆分到多个文件中。这样，如果某个文件不需要渲染特定网页，它将不会被装载，也不会阻塞渲染进程。

默认情况下，浏览器假设所有样式表都应阻塞渲染进程。但你可以通过使用 `media` 属性和媒体查询告诉浏览器何时应装载样式表。 

在此示例中，只有当文档正在被打印时，才会应用 `print.css` 样式表，因为它具有值为 `print` 的 `media` 属性：

```html
<link rel="stylesheet" href="print.css" media="print" />
```

你还可以在构建进程中自动压缩你的 CSS 文件以减小其大小，并在托管你的网页应用的服务器上压缩这些文件。

尽量保持你的 CSS 选择器尽可能简单，不要尝试选择超过必要数量的元素。

尽可能利用 CSS 的层叠特性。某些样式将被继承。

使用 `rel="preload"` 预加载资源是另一项重要技术。关键资源应优先装载。

请记住，CSS 动画可能会影响性能。

对某些 CSS 属性（例如维度、位置和版型）进行动画处理会触发一个称为 `"reflow"` 的进程，在此期间浏览器会重新计算页面上某些元素的位置和几何形状。这需要重绘，计算开销很大。

因此，建议尽可能减少 CSS 动画的数量，或者至少为用户提供一个切换它们开关的选项。

尽管字体可以增强你的网页应用的视觉表现，但它们可能会影响性能。有些字体文件很大，达到数兆字节。尽量最多使用两到三种字体，并尽可能使用网页安全字体。

你也可以考虑预装载最重要的 `font`，以便快速使用。

这些是网页开发中优化 CSS 的一些最重要且常用的技术。通过将它们纳入，你可以优化你的网站 `performance`，以创建流畅的 `user` 体验。
