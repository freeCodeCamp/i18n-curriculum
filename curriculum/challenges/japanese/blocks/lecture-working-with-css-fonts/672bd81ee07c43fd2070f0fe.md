# --description--

ウェブセーフフォントは、コンピューターやデバイスにインストールされている可能性が非常に高いフォントの部分集合です。異なるオペレーティングシステムやウェブブラウザで広くサポートされているため、一貫してレンダリングおよび表示される可能性が高いです。これがどのように機能するか見てみましょう。

ブラウザはウェブサイト上のフォントを解釈し表示する役割を担っています。ブラウザがフォントをレンダリングする際、ユーザーのシステム上でフォントファイルを探そうとします。しかし、フォントが見つからない場合は通常、デフォルトのシステムフォントにフォールバックします。これにより、ウェブサイトでレンダリングされるべき特定のフォントが欠けていても、コンテンツは読みやすいままになります。

ただし、ブラウザが選択するフォールバックフォントは、元々レンダリングされるはずだったフォントとは大きく異なる場合があります。これは全体のデザインやユーザー体験に重大な影響を与えることがあります。これを避けるために、可能な限りウェブセーフフォントを使用することをお勧めします。選択肢は2つあります。ウェブセーフフォントをプライマリフォントとして使うか、カスタムフォントを使う場合はウェブセーフフォントをフォールバックとして設定する方法です。こうすることで、カスタムフォントが見つからなかった場合のウェブサイトの見た目をコントロールできます。

いくつかのウェブセーフフォントの例を見てみましょう。前のレッスンで、サンセリフフォントは文字の端に小さな「足」や線がないため、画面上で読みやすく、ウェブ開発でよく使われることを学びました。ウェブセーフなサンセリフフォントの例は以下の通りです。

- Arial
- Verdana
- Trebuchet MS

一方、セリフフォントは文字の端に小さな「足」があり、伝統的な印刷物でよく使われます。しかし、ウェブ開発で使う必要がある場合、ウェブセーフなセリフフォントには以下があります。

- Times New Roman
- Georgia

ウェブセーフフォントを使うことで、デバイスやプラットフォーム間でデザインの一貫性を保てます。また、シンプルで読みやすいため、視覚障害のあるユーザーのアクセシビリティ向上にも役立ちます。さらに、既にインストールされているフォントなので、ページの読み込み時間を短縮できます。

ウェブセーフフォントが何かを理解したので、これらを使い始めたり、フォールバックとして代入してブラウザやプラットフォーム間で信頼できるユーザー体験を作成してください。
