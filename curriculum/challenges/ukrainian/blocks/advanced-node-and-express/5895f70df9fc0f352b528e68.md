# --description--

Стратегія — це спосіб автентифікації користувача. Ви можете використати стратегію, щоб дозволити користувачам автентифікуватися на основі локально збереженої інформації (якщо вони спочатку зареєструвалися) або через різних провайдерів, таких як Google чи GitHub. Для цього проєкту ми використаємо підпрограмне забезпечення Passport. Passport надає комплексний набір стратегій, які підтримують автентифікацію за допомогою імені користувача та пароля, GitHub, Google та інших.

`passport-local@~1.0.0` вже додано як залежність. Додайте його до свого сервера так:

```javascript
const LocalStrategy = require('passport-local');
```

Скажіть passport **використовувати** створений об’єкт `LocalStrategy` з кількома визначеними налаштуваннями. Переконайтеся, що це (як і все, що йде далі) знаходиться всередині підключення до бази даних, оскільки воно на ньому базується!:

```javascript
passport.use(new LocalStrategy((username, password, done) => {
  myDataBase.findOne({ username: username }, (err, user) => {
    console.log(`User ${username} attempted to log in.`);
    if (err) return done(err);
    if (!user) return done(null, false);
    if (password !== user.password) return done(null, false);
    return done(null, user);
  });
}));
```

Це визначає процес, який буде використовуватися, коли ви намагаєтеся автентифікувати когось локально. Спочатку він намагається знайти користувача у вашій базі даних за введеним іменем користувача. Потім перевіряє, чи пароль співпадає. Нарешті, якщо не виникло помилок, які ви перевіряли (наприклад, неправильний пароль), повертається об’єкт `user` і користувач автентифікований.

Багато стратегій налаштовуються з різними параметрами. Загалом, їх легко налаштувати, орієнтуючись на README у репозиторії цієї стратегії. Хорошим прикладом є стратегія GitHub, де вам не потрібно турбуватися про ім’я користувача чи пароль, бо користувача перенаправляють на сторінку автентифікації GitHub. Якщо він увійшов і погоджується, GitHub повертає його профіль для вашого використання.

У наступному кроці ви налаштуєте, як фактично викликати стратегію автентифікації для перевірки користувача на основі даних форми.

Відправте свою сторінку, коли вважатимете, що зробили все правильно. Якщо виникають помилки, ви можете <a href="https://forum.freecodecamp.org/t/advanced-node-and-express/567135#authentication-strategies-6" target="_blank" rel="noopener noreferrer nofollow">переглянути проєкт, виконаний до цього кроку</a>.
