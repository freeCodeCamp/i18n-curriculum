# --description--

让我们了解你可以使用的命令行快捷，以加快生产力。

终端和命令行快捷可以帮助你简化工作流程并最大化你的生产力。对于 Linux 和 macOS，这两者都可以追溯到 Unix，许多这些快捷将是相同的。但对于 Windows，会有一些差异。

第一个快捷键是向上箭头键。按下此键将允许你以一个命令为单位向后循环浏览你的命令历史。然后，向下箭头键可以用来向前循环浏览你的命令历史。这两个键允许你快速循环浏览你之前运行过的命令，以查找你需要再次运行的命令。

许多 shell 还会提供自动补全特色，根据你开始输入的内容推荐命令。可以使用 `Tab` 键填写建议的其余部分，快速将完整的语法填充到你的命令行中。这些建议的生成方式因 shell 而异。例如，在 zsh 中，这些建议基于你最近的命令历史。但在 PowerShell 中，它们基于可用的命令、变量和参数（对最近使用的权值较低）。

有时一个命令可能会在你的终端产生大量输出。这些额外的噪音可能会妨碍或干扰你的工作流程。在基于 *nix 的终端（如 Linux 和 macOS 设备上）中，你可以按 Control + L 来清除整个屏幕，并从一个干净的提示符开始。在 PowerShell 中，你需要运行 `cls` 命令（你可以将其绑定到像 Control + L 这样的键组合）。

如果你需要中止一个命令以停止其执行，可以使用 Control + C 来终止进程。这将结束该命令并创建一个新建的提示符，允许你继续处理其他需要完成的工作。对于 PowerShell 用户，Control + C 也用于拷贝文本——只有在上下文不模糊时（例如没有选中文本进行拷贝时）才会用于终止进程。

接下来的几个快捷只在基于 *nix 的终端中可用，且没有 PowerShell 备选。

有时你可能需要多任务处理，允许一个进程或命令在后台运行，同时你继续处理另一个任务。按下 Control + Z 会将当前进程置于后台任务，并返回命令行，让你可以继续工作。当你需要将焦点切换回后台任务时，可以使用 `fg` 来还原它。

最后，可能有些命令你需要连续运行两次。你可以按上箭头键向后循环到之前运行的命令，也可以在命令行中输入两个感叹号（`!!`）并按回车——这将自动运行上一次执行的命令。

许多终端提供更多快捷，我鼓励你查找你特定设置的文档，以找到最适合你的快捷。但这些基本的通用快捷应作为最大化你的生产力和成为终端高手的良好起点。
