# --interactive--

**注意**: いくつかのインタラクティブな例では、まだ学んでいないCSSが使われている場合があります。すべてのコードを理解しようと心配しないでください。これらの例の目的は、デザインの概念をプレビューとして示し、仕組みをよりよく理解してもらうことです。

「スケール」とは、何かの大きさを指します。

ウェブデザインでスケールを見るときは、異なる要素間のサイズの関係性や、それらの要素が異なる画面サイズにどのように適応するかを見ています。

要素に適切なスケールを使うことは、視覚的な階層構造で重要な役割を果たします。大きな要素はより多くの注目を集め、ユーザーを意図した通りにコンテンツへ導くことができます。

例えば、見出しと段落の間の視覚的な区切りは読者の注意を引きますが、スケールは目を引くテキストになるよう適切である必要があります。

:::interactive_editor

```html
<style>
  body {
    font-family: sans-serif;
    padding: 20px;
    background-color: #fdfdfd;
    color: #333;
  }

  .section {
    margin-bottom: 40px;
  }

  .big-heading {
    font-size: 40px;
    font-weight: bold;
    margin-bottom: 12px;
  }

  .paragraph {
    font-size: 16px;
    max-width: 600px;
    line-height: 1.6;
  }
</style>

<div class="section">
  <div class="big-heading">Discover the Power of Smart Design</div>
  <p class="paragraph">
    Great design isn’t just about colors or fonts — it’s also about scale. 
    A large heading like this one instantly grabs your attention, while the paragraph beneath it provides context and detail.
    Proper scaling creates a clear path for the reader’s eye to follow.
  </p>
</div>
```

:::

スケールはテキストだけに適用されるわけではありません。画像にも重要です。バナー画像のスケールはデスクトップのレイアウトでは適切でも、モバイルのレイアウトでは大きすぎる場合があります。

画像をより適切な比率に縮小することで、視覚的なインパクトを保ちつつ、サイト上の情報が見やすくなります。

:::interactive_editor

```html
<style>
  body {
    font-family: sans-serif;
    padding: 20px;
    background-color: #fefefe;
    color: #333;
  }

  .banner {
    max-width: 100%;
    height: auto;
    display: block;
    margin: 0 auto 20px auto;
    border-radius: 8px;
  }

  .content {
    max-width: 600px;
    margin: 0 auto;
    font-size: 16px;
    line-height: 1.6;
  }

  @media (max-width: 600px) {
    .banner {
      max-width: 90%;
    }

    .content {
      font-size: 15px;
    }
  }
</style>

<img 
  src="https://placehold.co/1200x400/png?text=Large+Banner+Image"
  alt="Banner" 
  class="banner"
>

<div class="content">
  <p>
    This banner image looks great on a large screen, but on smaller devices, it scales down automatically.
    That way, it still delivers a strong visual impression without pushing the actual content off the screen.
    Scaling images properly helps maintain balance and accessibility across layouts.
  </p>
</div>
```

:::

スケールはインタラクティビティやウェブサイトの使いやすさにも重要です。ナビゲーションバーのテキストが適切なスケールでないと、モバイルユーザーはリンクをタップしにくくなります。

適切にスケールを調整すれば、リンクは読みやすくなるだけでなく、モバイルユーザーがクリックしやすくなります。

:::interactive_editor

```html
<style>
  body {
    font-family: sans-serif;
    padding: 20px;
    margin: 0;
    background-color: #fafafa;
  }

  .navbar {
    background-color: #004080;
    padding: 15px 20px;
    display: flex;
    flex-wrap: wrap;
    gap: 20px;
  }

  .nav-link {
    color: white;
    text-decoration: none;
    font-size: 18px;
    padding: 10px 15px;
    display: inline-block;
    border-radius: 4px;
  }

  .nav-link:hover {
    background-color: #0059b3;
  }

  @media (max-width: 600px) {
    .nav-link {
      font-size: 20px;
      padding: 14px 18px;
    }
  }
</style>

<nav class="navbar">
  <a href="#" class="nav-link">Home</a>
  <a href="#" class="nav-link">About</a>
  <a href="#" class="nav-link">Services</a>
  <a href="#" class="nav-link">Contact</a>
</nav>
```

:::

スケールはデザインの多くの面で重要です。基本を説明したので、その重要性の基礎的な理解ができたはずです。
