# --interactive--

웹 디자인에서 배경은 웹페이지의 전체적인 분위기와 느낌을 정의하는 데 중요한 역할을 합니다.

하지만 배경을 디자인할 때는 시각 장애가 있는 사용자를 포함해 모든 사용자가 콘텐츠를 사용할 수 있고 읽을 수 있도록 접근성을 고려하는 것이 매우 중요합니다.

배경과 관련된 주요 접근성 문제 중 하나는 배경과 텍스트 사이에 충분한 명암 대비가 있는지 확인하는 것입니다.

충분한 대비가 없으면 저시력이나 색맹을 포함한 시각 장애가 있는 사용자가 페이지의 내용을 읽기 어려울 수 있습니다.

대비란 두 색상 간 밝기 또는 어둠의 차이를 의미합니다. 배경색과 텍스트 색상 사이에 충분한 대비가 있어야 가독성이 확보됩니다.

웹 콘텐츠 접근성 지침(WCAG)은 일반 텍스트에 대해 최소 명암 대비율 4.5:1, 큰 텍스트에 대해서는 3:1을 권장합니다.

예를 들어, 흰색 텍스트를 연한 회색 배경 위에 놓으면 대비가 낮아 텍스트를 읽기 어렵습니다. 반면, 흰색 텍스트를 어두운 파란색 배경 위에 놓으면 대비가 좋아 모든 사용자가 읽기 편해집니다.

다음은 대비가 좋지 않은 예입니다:

:::interactive_editor

```html
<p style="color: lightgray; background-color: whitesmoke;">
   This is an example of poor contrast.
</p>
```

:::

다음은 대비가 좋은 예입니다:

:::interactive_editor

```html
<p style="color: white; background-color: darkslategray;">
  This is an example of good contrast.
</p>
```

:::

또 다른 고려사항은 이미지나 여러 색상의 그라데이션처럼 복잡하거나 산만한 배경 위에 텍스트를 배치하지 않는 것입니다. 복잡한 배경은 대비와 상관없이 텍스트와 배경을 구분하기 어렵게 만듭니다.

배경을 디자인할 때 색상만으로 정보를 전달하는 것을 피하세요. 예를 들어, 오류 메시지를 빨간색으로, 성공 메시지를 초록색으로 표시하는 것만으로는 색맹 사용자가 문제를 겪을 수 있습니다.

색상 외에도 기호나 텍스트를 사용해 정보를 전달해야 합니다. 예를 들어, 빨간색 오류 메시지 옆에 아이콘이나 굵은 텍스트를 함께 사용해 오류임을 명확히 할 수 있습니다.

덜 흔하지만 배경 오디오나 비디오도 접근성에 영향을 줄 수 있습니다.

배경 음악이나 자동 재생 비디오는 특히 인지 장애가 있는 사용자에게 방해가 될 수 있습니다. 배경 오디오를 포함할 경우 사용자가 음소거하거나 일시 정지할 수 있는 방법을 반드시 제공하세요.

이러한 접근성 고려사항을 반영하면 모든 사용자가 능력에 상관없이 콘텐츠를 읽고 사용할 수 있는 더 포용적인 디자인을 만들 수 있습니다.
