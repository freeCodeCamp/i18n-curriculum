# --description--

يحتوي المتصفح على أدوات مطوِّر مدمجة يمكنك استخدامها لتصحيح أخطاء HTML وCSS وJavaScript.

مع ذلك، فهي ليست مثالية للعثور على الأخطاء البرمجية وإصلاحها في تطبيقات React. لذلك، طوّر فريق React أداة تسمى "أدوات مطوِّر React" (المعروفة أيضًا باسم React DevTools) لتتمكن من فحص وتصحيح وتحليل تطبيقات React.

تتوفر React DevTools كإضافة للمتصفح لكل من Chrome وEdge وFirefox. إذا كنت تستخدم Chrome أو Edge، توجه إلى متجر Chrome الإلكتروني، وابحث عن "React Developer Tools"، وأضفها إلى متصفحك.

وإذا كنت تستخدم Firefox، توجه إلى صفحة الإضافات الخاصة بـ Firefox، وابحث عن الأداة، وأضفها إلى متصفحك.

إذا كنت تستخدم Safari، يمكنك تثبيت React DevTools من npm عبر تشغيل الأمر `npm install -g react-devtools` أو `yarn global add react-devtools`.

بعد تثبيت وتمكين React DevTools، إذا فتحت تطبيق React في متصفحك ثم فتحت أدوات المطوِّر في المتصفح، سترى تبويبين إضافيين: Components وProfiler.

يعرض تبويب Components كل مكوِّن لك في شكل عرض شجري. يمكنك من خلاله:

- عرض تسلسل المكونات في التطبيق.
- فحص وتعديل الخصائص (props) والحالات (states) وقيم السياق (context) في الوقت الحقيقي.
- فحص الكود المصدري لكل مكوِّن محدد.
- تسجيل بيانات المكوِّن في وحدة التحكم.
- فحص عناصر DOM الخاصة بالمكوِّن.

من ناحية أخرى، يساعدك تبويب Profiler على تسجيل وتحليل أداء المكونات لتتمكن من تحديد عمليات إعادة العرض غير الضرورية، وعرض مدة الالتزام، والأمور التي يمكنك تحسينها.

إليك تطبيقًا بسيطًا يوضح لك كيف يمكنك فحص المكونات وأي خصائص وحالات تمتلكها. هذا مشابه للكود الذي استخدمناه في درس سابق عن تمرير الخصائص (prop drilling):

```jsx
import { useState } from "react";

export default function App() {
  const greeting = "Hello, Prop Drilling!";
  const response = "I'm not here to play!";

  return <Parent greeting={greeting} response={response} />;
}

const Parent = ({ greeting, response }) => {
  return <Child greeting={greeting} response={response} />;
};

const Child = ({ greeting, response }) => {
  return <Grandchild greeting={greeting} response={response} />;
};

const Grandchild = ({ greeting, response }) => {
  const [count, setCount] = useState(0);

  return (
    <>
      <h1>{greeting}</h1>
      <h2>{response}</h2>

      <button onClick={() => setCount(count + 1)}>Increase Count</button>
      <h2>Count: {count}</h2>
      <button onClick={() => setCount(count - 1)}>Decrease Count</button>
    </>
  );
};
```

إذا نظرت في تبويب Components في React DevTools، سترى العرض الشجري للمكونات. مكوِّن `App` في الأعلى، يليه مكونات `Parent` و`Child` و`Grandchild`.

إذا اخترت أيًا من هذه المكونات، يمكنك رؤية الخصائص والحالة فيها. إذا اخترت مكوِّن `Parent`، سترى خصائص `greeting` و`response`، وهما على التوالي `Hello, Prop Drilling!` و`I'm not here to play!`.

يمكنك رؤية الخصائص وتحديث الحالة في الوقت الحقيقي، وتغييرها إذا لزم الأمر. على سبيل المثال، يمكنك اختيار مكوِّن `Grandchild` وتغيير خاصية `greeting` من `Hello, Prop Drilling!` إلى `Hello, Welcome to Prop Drilling!`، وسترى التغيير ينعكس على الصفحة فورًا.

لتسجيل بيانات في مكوِّن إلى وحدة التحكم، افحص عناصر DOM المطابقة واطلع على الكود المصدري للمكوِّن. الأيقونات في الزاوية العلوية اليمنى تتيح لك ذلك. إذا اخترت مكوِّن `Grandchild` ونقرت على زر `Log the component data to the console`، فسيسجل الخصائص والحالة والهوكس والعقد وبيانات أخرى في وحدة التحكم.

خطأ شائع قد تواجهه في React يسمى عدم تطابق الخصائص (props mismatch).

على سبيل المثال، لنفترض أنك في مكوِّن `Child` مررت خاصية `reply` بدلًا من `response` عن طريق الخطأ:

```jsx
const Child = ({ greeting, response }) => {
  return <Grandchild greeting={greeting} reply={response} />;
};
```

تذكر أن مكوِّن `Grandchild` يتوقع خاصية `response`. وبما أن المكوِّن يستقبل خاصية مختلفة، لا يمكنه عرض ذلك النص على الصفحة، ويضيف عنصر `h2` فارغ إلى DOM. بدلاً من ذلك، سترى فقط عنصر `h1` بالنص `Hello, Prop Drilling!`، مع الأزرار والنصوص الأخرى الموجودة بالفعل على الصفحة. عنصر `h2` الفارغ موجود لكنه غير مرئي إلا إذا فحصت DOM.

لإصلاح ذلك، يمكنك فحص تقدم الخصائص من مكوِّن `Parent` إلى `Child` وتعديل اسم الخاصية مباشرة. إذا ذهبت إلى تبويب Components، واخترت مكوِّن `Child`، وغيرت خاصية `reply` إلى `response`، سترى عنصر `h2` على الصفحة بالنص `I'm not here to play!`.
