## 無障礙介紹

- **網頁內容無障礙指引**：網頁內容無障礙指引（WCAG）是一組讓網頁內容對身心障礙者更易取用的指引。WCAG 的四大原則是 POUR，分別代表可感知（Perceivable）、可操作（Operable）、可理解（Understandable）和強固（Robust）。

## 無障礙輔助技術

- **螢幕閱讀器**：讀出電腦螢幕內容的軟體程式。它們被盲人或視障者用來存取網路。
- **大型文字或點字鍵盤**：供視障者存取網路時使用。
- **螢幕放大鏡**：將電腦螢幕內容放大的軟體程式。它們供視力低下的人士用來存取網路。
- **替代指點裝置**：供行動不便者存取網路時使用。這包括搖桿、滾球和觸控板等裝置。
- **語音辨識**：供行動不便的人士存取網路。它允許使用者使用聲音來控制電腦。

## 無障礙性稽核工具

- **常見的無障礙工具**：Google Lighthouse、Wave、IBM Equal Accessibility Checker 和 axe DevTools 是用來稽核網站無障礙性的常見無障礙工具。

## 無障礙最佳實踐

- **正確的標題層級結構**：你應該使用正確的標題層級來創建內容的邏輯結構。這有助於使用輔助技術的人理解你網站的內容。
- **無障礙性與表格**：使用表格時，你應該使用 `th` 元素來定義標題儲存格，並使用 `td` 元素來定義資料儲存格。這有助於使用輔助技術的人理解表格的結構。使用 `caption` 元素，你可以撰寫表格的標題（或名稱），讓使用者，特別是使用輔助技術的使用者，能快速了解表格的目的和內容。你應該將 `caption` 元素放置在 `table` 元素的開啟標籤之後。這樣，螢幕閱讀器和其他輔助技術可以在讀取內容之前，透過宣告標題來提供更多的背景關係。
- **輸入欄位擁有相關的標籤的重要性**：你應該使用 `label` 元素將標籤與表單輸入欄位相關聯。這有助於使用輔助技術的人了解輸入欄位的用途。
- **良好 `alt` 文字的重要性**：你應該使用 `alt` 屬性為影像提供文字替代。這有助於使用輔助技術的人理解影像的內容。
- **良好聯結文字的重要性**：你應該使用具描述性的聯結文字來幫助使用者了解聯結的目的。這有助於使用輔助技術的人理解聯結的用途。
- **讓音訊和影片內容可存取的最佳做法**：你應該為音訊和影片內容提供字幕和文字稿，以便讓聽力障礙者能夠存取。你也應該為影片內容提供音訊描述，以便讓視力障礙者能夠存取。
- **`tabindex` 屬性**：用於使元素可聚焦並定義使用鍵盤導航時的相對順序。切勿將 `tabindex` 屬性的值設為大於 0。你應該使用 0 或 -1 作為值。

:::interactive_editor

```html
<p tabindex="-1">Sorry, there was an error with your submission.</p>
```

:::

- **`accesskey` 屬性**：用於定義元素的鍵盤快速鍵。這可以幫助有行動不便的使用者更輕鬆地瀏覽網站。

:::interactive_editor

```html
<button accesskey="s">Save</button>
<button accesskey="c">Cancel</button>
<a href="index.html" accesskey="h">Home</a>
```

:::

## WAI-ARIA、角色和屬性

- **WAI-ARIA**：代表網頁無障礙倡議－可存取的豐富網際網路應用程式。它是一組可以添加到 HTML 元素的屬性，用以提升無障礙性。它為輔助技術提供關於內容目的和結構的額外資訊。
- **ARIA roles**：一組可添加到 HTML 元素以定義其用途的預設角色。這有助於使用輔助技術的人理解網站內容。範例包括 `role="tab"`、`role="menu"` 和 `role="alert"`。

ARIA 角色有六個主要類別：

- **文件結構角色**：這些角色定義網頁的整體結構。透過這些角色，輔助技術可以理解不同區段之間的關係，並協助使用者瀏覽內容。
- **Widget 角色**：這些角色定義互動元素的目的和功能，例如捲軸。
- **地標角色**：這些角色用來分類並標籤網頁的主要區段。螢幕閱讀器使用它們來提供方便的導覽至頁面中的重要區段。
- **Live region roles**：這些角色定義內容會動態變化的元素。如此一來，螢幕閱讀器和其他輔助技術就能向視覺障礙使用者宣告變更。
- **視窗角色**：這些角色定義子視窗，例如彈出模態對話盒。這些角色包括 `alertdialog` 和 `dialog`。
- **抽象的角色**：這些角色有助於組織文件。它們僅供瀏覽器內部使用，而非開發者使用，所以你應該知道它們的存在，但不應該在你的網站或網頁應用程式中使用它們。

- **`aria-label` 和 `aria-labelledby` 屬性**：這些屬性用於為元素提供程式化（或可存取的）名稱，幫助使用輔助技術（例如螢幕閱讀器）的人了解該元素的用途。當元素的視覺標籤是影像或符號而非文字時，這些屬性經常被使用。`aria-label` 允許你直接在屬性中定義名稱，而 `aria-labelledby` 則允許你引用頁面上現有的文字。

```html
<button aria-label="Search">
    <i class="fas fa-search"></i>
</button>
```

:::interactive_editor

```html
<input type="text" aria-labelledby="search-btn">
<button type="button" id="search-btn">Search</button>
```

:::

- **`aria-hidden` 屬性**：用於將元素從輔助技術（例如螢幕閱讀器）中隱藏。舉例來說，這可用於隱藏不提供任何有意義內容的裝飾性影像。

:::interactive_editor

```html
<button>
    <i class="fa-solid fa-gear" aria-hidden="true"></i>
    <span class="label">Settings</span>
</button>
```

:::

- **`aria-describedby` 屬性**：用於透過將元素與包含資訊的另一個元素關聯，提供該元素的額外資訊。這讓使用螢幕閱讀器的人在導覽到該元素時能立即存取額外資訊。常見用法包括將格式說明關聯到文字輸入框，或在無效的表單提交後將錯誤訊息關聯到輸入框。

:::interactive_editor

```html
<form>
    <label for="password">Password:</label>
    <input type="password" id="password" aria-describedby="password-help" />
    <p id="password-help">Your password must be at least 8 characters long.</p>
</form>
```

:::
