# --interactive--

Flüssige Animationen auf einer Webseite zu erstellen, kann knifflig sein, besonders wenn Sie nicht genau wissen, wie Sie das Timing richtig handhaben. Die gute Nachricht ist, dass die `requestAnimationFrame()`-API das einfacher macht.

`requestAnimationFrame()` ist eine Methode, mit der Sie den nächsten Schritt Ihrer Animation vor dem nächsten Bildschirm-Refresh planen können, was zu einer flüssigen und optisch ansprechenden Darstellung führt.

Der nächste Bildschirm-Refresh bezeichnet den Moment, in dem der Browser die visuelle Anzeige der Webseite aktualisiert. Dies geschieht mehrmals pro Sekunde, typischerweise etwa 60 Mal (oder 60 Frames pro Sekunde) auf den meisten Displays.

Um die `requestAnimationFrame()`-Methode zu verwenden, müssen Sie sie nur aufrufen und eine Callback-Funktion übergeben:

```js
requestAnimationFrame(callback);
```

Der Aufruf von `requestAnimationFrame()` muss zuerst innerhalb einer Funktion erfolgen, die die Animation steuert, wie zum Beispiel `animate()`, zusammen mit einer Funktion, die die Animation aktualisiert, traditionell `update()` genannt:

```js
function animate() {
 // Update the animation...
 // for example, move an element, change a style, and more.
 update();
 // Request the next frame
 requestAnimationFrame(animate);
}
```

Die `update()`-Funktion ist der Ort, an dem die Magie passiert. Innerhalb dieser Funktion können Sie alles ändern, was Sie animieren möchten. Zum Beispiel einen Stil aktualisieren oder die Position eines Elements verändern:

```js
function update() {
 element.style.transform = `translateX(${position}px)`;
 position += 2;
}
```

Was die Animation schließlich startet, ist der Aufruf von `requestAnimationFrame()` und die Übergabe der `animate`-Funktion, diesmal außerhalb der `animate`-Funktion:

```js
requestAnimationFrame(animate);
```

Die Schleife läuft weiter, bis Sie sie stoppen.

Schauen wir uns nun ein weiteres Beispiel an. Das HTML für dieses Beispiel ist ein `div`-Element mit dem Text `freeCodeCamp is Awesome`. Das CSS macht aus dem `div` ein Rechteck und versteckt alles, was links oder rechts außerhalb des Viewports liegt. Der JavaScript-Code bewegt das Rechteck `2px` bei jedem Aufruf von `requestAnimationFrame(animate)` mit der `animate`-Funktion als Callback nach rechts:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css" />
<div id="rect" class="rect">freeCodeCamp is Awesome</div>
<script src="index.js"></script>
```

```css
body {
  overflow-x: hidden;
}

.rect {
  display: flex;
  align-items: center;
  justify-content: center;
  width: 400px;
  height: 250px;
  border-radius: 5px;
  background-color: #1b1b32;
  color: #f5f6f7;
  font-size: 2rem;
  position: absolute;
}
```

```js
const rect = document.getElementById("rect");

let position = 0;

function update() {
  // Move the rectangle 2px to the right
  rect.style.left = position + "px";
  position += 2;

  if (position > window.innerWidth) {
    // Move the rectangle just outside the left side of the screen
    position = -rect.offsetWidth;
  }
}

function animate() {
  update();

  //request the next frame
  requestAnimationFrame(animate);
}

// Start the animation
requestAnimationFrame(animate);
```

:::

Das Ergebnis im Browser ist eine animierte Titelkarte, die über den Bildschirm schwebt.
