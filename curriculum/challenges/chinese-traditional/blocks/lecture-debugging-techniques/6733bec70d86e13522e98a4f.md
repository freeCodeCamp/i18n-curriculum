# --description--

JavaScript 中的 `throw` 述語用來丟出使用者自訂的例外。在程式設計中，例外是指發生了意外事件，導致程式的正常流程被中斷。

作為程式設計師，處理這些例外非常重要，這樣當錯誤發生時，你的程式才不會意外當掉。以下是 `throw` 述語的基本語法：

```js
throw expression;
```

這裡的 `expression` 是指代表你想丟出的例外的物件或值。這類例外的範例包括內建的例外類別，如 `Error`、`TypeError` 或 `RangeError` 類別。以下是使用 `throw` 述語丟出 `TypeError` 的範例：

```js
function validateNumber(input) {
  if (typeof input !== "number") {
    throw new TypeError("Expected a number, but received " + typeof input);
  }
  return input * 2;
}
```

在這個範例中，我們正在檢查 `input` 的型別是否不是 `number`。如果不是，則會丟出一個帶有自訂訊息的 `TypeError`。否則，函式會傳回將 `input` 乘以 `2` 的結果。

如果你想要發出更一般化的錯誤訊息，則你可以像這樣取用 `Error` 建構式：

```js
function divide(numerator, denominator) {
  if (denominator === 0) {
    throw new Error("Cannot divide by zero");
  }
  return numerator / denominator;
}
```

這是一個會檢查分母是否為 `0` 的函式（程式）範例。如果是這種情況，則會發出一個訂製的錯誤訊息，內容為 `Cannot divide by zero`。

在下一課，我們將探討如何在 `try`/`catch` 區塊的背景關係中丟出錯誤訊息，這是用來優雅地處理 JavaScript 中的例外。
