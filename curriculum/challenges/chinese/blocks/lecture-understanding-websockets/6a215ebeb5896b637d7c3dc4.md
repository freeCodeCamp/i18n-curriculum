# --description--

Node.js 没有内置的 WebSocket 服务器。开发者通常使用外部包，比如 `ws`，来实现 WebSocket 协议并处理客户和服务器之间的实时通信。

`ws` 库轻量、快速且易于使用。它实现了 WebSocket 协议，因此你可以创建和管理 WebSocket 连接。

你可以用 npm 安装它：

```bash
npm install ws
```

安装完成后，通过导入 `ws` 库，调用它的 `Server` 方法，并传入一个端口，来创建 WebSocket 服务器：

```js
const WebSocket = require("ws");

const server = new WebSocket.Server({
  port: 3000,
});
```

这会启动一个监听 `3000` 端口连接的 WebSocket 服务器。

当客户连接时，服务器会触发一个 `connection` 事件。

```js
server.on("connection", (socket) => {
  console.log("Client connected");
});
```

把这个事件想象成服务器在说“有新客户加入了”。

`socket` 参数代表服务器和该客户之间的连接。你可以用它来发送和接收消息。

当已连接的客户发送数据时，服务器会触发一个 `message` 事件。

```js
server.on("connection", (socket) => {
  socket.on("message", (message) => {
    console.log(message.toString());
  });
});
```

在这个例子中，服务器监听传入消息并将其记录到控制台。

你可以使用 `send()` 方法向客户发送数据：

```js
socket.send("Hello from the server");
```

这会通过打开的 WebSocket 连接发送消息。

只要连接保持打开，客户和服务器都可以继续实时交换消息。

除了 `connection` 和 `message`，WebSocket 连接还会触发 `close` 和 `error` 事件。

`close` 在连接关闭时触发：

```js
socket.on("close", () => {
  console.log("Disconnected");
});
```

`error` 在出现问题时触发：

```js
socket.on("error", (error) => {
  console.error(error);
});
```

这些事件共同构成了大多数 WebSocket 应用的基础。

聊天应用中一个常见需求是广播。广播意味着向多个已连接的客户发送消息，而不仅仅是一个。

想象一个有三个用户的聊天室，分别是 **John Doe**、**Jane Doe** 和 **Wes Smith**。

如果 John Doe 发送消息，服务器可能需要将该消息转发给 Jane Doe 和 Wes Smith。

为此，服务器会跟踪已连接的客户，并在消息到达时循环遍历它们。

发送消息前，服务器会检查客户连接是否仍然打开：

```js
if (client.readyState === WebSocket.OPEN) {
  // do something
}
```

每个 WebSocket 连接都有一个 `readyState` 属性，描述其当前状态。

`WebSocket.OPEN` 表示连接处于活动状态，准备好发送和接收数据。

通过检查这个，服务器避免向已经断开连接或连接仍在建立中的客户发送消息：

```js
server.on("connection", (socket) => {
  socket.on("message", (message) => {
    server.clients.forEach((client) => {
      if (client.readyState === WebSocket.OPEN) {
        client.send(message.toString());
      }
    });
  });
});
```

在这个例子中，每个已连接的客户都会收到传入消息。这就是广播。

许多实时应用使用广播来传递数据，比如聊天消息、实时通知、体育更新和多人游戏事件。

使用 `ws` 库，在 Node.js 中构建 WebSocket 服务器非常简单。

你只需创建服务器、监听事件、交换消息，并向已连接客户广播更新。

这些构建模块构成了许多实时应用的基础。
