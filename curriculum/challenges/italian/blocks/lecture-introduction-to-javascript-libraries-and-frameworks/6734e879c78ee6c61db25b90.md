# --description--

I componenti sono i mattoni fondamentali delle applicazioni React che permettono agli sviluppatori di suddividere interfacce utente complesse in parti più piccole e gestibili, facilitando lo sviluppo e la manutenzione di applicazioni su larga scala.

I due tipi di componenti in React sono quelli funzionali e quelli basati su classi. Nel React moderno, gli sviluppatori useranno componenti funzionali e tutti gli esempi che vedremo oggi saranno componenti funzionali.

A un livello più alto, puoi pensare ai componenti come a funzioni JavaScript che restituiscono elementi che descrivono l’interfaccia utente.

Questa interfaccia utente è descritta usando JSX, un’estensione di sintassi per JavaScript che assomiglia all’HTML ma ti permette di scrivere elementi UI in modo più dichiarativo.

Diamo un’occhiata a un esempio di componente React:

```jsx
function Greeting() {
  const name = "John"
  {/* The result will be Hello John*/}
  return <h1 className="title">Hello {name}</h1>;
}
```

In questo esempio, abbiamo definito un componente chiamato `Greeting`. Le parentesi graffe `{}` all’interno dei tag `h1` ci permettono di incorporare espressioni JavaScript, consentendoci di consultare la variabile `name` all’interno dell’elemento `h1`.

Stiamo anche applicando una `className` chiamata `title` all’elemento `h1`.

Ma perché usiamo `className` invece di `class` come con i normali elementi HTML?

Questo perché in JavaScript `class` è un nome riservato. Quindi, dobbiamo usare `className` invece.

Stiamo anche usando un commento in JSX che mostra quale sarà il risultato. È importante notare che puoi usare la sintassi dei commenti normale così, ma deve essere racchiusa tra parentesi graffe affinché funzioni:

```jsx
{/* Block Comments */}
```

Un’altra cosa che potresti aver notato è che usiamo una lettera maiuscola all’inizio del nome del componente. Ma perché non possiamo usare tutte lettere minuscole?

Questo perché React tratta i componenti con la lettera maiuscola come componenti personalizzati, mentre gli elementi con lettere minuscole sono considerati elementi HTML integrati.

Quando React incontra un tag minuscolo, come `<div>` o `<span>`, assume che sia un elemento HTML standard. Tuttavia, se il nome del componente inizia con una lettera maiuscola, React lo tratterà come un componente definito dall’utente e lo renderizzerà di conseguenza. Questa distinzione aiuta React a differenziare tra tag HTML nativi e componenti che crei tu.

Per usare questo componente `Greeting` nella nostra applicazione, scriveremmo qualcosa del genere:

```jsx
<Greeting />
```

Questo renderizzerebbe un elemento `h1` con il testo `Hello John` sulla pagina. Ma guarda più da vicino la sintassi qui. Quando usiamo il componente, termina con una barra obliqua e poi il simbolo maggiore.

Quando lavori con JSX, tutti i tag e gli usi dei componenti devono essere chiusi esplicitamente. Quindi se il componente o il tag non ha figli, devi chiuderlo esplicitamente come mostrato qui:

```jsx
<Greeting /> {/* /> is required */}
```

Finora abbiamo visto solo come renderizzare un singolo elemento `h1`. Ma in realtà puoi renderizzare più elementi.

Diamo un’occhiata al seguente esempio di codice:

```jsx
function Greeting() {
  const name = "John";
  {/* This will throw an error */}
  return <h1>Hello {name}</h1> 
  <p>Nice to meet you.</p>
}
```

Stiamo cercando di aggiungere un’altra frase di `Nice to meet you` ma non viene renderizzata correttamente sulla pagina.

Invece sembra esserci un messaggio di errore. Il messaggio di errore dice `Adjacent JSX elements must be wrapped in an enclosing tag.`

Il motivo per cui ricevi quel messaggio di errore è che più elementi fratelli devono essere racchiusi in un elemento genitore. Anche se potresti racchiudere gli elementi `h1` e `p` in un semplice `div`, c’è un altro modo per eliminare l’errore.

I frammenti React sono usati per raggruppare elementi insieme. Ecco come sarà l’esempio rivisto:

```jsx
function Greeting() {
  const name = "John";
  return (
    <Fragment>
      <h1>Hello {name}</h1>
      <p>Nice to meet you.</p>
    </Fragment>
  );
}
```

Puoi anche scegliere di usare tag JSX vuoti che possono servire come abbreviazione per i frammenti:

```jsx
function Greeting() {
  const name = "John";
  return (
    <>
      <h1>Hello {name}</h1>
      <p>Nice to meet you.</p>
    </>
  );
}
```
