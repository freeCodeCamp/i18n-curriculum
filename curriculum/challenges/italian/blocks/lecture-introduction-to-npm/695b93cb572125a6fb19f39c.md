# --description--

In una lezione precedente, hai imparato cosa sono le dipendenze e come scegliere quelle giuste per il tuo progetto.

In questa lezione, imparerai come installare e rimuovere le dipendenze usando npm.

Per installare una dipendenza, usa il comando `npm install` così:

```bash
npm install dependencyName
```

Puoi anche abbreviare `install` in `i` così:

```bash
npm i dependencyName
```

Dopo aver installato un pacchetto, npm aggiunge una nuova sezione `dependencies` al tuo file `package.json` (se non è già presente) e elenca il pacchetto che hai installato. Per esempio, ecco come appare quando installi il pacchetto `chalk`:

```json
{
  "name": "install-uninstall-deps",
  "version": "1.0.0",
  "description": "a simple demo on how to install and uninstall dependencies",
  "main": "index.js",
  "scripts": {
    "test": "echo \"Error: no test specified\" && exit 1"
  },
  "keywords": ["npm", "deps"],
  "author": "Kolade Chris",
  "license": "ISC",
  "dependencies": {
    "chalk": "^5.6.2"
  }
}
```

Quando installi una dipendenza, due nuove cose appaiono nella cartella principale del tuo progetto:

* `node_modules/`: una cartella che contiene il codice effettivo dei pacchetti che hai installato.
    
* `package-lock.json`: un file che conserva informazioni dettagliate sulle versioni esatte di quei pacchetti, comprese le loro dipendenze. (Ne parlerai meglio in una lezione successiva.)
    

<img src="https://cdn.freecodecamp.org/curriculum/lecture-transcripts/how-can-you-install-and-remove-dependencies-1.png" alt="">

La maggior parte dei progetti necessita di più di una dipendenza. Invece di installarle una alla volta, npm ti permette di installare più pacchetti contemporaneamente inserendo una lista di nomi di pacchetti separati da spazi:

```bash
npm install express lodash bcrypt
```

Come prima, quando installi più dipendenze insieme, npm aggiorna i tuoi file `package.json` e `package-lock.json` e il tuo `node_modules` per includerli. Ecco come apparirebbe il `package.json`:

```json
{
  "name": "install-uninstall-deps",
  "version": "1.0.0",
  "description": "a simple demo on how to install and uninstall dependencies",
  "main": "index.js",
  "scripts": {
    "test": "echo \"Error: no test specified\" && exit 1"
  },
  "keywords": ["npm", "deps"],
  "author": "Kolade Chris",
  "license": "ISC",
  "dependencies": {
    "bcrypt": "^6.0.0",
    "chalk": "^5.6.2",
    "express": "^5.1.0",
    "lodash": "^4.17.21"
  }
}
```

Per installare una `devDependency`, aggiungi la flag `--save-dev` al comando `install`. Puoi anche usare la versione più corta `-D`:

```bash
npm i nodemon -D
```

Dopo aver eseguito questo comando, npm aggiorna il tuo file `package.json` e aggiunge il pacchetto sotto la sezione `devDependencies`. Ecco un esempio di come appare:

```json
{
  "name": "install-uninstall-deps",
  "version": "1.0.0",
  "description": "a simple demo on how to install and uninstall dependencies",
  "main": "index.js",
  "scripts": {
    "test": "echo \"Error: no test specified\" && exit 1"
  },
  "keywords": ["npm", "deps"],
  "author": "Kolade Chris",
  "license": "ISC",
  "dependencies": {
    "bcrypt": "^6.0.0",
    "chalk": "^5.6.2",
    "express": "^5.1.0",
    "lodash": "^4.17.21"
  },
  "devDependencies": {
    "nodemon": "^3.1.10"
  }
}
```

Ogni volta che esegui un comando `npm install`, npm mostrerà un messaggio che ti dice se l’installazione è andata a buon fine o meno. Un messaggio di successo di solito appare così:

```bash

added 72 packages, and audited 74 in 8s

17 packages are looking for funding
  run `npm fund` for details

found 0 vulnerabilities

user@Kolade ~/Desktop/fCC/script-code/node/npm/install-uninstall-deps  % npm install nodemon -D

added 27 packages, and audited 101 in 16s

21 packages are looking for funding
  run `npm fund` for details

found 0 vulnerabilities
```

Per disinstallare o rimuovere un pacchetto, usa il comando `npm uninstall`:

```bash
npm uninstall dependencyName
```

Per esempio, per rimuovere `chalk` dal tuo progetto, esegui questo comando:

```bash
npm uninstall chalk
```

Puoi anche disinstallare più pacchetti contemporaneamente elencandoli con spazi tra ogni nome di pacchetto:

```bash
npm uninstall express lodash bcrypt
```

Riceverai anche un messaggio che ti mostra se il comando di disinstallazione è riuscito o fallito:

```bash
user@Kolade ~/Desktop/fCC/script-code/node/npm/install-uninstall-deps  % npm uninstall chalk

added 1 package, removed 1 package, and audited 101 packages in 1s

20 packages are looking for funding
  run `npm fund` for details

found 0 vulnerabilities

user@Kolade ~/Desktop/fCC/script-code/node/npm/install-uninstall-deps  % npm uninstall express lodash bcrypt

removed 70 packages, and audited 31 in 1s

4 packages are looking for funding
  run `npm fund` for details

found 0 vulnerabilities
```

E il tuo file `package.json` verrà aggiornato automaticamente:

```json
{
  "name": "install-uninstall-deps",
  "version": "1.0.0",
  "description": "a simple demo on how to install and uninstall dependencies",
  "main": "index.js",
  "scripts": {
    "test": "echo \"Error: no test specified\" && exit 1"
  },
  "keywords": ["npm", "deps"],
  "author": "Kolade Chris",
  "license": "ISC",
  "devDependencies": {
    "nodemon": "^3.1.10"
  }
}
```

Per disinstallare una `devDependency`, puoi semplicemente eseguire:

```bash
npm uninstall nodemon
```

Nota che non ti serve la flag `-D` quando disinstalli. npm rimuoverà automaticamente il pacchetto dalla sezione corretta, in questo caso `devDependencies`.

Dopo aver disinstallato la `devDependency`, il tuo `package.json` verrà aggiornato di nuovo:

```json
{
  "name": "install-uninstall-deps",
  "version": "1.0.0",
  "description": "a simple demo on how to install and uninstall dependencies",
  "main": "index.js",
  "scripts": {
    "test": "echo \"Error: no test specified\" && exit 1"
  },
  "keywords": ["npm", "deps"],
  "author": "Kolade Chris",
  "license": "ISC"
}
```
