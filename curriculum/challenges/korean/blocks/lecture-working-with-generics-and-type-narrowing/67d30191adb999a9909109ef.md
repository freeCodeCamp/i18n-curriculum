# --description--

사실 이전 수업에서 제네릭 타입을 본 적이 있습니다: `Array<string>`. 하지만 보통 함수에서 제네릭 타입을 사용합니다. 사실 제네릭은 함수의 타입 정의 동작을 제어하기 위해 함수에 제공하는 특별한 매개변수로 생각할 수 있습니다.

이전에 배열에서 무작위 값을 가져오는 함수 예제로 돌아가 보겠습니다:

```js
const getRandomValue = (array: string[]): string => {
  return array[Math.floor(Math.random() * array.length)];
}
```

우리 함수는 문자열 배열을 받아 문자열을 반환합니다. 그런데 숫자 배열을 전달하고 숫자를 반환하고 싶다면 어떻게 할까요?

유니언 타입을 사용할 수 있습니다:

```js
const getRandomValue = (array: (string|number)[]): (string|number) => {
  return array[Math.floor(Math.random() * array.length)];
}
const val = getRandomValue([1, 2, 4])
```

하지만 이 방법은 숫자 배열만 전달했음에도 TypeScript가 `val` 변수가 숫자 또는 문자열일 수 있다고 판단합니다.

대신 함수에 제네릭 타입을 정의할 수 있습니다:

```js
const getRandomValue = <T>(array: T[]): T => {
  return array[Math.floor(Math.random() * array.length)];
}
const val = getRandomValue([1, 2, 4])
```

새 예제에는 몇 가지 중요한 요소가 있습니다. 먼저 `<T>` 구문은 함수에 제네릭 타입 `T`을 정의하고 있음을 TypeScript에 알립니다. `T`는 제네릭 타입에 흔히 쓰이는 이름이지만, 실제로는 JavaScript 변수 관례 내에서 원하는 이름을 사용할 수 있습니다.

그다음, 배열 매개변수가 제네릭 타입에 맞는 값들의 배열임을, 반환값이 같은 타입의 단일 요소임을 TypeScript에 알려줍니다. 이 덕분에 우리가 전달한 배열이 숫자로만 구성되어 있으므로 TypeScript가 `val`가 숫자임을 올바르게 감지할 수 있습니다.

그런데 제어할 수 없는 함수는 어떻게 할까요? `document.querySelector()` 메서드를 살펴보겠습니다:

```js
const email = document.querySelector("#email");
console.log(email.value);
```

ID 선택자로 쿼리하기 때문에 TypeScript는 어떤 종류의 요소를 쿼리하는지 모릅니다. 어떤 종류의 `Element`인지는 알지만, 최상위 요소 인터페이스에 `value` 속성이 없기 때문에 접근하려 하면 오류가 발생합니다.

다행히 함수 호출에 타입 인수를 전달할 수 있습니다:

```js
const email = document.querySelector<HTMLInputElement>("#email");
console.log(email.value);
```

이번에는 함수 호출에서 각괄호 구문(`<HTMLInputElement>`)을 다시 사용하는 것을 주목하세요. 이는 TypeScript에 우리가 찾으려는 요소가 `input` 요소임을 알려주며, input 요소에는 value 속성이 있으므로 더 이상 오류가 발생하지 않습니다. 물론 `querySelector`는 요소를 찾지 못하면 `null`를 반환할 수 있고, 이 경우 TypeScript는 우리가 그 상황을 처리하지 않았기 때문에 컴파일러 오류를 발생시킵니다.

하지만 그 부분은 다음 수업에서 배우게 됩니다.
