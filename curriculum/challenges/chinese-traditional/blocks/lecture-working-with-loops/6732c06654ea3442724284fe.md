# --interactive--

在之前的課程中，你學會了如何使用 `for` 迴圈、`for...in` 迴圈和 `for...of` 迴圈。在本課程中，你將學習 `while` 迴圈和 `do...while` 迴圈。

只要條件為真，`while` 迴圈就會執行一個區塊的程式碼。以下是 `while` 迴圈的基本語法：

```js
while (condition) {
  // code block to be executed
}
```

條件會在執行程式碼區塊之前被檢查。如果條件為假，程式碼區塊將不會被執行。

當你不知道需要執行多少次該區塊的程式碼時，`while` 迴圈非常有用。以下是使用 `while` 迴圈的範例：

:::interactive_editor

```js
let counter = 0;
while(counter < 5) {
  console.log(counter);
  counter++;
}
```

:::

在這個範例中，我們有一個名為 `counter` 的變數，初始值為 `0`。只要 `counter` 的值小於 `5`，`while` 迴圈就會持續執行。在迴圈內，我們將 `counter` 的值記錄到主控台，然後將 `counter` 增加 `1`。

另一個類似於 `while` 迴圈的迴圈是 `do...while` 迴圈。以下是基本語法：

```js
do {
  // code block to be executed
} while (condition);
```

`do...while` 迴圈和 `while` 迴圈之間的一個主要差異是，`do...while` 迴圈會在檢查條件之前至少執行一次區塊程式碼。

如果條件為真，程式碼區塊將繼續執行。如果條件為假，程式碼區塊將停止執行。

這裡是一個使用 `do...while` 迴圈的範例：

:::interactive_editor

```js
let counter = 0;
do {
  console.log(counter);
  counter++;
} while (counter < 5);  
```

:::

在這個範例中，我們有一個名為 `counter` 的變數，初始值為 `0`。`do...while` 迴圈會將 `counter` 的值記錄到主控台，然後將 `counter` 增加 `1`。在執行完程式碼區塊後，它會檢查 `counter` 的值是否小於 `5`。如果是，迴圈將繼續執行。如果不是，迴圈將停止。

在大多數情況下，你可能會比起 `do...while` 迴圈更常使用 `while` 迴圈。然而，了解這兩種迴圈型別以及何時使用它們是很重要的。

## --text--

`while` 迴圈和 `do...while` 迴圈的主要差異是什麼？

## --answers--

`while` 迴圈至少執行一次程式碼區塊，而 `do...while` 迴圈則不會。
