## 알고리즘과 빅오 표기법

- **알고리즘**: 문제를 해결하거나 작업을 수행하기 위한 명확한 지침 집합입니다. 알고리즘은 유한한 단계 내에 종료되어야 하며 각 단계는 정확하고 명확해야 합니다.

- **빅오 표기법**: 입력 크기가 증가할 때 알고리즘의 최악의 성능 또는 성장률을 나타냅니다. 상수 요소와 하위 차수 항은 무시하고 입력 크기에 따른 자원 사용량 증가에 초점을 맞춥니다.

### 일반적인 시간 복잡도

- **O(1) - 상수 시간**: 입력 크기와 관계없이 알고리즘이 동일한 시간 내에 실행됩니다.

:::interactive_editor

```js
function checkEvenOrOdd(number) {
  if (number % 2 === 0) {
    return "Even";
  } else {
    return "Odd";
  }
}

console.log(checkEvenOrOdd(4)) // "Even"
console.log(checkEvenOrOdd(5)) // "Odd"
```

:::

- **O(log n) - 로그 시간**: 입력이 증가해도 시간이 천천히 증가합니다. 문제 크기를 분수 단위로 반복해서 줄이는 알고리즘(예: 이진 탐색)에서 흔히 나타납니다.

- **O(n) - 선형 시간**: 실행 시간이 입력 크기에 비례하여 증가합니다.

```js
for (const grade of grades) { // grades is an array
  console.log(grade);
}
```

- **O(n log n) - 로그 선형 시간**: 병합 정렬과 퀵 정렬 같은 효율적인 정렬 알고리즘의 일반적인 시간 복잡도입니다.

- **O(n²) - 이차 시간**: 실행 시간이 제곱에 비례하여 증가합니다. 중첩된 반복문에서 자주 나타납니다.

:::interactive_editor

```js
const n = 3;

for (let i = 0; i < n; i++) {
  for (let j = 0; j < n; j++) {
    console.log("Hello, World!");
  }
}
```

:::

### 공간 복잡도

- **O(1) - 상수 공간**: 입력 크기와 관계없이 동일한 메모리를 사용합니다.
- **O(n) - 선형 공간**: 메모리 사용량이 입력 크기에 비례하여 증가합니다.
- **O(n²) - 이차 공간**: 메모리 사용량이 입력 크기의 제곱에 비례하여 증가합니다.

## 문제 해결 기법

- **문제 이해하기**: 문제 설명을 여러 번 읽으세요. 입력, 예상 출력, 입력을 출력으로 변환하는 방법을 파악하세요.

- **의사 코드**: 언어에 독립적인 알고리즘 논리의 고수준 설명입니다. 일반적인 문장과 `IF`, `ELSE`, `FOR`, `WHILE` 같은 프로그래밍 구성 요소를 혼합해 사용합니다.

```md
GET original_string
SET reversed_string = ""
FOR EACH character IN original_string:
  ADD character TO THE BEGINNING OF reversed_string
DISPLAY reversed_string
```

- **경계 사례**: 알고리즘이 처리해야 하는 범위의 경계에 위치한 특정하고 유효한 입력입니다. 항상 경계 사례를 고려하고 테스트하세요.

## 배열

- **정적 배열**: 초기화 시 크기가 고정됩니다. 요소들은 인접한 메모리 위치에 저장됩니다. 프로그램 실행 중 크기를 변경할 수 없습니다.

- **동적 배열**: 프로그램 실행 중 자동으로 크기가 늘어나거나 줄어듭니다. 필요할 때 더 큰 배열로 자동 복사를 통해 크기 조절을 처리합니다.

### JavaScript에서 동적 배열 구현

:::interactive_editor

```js
let numbers = [3, 4, 5, 6];
// Access elements
console.log(numbers[0])  // 3

// Update elements
numbers[2] = 16
console.log([...numbers]);  // [3, 4, 16, 6]

// Add elements
numbers.push(7)
console.log([...numbers]);  // [3, 4, 16, 6, 7] 

// Add at a specific index
numbers.splice(3, 0, 15);
console.log([...numbers]);  // [3, 4, 16, 15, 6, 7]

// Remove elements
numbers.splice(2, 1);  // Remove at specific index
console.log([...numbers]);  // [3, 4, 15, 6, 7]

numbers.pop()   // Remove last element
console.log([...numbers]); // [3, 4, 15, 6]
```

:::

### 동적 배열의 시간 복잡도

- **접근**: O(1)
- **끝에 삽입**: 평균 O(1), 크기 조절 시 O(n)
- **중간에 삽입**: O(n)
- **삭제**: 중간에서 O(n), 끝에서 O(1)

## 스택

- **스택**: 후입선출(LIFO) 자료 구조입니다. 요소는 오직 맨 위에서만 추가되고 제거됩니다.

- **푸시 연산**: 스택 맨 위에 요소를 추가합니다. 시간 복잡도: O(1).

- **팝 연산**: 스택 맨 위에서 요소를 제거합니다. 시간 복잡도: O(1).

:::interactive_editor

```js
// Using JavaScript array as a stack

let stack = [];

// Push operations
stack.push(1);
stack.push(2);
stack.push(3);
console.log([...stack]);  // [1, 2, 3]

// Pop operations
let topElement = stack.pop();
console.log(topElement);  // 3
console.log([...stack]);  // [1, 2]
```

:::

## 큐

- **큐**: 선입선출(FIFO) 자료 구조입니다. 요소는 뒤에서 추가되고 앞에서 제거됩니다.

- **인큐 연산**: 큐 뒤에 요소를 추가합니다. 시간 복잡도: O(1).

- **디큐 연산**: 큐 앞에서 요소를 제거합니다. 시간 복잡도: O(1).

:::interactive_editor

```js
// Using JavaScript array as a queue
let queue = [];

// Enqueue operations
queue.push(1);
queue.push(2);
queue.push(3);
console.log([...queue]);  // [1, 2, 3]

// Dequeue operations
let firstElement = queue.shift();
console.log(firstElement);  // 1
console.log([...queue]);  // [2, 3]
```

:::

## 연결 리스트

- **연결 리스트**: 각 노드가 데이터와 다음 노드에 대한 참조를 포함하는 선형 자료 구조입니다. 노드들이 사슬처럼 연결되어 있습니다.

### 단일 연결 리스트

- **구조**: 각 노드는 데이터와 다음 노드에 대한 하나의 참조를 가집니다.
- **순회**: 머리에서 꼬리 방향으로만 이동할 수 있습니다.
- **머리 노드**: 리스트의 첫 번째 노드로, 보통 직접 접근 가능한 유일한 노드입니다.
- **꼬리 노드**: 리스트의 마지막 노드로, `null`를 가리킵니다.

### 연산과 시간 복잡도

- **처음에 삽입**: O(1)
- **끝에 삽입**: O(n) - 끝까지 순회해야 함
- **중간에 삽입**: O(n) - 위치까지 순회해야 함
- **처음에서 삭제**: O(1)
- **끝에서 삭제**: O(n) - 이전 노드를 찾기 위해 순회해야 함
- **중간에서 삭제**: O(n) - 노드를 찾기 위해 순회해야 함

### 이중 연결 리스트

- **구조**: 각 노드가 데이터와 두 개의 참조(다음 노드와 이전 노드)를 가집니다.
- **순회**: 양방향으로 이동할 수 있습니다.
- **메모리**: 추가 참조 때문에 단일 연결 리스트보다 더 많은 메모리를 사용합니다.

## 각 자료 구조를 사용할 때

- **배열(동적 배열)**: 순서가 있고 인덱스로 접근해야 하며 크기를 미리 모를 때
- **스택**: 후입선출 작업(실행 취소 기능, 표현식 평가, 백트래킹)에 적합
- **큐**: 선입선출 작업(작업 스케줄링, 너비 우선 탐색)에 적합
- **연결 리스트**: 처음에서 빈번한 삽입/삭제가 필요하고 크기를 모를 때, 임의 접근이 필요 없을 때
