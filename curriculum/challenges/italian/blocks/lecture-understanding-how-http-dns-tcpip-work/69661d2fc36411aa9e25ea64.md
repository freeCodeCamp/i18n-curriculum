# --description--

Impariamo a conoscere i codici di risposta HTTP comuni.

I codici di risposta HTTP sono codici numerici a tre cifre che i server web usano per comunicare con i client e far sapere cosa è successo con le loro richieste.

Forniscono informazioni sul fatto che la richiesta sia stata gestita con successo o se si è verificato un errore durante la gestione della richiesta.

In questa lezione vedremo alcuni dei più comuni.

## Codici di successo

Iniziamo con i **codici di successo**. Questi codici iniziano con il numero 2.

Il codice **200 OK** è il codice di risposta HTTP più comune. Significa che la richiesta è stata completata con successo e il server ha restituito le informazioni richieste dal client.

Il codice **201 Created** viene restituito dal server quando il client richiede la creazione di una risorsa e il processo viene completato con successo. Per esempio, se un nuovo account utente viene creato con successo, il server restituisce questo codice di risposta al client.

Il codice **204 No Content** viene restituito quando la richiesta è stata gestita con successo ma il server non ha contenuti da restituire. Questo può essere usato per processi che eliminano o aggiornano una risorsa in un database ma non hanno dati da restituire.

## **Codici di reindirizzamento**

Ora parliamo dei **codici di reindirizzamento**.

Questi codici vengono restituiti dal server quando la risorsa richiesta dal client è stata spostata in una posizione diversa o quando il client deve compiere ulteriori passaggi per completare la richiesta.

Questi codici iniziano con il numero 3.

Il codice **301 Moved Permanently** viene restituito quando la risorsa richiesta dal client è stata spostata in modo permanente in un’altra posizione. In questo caso, il server dovrebbe includere un URL della nuova posizione nella risposta e una volta che il client riceve la risposta, l’utente dovrebbe essere reindirizzato a questo URL.

Il codice **302 Found** indica che la risorsa richiesta dal client è stata spostata temporaneamente in un’altra posizione. Il server dovrebbe fornire il nuovo URL nella risposta e il client dovrebbe reindirizzare l’utente a questo URL. Tuttavia, l’URL originale potrebbe essere disponibile in futuro poiché il cambiamento è temporaneo. Questa è la differenza principale tra i codici di risposta 301 e 302.

Il codice **304 Not Modified** viene restituito dal server per comunicare al client che la risorsa richiesta non è stata modificata dall’ultima volta che è stata richiesta. Il server invia questo codice per evitare di inviare dati già inviati se il contenuto non è cambiato. È una tecnica di ottimizzazione delle prestazioni.

## Codici di errore client

Ora iniziamo a entrare nei **codici di errore**.

Gli errori possono essere causati dal client o dal server. Parleremo prima dei codici di errore client.

I codici di errore client vengono restituiti quando la richiesta non può essere completata a causa di un problema generato dal client.

Questi codici di errore iniziano con il numero 4.

Il codice **400 Bad Request** viene restituito quando il server non riesce a comprendere la richiesta inviata dal client. Questo può accadere quando il client invia una richiesta con sintassi errata, con parametri non validi o dati non validi.

Il codice **401 Unauthorized** è legato all’autenticazione. Indica che il client deve fornire determinate credenziali di autenticazione per accedere alla risorsa richiesta. Queste credenziali possono includere un nome utente e una password.

Il codice **403 Forbidden** viene restituito quando il client non è autorizzato ad accedere alla risorsa richiesta. La differenza tra 401 e 403 è che nel 403 il client non ha i permessi necessari per la risorsa specifica che sta richiedendo, anche se ha credenziali di autenticazione valide.

Il codice **404 Not Found** viene restituito quando la risorsa non può essere trovata sul server. Forse è stata eliminata o c’è un errore di battitura nell’URL che impedisce al server di trovare la risorsa. Il server potrebbe anche avere un problema temporaneo che causa questo codice di errore.

## Codici di errore server

Infine, abbiamo i **codici di errore server**. Questi codici vengono restituiti dal server quando la richiesta non può essere elaborata a causa di un problema lato server.

Il codice di errore **500 Internal Server Error** indica che si è verificato un errore imprevisto sul server. Questo errore può essere causato da una vasta gamma di problemi, che vanno da problemi di configurazione a problemi temporanei.

Il codice di errore **502 Bad Gateway** viene restituito quando il server stava cercando di comunicare con un altro server ma ha ricevuto una risposta non valida da quel server. Ci sono molte cause possibili per questo tipo di errore.

Il codice di errore **503 Service Unavailable** indica che il server non sarà in grado di gestire la richiesta temporaneamente, quindi il client dovrebbe riprovare più tardi. Cause comuni di questo errore includono attività di manutenzione e un alto volume di richieste che il server al momento non riesce a gestire.

I codici di risposta HTTP sono essenziali per comprendere la comunicazione tra client e server web. Capendo questi codici, puoi risolvere problemi ed eseguire il debug delle tue applicazioni web.
