# --description--

React 19 では、サーバーコンポーネントとサーバーアクションという2つの注目すべき新機能が導入されました。

このバージョン以降、Next.jsのようなそれらをすぐにサポートするフレームワークでは、サーバーコンポーネントがデフォルトになりました。

一方、サーバーアクションは、APIエンドポイントを必要とせずにサーバー上でフォーム処理を可能にする関数です。

サーバーアクションは次のようになります：

```js
"use server";

async function submitForm(formData) {
  const name = formData.get("name");
  return { message: `Hello, ${name}!` };
}
```

このサーバーアクションはフォームから`name`フィールドを抽出し、その名前に対する挨拶の文字列を返します。

サーバーアクションの状態管理を簡素化し、シンプルなフォームでクライアント側のJavaScriptを不要にするために、Reactチームはバージョン19で`useActionState`フックを導入しました。

このフックを詳しく見て、その動作を確認しましょう。

Reactのドキュメントでは、`useActionState`フックを「フォームアクションの結果に基づいて状態を更新できるフック」と説明しています。

しかし、これは`useActionState`フックがフォームだけで使えるという意味ではありません。アクションがあれば、ボタンクリックや他のイベントの管理にも使えます。

そして、`useActionState`はフックなので、サーバーコンポーネント内で使うことはできないことに注意してください。

`useActionState`フックの基本構文は次の通りです：

```js
const [state, action, isPending] = useActionState(actionFunction, initialState, permalink);
```

- `state`はアクションが返す現在の状態です。

- `action`はサーバーアクションを呼び出す関数です。

- `isPending`はアクションが現在実行中かどうかを示すブール値です。

- `actionFunction`パラメータはサーバーアクションそのものです。

- `initialState`はアクションが実行される前の状態の開始点を表すパラメータです。

- `permalink`はフォームが変更するユニークなページURLを含むオプションの文字列です。

`useActionState`フックを使うには、まずアクションが用意されている必要があります。前の例のアクションを少し変えて使いましょう：

```js
"use server";

export async function submitForm(_, formData) {
  const name = formData.get("name");

  const hour = new Date().getHours();
  let greeting;

  if (hour < 12) {
    greeting = "Good morning";
  } else if (hour < 18) {
    greeting = "Good afternoon";
  } else {
    greeting = "Good evening";
  }

  return { message: `${greeting}, ${name}` };
}
```

コンポーネント内では、`useActionState`フックをインポートし、他のフックと同様にコンポーネント本体のトップレベル（return文の前）で呼び出します。アクションもインポートしてください：

```jsx
"use client";

// Import the useActionState hook
import { useActionState } from "react";

// Import the submitForm action
import { submitForm } from "./actions/submitForm";

const Greeter = () => {

 // Initialize the hook
 const [state, submit, isPending] = useActionState(submitForm, {
   message: "",
 });

  return (
    <div className="flex flex-col items-center justify-center min-h-screen bg-gray-100 p-6">
      {/* Rest of component */}
    </div>
  );
};

export default Greeter;
```

少しスタイルを加えた完全なコードは次のようになります：

```jsx
"use client";

import { useActionState } from "react";
import { submitForm } from "./actions/submitForm";

const Greeter = () => {
  const [state, submit, isPending] = useActionState(submitForm, {
    message: "",
  });

  return (
    <div className="flex flex-col items-center justify-center min-h-screen bg-gray-100 p-6">
      <form
        action={submit}
        className="bg-white p-6 rounded-2xl shadow-md w-full max-w-md"
      >
        <h2 className="text-2xl text-center font-semibold text-gray-700 mb-4">
          Greet Someone
        </h2>

        <input
          type="text"
          name="name"
          placeholder="Enter your name"
          required
          className="w-full p-3 border border-gray-300 rounded-lg focus:outline-none focus:ring-2 focus:ring-green-400"
        />

        <button
          type="submit"
          disabled={isPending}
          className="w-full mt-4 p-3 bg-green-500 text-white font-semibold rounded-lg hover:bg-green-600 disabled:bg-gray-400 transition-all"
        >
          {isPending ? "Greeting..." : "Greet"}
        </button>

        {state.message && (
          <p className="mt-4 text-green-600 text-center font-medium">
            {state.message}
          </p>
        )}
      </form>
    </div>
  );
};

export default Greeter;
```

ブラウザでは、フォームのボタンが`Greet`から`Greeting...`に変わり、アクションが`isPending`している間、挨拶はフォームが送信された時間帯に応じて`Good morning, {name}`、`Good afternoon, {name}`、または`Good evening, {name}`と表示されます。

`useActionState`フックはフォーム外でも使えるとお伝えしましたね？

この例では、ボタンクリックでJSONPlaceholderから5人のユーザーを取得します：

```js
"use server";

export async function getUsers() {
  const res = await fetch(
    "https://jsonplaceholder.typicode.com/users?_start=0&_limit=5/"
  );
  return await res.json();
}
```

スタイル付きのUIは次の通りです：

```jsx
"use client";

import { useActionState } from "react";
import { getUsers } from "./actions/getUsers";

export default function FetchUsers() {
  const [users, fetchAction, isPending] = useActionState(getUsers, []);

  return (
    <div className="p-6 max-w-lg mx-auto">
      <button
        onClick={fetchAction}
        disabled={isPending}
        className="px-4 py-2 cursor-pointer bg-green-500 text-white rounded-lg hover:bg-green-600 disabled:bg-gray-400 font-bold"
      >
        {isPending ? "Fetching Users..." : "Fetch Users"}
      </button>

      <ul className="mt-4 space-y-2">
        {users.map((user) => (
          <li key={user.id} className="p-3 bg-gray-100 rounded-lg">
            <p className="font-semibold">{user.name}</p>
            <p className="text-sm text-gray-600">{user.email}</p>
          </li>
        ))}
      </ul>
    </div>
  );
}
```

ブラウザでは、ボタンをクリックしてもテキストが`Fetching Users...`に更新されないことがわかります。

これは、Reactがデータ取得とレンダリングを`isPending`状態よりも優先し、その過程で`isPending`をブロックしてエラーをスローするためです。

この問題を解決するには、アクションを`startTransition`でラップする必要があります：

```jsx
"use client";

// import startTransition from React
import { useActionState, startTransition } from "react";
import { getUsers } from "./actions/getUsers";

export default function FetchUsers() {
  const [users, fetchAction, isPending] = useActionState(getUsers, []);

  return (
    <div className="p-6 max-w-lg mx-auto">
      <button
        {/* wrap fetchAction in startTransition */}
        onClick={() => startTransition(() => fetchAction())}
        disabled={isPending}
        className="px-4 py-2 bg-green-500 font-bold cursor-pointer text-white rounded-lg hover:bg-green-600 disabled:bg-gray-400"
      >
        {isPending ? 'Fetching Users...' : 'Fetch Users'}
      </button>

      <ul className="mt-4 space-y-2">
        {users.map((user) => (
          <li key={user.id} className="p-3 bg-gray-100 rounded-lg">
            <p className="font-semibold">{user.name}</p>
            <p className="text-sm text-gray-600">{user.email}</p>
          </li>
        ))}
      </ul>
    </div>
  );
}
```

`startTransition`とは何か気になるかもしれませんが、これはReactに状態更新が低優先度で中断可能であることを伝える関数です。これにより、サーバーアクションのような非同期更新を処理しながらUIのレスポンスを保ちます。

これがフォーム内外で`useActionState`フックを使う方法です。

## --text--

Reactにおける`useActionState`フックの目的は何ですか？

## --answers--

フォーム送信、ボタンクリック、その他のイベント駆動の更新を含むアクションの結果に基づいて状態を管理します。

---

フォームが送信されたときだけ状態を更新し、ボタンクリックやカスタムイベントハンドラのような他のタイプのアクションはサポートしません。

### --feedback--

---
