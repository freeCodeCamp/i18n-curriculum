# --interactive--

이전에 시맨틱 마크업과 ARIA 속성이 콘텐츠를 접근 가능하게 만드는 방법을 배웠습니다. 실제 앱에서는 콘텐츠가 거의 정적이지 않고, 페이지가 JavaScript를 통해 동적으로 자주 업데이트됩니다. 이런 경우, 이러한 변경 사항이 HTML에도 반영되도록 하는 것이 중요합니다. 그래야 화면 읽기 프로그램이 업데이트된 상태를 정확히 사용자에게 전달할 수 있습니다. 그렇지 않으면 보조 도구에 의존하는 사용자는 변경된 사실을 전혀 알지 못하거나 오래되었거나 오해를 불러일으키는 정보를 받을 수 있습니다.

JavaScript가 요소의 상태를 변경하는 일반적인 상황과 ARIA 속성을 업데이트하여 HTML을 어떻게 접근 가능하게 유지할 수 있는지 살펴보겠습니다.

드롭다운 메뉴를 열고 닫는 버튼이 있다고 가정해 보세요:

:::interactive_editor

```html
<button id="menuButton" aria-expanded="false" aria-controls="menuList">
  Menu
</button>

<ul id="menuList" hidden>
  <li>List item 1</li>
  <li>List item 2</li>
</ul>
<script src="index.js"></script>
```

```js
const button = document.getElementById("menuButton");
const menu = document.getElementById("menuList");

button.addEventListener("click", () => {
  const expanded = button.getAttribute("aria-expanded") === "true";
  button.setAttribute("aria-expanded", String(!expanded));
  menu.hidden = expanded;
});
```

:::

HTML은 드롭다운 메뉴를 제어하는 버튼을 설정합니다. 버튼은 메뉴가 열려 있는지 여부를 나타내기 위해 `aria-expanded`를 사용하고, 메뉴의 `aria-controls`를 참조하여 메뉴와 연결하기 위해 `id`를 사용합니다.

JavaScript는 메뉴 버튼의 클릭을 감지합니다. 클릭되면 `aria-expanded`의 값을 토글하고 메뉴의 `hidden` 속성을 업데이트합니다. 이렇게 하면 메뉴의 가시성과 접근성 정보가 동기화되어, 시각 사용자와 화면 읽기 프로그램 사용자 모두 메뉴가 열렸는지 닫혔는지 알 수 있습니다.

앞으로 수업에서 이러한 ARIA 속성에 대해 더 배우게 됩니다. 지금은 JavaScript가 요소의 상태를 변경할 때 관련 ARIA 속성도 업데이트하여 보조 기술이 화면에 실제로 표시된 내용을 정확히 반영하도록 해야 한다는 점만 기억하세요.
