# --description--

Beim Entwerfen Ihrer Anwendungen haben Sie oft eine Reihe kleiner Funktionen, die jeweils für eine Sache zuständig sind. Dies wird als „Single-Responsibility-Prinzip“ bezeichnet. Wenn Sie eine Reihe kleiner Funktionen haben, ist es am besten, diese Funktionen zu testen, um sicherzustellen, dass alles wie erwartet funktioniert. In dieser Lektion schauen wir uns an, wie man mit dem beliebten Jest-Testframework einen Unit-Test erstellt.

In diesem Beispiel erstellen wir eine Funktion, die dafür zuständig ist, einen neu formatierten String zurückzugeben:

```js
export function getFormattedWord(str) {
  if (!str) return '';
  return str.charAt(0).toUpperCase() + str.slice(1);
}
```

Diese `getFormattedWord`-Funktion hat einen Parameter namens `str` und prüft zuerst, ob der `str` leer ist. Falls ja, wird ein leerer String zurückgegeben. Andernfalls wird ein neuer String zurückgegeben, bei dem der erste Buchstabe großgeschrieben ist. Wir exportieren diese Funktion, damit wir sie in einer Testdatei verwenden können.

In einer separaten `getFormattedWord.test.js`-Datei können wir einige Tests schreiben, um zu überprüfen, ob die Funktion das tut, was sie tun soll.

Ganz oben in der `getFormattedWord.test.js`-Datei müssen wir die Funktion zuerst so importieren:

```js
import { getFormattedWord } from "./getFormattedWord.js";
```

Dann müssen wir das Jest-Paket mit `npm i jest` installieren. Danach können wir einen Test hinzufügen, der überprüft, ob ein Wort, das aus Kleinbuchstaben besteht, `hello`, korrekt formatiert wird zu `Hello`, das mit einem Großbuchstaben `H` beginnt:

```js
test('capitalizes the first letter of a word', () => {
  expect(getFormattedWord('hello')).toBe('Hello');
});
```

Die `expect`-Funktion wird verwendet, um einen Wert zu testen. Sie wird mit einem Matcher kombiniert, einer Funktion, die überprüft, ob sich der Wert wie erwartet verhält. In diesem Fall ist der Matcher `toBe()`. Jest bietet eine Vielzahl von Matchern, mit denen Sie auf Wahrheitswerte, Strings, Zahlen und mehr testen können.

Eine Möglichkeit, Ihre Tests auszuführen, besteht darin, ein npm-Skript in Ihrer `package.json`-Datei hinzuzufügen. Hier ist ein Beispielskript für das Jest-Framework:

```js
"scripts": {
  "test": "jest"
},
```

Dann führen Sie einfach `npm run test` im Terminal aus, um Ihre Tests zu starten. Wenn Ihre Tests bestanden werden, sollten Sie im Terminal so etwas sehen:

```sh
PASS ./getFormattedWord.test.js
  ✓ capitalizes the first letter of a word (1 ms)

Test Suites: 1 passed, 1 total
Tests:       1 passed, 1 total
Snapshots:   0 total
Time:        0.103 s, estimated 1 s
Ran all test suites.
```

Um zu sehen, wie ein fehlschlagender Test aussieht, können wir den Test absichtlich durch folgende Änderung der Funktion zum Scheitern bringen:

```js
export function getFormattedWord(str) {
  if (!str) return '';
  return "This is incorrect";
}
```

Wenn Sie jetzt den Befehl `npm run test` ausführen, erscheint eine Fehlermeldung, weil der Test ein anderes Ergebnis erwartet hat:

```sh
FAIL ./getFormattedWord.test.js
  ✕ capitalizes the first letter of a word (1 ms)

  ● capitalizes the first letter of a word

  expect(received).toBe(expected) // Object.is equality

  Expected: "Hello"
  Received: "This is incorrect"

    2 |
    3 | test('capitalizes the first letter of a word', () => {
    4 |   expect(getFormattedWord('hello')).toBe('Hello');
                                            ^
    5 | });
    6 |

    at Object.toBe (getFormattedWord.test.js:4:37)

Test Suites: 1 failed, 1 total
Tests:       1 failed, 1 total
Snapshots:   0 total
Time:        0.121 s, estimated 1 s
Ran all test suites.
```

Jetzt können wir unsere Funktion hier wieder auf die ursprüngliche Version zurücksetzen:

```js
export function getFormattedWord(str) {
  if (!str) return '';
  return str.charAt(0).toUpperCase() + str.slice(1);
}
```

Fügen Sie einen weiteren Test hinzu, der Fälle überprüft, in denen kein Argument an `getFormattedWord` übergeben wird:

```js
test("returns an empty string when no argument is provided", () => {
  expect(getFormattedWord()).toBe("");
});
```

Es gibt weitere Tests, die Sie hinzufügen können, um die Robustheit zu erhöhen, aber diese ersten Tests sind eine gute Einführung ins Unit-Testing.

Wenn es um das Testen von JavaScript-Anwendungen geht, gehören Jest, Mocha und Vitest zu den gängigen Testframeworks.

Unit-Testing ist wichtig, weil es Ihnen hilft, mehr Bugs in Ihren Programmen zu finden und sicherzustellen, dass alles wie erwartet funktioniert. Es kann auch als eine Art Dokumentation für Ihre Anwendung dienen, da es das erwartete Verhalten Ihres Codes darstellen soll.
