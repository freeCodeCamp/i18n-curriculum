# --interactive--

El atributo `id` añade un identificador único a un elemento HTML. 

Aquí tienes un ejemplo de un elemento `h1` con un `id` de `title`. 

Debajo del elemento `h1`, agrega un elemento `h2` con un `id` establecido en `"subtitle"`. Puedes escribir el texto que quieras para el `h2` y verás los cambios en la ventana de vista previa. Para interactuar con el ejemplo, necesitarás habilitar el editor interactivo.

:::interactive_editor

```html
<h1 id="title">Movie Review Page</h1>

```

:::

Puedes hacer referencia al nombre `id` de `title` dentro de tu JavaScript o CSS. Aquí tienes un ejemplo de CSS que hace referencia al `id` de `title` para cambiar el `color` del texto a `red`.

**NOTA**: Se ha proporcionado algo de CSS para ti en este ejemplo interactivo. No te preocupes por tratar de entender el código CSS porque aprenderás más sobre eso en lecciones futuras. Pero si quieres ver el cambio del color del texto a azul, activa el editor interactivo, haz clic en la pestaña `styles.css` y cambia `color: red;` a `color: blue;`. 

:::interactive_editor

```html
<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="utf-8" />
    <meta
       name="viewport"
       content="width=device-width, initial-scale=1.0" />
    <title>Review page Example</title>
    <link rel="stylesheet" href="./styles.css" />
  </head>
  <body>
    <h1 id="title">Movie Review Page</h1>
  </body>
</html>
```

```css
#title {
  color: red;
}
```

:::

The hash symbol (`#`) in front of `title`, tells the computer you want to target an `id` with that value. `id` names should not be used more than once, and they should always be unique. Otra cosa a tener en cuenta acerca de los valores de `id` es que no pueden tener espacios en ellos. Aquí hay un ejemplo de aplicar las palabras `main` y `heading` a un valor de atributo `id`:

```html
<h1 id="main heading">Main heading</h1>
```

Los navegadores verán este espacio como parte del `id`, lo que causará problemas no deseados al momento de aplicar estilos y scripts. Los valores del atributo `id` solo deben contener letras, dígitos, guiones bajos y guiones.

En contraste con el atributo `id`, el valor del atributo `class` no necesita ser único y puede contener espacios. 

Aquí tienes un ejemplo de cómo aplicar una clase llamada `box` a un elemento `div`.

```html
<div class="box"></div>
```

Si quieres agregar múltiples nombres de clase a un elemento, puedes hacerlo separando los nombres con un espacio. Aquí tienes un ejemplo actualizado aplicando múltiples clases a un elemento `div`.

```html
<div class="box red-box"></div>
```

Aquí tienes otro ejemplo de cómo aplicar múltiples clases a varios elementos `div`. 

**NOTA**: Se ha proporcionado algo de CSS para ti en este ejemplo interactivo. No te preocupes por tratar de entender el código CSS porque aprenderás más sobre eso en lecciones futuras. Pero si quieres cambiar el color de las cajas primera y tercera, activa el editor interactivo y haz clic en la pestaña `styles.css` y cambia `background-color: red;` por `background-color: black;`.

:::interactive_editor

```html
<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="utf-8" />
    <meta
       name="viewport"
       content="width=device-width, initial-scale=1.0" />
    <title>Colored boxes example</title>
    <link rel="stylesheet" href="./styles.css" />
  </head>
  <body>
    <div class="box red-box"></div>
    <div class="box blue-box"></div>
    <div class="box red-box"></div>
    <div class="box blue-box"></div>
  </body>
</html>
```

```css
.box {
  width: 100px;
  height: 100px;
}

.red-box {
  background-color: red;
}

.blue-box {
  background-color: blue;
}
```

:::

Entonces, para recapitular, ¿cuándo debes usar un `id` en lugar de una `clase`? Las clases son mejores para usar cuando quieres aplicar un conjunto de estilos a muchos elementos. Si quieres apuntar a un elemento específico, es mejor usar un `id` porque esos valores deben ser únicos.

## --answers--

Utiliza una `clase` cuando quieras un identificador único que sólo deba aplicarse a un elemento.
