# --description--

ISBN(국제 표준 도서 번호)는 상업용 도서에 할당되는 고유 식별자입니다. 10자리 또는 13자리 숫자일 수 있으며, 마지막 숫자는 나머지 숫자들로부터 계산된 검증 숫자입니다.

Camperbot이 자체 ISBN 검증기를 만들려고 했지만, 몇 가지 실수를 했습니다.

이 실습에서는 기존 코드를 고쳐 제대로 작동하게 만드세요.

예상 동작:
사용자가 프로그램을 실행하면 `Enter ISBN and length: ` 프롬프트가 표시됩니다. 사용자는 `ISBN,length` 형식으로 검증하려는 ISBN 코드를 입력할 수 있습니다. ISBN 코드는 하이픈 없이 입력하며, 그 뒤에 길이(`10` 또는 `13`)를 콤마로 구분해 입력합니다.

입력 예시:
ISBN-10 코드의 경우 `1530051126,10`.
ISBN-13 코드의 경우 `9781530051120,13`.

검증 숫자 찾는 방법:

- 이 실습에서는 상세 계산 로직을 알 필요 없습니다. `calculate_check_digit_10` 함수와 `calculate_check_digit_13` 함수가 계산을 처리해 문자열로 된 예상 검증 숫자를 반환합니다.
- ISBN-10 코드의 검증 숫자는 `0`부터 `9`까지 숫자 또는 대문자 `X`일 수 있습니다.
- ISBN-13 코드의 검증 숫자는 `0`부터 `9`까지 숫자일 수 있습니다.

**목표:** 아래 사용자 스토리를 모두 충족하고 모든 테스트를 통과해 실습을 완료하세요.

**사용자 스토리:**

1. 현재 코드에서 `IndentationError`을 고쳐야 합니다.
1. 사용자가 콤마로 구분된 값을 입력하지 않아도 프로그램이 `IndexError`를 처리해 크래시가 나지 않아야 합니다.
1. 사용자가 콤마로 구분된 값을 입력하지 않으면 콘솔에 `Enter comma-separated values.` 메시지가 표시되고 프로그램이 종료되어야 합니다.
1. 사용자가 길이에 숫자가 아닌 값을 입력해도 프로그램이 `ValueError`를 처리해 크래시가 나지 않아야 합니다.

1. `validate_isbn` 함수에서 경계값 오류를 고쳐야 합니다.
1. 사용자가 유효한 ISBN 코드를 입력할 때 발생하는 현재 코드의 `TypeError`를 고쳐야 합니다.

    if len(isbn) != length:
        print(f'ISBN-{length} code should be {length} digits long.')
        return

    check_digit_index = length - 1

    main_digits = isbn[0:check_digit_index]
    given_check_digit = isbn[check_digit_index]

    try:
        main_digits_list = [int(digit) for digit in main_digits]
