# --interactive--

前のレッスンでは、`addEventListener()` メソッドを使ってイベントを扱う方法を学びました。しかし、JavaScriptでイベントを扱う際に推奨されない別の方法もあります。

インラインイベントハンドラは、HTML要素に付けられる特別な属性で、イベントが発生したときにJavaScriptコードを実行するために使われます。

以下は、インラインのクリックイベントハンドラを持つ `button` 要素の例です。

:::interactive_editor

```html
<button onclick="alert('Hello World!')">Show alert</button>
```

:::

ユーザーがボタンをクリックすると、`alert` 関数が呼び出され、メッセージ `Hello World!` を表示するアラートダイアログが出ます。

インラインイベントハンドラの別の使い方として、HTMLドキュメント内の `script` タグで定義された関数を呼び出す方法があります。

以下は、`changeBgColor` という関数を定義し、インラインのクリックイベントハンドラから呼び出す例です。

:::interactive_editor

```html
<script>
  function changeBgColor() {
    document.body.style.backgroundColor = "lightblue";
  }
</script>

<button onclick="changeBgColor()">Change background color</button>
```

:::

ユーザーがボタンをクリックすると、`changeBgColor` 関数が呼び出され、ページの背景色がライトブルーに変わります。このようにインラインイベントハンドラを使うことは可能ですが、ベストプラクティスとは見なされていません。

その理由の一つは、インラインイベントハンドラは1つの要素に対して1つのイベントリスナーしか付けられないことです。同じ要素に複数のイベントリスナーを付けたい場合は、`addEventListener()` を使う必要があります。もう一つの理由は、インラインイベントハンドラはHTMLとJavaScriptのコードを混ぜてしまうため、コードの読みやすさや保守性が低下することです。HTMLコードとJavaScriptコードは分けて、`addEventListener()` を使ってイベントリスナーを要素に付けるほうが良いです。

インラインイベントハンドラは現代のJavaScriptでは推奨されていません。ですので、JavaScriptでイベントを扱う際は`addEventListener()` メソッドを使うのがベストプラクティスです。
