# --description--

Cuando trabajas en el terminal, los editores de texto estándar no siempre están disponibles, pero existen programas diseñados para editar archivos directamente desde el terminal.

Los entornos Linux típicamente incluyen un editor de texto. Vi, o Vim, suelen ser los más comunes. Emacs y Nano pueden necesitar ser instalados manualmente, pero están disponibles para casi todas las distribuciones.

El desafío con Vim es que tiene una curva de aprendizaje considerable. Si no estás familiarizado con él, ¡puede que ni siquiera sepas cómo cerrar la aplicación! Emacs es posiblemente un poco más amigable para el usuario, pero aún así tiene su propio tomo de atajos de teclado.

Nano es considerablemente más sencillo y fácil de usar, especialmente para quienes están aprendiendo. Puede ser mucho más rápido comenzar y editar un archivo con Nano, y la aplicación incluso ofrece ayuda en pantalla para los atajos básicos del teclado.

La desventaja de esto, sin embargo, es la falta de extensibilidad. A diferencia de Vim o Emacs, que pueden extenderse con plugins para convertirse en entornos completos de desarrollo, Nano es bastante básico.

Echemos un vistazo a lo que Nano puede hacer. Para abrir un archivo, puedes escribir `nano <filename>` en la terminal.

Una vez que un archivo esté abierto, verás la versión de Nano que estás usando y el nombre del archivo que tienes abierto en la parte superior de la pantalla.

Debajo de eso está el contenido del archivo. Puedes usar las teclas de flecha para mover el cursor e insertar o eliminar texto con tu teclado. El archivo se desplazará según sea necesario para seguir.

Luego, en la parte inferior, verás una lista de atajos de teclado disponibles, que se ve algo así:

```bash
^G Help   ^O Write Out    ^F Where Is   ^K Cut      ^T Execute  ^C Location     M-U Undo    M-A Set Mark    M-] To Bracket    M-B Previous
^X Exit   ^R Read File    ^\ Replace    ^U Paste    ^J Justify  ^/ Go To Line   M-E Redo    M-6 Copy        ^B Where Was      M-F Next
```

Los atajos que comienzan con el símbolo caret (`^`) indican que necesitas mantener presionada la tecla Ctrl y presionar la letra indicada. Los atajos que comienzan con `M-` requieren que mantengas presionada la "meta key", que en la mayoría de las configuraciones debería ser la tecla Alt, y presiones la letra.

Para editar un archivo en Nano, solo necesitas conocer algunos de los comandos de atajo. Después de hacer cambios en un archivo, puedes presionar `Ctrl + O` para "Write Out", o guardar, el archivo. Después de ingresar el comando, verás que el menú en la parte inferior cambia a esto:

```bash
File Name to write : <filename>
^G Get Help                 ^T  To Files
^C Cancel                   TAB Complete
```

Presiona enter para guardar el archivo y volver a editar el archivo, o `Ctrl + C` para cancelar y volver a editar el archivo.

Mientras estás en el editor, puedes presionar `Ctrl + X` para salir de Nano y volver al terminal. Si tienes cambios sin guardar al intentar salir, verás esto en el menú inferior:

```bash
Save modified buffer (ANSWERING "No" WILL DESTROY CHANGES) ?
                            Y Yes
^C Cancel                   N No
```

Presiona `Y` para guardar tus cambios no guardados, o `N` para descartarlos.

Con este conocimiento, deberías estar mejor preparado la próxima vez que necesites editar rápidamente un archivo en un servidor remoto, donde quizás no tengas acceso a un entorno gráfico completo.
