# --description--

Das Schlüsselwort `let` ist nicht die einzige neue Möglichkeit, Variablen zu deklarieren. In ES6 können Sie Variablen auch mit dem Schlüsselwort `const` deklarieren.

`const` hat alle großartigen Funktionen, die `let` bietet, mit dem zusätzlichen Vorteil, dass Variablen, die mit `const` deklariert werden, schreibgeschützt sind. Sie sind ein konstanter Wert, was bedeutet, dass eine Variable, die einmal mit `const` zugewiesen wurde, nicht erneut zugewiesen werden kann:

```js
const FAV_PET = "Cats";
FAV_PET = "Dogs";
```

Die Konsole zeigt einen Fehler an, weil der Wert von `FAV_PET` erneut zugewiesen wird.

Sie sollten Variablen, die Sie nicht neu zuweisen möchten, immer mit dem Schlüsselwort `const` benennen. Das hilft, wenn Sie versehentlich versuchen, eine Variable neu zuzuweisen, die konstant bleiben soll.

**Hinweis:** Es ist üblich, dass Entwickler für unveränderliche Werte Großbuchstaben als Variablennamen verwenden und Kleinbuchstaben oder CamelCase für veränderliche Werte (Objekte und Arrays). Mehr über Objekte, Arrays sowie unveränderliche und veränderliche Werte lernen Sie in späteren Aufgaben. Auch werden Sie dort Beispiele für Großbuchstaben-, Kleinbuchstaben- oder CamelCase-Variablennamen sehen.

# --instructions--

Ändern Sie den Code so, dass alle Variablen mit `let` oder `const` deklariert werden. Verwenden Sie `let`, wenn sich die Variable ändern soll, und `const`, wenn die Variable konstant bleiben soll. Benennen Sie außerdem Variablen, die mit `const` deklariert wurden, um, damit sie den gängigen Praktiken entsprechen. Ändern Sie nicht die Strings, die den Variablen zugewiesen sind.
