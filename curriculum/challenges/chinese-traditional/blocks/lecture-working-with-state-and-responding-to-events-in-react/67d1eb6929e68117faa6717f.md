# --description--

在 React 中，算繪是組件出現在使用者介面（UI）中的處理程序，通常是在瀏覽器中。

React 會將你所有的 JavaScript、JSX 和 CSS 程式碼整合，判斷它應該呈現的樣子，然後在使用者介面中顯示出來。

呈現處理程序包含三個階段：觸發、呈現和提交。讓我們更詳細地看看這些階段。

當 React 偵測到某些東西已改變且使用者介面可能需要更新時，會發生觸發階段。這種變化通常是由於 state 或 props 的更新。

例如，注意到該是晚餐時間可以觸發你進入廚房開始烹飪。

在下面的 `Counter` 範例中，點擊遞增或遞減按鈕會觸發 React 顯示新的 `count` 值：

```jsx
import { useState } from "react";

function Counter() {
  const [count, setCount] = useState(0);

  return (
    <div>
      <h1>{count}</h1>
      <button onClick={() => setCount(count - 1)}>Decrement</button>
      <button onClick={() => setCount(count + 1)}>Increment</button>
    </div>
  );
}

export default Counter;
```

一旦觸發發生，React 就會進入呈現階段。在這裡，React 會重新評估你的組件並計算要呈現的內容。

為此，React 使用「真實」DOM 的輕量複本。這稱為虛擬 DOM。透過虛擬 DOM，React 可以快速檢查組件中需要變更的部分。

將此階段視為你在廚房，已經準備好食材，並開始烹飪晚餐的時刻。

對於 `Counter` 組件，呈現階段是 React 使用新的 `count` 值再次執行函式的時刻。React 根據更新後的 `count` 值重新計算組件中 `<h1>{count}</h1>` 這部分應該呈現的樣子，但你不會在螢幕上看到任何變化，直到下一個階段－提交。

提交階段是 React 從虛擬 DOM 取得準備好的變更並將它們套用到真實 DOM 的階段。換句話說，這是你在螢幕上看到最終結果的階段。

為了達成這個目的，React 會比較虛擬 DOM 與實際 DOM，識別出只需要更新的零件，並將這些變更套用到真實 DOM，以更新使用者介面。

你可以將此階段視為你端出烹飪好的食物的時刻，使其可見，就像 React 在提交更新到實際 DOM 時所做的那樣。

至於 `Counter` 組件，提交階段是將新的 `count` 值套用到 `h1` 元素的時刻，你可以在頁面上看到變更。

這三個處理程序非常快速，因為 React 先在虛擬 DOM 中計算變更，然後只更新實際 DOM 中需要變更的零件。
