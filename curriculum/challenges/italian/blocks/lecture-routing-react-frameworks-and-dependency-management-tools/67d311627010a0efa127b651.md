# --description--

Nelle lezioni precedenti, hai imparato che React è un'applicazione a pagina singola. Le applicazioni a pagina singola sono applicazioni che contengono un solo file HTML e usano JavaScript per aggiornare dinamicamente qualsiasi contenuto sulla pagina.

Ma cosa succede quando devi aggiungere più "pagine" alla tua applicazione React? Come puoi navigare tra queste diverse viste?

Ecco dove entra in gioco React Router.

React Router è una libreria di terze parti che ti permette di aggiungere il routing alle tue applicazioni React. Per cominciare, dovrai installare React Router in un progetto React esistente così:

```sh
npm i react-router
```

Se controlli il file `package.json`, vedrai che `react-router` è stato aggiunto alla lista delle dipendenze:

```json
"dependencies": {
  "react": "^18.3.1",
  "react-dom": "^18.3.1",
  "react-router": "^7.2.0"
}
```

Poi, all'interno del tuo file `main.jsx` o `index.jsx`, dovrai importare `BrowserRouter` e renderizzare `BrowserRouter` attorno al componente `App`:

```jsx
import { StrictMode } from "react";
import { createRoot } from "react-dom/client";
import { BrowserRouter } from "react-router";
import App from "./App.jsx";

import "./index.css";

createRoot(document.getElementById("root")).render(
  <StrictMode>
    <BrowserRouter>
      <App />
    </BrowserRouter>
  </StrictMode>
);
```

Per abilitare le rotte nella tua applicazione, dovrai aggiornare la tua istruzione `import` per includere i componenti `Routes` e `Route` così:

```js
import { BrowserRouter, Routes, Route } from "react-router";
```

Poi, all'interno di `BrowserRouter`, aggiungi i componenti `Routes` e `Route`:

```jsx
createRoot(document.getElementById("root")).render(
  <StrictMode>
    <BrowserRouter>
      <Routes>
        <Route path="/" element={<App />} />
      </Routes>
    </BrowserRouter>
  </StrictMode>
);
```

I componenti `path` e `element` sono usati per collegare insieme l'URL e i componenti dell'interfaccia utente. In questo caso, stiamo configurando una rotta per la homepage che punta al componente `App`.

È comune in applicazioni più grandi avere più viste e rotte configurate in questo modo:

```jsx
<Routes>
  <Route index element={<Home />} />
  <Route path="about" element={<About />} />

  <Route path="products">
    <Route index element={<ProductsHome />} />
    <Route path=":category" element={<Category />} />
    <Route path=":category/:productId" element={<ProductDetail />} />
    <Route path="trending" element={<Trending />} />
  </Route>
</Routes>
```

La prop `index` in questi esempi serve a rappresentare la rotta predefinita per un dato segmento di percorso. Quindi il componente `Home` sarà mostrato al percorso radice (`/`) mentre il componente `ProductsHome` sarà mostrato al percorso `products`.

Avrai anche notato che stiamo annidando alcune rotte dentro un'altra rotta così:

```jsx
<Route path="products">
  <Route path="trending" element={<Trending />} />
</Route>
```

Questo significa che il percorso della rotta figlio sarà aggiunto in coda al percorso della rotta genitore. Quindi in questo esempio, il `path` per i prodotti di tendenza sarà `products/trending`.

Se il `path` inizia con due punti (`:`) allora rappresenta un segmento dinamico nella rotta:

```jsx
<Route path=":category" element={<Category />} />
```

In questo esempio abbiamo un segmento dinamico chiamato `category`. Quando un utente naviga a un URL come `products/brass-instruments`, la vista cambierà al componente `Category` e potrai recuperare dinamicamente i dati appropriati basandoti sul segmento.

Puoi consultare il valore del segmento dinamico usando il hook `useParams` all'interno del componente figlio così:

```jsx
import { useParams } from "react-router";

export default function Category() {
  let params = useParams();
   {/* Accessing the category param: params.category */}
   {/* rest of code goes here */}
}
```

Le rotte dinamiche sono utili perché ti permettono di creare componenti flessibili e riutilizzabili che possono renderizzare contenuti diversi basandosi sui parametri nell'URL. Invece di definire una lista fissa di percorsi per ogni possibile rotta, puoi usare segmenti dinamici per renderizzare vari contenuti in base a ciò che l'utente ha richiesto.
