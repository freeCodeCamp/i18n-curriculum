# --interactive--

在之前的模块中，你已经习惯于使用像这样的字串字面：

```js
const greeting = "Hello, World!";
```

但是 JavaScript 也有字符串对象。字符串对象和字符串原语都用于处理文本，但它们在底层的工作方式不同。字符串对象是使用字符串构造函数创建的，它将原语值包装在一个对象中。以下是你如何创建字符串对象：

:::interactive_editor

```js
const greetingObject = new String("Hello, World!");

console.log(typeof greetingObject); // "object"
```

:::

当我们使用 `typeof` 操作符时，我们可以看到结果是对象类型而不是字串类型。你可能一直在想的其中一件事是，如何在字串原语上使用像 `.length` 这样的属性。

当你在字符串原语上使用 `length` 属性时，JavaScript 会临时将字符串原语包装在字符串对象中，以执行该操作。这就是为什么你可以使用 `length` 属性和不同的方法，如 `repeat()`、`concat()` 和 `slice()`。这类方法和属性被称为实例方法、实例属性和静态方法。你将在后续模块中详细了解其工作原理。

字符串对象和字符串原语之间的一个关键区别在于它们与内存和性能的关系。字符串原语通常比字符串对象更高效且更快。虽然你主要在代码中使用字符串原语，但了解字符串对象的工作原理仍然很有用。
