## Reactでのフォーム操作

- **制御された入力**: これは入力フィールドの値を状態に保存し、`onChange` イベントを通じて更新する方法です。これによりフォームデータを完全にコントロールでき、即時のバリデーションや条件付きレンダリングが可能になります。

```jsx
import { useState } from "react";

function App() {
  const [name, setName] = useState("");

  const handleChange = (e) => {
    setName(e.target.value);
  };

  const handleSubmit = (e) => {
    e.preventDefault();
    console.log(name);
  };

  return (
    <>
      <form onSubmit={handleSubmit}>
        <label htmlFor="name">Your name</label> <br />
        <input value={name} id="name" onChange={handleChange} type="text" />
        <button type="submit">Submit</button>
      </form>
    </>
  );
}

export default App;
```

- **非制御入力**: `useState` フックを使って入力を扱う代わりに、HTMLの非制御入力はDOMの助けを借りて独自の内部状態を維持します。DOMが入力値を管理するため、`ref` を使って入力フィールドの値を取得する必要があります。

```jsx
import { useRef } from "react";

function App() {
 const nameRef = useRef();

 const handleSubmit = (e) => {
   e.preventDefault();
   console.log(nameRef.current.value);
 };

 return (
   <form onSubmit={handleSubmit}>
     <label htmlFor="name">Your</label>{" "}
     <input type="text" ref={nameRef} id="name" />
     <button type="submit">Submit</button>
   </form>
 );
}

export default App;
```

## `useActionState` フックの使い方

- **サーバーアクション**: これはサーバー上で実行される関数で、APIエンドポイントを必要とせずにサーバー上でフォーム処理を可能にします。Next.jsアプリケーションの例を示します。

```js
"use server";

async function submitForm(formData) {
 const name = formData.get("name");
 return { message: `Hello, ${name}!` };
}
```

`"user server"` ディレクティブは関数をサーバーアクションとしてマークします。

- **`useActionState` フック**: このフックはフォーム送信の結果に基づいて状態を更新します。`useActionState` フックの基本構文は以下の通りです。

```js
const [state, action, isPending] = useActionState(actionFunction, initialState, permalink);
```

- `state` はアクションが返す現在の状態です。
- `action` はサーバーアクションをトリガーする関数です。
- `isPending` はアクションが現在実行中かどうかを示すブール値です。
- `actionFunction` パラメータはサーバーアクションそのものです。
- `initialState` はアクション実行前の状態の開始点を表すパラメータです。
- `permalink` はフォームが変更するユニークなページURLを含むオプションの文字列です。

## Reactでのデータ取得

- **データ取得の選択肢**: Reactでデータを取得する方法は多様です。ネイティブのFetch APIやAxios、SWRなどのサードパーティツールを使えます。
- **データ取得時によく使う状態変数**: Reactでどの方法を選んでも、追跡すべき状態がいくつかあります。まずはデータそのもの。次にデータがまだ取得中かどうかを追跡する状態。そしてデータ取得中に発生した可能性のあるエラーを捕捉する状態変数です。

```js
const [data, setData] = useState(null);
const [loading, setLoading] = useState(true);
const [error, setError] = useState(null);
```

データ取得は副作用なので、`Fetch API` を `useEffect` フック内で使うのが最適です。

```js
useEffect(() => {
  const fetchData = async () => {
    try {
      const res = await fetch("https://jsonplaceholder.typicode.com/posts");
      
      if (!res.ok) {
        throw new Error("Network response was not ok");
      }

      const data = await res.json();
      setData(data);
    } catch (err) {
      setError(err);
    } finally {
      setLoading(false);
    }
  };

  fetchData();
}, []);
```

データ取得が完了していなければローディングメッセージを、エラーがあればエラーメッセージを、そうでなければ結果をレンダリングできます。

```jsx
if (loading) {
  return <p>Loading...</p>;
}

if (error) {
  return <p>{error.message}</p>;
}

return (
  <ul>
    {data.map((post) => (
      <li key={post.id}>{post.title}</li>
    ))}
  </ul>
);
```

Axiosを使いたい場合は、インストールしてインポートしてください。

```bash
npm i axios
```

```js
import axios from "axios";
```

次に`axios.get` を使ってデータを取得できます。

```jsx
const [data, setData] = useState(null);
const [loading, setLoading] = useState(true);
const [error, setError] = useState(null);

useEffect(() => {
  const fetchData = async () => {
    try {
      const res = await axios.get(
        "https://jsonplaceholder.typicode.com/users"
      );
      setData(res.data);
    } catch (err) {
      setError(err);
    } finally {
      setLoading(false);
    }
  };

  fetchData();
}, []);
```

`useSWR` フックを使ってデータを取得するには、まずインストールしてインポートする必要があります。

```bash
npm i swr
```

```js
import useSWR from "swr";
```

このフックを使ってデータを取得する方法は以下の通りです。

```jsx
import useSWR from "swr";

const fetcher = (url) => fetch(url).then((res) => res.json());

const FetchTodos = () => {
 const { data, error } = useSWR(
   "https://jsonplaceholder.typicode.com/todos",
   fetcher
 );

 if (!data) {
   return <h2>Loading...</h2>;
 }
 if (error) {
   return <h2>Error: {error.message}</h2>;
 }

 return (
   <>
     <h2>Todos</h2>
     <div>
       {data.map((todo) => (
         <h3 key={todo.id}>{todo.title}</h3>
       ))}
     </div>
   </>
 );
};

export default FetchTodos;
```

## `useOptimistic` フックの使い方

- **`useOptimistic` フック**: このフックは非同期アクションがバックグラウンドで完了するのを待つ間、UIをレスポンシブに保つために使います。これは「楽観的更新」を管理する手法で、サーバーからのレスポンスを待つ間に期待される結果に基づいて即座にUIを更新します。

基本構文は以下の通りです。

```js
const [optimisticState, addOptimistic] = useOptimistic(actualState, updateFunction);
```

- `optimisticState` はユーザー体験向上のために即座に更新される一時的な状態です。
- `addOptimistic` は実際の状態変更前に楽観的更新を適用する関数です。
- `actualState` はサーバーからのデータ取得などアクションの結果から得られる実際の状態値です。
- `updateFunction` は呼び出されたときに楽観的状態をどのように更新するか決める関数です。

`useOptimistic` フックを `TaskList` コンポーネントで使う例は以下の通りです。

```jsx
"use client";

import { useOptimistic } from "react";

export default function TaskList({ tasks, addTask }) {
  const [optimisticTasks, addOptimisticTask] = useOptimistic(
    tasks,
    (state, newTask) => [...state, { text: newTask, pending: true }]
  );

  async function handleSubmit(e) {
    e.preventDefault();
    const formData = new FormData(e.target);

    addOptimisticTask(formData.get("task"));

    addTask(formData);
    e.target.reset();
  }

  return <>{/* UI */}</>;
}
```

- **`startTransition`**: これはUIの一部をレンダリングし、状態更新を非緊急のトランジションとしてマークします。これにより高負荷の更新中でもUIがレスポンシブになります。基本構文は以下の通りです。

```js
startTransition(action);
```

`action` は状態更新やトランジション関連のロジックをトリガーします。これにより、入力やクリックなどの緊急のUI更新がブロックされません。

## `useMemo` フックの使い方

- **メモ化**: これは高コストな関数呼び出しの結果を特定の引数に基づいてキャッシュ（記憶）する最適化手法です。同じ引数が再度与えられた場合、再計算せずにキャッシュされた結果を返します。
- **`useMemo` フック**: このフックは計算済みの値をメモ化するために使います。大きな配列のソート結果をメモ化する例を示します。`expensiveSortFunction` は `largeArray` が変わったときだけ実行されます。

```js
const memoizedSortedArray = useMemo(
  () => expensiveSortFunction(largeArray),
  [largeArray]
);
```

## `useCallback` フックの使い方

- **`useCallback` フック**: これは関数の参照をメモ化するために使います。

```js
const handleClick = useCallback(() => {
  // code goes here
}, [dependency]);
```

- **`React.memo`**: これはプロップが変わっていない場合に不要な再レンダリングを防ぐためにコンポーネントをメモ化するために使います。

```jsx
const MemoizedComponent = React.memo(({ prop }) => {
 return (
   <>
     {/* Presentation */}
   </>
 )
});
```

## 依存関係管理ツール

- **依存関係の定義**: ソフトウェアでは、あるコンポーネントやモジュールが正しく機能するために別のものに依存していることを指します。依存関係は、開発者が他者が作成した既存の関数やツールを利用できるため、ソフトウェアアプリケーションで一般的です。Reactプロジェクトに必要な2つの主要な依存関係は `react` と `react-dom` パッケージです。

```json
"dependencies": {
  "react": "^18.3.1",
  "react-dom": "^18.3.1"
}
```

- **パッケージマネージャの定義**: プロジェクトのソフトウェア依存関係を管理するにはパッケージマネージャを使います。パッケージマネージャは依存関係のインストール、更新、削除に使うツールです。JavaScript、Python、Ruby、Javaなど多くの人気プログラミング言語がパッケージマネージャを使います。JavaScriptの代表的なパッケージマネージャはnpm、Yarn、pnpmです。
- **`package.json` ファイル**: これはプロジェクトの名前、バージョン、依存関係などのメタデータを含む重要な設定ファイルです。スクリプト、ライセンス情報、その他の設定も定義し、プロジェクトと依存関係の管理を助けます。
- **`package-lock.json` ファイル**: このファイルはプロジェクトで使われているすべてのパッケージの正確なバージョンを固定します。パッケージを更新すると、新しいバージョンがロックファイルにも反映されます。
- **`node_modules` フォルダー**: このフォルダーには `package.json` ファイルに記載された依存関係の実際のコードが含まれています。プロジェクトの直接の依存関係とそれらの依存関係の両方が含まれます。
- **開発依存関係**: これは開発時のみ使われ、本番環境では使われないパッケージです。例えばJestのようなテストライブラリが該当します。Jestはローカルでアプリケーションをテストするために必要ですが、本番環境でアプリを動かすためには不要なので開発依存関係としてインストールします。

```json
"devDependencies": {
  "@eslint/js": "^9.17.0",
  "@types/react": "^18.3.18",
  "@types/react-dom": "^18.3.5",
  "@vitejs/plugin-react": "^4.3.4",
  "eslint": "^9.17.0",
  "eslint-plugin-react": "^7.37.2",

  "eslint-plugin-react-refresh": "^0.4.16",
  "globals": "^15.14.0",
  "vite": "^6.0.5"
}
```

## React Router

- **概要**: React RouterはReactアプリにルーティング機能を追加するサードパーティライブラリです。まず既存のReactプロジェクトにReact Routerをインストールします。

```bash
npm i react-router
```

次に `main.jsx` または `index.jsx` ファイル内でルート構造を設定します。

```jsx
import { StrictMode } from "react";
import { createRoot } from "react-dom/client";
import { BrowserRouter, Routes, Route } from "react-router";
import App from "./App.jsx";

import "./index.css";

createRoot(document.getElementById("root")).render(
  <StrictMode>
    <BrowserRouter>
      <Routes>
        <Route path="/" element={<App />} />
      </Routes>
    </BrowserRouter>
  </StrictMode>
);
```

`path` と `element` はURLとUIコンポーネントを結びつけるために使います。この例ではホームページのルートを `App` コンポーネントに設定しています。

- **複数ビューとルート設定**: 大規模なアプリでは複数のビューとルートをこのように設定することが一般的です。

```jsx
<Routes>
  <Route index element={<Home />} />
  <Route path="about" element={<About />} />

  <Route path="products">
    <Route index element={<ProductsHome />} />
    <Route path=":category" element={<Category />} />
    <Route path=":category/:productId" element={<ProductDetail />} />
    <Route path="trending" element={<Trending />} />
  </Route>
</Routes>
```

これらの例の `index` プロップは特定のパスセグメントのデフォルトルートを表します。つまり `Home` コンポーネントはルート `/` パスで表示され、`ProductsHome` コンポーネントは `/products` パスで表示されます。

- **ルートのネスト**: ルートを他のルートの中にネストでき、子ルートのパスは親ルートのパスに追加されます。

```jsx
<Route path="products">
  <Route path="trending" element={<Trending />} />
</Route>
```

上の例では、トレンド商品のパスは products/trending になります。

- **動的セグメント**: 動的セグメントはURLパスの任意の部分が動的になるものです。

```jsx
<Route path=":category" element={<Category />} />
```

この例では `category` という動的セグメントがあります。ユーザーが products/brass-instruments のようなURLにアクセスすると、ビューは `Category` コンポーネントに切り替わり、そのセグメントに基づいて適切なデータを動的に取得できます。

- **`useParams` フック**: このフックはURLパスの動的パラメータにアクセスするために使います。

```jsx
import { useParams } from "react-router";

export default function Category() {
  let params = useParams();
  {/* Accessing the category param: params.category */}
  {/* rest of code goes here */}
}
```

## Reactフレームワーク

- **概要**: Reactフレームワークはルーティング、画像最適化、データ取得、認証などの機能を提供します。これにより特定のユースケースでフロントエンドとバックエンドのアプリを別々に設定する必要がなくなる場合があります。Reactフレームワークの例にはNext.jsやRemixがあります。
- **Next.jsのルーティング**: このルーティングシステムは動的ルート、並列ルート、ルートハンドラ、リダイレクト、多言語対応などをサポートしています。

カスタムリクエストハンドラを作成する例は以下の通りです。

```js
export async function GET() {
  const res = await fetch("https://example-api.com");
  const data = await res.json();

  return Response.json({ data });
}
```

- **Next.jsの画像最適化**: `Image` コンポーネントはネイティブのHTML `img` 要素を拡張し、ページの読み込み速度とサイズの最適化を実現します。これにより画像はビューポートに入ったときだけ読み込まれ、`Image` コンポーネントは各デバイスに適したサイズの画像を自動的に提供します。

```jsx
import Image from "next/image";

export default function Page() {
  return (
    <Image src="link-to-image-goes-here" alt="descriptive-title-goes-here" />
  );
}
```

## プロップドリリング

- **定義**: プロップドリリングは親コンポーネントから深くネストされた子コンポーネントにプロップスを渡す過程で、一部の子コンポーネントはそのプロップスを必要としない場合でも渡されることを指します。

## 状態管理

- **Context API**: コンテキストとは、親コンポーネントが子コンポーネントに明示的にプロップスを通さずに情報を提供することを指します。`createContext` は他のコンポーネントが読むコンテキストを表すコンテキストオブジェクトを作成するために使います。`Provider` は子コンポーネントにコンテキストの値を供給するために使います。

```jsx
import { useState, createContext } from "react";

const CounterContext = createContext();

const CounterProvider = ({ children }) => {
  const [count, setCount] = useState(0);

  return (
    <CounterContext.Provider value={{ count, setCount }}>
      {children}
    </CounterContext.Provider>
  );
};

export { CounterContext, CounterProvider };
```

- **Redux**: Reduxは中央のストアと状態更新の厳格な管理を提供して状態管理を行います。アクション、リデューサー、ミドルウェアという予測可能なパターンを使います。アクションはアプリケーションからReduxストアにデータを送る情報のペイロードで、ユーザー操作によってトリガーされることが多いです。リデューサーはアクションに応じて状態をどのように変更するかを指定する関数で、不変の方法で状態を更新します。ミドルウェアはアクションのディスパッチとリデューサーの間の橋渡しをし、ログ記録や非同期処理などReduxの機能を拡張できます。

- **Zustand**: この状態管理ソリューションは小規模から中規模のアプリに最適です。`useStore` フックを使ってコンポーネントやページで直接状態にアクセスします。これによりアクション、リデューサー、プロバイダーなしでデータの変更やアクセスが可能です。

## ReactコンポーネントのデバッグにReact開発者ツールを使う

- **React開発者ツール**: これはChrome、Firefox、Edgeで使えるブラウザ拡張機能で、Reactコンポーネントを検査しパフォーマンス問題を特定できます。Safariでは`react-devtools` npmパッケージをインストールする必要があります。React DevToolsをインストールしブラウザでReactアプリを開いたら、ブラウザの開発者ツールを開いてReact用の2つの追加タブ「Components」と「Profiler」にアクセスできます。
- **Componentsタブ**: このタブはコンポーネントをツリー形式で表示します。ここでできることは以下の通りです。
  - アプリのコンポーネント階層を確認する
  - プロップス、状態、コンテキストの値をリアルタイムで確認・変更する
  - 選択したコンポーネントのソースコードを確認する
  - コンポーネントのデータをコンソールにログ出力する
  - コンポーネントのDOM要素を検査する
- **Profilerタブ**: このタブはコンポーネントのパフォーマンス分析を助けます。コンポーネントのパフォーマンスを記録し、不要な再レンダリングを特定し、コミット時間を確認し、遅いコンポーネントを最適化できます。

## Reactサーバーコンポーネント

- **定義**: Reactサーバーコンポーネントはサーバー上でのみレンダリングされ、最終的なHTMLだけをクライアントに送ります。これによりサーバーサイドのリソースに直接アクセスでき、ブラウザに送るJavaScriptの量を大幅に減らせます。
