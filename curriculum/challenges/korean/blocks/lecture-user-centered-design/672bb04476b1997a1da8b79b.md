# --description--

게으른 등록은 사용자가 애플리케이션에 등록하지 않고도 탐색하고 상호작용할 수 있게 하는 UI 디자인 패턴입니다. 좋은 예로 전자상거래 사이트가 있습니다. 사용자는 제품을 둘러보고 몇 가지 아이템을 장바구니에 담을 수 있어야 합니다. 그런 다음 구매에 관심이 있으면 등록해야 합니다.

그 이유는 사용자가 정보를 제공하고 등록하기 전에 사이트가 제공하는 가치를 먼저 확인해야 하기 때문입니다. 애플리케이션을 설계할 때 사용자가 가치를 보고 정보를 제공해도 안전하다고 느껴야 합니다. 그렇지 않으면 등록을 꺼리고 잠재 고객을 잃게 됩니다.

사용자의 민감한 데이터가 보호되고 안전하다는 점을 반드시 알리세요. 이후 모듈에서 애플리케이션을 안전하게 보호하고 사용자의 데이터를 지키는 방법을 다룰 것입니다.

또 다른 좋은 예는 YouTube입니다. YouTube는 기술, 대중문화, 게임 등 모든 주제의 수백만 개 동영상을 공유하는 플랫폼입니다. YouTube를 방문하면 사용자는 로그인하거나 등록하지 않고도 원하는 만큼 동영상을 시청할 수 있습니다. 하지만 좋아요를 누르거나 주석을 달거나 채널을 구독하려면 등록해야 합니다.

사용자가 시청하는 콘텐츠를 좋아하거나 대화에 참여하고 싶으면 등록할 의지가 더 커집니다. 게으른 등록은 사용자가 정보를 제공하기 전에 애플리케이션의 가치를 확인할 수 있게 하는 유용한 디자인 패턴입니다.

다음에 애플리케이션을 설계할 때는 사용자 참여와 유지율을 높이기 위해 게으른 등록을 고려하세요.
