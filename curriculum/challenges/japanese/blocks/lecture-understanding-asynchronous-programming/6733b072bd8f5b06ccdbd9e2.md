# --description--

非同期JavaScriptについて学びましょう。非同期イベントは、それぞれ独立して異なるタイミングで発生します。

ソフトウェア開発の文脈では、「非同期」とは、プログラムのメインの流れとは独立してバックグラウンドで実行されるタスクを指します。非同期処理の最大の利点は、メインプログラムの実行をブロックしないことです。

これは、リモートサーバーからのデータ取得、大きなファイルの処理、ユーザー入力の処理、複雑な計算の実行など、時間がかかる可能性のあるタスクに特に役立ちます。これが非同期プログラミングとして知られている手法です。

このアプローチは、これまで取り組んできた従来の同期プログラミング手法とは対照的です。複数行のコードを書くとき、通常は何がいつ起こるか予測できます。最初の行が実行され、その次に2行目、そして書いた順に続きます。

この例では、変数を定義する最初の行が最初に実行され、次に2行目、最後に文字列がコンソールにログされる3行目が実行されます。

```js
const topic = "JavaScript";
const learning = `I'm learning ${topic}!`;
console.log(learning);
```

すべてが順序通りで予測可能です。各行は次の行が始まる前に完了します。このタイプのJavaScriptプログラムはシングルスレッドとして知られています。

スレッドの概念は非常に重要です。スレッドとは、プロセス内の実行単位のことです。プログラム内の別々の制御の流れのようなものです。

同期プログラミングでは、スレッドは順番に1つずつ実行されます。ユーザー入力を待つなどスレッドがブロックされると、そのスレッドが完了するまでプロセス全体が停止します。

非同期プログラミングでは、スレッドは同時に実行され、複数のスレッドが同時に動作します。これにより、1つのスレッドがブロックされていても、メインプログラムが応答しなくなることなく複数のタスクを同時に実行し続けられます。

非同期プログラミングでは、非ブロッキング操作を扱うためにコールバック、Promise、またはasync/awaitがよく使われます。

前のレッスンで学んだように、コールバックとは、後で呼び出されるために別の関数に渡す関数のことです。イベントハンドラは特定のタイプのコールバックで、以前に扱ったことがあります。JavaScriptで非同期関数を実装する最も一般的な方法でした。しかし、コールバック関数がさらに他のコールバック関数を受け取る場合、コードとロジックが非常に複雑になることがあります。

現在、JavaScriptで非同期プログラミングを実装する最も一般的な手法はPromiseです。`Promise`は、非同期処理の最終的な完了（または失敗）とその値を表すオブジェクトです。

Promiseの値は、Promiseが作成された時点ではわかりません。非同期処理が完了したときにのみわかります。例えば、ウェブアプリケーションのためにリモートAPIからデータを取得する処理は時間がかかる場合があります。その間、ユーザーには快適なユーザー体験を提供したいですよね？

これを実装するには、非同期処理が実行されている間、ユーザーインターフェースをアクティブでインタラクティブに保つためにPromiseを作成できます。処理が完了すると、PromiseにはAPIから返されたデータが含まれるので、アプリケーションはそれを適切に処理したりレンダリングしたりできます。

非同期プログラミングは効率的なJavaScriptアプリケーションを構築するための強力な手法です。同期プログラミングと非同期プログラミングの違いを理解することで、アプリケーションに適した方法を選び、より効率的なコードを書けるようになります。
