## 物件基礎

- **定義**：物件是一種由屬性組成的資料結構。屬性由鍵和值組成。要從物件存取資料，可以使用點記法或中括號記法。

:::interactive_editor

```js
const person = {
  name: "Alice",
  age: 30,
  city: "New York"
};

console.log(person.name);  // Alice
console.log(person["name"]); // Alice
```

:::

要設定現有物件的屬性，可以搭配指定運算子，使用點記法或中括號記法。

:::interactive_editor

```js
const person = {
  name: "Alice",
  age: 30
};

person.job = "Engineer"
person["hobby"] = "Knitting"
console.log(person);  // {name: 'Alice', age: 30, job: 'Engineer', hobby: 'Knitting'}
```

:::

## 從物件移除屬性

- **`delete` 運算子**：這個運算子用來從物件中移除屬性。

:::interactive_editor

```js
const person = {
  name: "Alice",
  age: 30,
  job: "Engineer"
};

delete person.job;

console.log(person.job); // undefined
```

:::

## 檢查物件是否有某個屬性

- **`hasOwnProperty()` 方法**：這個方法會傳回一個布林值，表示物件是否擁有指定的自有屬性。

:::interactive_editor

```js
const person = {
  name: "Alice",
  age: 30
};

console.log(person.hasOwnProperty("name")); // true
console.log(person.hasOwnProperty("job")); // false
```

:::

- **`Object.hasOwn()` 方法**：這是現代且推薦的方式，用來檢查物件是否擁有某個屬性（非繼承而來）。語法為 `Object.hasOwn(object, propertyName)`。如果該屬性存在於物件上，則傳回 `true`，不存在則傳回 `false`，不論該屬性的值為何。這使得它比 `hasOwnProperty()` 更安全，也比在值可能是 `0`、`false`、`null` 或 `undefined` 時使用 `if (obj.prop)` 更可靠。

:::interactive_editor

```js
const person = {
  name: "Alice",
  age: 30
};

console.log(Object.hasOwn(person, "name")); // true
console.log(Object.hasOwn(person, "job")); // false
```

:::

這裡有一個更詳細的範例，說明為什麼 `Object.hasOwn()` 比直接檢查值更可靠：

:::interactive_editor

```js
const settings = {
  darkMode: false,
  fontSize: 0,
  language: null
};

// Object.hasOwn() correctly sees these properties exist
console.log(Object.hasOwn(settings, "darkMode")); // true (value is false, but exists)
console.log(Object.hasOwn(settings, "fontSize")); // true (value is 0, but exists)
console.log(Object.hasOwn(settings, "theme"));    // false (property was never added)

// Using if() directly is unsafe for falsy values!
if (settings.darkMode) {
  console.log("Dark mode on"); // Does NOT print — misleading!
}

// Object.hasOwn() is the safe way
if (Object.hasOwn(settings, "darkMode")) {
  console.log("darkMode exists, value is:", settings.darkMode); // darkMode exists, value is: false
}
```

:::

- **`in` 運算子**：這個運算子如果屬性存在於物件中，會傳回 `true`。

:::interactive_editor

```js
const person = {
  name: "Bob",
  age: 25
};

console.log("name" in person);  // true
```

:::

## 存取巢狀物件的屬性

- **存取資料**：存取巢狀物件的屬性時，和存取一般物件屬性一樣，可以用點記法或中括號記法。不過，你需要串接這些存取方式，才能深入巢狀結構。

:::interactive_editor

```js
const person = {
  name: "Alice",
  age: 30,
  contact: {
    email: "alice@example.com",
    phone: {
      home: "123-456-7890",
      work: "098-765-4321"
    }
  }
};

console.log(person.contact.phone.work); // "098-765-4321"
```

:::

## 原生與非原生資料型別

- **原生資料型別**：這些型別包含數字、字串、布林值、`null`、`undefined` 和符號。這些型別稱為「原生」，因為它們只代表單一值，且不是物件。原生值是不可變的，也就是說一旦建立，其值就不能更改。
- **非原生資料型別**：在 JavaScript 中，這些是物件，包括一般物件、陣列和函式。與原生型別不同，非原生型別可以以屬性或元素的形式儲存多個值。

## 物件方法

- **定義**：物件方法是與物件相關聯的函式。它們被定義為物件的屬性，並且可以存取和操作物件的資料。在方法內部使用 `this` 關鍵字時，會指向該物件本身，讓你可以存取它的屬性。

:::interactive_editor

```js
const person = {
  name: "Bob",
  age: 30,
  sayHello: function() {
    return "Hello, my name is " + this.name;
  }
};

console.log(person.sayHello()); // "Hello, my name is Bob"
```

:::

## 物件建構式

- **定義**：在 JavaScript 中，建構式是一種特殊的函式，用來建立並初始化物件。它必須搭配 `new` 關鍵字呼叫，並可在新建立的物件上初始化屬性和方法。`Object()` 建構式會建立一個新的空物件。

```js
new Object()
```

## 使用選用串鏈運算子（`?.`）

- **定義**：這個運算子讓你可以安全地存取物件屬性或呼叫方法，而不用擔心它們是否存在。

:::interactive_editor

```ts
const user = {
  name: "John",
  profile: {
    email: "john@example.com",
    address: {
      street: "123 Main St",
      city: "Somewhere"
    }
  }
};

console.log(user.profile?.address?.street); // "123 Main St"
console.log(user.profile?.phone?.number);   // undefined
```

:::

## 物件解構賦值

- **定義**：物件解構賦值讓你可以從物件中擷取值，並更簡潔、易讀地指定給變數。

:::interactive_editor

```js
const person = { name: "Alice", age: 30, city: "New York" };

const { name, age } = person;

console.log(name); // Alice
console.log(age);  // 30
```

:::

## 使用 JSON

- **定義**：JSON 代表 JavaScript 物件標記法。它是一種輕量級、以文字為基礎的資料格式，常用於伺服器與網頁應用程式之間的資料交換。

```js
{
  "name": "Alice",
  "age": 30,
  "isStudent": false,
  "list of courses": ["Mathematics", "Physics", "Computer Science"]
}
```

- **`JSON.stringify()`**：這個方法用來將 JavaScript 物件轉換成 JSON 字串。當你想要以容易分享或傳輸的格式儲存或傳送資料時，這會很有用。

:::interactive_editor

```js
const user = {
  name: "John",
  age: 30,
  isAdmin: true
};

const jsonString = JSON.stringify(user);
console.log(jsonString); // '{"name":"John","age":30,"isAdmin":true}'
```

:::

- **`JSON.parse()`**：這個方法會將 JSON 字串轉換回 JavaScript 物件。當你從網頁伺服器或 localStorage 取得 JSON 資料，並需要在應用程式中操作這些資料時，這會很有用。

:::interactive_editor

```js
const jsonString = '{"name":"John","age":30,"isAdmin":true}';
const userObject = JSON.parse(jsonString);

// result: { name: 'John', age: 30, isAdmin: true }
console.log(userObject);
```

:::
