¿Cuáles son los roles del elemento HTML `audio` y del elemento `video`, y cómo funcionan?

Los elementos `audio` y `video` te permiten añadir contenido de sonido y video a tus documentos HTML. El elemento `audio` soporta formatos de audio populares como mp3, wav y ogg. El elemento `video` soporta formatos mp4, ogg y webm.

Para incluir contenido de audio en tu página web, puedes usar el elemento `audio` con el atributo `src` apuntando a la ubicación de tu archivo de audio:

```html
<audio src="CrystalizeThatInnerChild.mp3"></audio>
```

Si intentas ejecutar este ejemplo, verás que nada aparece en la página. Sin embargo, si inspeccionas la página con las herramientas para desarrolladores, verás que el elemento `audio` está efectivamente en la página. Si deseas ver el reproductor de audio en la página, puedes agregar el elemento `audio` con el atributo `controls`:

```html
<audio src="CrystalizeThatInnerChild.mp3" controls></audio>
```

El atributo `controls` permite a los usuarios gestionar la reproducción de audio, incluyendo ajustar el volumen, pausar, o reanudar la reproducción. El atributo `controls` es un atributo booleano que puede añadirse a un elemento para habilitar controles de reproducción integrados. Si se omite, no se mostrarán controles.

Además de los atributos `src` y `controls`, hay varios otros atributos que mejoran la funcionalidad del elemento `audio`. El atributo `loop` es un atributo booleano que hace que el audio se reproduzca continuamente. Aquí tienes un ejemplo de uso del atributo `loop` para reproducir una de las canciones de Quincy Larson titulada "Can't stay down":

```html
<audio
  src="https://cdn.freecodecamp.org/curriculum/js-music-player/can't-stay-down.mp3"
  loop
  controls
></audio>
```

Cuando la canción llega al final, vuelve al principio y se reproduce de nuevo. Otro atributo que puedes usar es el atributo `muted`. Cuando está presente en el elemento `audio`, este atributo booleano iniciará el audio en un estado silenciado. Aquí tienes un ejemplo de uso del atributo `muted`:

```html
<audio
  src="https://cdn.freecodecamp.org/curriculum/js-music-player/can't-stay-down.mp3"
  loop
  controls
  muted
></audio>
```

Cuando comiences la canción en el navegador, no escucharás nada. Para escuchar la música, necesitarás hacer clic en el ícono de volumen.

En cuanto a los tipos de archivos de audio, hay diferencias en qué navegadores soportan qué tipo. Para acomodar esto, puedes usar elementos `source` dentro del elemento `audio`, y el navegador seleccionará la primera fuente que comprenda. Aquí tienes un ejemplo de uso de múltiples elementos `source` para un elemento `audio`:

```html
<audio controls>
  <source src="audio.ogg" type="audio/ogg" />
  <source src="audio.wav" type="audio/wav" />
  <source src="audio.mp3" type="audio/mpeg" />
</audio>
```

El navegador empezará primero con el tipo ogg, y si no puede reproducir el audio, pasará al siguiente tipo en la lista.

Todos los atributos que hemos aprendido hasta ahora también son compatibles en el elemento `video`. Aquí tienes un ejemplo de uso de un elemento `video` con los atributos `loop`, `controls` y `muted`:

```html
<video
  src="https://archive.org/download/BigBuckBunny_124/Content/big_buck_bunny_720p_surround.mp4"
  loop
  controls
  muted
></video>
```

Para el `src`, o atributo fuente, estamos usando un video llamado "Big Buck Bunny" de archive.org. Si deseas mostrar una imagen mientras el video se descarga, puedes usar el atributo `poster`. Este atributo no está disponible para elementos `audio` y es único para el elemento `video`. Aquí tienes un ejemplo de uso del atributo `poster` con contenido proporcionado por peach.blender.org:

```html
<video
  src="https://archive.org/download/BigBuckBunny_124/Content/big_buck_bunny_720p_surround.mp4"
  loop
  controls
  muted
  poster="https://peach.blender.org/wp-content/uploads/title_anouncement.jpg?x11217"
  width="620"
></video>
```

Este ejemplo también usa el atributo `width` para establecer el ancho en 620 píxeles para que el video encaje mejor en la pantalla.
