## JavaScript-Bibliotheken und Frameworks

- JavaScript-Bibliotheken und Frameworks bieten schnelle Lösungen für häufige Probleme und beschleunigen die Entwicklung, indem sie vorgefertigten Code bereitstellen.
- Bibliotheken konzentrieren sich in der Regel stärker darauf, Lösungen für spezifische Aufgaben bereitzustellen, wie das Manipulieren des DOM, das Verwalten von Ereignissen oder das Handhaben von AJAX-Anfragen.
- Einige Beispiele für JavaScript-Bibliotheken sind jQuery und React.
- Frameworks hingegen bieten eine klarere Struktur zum Erstellen von Anwendungen. Sie kommen oft mit einem Satz von Regeln und Konventionen, die Entwickler befolgen müssen.
- Beispiele für Frameworks sind Angular und Next.js, ein Meta-Framework für React.
- **Single-Page-Anwendungen** (SPAs) sind Webanwendungen, die eine einzelne HTML-Seite laden und diese Seite dynamisch aktualisieren, während der Benutzer mit der Anwendung interagiert, ohne die gesamte Seite neu zu laden.
- SPAs verwenden JavaScript, um den Zustand der Anwendung zu verwalten und Inhalte zu rendern. Dies geschieht oft mit Frameworks, die großartige Werkzeuge zum Erstellen komplexer Benutzeroberflächen bieten.
- Einige Probleme im Zusammenhang mit SPAs sind:

- Screenreader haben Schwierigkeiten mit dynamisch aktualisierten Inhalten.
  - Die URL ändert sich nicht, wenn der Nutzer innerhalb der Anwendung navigiert, was das Setzen von Lesezeichen, das Zurückgehen oder das Teilen bestimmter Seiten erschweren kann.
  - Die anfängliche Ladezeit kann langsam sein, wenn die Anwendung groß ist, da alle Ressourcen im Voraus geladen werden müssen.

## React

- React ist eine beliebte JavaScript-Bibliothek zum Erstellen von Benutzeroberflächen und Webanwendungen.
- Ein Kernkonzept von React ist die Erstellung wiederverwendbarer UI-Komponenten, die sich unabhängig aktualisieren und rendern können, wenn sich Daten ändern.
- React ermöglicht es Entwicklern, zu beschreiben, wie die UI basierend auf dem Zustand der Anwendung aussehen soll. React aktualisiert und rendert dann die richtigen Komponenten, wenn sich die Daten oder der Zustand ändern.

## React-Komponenten

- Komponenten sind die Bausteine von React-Anwendungen, die es Entwicklern ermöglichen, komplexe Benutzeroberflächen in kleinere, handhabbare Teile zu zerlegen.
- Die UI wird mit JSX beschrieben, einer Erweiterung der JavaScript-Syntax, die es Entwicklern erlaubt, HTML-ähnlichen Code innerhalb von JavaScript zu schreiben.
- Komponenten sind im Grunde JS-Funktionen oder Klassen, die ein Stück UI zurückgeben.

Hier ist ein Beispiel für eine einfache React-Komponente, die eine Begrüßungsnachricht rendert:

```jsx
function Greeting() {
  const name = 'Anna';
  return <h1>Welcome, {name}!</h1>;
}
```

Um die Komponente zu verwenden, können Sie sie einfach aufrufen:
    
```jsx
  <Greeting />
```

## Importieren und Exportieren von React-Komponenten

- Ein Export macht eine Komponente für andere Dateien verfügbar. Ein Import erlaubt einer Datei, eine Komponente zu verwenden, die anderswo exportiert wurde.
- Es gibt zwei Arten von Exporten in React: Standardexporte und benannte Exporte.

**Standardexporte**

- Eine Datei kann nur einen Standardexport haben. Das ist ideal für eine Datei, die hauptsächlich eine einzelne Komponente enthält.
- Sie können eine Komponente als Standardexport so exportieren:

```jsx
// City.js
function City() {
  return <p>New York</p>;
}

export default City;
```

- Sie können auch in derselben Zeile wie die Komponentendefinition exportieren:

```jsx
export default function City() {
  return <p>New York</p>;
}
```

- Um einen Standardexport zu importieren, verwenden Sie das `import`-Schlüsselwort ohne geschweifte Klammern:

```jsx
// App.js
import City from './City';

function App() {
  return (
    <div>
      <h1>My favorite city is:</h1>
      <City />
    </div>
  );
}
```

**Benannte Exporte**

- Benannte Exporte erlauben es einer Datei, mehrere Komponenten zu teilen. Im Gegensatz zu Standardexporten müssen sie mit genau dem Namen importiert werden, mit dem sie exportiert wurden.
- Sie können mehrere Komponenten aus derselben Datei so exportieren:

```jsx
// Animals.jsx
export function Cat() {
  return <h2>Mr. Whiskers</h2>;
}

export function Dog() {
  return <h2>Fido</h2>;
}
```

- Um benannte Exporte zu importieren, verwenden Sie geschweifte Klammern mit den exakt exportierten Namen:

```jsx
import { Cat, Dog } from './Animals';
```

- Sie können einen benannten Export beim Import mit dem `as`-Schlüsselwort umbenennen:

```jsx
import { Cat as Kitty } from './Animals';
```

**Gemischte Standard- und benannte Exporte**

- Eine Datei kann gleichzeitig einen Standardexport und mehrere benannte Exporte haben:

```jsx
// Animals.jsx
export default function Cat() {
  return <h2>Mr. Whiskers</h2>;
}

export function Dog() {
  return <h2>Fido</h2>;
}
```

- Beim Importieren gemischter Exporte steht der Standardexport zuerst (ohne Klammern), gefolgt von den benannten Exporten (in Klammern):

```jsx
import Cat, { Dog } from './Animals';
```

## Einrichten eines React-Projekts mit Vite

- Projekt-Setup-Tools und CLIs bieten eine schnelle und einfache Möglichkeit, neue Projekte zu starten, sodass Entwickler sich auf das Schreiben von Code konzentrieren können, anstatt sich mit der Konfiguration zu beschäftigen.
- Vite ist ein beliebtes Projekt-Setup-Tool und kann mit React verwendet werden.
- Um ein neues Projekt mit Vite zu erstellen, können Sie folgenden Befehl im Terminal verwenden:

```bash
npm create vite@latest my-react-app -- --template react
```

Dieser Befehl erstellt ein neues React-Projekt namens `my-react-app` mit der React-Vorlage von Vite. Im Projektverzeichnis sehen Sie eine `package.json`-Datei mit den Projektabhängigkeiten und Befehlen.

- Um das Projekt auszuführen, navigieren Sie in das Projektverzeichnis und führen Sie folgende Befehle aus:

```bash
cd my-react-app # path to the project directory
npm install # installs the dependencies listed in the package.json file
```

- Sobald die Abhängigkeiten installiert sind, sollten Sie einen neuen Ordner in Ihrem Projekt namens `node_modules` sehen.

- Der `node_modules`-Ordner ist der Speicherort aller Pakete und Bibliotheken, die Ihr Projekt benötigt.

- Um Ihr Projekt zu starten, verwenden Sie folgenden Befehl:

```bash
npm run dev
```

- Danach öffnen Sie Ihren Browser und navigieren zu `http://localhost:5173`, um Ihre React-Anwendung laufen zu sehen.

- Um den Code der Starter-Vorlage tatsächlich zu sehen, können Sie in Ihrem Projekt in den `src`-Ordner gehen, dort sollten Sie die `App.jsx`-Datei sehen.

## Props in React-Komponenten übergeben

- In React sind Props (kurz für properties) eine Möglichkeit, Daten von einer übergeordneten Komponente an eine untergeordnete Komponente zu übergeben. Dieser Mechanismus ist notwendig, um wiederverwendbare und dynamische UI-Elemente zu erstellen.
- Props können jeden JavaScript-Wert enthalten. Um Props von einer übergeordneten an eine untergeordnete Komponente zu übergeben, fügen Sie die Props als Attribute hinzu, wenn Sie die untergeordnete Komponente im JSX der übergeordneten Komponente verwenden. Hier ist ein einfaches Beispiel:

```jsx
// Parent component
function Parent() {
  const name = 'Anna';
  return <Child name={name} />;
}

// Child component
function Child(props) {
  return <h1>Hello, {props.name}!</h1>;
}
```

Sie können mehrere Props mit dem Spread-Operator `(...)` übergeben, nachdem Sie sie in ein Objekt umgewandelt haben. Hier ein Beispiel:
    
```jsx
// Parent component
function Parent() {
  const person = {
    name: 'Anna',
    age: 25,
    city: 'New York'
  };
  return <Child {...person} />;
}
```

In diesem Code wandelt der Spread-Operator `{...person}` das person-Objekt in einzelne Props um, die an die Child-Komponente übergeben werden.

## Bedingtes Rendern in React

- Bedingtes Rendern in React ermöglicht es Ihnen, dynamische Benutzeroberflächen zu erstellen. Es wird verwendet, um je nach bestimmten Bedingungen oder Zuständen innerhalb Ihrer Anwendung unterschiedliche Inhalte anzuzeigen.
- Es gibt mehrere Möglichkeiten, Inhalte in React bedingt zu rendern. Eine gängige Methode ist die Verwendung des ternären Operators. Hier ein Beispiel:

```jsx
function Greeting({ isLoggedIn }) {
  return (
    <div>
      {isLoggedIn ? <h1>Welcome back!</h1> : <h1>Please log in</h1>}
    </div>
  );
}
```

- Eine andere Möglichkeit, Inhalte bedingt zu rendern, ist die Verwendung des logischen UND-Operators (`&&`). Das ist nützlich, wenn Sie Inhalte nur rendern möchten, wenn eine bestimmte Bedingung erfüllt ist. Hier ein Beispiel:

```jsx
function Greeting({ user }) {
  return (
    <div>
      {user && <h1>Welcome, {user.name}!</h1>}
    </div>
  );
}
```

Im obigen Code wird das `h1`-Element nur gerendert, wenn das user-Objekt wahrheitsgemäß ist.

Sie können auch eine direkte `if`-Anweisung so verwenden:

```js
function Greeting({ isLoggedIn }) {
  if (isLoggedIn) {
    return <h1>Welcome back!</h1>;
  }
  return <h1>Please sign in</h1>;
}
```

## Listen in React rendern

- Das Rendern von Listen in React ist eine häufige Aufgabe beim Erstellen von Benutzeroberflächen.
- Listen können mit der JS-Array-Methode `map()` gerendert werden, um ein Array von Elementen zu durchlaufen und ein neues Array von JSX-Elementen zurückzugeben.
- Zum Beispiel, wenn Sie ein Array von Namen haben, das Sie als Liste rendern möchten, können Sie Folgendes tun:

```jsx
function NameList({ names }) {
  return (
    <ul>
      {names.map((name, index) => (
        <li key={`${name}-${index}`}>{name}</li>
      ))}
    </ul>
  );
}
```

- Denken Sie immer daran, jedem Listenelement einen eindeutigen Schlüssel zu geben, um React bei der Verwaltung der Aktualisierungs- und Renderaufgaben zu unterstützen. Mit diesen Techniken können Sie flexible, effiziente und dynamische Listen in Ihren React-Anwendungen erstellen.

## Inline-Stile in React

- Inline-Stile in React erlauben es Ihnen, CSS-Stile direkt auf JSX-Elemente mit JavaScript-Objekten anzuwenden.
- Um Inline-Stile in React anzuwenden, können Sie das style-Attribut auf JSX-Elementen verwenden. Das style-Attribut nimmt ein Objekt, bei dem die Schlüssel CSS-Eigenschaften in CamelCase sind und die Werte die entsprechenden Werte. Hier ein Beispiel:

```js
function Greeting() {
  return (
    <h1
      style={{ color: 'blue', fontSize: '24px', backgroundColor: 'lightgray' }}
    >
      Hello, world!
    </h1>
  );
}

export default Greeting;
```

Sie können die Stile auch in ein separates Objekt auslagern und es im `style`-Attribut so referenzieren:

```jsx
function Greeting() {

  const styles = {
    color: 'blue',
    fontSize: '24px',
    backgroundColor: 'lightgray'
  };

  return <h1 style={styles}>Hello, world!</h1>;
}

export default Greeting;
```

- Inline-Stile unterstützen dynamisches Styling, indem Sie Stile bedingt basierend auf Props oder Zustand anwenden können. Hier ein Beispiel, wie Sie Stile basierend auf einer Prop bedingt anwenden können:

```jsx
function Greeting({ isImportant }) {

  const styles = {
    color: isImportant ? 'red' : 'black',
    fontSize: isImportant ? '24px' : '16px'
  };

  return <h1 style={styles}>Hello, world!</h1>;
}

export default Greeting;
```

- Im obigen Code werden die `color`- und `fontSize`-Stile bedingt basierend auf der `isImportant`-Prop gesetzt.

## Arbeiten mit Ereignissen in React

- **Synthetisches Ereignissystem**: Dies ist Reacts Methode zur Behandlung von Ereignissen. Es dient als Wrapper um die nativen Ereignisse wie die `click`, `keydown` und `submit` Ereignisse. Ereignishandler in React verwenden die camelCase-Benennungskonvention. (z. B. `onClick`, `onSubmit` usw.)

Hier ist ein Beispiel für die Verwendung des `onClick` Attributs für ein `button` Element in React:

```jsx
function handleClick() {
  console.log("Button clicked!");
}

<button onClick={handleClick}>Click Me</button>;
```

In React beginnen Ereignishandler-Funktionen üblicherweise mit dem Präfix `handle`, um anzuzeigen, dass sie für die Behandlung von Ereignissen zuständig sind, wie `handleClick` oder `handleSubmit`.

Wenn eine Benutzeraktion ein Ereignis auslöst, übergibt React ein synthetisches Ereignisobjekt an Ihren Handler. Dieses Objekt verhält sich ähnlich wie das native Event-Objekt in Vanilla JavaScript und stellt Eigenschaften wie `type`, `target` und `currentTarget` bereit.

Um Standardverhalten wie das Aktualisieren des Browsers während eines `onSubmit` Ereignisses zu verhindern, können Sie beispielsweise die Methode `preventDefault()` aufrufen:

```jsx
function handleSubmit(event) {
  event.preventDefault();
  console.log("Form submitted!");
}

<form onSubmit={handleSubmit}>
  <input type="text" />
  <button>Submit</button>
</form>;
```

Sie können eine Handler-Funktion auch in einer Pfeilfunktion wie folgt einbetten:

```jsx
function handleDelete(id) {
  console.log("Deleting item:", id);
}

<button onClick={() => handleDelete(1)}>Delete Item</button>;
```

## Arbeiten mit State und dem `useState` Hook

- **Definition von State**: In React ist State ein Objekt, das Daten für eine Komponente enthält. Wenn sich der State ändert, rendert die Komponente neu. React behandelt State als unveränderlich, das heißt, Sie sollten ihn nicht direkt ändern.
- **`useState()`-Hook**: Der `useState`-Hook ist eine Funktion, mit der Sie State-Variablen in Funktionskomponenten deklarieren können. Hier ist die grundlegende Syntax:

```js
const [stateVariable, setStateFunction] = useState(initialValue);
```

In der Zustandsvariablen haben Sie Folgendes:

- `stateVariable` enthält den aktuellen State-Wert
- `setStateFunction` (die Setter-Funktion) aktualisiert die Zustandsvariable
- `initialValue` legt den Anfangszustand fest

Hier ist ein vollständiges Beispiel für eine `Counter` Komponente:

```jsx
import { useState } from "react";

function Counter() {
  const [count, setCount] = useState(0);

  return (
    <div>
      <h2>{count}</h2>

      <button onClick={() => setCount(count - 1)}>Decrement</button>
      <button onClick={() => setCount(count + 1)}>Increment</button>
    </div>
  );
}

export default Counter;
```

## Rendern und React-Komponenten

- **Definition**: In React ist Rendern der Prozess, bei dem Komponenten in der Benutzeroberfläche (UI) erscheinen, üblicherweise im Browser. Der Renderprozess besteht aus drei Phasen: Auslösen, Rendern und Commit.

Die Auslösephase tritt ein, wenn React erkennt, dass sich etwas geändert hat und die Benutzeroberfläche (UI) möglicherweise aktualisiert werden muss. Diese Änderung ist oft auf eine Aktualisierung von State oder Props zurückzuführen.

Sobald das Auslösen erfolgt, beginnt React mit der Renderphase. Hier wertet React Ihre Komponenten neu aus und ermittelt, was angezeigt werden soll. Dazu verwendet React eine leichte Kopie des „echten“ DOM, das sogenannte virtuelle DOM. Mit dem virtuellen DOM kann React schnell prüfen, was sich in der Komponente ändern muss.

Die Commit-Phase ist der Schritt, in dem React die vorbereiteten Änderungen vom virtuellen DOM auf das echte DOM anwendet. Anders gesagt, in dieser Phase sehen Sie das endgültige Ergebnis auf dem Bildschirm.

## Aktualisieren von Objekten und Arrays im State

- **Aktualisieren von Objekten im State**: Wenn Sie ein Objekt im State aktualisieren müssen, sollten Sie zuerst ein neues Objekt erstellen oder ein bestehendes kopieren und dann den State für dieses neue Objekt setzen. Jedes Objekt, das in den State gesetzt wird, sollte als schreibgeschützt betrachtet werden. Hier ist ein Beispiel, wie Sie den Namen, das Alter und die Stadt eines Benutzers setzen. Die `handleChange` Funktion wird verwendet, um Aktualisierungen der Benutzerinformationen zu behandeln:

```jsx
import { useState } from "react";

function Profile() {
  const [user, setUser] = useState({ name: "John Doe", age: 31, city: "LA" });

  const handleChange = (e) => {
    const { name, value } = e.target;

    setUser((prevUser) => ({...prevUser, [name]: value}));
  };

  return (
    <div>
      <h1>User Profile</h1>
      <p>Name: {user.name}</p>
      <p>Age: {user.age}</p>
      <p>City: {user.city}</p>

      <h2>Update User Age </h2>
      <input type="number" name="age" value={user.age} onChange={handleChange} />

      <h2>Update User Name </h2>
      <input type="text" name="name" value={user.name} onChange={handleChange} />

      <h2>Update User City </h2>
      <input type="text" name="city" value={user.city} onChange={handleChange} />
    </div>
  );
}

export default Profile;
```

- **Aktualisieren von Arrays im State**: Beim Aktualisieren von Arrays im State ist es wichtig, das Array nicht direkt mit Methoden wie `push()` oder `pop()` zu verändern. Stattdessen sollten Sie ein neues Array erstellen, wenn Sie den State aktualisieren:

```jsx
const addItem = () => {
  const newItem = {
    id: items.length + 1,
    name: `Item ${items.length + 1}`,
  };

  // Creates a new array
  setItems((prevItems) => [...prevItems, newItem]);
};
```

Wenn Sie Elemente aus einem Array entfernen möchten, sollten Sie die `filter()` Methode verwenden, die ein neues Array zurückgibt, nachdem die zu entfernenden Elemente herausgefiltert wurden:

```jsx
const removeItem = (id) => {
  setItems((prevItems) => prevItems.filter((item) => item.id !== id));
};
```

## Werte mit Refs referenzieren

- **`ref` Attribut**: Sie können in React auf einen DOM-Knoten zugreifen, indem Sie das `ref` Attribut verwenden. Hier ist ein Beispiel, das einen `ref` zeigt, um den Fokus auf ein `input` Element zu setzen. Die `current` Eigenschaft wird verwendet, um den aktuellen Wert dieses `ref` zuzugreifen:

```jsx
import { useRef } from "react";

const Focus = () => {
  const inputRef = useRef(null);

  const handleFocus = () => {
    if (inputRef.current) {
      inputRef.current.focus();
    }
  };

  return (
    <div>
      <input ref={inputRef} type="text" placeholder="Enter text" />
      <button onClick={handleFocus}>Focus Input</button>
    </div>
  );
};

export default Focus;
```

## Arbeiten mit dem `useEffect` Hook

- **`useEffect()` Hook**: In React ist ein Effekt alles, was außerhalb des Renderprozesses einer Komponente passiert. Das heißt, alles, was React nicht direkt als Teil des Renderns der UI behandelt. Häufige Beispiele sind das Abrufen von Daten, das Aktualisieren des Titels des Browser-Tabs, das Lesen oder Schreiben im lokalen Speicher des Browsers, das Ermitteln des Standorts des Benutzers und vieles mehr. Diese Operationen interagieren mit der Außenwelt und werden als Nebeneffekte bezeichnet. React stellt den `useEffect` Hook bereit, mit dem Sie diese Nebeneffekte behandeln können. `useEffect` ermöglicht es Ihnen, eine Funktion auszuführen, nachdem die Komponente gerendert oder aktualisiert wurde.

```js
import { useEffect } from "react";

useEffect(() => {
 // Your side effect logic (usually a function) goes here
}, [dependencies]);
```

Die Effektfunktion läuft nach dem Rendern der Komponente, während das optionale `dependencies` Argument steuert, wann der Effekt ausgeführt wird.

Beachten Sie, dass `dependencies` ein Array von „reaktiven Werten“ (State, Props, Funktionen, Variablen usw.), ein leeres Array oder ganz weggelassen werden kann. So steuern all diese Optionen, wie `useEffect` funktioniert:

- Wenn `dependencies` ein Array ist, das einen oder mehrere reaktive Werte enthält, wird der Effekt immer dann ausgeführt, wenn sich diese ändern.

- Wenn `dependencies` ein leeres Array ist, läuft `useEffect` nur einmal, wenn die Komponente zum ersten Mal gerendert wird.

- Wenn Sie `dependencies` weglassen, läuft der Effekt jedes Mal, wenn die Komponente gerendert oder aktualisiert wird.

## Wie man benutzerdefinierte Hooks erstellt

- **Benutzerdefinierte Hooks**: Ein benutzerdefinierter Hook ermöglicht es Ihnen, wiederverwendbare Logik aus Komponenten zu extrahieren, wie z. B. das Abrufen von Daten, das Verwalten von State, Umschalten und Nebeneffekte wie das Verfolgen des Online-Status. In React beginnen alle eingebauten Hooks mit dem Wort `use`, daher sollte Ihr benutzerdefinierter Hook derselben Konvention folgen.

Hier ist ein Beispiel für die Erstellung eines `useDebounce` Hooks:

```jsx
function useDebounce(value, delay) {
  const [debouncedValue, setDebouncedValue] = useState(value);

  useEffect(() => {
    const handler = setTimeout(() => {
      setDebouncedValue(value);
    }, delay);

    return () => {
      clearTimeout(handler);
    };
  }, [value, delay]);

  return debouncedValue;
}

export { useDebounce };
```

## Arbeiten mit Formularen in React

- **Kontrollierte Eingaben**: Dabei speichern Sie den Wert des Eingabefelds im State und aktualisieren ihn über `onChange`-Ereignisse. Das gibt Ihnen die volle Kontrolle über die Formulardaten und ermöglicht sofortige Validierung und bedingtes Rendern.

```jsx
import { useState } from "react";

function App() {
  const [name, setName] = useState("");

  const handleChange = (e) => {
    setName(e.target.value);
  };

  const handleSubmit = (e) => {
    e.preventDefault();
    console.log(name);
  };

  return (
    <>
      <form onSubmit={handleSubmit}>
        <label htmlFor="name">Your name</label> <br />
        <input value={name} id="name" onChange={handleChange} type="text" />
        <button type="submit">Submit</button>
      </form>
    </>
  );
}

export default App;
```

- **Unkontrollierte Eingaben**: Anstatt die Eingaben über den `useState`-Hook zu steuern, behalten unkontrollierte Eingaben in HTML ihren eigenen internen Zustand mithilfe des DOM. Da das DOM die Eingabewerte kontrolliert, müssen Sie die Werte der Eingabefelder mit einem `ref` abrufen.

```jsx
import { useRef } from "react";

function App() {
 const nameRef = useRef();

 const handleSubmit = (e) => {
   e.preventDefault();
   console.log(nameRef.current.value);
 };

 return (
   <form onSubmit={handleSubmit}>
     <label htmlFor="name">Your</label>{" "}
     <input type="text" ref={nameRef} id="name" />
     <button type="submit">Submit</button>
   </form>
 );
}

export default App;
```

## Arbeiten mit dem `useActionState`-Hook

- **Serveraktionen**: Das sind Funktionen, die auf dem Server ausgeführt werden, um die Formularverarbeitung direkt auf dem Server zu ermöglichen, ohne dass API-Endpunkte benötigt werden. Hier ist ein Beispiel aus einer Next.js-Anwendung:

```js
"use server";

async function submitForm(formData) {
 const name = formData.get("name");
 return { message: `Hello, ${name}!` };
}
```

Die `"use server"`-Direktive kennzeichnet die Funktion als Server Action.

- **`useActionState`-Hook**: Dieser Hook aktualisiert den State basierend auf dem Ergebnis einer Formulareinsendung. Hier ist die Grundsyntax des `useActionState`-Hooks:

```js
const [state, action, isPending] = useActionState(actionFunction, initialState, permalink);
```

- `state` ist der aktuelle State, den die Aktion zurückgibt.
- `action` ist die Funktion, die die Serveraktion auslöst.
- `isPending` ist ein Boolean, der angibt, ob die Aktion gerade läuft oder nicht.
- `actionFunction` Parameter ist die Serveraktion selbst.
- `initialState` ist der Parameter, der den Ausgangszustand vor Ausführung der Aktion darstellt.
- `permalink` ist ein optionaler String, der die eindeutige Seiten-URL enthält, die das Formular ändert.

## Datenabruf in React

- **Optionen zum Datenabruf**: Es gibt viele verschiedene Möglichkeiten, Daten in React abzurufen. Sie können die native Fetch-API verwenden oder Drittanbieter-Tools wie Axios oder SWR.
- **Häufig verwendete State-Variablen beim Datenabruf**: Unabhängig davon, welche Methode Sie wählen, um Daten in React abzurufen, gibt es einige State-Teile, die Sie verfolgen müssen. Der erste ist die eigentliche Datenmenge. Der zweite verfolgt, ob die Daten noch abgerufen werden. Der dritte ist eine State-Variable, die eventuelle Fehler während des Datenabrufs erfasst.

```js
const [data, setData] = useState(null);
const [loading, setLoading] = useState(true);
const [error, setError] = useState(null);
```

Da der Datenabruf ein Nebeneffekt ist, ist es am besten, den `Fetch API` innerhalb eines `useEffect`-Hooks zu verwenden.

```js
useEffect(() => {
  const fetchData = async () => {
    try {
      const res = await fetch("https://jsonplaceholder.typicode.com/posts");
      
      if (!res.ok) {
        throw new Error("Network response was not ok");
      }

      const data = await res.json();
      setData(data);
    } catch (err) {
      setError(err);
    } finally {
      setLoading(false);
    }
  };

  fetchData();
}, []);
```

Dann können Sie eine Ladeanzeige rendern, wenn der Datenabruf noch nicht abgeschlossen ist, eine Fehlermeldung, wenn ein Fehler aufgetreten ist, oder die Ergebnisse.

```jsx
}

return (
  <ul>
    {data.map((post) => (
      <li key={post.id}>{post.title}</li>
    ))}
  </ul>
);
```

## Arbeiten mit dem `useOptimistic`-Hook

- **`useOptimistic`-Hook**: Dieser Hook wird verwendet, um UIs responsiv zu halten, während im Hintergrund eine asynchrone Aktion abgeschlossen wird. Er hilft dabei, „optimistische Updates“ in der UI zu verwalten, eine Technik, bei der Sie sofortige UI-Updates basierend auf dem erwarteten Ergebnis einer Aktion bereitstellen, etwa während Sie auf eine Serverantwort warten.

Hier ist die Grundsyntax:

```js
const [optimisticState, addOptimistic] = useOptimistic(actualState, updateFunction);
```

- `optimisticState` ist der temporäre State, der sofort aktualisiert wird, um die Benutzererfahrung zu verbessern.
- `addOptimistic` ist die Funktion, die das optimistische Update vor der tatsächlichen State-Änderung anwendet.
- `actualState` ist der echte State-Wert, der aus dem Ergebnis einer Aktion stammt, z. B. dem Abruf von Daten vom Server.
- `updateFunction` ist die Funktion, die bestimmt, wie der optimistische State aktualisiert werden soll, wenn sie aufgerufen wird.

Hier ist ein Beispiel für die Verwendung des `useOptimistic`-Hooks in einer `TaskList`-Komponente:

```js
"use client";

import { useOptimistic } from "react";

export default function TaskList({ tasks, addTask }) {
  const [optimisticTasks, addOptimisticTask] = useOptimistic(
    tasks,
    (state, newTask) => [...state, { text: newTask, pending: true }]
  );

  async function handleSubmit(e) {
    e.preventDefault();
    const formData = new FormData(e.target);

    addOptimisticTask(formData.get("task"));

    addTask(formData);
    e.target.reset();
  }

  return <>{/* UI */}</>;
}
```

- **`startTransition`**: Dies wird verwendet, um einen Teil der UI zu rendern und ein State-Update als nicht dringende Transition zu markieren. So bleibt die UI während aufwändiger Updates responsiv. Hier ist die Grundsyntax:

```js
startTransition(action)
```

Der `action` führt ein State-Update durch oder löst eine transition-bezogene Logik aus. So wird sichergestellt, dass dringende UI-Updates (wie Tippen oder Klicken) nicht blockiert werden.

## Arbeiten mit dem `useMemo`-Hook

- **Memoisierung**: Dies ist eine Optimierungstechnik, bei der das Ergebnis teurer Funktionsaufrufe basierend auf bestimmten Argumenten zwischengespeichert (erinnert) wird. Wenn dieselben Argumente erneut übergeben werden, wird das zwischengespeicherte Ergebnis zurückgegeben, anstatt die Funktion neu zu berechnen.
- **`useMemo`-Hook**: Dieser Hook wird verwendet, um berechnete Werte zu memoizieren. Hier ist ein Beispiel, wie das Ergebnis der Sortierung eines großen Arrays memoisiert wird. Der `expensiveSortFunction` wird nur ausgeführt, wenn sich `largeArray` ändert:

```js

  () => expensiveSortFunction(largeArray),
  [largeArray]
);
```

## React Router

- **Einführung**: React Router ist eine Drittanbieter-Bibliothek, mit der Sie Routing zu Ihren React-Anwendungen hinzufügen können. Um zu beginnen, müssen Sie React Router in einem bestehenden React-Projekt so installieren:

```bash
npm i react-router
```

Dann müssen Sie innerhalb der `main.jsx`- oder `index.jsx`-Datei die Routenstruktur so einrichten:

```jsx
import { StrictMode } from "react";
import { createRoot } from "react-dom/client";
import { BrowserRouter, Routes, Route } from "react-router";
import App from "./App.jsx";

import "./index.css";

createRoot(document.getElementById("root")).render(
  <StrictMode>
    <BrowserRouter>
      <Routes>
        <Route path="/" element={<App />} />
      </Routes>
    </BrowserRouter>
  </StrictMode>
);
```

Die `path` und `element` werden verwendet, um die URL und UI-Komponenten miteinander zu koppeln. In diesem Fall richten wir eine Route für die Startseite ein, die auf die `App`-Komponente zeigt.

- **Mehrere Ansichten und Routenaufbau**: In größeren Anwendungen ist es üblich, mehrere Ansichten und Routen so einzurichten:

```jsx
<Routes>
  <Route index element={<Home />} />
  <Route path="about" element={<About />} />

  <Route path="products">
    <Route index element={<ProductsHome />} />
    <Route path=":category" element={<Category />} />
    <Route path=":category/:productId" element={<ProductDetail />} />
    <Route path="trending" element={<Trending />} />
  </Route>
</Routes>
```

Die `index`-Prop in diesen Beispielen soll die Standardroute für ein bestimmtes Pfadsegment darstellen. Die `Home`-Komponente wird also am Root-`/`-Pfad angezeigt, während die `ProductsHome`-Komponente am `/products`-Pfad angezeigt wird.

- **Verschachtelte Routen**: Sie können Routen innerhalb anderer Routen verschachteln, wodurch der Pfad der Kindroute an den Pfad der Elternroute angehängt wird.

```jsx
<Route path="products">
  <Route path="trending" element={<Trending />} />
</Route>
```

Im obigen Beispiel lautet der Pfad für die Trendprodukte products/trending.

- **Dynamische Segmente**: Ein dynamisches Segment ist ein Teil des URL-Pfads, der dynamisch ist.

```jsx
<Route path=":category" element={<Category />} />
```

In diesem Beispiel haben wir ein dynamisches Segment namens `category`. Wenn ein Benutzer zu einer URL wie products/brass-instruments navigiert, ändert sich die Ansicht zur `Category`-Komponente, und Sie können die passenden Daten basierend auf dem Segment dynamisch abrufen.

- **`useParams`-Hook**: Dieser Hook wird verwendet, um auf die dynamischen Parameter eines URL-Pfads zuzugreifen.

```js
import { useParams } from "react-router";

export default function Category() {
  let params = useParams();
   {/* Accessing the category param: params.category */}
   {/* rest of code goes here */}
}
```

## Zustandsverwaltung

- **Context API**: Kontext bedeutet, dass eine übergeordnete Komponente Informationen für Kindkomponenten verfügbar macht, ohne sie explizit über Props weitergeben zu müssen. `createContext` wird verwendet, um ein Kontextobjekt zu erstellen, das den Kontext darstellt, den andere Komponenten lesen. `Provider` wird verwendet, um Kontextwerte an die Kindkomponenten zu liefern.

```js
import { useState, createContext } from "react";

const CounterContext = createContext();

const CounterProvider = ({ children }) => {
  const [count, setCount] = useState(0);

  return (
    <CounterContext.Provider value={{ count, setCount }}>
      {children}
    </CounterContext.Provider>
  );
};

export { CounterContext, CounterProvider };
```

- **Redux**: Redux verwaltet den State, indem es einen zentralen Store und strenge Kontrolle über State-Updates bereitstellt. Es verwendet ein vorhersehbares Muster mit Aktionen, Reducern und Middleware. Aktionen sind Nutzlasten von Informationen, die Daten von Ihrer Anwendung an den Redux-Store senden, oft ausgelöst durch Benutzerinteraktionen. Reducer sind Funktionen, die festlegen, wie sich der State als Reaktion auf diese Aktionen ändern soll, und sorgen dafür, dass der State unveränderlich aktualisiert wird. Middleware fungiert als Brücke zwischen dem Dispatchen von Aktionen und dem Reducer, sodass Sie die Funktionalität von Redux erweitern können (z. B. Logging, asynchrone Operationen), ohne den Kernfluss zu verändern.

- **Zustand**: Diese Lösung zur Zustandsverwaltung ist ideal für kleine bis mittelgroße Anwendungen. Sie funktioniert, indem sie einen `useStore`-Hook verwendet, um direkt in Komponenten und Seiten auf den State zuzugreifen. So können Sie Daten ändern und abrufen, ohne Aktionen, Reducer oder einen Provider zu benötigen.

## React-Server-Komponenten

## Unterschiede zwischen realer und wahrgenommener Performance

- **Wahrgenommene Performance**: So nehmen Nutzer die Performance einer Website wahr. Es geht darum, wie sie die Reaktionsfähigkeit und Zuverlässigkeit bewerten. Dies ist eine subjektive Messung, daher schwer zu quantifizieren, aber sehr wichtig, da die Nutzererfahrung über Erfolg oder Misserfolg einer Website entscheidet.
- **Reale Performance**: Dies ist die objektive und messbare Performance der Website. Sie wird mit Metriken wie Seitenladezeit, Serverantwortzeit und Renderzeit gemessen. Diese Messungen werden von verschiedenen Faktoren beeinflusst, die mit dem Netzwerk und dem Code selbst zusammenhängen.

## Techniken zur Verbesserung der wahrgenommenen Performance

- **Lazy Loading**: Diese Technik reduziert die anfängliche Ladezeit so weit wie möglich, indem nicht wesentliche Ressourcen im Hintergrund geladen werden.
- **Minimieren von Schriftart-Verzögerungen**: Wenn Ihre Website benutzerdefinierte Schriftarten verwendet, sollten Sie auch versuchen, Verzögerungen beim Laden der Schriftarten zu minimieren, da dies zu Flackern oder zur Anzeige der Fallback-Schriftart führen kann, während die benutzerdefinierte Schriftart geladen wird. Ein Vorschlag ist die Verwendung einer Fallback-Schriftart, die der benutzerdefinierten ähnlich ist, sodass die Änderung subtiler ausfällt.
- **Verwendung von Ladeanzeigen**: Das Anzeigen eines Ladeindikators bei einem lang andauernden Prozess, sobald der Nutzer auf ein Element klickt, kann dem Nutzer das Gefühl geben, verbunden und eingebunden zu sein, wodurch die Wartezeit kürzer erscheint.

## Grundlegende Performance-Konzepte

- **Quellreihenfolge**: Dies bezieht sich darauf, wie HTML-Elemente im Dokument strukturiert sind. Sie bestimmt, was zuerst geladen wird und kann die Performance und Barrierefreiheit erheblich beeinflussen.

Einige bewährte Vorgehensweisen für die Quellreihenfolge sind:

- Kritische Inhalte wie Überschriften, Navigation oder Haupttext höher in der HTML-Struktur platzieren.
- Nicht wesentliche Skripte wie solche für Analytics oder Drittanbieter-Widgets aufschieben, damit sie das Rendern nicht blockieren.
- Progressive Enhancement verwenden, um sicherzustellen, dass das Kernerlebnis bereits vor dem Laden von Styles und Skripten funktioniert. Progressive Enhancement ist eine Technik zum Erstellen von Websites und Anwendungen, die darauf basiert, dass die Seite zuerst mit HTML funktioniert.

Hier ist ein Beispiel für eine gute Quellreihenfolge unter Verwendung der gerade besprochenen bewährten Vorgehensweisen:

```html
<h1>Welcome to FastSite!</h1>
<p>Critical information loads first.</p>
<script src="slow-script.js" defer></script>
```

- **Critical Rendering Path**: Dies ist die Folge von Schritten, die der Browser durchläuft, um Code in Pixel auf dem Bildschirm umzuwandeln.
- **Latenz**: Dies ist die Zeit, die eine Anfrage benötigt, um zwischen Browser und Server zu reisen. Mit anderen Worten: hohe Latenz bedeutet langsame Seiten.

Einige Möglichkeiten zur Reduzierung der Latenz sind:

- Verwendung von CDNs (Content Delivery Networks), um Dateien von näheren Standorten auszuliefern.
- Aktivierung von Kompression, z. B. mit Gzip, um Dateigrößen zu reduzieren.
- Optimierung von Bildern und Verwendung von Lazy Loading.

```html
<img src="placeholder.jpg" data-src="real-image.jpg" loading="lazy">
```

## Verbesserung von INP

- **Definition**: INP (Interaction to Next Paint) bewertet die Gesamtreaktionsfähigkeit einer Seite, indem die Zeit vom Nutzerinteraktionsereignis, wie einem Klick oder Tastendruck, bis zum nächsten Browser-Update gemessen wird. Ein niedriger INP zeigt eine reaktionsschnellere Seite an.

Hier einige Möglichkeiten zur Verbesserung von INP:

- Reduzieren Sie die Arbeit des Hauptthreads, indem Sie lange JavaScript-Aufgaben aufteilen.
- Verwenden Sie `requestIdleCallback()` für nicht-kritische Skripte. Dies fügt eine Funktion in die Warteschlange ein, die während der Leerlaufzeiten des Browsers aufgerufen wird.
- Schieben Sie schwere Assets auf oder laden Sie sie lazy, wie zuvor beschrieben.
- Optimieren Sie Ereignishandler. Wenn diese zu häufig ausgeführt werden oder schwere Operationen enthalten, können sie die Seite verlangsamen und INP erhöhen. Die Lösung ist Debouncing. Debouncing sorgt dafür, dass die Funktion erst ausgeführt wird, wenn der Nutzer für eine kurze Verzögerung aufgehört hat zu tippen – zum Beispiel 300 ms. Dies verhindert unnötige Berechnungen und verbessert die Performance.

## Wie Rendering im Browser funktioniert

- **Wie Rendering funktioniert**: Zuerst parst der Browser das HTML und baut das DOM auf. Danach verarbeitet der Browser das CSS und erstellt das CSS Object Model (CSSOM). Dies ist eine weitere Baumstruktur, die bestimmt, wie Elemente gestylt werden sollen. Schließlich malt der Browser die Pixel auf den Bildschirm und rendert jedes Element basierend auf den berechneten Styles und dem Layout. Bei komplexen Seiten können mehrere Ebenen zusammengesetzt werden, um die endgültige visuelle Ausgabe zu erzeugen.

## Wie Performance die Nachhaltigkeit beeinflusst

- **Hintergrundinformationen**: Das Internet verursacht etwa 2 % der weltweiten CO2-Emissionen – das entspricht der Luftfahrtindustrie! Jedes übertragene Byte benötigt Strom, von Rechenzentren bis zu Nutzergeräten. Größere Dateien und ineffiziente Skripte bedeuten mehr Energieverbrauch. Eine leistungsstarke Website ist nicht nur schneller, sondern reduziert auch unnötige Verarbeitung und Energieverbrauch.

## Möglichkeiten zur Reduzierung der Seitenladezeiten

- **Optimieren von Medienassets**: Große Bilder und Videos sind häufige Ursachen für langsame Ladezeiten. Durch die Optimierung dieser Assets können Sie Ihre Seite deutlich beschleunigen. Dazu gehören Komprimierung von Bildern, Verwendung moderner Formate wie WebP und Lazy Loading für Assets.
- **Nutzung von Browser-Caching**: Caching ermöglicht es Browsern, Teile Ihrer Website lokal zu speichern, wodurch die Ladezeiten für wiederkehrende Besucher reduziert werden.
- **Minimieren und Komprimieren von Dateien**: Die Reduzierung der Dateigröße führt zu schnelleren Downloads. Dazu gehört die Verringerung der übertragenen Dateigrößen sowie das Minifizieren von CSS- und JavaScript-Dateien.

## Verbesserung der „time to usable“

- **Definition**: „time to usable“ ist das Intervall vom Zeitpunkt der Seitenanforderung bis zu dem Moment, an dem der Nutzer sinnvoll mit der Seite interagieren kann. Um „time to usable“ zu verbessern, können Sie Assets lazy laden oder renderblockierende Ressourcen minimieren.

## Wichtige Metriken zur Messung der Performance

- **First Contentful Paint oder FCP**: Misst, wie schnell das erste sichtbare Inhaltselement – Text oder Bild – auf dem Bildschirm erscheint. Ein guter FCP liegt unter 1,8 Sekunden, ein schlechter über 3 Sekunden. Sie können Ihren FCP mit Chrome DevTools im Performance-Tab überprüfen.
- **Total Blocking Time**: Zeigt, wie lange der Hauptthread durch schwere JavaScript-Aufgaben blockiert ist. Ist die Total Blocking Time (TBT) hoch, erleben Nutzer träge Interaktionen. Zur Verbesserung von TBT sollten lange Aufgaben aufgeteilt und nicht wesentliche Skripte aufgeschoben werden.
- **Absprungrate**: Der Prozentsatz der Besucher, die die Seite verlassen, ohne zu interagieren. Eine hohe Absprungrate kann darauf hindeuten, dass Ihre Seite zu langsam ist.
- **Eindeutige Nutzer**: Diese Metrik verfolgt, wie viele einzelne Besucher Ihre Seite besuchen. Um Absprungrate und eindeutige Nutzer zu sehen, können Sie Google Analytics verwenden. Es ermöglicht Ihnen, diese Metriken zu überwachen und die Nutzerbindung zu verbessern.

## Übliche Tools zur Performance-Messung

- **Chrome DevTools**: Chrome DevTools ist ein eingebautes Tool in Google Chrome, mit dem Sie Performance in Echtzeit analysieren und debuggen können. DevTools zeigt Ladezeiten, CPU-Auslastung und Render-Verzögerungen an. Es ist besonders nützlich, um First Contentful Paint (FCP) zu messen, also wie schnell ein Nutzer den ersten sichtbaren Inhalt sieht. Wenn Ihre Website langsam wirkt, hilft DevTools, Engpässe zu erkennen.
- **Lighthouse**: Ein automatisiertes Tool, das Performance, SEO und Barrierefreiheit überprüft.
- **WebPageTest**: Dieses Tool testet, wie Ihre Seite von verschiedenen Standorten und Geräten geladen wird. Es liefert eine detaillierte Aufschlüsselung des Speed Index, der Total Blocking Time und anderer wichtiger Performance-Metriken. Wenn Sie wissen möchten, wie echte Nutzer Ihre Seite weltweit erleben, ist WebPageTest das richtige Tool.
- **PageSpeed Insights**: Dieses Tool analysiert Ihre Website und gibt schnelle Verbesserungsvorschläge für Mobilgeräte und Desktop. Es zeigt, was Ihre Seite verlangsamt, und gibt konkrete Empfehlungen wie Bildoptimierung, Entfernen renderblockierender Skripte und Reduzierung der Serverantwortzeiten. PageSpeed Insights ist eine schnelle und einfache Möglichkeit, zu prüfen, wie Google die Performance Ihrer Seite sieht.
- **Real User Monitoring**: RUM-Tools verfolgen das tatsächliche Nutzerverhalten und zeigen, wie echte Besucher Ihre Seite erleben. Beliebte RUM-Tools sind Google Analytics, das Seitenladezeiten und Absprungraten misst, sowie New Relic oder Datadog, die Performance-Probleme in Echtzeit überwachen. Wenn Sie Daten von echten Nutzern möchten, sind RUM-Tools unverzichtbar.

## Arbeiten mit Performance Web APIs

- **Definition**: Performance Web APIs ermöglichen Entwicklern, die Effizienz des Ladens und der Reaktion einer Webseite direkt im Code zu verfolgen. Diese APIs erlauben es, Seitenladezeiten zu messen, Render- und Interaktionsverzögerungen zu verfolgen und die Ausführungszeit von JavaScript zu analysieren.
- **`performance.now()`**: Diese API liefert hochpräzise Zeitstempel (in Millisekunden), um zu messen, wie lange verschiedene Teile Ihrer Seite zum Laden benötigen.

```js
const start = performance.now();  
// Run some code here  
const end = performance.now();  

console.log(`Execution time: ${end - start}ms`);
```

- **Performance Timing API**: Diese API gibt eine Aufschlüsselung jeder Phase des Seitenladens, vom DNS-Lookup bis zu `DOMContentLoaded`.

```js
const timing = performance.timing;  

const pageLoadTime = timing.loadEventEnd - timing.navigationStart;  
console.log(`Page load time: ${pageLoadTime}ms`);
```

- **`PerformanceObserver`**: Diese API hört auf Performance-Ereignisse wie Layout-Verschiebungen, lange Aufgaben und Nutzerinteraktionen.

```js
const observer = new PerformanceObserver((list) => {  
  list.getEntries().forEach((entry) => {  
    console.log(`Long task detected: ${entry.duration}ms`);  
  });  
});  

observer.observe({ type: "longtask", buffered: true });
```

## Techniken zur Verbesserung der CSS-Performance

- **CSS-Animationen**: Das Animieren bestimmter CSS-Eigenschaften wie Dimensionen, Position und Layout löst einen Prozess namens „Reflow“ aus, bei dem der Browser die Position und Geometrie bestimmter Elemente auf der Seite neu berechnet. Dies erfordert ein Neuzeichnen, das rechenintensiv ist. Daher wird empfohlen, die Anzahl der CSS-Animationen so weit wie möglich zu reduzieren oder dem Nutzer zumindest die Möglichkeit zu geben, sie ein- oder auszuschalten.

## Techniken zur Verbesserung der JavaScript-Performance

- **Code-Splitting**: Das Aufteilen Ihres JavaScript-Codes in Module, die kritische und nicht-kritische Aufgaben ausführen, ist ebenfalls hilfreich. So können Sie die kritischen Module so früh wie möglich vorladen und die nicht-kritischen aufschieben, um die Seite so schnell wie möglich zu rendern.
- **DOM-Manipulation**: Denken Sie daran, dass DOM-Manipulation den Prozess bezeichnet, den Inhalt einer Seite dynamisch mit JavaScript zu ändern, indem mit dem Document Object Model (DOM) interagiert wird. Die Manipulation des DOM ist rechenintensiv. Durch die Reduzierung der DOM-Manipulation in Ihrem JavaScript-Code verbessern Sie die Performance.

## CSS-Frameworks

- **CSS-Frameworks**: CSS-Frameworks können Ihren Workflow beschleunigen, einen einheitlichen visuellen Stil über eine Webseite schaffen, Ihr Design über mehrere Browser hinweg konsistent aussehen lassen und Ihren CSS-Code besser organisieren.
- **Beliebte CSS-Frameworks**: Einige der beliebten CSS-Frameworks sind Tailwind CSS, Bootstrap, Materialize und Foundation.
- **Mögliche Nachteile**:
  - Das vom Framework bereitgestellte CSS kann mit Ihrem eigenen CSS in Konflikt geraten.
  - Ihre Webseite könnte anderen Webseiten, die dasselbe Framework verwenden, ähnlich sehen.
  - Große Frameworks können Leistungsprobleme verursachen.

## Zwei Arten von CSS-Frameworks

- **Utility-first CSS-Frameworks**: Diese Frameworks haben kleine Klassen mit spezifischen Zwecken, wie das Setzen von Außenabstand, Innenabstand oder Hintergrundfarbe. Sie können diese kleinen Klassen direkt den HTML-Elementen nach Bedarf zuweisen. Tailwind CSS wird als Utility-first CSS-Framework kategorisiert.

Hier ein Beispiel für die Verwendung von Tailwind CSS zur Gestaltung eines Buttons.

```html
<button class="bg-blue-500 text-white font-bold py-2 px-4 rounded-full hover:bg-blue-700">
  Button
</button>
```

- **Komponentenbasierte CSS-Frameworks**: Diese Frameworks haben vorgefertigte Komponenten mit vordefinierten Stilen, die Sie zu Ihrer Webseite hinzufügen können. Die Komponenten sind in der offiziellen Dokumentation des CSS-Frameworks verfügbar und können in Ihr Projekt kopiert und eingefügt werden. Bootstrap wird als komponentenbasiertes CSS-Framework kategorisiert.

Hier ein Beispiel für die Verwendung von Bootstrap zur Erstellung einer Listengruppe. Anstatt kleine Klassen auf Ihre HTML-Elemente anzuwenden, fügen Sie die gesamte Komponente einschließlich der HTML-Struktur hinzu.

```html
<div class="card" style="width: 25rem;">
  <ul class="list-group list-group-flush">
    <li class="list-group-item">HTML</li>
    <li class="list-group-item">CSS</li>
    <li class="list-group-item">JavaScript</li>
  </ul>
</div>
```

Tailwind ist ein Utility-first CSS-Framework. Anstatt benutzerdefinierte CSS-Regeln zu schreiben, bauen Sie Ihre Designs, indem Sie kleine Utility-Klassen direkt in Ihrem HTML kombinieren.

Tailwind verwendet Präfixe wie `sm:`, `md:` und `lg:`, um Stile bei verschiedenen Bildschirmgrößen anzuwenden.

Tailwind enthält Utilities für CSS Grid, wie `grid`, `grid-cols-1` und `md:grid-cols-3`.

### Spacing-Utilities

Utilities wie `mt-8`, `mx-auto`, `p-4` und `gap-4` helfen, konsistente Abstände zu schaffen, ohne CSS schreiben zu müssen.

```html
<div class="mt-8 p-4 bg-indigo-600 text-white">

</div>
```

### Typografie-Utilities

Utilities wie `uppercase`, `font-bold`, `font-semibold` und `text-4xl` steuern das Aussehen von Text.

Sie können Schriftgrößen einstellen, die sich an Breakpoints anpassen, wie `text-3xl` und `md:text-5xl`.

### Farben und Hover-Zustände

Tailwind bietet eine breite Farbpalette, wie `text-red-700`, `bg-indigo-600` und `bg-gray-100`.

Klassen wie `hover:bg-pink-600` machen interaktive Effekte einfach.

- **Ringe**: `ring-1 ring-gray-300` erzeugt outline-ähnliche Effekte, die oft für Fokus oder Karten verwendet werden.

Tailwind unterstützt Verlauf-Utilities wie `bg-gradient-to-r from-fuchsia-500 to-indigo-600`.

## CSS-Präprozessoren

- **CSS-Präprozessor**: Ein CSS-Präprozessor ist ein Werkzeug, das Standard-CSS erweitert. Es kompiliert den Code mit erweiterter Syntax in eine native CSS-Datei. Es kann hilfreich sein, um saubereren, wiederverwendbaren, weniger repetitiven und skalierbaren CSS-Code für komplexe Projekte zu schreiben.
- **Features**: Einige der Features, die CSS-Präprozessoren bieten können, sind Variablen, Mixins, Verschachtelung und Selektor-Vererbung.
- **Beliebte CSS-Präprozessoren**: Einige der beliebten CSS-Präprozessoren sind Sass, Less und Stylus.
- **Mögliche Nachteile**:
  - Das Kompilieren der CSS-Regeln in Standard-CSS kann Overhead verursachen.
  - Der kompilierte Code kann schwer zu debuggen sein.

## Sass

- **Sass**: Es ist einer der beliebtesten CSS-Präprozessoren. Sass steht für „Syntactically Awesome Style Sheets“.
- **Von Sass unterstützte Features**: Sass unterstützt Features wie Variablen, verschachtelte CSS-Regeln, Module, Mixins, Vererbung und Operatoren für grundlegende mathematische Operationen.

## Zwei von Sass unterstützte Syntaxen

- **SCSS-Syntax**: SCSS (Sassy CSS) erweitert die Basissyntax von CSS. Es ist die am weitesten verbreitete Syntax für Sass. SCSS-Dateien haben die `.scss`-Erweiterung.

Hier ein Beispiel für die Definition und Verwendung einer Variablen in SCSS.

```scss
$primary-color: #3498eb;

header {
  background-color: $primary-color;
}
```

- **Eingerückte Syntax**: Die eingerückte Syntax war die ursprüngliche Syntax von Sass und ist auch als „Sass-Syntax“ bekannt.

Hier ein Beispiel für die Definition und Verwendung einer Variablen in der eingerückten Syntax.

```sass
$primary-color: #3498eb

header
  background-color: $primary-color
```

### Mixins

- **Mixins**: Mixins erlauben es Ihnen, mehrere CSS-Eigenschaften und deren Werte unter einem Namen zu gruppieren und diesen CSS-Codeblock in Ihrem Stylesheet wiederzuverwenden.

Hier ein Beispiel für die Definition eines Mixins in SCSS-Syntax. In diesem Fall heißt das Mixin `center-flex` und enthält drei CSS-Eigenschaften, um Elemente mit Flexbox zu zentrieren.

## Manuelles und automatisiertes Testen

- **Manuelles Testen**: Beim manuellen Testen prüft eine Testperson jeden Teil der Anwendung manuell und testet verschiedene Funktionen, um sicherzustellen, dass alles korrekt funktioniert. Wenn während des Testens Bugs entdeckt werden, meldet die Testperson diese an das Softwareteam, damit sie behoben werden können.
- **Automatisiertes Testen**: Beim automatisierten Testen können Sie Ihre Tests automatisieren, indem Sie ein separates Programm schreiben, das überprüft, ob Ihre Anwendung wie erwartet funktioniert.

## Unit-Testing

- **Unit-Testing**: Beim Unit-Testing testen Sie jede Funktion, um sicherzustellen, dass alles wie erwartet funktioniert. Unit-Tests können auch als eine Art Dokumentation für Ihre Anwendung dienen, da sie das erwartete Verhalten Ihres Codes darstellen sollen.
- **Single-Responsibility-Prinzip**: Das Single-Responsibility-Prinzip empfiehlt, jede Funktion klein zu halten und nur für eine Sache verantwortlich zu machen.
- **Gängige JavaScript-Testframeworks**: Zu den gängigen Testframeworks gehören Jest, Mocha und Vitest. Jest ist ein beliebtes Testframework für Unit-Tests.

Hier ist ein Beispiel für Unit-Tests mit Jest.

Zuerst können Sie eine Funktion erstellen, die für die Rückgabe eines neu formatierten Strings verantwortlich ist:

```javascript
export function getFormattedWord(str) {
  if (!str) return '';
  return str.charAt(0).toUpperCase() + str.slice(1);
}
```

In einer separaten `getFormattedWord.test.js`-Datei können Sie einige Tests schreiben, um zu überprüfen, ob die Funktion wie erwartet funktioniert. Die `getFormattedWord.test.js`-Datei sieht so aus:

```javascript
import { getFormattedWord } from "./getFormattedWord.js";
import { test, expect } from "jest";

test('capitalizes the first letter of a word', () => {
  expect(getFormattedWord('hello')).toBe('Hello');
});
```

- **`expect`-Funktion**: Die `expect`-Funktion wird verwendet, um einen Wert zu testen.
- **Matcher**: Ein Matcher ist eine Funktion, die überprüft, ob sich der Wert wie erwartet verhält. Im obigen Beispiel ist der Matcher `toBe()`. Jest bietet eine Vielzahl von Matchern.

Um Jest zu verwenden, müssen Sie zuerst das `jest`-Paket mit `npm i jest` installieren. Außerdem müssen Sie ein Skript in Ihre `package.json`-Datei hinzufügen, das so aussieht:

```js
"scripts": {
  "test": "jest"
},
```

Dann können Sie den Befehl `npm run test` ausführen, um Ihre Tests zu starten.

## Softwareentwicklungszyklus

- **Verschiedene Phasen des Softwareentwicklungszyklus**:
  - **Planungsphase**: Das Entwicklungsteam sammelt Anforderungen für die geplante Arbeit von den Stakeholdern.
  - **Designphase**: Das Softwareteam zerlegt die Anforderungen und entscheidet sich für die besten Lösungsansätze.
  - **Implementierungsphase**: Das Softwareteam zerlegt die Anforderungen in handhabbare Aufgaben und implementiert sie.
  - **Testphase**: Diese umfasst manuelles und automatisiertes Testen der neuen Arbeit. Manchmal testet das Team die Anwendung während der gesamten Entwicklungsphase, um auftretende Probleme zu erkennen und zu beheben.
  - **Deployment-Phase**: Das Team bringt die neuen Änderungen in eine Build- oder Testumgebung.
  - **Wartungsphase**: Diese umfasst das Beheben von Problemen, die bei Kunden in der produktiven Anwendung auftreten.

- **Verschiedene Modelle des Softwareentwicklungszyklus**:
  - **Wasserfallmodell**: Beim Wasserfallmodell muss jede Phase des Zyklus abgeschlossen sein, bevor die nächste Phase beginnen kann.
  - **Agiles Modell**: Das agile Modell konzentriert sich auf iterative Entwicklung, indem die Arbeit in Sprints unterteilt wird.

## BDD und TDD

- **TDD**: Testgetriebene Entwicklung ist eine Methodik, die das Schreiben von Tests zuerst betont. Das Schreiben von Tests vor der Implementierung von Funktionen liefert Entwicklern während der Entwicklung Echtzeit-Feedback.
- **BDD**: Verhaltensgetriebene Entwicklung ist der Ansatz, eine Reihe von Tests mit Geschäftszielen abzustimmen. Die Testszenarien in BDD sollten in einer Sprache geschrieben sein, die sowohl technische als auch nicht-technische Personen verstehen können. Ein Beispiel für eine solche Syntax ist Gherkin.
- **BDD-Testframeworks**: Beispiele für BDD-Testframeworks sind Cucumber, JBehave und SpecFlow.

## Zusicherungen im Unit-Testing

- **Zusicherung**: Zusicherungen werden verwendet, um zu testen, ob der Code wie erwartet funktioniert.
- **Zusicherungsbibliotheken**: Chai ist eine häufig verwendete Zusicherungsbibliothek. Weitere gängige JavaScript-Zusicherungsbibliotheken sind `should.js` und `expect.js`.

Hier ist ein Beispiel für eine Zusicherung mit Chai, die überprüft, ob der Rückgabewert der `addThreeAndFour`-Funktion gleich der Zahl 7 ist:

```js
assert.equal(addThreeAndFour(), 7);
```

- **Bewährte Vorgehensweisen**: Unabhängig davon, welche Zusicherungsbibliothek Sie verwenden, sollten Sie klare assert- und Fehlermeldungen schreiben, die Ihnen helfen zu verstehen, welche Tests fehlschlagen und warum.

## Mocking, Faking und Stubbing

- **Mocking**: Mocking ist der Prozess, echte Daten durch falsche Daten zu ersetzen, die das Verhalten echter Komponenten simulieren. Zum Beispiel könnten Sie in Tests die API-Antwort mocken, anstatt ständig API-Aufrufe zur Datenabfrage zu machen.
- **Stubbing**: Stubs sind Objekte, die vordefinierte Antworten oder Dummy-Daten für ein erwartetes Verhalten in einer Anwendung zurückgeben. Zum Beispiel können Sie das Verhalten einer Datenbankverbindung in Ihren Tests stubben, ohne auf eine echte Datenbankverbindung angewiesen zu sein.
- **Faking**: Fakes sind vereinfachte Versionen echter Komponenten ohne Komplexität oder Nebeneffekte. Zum Beispiel können Sie eine Datenbank faken, indem Sie die Daten im Speicher speichern, anstatt mit der echten Datenbank zu interagieren. So können Sie Datenbankoperationen im Speicher nachahmen, was viel schneller ist als der Umgang mit der echten Datenbank.

## Funktionstests

- **Funktionstests**: Funktionstests prüfen, ob die Funktionen und Features der Anwendung wie erwartet funktionieren. Ziel der Funktionstests ist es, das System als Ganzes gegen verschiedene Szenarien zu testen.
- **Nicht-funktionale Tests**: Nicht-funktionale Tests konzentrieren sich auf Aspekte wie Leistung und Zuverlässigkeit.
- **Smoke-Tests**: Smoke-Tests sind eine erste Überprüfung des Systems auf grundlegende oder kritische Probleme, bevor umfangreichere Tests beginnen.

## End-to-End-Tests

- **End-to-End-Tests**: End-to-End-Tests, oder E2E, testen reale Szenarien aus der Perspektive des Nutzers. E2E-Tests helfen sicherzustellen, dass Ihre Anwendung korrekt funktioniert und für Nutzer vorhersehbar ist. Allerdings ist das Einrichten, Entwerfen und Warten zeitaufwendig.
- **End-to-End-Testframeworks**: Playwright ist ein beliebtes End-to-End-Testframework, das von Microsoft entwickelt wurde. Weitere Beispiele für End-to-End-Testtools sind Cypress, Selenium und Puppeteer.

Hier ist ein Beispiel für E2E-Tests aus dem freeCodeCamp-Codebase mit Playwright. Der `beforeEach`-Hook wird vor jedem Test ausgeführt. Die Tests prüfen, ob der Spender einen Unterstützerlink in der Menüleiste hat sowie einen speziellen stilisierten Rahmen um sein Avatarbild:

```js
test.beforeEach(async ({ page }) => {
  execSync("node ./tools/scripts/seed/seed-demo-user --set-true isDonating");
  await page.goto("/donate");
});

...

test("The menu should have a supporters link", async ({ page }) => {
  const menuButton = page.getByTestId("header-menu-button");
  const menu = page.getByTestId("header-menu");

  await expect(menuButton).toBeVisible();
  await menuButton.click();

  await expect(menu).toBeVisible();

  await expect(page.getByRole("link", { name: "Supporters" })).toBeVisible();
});

test("The Avatar should have a special border for donors", async ({ page }) => {
  const container = page.locator(".avatar-container");
  await expect(container).toHaveClass("avatar-container gold-border");
});
```

## Usability-Tests

- **Usability-Tests**: Usability-Tests finden statt, wenn echte Nutzer mit der Anwendung interagieren, um herauszufinden, ob es Design-, Benutzererlebnis- oder Funktionsprobleme in der App gibt. Usability-Tests konzentrieren sich auf die Intuitivität der Anwendung für die Nutzer.
- **Vier gängige Arten von Usability-Tests**:
  - **Explorativ**: Explorative Usability-Tests beinhalten, dass Nutzer mit verschiedenen Funktionen der Anwendung interagieren, um besser zu verstehen, wie sie funktionieren.
  - **Vergleichend**: Vergleichende Tests vergleichen das Benutzererlebnis Ihrer Anwendung mit ähnlichen Anwendungen auf dem Markt.
  - **Bewertung**: Bewertungstests untersuchen, wie intuitiv die Anwendung zu bedienen ist.
  - **Validierung**: Validierungstests identifizieren größere Probleme, die verhindern könnten, dass Nutzer die Anwendung effektiv verwenden.
- **Usability-Testtools**: Beispiele für Tools zur Usability-Testung sind Loop11, Maze, Userbrain, UserTesting und UXTweak.

## Kompatibilitätstests

- **Kompatibilitätstests**: Ziel der Kompatibilitätstests ist es sicherzustellen, dass Ihre Anwendung in verschiedenen Computerumgebungen funktioniert.
- **Verschiedene Arten von Kompatibilitätstests**:
  - **Abwärtskompatibilität**: Abwärtskompatibilität bedeutet, dass die Software mit früheren Versionen kompatibel ist.
  - **Aufwärtskompatibilität**: Aufwärtskompatibilität bedeutet, dass die Software und Systeme mit zukünftigen Versionen funktionieren.
  - **Hardware-Kompatibilität**: Hardware-Kompatibilität ist die Fähigkeit der Software, in verschiedenen Hardwarekonfigurationen richtig zu funktionieren.
  - **Betriebssystem-Kompatibilität**: Betriebssystem-Kompatibilität ist die Fähigkeit der Software, auf verschiedenen Betriebssystemen wie macOS, Windows und Linux-Distributionen wie Ubuntu und Fedora zu laufen.
  - **Netzwerk-Kompatibilität**: Netzwerk-Kompatibilität bedeutet, dass die Software unter verschiedenen Netzwerkbedingungen funktioniert, z. B. bei unterschiedlichen Netzwerkgeschwindigkeiten, Protokollen, Sicherheitseinstellungen usw.
  - **Browser-Kompatibilität**: Browser-Kompatibilität bedeutet, dass die Webanwendung in verschiedenen Browsern wie Google Chrome, Safari, Firefox usw. konsistent funktioniert.
  - **Mobile Kompatibilität**: Es ist wichtig sicherzustellen, dass Ihre Softwareanwendungen auf verschiedenen Android- und iOS-Geräten funktionieren, einschließlich Smartphones und Tablets.

## Leistungstests

- **Leistungstests**: Bei Leistungstests prüfen Sie die Geschwindigkeit, Reaktionsfähigkeit, Skalierbarkeit und Stabilität einer Anwendung unter verschiedenen Arbeitslasten. Ziel ist es, jegliche Leistungsengpässe zu beheben.
- **Verschiedene Arten von Leistungstests**:
  - **Lasttests**: Lasttests bestimmen, wie sich ein System während normaler und Spitzenlastzeiten verhält.
  - **Stresstests**: Stresstests prüfen, wie Ihre Anwendung unter extremen Lasten reagiert.
  - **Soak-Tests (Dauertests)**: Soak-Tests oder Dauertests sind eine Art Lasttest, bei dem das System über einen längeren Zeitraum mit hoher Last getestet wird.
  - **Spike-Tests**: Spike-Tests erhöhen und verringern die Last dramatisch und analysieren die Reaktionen des Systems auf diese Änderungen.
  - **Breakpoint-Tests (Kapazitätstests)**: Breakpoint-Tests oder Kapazitätstests erhöhen die Last langsam über die Zeit, bis das System zu versagen oder sich zu verschlechtern beginnt.

## Sicherheitstests

- **Sicherheitstests**: Sicherheitstests helfen, Schwachstellen und Sicherheitslücken zu identifizieren.
- **Sicherheitsgrundsätze**:
  - **Vertraulichkeit**: Schützt davor, dass sensible Informationen an andere Empfänger als den vorgesehenen weitergegeben werden.
  - **Integrität**: Verhindert, dass böswillige Nutzer Benutzerinformationen verändern.
  - **Authentifizierung**: Überprüft die Identität des Nutzers, um sicherzustellen, dass er berechtigt ist, das System zu verwenden.
  - **Autorisierung**: Bestimmt, welche Aktionen authentifizierte Nutzer ausführen dürfen oder auf welche Teile des Systems sie zugreifen dürfen.
  - **Verfügbarkeit**: Stellt sicher, dass Informationen und Dienste für berechtigte Nutzer verfügbar sind, wenn sie sie benötigen.
  - **Nichtabstreitbarkeit**: Stellt sicher, dass sowohl Absender als auch Empfänger einen Nachweis über die Zustellung und die Identität des Absenders haben. Schützt davor, dass der Absender die Übermittlung der Informationen abstreitet.
- **Gängige Sicherheitsbedrohungen**:
  - **Cross-Site-Scripting (XSS)**: XSS-Angriffe treten auf, wenn ein Angreifer bösartige Skripte in eine Webseite einschleust und diese im Kontext des Browsers des Opfers ausführt.
  - **SQL-Injection**: SQL-Injection ermöglicht es böswilligen Nutzern, schädlichen Code in eine Datenbank einzuschleusen.
  - **Denial-of-Service (DoS)-Angriff**: Ein DoS-Angriff ist, wenn böswillige Nutzer eine Webseite mit einer hohen Anzahl von Anfragen oder Traffic überfluten, wodurch der Server verlangsamt wird und möglicherweise abstürzt, sodass die Seite für Nutzer nicht mehr verfügbar ist.
- **Kategorien von Sicherheitstesttools**:
  - **Static Application Security Testing**: Diese Tools analysieren den Quellcode einer Anwendung, um Sicherheitslücken zu identifizieren.
  - **Dynamic Application Security Testing**: Diese Tools interagieren mit dem Frontend der Anwendung, um potenzielle Sicherheitslücken aufzudecken. DAST-Tools haben keinen Zugriff auf den Quellcode.
- **Penetrationstest (Pentest)**: Penetrationstests sind eine Art von Sicherheitstest, bei dem simulierte Cyberangriffe auf die Anwendung durchgeführt werden, um Schwachstellen im System zu identifizieren.

## A/B-Tests

- **A/B-Tests**: A/B-Tests vergleichen zwei Versionen einer Seite oder Anwendung und untersuchen, welche Version besser abschneidet. Sie werden auch als Bucket- oder Split-Tests bezeichnet. A/B-Tests ermöglichen es Ihnen, datengesteuerte Entscheidungen zu treffen und das Benutzererlebnis kontinuierlich zu verbessern.
- **Tools für A/B-Tests**: Beispiele für Tools zur Durchführung von A/B-Tests sind GrowthBook und LaunchDarkly.

## Alpha- und Beta-Tests

Sobald die erste Entwicklung und das Softwaretesten abgeschlossen sind, ist es wichtig, die Anwendung von Testern und echten Nutzern testen zu lassen. Hier kommen Alpha- und Beta-Tests ins Spiel.

- **Alpha-Tests**: Alpha-Tests werden von einer ausgewählten Gruppe von Testern durchgeführt, die die Anwendung durchgehen, um sicherzustellen, dass keine Bugs vorhanden sind, bevor sie auf den Markt gebracht wird. Alpha-Tests sind Teil der Abnahmetests und nutzen sowohl White-Box- als auch Black-Box-Testtechniken.
- **Beta-Tests**: Beta-Tests sind, wenn die Anwendung echten Nutzern zur Verfügung gestellt wird. Nutzer können mit der Anwendung interagieren und Feedback geben. Beta-Tests sind ebenfalls eine Form von Benutzerakzeptanztests.
- **Abnahmetests**: Abnahmetests stellen sicher, dass die Softwareanwendung die Geschäftsanforderungen und die Bedürfnisse der Nutzer vor der Veröffentlichung erfüllt.
- **Black-Box-Tests**: Black-Box-Tests konzentrieren sich nur auf das erwartete Verhalten der Anwendung.
- **White-Box-Tests**: White-Box-Tests beinhalten, dass der Tester die internen Komponenten kennt und Tests an ihnen durchführt.

## Regressionstests

- **Regression**: Regression bezeichnet Situationen, in denen neue Änderungen unbeabsichtigt bestehende Funktionalität zerstören.
- **Regressionstests**: Regressionstests helfen, Regressionen zu erkennen. Dabei werden funktionale Tests erneut auf Teile Ihrer Anwendung angewendet, um sicherzustellen, dass alles weiterhin wie erwartet funktioniert.
- **Tools für Regressionstests**: Tools, die Sie für Regressionstests verwenden können, sind Puppeteer, Playwright, Selenium und Cypress.
- **Techniken für Regressionstests**:
  - **Unit-Regressionstests**: Dabei gibt es eine Liste von Elementen, die bei jeder größeren Änderung oder Fehlerbehebung in der App getestet werden müssen.
  - **Partielle Regressionstests**: Diese beinhalten gezielte Ansätze, um sicherzustellen, dass neue Änderungen bestimmte Aspekte der Anwendung nicht zerstört haben.
  - **Vollständige Regressionstests**: Diese führen Tests für alle Funktionalitäten im Codebase aus. Dies ist die zeitaufwändigste und detaillierteste Option.
- **Retests**: Retests werden verwendet, um bekannte Probleme zu überprüfen und sicherzustellen, dass sie behoben wurden. Im Gegensatz dazu suchen Regressionstests nach unbekannten Problemen, die durch kürzliche Änderungen im Codebase entstanden sein könnten.

## Was ist TypeScript

- **JavaScript**: JavaScript ist eine dynamisch typisierte Sprache. Das bedeutet, dass Variablen zur Laufzeit beliebige Werte erhalten können. Die Herausforderung einer dynamisch typisierten Sprache besteht darin, dass das Fehlen von Typensicherheit Fehler verursachen kann.

Zum Beispiel können Sie eine JavaScript-Funktion, die ein Array erwartet, trotzdem mit einer Zahl aufrufen:

```javascript
const getRandomValue = (array) => {
  return array[Math.floor(Math.random() * array.length)];
}

console.log(getRandomValue(10));
```

Die `console` Ausgabe für das obige Beispiel wird `undefined` sein.

- **TypeScript**: TypeScript erweitert die JavaScript-Sprache um statische Typisierung, die dabei hilft, Fehler durch Typinkonsistenzen zu erkennen, bevor Sie Ihren Code ausführen.

Zum Beispiel können Sie einen Typ für den `array` Parameter wie folgt definieren:

```typescript
const getRandomValue = (array: string[]) => {
  return array[Math.floor(Math.random() * array.length)];
}
```

Diese Typdefinition sagt TypeScript, dass der `array` Parameter ein Array von Strings sein muss. Wenn Sie dann `getRandomValue` aufrufen und eine Zahl übergeben, erhalten Sie einen Fehler, der als Compiler-Fehler bezeichnet wird.

- **Compiler**: Sie müssen TypeScript-Code zuerst in reguläres JavaScript kompilieren. Wenn Sie den Compiler ausführen, bewertet TypeScript Ihren Code und wirft einen Fehler bei allen Problemen, bei denen die Typen nicht übereinstimmen.

## Datentypen in TypeScript

- **Primitive Datentypen in TypeScript**: Für die primitiven Datentypen `string`, `null`, `undefined`, `number`, `boolean` und `bigint` bietet TypeScript entsprechende Typ-Schlüsselwörter.

```typescript
let str: string = "Naomi";
let num: number = 42;
let bool: boolean = true;
let nope: null = null;
let nada: undefined = undefined;
```

- **Array**: Sie können ein Array eines bestimmten Typs mit zwei verschiedenen Syntaxen definieren.

```typescript
const arrOne: string[] = ["Naomi"];
const arrTwo: Array<string> = ["Camperchan"];
```

- **Objekte**: Sie können die genaue Struktur eines Objekts definieren.

```typescript
const obj: { a: string, b: number } = { a: "Naomi", b: 42 };
```

Wenn Sie ein Objekt mit beliebigen Schlüsseln möchten, bei dem aber alle Werte Strings sein müssen, gibt es zwei Möglichkeiten, dies zu definieren:

```typescript
const objOne: Record<string, string> = {};
const objTwo: { [key: string]: string } = {};
```

- **Weitere hilfreiche Typen in TypeScript**:
  - **`any`**: `any` zeigt an, dass ein Wert jeden Typ haben kann. Es sagt dem Compiler, dass er sich nicht mehr um den Typ dieser Variable kümmern soll.
  - **`unknown`**: `unknown` sagt TypeScript, dass Ihnen der Typ des Werts _wichtig_ ist, Sie aber tatsächlich nicht wissen, welcher Typ es ist. `unknown` wird im Allgemeinen gegenüber `any` bevorzugt.
  - **`void`**: Dies ist ein spezieller Typ, den Sie normalerweise nur beim Definieren von Funktionen verwenden. Funktionen, die keinen Rückgabewert haben, verwenden den Rückgabetyp `void`.
  - **`never`**: Er repräsentiert einen Typ, der niemals existieren wird.

- **`type` Schlüsselwort**: Dieses Schlüsselwort ist wie `const`, aber anstatt eine Variable zu deklarieren, können Sie einen Typ deklarieren.

Es ist nützlich, um benutzerdefinierte Typen wie Union-Typen oder Typen zu deklarieren, die nur bestimmte Werte enthalten:

```typescript
type stringOrNumber = string | number;
type bot = "camperchan" | "camperbot" | "naomi";
```

- **`interface`**: Interfaces sind wie Klassen für Typen. Sie können andere Interfaces implementieren oder erweitern, sind speziell Objekttypen und werden im Allgemeinen bevorzugt, es sei denn, Sie benötigen eine bestimmte Funktion, die eine `type` Deklaration bietet.

```typescript
interface wowie {
  zowie: boolean;
  method: () => void;
}
```

- **Rückgabetyp definieren**: Sie können auch den _Rückgabetyp_ der Funktion definieren.

Das folgende Beispiel definiert den Rückgabewert als String. Wenn Sie versuchen, etwas anderes zurückzugeben, gibt TypeScript einen Compiler-Fehler aus.

```typescript
const getRandomValue = (array: string[]): string => {
  return array[Math.floor(Math.random() * array.length)];
}
```

## Generics

- **Generischen Typ definieren**: Sie können einen generischen Typ definieren und in Ihrer Funktion darauf verweisen. Man kann ihn sich als einen speziellen Parameter vorstellen, den Sie einer Funktion geben, um das Verhalten der Typdefinition der Funktion zu steuern.

Hier ist ein Beispiel für die Definition eines generischen Typs für eine Funktion:

```typescript
const getRandomValue = <T>(array: T[]): T => {
  return array[Math.floor(Math.random() * array.length)];
}
const val = getRandomValue([1, 2, 4])
```

Die `<T>` Syntax sagt TypeScript, dass Sie einen generischen Typ `T` für die Funktion definieren. `T` ist ein gebräuchlicher Name für generische Typen, aber Sie können jeden Namen verwenden.

Dann sagen Sie TypeScript, dass der `array` Parameter ein Array von Werten ist, die dem generischen Typ entsprechen, und dass der zurückgegebene Wert ein einzelnes Element desselben Typs ist.

- **Typ-Argument im Funktionsaufruf angeben**: Sie können ein Typ-Argument an einen Funktionsaufruf übergeben, indem Sie spitze Klammern zwischen dem Funktionsnamen und seinen Parametern verwenden.

Hier ist ein Beispiel, wie Sie ein Typ-Argument an einen Funktionsaufruf übergeben:

```typescript
const email = document.querySelector<HTMLInputElement>("#email");
console.log(email.value);
```

Das sagt TypeScript, dass das Element, das Sie erwarten zu finden, ein Eingabeelement sein wird.

## Typverengung

- **Verengung durch Wahrheitswertprüfung**: Im folgenden Beispiel erhalten Sie einen Compiler-Fehler, wenn Sie versuchen, auf die `value` Eigenschaft von `email` zuzugreifen, weil `email` _möglicherweise_ `null` ist.

```typescript
const email = document.querySelector<HTMLInputElement>("#email");
console.log(email.value);
```

Sie können eine Bedingung verwenden, um zu bestätigen, dass `email` _wahr_ ist, bevor Sie auf die Eigenschaft zugreifen:

```typescript
const email = document.querySelector<HTMLInputElement>("#email");
if (email) {
  console.log(email.value);
}
```

Wahrheitswertprüfungen können auch in die entgegengesetzte Richtung funktionieren:

```typescript
const email = document.querySelector<HTMLInputElement>("#email");
if (!email) {
  throw new ReferenceError("Could not find email element!")
}
console.log(email.value);
```

Das Werfen eines Fehlers beendet die logische Ausführung dieses Codes, was bedeutet, dass wenn Sie den `console.log()` Aufruf erreichen, TypeScript weiß, dass `email` _nicht_ `null` sein kann.

- **Optionale Verkettung**: Optionale Verkettung `?.` ist ebenfalls eine Form der Typverengung, basierend auf der gleichen Prämisse, dass der Eigenschaftszugriff nicht stattfinden kann, wenn der `email` Wert `null` ist.

```typescript
const email = document.querySelector<HTMLInputElement>("#email");
console.log(email?.value);
```

- **`typeof` Operator**: Sie können eine Bedingung verwenden, um den Typ einer Variablen mit dem `typeof` Operator zu prüfen.

```typescript
const myVal = Math.random() > 0.5 ? 222 : "222";
if (typeof myVal === "number") {
  console.log(myVal / 10);
}
```

- **`instanceof` Schlüsselwort**: Wenn das Objekt von einer Klasse stammt, können Sie das `instanceof` Schlüsselwort verwenden, um den Typ zu verengen.

```typescript
const email = document.querySelector("#email");

if (email instanceof HTMLInputElement) {
    console.log(email.value);
}
```

- **Casting**: Wenn TypeScript den Typ eines Werts nicht automatisch bestimmen kann, wie das Ergebnis der `request.json()` Methode im folgenden Beispiel, erhalten Sie einen Compiler-Fehler. Eine Möglichkeit, dies zu beheben, ist das Casten des Typs, aber dadurch wird TypeScripts Fähigkeit, potenzielle Fehler zu erkennen, geschwächt.

```typescript
interface User {
    name: string;
    age: number;
}

const printAge = (user: User) => 
  console.log(`${user.name} is ${user.age} years old!`)

const request = await fetch("url")
const myUser = await request.json() as User;
printAge(myUser);
```

- **Type Guard**: Anstatt den Typ zu casten, können Sie einen Type Guard schreiben:

```typescript
interface User {
    name: string;
    age: number;
}

const isValidUser = (user: unknown): user is User => {
  return !!user && 
    typeof user === "object" &&
    "name" in user &&
    "age" in user;
}
```

Die `user is User` Syntax zeigt an, dass Ihre Funktion einen Boolean-Wert zurückgibt, der bei true bedeutet, dass der `user` Wert das `User` Interface erfüllt.

## `tsconfig` Datei

- **`tsconfig.json`**: Die Compiler-Einstellungen von TypeScript befinden sich in einer `tsconfig.json` Datei im Stammverzeichnis Ihres Projekts.

```json
{
  "compilerOptions": {
    "rootDir": "./src",
    "outDir": "./prod",
    "lib": ["ES2023"],
    "target": "ES2023",
    "module": "ES2022",
    "moduleResolution": "Node",
    "esModuleInterop": true,
    "skipLibCheck": true,
    "strict": true
  },
  "exclude": ["test/"]
}
```

Hier sind Beschreibungen der Compiler-Optionen, die im obigen Beispiel verwendet werden:

- **`compilerOptions`**: Die `compilerOptions` Eigenschaft steuert, wie sich der TypeScript-Compiler verhält.
- **`rootDir` und `outDir`**: Die `rootDir` und `outDir` geben TypeScript an, welches Verzeichnis Ihre Quelldateien enthält und welches Verzeichnis den transpilierten JavaScript-Code enthalten soll.
- **`lib`**: Die `lib` Eigenschaft bestimmt, welche Typdefinitionen der Compiler verwendet, und ermöglicht es Ihnen, Unterstützung für bestimmte ES-Versionen, das DOM und mehr einzuschließen.
- **`module` und `moduleResolution`**: Die `module` und `moduleResolution` arbeiten zusammen, um zu steuern, wie Ihr Paket Module verwendet – entweder CommonJS oder ECMAScript.
- **`esModuleInterop`**: Die `esModuleInterop` ermöglicht eine reibungslosere Interoperabilität zwischen CommonJS- und ES-Modulen, indem automatisch Namespace-Objekte für Importe erstellt werden.
- **`skipLibCheck`**: Die `skipLibCheck` Option überspringt die Validierung von `.d.ts` Dateien, die nicht durch Importe in Ihrem Code referenziert werden.
- **`strict`**: Das `strict` Flag aktiviert mehrere Prüfungen, wie die korrekte Behandlung von nullable Typen und warnt, wenn TypeScript auf `any` zurückfällt.
- **`exclude`**: Die oberste Ebene `exclude` Eigenschaft weist den Compiler an, diese TypeScript-Dateien während der Kompilierung zu ignorieren.
