# --interactive--

HTMLテーブルは、かつてほど頻繁には使われなくなりました。しかし、フロントエンド開発者としては、依然として理解しておくべきです。テーブルは1990年代にブラウザでデータを表示するために開発者が使い始めた最も古い方法の一つです。

こちらは米国労働統計局のデータからテーブルを生成するためのコード例です。

:::interactive_editor

```html
<table id="quickfacts">
  <thead>
    <tr>
      <th colspan="2">Quick Facts: Software Developers, Quality Assurance Analysts, and Testers</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <th>2023 Median Pay</th>
      <td>
        $130,160 per year
        <br>$62.58 per hour
      </td>
    </tr>
    <tr>
      <th>Typical Entry-Level Education</th>
      <td>Bachelor's degree</td>
    </tr>
    <tr>
      <th>Work Experience in a Related Occupation</th>
      <td>None</td>
    </tr>
    <tr>
      <th>On-the-job Training</th>
      <td>None</td>
    </tr>
    <tr>
      <th>Number of Jobs, 2022</th>
      <td>1,795,300</td>
    </tr>
    <tr>
      <th>Job Outlook, 2022-32</th>
      <td>25% (Much faster than average)</td>
    </tr>
    <tr>
      <th>Employment Change, 2022-32</th>
      <td>451,200</td>
    </tr>
  </tbody>
  <tfoot>
    <tr>
      <th>If this table had a footer it would be here.</th>
    </tr>
  </tfoot>
</table>
```

:::

ご覧の通り、メインの`table`要素があり、その中に`id`が`quickfacts`に設定されています。テーブルの中には、テーブルヘッド要素の`thead`、テーブルボディ要素の`tbody`、そしてテーブルフット要素の`tfoot`があります。

テーブルヘッド、ボディ、フットの各要素は複数のテーブル行、`tr`を含むことができます。そして各テーブル行は、その行のデータをラベル付けするテーブルヘッダー`th`を含むことができます。また、複数の個別データセル、テーブルデータ`td`を含むこともあります。

これだけ多くのHTML要素がありますが、怖がる必要はありません。これらはシンプルな階層構造に従っています。

これらすべての要素を含む最もシンプルなテーブルは以下の通りです。

:::interactive_editor

```html
<table>
  <thead>
    <tr>
      <th>The title of this table</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <th>First Row</th>
      <td>
        First Data Cell
      </td>
    </tr>
    <tr>
      <th>Second Row</th>
      <td>
        Second Data Cell
      </td>
    </tr>
  </tbody>
  <tfoot>
    <tr>
      <th>The footer of this table, which might contain date of publication, author credits, or other meta information.</th>
    </tr>
  </tfoot>
</table>
```

:::

つまり、データ自体は常に`tr`の中にあり、その`tr`要素の中にヘッダーを持つ`th`要素とデータを持つ`td`要素があります。

一部のウェブサイトでは、より適切な`div`要素の代わりに`table`を使って独自のテーブルを作ることがあります。

一般的な`div`要素を使って表形式のデータを表示することも可能ですが、`table`要素を使う方が望ましいです。

昔は、開発者がウェブページ上の非データ要素の配置に`table`を使うこともありましたが、これはベストプラクティスとは見なされていませんでした。しかし、まだそのようなテーブルの使い方をしているコードベースに出会うことがあります。

現在では、開発者はCSSのflexboxやgridを使ってレイアウトを組みます。freeCodeCampではこれらのツールを後ほど詳しく扱います。

今は、HTMLテーブルは本来の目的である表形式データの表示に使うだけにしてください。
