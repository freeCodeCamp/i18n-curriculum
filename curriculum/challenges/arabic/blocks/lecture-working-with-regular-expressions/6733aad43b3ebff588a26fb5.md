# --interactive--

التعبيرات النمطية، أو regex، هي ميزة تدعمها العديد من لغات البرمجة المختلفة.

التعبير النمطي هو صيغة نحوية خاصة لإنشاء "نمط"، يمكنك استخدامه بعد ذلك للتحقق من سلسلة نصية، استخراج نص، والمزيد.

لنلق نظرة على تعبير نمطي أساسي:

```js
const regex = /freeCodeCamp/;
```

لاحظ كيف في JavaScript، تُعرّف التعبير النمطي بإنشاء نمطك بين شرطتين مائلتين للأمام (`/`). حاول ألا تخلط بين هذا وبين التعليق، حيث يأتي النص بعد كلتا الشرطتين المائلتين.

هذا التعبير النمطي المحدد سيطابق النص `freeCodeCamp`، مع حرفي `C` كبيرين، في أي مكان داخل السلسلة النصية. لكن كيف يمكنك فعل ذلك فعليًا؟

هذا يقودنا إلى أول طريقة لدينا، وهي طريقة `test()`. طريقة `test()` موجودة على كائنات `RegExp`، وهي كائنات تمثل تعبيرًا نمطيًا (مثل الذي عرفناه للتو).

طريقة `test()` تقبل سلسلة نصية، وهي السلسلة التي نختبر وجود تطابق لها مع التعبير النمطي. على سبيل المثال، دعنا نجرب اختبار السلسلة `e`:

:::interactive_editor

```js
const regex = /freeCodeCamp/;
const test = regex.test("e");
console.log(test);
```

:::

يمكنك أن ترى أننا استدعينا طريقة `test()` على التعبير النمطي الجديد، ومررنا السلسلة `e` كمعلمة.

طريقة `test()` أعادت القيمة `false` لأن السلسلة `e` لا تطابق النمط `freeCodeCamp`. رغم أن النمط `freeCodeCamp` يحتوي على الحرف `e`، إلا أن هذا عكس كيفية عمل التعبيرات النمطية.

لنلق نظرة على بعض الأمثلة الأخرى. خذ لحظة للتفكير فيها:

:::interactive_editor

```js
const regex = /freeCodeCamp/;
console.log(regex.test("freeCodeCamp"));
console.log(regex.test("freeCodeCamp is great"));
console.log(regex.test("I love freeCodeCamp"));
console.log(regex.test("freecodecamp"));
console.log(regex.test("FREECODECAMP"));
console.log(regex.test("free"));
console.log(regex.test("code"));
console.log(regex.test("camp"));
```

:::

لاحظ كيف أن أول ثلاث سلاسل أعادت القيمة `true`. هذه السلاسل كلها تحتوي على النص `freeCodeCamp` بالضبط، في مكان ما داخل السلسلة.

السطور 5 و6 تعيد `false`. رغم احتوائها على النص `freecodecamp`، إلا أن حالة الأحرف لا تتطابق. التعبيرات النمطية حساسة لحالة الأحرف افتراضيًا.

أخيرًا، بينما تحتوي السلاسل الثلاث الأخيرة على جزء من النمط، إلا أنها لا تحتوي على النمط الكامل.

طريقة `test()` تعيد قيمة منطقية (`boolean`) تشير إلى ما إذا كانت السلسلة تطابق التعبير النمطي أم لا.

لكن ماذا لو أردت معلومات أكثر من ذلك؟ حسنًا، السلاسل النصية تحتوي على طريقة `match()`. هذه الطريقة تقبل تعبيرًا نمطيًا، رغم أنه يمكنك أيضًا تمرير سلسلة نصية سيتم تحويلها إلى تعبير نمطي.

طريقة `match()` تعيد مصفوفة التطابقات للسلسلة. ما هي مصفوفة التطابق؟ لنلق نظرة:

```js
const regex = /freeCodeCamp/;
const match = "freeCodeCamp".match(regex);
console.log(match);
```

إذا شغلنا هذا، نحصل على مصفوفة! لكنها تبدو غريبة بعض الشيء. لديها بعض الخصائص الإضافية:

```js
// [
//   'freeCodeCamp',
//   index: 0,
//   input: 'freeCodeCamp',
//   groups: undefined
// ]
```

خاصية `groups` ستُظهر أي مجموعات تم التقاطها. ستتعلم ما يعني ذلك في درس لاحق.

خاصية `index` تخبرك في أي حرف من السلسلة وُجد التطابق. في حالتنا، وُجد في بداية السلسلة.

خاصية `input` تخبرك بالسلسلة التي استُدعيت عليها طريقة `match()`.

دعنا نجرب المزيد مرة أخرى، ونرى كيف يتغير الناتج:

```js
const regex = /freeCodeCamp/;
console.log("freeCodeCamp".match(regex)); // ['freeCodeCamp', index: 0, input: 'freeCodeCamp', groups: undefined]
console.log("freeCodeCamp is great".match(regex)); // ['freeCodeCamp', index: 0, input: 'freeCodeCamp is great', groups: undefined]
console.log("I love freeCodeCamp".match(regex)); // ['freeCodeCamp', index: 7, input: 'I love freeCodeCamp', groups: undefined]
console.log("freecodecamp".match(regex)); // null
console.log("FREECODECAMP".match(regex)); // null
console.log("free".match(regex)); // null
```

نعلم بالفعل أن أول ثلاث سلاسل يجب أن تُنتج تطابقًا، فلنلق نظرة عليها:

```js
// ['freeCodeCamp', index: 0, input: 'freeCodeCamp', groups: undefined]
// ['freeCodeCamp', index: 0, input: 'freeCodeCamp is great', groups: undefined]
// ['freeCodeCamp', index: 7, input: 'I love freeCodeCamp', groups: undefined]
```

هل هذا ما توقعت؟ يمكنك أن ترى كيف تغيرت خاصيتي `input` و`index` حسب السلسلة المقدمة وموقع التطابق داخلها.

السطور الثلاثة الأخرى، التي لا تطابق، تعيد `null` بدلًا من مصفوفة.

الآن بعد أن يمكننا اختبار ومطابقة السلاسل باستخدام التعبير النمطي، ماذا لو أردنا استبدال محتوى سلسلة؟ ربما كتب شخص ما `freecodecamp` بأحرف صغيرة كلها، ونريد تصحيح الحالة تلقائيًا.

أولًا، نحتاج إلى تحديث التعبير النمطي لدينا ليطابق الشكل بأحرف صغيرة من `freecodecamp`، وننشئ سلسلة الاختبار:

```js
const regex = /freecodecamp/;
const str = "freecodecamp is rly kewl";
```

الآن، السلاسل النصية تحتوي على طريقة `replace()` التي تقبل معلمتين: التعبير النمطي للمطابقة (أو سلسلة نصية إذا لم تكن بحاجة لكل ميزات التعبيرات النمطية)، والسلسلة التي تستبدل بها التطابق (أو دالة تُنفذ على كل تطابق).

إذا أردنا استبدال `freecodecamp` بالحالة الصحيحة:

:::interactive_editor

```js
const regex = /freecodecamp/;
const str = "freecodecamp is rly kewl";
const replaced = str.replace(regex, "freeCodeCamp");
console.log(replaced);
```

:::

يمكنك أن ترى أن الدالة `replace()` تُرجع السلسلة النصية المحدثة مع النمط المُطابق الذي تم `replaced`.

التعبيرات النمطية، وكل الطرق المرتبطة بها، قد تبدو معقدة ومربكة. لكنك ستحصل على فرصة لاستكشافها أكثر في مجموعة الدروس التالية.
