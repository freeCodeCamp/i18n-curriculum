# --description--

CSP (Content Security Policy) und Permissions-Policy sind zwei mächtige HTTP-Sicherheitsheader, die Webanwendungen schützen, indem sie einschränken, welche Ressourcen geladen werden dürfen und auf welche Browserfunktionen eine Seite zugreifen kann.

Content Security Policy konzentriert sich hauptsächlich auf das Laden von Ressourcen. Sie fungiert als Sicherheitsfilter für Ihre Frontend-Skripte, Styles und Bilder.

Böswillige Akteure versuchen oft, schädliche Skripte in verwundbare Websites einzuschleusen. Eine starke CSP verhindert diese Cross-Site-Scripting-(XSS)-Angriffe, indem sie eine strenge Whitelist vertrauenswürdiger Quell-Domains festlegt.

Wenn ein Server den CSP-Header sendet, liest der Browser die spezifischen Direktiven. Zum Beispiel teilt die Direktive `script-src` dem Browser mit, welche Domains JavaScript sicher auf der Seite ausführen dürfen.

Versucht ein Hacker, ein bösartiges Skript von einem nicht genehmigten externen Server einzuschleusen, erkennt der Browser den Verstoß gegen die Richtlinie, verwirft das Skript sofort und protokolliert einen Sicherheitsfehler.

CSP kann auch Bildquellen, Formularziele und Stylesheet-Ursprünge einschränken.

Permissions-Policy konzentriert sich auf Browserfunktionen und Gerätehardware. Sie hat den älteren Feature-Policy-Header ersetzt.

Dieser Sicherheitsmechanismus erlaubt es Seitenbetreibern, bestimmte Browserfunktionen für ihre eigene Seite und eingebettete iframes explizit zu aktivieren oder zu deaktivieren und verhindert, dass Drittanbieter-Snippets die Privatsphäre der Nutzer missbrauchen.

Mit einem Permissions-Policy-Header können Sie den Zugriff auf Hardware wie Webcam, Mikrofon und Geolokalisierung steuern.

Wenn Ihre Website zum Beispiel den Standort eines Nutzers nicht benötigt, können Sie die Richtlinie so einstellen, dass die Geolocation-API vollständig blockiert wird. Selbst wenn ein Angreifer ein Skript auf Ihrer Seite kompromittiert, weigert sich der Browser, die Tracking-Funktion zu aktivieren.

Die Umsetzung der Permissions-Policy verwendet ein klares, strukturiertes Header-Format, das die Funktion benennt und ihre erlaubten Ursprünge definiert.

Um eine Hardwarefunktion strikt auf Ihre eigene Domain zu beschränken, verwenden Sie das Schlüsselwort `self` in runden Klammern, zum Beispiel `camera=(self)`. Möchten Sie eine Funktion für alle Nutzer auf der gesamten Seite deaktivieren, verwenden Sie stattdessen eine leere Liste, die so aussieht: `geolocation=()`.

Die Kombination beider Header bietet einen soliden Schutz für Webanwendungen. CSP sichert die Datenübertragung ab und verhindert Code-Injektionen, während Permissions-Policy das physische Gerät schützt und die Einhaltung des Datenschutzes gewährleistet.

Moderne Webentwickler nutzen beide Werkzeuge, um eine stark gesicherte Browserumgebung zu schaffen.
