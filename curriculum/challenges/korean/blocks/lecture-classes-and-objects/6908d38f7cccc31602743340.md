# --description--

이전 수업에서 속성은 객체에 속하는 변수라는 것을 배웠습니다. 즉, 속성은 객체의 상태나 동작을 설명하는 데이터를 담고 있습니다.

예를 들어, 자동차는 보통 브랜드와 모델을 가집니다. 브랜드와 모델은 `Car` 클래스의 속성이 될 수 있습니다:

```python
class Car: 
    def __init__(self, brand, model): 
        self.brand = brand 
        self.model = model 

my_car = Car('Lamborghini', 'Gallardo') 
print(my_car.brand) # Lamborghini 
print(my_car.model) # Gallardo 
```

하지만 때로는 프로그램이 실행되는 동안에야 어떤 속성이 필요한지 알 수 있을 때가 있습니다. 사용자가 입력하거나 설정 파일에서 속성 이름을 받는 스크립트를 작성한다고 상상해 보세요. 그런 속성들은 미리 코드에 하드코딩할 수 없습니다.

이럴 때 동적으로 속성을 다루는 방법이 필요합니다. 이렇게 하면 속성 이름을 변수처럼 사용해서 접근, 수정, 확인, 삭제할 수 있습니다. 코드에 고정된 이름으로 쓰지 않고 말이죠. 덕분에 프로그램이 다양한 데이터나 사용자 입력에 즉각 대응할 수 있는 유연성을 갖게 됩니다.

Python은 객체 속성을 동적으로 다루기 위한 네 가지 내장 함수를 제공합니다. `getattr()`, `setattr()`, `hasattr()`, `delattr()`가 그것입니다.

이 함수들은 변수 이름을 사용해 속성에 접근하고, 생성하고, 확인하고, 제거할 수 있게 해줍니다. 각각 어떻게 쓰이는지 살펴보겠습니다.

`getattr()`는 실행 시점까지 속성 이름을 모를 때 객체에서 속성을 읽을 수 있게 해줍니다. 만약 속성이 없으면 기본값을 제공하지 않는 한 `AttributeError`를 발생시킵니다.

사용법은 객체, 속성 이름, 그리고 선택적 기본값을 전달하는 것입니다:

```python
getattr(object, attribute_name, default_value) 
```

예를 들어:

```python
class Person: 
    def __init__(self, name, age): 
        self.name = name 
        self.age = age 

person = Person('John Doe', 30) 
 
print(getattr(person, 'name')) # John Doe 
print(getattr(person, 'age')) # 30 
print(getattr(person, 'city', 'Milano')) # Milano
```

위 예제에서 `Milano`는 기본값입니다. `city`는 `Person` 클래스에 존재하지 않기 때문입니다.

앞서 말했듯 `getattr()`의 진가는 속성 이름이 사용자 입력이나 파일에서 온 변수일 때 드러납니다.

이 경우 속성 이름이 고정되어 있지 않으므로 일반 `object.attribute_name` 구문을 사용할 수 없습니다.

```python
class Person: 
    def __init__(self, name, age): 
        self.name = name 
        self.age = age 

person = Person('John Doe', 30)

attr_name = input('Enter the attribute you want to see: ')
print(getattr(person, attr_name, 'Attribute not found'))
```

이 경우 사용자가 `name`를 입력하면 `John Doe`가 출력되고, `age`를 입력하면 `30`가 출력됩니다. 그리고 `email`처럼 클래스에 없는 것을 입력하면 `Attribute not found`가 출력됩니다.

바로 이 점이 동적 속성 다루기의 장점입니다. 코드가 이전에 본 적 없는 입력과 데이터에 대응할 수 있게 해줍니다.

또한, 이미 알고 있는 속성뿐 아니라 객체가 가진 모든 속성을 살펴보고 싶을 때도 있습니다. 내장 `dir()` 함수가 그 역할을 합니다. 이 함수는 객체의 모든 속성 이름을 리스트로 반환합니다. 사용법은 다음과 같습니다:

```python
class Person: 
    def __init__(self, name, age): 
        self.name = name 
        self.age = age 

person = Person('John Doe', 30)

# Loop through all attributes of the person object with dir() function
for attr in dir(person):
    # Ignore dunder methods like __init__ or __str__ and regular methods
    if not attr.startswith('__') and not callable(getattr(person, attr)): 
        value = getattr(person, attr)
        print(f'{attr}: {value}')

# Output
# age: 30
# name: John Doe
```

`setattr()` 함수는 동적으로 새 속성을 만들거나 기존 속성을 업데이트할 수 있게 해줍니다. 구문은 다음과 같습니다:

```python
setattr(object, attribute_name, value) 
```

예를 들어 설정 파일이나 환경 변수 파일에서 받은 데이터를 바탕으로 설정 속성을 지정하는 예는 다음과 같습니다:

```python
class Configuration:
    pass

# Data loaded at runtime (like from a config or env file)
settings_data = {
    'server_url': 'https://api.example.com',
    'timeout_sec': 30,
    'max_retries': 5
}

config_obj = Configuration()

# Dynamically set attributes using dictionary keys and values
for attr_name, attr_value in settings_data.items():
    setattr(config_obj, attr_name, attr_value)

print(config_obj.server_url) # https://api.example.com
print(config_obj.timeout_sec) # 30
```

또한 `hasattr()`가 있습니다. 속성으로 무언가를 하기 전에 또는 삭제하기 전에 속성이 존재하는지 확인하는 것이 좋은 습관입니다. `hasattr()`가 바로 그 역할을 합니다. 속성 존재 여부를 확인해 결과에 따라 `True` 또는 `False`를 반환합니다.

기본 구문은 다음과 같습니다:

```python
hasattr(object, attribute_name)  
```

다음은 `Product` 클래스 인스턴스에서 속성 존재 여부를 동적으로 확인하는 예입니다:

```python
class Product:
    def __init__(self, name, price):
        self.name = name
        self.price = price

product_a = Product('T-Shirt', 25)

required_attributes = ['name', 'price', 'inventory_id']

for attr in required_attributes:
    if not hasattr(product_a, attr):
        print(f"ERROR: Product is missing the required attribute: '{attr}'")
    else:
        # Access the attributes dynamically once their existence is confirmed
        print(f'{attr}: {getattr(product_a, attr)}')

# Output:
# name: T-Shirt
# price: 25
# ERROR: Product is missing the required attribute: 'inventory_id'
```

ERROR 출력은 `inventory_id`가 `Product` 클래스와 인스턴스에 없기 때문에 발생했습니다.

마지막으로 `delattr()`는 속성을 동적으로 삭제할 수 있게 해줍니다:

```python
delattr(object, attribute_name) 
```

예를 들어, 객체가 완전히 처리된 후 민감하거나 임시 속성을 정리하고 싶을 때가 있습니다. 그런 다음 `dir()`를 사용해 남은 속성을 반복할 수 있습니다:

```python
class UserSession:
    def __init__(self, user_id, token):
        self.user_id = user_id
        self.auth_token = token # sensitive
        self.temp_counter = 0 # temporary

session = UserSession(101, 'a1b2c3d4e5')

# List of attributes to remove dynamically before "saving" the session
attributes_to_clean = ['auth_token', 'temp_counter']

# Dynamically remove specified attributes
for attr in attributes_to_clean:
    if hasattr(session, attr):
        delattr(session, attr)
        print(f'Removed attribute: {attr}')

print('\nFinal attributes remaining:')

# Loop through the remaining attributes with dir()
for attr in dir(session):
    # Ignore dunder methods like __init__ or __str__ and regular methods
    if not attr.startswith('__') and not callable(getattr(session, attr)):
        print(f' - {attr}: {getattr(session, attr)}')

# Output:
# Removed attribute: auth_token
# Removed attribute: temp_counter

# Final attributes remaining:
#  - user_id: 101
```

이렇게 해서 속성을 동적으로 다루는 방법을 배웠습니다!
