# --description--

Керування пам’яттю — це важлива концепція в програмуванні, але вона може бути трохи заплутаною для початківців, тому давайте розберемо її простими словами.

Коли ви запускаєте програму, включно з кодом JavaScript у веббраузері, їй потрібна пам’ять для збереження всієї інформації, з якою вона працює. Це включає змінні, функції, об’єкти — фактично все, що створює і використовує ваш код. Керування пам’яттю — це процес контролю цієї пам’яті: виділення її, коли потрібно, і звільнення, коли вона більше не потрібна. У деяких мовах програмування розробники мають керувати пам’яттю вручну. Вони повинні явно вказувати комп’ютеру, коли виділити пам’ять для нових об’єктів і коли звільнити пам’ять, яка більше не потрібна. Це може бути потужним, але й складним, бо забуття звільнити пам’ять може призвести до витоків пам’яті.

JavaScript, однак, використовує автоматичне керування пам’яттю. Це означає, що JavaScript (точніше, JavaScript-двигун у вашому веббраузері) самостійно піклується про виділення і звільнення пам’яті. Вам не потрібно явно звільняти пам’ять у коді. Цей автоматичний процес часто називають збором сміття.

Ось як це працює простими словами. Спочатку виділення пам’яті відбувається, коли ви створюєте змінні, об’єкти або функції у вашому JavaScript-коді — пам’ять автоматично виділяється для їх збереження. Потім ви використовуєте цю виділену пам’ять, працюючи з цими змінними, об’єктами чи функціями у коді.

JavaScript-двигун має розумні способи визначити, коли щось у пам’яті більше не потрібне. Загалом, якщо ваша програма більше не має способу отримати доступ або використати певні дані, вони вважаються такими, що більше не потрібні. Періодично збирач сміття запускається, знаходить пам’ять, яка більше не потрібна, і звільняє її, роблячи доступною для подальшого використання. Цей процес відбувається автоматично, що чудово, бо вам не потрібно турбуватися про керування пам’яттю самостійно.

Проте важливо розуміти цю концепцію, бо іноді ви можете випадково зберігати референси на речі, які більше не потрібні, і це заважає збирачу сміття звільнити цю пам’ять. Наприклад:

```js
function createLargeArray() {
  let largeArray = new Array(1000000);
  return function() {
    console.log(largeArray.length);
  };
}

let printArrayLength = createLargeArray();
printArrayLength();
```

У цьому коді, навіть після того, як `createLargeArray` завершить виконання, `largeArray` не може бути зібраний сміттям, бо повернена функція все ще має доступ до нього. Це замикання, і хоча замикання корисні, вони іноді можуть призводити до більшого використання пам’яті, ніж ви очікуєте. Ви дізнаєтеся більше про замикання у майбутніх уроках.

Як початківець, вам не потрібно надто перейматися деталями керування пам’яттю. Автоматичний збір сміття в JavaScript піклується про більшість речей за вас. Проте, коли ви просунетеся у вивченні JavaScript, розуміння цих концепцій допоможе писати більш ефективний код, особливо для великих застосунків або при роботі з обмеженими ресурсами.

Пам’ятайте, що хороші практики кодування, як-от уникання глобальних змінних, коли це можливо, і уважність до того, що саме ваші функції замикають, допоможуть JavaScript-двигуну ефективніше керувати пам’яттю.
