# --description--

JavaScript로 프로젝트를 만들 때, 작업을 돕는 추가 도구가 자주 필요합니다. 간단한 애플리케이션조차도 데이터 저장, 로그인, 페이지 간 이동을 위한 라우팅 같은 기능이 필요할 수 있습니다.

이 모든 코드를 처음부터 직접 작성하는 대신, 다른 개발자들이 이미 만들어 공유한 오픈소스 패키지를 사용할 수 있습니다.

바로 이때 npm이 등장합니다. npm은 방대한 온라인 오픈소스 소프트웨어 저장소입니다. 코드를 위한 앱스토어라고 생각하세요 — 필요한 것을 검색하고 설치한 뒤 프로젝트에서 바로 사용할 수 있습니다.

많은 사람이 npm이 "Node Package Manager"의 약자라고 생각하지만, 공식적인 의미는 아닙니다. 예전 npm 문서에서는 "npm은 약어가 아니다"라는 재귀적 약어로 설명했습니다. 즉, npm은 공식적으로 어떤 약어도 아닙니다.

오늘날 npm은 세 가지 관련 부분을 가리킵니다:

* 패키지를 검색할 수 있는 웹사이트 npmjs.com
    
* 모든 패키지를 저장하는 대규모 공개 데이터베이스인 npm 저장소
    
* Node.js와 함께 설치되어 패키지를 설치하고 관리할 수 있는 npm 명령줄 도구
    

예를 들어, 프로젝트에서 상태 관리, 날짜 처리, 이메일 전송, HTTP 요청을 해야 한다면, 이미 그런 기능을 하는 npm 패키지가 있을 가능성이 큽니다. 간단한 `npm install` 명령으로 직접 작업하는 대신 이 준비된 소프트웨어를 사용할 수 있습니다.

npm은 프로젝트 의존성 관리도 도와줍니다. 즉, 프로젝트에서 사용하는 패키지와 각 패키지의 버전을 추적합니다. 이 작업은 앞으로 수업에서 배우게 될 `package.json`과 `package-lock.json` 파일로 이루어집니다.

Yarn, PNPM, Bun은 시간이 지나면서 인기를 얻은 npm의 대안입니다. 각각 설치 속도나 저장 효율 같은 특정 영역에서 장점을 제공하지만, npm이 여전히 가장 널리 사용됩니다.

앞으로 수업에서 다양한 npm 명령어, 패키지 설치 방법, `package.json`과 `package-lock.json` 파일, 그리고 자신만의 패키지를 npm 저장소에 게시하는 방법까지 배우게 됩니다.
