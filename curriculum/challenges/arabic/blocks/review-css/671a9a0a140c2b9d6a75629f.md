# --interactive--

راجع المفاهيم أدناه للتحضير للاختبار القادم.

## أساسيات CSS

- **ما هو CSS؟**: أوراق الأنماط المتتالية (CSS) هي لغة أوراق أنماط تُستخدم لتطبيق الأنماط على عناصر HTML. تُستخدم CSS للألوان، صور الخلفية، التخطيطات، والمزيد.
- **التركيب الأساسي لقاعدة CSS**: تتكون قاعدة CSS من جزأين رئيسيين: محدد وكتلة إعلان. المحدد هو نمط يُستخدم في CSS لتحديد واستهداف عناصر HTML معينة لتطبيق الأنماط عليها. كتلة الإعلان تطبق مجموعة من الأنماط لمحدد معين أو لمحددات.
- **عنصر `meta name="viewport"`**: هذا العنصر `meta` يعطي المتصفح تعليمات حول كيفية التحكم في أبعاد الصفحة وتكبيرها على أجهزة مختلفة، خصوصًا الهواتف المحمولة والأجهزة اللوحية.
- **أنماط المتصفح الافتراضية**: كل عنصر HTML له أنماط متصفح افتراضية تُطبق عليه. عادةً تشمل هذه العناصر مثل الهوامش والحشوات الافتراضية.

## CSS المضمّن، الداخلي، والخارجي

- **CSS المضمّن**: تُكتب هذه الأنماط مباشرة داخل عنصر HTML باستخدام الخاصية `style`. في معظم الأحيان لن تستخدم CSS المضمّن بسبب مبدأ فصل الاهتمامات.
- **CSS الداخلي**: تُكتب هذه الأنماط داخل وسم `<style>` في قسم `head` من وثيقة HTML. يمكن أن يكون هذا مفيدًا لإنشاء أمثلة شفرة قصيرة، لكن عادةً لن تستخدم CSS الداخلي.
- **CSS الخارجي**: تُكتب هذه الأنماط في ملف CSS منفصل ويرتبط بملف HTML باستخدام عنصر `link` في قسم `head`. في معظم المشاريع، ستستخدم ملف CSS خارجي بدلًا من الداخلي أو المضمّن.

## العمل مع خاصيتي `width` و `height`

- **خاصية `width`**: تحدد هذه الخاصية عرض العنصر. إذا لم تحدد عرضًا، فالقيمة الافتراضية تكون `auto`. هذا يسمح للمتصفح بتحديد عرض العنصر بناءً على محتواه، العنصر الأب، ونوع العرض.
- **خاصية `min-width`**: تحدد الحد الأدنى لعرض العنصر.
- **خاصية `max-width`**: تحدد الحد الأقصى لعرض العنصر.
- **خاصية `height`**: تحدد ارتفاع العنصر. بالمثل، الارتفاع افتراضيًا `auto`، مما يعني أنه يتكيف مع المحتوى الداخلي.
- **خاصية `min-height`**: تحدد الحد الأدنى لارتفاع العنصر.
- **خاصية `max-height`**: تحدد الحد الأقصى لارتفاع العنصر.

## أنواع مختلفة من مركبات CSS

- **مركب العنصر التابع (` `)\*مسافة واحدة بين المحددات**: يُستخدم هذا المركب لاستهداف العناصر التابعة لعنصر أب محدد.
- **مركب العنصر الابن (`>`)**: يُستخدم هذا المركب لاختيار العناصر التي هي أبناء مباشرة لعنصر أب محدد.
- **مركب الأخ التالي (`+`)**: يختار هذا المركب العنصر الذي يلي مباشرة عنصر أخ محدد.

- **مركب الأخ اللاحق (`~`)**: يختار هذا المركب جميع الإخوة لعنصر محدد الذين يأتون بعده.

## عناصر المستوى المضمّن، الكتلي، والمضمّن-كتلي

- **عناصر المستوى المضمّن**: تأخذ العناصر المضمّنة فقط العرض الذي تحتاجه ولا تبدأ في سطر جديد. تتدفق هذه العناصر ضمن المحتوى، مما يسمح للنص والعناصر المضمّنة الأخرى بالظهور بجانبها. من العناصر المضمّنة الشائعة `span`، `a`، و`img`.
- **عناصر المستوى الكتلي**: تأخذ عناصر المستوى الكتلي العرض الكامل المتاح لها افتراضيًا، وتمتد عبر عرض الحاوية الخاصة بها. من العناصر الكتلية الشائعة `div`، `p`، و`section`.
- **عناصر المستوى المضمّن-كتلي**: يمكنك تعيين عنصر إلى `inline-block` باستخدام خاصية `display`. تتصرف هذه العناصر كعناصر مضمّنة لكنها يمكن أن تحدد لها `width` و`height` مثل العناصر الكتلية.

## الهامش والحشو

- **خاصية `margin`**: تُستخدم هذه الخاصية لتطبيق مساحة خارج العنصر، بين حدود العنصر والعناصر المحيطة.
- **اختصار `margin`**: يمكنك تحديد من 1 إلى 4 قيم لتعيين جوانب الهامش. قيمة واحدة تطبق على جميع الجوانب الأربعة؛ قيمتان تعينان `top` و`bottom` ثم `right` و`left`؛ ثلاث قيم تعين `top`، الأفقي (`right` و`left`)، ثم `bottom`؛ أربع قيم تعين `top`، `right`، `bottom`، `left`.
- **خاصية `padding`**: تُستخدم هذه الخاصية لتطبيق مساحة داخل العنصر، بين المحتوى وحدوده.
- **اختصار `padding`**: يمكنك تحديد من 1 إلى 4 قيم لتعيين جوانب الحشو. قيمة واحدة تطبق على جميع الجوانب الأربعة؛ قيمتان تعينان `top` و`bottom` ثم `right` و`left`؛ ثلاث قيم تعين `top`، الأفقي (`right` و`left`)، ثم `bottom`؛ أربع قيم تعين `top`، `right`، `bottom`، `left`.

## خصوصية CSS

- **خصوصية CSS المضمّن**: CSS المضمّن لها أعلى خصوصية لأنها تُطبق مباشرة على العنصر. تتجاوز أي CSS داخلية أو خارجية. قيمة الخصوصية لأنماط المضمّن هي (1, 0, 0, 0).
- **خصوصية CSS الداخلية**: تُعرف CSS الداخلية داخل عنصر `style` في قسم `head` من وثيقة HTML. لها خصوصية أقل من أنماط المضمّن. تشترك كل من CSS الداخلية والخارجية في نفس مستوى الخصوصية، الذي يحدده المحددون وليس الموقع.
- **خصوصية CSS الخارجية**: ترتبط CSS الخارجية عبر عنصر `link` في قسم `head` وتُكتب في ملفات `.css` منفصلة. مثل CSS الداخلية، تحدد الخصوصية بواسطة المحددات المستخدمة. عندما يكون لقانونين نفس الخصوصية، يحدد ترتيب المصدر الفائز: القاعدة التي تظهر لاحقًا في الوثيقة لها الأولوية. توفر CSS الخارجية أفضل قابلية للصيانة للمشاريع الكبيرة.
- **المحدد العام (`*`)**: هذا نوع خاص من محددات CSS يطابق أي عنصر في الوثيقة. غالبًا ما يُستخدم لتطبيق نمط على جميع العناصر في الصفحة، وهو مفيد لإعادة تعيين أو توحيد الأنماط عبر المتصفحات المختلفة. للمحدد العام أقل قيمة خصوصية بين جميع المحددات. يساهم بصفر في جميع أجزاء قيمة الخصوصية (0, 0, 0, 0).
- **محددات النوع**: تستهدف هذه المحددات العناصر بناءً على اسم الوسم الخاص بها. لمحددات النوع خصوصية منخفضة نسبيًا مقارنة بالمحددات الأخرى. قيمة الخصوصية لمحدد النوع هي (0, 0, 0, 1).
- **محددات الصنف**: تُعرف هذه المحددات بنقطة (`.`) تليها اسم الصنف. قيمة الخصوصية لمحدد الصنف هي (0, 0, 1, 0). هذا يعني أن محددات الصنف يمكنها تجاوز محددات النوع، لكنها يمكن أن تُتجاوز بواسطة محددات المعرف وأنماط المضمّن.
- **محددات المعرف**: تُعرف محددات المعرف برمز الهاش (`#`) يليه اسم المعرف. لها خصوصية عالية جدًا، أعلى من محددات النوع والصنف، لكنها أقل من أنماط المضمّن. قيمة الخصوصية لمحدد المعرف هي (0, 1, 0, 0).
- **الكلمة المفتاحية `!important`**: تُستخدم لإعطاء قاعدة نمط أولوية قصوى، مما يسمح لها بتجاوز أي إعلانات أخرى لخاصية معينة. عند استخدامها، تجبر المتصفح على تطبيق النمط المحدد بغض النظر عن خصوصية المحددات الأخرى. يجب الحذر عند استخدام `!important` لأنها قد تجعل CSS أصعب في الصيانة وتصحيح الأخطاء.
- **خوارزمية التتابع (Cascade Algorithm)**: خوارزمية تُستخدم لتحديد أي قواعد CSS تُطبق عندما تستهدف أنماط متعددة نفس العنصر. تضمن استخدام الأنماط الأنسب بناءً على مجموعة من القواعد المحددة جيدًا.
- **وراثة CSS**: العملية التي تنتقل بها الأنماط من العناصر الأبوية إلى أبنائها. تسمح الوراثة بتعريف أنماط على مستوى أعلى في شجرة الوثيقة وتطبيقها على عدة عناصر دون تحديدها صراحة لكل عنصر.

## مصطلحات التصميم

- **التخطيط**: هو كيفية ترتيب العناصر البصرية على صفحة أو شاشة لنقل رسالة. قد تشمل هذه العناصر النصوص، الصور، والمساحات البيضاء.
- **المحاذاة**: هي كيفية وضع العناصر بالنسبة لبعضها البعض. استخدام المحاذاة بشكل صحيح يساعد على جعل التصميم نظيفًا ومنظمًا.
- **التكوين**: هو فعل ترتيب العناصر لإنشاء تصميم متناغم. يحدد كيف ترتبط العناصر مثل الصور، النصوص، والأشكال ببعضها وتساهم في التصميم بطريقة فنية.
- **التوازن**: هو كيفية توزيع الوزن البصري داخل التكوين. يهدف المصممون إلى خلق توازن من خلال الترتيبات المتماثلة أو غير المتماثلة.
- **المقياس**: يشير إلى مقارنة أبعاد أو حجم عنصر ما مع عنصر آخر.
- **التسلسل الهرمي**: يحدد ترتيب أهمية العناصر في التصميم. يتعلق بضمان ملاحظة المعلومات الأكثر أهمية أولًا.
- **التباين**: هو عملية خلق تمييز واضح بين العناصر. يمكن تحقيق ذلك من خلال اختلافات في اللون، الحجم، الشكل، الملمس، أو أي خاصية بصرية أخرى. التباين القوي يساعد أيضًا على تحسين قابلية القراءة.
- **المساحة البيضاء (المساحة السلبية)**: هي المساحة الفارغة في التصميم. هي المنطقة المحيطة بالعناصر.
- **واجهة المستخدم (UI)**: تشمل العناصر البصرية والتفاعلية التي يمكن للمستخدمين رؤيتها على شاشاتهم، مثل الأيقونات، الصور، النصوص، القوائم، الروابط، والأزرار.
- **تجربة المستخدم (UX)**: تتعلق بكيف يشعر المستخدمون عند استخدام منتج أو خدمة. التطبيق ذو تجربة مستخدم مصممة جيدًا يكون بديهيًا، سهل الاستخدام، فعالًا، متاحًا، وممتعًا.
- **موجز التصميم**: هو وثيقة تحدد أهداف ومتطلبات المشروع. هو خارطة طريق توجه عملية التصميم وتضمن أن المنتج النهائي يلبي احتياجات العميل.
- **التصميم القائم على المتجهات**: يتضمن إنشاء فن رقمي باستخدام صيغ رياضية لتعريف الخطوط، الأشكال، والألوان.
- **النمذجة الأولية**: تشير إلى عملية إنشاء نموذج تفاعلي لمنتج أو واجهة مستخدم.

## أساسيات تصميم واجهة المستخدم

- **تباين جيد بين ألوان الخلفية والمقدمة**: من المهم التأكد من أن ألوان الخلفية والمقدمة لها تباين جيد لجعل النص قابلًا للقراءة. توصي إرشادات إمكانية الوصول لمحتوى الويب (WCAG) بنسبة تباين لا تقل عن 4.5:1 للنص العادي و3:1 للنص الكبير.
- **تسلسل هرمي بصري جيد في التصميم**: يمكن للتسلسل الهرمي البصري القوي أن يوفر مسارًا واضحًا للعين لتتبعه، مما يضمن استهلاك المعلومات التي تنقلها بالترتيب الذي تريده.
- **الصور المتجاوبة**: الصور المتجاوبة هي الصور التي تتغير أبعادها لتناسب حجم الشاشة التي تُعرض عليها. هذا مهم لأنه يضمن أن تبدو صورك جيدة على جميع الأجهزة، من أجهزة الكمبيوتر المكتبية إلى الهواتف المحمولة.
- **التحسين التدريجي**: هو نهج تصميم يضمن أن جميع المستخدمين، بغض النظر عن المتصفح أو الجهاز، يمكنهم الوصول إلى المحتوى والوظائف الأساسية للتطبيق.
- **تصميم يركز على المستخدم**: هو نهج يعطي الأولوية للمستخدم النهائي، من احتياجاته إلى تفضيلاته وقيوده. هدف التصميم الذي يركز على المستخدم هو إنشاء صفحة ويب بديهية، سهلة الاستخدام، وممتعة لتفاعل المستخدمين معها.
- **بحث المستخدم**: هو الدراسة المنهجية للأشخاص الذين يستخدمون منتجك. الهدف هو قياس احتياجات المستخدمين، سلوكياتهم، ونقاط الألم لديهم.
- **اختبار المستخدم**: يشير إلى ممارسة جمع البيانات من المستخدمين أثناء تفاعلهم مع تطبيقك.
- **اختبار A/B**: هو عملية إطلاق ميزة جديدة لمجموعة فرعية عشوائية من قاعدة المستخدمين. يمكنك بعد ذلك الاستفادة من بيانات التحليلات لتحديد ما إذا كانت الميزة مفيدة.
- **متطلبات المستخدم**: تشير إلى القصص أو المعايير التي يجب أن يتبعها تطبيقك. قد تُحدد متطلبات المستخدم بواسطة بحث المستخدم، أو معايير الصناعة. يمكن أيضًا تحديدها بواسطة مدخلات أصحاب المصلحة.
- **الإفصاح التدريجي**: هو نمط تصميم يُستخدم لعرض المحتوى ذي الصلة فقط للمستخدمين بناءً على نشاطهم الحالي وإخفاء الباقي. يتم ذلك لتقليل الحمل المعرفي وجعل تجربة المستخدم أكثر بديهية.
- **التسجيل المؤجل/الكسول**: هو نمط تصميم واجهة مستخدم يسمح للمستخدمين بتصفح والتفاعل مع تطبيقك دون الحاجة إلى التسجيل.

## أفضل ممارسات التصميم

- **الوضع الداكن**: هي ميزة خاصة في تطبيقات الويب حيث يمكنك تغيير نظام الألوان الافتراضي الفاتح إلى نظام ألوان داكن. يجب استخدام ألوان منخفضة التشبع في الوضع الداكن. الألوان منخفضة التشبع هي ألوان أقل كثافة ولها مستوى تشبع أقل.
- **شريط التنقل (Breadcrumbs)**: هو مساعد تنقل يُظهر للمستخدم مكانه في هيكل الموقع. من الأفضل وضع شريط التنقل في أعلى الصفحة ليتمكن المستخدمون من العثور عليه بسهولة. كما يجب التأكد من أن شريط التنقل كبير بما يكفي ليُقرأ بسهولة، لكنه ليس كبيرًا جدًا ليأخذ مساحة كبيرة في الصفحة.
- **مكوِّن البطاقة**: يجب أن يكون مكوِّن البطاقة بسيطًا في التصميم، غير مزدحم بصريًا أو يعرض معلومات كثيرة. بالنسبة للوسائط، تأكد من اختيار صور وفيديوهات عالية الجودة لتعزيز تجربة المستخدم.
- **التمرير اللانهائي**: هو نمط تصميم يقوم بتحميل المزيد من المحتوى أثناء تمرير المستخدم لأسفل الصفحة. يجب التفكير في استخدام زر "تحميل المزيد" لأنه يمنح المستخدم التحكم في متى يريد رؤية المزيد من المحتوى. يمكنك أيضًا إضافة زر "عودة" ليتمكن المستخدمون من العودة إلى الصفحة السابقة دون الحاجة إلى التمرير إلى الأعلى.
- **مربع حوار منبثق (Modal Dialog)**: هو نوع من النوافذ المنبثقة التي تظهر فوق محتوى الصفحة الحالي. عادةً ما يكون لمحتوى الخلفية تراكب لوني خافت لمساعدة المستخدم على التركيز على محتوى المربع المنبثق. من الجيد دائمًا السماح للمستخدم بالنقر خارج المربع لإغلاقه. عند استخدام عنصر HTML `dialog`، ستحصل على الكثير من الوظائف وفوائد إمكانية الوصول المدمجة.
- **مؤشر التقدم لتسجيل النموذج**: هو طريقة لإظهار مدى تقدم المستخدم في عملية ما. يمكن استخدامه في النماذج، التسجيل، وعمليات الإعداد. يجب أن يكون تصميمك بسيطًا، سهل العثور عليه، ويسمح بالعودة إلى الخطوات السابقة.
- **عربة التسوق**: العربة هي مكان للمستخدمين لرؤية العناصر التي اختاروها بالفعل على منصة التجارة الإلكترونية. يجب أن تكون عربات التسوق مرئية دائمًا للمستخدم، تستخدم أيقونة شائعة مثل عربة، حقيبة، أو سلة، وتحتوي على زر دعوة لاتخاذ إجراء واضح للمستخدمين للمتابعة إلى الدفع.

## أدوات التصميم الشائعة

- **Figma**: هذه الأداة السحابية متخصصة في تصميم واجهة المستخدم وتجربة المستخدم (UI / UX). تتيح لفرق التصميم والتطوير التعاون من أي مكان، وتقدم ميزات مدمجة تشمل التصميم القائم على المتجهات، التخطيط التلقائي، نظام التعليقات والتغذية الراجعة، والمزيد.
- **Sketch**: هي أداة تصميم شهيرة تُستخدم لواجهتها البديهية وبساطتها، مما يجعلها مثالية للمطورين الذين يرغبون في إنشاء نماذج أولية بسرعة. كما يستخدمها المصممون على نطاق واسع لمهام مثل إنشاء واجهات المستخدم، الأيقونات، وتخطيطات الويب.
- **Adobe XD**: هي أداة تصميم ونمذجة أولية قائمة على المتجهات لتصميم UI/UX، معروفة بتكاملها السلس مع تطبيقات Adobe الأخرى مثل Photoshop وIllustrator وAfter Effects.
- **Canva**: تتيح هذه الأداة إنشاء مجموعة واسعة من المحتوى البصري، بما في ذلك الملصقات، صور الغلاف، العروض التقديمية، الفيديوهات القصيرة، والمزيد. تصميمها سهل الاستخدام وبسيط، مما يجعلها مثالية للمبتدئين.

## الوحدات المطلقة

- **`px` (بكسل)**: هذه الوحدة المطلقة هي وحدة قياس ثابتة في CSS، توفر تحكمًا دقيقًا في الأبعاد. هذا يعني أن 1px دائمًا يساوي 1/96 من البوصة.
- **`in` (بوصة)**: هذه الوحدة المطلقة تساوي 96px.
- **`cm` (سنتيمتر)**: هذه الوحدة المطلقة تساوي 25.2/64 من البوصة.
- **`mm` (ملليمتر)**: هذه الوحدة المطلقة تساوي 1/10 من السنتيمتر.
- **`q` (ربع ملليمتر)**: هذه الوحدة المطلقة تساوي 1/40 من السنتيمتر.
- **`pc` (بيكا)**: هذه الوحدة المطلقة تساوي 1/6 من البوصة.
- **`pt` (نقطة)**: هذه الوحدة المطلقة تساوي 1/72 من البوصة.

## الوحدات النسبية

- **النسب المئوية**: تتيح لك هذه الوحدات النسبية تعريف الأحجام والأبعاد وخصائص أخرى كنسبة من عنصرها الأصل. على سبيل المثال، إذا ضبطت `width: 50%;` على عنصر، فسيشغل نصف عرض حاويته الأصل.
- **الوحدة `em`**: هذه الوحدات نسبية إلى حجم خط العنصر. إذا كنت تستخدم `ems` للخاصية `font-size`، فسيكون حجم النص نسبيًا إلى حجم خط العنصر الأصل.
- **الوحدة `rem`**: هذه الوحدات نسبية إلى حجم خط العنصر الجذر، وهو عنصر `html`.
- **الوحدة `vh`**: ترمز `vh` إلى `"viewport height"` (ارتفاع إطار العرض) وتساوي `1vh` نسبة 1% من ارتفاع إطار العرض.
- **الوحدة `vw`**: ترمز `vw` إلى `"viewport width"` (عرض إطار العرض) وتساوي `1vw` نسبة 1% من عرض إطار العرض.

## دالة `calc`

- **دالة `calc()`**: باستخدام دالة `calc()`، يمكنك إجراء حسابات مباشرة داخل أوراق الأنماط لتحديد قيم الخصائص بشكل ديناميكي. هذا يعني أنه يمكنك إنشاء واجهات مستخدم مرنة ومتجاوبة بحساب الأبعاد بناءً على حجم إطار العرض أو عناصر أخرى.

## الفئات المزيفة الخاصة بتفاعل المستخدم

- **تعريف الفئات المزيفة**: هي كلمات مفتاحية خاصة في CSS تسمح لك باختيار عنصر بناءً على حالته أو موقعه المحدد.
- **فئات مزيفة لتفاعل المستخدم**: هي كلمات مفتاحية خاصة تسمح لك بتغيير مظهر العناصر بناءً على تفاعلات المستخدم، مما يحسن تجربة المستخدم العامة.
- **الفئة المزيفة `:active`**: تتيح لك اختيار الحالة النشطة لعنصر، مثل النقر على زر.
- **الفئة المزيفة `:hover`**: تعرف حالة المرور فوق عنصر.
- **الفئة المزيفة `:focus`**: تطبق أنماطًا عندما يحصل العنصر على التركيز، عادةً عبر التنقل باستخدام لوحة المفاتيح أو عند النقر داخل حقل نموذج.
- **الفئة المزيفة `:focus-within`**: تُستخدم لتطبيق أنماط على عنصر عندما يكون هو أو أي من عناصره التابعة في حالة تركيز.

## فئات مزيفة لعناصر الإدخال

- **فئات مزيفة لعناصر الإدخال**: تُستخدم هذه الفئات المزيفة لاستهداف عناصر HTML `input` بناءً على الحالة التي تكون عليها قبل وبعد تفاعل المستخدم.
- **الفئة المزيفة `:enabled`**: تُستخدم لاستهداف أزرار النماذج أو العناصر الأخرى التي تكون مفعلة حاليًا.
- **الفئة المزيفة `:disabled`**: تتيح لك تنسيق عنصر تفاعلي في وضع معطل.
- **الفئة المزيفة `:checked`**: تُستخدم للإشارة إلى أن العنصر محدد.
- **الفئة المزيفة `:valid`**: تستهدف حقول الإدخال التي تستوفي معايير التحقق.
- **الفئة المزيفة `:invalid`**: تستهدف حقول الإدخال التي لا تستوفي معايير التحقق.
- **الفئتان المزيفتان `:in-range` و`:out-of-range`**: تطبق أنماطًا على العناصر بناءً على ما إذا كانت قيمها ضمن أو خارج نطاق محدد.
- **الفئة المزيفة `:required`**: تستهدف عناصر `input` التي تحتوي على الخاصية `required`. تشير إلى أنه يجب على المستخدم ملء الحقل لإرسال النموذج.
- **الفئة المزيفة `:optional`**: تطبق أنماطًا على عناصر الإدخال التي ليست مطلوبة ويمكن تركها فارغة.
- **الفئة المزيفة `:autofill`**: تطبق أنماطًا على حقول الإدخال التي يملؤها المتصفح تلقائيًا بالبيانات المحفوظة.

## فئات مزيفة للموقع

- **فئات مزيفة للموقع**: تُستخدم هذه الفئات المزيفة لتنسيق الروابط والعناصر المستهدفة داخل الوثيقة الحالية.
- **الفئة المزيفة `:any-link`**: هي مزيج من الفئتين المزيفتين `:link` و`:visited`. تطابق أي عنصر وسم الرابط الذي يحتوي على خاصية `href`، بغض النظر عما إذا كان قد تم زيارته أم لا.
- **الفئة المزيفة `:link`**: تسمح لك باستهداف جميع الروابط غير المزارة على صفحة الويب. يمكنك استخدامها لتنسيق الروابط بشكل مختلف قبل نقر المستخدم عليها.
- **الفئة المزيفة `:local-link`**: تستهدف الروابط التي تشير إلى نفس الوثيقة. يمكن أن تكون مفيدة عندما تريد التمييز بين الروابط الداخلية والخارجية.
- **الفئة المزيفة `:visited`**: تستهدف رابطًا زاره المستخدم.
- **الفئة المزيفة `:target`**: تُستخدم لتطبيق أنماط على عنصر هو هدف جزء من عنوان URL.

## فئات مزيفة هيكلية شجرية

- **فئات مزيفة هيكلية شجرية**: تتيح لك هذه الفئات المزيفة استهداف العناصر وتنسيقها بناءً على موقعها داخل شجرة المستند.
- **فئة مزيفة `:root`**: عادةً ما تكون هذه الفئة المزيفة العنصر الجذري `html`. تساعدك على استهداف أعلى مستوى في المستند لتطبيق نمط مشترك على كامل المستند.
- **فئة مزيفة `:empty`**: تشمل العناصر الفارغة، أي العناصر التي لا تحتوي على أبناء سوى الفراغات البيضاء، وهي مدرجة أيضًا في شجرة المستند. لهذا السبب توجد فئة مزيفة `:empty` لاستهداف العناصر الفارغة.
- **فئة مزيفة `:nth-child(n)`**: تتيح لك هذه الفئة المزيفة اختيار العناصر بناءً على موقعها داخل العنصر الأب.
- **فئة مزيفة `:nth-last-child(n)`**: تمكنك هذه الفئة المزيفة من اختيار العناصر بالعد من النهاية.
- **فئة مزيفة `:first-child`**: تختار هذه الفئة المزيفة أول عنصر داخل عنصر أب أو داخل المستند.
- **فئة مزيفة `:last-child`**: تختار هذه الفئة المزيفة آخر عنصر داخل عنصر أب أو داخل المستند.
- **فئة مزيفة `:only-child`**: تختار هذه الفئة المزيفة العنصر الوحيد داخل عنصر أب أو داخل المستند.
- **فئة مزيفة `:first-of-type`**: تختار هذه الفئة المزيفة أول ظهور لنوع عنصر معين داخل عنصر الأب.
- **فئة مزيفة `:last-of-type`**: تختار هذه الفئة المزيفة آخر ظهور لنوع عنصر معين داخل عنصر الأب.
- **فئة مزيفة `:nth-of-type(n)`**: تتيح لك هذه الفئة المزيفة اختيار عنصر معين داخل عنصر الأب بناءً على موقعه بين الأشقاء من نفس النوع.
- **فئة مزيفة `:only-of-type`**: تختار هذه الفئة المزيفة العنصر إذا كان الوحيد من نوعه داخل عنصر الأب.


## الفئات المزيفة الوظيفية

- **الفئات المزيفة الوظيفية**: تسمح الفئات المزيفة الوظيفية باختيار العناصر بناءً على شروط أو علاقات أكثر تعقيدًا. على عكس الفئات المزيفة العادية التي تستهدف العناصر بناءً على حالة معينة (مثل `:hover`، `:focus`)، تقبل الفئات المزيفة الوظيفية معلمات.
- **فئة مزيفة `:is()`**: تأخذ هذه الفئة المزيفة قائمة من المحددات (مثل `ol`، `ul`) وتختار عنصرًا يطابق أحد المحددات في القائمة.
- **فئة مزيفة `:where()`**: تأخذ هذه الفئة المزيفة قائمة من المحددات (مثل `ol`، `ul`) وتختار عنصرًا يطابق أحد المحددات في القائمة. الفرق بين `:is()` و`:where()` هو أن الأخيرة لها خصوصية (specificity) تساوي 0.
- **فئة مزيفة `:has()`**: غالبًا ما تُسمى محدد الـ`"parent"` لأنها تتيح لك تنسيق العناصر التي تحتوي على عناصر فرعية محددة في قائمة المحددات.

## العناصر المزيفة

- **العنصر المزيف `::before`**: يستخدم هذا العنصر الخاصية `content` لإدراج محتوى تجميلي مثل الأيقونات قبل العنصر مباشرة.
- **العنصر المزيف `::after`**: يستخدم هذا العنصر الخاصية `content` لإدراج محتوى تجميلي مثل الأيقونات بعد العنصر مباشرة.
- **العنصر المزيف `::first-letter`**: يستهدف الحرف الأول من محتوى العنصر، مما يسمح لك بتنسيقه.
- **العنصر المزيف `::marker`**: يتيح لك اختيار العلامة (النقطة أو الترقيم) لعناصر القائمة لتنسيقها.


## نظرية اللون

- **تعريف نظرية اللون**: هي دراسة كيفية تفاعل الألوان مع بعضها وكيف تؤثر على إدراكنا. تشمل العلاقات اللونية، تناغم الألوان، والتأثير النفسي للون.
- **الألوان الأساسية**: هي الألوان الأصفر، الأزرق، والأحمر، وهي الألوان الأساسية التي تُشتق منها جميع الألوان الأخرى.
- **الألوان الثانوية**: تنتج من مزج كميات متساوية من لونين أساسيين. من الأمثلة عليها الأخضر، البرتقالي، والبنفسجي.
- **الألوان الثلاثية**: تنتج من دمج لون أساسي مع لون ثانوي مجاور له. من الأمثلة عليها الأصفر-الأخضر، الأزرق-الأخضر، والأزرق-البنفسجي.
- **الألوان الدافئة**: تشمل الأحمر، البرتقالي، والأصفر، وتثير مشاعر الراحة والدفء والحميمية.
- **الألوان الباردة**: تشمل الأزرق، الأخضر، والبنفسجي، وتثير مشاعر الهدوء والسكينة والاحترافية.
- **عجلة الألوان**: هي مخطط دائري يوضح كيف ترتبط الألوان ببعضها. أداة أساسية للمصممين تساعدهم في اختيار تركيبات الألوان.
- **مخططات الألوان المتجاورة**: تخلق تجارب متناسقة ومهدئة، حيث تكون الألوان متجاورة على عجلة الألوان.
- **مخططات الألوان التكاملية**: تخلق تباينًا عاليًا وتأثيرًا بصريًا قويًا، حيث تقع الألوان على نهايات متقابلة من عجلة الألوان.
- **مخطط الألوان الثلاثي**: يحتوي على ألوان زاهية، تتكون من ألوان متباعدة تقريبًا بمسافات متساوية، تشكل مثلثًا متساوي الأضلاع على عجلة الألوان.
- **مخطط الألوان أحادي اللون**: جميع الألوان مشتقة من لون أساسي واحد مع تعديل فاتحته، داكنته، وتشبع اللون. يخلق شعورًا بالوحدة والتناغم مع وجود تباين.

## طرق مختلفة للعمل مع الألوان في CSS

- **الألوان المسماة**: هي أسماء ألوان محددة مسبقًا ومعترف بها من المتصفحات. من الأمثلة عليها `blue`، `darkred`، `lightgreen`.
- **دالة `rgb()`**: تمثل الأحمر والأخضر والأزرق، وهي الألوان الأساسية للضوء. تُدمج هذه الألوان بكثافات مختلفة لإنشاء مجموعة واسعة من الألوان. تسمح دالة `rgb()` بتحديد الألوان باستخدام نموذج RGB.
- **دالة `rgba()`**: تضيف قيمة رابعة، ألفا، تتحكم في شفافية اللون.
- **دالة `hsl()`**: تمثل اللون (Hue)، التشبع (Saturation)، والسطوع (Lightness) — وهي ثلاثة مكونات رئيسية تحدد اللون.
- **دالة `hsla()`**: تضيف قيمة رابعة، ألفا، تتحكم في تعتيم اللون.
- **التمثيل السداسي عشري (Hexadecimal)**: هو رمز مكون من ستة أحرف يستخدم لتمثيل الألوان في نموذج RGB. يشير "hex" إلى نظام العد الست عشري الذي يستخدم الأرقام من 0 إلى 9 والحروف من A إلى F.

## خاصية `box-shadow`

- **التعريف**: خاصية `box-shadow` تُطبّق ظلًا واحدًا أو أكثر حول العنصر.
- **قيم الإزاحة**: يجب تحديد قيم أفقية (`offset-x`) وعمودية (`offset-y`). القيمة الموجبة لـ `offset-x` تحرك الظل إلى اليمين، والقيم السالبة تحركه إلى اليسار. القيمة الموجبة لـ `offset-y` تحرك الظل إلى الأسفل، والقيم السالبة تحركه إلى الأعلى. إذا كانت القيمة `0`، فلا حاجة لتضمين وحدة قياس.
- **نصف قطر التمويه**: هذه القيمة اختيارية تتحكم في مدى ضبابية الظل. إذا لم تُذكر، تكون القيمة الافتراضية `0`، مما ينتج حوافًا حادة. كلما زادت القيمة، أصبح الظل أكثر نعومة.
- **نصف قطر الانتشار**: هذه القيمة اختيارية تتحكم في مدى توسع الظل أو تقلصه. إذا لم تُذكر، تكون القيمة الافتراضية `0`.
- **لون الظل**: يمكنك تحديد اللون باستخدام الألوان المسماة، القيم السداسية، أو دوال `rgb()`, `rgba()`, `hsl()`, أو `hsla()`.
- **الكلمة المفتاحية `inset`**: إضافة الكلمة المفتاحية `inset` تضع الظل داخل العنصر بدلًا من خارجه.
- **تطبيق عدة ظلال على الصندوق**: يمكنك تطبيق عدة ظلال بفصلها بفواصل. تُرتب الظلال من الأمام إلى الخلف.

### الصيغة النحوية

box-shadow: offset-x offset-y blur-radius spread-radius color;

## التدرجات الخطية والدائرية

- **التدرجات الخطية**: تخلق تدرجًا تدريجيًا بين الألوان على طول خط مستقيم. يمكنك التحكم في اتجاه هذا الخط والألوان المستخدمة.
- **التدرجات الدائرية**: تخلق تدرجات دائرية أو بيضاوية تشع من نقطة مركزية.

## أفضل الممارسات لتنسيق حقول الإدخال

- **تنسيق حقول الإدخال**: كما هو الحال مع جميع عناصر النص، يجب التأكد من أن التنسيقات التي تطبقها على حقول الإدخال قابلة للوصول. هذا يعني أن الخط يجب أن يكون بحجم مناسب، وأن يكون اللون متباينًا بشكل كافٍ مع الخلفية. كما أن عناصر الإدخال يمكن تركيزها. عند تعديل التنسيقات، احرص على الحفاظ على مؤشر واضح عند تركيز العنصر، مثل حدود عريضة.

## استخدام `appearance: none` لحقول الإدخال

- **`appearance: none`**: تطبق المتصفحات تنسيقات افتراضية على العديد من العناصر. خاصية CSS `appearance: none` تمنحك تحكمًا كاملاً في التنسيق، لكنها تأتي مع بعض التحفظات. عند بناء تنسيقات مخصصة لحقول الإدخال، يجب التأكد من استمرار وجود مؤشرات التركيز والأخطاء.

## المشكلات الشائعة في تنسيق خصائص `datetime-local` و`color`

- **المشكلات الشائعة**: تعتمد هذه الأنواع الخاصة من حقول الإدخال على عناصر مزيفة معقدة لإنشاء أدوات اختيار التاريخ واللون. هذا يشكل تحديًا كبيرًا لتنسيق هذه الحقول. أحد التحديات هو أن التنسيق الافتراضي يعتمد كليًا على المتصفح، لذا قد تختلف الشفرة التي تكتبها لجعل أداة الاختيار تبدو كما تريد بين متصفح وآخر.

## خاصية تجاوز السعة (Overflow) في CSS

- **التعريف**: يشير تجاوز السعة إلى كيفية تعامل العناصر مع المحتوى الذي يتجاوز حجم العنصر الحاوي. تجاوز السعة ثنائي الأبعاد.
- **`overflow-x`**: المحور السيني يحدد تجاوز السعة الأفقي.
- **`overflow-y`**: المحور الصادي يحدد تجاوز السعة العمودي.

## خاصية التحويل (Transform) في CSS

- **التعريف**: تتيح هذه الخاصية تطبيق تحولات مختلفة على العناصر، مثل التدوير، التكبير، الإمالة، أو الترجمة (التحريك) في الفضاء ثنائي أو ثلاثي الأبعاد.
- **دالة `translate()`**: تُستخدم لتحريك عنصر من موقعه الحالي.
- **دالة `scale()`**: تسمح بتغيير حجم العنصر.
- **التحولات وإمكانية الوصول**: إذا كنت تستخدم التحويل لإخفاء أو إظهار المحتوى، تأكد من أن المحتوى لا يزال متاحًا لقارئات الشاشة والتنقل عبر لوحة المفاتيح. يجب أن يكون المحتوى المخفي مخفيًا فعليًا، مثل استخدام `display: none` أو `visibility: hidden`، وليس مجرد تحريكه خارج الشاشة بصريًا.

## نموذج الصندوق (Box Model)

- **التعريف**: في نموذج الصندوق في CSS، يُحيط بكل عنصر صندوق يتكون من أربعة مكونات: مساحة المحتوى، `padding`، `border`، و`margin`.
- **مساحة المحتوى**: هي الجزء الداخلي للصندوق، المساحة التي تحتوي على المحتوى الفعلي للعنصر، مثل النص أو الصور.
- **`padding`**: هو المساحة التي تلي مباشرة مساحة المحتوى، وهي المسافة بين المحتوى وحدود العنصر.
- **`border`**: هو الحافة الخارجية أو الإطار للعنصر في نموذج الصندوق.
- **`margin`**: هو المساحة خارج حدود العنصر، ويحدد المسافة بين العنصر والعناصر المحيطة به.

## تداخل الهوامش (Margin Collapse)

- **التعريف**: يحدث هذا السلوك عندما تتداخل الهوامش الرأسية للعناصر المتجاورة، مما ينتج هامشًا واحدًا يساوي الأكبر بينهما. ينطبق هذا السلوك فقط على الهوامش الرأسية (الأعلى والأسفل)، وليس الأفقية (اليسار واليمين).

## قيم خاصيتي `content-box` و`border-box`

- **خاصية `box-sizing`**: تُستخدم هذه الخاصية لتحديد كيفية حساب القيم النهائية لـ `width` و `height` لعنصر HTML.
- **قيمة `content-box`**: في نموذج `content-box`، تحدد قيمتي `width` و `height` التي تعيّنها لعنصر أبعاد مساحة المحتوى فقط، ولا تشمل الـ `padding` أو الـ `border` أو الـ `margin`.
- **قيمة `border-box`**: مع `border-box`، تشمل قيمتا `width` و `height` مساحة المحتوى، و`padding`، و`border`، لكنها لا تشمل الـ `margin`.

## إعادة تعيين CSS (CSS Reset)

- **التعريف**: إعادة تعيين CSS هي ورقة أنماط تزيل كل أو بعض التنسيقات الافتراضية التي يطبقها المتصفح على عناصر HTML. من الخيارات الخارجية لإعادة التعيين `sanitize.css` و`normalize.css`.

## خاصية فلتر CSS

- **التعريف**: يمكن استخدام هذه الخاصية لإنشاء تأثيرات مختلفة مثل التمويه، تغيير اللون، وتعديل التباين.
- **دالة `blur()`**: تطبق تمويهًا غاوسيًا على العنصر. تُحدد الكمية بالبكسل وتمثل نصف قطر التمويه.
- **دالة `brightness()`**: تضبط سطوع العنصر. القيمة 0% تجعل العنصر أسود تمامًا، والقيم فوق 100% تزيد السطوع.
- **دالة `grayscale()`**: تحول العنصر إلى تدرج رمادي. تُحدد الكمية كنسبة مئوية، حيث 100% تعني تدرج رمادي كامل و0% تعني عدم تغيير الصورة.
- **دالة `sepia()`**: تطبق نغمة سيبيا على العنصر. تستخدم قيمة مئوية مثل التدرج الرمادي.
- **دالة `hue-rotate()`**: تطبق تدويرًا للون على العنصر. تُحدد القيمة بالدرجات وتمثل تدويرًا حول دائرة الألوان.

## مقدمة إلى CSS Flexbox ونموذج Flex

- **التعريف**: CSS flexbox هو نموذج تخطيط أحادي البعد يسمح بترتيب العناصر في صفوف وأعمدة داخل حاوية.
- **نموذج Flex**: يحدد كيفية ترتيب العناصر المرنة داخل الحاوية المرنة. لكل حاوية مرنة محوران: المحور الرئيسي والمحور المتقاطع.

## خاصية `flex-direction`

- **التعريف**: تحدد هذه الخاصية اتجاه المحور الرئيسي. القيمة الافتراضية لـ `flex-direction` هي `row`، التي تضع كل العناصر المرنة في صف واحد، باتجاه لغة المتصفح الافتراضية (من اليسار إلى اليمين أو العكس).
- **`flex-direction: row-reverse;`**: يعكس ترتيب العناصر في الصف.
- **`flex-direction: column;`**: يصطف العناصر عموديًا بدلًا من أفقيًا.
- **`flex-direction: column-reverse;`**: يعكس ترتيب العناصر عموديًا.

## خاصية `flex-wrap`

- **التعريف**: تحدد هذه الخاصية كيفية التفاف العناصر المرنة داخل الحاوية المرنة لتناسب المساحة المتاحة. يمكن أن تأخذ `flex-wrap` ثلاث قيم: `nowrap`، `wrap`، و`wrap-reverse`.
- **`flex-wrap: nowrap;`**: القيمة الافتراضية. لا تُلف العناصر إلى سطر جديد حتى لو تجاوز عرضها عرض الحاوية.
- **`flex-wrap: wrap;`**: تلف العناصر عندما تتجاوز عرض الحاوية.
- **`flex-wrap: wrap-reverse;`**: تلف العناصر بترتيب عكسي.
- **خاصية `flex-flow`**: خاصية مختصرة لـ `flex-direction` و`flex-wrap`.

## خاصية `justify-content`

- **التعريف**: تصطف هذه الخاصية العناصر الفرعية على طول المحور الرئيسي للحاوية المرنة.
- **`justify-content: flex-start;`**: تصطف العناصر عند بداية المحور الرئيسي، أفقيًا أو عموديًا.
- **`justify-content: flex-end;`**: تصطف العناصر عند نهاية المحور الرئيسي، أفقيًا أو عموديًا.
- **`justify-content: center;`**: توسيط العناصر على المحور الرئيسي.
- **`justify-content: space-between;`**: توزع العناصر بالتساوي على المحور الرئيسي.
- **`justify-content: space-around;`**: توزع العناصر بالتساوي مع مساحة قبل أول عنصر وبعد آخر عنصر.
- **`justify-content: space-evenly;`**: توزع العناصر بالتساوي على المحور الرئيسي.
  
## خاصية `align-items`

- **التعريف**: تُستخدم لتوزيع العناصر على المحور المتقاطع، وهو عمودي على المحور الرئيسي.
- **`align-items: center;`**: توسيط العناصر على المحور المتقاطع.
- **`align-items: flex-start;`**: تصطف العناصر عند بداية المحور المتقاطع.
- **`align-items: stretch;`**: تمدد العناصر على المحور المتقاطع.

## مقدمة إلى الطباعة (Typography)

- **التعريف**: الطباعة هي فن اختيار الخطوط والتنسيق لجعل النص جذابًا بصريًا وسهل القراءة. "النوع" يشير إلى كيفية تصميم وترتيب الحروف الفردية.
- **تعريف نوع الخط (Typeface)**: هو التصميم العام والأسلوب لمجموعة من الحروف والأرقام والرموز. يشبه المخطط الأساسي لعائلة الخطوط.
- **تعريف الخط (Font)**: هو نسخة محددة من نوع الخط بخصائص معينة مثل الحجم، الوزن، الأسلوب، والعرض.
- **نوع الخط المزخرف (Serif)**: له أسلوب كلاسيكي مع خطوط صغيرة في نهاية الحروف. يُستخدم عادة في المواد المطبوعة مثل الكتب.
- **نوع الخط غير المزخرف (Sans Serif)**: له مظهر حديث بدون الخطوط الصغيرة في نهاية الحروف. يُستخدم كثيرًا في التصميم الرقمي لسهولة قراءته على الشاشة. من الأمثلة Helvetica، Arial، وRoboto.
- **وزن الخط**: سمك الحروف، مثل خفيف، عادي، عريض، وأسود.
- **أسلوب الخط**: ميلان واتجاه الحروف، مثل المائل والمائل المائل.
- **عرض الخط**: المساحة الأفقية التي تشغلها الحروف، مثل المضغوط والموسع.
- **الخط الأساسي (Baseline)**: الخط الخيالي الذي تستقر عليه معظم الحروف.
- **ارتفاع الحروف الكبيرة (Cap Height)**: ارتفاع الحروف الكبيرة من الخط الأساسي إلى الأعلى.
- **ارتفاع الحروف الصغيرة (X-height)**: متوسط ارتفاع الحروف الصغيرة، باستثناء الأجزاء المرتفعة والمنخفضة.
- **الأجزاء المرتفعة (Ascenders)**: أجزاء الحروف الصغيرة التي تمتد فوق ارتفاع الحروف الصغيرة، مثل رؤوس الحروف 'h'، 'b'، 'd'، و'f'.
- **الأجزاء المنخفضة (Descenders)**: أجزاء الحروف الصغيرة التي تمتد تحت الخط الأساسي، مثل ذيول الحروف 'y'، 'g'، 'p'، و'q'.
- **التباعد بين الحروف (Kerning)**: تعديل المسافة بين أزواج محددة من الحروف لتحسين القراءة والجمالية، مثل تقليل المسافة بين الحرفين A وV.
- **التتبع (Tracking)**: تعديل المسافة بين جميع الحروف في نص معين، يؤثر على كثافة النص وفتحه.
- **المسافة بين الأسطر (Leading)**: المسافة الرأسية بين أسطر النص، تُقاس من الخط الأساسي لسطر إلى الخط الأساسي للسطر التالي.
- **أفضل الممارسات في الطباعة**: اختر خطوطًا واضحة وبسيطة لجعل تصميماتك سهلة الفهم، خاصة للنص الرئيسي في موقعك. يميل المستخدمون إلى التفاعل أكثر إذا كان الخط سهل القراءة. استخدم خطين أو ثلاثة كحد أقصى لخلق تناسق بصري. استخدام الكثير من الخطوط يصعب القراءة ويضعف هوية العلامة التجارية. يؤثر ذلك أيضًا على تجربة المستخدم بزيادة وقت تحميل الموقع. يمكنك استخدام حجم الخط لإنشاء تسلسل هرمي بصري للعناوين والعناوين الفرعية والفقرات والعناصر الأخرى. على سبيل المثال، يجب أن يكون العنوان الرئيسي بحجم خط أكبر، يليه العناوين الفرعية بحجوم أصغر. هذا يعطي كل عنصر في التسلسل الهرمي حجم خط محدد يساعد المستخدمين على التنقل بصريًا.

  
## خاصية CSS `font-family`

- **التعريف**: عائلة الخطوط مجموعة من الخطوط تشترك في تصميم مشترك. وتستند جميع الخطوط التي تنتمي إلى العائلة نفسها إلى المحرف الأساسي نفسه، لكنها تختلف أيضًا في نمطها ووزنها وعرضها. ويمكنك تحديد عدة عائلات خطوط بترتيب الأولوية، من الأعلى إلى الأدنى، بفصلها بفواصل. وستعمل هذه الخطوط البديلة كخيارات احتياطية. وينبغي أن تُضمّن دائمًا عائلة خطوط عامة في نهاية قائمة font-family.
- **عائلات الخطوط الشائعة**: فيما يلي بعض عائلات الخطوط الشائعة المستخدمة في CSS: serif وsans-serif وmonospace وcursive وfantasy

## خطوط الويب الآمنة

- **التعريف**: هي مجموعة فرعية من الخطوط التي من المرجح جدًا أن تكون مثبتة على جهاز الكمبيوتر أو الجهاز. عندما يحتاج المتصفح إلى عرض خط، يحاول العثور على ملف الخط على نظام المستخدم. وإذا لم يُعثر على الخط، عادةً ما يعود إلى خط نظام افتراضي.

- **خطوط الويب الآمنة**:
  - تُستخدم خطوط sans-serif كثيرًا في تطوير الويب لأنها لا تحتوي على "أقدام" صغيرة أو خطوط في نهاية الحروف، مما يجعلها سهلة القراءة على الشاشة. من الأمثلة على خطوط sans-serif الآمنة للويب: Arial، Verdana، وTrebuchet MS.
  - بالمقابل، تحتوي خطوط serif على "أقدام" صغيرة في نهاية الحروف، لذا تُستخدم عادة في الطباعة التقليدية. من خطوط serif الآمنة للويب: Times New Roman وGeorgia.

## قواعد عند (At-rules) وقاعدة `@font-face`

- **التعريف**: قواعد عند هي تعليمات توجه المتصفح. يمكنك استخدامها لتعريف جوانب مختلفة من ورقة الأنماط، مثل استعلامات الوسائط، الإطارات المفتاحية، وجوه الخطوط، والمزيد.
- **`@font-face`**: تسمح لك بتعريف خط مخصص بتحديد ملف الخط، التنسيق، وخصائص مهمة أخرى مثل الوزن والأسلوب. لكي تكون قاعدة `@font-face` صالحة، يجب أيضًا تحديد خاصية `src` التي تحتوي على مراجع لمصادر الخط.
- **تنسيقات الخطوط**: لكل مصدر خط، يمكنك أيضًا تحديد التنسيق. هذا اختياري. هو تلميح للمتصفح حول تنسيق الخط. إذا تم حذف التنسيق، سيتم تنزيل المصدر وسيتم اكتشاف التنسيق بعد التنزيل. إذا كان التنسيق غير صالح، فلن يتم تنزيل المصدر. من تنسيقات الخطوط الممكنة: `collection`، `embedded-opentype`، `opentype`، `svg`، `truetype`، `woff`، و`woff2`.
- **`woff` و`woff2`**: `woff` تعني "Web Open Font Format"، وهو تنسيق خط شائع طورته موزيلا بالتعاون مع Type Supply وLettError ومنظمات أخرى. الفرق بين `woff` و`woff2` هو خوارزمية ضغط البيانات المستخدمة.
- **OpenType**: هو تنسيق لخطوط الحاسوب القابلة للتوسع طورته مايكروسوفت وأدوبي، يسمح للمستخدمين بالوصول إلى ميزات إضافية في الخط. يُستخدم على نطاق واسع عبر أنظمة التشغيل الرئيسية.
- **`tech()`**: تُستخدم لتحديد تقنية مصدر الخط. هذا اختياري.

## العمل مع الخطوط الخارجية

- **التعريف**: الخط الخارجي هو ملف خط غير مضمن مباشرة ضمن ملفات مشروعك. عادةً ما يكون مستضافًا على خادم منفصل. لإدراج هذه الخطوط الخارجية داخل مشروعك، يمكنك استخدام عنصر `link` أو تعليمة `@import` داخل CSS.
- **خطوط Google**: خدمة من Google تقدم مجموعة من الخطوط، العديد منها مصمم خصيصًا لتطوير الويب.
- **Font Squirrel**: مورد شائع آخر يمكنك استخدامه لإضافة خطوط خارجية مخصصة إلى مشاريعك.

## خاصية `text-shadow`

- **التعريف**: تُستخدم هذه الخاصية لتطبيق ظلال على النص. يجب تحديد إزاحة X وY، التي تمثل المسافة الأفقية والعمودية للظل عن النص، على التوالي. هذه القيم مطلوبة. القيم الموجبة لإزاحة X وY تحرك الظل إلى اليمين والأسفل، على التوالي، بينما القيم السالبة تحرك الظل إلى اليسار والأعلى.
- **لون الظل**: يمكنك أيضًا تخصيص لون الظل بتحديد هذه القيمة قبل أو بعد الإزاحة. إذا لم يُحدد اللون، يحدد المتصفح اللون تلقائيًا، لذا من الأفضل تعيينه صراحة.
- **نصف قطر التمويه**: نصف قطر التمويه اختياري لكنه يجعل الظل يبدو أكثر نعومة ودقة. القيمة الافتراضية للتمويه هي صفر. كلما زادت القيمة، زاد التمويه، مما يجعل الظل أخف.
- **تطبيق عدة ظلال نصية**: يمكن للنص أن يحتوي على أكثر من ظل. تُطبق الظلال على طبقات، من الأمام إلى الخلف، مع الظل الأول في الأعلى.

## أدوات تباين الألوان

- **أداة فحص تباين الألوان من WebAIM**: أداة عبر الإنترنت تتيح لك إدخال ألوان المقدمة والخلفية لتصميمك ورؤية فورية إذا كانت تلبي معايير إرشادات إمكانية الوصول لمحتوى الويب (WCAG).
- **محلل تباين الألوان TPGi**: أداة مجانية لفحص تباين الألوان تتيح لك التحقق مما إذا كانت مواقعك وتطبيقاتك تلبي معايير WCAG. تأتي هذه الأداة أيضًا مع ميزة محاكاة عمى الألوان لتتمكن من رؤية كيف يبدو موقعك أو تطبيقك للأشخاص الذين يعانون من مشاكل في رؤية الألوان.

## شجرة إمكانية الوصول

شجرة إمكانية الوصول هي هيكل تستخدمه التقنيات المساعدة، مثل قارئات الشاشة، لتفسير والتفاعل مع محتوى صفحة الويب.

## دالة `max()`

تعيد دالة `max()` أكبر قيمة من مجموعة قيم مفصولة بفواصل:

```css
img {
  width: max(250px, 25vw);
}
```

في المثال أعلاه، سيكون عرض الصورة 250 بكسل إذا كان عرض إطار العرض أقل من 1000 بكسل (لأن 250 بكسل أكبر من 25% من عرض إطار العرض). إذا كان عرض إطار العرض أكبر من 1000 بكسل، سيكون عرض الصورة 25vw. ذلك لأن 25vw تساوي 25% من عرض إطار العرض.

## أفضل الممارسات مع CSS وإمكانية الوصول

- **`display: none;`**: استخدام `display: none;` يعني أن قارئات الشاشة والتقنيات المساعدة الأخرى لن تتمكن من الوصول إلى هذا المحتوى، لأنه غير مدرج في شجرة إمكانية الوصول. لذلك، من المهم استخدام هذه الطريقة فقط عندما تريد إزالة المحتوى تمامًا من العرض البصري وإمكانية الوصول.
- **`visibility: hidden;`**: تخفي هذه الخاصية المحتوى بصريًا لكنها تبقيه في تدفق المستند، مما يعني أنه لا يزال يشغل مساحة في الصفحة. كما أن هذه العناصر لن تُقرأ بواسطة قارئات الشاشة لأنها تُزال من شجرة إمكانية الوصول.
- **فئة CSS `.sr-only`**: تقنية شائعة تُستخدم لإخفاء المحتوى بصريًا مع إبقائه متاحًا لقارئات الشاشة.

```css
.sr-only {
  position: absolute;
  width: 1px;
  height: 1px;
  padding: 0;
  margin: -1px;
  overflow: hidden;
  clip: rect(0, 0, 0, 0);
  white-space: nowrap;
  border: 0;
}
```

- **`scroll-behavior: smooth;`**: تُمكّن هذه الخاصية سلوك التمرير السلس.
- **ميزة `prefers-reduced-motion`**: هي ميزة وسائط تُستخدم لاكتشاف تفضيل المستخدم للحد من الرسوم المتحركة.

```css
@media (prefers-reduced-motion: no-preference) {
  * {
    scroll-behavior: smooth;
  }
}
```

في المثال أعلاه، يتم تمكين التمرير السلس إذا لم يكن لدى المستخدم تفضيل للرسوم المتحركة مضبوط على جهازه.

## إخفاء المحتوى باستخدام سمات HTML

- **سمة `aria-hidden`**: تُستخدم لإخفاء عنصر عن الأشخاص الذين يستخدمون تقنيات مساعدة مثل قارئات الشاشة. على سبيل المثال، يمكن استخدامها لإخفاء الصور الزخرفية التي لا تقدم محتوى ذا معنى.
- **سمة `hidden`**: تدعمها معظم المتصفحات الحديثة وتخفي المحتوى بصريًا ومن شجرة إمكانية الوصول. يمكن التبديل بينها بسهولة باستخدام جافاسكريبت.

:::interactive_editor

```html
<p aria-hidden>This paragraph is visible for sighted users, but is hidden from assistive technology.</p>
<p hidden>This paragraph is hidden from both sighted users and assistive technology.</p>
```

:::

## مشكلة إمكانية الوصول في خاصية `placeholder`

استخدام نص العنصر النائب ليس جيدًا لإمكانية الوصول. كثيرًا ما يخلط المستخدمون بين نص العنصر النائب وقيمة الإدخال الفعلية — يظنون أن هناك قيمة موجودة بالفعل في الإدخال.

## العمل مع محددات الخصائص المختلفة والروابط

- **التعريف**: يسمح محدد `attribute` باستهداف عناصر HTML بناءً على خصائصها مثل خصائص `href` أو `title`.
- **خاصية `title`**: توفر هذه الخاصية معلومات إضافية عن العنصر.

## استهداف العناصر باستخدام خاصيتي `lang` و`data-lang`

- **خاصية `lang`**: تُستخدم هذه الخاصية في HTML لتحديد لغة المحتوى داخل عنصر معين. قد ترغب في تنسيق العناصر بشكل مختلف بناءً على اللغة التي كُتبت بها، خاصةً في موقع ويب متعدد اللغات.
- **خاصية `data-lang`**: تُستخدم خصائص البيانات المخصصة مثل `data-lang` عادةً لتخزين معلومات إضافية داخل العناصر، مثل تحديد اللغة المستخدمة في قسم معين من النص.


## العمل مع محدد الخاصية، عناصر القائمة المرتبة، وخصيصة `type`

- **خاصية `type`**: عند العمل مع القوائم المرتبة في HTML، تتيح لك خاصية `type` تحديد نمط الترقيم المستخدم، مثل الأرقام، الحروف الأبجدية، أو الأرقام الرومانية.

## العمل مع العناصر العائمة (Floats)

- **التعريف**: تُستخدم العناصر العائمة لإخراج عنصر من تدفقه الطبيعي في الصفحة وتحديد موقعه إما على الجانب الأيسر أو الأيمن من الحاوية الخاصة به. عند حدوث ذلك، يلتف النص حول هذا العنصر العائم.
- **مسح العناصر العائمة**: تُستخدم خاصية `clear` لتحديد ما إذا كان يجب نقل عنصر أسفل المحتوى العائم. عند وجود عدة عناصر عائمة متجاورة، قد تحدث مشاكل في التداخل والانهيار في التخطيطات، لذا تم ابتكار حيلة `clearfix` لحل هذه المشكلة.

## التمركز الثابت، النسبي والمطلق

- **التمركز الثابت**: هذا هو التدفق الطبيعي للوثيقة. تُوضع العناصر من الأعلى إلى الأسفل، ومن اليسار إلى اليمين واحدة تلو الأخرى.
- **التمركز النسبي**: يسمح لك باستخدام خصائص `top`، `left`، `right`، و`bottom` لتحديد موقع العنصر ضمن التدفق الطبيعي للوثيقة. يمكنك أيضًا استخدام التمركز النسبي لجعل العناصر تتداخل مع عناصر أخرى في الصفحة.
- **التمركز المطلق**: يسمح لك بإخراج العنصر من التدفق الطبيعي للوثيقة، مما يجعله يتصرف بشكل مستقل عن العناصر الأخرى.

## التمركز الثابت واللزج

- **التمركز الثابت**: عندما يُحدد عنصر باستخدام `position: fixed`، يُزال من التدفق الطبيعي للوثيقة ويُوضع بالنسبة لإطار العرض، مما يعني أنه يبقى في نفس الموقع حتى عند تمرير المستخدم للصفحة. يُستخدم هذا غالبًا لعناصر مثل الأقسام العلوية أو أشرطة التنقل التي يجب أن تبقى مرئية دائمًا.
- **التمركز اللزج**: هذا النوع من التمركز يعمل كخليط بين التمركز النسبي والثابت. في البداية، يتصرف العنصر كما لو كان محددًا نسبيًا، ويبقى ضمن تدفق الوثيقة. ولكن بمجرد أن يمرر المستخدم العنصر بعد نقطة معينة، يصبح "لزجًا" في إطار العرض (عادةً في الأعلى) ويتصرف كما لو كان ثابتًا. هذا مفيد لإنشاء عناصر مثل أشرطة التنقل اللزجة التي تصبح ثابتة فقط عند التمرير إلى موقع معين.

## العمل مع خاصية `z-index`

- **التعريف**: تُستخدم خاصية `z-index` في CSS للتحكم في ترتيب تراكب العناصر المحددة الموضع عموديًا عندما تتداخل على الصفحة.

## تصميم الويب المتجاوب

- **التعريف**: المبدأ الأساسي للتصميم المتجاوب هو القدرة على التكيف: قدرة الموقع على تعديل تخطيطه ومحتواه بناءً على حجم الشاشة وإمكانيات الجهاز الذي يُعرض عليه.
- **الشبكات السائلة**: تستخدم وحدات نسبية مثل النسب المئوية بدلًا من الوحدات الثابتة مثل البكسل، مما يسمح للمحتوى بإعادة الحجم وإعادة التدفق بناءً على حجم الشاشة.
- **الصور المرنة**: تُضبط لتغيير حجمها داخل الحاويات الخاصة بها، مما يضمن عدم تجاوزها حدود الحاوية على الشاشات الصغيرة.

## استعلامات الوسائط

- **التعريف**: تتيح استعلامات الوسائط للمطورين تطبيق أنماط مختلفة بناءً على خصائص الجهاز، خصوصًا عرض إطار العرض.
- **نوع الوسائط `all`**: مناسب لجميع الأجهزة. هذا هو الافتراضي إذا لم يُحدد نوع وسائط.
- **أنواع الوسائط `print`**: مخصصة للمواد المطبوعة والوثائق المعروضة على الشاشة في وضع معاينة الطباعة.
- **أنواع الوسائط `screen`**: مخصصة أساسًا للشاشات.
- **`aspect-ratio`**: يصف نسبة العرض إلى الارتفاع لإطار العرض.
- **`orientation`**: يُستخدم لتحديد ما إذا كان الجهاز في وضع العرض الأفقي أو الرأسي.
- **`resolution`**: يُستخدم لوصف دقة جهاز الإخراج بوحدات النقاط لكل إنش (dpi) أو النقاط لكل سنتيمتر (dpcm).
- **`hover`**: يُستخدم لاختبار ما إذا كانت آلية الإدخال الرئيسية يمكنها التحويم فوق العناصر.
- **`prefers-color-scheme`**: يُستخدم لاكتشاف ما إذا كان المستخدم قد طلب سمة ألوان فاتحة أو داكنة.
- **استعلامات الوسائط والمعاملات المنطقية**: يُستخدم المعامل `and` لدمج ميزات وسائط متعددة، بينما يمكن استخدام `not` و`only` لنفي أو عزل استعلامات الوسائط.

## نقاط توقف الوسائط الشائعة

- **التعريف**: نقاط توقف الوسائط هي نقاط محددة في تصميم الموقع حيث يتغير التخطيط والمحتوى ليتناسب مع أحجام الشاشات المختلفة. هناك بعض نقاط التوقف العامة التي يمكنك استخدامها لاستهداف الهواتف، والأجهزة اللوحية، وشاشات سطح المكتب. لكن من غير الحكمة محاولة استهداف كل حجم شاشة ممكن لكل جهاز.
- **الأجهزة الصغيرة (الهواتف الذكية)**: حتى 640 بكسل
- **الأجهزة المتوسطة (الأجهزة اللوحية)**: من 641 بكسل إلى 1024 بكسل
- **الأجهزة الكبيرة (أجهزة سطح المكتب)**: 1025 بكسل وما فوق

## نهج التصميم أولًا للجوال

- **التعريف**: نهج التصميم أولًا للجوال هو فلسفة تصميم واستراتيجية تطوير في تصميم الويب المتجاوب تعطي الأولوية لإنشاء مواقع للجوال قبل تصميم الشاشات الأكبر.

## خصائص CSS المخصصة (متغيرات CSS)

- **التعريف**: خصائص CSS المخصصة، المعروفة أيضًا بمتغيرات CSS، هي كيانات يحددها مؤلفو CSS تحتوي على قيم محددة لإعادة استخدامها في جميع أنحاء المستند. إنها ميزة قوية تسمح بإنشاء أوراق أنماط أكثر كفاءة وقابلية للصيانة ومرونة. الخصائص المخصصة مفيدة بشكل خاص في إنشاء تصاميم قابلة للتخصيص. يمكنك تعريف مجموعة من الخصائص لمواضيع مختلفة.

## قاعدة `@property`

- **التعريف**: قاعدة `@property` هي ميزة قوية في CSS تتيح للمطورين تعريف خصائص مخصصة مع تحكم أكبر في سلوكها، بما في ذلك كيفية تحريكها وقيمها الأولية.

```css
@property --property-name {
  syntax: '<type>';
  inherits: true | false;
  initial-value: <value>;
}
```

- **`--property-name`**: هذا هو اسم الخاصية المخصصة التي تعرفها. مثل جميع الخصائص المخصصة، يجب أن تبدأ بشرطتين.
- **`syntax`**: يحدد نوع الخاصية، والتي يمكن أن تكون مثل `<color>`، `<length>`، `<number>`، `<percentage>`، أو أنواع أكثر تعقيدًا.
- **`inherits`**: يحدد ما إذا كانت الخاصية يجب أن ترث قيمتها من العنصر الأب.
- **`initial-value`**: يحدد القيمة الافتراضية للخاصية.
- **النظام البديل (Fallbacks)**: عند استخدام الخاصية المخصصة، يمكنك توفير قيمة بديلة باستخدام دالة `var()`، تمامًا كما تفعل مع الخصائص المخصصة القياسية.

## أساسيات شبكة CSS Grid

- **التعريف**: CSS Grid هو نظام تخطيط ثنائي الأبعاد يُستخدم لإنشاء تخطيطات معقدة في صفحات الويب. تحتوي الشبكات على صفوف وأعمدة مع فراغات بينها. لتعريف تخطيط شبكة، تضبط خاصية `display` على القيمة `grid`.
- **وحدة `fr` (الكسرية)**: تمثل هذه الوحدة جزءًا من المساحة داخل حاوية الشبكة. يمكنك استخدام وحدة `fr` لإنشاء شبكات مرنة.
- **إنشاء فراغات بين المسارات**: هناك ثلاث طرق لإنشاء فراغات بين المسارات في شبكة CSS. يمكنك استخدام خاصية `column-gap` لإنشاء فراغات بين الأعمدة. يمكنك استخدام خاصية `row-gap` لإنشاء فراغات بين الصفوف. أو يمكنك استخدام خاصية `gap` المختصرة لإنشاء فراغات بين الصفوف والأعمدة معًا.
- **دالة `repeat()`**: تُستخدم هذه الدالة لتكرار أقسام في قائمة المسارات. بدلًا من كتابة `grid-template-columns: 1fr 1fr 1fr;` يمكنك استخدام دالة `repeat()` بدلاً من ذلك.
- **الشبكة الصريحة**: يمكنك تحديد عدد الخطوط أو المسارات باستخدام خصائص `grid-template-columns` أو `grid-template-rows`.
- **الشبكة الضمنية**: عندما توضع العناصر خارج الشبكة، يتم إنشاء صفوف وأعمدة تلقائيًا لتلك العناصر الخارجية.
- **دالة `minmax()`**: تُستخدم هذه الدالة لتحديد الحد الأدنى والحد الأقصى لأحجام المسار.
- **التوزيع التلقائي للعناصر في الشبكة**: يضع المتصفح العناصر تلقائيًا باستخدام خوارزمية التوزيع التلقائي. تتحكم خاصية `grid-auto-flow` في هذا السلوك، باستخدام قيم مثل `row`، `column`، أو `dense` للتأثير على اتجاه التوزيع وما إذا كان سيتم ملء الفراغات الفارغة.
- **التحديد بناءً على الخطوط**: تحتوي جميع الشبكات على خطوط. لتحديد مكان بدء العنصر على خط، يمكنك استخدام خصائص `grid-column-start` و`grid-row-start`. لتحديد مكان انتهاء العنصر على الخط، يمكنك استخدام خصائص `grid-column-end` و`grid-row-end`. يمكنك أيضًا استخدام الخصائص المختصرة `grid-column` أو `grid-row` بدلاً من ذلك.
- **خاصية `grid-template-areas`**: تُستخدم هذه الخاصية لتوفير أسماء للعناصر التي ستوضع على الشبكة.
- **المحاذاة:** استخدم `justify-items` و`align-items` للتحكم في كيفية محاذاة العناصر داخل مناطق الشبكة الخاصة بها. استخدم `justify-self` و`align-self` على عناصر الشبكة الفردية لتجاوز المحاذاة المحددة على الحاوية.

## أساسيات تحريك CSS

- **التعريف**: تتيح تحريكات CSS إنشاء تأثيرات ديناميكية وجذابة بصريًا على صفحات الويب دون الحاجة إلى جافاسكريبت أو برمجة معقدة. توفر طريقة للانتقال السلس للعناصر بين أنماط مختلفة خلال مدة محددة.
- **قاعدة `@keyframes`**: تحدد هذه القاعدة مراحل وأنماط التحريك. توضح الأنماط التي يجب أن يمتلكها العنصر في نقاط مختلفة خلال التحريك.
- **خاصية `animation`**: هذه الخاصية المختصرة تُستخدم لتطبيق التحريكات.
- **`animation-name`**: تحدد اسم قاعدة `@keyframes` التي ستُستخدم.
- **`animation-duration`**: تحدد مدة إكمال التحريك.
- **`animation-timing-function`**: تحدد كيفية تقدم التحريك مع الزمن (مثل ease، linear، ease-in-out).
- **`animation-delay`**: تحدد تأخيرًا قبل بدء التحريك.
- **`animation-iteration-count`**: تحدد عدد مرات تكرار التحريك.
- **`animation-direction`**: تحدد ما إذا كان التحريك يجب أن يُشغل للأمام، للخلف، أو بالتناوب.
- **`animation-fill-mode`**: تحدد كيف يجب تنسيق العنصر قبل وبعد التحريك.
- **`animation-play-state`**: تتيح إيقاف واستئناف التحريك.

## إمكانية الوصول واستعلام الوسائط `prefers-reduced-motion`

- **استعلام الوسائط `prefers-reduced-motion`**: من أهم مخاوف إمكانية الوصول مع التحريكات أنها قد تسبب انزعاجًا أو حتى ضررًا جسديًا لبعض المستخدمين. قد يعاني الأشخاص الذين لديهم اضطرابات في الجهاز الدهليزي أو حساسية للحركة من دوار، غثيان، أو صداع عند تعرضهم لأنواع معينة من الحركة على الشاشة. يسمح استعلام الوسائط `prefers-reduced-motion` لمطوري الويب باكتشاف ما إذا كان المستخدم قد طلب تقليل التحريكات أو تأثيرات الحركة على مستوى النظام.
