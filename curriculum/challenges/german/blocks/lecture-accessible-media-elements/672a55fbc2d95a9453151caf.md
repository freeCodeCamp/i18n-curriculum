# --interactive--

Viele Nutzerinnen und Nutzer sind aufgrund körperlicher Einschränkungen, wiederholter Belastungsverletzungen oder persönlicher Vorlieben auf die Tastatur statt auf die Maus angewiesen. Dazu gehören auch Nutzerinnen und Nutzer von Screenreadern sowie solche, die keine funktionierende Maus haben.
Die Barrierefreiheit für die Tastatur stellt sicher, dass diese Nutzerinnen und Nutzer Webanwendungen effektiv und ohne Hindernisse bedienen können.

Schauen wir uns einige praktische Techniken an, mit denen Sie Webanwendungen tastaturzugänglich machen können.

Viele Nutzerinnen und Nutzer verwenden die `Tab`-Taste, um durch interaktive Elemente auf einer Webseite zu navigieren. Standardmäßig erlauben Browser das Tabben durch Elemente wie Links, Buttons und Formularfelder in der Reihenfolge, in der sie im HTML erscheinen. Dies nennt man die natürliche Tab-Reihenfolge.

Manchmal möchten Sie vielleicht festlegen, welche Elemente fokussierbar sind oder die Reihenfolge ihres Fokus ändern. Das `tabindex`-Attribut ermöglicht Ihnen dies.

Hier ist die grundlegende Syntax:

```html
<element tabindex="number">Element Text</element>
```

Der Wert von `tabindex` bestimmt, wie sich das Element bei der Tastaturnavigation verhält:

`tabindex="0"` fügt das Element der natürlichen Tab-Reihenfolge hinzu.

Das Tabben bewegt den Fokus vom `button` zum `div` und dann zum Link, entsprechend ihrer Reihenfolge im HTML.

Klicken Sie irgendwo in den Leerraum des Vorschaufensters. Verwenden Sie dann die `Tab`-Taste, um zu sehen, wie der Fokus zwischen den verschiedenen Elementen wechselt. Um mit dem Beispiel zu interagieren, müssen Sie den interaktiven Editor aktivieren.

:::interactive_editor

```html
<button>First</button>
<div tabindex="0">Second</div>
<a href="#">Third</a>
```

:::

`tabindex="-1"` macht ein Element programmatisch fokussierbar. Das ist nützlich, um den Fokus in Elementen zu steuern, die normalerweise nicht fokussierbar sind, wie Überschriften, Container, Dialoge oder Fehlermeldungen:

```html
<p tabindex="-1">Sorry, there was an error with your submission.</p>
```

In diesem Beispiel ist der Absatz nicht Teil der normalen Tab-Reihenfolge, sodass Nutzerinnen und Nutzer ihn nicht durch Drücken der `Tab`-Taste erreichen können. Wenn Sie jedoch den Fokus per Skript auf dieses Element setzen, wird die Nachricht der Aufmerksamkeit der Nutzerin oder des Nutzers zugeführt. Diese Technik lernen Sie in den JavaScript-Lektionen genauer kennen.

Wenn der `tabindex`-Wert größer als `0` ist, wird eine benutzerdefinierte Tab-Reihenfolge festgelegt. Elemente mit niedrigeren positiven Werten werden zuerst fokussiert.

In diesem Beispiel wird beim Tabben zuerst der `input` mit `tabindex="1"` fokussiert, dann `2` und danach `3`, unabhängig von ihrer Reihenfolge im HTML.

```html
<input tabindex="2">
<input tabindex="1">
<input tabindex="3">
```

Benutzerdefinierte positive Werte werden manchmal in komplexen Widgets verwendet, etwa in einer Symbolleiste, in der eine bestimmte Navigationsreihenfolge gewünscht ist. Diese Vorgehensweise wird jedoch nicht empfohlen, da sie die Navigation verwirrend und schwer wartbar machen kann, besonders wenn die Seite wächst oder sich ändert.

`accesskey` ist ein weiteres Attribut, mit dem Sie Ihre Webprojekte tastaturzugänglich machen können. Es erlaubt Ihnen, eine Taste zu definieren, die ein bestimmtes Element fokussiert oder aktiviert.

Hier ist ein interaktives Beispiel, das Sie ausprobieren können, indem Sie den interaktiven Editor aktivieren und den folgenden Vorschlägen folgen:

- `accesskey="s"` weist die Taste `S` dem `Save`-Button zu. In den meisten Browsern wird durch Drücken von `ALT + S` (unter Windows) und `CTRL + Option + S` (auf dem Mac) dieser Button aktiviert.
- `accesskey="c"` setzt die Taste `C` für den `Cancel`-Button, sodass Nutzerinnen und Nutzer ihn mit `ALT + C` (Windows) und `CTRL + Option + C` (Mac) aktivieren können.
- `accesskey="h"` weist die Taste `H` dem `Home`-Link zu, sodass Nutzerinnen und Nutzer mit `ALT + H` (Windows) und `CTRL + Option + H` (Mac) zur Startseite navigieren können.

Bitte beachten Sie, dass die genaue Tastenkombination zum Aktivieren des `accesskey` je nach Browser und Betriebssystem variieren kann. Üblicherweise ist es `ALT + Specified Key` unter Windows und `CTRL + Option + Specified Key` auf dem Mac.

Das `accesskey`-Attribut kann problematisch sein, da es mit bestehenden Tastenkombinationen kollidieren oder sich auf verschiedenen Geräten und Tastaturen inkonsistent verhalten kann. Außerdem ist es für Nutzerinnen und Nutzer oft nicht intuitiv, diese Tastenkombinationen zu entdecken oder sich zu merken.

Wenn Sie sich entscheiden, `accesskey` zu verwenden, sollte dies mit klaren Anweisungen für die Nutzerinnen und Nutzer geschehen. In den meisten Fällen ist es jedoch besser, `accesskey` in Websites und Webanwendungen zu vermeiden.

:::interactive_editor

```html
<button accesskey="s">Save</button>
<button accesskey="c">Cancel</button>
<a href="index.html" accesskey="h">Home</a>
```

:::

Eine weitere Möglichkeit, die Tastaturzugänglichkeit in Ihren Anwendungen zu verbessern, besteht darin, klare Fokusindikatoren bereitzustellen. Wenn Sie den standardmäßigen Fokusindikator des Browsers für nicht ausreichend halten, können Sie ihn überschreiben, indem Sie den Fokuszustand des Elements gezielt ansprechen.

Hier ist ein Beispiel, wie Sie den Fokuszustand für ein `button`-Element stylen. Klicken Sie irgendwo in den Leerraum des Vorschaufensters und drücken Sie dann die `Tab`-Taste, um den Button zu fokussieren. Um mit dem Beispiel zu interagieren, müssen Sie den interaktiven Editor aktivieren.

:::interactive_editor

```html
<link href="styles.css" rel="stylesheet">

<button>Example button</button>
```

```css
button:focus {
  outline: 2px solid #005fcc;
}
```

:::

Die `outline`-Eigenschaft wird verwendet, um die Umrandung um das Element zu definieren. In diesem Beispiel wird die Umrandung als durchgezogene blaue Linie mit einer Dicke von 2 Pixeln gesetzt. Der Fokusindikator sollte so gestaltet sein, dass klar erkennbar ist, welches Element gerade den Fokus hat. Um barrierefrei zu sein, muss der Indikator einen Mindestfarbkontrast von mindestens 3:1 zum Hintergrund haben, den er überdeckt.

Sie sollten auch Tastaturfallen vermeiden, die auftreten, wenn Nutzerinnen und Nutzer den Fokus in Komponenten wie Modals und Popups nicht von einem bestimmten Element wegbewegen können.
