# --description--

Geolocation API는 웹사이트가 사용자의 위치를 요청할 수 있는 방법을 제공합니다. 개인정보 보호를 위해 사용자가 위치 접근 권한을 허용해야만 웹사이트에서 위치에 접근할 수 있다는 점을 꼭 기억하세요.

오늘 집중할 주요 메서드는 `getCurrentPosition`입니다. 이 메서드는 기기의 지리적 위치를 수집하는 데 사용됩니다. `getCurrentPosition`을 사용하는 예시는 다음과 같습니다:

```js
navigator.geolocation.getCurrentPosition(
  (position) => {
    console.log("Latitude: " + position.coords.latitude);
    console.log("Longitude: " + position.coords.longitude);
  },
  (error) => {
    console.log("Error: " + error.message);
  }
);
```

이 코드에서는 `getCurrentPosition`를 호출하고, 위치가 성공적으로 얻어졌을 때 호출될 함수를 전달합니다. 이 위치 객체에는 여러 정보가 포함되어 있지만, 여기서는 `latitude`과 `longitude`에만 집중합니다.

위치 정보를 얻는 데 문제가 있으면 오류가 콘솔에 기록됩니다.

`getCurrentPosition` 메서드는 기기와 설정에 따라 GPS, Wi-Fi 네트워크, 또는 IP 주소 기반 위치 정보를 사용합니다. 위치가 확인되면 성공 콜백 함수가 위치 객체와 함께 호출됩니다.

위치 객체에는 여러 프로퍼티가 포함되며, 가장 자주 사용되는 것은 `latitude`과 `longitude`이지만 `altitude`, `accuracy`, `speed`, `heading` 등도 포함될 수 있습니다.

지리 위치를 사용할 때 중요한 점은 사용자 개인정보 보호입니다. 사용자에게 위치 데이터가 왜 필요한지, 어떻게 사용할 것인지 설명하세요.
