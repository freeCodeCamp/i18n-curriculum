# --description--

アプリケーションを設計するとき、1つのことに責任を持つ小さな関数がいくつかあることがよくあります。これは「単一責任の原則」として知られています。小さな関数が複数ある場合、それらの関数が期待通りに動作することを確認するためにテストするのが最善です。このレッスンでは、人気のあるJestテストフレームワークを使ってユニットテストを作成する方法を見ていきます。

この例では、新しくフォーマットされた文字列を返す関数を作成します：

```js
export function getFormattedWord(str) {
  if (!str) return '';
  return str.charAt(0).toUpperCase() + str.slice(1);
}
```

この`getFormattedWord`関数には`str`というパラメータがあり、まず`str`が空かどうかをチェックします。空の場合は空の文字列を返します。そうでなければ、最初の文字が大文字になった新しい文字列を返します。この関数はテストファイルで使えるようにエクスポートしています。

別の`getFormattedWord.test.js`ファイルで、この関数が期待通りに動作しているかどうかを検証するテストを書けます。

`getFormattedWord.test.js`ファイルの先頭で、まずこのように関数をインポートする必要があります：

```js
import { getFormattedWord } from "./getFormattedWord.js";
```

次に、`npm i jest`を使ってJestパッケージをインストールします。それから、小文字で構成された単語`hello`が、最初の文字が大文字の`Hello`に正しくフォーマットされているかをチェックするテストを追加できます。ここで大文字は`H`です：

```js
test('capitalizes the first letter of a word', () => {
  expect(getFormattedWord('hello')).toBe('Hello');
});
```

`expect`関数は値をテストするために使われます。これはマッチャーと組み合わせて使います。マッチャーは値が期待通りに動作しているかをチェックする関数です。この場合、マッチャーは`toBe()`です。Jestには真偽値、文字列、数値などをテストするためのさまざまなマッチャーがあります。

テストを実行する1つの方法は、`package.json`ファイルにnpmスクリプトを追加することです。Jestフレームワーク用の例を示します：

```js
"scripts": {
  "test": "jest"
},
```

あとはターミナルで`npm run test`を実行すればテストが走ります。テストが通れば、ターミナルに次のような表示が出ます：

```sh
PASS ./getFormattedWord.test.js
  ✓ capitalizes the first letter of a word (1 ms)

Test Suites: 1 passed, 1 total
Tests:       1 passed, 1 total
Snapshots:   0 total
Time:        0.103 s, estimated 1 s
Ran all test suites.
```

失敗したテストがどのように見えるかを確認するために、意図的に関数を次のように変更してテストを壊してみましょう：

```js
export function getFormattedWord(str) {
  if (!str) return '';
  return "This is incorrect";
}
```

`npm run test`コマンドを実行すると、テストが異なる結果を期待していたためエラーメッセージが表示されます：

```sh
FAIL ./getFormattedWord.test.js
  ✕ capitalizes the first letter of a word (1 ms)

  ● capitalizes the first letter of a word

  expect(received).toBe(expected) // Object.is equality

  Expected: "Hello"
  Received: "This is incorrect"

    2 |
    3 | test('capitalizes the first letter of a word', () => {
    4 |   expect(getFormattedWord('hello')).toBe('Hello');
                                            ^
    5 | });
    6 |

    at Object.toBe (getFormattedWord.test.js:4:37)

Test Suites: 1 failed, 1 total
Tests:       1 failed, 1 total
Snapshots:   0 total
Time:        0.121 s, estimated 1 s
Ran all test suites.
```

ここで関数を元の状態に戻します：

```js
export function getFormattedWord(str) {
  if (!str) return '';
  return str.charAt(0).toUpperCase() + str.slice(1);
}
```

`getFormattedWord`に引数が渡されなかった場合のテストも追加しましょう：

```js
test("returns an empty string when no argument is provided", () => {
  expect(getFormattedWord()).toBe("");
});
```

より堅牢にするために他のテストも追加できますが、最初の2つのテストはユニットテストの良い導入になります。

JavaScriptアプリケーションのテストでよく使われるフレームワークにはJest、Mocha、Vitestがあります。

ユニットテストは、プログラムのバグをより多く見つけられ、すべてが期待通りに動作していることを保証するために重要です。また、コードの期待される動作を表すドキュメントの役割も果たします。
