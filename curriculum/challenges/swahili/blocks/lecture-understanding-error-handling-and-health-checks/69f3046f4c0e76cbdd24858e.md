# --description--

Je, umewahi kujiuliza nini hutokea wakati kitu kinapokosea katika programu yako ya Express?

Je, basi kama muunganisho wa hifadhidata unashindwa? Au mtumizi anapotembelea njia ya data ambayo haipo?

Kuelewa jinsi mshughulikiaji wa hitilafu unavyofanya kazi katika Express ni muhimu ikiwa unataka kujenga programu za mtandao salama na za kuaminika.

Katika funzo hili, tutapitia jinsi Express inavyoshughulikia hitilafu, na jinsi unavyoweza kuzizuia na kutoa majibu kwa kutumia mifano halisi ya msimbo.

## Jinsi ya kushughulikia hitilafu rahisi katika Express

Katika Express, mshughulikiaji wa hitilafu umejengwa juu ya programu ya kati.

Kama vile kazi za kawaida za programu ya kati, mshughulikiaji wa hitilafu ana upatikanaji wa `req`, `res`, na `next`. Lakini una kigezo kimoja cha ziada mwanzoni: `err`.

Huu ndio muundo wa sintaksia:

```js
app.use((err, req, res, next) => {
  console.error(err.message);
  res.status(500).send("Internal Server Error");
});
```

Unaeleza programu hii ya kati mwishoni mwa safu yako ya programu ya kati.

Express inajua ni mshughulikiaji wa hitilafu kwa sababu ina vigezo vinne — kipya, `err`, pamoja na vigezo vya kawaida vya `req`, `res`, na `next` kwa kazi nyingine za programu ya kati.

Katika funzo lijalo, utajifunza zaidi kuhusu programu ya kati ya mshughulikiaji wa hitilafu na jinsi inavyolingana na safu pana ya programu ya kati ya Express.

## Jinsi ya kushughulikia hitilafu zisizo za wakati mmoja

Kushughulikia hitilafu zisizo za wakati mmoja kunategemea toleo la Express unalotumia.

Kumbuka, bila kujali toleo la Express unalotumia, bado unahitaji kueleza programu yako ya kati ya mshughulikiaji wa hitilafu mwishoni mwa safu yako, baada ya njia zako zote za data.

Huyu ndiye mshughulikiaji wa hitilafu tena:

```js
// App routes
// ...

// Error-handling middleware
app.use((err, req, res, next) => {
  console.error(err.message);
  res.status(500).send("Internal Server Error");
});
```

## Express 4

Katika Express 4 na chini, ambayo miradi mingi bado inaitumia, hitilafu zisizo za wakati mmoja **hazikamatwi moja kwa moja**.

Ikiwa unatumia `async`/`await` na kitu kinatupa ndani ya mshughulikiaji wa njia ya data ya `async`, Express haitakamata isipokuwa uzipe kwa mkono `next()`.

Huu ndio jinsi unavyoshughulikia hitilafu zisizo za wakati mmoja ipasavyo katika Express 4:

```js
app.get("/user", async (req, res, next) => {
  try {
    const user = await getUserFromDatabase();
    res.send(user);
  } catch (err) {
    next(err); // Manually pass the error to the error handler
  }
});
```

Kama utasahau kipande cha `try/catch` na `next(err)`, hitilafu inaweza:

* Kusababisha programu yako kuanguka
* Au kusababisha onyo la ahadi isiyoshughulikiwa

Ndiyo maana kufunika mantiki isiyo ya wakati mmoja katika `try`/`catch` ni muhimu katika Express 4.

Baadhi ya wasanidi programu hata huunda kazi ya msaada ili kuepuka kurudia `try/catch` kila mahali:

```js
function asyncHandler(fn) {
  return function (req, res, next) {
    Promise.resolve(fn(req, res, next)).catch(next);
  };
}
```

Kisha uitumie hivi:

```js
app.get("/user", asyncHandler(async (req, res) => {
  const user = await getUserFromDatabase();
  res.send(user);
}));
```

Hii inafanya usihitaji kutumia vipande vya `try/catch` katika njia zako zote za `async` huku bado ukisafirisha hitilafu ipasavyo.

## Express 5

Toleo la sasa la Express limeboresha tabia hii.

Kuanzia Express 5, ikiwa njia ya data ya `async` itatupa hitilafu au kurudisha ahadi iliyokataliwa, Express huisafirisha moja kwa moja kwa programu ya kati ya mshughulikiaji wa hitilafu.

Hii inamaanisha mfano huu unashughulikia hitilafu bila `try/catch`:

```js
app.get("/user", async (req, res) => {
  const user = await getUserFromDatabase();
  res.send(user);
});
```

Ikiwa `getUserFromDatabase()` itatupa hitilafu, Express 5 itakamata na kuipatia mshughulikiaji wa hitilafu kwa njia ya moja kwa moja.

Hakuna wito wa mkono wa `next(err)` unahitajika.

## Jinsi ya kushughulikia hitilafu za 404

Hitilafu za 404 ni tofauti kidogo. Hutokea wakati hakuna njia ya data inayolingana na ombi.

Unaweza kuzishughulikia kwa programu hii ya kati:

```js
app.use((req, res, next) => {
  res.status(404).send("Sorry, that route does not exist.");
});
```

**Muhimu**: Hii inapaswa kuwekwa baada ya njia zako za data, lakini kabla ya programu ya kati ya mshughulikiaji wa hitilafu.

Na hatimaye, hizi ni mbinu bora za kushughulikia hitilafu katika Express:

* Daima weka programu yako ya kati ya mshughulikiaji wa hitilafu mwishoni.
* Shughulikia hitilafu za 404 tofauti.
* Tumia `next(err)` kusukuma hitilafu kutoka njia au mantiki ya `async` katika Express 4 na chini.

Katika uzalishaji, usitume mfuatano wa makosa kwa watumiaji. Andika maelezo ndani na rudisha ujumbe rafiki kwa mtumiaji badala yake.

Kushughulikia hitilafu katika Express ni kuhusu kuanzisha programu ya kati inayosikiliza matatizo na kutoa majibu kwa njia wazi na salama. Iwe ni hitilafu katika msimbo wako au njia iliyokosekana, programu yako haipaswi kamwe kuanguka bila mpango wa dharura.
