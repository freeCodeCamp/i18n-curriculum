# --description--

Wenn Ihre Anwendung wächst, kann das Verwalten des Datenflusses zwischen Komponenten komplex werden.

Am Anfang reicht vielleicht der React-`useState`-Hook aus, aber mit zunehmenden Funktionen können Probleme auftreten bei:

- Dem Weitergeben von Props durch Komponenten, die sie nicht benötigen, auch bekannt als Prop Drilling
- Dem Synchronhalten von Daten in verschiedenen Teilen Ihrer Anwendung
- Dem Umgang mit komplexen Aktualisierungen, die mehrere Komponenten gleichzeitig betreffen

Diese und andere Herausforderungen können entstehen, was zu einem Codebestand führt, der schwerer zu warten, zu debuggen und zu testen ist. Hier kommen State-Management-Bibliotheken ins Spiel – sie bieten einen zentralen Ort, an dem Komponenten die benötigten Daten abrufen oder aktualisieren können.

Schauen wir uns einige verschiedene State-Management-Optionen an und wann Sie sie verwenden sollten.

Die Context API ist ein in React integrierter State-Manager, mit dem Sie State über Komponenten hinweg teilen können, ohne eine Drittanbieter-Bibliothek zu verwenden. Sie ist eine bewährte Verbesserung gegenüber dem `useState`-Hook und eignet sich daher perfekt für Fälle wie das Umschalten von Themes oder den Status der Benutzer-Authentifizierung.

Die Context API ist jedoch nicht gut geeignet für häufige Aktualisierungen und kann unnötige Re-Renders verursachen, weshalb sie weniger passend für komplexe State-Anforderungen in Anwendungen wie E-Commerce- und Social-Media-Plattformen ist.

Hier ist eine Counter-Komponente, die die grundlegende Verwendung der Context API demonstriert:

```jsx
import { useState, createContext } from 'react';

const CounterContext = createContext();

const CounterProvider = ({ children }) => {
  const [count, setCount] = useState(0);

  return (
    <CounterContext.Provider value={{ count, setCount }}>
      {children}
    </CounterContext.Provider>
  );
};

export { CounterContext, CounterProvider };
```

Dieser Code erstellt einen Context und einen Provider, um einen `count`-State in der gesamten Anwendung zu teilen.

`CounterProvider` verwendet den `useState`-Hook, um den `count`-State und dessen Setter zu initialisieren und zu verwalten. Beide werden dann über den `Provider` an Kindkomponenten weitergegeben.

Wenn Sie also Ihre gesamte Anwendung mit dem `CounterProvider` umschließen, ist der `count`-State überall in Ihrer Anwendung verfügbar.

So können Sie `CounterProvider` um Ihre Anwendung herum einwickeln:

```jsx
import { CounterProvider } from './context/CounterContext';

function App() {
  return (
    <CounterProvider>
        {/* App components */}
    </CounterProvider>
  );
}

export default App;
```

Und so können Sie den `count`-State verwenden:

```jsx
import React, { useContext } from 'react';
import { CounterContext } from '../context/CounterContext';

const Counter = () => {
  const { count, setCount } = useContext(CounterContext);

  return (
    <>
      <div style={{ textAlign: 'center' }}>
        <h1>Context API Counter</h1>
        <button style={{ marginRight: '5px' }} onClick={() => setCount(count - 1)}>
          Decrease
        </button>
        <span>{count}</span>
        <button style={{ marginLeft: '5px' }} onClick={() => setCount(count + 1)}>
          Increase
        </button>
      </div>
    </>
  );
};

export default Counter;
```

Wie Sie sehen, werden der `count`-State und seine Setter-Funktion `setCount` über die `useContext`-Funktion initialisiert.

Der aktuelle `count`-State wird dann angezeigt, und `setCount` wird verwendet, um den `count`-State zu erhöhen oder zu verringern, wenn der Benutzer die jeweiligen Buttons zum Dekrementieren und Inkrementieren klickt.

Eine weitere beliebte State-Management-Bibliothek ist Redux, eine der am häufigsten verwendeten Bibliotheken mit React. Sie gibt es schon lange und sie eignet sich ideal für größere Anwendungen wie E-Commerce- und Social-Media-Plattformen, Foren und so weiter.

Redux verwaltet den State, indem es einen zentralen Store und strenge Kontrolle über State-Aktualisierungen bereitstellt. Es verwendet ein vorhersehbares Muster mit Actions, Reducern und Middleware.

Actions sind Nutzlasten von Informationen, die Daten von Ihrer Anwendung an den Redux-Store senden, oft ausgelöst durch Benutzerinteraktionen.

Reducer sind Funktionen, die festlegen, wie sich der State als Reaktion auf diese Actions ändern soll, und sorgen dafür, dass der State unveränderlich aktualisiert wird.

Middleware fungiert als Brücke zwischen dem Dispatchen von Actions und dem Reducer und ermöglicht es Ihnen, die Funktionalität von Redux zu erweitern (zum Beispiel Logging, Umgang mit asynchronen Operationen), ohne den Kernablauf zu verändern.

Die häufigste Kritik an Redux betrifft den umfangreichen Boilerplate-Code, der zum Einstieg nötig ist. Als Antwort darauf hat das Redux-Team „Redux Toolkit“ und „RTK Query“ eingeführt, die den Einrichtungsprozess deutlich vereinfachen.

Normalerweise definieren Sie sowohl Actions als auch Reducer in einer einzigen Datei mit der `createSlice()`-Funktion. Es ist üblich, die Datei so zu benennen, dass sie mit dem Wort `Slice` endet, zum Beispiel `productSlice`, `userSlice`, `counterSlice` und so weiter.

Hier ist eine `counterSlice`-Datei, die Ihnen die Grundlagen zeigt:

```jsx
import { createSlice } from '@reduxjs/toolkit';

const counterSlice = createSlice({
  name: 'counter',

  initialState: { count: 0 },

  reducers: {
    increment: (state) => {
      state.count += 1;
    },
    decrement: (state) => {
      state.count -= 1;
    },
  },
});

export const { increment, decrement } = counterSlice.actions;

export default counterSlice.reducer;
```

Von hier aus müssen Sie dann die gesamte App mit dem `Provider` umschließen, ein Stück State aus dem Slice mit `useSelector()` auswählen und dann `useDispatch()` verwenden, um den State aktiv zu machen.

Eine weitere Option, die Sie in Betracht ziehen können, ist Zustand.

Zustand ist eine leichtgewichtige State-Management-Bibliothek mit einer einfachen API. Sie basiert auf Hooks, wodurch weniger Boilerplate im Vergleich zu Redux anfällt und die Einrichtung einfacher und schneller ist.

Zustand eignet sich ideal für kleine bis mittelgroße Anwendungen. Es funktioniert, indem es einen `useStore`-Hook verwendet, um direkt in Komponenten und Seiten auf den State zuzugreifen. So können Sie Daten ändern und abrufen, ohne Actions, Reducer oder einen Provider zu benötigen.

Hier ist ein `useCounterStore`, der eine weitere Counter-Funktionalität implementiert:

```jsx
import { create } from 'zustand';

const useCounterStore = create((set) => ({
  count: 0,
  increment: () => set((state) => ({ count: state.count + 1 })),
  decrement: () => set((state) => ({ count: state.count - 1 })),
}));

export default useCounterStore;
```

Und so initialisieren und verwenden Sie die States in Ihrer Anwendung:

```jsx
// Import the useCounterStore (it's just a hook)
import useCounterStore from '../useCounterStore';

const Counter = () => {
  // Initialize the states with the useCounterStore hook
  const { count, increment, decrement } = useCounterStore();

  return (
    <>
      <div style={{ textAlign: 'center' }}>
        <h1>Zustand Counter</h1>
        <button style={{ marginRight: '5px' }} onClick={() => decrement()}>
          Decrease
        </button>
        <span>{count}</span>
        <button style={{ marginLeft: '5px' }} onClick={() => increment()}>
          Increase
        </button>
      </div>
    </>
  );
};

export default Counter;
```

Obwohl sich das Frontend-Ökosystem ständig weiterentwickelt und regelmäßig neue State-Management-Bibliotheken entstehen, sind die hier besprochenen weit verbreitet in der Branche.
