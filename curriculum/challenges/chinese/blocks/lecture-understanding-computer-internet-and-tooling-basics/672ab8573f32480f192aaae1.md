# --description--

专业开发者依赖各种工具来提高生产力和代码质量。让我们了解这些工具，包括那些看起来非常明显的工具。

工具中最重要的是计算机。计算机是主要的开发环境。它可以是运行 Windows、macOS 或 Linux 操作系统的台式机，也可以是可移植的笔记本电脑。

专业开发者通常选择具有快速处理能力和大量随机存取存储器的计算机，以处理资源密集型任务。

计算机上有一个用于直接编写和编辑代码的程序。这就是代码编辑器或集成开发环境（IDE）。

像 Visual Studio、IntelliJ IDEA、JetBrains 和 PyCharm 这样的集成开发环境提供强大的特色，如代码补全、调试和集成终端支持。

Visual Studio Code（VS Code）本质上是一个代码编辑器，但它还通过其丰富的扩展库提供这些功能。

其他代码编辑器有 Sublime Text、Atom、Notepad++ 和 Brackets。

当你使用代码编辑器和集成开发环境编写代码时，你需要跟踪对它们所做的更改。让你跟踪这些更改的工具是版本控制系统。

Git 是开发社区中使用最广泛的版本控制系统。

像 GitHub、GitLab 和 Bitbucket 这样的平台为你的 Git 仓库提供基于云的托管服务。这些平台启用与其他开发者的协作，允许你在多个分支上工作，并促进代码更改的合并。

包管理器是另一个关键工具。它们帮助开发者简化添加、更新和移除库及项目依赖的进程。

流行的包管理器示例有：

* JavaScript 的 NPM、Yarn 和 PNPM
* Python 的 PIP
* PHP 的 Composer
* Java 的 Maven

在使用不同工具编写代码后，开发者会测试该代码以确保其按预期工作。

为此，开发者使用像 Cypress、Playwright、Selenium 和其他的测试框架。还有一些特定语言的测试框架。示例有：

* JavaScript 的 Jest
* Python 的 pytest
* Java 的 JUnit
* PHP 的 PHPUnit

你不仅需要测试代码以确保它按预期工作。你还需要测试代码在最终用户视觉上的呈现效果。开发者使用网页浏览器来完成这项工作。

现代浏览器如 Chrome、Firefox、Edge 和 Safari 提供用于检查超文本标记语言、CSS 和 JavaScript 代码的开发者工具。还有用于调试和性能分析的工具。

这些工具帮助开发者在不同浏览器中测试和优化他们的网页应用，以便最终用户使用。
