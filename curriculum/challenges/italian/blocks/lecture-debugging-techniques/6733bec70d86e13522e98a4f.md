# --description--

L’istruzione `throw` in JavaScript serve a lanciare un’eccezione definita dall’utente. Un’eccezione in programmazione è quando si verifica un evento imprevisto che interrompe il flusso normale del programma.

Come programmatori è importante gestire queste eccezioni, così i tuoi programmi non si bloccano inaspettatamente quando si verificano errori. Ecco la sintassi base per l’istruzione `throw`:

```js
throw expression;
```

Il `expression` in questo caso sarebbe l’oggetto o il valore che rappresenta l’eccezione che vuoi lanciare. Esempi di questo sono le classi di eccezioni integrate come la classe `Error`, `TypeError` o `RangeError`. Ecco un esempio di utilizzo dell’istruzione `throw` per lanciare un `TypeError`:

```js
function validateNumber(input) {
  if (typeof input !== "number") {
    throw new TypeError("Expected a number, but received " + typeof input);
  }
  return input * 2;
}
```

In questo esempio, controlliamo se il tipo di `input` non è di tipo `number`. Se non lo è, allora lanciamo un `TypeError` con un messaggio personalizzato. Altrimenti, la funzione restituirà il risultato della moltiplicazione di `input` per `2`.

Se vuoi lanciare un messaggio di errore più generico, puoi fare riferimento al costruttore `Error` in questo modo:

```js
function divide(numerator, denominator) {
  if (denominator === 0) {
    throw new Error("Cannot divide by zero");
  }
  return numerator / denominator;
}
```

Ecco un esempio di funzione che verifica se il denominatore è `0`. Se è così, lancerà un messaggio di errore personalizzato che dice `Cannot divide by zero`.

Nella prossima lezione vedremo come lanciare messaggi di errore nel contesto del blocco `try`/`catch`, che serve a gestire con eleganza le eccezioni in JavaScript.
