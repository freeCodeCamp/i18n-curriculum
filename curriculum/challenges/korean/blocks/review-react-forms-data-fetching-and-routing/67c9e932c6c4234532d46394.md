## React에서 폼 다루기

- **제어된 입력**: 입력 필드 값을 상태에 저장하고 `onChange` 이벤트를 통해 업데이트하는 경우입니다. 이렇게 하면 폼 데이터를 완전히 제어할 수 있고 즉각적인 유효성 검사와 조건부 렌더링이 가능합니다.

```jsx
import { useState } from "react";

function App() {
  const [name, setName] = useState("");

  const handleChange = (e) => {
    setName(e.target.value);
  };

  const handleSubmit = (e) => {
    e.preventDefault();
    console.log(name);
  };

  return (
    <>
      <form onSubmit={handleSubmit}>
        <label htmlFor="name">Your name</label> <br />
        <input value={name} id="name" onChange={handleChange} type="text" />
        <button type="submit">Submit</button>
      </form>
    </>
  );
}

export default App;
```

- **비제어 입력**: `useState` 훅을 통해 입력을 처리하는 대신, HTML의 비제어 입력은 DOM의 도움으로 자체 내부 상태를 유지합니다. DOM이 입력 값을 제어하기 때문에 `ref`를 사용해 입력 필드 값을 가져와야 합니다.

```jsx
import { useRef } from "react";

function App() {
 const nameRef = useRef();

 const handleSubmit = (e) => {
   e.preventDefault();
   console.log(nameRef.current.value);
 };

 return (
   <form onSubmit={handleSubmit}>
     <label htmlFor="name">Your</label>{" "}
     <input type="text" ref={nameRef} id="name" />
     <button type="submit">Submit</button>
   </form>
 );
}

export default App;
```

## `useActionState` 훅 다루기

- **서버 액션**: API 엔드포인트 없이 서버에서 바로 폼 처리를 할 수 있도록 서버에서 실행되는 함수입니다. 다음은 Next.js 애플리케이션의 예시입니다:

```js
"use server";

async function submitForm(formData) {
 const name = formData.get("name");
 return { message: `Hello, ${name}!` };
}
```

`"user server"` 지시어는 함수를 서버 액션으로 표시합니다.

- **`useActionState` 훅**: 이 훅은 폼 제출 결과에 따라 상태를 업데이트합니다. `useActionState` 훅의 기본 구문은 다음과 같습니다:

```js
const [state, action, isPending] = useActionState(actionFunction, initialState, permalink);
```

- `state`은 액션이 반환하는 현재 상태입니다.
- `action`은 서버 액션을 트리거하는 함수입니다.
- `isPending`은 액션이 현재 실행 중인지 여부를 나타내는 불리언입니다.
- `actionFunction` 매개변수는 서버 액션 자체입니다.
- `initialState`은 액션 실행 전 상태의 시작점을 나타내는 매개변수입니다.
- `permalink`은 폼이 수정하는 고유한 페이지 URL을 포함하는 선택적 문자열입니다.

## React에서 데이터 가져오기

- **데이터 가져오기 옵션**: React에서 데이터를 가져오는 방법은 다양합니다. 기본 Fetch API를 사용하거나 Axios, SWR 같은 서드파티 도구를 사용할 수 있습니다.
- **데이터 가져올 때 자주 사용하는 상태 변수**: 어떤 방법을 사용하든 React에서 데이터를 가져올 때는 몇 가지 상태를 추적해야 합니다. 첫째는 데이터 자체입니다. 둘째는 데이터가 아직 가져오는 중인지 추적하는 상태입니다. 셋째는 데이터 가져오기 과정에서 발생할 수 있는 오류를 포착하는 상태 변수입니다.

```js
const [data, setData] = useState(null);
const [loading, setLoading] = useState(true);
const [error, setError] = useState(null);
```

데이터 가져오기는 부수 효과이므로 `Fetch API`를 `useEffect` 훅 안에서 사용하는 것이 좋습니다.

```js
useEffect(() => {
  const fetchData = async () => {
    try {
      const res = await fetch("https://jsonplaceholder.typicode.com/posts");
      
      if (!res.ok) {
        throw new Error("Network response was not ok");
      }

      const data = await res.json();
      setData(data);
    } catch (err) {
      setError(err);
    } finally {
      setLoading(false);
    }
  };

  fetchData();
}, []);
```

그다음 데이터 가져오기가 완료되지 않았으면 로딩 메시지를, 오류가 있으면 오류 메시지를, 아니면 결과를 렌더링할 수 있습니다.

```jsx
if (loading) {
  return <p>Loading...</p>;
}

if (error) {
  return <p>{error.message}</p>;
}

return (
  <ul>
    {data.map((post) => (
      <li key={post.id}>{post.title}</li>
    ))}
  </ul>
);
```

Axios를 사용하려면 설치하고 가져와야 합니다:

```bash
npm i axios
```

```js
import axios from "axios";
```

그다음 `axios.get`를 사용해 데이터를 가져올 수 있습니다:

```jsx
const [data, setData] = useState(null);
const [loading, setLoading] = useState(true);
const [error, setError] = useState(null);

useEffect(() => {
  const fetchData = async () => {
    try {
      const res = await axios.get(
        "https://jsonplaceholder.typicode.com/users"
      );
      setData(res.data);
    } catch (err) {
      setError(err);
    } finally {
      setLoading(false);
    }
  };

  fetchData();
}, []);
```

`useSWR` 훅을 사용해 데이터를 가져오려면 먼저 설치하고 가져와야 합니다.

```bash
npm i swr
```

```js
import useSWR from "swr";
```

다음은 훅을 사용해 데이터를 가져오는 방법입니다:

```jsx
import useSWR from "swr";

const fetcher = (url) => fetch(url).then((res) => res.json());

const FetchTodos = () => {
 const { data, error } = useSWR(
   "https://jsonplaceholder.typicode.com/todos",
   fetcher
 );

 if (!data) {
   return <h2>Loading...</h2>;
 }
 if (error) {
   return <h2>Error: {error.message}</h2>;
 }

 return (
   <>
     <h2>Todos</h2>
     <div>
       {data.map((todo) => (
         <h3 key={todo.id}>{todo.title}</h3>
       ))}
     </div>
   </>
 );
};

export default FetchTodos;
```

## `useOptimistic` 훅 다루기

- **`useOptimistic` 훅**: 이 훅은 비동기 작업이 백그라운드에서 완료될 때까지 UI를 반응형으로 유지하는 데 사용합니다. 서버 응답을 기다리는 것처럼 예상 결과를 기반으로 UI에 즉각적인 업데이트를 제공하는 "낙관적 업데이트" 전략을 관리하는 데 도움을 줍니다.

기본 구문은 다음과 같습니다:

```js
const [optimisticState, addOptimistic] = useOptimistic(actualState, updateFunction);
```

- `optimisticState`는 더 나은 사용자 경험을 위해 즉시 업데이트되는 임시 상태입니다.
- `addOptimistic`은 실제 상태가 변경되기 전에 낙관적 업데이트를 적용하는 함수입니다.
- `actualState`은 서버에서 데이터를 가져오는 등 액션 결과로부터 오는 실제 상태 값입니다.
- `updateFunction`는 호출 시 낙관적 상태가 어떻게 업데이트될지 결정하는 함수입니다.

다음은 `useOptimistic` 훅을 `TaskList` 컴포넌트에서 사용하는 예시입니다:

```jsx
"use client";

import { useOptimistic } from "react";

export default function TaskList({ tasks, addTask }) {
  const [optimisticTasks, addOptimisticTask] = useOptimistic(
    tasks,
    (state, newTask) => [...state, { text: newTask, pending: true }]
  );

  async function handleSubmit(e) {
    e.preventDefault();
    const formData = new FormData(e.target);

    addOptimisticTask(formData.get("task"));

    addTask(formData);
    e.target.reset();
  }

  return <>{/* UI */}</>;
}
```

- **`startTransition`**: UI 일부를 렌더링하고 상태 업데이트를 비긴급 전환으로 표시하는 데 사용합니다. 이렇게 하면 비용이 많이 드는 업데이트 중에도 UI가 반응형으로 유지됩니다. 기본 구문은 다음과 같습니다:

```js
startTransition(action);
```

`action`는 상태 업데이트를 수행하거나 전환 관련 로직을 트리거합니다. 이를 통해 타이핑이나 클릭 같은 긴급 UI 업데이트가 차단되지 않도록 보장합니다.

## `useMemo` 훅 다루기

- **메모이제이션**: 특정 인수를 기반으로 비용이 많이 드는 함수 호출 결과를 캐시(기억)하는 최적화 기법입니다. 동일한 인수가 다시 제공되면 함수 재계산 대신 캐시된 결과를 반환합니다.
- **`useMemo` 훅**: 계산된 값을 메모이제이션하는 데 사용합니다. 다음은 큰 배열을 정렬한 결과를 메모이제이션하는 예시입니다. `expensiveSortFunction`는 `largeArray`이 변경될 때만 실행됩니다:

```js
const memoizedSortedArray = useMemo(
  () => expensiveSortFunction(largeArray),
  [largeArray]
);
```

## `useCallback` 훅 다루기

- **`useCallback` 훅**: 함수 참조를 메모이제이션하는 데 사용합니다.

```js
const handleClick = useCallback(() => {
  // code goes here
}, [dependency]);
```

- **`React.memo`**: 프로프가 변경되지 않았을 때 불필요한 재렌더링을 방지하기 위해 컴포넌트를 메모이제이션하는 데 사용합니다.

```jsx
const MemoizedComponent = React.memo(({ prop }) => {
 return (
   <>
     {/* Presentation */}
   </>
 )
});
```

## 의존성 관리 도구

- **의존성 정의**: 소프트웨어에서 의존성은 애플리케이션 내 한 컴포넌트나 모듈이 제대로 작동하려면 다른 컴포넌트나 모듈에 의존하는 경우를 말합니다. 의존성은 개발자가 다른 사람이 만든 미리 만들어진 함수나 도구를 사용할 수 있게 해주기 때문에 소프트웨어 애플리케이션에서 흔합니다. React 프로젝트에 필요한 두 가지 핵심 의존성은 `react`와 `react-dom` 패키지입니다:

```json
"dependencies": {
  "react": "^18.3.1",
  "react-dom": "^18.3.1"
}
```

- **패키지 관리자 정의**: 프로젝트에서 소프트웨어 의존성을 관리하려면 패키지 관리자를 사용해야 합니다. 패키지 관리자는 의존성 설치, 업데이트, 제거에 사용하는 도구입니다. JavaScript, Python, Ruby, Java 같은 인기 프로그래밍 언어는 모두 패키지 관리자를 사용합니다. JavaScript용 인기 패키지 관리자로는 npm, Yarn, pnpm이 있습니다.
- **`package.json` 파일**: 프로젝트에 대한 메타데이터(이름, 버전, 의존성 등)를 포함하는 핵심 구성 파일입니다. 또한 스크립트, 라이선스 정보, 기타 설정을 정의해 프로젝트와 의존성 관리를 돕습니다.
- **`package-lock.json` 파일**: 프로젝트에서 사용하는 모든 패키지의 정확한 버전을 고정하는 파일입니다. 패키지를 업데이트하면 잠금 파일도 새 버전으로 업데이트됩니다.
- **`node_modules` 폴더**: `package.json` 파일에 나열된 의존성의 실제 코드를 포함하는 폴더로, 프로젝트의 직접 의존성과 그 의존성의 의존성 모두를 포함합니다.
- **개발 의존성**: 개발 중에만 사용하고 프로덕션에서는 사용하지 않는 패키지입니다. 예를 들어 Jest 같은 테스트 라이브러리가 있습니다. Jest는 애플리케이션을 로컬에서 테스트할 때 필요하지만 프로덕션 실행에는 필요 없으므로 개발 의존성으로 설치합니다.

```json
"devDependencies": {
  "@eslint/js": "^9.17.0",
  "@types/react": "^18.3.18",
  "@types/react-dom": "^18.3.5",
  "@vitejs/plugin-react": "^4.3.4",
  "eslint": "^9.17.0",
  "eslint-plugin-react": "^7.37.2",
  "eslint-plugin-react-hooks": "^5.0.0",
  "eslint-plugin-react-refresh": "^0.4.16",
  "globals": "^15.14.0",
  "vite": "^6.0.5"
}
```

## React 라우터

- **소개**: React Router는 React 애플리케이션에 라우팅을 추가할 수 있게 해주는 서드파티 라이브러리입니다. 시작하려면 기존 React 프로젝트에 React Router를 이렇게 설치하세요:

```bash
npm i react-router
```

그다음 `main.jsx` 또는 `index.jsx` 파일 안에서 라우트 구조를 이렇게 설정해야 합니다:

```jsx
import { StrictMode } from "react";
import { createRoot } from "react-dom/client";
import { BrowserRouter, Routes, Route } from "react-router";
import App from "./App.jsx";

import "./index.css";

createRoot(document.getElementById("root")).render(
  <StrictMode>
    <BrowserRouter>
      <Routes>
        <Route path="/" element={<App />} />
      </Routes>
    </BrowserRouter>
  </StrictMode>
);
```

`path`와 `element`는 URL과 UI 컴포넌트를 연결하는 데 사용됩니다. 여기서는 홈페이지 라우트를 `App` 컴포넌트로 설정하고 있습니다.

- **여러 뷰와 라우트 설정**: 큰 애플리케이션에서는 여러 뷰와 라우트를 이렇게 설정하는 것이 일반적입니다:

```jsx
<Routes>
  <Route index element={<Home />} />
  <Route path="about" element={<About />} />

  <Route path="products">
    <Route index element={<ProductsHome />} />
    <Route path=":category" element={<Category />} />
    <Route path=":category/:productId" element={<ProductDetail />} />
    <Route path="trending" element={<Trending />} />
  </Route>
</Routes>
```

이 예시에서 `index` 프로프는 주어진 경로 세그먼트의 기본 라우트를 나타냅니다. 그래서 `Home` 컴포넌트는 루트 `/` 경로에서, `ProductsHome` 컴포넌트는 `/products` 경로에서 보여집니다.

- **라우트 중첩**: 라우트를 다른 라우트 안에 중첩할 수 있으며, 이 경우 자식 라우트 경로가 부모 라우트 경로에 덧붙여집니다.

```jsx
<Route path="products">
  <Route path="trending" element={<Trending />} />
</Route>
```

위 예시에서 트렌딩 상품 경로는 products/trending가 됩니다.

- **동적 세그먼트**: URL 경로의 일부가 동적인 경우를 말합니다.

```jsx
<Route path=":category" element={<Category />} />
```

이 예시에는 `category`라는 동적 세그먼트가 있습니다. 사용자가 products/brass-instruments 같은 URL로 이동하면 뷰가 `Category` 컴포넌트로 바뀌고 세그먼트에 따라 적절한 데이터를 동적으로 가져올 수 있습니다.

- **`useParams` 훅**: URL 경로에서 동적 매개변수에 접근하는 데 사용합니다.

```jsx
import { useParams } from "react-router";

export default function Category() {
  let params = useParams();
  {/* Accessing the category param: params.category */}
  {/* rest of code goes here */}
}
```

## React 프레임워크

- **소개**: React 프레임워크는 라우팅, 이미지 최적화, 데이터 가져오기, 인증 등 기능을 제공합니다. 이로 인해 특정 사용 사례에서는 별도의 프론트엔드와 백엔드 애플리케이션을 설정할 필요가 없을 수 있습니다. React 프레임워크 예로는 Next.js와 Remix가 있습니다.
- **Next.js 라우팅**: 이 라우팅 시스템은 동적 라우트, 병렬 라우트, 라우트 핸들러, 리다이렉트, 국제화 등을 지원합니다.

다음은 커스텀 요청 핸들러를 만드는 예시입니다:

```js
export async function GET() {
  const res = await fetch("https://example-api.com");
  const data = await res.json();

  return Response.json({ data });
}
```

- **Next.js 이미지 최적화**: `Image` 컴포넌트는 기본 HTML `img` 요소를 확장해 페이지 로드 속도와 크기 최적화를 가능하게 합니다. 즉, 이미지는 뷰포트에 들어올 때만 로드되고 `Image` 컴포넌트는 각 기기에 맞는 크기의 이미지를 자동으로 제공합니다.

```jsx
import Image from "next/image";

export default function Page() {
  return (
    <Image src="link-to-image-goes-here" alt="descriptive-title-goes-here" />
  );
}
```

## 프로프 드릴링

- **정의**: 프로프 드릴링은 부모 컴포넌트가 깊게 중첩된 자식 컴포넌트에 프로프를 전달하는 과정으로, 일부 자식 컴포넌트는 프로프가 필요 없더라도 전달하는 경우입니다.

## 상태 관리

- **Context API**: Context는 부모 컴포넌트가 자식 컴포넌트에 명시적으로 프로프를 통해 전달하지 않고도 정보를 제공하는 것을 말합니다. `createContext`는 다른 컴포넌트가 읽을 컨텍스트 객체를 생성하는 데 사용합니다. `Provider`는 자식 컴포넌트에 컨텍스트 값을 공급하는 데 사용합니다.

```jsx
import { useState, createContext } from "react";

const CounterContext = createContext();

const CounterProvider = ({ children }) => {
  const [count, setCount] = useState(0);

  return (
    <CounterContext.Provider value={{ count, setCount }}>
      {children}
    </CounterContext.Provider>
  );
};

export { CounterContext, CounterProvider };
```

- **Redux**: Redux는 중앙 저장소와 엄격한 상태 업데이트 제어를 통해 상태 관리를 처리합니다. 액션, 리듀서, 미들웨어라는 예측 가능한 패턴을 사용합니다. 액션은 애플리케이션에서 Redux 저장소로 데이터를 보내는 페이로드로, 주로 사용자 상호작용에 의해 트리거됩니다. 리듀서는 상태가 어떻게 변경되어야 하는지 지정하는 함수로, 상태를 불변 방식으로 업데이트합니다. 미들웨어는 액션 디스패치와 리듀서 사이에서 다리 역할을 하며, 핵심 흐름을 변경하지 않고 Redux 기능(예: 로깅, 비동기 작업 처리)을 확장할 수 있게 합니다.

- **Zustand**: 이 상태 관리 솔루션은 소규모에서 중간 규모 애플리케이션에 적합합니다. `useStore` 훅을 사용해 컴포넌트와 페이지에서 상태에 직접 접근합니다. 이를 통해 액션, 리듀서, 프로바이더 없이 데이터를 수정하고 접근할 수 있습니다.

## React 컴포넌트 디버깅: React 개발자 도구

- **React 개발자 도구**: Chrome, Firefox, Edge에서 React 컴포넌트를 검사하고 성능 문제를 식별하는 데 사용하는 브라우저 확장 프로그램입니다. Safari에서는 `react-devtools` npm 패키지를 설치해야 합니다. React DevTools를 설치하고 브라우저에서 React 앱을 열면, 브라우저 개발자 도구에서 React 디버깅용 두 개의 추가 탭인 Components와 Profiler를 사용할 수 있습니다.
- **Components 탭**: 이 탭은 앱의 컴포넌트 계층 구조를 트리 뷰 형식으로 보여줍니다. 여기서 할 수 있는 일은 다음과 같습니다:
  - 앱의 컴포넌트 계층 구조 보기
  - 프로프, 상태, 컨텍스트 값을 실시간으로 확인 및 수정
  - 선택한 컴포넌트의 소스 코드 확인
  - 컴포넌트 데이터를 콘솔에 기록
  - 컴포넌트의 DOM 요소 검사
- **Profiler 탭**: 이 탭은 컴포넌트 성능 분석을 돕습니다. 컴포넌트 성능을 기록해 불필요한 재렌더링을 식별하고 커밋 지속 시간을 확인하며 느린 컴포넌트를 최적화할 수 있습니다.

## React 서버 컴포넌트

- **정의**: React 서버 컴포넌트는 서버에서만 렌더링되는 React 컴포넌트로, 최종 HTML만 클라이언트에 전송합니다. 이로 인해 해당 컴포넌트는 서버 측 리소스에 직접 접근할 수 있고 브라우저로 전송되는 JavaScript 양을 크게 줄일 수 있습니다.
