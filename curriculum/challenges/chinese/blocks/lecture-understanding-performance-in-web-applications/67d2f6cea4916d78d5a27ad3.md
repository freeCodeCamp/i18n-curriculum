# --description--

让我们通过 Chrome DevTools 演示一种实用的方法来测量 Interaction to Next Paint，或 INP。

首先，按下 `F12` 键或右键点击并选择 `Inspect` 来打开开发者工具。

现在，假设你在网页上有一个搜索栏。

想象一个用户输入一个查询，但建议下拉菜单在 600 毫秒后才出现。Chrome DevTools 会将此标记为高 `INP` 值。

你可以通过推迟重 JavaScript、减少长任务和改进事件处理来优化。INP 将降至 150 毫秒，使交互感觉瞬间完成。

很酷，对吧？

现在，让我们来看一种实用的测量技术，它有助于识别交互延迟，从而让你创建更快更流畅的用户体验！

首先，我们会打开 Chrome DevTools。我们会再次通过打开 Google Chrome，导航到我们想要分析的网页，然后按下 `F12` 或右键点击页面并点击 `Inspect` 来完成。现在，让我们前往 `Performance` 标签（页）——这里有你需要的一切。

现在我们准备识别 Interaction to Next Paint，或 INP。我们将寻找记录的最长交互延迟。良好的 INP 通常低于 200ms，较差的 INP 高于 500ms。就这些！
