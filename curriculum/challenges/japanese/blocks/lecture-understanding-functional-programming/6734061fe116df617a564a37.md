# --interactive--

カリー化は、複数の引数を取る関数を、1つの引数だけを取る関数のシーケンスに変換する手法です。

まずは基本的な例から始めましょう。2つの数字を足す関数があると想像してください。

:::interactive_editor

```js
function add(a, b) {
  return a + b;
}

console.log(add(3, 4)); // 7
```

:::

これは2つの引数を取り、その合計を返す関数です。では、この関数をカリー化するとどうなるか見てみましょう。

:::interactive_editor

```js
function curriedAdd(a) {
  return function(b) {
    return a + b;
  }
}

console.log(curriedAdd(3)(4)); // 7
```

:::

このカリー化されたコードでは、一度に2つの引数を取る代わりに、最初の引数を取って別の関数を返します。この返された関数が2つ目の引数を取り、加算を行います。`curriedAdd(3)(4)`のように呼び出しますが、丸括弧のペアがそれぞれ関数の呼び出しを表しています。

でも、なぜこんなことをするのでしょうか？

カリー化を使うと、特別な関数を簡単に作成できます。例えば、どんな数字にも常に5を足す関数を作ることができます。

:::interactive_editor

```js
function curriedAdd(a) {
  return function(b) {
    return a + b;
  }
}

const addFive = curriedAdd(5);
console.log(addFive(10)); // 15
console.log(addFive(20)); // 25
```

:::

ここで、`addFive`は与えられた数字に常に5を足す準備ができている関数です。これは部分適用の簡単な例で、関数の引数の一部を固定して、より少ない引数を取る別の関数を生成します。

これまでの例は2つの引数を持つ関数に焦点を当てましたが、カリー化は任意の数の引数を持つ関数に適用できます。

JavaScriptの学習を続ける中で、カリー化がコードをよりシンプルで表現力豊かにする多くの場面に出会うでしょう。

## --text--

次のうち、JavaScriptでカリー化された関数の例はどれですか？

## --answers--

```js
function add(a, b, c) {
  return a + b + c;
}
```

### --feedback--

これはすべてのパラメータを一度に取る通常の関数です。

---

```js
function add(a) {
  return function (b) {
    return function (c) {
      return a + b + c;
    };
  };
}
```

---

```js
function add(a, b, c) {
  return a() + b() + c();
}
```

### --feedback--

この関数はすべてのパラメータを一度に受け取り、関数のように呼び出しています。これはカリー化関数の定義に合いません。

---

```js
function add(a) {
  function inner(b) {
    return a + b;
  }
}
```

### --feedback--

内部関数が返されないため、`add(2)`は`undefined`を返し、`add(2)(3)`はエラーをスローします。
