# --interactive--

L'attributo `aria-expanded` viene usato per scopi di accessibilità per indicare se un controllo è espanso o collassato. Viene usato insieme a widget collassabili come menu, accordion e altri widget di divulgazione che controllano la visibilità del contenuto. L'attributo `aria-expanded` viene impostato su `true` se il controllo è espanso, oppure su `false` se è collassato.

Le informazioni fornite da `aria-expanded` permettono alle persone che usano lettori di schermo di capire lo stato attuale del controllo (se è espanso o collassato).

L'attributo `aria-expanded` viene applicato all'elemento interattivo che attiva la visibilità di un widget collassabile. Per esempio, se un pulsante attiva un menu espandibile, l'attributo `aria-expanded` viene posto sul pulsante.

Quando il menu è espanso, l'attributo `aria-expanded` dovrebbe essere impostato su `true` come in questo esempio:

```html
<button aria-expanded="true">Menu</button>
```

Quando il menu è collassato, dovrebbe invece essere impostato su `false`.

```html
<button aria-expanded="false">Menu</button>
```

Devi sempre avere l'attributo `aria-expanded` impostato su `true` o `false` sull'elemento di controllo. Per esempio, se un pulsante attiva la visibilità di un menu, il valore predefinito per `aria-expanded` dovrebbe essere basato sulla visibilità predefinita del menu.

Se il menu è espanso per impostazione predefinita, `aria-expanded` dovrebbe inizialmente essere impostato su `true`. Se il menu è collassato per impostazione predefinita, `aria-expanded` dovrebbe inizialmente essere impostato su `false`.

Il valore di `aria-expanded` dovrebbe essere aggiornato dinamicamente usando JavaScript mentre l’utente interagisce con l’elemento.

Inoltre, le proprietà `aria-controls` e `aria-owns` possono essere usate in combinazione con `aria-expanded` per stabilire una connessione programmata tra l’elemento di controllo e l’elemento che controlla.

Iniziamo con `aria-controls`. Quando usato con `aria-expanded`, l'attributo `aria-controls` serve a specificare l’elemento controllato. Per esempio, un pulsante potrebbe espandere o collassare una lista che funge da menu. Il valore di `aria-controls` sarà l’`id` dell’elemento controllato (la lista del menu in questo esempio).

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<button aria-expanded="false" aria-controls="menu1">Menu</button>
<ul id="menu1" hidden>
  <li>...</li>
  <li>...</li>
</ul>

<script src="index.js"></script>
```

```css
button {
  background-color: #0078d4;
  color: white;
  padding: 8px 12px;
  border: none;
  border-radius: 4px;
  cursor: pointer;
}

button:hover {
  background-color: #005ea2;
}

ul {
  list-style: none;
  padding: 0;
  margin: 8px 0 0;
  border: 1px solid #ccc;
  border-radius: 4px;
  width: 150px;
  background: #fff;
}

  padding: 8px;

}

```

```js
const button = document.querySelector('button');
const menu = document.getElementById(button.getAttribute('aria-controls'));

button.addEventListener('click', () => {
  const expanded = button.getAttribute('aria-expanded') === 'true';
  button.setAttribute('aria-expanded', String(!expanded));
  menu.hidden = expanded;
});

```

Nota che la lista segue immediatamente il pulsante di controllo. Per controlli espandibili come questo, è meglio che il contenuto espanso segua immediatamente l’elemento che lo controlla nel DOM. Questo evita che gli utenti di lettori di schermo debbano cercare il contenuto espanso e facilita la navigazione con tastiera tra i controlli interattivi nel contenuto espanso.

Se non è possibile posizionare il contenuto espanso subito dopo l’elemento di controllo, l’attributo `aria-owns` permette di spostarlo virtualmente dopo il controllo nell’albero di accessibilità. Questo consente alla tecnologia assistiva come i lettori di schermo di fingere che il contenuto espanso sia posizionato direttamente dopo il controllo nel DOM.

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<button aria-owns="menu1" aria-expanded="true">Menu</button>
<main>
  <!-- an entire page worth of content --->
</main>
<ul id="menu1">
  <li>...</li>
  <li>...</li>
</ul>

<script src="index.js"></script>
```

  cursor: pointer;
  margin-bottom: 10px;
}

  background-color: #005ea2;
}

  list-style: none;
  padding: 0;
  margin: 8px 0;
  border: 1px solid #ccc;
  border-radius: 4px;
  width: 150px;

  padding: 8px;

```js
const button = document.querySelector('button');
const menu = document.getElementById(button.getAttribute('aria-owns'));

button.addEventListener('click', () => {
  const expanded = button.getAttribute('aria-expanded') === 'true';
  button.setAttribute('aria-expanded', String(!expanded));
  menu.hidden = expanded;
});

// Close the menu if user clicks outside
document.addEventListener('click', (e) => {
  if (!button.contains(e.target) && !menu.contains(e.target)) {
    button.setAttribute('aria-expanded', 'false');
    menu.hidden = true;
  }
});

```

Ci sono svantaggi nell’uso dell’attributo `aria-owns`. Crea una verbosità inutile per gli utenti di lettori di schermo, dato che la maggior parte dei lettori di schermo legge automaticamente tutto il contenuto dell’elemento espanso quando viene aperto. Inoltre, non cambia l’ordine di tabulazione, costringendo potenzialmente gli utenti da tastiera a tabulare attraverso tutti gli altri controlli interattivi della pagina prima di raggiungere il contenuto espanso, a meno che non gestisci l’ordine di tabulazione con JavaScript.

Idealmente, il contenuto espandibile dovrebbe essere posizionato dopo l’elemento di controllo, e l’attributo `aria-owns` dovrebbe essere usato solo in casi estremi quando ciò non è possibile. Se deve essere usato, dovrai testare accuratamente con un’ampia gamma di lettori di schermo e browser per assicurarti che la tua implementazione sia accessibile a tutti.

E solo un promemoria: quando usi `aria-controls` o `aria-owns`, il valore di `aria-expanded` deve continuare a essere aggiornato mentre il controllo viene espanso e collassato.

L’attributo `aria-expanded` indica se un controllo è espanso o collassato. Questa informazione è essenziale per gli utenti di lettori di schermo, aiutandoli a capire lo stato attuale di elementi collassabili come menu, accordion e altri widget di divulgazione simili.

Usando correttamente `aria-expanded`, puoi creare un’esperienza utente intuitiva per tutti.
