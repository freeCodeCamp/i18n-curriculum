# --description--

L’API Web Storage fornisce un meccanismo per i browser per archiviare coppie chiave-valore direttamente all’interno del browser, permettendo agli sviluppatori di conservare informazioni utilizzabili attraverso diversi ricaricamenti di pagina e sessioni.

I due componenti principali dell’API Web Storage sono `localStorage` e `sessionStorage`.

`localStorage` è la parte dell’API Web Storage che consente ai dati di persistere anche dopo che la finestra del browser è stata chiusa o la pagina è stata aggiornata. Questi dati rimangono disponibili finché non vengono rimossi esplicitamente dall’applicazione o dall’utente.

`sessionStorage` è un’altra parte dell’API Web Storage che conserva i dati per la durata della sessione della pagina, cioè i dati sono disponibili finché la scheda o la finestra del browser rimane aperta. Tuttavia, a differenza di `localStorage`, i dati in `sessionStorage` vengono cancellati quando la scheda o la finestra viene chiusa. Imparerai di più su `sessionStorage` nella prossima lezione.

Gli usi comuni di `localStorage` includono la memorizzazione delle impostazioni utente, come temi o preferenze di lingua, il ricordo dei dati dei moduli tra sessioni del browser e la memorizzazione temporanea di piccoli pezzi di informazioni per migliorare le prestazioni delle app web.

La memorizzazione temporanea (caching) consiste nel conservare dati consultati frequentemente in una posizione di archiviazione temporanea, chiamata cache, in modo che le richieste successive per quei dati possano essere servite più rapidamente senza doverli ricalcolare o recuperare da una fonte dati più lenta, come un database o un server esterno.

Alcuni metodi comuni di `localStorage` includono i metodi `setItem`, `getItem`, `removeItem` e `clear`.

Ecco un esempio di utilizzo del metodo `setItem()` che memorizza una coppia chiave-valore in `localStorage`.

```js
localStorage.setItem('username', 'JaneDoe');
```

Poi, se vogliamo recuperare il valore di una chiave specifica da `localStorage`, possiamo usare il metodo `getItem()`.

```js
let username = localStorage.getItem('username');
console.log(username); // JaneDoe
```

Per rimuovere un elemento da `localStorage` usando la sua chiave, puoi usare il metodo `removeItem()`.

```js
localStorage.removeItem('username');
```

Per cancellare tutti i dati in `localStorage`, puoi usare il metodo `clear()`.

```js
localStorage.clear();
```

Ora, diamo un’occhiata a un esempio in cui usiamo `localStorage` per memorizzare il tema preferito di un utente.

```js
// Store the user's theme preference
localStorage.setItem('theme', 'dark');

// Retrieve the stored theme preference
const userTheme = localStorage.getItem('theme');
console.log(userTheme); // 'dark'

// Remove the theme preference
localStorage.removeItem('theme');

// Clear all localStorage data
localStorage.clear();
```

In questo esempio:

- Prima memorizziamo una scelta di tema (`dark`) per l’utente.

- Poi recuperiamo quel tema e lo mostriamo nella console.

- Infine, dimostriamo come rimuovere un elemento specifico o cancellare tutti i dati memorizzati.

`localStorage` è molto utile per conservare piccoli pezzi di dati che devono persistere tra le sessioni, ma è importante notare che `localStorage` non dovrebbe essere usato per memorizzare informazioni sensibili, come password, perché può comportare rischi per la sicurezza.
