# --description--

- **JavaScript ya wakati mmoja** inatekelezwa kwa mfululizo na husubiri operesheni iliyotangulia imalizike kabla ya kuendelea na inayofuata.
- **JavaScript isiyo ya wakati mmoja** inaruhusu operesheni nyingi kutekelezwa nyuma ya pazia bila kuzuia nyuzi kuu.
- **Nyuzi** ni mfululizo wa maagizo yanayoweza kutekelezwa bila kutegemea mtiririko mkuu wa programu.
- **Vitendakazi vya mwitiko** ni vitendakazi vinavyopitishwa kama hoja kwa vitendakazi vingine na hufanywa baada ya operesheni kukamilika au kama matokeo ya tukio.

## Injini ya JavaScript na mazingira ya JavaScript

- **Injini ya JavaScript** ni programu inayotekeleza msimbo wa JavaScript katika kivinjari cha mtandao. Inafanya kazi kama mabadilishaji anayechukua msimbo wako, kuubadilisha kuwa maagizo ambayo kompyuta inaweza kuelewa na kufanya kazi kwa mujibu wake.
- V8 ni mfano wa injini ya JavaScript iliyotengenezwa na Google.
- **Mazingira ya utekelezaji ya JavaScript** ni mazingira ambayo msimbo wa JavaScript hufanyika ndani yake. Inajumuisha injini ya JavaScript inayosindika na kutekeleza msimbo, pamoja na vipengele vya ziada kama kivinjari cha mtandao au Node.js.

## Fetch API

- Fetch API inaruhusu programu za mtandao kufanya maombi ya mtandao, mara nyingi kuchukua au kutuma data kwenye seva. Inatoa njia ya `fetch()` unayoweza kutumia kufanya maombi haya.
- Unaweza kuchukua maandishi, picha, sauti, JSON, na aina nyingine za data kwa kutumia Fetch API.

## Njia za HTTP kwa Fetch API

Fetch API inaunga mkono njia mbalimbali za HTTP za kuwasiliana na seva. Njia za kawaida ni:

- **GET**: Hutumika kuchukua data kutoka kwenye seva. Kwa chaguo-msingi, Fetch API hutumia njia ya `GET` kuchukua data.

```js
fetch('https://api.example.com/data')
```

Ili kutumia data iliyochukuliwa, lazima ibadilishwe kuwa muundo wa JSON kwa kutumia njia ya `.json()`:

```js
fetch('https://api.example.com/data')
  .then(response => response.json())
  .then(data => console.log(data))
```

Katika msimbo huu, `fetch()` hurudisha Ahadi, na mshughulikiaji wa `.then()` hubadilisha jibu lililotatuliwa kuwa muundo wa JSON.

- **POST**: Hutumika kutuma data kwenye seva. Njia ya `POST` hutumika kuunda rasilimali mpya kwenye seva.

```js
fetch('https://api.example.com/users', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({
    name: 'John Doe',
    email: 'john@example.com'
  })
})
```

Katika mfano huu, tunatuma ombi la `POST` kuunda mtumiaji mpya. Tumeainisha njia kama `POST`, tumeweka vichwa vinavyofaa, na tumejumuisha mwili wenye data tunayotaka kutuma. Mwili lazima uwe mfuatano wa herufi, hivyo tunatumia `JSON.stringify()` kubadilisha kitu chetu kuwa mfuatano wa JSON.

- **PUT**: Hutumika kusasisha data kwenye seva. Njia ya `PUT` hutumika kusasisha rasilimali zilizopo kwenye seva.

```js
fetch('https://api.example.com/users/45', {
  method: 'PUT',
  headers: {
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({
    name: 'John Smith',
    email: 'john@example.com'
  })
})
```

Katika mfano huu, tunasasisha ID ya `45` iliyobainishwa mwishoni mwa URL. Tumeweka njia ya `PUT` kwenye msimbo na pia tumeainisha data kama mwili utakaotumika kusasisha data iliyotambuliwa.

- **DELETE**: Hutumika kufuta data kwenye seva. Njia ya `DELETE` hutumika kufuta rasilimali kwenye seva.

```js
fetch('https://api.example.com/users/45', {
  method: 'DELETE'
})
```

Katika mfano huu, tunatuma ombi la `DELETE` kuondoa mtumiaji mwenye ID ya `45`.

## Ahadi na uunganishaji wa ahadi (promise chaining)

- **Ahadi** ni vitu vinavyowakilisha kukamilika kwa hatima au kushindwa kwa operesheni isiyo ya wakati mmoja na thamani inayotokana nayo. Thamani ya ahadi inajulikana tu wakati operesheni isiyo ya wakati mmoja imemalizika.
- Hapa kuna mfano wa kuunda ahadi rahisi:

```js
const promise = new Promise((resolve, reject) => {
  setTimeout(() => {
    resolve('Data received successfully');
  }, 2000);
});
```

- Njia ya `.then()` hutumika kwenye Promise kubainisha nini kitafanyika ahadi ikitimia, wakati `.catch()` hutumika kushughulikia makosa yoyote yatakayotokea.
- Hapa kuna mfano wa kutumia `.then()` na `.catch()` na Promise:

```js
promise
  .then(data => {
    console.log(data);
  })
  .catch(error => {
    console.error(error);
  });
```

Katika mfano hapo juu, njia ya `.then()` hutumika kuchapisha data iliyopokelewa kutoka Promise, wakati njia ya `.catch()` hutumika kuchapisha makosa yoyote yatakayotokea.

- **Promise chaining**: Moja ya sifa zenye nguvu za Promise ni kwamba unaweza kuunganisha operesheni nyingi zisizo za wakati mmoja pamoja. Kila `.then()` inaweza kurejesha Promise mpya, na hivyo kukuwezesha kufanya mfululizo wa operesheni zisizo za wakati mmoja moja baada ya nyingine.
- Hapa kuna mfano wa Promise chaining:

```js
fetch('https://api.example.com/data')
  .then(response => response.json())
  .then(data => {
    console.log(data);
    return fetch('https://api.example.com/other-data');
  })
  .then(response => response.json())
  .then(otherData => {
    console.log(otherData);
  })
  .catch(error => {
    console.error(error);
  });
```

Katika mfano huu, kwanza tunachukua data kutoka URL moja, kisha tunachukua data kutoka URL nyingine kulingana na jibu la kwanza, na hatimaye tunachapisha data ya pili iliyopokelewa.

Njia ya `catch` ingeshughulikia makosa yoyote yatakayotokea wakati wa mchakato. Hii inamaanisha hutahitaji kuongeza ushughulikiaji wa makosa kwenye kila hatua, jambo linalorahisisha msimbo wako.

## Kutumia `async/await` kushughulikia ahadi

`async`/`await` hufanya uandishi na usomaji wa msimbo usio wa wakati mmoja kuwa rahisi zaidi, ambao umejengwa juu ya Ahadi.

- **async**: Neno `async` hutumika kufafanua kitendakazi kisicho cha wakati mmoja. Kitendakazi cha `async` hurejesha Promise, ambayo hutatuliwa na thamani inayorejeshwa na kitendakazi cha `async`.
- **await**: Neno `await` hutumika ndani ya kitendakazi cha `async` kusitisha utekelezaji wa kitendakazi hadi Promise itatuliwe. Inaweza kutumika tu ndani ya kitendakazi cha `async`.
- Hapa kuna mfano wa kutumia `async/await`:

```js
async function delayedGreeting(name) {
  console.log("A Messenger entered the chat...");
  await new Promise(resolve => setTimeout(resolve, 2000));
  console.log(`Hello, ${name}!`);
}

delayedGreeting("Alice");
console.log("First Printed Message!");
```

Katika mfano huu, kitendakazi cha `delayedGreeting` ni kitendakazi cha `async` kinachosubiri kwa sekunde 2 kabla ya kuchapisha ujumbe wa salamu. Neno `await` hutumika kusitisha utekelezaji wa kitendakazi hadi `Promise` itatuliwe.

- Moja ya faida kubwa za `async/await` ni ushughulikiaji wa makosa kupitia vizuizi vya `try/catch`. Hapa kuna mfano:

```js
async function fetchData() {
  try {
    const response = await fetch('https://api.example.com/data');
    const data = await response.json();
    console.log(data);
  } catch (error) {
    console.error(error);
  }
}

fetchData();
```

Katika mfano huu, kizuizi cha `try` kina msimbo ambao unaweza kutupa kosa, na kizuizi cha `catch` kinashughulikia kosa likitokea. Hii hufanya ushughulikiaji wa makosa kuwa rahisi na unaosomeka zaidi.

## Sifa ya `async`

- Sifa ya `async` inaeleza kivinjari kupakua faili la script isiyo ya wakati mmoja huku ikiendelea kuchambua hati ya HTML.
- Mara script inapopakuliwa, uchambuzi wa HTML unasitishwa, script inatekelezwa, kisha uchambuzi wa HTML unaendelea tena.
- Unapaswa kutumia `async` kwa script huru ambapo mpangilio wa utekelezaji haujaliwi.

## Sifa ya `defer`

- Sifa ya `defer` pia hupakua script isiyo ya wakati mmoja, lakini huahirisha utekelezaji wa script hadi baada ya hati ya HTML kuchambuliwa kikamilifu.
- Script za `defer` huhifadhi mpangilio wa utekelezaji kama zinavyoonekana katika hati ya HTML.

- Ni muhimu kufahamu kwamba sifa za `async` na `defer` hupitwa kwa skripti za ndani na hufanya kazi tu kwa faili za skripti za nje.

- Wakati sifa za `async` na `defer` zote zipo, sifa ya `async` ndiyo inayotangulizwa.

## API ya Geolocation

- Geolocation API inatoa njia kwa tovuti kuomba mahali alipo mtumiaji.

- Mfano uliopo chini unaonyesha njia ya API ya `getCurrentPosition()`, ambayo hutumika kupata eneo la mtumizi kwa sasa.

```js
navigator.geolocation.getCurrentPosition(
  (position) => {
    console.log("Latitude: " + position.coords.latitude);
    console.log("Longitude: " + position.coords.longitude);
  },
  (error) => {
    console.log("Error: " + error.message);
  }
);
```

Katika msimbo huu, tunaita `getCurrentPosition` na kuipatia kitendakazi, ambacho kitaitwa wakati nafasi itakapopatikana kwa mafanikio.

Kitu cha `position` kina taarifa mbalimbali, lakini hapa tumechagua `latitude` na `longitude` pekee.

Kama kuna tatizo kupata `position`, basi kosa litaandikwa kwenye konsoli.

- Ni muhimu kuheshimu faragha ya mtumizi na kuomba eneo lake tu wakati ni lazima.
