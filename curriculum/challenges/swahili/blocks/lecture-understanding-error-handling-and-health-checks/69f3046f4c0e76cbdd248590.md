# --description--

Je, umewahi kujiuliza jinsi ya kuweka programu yako ya Express imara na tayari kwa uzalishaji, hasa wakati iko chini ya mzigo mkubwa au inafunga?

Hapo ndipo ukaguzi wa afya na kufunga kwa heshima vinapoingia. Mifumo hii miwili inakusaidia kuweka programu yako inapatikana, na kuhakikisha inatoka kwa usafi bila kupoteza maombi au kuharibu data.

Hebu tuchambue.

Kwanza, ukaguzi wa afya ni nini hata katika Express?

Ukaguzi wa afya ni njia rahisi inayosema kama programu yako iko hai na inafanya kazi.

Kwa mfano:

```js
app.get('/health', (req, res) => {
  res.status(200).send('OK')
})
```

Vifaa kama wasawazishaji mzigo au watawala wa kontena - fikiria Kubernetes au AWS Elastic Beanstalk - vinaweza kupiga njia hii ya data ili kuamua kama programu yako iko katika hali nzuri.

Ikiwa njia hiyo inarudisha msimbo wa hali 200, huduma yako inachukuliwa kuwa yenye afya. Kumbuka - msimbo wa hali wa HTTP 200 unamaanisha kila kitu kiko sawa!

Ikiwa haitajibu, mtawala anaweza kuanzisha huduma tena moja kwa moja.

Na kufunga kwa heshima ni nini?

Kufunga kwa heshima ni kinachotokea wakati programu yako inakaribia kusimama, kama wakati seva inapoanzishwa upya au usambazaji wa programu unapotokea.

Badala ya kukata muunganisho mara moja, unampa programu muda wa kumaliza kile inachofanya.

Katika Express, unaweza kusikiliza `SIGTERM`, ambayo ni ishara ya mfumo inayotumwa wakati mchakato unapaswa kumalizika.

Hapa kuna mfano:

```js
process.on('SIGTERM', () => {
  server.close(() => {
    console.log('Server closed. Cleaning up...')
    // Close DB connections or other cleanup here
  })
})
```

Hii inamwambia Express kuacha kupokea miunganisho mipya na kusubiri ile iliyopo imalizike kabla ya kutoka.

Pia unapaswa kushughulikia `SIGINT`, ambayo ni ishara inayotumwa unapobofya Ctrl+C kusimamisha programu yako wakati wa ukuzaji wa ndani.

Wakati `SIGTERM` kawaida hutumiwa na watawala kama Kubernetes katika uzalishaji, `SIGINT` inashughulikia mtiririko wa kazi wa msanidi programu wa kila siku. Bila kuishughulikia, kubofya Ctrl+C hualiza mchakato mara moja, ikiruka usafi wowote ulioweka.

Kwa kuwa ishara zote mbili zinapaswa kusababisha tabia ile ile ya kufunga kwa heshima, wasanidi programu kawaida husikiliza zote mbili.

Sasa, tuchanganye yote pamoja

Fikiria programu yako inafanya kazi katika kontena. Kubernetes inatuma `SIGTERM` kuifunga kwa ajili ya sasisho.

Kwa kufunga kwa heshima tu, programu yako inakataa kupokea miunganisho mipya na inamaliza maombi yaliyopo. Mara kazi zote zinapomalizika, inakatisha muunganisho na hifadhidata na kutoka kwa usafi. Hakuna data iliyopotea, hakuna hali iliyoharibika.

Lakini kuna tatizo — wasawazishaji mzigo hawajui programu yako inafunga, hivyo wanaweza kuendelea kutuma trafiki mpya kwako. Hapo ndipo mfumo wa ziada hapa chini unapoingia.

Sasa, nina zawadi kidogo kwako. Tutaangalia "Kuweka alama programu kuwa haiko katika hali nzuri kabla ya kufunga".

Kabla ya kufunga, unaweza kusahihisha ukaguzi wa afya kurudisha msimbo wa hali 503. Kwa njia hiyo, wasawazishaji mzigo wanaona jibu la hali mbaya na kuacha kupeleka trafiki mpya kwa programu yako wakati wa dirisha la kufunga, kama ilivyoonyeshwa katika msimbo hapo juu.

Kumbuka: Msimbo wa hali `503` unamaanisha "Huduma haipatikani". Inatoa ishara kuwa seva haiwezi kushughulikia maombi kwa muda, ambayo ni hasa kinachotokea wakati wa kufunga kwa heshima.

```js
let isShuttingDown = false

app.get('/health', (req, res) => {
  if (isShuttingDown) {
    res.status(500).send('Shutting down')
  } else {
    res.status(200).send('OK')
  }
})

process.on('SIGTERM', () => {
  isShuttingDown = true

  server.close(() => {
    console.log('Graceful shutdown complete')
    // Perform cleanup
  })
})
```
