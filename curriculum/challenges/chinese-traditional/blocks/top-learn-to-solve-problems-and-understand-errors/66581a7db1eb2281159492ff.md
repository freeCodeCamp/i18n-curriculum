# --description--

錯誤的另一個重要零件是堆疊追蹤。這有助於你了解錯誤在你的應用程式中何時被發出，以及導致錯誤的函式（程式）呼叫了哪些。因此，例如，如果你有以下程式碼：

```javascript
const a = 5;
const b = 10;

function add() {
  return c;
}

function print() {
  add();
}

print();
```

我們的函式（程式）`print()` 應該呼叫 `add()`，該函式（程式）傳回一個名為 `c` 的變數，該變數目前尚未宣告。對應的錯誤如下：

<img src="https://cdn.statically.io/gh/TheOdinProject/curriculum/284f0cdc998be7e4751e29e8458323ad5d320303/foundations/javascript_basics/understanding_errors/imgs/01.png" width="100%" alt="An error showing with a stacktrace in the developer console">

堆疊追蹤告訴我們：

1. `add()` 第 5 行宣告的範圍內未定義 `c`。
1. `add()` 是由在第 9 行宣告的 `print()` 呼叫的。
1. `print()` 本身在第 12 行被呼叫。

因此，堆疊追蹤讓你追蹤錯誤的演變回到其起源，這裡是 `add()` 的宣告。

## --answers--  

堆疊追蹤顯示函式 `add()` 正確傳回 `c` 的值。

堆疊追蹤提供導致錯誤的函式呼叫詳細路徑，有助於識別 `c` 被錯誤取用的位置。
