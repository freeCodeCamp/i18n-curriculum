# --interactive--

정규 표현식, 또는 정규식은 여러 프로그래밍 언어에서 지원하는 기능입니다.

정규 표현식은 "패턴"을 만들기 위한 특별한 구문으로, 이를 사용해 문자열을 검사하거나 텍스트를 추출하는 등 다양한 작업을 할 수 있습니다.

기본적인 정규 표현식을 살펴보겠습니다:

```js
const regex = /freeCodeCamp/;
```

JavaScript에서 정규 표현식을 정의할 때 패턴을 두 개의 슬래시(`/`) 사이에 작성한다는 점을 주목하세요. 이것을 주석과 혼동하지 마세요. 주석은 두 개의 슬래시 뒤에 텍스트가 옵니다.

이 정규 표현식은 문자열 어디에서든 대문자 `freeCodeCamp`가 포함된 `C` 텍스트와 일치합니다. 그런데 실제로 어떻게 할 수 있을까요?

첫 번째 메서드인 `test()` 메서드가 바로 그 방법입니다. `test()` 메서드는 정규 표현식을 나타내는 `RegExp` 객체에 존재합니다(우리가 방금 정의한 것과 같은 객체입니다).

`test()` 메서드는 문자열을 인수로 받아 정규 표현식과 일치하는지 테스트합니다. 예를 들어, 문자열 `e`을 테스트해 봅시다:

:::interactive_editor

```js
const regex = /freeCodeCamp/;
const test = regex.test("e");
console.log(test);
```

:::

새로 만든 정규식에서 `test()` 메서드를 호출하고 문자열 `e`을 인수로 전달한 것을 볼 수 있습니다.

`test()` 메서드는 `false`을 반환했는데, 이는 문자열 `e`가 패턴 `freeCodeCamp`과 일치하지 않기 때문입니다. 패턴 `freeCodeCamp`에 문자 `e`가 포함되어 있지만, 정규 표현식이 작동하는 방향과는 반대입니다.

몇 가지 예를 더 살펴보겠습니다. 잠시 시간을 내어 생각해 보세요:

:::interactive_editor

```js
const regex = /freeCodeCamp/;
console.log(regex.test("freeCodeCamp"));
console.log(regex.test("freeCodeCamp is great"));
console.log(regex.test("I love freeCodeCamp"));
console.log(regex.test("freecodecamp"));
console.log(regex.test("FREECODECAMP"));
console.log(regex.test("free"));
console.log(regex.test("code"));
console.log(regex.test("camp"));
```

:::

처음 세 문자열이 `true`을 반환한 것을 주목하세요. 이 문자열들은 모두 문자열 어딘가에 `freeCodeCamp` 텍스트를 정확히 포함하고 있습니다.

5번째와 6번째 줄은 `false`을 반환합니다. `freecodecamp` 텍스트를 포함하지만 대소문자가 일치하지 않기 때문입니다. 정규 표현식은 기본적으로 대소문자를 구분합니다.

마지막 세 문자열은 패턴의 일부를 포함하지만, 전체 패턴을 포함하지는 않습니다.

`test()` 메서드는 문자열이 정규 표현식과 일치하는지 여부를 나타내는 `boolean`을 반환합니다.

그런데 더 많은 정보를 원한다면 어떻게 할까요? 문자열에는 `match()` 메서드가 있습니다. 이 메서드는 정규 표현식을 인수로 받으며, 문자열을 전달하면 정규 표현식으로 변환됩니다.

`match()`은 문자열에 대한 일치 배열을 반환합니다. 일치 배열이 무엇일까요? 살펴보겠습니다:

```js
const regex = /freeCodeCamp/;
const match = "freeCodeCamp".match(regex);
console.log(match);
```

실행하면 배열이 반환됩니다! 하지만 조금 이상한 배열입니다. 몇 가지 추가 속성이 있습니다:

```js
// [
//   'freeCodeCamp',
//   index: 0,
//   input: 'freeCodeCamp',
//   groups: undefined
// ]
```

`groups` 속성은 캡처된 그룹을 보여줍니다. 이 의미는 앞으로 수업에서 배우게 됩니다.

`index` 속성은 문자열에서 일치가 발견된 문자의 위치를 알려줍니다. 여기서는 문자열의 시작 부분에서 발견되었습니다.

`input` 속성은 `match()` 메서드가 호출된 문자열을 나타냅니다.

다시 몇 가지 예를 시도해 보고 결과가 어떻게 달라지는지 살펴보겠습니다:

```js
const regex = /freeCodeCamp/;
console.log("freeCodeCamp".match(regex)); // ['freeCodeCamp', index: 0, input: 'freeCodeCamp', groups: undefined]
console.log("freeCodeCamp is great".match(regex)); // ['freeCodeCamp', index: 0, input: 'freeCodeCamp is great', groups: undefined]
console.log("I love freeCodeCamp".match(regex)); // ['freeCodeCamp', index: 7, input: 'I love freeCodeCamp', groups: undefined]
console.log("freecodecamp".match(regex)); // null
console.log("FREECODECAMP".match(regex)); // null
console.log("free".match(regex)); // null
```

처음 세 문자열이 일치를 생성할 것이라는 것을 이미 알고 있으니, 그 결과를 살펴봅시다:

```js
// ['freeCodeCamp', index: 0, input: 'freeCodeCamp', groups: undefined]
// ['freeCodeCamp', index: 0, input: 'freeCodeCamp is great', groups: undefined]
// ['freeCodeCamp', index: 7, input: 'I love freeCodeCamp', groups: undefined]
```

예상한 대로인가요? 제공된 문자열과 일치 위치에 따라 `input`과 `index`가 어떻게 달라졌는지 볼 수 있습니다.

일치하지 않는 나머지 세 줄은 배열 대신 `null`을 반환합니다.

이제 정규 표현식으로 문자열을 테스트하고 일치시킬 수 있으니, 문자열 내용을 바꾸고 싶다면 어떻게 할까요? 예를 들어 누군가가 `freecodecamp`를 모두 소문자로 썼는데, 자동으로 대소문자를 고치고 싶을 수 있습니다.

먼저 `freecodecamp`의 소문자 형태와 일치하도록 정규 표현식을 업데이트하고 테스트 문자열을 만듭니다:

```js
const regex = /freecodecamp/;
const str = "freecodecamp is rly kewl";
```

문자열에는 `replace()` 메서드가 있는데, 이 메서드는 두 개의 인수를 받습니다: 일치시킬 정규 표현식(또는 정규식의 모든 기능이 필요 없으면 문자열)과 일치하는 부분을 대체할 문자열(또는 각 일치에 대해 실행할 함수)입니다.

그래서 `freecodecamp`를 올바른 대소문자로 바꾸고 싶다면:

:::interactive_editor

```js
const regex = /freecodecamp/;
const str = "freecodecamp is rly kewl";
const replaced = str.replace(regex, "freeCodeCamp");
console.log(replaced);
```

:::

`replace()`가 일치하는 패턴 `replaced`을 포함한 업데이트된 문자열을 반환하는 것을 볼 수 있습니다.

정규 표현식과 관련 메서드는 복잡하고 벅차 보일 수 있지만, 다음 수업에서 더 깊이 탐구할 기회가 있을 것입니다.
