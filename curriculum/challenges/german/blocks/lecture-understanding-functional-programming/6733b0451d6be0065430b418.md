# --interactive--

Funktionale Programmierung bedeutet, Code zu schreiben, der sauber, vorhersehbar und leichter zu testen und zu debuggen ist. Zwei wichtige Konzepte, die wir heute erkunden, sind reine Funktionen und das Vermeiden von Seiteneffekten.

Beginnen wir mit reinen Funktionen.

Eine reine Funktion ist eine Funktion, die bei derselben Eingabe immer dieselbe Ausgabe liefert und nichts außerhalb von sich selbst verändert.

Hier ist ein Beispiel für eine reine Funktion:

```js
function add(a, b) {
  return a + b;
}
```

Diese Funktion ist rein, weil sie für dieselben Eingaben immer dasselbe Ergebnis zurückgibt und nichts außerhalb von sich selbst ändert. Egal wie oft Sie `add(2, 3)` aufrufen, es wird immer `5` zurückgeben, und es beeinflusst keinen anderen Teil Ihres Programms.

Vergleichen wir das nun mit einer unreinen Funktion:

```js
let total = 0;
function addToTotal(value) {
  total += value;
  return total;
}
```

Diese Funktion ist unreif, weil sie eine Variable außerhalb von sich selbst (`total`) verändert. Das Ergebnis von `addToTotal(5)` hängt davon ab, was `total` vor dem Aufruf der Funktion war. Diese Unvorhersehbarkeit kann unseren Code schwerer verständlich und schwieriger zu debuggen machen.

Das führt uns zum nächsten wichtigen Konzept: den Seiteneffekten.

Ein Seiteneffekt ist jede Änderung, die im Zustand des Programms auftritt, wenn eine Funktion aufgerufen wird. Das kann das Verändern einer globalen Variable, das Schreiben in eine Datei oder das Ausführen eines API-Aufrufs sein.

Reine Funktionen haben keine Seiteneffekte, unreine Funktionen hingegen schon.

Das Vermeiden von Seiteneffekten ist ein zentrales Prinzip der funktionalen Programmierung. Wenn wir Seiteneffekte minimieren, wird unser Code vorhersehbarer und leichter zu testen.

Beim Schreiben von Code sollte unser Hauptziel sein, Funktionen so zu gestalten, dass sie genau eine Sache tun und diese sehr gut, ohne andere Teile unseres Programms zu verändern.

Hier ist ein Beispiel für eine Funktion mit einem Seiteneffekt:

:::interactive_editor

```js
function greet(name) {
  console.log(`Hello, ${name}!`);
}

greet("Alice");
```

:::

Diese Funktion hat einen Seiteneffekt, weil sie mit der Außenwelt interagiert, indem sie in die Konsole schreibt.

In der funktionalen Programmierung könnten wir das so umschreiben:

:::interactive_editor

```js
function greet(name) {
  return `Hello, ${name}!`;
}
console.log(greet("Alice"));
```

:::

Jetzt ist `greet` eine reine Funktion, die einfach einen String zurückgibt, und wir behandeln den Seiteneffekt (das Schreiben in die Konsole) separat.

Die funktionale Programmierung ermutigt uns, unsere Programme aus reinen Funktionen zusammenzusetzen, indem wir unseren Code in kleine, vorhersehbare Teile zerlegen. So können wir Programme erstellen, die leichter zu verstehen, zu testen und zu warten sind.
