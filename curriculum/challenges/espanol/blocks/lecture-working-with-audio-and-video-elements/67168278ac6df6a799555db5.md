# --interactive--

Los elementos `audio` y `video` te permiten añadir contenido de sonido y video a tus documentos HTML. El elemento `audio` soporta formatos de audio populares como mp3, wav y ogg. El elemento `video` soporta formatos mp4, ogg y webm.

Para incluir contenido de audio en tu página web, puedes usar el elemento `audio` con el atributo `src` que apunta a la ubicación de tu archivo de audio. 

Como puedes ver en la ventana de vista previa, no aparece nada en la página. Para ver las vistas previas, necesitarás habilitar el editor interactivo.

:::interactive_editor

```html
<audio src="https://cdn.freecodecamp.org/curriculum/js-music-player/cruising-for-a-musing.mp3"></audio>
```

:::

Si quieres ver el reproductor de audio en la página, entonces puedes agregar el elemento `audio` con el atributo `controls`.

Presiona play en la ventana de vista previa para escuchar una de las canciones de Quincy Larson. Para escuchar una canción diferente, cambia el valor de `src` a `"https://cdn.freecodecamp.org/curriculum/js-music-player/never-not-favored.mp3"`. Para ver las vistas previas, necesitarás habilitar el editor interactivo.

:::interactive_editor

```html
<audio src="https://cdn.freecodecamp.org/curriculum/js-music-player/cruising-for-a-musing.mp3" controls></audio>
```

:::

The `controls` attribute enables users to manage audio playback, including pausing or resuming playback. El atributo `controls` es un atributo booleano que puede añadirse a un elemento para habilitar controles de reproducción integrados. Si se omite, no se mostrarán controles.

Nota: Algunos navegadores, como Safari, pueden no mostrar un control de volumen por defecto incluso cuando el atributo `controls` está presente.

Además de los atributos `src` y `controls`, hay varios otros atributos que mejoran la funcionalidad del elemento `audio`. El atributo `loop` es un atributo booleano que hace que el audio se reproduzca continuamente. 

Aquí tienes un ejemplo de cómo usar el atributo `loop` para reproducir una de las canciones de Quincy Larson titulada "Can't stay down". Para ver el bucle en acción, activa el editor interactivo, mueve el cabezal de reproducción cerca del final de la canción y esta se reiniciará una vez que termine. 

:::interactive_editor

```html
<audio
  src="https://cdn.freecodecamp.org/curriculum/js-music-player/can't-stay-down.mp3"
  loop
  controls
></audio>
```

:::

Otro atributo que puedes usar es el atributo `muted`. Cuando está presente en el elemento `audio`, este atributo booleano iniciará el audio en un estado silenciado.  Here's an example of using the `muted` attribute.

Para escuchar la música, activa el editor interactivo y haz clic en el ícono de volumen en el reproductor de audio. 

:::interactive_editor

```html
<audio
  src="https://cdn.freecodecamp.org/curriculum/js-music-player/can't-stay-down.mp3"
  loop
  controls
  muted
></audio>
```

:::

En cuanto a los tipos de archivos de audio, hay diferencias en qué navegadores soportan qué tipo. Para acomodar esto, puedes usar elementos `source` dentro del elemento `audio`, y el navegador seleccionará la primera fuente que comprenda. Aquí tienes un ejemplo de uso de múltiples elementos `source` para un elemento `audio`:

```html
<audio controls>
  <source src="audio.ogg" type="audio/ogg" />
  <source src="audio.wav" type="audio/wav" />
  <source src="audio.mp3" type="audio/mpeg" />
</audio>
```

El navegador empezará primero con el tipo ogg, y si no puede reproducir el audio, pasará al siguiente tipo en la lista.

Todos los atributos que hemos aprendido hasta ahora también son compatibles con el elemento `video`. Aquí tienes un ejemplo de cómo usar un elemento `video` con los atributos `loop`, `controls` y `muted`. 

Agrega el atributo `autoplay` a la etiqueta de apertura `video` para que el video se reproduzca automáticamente. Para interactuar con el ejemplo, necesitarás habilitar el editor interactivo.

**NOTA**: El atributo `width` se está usando aquí para hacer que el video sea más pequeño y se ajuste mejor en la ventana de vista previa. Aprenderás más sobre el atributo `width` en lecciones futuras.

:::interactive_editor

```html
<video
  src="https://archive.org/download/BigBuckBunny_124/Content/big_buck_bunny_720p_surround.mp4"
  loop
  controls
  muted
  width="400"
></video>
```

:::

Para el `src`, o atributo fuente, estamos usando un video llamado "Big Buck Bunny" de archive.org. Si deseas mostrar una imagen mientras el video se descarga, puedes usar el atributo `poster`. Este atributo no está disponible para elementos `audio` y es único para el elemento `video`.  Here's an example of using the `poster` attribute with content provided by peach.blender.org.

```html
<video
  src="https://archive.org/download/BigBuckBunny_124/Content/big_buck_bunny_720p_surround.mp4"
  loop
  controls
  muted
  poster="https://peach.blender.org/wp-content/uploads/title_anouncement.jpg?x11217"
  width="400"
></video>
```

También puedes usar el elemento `source` dentro de un elemento `video`, tal como hiciste con el elemento `audio`. Esto te permite proporcionar el mismo video en múltiples formatos, y el navegador elegirá el primero que pueda reproducir.

```html
<video
  controls
  width="400"
  poster="https://peach.blender.org/wp-content/uploads/title_anouncement.jpg?x11217"
>
  <source
    src="https://archive.org/download/BigBuckBunny_124/Content/big_buck_bunny_720p_surround.mp4"
    type="video/mp4"
  />
  <source
    src="https://archive.org/download/BigBuckBunny_124/Content/big_buck_bunny_720p_surround.webm"
    type="video/webm"
  />
  Your browser does not support the video tag.
</video>
```
