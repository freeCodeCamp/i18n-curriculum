## データベース正規化

これはリレーショナルデータベースを整理して、データの冗長性を減らし整合性を向上させるプロセスです。

その利点は以下の通りです：

- 重複データを最小限に抑え、ストレージを節約し不整合を減らします。
- 主キーや外部キーを使ってデータの整合性を強制します。
- データベースの保守や理解がしやすくなります。

### 正規形

- **1NF（第一正規形）**
  - 各セルは単一（原子）値を含みます。
  - 各レコードは一意である（主キーで強制）。
  - 行や列の順序は関係ありません。
  - 例：複数の電話番号を `students` テーブルから別の `student_phones` テーブルに移動します。

- **2NF（第二正規形）**
  - 1NF の要件を満たします。
  - **部分関数従属**がない：すべての非キー属性は複合主キー全体に依存しなければなりません。
  - 例：`orders` テーブルを `order_header` と `order_items` に分割し、属性がキーの一部にのみ依存するのを避けます。

- **3NF（第三正規形）**
  - 2NF の要件を満たします。
  - **推移的関数従属**がない：非キー属性は他の非キー属性に依存できません。
  - 例：`city_postal_code` をすべての注文と一緒に保存するのではなく、`cities` テーブルに移動します。

- **BCNF（ボイス・コッド正規形）**
  - 3NF の要件を満たします。
  - すべての決定子（関数従属の左辺）はスーパーキーでなければなりません。

**ヒント**：ほとんどの設計では、整合性とパフォーマンスのバランスが良い 3NF を目指してください。

## 重要な SQL の概念

- SQL はリレーショナルデータベースと通信するための構造化クエリ言語です。
- **基本コマンド** → `SELECT`、`INSERT`、`UPDATE`、`DELETE`、`CREATE TABLE`、`ALTER TABLE` など。
- `Joins` → 複数のテーブルからデータを結合します（`INNER JOIN`、`LEFT JOIN`、`RIGHT JOIN`、`FULL JOIN`）。

## Bash での SQL コマンドの実行

PostgreSQL の `psql` コマンドラインクライアントや他のデータベース用の類似ツールを使って、コマンドラインから直接 SQL コマンドを実行できます。

例えば、PostgreSQL で SQL ファイルを実行するには：

```bash
psql -U username -d database_name -c "SELECT * FROM students;"
```

MySQL コマンドを直接実行することもできます：

```bash
mysql -u username -p database_name -e "SELECT * FROM students;"
```

### ファイルから SQL を実行する

```bash
# PostgreSQL
psql -U username -d database_name -f script.sql

# MySQL
mysql -u username -p database_name < script.sql
```

### Bash スクリプトに SQL を埋め込む

```bash
#!/bin/bash
DB_USER="school_admin"
DB_NAME="school"

# Insert student data
psql -U "$DB_USER" -d "$DB_NAME" -c \
"INSERT INTO students (name, age, major) VALUES ('Alice', 20, 'CS');"
```

### SQL での変数の使用

```bash
#!/bin/bash
DB_USER="school_admin"
DB_NAME="school"
STUDENT_NAME="Bob"
AGE=21

psql -U "$DB_USER" -d "$DB_NAME" -c \
"INSERT INTO students (name, age) VALUES ('$STUDENT_NAME', $AGE);"
```

**ヒント**：SQL インジェクションを避けるために変数をサニタイズしてください。

## Bash で SQL クエリ結果を取得して利用する

`psql` 経由で SQL クエリを実行すると、返された値を Bash スクリプト内で**取得**して**処理**できます。

### 単一値の取得

```bash
#!/bin/bash
DB_USER="school_admin"
DB_NAME="school"

# Get total student count
STUDENT_COUNT=$(psql -U "$DB_USER" -d "$DB_NAME" -t -A -c \
"SELECT COUNT(*) FROM students;")

echo "Total students: $STUDENT_COUNT"
```

出力 → 42

### 複数カラムの取得

```bash
#!/bin/bash
DB_USER="school_admin"
DB_NAME="school"

# Get top 3 students' names and ages
RESULTS=$(psql -U "$DB_USER" -d "$DB_NAME" -t -A -F"," -c \
"SELECT name, age FROM students LIMIT 3;")

echo "Top 3 students:"
echo "$RESULTS"
```

出力

```bash
Alice,20
Bob,21
Charlie,22
```

### クエリ結果をループ処理する

```bash
#!/bin/bash
DB_USER="school_admin"
DB_NAME="school"

# Get student names and majors
psql -U "$DB_USER" -d "$DB_NAME" -t -A -F"," -c \
"SELECT name, major FROM students;" | while IFS="," read -r name major
do
    echo "Student: $name | Major: $major"
done
```

出力の形

```bash
Student: Alice | Major: CS
Student: Bob   | Major: Math
Student: Carol | Major: Physics
```

## SQL インジェクション

これはウェブのセキュリティ脆弱性で、攻撃者が入力フィールドに悪意のある SQL コードを挿入してデータベースを操作するものです。

これにより以下のような危険な行為が起こります：

- 認証の回避。
- 機密データの窃取。
- レコードの変更や削除。

SQL インジェクション攻撃の例：

```sql
SELECT * FROM users WHERE username = ' " " OR "1"="1" -- ' AND password = 'anything';
```

このクエリは `OR "1"="1"` の条件が常に真になるため、すべてのユーザーを返し、攻撃者がログインチェックを回避できます。

### SQL インジェクションの防止

1. **プリペアドステートメントの使用**：SQL コードとデータを分離し、インジェクションを防ぎます。例（Node.js と pg）：

    ```sql
    client.query('SELECT * FROM users WHERE username = $1 AND password = $2', [username, password]);
    ```

2. **入力のバリデーション**：すべてのユーザー入力をサニタイズし、期待される形式に合っているか検証してください。

3. **最小権限の原則**：アプリケーションに必要な最小限の権限を持つデータベースアカウントを使用してください。

**注意**：アプリケーションアカウントに管理者権限を絶対に与えないでください。

## N+1 問題

N+1 問題は、アプリケーションがアイテムのリスト（N）を取得するために1回のクエリを実行し、その後各アイテムに関連するデータを取得するために追加で N 回のクエリを実行し、合計 N+1 回のクエリが発生する問題です。

**なぜ問題か**

- 各クエリはネットワークと処理のオーバーヘッドを増やします。
- 複数の小さなクエリは、1つの最適化されたクエリより遅くなります。

### N+1 パターンの例

```sql
-- 1: Get list of orders
SELECT * FROM orders LIMIT 50;

-- N: For each order, get customer
SELECT * FROM customers WHERE customer_id = ...;
```

**解決策**：`JOINs` や他のセットベースの操作を使います。

```sql
SELECT 
  orders.order_id,
  orders.product,
  orders.quantity,
  customers.customer_id,
  customers.name,
  customers.email,
  customers.address
FROM orders
JOIN customers 
  ON orders.customer_id = customers.customer_id
WHERE orders.order_id IN (SELECT order_id FROM orders LIMIT 50);
```

関連データを1つのクエリにまとめる機会を常に探してください。
