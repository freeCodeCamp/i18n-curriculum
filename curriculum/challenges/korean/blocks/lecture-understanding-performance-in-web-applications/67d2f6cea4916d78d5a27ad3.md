# --description--

Chrome 개발자 도구를 사용해 Interaction to Next Paint, 즉 INP를 측정하는 실용적인 방법을 살펴보겠습니다.

먼저, "F12" 키를 누르거나 마우스 오른쪽 버튼을 클릭한 후 "검사"를 클릭해 개발자 도구를 열어보세요.

이제 웹페이지에 검색창이 있다고 가정해봅시다.

사용자가 쿼리를 입력했는데 제안 드롭다운이 나타나기까지 600ms가 걸린다고 상상해보세요. Chrome 개발자 도구는 이 높은 INP 값을 표시할 것입니다.

무거운 JavaScript를 지연시키고, 긴 작업을 줄이며, 이벤트 처리를 개선하는 방식으로 최적화할 수 있습니다. 그러면 INP가 150ms로 떨어져 상호작용이 즉각적으로 느껴질 것입니다.

멋지죠?

이제 상호작용 지연을 식별하는 데 도움이 되는 실용적인 측정 기법을 살펴보고 더 빠르고 부드러운 사용자 경험을 만들어 보세요!

먼저 Chrome 개발자 도구를 엽니다. 다시 말해, Google Chrome을 열고 분석하려는 웹페이지로 이동한 후 "F12" 키를 누르거나 페이지를 마우스 오른쪽 버튼으로 클릭하고 "검사"를 클릭하세요. 이제 "성능" 탭으로 이동하면 필요한 모든 것이 있습니다.

이제 Interaction to Next Paint, 즉 INP를 식별할 준비가 되었습니다. 기록된 가장 긴 상호작용 지연을 찾아볼 것입니다. 좋은 INP는 일반적으로 200ms 미만이고, 나쁜 INP는 500ms 이상입니다. 이것으로 끝입니다!
