# --description--

Jetzt, da Sie eine grundlegende Express-Anwendung erstellt haben, ist es Zeit, Routing und seine Funktionsweise genauer zu betrachten.

In Express ist Routing die Art und Weise, wie Sie die Endpunkte einer Webanwendung definieren. Sie können sich Routen als Regeln vorstellen, die der Anwendung sagen, wie sie reagieren soll, wenn ein Benutzer eine bestimmte URL besucht.

Schauen wir uns unsere einfache Express-Anwendung noch einmal an:

```javascript
const express = require("express");
const app = express();
const port = 3000;

app.get("/", (req, res) => {
  res.send("Hello World!");
});

app.listen(port, () => {
  console.log(`Example app listening on port ${port}`);
});
```

In diesem Beispiel definiert die Anwendung eine Route für die Stamm-URL (`/`). Wenn ein Client eine GET-Anfrage an diesen Pfad sendet, antwortet der Server mit dem Text `Hello World!`. Die Anwendung beginnt dann, auf eingehende Anfragen am Port `3000` zu lauschen.

Aber es gibt noch andere Routinenmethoden.

Express ermöglicht es Ihnen, verschiedene Arten von HTTP-Anfragen mit unterschiedlichen Routing-Methoden zu behandeln. Die gebräuchlichsten sind:

* `app.get()` – ruft Daten ab
    
* `app.post()` – erstellt neue Daten
    
* `app.put()` – aktualisiert vorhandene Daten
    
* `app.delete()` – löscht Daten
    

Jede Methode definiert, wie Ihre Anwendung auf eine bestimmte HTTP-Anfrage an einer bestimmten Route reagieren soll.

Hier sind einige einfache Beispiele:

```javascript
app.post("/submit", (req, res) => {
  res.send("Form submitted!");
});

app.put("/update", (req, res) => {
  res.send("Item updated!");
});

app.delete("/delete", (req, res) => {
  res.send("Item deleted!");
});
```

In jedem Fall hört Express auf eine Anfrage mit einer bestimmten HTTP-Methode und URL-Pfad und führt dann die entsprechende Handler-Funktion aus, um eine Antwort an den Client zurückzusenden.

Sie können diese Routen mit Tools wie Postman oder `curl` testen, die es Ihnen ermöglichen, verschiedene Arten von HTTP-Anfragen an Ihren Server zu senden.

Und das ist Routing in Express – es geht einfach darum, zu definieren, wie Ihre Anwendung auf verschiedene Anfragen reagiert.
