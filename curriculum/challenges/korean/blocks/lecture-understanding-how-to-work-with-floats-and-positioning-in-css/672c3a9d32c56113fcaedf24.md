# --interactive--

고정 위치와 스티키 위치는 두 가지 중요한 CSS 위치 지정 전략으로, 절대 위치와 비교했을 때 각각 독특한 동작 방식을 제공합니다. 요소가 `position: fixed`로 위치 지정되면 일반 문서 흐름에서 제거되고 뷰포트를 기준으로 배치되어 사용자가 스크롤해도 같은 위치에 머무릅니다. 이는 항상 보여야 하는 헤더나 내비게이션 바 같은 요소에 자주 사용됩니다. 예를 들어, 제목을 페이지 상단에 고정하려면 다음 코드를 사용할 수 있습니다:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<h1>Fixed Header</h1>

<p>Lorem ipsum dolor sit amet, consectetur adipiscing elit. Integer nec odio. Praesent libero. Sed cursus ante dapibus diam.</p>
<p>Sed nisi. Nulla quis sem at nibh elementum imperdiet. Duis sagittis ipsum. Praesent mauris.</p>
<p>Fusce nec tellus sed augue semper porta. Mauris massa. Vestibulum lacinia arcu eget nulla.</p>
<p>Class aptent taciti sociosqu ad litora torquent per conubia nostra, per inceptos himenaeos.</p>
<p>Curabitur sodales ligula in libero. Sed dignissim lacinia nunc. Curabitur tortor.</p>
<p>Pellentesque nibh. Aenean quam. In scelerisque sem at dolor. Maecenas mattis.</p>
<p>Sed convallis tristique sem. Proin ut ligula vel nunc egestas porttitor. Morbi lectus risus.</p>
<p>Donec congue lacinia dui, a porttitor lectus condimentum laoreet. Nunc eu ullamcorper orci.</p>
<p>Quisque eget odio ac lectus vestibulum faucibus eget in metus. In pellentesque faucibus vestibulum.</p>
<p>Nulla at nulla justo, eget luctus tortor. Nulla facilisi. Duis aliquet egestas purus in blandit.</p>
```

```css
body {
  margin: 0;
  padding-top: 60px;
  font-family: Arial, sans-serif;
  line-height: 1.6;
}

h1 {
  position: fixed;
  top: 0;
  width: 500px;
  background: white;
  padding: 10px;
  border-bottom: 2px solid #ccc;
}

p {
  max-width: 600px;
  margin: 20px auto;
}
```

:::

이 예제에서 `h1` 요소는 뷰포트 상단에 붙어 있으며, 사용자가 스크롤해도 제자리에 남아 있습니다. 이는 스티키 헤더나 항상 보이는 내비게이션 같은 지속적인 UI 요소를 만들 때 특히 유용합니다.

`position: sticky`는 상대 위치와 고정 위치의 하이브리드처럼 동작합니다. 처음에는 요소가 상대 위치처럼 문서 흐름 내에 머무르지만, 사용자가 요소를 특정 지점까지 스크롤하면 뷰포트(보통 상단)에 "붙어서" 고정 위치처럼 동작합니다. 이는 사용자가 특정 위치까지 스크롤할 때만 고정되는 스티키 내비게이션 바 같은 요소를 만들기에 좋습니다. 스티키 위치를 적용하는 방법은 다음과 같습니다:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<h1>Sticky Header</h1>

<p>Lorem ipsum dolor sit amet, consectetur adipiscing elit. Integer nec odio. Praesent libero. Sed cursus ante dapibus diam.</p>
<p>Sed nisi. Nulla quis sem at nibh elementum imperdiet. Duis sagittis ipsum. Praesent mauris.</p>
<p>Fusce nec tellus sed augue semper porta. Mauris massa. Vestibulum lacinia arcu eget nulla.</p>
<p>Class aptent taciti sociosqu ad litora torquent per conubia nostra, per inceptos himenaeos.</p>
<p>Curabitur sodales ligula in libero. Sed dignissim lacinia nunc. Curabitur tortor.</p>
<p>Pellentesque nibh. Aenean quam. In scelerisque sem at dolor. Maecenas mattis.</p>
<p>Sed convallis tristique sem. Proin ut ligula vel nunc egestas porttitor. Morbi lectus risus.</p>
<p>Donec congue lacinia dui, a porttitor lectus condimentum laoreet. Nunc eu ullamcorper orci.</p>
<p>Quisque eget odio ac lectus vestibulum faucibus eget in metus. In pellentesque faucibus vestibulum.</p>
<p>Nulla at nulla justo, eget luctus tortor. Nulla facilisi. Duis aliquet egestas purus in blandit.</p>
```

```css
h1 {
  position: sticky;
  top: 30px;
  left: 30px;
}
```

:::

이 예제에서 `h1` 요소는 사용자가 `30px`만큼 상단에서 스크롤할 때까지는 정상적으로 동작하다가, 그 지점에 도달하면 뷰포트 상단에 "붙습니다".

절대 위치, 고정 위치, 스티키 위치의 주요 차이점은 다음과 같습니다:

- 절대 위치는 요소를 문서 흐름에서 제거하고 가장 가까운 위치 지정 조상이나 없으면 초기 포함 블록을 기준으로 배치합니다. 스크롤과 상관없이 그 위치에 머뭅니다.
- 고정 위치도 요소를 문서 흐름에서 제거하지만 뷰포트를 기준으로 고정하여 페이지가 스크롤되어도 같은 자리에 보입니다.
- 스티키 위치는 처음에는 요소를 정상 흐름에 두지만, 정의된 임계값을 지나 스크롤하면 제자리에 "붙도록" 허용합니다.

이 전략들은 떠다니는 요소, 스티키 헤더, 더 동적인 UI 컴포넌트를 포함하는 레이아웃을 만들 수 있게 하여 중요한 정보를 항상 접근 가능하게 만들어 사용자 경험을 향상합니다.

## --text--

`fixed` 위치 지정과 `sticky` 위치 지정의 주요 차이점은 무엇인가요?
