# --description--

I getter e i setter sono metodi che ti permettono di controllare come gli attributi di una classe vengono consultati e modificati. Con i getter recuperi un valore, e con i setter imposti un valore.

Queste azioni avvengono tramite quelle che sono chiamate proprietà. Sono ciò che collega getter e setter e permettono di consultare i dati.

Le proprietà si comportano come attributi ma agiscono come metodi sotto il cofano. Pensale come dati che definisci come metodi, ma che funzionano come attributi. Questo significa che puoi consultare le proprietà con la notazione a punto invece che con le parentesi tonde.

La cosa principale che fanno le proprietà è eseguire una logica extra dietro le quinte quando ottieni, imposti o elimini valori con esse. Questo le rende la scelta perfetta quando vuoi consultare o manipolare dati all’interno degli oggetti.

Allora perché usare le proprietà per questo invece dei metodi? È soprattutto una questione di leggibilità e convenzione. Rendono il tuo codice più pulito e facile da leggere.

Quando usi un metodo, devi sempre chiamarlo con le parentesi tonde. Ma con una proprietà, puoi consultarla proprio come un normale attributo usando la notazione a punto. Questo fa sembrare il tuo codice semplice anche quando sta facendo lavoro extra dietro le quinte.

Per esempio, potresti voler calcolare un valore o verificare che un nuovo valore sia valido prima di salvarlo. Invece di chiamare un metodo per questo, puoi usare un modo simile a un attributo per farlo.

In Python, un decoratore è una funzione che modifica le funzionalità di altre funzioni o classi senza cambiare il loro codice originale.

Anche se è possibile creare decoratori personalizzati, questo va oltre l’ambito di questa lezione. Sappi solo che per creare una proprietà, definisci un metodo e metti il decoratore `@property` sopra di esso. Questo trasforma il metodo in una proprietà, così può essere consultato come un attributo chiamando internamente il metodo decorato.

Questo ci porta ai getter. Ecco come crearne uno con il decoratore `@property`:

```py
class Circle:
    def __init__(self, radius):
        self._radius = radius

    @property
    def radius(self): # A getter to get the radius
        return self._radius
  
    @property
    def area(self):  # A getter to calculate area
        return 3.14 * (self._radius ** 2)

my_circle = Circle(3)

print(my_circle.radius) # 3
print(my_circle.area) # 28.26
```

Questo esempio ottiene un raggio e l’area di un cerchio.

Nota come abbiamo usato `_radius` invece di radius dentro la classe. L’underscore è una convenzione comune in Python per indicare che un attributo è destinato a essere privato. In altre parole, significa che è per uso interno e non dovrebbe essere consultato direttamente dall’esterno della classe.

Per creare un setter per impostare il raggio, per esempio, devi definire un altro metodo con lo stesso nome e usare `@<property_name>.setter` sopra di esso:

Usare `self.radius` dentro `__init__` assicura che il setter venga eseguito durante la creazione dell’oggetto, così i valori di raggio non validi vengono intercettati immediatamente.

```py
class Circle:
    def __init__(self, radius):
        self.radius = radius # Calling the setter

    @property
    def radius(self):  # A getter to get the radius
        return self._radius

    @radius.setter
    def radius(self, value):  # A setter to set the radius
        if value <= 0:
            raise ValueError('Radius must be positive')
        self._radius = value

my_circle = Circle(3)
print('Initial radius:', my_circle.radius) # Initial radius: 3

my_circle.radius = 8
print('After modifying the radius:', my_circle.radius) # After modifying the radius: 8
```

In questo esempio, il setter del raggio non si limita a impostare il raggio del cerchio, ma esegue anche una validazione che assicura che il raggio non sia un numero negativo.

Una volta definiti getter e setter, Python li chiama automaticamente sotto il cofano ogni volta che usi la sintassi normale degli attributi:

```py
my_circle.radius # This will call the getter
my_circle.radius = 4 # This will call the setter
```

Nota che dentro il setter non puoi usare lo stesso nome della proprietà quando assegni un nuovo valore. Questo perché `self.radius = value` chiamerebbe il setter all’interno del metodo setter stesso, portando a una ricorsione infinita e a un `RecursionError`. Quindi devi sempre usare la forma con prefisso underscore `self._radius = value`.

Proprio come puoi controllare come un attributo viene consultato tramite il getter e come viene modificato con il setter, puoi anche controllare come viene eliminato usando un deleter.

Un deleter esegue una logica personalizzata quando usi l’istruzione del su una proprietà. Per crearne uno, usi il decoratore `@<property_name>.deleter`:

```py
class Circle:
    def __init__(self, radius):
        self.radius = radius

    # Getter
    @property
    def radius(self):
        return self._radius

    # Setter
    @radius.setter
    def radius(self, value):
        if value <= 0:
            raise ValueError("Radius must be positive")
        self._radius = value

    # Deleter
    @radius.deleter
    def radius(self):
        print("Deleting radius...")
        del self._radius
```

Ecco come può essere usato il deleter:

```py
# Create circle object with a radius
my_circle = Circle(33)
print("Initial radius:", my_circle.radius)  # 33

# Delete the radius
# This calls the deleter
del my_circle.radius # Deleting radius...
print("Radius deleted!") # Radius deleted!

# Try to access radius after deletion
try:
    print(my_circle.radius)
except AttributeError as e:
    print("Error:", e) # Error: 'Circle' object has no attribute '_radius'
```

La conclusione è che:

- I getter ti permettono di recuperare un valore o anche di calcolare un valore al volo.
- I setter ti permettono di modificare i valori in modo sicuro eseguendo controlli prima dell’assegnazione.
- Le proprietà sono ciò che lega insieme questi getter e setter così puoi scrivere logica pur usando la notazione a punto.
- I deleter ti permettono di definire cosa succede quando un attributo viene eliminato.
