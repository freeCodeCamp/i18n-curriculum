# --description--

기본 Express 애플리케이션을 만들었으니, 이제 라우팅과 그 작동 방식을 자세히 살펴보세요.

Express에서 라우팅은 웹 애플리케이션의 엔드포인트를 정의하는 방법입니다. 라우트는 사용자가 특정 URL을 방문했을 때 애플리케이션이 어떻게 응답할지 알려주는 규칙이라고 생각할 수 있습니다.

다시 한 번 간단한 Express 애플리케이션을 살펴보겠습니다:

```javascript
const express = require("express");
const app = express();
const port = 3000;

app.get("/", (req, res) => {
  res.send("Hello World!");
});

app.listen(port, () => {
  console.log(`Example app listening on port ${port}`);
});
```

이 예제에서 애플리케이션은 루트 URL(`/`)에 대한 라우트를 정의합니다. 클라이언트가 이 경로로 GET 요청을 보내면 서버는 `Hello World!`라는 텍스트로 응답합니다. 그런 다음 애플리케이션은 포트 `3000`에서 들어오는 요청을 수신 대기합니다.

하지만 다른 라우트 메서드도 있습니다.

Express는 다양한 라우팅 메서드를 사용해 여러 종류의 HTTP 요청을 처리할 수 있게 해줍니다. 가장 일반적인 메서드는 다음과 같습니다:

* `app.get()` – 데이터를 조회합니다
    
* `app.post()` – 새 데이터를 생성합니다
    
* `app.put()` – 기존 데이터를 수정합니다
    
* `app.delete()` – 데이터를 삭제합니다
    

각 메서드는 특정 라우트에서 특정 HTTP 요청에 애플리케이션이 어떻게 응답할지 정의합니다.

몇 가지 간단한 예를 보겠습니다:

```javascript
app.post("/submit", (req, res) => {
  res.send("Form submitted!");
});

app.put("/update", (req, res) => {
  res.send("Item updated!");
});

app.delete("/delete", (req, res) => {
  res.send("Item deleted!");
});
```

각 경우에 Express는 특정 HTTP 메서드와 URL 경로에 대한 요청을 기다렸다가, 해당 핸들러 함수를 실행해 클라이언트에 응답을 보냅니다.

Postman이나 `curl` 같은 도구를 사용해 이러한 라우트를 테스트할 수 있습니다. 이 도구들은 서버에 다양한 종류의 HTTP 요청을 보낼 수 있게 해줍니다.

이것이 바로 Express에서의 라우팅입니다 — 애플리케이션이 다양한 요청에 어떻게 응답할지 정의하는 것입니다.
