# --interactive--

L'aspetto e il comportamento degli elementi input sono importanti quando si costruiscono moduli web. Un modulo con input che rispondono alle azioni dell'utente migliora molto l'esperienza utente e non dovrebbe causare confusione o frustrazione.

CSS offre diverse pseudo-classi per input che possono rendere i tuoi moduli più intuitivi e facili da usare. Vediamo queste pseudo-classi nel dettaglio.

La pseudo-classe `:focus` ti permette di selezionare un elemento input quando un utente lo seleziona o ci clicca sopra, attirando l'attenzione sul campo di input attivo. Questo aiuta gli utenti a identificare facilmente dove stanno digitando in quel momento:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css" />
<form>
  <input type="text" />
</form>
```

```css
input:focus {
  border: 2px solid crimson;
  outline: none;
}
```

:::

Come suggerisce il nome, la pseudo-classe `:hover` si attiva quando l'utente posiziona il cursore su un elemento. Fornisce un feedback visivo prima che l'utente interagisca con l'input, segnalando efficacemente che l'input è pronto per l'azione.

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css" />
<form>
  <input type="text" />
</form>
```

```css
input:hover {
  background-color: orange;
}
```

:::

Nell'esempio sopra, lo sfondo dell'elemento input cambia in arancione quando l'utente ci passa sopra con il cursore, facendogli sapere che il campo è pronto per l'interazione.

La pseudo-classe `:checked` ti consente di definire uno stile diverso per un pulsante di opzione o una casella di spunta quando un utente li seleziona. In questo modo, l'utente può vedere facilmente quale opzione ha scelto.

Ecco un esempio con una casella di spunta per accettare i termini su un sito web.

**NOTA**: Alcune proprietà CSS in questo esempio non sono ancora state trattate. Questo serve solo per darti un'idea di come creare una casella di spunta personalizzata. Imparerai come funziona tutto questo nelle lezioni e workshop futuri.

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css" />
<form>
  <label>
  Agree <input class="checkbox" type="checkbox" />
  </label>
</form>
```

```css
.checkbox {
  appearance: none;
  width: 18px;
  height: 18px;
  border: 2px solid #ccc;
  border-radius: 4px;
  display: inline-block;
  position: relative;
  cursor: pointer;
  transition: all 0.25s ease;
  vertical-align: middle; 
}

.checkbox:hover {
  border-color: #888;
}

.checkbox:checked {
  background-color: #4caf50;
  border-color: #4caf50;
}

.checkbox:checked::after {
  content: "";
  position: absolute;
  left: 4px;
  top: 0px;
  width: 5px;
  height: 10px;
  border: solid white;
  border-width: 0 2px 2px 0;
  transform: rotate(45deg);
}

.checkbox:focus {
  outline: 2px solid #90caf9;
  outline-offset: 2px;
}

```

:::

`:required` seleziona gli elementi input che hanno l'attributo `required`. Indica all'utente che deve compilare il campo per inviare il modulo.

Ecco un esempio con un modulo che ha alcuni campi di input obbligatori:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css" />
<form action="#">
  <label for="name">Name:</label>
  <input type="text" id="name" name="name" required />

  <label for="email">Email:</label>
  <input type="email" id="email" name="email" required />

  <label for="phone">Phone Number:</label>
  <input type="tel" id="phone" name="phone" />

  <button type="submit">Submit</button>
</form>
```

```css
input:required {
  border: 2px solid orange;
}
```

:::

Quando si validano i moduli, puoi usare la pseudo-classe `:valid` per definire lo stile dei campi di input che soddisfano i criteri di validazione, e `:invalid` per quelli che non li soddisfano. Di solito, si usa il verde per un input valido e il rosso per uno non valido.

Ecco un esempio che usa la pseudo-classe `:valid`:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css" />
<form>
  <label for="email">Email:</label>
  <input type="email" id="email" name="email" />
</form> 
```

```css
input:valid {
  border-color: green;
}
```

:::

Ecco un esempio di utilizzo della pseudo-classe `:invalid`:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css" />
<form>  
  <label for="email">Email:</label>
  <input type="email" id="email" name="email" />
</form>
```

```css
input:invalid {
  border-color: crimson;
}
```

:::

La pseudo-classe `:disabled` ti permette di selezionare e definire lo stile degli elementi input disabilitati. Questi elementi hanno l'attributo `disabled`, che impedisce agli utenti di interagire con essi. Quando un input è disabilitato, non può essere cliccato, messo a focus o modificato.

Ecco un esempio di un'etichetta e un elemento input disabilitato.

Nel CSS, stiamo selezionando quell'input disabilitato e gli diamo uno sfondo di colore `lightgray` e cambiamo il cursore da pointer a `not-allowed`.

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css" />
<form>
  <label for="name">Name:</label>
  <input class="text-input" type="text" id="name" name="name" disabled />
</form>
```

```css
.text-input:disabled {
  background-color: lightgray;
  cursor: not-allowed;
}
```

:::

Il valore della proprietà CSS `cursor: not-allowed;` cambia l'aspetto del cursore per indicare che un'azione non è permessa.

Quando viene applicato a un elemento, mostra un cursore con un simbolo di cerchio barrato (di solito un cerchio con una linea diagonale) per segnalare agli utenti che l'elemento non è interattivo o non può essere cliccato o usato.

Ecco altri esempi di pseudo-classi per input e cosa fanno:

- `:autofill`: applica stili ai campi di input che il browser riempie automaticamente con dati salvati.
- `:optional`: definisce lo stile degli elementi input che non sono obbligatori e possono essere lasciati vuoti.
- `:in-range` e `:out-of-range`: definiscono lo stile degli elementi in base al fatto che i loro valori siano entro o fuori dai limiti specificati.
