## Funzioni pure vs impure

- Una funzione pura è quella che produce sempre lo stesso output per lo stesso input e non ha effetti collaterali. Il suo output dipende solo dall’input e non modifica alcuno stato esterno.
- Le funzioni impure hanno effetti collaterali, cioè modifiche allo stato del programma osservabili all’esterno della funzione.

## Programmazione funzionale

- La programmazione funzionale è un approccio allo sviluppo software che enfatizza l’uso delle funzioni per risolvere problemi, concentrandosi su cosa deve essere fatto piuttosto che su come farlo.
- La programmazione funzionale incoraggia l’uso di tecniche che aiutano a evitare effetti collaterali, come l’uso di strutture dati immutabili e funzioni di ordine superiore.
- Se usati correttamente, i principi della programmazione funzionale portano a codice più pulito e più facile da mantenere.

## Currying

- Il currying è una tecnica di programmazione funzionale che trasforma una funzione con più argomenti in una sequenza di funzioni, ciascuna con un solo argomento.

Ecco un esempio di funzione normale rispetto a una funzione curried:

:::interactive_editor

```js
// Regular function
function average(a, b, c) {
  return (a + b + c) / 3;
}

console.log(average(2, 3, 4)); // 3

// Curried function
function curriedAverage(a) {
  return function(b) {
    return function(c) {
      return (a + b + c) / 3;
    };
  };
}

console.log(curriedAverage(2)(3)(4)); // 3
```

:::

- Il currying può essere particolarmente potente quando si lavora con funzioni che prendono molti argomenti.
- Il currying rende il tuo codice più flessibile e più facile da riutilizzare.
- Puoi usare le funzioni freccia per creare funzioni curried in modo più conciso:

```js
const curriedAverage = a => b => c => (a + b + c) / 3;
```

- Sebbene il currying possa portare a codice più flessibile e riutilizzabile, può anche rendere il codice più difficile da leggere se usato eccessivamente.
