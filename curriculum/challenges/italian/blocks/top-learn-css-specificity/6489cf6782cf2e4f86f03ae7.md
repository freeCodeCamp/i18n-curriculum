# --description--

```css
/* rule 1 */
.class.second-class {
  font-size: 12px;
}

/* rule 2 */
.class .second-class {
  font-size: 24px;
}
```

Qui sia la regola 1 che la regola 2 hanno la stessa specificità. La regola 1 usa un selettore concatenato (senza spazio) e la regola 2 usa un combinatore discendente (lo spazio vuoto). Ma entrambe le regole hanno due classi e il simbolo del combinatore di per sé non aggiunge specificità.

```css
/* rule 1 */
.class.second-class {
  font-size: 12px;
}
/* rule 2 */
.class > .second-class {
  font-size: 24px;
}
```

Questo esempio mostra la stessa cosa. Anche se la regola 2 usa un combinatore figlio (`>`), questo non cambia il valore della specificità. Entrambe le regole hanno ancora due classi quindi hanno gli stessi valori di specificità.

*Nota:* Non tutto aggiunge specificità

Quando confronti i selettori, potresti incontrare simboli speciali per il selettore universale (`*`) così come combinatori (`+`, `~`, `>` e uno spazio vuoto). Questi simboli non aggiungono specificità di per sé.
