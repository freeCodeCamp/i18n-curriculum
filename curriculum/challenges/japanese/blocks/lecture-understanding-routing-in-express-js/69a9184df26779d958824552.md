# --description--

静的ファイルの配信はウェブアプリケーションでよくある要件です。静的ファイルには、画像、CSSファイル、クライアントサイドのJavaScript、フォント、その他動的に変化しないリソースが含まれます。

Expressは組み込みの`express.static()`ミドルウェアでこれらのファイルの配信を簡単にします。

Expressにおける静的ファイルの配信とは、サーバーのファイルシステムからクライアントのブラウザへ、追加のサーバー側処理なしにファイルを直接届けることを意味します。

例えば、ブラウザが`/styles.css`、`/script.js`、または`/images/logo.png`のようなファイルをリクエストすると、Expressは指定されたフォルダー内でそれらのファイルを見つけて直接配信できます。ファイルが見つからない場合は、Expressが自動的に404で応答します。これらのファイルはサーバー上で生成や変更が不要なため、*静的*と呼ばれます。

これにより、スタイルシートや画像のようなアセットの配信が簡単になり、各ファイルのためにルートを手動で定義する必要がなくなります。

Expressで静的ファイル配信を設定する方法は次の通りです：

```javascript
const express = require("express");
const path = require("path");
const app = express();
const port = 3000;

// Serve static files from the "public" directory
app.use(express.static(path.join(__dirname, "public")));

app.get("/", (req, res) => {
  res.send("Welcome to the static file demo!");
});

app.listen(port, () => {
  console.log(`Server running at http://localhost:${port}`);
});
```

まず、サーバーを作成するために`express`をインポートし、異なるOS間で安全にファイルパスを構築するために`path`をインポートします。

次に、`app.use(express.static(path.join(__dirname, "public")))`がExpressに`public`フォルダーからファイルを配信するよう指示します。`express.static()`ミドルウェアはExpressに`public`フォルダーからファイルを配信するよう指示します。このディレクトリ内のファイルはすべてブラウザからアクセス可能になります。

また、`GET`のためのシンプルな`/`ルートを定義し、歓迎メッセージを送信します。これにより、静的ファイル配信が動的ルートと共存できることが示されます。

最後に、`app.listen()`がポート3000でサーバーを起動します。サーバーが起動すると、`public`フォルダー内の静的アセットへのリクエストは自動的に処理されます。

これを機能させるには、静的ファイルを保存するための`public`という名前のフォルダーが必要です。典型的なプロジェクト構造は次のようになります：

```md
my-app/
│
├── app.js
└── public/
    ├── index.html
    ├── styles.css
    └── image.png
```

ユーザーが例えば`http://localhost:3000/index.html`にアクセスすると、Expressは`public`ディレクトリ内にリクエストされたファイルが存在するかどうかを確認します。ファイルが存在すれば、Expressはそれを直接ブラウザに送信します。存在しなければ、Expressは次のミドルウェアに進むか404レスポンスを返します。

各ファイルのためにルートを定義する必要はないことを覚えておいてください。したがって、`http://localhost:3000/styles.css`へのリクエストはExpressが`public/styles.css`を探して自動的に応答することを意味します。

静的ファイルの配信方法はカスタマイズも可能です。例えば、複数のディレクトリからファイルを配信したり、キャッシュの動作を設定したりできます。

アプリに次のようなファイル構造があると想像してください：

```md
my-app/
│
├── app.js
│
├── public/
│   ├── index.html
│   └── styles.css
│
└── images/
    └── image.png
```

複数のディレクトリから静的ファイルを配信するために登録する方法は次の通りです：

```javascript
app.use(express.static(path.join(__dirname, "public")));
app.use("/images", express.static(path.join(__dirname, "images")));
```

ここでは前と同じように`app.use(express.static(path.join(__dirname, "public")));`を使っています。つまり、`public`ディレクトリ内のファイルはルートパスから配信されます。

しかし次の行を見ると、`"/images"`を`app.use()`の最初の引数として渡し、その後に`express.static(path.join(__dirname, "images"))`を渡しています。この場合、`"/images"`はマウントパスで、Expressが`/images`へのリクエストを受けたときだけこのミドルウェアを使うよう指示します。そうすると`express.static()`が起動し、`images`ディレクトリ内のファイルを探します。

この設定で、次のリクエストは次のように動作します：

*   GET `http://localhost:3000/styles.css`：Expressは`public`ディレクトリ内で`styles.css`を探し、ファイルを見つけてクライアントに送信します。
    
*   GET `http://localhost:3000/images/image.png`：このリクエストは`/images`で始まるため、Expressは`images`ディレクトリ内で`image.png`を探し、ファイルを見つけてクライアントに送信します。
    

Expressでの静的ファイル配信は簡単で強力です。`express.static()`を使えば、画像、スタイルシート、クライアントサイドJavaScriptのようなアセットを簡単に配信できます。

`public`のような専用フォルダーに静的ファイルをまとめたり、複数のファイルディレクトリやマウントパスを使ったりすることで、アプリケーションを整理しやすく、保守しやすく、拡張しやすくなります。
