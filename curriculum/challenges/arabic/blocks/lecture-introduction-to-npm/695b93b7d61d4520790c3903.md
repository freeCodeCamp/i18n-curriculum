# --description--

عندما تبني مشاريع بلغة JavaScript، غالبًا ما تحتاج إلى أدوات إضافية تساعدك على إنجاز المهام. حتى التطبيقات البسيطة قد تحتاج إلى ميزات مثل حفظ البيانات، تسجيل الدخول، التنقل بين الصفحات، وهكذا.

بدلًا من كتابة كل هذه الشفرة من الصفر، يمكنك استخدام حزم مفتوحة المصدر سبق أن أنشأها وشاركها مطوِّرون آخرون.

هنا يأتي دور npm. npm هو سجل ضخم على الإنترنت للبرمجيات مفتوحة المصدر. فكر فيه كمتجر تطبيقات للشفرة — يمكنك البحث عما تحتاجه، تثبيته، والبدء باستخدامه في مشروعك.

يعتقد كثيرون أن npm تعني "مدير حزم Node"، لكن هذا ليس المعنى الرسمي لها. في وثائق npm القديمة، وُصفت بأنها "اختصار تكراري ساخر لـ 'npm ليست اختصارًا'". بمعنى آخر، npm لا تعني رسميًا أي شيء.

اليوم، تشير npm إلى ثلاثة أجزاء مترابطة:

* موقع الويب npmjs.com، حيث يمكنك البحث عن الحزم في السجل.
    
* سجل npm، وهو قاعدة بيانات عامة كبيرة تخزن كل الحزم.
    
* أداة واجهة سطر الأوامر npm التي تُثبت مع Node.js وتتيح لك تثبيت وإدارة الحزم.
    

على سبيل المثال، إذا كنت تحتاج إلى إدارة الحالة، التعامل مع التواريخ، إرسال رسائل بريد إلكتروني، أو إجراء طلبات HTTP في مشروعك، فغالبًا ما توجد حزمة npm تقوم بذلك بالفعل. باستخدام أمر `npm install` البسيط، يمكنك البدء باستخدام هذا البرنامج الجاهز بدلًا من كتابة كل العمل بنفسك.

تساعدك npm أيضًا في إدارة تبعيات مشروعك. هذا يعني أنها تتعقب الحزم التي تستخدمها في مشاريعك، وإصدار كل منها. وتفعل ذلك عبر ملفي `package.json` و`package-lock.json`، اللذين ستتعرف عليهما في دروس لاحقة.

Yarn وPNPM وBun هي بعض البدائل لـ npm التي أصبحت شائعة مع الوقت. بينما يقدم كل منها فوائد في مجالات محددة مثل سرعة التثبيت وكفاءة التخزين، لا يزال npm هو الأكثر استخدامًا.

في الدروس القادمة، ستتعلم أوامر npm المختلفة، كيفية تثبيت الحزم، عن ملفي `package.json` و`package-lock.json`، وحتى كيفية نشر حزمة خاصة بك في سجل npm.
