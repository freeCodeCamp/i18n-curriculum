# --description--

У минулому уроці ви дізналися про мемоізацію та як працює хук `useMemo`.

У цьому уроці ви дізнаєтеся, як працюють хук `useCallback` і `React.memo`.

У минулому уроці ми також згадували, що `useCallback` призначений для мемоізації референсів функцій.

Для `React.memo` він дозволяє мемоізувати компонент, щоб уникнути непотрібних повторних відтворень, коли його проп не змінився.

Ось базовий синтаксис хука `useCallback`:

```js
const handleClick = useCallback(() => {
  // code goes here
}, [dependency]);
```

А ось базовий синтаксис `React.memo`:

```jsx
const MemoizedComponent = React.memo(({ prop }) => {
  return (
    <>
      {/* Presentation */}
    </>
  )
});
```

Розглянемо приклад хука `useCallback`:

```jsx
import { useState, useEffect } from "react";

function Counter() {
  const [count, setCount] = useState(0);

  const handleClick = () => {
    setCount((prevCount) => prevCount + 1);
  };

  useEffect(() => {
    console.log("useEffect runs");
  }, [handleClick]);

  return (
    <div>
      <p>Count: {count}</p>
      <button onClick={handleClick}>Increment</button>
    </div>
  );
}

export default Counter;
```

У компоненті ефект запускається щоразу, коли змінюється `handleClick`, тому що функція `handleClick` створюється заново при кожному відтворенні.

Щоб це виправити, потрібно сказати React, що функція `handleClick` має залишатися тією ж самою між відтвореннями, мемоізувавши її за допомогою хука `useCallback`, щоб вона не створювалася заново:

```jsx
import { useState, useEffect, useCallback } from "react";

function Counter() {
  const [count, setCount] = useState(0);

  // Memoize the handleClick function with useCallback
  const handleClick = useCallback(() => {
    setCount((prevCount) => prevCount + 1);
  }, []);

  useEffect(() => {
    console.log("useEffect runs");
  }, [handleClick]);

  return (
    <div>
      <p>Count: {count}</p>
      <button onClick={handleClick}>Increment</button>
    </div>
  );
}

export default Counter;
```

Тепер функція `handleClick` не створюється заново при кожному відтворенні.

Щоб показати, як працює функція вищого порядку `React.memo` (або `memo`) і хук `useCallback` разом, ось компонент `Counter` з функцією `handleClick`, яка потребує `useCallback`, але наразі її не використовує:

```jsx
import { useState, useEffect, useCallback } from "react";
import CounterChild from "./CounterChild";

function Counter() {
  const [count, setCount] = useState(0);
  const [timer, setTimer] = useState(new Date().toLocaleTimeString());

  const handleClick = () => {
    setCount(count + 1);
  };

  useEffect(() => {
    const interval = setInterval(() => {
      setTimer(new Date().toLocaleTimeString());
    }, 1000);

    return () => clearInterval(interval);
  }, []);

  return (
    <div>
      <h1>Time: {timer}</h1>
      <p>Count: {count}</p>
      <button onClick={handleClick}>Increment</button>
      <CounterChild onClick={handleClick} />
    </div>
  );
}

export default Counter;
```

У цій функції також є таймер у стані, який оновлюється щосекунди. Це змушує компонент повторно відтворюватися щоразу, коли змінюється `timer`, через що функція `handleClick` створюється заново при кожному відтворенні.

Ось чому функцію `handleClick` потрібно мемоізувати за допомогою `useCallback`.

Ось компонент `CounterChild`:

```jsx
const CounterChild = ({ onClick }) => {
  console.log("CounterChild component rendered");
  return <button onClick={onClick}>Increment from Child</button>;
};

export default CounterChild;
```

Цей компонент `CounterChild` приймає проп `onClick`, що дає змогу також збільшувати фішку з нього.

Оскільки компонент `CounterChild` є дочірнім елементом компонента `Counter`, він також відтворюється щоразу, коли `Counter` повторно відтворюється через змінний таймер. Тож `CounterChild` також потрібно мемоізувати.

Без мемоізації, оскільки компонент повторно відтворюється через оновлення таймера щосекунди, компонент `CounterChild` також повторно відтворюється.

Щоб цього уникнути, потрібно мемоізувати компонент `CounterChild` за допомогою `React.memo`:

```jsx
import React from "react";

const CounterChild = React.memo(({ onClick }) => {
  console.log("CounterChild component rendered");
  return <button onClick={onClick}>Increment from Child</button>;
});

export default CounterChild;
```

Навіть після мемоізації `CounterChild` за допомогою `React.memo` речі працюють не оптимально.

Це відбувається тому, що функція `handleClick` створюється заново при кожному відтворенні, тому її також потрібно мемоізувати за допомогою `useCallback`, щоб сказати React, що функція має залишатися тією ж самою між відтвореннями:

```js
const handleClick = useCallback(() => {
  setCount((prevCount) => prevCount + 1);
}, [count]);
```

Тепер компонент повторно відтворюється лише тоді, коли змінюється стан `count`.
