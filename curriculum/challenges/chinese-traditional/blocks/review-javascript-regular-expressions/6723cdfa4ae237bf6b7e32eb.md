## 正規表示式與常用方法

- **定義**：正規表示式（Regular Expressions，簡稱 Regex）用來建立一個「樣式」，你可以用這個樣式來檢查字串、擷取文字等。

```js
const regex = /freeCodeCamp/;
```

- **`test()` 方法**：這個方法接受一個字串，這個字串就是要用正規表示式測試是否符合的內容。這個方法會傳回一個布林值，表示該字串是否符合正規表示式。

:::interactive_editor

```js
const regex = /freeCodeCamp/;
const test = regex.test("e");
console.log(test); // false
```

:::

- **`match()` 方法**：這個方法接受一個正規表示式，你也可以傳入一個字串，該字串會被轉換成正規表示式。`match` 方法會傳回該字串的符合項目陣列。

```js
const regex = /freeCodeCamp/;
const match = "freeCodeCamp".match(regex);
console.log(match); // ['freeCodeCamp', index: 0, input: 'freeCodeCamp', groups: undefined]
```

- **`replace()` 方法**：這個方法接受兩個引數：要比對的正規表示式（或字串），以及要用來取代符合項目的字串（或針對每個符合項目執行的函式）。

:::interactive_editor

```js
const regex = /Jessica/;
const str = "Jessica is rly kewl";
const replaced = str.replace(regex, "freeCodeCamp");
console.log(replaced); // "freeCodeCamp is rly kewl"
```

:::

- **`replaceAll` 方法**：這個方法用來將指定樣式的所有出現位置都取代成新的字串。如果你給它一個沒有全域修飾詞的正規表示式，這個方法會丟出錯誤。

:::interactive_editor

```js
const text = "I hate JavaScript! I hate programming!";
const newText = text.replaceAll("hate", "love");
console.log(newText);  // "I love JavaScript! I love programming!"
```

:::

- **`matchAll` 方法**：這個方法用來擷取字串中所有符合指定正規表示式的項目（包含擷取群組），並以迭代器的形式傳回。迭代器是一種可以讓你逐一走訪（或「迭代」）一個項目群集的物件。

:::interactive_editor

```js
const str = "JavaScript, Python, JavaScript, Swift, JavaScript";
const regex = /JavaScript/g;

const iterator = str.matchAll(regex);

for (let match of iterator) {
  console.log(match[0]); // "JavaScript" for each match
}
```

:::

## 正規表示式修飾詞

- **定義**：修飾詞（Modifiers），又稱為「旗標」，用來改變正規表示式的行為。
- **`i` 旗標**：這個旗標讓正規表示式忽略大小寫。

:::interactive_editor

```js
const regex = /freeCodeCamp/i;
console.log(regex.test("freecodecamp")); // true
console.log(regex.test("FREECODECAMP")); // true
```

:::

- **`g` 旗標**：這個旗標，也就是全域修飾詞，讓你的正規表示式可以多次比對樣式。

:::interactive_editor

```js
const regex = /freeCodeCamp/gi;
console.log(regex.test("freeCodeCamp")); // true
console.log(regex.test("freeCodeCamp is great")); // false
```

:::

- **錨點定義**：`^` 錨點放在正規表示式的開頭，表示「比對字串的開頭」；`$` 錨點放在正規表示式的結尾，表示「比對字串的結尾」。

:::interactive_editor

```js
const start = /^freeCodeCamp/i;
const end = /freeCodeCamp$/i;
console.log(start.test("freecodecamp")); // true
console.log(end.test("freecodecamp")); // true
```

:::

- **`m` 旗標**：錨點會尋找整個字串的開頭和結尾。但你可以用 `m` 旗標（多行修飾詞）讓正規表示式處理多行內容。

:::interactive_editor

```js
const start = /^freecodecamp/im;
const end = /freecodecamp$/im;
const str = `I love 
freecodecamp
it's my favorite
`;
console.log(start.test(str)); // true
console.log(end.test(str)); // true
```

:::

- **`d` 旗標**：這個旗標會擴充你在符合物件中取得的資訊。

```js
const regex = /freecodecamp/di;
const string = "we love freecodecamp isn't freecodecamp great?";
console.log(string.match(regex));
```

結果如下：

```js
// [
//   'freecodecamp',
//   index: 8,
//   input: "we love freecodecamp isn't freecodecamp great?",
//   groups: undefined,
//   indices: [
//     0: [8, 20],
//     groups: undefined
//   ]
// ]
```

- **`u` Flag**：這個旗標擴展了正規表達式的功能，讓它能匹配特殊的 unicode 字元。`u` 旗標讓你可以使用像 `Extended_Pictographic` 這樣的特殊類別，以匹配大部分的表情符號。還有一個 `v` 旗標，進一步擴展了 unicode 匹配的功能。
- **`y` Flag**：sticky 修飾詞的行為與 global 修飾詞非常相似，但有一些例外。最大差異是 global 正規表達式會從 lastIndex 開始，搜尋字串剩餘部分的所有匹配；而 sticky 正規表達式如果在先前的 lastIndex 位置沒有立即匹配，會傳回 null 並將 lastIndex 重設為 0。
- **`s` Flag**：single-line 修飾詞允許正規表達式中的萬用字元（用 `.` 表示）匹配換行字元——實際上將字串視為單行文字。

## 字元類別

- **萬用字元 `.`**：字元類別是一種特殊語法，可以用來比對一組或子集的字元。你應該先學會的第一個字元類別是萬用字元類別。萬用字元用一個句點（dot）表示，可以比對任何單一字元，除了換行字元。如果你想讓萬用字元類別也能比對換行字元，記得要加上 `s` 旗標。

```js
const regex = /a./;
```

- **`\d`**：這會比對字串中的所有數字（`0-9`）。

```js
const regex = /\d/;
```

- **`\w`**：用來匹配字串中的任何字元字元（`a-zA-Z0-9_`）。字元字元定義為任何字母（a 到 z 或 A 到 Z）、數字（0 到 9）或底線字元。

```js
const regex = /\w/;
```

- **`\s`**：空白字元類別 `\s` 是由反斜線後接 `s` 表示。這個字元類別會匹配任何空白字元，包括換行、空格、製表符和特殊的 Unicode 空白字元。
- **否定特殊字元類別**：要否定這些字元類別，可以將反斜線後的小寫字母改成大寫字母。以下範例不匹配數字字元，而是匹配任何非數字字元的單一字元。

```js
const regex = /\D/;
```

- **自訂字元類別**：你可以將想匹配的字元放在一組方括號內，來建立自訂的字元類別。

```js
const regex = /[abcdf]/;
```

## 前瞻與後顧斷言

- **定義**：前瞻（lookahead）和後顧（lookbehind）斷言讓你可以根據前後是否有特定樣式來比對某個樣式。
- **正向前瞻斷言**：這種斷言會在某個樣式後面還有另一個樣式時才比對。要建立正向前瞻，先寫你想比對的樣式，然後用小括號包住你想當作條件的樣式。在開括號後面加上 `?=`，就能定義這個條件為正向前瞻。

```js
const regex = /free(?=code)/i;
```

- **負向前瞻斷言**：這是一種在正規表示式中用來檢查某個樣式後面沒有出現特定樣式的條件。

```js
const regex = /free(?!code)/i;
```

- **正向後顧斷言**：這種斷言只會在某個樣式前面有另一個特定樣式時才比對，而且不會把前面的樣式包含在比對結果裡。

```js
const regex = /(?<=free)code/i;
```

- **負向後顧斷言**：這種斷言確保某個樣式前面沒有另一個特定樣式。只有在指定樣式前面沒有出現給定序列時才會比對，而且不會把前面的序列包含在比對結果裡。

```js
const regex = /(?<!free)code/i;
```

## 正規表示式量詞

- **定義**：正規表示式中的量詞用來指定某個樣式（或樣式的一部分）應該出現幾次。它們可以幫你控制比對時字元或群組出現的次數。以下範例會比對前一個字元恰好出現四次。

```js
const regex = /^\d{4}$/;
```

- **`*`** ：比對前一個元素出現 0 次或多次。
- **`+`**：比對前一個元素出現 1 次或多次。
- **`?`**：比對前一個元素出現 0 次或 1 次。
- **`{n}`**：比對前一個元素恰好出現 n 次。
- **`{n,}`**：比對前一個元素出現 n 次或更多次。
- **`{n,m}`**：比對前一個元素出現 n 到 m 次。

## 擷取群組與反向參照

- **擷取群組**：擷取群組讓你可以「擷取」比對到的字串片段，方便你後續使用。擷取群組用小括號包住要擷取的樣式，前面不用加像前瞻那樣的特殊字元。

```js
const regex = /free(code)camp/i;
```

- **非捕獲群組**：非捕獲群組類似於捕獲群組，但不會儲存字串中符合的部分以供後續使用。非捕獲群組由 `(?:...)` 定義。

```js
const regex = /free(?:code)camp/i;
```

- **反向參照**：正規表示式中的反向參照是一種重複利用先前比對到的樣式的方法。它讓你可以用數字來指向某個擷取群組（也就是用括號包住的樣式）。例如，`$1` 代表第一個擷取群組。

:::interactive_editor

```js
const regex = /free(co+de)camp/i;
console.log("freecoooooooodecamp".replace(regex, "paid$1world"));
```

:::

- 你可以在正規表示式本身內使用反向引用，以反斜線和捕獲群組編號來匹配先前群組所捕獲的相同文字。例如：

:::interactive_editor

```js
const regex = /(hello) \1/i;
console.log(regex.test("hello hello")); // true
console.log(regex.test("hello world")); // false
```

:::
