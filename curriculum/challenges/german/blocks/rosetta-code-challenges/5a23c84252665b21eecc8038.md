# --description--

Subleq ist ein Beispiel für einen One-Instruction Set Computer (OISC).

Der Name leitet sich von seiner einzigen Anweisung ab, die **SU**btrahieren und **B**edingt **L**eiser oder **EQ**ual (kleiner oder gleich null) verzweigen heißt.

Ihre Aufgabe ist es, einen Interpreter zu erstellen, der eine solche Maschine emuliert.

Der Speicher der Maschine besteht aus einem Array von vorzeichenbehafteten ganzen Zahlen. Jede vernünftige Wortgröße ist geeignet, aber der Speicher muss sowohl negative als auch positive Zahlen speichern können.

Die Ausführung beginnt mit dem Befehlszeiger, der auf das erste Wort zeigt, also Adresse 0. Sie verläuft wie folgt:

<ol>
  <li>Seien A, B und C die Werte, die in den drei aufeinanderfolgenden Speicherstellen ab dem Befehlszeiger gespeichert sind.</li>
  <li>Bewegen Sie den Befehlszeiger um 3 Wörter weiter, sodass er auf die Adresse nach derjenigen zeigt, die C enthält.</li>
  <li>Wenn A -1 ist, wird ein Zeichen von der Standardeingabe gelesen und dessen Codepunkt an der Adresse gespeichert, die B angibt. C wird nicht verwendet.</li>
  <li>Wenn B -1 ist, wird die Zahl an der Adresse, die A angibt, als Codepunkt interpretiert und das entsprechende Zeichen ausgegeben. C wird ebenfalls nicht verwendet.</li>
  <li>Andernfalls werden sowohl A als auch B als Adressen von Speicherstellen behandelt. Die Zahl an der Adresse A wird von der Zahl an der Adresse B subtrahiert (und das Ergebnis an Adresse B zurückgeschrieben). Wenn das Ergebnis null oder negativ ist, wird der Wert C zum neuen Befehlszeiger.</li>
  <li>Wenn der Befehlszeiger negativ wird, stoppt die Ausführung.</li>
</ol>

Andere negative Adressen außer -1 können entweder wie -1 behandelt werden oder einen Fehler auslösen, je nachdem, wie Sie es umsetzen.

Ihre Lösung sollte ein Programm akzeptieren, das auf der Maschine ausgeführt wird, getrennt von der Eingabe, die an das Programm selbst übergeben wird.

Dieses Programm sollte roher Subleq-"Maschinencode" sein – dezimale Zahlen, getrennt durch Leerzeichen, ohne symbolische Namen oder andere Assemblerebene-Erweiterungen, die ab Adresse 0 in den Speicher geladen werden. Zeigen Sie die Ausgabe Ihrer Lösung, wenn Sie dieses "Hello, world!"-Programm ausführen. (Beachten Sie, dass das Beispiel ASCII oder eine Obermenge davon annimmt, wie eines der Latin-N-Zeichensätze oder Unicode. Sie können es in einen anderen Zeichensatz übersetzen, wenn Ihre Implementierung in einer nicht ASCII-kompatiblen Umgebung läuft.)

<pre>15 17 -1 17 -1 -1 16 1 -1 16 3 -1 15 15 0 0 -1 72 101 108 108 111 44 32 119 111 114 108 100 33 10 0</pre>

Das entspricht ungefähr folgendem in einer hypothetischen Assemblersprache:

<pre>start:
    zero, message, -1
    message, -1, -1
    neg1, start+1, -1
    neg1, start+3, -1
    zero, zero, start
zero: 0
neg1: -1
message: "Hello, world!\n\0"
</pre>

```js
function Subleq(mem) {
  var out = '';
  var instructionPointer = 0;
  do {
    var a = mem[instructionPointer];
    var b = mem[instructionPointer + 1];
    if (a === -1) {
    } else if (b === -1) {
      out += String.fromCharCode(mem[a]);
    } else {
      mem[b] -= mem[a];
      if (mem[b] < 1) {
        instructionPointer = mem[instructionPointer + 2];
        continue;
      }
    }
    instructionPointer += 3;
  } while (instructionPointer >= 0);

  return out;
}
```
