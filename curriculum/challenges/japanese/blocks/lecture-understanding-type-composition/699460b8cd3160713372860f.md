# --description--

時には、TypeScript よりも型について詳しく知っている場合があります。そんなときに型アサーションが役立ちます。

ここでは、`querySelector()` メソッドを使って `id` の `submit` をアクセスする例を示します。

```ts
const submitBtn = document.querySelector("#submit");
```

現在、TypeScript はこの `submitBtn` が何らかの `Element` またはおそらく `null` であることしか認識していません。TypeScript は対応する HTML を見て判断しません。

ここで型アサーションを使って、この要素がどの型であるかを TypeScript に伝えるのが良い方法です。例を示します。

```ts
const submitBtn = document.querySelector("#submit") as HTMLButtonElement;
```

この型アサーションのおかげで、TypeScript はこれを `button` 要素として扱います。

ただし、`as` キーワードの使用には制限があります。例えば、TypeScript がエラーをスローするような次のようなことはできません。

```ts
// Conversion of type 'string' to type 'number' may be a mistake because neither type sufficiently overlaps with the other. If this was intentional, convert the expression to 'unknown' first.
const age = "awesome" as number;
```

この例では、文字列 `"awesome"` を数値型として扱おうとするのはあまり意味がありません。TypeScript は互換性のある型や重なり合う型の間でのみ型アサーションを許可しています。

このエラーを回避する方法の一つは、次のような二重アサーションを使うことです。

```ts
const age = "awesome" as unknown as number;
```

TypeScript ではこれが動作しますが、論理的にはあまり意味がないため推奨されません。

型アサーションを書く別の方法として、次のような山括弧構文を使う方法があります。

```ts
const submitBtn = <HTMLButtonElement>document.querySelector("#submit");
```

`tsx` ファイルでこの構文を使うと、TypeScript はコンポーネントをレンダリングしようとしていると誤解し、次のようなエラーをスローするので注意が必要です。

```ts
// file: index.tsx
// This JSX tag requires 'React' to be in scope, but it could not be found.
// JSX element 'HTMLButtonElement' has no corresponding closing tag.
const submitBtn =  <HTMLButtonElement>document.querySelector("#submit");
```

ほとんどの場合、`as` キーワードを使った最初の構文が2番目の選択肢よりもよく使われます。
