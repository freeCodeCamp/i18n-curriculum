# --description--

Expressアプリを安定して本番環境で使える状態に保つには、特に負荷が高い時やシャットダウン時にどうすればいいか気になったことはありませんか？

そこで役立つのがヘルスチェックとグレースフルシャットダウンという2つのパターンです。これらはアプリの稼働を維持し、リクエストを落としたりデータを壊したりせずにきれいに終了させるのに役立ちます。

詳しく見ていきましょう。

まず、Expressでいう「ヘルスチェック」とは何でしょうか？

ヘルスチェックは、アプリが起動して動いているかどうかを教えてくれるシンプルなルートです。

例えば：

```js
app.get('/health', (req, res) => {
  res.status(200).send('OK')
})
```

ロードバランサーやコンテナオーケストレーター（KubernetesやAWS Elastic Beanstalkなど）がこのルートにpingを送って、アプリが健康かどうか判断できます。

もしこのルートが200ステータスコードを返せば、サービスは正常とみなされます。ちなみに200のHTTPステータスコードは「問題なし」を意味します！

応答がなければ、オーケストレーターは自動的にサービスを再起動できます。

では「グレースフルシャットダウン」とは何でしょう？

グレースフルシャットダウンは、サーバーの再起動やデプロイなどでアプリが停止しようとするときに行われる処理です。

接続をすぐに切るのではなく、アプリが今やっている処理を終わらせる時間を与えます。

Expressでは`SIGTERM`という、プロセスが終了すべきときに送られるシステム信号を監視できます。

例を見てみましょう：

```js
process.on('SIGTERM', () => {
  server.close(() => {
    console.log('Server closed. Cleaning up...')
    // Close DB connections or other cleanup here
  })
})
```

これはExpressに新しい接続を受け付けるのをやめ、既存の接続が終わるのを待ってから終了するよう指示します。

また、`SIGINT`も処理すべきです。これはローカル開発中にCtrl+Cを押してアプリを停止するときに送られる信号です。

`SIGTERM`は通常Kubernetesのようなオーケストレーターが本番環境で使いますが、`SIGINT`は日常の開発者の作業フローをカバーします。これを処理しないと、Ctrl+Cでプロセスが即座に終了し、設定したクリーンアップ処理がスキップされてしまいます。

両方の信号は同じグレースフルシャットダウンの動作を起こすべきなので、開発者は両方を監視することが多いです。

では、これらをまとめてみましょう。

アプリがコンテナで動いているとします。Kubernetesがアップデートのために`SIGTERM`を送ってシャットダウンを指示します。

基本的なグレースフルシャットダウンでは、アプリは新しい接続を受け付けるのをやめ、現在のリクエストを終わらせます。すべての処理が終わったらデータベースから切断し、きれいに終了します。データの損失も状態の破損もありません。

しかし問題があります。ロードバランサーはアプリがシャットダウン中だと知らないため、新しいトラフィックを送り続けるかもしれません。そこで次のボーナスパターンが役立ちます。

ここでボーナスとして「シャットダウン前にアプリを不健康としてマークする」方法を紹介します。

シャットダウン前にヘルスチェックのレスポンスを503ステータスに更新できます。そうすればロードバランサーは不健康なレスポンスを見て、シャットダウン中は新しいトラフィックをアプリに送るのをやめます。上のコード例で示した通りです。

補足：`503`のステータスコードは「サービス利用不可」を意味します。これはサーバーが一時的にリクエストを処理できないことを示し、まさにグレースフルシャットダウン中の状態を表しています。

```js
let isShuttingDown = false

app.get('/health', (req, res) => {
  if (isShuttingDown) {
    res.status(500).send('Shutting down')
  } else {
    res.status(200).send('OK')
  }
})

process.on('SIGTERM', () => {
  isShuttingDown = true

  server.close(() => {
    console.log('Graceful shutdown complete')
    // Perform cleanup
  })
})
```
