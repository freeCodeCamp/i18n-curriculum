# --description--

误差的另一个重要部分是栈追踪。这有助于你了解误差在你的应用中何时被抛出，以及导致该误差的函数调用。因此，例如，如果你有以下代码：

```javascript
const a = 5;
const b = 10;

function add() {
  return c;
}

function print() {
  add();
}

print();
```

我们的函数 `print()` 应该调用 `add()`，该函数返回一个名为 `c` 的变量，而该变量目前尚未声明。对应的误差如下：

<img src="https://cdn.statically.io/gh/TheOdinProject/curriculum/284f0cdc998be7e4751e29e8458323ad5d320303/foundations/javascript_basics/understanding_errors/imgs/01.png" width="100%" alt="An error showing with a stacktrace in the developer console">

栈追踪告诉我们：

1. `add()` 第 5 行声明的作用域中未定义 `c`。
1. `add()` 是由在第 9 行声明的 `print()` 调用的。
1. `print()` 本身在第 12 行被调用。

因此，栈追踪让你能够追踪误差的发展直到其起源，这里是 `add()` 的声明。
