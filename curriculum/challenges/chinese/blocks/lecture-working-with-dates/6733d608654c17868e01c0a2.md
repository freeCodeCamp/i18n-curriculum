# --interactive--

在上一课中，你学习了如何使用 JavaScript 中的 `Date` 对象。但在 JavaScript 中有几种不同的方式来格式化日期。

在本课中，我们将学习如何使用 `toISOString()`、`toString()` 和 `toLocaleDateString()` 方法来格式化 JavaScript 中的日期。

在我们查看不同的方法之前，先回顾一下 `Date` 对象的样子：

:::interactive_editor

```js
const date = new Date();
console.log(date);
```

:::

当你将 `date` 对象记录到控制台时，你将看到基于用户系统设置的当前日期和时间。以下是你可能看到的输出示例：

```js
Sun Sep 29 2024 19:45:37 GMT-0700 (Pacific Daylight Time)
```

如果你对 `date` 对象使用 `toString()` 方法，你将看到与上面相同的输出。

要以扩展的 ISO 格式（ISO 8601）格式化日期，你可以像这样使用 `toISOString()` 方法：

:::interactive_editor

```js
const date = new Date();
console.log(date.toISOString());
```

:::

ISO 8601 是一个用于表现日期和时间的国际标准。格式为 `YYYY-MM-DDTHH:mm:ss.sssZ`。

以下是你可能看到的输出示例：

```js
2024-09-30T02:47:20.292Z
```

另一种格式化日期的方法是使用 `toLocaleDateString()` 方法。此方法允许你根据用户的局部格式化日期。以下是使用 `toLocaleDateString()` 方法的基本语法：

:::interactive_editor

```js
const date = new Date();
console.log(date.toLocaleDateString());
```

:::

以下是你可能看到的输出示例：

```js
9/29/2024
```

`toLocaleDateString()` 方法接受两个可选的参数：`locales` 和 `options`。

`locales` 参数是一个表现要使用的区域设置的字串。例如，你可以传入 `en-US` 表示英语（美国）或 `fr-FR` 表示法语（法国）。如果你不传入 `locales` 参数，则使用默认区域设置。以下是如何使用 `locales` 参数的示例：

:::interactive_editor

```js
const date = new Date();
console.log(date.toLocaleDateString("fr-FR"));
```

:::

第二个可选的参数是 `options` 参数。该参数是一个对象，允许你指定日期字串的形式。以下是如何使用 `options` 参数的示例：

:::interactive_editor

```js
const date = new Date();
const options = {
  weekday: "long",
  year: "numeric",
  month: "long",
  day: "numeric",
};
console.log(date.toLocaleDateString("en-GB", options));
```

:::

在上述示例中，我们指定了选项为英语（英国）并包含完整的星期几、年份、月份和日期。 

以下是你可能看到的输出示例：

```js
Sunday, September 29, 2024
```

在本课中，我们只介绍了在 JavaScript 中格式化日期的几种方式。还有许多其他方法和库可以帮助你格式化 JavaScript 中的日期。但 `toISOString()` 和 `toLocaleDateString()` 方法是格式化 JavaScript 日期的良好起点。
