# --description--

陣列是計算機科學中一種基本的資料結構。所有陣列都儲存有序的資料群集，但根據它們的型別，背後的運作方式可能不同。

它們的底層行為可能對程式的效率產生重要影響，因此讓我們來了解動態陣列和靜態陣列及其差異，這樣你就能為你的程式選擇最高效的陣列。

我們將從靜態陣列開始。

**靜態陣列** 有固定大小。它們將元素儲存在相鄰的記憶體位置。

靜態陣列的大小是在陣列初始化時決定的。一旦該特定的記憶體區段被配置，它就是固定的，且在程式執行期間無法更改。這是靜態陣列的一個關鍵特性。

將元素儲存在相鄰的記憶體位置，使資料擷取處理程序更高效，因為程式可以儲存第一個元素的位置，然後使用索引進行簡單計算以找到記憶體中的其他元素。

多虧了這個，存取靜態陣列的值需要恆定時間 `O(1)`，這非常高效。

當你事先知道將要儲存的元素數量時，可以使用靜態陣列。當值將被非常頻繁地存取時，這也很有幫助，因為存取操作非常高效。

然而，這個資料結構無法擴展或縮減，所以如果將要儲存的元素數量會變動，你應該改用動態陣列。

嘗試增加靜態陣列的大小會涉及建立一個新的陣列，並將所有元素從舊陣列複製到新的陣列，這樣做效率不高。在這種情況下，動態陣列會更好，因為它會自動控制代碼這個處理程序。

JavaScript 沒有像 C 或 Java 那樣的傳統低階靜態陣列，但你可以使用內建的 `Array` 物件來模擬固定大小的陣列：

```js
let numbers = new Array(3); // fixed size of 3
numbers[0] = 10;
numbers[1] = 20;
numbers[2] = 30;
```

雖然 JavaScript 允許調整大小，但在概念上這表述為固定長度的陣列。

JavaScript 中的陣列預設是動態的，所以讓我們來看看那些。

**動態陣列** 更具彈性，因為它們可以在程式執行時自動增長或縮小。

它們透過一個自動調整大小的機制運作，當原始陣列已滿時，會將元素複製到新的陣列。這個處理程序是高效完成的，因為新陣列的大小是以一種高效的方式選擇，使這些計算上昂貴的操作行為較不頻繁。

存取動態陣列的元素需要恆定時間 `O(1)`，所以這個操作非常高效。

在陣列中間插入元素需要線性時間 `O(n)`，因為其後的元素需要重新定位。

如果動態陣列中仍有空間，將元素插入陣列末端需要恆定時間 `O(1)`，但如果陣列已滿且需要調整大小，這個操作行為的時間複雜度為 `O(n)`。

當你事先不知道需要在陣列中儲存多少個值時，你應該使用動態陣列。當你會頻繁插入和刪除元素時，它們也很有幫助。

要在 JavaScript 中創建一個陣列，你需要將元素寫在方括號內，並以逗號分隔：

```js
let numbers = [3, 4, 5, 6];
```

這會創建一個具有四個元素的動態陣列。

你可以透過寫出變數名稱，後面接著包含索引的方括號，來存取陣列中的元素。

索引從第一個元素的 0 開始，並且每個下一個元素增加 1。

```js
numbers[0]; // 3
numbers[1]; // 4
numbers[2]; // 5
numbers[3]; // 6
```

要更新一個值，你只需要重新指定它：

```js
numbers[2] = 16
```

現在，陣列變成 `[3, 4, 16, 6]`。

要將元素添加到陣列的末端，請使用 `.push()` 方法：

```js
numbers.push(7)
```

現在陣列變成 `[3, 4, 16, 6, 7]`：

要在特定索引插入元素，JavaScript 沒有像 Python 的 `.insert()`，但你可以使用 `.splice()` 方法。語法如下：

```js
array.splice(index, deleteCount, newElement)
```

這裡有一個範例：

```js
numbers.splice(3, 0, 15);
```

這個範例表示：

- 從索引 3 開始
- 移除 0 個元素
- 插入 15

如果陣列是 `[3, 4, 16, 6, 7]`，它現在變成 `[3, 4, 16, 15, 6, 7]`。

你可以使用相同的 `.splice` 方法在特定索引移除元素：

```js
numbers.splice(2, 1);
```

這表示從索引 2 開始並移除 1 個元素。

要移除最後一個元素，請使用 `.pop()` 方法：

```js
numbers.pop();
```

還有大量其他內建的陣列方法，你可以在 JavaScript 中用於操作陣列的文件中查看。

這就是動態陣列的威力。

一般來說，當你事先知道元素數量且需要頻繁存取時，應該使用靜態陣列；當元素數量未知或隨時間變動時，則應使用動態陣列。

你應該始終考慮靜態陣列的簡單性與動態陣列的彈性之間的權衡。它們兩者對特定的使用案例和情境都很有幫助。能夠為特定問題選擇最適合的那一種，是你隨著練習逐漸培養的解決問題技能的一部分。
