# --description--

تسمح لنا المركبات بدمج محددات متعددة بطريقة مختلفة عن التجميع أو التسلسل، إذ تُظهر علاقة بين المحددات. هناك أربعة أنواع من المركبات إجمالًا، لكن الآن سنعرض فقط المركب التابع، الذي يُمثَّل في CSS بمسافة واحدة بين المحددات. المركب التابع يختار العناصر التي تطابق المحدد الأخير فقط إذا كان لها عنصر أصل (والد، جد، وهكذا) يطابق المحدد السابق.

لذا، شيء مثل `.ancestor .child` سيختار عنصرًا يحمل الصنف `child` إذا كان له أصل يحمل الصنف `ancestor`. طريقة أخرى للتفكير هي أن العنصر child سيُختار فقط إذا كان مسكنًا داخل `ancestor`، مهما كان العمق. ألقِ نظرة سريعة على المثال أدناه وحاول تحديد العناصر التي ستُختار بناءً على قاعدة CSS المقدمة:

```html
<!-- index.html -->

<div class="ancestor"> <!-- A -->
  <div class="contents"> <!-- B -->
    <div class="contents"> <!-- C -->
    </div>
  </div>
</div>

<div class="contents"></div> <!-- D -->
```

```css
/* styles.css */

.ancestor .contents {
  /* some declarations */
}
```

في المثال أعلاه، العنصران الأولان اللذان يحملان الصنف `contents` (`B` و `C`) سيُختاران، لكن العنصر الأخير (`D`) لن يُختار. هل كان تخمينك صحيحًا؟

لا يوجد حد لعدد المركبات التي يمكنك إضافتها إلى قاعدة، لذا `.one .two .three .four` سيكون صحيحًا تمامًا. هذا سيختار عنصرًا يحمل الصنف `four` إذا كان له أصل يحمل الصنف `three`، وإذا كان لذلك الأصل أصله الخاص الذي يحمل الصنف `two`، وهكذا. لكن من الأفضل تجنب اختيار عناصر تحتاج إلى هذا المستوى من التسكين، لأنه قد يصبح مربكًا وطويلاً، وقد يسبب مشاكل في التحديدية.
