# --description--

Kommentare in der Programmierung werden verwendet, um zusätzlichen Kontext für den Code bereitzustellen oder Notizen für sich selbst und andere zu hinterlassen.

Kommentare sind Zeilen oder Textblöcke, die von der JavaScript-Engine ignoriert werden, wenn Ihr Code ausgeführt wird. Sie dienen ausschließlich dem Nutzen der Personen, die den Code lesen, sei es Sie selbst oder jemand anderes.

JavaScript bietet zwei Möglichkeiten, Kommentare in Ihren Code einzufügen: einzeilige Kommentare und mehrzeilige Kommentare.

Einzeilige Kommentare werden mit zwei Schrägstrichen (`//`) erstellt. Hier ist ein Beispiel:

```js
// I am a single line comment in JavaScript
```

Diese Art von Kommentar eignet sich hervorragend für kurze Erklärungen oder Klarstellungen.

Hier ist ein Beispiel aus den Projektdateien des freeCodeCamp-Lehrplans:

```js
// This is to allow English to build without having to download the i18n files.
// It fails when trying to resolve the i18n-curriculum path if they don't exist.
const curriculumLocale = process.env.CURRICULUM_LOCALE ?? 'english';
const I18N_CURRICULUM_DIR = path.resolve(
  __dirname,
  curriculumLocale === 'english' ? '.' : 'i18n-curriculum/curriculum'
);
```

Machen Sie sich keine Sorgen darüber, zu verstehen, was der Code tatsächlich macht, da er fortgeschrittener ist als das, was Sie bisher gelernt haben. Konzentrieren Sie sich stattdessen auf den Kommentar, den der Entwickler hinterlassen hat. Dieser Kommentar liefert wichtigen Kontext dafür, warum dieser Code existiert.

Solche Kommentare sind aus zwei Gründen wichtig für Teams:

1. Andere Entwickler, die am Projekt arbeiten, verstehen den Zweck dieses Codes.

1. Es hilft, unnötige Änderungen oder Löschungen ohne Rücksprache mit dem Team zu vermeiden, was zu Bugs oder Problemen führen könnte.

Eine andere Art von Kommentar sind mehrzeilige Kommentare. Hier ist die grundlegende Syntax:

```js
/*
 I am a multiline comment.
 This is helpful for longer explanations.
*/
```

Mehrzeilige Kommentare sind nützlich, wenn Sie längere Beschreibungen, Erklärungen oder Notizen in Ihrem Code schreiben müssen.

Werfen wir einen weiteren Blick auf die Projektdateien des freeCodeCamp-Lehrplans, um zu sehen, wie mehrzeilige Kommentare in der Praxis verwendet werden können.

```js
/* Since there can be more than one way to complete a certification (using the
legacy curriculum or the new one, for instance), we need a certification
field to track which certification this belongs to. */
const dupeCertifications = [
  {
    certification: 'responsive-web-design',
    dupe: '2022/responsive-web-design'
  }
];
const hasDupe = dupeCertifications.find(
  cert => cert.dupe === meta.superBlock
);
```

Wie zuvor ignorieren Sie den gesamten JavaScript-Code, da er Konzepte verwendet, die noch nicht gelehrt wurden. Konzentrieren Sie sich stattdessen auf den Kommentar des Entwicklers.

Ein Entwickler im Team oder sogar ein neuer Beitragender, der am Projekt arbeitet, kann verstehen, warum dieser Codeabschnitt hier ist, und hat den vollständigen Kontext, bevor er an diesem Bereich des Projekts arbeitet.

Obwohl Kommentare in der Programmierung nützlich sind, ist es wichtig, Überkommentierung zu vermeiden. Sie müssen nicht jede einzelne Codezeile kommentieren, besonders wenn der Code klar und selbsterklärend ist.

Hier ist ein Beispiel für Kommentare, die das Offensichtliche erklären:

```js
// This code uses the const keyword to create a new variable called price.
// We are assigning the number 10 to the price variable.
const price = 10;
```

In dieser Situation ist es nicht nötig, Kommentare hinzuzufügen, da der Code selbsterklärend ist. Das Ziel ist es, die Lesbarkeit zu verbessern, nicht den Code mit unnötigen Erklärungen zu überladen.

Wenn Sie beim Lernen des Programmierens Kommentare in Ihren persönlichen Projekten hinzufügen möchten, ist das in Ordnung. Sobald Sie jedoch an realen Projekten mit anderen Entwicklern arbeiten, ist es wichtig, keine Kommentare für selbsterklärenden Code zu verwenden.

Ebenso sollten Kommentare nicht dazu dienen, verwirrenden, zu komplizierten oder schlecht geschriebenen Code zu erklären. In solchen Fällen ist es besser, den Code zu überarbeiten oder zu ändern, damit andere Entwickler besser verstehen, was vor sich geht.

Kommentare sind mächtige Werkzeuge, um Ihren Code zu dokumentieren und verständlicher zu machen. Sie sollten Kommentare verwenden, um Kontext zu liefern oder Notizen für sich selbst und andere zu hinterlassen.
