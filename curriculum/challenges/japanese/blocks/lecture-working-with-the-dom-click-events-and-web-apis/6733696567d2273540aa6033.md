# --interactive--

`Web Animations` API（WAAPI）を使うと、JavaScript内で直接アニメーションを作成・制御できます。WAAPIを使うことで、アニメーションをより動的に扱いやすくなります。

WAAPIについて詳しく見ていきましょう。JavaScriptコード内で直接アニメーションを操作できるようになります。WAAPIの中心には`Animation`コンストラクタがあり、これには要素を動的にアニメーションさせるための複数のインスタンスメソッドやプロパティが用意されています。

`Animation`コンストラクタの重要なメソッドの一つが`animate()`です。これはキーフレームや、duration、direction、easing、iterationsなどのオプションを指定してアニメーションを作成できます。

`animate()`メソッドの基本的な構文は以下の通りです。

```js
element.animate(keyframes, options);
```

例を見てみましょう。

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css" />
<div class="square" id="square"></div>
<script src="index.js"></script>
```

```css
body {
  background: #f1f1f1;
  display: flex;
  align-items: center;
  justify-content: center;
  height: 100vh;
}

.square {
  background: #1b1b32;
  width: 10rem;
  aspect-ratio: 1/1;
}
```

```js
const square = document.querySelector("#square");

const animation = square.animate(
  [{ transform: "translateX(0px)" }, { transform: "translateX(100px)" }],
  {
    duration: 2000, // makes animation lasts 2 seconds
    iterations: Infinity, // loops indefinitely
    direction: "alternate", // moves back and forth
    easing: "ease-in-out" // smooth easing
  }
);
```

:::

ブラウザでの結果は、青い四角形が左右に動くアニメーションになります。

`Animation`コンストラクタのインスタンスメソッドには以下があります。

```js
play()
pause()
reverse()
finish()
cancel()
```

インスタンスプロパティには以下があります。

```md
playbackRate
currentTime
startTime
effect
timeline
playState
finished
onfinish
oncancel
```

例を修正して、`play()`、`pause()`メソッドと`onfinish`プロパティを使ってみましょう。新しいバージョンはこちらです。

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css" />
<div class="square" id="square"></div>

<button id="playBtn">Play</button>
<button id="pauseBtn">Pause</button>
<script src="index.js"></script>
```

```css
body {
  background: #f1f1f1;
  display: flex;
  flex-direction: column;
  align-items: center;
  justify-content: center;
  height: 100vh;
}
.square {
  background: #1b1b32;
  width: 10rem;
  aspect-ratio: 1/1;
  margin-bottom: 20px;
}
button {
  margin: 10px;
  padding: 10px 20px;
  font-size: 16px;
}
```

```js
const square = document.querySelector("#square");
const playBtn = document.querySelector("#playBtn");
const pauseBtn = document.querySelector("#pauseBtn");

const animation = square.animate(
  [{ transform: "translateX(0px)" }, { transform: "translateX(200px)" }],
  {
    duration: 5000, // Animation lasts 5 seconds
    // iterations: Infinity, // Loops indefinitely
    direction: "alternate", // Moves back and forth
    easing: "ease-in-out" // Smooth easing function
  }
);

// Set the onfinish property to log a message when the animation ends
animation.onfinish = () => {
  console.log("Animation finished!");
};

// Play the animation when the "Play" button is clicked
playBtn.addEventListener("click", () => {
  animation.play();
  console.log("You start the animation");
});

// Pause the animation when the "Pause" button is clicked
pauseBtn.addEventListener("click", () => {
  animation.pause();
  console.log("You pause the animation");
});
```

:::

ブラウザでの結果は、`play`ボタンをクリックすると青いボックスが左から右に動きます。

WAAPIはCSSアニメーションの機能を拡張し、JavaScript内でより動的にアニメーションを制御できるようにします。

CSSアニメーションでは、`animation-name`、`animation-duration`、`animation-timing-function`などのプロパティを使って宣言的にアニメーションを定義します。WAAPIの`animate()`メソッドを使って同じこともできます。

違いは、`animate()`メソッドで作成したアニメーションをより直接的かつ動的に制御できる点です。一方、CSSアニメーションではカスタムスタイルを定義し、JavaScript内でそれをトリガーする必要があり、より多くの作業が必要です。

CSSアニメーションは、ホバーエフェクトやトランジション、トリガー後にあまり操作が必要ないシンプルで宣言的なアニメーションに最適です。クリックやスクロールなどのユーザー操作に応じてアニメーションを制御したり、一時停止や逆再生、速度変更などを動的に行いたい場合はWAAPIの方が適しています。

基本設定はCSSのシンプルさを使いながら、実行時にアニメーションを制御したい場合はWAAPIとCSSアニメーションを組み合わせることも検討できます。
