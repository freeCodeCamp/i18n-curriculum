# --interactive--

إخفاء المحتوى في صفحة ويب ممارسة شائعة في تطوير الويب، لكن من الضروري القيام بذلك بطريقة لا تضر بإمكانية الوصول. تقنيات الإخفاء المختلفة تؤثر بطرق متنوعة على كيفية تفسير تقنية المساعدة للمحتوى وعرضه للمستخدمين. دعنا نستعرض بعض أفضل الممارسات لإخفاء المحتوى مع الحفاظ على إمكانية الوصول. إحدى الطرق الشائعة لإخفاء المحتوى هي استخدام `display: none`. إليك مثالًا:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<p class="hidden">Hidden text</p>
<p>Visible text</p>
```

```css
.hidden {
  display: none;
}
```

:::

بينما يخفي هذا المحتوى بصريًا بشكل فعال، فإنه أيضًا يزيله من شجرة إمكانية الوصول. شجرة إمكانية الوصول هي بنية تستخدمها تقنيات المساعدة، مثل قارئ الشاشة، لتفسير المحتوى والتفاعل معه في صفحة الويب. تمثل المحتوى ومعناه الدلالي بطريقة يمكن لتقنية المساعدة فهمها وعرضها للمستخدم.

استخدام `display: none` يعني أن قارئات الشاشة وتقنيات المساعدة الأخرى لن تتمكن من الوصول إلى هذا المحتوى، لأنه غير مدرج في شجرة إمكانية الوصول. لذلك، من المهم استخدام هذه الطريقة فقط عندما تريد إزالة المحتوى تمامًا من العرض البصري وإمكانية الوصول.

طريقة أخرى لإخفاء المحتوى هي استخدام `visibility: hidden`:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<p class="hidden">Hidden text</p>
<p>Visible text</p>
```

```css
.hidden {
  visibility: hidden;
}
```

:::

`visibility: hidden` يخفي المحتوى بصريًا لكنه يبقيه في تدفق المستند، مما يعني أنه لا يزال يشغل مساحة في الصفحة. مثل `display: none`، `visibility: hidden` أيضًا يزيل المحتوى من شجرة إمكانية الوصول. هذا يعني أن تقنيات المساعدة، مثل قارئ الشاشة، لن تتمكن من الوصول إلى المحتوى المخفي. استخدم `visibility: hidden` فقط عندما تريد إخفاء المحتوى عن الجميع، بما في ذلك الأشخاص الذين يستخدمون تقنية المساعدة.

للمحتوى الذي يجب إخفاؤه بصريًا لكنه يبقى متاحًا لقارئات الشاشة، يمكنك استخدام تقنية تُعرف غالبًا باسم "مخفي بصريًا" أو "خاص بقارئ الشاشة فقط". إليك مثالًا باستخدام صنف CSS `.sr-only`، وهي تقنية شائعة تُستخدم لإخفاء المحتوى بصريًا مع إبقائه متاحًا لقارئات الشاشة:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<p class="sr-only">Hidden text</p>
<p>Visible text</p>
```

```css
.sr-only {
  position: absolute;
  width: 1px;
  height: 1px;
  padding: 0;
  margin: -1px;
  overflow: hidden;
  clip: rect(0, 0, 0, 0);
  white-space: nowrap;
  border: 0;
}
```

:::

في هذا المثال، نستخدم خصائص مثل `position` و`clip` و`white-space`، والتي ستتعلم عنها في دروس لاحقة. الآن، فقط اعلم أن قاعدة CSS هذه تخفي المحتوى بصريًا مع إبقائه متاحًا لقارئات الشاشة. هذه التقنية مفيدة لتوفير سياق إضافي لمستخدمي قارئ الشاشة دون التأثير على التخطيط البصري.

للتبديل بين ظهور المحتوى وإخفائه، فكر في استخدام الخاصية `hidden`:

:::interactive_editor

```html
<p hidden>This content is hidden</p>
<p>This content is visible</p>
```

:::

الخاصية `hidden` مدعومة من معظم المتصفحات الحديثة وتخفي المحتوى بصريًا ومن شجرة إمكانية الوصول. يمكن التبديل بينها بسهولة باستخدام JavaScript. أخيرًا، كن حذرًا عند إخفاء المحتوى المهم. إذا كانت المعلومات ضرورية لفهم الموقع أو استخدامه، يجب أن تكون مرئية ومتاحة لجميع المستخدمين. أخفِ المحتوى فقط عندما يعزز ذلك تجربة المستخدم حقًا. باتباع هذه أفضل الممارسات، تضمن بقاء المحتوى متاحًا لجميع المستخدمين، بغض النظر عن كيفية تفاعلهم مع موقعك.
