# --interactive--

الدوال هي قطع قابلة لإعادة الاستخدام من الشفرة تؤدي مهمة محددة أو تحسب قيمة. فكر في الدوال كآلة تأخذ بعض المُدخلات، تُجري عليها عمليات، ثم تُنتج مخرجًا. إليك مثالًا على إعلان دالة:

```js
function greet() {
  console.log("Hello, Jessica!");
}
```

في هذا المثال، أعلنّا دالة تُسمى `greet`. داخل تلك الدالة، لدينا `console.log` يسجل الرسالة `Hello, Jessica!`. إذا حاولنا تشغيل هذه الشفرة، لن نرى الرسالة تظهر في وحدة التحكم. ذلك لأننا بحاجة إلى استدعاء الدالة.

استدعاء الدالة، أو تنفيذها، هو عندما نستخدم الدالة فعليًا أو ننفذها. لاستدعاء دالة، تحتاج إلى الإشارة إلى اسم الدالة متبوعًا بمجموعة من الأقواس العادية:

:::interactive_editor

```js
function greet() {
  console.log("Hello, Jessica!");
}

greet(); // "Hello, Jessica!"
```

:::

الآن ستُسجل رسالة `Hello, Jessica!` في وحدة التحكم. لكن ماذا لو أردنا أن تقول الرسالة `Hello, Nick!` أو `Hello, Anna!`؟ لا نريد كتابة دالة جديدة في كل مرة نحيي فيها مستخدمًا مختلفًا. بدلًا من ذلك، يمكننا إنشاء دالة قابلة لإعادة الاستخدام تستخدم معلمات الدالة والمعلمات الفعلية.

تعمل المعلمات كعناصر نائب للقيم التي ستمرر إلى الدالة عند استدعائها. تسمح الدوال بقبول مُدخلات والعمل عليها. المعلمات الفعلية هي القيم الحقيقية التي تُمرر إلى الدالة عند استدعائها. إليك نسخة محدثة من دالة `greet` تستخدم المعلمات والمعلمات الفعلية:

:::interactive_editor

```js
function greet(name) {
  console.log("Hello, " + name + "!");
}

greet("Alice"); // Hello, Alice!
greet("Nick"); // Hello, Nick!
```

:::

تعمل `name` كمعلمة بينما السلاسل النصية `Alice` و`Nick` تعمل كمعلمات فعلية. الآن لدينا دالة قابلة لإعادة الاستخدام يمكن استخدامها عشرات المرات في شفرتنا مع معلمات فعلية مختلفة.

عندما تنهي الدالة تنفيذها، ستُرجع دائمًا قيمة. افتراضيًا، ستكون قيمة الإرجاع `undefined`. إليك مثالًا:

:::interactive_editor

```js
function doSomething() {
  console.log("Doing something...");
}

let result = doSomething();
console.log(result); // undefined
```

:::

إذا كنت تحتاج إلى أن تُرجع دالتك قيمة محددة، فستحتاج إلى استخدام جملة `return`. إليك مثالًا على استخدام جملة `return` لإرجاع مجموع قيمتين:

:::interactive_editor

```js
function calculateSum(num1, num2) {
  return num1 + num2;
}

console.log(calculateSum(3, 4)); // 7
```

:::

غالبًا ما ستستخدم جملة `return`، لأنك تستطيع استخدام القيمة التي أُخرجت من الدالة لاحقًا في شفرتك.

حتى الآن، كنا نعمل مع دوال مسماة، لكن يمكنك أيضًا إنشاء ما يُسمى دالة مجهولة. الدالة المجهولة هي دالة بلا اسم يمكن تعيينها إلى متغير مثل هذا:

:::interactive_editor

```js
const sum = function (num1, num2) {
  return num1 + num2;
};

console.log(sum(3, 4)); // 7
```

:::

في هذا المثال، لدينا متغير `const` يُسمى `sum` ونُعيّن له دالة مجهولة تُرجع مجموع `num1` و`num2`. ثم نستطيع استدعاء `sum` وتمرير الرقمين `3` و`4` للحصول على النتيجة `7`.

تدعم الدوال المعلمات الافتراضية، مما يسمح لك بتعيين قيم افتراضية للمعلمات. تُستخدم هذه القيم الافتراضية إذا استُدعيت الدالة بدون معلمة فعلية لتلك المعلمة. إليك مثالًا:

:::interactive_editor

```js
function greetings(name = "Guest") {
  console.log("Hello, " + name + "!");
}

greetings(); // Hello, Guest!
greetings("Anna"); // Hello, Anna!
```

:::

في هذا المثال، إذا لم يُقدّم معلمة فعلية لـ `name`، فإنها تفترض القيمة `Guest` افتراضيًا.

باختصار، تسمح لك الدوال بكتابة شفرة قابلة لإعادة الاستخدام ومنظمة. يمكنها أخذ مُدخلات (معلمات)، تنفيذ إجراءات، وإرجاع مخرجات.
