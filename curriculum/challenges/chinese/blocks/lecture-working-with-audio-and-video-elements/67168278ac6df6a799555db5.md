# --interactive--

`audio` 和 `video` 元素可让你在 HTML 文档中添加声音和视频内容。 `audio` 元素支持流行的音频格式，如 mp3、wav 和 ogg。 `video` 元素支持 mp4、ogg 和 webm 格式。

要在你的网页中包含音频内容，可以使用带有指向音频文件位置的 `src` 属性的 `audio` 元素。 

正如你在预览窗口中看到的，页面上没有显示任何内容。要查看预览，你需要启用交互式编辑器。

:::interactive_editor

```html
<audio src="https://cdn.freecodecamp.org/curriculum/js-music-player/cruising-for-a-musing.mp3"></audio>
```

:::

如果你想在页面上看到音频播放器，那么你可以为你的 `audio` 元素添加 `controls` 属性。

在预览窗口中按播放以听取 Quincy Larson 的一首歌曲。要听不同的歌曲，请将 `src` 值更改为 `"https://cdn.freecodecamp.org/curriculum/js-music-player/never-not-favored.mp3"`。要查看预览，你需要启用交互式编辑器。

:::interactive_editor

```html
<audio src="https://cdn.freecodecamp.org/curriculum/js-music-player/cruising-for-a-musing.mp3" controls></audio>
```

:::

The `controls` attribute enables users to manage audio playback, including pausing or resuming playback. `controls` 属性是一个布尔属性，可添加到元素中以启用内置播放控制。 如果省略，则不会显示任何控件。

注意：某些浏览器，例如 Safari，即使存在 `controls` 属性，也可能默认不显示卷控件。

除了 `src` 和 `controls` 属性外，还有其他几个属性可以增强 `audio` 元素的功能。 `loop` 属性是一个布尔属性，用于使音频连续重放。 

这是一个使用 `loop` 属性播放 Quincy Larson 的歌曲 `"Can't stay down"` 的示例。要查看循环效果，请启用交互式编辑器，将播放头拖动到歌曲接近结尾的位置，歌曲播放结束后将重新开始。 

:::interactive_editor

```html
<audio
  src="https://cdn.freecodecamp.org/curriculum/js-music-player/can't-stay-down.mp3"
  loop
  controls
></audio>
```

:::

你可以使用的另一个属性是 `muted` 属性。 当出现在 `audio` 元素中时，此布尔属性将以静音状态启动音频。  Here's an example of using the `muted` attribute.

要听音乐，请启用交互式编辑器并点击音频播放器中的卷图标。 

:::interactive_editor

```html
<audio
  src="https://cdn.freecodecamp.org/curriculum/js-music-player/can't-stay-down.mp3"
  loop
  controls
  muted
></audio>
```

:::

说到音频文件类型，不同的浏览器支持不同的类型。 为此，你可以在 `audio` 元素内使用 `source` 元素，浏览器将选择它能理解的第一个源。 下面是为 `audio` 元素使用多个 `source` 元素的示例：

```html
<audio controls>
  <source src="audio.ogg" type="audio/ogg" />
  <source src="audio.wav" type="audio/wav" />
  <source src="audio.mp3" type="audio/mpeg" />
</audio>
```

浏览器会首先从 ogg 类型开始，如果无法播放音频，就会下移到列表中的下一个类型。

到目前为止我们学过的所有属性也都支持在 `video` 元素中。下面是一个使用带有 `loop`、`controls` 和 `muted` 属性的 `video` 元素的示例。 

为打开的 `video` 标签添加 `autoplay` 属性，使视频自动播放。要与示例交互，你需要启用交互式编辑器。

**注意**：这里使用了 `width` 属性来使视频更小并更好地适应预览窗口。你将在后续课程中学习更多关于 `width` 属性的内容。

:::interactive_editor

```html
<video
  src="https://archive.org/download/BigBuckBunny_124/Content/big_buck_bunny_720p_surround.mp4"
  loop
  controls
  muted
  width="400"
></video>
```

:::

对于 `src`，或源属性，我们使用的是 archive.org 上名为 "Big Buck Bunny" 的视频。 如果你想在加载视频时显示图像，可以使用 `poster` 属性。 此属性不适用于 `audio` 元素，是 `video` 元素独有的属性。  Here's an example of using the `poster` attribute with content provided by peach.blender.org.

```html
<video
  src="https://archive.org/download/BigBuckBunny_124/Content/big_buck_bunny_720p_surround.mp4"
  loop
  controls
  muted
  poster="https://peach.blender.org/wp-content/uploads/title_anouncement.jpg?x11217"
  width="400"
></video>
```

你也可以在 `video` 元素内使用 `source` 元素，就像你在 `audio` 元素中所做的那样。这允许你以多种格式提供相同的视频，浏览器将选择它能播放的第一个格式。

```html
<video
  controls
  width="400"
  poster="https://peach.blender.org/wp-content/uploads/title_anouncement.jpg?x11217"
>
  <source
    src="https://archive.org/download/BigBuckBunny_124/Content/big_buck_bunny_720p_surround.mp4"
    type="video/mp4"
  />
  <source
    src="https://archive.org/download/BigBuckBunny_124/Content/big_buck_bunny_720p_surround.webm"
    type="video/webm"
  />
  Your browser does not support the video tag.
</video>
```
