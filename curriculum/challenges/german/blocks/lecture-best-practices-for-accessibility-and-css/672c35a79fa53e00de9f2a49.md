# --interactive--

Inhalte auf einer Webseite zu verstecken ist eine gängige Praxis in der Webentwicklung, aber es ist entscheidend, dies so zu tun, dass die Barrierefreiheit nicht beeinträchtigt wird. Verschiedene Techniken zum Verstecken können unterschiedliche Auswirkungen darauf haben, wie Hilfstechnologien die Inhalte interpretieren und den Nutzenden präsentieren. Lassen Sie uns einige bewährte Vorgehensweisen zum Verstecken von Inhalten erkunden, während die Barrierefreiheit erhalten bleibt. Eine häufige Methode, Inhalte zu verstecken, ist die Verwendung von `display: none`. Hier ist ein Beispiel:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<p class="hidden">Hidden text</p>
<p>Visible text</p>
```

```css
.hidden {
  display: none;
}
```

:::

Obwohl dies den Inhalt visuell effektiv versteckt, entfernt es ihn auch aus dem Accessibility Tree. Der Accessibility Tree ist eine Struktur, die von Hilfstechnologien wie Screenreadern verwendet wird, um die Inhalte auf einer Webseite zu interpretieren und mit ihnen zu interagieren. Er stellt die Inhalte und deren semantische Bedeutung so dar, dass die Hilfstechnologie sie verstehen und dem Nutzenden präsentieren kann.

Die Verwendung von `display: none` bedeutet, dass Screenreader und andere Hilfstechnologien nicht auf diesen Inhalt zugreifen können, da er nicht im Accessibility Tree enthalten ist. Daher ist es wichtig, diese Methode nur dann zu verwenden, wenn Sie Inhalte vollständig sowohl aus der visuellen Darstellung als auch aus der Barrierefreiheit entfernen möchten.

Eine andere Möglichkeit, Inhalte zu verstecken, ist die Verwendung von `visibility: hidden`:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<p class="hidden">Hidden text</p>
<p>Visible text</p>
```

```css
.hidden {
  visibility: hidden;
}
```

:::

`visibility: hidden` versteckt den Inhalt visuell, behält ihn aber im Dokumentenfluss, das heißt, er nimmt weiterhin Platz auf der Seite ein. Wie `display: none` entfernt auch `visibility: hidden` Inhalte aus dem Accessibility Tree. Das bedeutet, dass Hilfstechnologien wie Screenreader nicht auf den versteckten Inhalt zugreifen können. Verwenden Sie `visibility: hidden` nur, wenn Sie Inhalte für alle, einschließlich Menschen, die Hilfstechnologien nutzen, verstecken möchten.

Für Inhalte, die visuell versteckt, aber für Screenreader zugänglich bleiben sollen, können Sie eine Technik verwenden, die oft als „visually hidden“ oder „screen reader only“ bezeichnet wird. Hier ist ein Beispiel mit der CSS-Klasse `.sr-only`, eine gängige Technik, um Inhalte visuell zu verstecken und gleichzeitig für Screenreader zugänglich zu halten:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<p class="sr-only">Hidden text</p>
<p>Visible text</p>
```

```css
.sr-only {
  position: absolute;
  width: 1px;
  height: 1px;
  padding: 0;
  margin: -1px;
  overflow: hidden;
  clip: rect(0, 0, 0, 0);
  white-space: nowrap;
  border: 0;
}
```

:::

In diesem Beispiel verwenden wir Eigenschaften wie `position`, `clip` und `white-space`, die Sie in zukünftigen Lektionen kennenlernen werden. Für den Moment wissen Sie nur, dass diese CSS-Regel den Inhalt visuell effektiv versteckt, während er für Screenreader zugänglich bleibt. Das ist nützlich, um zusätzlichen Kontext für Screenreader-Nutzende bereitzustellen, ohne das visuelle Layout zu beeinflussen.

Zum Umschalten der Sichtbarkeit von Inhalten sollten Sie das Attribut `hidden` in Betracht ziehen:

:::interactive_editor

```html
<p hidden>This content is hidden</p>
<p>This content is visible</p>
```

:::

Das Attribut `hidden` wird von den meisten modernen Browsern unterstützt und versteckt Inhalte sowohl visuell als auch aus dem Accessibility Tree. Es lässt sich leicht mit JavaScript umschalten. Seien Sie zuletzt vorsichtig beim Verstecken wichtiger Inhalte. Wenn Informationen entscheidend für das Verständnis oder die Nutzung der Webseite sind, sollten sie für alle Nutzenden sichtbar und zugänglich sein. Verstecken Sie Inhalte nur, wenn dies die Nutzererfahrung wirklich verbessert. Wenn Sie diese bewährten Vorgehensweisen befolgen, stellen Sie sicher, dass Ihre Inhalte für alle Nutzenden zugänglich bleiben, unabhängig davon, wie sie mit Ihrer Webseite interagieren.
