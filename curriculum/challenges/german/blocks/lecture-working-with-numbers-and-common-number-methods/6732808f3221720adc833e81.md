# --interactive--

`parseFloat()` und `parseInt()` sind zwei wichtige Methoden in JavaScript, um Strings in Zahlen umzuwandeln. Diese Methoden sind besonders nützlich, wenn Sie mit Benutzereingaben arbeiten oder Daten verarbeiten, die als String vorliegen, aber als numerische Werte behandelt werden müssen.

Beginnen wir mit `parseFloat()`. Diese Methode parst ein String-Argument und gibt eine Gleitkommazahl zurück. Sie ist dafür ausgelegt, eine Zahl vom Anfang eines Strings zu extrahieren, selbst wenn der String später nicht-numerische Zeichen enthält. Denken Sie daran, dass Floats Zahlen mit Dezimalstellen sind. So funktioniert `parseFloat()`:

:::interactive_editor

```js
console.log(parseFloat("3.14"));     // 3.14
console.log(parseFloat("3.14 abc")); // 3.14
console.log(parseFloat("3.14.5"));   // 3.14
console.log(parseFloat("abc 3.14")); // NaN
```

:::

Wie Sie sehen, beginnt `parseFloat()` mit dem Parsen am Anfang des Strings und fährt fort, bis es auf ein Zeichen trifft, das kein Teil einer Gleitkommazahl sein kann. Wenn es keine gültige Zahl am Anfang des Strings findet, gibt es `NaN` (Not a Number) zurück.

`parseInt()` hingegen parst ein String-Argument und gibt eine Ganzzahl zurück. Wie `parseFloat()` beginnt es am Anfang des Strings, stoppt jedoch beim ersten nicht-ziffern Zeichen. So funktioniert `parseInt()`:

:::interactive_editor

```js
console.log(parseInt("42"));       // 42
console.log(parseInt("42px"));     // 42
console.log(parseInt("3.14"));     // 3
console.log(parseInt("abc123"));   // NaN
```

:::

`parseInt()` stoppt das Parsen beim ersten nicht-ziffern Zeichen, das es findet. Bei Gleitkommazahlen gibt es nur den ganzzahligen Teil zurück. Wenn es keine gültige Ganzzahl am Anfang des Strings findet, gibt es `NaN` zurück.

Beide Methoden zeigen einige bemerkenswerte Verhaltensweisen. Sie ignorieren führende Leerzeichen:

:::interactive_editor

```js
console.log(parseFloat("  3.14"));  // 3.14
console.log(parseInt("  42"));      // 42
```

:::

Sie verarbeiten Plus- und Minuszeichen am Anfang des Strings:

:::interactive_editor

```js
console.log(parseFloat("+3.14"));  // 3.14
console.log(parseInt("-42"));      // -42
```

:::

Es ist wichtig zu beachten, dass diese Methoden zwar leistungsfähig sind, aber einige Einschränkungen haben. Zum Beispiel verarbeiten sie nicht alle Zahlenformate, wie wissenschaftliche Notation, direkt. Für komplexere Parsing-Anforderungen müssen Sie möglicherweise zusätzliche Techniken oder Bibliotheken verwenden.

Zusammenfassend sind `parseFloat()` und `parseInt()` wertvolle Werkzeuge, um Strings in Zahlen in JavaScript umzuwandeln. Wenn Sie verstehen, wie sie funktionieren und welche spezifischen Verhaltensweisen sie haben, können Sie numerische Daten in Ihren Anwendungen effektiver verarbeiten, insbesondere bei Benutzereingaben oder externen Datenquellen.
