# --description--

End-to-End-Testing, kurz E2E, testet reale Szenarien aus der Perspektive des Nutzers. Beispiele für E2E-Test-Szenarien sind das Testen des Registrierungs- und Anmeldeprozesses oder das Testen des Bestellvorgangs bei einer E-Commerce-Website.

Um besser zu verstehen, wie End-to-End-Testing funktioniert, schauen wir uns ein reales Beispiel aus dem freeCodeCamp-Codebestand an. Dieses Beispiel verwendet Playwright, ein beliebtes End-to-End-Testframework, das von Microsoft entwickelt wurde.

Im freeCodeCamp-Codebestand gibt es ein Verzeichnis namens `e2e`, das Tests für Spenden, Zertifizierungen, Prüfungen und mehr enthält. Innerhalb des Verzeichnisses gibt es eine `donate-page-donor.spec.ts`-Datei, die Tests für die authentifizierte Unterstützerseite von Spendern enthält.

Der erste Schritt ist, als Demo-Nutzer, der bereits Unterstützer ist, zur Spendenseite zu navigieren:

```js
test.beforeEach(async ({ page }) => {
  execSync("node ./tools/scripts/seed/seed-demo-user --set-true isDonating");
  await page.goto("/donate");
});
```

Da es in dieser Datei mehrere Tests gibt, wird der `beforeEach`-Hook vor jedem Test ausgeführt.

Sobald sich der Nutzer auf der Spendenseite befindet, sieht er eine Dankesnachricht mit Vorschlägen, wie er freeCodeCamp weiter unterstützen kann. Hier sind nur einige der Tests, die prüfen, ob bestimmte Texte auf der Spendenseite vorhanden sind:

```js
test("should render the donate page correctly", async ({ page }) => {
  await expect(
    page.getByText("Thank you for your continued support")
  ).toBeVisible();

  await expect(
    page.getByText(
      "Your contributions are crucial in creating resources that empower millions of people to learn new skills and support their families."
    )
  ).toBeVisible();

  ...
});
```

Es gibt auch Tests, die überprüfen, ob der Spender einen Unterstützer-Link in der Menüleiste hat, sowie einen speziell gestalteten Rahmen um sein Avatarbild, der anzeigt, dass er Unterstützer ist:

```js
test("The menu should have a supporters link", async ({ page }) => {
  const menuButton = page.getByTestId("header-menu-button");
  const menu = page.getByTestId("header-menu");

  await expect(menuButton).toBeVisible();
  await menuButton.click();

  await expect(menu).toBeVisible();

  await expect(page.getByRole("link", { name: "Supporters" })).toBeVisible();
});

test("The Avatar should have a special border for donors", async ({ page }) => {
  const container = page.locator(".avatar-container");
  await expect(container).toHaveClass("avatar-container gold-border");
});
```

Durch solche detaillierten Tests können Sie die Testabdeckung Ihrer Anwendung über reine Unit-Tests hinaus erhöhen. End-to-End-Tests helfen außerdem sicherzustellen, dass Ihre Anwendung korrekt funktioniert und für Nutzer vorhersehbar ist. Weitere Beispiele für End-to-End-Testtools sind Cypress, Selenium und Puppeteer.

Obwohl End-to-End-Testing wichtig ist, ist es aufwendig, da das Einrichten, Entwerfen und Pflegen viel Zeit in Anspruch nimmt. E2E-Tests eignen sich hervorragend für kritische Nutzerabläufe, während Unit-Tests ideal sind, um kleine Einheiten in der Anwendung zu testen.
