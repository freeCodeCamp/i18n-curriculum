# --description--

让我们了解命令选项和标记。

在之前的课程中，你学习了如何向诸如 `touch readme.md` 这样的命令传递参数。选项或标记是你可以传递给命令的特殊参数，它们会影响命令的行为方式。这两个术语可以互换使用，尽管“标记”通常更具体地用于作为开/关切换的选项。

选项通常以一个或两个连字符（`-`）符号为前缀，这有助于在选项和参数之间提供视觉区分。首先，让我们看看两个连字符，或称“长形式”的语法。

许多应用程序，例如 `ls`，接受 `--version` 标记，该标记打印应用程序的当前版本到终端，而不是运行应用程序的命令。

许多应用也提供 `--help` 标记，打印有关如何使用该应用的说明。

单破折号或“短形式”语法通常使用单个字母的选项。例如，`ls` 的 `-a` 标记会列出所有文件，包括以点号（`.`）开头的隐藏文件，如 `.env`。

这些短选项的优点是你可以将多个标记串联在一起。你可以使用 `ls -ahs` 一次性使用所有单个字母，而不是 `ls --all --human-readable --size`。

某些选项需要传递一个值。

使用长形式选项时，通常需要使用等号。在此语法中，值直接与选项通过等号（`=`）符号连接。以下是修改 `ls` 行为以包含或排除颜色的示例：

```bash
ls --color=never
```

使用短形式选项时，通常用空格分隔值。例如，以下是设置 `ls` 结果中 `width` 的长形式和短形式选项：

```bash
ls -w 50
```

```bash
ls --width=50
```

注意简写形式使用 `-w 50`，而长写形式使用 `--width=50`。了解这一区别很重要，以避免将选项值错误地作为位置参数传递。

但如果你不确定，记住你通常可以使用 `--help` 标记来查看选项的预期语法！

`命令` 选项的“长形式”语法和“短形式”语法有什么区别？
