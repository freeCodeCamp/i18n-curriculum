# --description--

단어 `issue`는 처리해야 할 문제나 사안을 가리킬 수 있습니다.

프로그래밍에서 `issue`는 종종 수정해야 할 코드 내 버그나 오류를 의미합니다. 예를 들어, `We found an issue in the latest software update.`

# --questions--

## --text--

프로그래밍 맥락에서 `issue`는 무슨 뜻인가요?

## --answers--

문서가 불완전하다는 뜻입니다.

### --feedback--

문서 문제도 `issues`로 간주될 수 있지만, 프로그래밍에서는 `issue`가 코드 자체 내 문제를 더 직접적으로 가리킵니다.

---

소프트웨어 애플리케이션의 사용자 설명서를 의미합니다.

### --feedback--

`issue`는 사용자 설명서와 관련된 것이 아니라 수정해야 할 코드 내 문제를 뜻합니다.

---

새로운 기능 요청을 나타냅니다.

### --feedback--

기능 요청은 `issues`와 다릅니다; `issues`는 문제를 나타내고, 기능 요청은 새로운 기능에 대한 제안입니다.
