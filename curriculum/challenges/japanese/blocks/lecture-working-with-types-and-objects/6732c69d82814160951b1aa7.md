# --interactive--

`Number()`コンストラクタは数値オブジェクトを作成するために使われます。数値オブジェクトには`isNaN`や`toFixed`メソッドのような便利なプロパティやメソッドがいくつか含まれています。以下は`Number()`コンストラクタを`new`キーワードと一緒に使った例です：

:::interactive_editor

```js
const myNum = new Number("34");
console.log(typeof myNum); // "object" 
```

:::

この例では文字列リテラルを`Number()`コンストラクタに渡しており、戻り値の型は`object`ではなく`string`型になっています。

`Number()`コンストラクタが`new`キーワードなしで関数として呼び出されると、戻り値はプリミティブな数値型になります。ほとんどの場合、他のデータ型を数値型に変換するために`Number()`コンストラクタを使います。文字列を数値に変換する例を示します：

:::interactive_editor

```js
const myNum = Number("100");
console.log(myNum); // 100

console.log(typeof myNum); // number
```

:::

これはユーザーからの入力を受け取り、その文字列入力を数値に変換して数学的な計算を行う必要がある場合に役立ちます。

空の文字列を`Number()`コンストラクタに渡そうとすると、結果は数値の`0`になります：

:::interactive_editor

```js
const num = Number("");
console.log(num); // 0
```

:::

これはJavaScriptが文字列を解析しようとし、数字が含まれていないため結果がゼロになるからです。

ランダムな文字が入った文字列を渡すと、結果は`NaN`、つまり「Not a Number」になります。

:::interactive_editor

```js
const num = Number("random");
console.log(num); // NaN
```

:::

ブール値を扱う場合、`true`は`1`を返します。なぜなら`true`は1に変換され、`false`は`0`を返し、`false`は0に変換されるからです。

:::interactive_editor

```js
const boolTrue = Number(true);
const boolFalse = Number(false);

console.log(boolTrue); // 1
console.log(boolFalse); // 0
```

:::

`null`を渡すと結果は`0`になり、`undefined`を渡すと結果は`NaN`になります。

:::interactive_editor

```js
const undefinedNum = Number(undefined);
const nullNum = Number(null);

console.log(undefinedNum); // NaN
console.log(nullNum); // 0
```

:::

配列を扱う場合、いくつか考慮すべきことがあります。

空の配列は`0`を返します。単一の数値を含む配列はその数値を返します。複数の数値を含む配列は`NaN`を返します。そして文字列を含む配列も`NaN`を返します。

:::interactive_editor

```js
const emptyArr = Number([]);
const arrOneNum = Number([7]);
const arrMultiNum = Number([7, 36, 12]);
const arrStr = Number(["str1"]);
const arrMultiStr = Number(["str1", "str2"]);

console.log(emptyArr); // 0
console.log(arrOneNum); // 7
console.log(arrMultiNum); // NaN
console.log(arrStr); // NaN
console.log(arrMultiStr); // NaN
```

:::

オブジェクトを扱う場合、結果は常に`NaN`になります。

:::interactive_editor

```js
const obj1 = Number({});
const obj2 = Number({2: 2});
const obj3 = Number({key: "val"});
const obj4 = Number({key: true});

console.log(obj1); // NaN
console.log(obj2); // NaN
console.log(obj3); // NaN
console.log(obj4); // NaN
```

:::

まとめると、数値や数値オブジェクトを作成するよりも、型変換のために`Number()`コンストラクタを使うことがほとんどです。
