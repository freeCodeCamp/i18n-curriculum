## 在我们开始之前

Flexbox 版型可能会有点复杂。在之前的课程中，你已经学习了如何使用浏览器的开发者工具来检查和调试内容。这些工具在接下来的课程中对你来说至关重要。如果某些内容的表现不符合你的预期，每次都应该首先在开发者工具中检查它。

Flexbox 并不一定比我们到目前为止介绍的其他概念更难，但它确实有更多的活动部分。在你学完所有内容并能够将它们整合起来之前，使用这些最初课程中学到的任何内容都会有些困难。随着学习的进行，请善待自己，<strong>多多玩弄所有的代码示例。</strong>

当你完成本节末尾的赋值时，你几乎肯定需要回过头来引用这些课程（或我们与你分享的几个资源），但如果你花时间并尝试我们提供的所有代码示例，到那时你会更清楚该去哪里查找。

## 让我们展示实力

Flexbox 是一种将项排列成行或列的方式。这些项将根据你可以定义的一些简单规则进行伸缩（即增长或缩小）。要开始，先来看一个简单的演示。

我们很快就会详细说明这里到底发生了什么。但现在，让我们通过移除包围它们的 `/*` 和 `*/` 标签来取消注释上面 Codepen 中与 flex 相关的两个 CSS 声明，然后查看结果。

> 评论防止浏览器将行解释为代码，并且被包裹在特定的标签之间。CSS 使用 `/*` 作为开启评论标签，`*/` 作为关闭评论标签，而超文本标记语言和 JavaScript 有它们自己的语法。被注释掉的代码行可以通过移除包围代码的评论标签来重新启用。

现在所有 3 个 div 应该水平排列。如果你使用 `1x`、`.5x` 和 `.25x` 按钮调整结果帧的大小，你还会看到 `div` 元素会“flex”。它们将填充可用区域，并且每个宽度相等。

如果你为 `.flex-container` 内的超文本标记语言添加另一个 `div`，它将与其他元素一起显示，且所有元素都会弹性调整以适应可用区域。

## --text--

当 CodePen 中与 flex 相关的两个 CSS 声明被取消注释时，会发生什么？

## --answers--

`div` 元素从视图中隐藏，失去在容器中的可见性。

---

所有 3 个 `div` 元素水平排列，并且它们通过弹性布局以相等宽度填充可用区域。

---

每个 `div` 元素都会放大并覆盖整个容器，彼此重叠。

---

`div` 元素垂直对齐在单个列中，忽略与 flex 相关的 CSS 声明。
