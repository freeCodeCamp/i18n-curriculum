# --description--

与处理较小的原生超文本标记语言、CSS 和 JavaScript 项目不同，启动一个新建的 React 项目需要更多步骤以确保一切正常运行。你无需自己尝试设置所有必要的配置，有工具可以为你完成这些工作。 

设置项目最流行的工具之一是 Vite。Vite 在法语中意为“快速”，是一种构建工具，旨在为现代网页项目提供更快的开发体验。Vite 可以与 React 一起使用，也可以与 Vue 或 Svelte 等其他库和框架一起使用。你甚至可以将它用于原生 JavaScript 项目。

要使用 Vite 创建一个新项目，你需要使用命令行。如果你使用的是 Windows 机器，那么你可以使用命令提示符或 Windows PowerShell。如果你使用的是 Mac，那么你可以使用终端应用程序。

一旦你打开了命令行，你就可以使用以下命令：

```bash
npm create vite@latest my-react-app -- --template react
```

此命令使用 Vite 的 React 模板创建一个名为 `my-react-app` 的新 React 项目。

你可以打开新建的项目，查看 Vite 为你提供的 React 公式化代码。

Vite 的最大优点是它只会提供启动你的 React `project` 所绝对必要的 `files`。

要实际按原样运行该项目，你需要使用命令行中的以下命令安装依赖项：

```bash
cd my-react-app
npm install
```

`cd my-react-app` 命令确保你切换到项目所在的正确目录。

`npm install` 命令用于安装正确运行你的 React 项目所需的依赖项。依赖项是你的 React 项目为了正常运行所需的库或包，例如 React 本身、ReactDOM，或提供额外功能性的其他第三方包。

运行 `npm install` 会读取你项目目录中的 `package.json` 文件并安装其中列出的所有依赖项，使你能够构建和运行你的 React 应用而不会缺少任何必要的组件。

`package.json` 文件是项目中的关键配置文件，包含有关你的项目的元数据，包括其名称、版本和依赖项。它还定义了脚本、许可信息以及帮助管理项目及其依赖项的其他设置。

依赖安装完成后，你应该会在你的项目中看到一个名为 `node_modules` 的新文件夹。

`node_modules` 文件夹是存放你的项目所需的所有包和库的地方。该文件夹包含 `package.json` 文件中列出的依赖项的实际代码，包括你项目的直接依赖项以及这些依赖项的任何依赖项。

要运行你的项目，运行 `npm run dev` 命令并在新的浏览器标签（页）中打开 `http://localhost:5173/`。

你应该看到 Vite 为你提供的入门模板。

如果你需要停止局部的服务器，请在命令行中按 `CTRL + C`。

要实际查看此入门模板的代码，你可以进入你的项目中的 `src` 文件夹，应该能看到 `App.jsx` 文件。

从那里你可以开始修改 `file`，储存你的更改并在浏览器中看到新建的更改显示。

这样，你就可以开始创建你的 React 应用了！
