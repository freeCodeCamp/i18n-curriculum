# --description--

在 React 中，行內樣式用於直接將 CSS 樣式套用到你的 JSX 程式碼中的 React 元素，而不是定義在獨立的 CSS 檔案中。

React 對行內樣式的處理方式是使用 JavaScript 物件來定義樣式，而非傳統的 CSS 字串。這表示你不是像在 CSS 檔案中那樣撰寫樣式，而是創建一個 JavaScript 物件，其中鍵是 CSS 屬性名稱的駝峰式寫法，值則是 CSS 值的字串。

這裡是一個如何為 `Button` 組件使用行內樣式的範例：

```jsx
function Button({ buttonText }) {
  const defaultStyles = {
    backgroundColor: "#007BFF",
    color: "white",
    border: "none",
    borderRadius: "4px",
    padding: "10px 20px",
    fontSize: "16px",
    fontWeight: "bold",
    cursor: "pointer",
    transition: "background-color 0.3s ease",
  };

  return <button style={defaultStyles}>{buttonText}</button>;
}
```

在這個範例中，我們定義了一個名為 `defaultStyles` 的樣式物件。接著，我們使用 `style` 屬性將這些樣式應用到按鈕元素。React 在呈現時會負責將這些樣式套用到該元素。

你也可以選擇直接將物件傳入 `style` 屬性。以下是一個修訂後的範例：

```jsx
function Button({ buttonText }) {
  return (
    <button
      style={{
        backgroundColor: "#007BFF",
        color: "white",
      }}
    >
      {buttonText}
    </button>
  );
}
```

請注意 `style` 屬性中的雙重大括號 `{{}}`。外層大括號顯示 JSX 中的 JavaScript 表達式，而內層大括號定義 JavaScript 物件字面值。此語法允許你直接在 JSX 屬性中嵌入 JavaScript 物件。

有時如果只有少數幾個屬性，如此處所示，你可能想直接傳入一個物件。否則，像第一個範例那樣傳入物件的名稱會比較好。

重要的是要注意，雖然 CSS 屬性名稱通常以連字元命名法（kebab case）書寫，例如 `font-size`，但在 React 的行內樣式中，我們使用駝峰式命名法（camel case），例如 `fontSize`。這是因為樣式物件是 JavaScript 物件，而連字元命名法的名稱在 JavaScript 中若不使用引號，無法作為物件鍵值。

React 中行內樣式的一大優點是它們支援基於組件狀態或 props 的動態樣式。<dfn>例如</dfn>：

```jsx
function DynamicButton({ isActive }) {
  const buttonStyles = {
    backgroundColor: isActive ? "green" : "red",
    color: "white",
    padding: "10px 15px",
    border: "none",
    cursor: "pointer",
  };

  return <button style={buttonStyles}>Login</button>;
}
```

在此範例中，按鈕的背景顏色會根據 `isActive` prop 變化。這種動態樣式對於建立互動且回應迅速的使用者介面非常強大。

總結來說，React 中的行內樣式提供了一種強大的方式，能夠直接在你的組件中套用和操作樣式。它們使用 JavaScript 物件而非 CSS 字串，屬性名稱需要使用駝峰式命名，並且可以輕鬆地整合動態值。它們是 React 開發者工具組中不可或缺的工具，特別適合用於建立高度訂製且互動性強的使用者介面。
