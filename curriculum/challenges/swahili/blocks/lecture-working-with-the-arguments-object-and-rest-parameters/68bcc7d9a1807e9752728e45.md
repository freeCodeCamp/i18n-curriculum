# --interactive--

Katika funzo lililopita, ulijifunza jinsi ya kufanya kazi na kitu cha `arguments` ambacho ni kitu kinachofanana na safu ya data kinachojumuisha thamani za hoja zilizopitishwa kwenye kitendakazi.

:::interactive_editor

```js
function logArgs() {
  for (const arg of arguments) {
    console.log(arg);
  }
}

logArgs(1, 2, 3);
// result:
// 1
// 2
// 3
```

:::

Ingawa hii ni njia halali ya kupata na kufanya kazi na seti inayobadilika ya hoja kutoka kwenye kitendakazi, programu za kisasa za JavaScript zitumie sintaksia ya kigezo cha rest badala yake.

Hapa kuna mfano uliosasishwa ukitumia sintaksia ya kigezo cha rest badala ya kitu cha `arguments`:

:::interactive_editor

```js
function logArgs(...args) {
  for (const arg of args) {
    console.log(arg);
  }
}

logArgs(1, 2, 3);
// result:
// 1
// 2
// 3
```

:::

Mfano huu uliosasishwa hauzitaji tena rejea za moja kwa moja kwa kitu cha `arguments`. Badala yake, kigezo cha mwisho katika ufafanuzi wa kitendakazi kina nukta tatu mbele yake. Hii husababisha kigezo hiki cha rest kuwekwa ndani ya kitu cha `Array`. Unaweza kumtaja kigezo hiki cha rest kwa jina lolote unalotaka. Hakikisha tu ni kigezo cha mwisho katika ufafanuzi wa kitendakazi kama hivi:

```js
function exampleFunction(a, b, ...restOfArgs) {
  // some code here
}

function anotherFunction(x, y, ...theArgs) {
  // some code here
}
```

Kuna vikwazo vichache zaidi unavyokabiliana navyo ukiwa unatumia sintaksia ya kigezo cha rest. Kikwazo kimoja ni kwamba ufafanuzi wa kitendakazi unaweza kuwa na kigezo kimoja tu cha rest. Hivyo mfano ufuatao hapa utaonekana kuwa batili.

```js
// Won't work

function badFunction(...args, ...moreArgs) {
  // some code here
}
```

Kikwazo kingine ni kwamba koma za mwisho haziruhusiwi baada ya kigezo cha rest: `function exampleFunction(a, b, ...restOfArgs, )`.

Kikwazo kingine ni kwamba kigezo cha rest hakiwezi kuwa na thamani ya msingi. Vinginevyo `SyntaxError` itatupwa.

```js
function badFunction(...args = [1,2]){
  // some code here
}
```

Basi ni tofauti gani kati ya kitu cha `arguments` na vigezo vya `rest`?

Tofauti kuu ni kwamba kitu cha `arguments` si safu halisi kwa hivyo hakitumi njia kama `includes`, `pop` na `push`. Lakini kigezo cha rest ni mfano wa `Array`. Hivyo unaweza kutumia njia halali za safu zilizojengwa bila ya kuhitaji kuibadilisha kuwa safu halisi kwanza.

:::interactive_editor

```js
function hasCat(...args) {
  return args.includes("cat");
}

console.log(hasCat("dog", "chicken", "cat")); // true
console.log(hasCat("dog", "chicken", "horse")); // false
```
