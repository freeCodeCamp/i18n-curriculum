# --description--

تحدثنا قليلاً عن Git وGitHub، والآن يمكننا البدء في تجميع كل شيء معًا. للقيام بذلك، تحتاج إلى مستودع. لكن ما هو المستودع؟ يمكنك التفكير في المستودع كحاوية لمشروع - إذا كنت تعمل على تطبيق، فستحتفظ بملفات ذلك التطبيق معًا في مستودع.

يمكن أن تكون المستودعات محلية على جهاز الكمبيوتر الخاص بك، أو بعيدة على خدمة مثل GitHub. لإنشاء مستودع، يمكنك البدء إما محليًا أو عن بُعد - النتيجة ستكون نفسها، لذا اختر الطريقة التي تناسب سير عملك أفضل. إذا كان سير العمل يبدأ بمستودع بعيد، يمكنك إنشاء واحد مباشرة عبر موقع GitHub باتباع هذه التعليمات:

زر <a href="https://github.com" target="_blank">موقع GitHub</a>، تأكد من تسجيل الدخول، ثم انقر على أيقونة الزائد في أعلى يمين شريط التنقل. سترى عدة خيارات، لكن الخيار الذي يهمك هو "مستودع جديد". اختره لفتح واجهة إنشاء مستودع جديد.

الخيار الأول الذي ستراه هناك هو إمكانية اختيار قالب. من المحتمل أنك لم تقم بإعداد واحد بعد، لكن القالب هو نوع خاص من المستودعات يمكنك استخدامه كنقطة انطلاق لمستودعاتك الجديدة. هذه القوالب مفيدة لتضمين التوثيق أو قوالب القضايا في جميع مشاريعك.

الخيار التالي هو اختيار مالك. يمكنك اختيار حساب المستخدم الخاص بك أو أي من المنظمات التي تملكها أو لديك وصول إليها. ثم يمكنك تسمية المستودع - بشكل عام، ستريد استخدام اسم يصف المشروع الذي تبنيه بشكل موجز. كأفضل الممارسات، يجب تجنب الفراغات في اسم المستودع. سيقوم GitHub تلقائيًا باستبدالها بشرطات.

بعد ذلك، يمكنك إضافة وصف اختياري سيظهر في الصفحة الرئيسية للمستودع.

يمكنك أيضًا اختيار ما إذا كان المستودع عامًا أو خاصًا. يمكن لأي شخص عرض المستودع العام وتنزيله. أما المستودع الخاص فلا يمكن الوصول إليه إلا من قبلك ومن تمنحهم وصولًا صريحًا.

أخيرًا، إذا لم تختر قالبًا، سترى خيارات لتوليد ملفات في المستودع تلقائيًا. سيظهر ملف README في الصفحة الرئيسية للمستودع، وهو طريقة رائعة لتوفير التوثيق ووصف مشروعك. يسمح ملف `.gitignore` بتحديد الملفات التي سيتجاهلها Git - وهذا مفيد لأشياء مثل تبعيات الحزم، وملفات بيئة العمل، وأشياء أخرى لا تريد حفظها. سيقدم GitHub قوالب لهذا الملف تستثني العناصر الشائعة للغة أو بيئة التشغيل المحددة. والترخيص هو وثيقة قانونية تحدد من يمكنه استخدام برنامجك، وما يمكنهم فعله به.

عندما تنتهي من إعداداتك كما تريد، يمكنك النقر على زر "إنشاء مستودع" لإكمال العملية. سيتم إعادة توجيهك تلقائيًا إلى صفحة الهبوط لمستودعك الجديد!

من هنا، يمكنك النقر على زر "الشفرة" للحصول على خيارات استنساخ المستودع إلى جهاز الكمبيوتر الخاص بك.

يمكنك تجاهل علامتي التبويب "Codespaces" و"Copilot" لهذا الدرس. بدلاً من ذلك، سنركز على الخيارات الثلاثة في علامة التبويب "محلي". الخياران الأولان، HTTPS وSSH، يُستخدمان بنفس الطريقة. تستخدم أمر `git clone` مع عنوان URL المقدم في الصندوق لاستنساخ المستودع البعيد إلى جهاز الكمبيوتر. الفرق بينهما هو طرق الاتصال والتوثيق.

لنسخ مستودع عام، لن تحتاج فعليًا إلى التوثيق. لكن لنسخ مستودع خاص، أو للدفع إلى مستودعات عامة أو خاصة، يجب أن تكون مُوثَّقًا. يستخدم عنوان URL البعيد HTTPS لتوثيقك عبر اسم المستخدم وكلمة المرور في GitHub. ومع ذلك، بسبب مخاوف أمنية، لم يعد هذا مدعومًا، ويُطلب منك بدلاً من ذلك استخدام اسم مستخدم ورمز وصول. حتى مع ذلك، هذه طريقة أقل أمانًا بكثير من SSH، التي تستخدم مفاتيحك الخاصة والعامة. ولها فائدة إضافية تتمثل في تسجيل دخولك تلقائيًا، بدلاً من الحاجة إلى تقديم رمز في كل مرة. سيكون هناك درس منفصل يتعمق في SSH وكيفية عمله، لكن يمكنك أيضًا قراءة وثائق GitHub حول كيفية إعداد SSH.

لاستنساخ المستودع، يمكنك تشغيل هذا في الطرفية:

```sh
git clone git@github.com:your-username/your-repository-name
```

سيُنشئ مجلدًا على جهاز الكمبيوتر باسم المستودع الخاص بك ويحتوي على جميع الملفات من المستودع بداخله. من هناك يمكنك استخدام `cd` (تغيير الدليل) إلى المستودع المستنسخ باستخدام:

```sh
cd name-of-cloned-repo
```

بالإضافة إلى ذلك، النسخة المحلية من المستودع ستحتوي على عنوان بعيد يشير إلى النسخة على GitHub. للتحقق من العناوين البعيدة، يمكنك تشغيل الأمر التالي:

```sh
git remote -v
```

خيار آخر لاستنساخ المستودع هو استخدام واجهة سطر الأوامر الخاصة بـ GitHub. تُستخدم هذه الأداة لأداء مهام خاصة بـ GitHub دون مغادرة سطر الأوامر. إذا لم تكن مثبتة لديك، يمكنك الحصول على تعليمات التثبيت من توثيق GitHub - لكن يجب أن تكون متاحة في مدير الحزم الخاص بنظامك.

لاستنساخ مستودع باستخدام أداة CLI، سأستخدم:

```sh
gh repo clone your-username/your-repository-name
```

النتيجة النهائية لهذه الأوامر هي نفسها: سيكون لديك نسخة من المستودع محليًا.

يمكنك أيضًا استخدام واجهة سطر الأوامر الخاصة بـ GitHub لإنشاء مستودع:

```sh
gh repo create
```

إذا لم تمرر أي معلمات، ستقدم لك CLI معالجًا تفاعليًا لإعداد المستودع. سيعطيك هذا المعالج كل الخيارات نفسها الموجودة في واجهة الويب. مع هذه الأداة، ستحصل أيضًا على خيار استنساخ المستودع الجديد فورًا، حتى لا تضطر إلى المرور بالخطوات اليدوية الموضحة سابقًا.

لديك الآن فهم أساسي لكيفية عمل المستودعات، وكيف يمكنك إنشاء واحد لبدء إصدار عملك بشكل صحيح. في الدروس القادمة، ستتعلم كيفية دفع مستودع محلي موجود إلى GitHub وكيفية دفع التغييرات إلى المستودعات البعيدة.
