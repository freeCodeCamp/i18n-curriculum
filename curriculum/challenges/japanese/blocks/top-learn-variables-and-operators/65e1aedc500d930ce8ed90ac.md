# --description--

式に複数の演算子がある場合、実行順序はそれらの優先度、つまり演算子のデフォルトの優先度順によって決まります。

学校で習ったように、式 `1 + 2 * 2` の掛け算は足し算より先に計算されます。これがまさに優先度の考え方です。掛け算は足し算より優先度が高いと言われます。

丸括弧はどんな優先度よりも優先されるので、デフォルトの順序に満足できない場合は丸括弧を使って変更できます。例えば、`(1 + 2) * 2` と書きます。

JavaScriptには多くの演算子があります。すべての演算子には対応する優先度の数値があり、数値が大きい方が先に実行されます。優先度が同じ場合は左から右へ実行されます。

以下は優先度テーブルの抜粋です（覚える必要はありませんが、単項演算子は対応する二項演算子より優先度が高いことに注意してください）：

| 優先度 | 名前             | 記号 |
|--------|------------------|------|
| ...    | ...              | ...  |
| 14     | 単項プラス       | +    |
| 14     | 単項マイナス     | -    |
| 13     | べき乗           | **   |
| 12     | 乗算             | *    |
| 12     | 除算             | /    |
| 11     | 加算             | +    |
| 11     | 減算             | -    |
| ...    | ...              | ...  |
| 2      | 代入             | =    |
| ...    | ...              | ...  |



# --questions--

## --text--

JavaScriptの式で複数の演算子を扱う場合、実行順序について正しいのはどれですか？

## --answers--

演算子は優先度に関係なく右から左に実行されます。

---

優先度が高い演算子が先に実行され、丸括弧はあらかじめ決められた優先度をオーバーライドします。

---

演算子の優先度は演算の実行速度によって決まります。

---

掛け算は常に足し算より優先され、単項演算子は二項演算子の後に実行されます。
