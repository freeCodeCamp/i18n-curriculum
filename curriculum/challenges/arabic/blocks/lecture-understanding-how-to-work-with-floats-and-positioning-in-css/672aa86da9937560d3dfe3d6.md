# --interactive--

تُعد الـ Floats في CSS تقنية صُممت في الأصل للسماح للنص بالالتفاف حول عنصر، مثل صورة. مع مرور الوقت، وجد المطوّرون طرقًا جديدة لاستخدام الـ Floats، حيث طبقوها على تصميم التخطيط بطرق إبداعية. رغم أن طرق التخطيط الحديثة مثل Flexbox وGrid أصبحت أكثر استخدامًا الآن، إلا أن فهم الـ Floats لا يزال مهمًا، خاصة عند العمل مع شفرة قديمة أو الحاجة لتحقيق تأثيرات تخطيط محددة.

عندما يُطفو عنصر، يُزال من التدفق الطبيعي للوثيقة ويُدفع إلى اليسار أو اليمين داخل الحاوية الخاصة به. المحتوى الذي يلي العنصر يلتف حوله، ويملأ المساحة المتبقية. أحد الاستخدامات الكلاسيكية هو لف النص حول الصور، حيث تُطفو الصورة إلى جانب واحد ويلتف النص حولها. لا تزال هذه التقنية مستخدمة على نطاق واسع، خاصة في المقالات والمدونات التي تحتاج إلى وضع الصور بجانب النص. إليك مثال الشفرة:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<div class="container">
  <img src="https://placehold.co/150x150" alt="Placeholder Image">
</div>
```

```css
.container {
  border: 1px solid black;
  padding: 10px;
}

img {
  float: left;
  margin-right: 20px;
}
```

:::

في المثال أعلاه، لا تلتف الحاوية حول الصورة التي تم تعويمها. الصورة خارج التدفق الطبيعي للوثيقة، وانكمشت الحاوية إلى ارتفاع صفر لأنها لا "ترى" العناصر الفرعية المعوّمة.

كانت الـ Floats شائعة أيضًا لإنشاء تخطيطات متعددة الأعمدة قبل انتشار Flexbox وGrid. من خلال تعويم العناصر بجانب بعضها، كان بإمكان المطوّرين إنشاء أعمدة تصطف أفقيًا. مع ذلك، عند استخدام الـ Floats، من المهم معالجة مشكلة انكماش العناصر الأصلية عندما تكون العناصر الفرعية معوّمة. تُطبق تقنية clearfix على عنصر الصنف `container` لحل هذه المشكلة.

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<div class="container">
  <img src="https://placehold.co/150x150" alt="Placeholder Image">
  <p> This is an example of text flowing around a floated image.</p>
</div>
```

```css
.container {
  border: 1px solid black;
}

/* Clearfix CSS */
.container::after {
  content: "";  
  display: block;
  clear: both;
}

img {
  float: left;
  margin-right: 20px;
}
```

:::

- `::after` هو عنصر مزيف يضيف كتلة غير مرئية بعد محتوى الحاوية.
- `content: ""` يضمن وجود العنصر المزيف دون عرض أي محتوى.
- `display: block` يجعل العنصر المزيف عنصرًا على مستوى الكتلة.
- `clear: both` يضمن أن يمسح العنصر المزيف كلا الجانبين من أي عناصر معوّمة فوقه.

في المثال أعلاه، أضفنا فقرة جديدة لجعل الانكماش أكثر وضوحًا. بما أن الفقرة تبقى في التدفق الطبيعي للوثيقة، تتوسع الحاوية بما يكفي لتلتف حولها. ثم نطبق تقنية clearfix لإصلاح الانكماش وجعل حدود الحاوية تظهر بشكل صحيح. يمكنك تجربة إزالة قاعدة clearfix لترى الحاوية تنكمش مرة أخرى.

تضمن تقنية clearfix أن يلتف العنصر الأصل حول أطفاله المعوّمين بشكل صحيح. تجبر clearfix الحاوية الأصلية على "رؤية" العناصر الفرعية المعوّمة بإضافة خاصية clear بعد المحتوى المعوّم.

رغم أن الـ Floats لم تعد الطريقة الأساسية للتخطيطات المعقدة بسبب التقنيات الحديثة مثل Flexbox وGrid، إلا أنها لا تزال تلعب دورًا أساسيًا في بعض السيناريوهات. سواء كنت تلتف النص حول الصور أو تعمل على مشاريع موروثة، فإن فهم كيفية عمل الـ Floats وكيفية مسحها بشكل صحيح هو مفتاح للحفاظ على تخطيطات منظمة ومتجاوبة.
