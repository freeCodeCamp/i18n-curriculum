# --interactive--

高階函式（Higher order functions）是 JavaScript 中一個強大的概念，能顯著提升你的程式碼能力，並使你的程式碼更具彈性和可重複使用性。

本質上，高階函式是指一個函式，它會將一個或多個函式作為引數，傳回一個函式，或兩者皆是。

要理解高階函式，讓我們先將函式視為 JavaScript 中的一等公民。這表示函式可以像其他任何值一樣被對待－它們可以被指定給變數、作為引數傳遞給其他函式，並且可以從函式中傳回。這種彈性就是啟用高階函式產生與使用的關鍵。

高階函式（Higher order functions）的一個常見用途是抽象化複雜的操作行為。

例如，你可能有一個函式（程式）對陣列中的每個元素執行特定的操作行為。你可以創建一個高階函式（程式），將操作行為作為引數傳入，而不是為不同的操作行為撰寫不同的函式（程式）。這讓你能夠用不同的行為重複使用相同的函式（程式）結構。

這裡有一個範例來說明這個概念：

:::interactive_editor

```js
function operateOnArray(arr, operation) {
  let result = [];
  for (let i = 0; i < arr.length; i++) {
    result.push(operation(arr[i]));
}
  return result;
}

function double(x) {
    return x * 2;
}

let numbers = [1, 2, 3, 4, 5];
let doubledNumbers = operateOnArray(numbers, double);
console.log(doubledNumbers); // [2, 4, 6, 8, 10]
```

:::

在這個範例中，`operateOnArray` 是一個高階函式（程式）。它接受一個陣列和一個函式（`operation`）作為引數。然後，它將該操作行為套用到陣列的每個元素。雙倍函式作為引數傳遞給 `operateOnArray`，展示了函式如何被用作值。

高階函式（程式）也可以傳回函式（程式）。這對於基於較一般的函式（程式）建立專門化函式（程式）特別有用。這個概念通常稱為函式（程式）工廠。以下是一個範例：

:::interactive_editor

```js
function multiplyBy(factor) {
  return function(number) {
    return number * factor;
  }
}

let double = multiplyBy(2);
let triple = multiplyBy(3);

console.log(double(5)); // 10
console.log(triple(5)); // 15
```

:::

在這種情況下，`multiplyBy` 是一個高階函式（程式），會傳回一個新的函式（程式）。這個新的函式（程式）會根據傳入 `multiplyBy` 的因子進行專門化。這讓我們能輕鬆地創建訂製的乘法函式（程式）。

高階函式不僅僅是理論上的概念－它們在 JavaScript 中被廣泛使用。

JavaScript 中許多內建的陣列方法，例如 `map()`、`filter()` 和 `reduce()`，都是高階函式（higher order functions）。這些方法會將函式作為引數，並以各種方式將其應用到陣列的元素上。你將在未來的課程中學習更多關於這些方法的內容。

使用高階函式（function）可以讓程式碼更具宣告性且更容易理解。

與其逐步描述如何完成一項任務（命令式程式設計），高階函式允許你描述你想完成的目標（宣告式程式設計）。這可以讓你的程式碼更易讀且更易維護。

當你繼續使用 JavaScript 時，你會經常遇到並使用高階函式（higher order functions）。它們是 JavaScript 函式式程式設計中的關鍵部分，對於撰寫乾淨、高效且靈活的程式碼至關重要。

理解並運用高階函式將大幅提升你撰寫精巧且優雅 JavaScript 程式的能力。
