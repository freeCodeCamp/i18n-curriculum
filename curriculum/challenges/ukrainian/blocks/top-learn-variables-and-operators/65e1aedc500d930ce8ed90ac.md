# --description--

Якщо у виразі є більше одного оператора, порядок виконання визначається їх пріоритетом, або, іншими словами, пріоритетним порядком операторів за замовчуванням.

Зі школи ви знаєте, що множення у виразі `1 + 2 * 2` має обчислюватися перед додаванням. Саме це і є пріоритет. Кажуть, що множення має вищий пріоритет, ніж додавання.

Круглі дужки заміщають будь-який пріоритет, тому якщо вас не влаштовує порядок за замовчуванням, ви можете їх використати, щоб змінити його. Наприклад, напишіть `(1 + 2) * 2`.

У JavaScript є багато операторів. Кожен оператор має відповідний номер пріоритету. Той, у кого більший номер, виконується першим. Якщо пріоритет однаковий, порядок виконання йде зліва направо.

Ось витяг із таблиці пріоритетів (вам не потрібно це запам’ятовувати, але зверніть увагу, що унарні оператори мають вищий пріоритет, ніж відповідні бінарні):

| Пріоритет | Назва            | Знак |
|-----------|------------------|------|
| ...       | ...              | ...  |
| 14        | унарний плюс     | +    |
| 14        | унарне заперечення | -    |
| 13        | піднесення до степеня | **   |
| 12        | множення         | *    |
| 12        | ділення          | /    |
| 11        | додавання        | +    |
| 11        | віднімання       | -    |
| ...       | ...              | ...  |
| 2         | присвоєння       | =    |
| ...       | ...              | ...  |



# --questions--

## --text--

Коли у виразі JavaScript використовується кілька операторів, яке твердження є правильним щодо порядку їх виконання?

## --answers--

Оператори виконуються справа наліво, незалежно від їх пріоритету.

---

Оператори з вищим пріоритетом виконуються першими, а круглі дужки можуть заміщати заданий пріоритет.

---

Пріоритет оператора визначається швидкістю його виконання.

---

Множення завжди має пріоритет над додаванням, а унарні оператори виконуються після бінарних.
