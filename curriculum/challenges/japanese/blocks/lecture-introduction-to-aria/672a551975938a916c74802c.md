# --interactive--

`aria-describedby` 属性は、ページ上の既存のコンテンツを参照することで、スクリーンリーダーユーザーに要素に関する追加情報を提供するために使われます。これは要素とコンテンツ（技術的にはアクセシブルな説明と呼ばれます）との間にプログラム的な関連付けを作り、スクリーンリーダーが要素とやり取りするときに追加情報をユーザーに伝えられるようにします。

`aria-describedby` の最も一般的な使い方は、フォームの入力欄に指示やエラーメッセージを関連付けることです。スクリーンリーダーユーザーはページ内をさまざまな方法で移動するため、入力欄間を移動するときにこれらのメッセージを見逃す場合があります。`aria-describedby` を使うことで、それらのメッセージを確実に聞けるようにします。

どのように機能するか理解するために、いくつか例を見てみましょう。最初の例では、パスワードを受け付ける`form` 要素があります。

インタラクティブエディターを有効にして、パスワードの入力フィールドにいくつか文字を入力してください。プレビューウィンドウでパスワードがマスクされているのがわかります。また、入力に8文字以上を入れるまで`password-help` のテキストが赤いままであることも確認できます。

注意：このインタラクティブな例では、CSSとJavaScriptを使って`password-help` のテキストの色を動的に更新しています。JavaScriptのコードを理解しようと心配しないでください。今後のモジュールでJavaScriptを学びます。

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<form>
  <label for="password">Password:</label>
  <input type="password" id="password" aria-describedby="password-help" />
  <p id="password-help">Your password must be at least 8 characters long.</p>
</form>

<script src="index.js"></script>
```

```css
#password-help {
  color: red;
}
```

```js
const passwordEl = document.getElementById("password");
const passwordHelpText = document.getElementById("password-help");

passwordEl.addEventListener("input", (e) => { 
  const userInput = e.target.value;
  passwordHelpText.style.color = userInput.length >= 8 ? "green" : "red";
});
```

:::

`label` 要素を`Password` のテキストに使い、それをパスワードの`input` フィールドに関連付けています。

また、パスワードの要件を説明する段落要素もあります。`aria-describedby` 属性を使って、パスワードの`input` フィールドと段落要素内のパスワード要件を関連付けています。スクリーンリーダーユーザーがこの`input` とやり取りすると、スクリーンリーダーは`input` の名前（`Password`）を読み上げ、その後パスワード要件も読み上げる場合があります。ただし、これは絶対的な保証ではありません。スクリーンリーダーによっては追加の内容を自動的に読み上げなかったり、特定の状況でのみ読み上げたりすることがあります。これは珍しいことではありません。スクリーンリーダーはそれぞれ異なり、ARIA属性の扱いも独自です。だからといって`aria-describedby` の使用が無意味になるわけではなく、一般的にスクリーンリーダーユーザーに役立ちます。

`aria-describedby` 属性のもう一つの良い使い方は、削除`button` がある場合です。以下は削除`button` と、そのボタンがクリックされたときに何が起こるかを説明するメッセージの例です。

:::interactive_editor

```html
<button aria-describedby="delete-message">Delete</button>

<p id="delete-message">Warning! All deletions are permanent.</p>
```

:::

先ほどの例と同様に、削除ボタンとメッセージを`aria-describedby` 属性で関連付けています。`id` の値と`aria-describedby` 属性の値は一致している必要があります。

`aria-describedby` 属性は、要素に関する追加情報がスクリーンリーダーユーザーに提供されるようにするための強力な属性です。主にフォームの入力欄に指示やエラーメッセージを関連付けるために使われ、スクリーンリーダーユーザーがフォームを移動するときにこれらのメッセージを見逃す可能性を減らします。
