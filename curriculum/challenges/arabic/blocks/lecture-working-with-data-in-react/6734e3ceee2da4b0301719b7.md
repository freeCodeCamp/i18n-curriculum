# --description--

في الدروس السابقة، تعلمنا كيفية بناء مكوِّنات صغيرة في React مثل هذا:

```jsx
function Greeting() {
  const developerName = "Jessica";
  return <h1>Hi {developerName}!</h1>;
}
```

يمكننا اختيار تضمين هذا المكوِّن داخل مكوِّن أب آخر أو داخل المكوِّن الجذري مثل هذا:

```jsx
function App() {
  return <Greeting />;
}

function Greeting() {
  const developerName = "Jessica";
  return <h1>Hi {developerName}!</h1>;
}
```

بينما ستعمل هذه الشفرة بشكل جيد وتعرض نتيجة `Hi Jessica!` على الشاشة، إلا أن هذا المكوِّن ليس مرنًا جدًا.

ماذا لو أردنا عرض اسم مختلف مثل Naomi أو Tom أو Oliver؟ هنا تأتي أهمية الخصائص في React. الخصائص، وهي اختصار لـ properties، هي الطريقة التي تمرر بها المكوِّنات الأب البيانات إلى المكوِّن الفرعي. يمكن أن تكون الخصائص من أي نوع: سلاسل نصية، أرقام، قيم منطقية، كائنات، أو مصفوفات.

لنحدّث مثالنا السابق ليقبل الآن خاصية `name`:

```jsx
function App() {
  return <Greeting name="Jessica" />;
}

export default App;

function Greeting(props) {
  console.log(props);
  return <h1>Hi {props.name}!</h1>;
}
```

بالنسبة للمكوِّن الفرعي المسمى `Greeting`، نستخدم الآن `props.name` بدلاً من كتابة الاسم `"Jessica"` بشكل ثابت. كما نسجل `props` في وحدة التحكم حيث تظهر ككائن.

ثم، داخل مكوِّن الأب `App`، نمرر القيمة إلى الخاصية `name` لكي تُمرر إلى المكوِّن الفرعي. ستكون النتيجة نفسها على الشاشة كما في السابق، لكننا أنشأنا مكوِّنًا أكثر مرونة.

الآن لدينا القدرة على إعادة استخدام المكوِّن الفرعي عدة مرات وتمرير أسماء مختلفة في كل مرة:

```jsx
function App() {
  return (
    <>
      <Greeting name="Naomi" />
      <Greeting name="Tom" />
      <Greeting name="Jessica" />
      <Greeting name="Oliver" />
    </>
  );
}
```

يمكنك أيضًا اختيار استخدام التفكيك في الخصائص لجعل الشفرة أكثر وضوحًا. إليك كيف يمكنك إعادة كتابة مكوِّن `Greeting`:

```jsx
function Greeting({ name }) {
  return <h1>Hi {name}!</h1>;
}
```

تحقق هذه الشفرة نفس النتيجة لكنها توضح الخصائص التي يتوقع المكوِّن استقبالها.

يوفر React أيضًا خاصية خاصة تسمى `children`. أي JSX تضعه بين وسم الفتح ووسم الإغلاق لمكوِّن يُمرر إلى المكوِّن كـ `children`.

على سبيل المثال:

```jsx
function Card({ children }) {
  return <div className="card">{children}</div>;
}

function App() {
  return (
    <Card>
      <h2>Hello</h2>
    </Card>
  );
}
```

في هذه الحالة، يصبح JSX داخل `<Card>...</Card>` هو الخاصية `children`. وسيكون الناتج المعروض في DOM هو:

```html
<div class="card">
  <h2>Hello</h2>
</div>
```

يُستخدم هذا النمط غالبًا لتكوين المكوِّنات، حيث يلتف مكوِّن حول عناصر واجهة مستخدم أخرى.

أحيانًا، قد يكون لديك الكثير من الخصائص التي يجب تمريرها كمُدخلات. بدلًا من تمريرها واحدة تلو الأخرى، يمكنك استخدام معامل النشر (`...`) بعد تحويلها إلى كائن.

إليك مثال على مكوِّن فرعي جديد يسمى `DeveloperCard`:

```jsx
function DeveloperCard({ name, age, country }) {
  return (
    <div className="developer-card">
      <h1>Developer: {name}</h1>
      <p>Age: {age}</p>
      <p>Country: {country}</p>
    </div>
  );
}
```

يقبل مكوِّن `DeveloperCard` هذا ثلاث خصائص: `name` و `age` و `country`.

في مكوِّن الأب `App`، يمكننا استخدام صيغة النشر لتمرير كل الخصائص من كائن كخصائص فردية إلى المكوِّن الفرعي:

```jsx
function App() {
  const developerObj = {
    name: "Alice",
    age: 30,
    country: "USA",
  };

  return (
    <div className="App">
      <DeveloperCard {...developerObj} />
    </div>
  );
}
```

هذا مفيد بشكل خاص عند العمل مع مصفوفات من الكائنات وتمرير مجموعات متعددة من الخصائص إلى المكوِّنات الفرعية. على سبيل المثال، قد يكون لديك قائمة من المطوِّرين حيث يحتوي كل كائن في المصفوفة على نفس الهيكل لكنه يمثل شخصًا مختلفًا.

ستتعلم المزيد عن كيفية عرض القوائم في المصفوفات في الدروس القادمة.

استخدام الخصائص في React يجعل مكوِّناتك أكثر مرونة وقابلية لإعادة الاستخدام، مما يسمح لك ببناء واجهات مستخدم أكثر تعقيدًا. مع ذلك، من المهم أن تعرف أن الخصائص غير قابلة للتغيير، أي لا يمكن تعديلها بعد تمريرها إلى المكوِّن. إذا كنت بحاجة إلى التعامل مع مدخلات المستخدم وتعديل البيانات، يجب عليك استخدام الحالة بدلاً من ذلك. ستتعلم المزيد عن إدارة الحالة في الدروس القادمة.
