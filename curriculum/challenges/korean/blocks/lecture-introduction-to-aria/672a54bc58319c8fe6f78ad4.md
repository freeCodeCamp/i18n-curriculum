# --interactive--

화면 읽기 프로그램 같은 보조 기술을 사용하는 사람들에게는 숨기면서도 콘텐츠를 표시해야 할 때 `aria-hidden` 속성을 사용할 수 있습니다.

숨기려는 HTML 요소에 이 속성을 추가하고 값을 `true`로 설정하기만 하면 됩니다. 여기서 볼 수 있듯이 `aria-hidden="true"`.

이 속성은 요소와 그 자식 모두를 접근성 트리에서 숨기면서 페이지에서는 보이도록 유지합니다. 일반적인 사용 사례는 다음과 같습니다:

- 장식용 아이콘과 이미지.
- 중복된 콘텐츠.

`aria-hidden`는 화면 읽기 프로그램 같은 보조 기술에서만 콘텐츠를 숨긴다는 점을 기억하는 것이 중요합니다. 만약 모든 사용자에게 콘텐츠를 숨겨야 한다면 `aria-hidden`를 사용해서는 안 됩니다. 예를 들어, 현재 접힌 상태인 햄버거 메뉴는 화면 읽기 프로그램 사용자뿐 아니라 모든 키보드 사용자에게 숨겨져야 합니다. 이 경우 메뉴가 접힐 때 DOM에서 제거하도록 CSS `display` 속성을 `none`로 설정할 수 있습니다.

키보드로 포커스할 수 있는 요소를 숨기기 위해 `aria-hidden`를 절대 사용하지 마세요. `aria-hidden` 속성은 요소를 접근성 트리에서만 제거합니다. DOM에서는 제거하지 않습니다. 따라서 화면 읽기 프로그램 사용자가 탭으로 요소에 접근할 수는 있지만, 화면 읽기 프로그램이 요소를 알리지 않아 사실상 "아무것도 없는" 곳에 포커스가 가게 됩니다.

다음은 `aria-hidden` 속성을 값 `true`로 추가해 아이콘을 접근성 트리에서 숨기는 예입니다.

아이콘과 텍스트가 같은 목적을 위해 중복될 때 혼란을 피하기 위해 텍스트만 보조 기술에 노출합니다.

**참고**: 이 인터랙티브 예제는 Font Awesome CDN을 포함하여 미리보기 창에 기어 아이콘이 표시됩니다. 미리보기를 보려면 인터랙티브 편집기를 활성화해야 합니다.

:::interactive_editor

```html
<head>
  <!-- Font Awesome CDN -->
  <link
    rel="stylesheet"
    href="https://cdn.jsdelivr.net/npm/@fortawesome/fontawesome-free@6.5.0/css/all.min.css"
  />
</head>

<button>
  <i class="fa-solid fa-gear" aria-hidden="true"></i>
  <span class="label">Settings</span>
</button>
```

:::

다음과 같은 경우에는 `aria-hidden`를 사용할 필요가 없습니다:

- HTML 요소에 이미 `hidden` 속성이 있는 경우.
- 요소나 조상 요소가 이미 `display: none`로 숨겨진 경우.
- 요소나 조상 요소가 이미 `visibility: hidden`로 숨겨진 경우.

이 세 경우 모두 요소가 이미 DOM에서 제거되어 접근성 트리에서도 숨겨지므로 `aria-hidden` 속성은 필요하지 않습니다.

모든 ARIA 속성을 사용할 때처럼 보조 기술로 항상 테스트하고, 가능하면 장애가 있는 분들이 직접 테스트해 이해하기 쉬운지 확인하세요. 숨겨진 요소가 있어도 이해하기 쉬워야 합니다.

또한 `aria-hidden`를 `false`로 설정해도 부모 중 하나가 이 속성을 `true`로 설정하면 요소가 보조 기술에 노출되지 않는다는 점을 알아두세요.

`aria-hidden` 속성은 화면 읽기 프로그램 사용자 같은 보조 기술 사용자에게 요소를 숨기는 데 사용됩니다.

순수 장식용 요소나 중복 콘텐츠를 숨기는 데 유용하지만 접근성을 저해하지 않도록 신중히 사용해야 합니다.

일반적으로 페이지에서 제공하는 모든 콘텐츠와 기능은 보조 기술 사용자도 이용할 수 있어야 합니다. `aria-hidden`의 사용 사례는 매우 제한적이며, 순수 장식용이나 중복 정보를 제거해 화면 읽기 프로그램 사용자의 경험을 깔끔하게 만드는 데 주로 한정해야 합니다. 화면 읽기 프로그램 사용자가 관심 없을 것 같다고 생각하는 콘텐츠를 `aria-hidden`로 숨기지 마세요. 화면 읽기 프로그램 사용자도 페이지의 모든 정보에 접근할 권리가 있습니다.

이 모범 사례를 따르고 사용자 경험을 테스트하면 모두를 위한 포용적인 온라인 경험을 만들 수 있습니다.
