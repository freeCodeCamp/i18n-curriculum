# --interactive--

تتيح لك الجمل الشرطية اتخاذ قرارات في شفرة JavaScript الخاصة بك. تسمح لبرنامجك بالتدفق بطريقة معينة بناءً على شروط محددة. لنلقِ نظرة على كيفية عمل `if`، و`else if`، و`else`، والمعامل الثلاثي للتحكم في تدفق الشفرة.

تأخذ جملة `if` شرطًا وتنفذ كتلة شفرة إذا كان هذا الشرط قيمة صحيحة. القيم الصحيحة هي أي قيم تُنتج `true` عند تقييمها في سياق قيمة منطقية مثل جملة `if`. إليك أمثلة على القيم الصحيحة:

- سلاسل نصية غير فارغة، مثل `hello`

- أي رقم غير `0` و`-0`، مثل `4`، و`-5`، وغيرها

- مصفوفات

- كائنات

- القيمة المنطقية `true`

من ناحية أخرى، القيم الخاطئة هي القيم التي تُقيّم إلى `false` في سياق قيمة منطقية. لدى JavaScript عدد قليل من القيم الخاطئة، مما يجعل تذكرها سهلاً. إليك بعض القيم الخاطئة:

- القيمة المنطقية `false`

- `0` (صفر)

- `""` (سلسلة نصية فارغة)

- `null`

- `undefined`

- `NaN` (ليس رقمًا)

الآن، بعد أن أصبح لدينا فهم أساسي للقيم الصحيحة والخاطئة، لنرَ كيف تعمل مع جمل `if`. في هذا المثال الأول، نستخدم بعض جمل `if` للتحقق من القيم الصحيحة والخاطئة:

:::interactive_editor

```js
if (null) {
  console.log("This will not run.");
}

if ("freeCodeCamp") {
  console.log("This will run.");
}
```

:::

بما أن `null` قيمة خاطئة، فلن يتم تسجيل الرسالة داخل الكتلة في وحدة التحكم أبدًا. أما في جملة `if` الثانية، فالسلسلة النصية `freeCodeCamp` قيمة صحيحة، وستُعتبر `true` في هذا السياق المنطقي لجملة `if`. ونتيجة لذلك، ستُسجل الرسالة `This will run.` في وحدة التحكم.

لنلقِ نظرة على بعض الأمثلة الإضافية حول كيفية عمل جمل `if` مع عوامل المقارنة المختلفة. إليك مثالًا على استخدام جملة `if` للتحقق مما إذا كان المستخدم مؤهلاً للتصويت:

:::interactive_editor

```js
const age = 22;

if (age >= 18) {
 console.log("You're eligible to vote"); // You're eligible to vote
}
```

:::

في هذا المثال، بما أن `age` حاليًا `22`، فهذا يعني أن الشرط سيُقيّم إلى `true` لأن `22` أكبر من أو يساوي `18`. لذا ستُسجل الرسالة `You're eligible to vote` في وحدة التحكم. إذا غيرنا المثال ليصبح `age` الآن `15`، فسيُقيّم الشرط إلى `false` ولن تُسجل الرسالة في وحدة التحكم:

:::interactive_editor

```js
const age = 15;

if (age >= 18) {
 console.log("You're eligible to vote"); // Code not running because age is less than 18
}
```

:::

عندما يكون الشرط `false`، يمكنك استخدام بند `else`:

:::interactive_editor

```js
const age = 15;

if (age >= 18) {
 console.log("You're eligible to vote");
} else {
 console.log("You're not eligible to vote"); // You're not eligible to vote
}
```

:::

في هذا المثال، `15` ليست أكبر من أو تساوي `18`، لذا سيكون الشرط `false`. ستُنفذ الشفرة داخل كتلة `else` في هذه الحالة.

إذا أردت التحقق من شروط متعددة، يمكنك استخدام كتلة `else if`. هذا يسمح لبرنامجك بالاختيار بين أكثر من مسارَين.

:::interactive_editor

```js
const score = 87;

if (score >= 90) {
 console.log('You got an A'); 
} else if (score >= 80) {
 console.log('You got a B'); // You got a B
} else if (score >= 70) {
 console.log('You got a C');
} else {
 console.log('You failed! You need to study more!');
}
```

:::

بما أن `score` حاليًا `87`، فستُسجل الرسالة `You got a B` في وحدة التحكم.

المعامل الثلاثي هو طريقة مختصرة لكتابة جمل `if/else` بسيطة. يتكون من ثلاثة أجزاء: شرط، نتيجة إذا كان الشرط صحيحًا، ونتيجة إذا كان خاطئًا. إليك الصيغة النحوية الأساسية:

```js
condition ? expressionIfTrue : expressionIfFalse;
```

إليك مثالًا يتعامل مع درجات الحرارة بالدرجة المئوية:

:::interactive_editor

```js
const temperature = 30;
const weather = temperature > 25 ? 'sunny' : 'cool';

console.log(`It's a ${weather} day!`);
```

:::

إذا كانت `temperature` أكبر من `25`، تسجل الشفرة أعلاه `It's a sunny day!`. وإذا كانت `temperature` أقل من أو تساوي `25`، تسجل `It's a cool day!`.

فأيّهما تستخدم بين جملة if والمعامل الثلاثي؟ استخدم المعامل الثلاثي عند التعامل مع شرط واحد أو تعبير واحد، أو عندما تريد صيغة مختصرة للمنطق البسيط. استخدم جمل `if/else` عند التعامل مع شروط معقدة وجمل متعددة، إذ تصبح الأمور غير قابلة للقراءة إذا قمت بتسكين معاملات ثلاثية.
