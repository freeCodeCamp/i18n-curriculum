# --description--

Same-Origin 정책은 웹 브라우저에 내장된 안전장치로, 한 웹사이트가 다른 웹사이트의 개인 데이터를 읽지 못하게 막습니다. 이 규칙은 개인 계정을 해커로부터 보호합니다.

Origin은 웹 프로토콜, 도메인 이름, 포트 번호의 세 가지 특정 부분으로 구성됩니다.

예를 들어, 주소 `https://example.com:443`에서 프로토콜은 HTTPS이고, 도메인 이름은 `example.com`, 포트 번호는 `443`입니다.

브라우저는 표준 포트일 경우 포트 번호를 숨기는 경우가 많습니다. HTTPS는 기본적으로 포트 443을, HTTP는 포트 80을 사용합니다. 숨겨져 있어도 브라우저는 이를 확인합니다. 세 부분이 모두 정확히 일치할 때만 두 웹사이트가 같은 origin을 가집니다.

이 엄격한 규칙에 따라 브라우저는 한 웹사이트가 다른 웹사이트를 엿보는 것을 차단합니다. 예를 들어, 한 탭에서 온라인 은행 계정을 열고 다른 탭에서 악성 웹사이트를 열었다고 가정해 보세요. Same-Origin 정책은 악성 웹사이트가 은행 정보를 훔치는 것을 막습니다.

하지만 현대 웹사이트는 종종 서로 다른 origin 간에 데이터를 공유해야 합니다. 예를 들어, 프론트엔드 웹페이지가 별도의 백엔드 서버에서 데이터를 가져와야 할 수 있습니다. 브라우저의 Same-Origin 규칙은 이러한 안전한 요청을 차단하므로 개발자는 안전하게 데이터를 공유할 방법이 필요했습니다. 이때 CORS가 등장합니다.

CORS는 서버가 브라우저 규칙을 통해 접근을 열 수 있게 하는 시스템입니다. 신뢰할 수 있는 사이트에 데이터 접근 권한을 부여하는 특별한 헤더를 사용합니다.

브라우저가 다른 사이트에 데이터를 요청할 때 일어나는 일은 다음과 같습니다:

* 요청하는 웹사이트 이름을 담은 `Origin` 헤더를 포함합니다
* 수신 서버가 이 이름을 확인합니다
* 서버가 웹사이트를 신뢰하면, 브라우저에 해당 웹사이트가 들어오는 데이터를 읽을 수 있다고 알려주는 특별한 권한 헤더로 응답합니다.

서버가 이 권한을 보내지 않으면 브라우저가 즉시 데이터를 차단합니다. 프론트엔드 웹사이트는 개인 파일 대신 오류 메시지를 받습니다.

Express를 사용하는 Node.js 애플리케이션에서는 `cors` 패키지로 CORS를 구현합니다:

```javascript
const express = require('express');
const cors = require('cors');
const app = express();

// this enables CORS for all origins and all routes for development
app.use(cors());
```

프로덕션 환경에서는 특정 도메인에 접근을 제한하기 위해 구성 객체를 전달해야 할 수도 있습니다:

```javascript
const corsOptions = {
  origin: ['https://yourapp.com', 'https://admin.yourapp.com'],
  credentials: true
};

app.use(cors(corsOptions));
```

순수 Node.js를 사용할 경우, 모든 요청에 대해 직접 CORS 응답 헤더를 설정하여 수동으로 처리해야 합니다:

```javascript
res.setHeader('Access-Control-Allow-Origin', 'https://example.com');
res.setHeader('Access-Control-Allow-Methods', 'GET, POST, PUT, DELETE, OPTIONS');
res.setHeader('Access-Control-Allow-Headers', 'Content-Type, Authorization');
```
