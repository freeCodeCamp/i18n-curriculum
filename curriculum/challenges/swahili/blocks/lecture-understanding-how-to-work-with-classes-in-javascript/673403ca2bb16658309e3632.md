# --interactive--

Umejifunza kwamba madarasa ni kama ramani za jumla unazoweza kutumia kuunda vitu.

Ramani hizi zinahitaji njia ya jumla ya kurejelea kitu maalum kinachoundwa, kinachofikiliwa, au kinachobadilishwa ndani ya msimbo. Hicho ndicho hasa neno la kifunguo `this` linapotumika. Linarejelea muktadha ambapo msimbo unapaswa kuendeshwa. Ukilitumia ndani ya njia, thamani ya `this` itakuwa rejea kwa kitu kinachohusiana na njia hiyo. Kwa njia hii, unaweza kufikia sifa na njia za kitu ndani ya darasa na kutumia tena njia hiyo kwa vitu tofauti.

Kwa mfano, hapa tuna darasa la `Dessert`. Wakati huu, tunafafanua sifa ya `hasPeanuts`.

```js
class Dessert {
  constructor(hasPeanuts) {
    this.hasPeanuts = hasPeanuts;
  }
}
```

Angalia kwamba tunatumia neno la kifunguo `this` ndani ya kijenzi. Kumbuka kwamba kijenzi huitwa moja kwa moja wakati kitu kipya cha darasa kinapoundwa.

Neno la kifunguo `this` linaambia programu kwamba `hasPeanuts` inapaswa kuwa sifa ya kitu kinachoundwa kwa sasa.

Sehemu ya kulia ya taarifa ya mgawo wa thamani inaweka thamani ya kigezo cha `hasPeanuts` kwa sifa ya `hasPeanuts` ya kitu kipya.

```js
this.hasPeanuts = hasPeanuts;
```

Hiyo ni katika muktadha wa kijenzi lakini pia unaweza kutumia neno la kifunguo `this` ndani ya njia nyingine.

Huu ni darasa letu la `Dessert` lenye mabadiliko machache. Sasa tuna sifa mbili: `name` na `price`.

```js
class Dessert {
  constructor(name, price) {
    this.name = name;
    this.price = price;
  }

  showPrice() {
    console.log(`The price of ${this.name} is $${this.price}.`);
  }
}
```

Pia tuna njia mpya ya `showPrice`. Katika njia hii, tunachapisha ujumbe wa maelezo kwenye konsoli ukiwa na jina na bei ya kitu cha dessert.

Angalia jinsi tunavyotumia neno la kifunguo `this` kufikia sifa za kitu ndani ya njia ya `showPrice`.

Thamani gani itakuwa ya neno la kifunguo `this` katika muktadha wa njia hii? Itarejelea kitu ambacho njia hii inaitwa juu yake. Katika kesi hii, tunaunda mfano mpya wa Dessert wenye jina `"Brownie"` na bei `$5.99`:

```js
const brownie = new Dessert("Brownie", 5.99);
```

Ikiwa tutaita njia ya `.showPrice()` kwenye kitu hiki:

```js
brownie.showPrice();
```

Hapa ni mfano kamili:

:::interactive_editor

```js
class Dessert {
  constructor(name, price) {
    this.name = name;
    this.price = price;
  }

  showPrice() {
    console.log(`The price of ${this.name} is $${this.price}.`);
  }
}

const brownie = new Dessert("Brownie", 5.99);
brownie.showPrice();
console.log(brownie);
```

:::

Jina na bei ya kitu cha sasa (`brownie`) zimebadilishwa katika mfuatano wa herufi.

Hii yote ni shukrani kwa neno la kifunguo `this`, ambalo linakupa njia ya jumla ya kurejelea kitu cha sasa ndani ya darasa. Hii inakuwezesha kutumia tena njia hiyo kwa vitu tofauti.

Hii ni katika muktadha wa njia za vitu lakini neno la kifunguo `this` pia litakuwa na thamani katika vitendakazi huru na vitendakazi vya mshale.

Thamani yake ndani ya kitendakazi huru kawaida hutaja kitu cha ulimwengu (katika hali isiyo kali) au `undefined` (katika hali kali).

Thamani ya neno la kifunguo `this` ni tofauti kidogo katika vitendakazi vya mshale. Vitendakazi vya mshale vinarithi thamani ya `this` kutoka kwa wigo unaozunguka ambapo vimefafanuliwa. Havundwi na uhusiano wao wa `this`. Hii inamaanisha kwamba thamani ya `this` inategemea muktadha ambapo kitendakazi cha mshale kimefafanuliwa, si mahali au jinsi kitendakazi hicho kinavyotumika. Hii hufanya vitendakazi vya mshale kuwa na msaada kwa wito wa kurudisha au kuhifadhi muktadha.

Hata hivyo, kwa sababu hii, thamani ya `this` haitarejelea kitu cha sasa katika njia za vitu, hivyo hili ni jambo unalopaswa kulizingatia unapotumia vitendakazi vya mshale.

Hali hiyo hiyo inahusiana na vitendakazi vya mshale vilivyofafanuliwa ndani ya vitendakazi vingine. Vitarithi thamani ya `this` kutoka kwa wigo wa mzazi wao.

Neno la kifunguo `this` ni chombo chenye nguvu cha kufikia na kudhibiti vitu. Kuelewa jinsi linavyofanya kazi ni muhimu kwa kumiliki programu za vitu katika JavaScript.
