# --description--

Cuando construyes proyectos en JavaScript, a menudo necesitas herramientas adicionales para ayudarte a completar las tareas. Incluso las aplicaciones simples pueden necesitar funciones como guardar datos, iniciar sesión, enrutamiento para moverse entre páginas, y así sucesivamente.

En lugar de escribir todo este código desde cero, puedes usar paquetes de código abierto que otros desarrolladores ya han creado y compartido.

Aquí es donde entra npm. npm es un enorme registro en línea de software de código abierto. Piensa en ello como una tienda de aplicaciones para código: puedes buscar lo que necesitas, instalarlo y comenzar a usarlo en tu proyecto.

Mucha gente piensa que npm significa "Node Package Manager," pero ese no es su significado oficial. En documentación antigua de npm, se describía como una "abreviatura bacrónica recursiva de 'npm is not an acronym'." En otras palabras, npm no significa oficialmente nada.

Hoy, npm se refiere a tres partes relacionadas:

* El sitio web npmjs.com, donde puedes buscar paquetes en el registro.
    
* El registro de npm, que es una gran base de datos pública que almacena todos los paquetes.
    
* La herramienta CLI de npm que se instala con Node.js y te permite instalar y gestionar paquetes.
    

Por ejemplo, si necesitas gestionar estado, manejar fechas, enviar correos electrónicos o hacer solicitudes HTTP en tu proyecto, probablemente ya exista un paquete npm que haga eso. Con un simple comando `npm install`, puedes empezar a usar este software listo en lugar de hacer todo el trabajo tú mismo.

npm también te ayuda a gestionar las dependencias de tu proyecto. Esto significa que realiza un seguimiento de los paquetes que usas en tus proyectos y la versión de cada uno de esos paquetes. Lo hace con los archivos `package.json` y `package-lock.json`, sobre los cuales aprenderás en lecciones futuras.

Yarn, PNPM y Bun son algunas alternativas a npm que se han vuelto populares con el tiempo. Aunque cada una ofrece beneficios en áreas específicas como la velocidad de instalación y la eficiencia del almacenamiento, npm sigue siendo la más utilizada.

En las próximas lecciones, aprenderás varios comandos de npm, cómo instalar paquetes, sobre los archivos `package.json` y `package-lock.json`, e incluso cómo publicar tu propio paquete en el registro de npm.
