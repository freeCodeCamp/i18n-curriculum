# --interactive--

障害を持つ方を含め、すべてのユーザーが問題なくウェブサイトにアクセスできることは非常に重要です。

スクリーンリーダーを使う人にとって、`aria-label` と `aria-labelledby` 属性は、わかりにくかったり見えなかったりするページ要素に関する重要な情報を提供します。

ここでは `aria-label` と `aria-labelledby` 属性が何であるか、そして視覚障害や関連する障害を持つ人々にウェブをアクセシブルにするために果たす役割を見ていきましょう。

`aria-label` と `aria-labelledby` の両方が aria という接頭辞付きであることに気づくでしょう。これは何を意味するのでしょうか？ARIA は Accessible Rich Internet Applications の略です。これは `aria-` という接頭辞が付いた属性のセットで、開発者が要素の目的を支援技術に伝えられるようにします。`aria-label` 属性はインタラクティブな要素のための見えないラベルです。スクリーンリーダーが読み上げられるテキストラベルを要素に追加します。

`aria-label` は、目に見えるテキストがないけれどもスクリーンリーダーで説明する必要がある要素に特に役立ちます。例えば、アイコンだけのボタンは、その目的を伝えるために `aria-label` が必要なことが多いです。

例を見てみましょう：

```html
<button aria-label="Search">
  <i class="fa-solid fa-magnifying-glass"></i>
</button>
```

この場合、ボタンにはアイコンしか含まれていませんが、スクリーンリーダーはこのボタンを `Search, button` として読み上げるかもしれません。`aria-label` 属性は、アイコンの代わりにスクリーンリーダーが使うテキストを指定します。

もしボタンにアイコンの代わりに「Search」というテキストが含まれていれば、テキストがボタンのラベルになるため、`aria-label` 属性は必要ありません。

入力要素の場合、`aria-label` 属性は、入力に関連付けられた見えるラベルがない場合に直接ラベルを提供します。

`aria-labelledby` 属性は `aria-label` 属性とまったく同じことをしますが、属性内に直接テキストを定義する代わりに、ページ上にすでに存在するテキストへの参照を使います。既存のテキストには `id` 属性が必要で、`aria-labelledby` 属性の参照値として使われます。

例を見てみましょう：

:::interactive_editor

```html
<input type="text" aria-labelledby="search-btn">
<button type="button" id="search-btn">Search</button>
```

:::

この場合、ボタンのテキストが検索入力のラベルとして使われています。スクリーンリーダーは入力を `Search, edit` のように読み上げます。後でボタンのテキストを `Find` に変更したい場合でも、入力のラベルはボタンのテキストを参照しているため自動的に新しいテキストに更新されます。
複数の `id` の値を1つの `aria-labelledby` 属性値にまとめることも可能です。やり方は次の通りです：

:::interactive_editor

```html
<div>
  <span id="volume-label">Volume</span>
  <span id="volume-details">Adjust the volume level</span>
  <input
    type="range"
    min="0"
    max="100"
    value="30"
    aria-labelledby="volume-label volume-details">
</div>
```

:::

スライダーの場合、スクリーンリーダーは `volume-label` と `volume-details` 要素の内容を参照して `Volume Adjust the volume level` と読み上げます。

`aria-label` と `aria-labelledby` の両方の属性がスクリーンリーダーに要素の役割を理解させるのに役立つことがわかりました。では、どちらを使うべきでしょうか？どちらも同じ機能を提供するためどちらでも使えますが、`aria-labelledby` を `aria-label` より使うことにいくつかの利点があります：

- ページの内容を翻訳サービスで翻訳する場合、`aria-label` 属性内のテキストは必ずしも翻訳されないことがあります。
- `aria-labelledby` を使うと、見えるラベルテキストとスクリーンリーダー用の見えないラベルの不一致を防げます。見えるテキストを更新すると見えないラベルも自動的に更新されるためです。
- `aria-labelledby` は複数のテキストソースからなる複雑な見えないラベルをプログラム的に作成するのをずっと簡単にします。

最後に注意点ですが、`aria-label` と `aria-labelledby` を同じ要素に同時に使わないでください。この場合、スクリーンリーダー用の見えないラベルは常に `aria-labelledby` によって決まり、`aria-label` は完全に無視されます。
