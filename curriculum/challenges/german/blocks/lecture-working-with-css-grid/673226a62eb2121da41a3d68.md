# --interactive--

Die `minmax()`-Funktion definiert den Bereich für die Größe einer Grid-Spur und legt fest, wie viel Platz eine Zeile oder Spalte einnehmen kann.

Denken Sie daran, dass Sie die Spurgröße mit Einheiten wie `px` (Pixel), `rem` oder sogar `em` sowie mit Bruch-Einheiten (`fr`) festlegen können.

Die `minmax()`-Funktion geht noch einen Schritt weiter, indem Sie eine Mindestgröße und eine Höchstgröße für die Grid-Spur festlegen können.

Hier ist die Syntax der `minmax()`-Funktion:

```css
minmax(min, max)
```

`min` ist die Mindestgröße der Grid-Spur, die Sie in Pixeln, Prozent oder `auto` angeben können. Und `max` ist die Höchstgröße der Grid-Spur, die Sie mit denselben Einheiten festlegen können.

Die beiden Werte wirken zusammen wie folgt:

- Der `min`-Wert stellt sicher, dass die Grid-Spur niemals kleiner als eine festgelegte Größe wird.

- Der `max`-Wert begrenzt, wie groß die Grid-Spur werden kann.

Die Größe der Grid-Spur passt sich dynamisch zwischen den `min`- und `max`-Werten an, abhängig vom Inhalt und der Größe des Containers.

Schauen wir uns ein praktisches Beispiel an:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<div class="grid-container">
  <div>
    <h2>Item 1</h2>
  </div>
  <div>
    <h2>Item 2</h2>
  </div>
</div>
```

```css
.grid-container {
  display: grid;
  grid-template-columns: minmax(150px, 300px) 1fr;
  gap: 20px;
}

.grid-container > div {
  background: crimson;
  padding: 20px;
  text-align: center;
}
```

:::

Was passiert hier?

Die erste Spalte, `minmax(150px, 300px)`, ist immer mindestens `150px` und höchstens `300px`, je nach verfügbarem Platz.

Die zweite Spalte hingegen, `1fr`, nimmt den restlichen verfügbaren Platz im Grid-Container ein, da keine weiteren Spalten vorhanden sind, mit denen der Platz geteilt werden müsste.

Der Vorteil der `minmax()`-Funktion gegenüber festen Größen und sogar `fr`-Einheiten ist, dass sie flexibler ist und sich daher ideal für Anpassungsfähigkeit und Responsivität eignet.
