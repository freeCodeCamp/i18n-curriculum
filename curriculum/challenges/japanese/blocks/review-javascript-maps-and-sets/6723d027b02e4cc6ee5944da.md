## JavaScript の Sets

- `Set` はデータコレクションを管理するための組み込みのオプションです。
- Sets はその中の各値が一度だけ現れることを保証するため、配列から重複を取り除いたり、異なる値のコレクションを扱うのに便利です。
- `Set` コンストラクタを使って `Set()` を作成できます。

:::interactive_editor

```js
const set = new Set([1, 2, 3, 4, 5]);
console.log(set); // Set { 1, 2, 3, 4, 5 }
```

:::

- Sets は次のメソッドで操作できます。

  - `add()`: `Set` に新しい要素を追加します。
  - `delete()`: `Set` から要素を削除します。
  - `has()`: `Set` に要素が存在するか確認します。
  - `clear()`: `Set` からすべての要素を削除します。
  - `keys()` と `values()`: どちらも `SetIterator` の値を含む `Set` を返します。`keys()` は `values()` の別名なので同じです。
  - `forEach()`: `Set` の値を繰り返し処理するためのものです。

## JavaScript の Weaksets

- `WeakSet` は弱く保持されたオブジェクトを格納できるオブジェクトのコレクションです。

## Sets と WeakSets の違い

- Sets と違い、`WeakSet` は数値や文字列のようなプリミティブをサポートしません。
- `WeakSet` はオブジェクトのみを格納し、それらのオブジェクトへの参照は「弱い」ため、コードの他の場所で使われていなければ自動的に削除されてメモリを解放します。

## JavaScript の Maps

- `Map` はキーと値のペアを保持する組み込みオブジェクトで、オブジェクトと同様です。
- Maps は任意の型のキー（オブジェクトや関数も含む）を許可できる点で標準の JavaScript オブジェクトと異なります。
- `Map` はキーと値のペアの頻繁な追加や削除において標準オブジェクトより優れたパフォーマンスを提供します。
- `Map` コンストラクタを使って `Map()` を作成できます。

:::interactive_editor

```js
const map = new Map([
  ['flower', 'rose'],
  ['fruit', 'apple'],
  ['vegetable', 'carrot']
]);
console.log(map); // Map(3) { 'flower' => 'rose', 'fruit' => 'apple', 'vegetable' => 'carrot' }
```

:::

- Maps は次のメソッドで操作できます。

  - `set()`: `Map` に新しいキーと値のペアを追加します。
  - `get()`: `Map` からキーの値を取得します。
  - `delete()`: `Map` からキーと値のペアを削除します。
  - `has()`: `Map` にキーが存在するか確認します。
  - `clear()`: `Map` からすべてのキーと値のペアを削除します。

Maps と Sets の両方に、ユニークな要素の数を返す `size` プロパティがあることに注意してください。

## JavaScript の WeakMaps

- `WeakMap` は `Map` と同様にキーと値のペアのコレクションですが、キーへの参照が弱いものです。キーはオブジェクトでなければならず、値は任意です。

## Maps と WeakMaps の違い

- WeakMaps は WeakSets と似ており、オブジェクトのみを格納し、それらのオブジェクトへの参照は「弱い」です。
