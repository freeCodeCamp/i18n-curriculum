# --description--

配列はいつでも変更（ミューテート）できるため、特定のデータが配列のどこにあるか、あるいはその要素自体がまだ存在しているかは保証されません。幸いなことに、JavaScript には `indexOf()` という組み込みメソッドがあり、配列に要素が存在するかどうかを簡単かつ素早く確認できます。`indexOf()` は要素をパラメータとして受け取り、呼び出すとその要素の位置（インデックス）を返します。もしその要素が配列に存在しなければ `-1` を返します。

例えば：

```js
let fruits = ['apples', 'pears', 'oranges', 'peaches', 'pears'];

fruits.indexOf('dates');
fruits.indexOf('oranges');
fruits.indexOf('pears');
```

`indexOf('dates')` は `-1` を返し、`indexOf('oranges')` は `2` を返し、`indexOf('pears')` は `1` を返します（それぞれの要素が最初に存在するインデックスです）。

# --instructions--

`indexOf()` は配列に要素が存在するかどうかを素早く確認するのに非常に便利です。配列と要素を引数に取る `quickCheck` という関数を定義しています。`indexOf()` を使って関数を修正し、渡された要素が配列に存在する場合は `true` を返し、存在しない場合は `false` を返すようにしてください。

# --hints--

`quickCheck` 関数は文字列（`true` や `false`）ではなく、ブール値（`"true"` または `"false"`）を返すべきです。

```js
assert.isBoolean(quickCheck(['squash', 'onions', 'shallots'], 'mushrooms'));
```

`quickCheck(["squash", "onions", "shallots"], "mushrooms")` は `false` を返すべきです。

```js
assert.strictEqual(
  quickCheck(['squash', 'onions', 'shallots'], 'mushrooms'),
  false
);
```

`quickCheck(["onions", "squash", "shallots"], "onions")` は `true` を返すべきです。

```js
assert.strictEqual(
  quickCheck(['onions', 'squash', 'shallots'], 'onions'),
  true
);
```

`quickCheck([3, 5, 9, 125, 45, 2], 125)` は `true` を返すべきです。

```js
assert.strictEqual(quickCheck([3, 5, 9, 125, 45, 2], 125), true);
```

`quickCheck([true, false, false], undefined)` は `false` を返すべきです。

```js
assert.strictEqual(quickCheck([true, false, false], undefined), false);
```

`quickCheck` 関数は `indexOf()` メソッドを利用するべきです。

```js
assert.notStrictEqual(quickCheck.toString().search(/\.indexOf\(/), -1);
```
