# --description--

Servir arquivos estáticos é uma necessidade comum em aplicações web. Arquivos estáticos incluem assets como imagens, arquivos CSS, JavaScript do lado do cliente, fontes e outros recursos que não mudam dinamicamente.

O Express facilita servir esses arquivos com seu middleware embutido `express.static()`.

No Express, servir arquivos estáticos significa entregar arquivos diretamente do sistema de arquivos do servidor para o navegador do cliente sem qualquer processamento adicional no servidor.

Por exemplo, quando um navegador solicita arquivos como `/styles.css`, `/script.js` ou `/images/logo.png`, o Express pode localizar esses arquivos em uma pasta especificada e servi-los diretamente. Se um arquivo não for encontrado, o Express responde automaticamente com um 404. Como esses arquivos não precisam ser gerados ou modificados no servidor, eles são chamados de *estáticos*.

Isso facilita servir assets como folhas de estilo e imagens, e elimina a necessidade de definir rotas manualmente para cada arquivo.

Veja como configurar o serviço de arquivos estáticos no Express:

```javascript
const express = require("express");
const path = require("path");
const app = express();
const port = 3000;

// Serve static files from the "public" directory
app.use(express.static(path.join(__dirname, "public")));

app.get("/", (req, res) => {
  res.send("Welcome to the static file demo!");
});

app.listen(port, () => {
  console.log(`Server running at http://localhost:${port}`);
});
```

Primeiro, importamos `express` para criar nosso servidor e `path` para construir caminhos de arquivos de forma segura em diferentes sistemas operacionais.

Em seguida, `app.use(express.static(path.join(__dirname, "public")))` informa ao Express para servir arquivos da pasta `public`. O middleware `express.static()` indica ao Express para servir arquivos da pasta `public`. Qualquer arquivo dentro desse diretório fica acessível ao navegador.

Também definimos uma rota simples `GET` para `/` que envia uma mensagem de boas-vindas. Isso mostra que o serviço de arquivos estáticos pode coexistir com rotas dinâmicas.

Por fim, `app.listen()` inicia o servidor na porta 3000. Uma vez que o servidor está rodando, as requisições por assets estáticos na pasta `public` são tratadas automaticamente.

Para isso funcionar, você precisa de uma pasta chamada `public` para armazenar seus arquivos estáticos. Uma estrutura típica de projeto pode ser assim:

```md
my-app/
│
├── app.js
└── public/
    ├── index.html
    ├── styles.css
    └── image.png
```

Quando um usuário faz uma requisição, por exemplo, visitando `http://localhost:3000/index.html`, o Express verifica se o arquivo solicitado existe dentro do diretório `public`. Se o arquivo existir, o Express o envia diretamente para o navegador. Se o arquivo não existir, o Express passa para o próximo middleware ou retorna uma resposta 404.

Lembre-se que você não precisa definir rotas para cada arquivo, então uma requisição para `http://localhost:3000/styles.css` significa que o Express procura o `public/styles.css` e responde com ele automaticamente.

Você também pode personalizar como os arquivos estáticos são servidos. Por exemplo, pode servir arquivos de múltiplos diretórios ou configurar o comportamento de cache.

Imagine que seu app tem esta estrutura de arquivos:

```md
my-app/
│
├── app.js
│
├── public/
│   ├── index.html
│   └── styles.css
│
└── images/
    └── image.png
```

Veja como registrar múltiplos diretórios para servir arquivos estáticos:

```javascript
app.use(express.static(path.join(__dirname, "public")));
app.use("/images", express.static(path.join(__dirname, "images")));
```

Aqui usamos `app.use(express.static(path.join(__dirname, "public")));` como antes, o que significa que arquivos dentro do diretório `public` são servidos a partir do caminho raiz, como antes.

Mas se você olhar a linha seguinte, verá que passamos `"/images"` como primeiro argumento para `app.use()`, seguido de `express.static(path.join(__dirname, "images"))`. Nesse caso, `"/images"` é um caminho de montagem, que informa ao Express para usar esse middleware apenas quando receber uma requisição para `/images`. Quando isso acontece, `express.static()` é acionado e procura arquivos dentro do diretório `images`.

Então, com essa configuração, veja como as seguintes requisições funcionariam:

*   GET `http://localhost:3000/styles.css`: o Express procura no diretório `public` pelo `styles.css`, encontra o arquivo e o envia para o cliente.
    
*   GET `http://localhost:3000/images/image.png`: como essa requisição começa com `/images`, o Express procura no diretório `images` pelo `image.png`, encontra o arquivo e o envia para o cliente.
    

Servir arquivos estáticos com Express é simples e poderoso. Usando `express.static()`, você pode facilmente servir assets como imagens, folhas de estilo e JavaScript do lado do cliente.

Manter arquivos estáticos em uma pasta dedicada como `public`, ou usar múltiplos diretórios de arquivos e caminhos de montagem, ajuda a manter sua aplicação organizada, fácil de manter e escalável.
