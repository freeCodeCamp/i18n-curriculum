## `Audio` 构造函数和常用方法

- **定义**：`Audio` 构造函数，像其他构造函数一样，是一个使用 `new` 关键字调用的特殊函数。它返回一个 `HTMLAudioElement`，你可以用它为用户播放音频，或追加到 DOM 中让用户自行控制。当你调用构造函数时，可以可选地传入一个 URL 作为（唯一）参数。该 URL 应指向你想播放的音频文件的源。或者，如果你需要动态更改源，可以将 URL 赋值给返回的音频元素的 `src` 属性。
- **`play()` 方法**：此方法用于 `audio` 或 `video` 元素以开始媒体播放。

:::interactive_editor

```html
<audio id="audio" controls>
  <source src="https://www.w3schools.com/html/horse.mp3" type="audio/mpeg">
</audio>

<button id="playBtn">Play Audio</button>

<script>
  const audio = document.getElementById("audio");
  const playBtn = document.getElementById("playBtn");

  playBtn.addEventListener("click", () => {
    audio.play();
  });
</script>
```

:::

- **`pause()` 方法**：此方法用于 `audio` 或 `video` 元素以暂停媒体播放。它维护当前播放位置，因此如果调用 `play()`，会从该位置开始播放。

:::interactive_editor

```html
<audio id="myAudio" controls>
  <source src="https://www.w3schools.com/html/horse.mp3" type="audio/mpeg">
</audio>

<br>

<button id="pauseBtn">Pause Audio</button>

<script>
  document.addEventListener("DOMContentLoaded", () => {
    const audio = document.getElementById("myAudio");
    const pauseBtn = document.getElementById("pauseBtn");

    pauseBtn.addEventListener("click", () => {
      audio.pause();
    });
  });
</script>
```

:::

- **`addTextTrack()` 方法**：此方法允许你为媒体元素指定一个文本轨道——这对于给视频添加字幕特别有用。
- **`fastSeek()` 方法**：此方法允许你将播放位置移动到媒体中的特定时间。

`Audio()` 构造函数既有方法，也有属性。这些属性包括：

- `currentTime`：获取音频当前的播放时间
- `loop`：使音频循环播放，播放结束时自动重新开始
- `muted`：无论音量设置如何，静音媒体元素的音频输出

## 不同的音频和视频格式

- **MIME 类型**：MIME 类型，全称多用途互联网邮件扩展，是一种以编程方式指示文件类型的标准化方式。MIME 类型可以告诉应用程序（例如浏览器）如何处理特定文件。对于音频和视频，MIME 类型表示它是可以嵌入网页的多媒体格式。
- **`source` 元素**：用于指定文件类型和源——可以通过使用多个 `source` 元素包含多种不同类型。当你这样做时，浏览器会根据用户当前环境确定最佳格式。
- **MP3**：一种用于存储音乐、音频或声音的数字文件格式。它是声音录音的压缩版本，使文件大小更小，更易于保存和分享。MP3 拥有最广泛的浏览器支持，MIME 类型为 `audio/mpeg`。
- **MP4**：一种用于存储视频和音频的数字文件格式。它作为一个容器，将视频（图像）和声音（音乐或语音）保存在一个文件中。MP4 的 MIME 类型可以是 `audio/mp4` 或 `video/mp4`，取决于是视频文件还是仅音频。
- 其他格式包括与 Windows Media Player 关联的 `WMV`，以及 `OGG`、`MKV`、`AVI` 等。

## 编解码器（codecs）

- **定义**：编解码器，英文全称 encoder/decoder，是一种可以在模拟和数字格式之间转换音频和视频的算法或软件。编解码器可以作为 MIME 类型的一部分指定。定义编解码器的基本语法是在媒体类型后添加分号，然后是 `codecs=` 和编解码器名称。

## HTMLMediaElement API

- **定义**：`HTMLMediaElement` API 用于控制页面上的音频和视频元素的行为。它扩展了基础的 `HTMLElement` 接口，因此你可以访问基础属性、方法以及诸如 `waiting`、`ended`、`canplay`、`canplaythrough` 等有用事件。方法示例包括 `play()`、`fastSeek()`、`pause()`，以及用于检查浏览器是否可能播放某音频文件的 `canPlayType()`。

## 媒体捕捉和流 API

- **定义**：媒体捕捉和流 API，或称 MediaStream API，用于从设备捕捉音频和视频。要使用该 API，你需要创建一个 `MediaStream` 对象。你可以用构造函数创建，但它不会绑定到用户硬件。相反，全局 `navigator` 对象的 `mediaDevices` 属性提供了一个 `getUserMedia()` 方法供你使用。

```js
window.navigator.mediaDevices.getUserMedia({
  audio: true,
  video: {
    width: {
      min: 1280,
      ideal: 1920,
      max: 3840
    },
    height: {
      min: 720,
      ideal: 1080,
      max: 2160
    }
  }
});
```

## 屏幕捕捉 API

- **定义**：屏幕捕捉 API 用于记录用户的屏幕。该 API 通过调用 `mediaDevices` 对象的 `getDisplayMedia()` 方法并使用返回的媒体流来暴露。

## MediaStream 记录 API

- **定义**：MediaStream Recording API 与 MediaStreams API 协同工作，允许你录制一个 MediaStream（甚至直接录制一个 `HTMLMediaElement`）。

## 媒体源扩展名 API

- **主题**：Media Source Extensions API 允许你通过 `srcObject` 属性将用户的摄像头视频直接传递给视频元素。

## Web 音频 API

- **定义**：Web Audio API 是驱动网页上所有可听内容的基础。该 API 包含重要的对象，如表示专门包含音频数据的缓冲区的 `AudioBuffer`，或用于表现音频处理图的 `AudioContext`。
