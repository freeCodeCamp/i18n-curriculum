# --description--

Die `.env`-Datei ist eine versteckte Datei, die verwendet wird, um Umgebungsvariablen an Ihre Anwendung zu übergeben. Diese Datei ist geheim, nur Sie können darauf zugreifen, und sie kann verwendet werden, um Daten zu speichern, die Sie privat oder verborgen halten möchten. Zum Beispiel können Sie API-Schlüssel von externen Diensten oder Ihre Datenbank-URI speichern. Sie können sie auch verwenden, um Konfigurationsoptionen zu speichern. Durch das Festlegen von Konfigurationsoptionen können Sie das Verhalten Ihrer Anwendung ändern, ohne Code neu schreiben zu müssen.

Die Umgebungsvariablen sind aus der App als `process.env.VAR_NAME` zugänglich. Das `process.env`-Objekt ist ein globales Node-Objekt, und Variablen werden als Strings übergeben. Nach Konvention sind die Variablennamen alle in Großbuchstaben, wobei Wörter durch einen Unterstrich getrennt sind. Die `.env` ist eine Shell-Datei, daher müssen Sie Namen oder Werte nicht in Anführungszeichen setzen. Es ist auch wichtig zu beachten, dass beim Zuweisen von Werten an Ihre Variablen kein Leerzeichen um das Gleichheitszeichen stehen darf, z. B. `VAR_NAME=value`. Normalerweise schreiben Sie jede Variablendefinition in eine eigene Zeile.

# --instructions--

Fügen Sie eine Umgebungsvariable als Konfigurationsoption hinzu.

Erstellen Sie eine `.env`-Datei im Stammverzeichnis Ihres Projekts und speichern Sie die Variable `MESSAGE_STYLE=uppercase` darin.

Greifen Sie dann im `/json`-GET-Route-Handler, den Sie in der letzten Aufgabe erstellt haben, auf `process.env.MESSAGE_STYLE` zu und wandeln Sie den `message` des Antwortobjekts in Großbuchstaben um, wenn die Variable `uppercase` entspricht. Das Antwortobjekt sollte entweder `{"message": "Hello json"}` oder `{"message": "HELLO JSON"}` sein, abhängig vom Wert von `MESSAGE_STYLE`. Beachten Sie, dass Sie den Wert von `process.env.MESSAGE_STYLE` **innerhalb** des Route-Handlers lesen müssen, nicht außerhalb, aufgrund der Art, wie unsere Tests ausgeführt werden.

Sie müssen das `dotenv`-Paket verwenden. Es lädt Umgebungsvariablen aus Ihrer `.env`-Datei in `process.env`. Das `dotenv`-Paket wurde bereits installiert und befindet sich in der `package.json`-Datei Ihres Projekts. Fügen Sie am Anfang Ihrer `myApp.js`-Datei `require('dotenv').config()` hinzu, um die Umgebungsvariablen zu laden.

# --hints--

Die Antwort des Endpunkts `/json` sollte sich entsprechend der Umgebungsvariable `MESSAGE_STYLE` ändern

  const response = await fetch(code + '/_api/use-env-vars');
