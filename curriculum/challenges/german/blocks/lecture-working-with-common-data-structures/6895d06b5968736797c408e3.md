# --description--

Während Ihrer Lernreise sollten Sie daran arbeiten, starke Problemlösungsfähigkeiten zu entwickeln. Diese Kernkompetenzen sind entscheidend, um reale Probleme in Ihrem Arbeitsalltag zu bewältigen.

Das Lösen algorithmischer Aufgaben ist eine großartige Möglichkeit zum Üben. Es erfordert eine analytische Denkweise, die Fähigkeit, das Problem in seine Kernkomponenten zu zerlegen, und eine Lösung zu finden, die die richtige Ausgabe effizient erzeugt.

Aber wo fängt man an?

Es gibt verschiedene Techniken zur Problemlösung, die Sie nutzen können, um sich diesen Herausforderungen zu nähern.

Als Beispiel werden wir einen String in Python umkehren.

Das ist die Aufgabe:

„Gegeben ist ein String. Schreiben Sie einen Algorithmus, der einen neuen String mit den Zeichen in umgekehrter Reihenfolge zurückgibt.“

Das Erste, was Sie tun sollten, wenn Sie auf diese Art von Problem stoßen, ist, die Beschreibung mehrmals zu lesen, um sicherzugehen, dass Sie sie verstehen. Wenn Sie diesen Schritt überspringen oder zu schnell lesen, könnten Sie wichtige Informationen übersehen.

Sobald Sie mit dem Problem vertraut sind, beginnen Sie damit, es in seine Kernkomponenten zu zerlegen.

Fragen Sie sich:

* „Was ist die Eingabe?“
    
* „Was ist die erwartete Ausgabe?“
    
* „Wie kann ich die Eingabe in die erwartete Ausgabe umwandeln?“
    

In diesem Problem können Sie erkennen, dass die Eingabe ein String ist, weil die Aufgabe mit „Gegeben ist ein String…“ beginnt.

Die Ausgabe ist „ein neuer String mit den Zeichen in umgekehrter Reihenfolge.“

Sie müssen also den ursprünglichen String nehmen und umkehren.

Diese erste Analyse mag anfangs etwas repetitiv erscheinen, aber sie ist sehr hilfreich, um sicherzustellen, dass Sie die Anforderungen vollständig verstehen.

Dann sollten Sie darüber nachdenken, wie der Algorithmus, den Sie entwickeln, die Eingabe in die Ausgabe umwandeln wird.

Während dieser Planungs- und Analysephase ist es üblich, Pseudocode zu verwenden, um die notwendigen Schritte zu skizzieren.

**Pseudocode** ist eine abstrakte Beschreibung der Logik des Algorithmus, die allgemein gehalten ist und nicht auf einer bestimmten Programmiersprache basiert.

Pseudocode ist nicht so formell wie echter Code, da er nur für Menschen lesbar sein soll. Er sollte auf einen Blick leicht verständlich sein. Sein Hauptzweck ist es, eine klare Vorstellung von der Folge der auszuführenden Schritte zu geben.

Pseudocode ist üblicherweise eine Mischung aus einer gebräuchlichen Schriftsprache, wie Englisch, mit Programmierkonstrukten wie `IF`, `ELSE`, `FOR` und `WHILE`.

Dies ist ein Beispiel für Pseudocode, den Sie schreiben könnten, um die Aufgabe „String umkehren“ zu lösen.

```md
GET original_string

SET reversed_string = ""

FOR EACH character IN original_string:
  ADD character TO THE BEGINNING OF reversed_string

DISPLAY reversed_string
```

Beachten Sie, wie die Schritte so dargestellt sind, dass sie leicht verständlich sind. Die Wörter und Konstrukte selbst können je nach den von Ihnen verwendeten Standards variieren.

Wenn Sie möchten, könnten Sie diese Schritte in mehreren Programmiersprachen implementieren, die gleiche Logik befolgend, da der Pseudocode unabhängig von der Programmiersprache ist.

Bis zu diesem Punkt haben Sie vielleicht schon erkannt, dass dieses Problem auf viele verschiedene Arten gelöst werden kann. Dies ist nicht der einzige Weg, einen String umzukehren.

Aber denken Sie daran, dass die Wahl des richtigen Algorithmus wichtig ist.

In einer vorherigen Lektion haben Sie etwas über algorithmische Komplexität gelernt und warum es wichtig ist, Algorithmen zu wählen, die in Bezug auf Zeit und Speicher effizient sind.

Hier spielen Sie als Entwickler eine entscheidende Rolle. Sie müssen den effizientesten Algorithmus auswählen, um die Aufgabe zu lösen.

Das Nachdenken über verschiedene verfügbare Algorithmen ist eine wichtige Problemlösungskompetenz, die Sie üben sollten. Nehmen Sie sich einen Moment Zeit und fragen Sie sich, ob die Lösung, die Sie in Ihrem Pseudocode vorschlagen, die beste ist oder nicht.

Zum Beispiel gibt es viele verschiedene Algorithmen zum Sortieren von Elementen, aber einige sind effizienter als andere. Bubble Sort ist zum Beispiel sehr ineffizient für das Sortieren großer Listen, während Quick Sort normalerweise effizienter ist.

Für unsere Aufgabe „String umkehren“ könnten wir eine der folgenden Herangehensweisen verwenden, vorausgesetzt, wir planen, unseren Algorithmus in Python zu implementieren:

* Die erweiterte Slice-Syntax `[::-1]` verwenden, um einen neuen umgekehrten String zu erhalten.
    
* Über die Zeichen von links nach rechts iterieren und das neue Zeichen am Anfang des neuen Strings hinzufügen.
    
* Die Funktion `reversed()` aufrufen, um einen Iterator mit allen Zeichen in umgekehrter Reihenfolge zu erhalten, und dann die Methode `““.join()`, um sie wieder zu einem String zusammenzufügen.
    

Welche sollten Sie verwenden? Das liegt bei Ihnen.

Diese Entscheidungen basierend auf Ihrem Wissen und Ihrer Erfahrung zu treffen, kann einen großen Unterschied in der finalen Leistung Ihrer Anwendung machen. Berücksichtigen Sie verschiedene Ansätze, deren Effizienz, Auswirkungen und Umsetzung.

Fragen Sie sich:

* „Wie werde ich dieses Problem angehen?“
    
* „Welche Datenstrukturen werde ich verwenden?“
    
* „Sind die von mir gewählten Datenstrukturen die effizientesten für das vorliegende Problem?“
    
* „Decke ich alle möglichen Randfälle ab?“
    

Randfälle sind spezifische, gültige Eingaben oder Bedingungen, die an den Grenzen dessen auftreten, was ein Algorithmus verarbeiten sollte.

Zum Beispiel wäre in der Aufgabe „String umkehren“ ein Randfall, einen leeren String als Eingabe zu nehmen. Behandeln Sie diesen korrekt? Wenn nicht, überlegen Sie, wie Sie diesen Randfall am besten behandeln und fügen Sie ihn Ihrem Pseudocode hinzu.

Sobald Sie mit Ihrem Plan zufrieden sind, können Sie zur Implementierungsphase übergehen. In dieser Phase implementieren Sie Ihren Algorithmus in einer Programmiersprache.

Beim Strukturieren Ihres Programms sollten Sie modularen Code schreiben, der leicht zu lesen und zu verstehen ist.

Nutzen Sie die Werkzeuge der Programmiersprache basierend auf Ihrem aktuellen Wissen. Einige Programmiersprachen enthalten eingebaute Lösungen für häufige Probleme und Aufgaben. Verwenden Sie diese, wenn möglich.

Um konsistent zu bleiben, folgen Sie den bewährten Vorgehensweisen der von Ihnen gewählten Programmiersprache.

Testen Sie Ihren Code während des Schreibens und stellen Sie sicher, dass Sie Randfälle angemessen behandeln.

Sobald Ihre Lösung implementiert ist, prüfen Sie, ob sie für alle Beispiele korrekt funktioniert, und überarbeiten Sie Ihren Code gegebenenfalls, um ihn klarer oder einfacher zu machen.

Zurück zu Ihrer Lösung zu gehen, ist sehr wichtig. Entwicklung ist nicht unbedingt ein linearer, schrittweiser Prozess. Sie können jederzeit zu Ihrem Code zurückkehren und Ihre kritischen Denkfähigkeiten nutzen, um ihn zu verbessern.

Dies sind einige gängige Techniken zur Problemlösung, denen Sie folgen können, um algorithmische Aufgaben anzugehen. Wenn Sie konsequent üben, werden Sie Ihre Problemlösungsfähigkeiten nach und nach entwickeln.
