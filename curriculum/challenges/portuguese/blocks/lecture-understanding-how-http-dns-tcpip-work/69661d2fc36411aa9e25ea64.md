# --description--

Vamos aprender sobre os códigos comuns de resposta HTTP.

Códigos de resposta HTTP são códigos numéricos de três dígitos que servidores web usam para se comunicar com clientes para informá-los sobre o que aconteceu com suas requisições.

Eles fornecem informações sobre se a solicitação foi processada com sucesso ou se houve um erro ao processar a solicitação.

Vamos cobrir alguns dos mais comuns nesta aula.

## Códigos de Sucesso

Vamos começar com os **códigos de sucesso**. Esses códigos começam com o número 2.

O código **200 OK** é o código de resposta HTTP mais comum. Significa que a requisição foi bem-sucedida e o servidor retornou as informações solicitadas pelo cliente.

O código **201 Created** é retornado pelo servidor quando o cliente solicita a criação de um recurso e o processo é concluído com sucesso. Por exemplo, se uma nova conta de usuário for criada com sucesso, o servidor retorna esse código de resposta para o cliente.

O código **204 No Content** é retornado quando a requisição foi processada com sucesso mas o servidor não tem conteúdo para retornar. Isso pode ser usado para processos que deletam ou atualizam um recurso em um banco de dados mas não têm dados para retornar.

## **Códigos de Redirecionamento**

Agora vamos falar sobre **códigos de redirecionamento**.

Esses códigos são retornados pelo servidor quando a solicitação de recurso pelo cliente foi movida para um local diferente ou quando o cliente precisa tomar etapas adicionais para completar a solicitação.

Esses códigos começam com o número 3.

O código **301 Moved Permanently** é retornado quando o recurso solicitado pelo cliente foi movido permanentemente para outro local. Nesse caso, o servidor deve incluir uma URL para o novo local na resposta e, uma vez que o cliente receba a resposta, o usuário deve ser redirecionado para essa URL.

O código **302 Found** indica que o recurso solicitado pelo cliente foi temporariamente movido para outro local. O servidor deve fornecer a nova URL na resposta e o cliente deve redirecionar o usuário para essa URL. No entanto, a URL original pode estar disponível no futuro, já que a mudança é temporária. Essa é a principal diferença entre os códigos de resposta 301 e 302.

O código **304 Not Modified** é retornado pelo servidor para informar ao cliente que o recurso solicitado não foi modificado desde a última vez que foi requisitado. O servidor envia esse código para evitar o envio de dados que já foram enviados se o conteúdo não mudou. É uma técnica de otimização de desempenho.

## Códigos de Erro do Cliente

Agora vamos começar a explorar os **códigos de erro**.

Erros podem ser causados pelo cliente e pelo servidor. Vamos falar primeiro sobre os códigos de erro do cliente.

Códigos de erro do cliente são retornados quando a solicitação não pôde ser concluída devido a um problema gerado pelo cliente.

Esses códigos de erro começam com o número 4.

O código **400 Bad Request** é retornado quando o servidor não conseguiu entender a requisição enviada pelo cliente. Isso pode ocorrer quando o cliente envia uma requisição com sintaxe incorreta, com parâmetros inválidos ou dados inválidos.

O código **401 Unauthorized** está relacionado à autenticação. Ele indica que o cliente precisa fornecer certas credenciais de autenticação para acessar o recurso que está sendo solicitado. Essas credenciais podem incluir um nome de usuário e uma senha.

O código **403 Forbidden** é retornado quando o cliente não está autorizado a acessar o recurso solicitado. A diferença entre 401 e 403 é que no 403, o cliente não tem as permissões necessárias para o recurso específico que está solicitando, mesmo que possua credenciais de autenticação válidas.

O código **404 Not Found** é retornado quando o recurso não pôde ser encontrado no servidor. Talvez ele tenha sido deletado ou haja um erro de digitação na URL que impede o servidor de encontrar o recurso. O servidor também pode estar enfrentando um problema temporário que resulta neste código de erro.

## Códigos de Erro do Servidor

Em seguida, temos os **códigos de erro do servidor**. Esses códigos são retornados pelo servidor quando a solicitação não pôde ser processada devido a um problema no lado do servidor.

O código de erro **500 Internal Server Error** indica que houve um erro inesperado no servidor. Esse erro pode ser causado por uma ampla variedade de problemas, desde problemas de configuração até problemas temporários.

O código de erro **502 Bad Gateway** é retornado quando o servidor estava tentando se comunicar com outro servidor mas recebeu uma resposta inválida desse servidor. Há uma ampla variedade de causas para esse tipo de erro.

O código de erro **503 Service Unavailable** indica que o servidor não poderá processar a solicitação temporariamente, então o cliente deve tentar novamente mais tarde. Causas comuns para esse erro incluem tarefas de manutenção e alto volume de solicitações que o servidor atualmente não consegue processar.

Os códigos de resposta HTTP são essenciais para entender a comunicação entre clientes e servidores web. Ao entender esses códigos, você pode solucionar problemas e depurar suas aplicações web.
