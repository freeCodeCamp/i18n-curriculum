# --interactive--

تُستخدم الخاصية `aria-controls` لإنشاء علاقة برمجية بين عنصر يتحكم في وجود عنصر آخر على الصفحة، والعنصر الذي يتحكم به. تساعد هذه العلاقة مستخدمي قارئ الشاشة على فهم كيفية عمل التحكم بشكل أفضل. من الاستخدامات الشائعة مجموعة من علامات التبويب التي تتحكم في مستوى الظهور لأقسام مختلفة من المحتوى، أو زر يبدل مستوى الظهور لقائمة.

لنلقِ نظرة على مثال لنرى كيف يعمل هذا. في هذا المثال لواجهة بعلامات تبويب، لدينا عنصر `div` يحتوي على مجموعة من ثلاثة أزرار:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<div role="tablist">
  <button role="tab" id="tab1" aria-controls="section1" aria-selected="true">
    Tab 1
  </button>
  <button role="tab" id="tab2" aria-controls="section2" aria-selected="false">
    Tab 2
  </button>
  <button role="tab" id="tab3" aria-controls="section3" aria-selected="false">
    Tab 3
  </button>
</div>
```

```css
[role="tablist"] {
  display: flex;
  border-bottom: 2px solid #ccc;
  margin-bottom: 1em;
  gap: 4px;
}

[role="tab"] {
  background: #f5f5f5;

}

[role="tab"]:hover,
[role="tab"]:focus {
  background: #e8f0fe;
  outline: none;
}

[role="tab"][aria-selected="true"] {
  background: #fff;
  border-color: #ccc;
  border-bottom: 2px solid #fff;
```

يستخدم العنصر `div` الخاصية `role="tablist"` للدلالة على أنه يعمل كحاوية لمجموعة من علامات التبويب.

يمثل كل زر علامة تبويب وله خاصية `role` مضبوطة على `tab`. في معظم واجهات علامات التبويب، يكون أول قسم علامة تبويب ظاهرًا افتراضيًا، لذا يحتوي زر علامة التبويب الأول على الخاصية `aria-selected` مضبوطة على `true` للدلالة على أن القسم المرتبط به ظاهر حاليًا. تُستخدم الخاصية `aria-controls` لربط كل زر بالقسم الذي يتحكم به.

إليك الأقسام الثلاثة من المحتوى التي تتوافق مع علامات التبويب:

```js
const tabs = document.querySelectorAll('[role="tab"]');
const tabList = document.querySelector('[role="tablist"]');

tabs.forEach(tab => {
  tab.addEventListener('click', () => {
    activateTab(tab);
  });

  tab.addEventListener('keydown', (e) => {
    const key = e.key;
    const currentIndex = Array.from(tabs).indexOf(tab);
    let newIndex = null;

    if (key === 'ArrowRight') {
      newIndex = (currentIndex + 1) % tabs.length;
    } else if (key === 'ArrowLeft') {
      newIndex = (currentIndex - 1 + tabs.length) % tabs.length;
    } else if (key === 'Home') {
      newIndex = 0;
    } else if (key === 'End') {
      newIndex = tabs.length - 1;
    }

    if (newIndex !== null) {
      tabs[newIndex].focus();
      activateTab(tabs[newIndex]);
    }
  });
});

function activateTab(tab) {
  const tabPanels = document.querySelectorAll('[role="tabpanel"]');

  tabs.forEach(t => {
    t.setAttribute('aria-selected', 'false');
    t.setAttribute('tabindex', '-1');
  });

  tabPanels.forEach(panel => panel.hidden = true);

  tab.setAttribute('aria-selected', 'true');
  tab.removeAttribute('tabindex');

  const panelId = tab.getAttribute('aria-controls');
  const panel = document.getElementById(panelId);
  panel.hidden = false;
  tab.focus();
}

```

:::

كل قسم من المحتوى يحتوي على خاصية `role` مضبوطة على `tabpanel` وخاصية `aria-labelledby` تشير إلى التبويب المقابل لمنح اللوحة اسمًا يمكن الوصول إليه. تُستخدم الخاصية `hidden` لإخفاء أقسام المحتوى التي لم تُحدد حاليًا. يتطابق معرف كل قسم مع قيمة الخاصية `aria-controls` على الزر المقابل. معرف `section1` يتطابق مع الخاصية `aria-controls` على الزر الأول، و`section2` يتطابق مع الخاصية `aria-controls` على الزر الثاني، و`section3` يتطابق مع الخاصية `aria-controls` على الزر الثالث. هكذا يُنشأ الارتباط بين التبويبات وأقسام المحتوى الخاصة بها.

للتبديل بين الأقسام المختلفة، ستحتاج إلى استخدام JavaScript لتحديث الخاصية `hidden` على القسم والخاصية `aria-selected` على الزر بناءً على القسم الظاهر حاليًا. يمكن أن يكون قسم واحد فقط ظاهرًا في كل مرة، ويجب ألا يحتوي على الخاصية `hidden` ويجب ضبط `aria-selected` على `true` على زره. يجب أن تحتوي الأقسام المخفية المتبقية على الخاصية `hidden` ويجب ضبط `aria-selected` على `false` على أزرارها.

تُعد علامات التبويب حالة استخدام شائعة للخاصية `aria-controls`، لكنها يمكن أن تُستخدم في سيناريوهات أخرى حيث يتحكم عنصر في مستوى الظهور أو سلوك عنصر آخر. من الأمثلة الأخرى الأكورديونات، قوائم السحب، والنوافذ المنبثقة.

في المرة القادمة التي تصمم فيها واجهة مستخدم تتضمن التحكم في مستوى الظهور للعناصر، فكر في استخدام الخاصية `aria-controls` لإنشاء العلاقة بين العنصر المتحكم والعنصر المتحكم به.
