# --interactive--

在程式設計中，了解字元如何被表述為數字是基本的。這就是 `ASCII` 派上用場的地方。`ASCII`，即 American Standard Code for Information Interchange 的縮寫，是一種用於電腦中表述文字的字元編碼標準。它為每個字元指定一個數值，該數值被機器普遍識別。

在本課程中，我們將探討什麼是 ASCII、它如何運作，以及像 `charCodeAt()` 和 `fromCharCode()` 這些 JavaScript 方法如何與字元編碼相關。雖然 JavaScript 字串內部使用 Unicode（UTF-16），但 ASCII 值與前 128 個 Unicode 字元相符，這就是為什麼基於 ASCII 的範例能在 JavaScript 中運作的原因。

ASCII 是一種將字母、數字和符號等字元編碼成數值的系統。每個字元都對應到特定的數字。

例如，大寫字母 `A` 在 ASCII 中由數字 `65` 表現，而小寫字母 `a` 則由 `97` 表現。這種編碼允許電腦儲存和操作文字。

ASCII 標準涵蓋 128 個字元，包括：

- 大寫和小寫英文字母（A-Z、a-z）。
- 數字（0-9）。
- 常見的標點符號和符號（!、@、# 等等）。
- 控制字元（例如換行和製表符）。

在 JavaScript 中，你可以使用 `charCodeAt()` 方法來存取字元的數字程式碼。此方法會傳回指定索引處字元的 UTF-16 程式碼單元。對於前 128 個字元，此值與 ASCII 程式碼相符。

讓我們來看一個範例：

:::interactive_editor

```js
let letter = "A";
console.log(letter.charCodeAt(0));  // 65
```

:::

在此範例中，`A` 是字串的第一個字元，而呼叫 `charCodeAt(0)` 傳回它的數值程式碼（對於基本拉丁字元而言，該數值與其 ASCII 值相符），即 `65`。

你也可以將此方法用於其他字元，以尋找它們的數值程式碼。

:::interactive_editor

```js
let symbol = "!";
console.log(symbol.charCodeAt(0));  // 33
```

:::

這裡，驚嘆號 `!` 的數值程式碼被傳回為 `33`（與其 ASCII 值相符）。

當 `charCodeAt()` 幫助你取得字元的數值代碼時，`fromCharCode()` 方法則允許你做相反的事：將 UTF-16 代碼單元（對基本字元而言與 ASCII 相符）轉換成對應的字元。

讓我們來看看這個實際操作：

:::interactive_editor

```js
let char = String.fromCharCode(65);
console.log(char);  //  A
```

:::

在此範例中，`fromCharCode(65)` 將數字程式碼 `65`（對應 `A` 的 ASCII 值）轉換回字元 `A`。

另一個範例是將數字 `97` 轉換成對應的小寫字母：

:::interactive_editor

```js
let char = String.fromCharCode(97);
console.log(char);  // a
```

:::

當你需要根據字元的數值程式碼來操作或比較字元時，這些方法特別有用。

例如，你可能會使用 `charCodeAt()` 透過比較其 ASCII 值來檢查字元是大寫、小寫還是數字。

另一方面，`fromCharCode()` 可用來從它們的 ASCII 碼動態產生字元。
