# --description--

Раніше ви ознайомилися з функцією підпрограмного забезпечення `express.static()`. Тепер настав час детальніше розглянути, що таке підпрограмне забезпечення. Функції підпрограмного забезпечення — це функції, які приймають 3 аргументи: об’єкт запиту, об’єкт відповіді та наступну функцію в циклі запит-відповідь застосунку. Ці функції виконують код, який може мати побічні ефекти на застосунок, і зазвичай додають інформацію до об’єктів запиту або відповіді. Вони також можуть завершити цикл, відправивши відповідь, коли виконується певна умова. Якщо вони не відправляють відповідь після завершення, вони запускають виконання наступної функції в стеку. Це викликає виклик третього аргументу, `next()`.

Подивіться на наступний приклад:

```js
function(req, res, next) {
  console.log("I'm a middleware...");
  next();
}
```

Припустимо, ви змонтували цю функцію на маршруті. Коли запит співпадає з маршрутом, вона виводить рядок “I’m a middleware…”, а потім виконує наступну функцію в стеку. У цьому завданні ви створите підпрограмне забезпечення на кореневому рівні. Як ви бачили у завданні 4, щоб змонтувати функцію підпрограмного забезпечення на кореневому рівні, можна використати метод `app.use(<mware-function>)`. У цьому випадку функція буде виконуватися для всіх запитів, але ви також можете встановити більш конкретні умови. Наприклад, якщо ви хочете, щоб функція виконувалася лише для POST-запитів, можна використати `app.post(<mware-function>)`. Аналогічні методи існують для всіх HTTP-дієслів (GET, DELETE, PUT тощо).

# --instructions--

Створіть простий логер. Для кожного запиту він має виводити в консоль рядок у такому форматі: `method path - ip`. Приклад виглядатиме так: `GET /json - ::ffff:127.0.0.1`. Зверніть увагу, що між `method` і `path` є пробіл, а дефіс, який розділяє `path` і `ip`, оточений пробілами з обох боків. Ви можете отримати метод запиту (HTTP-дієслово), відносний шлях маршруту та IP виклику з об’єкта запиту, використовуючи `req.method`, `req.path` і `req.ip`. Не забудьте викликати `next()` після завершення, інакше ваш сервер зависне назавжди. Обов’язково відкрийте вкладку «Логи» і подивіться, що відбувається, коли надходить запит.

**Примітка:** Express виконує функції в тому порядку, в якому вони з’являються в коді. Це стосується і підпрограмного забезпечення. Якщо ви хочете, щоб воно працювало для всіх маршрутів, його слід змонтувати перед ними.
