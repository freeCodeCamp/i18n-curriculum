# --description--

Finora hai visto come JSX differisca da HTML in un aspetto chiave, cioè l’uso di `className` rispetto a `class` per definire le classi HTML.

Un altro modo importante in cui JSX differisce da HTML riguarda il concetto di tag a chiusura automatica.

In HTML, quasi tutti i tag hanno sia un tag di apertura sia un tag di chiusura: `<div></div>`; il tag di chiusura ha sempre una barra obliqua prima del nome del tag che stai chiudendo. Tuttavia, ci sono casi speciali in HTML chiamati <dfn>elementi void</dfn>, cioè elementi che non richiedono sia il tag di apertura sia quello di chiusura prima che possa iniziare un altro elemento.

Per esempio, il tag di interruzione di riga può essere scritto come `<br>` oppure come `<br />`, ma non dovrebbe mai essere scritto come `<br></br>`, dato che non contiene alcun contenuto.

In JSX, le regole sono un po’ diverse. Qualsiasi elemento JSX può essere scritto con un tag a chiusura automatica, e ogni elemento deve essere chiuso. Il tag di interruzione di riga, per esempio, deve sempre essere scritto come `<br />` per essere un JSX valido che può essere transpiled. Un `<div>`, invece, può essere scritto come `<div />` oppure `<div></div>`. La differenza è che nella prima versione di sintassi non c’è modo di includere qualcosa dentro il `<div />`. Vedrai nelle sfide successive che questa sintassi è utile quando si renderizzano componenti React.

# --instructions--

Correggi gli errori nell’editor di codice in modo che sia JSX valido e si possa transpile correttamente. Assicurati di non modificare alcun contenuto: devi solo chiudere i tag dove serve.

# --hints--

La costante `JSX` dovrebbe restituire un elemento `div`.

```js
assert.strictEqual(JSX.type, 'div');
```

Il `div` dovrebbe contenere un tag `br`.

```js
assert(Enzyme.shallow(JSX).find('br').length === 1);
```

Il `div` dovrebbe contenere un tag `hr`.

```js
assert(Enzyme.shallow(JSX).find('hr').length === 1);
```
