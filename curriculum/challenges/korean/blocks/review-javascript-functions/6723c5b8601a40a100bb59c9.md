## JavaScript 함수

- 함수는 특정 작업을 수행하는 재사용 가능한 코드 블록입니다.
- 함수는 `function` 키워드 다음에 이름, 매개변수 목록, 작업을 수행하는 코드 블록을 작성하여 정의할 수 있습니다.

:::interactive_editor

```js
function addNumbers(x, y, z) {
  return x + y + z;
}

console.log(addNumbers(5, 3, 8)); // Output: 16
```

:::

- 인수는 함수가 호출될 때 전달되는 값입니다.
- 함수 호출은 함수 이름 뒤에 소괄호를 붙여 프로그램에서 함수를 실행하는 과정이며, 소괄호 안에 인수를 포함할 수도 있습니다.
- 함수가 실행을 마치면 항상 값을 반환합니다.
- 기본적으로 함수의 반환값은 `undefined`입니다.
- `return` 키워드는 함수에서 반환할 값을 지정하고 함수 실행을 종료하는 데 사용됩니다.
- 기본 매개변수는 함수가 호출될 때 인수가 제공되지 않으면 미리 정해진 값을 사용하도록 하여, 함수를 더 유연하게 만들고 특정 인수가 생략될 때 발생할 수 있는 오류를 방지합니다.

:::interactive_editor

```js
const calculateTotal = (amount, taxRate = 0.05) => {
  return amount + (amount * taxRate);
};

console.log(calculateTotal(100)); // Output: 105
```

:::

- 익명 함수는 이름이 없는 함수로, 변수에 할당할 수 있습니다. 변수에 할당하면 해당 변수가 접근 가능한 어디서든 재사용할 수 있습니다.

:::interactive_editor

```js
const multiplyNumbers = function(firstNumber, secondNumber) {
  return firstNumber * secondNumber;
};

console.log(multiplyNumbers(4, 5)); // Output: 20
```

:::

## 화살표 함수

- 화살표 함수는 JavaScript에서 함수를 더 간결하게 작성하는 방법입니다.

:::interactive_editor

```js
const calculateArea = (length, width) => {
  const area = length * width;
  return `The area of the rectangle is ${area} square units.`;
};

console.log(calculateArea(5, 10)); // Output: "The area of the rectangle is 50 square units."
```

:::

- 화살표 함수를 정의할 때 `function` 키워드는 필요하지 않습니다.
- 매개변수가 하나일 경우 매개변수 목록을 감싸는 소괄호를 생략할 수 있습니다.

:::interactive_editor

```js
const cube = x => {
  return x * x * x;
};

console.log(cube(3)); // Output: 27
```

:::

- 함수 본문이 단일 표현식으로 이루어졌다면 중괄호와 `return` 키워드를 생략할 수 있습니다.

:::interactive_editor

```js
const square = number => number * number;

console.log(square(5)); // Output: 25
```

:::

## 프로그래밍에서의 범위

- **전역 범위**: JavaScript에서 가장 바깥쪽 범위입니다. 전역 범위에 선언된 변수는 코드 어디서든 접근할 수 있으며 전역 변수라고 합니다.
- **지역 범위**: 함수 내에 선언된 변수를 가리킵니다. 이 변수들은 선언된 함수 내에서만 접근 가능하며 지역 변수라고 합니다.
- **블록 범위**: 블록은 `{}`로 감싼 일련의 문장 집합이며, `if` 문장이나 반복문에서 사용됩니다.
- `let`와 `const`를 사용한 블록 범위는 변수 접근을 더 세밀하게 제어하여 오류를 방지하고 코드를 더 예측 가능하게 만듭니다.
