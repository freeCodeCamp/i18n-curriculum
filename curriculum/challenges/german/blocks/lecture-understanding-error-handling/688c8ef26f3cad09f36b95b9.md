# --description--

Debugging ist eine unverzichtbare Fähigkeit für jede*n Python-Entwickler*in. Das Verständnis grundlegender Techniken hilft Ihnen dabei, Probleme effizient zu erkennen und zu beheben.

Debugging ist der Prozess, Fehler oder Bugs in Ihrem Code zu identifizieren und zu lösen. Dabei untersuchen Sie den Code, verstehen den Ablauf und nutzen Werkzeuge, um die Ursache von Problemen zu finden.

In dieser Lektion gehen wir auf gängige Debugging-Techniken ein, die Sie in Ihrem nächsten Python-Projekt verwenden können.

## Verwendung der print-Funktion und f-Strings

Zunächst kann die Verwendung der `print()`-Funktion und von f-Strings an verschiedenen Stellen im Code Ihnen helfen, den Ablauf und den Zustand von Variablen zu verstehen. Zum Beispiel:

```py
def add(a, b):
    result = a + b
    print(f'Adding {a} and {b} gives {result}')
    return result
```

Indem Sie die Werte von `a`, `b` und `result` ausgeben, können Sie überprüfen, ob die Funktion wie erwartet arbeitet.

## Interaktives Debuggen mit dem `pdb`-Modul

Als Nächstes können Sie das eingebaute `pdb`-Modul von Python für interaktives Debugging nutzen:

```py
import pdb

def divide(a, b):
    pdb.set_trace()
    return a / b

print(divide(10, 2))
```

Indem Sie mit der `set_trace()`-Funktion eine Spur setzen, können Sie den Code schrittweise durchgehen, Variablen inspizieren und das Verhalten des Programms verstehen.

Wenn Sie den obigen Code ausführen, sehen Sie eine Ausgabe, die den Speicherort der Datei zeigt, die Sie ausführen, die Zeile, in der Sie die `set_trace()`-Funktion aufgerufen haben, und den Code direkt danach sowie eine interaktive `pdb`-Eingabeaufforderung:

```py
> /Users/fcc/Desktop/debugging.py(5)divide()
-> return a / b
(Pdb)
```

Wenn Sie in der Eingabeaufforderung help eingeben, sehen Sie eine Liste von Befehlen, die Sie verwenden können:

```py
(Pdb) help

Documented commands (type help <topic>):
========================================
EOF    c          d        h         list      q        rv       undisplay
a      cl         debug    help      ll        quit     s        unt      
alias  clear      disable  ignore    longlist  r        source   until    
args   commands   display  interact  n         restart  step     up       
b      condition  down     j         next      return   tbreak   w        
break  cont       enable   jump      p         retval   u        whatis   
bt     continue   exit     l         pp        run      unalias  where    

Miscellaneous help topics:
==========================
exec  pdb
```

Dann können Sie die Befehle nutzen, um Ihren Code zu debuggen.

Wenn Sie zum Beispiel den Typ der Elemente in Ihrem Code zu diesem Zeitpunkt ansehen möchten, können Sie den `whatis`-Befehl verwenden:

```py
(Pdb) whatis a
<class 'int'>
(Pdb) whatis divide
Function divide
```

Wie Sie sehen, ist zum Zeitpunkt des Aufrufs von `.set_trace()` der Typ des Parameters a ein Integer, und divide ist eine Funktion.

Um die Ausführung Ihres Codes fortzusetzen, können Sie den `continue`-Befehl oder einen seiner Aliase, `cont` oder `c`, verwenden:

```py
(Pdb) continue
5.0
```

## Debugging-Tools in IDEs

Viele integrierte Entwicklungsumgebungen (IDEs) bieten erweiterte Debugging-Tools wie Haltepunkte, schrittweise Ausführung und Variableninspektion.

## Verwendung des Debuggers in VS Code

Wenn Sie VS Code verwenden, können Sie Haltepunkte in Ihrem Code setzen und den Debugger starten, um die Ausführung an diesen Stellen anzuhalten. So debuggen Sie dieselbe `divide`-Funktion:

**Schritt 1: Richten Sie Ihren Code ein** Erstellen Sie eine Datei namens `main.py` mit folgendem Inhalt:

```py
def divide(a, b):
    result = a / b
    return result

print(divide(10, 2))
print(divide(15, 3))
```

**Schritt 2: Setzen Sie einen Haltepunkt**

1. Klicken Sie im Spaltenabstand (linker Außenabstand) neben Zeile 2 (`result = a / b`), um einen Haltepunkt zu setzen

2. Ein roter Punkt erscheint, der anzeigt, dass der Haltepunkt gesetzt ist

**Schritt 3: Starten Sie das Debuggen**

1. Drücken Sie `F5` oder gehen Sie zu Ausführen > Debuggen starten

2. Wählen Sie „Python-Datei“ aus, wenn Sie dazu aufgefordert werden

3. Der Debugger hält die Ausführung an Ihrem Haltepunkt an

**Schritt 4: Variablen inspizieren**

- Fahren Sie mit der Maus über Variablen, um deren aktuellen Wert zu sehen

- Verwenden Sie das Variablenfenster links, um alle lokalen Variablen anzuzeigen

- Nutzen Sie die Debug-Konsole unten, um Ausdrücke auszuwerten

**Schritt 5: Schrittweise durch den Code gehen**

Verwenden Sie die Debug-Symbolleiste, um:

- **Fortsetzen (F5)**: Die Ausführung bis zum nächsten Haltepunkt fortsetzen

- **Übergehen (F10)**: Die aktuelle Zeile ausführen und zur nächsten wechseln

- **Eintreten (F11)**: In Funktionsaufrufe hineingehen

- **Heraustreten (Shift+F11)**: Die aktuelle Funktion verlassen

Die Debugging-Tools in IDEs bieten eine visuelle Oberfläche, um den Zustand Ihres Programms zu untersuchen, was das Erkennen und Beheben von Problemen im Vergleich zu reinen print-Anweisungen erleichtert.

Wenn Sie diese grundlegenden Debugging-Techniken – Verwendung von `print()`-Anweisungen, das pdb-Modul und IDE-Tools – beherrschen, können Sie Probleme in Ihrem Python-Code effektiv erkennen und lösen. Jede Technik hat ihren Platz: `print()`-Anweisungen für schnelle Überprüfungen, pdb für interaktive Erkundung und IDE-Debugger für visuelle Inspektion.
