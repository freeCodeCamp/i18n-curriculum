# --description--

讓我們透過 Chrome DevTools 實際操作測量 Interaction to Next Paint，或稱 INP。

首先，按下「F12」或右鍵點擊並選擇「檢查」來開啟 DevTools。

現在，假設你的網頁上有一個搜尋欄。

想像一個使用者輸入查詢，但建議下拉選單在 600 毫秒後才出現。Chrome DevTools 會標記這個高 `INP` 值。

你可以透過延緩執行大量的 JavaScript、減少長時間任務，以及改善 `event` 處理來最佳化。這樣 INP 就會降到 150 毫秒，使互動感覺瞬間完成。

很酷，對吧？

現在，讓我們來看看一種實用的測量技術，幫助你識別互動延遲，從而創建更快速且更流暢的使用者體驗！

首先，我們會打開 Chrome DevTools。我們會再次透過開啟 Google Chrome，前往你想分析的網頁，然後按下「F12」或右鍵點擊頁面並點選「檢查」來完成。現在，讓我們前往「Performance」標籤－裡面有你所需要的一切。

現在我們準備好識別 Interaction to Next Paint，或稱 INP。 我們將尋找記錄中最長的互動延遲。 一般來說，良好的 INP 低於 200ms，而不良的 INP 則高於 500ms。 就是這樣！
