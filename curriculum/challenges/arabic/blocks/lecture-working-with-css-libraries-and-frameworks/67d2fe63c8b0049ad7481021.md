# --description--

معالج مسبق لـ CSS هو أداة توسع CSS القياسي بميزات قوية مثل المتغيرات، والخليط، والتسكين، ووراثة المحددات.

بعض هذه الميزات، مثل المتغيرات والتسكين، مدعومة الآن أو تحصل على دعم أكبر في CSS الأصلي، ومن المرجح أن يستمر هذا الاتجاه. مع ذلك، الإصدارات السابقة من CSS الأصلي لم تدعم هذه الميزات.

لهذا السبب أصبح معالجو CSS المسبقين مستخدمين على نطاق واسع في البداية. فهم يأخذون قواعد وأنماط CSS التي تكتبها، مع صياغتها الموسعة، ويحولونها إلى ملف CSS أصلي يمكن للمتصفحات فهمه.

باستخدام معالج مسبق لـ CSS، يمكنك تنظيم شفرة CSS بطريقة أكثر قابلية لإعادة الاستخدام ومنطقية. كما ستحصل على ميزات قوية، مثل الخليط، التي لا يدعمها CSS مباشرة.

بعض أشهر معالجات CSS المسبقين هي Sass و Less و Stylus.

دعنا نتحدث عن Sass.

Sass تعني "أوراق الأنماط الرائعة من حيث الصياغة". هو متوافق مع جميع إصدارات CSS ويُصان بواسطة مجتمع كبير من المطورين.

يدعم Sass ميزات مثل:

- المتغيرات، لتخزين القيم وإعادة استخدامها في جميع أنحاء ملف الأنماط.

- قواعد CSS المسكنة، لإنشاء تسلسل هرمي بصري في ملفك.

- الوحدات، لتقسيم أنماطك إلى عدة ملفات أنماط.

- الخليط، لإعادة استخدام إعلانات CSS في جميع أنحاء موقعك.

- الوراثة، ليشارك عدة محددات CSS الخصائص.

- والعوامل، للعمليات الرياضية الأساسية.

يدعم Sass أيضًا صياغتين.

صياغة SCSS تعني Sassy CSS، وهي مجموعة فرعية من CSS. هذا يعني أن SCSS توسع الصياغة الأساسية لـ CSS.

SCSS هي الصياغة الأكثر شيوعًا التي ستستخدمها وتجدها عند العمل مع Sass. تتطلب استخدام الأقواس المتعرجة (`{` و `}`) حول خصائص CSS والفواصل المنقوطة (`;`) في نهاية الإعلانات، تمامًا مثل CSS الأصلي.

هذه الملفات تحمل امتداد `.scss`. إليك مثالًا:

```scss
$primary-color: #3498eb;

header {
  background-color: $primary-color;
}
```

ما تراه هنا، في الأعلى، هو متغير معرف في SCSS. يُستخدم هذا المتغير في قاعدة CSS أدناه.

دعنا نقارن SCSS مع صياغة أقل استخدامًا، وهي الصياغة المسكنة. تُعرف أيضًا باسم "صياغة Sass" لأنها الصياغة الأصلية لـ Sass.

تعتمد هذه الصياغة على التسكين لتعريف القواعد. إليك مثالًا:

```scss
$primary-color: #3498eb

header
  background-color: $primary-color
```

لاحظ أنه لا توجد أقواس متعرجة حول قاعدة CSS، ولا فواصل منقوطة في نهاية الإعلانات. في الأعلى، يمكنك أيضًا رؤية متغير. لاحظ أن اسمه يبدأ بعلامة الدولار (`$`) في هذه الصياغة، وهو مختلف عن متغيرات CSS الأصلية، لكنه يعمل بطريقة مشابهة جدًا.

يدعم Sass أيضًا ميزة قوية تسمى الخليط.

تسمح الخليطات بتجميع عدة خصائص CSS وقيمها تحت اسم واحد وإعادة استخدام ذلك المقطع من شفرة CSS في جميع أنحاء ملف الأنماط.

هذا يجعل شفرة CSS أقل تكرارًا وأسهل في الصيانة، لأنه إذا غيرت شيئًا في ذلك المقطع، يُطبق التغيير في كل مكان تستخدم فيه الخليط.

وبما أن الخليطات يمكن أن تحمل أسماء مخصصة ووصفية، فهي تساعد على فهم وظيفة كل مقطع من شفرة CSS، مما يجعل ملفات الأنماط أسهل في الفهم.

كما تعزز الخليطات الاتساق في ملفات الأنماط من خلال تطبيق أنماط موحدة.

هذا مثال على خليط في صياغة SCSS:

```scss
@mixin center-flex {
  display: flex;
  justify-content: center;
  align-items: center;
}
```

لتعريف خليط، تبدأ بكتابة قاعدة عند `@mixin`، تليها اسم الخليط. في هذه الحالة، الخليط يسمى `center-flex`. يحتوي على ثلاث خصائص CSS لمركزة العناصر باستخدام Flexbox.

ثم، بعد تعريف الخليط، تستخدم قاعدة عند `@include` لإدراج تلك الخصائص في قاعدة CSS. فقط اكتب `@include` متبوعًا باسم الخليط. في هذه الحالة، الاسم هو `center-flex`:

```scss
section {
  @include center-flex;
  height: 500px;
  background-color: #3289a8;
}
```

هذا هو الشفرة الكاملة مع الخليط وقاعدة CSS:

```scss
@mixin center-flex {
  display: flex;
  justify-content: center;
  align-items: center;
}

section {
  @include center-flex;
  height: 500px;
  background-color: #3289a8;
}
```

يمكنك إدراج الخليط في عدد غير محدود من قواعد CSS حسب الحاجة. إذا احتجت إلى إجراء تغييرات، يكفي تعديل الخليط نفسه فقط، وسيُطبق التغيير في كل مكان تلقائيًا. هذا المثال بصياغة SCSS. لاحظ وجود الأقواس المتعرجة والفواصل المنقوطة.

إليك المعادل بصياغة التسكين، المعروفة أيضًا بصياغة Sass، بدون الأقواس المتعرجة أو الفواصل المنقوطة:

```scss
@mixin center-flex
  display: flex
  justify-content: center
  align-items: center

section
  @include center-flex
  height: 500px
  background-color: #3289a8
```

وهذه هي شفرة CSS المجمعة، التي سيُفسرها المتصفح فعليًا بعد تحويل الملف إلى CSS أصلي:

```css
section {
  display: flex;
  justify-content: center;
  align-items: center;
  height: 500px;
  background-color: #3289a8;
}
```

لاحظ أنها تتضمن الخصائص الثلاثة للخليط في الأعلى: `display` و `justify-content` و `align-items`.

هذا مجرد مثال بسيط على ما يمكن لمعالجات CSS المسبقين القيام به. لديهم العديد من الميزات القوية التي تستحق التعلم والتطبيق في سير عملك اليومي.

مع ذلك، لديهم بعض العيوب المحتملة التي يجب أن تكون على علم بها.

أولًا، يجب على معالجات CSS المسبقين تحويل قواعد CSS إلى CSS قياسي. رغم أن العبء المحتمل لهذه العملية غالبًا ما يكون ضئيلًا من حيث الوقت والموارد، إلا أنه أمر يجب أخذه في الاعتبار في المشاريع الواقعية.

بعد ذلك، يمكن أن تخلق تحديات في تصحيح الأخطاء، لأن المتصفحات تستخدم CSS المجمعة مباشرة. معرفة مصدر نمط إشكالي من الصياغة الموسعة قد يتطلب خطوات إضافية مقارنة بـ CSS القياسي.

مع ذلك، عادةً ما تفوق مزايا معالجات CSS المسبقين عيوبها، خاصة في المشاريع المعقدة. فهي مفيدة جدًا لكتابة CSS أنظف، قابلة لإعادة الاستخدام، أقل تكرارًا، وقابلة للتوسع.
