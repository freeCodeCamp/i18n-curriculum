# --interactive--

Katika funzo lililopita, ulijifunza jinsi ya kutupa exceptions katika programu zako. Katika funzo hili, tutaangalia jinsi ya kushughulikia makosa haya kwa hila ndani ya kipengele cha `try…catch…finally`.

Kipengele cha `try` hutumika kufunika msimbo ambao unaweza kutupa kosa. Hufanya kama sehemu salama ya kujaribu kitu ambacho kinaweza kushindikana.

Kipengele cha `catch` hukamata na kushughulikia makosa yanayotokea katika kipengele cha `try`. Unaweza kutumia kitu cha `Error` ndani ya `catch` kuchunguza kilichosababisha kosa.

Kipengele cha `finally` kinaendeshwa baada ya vipengele vya `try` na `catch`, bila kujali kama kosa lilitokea au la. Mara nyingi hutumika kwa kazi za kusafisha, kama kufunga mafaili au kuachilia rasilimali.

Hapa kuna mfano wa kutumia kipengele cha `try…catch`:

:::interactive_editor

```js
function processInput(input) {
  if (typeof input !== "string") {
    throw new TypeError("Input must be a string.");
  }

  return input.toUpperCase();
}

try {
  console.log("Starting to process input...");
  const result = processInput(9);
  console.log("Processed result:", result);
} catch (error) {
  console.error("Error occurred:", error.message);
} 
```

:::

Katika mfano huu, tuna kitendakazi kinachoitwa `processInput` ambacho kwanza kinakagua kama `input` si aina ya `string`. Ikiwa hivyo ndicho kilicho, basi tunatupa kosa. Vinginevyo, tunarudisha matokeo ya kutumia njia ya `toUpperCase` kwenye `input`.

Tunaita kitendakazi ndani ya kipengele cha `try`. Kwa kuwa wito wa kitendakazi unatupa kosa, basi litakamatwa ndani ya kipengele cha `catch` na ujumbe wa kosa utaonyeshwa kwenye konsoli.

Kosa lililotumwa ndani ya kipengele cha `catch` ni kitu cha `Error` ambacho kina taarifa kuhusu kosa hilo. Katika kesi hii, tunatumia sifa ya `message` ambayo inaonyesha taarifa inayoweza kueleweka na binadamu kwa mtumizi.

Tunatumia `console.error` kwa sababu imeundwa mahsusi kwa ajili ya kurekodi makosa. Katika vivinjari vingi vya kisasa, matokeo ya `console.error()` huonekana kwa rangi nyekundu kwenye konsoli.

Sentensi ya `finally` hufanyika bila kujali kama exception ilitupwa au la.

```js
try {
  // Code that might throw an error
} catch (error) {
  // Code to handle the error
} finally {
  // Code that runs regardless of whether an error occurred or not
}
```

Matumizi mazuri ya sentensi ya `finally` ni kama unafanya kazi na mafaili. Katika JavaScript, unaweza kufungua faili, tumia kipengele cha `try` kuandika data kwenye faili. Ikiwa kuna makosa yoyote, unaweza kutumia `catch` kukamata makosa hayo. Kisha tumia sentensi ya `finally` kufunga faili.
