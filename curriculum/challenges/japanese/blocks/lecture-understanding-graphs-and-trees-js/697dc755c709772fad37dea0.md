# --description--

**優先キュー**は、キューやスタックと似た動作をする抽象データ型（ADT）ですが、重要な違いがあります。

ご存じの通り、標準的なキューはFIFO（先入れ先出し）方式で動作し、キューに最初に追加された要素が最初に取り出されます。

スタックはLIFO（後入れ先出し）方式で動作し、スタックに最後に追加された要素が最初に取り出されます。

キューとスタックは要素の挿入順序のみを考慮します。

しかし、優先キューは要素の「優先度」を考慮します。優先度は、どの要素を次に取り出すかを決定するために使われます。

通常は、最も優先度の高い要素が最初に取り出されますが、実装によっては最も優先度の低い要素を最初に取り出す場合もあります。これはプログラムの要件によって異なります。

優先キューは、2地点間の最短経路の探索、オペレーティングシステムのタスクスケジューリング、交通シミュレーション、データ圧縮、ネットワーク管理などの実用的な用途で非常に役立ちます。

実際には、優先キューはヒープというデータ構造を使って実装されることが多いです。

**ヒープ**は、**ヒープ特性**と呼ばれる非常に特定の性質を持つ木構造のデータ構造です。この特性は、ヒープの種類に基づいて各ノードとその子ノードの関係を決定します。

ヒープには主に2種類あります：

* 最大ヒープ（Max-heap）
    
* 最小ヒープ（Min-heap）
    

**最大ヒープ**では、各ノードの値はその子ノードの値以上です。

この例では、8、7、5、2、1のノードを持つ木構造が示されています。ノード7はノード2とノード1より大きく、ヒープ特性を満たしています。これは他のすべてのノードにも当てはまります。

<img src="https://cdn.freecodecamp.org/curriculum/lecture-transcripts/how-do-priority-queues-and-heaps-work-1.png" alt="A max-heap tree structure showing nodes with values 8 at the root, 7 and 5 as children of 8, and 2 and 1 as children of 7, demonstrating that each parent node is greater than its children.">

一方、**最小ヒープ**では、各ノードの値はその子ノードの値以下です。

この例では、4、7、9、12、15のノードがあります。例えば、ノード7はノード12とノード15より小さく、ヒープ特性を満たしています。これも他のすべてのノードに当てはまります。

<img src="https://cdn.freecodecamp.org/curriculum/lecture-transcripts/how-do-priority-queues-and-heaps-work-2.png" alt="A min-heap tree structure showing nodes with values 4 at the root, 7 and 9 as children of 4, and 12 and 15 as children of 7, demonstrating that each parent node is less than its children.">

ヒープ特性は重要で、最大（または最小）要素が常にトップに位置することを保証し、取り出しを非常に簡単にします。

実際には、ヒープは親ノードと子ノードに効率的にアクセスできるように配列として実装されることが多いです。

配列を使うことで、ヒープが完全二分木の構造を保っている場合、要素のメモリ上の位置をインデックスに基づく単純な数学的操作で見つけられるため、これらの値や「ノード」へのアクセスのロジックが簡単になります。

JavaScriptには組み込みのヒープモジュールはありませんが、配列を使って最小ヒープを実装できます。

以下はJavaScriptでの基本的な最小ヒープの実装例です：

```js
class MinHeap {
  constructor(compare = (a, b) => a - b) {
    this.data = [];
    this.compare = compare;
  }

  peek() {
    return this.data[0];
  }

  push(value) {
    this.data.push(value);
    this.#bubbleUp(this.data.length - 1);
  }

  pop() {
    if (this.data.length === 0) return undefined;

    const top = this.data[0];
    const last = this.data.pop();

    if (this.data.length > 0) {
      this.data[0] = last;
      this.#bubbleDown(0);
    }

    return top;
  }

  pushPop(value) {
    if (this.data.length === 0) return value;

    if (this.compare(this.data[0], value) < 0) {
      const top = this.data[0];
      this.data[0] = value;
      this.#bubbleDown(0);
      return top;
    }

    return value;
  }

  heapify(arr) {
    this.data = arr.slice();
    for (let i = Math.floor(this.data.length / 2) - 1; i >= 0; i--) {
      this.#bubbleDown(i);
    }
  }

  #bubbleUp(i) {
    while (i > 0) {
      const p = Math.floor((i - 1) / 2);
      if (this.compare(this.data[i], this.data[p]) >= 0) break;
      [this.data[i], this.data[p]] = [this.data[p], this.data[i]];
      i = p;
    }
  }

  #bubbleDown(i) {
    const n = this.data.length;

    while (true) {
      let smallest = i;
      const l = 2 * i + 1;
      const r = 2 * i + 2;

      if (l < n && this.compare(this.data[l], this.data[smallest]) < 0) smallest = l;
      if (r < n && this.compare(this.data[r], this.data[smallest]) < 0) smallest = r;

      if (smallest === i) break;

      [this.data[i], this.data[smallest]] = [this.data[smallest], this.data[i]];
      i = smallest;
    }
  }
}
````

このヒープを使うには、まず空のヒープを作成します。これがヒープの基盤となるデータ構造です：

```js
const myHeap = new MinHeap();
```

ヒープに要素を追加するには`push()`を呼び出します。これにより、ヒープ特性を保つために要素が適切な位置に自動的に追加されます：

```js
myHeap.push(9);
```

最も優先度の高い要素（この場合は最小値）を取得するには`pop()`を呼び出します：

```js
myHeap.pop();
```

`pushPop()`はこれら両方の操作を1回の呼び出しで行います。

これは特にヒープが大きい場合に、別々に呼び出すより効率的で、再配置操作が1回だけで済みます：

```js
myHeap.pushPop(15);
```

すでに配列がある場合、それをヒープに変換するには`heapify()`を呼び出せます：

```js
myHeap.heapify([9, 2, 7, 1]);
```

しかし、今は値で要素をソートしていますよね？

もし「優先度」でソートしたい場合はどうしますか？

このような構造の配列を格納することで実現できます：`[priority, element]`。

JavaScriptでは配列は自動的に要素ごとに比較されないため、優先度を最初に比較する比較関数をヒープに渡します。

この場合、値が小さいほど優先度が高いことに注意してください。つまり、優先度1のアイテムは優先度3のアイテムより**高い**優先度を持ちます：

```js
const myHeap = new MinHeap((a, b) => a[0] - b[0]);

myHeap.push([3, "A"]);
myHeap.push([2, "B"]);
myHeap.push([1, "C"]);
```

同じ優先度の要素を挿入順に取り出したい場合は、アイテムの2番目の要素として一意のカウンターを含めて、優先度が同じ場合の判定に使うことができます。例えばこのように`[priority, counter, element]`。

優先度を最初に比較し、次にカウンターを比較する例です：

```js
let counter = 0;
const myHeap = new MinHeap((a, b) => (a[0] - b[0]) || (a[1] - b[1]));

myHeap.push([3, counter++, "A"]);
myHeap.push([2, counter++, "B"]);
myHeap.push([2, counter++, "C"]);
```

では、ヒープの効率について話しましょう。

ヒープから最小値または最大値を挿入・抽出する操作の平均および最悪ケースの時間計算量は対数時間で、`O(log n)`です。これは必要な交換回数がおおよそヒープの高さ（log(n)）に比例するためです。

「peek」操作（最小値または最大値を取り出さずに取得する）は平均および最悪ケースで定数時間、O(1)です。

未ソートリストからヒープを構築する「heapify」操作は平均および最悪ケースで線形時間、O(n)です。

同様に、任意の要素の検索や削除も平均および最悪ケースで線形時間、O(n)です。これはヒープ全体を走査する必要があるためです。

必要な空間はどのくらいでしょうか？

ヒープの空間計算量は線形、O(n)で、`n`は要素数です。要素と配列オブジェクト自体のわずかな追加オーバーヘッドのみが必要です。

優先キューとヒープはコンピュータサイエンスで非常に重要です。リストから最も重要な要素を素早く見つけて使うことができ、この効率性は地図上の最速ルート探索など、現実世界の重要なタスクを実行する多くのプログラムに不可欠です。
