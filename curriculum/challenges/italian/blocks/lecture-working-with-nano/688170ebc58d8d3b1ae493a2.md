# --description--

Quando lavori nel terminale, gli editor di testo standard non sono sempre disponibili, ma esistono programmi progettati per modificare i file direttamente dal terminale.

Gli ambienti Linux includono tipicamente un editor di testo. Vi, o Vim, sono i più comuni. Emacs e Nano potrebbero dover essere installati manualmente, ma sono facilmente reperibili per quasi tutte le distribuzioni.

La sfida con Vim è che ha una curva di apprendimento piuttosto ripida. Se non lo conosci già, potresti non sapere nemmeno come chiudere l’applicazione! Emacs è probabilmente un po’ più facile da usare, ma ha comunque un gran numero di scorciatoie da tastiera da imparare.

Nano è decisamente più semplice e intuitivo, soprattutto per chi è alle prime armi. Puoi iniziare a modificare un file con Nano molto più rapidamente, e l’applicazione offre anche un aiuto a schermo per le scorciatoie da tastiera di base.

Il compromesso, però, è la mancanza di estendibilità. A differenza di Vim o Emacs, che possono essere estesi con plugin per diventare veri ambienti di sviluppo, Nano è piuttosto essenziale.

Diamo un’occhiata a cosa può fare Nano. Per aprire un file, puoi digitare `nano <filename>` nel terminale.

Una volta aperto un file, vedrai la versione di Nano che stai usando e il nome del file aperto nella parte superiore dello schermo.

Sotto c’è il contenuto del file. Puoi usare i tasti freccia per muovere il cursore e inserire o eliminare testo con la tastiera. Il file scorrerà automaticamente se necessario.

Infine, in basso, vedrai una lista di scorciatoie da tastiera disponibili, che appare più o meno così:

```bash
^G Help   ^O Write Out    ^F Where Is   ^K Cut      ^T Execute  ^C Location     M-U Undo    M-A Set Mark    M-] To Bracket    M-B Previous
^X Exit   ^R Read File    ^\ Replace    ^U Paste    ^J Justify  ^/ Go To Line   M-E Redo    M-6 Copy        ^B Where Was      M-F Next
```

Le scorciatoie che iniziano con il simbolo dell’accento circonflesso (`^`) indicano che devi tenere premuto il tasto Ctrl e premere la lettera indicata. Le scorciatoie che iniziano con `M-` richiedono di tenere premuto il “tasto meta”, che nella maggior parte delle configurazioni corrisponde al tasto Alt, e premere la lettera.

Per modificare un file in Nano, devi conoscere solo alcune scorciatoie. Dopo aver fatto modifiche, puoi premere `Ctrl + O` per “Scrivere”, cioè salvare, il file. Dopo aver dato il comando, vedrai il menu in basso cambiare così:

```bash
File Name to write : <filename>
^G Get Help                 ^T  To Files
^C Cancel                   TAB Complete
```

Premi invio per salvare il file e tornare a modificarlo, oppure `Ctrl + C` per annullare e tornare a modificare il file.

Mentre sei nell’editor, puoi premere `Ctrl + X` per uscire da Nano e tornare al terminale. Se hai modifiche non salvate quando provi a uscire, vedrai questo nel menu in basso:

```bash
Save modified buffer (ANSWERING "No" WILL DESTROY CHANGES) ?
                            Y Yes
^C Cancel                   N No
```

Premi `Y` per salvare le modifiche non salvate, oppure `N` per scartarle.

Con queste informazioni, sarai meglio preparato la prossima volta che dovrai modificare rapidamente un file su un server remoto, dove potresti non avere accesso a un ambiente grafico completo.
