## Вступ до реляційних баз даних

- **Реляційні бази даних**: Організовують дані у пов’язані таблиці, що складаються з рядків і стовпців. Кожен рядок представляє запис, а кожен стовпець — атрибут даних.
- **Переваги реляційних баз даних**: Масштабовані, широко застосовуються в різних сферах (наприклад, охорона здоров’я, бізнес, ігри) і структуровані для підтримки надійності даних.
- **Загальні випадки використання**: Веб-розробка, системи обліку, електронна комерція, охорона здоров’я та корпоративні застосунки.

## Ключові поняття

- **Схема**: Реляційна база даних потребує схеми, яка визначає її структуру — таблиці, стовпці, типи даних, обмеження та зв’язки.
- **Первинні ключі**: Унікальні ідентифікатори для кожного рядка в таблиці. Вони необхідні для цілісності даних і використовуються для зв’язку записів між таблицями через зовнішні ключі.
- **Зовнішні ключі**: Посилання на первинні ключі в іншій таблиці, які використовуються для зв’язку пов’язаних даних між таблицями.
- **Відносини**: Завдяки зв’язку таблиць через первинні та зовнішні ключі можна структурувати нормалізовані дані та виконувати змістовні запити.
- **Діаграми сутність-зв’язок (ERD)**: Візуалізують, як сутності (таблиці) пов’язані між собою у схемі бази даних.
- **Цілісність даних**: Забезпечується за допомогою ключів і типів даних. Гарантує послідовність і точність збережених даних.

## Основи SQL

- **Запити**: Запити для отримання конкретних даних із бази даних.

```sql
SELECT * FROM dogs WHERE age < 3;
```

- **Умова WHERE**: Фільтрує результати на основі умов. Використовуйте оператори порівняння, такі як `<`, `=`, `>` тощо.
- **Вибірка з ORDER BY**: Отримує і сортує результати за стовпцем.

```sql
SELECT columns FROM table_name ORDER BY column_name;
```

## Операції з таблицями

- **Інструкція `CREATE TABLE`**: Ця інструкція використовується для створення нової таблиці в базі даних.

```sql
CREATE TABLE first_table();
```

- **Інструкція `ALTER TABLE ADD COLUMN`**: Ця інструкція використовується для додавання стовпця до існуючої таблиці.

```sql
ALTER TABLE table_name ADD COLUMN column_name DATATYPE;
```

- **Інструкція `ALTER TABLE DROP COLUMN`**: Ця інструкція використовується для видалення стовпця з існуючої таблиці.

```sql
ALTER TABLE table_name DROP COLUMN column_name;
```

- **Інструкція `ALTER TABLE RENAME COLUMN`**: Ця інструкція використовується для перейменування стовпця в таблиці.

```sql
ALTER TABLE table_name RENAME COLUMN column_name TO new_name;
```

- **Інструкція `DROP TABLE`**: Ця інструкція використовується для видалення всієї таблиці з бази даних.

```sql
DROP TABLE table_name;
```

- **Інструкція `ALTER DATABASE RENAME`**: Ця інструкція використовується для перейменування бази даних.

```sql
ALTER DATABASE database_name RENAME TO new_database_name;
```

- **Інструкція `DROP DATABASE`**: Ця інструкція використовується для видалення всієї бази даних.

```sql
DROP DATABASE database_name;
```

## Обмеження та цілісність даних

- **Інструкція `ALTER TABLE ADD COLUMN` з обмеженням**: Ця інструкція використовується для додавання стовпця з обмеженням до існуючої таблиці.

```sql
ALTER TABLE table_name ADD COLUMN column_name DATATYPE CONSTRAINT;
```

- **Обмеження `NOT NULL`**: Це обмеження гарантує, що стовпець не може містити NULL-значення.

```sql
column_name VARCHAR(50) NOT NULL
```

- **Інструкція `ALTER TABLE ADD PRIMARY KEY`**: Ця інструкція використовується для додавання обмеження первинного ключа до таблиці.

```sql
ALTER TABLE table_name ADD PRIMARY KEY(column_name);
```

- **Інструкція `ALTER TABLE DROP CONSTRAINT`**: Ця інструкція використовується для видалення обмеження з таблиці.

```sql
ALTER TABLE table_name DROP CONSTRAINT constraint_name;
```

- **Інструкція `ALTER TABLE ADD COLUMN` з зовнішнім ключем**: Ця інструкція використовується для додавання стовпця зовнішнього ключа, що посилається на іншу таблицю.

```sql
ALTER TABLE table_name ADD COLUMN column_name DATATYPE REFERENCES referenced_table_name(referenced_column_name);
```

- **Інструкція `ALTER TABLE ADD UNIQUE`**: Ця інструкція використовується для додавання обмеження UNIQUE до стовпця.

```sql
ALTER TABLE table_name ADD UNIQUE(column_name);
```

- **Інструкція `ALTER TABLE ALTER COLUMN SET NOT NULL`**: Ця інструкція використовується для встановлення обмеження NOT NULL на існуючий стовпець.

```sql
ALTER TABLE table_name ALTER COLUMN column_name SET NOT NULL;
```

- **Інструкція `INSERT` з NULL-значеннями**: Ця інструкція демонструє, як вставляти NULL-значення в таблицю.

```sql
INSERT INTO table_name(column_a) VALUES(NULL);
-- or
INSERT INTO table_name(column_b) VALUES('value'); -- if column_a allows nulls
```

- **Складений первинний ключ**: Це обмеження визначає первинний ключ, що складається з кількох стовпців.

```sql
CREATE TABLE course_enrollments (
    student_id INT,
    course_id INT,
    PRIMARY KEY (student_id, course_id)
);
```

## Маніпуляції з даними (CRUD)

- **Інструкція `INSERT`**: Ця інструкція використовується для вставлення одного рядка в таблицю.

```sql
INSERT INTO table_name(column_1, column_2) VALUES(value1, value2);
```

- **Інструкція `INSERT` з пропущеними стовпцями**: Ця інструкція показує, як вставляти значення без явного переліку імен стовпців, покладаючись на порядок стовпців за замовчуванням у таблиці.

```sql
INSERT INTO dogs VALUES ('Gino', 3);
```

- **Інструкція `INSERT` з кількома рядками**: Ця інструкція використовується для вставлення кількох рядків у таблицю за одну операцію.

```sql
INSERT INTO dogs (name, age) VALUES 
('Gino', 3),
('Nora', 2);
```

- **Інструкція `UPDATE`**: Ця інструкція використовується для оновлення існуючих даних у таблиці на основі умови.

```sql
UPDATE table_name SET column_name=new_value WHERE condition;
```

- **Інструкція `DELETE`**: Ця інструкція використовується для видалення рядків із таблиці на основі умови.

```sql
DELETE FROM table_name WHERE condition;
```

## Типи даних

- **Тип даних `NUMERIC`**: Цей тип даних використовується для збереження точних десяткових чисел із заданою точністю та масштабом.

```sql
price NUMERIC(10, 2)
```

- **Тип даних `TEXT`**: Цей тип даних використовується для збереження рядків змінної довжини без конкретного обмеження довжини.

```sql
column_name TEXT
```

- **Тип даних `INTEGER`**: Цей тип даних використовується для збереження цілих чисел без десяткових знаків.

```sql
units_sold INTEGER
```

- **Типи даних `SMALLINT` і `BIGINT`**: Це варіанти INTEGER з меншим і більшим діапазоном відповідно.

- **Тип даних `SERIAL`**: Цей тип даних використовується для створення стовпців цілих чисел з автоінкрементом у PostgreSQL.

```sql
id SERIAL
```

- **Атрибут `AUTO_INCREMENT`**: Цей атрибут використовується в MySQL для створення стовпців цілих чисел з автоінкрементом.

```sql
id INT AUTO_INCREMENT
```

- **Тип даних `VARCHAR`**: Цей тип даних використовується для збереження рядків змінної довжини з заданою максимальною довжиною.

```sql
name VARCHAR(50)
```

- **Тип даних `DATE`**: Цей тип даних використовується для збереження дат (рік, місяць, день).

```sql
event_date DATE
```

- **Тип даних `TIME`**: Цей тип даних використовується для збереження часу (година, хвилина, секунда).

```sql
start_time TIME
```

- **Тип даних `TIMESTAMP`**: Цей тип даних використовується для збереження дати і часу, опційно з інформацією про часовий пояс.

```sql
event_timestamp TIMESTAMP
event_timestamp TIMESTAMP WITH TIME ZONE
```

- **Тип даних `BOOLEAN`**: Цей тип даних використовується для збереження значень істина/хиба.

```sql
is_active BOOLEAN
```

## Відносини в базі даних

- **Типи відносин**: Це різні способи, якими таблиці можуть бути пов’язані між собою в реляційній базі даних.
  - Один до одного
  - Один до багатьох
  - Багато до одного
  - Багато до багатьох
  - Самопосилання (рекурсивні)

- **Відношення один до одного**: Цей тип відношення означає, що кожен запис в одній таблиці відповідає рівно одному запису в іншій таблиці.

```md
One employee is assigned exactly one vehicle.
Tables: employees, vehicles
```

- **Відношення один до багатьох**: Цей тип відношення означає, що один запис у таблиці може бути пов’язаний з кількома записами в іншій таблиці.

```md
One customer can have many orders.
Tables: customers → orders
```

- **Відношення багато до багатьох через проміжну таблицю**: Цей тип відношення реалізується за допомогою проміжної таблиці, яка містить зовнішні ключі обох пов’язаних таблиць.

```sql
CREATE TABLE books_authors (
    author_id INT REFERENCES authors(id),
    book_id INT REFERENCES books(id)
);
```

- **Самопосилання**: Цей тип відношення виникає, коли таблиця посилається сама на себе, створюючи ієрархічну структуру.

```md
An employee table where each employee may report to another employee.
```

## Розширений SQL (JOIN)

- **Інструкція `INNER JOIN`**: Цей JOIN повертає лише ті рядки, які мають відповідні значення в обох таблицях.

```sql
SELECT *
FROM products
INNER JOIN sales ON products.product_id = sales.product_id;
```

- **Інструкція `FULL OUTER JOIN`**: Цей JOIN повертає всі рядки з обох таблиць, включно з нерівнозначними рядками з будь-якої таблиці.

```sql
SELECT *
FROM products
FULL OUTER JOIN sales ON products.product_id = sales.product_id;
```

- **Інструкція `LEFT OUTER JOIN`**: Цей JOIN повертає всі рядки з лівої таблиці та відповідні рядки з правої таблиці.

```sql
SELECT *
FROM products
LEFT JOIN sales ON products.product_id = sales.product_id;
```

- **Інструкція `RIGHT OUTER JOIN`**: Цей JOIN повертає всі рядки з правої таблиці та відповідні рядки з лівої таблиці.

```sql
SELECT *
FROM products
RIGHT JOIN sales ON products.product_id = sales.product_id;
```

- **Інструкція `SELF JOIN`**: Цей JOIN використовується для з’єднання таблиці з самою собою для порівняння рядків у межах однієї таблиці.

```sql
SELECT A.column_name, B.column_name
FROM table_name A
JOIN table_name B ON A.related_column = B.related_column;
```

- **Інструкція `CROSS JOIN`**: Цей JOIN повертає декартовий добуток двох таблиць, поєднуючи кожен рядок першої таблиці з кожним рядком другої таблиці.

```sql
SELECT *
FROM table1
CROSS JOIN table2;
```

## Специфічні команди PostgreSQL

- **Команда входу `psql`**: Ця команда використовується для входу в PostgreSQL з конкретним ім’ям користувача та базою даних.

```bash
psql --username=freecodecamp --dbname=postgres
```

- **Команда `\l`**: Ця команда виводить список усіх баз даних у екземплярі PostgreSQL.

```sql
\l
```

- **Команди `CREATE DATABASE` і `\c`**: Ці команди використовуються для створення нової бази даних і підключення до неї.

```sql
CREATE DATABASE database_name;
\c database_name
```

- **Команда `\d`**: Ця команда виводить список усіх таблиць у поточній базі даних.

```sql
\d
```

- **Команда `\d table_name`**: Ця команда показує схему/структуру конкретної таблиці.

```sql
\d table_name
```

- **Команда `\q`**: Ця команда виходить із клієнта PostgreSQL.

```bash
\q
```

## Реляційні проти нереляційних

- **Нереляційні (NoSQL) бази даних**: Зберігають неструктуровані або напівструктуровані дані. Не потребують жорсткої схеми і більш гнучкі для змінних моделей даних.
- **Вибір між реляційними та нереляційними**: Залежить від природи ваших даних і вимог застосунку.
- **Реляційні проти нереляційних**: Обирайте реляційні для структурованих даних і послідовності; NoSQL — для гнучкості та швидкозмінних даних.

## Популярні системи РБД

- **MySQL**: Відкрите, надійне, широко використовується у веб-розробці, підтримується великою спільнотою.
- **PostgreSQL**: Відкрите, просунуте, розширюване. Підтримує кастомні типи даних і серверне програмування.
- **SQLite**: Легковагове, файлове, безсерверне. Ідеально для невеликих застосунків.
- **Microsoft SQL Server**: Пропрієтарна, корпоративна база даних.
- **Oracle Database**: Комерційна РБД, відома великомасштабною продуктивністю та масштабованістю.
