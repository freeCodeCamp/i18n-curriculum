# --description--

Політика same-origin — це вбудований захід безпеки у ваших веббраузерах, який забороняє одному сайту читати приватні дані з іншого сайту. Це правило захищає ваші особисті акаунти від хакерів.

Походження складається з трьох конкретних частин: вебпротоколу, доменного імені та номера порту.

Наприклад, у адресі `https://example.com:443` протокол — HTTPS, доменне ім’я — `example.com`, а номер порту — `443`.

Браузери часто приховують номер порту, якщо він стандартний. HTTPS за замовчуванням використовує порт 443, а HTTP — порт 80. Навіть якщо порт прихований, браузер все одно його перевіряє. Два сайти мають однакове походження лише тоді, коли всі три ці частини точно співпадають.

За цим суворим правилом браузер блокує сайт, який намагається шпигувати за іншим сайтом. Уявіть, що ви відкрили свій онлайн-банкінг в одній вкладці браузера, а в іншій — шкідливий сайт. Політика same-origin не дозволить шкідливому сайту вкрасти ваші банківські дані.

Однак сучасним сайтам часто потрібно обмінюватися даними між різними походженнями. Наприклад, фронтенд-сторінка може потребувати отримати дані з окремого бекенд-сервера. Правила same-origin у браузерах блокують такі безпечні запити, тому розробникам знадобився спосіб безпечно ділитися даними. Саме тут на допомогу приходить CORS.

CORS — це система, яка дозволяє серверам відкривати доступ через правила браузера. Вона використовує спеціальні заголовки, щоб надати доступ до даних довіреним сайтам.

Ось що відбувається, коли ваш браузер запитує дані з іншого сайту:

* Він додає заголовок `Origin`, який вказує сайт, що робить запит
* Сервер, що отримує запит, перевіряє це ім’я
* Якщо сервер довіряє сайту, він відповідає спеціальним заголовком дозволу, який повідомляє браузеру, що сайту дозволено читати отримані дані.

Якщо сервер не надсилає цей дозвіл, браузер миттєво блокує дані. Тоді фронтенд-сайт отримує повідомлення про помилку замість приватних файлів.

У застосунках Node.js з Express CORS реалізується за допомогою пакета `cors`:

```javascript
const express = require('express');
const cors = require('cors');
const app = express();

// this enables CORS for all origins and all routes for development
app.use(cors());
```

У продакшені вам може знадобитися передати об’єкт конфігурації, щоб обмежити доступ до конкретного домену:

```javascript
const corsOptions = {
  origin: ['https://yourapp.com', 'https://admin.yourapp.com'],
  credentials: true
};

app.use(cors(corsOptions));
```

Якщо ви використовуєте чистий Node.js, доведеться вручну встановлювати заголовки відповіді CORS для кожного запиту:

```javascript
res.setHeader('Access-Control-Allow-Origin', 'https://example.com');
res.setHeader('Access-Control-Allow-Methods', 'GET, POST, PUT, DELETE, OPTIONS');
res.setHeader('Access-Control-Allow-Headers', 'Content-Type, Authorization');
```
