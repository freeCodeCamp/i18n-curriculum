# --description--

오류의 또 다른 중요한 부분은 스택 추적입니다. 이것은 애플리케이션에서 오류가 발생한 시점과 오류로 이어진 함수 호출들을 이해하는 데 도움을 줍니다. 예를 들어, 다음과 같은 코드가 있다고 가정해 보세요:

```javascript
const a = 5;
const b = 10;

function add() {
  return c;
}

function print() {
  add();
}

print();
```

우리 함수 `print()`는 `add()`를 호출해야 하며, 이 함수는 현재 선언되지 않은 `c`라는 변수를 반환합니다. 이에 해당하는 오류는 다음과 같습니다:

<img src="https://cdn.statically.io/gh/TheOdinProject/curriculum/284f0cdc998be7e4751e29e8458323ad5d320303/foundations/javascript_basics/understanding_errors/imgs/01.png" width="100%" alt="개발자 콘솔에 스택 추적과 함께 표시된 오류">

스택 추적은 다음을 알려줍니다:

1. `c is not defined`가 5번째 줄에 선언된 `add()`의 범위 내에 있습니다.
1. `add()`는 9번째 줄에 선언된 `print()`에 의해 호출되었습니다.
1. `print()` 자체는 12번째 줄에서 호출되었습니다.

따라서 스택 추적을 통해 오류가 발생한 경로를 그 기원까지 추적할 수 있는데, 여기서는 `add()`의 선언이 그 기원입니다.

## --answers--  

스택 추적은 함수 `add()`가 `c`의 값을 올바르게 반환함을 보여줍니다.

---

스택 추적은 구문 오류를 수정하기 위해 코드의 어느 줄을 편집해야 하는지 알려줍니다.

---

스택 추적은 오류로 이어진 함수 호출의 자세한 경로를 제공하여 `c`가 잘못 참조된 위치를 식별하는 데 도움을 줍니다.

---

스택 추적은 `print()`가 호출되었음을 나타내지만 `add()`의 오류에 대한 세부 정보는 제공하지 않습니다.
