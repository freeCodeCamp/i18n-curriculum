# --description--

In der Softwareentwicklung ist eine Bibliothek wie ein Werkzeugkasten für Entwickler.

Anstatt jeden einzelnen Teil des Codes selbst von Grund auf neu zu implementieren, stellt Ihnen eine Bibliothek vorgefertigten und wiederverwendbaren Code zur Verfügung, wie Funktionen, Klassen und Datenstrukturen, die Sie in Ihren Projekten verwenden können.

Python verfügt über eine umfangreiche Standardbibliothek mit vielen verschiedenen eingebauten Modulen. Sie sind alle standardisierte, gut geprüfte Lösungen für viele der Probleme und Aufgaben, denen Sie als Programmierer täglich begegnen, wie zum Beispiel:

* Interaktion mit dem Betriebssystem.
    
* Arbeiten mit Dateien.
    
* Netzwerken.
    
* Arbeiten mit Datum und Uhrzeit.
    
* Ausführen mathematischer Operationen.
    
* Verwendung von regulären Ausdrücken.
    
* Testen und Debuggen Ihres Codes.
    
* Und vieles mehr!
    

Einige Beispiele für beliebte eingebaute Module sind `math`, `random`, `re` (Abkürzung für „reguläre Ausdrücke“) und `datetime`.

Das `math`-Modul enthält hilfreiche Funktionen für komplexere mathematische Operationen.

Das `random`-Modul ist nützlich, um Zufallszahlen zu erzeugen.

Das `re`-Modul wird für die Arbeit mit regulären Ausdrücken verwendet.

Und das `datetime`-Modul ist hilfreich für die Arbeit mit Datum und Uhrzeit in Python.

Aber wie können Sie auf die Variablen, Konstanten, Funktionen und Klassen zugreifen, die in diesen eingebauten Modulen definiert sind?

Sie verwenden eine Import-Anweisung. Diese Anweisungen ermöglichen es Ihnen, Module in Ihr Python-Skript zu importieren. Import-Anweisungen werden in der Regel am Anfang der Datei geschrieben. Außerdem können Sie sie je nach Bedarf anpassen. Zuerst verwenden Sie die `import`-Anweisung, gefolgt vom Namen des Moduls:

```python
import module_name
```

Angenommen, Sie möchten das `math`-Modul importieren. In diesem Fall würden Sie dies am Anfang Ihrer Datei schreiben:

```python
import math
```

Wenn Sie dann eine Funktion aus diesem Modul in Ihrem Python-Skript aufrufen müssen, verwenden Sie die Punktnotation, mit dem Namen des Moduls gefolgt vom Namen der Funktion:

```python
module_name.function_name() 
```

Zum Beispiel, um die Quadratwurzel von 36 zu erhalten, schreiben Sie `math` gefolgt von einem Punkt und dann `sqrt`, eine Abkürzung für Quadratwurzel, und innerhalb der runden Klammern übergeben Sie die notwendigen Argumente. In diesem Fall müssen wir nur die Zahl übergeben, von der wir die Quadratwurzel wollen:

```python
math.sqrt(36)
```

Dies ist die einfachste Form einer Import-Anweisung, aber es gibt auch andere Alternativen.

Wenn Sie das Modul unter einem anderen Namen importieren möchten (auch bekannt als „Alias“), können Sie diese Syntax verwenden, mit `as` gefolgt vom Alias am Ende der Import-Anweisung:

```python
import module_name as module_alias
```

Dies wird oft verwendet, um lange Modulnamen abzukürzen oder um Namenskonflikte zu vermeiden.

Zum Beispiel, um im Code auf das `math`-Modul als `m` zu verweisen, können Sie ihm einen Alias zuweisen, so:

```python
import math as m 
```

Dann können Sie die Elemente des Moduls mit dem Alias aufrufen:

```python
m.sqrt(36)
```

Manchmal müssen Sie aber nicht alles aus einem Modul importieren. Vielleicht brauchen Sie nur eine oder zwei bestimmte Funktionen oder Klassen. Python hat genau das, was Sie in diesem Fall brauchen.

Jetzt beginnt die Import-Anweisung mit `from`, gefolgt vom Namen des Moduls, und dann dem Schlüsselwort `import`, gefolgt von den Namen der Elemente, die Sie importieren möchten:

```python
from module_name import name1, name2
```

Dann können Sie diese Namen ohne das Modulpräfix in Ihrem Python-Skript verwenden.

Wenn Sie diesen Namen Aliase zuweisen möchten, können Sie das mit dem Schlüsselwort `as` nach jedem Namen tun, gefolgt vom Alias, den Sie verwenden möchten:

```python
from module_name import name1 as alias1, name2 as alias2
```

Angenommen, Sie möchten nur die Funktionen radians, sine und cosine aus dem `math`-Modul importieren. Sie würden schreiben:

```python
from math import radians, sin, cos
```

Jetzt können Sie diese Funktionen direkt in Ihrem Code aufrufen, ohne das `math`-Modul als Präfix.

Hier haben wir ein detaillierteres Beispiel:

Um den Sinus und Kosinus eines bestimmten Winkels, der ursprünglich in Grad angegeben ist, zu finden, können wir die `radians`-Funktion aufrufen, um ihn in Radiant umzuwandeln, und dann die Sinus- und Kosinusfunktionen aufrufen, wobei wir den Winkel in Radiant übergeben:

```python
from math import radians, sin, cos

angle_degrees = 40
angle_radians = radians(angle_degrees)

sine_value = sin(angle_radians)
cos_value = cos(angle_radians)

print(sine_value) # 0.6427876096865393
print(cos_value)  # 0.766044443118978
```

Beachten Sie, wie wir die Funktionen direkt aufrufen, ohne den Namen des Moduls als Präfix. Das liegt daran, dass wir die Funktionen mit dieser alternativen Syntax importiert haben.

Das ist hilfreich, kann aber zu Namenskonflikten führen, wenn Sie bereits Funktionen oder Variablen mit demselben Namen im Python-Skript definiert haben. Das sollten Sie bei der Wahl der Import-Anweisung berücksichtigen.

Und schließlich gibt es diese Import-Anweisung, die mit einem Sternchen endet. Das Sternchen sagt Python, dass Sie alles aus diesem Modul importieren möchten, aber so, dass Sie den Namen des Moduls nicht als Präfix verwenden müssen:

```python
from module_name import *
```

Wenn Sie zum Beispiel das `math`-Modul so importieren, können Sie jede Funktion aus diesem Modul aufrufen, ohne den Namen des Moduls als Präfix anzugeben. Hier sind einige Beispiele:

```python
from math import *
print(sqrt(36))  # 6.0
print(pow(5, 2)) # 25.0
print(exp(1))    # 2.718281828459045
```

Dies wird jedoch allgemein nicht empfohlen, da es zu Namensraumkonflikten führen kann und es schwieriger macht, nachzuvollziehen, woher bestimmte Namen stammen.

Import-Anweisungen funktionieren genau gleich für Funktionen, Klassen, Konstanten, Variablen und alle anderen Elemente, die im Modul definiert sind.

Hier ist ein Beispiel für eine Konstante aus dem `math`-Modul, die Zahl `pi`:

```python
import math
print(math.pi)
```

Und hier ist ein Beispiel für eine Klasse aus dem `datetime`-Modul. Wir erstellen ein Datumsobjekt, das den 15. Juli 1959 darstellt. Dann weisen wir dieses `date`-Objekt einer Variablen zu und greifen mit Punktnotation einzeln auf Tag, Monat und Jahr zu:

```python
import datetime
birthday = datetime.date(1959, 7, 15)
print(birthday.day)    # 15
print(birthday.month)  # 7
print(birthday.year)   # 1959
```

Weitere Informationen zum Inhalt des Moduls finden Sie in der offiziellen Python-Dokumentation zu diesem Modul.

Sehr gut. Nun, da Sie mehr über Module wissen, sollten Sie auch dieses sehr wichtige Idiom in Python-Skripten kennen, da sie sehr eng miteinander verbunden sind:

```python
if __name__ == '__main__': 
    # Code
```

`__name__` ist eine spezielle eingebaute Variable in Python.

Wenn eine Python-Datei direkt ausgeführt wird, setzt Python den Wert dieser Variable auf den String `"__main__"`.

Wenn die Python-Datei jedoch als Modul in ein anderes Python-Skript importiert wird, wird der Wert der `__name__`-Variablen auf den Namen dieses Moduls gesetzt (normalerweise der Dateiname ohne die `.py`-Erweiterung).

Deshalb finden Sie in Python-Skripten oft diese Bedingung. Sie enthält den Code, den Sie **nur ausführen möchten, wenn** das Python-Skript als Hauptprogramm läuft:

```python
if __name__ == '__main__': 
    # Code
```

Wenn das Skript als Modul importiert wird, wird der Code in diesem Block nicht ausgeführt.

Das ist hilfreich, weil es Python-Skripten erlaubt, zwei Zwecke zu erfüllen. Sie können direkt ausgeführt werden, um ihre Hauptlogik auszuführen, oder sie können in ein anderes Modul importiert werden, ohne ihre Hauptlogik auszuführen.
