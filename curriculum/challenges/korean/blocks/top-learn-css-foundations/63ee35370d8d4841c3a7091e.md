# --description--

조합자는 선택자들을 그룹화하거나 체이닝하는 것과는 다르게, 선택자들 간의 관계를 나타내면서 여러 선택자를 결합할 수 있게 해줍니다. 조합자는 총 네 가지 유형이 있지만, 지금은 CSS에서 선택자 사이에 공백 하나로 나타내는 자손 요소 조합자만 보여드리겠습니다. 자손 요소 조합자는 마지막 선택자와 일치하는 요소가 이전 선택자와 일치하는 조상 요소(부모, 조부모 등)를 가지고 있을 때만 선택됩니다.

예를 들어 `.ancestor .child`는 `child` 클래스를 가진 조상 요소가 있을 때 `ancestor` 클래스를 가진 요소를 선택합니다. 다른 말로 하면, 자식 요소는 `ancestor` 안에 중첩되어 있을 때만 선택되며, 중첩 깊이에 상관없습니다. 아래 예제를 잠깐 살펴보고, 제공된 CSS 규칙에 따라 어떤 요소가 선택될지 맞혀보세요:

```html
<!-- index.html -->

<div class="ancestor"> <!-- A -->
  <div class="contents"> <!-- B -->
    <div class="contents"> <!-- C -->
    </div>
  </div>
</div>

<div class="contents"></div> <!-- D -->
```

```css
/* styles.css */

.ancestor .contents {
  /* some declarations */
}
```

위 예제에서 `contents` 클래스를 가진 처음 두 요소(`B`와 `C`)는 선택되지만, 마지막 요소(`D`)는 선택되지 않습니다. 맞게 맞혔나요?

규칙에 조합자를 몇 개나 추가해도 제한이 없으므로 `.one .two .three .four`도 완전히 유효합니다. 이 규칙은 `four` 클래스를 가진 조상 요소가 있고, 그 조상 요소가 다시 `three` 클래스를 가진 조상을 가지고 있을 때 `two` 클래스를 가진 요소를 선택합니다. 하지만 이렇게 깊게 중첩된 요소를 선택하려고 하면 혼란스럽고 길어질 수 있으며, 특이성 문제도 발생할 수 있으니 일반적으로 피하는 것이 좋습니다.
