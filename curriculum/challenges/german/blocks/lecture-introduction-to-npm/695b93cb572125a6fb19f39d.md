# --description--

Ihre Abhängigkeiten aktuell zu halten – und zu wissen, wie man eine bestimmte Version installiert – ist wichtig, um Ihr Projekt stabil zu halten. So vermeiden Sie unerwartete Bugs und haben mehr Kontrolle, wenn neue Versionen veröffentlicht werden.

Um eine bestimmte Version eines Pakets zu installieren, verwenden Sie dieses Format:

```bash
npm install package@version-number
```

Zum Beispiel, wenn Ihr Projekt eine ältere Version von Express benötigt, sagen wir `4.21.2`, können Sie sie so installieren:

```bash
npm install express@4.21.2
```

Wenn Sie keine Versionsnummer angeben, installiert npm automatisch die neueste verfügbare Version.

Wenn Sie nicht sicher sind, welche Versionen existieren, können Sie sie mit diesem Befehl überprüfen:

```bash
npm view package-name versions
```

Zum Beispiel:

```bash
npm view express versions
```

Um Pakete zu aktualisieren, können Sie sie einzeln oder alle auf einmal aktualisieren. Es ist hilfreich, vorher zu sehen, welche veraltet sind:

```bash
npm outdated
```

Dies zeigt Ihnen, welche Abhängigkeiten neuere Versionen haben, und Sie sollten eine Ausgabe ähnlich dieser sehen:

```bash
user@Kolade ~/Desktop/fCC/script-code/node/npm/install-spec-version  % npm outdated
Package  Current  Wanted  Latest  Location              Depended by
chalk      5.5.0   5.5.0   5.6.2  node_modules/chalk    install-spec-version
cleave     1.0.1   1.0.1   1.0.2  node_modules/cleave   install-spec-version
express   4.21.2  4.21.2   5.2.1  node_modules/express  install-spec-version
```

Sobald Sie überprüft haben, welche Abhängigkeiten veraltet sind, können Sie mit dem Aktualisieren beginnen.

Um eine einzelne Abhängigkeit zu aktualisieren, verwenden Sie Folgendes:

```bash
npm update package-name
```

Dies aktualisiert das Paket nur innerhalb des von Ihrer `package.json` Datei erlaubten Versionsbereichs. Zum Beispiel können Sie Express so aktualisieren:

```bash
npm update express
```

Wenn Sie den Versionsbereich überspringen und die neueste Version eines Pakets installieren möchten, verwenden Sie diesen Befehl:

```bash
npm install package-name@latest
```

Um alle Abhängigkeiten zu aktualisieren, die innerhalb der erlaubten Versionsbereiche liegen, führen Sie aus:

```bash
npm update
```

Beachten Sie jedoch, dass `npm update` ein Paket nicht auf eine neue Hauptversion aktualisiert, wenn Ihre `package.json` dies nicht erlaubt. Hauptversionen enthalten oft Breaking Changes, daher wendet npm diese nicht automatisch an.

Wenn Sie speziell die neueste Hauptversion möchten, müssen Sie Folgendes ausführen:

```bash
npm install package-name@latest
```

Dies zwingt npm, die aktuellste Version zu installieren, egal welcher Versionsbereich in Ihrer `package.json` steht.
