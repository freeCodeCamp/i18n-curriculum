# --interactive--

固定配置とスティッキー配置は、絶対配置とは異なる動作を持つ重要なCSSの配置戦略です。要素が`position: fixed`で配置されると、通常の文書の流れから外れ、ビューポートに対して相対的に配置されます。つまり、ユーザーがスクロールしても同じ位置に留まります。これは、常に表示しておきたいヘッダーやナビゲーションバーなどの要素に使われます。例えば、見出しをページの上部に固定したい場合、次のコードを使えます。

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<h1>Fixed Header</h1>

<p>Lorem ipsum dolor sit amet, consectetur adipiscing elit. Integer nec odio. Praesent libero. Sed cursus ante dapibus diam.</p>
<p>Sed nisi. Nulla quis sem at nibh elementum imperdiet. Duis sagittis ipsum. Praesent mauris.</p>
<p>Fusce nec tellus sed augue semper porta. Mauris massa. Vestibulum lacinia arcu eget nulla.</p>
<p>Class aptent taciti sociosqu ad litora torquent per conubia nostra, per inceptos himenaeos.</p>
<p>Curabitur sodales ligula in libero. Sed dignissim lacinia nunc. Curabitur tortor.</p>
<p>Pellentesque nibh. Aenean quam. In scelerisque sem at dolor. Maecenas mattis.</p>
<p>Sed convallis tristique sem. Proin ut ligula vel nunc egestas porttitor. Morbi lectus risus.</p>
<p>Donec congue lacinia dui, a porttitor lectus condimentum laoreet. Nunc eu ullamcorper orci.</p>
<p>Quisque eget odio ac lectus vestibulum faucibus eget in metus. In pellentesque faucibus vestibulum.</p>
<p>Nulla at nulla justo, eget luctus tortor. Nulla facilisi. Duis aliquet egestas purus in blandit.</p>
```

```css
body {
  margin: 0;
  padding-top: 60px;
  font-family: Arial, sans-serif;
  line-height: 1.6;
}

h1 {
  position: fixed;
  top: 0;
  width: 500px;
  background: white;
  padding: 10px;
  border-bottom: 2px solid #ccc;
}

p {
  max-width: 600px;
  margin: 20px auto;
}
```

:::

この例では、`h1`要素がビューポートの上部に固定され、ユーザーがスクロールしてもその位置に留まります。これは、スティッキーヘッダーや常に表示されるナビゲーションなど、持続的なUI要素を作るのに特に便利です。

`position: sticky`は相対配置と固定配置のハイブリッドのように動作します。最初は相対配置のように文書の流れの中に留まりますが、ユーザーが要素をある位置までスクロールすると、その位置で「固定」され、固定配置のように振る舞います。これは、ユーザーが特定の位置までスクロールしたときにのみ固定されるスティッキーナビゲーションバーのような要素を作るのに適しています。スティッキー配置を適用する方法は次の通りです。

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<h1>Sticky Header</h1>

<p>Lorem ipsum dolor sit amet, consectetur adipiscing elit. Integer nec odio. Praesent libero. Sed cursus ante dapibus diam.</p>
<p>Sed nisi. Nulla quis sem at nibh elementum imperdiet. Duis sagittis ipsum. Praesent mauris.</p>
<p>Fusce nec tellus sed augue semper porta. Mauris massa. Vestibulum lacinia arcu eget nulla.</p>
<p>Class aptent taciti sociosqu ad litora torquent per conubia nostra, per inceptos himenaeos.</p>
<p>Curabitur sodales ligula in libero. Sed dignissim lacinia nunc. Curabitur tortor.</p>
<p>Pellentesque nibh. Aenean quam. In scelerisque sem at dolor. Maecenas mattis.</p>
<p>Sed convallis tristique sem. Proin ut ligula vel nunc egestas porttitor. Morbi lectus risus.</p>
<p>Donec congue lacinia dui, a porttitor lectus condimentum laoreet. Nunc eu ullamcorper orci.</p>
<p>Quisque eget odio ac lectus vestibulum faucibus eget in metus. In pellentesque faucibus vestibulum.</p>
<p>Nulla at nulla justo, eget luctus tortor. Nulla facilisi. Duis aliquet egestas purus in blandit.</p>
```

```css
h1 {
  position: sticky;
  top: 30px;
  left: 30px;
}
```

:::

この例では、`h1`要素はユーザーが`30px`だけ上からスクロールするまでは通常通りに動作し、その後ビューポートの上部に「固定」されます。

絶対配置、固定配置、スティッキー配置の主な違いは次の通りです。

- 絶対配置は要素を文書の流れから外し、最も近い配置された祖先要素、または存在しなければ初期の包含ブロックに対して相対的に配置します。スクロールしてもその位置に留まります。
- 固定配置も要素を文書の流れから外しますが、ビューポートに対して固定されるため、ページをスクロールしても同じ場所に表示され続けます。
- スティッキー配置は最初は通常の文書の流れに留まりますが、定義された閾値を超えてスクロールすると、その位置に「固定」されます。

これらの戦略を使うことで、浮動要素やスティッキーヘッダー、より動的なUIコンポーネントを作成でき、重要な情報を常にアクセスしやすくしてユーザー体験を向上させられます。
