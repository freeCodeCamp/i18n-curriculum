# --description--

في JavaScript، `Set` هو كائن مدمج لإدارة مجموعات البيانات. يسمح لك بتخزين قيم فريدة من أي نوع، سواء كانت بدائية أو مراجع كائنات. يضمن `Set` أن كل قيمة تظهر مرة واحدة فقط، مما يجعله مفيدًا لإزالة التكرارات من مصفوفة أو التعامل مع مجموعات من القيم المميزة.

أما بالنسبة لـ `WeakSet`، فهو نوع خاص من `Set` بميزات أقل يسمح لك بتخزين مراجع كائنات ورموز محتفظ بها بشكل ضعيف. على عكس `Set`، لا يدعم `WeakSet` القيم البدائية مثل الأرقام أو السلاسل النصية.

على عكس `Set` العادي، يخزن `WeakSet` الكائنات فقط، والمراجع إلى تلك الكائنات "ضعيفة"، مما يعني أن WeakSets لا تمنع جمع القمامة من إزالة الكائنات المخزنة إذا لم تكن هناك مراجع أخرى لها. ببساطة، إذا لم يكن الكائن مستخدمًا في مكان آخر في شفرتك، فسيتم حذفه تلقائيًا لتحرير الذاكرة.

لإنشاء `Set`، تستخدم دالة البناء `Set` وتعيّنها إلى متغير:

```js
const myFirstSet = new Set();
```

يمكنك أيضًا تهيئة `Set` بالقيم:

```js
const treeSet = new Set(["Baobab", "Jackalberry", "Mopane Tree", "Breadfruit"]);
```

إذا سجلت `Set` في وحدة التحكم، فسيبدو الناتج هكذا:

```js
/*
Set(4) {"Baobab", "Jackalberry", "Mopane Tree", "Breadfruit"}
  [[Entries]]
  0: "Baobab"
    value: "Baobab"
  1: "Jackalberry"
    value: "Jackalberry"
  2: "Mopane Tree"
    value: "Mopane Tree"
  3: "Breadfruit"
    value: "Breadfruit"
  size: 4
  [[Prototype]]: Set
*/
```

إذا لم تهيئ `Set` بالقيم، يمكنك استخدام الطريقة `add()` لإضافة عنصر إلى `Set`:

```js
const treeSet = new Set();

// Add items to the treeSet
treeSet.add("Baobab");
treeSet.add("Jackalberry");
treeSet.add("Mopane Tree");
treeSet.add("Breadfruit");
```

يبقى الناتج ومظهره في وحدة التحكم كما هو.

لا تنسَ أن العناصر المكررة ستُتجاهل في `Set`:

```js
const treeSet = new Set();

// Add items to the treeSet
treeSet.add("Baobab");
treeSet.add("Jackalberry");
treeSet.add("Mopane Tree");
treeSet.add("Breadfruit");
treeSet.add("Baobab"); //duplicate item will be ignored

console.log(treeSet);
// Set(4) {"Baobab", "Jackalberry", "Mopane Tree", "Breadfruit"}
```

الطرق الأخرى التي يمكنك استخدامها للتعامل مع `Set` هي:

- `delete()`  
- `clear()`
- `has()`
- `entries()`
- `forEach()`
- `keys()`
- `values()`

لننظر كيف تعمل هذه الطرق واحدة تلو الأخرى. تزيل `delete()` عنصرًا محددًا من `Set`:

```js
const treeSet = new Set();

// Add items to the treeSet
treeSet.add("Baobab");
treeSet.add("Jackalberry");
treeSet.add("Mopane Tree");
treeSet.add("Breadfruit");

treeSet.delete("Breadfruit");

console.log(treeSet); // Set(3) {"Baobab", "Jackalberry", "Mopane Tree"}
```

تتحقق `has()` مما إذا كانت قيمة محددة موجودة في `Set`:

```js
const treeSet = new Set();

// Add items to the treeSet
treeSet.add("Baobab");
treeSet.add("Jackalberry");
treeSet.add("Mopane Tree");
treeSet.add("Breadfruit");

treeSet.delete("Breadfruit");

console.log(treeSet.has("Breadfruit")); // false
```

تعيد `entries()` مكرّرًا لـ `Set` يحتوي على مصفوفة من القيم بصيغة `[value, value]`:

```js
const treeSet = new Set();

// Add items to the treeSet
treeSet.add("Baobab");
treeSet.add("Jackalberry");
treeSet.add("Mopane Tree");
treeSet.add("Breadfruit");

treeSet.delete("Breadfruit");

console.log(treeSet.entries());
// SetIterator {"Baobab" => "Baobab", "Jackalberry" => "Jackalberry", "Mopane Tree" => "Mopane Tree"}
```

تعرض `keys()` و`values()` القيم في `Set`. `keys()` هي مجرد اسم مستعار لطريقة `values()`:

```js
const treeSet = new Set();

// Add items to the treeSet
treeSet.add("Baobab");
treeSet.add("Jackalberry");
treeSet.add("Mopane Tree");
treeSet.add("Breadfruit");

treeSet.delete("Breadfruit");

console.log("Keys: ", treeSet.keys());
console.log("Values: ", treeSet.values());
// Keys: SetIterator {"Baobab", "Jackalberry", "Mopane Tree"}
// Values: SetIterator {"Baobab", "Jackalberry", "Mopane Tree"}
```

تتيح لك `forEach()` التكرار خلال `Set`:

```js
const treeSet = new Set();

// Add items to the treeSet
treeSet.add("Baobab");
treeSet.add("Jackalberry");
treeSet.add("Mopane Tree");
treeSet.add("Breadfruit");

treeSet.delete("Breadfruit");

treeSet.forEach((tree) => console.log(tree));
/*
Baobab
Jackalberry
Mopane Tree
*/
```

تزيل `clear()` كل عناصر المصفوفة:

```js
const treeSet = new Set();

// Add items to the treeSet
treeSet.add("Baobab");
treeSet.add("Jackalberry");
treeSet.add("Mopane Tree");
treeSet.add("Breadfruit");

treeSet.delete("Breadfruit");

treeSet.clear();

console.log(treeSet); // Set(0) {size: 0}
```

ومن الجدير بالذكر أيضًا وجود خاصية `size` التي تعيد عدد العناصر في `Set`:

```js
const treeSet = new Set();

// Add items to the treeSet
treeSet.add("Baobab");
treeSet.add("Jackalberry");
treeSet.add("Mopane Tree");
treeSet.add("Breadfruit");

treeSet.delete("Breadfruit");

console.log(treeSet.size); // 3
```

تمامًا مثل `Set`، هناك أيضًا دالة البناء `WeakSet` التي يمكنك استخدامها لإنشاء `WeakSet`:

```javascript
const treeWeakSet = new WeakSet();
```

يحتوي `WeakSet` أيضًا على الطرق `add()` و`delete()` و`has()`:

```javascript
const treeWeakSet = new WeakSet();

const baobab = { name: "Baobab" };
const jackalberry = { name: "Jackalberry" };
const mopaneTree = { name: "Mopane Tree" };
const breadfruit = { name: "Breadfruit" };

treeWeakSet.add(baobab);
treeWeakSet.add(jackalberry);
treeWeakSet.add(mopaneTree);
treeWeakSet.add(breadfruit);

treeWeakSet.delete(jackalberry);
console.log(treeWeakSet.has(jackalberry)); // false

console.log(treeWeakSet);
```

في الناتج، تظهر محتويات `WeakSet` بهذا الشكل:

```javascript
/*
WeakSet {{…}, {…}, {…}}
  [[Entries]]
    0: value: {name: "Mopane Tree"}
    1: value: {name: "Baobab"}
    2: value: {name: "Breadfruit"}
  [[Prototype]]: WeakSet
*/
```

المحتويات مرئية هنا لأن متغيرات الكائنات لا تزال تحتفظ بمراجع قوية، مما يبقي تلك الكائنات في الذاكرة. تذكر أن WeakSets غير قابلة للتكرار، لذلك لا توجد طريقة للتكرار عبر مدخلاتها أو الوصول إليها برمجيًا. ما تراه في الناتج هو فقط لمساعدة تصحيح الأخطاء.

لا تنسَ أن الرموز والكائنات ذات المفاتيح والقيم المعرفة جيدًا فقط هي المدعومة. إضافة قيمة بدائية، مثل الأرقام أو السلاسل النصية، ستؤدي إلى حدوث خطأ:

```js
treeWeakSet.add("Alan Smith");

console.log(treeWeakSet); // Invalid value used in weak set
//    at WeakSet.add (<anonymous>)
```

الفرق الرئيسي بين `Set` و`WeakSet` هو أن `Set` يخزن أي قيمة، بينما يمكن لـ `WeakSet` تخزين الكائنات فقط.

إليك بعض الفروقات الملحوظة الأخرى بين `Set` و`WeakSet`:

| الميزة | Set | WeakSet |
| --- | --- | --- |
| نوع القيم المخزنة | يخزن أي نوع من البيانات | يخزن الكائنات فقط |
| نوع المرجعية | مراجع قوية | مراجع ضعيفة |
| التكرار | يدعم التكرار باستخدام `forEach` والحلقات | لا يدعم التكرار |
| الطرق والخصائص | `add()`, `delete()`, `has()`, `keys()`, `values()`, `size`، وأكثر | فقط `add()`, `delete()`, و`has()` |
| حالة الاستخدام | مجموعة عامة من القيم الفريدة وإزالة التكرارات من المصفوفات | تتبع الذاكرة بكفاءة لمراجع الكائنات |

يمكنك رؤية الفروقات في أنواع القيم التي يمكن للمجموعتين تخزينها، ودعمهما للتكرار عبر الكائنات المخزنة، وحالات الاستخدام المثالية لكل منهما. خذ لحظة لقراءة محتوى هذا الجدول.
