# --description--

`repeating-linear-gradient()` 関数は `linear-gradient()` と非常に似ていますが、大きな違いは指定したグラデーションパターンを繰り返すことです。`repeating-linear-gradient()` はさまざまな値を受け入れますが、このチャレンジでは簡単のために角度の値とカラーストップの値を使います。

角度の値はグラデーションの方向を示します。カラーストップは幅の値のようなもので、どこで色の変化が起きるかを示し、パーセンテージかピクセル数で指定します。

コードエディターで示された例では、グラデーションは `yellow` の色で0ピクセルから始まり、そこから40ピクセル離れたところで2番目の色 `blue` に混ざります。次のカラーストップも40ピクセルにあるため、グラデーションはすぐに3番目の色 `green` に変わり、そこから80ピクセル離れたところで4番目の色 `red` に混ざります。

この例では、カラーストップを2つずつのペアとして考えるとわかりやすく、2色ずつが混ざり合っています。

```css
0px [yellow -- blend -- blue] 40px [green -- blend -- red] 80px
```

もし2つのカラーストップの値が同じ色であれば、混ざり合いは目立たず、同じ色の間の変化なので、次の色への急な変化が起きてストライプになります。

# --instructions--

`repeating-linear-gradient()` を変更してグラデーションの角度を `45deg` にし、最初の2つのカラーストップを `yellow` に、次の2つのカラーストップを `black` に設定してストライプを作ってください。

# --hints--

`repeating-linear-gradient()` の角度は45degにしてください。

```js
assert.match(code,/background:\s*?repeating-linear-gradient\(\s*?45deg/gi);
```

`repeating-linear-gradient()` の角度はもはや90degではありません。

```js
assert.notMatch(code, /90deg/gi);
```

0ピクセルのカラーストップは `yellow` にしてください。

```js
assert.match(code, /yellow\s+?0(px)?/gi);
```

40ピクセルの最初のカラーストップは `yellow` にしてください。

```js
assert.match(code, /yellow\s+?40px/gi);
```

40ピクセルの2番目のカラーストップは `black` にしてください。

```js
assert.match(code, /yellow\s+?40px,\s*?black\s+?40px/gi);
```

80ピクセルの最後のカラーストップは `black` にしてください。

```js
assert.match(code, /black\s+?80px/gi);
```
