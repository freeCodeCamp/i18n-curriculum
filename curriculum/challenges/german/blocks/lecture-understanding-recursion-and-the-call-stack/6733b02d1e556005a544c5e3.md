# --interactive--

Rekursion ist eine komplexe Funktion, die es Ihnen erlaubt, eine Funktion wiederholt aufzurufen, bis ein Basisfall erreicht ist. Im Gegensatz zu einer herkömmlichen Schleife ermöglicht Rekursion die Verarbeitung von etwas mit unbekannter Tiefe, wie zum Beispiel tief verschachtelte Objekte/Arrays oder einen Dateibaum. Sie können sie aber auch für einfachere Aufgaben verwenden, wie das Herunterzählen von einer gegebenen Zahl.

Lassen Sie uns eine Funktion erstellen, die genau das macht. Wir nennen unsere Funktion `recursiveCountdown` und sie soll eine Zahl akzeptieren. Wir lassen sie diese Zahl in der Konsole ausgeben:

:::interactive_editor

```js
const recursiveCountdown = (number) => {
  console.log(number);
};

recursiveCountdown(5);
```

:::

Wenn wir diese Funktion nun aufrufen und die Zahl 5 übergeben, sehen wir die Zahl in unserem Terminal ausgegeben. Aber es passiert sonst nichts – und die Zahl 5 ist sicherlich kein Countdown.

Bevor wir den rekursiven Teil unserer Funktion erstellen, müssen wir zuerst unseren Basisfall festlegen. Wenn Sie keinen Basisfall definieren, läuft Ihr Code so lange, bis der Speicher überschritten wird und er abstürzt.

:::interactive_editor

```js
const recursiveCountdown = (number) => {
  if (number < 1) {
    return;
  }
  console.log(number);
};

recursiveCountdown(5);
```

:::

Für unseren Basisfall soll der Countdown stoppen, wenn die Zahl kleiner als 1 ist. Wenn wir diesen Basisfall erreichen, können wir mit return die Ausführung der Funktion abbrechen.

Jetzt, wo wir sicher einen Basisfall vorbereitet haben, können wir die Rekursion einrichten. Der entscheidende Punkt, der eine Funktion rekursiv macht, ist, dass sie sich selbst während der Ausführung aufruft. In diesem Fall wollen wir die Funktion nach der Ausgabe der Zahl aufrufen. Aber um herunterzuzählen, muss unsere neue Zahl um eins kleiner sein:

:::interactive_editor

```js
const recursiveCountdown = (number) => {
  if (number < 1) {
    return;
  }
  console.log(number);
  recursiveCountdown(number - 1);
};

recursiveCountdown(5); 
```

:::

Das würde die Zahlen 5, 4, 3, 2 und 1 in der Konsole ausgeben.

Wir erhalten also unsere fünf Zahlen! Aber was, wenn wir stattdessen hochzählen wollen? Anstatt eine komplett neue Funktion zu schreiben, können wir die Reihenfolge unserer Ausgabe und unseres rekursiven Aufrufs tauschen:

:::interactive_editor

```js
const recursiveCountdown = (number) => {
    if (number < 1) {
        return;
    }
    recursiveCountdown(number - 1);
    console.log(number);
  };

recursiveCountdown(5);
```

:::

Das würde die Zahlen 1, 2, 3, 4 und 5 in der Konsole ausgeben.

Aber warum funktioniert das? Um das zu verstehen, müssen Sie den Aufrufstapel verstehen. Der Aufrufstapel ist die Art und Weise, wie JavaScript Funktionsaufrufe verfolgt und auflöst. Der Stapel funktioniert wie eine Art Last-in-first-out-Warteschlange. Um das besser zu verstehen, fügen wir unserer Funktion einige Ausgaben hinzu:

:::interactive_editor

```js
const recursiveCountdown = (number) => {
  console.log(`Function execution started for number: ${number}`);
  if (number < 1) {
    console.log(`Base case reached, begin resolving stack`);
    return;
  }
  console.log(`Calling recursiveCountdown with number: ${number - 1}`);
  recursiveCountdown(number - 1);
  console.log(`Function execution completed for number: ${number}`);
};

recursiveCountdown(5);
```

:::

Wir haben hier vier wichtige Ausgaben hinzugefügt. Die erste Ausgabe läuft, wenn ein Funktionsaufruf beginnt. Die dritte Ausgabe läuft kurz bevor die rekursive Funktion aufgerufen wird. Und die vierte Ausgabe läuft, wenn die Ausführung der Funktion beendet ist. Das Ergebnis ist:

```md
Function execution started for number: 5
Calling recursiveCountdown with number: 4
Function execution started for number: 4
Calling recursiveCountdown with number: 3
Function execution started for number: 3
Calling recursiveCountdown with number: 2
Function execution started for number: 2
Calling recursiveCountdown with number: 1
Function execution started for number: 1
Calling recursiveCountdown with number: 0
Function execution started for number: 0
Base case reached, begin resolving stack
Function execution completed for number: 1
Function execution completed for number: 2
Function execution completed for number: 3
Function execution completed for number: 4
Function execution completed for number: 5
```

Aber wie passiert das? Hier kommt der Aufrufstapel ins Spiel. Wenn wir `recursiveCountdown(5)` aufrufen, wird dieser Funktionsaufruf zum Aufrufstapel hinzugefügt.

Wenn dieser Funktionsaufruf an den Punkt kommt, an dem er `recursiveCountdown(4)` aufrufen muss, muss er anhalten und auf das Ergebnis warten. In der Zwischenzeit wird unser `recursiveCountdown(4)` oben auf den Aufrufstapel gelegt, über `recursiveCountdown(5)`.

Wenn dieser Funktionsaufruf an den Punkt kommt, an dem er `recursiveCountdown(3)` aufrufen muss, muss er anhalten und auf das Ergebnis warten. In der Zwischenzeit wird unser `recursiveCountdown(3)` oben auf den Aufrufstapel gelegt, über `recursiveCountdown(4)`.

Wenn dieser Funktionsaufruf an den Punkt kommt, an dem er `recursiveCountdown(2)` aufrufen muss, muss er anhalten und auf das Ergebnis warten. In der Zwischenzeit wird unser `recursiveCountdown(2)` oben auf den Aufrufstapel gelegt, über `recursiveCountdown(3)`.

Wenn dieser Funktionsaufruf an den Punkt kommt, an dem er `recursiveCountdown(1)` aufrufen muss, muss er anhalten und auf das Ergebnis warten. In der Zwischenzeit wird unser `recursiveCountdown(1)` oben auf den Aufrufstapel gelegt, über `recursiveCountdown(2)`.

Und schließlich, wenn dieser Funktionsaufruf an den Punkt kommt, an dem er `recursiveCountdown(0)` aufrufen muss, muss er anhalten und auf das Ergebnis warten. In der Zwischenzeit wird unser `recursiveCountdown(0)` oben auf den Aufrufstapel gelegt, über `recursiveCountdown(1)`.

Aber `recursiveCountdown(0)` ruft keine weitere Funktion auf – er trifft auf unseren Basisfall, bei dem er frühzeitig zurückkehrt. Da die Ausführung dieser Funktion beendet ist, kann der Funktionsaufruf als „aufgelöst“ betrachtet werden. Wenn der Aufruf aufgelöst ist, wird er vom Stapel entfernt.

Jetzt wartet unser `recursiveCountdown(1)` nicht mehr auf diesen Aufruf – er steht ganz oben auf dem Stapel und kann die Ausführung fortsetzen.

`recursiveCountdown(1)` wird aufgelöst, vom Stapel entfernt und ermöglicht `recursiveCountdown(2)` die Fortsetzung der Ausführung.

`recursiveCountdown(2)` wird aufgelöst, vom Stapel entfernt und ermöglicht `recursiveCountdown(3)` die Fortsetzung der Ausführung.

`recursiveCountdown(3)` wird aufgelöst, vom Stapel entfernt und ermöglicht `recursiveCountdown(4)` die Fortsetzung der Ausführung.

`recursiveCountdown(4)` wird aufgelöst, vom Stapel entfernt und ermöglicht `recursiveCountdown(5)` die Fortsetzung der Ausführung.

Und `recursiveCountdown(5)` wird aufgelöst und vom Stapel entfernt. Unser Aufrufstapel ist jetzt leer, also ist die Rekursion abgeschlossen!

Dies ist eine grundlegende Übersicht darüber, wie Rekursion in JavaScript funktioniert. Es ist ein komplexes Konzept, und Sie sollten mit etwas Code und Ausgaben experimentieren, bis Sie sich mit dem Verhalten des Aufrufstapels wohlfühlen.

Als kleine interessante Information: Wir haben darüber gesprochen, dass das Fehlen eines Basisfalls dazu führen kann, dass Ihr Code abstürzt, wenn der Speicher ausgeht. Das liegt daran, dass die Rekursion immer mehr Funktionsaufrufe in den Aufrufstapel legt, bis der Stapel überläuft. Genau wie der Name der beliebten Programmier-Community.
