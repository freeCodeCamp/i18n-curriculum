## Що таке Python?

- **Вступ**: Python — це мова програмування загального призначення, відома своєю простотою та легкістю у використанні. Python застосовується у багатьох сферах, таких як наука про дані та машинне навчання, веброзробка, скриптування та автоматизація, вбудовані системи та IoT, і багато іншого.
- **Поширені випадки використання**: Python використовується у науці про дані, машинному навчанні, веброзробці, кібербезпеці, автоматизації та на мікрокомп’ютерах, таких як Raspberry Pi та плати, сумісні з MicroPython.

## Змінні

- **Оголошення змінних**: Щоб оголосити змінну, починайте з імені змінної, потім оператор присвоєння (`=`) і далі значення. Це може бути число, рядок, булевий тип тощо. Ось кілька прикладів:

```py
name = 'John Doe'
age = 25
```

- **Конвенції іменування змінних**: Ось правила іменування змінних, яких слід дотримуватися:

  - Імена змінних можуть починатися лише з літери або підкреслення (_), а не з числа.
  - Імена змінних можуть містити лише алфавітно-цифрові символи (a-z, A-Z, 0-9) та підкреслення (_).
  - Імена змінних чутливі до регістру — `age`, `Age` і `AGE` вважаються різними.
  - Імена змінних не можуть бути зарезервованими ключовими словами Python, такими як `if`, `class` або `def`.
  - Імена змінних з кількома словами розділяються підкресленнями. Наприклад, `snake_case`.


## Коментарі

- **Однорядкові коментарі**: Цей тип коментарів використовуйте для коротких нотаток у коді.

```py

```

- **Багаторядкові рядки**: Цей тип рядків можна використовувати для більших нотаток або щоб перетворити частини коду на коментарі.

```py
"""

Here is some code commented out.

```

```py
print('Hello world!') # Hello world!
```

## Поширені типи даних у Python

- **Вступ**: Python — це мова з динамічною типізацією, як і JavaScript, тобто вам не потрібно явно оголошувати типи змінних. Мова визначає тип змінної на основі призначеного їй значення.
- **Ціле число**: Ціле число без десяткових знаків:

```py
my_integer_var = 10
print('Integer:', my_integer_var) # Integer: 10
```

- **Число з плаваючою крапкою**: Число з десятковими знаками:

```py
my_float_var = 4.50
print('Float:', my_float_var) # Float: 4.5
```

- **Рядок**: Послідовність символів, укладена в лапки:

```py
my_string_var = 'hello'
print('String:', my_string_var) # String: hello
```

- **Булевий тип**: Значення, що представляє `True` або `False`:

```py
my_boolean_var = True
print('Boolean:', my_boolean_var) # Boolean: True
```

- **Множина**: Неупорядкована колекція унікальних елементів:

```py
my_set_var = {7, 5, 8}
print('Set:', my_set_var) # Set: {7, 5, 8}
```

- **Словник**: Колекція пар ключ-значення, укладена у фігурні дужки:

```py
my_dictionary_var = {"name": "Alice", "age": 25}
print('Dictionary:', my_dictionary_var) # Dictionary: {'name': 'Alice', 'age': 25}
```

- **Кортеж**: Незмінна впорядкована колекція, укладена в круглі дужки:

```py
my_tuple_var = (7, 5, 8)
print('Tuple:', my_tuple_var) # Tuple: (7, 5, 8)
```

- **Діапазон**: Послідовність чисел, часто використовується в циклах:

```py
my_range_var = range(5)
print(my_range_var) # range(0, 5)
```

- **Список**: Впорядкована колекція елементів, що підтримує різні типи даних:

```py
my_list = [22, 'Hello world', 3.14, True]
print(my_list) # [22, 'Hello world', 3.14, True]
```

- **None**: Спеціальне значення, що означає відсутність значення:

```py
my_none_var = None
print('None:', my_none_var) # None: None
```

## Незмінні та змінні типи

- **Незмінні типи**: Ці типи не можна змінювати після оголошення, хоча можна переназначити змінну на інше значення. До них належать ціле число, число з плаваючою крапкою, комплексне число, булевий, рядок, кортеж, діапазон і `None`.
- **Змінні типи**: Ці типи можна змінювати після оголошення. Ви можете додавати, видаляти або оновлювати їхні елементи. До них належать колекції, такі як список, множина і словник.
- **Функція `type()`**: Щоб побачити тип змінної, можна використати функцію `type()` так:

```py
greeting = 'Hello there!'
age = 21

print(type(greeting)) # <class 'str'>
print(type(age)) # <class 'int'>
```

- **Функція `isinstance()`**: Використовується для перевірки, чи відповідає змінна певному типу даних:

```py
greeting = 'Hello world'
name = 'John Doe'

print(isinstance(greeting, str)) # True
print(isinstance(name, int)) # False
```

## Робота з рядками

- **Визначення**: Як ви пам’ятаєте з JavaScript, рядки незмінні, тобто їх не можна змінити після створення. У Python можна використовувати як одинарні, так і подвійні лапки. Рекомендується вибрати один стиль і дотримуватися його:

```py
developer = 'Jessica'
city = "Los Angeles"
```

- **Доступ до символів рядка**: Ви можете отримати символи рядка за допомогою дужкової нотації так:

```py
my_str = 'Hello world'

print(my_str[0])  # H
print(my_str[6])  # w

print(my_str[-1])  # d
print(my_str[-2]) # l
```

- **Екранізація рядків**: Якщо рядок містить лапки, можна використати зворотну скісну риску (`\`) так:

```py
msg = 'It\'s a sunny day'
quote = "She said, \"Hello!\""
```

- **Конкатенація рядків**: Щоб об’єднати рядки, можна використати оператор `+` так:

```py
developer = 'Jessica'
print('My name is ' + developer + '.') # My name is Jessica.
```

Інший спосіб конкатенації — оператор `+=`, який виконує конкатенацію і присвоєння одночасно:

```py
greeting = 'My name is '
developer = 'Jessica.'

greeting += developer
print(greeting) # My name is Jessica.
```

- **`f-strings`**: Це скорочення для форматованих рядкових літералів. Вони дозволяють робити інтерполяцію та конкатенацію з компактним і читабельним синтаксисом:

```py
developer = 'Jessica'
greeting = f'My name is {developer}.'
print(greeting) # My name is Jessica.
```

- **Вирізання рядків (slicing)**: Це коли ви можете витягти частину рядка. Ось базовий синтаксис:

```py
str[start:stop:step]
```

Початкова позиція — це індекс, з якого починається вирізання. Кінцева позиція — де вирізання закінчується, ця позиція не включається. Крок — інтервал, з яким відбувається вирізання. Ось кілька прикладів:

```py
message = 'Python is fun!'

print(message[0:6])  # Python
print(message[7:])  # is fun!
print(message[::2])  # Pto sfn
```

- **Отримання довжини рядка**: Функція `len()` повертає кількість символів у рядку:

```py
developer = 'Jessica'

print(len(developer)) # 7
```

## Робота з оператором `in`

- **Оператор `in`**: Повертає булеве значення, яке вказує, чи існує символ або символи у рядку:

```py
my_str = 'Hello world'

print('Hello' in my_str)  # True
print('hey' in my_str)    # False
print('hi' in my_str)    # False
print('e' in my_str)  # True
print('f' in my_str)  # False
```

## Поширені методи рядків

- **`upper()`**: Цей метод рядка повертає новий рядок, у якому всі символи перетворені на верхній регістр:

```py
developer = 'Jessica'

print(developer.upper()) # JESSICA
```

- **`lower()`**: Цей метод рядка повертає новий рядок, у якому всі символи перетворені на нижній регістр:

```py
developer = 'Jessica'

print(developer.lower()) # jessica
```

- **`strip()`**: Цей метод повертає копію рядка з видаленими вказаними провідними та кінцевими символами (якщо аргумент не передано, видаляє пробіли на початку і в кінці).

```py
greeting = '  hello world  '

trimmed_my_str = greeting.strip()
print(trimmed_my_str)  # 'hello world'
```

- **`replace()`**: Повертає новий рядок, у якому всі входження старого рядка замінені на новий.

```py
greeting = 'hello world'

replaced_my_str = greeting.replace('hello', 'hi')
print(replaced_my_str)  # 'hi world'
```

- **`split()`**: Використовується для розбиття рядка на список за вказаним роздільником. Роздільник — це рядок, що вказує, де відбувається розбиття.

```py
dashed_name = 'example-dashed-name'

split_words = dashed_name.split('-')
print(split_words)  # ['example', 'dashed', 'name']
```

- **`join()`**: Використовується для об’єднання елементів ітерованого об’єкта в рядок із роздільником. Ітерований об’єкт — це колекція елементів, по якій можна ітерувати, наприклад список, рядок або кортеж.

```py
example_list = ['example', 'dashed', 'name']

joined_str = ' '.join(example_list)
print(joined_str)  # example dashed name
```

- **`startswith(prefix)`**: Цей метод рядка повертає булеве значення, яке вказує, чи починається рядок із вказаного префікса:

```py
developer = 'Naomi'

result = developer.startswith('N')
print(result)  # True
```

- **`endswith(suffix)`**: Цей метод рядка повертає булеве значення, яке вказує, чи закінчується рядок вказаним суфіксом:

```py
developer = 'Naomi'

result = developer.endswith('N')
print(result)  # False
```

- **`find()`**: Цей метод рядка повертає індекс першого входження підрядка. Якщо підрядок не знайдено, повертається `-1`:

```py
developer = 'Naomi'

result = developer.find('N')
print(result)  # 0

city = 'Los Angeles'
print(city.find('New')) # -1
```

- **`count(substring)`**: Цей метод рядка рахує, скільки разів підрядок зустрічається у рядку:

```py
city = 'Los Angeles'
print(city.count('e')) # 2
```

- **`capitalize()`**: Цей метод рядка повертає новий рядок, у якому перший символ — великий, а всі інші — малі:

```py
dessert = 'chocolate cake'
print(dessert.capitalize()) # Chocolate cake
```

- **`isupper()`**: Цей метод рядка повертає `True`, якщо всі літери у рядку — великі, і `False`, якщо ні:

```py
dessert = 'chocolate cake'
print(dessert.isupper()) # False
```

- **`islower()`**: Цей метод рядка повертає `True`, якщо всі літери у рядку — малі, і `False`, якщо ні:

```py
dessert = 'chocolate cake'
print(dessert.islower()) # True
```

- **`title()`**: Цей метод рядка повертає новий рядок, у якому перша літера кожного слова — велика:

```py
city = 'los angeles'
print(city.title()) # Los Angeles
```

- **`maketrans()`**: Цей метод рядка створює таблицю відповідностей символів для трансляції. Зазвичай використовується разом із методом `translate()`, який застосовує цю таблицю до рядка і повертає трансформований результат.

```py
trans_table = str.maketrans('abc', '123')
print(trans_table) # {97: 49, 98: 50, 99: 51}

result = 'abcabc'.translate(trans_table)
print(result)  # 123123
```

## Поширені операції з цілими числами та числами з плаваючою крапкою

- **Основні математичні операції**: У Python можна виконувати базові математичні операції з цілими числами та числами з плаваючою крапкою: додавання, віднімання, множення та ділення:

```py
int_1 = 56
int_2 = 12
float_1 = 5.4
float_2 = 12.0

# Addition

print('Integer Addition:', int_1 + int_2) # Integer Addition: 68
print('Float Addition:', float_1 + float_2) # Float Addition: 17.4

# Subtraction

print('Int Subtraction:', int_1 - int_2) # Int Subtraction: 44
print('Float Subtraction:',  float_2 - float_1) # Float Subtraction: 6.6

# Multiplication

print('Int Multiplication:', int_1 * int_2) # Int Multiplication: 672
print('Float Multiplication:', float_2 * float_1) # Float Multiplication: 64.80000000000001

# Division

print('Int Division:', int_1 / int_2) # Int Division: 4.666666666666667
print('Float Division:', float_2 / float_1) # Float Division: 2.222222222222222
```

Якщо додати число з плаваючою крапкою і ціле число, результат буде числом з плаваючою крапкою:

```py
int_1 = 56
float_1 = 5.4

print(int_1 + float_1) # 61.4
```

- **Оператор взяття остачі (`%`)**: Повертає остачу від ділення одного числа на інше:

```py
int_1 = 56
int_2 = 12

print(int_1 % int_2) # 8
```

- **Цілочисельне ділення (`//`)**: Ділить два числа і округлює результат вниз до найближчого цілого:

```py
int_1 = 56
int_2 = 12

print(int_1 // int_2) # 4
```

- **Оператор піднесення до степеня (`**`)**: Підносить число до степеня іншого числа:

```py
int_1 = 4
int_2 = 2

print(int_1 ** int_2) # 16
```

- **Функція `float()`**: Використовується для перетворення цілого числа на число з плаваючою крапкою.

```py
num = 4

print(float(num)) # 4.0
```

- **Функція `int()`**: Використовується для перетворення числа з плаваючою крапкою на ціле число.

```py
num = 4.0

print(int(num)) # 4
```

- **Функція `round()`**: Використовується для округлення числа до найближчого цілого:

```py
num_1 = 3.4
num_2 = 7.7

print(round(num_1)) # 3
print(round(num_2)) # 8
```

- **Функція `abs()`**: Використовується для отримання абсолютного значення числа:

```py
num = -13

print(abs(num)) # 13
```

- **Функція `bin()`**: Використовується для перетворення цілого числа у двійковий рядок:

```py
num = 56

print(bin(num))  # 0b111000
```

- **Функція `oct()`**: Використовується для перетворення цілого числа в вісімковий рядок:

```py
num = 56

print(oct(num))  # 0o70
```

- **Функція `hex()`**: Використовується для перетворення цілого числа в шістнадцятковий рядок:

```py
num = 56

print(hex(num))  # 0x38
```

- **Функція `pow()`**: Використовується для піднесення числа до степеня іншого числа:

```py
result = pow(2, 3) 
print(result)  # 8
```

## Розширені присвоєння

- **Визначення**: Розширене присвоєння поєднує бінарну операцію з присвоєнням в одному кроці. Воно бере змінну, застосовує до неї операцію з іншим значенням і зберігає результат у тій самій змінній.

```py
# Addition assignment 
my_var = 10
my_var += 5

print(my_var) # 15

# Subtraction assignment
count = 14
count -= 3

print(count) # 11

# Multiplication assignment 
product = 65
product *= 7

print(product) # 455

# Division assignment 
price = 100
price /= 4

print(price) # 25.0

# Floor Division assignment 
total_pages = 23
total_pages //= 5

print(total_pages) # 4

# Modulo assignment 
bits = 35
bits %= 2

print(bits) # 1

# Exponentiation assignment 
power = 2
power **= 3

print(power) # 8
```

Існують й інші оператори розширеного присвоєння, наприклад для побітових операцій: `&=`, `^=`, `>>=` і `<<=`.

## Робота з функціями

- **Визначення**: Функції — це повторно використовувані частини коду, які приймають вхідні дані (аргументи) і повертають результат. Щоб викликати функцію, потрібно звернутися до її імені, а потім поставити круглі дужки:

```py
# Defining a function

def get_sum(num_1, num_2):
    return num_1 + num_2

result = get_sum(3, 4) # function call
print(result) # 7
```

Якщо функція явно не повертає значення, то за замовчуванням повертається `None`:

```py
def greet():
    print('hello') 

result = greet() # hello
print(result) # None
```

Також можна задавати значення параметрів за замовчуванням так:

```py
def get_sum(num_1, num_2=2):
    return num_1 + num_2

result = get_sum(3) 
print(result) # 5
```

Якщо викликати функцію з неправильним числом аргументів, отримаєте `TypeError`:

```py
def calculate_sum(a, b):
    print(a + b)

calculate_sum()

# TypeError: calculate_sum() missing 2 required positional arguments: 'a' and 'b'
```

name = input('What is your name?') # User types 'Kolade' and presses Enter  
print('Hello', name) # Hello Kolade

```py
print(int(3.14)) # 3
print(int('42')) # 42
print(int(True)) # 1
print(int(False)) # 0 
```

## Область видимості в Python

- **Локальна область видимості**: Змінна, оголошена всередині функції або класу, доступна лише в межах цієї функції або класу.

```py
def my_func():
    num = 10
    print(num)
```

- **Огортаюча область видимості**: Функція, вкладена в іншу функцію, може отримати доступ до змінних зовнішньої функції.

```py
def outer_func():
    msg = 'Hello there!'
    
    def inner_func():
        print(msg)
    inner_func()

print(outer_func()) # Hello there!
```

- **Глобальна область видимості**: Змінні, оголошені поза будь-якими функціями чи класами, доступні з будь-якої частини програми.

```py
tax = 0.70 

def get_total(subtotal):
    total = subtotal + (subtotal * tax)
    return total

print(get_total(100))  # 170.0
```

- **Вбудована область видимості**: Зарезервовані імена Python для вбудованих функцій, модулів, ключових слів і об’єктів.

```py
print(str(45)) # '45'
print(type(3.14)) # <class 'float'>
print(isinstance(3, str)) # False
```

## Оператори порівняння

- **Рівність (`==`)**: Перевіряє, чи два значення рівні:

```py
print(3 == 4) # False
```

- **Нерівність (`!=`)**: Перевіряє, чи два значення не рівні:

```py
print(3 != 4) # True
```

- **Строго більше (`>`)**: Перевіряє, чи одне значення більше за інше:

```py
print(3 > 4) # False
```

- **Строго менше (`<`)**: Перевіряє, чи одне значення менше за інше:

```py
print(3 < 4) # True
```

- **Більше або дорівнює (`>=`)**: Перевіряє, чи одне значення більше або дорівнює іншому:

```py
print(3 >= 4) # False
```

- **Менше або дорівнює (`<=`)**: Перевіряє, чи одне значення менше або дорівнює іншому:

```py
print(3 <= 4) # True
```

## Робота з інструкціями `if`, `elif` і `else`

- **Інструкції `if`**: Умови, які визначають, чи є щось правдивим. Якщо умова оцінюється як `True`, виконується відповідний блок коду.

```py
age = 18

if age >= 18:
    print('You are an adult') # You are an adult
```

- **Інструкція `elif`**: Умови, що йдуть після інструкції `if`. Якщо умова `elif` оцінюється як `True`, виконується відповідний блок коду.

```py
age = 16

if age >= 18:
    print('You are an adult')
elif age >= 13:
    print('You are a teenager')  # You are a teenager
```

- **Умова `else`**: Виконується, якщо жодна інша умова не оцінюється як `True`.

```py
age = 12

if age >= 18:
    print('You are an adult')
elif age >= 13:
    print('You are a teenager')
else:
    print('You are a child')  # You are a child
```

Також можна використовувати вкладені інструкції `if` так:

```py
is_citizen = True
age = 25

if is_citizen:
    if age >= 18:
        print('You are eligible to vote') # You are eligible to vote
else:
    print('You are not eligible to vote')
```

## Правдиві та хибні значення

- **Визначення**: У Python кожне значення має вбудоване булеве значення, яке визначає, чи слід його вважати `True` або `False` у логічному контексті. Багато значень вважаються правдивими (truthy), тобто оцінюються як `True`. Інші — хибними (falsy), тобто оцінюються як `False`. Ось кілька прикладів хибних значень:

```md
None
False
Integer 0
Float 0.0
Empty strings ''
```

Інші значення, як-от ненульові числа та непорожні рядки, є правдивими.

## Робота з функцією `bool()`

- **Визначення**: Щоб перевірити, чи є значення правдивим або хибним, можна використати вбудовану функцію `bool()`. Вона явно перетворює значення на булевий еквівалент і повертає `True` для правдивих значень і `False` для хибних. Ось кілька прикладів:

```py
print(bool(False)) # False
print(bool(0))  # False
print(bool('')) # False

print(bool(True)) # True
print(bool(1)) # True
print(bool('Hello')) # True
```

## Булеві оператори та коротке замикання

- **Визначення**: Це спеціальні оператори, які дозволяють комбінувати кілька виразів для створення складнішої логіки прийняття рішень у коді. У Python є три булеві оператори: `and`, `or` і `not`.
- **Оператор `and`**: Приймає два операнди і повертає перший, якщо він хибний, інакше — другий. Обидва операнди мають бути правдивими, щоб вираз був правдивим.

```py
is_citizen = True
age = 25

print(is_citizen and age) # 25
```

Також можна використовувати оператор `and` в умовах так:

```py
is_citizen = True
age = 25

if is_citizen and age >= 18:
    print('You are eligible to vote') # You are eligible to vote
else:
    print('You are not eligible to vote')
```

- **Оператор `or`**: Повертає перший операнд, якщо він правдивий, інакше другий. Вираз `or` є правдивим, якщо хоча б один операнд правдивий. Ось приклад:

```py
age = 19
is_employed = False

print(age or is_employed) # 19
```

Як і з оператором `and`, можна використовувати оператор `or` в умовах так:

```py
age = 19
is_student = True

if age < 18 or is_student:
    print('You are eligible for a student discount') # You are eligible for a student discount
else:
    print('You are not eligible for a student discount')
```

- **Коротке замикання**: Оператори `and` і `or` називають операторами короткого замикання. Це означає, що Python перевіряє значення зліва направо і зупиняється, щойно визначає кінцевий результат.
- **Оператор `not`**: Приймає один операнд і інвертує його булеве значення. Перетворює правдиві значення на `False` і хибні на `True`. На відміну від попередніх операторів, `not` завжди повертає `True` або `False`. Ось кілька прикладів:

```py
print(not '') # True, because empty string is falsy
print(not 'Hello') # False, because non-empty string is truthy
print(not 0) # True, because 0 is falsy
print(not 1) # False, because 1 is truthy
print(not False) # True, because False is falsy
print(not True) # False, because True is truthy
```

Ось приклад використання оператора `not` в умові:

```py
is_admin = False

if not is_admin:
    print('Access denied for non-administrators.') # Access denied for non-administrators.
else:
    print('Welcome, Administrator!')
```

## Списки в Python

- **Вступ**: У Python тип даних список — це впорядкована послідовність елементів, які можуть бути рядками, числами або навіть іншими списками. Списки змінні і мають індексацію, починаючи з нуля.

```python
cities = ['Los Angeles', 'London', 'Tokyo']
```

- **Доступ до елементів списку**: Щоб отримати елемент зі списку `cities`, зверніться до його індексу в послідовності:

```python
cities = ['Los Angeles', 'London', 'Tokyo']
cities[0] # Los Angeles
```

- **Доступ до елементів за від’ємною індексацією**: Щоб отримати останній елемент будь-якого списку, можна використати `-1` як індекс:

```python
cities = ['Los Angeles', 'London', 'Tokyo']
cities[-1] # Tokyo
```

- Від’ємна індексація використовується для доступу до елементів, починаючи з кінця списку, а не з початку з індексу `0`.

- **Створення списків за допомогою конструктора `list()`**: Списки також можна створювати за допомогою конструктора `list()`. Конструктор `list()` використовується для перетворення ітерованого об’єкта у список:

```python
developer = 'Jessica'

print(list(developer)) 
# Result: ['J', 'e', 's', 's', 'i', 'c', 'a']
```

- **Знаходження довжини списку**: Для отримання довжини списку можна використати функцію `len()`:

```python
numbers = [1, 2, 3, 4, 5]
len(numbers) # 5
```

- **Змінність списків**: Списки змінні, тобто можна оновити будь-який елемент, якщо вказати дійсний індекс. Щоб оновити елемент за індексом, призначте нове значення цьому індексу:

```python
programming_languages = ['Python', 'Java', 'C++', 'Rust']
programming_languages[0] = 'JavaScript'
print(programming_languages) # ['JavaScript', 'Java', 'C++', 'Rust']
```

- **Помилка виходу за межі індексу**: Якщо передати індекс (позитивний або від’ємний), який виходить за межі списку, отримаєте `IndexError`:

```python
programming_languages = ['Python', 'Java', 'C++', 'Rust']
programming_languages[10] = 'JavaScript'

"""
Traceback (most recent call last):
  File "<stdin>", line 1, in <module>
IndexError: list assignment index out of range
"""
```

- **Видалення елементів зі списку**: Елементи можна видалити за допомогою ключового слова `del`:

```python
developer = ['Jane Doe', 23, 'Python Developer']
del developer[1]
print(developer) # ['Jane Doe', 'Python Developer']
```

- **Перевірка наявності елемента у списку**: Ключове слово `in` використовується для перевірки, чи існує елемент у списку:

```py
programming_languages = ['Python', 'Java', 'C++', 'Rust']

'Rust' in programming_languages # True
'JavaScript' in programming_languages # False
```

- **Вкладені списки**: Списки можуть бути вкладені в інші списки:

```python
developer = ['Alice', 25, ['Python', 'Rust', 'C++']]
```

- Щоб отримати вкладений список, зверніться до нього за індексом `2`, оскільки списки індексуються, починаючи з нуля.

```python
developer = ['Alice', 25, ['Python', 'Rust', 'C++']]
developer[2] # ['Python', 'Rust', 'C++']
```

- Щоб отримати другу мову з цього вкладеного списку, зверніться до неї за індексом `1`:

```python
developer = ['Alice', 25, ['Python', 'Rust', 'C++']]
developer[2][1] # Rust
```

- **Розпакування значень зі списку**: Розпакування — це техніка призначення значень зі списку новим змінним. Ось приклад розпакування списку `developer` у змінні `name`, `age` і `job`:

```py
developer = ['Alice', 34, 'Rust Developer']
name, age, job = developer
```

- Якщо кількість змінних зліва від оператора присвоєння не співпадає з кількістю елементів у списку, отримаєте `ValueError`.

- **Збір залишкових елементів зі списку**: Щоб зібрати залишкові елементи зі списку, використайте оператор зірочки (`*`) так:

```py
developer = ['Alice', 34, 'Rust Developer']
name, *rest = developer
```

- **Вирізання списків (slicing)**: Вирізання — це доступ до частини списку за допомогою оператора вирізання `:`. Щоб вирізати список, починаючи з індексу `1` і закінчуючи перед індексом `3`, використайте такий синтаксис:

```python
desserts = ['Cake', 'Cookies', 'Ice Cream', 'Pie']
desserts[1:3] # ['Cookies', 'Ice Cream']
```

- **Кроки інтервалу**: Можна також вказати крок, який визначає, наскільки збільшувати індекси. Ось приклад, як вирізати список лише з парними числами:

```python
numbers = [1, 2, 3, 4, 5, 6]
numbers[1::2] # [2, 4, 6]
```

## Методи списків

- **append()**: Використовується для додавання елемента в кінець списку. Ось приклад використання методу `append()` для додавання числа `6` у список `numbers`:

```py
numbers = [1, 2, 3, 4, 5]
numbers.append(6)
print(numbers) # [1, 2, 3, 4, 5, 6]
```

- **Додавання списків**: Метод `append()` також можна використати для додавання одного списку в кінець іншого:

```py
numbers = [1, 2, 3, 4, 5]
even_numbers = [6, 8, 10]

numbers.append(even_numbers)
print(numbers) # [1, 2, 3, 4, 5, [6, 8, 10]]
```

- **extend()**: Використовується для додавання кількох елементів у кінець списку. Ось приклад додавання чисел `6`, `8` і `10` у кінець списку `numbers`:

```py
numbers = [1, 2, 3, 4, 5]
even_numbers = [6, 8, 10]

numbers.extend(even_numbers)
print(numbers) # [1, 2, 3, 4, 5, 6, 8, 10]
```

- **insert()**: Використовується для вставки елемента за певним індексом у списку. Ось приклад використання методу `insert()`:

```py
numbers = [1, 2, 3, 4, 5]
numbers.insert(2, 2.5)

print(numbers) # [1, 2, 2.5, 3, 4, 5]
```

- **remove()**: Використовується для видалення елемента зі списку. Метод `remove()` видаляє лише перше входження елемента у списку:

```py
numbers = [1, 2, 3, 4, 5, 5, 5]
numbers.remove(5)

print(numbers) # [1, 2, 3, 4, 5, 5]

```

- **pop()**: Використовується для видалення певного елемента зі списку з поверненням цього елемента:

```py
numbers = [1, 2, 3, 4, 5]
numbers.pop(1) # The number 2 is returned
```

- Якщо не вказати елемент для методу `pop`, буде видалено останній елемент.

```py
numbers = [1, 2, 3, 4, 5]
numbers.pop() # The number 5 is returned
```

- **clear()**: Використовується для видалення всіх елементів зі списку:

```py
numbers = [1, 2, 3, 4, 5]
numbers.clear()

print(numbers) # []
```

- **sort()**: Метод `sort()` сортує елементи на місці. Ось приклад сортування випадкового списку `numbers`:

```py
numbers = [19, 2, 35, 1, 67, 41]
numbers.sort()

print(numbers) # [1, 2, 19, 35, 41, 67]
```

- **sorted()**: Вбудована функція, яка повертає новий відсортований список, не змінюючи оригінал:

```py
numbers = [19, 2, 35, 1, 67, 41]
sorted_numbers = sorted(numbers)

print(sorted_numbers) # [1, 2, 19, 35, 41, 67]
print(numbers) # [19, 2, 35, 1, 67, 41]
```

- **reverse()**: Використовується для обертання порядку елементів у списку:

```py
numbers = [6, 5, 4, 3, 2, 1]
numbers.reverse()

print(numbers) # [1, 2, 3, 4, 5, 6]
```

- **index()**: Використовується для пошуку першого індексу, де знаходиться елемент у списку:

```python
programming_languages = ['Rust', 'Java', 'Python', 'C++']
programming_languages.index('Java') # 1
```

- Якщо елемент не знайдено за допомогою методу `index()`, результатом буде `ValueError`.

## Кортежі в Python

- **Визначення**: Кортеж — це тип даних Python, який створює впорядковану послідовність значень. Кортежі можуть містити змішані типи даних:

```py
developer = ('Alice', 34, 'Rust Developer')
```

- Кортежі незмінні, тобто елементи не можна змінювати після створення. Якщо спробувати оновити елемент, отримаєте `TypeError`:

```py
programming_languages = ('Python', 'Java', 'C++', 'Rust')
programming_languages[0] = 'JavaScript'

"""
Traceback (most recent call last):
  File "<stdin>", line 1, in <module>
TypeError: "tuple" object does not support item assignment
"""
```

- **Доступ до елементів кортежу**: Щоб отримати елемент, використовуйте дужкову нотацію і індекс:

```py
developer = ('Alice', 34, 'Rust Developer')
developer[1] # 34
```

- Можна використовувати від’ємну індексацію для доступу з кінця кортежу:

```py
numbers = (1, 2, 3, 4, 5)
numbers[-2] # 4
```

- Якщо передати індекс, що перевищує або дорівнює довжині кортежу, отримаєте `IndexError`:

```py
numbers = (1, 2, 3, 4, 5)
numbers[7]

"""
Traceback (most recent call last):
  File "<stdin>", line 1, in <module>
IndexError: tuple index out of range
"""
```

- Кортеж можна створити за допомогою конструктора `tuple()`. У конструктор можна передати різні ітеровані об’єкти, як-от рядки, списки або інші кортежі.

```python
developer = 'Jessica'

print(tuple(developer)) 
# Result: ('J', 'e', 's', 's', 'i', 'c', 'a')
```

- **Перевірка наявності елемента в кортежі**: Щоб перевірити, чи є елемент у кортежі, використовуйте ключове слово `in` так:

```py
programming_languages = ('Python', 'Java', 'C++', 'Rust')

'Rust' in programming_languages # True
'JavaScript' in programming_languages # False
```

- **Розпакування кортежів**: Елементи можна розпакувати так:

```py
developer = ('Alice', 34, 'Rust Developer')
name, age, job = developer
```

- Щоб зібрати залишкові елементи з кортежу, використовуйте оператор зірочки (`*`) так:

```py
developer = ('Alice', 34, 'Rust Developer')
name, *rest = developer
```

- **Вирізання кортежів (slicing)**: Вирізання можна використовувати для отримання частини кортежу. Наприклад, елементи `pie` і `cookies` можна вирізати в окремий кортеж:

```py
desserts = ('cake', 'pie', 'cookies', 'ice cream')
desserts[1:3] # ('pie', 'cookies')
```

- **Видалення елементів з кортежів**: Видалення елемента з кортежу викличе `TypeError`, оскільки кортежі незмінні:

```py
developer = ('Jane Doe', 23, 'Python Developer')
del developer[1]

"""
Traceback (most recent call last):
  File "<stdin>", line 1, in <module>
TypeError: "tuple" object doesn't support item deletion
"""
```

- **Коли використовувати кортеж замість списку?**: Якщо потрібна динамічна колекція, де можна додавати, видаляти і оновлювати елементи, використовуйте список. Якщо колекція фіксована і незмінна, використовуйте кортеж.

## Поширені методи кортежів

- **`count()`**: Використовується для визначення, скільки разів елемент зустрічається в кортежі. Наприклад, можна перевірити, скільки разів мова `'Rust'` зустрічається в кортежі:

```py
programming_languages = ('Rust', 'Java', 'Python', 'C++', 'Rust')
programming_languages.count('Rust') # 2
```

- Якщо в методі `count()` вказаний елемент відсутній у кортежі, повертається `0`:

```py
programming_languages = ('Rust', 'Java', 'Python', 'C++', 'Rust')
programming_languages.count('JavaScript') # 0
```

- Якщо аргументи не передані методу `count()`, Python поверне `TypeError`.

- **index()**: Використовується для пошуку індексу, де знаходиться певний елемент у кортежі. Ось приклад використання методу `index()` для пошуку індексу мови `'Java'`:

```py
programming_languages = ('Rust', 'Java', 'Python', 'C++', 'Rust')
programming_languages.index('Java') # 1
```

- Якщо елемент не знайдено, Python поверне `ValueError`.

- Можна передати необов’язковий початковий індекс у метод `index()`, щоб вказати, з якого місця починати пошук:

```py
programming_languages = ('Rust', 'Java', 'Python', 'C++', 'Rust', 'Python')
programming_languages.index('Python', 3) # 5
```

- Можна також передати необов’язковий кінцевий індекс у метод `index()`, щоб вказати, де припинити пошук:

```py
programming_languages = ('Rust', 'Java', 'Python', 'C++', 'Rust', 'Python', 'JavaScript', 'Python')
programming_languages.index('Python', 2, 5) # 2
```

- **`sorted()`**: Використовується для сортування елементів будь-якого ітерованого об’єкта і повертає новий відсортований список. Ось приклад створення нового списку чисел за допомогою функції `sorted()`:

```py
numbers = (13, 2, 78, 3, 45, 67, 18, 7)
sorted(numbers) # [2, 3, 7, 13, 18, 45, 67, 78]
```

- **Зміна поведінки сортування**: Можна налаштувати поведінку сортування за допомогою необов’язкових аргументів `reverse` і `key`. Ось приклад використання аргументу `key` для сортування елементів кортежу за довжиною:

```py
programming_languages = ('Rust', 'Java', 'Python', 'C++', 'Rust', 'Python')
sorted(programming_languages, key=len)

# Result
# ['C++', 'Rust', 'Java', 'Rust', 'Python', 'Python']
```

- Можна створити новий список у зворотному порядку, використовуючи аргумент `reverse` так:

```py
programming_languages = ('Rust', 'Java', 'Python', 'C++', 'Rust', 'Python')

print(sorted(programming_languages, reverse=True))

# Result
# ['Rust', 'Rust', 'Python', 'Python', 'Java', 'C++']
```

## Цикли в Python

- **Визначення**: Цикли використовуються для повторення блоку коду певну кількість разів.

- **Цикл `for`**: Використовується для ітерації по послідовності (наприклад, списку, кортежу або рядку) і виконання блоку коду для кожного елемента. Ось приклад використання циклу `for` для ітерації списком і виведення кожної мови в консоль:

```py
programming_languages = ['Rust', 'Java', 'Python', 'C++']

for language in programming_languages:
    print(language)

"""
Result 

Rust
Java
Python
C++
"""
```

- Ось приклад використання циклу `for` для проходження рядка `code` і виведення кожного символу:

```py
for char in 'code':
    print(char)

"""
Result 

c
o
d
e
"""
```

- Цикли `for` можна вкладати. Ось приклад вкладеного циклу `for`:

```py
categories = ['Fruit', 'Vegetable']
foods = ['Apple', 'Carrot', 'Banana']

for category in categories:
    for food in foods:
        print(category, food)

"""
Result

Fruit Apple
Fruit Carrot
Fruit Banana
Vegetable Apple
Vegetable Carrot
Vegetable Banana
"""

```

- **Цикл `while`**: Повторює блок коду, доки умова не стане `False`. Ось приклад використання циклу `while` для гри в угадайку:

```py
secret_number = 3
guess = 0

while guess != secret_number:
    guess = int(input('Guess the number (1-5): '))
    if guess != secret_number:
        print('Wrong! Try again.')

print('You got it!')

"""
Result

Guess the number (1-5): 2
Wrong! Try again.
Guess the number (1-5): 1
Wrong! Try again.

```

- **Інструкції `break` і `continue`**: Використовуються в циклах для зміни виконання циклу.

- Інструкція `break` використовується для негайного виходу з циклу, коли виконується певна умова. Ось приклад використання інструкції `break` для списку `developer_names`:

```py
developer_names = ['Jess', 'Naomi', 'Tom']

for developer in developer_names:
    if developer == 'Naomi':
        break
    print(developer)
```

- Інструкція `continue` використовується для пропуску поточної ітерації і переходу до наступної. Ось приклад використання інструкції `continue` замість `break`:

```py
developer_names = ['Jess', 'Naomi', 'Tom']

for developer in developer_names:
    if developer == 'Naomi':
        continue
    print(developer)
```

- Цикли `for` і `while` можна поєднувати з інструкцією `else`, яка виконується лише якщо цикл не був завершений інструкцією `break`:

```python
words = ['sky', 'apple', 'rhythm', 'fly', 'orange']

for word in words:
    for letter in word:
        if letter.lower() in 'aeiou':
            print(f"'{word}' contains the vowel '{letter}'")
            break
    else:
        print(f"'{word}' has no vowels")
```

## Діапазони та їх використання в циклах

- **Функція `range()`**: Використовується для генерації послідовності цілих чисел.

```py
range(start, stop, step)
```

- Обов’язковий аргумент `stop` — ціле число (не включно), що позначає кінцеву точку послідовності. Ось приклад використання функції `range()`:

```py
for num in range(3):
    print(num)
```

- Якщо аргумент `start` не вказано, за замовчуванням він дорівнює `0`. За замовчуванням послідовність збільшується на `1`. Можна використати необов’язковий аргумент `step`, щоб змінити крок. Ось приклад генерації послідовності парних чисел від 2 до 10 включно:

```py
for num in range(2, 11, 2):
    print(num)
```

- Якщо не передати аргументи функції `range()`, отримаєте `TypeError`.

- Функція `range()` приймає лише цілі числа, а не числа з плаваючою крапкою. Використання чисел з плаваючою крапкою призведе до `TypeError`:

```py
ERROR!
Traceback (most recent call last):
  File "<main.py>", line 1, in <module>
TypeError: 'float' object cannot be interpreted as an integer
```

- Можна використати від’ємне ціле число для аргументу `step`, щоб згенерувати послідовність у спадному порядку:

```py
for num in range(40, 0, -10):
    print(num)
```

- Функцію `range()` також можна використати для створення списку цілих чисел разом із конструктором `list`. Конструктор `list` перетворює ітерований об’єкт у список. Ось приклад генерації списку парних чисел від 2 до 10 включно:

```py
numbers = list(range(2, 11, 2))
print(numbers) # [2, 4, 6, 8, 10]
```

## Функції `enumerate()` і `zip()` у Python

- **`enumerate()`**: Використовується для ітерації по послідовності з одночасним відстеженням індексу кожного елемента. Функція `enumerate()` приймає ітерований об’єкт і повертає об’єкт `enumerate`, що містить індекс і значення кожного елемента.

```python
languages = ['Spanish', 'English', 'Russian', 'Chinese']

for index, language in enumerate(languages):
    print(f'Index {index} and language {language}')

# Result
# Index 0 and language Spanish
# Index 1 and language English
# Index 2 and language Russian
# Index 3 and language Chinese

```

- Функцію `enumerate()` можна також використовувати поза циклом `for`:

```python
languages = ['Spanish', 'English', 'Russian', 'Chinese']

print(list(enumerate(languages)))
# [(0, 'Spanish'), (1, 'English'), (2, 'Russian'), (3, 'Chinese')]
```

- Функція `enumerate()` приймає необов’язковий аргумент `start`, який задає початкове значення лічильника. Якщо аргумент пропущено, лічильник починається з `0`.

- **`zip()`**: Використовується для ітерації одночасно по кількох ітерованих об’єктах. Ось приклад використання функції `zip()` для ітерації по `developers` і `ids`:

```python
developers = ['Naomi', 'Dario', 'Jessica', 'Tom']
ids = [1, 2, 3, 4]

for name, id in zip(developers, ids):
    print(f'Name: {name}')
    print(f'ID: {id}')


"""
Result

Name: Naomi
ID: 1
Name: Dario
ID: 2
Name: Jessica
ID: 3
Name: Tom
ID: 4
"""

```

## Генератори списків у Python

- **Визначення**: Генератор списку дозволяє створити новий список в одному рядку, поєднуючи цикл і умову безпосередньо в квадратних дужках. Це робить код коротшим і часто легшим для читання.

```py
even_numbers = [num for num in range(21) if num % 2 == 0]
print(even_numbers)
```

## Функції для роботи з ітерованими об’єктами

- **`filter()`**: Використовується для фільтрації елементів ітерованого об’єкта за умовою. Повертає ітератор, що містить лише елементи, які задовольняють умову. Ось приклад створення нового списку слів довжиною більше чотирьох символів:

```py
words = ['tree', 'sky', 'mountain', 'river', 'cloud', 'sun']

def is_long_word(word):
    return len(word) > 4

long_words = list(filter(is_long_word, words))
print(long_words) # ['mountain', 'river', 'cloud']
```

- **`map()`**: Використовується для застосування функції до кожного елемента ітерованого об’єкта і повертає новий ітератор з результатами. Ось приклад використання функції `map()` для перетворення списку температур у Цельсіях у Фаренгейти:

```py
celsius = [0, 10, 20, 30, 40]

def to_fahrenheit(temp):
    return (temp * 9/5) + 32

fahrenheit = list(map(to_fahrenheit, celsius))
print(fahrenheit) # [32.0, 50.0, 68.0, 86.0, 104.0]
```

- **`sum()`**: Використовується для отримання суми елементів ітерованого об’єкта, наприклад списку або кортежу. Ось приклад використання функції `sum()`:

```py
numbers = [5, 10, 15, 20]
total = sum(numbers)
print(total) # Result: 50
```

- Можна також передати необов’язковий аргумент `start`, який задає початкове значення суми. Ось оновлений приклад із використанням позиційного аргументу `start`:

```py
numbers = [5, 10, 15, 20]
total = sum(numbers, 10) # positional argument
print(total) # 60
```

- Можна також використати аргумент `start` як ключове слово так:

```py
numbers = [5, 10, 15, 20]
total = sum(numbers, start=10) # keyword argument
print(total) # 60
```

## Лямбда-функції

- **Визначення**: Лямбда-функція в Python — це короткий спосіб створити функцію без імені (анонімну функцію).
- Лямбда-функції часто використовуються як аргументи для інших функцій. Ось приклад лямбда-функції:

```py
numbers = [1, 2, 3, 4, 5]

even_numbers = list(filter(lambda x: x % 2 == 0, numbers))
print(even_numbers)  # [2, 4]
```

- Найкращі практики використання лямбда-функцій: не призначати їх змінним, тримати простими і читабельними, використовувати для коротких одноразових функцій.

## Словники

- **Словники**: Словники — це вбудовані структури даних, які зберігають колекції пар ключ-значення. Ключі мають бути незмінними типами даних. Ось загальний синтаксис словника Python:

```python
dictionary = {
    key1: value1,
    key2: value2
}
```

- **Конструктор `dict()`**: Конструктор `dict()` — альтернативний спосіб створення словника. Ви передаєте список кортежів як аргумент конструктору `dict()`. Кортежі містять ключ як перший елемент і значення як другий.

```python
pizza = dict([('name', 'Margherita Pizza'), ('price', 8.9), ('calories_per_slice', 250), ('toppings', ['mozzarella', 'basil'])])
```

- **Дужкова нотація**: Щоб отримати значення за ключем, використовуйте дужкову нотацію.

```python
dictionary[key]
```

## Поширені методи словників

- **Метод `get()`**: Метод `get()` отримує значення за ключем. Це схоже на дужкову нотацію, але дозволяє задати значення за замовчуванням, щоб уникнути помилок, якщо ключ відсутній.

```python
dictionary.get(key, default)
```

- **Методи `keys()` і `values()`**: Методи `keys()` і `values()` повертають об’єкти перегляду з усіма ключами і значеннями словника відповідно. Об’єкт перегляду дозволяє бачити вміст словника без створення копії.

```python
pizza = {
    'name': 'Margherita Pizza',
    'price': 8.9,
    'calories_per_slice': 250
}

pizza.keys()
# dict_keys(['name', 'price', 'calories_per_slice'])

pizza.values()
# dict_values(['Margherita Pizza', 8.9, 250])
```

- **Метод `items()`**: Метод `items()` повертає об’єкт перегляду з усіма парами ключ-значення словника.

```python
pizza.items()
# dict_items([('name', 'Margherita Pizza'), ('price', 8.9), ('calories_per_slice', 250)])
```

- **Метод `clear()`**: Метод `clear()` видаляє всі пари ключ-значення зі словника.

```python
pizza.clear()
```

- **Метод `pop()`**: Метод `pop()` видаляє пару ключ-значення за вказаним ключем і повертає її значення. Якщо ключ відсутній, повертає значення за замовчуванням, якщо воно вказане. Якщо ні — викликає `KeyError`.

```python
pizza.pop('price', 10)
pizza.pop('total_price') # KeyError
```

- **Метод `popitem()`**: У Python 3.7 і вище метод `popitem()` видаляє останній доданий елемент.

```python
pizza.popitem()
```

- **Метод `update()`**: Метод `update()` оновлює словник парами ключ-значення з іншого словника. Якщо ключі співпадають, їхні значення замінюються. Нові ключі додаються.

```python
pizza.update({ 'price': 15, 'total_time': 25 })
```

## Ітерація по словнику

- **Ітерація по значеннях**: Щоб ітеруватися по значеннях словника, напишіть цикл `for` з `values()`, щоб отримати всі значення словника.

```python
products = {
    'Laptop': 990,
    'Smartphone': 600,
    'Tablet': 250,
    'Headphones': 70,
}

for price in products.values():
    print(price)
```

Вивід:

```md
990
600
250
70
```

- **Ітерація по ключах**: Щоб ітеруватися по ключах словника `products`, можна написати `products.keys()` або `products` безпосередньо.

```python
for product in products.keys():
    print(product)
    
# Or

for product in products:
    print(product)
```

Вивід:

```md
Laptop
Smartphone
Tablet
Headphones
```

- **Ітерація по парах ключ-значення**: Щоб ітеруватися одночасно по ключах і значеннях, ітеруйтеся по `products.items()`. Ви отримаєте окремі кортежі з ключами і відповідними значеннями.

```python
for product in products.items():
    print(product)
```

Вивід:

```md
('Laptop', 990)
('Smartphone', 600)
('Tablet', 250)
('Headphones', 70)
```

Щоб зберегти ключ і значення у різні змінні циклу, розділіть їх комою. Перша змінна зберігає ключ, друга — значення.

```python
for product, price in products.items():
    print(product, price)
```

Вивід:

```md

Smartphone 600
Tablet 250
Headphones 70
```

- **Функція `enumerate()`**: Щоб ітеруватися по словнику з лічильником, викличте функцію `enumerate()`. Вона повертає об’єкт `enumerate`, який призначає кожному елементу ціле число, як лічильник. Лічильник можна почати з будь-якого числа, за замовчуванням — з 0.

Призначення індексу і елемента в окремі змінні циклу — поширений спосіб використання `enumerate()`. Наприклад, з `products.items()` можна отримати повну пару ключ-значення разом з індексом:

```python
for index, product in enumerate(products.items()):

```

Вивід:

```md
0 ('Laptop', 990)
1 ('Smartphone', 600)
2 ('Tablet', 250)
3 ('Headphones', 70)
```

Щоб налаштувати початкове значення лічильника, передайте другий аргумент у `enumerate()`. Ось приклад початку лічильника з 1.

```python
for index, product in enumerate(products.items(), 1):
    print(index, product)
```

Вивід:

```md
1 ('Laptop', 990)
2 ('Smartphone', 600)
3 ('Tablet', 250)
4 ('Headphones', 70)
```

## Множини

- **Множини**: Множини — це вбудовані структури даних Python, які не допускають дублікати. Вони змінні і неупорядковані, тобто елементи не зберігаються у певному порядку, тому доступ за індексом або ключем неможливий. Множини можуть містити лише незмінні типи даних, як-от числа, рядки і кортежі.

- **Оголошення множини**: Щоб оголосити множину, запишіть її елементи у фігурних дужках, розділені комами.

```python
my_set = {1, 2, 3, 4, 5}
```

- **Оголошення порожньої множини**: Щоб оголосити порожню множину, потрібно використати функцію `set()`. Просто написати порожні фігурні дужки створить словник.

```python
set() # Set
{}    # Dictionary
```

## Поширені методи множин

- **Метод `add()`**: Додає елемент у множину за допомогою методу `add()`, передаючи новий елемент як аргумент.

```python
my_set.add(6)
```

- **Методи `remove()` і `discard()`**: Щоб видалити елемент з множини, можна використати метод `remove()` або `discard()`, передаючи елемент для видалення. Метод `remove()` викликає `KeyError`, якщо елемент не знайдено, а `discard()` — ні.

```python
my_set.remove(4)
my_set.discard(4)
```

- **Метод `clear()`**: Метод `clear()` видаляє всі елементи множини.

```python
my_set.clear()
```

## Математичні операції над множинами

- **Методи `issubset()` і `issuperset()`**: Методи `issubset()` і `issuperset()` перевіряють, чи є множина підмножиною або надмножиною іншої множини відповідно.

```python
my_set = {1, 2, 3, 4, 5}
your_set = {2, 3, 4, 5}

print(your_set.issubset(my_set)) # True
print(my_set.issuperset(your_set)) # True
```

- **Метод `isdisjoint()`**: Метод `isdisjoint()` перевіряє, чи множини не мають спільних елементів.

```python
my_set = {1, 2, 3}
your_set = {4, 5, 6}

print(my_set.isdisjoint(your_set)) # True
```

- **Оператор об’єднання (`|`)**: Оператор об’єднання `|` повертає нову множину з усіма елементами обох множин.

```python
my_set = {1, 2, 3}
your_set = {4, 5, 6}

my_set | your_set # {1, 2, 3, 4, 5, 6}
```

- **Оператор перетину (`&`)**: Оператор перетину `&` повертає нову множину з елементами, спільними для обох множин.

```python
my_set = {1, 2, 3, 4, 5}
your_set = {2, 3, 4, 6}

my_set & your_set # {2, 3, 4}
```

- **Оператор різниці (`-`)**: Оператор різниці `-` повертає нову множину з елементами першої множини, яких немає в другій.

```python
my_set = {1, 2, 3, 4, 5}
your_set = {2, 3, 4, 6}

my_set - your_set # {1, 5}
```

- **Оператор симетричної різниці (`^`)**: Оператор симетричної різниці `^` повертає нову множину з елементами, які є або в першій, або в другій множині, але не в обох одночасно.

```python
my_set = {1, 2, 3, 4, 5}
your_set = {2, 3, 4, 6}

my_set ^ your_set # {1, 5, 6}
```

- **Оператор `in`**: За допомогою оператора `in` можна перевірити, чи є елемент у множині.

```python
print(5 in my_set)
```

## Інструкція імпорту

- **Інструкція імпорту**: Щоб отримати доступ до елементів вбудованих модулів, використовуйте інструкцію імпорту. Зазвичай імпорти пишуть на початку файлу. Вони працюють однаково для функцій, класів, констант, змінних і інших елементів модуля.

- **Базова інструкція імпорту**: Використовуйте ключове слово `import` і назву модуля:

```python
import module_name
```

Потім, щоб викликати функцію з модуля, використовуйте точкову нотацію: ім’я модуля, крапка, ім’я функції.

```python
module_name.function_name()
```

Наприклад, щоб імпортувати модуль `math` і отримати квадратний корінь з 36:

```python
import math

math.sqrt(36)
```

- **Імпорт модуля з іншим ім’ям**: Щоб імпортувати модуль під іншим ім’ям (псевдонімом), використовуйте `as` і псевдонім у кінці інструкції імпорту. Це корисно для довгих імен модулів або уникнення конфліктів.

```python
import module_name as module_alias
```

Наприклад, щоб звертатися до модуля `math` як `m` у коді:

```python
import math as m
```

Потім можна звертатися до елементів модуля через псевдонім:

```python
m.sqrt(36)
```

- **Імпорт конкретних елементів**: Якщо потрібні не всі елементи модуля, можна імпортувати конкретні за допомогою `from`. Інструкція починається з `from`, потім ім’я модуля, ключове слово `import` і перелік елементів.

```python
from module_name import name1, name2
```

Потім ці імена можна використовувати без префікса модуля у скрипті. Наприклад:

```python
from math import radians, sin, cos

angle_degrees = 40
angle_radians = radians(angle_degrees)

sine_value = sin(angle_radians)
cos_value = cos(angle_radians)

print(sine_value) # 0.6427876096865393
print(cos_value)  # 0.766044443118978
```

Це зручно, але може призвести до конфліктів імен, якщо у вас вже є функції або змінні з такими іменами. Враховуйте це при виборі типу імпорту.

Якщо потрібно призначити псевдоніми цим іменам, можна зробити це за допомогою ключового слова `as` і псевдоніма.

```python
from module_name import name1 as alias1, name2 as alias2
```

- **Інструкція імпорту з зірочкою (`*`)**: Зірочка каже Python імпортувати все з модуля, але без префікса імені модуля.

```python
from module_name import *
```

Наприклад, якщо імпортувати модуль `math` таким чином, можна викликати будь-яку функцію з модуля без вказання імені модуля.

```python
from math import *
print(sqrt(36))  # 6.0
```

Однак це не рекомендується, бо може призвести до конфліктів імен і ускладнити розуміння походження імен.

## `if __name__ == '__main__'`

Саме тому у скриптах часто зустрічається така умова. Вона містить код, який потрібно виконувати **тільки** якщо скрипт запускається як головна програма.

```python
if __name__ == '__main__': 
    # Code
```

## Поширені помилки в Python

- **SyntaxError**: Помилка, яку викликає Python, якщо код не відповідає правилам синтаксису. Наприклад, код `print("Hello there"` викличе синтаксичну помилку з повідомленням `SyntaxError: '(' was never closed`, бо бракує закриваючої дужки.
- **NameError**: Python викликає `NameError`, якщо ви намагаєтеся звернутися до змінної або функції, яку не оголосили. Наприклад, якщо у коді є рядок `print(username)` без оголошення змінної `username`, отримаєте помилку з повідомленням `NameError: name 'username' is not defined`.
- **TypeError**: Помилка, яку викликає Python, коли виконується операція над несумісними типами даних. Наприклад, якщо додати рядок до числа, отримаєте помилку `TypeError: can only concatenate str (not "int") to str`.
- **IndexError**: Отримаєте `IndexError`, якщо звернетеся до індексу, якого немає у списку або інших послідовностях, як кортеж чи рядок. Наприклад, у рядку `Hello world` індекс останнього символу — `10`. Якщо звернутися так: `greet = "hello world"; print(greet[11])`, отримаєте помилку з повідомленням `IndexError: string index out of range`.
- **AttributeError**: Python викликає цю помилку, якщо ви намагаєтеся використати метод або властивість, якої немає у об’єкта такого типу. Наприклад, виклик `.append()` для рядка `"hello".append("!")` викличе помилку з повідомленням `AttributeError: 'str' object has no attribute 'append'`.

## Обробка виключень

- **`try...except`**: Виконує блок коду, який може викликати виключення. Блок `try` — це місце, де очікується помилка, а блок `except` обробляє вказане виключення. Ось приклад:

  ```py
  try:
    print(22 / 0)
  except ZeroDivisionError:
    print('You can\'t divide by zero!')
    # You can't divide by zero!
  ```

  Можна ланцюжити кілька блоків `except`, щоб обробляти різні типи виключень:

  ```py
  try:
    number = int(input('Enter a number: '))
    print(22 / number)
  except ZeroDivisionError:
    print('You cannot divide by zero!')
    # You cannot divide by zero! prints when you enter 0
  except ValueError:
    print('Please enter a valid number!')
    # Please enter a valid number! prints when you don't enter an integer
  ```

- **`else` і `finally`**: Ці блоки розширюють `try...except`. Якщо виключення не виникає, виконується блок `else`. Блок `finally` виконується завжди, незалежно від помилок.

  ```py
  try:
    result = 100 / 4
  except ZeroDivisionError:
    print('You cannot divide by zero!') # This will not run
  else:
    print(f'Result is {result}') # Result is 25.0
  finally:
    print('Execution complete!') # Execution complete!
  ```

- **Об’єкт виключення**: Дозволяє отримати доступ до самого виключення для кращого налагодження і виведення повідомлення про помилку. Щоб отримати об’єкт виключення, використовуйте ключове слово `as`. Ось приклад:

  ```py
  try:
      value = int('This will raise an error')
  except ValueError as e:
      print(f'Caught an error: {e}')
      # Caught an error: invalid literal for int() with base 10: 'This will raise an error'
  ```

- **Інструкція `raise`**: Дозволяє вручну викликати виключення. Використовуйте її, щоб кинути виключення при виконанні певної умови. Ось приклад:

  ```py
  def divide(a, b):
      if b == 0:
          raise ZeroDivisionError('You cannot divide by zero')
      return a / b
  ```
  
## Сигналізація виключень
  
  Інструкція `raise` корисна при створенні власних кастомних виключень, бо дозволяє кинути виключення з власним повідомленням. Ось приклад:

  ```py
  class InvalidCredentialsError(Exception):
      def __init__(self, message="Invalid username or password"):
          self.message = message
          super().__init__(self.message)

  def login(username, password):
      stored_username = "admin"
      stored_password = "password123"
      
      if username != stored_username or password != stored_password:
          raise InvalidCredentialsError()
      
      return f"Welcome, {username}!"
  ```  

  Ось як можна використати функцію `login` з виключенням `InvalidCredentialsError`:

  ```py
  # failed login attempt
  try:
      message = login("user", "wrongpassword")
  except InvalidCredentialsError as e:
      print(f"Login failed: {e}")
  else:
      print(message)

  # successful login attempt
  try:
      message = login("admin", "password123")
  except InvalidCredentialsError as e:
      # This block is not executed because the login was successful
      print(f"Login failed: {e}")
  else:
      # The else block runs if the 'try' block completes without an exception
      print(message)
  ```

  Інструкція `raise` також може використовуватися з ключовим словом `from` для ланцюжка виключень, показуючи зв’язок між різними помилками:

  ```py
  def parse_config(filename):
    try:
        with open(filename, 'r') as file:
            data = file.read()
            return int(data)
    except FileNotFoundError:
        raise ValueError('Configuration file is missing') from None
    except ValueError as e:
        raise ValueError('Invalid configuration format') from e

  config = parse_config('config.txt')
```

## Python Classes and Objects

- **Class Definition**: A class is a blueprint for creating objects. It defines the behavior an object will have through its attributes and methods. Here is a basic example of a class definition in Python:

```python
class Dog:
    def __init__(self, name, age):
        self.name = name
        self.age = age

    def bark(self):
        print(f'{self.name.upper()} says woof woof!')
```

- **Creating Objects**: Objects are instances of a class. They are created by calling the class with the necessary arguments.

```python
dog1 = Dog('Jack', 3)
dog2 = Dog('Thatcher', 5)

dog1.bark()  # JACK says woof woof!
dog2.bark()  # THATCHER says woof woof!
```

- **Calling Methods on Objects**: You can call methods on objects to perform actions or retrieve information.

```python
object_name1.method_name()
object_name2.method_name()
```

- **Difference Between Class and Object**: A class is a reusable template, while an object is a specific instance of that class with actual data.

## Attributes

- **Instance Attributes**: Defined in `__init__()` using `self`, and unique to each object.
- **Class Attributes**: Defined directly inside the class and shared by all instances.

```python
class Dog:
    species = 'French Bulldog'  # Class attribute

    def __init__(self, name):
        self.name = name  # Instance attribute

print(Dog.species) # French Bulldog

jack = Dog('Jack')
print(jack.name)     # Jack
print(jack.species)  # French Bulldog
```

## Methods

- **Methods**: Functions defined inside a class that operate on the object's attributes.

```python
class Car:
    def __init__(self, color, model):
        self.color = color
        self.model = model

    def describe(self):
        return f'This car is a {self.color} {self.model}'

my_car_1 = Car('red', 'Tesla Model S')
print(my_car_1.describe())  # This car is a red Tesla Model S
```

- **Accessing Methods**: Call methods on objects using the dot notation. Here is an example of calling the `describe` method on two different car objects:

```python
class Car:
    def __init__(self, color, model):
        self.color = color  
        self.model = model  

    def describe(self):
        return f'This car is a {self.color} {self.model}'

my_car_1 = Car('red', 'Tesla Model S')
my_car_2 = Car('green', 'Lamborghini Revuelto')

print(my_car_1.describe()) # Calling method using the dot notation

print(my_car_2.describe()) # Calling method using the dot notation
```

## Dunder (Magic) Methods

- **Definition**: Special methods that start and end with a double underscore (e.g., `__init__`, `__len__`, `__str__`, `__eq__`). Python uses them internally for built-in operations.

```python
class Book:
    def __init__(self, title, pages):
        self.title = title
        self.pages = pages

    def __len__(self):
        return self.pages

    def __str__(self):
        return f"'{self.title}' has {self.pages} pages"

    def __eq__(self, other):
        return self.pages == other.pages

book1 = Book('Built Wealth Like a Boss', 420)
print(len(book1))        # 420
print(str(book1))        # 'Built Wealth Like a Boss' has 420 pages
```

- **Calling dunder methods indirectly**: You don't need to call dunder methods directly. Instead, Python automatically calls them when certain actions happen. These operations include:

  - **arithmetic operations like addition, subtraction, multiplication, division, and others**. `__add__()` is called for addition, `__sub__()` for subtraction, `__mul__()` for multiplication, and `__truediv__()` for division.
        
  - **string operations like concatenation, repetition, formatting, and conversion to text**. `__add__()` is called for concatenation, `__mul__()` for repetition, `__format__()` for formatting, `__str__()` and `__repr__()` for text conversion, and so on.
        
  - **comparison operations like equality, less-than, greater-than, and others**. `__eq__()` is called for equality checks, `__lt__()` for less-than, `__gt__()` for greater-than, and so on.
        
  - **iteration operations like making an object iterable and advancing through items**. `__iter__()` is called to return an iterator and  `__next__()` to fetch the next item.

## Real World Example: Shopping Cart

- **Cart Class with Dunder Methods**: Allows adding, removing, iterating, and checking contents with built-in behavior.

```python
class Cart:
    def __init__(self):
        self.items = []

    def add(self, item):
        self.items.append(item)

    def remove(self, item):
        if item in self.items:
            self.items.remove(item)
        else:
            print(f'{item} is not in cart')

    def list_items(self):
        return self.items

    def __len__(self):
        return len(self.items)

    def __getitem__(self, index):
        return self.items[index]

    def __contains__(self, item):
        return item in self.items

    def __iter__(self):
        return iter(self.items)

cart = Cart()
cart.add('Laptop')
print(len(cart))        # 1
print('Laptop' in cart) # True
```

## What is Object-Oriented Programming?

- **Object-oriented programming**: A programming style in which developers treat everything in their code like a real-world object. It is popularly called OOP. The four key principles that help you organize and manage code effectively are **encapsulation**, **inheritance**, **polymorphism**, and **abstraction**
- **Classes**: The blueprint for creating objects. Every single object created from a class has attributes that define data and methods that determine the behaviors of the object.

## What is Encapsulation?

- **Encapsulation**: The bundling of the attributes and methods of an object into a single unit. It lets you hide the internal state of the object behind a simple set of public methods and attributes that act like doors. Behind those doors are private attributes and methods that control how the data changes and who can see it.
- **Example of Encapsulation**: If you want to track a wallet balance, you will allow deposit and withdrawal, but you won't want anyone to tamper with the wallet balance itself:

```py
class Wallet:
   def __init__(self, balance):
       self.__balance = balance # Private attribute

   def deposit(self, amount):
       if amount > 0:
           self.__balance += amount # Add to the balance safely

   def withdraw(self, amount):
       if 0 < amount <= self.__balance:
           self.__balance -= amount # Remove from the balance safely

account = Wallet(500)
print(account.__balance) # AttributeError: 'Wallet' object has no attribute '__balance'
```

- **Difference Between Prefixing Attributes with Single and Double Underscore**: Prefixing attributes and methods with a single underscore means they are meant for internal use. This is a convention, and it doesn't enforce accessing attributes from the outside. Prefixing attributes and methods with a double underscore effectively prevents them from being accessed from outside of their class.

## What Are Getters and Setters?

- **Getters and Setters**: Methods that let you control how the attributes of a class are accessed and modified. You retrieve values with getters and you set values with setters.
- **Properties**: They connect getters and setters, and allow access to data. They run extra logic behind the scenes when you get, set, or delete values.
- **Why Properties Instead of Methods**: Properties are used instead of methods for better readability and cleaner code. They let you access values with dot notation, like regular attributes, without parentheses.
- **Creating a Getter**: To create a getter, you use the `@property` decorator. Here's a getter that gets the radius of a circle:

```py
class Circle:
    def __init__(self, radius):
        self._radius = radius

    @property
    def radius(self): # A getter to get the radius
        return self._radius
  
    @property
    def area(self):  # A getter to calculate area
        return 3.14 * (self._radius ** 2)

my_circle = Circle(3)

print(my_circle.radius) # 3
print(my_circle.area) # 28.26
```

- **Creating a Setter**: To create the setter that will set the radius, you have to define another method with the same name and use `@<property_name>.setter` above it:

```py
class Circle:
    def __init__(self, radius):
        self._radius = radius

    @property
    def radius(self):  # A getter to get the radius
        return self._radius

    @radius.setter
    def radius(self, value):  # A setter to set the radius
        if value <= 0:
            raise ValueError('Radius must be positive')
        self._radius = value

my_circle = Circle(3)
print('Initial radius:', my_circle.radius) # Initial radius: 3

my_circle.radius = 8
print('After modifying the radius:', my_circle.radius) # After modifying the radius: 8
```

- **How Python Handles Getters and Setters**: Once you define getters and setters, Python automatically calls them under the hood whenever you use normal attribute syntax this way:

```py
my_circle.radius # This will call the getter
my_circle.radius = 4 # This will call the setter
```

 When setting a value, you should not assign to the property name itself because that will cause a `RecursionError`. Use a separate internal name, often with an underscore, to store the value.

- **Deleter**: After setting and getting a value with setter and getter, you can control how it is deleted with a `deleter`. A deleter runs custom logic when you use the `del` statement on a property. To create a deleter, you use the `@<property_name>.deleter` decorator.

```py
  # Deleter
    @radius.deleter
    def radius(self):
        print("Deleting radius...")
        del self._radius
```

## What Is Inheritance and How Does It Promote Code Reuse?

- **Inheritance**: The process by which a child class uses the attributes and methods of a parent class. Inheritance promotes code reuse, provides clear hierarchies, and customizes behavior without rewriting everything. To implement inheritance, a child class takes in the name of a parent class:

```py
class Parent:
    # Parent attributes and methods

class Child(Parent):
    # Child inherits, extends, and/or overrides where necessary
```

- **Single and Multiple Inheritance**: When a child class inherits properties and methods from a single parent, as you can see above, the process is called **single inheritance**. When a child class inherits properties and methods from more than one parent, that is **multiple inheritance**. Here's the syntax for that:

```py
class Parent:
    # Attributes and methods for Parent

class Child:
    # Attributes and methods for Child

class GrandChild(Parent, Child):
    # GrandChild inherits from both Parent and Child
    # GrandChild can combine or override behavior from each
```

- **`super()` Function**: A function that lets you **call** a method from a parent class, when a class has a different implementation of that method, or it extends the method, without duplicating code.

## What Is Polymorphism and How Does It Promote Code Reuse?

- **Polymorphism**: The OOP principle that lets different classes use the same method name, but each class implements it differently when called. Here's the syntax for it:

```py
class A:
   def action(self): ...

class B:
   def action(self): ...

class C:
   def action(self): ...

Class().method()  # Works for A, B, or C
```

- **Inheritance-based polymorphism**: A parent sets up a method, and each child class twists it to their use.

## What is Name Mangling and How Does it Work?

- **Name Mangling**: A process in which Python internally renames an attribute prefixed with a double underscore by adding an underscore and the class name as a prefix, turning `__attribute` into `_ClassName__attribute`.
- **The Purpose of Name Mangling**: The main purpose of name mangling is to prevent accidental attribute and method overriding when you use inheritance. Here's a code that makes that more understandable:

```py
class Parent:
    def __init__(self):
        self.__data = 'Parent data'

class Child(Parent):
    def __init__(self):
        super().__init__()
        self.__data = 'Child data'

c = Child()
print(c.__dict__) # {'_Parent__data': 'Parent data', '_Child__data': 'Child data'}
```

## What Is Abstraction and How Does It Help Keep Complex Systems Organized?

- **Abstraction**: A programming concept in which complex implementation details of object or system are hidden and only the essential features are shown. In Python and other programming languages, abstraction simplifies complex systems by increasing reusability.
- **Example of Abstraction**: A good example of abstraction in everyday life is a car letting you just use the wheel, pedals, and shifter without knowing how the engine or brakes work.
- **How Python Implements Abstraction**: Python implements abstraction through the `abc` module. The module provides the `ABC` class (abstract base class) and the `@abstractmethod` decorator. An abstract base class (ABC) defines the common methods and properties subclasses must implement. It can't be instantiated.
- **How Abstract Method is Defined**: An abstract method is defined with `@abstractmethod` and must be overridden in subclasses, even if it has a default implementation. The basic syntax of abstraction looks like this:

```py
from abc import ABC, abstractmethod

# Define an abstract base class
class AbstractClass(ABC):
    @abstractmethod
    def abstract_method(self):
        pass

# Concrete subclass that implements the abstract method
class ConcreteClassOne(AbstractClass):
    def abstract_method(self):
        print('Implementation in ConcreteClassOne')

# Another concrete subclass
class ConcreteClassTwo(AbstractClass):
    def abstract_method(self):
        print('Implementation in ConcreteClassTwo')
```

## Algorithms and Big O Notation

- **Algorithms**: A set of unambiguous instructions for solving a problem or carrying out a task. Algorithms must finish in a finite number of steps and each step must be precise and unambiguous.

- **Big O Notation**: Describes the worst-case performance, or growth rate, of an algorithm as the input size increases. It focuses on how resource usage grows with input size, ignoring constant factors and lower-order terms.

### Common Time Complexities

- **O(1) - Constant Time**: Algorithm takes the same amount of time regardless of input size.

```python
def check_even_or_odd(number):
    if number % 2 == 0:
        return 'Even'
    else:
        return 'Odd'
```

- **O(log n) - Logarithmic Time**: Time increases slowly as input grows. Common in algorithms that repeatedly reduce problem size by a fraction (like Binary Search).

- **O(n) - Linear Time**: Running time increases proportionally to input size.

```python
for grade in grades:
    print(grade)
```

- **O(n log n) - Log-Linear Time**: Common time complexity of efficient sorting algorithms like Merge Sort and Quick Sort.

- **O(n²) - Quadratic Time**: Running time increases quadratically. Often seen in nested loops.

```python
for i in range(n):
    for j in range(n):
        print("Hello, World!")
```

### Space Complexity

- **O(1) - Constant Space**: Algorithm uses same amount of memory regardless of input size.
- **O(n) - Linear Space**: Memory usage grows proportionally with input size.
- **O(n²) - Quadratic Space**: Memory usage grows quadratically with input size.

## Problem-Solving Techniques

- **Understanding the Problem**: Read the problem statement multiple times. Identify the input, expected output, and how to transform input to output.

- **Pseudocode**: High-level description of algorithm logic that is language-independent. Uses common written language mixed with programming constructs like `IF`, `ELSE`, `FOR`, `WHILE`.

```md
GET original_string
SET reversed_string = ""
FOR EACH character IN original_string:
  ADD character TO THE BEGINNING OF reversed_string

```

- **Edge Cases**: Specific, valid inputs that occur at the boundaries of what an algorithm should handle. Always consider and test edge cases.

## Arrays

- **Static Arrays**: Have a fixed size determined at initialization. Elements stored in adjacent memory locations. Size cannot be changed during program execution.

- **Dynamic Arrays**: Can grow or shrink automatically during program execution. Handle resizing through automatic copying to larger arrays when needed.

### Python Lists (Dynamic Arrays)

```python
numbers = [3, 4, 5, 6]

# Access elements
numbers[0]  # 3

# Update elements
numbers[2] = 16

# Add elements
numbers.append(7)
numbers.insert(3, 15)  # Insert at specific index

# Remove elements
numbers.pop(2)  # Remove at specific index
numbers.pop()   # Remove last element
```

### Time Complexities for Dynamic Arrays

- **Access**: O(1)
- **Insert at end**: O(1) average, O(n) when resizing needed
- **Insert in middle**: O(n)
- **Delete**: O(n) for middle, O(1) for end

## Stacks

- **Stacks**: Last-In, First-Out (LIFO) data structure. Elements added and removed from the top only.

- **Push Operation**: Adding an element to the top of the stack. Time complexity: O(1).

- **Pop Operation**: Removing an element from the top of the stack. Time complexity: O(1).

```python
# Using Python list as stack
stack = []

# Push operations
stack.append(1)
stack.append(2)
stack.append(3)

# Pop operations
top_element = stack.pop()  # Returns 3
```

## Queues

- **Queues**: First-In, First-Out (FIFO) data structure. Elements added to the back and removed from the front.

- **Enqueue Operation**: Adding an element to the back of the queue. Time complexity: O(1).

- **Dequeue Operation**: Removing an element from the front of the queue. Time complexity: O(1).

```python
from collections import deque

# Using deque for efficient queue operations
queue = deque()

# Enqueue operations
queue.append(1)
queue.append(2)
queue.append(3)

# Dequeue operations
first_element = queue.popleft()  # Returns 1
```

## Linked Lists

- **Linked Lists**: Linear data structure where each node contains data and a reference to the next node. Nodes are connected like a chain.

### Singly Linked Lists

- **Structure**: Each node has data and one reference to the next node.
- **Traversal**: Can only move forward from head to tail.
- **Head Node**: First node in the list, usually the only directly accessible node.
- **Tail Node**: Last node in the list, points to `None`.

### Operations and Time Complexities

- **Insert at beginning**: O(1)
- **Insert at end**: O(n) - must traverse to end
- **Insert in middle**: O(n) - must traverse to position
- **Delete from beginning**: O(1)
- **Delete from end**: O(n) - must traverse to find previous node
- **Delete from middle**: O(n) - must traverse to find node

### Doubly Linked Lists

- **Structure**: Each node has data and two references: next node and previous node.
- **Traversal**: Can move in both directions.
- **Memory**: Requires more memory than singly linked lists due to extra reference.

## Hash Maps and Sets

### Maps and Hash Maps

- **Map (Abstract Data Type)**: Manages collections of key-value pairs. Every key must be unique, but values can be repeated.

- **Hash Map**: Concrete implementation of map ADT using hashing technique. Uses hash function to generate hash values for keys, which determine storage location in underlying array.

### Python Dictionaries (Hash Maps)

```python
# Creating dictionaries
my_dictionary = {
    "A": 1,
    "B": 2, 
    "C": 3
}

# Alternative creation
my_dictionary = dict(A=1, B=2, C=3)

# Access and modify
value = my_dictionary["A"]  # 1
my_dictionary["A"] = 4      # Update value
del my_dictionary["A"]      # Remove key-value pair

# Check membership
"C" in my_dictionary

# Get keys, values, items
my_dictionary.keys()
my_dictionary.values()
my_dictionary.items()
```

### Time Complexities for Hash Maps

- **Average case**: O(1) for insert, get, delete
- **Worst case**: O(n) when many hash collisions occur

### Sets

- **Sets**: Unordered collections of unique elements. No duplicates allowed, no specific order maintained.

- **Immutable Elements Only**: Sets can only contain immutable data types (numbers, strings, tuples) because hash values must remain constant.

```python
# Creating sets
numbers = {1, 2, 3, 4}
empty_set = set()  # Must use set(), not {}

# Add and remove elements
numbers.add(5)
numbers.remove(4)      # Raises KeyError if not found
numbers.discard(4)     # No error if not found

# Set operations
set_a = {1, 2, 3, 4}
set_b = {2, 3, 4, 5, 6}

# Union, intersection, difference, symmetric difference
set_a.union(set_b)                    # or set_a | set_b
set_a.intersection(set_b)             # or set_a & set_b
set_a.difference(set_b)               # or set_a - set_b
set_a.symmetric_difference(set_b)     # or set_a ^ set_b

# Subset and superset checks
set_a.issubset(set_b)
set_a.issuperset(set_b)
set_a.isdisjoint(set_b)

# Membership testing
5 in numbers
```

### Time Complexities for Sets

- **Average case**: O(1) for add, remove, membership testing
- **Worst case**: O(n) due to hash collisions

## Hash Collisions

- **Hash Collision**: Occurs when two different keys produce the same hash value.

- **Collision Resolution Strategies**:
  - **Chaining**: Each array index points to a linked list storing all elements with same hash value
  - **Open Addressing**: Search for next available index using predefined sequence

## When to Use Each Data Structure

- **Lists**: When you need ordered, indexed access and don't know size in advance
- **Stacks**: For LIFO operations (undo functionality, expression evaluation, backtracking)
- **Queues**: For FIFO operations (task scheduling, breadth-first search)
- **Linked Lists**: When frequent insertion/deletion at beginning, unknown size, no random access needed
- **Hash Maps**: For fast key-value lookups, counting occurrences, caching
- **Sets**: For uniqueness checking, mathematical set operations, removing duplicates

## Searching Algorithms

Searching algorithms let you search for a target within a certain list of items.

In computer science, there are two searching algorithms you should know about. They are **linear search** and **binary search** algorithms. It is important to understand the differences between the two algorithms and when to use each one.

### Linear Search

- Linear search iterates through a list of items, checking each item from the beginning until the target item is found.
- If the target item is found, the index where it is located in the list is returned.
- If the target is not found, it returns `-1`, which means **invalid index** in most programming languages.
- Because linear search checks each item until it finds the target, it is not efficient for a large list of items.
- The time complexity of linear search is  `O(n)` because the time it takes to search through the list grows linearly with the size of the list.
- The space complexity of linear search is `O(1)` because it doesn't require any additional space to search through the list.

### Binary Search

- Binary search works by dividing a list of items in half, and checking if the target value is in the middle of the list.
- The condition for binary search to work is that the items in the list must be in ascending order.
- Binary search is a more efficient algorithm for searching through a large list of items because it divides the list of items in half and ignores any half where the target is not found.
- If the target item is found in the middle of the list, the index of the target item is returned.
- If the item is not found, the algorithm checks if the target item is in the left or right half of the list.
- It continues to divide the remaining parts of the list into halves until the target item is found.
- If the target item is finally not found in the list, it returns `-1`
- The time complexity of binary search is `O(log n)` because the time it takes to search through the list grows logarithmically with the size of the list.
- The space complexity of binary search is `O(1)` because it doesn't require any additional space to search through the list.

### How Linear Search Differs from Binary Search 

- Binary search is more suitable for a large list of items compared to linear search.
- The time complexity of linear search is  `O(n)` because the time it takes to search through the list grows linearly with the size of the list.
- The time complexity of binary search is `O(log n)` because the time it takes to search through the list grows logarithmically with the size of the list.

## Sorting Algorithms and Divide-and-Conquer

In computer science, divide-and-conquer is a technique used to break down a problem into smaller sub-problems so they are easier to solve. Recursion is the technique often employed in divide-and-conquer, and divide-and-conquer is a powerful strategy used to implement many efficient sorting algorithms like merge sort.

### Merge Sort

- Merge sort is a sorting algorithm that follows the divide-and-conquer approach.
- It works by recursively dividing a list into smaller sub-lists until each sub-list contains only one element.
- It then repeatedly merges the sub-lists back together in a sorted order.
- The time complexity for merge sort is `O(n log n)` because the list is continuously divided in half `(log n)` and then merged together `(O(n))`.
- The space complexity of merge sort is `O(n)` because it is not an in-place sorting algorithm.

## Graphs Overview

A graph is a set of nodes (vertices) connected by edges (connections). Each node can connect to multiple other nodes, forming a network. The different types of graphs include:

- Directed: edges have a direction (from one node to another), often represented with straight lines and arrows.
- Undirected: edges have no direction, represented with simple lines.
- Vertex: each node is associated with a label or identifier.
- Cyclic: contains cycles (a path that starts and ends at the same node).
- Acyclic (DAG): does not contain cycles.
- Edge labeled: each edge has a label usually drawn next to corresponding edge.
- Weighted: edges have weights (values) associated with them, that can be used to perform arithmetic operations. 
- Disconnected: contains two or more nodes that are not connected by any edges.

Graphs are used in various applications such as maps, networks, recommendation systems, dependency resolution.

## Graph Traversals

This involves visiting all the nodes in a graph. The two main algorithms are:

- **Breadth-First Search (BFS)**
  - Uses a queue.
  - Explores level by level.
  - Finds shortest path in unweighted graphs.

- **Depth-First Search (DFS)**
  - Uses a stack (or recursion).
  - Explores a branch fully before backtracking.
  - Useful for cycle detection and path finding.

## Graph Representations

Graphs can be represented in two main ways:

- **Adjacency List**
  - Each node has a list of its neighbors.
  - Space efficient for sparse graphs.
  - Easy to iterate over neighbors.

- **Adjacency Matrix**
  - A 2D array where rows and columns represent nodes.
  - Space intensive for large graphs.
  - Fast to check if an edge exists between two nodes.

## Trees

A tree is a special type of graph that is acyclic and connected. Key properties include:

- They have no loops or cycles (paths where the start and end nodes are the same).
- They must be connected (every node can be reached from every other node).

### Common types of trees

The most common types of trees are:

- Binary Trees
  - Each node has at most two children, a left and a right child.

- Binary Search Trees (BST)
  - A binary tree in which every left child is less than its parent, and every right child is greater than its parent.  


## Tries

Also known as prefix trees, they are used to store sets of strings, where each node represents a character.

Shared prefixes are stored only once, making them efficient for tasks like autocomplete and spell checking.

Search and insertion operations have a time complexity of O(L), where L is the length of the string.

## Priority Queues

A priority queue is an abstract data type where each element has a priority.

Queues and stacks consider only the order of insertion, while priority queues consider the priority of elements. 

Standard queues follow FIFO (First In First Out) and stacks follow LIFO (Last In First Out). However, in a priority queue, elements with higher priority are served before those with lower priority, regardless of their insertion order.

## Heaps

It's a specialized tree-based data structure with a very specific property called the heap property. 

The heap property determines the relationship between parent and child nodes. There are two types of heaps:

- Max-Heap
  - The value of each parent node is greater than or equal to the values of its children.
  - The largest element is at the root.

- Min-Heap
  - The value of each parent node is less than or equal to the values of its children.
  - The smallest element is at the root.

### Python `heapq` module example

```py
import heapq

# Create empty heap
my_heap = []

# Insert elements
heapq.heappush(my_heap, 9)
heapq.heappush(my_heap, 3)
heapq.heappush(my_heap, 5)
print(my_heap) # [3, 9, 5]

# Remove smallest element
print(heapq.heappop(my_heap))  # 3
print(my_heap) # [5, 9]

# Push + Pop in one step
print(heapq.heappushpop(my_heap, 7)) # 5
print(my_heap) # [7, 9]

# Transform list into heap
nums = [5, 7, 3, 1]
heapq.heapify(nums)
```

### Using Priorities

```py
my_heap = []
heapq.heappush(my_heap, (3, "A"))
heapq.heappush(my_heap, (2, "B"))
heapq.heappush(my_heap, (1, "C"))

# Removes lowest number = highest priority
print(heapq.heappop(my_heap))  # (1, "C")
```

## Introduction to Dynamic Programming

- **Definition**: Dynamic programming is an algorithmic technique that solves complex problems by breaking them down into simpler subproblems and storing the results to avoid redundant calculations. 
- **Overlapping Subproblems**: The same smaller problems appear multiple times when solving the larger problem. Instead of recalculating these subproblems repeatedly, we store their solutions.
- **Optimal Substructure**: The optimal solution to the problem contains optimal solutions to its subproblems. This means we can build up the best solution by combining the best solutions to smaller parts.

## Dynamic Programming Solutions

- **Memoization (Top-Down Approach)**: Memoization stores the results of expensive function calls and returns the cached result when the same inputs occur again. 

```py
def climb_stairs_memo(n, memo={}):
    """Dynamic programming with memoization"""
    # Check if we've already calculated this value
    if n in memo:
        return memo[n]  # Return cached result - O(1) lookup!
    
    # Base cases
    if n <= 2:
        return n
    
    # Calculate once and store in memo for future use
    memo[n] = climb_stairs_memo(n-1, memo) + climb_stairs_memo(n-2, memo)
    return memo[n]
```

- **Tabulation (Bottom-Up Approach)**: Tabulation builds the solution from the ground up, filling a table with solutions to subproblems.

```py
def climb_stairs_tabulation(n):
    """Dynamic programming with tabulation"""
    if n <= 2:
        return n
    
    # Create array to store results for all steps from 0 to n
    dp = [0] * (n + 1)
    dp[1] = 1  # 1 way to reach step 1
    dp[2] = 2  # 2 ways to reach step 2
    
    # Build up the solution iteratively
    for i in range(3, n + 1):
        # Ways to reach step i = ways to reach (i-1) + ways to reach (i-2)
        dp[i] = dp[i-1] + dp[i-2]
    
    return dp[n]
```
