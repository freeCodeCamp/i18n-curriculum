# --description--

念のため、このプロジェクトは<a href="https://github.com/freeCodeCamp/boilerplate-mochachai/" target="_blank" rel="noopener noreferrer nofollow">GitHub</a>からクローンした以下のスタータープロジェクトを基に構築されています。

Mochaは`chai-http`というプラグインを使うことで、APIエンドポイントへの呼び出しのような非同期操作をテストできます。

以下は`chai-http`を使った`'GET /hello?name=[name] => "hello [name]"'`というスイートのテスト例です。

```js
suite('GET /hello?name=[name] => "hello [name]"', function () {
  test('?name=John', function (done) {
    chai
      .request(server)
      .keepOpen()
      .get('/hello?name=John')
      .end(function (err, res) {
        assert.equal(res.status, 200, 'Response status should be 200');
        assert.equal(res.text, 'hello John', 'Response should be "hello John"');
        done();
      });
  });
});
```

このテストは名前をURLクエリ文字列（`GET`）としてサーバーに`?name=John`リクエストを送信します。`end`メソッドのコールバック関数内でレスポンスオブジェクト（`res`）を受け取り、その中に`status`プロパティが含まれています。

最初の`assert.equal`はステータスが`200`と等しいかどうかをチェックします。2つ目の`assert.equal`はレスポンスの文字列（`res.text`）が`"hello John"`と等しいかどうかをチェックします。

また、テストのコールバック関数内の`done`パラメータにも注目してください。非同期操作が完了したことを示すために、テストの最後で引数なしで呼び出す必要があります。

最後に、`keepOpen`メソッドの直後にある`request`メソッドに注意してください。通常はコマンドラインからテストを実行したり、自動統合プロセスの一部として実行したりし、`chai-http`にサーバーの起動と停止を自動で任せることができます。

しかし、プロジェクトのリンクを提出した際に実行されるテストはサーバーが起動している必要があるため、`keepOpen`がサーバーを停止するのを防ぐために`chai-http`メソッドを使う必要があります。

# --instructions--
