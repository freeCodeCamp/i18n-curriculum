### --question--

#### --text--

HTTP 通信は WebSocket 通信とどのように異なりますか？

#### --distractors--

HTTP は双方向で WebSocket は単方向である。

---

HTTP は永続的な接続を維持し、WebSocket は交換ごとに接続を閉じる。

---

HTTP と WebSocket はどちらも同じリクエスト-レスポンスパターンに従う。

#### --answer--

HTTP は `Client --> Request --> Server --> Response` に従い、WebSocket は `Client <--> Server` である。

#### --text--

Pub/Sub のメッセージフローはどのようになっていますか？

#### --distractors--

パブリッシャーがサブスクライバーに送信 --> サブスクライバーがトピックに転送 --> トピックがメッセージを保存。

---

トピックがメッセージを送信 --> パブリッシャーが受け取る --> パブリッシャーがサブスクライバーに転送。

---

サブスクライバーがパブリッシャーに接続 --> パブリッシャーがデータ送信 --> サブスクライバーが受領確認。

#### --answer--

サブスクライバーがトピックを購読 --> パブリッシャーがそのトピックにメッセージを送信 --> メッセージシステムがそのトピックのすべてのサブスクライバーに配信。

#### --text--

WebSocket 接続の流れを正しく説明しているのはどれですか？

#### --distractors--

---

クライアントがデータ送信 --> サーバーがアップグレード要求 --> 接続開始 --> クライアントが確認。
