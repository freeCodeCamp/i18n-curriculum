# --description--

當你在建置你的專案時，你經常會遇到你的程式無法如預期運作的問題。

程式設計師經常將問題稱為錯誤。尋找並修正這些錯誤的處理程序稱為偵錯。

要偵錯你的程式碼，你需要使用瀏覽器提供的一些工具。

兩個重要的工具是 DOM 檢查器和開發者工具。

DOM 檢查器允許你檢查你所在頁面的 HTML 結構。

DOM 代表文件物件模型。它是一種樹狀結構，表述頁面上的元素。你將在後續模組中學習更多關於 DOM 的內容。

開發者工具允許你檢查你所在頁面的 HTML、CSS 和 JavaScript。

讓我們來看看一個包含錨點元素小錯誤的 HTML 範例：

```html
<a href="https://www.freecodecamp.org/larn/">freeCodeCamp curriculum</a>
```

當你點擊該聯結時，將會導向 404 頁面。404 頁面是當使用者嘗試存取伺服器上不存在的網頁時所出現的錯誤頁面。

目的是讓該聯結導向 freeCodeCamp 課程。

若要查看可能的問題，你可以使用開發者工具。

要開啟瀏覽器中的開發者工具，你可以在頁面上按右鍵並選擇 `Inspect`。

你也可以在你的 PC 鍵盤上使用 `Control Shift I`，或在你的 Mac 上使用 `Command Option I`。

當你在 Google Chrome 中開啟開發者工具時，你會看到多個分頁。第一個分頁稱為 `Elements` 分頁。此分頁會顯示你所在頁面的 HTML 結構。

第二個分頁稱為 `Console` 分頁。此分頁會顯示頁面上可能發生的任何錯誤。

在連結損壞的情況下，你可以檢查主控台以查看該損壞連結的錯誤訊息。持續顯示的常見損壞連結訊息是 404 錯誤。404 錯誤顯示該頁面未找到。

這讓我們知道問題出在錨點元素中的 URL。當你檢查 `href` 值時，你會看到有一個拼寫錯誤。

目前主控台訊息顯示 `/larn` 對應 404，但正確的 URL 應該是 `/learn`。當聯結修正後，就會如預期般運作。

你將在整個認證過程中學習更多關於使用開發者工具的知識，但這只是一個簡短的範例，說明它如何幫助你偵錯你的程式碼。
