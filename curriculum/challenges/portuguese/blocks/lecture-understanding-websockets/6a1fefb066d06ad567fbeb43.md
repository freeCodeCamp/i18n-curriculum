# --description--

WebSockets permitem que um cliente e um servidor se comuniquem em tempo real.

Normalmente, aplicações do lado do cliente, como apps web e móveis, usam HTTP. Com HTTP, o cliente envia uma requisição e o servidor responde. Isso é chamado de ciclo de requisição e resposta, como você já aprendeu.

Depois que a conexão é estabelecida e os dados são trocados, a conexão é fechada. Isso funciona bem para a maioria das coisas, como carregar uma página, buscar ou enviar dados. Mas fica lento e pouco confiável para aplicações em tempo real.

Por exemplo, pense em um app de chat. Se você decidir usar HTTP para ele, o cliente precisa ficar perguntando ao servidor pelos dados de tempos em tempos, com a conexão abrindo, os dados chegando e a conexão fechando depois para cada mensagem.

Esse é o problema que os WebSockets resolvem. Eles criam uma conexão permanente entre o cliente e o servidor. Uma vez que a conexão abre, ambos os lados podem trocar dados a qualquer momento em um canal em tempo real até que um sinal de fechamento seja enviado de um para o outro.

Embora os WebSockets sejam diferentes do HTTP, toda conexão WebSocket começa com HTTP, em um processo chamado handshake.

O cliente então solicita que a conexão seja atualizada para WebSocket. Quando o servidor concorda, ele envia uma `101 Switching Protocols` resposta que atualiza a conexão para WebSocket. Depois disso, a conexão permanece aberta e a comunicação em tempo real começa.

O fluxo é assim:

1. Cliente conecta ao servidor via HTTP  
2. Servidor aceita e atualiza a conexão para WebSocket  
3. Conexão permanece aberta  
4. Ambos os lados trocam dados a qualquer momento até a conexão fechar  

Esse fluxo é bidirecional, não de requisição-resposta como o HTTP.

```md
Client <---> Server
```

Com HTTP, o fluxo normalmente segue um modelo de requisição-resposta:

```md
Client ---> Request
Server ---> Response
```

Sobre como os WebSockets diferem do HTTP, HTTP é como trocar cartas, e WebSocket é como uma ligação telefônica. Você escreve uma carta, espera uma resposta e a conversa termina. Mas numa ligação, você e o destinatário conversam em tempo real.

WebSockets alimentam apps em tempo real como apps de chat, jogos multiplayer online, atualizações ao vivo de esportes, painéis do mercado financeiro e ferramentas colaborativas como o Google Docs.
