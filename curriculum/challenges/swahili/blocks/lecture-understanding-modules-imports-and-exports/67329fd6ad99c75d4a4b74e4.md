# --description--

Katika JavaScript, moduli ni kitengo cha msimbo chenye kujitegemea kinachojumuisha kitendakazi, madarasa ya vitu, au vigezo vinavyohusiana.

Fikiria moduli kama kipande cha ujenzi cha programu yako, kama vile sura katika kitabu. Kila moduli inazingatia utendaji maalum, na kufanya msimbo wako kuwa na mpangilio mzuri, rahisi kudumisha, na unaoweza kutumika tena. Moduli husaidia kuzuia migongano ya majina na kuruhusu kuunda programu yako katika vipande tofauti vinavyoshirikiana.

Dhana ya moduli katika JavaScript imebadilika kwa muda, lakini njia inayotumika zaidi na inayoungwa mkono ni mfumo wa moduli wa ES6 (ECMAScript 2015). Mfumo huu hutoa njia ya kawaida ya kufafanua na kutumia moduli katika mazingira tofauti ya JavaScript.

Ili kuunda moduli, unaandika msimbo wako wa JavaScript katika faili tofauti. Vigezo vyovyote, vitendakazi, au madarasa ya vitu unayotaka kuyafanya kupatikana kwa sehemu nyingine za programu yako lazima yaeexportwe waziwazi. Unaweza kufanya hivi kwa kutumia neno kuu la `export`.

Kwa mfano, tuseme una faili inayoitwa `math.js` yenye baadhi ya vitendakazi vya hesabu:

```js
export function add(a, b) {
    return a + b;
}

export function subtract(a, b) {
    return a - b;
}

const PI = 3.14159;
export { PI };
```

Katika mfano huu, tunaeexport vitendakazi viwili (`add` na `subtract`) na thabiti (`PI`). Unaweza kuuexport vitu vingi kadri unavyohitaji kutoka moduli moja.

Ili kutumia vitu hivi vilivyoexportwa katika sehemu nyingine ya programu yako, unahitaji kuviimport. Hii hufanywa kwa kutumia neno kuu la `import`.

Tuseme unataka kutumia vitendakazi hivi vya hesabu katika faili inayoitwa `app.js`:

```js
import { add, subtract, PI } from './math.js';

console.log(add(5, 3));        // Outputs: 8
console.log(subtract(10, 4));  // Outputs: 6
console.log(PI);               // Outputs: 3.14159
```

Hapa, tunaviimport vitendakazi maalum na thabiti tunazohitaji kutoka moduli ya `math.js`. Sehemu ya `'./math.js'` inaeleza JavaScript wapi pa kupata faili la moduli kulingana na faili ya sasa.

Wakati mwingine, unaweza kutaka kuimport kila kitu ambacho moduli inaexport. Unaweza kufanya hivi kwa kutumia sintaksia ya nyota (`*`):

```js
import * as Math from './math.js';

console.log(Math.add(5, 3));        // Outputs: 8
console.log(Math.subtract(10, 4));  // Outputs: 6
console.log(Math.PI);               // Outputs: 3.14159
```

Katika kesi hii, vitu vyote vilivyoexportwa kutoka `math.js` vinaimportwa kama mali za kitu kinachoitwa `Math`.

Mfumo mwingine wa kawaida ni kuwa na export ya default katika moduli. Hii kawaida hutumika wakati moduli inaeexport kitendakazi kimoja hasa. Unaweza kuwa na export moja tu ya default kwa moduli moja.

Hivi ndivyo inavyoonekana:

```js
// In math.js
export default function multiply(a, b) {
    return a * b;
}

// In app.js
import multiply from './math.js';

console.log(multiply(4, 5));  // Outputs: 20
```

Angalia kwamba unapoimport export ya default, huna haja ya kutumia mabano ya mviringo, na unaweza kuitaja import kwa jina lolote unalotaka.

Ni muhimu kutambua kwamba ili kutumia moduli za ES6 katika kivinjari, unahitaji kubainisha `type` kama `module` katika lebo yako ya `script`:

```html
<script type="module" src="app.js"></script>
```

Moduli hutoa njia yenye nguvu ya kupanga na kuunda msimbo wako wa JavaScript. Zinakuwezesha kugawanya programu yako katika vipande vidogo, vinavyoweza kudhibitiwa, kukuza matumizi tena ya msimbo, na kusaidia kudumisha mgawanyiko safi wa majukumu. Unapoendelea kujenga programu kubwa, utagundua kwamba moduli zinakuwa chombo muhimu katika zana zako za ukuzaji wa JavaScript.
