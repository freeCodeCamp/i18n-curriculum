# --description--

Jetzt sind Sie bereit, Ihre ersten Schritte in der Welt von Node.js zu machen, indem Sie es auf Ihrem Computer installieren. Aber bevor Sie Node.js installieren, empfehlen wir Ihnen, ein Tool namens NVM (Node Version Manager) zu installieren. Wie der Name schon sagt, ist NVM ein Tool zur Verwaltung mehrerer Versionen von Node.js auf demselben Computer.

Die Einrichtung dieses Tools von Anfang an wird Ihnen in Zukunft helfen, da Sie möglicherweise zwischen verschiedenen Versionen von Node.js für unterschiedliche Projekte wechseln müssen. Mit NVM können Sie das sehr einfach tun.

Sehen wir uns an, wie Sie NVM auf macOS/Linux und Windows installieren können und wie Sie es verwenden, um Node.js zu installieren.

## Wie man NVM auf macOS/Linux installiert

Auf macOS und Linux können Sie NVM direkt über ein Shell-Skript installieren.

Beachten Sie jedoch, dass die offizielle Dokumentation für NVM bei macOS erwähnt, dass „Sie die Xcode-Befehlszeilentools manuell installieren müssen, bevor Sie das Installationsskript ausführen, sonst schlägt es fehl.“

Auf macOS können Sie die Xcode-Befehlszeilentools installieren, indem Sie diesen Befehl in Ihrem Terminal ausführen:

```bash
xcode-select --install
```

Sobald die Installation abgeschlossen ist, können Sie die Versionsnummer überprüfen und die Installation mit diesem Befehl verifizieren:

```bash
xcode-select --version
```

Dann ist es Zeit, NVM zu installieren. Sie können das NVM-Installationsskript herunterladen und ausführen, indem Sie einen dieser Befehle ausführen:

```bash
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.3/install.sh | bash
```

```bash
wget -qO- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.3/install.sh | bash
```

Um die neueste Version von NVM zum Zeitpunkt des Lesens dieses Artikels zu erhalten, konsultieren Sie bitte die offizielle NVM-Dokumentation: `https://github.com/nvm-sh/nvm?tab=readme-ov-file#install--update-script`. Dort finden Sie diese Befehle mit der neuesten Versionsnummer.

Möglicherweise müssen Sie Ihr Terminal neu starten, wenn der Vorgang abgeschlossen ist, damit der neue Befehl und die neue Konfiguration geladen werden.

Führen Sie dann diesen Befehl aus, um zu bestätigen, dass NVM korrekt installiert wurde, und um Ihre aktuelle Version zu überprüfen:

```bash
nvm -v
```

Sie sollten Ihre Version von NVM in der Ausgabe sehen.

## Wie man Node.js mit NVM auf macOS/Linux installiert

Jetzt, wo Sie NVM auf macOS/Linux installiert haben, können Sie Node.js installieren.

Um die neueste Long-Term-Support-(LTS-)Version von Node.js zu installieren, führen Sie diesen Befehl aus:

```bash
nvm install --lts
```

Beachten Sie die `--lts`-Flagge. Diese Flagge zeigt an, dass Sie die aktuelle Long-Term-Support-(LTS-)Version von Node.js installieren möchten. Diese Version wird normalerweise empfohlen, da sie Stabilität, Zuverlässigkeit und Sicherheit über neue oder experimentelle Funktionen stellt. Sie ist gründlich getestet und produktionsbereit. Außerdem ist garantiert, dass sie längere Support-Zeiträume mit Fokus auf Bugfixes und Sicherheitsupdates hat.

Alternativ können Sie eine bestimmte Version von Node.js installieren, indem Sie die Versionsnummer im Befehl angeben:

```bash
nvm install <version>
```

Zum Beispiel so können Sie Node.js v20 installieren:

```bash
nvm install 20
```

Nachdem Sie eine Version von Node.js installiert haben, können Sie mit diesem Befehl zu dieser Version wechseln:

```bash
nvm use <version>
```

Zum Beispiel so wechseln Sie zu Version 20:

```bash
nvm use 20
```

Schließlich können Sie mit diesem Befehl die aktuelle Version von Node.js überprüfen:

```bash
node -v
```

## Wie man NVM auf Windows installiert

Sehen wir uns nun an, wie Sie NVM auf Windows installieren und verwenden können.

Offizielle Versionen von NVM unterstützen Windows nur in bestimmten Fällen. Wenn Sie das Windows Subsystem for Linux (WSL) auf Ihrem Gerät eingerichtet haben, können Sie der oben genannten macOS/Linux-Anleitung folgen, um NVM zu installieren und zu verwenden.

Für Windows selbst gibt es jedoch einige Alternativen. Eine davon ist `nvm-windows`, ein Node.js-Versionsverwaltungs-Tool für Windows. Dies ist ein völlig separates Projekt vom offiziellen NVM-Projekt. Aber der Zweck dieser Tools ist derselbe – Ihnen zu helfen, mit mehreren Versionen von Node.js auf demselben Gerät zu arbeiten.

Bevor Sie `nvm-windows` installieren, sollten Sie wissen, dass das offizielle Repository empfiehlt, alle vorherigen Node-Installationen zu deinstallieren, bevor Sie NVM für Windows installieren. Wenn dies Ihre erste Node.js-Installation ist, sollten Sie in Ordnung sein. Wenn Sie bereits andere Versionen von Node.js installiert haben, wird empfohlen, diese zuerst zu deinstallieren.

Um `nvm-windows` zu installieren:

- Gehen Sie zum Releases-Abschnitt des offiziellen Repositorys: `https://github.com/coreybutler/nvm-windows/releases`.
- Laden Sie den neuesten Installer herunter.
- Folgen Sie den Anweisungen.  

Nach Abschluss der Installation müssen Sie möglicherweise Ihr Terminal neu starten, falls der `nvm`-Befehl nicht sofort verfügbar ist.

Sie können überprüfen, ob die Installation erfolgreich war, und Ihre aktuelle Version von `nvm-windows` sehen, indem Sie diesen Befehl ausführen:

```powershell
nvm -v
```

## Wie man Node.js mit NVM für Windows installiert

Jetzt, wo Sie `nvm-windows` auf Windows installiert haben, sind Sie bereit, Node.js zu installieren.

Um die neueste Long-Term-Support-(LTS-)Version von Node.js zu installieren, führen Sie diesen Befehl aus:

```powershell
nvm install lts
```

Die Long-Term-Support-(LTS-)Version wird normalerweise empfohlen, aber Sie können eine andere Version installieren, indem Sie die Versionsnummer im Befehl angeben:

```powershell
nvm install <version>
```

Zum Beispiel:

```powershell
nvm install 20
```

Wenn die Installation abgeschlossen ist, können Sie mit diesem Befehl zu dieser Node.js-Version wechseln:

```powershell
nvm use <version>
```

Zum Beispiel:

```powershell
nvm use 20
```

Sie können auch Ihre aktuelle Version von Node.js mit diesem Befehl überprüfen:

```bash
node -v
```

## Grundlegende Befehle

Jetzt, wo Sie wissen, wie man NVM und Node.js installiert, sehen wir uns einige grundlegende Befehle an, die Ihnen bei der Arbeit mit Node.js und NVM hilfreich sein werden.

Obwohl all diese Befehle für NVM auf macOS/Linux, einschließlich WSL, funktionieren, können sie für `nvm-windows` anders funktionieren oder gar nicht funktionieren. Wir geben alternative Befehle für `nvm-windows`, wann immer möglich.

### Wie man die Ausführungsrichtlinie von Powershell anpasst

Wenn Sie Windows verwenden und mit Powershell und `nvm-windows` arbeiten, müssen Sie möglicherweise die Ausführungsrichtlinie von Powershell anpassen. Die Standard-Ausführungsrichtlinie für Powershell ist `Restricted`, eine Sicherheitsmaßnahme, um das Ausführen von Skripten zu verhindern.

Führen Sie zuerst Powershell als Administrator aus, indem Sie mit der rechten Maustaste auf Powershell klicken und dann „Als Administrator ausführen“ wählen.

Um die aktuelle Ausführungsrichtlinie der aktuellen Powershell-Sitzung zu sehen, führen Sie den folgenden Befehl aus:

```powershell
Get-ExecutionPolicy
```

Um schließlich die Ausführungsrichtlinie von Powershell auf `RemoteSigned` zu setzen, die lokale Skripte und `npm` ausführen lässt, verwenden Sie den folgenden Befehl:

```powershell
Set-ExecutionPolicy RemoteSigned
```

Nachdem Sie den obigen Befehl ausgeführt haben, können Sie das Administratorfenster von Powershell schließen und ein nicht-administratives Powershell-Fenster öffnen.

### Alle Versionen auflisten

Um alle installierten Versionen von Node.js aufzulisten und die aktuell verwendete zu überprüfen, führen Sie diesen Befehl aus:

```bash
nvm ls
```

### Standard-Node.js-Version festlegen (nur NVM auf macOS/Linux)

Dieser Befehl legt die Standardversion von Node.js fest, die für neue Terminal-Sitzungen verwendet wird:

```bash
nvm alias default <version>
```

Zum Beispiel:

```bash
nvm alias default 22.20.0
```

Sie können auch benutzerdefinierte Aliase erstellen, um einer Version einen spezifischeren Namen zuzuweisen:

```bash
nvm alias <new_name> <version>
```

Zum Beispiel:

```bash
nvm alias new-api 18.20.8
```

Nachdem Sie den Alias eingerichtet haben, können Sie diesen neuen Namen verwenden, um zu dieser Version zu wechseln:

```bash
nvm use <new_name>
```

Zum Beispiel:

```bash
nvm use new-api
```

### Eine Node.js-Version entfernen

Um eine Version von Node.js zu entfernen, führen Sie diesen Befehl aus:

```bash
nvm uninstall <version>
```

Zum Beispiel:

```bash
nvm uninstall 20.19.5
```

Mit NVM für macOS/Linux können Sie eine Version von Node.js entfernen, indem Sie den `uninstall`-Befehl mit einer Hauptversion wie `22` oder `20` ausführen. Zum Beispiel:

```bash
nvm uninstall 20
```

Wenn Sie mehrere Releases für diese Hauptversion installiert haben, wie `20.14.0` und `20.12.2`, wird zuerst die neueste Version entfernt. In diesem Fall entfernt das Ausführen von `nvm uninstall 20` zuerst `20.14.0`.

### Ihre Node.js-Version überprüfen

Um Ihre aktuelle Version von Node.js zu überprüfen, können Sie Folgendes verwenden:

```bash
node -v
```

### Ihre npm-Version überprüfen

`npm` ist ein Paketmanager für Node.js. Sie können ihn verwenden, um Softwarepakete zu installieren, zu veröffentlichen und zu verwalten. Er wird automatisch installiert, wenn Sie Node.js installieren.

Um Ihre aktuelle Version von `npm` zu überprüfen, führen Sie diesen Befehl aus:

```bash
npm -v
```

### Eine JavaScript-Datei ausführen

Dieser Befehl führt die JavaScript-Datei aus, die Sie in der aktuell aktiven Node.js-Laufzeit angeben:

```bash
node <file.js>
```

Zum Beispiel:

```bash
node app.js
```

### Ein Node.js-Projekt erstellen

Dieser Befehl erstellt ein neues Node.js-Projekt, indem er eine `package.json`-Datei erstellt, die dessen Details und Abhängigkeiten verfolgt.

```bash
npm init
```

Wenn Sie den `npm init`-Befehl ausführen, führt er Sie durch eine interaktive Befehlszeilenschnittstelle zur Einrichtung der `package.json`-Datei. Er stellt Ihnen Fragen zum Projektnamen, zur Beschreibung, Version usw. Wenn Sie mit einer Standard-`package.json`-Datei starten möchten, können Sie den `npm init -y`- oder `npm init --yes`-Befehl verwenden.

### Ein Paket installieren

Um ein bestimmtes Paket zu installieren, führen Sie aus:

```bash
npm install <package>
```

Zum Beispiel:

```bash
npm install express
```

### Abhängigkeiten installieren

Um alle in der `package.json`-Datei eines Projekts aufgeführten Abhängigkeiten zu installieren, führen Sie aus:

```bash
npm install
```
