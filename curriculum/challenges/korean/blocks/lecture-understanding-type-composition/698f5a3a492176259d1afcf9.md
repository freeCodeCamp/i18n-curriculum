# --description--

이전 수업에서 타입 별칭을 다루는 방법과 객체 타입에 이름을 붙이는 예시를 배웠습니다.

기억하시겠지만, 객체 타입에 이름을 붙이는 한 가지 방법은 `type` 키워드를 사용하는 것입니다:

```ts
type ID = number | string;

type User = {
  id: ID;
  name: string;
};
```

또 다른 방법은 `interface` 선언을 사용하는 것입니다:

```ts
type ID = number | string;

interface User {
  id: ID;
  name: string;
}
```

`type`와 `interface`는 비슷하지만 한 가지 중요한 차이점은 인터페이스가 `extends` 키워드를 사용해 다른 인터페이스를 확장하는 반면, `type` 별칭은 교차 타입(`&`)을 사용해 타입을 결합한다는 점입니다.

몇 가지 예시를 살펴보겠습니다:

```ts
interface User {
  id: number | string;
  name: string;
}

interface Admin extends User {
  role: string;
}

const adminUser: Admin = {
  id: 101,
  name: "Alice",
  role: "superadmin"
};
```

이 예시에서 `User`는 `id`와 `name`로 시작합니다. `Admin` 인터페이스는 `User`를 확장하는데, 이는 `id`와 `name`를 포함하고 `role`라는 새 속성을 추가한다는 뜻입니다.

`type`를 확장하고 싶다면, 다음과 같이 교차 타입(`&`)을 사용합니다:

```ts
type User = {
  id: number | string;
  name: string;
};

type Admin = User & {
  role: string;
};

const adminUser: Admin = {
  id: 101,
  name: "Alice",
  role: "superadmin"
};
```

`Admin`는 `User` 타입과 `role`를 포함하는 새 객체 타입을 결합합니다. 결과는 인터페이스 예시와 동일한 형태로, `id`, `name`, `role`를 가진 객체입니다.

`type`와 `interface` 모두 객체의 형태를 설명할 수 있습니다. 어떤 것을 사용해야 할지 확실하지 않다면, `interface`는 객체 형태를 위해 특별히 설계되었고, `type`는 더 유연하여 유니언 타입, 원시 타입, 더 복잡한 타입 조합도 나타낼 수 있다는 점을 기억하세요.
