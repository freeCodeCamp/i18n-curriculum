# --description--

Stacks und Queues sind Datenstrukturen, die häufig in der Informatik verwendet werden.

Sie sind lineare Datenstrukturen, die bestimmten Regeln zum Hinzufügen und Entfernen von Elementen folgen.

## Stacks

Beginnen wir mit **Stacks**.

Ein **Stack** ist eine Last-in, First-out (LIFO) Datenstruktur.

Das bedeutet, dass das zuletzt zum Stack hinzugefügte Element als erstes entfernt wird.

Stacks haben zwei Enden, die wir als oben und unten kennen.

Elemente werden oben auf den Stack hinzugefügt und von dort entfernt.

Man kann sich einen Stack wie einen Stapel von Tellern vorstellen, bei dem man Teller nur oben auf den Stapel legen und von oben nehmen kann.

<img src="https://cdn.freecodecamp.org/curriculum/lecture-transcripts/how-do-stacks-and-queues-work-1.png" alt="stack data structure visualization">

Diese Operationen des Hinzufügens und Entfernens von Elementen haben in diesem Zusammenhang spezielle Namen.

Das Hinzufügen eines Elements zu einem Stack wird als „push“-Operation bezeichnet. Wir sagen, dass wir ein Element auf den Stack „pushen“, wenn wir es oben auf den Stack legen.

<img src="https://cdn.freecodecamp.org/curriculum/lecture-transcripts/how-do-stacks-and-queues-work-2.png" alt="stack push operation visualization">

Das Entfernen eines Elements von einem Stack wird als „pop“-Operation bezeichnet. Wir sagen, dass wir ein Element vom Stack „poppen“, wenn wir es von oben vom Stack nehmen.

<img src="https://cdn.freecodecamp.org/curriculum/lecture-transcripts/how-do-stacks-and-queues-work-3.png" alt="stack pop operation visualization">

Sie sehen, dass wir am unteren Ende des Stacks eigentlich keine Operationen durchführen, es aber als Referenz behalten.

Die Zeitkomplexität der push- und pop-Operationen ist typischerweise `O(1)`, eine konstante Zeitkomplexität.

Wenn Sie ein Element auf den Stack pushen, wird das Element einfach oben hinzugefügt.

Wenn Sie ein Element vom Stack poppen, wird das Element oben entfernt.

Daher bleibt die Zeit, die für diese Operationen benötigt wird, unabhängig von der Größe des Stacks konstant.

Die Speicherkomplexität der push- und pop-Operationen ist normalerweise konstant `O(1)`. Das bedeutet, dass die Menge an Speicher, die für diese Operationen benötigt wird, unabhängig von der Größe des Stacks konstant bleibt.

## Queues

Jetzt, wo Sie mehr über Stacks wissen, lernen wir **Queues** kennen.

Eine Queue ist eine First-in First-out (FIFO) lineare Datenstruktur. Das bedeutet, dass das zuerst zur Queue hinzugefügte Element als erstes entfernt wird.

Queues haben zwei Enden: vorne und hinten.

<img src="https://cdn.freecodecamp.org/curriculum/lecture-transcripts/how-do-stacks-and-queues-work-4.webp" alt="queue data structure visualization">

Elemente werden hinten an die Queue angefügt und vorne aus der Queue entfernt.

Man kann sich eine Queue wie eine Warteschlange von Personen vorstellen, die an der Supermarktkasse bezahlen wollen. Die erste Person in der Schlange ist die erste, die zur Kasse geht, während neue Personen hinten an der Schlange dazukommen.

Die Operationen des Hinzufügens und Entfernens von Elementen haben im Kontext einer Queue spezielle Namen.

Das Hinzufügen eines Elements hinten an eine Queue wird als „enqueue“-Operation bezeichnet.

Bei einer enqueue-Operation wird das neue Element ans Ende der Queue angefügt und wird somit das Ende der Schlange.

<img src="https://cdn.freecodecamp.org/curriculum/lecture-transcripts/how-do-stacks-and-queues-work-5.png" alt="queue enqueue operation visualization">

Das Entfernen eines Elements vorne aus der Queue wird als „dequeue“-Operation bezeichnet.  
  
Bei der dequeue-Operation wird das Element vorne aus der Queue entfernt, und das nächste Element in der Schlange wird zum neuen Anfang.

<img src="https://cdn.freecodecamp.org/curriculum/lecture-transcripts/how-do-stacks-and-queues-work-6.png" alt="queue dequeue operation visualization">

Die Zeitkomplexität der enqueue- und dequeue-Operationen ist `O(1)`, konstante Zeit. Die Zeit, die für diese Operationen benötigt wird, bleibt unabhängig von der Größe der Queue konstant.

Die Speicherkomplexität der enqueue- und dequeue-Operationen ist normalerweise konstant `O(1)`. Das bedeutet, dass die Menge an Speicher, die für diese Operationen benötigt wird, unabhängig von der Größe der Queue konstant bleibt.

Stacks und Queues sind Datenstrukturen, die in der Informatik zum Organisieren und Verwalten von Elementen verwendet werden. Sie zu verstehen ist essenziell, um effiziente Algorithmen in verschiedenen Programmieranwendungen zu erstellen.
