# --description--

リポジトリをGitHubにプッシュする前に、認証が必要です。SSHキーとGPGキーの両方は公開鍵と秘密鍵のペアを使います。つまり、秘密鍵はローカルマシンに保持し、それを使って自分自身を認証します。公開鍵は認証を検証する必要がある人やサービスと共有します。言い換えれば、秘密鍵は操作を実行するために必要で、公開鍵はその操作を検証するために使われます。

GPG（Gnu Privacy Guard）キーは通常、ファイルやコミットに署名するために使われます。誰かがあなたの公開GPGキーを使って、そのファイルの署名があなたのキーによるものであり、ファイルの内容が改ざんされていないことを検証できます。SSH（Secure SHell）キーは通常、`ssh`ユーティリティを使ってサーバーへのリモート接続を認証するために使われます。しかし、このレッスンで学ぶように、SSHキーを使ってコミットに署名することもできます。

GPGキーを生成するには、次のコマンドを実行してください：

```sh
gpg --full-generate-key
```

SSHキーの場合は、次のコマンドを実行します：

```sh
ssh-keygen -t ed25519 -C "your_email@example.com"
```

実際のメールアドレスを必ず入力してください。

これらのコマンドはどちらもキーを保護するためのパスフレーズを求められます。これは任意ですが、強くお勧めします。

SSHキーの場合、秘密鍵用と公開鍵用の2つのファイルが生成されます。公開鍵のファイル名は`.pub`で終わるので、秘密鍵を誤って共有しないようになっています。これらのファイルは`~/.ssh`に保存されます。

一方、GPGキーは隠しかつ暗号化されたディレクトリに保存されます。代わりに、公開鍵の一覧を表示するには次のコマンドを実行します：

```sh
gpg --list-secret-keys --keyid-format=long
```

公開鍵を取得するには、次のコマンドを使います：

```sh
gpg --armor --export "<key id>"
```

では、これらの新しいキーで何ができるのでしょうか？コミットに署名することができ、これはどちらのキーでも使えます。

GPGキーでコミットに署名するには、秘密鍵ではなく公開鍵をGitHubアカウントにアップロードする必要があります。次に、先ほどキーの一覧表示で得た短いIDを使い、gitの署名キーとして設定するには次のコマンドを実行します：

```sh
git config --global user.signingkey <your_gpg_key_id>
```

その後、特定のコミットに署名するには、`-S`フラグを`git commit`コマンドに渡します。パスフレーズの入力が必要です。あるいは、すべてのコミットに自動的に署名したい場合は、autosign設定を`true`に設定してください：

```sh
git config --global commit.gpgsign true
```

SSHキーで署名する場合は、GitHubで比較的新しい機能ですが、まずキーをGitHubアカウントにアップロードします。次に、gitの署名モードをSSHに設定します：

```sh
git config --global gpg.format ssh
```

署名キーを設定するには、IDの代わりにファイルパスを渡します：

```sh
git config --global user.signingkey <path_to_your_ssh_keys>
```

最後に、GPGキーと同様に自動署名を有効にできます。

これらのどちらかの方法を設定すれば、GitHubで特別なバッジが付く検証済みコミットができるようになります。
