# --description--

전문 개발자는 생산성과 코드 품질을 높이기 위해 다양한 도구를 사용합니다. 아주 명확해 보이는 도구들도 포함해서 그런 도구들을 함께 알아보겠습니다.

도구 중 가장 먼저 꼽히는 것은 컴퓨터입니다. 컴퓨터는 기본 개발 환경입니다. 무거운 데스크톱일 수도 있고, Windows, macOS, Linux 운영체제를 사용하는 휴대용 노트북일 수도 있습니다.

전문 개발자는 자원 집약적인 작업을 처리하기 위해 빠른 처리 속도와 충분한 RAM을 갖춘 컴퓨터를 선호합니다.

컴퓨터 다음으로는 그 컴퓨터에서 직접 코드를 작성하고 편집하는 프로그램이 있습니다. 그것이 코드 편집기 또는 통합 개발 환경(IDE)입니다.

Visual Studio, IntelliJ IDEA, JetBrains, PyCharm 같은 IDE는 코드 완성, 디버깅, 통합 터미널 지원 같은 강력한 기능을 제공합니다.

Visual Studio Code(VS Code)는 본질적으로 코드 편집기이지만, 풍부한 확장 라이브러리를 통해 이러한 기능도 제공합니다.

다른 코드 편집기로는 Sublime Text, Atom, Notepad++, Brackets가 있습니다.

코드 편집기와 IDE로 코드를 작성할 때, 변경 사항을 추적해야 합니다. 변경 사항을 추적할 수 있게 해 주는 도구가 버전 관리 시스템입니다.

Git은 개발 커뮤니티에서 가장 널리 사용되는 버전 관리 시스템입니다.

GitHub, GitLab, Bitbucket 같은 플랫폼은 Git 저장소를 위한 클라우드 기반 호스팅 서비스를 제공합니다. 이 플랫폼들은 다른 개발자와 협업할 수 있게 하고, 여러 브랜치에서 작업하며, 코드 변경 사항을 병합할 수 있게 합니다.

패키지 관리자는 또 다른 중요한 도구입니다. 라이브러리와 프로젝트 의존성을 추가, 업데이트, 제거하는 과정을 단순화하는 데 도움을 줍니다.

인기 있는 패키지 관리자 예시는 다음과 같습니다:

* JavaScript용 NPM, Yarn, PNPM
* Python용 PIP
* PHP용 Composer
* Java용 Maven

다양한 도구로 코드를 작성한 후, 개발자는 코드가 예상대로 작동하는지 테스트합니다.

이를 위해 Cypress, Playwright, Selenium 같은 테스트 프레임워크를 사용합니다. 언어별 테스트 프레임워크도 있습니다. 예시는 다음과 같습니다:

* JavaScript용 Jest
* Python용 pytest
* Java용 JUnit
* PHP용 PHPUnit

코드가 예상대로 작동하는지 테스트하는 것뿐 아니라, 최종 사용자가 보는 시각적 모습도 테스트해야 합니다. 개발자는 이를 위해 웹 브라우저를 사용합니다.

Chrome, Firefox, Edge, Safari 같은 최신 브라우저는 HTML, CSS, JavaScript 코드를 검사할 수 있는 개발자 도구를 제공합니다. 디버깅과 성능 프로파일링 도구도 있습니다.

이 도구들은 개발자가 다양한 브라우저에서 웹 애플리케이션을 테스트하고 최적화하는 데 도움을 줍니다.
