# --description--

Express.js、または略してExpressは、Node.jsの**上に構築された**最小限で柔軟なウェブフレームワークです。ウェブサーバーやAPIの構築を簡単にします。

Nodeのネイティブ`http`モジュールで大量のボイラープレートコードを書く代わりに、Expressを使えばわずか数行で堅牢なウェブアプリケーションを作成できます。

Node.jsをより使いやすく、より強力にする高速で自由度の高いレイヤーと考えてください。

開発者がExpressを愛用する主な理由は次の3つです：

1. **最小限で高速** – Expressは必要最低限の機能だけを提供し、ミドルウェアやプラグインを通じて必要に応じて機能を追加できるため、完全なコントロールが可能です。

2. **巨大なエコシステム** – ロギング、認証、セキュリティ、セッションなどのための膨大なミドルウェアやプラグインのコレクションがあります。

3. **APIに最適** – シンプルなルーティングとリクエスト処理モデルにより、RESTful APIの構築に特に適しています。

Expressを使えば、以下のようなものを作成できます：

* ウェブアプリ

* RESTful API

* シングルページアプリケーション（SPA）のバックエンド

* ミドルウェアを多用するサービス

* HTTPを使ったサーバーサイドのあらゆるもの


## Expressのインストール方法

まず、Node.jsとnpmがインストールされていることを確認してください。nodeとnpmがインストールされているか確認する方法は以下の通りです：

```bash
node -v
npm -v
```

バージョン番号が表示されればインストール済みです。表示されなければインストールが必要です。インストール手順は以前のレッスンか公式Node.jsウェブサイトで確認できます。

次に、ターミナルで以下のコマンドを入力してください：

```bash
mkdir my-app
cd my-app
npm init -y
npm install express
```

ミドルウェアやプラグインについては今後のレッスンで詳しく学びますが、簡単に言うと、ミドルウェアはリクエストを受け取ってレスポンスを送る間に実行されるコードです。エラーハンドリング、入力のバリデーション、リクエストやレスポンスのデータ修正などに使われます。ミドルウェアを連結して柔軟で再利用可能なリクエスト処理ロジックを構築できます。

プラグインは通常、Expressの機能を拡張する外部のサードパーティ製Node.jsモジュールです。npmで別途インストールでき、多くはミドルウェアとして実装されており、少しの設定でアプリに組み込めます。共通機能を一から書く代わりに、ロギング、認証、セキュリティのベストプラクティスなどをプラグインで処理できます。

ExpressはNode.jsのサーバーサイド開発で人気のフレームワークです。サーバーの立ち上げ、ルートの管理、ミドルウェアの扱い、APIの構築を素早く行えるビルディングブロックを提供し、コードはシンプルで読みやすいです。
