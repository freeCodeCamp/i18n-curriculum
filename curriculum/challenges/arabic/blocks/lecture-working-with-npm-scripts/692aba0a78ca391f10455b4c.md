# --description--

الآن بعد أن فهمت نظام وحدات CommonJS، دعنا نلقي نظرة على نظام وحدات ES.

وحدات ES، أو وحدات ECMAScript، هي صيغة الوحدة الرسمية والموحدة لجافاسكريبت حسب مواصفة ECMAScript. تستخدم بيانات `import` و`export`، وفي Node.js تُحمّل بشكل غير متزامن، مما يجعلها أكثر كفاءة.

يتعامل Node.js مع الملفات كوحدات ES في الحالات التالية:

* الملفات التي تحمل امتداد `.mjs`.
    
* والملفات التي تحمل امتداد `.js` عندما يحتوي أقرب ملف `package.json` أبوي على حقل من المستوى الأعلى باسم `"type"` وقيمته `"module"`.
    

هنا لدينا نفس دالة `multiply` التي رأيتها سابقًا، لكننا الآن سنستخدم نظام ESM. لهذا، نُعيّن امتداد `.mjs` لكلتا الوحدتين.

لتصدير دالة `multiply` من وحدة `math.mjs`، استخدم الكلمة المفتاحية `export` متبوعة باسم الدالة أو المتغير أو الكائن الذي تريد تصديره داخل أقواس متعرجة:

```javascript
function multiply(a, b) {
  return a * b;
}

export { multiply };
```

يمكنك أيضًا تصدير الدالة مباشرة باستخدام الكلمة المفتاحية `export` قبل كلمة `function`. يُعرف هذا بالتصدير المسمي. يمكنك أن تملك عدة تصديرات مسمية في الوحدة الواحدة:

```javascript
export function multiply(a, b) {
  return a * b;
}
```

بعد ذلك، لاستيراد هذه الوحدة، استخدم الكلمة المفتاحية `import` متبوعة بأقواس متعرجة. لاحظ أننا نستخدم `import` بدلًا من `require()`. هذه واحدة من الفروقات الرئيسية بين CommonJS ووحدات ES.

داخل الأقواس المتعرجة، نكتب اسم الدالة أو المتغير أو الكائن الذي نريد استيراده. في هذه الحالة، هو `multiply`. ثم نستخدم الكلمة المفتاحية `from` والمسار النسبي للوحدة. ستعمل هذه الشفرة بنجاح وسيكون الناتج `42`.

```javascript
import { multiply } from './math.mjs';

console.log(multiply(6, 7)); // 42
```

في نظام ES، يمكنك تعيين تصدير افتراضي واحد لكل وحدة. بهذه الطريقة، لا تحتاج لاستخدام الأقواس المتعرجة لاستيراده.

إليك مثالًا حيث نُعيّن دالة `multiply` كتصدير افتراضي لوحدة `math.mjs` بأكملها:

```javascript
export default function multiply(a, b) {
  return a * b;
}
```

الآن لا نحتاج الأقواس المتعرجة، لأن الدالة ستُستورد وتُعيّن افتراضيًا:

```javascript
import multiply from './math.mjs';
```

إذا لم تستخدم امتدادات الملفات أو ملف `package.json` لتخبر Node.js صراحةً نوع نظام الوحدة الذي تريد استخدامه، فسيُفحص الكود المصدري بحثًا عن صيغة وحدات ES. إذا وُجدت هذه الصيغة، سيُشغّل الكود كوحدة ES؛ وإلا، سيُشغّله كوحدة CommonJS.

بينما كان CommonJS هو المعيار الافتراضي لـ Node.js في بداياته، ومع اعتماد المطورين المتزايد لـ ESM لكفاءته وتوافقه مع جافاسكريبت الحديثة، من المهم معرفة الفروقات بينهما وكيفية استخدام كلا النظامين.
