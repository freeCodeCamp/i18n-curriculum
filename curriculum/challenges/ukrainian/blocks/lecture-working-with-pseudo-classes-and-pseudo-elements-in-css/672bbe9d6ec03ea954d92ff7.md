# --interactive--

Зовнішній вигляд і поведінка елементів input мають значення при створенні веб-форм. Форма з input, які реагують на дії користувача, значно покращує користувацький досвід і не повинна викликати плутанину чи розчарування.

CSS надає кілька псевдокласів для input, які можуть зробити ваші форми більш інтуїтивними та зручними для користувача. Розглянемо ці псевдокласи детальніше.

Псевдоклас `:focus` дозволяє націлитися на елемент input, коли користувач вибирає або натискає на нього, привертаючи увагу до активного поля введення. Це допомагає користувачам легко визначити, де вони зараз вводять текст:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css" />
<form>
  <input type="text" />
</form>
```

```css
input:focus {
  border: 2px solid crimson;
  outline: none;
}
```

:::

Як випливає з назви, псевдоклас `:hover` спрацьовує, коли користувач наводить курсор на елемент. Він надає візуальний зворотний зв’язок перед взаємодією з input, ефективно повідомляючи, що поле готове до дії.

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css" />
<form>
  <input type="text" />
</form>
```

```css
input:hover {
  background-color: orange;
}
```

:::

У наведеному вище прикладі фон елемента input змінюється на помаранчевий колір, коли користувач наводить на нього курсор, даючи зрозуміти, що поле готове до взаємодії.

Псевдоклас `:checked` дозволяє стилізувати радіокнопку або прапорець інакше, коли користувач їх вибирає. Таким чином користувач може легко побачити, який варіант він обрав.

Ось приклад із прапорцем для згоди з умовами на сайті.

**ПРИМІТКА**: Частина CSS у цьому прикладі використовує властивості, які ще не були розглянуті. Це лише для того, щоб дати уявлення про створення кастомного прапорця. Ви дізнаєтеся, як це все працює, у майбутніх уроках і практичних заняттях.

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css" />
<form>
  <label>
  Agree <input class="checkbox" type="checkbox" />
  </label>
</form>
```

```css
.checkbox {
  appearance: none;
  width: 18px;
  height: 18px;
  border: 2px solid #ccc;
  border-radius: 4px;
  display: inline-block;
  position: relative;
  cursor: pointer;
  transition: all 0.25s ease;
  vertical-align: middle; 
}

.checkbox:hover {
  border-color: #888;
}

.checkbox:checked {
  background-color: #4caf50;
  border-color: #4caf50;
}

.checkbox:checked::after {
  content: "";
  position: absolute;
  left: 4px;
  top: 0px;
  width: 5px;
  height: 10px;
  border: solid white;
  border-width: 0 2px 2px 0;
  transform: rotate(45deg);
}

.checkbox:focus {
  outline: 2px solid #90caf9;
  outline-offset: 2px;
}

```

:::

`:required` націлюється на input елементи, які мають атрибут `required`. Це сигналізує користувачу, що поле потрібно заповнити, щоб відправити форму.

Ось приклад форми з деякими обов’язковими полями введення:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css" />
<form action="#">
  <label for="name">Name:</label>
  <input type="text" id="name" name="name" required />

  <label for="email">Email:</label>
  <input type="email" id="email" name="email" required />

  <label for="phone">Phone Number:</label>
  <input type="tel" id="phone" name="phone" />

  <button type="submit">Submit</button>
</form>
```

```css
input:required {
  border: 2px solid orange;
}
```

:::

Під час валідації форм ви можете використовувати псевдоклас `:valid` для стилізації полів введення, які відповідають критеріям валідації, і `:invalid` для стилізації полів, які не відповідають критеріям. Зазвичай для валідного введення використовують зелений колір, а для невалідного — червоний.

Ось приклад із використанням псевдокласу `:valid`:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css" />
<form>
  <label for="email">Email:</label>
  <input type="email" id="email" name="email" />
</form> 
```

```css
input:valid {
  border-color: green;
}
```

:::

Ось приклад використання псевдокласу `:invalid`:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css" />
<form>  
  <label for="email">Email:</label>
  <input type="email" id="email" name="email" />
</form>
```

```css
input:invalid {
  border-color: crimson;
}
```

:::

Псевдоклас `:disabled` дозволяє вибирати та стилізувати input елементи, які вимкнені. Ці елементи мають атрибут `disabled`, що забороняє користувачам взаємодіяти з ними. Коли input вимкнений, його не можна натиснути, сфокусувати або редагувати.

Ось приклад мітки та вимкненого input елемента.

У CSS ми націлюємося на цей вимкнений input і задаємо йому фон кольору `lightgray` та змінюємо курсор на `not-allowed`.

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css" />
<form>
  <label for="name">Name:</label>
  <input class="text-input" type="text" id="name" name="name" disabled />
</form>
```

```css
.text-input:disabled {
  background-color: lightgray;
  cursor: not-allowed;
}
```

:::

Значення CSS-властивості `cursor: not-allowed;` змінює вигляд курсора, щоб показати, що дія заборонена.

Коли застосовується до елемента, воно показує курсор із символом кола зі слешем (зазвичай коло з діагональною лінією), що сигналізує користувачам, що елемент неінтерактивний або на нього не можна натиснути чи взаємодіяти.

Ось ще кілька прикладів псевдокласів для input і що вони роблять:

- `:autofill`: застосовує стилі до полів введення, які браузер автоматично заповнює збереженими даними.
- `:optional`: стилізує input елементи, які не є обов’язковими і можуть залишатися порожніми.
- `:in-range` і `:out-of-range`: стилізують елементи залежно від того, чи їхні значення знаходяться в межах або поза межами заданих обмежень.
