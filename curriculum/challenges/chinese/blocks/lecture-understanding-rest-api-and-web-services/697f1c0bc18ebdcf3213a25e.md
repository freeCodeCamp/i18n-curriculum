# --description--

**REST** 代表表现层状态转移。可以将其视为一组指导原则，描述了网络客户和服务器如何通过互联网相互通信。

在本课中，你将学习什么是 REST、它的关键概念以及它是如何工作的。

REST 定义了一种通过使用标准网页技术让不同系统交互的方式。通过 REST，诸如网页浏览器、移动应用和 API 等客户使用常见的超文本传输协议 `GET`、`POST`、`PUT`、`PATCH` 和 `DELETE` 方法与服务器通信，这些方法我们在之前的课程中讨论过。

REST 的一些核心概念包括**无状态性**、**资源**、**HTTP methods**和**响应或状态代码**。

**无状态性** 意味着客户发送到服务器的每个请求都是独立处理的。服务器不会记住之前的请求。正因为如此，每个请求必须包含处理它所需的所有信息。例如，如果用户已通过认证，则他们的认证令牌必须随每个请求一起发送。

在 REST 中，一切都被视为资源。资源是一段可以被访问或修改的数据。每个资源都由一个 URL 标识，例如 `/products` 表示所有产品，或 `/products/1` 表示特定产品。

以下是超文本传输协议方法、状态代码的概述，以及 REST 架构工作原理的逐步示例。

## 超文本传输协议 方法

超文本传输协议 方法定义对资源执行的操作：

* `GET`：检索资源
    
* `POST`：创建一个新资源
    
* `PUT / PATCH`：更新现有资源
    
* `DELETE`：删除资源
    

## 超文本传输协议 `Status` 码

在处理完一个 `request` 后，`server` 会以一个表示结果的 **状态码** 响应：

* `200` **（成功）**：请求成功
    
* `201` **（已创建）**：已创建一个新资源
    
* `400` **（错误请求）**：客户发送了无效的数据
    
* `404` **（未找到）**：请求的资源不存在
    
* `500` **（内部服务器误差）**：服务器上发生了错误
    

## REST 的工作原理：逐步示例

### 1\. 客户发送一个请求

* 客户使用诸如 `GET` 或 `POST` 的方法发送超文本传输协议请求
    
* 该请求包含标识资源的 URL
    
* 它还可能包括头部、元数据和请求体
    

### 2\. 服务器接收请求

* 服务器验证请求形式和认证（如果需要）
    
* 它从 URL 中识别目标资源
    
* 它根据 HTTP `method` 确定请求的操作
    
* 它检查权限和访问控件
    

### 3\. 服务器执行该操作

* `GET`：检索资源
    
* `POST`：创建一个新资源
    
* `PUT / PATCH`：更新资源
    
* `DELETE`：删除资源
    
* 执行任何其他必要的逻辑
    

### 4\. 服务器构建响应

* 服务器选择一个合适的状态码（例如，`200`、`201` 或 `404`）
    
* 它格式化响应 `data`，通常为 **JSON**
    
* 它添加相关的头部，例如 `Content-Type` 或 `Cache-Control`
    

### 5\. 服务器发送响应

* 响应包括 `status` 码、`header` 和可选的 `body`
    
* 服务器上不保存任何客户状态（无状态）
    

### 6\. 客户处理器响应

* 客户查看 `status code` 以确定成功或失败
    
* 它处理服务器返回的任何数据
