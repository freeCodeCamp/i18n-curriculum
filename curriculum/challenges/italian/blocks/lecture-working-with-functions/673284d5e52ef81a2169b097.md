# --interactive--

Nella lezione precedente, hai imparato a lavorare con le funzioni, che sono pezzi di codice riutilizzabili che aiutano a rendere il tuo codice più modulare, più facile da mantenere e più efficiente. Tutti gli esempi precedenti usavano la sintassi regolare delle funzioni, come questa:

```js
function greetings(name) {
  console.log("Hello, " + name + "!");
}
```

Ma un altro modo per scrivere funzioni in JavaScript è creare un’espressione di funzione freccia. Ecco come puoi rifattorizzare l’esempio precedente per usare invece la sintassi della funzione freccia:

```js
const greetings = (name) => {
  console.log("Hello, " + name + "!");
};
```

In questo esempio rivisto, stiamo creando una variabile `const` chiamata `greetings` e assegnandole una funzione anonima. La maggior parte della sintassi ti sarà familiare tranne per la mancanza della parola chiave `function` e l’aggiunta della freccia (`=>`) tra il parametro `name` e il corpo della funzione. Se la tua lista di parametri ha solo un parametro, puoi rimuovere le parentesi tonde così:

```js
const greetings = name => {
  console.log("Hello, " + name + "!");
};
```

Se la tua funzione freccia non ha parametri, allora devi usare le parentesi tonde così:

```js
const greetings = () => {
  console.log("Hello");
};
```

Quando hai iniziato a imparare le funzioni, dovevi racchiudere il corpo della funzione nelle parentesi graffe. Ma se il corpo della funzione contiene solo una riga di codice, puoi rimuovere le parentesi graffe così:

```js
const greetings = name => console.log("Hello, " + name + "!");
```

È importante notare che rimuovere le parentesi tonde e le parentesi graffe nella sintassi regolare delle funzioni non funziona. Otterresti errori se provassi a fare qualcosa del genere:

```js
// This will produce syntax errors 
function greetings name console.log("Hello, " + name + "!");
```

Questi tipi di funzioni a riga singola funzionano solo se usi la sintassi della funzione freccia. Un altro concetto chiave è l’istruzione `return`. Ecco un esempio di utilizzo della sintassi della funzione freccia per calcolare l’area:

:::interactive_editor

```js
const calculateArea = (width, height) => {
  const area = width * height;
  return area;
};

console.log(calculateArea(5, 3)); // 15
```

:::

Stiamo creando una variabile dentro la funzione chiamata `area` e poi la restituiamo. Ma potremmo semplificare il codice e restituire direttamente il calcolo:

:::interactive_editor

```js
const calculateArea = (width, height) => {
  return width * height;
}; 

console.log(calculateArea(5, 3)); // 15
```

:::

Se provassi a rimuovere le parentesi graffe e mettere il calcolo sulla stessa riga, otterresti un messaggio di errore `Uncaught SyntaxError: Unexpected token 'return'`:

```js
const calculateArea = (width, height) => return width * height;
```

Il motivo per cui ricevi questo errore è che devi rimuovere l’istruzione `return`. Quando rimuovi quell’istruzione `return`, l’errore scompare e la funzione restituisce implicitamente il calcolo.

```js
const calculateArea = (width, height) => width * height;
```

Quindi, quando dovresti usare la sintassi della funzione freccia? Beh, dipende. Molti sviluppatori la usano sempre nei loro progetti personali. Tuttavia, quando lavori in team, la scelta di solito dipende dal fatto che il codice esistente usi funzioni regolari o funzioni freccia. Nelle lezioni future vedremo quando usare le funzioni freccia e quando evitarle.
