## Häufige Fehler in Python

- **SyntaxError**: Der Fehler, den Python auslöst, wenn Ihr Code nicht den Syntaxregeln entspricht. Zum Beispiel führt der Code `print("Hello there"` zu einem Syntaxfehler mit der Meldung `SyntaxError: '(' was never closed`, weil eine schließende Klammer fehlt.
- **NameError**: Python löst einen `NameError` aus, wenn Sie versuchen, auf eine Variable oder Funktion zuzugreifen, die Sie nicht definiert haben. Wenn Sie zum Beispiel die Zeile `print(username)` in Ihrem Code haben, ohne zuvor eine `username`-Variable definiert zu haben, erhalten Sie einen NameError mit der Meldung `NameError: name 'username' is not defined`.
- **TypeError**: Dies ist der Fehler, den Python wirft, wenn Sie eine Operation mit zwei oder mehr inkompatiblen Datentypen ausführen. Wenn Sie zum Beispiel versuchen, einen String zu einer Zahl zu addieren, erhalten Sie den Fehler `TypeError: can only concatenate str (not "int") to str`.
- **IndexError**: Sie erhalten einen `IndexError`, wenn Sie auf einen Index zugreifen, der in einer Liste oder anderen Folgen wie Tuple und String nicht existiert. In einem `Hello world`-String ist der Index des letzten Zeichens `11`. Wenn Sie versuchen, ein Zeichen auf diese Weise zuzugreifen, `greet = "hello world"; print(greet[12])`, erhalten Sie einen Fehler mit der Meldung `IndexError: string index out of range`.
- **AttributeError**: Python löst diesen Fehler aus, wenn Sie versuchen, eine Methode oder Eigenschaft zu verwenden, die in einem Objekt dieses Typs nicht existiert. Zum Beispiel führt der Aufruf von `.append()` bei einem String wie `"hello".append("!")` zu einem Fehler mit der Meldung `AttributeError: 'str' object has no attribute 'append'`.

## Gute Debugging-Techniken in Python

- **Verwendung der `print`-Funktion**: Das Einfügen von `print()`-Anweisungen an verschiedenen Stellen im Code während des Debuggens hilft Ihnen, die Werte von Variablen und den Ablauf Ihres Codes zu sehen.
- **Verwendung des eingebauten Python-Debuggers (`pdb`)**: Python stellt ein `pdb`-Modul zum Debuggen bereit. Es ist Teil der Standardbibliothek von Python und daher immer verfügbar. Mit `pdb` können Sie mit der `set_trace()`-Funktion eine Spur setzen, um interaktiv durch den Code zu gehen und Variablen zu überprüfen.
- **Nutzung von Debugging-Tools in IDEs**: Viele integrierte Entwicklungsumgebungen (IDEs) und Code-Editoren wie Pycharm und VS Code bieten Debugging-Tools mit Haltepunkten, schrittweiser Ausführung, Variableninspektion und weiteren Debugging-Funktionen.

## Ausnahmebehandlung

- **`try...except`**: Wird verwendet, um einen Codeblock auszuführen, der eine Ausnahme auslösen könnte. Der `try`-Block ist der Bereich, in dem Sie einen Fehler erwarten, während der `except`-Block eine bestimmte Ausnahme behandelt, falls dieser Fehler auftritt. Hier ein Beispiel:

  ```py
  try:
    print(22 / 0)
  except ZeroDivisionError:
    print('You can\'t divide by zero!')
    # You can't divide by zero!
  ```

  Sie können auch mehrere `except`-Blöcke verketten, um verschiedene Arten von Ausnahmen zu behandeln:

  ```py
  try:
    number = int(input('Enter a number: '))
    print(22 / number)
  except ZeroDivisionError:
    print('You cannot divide by zero!')
    # You cannot divide by zero! prints when you enter 0
  except ValueError:
    print('Please enter a valid number!')
    # Please enter a valid number! prints when you enter a string  
  ```

- **`else` und `finally`**: Diese Blöcke erweitern `try...except`. Wenn keine Ausnahme auftritt, wird der `else`-Block ausgeführt. Der `finally`-Block wird immer ausgeführt, unabhängig von Fehlern.

  ```py
  try:
    result = 100 / 4
  except ZeroDivisionError:
    print('You cannot divide by zero!') # This will not run
  else:
    print(f'Result is {result}') # Result is 25.0
  finally:
    print('Execution complete!') # Execution complete!
  ```

- **Ausnahmeobjekt**: Damit können Sie auf die Ausnahme selbst zugreifen, um besser zu debuggen und die direkte Fehlermeldung auszugeben. Um auf das Ausnahmeobjekt zuzugreifen, verwenden Sie das Schlüsselwort `as`. Hier ein Beispiel:

  ```py
  try:
      value = int('This will raise an error')
  except ValueError as e:
      print(f'Caught an error: {e}')
      # Caught an error: invalid literal for int() with base 10: 'This will raise an error'
  ```

- **Die `raise`-Anweisung**: Damit können Sie manuell eine Ausnahme auslösen. Sie können sie verwenden, um eine Ausnahme zu werfen, wenn eine bestimmte Bedingung erfüllt ist. Hier ein Beispiel:

  ```py
  def divide(a, b):
      if b == 0:
          raise ZeroDivisionError('You cannot divide by zero')
      return a / b
  ```
  
## Ausnahme-Signalisierung
  
  Die `raise`-Anweisung ist auch nützlich, wenn Sie eigene benutzerdefinierte Ausnahmen erstellen, da Sie damit eine Ausnahme mit einer benutzerdefinierten Meldung werfen können. Hier ein Beispiel dafür:

  ```py
  class InvalidCredentialsError(Exception):
      def __init__(self, message="Invalid username or password"):
          self.message = message
          super().__init__(self.message)

  def login(username, password):
      stored_username = "admin"
      stored_password = "password123"
      
      if username != stored_username or password != stored_password:
          raise InvalidCredentialsError()
      
      return f"Welcome, {username}!"
  ```  

  So können Sie die `login`-Funktion mit der `InvalidCredentialsError`-Ausnahme verwenden:

  ```py
  # failed login attempt
  try:
      message = login("user", "wrongpassword")
  except InvalidCredentialsError as e:
      print(f"Login failed: {e}")
  else:
      print(message)

  # successful login attempt
  try:
      message = login("admin", "password123")
  except InvalidCredentialsError as e:
      # This block is not executed because the login was successful
      print(f"Login failed: {e}")
  else:
      # The else block runs if the 'try' block completes without an exception
      print(message)
  ```

  Die `raise`-Anweisung kann auch mit dem Schlüsselwort `from` verwendet werden, um Ausnahmen zu verketten und so die Beziehung zwischen verschiedenen Fehlern zu zeigen:

  ```py
  def parse_config(filename):
    try:
        with open(filename, 'r') as file:
            data = file.read()
            return int(data)
    except FileNotFoundError:
        raise ValueError('Configuration file is missing') from None
    except ValueError as e:
        raise ValueError('Invalid configuration format') from e

  config = parse_config('config.txt')
  ```
