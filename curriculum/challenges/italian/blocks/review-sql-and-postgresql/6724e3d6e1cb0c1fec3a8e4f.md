## Introduzione ai database relazionali

- **Database relazionali**: organizzano i dati in tabelle correlate composte da righe e colonne. Ogni riga rappresenta un record e ogni colonna rappresenta un attributo del dato.
- **Vantaggi dei database relazionali**: scalabili, ampiamente applicabili in diversi ambiti (ad esempio, sanità, business, gaming) e strutturati per mantenere dati affidabili.
- **Casi d'uso comuni**: sviluppo web, sistemi di inventario, e-commerce, sanità e applicazioni aziendali.

## Concetti chiave

- **Schema**: un database relazionale richiede uno schema che definisca la sua struttura — tabelle, colonne, tipi di dati, vincoli e relazioni.
- **Chiavi primarie**: identificatori unici per ogni riga di una tabella. Sono essenziali per l'integrità dei dati e vengono usate per collegare record tra tabelle tramite chiavi esterne.
- **Chiavi esterne**: riferimenti a chiavi primarie in un'altra tabella, usate per collegare dati correlati tra tabelle.
- **Relazioni**: collegando le tabelle tramite chiavi primarie e esterne, puoi strutturare dati normalizzati ed eseguire query significative.
- **Diagrammi entità-relazione (ERD)**: visualizzano come le entità (tabelle) si relazionano tra loro in uno schema di database.
- **Integrità dei dati**: garantita usando chiavi e tipi di dati. Assicura coerenza e accuratezza dei dati archiviati.

## Nozioni di base su SQL

- **Query**: richieste per recuperare dati specifici dal database.

```sql
SELECT * FROM dogs WHERE age < 3;
```

- **Clausola WHERE**: filtra i risultati in base a condizioni. Usa operatori di confronto come `<`, `=`, `>`, ecc.
- **Select con ORDER BY**: recupera e ordina i risultati in base a una colonna.

```sql
SELECT columns FROM table_name ORDER BY column_name;
```

## Operazioni sulle tabelle

- **Istruzione `CREATE TABLE`**: questa istruzione serve a creare una nuova tabella in un database.

```sql
CREATE TABLE first_table();
```

- **Istruzione `ALTER TABLE ADD COLUMN`**: questa istruzione serve ad aggiungere una colonna a una tabella esistente.

```sql
ALTER TABLE table_name ADD COLUMN column_name DATATYPE;
```

- **Istruzione `ALTER TABLE DROP COLUMN`**: questa istruzione serve a eliminare una colonna da una tabella esistente.

```sql
ALTER TABLE table_name DROP COLUMN column_name;
```

- **Istruzione `ALTER TABLE RENAME COLUMN`**: questa istruzione serve a rinominare una colonna in una tabella.

```sql
ALTER TABLE table_name RENAME COLUMN column_name TO new_name;
```

- **Istruzione `DROP TABLE`**: questa istruzione serve a eliminare un'intera tabella dal database.

```sql
DROP TABLE table_name;
```

- **Istruzione `ALTER DATABASE RENAME`**: questa istruzione serve a rinominare un database.

```sql
ALTER DATABASE database_name RENAME TO new_database_name;
```

- **Istruzione `DROP DATABASE`**: questa istruzione serve a eliminare un intero database.

```sql
DROP DATABASE database_name;
```

## Vincoli e integrità dei dati

- **`ALTER TABLE ADD COLUMN` con vincolo**: questa istruzione serve ad aggiungere una colonna con un vincolo a una tabella esistente.

```sql
ALTER TABLE table_name ADD COLUMN column_name DATATYPE CONSTRAINT;
```

- **Vincolo `NOT NULL`**: questo vincolo assicura che una colonna non possa contenere valori NULL.

```sql
column_name VARCHAR(50) NOT NULL
```

- **Istruzione `ALTER TABLE ADD PRIMARY KEY`**: questa istruzione serve ad aggiungere un vincolo di chiave primaria a una tabella.

```sql
ALTER TABLE table_name ADD PRIMARY KEY(column_name);
```

- **Istruzione `ALTER TABLE DROP CONSTRAINT`**: questa istruzione serve a eliminare un vincolo da una tabella.

```sql
ALTER TABLE table_name DROP CONSTRAINT constraint_name;
```

- **`ALTER TABLE ADD COLUMN` con chiave esterna**: questa istruzione serve ad aggiungere una colonna chiave esterna che fa riferimento a un'altra tabella.

```sql
ALTER TABLE table_name ADD COLUMN column_name DATATYPE REFERENCES referenced_table_name(referenced_column_name);
```

- **Istruzione `ALTER TABLE ADD UNIQUE`**: questa istruzione serve ad aggiungere un vincolo UNIQUE a una colonna.

```sql
ALTER TABLE table_name ADD UNIQUE(column_name);
```

- **Istruzione `ALTER TABLE ALTER COLUMN SET NOT NULL`**: questa istruzione serve a impostare un vincolo NOT NULL su una colonna esistente.

```sql
ALTER TABLE table_name ALTER COLUMN column_name SET NOT NULL;
```

- **Istruzione `INSERT` con valori NULL**: questa istruzione dimostra come inserire valori NULL in una tabella.

```sql
INSERT INTO table_name(column_a) VALUES(NULL);
-- or
INSERT INTO table_name(column_b) VALUES('value'); -- if column_a allows nulls
```

- **Chiave primaria composta**: questo vincolo definisce una chiave primaria composta da più colonne.

```sql
CREATE TABLE course_enrollments (
    student_id INT,
    course_id INT,
    PRIMARY KEY (student_id, course_id)
);
```

## Manipolazione dei dati (CRUD)

- **Istruzione `INSERT`**: questa istruzione serve a inserire una singola riga in una tabella.

```sql
INSERT INTO table_name(column_1, column_2) VALUES(value1, value2);
```

- **Istruzione `INSERT` con colonne omesse**: questa istruzione mostra come inserire valori senza elencare esplicitamente i nomi delle colonne, affidandosi all'ordine predefinito delle colonne nella tabella.

```sql
INSERT INTO dogs VALUES ('Gino', 3);
```

- **Istruzione `INSERT` con più righe**: questa istruzione serve a inserire più righe in una tabella in un'unica operazione.

```sql
INSERT INTO dogs (name, age) VALUES 
('Gino', 3),
('Nora', 2);
```

- **Istruzione `UPDATE`**: questa istruzione serve ad aggiornare dati esistenti in una tabella in base a una condizione.

```sql
UPDATE table_name SET column_name=new_value WHERE condition;
```

- **Istruzione `DELETE`**: questa istruzione serve a eliminare righe da una tabella in base a una condizione.

```sql
DELETE FROM table_name WHERE condition;
```

## Tipi di dati

- **Tipo di dato `NUMERIC`**: questo tipo di dato serve a memorizzare numeri decimali precisi con una precisione e scala specificate.

```sql
price NUMERIC(10, 2)
```

- **Tipo di dato `TEXT`**: questo tipo di dato serve a memorizzare stringhe di caratteri a lunghezza variabile senza un limite specifico di lunghezza.

```sql
column_name TEXT
```

- **Tipo di dato `INTEGER`**: questo tipo di dato serve a memorizzare numeri interi senza cifre decimali.

```sql
units_sold INTEGER
```

- **Tipi di dato `SMALLINT` e `BIGINT`**: sono varianti di INTEGER con intervalli rispettivamente più piccoli e più grandi.

- **Tipo di dato `SERIAL`**: questo tipo di dato serve a creare colonne intere auto-incrementanti in PostgreSQL.

```sql
id SERIAL
```

- **Attributo `AUTO_INCREMENT`**: questo attributo è usato in MySQL per creare colonne intere auto-incrementanti.

```sql
id INT AUTO_INCREMENT
```

- **Tipo di dato `VARCHAR`**: questo tipo di dato serve a memorizzare stringhe di caratteri a lunghezza variabile con una lunghezza massima specificata.

```sql
name VARCHAR(50)
```

- **Tipo di dato `DATE`**: questo tipo di dato serve a memorizzare valori di data (anno, mese, giorno).

```sql
event_date DATE
```

- **Tipo di dato `TIME`**: questo tipo di dato serve a memorizzare valori di ora (ora, minuto, secondo).

```sql
start_time TIME
```

- **Tipo di dato `TIMESTAMP`**: questo tipo di dato serve a memorizzare valori di data e ora, opzionalmente con informazioni sul fuso orario.

```sql
event_timestamp TIMESTAMP
event_timestamp TIMESTAMP WITH TIME ZONE
```

- **Tipo di dato `BOOLEAN`**: questo tipo di dato serve a memorizzare valori vero/falso.

```sql
is_active BOOLEAN
```

## Relazioni nel database

- **Tipi di relazioni**: questi sono i diversi modi in cui le tabelle possono essere correlate tra loro in un database relazionale.
  - Uno a uno
  - Uno a molti
  - Molti a uno
  - Molti a molti
  - Auto-riferimento (ricorsivo)

- **Relazione uno a uno**: questo tipo di relazione significa che ogni record in una tabella corrisponde esattamente a un record in un'altra tabella.

```md
One employee is assigned exactly one vehicle.
Tables: employees, vehicles
```

- **Relazione uno a molti**: questo tipo di relazione significa che un record in una tabella può essere associato a più record in un'altra tabella.

```md
One customer can have many orders.
Tables: customers → orders
```

- **Relazione molti a molti tramite tabella di giunzione**: questo tipo di relazione si implementa usando una tabella di giunzione che contiene chiavi esterne da entrambe le tabelle correlate.

```sql
CREATE TABLE books_authors (
    author_id INT REFERENCES authors(id),
    book_id INT REFERENCES books(id)
);
```

- **Relazioni auto-riferite**: questo tipo di relazione si verifica quando una tabella fa riferimento a se stessa, creando una struttura gerarchica.

```md
An employee table where each employee may report to another employee.
```

## SQL avanzato (join)

- **Istruzione `INNER JOIN`**: questa join restituisce solo le righe che hanno valori corrispondenti in entrambe le tabelle.

```sql
SELECT *
FROM products
INNER JOIN sales ON products.product_id = sales.product_id;
```

- **Istruzione `FULL OUTER JOIN`**: questa join restituisce tutte le righe di entrambe le tabelle, incluse quelle non corrispondenti di entrambe le tabelle.

```sql
SELECT *
FROM products
FULL OUTER JOIN sales ON products.product_id = sales.product_id;
```

- **Istruzione `LEFT OUTER JOIN`**: questa join restituisce tutte le righe della tabella di sinistra e le righe corrispondenti della tabella di destra.

```sql
SELECT *
FROM products
LEFT JOIN sales ON products.product_id = sales.product_id;
```

- **Istruzione `RIGHT OUTER JOIN`**: questa join restituisce tutte le righe della tabella di destra e le righe corrispondenti della tabella di sinistra.

```sql
SELECT *
FROM products
RIGHT JOIN sales ON products.product_id = sales.product_id;
```

- **Istruzione `SELF JOIN`**: questa join serve a unire una tabella con se stessa per confrontare righe all'interno della stessa tabella.

```sql
SELECT A.column_name, B.column_name
FROM table_name A
JOIN table_name B ON A.related_column = B.related_column;
```

- **Istruzione `CROSS JOIN`**: questa join restituisce il prodotto cartesiano di due tabelle, combinando ogni riga della prima tabella con ogni riga della seconda tabella.

```sql
SELECT *
FROM table1
CROSS JOIN table2;
```

## Comandi specifici di PostgreSQL

- **Comando di login `psql`**: questo comando serve per accedere a PostgreSQL con un nome utente e database specifici.

```bash
psql --username=freecodecamp --dbname=postgres
```

- **Comando `\l`**: questo comando elenca tutti i database nell'istanza di PostgreSQL.

```sql
\l
```

- **Comandi `CREATE DATABASE` e `\c`**: questi comandi servono a creare un nuovo database e a connettersi ad esso.

```sql
CREATE DATABASE database_name;
\c database_name
```

- **Comando `\d`**: questo comando elenca tutte le tabelle nel database corrente.

```sql
\d
```

- **Comando `\d table_name`**: questo comando mostra lo schema/struttura di una tabella specifica.

```sql
\d table_name
```

- **Comando `\q`**: questo comando esce dal client PostgreSQL.

```bash
\q
```

## Relazionale vs non relazionale

- **Database non relazionali (NoSQL)**: archiviano dati non strutturati o semi-strutturati. Non richiedono uno schema rigido e sono più flessibili per modelli di dati in evoluzione.
- **Scegliere tra relazionale e non relazionale**: dipende dalla natura dei dati e dai requisiti dell'applicazione.
- **Relazionale vs non relazionale**: scegli relazionale per dati strutturati e coerenza; NoSQL per flessibilità e dati in rapido cambiamento.

## Sistemi RDBMS popolari

- **MySQL**: open source, affidabile, ampiamente usato nello sviluppo web, supportato da una grande comunità.
- **PostgreSQL**: open source, avanzato, estensibile. Supporta tipi di dati personalizzati e programmazione lato server.
- **SQLite**: leggero, basato su file, senza server. Ideale per applicazioni piccole.
- **Microsoft SQL Server**: database proprietario di livello enterprise.
- **Oracle Database**: RDBMS commerciale noto per prestazioni e scalabilità su larga scala.

## Buone pratiche

- **Convenzione di denominazione**: usa `snake_case` (ad esempio `delivery_orders`) per nomi di tabelle e colonne.
