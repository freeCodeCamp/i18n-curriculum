# --interactive--

Modifikatoren, oft auch „Flags“ genannt, verändern das Verhalten eines regulären Ausdrucks. Erinnern wir uns an unser Beispiel aus einer früheren Lektion:

:::interactive_editor

```js
const regex = /freeCodeCamp/;
console.log(regex.test("freeCodeCamp")); // true
console.log(regex.test("freeCodeCamp is great")); // true
console.log(regex.test("I love freeCodeCamp")); // true
console.log(regex.test("freecodecamp")); // false
console.log(regex.test("FREECODECAMP")); // false
console.log(regex.test("free")); // false
console.log(regex.test("code")); // false
console.log(regex.test("camp")); // false
```

:::

Wenn Sie sich erinnern, haben die komplett in Kleinbuchstaben und komplett in Großbuchstaben geschriebenen `freeCodeCamp` Strings nicht zum Muster gepasst. Das liegt daran, dass reguläre Ausdrücke standardmäßig zwischen Groß- und Kleinschreibung unterscheiden.

Aber was, wenn wir dem regulären Ausdruck sagen könnten, er soll die Groß- und Kleinschreibung ignorieren? Dafür gibt es einen Modifikator. Das `i` Flag sorgt dafür, dass ein Regex die Groß- und Kleinschreibung ignoriert. Wie können wir es verwenden? Flags stehen nach dem abschließenden Schrägstrich in einem regulären Ausdruck:

```js
const regex = /freeCodeCamp/i;
```

Beachten Sie die Änderung im regulären Ausdruck in der ersten Zeile. Jetzt können wir prüfen, wie sich das auswirkt:

:::interactive_editor

```js
const regex = /freeCodeCamp/i;

console.log(regex.test("freeCodeCamp")); // true
console.log(regex.test("freeCodeCamp is great")); // true
console.log(regex.test("I love freeCodeCamp")); // true
console.log(regex.test("freecodecamp")); // true
console.log(regex.test("FREECODECAMP")); // true
console.log(regex.test("free")); // false
console.log(regex.test("code")); // false
console.log(regex.test("camp")); // false
```

:::

Da unser regulärer Ausdruck jetzt nicht mehr zwischen Groß- und Kleinschreibung unterscheidet, haben die komplett in Kleinbuchstaben und komplett in Großbuchstaben geschriebenen Strings den Test „bestanden“. Das funktioniert auch bei einem String mit einer zufälligen Mischung aus Groß- und Kleinbuchstaben:

:::interactive_editor

```js
const regex = /freeCodeCamp/i;

console.log(regex.test("dO yOu LoVe fReEcOdEcAmP?")); // true
```

:::

Es gibt noch einige andere Flags, die Sie verwenden können. Das `g` Flag, auch globaler Modifikator genannt, erlaubt es Ihrem regulären Ausdruck, ein Muster mehr als einmal zu finden.

Sehen wir uns an, wie sich das auf unseren Code auswirkt. Sie werden bemerken, dass wir das `i`-Flag beibehalten haben: Ein regulärer Ausdruck kann mehrere Flags verwenden (so viele wie nötig), um das gewünschte Verhalten zu erreichen:

```js
const regex = /freeCodeCamp/gi;
```

Moment mal... was ist das? Es scheint, als würde der globale Modifikator einige unserer Strings, die eigentlich passen sollten, als nicht passend bewerten:

:::interactive_editor

```js
const regex = /freeCodeCamp/gi;

console.log(regex.test("freeCodeCamp")); // true
console.log(regex.test("freeCodeCamp is great")); // false
console.log(regex.test("I love freeCodeCamp")); // true
console.log(regex.test("freecodecamp")); // false
console.log(regex.test("FREECODECAMP")); // true
console.log(regex.test("free")); // false
console.log(regex.test("code")); // false
console.log(regex.test("camp")); // false
```

:::

Warum? Der globale Modifikator macht Ihren regulären Ausdruck zustandsbehaftet. Das bedeutet, er merkt sich, wo er zuvor ein Muster gefunden hat. Wenn er also den ersten `freeCodeCamp` String matched, merkt er sich, dass er eine Übereinstimmung ab Index `0` gefunden hat.

Dann testen wir ihn gegen `freeCodeCamp is great`, aber dieser beginnt nicht bei Index `0`. Der reguläre Ausdruck „weiß“, dass er bereits eine Übereinstimmung bei Index `0` gefunden hat, also beginnt er, obwohl es ein anderer String ist, ab dem Endindex der vorherigen Übereinstimmung.

`freeCodeCamp` ist `12` Zeichen lang, also endet eine Übereinstimmung bei `0` am Index `11`. Die Suche wird bei Index `12` fortgesetzt. Und da `is great` nicht zu `freeCodeCamp` passt, gibt er `false` zurück.

Dann, weil keine Übereinstimmung gefunden wird, „verliert“ er seinen Zustand und beginnt die nächste Suche wieder bei `0`.

Wenn wir unsere Logs so anordnen, dass ein String mit einer Übereinstimmung bei `0` direkt von einem String gefolgt wird, der eine Übereinstimmung später als bei Index `11` hat:

:::interactive_editor

```js
const regex = /freeCodeCamp/gi;

console.log(regex.test("freeCodeCamp")); // true
console.log(regex.test("I loooooooove freeCodeCamp")); // true
```

:::

Wenn ein regulärer Ausdruck global ist, erhält er eine neue Eigenschaft namens `lastIndex`. Schauen wir uns mit unserem vorherigen Code an, wie diese Eigenschaft funktioniert:

:::interactive_editor

```js
const regex = /freeCodeCamp/gi;

console.log(regex.lastIndex); // 0
console.log(regex.test("freeCodeCamp")); // true
console.log(regex.lastIndex); // 12
console.log(regex.test("freeCodeCamp is great")); // false
console.log(regex.lastIndex); // 0
console.log(regex.test("I love freeCodeCamp")); // true
console.log(regex.lastIndex); // 19
console.log(regex.test("freecodecamp")); // false
console.log(regex.lastIndex); // 0
console.log(regex.test("FREECODECAMP")); // true
console.log(regex.lastIndex); // 12
console.log(regex.test("free")); // false
console.log(regex.lastIndex); // 0
console.log(regex.test("code")); // false
console.log(regex.lastIndex); // 0
console.log(regex.test("camp")); // false
```

:::

Anhand dieses Beispiels sehen Sie, wie sich der Zustand des regulären Ausdrucks mit jedem Testaufruf ändert, wobei `lastIndex` seine vorherigen Übereinstimmungen verfolgt.

Das globale Flag ist großartig, wenn Sie mehrere Übereinstimmungen in einem einzigen String finden möchten. Wenn Sie jedoch mehrere Strings mit demselben regulären Ausdruck testen, ist es besser, das `g` Flag auszuschalten.

Bevor Sie den nächsten Flag kennenlernen, müssen Sie etwas über Anker wissen. Der Caret-Anker (`^`) am Anfang des regulären Ausdrucks bedeutet „mache eine Übereinstimmung am Anfang des Strings“:

```js
const start = /^freecodecamp/i;
```

Das Dollarzeichen-Anker (`$`) am Ende des regulären Ausdrucks bedeutet „mache eine Übereinstimmung am Ende des Strings“:

```js
const end = /freecodecamp$/i;
```

Nehmen Sie sich einen Moment Zeit, um die Ausgaben rechts zu vergleichen:

:::interactive_editor

```js
const start = /^freecodecamp/i;
const end = /freecodecamp$/i;
console.log(start.test("freecodecamp")); // true
console.log(end.test("freecodecamp")); // true
console.log(start.test("freecodecamp is great")); // true
console.log(end.test("freecodecamp is great")); // false
console.log(start.test("i love freecodecamp")); // false
console.log(end.test("i love freecodecamp")); // true
console.log(start.test("have met freecodecamp's founder")); // false
console.log(end.test("have met freecodecamp's founder")); // false
```

:::

Sehen Sie, wie der Startanker nur am Anfang des Strings passt und der Endanker nur am Ende des Strings? Aber wie sieht es mit Übereinstimmungen über mehrere Zeilen aus? Schauen wir uns das an:

:::interactive_editor

```js
const start = /^freecodecamp/i;
const end = /freecodecamp$/i;
const string = `I really love
freecodecamp
it's my favorite`;
console.log(start.test(string)); // false
console.log(end.test(string)); // false
```

:::

Obwohl `freecodecamp` in einer eigenen Zeile steht, schlägt beides fehl. Das liegt daran, dass Anker standardmäßig nur den Anfang und das Ende des gesamten Strings suchen.

Aber Sie können einen Regex mit dem `m` Flag, dem Mehrzeilen-Modifikator, so einstellen, dass er mehrere Zeilen behandelt. Fügen wir das unseren regulären Ausdrücken hinzu, um zu sehen, was passiert:

:::interactive_editor

```js
const start = /^freecodecamp/im;
const end = /freecodecamp$/im;
const string = `I really love
freecodecamp
it's my favorite`;
console.log(start.test(string)); // true
console.log(end.test(string)); // true
```

:::

Jetzt passen beide! Weil `freecodecamp` ganz allein in einer Zeile steht, passt der Startanker zum Anfang dieser Zeile und der Endanker zum Ende dieser Zeile.

Schließlich gibt es noch das `d` Flag, den Indizes-Modifikator. Denken Sie daran, dass das `i` Flag für die Groß-/Kleinschreibung zuständig ist, daher brauchte der Indizes-Modifikator ein anderes Flag.

Das `d` Flag erweitert die Informationen, die Sie in einem Match-Objekt erhalten. Fügen wir es unserem regulären Ausdruck hinzu:

```js
const regex = /freecodecamp/di;
const string = "we love freecodecamp isn't freecodecamp great?";
console.log(string.match(regex));
```

Und das Ergebnis ist:

```js
// [
//   'freecodecamp',
//   index: 8,
//   input: "we love freecodecamp isn't freecodecamp great?",
//   groups: undefined,
//   indices: [
//     0: [8, 20],
//     groups: undefined
//   ]
// ]
```

Unser Match-Objekt erhält eine neue `indices` Eigenschaft! Diese Eigenschaft ist ein Array aus zwei Zahlen: Die erste ist der Index im ursprünglichen String, an dem die Übereinstimmung beginnt, und die zweite ist der Index direkt nach dem Ende der Übereinstimmung. Dieses Array hat außerdem eine zusätzliche `groups` Eigenschaft, die für benannte Capture-Gruppen gedacht ist.

Es gibt noch einige andere Flags, die Ihnen zur Verfügung stehen, die aber in typischem Code weniger häufig verwendet werden.

Das erste ist der Unicode-Modifikator, oder das `u` Flag. Dieser erweitert die Funktionalität eines regulären Ausdrucks, sodass er spezielle Unicode-Zeichen erkennen kann.

Sie werden in einer späteren Lektion mehr über Zeichenklassen lernen, aber das `u` Flag gibt Ihnen Zugriff auf spezielle Klassen wie `Extended_Pictographic`, um die meisten Emojis zu erkennen:

:::interactive_editor

```js
const regex = /🍎/u;

const str = "I have an apple 🍎";
console.log(regex.test(str)); // true
```

:::

Es gibt auch ein `v` Flag, das die Funktionalität der Unicode-Erkennung noch weiter ausbaut.

Das zweite ist der Sticky-Modifikator, oder das `y` Flag. Der Sticky-Modifikator verhält sich sehr ähnlich zum globalen Modifikator, aber mit einigen Ausnahmen.

Die wichtigste ist, dass ein globaler regulärer Ausdruck ab `lastIndex` sucht und den gesamten Rest des Strings durchsucht, um eine weitere Übereinstimmung zu finden, während ein Sticky-Regulärer Ausdruck `null` zurückgibt und den `lastIndex` auf `0` zurücksetzt, wenn nicht sofort eine Übereinstimmung an der vorherigen `lastIndex` gefunden wird.

Und zuletzt gibt es den Single-Line-Modifier, oder das `s`-Flag. Denken Sie daran, dass der Multi-Line-Modifier Start- und Endanker dazu bringt, den Anfang und das Ende einer Zeile zu erkennen, statt des gesamten Strings.

Der Single-Line-Modifier erlaubt es einem Wildcard-Zeichen, dargestellt durch einen Punkt (`.`) im Regex, Zeilenumbrüche zu matchen und behandelt den String effektiv als eine einzige Textzeile.

Es gibt ziemlich viele dieser Modifikatoren, aber die `i` und `g` Flags sind die, die Sie am häufigsten verwenden werden und die am wichtigsten sind, sich zu merken.
