# --description--

前のレッスンでは、ユニオン型の扱い方を学びました。ユニオン型を扱う一般的な方法の一つが型エイリアスを使うことです。

型エイリアスとは、型に名前を付けることです。以下はユニオン型に名前を付ける例です：

```ts
type ID = number | string;
```

プロジェクトでは、型エイリアスを作成し、プログラム全体でこのように使うことがよくあります：

```ts
type ID = number | string;

type User = {
  id: ID;
  name: string;
};

function getUserId(user: User): ID {
  return user.id;
}
```

この例では、`ID` と `User` という2つの型を使っています。`ID` は `number` または `string` のユニオン型であり、`User` は `id` と `name` プロパティを持つオブジェクト型です。`id` は `ID` 型で指定され、`name` は `string` 型を持っています。

`getUserId` 関数は `user` というパラメータを1つ持ち、その型は `User` です。`getUserId` 関数の戻り値の型は `ID` となります。

型エイリアスはオブジェクトの形状を表現するのにとても便利です。オブジェクト型を作成し、配列で使うこともよくあります：

```ts
type User = {
  id: number | string;
  name: string;
};

const users: User[] = [
  { id: 101, name: "Alice" },
  { id: "A102", name: "Bob" },
  { id: 103, name: "Charlie" }
];
```

この例では、`users` 配列は `User` オブジェクトのみで埋める必要があります。もしオブジェクトの中でプロパティを忘れると、TypeScriptは次のようなエラーをスローします：

```ts
type User = {
  id: number | string;
  name: string;
};

const users: User[] = [
  // Property 'name' is missing in type '{ id: number; }' but required in type 'User'.
  { id: 101 },
  { id: "A102", name: "Bob" },
  { id: 103, name: "Charlie" }
];
```

型エイリアスの名前付けに PascalCase が使われていることに気づいたかもしれません。これは慣例として、PascalCase を使うことで型と変数を区別しやすくし、可読性を高めるためです。`User[]` を見ると、それが `User` 型のオブジェクトの配列であることがわかります。
