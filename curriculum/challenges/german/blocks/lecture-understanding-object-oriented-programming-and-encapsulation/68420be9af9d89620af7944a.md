# --description--

Objektorientierte Programmierung, auch bekannt als OOP, ist ein Programmierstil, bei dem Entwickler alles in ihrem Code wie ein reales Objekt behandeln.

Eine Klasse ist wie ein Bauplan zur Erstellung von Objekten. Jedes einzelne Objekt, das aus einer Klasse erstellt wird, hat Attribute, die Daten definieren, und Methoden, die das Verhalten der Objekte festlegen.

In einer vorherigen Lektion haben Sie gelernt, wie man Klassen erstellt. Hier eine Erinnerung an die Syntax:

```py
class ClassName:
   def __init__(self, parameters):
       attribute = value

   def method_name(self):
       # method logic
```

Hier ist ein Beispiel für eine Klasse, die die `__init__` spezielle Methode verwendet, um die Attribute Marke und Farbe zu initialisieren, wann immer ein Objekt mit der Klasse erstellt wird:

```py
class Car:
   def __init__(self, brand, color):
       self.brand = brand
       self.color = color

# create two objects from the Car class
car1 = Car('Toyota', 'red')
car2 = Car('Lambo', 'green')

print('Car 1 Brand:', car1.brand) # Car 1 Brand: Toyota
print('Car 1 Color:', car1.color) # Car 1 Color: red

print('Car 2 Brand:', car2.brand) # Car 2 Brand: Lambo
print('Car 2 Color:', car2.color) # Car 2 Color: green
```

Die objektorientierte Programmierung hat vier Grundsätze, die Ihnen helfen, Code effektiv zu organisieren und zu verwalten. Diese sind Kapselung, Vererbung, Polymorphismus und Abstraktion.

Der Rest dieser Lektion konzentriert sich darauf, wie Kapselung funktioniert.

Kapselung ist das Bündeln der Attribute und Methoden eines Objekts zu einer einzigen Einheit, der Klasse.

Mit Kapselung können Sie den internen Zustand des Objekts hinter einer einfachen Menge öffentlicher Methoden und Attribute verbergen, die wie Türen funktionieren. Hinter diesen Türen befinden sich private Attribute und Methoden, die steuern, wie sich die Daten ändern und wer sie sehen kann.

Angenommen, Sie möchten den Kontostand eines Geldbeutels verfolgen. Sie möchten erlauben, dass Menschen Geld einzahlen oder abheben, aber niemand sollte den Kontostand direkt manipulieren können.

In diesem Fall können Sie `deposit()` und `withdraw()` als öffentliche Methoden definieren und den Kontostand unter dem `_balance` Attribut verbergen:

```py
class Wallet:
   def __init__(self, balance):
       self._balance = balance # For internal use by convention

   def deposit(self, amount):
       if amount > 0:
           self._balance += amount # Add to the balance safely

   def withdraw(self, amount):
       if 0 < amount <= self._balance:
           self._balance -= amount # Remove from the balance safely
```

Konventionell bedeutet das Präfix eines Attributs oder einer Methode mit einem einzelnen Unterstrich, dass sie für den internen Gebrauch bestimmt sind. Niemand sollte direkt von außerhalb der Klasse auf sie zugreifen, da dies den Grundsätzen der Kapselung widerspricht und zu Bugs führen kann.

Während ein einzelner Unterstrich nur eine Konvention ist, verhindert das Präfix von Attributen und Methoden mit einem doppelten Unterstrich effektiv, dass sie von außerhalb ihrer Klasse aufgerufen werden können, wodurch diese Attribute und Methoden privat werden.

```py
class Wallet:
   def __init__(self, balance):
       self.__balance = balance # Private attribute

   def deposit(self, amount):
       if amount > 0:
           self.__balance += amount # Add to the balance safely

   def withdraw(self, amount):
       if 0 < amount <= self.__balance:
           self.__balance -= amount # Remove from the balance safely

account = Wallet(500)
print(account.__balance) # AttributeError: 'Wallet' object has no attribute '__balance'
```

Um den aktuellen Wert von `__balance` zu erhalten, können Sie eine get_balance-Methode definieren. Zum Beispiel:

```py
class Wallet:
   def __init__(self, balance):
       self.__balance = balance

   def deposit(self, amount):
       if amount > 0:
           self.__balance += amount

   def withdraw(self, amount):
       if 0 < amount <= self.__balance:
           self.__balance -= amount
  
   def get_balance(self):
       return self.__balance


acct_one = Wallet(100)
acct_one.deposit(50)
print(acct_one.get_balance()) # 150

acct_two = Wallet(450)
acct_two.withdraw(28)
print(acct_two.get_balance()) # 422

acct_two.deposit(150)
print(acct_two.get_balance()) # 572
```

Sie können auch eine private `__validate` Methode definieren, um zu überprüfen, ob jeder Einzahlungs- oder Abhebungsbetrag eine positive Zahl ist:

```py
class Wallet:
   def __init__(self):
       self.__balance = 0

   def __validate(self, amount):
       if amount < 0:
           raise ValueError('Amount must be positive')

   def deposit(self, amount):
       self.__validate(amount)
       self.__balance += amount

   def withdraw(self, amount):
       self.__validate(amount)
       if amount > self.__balance:
           raise ValueError('Insufficient funds')
       self.__balance -= amount

   def get_balance(self):
       return self.__balance

acct_one = Wallet()
acct_one.deposit(3)
print(acct_one.get_balance()) # 3

acct_one.deposit(50)
print(acct_one.get_balance()) # 53

acct_one.deposit(-4)  # ValueError: Amount must be positive
acct_one.withdraw(-8) # ValueError: Amount must be positive
acct_one.withdraw(58) # ValueError: Insufficient funds
```

Wie Sie sehen, ist die `__validate` Methode privat und läuft hinter den Kulissen in den öffentlichen Methoden `deposit()` und `withdraw()`, um sicherzustellen, dass der Betrag immer gültig ist.

In einer kommenden Lektion lernen Sie mehr darüber, wie Attribute mit doppeltem Unterstrich funktionieren.

Zusammenfassend sperrt Kapselung interne Daten hinter klaren öffentlichen Methoden ab. So schützen Sie Ihre Klassen vor Manipulation und zentralisieren die Validierung an einem Ort. Sie können Ihren Code frei aktualisieren oder erweitern, in dem Wissen, dass externer Code nur die Schnittstellen berührt, die Sie verfügbar machen.
