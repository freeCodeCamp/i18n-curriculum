# --description--

In Express, i parametri di rotta sono valori dinamici incorporati nel percorso URL. Permettono alla tua applicazione di catturare parti dell’URL che possono cambiare a ogni richiesta. Questo è utile per costruire pagine dinamiche, come mostrare un utente specifico, un post del blog o un prodotto in base all’URL.

Ecco un esempio:

```javascript
// Route with a parameter in the path
app.get("/post/:postId", (req, res) => {
  const postId = req.params.postId;
  res.send(`Viewing post with ID: ${postId}`);
});
```

Qui, la rotta ascolta le richieste GET a `/post/:postId`.

* `:postId` è un parametro di rotta, un segnaposto nell’URL che verrà sostituito da un valore reale. Per esempio, una richiesta a `/post/42` corrisponde a questa rotta, e `42` diventa il valore di `postId`.
    
* Puoi consultare i parametri di rotta usando `req.params`. L’oggetto `req.params` contiene tutti i parametri di rotta, e in questo caso `req.params.postId` contiene il valore preso dall’URL, `42`.
    

Infine, `res.send()` invia una risposta al client. Quindi se l’URL è `/post/42`, il client vede `Viewing post with ID: 42`.

Come vedi, i parametri di rotta rendono le tue rotte flessibili e dinamiche senza dover creare una rotta separata per ogni possibile valore.

Ora che hai capito come funzionano i parametri di rotta, torniamo a un esempio della lezione precedente e vediamo come puoi usare rotte dinamiche per diversi tipi di menu:

```javascript
app.get("/menu/:category", (req, res) => {
  const category = req.params.category;
  res.send(`You are viewing the ${category} menu!`);
});
```

Ecco una spiegazione di come funziona:

1. Un utente visita [`http://localhost:3000/menu/drinks`](http://localhost:3000/menu/drinks).
    
2. Il server associa questa richiesta alla rotta `/menu/:category` e cattura il parametro `category` come `drinks`.
    
3. Il server invia questa risposta al browser: `You are viewing the drinks menu!`
    

Allo stesso modo, se l’utente visita [`http://localhost:3000/menu/food`](http://localhost:3000/menu/food), il server risponde con: `You are viewing the food menu!`

I parametri di rotta ti permettono di mostrare contenuti dinamici in base all’URL. Invece di creare una rotta separata per ogni categoria, puoi usare una singola rotta per gestire più categorie come bevande, cibo, dessert e altro, semplicemente cambiando l’URL.

## --text--

Qual è il ruolo di `:postId` nella rotta `/post/:postId`?

## --answers--

È una parte statica dell’URL.

### --feedback--

`:` indica un parametro, non un segmento fisso.

---

Cattura un valore dinamico dall’URL.

---

È un parametro di query.

### --feedback--

I parametri di query appaiono dopo `?`, non nel percorso.

---

Definisce un’intestazione personalizzata.

### --feedback--

I parametri di rotta si trovano nel percorso URL, non nelle intestazioni.

Per archiviare valori statici nell’URL.
