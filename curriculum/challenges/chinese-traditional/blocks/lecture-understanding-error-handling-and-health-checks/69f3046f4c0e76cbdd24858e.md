# --description--

你是否曾經想過，當你的 Express 應用程式出錯時會發生什麼事？

如果資料庫連線失敗呢？或者使用者造訪了一個不存在的路由？

了解 Express 中錯誤處理的運作方式，對於建置安全且可靠的網頁應用程式至關重要。

在本課程中，我們將帶你了解 Express 如何處理錯誤，以及如何使用實際的程式碼範例來捕捉並回應錯誤。

## 如何在 Express 中處理簡單錯誤

在 Express 中，錯誤處理是建立在中介層上的。

就像一般的中介層函式一樣，錯誤處理中介層可以存取 `req`、`res` 和 `next`。但它在最前面多了一個參數：`err`。

語法如下：

```js
app.use((err, req, res, next) => {
  console.error(err.message);
  res.status(500).send("Internal Server Error");
});
```

你會將這個中介層定義在中介層堆疊的最後。

Express 會知道它是錯誤處理常式，因為它有四個參數 — 新增的 `err`，以及其他中介層函式常見的 `req`、`res` 和 `next`。

在接下來的課程中，你會學到更多關於錯誤處理中介層，以及它如何融入更廣泛的 Express 中介層堆疊。

## 如何處理非同步錯誤

非同步錯誤的處理方式取決於你使用的 Express 版本。

請注意，無論你使用哪個版本的 Express，都必須將錯誤處理中介層定義在堆疊的最後，所有路由之後。

錯誤處理常式如下：

```js
// App routes
// ...

// Error-handling middleware
app.use((err, req, res, next) => {
  console.error(err.message);
  res.status(500).send("Internal Server Error");
});
```

## Express 4

在 Express 4 及以下版本（許多專案仍在使用），非同步錯誤**不會自動被捕捉**。

如果你使用 `async` / `await`，且在 `async` 路由處理常式中發生錯誤，Express 不會捕捉它，除非你手動將錯誤傳給 `next()`。

以下是在 Express 4 中正確處理非同步錯誤的方法：

```js
app.get("/user", async (req, res, next) => {
  try {
    const user = await getUserFromDatabase();
    res.send(user);
  } catch (err) {
    next(err); // Manually pass the error to the error handler
  }
});
```

如果你忘記使用 `try/catch` 區塊和 `next(err)`，錯誤可能會：

* 造成應用程式崩潰
* 或導致未處理的 promise 拒絕警告

這就是為什麼在 Express 4 中，將非同步邏輯包裹在 `try` / `catch` 中非常重要。

有些開發者甚至會建立一個輔助函式，避免到處重複 `try/catch`：

```js
function asyncHandler(fn) {
  return function (req, res, next) {
    Promise.resolve(fn(req, res, next)).catch(next);
  };
}
```

然後這樣使用它：

```js
app.get("/user", asyncHandler(async (req, res) => {
  const user = await getUserFromDatabase();
  res.send(user);
}));
```

這樣你就不需要在所有 `try/catch` 路由中使用 `async` 區塊，同時仍能正確轉發錯誤。

## Express 5

目前的 Express 版本改善了這個行為。

從 Express 5 開始，如果 `async` 路由丟出錯誤或回傳拒絕的 promise，Express 會自動將它轉發給錯誤處理中介層。

這表示以下範例可以在沒有 `try/catch` 的情況下處理錯誤：

```js
app.get("/user", async (req, res) => {
  const user = await getUserFromDatabase();
  res.send(user);
});
```

如果 `getUserFromDatabase()` 丟出錯誤，Express 5 會自動捕捉並傳給你的錯誤處理常式。

不需要手動呼叫 `next(err)`。

## 如何處理 404 錯誤

404 錯誤有點不同。當沒有路由符合請求時，就會發生。

你可以用這個中介層來處理：

```js
app.use((req, res, next) => {
  res.status(404).send("Sorry, that route does not exist.");
});
```

**重要**：這個中介層應該放在路由之後，但在錯誤處理中介層之前。

最後，這裡有一些在 Express 中處理錯誤的最佳實務：

* 永遠將錯誤處理中介層放在最後。
* 分別處理 404 錯誤。
* 在 Express 4 及以下版本，使用 `next(err)` 來將錯誤從路由或 `async` 邏輯中冒泡上來。

在生產環境中，不要將堆疊追蹤傳給使用者。請在內部記錄詳細資訊，並回傳使用者友善的訊息。

Express 中的錯誤處理就是設定中介層來監聽問題，並以清楚且安全的方式回應。無論是程式錯誤還是缺少路由，你的應用程式都不應該在沒有備援方案的情況下崩潰。
