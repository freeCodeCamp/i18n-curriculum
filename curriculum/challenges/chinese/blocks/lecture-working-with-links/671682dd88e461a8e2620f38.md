# --description--

路径是指在文件系统中指定文件或目录位置的字符串。 在网页开发中，路径能让开发者链接到诸如图片、样式表、脚本以及其他网页之类的资源。 有绝对路径和相对路径——在指定文件系统中的文件位置时，这两种路径都是必不可少的。 让我们看看两者，这样你就能决定使用哪一个以及在什么时候使用。

绝对路径是指向某一资源的完整链接。 它从根目录开始，包括所有相关目录，最后是文件名和扩展名。 “根目录”指的是层级结构中的顶层目录或文件夹。

如果你要链接到你本地机器上的资源，请使用绝对路径，其中包含该文件的完整目录位置。以下是如何使用绝对路径链接到 `about.html` 文件：

```html
<p>
  Read more on the
  <a
    href="/Users/user/Desktop/fCC/script-code/absolute-vs-relative-paths/pages/about.html"
    >About Page</a
    >
</p>
```

看起来是这样的，因为我们从根目录开始，进入一个名为 `Users` 的文件夹，然后进入一个名为 `user` 的文件夹，再进入一个名为 `Desktop` 的文件夹，然后进入一个名为 `fCC` 的文件夹，再进入一个名为 `script-code` 的文件夹，然后进入一个名为 `absolute-vs-relative-paths` 的文件夹，最后进入一个名为 `pages` 的文件夹，最终获取 `about.html` 文件。

绝对 URL 是用于访问资源的完整地址。它包括协议——可能是 `http`、`https` 和 `file`，以及如果资源位于网络上则包含域名。以下是一个指向 freeCodeCamp 标志的绝对 URL 示例：

```html
<a href="https://design-style-guide.freecodecamp.org/img/fcc_secondary_small.svg">
  View fCC Logo
</a>
```

在这个示例中，协议是 `https` ，域名是 `design-style-guide.freecodecamp.org` ，文件名是 `fcc_secondary_small.svg`。

这是在浏览器地址栏中该 URL 绝对地址的形式：

```sh
file:///Users/user/Desktop/fCC/script-code/absolute-vs-relative-paths/pages/about.html
```

该 URL 包含协议，即 `file://`。  It also includes the path, which looks like this: `/Users/user/Desktop/fCC/script-code/absolute-vs-relative-paths/pages/`, and represents the series of folders that lead to the file. 最后，它还包含了 `about.html` 这个文件名及其扩展名。

绝对路径显示文件系统中一个文件的完整位置，通常用于局部机器上的资源。绝对 URL 包含访问信息——例如协议，以及对于网络资源的域名——这些信息告诉浏览器如何以及从哪里检索资源。

Now, let's look at the relative path. 相对路径指定了文件相对于当前文件所在目录的位置。 它不包含协议或域名，这使得同一网站内的内部链接显得更简洁和灵活。 以下是一个从 `contact.html` 页面链接到 `about.html` 页面的示例，这两个页面都在同一文件夹中：

```html
<p>
  Read more on the
  <a href="about.html">About Page</a>
</p>
```

那么想象你正在浏览 `contact.html` 页面，而由于 `about.html` 页面也在同一位置，所以你只需获取文件名即可。 这是一个使用相对文件路径的例子。

那么，你应该使用哪种以及何时使用：绝对路径、绝对 URL 还是相对路径？以下是你应该遵循的规则：

- 当你想从固定位置引用资源时，例如从你网站的根目录或你本地机器上的已知目录，使用绝对路径。

- 当链接到托管在外部网站上的资源时，请使用绝对 URL。

- 在当链接同一网站内的资源时，请使用相对路径。

- 如果你希望在开发过程中保持代码的整洁并易于维护，请使用相对路径。

- 在本地测试时使用相对路径，以确保在没有网络连接的情况下链接也能正常工作。
