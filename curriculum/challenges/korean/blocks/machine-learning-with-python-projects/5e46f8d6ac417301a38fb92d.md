# --instructions--

`RPS.py` 파일에는 `player`라는 함수가 제공됩니다. 이 함수는 상대방의 마지막 수를 나타내는 문자열 인수("R", "P", 또는 "S")를 받습니다. 함수는 다음에 낼 수를 나타내는 문자열("R", "P", 또는 "S")을 반환해야 합니다.

경기 첫 게임에서는 이전 수가 없으므로 플레이어 함수는 빈 문자열을 인수로 받습니다.

`RPS.py` 파일에는 업데이트해야 할 예제 함수가 있습니다. 예제 함수는 두 개의 인수(`player(prev_play, opponent_history = [])`)로 정의되어 있습니다. 두 번째 인수는 호출될 때 전달되지 않으므로 완전히 선택 사항입니다. 예제 함수에 두 번째 인수(`opponent_history = []`)가 있는 이유는 `player` 함수가 연속 호출될 때 상태를 저장하는 유일한 방법이기 때문입니다. 상대방 기록을 추적하려면 `opponent_history` 인수가 필요합니다.

*힌트: 네 명의 상대를 모두 이기려면 상대의 수에 따라 전략을 바꾸는 여러 전략이 필요할 수 있습니다.*

## 개발

`RPS_game.py`는 수정하지 마세요. 모든 코드는 `RPS.py`에 작성하세요. 개발 중에는 `main.py`를 사용해 코드를 테스트할 수 있습니다.

`main.py`는 `RPS_game.py`에서 게임 함수와 봇을 가져옵니다.

코드를 테스트하려면 `play` 함수를 사용해 게임을 하세요. `play` 함수는 네 개의 인수를 받습니다:

- 서로 대결할 두 플레이어(플레이어는 실제로 함수입니다)
- 경기에서 할 게임 수
- 각 게임의 로그를 보고 싶을 때 사용하는 선택적 인수. `True`로 설정하면 메시지를 볼 수 있습니다.

```py
play(player1, player2, num_games[, verbose])
```

예를 들어, `player`와 `quincy`가 1000게임을 대결하고 각 게임 결과를 보고 싶다면 이렇게 함수를 호출합니다:

```py
play(player, quincy, 1000, verbose=True)
```
