## Lavorare con gli eventi in React

- **Sistema di eventi sintetici**: Questo è il modo in cui React gestisce gli eventi. Funziona come un involucro attorno agli eventi nativi come gli eventi `click`, `keydown` e `submit`. I gestori di eventi in React usano la convenzione di denominazione camel case. (Es. `onClick`, `onSubmit`, ecc.)

Ecco un esempio di utilizzo dell'attributo `onClick` per un elemento `button` in React:

```jsx
function handleClick() {
  console.log("Button clicked!");
}

<button onClick={handleClick}>Click Me</button>;
```

In React, le funzioni gestore di eventi solitamente iniziano con il prefisso `handle` per indicare che si occupano di gestire eventi, come `handleClick` o `handleSubmit`.

Quando un'azione dell'utente attiva un evento, React passa un oggetto Evento Sintetico al tuo gestore. Questo oggetto si comporta molto come l'oggetto Evento nativo in JavaScript vanilla, fornendo proprietà come `type`, `target` e `currentTarget`.

Per evitare comportamenti predefiniti come l'aggiornamento del browser durante un evento `onSubmit`, ad esempio, puoi chiamare il metodo `preventDefault()`:

```jsx
function handleSubmit(event) {
  event.preventDefault();
  console.log("Form submitted!");
}

<form onSubmit={handleSubmit}>
  <input type="text" />
  <button>Submit</button>
</form>;
```

Puoi anche racchiudere una funzione gestore in una funzione freccia in questo modo:

```jsx
function handleDelete(id) {
  console.log("Deleting item:", id);
}

<button onClick={() => handleDelete(1)}>Delete Item</button>;
```

## Lavorare con lo stato e l'hook `useState`

- **Definizione di stato**: In React, lo stato è un oggetto che contiene dati per un componente. Quando lo stato si aggiorna, il componente viene renderizzato di nuovo. React considera lo stato immutabile, quindi non dovresti modificarlo direttamente.  
- **Hook `useState()`**: L'hook `useState` è una funzione che ti permette di dichiarare variabili di stato nei componenti funzionali. Ecco la sintassi di base:

```js
const [stateVariable, setStateFunction] = useState(initialValue);
```

Nella variabile di stato hai quanto segue:

- `stateVariable` contiene il valore corrente dello stato
- `setStateFunction` (la funzione setter) aggiorna la variabile di stato
- `initialValue` imposta lo stato iniziale

Ecco un esempio completo per un componente `Counter`:

```jsx
import { useState } from "react";

function Counter() {
  const [count, setCount] = useState(0);

  return (
    <div>
      <h2>{count}</h2>

      <button onClick={() => setCount(count - 1)}>Decrement</button>
      <button onClick={() => setCount(count + 1)}>Increment</button>
    </div>
  );
}

export default Counter;
```

## Renderizzazione e componenti React

- **Definizione**: In React, la renderizzazione è il processo con cui i componenti appaiono nell'interfaccia utente (UI), di solito nel browser. Il processo di renderizzazione consiste in tre fasi: trigger, render e commit.

La fase di trigger avviene quando React rileva che qualcosa è cambiato e l'interfaccia utente potrebbe aver bisogno di essere aggiornata. Questo cambiamento è spesso dovuto a un aggiornamento dello stato o delle props.

Una volta che il trigger avviene, React entra nella fase di render. Qui React rivaluta i tuoi componenti e decide cosa mostrare. Per farlo, React usa una copia leggera del DOM "reale" chiamata DOM virtuale. Con il DOM virtuale, React può rapidamente verificare cosa deve cambiare nel componente.

La fase di commit è quella in cui React prende le modifiche preparate dal DOM virtuale e le applica al DOM reale. In altre parole, questa è la fase in cui vedi il risultato finale sullo schermo.

## Aggiornare oggetti e array nello stato

- **Aggiornare oggetti nello stato**: Se devi aggiornare un oggetto nello stato, dovresti prima creare un nuovo oggetto o copiare un oggetto esistente, poi impostare lo stato per quel nuovo oggetto. Qualsiasi oggetto inserito nello stato dovrebbe essere considerato di sola lettura. Ecco un esempio di impostazione del nome, età e città di un utente. La funzione `handleChange` viene usata per gestire gli aggiornamenti delle informazioni dell'utente:

```jsx
import { useState } from "react";

function Profile() {
  const [user, setUser] = useState({ name: "John Doe", age: 31, city: "LA" });

  const handleChange = (e) => {
    const { name, value } = e.target;

    setUser((prevUser) => ({...prevUser, [name]: value}));
  };

  return (
    <div>
      <h1>User Profile</h1>
      <p>Name: {user.name}</p>
      <p>Age: {user.age}</p>
      <p>City: {user.city}</p>

      <h2>Update User Age </h2>
      <input type="number" name="age" value={user.age} onChange={handleChange} />

      <h2>Update User Name </h2>
      <input type="text" name="name" value={user.name} onChange={handleChange} />

      <h2>Update User City </h2>
      <input type="text" name="city" value={user.city} onChange={handleChange} />
    </div>
  );
}

export default Profile;
```

- **Aggiornare array nello stato**: Quando aggiorni array nello stato, è importante non modificare direttamente l'array usando metodi come `push()` o `pop()`. Invece dovresti creare un nuovo array quando aggiorni lo stato:

```js
const addItem = () => {
  const newItem = {
    id: items.length + 1,
    name: `Item ${items.length + 1}`,
  };

  // Creates a new array
  setItems((prevItems) => [...prevItems, newItem]);
};
```

Se vuoi rimuovere elementi da un array, dovresti usare il metodo `filter()`, che restituisce un nuovo array dopo aver filtrato ciò che vuoi rimuovere:

```js
const removeItem = (id) => {
  setItems((prevItems) => prevItems.filter((item) => item.id !== id));
};
```

## Consultare valori usando i ref

- **Attributo `ref`**: Puoi consultare un nodo DOM in React usando l'attributo `ref`. Ecco un esempio che mostra un `ref` per mettere il focus su un elemento `input`. La proprietà `current` viene usata per consultare il valore corrente di quel `ref`:

```jsx
import { useRef } from "react";

const Focus = () => {
  const inputRef = useRef(null);

  const handleFocus = () => {
    if (inputRef.current) {
      inputRef.current.focus();
    }
  };

  return (
    <div>
      <input ref={inputRef} type="text" placeholder="Enter text" />
      <button onClick={handleFocus}>Focus Input</button>
    </div>
  );
};

export default Focus;
```

## Lavorare con l'hook `useEffect`

- **Hook `useEffect()`**: In React, un effetto è qualsiasi cosa che accade al di fuori del processo di renderizzazione del componente. Cioè, tutto ciò che React non gestisce direttamente come parte della renderizzazione dell'interfaccia utente. Esempi comuni includono il recupero di dati, l'aggiornamento del titolo della scheda del browser, la lettura o scrittura nella memoria locale del browser, ottenere la posizione dell'utente e molto altro. Queste operazioni interagiscono con il mondo esterno e sono note come effetti collaterali. React fornisce l'hook `useEffect` per permetterti di gestire questi effetti collaterali. `useEffect` ti permette di eseguire una funzione dopo che il componente è stato renderizzato o aggiornato.

```js
import { useEffect } from "react";

useEffect(() => {
 // Your side effect logic (usually a function) goes here
}, [dependencies]);
```

La funzione effetto viene eseguita dopo che il componente è stato renderizzato, mentre l'argomento opzionale `dependencies` controlla quando l'effetto viene eseguito.

Nota che `dependencies` può essere un array di "valori reattivi" (stato, props, funzioni, variabili e così via), un array vuoto o essere omesso del tutto. Ecco come tutte queste opzioni controllano il funzionamento di `useEffect`:

- Se `dependencies` è un array che include uno o più valori reattivi, l'effetto verrà eseguito ogni volta che cambiano.

- Se `dependencies` è un array vuoto, `useEffect` viene eseguito solo una volta quando il componente viene renderizzato per la prima volta.

- Se ometti `dependencies`, l'effetto viene eseguito ogni volta che il componente viene renderizzato o aggiornato.

## Come creare hook personalizzati

- **Hook personalizzati**: Un hook personalizzato ti permette di estrarre logica riutilizzabile dai componenti, come il recupero dati, la gestione dello stato, l'attivazione e gli effetti collaterali come il monitoraggio dello stato online. In React, tutti gli hook integrati iniziano con la parola `use`, quindi il tuo hook personalizzato dovrebbe seguire la stessa convenzione.

Ecco un esempio di creazione di un hook `useDebounce`:

```js
function useDebounce(value, delay) {
  const [debouncedValue, setDebouncedValue] = useState(value);

  useEffect(() => {
    const handler = setTimeout(() => {
      setDebouncedValue(value);
    }, delay);

    return () => {
      clearTimeout(handler);
    };
  }, [value, delay]);

  return debouncedValue;
}

export { useDebounce };
```
