# --description--

JavaScript는 동적 타이핑 언어로, 변수를 선언할 때 데이터 타입을 지정할 필요가 없습니다. 대신, 프로그램이 실행되는 동안 변수에 할당된 값에 따라 타입이 결정됩니다. 이 덕분에 프로그램 전체에서 변수의 타입을 변경할 수 있습니다.

예제를 살펴보겠습니다:

```js
let example = "Hello";
example = 42;
```

이 예제에서는 `example`라는 변수에 문자열 타입이 할당되어 있습니다. 하지만 이후에 value를 숫자로 업데이트합니다.

동적 타이핑의 유연성 덕분에 JavaScript는 빠른 스크립팅에 더 관대하고 다루기 쉽지만, 프로그램이 커질수록 잡기 어려운 버그가 발생할 수도 있습니다.

C#이나 C++ 같은 정적 타이핑 언어에서는 변수를 생성할 때 데이터 타입을 반드시 선언해야 하며, 그 타입은 변경할 수 없습니다.

예를 들어, 변수를 `integer` 타입으로 선언하면 정수 값만 할당할 수 있습니다. 다른 타입을 할당하려 하면 프로그램이 오류를 발생시킵니다.

다음은 C# 언어 예제입니다:

```csharp
int data = 42; // data must always be an integer
data = "Hello"; // This would cause an error in C#
```

동적 타이핑과 정적 타이핑의 차이는 코드의 유연성과 안전성에 있습니다. 동적 타이핑 언어는 유연성을 제공하지만 잠재적인 런타임 오류 위험이 있습니다.

정적 타이핑 언어는 특정 오류를 방지할 수 있는 엄격한 규칙을 적용하지만, 더 많은 사전 선언이 필요하고 타입 변경에 덜 유연합니다.

다음은 `number` 타입으로 변수를 생성한 후 나중에 `string` 타입으로 변경하는 예제입니다:

```js
let data = 100;  // Initially a number
data = "New data";  // Dynamically changes to a string
```

정적 타이핑 언어에서는 데이터 타입이 고정되어 있기 때문에 이런 변경이 허용되지 않습니다.

결론적으로, JavaScript의 동적 타이핑은 변수 타입을 자유롭게 변경할 수 있어 유연성을 제공하지만 실행 중 예상치 못한 오류가 발생할 수 있습니다.

Java 같은 정적 타이핑 언어는 변수 타입을 미리 지정해야 하므로 프로그램 실행 전에 오류를 잡을 수 있지만 유연성은 떨어집니다.
