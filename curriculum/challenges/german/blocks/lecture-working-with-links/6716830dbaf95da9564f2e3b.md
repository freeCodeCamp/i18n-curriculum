# --description--

Sie haben vielleicht schon Links wie `/public/logo.png`, `./script.js` oder `../styles.css` gesehen. Aber was bedeuten diese speziellen Arten von Links? Diese nennt man Dateipfade. Es gibt drei wichtige Syntaxarten, die Sie kennen sollten. Zuerst der Slash, der je nach Betriebssystem ein Backslash (`\`) oder ein Vorwärtsslash (`/`) sein kann. Zweitens der einzelne Punkt (`.`). Und schließlich die zwei Punkte (`..`).

Der Slash wird als „Pfadtrenner“ bezeichnet. Er wird verwendet, um eine Trennung im Text zwischen Ordner- oder Dateinamen anzuzeigen. So weiß Ihr Computer, dass `naomis-files/` auf ein Verzeichnis mit demselben Namen verweist, während `naomis/files/` auf ein `files`-Verzeichnis im `naomis`-Verzeichnis zeigt.

Ein einzelner Punkt zeigt auf das aktuelle Verzeichnis, und zwei Punkte zeigen auf das übergeordnete Verzeichnis. In der Regel sehen Sie einen einzelnen Punkt, um sicherzustellen, dass der Pfad als relativer Pfad erkannt wird. Denken Sie daran, dass Sie in einer vorherigen Lektion über relative und absolute Pfade gelernt haben.

Dagegen werden zwei Punkte häufiger verwendet, um auf Dateien außerhalb des aktuellen Arbeitsverzeichnisses zuzugreifen.

Zum Beispiel, gegeben dieser Dateibaum:

```sh
my-app/
├─ public/
│  ├─ favicon.ico
│  ├─ index.html
├─ src/
│  ├─ index.css
│  ├─ index.js
```

Wenn Ihre `public/index.html`-Datei die `favicon.ico`-Datei laden müsste, würden Sie einen relativen Pfad mit einem einzelnen Punkt verwenden, um auf das aktuelle Verzeichnis zuzugreifen: `./favicon.ico`. Wenn Ihre `public/index.html`-Datei die `index.css`-Datei laden müsste, würden Sie einen relativen Pfad mit zwei Punkten verwenden, um zuerst zum übergeordneten `my-app`-Verzeichnis zu navigieren, dann zum `src`-Verzeichnis und schließlich zu Ihrer Datei: `../src/index.css`.
