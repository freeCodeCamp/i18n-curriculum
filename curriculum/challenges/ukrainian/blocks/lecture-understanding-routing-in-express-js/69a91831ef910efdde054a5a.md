# --description--

У Express параметри маршруту — це динамічні значення, вбудовані в шлях URL. Вони дозволяють вашому застосунку захоплювати частини URL, які можуть змінюватися з кожним запитом. Це корисно для створення динамічних сторінок, наприклад, для відображення конкретного користувача, блогу чи продукту залежно від URL.

Ось приклад:

```javascript
// Route with a parameter in the path
app.get("/post/:postId", (req, res) => {
  const postId = req.params.postId;
  res.send(`Viewing post with ID: ${postId}`);
});
```

Тут маршрут слухає GET-запити до `/post/:postId`.

* `:postId` — це параметр маршруту, заповнювач у URL, який буде замінений на фактичне значення. Наприклад, запит до `/post/42` відповідає цьому маршруту, і `42` стає значенням `postId`.
    
* Ви можете отримати доступ до параметрів маршруту за допомогою `req.params`. Об’єкт `req.params` містить усі параметри маршруту, і в цьому випадку `req.params.postId` містить значення з URL, `42`.
    

Нарешті, `res.send()` відправляє відповідь клієнту. Тож якщо URL — `/post/42`, клієнт побачить `Viewing post with ID: 42`.

Отже, як бачите, параметри маршруту роблять ваші маршрути гнучкими та динамічними без потреби створювати окремий маршрут для кожного можливого значення.

Тепер, коли ви розумієте, як працюють параметри маршруту, повернемося до прикладу з минулого уроку і подивимося, як можна використовувати динамічні маршрути для різних типів меню:

```javascript
app.get("/menu/:category", (req, res) => {
  const category = req.params.category;
  res.send(`You are viewing the ${category} menu!`);
});
```

Ось пояснення, як це працює:

1. Користувач відвідує [`http://localhost:3000/menu/drinks`](http://localhost:3000/menu/drinks).
    
2. Сервер співставляє цей запит з маршрутом `/menu/:category` і захоплює параметр `category` як `drinks`.
    
3. Сервер відправляє цю відповідь назад у браузер: `You are viewing the drinks menu!`
    

Аналогічно, якщо користувач відвідує [`http://localhost:3000/menu/food`](http://localhost:3000/menu/food), сервер відповідає так: `You are viewing the food menu!`

Параметри маршруту дозволяють відображати динамічний вміст залежно від URL. Замість створення окремого маршруту для кожної категорії, ви можете використовувати один маршрут для обробки кількох категорій, таких як напої, їжа, десерти та інші, просто змінюючи URL.
