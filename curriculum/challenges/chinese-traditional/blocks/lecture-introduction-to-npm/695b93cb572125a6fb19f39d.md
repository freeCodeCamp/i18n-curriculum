# --description--

保持你的相依性為最新狀態——並且知道如何安裝特定版本——對於維持你的專案穩定性非常重要。它有助於你避免意外的錯誤，並在新版本釋出時給你更多控制權。

要安裝特定版本的軟體包，請使用此格式：

```bash
npm install package@version-number
```

例如，如果你的專案需要較舊版本的 Express，例如 `4.21.2`，你可以這樣安裝：

```bash
npm install express@4.21.2
```

如果你沒有包含版本號，npm 將自動安裝可用的最新版本。

如果你不確定存在哪些版本，可以使用此指令檢查。

```bash
npm view package-name versions
```

例如：

```bash
npm view express versions
```

要更新軟體包，你可以單獨更新它們或一次全部更新。在此之前，先查看哪些已過期會很有幫助：

```bash
npm outdated
```

這將顯示哪些依賴項有較新的版本可用，你應該會看到類似這樣的輸出：

```bash
user@Kolade ~/Desktop/fCC/script-code/node/npm/install-spec-version  % npm outdated
Package  Current  Wanted  Latest  Location              Depended by
chalk      5.5.0   5.5.0   5.6.2  node_modules/chalk    install-spec-version
cleave     1.0.1   1.0.1   1.0.2  node_modules/cleave   install-spec-version
express   4.21.2  4.21.2   5.2.1  node_modules/express  install-spec-version
```

一旦你檢查完哪些依賴項已過時，你就可以開始更新它們。

要更新單一相依性，請使用以下指令：

```bash
npm update package-name
```

這會僅在你的 `package.json` 檔案允許的版本範圍內更新軟體包。 例如，要更新 Express，你可以使用以下指令：

```bash
npm update express
```

如果你想跳過版本範圍並安裝軟體包的最新版本，請使用以下指令：

```bash
npm install package-name@latest
```

要更新所有落在允許版本範圍內的相依性，請執行：

```bash
npm update
```

但是，請注意，如果你的 `package.json` 不允許，`npm update` 不會將軟體包升級到新的主要版本。主要版本通常包含破壞性變更，因此 npm 不會自動套用它們。

如果你特別想要最新的主要版本，必須執行：

```bash
npm install package-name@latest
```

這會強制 npm 安裝最新版本，不論你的 `package.json` 中的版本範圍為何。
