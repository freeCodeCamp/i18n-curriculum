# --interactive--

L'attributo `aria-controls` viene usato per creare una relazione programmabile tra un elemento che controlla la presenza di un altro elemento nella pagina e l'elemento che controlla. Questa relazione può aiutare gli utenti di lettori di schermo a capire meglio come funziona il controllo. Gli usi comuni includono un insieme di schede che controllano la visibilità di diverse sezioni di contenuto o un pulsante che attiva o disattiva la visibilità di un menu.

Diamo un’occhiata a un esempio per vedere come funziona. In questo esempio di interfaccia a schede, abbiamo un elemento `div` con un insieme di tre pulsanti:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<div role="tablist">
  <button role="tab" id="tab1" aria-controls="section1" aria-selected="true">
    Tab 1
  </button>
  <button role="tab" id="tab2" aria-controls="section2" aria-selected="false">
    Tab 2
  </button>
  <button role="tab" id="tab3" aria-controls="section3" aria-selected="false">
    Tab 3
  </button>
</div>
```

```css
[role="tablist"] {
  display: flex;
  border-bottom: 2px solid #ccc;
  margin-bottom: 1em;
  gap: 4px;
}

[role="tab"] {
  background: #f5f5f5;

}

  background: #e8f0fe;
  outline: none;
}

[role="tab"][aria-selected="true"] {
  background: #fff;
  border-color: #ccc;
  border-bottom: 2px solid #fff;
  color: #000;
  font-weight: 600;
  position: relative;
  top: 1px; 
}

[role="tab"]:focus-visible {
  outline: 2px solid #0078d4;
  outline-offset: 2px;
}

```

:::

L'elemento `div` usa `role="tablist"` per indicare che funge da contenitore per un gruppo di schede.

Ogni pulsante rappresenta una scheda e ha un attributo `role` impostato su `tab`. Nella maggior parte delle interfacce a schede, il primo pannello della scheda è visibile per impostazione predefinita, quindi il primo pulsante della scheda ha un attributo `aria-selected` impostato su `true` per indicare che la sezione di contenuto associata è attualmente visibile. L'attributo `aria-controls` viene usato per associare ogni pulsante alla sezione di contenuto che controlla.

Ecco le tre sezioni di contenuto corrispondenti alle schede:

```js
const tabs = document.querySelectorAll('[role="tab"]');
const tabList = document.querySelector('[role="tablist"]');

tabs.forEach(tab => {
  tab.addEventListener('click', () => {
    activateTab(tab);
  });

  tab.addEventListener('keydown', (e) => {
    const key = e.key;
    const currentIndex = Array.from(tabs).indexOf(tab);
    let newIndex = null;

    if (key === 'ArrowRight') {
      newIndex = (currentIndex + 1) % tabs.length;
    } else if (key === 'ArrowLeft') {
      newIndex = (currentIndex - 1 + tabs.length) % tabs.length;
    } else if (key === 'Home') {
      newIndex = 0;
    } else if (key === 'End') {
      newIndex = tabs.length - 1;
    }

    if (newIndex !== null) {
      tabs[newIndex].focus();
      activateTab(tabs[newIndex]);
    }
  });
});

function activateTab(tab) {
  const tabPanels = document.querySelectorAll('[role="tabpanel"]');

  tabs.forEach(t => {
    t.setAttribute('aria-selected', 'false');
    t.setAttribute('tabindex', '-1');
  });

  tabPanels.forEach(panel => panel.hidden = true);

  tab.setAttribute('aria-selected', 'true');
  tab.removeAttribute('tabindex');

  const panelId = tab.getAttribute('aria-controls');
  const panel = document.getElementById(panelId);
  panel.hidden = false;
  tab.focus();
}

```

:::

Ogni sezione di contenuto ha un attributo `role` impostato su `tabpanel` e un attributo `aria-labelledby` che punta alla scheda corrispondente per dare al pannello un nome accessibile. L'attributo `hidden` viene usato per nascondere le sezioni di contenuto che non sono attualmente selezionate. Ogni ID della sezione corrisponde al valore dell'attributo `aria-controls` sul pulsante corrispondente. L'ID `section1` corrisponde all'attributo `aria-controls` sul primo pulsante, `section2` corrisponde all'attributo `aria-controls` sul secondo pulsante e `section3` corrisponde all'attributo `aria-controls` sul terzo pulsante. Così si stabilisce l’associazione tra le schede e le loro sezioni di contenuto.

Per passare tra le diverse sezioni dovrai usare JavaScript per aggiornare l'attributo `hidden` sulla sezione e l'attributo `aria-selected` sul pulsante in base a quale sezione è attualmente visibile. Solo una sezione può essere visibile alla volta e non dovrebbe avere l'attributo `hidden` mentre `aria-selected` dovrebbe essere impostato su `true` sul suo pulsante. Le restanti sezioni nascoste dovrebbero avere tutte l'attributo `hidden` e `aria-selected` dovrebbe essere impostato su `false` sui loro pulsanti.

Le schede sono un caso d’uso comune per l'attributo `aria-controls`, ma può essere usato in altri scenari in cui un elemento controlla la visibilità o il comportamento di un altro elemento. Altri esempi includono gli accordion, i menu a discesa e i modali.

La prossima volta che progetti un’interfaccia utente che coinvolge il controllo della visibilità degli elementi, considera di usare l'attributo `aria-controls` per stabilire la relazione tra l’elemento che controlla e l’elemento controllato.
