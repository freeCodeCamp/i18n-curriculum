# --description--

在本課程中，你將學習如何為物件和陣列添加型別。

讓我們先來看看陣列。

這是一個程式語言陣列的範例： 

```js
const programmingLanguages = ["Java", "C++", "Python"];
```

如果你想將該陣列明確地定義為字串陣列，那麼你可以將它定義為 `string[]`：

```ts
const programmingLanguages:string[] = ["Java", "C++", "Python"];
```

如你先前所學，如果你只寫這個，TypeScript 就會聰明地推斷型別為字串陣列：

```js
const programmingLanguages = ["Java", "C++", "Python"];
```

但明確地指定型別會在你嘗試像這樣將另一種型別添加到陣列時有所幫助：

```ts
// Type 'number' is not assignable to type 'string'.(2322)
const programmingLanguages:string[] = ["Java", 9, "Python"];
```

另一種輸入陣列的方式是使用此語法：

```ts
const programmingLanguages:Array<string> = ["Java", "C++", "Python"];
```

基本上，這兩種語法是相同的，兩者之間的選擇通常是偏好的問題。

現在，讓我們來看看如何為物件打字。

這是一個物件字面值的範例：

```js
const developer = {
  firstName: "Jessica",
  isEmployed: true
}
```

要明確地為物件屬性指定型別，你可以使用這個語法：

```ts
const developer: { firstName: string, isEmployed: boolean } = {
  firstName: "Jessica",
  isEmployed: true
}
```

如果你嘗試從該物件添加或移除屬性，則會收到類似這樣的錯誤：

```ts
// Property 'isEmployed' is missing in type '{ firstName: string; }' but required in type '{ firstName: string; isEmployed: boolean; }'
const developer: { firstName: string, isEmployed: boolean } = {
  firstName: "Jessica",
}
```

你可以在屬性名稱後使用 `?` 來使物件屬性變成選用，如下所示：

```ts
const developer: { firstName: string, isEmployed?: boolean } = {
  firstName: "Jessica",
}
```

TypeScript 不會再顯示錯誤，因為 `isEmployed` 屬性已被標記為選用。

到目前為止，所有的物件範例都有明確的限制屬性。這表示我們明確列出了物件允許擁有的每個屬性：

```ts
const developer: { firstName: string, isEmployed: boolean } = {
  firstName: "Jessica",
  isEmployed: true
}
```

但是如果你事先不知道所有的屬性名稱怎麼辦？如果你想要一個可以有任意數量屬性的物件，但所有的值都必須遵循特定的型別怎麼辦？

好吧，有兩種方法可以達成這個目的。第一種方法是使用 `Record`，它是 TypeScript 中內建的泛型工具型別。

這是基本語法：

```ts
Record<Keys, ValueType>
```

第一個型別表述鍵的型別，而第二個型別表述值的型別。

這裡是一個使用 `Record` 來建立 `userRoles` 物件的範例：

```ts
const userRoles: Record<string, string> = {
  admin: "full-access",
  editor: "limited-access",
  viewer: "read-only"
};
```

這個 `userRoles` 物件可以有任何字串鍵，且所有值必須是字串。如果你嘗試指定非字串值，將會出現錯誤：

```ts
// Type 'number' is not assignable to type 'string'
const userRoles: Record<string, string> = {
  admin: 1
};
```

第二種寫出相同想法的方法是使用索引標記：

```ts
const obj: { [key: string]: string } = {};
```

上述語法表示任何具有字串鍵的屬性必須有字串值。以下是 `settings` 物件的範例：

```ts
const settings: { [key: string]: string } = {
  theme: "dark",
  language: "en",
  layout: "grid"
};
```

就像之前一樣，如果你有錯誤的值型別，TypeScript 會顯示錯誤訊息：

```ts
// Type 'boolean' is not assignable to type 'string'
const settings: { [key: string]: string } = {
  darkMode: true
};
```

`Record` 和索引標記在本質上是相同的。所以在它們之間做選擇大多是偏好的問題。
