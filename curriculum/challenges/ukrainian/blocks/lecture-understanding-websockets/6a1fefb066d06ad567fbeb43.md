# --description--

WebSockets дозволяють клієнту і серверу спілкуватися в режимі реального часу.

Зазвичай клієнтські застосунки, такі як веб- і мобільні застосунки, використовують HTTP. При HTTP клієнт надсилає запит, а сервер повертає відповідь. Це називається циклом запит-відповідь, як ви вже вивчали.

Після встановлення з’єднання і обміну даними з’єднання закривається. Це добре працює для більшості випадків, наприклад, завантаження сторінки, отримання або відправлення даних. Але для застосунків у режимі реального часу це стає повільним і ненадійним.

Наприклад, уявіть чат-застосунок. Якщо ви вирішите використовувати HTTP для нього, клієнт повинен періодично запитувати сервер про нові дані, при цьому з’єднання відкривається, дані надходять, а потім з’єднання закривається для кожного повідомлення.

Цю проблему вирішують WebSockets. Вони створюють постійне з’єднання між клієнтом і сервером. Після відкриття з’єднання обидві сторони можуть обмінюватися даними в будь-який момент у каналі реального часу, доки одна зі сторін не надішле сигнал закриття іншій.

Хоча WebSockets відрізняються від HTTP, кожне WebSocket-з’єднання починається з HTTP у процесі, який називається рукопотисканням.

Клієнт запитує оновлення з’єднання до WebSocket. Коли сервер погоджується, він надсилає `101 Switching Protocols` відповідь, яка оновлює з’єднання до WebSocket. Після цього з’єднання залишається відкритим, і починається обмін у режимі реального часу.

Потік виглядає так:

1. Клієнт підключається до сервера через HTTP  
2. Сервер приймає і оновлює з’єднання до WebSocket  
3. З’єднання залишається відкритим  
4. Обидві сторони обмінюються даними у будь-який момент, доки з’єднання не закриється  

Цей потік є двонапрямленим, а не запит-відповідь, як у HTTP.

```md
Client <---> Server
```

При HTTP потік зазвичай слідує моделі запит-відповідь:

```md
Client ---> Request
Server ---> Response
```

Щодо відмінностей WebSockets від HTTP, HTTP — це як обмін листами, а WebSocket — як телефонна розмова. Ви пишете листа, чекаєте відповіді, і розмова закінчується. Але під час телефонної розмови ви і співрозмовник говорите в режимі реального часу.

WebSockets живлять застосунки реального часу, такі як чат-застосунки, онлайн-ігри з багатьма гравцями, оновлення спортивних подій, інформаційні панелі фондового ринку та спільні інструменти, як Google Docs.
