# --interactive--

在 JavaScript 中，`null` 和 `undefined` 是兩種不同的資料型別，表述值的缺失，但它們在比較中表現不同。了解這些型別在各種比較情境中的互動方式，對於撰寫強固且無錯誤的程式碼至關重要。

讓我們從 `undefined` 型別開始。當變數已宣告但尚未指定值時，該變數即為 `undefined`。這是未初始化變數和未提供引數的函式參數的預設值。

另一方面，`null` 型別是一種指定值，用來表述刻意的無值。它常用來顯示變數刻意沒有值。

當使用等號運算子（`==`）比較 `null` 和 `undefined` 時，JavaScript 會執行型別強制轉換。這表示它會嘗試將運算元轉換為相同的型別後再進行比較。在這種情況下，`null` 和 `undefined` 被視為相等：

:::interactive_editor

```js
console.log(null == undefined); // true
```

:::

然而，當使用嚴格等號運算子（`===`），該運算子會檢查值和型別且不進行型別強制轉換時，`null` 和 `undefined` 並不相等：

:::interactive_editor

```js
console.log(null === undefined); // false
```

:::

在撰寫條件述語或執行相等性檢查時，記住這個差異非常重要。當使用等號運算子（`==`）比較 `null` 或 `undefined` 與其他值時，行為可能會出乎意料。<dfn>例如</dfn>：

:::interactive_editor

```js
console.log(null == 0);  // false
console.log(null == ''); // false
console.log(undefined == 0); // false
console.log(undefined == ''); // false
```

:::

這些比較傳回 `false`，因為使用等號運算子時，`null` 和 `undefined` 只會彼此（以及自身）相等。`null` 在其他比較中的行為尤其棘手：

:::interactive_editor

```js
console.log(null > 0);  // false
console.log(null == 0); // false
console.log(null >= 0); // true
```

:::

另一方面，`undefined` 在數值上下文中總是轉換為 `NaN`，這使得所有與 `undefined` 的數值比較都傳回 `false`：

:::interactive_editor

```js
console.log(undefined > 0);  // false
console.log(undefined < 0);  // false
console.log(undefined == 0); // false
```

:::

鑑於這些細微差異，通常建議在比較值時使用嚴格等號運算子，特別是在處理 `null` 和 `undefined` 時。這種方法有助於避免意外的型別強制轉換，並使你的程式碼行為更可預測。

總結來說，雖然 `null` 和 `undefined` 都用來表述值的缺失，但它們在比較時的行為不同。理解這些差異是撰寫清晰且無錯誤 JavaScript 程式碼的關鍵。
