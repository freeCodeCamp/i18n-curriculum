# --description--

これまでに変更をコミットしてリモートリポジトリにプッシュする方法を学びましたが、それはすべてデフォルトの`main`ブランチで行ってきました。小規模なプロジェクトならそれでも問題ありませんが、他のオープンソースプロジェクトに貢献したり、プロジェクトが大きくなるにつれて、ブランチは不可欠になります。

例えば、リポジトリをフォークして自分のフォークの`main`ブランチにコミットを行った場合、元のリポジトリが受け入れていない変更がフォークに残ることがあります。そうなると、元のリポジトリで受け入れられた変更を自分のブランチに取り込むのが非常に難しくなります。また、自分のプロジェクトで作業しているときに、ブランチを使わずに異なるタスクを切り替えるのはずっと大変です。どの変更がどの機能や修正に関係しているかを手動で管理しなければならなくなります。

では、ブランチとは具体的に何でしょうか？`main`ブランチを高速道路に例えると、各コミットはその高速道路に入る車のようなものです。新しい機能を作るなど何かを試したい場合は、高速道路から少し降りて別の道を行く必要があります。

ブランチは基本的にメインの履歴からの分岐で、新しい機能やバグ修正のために自由に変更を試せる場所です。変更に満足したら、そのブランチを`main`にマージして高速道路に戻れます。満足しなければ、そのブランチを削除して`main`に切り替え、変更を取り込まずに戻れます。

新しいブランチを作成する前に、`git branch`コマンドで現在のブランチを確認します。出力は次のようになるかもしれません：

```sh
* main
```

今はデフォルトの`main`ブランチが1つだけあります。アスタリスク（`*`）は現在「チェックアウト」しているブランチを示しています。これについては後で詳しく学びます。

`git branch`コマンドに新しいブランチ名を引数として渡すことで、新しいブランチを作成できます：

```sh
git branch feature
```

これで`feature`という名前の新しいブランチが作成されました。`git branch`で再度ブランチを確認してみましょう：

```sh
  feature
* main
```

新しいブランチが見えますが、まだ`main`ブランチにいます。新しいブランチに「チェックアウト」して切り替える必要があります。チェックアウトすると、その時点のファイルの状態がGitによって読み込まれます。

次のように新しいブランチをチェックアウトしましょう：

```sh
git checkout feature
```

または

```sh
git switch feature
```

そしてもう一度`git branch`を実行すると：

```sh
* feature
  main
```

アスタリスクが現在「チェックアウト」しているブランチに移動しているのがわかります。

新しいブランチを作成したら、すぐにチェックアウトしたいことがよくあります。ブランチを作成してからチェックアウトする2つのコマンドを使う代わりに、1つのコマンドで作成と切り替えを同時に行うこともできます：

```sh
git checkout -b feature
```

または

```sh
git switch -c feature
```

これで新しいブランチを作成してチェックアウトしました。このブランチは`main`ブランチのクローンで、作成時点の`main`と同じコードをすべて持っています。これで`main`ブランチに影響を与えずにこのブランチにコードを追加できます。つまり、新しい機能を作るために高速道路から降りた状態です。`git status`を確認すると、次のようになっています：

```sh
On branch feature
nothing to commit, working tree clean
```

次に、`feature.md`ファイルを作成します。次のコマンドを実行してください：

```sh
echo "This is our new feature" > feature.md
```

するとステータスは次のようになります：

```sh
On branch feature
Untracked files:
  (use "git add <file>..." to include in what will be committed)
	feature.md

nothing added to commit but untracked files present (use "git add" to track)
```

次に、新しい機能をコミットする準備です。まず、ファイルをステージングに追加します：

```sh
git add feature.md
```

次に、変更をコミットします：

```sh
git commit -m "my new feature"
```

最後に、`feature`ブランチをリポジトリにプッシュします：

```sh
git push -u origin feature
```

`-u`フラグはローカルの`feature`ブランチとリモートのブランチを連携させるので、今後は`git push`だけでプッシュできます。
