# --interactive--

У попередньому уроці ви дізналися, як працювати з функціями — це повторно використовувані частини коду, які допомагають зробити ваш код більш модульним, легшим для підтримки та ефективнішим. Усі попередні приклади використовували звичайний синтаксис функцій, як-от:

```js
function greetings(name) {
  console.log("Hello, " + name + "!");
}
```

Але інший спосіб написання функцій у JavaScript — це створення виразу стрілкової функції. Ось як ви можете переписати попередній приклад, щоб використати синтаксис стрілкової функції:

```js
const greetings = (name) => {
  console.log("Hello, " + name + "!");
};
```

У цьому оновленому прикладі ми створюємо змінну `const` з назвою `greetings` і призначаємо їй анонімну функцію. Більшість синтаксису буде вам знайомою, окрім відсутнього ключового слова `function` і додавання стрілки (`=>`) між параметром `name` і тілом функції. Якщо у вашому списку параметрів лише один параметр, то ви можете прибрати круглі дужки ось так:

```js
const greetings = name => {
  console.log("Hello, " + name + "!");
};
```

Якщо у вашої стрілкової функції немає параметрів, тоді потрібно використовувати круглі дужки ось так:

```js
const greetings = () => {
  console.log("Hello");
};
```

Коли ви вперше вивчали функції, потрібно було обгорнути тіло функції у фігурні дужки. Але якщо тіло функції містить лише один рядок коду, ви можете прибрати фігурні дужки ось так:

```js
const greetings = name => console.log("Hello, " + name + "!");
```

Важливо зазначити, що прибирати круглі дужки і фігурні дужки у звичайному синтаксисі функцій не вийде. Ви отримаєте помилки, якщо спробуєте зробити щось подібне:

```js
// This will produce syntax errors 
function greetings name console.log("Hello, " + name + "!");
```

Такі однорядкові функції працюють лише зі стрілковим синтаксисом. Ще одна ключова концепція — це інструкція `return`. Ось приклад використання стрілкового синтаксису для обчислення площі:

:::interactive_editor

```js
const calculateArea = (width, height) => {
  const area = width * height;
  return area;
};

console.log(calculateArea(5, 3)); // 15
```

:::

Ми створюємо змінну всередині функції з назвою `area` і потім повертаємо цю змінну. Але ми можемо трохи спростити код і повертати сам розрахунок:

:::interactive_editor

```js
const calculateArea = (width, height) => {
  return width * height;
}; 

console.log(calculateArea(5, 3)); // 15
```

:::

Якщо ви спробуєте прибрати фігурні дужки і розмістити розрахунок в одному рядку, то отримаєте повідомлення про помилку `Uncaught SyntaxError: Unexpected token 'return'`:

```js
const calculateArea = (width, height) => return width * height;
```

Причина цієї помилки в тому, що потрібно прибрати інструкцію `return`. Коли ви видалите цю інструкцію `return`, помилка зникне, і функція все одно неявно поверне результат розрахунку.

```js
const calculateArea = (width, height) => width * height;
```

Отже, коли слід використовувати синтаксис стрілкових функцій? Це залежить. Багато розробників послідовно використовують його у своїх особистих проєктах. Однак у командній роботі вибір зазвичай залежить від того, чи використовує існуюча кодова база звичайні функції чи стрілкові. У майбутніх уроках ми розглянемо, коли варто використовувати стрілкові функції, а коли їх краще уникати.
