# --description--

스택과 큐는 컴퓨터 과학에서 흔히 사용되는 자료 구조입니다.

이들은 요소를 추가하고 제거하는 데 특정 규칙을 따르는 선형 자료 구조입니다.

## Stacks

먼저 **스택**부터 시작해 보겠습니다.

**스택**은 후입선출(LIFO) 자료 구조입니다.

즉, 스택에 마지막으로 추가된 요소가 가장 먼저 제거됩니다.

스택은 위와 아래, 두 끝이 있습니다.

요소는 스택의 위쪽에서 추가되고 제거됩니다.

스택을 접시 더미로 생각할 수 있는데, 접시는 더미의 위쪽에만 올릴 수 있고 위쪽에서만 꺼낼 수 있습니다.

<img src="https://cdn.freecodecamp.org/curriculum/lecture-transcripts/how-do-stacks-and-queues-work-1.png" alt="stack data structure visualization">

요소를 추가하고 제거하는 이 작업들은 이 맥락에서 특별한 이름을 가집니다.

스택에 요소를 추가하는 작업을 "push" 연산이라고 합니다. 스택의 위쪽에 요소를 추가할 때 "push"한다고 말합니다.

<img src="https://cdn.freecodecamp.org/curriculum/lecture-transcripts/how-do-stacks-and-queues-work-2.png" alt="stack push operation visualization">

스택에서 요소를 제거하는 작업을 "pop" 연산이라고 합니다. 스택의 위쪽에서 요소를 제거할 때 "pop"한다고 말합니다.

<img src="https://cdn.freecodecamp.org/curriculum/lecture-transcripts/how-do-stacks-and-queues-work-3.png" alt="stack pop operation visualization">

스택의 아래쪽에서는 실제로 어떤 작업도 수행하지 않지만 참조용으로 남겨둡니다.

push와 pop 연산의 시간 복잡도는 일반적으로 `O(1)`, 즉 상수 시간 복잡도입니다.

요소를 스택에 push할 때는 단순히 위쪽에 추가됩니다.

요소를 스택에서 pop할 때는 위쪽의 요소가 제거됩니다.

따라서 이 연산들을 수행하는 데 걸리는 시간은 스택 크기에 상관없이 일정합니다.

push와 pop 연산의 공간 복잡도도 보통 상수 `O(1)`입니다. 이는 이 연산들을 수행하는 데 필요한 메모리 양이 스택 크기와 무관하게 일정하다는 뜻입니다.

## Queues

스택에 대해 더 알게 되었으니, 이제 **큐**에 대해 배워 보겠습니다.

큐는 선입선출(FIFO) 선형 자료 구조입니다. 즉, 큐에 가장 먼저 추가된 요소가 가장 먼저 제거됩니다.

큐는 앞과 뒤, 두 끝이 있습니다.

<img src="https://cdn.freecodecamp.org/curriculum/lecture-transcripts/how-do-stacks-and-queues-work-4.webp" alt="queue data structure visualization">

요소는 큐의 뒤쪽에 추가되고 앞쪽에서 제거됩니다.

큐를 슈퍼마켓 계산대에서 결제하려고 줄 서 있는 사람들로 생각할 수 있습니다. 줄의 맨 앞 사람이 가장 먼저 계산대에 가고, 새로 온 사람들은 줄의 끝에 합류합니다.

요소를 추가하고 제거하는 작업들은 큐 맥락에서 특별한 이름을 가집니다.

큐의 뒤쪽에 요소를 추가하는 작업을 "enqueue" 연산이라고 합니다.

enqueue 연산에서는 새 요소가 큐의 끝에 추가되어 줄의 끝이 됩니다.

<img src="https://cdn.freecodecamp.org/curriculum/lecture-transcripts/how-do-stacks-and-queues-work-5.png" alt="queue enqueue operation visualization">

큐의 앞쪽에서 요소를 제거하는 작업을 "dequeue" 연산이라고 합니다.

dequeue 연산에서는 큐의 앞쪽 요소가 제거되고, 다음 요소가 새 앞쪽이 됩니다.

<img src="https://cdn.freecodecamp.org/curriculum/lecture-transcripts/how-do-stacks-and-queues-work-6.png" alt="queue dequeue operation visualization">

enqueue와 dequeue 연산의 시간 복잡도는 `O(1)`, 즉 상수 시간입니다. 이 연산들을 수행하는 데 걸리는 시간은 큐 크기와 무관하게 일정합니다.

enqueue와 dequeue 연산의 공간 복잡도도 보통 상수 `O(1)`입니다. 이는 이 연산들을 수행하는 데 필요한 메모리 양이 큐 크기와 무관하게 일정하다는 뜻입니다.

스택과 큐는 컴퓨터 과학에서 요소를 조직하고 관리하는 데 사용되는 자료 구조입니다. 이를 이해하는 것은 다양한 프로그래밍 애플리케이션에서 효율적인 알고리즘을 만드는 데 필수적입니다.
