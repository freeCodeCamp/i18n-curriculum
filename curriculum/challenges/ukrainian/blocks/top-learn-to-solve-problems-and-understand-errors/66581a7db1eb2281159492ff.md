# --description--

Ще одна важлива частина помилки — це стек викликів. Він допомагає зрозуміти, коли саме в застосунку виникла помилка і які функції були викликані, що призвело до цієї помилки. Наприклад, якщо у вас є такий код:

```javascript
const a = 5;
const b = 10;

function add() {
  return c;
}

function print() {
  add();
}

print();
```

Наша функція `print()` має викликати `add()`, яка повертає змінну з ідентифікатором `c`, яка наразі не оголошена. Відповідна помилка виглядає так:

<img src="https://cdn.statically.io/gh/TheOdinProject/curriculum/284f0cdc998be7e4751e29e8458323ad5d320303/foundations/javascript_basics/understanding_errors/imgs/01.png" width="100%" alt="An error showing with a stacktrace in the developer console">

Стек викликів повідомляє нам, що:

1. `c is not defined` у області видимості `add()`, яка оголошена на лінії 5.
1. `add()` був викликаний `print()`, яка була оголошена на лінії 9.
1. `print()` сама була викликана на лінії 12.

Отже, стек викликів дозволяє відстежити розвиток помилки до її джерела, яким тут є оголошення `add()`.

# --questions--

## --text--

Як стек викликів допомагає налагодити помилку у наведеному коді JavaScript?

## --answers--  

Стек викликів показує, що функція `add()` правильно повертає значення `c`.

---

Стек викликів вказує, яку лінію коду потрібно відредагувати, щоб виправити синтаксичну помилку.

---

Стек викликів надає детальний шлях викликів функцій, що призвели до помилки, допомагаючи визначити, де `c` неправильно використано.

---

Стек викликів лише вказує, що `print()` було викликано, але не дає деталей про помилку в `add()`.
