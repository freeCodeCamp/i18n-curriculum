# --description--

Bevor Sie Versionskontrolle und Seiten wie GitHub wirklich nutzen können, müssen Sie Git installiert und einsatzbereit haben.

Um zu überprüfen, ob Git bereits auf Ihrem Rechner installiert ist, können Sie den folgenden Befehl im Terminal ausführen:

```sh
git --version
```

Wenn Sie eine Versionsnummer sehen, ist Git installiert. Andernfalls müssen Sie es installieren.

Bei Linux-Systemen ist Git oft standardmäßig vorinstalliert. Falls nicht, können Sie es mit Paketmanager-Befehlen wie `sudo apt-get install git` oder `sudo pacman -S git` installieren.

Für Mac-Nutzer können Sie Git über Homebrew mit `brew install git` installieren oder den ausführbaren Installer von der Git-Website herunterladen.

Für Windows können Sie den ausführbaren Installer von der Git-Website herunterladen. Alternativ, wenn Sie Chocolatey eingerichtet haben, können Sie `choco install git.install` in PowerShell ausführen. Beachten Sie, dass Sie unter Windows auch Git Bash herunterladen sollten, um eine Unix-ähnliche Shell-Umgebung zu haben.

Um sicherzugehen, dass die Installation funktioniert hat, führen Sie den `git --version`-Befehl erneut im Terminal aus.

Sobald Git installiert ist, müssen Sie einige Änderungen an der Konfiguration vornehmen. Diese Änderungen müssen Sie nur einmal auf Ihrem Computer vornehmen und nicht bei jedem Upgrade erneut.

`git config` wird verwendet, um Konfigurationsvariablen festzulegen, die bestimmen, wie Git auf Ihrem Rechner arbeitet. Um Ihre aktuellen Einstellungvariablen und deren Speicherort auf Ihrem System anzuzeigen, können Sie den folgenden Befehl ausführen:

```sh
git config --list --show-origin
```

Im Moment sollten Sie nur systemweite Konfigurationseinstellungen sehen, wenn Sie Git gerade zum ersten Mal installiert haben. Falls Sie Git bereits vor dieser Lektion installiert hatten, sehen Sie möglicherweise Benutzereinstellungen für einen Benutzernamen und eine E-Mail-Adresse.

Um Ihren Benutzernamen zu setzen, führen Sie folgenden Befehl aus:

```sh
git config --global user.name "Jane Doe"
```

Das `--global`-Flag wird hier verwendet, um den Benutzernamen für alle Projekte auf Ihrem System zu setzen, die Git verwenden. Wenn Sie den Benutzernamen für ein bestimmtes Projekt überschreiben möchten, können Sie den Befehl in diesem Projektverzeichnis ohne das `--global`-Flag ausführen.

Um die Benutzer-E-Mail-Adresse zu setzen, führen Sie folgenden Befehl aus:

```sh
git config --global user.email janedoe@example.com
```

Die Konfiguration Ihres Benutzernamens und Ihrer E-Mail-Adresse ist wichtig, weil Git diese Informationen für jeden Commit verwendet, den Sie in einem Projekt machen. Mehr über Commits lernen Sie in einer zukünftigen Lektion.

Eine weitere Konfiguration ist der bevorzugte Editor, den Git verwenden soll. Hier ein Beispiel, um den Editor auf Emacs zu setzen:

```sh
git config --global core.editor emacs
```

Weitere Optionen sind Vim, Nano, VS Code und mehr. Wenn Sie zum Beispiel VS Code als Editor verwenden, können Sie Git so konfigurieren:

```sh
git config --global core.editor "code --wait"
```

Wenn Sie keinen bevorzugten Editor setzen, verwendet Git den Standardeditor Ihres Systems.

Wenn Sie einen Windows-Rechner verwenden und Ihren Editor ändern möchten, müssen Sie den vollständigen Pfad zur ausführbaren Datei angeben, zum Beispiel so:

```sh
git config --global core.editor "'C:/Program Files/Notepad++/notepad++.exe' -multiInst -notabbar -nosession -noPlugin"
```

Um Ihre Liste der Konfigurationen zu sehen, können Sie den folgenden Befehl ausführen:

```sh
git config --list
```

Jetzt, da Sie Git installiert und eingerichtet haben, sind Sie bereit, Versionskontrolle zu Ihren Projekten hinzuzufügen.
