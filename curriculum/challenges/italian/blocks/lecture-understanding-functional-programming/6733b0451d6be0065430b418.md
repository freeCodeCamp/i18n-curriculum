# --interactive--

La programmazione funzionale riguarda la scrittura di codice pulito, prevedibile e più facile da testare e eseguire il debug. Due concetti chiave che esploreremo oggi sono le funzioni pure e l’evitare gli effetti collaterali.

Iniziamo con le funzioni pure.

Una funzione pura è una funzione che, dato lo stesso input, restituisce sempre lo stesso output e non modifica nulla al di fuori di sé stessa.

Ecco un esempio di funzione pura:

```js
function add(a, b) {
  return a + b;
}
```

Questa funzione è pura perché restituisce sempre lo stesso risultato per gli stessi input e non cambia nulla al di fuori di sé stessa. Non importa quante volte chiami `add(2, 3)`, restituirà sempre `5` e non influenzerà nessun’altra parte del tuo programma.

Ora, confrontiamola con una funzione impura:

```js
let total = 0;
function addToTotal(value) {
  total += value;
  return total;
}
```

Questa funzione è impura perché modifica una variabile esterna a sé stessa (`total`). Il risultato di `addToTotal(5)` sarà diverso a seconda di cosa fosse `total` prima che la funzione venisse chiamata. Questa imprevedibilità può rendere il nostro codice più difficile da capire e da eseguire il debug.

Questo ci porta al nostro prossimo concetto importante, ovvero gli effetti collaterali.

Un effetto collaterale è qualsiasi cambiamento che avviene nello stato del programma quando una funzione viene chiamata. Questo può includere modificare una variabile globale, scrivere su un file o fare una chiamata API.

Le funzioni pure non hanno effetti collaterali, mentre le funzioni impure li hanno.

Evitare gli effetti collaterali è un principio chiave della programmazione funzionale. Quando minimizziamo gli effetti collaterali, il nostro codice diventa più prevedibile e più facile da testare.

Quando scriviamo codice, il nostro obiettivo principale dovrebbe essere far sì che le funzioni facciano una cosa sola e che la facciano molto bene senza modificare altre parti del nostro programma.

Ecco un esempio di funzione con un effetto collaterale:

:::interactive_editor

```js
function greet(name) {
  console.log(`Hello, ${name}!`);
}

greet("Alice");
```

:::

Questa funzione ha un effetto collaterale perché interagisce con il mondo esterno scrivendo nella console.

Nella programmazione funzionale, potremmo riscriverla così:

:::interactive_editor

```js
function greet(name) {
  return `Hello, ${name}!`;
}
console.log(greet("Alice"));
```

:::

Ora `greet` è una funzione pura che semplicemente restituisce una stringa, e gestiamo l’effetto collaterale (scrivere nella console) separatamente.

La programmazione funzionale ci incoraggia a comporre i nostri programmi usando funzioni pure, suddividendo il codice in piccoli pezzi prevedibili. In questo modo possiamo costruire programmi più facili da capire, testare e mantenere.
