# --interactive--

## Основи об’єктів

- **Визначення**: об’єкт — це структура даних, яка складається з властивостей. Властивість містить ключ і значення. Щоб отримати доступ до даних об’єкта, можна використовувати або точкову нотацію, або дужкову нотацію.

:::interactive_editor

```js
const person = {
  name: "Alice",
  age: 30,
  city: "New York"
};

console.log(person.name);  // Alice
console.log(person["name"]); // Alice
```

:::

Щоб призначити властивість існуючого об’єкта, можна використовувати або точкову нотацію, або дужкову нотацію разом з оператором присвоєння.

:::interactive_editor

```js
const person = {
  name: "Alice",
  age: 30
};

person.job = "Engineer"
person["hobby"] = "Knitting"
console.log(person);  // {name: 'Alice', age: 30, job: 'Engineer', hobby: 'Knitting'}
```

:::

## Видалення властивостей з об’єкта

- **Оператор `delete`**: цей оператор використовується для видалення властивості з об’єкта.

:::interactive_editor

```js
const person = {
  name: "Alice",
  age: 30,
  job: "Engineer"
};

delete person.job;

console.log(person.job); // undefined
```

:::

## Перевірка, чи має об’єкт властивість

- **Метод `hasOwnProperty()`**: цей метод повертає булеве значення, яке вказує, чи має об’єкт вказану властивість як власну властивість.

:::interactive_editor

```js
const person = {
  name: "Alice",
  age: 30
};

console.log(person.hasOwnProperty("name")); // true
console.log(person.hasOwnProperty("job")); // false
```

:::

- **`Object.hasOwn()` метод**: Це сучасний, рекомендований спосіб перевірити, чи має об’єкт власну (не успадковану) властивість. Синтаксис: `Object.hasOwn(object, propertyName)`. Він повертає `true`, якщо властивість існує в об’єкті, і `false`, якщо ні, незалежно від значення властивості. Це робить його безпечнішим за `hasOwnProperty()` і надійнішим за `if (obj.prop)`, коли значення можуть бути `0`, `false`, `null` або `undefined`.

:::interactive_editor

```js
const person = {
  name: "Alice",
  age: 30
};

console.log(Object.hasOwn(person, "name")); // true
console.log(Object.hasOwn(person, "job")); // false
```

:::

Ось детальніший приклад, який показує, чому `Object.hasOwn()` надійніший за пряму перевірку значення:

:::interactive_editor

```js
const settings = {
  darkMode: false,
  fontSize: 0,
  language: null
};

// Object.hasOwn() correctly sees these properties exist
console.log(Object.hasOwn(settings, "darkMode")); // true (value is false, but exists)
console.log(Object.hasOwn(settings, "fontSize")); // true (value is 0, but exists)
console.log(Object.hasOwn(settings, "theme"));    // false (property was never added)

// Using if() directly is unsafe for falsy values!
if (settings.darkMode) {
  console.log("Dark mode on"); // Does NOT print — misleading!
}

// Object.hasOwn() is the safe way
if (Object.hasOwn(settings, "darkMode")) {
  console.log("darkMode exists, value is:", settings.darkMode); // darkMode exists, value is: false
}
```

:::

- **Оператор `in`**: цей оператор поверне `true`, якщо властивість існує в об’єкті.

:::interactive_editor

```js
const person = {
  name: "Bob",
  age: 25
};

console.log("name" in person);  // true
```

:::

## Доступ до властивостей вкладених об’єктів

- **Доступ до даних**: доступ до властивостей вкладених об’єктів здійснюється за допомогою точкової або дужкової нотації, так само як і для простих об’єктів. Проте потрібно ланцюжком викликати ці доступи, щоб заглибитися у вкладену структуру.

:::interactive_editor

```js
const person = {
  name: "Alice",
  age: 30,
  contact: {
    email: "alice@example.com",
    phone: {
      home: "123-456-7890",
      work: "098-765-4321"
    }
  }
};

console.log(person.contact.phone.work); // "098-765-4321"
```

:::

## Примітивні та непримітивні типи даних

- **Примітивні типи даних**: ці типи даних включають числа, рядки, булеві значення, `null`, `undefined` та символи. Ці типи називають «примітивними», тому що вони представляють окремі значення і не є об’єктами. Примітивні значення є незмінними, тобто після створення їхнє значення не можна змінити.  
- **Непримітивні типи даних**: у JavaScript це об’єкти, які включають звичайні об’єкти, масиви та функції. На відміну від примітивів, непримітивні типи можуть містити кілька значень у вигляді властивостей або елементів.

## Методи об’єктів

- **Визначення**: методи об’єктів — це функції, які пов’язані з об’єктом. Вони визначаються як властивості об’єкта і можуть отримувати доступ до даних об’єкта та змінювати їх. Ключове слово `this` всередині методу посилається на сам об’єкт, що дає змогу отримувати доступ до його властивостей.

:::interactive_editor

```js
const person = {
  name: "Bob",
  age: 30,
  sayHello: function() {
    return "Hello, my name is " + this.name;
  }
};

console.log(person.sayHello()); // "Hello, my name is Bob"
```

:::

## Конструктор об’єкта

- **Визначення**: конструктор у JavaScript — це спеціальний тип функції, який використовується для створення та ініціалізації об’єктів. Його викликають за допомогою ключового слова `new` і він може ініціалізувати властивості та методи на новоствореному об’єкті. Конструктор `Object()` створює новий порожній об’єкт.

```js
new Object()
```

## Робота з оператором необов’язкового ланцюжка (`?.`)

- **Визначення**: цей оператор дозволяє безпечно отримувати доступ до властивості об’єкта або викликати методи, не турбуючись про те, чи вони існують.

:::interactive_editor

```ts
const user = {
  name: "John",
  profile: {
    email: "john@example.com",
    address: {
      street: "123 Main St",
      city: "Somewhere"
    }
  }
};

console.log(user.profile?.address?.street); // "123 Main St"
console.log(user.profile?.phone?.number);   // undefined
```

:::

## Деструктуризація об’єктів

- **Визначення**: деструктуризація об’єктів дозволяє витягувати значення з об’єктів і призначати їх до змінних більш лаконічним і зрозумілим способом.

:::interactive_editor

```js
const person = { name: "Alice", age: 30, city: "New York" };

const { name, age } = person;

console.log(name); // Alice
console.log(age);  // 30
```

:::

## Робота з JSON

- **Визначення**: JSON означає JavaScript Object Notation. Це легкий, текстовий формат даних, який часто використовується для обміну даними між сервером і вебзастосунком.

```js
{
  "name": "Alice",
  "age": 30,
  "isStudent": false,
  "list of courses": ["Mathematics", "Physics", "Computer Science"]
}
```

- **`JSON.stringify()`**: цей метод використовується для перетворення JavaScript-об’єкта в JSON-рядок. Це корисно, коли потрібно зберегти або передати дані у форматі, яким можна легко поділитись або передати між системами.

:::interactive_editor

```js
const user = {
  name: "John",
  age: 30,
  isAdmin: true
};

const jsonString = JSON.stringify(user);
console.log(jsonString); // '{"name":"John","age":30,"isAdmin":true}'
```

:::

- **`JSON.parse()`**: цей метод перетворює JSON-рядок назад у JavaScript-об’єкт. Це корисно, коли ви отримуєте JSON-дані з вебсервера або localStorage і потрібно опрацювати ці дані у своєму застосунку.

:::interactive_editor

```js
const jsonString = '{"name":"John","age":30,"isAdmin":true}';
const userObject = JSON.parse(jsonString);

// result: { name: 'John', age: 30, isAdmin: true }
console.log(userObject);
```

:::
