# --description--

在計算機科學中，搜尋一個項目列表是常見的情況。當談到搜尋時，你應該知道兩個關鍵的演算法：線性搜尋和二分搜尋法。

線性搜尋從列表的開頭開始，並迭代每個項目，直到找到它正在尋找的標的值。

如果找到標的值，則傳回它在列表中的索引。如果找不到標的值，則傳回 `-1`。我們傳回 `-1` 是因為它在大多數程式語言中不是有效的索引。

以下是線性搜尋的 `code` 範例：

```python
def linear_search(arr, target):
    for i in range(len(arr)):
        if arr[i] == target:
            return i
    return -1
```

如果我們要搜尋的列表是 `[13, 4, 7, 9, 10]`，且標的值是 `9`，則函式會傳回 `3`，因為 `9` 位於索引 `3`。

如果我們將標的值改為 `5`，函式會傳回 `-1`，因為 `5` 不在列表中。

雖然這是一個相對簡單的演算法，但它不是最高效的。如果你有一個大型的項目列表，線性搜尋可能需要很長時間才能找到標的值。

線性搜尋的時間複雜度是 `O(n)`，因為搜尋整個列表所需的時間會隨著列表大小線性增加。

線性搜尋的空間複雜度是 `O(1)`，因為它不需要任何額外空間來搜尋整個列表。

二分搜尋法是一種用於搜尋大量條款的高效演算法。條件是列表必須以遞增順序排序。

二分搜尋法的運作方式是將列表對半分割，並檢查標的值是否在列表的中間。如果標的值在列表的中間，則傳回標的值的索引。否則，演算法會檢查標的值是在列表的左半部還是右半部。

它會持續將列表中剩餘的零件分成兩半，直到找到標的值。如果列表中沒有標的值，則傳回 `-1`。

以下是 `binary search` 的程式碼範例：

```python
def binary_search(arr, target):
    low = 0
    high = len(arr) - 1

    while low <= high:
        mid = (low + high) // 2  

        if arr[mid] == target:
            return mid
        elif arr[mid] < target:
            low = mid + 1
        else:
            high = mid - 1

    return -1
```

我們從識別一個 `low` 和 `high` 索引開始。這表述了我們正在搜尋的列表範圍。

接著我們檢查 `low` 是否小於或相等於 `high` 的條件。如果 `low` 大於 `high`，表示我們已經搜尋完整個列表且未找到標的值。在這種情況下，我們停止搜尋並傳回 `-1`。

如果 `low` 索引小於或相等於 `high` 索引，我們會計算列表的中間索引 `mid`。接著我們檢查標的值是否位於中間索引。如果是，我們傳回中間索引。

否則，我們會檢查中點的值是否小於標的。如果是，我們會將低索引更新為中間索引加一。這表示我們將搜尋列表的右半部。

最後，如果其他條件都不是 `True`，我們會將 `high` 索引更新為中間索引減一。這表示我們將搜尋列表的左半部。

我們會持續重複這個處理程序，直到找到標的或確定標的不在列表中。

二分搜尋法的時間複雜度是 `O(log n)`，因為搜尋列表所需的時間會隨著列表大小以對數方式成長。

二分搜尋法的空間複雜度是 `O(1)`，因為它在搜尋列表時不需要任何額外空間。

二分搜尋法和線性搜尋可以用於你在電腦科學中會遇到的各種問題。了解這兩種演算法之間的差異以及何時使用每一種非常重要。
