# --explanation--

`Quality assurance`는 제품이 정해진 품질 기준을 충족하는지 확인하는 과정입니다. 예를 들어:

`The quality assurance team checks the software for bugs before release.` - 여기서 `quality assurance`는 제품의 기준을 유지하는 팀의 역할을 강조합니다.

`Testing`는 제품의 성능을 확인하기 위해 평가하거나 검사하는 것을 의미합니다. 예를 들어:

`Developers are testing the new features for potential issues.` - 이는 기능이 제대로 작동하는지 적극적으로 점검하고 있음을 보여줍니다.

`Make sure the subtitles are syncing with the video.` - 이는 자막이 영상 속 말하는 타이밍과 일치해야 한다는 의미입니다.
