# --interactive--

**ملاحظة**: قد تستخدم بعض الأمثلة التفاعلية CSS لم تتعلمها بعد. لا تقلق بشأن محاولة فهم كل الشفرة. الهدف من الأمثلة هو عرض معاينات لهذه المفاهيم التصميمية لتفهم كيف تعمل الأمور بشكل أفضل. لعرض المعاينات، يجب تفعيل المحرر التفاعلي.

عندما يتعلق الأمر بتصميم الويب، هناك أنواع كثيرة من وسائل التنقل التي يمكنك استخدامها. تشمل الأمثلة أشرطة التنقل العلوية، الأشرطة الجانبية، والتذييلات. لكن إذا كان موقعك أكثر تعقيدًا ويحتوي على مستويات أعمق من التنقل، فقد ترغب في استخدام مسارات التنقل.

مسارات التنقل هي وسيلة تنقل تُظهر للمستخدم موقعه في هيكل الموقع. إليك مثالًا على شكل مسارات التنقل في موقع إلكترونيات تجريبي:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<div class="breadcrumbs">
  Home / Electronics / Phones / Smartphone XYZ
</div>
```

```css
.breadcrumbs {
  font-size: 0.9rem;
  color: #555;
  margin: 1rem 0;
}
```

:::

في معظم المواقع، تُعرض مسارات التنقل في أعلى الصفحة، وتُظهر للمستخدم المسار الذي اتبعه للوصول إلى الصفحة الحالية. بدءًا من `Homepage`، تنقل المستخدم إلى فئة `Electronics`، ثم إلى فئة `Phones`، وأخيرًا إلى منتج `Smartphone XYZ`. ربما تفاعلت مع مسارات التنقل أثناء البحث عن منتج أو معلومة محددة على موقع ما.

استخدام مسارات التنقل مفيد لأنه يساعد المستخدمين على فهم موقعهم في هيكل الموقع وكيفية العودة إلى الصفحات السابقة. هذا مفيد بشكل خاص عندما يأتي المستخدم من نتيجة بحث أو رابط خارجي ويحتاج لفهم سياق الصفحة التي هو عليها.

عند تصميم مسارات التنقل، هناك بعض الأمور التي يجب مراعاتها. أولها تحديد الفاصل الذي سيُستخدم. الفاصل هو الحرف الذي يفصل بين مستويات الهيكل المختلفة. الفواصل الشائعة تشمل علامة أكبر من (`>`)، علامات الاقتباس الزاوي اليمنى (`»`)، والشرطة المائلة الأمامية (`/`).

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<div class="breadcrumbs">
  Home &gt; Electronics &gt; Phones &gt; Smartphone XYZ
</div>
```

```css
.breadcrumbs {
  font-size: 0.9rem;
  color: #555;
  margin: 1rem 0;
}
```

:::

الاعتبار الثاني هو مكان وضع مسارات التنقل. عادةً ما توضع مسارات التنقل في أعلى الصفحة، إما فوق أو تحت شريط التنقل الرئيسي. يجب ألا يواجه المستخدمون صعوبة في العثور على مسارات التنقل، فتأكد من أنها واضحة وسهلة التحديد.

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<nav class="main-nav">
  <a href="#">Home</a>
  <a href="#">Electronics</a>
  <a href="#">Phones</a>
</nav>
<div class="breadcrumbs">
  Home &gt; Electronics &gt; Phones &gt; Smartphone XYZ
</div>
```

```css
.main-nav {
  background-color: #333;
  padding: 1rem;
}

.main-nav a {
  color: white;
  margin-right: 1rem;
  text-decoration: none;
}

.breadcrumbs {
  font-size: 0.9rem;
  color: #555;
  margin: 1rem 0;
}
```

:::

اعتبار آخر هو حجم مسارات التنقل. يجب أن تكون كبيرة بما يكفي لتُقرأ بسهولة، لكن ليست كبيرة جدًا لتشغل مساحة كبيرة في الصفحة. تذكر أن مسارات التنقل ليست أداة تنقل رئيسية، بل وسيلة تنقل ثانوية. في المواقع التي تحتوي على الكثير من المعلومات في الصفحة، يمكن للمستخدمين رؤية موقعهم في الهيكل والتنقل بسهولة إلى الصفحات السابقة باستخدام مسارات التنقل.

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<div class="breadcrumbs">
  Home &gt; Electronics &gt; Phones &gt; Smartphone XYZ
</div>
<p>
  Lorem ipsum dolor sit amet, consectetur adipiscing elit. Sed do eiusmod tempor incididunt ut labore et dolore magna aliqua.
</p>
```

```css
.breadcrumbs {
  font-size: 1rem;
  color: #555;
  margin: 1rem 0;
}   

p {
  font-size: 1.2rem;
  line-height: 1.6;
}
```

:::

متى يجب استخدام مسارات التنقل في موقعك؟ إذا كان موقعك بسيطًا وله هيكل تنقل ضحل، فقد لا تكون مسارات التنقل ضرورية. الهيكل الضحل يعني أن الموقع يحتوي على عدد قليل من مستويات التنقل، لذا قد لا تضيف مسارات التنقل قيمة كبيرة. عادةً ما ترى مسارات التنقل في مواقع التجارة الإلكترونية، مواقع الأخبار، والمواقع التي تحتوي على هيكل عميق مثل مواقع التوثيق التقنية.
