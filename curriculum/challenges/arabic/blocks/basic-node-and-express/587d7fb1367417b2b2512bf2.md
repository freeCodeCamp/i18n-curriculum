# --description--

ملف `.env` هو ملف مخفي يُستخدم لتمرير متغيرات البيئة إلى تطبيقك. هذا الملف سري، ولا يمكن لأحد غيرك الوصول إليه، ويمكن استخدامه لتخزين بيانات تريد الاحتفاظ بها خاصة أو مخفية. على سبيل المثال، يمكنك تخزين مفاتيح API من خدمات خارجية أو عنوان URI لقاعدة البيانات الخاصة بك. يمكنك أيضًا استخدامه لتخزين خيارات التهيئة. من خلال تعيين خيارات التهيئة، يمكنك تغيير سلوك التطبيق دون الحاجة إلى إعادة كتابة بعض الشفرة.

يمكن الوصول إلى متغيرات البيئة من التطبيق عبر `process.env.VAR_NAME`. كائن `process.env` هو كائن عالمي في Node، وتُمرر المتغيرات كسلاسل نصية. حسب الاصطلاح، تكون أسماء المتغيرات كلها بحروف كبيرة، مع فصل الكلمات بشرطة سفلية. ملف `.env` هو ملف شل، لذا لا تحتاج إلى وضع الأسماء أو القيم بين علامات اقتباس. من المهم أيضًا ملاحظة أنه لا يمكن وجود مسافة حول علامة المساواة عند تعيين القيم للمتغيرات، مثل `VAR_NAME=value`. عادةً، تضع تعريف كل متغير في سطر منفصل.

# --instructions--

لنُضف متغير بيئة كخيار تهيئة.

أنشئ ملف `.env` في جذر دليل مشروعك، وخزن فيه المتغير `MESSAGE_STYLE=uppercase`.

ثم، في معالِج مسار GET `/json` الذي أنشأته في التحدي السابق، وصول إلى `process.env.MESSAGE_STYLE` وحوّل قيمة `message` في كائن الاستجابة إلى حروف كبيرة إذا كان المتغير يساوي `uppercase`. يجب أن يكون كائن الاستجابة إما `{"message": "Hello json"}` أو `{"message": "HELLO JSON"}`، حسب قيمة `MESSAGE_STYLE`. لاحظ أنه يجب قراءة قيمة `process.env.MESSAGE_STYLE` **داخل** معالِج المسار، وليس خارجه، بسبب طريقة تشغيل اختباراتنا.

ستحتاج إلى استخدام حزمة `dotenv`. تقوم هذه الحزمة بتحميل متغيرات البيئة من ملف `.env` إلى `process.env`. حزمة `dotenv` مثبتة بالفعل، وموجودة في ملف `package.json` الخاص بمشروعك. في أعلى ملف `myApp.js`، أضف `require('dotenv').config()` لتحميل متغيرات البيئة.

# --hints--

يجب أن تتغير استجابة نقطة النهاية `/json` حسب متغير البيئة `MESSAGE_STYLE`.

```js
  const response = await fetch(code + '/_api/use-env-vars');
  if (!response.ok) {
    throw new Error(await response.text());
  }
  const data = await response.json();
  assert.isTrue(
    data.passed,
    'The response of "/json" does not change according to MESSAGE_STYLE'
  );
```
