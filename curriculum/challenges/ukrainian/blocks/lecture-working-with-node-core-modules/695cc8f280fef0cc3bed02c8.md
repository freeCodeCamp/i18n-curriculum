# --description--

Crypto — це ще один основний модуль, вбудований у Node.js. Він містить інструменти для таких речей, як хешування, шифрування, дешифрування та створення цифрових підписів, які використовуються для захисту конфіденційної інформації та забезпечення безпеки вашого застосунку.

Проте `crypto` надає вам низькорівневі будівельні блоки, а не готову до використання безпеку. Написання власного коду для шифрування або автентифікації може бути небезпечним, якщо не бути обережним. У більшості випадків краще використовувати добре перевірені бібліотеки, такі як `bcrypt` для хешування паролів або `jsonwebtoken` (JWT) для обробки входів і токенів.

Водночас корисно розуміти, як працюють деякі методи модуля crypto.

Щоб використовувати ці методи, потрібно імпортувати модуль `crypto`:

```js
const crypto = require("crypto");
```

Деякі методи використовуються для трансформації даних, наприклад, для хешування та шифрування, а інші — для керування ключами та безпекою, наприклад, для генерації випадкових значень і створення секретів.

Спершу розглянемо методи для хешування та шифрування даних.

Метод `createHash()` створює об’єкт хешу, приймаючи алгоритми, такі як `sha256`, `sha512` або `md5`. Це одностороння операція, тому її не можна звернути назад.

`createHash()` корисний для хешування паролів і створення відбитків файлів. Щоб його використати, передайте алгоритм, застосуйте метод `update()` для подачі даних і нарешті використайте `digest()` з кодуванням, щоб отримати значення хешу:

```js
const crypto = require("crypto");

const hashedPassword = crypto
  .createHash("sha256")
  .update("myStrongPassword")
  .digest("hex");

console.log("createHash result:", hashedPassword);
// createHash result: f92c9cfa0ead1bcec05ca75888a4074ba994ad237e5e2a8c7cc6a620378c061d
```

`createHmac()` робить майже те саме, що і `createHash()`, але піднімає це на новий рівень, приймаючи секретний ключ, тому лише той, хто має цей ключ, може перевірити хеш. Це ідеально для автентифікації та перевірки цілісності даних:

```js
const crypto = require("crypto");

const hashedMessage = crypto
  .createHmac("sha256", "secretkey")
  .update("important-secret-message")
  .digest("hex");

console.log("createHmac result:", hashedMessage);
// createHmac result: da48d6f026b6036286b1fb872c63264130d5cc4271f3a213bb6ddca5a023e77e
```

Методи `createCipheriv()` і `createDecipheriv()` шифрують і дешифрують дані. Вони приймають алгоритм, ключ і `iv`, який є блоком випадкових або унікальних даних, що використовується на початку процесу шифрування:

```js
createCipheriv(algorithm, key, iv);
createDecipheriv(algorithm, key, iv);
```

Щоб дешифрувати дані, ключ має бути тим самим, інакше дешифрування не вдасться, і ви отримаєте помилку або нечитаємий набір символів замість оригінального повідомлення.

Ось як вони працюють на практиці:

```js
const crypto = require("crypto");

// A key must match the algorithm length. Here AES-256 is 32 bytes
const key = Buffer.from("12345678901234567890123456789012");

// A fixed IV, 16 bytes for AES
const iv = Buffer.from("1234567890123456");

const cipher = crypto.createCipheriv("aes-256-cbc", key, iv);

let encrypted = cipher.update("Hello campers!", "utf8", "hex");
encrypted += cipher.final("hex");

console.log("Encrypted data:", encrypted);
// Encrypted data: 4ee93aa398ab44e3540e4a67ca96bc8c

// Decrypt the "Hello campers!" message
const decipher = crypto.createDecipheriv("aes-256-cbc", key, iv);
let decrypted = decipher.update(encrypted, "hex", "utf8");
decrypted += decipher.final("utf8");

console.log("Decrypted data:", decrypted);
// Decrypted data: Hello campers!
```

Ще один метод crypto для трансформації даних — це `sign()` і `verify()`.

`sign()` створює цифровий підпис із деяких даних за допомогою приватного ключа. Цей підпис доводить, що дані походять від власника приватного ключа і не були змінені.

`verify()` перевіряє цей підпис, і він не проходить перевірку, якщо дані або підпис не співпадають.

Тепер розглянемо методи crypto для генерації випадкових значень і створення секретів.

`randomBytes()` приймає `size` і генерує криптографічно захищені токени. Це робить його корисним для генерації UUID (універсально унікальних ідентифікаторів). Крім того, це гарна заміна `Math.random()`, який не є безпечним для токенів і ключів.

```js
console.log("Random Bytes:", crypto.randomBytes(16));
// Random Bytes: <Buffer 01 88 aa 1e 2c 38 48 39 26 e1 6b a9 d8 c5 ed 49>
```

Вихідні дані за замовчуванням — це буфер. Як ви дізналися в уроці про модуль Buffer, ви можете конвертувати цей буфер у рядок за допомогою методу `toString()`:

```js
console.log("Random Bytes:", crypto.randomBytes(16).toString("hex"));
// Random Bytes: a6154ef5a296fa176ad0f332bd94d712
```

Аргумент `'hex'` у `toString('hex')` тут каже Node кодувати двійкові дані з буфера у шістнадцятковий рядок.

Метод `randomInt()` приймає значення `min` і `max` і генерує безпечне випадкове ціле число між ними. Це корисно для OTP і випадкового вибору.

```js
console.log("Random Int:", crypto.randomInt(0, 100)); // 89
```

Знову ж таки, перевага над `Math.random()` і `Math.floor()` у тому, що метод використовує криптографічно безпечну випадковість, тому зловмисники не можуть передбачити отримане випадкове число.

Ще один метод — `createSecretKey()`. Він приймає буфер і генерує сирий байт, загорнутий у `KeyObject`:

```js
const crypto = require("crypto");

const secret = crypto.createSecretKey(crypto.randomBytes(32));
console.log(secret); // SecretKeyObject [KeyObject] { [Symbol(kKeyType)]: 'secret' }
```

Потім ви можете використати метод `export`, щоб надіслати цей `KeyObject`:

```js
const secret = crypto.createSecretKey(crypto.randomBytes(32));
console.log(secret.export());
// <Buffer 53 06 a1 c7 75 69 8b 38 8b a4 b2 f7 1b bc b8
// ae e2 d1 bf 67 af 1a 6a 0a 6e a0 29 62 bb 52 52 32>
```

І нарешті, використайте `toString()` на буфері, щоб побачити його рядкове представлення:

```js
const secret = crypto.createSecretKey(crypto.randomBytes(32));
console.log(secret.export().toString('hex'));
// 32dfe5917668580160986f1623bf8152913329c71163be9c3404a110cd78efd6
```

Окрім цього, є:

- `createPublicKey()` і `createPrivateKey()`, які дозволяють працювати з ключами, згенерованими в іншому місці
- `createDiffieHellman()` для двох сторін, щоб згенерувати спільний секрет без прямої передачі секрету
- `Certificate()` для роботи з тим, що використовується в HTTPS, щоб ви могли аналізувати, експортувати та перевіряти вміст сертифікатів

## --answers--

`sign()` використовує публічний ключ для створення підпису, а `verify()` перевіряє його за допомогою приватного ключа.

---

### --feedback--

---

`sign()` створює цифровий підпис за допомогою приватного ключа, а `verify()` перевіряє його відповідним публічним ключем.

---

`verify()` може перевіряти підписи без потреби в будь-якому ключі.
