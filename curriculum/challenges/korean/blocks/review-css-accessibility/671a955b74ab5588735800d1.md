## 접근성 트리

접근성 트리는 화면 읽기 프로그램과 같은 보조 기술이 웹페이지의 콘텐츠를 해석하고 상호작용하는 데 사용하는 구조입니다.

## `max()` 함수

`max()` 함수는 쉼표로 구분된 값들 중 가장 큰 값을 반환합니다:

:::interactive_editor

```html
<head>
  <link rel="stylesheet" href="styles.css">
</head>
<body>
  <img src="https://cdn.freecodecamp.org/curriculum/css-photo-gallery/9.jpg" alt="A cat sitting on a table">
</body>
```

```css
img {
  width: max(250px, 25vw);
}
```

:::

위 예제에서 뷰포트 너비가 1000픽셀 미만이면 이미지 너비는 250px가 됩니다. 뷰포트 너비가 1000픽셀보다 크면 이미지 너비는 25vw가 됩니다. 이는 25vw가 뷰포트 너비의 25%와 같기 때문입니다.

## CSS와 접근성의 모범 사례

- **`display: none;`**: `display: none;`를 사용하면 화면 읽기 프로그램과 다른 보조 기술이 이 콘텐츠에 접근할 수 없는데, 이는 접근성 트리에 포함되지 않기 때문입니다. 따라서 이 방법은 시각적 표현과 접근성 모두에서 콘텐츠를 완전히 제거하려는 경우에만 사용하는 것이 중요합니다.
- **`visibility: hidden;`**: 이 속성과 값은 콘텐츠를 시각적으로 숨기지만 문서 흐름에는 남겨두어 페이지에서 공간을 차지합니다. 이 요소들은 접근성 트리에서 제거되어 화면 읽기 프로그램이 더 이상 읽지 않습니다.
- **`.sr-only` CSS 클래스**: 화면 읽기 프로그램에는 접근 가능하지만 시각적으로는 숨기는 데 자주 사용되는 기법입니다.

:::interactive_editor

```html
<head>
  <link rel="stylesheet" href="styles.css">
</head>
<body>
  <span>This span element can be read</span>
  <span class="sr-only">This span element can only be read by screen readers</span>
  <p style="display: none">
    This text is hidden with display: none and is NOT accessible to screen readers.
  </p>
  <p style="visibility: hidden">
    This text is hidden with visibility: hidden and is NOT read by screen readers,
    but it still takes up space.
  </p>
</body>

```

```css
.sr-only {
  position: absolute;
  width: 1px;
  height: 1px;
  padding: 0;
  margin: -1px;
  overflow: hidden;
  clip: rect(0, 0, 0, 0);
  white-space: nowrap;
  border: 0;
}

```

:::

- **`scroll-behavior: smooth;`**: 이 속성과 값은 부드러운 스크롤 동작을 활성화합니다.
- **`prefers-reduced-motion` 기능**: 사용자의 애니메이션 선호도를 감지하는 데 사용할 수 있는 미디어 기능입니다.

## HTML 속성으로 콘텐츠 숨기기

- **`aria-hidden` 속성**: 화면 읽기 프로그램과 같은 보조 기술을 사용하는 사람들에게 요소를 숨기는 데 사용합니다. 예를 들어, 의미 있는 콘텐츠를 제공하지 않는 장식용 이미지를 숨길 때 사용할 수 있습니다.
- **`hidden` 속성**: 대부분의 최신 브라우저에서 지원하며 콘텐츠를 시각적으로 그리고 접근성 트리에서도 숨깁니다. JavaScript로 쉽게 토글할 수 있습니다.

:::interactive_editor

```html
<p aria-hidden>This paragraph is visible for sighted users, but is hidden from assistive technology.</p>
<p hidden>This paragraph is hidden from both sighted users and assistive technology.</p>
<p>This is a normal paragraph</p>

```

:::

## `placeholder` 속성의 접근성 문제

자리 표시자 텍스트는 접근성에 좋지 않습니다. 사용자들이 자리 표시자 텍스트를 실제 입력 값과 혼동하는 경우가 많아 입력란에 이미 값이 있다고 생각합니다.
