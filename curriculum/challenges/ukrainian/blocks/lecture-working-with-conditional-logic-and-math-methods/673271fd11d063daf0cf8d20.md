# --interactive--

Умовні інструкції дозволяють приймати рішення у вашому JavaScript коді. Вони дають змогу вашій програмі рухатися певним потоком залежно від певних умов. Давайте розглянемо, як працюють `if`, `else if`, `else` та тернарний оператор, щоб ви могли контролювати потік свого коду.

Інструкція `if` приймає умову і виконує блок коду, якщо ця умова є правдивою. Правдиві значення — це будь-які значення, які при оцінці в булевому контексті, наприклад в операторі `true`, дають `if`. Ось приклади правдивих значень:

- непорожні рядки, наприклад `hello`

- будь-яке число, окрім `0` та `-0`, наприклад `4`, `-5` та інші

- масиви

- об’єкти

- булеве значення `true`

З іншого боку, хибні значення — це значення, які в булевому контексті оцінюються як `false`. У JavaScript є небагато хибних значень, тому їх легко запам’ятати. Ось кілька хибних значень:

- булеве `false`

- `0` (нуль)

- `""` (порожній рядок)

- `null`

- `undefined`

- `NaN` (Not a Number)

Тепер, коли ми маємо базове розуміння правдивих і хибних значень, давайте подивимось, як це працює з інструкціями `if`. У цьому першому прикладі ми використовуємо кілька операторів `if` для перевірки правдивих і хибних значень:

:::interactive_editor

```js
if (null) {
  console.log("This will not run.");
}

if ("freeCodeCamp") {
  console.log("This will run.");
}
```

:::

Оскільки `null` є хибним значенням, повідомлення всередині блоку ніколи не буде виведене в консоль. Але для другого оператора `if` рядок `freeCodeCamp` є правдивим значенням і буде вважатися `true` у цьому булевому контексті оператора `if`. В результаті повідомлення `This will run.` буде виведене в консоль.

Розглянемо ще кілька прикладів, як оператори `if` працюють з різними операторами порівняння. Ось приклад використання оператора `if` для перевірки, чи користувач має право голосувати:

:::interactive_editor

```js
const age = 22;

if (age >= 18) {
 console.log("You're eligible to vote"); // You're eligible to vote
}
```

:::

У цьому прикладі, оскільки `age` наразі є `22`, це означає, що умова оціниться як `true`, бо `22` більше або дорівнює `18`. Тож повідомлення `You're eligible to vote` буде виведене в консоль. Якщо змінити приклад так, що `age` тепер `15`, тоді умова оціниться як `false` і повідомлення не буде виведене в консоль:

:::interactive_editor

```js
const age = 15;

if (age >= 18) {
 console.log("You're eligible to vote"); // Code not running because age is less than 18
}
```

:::

Коли умова є `false`, тоді можна використати блок `else`:

:::interactive_editor

```js
const age = 15;

if (age >= 18) {
 console.log("You're eligible to vote");
} else {
 console.log("You're not eligible to vote"); // You're not eligible to vote
}
```

:::

У цьому прикладі `15` не є більшим або рівним `18`, тож умова буде `false`. Код всередині блоку `else` виконається у цьому випадку.

Якщо хочете перевірити кілька умов, можна використати блок `else if`. Це дозволяє вашій програмі обирати між більш ніж двома шляхами.

:::interactive_editor

```js
const score = 87;

if (score >= 90) {
 console.log('You got an A'); 
} else if (score >= 80) {
 console.log('You got a B'); // You got a B
} else if (score >= 70) {
 console.log('You got a C');
} else {
 console.log('You failed! You need to study more!');
}
```

:::

Оскільки `score` наразі є `87`, тоді повідомлення `You got a B` буде виведене в консоль.

Тернарний оператор — це компактний спосіб записати прості інструкції `if/else`. Він має три частини: умову, результат, якщо умова правдива, і результат, якщо вона хибна. Ось базовий синтаксис:

```js
condition ? expressionIfTrue : expressionIfFalse;
```

Ось приклад, що працює з температурою повітря в Цельсіях:

:::interactive_editor

```js
const temperature = 30;
const weather = temperature > 25 ? 'sunny' : 'cool';

console.log(`It's a ${weather} day!`);
```

:::

Якщо `temperature` більше за `25`, код вище виведе `It's a sunny day!`. Якщо `temperature` буде меншим або рівним `25`, він виведе `It's a cool day!`.

Отже, що краще використовувати — оператор if чи тернарний? Використовуйте тернарний оператор, коли маєте справу з однією умовою або одним виразом, або коли хочете компактний синтаксис для простої логіки. Використовуйте оператори `if/else`, коли маєте складні умови та кілька інструкцій, бо вкладені тернарні оператори стають важкими для читання.
