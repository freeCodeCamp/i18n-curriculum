# --description--

Mentre un server HTML serve pagine HTML, un'API serve dati. Un'API <dfn>REST</dfn> (trasferimento di stato rappresentazionale) permette lo scambio di dati in modo semplice, senza che i client debbano conoscere dettagli sul server. Il client deve solo sapere dove si trova la risorsa (l'URL) e quale azione vuole eseguire su di essa (il verbo). Il verbo GET si usa quando si vuole ottenere qualche informazione senza modificare nulla. Oggi il formato dati preferito per spostare informazioni sul web è JSON. In parole semplici, JSON è un modo comodo per rappresentare un oggetto JavaScript come stringa, così da poterlo trasmettere facilmente.

Creiamo un'API semplice creando una rotta che risponde con JSON al percorso `/json`. Puoi farlo come al solito, con il metodo `app.get()`. All'interno del gestore della rotta, usa il metodo `res.json()`, passando un oggetto come argomento. Questo metodo chiude il ciclo richiesta-risposta, restituendo i dati. Dietro le quinte, converte un oggetto JavaScript valido in una stringa, poi imposta le intestazioni appropriate per dire al browser che stai servendo JSON e invia i dati indietro. Un oggetto valido ha la struttura usuale `{key: data}`. `data` può essere un numero, una stringa, un oggetto annidato o un array. `data` può anche essere una variabile o il risultato di una chiamata a funzione, nel qual caso verrà valutato prima di essere convertito in stringa.

# --instructions--

Servi l'oggetto `{"message": "Hello json"}` come risposta, in formato JSON, alle richieste GET sulla rotta `/json`. Poi punta il browser su `your-app-url/json`, dovresti vedere il messaggio sullo schermo.

# --hints--

L'endpoint `/json` dovrebbe servire l'oggetto JSON `{"message": "Hello json"}`

```js
  const response = await fetch(code + '/json');
  if (!response.ok) {
    throw new Error(await response.text());
  }
  const data = await response.json();
  assert.equal(
    data.message,
    'Hello json',
    "The '/json' endpoint does not serve the right data"
  );
```
