# --description--

Server sind für das moderne Web unverzichtbar. Sie ermöglichen die Online-Erlebnisse, auf die wir als Nutzer täglich angewiesen sind, und sie werden die meisten der Erlebnisse antreiben, die Sie als Entwickler schaffen werden.

Lernen wir mehr über sie.

Wie Sie vielleicht wissen, ist das Internet ein globales System miteinander verbundener Computernetzwerke. Server sind die zentralen Komponenten, die diese Kommunikation ermöglichen, indem sie den Prozess des Datenaustauschs abwickeln.

Ein **Server** ist ein Computer oder ein Computerprogramm, das Anfragen von anderen Computern empfängt und Daten oder Dienste als Antwort über das Netzwerk an sie sendet.

Server ist ein doppeltes Konzept, weil es sich beziehen kann auf:

* Die tatsächliche physische, leistungsstarke Computer-**Hardware**, die dafür ausgelegt ist, effizient und kontinuierlich zu laufen, um diese Dienste bereitzustellen. Diese Maschinen sind für Zuverlässigkeit gebaut und konzipiert.
    
* Oder auf die tatsächliche Computer-**Software**, die auf dieser physischen Hardware läuft, um die Dienste bereitzustellen.
    
Die physischen Computer, die wir als „Server“ bezeichnen, werden üblicherweise in Rechenzentren aufbewahrt, das sind große und sichere Einrichtungen mit kontrollierten Umgebungen.

Server sind ein wesentlicher Bestandteil des **Client-Server-Modells**.

Das Client-Server-Modell ist eine Architektur verteilter Systeme, bei der Komponenten entweder Clients oder Server sind.

* Der **Client** ist der Prozess oder die Anwendung, die Anfragen an Server für Daten oder Dienste sendet. Zum Beispiel ein Webbrowser.
    
* Der **Server** ist das physische Gerät oder Computerprogramm, das diese Anfragen bearbeitet und eine Antwort zurückgibt.

Im Wesentlichen sind Client und Server Teil eines Anfrage-Antwort-Zyklus, bei dem der Client eine Anfrage an den Server sendet und der Server eine Antwort an den Client zurückgibt.

Zum Beispiel, wenn Sie Ihren Browser (den Client) verwenden, um sich mit Ihrem Benutzernamen und Passwort bei einer Anwendung zu authentifizieren, werden Ihre Informationen sicher an einen Server gesendet. Der Server prüft dann, ob der eingegebene Benutzername und das Passwort mit den Informationen übereinstimmen, die die Plattform aktuell in ihrer Datenbank hat. Wenn eine Übereinstimmung vorliegt, gibt der Server eine erfolgreiche Antwort an den Client (Ihren Browser) zurück, und Sie sind erfolgreich auf der Seite authentifiziert.

Wenn keine Übereinstimmung gefunden wird, gibt der Server eine Fehlermeldung an den Client zurück, und Sie sehen typischerweise eine Fehlermeldung oder eine Art Rückmeldung, die erklärt, warum Ihr Anmeldeversuch nicht erfolgreich war.

Die gesamte Authentifizierungslogik läuft auf dem Server. Der Server ist verantwortlich für die Validierung der Zugangsdaten, die Interaktion mit der Datenbank und die Entscheidung, wie auf jede Anfrage reagiert wird.

Jede Serverantwort enthält einen HTTP-Statuscode, der dem Client mitteilt, was passiert ist. Diese Statuscodes informieren den Client darüber, ob eine Anfrage erfolgreich war oder fehlgeschlagen ist und warum.

Zum Beispiel:

* 200 bedeutet, dass alles in Ordnung war.
    
* 404 bedeutet, dass die Ressource auf dem Server nicht gefunden wurde.
    
* 500 bedeutet, dass ein interner Serverfehler aufgetreten ist.
    
* 503 bedeutet, dass der Server vorübergehend nicht verfügbar ist.
    
* Und so weiter…
    

Jeder Statuscode hat seine eigene eindeutige Bedeutung.

Beispiele für reale Anwendungen, die Server verwenden, sind E-Commerce, Suchmaschinen, Gaming, Video-Streaming, Cloud-Speicher- und Kollaborationsplattformen, soziale Medien, Finanzanwendungen und mehr.

Wenn die Daten dynamisch aktualisiert werden oder die Anwendung eine Art Kommunikation mit einer Datenbank benötigt, sind Server sehr wahrscheinlich Schlüsselkomponenten des Kommunikationsprozesses.

Dies sind einige der Haupttypen von Servern:

* **Webserver** liefern Webinhalte wie HTML-Seiten, Bilder und CSS-Dateien an Browser.
    
* **Anwendungsserver** bearbeiten Geschäftslogik, verarbeiten Benutzereingaben und koordinieren Antworten.
    
* **Datenbankserver** betreiben Datenbankmanagementsysteme und speichern, rufen ab und verwalten Daten.
    
* **Mailserver** senden, empfangen und speichern E-Mail-Nachrichten.
    
* **Dateiserver** verwalten den Zugriff auf Dateien und ermöglichen deren Speicherung und gemeinsame Nutzung über ein Netzwerk.
    
* **Proxyserver** fungieren als Vermittler zwischen Clients und anderen Servern, oft verwendet für Caching, Sicherheit oder Verkehrssteuerung.

Server sind das Rückgrat der leistungsstarken Online-Plattformen, die wir täglich nutzen. Sie ermöglichen es, komplexe Logik auszuführen, große Datenmengen zu speichern und zu verarbeiten und Millionen von Nutzern gleichzeitig zu unterstützen. Ohne Server würde die moderne digitale Welt, auf die wir angewiesen sind, nicht existieren.
