# --description--

GitHub Pages es una opción realmente sólida para desplegar sitios estáticos o aplicaciones que no requieren un servidor back-end para manejar la lógica. Es decir, las aplicaciones que se ejecutan completamente del lado del cliente, o en el navegador del usuario, pueden desplegarse por completo en esta plataforma.

Además, GitHub Pages es completamente gratuito para proyectos de código abierto. Hay muchas buenas razones para hacer que tu proyecto sea de código abierto, y Pages añade una más.

¿Pero cómo despliegas realmente en GitHub Pages? Bueno, eso depende de cómo funciona tu aplicación.

Comencemos con una página HTML básica. Aquí tienes una que puedes agregar a tu repositorio existente si quieres:

```html
<!DOCTYPE html>
<html>
  <head>
    <title>freeCodeCamp</title>
    <link rel="stylesheet" href="./style.css" />
  </head>
  <body>
    <h1>Welcome to freeCodeCamp's Lessons!</h1>
    <p>Let's have some fun learning about Git and GitHub!</p>
  </body>
</html>
```

Ahora necesitamos visitar nuestro repositorio y hacer clic en la pestaña "Settings" para ir a la configuración del repositorio. Luego necesitamos seleccionar "Pages" en la barra lateral.

Al principio hay solo unas pocas opciones, pero eso va a cambiar a medida que empecemos a configurar los ajustes. Primero, necesitamos decirle a GitHub qué desplegar desde.

Querremos dejar esto como "Deploy from a branch", porque no estamos usando un framework ni hemos configurado ninguna acción de GitHub. Aprenderás más sobre qué son en una lección próxima.

Luego, configura la rama desde la que estamos desplegando a `main` para desplegar lo que está en esa.

Luego tendrás la opción de desplegar desde la carpeta raíz de tu proyecto o desde una carpeta `docs`. Esta función es utilizada por proyectos como freeCodeCamp: solíamos empaquetar la documentación con nuestra app, para poder desplegar solo la documentación en GitHub pages a través de la carpeta `docs`. Por ahora, déjalo como la carpeta `/` (raíz) y presiona el botón "Save" junto a este menú desplegable para guardar tus cambios.

Una vez que hayas guardado, verás una nueva entrada para configurar un dominio personalizado. Si ya tienes uno y quieres usarlo, puedes seguir el enlace a la documentación para hacerlo. Sin embargo, para esta lección, vamos a dejar esto en blanco.

También puedes ver el dominio al que se ha desplegado tu nuevo sitio de GitHub Pages. Sin un dominio personalizado, esto por defecto será `<username>.github.io`, donde `<username>` es el nombre de tu cuenta (o la cuenta de la organización si estás desplegando desde un repositorio propiedad de una organización). Y el repositorio se servirá en una ruta que coincide con el nombre del repositorio, por ejemplo, `<username>.github.io/<repo-name>`. ¡Si visitas la URL, verás tu sitio!

Ahora, cuando hagas push de nuevos cambios a tu rama `main`, GitHub Pages actualizará automáticamente nuestro sitio desplegado con los cambios más recientes. Este proceso puede tardar unos minutos, dependiendo de la carga del servidor, y puede que necesites hacer un hard refresh para limpiar la caché.

Y con eso, hemos desplegado con éxito una aplicación web básica en GitHub Pages. También puedes incluir archivos CSS y JavaScript en tu despliegue de Pages, para que puedas crear sitios hermosos e interactivos. ¡Y una vez que aprendas un framework como React o Angular, incluso puedes desplegarlos en GitHub Pages también!
