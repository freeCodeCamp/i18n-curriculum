# --description--

HTTPS는 Hypertext Transfer Protocol Secure의 약자입니다. HTTP의 보안 버전입니다.

이전 강의에서 배웠듯이, HTTP는 브라우저와 웹사이트 간에 데이터를 전송합니다. 일반 HTTP는 이 데이터를 평문으로 보내기 때문에 누구나 그 평문 데이터를 가로챌 수 있습니다.

이 때문에 HTTPS는 Transport Layer Security(TLS)라는 암호화 프로토콜을 사용해 이 데이터 전송을 보호합니다. TLS는 두 지점 간의 통신 채널을 암호화합니다. 이전 시스템은 Secure Sockets Layer(SSL)를 사용했으며, 많은 웹 호스팅 플랫폼이 오늘날에도 이를 사용합니다.

암호화는 그 데이터를 읽을 수 없는 코드로 뒤섞어 사용자의 브라우저와 웹 서버만이 해독할 수 있는 키를 갖게 합니다.

예를 들어, 웹사이트에 비밀번호를 입력할 때 HTTPS는 즉시 이를 뒤섞어 신용카드 정보, 개인 주소, 로그인 자격 증명 및 기타 민감한 정보를 보호합니다. 데이터를 가로채는 해커는 무작위 문자만 보게 되어 도난당한 정보를 사용할 수 없습니다.

이 보호는 공용 Wi-Fi 네트워크에서 매우 중요합니다. 커피숍, 공항 등 공공장소의 개방형 Wi-Fi 네트워크는 기본 보안이 없어 사이버 범죄자가 이 네트워크의 암호화되지 않은 트래픽을 감시할 수 있습니다. HTTPS는 공용 네트워크를 통해 개인 터널을 만들어 근처의 악의적 행위자로부터 브라우징 기록을 안전하게 지킵니다.

HTTPS는 또한 인증을 제공하여 웹사이트가 합법적임을 증명합니다. HTTPS를 사용하려면 웹사이트 소유자가 신뢰받는 기관이 발급한 디지털 인증서를 받아야 합니다. 이 인증서는 웹사이트 소유자의 진짜 신원을 확인합니다.

웹사이트 주소가 "http"로만 시작하면, 이는 웹사이트 소유자가 인증서를 받지 않아 웹사이트가 안전하지 않다는 신호입니다. 최신 브라우저는 이런 웹사이트에 대해 경고도 제공합니다.

마지막으로, 데이터 무결성도 HTTPS의 중요한 이점입니다. 이는 전송 중에 아무도 데이터를 변경할 수 없다는 뜻입니다. 해커가 웹사이트 파일에 악성 코드를 주입할 수 없으며, 웹사이트 소유자가 보낸 정확한 데이터를 받게 됩니다.

Google 같은 검색 엔진은 HTTPS 웹사이트를 우선시하며, 안전한 웹사이트가 검색 결과에서 더 높은 순위를 차지합니다. 최신 브라우저는 지리 위치, 푸시 알림, 웹캠 같은 고급 기능에 HTTPS를 필수로 요구합니다.
