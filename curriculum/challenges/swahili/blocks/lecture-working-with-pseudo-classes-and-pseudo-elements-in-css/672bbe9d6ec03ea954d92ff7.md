# --interactive--

Muonekano na tabia za vipengele vya ingizo ni muhimu wakati wa kujenga fomu za mtandao. Fomu yenye vigezo vinavyojibu vitendo vya mtumizi husaidia sana kuboresha uzoefu wa mtumizi, na haipaswi kusababisha mkanganyiko au kuchanganyikiwa.

CSS hutoa darasa la bandia la ingizo kadhaa ambalo linaweza kufanya fomu zako kuwa za kueleweka zaidi na rafiki kwa mtumizi. Tuchunguze darasa hizi za bandia kwa undani.

Darasa la bandia la `:focus` linakuwezesha kulenga kipengele cha ingizo wakati mtumizi anachagua au kubofya, likionyesha makini kwa sehemu ya ingizo inayotumika. Hii husaidia watumizi kutambua kwa urahisi wanapokuwa wakiandika:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css" />
<form>
  <input type="text" />
</form>
```

```css
input:focus {
  border: 2px solid crimson;
  outline: none;
}
```

:::

Kama jina linavyosema, darasa la bandia la `:hover` huchochewa wakati mtumizi anaweka kidole cha kuonyesha (cursor) juu ya kipengele. Hutoa mrejesho wa kuona kabla mtumizi hajashirikiana na ingizo, kwa ufanisi kuwaonya kuwa ingizo liko tayari kwa kitendo.

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css" />
<form>
  <input type="text" />
</form>
```

```css
input:hover {
  background-color: orange;
}
```

:::

Katika mfano hapo juu, rangi ya nyuma ya kipengele cha ingizo hubadilika kuwa rangi ya chungwa wakati mtumizi anapobeba kidole cha kuonyesha juu yake, ikimjulisha kuwa sehemu hiyo iko tayari kushirikiana nayo.

Darasa la bandia la `:checked` linakuwezesha kupamba kitufe cha radio au kisanduku cha kuchagua kwa njia tofauti wakati mtumizi anachagua. Hii inamruhusu mtumizi kuona kwa urahisi chaguo alilochagua.

Hapa kuna mfano wa kisanduku cha kuchagua kukubaliana na masharti kwenye tovuti.

**KUMBUKA**: Baadhi ya CSS katika mfano huu hutumia sifa ambazo bado hazijafundishwa. Hii ni kwa ajili ya kukupa wazo la jinsi ya kuunda kisanduku cha kuchagua maalum. Utajifunza jinsi yote haya yanavyofanya kazi katika mafundisho na warsha zijazo.

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css" />
<form>
  <label>
  Agree <input class="checkbox" type="checkbox" />
  </label>
</form>
```

```css
.checkbox {
  appearance: none;
  width: 18px;
  height: 18px;
  border: 2px solid #ccc;
  border-radius: 4px;
  display: inline-block;
  position: relative;
  cursor: pointer;
  transition: all 0.25s ease;
  vertical-align: middle; 
}

.checkbox:hover {
  border-color: #888;
}

.checkbox:checked {
  background-color: #4caf50;
  border-color: #4caf50;
}

.checkbox:checked::after {
  content: "";
  position: absolute;
  left: 4px;
  top: 0px;
  width: 5px;
  height: 10px;
  border: solid white;
  border-width: 0 2px 2px 0;
  transform: rotate(45deg);
}

.checkbox:focus {
  outline: 2px solid #90caf9;
  outline-offset: 2px;
}

```

:::

`:required` inalenga vipengele vya ingizo vyenye sifa ya `required`. Hii inaashiria kwa mtumizi kuwa lazima ajaze sehemu hiyo kabla ya kuwasilisha fomu.

Hapa kuna mfano wa fomu yenye sehemu za ingizo zinazotakiwa:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css" />
<form action="#">
  <label for="name">Name:</label>
  <input type="text" id="name" name="name" required />

  <label for="email">Email:</label>
  <input type="email" id="email" name="email" required />

  <label for="phone">Phone Number:</label>
  <input type="tel" id="phone" name="phone" />

  <button type="submit">Submit</button>
</form>
```

```css
input:required {
  border: 2px solid orange;
}
```

:::

Wakati wa kuthibitisha fomu, unaweza kutumia darasa la bandia la `:valid` kupamba sehemu za ingizo zinazokidhi vigezo vya uthibitisho, na `:invalid` kupamba sehemu za ingizo ambazo hazikidhi vigezo. Kawaida, utatumia kijani kwa ingizo halali na nyekundu kwa ingizo lisilo halali.

Hapa kuna mfano wa kutumia darasa la bandia la `:valid`:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css" />
<form>
  <label for="email">Email:</label>
  <input type="email" id="email" name="email" />
</form> 
```

```css
input:valid {
  border-color: green;
}
```

:::

Hapa kuna mfano wa kutumia darasa la bandia la `:invalid`:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css" />
<form>  
  <label for="email">Email:</label>
  <input type="email" id="email" name="email" />
</form>
```

```css
input:invalid {
  border-color: crimson;
}
```

:::

Darasa la bandia la `:disabled` linakuwezesha kuchagua na kupamba vipengele vya ingizo vilivyozimwa. Vipengele hivi vina sifa ya `disabled` ambayo inazuia watumizi kushirikiana navyo. Ingizo lililozimwa haliwezi kubofyiwa, kupewa makini, au kuhaririwa.

Hapa kuna mfano wa lebo na kipengele cha ingizo kilichozimwa.

Katika CSS, tunalenga ingizo hilo lililozimwa na kumpa rangi ya nyuma ya `lightgray` na kubadilisha kidole cha kuonyesha kuwa `not-allowed`.

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css" />
<form>
  <label for="name">Name:</label>
  <input class="text-input" type="text" id="name" name="name" disabled />
</form>
```

```css
.text-input:disabled {
  background-color: lightgray;
  cursor: not-allowed;
}
```

:::

Thamani ya sifa ya CSS ya `cursor: not-allowed;` hubadilisha muonekano wa kidole cha kuonyesha kuashiria kuwa kitendo hakiruhusiwi.

Inapotumika kwa kipengele, inaonyesha kidole cha kuonyesha chenye alama ya mduara wenye mstari wa msalaba (kawaida mduara wenye mstari wa mwinuko) kuashiria kwa watumizi kuwa kipengele hakishirikiani au hakiwezi kubofyiwa au kushirikiana nacho.

Hapa kuna mifano mingine ya darasa la bandia la ingizo na kile kinachofanya:

- `:autofill`: hutumia mitindo kwa sehemu za ingizo ambazo kivinjari hujazwa kiotomatiki na data iliyohifadhiwa.
- `:optional`: hupamba vipengele vya ingizo ambavyo havihitajiki na vinaweza kuachwa tupu.
- `:in-range` na `:out-of-range`: hupamba vipengele kulingana na kama thamani zao ziko ndani au nje ya vizingiti vilivyowekwa.
