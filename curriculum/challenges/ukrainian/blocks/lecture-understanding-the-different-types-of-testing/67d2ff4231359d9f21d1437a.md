# --description--

End-to-end тестування, або скорочено E2E, перевіряє реальні сценарії з точки зору користувача. Прикладами сценаріїв E2E тестування є перевірка процесу реєстрації та входу або перевірка оформлення замовлення на сайті електронної комерції.

Щоб краще зрозуміти, як працює end-to-end тестування, давайте розглянемо реальний приклад із кодової бази freeCodeCamp. Цей приклад використовує Playwright, популярний фреймворк для end-to-end тестування, розроблений Microsoft.

У кодовій базі freeCodeCamp є директорія `e2e`, яка містить тести для пожертв, сертифікацій, екзаменів та іншого. Усередині цієї директорії є файл `donate-page-donor.spec.ts`, у якому є тести для сторінки автентифікованого донора-підтримувача.

Перший крок — перейти на сторінку пожертв як демонстраційний користувач, який уже є підтримувачем:

```js
test.beforeEach(async ({ page }) => {
  execSync("node ./tools/scripts/seed/seed-demo-user --set-true isDonating");
  await page.goto("/donate");
});
```

Оскільки у цьому файлі є кілька тестів, хук `beforeEach` буде запускатися перед кожним із них.

Коли користувач перебуває на сторінці пожертв, він побачить подячне повідомлення з пропозиціями, як ще більше підтримати freeCodeCamp. Ось кілька тестів, які перевіряють наявність деякого тексту на сторінці пожертв:

```js
test("should render the donate page correctly", async ({ page }) => {
  await expect(
    page.getByText("Thank you for your continued support")
  ).toBeVisible();

  await expect(
    page.getByText(
      "Your contributions are crucial in creating resources that empower millions of people to learn new skills and support their families."
    )
  ).toBeVisible();

  ...
});
```

Також є тести, які перевіряють, що у донора в меню є посилання підтримувача, а також спеціальний стилізований кордон навколо його аватара, що вказує на те, що він є підтримувачем:

```js
test("The menu should have a supporters link", async ({ page }) => {
  const menuButton = page.getByTestId("header-menu-button");
  const menu = page.getByTestId("header-menu");

  await expect(menuButton).toBeVisible();
  await menuButton.click();

  await expect(menu).toBeVisible();

  await expect(page.getByRole("link", { name: "Supporters" })).toBeVisible();
});

test("The Avatar should have a special border for donors", async ({ page }) => {
  const container = page.locator(".avatar-container");
  await expect(container).toHaveClass("avatar-container gold-border");
});
```

Маючи такі детальні тести, ви можете збільшити покриття тестами вашого застосунку не лише юніт-тестами. End-to-end тести також допомагають переконатися, що ваш застосунок працює правильно і передбачувано для користувачів. Інші приклади інструментів для end-to-end тестування включають Cypress, Selenium і Puppeteer.

Хоча end-to-end тестування важливе, воно є дорогим, бо потребує багато часу на налаштування, проєктування та підтримку. E2E тести чудово підходять для критичних користувацьких потоків, тоді як юніт-тести — для перевірки невеликих одиниць у застосунку.
