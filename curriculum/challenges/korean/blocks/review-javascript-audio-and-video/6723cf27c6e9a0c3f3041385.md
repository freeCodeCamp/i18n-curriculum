## `Audio` 생성자와 공통 메서드

- **정의**: `Audio` 생성자는 다른 생성자들처럼 `new` 키워드로 호출하는 특별한 함수입니다. 이 생성자는 `HTMLAudioElement`를 반환하며, 이를 사용해 사용자를 위해 오디오를 재생하거나 사용자가 직접 제어할 수 있도록 DOM에 추가할 수 있습니다. 생성자를 호출할 때 선택적으로 URL을 (유일한) 인수로 전달할 수 있습니다. 이 URL은 재생하려는 오디오 파일의 소스를 가리켜야 합니다. 또는 소스를 동적으로 변경해야 할 경우, 반환된 오디오 요소의 `src` 속성에 URL을 할당할 수 있습니다.
- **`play()` 메서드**: 이 메서드는 `audio` 또는 `video` 요소와 함께 미디어 재생을 시작하는 데 사용됩니다.

:::interactive_editor

```html
<audio id="audio" controls>
  <source src="https://www.w3schools.com/html/horse.mp3" type="audio/mpeg">
</audio>

<button id="playBtn">Play Audio</button>

<script>
  const audio = document.getElementById("audio");
  const playBtn = document.getElementById("playBtn");

  playBtn.addEventListener("click", () => {
    audio.play();
  });
</script>
```

:::

- **`pause()` 메서드**: 이 메서드는 `audio` 또는 `video` 요소와 함께 미디어 재생을 일시 중지하는 데 사용됩니다. 현재 위치를 유지하므로 `play()`이 호출되면 그 위치에서 재생이 시작됩니다.

:::interactive_editor

```html
<audio id="myAudio" controls>
  <source src="https://www.w3schools.com/html/horse.mp3" type="audio/mpeg">
</audio>

<br>

<button id="pauseBtn">Pause Audio</button>

<script>
  document.addEventListener("DOMContentLoaded", () => {
    const audio = document.getElementById("myAudio");
    const pauseBtn = document.getElementById("pauseBtn");

    pauseBtn.addEventListener("click", () => {
      audio.pause();
    });
  });
</script>
```

:::

- **`addTextTrack()` 메서드**: 이 메서드는 미디어 요소에 텍스트 트랙을 지정할 수 있게 해주며, 특히 비디오에 자막을 추가할 때 유용합니다.
- **`fastSeek()` 메서드**: 이 메서드는 재생 위치를 미디어 내 특정 시간으로 이동할 수 있게 해줍니다.

`Audio()` 생성자는 메서드뿐 아니라 속성도 가지고 있습니다. 그 속성들은 다음과 같습니다:

- `currentTime`: 오디오의 현재 재생 시간을 가져오는 속성
- `loop`: 끝에 도달하면 자동으로 다시 시작하여 오디오를 계속 재생하게 하는 속성
- `muted`: 볼륨 설정과 상관없이 미디어 요소의 오디오 출력을 음소거하는 속성

## 다양한 오디오 및 비디오 포맷

- **MIME 타입**: MIME 타입은 Multipurpose Internet Mail Extensions의 약자로, 프로그래밍적으로 파일 유형을 표준화해 나타내는 방식입니다. MIME 타입은 브라우저 같은 애플리케이션이 특정 파일을 어떻게 처리할지 알려줍니다. 오디오와 비디오의 경우 MIME 타입은 웹 페이지에 임베드할 수 있는 멀티미디어 형식임을 나타냅니다.
- **`source` 요소**: 이 요소는 파일 유형과 소스를 지정하는 데 사용되며, 여러 개의 소스 요소를 사용해 다양한 유형을 포함할 수 있습니다. 이렇게 하면 브라우저가 사용자의 현재 환경에 가장 적합한 포맷을 결정합니다.
- **MP3**: 음악, 오디오 또는 소리를 저장하는 데 사용되는 디지털 파일 형식입니다. 파일 크기를 줄여 저장과 공유를 쉽게 하는 압축된 사운드 녹음 버전입니다. MP3는 가장 넓은 브라우저 지원을 가지며 MIME 타입은 `audio/mpeg`입니다.
- **MP4**: 비디오와 오디오를 저장하는 디지털 파일 형식입니다. 비디오(이미지)와 사운드(음악 또는 음성)를 하나의 파일에 담는 컨테이너 역할을 합니다. MP4는 비디오 파일인지 오디오 전용인지에 따라 MIME 타입이 `audio/mp4` 또는 `video/mp4`일 수 있습니다.
- 그 밖의 포맷으로는 Windows Media Player 앱과 관련된 `WMV`, `OGG`, `MKV`, `AVI` 등이 있습니다.

## 코덱

- **정의**: 코덱은 "인코더/디코더"의 줄임말로, 아날로그와 디지털 형식 간에 오디오와 비디오를 변환하는 알고리즘 또는 소프트웨어입니다. 코덱은 MIME 타입의 일부로 지정할 수 있습니다. 코덱을 정의하는 기본 구문은 미디어 타입 뒤에 세미콜론을 붙이고 `codecs=`와 코덱을 추가하는 방식입니다.

## HTMLMediaElement API

- **정의**: `HTMLMediaElement` API는 페이지의 오디오와 비디오 요소 동작을 제어하는 데 사용됩니다. 기본 `HTMLElement` 인터페이스를 확장하여 기본 속성과 메서드뿐 아니라 `waiting`, `ended`, `canplay`, `canplaythrough` 같은 유용한 이벤트도 사용할 수 있습니다. 메서드 예시로는 `play()`, `fastSeek()`, `pause()`, `canPlayType()`가 있으며, 브라우저가 오디오 파일을 재생할 수 있을지 확인하는 데 사용됩니다.

## Media Capture and Streams API

- **정의**: Media Capture and Streams API, 또는 MediaStream API는 장치에서 오디오와 비디오를 캡처하는 데 사용됩니다. 이 API를 사용하려면 `MediaStream` 객체를 생성해야 합니다. 생성자를 사용할 수도 있지만, 이는 사용자의 하드웨어와 연결되지 않습니다. 대신 `mediaDevices` 내비게이터 객체의 `global` 속성에 `getUserMedia()` 메서드가 있어 이를 사용합니다.

```js
window.navigator.mediaDevices.getUserMedia({
  audio: true,
  video: {
    width: {
      min: 1280,
      ideal: 1920,
      max: 3840
    },
    height: {
      min: 720,
      ideal: 1080,
      max: 2160
    }
  }
});
```

## Screen Capture API

- **정의**: Screen Capture API는 사용자의 화면을 녹화하는 데 사용됩니다. 이 API는 `getDisplayMedia()` 객체의 `mediaDevices` 메서드를 호출해 노출되며, 반환된 미디어 스트림을 사용합니다.

## MediaStream Recording API

- **정의**: MediaStream Recording API는 MediaStreams API와 함께 작동하여 MediaStream(또는 `HTMLMediaElement` 직접)을 녹화할 수 있게 합니다.

## Media Source Extensions API

- **주제**: Media Source Extensions API는 사용자의 웹캠 피드를 `srcObject` 속성을 통해 비디오 요소에 직접 전달할 수 있게 해줍니다.

## Web Audio API

- **정의**: Web Audio API는 웹에서 들리는 모든 소리를 담당합니다. 이 API에는 오디오 데이터를 담은 버퍼를 나타내는 `AudioBuffer` 같은 중요한 객체와 오디오 처리 그래프를 나타내는 `AudioContext`가 포함되어 있습니다.
