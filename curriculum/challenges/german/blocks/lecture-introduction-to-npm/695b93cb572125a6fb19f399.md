# --description--

npm benötigt eine Möglichkeit, wichtige Details zu Ihrem Projekt zu speichern – Dinge wie welche Pakete es verwendet, die Version dieser Pakete und welche Befehle es ausführen kann. Dies geschieht mit der `package.json`-Datei.

Betrachten Sie die `package.json`-Datei als eine npm-Konfigurationsdatei für das Projekt. Es ist eine einfache JSON-Datei, die Ihr Projekt mit Schlüssel-Wert-Paaren beschreibt, um Dinge wie Folgendes zu verfolgen:

* den Namen und die Version des Projekts
    
* den Autor
    
* die Hauptdatei, die zuerst ausgeführt werden soll
    
* die Lizenz
    
* das Code-Repository, Schlüsselwörter und Beschreibungen
    
* alle Abhängigkeiten, die das Projekt benötigt
    

So könnte eine `package.json`-Datei aussehen:

```json
{
  "name": "my-app",
  "version": "1.0.0",
  "description": "A simple NPM project",
  "main": "index.js",
  "scripts": {
    "start": "node index.js",
    "test": "echo \"Running tests...\""
  },
  "repository": {
    "type": "git",
    "url": "git+https://github.com/username/project.git"
  },
  "keywords": ["fCC", "js"],
  "author": "Kolade Chris",
  "license": "ISC",
  "dependencies": {
    "react": "^19.2.1"
  },
  "devDependencies": {
    "cypress": "^15.7.1"
  }
}
```

Das `main`-Feld ist oft auf `index.js` gesetzt und teilt Node.js mit, welche Datei zuerst ausgeführt werden soll.

Das `license`-Feld wird verwendet, um die rechtlichen Bedingungen anzugeben, unter denen andere Personen den Code in Ihrem Paket verwenden, ändern und verbreiten dürfen. Es gibt viele verschiedene Lizenzen wie `MIT`, `ISC` oder `GPL`, die alle unterschiedliche Bedingungen und Berechtigungen haben. Standardmäßig ist es `ISC`, die Internet Systems Consortium License, eine sehr großzügige Open-Source-Lizenz.

Während Ihr Projekt wächst und Sie Pakete installieren, fügt npm diese unter einem von zwei Abschnitten hinzu:

* `dependencies`, das sind Pakete, die Ihre App benötigt, um in der Produktion zu laufen
    
* `devDependencies`, das sind Pakete, die Sie nur während der Entwicklung benötigen
    

Zum Beispiel, wenn Ihr Projekt React für die Benutzeroberfläche verwendet, gehört das in `dependencies`, weil es für das Funktionieren Ihres Projekts nach der Bereitstellung erforderlich ist. Ein Tool wie Cypress, das nur für End-to-End-Tests vor der Produktion verwendet wird, gehört in `devDependencies`.

Es gibt ein paar einfache Möglichkeiten, eine `package.json`-Datei zu erstellen. Die gebräuchlichste Methode ist, den folgenden Befehl auszuführen:

```bash
npm init
```

Sie sollten diesen im Stammordner Ihres Projekts ausführen. Dieser Befehl startet einen interaktiven Einrichtungsprozess, bei dem npm Ihnen eine Reihe von Fragen stellt. Ihre Antworten werden verwendet, um Felder wie den Namen des Projekts, die Version, Beschreibung, Skripte, Schlüsselwörter, Autor und Lizenz auszufüllen.

Wenn Sie die Fragen überspringen und eine `package.json`-Datei mit Standardwerten erstellen möchten, können Sie Folgendes ausführen:

```bash
npm init -y
```

oder

```bash
npm init --yes
```

Dies erstellt sofort eine `package.json`-Datei für Sie mit Standardwerten in vielen Feldern.

Es ist normalerweise am besten, `npm init` ohne die `-y`- oder `--yes`-Flagge zu verwenden, damit Sie jedes Feld durchgehen und die Informationen Ihres Projekts anpassen können. Wenn Sie eine Frage nicht beantworten, verwendet npm einfach einen Standardwert. Zum Beispiel, wenn Sie die Lizenzfrage leer lassen, setzt npm sie standardmäßig auf `ISC`.

Sie werden bemerken, dass `dependencies` und `devDependencies` noch nicht aufgeführt sind. Das liegt daran, dass diese Abschnitte erst hinzugefügt werden, nachdem Sie Ihr erstes Paket installiert haben.

Sie werden auch bemerken, dass das `repository`-Feld fehlt. Obwohl npm während `npm init` danach gefragt hat, füllt es dieses Feld später nicht automatisch aus. Zum Beispiel, wenn Sie Befehle wie `git remote add origin` oder `git push` ausführen, aktualisiert npm das Repository-Feld nicht für Sie – Sie müssen es manuell hinzufügen, wenn Sie es einbeziehen möchten.
