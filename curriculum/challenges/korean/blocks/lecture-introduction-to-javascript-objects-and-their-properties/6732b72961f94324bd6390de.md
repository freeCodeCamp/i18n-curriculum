# --interactive--

JavaScript에서는 객체에 특정 속성이 있는지 확인하는 여러 가지 방법이 있습니다. 이러한 메서드를 이해하는 것은 외부 소스에서 데이터를 다루거나 특정 속성이 존재하는지 확인한 후 사용해야 할 때 객체를 효과적으로 다루는 데 중요합니다.

일반적인 접근법으로는 `hasOwnProperty()` 메서드, `Object.hasOwn()` 메서드, `in` 연산자, 그리고 `undefined`와 비교하는 방법이 있습니다.

먼저 `hasOwnProperty()` 메서드부터 살펴보겠습니다. 이 메서드는 객체가 지정한 속성을 자신의 속성으로 가지고 있는지 여부를 불리언으로 반환합니다. 예시는 다음과 같습니다:

:::interactive_editor

```js
const person = {
  name: "Alice",
  age: 30
};

console.log(person.hasOwnProperty("name")); // true
console.log(person.hasOwnProperty("job")); // false
```

:::

이 예제에서는 `person`라는 객체가 `name`와 `age` 두 개의 속성을 가지고 있습니다. `name`가 `person` 객체의 속성인지 확인하려면 `hasOwnProperty()` 메서드를 사용합니다. `name`가 속성이므로 `true`를 반환합니다. 하지만 `hasOwnProperty()`를 사용해 `job`가 속성인지 확인하면 객체에 존재하지 않기 때문에 `false`를 반환합니다.

`Object.hasOwn()`는 객체가 자신의 속성을 가지고 있는지 확인하는 현대적이고 권장되는 방법입니다(상속된 속성은 제외). `hasOwnProperty()`의 업그레이드된, 더 안전한 버전이라고 생각하세요. 구문은 `Object.hasOwn(object, propertyName)`이며, 첫 번째 인수로 객체를, 두 번째 인수로 속성 이름을 전달합니다.

기본 예시는 다음과 같습니다:

:::interactive_editor

```js
const person = {
  name: "Alice",
  age: 30
};

console.log(Object.hasOwn(person, "name")); // true
console.log(Object.hasOwn(person, "job")); // false
```

:::

이 예제에서 `Object.hasOwn(person, "name")`는 `true`가 `name` 객체에 직접 존재하므로 `person`를 반환합니다. `Object.hasOwn(person, "job")`는 `false`가 객체에 추가되지 않았기 때문에 `job`를 반환합니다.

매우 중요한 점은 `Object.hasOwn()`가 속성이 **존재하는지**만 확인하며, 속성의 값은 신경 쓰지 않는다는 것입니다. 즉, 값이 `true`, `0`, `false`, 또는 `null`일 때도 여전히 `undefined`를 반환합니다:

:::interactive_editor

```js
const user = {
  username: "coder123",
  score: 0,
  isActive: false,
  nickname: null
};

// Object.hasOwn() correctly reports these all exist
console.log(Object.hasOwn(user, "score"));    // true  (value is 0, but property exists)
console.log(Object.hasOwn(user, "isActive")); // true  (value is false, but property exists)
console.log(Object.hasOwn(user, "nickname")); // true  (value is null, but property exists)
console.log(Object.hasOwn(user, "email"));   // false (property was never added)

// Danger! Using if() directly gives wrong results for falsy values
if (user.score) {
  console.log("Has score"); // This will NOT print even though score exists!
}

// Safe! Object.hasOwn() gives correct result
if (Object.hasOwn(user, "score")) {
  console.log("Has score:", user.score); // Has score: 0
}
```

:::

객체에 속성이 존재하는지 확인하는 또 다른 방법은 `in` 연산자를 사용하는 것입니다. `hasOwnProperty()`처럼 `in` 연산자는 속성이 객체에 존재하면 `true`를 반환합니다. 사용법은 다음과 같습니다:

:::interactive_editor

```js
const person = {
  name: "Bob",
  age: 25
};
console.log("name" in person);  // true
```

:::

이 예제에서 `"name" in person`는 `true`가 `name`의 속성이므로 `person`를 반환합니다.

세 번째 방법은 속성이 `undefined`인지 확인하는 것입니다. 이 방법은 유용할 수 있지만 몇 가지 제한점이 있습니다. 예시는 다음과 같습니다:

:::interactive_editor

```js
const car = {
  brand: "Toyota",
  model: "Corolla",
  year: 2020
};

console.log(car.brand !== undefined); // true
console.log(car.color !== undefined); // false
```

:::

이 코드에서는 `car.brand`와 `car.color`가 `undefined`가 아닌지 확인합니다. 객체에서 존재하지 않는 속성에 접근하면 `undefined`를 반환하기 때문에 이 방법이 작동합니다. 하지만 속성이 명시적으로 `undefined` 값을 가진 경우에는 잘못된 부정 결과를 줄 수 있습니다.

실제로 이 방법들 중 어떤 것을 선택할지는 코드의 구체적인 요구 사항에 따라 달라집니다. 각 방법의 차이를 이해하면 다양한 상황에서 올바른 선택을 할 수 있습니다.
