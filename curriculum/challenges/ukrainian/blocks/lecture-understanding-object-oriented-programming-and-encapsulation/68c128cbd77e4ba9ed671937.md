# --description--

Геттери та сеттери — це методи, які дозволяють контролювати, як отримують доступ до властивостей класу та змінюють їх. За допомогою геттерів ви отримуєте значення, а за допомогою сеттерів — призначаєте значення.

Ці дії виконуються через те, що називають властивостями. Вони з’єднують геттери та сеттери і дозволяють отримувати доступ до даних.

Властивості поводяться як атрибути, але під капотом працюють як методи. Уявіть їх як дані, які ви визначаєте як методи, але вони працюють як атрибути. Це означає, що ви можете отримувати доступ до властивостей за допомогою точкової нотації замість круглих дужок.

Головне, що роблять властивості, — це виконують додаткову логіку поза увагою, коли ви отримуєте, призначаєте або видаляєте значення через них. Це робить їх ідеальним вибором, коли ви хочете отримати доступ або маніпулювати даними всередині об’єктів.

Чому ж використовувати властивості замість методів? Це здебільшого питання читабельності та конвенції. Вони роблять ваш код чистішим і легшим для розуміння.

Коли ви використовуєте метод, ви завжди маєте викликати його з круглими дужками. Але з властивістю ви можете отримати доступ до неї так само, як до звичайного атрибута, використовуючи точкову нотацію. Це робить ваш код простим, навіть якщо він виконує додаткову роботу поза увагою.

Наприклад, ви можете захотіти обчислити значення або перевірити, чи є нове значення дійсним перед збереженням. Замість виклику методу для цього ви можете використати спосіб, схожий на атрибут.

У Python декоратор — це функція, яка змінює функціональність інших функцій або класів без зміни їхнього початкового коду.

Хоча можна створювати кастомні декоратори, це виходить за межі цього уроку. Просто знайте, що щоб створити властивість, ви визначаєте метод і розміщуєте над ним декоратор `@property`. Це перетворює метод на властивість, тож до нього можна отримати доступ як до атрибута, при цьому внутрішньо викликаючи декорований метод.

Тепер перейдемо до геттерів. Ось як створити геттер за допомогою декоратора `@property`:

```py
class Circle:
    def __init__(self, radius):
        self._radius = radius

    @property
    def radius(self): # A getter to get the radius
        return self._radius
  
    @property
    def area(self):  # A getter to calculate area
        return 3.14 * (self._radius ** 2)

my_circle = Circle(3)

print(my_circle.radius) # 3
print(my_circle.area) # 28.26
```

У цьому прикладі отримують радіус і площу кола.

Зверніть увагу, що всередині класу ми використали `_radius` замість radius. Підкреслення — це загальна конвенція Python, яка показує, що атрибут призначений для приватного використання. Іншими словами, це означає, що він для внутрішнього використання і не повинен отримуватися безпосередньо ззовні класу.

Щоб створити сеттер для радіуса, наприклад, потрібно визначити інший метод з таким самим ім’ям і використати над ним `@<property_name>.setter`:

Використання `self.radius` всередині `__init__` гарантує, що сеттер виконається під час створення об’єкта, тому недійсні значення радіуса будуть одразу виявлені.

```py
class Circle:
    def __init__(self, radius):
        self.radius = radius # Calling the setter

    @property
    def radius(self):  # A getter to get the radius
        return self._radius

    @radius.setter
    def radius(self, value):  # A setter to set the radius
        if value <= 0:
            raise ValueError('Radius must be positive')
        self._radius = value

my_circle = Circle(3)
print('Initial radius:', my_circle.radius) # Initial radius: 3

my_circle.radius = 8
print('After modifying the radius:', my_circle.radius) # After modifying the radius: 8
```

У цьому прикладі сеттер радіуса не просто призначає радіус колу, а й виконує валідацію, яка перевіряє, що радіус не є від’ємним числом.

Після визначення геттерів і сеттерів Python автоматично викликає їх поза увагою щоразу, коли ви використовуєте звичайний синтаксис атрибута:

```py
my_circle.radius # This will call the getter
my_circle.radius = 4 # This will call the setter
```

Зверніть увагу, що всередині сеттера ви не можете використовувати те саме ім’я властивості при призначенні нового значення. Це тому, що `self.radius = value` викличе сеттер всередині самого методу сеттера, що призведе до нескінченної рекурсії і `RecursionError`. Тому ви завжди маєте використовувати форму з префіксом підкреслення `self._radius = value`.

Так само, як ви можете контролювати, як атрибут отримується через геттер і як він змінюється через сеттер, ви також можете контролювати, як він видаляється за допомогою делетера.

Делетер виконує кастомну логіку, коли ви використовуєте інструкцію del для властивості. Щоб створити делетер, використовуйте декоратор `@<property_name>.deleter`:

```py
class Circle:
    def __init__(self, radius):
        self.radius = radius

    # Getter
    @property
    def radius(self):
        return self._radius

    # Setter
    @radius.setter
    def radius(self, value):
        if value <= 0:
            raise ValueError("Radius must be positive")
        self._radius = value

    # Deleter
    @radius.deleter
    def radius(self):
        print("Deleting radius...")
        del self._radius
```

Ось як можна використати делетер:

```py
# Create circle object with a radius
my_circle = Circle(33)
print("Initial radius:", my_circle.radius)  # 33

# Delete the radius
# This calls the deleter
del my_circle.radius # Deleting radius...
print("Radius deleted!") # Radius deleted!

# Try to access radius after deletion
try:
    print(my_circle.radius)
except AttributeError as e:
    print("Error:", e) # Error: 'Circle' object has no attribute '_radius'
```

Основні висновки такі:

- Геттери дозволяють отримувати значення або навіть обчислювати його на льоту.
- Сеттеры дозволяють безпечно змінювати значення, виконуючи перевірки перед присвоєнням.
- Властивості пов’язують геттери і сеттери, щоб ви могли писати логіку, використовуючи точкову нотацію.
- Делетери дозволяють визначити, що відбувається, коли атрибут видаляють.
