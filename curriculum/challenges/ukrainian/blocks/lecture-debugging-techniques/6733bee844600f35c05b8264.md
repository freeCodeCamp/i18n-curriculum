# --interactive--

У попередньому уроці ви дізналися, як кидати виключення у своїх програмах. У цьому уроці ми розглянемо, як коректно обробляти ці помилки у блоці `try…catch…finally`.

Блок `try` використовується для обгортання коду, який може викликати помилку. Це безпечний простір, щоб спробувати щось, що може не вийти.

Блок `catch` ловить і обробляє помилки, які виникають у блоці `try`. Ви можете використовувати об’єкт `Error` всередині `catch`, щоб перевірити, що пішло не так.

Блок `finally` виконується після блоків `try` і `catch`, незалежно від того, чи сталася помилка. Його часто використовують для прибирання, наприклад, закриття файлів або звільнення ресурсів.

Ось приклад використання блоку `try…catch`:

:::interactive_editor

```js
function processInput(input) {
  if (typeof input !== "string") {
    throw new TypeError("Input must be a string.");
  }

  return input.toUpperCase();
}

try {
  console.log("Starting to process input...");
  const result = processInput(9);
  console.log("Processed result:", result);
} catch (error) {
  console.error("Error occurred:", error.message);
} 
```

:::

У цьому прикладі у нас є функція `processInput`, яка спочатку перевіряє, чи `input` не є типом `string`. Якщо це так, ми кидаємо помилку. Інакше повертаємо результат використання методу `toUpperCase` на `input`.

Ми викликаємо функцію всередині блоку `try`. Оскільки виклик функції кидає помилку, вона буде спіймана всередині блоку `catch`, і повідомлення про помилку відобразиться в консолі.

Помилка, передана в блок `catch`, є об’єктом `Error`, який містить інформацію про цю помилку. У цьому випадку ми використовуємо властивість `message`, яка показує інформацію у зручному для користувача вигляді.

Ми використовуємо `console.error`, бо він спеціально призначений для логування помилок. У багатьох сучасних браузерах вихідні дані `console.error()` відображаються червоним у консолі.

Інструкція `finally` виконується незалежно від того, чи було кинуто виключення.

```js
try {
  // Code that might throw an error
} catch (error) {
  // Code to handle the error
} finally {
  // Code that runs regardless of whether an error occurred or not
}
```

Добрим випадком для інструкції `finally` є робота з файлами. У JavaScript ви можете відкрити файл, використати блок `try` для запису даних у файл. Якщо виникнуть помилки, ви можете використати `catch` для їх ловлення. Потім використайте інструкцію `finally` для закриття файлу.
