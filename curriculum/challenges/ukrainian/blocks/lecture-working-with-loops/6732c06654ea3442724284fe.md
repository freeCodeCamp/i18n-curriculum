# --interactive--

У попередніх уроках ви навчились працювати з циклами `for`, циклами `for...in` та циклами `for...of`. У цьому уроці ви дізнаєтеся про цикл `while` та цикл `do...while`.

Цикл `while` виконуватиме блок коду доти, доки умова є істинною. Ось базовий синтаксис циклу `while`:

```js
while (condition) {
  // code block to be executed
}
```

Умова перевіряється перед виконанням блоку коду. Якщо умова хибна, блок коду не буде виконано.

Цикли `while` корисні, коли ви не знаєте, скільки разів потрібно виконати блок коду. Ось приклад використання циклу `while`:

:::interactive_editor

```js
let counter = 0;
while(counter < 5) {
  console.log(counter);
  counter++;
}
```

:::

У цьому прикладі у нас є змінна з назвою `counter`, яка ініціалізована значенням `0`. Цикл `while` продовжуватиме виконуватися доти, доки значення `counter` менше за `5`. Усередині циклу ми виводимо значення `counter` у консоль, а потім збільшуємо `counter` на `1`.

Іншим циклом, схожим на цикл `while`, є цикл `do...while`. Ось базовий синтаксис:

```js
do {
  // code block to be executed
} while (condition);
```

Головна відмінність між циклом `do...while` і циклом `while` полягає в тому, що цикл `do...while` виконає блок коду принаймні один раз перед перевіркою умови.

Якщо умова істинна, блок коду продовжить виконуватися. Якщо умова хибна, виконання блоку коду припиниться.

Ось приклад використання циклу `do...while`:

:::interactive_editor

```js
let counter = 0;
do {
  console.log(counter);
  counter++;
} while (counter < 5);  
```

:::

У цьому прикладі у нас є змінна з назвою `counter`, яка ініціалізована значенням `0`. Цикл `do...while` виведе значення `counter` у консоль, а потім збільшить `counter` на `1`. Після виконання блоку коду він перевіряє, чи значення `counter` менше за `5`. Якщо так, цикл продовжить виконання. Якщо ні, цикл зупиниться.

У більшості випадків ви, ймовірно, частіше використовуватимете цикл `while`, ніж цикл `do...while`. Проте добре знати обидва типи циклів і коли їх застосовувати.

## --answers--
