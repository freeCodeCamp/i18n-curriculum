# --description--

애플리케이션이 커질수록 컴포넌트 간 데이터 흐름을 관리하는 일이 복잡해질 수 있습니다.

처음 시작할 때는 React의 `useState` 훅만으로도 충분할 수 있지만, 기능을 추가하면서 다음과 같은 문제를 겪을 수 있습니다:

- 필요하지 않은 컴포넌트에 프로프스를 전달하는, 즉 프로프 드릴링
- 애플리케이션 여러 부분에서 데이터 동기화 유지
- 여러 컴포넌트에 동시에 영향을 주는 복잡한 업데이트 처리

이런 문제들은 코드베이스를 유지, 디버그, 테스트하기 어렵게 만들 수 있습니다. 이럴 때 상태 관리 라이브러리가 필요합니다 — 컴포넌트가 필요한 데이터를 얻거나 업데이트할 수 있는 중앙 집중식 공간을 제공합니다.

여러 상태 관리 옵션과 언제 사용해야 하는지 살펴보겠습니다.

Context API는 React에 내장된 상태 관리자로, 서드파티 라이브러리 없이 컴포넌트 간 상태를 공유할 수 있게 해줍니다. `useState` 훅보다 잘 확립된 업그레이드로, 테마 토글이나 사용자 인증 상태 같은 경우에 적합합니다.

하지만 Context API는 빈번한 업데이트를 잘 처리하지 못하고 불필요한 재렌더링을 유발할 수 있어, 전자상거래나 소셜 미디어 플랫폼 같은 복잡한 상태가 필요한 애플리케이션에는 덜 적합합니다.

다음은 Context API의 기본 사용법을 보여주는 카운터 컴포넌트입니다:

```jsx
import { useState, createContext } from 'react';

const CounterContext = createContext();

const CounterProvider = ({ children }) => {
  const [count, setCount] = useState(0);

  return (
    <CounterContext.Provider value={{ count, setCount }}>
      {children}
    </CounterContext.Provider>
  );
};

export { CounterContext, CounterProvider };
```

이 코드는 컨텍스트와 프로바이더를 생성해 애플리케이션 전반에 `count` 상태를 공유합니다.

`CounterProvider`는 `useState` 훅을 사용해 `count` 상태와 그 설정자를 초기화하고 관리합니다. 둘 다 `Provider`를 통해 자식 컴포넌트에 전달됩니다.

그래서 애플리케이션 전체를 `CounterProvider`로 감싸면 `count` 상태가 애플리케이션 어디서든 사용 가능합니다.

애플리케이션을 `CounterProvider`로 감싸는 방법은 다음과 같습니다:

```jsx
import { CounterProvider } from './context/CounterContext';

function App() {
  return (
    <CounterProvider>
        {/* App components */}
    </CounterProvider>
  );
}

export default App;
```

그리고 `count` 상태를 사용하는 방법은 다음과 같습니다:

```jsx
import React, { useContext } from 'react';
import { CounterContext } from '../context/CounterContext';

const Counter = () => {
  const { count, setCount } = useContext(CounterContext);

  return (
    <>
      <div style={{ textAlign: 'center' }}>
        <h1>Context API Counter</h1>
        <button style={{ marginRight: '5px' }} onClick={() => setCount(count - 1)}>
          Decrease
        </button>
        <span>{count}</span>
        <button style={{ marginLeft: '5px' }} onClick={() => setCount(count + 1)}>
          Increase
        </button>
      </div>
    </>
  );
};

export default Counter;
```

보시다시피 `count`과 그 설정 함수 `setCount`는 `useContext` 함수로 초기화됩니다.

현재 `count` 상태가 표시되고, `setCount`가 각각 감소 및 증가 버튼을 클릭할 때 `count` 상태를 증가 및 감소시키는 데 사용됩니다.

또 다른 인기 있는 상태 관리 라이브러리는 Redux로, React와 함께 사용하기 가장 인기 있는 라이브러리 중 하나입니다. 오랫동안 사용되어 왔으며, 전자상거래, 소셜 미디어 플랫폼, 포럼 등 대규모 애플리케이션에 이상적입니다.

Redux는 중앙 저장소와 상태 업데이트에 대한 엄격한 제어를 제공해 상태 관리를 처리합니다. 액션, 리듀서, 미들웨어라는 예측 가능한 패턴을 사용합니다.

액션은 애플리케이션에서 Redux 저장소로 데이터를 보내는 페이로드로, 주로 사용자 상호작용에 의해 트리거됩니다.

리듀서는 이러한 액션에 따라 상태가 어떻게 변경되어야 하는지 지정하는 함수로, 상태를 불변 방식으로 업데이트하도록 보장합니다.

미들웨어는 액션 디스패치와 리듀서 사이의 다리 역할을 하며, Redux의 핵심 흐름을 수정하지 않고 기능(예: 로깅, 비동기 작업 처리)을 확장할 수 있게 합니다.

Redux에 대한 가장 흔한 불만은 시작할 때 필요한 보일러플레이트 코드가 많다는 점입니다. 이에 대응해 Redux 팀은 설정 과정을 크게 단순화한 "Redux Toolkit"과 "RTK Query"를 도입했습니다.

보통 `createSlice()` 함수를 사용해 액션과 리듀서를 하나의 파일에 정의합니다. 파일 이름은 `Slice`로 끝나도록 하는 경우가 많으며, 예를 들어 `productSlice`, `userSlice`, `counterSlice` 등이 있습니다.

기본을 보여주는 `counterSlice` 파일은 다음과 같습니다:

```jsx
import { createSlice } from '@reduxjs/toolkit';

const counterSlice = createSlice({
  name: 'counter',

  initialState: { count: 0 },

  reducers: {
    increment: (state) => {
      state.count += 1;
    },
    decrement: (state) => {
      state.count -= 1;
    },
  },
});

export const { increment, decrement } = counterSlice.actions;

export default counterSlice.reducer;
```

여기서 애플리케이션 전체를 `Provider`로 감싸고, 슬라이스에서 `useSelector()`로 상태 일부를 선택한 다음 `useDispatch()`를 사용해 상태를 활성화해야 합니다.

또 다른 고려할 만한 옵션은 Zustand입니다.

Zustand는 간단한 API를 가진 경량 상태 관리 라이브러리입니다. 훅 기반이라 Redux보다 보일러플레이트가 적어 설정이 더 쉽고 빠릅니다.

Zustand는 소규모에서 중간 규모 애플리케이션에 적합합니다. `useStore` 훅을 사용해 컴포넌트와 페이지에서 상태에 직접 접근할 수 있습니다. 덕분에 액션, 리듀서, 프로바이더 없이도 데이터를 수정하고 접근할 수 있습니다.

다음은 또 다른 카운터 기능을 구현한 `useCounterStore`입니다:

```jsx
import { create } from 'zustand';

const useCounterStore = create((set) => ({
  count: 0,
  increment: () => set((state) => ({ count: state.count + 1 })),
  decrement: () => set((state) => ({ count: state.count - 1 })),
}));

export default useCounterStore;
```

애플리케이션에서 상태를 초기화하고 사용하는 방법은 다음과 같습니다:

```jsx
// Import the useCounterStore (it's just a hook)
import useCounterStore from '../useCounterStore';

const Counter = () => {
  // Initialize the states with the useCounterStore hook
  const { count, increment, decrement } = useCounterStore();

  return (
    <>
      <div style={{ textAlign: 'center' }}>
        <h1>Zustand Counter</h1>
        <button style={{ marginRight: '5px' }} onClick={() => decrement()}>
          Decrease
        </button>
        <span>{count}</span>
        <button style={{ marginLeft: '5px' }} onClick={() => increment()}>
          Increase
        </button>
      </div>
    </>
  );
};

export default Counter;
```

프론트엔드 생태계는 끊임없이 진화하고 새로운 상태 관리 라이브러리가 계속 등장하지만, 지금까지 살펴본 라이브러리들은 업계에서 널리 사용되고 있습니다.
