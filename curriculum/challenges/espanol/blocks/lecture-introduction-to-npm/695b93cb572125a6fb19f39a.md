# --description--

En una lección anterior, aprendiste qué son las dependencias. Aquí, aprenderás más sobre ellas y cosas a considerar al elegir un paquete externo para usar en tu proyecto.

Una dependencia es otra pieza de código que tu proyecto necesita para funcionar. Generalmente, son paquetes externos que otros desarrolladores han creado y compartido en el registro npm.

Usar dependencias te ahorra mucho tiempo. En lugar de construir todo tú mismo, puedes instalar un paquete que ya resuelve un problema común — cosas como crear un servidor web, manejar el enrutamiento, obtener datos, gestionar la autenticación y más.

En tu archivo `package.json`, todos los paquetes de los que depende tu proyecto están listados bajo la clave `dependencies`:

```json
{
  "name": "package-dependencies",
  "version": "1.0.0",
  "description": "a demo on how package dependencies work",
  "main": "index.js",
  "scripts": {
    "test": "echo \"Error: no test specified\" && exit 1"
  },
  "keywords": [
    "Node",
    "NPM"
  ],
  "author": "Kolade Chris",
  "license": "ISC",
  "dependencies": {
    "chalk": "^5.6.2",
    "lodash": "^4.17.21"
  }
}
```

En el archivo de ejemplo `package.json` arriba, `chalk` y `lodash` están listados como las dependencias del proyecto. Eso significa que el proyecto depende de estos paquetes para funcionar. Si no están instalados—y listados en el archivo `package.json`—cualquier función que los use dejará de funcionar.

También hay otra categoría de dependencias llamada `devDependencies`. Estos son paquetes que solo necesitas mientras construyes o pruebas tu proyecto, no cuando está en producción. Ejemplos comunes incluyen:

* Nodemon para reiniciar automáticamente el proyecto cuando hay actualizaciones
    
* Jest o Mocha para pruebas
    
* ESLint para el linting de código
    

Como esperas, `devDependencies` aparecen bajo la sección `devDependencies` del archivo `package.json`:

```json
{
  "name": "package-dependencies",
  "version": "1.0.0",
  "description": "a demo on how package dependencies work",
  "main": "index.js",
  "scripts": {
    "test": "echo \"Error: no test specified\" && exit 1"
  },
  "keywords": [
    "Node",
    "NPM"
  ],
  "author": "Kolade Chris",
  "license": "ISC",
  "dependencies": {
    "chalk": "^5.6.2",
    "lodash": "^4.17.21"
  },
  "devDependencies": {
    "jest": "^30.2.0",
    "nodemon": "^3.1.10"
  }
}
```

Al elegir qué paquetes usar en tu proyecto, hay algunas cosas importantes que debes tener en cuenta.

La seguridad siempre debe ser lo primero. Incluso un solo paquete inseguro puede crear problemas para todo tu proyecto. También debes considerar el rendimiento, porque un paquete grande o mal optimizado puede ralentizar tu proyecto.

También ayuda elegir paquetes que sean populares y bien soportados. Los paquetes populares usualmente tienen comunidades activas, buena documentación y actualizaciones frecuentes. Una forma fácil de verificar la popularidad es mirar el número de descargas semanales en npm. Por ejemplo, el paquete `chalk` recibe más de 300,000 descargas cada semana:

<img src="https://cdn.freecodecamp.org/curriculum/lecture-transcripts/what-are-package-dependencies-and-what-are-some-considerations-for-choosing-an-external-package-1.webp" alt="screenshot from npm of chalk package statistics">

El mantenimiento también es importante. Los paquetes bien mantenidos se actualizan regularmente y funcionan mejor con nuevas versiones de otros programas.

Finalmente, no pases por alto la documentación. Una documentación clara y completa facilita mucho entender cómo usar un paquete y te ahorra mucho tiempo al integrarlo en tu proyecto.
