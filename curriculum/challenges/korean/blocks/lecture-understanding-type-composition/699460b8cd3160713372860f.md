# --description--

때때로 TypeScript보다 타입에 대해 더 잘 아는 상황이 있을 수 있습니다. 이럴 때 타입 어서션이 유용합니다.

다음은 `querySelector()` 메서드를 사용해 `id`의 `submit` 요소에 접근하는 예입니다:

```ts
const submitBtn = document.querySelector("#submit");
```

현재 TypeScript는 이 `submitBtn`가 어떤 종류의 `Element`인지, 아니면 아마도 `null`인지밖에 알지 못합니다. TypeScript는 해당 HTML을 직접 확인하지 않습니다.

이럴 때 타입 어서션을 사용해 TypeScript에 이 요소가 어떤 타입인지 알려줄 수 있습니다. 예시는 다음과 같습니다:

```ts
const submitBtn = document.querySelector("#submit") as HTMLButtonElement;
```

이제 TypeScript는 이 타입 어서션 덕분에 이 요소를 `button` 요소로 취급합니다.

`as` 키워드를 사용하는 데는 한계가 있습니다. 예를 들어, TypeScript가 오류를 발생시키지 않고 다음과 같은 작업을 할 수는 없습니다:

```ts
// Conversion of type 'string' to type 'number' may be a mistake because neither type sufficiently overlaps with the other. If this was intentional, convert the expression to 'unknown' first.
const age = "awesome" as number;
```

이 예에서는 문자열 `"awesome"`를 숫자 타입으로 어서션하는 것이 논리적으로 맞지 않습니다. TypeScript는 호환 가능하거나 겹치는 타입 사이에서만 타입 어서션을 허용합니다.

이 오류를 없애는 한 가지 방법은 다음과 같이 이중 어서션을 사용하는 것입니다:

```ts
const age = "awesome" as unknown as number;
```

이 방법은 TypeScript에서 작동하지만 논리적으로는 권장되지 않습니다.

타입 어서션을 작성하는 또 다른 방법은 다음과 같이 꺾쇠 괄호 구문을 사용하는 것입니다:

```ts
const submitBtn = <HTMLButtonElement>document.querySelector("#submit");
```

`tsx` 파일을 사용할 때 이 구문을 조심해야 합니다. TypeScript가 이를 컴포넌트를 렌더링하려는 시도로 오해해 다음과 같은 오류를 발생시킬 수 있기 때문입니다:

```ts
// file: index.tsx
// This JSX tag requires 'React' to be in scope, but it could not be found.
// JSX element 'HTMLButtonElement' has no corresponding closing tag.
const submitBtn =  <HTMLButtonElement>document.querySelector("#submit");
```

대부분 상황에서는 두 번째 옵션보다 `as` 키워드를 사용하는 첫 번째 구문을 더 자주 보게 됩니다.
