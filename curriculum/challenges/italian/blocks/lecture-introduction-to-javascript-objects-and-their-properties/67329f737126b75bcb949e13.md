# --interactive--

In JavaScript, un oggetto è una struttura dati fondamentale che ti permette di archiviare e organizzare dati e funzionalità correlate.

Puoi pensare a un oggetto come a un contenitore che contiene vari pezzi di informazioni, proprio come un archivio contiene diverse cartelle e documenti.

Questi pezzi di informazioni si chiamano proprietà e sono composti da un nome (o chiave) e un valore.

```js
const exampleObject = {
  propertyName: value,
}
```

Gli oggetti sono incredibilmente versatili e costituiscono la spina dorsale di JavaScript. Infatti, quasi tutto in JavaScript è un oggetto o può essere trattato come tale. Questo include array, funzioni e persino tipi di dati primitivi come stringhe e numeri quando usati in certi modi.

Questa natura centrata sugli oggetti di JavaScript è uno dei motivi per cui è un linguaggio così flessibile e potente. Vediamo come puoi creare un oggetto:

```js
const person = {
  name: "Alice",
  age: 30,
  city: "New York"
};
```

In questo esempio, abbiamo creato un oggetto chiamato `person` con tre proprietà: `name`, `age` e `city`. Ogni proprietà ha un nome e un valore, separati da due punti.

Ora, esploriamo come puoi consultare queste proprietà. Ci sono due modi principali per consultare le proprietà di un oggetto in JavaScript: la notazione a punto e la notazione a parentesi.

La notazione a punto è il modo più comune e semplice per consultare le proprietà di un oggetto. Ecco la sintassi base per la notazione a punto:

```js
objectName.propertyName
```

Ecco come useresti la notazione a punto con il nostro oggetto `person`:

:::interactive_editor

```js
const person = {
  name: "Alice",
  age: 30,
  city: "New York"
};

console.log(person.name);  // Alice
console.log(person.age);   // 30
```

:::

La notazione a punto è concisa e facile da leggere, quindi è la scelta preferita quando conosci esattamente il nome della proprietà che vuoi consultare e quel nome è un identificatore JavaScript valido (cioè non inizia con un numero e non contiene caratteri speciali o spazi).

La notazione a parentesi, invece, ti permette di consultare le proprietà di un oggetto usando una stringa all’interno di parentesi quadre. Ecco come useresti la notazione a parentesi:

:::interactive_editor

```js
const person = {
  name: "Alice",
  age: 30,
  city: "New York"
};

console.log(person["name"]); // Alice
console.log(person["age"]); //  30
```

:::

La notazione a parentesi è più flessibile della notazione a punto perché ti consente di usare nomi di proprietà che non sono identificatori JavaScript validi. Per esempio, se avessi un nome di proprietà con spazi o che inizia con un numero, dovresti usare la notazione a parentesi:

:::interactive_editor

```js
const oddObject = {
  "1stProperty": "Hello",
  "property with spaces": "World"
};

console.log(oddObject["1stProperty"]);  // Hello
console.log(oddObject["property with spaces"]);  // World
```

:::

Un altro vantaggio della notazione a parentesi è che ti permette di usare variabili per consultare le proprietà in modo dinamico:

:::interactive_editor

```js
const person = {
  name: "Alice",
  age: 30,
  city: "Wonderland"
};

let propertyName = "city";
console.log(person[propertyName]); // Wonderland
```

:::

Questa flessibilità rende la notazione a parentesi particolarmente utile quando non conosci il nome esatto della proprietà al momento di scrivere il codice, o quando lavori con nomi di proprietà che provengono da input dell’utente o da qualche altra fonte dinamica.

Vale la pena notare che gli oggetti in JavaScript sono incredibilmente potenti e versatili. Possono contenere non solo valori semplici come stringhe e numeri, ma anche array o altri oggetti.

Capire gli oggetti e come lavorarci è fondamentale in JavaScript perché sono usati ampiamente in tutto il linguaggio e in molte librerie e framework JavaScript.

Man mano che continui a imparare e lavorare con JavaScript, scoprirai che padroneggiare gli oggetti apre un mondo di possibilità per creare applicazioni complesse e potenti.
