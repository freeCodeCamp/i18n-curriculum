# --description--

Умовне відтворення в React дозволяє створювати динамічні інтерфейси користувача. Його використовують, щоб показувати різний вміст залежно від певних умов або станів у вашому застосунку.

Найпоширеніші підходи до умовного відтворення включають використання `if` інструкцій, тернарного (`?:`) оператора та логічного AND (`&&`) оператора.

Найпростіша форма умовного відтворення використовує `if` інструкцію. Ось приклад:

```jsx
function Greeting({ isLoggedIn }) {
  if (isLoggedIn) {
    return <h1>Welcome back!</h1>;
  }
  return <h1>Please sign in</h1>;
}
```

У цьому прикладі компонент `Greeting` перевіряє проп `isLoggedIn`. Якщо він `true`, він повертає привітальне повідомлення, інакше пропонує користувачу увійти.

Ось приклад із використанням компонента `Greeting` всередині батьківського компонента `App`:

```jsx
function App() {
  return (
    <div className="App">
      <Greeting isLoggedIn={false} />
    </div>
  );
}
```

Для простіших умов часто безпосередньо в JSX використовують тернарний оператор (`?:`). Це дозволяє робити рядкове умовне відтворення, що робить ваш код більш лаконічним:

```jsx
function Greeting({ isLoggedIn }) {
  return <h1>{isLoggedIn ? "Welcome back!" : "Please sign in."}</h1>;
}
```

Цей код досягає того самого результату, що й попередній приклад, але в більш компактній формі. Тернарний оператор перевіряє `isLoggedIn` і відтворює відповідне повідомлення.

Ще один поширений шаблон умовного відтворення — використання логічного AND (`&&`) оператора. Це особливо корисно, коли потрібно відтворити щось або нічого залежно від умови:

```jsx
function Notification({ message }) {
  return (
    <div>
      {message && <p>{message}</p>}
    </div>
  );
}
```

У цьому прикладі абзац із повідомленням відтворюється лише якщо проп `message` є істинним. Якщо `message` є хибним — тобто порожній рядок, `null` або `undefined` — на екрані нічого не відтворюється.

Оволодівши цими техніками умовного відтворення, ви зможете створювати більш інтерактивні та зручні для користувача застосунки, які адаптуються до змінних даних і взаємодій користувача.
