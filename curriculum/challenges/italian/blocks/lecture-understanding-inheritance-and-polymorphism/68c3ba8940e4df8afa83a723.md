# --description--

L'ereditarietà è il prossimo concetto chiave della programmazione orientata agli oggetti (OOP) che affronteremo.

Esaminiamo più a fondo questo concetto e come ti permette di scrivere codice riutilizzabile.

Con l'ereditarietà, una sottoclasse (o classe figlio) può usare gli attributi e i metodi di una classe base (o classe genitore). Questo ti consente di riutilizzare il codice, creare gerarchie di classi chiare e personalizzare il comportamento senza riscrivere tutto. Puoi personalizzare estendendo i metodi esistenti o sovrascrivendoli nella classe figlio.

Ecco la sintassi base per l'ereditarietà:

```py
class Parent:
    # Parent attributes and methods

class Child(Parent):
    # Child inherits, extends, and/or overrides where necessary
```

Perché la classe `Child` erediti dalla classe `Parent`, devi passare il genitore al figlio.

Questo stile si chiama ereditarietà singola, dato che una classe figlio eredita da esattamente una classe genitore.

Ecco un esempio:

```py
class Animal:
    def __init__(self, name):
        self.name = name

    def sound(self):
        return f'{self.name} makes a sound'

class Dog(Animal):
    bark = 'woof! woof!! woof!!!'

jack = Dog('Jack')
print(jack.sound())  # Jack makes a sound
print(jack.bark)  # woof! woof!! woof!!!
```

Puoi vedere che siamo in grado di riutilizzare l'attributo `self.name` e il metodo `sound()` dalla classe genitore `Animal` nella classe figlio `Dog`.

Sovrascriviamo il metodo `sound()` dalla classe genitore `Animal` nella classe figlio `Dog` così possiamo far usare a `sound()` la variabile di classe `bark`:

```py
class Animal:
    def __init__(self, name):
        self.name = name

    def sound(self):
        return f'{self.name} makes a sound.'

class Dog(Animal):
    bark = 'woof! woof!! woof!!!'

    # Override sound() to use bark class variable
    def sound(self):
        return f'{self.name} barks {self.bark}'

jack = Dog('Jack')
print(jack.sound())  # Jack barks woof! woof!! woof!!!
```

Se vuoi mantenere il valore di ritorno di `sound()` e aggiungere la variabile di classe bark in seguito, puoi estendere `sound()` usando la funzione `super()`:

```py
class Animal:
    def __init__(self, name):
        self.name = name

    def sound(self):
        return f'{self.name} makes a sound'

class Dog(Animal):
    bark = 'woof! woof!! woof!!!'

    # Call Animal.sound(), then append bark
    def sound(self):
        base = super().sound()
        return f'{base}, then {self.name} barks {self.bark}'

jack = Dog('Jack')
print(jack.sound())  # Jack makes a sound, then Jack barks woof! woof!! woof!!!
```

In questo esempio, `base` è il risultato della chiamata al metodo `sound()` dalla classe `Animal`, e poi aggiungiamo il suono specifico della classe `Dog`. In questo modo puoi estendere la funzionalità della classe genitore `Animal` mantenendo comunque il suo comportamento originale.

Esiste anche l'ereditarietà multipla, dove una classe figlio può ereditare da più di una classe genitore.

Ecco la sintassi base dell'ereditarietà multipla:

```py
class Parent:
    # Attributes and methods for Parent

class Child:
    # Attributes and methods for Child

class GrandChild(Parent, Child):
    # GrandChild inherits from both Parent and Child
    # GrandChild can combine or override behavior from each
```

Un modo semplice per dimostrare l'ereditarietà multipla è con una rana, che può sia camminare sulla terra sia nuotare nell'acqua:

```py
class Walker:
    def walk(self):
        return 'I can walk on land'

class Swimmer:
    def swim(self):
        return 'I can swim in water'

# Amphibian inherits from both Walker and Swimmer
class Amphibian(Walker, Swimmer):
    def __init__(self, name):
        self.name = name

    def introduce(self):
        return f"I'm {self.name} the frog. {self.walk()} and {self.swim()}."

frog = Amphibian('Freddy')
print(frog.introduce())
# Output: I'm Freddy the frog. I can walk on land and I can swim in water.
```
