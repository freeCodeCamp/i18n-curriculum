# --description--

React Server Components (RSCs) — це новий тренд, який змінив підхід розробників React. Завдяки RSC більше роботи переноситься на сервер, що має багато переваг.

Давайте розглянемо, що таке серверні компоненти, як вони працюють і що призвело до їх появи.

React Server Components — це React-компоненти, які відтворюються виключно на сервері, який надсилає клієнту лише фінальний HTML. Це означає, що такі компоненти можуть безпосередньо отримувати доступ до ресурсів на сервері і суттєво зменшувати обсяг JavaScript, що надсилається в браузер.

Традиційно React-застосунки використовували систему «клієнтських компонентів», яка обробляє все в типовому React-застосунку: відтворення, інтерактивність і побічні ефекти. Термін «клієнтський компонент» рідко використовувався до появи React server components.

Але система клієнтських компонентів має недоліки, такі як великі JavaScript-бандли і повільніший початковий час завантаження.

React-фреймворки, як Next.js і Gatsby, знайшли обхідні шляхи, щоб перенести частину процесів на сервер і вирішити ці проблеми, але жоден із них не був стандартизований. Якщо ви користувалися будь-яким із цих фреймворків, ви, ймовірно, чули про `getServerSideProps` і `getServerData`.

Потім з’явилися React Server Components, які дозволяють запускати деякі компоненти повністю на сервері, щоб ви могли робити такі речі, як отримання даних і обчислення, ще до запуску будь-якого коду в браузері користувача.

Серверні компоненти вперше стали популярними і доступними в Next.js. Інші фреймворки, як Remix і Gatsby, наздоганяють, а для Vite є експериментальний плагін `vite-plugin-react-server`, який дозволяє створювати серверні компоненти.

Отже, як працюють серверні компоненти?

Один із найкращих способів продемонструвати React Server Components — це отримання даних.

У традиційних React клієнтських компонентах браузер обробляє API-запити. Оскільки отримання даних — це побічний ефект, ви робите цей API-виклик у хуку `useEffect`.

Також гарною практикою є встановлення змінних стану, таких як loading, data і error, щоб показати, що дані завантажуються, відобразити дані, коли вони готові, або показати помилку у вашому застосунку.

З React Server Components ви можете перенести весь компонент на сервер і отримувати дані там без необхідності використовувати `useState` або `useEffect`:

```jsx
const Users = async () => {
  const res = await fetch("https://jsonplaceholder.typicode.com/users");
  const users = await res.json();

  return (
    <>
      <h1 className="text-4xl text-center mt-6">Users</h1>
      <ul className="text-center mt-3">
        {users.map((user) => (
          <li key={user.id}>{user.name}</li>
        ))}
      </ul>
    </>
  );
};

export default Users;
```

Оскільки React Server Components запускаються лише на сервері, ви можете просто отримувати дані з API і відтворювати їх один раз. Також, оскільки отримання даних відбувається на сервері, ближче до джерела, ваш застосунок може працювати швидше, особливо для користувачів із повільним інтернет-з’єднанням.

Одна важлива особливість полягає в тому, що весь код серверних компонентів залишається на сервері і не надсилається в браузер. Це означає, що ви не можете використовувати React hooks із ними, і вони не мають доступу до Web API або слухачів подій браузера. Тож як додати інтерактивність?

В маршрутизаторі застосунку Next.js усі компоненти за замовчуванням є серверними. Якщо ви хочете додати інтерактивність, потрібно позначити компонент як клієнтський за допомогою директиви `"use client"`.

Припустимо, ви хочете зробити попередній приклад клієнтським компонентом. Ось як це можна зробити:

```jsx
"use client";

import { useState, useEffect } from "react";

const Users2 = () => {
  const [status, setStatus] = useState({
    users: [],
    loading: true,
    error: null,
  });

  async function fetchUsers() {
    try {
      const res = await fetch("https://jsonplaceholder.typicode.com/users");
      const data = await res.json();
      setStatus((prevStatus) => ({
        ...prevStatus,
        users: data,
        loading: false,
      }));
    } catch (err) {
      setStatus((prevStatus) => ({
        ...prevStatus,
        error: err.message,
        loading: false,
      }));
    }
  }

  useEffect(() => {
    fetchUsers();
  }, []);

  if (status.loading) {
    return <p>Loading Users...</p>;
  }
  if (status.error) {
    return <p>Error getting users: {status.error}</p>;
  }

  return (
    <>
      <h1 className="text-4xl text-center mt-6">Users</h1>
      <ul className="text-center mt-3">
        {status.users.map((user) => (
          <li key={user.id}>{user.name}</li>
        ))}
      </ul>
    </>
  );
};

export default Users2;
```

Якщо ви хочете додати інтерактивність, наприклад події кліку, компонент також має бути позначений як клієнтський:

```jsx
"use client";

import { useState } from "react";

const Counter = () => {
  const [count, setCount] = useState(0);

  return (
    <>
      <h1>Counter</h1>
      <button onClick={() => setCount(count + 1)}>Increment</button>
      <h2>{count}</h2>
      <button onClick={() => setCount(count - 1)}>Decrement</button>
    </>
  );
};

export default Counter;
```

Якщо не додати директиву use client до компонента, ви отримаєте помилку з повідомленням: «Ви імпортуєте серверний компонент, який потребує `useState`. Цей React hook працює лише в клієнтському компоненті. Щоб виправити, позначте файл (або його батьківський елемент) директивою `"use client"`».

Головні переваги React Server Components полягають у тому, що отримання даних стає простішим, код легше читати, а складність клієнта зменшується.
