# --description--

CSP (Content Security Policy) і Permissions-Policy — це два потужні HTTP-заголовки безпеки, які допомагають захистити веб-застосунки, обмежуючи, які ресурси можуть завантажуватися та до яких функцій браузера може мати доступ сторінка.

Content Security Policy зосереджується насамперед на завантаженні ресурсів. Він діє як фільтр безпеки для ваших фронтенд-скриптів, стилів і зображень.

Зловмисники часто намагаються впровадити шкідливі скрипти у вразливі вебсайти. Сильний CSP зупиняє ці атаки Cross-Site Scripting (XSS), встановлюючи суворий білий список довірених доменів джерел.

Коли сервер надсилає заголовок CSP, браузер читає конкретні директиви. Наприклад, директива `script-src` повідомляє браузеру, з яких доменів можна безпечно запускати JavaScript на сторінці.

Якщо хакер намагається впровадити шкідливий скрипт з непогодженого зовнішнього сервера, браузер виявляє порушення політики, миттєво відкидає скрипт і реєструє помилку безпеки.

CSP також може обмежувати джерела зображень, призначення форм і походження таблиць стилів.

Permissions-Policy зосереджується на функціях браузера та апаратному забезпеченні пристрою. Він замінив старіший заголовок Feature-Policy.

Цей механізм безпеки дозволяє власникам сайтів явно увімкнути або вимкнути конкретні функції браузера для своєї сторінки та будь-яких вставлених iframe, а також запобігає зловживанню приватністю користувача сторонніми фрагментами коду.

Використовуючи заголовок Permissions-Policy, ви можете контролювати доступ до апаратного забезпечення, такого як вебкамера, мікрофон і відстеження геолокації.

Наприклад, якщо вашому сайту не потрібно знати місцезнаходження користувача, ви можете встановити політику, щоб повністю заблокувати API геолокації. Навіть якщо зловмисник скомпрометує скрипт на вашій сторінці, браузер відмовиться активувати функцію відстеження.

Реалізація Permissions-Policy використовує чіткий, структурований формат заголовка, який називає функцію та визначає дозволені джерела.

Щоб обмежити апаратну функцію виключно вашим доменом, використовуйте ключове слово `self`, взяте в круглі дужки, наприклад `camera=(self)`. Якщо ж ви хочете вимкнути функцію на всій сторінці для всіх користувачів, використовуйте порожній список, який виглядає як `geolocation=()`.

Поєднання обох заголовків забезпечує надійний захист веб-застосунків. CSP контролює доставку даних і запобігає впровадженню коду, а Permissions-Policy захищає фізичний пристрій і гарантує дотримання приватності.

Сучасні веб-розробники використовують обидва інструменти, щоб створити максимально захищене середовище для перегляду.
