# --description--

- La ricorsione è un concetto di programmazione che ti permette di chiamare una funzione ripetutamente fino a raggiungere un caso base.

Ecco un esempio di funzione ricorsiva che calcola il fattoriale di un numero:

```js
function findFactorial(n) {
  if (n === 0) {
    return 1;
  }
  return n * findFactorial(n - 1);
}
```

Nell'esempio sopra, la funzione `findFactorial` viene chiamata ricorsivamente fino a quando `n` raggiunge `0`. Quando `n` è `0`, si raggiunge il caso base e la funzione restituisce `1`. La funzione poi restituisce il prodotto di `n` e il risultato della chiamata ricorsiva a `findFactorial(n - 1)`.

- La ricorsione ti permette di gestire qualcosa con una profondità sconosciuta, come oggetti/array profondamente annidati o un albero dei file.
- Uno stack di chiamate viene usato per tenere traccia delle chiamate di funzione in una funzione ricorsiva. Ogni volta che una funzione viene chiamata, viene aggiunta allo stack di chiamate. Quando si raggiunge il caso base, le chiamate di funzione vengono rimosse dallo stack.
- Devi definire con attenzione il caso base perché chiamarlo indefinitamente può far crashare il tuo codice. Questo succede perché la ricorsione continua ad accumulare sempre più chiamate di funzione finché il sistema esaurisce la memoria.
- Le ricorsioni trovano impiego nella risoluzione di problemi matematici come il fattoriale e Fibonacci, nell'attraversamento di alberi e grafi, nella generazione di permutazioni e combinazioni e molto altro.
