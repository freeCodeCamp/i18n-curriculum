# --description--

これまで、JavaScriptでクライアントサイドのコードを書いてきました。これは通常ウェブブラウザ上で動作するウェブアプリケーションのクライアント側のコードです。

クライアントサイドのJavaScriptは非常に強力ですが、より複雑なタスクにはいくつかの制限があります。

例えば：

- ローカルファイルへのアクセスが非常に制限されています。
- 複雑なアプリケーションロジックの処理にはあまり向いていません。
- セキュリティ上の懸念があります。例えば、データベースの認証情報などの機密情報をクライアントサイドのコードに公開したくありません。
    
クライアントサイドのJavaScriptにこれらの制限があるのは、もともとウェブブラウザ上でのみ動作するように設計されていたためです。

ウェブブラウザはJavaScriptコードを実行するために必要な環境を提供しており、JavaScriptエンジンも含まれています。また、Document Object Model（DOM）へのアクセスも提供しているため、コード内でHTML要素にアクセスできます。

以前はブラウザの外でJavaScriptを実行することはできませんでしたが、今はNode.jsで可能になりました。

Node.jsはJavaScriptのランタイム環境です。公式には「オープンソースでクロスプラットフォームのJavaScriptランタイム環境」と定義されています。

これが意味することを見てみましょう：

- Node.jsはオープンソースであり、そのコードは公開されていて、多くの開発者コミュニティによってメンテナンスされています。
- クロスプラットフォームであり、Windows、macOS、Linuxなどどのオペレーティングシステムでも動作します。
- JavaScriptランタイム環境であるため、ブラウザの外でJavaScriptコードを実行できます。
    
Node.jsはさまざまな業界で使われています。開発者はHTTPリクエストを処理するウェブサーバーやAPI、ウェブやモバイルのアプリケーションを構築するために使います。

データベースと連携したり複雑なデータを扱うウェブサイトやウェブアプリケーションを使ったり開発したりする場合、どこかでNode.jsが裏側で使われている可能性が高いです。

Node.jsは効率的でスケーラブルであり、世界中に大きな開発者とメンテナのコミュニティがあります。

Node.jsが何かがわかったところで、ブラウザとNodeランタイム環境の違いをいくつか見てみましょう。

ブラウザ環境は主にフロントエンドのウェブ開発向けに設計されているため、クライアントサイドのJavaScriptを実行します。一方、Nodeランタイム環境は主にバックエンドのウェブ開発向けに設計されているため、サーバーサイドのJavaScriptを実行します。これが提供されるAPIの違いにもつながります。

例えば、ブラウザからはDOM APIにアクセスできますが、ローカルファイルシステムへのアクセスには制限があります。対してNodeランタイム環境からはファイルシステムを含むほぼすべてのシステムリソースにアクセスできますが、DOMにはアクセスできません。

両方のJavaScript環境はグローバルオブジェクトを提供します。ブラウザのJavaScriptランタイムではこれを`window`と呼びます。この`window`オブジェクトはDOM操作、クッキー管理、ブラウザイベント処理などブラウザ関連の機能にアクセスできます。Node.js環境ではこのグローバルオブジェクトは`global`と呼ばれます。`global`オブジェクトはローカルファイル操作、ネットワーク、オペレーティングシステムとの連携などNode.js固有の機能にアクセスできます。

最後にもう一つ重要な違いは、サーバーで使うNode.jsのバージョンは選べますが、ユーザーがウェブサイトを訪れる際のブラウザ環境のバージョンは制御できません。

Node.jsは世界中の開発者にとって非常に重要な存在になっています。強力でスケーラブルかつ多用途なツールであり、JavaScriptを使ってフロントエンドとバックエンドの両方を構築できることでウェブ開発を変革しました。これにより、バックエンド開発のために新しいプログラミング言語を学ぶ必要がなくなり、開発プロセスがより効率的になりました。
