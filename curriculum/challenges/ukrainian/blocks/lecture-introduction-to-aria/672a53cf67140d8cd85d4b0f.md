# --interactive--

Зробити статичний вміст доступним може бути досить просто, але з динамічним вмістом це може бути складніше. Саме тут на допомогу приходить WAI-ARIA.

Давайте розглянемо, що таке WAI-ARIA, яка його мета, як він працює та кілька прикладів.

WAI-ARIA означає Web Accessibility Initiative - Accessible Rich Internet Applications. Це специфікація, яка покращує доступність динамічного вмісту та UI (користувацьких інтерфейсів).

Зверніть увагу, що WCAG і WAI-ARIA — це не одне й те саме. WCAG надає загальні рекомендації щодо доступності вебу, тоді як WAI-ARIA пропонує конкретні правила для забезпечення доступності динамічного та інтерактивного вмісту для користувачів допоміжних технологій.

Отже, основна мета WAI-ARIA — покращити доступність динамічного вмісту та UI-компонентів, які не мають нативних HTML-еквівалентів.

WAI-ARIA працює, вводячи набір атрибутів, які ви можете додати до HTML-елементів, щоб надати додаткову семантичну інформацію. Ці атрибути поділяються на ролі, стани та властивості.

ARIA роль визначає призначення елемента на вебсайті або у вебзастосунку. Ось приклад встановлення ролі `button` для елемента `div`.

```html
<div role="button">Click Me</div>
```

Таким чином ви повідомляєте допоміжним технологіям, що цей елемент є кнопкою. Проте ролі не надають жодної функціональності. Просто додати цьому `div` роль `role` зі значенням `button` не змусить його поводитися як кнопка. Щоб він виглядав і працював як кнопка, потрібно використовувати CSS і JavaScript для досягнення бажаного результату.

Ось приклад використання HTML, CSS і JavaScript для створення кастомного `button` елемента.

**ПРИМІТКА**: Не хвилюйтеся, якщо не розумієте CSS і JavaScript-код. Ви вивчите ці мови у майбутніх модулях.

:::interactive_editor

```html
<link href="styles.css" rel="stylesheet">
<div id="custom-btn" role="button">Click Me</div>
<script src="index.js"></script>
```

```css
#custom-btn {
  display: inline-block;
  padding: 0.4em 1em;
  font-size: 1rem;
  font-family: sans-serif;
  color: buttontext;
  background-color: buttonface;
  border: 1px solid #ccc;
  border-radius: 4px;
  cursor: pointer;
  user-select: none;
  text-align: center;
}

#custom-btn:focus {
  outline: 2px solid Highlight;
  outline-offset: 2px;
}

#custom-btn:active {
  background-color: #ddd;
}
```

```js
const button = document.getElementById("custom-btn");

button.addEventListener("click", () => {
  alert("Button clicked!");
});

button.addEventListener("keydown", (e) => {
  if (e.key === "Enter" || e.key === " ") {
    e.preventDefault(); 
    button.click();
  }
});
```

:::

Завжди краще використовувати нативний `button` або `input` елемент з `type="button"` замість цього.

ARIA властивості надають додаткові деталі про елементи. Наприклад, властивість `aria-labelledby` дозволяє зв’язати елемент із конкретною міткою:

```html
<h2 id="header-id">About freeCodeCamp</h2>
<button id="button-id" aria-labelledby="header-id button-id">Learn More</button>
```

Це зробить елементи зрозумілими та зручними для навігації користувачам допоміжних технологій.

Щоб максимально ефективно використовувати WAI-ARIA, намагайтеся використовувати нативний HTML, коли це можливо, оскільки він зазвичай забезпечує кращу доступність «з коробки».

Використовуйте WAI-ARIA лише тоді, коли HTML не справляється, і не забувайте тестувати з допоміжними технологіями, як-от читачами екрана, або залучайте людей з інвалідністю для тестування вашої роботи. Також переконайтеся, що стани та властивості WAI-ARIA оновлюються разом із вмістом у реальному часі. Уникайте надмірного використання ARIA, бо це може збивати з пантелику.
