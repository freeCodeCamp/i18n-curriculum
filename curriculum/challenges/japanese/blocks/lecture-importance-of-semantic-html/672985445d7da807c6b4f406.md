# --interactive--

構造的階層を作成する上で最も重要な点は、見出し要素を正しく使うことです。見出し要素は`h1`、`h2`、`h3`のように番号が付けられています。これらの番号は、その要素の見出しレベルを表しています。

テキストドキュメントと同じように、見出しレベルを間違った順序で使いたくありません。`h1`要素は最上位の見出しです。ページに複数あることはほとんどなく、通常はすべてのコンテンツの前に置きます。

`h2`要素はサブ見出しです。常に`h1`の後に来るべきで、導入文の後に来る場合もあります。`h1`要素とは異なり、ページに複数の`h2`要素を置くことができます。これは異なるセクションを区切るためによく使われます。

このパターンに従うと、`h3`要素は常に`h2`要素の後に来るべきです。つまり、`h1`から直接`h3`に飛ぶことは避けるべきです。ただし、同じレベルの見出し要素を複数置くことは可能です。例えば、このコードは正しいです：

```html
<section>
  <h1>freeCodeCamp</h1>
  <h2>Learn Front-End Development</h2>
  <h2>Learn Back-End Development</h2>
</section>
```

しかし、このコードは正しくありません。なぜなら`h3`が`h2`の前に来ているからです。

`h2`要素を`h3`の上に移動させて、意味的に正しくしてください。コードを操作するには、インタラクティブエディターを有効にする必要があります。

:::interactive_editor

```html
<section>
  <h1>freeCodeCamp</h1>
  <h3>Introduction to HTML</h3>
  <h2>Learn Front-End Development</h2>
</section>
```

:::

スタイルのために特定の見出し要素を使いたくなるかもしれません。例えば、大きなテキストのために`h1`を使う場合です：

:::interactive_editor

```html
<article>
  <p>
    Here is some
    <h1>Large Text</h1>
  </p>
</article>
```

:::

しかし、ドキュメント構造に適した要素を選び、CSSで望むスタイルを実現するべきです。

正しい階層を使うことはアクセシビリティにとって重要です。スクリーンリーダーのような支援技術は、ウェブページの構造をもとに解析し、ユーザーに読み上げます。`h3`要素の後に`h1`要素を使うと、`h2`要素がないためにスクリーンリーダーユーザーが重要な内容を飛ばしたと誤解する場合があります。

適切な構造はSEOにも重要です。検索エンジンは自動でウェブページの内容を解析し、どの検索結果に表示するかを決めます。構造が正しくないと、関連する検索結果での評価が低くなる場合があります。

最後に、構造があまりにも間違っていると、HTMLが技術的に正しくない場合があります。その場合、ブラウザは意図を推測して表示しますが、その推測が必ずしも望んだものとは限りません。

今日学んだように、ページに適切な構造的階層を使う理由はたくさんあります。新しいプロジェクトを作るときはこれを忘れないでください。
