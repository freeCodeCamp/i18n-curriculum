# --interactive--

CSSリセットとは、ウェブブラウザーがHTML要素に適用するすべてまたは一部のデフォルトの書式設定を取り除くスタイルシートのことです。たとえば、段落や見出しなどの特定の要素には、カスタムスタイルシートで明示的に定義しなくても、デフォルトでマージンが設定されていることに気づいたかもしれません。

これはさまざまなHTML要素の多くのプロパティでも見られます。ブラウザーによってHTML要素に適用されるデフォルトスタイルが異なる場合もあります。Google ChromeのデフォルトスタイルはMozilla FirefoxやMicrosoft Edgeとまったく同じではないかもしれません。これによりブラウザー間でスタイルの不一致が生じることがあり、できるだけ避けるべきです。

これに対処するために、CSSリセットはデフォルトスタイルを取り除きます。すべてまたは一部のデフォルトスタイルを取り除くことで、デザインの一貫した基準を得て、ブラウザーやデバイス間の不一致の可能性を最小限にできます。デフォルトスタイルを取り除くことで、カスタムスタイルシートで明示的に適用したスタイルだけが見えるようになり、スタイリングの作業も簡単になります。CSSリセットには主に2つのアプローチがあります。カスタムCSSリセットを定義するか、サードパーティのCSSリセットを使うかです。

カスタムCSSリセットは、プロジェクトのニーズに合わせて一から作成するスタイルシートです。これにより、リセットする特定のスタイルを柔軟にコントロールできます。ただし、スタイルシートの開発とメンテナンスに時間をかける必要があります。以下は、すべてのHTML要素のマージンとパディングをリセットする非常に一般的なCSSルールの例です。

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<h1>Example Heading</h1>
<p>This is a paragraph.</p>
```

```css
* {
  margin: 0;
  padding: 0;
}
```

:::

通常、CSSスタイルシートの先頭に書かれます。アスタリスクセレクターはすべてのHTML要素にマッチするワイルドカードセレクターなので、すべての要素のマージンとパディングが四辺ともゼロにリセットされます。これによりスタート地点ができ、その後スタイルシートのより具体的なCSSセレクターでカスタマイズできます。

この方法を使えば、任意のHTML要素を選択してデフォルトスタイルをリセットできます。カスタムスタイルシートを作成し、適切なCSSセレクターで要素を指定してスタイルを設定してください。

しかしこれは時間がかかる作業です。時間を節約したい場合は、サードパーティのCSSリセットを使うこともできます。これらのスタイルシートはすでに作成済みなので、ダウンロードしてプロジェクトに直接追加できます。サードパーティのCSSリセットの優れた例が`Normalize.css`です。このスタイルシートは幅広いHTML要素のスタイルを正規化しつつ、アクセシビリティに重要な役立つデフォルトスタイルは残しています。また、一般的なバグやスタイルの不一致も修正します。

もう一つの選択肢は`sanitize.css`です。これは主要な最新ブラウザー間でデフォルトスタイルの一貫性を確保するためのCSSライブラリです。このライブラリは`Normalize.css`と共に開発されており、一緒に進化しています。フォームやタイポグラフィの正規化など特定の目的向けに個別のスタイルシートもダウンロードできます。

多くの選択肢がありますが、プロジェクトのニーズに最適なものを選んでください。サードパーティのCSSリセットとカスタムリセットを組み合わせることも可能です。

CSSリセットを使う際はアクセシビリティも考慮することが重要です。ウェブアプリケーションは誰にでもアクセスできるべきです。スクリーンリーダーや他の支援技術に役立つスタイルはリセットしないようにしてください。また、これらの追加スタイルシートがアプリケーションのパフォーマンスに与える影響も考慮してください。カスタムスタイルが適用される前にダウンロードされる必要があるためです。

デフォルトスタイルを取り除くことで、CSSリセットはデザインを実装するための白紙のスタート地点を提供します。これによりブラウザーやデバイス間でより均一で一貫したユーザー体験が実現します。
