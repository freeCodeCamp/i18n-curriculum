# --description--

En esta lección, aprenderás a trabajar con tipos de `function`. Comencemos con las anotaciones de tipo de parámetro.

Aquí tienes un ejemplo de una función básica:

```ts
function circleArea(radius) {
  return Math.PI * radius * radius;
}
```

En este momento, TypeScript mostraría un mensaje de `Parameter 'radius' implicitly has an 'any' type`. El tipo `any` indica que un valor puede tener cualquier tipo. Aprenderás más sobre este tipo en una lección futura.

Para eliminar el error y escribir explícitamente el tipo del parámetro, puedes escribirlo así:

```ts
function circleArea(radius: number) {
  return Math.PI * radius * radius;
}
```

Si intentas llamar a la función `circleArea` con un tipo diferente que no sea un número, TypeScript mostrará un error como este:

```ts
function circleArea(radius: number) {
  return Math.PI * radius * radius;
}

// Argument of type 'string' is not assignable to parameter of type 'number'
circleArea("3");
```

Si intentas proporcionar más argumentos de los que la función espera, TypeScript mostrará un error:

```ts
function circleArea(radius: number) {
  return Math.PI * radius * radius;
}

// Expected 1 arguments, but got 2.
circleArea(3, 4);
```

Si proporcionas muy pocos argumentos, entonces TypeScript también mostrará un error:

```ts
function circleArea(radius: number) {
  return Math.PI * radius * radius;
}

// Expected 1 arguments, but got 0.
// An argument for 'radius' was not provided.
circleArea();
```

Puedes hacer que los parámetros sean opcionales usando el símbolo `?` cerca del parámetro así:

```ts
function area(radius: number, height?: number): number {
  const baseArea = Math.PI * radius * radius;

  if (height !== undefined) {
    const sideArea = 2 * Math.PI * radius * height;
    return baseArea + sideArea;
  }

  return baseArea;
}
```

La verificación condicional dentro de la función está ahí en caso de que no se proporcione el argumento `height`. 

Si quieres escribir los valores de retorno, puedes agregar anotaciones de tipo de retorno así:

```ts
function circleArea(radius: number): number {
  return Math.PI * radius * radius;
}
```

La anotación del tipo de retorno va después de la lista de parámetros. Similar a las anotaciones de tipo de variable, la mayoría de las veces TypeScript será lo suficientemente inteligente para inferir el tipo de retorno. Por eso, usa una anotación explícita del tipo de retorno cuando quieras dejar más claro el propósito de la función o evitar cambios no deseados en el valor de retorno.

El siguiente ejemplo que veremos tratará sobre funciones asíncronas y promesas. Si tu función va a devolver una promesa, puedes usar el tipo `Promise` así:

```ts
async function getCityTemperature(city: string): Promise<number> {
  const response = await fetch(`https://api.example.com/weather?city=${city}`);
  const data = await response.json(); 
  return data.temperature;
}
```

En esta situación, si intentas eliminar el tipo de retorno, TypeScript podría inferir el retorno como `Promise<any>`. Así que esta sería una buena situación para tipar explícitamente el valor de retorno.

Hasta este punto, solo hemos visto ejemplos de funciones nombradas. Pero, ¿cómo funcionan los tipos con funciones anónimas?

Aquí tienes un ejemplo usando una función anónima en el callback pasado a un `forEach`:

```ts
const languages = ["JavaScript", "Python", "TypeScript"];

languages.forEach((lang) => {
  console.log(`${lang.toUpperCase()} has ${lang.length} characters`);
});
```

El parámetro `lang` no tiene una anotación de tipo. En cambio, TypeScript usa tipado contextual, lo que significa que infiere el tipo del contexto en el que aparece.
