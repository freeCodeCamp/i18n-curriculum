# --interactive--

Коли ви працюєте з масивами в JavaScript, часто виникає потреба перевірити, чи всі елементи масиву відповідають певній умові, або чи принаймні один елемент відповідає цій умові.

Саме тут стають у пригоді методи `every()` і `some()`. Ці методи — потужні інструменти, які можуть спростити ваш код і зробити його більш читабельним.

Почнемо з методу `every()`. Цей метод перевіряє, чи всі елементи масиву проходять тест, реалізований за допомогою наданої функції. Простими словами, він перевіряє, чи кожен елемент вашого масиву задовольняє умову, яку ви вказали.

Метод `every()` повертає `true`, якщо надана функція повертає `true` для всіх елементів масиву. Якщо будь-який елемент не проходить тест, метод одразу повертає `false` і припиняє перевірку решти елементів.

Ось приклад, щоб проілюструвати, як працює `every()`:

:::interactive_editor

```js
const numbers = [2, 4, 6, 8, 10];
const hasAllEvenNumbers = numbers.every((num) => num % 2 === 0);

console.log(hasAllEvenNumbers); // true
```

:::

У цьому прикладі ми перевіряємо, чи всі числа в масиві парні. Функція, яку ми передаємо `every()`, перевіряє, чи ділиться кожне число на `2` без остачі. Оскільки всі числа в нашому масиві справді парні, `hasAllEvenNumbers` буде true.

Тепер подивімося на метод `some()`.

Якщо `every()` перевіряє, чи всі елементи проходять тест, то `some()` перевіряє, чи принаймні один елемент проходить тест. Метод `some()` повертає `true` щойно знаходить елемент, який проходить тест. Якщо жоден елемент не проходить тест, він повертає `false`.

Ось приклад, як працює `some()`:

:::interactive_editor

```js
const numbers = [1, 3, 5, 7, 8, 9];
const hasSomeEvenNumbers = numbers.some((num) => num % 2 === 0);

console.log(hasSomeEvenNumbers); // true
```

:::

У цьому прикладі ми перевіряємо, чи є в масиві хоча б одне парне число. Функція, яку ми передаємо `some()`, така сама, як і раніше. Хоча більшість чисел у нашому масиві непарні, `hasEven` буде `true`, бо в масиві є принаймні одне парне число (`8`).

Обидва методи `every()` і `some()` дуже корисні, коли потрібно перевірити дані або умови в масивах. Вони часто можуть замінити більш громіздкі цикли та умовні інструкції, роблячи ваш код чистішим і виразнішим.

Важливо пам’ятати, що обидва методи припиняють виконання, щойно можуть визначити результат. Для `every()` це означає, що він зупиняється, щойно знаходить результат `false`. Для `some()` він зупиняється, щойно знаходить результат `true`. Це може бути корисним для продуктивності, особливо з великими масивами.
