# --description--

OS 모듈은 Node.js에 기본 내장된 또 다른 표준 모듈입니다.

이 모듈을 사용하면 Node가 실행 중인 현재 운영체제와 상호작용할 수 있어 OS 유형, CPU 세부 정보, 사용 가능한 메모리, 전체 메모리, 네트워크 인터페이스 등 중요한 정보를 접근할 수 있습니다.

OS 모듈을 사용하려면 다음과 같이 가져오세요:

```js
const os = require("os");
```

OS 모듈은 애플리케이션을 시스템에 더 잘 맞게 만들 수 있도록 여러 메서드를 제공합니다. 자주 쓰이는 몇 가지를 살펴보겠습니다.

`platform()` 메서드는 Node가 현재 실행 중인 운영체제를 가져옵니다:

```js
const os = require("os");
console.log(os.platform()); // darwin
```

이것은 크로스 플랫폼 스크립팅을 구현할 때 유용할 수 있습니다:

```js
if (os.platform() === 'win32') {
 // Windows specific code
} else {
 // Non-Windows specific code
}
```

`arch()` 메서드는 Node.js가 컴파일된 CPU 아키텍처를 나타내는 문자열을 보여줍니다:

```js
const os = require("os");
console.log(os.arch()); // arm64
```

이것은 사용자가 특정 아키텍처에 맞는 올바른 바이너리와 종속성을 다운로드하도록 할 때 유용합니다.

다른 가능한 값으로는 `'x64'`, `'arm'`, `'arm64'`, `'ia32'`, `'mips'`, `'ppc'` 등이 있습니다.

`type()`는 공식 OS 이름을 가져와 프로그래밍적으로 운영체제를 식별할 수 있게 합니다:

```js
const os = require("os");
console.log(os.type()); // Darwin (the core OS for macOS, iOS, and other Apple products)
```

`release()`는 시스템의 OS 커널 버전을 보여줍니다. OS 커널은 시스템 자원과 하드웨어와 소프트웨어 컴포넌트 간 통신을 관리하는 운영체제의 핵심 부분입니다. 이 메서드는 OS 커널 버전과 서버 요구사항 간 호환성을 추적할 때 유용합니다.

이 값은 `'20.6.0'` 같은 문자열로 반환됩니다:

```js
const os = require("os");
console.log(os.release()); // 25.0.0
```

`version()`은 `release()` 메서드보다 더 자세한 특정 운영체제 버전을 반환합니다:

```js
const os = require("os");
console.log(os.version());
// Darwin Kernel Version 25.0.0: Wed Sep 17 21:41:39 PDT 2025;
// root:xnu-12377.1.9~141/RELEASE_ARM64_T8103
```

`cpus()`은 각 논리 CPU 코어에 대한 세부 정보를 담은 객체 배열을 반환합니다. CPU 부하를 모니터링하는 데 도움이 됩니다:


`uptime()`는 시스템이 부팅된 이후 경과 시간을 보여줍니다. 서버가 얼마나 오래 실행 중인지 확인할 때 유용합니다:

```js
const os = require("os");
console.log(os.uptime()); // 23047
```

`totalmem()`와 `freemem()`는 각각 바이트 단위의 전체 시스템 메모리와 사용 가능한 시스템 메모리를 보여줍니다:

```js
const os = require("os");
console.log(os.totalmem()); // 8589934592 (8 GB)
console.log(os.freemem()); // 93585408 (87 MB)
```

`userInfo()`은 현재 시스템 사용자에 관한 정보를 담은 객체를 반환합니다:

```js
const os = require("os");
console.log(os.userInfo());

/*
```
