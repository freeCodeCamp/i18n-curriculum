# --description--

前のレッスンで依存関係とは何かを学びました。ここでは、依存関係についてさらに詳しく学び、プロジェクトで使う外部パッケージを選ぶ際に考慮すべきことを説明します。

依存関係とは、プロジェクトが動作するために必要な別のコードのことです。通常は他の開発者が作成し、npmレジストリで共有している外部パッケージです。

依存関係を使うことで多くの時間を節約できます。すべてを自分で作る代わりに、すでに一般的な問題を解決しているパッケージをインストールできます。例えば、ウェブサーバーの作成、ルーティングの処理、データの取得、認証の管理などです。

`package.json` ファイルでは、プロジェクトが依存しているすべてのパッケージが `dependencies` キーの下にリストされています。

```json
{
  "name": "package-dependencies",
  "version": "1.0.0",
  "description": "a demo on how package dependencies work",
  "main": "index.js",
  "scripts": {
    "test": "echo \"Error: no test specified\" && exit 1"
  },
  "keywords": [
    "Node",
    "NPM"
  ],
  "author": "Kolade Chris",
  "license": "ISC",
  "dependencies": {
    "chalk": "^5.6.2",
    "lodash": "^4.17.21"
  }
}
```

上の例の `package.json` ファイルでは、`chalk` と `lodash` がプロジェクトの依存関係としてリストされています。これはプロジェクトがこれらのパッケージに依存していることを意味します。これらがインストールされておらず、`package.json` ファイルにリストされていなければ、それらを使う機能は動作しなくなります。

また、`devDependencies` と呼ばれる別の依存関係のカテゴリもあります。これはプロジェクトのビルドやテスト時にのみ必要で、本番環境で動作させる際には不要なパッケージです。一般的な例は以下の通りです。

* プロジェクトに更新があったときに自動で再起動する Nodemon
    
* テスト用の Jest や Mocha
    
* コードのリンティング用 ESLint
    

予想通り、`devDependencies` は `devDependencies` ファイルの `package.json` セクションに表示されます。

```json
{
  "name": "package-dependencies",
  "version": "1.0.0",
  "description": "a demo on how package dependencies work",
  "main": "index.js",
  "scripts": {
    "test": "echo \"Error: no test specified\" && exit 1"
  },
  "keywords": [
    "Node",
    "NPM"
  ],
  "author": "Kolade Chris",
  "license": "ISC",
  "dependencies": {
    "chalk": "^5.6.2",
    "lodash": "^4.17.21"
  },
  "devDependencies": {
    "jest": "^30.2.0",
    "nodemon": "^3.1.10"
  }
}
```

プロジェクトで使うパッケージを選ぶ際には、いくつか重要なポイントがあります。

まずはセキュリティが最優先です。たった一つの安全でないパッケージがプロジェクト全体に問題を引き起こす可能性があります。パフォーマンスも考慮すべきです。大きすぎたり最適化されていないパッケージはプロジェクトの動作を遅くすることがあります。

また、人気がありサポートが充実しているパッケージを選ぶのも役立ちます。人気のパッケージは活発なコミュニティがあり、ドキュメントが充実し、頻繁にアップデートされています。人気を簡単に確認する方法の一つは、npmでの週間ダウンロード数を見ることです。例えば、`chalk` パッケージは毎週30万回以上ダウンロードされています。

<img src="https://cdn.freecodecamp.org/curriculum/lecture-transcripts/what-are-package-dependencies-and-what-are-some-considerations-for-choosing-an-external-package-1.webp" alt="screenshot from npm of chalk package statistics">

メンテナンスも重要です。よくメンテナンスされているパッケージは定期的に更新され、他のソフトウェアの新しいバージョンともより良く連携します。

最後に、ドキュメントを見落とさないでください。わかりやすく充実したドキュメントは、パッケージの使い方を理解しやすくし、プロジェクトへの統合にかかる時間を大幅に節約できます。
