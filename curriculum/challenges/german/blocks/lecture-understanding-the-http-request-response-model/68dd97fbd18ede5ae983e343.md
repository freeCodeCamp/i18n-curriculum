# --description--

Webseiten und Apps müssen Daten sammeln und verarbeiten oder etwas erzeugen, das der Benutzer benötigt, wie zum Beispiel Suchergebnisse. Deshalb sind Formulare, genau wie Text, ein grundlegender Bestandteil des Webs.

Mit Formularen können Benutzer sich auf einer Seite registrieren und anmelden, Nachrichten senden und nach Elementen suchen. Zum Beispiel gelangen Sie zur Google-Suchergebnisseite, indem Sie Ihre Abfrage in das beliebteste Formular im Web, das Google-Such-Eingabefeld, eingeben.

Schauen wir uns an, woraus ein Formular besteht, wie Eingabefelder, das `action`-Attribut, Formularmethoden und vor allem, was passiert, wenn Sie ein Formular absenden.

Formulare werden mit dem `form`-Element auf einer Webseite eingebettet, aber damit Eingaben angezeigt werden, benötigen Sie auch die `input`-Elemente, je nach Bedarf. Einige dieser Eingabeelemente sind Text, E-Mail, Passwort, Auswahl, Datei, versteckte Eingaben und mehr.

Um diese Eingabefelder zu verwenden, müssen Sie sie mit dem `form`-Element umschließen. Jedes der Eingabefelder benötigt ein `name`-Attribut, damit der Server weiß, welche Daten zu welchem Feld gehören.

Hier ist ein Beispiel:

```html
<form>
   <input type="text" name="username" />
   <input type="email" name="email" />
   <input type="file" name="profilePicture" />
   <input type="radio" name="gender" />
   <input type="checkbox" name="subscribe" />
   <input type="hidden" name="userId" />
</form>
```

Die Eingabefelder müssen außerdem validiert werden, um sicherzustellen, dass der Benutzer die korrekten Informationen eingibt.

Diese Validierung kann sowohl auf der Client-Seite als auch auf der Server-Seite erfolgen. HTML ermöglicht grundlegende Client-seitige Validierung mit den `required`- und `pattern`-Attributen. Trotzdem sollten Sie auch serverseitig validieren, um die Sicherheit zu erhöhen, da die HTML-Validierung hauptsächlich für die Benutzererfahrung und sofortiges Feedback gedacht ist.

Das `required`-Attribut gibt an, dass das Eingabefeld ausgefüllt werden muss, während das `pattern`-Attribut reguläre Ausdrücke zulässt, die den Benutzer zwingen, die Informationen so einzugeben, wie Sie es benötigen.

Hier ist ein Beispiel dafür für ein Benutzername-Feld:

```html
<input type="text" name="username" required pattern="[A-Za-z0-9]{3,}" />
```

Nachdem die Eingabefelder validiert wurden, sollte die Einsendung erfolgen. Was passiert also, wenn das Formular abgesendet wird?

Zuerst benötigt das Formular für die Einsendung ein `action`-Attribut, das angibt, wohin der Browser die Werte der Eingabefelder senden soll, und ein `method`-Attribut, das angibt, wie die Formulardaten abgesendet werden. Mehr zum `method`-Attribut später.

So könnte ein action-Attribut aussehen:

```html
<form action="/register">
	<!-- input fields -->
</form>
```

Das `method`-Attribut kann entweder `GET`, `POST`, `PUT`, `PATCH` oder `DELETE` sein:

* `GET` ruft etwas vom Server ab, zum Beispiel Suchergebnisse.
    
* `POST` sendet etwas an den Server, zum Beispiel Login- oder Registrierungsinformationen, die über die Eingabefelder eingegeben wurden.
    
* `PUT` ändert etwas auf dem Server.
    
* `PATCH` ändert etwas teilweise auf dem Server.
    
* `DELETE` entfernt etwas vom Server.
    
Beachten Sie, dass HTML die Methoden `PUT`, `PATCH` und `DELETE` nicht direkt unterstützt, aber es gibt Umgehungsmöglichkeiten. Wenn Sie mit Vanilla JavaScript arbeiten, können Sie die Fetch API verwenden, um die Methode anzugeben, und in Node.js und Express können Sie das `method-override`-Paket verwenden.

So könnte ein Formular aussehen, das eine `GET`-Anfrage macht:

```html
<form action="/search" method="GET">
   <input type="text" name="q">
   <button type="submit">Search</button>
</form>
```

Und so könnte ein Formular aussehen, das eine `POST`-Anfrage macht:

```html
<form action="/submit" method="POST">
   <input type="text" name="username" required pattern="[A-Za-z0-9]{3,}" />
   <input type="password" name="password" />
   <button type="submit">Submit</button>
</form>
```

Nach der `GET`- oder `POST`-Anfrage prüft der Server die serverseitige Validierung und verarbeitet die Eingabewerte entweder oder lehnt sie ab.

Wenn die Validierung erfolgreich ist, sendet der Server die angeforderten Informationen oder wirft einen Fehler, wenn die Validierung fehlschlägt.

Wenn die Validierung erfolgreich ist, könnte die Antwort des Servers eine Umleitung bei Authentifizierung oder Suche sein oder eine Erfolgsmeldung, die die Aktion bestätigt.

Hier eine Zusammenfassung dessen, was passiert, wenn ein Formular abgesendet wird:

* Der Benutzer füllt die Formularfelder aus.
    
* Der Browser prüft die clientseitige Validierung (falls vorhanden) und fährt fort, wenn die Validierung erfolgreich ist.
    
* Der Browser sendet die Formulardaten basierend auf den action- und method-Attributen an den Server.
    
* Der Server empfängt die Daten und verarbeitet sie (speichert sie in einer Datenbank, registriert einen Benutzer, meldet einen Benutzer an, sendet eine Suchanfrage usw.).
    
* Falls eine serverseitige Validierung vorhanden ist, überprüft der Server die Daten vor der Verarbeitung erneut.
    
* Der Server antwortet mit einer Erfolgsmeldung, einem Fehler oder leitet den Benutzer basierend auf dem Validierungsstatus (Erfolg oder Fehler) weiter.
    
* Der Benutzer sieht eine Rückmeldung basierend auf der Serverantwort.
