## مقدمة إلى قواعد البيانات العلائقية

- **قواعد البيانات العلائقية**: تنظم البيانات في جداول مترابطة تتكون من صفوف وأعمدة. كل صف يمثل سجلًا، وكل عمود يمثل خاصية من بيانات السجل.
- **مزايا قواعد البيانات العلائقية**: قابلة للتوسع، وتُستخدم على نطاق واسع في مجالات متعددة (مثل الرعاية الصحية، الأعمال، الألعاب)، ومهيكلة للحفاظ على بيانات موثوقة.
- **حالات الاستخدام الشائعة**: تطوير الويب، أنظمة الجرد، التجارة الإلكترونية، الرعاية الصحية، وتطبيقات المؤسسات.

## المفاهيم الأساسية

- **المخطط (Schema)**: تتطلب قاعدة البيانات العلائقية مخططًا يحدد هيكلها—الجداول، الأعمدة، أنواع البيانات، القيود، والعلاقات.
- **المفاتيح الأساسية**: معرفات فريدة لكل صف في جدول. ضرورية لضمان سلامة البيانات وتُستخدم لربط السجلات بين الجداول عبر المفاتيح الخارجية.
- **المفاتيح الخارجية**: مراجع للمفاتيح الأساسية في جدول آخر، تُستخدم لربط البيانات ذات الصلة عبر الجداول.
- **العلاقات**: بربط الجداول عبر المفاتيح الأساسية والخارجية، يمكنك تنظيم البيانات المطَبَّعة وإجراء استعلامات ذات معنى.
- **مخططات علاقة الكيانات (ERDs)**: تصور كيف ترتبط الكيانات (الجداول) ببعضها في مخطط قاعدة البيانات.
- **سلامة البيانات**: تُفرض باستخدام المفاتيح وأنواع البيانات. تضمن اتساق ودقة البيانات المخزنة.

## أساسيات SQL

- **الاستعلامات**: طلبات لاسترجاع بيانات محددة من قاعدة البيانات.

```sql
SELECT * FROM dogs WHERE age < 3;
```

- **بند WHERE**: تصفية النتائج بناءً على شروط. استخدم عوامل المقارنة مثل `<`، `=`، `>`، وغيرها.
- **الاختيار مع ORDER BY**: استرجاع النتائج وترتيبها بناءً على عمود معين.

```sql
SELECT columns FROM table_name ORDER BY column_name;
```

## عمليات الجدول

- **تعليمة `CREATE TABLE`**: تُستخدم هذه التعليمة لإنشاء جدول جديد في قاعدة البيانات.

```sql
CREATE TABLE first_table();
```

- **تعليمة `ALTER TABLE ADD COLUMN`**: تُستخدم هذه التعليمة لإضافة عمود إلى جدول موجود.

```sql
ALTER TABLE table_name ADD COLUMN column_name DATATYPE;
```

- **تعليمة `ALTER TABLE DROP COLUMN`**: تُستخدم هذه التعليمة لحذف عمود من جدول موجود.

```sql
ALTER TABLE table_name DROP COLUMN column_name;
```

- **تعليمة `ALTER TABLE RENAME COLUMN`**: تُستخدم هذه التعليمة لإعادة تسمية عمود في جدول.

```sql
ALTER TABLE table_name RENAME COLUMN column_name TO new_name;
```

- **تعليمة `DROP TABLE`**: تُستخدم هذه التعليمة لحذف جدول كامل من قاعدة البيانات.

```sql
DROP TABLE table_name;
```

- **تعليمة `ALTER DATABASE RENAME`**: تُستخدم هذه التعليمة لإعادة تسمية قاعدة بيانات.

```sql
ALTER DATABASE database_name RENAME TO new_database_name;
```

- **تعليمة `DROP DATABASE`**: تُستخدم هذه التعليمة لحذف قاعدة بيانات كاملة.

```sql
DROP DATABASE database_name;
```

## القيود وسلامة البيانات

- **تعليمة `ALTER TABLE ADD COLUMN` مع قيد**: تُستخدم هذه التعليمة لإضافة عمود مع قيد إلى جدول موجود.

```sql
ALTER TABLE table_name ADD COLUMN column_name DATATYPE CONSTRAINT;
```

- **قيد `NOT NULL`**: يضمن هذا القيد ألا يحتوي العمود على قيم NULL.

```sql
column_name VARCHAR(50) NOT NULL
```

- **تعليمة `ALTER TABLE ADD PRIMARY KEY`**: تُستخدم هذه التعليمة لإضافة قيد المفتاح الأساسي إلى جدول.

```sql
ALTER TABLE table_name ADD PRIMARY KEY(column_name);
```

- **تعليمة `ALTER TABLE DROP CONSTRAINT`**: تُستخدم هذه التعليمة لحذف قيد من جدول.

```sql
ALTER TABLE table_name DROP CONSTRAINT constraint_name;
```

- **تعليمة `ALTER TABLE ADD COLUMN` مع مفتاح خارجي**: تُستخدم هذه التعليمة لإضافة عمود مفتاح خارجي يشير إلى جدول آخر.

```sql
ALTER TABLE table_name ADD COLUMN column_name DATATYPE REFERENCES referenced_table_name(referenced_column_name);
```

- **تعليمة `ALTER TABLE ADD UNIQUE`**: تُستخدم هذه التعليمة لإضافة قيد UNIQUE إلى عمود.

```sql
ALTER TABLE table_name ADD UNIQUE(column_name);
```

- **تعليمة `ALTER TABLE ALTER COLUMN SET NOT NULL`**: تُستخدم هذه التعليمة لتعيين قيد NOT NULL على عمود موجود.

```sql
ALTER TABLE table_name ALTER COLUMN column_name SET NOT NULL;
```

- **تعليمة `INSERT` مع قيم NULL**: توضح هذه التعليمة كيفية إدخال قيم NULL في جدول.

```sql
INSERT INTO table_name(column_a) VALUES(NULL);
-- or
INSERT INTO table_name(column_b) VALUES('value'); -- if column_a allows nulls
```

- **المفتاح الأساسي المركب**: يحدد هذا القيد مفتاحًا أساسيًا يتكون من عدة أعمدة.

```sql
CREATE TABLE course_enrollments (
    student_id INT,
    course_id INT,
    PRIMARY KEY (student_id, course_id)
);
```

## التلاعب بالبيانات (CRUD)

- **تعليمة `INSERT`**: تُستخدم هذه التعليمة لإدخال صف واحد في جدول.

```sql
INSERT INTO table_name(column_1, column_2) VALUES(value1, value2);
```

- **تعليمة `INSERT` مع حذف أسماء الأعمدة**: توضح هذه التعليمة كيفية إدخال القيم دون ذكر أسماء الأعمدة صراحة، مع الاعتماد على ترتيب الأعمدة الافتراضي في الجدول.

```sql
INSERT INTO dogs VALUES ('Gino', 3);
```

- **تعليمة `INSERT` مع عدة صفوف**: تُستخدم هذه التعليمة لإدخال عدة صفوف في جدول في عملية واحدة.

```sql
INSERT INTO dogs (name, age) VALUES 
('Gino', 3),
('Nora', 2);
```

- **تعليمة `UPDATE`**: تُستخدم هذه التعليمة لتحديث بيانات موجودة في جدول بناءً على شرط.

```sql
UPDATE table_name SET column_name=new_value WHERE condition;
```

- **تعليمة `DELETE`**: تُستخدم هذه التعليمة لحذف صفوف من جدول بناءً على شرط.

```sql
DELETE FROM table_name WHERE condition;
```

## أنواع البيانات

- **نوع البيانات `NUMERIC`**: يُستخدم هذا النوع لتخزين أرقام عشرية دقيقة مع دقة ومقياس محددين.

```sql
price NUMERIC(10, 2)
```

- **نوع البيانات `TEXT`**: يُستخدم هذا النوع لتخزين سلاسل نصية بطول متغير بدون حد معين للطول.

```sql
column_name TEXT
```

- **نوع البيانات `INTEGER`**: يُستخدم هذا النوع لتخزين أعداد صحيحة بدون فواصل عشرية.

```sql
units_sold INTEGER
```

- **أنواع البيانات `SMALLINT` و`BIGINT`**: هي متغيرات من INTEGER بنطاقات أصغر وأكبر على التوالي.

- **نوع البيانات `SERIAL`**: يُستخدم هذا النوع لإنشاء أعمدة أعداد صحيحة متزايدة تلقائيًا في PostgreSQL.

```sql
id SERIAL
```

- **خاصية `AUTO_INCREMENT`**: تُستخدم هذه الخاصية في MySQL لإنشاء أعمدة أعداد صحيحة متزايدة تلقائيًا.

```sql
id INT AUTO_INCREMENT
```

- **نوع البيانات `VARCHAR`**: يُستخدم هذا النوع لتخزين سلاسل نصية بطول متغير مع حد أقصى محدد.

```sql
name VARCHAR(50)
```

- **نوع البيانات `DATE`**: يُستخدم هذا النوع لتخزين قيم التاريخ (السنة، الشهر، اليوم).

```sql
event_date DATE
```

- **نوع البيانات `TIME`**: يُستخدم هذا النوع لتخزين قيم الوقت (الساعة، الدقيقة، الثانية).

```sql
start_time TIME
```

- **نوع البيانات `TIMESTAMP`**: يُستخدم هذا النوع لتخزين قيم التاريخ والوقت، مع إمكانية تضمين معلومات المنطقة الزمنية.

```sql
event_timestamp TIMESTAMP
event_timestamp TIMESTAMP WITH TIME ZONE
```

- **نوع البيانات `BOOLEAN`**: يُستخدم هذا النوع لتخزين قيم صحيحة/خاطئة.

```sql
is_active BOOLEAN
```

## علاقات قواعد البيانات

- **أنواع العلاقات**: هذه هي الطرق المختلفة التي يمكن أن ترتبط بها الجداول في قاعدة بيانات علائقية.
  - واحد إلى واحد
  - واحد إلى متعدد
  - متعدد إلى واحد
  - متعدد إلى متعدد
  - الإشارة إلى الذات (العلاقات العودية)

- **علاقة واحد إلى واحد**: تعني هذه العلاقة أن كل سجل في جدول يرتبط بسجل واحد فقط في جدول آخر.

```md
One employee is assigned exactly one vehicle.
Tables: employees, vehicles
```

- **علاقة واحد إلى متعدد**: تعني هذه العلاقة أن سجلًا واحدًا في جدول يمكن أن يرتبط بعدة سجلات في جدول آخر.

```md
One customer can have many orders.
Tables: customers → orders
```

- **علاقة متعدد إلى متعدد عبر جدول ربط**: تُنفذ هذه العلاقة باستخدام جدول ربط يحتوي على مفاتيح خارجية من كلا الجدولين المرتبطين.

```sql
CREATE TABLE books_authors (
    author_id INT REFERENCES authors(id),
    book_id INT REFERENCES books(id)
);
```

- **العلاقات ذاتية الإشارة**: تحدث هذه العلاقة عندما يشير جدول إلى نفسه، مما يخلق هيكلًا هرميًا.

```md
An employee table where each employee may report to another employee.
```

## SQL المتقدم (الانضمامات)

- **تعليمة `INNER JOIN`**: تُرجع هذه الانضمامة الصفوف التي لها قيم متطابقة في كلا الجدولين فقط.

```sql
SELECT *
FROM products
INNER JOIN sales ON products.product_id = sales.product_id;
```

- **تعليمة `FULL OUTER JOIN`**: تُرجع هذه الانضمامة جميع الصفوف من كلا الجدولين، بما في ذلك الصفوف غير المتطابقة من أي منهما.

```sql
SELECT *
FROM products
FULL OUTER JOIN sales ON products.product_id = sales.product_id;
```

- **تعليمة `LEFT OUTER JOIN`**: تُرجع هذه الانضمامة جميع الصفوف من الجدول الأيسر والصفوف المتطابقة من الجدول الأيمن.

```sql
SELECT *
FROM products
LEFT JOIN sales ON products.product_id = sales.product_id;
```

- **تعليمة `RIGHT OUTER JOIN`**: تُرجع هذه الانضمامة جميع الصفوف من الجدول الأيمن والصفوف المتطابقة من الجدول الأيسر.

```sql
SELECT *
FROM products
RIGHT JOIN sales ON products.product_id = sales.product_id;
```

- **تعليمة `SELF JOIN`**: تُستخدم هذه الانضمامة لربط جدول بنفسه لمقارنة الصفوف داخل نفس الجدول.

```sql
SELECT A.column_name, B.column_name
FROM table_name A
JOIN table_name B ON A.related_column = B.related_column;
```

- **تعليمة `CROSS JOIN`**: تُرجع هذه الانضمامة حاصل الضرب الديكارتي لجدولين، حيث تجمع كل صف من الجدول الأول مع كل صف من الجدول الثاني.

```sql
SELECT *
FROM table1
CROSS JOIN table2;
```

## أوامر خاصة بـ PostgreSQL

- **أمر تسجيل الدخول `psql`**: يُستخدم هذا الأمر لتسجيل الدخول إلى PostgreSQL باستخدام اسم مستخدم وقاعدة بيانات محددين.

```bash
psql --username=freecodecamp --dbname=postgres
```

- **أمر `\l`**: يعرض هذا الأمر جميع قواعد البيانات في مثيل PostgreSQL.

```sql
\l
```

- **أوامر `CREATE DATABASE` و`\c`**: تُستخدم هذه الأوامر لإنشاء قاعدة بيانات جديدة والاتصال بها.

```sql
CREATE DATABASE database_name;
\c database_name
```

- **أمر `\d`**: يعرض هذا الأمر جميع الجداول في قاعدة البيانات الحالية.

```sql
\d
```

- **أمر `\d table_name`**: يعرض هذا الأمر مخطط/هيكل جدول محدد.

```sql
\d table_name
```

- **أمر `\q`**: يخرج هذا الأمر من عميل PostgreSQL.

```bash
\q
```

## قواعد البيانات العلائقية مقابل غير العلائقية

- **قواعد البيانات غير العلائقية (NoSQL)**: تخزن بيانات غير مهيكلة أو شبه مهيكلة. لا تتطلب مخططًا صارمًا وتتمتع بمرونة أكبر لنماذج البيانات المتغيرة.
- **الاختيار بين العلائقية وغير العلائقية**: يعتمد على طبيعة بياناتك ومتطلبات التطبيق.
- **العلاقاتية مقابل غير العلائقية**: اختر العلائقية للبيانات المهيكلة والاتساق؛ وNoSQL للمرونة والبيانات سريعة التغير.

## أنظمة إدارة قواعد البيانات العلائقية الشهيرة

- **MySQL**: مفتوحة المصدر، موثوقة، مستخدمة على نطاق واسع في تطوير الويب، مدعومة من مجتمع كبير.
- **PostgreSQL**: مفتوحة المصدر، متقدمة، قابلة للتوسيع. تدعم أنواع بيانات مخصصة وبرمجة على جانب الخادم.
- **SQLite**: خفيفة الوزن، قائمة على ملف، بدون خادم. مثالية للتطبيقات الصغيرة.
- **Microsoft SQL Server**: قاعدة بيانات مملوكة، موجهة للمؤسسات.
- **Oracle Database**: نظام RDBMS تجاري معروف بالأداء العالي وقابلية التوسع على نطاق واسع.
