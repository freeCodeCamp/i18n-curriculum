# --description--

React stellt viele eingebaute Hooks bereit, mit denen Sie verschiedene Funktionen in Ihren Projekten implementieren können. Dazu gehören `useState`, `useEffect`, `useContext` und andere.

Manchmal benötigen Sie jedoch eine Funktion, bei der Ihnen keiner der eingebauten Hooks weiterhelfen kann. Glücklicherweise können Sie in React Ihre eigenen benutzerdefinierten Hooks erstellen.

Benutzerdefinierte Hooks sind nicht so kompliziert, wie sie vielleicht erscheinen. Es sind einfach wiederverwendbare Funktionen, mit denen Sie Logik über mehrere Komponenten hinweg teilen können. Das bedeutet, Wiederverwendbarkeit ist ein weiterer Grund, warum Sie Ihren eigenen Hook bauen möchten.

Mit einem benutzerdefinierten Hook können Sie Logik aus den Komponenten auslagern, die ihn verwenden, wie zum Beispiel das Abrufen von Daten, das Verwalten von Zuständen, das Umschalten, Nebeneffekte wie das Überprüfen des Online- oder Offline-Status von Benutzern und so weiter.

Sie können den Hook dann importieren und in jeder Komponente verwenden, sodass Sie sich in diesen Komponenten auf das Rendern und die Darstellung konzentrieren können. Das bedeutet weniger Wiederholungen und weniger Duplikate, was wiederum weniger Stellen bedeutet, an denen Sie Änderungen vornehmen müssen, wenn Sie Aktualisierungen vornehmen wollen.

Schauen wir uns nun an, wie Sie Ihren eigenen benutzerdefinierten Hook erstellen können.

In React beginnen alle eingebauten Hooks mit dem Wort `use`, daher sollte Ihr benutzerdefinierter Hook der gleichen Konvention folgen. Der Name Ihres benutzerdefinierten Hooks sollte auch klar kommunizieren, was er tut.

Wenn Ihr benutzerdefinierter Hook also…

- Daten abruft, können Sie ihn `useFetch` nennen

- etwas an- und ausschaltet, können Sie ihn `useToggle` nennen

- oder wenn er Debouncing implementiert, ist `useDebounce` ein guter Name

Angenommen, Sie möchten einen benutzerdefinierten Hook erstellen, um Debouncing in Ihre App einzubauen.

Debouncing ist eine Programmiertechnik, die begrenzt, wie oft eine Funktion ausgeführt wird. Sie funktioniert, indem sie wartet, bis ein Benutzer eine Aktion für eine bestimmte Zeitspanne unterbricht, bevor die Funktion ausgeführt wird. Zum Beispiel macht Debouncing in einem Suchfeld nicht für jeden Tastendruck einen API-Aufruf, sondern wartet, bis der Benutzer für etwa 500 Millisekunden mit dem Tippen pausiert.

Um einen Debouncing-Hook zu erstellen, müssen Sie zuerst eine `useDebounce.jsx`- oder `useDebounce.js`-Datei anlegen. Üblicherweise werden Dateien für benutzerdefinierte Hooks in einem `hooks`-Ordner gespeichert.

Sie können einige eingebaute Hooks innerhalb Ihres eigenen benutzerdefinierten Hooks verwenden. Für Debouncing benötigen Sie die Hooks `useState` und `useEffect`, also importieren Sie diese am Anfang Ihrer Datei:

```js
import { useState, useEffect } from "react";
```

Erstellen Sie als Nächstes eine `useDebounce`-Funktion, die `value` und `delay` als Parameter entgegennimmt. `value` ist die Ressource, auf die Sie warten möchten, und `delay` ist die Zeitspanne, die Sie abwarten wollen. Da Sie für eine bestimmte Zeit warten möchten, sind die Funktionen `setTimeout` und `clearTimeout` nützlich:

```js
function useDebounce(value, delay) {
  const [debouncedValue, setDebouncedValue] = useState(value);

  useEffect(() => {
    const handler = setTimeout(() => {
      setDebouncedValue(value);
    }, delay);

    return () => {
      clearTimeout(handler);
    };
  }, [value, delay]);

  return debouncedValue;
}

export { useDebounce };
```

Der `debouncedValue`-Zustand hält den verzögerten Wert und gibt ihn zurück, der erst nach Ablauf des angegebenen Zeitlimits aktualisiert wird.

`useEffect` ist der Ort, an dem die Magie wirklich passiert. Wenn Sie sich an die vorherige Lektion erinnern, ist alles, was außerhalb des React-Renderzyklus existiert, wie das Setzen und Löschen eines Timers, ein Nebeneffekt, und Sie sollten den `useEffect`-Hook verwenden, um diese zu handhaben.

Innerhalb des `useEffect`-Hooks hier verwenden Sie `setTimeout`, um den `debouncedValue` zu setzen. Sie geben dann eine Aufräumfunktion zurück, die `clearTimeout` verwendet, um den vorherigen Timeout zu löschen, wann immer sich `value` oder `delay` ändert oder die Komponente unmountet.

Und hier ist eine `FootballerSearch`-Komponente, die den `useDebounce`-Hook verwendet, um die Suche 1 Sekunde zu verzögern, nachdem der Benutzer aufgehört hat zu tippen:

```jsx
import { useState, useEffect } from "react";
import { useDebounce } from "./hooks/useDebounce";
import footballers from "./footballers";

const FootballerSearch = () => {
  const [query, setQuery] = useState("");
  const debouncedQuery = useDebounce(query, 1000); // Start searching 1 second after the user stops typing

  useEffect(() => {
    if (debouncedQuery) {
      const results = footballers.filter((footballer) =>
        footballer.toLowerCase().includes(debouncedQuery.toLowerCase()),
      );
      console.log("Search results:", results);
    } else {
      console.log("Search results: []");
    }
  }, [debouncedQuery]);

  return (
    <>
      <h1 style={{ textAlign: "center" }}>Footballer Search App</h1>
      <div style={{ textAlign: "center" }}>
        <input
          style={{ padding: "0.5rem", width: "30%" }}
          type="text"
          value={query}
          onChange={(e) => setQuery(e.target.value)}
          placeholder="Search for a footballer..."
        />
      </div>
    </>
  );
};

export default FootballerSearch;
```

Wie Sie sehen, ist die Variable `debouncedQuery` das, was den `useDebounce`-Hook mit dem Abfragezustand (was der Benutzer tippt) und der Verzögerung von 1.000 Millisekunden bzw. 1 Sekunde initialisiert. Die Suche selbst wird innerhalb des `useEffect`-Hooks gehandhabt, und die Suchergebnisse werden in der Konsole protokolliert.
