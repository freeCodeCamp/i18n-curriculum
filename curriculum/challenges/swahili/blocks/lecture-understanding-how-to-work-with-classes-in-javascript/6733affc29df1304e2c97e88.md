# --interactive--

Katika JavaScript ya kisasa, madarasa ya vitu ni kama ramani za muundo ambazo unaweza kuziandika katika msimbo kwa ajili ya kuunda vitu vingi vya aina moja.

Kwa mfano, ikiwa unaunda mfumo wa hesabu za bidhaa, utahitaji kuunda vitu vingi kwa kila aina ya bidhaa.

Kila bidhaa binafsi itakuwa na sifa na vitendo vyake, lakini bidhaa za aina moja zinaweza kuundwa kutoka kwa "ramani" ile ile. Hiyo ndiyo madarasa ya vitu yanayotumika.

Madarasa ya vitu yanaweza kufafanuliwa kama usemi wa darasa la vitu au tamko la darasa la vitu.

Hapa kuna sintaksia ya msingi kwa tamko la darasa la vitu:

```js
class MyClassName {
  // Class Methods
  constructor() { ... }
  method1() { ... }
  method2() { ... }
  ...
}
```

Unaanza kwa kuandika neno kuu la `class` ikifuatiwa na jina la darasa la vitu. Unapotoa majina ya madarasa ya vitu, unapaswa kutumia mkataba wa majina wa `PascalCase` badala ya `camelCase`.

Yaliyomo ndani ya mabano ya mviringo huitwa mwili wa darasa la vitu. Hapa ndipo kijenzi na njia zitakapoandikwa.

Kijenzi ni njia maalum inayoitwa moja kwa moja wakati kitu kipya kinaundwa kutoka kwa darasa la vitu. Kwa kawaida hutumika kuanzisha sifa za kitu hicho.

Katika mfano huu, unaweza kuona darasa la `Dog` lenye kijenzi.

```js
class Dog {
  constructor(name) {
    this.name = name;
  }
}
```

Neno kuu la `this` linatumika hapa kurejelea mfano wa sasa wa darasa la `Dog`, likikuruhusu kuweka sifa ya `name` kwa kitu kinachoanzishwa. Utajifunza zaidi kuhusu neno kuu la `this` katika funzo lijalo.

Chini ya kijenzi, unaweza kuwa na njia zinazojulikana kama methods. Njia ni kitendakazi kilichofafanuliwa ndani ya darasa la vitu kinachofanya vitendo au operesheni. Zinatumika kufafanua tabia ambazo mifano ya darasa hilo inaweza kufanya.

Hapa kuna mfano wa kuongeza njia inayoitwa `bark` kwa darasa la `Dog`.

```js
class Dog {
  constructor(name) {
    this.name = name;
  }

  bark() {
    console.log(`${this.name} says woof!`);
  }
}
```

Wakati njia ya `bark` inapoitwa, ujumbe utaandikwa kwenye konsoli ukiwa na jina halisi lililopitishwa ndani yake. Ikiwa utaunda mfano wa `Dog` wenye jina `"Buddy"`, kuitisha njia ya `bark()` kwenye mfano huo itachapisha `"Buddy says woof!"` kwenye konsoli.

Mara tu unapokuwa na darasa lako la vitu limefafanuliwa, unaweza kuunda mfano wa darasa hilo kwa kutumia neno kuu la `new`, jina la darasa, na hoja zake.

Katika mfano huu, tunaunda mfano mpya wa `Dog` mwenye jina `Gino` kisha tunaweka thamani yake kwenye mabadiliko ya `dog`.

```js
const dog = new Dog("Gino");
```

Unapoandika `new Dog("Gino")` katika msimbo wako, kitu kipya kinaundwa na kijenzi kinaitwa moja kwa moja kwa hoja zilizotolewa.

Kisha unaweza kupata sifa hii kwa kutumia uandishi wa nukta, kama unavyoona katika mstari ufuatao wa msimbo:

```js
console.log(dog.name);
```

Hapa kuna mfano kamili:

:::interactive_editor

```js
class Dog {
  constructor(name) {
    this.name = name;
  }

  bark() {
    console.log(`${this.name} says woof!`);
  }
}

const dog = new Dog("Gino");
console.log(dog.name);
```

:::

Unaweza pia kuita njia. Katika mfano huu, tunaita njia ya `bark()` kwenye mfano wa `dog`.

```js
dog.bark();
```

Hapa kuna mfano kamili ukiwa na wito wa njia umejumuishwa:

:::interactive_editor

```js
class Dog {
  constructor(name) {
    this.name = name;
  } 
  bark() {
    console.log(`${this.name} says woof!`);
  }
}

const dog = new Dog("Gino");
console.log(dog.name);
dog.bark();
```

:::

Kama ilivyotajwa awali, unaweza pia kufafanua madarasa ya vitu kama usemi wa darasa la vitu. Hii ni pale darasa la vitu linapokuwa lisilo na jina na limepewa thamani kwenye mabadiliko.

Hivi ndivyo mfano wa awali unavyoonekana kama usemi wa darasa la vitu:

```js
const Dog = class {
  constructor(name) {
    this.name = name;
  }

  bark() {
    console.log(`${this.name} says woof!`);
  }
};
```

Mara nyingi, utatumia tamko la darasa la vitu, lakini kuelewa usemi wa darasa la vitu pia ni muhimu, kwani hutoa uhuru zaidi, hasa katika hali za programu zinazobadilika.

Kwa kutumia madarasa ya vitu na kuelewa sintaksia yao ya msingi, unaweza kuunda msimbo unaoweza kutumika tena, kuupanga katika vitengo vya mantiki, na kukuza muundo wa msimbo unaojitegemea.
