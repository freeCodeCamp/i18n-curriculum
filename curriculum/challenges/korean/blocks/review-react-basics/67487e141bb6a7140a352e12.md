## JavaScript 라이브러리와 프레임워크

- JavaScript 라이브러리와 프레임워크는 자주 발생하는 문제에 대한 빠른 해결책을 제공하고, 미리 작성된 코드를 통해 개발 속도를 높여줍니다.
- 라이브러리는 일반적으로 DOM 조작, 이벤트 처리, AJAX 요청 관리 같은 특정 작업에 대한 해결책 제공에 더 집중합니다.
- JavaScript 라이브러리의 예로는 jQuery와 React가 있습니다.
- 반면 프레임워크는 애플리케이션을 구축하기 위한 보다 명확한 구조를 제공합니다. 개발자가 따라야 할 규칙과 관례를 함께 제공하는 경우가 많습니다.
- 프레임워크의 예로는 Angular와 React용 메타 프레임워크인 Next.js가 있습니다.
- **싱글 페이지 애플리케이션**(SPA)은 단일 HTML 페이지를 로드하고, 사용자가 애플리케이션과 상호작용할 때 전체 페이지를 다시 로드하지 않고 해당 페이지를 동적으로 업데이트하는 웹 애플리케이션입니다.
- SPA는 애플리케이션 상태를 관리하고 콘텐츠를 렌더링하기 위해 JavaScript를 사용합니다. 이는 복잡한 사용자 인터페이스를 구축하는 데 유용한 도구를 제공하는 프레임워크를 사용해 이루어지는 경우가 많습니다.
- SPA와 관련된 몇 가지 문제는 다음과 같습니다:

  - 화면 읽기 프로그램이 동적으로 업데이트되는 콘텐츠를 처리하는 데 어려움을 겪습니다.
  - 사용자가 애플리케이션 내에서 이동해도 URL이 변경되지 않아 특정 페이지를 북마크하거나 뒤로 가기, 공유하기가 어렵습니다.
  - 애플리케이션이 크면 초기 로드 시간이 느릴 수 있는데, 모든 자산을 미리 로드해야 하기 때문입니다.

## React

- React는 사용자 인터페이스와 웹 애플리케이션을 구축하기 위한 인기 있는 JavaScript 라이브러리입니다.
- React의 핵심 개념은 데이터가 변경될 때 독립적으로 업데이트되고 렌더링되는 재사용 가능한 UI 컴포넌트를 만드는 것입니다.
- React는 애플리케이션 상태에 따라 UI가 어떻게 보여야 하는지 설명할 수 있게 하며, 데이터나 상태가 변경되면 적절한 컴포넌트를 업데이트하고 렌더링합니다.

## React 컴포넌트

- 컴포넌트는 React 애플리케이션의 구성 요소로, 복잡한 사용자 인터페이스를 더 작고 관리하기 쉬운 조각으로 나눌 수 있게 합니다.
- UI는 JSX를 사용해 설명하는데, JSX는 JavaScript 구문의 확장으로, 개발자가 JavaScript 안에서 HTML과 유사한 코드를 작성할 수 있게 합니다.
- 컴포넌트는 기본적으로 UI 조각을 반환하는 JS 함수나 클래스입니다.

다음은 인사 메시지를 렌더링하는 간단한 React 컴포넌트 예시입니다:

```jsx
function Greeting() {
  const name = 'Anna';
  return <h1>Welcome, {name}!</h1>;
}
```

컴포넌트를 사용하려면 다음과 같이 호출할 수 있습니다:
    
```jsx
  <Greeting />
```

## React 컴포넌트 가져오기와 내보내기

- React 컴포넌트는 한 파일에서 내보내고 다른 파일에서 가져올 수 있습니다.
- 예를 들어 `City`라는 컴포넌트가 `City.js`라는 파일에 있다고 가정해 보겠습니다. `export` 키워드를 사용해 컴포넌트를 내보낼 수 있습니다:

```jsx
// City.js
function City() {
  return <p>New York</p>;
}

export default City;
```

- 다른 파일에서 `City` 컴포넌트를 가져오려면 `import` 키워드를 사용하세요:

```jsx
// App.js
import City from './City';

function App() {
  return (
    <div>
      <h1>My favorite city is:</h1>
      <City />
    </div>
  );
}
```

- `default` 키워드는 `City.js` 파일에서 기본 내보내기이기 때문에 사용됩니다.

- 컴포넌트 정의와 동시에 같은 줄에서 내보내기를 선택할 수도 있습니다:

```jsx
export default function City() {
  return <p>New York</p>;
}
```

## Vite를 사용한 React 프로젝트 설정

- 프로젝트 설정 도구와 명령줄 인터페이스는 새 프로젝트를 빠르고 쉽게 시작할 수 있게 하여, 개발자가 설정 대신 코드 작성에 집중할 수 있게 합니다.
- Vite는 인기 있는 프로젝트 설정 도구이며 React와 함께 사용할 수 있습니다.
- 터미널에서 다음 명령어를 사용해 Vite로 새 프로젝트를 만들 수 있습니다:

```bash
npm create vite@latest my-react-app -- --template react
```

이 명령어는 Vite의 React 템플릿을 사용해 `my-react-app`라는 새 React 프로젝트를 만듭니다. 프로젝트 디렉터리에는 프로젝트 의존성과 명령어가 나열된 `package.json` 파일이 있습니다.

- 프로젝트를 실행하려면 프로젝트 디렉터리로 이동한 후 다음 명령어를 실행하세요:

```bash
cd my-react-app # path to the project directory
npm install # installs the dependencies listed in the package.json file
```

- 의존성이 설치되면 프로젝트 내에 `node_modules`라는 새 폴더가 생성된 것을 확인할 수 있습니다.

- `node_modules` 폴더는 프로젝트에 필요한 모든 패키지와 라이브러리가 저장되는 곳입니다.

- 프로젝트를 실행하려면 다음 명령어를 사용하세요:

```bash
npm run dev
```

- 그 후 브라우저를 열고 `http://localhost:5173`로 이동하면 React 애플리케이션이 실행되는 것을 볼 수 있습니다.

- 시작 템플릿의 코드를 실제로 보려면 프로젝트 내 `src` 폴더로 들어가 `App.jsx` 파일을 확인하세요.

## React 컴포넌트에 프로프스 전달하기

- React에서 프로프스(속성의 줄임말)는 부모 컴포넌트에서 자식 컴포넌트로 데이터를 전달하는 방법입니다. 이 메커니즘은 재사용 가능하고 동적인 UI 요소를 만들 때 필요합니다.
- 프로프스는 어떤 JavaScript 값도 될 수 있습니다. 부모 컴포넌트에서 자식 컴포넌트로 프로프스를 전달하려면, 부모의 JSX에서 자식 컴포넌트를 사용할 때 프로프스를 속성으로 추가하면 됩니다. 간단한 예는 다음과 같습니다:

```jsx
// Parent component
function Parent() {
  const name = 'Anna';
  return <Child name={name} />;
}

// Child component
function Child(props) {
  return <h1>Hello, {props.name}!</h1>;
}
```

- 전개 연산자 `(...)`를 사용해 객체로 변환한 후 여러 프로프스를 전달할 수 있습니다. 예시는 다음과 같습니다:
    
```jsx
// Parent component
function Parent() {
  const person = {
    name: 'Anna',
    age: 25,
    city: 'New York'
  };
  return <Child {...person} />;
}
```

이 코드에서 전개 연산자 `{...person}`는 person 객체를 개별 프로프스로 변환해 Child 컴포넌트에 전달합니다.

## React에서 조건부 렌더링

- React의 조건부 렌더링은 동적인 사용자 인터페이스를 만들 수 있게 합니다. 애플리케이션 내 특정 조건이나 상태에 따라 다른 콘텐츠를 보여줄 때 사용합니다.
- React에서 조건부 렌더링을 하는 여러 방법이 있습니다. 흔한 방법 중 하나는 삼항 연산자를 사용하는 것입니다. 예시는 다음과 같습니다:

```jsx
function Greeting({ isLoggedIn }) {
  return (
    <div>
      {isLoggedIn ? <h1>Welcome back!</h1> : <h1>Please log in</h1>}
    </div>
  );
}
```

- 또 다른 방법은 논리 AND (`&&`) 연산자를 사용하는 것입니다. 특정 조건이 참일 때만 콘텐츠를 렌더링할 때 유용합니다. 예시는 다음과 같습니다:

```jsx
function Greeting({ user }) {
  return (
    <div>
      {user && <h1>Welcome, {user.name}!</h1>}
    </div>
  );
}
```

위 코드에서 `h1` 요소는 user 객체가 참일 때만 렌더링됩니다.

직접 `if` 문을 이렇게 사용할 수도 있습니다:

```jsx
function Greeting({ isLoggedIn }) {
  if (isLoggedIn) {
    return <h1>Welcome back!</h1>;
  }
  return <h1>Please sign in</h1>;
}
```

## React에서 리스트 렌더링

- React에서 리스트 렌더링은 사용자 인터페이스를 구축할 때 흔히 하는 작업입니다.
- JS 배열 `map()` 메서드를 사용해 배열의 항목을 순회하며 새로운 JSX 요소 배열을 반환해 리스트를 렌더링할 수 있습니다.
- 예를 들어, 이름 배열을 리스트로 렌더링하려면 다음과 같이 할 수 있습니다:

```jsx
function NameList({ names }) {
  return (
    <ul>
      {names.map((name, index) => (
        <li key={`${name}-${index}`}>{name}</li>
      ))}
    </ul>
  );
}
```

- 각 리스트 항목에 고유한 키를 제공해 React가 업데이트와 렌더링 역할을 관리할 수 있게 하는 것을 항상 기억하세요. 이 기법들을 사용하면 React 애플리케이션에서 유연하고 효율적이며 동적인 리스트를 만들 수 있습니다.

## React에서 인라인 스타일

- React의 인라인 스타일은 JavaScript 객체를 사용해 JSX 요소에 CSS 스타일을 직접 적용할 수 있게 합니다.
- React에서 인라인 스타일을 적용하려면 JSX 요소에 style 속성을 사용하세요. style 속성은 키가 카멜 케이스인 CSS 속성이고 값은 해당 속성 값인 객체를 받습니다. 예시는 다음과 같습니다:

```jsx
function Greeting() {
  return (
    <h1
      style={{ color: 'blue', fontSize: '24px', backgroundColor: 'lightgray' }}
    >
      Hello, world!
    </h1>
  );
}

export default Greeting;
```

- 스타일을 별도의 객체로 추출해 `style` 속성에서 참조할 수도 있습니다:

```jsx
function Greeting() {
  const styles = {
    color: 'blue',
    fontSize: '24px',
    backgroundColor: 'lightgray'
  };

  return <h1 style={styles}>Hello, world!</h1>;
}

export default Greeting;
```

- 인라인 스타일은 프로프스나 상태에 따라 조건부로 스타일을 적용하는 동적 스타일링을 지원합니다. 다음은 프로프스에 따라 조건부로 스타일을 적용하는 예시입니다:

```jsx
function Greeting({ isImportant }) {
  const styles = {
    color: isImportant ? 'red' : 'black',
    fontSize: isImportant ? '24px' : '16px'
  };

  return <h1 style={styles}>Hello, world!</h1>;
}

export default Greeting;
```

- 위 코드에서 `color`과 `fontSize` 스타일은 `isImportant` 프로프스에 따라 조건부로 설정됩니다.
