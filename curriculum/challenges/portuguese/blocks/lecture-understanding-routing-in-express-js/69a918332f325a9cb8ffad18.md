# --description--

No Express, **manipuladores de rota** (também chamados de funções de callback de rota ou funções controladoras) são funções que definem o que acontece quando um cliente envia uma requisição para uma rota específica.

Um manipulador de rota processa a requisição recebida, executa qualquer lógica necessária como consultar um banco de dados ou realizar cálculos, e então envia uma resposta de volta para o cliente.

Vamos detalhar isso com um exemplo simples (que você na verdade já vem usando o tempo todo):

```javascript
app.get("/menu", (req, res) => {
  res.send("Welcome to the menu!");
});
```

O método `.get()` é um dos métodos de rota do Express. Ele escuta requisições **GET** em uma rota especificada. Neste exemplo, ele escuta requisições para `/menu`.

A função `(req, res) => { ... }` é o **manipulador de rota**. Ela é executada sempre que um cliente envia uma requisição GET para `/menu`.

Dentro do manipulador:

* `req` (requisição) é um objeto que contém informações sobre a requisição recebida, como parâmetros da rota, strings de consulta, cabeçalhos e dados do corpo.
    
* `res` (resposta) é um objeto usado para enviar uma resposta de volta para o cliente.
    

Neste caso, `res.send("Welcome to the menu!")` envia uma resposta em texto simples para o cliente.

Quando um usuário navega para `/menu` no navegador — ou envia uma requisição GET para essa rota — o servidor executa esse manipulador e responde com a mensagem `Welcome to the menu!`.

Manipuladores de rota são o núcleo de uma aplicação Express. Eles permitem que você defina como diferentes rotas da sua aplicação web vão responder às requisições. Você também pode adicionar lógica a esses manipuladores, como processar dados de formulários, interagir com bancos de dados ou realizar validação.
