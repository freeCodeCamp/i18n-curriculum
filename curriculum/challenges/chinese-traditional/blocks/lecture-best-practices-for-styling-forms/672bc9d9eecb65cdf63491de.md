# --interactive--

讓我們了解何時使用 `appearance: none` 來解決搜尋輸入框、核取方塊和圓鈕的樣式問題。

瀏覽器會對許多元素套用預設樣式。在輸入元素的情況下，你用 CSS 來樣式化它們的能力可能會感覺相當受限。因此，你可能想使用 `appearance: none` 來隱藏預設元素的某些部分，並改為建置你自己的樣式。舉例來說，這會隱藏核取方塊輸入的預設核取方塊，讓你可以使用訂製的指示器，例如綠色勾勾和紅色叉叉來顯示狀態。對於搜尋輸入，基於 WebKit 的瀏覽器會顯示預設的搜尋圖示和取消按鈕。隱藏這些可以讓你創建自己的指示器，並在所有瀏覽器上顯示。

.checkbox {
  appearance: none;
  width: 18px;
  height: 18px;
  border: 2px solid #ccc;
  border-radius: 4px;
  display: inline-block;
  position: relative;
  cursor: pointer;
  transition: all 0.25s ease;
  vertical-align: middle; 
}

.checkbox:hover {
  border-color: #888;
}

.checkbox:checked {
  background-color: #4caf50;
  border-color: #4caf50;
}

.checkbox:checked::after {
  content: "";
  position: absolute;
  left: 4px;
  top: 0px;
  width: 5px;
  height: 10px;
  border: solid white;
  border-width: 0 2px 2px 0;
  transform: rotate(45deg);
}

.checkbox:focus {
  outline: 2px solid #90caf9;
  outline-offset: 2px;
}

WebKit 是一個幫助網頁瀏覽器顯示網站的軟體引擎。像 Safari 這樣的瀏覽器使用 WebKit 來確保網頁的外觀和功能正確。這個 `appearance: none` CSS 屬性讓你可以完全控制樣式，但確實有一些需要注意的地方。輸入元素的預設互動組件包含像是焦點和錯誤指示器等功能，你需要確保這些不會遺失。

建立一致的跨平台樣式是使用此屬性的絕佳理由。你也可以使用它來確保行動裝置上的點擊標的足夠大，或是核取方塊的顏色具有足夠的對比度。
