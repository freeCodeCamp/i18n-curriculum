# --description--

HTML `async`와 `defer` 속성은 `script` 요소에서 JavaScript 파일이 웹 페이지에 로드되고 실행되는 방식을 결정하는 데 중요한 역할을 합니다. 이를 이해하면 웹사이트 성능과 사용자 경험을 향상시킬 수 있습니다. HTML 파일에 `script`를 포함하면 다음과 같이 보입니다:

```js
<script src="example.js"></script>
```

브라우저가 이 `script` 태그를 만나면 HTML 파싱을 중단하고, 스크립트를 다운로드한 후 실행하고 다시 HTML 파싱을 계속합니다. 이 과정은 특히 큰 스크립트가 있을 때 웹 페이지 로딩 속도를 느리게 할 수 있습니다.

이때 `async`와 `defer` 속성이 등장합니다. 이들은 스크립트를 더 효율적으로 로드하는 방법을 제공합니다.

먼저 `async` 속성부터 살펴보겠습니다:

```js
<script src="example.js" async></script>
```

`async` 속성을 `script` 태그에 추가하면, 브라우저는 스크립트를 다운로드하는 동안 HTML 파싱을 계속합니다. 스크립트가 완전히 다운로드되면 브라우저는 HTML 파싱을 잠시 멈추고 스크립트를 실행한 후 다시 HTML 파싱을 재개합니다. 이 방법은 페이지 로딩 속도를 크게 향상시킬 수 있습니다.

async 스크립트는 다운로드가 완료되는 즉시 실행되므로, 원하는 순서대로 실행되지 않을 수 있다는 점을 기억하세요. 이 문제를 해결하는 것이 `defer` 속성입니다. `defer` 속성이 어떻게 생겼는지 살펴보겠습니다:

```js
<script src="example.js" defer></script>
```

`defer` 속성은 `async` 속성과 비슷합니다. 하지만 defer 스크립트는 다운로드 직후 실행되지 않고, HTML 파싱이 완료될 때까지 기다립니다. 또한 defer 스크립트는 HTML 코드에 나타난 순서대로 실행됩니다.

요약하자면, 실행 순서가 중요하지 않은 스크립트에는 `async`를 사용하고, 스크립트가 올바른 순서로 실행되어야 할 때는 `defer`를 사용하세요. 두 속성 모두 스크립트가 백그라운드에서 다운로드되는 동안 브라우저가 HTML 파싱을 계속할 수 있게 하여 페이지 로딩 시간을 크게 줄여줍니다.
