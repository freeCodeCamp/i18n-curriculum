# --description--

Molti linguaggi e framework offrono modi per lavorare con file e cartelle. Node.js ti permette di farlo con il modulo `fs` o modulo del file system.

Questa lezione si concentrerà sul modulo Node `fs`, come funziona e i tre diversi modi in cui puoi usare i metodi che fornisce.

Il modulo Node `fs` ti offre metodi per lavorare con file e cartelle, inclusi operazioni di apertura e chiusura, lettura e scrittura, ed eliminazione.

Il modulo `fs` è un modulo standard di Node.js, quindi è disponibile finché hai Node.js installato nel tuo ambiente. Per usare il modulo, lo importi in questo modo:

```js
const fs = require("fs");
```

La flessibilità che offre il modulo `fs` è che ti permette di usare i suoi metodi sia in modo sincrono sia asincrono. I metodi del modulo `fs` sono asincroni per impostazione predefinita, ma per ogni metodo esiste una forma sincrona:

```js
fs.writeFile() // Asynchronous file writing
fs.writeFileSync() // Synchronous file writing

fs.readFile() // Asynchronous file reading
fs.readFileSync() // Synchronous file reading

fs.open() // Opens a file
fs.openAsBlob() // Opens as blob
fs.openSync() // Synchronous open
fs.opendir() // Opens directory
fs.opendirSync() // Synchronous directory open
```

Non finisce qui. A seconda delle tue esigenze, puoi usare i metodi in tre modi:

- con callback, per esempio `fs.writeFile()` (asincrono)
- con promise se preferisci la sintassi `async/await`, per esempio `fs.promises.writeFile()`
- in modo sincrono, per esempio `fs.writeFileSync()`

Ecco la sintassi base per l’utilizzo asincrono dei metodi:

```js
fs.writeFile("filePath", "content", "utf8", (err) => {
  if (err) {
    throw err;
  }
  console.log("File written to!");
});
```

Per la versione con promise, puoi importarla da `fs/promises` o concatenare `promises` al tuo import `fs`. Ecco la sintassi:

```js
async function promisesExample() {
  try {
    await fs.promises.writeFile("filePath", "content", "utf8");
    console.log("File written to!");
  } catch (err) {
    console.error("Error:", err);
  }
}

promisesExample();
```

Ecco la versione sincrona:

```js
try {
  fs.writeFileSync("filePath", "content", "utf8");
  console.log("File written to!");
} catch (err) {
  console.error("Error:", err);
}
```

Nota che usare uno qualsiasi dei metodi in modo sincrono è bloccante. In altre parole, il tuo programma si fermerà e aspetterà che l’operazione finisca prima di passare alla riga successiva di codice.

Per script veloci, progetti piccoli o attività una tantum, i metodi sincroni vanno probabilmente bene. Ma nelle applicazioni reali, l’I/O bloccante diventa un problema perché può congelare altre parti della tua app mentre Node aspetta che il file system completi le operazioni. Questo rallenta tutto e peggiora le prestazioni e l’esperienza utente.

Ecco perché esistono i metodi asincroni. Permettono al resto del programma di continuare a funzionare mentre Node gestisce le operazioni sui file in background. Tradizionalmente si usavano le callback, ma possono diventare complicate man mano che la tua app cresce.

Promise e `async/await` risolvono questo problema. Sono ancora non bloccanti, ma il codice si legge come se fosse sincrono normale ed è molto più facile da mantenere.

Per questo motivo, oggi si preferisce generalmente l’approccio basato sulle promise, ed è quello che useremo nel resto di questa lezione.

Il metodo `writeFile()` ti permette di scrivere in un file esistente. Se il file non esiste, lo crea nella directory corrente e poi ci scrive il contenuto specificato.

Ecco come usare il metodo `writeFile()`:

```js
const fs = require("fs/promises");

async function writeToFile() {
  try {
    await fs.writeFile(
      "article.md",
      "## Node `fs` Module: The Complete Guide",
      "utf8",
    );
    console.log("File written to!");
  } catch (err) {
    console.error("Error writing to file:", err);
  }
}

writeToFile(); // File written to!
```

Ecco il contenuto del file article.md dopo:

```md
## Node `fs` Module: The Complete Guide
```

Il metodo `appendFile()` ti permette di aggiungere al contenuto di un file esistente. Ecco come usarlo:

```js
const fs = require("fs/promises");

async function appendToFile() {
  try {
    await fs.appendFile(
      "article.md",
      "\nIn this article, you will learn all there is to know about the Node fs module...",
      "utf8",
    );
    console.log("File appended to!");
  } catch (err) {
    console.log("Error appending to file:", err);
  }
}

appendToFile(); // File appended to!
```

Il metodo `readFile()` ti permette di vedere qual è il contenuto di un file:

```js
const fs = require("fs/promises");

async function readFileContent() {
  try {
    const fileContent = await fs.readFile("article.md", "utf8");
    console.log("File content:", fileContent);
  } catch (err) {
    console.log("File read successfully");
  }
}

readFileContent();

/*
File content: ## Node `fs` Module: The Complete Guide

In this article, you will learn all there is to know about the Node fs module...
*/
```

Se non specifichi la codifica dei caratteri `utf8`, otterrai il contenuto del file come buffer:

```js
async function readFileContent() {
  try {
    const fileContent = await fs.readFile("article.md");
    console.log("File content:", fileContent);
  } catch (err) {
    console.log("File read successfully");
  }
}

readFileContent();

/*
File content: <Buffer 23 23 20 4e 6f 64 65 20 60 66 73 60 20 4c 69 62 72 61 72 79 3a 20
 54 68 65 20 43 6f 6d 70 6c 65 74 65 20 47 75 69 64 65 0a 0a 49 6e 20 74 68 69 73 20 ... 
72 more bytes>
*/
```

Infine, il metodo `unlink()` ti permette di eliminare un file:

```js
const fs = require("fs/promises");

async function deleteFile() {
  try {
    await fs.unlink("article.ts");
    console.log("File deleted successfully!");
  } catch (err) {
    console.error("Error deleting file:", err);
  }
}

deleteFile();
```

Se il file non esiste, riceverai un messaggio di errore simile a questo:

```bash
Error deleting file: Error: ENOENT: no such file or directory, unlink 'article.ts'
```

Se esiste, il file viene eliminato e riceverai il messaggio di successo che hai impostato.
