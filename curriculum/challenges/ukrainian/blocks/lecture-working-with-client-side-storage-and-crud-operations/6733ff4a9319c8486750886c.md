# --description--

Web Storage API надає механізм для браузерів зберігати пари ключ-значення безпосередньо в браузері, що дозволяє розробникам зберігати інформацію, яку можна використовувати під час різних перезавантажень сторінок і сесій.

Два основні компоненти в Web Storage API — це `localStorage` і `sessionStorage`.

`localStorage` — це частина Web Storage API, яка дозволяє даним зберігатися навіть після закриття вікна браузера або оновлення сторінки. Ці дані залишаються доступними, доки їх явно не видалить застосунок або користувач.

`sessionStorage` — це інша частина Web Storage API, яка зберігає дані протягом сесії сторінки, тобто дані доступні, поки відкрита вкладка або вікно браузера. Однак, на відміну від `localStorage`, дані в `sessionStorage` очищуються, коли вкладку або вікно закривають. Ви дізнаєтеся більше про `sessionStorage` у наступному уроці.

Загальні випадки використання `localStorage` включають збереження налаштувань користувача, таких як теми або мовні уподобання, запам’ятовування даних форм між сесіями браузера та кешування невеликих фрагментів інформації для покращення продуктивності веб-застосунків.

Кешування означає зберігання часто використовуваних даних у тимчасовому сховищі, відомому як кеш, щоб наступні запити цих даних оброблялися швидше без необхідності повторного обчислення або отримання їх із повільнішого джерела, такого як база даних або зовнішній сервер.

Деякі загальні методи `localStorage` включають методи `setItem`, `getItem`, `removeItem` і `clear`.

Ось приклад використання методу `setItem()`, який зберігає пару ключ-значення в `localStorage`.

```js
localStorage.setItem('username', 'JaneDoe');
```

Якщо ми хочемо отримати значення за певним ключем із `localStorage`, ми можемо використати метод `getItem()`.

```js
let username = localStorage.getItem('username');
console.log(username); // JaneDoe
```

Щоб видалити елемент із `localStorage` за його ключем, можна використати метод `removeItem()`.

```js
localStorage.removeItem('username');
```

Щоб очистити всі дані в `localStorage`, можна використати метод `clear()`.

```js
localStorage.clear();
```

Тепер розглянемо приклад, де ми використовуємо `localStorage` для збереження обраної користувачем теми.

```js
// Store the user's theme preference
localStorage.setItem('theme', 'dark');

// Retrieve the stored theme preference
const userTheme = localStorage.getItem('theme');
console.log(userTheme); // 'dark'

// Remove the theme preference
localStorage.removeItem('theme');

// Clear all localStorage data
localStorage.clear();
```

У цьому прикладі:

- Спочатку ми зберігаємо вибір теми (`dark`) для користувача.

- Потім отримуємо цю тему і виводимо її в консоль.

- Нарешті, демонструємо, як видалити конкретний елемент або очистити всі збережені дані.

`localStorage` дуже корисний для зберігання невеликих фрагментів даних, які мають зберігатися між сесіями, але важливо пам’ятати, що `localStorage` не слід використовувати для зберігання конфіденційної інформації, такої як паролі, оскільки це може становити загрозу безпеці.

## --text--

Чим `localStorage` відрізняється від `sessionStorage`?

## --answers--

Дані `localStorage` очищуються при закритті вкладки, `sessionStorage` зберігаються постійно.

### --feedback--

Подумайте, що відбувається з `localStorage`, коли ви закриваєте і знову відкриваєте браузер.

---

---
