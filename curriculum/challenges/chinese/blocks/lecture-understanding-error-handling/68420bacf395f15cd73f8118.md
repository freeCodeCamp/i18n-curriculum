# --description--

编写 Python 代码时，常常会遇到错误。理解这些错误是快速且高效调试你的代码的关键。如果你知道如何读取这些消息，它们会准确告诉你出了什么问题。

常见的 Python 误差包括 `SyntaxError`、`NameError`、`TypeError`、`IndexError` 和 `AttributeError`。当 Python 无法理解你的代码，或者你的逻辑与正在处理的数据不匹配时，就会发生这些情况。

这是一个 `SyntaxError` 的示例：

```py
print("Hello, world!"
# SyntaxError: unexpected EOF while parsing
```

这一行缺少一个闭合括号。Python 引起 `SyntaxError`，因为代码不符合正确的语法规则。

这是一个 `NameError` 的示例：

```py
print(name)
# NameError: name 'name' is not defined
```

你正在尝试打印一个尚未定义的变量。当 Python 无法查找该名称的变量时，会引起一个 `NameError`。

这是一个 `TypeError` 的示例：

```py
5 + "5"
# TypeError: unsupported operand type(s) for +: 'int' and 'str'
```

你不能将整数和字串相加。当你尝试对不兼容的数据类型执行操作时，Python 会引起 `TypeError`。

这是一个 `IndexError` 的示例：

```py
my_list = [1, 2, 3]
print(my_list[5])
# IndexError: list index out of range
```

你正在尝试访问列表中不存在的索引。当你超出边界时，Python 会引起 `IndexError`。

这是一个 `AttributeError` 的示例：

```py
num = 42
num.append(5)
# AttributeError: 'int' object has no attribute 'append'
```

`int` 对象没有 `append()` 方法。当你尝试使用该数据类型不存在的方法或属性时，Python 会引起 `AttributeError`。

识别常见的 Python 错误信息可以帮助你更快地解决问题。不要猜测，要仔细读取错误信息，它通常会准确告诉你出了什么问题以及该去哪里查找。
