# --description--

**C**RUD Teil I – CREATE

Zuerst brauchen wir ein Schema. Jedes Schema entspricht einer MongoDB-Collection. Es definiert die Struktur der Dokumente innerhalb dieser Collection. Schemata sind Bausteine für Modelle. Sie können verschachtelt werden, um komplexe Modelle zu erstellen, aber in diesem Fall halten wir es einfach. Ein Modell ermöglicht es Ihnen, Instanzen Ihrer Objekte zu erstellen, die Dokumente genannt werden.

Auf Servern finden die Interaktionen mit der Datenbank in Handler-Funktionen statt. Diese Funktionen werden ausgeführt, wenn ein Ereignis eintritt (z. B. wenn jemand einen Endpunkt Ihrer API aufruft). Wir folgen in diesen Übungen demselben Ansatz. Die `done()`-Funktion ist ein Callback, der uns mitteilt, dass wir nach Abschluss einer asynchronen Operation wie Einfügen, Suchen, Aktualisieren oder Löschen fortfahren können. Sie folgt der Node-Konvention und sollte bei Erfolg als `done(null, data)` oder bei einem Fehler als `done(err)` aufgerufen werden.

Hinweis – Bei der Interaktion mit entfernten Diensten können Fehler auftreten!

```js
/* Example */

const someFunc = function(done) {
  //... do something (risky) ...
  if (error) return done(error);
  done(null, result);
};
```

# --instructions--

Erstellen Sie ein Personenschema namens `personSchema` mit folgender Struktur:

* Ein erforderliches `name`-Feld vom Typ `String`
* Ein `age`-Feld vom Typ `Number`
* Ein `favoriteFoods`-Feld vom Typ `[String]`

Verwenden Sie die grundlegenden Schema-Typen von Mongoose. Wenn Sie möchten, können Sie auch weitere Felder hinzufügen, einfache Validatoren wie required oder unique verwenden und Standardwerte festlegen. Siehe unseren <a href="https://www.freecodecamp.org/news/introduction-to-mongoose-for-mongodb-d2a7aa593c57/" target="_blank" rel="noopener noreferrer nofollow">Mongoose-Artikel</a>.

Erstellen Sie nun ein Modell aus dem `personSchema` und weisen Sie es der vorhandenen Variablen `Person` zu.
