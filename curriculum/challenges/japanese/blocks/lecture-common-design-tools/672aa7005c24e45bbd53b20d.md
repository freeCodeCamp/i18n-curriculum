# --description--

新しい機能やアプリケーションを設計する際、最初の良いステップはデザインブリーフを作成することです。

デザインブリーフは、プロジェクトの目的、目標、要件をまとめたドキュメントです。これは設計プロセスの道しるべとなり、最終的な成果物がクライアントのニーズを満たすことを保証します。

通常、クライアントがデザインブリーフを書き、それが作業用の草案として使われます。時にはデザイナーが作成し、クライアントと相談してニーズに合っているか確認する場合もあります。

デザインブリーフに含めるべき重要な要素がいくつかあります。

最初の要素は、プロジェクトとビジネスの概要です。この概要には、会社の詳細、ミッション、価値観、独自の強み、製品やサービスが含まれるべきです。

次の重要な要素は、プロジェクトの目標と目的を文書化することです。これにはプロジェクトの目的や望ましい成果が含まれます。

目標の例としては、サイトへのトラフィックを増やすことや、月間ページ訪問数をXパーセント増やすことなどがあります。

もう一つの重要な要素はターゲットオーディエンスです。デザインブリーフには、ターゲットとなる層の属性、興味、ニーズに関する情報を含めるべきです。

また、競合についての情報や、プロジェクトが競合とどう差別化するかについても含めるべきです。

さらに重要な要素はプロジェクトのスコープです。これには成果物、スケジュール、予算が含まれます。成果物には、モックアップや最終デザインなど、プロジェクトの一部として作成されるすべてのアイテムのリストが含まれます。

プロジェクトのスコープを明確に定義しないと、物事が手に負えなくなり予算オーバーになることがあります。ですから、何をいつまでに納品するかをできるだけ詳細にすることが重要です。

プロジェクト設計で難しい点の一つは、スケジュールと予算です。与えられた期間と予算内で何が達成できるかを現実的に考えることが大切です。したがって、これらの制約を明示したデザインブリーフが重要になります。

これらの詳細がすべて話し合われ文書化されたら、プロジェクト開始前にすべての関係者がデザインブリーフをレビューし承認する必要があります。その時点でデザイナーは作業を開始できます。

では、開発者の役割は何でしょうか？開発者の役割は、デザインを受け取り、プロジェクトの要件を理解し、それを動作する製品に変えることです。

これにはコードを書くこと、テストすること、そしてアプリケーションがデザインブリーフに記載された要件を満たしているかデバッグすることが含まれます。

多くの場合、開発者は複数人のチームで作業を分担します。

また通常、プロジェクトマネージャーが作業の調整とプロジェクトの進行管理を担当します。

ですから、開発者として設計や初期の意思決定プロセスに関わらない場合でも、デザインブリーフを理解し、それが自分の作業にどう影響するかを知ることは重要です。
