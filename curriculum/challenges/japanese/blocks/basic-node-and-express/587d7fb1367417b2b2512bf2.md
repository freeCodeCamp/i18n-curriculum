# --description--

`.env` ファイルは、環境変数をアプリケーションに渡すために使われる隠しファイルです。このファイルは秘密で、あなた以外はアクセスできません。APIキーやデータベースのURIなど、非公開または隠しておきたいデータを保存できます。また、設定オプションを保存することも可能です。設定オプションを設定することで、一部のコードを書き換えることなくアプリケーションの動作を変更できます。

環境変数はアプリから `process.env.VAR_NAME` としてアクセスできます。`process.env` オブジェクトはグローバルなNodeオブジェクトで、変数はすべて文字列として渡されます。慣例として、変数名はすべて大文字で、単語はアンダースコアで区切られます。`.env` はシェルファイルなので、名前や値を引用符で囲む必要はありません。変数に値を代入するとき、イコールの前後にスペースを入れてはいけないことも重要です。例：`VAR_NAME=value`。通常、変数定義はそれぞれ別の行に書きます。

# --instructions--

設定オプションとして環境変数を追加しましょう。

プロジェクトディレクトリのルートに `.env` ファイルを作成し、その中に変数 `MESSAGE_STYLE=uppercase` を保存してください。

次に、前のチャレンジで作成した `/json` のGETルートハンドラ内で `process.env.MESSAGE_STYLE` にアクセスし、変数が `message` と等しい場合はレスポンスオブジェクトの `uppercase` を大文字に変換してください。レスポンスオブジェクトは `{"message": "Hello json"}` の値によって `{"message": "HELLO JSON"}` または `MESSAGE_STYLE` のいずれかになります。テストの実行方法のため、`process.env.MESSAGE_STYLE` の値はルートハンドラの**内部**で読み取る必要があることに注意してください。

`dotenv` パッケージを使う必要があります。これは `.env` ファイルから環境変数を読み込み `process.env` に設定します。`dotenv` パッケージはすでにインストールされており、プロジェクトの `package.json` ファイルにあります。`myApp.js` ファイルの先頭に `require('dotenv').config()` を追加して環境変数を読み込んでください。

# --hints--

エンドポイント `/json` のレスポンスは環境変数 `MESSAGE_STYLE` によって変わるはずです。

```js
  const response = await fetch(code + '/_api/use-env-vars');
  if (!response.ok) {
    throw new Error(await response.text());
  }
  const data = await response.json();
  assert.isTrue(
    data.passed,
    'The response of "/json" does not change according to MESSAGE_STYLE'
  );
```
