# --description--

路徑是用來指定檔案或目錄在檔案系統中位置的字串。在網頁開發中，路徑讓開發者能夠連結到像是影像、樣式表、腳本和其他網頁等資源。路徑有絕對路徑和相對路徑兩種－兩者在指定檔案位置時都很重要。讓我們來看看這兩種路徑，這樣你就能決定何時使用哪一種。

絕對路徑是指向資源的完整聯結。它從根目錄開始，包含每個其他目錄，最後是檔名和擴充套件。「根目錄」指的是階層體系中的頂層目錄或資料夾。

絕對路徑也包含協定－可能是 `http`、`https` 和 `file`，以及如果資源位於網路上則包含網域名稱。以下是一個連結到 freeCodeCamp 標誌的絕對路徑範例：

```html
<a href="https://design-style-guide.freecodecamp.org/img/fcc_secondary_small.svg">
  View fCC Logo
</a>
```

在此範例中，協定是 `https`，網域名稱是 `design-style-guide.freecodecamp.org`，而檔案名稱是 `fcc_secondary_small.svg`。

現在，如果你想用絕對路徑連結的資源是在你的區域的機器上，該怎麼辦？以下是如何用絕對路徑連結 `about.html` 檔案的方法：

```html
<p>
  Read more on the
  <a
    href="/Users/user/Desktop/fCC/script-code/absolute-vs-relative-paths/pages/about.html"
    >About Page</a
    >
</p>
```

看起來是這樣，因為我們先進入名為 `Users` 的資料夾，再進入名為 `user` 的資料夾，接著進入名為 `Desktop` 的資料夾，再進入名為 `fCC` 的資料夾，然後進入名為 `script-code` 的資料夾，再進入名為 `absolute-vs-relative-paths` 的資料夾，接著進入名為 `pages` 的資料夾，最後取得 `about.html` 檔案。

以下是在瀏覽器位址列中絕對 URL 的顯示方式：

```sh
file:///Users/user/Desktop/fCC/script-code/absolute-vs-relative-paths/pages/about.html
```

URL 包含了協定、`file://`。它也包含路徑，看起來像這樣：`/Users/user/Desktop/fCC/script-code/absolute-vs-relative-paths/pages/`，並表述通往該檔案的一連串資料夾。最後，它也包含了 `about.html`，這是檔名和擴充套件。

現在，讓我們來看看相對路徑。相對路徑指定檔案相對於目前檔案所在目錄的位置。它不包含協定或網域名稱，使其在同一網站內部聯結時更短且更具彈性。以下是從同一資料夾中的 `contact.html` 頁面連結到 `about.html` 頁面的範例：

```html
<p>
  Read more on the
  <a href="about.html">About Page</a>
</p>
```

所以想像你在 `contact.html` 頁面，因為 `about.html` 頁面在相同位置，你只需取得檔案名稱。這是使用相對檔案路徑的範例。

那麼，你應該使用哪一種路徑，絕對路徑還是相對路徑？以下是你應該遵循的規則：

- 當連結到託管於外部網站的資源時，請使用絕對路徑。

- 當你需要聯結到頁面或資源能夠在網站內的文件位置不變時持續有效，請使用絕對路徑。

- 當在同一個網站內聯結到資源時，請使用相對路徑。

- 如果你想在開發過程中讓你的程式碼更乾淨且更容易維護，請使用相對路徑。

- 在區域測試期間使用相對路徑，以確保聯結在沒有網際網路連接時仍能運作。
