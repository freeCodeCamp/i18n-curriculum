# --description--

Kabla hujaanza kutumia udhibiti wa toleo kwa kweli, na tovuti kama GitHub, unahitaji kuwa na Git imesakinishwa na iko tayari kutumika.

Ili kuangalia kama Git tayari imesakinishwa kwenye kompyuta yako unaweza kuendesha amri ifuatayo kwenye terminal:

```sh
git --version
```

Kama unaona nambari ya toleo, hiyo inamaanisha Git imesakinishwa. Kama sivyo, basi utahitaji kuisakinisha.

Kwa mifumo ya Linux, Git mara nyingi huja tayari imewekwa na distros nyingi. Kama huna Git imewekwa tayari, unapaswa kuweza kuisakinisha kwa amri za meneja wa pakiti kama `sudo apt-get install git` au `sudo pacman -S git`.

Kwa watumiaji wa Mac, unaweza kusakinisha Git kupitia Homebrew kwa `brew install git`, au unaweza kupakua kifurushi cha kusakinisha kutoka kwenye tovuti ya Git.

Kwa Windows, unaweza kupakua kifurushi cha kusakinisha kutoka kwenye tovuti ya Git. Au, kama umeweka Chocolatey, unaweza kuendesha `choco install git.install` kwenye PowerShell. Kumbuka kwamba kwenye Windows, pia unaweza kutaka kupakua GitBash ili uwe na mazingira ya shell yanayofanana na Unix.

Ili kuhakikisha usakinishaji umefanikiwa, endesha amri ya `git --version` tena kwenye terminal.

Mara tu unapokuwa na Git imesakinishwa, utahitaji kufanya mabadiliko machache kwenye usanidi. Unapaswa kufanya mabadiliko haya mara moja tu kwenye kompyuta yako na hutahitaji kuendelea kuyafanya kila unaposasisha.

`git config` hutumika kuweka vigezo vya usanidi vinavyohusika na jinsi Git inavyofanya kazi kwenye kompyuta yako. Ili kuona vigezo vyako vya sasa vya usanidi na mahali vinahifadhiwa kwenye mfumo wako, unaweza kuendesha amri ifuatayo:

```sh
git config --list --show-origin
```

Kwa sasa unapaswa kuona tu usanidi wa ngazi ya mfumo ikiwa umeweka Git kwa mara ya kwanza. Kama tayari umeweka Git kabla ya funzo hili, basi huenda unaona usanidi wa mtumiaji kwa jina la mtumiaji na barua pepe.

Ili kuweka jina lako la mtumiaji, unaweza kuendesha amri ifuatayo:

```sh
git config --global user.name "Jane Doe"
```

Bendera ya `--global` hutumika hapa kuweka jina la mtumiaji kwa miradi yote kwenye mfumo wako inayotumia Git. Ikiwa unahitaji kubadilisha jina la mtumiaji kwa mradi fulani, basi unaweza kuendesha amri hiyo kwenye saraka ya mradi husika bila bendera ya `--global`.

Ili kuweka anwani ya barua pepe ya mtumiaji, unaweza kuendesha amri ifuatayo:

```sh
git config --global user.email janedoe@example.com
```

Kusanidi jina lako la mtumiaji na barua pepe ni muhimu kwa sababu Git itatumia taarifa hii kwa kila commit unayofanya kwenye mradi. Utajifunza zaidi kuhusu commits katika funzo lijalo.

Usanidi mwingine unaoweza kuweka ni mhariri unaopendelea Git atumie. Hapa kuna mfano wa jinsi ya kuweka mhariri unaopendelea kuwa Emacs:

```sh
git config --global core.editor emacs
```

Chaguzi nyingine ni pamoja na Vim, Nano, VS Code, na zaidi. Kwa mfano, kama unatumia VS Code kama mhariri wako, basi unaweza kusanidi usanidi wa Git kama ifuatavyo:

```sh
git config --global core.editor "code --wait"
```

Kama hutaki kuweka mhariri unaopendelea, basi Git itatumia mhariri wa chaguo-msingi wa mfumo wako.

Kama unatumia kompyuta ya Windows na unataka kubadilisha mhariri wako, utahitaji kutoa njia kamili ya faili la executable kama ifuatavyo:

```sh
git config --global core.editor "'C:/Program Files/Notepad++/notepad++.exe' -multiInst -notabbar -nosession -noPlugin"
```

Ili kuona orodha yako ya usanidi, unaweza kuendesha amri ifuatayo:

```sh
git config --list
```

Sasa umeweka na kusanidi Git, uko tayari kuanza kuongeza udhibiti wa toleo kwenye miradi yako.
