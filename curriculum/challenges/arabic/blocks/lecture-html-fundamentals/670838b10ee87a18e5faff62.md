# --interactive--

يُستخدم عنصر `script` لتضمين شفرة قابلة للتنفيذ. يستخدم معظم المطوّرين هذا العنصر لتنفيذ شفرة JavaScript. تُستخدم JavaScript لإضافة التفاعلية إلى صفحات الويب الخاصة بك. من الأمثلة الشائعة على استخدام JavaScript الألعاب التفاعلية، وعروض الصور المتحركة، والنماذج الديناميكية التي تتحقق من صحة مُدخلات المستخدم في الوقت الحقيقي.

إليك مثالًا على استخدام عنصر `script` في مستند HTML. احذف `//` من أمام `alert("Welcome to freeCodeCamp");` وسترى نافذة تنبيه تظهر. لعرض المعاينات، يجب تفعيل المحرر التفاعلي.

:::interactive_editor

```html
<body>
  <script>
    // alert("Welcome to freeCodeCamp");
  </script>
</body>
```

:::

رغم أنه يمكنك كتابة كل شفرة JavaScript داخل وسم `script`، إلا أن أفضل الممارسات توصي بربط ملف JavaScript خارجي بدلاً من ذلك. إليك مثالًا على استخدام عنصر `script` لربط ملف JavaScript خارجي:

```html
<script src="path-to-javascript-file.js"></script>
```

تُستخدم الخاصية `src` هنا لتحديد موقع ملف JavaScript الخارجي. تعني `src` "المصدر". السبب في عدم تشجيع وضع كل شفرة JavaScript داخل مستند HTML هو مبدأ فصل الاهتمامات. فصل الاهتمامات هو مبدأ تصميم تفصل فيه برامجك إلى أقسام مميزة، ويعالج كل قسم اهتمامًا منفصلًا. في هذه الحالة، نريد فصل شفرة JavaScript عن شفرة HTML.
