# --description--

Nella lezione precedente, abbiamo imparato a lavorare con `localStorage` e siamo stati brevemente introdotti a `sessionStorage`.

Ricorda che `sessionStorage` indica quando i dati vengono cancellati non appena l’utente chiude la scheda o la finestra in cui gira l’applicazione web. È ideale per situazioni in cui i dati devono persistere solo per la durata di una singola sessione, come mantenere i dati di un modulo durante la navigazione o memorizzare informazioni temporanee di stato durante un processo di checkout.

Proprio come `localStorage`, `sessionStorage` usa coppie chiave-valore per archiviare e recuperare dati. I metodi usati con `sessionStorage` sono gli stessi di `localStorage`, con l’unica vera differenza nella durata della memorizzazione dei dati.

Ecco alcuni esempi di come lavorare con i diversi metodi:

- `sessionStorage.setItem()`: Memorizza una coppia chiave-valore in `sessionStorage`.

```js
sessionStorage.setItem('cart', '3 items');
```

- `sessionStorage.getItem()`: Recupera il valore di una chiave specifica da `sessionStorage`.

```js
let cart = sessionStorage.getItem('cart');
console.log(cart); // Outputs: '3 items'
```

- `sessionStorage.removeItem()`: Rimuove un elemento specifico da `sessionStorage` usando la sua chiave.

```js
sessionStorage.removeItem('cart');
```

- `sessionStorage.clear()`: Cancella tutti i dati memorizzati in `sessionStorage`.

```js
sessionStorage.clear();
```

Vediamo un esempio in cui memorizziamo dati in `sessionStorage` che durano solo finché la scheda o la finestra del browser è aperta:

```js
// Store data in sessionStorage
sessionStorage.setItem('currentUser', 'JohnDoe');

// Retrieve the stored data
const user = sessionStorage.getItem('currentUser');
console.log(user); // 'JohnDoe'

// Remove a specific key from sessionStorage
sessionStorage.removeItem('currentUser');

// Clear all sessionStorage data
sessionStorage.clear();
```

In questo esempio:

1. Memorizziamo il nome dell’utente corrente (`JohnDoe`) in `sessionStorage`.

2. Lo recuperiamo e lo mostriamo.

3. Rimuoviamo l’elemento associato alla chiave `currentUser`.

4. Cancelliamo tutti i dati di `sessionStorage`.

La differenza principale rispetto a `localStorage` è che non appena l’utente chiude la scheda, tutti i dati di sessione memorizzati vengono persi.

`sessionStorage` è particolarmente utile in scenari come:

- Memorizzare dati temporanei come le voci di un modulo durante un processo multi-pagina.

- Memorizzare selezioni o preferenze temporanee che non devono persistere tra le sessioni.

- Mantenere lo stato in un’applicazione a pagina singola che non deve essere ricordato una volta chiusa la scheda.

`sessionStorage` garantisce che una volta che l’utente lascia la pagina, i dati di sessione vengano cancellati, il che è ottimo per scenari in cui non vuoi conservare informazioni oltre la sessione corrente.
