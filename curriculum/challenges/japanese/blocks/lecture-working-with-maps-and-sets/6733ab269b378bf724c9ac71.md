# --description--

JavaScriptでは、`Set`はデータコレクションを管理するための組み込みオブジェクトです。プリミティブ型でもオブジェクト参照でも、あらゆる型の一意な値を保存できます。`Set`は、その中の各値が一度だけ現れることを保証するため、配列から重複を取り除いたり、異なる値のコレクションを扱うのに便利です。

`WeakSet`は、機能が少ない特別なタイプの`Set`で、弱く保持されたオブジェクト参照やシンボルを保存できます。`Set`とは異なり、`WeakSet`は数値や文字列のようなプリミティブはサポートしません。

通常の`Set`とは違い、`WeakSet`はオブジェクトのみを保存し、それらのオブジェクトへの参照は「弱い」ため、WeakSetは他に参照がなければ保存されたオブジェクトのガベージコレクションを妨げません。簡単に言うと、コードの他の場所で使われていないオブジェクトは自動的に削除され、メモリが解放されます。

`Set`を作成するには、`Set`コンストラクタを使い、変数に代入します。

```js
const myFirstSet = new Set();
```

また、`Set`を値で初期化することもできます。

```js
const treeSet = new Set(['Baobab', 'Jackalberry', 'Mopane Tree', 'Breadfruit']);
```

`Set`をコンソールに出力すると、次のように表示されます。

```js
/*
Set(4) {'Baobab', 'Jackalberry', 'Mopane Tree', 'Breadfruit'}
  [[Entries]]
  0: "Baobab"
    value: "Baobab"
  1: "Jackalberry"
    value: "Jackalberry"
  2: "Mopane Tree"
    value: "Mopane Tree"
  3: "Breadfruit"
    value: "Breadfruit"
  size: 4
  [[Prototype]]: Set
*/
```

もし`Set`を値で初期化していなければ、`add()`メソッドを使ってアイテムを`Set`に追加できます。

```js
const treeSet = new Set();

// Add items to the treeSet
treeSet.add('Baobab');
treeSet.add('Jackalberry');
treeSet.add('Mopane Tree');
treeSet.add('Breadfruit');
```

結果とコンソールでの見た目は同じです。

`Set`では重複したアイテムは無視されることを忘れないでください。

```js
const treeSet = new Set();

// Add items to the treeSet
treeSet.add('Baobab');
treeSet.add('Jackalberry');
treeSet.add('Mopane Tree');
treeSet.add('Breadfruit');
treeSet.add('Baobab'); //duplicate item will be ignored

console.log(treeSet);
// Set(4) {'Baobab', 'Jackalberry', 'Mopane Tree', 'Breadfruit'}
```

`Set`を操作するために使える他のメソッドは以下の通りです。

- `delete()`  
- `clear()`
- `has()`
- `entries()`
- `forEach()`
- `keys()`
- `values()`

これらのメソッドがどのように動作するか、一つずつ見ていきましょう。`delete()`は`Set`から指定したアイテムを削除します。

```js
const treeSet = new Set();

// Add items to the treeSet
treeSet.add('Baobab');
treeSet.add('Jackalberry');
treeSet.add('Mopane Tree');
treeSet.add('Breadfruit');

treeSet.delete('Breadfruit');

console.log(treeSet); // Set(3) {'Baobab', 'Jackalberry', 'Mopane Tree'}
```

`has()`は`Set`に指定した値が存在するかどうかをチェックします。

```js
const treeSet = new Set();

// Add items to the treeSet
treeSet.add('Baobab');
treeSet.add('Jackalberry');
treeSet.add('Mopane Tree');
treeSet.add('Breadfruit');

treeSet.delete('Breadfruit');

console.log(treeSet.has('Breadfruit')); // false
```

`entries()`は`Set`イテレータを返し、`[value, value]`形式の配列に含まれる値を取得できます。

```js
const treeSet = new Set();

// Add items to the treeSet
treeSet.add('Baobab');
treeSet.add('Jackalberry');
treeSet.add('Mopane Tree');
treeSet.add('Breadfruit');

treeSet.delete('Breadfruit');

console.log(treeSet.entries());
// SetIterator {'Baobab' => 'Baobab', 'Jackalberry' => 'Jackalberry', 'Mopane Tree' => 'Mopane Tree'}
```

`keys()`と`values()`は`Set`の値を表示します。`keys()`は`values()`メソッドの別名です。

```js
const treeSet = new Set();

// Add items to the treeSet
treeSet.add('Baobab');
treeSet.add('Jackalberry');
treeSet.add('Mopane Tree');
treeSet.add('Breadfruit');

treeSet.delete('Breadfruit');

console.log('Keys: ', treeSet.keys());
console.log('Values: ', treeSet.values());
// Keys: SetIterator {'Baobab', 'Jackalberry', 'Mopane Tree'}
// Values: SetIterator {'Baobab', 'Jackalberry', 'Mopane Tree'}
```

`forEach()`は`Set`を繰り返し処理できます。

```js
const treeSet = new Set();

// Add items to the treeSet
treeSet.add('Baobab');
treeSet.add('Jackalberry');
treeSet.add('Mopane Tree');
treeSet.add('Breadfruit');

treeSet.delete('Breadfruit');

treeSet.forEach((tree) => console.log(tree));
/*
Baobab
Jackalberry
Mopane Tree
*/
```

`clear()`は配列のすべてのアイテムを削除します。

```js
const treeSet = new Set();

// Add items to the treeSet
treeSet.add('Baobab');
treeSet.add('Jackalberry');
treeSet.add('Mopane Tree');
treeSet.add('Breadfruit');

treeSet.delete('Breadfruit');

treeSet.clear();

console.log(treeSet); // Set(0) {size: 0}
```

また、`size`プロパティがあり、`Set`内のアイテム数を返します。

```js
const treeSet = new Set();

// Add items to the treeSet
treeSet.add('Baobab');
treeSet.add('Jackalberry');
treeSet.add('Mopane Tree');
treeSet.add('Breadfruit');

treeSet.delete('Breadfruit');

console.log(treeSet.size); // 3
```

`Set`と同様に、`WeakSet`コンストラクタを使って`WeakSet`を作成できます。

```javascript
const treeWeakSet = new WeakSet();
```

`WeakSet`には`add()`、`delete()`、`has()`メソッドもあります。

```javascript
const treeWeakSet = new WeakSet();

treeWeakSet.add({ name: 'Baobab' });
treeWeakSet.add({ name: 'Jackalberry' });
treeWeakSet.add({ name: 'Mopane Tree' });
treeWeakSet.add({ name: 'Breadfruit' });

treeWeakSet.delete('Jackalberry');
console.log(treeWeakSet.has('Jackalberry')); // false

console.log(treeWeakSet);
```

出力では、`WeakSet`の内容はこのように表示されます。

```javascript
/*
WeakSet {{…}, {…}, {…}, {…}}
  [[Entries]]
    No properties
  [[Prototype]]: WeakSet
    .
    .
    .
*/
```

WeakSetはイテラブルではなく、内容を直接公開しないため、内容は空に見えます。

シンボルやキーと値が明確に定義されたオブジェクトのみがサポートされることも覚えておいてください。数値や文字列のようなプリミティブを追加するとエラーになります。

```js
treeWeakSet.add('Alan Smith');

console.log(treeWeakSet); // Invalid value used in weak set
//    at WeakSet.add (<anonymous>)
```

`Set`と`WeakSet`の主な違いは、`Set`はあらゆる値を保存できるのに対し、`WeakSet`はオブジェクトのみを保存できることです。

`Set`と`WeakSet`の他の顕著な違いは以下の通りです。

| 機能 | Set | WeakSet |
| --- | --- | --- |
| 保存する値の種類 | あらゆるデータ型を保存 | オブジェクトのみを保存 |
| 参照の種類 | 強い参照 | 弱い参照 |
| イテレーション | `forEach`やループでの繰り返し処理をサポート | イテレーションをサポートしない |
| メソッドとプロパティ | `add()`、`delete()`、`has()`、`keys()`、`values()`、`size`など多数 | `add()`、`delete()`、`has()`のみ |
| 利用ケース | 一意な値の一般的なコレクションや配列の重複除去 | オブジェクト参照の効率的なメモリ管理 |

この表で、両者が保存できる値の種類、保存されたオブジェクトの繰り返し処理のサポート、理想的な利用ケースの違いがわかります。ぜひ内容をじっくり読んでください。
