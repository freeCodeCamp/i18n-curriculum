# --description--

زيادة أو نقصان رقم واحد هو من أكثر العمليات العددية شيوعًا.

لذلك، هناك مشغلات خاصة لذلك:

- الزيادة `++` تزيد المتغير بمقدار 1:

```js
let counter = 2;

// works the same as counter = counter + 1, but is shorter
counter++;      

console.log(counter); // 3
```

- النقصان `--` ينقص المتغير بمقدار 1:

```js
let counter = 2;

// works the same as counter = counter - 1, but is shorter
counter--;

console.log(counter); // 1
```

يمكن وضع المشغلين `++` و `--` قبل المتغير أو بعده.

- عندما يأتي المشغل بعد المتغير، يكون في "شكل البادئة اللاحقة": `counter++`.

- "شكل البادئة" هو عندما يأتي المشغل قبل المتغير: `++counter`.

هاتان العبارتان تفعلان الشيء نفسه: زيادة `counter` بمقدار `1`.

هل هناك فرق؟ نعم، لكن يمكنك رؤيته فقط إذا استخدمت القيمة المرجعة من `++/--`.

للتوضيح. كما تعلم، كل المشغلات تُرجع قيمة. الزيادة/النقصان ليست استثناء. شكل البادئة يرجع القيمة الجديدة بينما شكل البادئة اللاحقة يرجع القيمة القديمة (قبل الزيادة/النقصان).

لرؤية الفرق، إليك مثال:

```js
let counter = 1;
let a = ++counter; // (*)

console.log(a); // 2
```

لتلخيص:

- إذا لم تُستخدم نتيجة الزيادة/النقصان، فلا فرق في أي شكل تستخدم:

```js
let counter = 0;
counter++;
++counter;

console.log( counter ); // 2, the lines above did the same
```

- إذا أردت زيادة قيمة واستخدام نتيجة المشغل فورًا، تحتاج إلى شكل البادئة:

```js
let counter = 0;
console.log( ++counter ); // 1
```

- إذا أردت زيادة قيمة لكن تستخدم قيمتها السابقة، تحتاج إلى شكل البادئة اللاحقة:

```js 
let counter = 0;
console.log( counter++ ); // 0
```

## --text--

ما هي نواتج عبارات `console.log` الاثنتين في شفرة JavaScript أدناه؟

```js
let counter = 1;
console.log(2 * ++counter); // Statement A
```

```js
let counter = 1;
console.log(2 * counter++); // Statement B
```
