Durante lo sviluppo è importante poter controllare cosa succede nel tuo codice.

Node è semplicemente un ambiente JavaScript. Come per JavaScript lato client, puoi usare la console per mostrare informazioni utili per eseguire il debug. Sul tuo computer locale vedresti l’output della console in un terminale.

Ti consigliamo di tenere il terminale aperto mentre lavori a queste sfide. Leggendo l’output nel terminale puoi vedere eventuali errori che si verificano.

Il server deve essere riavviato dopo aver modificato i suoi file.

Puoi fermare il server dal terminale usando `Ctrl + C` e avviarlo direttamente con Node (`node mainEntryFile.js`) oppure usando uno script di esecuzione nel file `package.json` con `npm run`.

Ad esempio, lo script `"start": "node server.js"` verrebbe eseguito dal terminale usando `npm run start`.

Per implementare il riavvio automatico del server al salvataggio del file, Node fornisce la flag `--watch` che puoi aggiungere allo script di avvio `"start": "node --watch server.js"` oppure puoi installare un pacchetto npm come `nodemon`. Ti lasciamo questa come esercizio.
