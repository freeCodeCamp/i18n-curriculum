# --interactive--

Lernen wir, wann Sie `appearance: none` verwenden sollten, um Styling-Probleme bei Suchfeldern, Kontrollkästchen und Optionsschaltflächen zu beheben.

Browser wenden für viele Elemente eine Standardformatierung an. Bei Eingabeelementen kann es sich so anfühlen, als wäre Ihre Möglichkeit, sie mit CSS zu gestalten, ziemlich eingeschränkt. Deshalb möchten Sie möglicherweise `appearance: none` verwenden, um Aspekte des Standard-Elements auszublenden und stattdessen Ihr eigenes zu erstellen. Zum Beispiel würde dies die Standard-Kontrollkästchen für ein Kontrollkästchen-Eingabefeld ausblenden, sodass Sie benutzerdefinierte Indikatoren wie einen grünen Haken und ein rotes X verwenden können, um den Zustand anzuzeigen. Bei einer Sucheingabe zeigen WebKit-basierte Browser ein Standardsuchsymbol und eine Abbrechen-Schaltfläche an. Das Ausblenden dieser Elemente ermöglicht es Ihnen, eigene Indikatoren zu erstellen, die in allen Browsern angezeigt werden.

WebKit ist eine Software-Engine, die Webbrowsern hilft, Webseiten darzustellen. Browser wie Safari verwenden WebKit, um sicherzustellen, dass Webseiten korrekt aussehen und funktionieren. Diese `appearance: none`-CSS-Eigenschaft gibt Ihnen die vollständige Kontrolle über das Styling, bringt aber auch einige Dinge mit sich, auf die Sie achten müssen. Die standardmäßigen interaktiven Komponenten von Eingabeelementen umfassen Funktionen wie Fokus- und Fehlerindikatoren, die Sie sicherstellen müssen, dass sie nicht verloren gehen.

Eine konsistente plattformübergreifende Gestaltung zu schaffen, ist ein guter Grund, diese Eigenschaft zu verwenden. Sie können sie auch nutzen, um sicherzustellen, dass Tippziele auf einem mobilen Gerät groß genug sind oder dass die Farben eines Kontrollkästchens genügend Kontrast haben.
