# --description--

Нагадаємо, цей проєкт створюється на основі стартового проєкту, скопійованого з <a href="https://github.com/freeCodeCamp/boilerplate-infosec/" target="_blank" rel="noopener noreferrer nofollow">GitHub</a>.

Це завдання висвітлює одну перспективну нову захисну міру, яка може значно знизити ризик і наслідки багатьох типів атак у сучасних браузерах. Встановлюючи та налаштовуючи політику безпеки вмісту, ви можете запобігти впровадженню будь-чого небажаного на вашу сторінку. Це захистить ваш застосунок від вразливостей XSS, небажаного відстеження, шкідливих фреймів і багато іншого. CSP працює, визначаючи список дозволених джерел вмісту, яким довіряють. Ви можете налаштувати їх для кожного типу ресурсу, який може знадобитися вебсторінці (скрипти, таблиці стилів, шрифти, фрейми, медіаконтент тощо). Існує багато директив, тож власник сайту може мати детальний контроль. Детальніше дивіться на HTML 5 Rocks, KeyCDN. На жаль, CSP не підтримується старими браузерами.

За замовчуванням директиви дуже відкриті, тому важливо встановити директиву defaultSrc як запасний варіант. Helmet підтримує обидва стилі іменування defaultSrc і default-src. Запасний варіант застосовується для більшості невказаних директив.

# --instructions--

У цьому завданні використайте `helmet.contentSecurityPolicy()`. Налаштуйте його, додавши об’єкт `directives`. В об’єкті встановіть `defaultSrc` у `["'self'"]` (список дозволених джерел має бути у масиві), щоб за замовчуванням довіряти лише адресі вашого вебсайту. Також встановіть директиву `scriptSrc`, щоб дозволити завантаження скриптів лише з вашого вебсайту (`'self'`) та з домену `'trusted-cdn.com'`.

Підказка: у ключовому слові `'self'` одинарні лапки є частиною самого ключового слова, тому його потрібно взяти в подвійні лапки, щоб воно працювало.

# --hints--

підпрограмне забезпечення helmet.contentSecurityPolicy() має бути правильно підключене

```js
const response = await fetch(code + '/_api/app-info');
if (!response.ok) {
  throw Error(await response.text());
}
const data = await response.json();
assert.include(data.appStack, 'csp');
```

Ваша конфігурація csp неправильна. defaultSrc має бути ["'self'"], а scriptSrc має бути ["'self'", 'trusted-cdn.com']

```js
const response = await fetch(code + '/_api/app-info');
if (!response.ok) {
  throw Error(await response.text());
}
const data = await response.json();
var cspHeader = Object.keys(data.headers).filter(function (k) {
  return (
    k === 'content-security-policy' ||
    k === 'x-webkit-csp' ||
    k === 'x-content-security-policy'
  );
})[0];
assert.equal(
  data.headers[cspHeader],
  "default-src 'self'; script-src 'self' trusted-cdn.com"
);
```
