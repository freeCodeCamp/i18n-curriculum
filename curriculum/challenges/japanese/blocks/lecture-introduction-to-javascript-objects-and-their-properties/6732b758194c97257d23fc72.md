# --interactive--

JavaScriptでは、コンストラクタはオブジェクトを作成し初期化するための特別な種類の関数です。`new` キーワードで呼び出され、新しく作成されたオブジェクトにプロパティやメソッドを初期化できます。

このレッスンでは、`Object()` コンストラクタの使い方を見ていきます。`Object()` コンストラクタは新しい空のオブジェクトを作成します。以下はその例です：

```js
new Object()
```

`new Object()` を呼び出すと、値を格納するために使える新しいオブジェクトが返されます。

`Object()` コンストラクタは `new` キーワードの有無にかかわらず使えます。`new` なしで関数として呼び出された場合、渡された値の型によって動作が異なります。以下は `Object()` コンストラクタを `new` キーワードなしで使った例です：

:::interactive_editor

```js
const num = 42;
const numObj = Object(num); // Creates an object wrapper for the number

console.log(numObj);
console.log(typeof numObj); // "object"
```

:::

2つ目の `console.log` でわかるように、`numObj` はオブジェクトです。これは、数値の入力をオブジェクトに変換するために `Object()` コンストラクタを使ったためです。

`null` や `undefined` を `Object()` コンストラクタに渡すとどうなるでしょうか？

:::interactive_editor

```js
const newObj = new Object(undefined);
console.log(newObj); // {}
```

:::

結果は空のオブジェクトになります。`Object()` コンストラクタのもう一つの使い道は、型が不明な値を扱うときに、それがオブジェクトであることを保証したい場合です。次の例を見てみましょう：

:::interactive_editor

```js
function toObject(value) {
  if (value === null || value === undefined) {
    return {};
  }

  if (typeof value === "object") {
    return value;
  }

  return Object(value);
}

console.log(toObject(null));

console.log(toObject(true));

console.log(toObject([1, 2, 3]));
```

:::

この例では、`toObject` という関数があります。2つ目の条件は値がオブジェクト型かどうかをチェックし、条件が `true` の場合はその値を返します。この条件は配列もチェックします。配列は特別なタイプのオブジェクトだからです。

どちらの条件も真でなければ、関数は `Object(value)` を返します。これは入力をオブジェクトに変換します。これは数値、文字列、ブール値のような値に対して機能します。

ほとんどの場合、新しいオブジェクトを作成するのに `Object()` コンストラクタは使いません。代わりにオブジェクトリテラル構文（例：`const objectLiteral = { name: "Beau" }`）を使うからです。しかし、Object コンストラクタの基本を理解しておくことは役に立ちます。
