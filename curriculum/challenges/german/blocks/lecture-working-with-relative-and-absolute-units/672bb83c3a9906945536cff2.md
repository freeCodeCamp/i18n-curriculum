# --interactive--

In CSS sind `vh` und `vw` viewport-relative Einheiten, mit denen Sie Elemente basierend auf den Abmessungen des Browserfensters skalieren können. Diese Einheiten sind besonders nützlich, um responsives Design zu erstellen, das sich an verschiedene Bildschirmgrößen anpasst.

`vh` steht für „Viewport-Höhe“ und `1vh` entspricht 1 % der Höhe des Viewports.

Ebenso steht `vw` für „Viewport-Breite“ und `1vw` entspricht 1 % der Breite des Viewports.

Das bedeutet, wenn Sie die Höhe eines Elements auf `100vh` setzen, nimmt es die volle Höhe des Viewports ein, unabhängig von den tatsächlichen Pixelmaßen des Geräts.

Diese Einheiten sind besonders praktisch, wenn Sie Vollbild-Layouts oder Elemente erstellen möchten, die ein bestimmtes Verhältnis zum Bildschirm beibehalten.

Zum Beispiel könnten Sie sie verwenden, um einen Hero-Abschnitt zu erstellen, der immer den gesamten Bildschirm ausfüllt (um mit dem Beispiel zu interagieren, müssen Sie den interaktiven Editor aktivieren).

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css" />

<section class="hero">
  <h1>100vh / 100vw Example</h1>
  <p>This section fills the entire browser window.</p>
</section>
```

```css
body {
  margin: 0;
  font-family: sans-serif;
  border: 5px dashed #333;
}

.hero {
  height: 100vh;
  width: 100vw;
  background-color: #add8e6; 
  padding: 2em;
}

.hero h1 {
  font-size: 2em;
  margin-bottom: 0.5em;
}

.hero p {
  font-size: 1em;
}
```

:::

Dieses CSS stellt sicher, dass der Hero-Abschnitt immer genau die Größe des Viewports hat, unabhängig von der Bildschirmgröße des Geräts.

`vh` und `vw` Einheiten können auch für Typografie verwendet werden, um responsive Textgrößen zu erzeugen.

Aktivieren Sie den interaktiven Editor und versuchen Sie, die Größe des Vorschaufensters zu ändern, um zu sehen, wie der Text mit der Viewport-Größe skaliert:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css" />
<h1>Responsive Heading</h1>
<p>The h1 heading scales with the viewport width.</p>
```

```css
h1 {
  font-size: 5vw;
}
```

:::

Dies setzt die Schriftgröße von `h1` Elementen auf 5 % der Viewport-Breite, sodass der Text sich fließend an die Größe des Browserfensters anpasst.

Ein Vorteil von `vh` und `vw` Einheiten ist, dass sie in Echtzeit auf Änderungen der Viewport-Größe reagieren. Das bedeutet, wenn ein Nutzer sein Browserfenster vergrößert oder verkleinert, passen sich die mit diesen Einheiten dimensionierten Elemente automatisch an, ohne dass die Seite neu geladen werden muss. Allerdings ist es wichtig, diese Einheiten mit Bedacht einzusetzen. Wenn Sie zum Beispiel Schriftgrößen ausschließlich mit `vw` Einheiten festlegen, kann der Text auf schmalen Bildschirmen zu klein oder auf breiten Bildschirmen zu groß werden.

Ein weiterer Punkt ist, dass sich auf mobilen Geräten die Viewport-Höhe ändern kann, wenn die Adressleiste des Browsers ein- oder ausgeblendet wird, was unerwartete Layout-Verschiebungen verursachen kann, wenn Sie `vh` Einheiten intensiv verwenden.

Zusammenfassend sind `vh` und `vw` Einheiten leistungsstarke Werkzeuge, um responsive Layouts und Elemente zu erstellen, die sich an die Viewport-Größe anpassen. Sie sind besonders nützlich für Vollbild-Abschnitte, das Beibehalten von Seitenverhältnissen und das Erzeugen von fließend skalierenden Designs. Dennoch sollten sie bedacht und oft in Kombination mit anderen CSS-Techniken eingesetzt werden, um die beste Nutzererfahrung auf allen Geräten zu gewährleisten.
