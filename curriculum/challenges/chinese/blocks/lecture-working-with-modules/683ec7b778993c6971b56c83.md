# --description--

在软件开发中，库就像开发者的工具箱。

库为你提供了预先编写且可重用的代码，如函数、类和数据结构，供你在项目中使用，而无需从头实现代码的每一个部分。

Python 拥有丰富的标准函数库，包含许多不同的内置模块。它们都是针对程序员日常会遇到的许多问题和任务的标准化、经过充分验证的方案，例如：

* 与操作系统交互。
    
* 处理文件。
    
* 网络。
    
* 处理日期和时间。
    
* 执行数学操作。
    
* 使用正则表达式。
    
* 测试和调试你的代码。
    
* 还有更多！
    

一些流行的内置模块示例有 `math`、`random`、`re`（“regular expressions”的缩写）和 `datetime`。

`math` 模块有用于执行更复杂数学操作的帮助函数。

`random` 模块对于生成随机数很有帮助。

`re` 模块用于处理正则表达式。

而且 `datetime` 模块对于在 Python 中处理日期和时间非常有帮助。

但是你如何访问这些内置模块中定义的变量、常量、函数和类？

你使用 `import` 语句。这些语句允许你将模块导入到你的 Python 脚本中。导入语句通常写在文件的顶部。此外，你可以根据你的需求自定义它们。首先，你使用 `import` 语句，后跟模块的名称：

```python
import module_name
```

假设你想导入 `math` 模块。在这种情况下，你会在你的文件顶部写下这段代码：

```python
import math
```

然后，如果你需要在你的 Python 脚本中调用该模块的一个方法，你将使用点表示法，模块名称后跟方法名称：

```python
module_name.method_name() 
```

例如，要获取 36 的平方根，你需要写 `math`，后跟一个点，然后写 `sqrt`，这是平方根的缩略语，并且在括号内传入任何必要的参数。在这种情况下，我们只需传入我们想要计算平方根的数字：

```python
math.sqrt(36)
```

这是最基本版本的 `import` 语句，但还有其他备选。

如果你需要以不同的名称（也称为“别名”）导入模块，可以使用这种语法，在导入语句末尾使用 `as` 后跟别名：

```python
import module_name as module_alias
```

这通常用于缩短冗长的模块名称，或避免命名冲突。

例如，要在你的代码中将 `math` 模块别名为 `m`，你可以为它赋值一个别名，像这样：

```python
import math as m 
```

然后，你可以使用别名访问模块的元素：

```python
m.sqrt(36)
```

但有时你不需要从一个模块中导入所有内容。也许你只需要一两个特定的函数或类。Python 在这种情况下正好满足你的需求。

现在 `import` 语句以 `from` 开头，后跟模块名称，然后是 `import` 关键字，后面跟着你想要导入的元素名称：

```python
from module_name import name1, name2
```

然后，你可以在你的 Python 脚本中使用这些名称而无需模块前缀。

如果你想为这些名称赋值别名，可以在每个名称后使用 `as` 关键字，后面跟上你想使用的别名：

```python
from module_name import name1 as alias1, name2 as alias2
```

假设你只想从 `math` 模块导入 radians、sine 和 cosine 函数。你可以这样写：

```python
from math import radians, sin, cos
```

现在你可以直接在你的代码中调用这些函数，而无需以 `math` 模块作为前缀。

这里有一个更详细的示例：

要查找最初以度数表示的特定角度的正弦和余弦，我们可以调用 `radians` 函数将其转换为弧度，然后调用正弦和余弦函数，传入弧度角：

```python
from math import radians, sin, cos

angle_degrees = 40
angle_radians = radians(angle_degrees)

sine_value = sin(angle_radians)
cos_value = cos(angle_radians)

print(sine_value) # 0.6427876096865393
print(cos_value)  # 0.766044443118978
```

注意我们是直接调用函数，而没有使用模块名称作为前缀。这是因为我们使用了这种备选语法导入了函数。

这很有帮助，但如果你已经在 Python 脚本中定义了同名的函数或变量，可能会导致命名冲突。因此，在选择要使用的导入语句类型时需要注意这一点。

最后，我们查找这个以星号结尾的导入语句。星号告诉 Python 你想导入该模块中的所有内容，但你想导入它，以便不需要使用模块名称作为前缀：

```python
from module_name import *
```

例如，如果你在导入 `math` 模块时这样做，你将能够调用该模块中定义的任何函数，而无需指定模块名称作为前缀。以下是一些示例：

```python
from math import *
print(sqrt(36))  # 6.0
print(pow(5, 2)) # 25.0
print(exp(1))    # 2.718281828459045
```

但是，这通常不被推荐，因为它可能导致命名空间冲突，并且使得更难知道某些名称的来源。

导入语句对于模块中定义的函数、类、常量、变量以及任何其他元素的工作方式完全相同。

这是来自 `math` 模块的一个常量示例，数字 `pi`：

```python
import math
print(math.pi)
```

下面是来自 `datetime` 模块的一个类的示例。我们创建一个表现 1959 年 7 月 15 日的日期对象。然后，我们将该 `date` 对象赋值给一个变量，并使用点符号分别访问日、月和年：

```python
import datetime
birthday = datetime.date(1959, 7, 15)
print(birthday.day)    # 15
print(birthday.month)  # 7
print(birthday.year)   # 1959
```

你可以在该模块的官方 Python 文档中查找有关模块内容的更多信息。

很好。既然你对模块有了更多了解，你也应该知道 Python 脚本中这个非常重要的惯用法，因为它们关系非常密切：

```python
if __name__ == '__main__': 
    # Code
```

`__name__` 是 Python 中一个特殊的内置变量。

当一个 Python 文件被直接执行时，Python 会将这个变量的值设置为字串 `"__main__"`。

但是如果 Python 文件作为模块被导入到另一个 Python 脚本中，`__name__` 变量的值将被设置为该模块的名称（通常是没有 `.py` 扩展名的文件名）。

这就是为什么你经常会在 Python 脚本中看到这个条件式。它包含你只想在 Python 脚本作为主程序运行时才运行的代码：

```python
if __name__ == '__main__': 
    # Code
```

但是如果脚本作为模块被导入，那个块内的代码不会运行。

这很有帮助，因为它允许 Python 脚本有两个用途。它们可以直接运行以执行其主要逻辑，或者可以被导入到另一个模块中而不执行其主要逻辑。
