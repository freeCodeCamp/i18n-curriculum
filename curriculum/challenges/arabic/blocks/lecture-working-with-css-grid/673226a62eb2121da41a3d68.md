# --interactive--

تعرف دالة `minmax()` النطاق لحجم مسار الشبكة، محددة مقدار المساحة التي يمكن أن يشغلها صف أو عمود.

تذكر أنه يمكنك تعيين حجم المسار باستخدام وحدات مثل `px` (بكسل)، `rem`، أو حتى `em`، وأيضًا باستخدام الوحدات الكسرية (`fr`).

تأخذ دالة `minmax()` الأمور خطوة أبعد بالسماح لك بتعيين حجم الحد الأدنى وحجم الحد الأقصى لمسار الشبكة.

إليك صيغة دالة `minmax()`:

```css
minmax(min, max)
```

`min` هو الحد الأدنى لحجم مسار الشبكة، ويمكن تعيينه باستخدام البكسل أو النسبة المئوية أو `auto`. و`max` هو الحد الأقصى لحجم مسار الشبكة الذي يمكنك تعيينه بنفس الوحدات.

تعمل القيمتان معًا بهذه الطريقة:

- تضمن قيمة `min` ألا ينكمش مسار الشبكة أبدًا إلى أقل من حجم محدد.

- تحدد قيمة `max` الحد الأقصى لحجم مسار الشبكة.

يتغير حجم مسار الشبكة ديناميكيًا بين قيمتي `min` و`max` بناءً على المحتوى وحجم الحاوية.

لننظر إلى مثال عملي:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<div class="grid-container">
  <div>
    <h2>Item 1</h2>
  </div>
  <div>
    <h2>Item 2</h2>
  </div>
</div>
```

```css
.grid-container {
  display: grid;
  grid-template-columns: minmax(150px, 300px) 1fr;
  gap: 20px;
}

.grid-container > div {
  background: crimson;
  padding: 20px;
  text-align: center;
}
```

:::

ما الذي يحدث هنا؟

العمود الأول، `minmax(150px, 300px)`, سيكون دائمًا على الأقل `150px` وعلى الأكثر `300px`، حسب المساحة المتاحة.

أما العمود الثاني، `1fr`، فسيشغل أي مساحة متبقية متاحة في حاوية الشبكة لأنه لا توجد أعمدة إضافية لتقاسم المساحة معها.

ميزة دالة `minmax()` مقارنة بالأحجام الثابتة وحتى وحدات `fr` هي أنها أكثر مرونة، مما يجعلها مثالية للتكيف والتجاوب.
