## 디자인 용어

- **레이아웃**: 메시지를 전달하기 위해 페이지나 화면에 시각적 요소를 배치하는 방식입니다. 이 요소에는 텍스트, 이미지, 여백이 포함될 수 있습니다.
- **정렬**: 요소들이 서로 어떻게 배치되는지를 뜻합니다. 정렬을 올바르게 사용하면 디자인이 깔끔하고 정돈된 느낌을 줍니다.
- **구성**: 요소들을 조화롭게 배치하여 디자인을 만드는 행위입니다. 이미지, 텍스트, 도형 같은 요소들이 서로 어떻게 관계를 맺고 예술적으로 디자인에 기여하는지를 결정합니다.
- **균형**: 구성 내에서 시각적 무게가 어떻게 분배되는지를 뜻합니다. 디자이너는 대칭 또는 비대칭 배치를 통해 균형을 이루려고 합니다.
- **비율**: 한 요소의 크기나 치수를 다른 요소와 비교하는 것을 의미합니다.
- **계층 구조**: 디자인 내 요소들의 중요도 순서를 정하는 것입니다. 가장 중요한 정보가 먼저 눈에 띄도록 만드는 것이 목적입니다.
- **대비**: 요소들 간에 명확한 구분을 만드는 과정입니다. 색상, 크기, 형태, 질감 또는 다른 시각적 특성의 변화를 통해 할 수 있습니다. 강한 대비는 가독성 향상에도 도움이 됩니다.
- **여백 (음영 공간)**: 디자인에서 비어 있는 공간입니다. 요소들을 둘러싼 영역을 뜻합니다.
- **UI (사용자 인터페이스)**: 사용자가 화면에서 볼 수 있는 아이콘, 이미지, 텍스트, 메뉴, 링크, 버튼 같은 시각적이고 상호작용 가능한 요소를 포함합니다.
- **UX (사용자 경험)**: 사용자가 제품이나 서비스를 사용할 때 느끼는 경험입니다. 잘 설계된 사용자 경험을 가진 애플리케이션은 직관적이고, 사용하기 쉽고, 효율적이며, 접근 가능하고, 즐거움을 줍니다.
- **디자인 브리프**: 프로젝트의 목표, 목적, 요구사항을 개요로 정리한 문서입니다. 디자인 과정을 안내하고 최종 결과물이 클라이언트의 요구를 충족하도록 돕는 로드맵입니다.
- **벡터 기반 디자인**: 선, 도형, 색상을 수학적 공식으로 정의하여 디지털 아트를 만드는 방식입니다.
- **프로토타이핑**: 제품이나 사용자 인터페이스의 상호작용 모델을 만드는 과정을 뜻합니다.

## UI 디자인 기본

- **배경과 전경 색상의 좋은 대비**: 텍스트를 읽기 쉽게 하려면 배경과 전경 색상이 충분한 대비를 가져야 합니다. 웹 콘텐츠 접근성 지침(WCAG)은 일반 텍스트에 대해 최소 명암 대비율 4.5:1, 큰 텍스트에 대해 3:1을 권장합니다.
- **좋은 시각적 계층 구조**: 강력한 시각적 계층 구조는 눈이 따라가야 할 명확한 경로를 제공하여 전달하려는 정보를 의도한 순서대로 소비하게 합니다.
- **반응형 이미지**: 반응형 이미지는 화면 크기에 맞게 크기가 조절되는 이미지입니다. 데스크톱부터 모바일까지 모든 기기에서 이미지가 잘 보이도록 하는 데 중요합니다.
- **점진적 향상**: 모든 사용자, 브라우저나 기기에 관계없이 애플리케이션의 필수 콘텐츠와 기능에 접근할 수 있도록 하는 디자인 접근법입니다.
- **사용자 중심 디자인**: 최종 사용자의 요구, 선호, 한계를 우선시하는 접근법입니다. 사용자 중심 디자인의 목표는 직관적이고 효율적이며 사용자가 상호작용하기에 즐거운 웹 페이지를 만드는 것입니다.
- **사용자 조사**: 제품을 사용하는 사람들을 체계적으로 연구하는 것입니다. 사용자 요구, 행동, 문제점을 측정하는 것이 목표입니다.
- **퇴사 인터뷰**: 사용자가 계정을 취소할 때 제공하는 설문조사입니다. 사용자가 떠나는 이유를 이해하고 이탈률을 줄이기 위한 방안을 찾는 데 도움이 됩니다.
- **사용자 테스트**: 사용자가 애플리케이션과 상호작용하는 동안 데이터를 수집하는 실습입니다.
- **A/B 테스트**: 새로운 기능을 무작위로 선택된 사용자 부분집합에 배포하는 과정입니다. 분석 데이터를 활용해 기능이 유용한지 판단할 수 있습니다.
- **사용자 요구사항**: 애플리케이션이 따라야 하는 스토리나 기준을 뜻합니다. 사용자 조사, 업계 표준, 이해관계자 의견으로 정의될 수 있습니다.
- **점진적 공개**: 사용자의 현재 활동에 따라 관련 콘텐츠만 보여주고 나머지는 숨기는 디자인 패턴입니다. 인지 부하를 줄이고 사용자 경험을 더 직관적으로 만듭니다.
- **지연/레이지 등록**: 사용자가 등록하지 않고도 애플리케이션을 탐색하고 상호작용할 수 있게 하는 UI 디자인 패턴입니다.

## 디자인 모범 사례

- **다크 모드**: 웹 애플리케이션에서 기본 밝은 색상 체계를 어두운 색상 체계로 변경할 수 있는 특별 기능입니다. 다크 모드에서는 채도가 낮은 색상을 사용해야 합니다. 채도가 낮은 색상은 강도가 약하고 채도 수준이 낮은 색상입니다.
- **브레드크럼**: 사용자가 사이트 계층 구조에서 현재 위치를 보여주는 내비게이션 도구입니다. 페이지 상단에 배치해 사용자가 쉽게 찾을 수 있도록 하는 것이 좋습니다. 또한 브레드크럼은 읽기 쉽도록 충분히 크되 페이지 공간을 너무 많이 차지하지 않도록 해야 합니다.
- **카드 컴포넌트**: 카드 컴포넌트는 디자인이 단순하고 시각적으로 복잡하지 않으며 너무 많은 정보를 표시하지 않아야 합니다. 미디어의 경우 고품질 이미지와 비디오를 선택해 사용자 경험을 향상시키세요.
- **무한 스크롤**: 사용자가 페이지를 아래로 스크롤할 때 더 많은 콘텐츠를 불러오는 디자인 패턴입니다. 사용자가 더 많은 콘텐츠를 보고 싶을 때 직접 선택할 수 있도록 ‘더 보기’ 버튼을 사용하는 것을 고려하세요. 또한 사용자가 이전 페이지로 쉽게 돌아갈 수 있도록 뒤로 가기 버튼을 추가할 수도 있습니다.
- **모달 대화상자**: 기존 페이지 콘텐츠 위에 표시되는 팝업 유형입니다. 보통 배경 콘텐츠에 어두운 색 오버레이를 씌워 사용자가 모달 콘텐츠에 더 집중할 수 있게 합니다. 모달 외부를 클릭해 닫을 수 있도록 하는 것도 좋은 방법입니다. HTML `dialog` 요소를 사용하면 많은 기능과 접근성 이점을 내장으로 얻을 수 있습니다.
- **폼 등록 진행 표시**: 사용자가 프로세스 진행 상황을 알 수 있게 하는 방법입니다. 폼, 등록, 설정 과정에서 사용할 수 있습니다. 디자인은 단순하고 찾기 쉬우며 이전 단계로 돌아갈 수 있어야 합니다.
- **쇼핑 카트**: 전자상거래 플랫폼에서 사용자가 이미 선택한 항목을 볼 수 있는 공간입니다. 카트는 항상 사용자에게 보여야 하며, 카트, 가방, 바구니 같은 일반적인 아이콘을 사용하고, 결제로 진행할 수 있는 명확한 행동 유도 버튼이 있어야 합니다.

## 일반 디자인 도구

- **Figma**: 클라우드 기반 도구로 UI/UX 디자인에 특화되어 있습니다. 디자인과 개발 팀이 어디서나 협업할 수 있게 하며, 벡터 기반 디자인, 자동 레이아웃, 주석 및 피드백 시스템 등 내장 기능을 제공합니다.
- **Sketch**: 직관적인 인터페이스와 단순함으로 인기가 높은 디자인 도구로, 빠르게 프로토타입을 만들고자 하는 개발자에게 적합합니다. UI, 아이콘, 웹 레이아웃 제작에도 디자이너들이 널리 사용합니다.
- **Adobe XD**: UI/UX 디자인을 위한 벡터 기반 디자인 및 프로토타이핑 도구로, Photoshop, Illustrator, After Effects 같은 다른 Adobe 앱과 원활하게 통합됩니다.
- **Canva**: 포스터, 커버 사진, 프레젠테이션, 짧은 동영상 등 다양한 시각 콘텐츠를 만들 수 있는 도구입니다. 사용자 친화적이고 단순한 디자인으로 초보자에게 적합합니다.
