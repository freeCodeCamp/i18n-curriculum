# --description--

Em uma lição anterior, você aprendeu o que são dependências. Aqui, você aprenderá mais sobre elas e coisas a considerar ao escolher um pacote externo para usar no seu projeto.

Uma dependência é outro pedaço de código que seu projeto precisa para funcionar. Normalmente, são pacotes externos que outros desenvolvedores criaram e compartilharam no registro npm.

Usar dependências economiza muito tempo. Em vez de construir tudo você mesmo, você pode instalar um pacote que já resolve um problema comum — coisas como criar um servidor web, lidar com roteamento, buscar dados, gerenciar autenticação e mais.

No seu arquivo `package.json`, todos os pacotes dos quais seu projeto depende estão listados sob a chave `dependencies`:

```json
{
  "name": "package-dependencies",
  "version": "1.0.0",
  "description": "a demo on how package dependencies work",
  "main": "index.js",
  "scripts": {
    "test": "echo \"Error: no test specified\" && exit 1"
  },
  "keywords": [
    "Node",
    "NPM"
  ],
  "author": "Kolade Chris",
  "license": "ISC",
  "dependencies": {
    "chalk": "^5.6.2",
    "lodash": "^4.17.21"
  }
}
```

No exemplo do arquivo `package.json` acima, `chalk` e `lodash` estão listados como dependências do projeto. Isso significa que o projeto depende desses pacotes para funcionar. Se eles não estiverem instalados e listados no arquivo `package.json`, quaisquer funcionalidades que os utilizem irão falhar.

Também existe outra categoria de dependências chamada `devDependencies`. São pacotes que você precisa apenas durante a construção ou teste do seu projeto, não quando ele está em execução na produção. Exemplos comuns incluem:

* Nodemon para reiniciar automaticamente o projeto quando houver atualizações
    
* Jest ou Mocha para testes
    
* ESLint para linting de código
    

Como você esperaria, `devDependencies` aparecem na seção `devDependencies` do arquivo `package.json`:

```json
{
  "name": "package-dependencies",
  "version": "1.0.0",
  "description": "a demo on how package dependencies work",
  "main": "index.js",
  "scripts": {
    "test": "echo \"Error: no test specified\" && exit 1"
  },
  "keywords": [
    "Node",
    "NPM"
  ],
  "author": "Kolade Chris",
  "license": "ISC",
  "dependencies": {
    "chalk": "^5.6.2",
    "lodash": "^4.17.21"
  },
  "devDependencies": {
    "jest": "^30.2.0",
    "nodemon": "^3.1.10"
  }
}
```

Ao escolher quais pacotes usar no seu projeto, há algumas coisas importantes a serem lembradas.

A segurança deve sempre vir em primeiro lugar. Mesmo um único pacote inseguro pode criar problemas para todo o seu projeto. Você também deve considerar o desempenho porque um pacote grande ou mal otimizado pode deixar seu projeto mais lento.

Também ajuda escolher pacotes que são populares e bem suportados. Pacotes populares geralmente têm comunidades ativas, boa documentação e atualizações frequentes. Uma maneira fácil de verificar a popularidade é olhar o número de downloads semanais no npm. Por exemplo, o pacote `chalk` recebe mais de 300.000 downloads a cada semana:

<img src="https://cdn.freecodecamp.org/curriculum/lecture-transcripts/what-are-package-dependencies-and-what-are-some-considerations-for-choosing-an-external-package-1.webp" alt="screenshot from npm of chalk package statistics">

A manutenção também é importante. Pacotes bem mantidos são atualizados regularmente e funcionam melhor com novas versões de outros softwares.

Por fim, não negligencie a documentação. Documentação clara e completa facilita muito o entendimento de como usar um pacote e economiza muito tempo ao integrá-lo ao seu projeto.
