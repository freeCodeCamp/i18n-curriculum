# --description--

`/public/logo.png`, `./script.js`, 또는 `../styles.css` 같은 링크를 본 적이 있을 것입니다. 그런데 이런 특별한 유형의 링크는 무엇을 의미할까요? 이것들은 파일 경로라고 합니다. 알아야 할 세 가지 주요 구문이 있습니다. 첫 번째는 슬래시로, 운영 체제에 따라 역슬래시(`\`) 또는 정방향 슬래시(`/`)일 수 있습니다. 두 번째는 한 점(`.`)입니다. 마지막으로 두 점(`..`)이 있습니다.

슬래시는 "경로 구분자"로 알려져 있습니다. 폴더나 파일 이름 사이의 구분을 나타내는 데 사용합니다. 이렇게 해서 컴퓨터는 `naomis-files/`가 같은 이름의 디렉터리를 가리키고, `naomis/files/`는 `files` 디렉터리 안의 `naomis` 디렉터리를 가리킨다는 것을 알 수 있습니다.

한 점은 현재 디렉터리를 가리키고, 두 점은 상위 디렉터리를 가리킵니다. 일반적으로 한 점은 경로가 상대 경로임을 인식시키기 위해 사용됩니다. 이전 수업에서 상대 경로와 절대 경로에 대해 배운 것을 기억하세요.

하지만 두 점은 현재 작업 디렉터리 밖의 파일에 접근할 때 훨씬 더 자주 사용됩니다.

예를 들어, 다음과 같은 파일 트리가 있다고 가정해 봅시다:

```sh
my-app/
├─ public/
│  ├─ favicon.ico
│  ├─ index.html
├─ src/
│  ├─ index.css
│  ├─ index.js
```

만약 `public/index.html` 파일이 `favicon.ico` 파일을 불러와야 한다면, 현재 디렉터리에 접근하기 위해 한 점을 사용한 상대 경로 `./favicon.ico`을 사용합니다. `public/index.html` 파일이 `index.css` 파일을 불러와야 한다면, 먼저 상위 `my-app` 디렉터리로 이동한 다음 `src` 디렉터리로 가서 마지막으로 파일에 접근하는 두 점을 사용한 상대 경로 `../src/index.css`을 사용합니다.
