# --description--

依存関係を最新の状態に保ち、特定のバージョンをインストールする方法を知ることは、プロジェクトの安定性を維持するために重要です。これにより、予期しないバグを避けられ、新しいバージョンがリリースされたときにより細かくコントロールできます。

パッケージの特定のバージョンをインストールするには、次の形式を使います。

```bash
npm install package@version-number
```

たとえば、プロジェクトで古いバージョンのExpress、例えば`4.21.2`が必要な場合は、次のようにインストールできます。

```bash
npm install express@4.21.2
```

バージョン番号を指定しない場合、npmは自動的に利用可能な最新バージョンをインストールします。

どのバージョンが存在するかわからない場合は、次のコマンドで確認できます。

```bash
npm view package-name versions
```

たとえば：

```bash
npm view express versions
```

パッケージを更新するには、個別に更新するか一括で更新できます。その前に、どのパッケージが古くなっているかを確認すると便利です。

```bash
npm outdated
```

これにより、どの依存関係に新しいバージョンがあるかが表示され、次のような出力が得られます。

```bash
user@Kolade ~/Desktop/fCC/script-code/node/npm/install-spec-version  % npm outdated
Package  Current  Wanted  Latest  Location              Depended by
chalk      5.5.0   5.5.0   5.6.2  node_modules/chalk    install-spec-version
cleave     1.0.1   1.0.1   1.0.2  node_modules/cleave   install-spec-version
express   4.21.2  4.21.2   5.2.1  node_modules/express  install-spec-version
```

古くなっている依存関係を確認したら、更新を始められます。

単一の依存関係を更新するには、次のコマンドを使います。

```bash
npm update package-name
```

これは、`package.json`ファイルで許可されているバージョン範囲内でのみパッケージを更新します。たとえば、Expressを更新するには次のようにします。

```bash
npm update express
```

バージョン範囲を無視してパッケージの最新バージョンをインストールしたい場合は、次のコマンドを使います。

```bash
npm install package-name@latest
```

許可されたバージョン範囲内のすべての依存関係を更新するには、次を実行します。

```bash
npm update
```

ただし、`npm update`は、`package.json`で許可されていない場合、新しいメジャーバージョンにはアップグレードしません。メジャーバージョンは破壊的変更を含むことが多いため、npmは自動的に適用しません。

最新のメジャーバージョンを特にインストールしたい場合は、次を実行する必要があります。

```bash
npm install package-name@latest
```

これにより、`package.json`のバージョン範囲に関係なく、npmは最新のリリースを強制的にインストールします。
