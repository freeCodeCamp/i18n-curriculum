# --description--

Das Event-Handling ist ein wesentlicher Bestandteil jeder interaktiven Website.

React bietet eine leistungsstarke und konsistente Möglichkeit, Ereignisse über sein Synthetic Event System zu handhaben, das eine Hülle um native Ereignisse wie `click`, `keydown` und `submit` ist, die Sie in früheren Lektionen kennengelernt haben.

Diese browserübergreifende Hülle stellt sicher, dass Ereignisse in allen Browsern gleich funktionieren, sodass es keine Inkonsistenzen gibt.

Sehen wir uns an, wie Ereignisse in React funktionieren, damit Sie sie in Ihren Projekten verwenden können.

In React funktionieren Event-Handler ähnlich wie native Browser-Ereignisse, aber mit einigen Anpassungen.

Anstelle von Event-Attributnamen in Kleinbuchstaben wie `onclick` und `onsubmit` verwendet React CamelCase, wie `onClick` und `onSubmit`.

Außerdem erwartet React anstelle von Strings zur Angabe der Art des Ereignisses eine Funktion als Event-Handler.

Die Event-Handler-Funktion wird als Prop an das Element übergeben, und der Ereignistyp wie `onClick` oder `onSubmit` wird als Attribut in JSX verwendet.

Hier eine Erinnerung daran, wie man mit einem Klick-Ereignis in normalem HTML arbeitet:

```html
<button onclick="alert('Button clicked!')">Click Me</button>
```

Und so machen Sie dasselbe in React:

```jsx
function handleClick() {
  console.log("Button clicked!");
}

<button onClick={handleClick}>Click Me</button>;
```

In diesem Beispiel protokolliert `handleClick` eine Nachricht in der Konsole, wenn der Benutzer auf die Schaltfläche klickt.

Beachten Sie, dass Sie nach `handleClick` im `onClick`-Attribut keine runden Klammern benötigen, da Sie eine Referenz auf die Funktion übergeben und sie nicht aufrufen.

In React beginnen Event-Handler-Funktionen normalerweise mit dem Präfix `handle`, um anzuzeigen, dass sie für das Handhaben von Ereignissen zuständig sind, wie `handleClick` oder `handleSubmit`.

Wenn eine Benutzeraktion ein Ereignis auslöst, übergibt React ein Synthetic Event-Objekt an Ihren Handler. Dieses Objekt verhält sich ähnlich wie das native Ereignisobjekt in Vanilla JavaScript und bietet Eigenschaften wie `type`, `target` und `currentTarget`.

Sie können `event` als Parameter an die Handler-Funktion übergeben und es in der Konsole protokollieren, um sich seine Struktur anzusehen:

```js
function handleClick(event) {
  console.log(event);
}
```

Um Standardverhalten wie das Aktualisieren des Browsers während eines `onSubmit`-Ereignisses zu verhindern, können Sie beispielsweise die Methode `preventDefault()` aufrufen:

```jsx
function handleSubmit(event) {
  event.preventDefault();
  console.log("Form submitted!");
}

<form onSubmit={handleSubmit}>
  <input type="text" />
  <button>Submit</button>
</form>;
```

Sie können auch verhindern, dass ein Ereignis zu übergeordneten Elementen hochblubbert, indem Sie `event.stopPropagation()` aufrufen.

Manchmal möchten Sie beim Handhaben spezieller Fälle wie Löschen- und Bearbeitungsfunktionen zusätzliche Daten an einen Event-Handler übergeben. Das können Sie tun, indem Sie den Handler in einer inline Pfeilfunktion verpacken:

```jsx
function handleDelete(id) {
  console.log("Deleting item:", id);
}

<button onClick={() => handleDelete(1)}>Delete Item</button>;
```

Es ist in Ordnung, inline Event-Handler in React zu verwenden, da React das erneute Rendern effizient verwaltet und Leistungsprobleme vermeidet.

In Vanilla JavaScript können inline Event-Handler zu Leistungsproblemen führen, weil bei jedem Rendern neue Funktionen erstellt werden, da es keinen virtuellen DOM gibt, der den Prozess optimiert.
