# --description--

`Promise`는 비동기 작업의 최종 완료 또는 실패를 나타내는 객체입니다. 처음에는 대기 상태에 있습니다. 작업이 성공하면 이행 상태로, 실패하면 거부 상태로 전환할 수 있습니다. 다음은 `Promise`를 생성하는 예시입니다:

```js
const aPromise = new Promise((resolve, reject) => {
  setTimeout(() => {
    resolve("Operation successful!");
  }, 1000);
});
```

이 예시에서는 `setTimeout`를 사용해 비동기 작업을 시뮬레이션하는 프로미스를 만듭니다. 1초 후에 프로미스는 `Operation successful!` 메시지와 함께 해결됩니다.

프로미스를 다루는 또 다른 방법은 `.then`와 `.catch` 메서드를 사용하는 것입니다.

`.then()` 메서드는 프로미스가 이행되었을 때 실행할 작업을 지정하는 데 사용되고, `.catch()`는 발생한 오류를 처리하는 데 사용됩니다. 프로미스와 함께 `.then`와 `.catch`를 어떻게 사용하는지 살펴보겠습니다:

```js
aPromise.then((result) => {
  console.log(result);  // Outputs: "Operation successful!"
}).catch((error) => {
  console.error(error);
});
```

이 코드에서 프로미스가 이행되었을 때 실행할 명령어 또는 함수가 `.then()`에 전달된 함수로 호출됩니다. 오류가 발생하면 `.catch()`에 전달된 함수가 대신 호출됩니다.

이제 프로미스 체이닝에 대해 이야기해 보겠습니다. 프로미스의 강력한 기능 중 하나는 여러 비동기 작업을 연속으로 연결할 수 있다는 점입니다. 각 `.then()`는 새로운 프로미스를 반환할 수 있어, 비동기 작업 수열을 순차적으로 수행할 수 있습니다. 다음은 예시입니다:

```js
fetch('https://api.example.com/data')
  .then(response => response.json())
  .then(data => {
    console.log(data);
    return fetch('https://api.example.com/data2');
  })
  .then(response => response.json())
  .then(data2 => console.log(data2))
  .catch(error => console.error('Error:', error));
```

이 예시에서는 두 개의 API 호출을 순서대로 수행합니다. 첫 번째 `.then()`는 응답을 JSON으로 파싱합니다. 두 번째 `.then()`는 데이터를 기록하고 또 다른 API 호출을 합니다. 세 번째 `.then()`는 두 번째 API 호출의 응답을 파싱하고, 네 번째 `.then()`는 그 데이터를 기록합니다. 이 체인 어디에서든 오류가 발생하면 마지막의 `.catch()`가 이를 잡아냅니다.

중요한 점은 `.catch()`가 체인의 이전 단계들에서 발생한 오류를 모두 잡아낸다는 것입니다. 따라서 각 단계마다 오류 처리를 추가할 필요가 없어 코드를 훨씬 간단하게 만들 수 있습니다.
