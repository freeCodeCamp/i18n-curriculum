# --description--

In den vorherigen Lektionen haben wir gelernt, wie man kleine Komponenten in React wie diese erstellt:

```jsx
function Greeting() {
  const developerName = "Jessica";
  return <h1>Hi {developerName}!</h1>;
}
```

Wir können diese Komponente in eine andere Elternkomponente oder die Root-Komponente verschachteln, wie hier:

```jsx
function App() {
  return <Greeting />;
}

function Greeting() {
  const developerName = "Jessica";
  return <h1>Hi {developerName}!</h1>;
}
```

Obwohl dieser Code problemlos ausgeführt wird und das Ergebnis von `Hi Jessica!` auf dem Bildschirm anzeigt, ist die Komponente nicht sehr flexibel.

Was, wenn wir einen anderen Namen wie Naomi, Tom oder Oliver anzeigen möchten? Hier kommen React-Props ins Spiel. Props, kurz für Eigenschaften, sind der Weg, wie Elternkomponenten Daten an die Kindkomponente weitergeben. Props können jeden Typ haben: Strings, Zahlen, Booleans, Objekte oder Arrays.

Lassen Sie uns unser vorheriges Beispiel aktualisieren, damit es jetzt eine `name`-Prop akzeptiert:

```jsx
function App() {
  return <Greeting name="Jessica" />;
}

export default App;

function Greeting(props) {
  console.log(props);
  return <h1>Hi {props.name}!</h1>;
}
```

Für die Kindkomponente namens `Greeting` verwenden wir jetzt `props.name` anstelle eines fest codierten Namens `"Jessica"`. Außerdem protokollieren wir `props` in der Konsole, das als Objekt angezeigt wird.

Dann übergeben wir in der Elternkomponente `App` den Wert an die `name`-Prop, damit sie an das Kind weitergegeben werden kann. Das Ergebnis auf dem Bildschirm ist wie zuvor, aber jetzt haben wir eine flexiblere Komponente erstellt.

Jetzt können wir die Kindkomponente mehrfach wiederverwenden und jedes Mal unterschiedliche Namen übergeben:

```jsx
function App() {
  return (
    <>
      <Greeting name="Naomi" />
      <Greeting name="Tom" />
      <Greeting name="Jessica" />
      <Greeting name="Oliver" />
    </>
  );
}
```

Sie können auch die Destrukturierung der Props verwenden, um den Code lesbarer zu machen. So könnten Sie die `Greeting`-Komponente umschreiben:

```jsx
function Greeting({ name }) {
  return <h1>Hi {name}!</h1>;
}
```

Dieser Code erzielt dasselbe Ergebnis, macht aber klarer, welche Props die Komponente erwartet.

React bietet außerdem eine spezielle Prop namens `children`. Jeder JSX-Inhalt, den Sie zwischen den öffnenden und schließenden Tags einer Komponente platzieren, wird als `children` an die Komponente übergeben.

Zum Beispiel:

```jsx
function Card({ children }) {
  return <div className="card">{children}</div>;
}

function App() {
  return (
    <Card>
      <h2>Hello</h2>
    </Card>
  );
}
```

In diesem Fall wird der JSX-Inhalt innerhalb von `<Card>...</Card>` zur `children`-Prop. Das gerenderte Ergebnis im DOM sieht so aus:

```html
<div class="card">
  <h2>Hello</h2>
</div>
```

Dieses Muster wird oft für die Kompositionsstruktur von Komponenten verwendet, bei der eine Komponente andere UI-Elemente umschließt.

Manchmal gibt es viele Eigenschaften, die Sie als Props übergeben müssen. Anstatt sie einzeln zu übergeben, können Sie den Spread-Operator (`...`) verwenden, nachdem Sie sie in ein Objekt umgewandelt haben.

Hier ist ein Beispiel für eine neue Kindkomponente namens `DeveloperCard`:

```jsx
function DeveloperCard({ name, age, country }) {
  return (
    <div className="developer-card">
      <h1>Developer: {name}</h1>
      <p>Age: {age}</p>
      <p>Country: {country}</p>
    </div>
  );
}
```

Diese `DeveloperCard`-Komponente akzeptiert drei Props: `name`, `age` und `country`.

In der Elternkomponente `App` können wir die Spread-Syntax verwenden, um alle Eigenschaften eines Objekts als einzelne Props an die Kindkomponente zu übergeben:

```jsx
function App() {
  const developerObj = {
    name: "Alice",
    age: 30,
    country: "USA",
  };

  return (
    <div className="App">
      <DeveloperCard {...developerObj} />
    </div>
  );
}
```

Das ist besonders nützlich, wenn Sie mit Arrays von Objekten arbeiten und mehrere Eigenschaftensätze an Kindkomponenten übergeben. Zum Beispiel könnten Sie eine Liste von Entwicklern haben, bei der jedes Objekt im Array dieselbe Struktur hat, aber eine andere Person darstellt.

Sie werden in zukünftigen Lektionen mehr darüber lernen, wie man Listen in Arrays rendert.

Die Verwendung von Props in React macht Ihre Komponenten flexibler und wiederverwendbarer, sodass Sie komplexere Benutzeroberflächen erstellen können. Es ist jedoch wichtig zu beachten, dass Props unveränderlich sind, das heißt, sie können nach der Übergabe an eine Komponente nicht geändert werden. Wenn Sie Benutzereingaben verarbeiten und Daten ändern müssen, sollten Sie stattdessen State verwenden. Mehr zum Verwalten von State lernen Sie in zukünftigen Lektionen.
