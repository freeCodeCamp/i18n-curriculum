## Cos’è npm?

- **Definizione**: npm è un archivio online per software open source, che funziona come un “app store” per il codice.
- **Significato ufficiale**: npm non sta ufficialmente per “Node Package Manager”; è un’abbreviazione ricorsiva che significa “npm non è un acronimo”.
- **Tre parti correlate**:

  - **Sito web (npmjs.com)**: Per consultare i pacchetti nell’archivio.
  - **Registry**: Un grande database pubblico che conserva tutti i pacchetti.
  - **CLI (interfaccia a riga di comando)**: Uno strumento installato con Node.js per gestire le dipendenze del progetto.

- **Alternative**: Anche se npm è il più usato, alternative popolari includono **Yarn**, **PNPM** e **Bun**.

## Il file package.json

- **Scopo**: Un file di configurazione che conserva metadati, dipendenze e script per un progetto Node.js.
- **Campi chiave**:

  - `name` e `version`: Identificano il progetto.
  - `main`: Specifica il file di ingresso, spesso `index.js`.
  - `license`: Definisce i termini legali (predefinito è `ISC`).
  - `dependencies`: Pacchetti necessari per eseguire l’app in produzione.
  - `devDependencies`: Strumenti necessari solo durante lo sviluppo, come framework di test.

- **Creazione**:

  - `npm init`: Avvia un processo di configurazione interattivo.
  - `npm init -y`: Installa un `package.json` con valori predefiniti all’istante.

## Versionamento semantico (SemVer)

- **Formato**: `MAJOR.MINOR.PATCH`.
- **Regole**:

  - **MAJOR**: Cambiamenti incompatibili che possono influenzare il codice esistente.
  - **MINOR**: Nuove funzionalità compatibili con le versioni precedenti.
  - **PATCH**: Piccole correzioni di bug e miglioramenti.

- **Simboli di intervallo**:

  - `^` (accento circonflesso): Permette aggiornamenti minori e patch (es. `^1.2.3` permette `1.x.x`).
  - `~` (tilde): Permette solo aggiornamenti di patch (es. `~1.2.3` permette `1.2.x`).
  - `*` (asterisco): Corrisponde a qualsiasi versione; usato principalmente per test.

## Gestione delle dipendenze

- **Installazione**:

  - `npm install <name>`: Aggiunge una dipendenza di produzione.
  - `npm install <name> -D`: Aggiunge una dipendenza di sviluppo.
  - `npm install <name>@<version>`: Installa una versione specifica.

- **Node modules e lock**:

  - `node_modules/`: Contiene il codice effettivo dei pacchetti installati.
  - `package-lock.json`: Blocca le versioni esatte di tutte le dipendenze e delle “dipendenze figlie” per garantire coerenza tra ambienti.

- **Rimozione**: `npm uninstall <name>` rimuove il pacchetto e aggiorna `package.json`.
- **Manutenzione**:

  - `npm outdated`: Identifica quali pacchetti hanno versioni più recenti disponibili.
  - `npm update`: Aggiorna i pacchetti entro l’intervallo consentito dal versionamento semantico.

## Script npm

- **Funzione**: Comandi personalizzati definiti nella sezione `scripts` di `package.json` per automatizzare attività.
- **Esecuzione**: Si eseguono tramite `npm run <script_name>`.
- Esistono abbreviazioni per `start`, `test`, `stop` e `restart` (es. `npm start`).
- **Esecuzione sequenziale**:
- `&&`: Esegue il secondo comando solo se il primo ha successo.
- `;`: Esegue i comandi uno dopo l’altro indipendentemente dal successo.
- **Esecuzione concorrente**: Usa l’operatore `&` per eseguire processi in background.
- **Argomenti**: Passa argomenti agli script usando il separatore `--` (es. `npm start -- 8000`).

## Pubblicare nel registry npm

- **Accesso al registry**: Un database di pacchetti pubblici e privati.
- **Passaggi per la pubblicazione**:

1. Crea un account gratuito su `npmjs.com`.
2. Accedi tramite terminale usando `npm login`.
3. Assicurati che `name` sia unico e `version` valido in `package.json`.
4. Crea un `README.md` e un `.npmignore` (per escludere file sensibili).
5. Esegui `npm publish` (o `npm publish --access public` per pacchetti con scope).

- **Scope**: Usati per evitare conflitti di nome pubblicando sotto `@username/package-name`.

## Sistemi di moduli in Node.js

### CommonJS (CJS)

- **Sistema predefinito**: Il modo originale in cui Node.js gestisce i moduli; carica in modo sincrono.
- **Sintassi**:
- **Importare**: `const module = require('./path')`.
- **Esportare**: `module.exports = { functionName }`.
- **Identificazione**: Usato per file `.cjs` o file `.js` quando non è specificato `type` in `package.json`.

### ES Modules (ESM)

- **Formato standardizzato**: Standard moderno di JavaScript; carica in modo asincrono per efficienza.
- **Sintassi**:

  - **Esportare**: `export { name }` o `export default function`.
  - **Importare**: `import { name } from './path.mjs'`.
  - **Identificazione**: Usato per file `.mjs` o file `.js` quando `package.json` contiene `"type": "module"`.
