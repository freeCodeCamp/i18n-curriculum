# --description--

- تُنفذ **جافاسكريبت المتزامن** بشكل متسلسل وتنتظر انتهاء العملية السابقة قبل الانتقال إلى التالية.
- تسمح **جافاسكريبت غير المتزامن** بتنفيذ عدة عمليات في الخلفية دون حجب الخيط الرئيسي.
- **الخيط** هو متتالية من التعليمات يمكن تنفيذها بشكل مستقل عن تدفق البرنامج الرئيسي.
- **دوال الاستدعاء الراجع** هي دوال تُمرر كمعلمات إلى دوال أخرى وتُنفذ بعد إتمام العملية أو نتيجة لحدث.

## محرك جافاسكريبت وبيئة تشغيل جافاسكريبت

- **محرك جافاسكريبت** هو برنامج ينفذ شفرة جافاسكريبت في متصفح الويب. يعمل كمحول يأخذ شفرتك، ويحوّلها إلى تعليمات يفهمها الكمبيوتر وينفذها.
- V8 مثال على محرك جافاسكريبت طورته Google.
- **بيئة تشغيل جافاسكريبت** هي البيئة التي تُنفذ فيها شفرة جافاسكريبت. تشمل محرك جافاسكريبت الذي يعالج وينفذ الشفرة، وميزات إضافية مثل متصفح الويب أو Node.js.

## واجهة برمجة تطبيقات Fetch

- تتيح واجهة برمجة تطبيقات Fetch لتطبيقات الويب إجراء طلبات شبكة، عادة لاسترجاع أو إرسال بيانات إلى الخادم. توفر طريقة `fetch()` يمكنك استخدامها لإجراء هذه الطلبات.
- يمكنك استرجاع نصوص، صور، صوت، JSON، وأنواع أخرى من البيانات باستخدام Fetch API.

## طرق HTTP لواجهة Fetch API

تدعم Fetch API عدة طرق HTTP للتفاعل مع الخادم. أكثر الطرق شيوعًا هي:

- **GET**: تُستخدم لاسترجاع البيانات من الخادم. افتراضيًا، تستخدم Fetch API طريقة `GET` لاسترجاع البيانات.

```js
fetch('https://api.example.com/data')
```

لاستخدام البيانات المسترجعة، يجب تحويلها إلى صيغة JSON باستخدام طريقة `.json()`:

```js
fetch('https://api.example.com/data')
  .then(response => response.json())
  .then(data => console.log(data))
```

في هذه الشفرة، تُعيد `fetch()` وعدًا، ويحوّل معالج `.then()` الاستجابة المحلولة إلى صيغة JSON.

- **POST**: تُستخدم لإرسال بيانات إلى الخادم. تُستخدم طريقة `POST` لإنشاء موارد جديدة على الخادم.

```js
fetch('https://api.example.com/users', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({
    name: 'John Doe',
    email: 'john@example.com'
  })
})
```

في هذا المثال، نرسل طلب `POST` لإنشاء مستخدم جديد. حددنا الطريقة كـ `POST`، وضبطنا الرؤوس المناسبة، وأدرجنا جسمًا يحتوي على البيانات التي نريد إرسالها. يجب أن يكون الجسم سلسلة نصية، لذا نستخدم `JSON.stringify()` لتحويل كائننا إلى سلسلة JSON.

- **PUT**: تُستخدم لتحديث البيانات على الخادم. تُستخدم طريقة `PUT` لتحديث الموارد الموجودة على الخادم.

```js
fetch('https://api.example.com/users/45', {
  method: 'PUT',
  headers: {
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({
    name: 'John Smith',
    email: 'john@example.com'
  })
})
```

في هذا المثال، نحدث المعرف `45` المحدد في نهاية عنوان URL. استخدمنا طريقة `PUT` في الشفرة وحددنا البيانات كجسم يُستخدم لتحديث البيانات المحددة.

- **DELETE**: تُستخدم لحذف البيانات على الخادم. تُستخدم طريقة `DELETE` لحذف الموارد على الخادم.

```js
fetch('https://api.example.com/users/45', {
  method: 'DELETE'
})
```

في هذا المثال، نرسل طلب `DELETE` لإزالة مستخدم بالمعرف `45`.

## الوعد وتسلسل الوعود

- **الوعود** هي كائنات تمثل إتمام أو فشل عملية غير متزامنة وقيمتها الناتجة. تُعرف قيمة الوعد فقط عند إتمام العملية غير المتزامنة.
- إليك مثال لإنشاء وعد بسيط:

```js
const promise = new Promise((resolve, reject) => {
  setTimeout(() => {
    resolve('Data received successfully');
  }, 2000);
});
```

- تُستخدم طريقة `.then()` في الوعد لتحديد ما يجب أن يحدث عند تحقيق الوعد، بينما تُستخدم `.catch()` لمعالجة أي أخطاء تحدث.
- إليك مثالًا لاستخدام `.then()` و`.catch()` مع وعد:

```js
promise
  .then(data => {
    console.log(data);
  })
  .catch(error => {
    console.error(error);
  });
```

في المثال أعلاه، تُستخدم طريقة `.then()` لتسجيل البيانات المستلمة من الوعد، بينما تُستخدم `.catch()` لتسجيل أي أخطاء تحدث.

- **تسلسل الوعود**: من الميزات القوية للوعود أنه يمكننا تسلسل عدة عمليات غير متزامنة معًا. كل `.then()` يمكن أن يعيد وعدًا جديدًا، مما يسمح بأداء متتالية من العمليات غير المتزامنة واحدة تلو الأخرى.
- إليك مثالًا على تسلسل الوعود:

```js
fetch('https://api.example.com/data')
  .then(response => response.json())
  .then(data => {
    console.log(data);
    return fetch('https://api.example.com/other-data');
  })
  .then(response => response.json())
  .then(otherData => {
    console.log(otherData);
  })
  .catch(error => {
    console.error(error);
  });
```

في المثال أعلاه، نسترجع أولًا بيانات من عنوان URL واحد، ثم نسترجع بيانات من عنوان URL آخر بناءً على الاستجابة الأولى، وأخيرًا نسجل البيانات الثانية المستلمة.

طريقة `catch` ستتعامل مع أي أخطاء تحدث أثناء العملية. هذا يعني أنك لا تحتاج لإضافة معالجة أخطاء لكل خطوة، مما يبسط شفرتك كثيرًا.

## استخدام `async/await` لمعالجة الوعود

`async`/`await` يجعل كتابة وقراءة الشفرة غير المتزامنة أسهل، وهو مبني على الوعود.

- **async**: تُستخدم الكلمة المفتاحية `async` لتعريف دالة غير متزامنة. تُعيد الدالة `async` وعدًا (Promise) يُحلّ بالقيمة التي تُعيدها الدالة `async`.
- **await**: تُستخدم الكلمة المفتاحية `await` داخل دالة `async` لإيقاف تنفيذ الدالة مؤقتًا حتى يُحل الوعد. يمكن استخدامها فقط داخل دالة `async`.
- فيما يلي مثال على استخدام `async/await`:

```js
async function delayedGreeting(name) {
  console.log("A Messenger entered the chat...");
  await new Promise(resolve => setTimeout(resolve, 2000));
  console.log(`Hello, ${name}!`);
}

delayedGreeting("Alice");
console.log("First Printed Message!");
```

في المثال أعلاه، دالة `delayedGreeting` هي دالة `async` تتوقف لمدة ثانيتين قبل طباعة رسالة الترحيب. تُستخدم الكلمة المفتاحية `await` لإيقاف تنفيذ الدالة حتى يتم حل الـ `Promise`.

- من أكبر مزايا `async/await` هو معالجة الأخطاء عبر كتل `try/catch`. إليك مثالًا:

```js
async function fetchData() {
  try {
    const response = await fetch('https://api.example.com/data');
    const data = await response.json();
    console.log(data);
  } catch (error) {
    console.error(error);
  }
}

fetchData();
```

في المثال أعلاه، يحتوي كتلة `try` على الشفرة التي قد ترمي خطأ، وكتلة `catch` تتعامل مع الخطأ إذا حدث. هذا يجعل معالجة الأخطاء أكثر وضوحًا وسهولة قراءة.

## الخاصية `async`

- تخبر الخاصية `async` المتصفح بتحميل ملف السكريبت بشكل غير متزامن مع استمرار تحليل مستند HTML.
- بمجرد تحميل السكريبت، يتوقف تحليل HTML، يُنفذ السكريبت، ثم يستأنف تحليل HTML.
- يجب استخدام `async` للسكريبتات المستقلة التي لا يهم ترتيب تنفيذها.

## الخاصية `defer`

- تقوم الخاصية `defer` أيضًا بتحميل السكريبت بشكل غير متزامن، لكنها تؤجل تنفيذ السكريبت حتى يتم تحليل مستند HTML بالكامل.
- تحافظ سكريبتات `defer` على ترتيب التنفيذ كما تظهر في مستند HTML.

- من المهم ملاحظة أن الخاصيتين `async` و`defer` تُتجاهلان للسكريبتات المضمّنة وتعملان فقط مع ملفات السكريبت الخارجية.

- عندما تكون الخاصيتان `async` و`defer` موجودتين معًا، تُعطى الأولوية للخاصية `async`.

## واجهة برمجة تطبيقات تحديد الموقع الجغرافي

- توفر واجهة تحديد الموقع الجغرافي طريقة لمواقع الويب لطلب موقع المستخدم.

- المثال أدناه يوضح طريقة `getCurrentPosition()` من الواجهة، والتي تُستخدم للحصول على الموقع الحالي للمستخدم.

```js
navigator.geolocation.getCurrentPosition(
  (position) => {
    console.log("Latitude: " + position.coords.latitude);
    console.log("Longitude: " + position.coords.longitude);
  },
  (error) => {
    console.log("Error: " + error.message);
  }
);
```

في هذه الشفرة، نستدعي `getCurrentPosition` ونمرر لها دالة، تُستدعى عند الحصول على الموقع بنجاح.

يحتوي كائن `position` على معلومات متنوعة، لكننا اخترنا هنا فقط `latitude` و`longitude`.

إذا حدثت مشكلة في الحصول على `position`، فسيتم تسجيل الخطأ في وحدة التحكم.

- من المهم احترام خصوصية المستخدم وطلب موقعه فقط عند الضرورة.
