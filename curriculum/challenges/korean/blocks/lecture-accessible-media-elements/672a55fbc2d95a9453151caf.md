# --interactive--

많은 사용자가 신체적 장애, 반복적인 긴장 부상, 또는 개인적 선호로 인해 마우스 대신 키보드를 사용합니다. 여기에는 화면 읽기 프로그램 사용자와 작동하는 마우스가 없는 사용자도 포함됩니다.
키보드 접근성은 이러한 사용자가 웹 애플리케이션을 장애 없이 효과적으로 탐색할 수 있도록 보장합니다.

웹 애플리케이션을 키보드로 접근 가능하게 만드는 실용적인 기법 몇 가지를 살펴보겠습니다.

많은 사용자가 웹페이지의 인터랙티브 요소를 이동하기 위해 `Tab` 키를 사용합니다. 기본적으로 브라우저는 링크, 버튼, 폼 필드 같은 요소를 HTML에 나타나는 순서대로 탭으로 이동할 수 있게 합니다. 이를 자연스러운 탭 순서라고 합니다.

때로는 포커스 가능한 요소를 조정하거나 포커스 순서를 변경하고 싶을 수 있습니다. `tabindex` 속성이 이를 가능하게 합니다.

기본 구문은 다음과 같습니다:

```html
<element tabindex="number">Element Text</element>
```

`tabindex`의 값은 요소가 키보드 탐색에서 어떻게 동작하는지 결정합니다:

`tabindex="0"`는 요소를 자연스러운 탭 순서에 추가합니다.

탭을 누르면 `button`에서 `div`로, 그리고 링크로 포커스가 이동하며 HTML 순서를 따릅니다.

미리보기 창의 공백 부분을 아무 곳이나 클릭하세요. 그런 다음 `Tab` 키를 눌러 포커스가 서로 다른 요소 사이를 이동하는 것을 확인하세요. 예제와 상호작용하려면 인터랙티브 편집기를 활성화해야 합니다.

:::interactive_editor

```html
<button>First</button>
<div tabindex="0">Second</div>
<a href="#">Third</a>
```

:::

`tabindex="-1"`는 프로그래밍 방식으로 요소를 포커스 가능하게 만듭니다. 이는 제목, 컨테이너, 대화상자, 오류 메시지처럼 보통 포커스가 불가능한 요소의 포커스를 관리할 때 유용합니다:

```html
<p tabindex="-1">Sorry, there was an error with your submission.</p>
```

이 예제에서 단락은 일반 탭 순서에 포함되지 않아 사용자가 `Tab` 키를 눌러 접근할 수 없습니다. 하지만 스크립트를 통해 이 요소에 포커스를 설정하면 메시지가 사용자에게 주목됩니다. 이 기법은 JavaScript 수업에서 더 자세히 배우게 됩니다.

`tabindex`가 `0`보다 크면 사용자 지정 탭 순서를 설정합니다. 따라서 낮은 양수 값을 가진 요소가 먼저 포커스됩니다.

이 예제에서는 탭을 누르면 `input`가 `tabindex="1"`와 함께 먼저 포커스되고, 그 다음 `2`, 그리고 `3`가 포커스됩니다. 이는 HTML 순서와 상관없습니다.

```html
<input tabindex="2">
<input tabindex="1">
<input tabindex="3">
```

복잡한 위젯, 예를 들어 특정 탐색 순서가 필요한 툴바에서 사용자 지정 양수 값을 사용하는 경우가 있습니다. 하지만 이 방법은 페이지가 커지거나 변경될 때 탐색이 혼란스럽고 유지 관리가 어려워질 수 있으므로 권장하지 않습니다.

`accesskey`는 웹 프로젝트를 키보드로 접근 가능하게 만드는 또 다른 속성입니다. 특정 요소에 포커스하거나 활성화하는 키를 정의할 수 있습니다.

아래 제안에 따라 시도할 수 있는 인터랙티브 예제가 있습니다(먼저 인터랙티브 편집기를 활성화하세요):

- `accesskey="s"`는 `S` 키를 `Save` 버튼에 할당합니다. 대부분 브라우저에서 `ALT + S`(Windows)와 `CTRL + Option + S`(Mac)을 누르면 이 버튼이 활성화됩니다.
- `accesskey="c"`는 `C` 키를 `Cancel` 버튼에 설정하여 사용자가 `ALT + C`(Windows)와 `CTRL + Option + C`(Mac)으로 버튼을 활성화할 수 있게 합니다.
- `accesskey="h"`는 `H` 키를 `Home` 링크에 할당하여 사용자가 `ALT + H`(Windows)와 `CTRL + Option + H`(Mac)으로 홈페이지로 이동할 수 있게 합니다.

접근키를 활성화하는 정확한 키 조합은 브라우저와 운영체제에 따라 다를 수 있습니다. 보통 Windows에서는 `ALT + Specified Key`, Mac에서는 `CTRL + Option + Specified Key`입니다.

:::interactive_editor

```html
<button accesskey="s">Save</button>
<button accesskey="c">Cancel</button>
<a href="index.html" accesskey="h">Home</a>
```

:::

앱에서 키보드 접근성을 높이는 또 다른 방법은 명확한 포커스 표시기를 제공하는 것입니다. 기본 브라우저 포커스 표시기가 부족하다고 느껴지면 요소의 포커스 상태를 타겟팅하여 재정의할 수 있습니다.

다음은 `button` 요소의 포커스 상태를 스타일링한 예입니다. 미리보기 창의 공백 부분을 클릭한 후 `Tab` 키를 눌러 버튼에 포커스를 맞추세요. 예제와 상호작용하려면 인터랙티브 편집기를 활성화해야 합니다.

:::interactive_editor

```html
<link href="styles.css" rel="stylesheet">

<button>Example button</button>
```

```css
button:focus {
  outline: 2px solid #005fcc;
}
```

:::

`outline` 속성은 요소 주위의 윤곽선을 정의하는 데 사용됩니다. 이 예제는 윤곽선을 두께 2픽셀의 실선 파란색으로 설정했습니다. 포커스 표시기는 현재 포커스가 있는 요소를 명확히 알 수 있도록 스타일링해야 합니다. 접근성을 위해 표시기는 배경색과 최소 3:1 이상의 색 대비를 가져야 합니다.

또한 모달과 팝업 같은 컴포넌트에서 사용자가 특정 요소에서 포커스를 이동할 수 없게 되는 키보드 함정을 피해야 합니다.
