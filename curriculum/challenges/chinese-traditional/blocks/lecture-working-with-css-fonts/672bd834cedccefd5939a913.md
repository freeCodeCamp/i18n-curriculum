# --description--

在我們深入之前，你應該知道 CSS 中的 at-rule 是什麼。At-rule 是提供瀏覽器指令的述語。你可以使用它們來定義樣式表的各種面向，例如媒體查詢、關鍵影格、字型字體等。你將在未來的課程中學習更多關於媒體查詢和關鍵影格等概念。這次，你將學習 `@font-face` at-rule。

使用 `@font-face`，你可以透過指定字型檔案、格式以及其他重要屬性，如粗細和樣式，來定義訂製字型。這是基本語法。你可以看到 `@font-face` 後面接著一個由大括號包圍的區塊：

```css
@font-face {
  /* Descriptors */
}
```

在大括號內，你需要包含描述子來自訂你的字型。讓我們看看一些最常用的描述子。`font-family` 描述子指定你將在整個樣式表中用來指涉該字型的名稱。舉例來說，假設你定義了這個 `@font-face rule`。它有定義 `font-family` 描述子，且其值為 `MyCustomFont`：

```css
@font-face {
  font-family: "MyCustomFont";
}
```

在那種情況下，你需要在你的樣式表中使用該名稱，無論何處你想將該訂製字型家族指定給所有被 CSS 選擇器匹配的元素。<dfn>例如</dfn>，你會將它用作 `font-family` 的值：

```css
body { 
  font-family: "MyCustomFont"; 
}
```

但要使 `@font-face` at-rule 有效，你也需要指定 `src`。這包含對字型資源的引用。它基本上是一個由逗號分隔的外部引用或區域安裝的字型名稱列表。它也可以包含有關字型資源格式和技術的提示。

在以下程式碼中，定義了 `src`。該值是一個由逗號分隔並放在不同列以提升可讀性的 URL 列表：

```css
@font-face {
  font-family: "MyCustomFont"; 
  src: url("path/to/font.woff2"),
    url("path/to/font.woff"),
    url("path/to/font.otf");
}
```

你可以呼叫 `url()` 函式（程式）將檔案包含在你的樣式表中。在這個例子中，我們正在包含字型檔案。你需要在小括弧和引號內寫入檔案路徑，包含副檔名。為了提升可讀性，你可以將每個資源寫在不同的行，但最後一個應該以分號結尾。

對於每個字型資源，你也可以指定格式。這是選用的。這是給瀏覽器關於字型格式的提示。如果省略格式，資源將會被下載，並在下載後偵測格式。如果格式無效，資源將不會被下載。

可能的字型格式包括 `collection`、`embedded-opentype`、`opentype`、`svg`、`truetype`、`woff` 和 `woff2`。

這裡有一個字型格式的範例。請注意我們如何在小括弧和引號內寫出特定格式：

```css
@font-face {
  font-family: "MyCustomFont"; 
  src: url("path/to/font.woff2") format("woff2"),
    url("path/to/font.otf") format("opentype"),
    url("path/to/font.woff") format("woff");
}
```

在這個範例中，你可以看到我們正在指定 `WOFF2` 型別、`OpenType` 型別和 `WOFF` 型別。

woff 代表「Web Open Font Format」。WOFF 和 WOFF2 之間的差異在於用來壓縮資料的演算法。OpenType 是由微軟和 Adobe 開發的可縮放電腦字型格式，允許使用者存取字型中的額外功能。它被廣泛應用於主要作業系統。

除了指定格式之外，你也可以指定字型資源的技術。這也是選用的。以下是一個範例，我們指定第二個字型資源的技術。

```css
@font-face {
  font-family: "MyCustomFont"; 
  src: url("path/to/font.woff2") format("woff2"),
    url("path/to/font.otf") format("opentype") tech(color-COLRv1),
    url("path/to/font.woff") format("woff");
}
```

這些是 `@font-face` 規則的基本原理。使用這個 at-rule，你可以指定字型檔案來定義訂製字體，以符合你獨特的設計。
