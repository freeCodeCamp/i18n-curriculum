# --description--

CommonJS 모듈 시스템을 이해했으니, 이제 ES 모듈 시스템을 살펴보겠습니다.

ES 모듈, 또는 ECMAScript 모듈은 ECMAScript 명세에 정의된 JavaScript의 공식 표준 모듈 형식입니다. `import`와 `export` 문을 사용하며, Node.js에서는 비동기로 로드되어 더 효율적입니다.

Node.js는 다음과 같은 경우에 파일을 ES 모듈로 처리합니다:

* `.mjs` 확장자를 가진 파일
    
* 그리고 가장 가까운 상위 `.js` 파일에 최상위 `package.json` 필드가 `"type"` 값으로 설정된 `"module"` 확장자를 가진 파일
    

여기에는 이전에 본 것과 같은 `multiply` 함수가 있지만, 이제 ESM 시스템을 사용할 것입니다. 이를 위해 두 모듈 모두에 `.mjs` 확장자를 할당합니다.

`multiply` 모듈에서 `math.mjs` 함수를 내보내려면, 중괄호 안에 내보내고자 하는 함수, 변수 또는 객체 이름을 `export` 키워드 뒤에 작성하세요:

```javascript
function multiply(a, b) {
  return a * b;
}

export { multiply };
```

또한 `export` 키워드를 `function` 키워드 앞에 사용하여 함수를 직접 내보낼 수도 있습니다. 이것을 명명된 내보내기라고 하며, 모듈당 여러 개의 명명된 내보내기를 가질 수 있습니다:

```javascript
export function multiply(a, b) {
  return a * b;
}
```

그다음 이 모듈을 가져오려면, 중괄호 뒤에 `import` 키워드를 사용하세요. 여기서는 `import` 대신 `require()`를 사용하고 있습니다. 이것이 CommonJS와 ES 모듈 간 주요 차이점 중 하나입니다.

중괄호 안에는 가져오고자 하는 함수, 변수 또는 객체 이름을 작성합니다. 이 경우 `multiply`입니다. 그리고 `from` 키워드와 모듈의 상대 경로를 사용하세요. 이 코드는 성공적으로 실행되며 출력은 `42`가 됩니다.

```javascript
import { multiply } from './math.mjs';

console.log(multiply(6, 7)); // 42
```

ES 시스템에서는 모듈당 하나의 기본 내보내기를 설정할 수 있습니다. 이렇게 하면 가져올 때 중괄호를 사용할 필요가 없습니다.

다음은 `multiply` 모듈 전체에 대해 `math.mjs` 함수를 기본 내보내기로 설정하는 예입니다:

```javascript
export default function multiply(a, b) {
  return a * b;
}
```

이제 함수가 기본으로 가져와져 할당되므로 중괄호가 필요 없습니다:

```javascript
import multiply from './math.mjs';
```

파일 확장자나 `package.json` 파일을 사용해 Node.js에 명시적으로 어떤 모듈 시스템을 사용할지 알려주지 않으면, 소스 코드를 검사해 ES 모듈 구문이 있는지 확인합니다. 구문이 발견되면 ES 모듈로 실행하고, 그렇지 않으면 CommonJS 모듈로 실행합니다.

초기에는 CommonJS가 Node.js의 기본 표준이었지만, 개발자들이 효율성과 최신 JavaScript와의 호환성 때문에 점점 ESM을 채택하고 있으므로, 두 시스템의 차이점과 사용법을 아는 것이 중요합니다.
