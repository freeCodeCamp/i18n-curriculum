# --interactive--

ホイスティングはJavaScriptの挙動の一つで、初心者をよく混乱させますが、理解するとコードの微妙なバグを避けるのに役立ちます。簡単に言うと、ホイスティングとはコードが実行される前のコンパイル段階で、宣言がそれぞれのスコープの先頭に移動されるJavaScriptのデフォルトの動作です。

ホイスティングを理解するには、JavaScriptがコンパイル段階と実行段階の2つのフェーズで動作することを知ることが重要です。コンパイル段階では、JavaScriptエンジンがコードを読み、変数や関数のためのメモリ領域を確保します。ここでホイスティングが関係してきます。

まず変数のホイスティングから始めましょう。`var` キーワードを使って変数を宣言すると、JavaScriptはその宣言をスコープの先頭にホイスティングします。ただし、重要なのは宣言だけがホイスティングされ、初期化はされないということです。つまり、変数を宣言する前にコード内で使うことはできますが、その値は実際に値を代入するまでは `undefined` になります。

:::interactive_editor

```js
console.log(x); // undefined
var x = 5;
console.log(x); // 5
```

:::

このコードでは、`x` を宣言する前に使っていますがエラーは出ず、代わりに `undefined` が返されます。これはJavaScriptが宣言 `var x` をスコープの先頭にホイスティングする一方で、初期化 `x = 5` はホイスティングしないためです。コードは次のように書き換えられたかのように動作します。

:::interactive_editor

```js
var x;
console.log(x); // undefined
x = 5;
console.log(x); // 5
```

:::

関数のホイスティングは少し異なります。`function` 宣言構文を使って関数を宣言すると、関数名と関数本体の両方がホイスティングされます。つまり、コード内で関数を宣言する前に呼び出すことができます。関数ホイスティングの例を見てみましょう。

:::interactive_editor

```js
sayHello(); // "Hello, World!"

function sayHello(){
  console.log("Hello, World!");
}
```

:::

この場合、関数全体がスコープの先頭にホイスティングされるため、`sayHello()` を宣言前に呼び出せます。

ES6で導入された`let` と `const` 宣言ではホイスティングの動作が異なることに注意してください。

```js
console.log(y); // Throws a ReferenceError
let y = 10;
```

これらの宣言はホイスティングされますが初期化されず、コード内で実際に宣言される前にアクセスできません。これは「一時的デッドゾーン」と呼ばれます。

ホイスティングを理解すると、よりクリーンで予測可能なコードを書くのに役立ちます。ただし、ホイスティングに頼りすぎるとコードの読みやすさや保守性が下がる場合があります。ベストプラクティスとしては、ホイスティングに関係なく、変数はスコープの先頭で、関数は使う前に宣言することをお勧めします。これによりコードの動作が明確になり、自分自身や他の人が理解しやすくなります。
