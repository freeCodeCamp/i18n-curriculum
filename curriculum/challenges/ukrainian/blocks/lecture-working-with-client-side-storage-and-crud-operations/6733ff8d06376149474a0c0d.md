# --description--

API `Cache` використовується для збереження мережевих запитів і відповідей, що робить вебзастосунки більш ефективними та навіть дозволяє їм працювати офлайн. Він є частиною ширшого API Service Worker і є критично важливим для створення прогресивних вебзастосунків (PWA), які можуть працювати в умовах ненадійного або повільного інтернет-з’єднання.

Перш ніж продовжити з API `Cache`, нам потрібно спочатку зрозуміти, як працюють service workers, PWA та мережеві запити на базовому рівні.

Мережевий запит — це запит, який веббраузер або застосунок надсилає на сервер, щоб отримати дані або ресурси через інтернет. Наприклад, коли ви відвідуєте вебсайт, ваш браузер надсилає мережевий запит на вебсервер, щоб отримати файли (наприклад, HTML, зображення чи відео), необхідні для відтворення сторінки.

Service worker — це спеціальний тип JavaScript-файлу, який працює у фоновому режимі вебзастосунку, окремо від основного потоку браузера. Він виступає посередником між вебсторінкою та мережею, дозволяючи розробникам перехоплювати мережеві запити, кешувати ресурси та обробляти такі речі, як push-повідомлення або синхронізація у фоновому режимі.

PWA, або прогресивний вебзастосунок, — це тип вебзастосунку, який використовує сучасні вебтехнології для надання досвіду, схожого на нативний застосунок. PWA розроблені так, щоб надійно працювати на будь-якому пристрої, незалежно від умов мережі, і можуть бути доступні через браузер або встановлені на пристрій користувача, як традиційний застосунок.

Тепер, коли ми краще розуміємо service workers, PWA та мережеві запити, можемо заглибитися в Cache API.

API `Cache` — це механізм збереження, який зберігає об’єкти `Request` та `Response`. Коли надсилається запит на сервер, застосунок може зберегти відповідь і пізніше отримати її з кешу замість того, щоб робити новий мережевий запит. Це зменшує час завантаження, економить трафік і покращує загальний досвід користувача.

Браузер надає область збереження, відому як `CacheStorage`, де розробники можуть зберігати пари ключ-значення мережевих запитів і відповідних відповідей.

За допомогою заголовка `Cache-Control` розробники можуть вказати, як довго кешований ресурс має зберігатися, а також чи потрібно його перевіряти або віддавати безпосередньо з кешу.

Використовуючи API `Cache`, розробники можуть створювати вебзастосунки з пріоритетом офлайн-режиму, дозволяючи, наприклад, PWA віддавати кешовані ресурси, коли користувач відключений від мережі.
