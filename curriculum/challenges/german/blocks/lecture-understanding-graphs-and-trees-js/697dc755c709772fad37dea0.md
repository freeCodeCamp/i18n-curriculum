# --description--

Eine **Prioritätswarteschlange** ist ein abstrakter Datentyp (ADT), der ähnlich wie eine Warteschlange oder ein Stapel funktioniert, aber mit einem entscheidenden Unterschied.

Wie Sie vielleicht schon wissen, folgen Standardwarteschlangen dem FIFO-Prinzip (First-in, First-out), bei dem das zuerst in die Warteschlange eingefügte Element auch als erstes wieder entfernt wird.

Stapel folgen dem LIFO-Prinzip (Last-in, First-out), bei dem das zuletzt in den Stapel eingefügte Element als erstes wieder entfernt wird.

Warteschlangen und Stapel berücksichtigen nur die Reihenfolge, in der die Elemente eingefügt wurden.

Prioritätswarteschlangen hingegen berücksichtigen die „Priorität“ der Elemente. Die Priorität wird verwendet, um zu bestimmen, welches Element als nächstes entfernt werden soll.

In der Regel wird das Element mit der höchsten Priorität zuerst entfernt, aber einige Implementierungen können auch das Element mit der niedrigsten Priorität zuerst entfernen. Das hängt von den Anforderungen Ihres Programms ab.

Prioritätswarteschlangen sind sehr hilfreich für praktische Anwendungen wie das Finden des kürzesten Wegs zwischen zwei Orten, das Planen von Aufgaben in Betriebssystemen, die Simulation von Verkehr, die Datenkompression und das Verwalten von Netzwerken.

In der Praxis werden Prioritätswarteschlangen häufig mit einer Heap-Datenstruktur implementiert.

Ein **Heap** ist eine Baum-Datenstruktur mit einer ganz bestimmten Eigenschaft, der **Heap-Eigenschaft**. Diese Eigenschaft bestimmt die Beziehung zwischen jedem Knoten und seinen Kindknoten, basierend auf der Art des Heaps.

Es gibt zwei Haupttypen von Heaps:

* Max-Heap
    
* Min-Heap
    

In einem **Max-Heap** ist der Wert jedes Knotens größer oder gleich dem Wert seiner Kindknoten.

In diesem Beispiel sehen Sie eine Baumstruktur mit den Knoten 8, 7, 5, 2 und 1. Beachten Sie, dass der Knoten 7 größer ist als die Knoten 2 und 1, entsprechend der Heap-Eigenschaft. Das gilt auch für alle anderen Knoten.

<img src="https://cdn.freecodecamp.org/curriculum/lecture-transcripts/how-do-priority-queues-and-heaps-work-1.png" alt="A max-heap tree structure showing nodes with values 8 at the root, 7 and 5 as children of 8, and 2 and 1 as children of 7, demonstrating that each parent node is greater than its children.">

Im Gegensatz dazu ist in einem **Min-Heap** der Wert jedes Knotens kleiner oder gleich dem Wert seiner Kindknoten.

In diesem Beispiel haben wir Knoten mit den Werten 4, 7, 9, 12 und 15. Zum Beispiel ist der Knoten 7 kleiner als die Knoten 12 und 15, entsprechend der Heap-Eigenschaft. Das gilt auch für alle anderen Knoten.

<img src="https://cdn.freecodecamp.org/curriculum/lecture-transcripts/how-do-priority-queues-and-heaps-work-2.png" alt="A min-heap tree structure showing nodes with values 4 at the root, 7 and 9 as children of 4, and 12 and 15 as children of 7, demonstrating that each parent node is less than its children.">

Die Heap-Eigenschaft ist entscheidend, weil sie sicherstellt, dass das Maximum (oder Minimum) – je nach Art des Heaps – immer an der Spitze bleibt, was das Entfernen sehr einfach macht.

In der Praxis werden Heaps typischerweise als Arrays implementiert, um effizient auf Eltern- und Kindknoten zugreifen zu können.

Die Verwendung von Arrays vereinfacht die Logik für den Zugriff auf diese Werte oder „Knoten“, weil hinter den Kulissen, wenn der Heap die Struktur eines vollständigen binären Baums beibehält, die Array-Implementierung nur einfache mathematische Operationen basierend auf den Indizes benötigt, um zu bestimmen, wo die Elemente im Speicher liegen.

JavaScript hat kein eingebautes Heap-Modul, aber Sie können einen Min-Heap mit einem Array implementieren.

Hier ist eine grundlegende Min-Heap-Implementierung in JavaScript:

```js
class MinHeap {
  constructor(compare = (a, b) => a - b) {
    this.data = [];
    this.compare = compare;
  }

  peek() {
    return this.data[0];
  }

  push(value) {
    this.data.push(value);
    this.#bubbleUp(this.data.length - 1);
  }

  pop() {
    if (this.data.length === 0) return undefined;

    const top = this.data[0];
    const last = this.data.pop();

    if (this.data.length > 0) {
      this.data[0] = last;
      this.#bubbleDown(0);
    }

    return top;
  }

  pushPop(value) {
    if (this.data.length === 0) return value;

    if (this.compare(this.data[0], value) < 0) {
      const top = this.data[0];
      this.data[0] = value;
      this.#bubbleDown(0);
      return top;
    }

    return value;
  }

  heapify(arr) {
    this.data = arr.slice();
    for (let i = Math.floor(this.data.length / 2) - 1; i >= 0; i--) {
      this.#bubbleDown(i);
    }
  }

  #bubbleUp(i) {
    while (i > 0) {
      const p = Math.floor((i - 1) / 2);
      if (this.compare(this.data[i], this.data[p]) >= 0) break;
      [this.data[i], this.data[p]] = [this.data[p], this.data[i]];
      i = p;
    }
  }

  #bubbleDown(i) {
    const n = this.data.length;

    while (true) {
      let smallest = i;
      const l = 2 * i + 1;
      const r = 2 * i + 2;

      if (l < n && this.compare(this.data[l], this.data[smallest]) < 0) smallest = l;
      if (r < n && this.compare(this.data[r], this.data[smallest]) < 0) smallest = r;

      if (smallest === i) break;

      [this.data[i], this.data[smallest]] = [this.data[smallest], this.data[i]];
      i = smallest;
    }
  }
}
````

Um diesen Heap zu verwenden, können Sie einen leeren Heap erstellen. Dies wird die zugrundeliegende Datenstruktur für den Heap sein:

```js
const myHeap = new MinHeap();
```

Um Elemente zum Heap hinzuzufügen, rufen Sie `push()` auf. Dies fügt das Element automatisch an der richtigen Stelle ein, um die Heap-Eigenschaft zu erhalten:

```js
myHeap.push(9);
```

Um das Element mit der höchsten Priorität (in diesem Fall den kleinsten Wert) zu erhalten, rufen Sie `pop()` auf:

```js
myHeap.pop();
```

`pushPop()` kombiniert beide Operationen in einem Aufruf.

Das ist effizienter als sie nacheinander aufzurufen, besonders wenn der Heap groß ist, da nur eine Umordnung durchgeführt wird:

```js
myHeap.pushPop(15);
```

Wenn Sie bereits ein Array haben und es in einen Heap umwandeln möchten, können Sie `heapify()` aufrufen:

```js
myHeap.heapify([9, 2, 7, 1]);
```

Aber bisher sortieren wir die Elemente nach ihren Werten, richtig?

Was ist, wenn wir sie stattdessen nach ihrer „Priorität“ sortieren wollen?

Das könnten Sie tun, indem Sie Arrays mit dieser Struktur speichern: `[priority, element]`.

In JavaScript werden Arrays nicht automatisch Element für Element zum Sortieren verglichen, deshalb übergeben Sie dem Heap eine Vergleichsfunktion, die zuerst die Prioritäten vergleicht.

Bitte beachten Sie, dass in diesem Fall niedrigere Werte eine höhere Priorität darstellen. Das bedeutet, dass ein Element mit Priorität 1 eine **höhere** Priorität hat als ein Element mit Priorität 3:

```js
const myHeap = new MinHeap((a, b) => a[0] - b[0]);

myHeap.push([3, "A"]);
myHeap.push([2, "B"]);
myHeap.push([1, "C"]);
```

Wenn Elemente mit derselben Priorität in der Reihenfolge entfernt werden sollen, in der sie eingefügt wurden, könnten Sie einen eindeutigen Zähler als zweites Element Ihres Items hinzufügen, um Gleichstände zu brechen, so wie hier `[priority, counter, element]`.

Zum Beispiel können Sie zuerst nach Priorität und dann nach dem Zähler vergleichen:

```js
let counter = 0;
const myHeap = new MinHeap((a, b) => (a[0] - b[0]) || (a[1] - b[1]));

myHeap.push([3, counter++, "A"]);
myHeap.push([2, counter++, "B"]);
myHeap.push([2, counter++, "C"]);
```

Kommen wir nun zur Effizienz von Heaps.

Die durchschnittliche und schlechteste Zeitkomplexität für das Einfügen und Extrahieren des Minimums oder Maximums aus einem Heap (je nach Art des Heaps) ist logarithmisch, `O(log n)`, weil die Anzahl der erforderlichen Vertauschungen normalerweise proportional zur Höhe des Heaps ist, die log(n) beträgt.

Die durchschnittliche und schlechteste Zeitkomplexität für die „Peek“-Operation ist konstant, O(1). Peek bedeutet, das Minimum oder Maximum (je nach Art des Heaps) zu erhalten, ohne es zu entfernen.

Die „Heapify“-Operation, bei der der Heap aus einer unsortierten Liste aufgebaut wird, hat eine lineare Zeitkomplexität, O(n), im Durchschnitt und im schlechtesten Fall.

Ebenso haben sowohl die Suche nach als auch das Löschen eines beliebigen Elements eine lineare durchschnittliche und schlechteste Zeitkomplexität von O(n), da sie potenziell den gesamten Heap durchlaufen müssen.

Und wie viel Speicher benötigen sie?

Die Speicherkomplexität des Heaps ist linear, O(n), wobei `n` die Anzahl der enthaltenen Elemente ist. Es muss nur die Elemente und einen kleinen zusätzlichen Overhead für das Array-Objekt selbst speichern.

Prioritätswarteschlangen und Heaps sind in der Informatik sehr wichtig. Sie ermöglichen es Ihnen, schnell die wichtigsten Elemente aus einer Liste zu finden und zu verwenden. Diese Effizienz ist entscheidend für viele Computerprogramme, die kritische reale Aufgaben ausführen, wie zum Beispiel die schnellste Route auf einer Karte zu finden.
