# --interactive--

继承是 CSS 中一个关键的概念，它决定了样式如何从父元素传递到其子元素。

就像现实世界中，孩子经常从父母那里继承特征一样，在 CSS 中，某些属性可以由子元素继承自其父元素。

这允许以更高效的方式在整个文档中应用一致的样式。

在 CSS 中，并非所有属性都是默认继承的。例如，像 `color`、`font-family` 和 `line-height` 这样的属性是继承的。这意味着如果你在父元素上设置了文本颜色，除非你特别覆写它，否则其所有子元素都会继承该颜色。

例如，考虑以下示例，其中父 `div` 元素使用内联样式设置其颜色，子 `p` 元素继承该颜色：

:::interactive_editor

```html
<div style="color: blue;">
  This is the parent element.
  <p>This is the child element inheriting the color.</p>
</div>
```

:::

在这个分支中，父 `div` 和子 `p` 都会以蓝色显示它们的文本，因为颜色是继承的。

另一方面，像 `margin`、`padding`、`border` 和 `background` 这样的属性默认不会被继承。如果你想让子元素继承这些样式，你需要显式地设置它们，要么直接在子元素上设置，要么使用 `inherit` 关键字。

`inherit` 关键字可用于迫使从父元素继承某个属性，即使该属性通常不被继承。

例如，如果你想让特定的子元素具有与其父元素相同的 `padding`，你可以在子元素上设置 `padding: inherit`：

:::interactive_editor

```html
<div style="padding: 20px;">
  This is the parent element with padding.
  <p style="padding: inherit;">This is the child element inheriting the padding.</p>
</div>
```

:::

在这种情况下，子 `p` 元素将继承其父 `div` 元素的 `20px` 内边距。

继承对于维护样式表的一致性和减少冗余尤其有用。

你可以在父元素上定义一次样式规则，而不是为多个元素编写相同的样式规则，子元素将继承该规则。这可以使你的 CSS 更简洁且更易于管理。

但是，重要的是要记住继承只在一个方向上起作用——从父元素到子元素。如果你在子元素上覆写样式，它不会影响父元素。
