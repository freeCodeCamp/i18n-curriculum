# --description--

Beim Programmieren, besonders bei der Verwendung von Versionskontrollsystemen wie Git, bedeutet `push`, Änderungen von einem lokalen Repository in ein entferntes hochzuladen.

Die Vergangenheitsform `pushed` zeigt an, dass diese Aktion bereits abgeschlossen wurde.

Zum Beispiel bedeutet `He pushed his code to the GitHub repository`, dass er seine jüngsten Codeänderungen auf GitHub hochgeladen hat.

# --questions--

## --text--

Was bedeutet `push` im Kontext der Versionskontrolle?

## --answers--

Änderungen von einem entfernten Repository herunterladen

### --feedback--

`Push` bedeutet tatsächlich, Änderungen in ein entferntes Repository hochzuladen, nicht herunterzuladen.

---

Änderungen aus einem Repository löschen

### --feedback--

`Push` bezieht sich nicht darauf, Änderungen aus einem Repository zu löschen.

### --feedback--

`push` bezieht sich nicht darauf, Änderungen in einem Repository zu überprüfen.
