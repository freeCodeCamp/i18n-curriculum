# --description--

Una **coda di priorità** è un tipo astratto di struttura dati (ADT) che funziona in modo simile a una coda o a uno stack, ma con una differenza fondamentale.

Come forse già sai, le code standard seguono un approccio FIFO (First-in, First-out), dove il primo elemento aggiunto alla coda è il primo a essere rimosso dalla coda.

Gli stack seguono un approccio LIFO (Last-in, First-out), dove l’ultimo elemento aggiunto allo stack è il primo a essere rimosso dallo stack.

Code e stack considerano solo l’ordine di inserimento degli elementi.

Tuttavia, le code di priorità tengono conto della "priorità" degli elementi. La priorità serve a determinare quale elemento deve essere rimosso per primo.

Di solito, l’elemento con la priorità più alta viene rimosso per primo, ma alcune implementazioni possono scegliere di rimuovere per prime le priorità più basse. Questo dipende dai requisiti del tuo programma.

Le code di priorità sono molto utili in applicazioni pratiche come trovare il percorso più breve tra due luoghi, pianificare attività nei sistemi operativi, simulare il traffico, comprimere dati e gestire reti.

In pratica, le code di priorità sono comunemente implementate usando una struttura dati chiamata heap.

Un **heap** è una struttura dati ad albero con una proprietà molto specifica chiamata **proprietà di heap**. Questa proprietà determina la relazione tra ogni nodo e i suoi figli, in base al tipo di heap.

Esistono due tipi principali di heap:

* Max-heap
    
* Min-heap
    

In un **max-heap**, il valore di ogni nodo è maggiore o uguale al valore dei suoi figli.

In questo esempio, puoi vedere una struttura ad albero con i nodi 8, 7, 5, 2 e 1. Nota che il nodo 7 è maggiore del nodo 2 e del nodo 1, seguendo la proprietà di heap. Questo vale per tutti gli altri nodi.

Al contrario, in un **min-heap**, il valore di ogni nodo è minore o uguale al valore dei suoi figli.

In questo esempio, abbiamo nodi con valori 4, 7, 9, 12 e 15. Per esempio, il nodo 7 è minore del nodo 12 e del nodo 15, seguendo la proprietà di heap. Questo vale anche per tutti gli altri nodi.

La proprietà di heap è fondamentale perché garantisce che l’elemento massimo (o minimo) (a seconda del tipo di heap) rimanga sempre in cima, rendendo molto facile la sua rimozione.

In pratica, gli heap sono tipicamente implementati come array per consultare efficientemente i nodi genitori e figli.

Usare gli array semplifica la logica per consultare questi valori o "nodi" perché, dietro le quinte, se l’heap mantiene la struttura di un albero binario completo, l’implementazione con array richiede solo semplici operazioni matematiche basate sugli indici per trovare dove sono memorizzati gli elementi.

JavaScript non ha un modulo heap integrato, ma puoi implementare un min-heap usando un array.

Ecco una semplice implementazione di un min-heap in JavaScript:

```js
class MinHeap {
  constructor(compare = (a, b) => a - b) {
    this.data = [];
    this.compare = compare;
  }

  peek() {
    return this.data[0];
  }

  push(value) {
    this.data.push(value);
    this.#bubbleUp(this.data.length - 1);
  }

  pop() {
    if (this.data.length === 0) return undefined;

    const top = this.data[0];
    const last = this.data.pop();

    if (this.data.length > 0) {
      this.data[0] = last;
      this.#bubbleDown(0);
    }

    return top;
  }

  pushPop(value) {
    if (this.data.length === 0) return value;

    if (this.compare(this.data[0], value) < 0) {
      const top = this.data[0];
      this.data[0] = value;
      this.#bubbleDown(0);
      return top;
    }

    return value;
  }

  heapify(arr) {
    this.data = arr.slice();
    for (let i = Math.floor(this.data.length / 2) - 1; i >= 0; i--) {
      this.#bubbleDown(i);
    }
  }

  #bubbleUp(i) {
    while (i > 0) {
      const p = Math.floor((i - 1) / 2);
      if (this.compare(this.data[i], this.data[p]) >= 0) break;
      [this.data[i], this.data[p]] = [this.data[p], this.data[i]];
      i = p;
    }
  }

  #bubbleDown(i) {
    const n = this.data.length;

    while (true) {
      let smallest = i;
      const l = 2 * i + 1;
      const r = 2 * i + 2;

      if (l < n && this.compare(this.data[l], this.data[smallest]) < 0) smallest = l;
      if (r < n && this.compare(this.data[r], this.data[smallest]) < 0) smallest = r;

      if (smallest === i) break;

      [this.data[i], this.data[smallest]] = [this.data[smallest], this.data[i]];
      i = smallest;
    }
  }
}
````

Per usare questo heap, puoi creare un heap vuoto. Questa sarà la struttura dati sottostante per l’heap:

```js
const myHeap = new MinHeap();
```

Per aggiungere elementi all’heap, chiameresti `push()`. Questo aggiungerà automaticamente l’elemento nel posto giusto, per preservare la proprietà di heap:

```js
myHeap.push(9);
```

Per ottenere l’elemento con la priorità più alta (in questo caso, il valore più piccolo), chiameresti `pop()`:

```js
myHeap.pop();
```

`pushPop()` combina entrambe le operazioni in una sola chiamata.

Questo è più efficiente che chiamarle separatamente in sequenza, soprattutto quando l’heap è grande, perché esegue una sola operazione di riordinamento:

```js
myHeap.pushPop(15);
```

Se hai già un array e vuoi trasformarlo in un heap, potresti chiamare `heapify()`:

```js
myHeap.heapify([9, 2, 7, 1]);
```

Ma attualmente stiamo ordinando gli elementi in base ai loro valori, giusto?

E se volessimo ordinarli invece in base alla loro "priorità"?

Potresti farlo archiviando array con questa struttura: `[priority, element]`.

In JavaScript, gli array non vengono confrontati automaticamente elemento per elemento per l’ordinamento, quindi passi una funzione di confronto all’heap che confronta prima le priorità.

Nota che, in questo caso, valori più bassi rappresentano priorità più alte. Questo significa che un elemento con priorità 1 avrà una **priorità più alta** di un elemento con priorità 3:

```js
const myHeap = new MinHeap((a, b) => a[0] - b[0]);

myHeap.push([3, "A"]);
myHeap.push([2, "B"]);
myHeap.push([1, "C"]);
```

Se hai bisogno che gli elementi con la stessa priorità vengano rimossi nell’ordine in cui sono stati inseriti, potresti considerare di includere un contatore univoco come secondo elemento del tuo elemento per rompere il pareggio, così `[priority, counter, element]`.

Per esempio, puoi confrontare prima per priorità e poi per contatore:

```js
let counter = 0;
const myHeap = new MinHeap((a, b) => (a[0] - b[0]) || (a[1] - b[1]));

myHeap.push([3, counter++, "A"]);
myHeap.push([2, counter++, "B"]);
myHeap.push([2, counter++, "C"]);
```

Ora parliamo dell’efficienza degli heap.

La complessità temporale media e nel peggior caso per inserire ed estrarre il valore minimo o massimo da un heap (a seconda del tipo di heap) è logaritmica, `O(log n)`, perché il numero di scambi richiesti è solitamente proporzionale all’altezza dell’heap, che è log(n).

La complessità temporale media e nel peggior caso per l’operazione di "peek" è costante, O(1). Peek significa ottenere il valore minimo o massimo (a seconda del tipo di heap) senza rimuoverlo.

L’operazione di "heapify", dove l’heap viene costruito da una lista non ordinata, ha complessità temporale lineare, O(n), nei casi medi e peggiori.

Allo stesso modo, sia la ricerca che la cancellazione di un elemento arbitrario hanno complessità temporale lineare media e nel peggior caso di O(n), perché potenzialmente richiedono di attraversare tutto l’heap.

E quanta memoria richiedono?

La complessità spaziale dell’heap è lineare, O(n), dove `n` è il numero di elementi che contiene. Serve solo per archiviare gli elementi e un piccolo overhead aggiuntivo per l’oggetto array stesso.

Le code di priorità e gli heap sono molto importanti nell’informatica. Ti permettono di trovare e usare rapidamente gli elementi più importanti di una lista. Questa efficienza è cruciale per molti programmi che svolgono compiti reali critici, come trovare il percorso più veloce su una mappa.
