# --description--

La funzione `mapDispatchToProps()` serve per fornire specifici action creator ai tuoi componenti React in modo che possano dispatchare azioni contro l'archivio Redux. Ha una struttura simile alla funzione `mapStateToProps()` che hai scritto nella sfida precedente. Restituisce un oggetto che associa azioni di dispatch a nomi di proprietà, che diventano props del componente `props`. Tuttavia, invece di restituire un pezzo di stato `state`, ogni proprietà restituisce una funzione che chiama `dispatch` con un action creator e i dati rilevanti dell'azione. Hai accesso a questo `dispatch` perché viene passato a `mapDispatchToProps()` come parametro quando definisci la funzione, proprio come hai passato `state` a `mapStateToProps()`. Dietro le quinte, React Redux usa il `store.dispatch()` di Redux per effettuare questi dispatch con `mapDispatchToProps()`. Questo è simile a come usa `store.subscribe()` per i componenti mappati allo stato `state`.

Per esempio, hai un action creator `loginUser()` che prende un `username` come payload dell'azione. L'oggetto restituito da `mapDispatchToProps()` per questo action creator sarebbe qualcosa del genere:

```jsx
{
  submitLoginUser: function(username) {
    dispatch(loginUser(username));
  }
}
```

# --instructions--

L'editor di codice fornisce un action creator chiamato `addMessage()`. Scrivi la funzione `mapDispatchToProps()` che prende `dispatch` come argomento, poi restituisce un oggetto. L'oggetto dovrebbe avere una proprietà `submitNewMessage` impostata sulla funzione dispatch, che prende un parametro per il nuovo messaggio da aggiungere quando dispatcha `addMessage()`.
