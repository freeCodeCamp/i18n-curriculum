# --description--

Viumumvisho vinatuwezesha kuunganisha vichaguzi vingi kwa njia tofauti na kuviunganisha kwa vikundi au kwa mfululizo, kwani vinaonyesha uhusiano kati ya vichaguzi. Kuna aina nne za viumvisho kwa jumla, lakini kwa sasa tutakuonyesha tu viumvisho vya mzao, vinavyowakilishwa katika CSS na nafasi moja kati ya vichaguzi. Viumvisho vya mzao vitachagua vipengele vinavyolingana na kichaguzi cha mwisho tu ikiwa pia vina mzazi (mzazi, babu, n.k.) anayelingana na kichaguzi kilichotangulia.

Kwa hivyo kitu kama `.ancestor .child` kitachagua kipengele chenye darasa la vitu `child` ikiwa kina mzazi mwenye darasa la vitu `ancestor`. Njia nyingine ya kufikiria ni kwamba mtoto atachaguliwa tu ikiwa amepangwa ndani ya `ancestor`, haijalishi ni kwa kina kiasi gani. Tazama mfano hapa chini kwa haraka na uone kama unaweza kusema ni vipengele gani vitachaguliwa kulingana na kanuni ya CSS iliyotolewa:

```html
<!-- index.html -->

<div class="ancestor"> <!-- A -->
  <div class="contents"> <!-- B -->
    <div class="contents"> <!-- C -->
    </div>
  </div>
</div>

<div class="contents"></div> <!-- D -->
```

```css
/* styles.css */

.ancestor .contents {
  /* some declarations */
}
```

Katika mfano huu wa juu, vipengele viwili vya kwanza vyenye darasa la vitu `contents` (`B` na `C`) vitachaguliwa, lakini kipengele cha mwisho (`D`) hakitachaguliwa. Makisio yako yalikuwa sahihi?

Hakuna kikomo cha idadi ya viumvisho unavyoweza kuongeza kwenye kanuni, kwa hivyo `.one .two .three .four` itakuwa halali kabisa. Hii itachagua kipengele chenye darasa la vitu `four` ikiwa kina mzazi mwenye darasa la vitu `three`, na ikiwa mzazi huyo ana mzazi wake mwenye darasa la vitu `two`, na kadhalika. Kwa kawaida unapaswa kuepuka kujaribu kuchagua vipengele vinavyohitaji kiwango hiki cha upangaji ndani, kwani inaweza kuwa ngumu na ndefu, na inaweza kusababisha matatizo linapokuja suala la maalum.
