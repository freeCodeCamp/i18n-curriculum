# --description--

`Cache` API는 네트워크 요청과 응답을 저장하는 데 사용되어 웹 애플리케이션이 더 효율적으로 작동하고 오프라인에서도 기능할 수 있게 합니다. 이 API는 더 넓은 범위의 Service Worker API의 일부이며, 불안정하거나 느린 네트워크 환경에서도 작동할 수 있는 Progressive Web Apps(PWA)를 만드는 데 매우 중요합니다.

`Cache` API를 계속하기 전에, 먼저 서비스 워커, PWA, 그리고 네트워크 요청이 기본적으로 어떻게 작동하는지 이해해야 합니다.

네트워크 요청은 웹 브라우저나 애플리케이션이 서버에 데이터를 가져오거나 리소스를 요청하는 행위입니다. 예를 들어, 웹사이트를 방문할 때 브라우저는 웹 서버에 네트워크 요청을 보내 페이지를 표시하는 데 필요한 파일들(HTML, 이미지, 비디오 등)을 가져옵니다.

서비스 워커는 웹 애플리케이션의 백그라운드에서 실행되는 특별한 유형의 JavaScript 파일로, 메인 브라우저 스레드와는 별개로 작동합니다. 웹 페이지와 네트워크 사이에서 중개자 역할을 하며, 개발자가 네트워크 요청을 가로채고, 리소스를 캐시하며, 푸시 알림이나 백그라운드 동기화 같은 기능을 처리할 수 있게 합니다.

PWA, 즉 Progressive Web App은 최신 웹 기술을 사용해 웹에서 네이티브 앱과 같은 경험을 제공하는 웹 애플리케이션 유형입니다. PWA는 네트워크 상태와 상관없이 어떤 기기에서도 안정적으로 작동하도록 설계되었으며, 브라우저를 통해 접근하거나 전통적인 앱처럼 사용자의 기기에 설치할 수 있습니다.

서비스 워커, PWA, 네트워크 요청에 대해 더 잘 이해했으니, 이제 캐시 API에 대해 더 깊이 알아보겠습니다.

`Cache` API는 `Request`와 `Response` 객체를 저장하는 저장 메커니즘입니다. 서버에 요청이 이루어질 때, 애플리케이션은 응답을 저장소에 저장하고 나중에 새 네트워크 요청 대신 캐시에서 이를 가져올 수 있습니다. 이는 로드 시간을 줄이고, 대역폭을 절약하며, 전반적인 사용자 경험을 향상시킵니다.

브라우저는 `CacheStorage`라는 저장 영역을 제공하며, 개발자는 네트워크 요청과 해당 응답의 키-값 쌍을 저장할 수 있습니다.

`Cache-Control` 헤더를 사용하면 개발자가 캐시된 리소스가 얼마나 오래 저장될지, 그리고 재검증할지 아니면 캐시에서 바로 제공할지 지정할 수 있습니다.

`Cache` API를 사용하면 개발자가 오프라인 우선 웹 애플리케이션을 구축할 수 있어, 예를 들어 PWA가 네트워크 연결이 끊겼을 때 캐시된 자산을 제공할 수 있습니다.
