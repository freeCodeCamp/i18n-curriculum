# --interactive--

고차 함수는 JavaScript에서 매우 강력한 개념으로, 코딩 능력을 크게 향상시키고 코드를 더 유연하고 재사용 가능하게 만듭니다.

본질적으로 고차 함수는 하나 이상의 함수를 인수로 받거나, 함수를 반환하거나, 또는 둘 다 하는 함수입니다.

고차 함수를 이해하려면 먼저 JavaScript에서 함수가 일급 객체라는 점을 생각해 보세요. 이는 함수가 다른 값처럼 취급될 수 있다는 뜻입니다. 변수에 할당할 수 있고, 다른 함수에 인수로 전달할 수 있으며, 함수에서 반환할 수도 있습니다. 이러한 유연성이 고차 함수의 생성과 사용을 가능하게 합니다.

고차 함수의 일반적인 용도 중 하나는 복잡한 작업을 추상화하는 것입니다.

예를 들어, 배열의 각 요소에 특정 작업을 수행하는 함수가 있다고 해봅시다. 서로 다른 작업을 위해 별도의 함수를 작성하는 대신, 작업을 인수로 받는 고차 함수를 만들 수 있습니다. 이렇게 하면 같은 함수 구조를 다양한 동작으로 재사용할 수 있습니다.

이 개념을 설명하는 예시는 다음과 같습니다:

:::interactive_editor

```js
function operateOnArray(arr, operation) {
  let result = [];
  for (let i = 0; i < arr.length; i++) {
    result.push(operation(arr[i]));
}
  return result;
}

function double(x) {
    return x * 2;
}

let numbers = [1, 2, 3, 4, 5];
let doubledNumbers = operateOnArray(numbers, double);
console.log(doubledNumbers); // [2, 4, 6, 8, 10]
```

:::

이 예시에서 `operateOnArray`는 고차 함수입니다. 배열과 함수(`operation`)를 인수로 받습니다. 그런 다음 배열의 각 요소에 작업을 적용합니다. double 함수가 `operateOnArray`에 인수로 전달되어 함수가 값처럼 사용될 수 있음을 보여줍니다.

고차 함수는 함수를 반환할 수도 있습니다. 이는 보다 일반적인 함수에서 특화된 함수를 만드는 데 특히 유용합니다. 이 개념은 종종 함수 공장이라고 불립니다. 다음 예시를 보세요:

:::interactive_editor

```js
function multiplyBy(factor) {
  return function(number) {
    return number * factor;
  }
}

let double = multiplyBy(2);
let triple = multiplyBy(3);

console.log(double(5)); // 10
console.log(triple(5)); // 15
```

:::

이 경우 `multiplyBy`는 새 함수를 반환하는 고차 함수입니다. 이 새 함수는 `multiplyBy`에 전달된 인수에 따라 특화됩니다. 이를 통해 손쉽게 맞춤형 곱셈 함수를 만들 수 있습니다.

고차 함수는 단순한 이론적 개념이 아니라 JavaScript에서 널리 사용됩니다.

JavaScript 배열의 많은 내장 메서드들, 예를 들어 `map()`, `filter()`, `reduce()`는 고차 함수입니다. 이 메서드들은 함수를 인수로 받아 배열 요소에 다양한 방식으로 적용합니다. 이러한 메서드에 대해서는 앞으로 수업에서 더 배우게 됩니다.

고차 함수를 사용하면 더 선언적이고 이해하기 쉬운 코드를 작성할 수 있습니다.

작업을 단계별로 어떻게 수행할지 설명하는 대신(명령형 프로그래밍), 고차 함수는 무엇을 달성하고 싶은지 설명할 수 있게 해줍니다(선언형 프로그래밍). 이는 코드를 더 읽기 쉽고 유지보수하기 쉽게 만듭니다.

JavaScript를 계속 사용하면서 고차 함수를 자주 만나고 사용하게 될 것입니다. 고차 함수는 JavaScript 함수형 프로그래밍의 핵심이며, 깔끔하고 효율적이며 유연한 코드를 작성하는 데 필수적입니다.

고차 함수를 이해하고 활용하면 정교하고 우아한 JavaScript 프로그램을 작성하는 능력이 크게 향상됩니다.
