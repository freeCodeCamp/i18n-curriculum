# --description--

<ol>
  <li>Take an integer <code>n₀</code>, greater than zero.</li>
  <li>Form the next number <code>n</code> of the series by reversing <code>n₀</code> and adding it to <code>n₀</code></li>
  <li>Stop when <code>n</code> becomes palindromic - i.e. the digits of <code>n</code> in reverse order == <code>n</code>.</li>
</ol>

La relación de recurrencia anterior, cuando se aplica a la mayoría de los números iniciales `n` = 1, 2, ... termina en un palíndromo bastante rápido.

Por ejemplo, si `n₀` = 12 obtenemos:

```md
12
12 + 21 = 33,  a palindrome!
```

Y si `n₀` = 55 obtenemos:

```md
55
55 + 55 = 110
110 + 011 = 121,  a palindrome!
```

Nota que la verificación de un palíndromo ocurre *después* de una adición.

Algunos números iniciales parecen no tener fin; la relación de recurrencia para 196 se ha calculado durante millones de repeticiones formando números con millones de dígitos, sin formar un palíndromo. Estos números que no terminan en un palíndromo se llaman **números Lychrel**.

Para los propósitos de esta tarea, un número Lychrel es cualquier número inicial que no forme un palíndromo dentro de 500 (o más) iteraciones.

**Números semilla y números Lychrel relacionados:**

Cualquier entero producido en la secuencia de un número Lychrel también es un número Lychrel.

En general, cualquier secuencia de un número Lychrel *podría* converger para unirse a la secuencia de un candidato a número Lychrel anterior; por ejemplo, las secuencias para los números 196 y luego 689 comienzan:

```md
    196
    196 + 691 = 887
    887 + 788 = 1675
    1675 + 5761 = 7436
    7436 + 6347 = 13783
    13783 + 38731 = 52514
    52514 + 41525 = 94039
    ...
    689
    689 + 986 = 1675
    1675 + 5761 = 7436
    ...
```

Así que vemos que la secuencia que comienza con 689 converge a, y continúa con los mismos números que la de 196.

Debido a esto, podemos dividir aún más los números Lychrel en verdaderos candidatos a números Lychrel **Seed**, y números **Related** que no producen palíndromos pero tienen enteros en su secuencia que se ven como parte de la secuencia generada a partir de un número Lychrel inferior.

# --instructions--

Escribe una función que tome un número como parámetro. Devuelve true si el número es un número Lynchrel. De lo contrario, devuelve false. Recuerda que el límite de iteraciones es 500.

# --solutions--

```js
function isLychrel(n) {
  function reverse(num) {
    return parseInt(
      num
        .toString()
        .split('')
        .reverse()
        .join('')
    );
  }

  var i;
  for (i = 0; i < 500; i++) {
    n = n + reverse(n);
    if (n == reverse(n)) break;
  }

  return i == 500;
}
```
