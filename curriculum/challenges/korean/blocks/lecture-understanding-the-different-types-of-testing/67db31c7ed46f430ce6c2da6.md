# --description--

모킹은 실제 데이터를 가짜 데이터로 대체하여 실제 컴포넌트의 동작을 시뮬레이션하는 과정입니다. 이를 통해 테스트 담당자와 개발자는 외부 의존성에 의존하지 않고 애플리케이션 내 특정 컴포넌트를 테스트할 수 있습니다.

모킹 데이터의 일반적인 사용 사례는 API 호출을 다룰 때입니다. 애플리케이션에 다음과 같은 fetch 호출이 있다고 가정해 보세요:

```js
export const postService = {
  async getPosts() {
    const response = await fetch("https://api.example.com/posts");
    return response.json();
  },
};
```

`postService` 함수를 테스트해야 한다면, 데이터를 가져오기 위해 계속해서 API를 호출하는 대신 API 응답을 모킹할 수 있습니다. API에서 데이터를 가져오는 데는 시간이 걸리고 네트워크 가용성이나 서버 응답 같은 외부 요인에 의존합니다. API 응답을 모킹하면 이러한 외부 요인을 제거하고 예측 가능하며 제어된 데이터 세트로 작업할 수 있습니다.

다음은 모킹된 API 응답 예시입니다:

```json
[
  {
    "id": 1,
    "title": "Understanding Async/Await in JavaScript",
    "content": "Async/Await makes asynchronous code look and behave like synchronous code...",
    "author": "Jane Doe",
    "created_at": "2025-04-10T14:32:00Z"
  },
  {
    "id": 2,
    "title": "10 Tips for Writing Clean Code",
    "content": "Clean code is not just about code formatting. It involves naming, architecture...",
    "author": "John Smith",
    "created_at": "2025-04-08T09:21:00Z"
  },
  {
    "id": 3,
    "title": "Exploring React 19 Features",
    "content": "React 19 comes with exciting features like new hooks and better performance...",
    "author": "Alex Lee",
    "created_at": "2025-04-07T18:47:00Z"
  }
]
```

이제 모킹이 무엇인지 이해했으니, 스터빙과 비교해 보겠습니다. 스터브는 애플리케이션에서 예상되는 동작에 대해 미리 정의된 응답이나 더미 데이터를 반환하는 객체입니다. 예를 들어, 실제 데이터베이스 연결에 의존하지 않고 테스트에서 데이터베이스 연결 동작을 스터빙할 수 있습니다.

다음은 Jest에서 스터브 예시입니다:

```js
import { postService } from "./postService";
import { db } from "./db";

jest.mock("./db", () => ({
  db: {
    fetchPosts: jest.fn(),
  },
}));

test("should return stubbed posts", async () => {
  const fakePosts = [
    { id: 1, title: "Stubbed Post", content: "This is a stubbed post." },
  ];

  db.fetchPosts.mockResolvedValue(fakePosts);

  const posts = await postService.getPosts();

  expect(posts).toEqual(fakePosts);
});
```

이 예시에서 `db.fetchPosts.mockResolvedValue(fakePosts);`이 스터브입니다.

마지막으로 페이크에 대해 이야기해 보겠습니다. 페이크는 실제 컴포넌트의 복잡성이나 부작용 없이 단순화된 버전입니다. 예를 들어, 실제 데이터베이스와 상호작용하는 대신 메모리에 데이터를 저장하여 데이터베이스를 페이크할 수 있습니다. 이렇게 하면 실제 데이터베이스를 다루는 것보다 훨씬 빠르게 메모리 내에서 데이터베이스 작업을 흉내 낼 수 있습니다. 또 다른 일반적인 예는 파일 시스템 작업 시 페이크를 사용하는 것입니다. 테스트할 때 실제 파일 시스템 대신 페이크 파일 시스템을 만들 수 있습니다.

더 복잡한 소프트웨어 애플리케이션을 구축할 때 적절한 경우 모킹, 스터빙, 페이크를 테스트에 활용해 보세요.
