# --description--

Sasa umejenga programu ya msingi ya Express, ni wakati wa kuangalia kwa karibu routing na jinsi inavyofanya kazi.

Katika Express, routing ni jinsi unavyobainisha viishio vya programu ya mtandao. Unaweza kufikiria njia za data kama sheria zinazomwambia programu jinsi ya kujibu wakati mtumizi anapotembelea URL fulani.

Tuchukulie tena programu yetu rahisi ya Express:

```javascript
const express = require("express");
const app = express();
const port = 3000;

app.get("/", (req, res) => {
  res.send("Hello World!");
});

app.listen(port, () => {
  console.log(`Example app listening on port ${port}`);
});
```

Katika mfano huu, programu inabainisha njia ya URL ya mzizi (`/`). Mteja anapotuma ombi la GET kwa njia hii, seva inajibu kwa maandishi `Hello World!`. Kisha programu inaanza kusikiliza maombi yanayoingia kwenye port `3000`.

Lakini kuna njia nyingine za njia ya data.

Express inakuwezesha kushughulikia aina tofauti za maombi ya HTTP kwa kutumia njia tofauti za routing. Zaidi ya kawaida ni:

* `app.get()` – inapata data
    
* `app.post()` – inaunda data mpya
    
* `app.put()` – inasasisha data iliyopo
    
* `app.delete()` – inafuta data
    

Kila njia inaeleza jinsi programu yako inavyopaswa kujibu ombi maalum la HTTP kwenye njia ya data maalum.

Hapa kuna mifano michache rahisi:

```javascript
app.post("/submit", (req, res) => {
  res.send("Form submitted!");
});

app.put("/update", (req, res) => {
  res.send("Item updated!");
});

app.delete("/delete", (req, res) => {
  res.send("Item deleted!");
});
```

Kila mara, Express husikiliza ombi lenye njia ya HTTP na njia ya URL fulani, kisha hufanya kitendakazi kinachofaa ili kutuma jibu kwa mteja.

Unaweza kujaribu njia hizi kwa kutumia zana kama Postman au `curl`, ambazo hukuruhusu kutuma aina tofauti za maombi ya HTTP kwa seva yako.

Na hiyo ndiyo routing katika Express - ni kuhusu kubainisha jinsi programu yako inavyojibu maombi tofauti.
