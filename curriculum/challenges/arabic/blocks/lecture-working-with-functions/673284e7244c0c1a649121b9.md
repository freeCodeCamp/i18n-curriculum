# --interactive--

يشير النطاق في البرمجة إلى مستوى الظهور وإمكانية الوصول إلى المتغيرات في أجزاء مختلفة من شفرتك. يحدد أين يمكن الوصول إلى المتغيرات أو تعديلها. في JavaScript، فهم النطاق ضروري لكتابة شفرة نظيفة وفعالة وخالية من الأخطاء البرمجية. هناك ثلاثة أنواع رئيسية من النطاق: النطاق العام، النطاق المحلي، ونطاق الكتلة.

النطاق العام هو أوسع نطاق في برنامج JavaScript. المتغيرات المعلنة في النطاق العام يمكن الوصول إليها من أي مكان في شفرتك، بما في ذلك داخل الدوال والكتل. غالبًا ما تُسمى هذه المتغيرات متغيرات عامة. رغم أن المتغيرات العامة قد تكون مريحة، يجب استخدامها بحذر لأنها قد تؤدي إلى تعارض في الأسماء وتجعل شفرتك أصعب في الصيانة. إليك مثالًا على متغير عام:

:::interactive_editor

```js
let globalVar = "I'm a global variable";

function printGlobalVar() {
    console.log(globalVar);
}

printGlobalVar(); // "I'm a global variable"
```

:::

في هذا المثال، تم إعلان `globalVar` في النطاق العام ويمكن الوصول إليه داخل الدالة `printGlobalVar`.

أما النطاق المحلي، فيشير إلى المتغيرات التي يمكن الوصول إليها فقط داخل دالة معينة. إليك مثالًا على النطاق المحلي:

:::interactive_editor

```js
function greet() {
    let message = "Hello, local scope!";
    console.log(message);
}

greet(); // "Hello, local scope!"
// console.log(message); // This will throw an error
```

:::

في هذه الشفرة، `message` هو متغير محلي داخل الدالة `greet`. يمكن استخدامه داخل الدالة، لكن محاولة الوصول إليه خارج الدالة ستؤدي إلى خطأ.

نطاق الكتلة هو مفهوم تم تقديمه مع الكلمات المفتاحية `let` و `const` في ES6. الكتلة هي أي قسم شفرة داخل أقواس متعرجة، `{}`، مثل جمل `if`، أو حلقات `for`، أو حلقات `while`. سيتم تدريس مفهوم الحلقات في درس قادم.

المتغيرات المعلنة باستخدام `let` أو `const` داخل كتلة يمكن الوصول إليها فقط داخل تلك الكتلة. إليك مثالًا على نطاق الكتلة:

```js
if (true) {
    let blockVar = "I'm in a block";
    console.log(blockVar); // "I'm in a block"
}
console.log(blockVar); // This will throw an error
```

في هذا المثال، `blockVar` يمكن الوصول إليه فقط داخل كتلة `if`. محاولة الوصول إليه خارج الكتلة ستؤدي إلى خطأ. فهم هذه الأنواع المختلفة من النطاق ضروري لإدارة إمكانية الوصول إلى المتغيرات وتجنب التأثيرات الجانبية غير المقصودة في شفرتك.

يجب استخدام المتغيرات العامة بحذر، لأنها قد تؤدي إلى تعارض في الأسماء وتجعل شفرتك أصعب في الصيانة. تساعد المتغيرات المحلية في عزل أجزاء مختلفة من شفرتك، وهو أمر مفيد بشكل خاص في البرامج الكبيرة. يوفر نطاق الكتلة باستخدام `let` و `const` تحكمًا أدق في إمكانية الوصول إلى المتغيرات، مما يساعد على منع الأخطاء وجعل شفرتك أكثر توقعًا. إتقان هذه المفاهيم الأساسية للنطاق العام والمحلي ونطاق الكتلة سيؤسس قاعدة قوية لفهم المواضيع الأكثر تقدمًا.
