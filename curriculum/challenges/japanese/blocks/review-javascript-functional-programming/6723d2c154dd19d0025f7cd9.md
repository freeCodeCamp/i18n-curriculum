## 純粋な関数と不純な関数

- 純粋な関数とは、同じ入力に対して常に同じ出力を返し、副作用がない関数です。出力は入力のみに依存し、外部の状態を変更しません。
- 不純な関数は副作用を持ち、関数の外部から観察可能なプログラムの状態の変化を引き起こします。

## 関数型プログラミング

- 関数型プログラミングは、問題を解決するために関数の利用を重視し、「何をするか」にフォーカスするソフトウェア開発の手法です。
- 関数型プログラミングは、副作用を避けるために不変のデータ構造や高階関数の使用などの手法を推奨します。
- 正しく使うと、関数型プログラミングの原則はよりクリーンで保守しやすいコードにつながります。

## カリー化

- カリー化は、複数の引数を取る関数を、1つの引数を取る関数のシーケンスに変換する関数型プログラミングの手法です。

以下は通常の関数とカリー化された関数の例です：

:::interactive_editor

```js
// Regular function
function average(a, b, c) {
  return (a + b + c) / 3;
}

console.log(average(2, 3, 4)); // 3

// Curried function
function curriedAverage(a) {
  return function(b) {
    return function(c) {
      return (a + b + c) / 3;
    };
  };
}

console.log(curriedAverage(2)(3)(4)); // 3
```

:::

- カリー化は、多くの引数を取る関数を扱うときに特に強力です。
- カリー化によりコードがより柔軟で再利用しやすくなります。
- アロー関数を使うと、カリー化された関数をより簡潔に作成できます：

```js
const curriedAverage = a => b => c => (a + b + c) / 3;
```

- カリー化はコードを柔軟で再利用しやすくしますが、使いすぎるとコードの読みやすさが低下する場合があります。
