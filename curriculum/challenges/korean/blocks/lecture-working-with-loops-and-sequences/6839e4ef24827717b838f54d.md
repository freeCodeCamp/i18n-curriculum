# --description--

이전 수업들에서 `def` 키워드를 사용해 함수를 정의하는 데 익숙해졌을 것입니다:

```py
def square(num):
    return num ** 2

print(square(4)) # 16
```

하지만 `map()`과 `filter()` 같은 고차 함수와 작업할 때는 익명 인라인 함수를 사용할 수 있습니다. 바로 이때 람다 함수가 등장합니다.

다음은 `square()` 함수를 람다 함수로 리팩터링한 모습입니다:

```py
lambda num: num ** 2
```

앞서 언급했듯이 람다 함수는 익명이므로 이 함수에는 더 이상 `square`라는 이름이 붙지 않습니다. 람다 함수는 다음과 같이 고차 함수에서 사용할 때 매우 유용합니다:

```py
numbers = [1, 2, 3, 4, 5]

even_numbers = list(filter(lambda x: x % 2 == 0, numbers))
print(even_numbers)  # [2, 4]
```

이 예제에서는 숫자 목록이 있고 짝수만 포함하는 새 목록을 만들고자 합니다. 그래서 `filter()` 함수의 인수 중 하나로 람다 함수를 전달해 `2`와 `4` 숫자를 포함하는 새 목록을 얻습니다.

람다 함수를 사용할 때는 모범 사례를 염두에 두는 것이 중요합니다. 예를 들어, 다음과 같이 람다 함수를 변수에 할당하는 것은 좋은 습관이 아닙니다:

```py
numbers = [1, 2, 3, 4, 5]

square = lambda x: x ** 2
squared_numbers = list(map(square, numbers))
print(squared_numbers) # [1, 4, 9, 16, 25]
```

이렇게 하면 익명 함수 사용의 목적이 무색해집니다. 이 경우에는 다음과 같이 일반 함수를 사용하는 것이 좋습니다:

```py
numbers = [1, 2, 3, 4, 5]

def square(num):
    return num ** 2

squared_numbers = list(map(square, numbers))
print(squared_numbers) # [1, 4, 9, 16, 25]
```

또한 다음과 같이 읽기 어렵거나 불필요하게 복잡한 람다 함수는 피해야 합니다:

```py
result = (lambda x: (x**2 + 2*x - 1) if x > 0 else (x**3 - x + 4))(3)
print(result)  # 14
```

이 함수는 잘 실행되고 `14`라는 올바른 결과를 내지만 읽거나 보기 쉽지 않습니다. 이런 경우에는 `if/else` 문이 포함된 별도의 함수를 만들고 그 함수를 호출하는 편이 낫습니다:

```py
def calculate_expression(x):
    if x > 0:
        return x**2 + 2*x - 1
    else:
        return x**3 - x + 4

print(calculate_expression(3))  # 14
```

일반 함수와 람다 함수는 모두 Python 프로그램에서 용도가 있습니다. 단일 인라인 표현식을 다룰 때는 람다 함수를 고려할 수 있습니다. 그렇지 않으면 일반 함수를 사용하는 것이 좋습니다.
