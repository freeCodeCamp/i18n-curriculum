# --interactive--

CSS는 웹페이지에 인라인, 내부, 외부 세 가지 주요 방식으로 적용할 수 있습니다.

각 방식은 고유한 사용 사례, 장점, 한계가 있으며, 언제 어떤 방식을 써야 하는지 아는 것은 깔끔하고 효율적이며 유지보수하기 쉬운 코드를 작성하는 데 필수적입니다.

세 가지 CSS 유형과 각각 언제 사용해야 하는지 살펴보겠습니다.

인라인 CSS는 HTML 요소 내에서 `style` 속성을 사용해 직접 작성합니다. 특정 요소에 스타일을 적용합니다.

인라인 CSS 예시는 다음과 같습니다:

:::interactive_editor

```html
<p style="color: green;">This is an inline-styled paragraph.</p>
```

:::

이 예시에서는 `style` 속성을 사용해 문단 텍스트를 `green`로 설정했습니다.

인라인 CSS는 보통 빠르게 한 번만 적용할 스타일이나 특정 요소의 다른 스타일을 재정의할 때 사용합니다.

하지만 대부분의 경우 HTML이 복잡해지고 코드 유지보수가 어려워지므로 피하는 게 좋습니다.

대부분은 스타일을 정리하고 유지보수하기 쉽게 내부 또는 외부 CSS를 사용하는 편이 낫습니다.

내부 CSS는 HTML 문서의 `style` 섹션 내 `head` 태그 안에 작성합니다. 페이지 전체에 스타일을 적용하며, 단일 문서에 스타일을 적용할 때 유용합니다.

내부 CSS 예시는 다음과 같습니다:

:::interactive_editor

```html
<head>
  <style>
    p {
      color: blue;
    }
  </style>
</head>
<body>
  <p>This paragraph is styled using internal CSS.</p>
</body>
```

:::

위 예시에서 내부 CSS는 페이지 내 모든 `p` 요소에 파란색 텍스트를 적용합니다.

내부 CSS는 여러 페이지가 아닌 특정 페이지에만 스타일을 적용할 때 가장 적합합니다. 단일 페이지 웹사이트나 스타일을 다른 곳에서 재사용할 필요가 없을 때 유용합니다.

하지만 여러 페이지에서 재사용하기 어렵고, 인라인 CSS처럼 HTML과 CSS가 섞여 있어 큰 프로젝트에서는 유지보수가 어려워지는 단점이 있습니다.

외부 CSS는 별도의 `.css` 파일에 작성하고, `link` 섹션의 `head` 요소를 통해 HTML 문서에 연결합니다.

여러 페이지에 일관된 스타일을 적용할 수 있어 전문적인 웹 개발에서 선호되는 방법입니다.

모든 문단 요소에 스타일을 적용하는 예시는 다음과 같습니다:

:::interactive_editor

```html
<head>
  <link rel="stylesheet" href="styles.css">
</head>
<body>
  <p>This paragraph is styled using external CSS.</p>
</body>
```

```css
p {
  color: red;
}
```

:::

이전 수업에서 `link` 요소가 현재 문서와 연결된 리소스 간의 관계를 지정하는 `rel` 속성을 가진다는 것을 배웠습니다. 예를 들어 스타일시트나 외부 리소스에 연결할 때 사용합니다.

반면 `href` 속성은 연결된 리소스의 URL을 지정해 리소스를 어디서 가져올지 알려줍니다.

외부 CSS는 여러 페이지에 걸쳐 일관된 스타일을 유지해야 하는 큰 프로젝트에 이상적입니다.

HTML은 구조를, CSS는 스타일을 담당하게 하여 역할 분리를 촉진하므로 코드 유지보수와 확장성이 좋아집니다.

각 CSS 유형을 언제 사용해야 하는지 이해하는 것은 효율적이고 효과적인 웹 개발에 매우 중요합니다.

대부분의 경우, 특히 크고 복잡한 프로젝트에서는 외부 CSS를 기본으로 사용하는 게 좋습니다.
