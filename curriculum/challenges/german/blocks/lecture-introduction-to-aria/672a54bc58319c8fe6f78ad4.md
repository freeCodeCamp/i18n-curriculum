# --interactive--

Wenn Sie Inhalte anzeigen müssen, diese aber gleichzeitig für Menschen, die Hilfstechnologien wie Screenreader verwenden, ausblenden möchten, können Sie das `aria-hidden`-Attribut verwenden.

Sie müssen es nur zu dem HTML-Element hinzufügen, das Sie ausblenden möchten, und seinen Wert auf `true` setzen, wie Sie hier sehen können: `aria-hidden="true"`.

Dieses Attribut blendet das Element und alle seine Kinder aus dem Barrierefreiheitsbaum aus, während sie auf der Seite sichtbar bleiben. Häufige Anwendungsfälle sind:

- Symbole und Bilder, die nur dekorativen Zwecken dienen.
- Duplizierte Inhalte.

Es ist wichtig zu beachten, dass `aria-hidden` Inhalte nur vor Hilfstechnologien wie Screenreadern verbirgt. Wenn der Inhalt für alle verborgen sein soll, sollten Sie `aria-hidden` nicht verwenden. Zum Beispiel muss ein aktuell eingeklapptes Hamburger-Menü für alle Tastaturnutzenden verborgen sein, nicht nur für Screenreader-Nutzende. In diesem Fall könnten Sie die CSS-`display`-Eigenschaft auf `none` setzen, um das Menü aus der gerenderten Seite und dem Barrierefreiheitsbaum zu entfernen, wenn es eingeklappt ist.

Sie sollten niemals `aria-hidden` verwenden, um ein Element auszublenden, das mit der Tastatur fokussierbar ist. Das `aria-hidden`-Attribut entfernt das Element nur aus dem Barrierefreiheitsbaum. Es entfernt es nicht aus dem DOM. Daher können Screenreader-Nutzer weiterhin mit der Tabulatortaste zu dem Element navigieren, aber ihr Screenreader wird das Element nicht ankündigen, was dazu führt, dass sie effektiv auf „nichts“ fokussieren.

Hier ist ein Beispiel, bei dem wir ein Symbol aus dem Barrierefreiheitsbaum ausblenden, indem wir das `aria-hidden`-Attribut mit dem Wert `true` hinzufügen.

Wir halten nur den Text für Hilfstechnologien verfügbar, um Verwirrung zu vermeiden, die durch die Redundanz entstehen könnte, wenn sowohl ein Symbol als auch Text für denselben Zweck vorhanden sind.

**HINWEIS**: Dieses interaktive Beispiel enthält das Font Awesome CDN, damit Sie das Zahnrad-Symbol im Vorschaufenster sehen können. Um die Vorschauen zu sehen, müssen Sie den interaktiven Editor aktivieren.

:::interactive_editor

```html
<head>
  <!-- Font Awesome CDN -->
  <link
    rel="stylesheet"
    href="https://cdn.jsdelivr.net/npm/@fortawesome/fontawesome-free@6.5.0/css/all.min.css"
  />
</head>

<button>
  <i class="fa-solid fa-gear" aria-hidden="true"></i>
  <span class="label">Settings</span>
</button>
```

:::

Sie müssen `aria-hidden` nicht verwenden, wenn:

- Das HTML-Element bereits ein `hidden`-Attribut hat.
- Das Element oder ein Vorfahre des Elements bereits mit `display: none` ausgeblendet ist.
- Das Element oder ein Vorfahre des Elements bereits mit `visibility: hidden` ausgeblendet ist.

In diesen drei Fällen ist das Element bereits aus dem Barrierefreiheitsbaum verborgen, daher ist das `aria-hidden`-Attribut nicht notwendig.

Wie bei allen ARIA-Attributen sollten Sie immer mit Hilfstechnologien testen und idealerweise Menschen mit Behinderungen Ihre Arbeit überprüfen lassen, um sicherzustellen, dass sie auch mit diesen ausgeblendeten Elementen leicht verständlich ist.

Sie sollten auch wissen, dass das Setzen von `aria-hidden` auf `false` das Element nicht für Hilfstechnologien verfügbar macht, wenn eines seiner übergeordneten Elemente dieses Attribut auf `true` gesetzt hat.

Das `aria-hidden`-Attribut wird verwendet, um Elemente für Menschen, die Hilfstechnologien wie Screenreader verwenden, auszublenden.

Obwohl es hilfreich sein kann, rein dekorative Elemente und duplizierte Inhalte auszublenden, sollte es sparsam eingesetzt werden, um die Barrierefreiheit nicht zu beeinträchtigen.

Im Allgemeinen sollten alle Inhalte und Funktionen, die auf der Seite verfügbar sind, auch für Menschen zugänglich sein, die Hilfstechnologien verwenden. Der Anwendungsfall für `aria-hidden` ist sehr eng gefasst und sollte hauptsächlich dazu dienen, die Erfahrung für Screenreader-Nutzende zu verbessern, indem rein dekorative oder doppelte Informationen entfernt werden. Verwenden Sie `aria-hidden` nicht, um Inhalte zu verbergen, von denen Sie denken, dass Screenreader-Nutzende daran kein Interesse hätten. Screenreader-Nutzende verdienen es, Zugang zu allen Informationen auf der Seite zu haben.

Wenn Sie diese bewährten Vorgehensweisen befolgen und die Benutzererfahrung testen, können Sie inklusive Online-Erlebnisse für alle schaffen.
