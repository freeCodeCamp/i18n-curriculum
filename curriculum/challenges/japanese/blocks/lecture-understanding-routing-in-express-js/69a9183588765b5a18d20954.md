# --description--

Expressでは、`res`（レスポンス）オブジェクトを使って、受け取ったリクエストを処理した後にクライアントへレスポンスを返します。Expressは、さまざまな種類のレスポンスを送るための組み込みメソッドをいくつか提供しています。たとえば、プレーンテキスト、JSONデータの送信、ユーザーを別のURLにリダイレクト、テンプレートのレンダリングなどが可能です。

最もよく使われるレスポンスメソッドの一つが`res.send()`です。

```javascript
app.get("/text", (req, res) => {
  res.send("Hello, World!");
});
```

`res.send()`メソッドはクライアントにレスポンスを返します。プレーンテキスト、HTML、JSON、さらにはバイナリデータも送ることができます。この例では、クライアントが`/text`にアクセスすると、サーバーはテキスト`Hello, World!`を返します。

もう一つよく使われるレスポンスメソッドが`res.json()`です。

このメソッドはクライアントにJSONレスポンスを送ります。APIを構築するときに特に便利で、オブジェクトや配列のような構造化されたデータを返す必要がある場合に使います。ExpressはJavaScriptのオブジェクトを自動的にJSON形式の文字列に変換します。

```javascript
app.get("/user", (req, res) => {
  res.json({ name: "John Doe", age: 30 });
});
```

クライアントが`/user`にリクエストを送ると、サーバーは以下のレスポンスを返します。

```json
{
  "name": "John Doe",
  "age": 30
}
```

レスポンスのHTTPステータスコードを制御したい場合は、`res.status()`を使います。このメソッドはステータスコードを設定し、通常は`res.send()`や`res.json()`のような他のレスポンスメソッドとチェーンで使います。

```javascript
app.get("/notfound", (req, res) => {
  res.status(404).send("Not Found");
});
```

ここでは、サーバーがステータスコードを`404`に設定し、メッセージ`Not Found`をクライアントに返しています。

ユーザーを別のURLに送る場合は、`res.redirect()`を使います。

```javascript
app.get("/oldpage", (req, res) => {
  res.redirect("/newpage");
});
```

クライアントが`/oldpage`にアクセスすると、自動的に`/newpage`にリダイレクトされます。

最後に、`res.render()`はPug、EJS、Handlebarsなどのテンプレートエンジンを使うときに使います。ビューのテンプレートをレンダリングし、生成されたHTMLをクライアントに送ります。

```javascript
app.get("/profile", (req, res) => {
  res.render("profile", { username: "john_doe" });
});
```

この例では、サーバーが`profile`ビューをレンダリングし、`username`変数を渡してHTML内で表示できるようにしています。

これらのレスポンスメソッドを使うことで、リクエストの内容に応じてさまざまな種類のデータを返せます。テキストやJSONデータを送る場合でも、クライアントをリダイレクトする場合でも、Expressは柔軟に対応できるツールを提供しています。
