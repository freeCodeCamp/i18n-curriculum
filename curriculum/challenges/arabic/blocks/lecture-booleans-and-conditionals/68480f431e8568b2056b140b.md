# --description--

في الدرس السابق، تعلمت كيفية استخدام معاملات المقارنة والعبارات الشرطية للتحكم في تدفق برامجك.

رغم أن هذه الأدوات قوية جدًا، ستواجه غالبًا مواقف تحتاج فيها إلى مقارنة عدة قيم في آن واحد. هذا قد يؤدي إلى عبارات شرطية مسكنة، على سبيل المثال:

```python
is_citizen = True
age = 25

if is_citizen:
    if age >= 18:
        print('You are eligible to vote') # You are eligible to vote
else:
    print('You are not eligible to vote')
```

المثال أعلاه سيتحقق أولًا مما إذا كانت `is_citizen` تساوي `True`. إذا كان الأمر كذلك، سينتقل إلى العبارة الشرطية المسكنة `if` ويتحقق مما إذا كان `age` أكبر من أو يساوي `18`. بما أن `age` أكبر من أو يساوي `18`، ستُطبع الرسالة `You are eligible to vote` على الطرفية. إذا كانت `is_citizen` تساوي `False`، فستُطبع الرسالة `You are not eligible to vote` على الطرفية.

إذا كنت تعمل مع عبارات شرطية أكثر تعقيدًا، يمكنك استخدام معاملات `and` و `or` و `not` في بايثون.

لكن قبل أن نغوص في هذه المعاملات، دعنا نلقي نظرة على ما هي القيم الصحيحة والخاطئة.

في بايثون، لكل قيمة قيمة منطقية ضمنية، أو إحساس مدمج بما إذا كان يجب التعامل معها كـ `True` أو `False` في سياق منطقي. تُعتبر العديد من القيم **صحيحة**، أي أنها تُقيّم إلى `True` في سياق منطقي. وأخرى تُعتبر **خاطئة**، أي أنها تُقيّم إلى `False`.

إليك بعض القيم الخاطئة:

- `None`
- `False`
- العدد الصحيح `0`
- العدد العشري `0.0`
- السلاسل النصية الفارغة `""`

القيم الأخرى مثل الأعداد غير الصفرية، والسلاسل النصية غير الفارغة تُعتبر صحيحة.

إذا أردت التحقق مما إذا كانت القيمة صحيحة أو خاطئة، يمكنك استخدام الدالة المدمجة `bool()`. فهي تحول القيمة صراحة إلى ما يعادلها المنطقي وتُرجع `True` للقيم الصحيحة و`False` للقيم الخاطئة. إليك بعض الأمثلة:

```py
print(bool(False)) # False
print(bool(0))  # False
print(bool('')) # False

print(bool(True)) # True
print(bool(1)) # True
print(bool('Hello')) # True
```

الآن بعد أن فهمت القيم الصحيحة والخاطئة، يمكننا إلقاء نظرة على المعاملات المنطقية، المعروفة أيضًا بالمعاملات البولينية. هذه معاملات خاصة تسمح لك بدمج عدة تعبيرات لإنشاء منطق اتخاذ قرار أكثر تعقيدًا في شفرتك.

هناك ثلاثة معاملات بولينية في بايثون: `and` و `or` و `not`.

لنبدأ بالنظر إلى المعامل `and`.

المعامل `and` يأخذ معاملين ويُرجع المعامل الأول إذا كان خاطئًا، وإلا يُرجع المعامل الثاني. يجب أن يكون كلا المعاملين صحيحين لكي تُنتج التعبير قيمة صحيحة.

إليك مثالًا:

```python
is_citizen = True
age = 25

print(is_citizen and age) # 25
```

في المثال أعلاه، يُطبع الرقم 25 على الطرفية لأن المعامل `and` سيُقيّم المعامل الثاني إذا كان المعامل الأول `True`. يُعرف المعامل `and` بأنه معامل توقف قصير. التوقف القصير يعني أن بايثون يتحقق من القيم من اليسار إلى اليمين ويتوقف بمجرد أن يحدد النتيجة النهائية.

غالبًا ما تستخدم `and` داخل جمل `if` للتحقق من تحقق شروط متعددة. إليك كيف تعيد صياغة المثال السابق لاستخدام عامل `and` بدلاً من جمل `if` المتداخلة:

```python
is_citizen = True
age = 25

if is_citizen and age >= 18:
    print('You are eligible to vote') # You are eligible to vote
else:
    print('You are not eligible to vote')
```

في المثال أعلاه، `is_citizen` تساوي `True`، و`age >= 18` تُقيّم إلى `True`. بما أن كلا المعاملين في `and` صحيحان، فإن الشرط `is_citizen and age >= 18` يُقيّم إلى `True`، ويتم تنفيذ استدعاء `print` داخل كتلة `if`.

الآن لننظر إلى المعامل `or`. هذا المعامل يُرجع المعامل الأول إذا كان صحيحًا، وإلا يُرجع المعامل الثاني. يُنتج تعبير `or` قيمة صحيحة إذا كان على الأقل أحد المعاملين صحيحًا. يُعرف المعامل `or` أيضًا بأنه معامل توقف قصير. إليك مثالًا:

```python
age = 19
is_employed = False

print(age or is_employed) # 19
```

الشيفرة التالية ستطبع الرقم 19 لأن المعامل الأول `age` صحيح.

إذا كنت بحاجة للتحقق مما إذا كان تعبير واحد أو أكثر يساوي `True`، يمكنك استخدام المعامل `or` في شرط مثل هذا:

```python
age = 19
is_student = True

if age < 18 or is_student:
    print('You are eligible for a student discount') # You are eligible for a student discount
else:
    print('You are not eligible for a student discount')
```

في هذه الحالة، `age < 18` تساوي `False`، لكن `is_student` تساوي `True`. بما أن شرطًا واحدًا على الأقل صحيح، فإن التعبير الكامل `or` يُقيّم إلى `True`، وتُطبع رسالة الخصم داخل كتلة `if`.

المعامل الأخير الذي سننظر إليه هو المعامل `not` الذي يأخذ معاملًا واحدًا ويعكس قيمته المنطقية. يحول القيم الصحيحة إلى `False` والقيم الخاطئة إلى `True`. على عكس المعاملات السابقة، `not` يُرجع دائمًا `True` أو `False`.

إليك بعض الأمثلة:

```python
print(not '') # True, because empty string is falsy
print(not 'Hello') # False, because non-empty string is truthy
print(not 0) # True, because 0 is falsy
print(not 1) # False, because 1 is truthy
print(not False) # True, because False is falsy
print(not True) # False, because True is truthy
```

من الشائع استخدام المعامل `not` في الشروط للتحقق مما إذا كان شيء ما ليس `True` أو `False`، مثل هذا:

```python
is_admin = False

if not is_admin:
    print('Access denied for non-administrators.') # Access denied for non-administrators.
else:
    print('Welcome, Administrator!')
```

بما أن `is_admin` تساوي `False`، فإن `not is_admin` تعني ليس `False`، أي `True`. لذا ستُطبع الرسالة `Access denied for non-administrators.`.

الآن بعد أن فهمت القيم الصحيحة والخاطئة، والمعاملات `and` و `or` و `not`، وكيف يعمل التوقف القصير، يمكنك كتابة منطق شرطي أكثر مرونة وقابلية للقراءة.
