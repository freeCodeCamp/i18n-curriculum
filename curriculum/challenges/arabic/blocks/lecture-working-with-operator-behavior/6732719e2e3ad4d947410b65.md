# --interactive--

إذا كتبت تعبيرًا يحتوي على عدة عوامل في JavaScript، كيف يقرر JavaScript أيها يُنفذ أولًا؟ هنا يأتي دور أسبقية العمليات. دعنا نستكشف أسبقية العمليات بالتفصيل مع أمثلة شفرة، وأيضًا ماذا يحدث عندما تكون للعوامل نفس الأسبقية.

تحدد أسبقية العمليات ترتيب تنفيذ العمليات في التعبير. تُنفذ العوامل ذات الأسبقية الأعلى قبل تلك ذات الأسبقية الأقل. فكر في أسبقية العمليات كما في الرياضيات، حيث تتم عملية القسمة والضرب قبل الجمع والطرح.

بدون اتباع هذه القاعدة، ستعطي المعادلات الأساسية إجابة خاطئة. يعمل JavaScript بنفس الطريقة. لديه قواعده الخاصة لتحديد أي العوامل تأتي أولًا، ثانيًا، وهكذا. على سبيل المثال، انظر إلى التعبير أدناه:

:::interactive_editor

```js
const result = 2 + 3 * 4;

console.log(result); // 14
```

:::

لو أن JavaScript نفذ هذا التعبير من اليسار إلى اليمين، كنت تتوقع أن يكون `2 + 3 = 5`، ثم `5 * 4 = 20`. لكن لأن الضرب له أسبقية أعلى من الجمع، ينفذ JavaScript الجزء `3 * 4` أولًا، مما ينتج `2 + 12 = 14`.

أحيانًا، قد تريد أن تُنفذ أجزاء معينة من التعبير أولًا بغض النظر عن قواعد الأسبقية. يمكنك استخدام الأقواس العادية، `()`, لفعل ذلك. أي شيء داخل الأقواس يُنفذ أولًا مهما كان. دعنا نجعل الجزء `2 + 3` من المثال السابق يُنفذ أولًا:

:::interactive_editor

```js
const result = (2 + 3) * 4;

console.log(result); // 20
```

:::

في المثال أعلاه، تجبر الأقواس JavaScript على تنفيذ `2 + 3` أولًا، ثم ضرب الناتج في `4`. هذا يعطيك `20` بدلًا من `14`.

عامل القسمة له أسبقية أعلى من الجمع أو الطرح أيضًا:

:::interactive_editor

```js
const result = 2 + 6 / 3;

console.log(result); // 4
```

:::

لو أن JavaScript نفذ هذا التعبير من اليسار إلى اليمين، قد تتوقع أن يكون `2 + 6 = 8`، ثم `8 / 3 = 2.67`. لكن بما أن القسمة لها أسبقية أعلى من الجمع، ينفذ JavaScript القسمة أولًا: `6 / 3 = 2`، ثم يضيف `2 + 2`، مما يعطي النتيجة `4`.

لذا، في كل من الضرب والقسمة، ستحدث هذه العمليات دائمًا قبل الجمع والطرح ما لم تستخدم الأقواس لتغيير الترتيب. فماذا يحدث إذا كانت العوامل لها نفس الأسبقية؟ يستخدم JavaScript الترابط لتحديد ترتيب التنفيذ.

الترابط هو ما يخبر JavaScript ما إذا كان ينفذ العوامل من اليسار إلى اليمين أو من اليمين إلى اليسار. بالنسبة لمعظم العوامل مثل الجمع والضرب، يكون الترابط من اليسار إلى اليمين. لذا، يعالج JavaScript هذه العوامل من أقصى اليسار في التعبير إلى اليمين:

:::interactive_editor

```js
const result = 10 - 2 + 3;

console.log(result); // 11
```

:::

أولًا، `10 - 2 = 8`، ثم `8 + 3 = 11`. يتحرك JavaScript من اليسار إلى اليمين في هذه الحالة. بعض العوامل، مثل التعيين (`=`)، لها ترابط من اليمين إلى اليسار. هذا يعني أن الجانب الأيمن من التعبير يُنفذ أولًا:

:::interactive_editor

```js
let a, b;
a = b = 5;

console.log(a); // 5
console.log(b); // 5
console.log(a + b); // 10
```

:::

في المثال أعلاه، يعين JavaScript القيمة `5` إلى `b` أولًا، ثم يعين `b` (التي أصبحت الآن `5`) إلى `a`.

عامل الأس أيضًا له ترابط من اليمين إلى اليسار:

:::interactive_editor

```js
const result = 2 ** 3 ** 2;

console.log(result); // 512
```

:::

أولًا، ينفذ JavaScript `3 ** 2`، الذي يساوي `9`، ثم ينفذ `2 ** 9`، الذي يساوي `512`. لو كان عامل الأس له ترابط من اليسار إلى اليمين، كان JavaScript سيحسب `2 ** 3` أولًا ليحصل على `8`، ثم `8 ** 2` ليحصل على `64`.
