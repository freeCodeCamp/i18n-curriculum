# --description--

En lecciones anteriores, has aprendido a trabajar con tipos primitivos, así como con tipos unión e interfaces. 

Pero TypeScript tiene algunos tipos especiales que debes conocer. Comencemos viendo el tipo `any`. 

El tipo `any` se usa para representar cualquier tipo de valor. 

```ts
let randomValue: any;

randomValue = 42;           

randomValue = "Hello";     
 
randomValue = true;        

randomValue = { name: "Alice" }; 
```

Con este tipo, tienes que tener cuidado porque puede ser fácilmente sobreutilizado y mal utilizado. No quieres escribir todo con el tipo `any` solo para silenciar los mensajes de error de TypeScript. Eso va en contra del propósito de usar TypeScript y agregar seguridad de tipos desde el principio. 

Un equivalente más seguro al tipo `any` sería el tipo `unknown`. `unknown` es similar a `any` pero con el tipo `unknown`, tienes que hacer una verificación de tipo para la variable antes de usarla.

Aquí hay un ejemplo:

```ts
function doubleValue(value: unknown) {
  if (typeof value === "number") {
    console.log(value * 2);
  } else if (typeof value === "string") {
    console.log(value + value);
  }
}

doubleValue(10);    
doubleValue("Hi "); 
doubleValue(true);   
```

En este ejemplo, las comprobaciones de tipo aseguran que las operaciones solo se realicen en valores del tipo correcto. Los valores que no coinciden con los tipos esperados se ignoran.

El siguiente tipo que veremos es el tipo `never`. Este tipo representa algo que nunca sucederá.

En la mayoría de las situaciones, no escribirás anotaciones de tipo usando `never`. En cambio, lo más probable es que veas el tipo `never` aparecer en mensajes de error como este:

```ts
function processValue(value: string | number) {
  if (typeof value === "string") {
    console.log("String value:", value.toUpperCase());
  } else if (typeof value === "number") {
    console.log("Number value:", value * 2);
  } else {
    console.log(value); 
  }
}
```

Dentro de la cláusula `else` del ejemplo anterior, el `value` tiene un tipo `never`. Esto se debe a que la rama `else` ahora es imposible de alcanzar y no queda ningún tipo restante. Si intentas llamar a la función con un valor distinto de `string` o `number`, mostrará un error como este:

```ts
function processValue(value: string | number) {
  if (typeof value === "string") {
    console.log("String value:", value.toUpperCase());
  } else if (typeof value === "number") {
    console.log("Number value:", value * 2);
  } else {
    console.log(value); 
  }
}

// Argument of type 'boolean' is not assignable to parameter of type 'string | number'.
processValue(true); 
```

El último tipo que veremos es el tipo `void`. Cuando tienes una función que no devuelve un valor, ese sería un tipo `void`. 

Aquí hay un ejemplo:

```ts
type Status = "loading" | "success" | "error";

type Handler = {
  status: Status;
  onChange: (newStatus: Status) => void; 
};
```

En este ejemplo, la propiedad `onChange` para el tipo `Handler` no devuelve nada, por lo que se está usando `void`.
