# --description--

El módulo OS es otro módulo estándar que viene incorporado en Node.js.

Te permite interactuar con el sistema operativo en el que Node se está ejecutando para que puedas acceder a información vital como el tipo de SO, detalles de la CPU, memoria disponible, memoria total, interfaces de red y más.

Para usar el módulo OS, impórtalo de esta manera:

```js
const os = require("os");
```

El módulo OS proporciona varios métodos que puedes usar para hacer que tu aplicación sea más consciente del sistema. Veamos algunos comunes.

El método `platform()` obtiene el sistema operativo en el que Node se está ejecutando actualmente:

```js
const os = require("os");
console.log(os.platform()); // darwin
```

Puede ser útil para implementar scripting multiplataforma:

```js
if (os.platform() === 'win32') {
 // Windows specific code
} else {
 // Non-Windows specific code
}
```

El método `arch()` muestra una cadena que representa la arquitectura de CPU para la que se compiló Node.js:

```js
const os = require("os");
console.log(os.arch()); // arm64
```

Esto puede ser útil si quieres que los usuarios descarguen los binarios y dependencias correctos para una arquitectura dada.

Otros valores posibles son `'x64'`, `'arm'`, `'arm64'`, `'ia32'`, `'mips'`, `'ppc'` y otros.

`type()` obtiene el nombre oficial del SO, para que puedas identificar programáticamente los sistemas operativos:

```js
const os = require("os");
console.log(os.type()); // Darwin (the core OS for macOS, iOS, and other Apple products)
```

`release()` muestra la versión del kernel del sistema operativo, la parte central del sistema operativo que gestiona los recursos del sistema y la comunicación entre los componentes de hardware y software. Este método puede ser útil para rastrear la compatibilidad entre las versiones del kernel del sistema operativo y los requisitos del servidor.

Devuelve esto como una cadena como `'20.6.0'`:

```js
const os = require("os");
console.log(os.release()); // 25.0.0
```

`version()` devuelve la versión específica del sistema operativo con más detalles que el método `release()`:

```js
const os = require("os");
console.log(os.version());
// Darwin Kernel Version 25.0.0: Wed Sep 17 21:41:39 PDT 2025;
// root:xnu-12377.1.9~141/RELEASE_ARM64_T8103
```

`cpus()` devuelve un arreglo de objetos con detalles sobre cada núcleo lógico de la CPU. Esto puede ayudar a monitorear la carga de la CPU:

```js

console.log(os.cpus());
```

`uptime()` muestra el tiempo desde que el sistema se inició. Puede ayudar a determinar cuánto tiempo han estado funcionando los servidores:

```js
const os = require("os");
console.log(os.uptime()); // 23047
```

`totalmem()` y `freemem()` muestran la cantidad total de memoria del sistema en bytes y la memoria libre del sistema en bytes, respectivamente:

```js
const os = require("os");
console.log(os.totalmem()); // 8589934592 (8 GB)
console.log(os.freemem()); // 93585408 (87 MB)
```

`userInfo()` devuelve un objeto que contiene información sobre el usuario actual del sistema:

```js
const os = require("os");
console.log(os.userInfo());

/*
[Object: null prototype] {
 uid: 502,
 gid: 20,
 username: 'user',
 homedir: '/Users/user',
 shell: '/bin/zsh'
}
*/
```
