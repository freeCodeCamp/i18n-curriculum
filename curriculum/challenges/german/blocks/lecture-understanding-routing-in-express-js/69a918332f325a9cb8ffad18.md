# --description--

In Express sind **Route-Handler** (auch Route-Callback-Funktionen oder Controller-Funktionen genannt) Funktionen, die definieren, was passiert, wenn ein Client eine Anfrage an eine bestimmte Route sendet.

Ein Route-Handler verarbeitet die eingehende Anfrage, führt alle notwendigen Logiken aus, wie zum Beispiel das Abfragen einer Datenbank oder das Durchführen von Berechnungen, und sendet dann eine Antwort zurück an den Client.

Lassen Sie uns das mit einem einfachen Beispiel aufschlüsseln (das Sie tatsächlich die ganze Zeit schon verwendet haben):

```javascript
app.get("/menu", (req, res) => {
  res.send("Welcome to the menu!");
});
```

Die `.get()`-Methode ist eine der Route-Methoden von Express. Sie hört auf **GET**-Anfragen an einer bestimmten Route. In diesem Beispiel hört sie auf Anfragen an `/menu`.

Die Funktion `(req, res) => { ... }` ist der **Route-Handler**. Sie wird immer ausgeführt, wenn ein Client eine GET-Anfrage an `/menu` sendet.

Innerhalb des Handlers:

* `req` (request) ist ein Objekt, das Informationen über die eingehende Anfrage enthält, wie zum Beispiel Routenparameter, Query-Strings, Kopfzeilen und Body-Daten.
    
* `res` (response) ist ein Objekt, das verwendet wird, um eine Antwort an den Client zu senden.
    

In diesem Fall sendet `res.send("Welcome to the menu!")` eine Antwort im Klartext an den Client.

Wenn ein Nutzer im Browser zu `/menu` navigiert oder eine GET-Anfrage an diese Route sendet, führt der Server diesen Handler aus und antwortet mit der Nachricht `Welcome to the menu!`.

Route-Handler sind das Herzstück einer Express-Anwendung. Sie ermöglichen es Ihnen, zu definieren, wie verschiedene Routen Ihrer Webanwendung auf Anfragen reagieren. Sie können auch Logik in diese Handler einbauen, wie zum Beispiel das Verarbeiten von Formulardaten, die Interaktion mit Datenbanken oder das Durchführen von Validierungen.
