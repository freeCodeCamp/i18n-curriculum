# --description--

Працюючи з CSS, ви, ймовірно, рано чи пізно зіткнетеся з проблемами сумісності браузерів. Саме тому важливо надавати запасні варіанти для браузерів, щоб уникнути потенційних проблем.

Коли ваш браузер аналізує CSS сторінки, він ігнорує будь-які властивості, які не розпізнає або не підтримує. Наприклад, якщо ви використовуєте CSS-змінну для призначення кольору фону на сайті, Internet Explorer ігноруватиме цей колір фону, бо не підтримує CSS-змінні. У такому випадку браузер використає будь-яке значення, яке він має для цієї властивості. Якщо він не знайде жодного іншого призначеного значення для цієї властивості, він повернеться до значення за замовчуванням, що зазвичай не є ідеальним.

Це означає, що якщо ви хочете надати запасний варіант для браузера, це так само просто, як додати інше, більш широко підтримуване значення безпосередньо перед вашим оголошенням. Таким чином, старіший браузер матиме на що спиратися, а новіший браузер просто інтерпретуватиме оголошення, яке йде пізніше у каскаді.

# --instructions--

Здається, що змінна використовується для призначення кольору фону класу `.red-box`. Давайте покращимо сумісність з браузерами, додавши інше оголошення `background` безпосередньо перед існуючим оголошенням і призначимо йому значення `red`.

# --hints--

Ваше правило `.red-box` має містити запасний варіант із множиною `background`, встановленою в `red`, безпосередньо перед існуючим оголошенням `background`.

```js
const spacelessCode = __helpers.removeWhiteSpace(__helpers.removeCssComments(code));
assert.match(
  spacelessCode,
  /\.red-box{background:(red|#ff0000|#f00|rgb\(255,0,0\)|rgb\(100%,0%,0%\)|hsl\(0,100%,50%\));background:var\(--red-color\);height:200px;width:200px;}/gi
);
```

## --seed-contents--

```html
<style>
  :root {
    --red-color: red;
  }
  .red-box {

    background: var(--red-color);
    height: 200px;
    width:200px;
  }
</style>
<div class="red-box"></div>
```
