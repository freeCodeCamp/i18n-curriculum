# --interactive--

`audio`와 `video` 요소는 HTML 문서에 사운드와 비디오 콘텐츠를 추가할 수 있게 해줍니다. `audio` 요소는 mp3, wav, ogg 같은 인기 있는 오디오 형식을 지원합니다.
`video` 요소는 mp4, ogg, webm 형식을 지원합니다.

웹 페이지에 오디오 콘텐츠를 포함하려면 `audio` 요소를 사용하고 `src` 속성에 오디오 파일 위치를 지정하면 됩니다.

미리보기 창에서 보시다시피, 페이지에는 아무것도 표시되지 않습니다. 미리보기를 보려면 인터랙티브 편집기를 활성화해야 합니다.

:::interactive_editor

```html
<audio src="https://cdn.freecodecamp.org/curriculum/js-music-player/cruising-for-a-musing.mp3"></audio>
```

:::

오디오 플레이어를 페이지에 표시하려면 `audio` 요소에 `controls` 속성을 추가하세요.

미리보기 창에서 재생 버튼을 눌러 Quincy Larson의 노래 중 하나를 들어보세요. 다른 노래를 듣고 싶으면 `src` 값을 `"https://cdn.freecodecamp.org/curriculum/js-music-player/never-not-favored.mp3"`로 바꾸세요. 미리보기를 보려면 인터랙티브 편집기를 활성화해야 합니다.

:::interactive_editor

```html
<audio src="https://cdn.freecodecamp.org/curriculum/js-music-player/cruising-for-a-musing.mp3" controls></audio>
```

:::

`controls` 속성은 사용자가 오디오 재생을 일시 중지하거나 다시 시작하는 등 재생을 제어할 수 있게 해줍니다. `controls` 속성은 불리언 속성으로, 요소에 추가하면 내장 재생 컨트롤이 활성화됩니다. 생략하면 컨트롤이 표시되지 않습니다.

참고: Safari 같은 일부 브라우저는 `controls` 속성이 있어도 기본적으로 볼륨 컨트롤을 표시하지 않을 수 있습니다.

`src`와 `controls` 속성 외에도 `audio` 요소의 기능을 향상시키는 여러 속성이 있습니다. `loop` 속성은 불리언 속성으로 오디오를 계속 반복 재생하게 만듭니다.

다음은 Quincy Larson의 노래 "Can't stay down"을 재생하는 데 `loop` 속성을 사용하는 예입니다. 반복 재생을 확인하려면 인터랙티브 편집기를 활성화하고 재생 헤드를 노래 끝부분 근처로 이동하면 노래가 끝난 후 다시 시작됩니다.

:::interactive_editor

```html
<audio
  src="https://cdn.freecodecamp.org/curriculum/js-music-player/can't-stay-down.mp3"
  loop
  controls
></audio>
```

:::

또 다른 속성으로 `muted` 속성이 있습니다. `audio` 요소에 이 불리언 속성이 있으면 오디오가 음소거 상태로 시작됩니다. 다음은 `muted` 속성을 사용하는 예입니다.

음악을 듣기 위해 인터랙티브 편집기를 활성화하고 오디오 플레이어의 볼륨 아이콘을 클릭하세요.

:::interactive_editor

```html
<audio
  src="https://cdn.freecodecamp.org/curriculum/js-music-player/can't-stay-down.mp3"
  loop
  controls
  muted
></audio>
```

:::

오디오 파일 형식에 관해서는 브라우저마다 지원하는 형식이 다릅니다. 이를 해결하려면 `source` 요소를 `audio` 요소 안에 넣고, 브라우저가 이해하는 첫 번째 소스를 선택하게 할 수 있습니다. 다음은 `source` 요소에 여러 `audio` 요소를 사용하는 예입니다:

```html
<audio controls>
  <source src="audio.ogg" type="audio/ogg" />
  <source src="audio.wav" type="audio/wav" />
  <source src="audio.mp3" type="audio/mpeg" />
</audio>
```

브라우저는 먼저 ogg 형식부터 시작하고, 재생할 수 없으면 목록의 다음 형식으로 넘어갑니다.

지금까지 배운 모든 속성은 `video` 요소에서도 지원됩니다. 다음은 `video`, `loop`, `controls` 속성을 가진 `muted` 요소를 사용하는 예입니다.

비디오가 자동으로 재생되도록 하려면 시작 `autoplay` 태그에 `video` 속성을 추가하세요. 예제와 상호작용하려면 인터랙티브 편집기를 활성화해야 합니다.

**참고**: `width` 속성은 비디오를 더 작게 만들어 미리보기 창에 잘 맞게 하기 위해 사용되었습니다. `width` 속성에 대해서는 앞으로 수업에서 더 배우게 됩니다.

:::interactive_editor

```html
<video
  src="https://archive.org/download/BigBuckBunny_124/Content/big_buck_bunny_720p_surround.mp4"
  loop
  controls
  muted
  width="400"
></video>
```

:::

`src` 또는 소스 속성에는 archive.org의 "Big Buck Bunny"라는 비디오를 사용했습니다. 비디오가 다운로드되는 동안 이미지를 표시하려면 `poster` 속성을 사용할 수 있습니다. 이 속성은 `audio` 요소에는 없고 `video` 요소에만 있습니다. 다음은 peach.blender.org에서 제공한 콘텐츠와 함께 `poster` 속성을 사용하는 예입니다.

```html
<video
  src="https://archive.org/download/BigBuckBunny_124/Content/big_buck_bunny_720p_surround.mp4"
  loop
  controls
  muted
  poster="https://peach.blender.org/wp-content/uploads/title_anouncement.jpg?x11217"
  width="400"
></video>
```

`source` 요소를 `video` 요소 안에 넣어 `audio` 요소처럼 여러 형식의 동일한 비디오를 제공할 수도 있습니다. 그러면 브라우저가 재생할 수 있는 첫 번째 비디오를 선택합니다.

```html
<video
  controls
  width="400"
  poster="https://peach.blender.org/wp-content/uploads/title_anouncement.jpg?x11217"
>
  <source
    src="https://archive.org/download/BigBuckBunny_124/Content/big_buck_bunny_720p_surround.mp4"
    type="video/mp4"
  />
  <source
    src="https://archive.org/download/BigBuckBunny_124/Content/big_buck_bunny_720p_surround.webm"
    type="video/webm"
  />
  Your browser does not support the video tag.
</video>
```
