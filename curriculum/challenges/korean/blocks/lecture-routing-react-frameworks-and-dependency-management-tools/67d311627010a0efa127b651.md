# --description--

이전 수업에서 React가 싱글 페이지 애플리케이션이라는 것을 배웠습니다. 싱글 페이지 애플리케이션은 하나의 HTML 파일을 포함하고 JavaScript를 사용해 페이지의 모든 콘텐츠를 동적으로 업데이트하는 애플리케이션입니다.

그렇다면 React 애플리케이션에 여러 개의 "페이지"를 추가해야 할 때는 어떻게 할까요? 서로 다른 뷰로 어떻게 이동할 수 있을까요?

바로 이때 React Router가 필요합니다.

React Router는 React 애플리케이션에 라우팅을 추가할 수 있게 해주는 서드파티 라이브러리입니다. 시작하려면 기존 React 프로젝트에 React Router를 이렇게 설치해야 합니다:

```sh
npm i react-router
```

`package.json` 파일을 확인하면 `react-router`가 의존성 목록에 추가된 것을 볼 수 있습니다:

```json
"dependencies": {
  "react": "^18.3.1",
  "react-dom": "^18.3.1",
  "react-router": "^7.2.0"
}
```

그다음 `main.jsx` 또는 `index.jsx` 파일 안에서 `BrowserRouter`를 가져오고 `BrowserRouter` 컴포넌트 주위에 `App`를 렌더링해야 합니다:

```jsx
import { StrictMode } from "react";
import { createRoot } from "react-dom/client";
import { BrowserRouter } from "react-router";
import App from "./App.jsx";

import "./index.css";

createRoot(document.getElementById("root")).render(
  <StrictMode>
    <BrowserRouter>
      <App />
    </BrowserRouter>
  </StrictMode>
);
```

애플리케이션에서 라우트를 활성화하려면 `import` 문을 업데이트하여 `Routes`와 `Route` 컴포넌트를 포함해야 합니다:

```js
import { BrowserRouter, Routes, Route } from "react-router";
```

그다음 `BrowserRouter` 안에 `Routes`와 `Route` 컴포넌트를 추가하세요:

```jsx
createRoot(document.getElementById("root")).render(
  <StrictMode>
    <BrowserRouter>
      <Routes>
        <Route path="/" element={<App />} />
      </Routes>
    </BrowserRouter>
  </StrictMode>
);
```

`path`와 `element`는 URL과 UI 컴포넌트를 연결하는 데 사용됩니다. 여기서는 홈페이지 라우트를 설정하여 `App` 컴포넌트를 가리키도록 했습니다.

대규모 애플리케이션에서는 여러 뷰와 라우트를 이렇게 설정하는 것이 일반적입니다:

```jsx
<Routes>
  <Route index element={<Home />} />
  <Route path="about" element={<About />} />

  <Route path="products">
    <Route index element={<ProductsHome />} />
    <Route path=":category" element={<Category />} />
    <Route path=":category/:productId" element={<ProductDetail />} />
    <Route path="trending" element={<Trending />} />
  </Route>
</Routes>
```

이 예제에서 `index` 프로프는 주어진 경로 세그먼트의 기본 라우트를 나타냅니다. 그래서 `Home` 컴포넌트는 루트 경로(`/`)에서 보여지고 `ProductsHome` 컴포넌트는 `products` 경로에서 보여집니다.

또한 몇몇 라우트를 다른 라우트 안에 중첩한 것도 확인했을 겁니다:

```jsx
<Route path="products">
  <Route path="trending" element={<Trending />} />
</Route>
```

이것은 자식 라우트의 경로가 부모 라우트 경로에 덧붙여진다는 뜻입니다. 이 예제에서 트렌딩 상품의 `path` 경로는 `products/trending`가 됩니다.

만약 `path`이 콜론(`:`)으로 시작한다면, 이는 라우트에서 동적 세그먼트를 나타냅니다:

```jsx
<Route path=":category" element={<Category />} />
```

이 예제에는 `category`라는 동적 세그먼트가 있습니다. 사용자가 `products/brass-instruments` 같은 URL로 이동하면 뷰가 `Category` 컴포넌트로 바뀌고 세그먼트에 따라 적절한 데이터를 동적으로 가져올 수 있습니다.

자식 컴포넌트 안에서 `useParams` 훅을 사용해 동적 세그먼트 값을 접근할 수 있습니다:

```jsx
import { useParams } from "react-router";

export default function Category() {
  let params = useParams();
   {/* Accessing the category param: params.category */}
   {/* rest of code goes here */}
}
```

동적 라우트는 URL의 매개변수에 따라 다양한 콘텐츠를 렌더링할 수 있는 유연하고 재사용 가능한 컴포넌트를 만들 수 있게 해주기 때문에 유용합니다. 모든 가능한 경로에 대해 고정된 경로 목록을 정의하는 대신, 사용자가 요청한 내용에 따라 다양한 콘텐츠를 렌더링하도록 동적 세그먼트를 사용할 수 있습니다.
