# --interactive--

تُعد جمل `switch` وسلاسل `if/else if/else` كلاهما هياكل تحكم في التدفق في البرمجة تتيح لنا تنفيذ كتل شفرة مختلفة بناءً على شروط معينة. مع ذلك، لكل منهما خصائص وحالات استخدام مميزة.

يقوم تعبير `switch` بتقييم تعبير ومطابقة قيمته مع سلسلة من بنود `case`. عند العثور على تطابق، يُنفذ كتلة الشفرة المرتبطة بذلك البند `case`. إليك الهيكل الأساسي لتعبير `switch`:

```js
switch (expression) {
  case value1:
    // code to be executed if expression === value1
    break;
  case value2:
    // code to be executed if expression === value2
    break;
  default:
    // code to be executed if expression doesn't match any case
}
```

جملة `break` في نهاية كل `case` ضرورية جدًا. تخبر البرنامج بالخروج من كتلة switch بمجرد تنفيذ `case` المطابقة. بدونها، يستمر البرنامج في تنفيذ الحالات التالية، وهو سلوك يُعرف بـ "السقوط المتتابع".

تُستخدم جمل `switch` عادةً عند مقارنة متغير واحد مع عدة قيم محتملة. وهي مفيدة بشكل خاص عندما يكون لديك العديد من الشروط المحتملة للتحقق من متغير واحد. إليك مثالًا يستخدم جملة `switch` لأيام الأسبوع:

:::interactive_editor

```js
let dayOfWeek = 3; 

switch (dayOfWeek) {
    case 1:
        console.log("It's Monday! Time to start the week strong.");
        break;
    case 2:
        console.log("It's Tuesday! Keep the momentum going.");
        break;
    case 3:
        console.log("It's Wednesday! We're halfway there.");
        break;
    case 4:
        console.log("It's Thursday! Almost the weekend.");
        break;
    case 5:
        console.log("It's Friday! The weekend is near.");
        break;
    case 6:
        console.log("It's Saturday! Enjoy your weekend.");
        break;
    case 7:
        console.log("It's Sunday! Rest and recharge.");
        break;
    default:
        console.log("Invalid day! Please enter a number between 1 and 7.");
}
```

:::

يمكن أن تكون جمل `switch` أكثر وضوحًا واختصارًا عند التعامل مع العديد من القيم المحتملة لمتغير واحد.

من ناحية أخرى، تُعد جمل `if/else if` أكثر مرونة. يمكنها تقييم شروط معقدة ومتغيرات مختلفة في كل بند. هذا يجعلها مناسبة لمجموعة أوسع من السيناريوهات. إليك مثالًا على متى قد تستخدم جملة `if/else` بدلًا من جملة `switch`:

:::interactive_editor

```js
let creditScore = 720; 
let annualIncome = 60000; 
let loanAmount = 200000; 

let eligibilityStatus;

if (creditScore >= 750 && annualIncome >= 80000) {
    eligibilityStatus = "Eligible for premium loan rates.";
} else if (creditScore >= 700 && annualIncome >= 50000) {
    eligibilityStatus = "Eligible for standard loan rates.";
} else if (creditScore >= 650 && annualIncome >= 40000) {
    eligibilityStatus = "Eligible for subprime loan rates.";
} else if (creditScore < 650) {
    eligibilityStatus = "Not eligible due to low credit score.";
} else {
    eligibilityStatus = "Not eligible due to insufficient income.";
}

console.log(eligibilityStatus);
```

:::

في هذا المثال، لدينا دخل سنوي للشخص ونقاط ائتمانية ونفحص أنواع القروض التي قد يتأهل لها. بما أننا نتعامل مع تقييمات منطقية أكثر تعقيدًا ومتغيرات متعددة، فمن الأفضل استخدام جملة `if/else` هنا بدلًا من جملة `switch`.

يجدر بالذكر أن جمل `switch` في JavaScript تستخدم المقارنة الصارمة (`===`)، مما يعني أنها لا تُجري إجبار النوع. يمكن أن يكون هذا ميزة من حيث التنبؤ وتجنب الأخطاء البرمجية الدقيقة.

باختصار، بينما تسمح كل من جمل `switch` وسلاسل `if/else if` بتنفيذ منطق متعدد الفروع في شفرتك، لكل منهما نقاط قوة مختلفة. تتفوق جمل `switch` في التعامل مع قيم متعددة محتملة لمتغير واحد، بينما توفر سلاسل `if/else if` مرونة أكبر للشروط المعقدة. يعتمد الاختيار بينهما غالبًا على متطلبات الشفرة الخاصة بك وتفضيلات أسلوب البرمجة الشخصية أو الجماعية.
