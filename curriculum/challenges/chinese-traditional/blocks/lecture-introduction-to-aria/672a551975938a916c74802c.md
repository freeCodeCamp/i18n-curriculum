# --interactive--

`aria-describedby` 屬性用於透過取用頁面上現有的內容，向螢幕閱讀器使用者提供關於元素的額外資訊。它在元素與內容之間建立程式化的關聯（技術上稱為可存取描述），當使用者與該元素互動時，螢幕閱讀器可以使用此關聯來告知使用者額外的資訊。 

`aria-describedby` 最常見的用途是將說明和錯誤訊息與表單輸入項目關聯起來。由於螢幕閱讀器使用者有各種在頁面中導航的方法，他們在輸入項目之間切換時可能會錯過這些訊息。使用 `aria-describedby` 有助於確保他們能聽到這些訊息。

讓我們看幾個範例來了解它的運作方式。在這個第一個範例中，我們有一個接受密碼的 `form` 元素。

啟用互動式編輯器，並在密碼輸入欄中輸入幾個字元。你會看到密碼在預覽視窗中被遮蔽。你也應該看到 `password-help` 文字保持紅色，直到你在輸入欄中提供 8 個或以上的字元。 

注意：此互動範例使用 CSS 和 JavaScript 動態更新 `password-help` 文字的顏色。不要擔心嘗試理解 JavaScript 程式碼，因為你會在未來的模組中學習 JavaScript。

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<form>
  <label for="password">Password:</label>
  <input type="password" id="password" aria-describedby="password-help" />
  <p id="password-help">Your password must be at least 8 characters long.</p>
</form>

<script src="index.js"></script>
```

```css
#password-help {
  color: red;
}
```

```js
const passwordEl = document.getElementById("password");
const passwordHelpText = document.getElementById("password-help");

passwordEl.addEventListener("input", (e) => { 
  const userInput = e.target.value;
  passwordHelpText.style.color = userInput.length >= 8 ? "green" : "red";
});
```

:::

我們使用 `label` 元素來表示 `Password` 文字，並將其與密碼 `input` 欄位關聯。

我們也有一個段落元素用來描述密碼要求。我們使用 `aria-describedby` 屬性將密碼 `input` 欄位與段落元素中的密碼要求關聯起來。當螢幕閱讀器使用者與此 `input` 交談時，他們的螢幕閱讀器會讀出該 `input` 的名稱（`Password`），並且可能也會讀出密碼要求。然而，這並非絕對保證，因為有些螢幕閱讀器可能不會自動讀出額外內容，或僅在特定情況下讀出。這並不罕見。每個螢幕閱讀器都不同，並以自己的方式處理 ARIA 屬性。這並不否定使用 `aria-describedby`，因為它通常會對螢幕閱讀器使用者有利益。

另一個 `aria-describedby` 屬性的良好使用案例是當你有一個刪除 `button`。以下是一個刪除 `button`，後面跟著描述按下該按鈕時會發生什麼的訊息範例：

:::interactive_editor

```html
<button aria-describedby="delete-message">Delete</button>

<p id="delete-message">Warning! All deletions are permanent.</p>
```

:::

就像先前的範例一樣，我們使用 `aria-describedby` 屬性將刪除按鈕與訊息關聯起來。`id` 值與 `aria-describedby` 屬性的值必須相符。 

`aria-describedby` 屬性是一個強大的屬性，可用來確保當使用者與元素互動時，能向螢幕閱讀器使用者提供關於該元素的額外資訊。它最常用於將指示和錯誤訊息與表單輸入欄位關聯，以減少螢幕閱讀器使用者在瀏覽表單時遺漏這些訊息的機會。
