# --description--

배열은 컴퓨터 과학에서 기본적인 자료 구조입니다. 모든 배열은 순서가 있는 데이터 모음을 저장하지만, 유형에 따라 내부 동작 방식이 다를 수 있습니다.

이들의 기본 동작은 프로그램의 효율성에 중요한 영향을 미칠 수 있으니, 동적 배열과 정적 배열 및 그 차이점을 배워서 프로그램에 가장 효율적인 것을 선택할 수 있도록 하세요.

먼저 정적 배열부터 시작하겠습니다.

**정적 배열**은 크기가 고정되어 있습니다. 인접한 메모리 위치에 요소를 저장합니다.

정적 배열의 크기는 배열이 초기화될 때 결정됩니다. 특정 메모리 블록이 할당되면 고정되어 프로그램 실행 중에는 변경할 수 없습니다. 이것이 정적 배열의 핵심 특징입니다.

요소를 인접한 메모리 위치에 저장하면 프로그램이 첫 번째 요소의 위치를 저장한 후 인덱스를 사용해 간단한 계산으로 메모리 내 다른 요소를 찾을 수 있어 데이터 검색 과정이 더 효율적입니다.

덕분에 정적 배열의 값에 접근하는 데는 일정한 시간 `O(1)`이 걸리며, 매우 효율적입니다.

저장할 요소의 개수를 미리 알고 있을 때 정적 배열을 사용할 수 있습니다. 또한 접근 작업이 매우 효율적이므로 값에 자주 접근할 때도 유용합니다.

하지만 이 자료 구조는 크기를 늘리거나 줄일 수 없으므로, 저장할 요소 수가 변할 수 있다면 동적 배열을 사용해야 합니다.

정적 배열 크기를 늘리려면 새 배열을 만들고 기존 배열의 모든 요소를 새 배열로 복사해야 하는데, 이는 비효율적입니다. 이럴 때는 이 과정을 자동으로 처리하는 동적 배열이 훨씬 낫습니다.

Python에는 전통적인 정적 배열이 내장 자료 구조로 포함되어 있지 않습니다.

하지만 Java 같은 다른 프로그래밍 언어는 이를 지원합니다. 다음은 세 개의 정수를 저장할 수 있는 Java의 정적 배열 예시입니다:

```java
int[] numbers = new int[3]; 
```

Python의 배열은 동적이므로, 이제 동적 배열을 살펴보겠습니다.

**동적 배열**은 프로그램 실행 중에 자동으로 크기가 늘어나거나 줄어들 수 있어 더 유연합니다.

원래 배열이 가득 차면 요소를 새 배열로 복사하는 자동 크기 조절 메커니즘으로 작동합니다. 새 배열 크기를 효율적으로 선택해 계산 비용이 많이 드는 작업이 덜 자주 발생하도록 처리합니다.

동적 배열의 요소에 접근하는 데는 일정한 시간 `O(1)`이 걸리므로 이 작업도 매우 효율적입니다.

배열 중간에 요소를 삽입하는 데는 선형 시간 `O(n)`이 걸리는데, 그 뒤의 요소들을 다시 배치해야 하기 때문입니다.

배열 끝에 요소를 삽입하는 데는 동적 배열에 아직 공간이 있으면 일정한 시간 `O(1)`이 걸리지만, 배열이 가득 차서 크기 조절이 필요하면 이 작업은 `O(n)` 복잡도를 가집니다.

저장할 값의 수를 미리 모를 때 동적 배열을 사용하세요. 요소를 자주 삽입하거나 삭제할 때도 유용합니다.

Python의 내장 `list` 자료 구조는 동적 배열처럼 작동합니다. 대괄호 안에 요소를 쉼표로 구분해 작성하면 리스트를 만들 수 있습니다.

```python
numbers = [3, 4, 5, 6]
```

리스트의 요소에 접근하려면 리스트를 담은 변수 이름 뒤에 대괄호를 쓰고, 대괄호 안에 해당 인덱스를 적으세요.

인덱스는 첫 번째 요소가 0부터 시작하며, 이후 요소마다 1씩 증가합니다:

```python
numbers[0]  # 3
numbers[1]  # 4
numbers[2]  # 5
numbers[3]  # 6
```

값을 업데이트하려면 다시 할당하면 됩니다:

```python
numbers[2] = 16
```

`.append()` 메서드로 리스트에 요소를 추가할 수 있습니다:

```python
numbers.append(7)
```

`.insert()` 메서드로 특정 인덱스에 요소를 삽입할 수 있는데, 첫 번째 인수로 인덱스, 두 번째 인수로 요소를 전달하세요.

```python
numbers.insert(3, 15)
```

`.pop()` 메서드로 특정 인덱스의 요소를 제거할 수 있습니다:

```python
numbers.pop(2)
```

인덱스를 지정하지 않으면 `.pop()`가 마지막 요소를 제거합니다.

요소를 쉽게 추가하거나 제거할 수 있는 다른 내장 리스트 메서드도 문서에서 확인할 수 있습니다.

이것이 동적 배열, 즉 여기서는 리스트의 강점입니다.

일반적으로 요소 수를 미리 알고 자주 접근해야 할 때는 정적 배열을, 요소 수가 미지수이거나 시간이 지나면서 변할 때는 동적 배열을 사용하세요.

정적 배열의 단순함과 동적 배열의 유연성 사이의 균형을 항상 고려해야 합니다. 두 자료 구조 모두 특정 용도와 상황에 유용합니다. 주어진 문제에 가장 적합한 것을 선택하는 능력은 연습을 통해 점차 키워갈 문제 해결 능력의 일부입니다.
