# --description--

JavaScriptのすべてのオブジェクト（いくつかの例外を除く）には`prototype`があります。また、オブジェクトの`prototype`自体もオブジェクトです。

```js
function Bird(name) {
  this.name = name;
}

typeof Bird.prototype;
```

`prototype`はオブジェクトなので、`prototype`は独自の`prototype`を持つことができます！この場合、`prototype`の`Bird.prototype`は`Object.prototype`です：

```js
Object.prototype.isPrototypeOf(Bird.prototype);
```

これはどのように役立つでしょうか？前のチャレンジで`hasOwnProperty`メソッドを覚えているかもしれません：

```js
let duck = new Bird("Donald");
duck.hasOwnProperty("name");
```

`hasOwnProperty`メソッドは`Object.prototype`に定義されており、`Bird.prototype`からアクセスでき、さらに`duck`からアクセスできます。これは`prototype`チェーンの例です。この`prototype`チェーンでは、`Bird`が`supertype`の`duck`であり、`duck`が`subtype`です。`Object`は`supertype`と`Bird`の両方の`duck`です。`Object`はJavaScriptのすべてのオブジェクトの`supertype`です。したがって、どのオブジェクトも`hasOwnProperty`メソッドを使うことができます。
