# --description--

Ви вже дізналися про `HTMLMediaElement` та MediaStreams API, але існує ще кілька API, які можуть бути дуже корисними для роботи з відео та аудіо.

Перший — це Screen Capture API. Як можна здогадатися з назви, цей API дозволяє записувати екран користувача. Цей API відкривається викликом методу `getDisplayMedia()` об’єкта `mediaDevices` та використанням отриманого медіапотоку.

Далі є MediaStream Recording API. Цей API працює разом із MediaStreams API, дозволяючи записувати MediaStream (або навіть безпосередньо `HTMLMediaElement`). Потім він генерує події `dataavailable` з корисним навантаженням у вигляді Blob, які можна записати у локальне файлове сховище.

Основою всієї цієї технології є Media Source Extensions API. Media Source Extensions API дозволяє, наприклад, безпосередньо передавати відеопотік з вебкамери користувача до відеоелемента через властивість `srcObject`.

І нарешті, Web Audio API, який відповідає за все, що можна почути в інтернеті. Цей API включає важливі об’єкти, такі як `AudioBuffer` (який представляє буфер, що містить аудіодані) або `AudioContext`.

На цьому ми завершуємо наші уроки з аудіо та відео!
