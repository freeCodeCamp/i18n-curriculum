# --description--

Inline-`CSS` ermöglicht es, Styles direkt zu `HTML`-Elementen hinzuzufügen, obwohl diese Methode nicht so empfohlen wird:

```html
<body>
  <div style="color: white; background-color: black;">...</div>
</body>
```

Das Erste, was Sie beachten sollten, ist, dass hier keine Selektoren verwendet werden, da die Styles direkt im öffnenden `<div>`-Tag selbst hinzugefügt werden. Als Nächstes haben Sie das `style`-Attribut, dessen Wert innerhalb der Anführungszeichen die Deklarationen enthält.

Wenn Sie einen einzigartigen Stil für ein einzelnes Element hinzufügen müssen, kann diese Methode gut funktionieren. Im Allgemeinen ist dies jedoch aus einigen Gründen keine empfohlene Art, CSS zu HTML hinzuzufügen:

Es kann schnell ziemlich unübersichtlich werden, sobald Sie viele Deklarationen zu einem einzelnen Element hinzufügen, wodurch Ihre HTML-Datei unnötig aufgebläht wird.
Wenn viele Elemente denselben Stil haben sollen, müssten Sie denselben Stil in jedes einzelne Element kopieren und einfügen, was zu viel unnötiger Wiederholung und mehr Aufblähung führt.
Jedes inline CSS überschreibt die anderen beiden Methoden, was unerwartete Ergebnisse verursachen kann. (Obwohl Sie hier nicht tiefer darauf eingehen, kann dies tatsächlich ausgenutzt werden).

# --questions--    

## --text--

Was ist der Hauptnachteil bei der Verwendung von inline CSS?

## --answers--

Es kann schnell ziemlich unübersichtlich werden, sobald Sie viele Deklarationen zu einem einzelnen Element hinzufügen, wodurch Ihre HTML-Datei unnötig aufgebläht wird.

---

Es erfordert die Verwendung von Selektoren, was für Anfänger kompliziert sein kann.

---

Jedes inline CSS überschreibt die anderen beiden Methoden (intern und extern), was unerwartete Ergebnisse verursachen kann.
