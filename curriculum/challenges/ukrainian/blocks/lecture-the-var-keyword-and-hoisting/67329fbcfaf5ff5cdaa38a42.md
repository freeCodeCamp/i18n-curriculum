# --interactive--

Ключове слово `var` у JavaScript — це один із початкових способів оголосити змінні. Воно було частиною мови з самого початку і багато років залишалося основним методом створення змінних. Однак із розвитком JavaScript і набуттям розробниками більшого досвіду з мовою, стали очевидні певні недоліки використання `var`, що призвело до появи `let` і `const` у 2015 році.

Коли ви оголошуєте змінну за допомогою `var`, вона має область видимості функції або глобальну область видимості. Це означає, що якщо ви оголосите змінну всередині функції за допомогою `var`, вона буде доступна лише в цій функції. Однак якщо оголосити її поза будь-якою функцією, вона стане глобальною змінною, доступною в усьому вашому скрипті. Така поведінка іноді може призводити до несподіваних результатів і ускладнювати розуміння коду.

Проблема з `var` полягає в тому, що воно дозволяє оголошувати одну й ту саму змінну кілька разів без помилки. Це може призводити до випадкових перезаписів і ускладнювати налагодження.

:::interactive_editor

```js
var num = 5;
console.log(num); // 5

// This is allowed and doesn't throw an error
var num = 10;
console.log(num); // 10
```

:::

Найсуттєвіша проблема з `var` — відсутність блочної області видимості. Змінні, оголошені за допомогою `var` всередині блоку, наприклад інструкції `if` або циклу `for`, залишаються доступними поза межами цього блоку.

:::interactive_editor

```js
if (true) {
  var num = 5;
}
console.log(num); // 5
```

:::

Ця поведінка може призводити до небажаних витоків змінних і робить ваш код більш схильним до помилок.

Через ці проблеми сучасна розробка на JavaScript здебільшого відмовилася від `var` на користь `let` і `const`. Ці ключові слова забезпечують блочну область видимості, що більше відповідає тому, як область видимості працює в багатьох інших мовах програмування.

Вони також не дозволяють оголошувати змінну повторно в одній і тій самій області видимості, що допомагає уникнути випадкових заміщень.

Хоча `var` досі є частиною JavaScript і працює у всіх браузерах, загалом рекомендується використовувати `let` і `const` у сучасній розробці на JavaScript. Вони забезпечують чіткі правила області видимості, допомагають уникнути поширених помилок і роблять поведінку вашого коду більш передбачуваною.
