# --description--

웹사이트와 앱은 데이터를 수집하고 처리하거나, 사용자가 필요로 하는 검색 결과 같은 것을 생성해야 합니다. 그래서 텍스트와 마찬가지로 폼도 웹의 기본적인 요소입니다.

폼을 통해 사용자는 사이트에 등록하고 로그인하며, 메시지를 보내고, 항목을 검색할 수 있습니다. 예를 들어, 웹에서 가장 인기 있는 폼인 구글 검색 입력에 쿼리를 입력하면 구글 검색 결과 페이지에 접근할 수 있습니다.

입력 필드, `action` 속성, 폼 메서드, 그리고 가장 중요한 폼 제출 시 일어나는 일을 살펴보겠습니다.

폼은 `form` 요소로 웹 페이지에 임베드되지만, 입력 필드를 표시하려면 필요에 따라 `input` 요소도 필요합니다. 이 입력 요소에는 텍스트, 이메일, 비밀번호, 선택, 파일, 숨김 입력 등이 있습니다.

이 입력 필드를 사용하려면 `form` 요소로 감싸야 합니다. 각 입력에는 서버가 어떤 데이터가 어떤 필드에 해당하는지 알 수 있도록 `name` 속성이 필요합니다.

예시는 다음과 같습니다:

```html
<form>
   <input type="text" name="username" />
   <input type="email" name="email" />
   <input type="file" name="profilePicture" />
   <input type="radio" name="gender" />
   <input type="checkbox" name="subscribe" />
   <input type="hidden" name="userId" />
</form>
```

입력 필드는 사용자가 올바른 정보를 입력했는지 확인하기 위해 유효성 검사를 해야 합니다.

이 유효성 검사는 클라이언트 측과 서버 측 모두에서 일어날 수 있습니다. HTML은 `required`와 `pattern` 속성으로 기본적인 클라이언트 측 유효성 검사를 할 수 있게 해줍니다. 하지만 HTML 유효성 검사는 주로 사용자 경험과 즉각적인 피드백을 위한 것이므로, 보안을 위해 서버 측 유효성 검사도 반드시 해야 합니다.

`required` 속성은 입력 필드를 반드시 채워야 한다고 지정하며, `pattern` 속성은 정규 표현식을 입력할 수 있게 하여 사용자가 원하는 형식으로 정보를 입력하도록 강제합니다.

사용자 이름 필드에 대한 예시는 다음과 같습니다:

```html
<input type="text" name="username" required pattern="[A-Za-z0-9]{3,}" />
```

입력 필드가 유효성 검사를 통과하면 제출이 이루어져야 합니다. 그렇다면 폼이 제출될 때 무슨 일이 일어날까요?

먼저 제출이 이루어지려면, 폼에 입력 필드 값을 보낼 위치를 지정하는 `action` 속성과 폼 데이터를 어떻게 제출할지 지정하는 `method` 속성이 있어야 합니다. `method` 속성에 대해서는 나중에 더 설명하겠습니다.

action 속성 예시는 다음과 같습니다:

```html
<form action="/register">
	<!-- input fields -->
</form>
```

`method` 속성은 `GET`, `POST`, `PUT`, `PATCH`, 또는 `DELETE` 중 하나일 수 있습니다:

* `GET`는 서버에서 무언가를 가져옵니다. 예를 들어, 검색 결과입니다.
    
* `POST`는 서버로 무언가를 보냅니다. 예를 들어, 입력 필드를 통해 입력된 로그인 또는 등록 정보입니다.
    
* `PUT`는 서버의 무언가를 수정합니다.
    
* `PATCH`는 서버의 무언가를 부분적으로 수정합니다.
    
* `DELETE`는 서버의 무언가를 삭제합니다.
    
HTML은 `PUT`, `PATCH`, `DELETE` 메서드를 직접 지원하지 않지만, 우회 방법이 있습니다. 바닐라 JavaScript를 사용할 경우 Fetch API로 메서드를 지정할 수 있고, Node.js와 Express에서는 `method-override` 패키지를 사용할 수 있습니다.

`GET` 요청을 하는 폼 예시는 다음과 같습니다:

```html
<form action="/search" method="GET">
   <input type="text" name="q">
   <button type="submit">Search</button>
</form>
```

`POST` 요청을 하는 폼 예시는 다음과 같습니다:

```html
<form action="/submit" method="POST">
   <input type="text" name="username" required pattern="[A-Za-z0-9]{3,}" />
   <input type="password" name="password" />
   <button type="submit">Submit</button>
</form>
```

`GET` 또는 `POST` 요청 후 서버는 서버 측 유효성 검사를 확인하고 입력 값을 처리하거나 거부합니다.

유효성 검사가 성공하면 서버는 요청된 정보를 보내거나, 실패하면 오류를 발생시킵니다.

유효성 검사가 성공하면, 인증이나 검색의 경우 서버 응답은 리다이렉트일 수 있고, 작업 완료를 확인하는 성공 메시지일 수 있습니다.

폼 제출 시 일어나는 일을 요약하면 다음과 같습니다:

* 사용자가 폼 필드를 작성합니다.
    
* 브라우저가 클라이언트 측 유효성 검사(있다면)를 확인하고, 성공하면 진행합니다.
    
* 브라우저가 action과 method 속성에 따라 폼 데이터를 서버로 보냅니다.
    
* 서버가 데이터를 받아 처리합니다(데이터베이스에 저장, 사용자 등록, 로그인, 검색 쿼리 전송 등).
    
* 서버 측 유효성 검사가 있다면, 서버가 처리 전에 데이터를 다시 확인합니다.
    
* 서버가 성공 메시지, 오류, 또는 유효성 검사 상태(성공 또는 실패)에 따라 사용자를 리다이렉트합니다.
    
* 사용자는 서버 응답에 따른 피드백을 받습니다.
