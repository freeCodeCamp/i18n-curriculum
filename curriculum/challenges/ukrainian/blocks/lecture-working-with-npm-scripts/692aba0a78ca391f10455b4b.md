# --description--

У сучасній веб-розробці ви можете розбивати свій код на кілька файлів, які називаються «модулями».

Модуль можна уявити як автономний та багаторазово використовуваний блок коду.

Імпортуючи модулі в інші модулі, ви можете повторно використовувати їхню функціональність, при цьому зберігаючи код у різних файлах, що робить його більш підтримуваним, особливо для великих і складних веб-застосунків.

У Node.js кожен файл вважається модулем.

Node має дві різні системи для роботи з модулями: CommonJS та ES.

CommonJS — це оригінальний і за замовчуванням спосіб роботи з пакетами в Node.js. Він завантажує модулі синхронно, тобто має чекати, поки модуль повністю завантажиться, перш ніж почати завантаження наступного або продовжити виконання коду.

Він досі широко використовується, але поступово замінюється новішим синтаксисом ES.

CommonJS використовує функцію `require()` для імпорту модуля, приймаючи його відносний шлях як аргумент.

Наприклад, припустимо, що у нас є модуль під назвою `math.js`, де ми визначаємо базові математичні операції.

Тут ми використаємо лише функцію множення, щоб цей приклад був простим і сфокусованим:

```js
function multiply(a, b) {
  return a * b;
}
```

Як ми можемо повторно використовувати цю функцію поза межами нашого поточного модуля? За замовчуванням функції та змінні, визначені в модулі Node.js, є **приватними** — інші модулі не можуть отримати до них доступ, якщо ми явно не експортуємо їх. Існує кілька способів зробити це.

Один із загальних способів — призначити значення `module.exports`. Цей об’єкт представляє те, що модуль поверне, коли його імпортують в іншому місці, надаючи іншим модулям доступ до функцій, змінних або об’єктів, які ви вирішили відкрити.

У цьому прикладі ми призначаємо об’єкт із властивістю `multiply` для `module.exports`. Значення цієї властивості — референс на функцію `multiply`, яку ми визначили раніше. Це експортує функцію, тож ми зможемо викликати її з інших модулів, якщо потрібно.

```javascript
function multiply(a, b) {
  return a * b;
}

module.exports = {
  multiply: multiply
};
```

Припустимо, ми намагаємося імпортувати модуль `math.js` в інший модуль. Ми викликаємо функцію `require()` і передаємо відносний шлях як аргумент. Потім призначаємо об’єкт, який повертає `require()`, змінній `math`.

```javascript
const math = require('./math');
```

Перевіримо значення `math` у консолі, вивівши його за допомогою `console.log()`:

```javascript
console.log(math);
```

Ось вихідні дані. Це об’єкт із властивістю `multiply` і функцією як значенням:

```javascript
{ multiply: [Function: multiply] }
```

Якщо викликати цю функцію з `math.multiply()`, передаючи числа `6` і `7` як аргументи, вихідні дані будуть `42`. Функція успішно виконується, і при цьому вона знаходиться в іншому файлі! Отже, наш оператор імпорту спрацював правильно.

```javascript
const math = require('./math');

console.log(math.multiply(6, 7)); // 42
```

Але існують і інші способи експортувати функцію, змінну або об’єкт.

Наприклад, якщо потрібно експортувати лише одну функцію, змінну або об’єкт, ви можете призначити її безпосередньо як значення `module.exports`:

```javascript
function multiply(a, b) {
  return a * b;
}

module.exports = multiply;
```

Якщо потрібно експортувати кілька функцій, просто додайте їх до об’єкта `module.exports`, розділяючи комою.

Тут ви бачите, що експортований об’єкт має дві властивості: `subtract` і `multiply`. Їхні значення — референси на відповідні функції, тож ви матимете доступ до обох у будь-якому модулі, який імпортує `math.js`:

```javascript
function subtract(a, b) {
  return a - b;
}

function multiply(a, b) {
  return a * b;
}

module.exports = {
  subtract: subtract,
  multiply: multiply
};
```

Коли об’єкт `module.exports` має кілька властивостей, у вас буде можливість вибрати, чи імпортувати все одразу, чи імпортувати властивості окремо.

У цьому прикладі ми імпортуємо весь об’єкт, який експортував `math.js`, тож матимемо доступ і до `subtract`, і до `multiply`:

```javascript
const math = require('./math');

console.log(math.subtract(6, 7));  // -1
console.log(math.multiply(6, 7));  // 42
```

Якщо ми хочемо імпортувати лише `multiply`, потрібно просто написати його ім’я в фігурних дужках після ключового слова `const`. Фактично, ви використовуєте синтаксис деструктурованого присвоєння, щоб отримати його значення з експортованого об’єкта. Потім можна звертатися до нього безпосередньо за ім’ям у коді:

```javascript
const { multiply } = require('./math');

console.log(multiply(6, 7));
```

Це працює і для кількох елементів. Просто розділіть їх комою у фігурних дужках. Тут ми імпортуємо функції `subtract` і `multiply`:

```javascript
const { subtract, multiply } = require('./math');
```

За замовчуванням Node.js розглядає такі файли як модулі CommonJS:

* Файли з розширенням `.cjs`.
    
* Файли з розширенням `.js`, якщо найближчий батьківський файл `package.json` містить на верхньому рівні поле `"type"` зі значенням `"commonjs"`.
    
* Файли з розширенням `.js` або без розширення, якщо найближчий батьківський файл `package.json` не містить на верхньому рівні поля `"type"`, або якщо в жодній батьківській папці немає `package.json`.
    
* Файли з розширенням, яке не є `.mjs`, `.cjs`, `.json`, `.node` або `.js`.
    

Хоча ES-модулі поступово замінюють модулі CommonJS, розуміння останніх досі є важливим для розробників, які працюють із існуючими кодовими базами Node.js і багатьма пакетами npm.
