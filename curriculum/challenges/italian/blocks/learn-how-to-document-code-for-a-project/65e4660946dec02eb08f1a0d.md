# --description--

Un file `README` è una parte importante di molti progetti, specialmente nello sviluppo software. Spesso è il primo documento che le persone vedono quando guardano un progetto. Un file `README` spiega di cosa tratta il progetto, come usarlo e a volte come contribuire. Capire a cosa serve un file `README` è fondamentale nel contesto della documentazione di un progetto.

# --questions--

## --text--

A cosa serve comunemente un file `README` in un progetto?

## --answers--

Memorizzare codice

### --feedback--

Anche se il codice è una parte integrante di un progetto, il file `README` non serve a memorizzare codice ma a fornire informazioni sul progetto.

---

Istruzioni sul progetto

---

Informazioni di contatto

### --feedback--

Anche se le informazioni di contatto possono essere incluse, lo scopo principale di un file `README` è fornire istruzioni e informazioni sul progetto, non solo dettagli di contatto.

---

Registro delle modifiche

### --feedback--

Un registro delle modifiche, spesso chiamato changelog, è solitamente separato da un file `README`. Il file `README` si concentra sull’introduzione e la spiegazione del progetto.
