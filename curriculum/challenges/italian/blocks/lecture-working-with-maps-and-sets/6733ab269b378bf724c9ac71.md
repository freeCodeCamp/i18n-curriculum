# --description--

In JavaScript, `Set` è un oggetto integrato per gestire collezioni di dati. Ti permette di archiviare valori unici di qualsiasi tipo, sia primitivi sia riferimenti a oggetti. `Set` garantisce che ogni valore al suo interno appaia una sola volta, rendendolo utile per eliminare duplicati da un array o per gestire collezioni di valori distinti.

Per quanto riguarda `WeakSet`, è un tipo speciale di `Set` con meno funzionalità che consente di archiviare riferimenti a oggetti e simboli debolmente mantenuti. A differenza di `Set`, `WeakSet` non supporta primitivi come numeri o stringhe.

A differenza di un `Set` normale, un `WeakSet` memorizza solo oggetti, e i riferimenti a quegli oggetti sono "deboli", il che significa che i WeakSet non impediscono che gli oggetti memorizzati vengano raccolti dal garbage collector se non ci sono altri riferimenti a essi. In termini più semplici, se l’oggetto non viene usato altrove nel tuo codice, viene rimosso automaticamente per liberare memoria.

Per creare un `Set`, usi il costruttore `Set` e lo assegni a una variabile:

```js
const myFirstSet = new Set();
```

Puoi anche inizializzare il `Set` con dei valori:

```js
const treeSet = new Set(['Baobab', 'Jackalberry', 'Mopane Tree', 'Breadfruit']);
```

Se stampi il `Set` nella console, questo è l’output che ottieni:

```js
/*
Set(4) {'Baobab', 'Jackalberry', 'Mopane Tree', 'Breadfruit'}
  [[Entries]]
  0: "Baobab"
    value: "Baobab"
  1: "Jackalberry"
    value: "Jackalberry"
  2: "Mopane Tree"
    value: "Mopane Tree"
  3: "Breadfruit"
    value: "Breadfruit"
  size: 4
  [[Prototype]]: Set
*/
```

Se non hai inizializzato il `Set` con valori, puoi usare il metodo `add()` per aggiungere un elemento al `Set`:

```js
const treeSet = new Set();

// Add items to the treeSet
treeSet.add('Baobab');
treeSet.add('Jackalberry');
treeSet.add('Mopane Tree');
treeSet.add('Breadfruit');
```

Il risultato e l’aspetto in console rimangono gli stessi.

Non dimenticare che gli elementi duplicati verranno ignorati nel `Set`:

```js
const treeSet = new Set();

// Add items to the treeSet
treeSet.add('Baobab');
treeSet.add('Jackalberry');
treeSet.add('Mopane Tree');
treeSet.add('Breadfruit');
treeSet.add('Baobab'); //duplicate item will be ignored

console.log(treeSet);
// Set(4) {'Baobab', 'Jackalberry', 'Mopane Tree', 'Breadfruit'}
```

Gli altri metodi che puoi usare per manipolare un `Set` sono:

- `delete()`  
- `clear()`
- `has()`
- `entries()`
- `forEach()`
- `keys()`
- `values()`

Vediamo come funzionano uno per uno. `delete()` rimuove un elemento specificato dal `Set`:

```js
const treeSet = new Set();

// Add items to the treeSet
treeSet.add('Baobab');
treeSet.add('Jackalberry');
treeSet.add('Mopane Tree');
treeSet.add('Breadfruit');

treeSet.delete('Breadfruit');

console.log(treeSet); // Set(3) {'Baobab', 'Jackalberry', 'Mopane Tree'}
```

`has()` verifica se un valore specificato esiste nel `Set`:

```js
const treeSet = new Set();

// Add items to the treeSet
treeSet.add('Baobab');
treeSet.add('Jackalberry');
treeSet.add('Mopane Tree');
treeSet.add('Breadfruit');

treeSet.delete('Breadfruit');

console.log(treeSet.has('Breadfruit')); // false
```

`entries()` restituisce un iteratore `Set` contenente un array dei valori in formato `[value, value]`:

```js
const treeSet = new Set();

// Add items to the treeSet
treeSet.add('Baobab');
treeSet.add('Jackalberry');
treeSet.add('Mopane Tree');
treeSet.add('Breadfruit');

treeSet.delete('Breadfruit');

console.log(treeSet.entries());
// SetIterator {'Baobab' => 'Baobab', 'Jackalberry' => 'Jackalberry', 'Mopane Tree' => 'Mopane Tree'}
```

`keys()` e `values()` mostrano i valori nel `Set`. `keys()` è solo un alias per il metodo `values()`:

```js
const treeSet = new Set();

// Add items to the treeSet
treeSet.add('Baobab');
treeSet.add('Jackalberry');
treeSet.add('Mopane Tree');
treeSet.add('Breadfruit');

treeSet.delete('Breadfruit');

console.log('Keys: ', treeSet.keys());
console.log('Values: ', treeSet.values());
// Keys: SetIterator {'Baobab', 'Jackalberry', 'Mopane Tree'}
// Values: SetIterator {'Baobab', 'Jackalberry', 'Mopane Tree'}
```

`forEach()` ti permette di iterare sul `Set`:

```js
const treeSet = new Set();

// Add items to the treeSet
treeSet.add('Baobab');
treeSet.add('Jackalberry');
treeSet.add('Mopane Tree');
treeSet.add('Breadfruit');

treeSet.delete('Breadfruit');

treeSet.forEach((tree) => console.log(tree));
/*
Baobab
Jackalberry
Mopane Tree
*/
```

`clear()` rimuove tutti gli elementi dell’array:

```js
const treeSet = new Set();

// Add items to the treeSet
treeSet.add('Baobab');
treeSet.add('Jackalberry');
treeSet.add('Mopane Tree');
treeSet.add('Breadfruit');

treeSet.delete('Breadfruit');

treeSet.clear();

console.log(treeSet); // Set(0) {size: 0}
```

Vale anche la pena menzionare che esiste una proprietà `size` che restituisce il numero di elementi nel `Set`:

```js
const treeSet = new Set();

// Add items to the treeSet
treeSet.add('Baobab');
treeSet.add('Jackalberry');
treeSet.add('Mopane Tree');
treeSet.add('Breadfruit');

treeSet.delete('Breadfruit');

console.log(treeSet.size); // 3
```

Proprio come `Set`, esiste anche un costruttore `WeakSet` che puoi usare per creare un `WeakSet`:

```javascript
const treeWeakSet = new WeakSet();
```

`WeakSet` ha anche i metodi `add()`, `delete()` e `has()`:

```javascript
const treeWeakSet = new WeakSet();

treeWeakSet.add({ name: 'Baobab' });
treeWeakSet.add({ name: 'Jackalberry' });
treeWeakSet.add({ name: 'Mopane Tree' });
treeWeakSet.add({ name: 'Breadfruit' });

treeWeakSet.delete('Jackalberry');
console.log(treeWeakSet.has('Jackalberry')); // false

console.log(treeWeakSet);
```

Nell’output, il contenuto del `WeakSet` appare così:

```javascript
/*
WeakSet {{…}, {…}, {…}, {…}}
  [[Entries]]
    No properties
  [[Prototype]]: WeakSet
    .
    .
    .
*/
```

Il contenuto appare vuoto perché i WeakSet non sono iterabili e non espongono direttamente il loro contenuto.

Non dimenticare che sono supportati solo simboli e oggetti con chiavi e valori ben definiti. Aggiungere un primitivo, come numeri o stringhe, genera un errore:

```js
treeWeakSet.add('Alan Smith');

console.log(treeWeakSet); // Invalid value used in weak set
//    at WeakSet.add (<anonymous>)
```

La differenza chiave tra un `Set` e un `WeakSet` è che un `Set` può memorizzare qualsiasi valore, mentre un `WeakSet` può memorizzare solo oggetti.

Ecco alcune altre differenze evidenti tra un `Set` e un `WeakSet`:

| Funzionalità | Set | WeakSet |
| --- | --- | --- |
| Tipo di valori memorizzati | Memorizza qualsiasi tipo di dato | Memorizza solo oggetti |
| Riferimenti | Riferimenti forti | Riferimenti deboli |
| Iterazione | Supporta l’iterazione con `forEach` e loop | Non supporta l’iterazione |
| Metodi e proprietà | `add()`, `delete()`, `has()`, `keys()`, `values()`, `size` e altri | Solo `add()`, `delete()` e `has()` |
| Caso d’uso | Collezione generica di valori unici e rimozione duplicati da array | Tracciamento efficiente in memoria di riferimenti a oggetti |

Puoi vedere le differenze nei tipi di valori che i due tipi di insieme possono memorizzare, il loro supporto per iterare sugli oggetti memorizzati e i loro casi d’uso ideali. Prenditi un momento per leggere il contenuto di questa tabella.
