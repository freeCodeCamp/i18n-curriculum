# --interactive--

CSS-Spezifität ist ein grundlegendes Konzept, das bestimmt, welche Styles auf ein Element angewendet werden, wenn mehrere Regeln zutreffen könnten.

Das Verständnis der Spezifität hilft Entwicklern, Konflikte zwischen verschiedenen CSS-Regeln zu lösen und sicherzustellen, dass die gewünschten Styles konsequent angewendet werden.

Die CSS-Spezifität wird basierend auf der Art der verwendeten Selektoren berechnet.

Die höchste Spezifität wird inline-Styles zugeschrieben, die direkt über das `style`-Attribut auf ein Element angewendet werden.

In diesem Beispiel wird der erste Absatz rot sein, während die anderen `p`-Elemente blau sind. Das liegt daran, dass inline-Styles eine höhere Spezifität haben als Typselektoren wie der `p`-Selektor, der in der `styles.css`-Datei gezeigt wird.

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<p style="color: red;">Red paragraph</p>
<p>Other paragraph</p>
<p>Another paragraph</p>
<p>Yet another paragraph</p>
```

```css
p {
  color: blue;
}
```

:::

Danach haben ID-Selektoren eine hohe Spezifität.

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<p id="para">Red paragraph</p>
<p>Other paragraph</p>
<p>Another paragraph</p>
<p>Yet another paragraph</p>
```

```css
#para {
  color: red; 
}

p {
  color: blue;
}
```

:::

Als Nächstes kommen Klassen-Selektoren, Attribut-Selektoren und Pseudoklassen ins Spiel.

Beispiele sind Klassen-Selektoren wie `.container` und `.button`, Attribut-Selektoren wie `[type="text"]` und Pseudoklassen wie `:hover`. Diese haben eine mittlere Spezifität.

**HINWEIS**: Sie werden in zukünftigen Lektionen mehr über Pseudoklassen lernen.

In diesem Beispiel wird der erste Absatz rot sein, weil ein ID-Selektor eine höhere Spezifität hat als Klassen- und Typselektoren. Die `.example-para`-Elemente werden grün sein, während die übrigen Absatz-Elemente blau gefärbt sind.

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<p id="para">Example paragraph</p>
<p class="example-para">Other paragraph</p>
<p class="example-para">Another paragraph</p>
<p>Yet another paragraph</p>
```

```css
#para {
  color: red; 
}

.example-para {
  color: green;
}

p {
  color: blue;
}
```

:::

Typselektoren wie `div` und `h1` sowie Pseudoelemente wie `::before` und `::after` haben eine geringere Spezifität im Vergleich zu den vorherigen Gruppen.

**HINWEIS**: Sie werden in zukünftigen Lektionen mehr über Pseudoelemente lernen.

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<p id="para">Example paragraph</p>
<p>Other paragraph</p>
<p>Yet another paragraph</p>
```

```css
#para {
  color: red; 
}

p {
  color: blue;
}
```

:::

Schließlich hat der universelle Selektor, dargestellt durch ein Sternchen `*`, die niedrigste Spezifität von allen.

Hier ist ein Beispiel, wie Sie mit dem `*`-Selektor die Farbe aller Elemente auf Rot setzen. Sie werden jedoch keine roten Elemente sehen, da ID- und Typselektoren diese Stile überschreiben, was die geringe Spezifität des `*`-Selektors verdeutlicht.
:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<p id="para">Example paragraph</p>
<p>Other paragraph</p>
<p>Yet another paragraph</p>
```

```css
* {
  color: red;
}

#para {
  color: green; 
}

p {
  color: blue;
}
```

:::

Spezifitätswerte werden als vierteiliger Wert `(a, b, c, d)` berechnet:

- `a`: Inline-Styles (`1` oder `0`).
- `b`: Anzahl der ID-Selektoren.
- `c`: Anzahl der Klassen-Selektoren, Attribut-Selektoren und Pseudoklassen.
- `d`: Anzahl der Typselektoren, Pseudoelemente und universellen Selektoren.

Jeder Teil des Spezifitätswerts hat ein unterschiedliches Gewicht:

- **Inline-Styles** (`a`) haben das höchste Gewicht und tragen mit einem Wert von `1` zum ersten Teil des Spezifitätswerts bei.
    
- **ID-Selektoren** (`b`) haben das nächsthöhere Gewicht, wobei jeder ID-Selektor `1` zum zweiten Teil des Spezifitätswerts beiträgt.
    
- **Klassen-Selektoren, Attribut-Selektoren und Pseudoklassen** (`c`) haben ein mittleres Gewicht, wobei jeder `1` zum dritten Teil des Spezifitätswerts beiträgt.
    
- **Typselektoren und Pseudoelemente** (`d`) haben das niedrigste Gewicht, wobei jeder `1` zum vierten Teil des Spezifitätswerts beiträgt.
    
- **Universeller Selektor** (`*`): Der universelle Selektor trägt `0` zur Spezifitätsberechnung bei und beeinflusst die Spezifität nicht. Seine Einbeziehung in einen Selektor ändert den Spezifitätswert nicht.

Inline-CSS hat die höchste Spezifität, weil es direkt auf das Element angewendet wird. Es überschreibt jedes interne oder externe CSS. Der Spezifitätswert für Inline-Styles ist `(1, 0, 0, 0)`.

Hier ist ein weiteres Beispiel für einen Inline-Style:

:::interactive_editor

```html
<p style="color: red;">This text is red.</p>
```

:::

Internes CSS wird innerhalb eines `style`-Elements im `head`-Abschnitt des HTML-Dokuments definiert. Es hat eine geringere Spezifität als Inline-Styles. Sowohl internes als auch externes CSS haben das gleiche Spezifitätsniveau – die Spezifität wird durch die verwendeten Selektoren bestimmt, nicht durch den Ort, an dem das CSS definiert ist. Wenn zwei Regeln die gleiche Spezifität haben, gewinnt diejenige, die später im Dokument erscheint. Wenn also ein externes Stylesheet nach einem internen `style`-Block verlinkt wird, haben die externen Regeln Vorrang. Wenn es davor verlinkt wird, gewinnen die internen Regeln.

Der Spezifitätswert für interne Styles wird durch die verwendeten Selektoren bestimmt. Zum Beispiel hat ein ID-Selektor innerhalb von internem CSS einen Spezifitätswert von `(0, 1, 0, 0)`.

:::interactive_editor

```html
<head>
  <style>
    #text {
      color: blue;
    }
  </style>
</head>
<body>
  <div id="text">This text is blue.</div>
</body>
```

:::

In diesem Beispiel wird der Text blau sein, es sei denn, es wird ein Inline-Style oder ein spezifischerer Selektor angewendet.

Externes CSS wird über ein `link`-Element im `head`-Abschnitt verlinkt und in separaten `.css`-Dateien geschrieben. Wie bei internem CSS wird die Spezifität externer Styles durch die verwendeten Selektoren bestimmt. Externes CSS bietet die beste Wartbarkeit für größere Projekte.

Der Spezifitätswert für externe Styles wird ebenfalls durch die verwendeten Selektoren bestimmt. Zum Beispiel hat ein Klassen-Selektor innerhalb von externem CSS einen Spezifitätswert von `(0, 0, 1, 0)`.

:::interactive_editor

```html
<head>
  <link rel="stylesheet" href="styles.css">
</head>
<body>
  <div class="text">This text's color is defined in an external CSS file.</div>
</body>
```

```css
.text {
  color: purple;
}
```

:::

In diesem Beispiel ist die Textfarbe in der `styles.css`-Datei definiert und wird angewendet, sofern sie nicht von einem spezifischeren Selektor oder einem Inline-Style überschrieben wird.
