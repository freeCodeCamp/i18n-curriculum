# --description--

Cookies, auch bekannt als Web-Cookies oder Browser-Cookies, sind kleine Datenstücke, die ein Server an den Webbrowser eines Nutzers sendet. Diese Cookies werden auf dem Gerät des Nutzers gespeichert und bei nachfolgenden Anfragen an den Server zurückgesendet.

Cookies sind entscheidend, damit Webanwendungen den Zustand aufrechterhalten und Benutzerinformationen speichern können, was besonders wichtig ist, da HTTP ein zustandsloses Protokoll ist.

Cookies können verschiedene Informationen speichern, wie Benutzerpräferenzen, Sitzungsdaten oder Tracking-Informationen.

Cookies werden immer als Name-Wert-Paare gespeichert. Das bedeutet, dass jedes Cookie einen Namen (Schlüssel) und einen zugehörigen Wert hat.

Zum Beispiel könnte ein Cookie die Sitzungs-ID eines Nutzers so speichern: `sessionId=abc123`. Der Schlüssel ist `sessionId` und der Wert `abc123`. Jedes Mal, wenn der Browser mit dem Server kommuniziert, sendet der Browser diese Cookies in Form von Name-Wert-Paaren.

Wenn ein Nutzer eine Webseite besucht, kann der Server ein oder mehrere Cookies an den Browser des Nutzers senden, indem er einen Set-Cookie-Kopfzeile in der HTTP-Antwort einschließt. Eine Kopfzeile ist ein Schlüssel-Wert-Paar, das zusätzliche Informationen über die HTTP-Anfrage oder -Antwort liefert. Mehr über HTTP-Anfragen und -Antworten lernen Sie in zukünftigen Lektionen.

Sobald die Cookies gesetzt sind, speichert der Browser sie und fügt sie automatisch in der Cookie-Kopfzeile bei jeder nachfolgenden Anfrage an dieselbe Domain hinzu. So kann der Server auf die gespeicherten Cookies zugreifen und sie für Dinge wie das Aufrechterhalten von Benutzersitzungen oder das Verfolgen von Präferenzen verwenden.

Hier ist ein Beispiel, wie ein Cookie in einer HTTP-Antwort gesetzt wird:

```http
Set-Cookie: sessionId=abc123; Expires=Wed, 21 Oct 2021 07:28:00 GMT; Secure; HttpOnly
```

Der Browser speichert das Cookie, und bei zukünftigen Anfragen an denselben Server wird das Cookie in der `Cookie`-Kopfzeile mitgesendet:

```http
Cookie: sessionId=abc123
```

Der Server kann dann das Cookie lesen und die gespeicherte Sitzungs-ID verwenden, um Informationen über den Nutzer abzurufen, zum Beispiel ob er eingeloggt ist.

Hier ist eine Übersicht über die verschiedenen Arten von Cookies.

- Session-Cookies bestehen nur für die Dauer der Sitzung des Nutzers auf der Webseite. Sobald der Nutzer den Browser oder Tab schließt, wird das Session-Cookie gelöscht. Diese Cookies werden typischerweise für Aufgaben wie das Eingeloggtbleiben während des Besuchs verwendet.

- Persistent-Cookies haben ein Ablaufdatum und bleiben auf dem Gerät des Nutzers gespeichert, bis dieses Datum erreicht ist. Persistent-Cookies werden oft verwendet, um Benutzerpräferenzen oder Login-Daten über Sitzungen hinweg zu speichern.

- Secure-Cookies werden nur über HTTPS gesendet, wodurch sichergestellt wird, dass sie unterwegs nicht von Angreifern abgefangen werden können.

- HttpOnly-Cookies können von JavaScript, das im Browser läuft, nicht aufgerufen oder verändert werden, was sie sicherer gegen Cross-Site-Scripting-(XSS-)Angriffe macht. Cross-Site-Scripting-Angriffe passieren, wenn ein Angreifer bösartige Skripte in eine Webseite einschleust, die von anderen Nutzern angesehen wird. Diese Skripte können dann im Kontext des Browsers des Opfers ausgeführt werden, womöglich Cookies, Sitzungsdaten stehlen oder andere schädliche Aktionen ohne Wissen oder Zustimmung des Nutzers ausführen. Indem Cookies als HttpOnly markiert werden, sind sie davor geschützt, über JavaScript aufgerufen zu werden, was das Risiko solcher Angriffe verringert.

Sie können Cookies über Serverantworten mit der `Set-Cookie`-Kopfzeile oder über JavaScript mit `document.cookie` erstellen.

Hier ist ein Beispiel, wie man ein Cookie mit JavaScript setzt:

```js
document.cookie = "username=JohnDoe; expires=Fri, 31 Dec 2021 23:59:59 GMT; path=/";
```

Dieser Befehl setzt ein Cookie mit dem Namen `username` und dem Wert `"JohnDoe"`, das bis Ende 2021 abläuft. Sie können ein bestehendes Cookie aktualisieren, indem Sie es einfach erneut mit einem neuen Wert setzen.

Um ein Cookie zu löschen, setzen Sie sein Ablaufdatum in die Vergangenheit:

```js
document.cookie = "username=; expires=Thu, 01 Jan 1970 00:00:00 GMT; path=/";
```

Dadurch wird das `username`-Cookie aus dem Browser entfernt.
