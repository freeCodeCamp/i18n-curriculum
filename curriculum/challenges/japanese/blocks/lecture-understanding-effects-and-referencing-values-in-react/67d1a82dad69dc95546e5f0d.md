# --description--

React では、DOM 要素に直接アクセスする必要がある場合があります。そんなときに便利なのが「refs」です。refs は変更可能な値を保存することもできますが、その場合は state を使うほうが適しています。

バニラ JavaScript では、`getElementById()` と `querySelector()` メソッドを使って DOM 要素にアクセスしていました。しかし React では、refs を使って DOM の要素にアクセスします。

refs の大きな違いの一つは、ID やクラスのような識別子を使わずに要素を参照できることです。

では、refs をどのように作成して使うのでしょうか？React はそれを可能にする `useRef()` フックを提供しています。

まず最初に、React からそのフックをインポートします。

```js
import { useRef } from "react";
```

次に、`useRef` フックの中で初期値を持つ ref を保持する変数を作成します。例えば、`sectionRef` を `null` で初期化します。

```js
const sectionRef = useRef(null);
```

最後に、JSX 内の要素に `ref` 属性を使って ref 変数を割り当てます。

```jsx
<section ref={sectionRef}>
  {/* Section content */}
</section>
```

ref をコンソールにログ出力すると、current 値を持つオブジェクトであることがわかります。ここでは `null` です。

```js
console.log(sectionRef); // { current: null }
```

また、`current` プロパティを使って current の値を直接コンソールにログ出力することもできます。

```js
console.log(sectionRef.current); // null
```

ref のその後の値はコンポーネントのライフサイクルに依存します。

例えば、`sectionRef` の初期値は常に `null` です。これは初期化時の値だからです。コンポーネントがマウントされた後は、ref の値は ref が割り当てられた `section` 要素になります。

コンポーネントがアンマウントされると、ref の値は初期値の `null` に戻ります。

ref の典型的な例としては、レンダリング時やボタンをクリックしたときに入力要素にフォーカスを当てることがあります。

ボタンをクリックしたときにそれを行う方法は以下の通りです。

```jsx
import { useRef } from "react";

const Focus = () => {
  const inputRef = useRef(null);

  const handleFocus = () => {
    if (inputRef.current) {
      inputRef.current.focus();
    }
  };

  return (
    <div>
      <input ref={inputRef} type="text" placeholder="Enter text" />
      <button onClick={handleFocus}>Focus Input</button>
    </div>
  );
};

export default Focus;
```

上記のコードでは、`inputRef` が作成され `input` 要素に割り当てられています。また、`onClick` イベントを持つボタンがあり、`handleFocus` 関数を呼び出します。

`handleFocus` 関数は `focus()` 要素の `input` メソッドを呼び出すだけです。`input` は React に組み込みのコンポーネントなので、実際の `input` DOM 要素は ref の `current` プロパティに設定されます。したがって `focus()` を使って `input.current.focus()` メソッドを呼び出します。

refs を使う際に知っておくべきベストプラクティスは以下の通りです。

- refs は主に DOM とやり取りするために使ってください。変更可能なデータにも使えますが、その場合は state を使うほうがよいです。

- 基本的な状態管理には refs を使わないでください。これは `useState` の役割です。

- `ref.current` が存在するかどうかを必ず確認してから、そのプロパティにアクセスしてください。確認方法は以下の通りです。

```js
const handleFocus = () => {
  if (inputRef.current) {
    inputRef.current.focus();
  }
};
```

これにより、ref が DOM に割り当てられる前や削除された後にアクセスしてもエラーが発生しません。
