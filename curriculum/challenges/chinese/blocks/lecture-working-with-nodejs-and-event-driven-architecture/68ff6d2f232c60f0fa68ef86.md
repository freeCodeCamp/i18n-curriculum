# --description--

到目前为止，你一直在使用 JavaScript 编写客户端代码，这些代码运行在网络应用的客户端，通常是网页浏览器。

客户端 JavaScript 可以非常强大，但对于更复杂的任务确实存在某些限制。

例如：

- 它对局部的文件有非常严格的访问权限。
- 它并非真正为处理复杂应用逻辑而设计。
- 而且它存在潜在的安全性问题。例如，你不希望将敏感信息，如数据库凭据，暴露给客户端代码。
    
客户端 JavaScript 有这些限制，因为它最初设计为专门在网页浏览器上 `Run`。

网页浏览器提供运行 JavaScript 代码所需的环境，包括 JavaScript 引擎。它们还提供对文档对象模型（DOM）的访问，因此你可以在代码中访问 HTML 元素。

以前，我们无法在浏览器外运行 JavaScript，但现在我们可以使用 Node.js。

Node.js 是一个 JavaScript 运行期环境。它被官方定义为“一个开源且跨平台的 JavaScript 运行期环境”。

让我们看看这意味着什么：

- Node.js 是开源的，因为它的代码公开可用，并由大量开发者社区维护。
- 它是跨平台的，因为它可以在任何操作系统上运行，包括 Windows、macOS 和 Linux。
- 它是一个 JavaScript 运行期环境，因为它允许你在浏览器外运行 JavaScript 代码。
    
Node.js 被广泛应用于各行各业。开发者使用它来构建处理超文本传输协议请求的网络服务器和 API，以及网页和移动应用。

如果你正在使用或开发一个与数据库交互或处理复杂数据的网站或网页应用，那么很有可能你会在某个阶段在后台使用 Node.js。

它高效、可扩展，并且拥有全球大量的开发者和维护者社区。

既然你知道了什么是 Node.js，接下来让我们看看浏览器和 Node 运行期环境之间的一些区别。

浏览器环境主要为前端网页开发设计，因此它运行客户端 JavaScript。相比之下，Node 运行期环境主要为后端网页开发设计，因此它运行服务器端 JavaScript。这也决定了它们提供的 API。

例如，从浏览器中，你可以访问 DOM API，但访问局部的文件系统有一定限制。相比之下，从 Node 运行期环境中，你可以访问几乎所有系统资源，包括文件系统，但不能访问 DOM。

两个 JavaScript 环境都提供一个全局对象。在浏览器 JavaScript 运行期中，这个对象称为 `window`。这个 `window` 对象提供对浏览器相关功能的访问，例如访问用于操作 DOM、管理 cookie 和处理浏览器事件的方法。在 Node.js 环境中，这个全局对象称为 `global`。`global` 对象提供对 Node.js 特定功能的访问，例如用于处理局部文件、网络和与操作系统交互的内置模块。

最后，另一个重要的区别是你可以选择在你的服务器中使用的 Node.js 版本。然而，你无法控制用户访问你的网站时所使用的浏览器环境版本。

Node.js 已经成为全球开发者非常重要的工具。它是一个强大、可扩展且多功能的工具，通过使开发者能够使用 JavaScript 构建全栈应用的前端和后端，彻底改变了网页开发。这使得开发过程更加高效，因为开发者不需要为了开发后端而学习新的编程语言。
