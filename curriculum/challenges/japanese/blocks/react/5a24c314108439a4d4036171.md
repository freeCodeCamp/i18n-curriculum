# --description--

コンポーネントの初期 `state` を定義すると、レンダリングされる UI の任意の部分にその値を表示できます。コンポーネントがステートフルであれば、常に `state` のデータに `render()` メソッド内でアクセスできます。`this.state` を使ってデータにアクセスできます。

レンダーメソッドの `return` 内で `state` の値にアクセスしたい場合は、その値を波括弧で囲む必要があります。

`state` は React のコンポーネントにおける最も強力な機能の一つです。アプリ内の重要なデータを追跡し、そのデータの変化に応じて UI をレンダリングできます。データが変われば UI も変わります。React は仮想 DOM と呼ばれる仕組みを使い、裏側で変更を管理しています。`state` のデータが更新されると、そのデータを使っているコンポーネント（そのデータをプロップとして受け取った子コンポーネントも含む）が再レンダリングされます。React は実際の DOM を更新しますが、必要な部分だけに限定されます。つまり、DOM の変更を気にする必要はなく、UI がどうあるべきかを宣言するだけで済みます。

コンポーネントをステートフルにしても、他のコンポーネントはその `state` を知りません。`state` は完全にカプセル化されており、そのコンポーネント内だけでローカルに管理されます。ただし、`state` のデータを子コンポーネントに `props` として渡す場合は別です。このカプセル化された `state` の概念は非常に重要で、特定のロジックを一箇所にまとめて隔離できるようにします。

# --instructions--

コードエディターでは、`MyComponent` はすでにステートフルになっています。コンポーネントのレンダーメソッド内に `h1` タグを定義し、コンポーネントの `state` から `name` の値をレンダリングしてください。

**注意:** `h1` は `state` の値だけをレンダリングし、それ以外はレンダリングしないようにしてください。JSX では、波括弧で囲んだコード `{ }` は JavaScript として扱われます。したがって、`state` から値にアクセスするには、その参照を波括弧で囲んでください。

# --hints--

`MyComponent` は、`state` に `name` というキーを持ち、その値は `freeCodeCamp` である必要があります。

```js
assert(
  Enzyme.mount(React.createElement(MyComponent)).state('name') ===
    'freeCodeCamp'
);
```

`MyComponent` は、単一の `div` で囲まれた `h1` 見出し要素をレンダリングする必要があります。

```js
assert(
  /<div><h1>.*<\/h1><\/div>/.test(
    Enzyme.mount(React.createElement(MyComponent)).html()
  )
);
```

レンダリングされる `h1` 見出し要素は、コンポーネントの `state` からレンダリングされたテキストだけを含むべきです。

```js
  const waitForIt = (fn) =>
    new Promise((resolve, reject) => setTimeout(() => resolve(fn()), 250));
  const mockedComponent = Enzyme.mount(React.createElement(MyComponent));
  const first = () => {
    mockedComponent.setState({ name: 'TestName' });
    return waitForIt(() => mockedComponent.html());
  };
  const firstValue = await first();
  const getValue = firstValue.replace(/\s/g, '');
  assert(getValue === '<div><h1>TestName</h1></div>');
```
