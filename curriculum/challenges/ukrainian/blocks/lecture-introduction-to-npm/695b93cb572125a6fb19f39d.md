# --description--

Підтримувати залежності в актуальному стані — і знати, як встановити конкретну версію — важливо для стабільності вашого проєкту. Це допомагає уникнути несподіваних помилок і дає більше контролю, коли виходять нові версії.

Щоб встановити конкретну версію пакета, використовуйте такий формат:

```bash
npm install package@version-number
```

Наприклад, якщо вашому проєкту потрібна стара версія Express, скажімо `4.21.2`, ви можете встановити її так:

```bash
npm install express@4.21.2
```

Якщо не вказувати номер версії, npm автоматично встановить найновішу доступну версію.

Якщо ви не впевнені, які версії існують, можна перевірити їх за допомогою цієї команди:

```bash
npm view package-name versions
```

Наприклад:

```bash
npm view express versions
```

Щоб оновити пакети, можна оновлювати їх по одному або всі одразу. Перед цим корисно спочатку побачити, які з них застаріли:

```bash
npm outdated
```

Це покаже, які залежності мають новіші версії, і ви побачите вихідні дані, схожі на такі:

```bash
user@Kolade ~/Desktop/fCC/script-code/node/npm/install-spec-version  % npm outdated
Package  Current  Wanted  Latest  Location              Depended by
chalk      5.5.0   5.5.0   5.6.2  node_modules/chalk    install-spec-version
cleave     1.0.1   1.0.1   1.0.2  node_modules/cleave   install-spec-version
express   4.21.2  4.21.2   5.2.1  node_modules/express  install-spec-version
```

Після того, як ви перевірили, які залежності застаріли, можна починати їх оновлювати.

Щоб оновити одну залежність, використовуйте таку команду:

```bash
npm update package-name
```

Це оновить пакет лише в межах версійного діапазону, дозволеного вашим файлом `package.json`. Наприклад, щоб оновити Express, можна використати таку команду:

```bash
npm update express
```

Якщо хочете пропустити версійний діапазон і встановити найновішу версію пакета, використовуйте таку команду:

```bash
npm install package-name@latest
```

Щоб оновити всі залежності, які потрапляють у дозволені версійні діапазони, запустіть:

```bash
npm update
```

Однак зверніть увагу, що `npm update` не оновлює пакет до нової мажорної версії, якщо ваш `package.json` цього не дозволяє. Мажорні версії часто містять сумісні зміни, тому npm не застосовує їх автоматично.

Якщо ви хочете саме найновішу мажорну версію, потрібно виконати:

```bash
npm install package-name@latest
```

Це змусить npm встановити найновіший реліз, незалежно від версійного діапазону у вашому `package.json`.
