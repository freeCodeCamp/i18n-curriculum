# --interactive--

在建置網頁表單時，輸入元素的外觀和行為非常重要。具有回應使用者操作的輸入欄位的表單，能大幅提升使用者體驗，且不應造成混淆或挫折感。

CSS 提供了幾個輸入偽類，能讓你的表單更直覺且更友善於使用者。讓我們詳細看看這些偽類。

`:focus` 偽類允許你在使用者選取或點擊輸入元素時，將注意力集中在作用中的輸入欄位。這有助於使用者輕鬆識別他們目前正在輸入的位置：

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css" />
<form>
  <input type="text" />
</form>
```

```css
input:focus {
  border: 2px solid crimson;
  outline: none;
}
```

:::

顧名思義，當使用者將游標暫留在元素上時，會觸發 `:hover` 偽類別。它在使用者與輸入交談之前提供視覺回授，有效地提醒他們該輸入已準備好進行操作。

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css" />
<form>
  <input type="text" />
</form>
```

```css
input:hover {
  background-color: orange;
}
```

:::

在上述範例中，當使用者暫留在輸入元素上時，輸入元素的背景會變成橘色，讓他們知道該欄位已準備好進行互動。

`:checked` 偽類讓你在使用者選取時，對圓鈕或核取方塊套用不同的樣式。這樣，使用者就能輕鬆看出他們選擇了哪個選項。

這裡有一個範例，包含一個用於同意網站條款的核取方塊。 

**注意**：此範例中的部分 CSS 使用了尚未涵蓋的屬性。這只是為了讓你了解如何創建訂製的核取方塊。你將在未來的課程和工作坊中學習這些內容的運作方式。 

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css" />
<form>
  <label>
  Agree <input class="checkbox" type="checkbox" />
  </label>
</form>
```

```css
.checkbox {
  appearance: none;
  width: 18px;
  height: 18px;
  border: 2px solid #ccc;
  border-radius: 4px;
  display: inline-block;
  position: relative;
  cursor: pointer;
  transition: all 0.25s ease;
  vertical-align: middle; 
}

.checkbox:hover {
  border-color: #888;
}

.checkbox:checked {
  background-color: #4caf50;
  border-color: #4caf50;
}

.checkbox:checked::after {
  content: "";
  position: absolute;
  left: 4px;
  top: 0px;
  width: 5px;
  height: 10px;
  border: solid white;
  border-width: 0 2px 2px 0;
  transform: rotate(45deg);
}

.checkbox:focus {
  outline: 2px solid #90caf9;
  outline-offset: 2px;
}

```

:::

`:required` 標的為具有 `required` 屬性的輸入元素。它向使用者發出必須填寫該欄位才能提交表單的信號。

這裡有一個範例，包含一些必填輸入欄的表單：

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css" />
<form action="#">
  <label for="name">Name:</label>
  <input type="text" id="name" name="name" required />

  <label for="email">Email:</label>
  <input type="email" id="email" name="email" required />

  <label for="phone">Phone Number:</label>
  <input type="tel" id="phone" name="phone" />

  <button type="submit">Submit</button>
</form>
```

```css
input:required {
  border: 2px solid orange;
}
```

:::

在驗證表單時，你可以使用 `:valid` 偽類別來設定符合驗證條件的輸入欄位樣式，並使用 `:invalid` 來設定不符合條件的輸入欄位樣式。通常，你會為有效輸入使用綠色，為無效輸入使用紅色。 

以下是使用 `:valid` 偽類的範例：

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css" />
<form>
  <label for="email">Email:</label>
  <input type="email" id="email" name="email" />
</form> 
```

```css
input:valid {
  border-color: green;
}
```

:::

以下是使用 `:invalid` 偽類的範例：

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css" />
<form>  
  <label for="email">Email:</label>
  <input type="email" id="email" name="email" />
</form>
```

```css
input:invalid {
  border-color: crimson;
}
```

:::

`:disabled` 偽類允許你選取並樣式化被停用的輸入元素。這些元素具有 `disabled` 屬性，該屬性防止使用者與其互動。當輸入被停用時，無法點擊、聚焦或編輯。

這裡是一個標籤和停用輸入元素的範例。

在 CSS 中，我們針對該停用的輸入欄位設定 `lightgray` 的背景色，並將游標指標改為 `not-allowed`。 

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css" />
<form>
  <label for="name">Name:</label>
  <input class="text-input" type="text" id="name" name="name" disabled />
</form>
```

```css
.text-input:disabled {
  background-color: lightgray;
  cursor: not-allowed;
}
```

:::

`cursor: not-allowed;` CSS 屬性值會改變游標的外觀，以顯示該動作不被允許。 

當套用到元素時，它會顯示一個帶有圓圈斜線符號（通常是一個帶有對角線的圓圈）的游標，以向使用者表示該元素不可互動或無法點擊或互動。

以下是一些其他輸入偽類別及其功能的範例：

- `:autofill`：將樣式套用到瀏覽器自動填入已儲存資料的輸入欄位。
- `:optional`：為非必填且可留空的輸入元素套用樣式。
- `:in-range` 和 `:out-of-range`：根據其值是否在指定的範圍條件內或外，來設定樣式元素。
