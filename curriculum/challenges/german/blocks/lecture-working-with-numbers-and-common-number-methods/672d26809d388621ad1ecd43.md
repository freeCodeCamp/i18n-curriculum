# --interactive--

In JavaScript steht `NaN` für „Not a Number“. Es ist ein spezieller Wert, der ein nicht darstellbares oder undefiniertes numerisches Ergebnis darstellt. `NaN` ist eine Eigenschaft des globalen Objekts und wird in JavaScript auch als eine Art Zahl betrachtet, was auf den ersten Blick widersprüchlich erscheinen mag.

`NaN` ist typischerweise das Ergebnis von Operationen, die eine Zahl zurückgeben sollten, aber keinen sinnvollen numerischen Wert erzeugen können. Zum Beispiel:

:::interactive_editor

```js
let result = 0 / 0;
console.log(result); // NaN
```

:::

In diesem Fall ist die Division von Null durch Null mathematisch undefiniert, daher gibt JavaScript `NaN` zurück. Eine eigenartige Eigenschaft von `NaN` ist, dass es mit nichts gleich ist, auch nicht mit sich selbst:

:::interactive_editor

```js
console.log(NaN === NaN); // false
```

:::

Dieses einzigartige Verhalten macht es schwierig, mit den üblichen Vergleichsoperatoren zu prüfen, ob ein Wert `NaN` ist. Um dieses Problem zu lösen, stellt JavaScript die `isNaN()`-Funktion bereit. Die `isNaN()`-Funktion wird verwendet, um zu bestimmen, ob ein Wert `NaN` ist oder nicht. Es ist jedoch wichtig zu verstehen, wie `isNaN()` funktioniert, da es manchmal unerwartete Ergebnisse liefern kann. So verhält sich `isNaN()`:

:::interactive_editor

```js
console.log(isNaN(NaN));       // true
console.log(isNaN(undefined)); // true
console.log(isNaN({}));        // true

console.log(isNaN(true));      // false
console.log(isNaN(null));      // false
console.log(isNaN(37));        // false

console.log(isNaN("37"));      // false: "37" is converted to 37
console.log(isNaN("37.37"));   // false: "37.37" is converted to 37.37
console.log(isNaN(""));        // false: empty string is converted to 0
console.log(isNaN(" "));       // false: string with a space is converted to 0

console.log(isNaN("blabla"));  // true: "blabla" is not a number
```

:::

Wie Sie sehen, versucht `isNaN()` zuerst, den Parameter in eine Zahl umzuwandeln. Wenn dies nicht möglich ist, gibt es `true` zurück. Dieses Verhalten kann zu überraschenden Ergebnissen führen, besonders bei Strings, die in Zahlen umgewandelt werden können.

Aufgrund dieser möglichen Inkonsistenzen führte ES6 (die sechste Ausgabe von JavaScript, veröffentlicht 2015) `Number.isNaN()` ein. Diese Methode versucht nicht, den Parameter vor dem Test in eine Zahl umzuwandeln. Sie gibt nur dann `true` zurück, wenn der Wert genau `NaN` ist:

:::interactive_editor

```js
console.log(Number.isNaN(NaN));        // true
console.log(Number.isNaN(Number.NaN)); // true
console.log(Number.isNaN(0 / 0));      // true

console.log(Number.isNaN("NaN"));      // false
console.log(Number.isNaN(undefined));  // false
console.log(Number.isNaN({}));         // false
console.log(Number.isNaN("blabla"));   // false
```

:::

`Number.isNaN()` bietet eine zuverlässigere Möglichkeit, auf `NaN`-Werte zu prüfen, insbesondere in Fällen, in denen Typumwandlung zu unerwarteten Ergebnissen mit der globalen `isNaN()`-Funktion führen kann. In der Praxis ist es bei numerischen Operationen oder Benutzereingaben, die Zahlen sein sollten, oft notwendig, auf `NaN` zu prüfen, um Fehler oder unerwartete Eingaben elegant zu behandeln. Zum Beispiel:

:::interactive_editor

```js
let a = 0;
let b = 0;
let result = a / b;

if (Number.isNaN(result)) {
  result = "Error: Division resulted in NaN";
}

console.log(result); // "Error: Division resulted in NaN"
```

:::

In diesem Beispiel verwenden wir `Number.isNaN()`, um Fälle abzufangen, in denen die Division zu `NaN` führt, sodass wir dieses Szenario angemessen behandeln können. Das Verständnis von `NaN` und wie man richtig darauf prüft, ist entscheidend, um robusten JavaScript-Code zu schreiben, besonders bei mathematischen Operationen oder beim Parsen von Benutzereingaben.
