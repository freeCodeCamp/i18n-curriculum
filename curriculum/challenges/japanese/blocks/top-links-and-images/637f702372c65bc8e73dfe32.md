# --description--

ウェブサイトがテキストだけを表示できるとしたら、かなり退屈なものになってしまいます。幸いにもHTMLはさまざまな種類のメディアを表示するための多彩な要素を提供しています。その中で最も広く使われているのが画像要素です。

HTMLで画像を表示するには`<img>`要素を使います。これまでに出てきた他の要素とは異なり、`<img>`は空要素です。空の、つまりvoidなHTML要素は終了タグを必要としません。

開閉タグでコンテンツを囲む代わりに、`src`属性を使って画像をページに埋め込みます。この属性はブラウザに画像ファイルの場所を伝えます。`src`属性はアンカーの`href`属性とよく似ています。絶対パスと相対パスの両方で画像を埋め込むことができます。

例えば、絶対パスを使ってThe Odin Projectサイトにある画像を表示できます：

自分のウェブサイトにある画像を使う場合は、相対パスを使えます。

- `images`プロジェクト内に`odin-links-and-images`という名前の新しいディレクトリを作成してください。

- 次に、<a href="https://unsplash.com/photos/Mv9hjnEUHR4/download?force=true&w=640" target="_blank">この画像</a>をダウンロードして、先ほど作成した`images`ディレクトリに移動してください。

- 画像の名前を`dog.jpg`に変更してください。

最後に、画像を`index.html`ファイルに追加します：

```html
<body>
  <h1>Homepage</h1>
	<a href="https://www.theodinproject.com/about">click me</a>

	<a href="pages/about.html">About</a>

	<img src="images/dog.jpg">
</body>
```

`index.html`ファイルを保存してブラウザで開き、チャールズの姿を確認してください。


## 親ディレクトリ

`about`ページで犬の画像を使いたい場合はどうすればいいでしょうか？まずpagesディレクトリから一つ上の親ディレクトリに移動し、そこからimagesディレクトリにアクセスする必要があります。

親ディレクトリに移動するには、相対ファイルパスで2つのドットを使います：`../.`。これを実際に見てみましょう。`body`ファイルの`about.html`内で、先ほど追加した見出しの下に次の画像を追加してください：

```html
<img src="../images/dog.jpg">
```

これを分解すると：

- まず、pagesディレクトリの親ディレクトリである`odin-links-and-images`に移動します。

- 次に、親ディレクトリから`images`ディレクトリに入ります。

- 最後に、`dog.jpg`ファイルにアクセスします。

先ほどの比喩を使うと、ファイルパスで`../`を使うのは、今いる部屋から一歩出てメインの廊下に出て、別の部屋に行くようなものです。

## `Alt`属性

`src`属性のほかに、すべての画像要素には`alt`（代替テキスト）属性も必要です。

`alt`属性は画像を説明するために使われます。画像が読み込めない場合に代わりに表示されます。また、スクリーンリーダーと一緒に使われ、視覚障害のあるユーザーに画像の内容を伝えます。
