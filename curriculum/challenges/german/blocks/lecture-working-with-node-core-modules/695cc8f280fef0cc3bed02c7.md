# --description--

JavaScript wurde ursprünglich entwickelt, um in Webbrowsern zu laufen, wo seine Hauptrolle darin bestand, Webseiten interaktiv zu machen. Deshalb konzentrierte sich frühes JavaScript hauptsächlich darauf, Text in Formularen zu verarbeiten und das Document Object Model (DOM) zu manipulieren.

Allerdings sind nicht alle Daten im Web Text. Dateien, Bilder und Videos sind Binärdaten, die andere Verarbeitungsmechanismen benötigen. In Browsern werden diese Datentypen typischerweise von spezialisierten Komponenten des Browsers verarbeitet und nicht direkt von JavaScript.

Moderne Browser verwenden Rendering-Engines und JavaScript-Engines, um diese Aufgaben zu bewältigen – zum Beispiel Blink (mit V8) in Chrome, WebKit (mit JavaScriptCore) in Safari und Gecko (mit SpiderMonkey) in Firefox.

Node.js läuft nicht im Browser, daher benötigte es eine eigene Methode, um Binärdaten zu verarbeiten, insbesondere bei der Ein- und Ausgabe (I/O) von Dateien und TCP-Streams, bei denen Daten in Stücken ankommen. Hier kommt das `Buffer`-Modul ins Spiel.

Das Node.js `Buffer`-Modul ermöglicht es Ihnen, direkt mit Binärdaten wie Dateien, Bildern oder Netzwerkstreams zu arbeiten. Damit können Sie Binärdaten direkt im Speicher speichern und manipulieren.

Genau wie das `fs`-Modul ist `Buffer` eines der Kernmodule von Node.js, sodass Sie es nicht separat installieren müssen, bevor Sie es verwenden.

Um es zu nutzen, importieren Sie das Modul zuerst per Destrukturierung:

```js
const { Buffer } = require("buffer");
```

Dann rufen Sie `Buffer` mit den bereitgestellten Methoden auf. Zum Beispiel können Sie mit `Buffer.from()` einen Puffer aus einem String, Array oder anderen Rohdaten erstellen.

So erstellen Sie einen `Buffer` aus einem String und einem Array:

```js
// Create a buffer from a string
const myStrBuffer = Buffer.from("freeCodeCamp");
console.log(myStrBuffer); // <Buffer 66 72 65 65 43 6f 64 65 43 61 6d 70>

// Create a buffer from an array of numbers
const myNumBuffer = Buffer.from([
  70, 82, 69, 69, 67, 79, 68, 69, 67, 65, 77, 80,
]);

console.log(myNumBuffer); // <Buffer 46 52 45 45 43 4f 44 45 43 41 4d 50>
```

Obwohl es möglich ist, einige Methoden des `Buffer`-Moduls ohne vorherigen Import zu verwenden, sind andere Methoden nur verfügbar, wenn Sie `Buffer` explizit importieren. Deshalb wird empfohlen, Buffer immer zu importieren, wenn Sie es in Ihren Projekten verwenden.

Sie können auf einzelne `buffer`-Elemente genauso zugreifen wie bei einem Array:

```js
console.log(myNumBuffer[0]); // 70
console.log(myStrBuffer[0]); // 102
```

Sie können auch die `toString()`-Methode auf den Puffern verwenden, um zu sehen, wie sie wirklich aussehen:

```js
console.log(myStrBuffer.toString()); // freeCodeCamp
console.log(myNumBuffer.toString()); // FREECODECAMP
```

`Buffer.alloc()` ermöglicht es Ihnen, einen neuen Puffer mit einer bestimmten Größe (Anzahl der Bytes) zu erstellen. Jedes Byte darin wird automatisch mit `0` gefüllt:

```js
const someBuffer = Buffer.alloc(10);
console.log(someBuffer); // <Buffer 00 00 00 00 00 00 00 00 00 00>
```

Sie sehen, dass der Puffer mit Nullen initialisiert wird, basierend auf der Größe, die an die `alloc()`-Methode übergeben wurde.

Sie können die `Buffer.write()`-Methode verwenden, um in diesen Puffer zu schreiben:

```js
someBuffer.write("Hello fCC");

console.log(someBuffer); // <Buffer 48 65 6c 6c 6f 20 66 43 43 00>
console.log(someBuffer.toString()); // Hello fCC
```

Wenn Sie mehr Daten schreiben, als der Puffer aufnehmen kann, werden diese abgeschnitten:

```js
someBuffer.write("Hello freeCodeCamp");

console.log(someBuffer); // <Buffer 48 65 6c 6c 6f 20 66 72 65 65>
console.log(someBuffer.toString()); // Hello free
```

Schließlich können Sie `Buffer.byteLength()` verwenden, um die Anzahl der Bytes anzuzeigen, die benötigt werden, um einen String in einer bestimmten Kodierung zu speichern:

```js
console.log(Buffer.byteLength("Hello freeCodeCamp")); // 18
```

Weitere `Buffer`-Methoden sind:

- `Buffer.isBuffer()`: prüft, ob ein gegebenes Objekt ein Puffer ist
- `Buffer.compare()`: vergleicht zwei Puffer und gibt deren Sortierreihenfolge zurück
- `Buffer.concat()`: verbindet mehrere Puffer zu einem einzigen
