# --description--

الآن ستتعلم عن سكربتات npm.

قبل أن نبدأ، دعنا نراجع بعض المفاهيم المهمة:

* **npm** هو أداة سطر أوامر يمكنك استخدامها لتثبيت وإدارة حزم JavaScript.
    
* **الحزمة** هي مجلد يجمع شفرة مرتبطة في هيكل منطقي.
    
* **السكربت** هو تسلسل من التعليمات أو الأوامر مكتوبة بلغة برمجة نصية.
    

**سكربتات npm** هي أوامر مخصصة يمكنك تعريفها في ملف `package.json` الخاص بتطبيقك.

تساعد هذه السكربتات في أتمتة المهام وسير العمل التي ستستخدمها بشكل متكرر خلال مراحل التطوير والاختبار ومرحلة التشغيل لمشروعك.

على سبيل المثال، أحد استخدامات سكربتات npm هو تخصيص `npm start` لبدء بيئة التطوير بأكملها. بدلًا من كتابة كل الأوامر يدويًا واحدة تلو الأخرى في كل مرة تحتاج فيها لبدء بيئة التطوير، يمكنك فقط تشغيل `npm start` وسيتم تنفيذ العملية بأكملها.

يمكنك أيضًا استخدام سكربتات npm لأتمتة فحص تنسيق الشفرة عبر قاعدة الشفرة الخاصة بك وتشغيل مجموعات الاختبار، من بين استخدامات أخرى.

هي قوية جدًا، فلنراجع أساسياتها.

## كيفية إنشاء ملف package.json

لكي تتمكن من إنشاء وتشغيل سكربتات npm مخصصة، ستحتاج إلى ملف `package.json`.

ملف `package.json` ضروري لمشاريع Node.js. يحتوي هذا الملف على البيانات الوصفية، التبعيات، والسكربتات الخاصة بالمشروع.

كتذكير، لإنشاء ملف `package.json`، يجب تشغيل الأمر `npm init` في الدليل الجذري لمشروعك. بعدها يمكنك إدخال اسم الحزمة، الإصدار، الوصف، وتفاصيل أخرى يدويًا، أو ترك هذه الحقول فارغة لاستخدام القيم الافتراضية.

بعد إكمال هذه العملية، سيكون لديك ملف `package.json` في الدليل الجذري لمشروعك.

إذا احتفظت بكل القيم الافتراضية، سترى أزواج المفتاح-القيمة هذه عند فتح `package.json` في محرر الشفرة:

```json
{
  "name": "<project_name>",
  "version": "1.0.0",
  "main": "index.js",
  "scripts": {
    "test": "echo \"Error: no test specified\" && exit 1"
  },
  "author": "",
  "license": "ISC",
  "description": ""
}
```

لاحظ زوج المفتاح-القيمة هذا هنا:

```json
"scripts": {
  "test": "echo \"Error: no test specified\" && exit 1"
}
```

هنا ستحدد سكربتات npm الخاصة بك، تحت المفتاح `scripts`.

سترى سكربتًا واحدًا هناك افتراضيًا، وهو سكربت الاختبار:

```json
"test": "echo \"Error: no test specified\" && exit 1"
```

هذا مثال على سكربت npm معرف في `package.json`.

إليك الصيغة النحوية الأساسية لتعريف سكربت npm:

```json
"<script_name>": "<commands>"
```

إلى يسار النقطتين، نجد اسم السكربت. من الأسماء الشائعة `run`، `start`، `test`، `build`، `dev`، و`lint`. هذه الأسماء تعمل كـ"اختصارات" للأوامر الأساسية على اليمين.

إلى يمين النقطتين، نجد الأوامر الأساسية نفسها.

على سبيل المثال:

```json
"scripts": {
  "start": "node app.js"
}
```

لتشغيل السكربت، أدخل هذا في سطر الأوامر:

```bash
npm run <script_name>
```

وهكذا يمكنك تشغيل أي سكربت مخصص معرف في `package.json`.

أحيانًا يمكنك استخدام هذا الشكل المختصر بدون كلمة `run` لتنفيذ السكربتات:

```bash
npm <script_name>
```

لكن هذا يعمل فقط مع سكربتات npm محددة تسمى `start`، `test`، `stop`، و`restart`.

بالعودة إلى مثالنا، إذا كان لدينا هذا في `package.json`:

```json
"scripts": {
  "start": "node app.js"
}
```

لتشغيل سكربت npm، تحتاج إلى إدخال `npm run start` في سطر الأوامر. عند القيام بذلك، الأمر الأساسي الذي يُنفذ هو `node app.js`.

بما أن هذا السكربت لديه نسخة مختصرة، يمكنك أيضًا استخدام `npm start` وستكون النتيجة نفسها.

في هذه الحالة، أنت فقط تستبدل أمرًا بسيطًا، `node app.js`، بآخر، `npm start`. لكن تخيل لو كان لديك سلسلة من الأوامر المعقدة التي يجب تنفيذها بالتتابع أو بالتوازي، ويمكنك فقط استخدام `npm start` لتشغيلها جميعًا. هذا سيوفر عليك الكثير من الوقت، أليس كذلك؟ هنا يظهر قوة سكربتات npm حقًا.

إليك نظرة سريعة على بعض سكربتات npm الأكثر استخدامًا:

* `npm run`: تُستخدم لتشغيل سكربت npm مخصص.
    
* `npm start`: تُستخدم لبدء التطبيق الرئيسي أو خادم التطوير.
    
* `npm lint`: تُستخدم لتشغيل مدقق تنسيق الشفرة للتحقق من توافق شفرتك مع معايير وأفضل الممارسات.
    
* `npm test`: تُستخدم لتشغيل مجموعة اختبارات المشروع.
    
هناك العديد من سكربتات npm لأغراض مختلفة، لكن هذه هي الأكثر شيوعًا التي سترىها.

أيضًا، من الاصطلاحات الشائعة استخدام النقطتين لتجميع السكربتات المرتبطة تحت نفس النطاق. على سبيل المثال، `start:server` و`start:ui`.

## كيفية تشغيل الأوامر بالتتابع

إذا كنت بحاجة لتشغيل أوامر متعددة بالتتابع، لديك خياران. يمكنك استخدام عامل AND المنطقي (`&&`) أو الفاصلة المنقوطة (`;`). دعنا نراجع اختلافاتهما.

عامل AND المنطقي يشغل الأمر التالي فقط إذا نجح الأمر الأول.

على سبيل المثال:

```json
"scripts": {
  "build": "webpack --mode production",
  "start": "npm run build && node server.js"
}
```

لاحظ وجود عامل AND المنطقي في أمر `start`:

```json
"start": "npm run build && node server.js"
```

هنا، سيحاول `npm start` تشغيل `npm run build` أولًا، وفقط إذا نجح هذا السكربت، سيتم تنفيذ `node server.js`.

هذا نهج شائع جدًا لأنه يضمن الموثوقية ويمنع إضاعة الوقت والموارد على أوامر لاحقة قد تفشل أو ترمي أخطاء إذا فشل الأمر السابق.

بدلًا من ذلك، يمكنك استخدام عامل الفاصلة المنقوطة لتشغيل الأوامر بالتتابع، بغض النظر عما إذا كان الأمر السابق ناجحًا أم لا.

في مثالنا، هذا التغيير البسيط سيكون له تأثير مهم على كيفية تشغيل الأوامر:

```json
"scripts": {
  "build": "webpack --mode production",
  "start": "npm run build; node server.js"
}
```

ركز على أمر `start` مرة أخرى:

```json
"start": "npm run build; node server.js"
```

الآن، بما أننا نستخدم الفاصلة المنقوطة بدلًا من عامل `&&`، فإن أمر `node server.js` سيُشغل دائمًا بعد `npm run build`، حتى لو لم يكن `node server.js` ناجحًا.

```json
"start": "npm run build; node server.js"
```

## كيفية تشغيل الأوامر بالتوازي

إذا كنت بحاجة لتشغيل الأوامر بالتوازي (في نفس الوقت)، يمكنك استخدام عامل `&`.

```json
"scripts": {
  "start:server": "node server/index.js",
  "start:client": "npm run build:watch",
  "dev": "npm run start:server & npm run start:client"
}
```

لكن يجب أن تعرف أن هذا الخيار يشغل العمليات في الخلفية.

هذا يعني أنه إذا شغلتها في نفس جلسة الطرفية، قد ترى مخرجاتها مجتمعة، مما قد يصعب فهمها.

خيارات أخرى لتشغيل الأوامر بالتوازي تشمل أدوات مثل `concurrently` و`npm-run-all`.

## كيفية تمرير المعلمات إلى سكربتات npm

إذا كنت بحاجة لتمرير معلمات مباشرة إلى الأمر الذي يُنفذ داخل سكربت npm، يمكنك تحديدها في الأمر بكتابة شرطتين (`--`) متبوعتين بالمعلمات.

الشرطتين `--` تخبران npm أن كل ما يليها سيكون معلمات للأوامر الأساسية التي سينفذها سكربت npm.

هذه هي الصيغة العامة:

```bash
npm run <script_name> -- <argument_1> <argument_2> <argument_3>
```

على سبيل المثال، لنفترض أنك تعرف هذا السكربت `npm start`:

```json
{
  "name": "project-example",
  "scripts": {
    "start": "node app.js"
  }
}
```

كيف يمكنك تحديد المنفذ الذي تريد تشغيل الخادم عليه إذا استخدمت `npm start`؟

لنفترض أننا نريد تمرير المنفذ مباشرة كمعلمة في سطر الأوامر. ستكتب هذا:

```bash
npm start -- 8000
```

في شفرتك، يمكنك الوصول إلى هذه المعلمات في مصفوفة `process.argv` باستخدام الفهارس:

```bash
process.argv[0] # The path to the node executable.
process.argv[1] # The script file being executed.
process.argv[2] # The first argument (8000).
```

لذلك، يمكنك الوصول إلى رقم المنفذ واستخدامه حيثما تحتاج مع `process.argv[2]`. هذه هي قوة معلمات سطر الأوامر مع سكربتات npm.

كما ترى، سكربتات npm أدوات أساسية لتبسيط سير عمل التطوير. تساعدك على أتمتة وتحويل تسلسلات الأوامر المعقدة والقابلة لإعادة الاستخدام إلى اختصارات بسيطة ومتسقة يمكنك استخدامها في عملك اليومي.
