# --interactive--

HTML, що розшифровується як Hypertext Markup Language, — це мова розмітки для створення вебсторінок. Коли ви заходите на вебсайт і бачите вміст, як-от абзаци, заголовки, посилання, зображення та відео — це HTML.

Ось невеликий приклад із використанням HTML-елементів. Спробуйте змінити деякий текст у редакторі та подивіться, як зміни відобразяться у вікні попереднього перегляду.

:::interactive_editor

```html
<h1>Main heading element</h1>

<p>I am a paragraph element.</p>
```

:::

HTML представляє вміст і структуру вебсторінки за допомогою елементів. Більшість елементів мають початковий тег і кінцевий тег. Іноді ці теги називають стартовими та кінцевими тегами. Між цими двома тегами розміщується вміст. Цей вміст може бути текстом або іншими HTML-елементами.

Ось ще один приклад елемента абзацу. Змініть текст у редакторі на `I love coding!` і подивіться результати у вікні попереднього перегляду.

:::interactive_editor

```html
<p>I am a paragraph element.</p>
```

:::

І початкові, і кінцеві теги починаються з лівої кутової дужки (`<`) і закінчуються правою кутовою дужкою (`>`), а між цими дужками розміщується назва тегу. Хоча назви HTML-тегів нечутливі до регістру, загальноприйнятою конвенцією і найкращою практикою є писати їх у нижньому регістрі.

Ось детальніший погляд лише на початковий і кінцевий теги абзацу:

```html
<p>
```

```html
</p>
```

Що відрізняє початковий тег від кінцевого — це коса риска (`/`), яка розміщується одразу після лівої кутової дужки в кінцевому тегу. Деякі HTML-елементи не мають кінцевого тегу. Вони відомі як void-елементи.

Ось приклад елемента зображення, який є void-елементом:

```html
<img>
```

Зверніть увагу, що цей елемент зображення не має кінцевого тегу і не містить вмісту. Void-елементи не можуть мати вміст і мають лише початковий тег.

Іноді ви можете побачити void-елементи, які використовують `/` перед `>`, як ось тут:

```html
<img />
```

Хоча багато форматувачів коду, як-от Prettier, вибирають додавати `/` у void-елементи, специфікація HTML зазначає, що наявність `/` «не позначає початковий тег як самозакриваючийся, а є зайвою і не має жодного ефекту».

У реальній розробці ви побачите обидві форми, тому важливо знати обидві.

Якщо ви хочете показати зображення, потрібно додати кілька атрибутів у ваш елемент зображення. Атрибут — це спеціальне значення, яке використовується для налаштування поведінки HTML-елемента.

Ось приклад елемента зображення з атрибутом `src`. Оновіть значення атрибута `src` на `"https://cdn.freecodecamp.org/curriculum/cat-photo-app/cats.jpg"`, і ви побачите, як зображення зміниться на двох котів, що мирно сплять.

:::interactive_editor

```html
<img src="https://cdn.freecodecamp.org/curriculum/cat-photo-app/running-cats.jpg" />
```

:::

Атрибут `src` використовується для вказання розташування цього зображення. Для елементів зображення корисно додати ще один атрибут, який називається атрибутом `alt`. Атрибут `alt` використовується для надання короткого описового тексту для зображень.

Ось приклад елемента зображення з атрибутами `src` і `alt`. Спробуйте зламати зображення, оновивши значення `src` на `"https://.freecodecamp.org/curriculum/cat-photo-app/cats.jpg"`. Ви побачите, що зображення зникне, і відобразиться лише текст `alt`.

:::interactive_editor

```html
<img src="https://cdn.freecodecamp.org/curriculum/cat-photo-app/cats.jpg" alt="Two tabby kittens sleeping together on a couch." />
```

:::

Отже, ви можете запитати, чи достатньо лише HTML для створення вебсайту. Відповідь: це залежить. Якщо ви створюєте невеликий практичний проєкт, який лише відображає текст і зображення, HTML сам по собі може бути достатнім. Однак якщо ви створюєте сучасний професійний вебсайт, вам знадобляться HTML, CSS і JavaScript.

HTML відповідає за вміст і структуру. CSS — за стилі. JavaScript — за додавання інтерактивності на ваші вебсторінки. Добре порівняти HTML, CSS і JavaScript із повноцінною будівлею.

HTML — це блоки, бетон і арматура, які складають стіни. Це фундамент, що робить будівлю міцною. CSS — це внутрішній і зовнішній дизайн, який робить будинок красивим. JavaScript — це електрична і водопровідна система, що забезпечує безперебійний доступ до води та електрики.
