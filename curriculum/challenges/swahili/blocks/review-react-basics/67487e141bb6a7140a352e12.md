## Maktaba na mifumo ya kazi ya JavaScript

- Maktaba na mifumo ya kazi ya JavaScript hutoa suluhisho za haraka kwa matatizo ya kawaida na kuharakisha ukuzaji kwa kutoa msimbo uliotengenezwa tayari.
- Maktaba kwa ujumla hujikita zaidi katika kutoa suluhisho kwa kazi maalum, kama vile kusimamia DOM, kushughulikia matukio, au kusimamia maombi ya AJAX.
- Mifano michache ya maktaba za JavaScript ni jQuery na React.
- Mifumo ya kazi, kwa upande mwingine, hutoa muundo ulio wazi zaidi wa kujenga programu. Mara nyingi huja na seti ya sheria na mikataba ambayo wasanidi programu wanapaswa kufuata.
- Mifano ya mifumo ya kazi ni Angular na Next.js, mfumo wa meta kwa React.
- **Programu za ukurasa mmoja** (SPAs) ni programu za mtandao zinazopakua ukurasa mmoja wa HTML na kusasisha ukurasa huo kwa nguvu kadri mtumizi anavyoshirikiana na programu bila kupakua ukurasa mzima tena.
- SPAs hutumia JavaScript kusimamia hali ya programu na kuonyesha maudhui. Hii mara nyingi hufanywa kwa kutumia mifumo ya kazi inayotoa zana bora za kujenga kiolesura cha mtumiaji tata.
- Baadhi ya matatizo yanayohusiana na SPAs ni:

  - Visomaji vya skrini hushindwa vizuri na maudhui yanayosasishwa kwa nguvu.
  - URL haibadiliki wakati mtumizi anavinjari ndani ya programu, jambo linaloweza kufanya iwe vigumu kuweka alama, kurudi nyuma au kushiriki kurasa maalum.
  - Muda wa kupakua awali unaweza kuwa mrefu ikiwa programu ni kubwa kwani mali zote zinapaswa kupakuliwa mapema.

## React

- React ni maktaba maarufu ya JavaScript kwa ajili ya kujenga violesura vya mtumiaji na programu za mtandao.
- Dhana kuu ya React ni uundaji wa sehemu za UI zinazoweza kutumika tena ambazo zinaweza kusasishwa na kuonyeshwa kwa uhuru kadri data inavyobadilika.
- React inaruhusu wasanidi programu kuelezea jinsi UI inavyopaswa kuonekana kulingana na hali ya programu. Kisha React husasisha na kuonyesha sehemu sahihi wakati data au hali inavyobadilika.

## Sehemu za React

- Sehemu ni vipengele vya msingi vya programu za React vinavyowezesha wasanidi programu kugawanya violesura vya mtumiaji tata kuwa vipande vidogo, vinavyoweza kudhibitiwa.
- UI huandikwa kwa kutumia JSX, nyongeza ya sintaksia ya JavaScript, inayowezesha wasanidi programu kuandika msimbo unaofanana na HTML ndani ya JavaScript.
- Sehemu ni kwa msingi wake kazi za JS au madarasa yanayorejesha kipande cha UI.

Hapa kuna mfano wa sehemu rahisi ya React inayonyesha ujumbe wa salamu:

```jsx
function Greeting() {
  const name = 'Anna';
  return <h1>Welcome, {name}!</h1>;
}
```

Ili kutumia sehemu hii, unaweza tu kuita:

```jsx
  <Greeting />
```

## Kuleta na kusafirisha sehemu za React

- Kusafirisha hufanya sehemu ipatikane kwa mafaili mengine. Kuleta huruhusu faili kutumia sehemu iliyosafirishwa mahali pengine.
- Kuna aina mbili za usafirishaji katika React: usafirishaji wa default na usafirishaji wa majina.

**Usafirishaji wa default**

- Faili inaweza kuwa na usafirishaji mmoja wa default tu. Hii ni bora kwa faili inayojumuisha sehemu moja kuu.
- Unaweza kusafirisha sehemu kama usafirishaji wa default hivi:

```jsx
// City.js
function City() {
  return <p>New York</p>;
}

export default City;
```

- Pia unaweza kusafirisha kwenye mstari mmoja na ufafanuzi wa sehemu:

```jsx
export default function City() {
  return <p>New York</p>;
}
```

- Ili kuleta usafirishaji wa default, tumia neno `import` bila mabano ya mviringo:

```jsx
// App.js
import City from './City';

function App() {
  return (
    <div>
      <h1>My favorite city is:</h1>
      <City />
    </div>
  );
}
```

**Usafirishaji wa majina**

- Usafirishaji wa majina huruhusu faili kushiriki sehemu nyingi. Tofauti na usafirishaji wa default, lazima zilete kwa kutumia jina halisi walilosafirishwa nalo.
- Unaweza kusafirisha sehemu nyingi kutoka faili moja hivi:

```jsx
// Animals.jsx
export function Cat() {
  return <h2>Mr. Whiskers</h2>;
}

export function Dog() {
  return <h2>Fido</h2>;
}
```

- Ili kuleta usafirishaji wa majina, tumia mabano ya mviringo na majina halisi yaliyosafirishwa:

```jsx
import { Cat, Dog } from './Animals';
```

- Unaweza kubadilisha jina la usafirishaji wa jina wakati wa kuleta kwa kutumia neno `as`:

```jsx
import { Cat as Kitty } from './Animals';
```

**Mchanganyiko wa usafirishaji wa default na majina**

- Faili inaweza kuwa na usafirishaji mmoja wa default na usafirishaji wa majina mengi kwa wakati mmoja:

```jsx
// Animals.jsx
export default function Cat() {
  return <h2>Mr. Whiskers</h2>;
}

export function Dog() {
  return <h2>Fido</h2>;
}
```

- Unapolleta usafirishaji mchanganyiko, usafirishaji wa default huja kwanza (bila mabano), ukifuatiwa na usafirishaji wa majina (ndani ya mabano):

```jsx
import Cat, { Dog } from './Animals';
```

## Kuanzisha mradi wa React kwa kutumia Vite

- Zana za kuanzisha miradi na CLI hutoa njia rahisi na ya haraka ya kuanza miradi mipya, zikimruhusu msanidi programu kuzingatia kuandika msimbo badala ya kusimamia usanidi.
- Vite ni zana maarufu ya kuanzisha mradi na inaweza kutumika na React.
- Kuunda mradi mpya kwa Vite, unaweza kutumia amri ifuatayo kwenye terminal yako:

```bash
npm create vite@latest my-react-app -- --template react
```

Amri hii huunda mradi mpya wa React uitwao `my-react-app` kwa kutumia templeti ya React ya Vite. Katika saraka ya mradi, utaona faili la `package.json` lenye utegemezi wa mradi na amri zilizoorodheshwa ndani yake.

- Ili kuendesha mradi, nenda kwenye saraka ya mradi na tumia amri zifuatazo:

```bash
cd my-react-app # path to the project directory
npm install # installs the dependencies listed in the package.json file
```

- Mara utegemezi unapowekwa, utagundua folda mpya katika mradi wako iitwayo `node_modules`.

- Folda ya `node_modules` ndiyo mahali ambapo vifurushi na maktaba yote yanayohitajika na mradi wako huhifadhiwa.

- Ili kuendesha mradi wako, tumia amri ifuatayo:

```bash
npm run dev
```

- Baada ya hapo, fungua kivinjari chako na nenda kwenye `http://localhost:5173` ili kuona programu yako ya React ikifanya kazi.

- Ili kuona msimbo wa templeti ya kuanzia, unaweza kuingia ndani ya mradi wako katika folda ya `src` na utapata faili la `App.jsx`.

## Kupitisha vigezo vinavyopitishwa (props) katika sehemu za React

- Katika React, props (vigezo vinavyopitishwa) ni njia ya kupitisha data kutoka sehemu ya mzazi kwenda sehemu ya mtoto. Mfumo huu unahitajika kuunda vipengele vya UI vinavyoweza kutumika tena na kuwa na mabadiliko.
- Props zinaweza kuwa thamani yoyote ya JavaScript. Ili kupitisha props kutoka mzazi kwenda mtoto, unaongeza props kama sifa wakati unatumia sehemu ya mtoto ndani ya JSX ya mzazi. Hapa kuna mfano rahisi:

```jsx
// Parent component
function Parent() {
  const name = 'Anna';
  return <Child name={name} />;
}

// Child component
function Child(props) {
  return <h1>Hello, {props.name}!</h1>;
}
```

Unaweza kupitisha props nyingi kwa kutumia opereta ya spread `(...)`, baada ya kuzibadilisha kuwa kitu. Hapa kuna mfano:

```jsx
// Parent component
function Parent() {
  const person = {
    name: 'Anna',
    age: 25,
    city: 'New York'
  };
  return <Child {...person} />;
}
```

Katika msimbo huu, opereta ya spread `{...person}` hubadilisha kitu cha person kuwa props binafsi zinazopitishwa kwa sehemu ya Child.

## Kuonyesha kwa masharti katika React

- Kuonyesha kwa masharti katika React kunakuwezesha kuunda violesura vya mtumiaji vinavyobadilika. Hutumika kuonyesha maudhui tofauti kulingana na masharti au hali ndani ya programu yako.
- Kuna njia kadhaa za kuonyesha maudhui kwa masharti katika React. Njia moja ya kawaida ni kutumia opereta ya ternary. Hapa kuna mfano:

```jsx
function Greeting({ isLoggedIn }) {
  return (
    <div>
      {isLoggedIn ? <h1>Welcome back!</h1> : <h1>Please log in</h1>}
    </div>
  );
}
```

- Njia nyingine ya kuonyesha maudhui kwa masharti ni kutumia opereta ya mantiki AND (`&&`). Hii ni muhimu unapotaka kuonyesha maudhui tu ikiwa sharti fulani limekamilika. Hapa kuna mfano:

```jsx
function Greeting({ user }) {
  return (
    <div>
      {user && <h1>Welcome, {user.name}!</h1>}
    </div>
  );
}
```

Katika msimbo huu hapo juu, kipengele cha `h1` kinaonyeshwa tu ikiwa kitu cha user ni kweli.

Unaweza pia kutumia kauli ya `if` moja kwa moja hivi:

```jsx
function Greeting({ isLoggedIn }) {
  if (isLoggedIn) {
    return <h1>Welcome back!</h1>;
  }
  return <h1>Please sign in</h1>;
}
```

## Kuonyesha orodha katika React

- Kuonyesha orodha katika React ni kazi ya kawaida wakati wa kujenga violesura vya mtumiaji.
- Orodha zinaweza kuonyeshwa kwa kutumia njia ya safu ya JS `map()` kurudia juu ya safu ya vitu na kurejesha safu mpya ya vipengele vya JSX.
- Kwa mfano, ikiwa una safu ya majina unayotaka kuonyesha kama orodha, unaweza kufanya yafuatayo:

```jsx
function NameList({ names }) {
  return (
    <ul>
      {names.map((name, index) => (
        <li key={`${name}-${index}`}>{name}</li>
      ))}
    </ul>
  );
}
```

- Kumbuka kila wakati kutoa ufunguo wa kipekee kwa kila kipengele cha orodha kusaidia React kusimamia majukumu ya kusasisha na kuonyesha. Kwa mbinu hizi, unaweza kuunda orodha zinazobadilika, zenye ufanisi, na zenye nguvu katika programu zako za React.

## Mitindo ndani ya mstari katika React

- Mitindo ndani ya mstari katika React inakuwezesha kutumia mitindo ya CSS moja kwa moja kwa vipengele vya JSX kwa kutumia vitu vya JavaScript.
- Ili kutumia mitindo ndani ya mstari katika React, unaweza kutumia sifa ya style kwenye vipengele vya JSX. Sifa ya style inachukua kitu ambacho funguo zake ni mali za CSS kwa camelCase na thamani ni zile zinazolingana. Hapa kuna mfano:

```jsx
function Greeting() {
  return (
    <h1
      style={{ color: 'blue', fontSize: '24px', backgroundColor: 'lightgray' }}
    >
      Hello, world!
    </h1>
  );
}

export default Greeting;
```

Unaweza pia kutoa mitindo hiyo katika kitu tofauti na kuitaja katika sifa ya `style` hivi:

```jsx
function Greeting() {
  const styles = {
    color: 'blue',
    fontSize: '24px',
    backgroundColor: 'lightgray'
  };

  return <h1 style={styles}>Hello, world!</h1>;
}

export default Greeting;
```

- Mitindo ndani ya mstari inaunga mkono mitindo inayobadilika kwa kuruhusu kutumia mitindo kwa masharti kulingana na props au hali. Hapa kuna mfano wa jinsi unavyoweza kutumia mitindo kwa masharti kulingana na prop:

```jsx
function Greeting({ isImportant }) {
  const styles = {
    color: isImportant ? 'red' : 'black',
    fontSize: isImportant ? '24px' : '16px'
  };

  return <h1 style={styles}>Hello, world!</h1>;
}

export default Greeting;
```

- Katika msimbo huu hapo juu, mitindo ya `color` na `fontSize` imewekwa kwa masharti kulingana na prop ya `isImportant`.
