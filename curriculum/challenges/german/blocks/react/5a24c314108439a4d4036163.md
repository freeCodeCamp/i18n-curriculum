# --description--

Die andere Möglichkeit, eine React-Komponente zu definieren, ist mit der ES6 `class`-Syntax. Im folgenden Beispiel erweitert `Kitten` `React.Component`:

```jsx
class Kitten extends React.Component {
  constructor(props) {
    super(props);
  }

  render() {
    return (
      <h1>Hi</h1>
    );
  }
}
```

Dies erstellt eine ES6-Klasse `Kitten`, die die Klasse `React.Component` erweitert. Die Klasse `Kitten` hat somit Zugriff auf viele nützliche React-Funktionen, wie lokalen State und Lifecycle-Hooks. Machen Sie sich keine Sorgen, wenn Ihnen diese Begriffe noch nicht vertraut sind, sie werden in späteren Aufgaben ausführlicher behandelt. Beachten Sie auch, dass die Klasse `Kitten` eine `constructor` enthält, die `super()` aufruft. Sie verwendet `super()`, um den Konstruktor der Elternklasse aufzurufen, in diesem Fall `React.Component`. Der Konstruktor ist eine spezielle Methode, die während der Initialisierung von Objekten verwendet wird, die mit dem Schlüsselwort `class` erstellt werden. Es ist bewährte Vorgehensweise, den `constructor` einer Komponente mit `super` aufzurufen und `props` an beide zu übergeben. So wird sichergestellt, dass die Komponente korrekt initialisiert wird. Für den Moment sollten Sie wissen, dass es Standard ist, diesen Code einzuschließen. Bald werden Sie weitere Verwendungen für den Konstruktor sowie `props` sehen.

# --instructions--

`MyComponent` ist im Code-Editor mit der Klassensyntax definiert. Beenden Sie die `render`-Methode so, dass sie ein `div`-Element zurückgibt, das eine `h1` mit dem Text `Hello React!` enthält.
