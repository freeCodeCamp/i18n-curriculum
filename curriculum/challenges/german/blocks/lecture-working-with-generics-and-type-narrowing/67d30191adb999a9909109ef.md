# --description--

Sie haben tatsächlich schon einen generischen Typ in einer vorherigen Lektion gesehen: `Array<string>`. Aber normalerweise verwenden Sie generische Typen in Funktionen. Man kann sie tatsächlich als einen speziellen Parameter betrachten, den Sie einer Funktion übergeben, um das Verhalten der Typdefinition der Funktion zu steuern.

Kehren wir zu unserem vorherigen Beispiel einer Funktion zurück, die einen zufälligen Wert aus einem Array holt:

```js
const getRandomValue = (array: string[]): string => {
  return array[Math.floor(Math.random() * array.length)];
}
```

Unsere Funktion akzeptiert ein Array von Strings und gibt einen String zurück. Aber was, wenn wir ein Array von Zahlen übergeben und eine Zahl zurückgeben wollen?

Sie könnten einen Union-Typ verwenden:

```js
const getRandomValue = (array: (string|number)[]): (string|number) => {
  return array[Math.floor(Math.random() * array.length)];
}
const val = getRandomValue([1, 2, 4])
```

Aber bei diesem Ansatz bestimmt TypeScript, obwohl wir nur ein Array von Zahlen übergeben haben, dass die Variable `val` eine Zahl ODER ein String sein könnte.

Stattdessen können Sie einen generischen Typ für die Funktion definieren:

```js
const getRandomValue = <T>(array: T[]): T => {
  return array[Math.floor(Math.random() * array.length)];
}
const val = getRandomValue([1, 2, 4])
```

In unserem neuen Beispiel gibt es ein paar wichtige Komponenten. Zuerst sagt die `<T>`-Syntax TypeScript, dass Sie einen generischen Typ `T` für die Funktion definieren. `T` ist ein gebräuchlicher Name für generische Typen, aber Sie können ihn tatsächlich beliebig benennen (innerhalb der JavaScript-Variablenkonventionen).

Dann sagen wir TypeScript, dass der Array-Parameter ein Array von Werten ist, die dem generischen Typ entsprechen, und dass der zurückgegebene Wert ein einzelnes Element dieses gleichen Typs ist. Das ermöglicht TypeScript, korrekt zu erkennen, dass `val` eine Zahl sein wird, weil das übergebene Array ausschließlich aus Zahlen besteht.

Aber wie sieht es mit Funktionen aus, die Sie nicht kontrollieren? Schauen wir uns die `document.querySelector()`-Methode an:

```js
const email = document.querySelector("#email");
console.log(email.value);
```

Da wir mit einem ID-Selektor abfragen, weiß TypeScript nicht, welche Art von Element wir abfragen. Es weiß, dass es sich um eine Art `Element` handelt, aber unser Versuch, auf die Eigenschaft `value` zuzugreifen, wirft einen Fehler, weil das oberste Element-Interface diese Eigenschaft nicht hat.

Glücklicherweise können wir der Funktionsaufruf einen Typ-Parameter übergeben:

```js
const email = document.querySelector<HTMLInputElement>("#email");
console.log(email.value);
```

Beachten Sie, wie wir wieder die spitze Klammer-Syntax verwenden (`<HTMLInputElement>`), diesmal aber in einem Funktionsaufruf. Das sagt TypeScript, dass das Element, das wir erwarten zu finden, ein `input`-Element sein wird, und wir erhalten keinen Fehler mehr bei der Eigenschaft value, weil Eingabeelemente diese Eigenschaft haben. Natürlich kann `querySelector` `null` zurückgeben, wenn kein Element gefunden wird, und TypeScript wirft hier einen Compiler-Fehler, weil wir diesen Fall nicht behandeln.

Aber das lernen Sie in der nächsten Lektion.
