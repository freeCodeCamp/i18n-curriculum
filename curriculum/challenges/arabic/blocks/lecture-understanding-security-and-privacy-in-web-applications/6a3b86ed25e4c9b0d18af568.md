# --description--

ملف `.env` ومكتبة `dotenv` أدوات تُستخدم في تطوير البرمجيات الحديثة لفصل بيانات التهيئة الحساسة عن شفرة التطبيق.

ملف `.env` هو ملف نصي عادي يُوضع في جذر مجلد المشروع ويخزن إعدادات التهيئة والأسرار على شكل أزواج المفتاح-القيمة.

بدلاً من تضمين مفاتيح API أو كلمات مرور قاعدة البيانات أو أرقام المنافذ مباشرة في الكود المصدري مع خطر كشفها إذا تم رفع الكود إلى مستودع عام مثل GitHub، يحفظ المطوّرون هذه القيم محليًا في هذا الملف.

هيكل ملف `.env` بسيط وواضح:

```bash
PORT=3000
DB_HOST=localhost
API_KEY=xyz123secretkey
```

بما أن هذا الملف يحتوي على مفاتيح حساسة جدًا، فمن الممارسات الشائعة إضافة `.env` إلى ملف `.gitignore` الخاص بالمشروع حتى لا يتم تتبعه بواسطة Git.

حزمة `dotenv` هي وحدة خفيفة الوزن تربط ملف `.env` بتطبيقك.

عند بدء تشغيل برنامجك، تقرأ `dotenv` ملف `.env`، تُحلل المفاتيح، وتُدخلها في بيئة التشغيل. في تطبيق Node.js، على سبيل المثال، تهيئ الحزمة في أعلى ملف الدخول. بعد التحميل، تصبح المتغيرات متاحة عالميًا عبر كائن `process.env`.

```javascript
const express = require('express');
const app = express();

// initialize and configure dotenv
require('dotenv').config();

// access the variables securely from the environment
const port = process.env.PORT || 5000;
const apiKey = process.env.API_KEY;

app.get('/', (req, res) => {
  res.send(`Connected successfully using API Key: ${apiKey}`);
});

app.listen(port, () => {
  console.log(`Server running on port ${port}`);
});
```

يوفر هذا الإعداد فائدتين رئيسيتين. أولاً، يعزز الأمان عبر إبقاء بيانات الاعتماد الخاصة خارج الكود المصدري. ثانيًا، يمنح مرونة بالسماح لنفس قاعدة الشفرة بأن تتصرف بشكل مختلف عبر بيئات التطوير والاختبار والإنتاج بمجرد استبدال ملف `.env` المحلي على كل خادم استضافة خاص بتلك البيئة.
