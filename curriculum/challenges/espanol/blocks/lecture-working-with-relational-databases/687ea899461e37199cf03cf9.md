# --description--

Para definir una tabla en SQL, necesitas especificar el tipo de dato de cada columna.

Recuerda que esta es la sintaxis básica para crear una tabla en SQL. Cada columna tiene su propio tipo de dato.

```sql
CREATE TABLE table_name(
  column1 data_type column_constraint,
  column2 data_type column_constraint,
  column3 data_type column_constraint,
  ... etc
);
```

Como desarrollador, tu objetivo es elegir estos tipos de datos con la mayor precisión posible.

Hay seis categorías populares de tipos de datos en SQL:

- Numérico, como `INTEGER`, `FLOAT`, `SERIAL` y `DECIMAL`.
- Fecha y hora, como `TIMESTAMP`, `DATE` y `TIME`.
- Carácter y cadena. Estos incluyen `CHAR`, `VARCHAR` y `TEXT`.
- Unicode, incluyendo `NTEXT` y `NVARCHAR`. Estos se usan para asegurar que el texto se almacene y recupere correctamente, sin importar el origen de los caracteres.
- Binario, usado para almacenar datos no textuales, como imágenes, audio y archivos de video.
- Y otros tipos de datos diversos, como `XML` y `TABLE`.

En esta lección, cubriremos algunos de los más usados con PostgreSQL. Los tipos de datos varían entre los sistemas de gestión de bases de datos, pero generalmente son bastante similares. Así que necesitarás revisar la lista completa en la documentación.

Comenzaremos con valores numéricos.

Primero, tenemos el tipo de dato `INTEGER`:

```sql
units_sold INTEGER
```

En este ejemplo, asignamos este tipo de dato a una columna `units_sold`.

Los tipos `INTEGER` consumen 4 bytes en la base de datos y pueden tener un rango de valores desde -2,147,483,648 hasta 2,147,483,647. La documentación oficial de PostgreSQL lo describe como la "elección típica para enteros."

También tenemos `SMALLINT` y `BIGINT`, que son básicamente iguales a `INTEGER` excepto que tienen un rango de números más pequeño y más grande, respectivamente, debido a la cantidad de espacio que se les asigna en la base de datos.

Una característica útil disponible en PostgreSQL es crear una columna usando la palabra clave `SERIAL`. Aquí, se crea una columna `id` usando `SERIAL`:

```sql
id SERIAL
```

Aunque no es un tipo de dato verdadero, es muy útil para crear columnas de identificadores únicos porque la columna tendrá un tipo `INTEGER`, no permitirá valores `NULL` y se incrementará automáticamente cuando se agreguen filas.

Por ejemplo, la primera fila tendrá automáticamente un `id` de `1`, la segunda fila tendrá un `id` de `2`, y así sucesivamente, creando un ID único para cada registro.

En MySQL, el equivalente de `SERIAL` sería el atributo `AUTO_INCREMENT`, usado para generar enteros secuenciales automáticamente:

```sql
id INT AUTO_INCREMENT
```

Puedes ver que diferentes sistemas de gestión de bases de datos pueden tener diferentes maneras de lograr exactamente la misma funcionalidad en tu base de datos.

Estos son los tipos de datos numéricos más comúnmente usados, pero a veces, puede que necesites representar texto o secuencias de caracteres.

Para esto, tienes el tipo de dato `VARCHAR`. Este tipo de dato se usa para una longitud variable de cadena. Puedes establecer la longitud máxima de caracteres dentro de paréntesis:

```sql
name VARCHAR(50)
```

En este ejemplo, definimos una columna `name`, donde los valores pueden tener hasta 50 caracteres de longitud.

Este tipo de dato establece una longitud máxima para las cadenas, pero si necesitas almacenar cadenas de cualquier longitud, puedes usar `TEXT` en su lugar:

```sql
name TEXT
```

Además de números y cadenas, también es común almacenar fechas y horas.

Por ejemplo, si creas una tabla para almacenar eventos, puede que necesites guardar la fecha de cada evento en un formato específico. Para esto, tienes el tipo de dato `DATE`:

```sql
event_date DATE
```

Para almacenar tiempo, también puedes usar el tipo de datos `TIME`. Por ejemplo, puedes almacenar la hora en que comienza un evento:

```sql
start_time TIME
```

El tipo de datos `TIMESTAMP` combina ambos. Incluye tanto la fecha como la hora. También puede incluir la zona horaria si escribes `TIMESTAMP WITH TIME ZONE`:

```sql
event_timestamp TIMESTAMP
```

```sql
event_timestamp TIMESTAMP WITH TIME ZONE
```

Y finalmente, si necesitas almacenar un valor booleano, `TRUE` o `FALSE`, puedes usar el tipo de dato `BOOLEAN`:

```sql
is_active BOOLEAN
```

Ahora conoces algunos de los tipos de datos más comunes en SQL, pero hay muchos más. Puedes consultar la documentación de tu sistema de gestión de bases de datos para encontrar más información sobre los tipos de datos que puedes usar.
