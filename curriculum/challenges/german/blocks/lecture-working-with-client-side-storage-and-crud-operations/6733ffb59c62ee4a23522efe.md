# --interactive--

`IndexedDB` ist ein Werkzeug zum Speichern strukturierter Daten im Browser. Es ist in moderne Webbrowser integriert und ermöglicht Webanwendungen, JavaScript-Objekte effizient zu speichern und abzurufen.

Im Gegensatz zu anderen Speichermechanismen wie `localStorage`, die auf das Speichern von Strings beschränkt sind, kann `IndexedDB` JavaScript-Objekte, Dateien und praktisch jede andere Art von Daten speichern. Das macht es einfach für Webanwendungen, die mit großen und komplexen Datenstrukturen arbeiten müssen.

Schauen wir uns nun an, wie `IndexedDB` funktioniert.

Der erste Schritt ist das Öffnen einer Datenbank. Hier ist ein Beispiel:

:::interactive_editor

```js
let request = indexedDB.open("Sample DB", 1);

request.onerror = function(event) {
  console.log("Error opening database");
};

request.onsuccess = function(event) {
  let db = event.target.result;
  console.log("Database opened successfully");
};
```

:::

In diesem Code öffnen wir eine Datenbank namens `"Sample DB"` mit Version 1. Wir übergeben zwei Callback-Funktionen: eine zur Fehlerbehandlung und eine andere, die ausgeführt wird, wenn die Datenbank erfolgreich geöffnet wurde. Das `db`-Objekt, das wir im Erfolgs-Callback erhalten, verwenden wir, um mit der Datenbank zu interagieren.

Wenn Sie die Anwendungsschnittstelle der Browser-Entwicklertools überprüfen, sehen Sie, dass Ihre `Sample DB` im Abschnitt `IndexedDb` geöffnet wurde. Sobald Ihre Datenbank geöffnet ist, können Sie mit Objekt-Stores arbeiten.

Objekt-Stores in IndexedDB sind ähnlich wie Tabellen in traditionellen Datenbanken. Sie enthalten die tatsächlichen Daten, die Sie speichern möchten. So erstellen Sie einen Objekt-Store:

```js
let request = indexedDB.open("Sample DB", 1);

request.onupgradeneeded = function(event) {
  let db = event.target.result;
  let objectStore = db.createObjectStore("customers", { keyPath: "id" });
};
```

Dieser Code erstellt einen Objekt-Store namens `"customers"` mit `"id"` als Schlüsselpfad. Der Schlüsselpfad ist wie ein Primärschlüssel in einer traditionellen Datenbank – er wird verwendet, um jeden Datensatz eindeutig zu identifizieren.

Um Daten zu unserem Objekt-Store hinzuzufügen, machen wir so etwas. Das `db` in diesem Beispiel steht für die `IndexedDB`-Datenbankinstanz.

```js
let transaction = db.transaction(["customers"], "readwrite");
let objectStore = transaction.objectStore("customers");
let request = objectStore.add({ id: 1, name: "John Doe", email: "john@example.com" });

request.onerror = function(event) {
  console.log("Error adding data");
};

request.onsuccess = function(event) {
  console.log("Data added successfully");
};
```

Dieser Code fügt einen neuen Kunden zu unserem `"customers"`-Objekt-Store hinzu. Wir starten eine Transaktion (so gruppieren wir Datenbankoperationen), holen eine Referenz zu unserem Objekt-Store und fügen dann unsere Daten hinzu.

Das Abrufen von Daten funktioniert ähnlich. Wir starten eine Transaktion, holen unseren Objekt-Store und verwenden dann Methoden wie `get`, um Daten abzurufen:

```js
let transaction = db.transaction(["customers"]);
let objectStore = transaction.objectStore("customers");
let request = objectStore.get(1);

request.onerror = function(event) {
  console.log("Error retrieving data");
};

request.onsuccess = function(event) {
  console.log("Customer:", request.result);
};
```

Dieser Code ruft den Kunden mit dem `id` von `1` aus unserem `"customers"`-Objekt-Store ab.

Eine der wichtigsten Funktionen von `IndexedDB` ist, dass es asynchron ist.

Das bedeutet, dass beim Interagieren mit `IndexedDB` Operationen den Haupt-Thread der Webanwendung nicht blockieren. So bleibt Ihre Webanwendung responsiv, selbst wenn große Datenmengen verarbeitet werden.

Obwohl `IndexedDB` leistungsstarke Möglichkeiten bietet, ist die Lernkurve im Vergleich zu anderen einfacheren Speicher-APIs steiler, was für Anfänger herausfordernd sein kann. Für Anwendungen, die große Mengen strukturierter Daten auf der Client-Seite verarbeiten müssen, bietet `IndexedDB` jedoch unvergleichliche Fähigkeiten.
