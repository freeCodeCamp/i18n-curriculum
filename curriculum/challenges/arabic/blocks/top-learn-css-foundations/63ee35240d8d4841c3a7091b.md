# --description--

محدِّدات الصنف تختار كل العناصر التي تحمل `class` معين، وهو مجرد خاصية تضعها على عنصر HTML. إليك كيف تضيف صنفًا إلى وسم HTML وتختاره في CSS:

```html
<!-- index.html -->

<div class="alert-text">
  Please agree to our terms of service.
</div>
```

```css
/* styles.css */

.alert-text {
  color: red;
}
```

لاحظ صيغة محددات `class`: نقطة تليها مباشرة القيمة الحساسة لحالة الأحرف لخاصية الصنف. لا يشترط أن تكون الأصناف فريدة، لذا يمكنك استخدام نفس `class` على عدد من العناصر كما تريد.

يمكنك أيضًا باستخدام خاصية `class` إضافة عدة أصناف إلى عنصر واحد كقائمة مفصولة بمسافة، مثل `class="alert-text severe-alert"`. بما أن الفراغ يُستخدم للفصل بين أسماء الـ `class` بهذه الطريقة، لا تستخدم الفراغات في الأسماء المركبة، بل استخدم الشرطة (-) بدلًا منها.

## محددات المعرف
محددات المعرف تشبه محددات `class`. تختار عنصرًا يحمل `id` معين، وهي خاصية أخرى تضعها على عنصر HTML:

```html
<!-- index.html -->

<div id="title">My Awesome 90's Page</div>
```

```css
/* styles.css */

#title {
  background-color: red;
}
```

بدلًا من النقطة، تستخدم علامة الهاشتاغ تليها مباشرة القيمة الحساسة لحالة الأحرف لخاصية `id`. من الأخطاء الشائعة الإفراط في استخدام خاصية `id` عندما لا يكون ذلك ضروريًا، وحين تكفي الأصناف. رغم وجود حالات يكون فيها استخدام `id` منطقيًا أو ضروريًا، مثل الاستفادة من التحديدية أو وجود روابط تعيد التوجيه إلى قسم في الصفحة الحالية، يجب استخدام `id` بحذر (إن استُخدم أصلاً).

الفرق الرئيسي بين الأصناف والمعرفات هو أن العنصر يمكن أن يحتوي على `id` واحد فقط. لا يمكن تكرار أي `id` في الصفحة الواحدة، ويجب ألا تحتوي خاصية `id` على أي فراغات على الإطلاق.
