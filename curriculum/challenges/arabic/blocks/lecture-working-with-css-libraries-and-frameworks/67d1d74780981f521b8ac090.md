# --description--

أثناء عملك على CSS، قد تواجه تحديات معينة، مثل الحفاظ على تناسق الأنماط وتوافقها عبر المتصفحات، وتجنب التكرار، والتعامل مع تعقيد وتزايد عدد ملفات CSS في مشروعك. تساعد أُطُر عمل CSS كثيرًا في منع هذه المشاكل من البداية.

إطار عمل CSS يتكون من شفرة CSS مكتوبة مسبقًا يمكنك استخدامها لتنسيق عناصر HTML الخاصة بك.

باستخدام إطار عمل CSS، يمكنك تسريع سير عملك، وإنشاء نمط بصري موحد عبر الموقع، وجعل تصميمك يبدو متناسقًا عبر متصفحات متعددة، وبشكل عام الحفاظ على تنظيم شفرة CSS الخاصة بك.

يمكنك التفكير في إطار عمل CSS كصندوق أدوات، حيث يمكنك بسرعة العثور على نمط أو مكوِّن يناسب احتياجاتك واستخدامه فورًا، دون الحاجة إلى تحديده بنفسك باستخدام خصائص CSS. غالبًا ما تتضمن هذه الأُطُر أنماطًا ومكوِّنات معرفة مسبقًا يمكنك استخدامها بسهولة، بما في ذلك أنماط لإنشاء تخطيطات متجاوبة.

بعض أشهر أُطُر عمل CSS هي Tailwind CSS وBootstrap وMaterialize وFoundation.

دعنا نتحدث قليلًا عن نوعي أُطُر عمل CSS: أُطُر عمل CSS التي تعتمد على الأدوات أولًا، وأُطُر عمل CSS التي تعتمد على المكوِّنات.

Tailwind CSS هو إطار عمل CSS يعتمد على الأدوات أولًا. هذا يعني أنه يحتوي على أصناف صغيرة لأغراض محددة جدًا، مثل تعيين الهامش، الحشو، أو لون الخلفية. يمكنك عادةً تعيين هذه الأصناف الصغيرة مباشرة إلى عناصر HTML.

على سبيل المثال:

```html
<button class="bg-blue-500 text-white font-bold py-2 px-4 rounded-full hover:bg-blue-700">
  Button
</button>
```

سيكون لهذا الزر خلفية زرقاء مع نص أبيض وعريض، حشو عمودي بمقدار 2 ريم، حشو أفقي بمقدار 4 ريم، وحدود مستديرة. الصنف `hover:bg-blue-700` سيجعل الخلفية زرقاء داكنة عندما يحوم المستخدم فوقه بالماوس.

كما ترى، أصناف Tailwind CSS دقيقة جدًا، لذا يمكن تطبيقها على عناصر HTML الفردية حسب الحاجة.

على النقيض من ذلك، Bootstrap هو إطار عمل CSS يعتمد على المكوِّنات. يحتوي على مكوِّنات مبنية مسبقًا بأنماط معرفة مسبقًا يمكنك إضافتها بسهولة إلى موقعك. عادةً ما تكون هذه المكوِّنات متاحة في الوثائق الرسمية لإطار العمل، حيث يمكنك نسخها ولصقها بسرعة في مشروعك.

على سبيل المثال، إذا أردت إنشاء مجموعة قائمة في Bootstrap، ستكتب شيئًا مثل هذا في HTML:

```html
<div class="card" style="width: 25rem;">
  <ul class="list-group list-group-flush">
    <li class="list-group-item">HTML</li>
    <li class="list-group-item">CSS</li>
    <li class="list-group-item">JavaScript</li>
  </ul>
</div>
```

هذه الأصناف المعرفة مسبقًا ستطبق تلقائيًا أنماطًا قياسية على البطاقة، مجموعة القائمة، وعناصر مجموعة القائمة.

لاحظ هنا، بدلاً من إضافة أصناف صغيرة وعامة، نضيف المكوِّن بالكامل، بما في ذلك هيكل HTML. لهذا السبب Bootstrap يعتمد على المكوِّنات.

```html
<div class="card" style="width: 25rem;">
  <ul class="list-group list-group-flush">
    <li class="list-group-item">HTML</li>
    <li class="list-group-item">CSS</li>
    <li class="list-group-item">JavaScript</li>
  </ul>
</div>
```

بينما لأُطُر عمل CSS العديد من المزايا، إلا أن لها أيضًا بعض العيوب. من المهم أن تعرف كلاهما لاتخاذ قرار مستنير.

بما أن أُطُر عمل CSS تعتمد على أنماط معرفة مسبقًا، إذا حاولت إضافة CSS مخصص فوقها، قد تظهر مشاكل في التحديدية. قد تكون محددات الإطار أكثر تحديدًا من المحددات التي تستخدمها في CSS المخصص، لذا قد لا يظهر النمط الصحيح. لحل ذلك، ستحتاج إلى استخدام محدد CSS أكثر تحديدًا، وهذا قد يجلب بعض التحديات الإضافية.

يمكنها أيضًا أن تحد من التفرد إلى حد ما لأن أنماطها القياسية تُستخدم على نطاق واسع في العديد من المواقع. مع ذلك، يمكن معالجة هذا بتخصيص CSS بأنماطك الفريدة.

وأخيرًا، يجب أن تأخذ في الاعتبار المشاكل المحتملة في الأداء التي قد تسببها أُطُر عمل CSS. بعض الأُطُر كبيرة الحجم، وقد تستغرق وقتًا للتحميل.

بصفتك مطوِّرًا، ستقرر ما إذا كان استخدام إطار عمل CSS مفيدًا لمشروعك، أو إذا كان استخدام CSS مخصص بالكامل أفضل، أو ربما مزيج من الاثنين.
