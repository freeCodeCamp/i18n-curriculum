# --interactive--

미디어 쿼리는 반응형 웹 디자인의 기본 컴포넌트로, 개발자가 사용자의 장치나 브라우저 특성에 따라 다른 스타일을 적용할 수 있게 합니다. 콘텐츠 자체를 변경하지 않고도 다양한 장치에 맞게 콘텐츠 표현을 맞춤화할 수 있는 방법을 제공합니다.

미디어 쿼리의 핵심은 미디어 유형과 하나 이상의 특정 조건을 검사하는 표현식으로 구성됩니다. 이 조건들이 참이면 해당 스타일이 적용됩니다. 이 메커니즘 덕분에 화면 크기, 해상도, 장치 기능에 따라 적응하는 반응형 디자인을 만들 수 있습니다.

CSS에서 미디어 쿼리의 기본 구문은 다음과 같습니다:

```css
@media mediatype and (feature: value) {
  /* CSS rules go here */
}
```

이 구조에서 mediatype은 쿼리가 적용되는 미디어 유형을 지정하고, feature: value 쌍은 스타일이 적용되기 위한 조건을 정의합니다.

미디어 유형은 장치의 일반적인 범주를 설명합니다. 가장 일반적으로 사용되는 미디어 유형을 살펴보겠습니다:

- `all`는 모든 장치에 적합합니다. 미디어 유형이 지정되지 않으면 기본값입니다.

- `print`는 인쇄 미리보기 모드에서 화면에 표시되는 페이지 형식 자료와 문서에 적합합니다.

- `screen`는 주로 화면용입니다.

과거에는 `handheld`와 `tv` 같은 더 많은 미디어 유형이 있었지만, 대부분은 기능을 사용해 장치를 더 정확히 타겟팅하는 방식으로 대체되어 더 이상 권장되지 않습니다.

미디어 기능은 사용자 에이전트, 출력 장치 또는 환경의 특정 특성을 설명합니다. 가장 일반적으로 사용되는 미디어 기능 몇 가지를 살펴보겠습니다. `width`와 `height`는 뷰포트의 너비와 높이를 나타내며, `min-` 또는 `max-` 접두사와 함께 범위 쿼리에 자주 사용됩니다. 예시는 다음과 같습니다:

```css
@media screen and (min-width: 768px) {
  /* Styles for screens at least 768px wide */
}
```

`aspect-ratio`는 뷰포트의 너비와 높이 비율을 설명합니다. 예시는 다음과 같습니다:

```css
@media screen and (aspect-ratio: 16/9) {
  /* Styles for screens with a 16:9 aspect ratio */
}
```

`orientation` 기능은 장치가 가로 방향인지 세로 방향인지를 나타냅니다. 예시는 다음과 같습니다:

```css
@media screen and (orientation: landscape) {
  /* Styles for landscape orientation */
}
```

`resolution` 기능은 출력 장치의 해상도를 인치당 점(dpi) 또는 센티미터당 점(dpcm)으로 설명합니다. 예시는 다음과 같습니다:

```css
@media screen and (min-resolution: 300dpi) {
  /* Styles for high-resolution screens */
}
```

`hover` 기능은 기본 입력 장치가 요소 위에 마우스를 올릴 수 있는지 테스트합니다. 예시는 다음과 같습니다:

```css
@media (hover: hover) {
  /* Styles for devices that support hover */
}
```

`prefers-color-scheme` 기능은 사용자가 밝은 테마나 어두운 테마를 요청했는지 감지합니다. 예시는 다음과 같습니다:

```css
@media (prefers-color-scheme: dark) {
  /* Styles for dark mode */
}
```

미디어 쿼리는 논리 연산자를 사용해 여러 조건을 결합할 수도 있습니다. `and` 연산자는 여러 미디어 기능을 결합하는 데 사용되고, `not`와 `only`는 미디어 쿼리를 부정하거나 분리하는 데 사용됩니다. 여러 기능을 결합한 예시는 다음과 같습니다:

```css
@media screen and (min-width: 768px) and (orientation: landscape) {
  /* Styles for landscape screens at least 768px wide */
}
```

또한 쉼표로 구분된 목록에서 여러 쿼리를 타겟팅할 수 있는데, 이는 "또는" 연산자처럼 작동합니다:

```css
@media screen and (min-width: 768px), print {
  /* Styles for screens at least 768px wide OR for print */
}
```

미디어 쿼리를 사용할 때는 캐스케이드를 고려하는 것이 중요합니다. 미디어 쿼리는 특이성을 높이지 않고 조건부 규칙을 그룹화할 뿐입니다. 각 미디어 쿼리 내에서는 CSS 캐스케이드의 일반 규칙이 여전히 적용됩니다.

실제로 미디어 쿼리는 반응형 레이아웃을 만드는 데 자주 사용됩니다. 일반적인 패턴은 모바일 장치에 기본 스타일을 정의하고, 미디어 쿼리를 사용해 더 큰 화면에 맞게 레이아웃을 향상시키는 것입니다:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<div class="container">
  <p>Lorem ipsum dolor sit amet, consectetur adipiscing elit. Integer nec odio. Praesent libero. Sed cursus ante dapibus diam.</p>
  <p>Sed nisi. Nulla quis sem at nibh elementum imperdiet. Duis sagittis ipsum. Praesent mauris. Fusce nec tellus sed augue semper porta.</p>
  <p>Mauris massa. Vestibulum lacinia arcu eget nulla. Class aptent taciti sociosqu ad litora torquent per conubia nostra, per inceptos himenaeos.</p>
</div>
```

```css
/* Base styles for mobile */
.container {
  width: 100%;
  padding: 15px;
}

/* Styles for tablets */
@media screen and (min-width: 768px) {
  .container {
    width: 750px;
    margin: 0 auto;
  }
}

/* Styles for desktops */
@media screen and (min-width: 1024px) {
  .container {
    width: 960px;
  }
}
```

:::

이 방법은 "모바일 우선" 반응형 디자인으로 알려져 있으며, 기본 스타일이 모바일 장치에 적합하도록 하고 더 큰 화면에 대한 향상을 추가합니다.

결론적으로, 미디어 쿼리는 CSS에서 반응형이고 적응 가능한 웹 디자인을 만드는 강력한 도구입니다. 다양한 미디어 유형과 기능을 이해하면 개발자가 다양한 장치와 선호도에 최적화된 사용자 경험을 제공하는 웹사이트를 만들 수 있습니다. 웹 기술이 계속 발전함에 따라 새로운 미디어 기능을 숙지하는 것은 더 세밀하고 사용자 친화적인 반응형 디자인을 만드는 데 도움이 됩니다.
