## Grundlagen von Objekten

- **Definition**: Ein Objekt ist eine Datenstruktur, die aus Eigenschaften besteht. Eine Eigenschaft besteht aus einem Schlüssel und einem Wert. Um auf Daten eines Objekts zuzugreifen, können Sie entweder die Punktnotation oder die Klammernotation verwenden.

:::interactive_editor

```js
const person = {
  name: "Alice",
  age: 30,
  city: "New York"
};

console.log(person.name);  // Alice
console.log(person["name"]); // Alice
```

:::

Um eine Eigenschaft eines bestehenden Objekts zu setzen, können Sie entweder die Punktnotation oder die Klammernotation zusammen mit dem Zuweisungsoperator verwenden.

:::interactive_editor

```js
const person = {
  name: "Alice",
  age: 30
};

person.job = "Engineer"
person["hobby"] = "Knitting"
console.log(person);  // {name: 'Alice', age: 30, job: 'Engineer', hobby: 'Knitting'}
```

:::

## Eigenschaften aus einem Objekt entfernen

- **`delete` Operator**: Dieser Operator wird verwendet, um eine Eigenschaft aus einem Objekt zu entfernen.

:::interactive_editor

```js
const person = {
  name: "Alice",
  age: 30,
  job: "Engineer"
};

delete person.job;

console.log(person.job); // undefined
```

:::

## Überprüfen, ob ein Objekt eine Eigenschaft besitzt

- **`hasOwnProperty()` Methode**: Diese Methode gibt einen Boolean zurück, der angibt, ob das Objekt die angegebene Eigenschaft als eigene Eigenschaft besitzt.

:::interactive_editor

```js
const person = {
  name: "Alice",
  age: 30
};

console.log(person.hasOwnProperty("name")); // true
console.log(person.hasOwnProperty("job")); // false
```

:::

- **`Object.hasOwn()` Methode**: Dies ist die moderne, empfohlene Methode, um zu überprüfen, ob ein Objekt eine Eigenschaft als eigene (nicht geerbte) besitzt. Die Syntax lautet `Object.hasOwn(object, propertyName)`. Sie gibt `true` zurück, wenn die Eigenschaft im Objekt existiert, und `false`, wenn nicht – unabhängig vom Wert der Eigenschaft. Das macht sie sicherer als `hasOwnProperty()` und zuverlässiger als `if (obj.prop)`, wenn Werte `0`, `false`, `null` oder `undefined` sein können.

:::interactive_editor

```js
const person = {
  name: "Alice",
  age: 30
};

console.log(Object.hasOwn(person, "name")); // true
console.log(Object.hasOwn(person, "job")); // false
```

:::

Hier ist ein detaillierteres Beispiel, das zeigt, warum `Object.hasOwn()` zuverlässiger ist als die direkte Wertüberprüfung:

:::interactive_editor

```js
const settings = {
  darkMode: false,
  fontSize: 0,
  language: null
};

// Object.hasOwn() correctly sees these properties exist
console.log(Object.hasOwn(settings, "darkMode")); // true (value is false, but exists)
console.log(Object.hasOwn(settings, "fontSize")); // true (value is 0, but exists)
console.log(Object.hasOwn(settings, "theme"));    // false (property was never added)

// Using if() directly is unsafe for falsy values!
if (settings.darkMode) {
  console.log("Dark mode on"); // Does NOT print — misleading!
}

// Object.hasOwn() is the safe way
if (Object.hasOwn(settings, "darkMode")) {
  console.log("darkMode exists, value is:", settings.darkMode); // darkMode exists, value is: false
}
```

:::

- **`in` Operator**: Dieser Operator gibt `true` zurück, wenn die Eigenschaft im Objekt existiert.

:::interactive_editor

```js
const person = {
  name: "Bob",
  age: 25
};

console.log("name" in person);  // true
```

:::

## Zugriff auf Eigenschaften aus verschachtelten Objekten

- **Datenzugriff**: Der Zugriff auf Eigenschaften aus verschachtelten Objekten erfolgt ähnlich wie bei einfachen Objekten mit der Punktnotation oder Klammernotation. Allerdings müssen Sie diese Zugriffe verkettet verwenden, um in die verschachtelte Struktur hineinzudringen.

:::interactive_editor

```js
const person = {
  name: "Alice",
  age: 30,
  contact: {
    email: "alice@example.com",
    phone: {
      home: "123-456-7890",
      work: "098-765-4321"
    }
  }
};

console.log(person.contact.phone.work); // "098-765-4321"
```

:::

## Primitive und nicht primitive Datentypen

- **Primitive Datentypen**: Diese Datentypen umfassen Zahlen, Strings, Booleans, `null`, `undefined` und Symbole. Diese Typen werden „primitiv“ genannt, weil sie einzelne Werte darstellen und keine Objekte sind. Primitive Werte sind unveränderlich, das heißt, sobald sie erstellt sind, kann ihr Wert nicht mehr geändert werden.
- **Nicht primitive Datentypen**: In JavaScript sind dies Objekte, zu denen reguläre Objekte, Arrays und Funktionen gehören. Im Gegensatz zu primitiven Typen können nicht primitive Typen mehrere Werte als Eigenschaften oder Elemente speichern.

## Methoden von Objekten

- **Definition**: Objektmethoden sind Funktionen, die mit einem Objekt verknüpft sind. Sie werden als Eigenschaften eines Objekts definiert und können auf die Daten des Objekts zugreifen und diese manipulieren. Das Schlüsselwort `this` innerhalb der Methode bezieht sich auf das Objekt selbst und ermöglicht so den Zugriff auf seine Eigenschaften.

:::interactive_editor

```js
const person = {
  name: "Bob",
  age: 30,
  sayHello: function() {
    return "Hello, my name is " + this.name;
  }
};

console.log(person.sayHello()); // "Hello, my name is Bob"
```

:::

## Objektkonstruktor

- **Definition**: In JavaScript ist ein Konstruktor eine spezielle Art von Funktion, die verwendet wird, um Objekte zu erstellen und zu initialisieren. Er wird mit dem Schlüsselwort `new` aufgerufen und kann Eigenschaften und Methoden auf dem neu erstellten Objekt initialisieren. Der `Object()` Konstruktor erstellt ein neues leeres Objekt.

```js
new Object()
```

## Arbeiten mit dem optionalen Verkettungsoperator (`?.`)

- **Definition**: Dieser Operator ermöglicht es Ihnen, sicher auf Objekteigenschaften zuzugreifen oder Methoden aufzurufen, ohne sich Sorgen machen zu müssen, ob sie existieren.

:::interactive_editor

```ts
const user = {
  name: "John",
  profile: {
    email: "john@example.com",
    address: {
      street: "123 Main St",
      city: "Somewhere"
    }
  }
};

console.log(user.profile?.address?.street); // "123 Main St"
console.log(user.profile?.phone?.number);   // undefined
```

:::

## Objekt-Destrukturierung

- **Definition**: Die Objekt-Destrukturierung erlaubt es Ihnen, Werte aus Objekten zu extrahieren und sie auf eine prägnantere und lesbarere Weise Variablen zuzuweisen.

:::interactive_editor

```js
const person = { name: "Alice", age: 30, city: "New York" };

const { name, age } = person;

console.log(name); // Alice
console.log(age);  // 30
```

:::

## Arbeiten mit JSON

- **Definition**: JSON steht für JavaScript Object Notation. Es ist ein leichtgewichtiges, textbasiertes Datenformat, das häufig zum Austausch von Daten zwischen einem Server und einer Webanwendung verwendet wird.

```js
{
  "name": "Alice",
  "age": 30,
  "isStudent": false,
  "list of courses": ["Mathematics", "Physics", "Computer Science"]
}
```

- **`JSON.stringify()`**: Diese Methode wird verwendet, um ein JavaScript-Objekt in einen JSON-String umzuwandeln. Das ist nützlich, wenn Sie Daten in einem Format speichern oder übertragen möchten, das leicht zwischen Systemen geteilt oder übertragen werden kann.

:::interactive_editor

```js
const user = {
  name: "John",
  age: 30,
  isAdmin: true
};

const jsonString = JSON.stringify(user);
console.log(jsonString); // '{"name":"John","age":30,"isAdmin":true}'
```

:::

- **`JSON.parse()`**: Diese Methode wandelt einen JSON-String zurück in ein JavaScript-Objekt um. Das ist nützlich, wenn Sie JSON-Daten von einem Webserver oder localStorage abrufen und die Daten in Ihrer Anwendung verarbeiten müssen.

:::interactive_editor

```js
const jsonString = '{"name":"John","age":30,"isAdmin":true}';
const userObject = JSON.parse(jsonString);

// result: { name: 'John', age: 30, isAdmin: true }
console.log(userObject);
```

:::
