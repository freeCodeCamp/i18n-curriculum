# --description--

セマンティックバージョニング（SemVer）は、バージョン番号に明確な意味を持たせるシステムで、開発者がソフトウェアにどのような変更が加えられたかをすぐに理解できるようにします。

SemVerは次の形式を使います：

```bash
MAJOR.MINOR.PATCH
```

例えば、人気のあるライブラリ`lodash`は現在バージョン`4.17.21`です。

各部分の意味は次の通りです：

* MAJOR — 既存のコードを壊す大きな変更。例：既存のコードが依存している関数を削除または名前変更すること。
    
* MINOR — 既存のコードを壊さない新機能。例：新しい関数やオプションのパラメータを追加すること。
    
* PATCH — 小さな修正や改善。例：動作を変えずにバグやタイプミスを修正すること。
    

<img src="https://cdn.freecodecamp.org/curriculum/lecture-transcripts/what-is-semantic-versioning-1.png" alt="In the version number 4.17.21, 4 is the Major Version, it's changed for breaking changes, the 17 is the minor version, it's changed for non-breaking changes, and the 21 is the patch, it's changed for bug fixes and small features">

例えば、`my-package`というパッケージがあるとします。セマンティックバージョニングを使った場合のバージョン番号の変化例は次の通りです：

| バージョン | 変更内容 |
| --- | --- |
| 1.0.0 | 初期の安定リリース |
| 1.1.0 | 新しいオプション機能 |
| 1.1.1 | 小さなバグ修正や機能追加 |
| 2.0.0 | 破壊的変更の導入 |

しかし、npmのバージョン番号にはさらに意味があります。パッケージをインストールすると、`package.json`ファイルのバージョンは次のようになっているかもしれません：

```json
"lodash": "^4.17.21" 
```

すでに`4`がメジャーバージョン、`17`がマイナーバージョン、`21`がパッチバージョンであることは学びました。しかし、前にあるキャレット記号（`^`）は何を意味するのでしょうか？

キャレット記号は、npmがマイナーおよびパッチの新しいバージョン（`4.x.x`）を自動的にインストールできるようにしますが、メジャーバージョンの新しいもの（`5.x.x`）はインストールしません。なぜなら、メジャーアップデートは破壊的変更を含む場合があるからです。

メジャーバージョンが`0`の場合、キャレットはより制限的になります。例えば、`^0.2.3`は`0.2.4`のようなアップデートを許可しますが、`0.3.0`は許可しません。なぜなら、`0`で始まるメジャーバージョンは安定性が低いと見なされるからです。

キャレットは、`npm`や他のJavaScriptパッケージマネージャーが依存関係のバージョン管理に使う記号の一つに過ぎません。

チルダ記号（`~`）はパッチバージョンのアップデートのみを許可します。つまり、`~1.2.3`は`1.2.4`や`1.2.9`を受け入れますが、`1.3.0`は受け入れません。

また、より大きい（`>`）、より小さい（`<`）、およびそれらの「または等しい」バージョンを使って範囲内のバージョンを受け入れることもできます。例えば、`>=1.2.3 <2.0.0`は`1.2.3`から`2.0.0`未満の任意のバージョンを意味します。

アスタリスク記号（`*`）は任意のバージョンにマッチし、すべてを受け入れます。これはテストには理想的ですが、本番環境には推奨されません。

最後に、バージョン番号に記号を付けずに正確なバージョンを固定することもできます。例えば、バージョン`1.2.3`は`1.2.3`のみを受け入れます。

簡単なまとめの表はこちらです：

| 記号 | 意味 | 例の範囲 |
| --- | --- | --- |
| `^` | マイナー＋パッチのアップデートを許可 | `^1.2.3` → `1.x.x` |
| `~` | パッチのアップデートのみ許可 | `~1.2.3` → `1.2.x` |
| `>`, `<`, `>=`, `<=` | カスタム範囲 | `>=1.2.3` または `<2.0.0` |
| **\*** | 任意のバージョン | すべてのバージョン |
| なし | 正確なバージョン | `1.2.3` |
