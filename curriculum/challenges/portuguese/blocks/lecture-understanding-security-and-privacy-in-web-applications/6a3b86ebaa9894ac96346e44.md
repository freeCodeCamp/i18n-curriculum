# --description--

CSP (Content Security Policy) e Permissions-Policy são dois poderosos cabeçalhos de segurança HTTP que ajudam a proteger aplicações web ao restringir quais recursos podem ser carregados e quais recursos do navegador uma página pode acessar.

Content Security Policy foca principalmente no carregamento de recursos. Ela atua como um filtro de segurança para seus scripts, estilos e imagens no frontend.

Atacantes maliciosos frequentemente tentam injetar scripts nocivos em sites vulneráveis. Uma CSP forte impede esses ataques de Cross-Site Scripting (XSS) ao estabelecer uma lista branca rigorosa de domínios confiáveis.

Quando um servidor envia o cabeçalho CSP, o navegador lê as diretivas específicas. Por exemplo, a diretiva `script-src` informa ao navegador quais domínios podem executar JavaScript com segurança na página.

Se um hacker tentar injetar um script malicioso de um servidor externo não aprovado, o navegador detecta a violação da política, descarta o script imediatamente e registra um erro de segurança.

A CSP também pode restringir fontes de imagens, destinos de formulários e origens de folhas de estilo.

Permissions-Policy foca em recursos do navegador e hardware do dispositivo. Ela substituiu o antigo cabeçalho Feature-Policy.

Esse mecanismo de segurança permite que os proprietários do site habilitem ou desabilitem explicitamente recursos específicos do navegador para sua própria página e quaisquer iframes incorporados, além de impedir que trechos de terceiros abusem da privacidade do usuário.

Usando o cabeçalho Permissions-Policy, você pode controlar o acesso a hardware como webcam, microfone e rastreamento de geolocalização.

Por exemplo, se seu site não precisa saber a localização do usuário, você pode configurar a política para bloquear totalmente a API de geolocalização. Mesmo que um invasor comprometa um script na sua página, o navegador recusará ativar o recurso de rastreamento.

A implementação do Permissions-Policy usa um formato claro e estruturado no cabeçalho que nomeia o recurso e define suas origens permitidas.

Para restringir um recurso de hardware estritamente ao seu próprio domínio, você usa a palavra-chave `self` entre parênteses, como em `camera=(self)`. E se quiser desabilitar um recurso em toda a página para todos os usuários, usa-se uma lista vazia, que fica assim: `geolocation=()`.

Combinar ambos os cabeçalhos oferece uma defesa sólida para aplicações web. A CSP bloqueia a entrega de dados e previne injeção de código, e a Permissions-Policy protege o dispositivo físico e garante conformidade com a privacidade.

Desenvolvedores web modernos usam ambas as ferramentas para criar um ambiente de navegação fortemente protegido.
