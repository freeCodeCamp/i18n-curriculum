# --interactive--

`DOMContentLoaded` 이벤트는 HTML 문서의 모든 내용이 로드되고 구문 분석되었을 때 발생합니다. 외부 스타일시트나 이미지가 있다면, `DOMContentLoaded` 이벤트는 그것들이 로드될 때까지 기다리지 않습니다. 오직 HTML이 로드될 때까지만 기다립니다.

이는 외부 스타일시트, 이미지 등을 포함한 모든 것이 로드될 때까지 기다리는 `load` 이벤트와 다릅니다.

다음은 `DOMContentLoaded` 이벤트를 사용하는 예제 구문입니다:

```js
document.addEventListener("DOMContentLoaded", function () {
  console.log("DOM is loaded.");
});
```

DOM이 로드되면 함수가 실행되고 메시지 `DOM is loaded.`가 콘솔에 기록됩니다.

이제 `DOMContentLoaded` 이벤트를 사용하는 또 다른 예제를 살펴보겠습니다. 이 예제에서는 HTML 안에 이미지가 있습니다. 이미지를 업데이트하려면 이미지의 `src`와 `alt` 속성을 변경하는 함수를 만들 수 있습니다:

:::interactive_editor

```html
<h1>Image Change on DOM Loaded</h1>
<img
  id="example-img"
  src="https://cdn.freecodecamp.org/curriculum/cat-photo-app/relaxing-cat.jpg"
  alt="Cat relaxing"
/>

<script src="index.js"></script>
```

```js
function changeImg() {
  const img = document.getElementById("example-img");
  img.src =
    "https://cdn.freecodecamp.org/curriculum/responsive-web-design-principles/FCCStickers-CamperBot200x200.jpg";
  img.alt = "CamperBot sticker";
  console.log("image was just changed");
}

changeImg();
```

:::

그다음 DOM이 아직 로드 중인지 확인하고 `DOMContentLoaded` 이벤트에 대한 이벤트 리스너를 추가할 수 있습니다. 만약 `DOMContentLoaded` 이벤트가 이미 발생했다면 `changeImg` 함수를 직접 호출할 수 있습니다:

```js
if (document.readyState === "loading") {
  document.addEventListener("DOMContentLoaded", changeImg);
} else {
  console.log("DOMContentLoaded has already fired");
  changeImg();
}
```

이 코드를 실행하고 페이지를 새로 고침하면 DOM이 로드된 후 이미지가 새 이미지로 바뀌는 것을 볼 수 있습니다. 꽤 빠르게 일어나지만 이미지가 바뀔 때 약간 깜박임이 보일 것입니다.

`DOMContentLoaded` 이벤트는 DOM이 로드되자마자 JavaScript 코드를 실행하고 싶지만 이미지나 스타일시트 같은 모든 외부 리소스가 로드되기 전일 때 유용합니다.
