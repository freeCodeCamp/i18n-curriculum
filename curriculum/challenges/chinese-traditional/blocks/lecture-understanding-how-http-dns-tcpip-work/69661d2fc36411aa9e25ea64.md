# --description--

讓我們來了解常見的 HTTP 回應碼。

HTTP 回應碼是三位數字碼，網頁伺服器用來與客戶端溝通，讓他們知道其請求的處理結果。

它們提供有關請求是否成功處理或在處理請求時是否發生錯誤的資訊。

我們將在本講座中涵蓋一些最常見的項目。

## 成功代碼

讓我們從**成功代碼**開始。這些代碼以數字 2 開頭。

狀態碼 **200 OK** 是最常見的 HTTP 回應碼。它表示請求成功，且伺服器傳回了客戶端所要求的資訊。

當客戶端請求創建資源且處理程序成功完成時，伺服端會傳回狀態碼 **201 Created**。例如，如果成功創建一個新的使用者帳號，伺服端會將此回應碼傳回給客戶端。

當請求已成功被控制代碼處理程序，但伺服端沒有內容可傳回時，會傳回狀態碼 **204 No Content**。這可用於刪除或更新資料庫中資源但沒有資料可傳回的處理程序。

## **重新導向代碼**

現在讓我們來談談**重新導向代碼**。

當客戶端請求的資源已移動到不同位置或客戶端需要採取額外步驟以完成請求時，這些代碼會由伺服器傳回。

這些程式碼以數字 3 開頭。

當客戶端請求的資源已永久移動到另一個位置時，會傳回狀態碼 **301 Moved Permanently**。在這種情況下，伺服器應在回應中包含新位置的 URL，且當客戶端收到回應後，使用者應被重新導向到此 URL。

程式碼 **302 Found** 顯示客戶端請求的資源已暫時移動到另一個位置。伺服器應在回應中提供新的 URL，客戶端應將使用者重新導向到此 URL。然而，原始 URL 未來可能仍可用，因為此變更是暫時的。這是回應碼 301 和 302 之間的主要差異。

伺服器傳回 **304 Not Modified** 程式碼，以告訴客戶端所請求的資源自上次請求以來未被修改。若內容未改變，伺服器會傳送此程式碼以避免重複傳送已傳送的資料。這是一種效率優化技術。

## 客戶端錯誤代碼

現在我們將開始深入探討**錯誤代碼**。

錯誤可能由客戶端和伺服端引起。我們將先討論客戶端錯誤代碼。

當請求因客戶端產生的問題而無法完成時，會傳回客戶端錯誤代碼。

這些錯誤代碼以數字 4 開頭。

當伺服器無法理解客戶端所傳送的請求時，會傳回 **400 Bad Request** 代碼。當客戶端傳送語法錯誤、參數無效或資料無效的請求時，可能會發生此情況。

**401 Unauthorized** 狀態碼與驗證有關。它顯示客戶端必須提供某些驗證憑證，才能存取所請求的資源。這些憑證可能包括使用者名稱和密碼。

當客戶端未被授權存取所請求的資源時，會傳回 **403 Forbidden** 代碼。401 與 403 的差異在於，即使客戶端擁有有效的驗證憑證，403 表示該客戶端對其所請求的特定資源沒有必要的權限。

當伺服器找不到資源時，會傳回 **404 Not Found** 代碼。可能是資源已被刪除，或是 URL 中有拼寫錯誤，導致伺服器無法找到該資源。伺服器也可能遇到暫時性問題，導致此錯誤代碼。

## 伺服端錯誤代碼

接下來，我們有**伺服器錯誤代碼**。當請求因伺服器端的問題無法處理時，這些代碼會由伺服器傳回。

**500 Internal Server Error** 錯誤代碼顯示伺服端發生了意外錯誤。此錯誤可能由多種問題引起，範圍從設定問題到暫時性問題。

當伺服器嘗試與另一台伺服器通訊但收到該伺服器的無效回應時，會傳回 **502 Bad Gateway** 錯誤代碼。此類錯誤有廣泛的可能原因。

**503 Service Unavailable** 錯誤代碼顯示伺服器暫時無法控制代碼該請求，因此客戶端應該稍後再試。此錯誤的常見原因包括維護工作以及伺服器目前無法控制代碼的高流量請求。

HTTP 回應碼對於理解客戶端與網頁伺服器之間的通訊至關重要。透過理解這些碼，你可以排除故障並偵錯你的網頁應用程式。
