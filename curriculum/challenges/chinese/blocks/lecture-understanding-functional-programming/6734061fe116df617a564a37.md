# --interactive--

柯里化是一种技术，我们将一个接受多个参数的函数转换为一系列函数，每个函数只接受一个参数。

让我们从一个基本示例开始。假设我们有一个将两个数字相加的函数：

:::interactive_editor

```js
function add(a, b) {
  return a + b;
}

console.log(add(3, 4)); // 7
```

:::

这是一个接受两个参数并返回它们和的函数。现在，让我们看看如何对这个函数进行柯里化：

:::interactive_editor

```js
function curriedAdd(a) {
  return function(b) {
    return a + b;
  }
}

console.log(curriedAdd(3)(4)); // 7
```

:::

在这个柯里转换的代码中，我们不是一次接受两个参数，而是有一个函数接受第一个参数并返回另一个函数。这个返回的函数随后接受第二个参数并执行加法。我们像 `curriedAdd(3)(4)` 这样调用它，其中每对括号表现一次函数调用。

但是为什么我们要这样做？ 

柯里化使我们能够轻松创建一些特殊的函数。例如，我们可以创建一个函数，总是将五添加到任何数字上：

:::interactive_editor

```js
function curriedAdd(a) {
  return function(b) {
    return a + b;
  }
}

const addFive = curriedAdd(5);
console.log(addFive(10)); // 15
console.log(addFive(20)); // 25
```

:::

这里，`addFive` 是一个函数，它总是准备好为我们提供的任何数字添加五。这是部分应用的一个简单示例，我们将一定数量的参数固定到一个函数上，从而生成一个接受更少参数的函数。

虽然我们的示例集中在具有两个参数的函数上，但柯里化可以应用于具有任意数量参数的函数。

随着你在 JavaScript 中的旅程继续，你很可能会发现许多情况下柯里化可以使你的代码更简洁、更具表现力。

## --text--

以下哪项是 JavaScript 中柯里的函数示例？

## --answers--

```js
function add(a, b, c) {
  return a + b + c;
}
```

### --feedback--

这是一个一次性接收所有参数的普通函数。

---

```js
function add(a) {
  return function (b) {
    return function (c) {
      return a + b + c;
    };
  };
}
```

---

```js
function add(a, b, c) {
  return a() + b() + c();
}
```

### --feedback--

此函数一次接受所有参数并像调用函数一样调用它们，这不符合柯里函数的定义。

---

```js
function add(a) {
  function inner(b) {
    return a + b;
  }
}
```

### --feedback--

内部函数从不被返回，所以 `add(2)` 会返回 `undefined`，而 `add(2)(3)` 会抛出误差。
