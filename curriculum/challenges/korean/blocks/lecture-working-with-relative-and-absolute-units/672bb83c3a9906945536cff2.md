# --interactive--

CSS에서 `vh`와 `vw`는 뷰포트 기준 단위로, 브라우저 창 크기에 따라 요소 크기를 조절할 수 있게 해줍니다. 이 단위들은 다양한 화면 크기에 맞춰 조정되는 반응형 디자인을 만들 때 특히 유용합니다.

`vh`는 "뷰포트 높이"를 의미하며, `1vh`는 뷰포트 높이의 1%와 같습니다.

마찬가지로, `vw`는 "뷰포트 너비"를 의미하며, `1vw`는 뷰포트 너비의 1%와 같습니다.

즉, 요소의 높이를 `100vh`로 설정하면, 기기의 실제 픽셀 크기와 상관없이 뷰포트 전체 높이를 차지하게 됩니다.

이 단위들은 전체 화면 레이아웃이나 화면의 특정 비율을 유지하는 요소를 만들 때 특히 편리합니다.

예를 들어, 항상 화면 전체를 채우는 히어로 섹션을 만들고 싶을 때 사용할 수 있습니다(예제를 직접 조작하려면 인터랙티브 편집기를 활성화하세요).

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css" />

<section class="hero">
  <h1>100vh / 100vw Example</h1>
  <p>This section fills the entire browser window.</p>
</section>
```

```css
body {
  margin: 0;
  font-family: sans-serif;
  border: 5px dashed #333;
}

.hero {
  height: 100vh;
  width: 100vw;
  background-color: #add8e6; 
  padding: 2em;
}

.hero h1 {
  font-size: 2em;
  margin-bottom: 0.5em;
}

.hero p {
  font-size: 1em;
}
```

:::

이 CSS는 히어로 섹션이 기기의 화면 크기와 상관없이 항상 뷰포트 크기와 정확히 일치하도록 보장합니다.

`vh`와 `vw` 단위는 타이포그래피에도 사용되어 반응형 텍스트 크기를 만들 수 있습니다.

인터랙티브 편집기를 활성화하고 미리보기 창 크기를 조절해 텍스트가 뷰포트 크기에 따라 어떻게 조정되는지 확인하세요:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css" />
<h1>Responsive Heading</h1>
<p>The h1 heading scales with the viewport width.</p>
```

```css
h1 {
  font-size: 5vw;
}
```

:::

이렇게 하면 `h1` 요소의 폰트 크기가 뷰포트 너비의 5%로 설정되어, 브라우저 창 크기에 따라 텍스트가 부드럽게 조절됩니다.

`vh`와 `vw` 단위의 장점 중 하나는 뷰포트 크기 변경에 실시간으로 반응한다는 점입니다. 즉, 사용자가 브라우저 창 크기를 조절하면, 이 단위로 크기 지정된 요소가 페이지를 다시 로드하지 않고도 자동으로 조정됩니다. 하지만 이 단위를 신중하게 사용해야 합니다. 예를 들어, 폰트 크기를 `vw` 단위만으로 설정하면 좁은 화면에서는 텍스트가 너무 작아지거나 넓은 화면에서는 너무 커질 수 있습니다.

또 다른 고려사항은 모바일 기기에서 브라우저 주소 표시줄이 나타나거나 사라질 때 뷰포트 높이가 변할 수 있다는 점입니다. `vh` 단위를 많이 사용하면 예상치 못한 레이아웃 변화가 발생할 수 있습니다.

요약하자면, `vh`와 `vw` 단위는 뷰포트 크기에 맞춰 조정되는 반응형 레이아웃과 요소를 만드는 데 강력한 도구입니다. 전체 화면 섹션, 비율 유지, 부드럽게 크기 조절되는 디자인에 특히 유용합니다. 하지만 최상의 사용자 경험을 위해 다른 CSS 기법과 함께 신중하게 사용하는 것이 좋습니다.
