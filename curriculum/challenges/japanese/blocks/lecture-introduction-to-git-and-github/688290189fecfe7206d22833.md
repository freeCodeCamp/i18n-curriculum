# --description--

前のレッスンでGitHubについて簡単に触れました。GitHubとGitLabはどちらもバージョン管理プロバイダーです。これは、クラウドベースのソリューションで、バージョン管理されたプロジェクトを「リポジトリ」と呼ばれる場所に保存し、そのプロジェクトで使えるコラボレーション機能を提供することを意味します。

これらのプラットフォームが提供する機能に入る前に、まずプロジェクトの種類について説明します。多くのプロジェクトは「オープンソース」と見なされ、公開したコードを誰でも見られ、変更を提案したり、問題を報告したり、修正したバージョンを実行したりできます。

しかし、時にはチームメイトだけにコードを見せて作業してほしい場合もあります。その場合は「クローズドソース」としてプロジェクトを公開できます。これは、明示的に許可した人だけがプロジェクトを見て操作できることを意味します。

どちらの方法を取るにしても、GitHubやGitLabのようなサイトはチームのコラボレーションを大幅に向上させます。

例えば、機能を作っているチームメイトが行き詰まって質問があるとします。コードをDiscordにコピー＆ペーストして読んでもらったり、画面共有のために通話したりする代わりに、変更を直接GitHubにアップロードできます。そこからブラウザで変更を確認したり、変更をダウンロードしてローカルで実行・デバッグしたりできます。

チームメイトが機能を完成させたら、これらのウェブサイトを通じて変更を提案できます。あなたは変更をレビューし、特定のファイルや行にコメントを残して意見や質問を伝えられます。ブラウザ上で提案された編集を適用することも可能です！

変更に満足したら、それを受け入れてメインブランチにマージできます。これにより、チーム全員が最新の変更をローカルにダウンロードでき、次の作業を始めるときに最新のコードを使えます。

作業と言えば、これらのサイトは強力なイシュートラッカーも提供しています。チームメンバーにチケットを割り当てたり、一般からのバグ報告を受け付けたり、プロジェクトボードのような機能でスプリント全体を計画したりできます。

これらのサイトには他にも多くの機能があります。次のレッスンで主要なポイントをカバーし、すぐにコラボレーションを始められるようにします！
