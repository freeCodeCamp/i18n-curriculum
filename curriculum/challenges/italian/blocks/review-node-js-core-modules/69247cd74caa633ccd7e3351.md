## Introduzione ai moduli core di Node.js

- **Moduli core di Node.js**: moduli integrati che vengono preinstallati con Node.js, offrendo funzionalità essenziali per operazioni su file, crittografia, interazioni con il sistema operativo e altro.
- **Moduli standard**: non richiedono installazioni separate; disponibili finché Node.js è installato.
- **Sintassi di importazione**: usa `require()` per importare i moduli core nella tua applicazione.

```javascript
const fs = require("fs");
const crypto = require("crypto");
const os = require("os");
```

## Modulo File System (fs)

- **Scopo**: fornisce metodi per lavorare con file e cartelle, inclusi apertura, chiusura, lettura, scrittura ed eliminazione.
- **Importazione**: `const fs = require("fs");`
- **Flessibilità**: metodi disponibili in tre forme: callback, promise e sincrona.

### Pattern di utilizzo dei metodi

- **Asincrono con callback**: comportamento predefinito, non bloccante.

```javascript
fs.writeFile("filePath", "content", "utf8", (err) => {
  if (err) throw err;
  console.log("File written to!");
});
```

- **Basato su promise**: preferito per applicazioni moderne, usa `async/await`.

```javascript
const fs = require("fs/promises");

async function writeToFile() {
  try {
    await fs.writeFile("article.md", "## Node fs Module");
    console.log("File written to!");
  } catch (err) {
    console.error(err);
  }
}
```

- **Sincrono**: operazione bloccante, adatta a script semplici.

```javascript
try {
  fs.writeFileSync("article.md", "## Node fs Module", "utf8");
  console.log("File written to!");
} catch (err) {
  console.error(err);
}
```

### Operazioni core sui file

- **Metodo `writeFile()`**: crea o sovrascrive un file con contenuto specificato.

```javascript
await fs.writeFile("article.md", "## Node fs Module: The Complete Guide");
```

- **Metodo `appendFile()`**: aggiunge contenuto alla fine di un file esistente.

```javascript
await fs.appendFile("article.md", "\n\nIn this article, you will learn...");
```

- **Metodo `readFile()`**: legge il contenuto di un file.

```javascript
const content = await fs.readFile("article.md", "utf8");
console.log("File content:", content);
```

- **Metodo `unlink()`**: elimina un file dal file system.

```javascript
await fs.unlink("article.md");
console.log("File deleted successfully");
```

### Codifica dei caratteri

- **Codifica UTF-8**: specifica `"utf8"` per leggere i file come stringhe di testo.
- **Formato buffer**: senza specifica di codifica, i file vengono letti come oggetti buffer binari.

```javascript
// Returns string
const textContent = await fs.readFile("file.txt", "utf8");

// Returns Buffer
const bufferContent = await fs.readFile("file.txt");
```

## Modulo Buffer

- **Scopo**: gestisce dati binari direttamente in memoria per file, immagini e flussi di rete.
- **Importazione**: `const { Buffer } = require("buffer");`
- **Casi d'uso**: I/O su file, flussi TCP, elaborazione immagini e manipolazione di dati binari.

### Creazione di buffer

- **Metodo `Buffer.from()`**: crea buffer da stringhe, array o altri dati.

```javascript
// From string
const myStrBuffer = Buffer.from("freeCodeCamp");
console.log(myStrBuffer); // <Buffer 66 72 65 65 43 6f 64 65 43 61 6d 70>

// From array of numbers
const myNumBuffer = Buffer.from([70, 82, 69, 69, 67, 79, 68, 69, 67, 65, 77, 80]);
```

- **Metodo `Buffer.alloc()`**: crea buffer di dimensione specificata riempito di zeri.

```javascript
const someBuffer = Buffer.alloc(10);
console.log(someBuffer); // <Buffer 00 00 00 00 00 00 00 00 00 00>
```

### Operazioni su buffer

- **Accesso simile ad array**: accedi agli elementi del buffer usando la notazione con indice.

```javascript
console.log(myStrBuffer[0]); // 102 (byte value for 'f')
```

- **Metodo `toString()`**: converte il buffer in una stringa leggibile.

```javascript
console.log(myStrBuffer.toString()); // "freeCodeCamp"
```

- **Metodo `Buffer.write()`**: scrive dati in un buffer allocato.

```javascript
someBuffer.write("Hello fCC");
console.log(someBuffer.toString()); // "Hello fCC"
```

- **Metodo `Buffer.byteLength()`**: restituisce il numero di byte necessari per memorizzare una stringa.

```javascript
console.log(Buffer.byteLength("Hello freeCodeCamp")); // 18
```

### Metodi aggiuntivi per buffer

- **`Buffer.isBuffer()`**: verifica se un oggetto è un buffer.
- **`Buffer.compare()`**: confronta due buffer e restituisce l’ordine per ordinamento.
- **`Buffer.concat()`**: unisce più buffer in uno solo.

## Modulo Crypto

- **Scopo**: fornisce funzionalità crittografiche, inclusi hashing, crittografia, decrittografia e firme digitali.
- **Importazione**: `const crypto = require("crypto");`
- **Nota sulla sicurezza**: usa librerie ben testate come `bcrypt` o `jsonwebtoken` per l’autenticazione in produzione.

### Metodi di hashing

- **Metodo `createHash()`**: crea hash unidirezionali usando algoritmi come SHA256, SHA512 o MD5.

```javascript
const hashedPassword = crypto
  .createHash("sha256")
  .update("myPassword123")
  .digest("hex");

console.log("createHash result:", hashedPassword);
```

- **Metodo `createHmac()`**: crea un hash con chiave segreta per autenticazione e integrità dei dati.

```javascript
const hashedMessage = crypto
  .createHmac("sha256", "secret-key")
  .update("Hello World")
  .digest("hex");
```

### Crittografia e decrittografia

- **Metodi `createCipheriv()` e `createDecipheriv()`**: crittografano e decrittografano dati usando algoritmi, chiavi e vettori di inizializzazione.

```javascript
const key = Buffer.from("12345678901234567890123456789012"); // 32 bytes for AES-256
const iv = Buffer.from("1234567890123456"); // 16 bytes for AES

// Encryption
const cipher = crypto.createCipheriv("aes-256-cbc", key, iv);
let encrypted = cipher.update("Hello campers!", "utf8", "hex");
encrypted += cipher.final("hex");

// Decryption
const decipher = crypto.createDecipheriv("aes-256-cbc", key, iv);
let decrypted = decipher.update(encrypted, "hex", "utf8");
decrypted += decipher.final("utf8");
```

### Generazione di valori casuali

- **Metodo `randomBytes()`**: genera byte casuali crittograficamente sicuri.

```javascript
console.log("Random Bytes:", crypto.randomBytes(16).toString("hex"));
// Output: a6154ef5a296fa176ad0f332bd94d712
```

- **Metodo `randomInt()`**: genera numeri interi casuali sicuri entro un intervallo specificato.

```javascript
console.log("Random Int:", crypto.randomInt(0, 100)); // Random number 0-99
```

### Gestione delle chiavi

- **Metodo `createSecretKey()`**: crea oggetti chiave crittografica da byte grezzi.

```javascript
const secret = crypto.createSecretKey(crypto.randomBytes(32));
console.log(secret.export().toString('hex'));
```

### Firme digitali

- **Metodi `sign()` e `verify()`**: creano e validano firme digitali usando coppie di chiavi private/pubbliche.
- **Caso d’uso**: dimostra autenticità e integrità dei dati.

### Metodi aggiuntivi di Crypto

- **`createPublicKey()` e `createPrivateKey()`**: lavorano con chiavi generate esternamente.
- **`createDiffieHellman()`**: abilita la generazione sicura di segreti condivisi.
- **`Certificate()`**: gestisce operazioni su certificati HTTPS.

## Modulo OS

- **Scopo**: fornisce utility relative al sistema operativo per informazioni di sistema e compatibilità multipiattaforma.
- **Importazione**: `const os = require("os");`
- **Casi d’uso**: monitoraggio sistema, scripting multipiattaforma, informazioni hardware.

### Metodi per informazioni di sistema

- **Metodo `platform()`**: restituisce la piattaforma del sistema operativo.

```javascript
console.log(os.platform()); // 'darwin', 'win32', 'linux', etc.

// Cross-platform usage
if (os.platform() === 'win32') {
  // Windows-specific code
} else {
  // Unix-like systems
}
```

- **Metodo `arch()`**: restituisce l’architettura della CPU.

```javascript
console.log(os.arch()); // 'arm64', 'x64', 'arm', 'ia32', etc.
```

- **Metodo `type()`**: restituisce il nome ufficiale del sistema operativo.

```javascript
console.log(os.type()); // 'Darwin', 'Linux', 'Windows_NT'
```

### Informazioni sulla versione

- **Metodo `release()`**: mostra la versione del kernel OS.

```javascript
console.log(os.release()); // '25.0.0'
```

- **Metodo `version()`**: restituisce informazioni dettagliate sulla versione OS.

```javascript
console.log(os.version());
// Darwin Kernel Version 25.0.0: Wed Sep 17 21:41:39 PDT 2025
```

### Informazioni hardware

- **Metodo `cpus()`**: restituisce un array con informazioni sui core CPU.

```javascript
console.log(os.cpus()); // Array of CPU core objects with model, speed, times
```

- **Metodo `uptime()`**: restituisce il tempo di attività del sistema in secondi.

```javascript
console.log(os.uptime()); // 23047 (seconds since system boot)
```

### Informazioni sulla memoria

- **Metodo `totalmem()`**: restituisce la memoria totale del sistema in byte.
- **Metodo `freemem()`**: restituisce la memoria disponibile del sistema in byte.

## Buone pratiche

### Considerazioni sulle prestazioni

- **Evita metodi sincroni**: usa metodi asincroni nelle applicazioni di produzione per evitare blocchi.
- **Preferisci le promise**: usa `async/await` con API basate su promise per codice più pulito e manutenibile.
- **Gestione degli errori**: implementa sempre una corretta gestione degli errori con blocchi try-catch o callback di errore.

### Linee guida per la sicurezza

- **Modulo Crypto**: usa librerie consolidate per l’autenticazione invece di creare soluzioni crittografiche personalizzate.
- **Validazione input**: valida i percorsi dei file e gli input utente per prevenire vulnerabilità di sicurezza.
- **Gestione delle chiavi**: conserva le chiavi crittografiche in modo sicuro, mai hardcodate nel codice sorgente.

### Sviluppo multipiattaforma

- **Uso del modulo OS**: usa i metodi del modulo OS per la compatibilità multipiattaforma.
- **Gestione dei percorsi**: usa il modulo `path` per operazioni sui percorsi file multipiattaforma.
- **Variabili d’ambiente**: controlla variabili d’ambiente e configurazioni specifiche della piattaforma.

## Casi d’uso comuni
