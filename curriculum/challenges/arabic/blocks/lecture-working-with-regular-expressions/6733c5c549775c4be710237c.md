# --description--

دعنا نتعلم كيف نطابق أو نستبدل كل التكرارات لنمط معين في سلسلة نصية.

لقد تعلمت سابقًا عن الطريقتين `replace()` و`match()`، بالإضافة إلى المعدل العالمي `g`. الآن يمكنك دمج تلك المعرفة للتعامل مع كل الأنماط في السلسلة النصية. دعنا نسترجع شفرة المطابقة الأصلية:

```js
const regex = /freeCodeCamp/;
const match = "freeCodeCamp".match(regex);
console.log(match);
```

وها هو كائن المطابقة الناتج لدينا:

```js
// [
//   'freeCodeCamp',
//   index: 0,
//   input: 'freeCodeCamp',
//   groups: undefined
// ]
```

لكن ماذا لو كانت لدينا سلسلة تحتوي على تكرارات متعددة لـ `freecodecamp` للمطابقة؟ لنلق نظرة على كيفية تصرف `match()` في هذه الحالة. سنضيف أيضًا مثالنا القديم لـ `replace()` للمقارنة:

```js
const regex = /freecodecamp/;
const str = "freecodecamp is the best we love freecodecamp";
const matched = str.match(regex);
const replaced = str.replace(regex, "freeCodeCamp");
console.log(matched);
console.log(replaced);
```

والنتيجة هي هذه:

```js
// [
//   'freecodecamp',
//   index: 0,
//   input: 'freecodecamp is the best we love freecodecamp',
//   groups: undefined
// ]
// freeCodeCamp is the best we love freecodecamp
```

يا للأسف! `match()` أعادت فقط أول تطابق، و`replace()` استبدلت فقط أول تطابق. هذا لأن `match()` و`replace()` يعملان افتراضيًا فقط على أول ظهور للنمط.

لحسن الحظ، يمكنك تجنب ذلك باستخدام المعدل العالمي على التعبير النمطي الخاص بك. دعنا نضيفه إلى تعبيرنا:

```js
const regex = /freecodecamp/g;
const str = "freecodecamp is the best we love freecodecamp";
const matched = str.match(regex);
const replaced = str.replace(regex, "freeCodeCamp");
console.log(matched);
console.log(replaced);
```

ولنتأكد من النتيجة:

```js
// [ 'freecodecamp', 'freecodecamp' ]
// freeCodeCamp is the best we love freeCodeCamp
```

نجح ذلك! استدعاء الاستبدال لدينا استبدل كل سلاسل `freecodecamp` المكتوبة بأحرف صغيرة، وطريقة `match()` طابقت كلاهما.

الشيء المثير هنا هو أنه عند استخدام المعدل العالمي مع `match()`، تفقد المعلومات الإضافية عن مجموعات الالتقاط وفهارس السلسلة التي تأتي في مصفوفة المطابقة.

لحسن الحظ، جلب تحديث ECMAScript لعام 2019 طريقتين جديدتين: `matchAll()` و`replaceAll()`. مثل نظيراتها المفردة، تقبل هذه الطرق سلسلة نصية أو تعبيرًا نمطيًا، و`replaceAll()` تقبل أيضًا معلمة ثانية كسلسلة للاستبدال.

لكن على عكس الطرق السابقة، سترمي `replaceAll()` و`matchAll()` خطأً إذا أعطيتها تعبيرًا نمطيًا بدون المعدل العالمي. دعنا نحدث شفرتنا لاستخدام هذه الطرق الجديدة:

```js
const pattern = "freecodecamp";
const str = "freecodecamp is the best we love freecodecamp";
const matched = str.matchAll(pattern);
const replaced = str.replaceAll(pattern, "freeCodeCamp");
console.log(matched);
console.log(replaced);
```

وها هي النتيجة:

```js
// {}
// freeCodeCamp is the best we love freeCodeCamp
```

أخبار جيدة! عملت `replaceAll()` تمامًا كما أردنا: استبدلت كل تكرارات `freecodecamp` بالأحرف الصغيرة بالإصدار الصحيح بتنسيق سنام الجمل.

لكن ما هذا الكائن الفارغ؟ حسنًا، تعيد `matchAll()` نوعًا خاصًا من الكائنات يسمى `Iterator`، والذي لا تتعامل معه وحدة تحكم freeCodeCamp.

إذا ألقينا نظرة في وحدة تحكم المتصفح، فإن `Iterator` يحتوي على طريقة `next()`، يمكننا استدعاؤها للحصول على القيمة التالية:

```js
// RegExpStringIterator {  }
//  <prototype>: RegExp String Iterator {
//    next: ƒ next(),
//    Symbol(Symbol.toStringTag): "RegExp String Iterator"
//    <prototype>: Object { ... }
//  }
```

لنستدعي `matched.next()`، ونسجل النتيجة:

```js
// {
//   "done": false,
//   "value": [
//     0: "freecodecamp"
//     groups: undefined
//     index: 0
//     input: "freecodecamp is the best we love freecodecamp"
//   ]
// }
```

ها هي مصفوفة المطابقة! تعيد `next()` كائنًا بقيمتين: `done` التي تكون `false` عندما تتوفر عناصر أخرى في المكرر، و`value` وهي القيمة التي تكررناها للتو. لذا، إذا استدعيناها مرة أخرى:

```js
const regex = /freecodecamp/g;
const str = "freecodecamp is the best we love freecodecamp";
const matched = str.matchAll(regex);
const replaced = str.replaceAll(regex, "freeCodeCamp");
console.log(matched);
console.log(replaced);
console.log(matched.next());
console.log(matched.next());
```

```js
// {
//   "done": false,
//   "value": Array [ "freecodecamp"]
// }
//
// {
//   "done": false,
//   "value": Array [ "freecodecamp"]
// }
```

لماذا تقول `done` أنها لا تزال `false`؟ يجب أن يكون هناك تطابقان فقط في المصفوفة، أليس كذلك؟ دعنا نستدعيها للمرة الثالثة ونرى ماذا نحصل:

```js
// {
//   "done": false,
//   "value": Array [ "freecodecamp"]
// }
//
// {
//   "done": false,
//   "value": Array [ "freecodecamp"]
// }
//
// {
//   "done": true,
//   "value": undefined
// }
```

أخيرًا `done` أصبحت `true`، لكن لماذا القيمة `undefined`؟ حسنًا، كما يتضح، المكرر `matchAll()` كسول. لا يجد كل التطابقات دفعة واحدة. بل يجد تطابقًا فقط عندما تطلب ذلك باستدعاء `next()`.

طالما وجد تطابقًا، فهو ليس `done`. بمجرد أن يفشل في العثور على تطابق ويعيد `undefined`، يصبح `done`. قد يبدو هذا غير مريح، لكنه مفيد جدًا عندما يكون التعبير النمطي مكلفًا حسابيًا.

إذا كان المثال الخاص بك أقل تكلفة، مثل مثالنا، يمكنك تخطي هذه الميزة واستخراج كل التطابقات دفعة واحدة بتحويله إلى مصفوفة. يتم ذلك باستدعاء `Array.from()` وتمرير المكرر كمعلمة.

دعنا نحدث شفرتنا لاستخدام ذلك: سننظف استدعاءات `replaceAll` لأننا نعلم أنها تعمل:

```js
const regex = /freecodecamp/g;
const str = "freecodecamp is the best we love freecodecamp";
const matched = str.matchAll(regex);
console.log(Array.from(matched));
```

وأخيرًا نحصل على مصفوفة التطابقات:

```js
// [
//   'freecodecamp',
//   index: 0,
//   input: 'freecodecamp is the best we love freecodecamp',
//   groups: undefined
// ]
//
// [
//   'freecodecamp',
//   index: 33,
//   input: 'freecodecamp is the best we love freecodecamp',
//   groups: undefined
// ]
```

يمكن لهذه الطرق القوية مساعدتك في التلاعب بالبيانات واستخراجها من السلاسل النصية دون التضحية بالأداء أو سهولة القراءة.
