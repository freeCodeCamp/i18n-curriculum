# --description--

Ein weiterer wichtiger Teil eines Fehlers ist der Stacktrace. Dieser hilft Ihnen zu verstehen, wann der Fehler in Ihrer Anwendung geworfen wurde und welche Funktionen aufgerufen wurden, die zum Fehler geführt haben. Zum Beispiel, wenn Sie folgenden Code haben:

```javascript
const a = 5;
const b = 10;

function add() {
  return c;
}

function print() {
  add();
}

print();
```

Unsere Funktion `print()` sollte `add()` aufrufen, welche eine Variable namens `c` zurückgibt, die derzeit nicht deklariert wurde. Der entsprechende Fehler sieht folgendermaßen aus:

<img src="https://cdn.statically.io/gh/TheOdinProject/curriculum/284f0cdc998be7e4751e29e8458323ad5d320303/foundations/javascript_basics/understanding_errors/imgs/01.png" width="100%" alt="Ein Fehler mit Stacktrace in der Entwicklerkonsole">

Der Stacktrace sagt uns:

1. `c is not defined` im Gültigkeitsbereich von `add()`, die in Zeile 5 deklariert wurde.
1. `add()` wurde von `print()` aufgerufen, die in Zeile 9 deklariert wurde.
1. `print()` selbst wurde in Zeile 12 aufgerufen.

So ermöglicht Ihnen der Stacktrace, die Entwicklung eines Fehlers bis zu seiner Ursache zurückzuverfolgen, die hier die Deklaration von `add()` ist.

## --text--

Wie hilft der Stacktrace beim Debuggen des Fehlers im bereitgestellten JavaScript-Code?

## --answers--  

Der Stacktrace zeigt, dass die Funktion `add()` korrekt den Wert von `c` zurückgibt.

---

Der Stacktrace zeigt an, welche Zeile im Code bearbeitet werden muss, um einen Syntaxfehler zu korrigieren.

---

Der Stacktrace liefert einen detaillierten Pfad der Funktionsaufrufe, die zum Fehler führen, und hilft dabei zu erkennen, wo `c` falsch referenziert wird.

---

Der Stacktrace zeigt nur an, dass `print()` aufgerufen wurde, gibt aber keine Details zum Fehler in `add()`.
