# --description--

CSP（內容安全政策）和 Permissions-Policy 是兩個強大的 HTTP 安全標頭，透過限制可載入的資源和頁面可存取的瀏覽器功能，幫助保護網頁應用程式。

內容安全政策主要聚焦於資源載入。它就像你前端腳本、樣式和圖片的安全過濾器。

惡意攻擊者經常試圖將有害腳本注入易受攻擊的網站。強大的 CSP 透過建立嚴格的可信來源白名單，阻止這些跨站腳本攻擊（XSS）。

當伺服器傳送 CSP 標頭時，瀏覽器會讀取特定的指令。例如，`script-src` 指令告訴瀏覽器哪些網域可以安全地在頁面上執行 JavaScript。

如果駭客試圖從未經批准的外部伺服器注入惡意腳本，瀏覽器會偵測到政策違規，立即丟棄該腳本並記錄安全錯誤。

CSP 也可以限制圖片來源、表單目的地和樣式表來源。

Permissions-Policy 聚焦於瀏覽器功能和裝置硬體。它取代了舊有的 Feature-Policy 標頭。

這個安全機制允許網站擁有者明確啟用或停用特定瀏覽器功能，適用於自己的頁面和任何嵌入的 iframe，並防止第三方程式碼濫用使用者隱私。

使用 Permissions-Policy 標頭，你可以控制對攝影機、麥克風和地理位置追蹤等硬體的存取權限。

例如，如果你的網站不需要知道使用者位置，你可以設定政策完全封鎖地理位置 API。即使攻擊者入侵你頁面上的腳本，瀏覽器也會拒絕啟動追蹤功能。

Permissions-Policy 的實作使用清晰且結構化的標頭格式，命名功能並定義其允許的來源。

若要嚴格限制硬體功能只限於你自己的網域，使用括號包住的 `self` 關鍵字，例如 `camera=(self)`。若想在整個頁面上對所有使用者停用某功能，則使用空列表，格式為 `geolocation=()`。

結合這兩個標頭能為網頁應用程式提供堅實防護。CSP 鎖定資料傳輸並防止程式碼注入，Permissions-Policy 則保護實體裝置並確保隱私合規。

現代網頁開發者會同時使用這兩種工具，打造高度防護的瀏覽環境。
