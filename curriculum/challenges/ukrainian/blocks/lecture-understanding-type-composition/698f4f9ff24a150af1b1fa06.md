# --description--

У попередніх уроках ви навчились працювати з примітивними типами, а також з об’єднаними типами та інтерфейсами.

Але в TypeScript є деякі спеціальні типи, про які варто знати. Почнемо з розгляду типу `any`.

Тип `any` використовується для представлення будь-якого типу значення.

```ts
let randomValue: any;

randomValue = 42;           

randomValue = "Hello";     
 
randomValue = true;        

randomValue = { name: "Alice" }; 
```

З цим типом потрібно бути обережним, бо його легко можна надмірно використовувати або застосовувати неправильно. Не варто типізувати все типом `any` лише для того, щоб заглушити повідомлення про помилки TypeScript. Це суперечить самій ідеї використання TypeScript і додавання типобезпеки.

Безпечнішою альтернативою типу `any` буде тип `unknown`. `unknown` схожий на `any`, але з типом `unknown` потрібно виконувати перевірку типу змінної перед її використанням.

Ось приклад:

```ts
function doubleValue(value: unknown) {
  if (typeof value === "number") {
    console.log(value * 2);
  } else if (typeof value === "string") {
    console.log(value + value);
  }
}

doubleValue(10);    
doubleValue("Hi "); 
doubleValue(true);   
```

У цьому прикладі перевірки типів гарантують, що операції виконуються лише над значеннями правильного типу. Значення, які не відповідають очікуваним типам, ігноруються.

Наступний тип, який ми розглянемо, — це тип `never`. Цей тип представляє те, що ніколи не станеться.

У більшості випадків ви не будете писати анотації типів із `never`. Натомість ви найчастіше побачите тип `never` у повідомленнях про помилки, як-от це:

```ts
function processValue(value: string | number) {
  if (typeof value === "string") {
    console.log("String value:", value.toUpperCase());
  } else if (typeof value === "number") {
    console.log("Number value:", value * 2);
  } else {
    console.log(value); 
  }
}
```

У умові `else` у наведеному вище прикладі `value` має тип `never`. Це тому, що гілка `else` тепер недосяжна, і залишився тип відсутній. Якщо ви спробуєте викликати функцію зі значенням, відмінним від `string` або `number`, з’явиться помилка такого вигляду:

```ts
function processValue(value: string | number) {
  if (typeof value === "string") {
    console.log("String value:", value.toUpperCase());
  } else if (typeof value === "number") {
    console.log("Number value:", value * 2);
  } else {
    console.log(value); 
  }
}

// Argument of type 'boolean' is not assignable to parameter of type 'string | number'.
processValue(true); 
```

Останній тип, який ми розглянемо, — це тип `void`. Коли у вас є функція, яка не повертає значення, це буде тип `void`.

Ось приклад:

```ts
type Status = "loading" | "success" | "error";

type Handler = {
  status: Status;
  onChange: (newStatus: Status) => void; 
};
```

У цьому прикладі властивість `onChange` для типу `Handler` нічого не повертає, тому використовується `void`.
