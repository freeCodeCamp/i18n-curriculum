## 접근성 소개

- **웹 콘텐츠 접근성 지침**: 웹 콘텐츠 접근성 지침(WCAG)은 장애가 있는 사람들이 웹 콘텐츠에 더 쉽게 접근할 수 있도록 만든 지침 모음입니다. WCAG의 네 가지 원칙은 인지 가능(Perceivable), 조작 가능(Operable), 이해 가능(Understandable), 견고함(Robust)을 뜻하는 POUR입니다.

## 접근성을 위한 보조 기술

- **화면 읽기 프로그램**: 컴퓨터 화면의 내용을 소리 내어 읽어 주는 소프트웨어입니다. 시각 장애인이나 시각이 약한 사람들이 웹에 접근할 때 사용합니다.
- **큰 글자 또는 점자 키보드**: 시각 장애인이 웹에 접근할 때 사용하는 장치입니다.
- **화면 확대기**: 컴퓨터 화면의 내용을 확대해 주는 소프트웨어입니다. 시력이 약한 사람들이 웹에 접근할 때 사용합니다.
- **대체 포인팅 장치**: 이동성 장애인이 웹에 접근할 때 사용하는 장치입니다. 조이스틱, 트랙볼, 터치패드 등이 포함됩니다.
- **음성 인식**: 이동성 장애인이 음성으로 컴퓨터를 제어할 수 있게 해 주는 기술입니다.

## 접근성 감사 도구

- **일반적인 접근성 도구**: Google Lighthouse, Wave, IBM Equal Accessibility Checker, axe DevTools는 웹사이트의 접근성을 감사하는 데 자주 사용되는 도구입니다.

## 접근성 모범 사례

- **적절한 제목 수준 구조**: 콘텐츠에 논리적인 구조를 만들기 위해 적절한 제목 수준을 사용하세요. 이는 보조 기술을 사용하는 사람들이 웹사이트의 내용을 이해하는 데 도움이 됩니다.
- **표와 접근성**: 표를 사용할 때는 `th` 요소로 헤더 셀을 정의하고 `td` 요소로 데이터 셀을 정의해야 합니다. 이는 보조 기술 사용자가 표의 구조를 이해하는 데 도움이 됩니다. `caption` 요소를 사용해 표의 캡션(또는 제목)을 작성할 수 있으며, 특히 보조 기술 사용자가 표의 목적과 내용을 빠르게 이해할 수 있습니다. `caption` 요소는 `table` 요소의 여는 태그 바로 다음에 배치해야 합니다. 이렇게 하면 화면 읽기 프로그램과 다른 보조 기술이 내용을 읽기 전에 캡션을 먼저 안내해 더 많은 맥락을 제공할 수 있습니다.
- **입력과 연관된 레이블의 중요성**: `label` 요소를 사용해 레이블을 폼 입력과 연결하세요. 이는 보조 기술 사용자가 입력의 목적을 이해하는 데 도움이 됩니다.
- **좋은 `alt` 텍스트의 중요성**: `alt` 속성을 사용해 이미지에 대한 텍스트 대체물을 제공하세요. 이는 보조 기술 사용자가 이미지 내용을 이해하는 데 도움이 됩니다.
- **좋은 링크 텍스트의 중요성**: 링크의 목적을 이해할 수 있도록 설명적인 링크 텍스트를 사용하세요. 이는 보조 기술 사용자가 링크의 목적을 이해하는 데 도움이 됩니다.
- **오디오 및 비디오 콘텐츠 접근성 모범 사례**: 청각 장애인을 위해 오디오 및 비디오 콘텐츠에 캡션과 대본을 제공하세요. 시각 장애인을 위해 비디오 콘텐츠에 오디오 설명도 제공해야 합니다.
- **`tabindex` 속성**: 요소를 포커스 가능하게 만들고 키보드로 탐색할 때의 상대적 순서를 정의하는 데 사용됩니다. `tabindex` 속성에 0보다 큰 값을 절대 사용하지 말아야 하며, 대신 0 또는 -1 값을 사용해야 합니다.

:::interactive_editor

```html
<p tabindex="-1">Sorry, there was an error with your submission.</p>
```

:::

- **`accesskey` 속성**: 요소에 키보드 단축키를 정의하는 데 사용됩니다. 이는 이동성 장애인이 웹사이트를 더 쉽게 탐색하는 데 도움이 됩니다.

:::interactive_editor

```html
<button accesskey="s">Save</button>
<button accesskey="c">Cancel</button>
<a href="index.html" accesskey="h">Home</a>
```

:::

## WAI-ARIA, 역할, 속성

- **WAI-ARIA**: 웹 접근성 이니셔티브 - 접근 가능한 리치 인터넷 애플리케이션(Web Accessibility Initiative - Accessible Rich Internet Applications)의 약자입니다. HTML 요소에 추가할 수 있는 속성 모음으로, 보조 기술에 콘텐츠의 목적과 구조에 대한 추가 정보를 제공합니다.
- **ARIA 역할**: HTML 요소에 추가해 그 목적을 정의하는 미리 정의된 역할 모음입니다. 이는 보조 기술 사용자가 웹사이트 내용을 이해하는 데 도움이 됩니다. 예로 `role="tab"`, `role="menu"`, `role="alert"`이 있습니다.

ARIA 역할은 여섯 가지 주요 범주로 나뉩니다:

- **문서 구조 역할**: 웹 페이지의 전체 구조를 정의하는 역할입니다. 이 역할을 통해 보조 기술은 서로 다른 섹션 간 관계를 이해하고 사용자가 콘텐츠를 탐색하도록 돕습니다.
- **위젯 역할**: 스크롤바 같은 인터랙티브 요소의 목적과 기능을 정의하는 역할입니다.
- **랜드마크 역할**: 웹 페이지 주요 섹션을 분류하고 레이블을 붙이는 역할입니다. 화면 읽기 프로그램은 이를 사용해 중요한 섹션으로 편리하게 이동할 수 있게 합니다.
- **라이브 영역 역할**: 동적으로 내용이 변경되는 요소를 정의하는 역할입니다. 화면 읽기 프로그램과 다른 보조 기술이 시각 장애 사용자에게 변경 사항을 알릴 수 있습니다.
- **윈도우 역할**: 팝업 모달 대화상자 같은 하위 창을 정의하는 역할입니다. `alertdialog`와 `dialog`가 포함됩니다.
- **추상 역할**: 문서 조직을 돕는 역할로, 브라우저 내부에서만 사용되며 개발자가 웹사이트나 애플리케이션에 직접 사용하지 않아야 합니다.

- **`aria-label`와 `aria-labelledby` 속성**: 요소에 프로그래밍 가능한(또는 접근 가능한) 이름을 부여하는 데 사용됩니다. 이는 화면 읽기 프로그램 같은 보조 기술 사용자가 요소의 목적을 이해하는 데 도움이 됩니다. 시각적 레이블이 텍스트가 아닌 이미지나 기호일 때 자주 사용됩니다. `aria-label`은 이름을 속성에 직접 정의할 수 있게 하고, `aria-labelledby`는 페이지 내 기존 텍스트를 참조할 수 있게 합니다.

```html
<button aria-label="Search">
    <i class="fas fa-search"></i>
</button>
```

:::interactive_editor

```html
<input type="text" aria-labelledby="search-btn">
<button type="button" id="search-btn">Search</button>
```

:::

- **`aria-hidden` 속성**: 화면 읽기 프로그램 같은 보조 기술에서 요소를 숨기는 데 사용됩니다. 예를 들어, 의미 있는 내용이 없는 장식용 이미지를 숨길 때 사용합니다.

:::interactive_editor

```html
<button>
    <i class="fa-solid fa-gear" aria-hidden="true"></i>
    <span class="label">Settings</span>
</button>
```

:::

- **`aria-describedby` 속성**: 요소에 대한 추가 정보를 다른 요소와 연결해 제공하는 데 사용됩니다. 화면 읽기 프로그램 사용자가 해당 요소로 이동할 때 추가 정보를 즉시 확인할 수 있습니다. 일반적으로 서식 지침을 텍스트 입력과 연결하거나, 잘못된 폼 제출 후 입력에 오류 메시지를 연결할 때 사용합니다.

:::interactive_editor

```html
<form>
    <label for="password">Password:</label>
    <input type="password" id="password" aria-describedby="password-help" />
    <p id="password-help">Your password must be at least 8 characters long.</p>
</form>
```

:::
