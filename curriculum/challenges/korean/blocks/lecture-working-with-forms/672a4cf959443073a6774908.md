# --interactive--

HTML에서 입력과 같은 폼 컨트롤은 `focused` 상태, `readonly` 상태, 또는 `disabled` 상태와 같은 여러 단계나 조건에 있을 수 있습니다.

첫 번째 상태는 `default` 상태로 간주됩니다. 이메일 주소 입력의 기본 상태는 빈 입력입니다. 이것이 페이지에 처음 렌더링될 때 이메일 입력이 보이는 모습입니다.

:::interactive_editor

```html
<input type="email" name="email" id="email" />
```

:::

사용자가 폼 컨트롤을 클릭하거나 키보드의 탭 키로 선택하면 `focused` 상태가 됩니다. 입력이 `focused` 상태일 때 대부분의 브라우저는 입력 주위에 파란색 하이라이트 테두리를 표시합니다. 하지만 CSS에서 추가 스타일을 지정할 수도 있습니다.

미리보기 창의 공백 부분 아무 곳이나 클릭한 다음 `tab` 키를 눌러 포커스 상태를 확인하세요. 미리보기를 보려면 인터랙티브 편집기를 활성화해야 합니다.

:::interactive_editor

```html
<input type="email" name="email" id="email" />
```

:::

또 다른 폼 상태는 `disabled` 상태입니다. 이 상태는 사용자가 입력에 포커스하거나 활성화할 수 없음을 보여줍니다.

인터랙티브 편집기를 활성화하고 이메일 입력을 클릭해 보세요. 더 이상 포커스가 가지 않는 것을 확인할 수 있습니다.

:::interactive_editor

```html
<input disabled type="email" name="email" id="email" />
```

:::

`focused` 상태와 비슷하게, CSS를 사용해 `disabled` 상태에 대한 추가 스타일을 지정할 수 있습니다.

또 다른 폼 상태 유형은 `readonly` 상태입니다. 이것은 입력과 같은 폼 컨트롤이 사용자가 편집할 수 없을 때입니다. 다음은 이메일 입력을 읽기 전용으로 설정하는 예입니다. 입력 필드 안에 표시되는 값을 설정하는 데 `value` 속성이 사용됩니다.

인터랙티브 편집기를 활성화하고 미리보기 창에서 `example@email.com`의 현재 값을 편집해 보세요. 편집이 불가능한 것을 알 수 있습니다.

:::interactive_editor

```html
<input
  readonly
  type="email"
  name="email"
  id="email"
  value="example@email.com"
/>
```

:::

`disabled` 상태와 `readonly` 상태의 주요 차이점은 `readonly`는 포커스가 가능하지만 `disabled` 상태는 그렇지 않다는 점입니다.

다양한 폼 상태를 이해하는 것은 명확한 피드백과 안내를 제공하며 오류를 처리하는 동안 원활한 사용자 경험을 보장하기 때문에 중요합니다.
