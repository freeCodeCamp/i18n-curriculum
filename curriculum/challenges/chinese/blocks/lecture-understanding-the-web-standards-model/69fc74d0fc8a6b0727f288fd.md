# --description--

开放的网络不会自己运行。每当浏览器渲染页面、播放视频或绘制 3D 场景时，它都是在遵循多年来由工程师和公司代表在工作组中制定的指令。

这些指令称为网络标准。负责制定这些标准的组织称为标准机构。你可以把它们看作互联网的立法者。

目前在网络标准工作中承担主要任务的组织包括 **W3C**、**WHATWG**、**ECMA（TC39）** 和 **Khronos Group**。它们各自管理栈的不同层，并且运作方式略有不同。

**Web 超文本应用技术工作组**（WHATWG）维护 HTML 和 DOM，作为一个持续更新的“活标准”。它没有版本发布，规范实时演进，由一个拥有强大编辑权的小组编辑。决策权掌握在 Chromium、Gecko 和 WebKit 等主要浏览器引擎团队手中。

**万维网联盟**（W3C）是网络的主要标准组织。它拥有广泛的成员，包括浏览器厂商、政府和研究机构。

W3C 负责监督 CSS、SVG、无障碍指南以及众多网络 API。它通过一个缓慢且基于共识的流程工作，提案需经过多个审查阶段才能成为正式标准。

**Ecma 国际**（前身为欧洲计算机制造商协会，简称 ECMA）是一个通用标准组织，最初专注于计算和通信技术。如今，它正式发布 ECMAScript 标准，JavaScript 的实现基于此。语言标准的技术开发由 Ecma 的技术委员会 39（**TC39**）负责，而 Ecma 国际提供组织框架、发布流程和标准治理。

该委员会大约每两个月召开一次会议，成员包括来自 Google、Apple、Mozilla、Microsoft 等公司的代表。新语言特性通过五个阶段的提案流程，从第 0 阶段的草图到第 4 阶段的完整规范和实现。像 async/await、可选链和 ES 模块等特性都经过了这一流程，最终进入 JavaScript 语言。

**Khronos Group** 是一个由成员资助的联盟，专注于图形、计算和媒体标准。成员包括来自 NVIDIA 和 AMD 等 GPU 制造商的代表，以及浏览器厂商。对于网络来说，它最重要的贡献是 WebGL 和 WebGL 2，它们为浏览器带来了 GPU 加速的 3D 渲染。
