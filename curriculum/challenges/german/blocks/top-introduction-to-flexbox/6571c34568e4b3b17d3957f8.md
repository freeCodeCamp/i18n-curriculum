# --description--

`flex-shrink` ist ähnlich wie `flex-grow`, legt jedoch den „Schrumpffaktor“ eines Flex-Elements fest. `flex-shrink` wird nur angewendet, wenn die Größe aller Flex-Elemente größer als ihr übergeordneter Container ist. Zum Beispiel, wenn unsere 3 divs von oben eine Breiten-Deklaration wie `width: 100px` hätten und `.flex-container` kleiner als `300px` wäre, müssten unsere divs schrumpfen, um zu passen.

Der Standard-Schrumpffaktor ist `flex-shrink: 1`, was bedeutet, dass alle Elemente gleichmäßig schrumpfen. Wenn Sie nicht möchten, dass ein Element schrumpft, können Sie `flex-shrink: 0;` angeben. Sie können auch höhere Zahlen angeben, damit bestimmte Elemente schneller schrumpfen als normal.

Hier ein Beispiel: Wenn Sie Ihr Browserfenster verkleinern, werden Sie feststellen, dass `.two` niemals kleiner wird als die angegebene Breite von `250px`, obwohl die `flex-grow`-Regel sonst vorschreiben würde, dass jedes Element gleich groß sein sollte.

Eine wichtige Erkenntnis hierbei ist, dass wenn Sie `flex-grow` oder `flex-shrink` angeben, Flex-Elemente nicht unbedingt Ihre vorgegebenen Werte für `width` respektieren. Im obigen Beispiel erhalten alle 3 divs eine `width` von `250px`, aber wenn ihr übergeordneter Container groß genug ist, wachsen sie, um ihn auszufüllen. Ebenso schrumpfen sie bei zu kleinem Container standardmäßig, um zu passen. Dies ist kein Bug, kann aber verwirrend sein, wenn Sie es nicht erwarten.
