# --description--

No Express, parâmetros de rota são valores dinâmicos incorporados no caminho da URL. Eles permitem que sua aplicação capture partes da URL que podem mudar a cada requisição. Isso é útil para construir páginas dinâmicas, como exibir um usuário específico, uma postagem de blog ou um produto com base na URL.

Aqui está um exemplo:

```javascript
// Route with a parameter in the path
app.get("/post/:postId", (req, res) => {
  const postId = req.params.postId;
  res.send(`Viewing post with ID: ${postId}`);
});
```

Aqui, a rota escuta requisições GET para `/post/:postId`.

* `:postId` é um parâmetro de rota, um marcador de posição na URL que será substituído por um valor real. Por exemplo, uma requisição para `/post/42` corresponde a essa rota, e `42` se torna o valor de `postId`.
    
* Você pode acessar os parâmetros de rota usando `req.params`. O objeto `req.params` contém todos os parâmetros de rota, e neste caso, `req.params.postId` contém o valor da URL, `42`.
    

Por fim, `res.send()` envia uma resposta de volta para o cliente. Então, se a URL for `/post/42`, o cliente verá `Viewing post with ID: 42`.

Como você pode ver, parâmetros de rota tornam suas rotas flexíveis e dinâmicas sem precisar de uma rota separada para cada valor possível.

Agora que você entende como os parâmetros de rota funcionam, vamos voltar a um exemplo da lição anterior e ver como você pode usar rotas dinâmicas para diferentes tipos de menus:

```javascript
app.get("/menu/:category", (req, res) => {
  const category = req.params.category;
  res.send(`You are viewing the ${category} menu!`);
});
```

E aqui está uma explicação de como isso funciona:

1. Um usuário visita [`http://localhost:3000/menu/drinks`](http://localhost:3000/menu/drinks).
    
2. O servidor corresponde essa requisição à rota `/menu/:category` e captura o parâmetro `category` como `drinks`.
    
3. O servidor envia essa resposta de volta ao navegador: `You are viewing the drinks menu!`
    

Da mesma forma, se o usuário visitar [`http://localhost:3000/menu/food`](http://localhost:3000/menu/food), o servidor responde com o seguinte: `You are viewing the food menu!`

Parâmetros de rota permitem que você exiba conteúdo dinâmico com base na URL. Em vez de criar uma rota separada para cada categoria, você pode usar uma única rota para lidar com múltiplas categorias como bebidas, comidas, sobremesas e mais, simplesmente mudando a URL.

## --text--

Como você pode usar parâmetros de rota na sua aplicação?

## --answers--

Para armazenar valores estáticos na URL.

### --feedback--

Valores estáticos não se beneficiam de marcadores de posição para parâmetros.

---

Para capturar dinamicamente partes da URL para processamento.

---

Para validar a entrada do usuário.

### --feedback--

A validação pode ocorrer após capturar o valor, mas o parâmetro em si captura os dados.

---

Para definir cabeçalhos para requisições.

### --feedback--

Parâmetros de rota pertencem ao caminho e não definem cabeçalhos.
