# --description--

栈和队列是计算机科学中常用的数据结构。

它们是线性数据结构，遵循特定规则来添加和移除元素。

## 栈

让我们从 **栈** 开始。

**栈** 是一种后进先出（LIFO）的数据结构。

这意味着最后添加到栈的元素是第一个被移除的。

栈有两个端点，我们称之为顶部和底部。

元素从栈顶添加和移除。

你可以将 `stack` 想象成一堆盘子，你只能将盘子放在堆的顶部，也只能从堆的顶部取盘子。

<img src="https://cdn.freecodecamp.org/curriculum/lecture-transcripts/how-do-stacks-and-queues-work-1.png" alt="stack data structure visualization">

在此上下文中，添加和移除元素的这些操作有特殊名称。

将元素添加到栈中称为“push”操作。当我们将元素添加到栈顶时，我们说将元素“push”到栈上。

<img src="https://cdn.freecodecamp.org/curriculum/lecture-transcripts/how-do-stacks-and-queues-work-2.png" alt="stack push operation visualization">

从栈中移除一个元素称为“pop”操作。当我们从栈顶移除元素时，我们说我们从栈中“pop”了一个元素。

<img src="https://cdn.freecodecamp.org/curriculum/lecture-transcripts/how-do-stacks-and-queues-work-3.png" alt="stack pop operation visualization">

你可以看到我们实际上并没有在栈底执行任何操作，但我们将其保留在那里作为引用。

push 和 pop 操作的时间复杂度通常是 `O(1)`，一个常量时间复杂度。

当你将一个元素推入栈时，该元素会被简单地添加到顶部。

当你从 `stack` 弹出一个 `Element` 时，顶部的 `Element` 会被移除。

因此，执行这些操作所需的时间无论 `stack` 的大小如何都保持常量。

`push` 和 `pop` 操作的空间复杂度通常是常量 `O(1)`。这意味着执行这些操作所需的内存量无论栈的大小如何都保持不变。

## 队列

既然你已经了解了更多关于栈的内容，现在让我们学习 **队列**。

队列是一种先进先出（FIFO）的线性数据结构。这意味着第一个添加到队列的元素是第一个被移除的。

队列有两个端点：前端和后端。

<img src="https://cdn.freecodecamp.org/curriculum/lecture-transcripts/how-do-stacks-and-queues-work-4.webp" alt="queue data structure visualization">

元素被添加到队列的末尾，并且从队列的前端移除。

你可以将队列想象成在超市排队结账的人群。排在最前面的人是第一个去收银台的人，而新的人则加入队列的末尾。

在 `queue` 的上下文中，添加和移除元素的操作有特殊名称。

将元素添加到队列尾部称为“enqueue”操作。

在入队操作中，新的元素被添加到队列的末尾，成为队列的最后一个。

<img src="https://cdn.freecodecamp.org/curriculum/lecture-transcripts/how-do-stacks-and-queues-work-5.png" alt="queue enqueue operation visualization">

从队列前端移除一个元素称为 `"dequeue"` 操作。  
  
在 `dequeue` 操作中，队列前端的元素被移除，队列中的下一个元素成为新的前端。

<img src="https://cdn.freecodecamp.org/curriculum/lecture-transcripts/how-do-stacks-and-queues-work-6.png" alt="queue dequeue operation visualization">

入队和出队操作的时间复杂度是 `O(1)`，常量时间。执行这些操作所需的时间保持不变，无论队列的大小如何。

入队和出队操作的空间复杂度通常是常量 `O(1)`。这意味着执行这些操作所需的内存量无论队列的大小如何都保持不变。

栈和队列是计算机科学中用于组织和管理元素的数据结构。理解它们对于在各种编程应用中创建高效算法至关重要。
