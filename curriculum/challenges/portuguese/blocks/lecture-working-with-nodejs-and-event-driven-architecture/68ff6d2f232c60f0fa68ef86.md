# --description--

Até agora, você tem escrito código do lado do cliente com JavaScript, código que é executado no lado do cliente de uma aplicação web, geralmente um navegador web.

JavaScript do lado do cliente pode ser muito poderoso, mas tem certas limitações para tarefas mais complexas.

Por exemplo:

- Ele tem acesso muito restrito a arquivos locais.
- Não é realmente projetado para lidar com lógica complexa de aplicação.
- E isso tem potenciais preocupações de segurança. Por exemplo, você não gostaria de expor informações sensíveis, como credenciais de banco de dados, para código do lado do cliente.
    
O JavaScript do lado do cliente tem essas limitações porque foi inicialmente projetado para rodar exclusivamente em navegadores web.

Os navegadores web fornecem o ambiente necessário para executar código JavaScript, incluindo o JavaScript Engine. Eles também fornecem acesso ao Document Object Model (DOM), para que você possa acessar elementos HTML no seu código.

Anteriormente, não podíamos executar JavaScript fora de um navegador, mas agora podemos com Node.js.

Node.js é um ambiente de execução JavaScript. É oficialmente definido como “um ambiente de execução JavaScript de código aberto e multiplataforma.”

Vamos ver o que isso significa:

- Node.js é open source porque seu código está publicamente disponível e é mantido por uma grande comunidade de desenvolvedores.
- É multiplataforma porque funciona em qualquer sistema operacional, incluindo Windows, macOS e Linux.
- E é um ambiente de execução JavaScript porque permite que você execute código JavaScript fora de um navegador.
    
Node.js é usado em uma ampla variedade de indústrias. Desenvolvedores o utilizam para construir servidores web e APIs que lidam com requisições HTTP, assim como aplicações web e móveis.

Se você está usando ou desenvolvendo um site ou aplicação web que interage com um banco de dados ou lida com dados complexos, há uma grande chance de que você use Node.js nos bastidores em algum momento.

É eficiente, escalável e possui uma grande comunidade de desenvolvedores e mantenedores ao redor do mundo.

Agora que você sabe o que é Node.js, vamos ver algumas das diferenças entre o navegador e o ambiente de execução Node.

O ambiente do navegador é projetado principalmente para desenvolvimento web front-end, então ele executa JavaScript do lado do cliente. Em contraste, o ambiente de execução Node é projetado principalmente para desenvolvimento web back-end, então ele executa JavaScript do lado do servidor. Isso também determina as APIs que eles oferecem.

Por exemplo, do navegador, você pode acessar a API DOM, mas há restrições para acessar o sistema de arquivos local. Em contraste, do ambiente de execução Node, você pode acessar quase todos os recursos do sistema, incluindo o sistema de arquivos, mas não o DOM.

Ambos os ambientes JavaScript fornecem um objeto global. No runtime JavaScript do navegador, isso é chamado de `window`. Esse objeto `window` fornece acesso a funcionalidades relacionadas ao navegador, como acesso a métodos para manipular o DOM, gerenciar cookies e lidar com eventos do navegador. No ambiente Node.js, esse objeto global é chamado de `global`. O objeto `global` fornece acesso a funcionalidades específicas do Node.js, como módulos integrados para trabalhar com arquivos locais, redes e interagir com o sistema operacional.

Finalmente, outra diferença importante é que você pode escolher a versão do Node.js que deseja usar no seu servidor. No entanto, você não tem controle sobre a versão do ambiente do navegador que seus usuários usarão para visitar seu site.

Node.js se tornou muito importante para desenvolvedores em todo o mundo. É uma ferramenta poderosa, escalável e versátil que transformou o desenvolvimento web ao permitir que desenvolvedores usem JavaScript para construir tanto o frontend e o backend de aplicações full-stack. Isso tornou o processo de desenvolvimento mais eficiente porque os desenvolvedores não precisam aprender uma nova linguagem de programação apenas para desenvolver o backend.
