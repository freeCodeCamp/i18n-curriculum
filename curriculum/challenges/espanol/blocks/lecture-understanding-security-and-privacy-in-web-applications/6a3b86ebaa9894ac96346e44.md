# --description--

CSP (Content Security Policy) y Permissions-Policy son dos potentes encabezados de seguridad HTTP que ayudan a proteger las aplicaciones web al restringir qué recursos pueden cargarse y a qué funcionalidades del navegador puede acceder una página.

Content Security Policy se centra principalmente en la carga de recursos. Actúa como un filtro de seguridad para tus scripts, estilos e imágenes del frontend.

Los actores maliciosos a menudo intentan inyectar scripts dañinos en sitios web vulnerables. Un CSP fuerte detiene estos ataques de Cross-Site Scripting (XSS) estableciendo una lista blanca estricta de dominios de origen confiables.

Cuando un servidor envía el encabezado CSP, el navegador lee las directivas específicas. Por ejemplo, la directiva `script-src` indica al navegador qué dominios pueden ejecutar JavaScript de forma segura en la página.

Si un hacker intenta inyectar un script malicioso desde un servidor externo no aprobado, el navegador detecta la violación de la política, descarta el script al instante y registra un error de seguridad.

CSP también puede restringir las fuentes de imágenes, destinos de formularios y orígenes de hojas de estilo.

Permissions-Policy se enfoca en las funcionalidades del navegador y el hardware del dispositivo. Reemplazó al antiguo encabezado Feature-Policy.

Este mecanismo de seguridad permite a los propietarios del sitio habilitar o deshabilitar explícitamente funcionalidades específicas del navegador para su propia página y cualquier iframe incrustado, y evita que fragmentos de terceros abusen de la privacidad del usuario.

Usando un encabezado Permissions-Policy, puedes controlar el acceso a hardware como la cámara web, el micrófono y el seguimiento de geolocalización.

Por ejemplo, si tu sitio web no necesita conocer la ubicación del usuario, puedes configurar la política para bloquear completamente la API de geolocalización. Incluso si un atacante compromete un script en tu página, el navegador se negará a activar la función de seguimiento.

La implementación de Permissions-Policy usa un formato claro y estructurado en el encabezado que nombra la funcionalidad y define sus orígenes permitidos.

Para restringir una funcionalidad de hardware estrictamente a tu propio dominio, usas la palabra clave `self` entre paréntesis, como `camera=(self)`. Y si quieres deshabilitar una funcionalidad en toda la página para todos los usuarios, usas una lista vacía, que se ve así: `geolocation=()`.

Combinar ambos encabezados proporciona una defensa sólida para las aplicaciones web. CSP bloquea la entrega de datos y previene la inyección de código, y Permissions-Policy protege el dispositivo físico y asegura el cumplimiento de la privacidad.

Los desarrolladores web modernos usan ambas herramientas para crear un entorno de navegación fuertemente protegido.
