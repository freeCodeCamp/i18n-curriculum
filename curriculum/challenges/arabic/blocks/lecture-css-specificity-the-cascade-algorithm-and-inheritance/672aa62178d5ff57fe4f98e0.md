# --interactive--

خصوصية CSS مفهوم أساسي يحدد أي الأنماط تُطبَّق على عنصر عندما يمكن تطبيق قواعد متعددة.

فهم الخصوصية يساعد المطوِّرين على حل النزاعات بين قواعد CSS المختلفة ويضمن تطبيق الأنماط المطلوبة باستمرار.

تُحسب خصوصية CSS بناءً على نوع المحددات المستخدمة.

أعلى خصوصية تُنسب إلى الأنماط المضمَّنة، التي تُطبَّق مباشرة على العنصر عبر الخاصية `style`.

في هذا المثال، الفقرة الأولى ستكون باللون الأحمر بينما العناصر الأخرى من نوع `p` ستكون باللون الأزرق. ذلك لأن الأنماط المضمَّنة لها خصوصية أعلى من محددات النوع مثل محدد `p` الموضح في ملف `styles.css`.

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<p style="color: red;">Red paragraph</p>
<p>Other paragraph</p>
<p>Another paragraph</p>
<p>Yet another paragraph</p>
```

```css
p {
  color: blue;
}
```

:::

بعد ذلك، لمحددات المعرف (ID) خصوصية عالية.

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<p id="para">Red paragraph</p>
<p>Other paragraph</p>
<p>Another paragraph</p>
<p>Yet another paragraph</p>
```

```css
#para {
  color: red; 
}

p {
  color: blue;
}
```

:::

بعدها تأتي محددات الصنف، والمحددات الخاصة بالخاصيات، والفئات المزيفة.

تشمل الأمثلة محددات الصنف مثل `.container` و`.button`، ومحددات الخاصية مثل `[type="text"]`، والفئات المزيفة مثل `:hover`. هذه لها مستوى متوسط من الخصوصية.

**ملاحظة**: ستتعلم المزيد عن الفئات المزيفة في الدروس القادمة.

في هذا المثال، ستكون الفقرة الأولى باللون الأحمر لأن محدد المعرف له خصوصية أعلى من الصنف ومحددات النوع. عناصر `.example-para` ستكون باللون الأخضر بينما بقية عناصر الفقرة ستكون باللون الأزرق.

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<p id="para">Example paragraph</p>
<p class="example-para">Other paragraph</p>
<p class="example-para">Another paragraph</p>
<p>Yet another paragraph</p>
```

```css
#para {
  color: red; 
}

.example-para {
  color: green;
}

p {
  color: blue;
}
```

:::

محددات النوع، مثل `div` و`h1`، والعناصر المزيفة مثل `::before` و`::after`، لها خصوصية أقل مقارنة بالمجموعات السابقة.

**ملاحظة**: ستتعلم المزيد عن العناصر المزيفة في الدروس القادمة.

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<p id="para">Example paragraph</p>
<p>Other paragraph</p>
<p>Yet another paragraph</p>
```

```css
#para {
  color: red; 
}

p {
  color: blue;
}
```

:::

أخيرًا، محدد الكل، الممثَّل بالنجمة `*`، له أقل خصوصية على الإطلاق.

إليك مثالًا على تعيين اللون لجميع العناصر إلى الأحمر باستخدام محدد `*`. لن ترى أي عناصر حمراء لأن هناك محددات معرف ونوع تتجاوز تلك الأنماط، مما يبرز الخصوصية المنخفضة لمحدد `*`.
:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<p id="para">Example paragraph</p>
<p>Other paragraph</p>
<p>Yet another paragraph</p>
```

```css
* {
  color: red;
}

#para {
  color: green; 
}

p {
  color: blue;
}
```

:::

تُحسب قيم الخصوصية كقيمة ذات أربعة أجزاء `(a, b, c, d)`:

- `a`: الأنماط المضمَّنة (`1` أو `0`).
- `b`: عدد محددات المعرف.
- `c`: عدد محددات الصنف، ومحددات الخاصية، والفئات المزيفة.
- `d`: عدد محددات النوع، والعناصر المزيفة، ومحددات الكل.

كل جزء من قيمة الخصوصية يحمل وزنًا مختلفًا:

- **الأنماط المضمَّنة** (`a`) لها أعلى وزن، تساهم بقيمة `1` في الجزء الأول من قيمة الخصوصية.
    
- **محددات المعرف** (`b`) تحمل الوزن التالي الأعلى، حيث يساهم كل معرف بقيمة `1` في الجزء الثاني من قيمة الخصوصية.
    
- **محددات الصنف، ومحددات الخاصية، والفئات المزيفة** (`c`) تحمل وزنًا متوسطًا، حيث يساهم كل منها بقيمة `1` في الجزء الثالث من قيمة الخصوصية.
    
- **محددات النوع والعناصر المزيفة** (`d`) لها أقل وزن، حيث يساهم كل منها بقيمة `1` في الجزء الرابع من قيمة الخصوصية.
    
- **محدد الكل** (`*`): يساهم محدد الكل بقيمة `0` في حساب الخصوصية ولا يؤثر على الخصوصية. إدراجه في محدد لا يغير قيمة الخصوصية.

لدى CSS المضمَّن أعلى خصوصية لأنه يُطبَّق مباشرة على العنصر. يتجاوز أي CSS داخلي أو خارجي. قيمة الخصوصية للأنماط المضمَّنة هي `(1, 0, 0, 0)`.

إليك مثالًا آخر على نمط مضمَّن:

:::interactive_editor

```html
<p style="color: red;">This text is red.</p>
```

:::

يُعرَّف CSS الداخلي داخل عنصر `style` في قسم `head` من وثيقة HTML. له خصوصية أقل من الأنماط المضمَّنة. يشترك كل من CSS الداخلي والخارجي في نفس مستوى الخصوصية — تُحدد الخصوصية بواسطة المحددات المستخدمة، وليس بمكان تعريف CSS. عندما تتساوى خصوصية قاعدتين، تفوز القاعدة التي تظهر لاحقًا في الوثيقة. لذا إذا تم ربط ملف CSS خارجي بعد كتلة `style` داخلية، فإن قواعد الخارجي لها الأسبقية. وإذا تم ربطه قبلها، تفوز القواعد الداخلية.

تُحدد قيمة الخصوصية للأنماط الداخلية بواسطة المحددات المستخدمة. على سبيل المثال، لمحدد معرف داخل CSS الداخلي قيمة خصوصية `(0, 1, 0, 0)`.

:::interactive_editor

```html
<head>
  <style>
    #text {
      color: blue;
    }
  </style>
</head>
<body>
  <div id="text">This text is blue.</div>
</body>
```

:::

في هذا المثال، سيكون النص أزرق ما لم يُطبَّق نمط مضمَّن أو محدد أكثر خصوصية.

يرتبط CSS الخارجي عبر عنصر `link` في قسم `head` ويُكتب في ملفات `.css` منفصلة. مثل CSS الداخلي، تُحدد خصوصية الأنماط الخارجية بواسطة المحددات المستخدمة. يوفر CSS الخارجي أفضل قابلية للصيانة للمشاريع الأكبر.

تُحدد قيمة الخصوصية للأنماط الخارجية أيضًا بواسطة المحددات المستخدمة. على سبيل المثال، لمحدد صنف داخل CSS الخارجي قيمة خصوصية `(0, 0, 1, 0)`.

:::interactive_editor

```html
<head>
  <link rel="stylesheet" href="styles.css">
</head>
<body>
  <div class="text">This text's color is defined in an external CSS file.</div>
</body>
```

```css
.text {
  color: purple;
}
```

:::

في هذا المثال، يُعرَّف لون النص في ملف `styles.css` وسيُطبَّق ما لم يتم تجاوزه بواسطة محدد أكثر خصوصية أو نمط مضمَّن.
