# --description--

Давайте дізнаємося, що таке термінал і чим він відрізняється від командного рядка.

Командний рядок — це базовий текстовий інтерфейс введення, який дозволяє користувачу вводити "команди", зазвичай у вигляді послідовності символів, і відправляти або виконувати їх, зазвичай натискаючи клавішу "Enter". Найчастіше ви побачите інтерфейси командного рядка всередині терміналу.

Термінал — це спеціальний застосунок, який пропонує інтерфейс командного рядка для виконання системних команд, що виходять за межі базових операцій читання/запису. Ви також можете почути про "емулятори терміналу" — це, по суті, застосунки, які обгортають базовий інтерфейс терміналу, щоб надати додаткові функції або можливості. Але для більшості загальних випадків ви можете безпечно називати їх просто "терміналами".

Нарешті, нам також потрібно поговорити про "шелли". Шелл — це програмне забезпечення, яке обгортає командний рядок і інтерпретує ваші введення як команди, повертаючи вихідні дані.

Але як ви можете отримати доступ до терміналу у своїй системі?

Якщо у вас сучасна версія Windows, у вас, ймовірно, є два варіанти. Windows пропонує два окремі застосунки шеллів: Powershell і Command Prompt.

Обидва ці шелли мають власні застосунки терміналу (які мають назву відповідного шеллу), тож ви можете скористатися меню застосунків, щоб запустити термінал Powershell або термінал Command Prompt.

Windows також пропонує відносно новий і модернізований емулятор терміналу з назвою Microsoft Terminal.

Ви можете встановити цей застосунок із Windows Store і запускати його так само, як і термінали Powershell або Command Prompt. Перевага Microsoft Terminal у тому, що він дозволяє отримати доступ до обох шеллів — Powershell і Command Prompt — з одного застосунку, а також до відповідного Linux шеллу, якщо ви налаштували Windows Subsystems for Linux.

MacOS пропонує термінал з назвою "Terminal". Ви можете отримати до нього доступ через пошук Spotlight або через запуск застосунків.

MacOS також підтримує сторонні емулятори терміналу, такі як iTerm або Ghostty. Ви можете встановити ці застосунки з їхніх домашніх сторінок.

У Linux термінал за замовчуванням (і спосіб його запуску) сильно залежить від дистрибутива та середовища робочого столу, яке ви використовуєте. Існує багато підтримки сторонніх емуляторів терміналу. Наприклад, ви можете налаштувати kitty в Arch Linux — і запускати його через менеджер застосунків, такий як wofi.

Linux, мабуть, має найбільший вибір застосунків терміналу. Більшість із них можна встановити через менеджер пакетів вашого дистрибутива.

Розрізнення між "терміналом", "шеллом" і "командним рядком" іноді може здаватися надто педантичним. І в більшості випадків ці терміни можна використовувати взаємозамінно. Але важливо знати, що вони мають конкретні значення і розуміти, які саме.
