# --description--

戦略とはユーザーを認証する方法のことです。戦略を使うことで、ユーザーが最初に登録していればローカルに保存された情報に基づいて認証したり、GoogleやGitHubなどのさまざまなプロバイダーから認証を行ったりできます。このプロジェクトでは、Passportミドルウェアを使用します。Passportは、ユーザー名とパスワード、GitHub、Googleなどを使った認証をサポートする包括的な戦略セットを提供しています。

`passport-local@~1.0.0` はすでに依存関係として追加されています。サーバーに次のように追加してください。

```javascript
const LocalStrategy = require('passport-local');
```

Passportに対して、いくつかの設定が定義されたインスタンス化済みの`LocalStrategy`オブジェクトを**使用する**よう指示します。これ（およびここから先のすべて）は、データベース接続内にカプセル化されていることを必ず確認してください。なぜなら、これに依存しているからです！:

```javascript
passport.use(new LocalStrategy((username, password, done) => {
  myDataBase.findOne({ username: username }, (err, user) => {
    console.log(`User ${username} attempted to log in.`);
    if (err) return done(err);
    if (!user) return done(null, false);
    if (password !== user.password) return done(null, false);
    return done(null, user);
  });
}));
```

これは、ローカルでユーザーを認証しようとするときに使う処理を定義しています。まず、入力されたユーザー名でデータベース内のユーザーを探します。次に、パスワードが一致するかを確認します。最後に、（例えばパスワードが間違っているなど）チェックしたエラーがなければ、`user`オブジェクトが返され、認証されます。

多くの戦略は異なる設定でセットアップされます。一般的には、その戦略のリポジトリのREADMEに基づいて簡単にセットアップできます。良い例がGitHub戦略で、ユーザー名やパスワードを気にする必要はありません。ユーザーはGitHubの認証ページに送られ、ログインして同意すればGitHubがプロフィールを返してくれます。

次のステップでは、フォームデータに基づいてユーザーを検証するために実際に認証戦略を呼び出す方法を設定します。
