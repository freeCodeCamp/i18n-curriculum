## Arbeiten mit Formularen in React

- **Kontrollierte Eingaben**: Dabei speichern Sie den Wert des Eingabefelds im State und aktualisieren ihn über `onChange`-Ereignisse. Das gibt Ihnen die volle Kontrolle über die Formulardaten und ermöglicht sofortige Validierung und bedingtes Rendern.

```jsx
import { useState } from "react";

function App() {
  const [name, setName] = useState("");

  const handleChange = (e) => {
    setName(e.target.value);
  };

  const handleSubmit = (e) => {
    e.preventDefault();
    console.log(name);
  };

  return (
    <>
      <form onSubmit={handleSubmit}>
        <label htmlFor="name">Your name</label> <br />
        <input value={name} id="name" onChange={handleChange} type="text" />
        <button type="submit">Submit</button>
      </form>
    </>
  );
}

export default App;
```

- **Unkontrollierte Eingaben**: Anstatt die Eingaben über den `useState`-Hook zu steuern, behalten unkontrollierte Eingaben in HTML ihren eigenen internen Zustand mithilfe des DOM. Da das DOM die Eingabewerte kontrolliert, müssen Sie die Werte der Eingabefelder mit einem `ref` abrufen.

```jsx
import { useRef } from "react";

function App() {
 const nameRef = useRef();

 const handleSubmit = (e) => {
   e.preventDefault();
   console.log(nameRef.current.value);
 };

 return (
   <form onSubmit={handleSubmit}>
     <label htmlFor="name">Your</label>{" "}
     <input type="text" ref={nameRef} id="name" />
     <button type="submit">Submit</button>
   </form>
 );
}

export default App;
```

## Arbeiten mit dem `useActionState`-Hook

- **Serveraktionen**: Das sind Funktionen, die auf dem Server ausgeführt werden, um die Formularverarbeitung direkt auf dem Server zu ermöglichen, ohne dass API-Endpunkte benötigt werden. Hier ist ein Beispiel aus einer Next.js-Anwendung:

```js
"use server";

async function submitForm(formData) {
 const name = formData.get("name");
 return { message: `Hello, ${name}!` };
}
```

Die `"user server"`-Direktive kennzeichnet die Funktion als Serveraktion.

- **`useActionState`-Hook**: Dieser Hook aktualisiert den State basierend auf dem Ergebnis einer Formulareinsendung. Hier ist die Grundsyntax des `useActionState`-Hooks:

```js
const [state, action, isPending] = useActionState(actionFunction, initialState, permalink);
```

- `state` ist der aktuelle State, den die Aktion zurückgibt.
- `action` ist die Funktion, die die Serveraktion auslöst.
- `isPending` ist ein Boolean, der angibt, ob die Aktion gerade läuft oder nicht.
- `actionFunction` Parameter ist die Serveraktion selbst.
- `initialState` ist der Parameter, der den Ausgangszustand vor Ausführung der Aktion darstellt.
- `permalink` ist ein optionaler String, der die eindeutige Seiten-URL enthält, die das Formular ändert.

## Datenabruf in React

- **Optionen zum Datenabruf**: Es gibt viele verschiedene Möglichkeiten, Daten in React abzurufen. Sie können die native Fetch-API verwenden oder Drittanbieter-Tools wie Axios oder SWR.
- **Häufig verwendete State-Variablen beim Datenabruf**: Unabhängig davon, welche Methode Sie wählen, um Daten in React abzurufen, gibt es einige State-Teile, die Sie verfolgen müssen. Der erste ist die eigentliche Datenmenge. Der zweite verfolgt, ob die Daten noch abgerufen werden. Der dritte ist eine State-Variable, die eventuelle Fehler während des Datenabrufs erfasst.

```js
const [data, setData] = useState(null);
const [loading, setLoading] = useState(true);
const [error, setError] = useState(null);
```

Da der Datenabruf ein Nebeneffekt ist, ist es am besten, den `Fetch API` innerhalb eines `useEffect`-Hooks zu verwenden.

```js
useEffect(() => {
  const fetchData = async () => {
    try {
      const res = await fetch("https://jsonplaceholder.typicode.com/posts");
      
      if (!res.ok) {
        throw new Error("Network response was not ok");
      }

      const data = await res.json();
      setData(data);
    } catch (err) {
      setError(err);
    } finally {
      setLoading(false);
    }
  };

  fetchData();
}, []);
```

Dann können Sie eine Ladeanzeige rendern, wenn der Datenabruf noch nicht abgeschlossen ist, eine Fehlermeldung, wenn ein Fehler aufgetreten ist, oder die Ergebnisse.

```jsx
if (loading) {
  return <p>Loading...</p>;
}

if (error) {
  return <p>{error.message}</p>;
}

return (
  <ul>
    {data.map((post) => (
      <li key={post.id}>{post.title}</li>
    ))}
  </ul>
);
```

Wenn Sie Axios verwenden möchten, müssen Sie es installieren und importieren:

```bash
npm i axios
```

```js
import axios from "axios";
```

Dann können Sie die Daten mit `axios.get` abrufen:

```jsx
const [data, setData] = useState(null);
const [loading, setLoading] = useState(true);
const [error, setError] = useState(null);

useEffect(() => {
  const fetchData = async () => {
    try {
      const res = await axios.get(
        "https://jsonplaceholder.typicode.com/users"
      );
      setData(res.data);
    } catch (err) {
      setError(err);
    } finally {
      setLoading(false);
    }
  };

  fetchData();
}, []);
```

Um Daten mit dem `useSWR`-Hook abzurufen, müssen Sie ihn zuerst installieren und importieren.

```bash
npm i swr
```

```js
import useSWR from "swr";
```

So können Sie den Hook zum Datenabruf verwenden:

```jsx
import useSWR from "swr";

const fetcher = (url) => fetch(url).then((res) => res.json());

const FetchTodos = () => {
 const { data, error } = useSWR(
   "https://jsonplaceholder.typicode.com/todos",
   fetcher
 );

 if (!data) {
   return <h2>Loading...</h2>;
 }
 if (error) {
   return <h2>Error: {error.message}</h2>;
 }

 return (
   <>
     <h2>Todos</h2>
     <div>
       {data.map((todo) => (
         <h3 key={todo.id}>{todo.title}</h3>
       ))}
     </div>
   </>
 );
};

export default FetchTodos;
```

## Arbeiten mit dem `useOptimistic`-Hook

- **`useOptimistic`-Hook**: Dieser Hook wird verwendet, um UIs responsiv zu halten, während im Hintergrund eine asynchrone Aktion abgeschlossen wird. Er hilft dabei, „optimistische Updates“ in der UI zu verwalten, eine Technik, bei der Sie sofortige UI-Updates basierend auf dem erwarteten Ergebnis einer Aktion bereitstellen, etwa während Sie auf eine Serverantwort warten.

Hier ist die Grundsyntax:

```js
const [optimisticState, addOptimistic] = useOptimistic(actualState, updateFunction);
```

- `optimisticState` ist der temporäre State, der sofort aktualisiert wird, um die Benutzererfahrung zu verbessern.
- `addOptimistic` ist die Funktion, die das optimistische Update vor der tatsächlichen State-Änderung anwendet.
- `actualState` ist der echte State-Wert, der aus dem Ergebnis einer Aktion stammt, z. B. dem Abruf von Daten vom Server.
- `updateFunction` ist die Funktion, die bestimmt, wie der optimistische State aktualisiert werden soll, wenn sie aufgerufen wird.

Hier ist ein Beispiel für die Verwendung des `useOptimistic`-Hooks in einer `TaskList`-Komponente:

```jsx
"use client";

import { useOptimistic } from "react";

export default function TaskList({ tasks, addTask }) {
  const [optimisticTasks, addOptimisticTask] = useOptimistic(
    tasks,
    (state, newTask) => [...state, { text: newTask, pending: true }]
  );

  async function handleSubmit(e) {
    e.preventDefault();
    const formData = new FormData(e.target);

    addOptimisticTask(formData.get("task"));

    addTask(formData);
    e.target.reset();
  }

  return <>{/* UI */}</>;
}
```

- **`startTransition`**: Dies wird verwendet, um einen Teil der UI zu rendern und ein State-Update als nicht dringende Transition zu markieren. So bleibt die UI während aufwändiger Updates responsiv. Hier ist die Grundsyntax:

```js
startTransition(action);
```

Der `action` führt ein State-Update durch oder löst eine transition-bezogene Logik aus. So wird sichergestellt, dass dringende UI-Updates (wie Tippen oder Klicken) nicht blockiert werden.

## Arbeiten mit dem `useMemo`-Hook

- **Memoisierung**: Dies ist eine Optimierungstechnik, bei der das Ergebnis teurer Funktionsaufrufe basierend auf bestimmten Argumenten zwischengespeichert (erinnert) wird. Wenn dieselben Argumente erneut übergeben werden, wird das zwischengespeicherte Ergebnis zurückgegeben, anstatt die Funktion neu zu berechnen.
- **`useMemo`-Hook**: Dieser Hook wird verwendet, um berechnete Werte zu memoizieren. Hier ist ein Beispiel, wie das Ergebnis der Sortierung eines großen Arrays memoisiert wird. Der `expensiveSortFunction` wird nur ausgeführt, wenn sich `largeArray` ändert:

```js
const memoizedSortedArray = useMemo(
  () => expensiveSortFunction(largeArray),
  [largeArray]
);
```

## Arbeiten mit dem `useCallback`-Hook

- **`useCallback`-Hook**: Dieser Hook wird verwendet, um Funktionsreferenzen zu memoizieren.

```js
const handleClick = useCallback(() => {
  // code goes here
}, [dependency]);
``` 

- **`React.memo`**: Dies wird verwendet, um eine Komponente zu memoizieren, um unnötige Re-Renders zu verhindern, wenn sich ihre Prop nicht geändert hat.

```jsx
const MemoizedComponent = React.memo(({ prop }) => {
 return (
   <>
     {/* Presentation */}
   </>
 )
});
``` 

## Abhängigkeitsverwaltungstools

- **Definition von Abhängigkeiten**: In der Software ist eine Abhängigkeit gegeben, wenn eine Komponente oder ein Modul in einer Anwendung auf eine andere angewiesen ist, um richtig zu funktionieren. Abhängigkeiten sind in Softwareanwendungen üblich, weil sie Entwicklern erlauben, vorgefertigte Funktionen oder Werkzeuge zu verwenden, die von anderen erstellt wurden. Die zwei Kernabhängigkeiten, die für ein React-Projekt benötigt werden, sind die `react`- und `react-dom`-Pakete:

```json
"dependencies": {
  "react": "^18.3.1",
  "react-dom": "^18.3.1"
}
```

- **Definition eines Paketmanagers**: Um Softwareabhängigkeiten in einem Projekt zu verwalten, benötigen Sie einen Paketmanager. Ein Paketmanager ist ein Werkzeug für Installation, Updates und Entfernung von Abhängigkeiten. Viele populäre Programmiersprachen wie JavaScript, Python, Ruby und Java verwenden Paketmanager. Beliebte Paketmanager für JavaScript sind npm, Yarn und pnpm.
- **`package.json`-Datei**: Dies ist eine wichtige Konfigurationsdatei in Projekten, die Metadaten über Ihr Projekt enthält, einschließlich Name, Version und Abhängigkeiten. Sie definiert auch Skripte, Lizenzinformationen und andere Einstellungen, die bei der Verwaltung des Projekts und seiner Abhängigkeiten helfen.
- **`package-lock.json`-Datei**: Diese Datei sperrt die exakten Versionen aller Pakete, die Ihr Projekt verwendet. Wenn Sie ein Paket aktualisieren, werden die neuen Versionen auch in der Lock-Datei aktualisiert.
- **`node_modules`-Ordner**: Dieser Ordner enthält den tatsächlichen Code für die Abhängigkeiten, die in Ihrer `package.json`-Datei aufgelistet sind, einschließlich der direkten Abhängigkeiten Ihres Projekts und aller Abhängigkeiten dieser Abhängigkeiten.
- **Dev-Abhängigkeiten**: Das sind Pakete, die nur für die Entwicklung verwendet werden und nicht in der Produktion. Ein Beispiel dafür ist eine Testbibliothek wie Jest. Sie würden Jest als Dev-Abhängigkeit installieren, weil es für das lokale Testen Ihrer Anwendung benötigt wird, aber nicht, um die Anwendung in der Produktion auszuführen.

## React Router

- **Einführung**: React Router ist eine Drittanbieter-Bibliothek, mit der Sie Routing zu Ihren React-Anwendungen hinzufügen können. Um zu beginnen, müssen Sie React Router in einem bestehenden React-Projekt so installieren:

```bash
npm i react-router
```

Dann müssen Sie innerhalb der `main.jsx`- oder `index.jsx`-Datei die Routenstruktur so einrichten:

```jsx
import { StrictMode } from "react";
import { createRoot } from "react-dom/client";
import { BrowserRouter, Routes, Route } from "react-router";
import App from "./App.jsx";

import "./index.css";

createRoot(document.getElementById("root")).render(
  <StrictMode>
    <BrowserRouter>
      <Routes>
        <Route path="/" element={<App />} />
      </Routes>
    </BrowserRouter>
  </StrictMode>
);
```

Die `path` und `element` werden verwendet, um die URL und UI-Komponenten miteinander zu koppeln. In diesem Fall richten wir eine Route für die Startseite ein, die auf die `App`-Komponente zeigt.

- **Mehrere Ansichten und Routenaufbau**: In größeren Anwendungen ist es üblich, mehrere Ansichten und Routen so einzurichten:

```jsx
<Routes>
  <Route index element={<Home />} />
  <Route path="about" element={<About />} />

  <Route path="products">
    <Route index element={<ProductsHome />} />
    <Route path=":category" element={<Category />} />
    <Route path=":category/:productId" element={<ProductDetail />} />
    <Route path="trending" element={<Trending />} />
  </Route>
</Routes>
```

Die `index`-Prop in diesen Beispielen soll die Standardroute für ein bestimmtes Pfadsegment darstellen. Die `Home`-Komponente wird also am Root-`/`-Pfad angezeigt, während die `ProductsHome`-Komponente am `/products`-Pfad angezeigt wird.

- **Verschachtelte Routen**: Sie können Routen innerhalb anderer Routen verschachteln, wodurch der Pfad der Kindroute an den Pfad der Elternroute angehängt wird.

```jsx
<Route path="products">
  <Route path="trending" element={<Trending />} />
</Route>
```

Im obigen Beispiel lautet der Pfad für die Trendprodukte products/trending.

- **Dynamische Segmente**: Ein dynamisches Segment ist ein Teil des URL-Pfads, der dynamisch ist.

```jsx
<Route path=":category" element={<Category />} />
```

In diesem Beispiel haben wir ein dynamisches Segment namens `category`. Wenn ein Benutzer zu einer URL wie products/brass-instruments navigiert, ändert sich die Ansicht zur `Category`-Komponente, und Sie können die passenden Daten basierend auf dem Segment dynamisch abrufen.

- **`useParams`-Hook**: Dieser Hook wird verwendet, um auf die dynamischen Parameter eines URL-Pfads zuzugreifen.

```jsx
import { useParams } from "react-router";

export default function Category() {
  let params = useParams();
  {/* Accessing the category param: params.category */}
  {/* rest of code goes here */}
}
```

## React-Frameworks

- **Einführung**: React-Frameworks bieten Funktionen wie Routing, Bildoptimierung, Datenabruf, Authentifizierung und mehr. Das bedeutet, dass Sie für bestimmte Anwendungsfälle möglicherweise keine separaten Frontend- und Backend-Anwendungen einrichten müssen. Beispiele für React-Frameworks sind Next.js und Remix.
- **Next.js Routing**: Dieses Routingsystem unterstützt dynamische Routen, parallele Routen, Routen-Handler, Umleitungen, Internationalisierung und mehr.

Hier ist ein Beispiel für die Erstellung eines benutzerdefinierten Request-Handlers:

```js
export async function GET() {
  const res = await fetch("https://example-api.com");
  const data = await res.json();

  return Response.json({ data });
}
```

- **Next.js Bildoptimierung**: Die `Image`-Komponente erweitert das native HTML-`img`-Element und ermöglicht schnellere Seitenladezeiten und Größenoptimierungen. Das bedeutet, dass Bilder nur geladen werden, wenn sie in den Viewport gelangen, und die `Image`-Komponente automatisch korrekt dimensionierte Bilder für jedes Gerät bereitstellt.

```jsx
import Image from "next/image";

export default function Page() {
  return (
    <Image src="link-to-image-goes-here" alt="descriptive-title-goes-here" />
  );
}
``` 

## Prop Drilling

- **Definition**: Prop Drilling ist der Prozess, Props von einer übergeordneten Komponente an tief verschachtelte Kindkomponenten weiterzugeben, selbst wenn einige der Kindkomponenten die Props nicht benötigen.

## Zustandsverwaltung

- **Context API**: Kontext bedeutet, dass eine übergeordnete Komponente Informationen für Kindkomponenten verfügbar macht, ohne sie explizit über Props weitergeben zu müssen. `createContext` wird verwendet, um ein Kontextobjekt zu erstellen, das den Kontext darstellt, den andere Komponenten lesen. `Provider` wird verwendet, um Kontextwerte an die Kindkomponenten zu liefern.

```jsx
import { useState, createContext } from "react";

const CounterContext = createContext();

const CounterProvider = ({ children }) => {
  const [count, setCount] = useState(0);

  return (
    <CounterContext.Provider value={{ count, setCount }}>
      {children}
    </CounterContext.Provider>
  );
};

export { CounterContext, CounterProvider };
```

- **Redux**: Redux verwaltet den State, indem es einen zentralen Store und strenge Kontrolle über State-Updates bereitstellt. Es verwendet ein vorhersehbares Muster mit Aktionen, Reducern und Middleware. Aktionen sind Nutzlasten von Informationen, die Daten von Ihrer Anwendung an den Redux-Store senden, oft ausgelöst durch Benutzerinteraktionen. Reducer sind Funktionen, die festlegen, wie sich der State als Reaktion auf diese Aktionen ändern soll, und sorgen dafür, dass der State unveränderlich aktualisiert wird. Middleware fungiert als Brücke zwischen dem Dispatchen von Aktionen und dem Reducer, sodass Sie die Funktionalität von Redux erweitern können (z. B. Logging, asynchrone Operationen), ohne den Kernfluss zu verändern.

- **Zustand**: Diese Lösung zur Zustandsverwaltung ist ideal für kleine bis mittelgroße Anwendungen. Sie funktioniert, indem sie einen `useStore`-Hook verwendet, um direkt in Komponenten und Seiten auf den State zuzugreifen. So können Sie Daten ändern und abrufen, ohne Aktionen, Reducer oder einen Provider zu benötigen.

## Debuggen von React-Komponenten mit den React-Entwicklerwerkzeugen

- **React Developer Tools**: Dies ist eine Browsererweiterung, die Sie in Chrome, Firefox und Edge verwenden können, um React-Komponenten zu inspizieren und Leistungsprobleme zu identifizieren. Für Safari müssen Sie das `react-devtools`-npm-Paket installieren. Nach der Installation von React DevTools und dem Öffnen einer React-App im Browser öffnen Sie die Entwicklerwerkzeuge des Browsers, um auf die zwei zusätzlichen Tabs zum Debuggen von React – Komponenten und Profiler – zuzugreifen.
- **Komponenten-Tab**: Dieser Tab zeigt jede Komponente in einer Baumansicht an. Hier können Sie:
  - die Komponenten-Hierarchie der App ansehen
  - Props, States und Kontextwerte in Echtzeit prüfen und ändern
  - den Quellcode jeder ausgewählten Komponente ansehen
  - die Komponentendaten in der Konsole protokollieren
  - die DOM-Elemente der Komponente inspizieren
- **Profiler-Tab**: Dieser Tab hilft Ihnen, die Leistung von Komponenten zu analysieren. Sie können die Leistung der Komponenten aufzeichnen, um unnötige Re-Renders zu identifizieren, Commit-Dauern ansehen und anschließend langsame Komponenten optimieren.

## React Server Components

- **Definition**: React Server Components sind React-Komponenten, die ausschließlich auf dem Server gerendert werden und nur das finale HTML an den Client senden. Das bedeutet, dass diese Komponenten direkt auf serverseitige Ressourcen zugreifen können und die Menge an JavaScript, die an den Browser gesendet wird, drastisch reduzieren.
