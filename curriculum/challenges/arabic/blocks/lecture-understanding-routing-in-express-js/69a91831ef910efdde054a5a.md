# --description--

في Express، معلمات المسار هي قيم ديناميكية مضمّنة في مسار URL. تتيح لتطبيقك التقاط أجزاء من URL يمكن أن تتغير مع كل طلب. هذا مفيد لبناء صفحات ديناميكية، مثل عرض مستخدم معين أو منشور مدونة أو منتج بناءً على URL.

إليك مثالاً:

```javascript
// Route with a parameter in the path
app.get("/post/:postId", (req, res) => {
  const postId = req.params.postId;
  res.send(`Viewing post with ID: ${postId}`);
});
```

هنا، يستمع المسار لطلبات GET إلى `/post/:postId`.

* `:postId` هي معلمة مسار، عنصر نائب في URL سيتم استبداله بقيمة فعلية. على سبيل المثال، الطلب إلى `/post/42` يطابق هذا المسار، و`42` تصبح قيمة `postId`.
    
* يمكنك الوصول إلى معلمات المسار باستخدام `req.params`. يحتوي كائن `req.params` على جميع معلمات المسار، وفي هذه الحالة، يحتوي `req.params.postId` على القيمة من URL، وهي `42`.
    

أخيرًا، ترسل الدالة `res.send()` استجابة إلى العميل. لذا إذا كان URL هو `/post/42`، يرى العميل النص `Viewing post with ID: 42`.

كما ترى، تجعل معلمات المسار مساراتك مرنة وديناميكية دون الحاجة إلى مسار منفصل لكل قيمة ممكنة.

الآن بعد أن فهمت كيف تعمل معلمات المسار، دعنا نعود إلى مثال من الدرس السابق ونرى كيف يمكنك استخدام المسارات الديناميكية لأنواع مختلفة من القوائم:

```javascript
app.get("/menu/:category", (req, res) => {
  const category = req.params.category;
  res.send(`You are viewing the ${category} menu!`);
});
```

وهنا تفصيل لكيفية عمل ذلك:

1. يزور المستخدم [`http://localhost:3000/menu/drinks`](http://localhost:3000/menu/drinks).
    
2. يطابق الخادم هذا الطلب مع المسار `/menu/:category` ويلتقط معلمة `category` كـ `drinks`.
    
3. يرسل الخادم هذه الاستجابة إلى المتصفح: `You are viewing the drinks menu!`
    

بنفس الطريقة، إذا زار المستخدم [`http://localhost:3000/menu/food`](http://localhost:3000/menu/food)، يرد الخادم بالرسالة التالية: `You are viewing the food menu!`

تتيح معلمات المسار عرض محتوى ديناميكي بناءً على URL. بدلاً من إنشاء مسار منفصل لكل فئة، يمكنك استخدام مسار واحد للتعامل مع فئات متعددة مثل المشروبات، الطعام، الحلويات، والمزيد، فقط بتغيير URL.
