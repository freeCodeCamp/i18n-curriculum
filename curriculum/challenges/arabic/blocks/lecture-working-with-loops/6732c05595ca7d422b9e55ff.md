# --interactive--

تُستخدم حلقة `for...in` بشكل أفضل عندما تحتاج إلى التكرار على خصائص كائن. ستكرر هذه الحلقة جميع الخصائص القابلة للعدّ في الكائن، بما في ذلك الخصائص الموروثة والخصائص غير الرقمية.

الخاصية الموروثة هي خاصية تُورث من سلسلة النموذج الأولي للكائن. الخاصية غير الرقمية هي خاصية ليست رقمًا أو سلسلة نصية يمكن تحويلها إلى رقم.

إليك الصيغة النحوية الأساسية لحلقة `for...in`:

```js
for (variable in object) {
  // code block to be executed
}
```

المتغير في المثال يمثل الخاصية الحالية للكائن التي يتم التكرار عليها.

لنلقِ نظرة على بعض الأمثلة لتفهم كيف تعمل حلقة `for...in` بشكل أفضل.

في هذا المثال الأول لدينا كائن `fruit` ونريد التكرار على كل خاصية وتسجيل القيمة في وحدة التحكم.

:::interactive_editor

```js
const fruit = {
  name: 'apple',
  color: 'red',
  price: 0.99
};

for (const prop in fruit) {
  console.log(fruit[prop]);
}
```

:::

المتغير `prop` يمثل الخاصية الحالية للكائن. تُستخدم `fruit[prop]` للوصول إلى قيمة كل خاصية.

في التكرار الأول، ستكون قيمة `prop` هي `name`. في التكرار الثاني، ستكون `prop` هي `color`، وهكذا.

النتائج التي ستُسجل في وحدة التحكم ستكون `apple`، و`red`، و`0.99`.

في هذا المثال الثاني، لدينا كائن مسكن ونريد التكرار على كل خاصية وتسجيل القيمة في وحدة التحكم.

:::interactive_editor

```js
const person = {
  name: 'John',
  age: 30,
  address: {
    street: '123 Main St',
    city: 'Anytown',
    state: 'CA'
  }
};

for (const prop in person) {
  console.log(person[prop]);
}
```

:::

الخاصية `address` هي كائن بحد ذاته. ستكرر حلقة `for...in` أيضًا على خصائص كائن `person` وتُسجل كائن `address` بالكامل في وحدة التحكم.

إليك كيف سيبدو الناتج في وحدة التحكم:

```md
John
30
{ street: '123 Main St', city: 'Anytown', state: 'CA' }
```

إذا أردت التكرار على خصائص كائن `address`، يمكنك تسكين حلقة `for...in` أخرى داخل الأولى.

:::interactive_editor

```js
const person = {
  name: 'John',
  age: 30,
  address: {
    street: '123 Main St',
    city: 'Anytown',
    state: 'CA'
  }
};

function isObject(obj) {
  return typeof obj === 'object' && !Array.isArray(obj) && obj !== null;
}

for (const prop in person) {
  if (isObject(person[prop])) {
    for (const nestedProp in person[prop]) {
      console.log(person[prop][nestedProp]);
    }
  } else {
    console.log(person[prop]);
  }
}
```

:::

في هذا المثال، لدينا دالة مخصصة `isObject` تتحقق مما إذا كانت القيمة كائنًا.

تُستخدم الطريقة `Array.isArray` للتحقق مما إذا كانت القيمة مصفوفة. بوضع عامل النفي المنطقي (`!`) أمام الطريقة، نتحقق مما إذا كانت القيمة ليست مصفوفة.

السبب في أننا لا نستطيع فقط استخدام `typeof` يساوي `'object'` هو أن المصفوفات تُعتبر أيضًا كائنات في JavaScript. نريد استبعاد المصفوفات من التحقق.

أيضًا، بسبب خطأ تاريخي في JavaScript، تُرجع `typeof null` القيمة `'object'`. لذلك نريد أيضًا استبعاد القيم `null` من التحقق.

إذا تحقق الشرط، نسكن حلقة `for...in` أخرى ستكرر على خصائص الكائن المسكن وتُسجل القيمة في وحدة التحكم.

المتغير `nestedProp` يمثل الخاصية الحالية للكائن المسكن.

إليك كيف سيبدو الناتج المعدل في وحدة التحكم:

```md
"John"
30
"123 Main St"
"Anytown"
"CA"
```

تُعتبر حلقة `for...in` مفيدة عندما تحتاج إلى التكرار على خصائص كائن.

لا يُنصح باستخدام حلقة `for...in` للتكرار على عناصر مصفوفة. بدلاً من ذلك، استخدم حلقة `for...of` أو طرق المصفوفة الأخرى مثل `forEach`، و`map`، و`filter`، و`reduce`، التي ستتعلمها في دروس لاحقة.
