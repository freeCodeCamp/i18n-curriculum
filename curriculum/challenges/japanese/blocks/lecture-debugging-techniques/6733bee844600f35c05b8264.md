# --interactive--

前のレッスンでは、プログラムで例外をスローする方法を学びました。このレッスンでは、これらのエラーを `try…catch…finally` ブロックで上手に処理する方法を見ていきます。

`try` ブロックは、エラーをスローする可能性があるコードを囲むために使います。失敗するかもしれない処理を試す安全な場所として機能します。

`catch` ブロックは、`try` ブロックで発生したエラーをキャッチして処理します。`Error` 内で `catch` オブジェクトを使って、何が問題だったのかを調べることができます。

`finally` ブロックは、エラーの有無にかかわらず `try` と `catch` ブロックの後に実行されます。ファイルを閉じたりリソースを解放したりするクリーンアップ処理に使われることが多いです。

以下は `try…catch` ブロックを使った例です：

:::interactive_editor

```js
function processInput(input) {
  if (typeof input !== "string") {
    throw new TypeError("Input must be a string.");
  }

  return input.toUpperCase();
}

try {
  console.log("Starting to process input...");
  const result = processInput(9);
  console.log("Processed result:", result);
} catch (error) {
  console.error("Error occurred:", error.message);
} 
```

:::

この例では、`processInput` という関数があり、まず `input` が `string` 型でないかをチェックしています。そうであればエラーをスローします。そうでなければ、`toUpperCase` に対して `input` メソッドを使った結果を返します。

この関数呼び出しは `try` ブロック内で行われています。関数呼び出しがエラーをスローするため、`catch` ブロック内でキャッチされ、エラーメッセージがコンソールに表示されます。

`catch` ブロックに渡されるエラーは `Error` オブジェクトで、そのエラーに関する情報が含まれています。この場合、ユーザーにわかりやすい情報を表示する `message` プロパティを使っています。

`console.error` を使っているのは、エラーをログに記録するために特化しているからです。多くの最新ブラウザでは、`console.error()` の出力がコンソールで赤色で表示されます。

`finally` 文は、例外がスローされたかどうかにかかわらず実行されます。

```js
try {
  // Code that might throw an error
} catch (error) {
  // Code to handle the error
} finally {
  // Code that runs regardless of whether an error occurred or not
}
```

`finally` 文の良い使い方は、ファイルを扱う場合です。JavaScriptではファイルを開き、`try` ブロックでファイルにデータを書き込みます。エラーがあれば `catch` でキャッチし、`finally` 文でファイルを閉じます。
