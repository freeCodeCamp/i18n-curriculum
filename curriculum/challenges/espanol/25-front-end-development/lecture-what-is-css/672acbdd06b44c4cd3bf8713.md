¿Cuáles son algunos estilos predeterminados de los navegadores aplicados al HTML?

Cuando empiezas a trabajar con HTML y CSS, notarás que algunos estilos se aplican a tus páginas web incluso antes de que escribas cualquier CSS. Estos estilos se llaman "estilos predeterminados de navegador" o "estilos de usuario-agente".

Estos estilos predeterminados del navegador proporcionan un formato básico para asegurar que los elementos HTML se muestren de una manera legible en todos los navegadores.

Sin embargo, estos estilos pueden variar ligeramente de un navegador a otro, por lo que es importante entenderlos cuando diseñes un aspecto consistente para tu sitio web.

Echemos un vistazo a algunos estilos predeterminados comunes del navegador aplicados a varios elementos HTML.

Los encabezados son uno de los elementos HTML más usados comúnmente y están estilizados por defecto para tener tamaños y pesos variados.

Por ejemplo, `h1` (el encabezado de nivel más alto) suele ser en negrita y de mayor tamaño de fuente en comparación con los encabezados de nivel inferior como `h2`, `h3`, y así sucesivamente.

Estas etiquetas de encabezado ayudan a definir la jerarquía y estructura de tu contenido.

El siguiente ejemplo muestra los seis elementos de encabezado desde `h1` a `h6`.

```html
<h1>Heading 1</h1>
<h2>Heading 2</h2>
<h3>Heading 3</h3>
<h4>Heading 4</h4>
<h5>Heading 5</h5>
<h6>Heading 6</h6>
```

Otro elemento con estilos predeterminados es el elemento `hr`, que crea una regla horizontal que se usa a menudo para separar visualmente secciones del contenido. Los navegadores generalmente aplican un estilo de línea simple a este elemento.

Para entenderlo mejor, veamos este ejemplo de código:

```html
<p>Paragraph 1</p>
<p>Paragraph 2</p>
<hr>
<p>Paragraph 3</p>
<p>Paragraph 4</p>
```

La regla horizontal aparece como una línea delgada con espaciado arriba y abajo del texto para distinguir entre secciones del contenido.

A continuación, observaremos el elemento `blockquote`.

Las citas en bloque se usan para indicar una sección de texto que es una cita de otra fuente. Los navegadores normalmente añaden sangría y a veces inclinan el texto.

La sangría ayuda a distinguir las citas en bloque del resto del texto, haciendo claro que es contenido citado de otra fuente.

Echemos un vistazo a este ejemplo de código:

```html
<p>Paragraph 1</p>
<p>Paragraph 2</p>
<blockquote>I think, therefore I am. (Rene Descartes)</blockquote>
<p>Paragraph 3</p>
<p>Paragraph 4</p>
```

En el ejemplo anterior, el elemento `blockquote` producirá una sangría en el texto, aplicada por los estilos predeterminados del navegador.

Otro elemento con estilos predeterminados es el elemento de anclaje (`<a>`), el cual se estiliza con un color azul predeterminado y subrayado para que sea reconocible como enlace.

Vamos a ver el siguiente ejemplo de HTML:

```html
<p>Paragraph 1</p>
<p>Paragraph 2</p>
<a href="https://freecodecamp.org/">Visit the freeCodeCamp learn page</a>
<p>Paragraph 3</p>
<p>Paragraph 4</p>
```

El código anterior tiene cuatro elementos de párrafo con un elemento de anclaje en el medio. El elemento de anclaje está estilizado en azul con un subrayado para mostrar a los usuarios cómo ir a la página de aprendizaje de freeCodeCamp.

Finalmente, veremos los elementos de lista no ordenada y lista ordenada.

Los navegadores añaden un formato básico a las listas, incluyendo sangría y puntos o números, dependiendo de si estás usando una lista no ordenada (`ul`) o una lista ordenada (`ol`).

Veamos un ejemplo:

```html
<p>Sample Paragraph</p>
<ol>
  <li>item 1</li>
  <li>item 2</li>
  <li>item 3</li>
</ol>
<ul>
  <li>item</li>
  <li>another item</li>
  <li>yet another item</li>
</ul>
<p>Ending Paragraph</p>
```

En el ejemplo de código anterior, estamos usando una lista no ordenada y una lista ordenada. Ambas listas estarán ligeramente sangradas a la derecha por defecto.

En todos estos ejemplos, viste cómo el navegador aplicó espaciado predeterminado, tamaños de fuente variados y pesos a estos elementos HTML.

En videos de lecturas posteriores, aprenderás cómo restablecer algunos de estos estilos predeterminados usando un reset de CSS.
