# --description--

현대 웹 개발에서는 코드를 "모듈"이라는 여러 파일로 나눌 수 있습니다.

모듈은 독립적이고 재사용 가능한 코드 블록으로 생각할 수 있습니다.

모듈을 다른 모듈에 가져오면 기능을 재사용하면서도 코드를 별도의 파일에 유지할 수 있어, 특히 크고 복잡한 웹 애플리케이션에서 유지 관리가 더 쉬워집니다.

Node.js에서는 각 파일이 하나의 모듈로 간주됩니다.

Node에는 모듈을 다루는 두 가지 시스템이 있습니다: CommonJS와 ES입니다.

CommonJS는 Node.js에서 패키지를 다루는 원래이자 기본 방식입니다. 모듈을 동기적으로 로드하는데, 이는 모듈이 완전히 로드될 때까지 다음 모듈을 로드하거나 코드를 계속 실행하지 않고 기다린다는 뜻입니다.

여전히 널리 사용되지만 점차 새로운 ES 구문으로 대체되고 있습니다.

CommonJS는 `require()` 함수를 사용해 모듈을 가져오며, 상대 경로를 인수로 받습니다.

예를 들어, 기본 수학 연산을 정의한 `math.js`라는 모듈이 있다고 가정해 봅시다.

이 예제를 간단하고 집중적으로 유지하기 위해 곱셈 함수만 사용하겠습니다:

```js
function multiply(a, b) {
  return a * b;
}
```

현재 모듈 외부에서 이 함수를 어떻게 재사용할 수 있을까요? 기본적으로 Node.js 모듈에서 정의한 함수와 변수는 **비공개**입니다 — 다른 모듈이 명시적으로 내보내지 않으면 접근할 수 없습니다. 이를 위한 여러 방법이 있습니다.

일반적인 방법 중 하나는 `module.exports`에 값을 할당하는 것입니다. 이 객체는 모듈이 다른 곳에서 가져올 때 반환할 내용을 나타내며, 다른 모듈이 노출하기로 선택한 함수, 변수 또는 객체에 접근할 수 있게 합니다.

이 예제에서는 `multiply` 속성을 가진 객체를 `module.exports`에 할당했습니다. 그 속성의 값은 앞서 정의한 `multiply` 함수에 대한 참조입니다. 이렇게 하면 함수를 내보내서 필요할 때 다른 모듈에서 호출할 수 있습니다.

```javascript
function multiply(a, b) {
  return a * b;
}

module.exports = {
  multiply: multiply
};
```

다른 모듈에서 `math.js` 모듈을 가져온다고 가정해 봅시다. `require()` 함수를 호출하고 상대 경로를 인수로 전달합니다. 그런 다음 `require()`이 반환한 객체를 `math` 변수에 할당합니다.

```javascript
const math = require('./math');
```

`math`의 값을 콘솔에서 `console.log()`로 출력해 확인해 봅시다:

```javascript
console.log(math);
```

출력 결과입니다. `multiply` 속성과 함수가 값인 객체입니다:

```javascript
{ multiply: [Function: multiply] }
```

이 함수에 `math.multiply()`를 호출하고 `6`와 `7` 숫자를 인수로 전달하면 출력은 `42`입니다. 함수가 성공적으로 실행되었고, 다른 파일에 있음에도 불구하고 가져오기 문이 제대로 작동한 것입니다.

```javascript
const math = require('./math');

console.log(math.multiply(6, 7)); // 42
```

하지만 함수, 변수, 객체를 내보내는 다른 방법도 있습니다.

예를 들어, 단일 함수, 변수, 객체만 내보내야 한다면 `module.exports`의 값으로 직접 할당할 수 있습니다:

```javascript
function multiply(a, b) {
  return a * b;
}

module.exports = multiply;
```

여러 함수를 내보내야 한다면, 쉼표로 구분하여 `module.exports` 객체에 추가하면 됩니다.

여기서는 내보낸 객체가 `subtract`와 `multiply` 두 속성을 가지고 있습니다. 이들의 값은 각각 대응하는 함수에 대한 참조이므로, `math.js`을 가져오는 어떤 모듈에서도 두 함수 모두에 접근할 수 있습니다:

```javascript
function subtract(a, b) {
  return a - b;
}

function multiply(a, b) {
  return a * b;
}

module.exports = {
  subtract: subtract,
  multiply: multiply
};
```

`module.exports` 객체에 여러 속성이 있을 때는 모든 속성을 한꺼번에 가져오거나 개별적으로 가져올지 선택할 수 있습니다.

이 예제에서는 `math.js`이 내보낸 전체 객체를 가져오므로 `subtract`와 `multiply` 모두에 접근할 수 있습니다:

```javascript
const math = require('./math');

console.log(math.subtract(6, 7));  // -1
console.log(math.multiply(6, 7));  // 42
```

만약 `multiply`만 가져오고 싶다면, `const` 키워드 뒤 중괄호 안에 이름을 적으면 됩니다. 기본적으로 구조 분해 할당 구문을 사용해 내보낸 객체에서 값을 가져오는 것입니다. 그 후 코드에서 이름으로 직접 참조할 수 있습니다:

```javascript
const { multiply } = require('./math');

console.log(multiply(6, 7));
```

이 방법은 여러 요소에도 적용됩니다. 중괄호 안에서 쉼표로 구분하면 됩니다. 여기서는 `subtract`와 `multiply` 함수를 모두 가져옵니다:

```javascript
const { subtract, multiply } = require('./math');
```

기본적으로 Node.js는 다음을 CommonJS 모듈로 처리합니다:

* `.cjs` 확장자를 가진 파일
    
* 가장 가까운 상위 `.js` 파일에 최상위 필드 `package.json`가 `"type"` 값으로 포함된 경우 `"commonjs"` 확장자를 가진 파일
    
* 가장 가까운 상위 `.js` 파일에 최상위 필드 `package.json`가 없거나 상위 폴더에 `"type"`가 없을 때 `package.json` 확장자 또는 확장자가 없는 파일
    
* `.mjs`, `.cjs`, `.json`, `.node`, `.js`가 아닌 확장자를 가진 파일
    

ES 모듈이 점차 CommonJS 모듈을 대체하고 있지만, 기존 Node.js 코드베이스와 많은 npm 패키지를 다루는 개발자에게는 여전히 CommonJS를 이해하는 것이 중요합니다.
