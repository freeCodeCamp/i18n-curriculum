# --description--

Preprocessor ya CSS ni chombo kinachoongeza vipengele vyenye nguvu kama vile vigezo, mixin, upangaji ndani, na urithi wa kichaguzi kwenye CSS ya kawaida.

Baadhi ya vipengele hivi, kama vigezo na upangaji ndani, sasa vinaungwa mkono au vinaongezewa msaada katika CSS ya asili, na mwelekeo huu huenda ukaendelea. Hata hivyo, matoleo ya awali ya CSS ya asili hayakuunga mkono vipengele hivi.

Ndiyo maana preprocessors za CSS zilianza kutumika sana hapo awali. Zinachukua kanuni na mitindo ya CSS unayoandika, pamoja na sintaksia yake iliyoongezwa, na kuzibadilisha kuwa faili ya CSS ya asili ambayo vivinjari vinaweza kuelewa.

Kwa kutumia preprocessor ya CSS, unaweza kupanga msimbo wako wa CSS kwa njia inayoweza kutumika tena na yenye mantiki zaidi. Pia utakuwa na upatikanaji wa vipengele vyenye nguvu, kama mixin, ambavyo haviji moja kwa moja na CSS.

Baadhi ya preprocessors maarufu za CSS ni Sass, Less, na Stylus.

Tuzungumzie Sass.

Sass inasimama kwa "Syntactically Awesome Style Sheets." Inalingana na matoleo yote ya CSS na inadumishwa na jamii kubwa ya wasanidi programu.

Sass inaunga mkono vipengele kama:

- Vigezo, ili uweze kuhifadhi na kutumia tena thamani katika karatasi nzima ya mtindo.

- Kanuni za CSS zilizopangwa ndani, ili uweze kuunda muundo wa kuona katika faili yako.

- Moduli, ili uweze kugawanya mitindo yako katika mafaili mengi ya mtindo.

- Mixins, ili uweze kutumia tena tamko za CSS katika tovuti yako yote.

- Urithi, ili wachaguzi wengi wa CSS waweze kushiriki mali.

- Na waendeshaji, kwa shughuli za msingi za kihesabu.

Sass pia inaunga mkono sintaksia mbili.

Sintaksia ya SCSS inasimama kwa Sassy CSS, ambayo ni toleo lililopanuliwa la CSS. Hii inamaanisha kuwa SCSS inaongeza sintaksia ya msingi ya CSS.

SCSS ndiyo sintaksia inayotumika zaidi utakayokutana nayo unapotumia Sass. Inahitaji matumizi ya mabano ya mviringo (`{` na `}`) kuzunguka mali za CSS na alama za nukta-koma (`;`) mwishoni mwa tamko za CSS, kama vile CSS ya asili.

Mafaili haya yana nyongeza ya `.scss`. Hapa kuna mfano:

```scss
$primary-color: #3498eb;

header {
  background-color: $primary-color;
}
```

Unachoona hapa juu ni kigezo kilichofafanuliwa katika SCSS. Kigezo hiki kinatumika katika kanuni ya CSS hapa chini.

Tufananishe SCSS na sintaksia inayotumika kidogo, sintaksia iliyopangwa kwa alama za nafasi. Hii pia inajulikana kama "sintaksia ya Sass" kwa kuwa ilikuwa sintaksia ya awali ya Sass.

Sintaksia hii hutegemea alama za nafasi kufafanua kanuni. Hapa kuna mfano:

```scss
$primary-color: #3498eb

header
  background-color: $primary-color
```

Angalia kuwa hakuna mabano ya mviringo kuzunguka kanuni ya CSS, wala nukta-koma mwishoni mwa tamko za CSS. Juu, pia unaweza kuona kigezo. Angalia kuwa jina lake linaanza na alama ya dola (`$`) katika sintaksia hii, ambayo ni tofauti na vigezo vya CSS ya asili, lakini hufanya kazi kwa njia inayofanana.

Sass pia ina kipengele chenye nguvu kinachoitwa mixins.

Mixins hukuruhusu kuunganisha mali nyingi za CSS na thamani zao chini ya jina na kutumia tena sehemu hiyo ya msimbo wa CSS katika karatasi yako ya mtindo.

Hii inafanya msimbo wako wa CSS usirudiwe mara nyingi na uwe rahisi kudumishwa kwa sababu, ukibadilisha kitu katika sehemu hiyo, mabadiliko yanatumika kila mahali unapotumia mixin.

Kwa kuwa mixins zinaweza kuwa na majina maalum na ya kueleweka, zinaweza kusaidia kuelewa kile kila sehemu ya msimbo wa CSS inachofanya, jambo linaloweza kufanya karatasi zako za mtindo ziwe rahisi kueleweka.

Na mixins pia huongeza uthabiti katika karatasi za mtindo kwa kutumia mitindo ya kawaida.

Huu ni mfano wa mixin katika sintaksia ya SCSS:

```scss
@mixin center-flex {
  display: flex;
  justify-content: center;
  align-items: center;
}
```

Ili kufafanua mixin, unaanza kwa kuandika tamko la `@mixin` la at-rule, ikifuatiwa na jina la mixin. Katika kesi hii, mixin inaitwa `center-flex`. Ina mali tatu za CSS za kuweka vitu katikati kwa kutumia Flexbox.

Kisha, mara tu unapokuwa na mixin yako imefafanuliwa, utatumia tamko la `@include` la at-rule kuingiza mali hizo katika kanuni ya CSS. Unahitaji tu kuandika `@include` ikifuatiwa na jina la mixin. Katika kesi hii, jina ni `center-flex`:

```scss
section {
  @include center-flex;
  height: 500px;
  background-color: #3289a8;
}
```

Huu ni msimbo kamili wenye mixin na kanuni ya CSS:

```scss
@mixin center-flex {
  display: flex;
  justify-content: center;
  align-items: center;
}

section {
  @include center-flex;
  height: 500px;
  background-color: #3289a8;
}
```

Unaweza kuingiza mixin katika kanuni za CSS kadri unavyohitaji. Ikiwa unahitaji kufanya mabadiliko yoyote, unahitaji kubadilisha mixin yenyewe tu na mabadiliko yatafanyika kila mahali moja kwa moja. Mfano huu uko katika sintaksia ya SCSS. Angalia kuwa una mabano ya mviringo na nukta-koma.

Huu ni sawa katika sintaksia iliyopangwa kwa alama za nafasi, inayojulikana pia kama sintaksia ya Sass, bila mabano ya mviringo au nukta-koma:

```scss
@mixin center-flex
  display: flex
  justify-content: center
  align-items: center

section
  @include center-flex
  height: 500px
  background-color: #3289a8
```

Na huu ni msimbo wa CSS uliotafsiriwa, ambao kivinjari kitatafsiri baada ya faili kubadilishwa kuwa CSS ya asili:

```css
section {
  display: flex;
  justify-content: center;
  align-items: center;
  height: 500px;
  background-color: #3289a8;
}
```

Angalia kuwa unajumuisha mali tatu za mixin juu: `display`, `justify-content`, na `align-items`.

Huu ni mfano rahisi tu wa kile preprocessors za CSS zinaweza kufanya. Zina vipengele vingi vyenye nguvu vinavyostahili kujifunza na kutumia katika mtiririko wako wa kazi wa kila siku.

Hata hivyo, zina hasara zinazoweza kuwepo ambazo unapaswa kuzijua.

Kwanza, preprocessors za CSS lazima zibadilishe kanuni za CSS kuwa CSS ya kawaida. Ingawa mzigo wa ziada wa mchakato huu wa kubadilisha mara nyingi ni mdogo kwa muda na rasilimali, ni jambo unalopaswa kuzingatia kwa miradi halisi.

Pia, zinaweza kuleta changamoto za kurekebisha makosa, kwa kuwa vivinjari hutumia CSS iliyotafsiriwa moja kwa moja. Kugundua ni nini kinachosababisha mtindo wenye tatizo kutoka kwa sintaksia iliyopanuliwa kunaweza kuchukua hatua za ziada ikilinganishwa na CSS ya kawaida.

Hata hivyo, faida za preprocessors za CSS kwa kawaida huwa kubwa kuliko hasara zao, hasa kwa miradi tata. Zinaweza kusaidia sana kuandika CSS safi, inayoweza kutumika tena, isiyorudiwa mara nyingi, na inayoweza kupanuka.
