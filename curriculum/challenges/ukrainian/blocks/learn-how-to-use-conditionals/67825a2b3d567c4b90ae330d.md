# --explanation--

Марія пропонує планувати створення запасних копій `at the end of each day`. Вона чітко вказує цю частоту як надійний підхід, щоб гарантувати, що вся робота за день буде надійно збережена. Цей щоденний графік забезпечує послідовність і зменшує ризик втрати даних.

Зверніть увагу на її формулювання, яке відображає практичне рішення, спрямоване на баланс між автоматизацією та існуючим потоком роботи команди.
