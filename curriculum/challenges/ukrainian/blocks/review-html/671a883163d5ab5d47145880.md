## Основи HTML

- **Роль HTML**: HTML (Hypertext Markup Language) є основою структури вебсторінки, визначаючи елементи сторінки.
- **Елементи HTML**: Використовуються для представлення вмісту на сторінці. Більшість з них складаються з відкриваючого та кінцевого тегів (наприклад, `<h1></h1>`, `<p></p>`).
- **Структура HTML**: HTML складається з `head` та `body`, де структуровані метадані, стилі та вміст.
- **Загальні елементи HTML**: Заголовки (`<h1>` - `<h6>`), абзаци (`<p>`) та контейнери div (`<div>`).
- **Елементи `div`**: Елемент `div` — це загальний HTML-елемент, який не має семантичного значення. Використовується як загальний контейнер для інших HTML-елементів.
- **Порожні елементи**: Не мають кінцевого тегу (наприклад, `<img>`).
- **Атрибути**: Додають метадані та поведінку елементам.

## Ідентифікатори та групування

- **Ідентифікатори (ID)**: Унікальні ідентифікатори елементів.
- **Класи**: Групування елементів для стилізації та поведінки.

## Спеціальні символи та посилання

- **HTML-сутності**: Використання спеціальних символів, таких як `&amp;` та `&lt;`.
- **Елемент `link`**: Підключення зовнішніх таблиць стилів.
- **Елемент `script`**: Вставлення зовнішніх JavaScript-файлів.

## Шаблонний код і кодування

- **Шаблонний код HTML**: Базова структура вебсторінки, що включає елементи `DOCTYPE`, `html`, `head` та `body`. Його слід використовувати як відправну точку для HTML-документа.
- **Кодування символів UTF-8**: Забезпечення універсального відображення символів.

## SEO та соціальний обмін

- **Мета-теги (`description`)**: Надають короткий опис вебсторінки та впливають на SEO.
- **Теги Open Graph**: Покращують обмін у соціальних мережах.

## Медіаелементи та оптимізація

- **Замінені елементи**: Вбудований вміст (наприклад, зображення, iframe).
- **Оптимізація медіа**: Техніки покращення продуктивності медіа.
- **Формати зображень і ліцензії**: Розуміння прав використання та типів.
- **SVG**: Масштабовані векторні графіки для чітких зображень.

## Інтеграція мультимедіа

- **Елементи аудіо та відео HTML**: Вставлення мультимедіа.
- **Вставлення за допомогою `<iframe>`**: Інтеграція зовнішнього відеоконтенту.

## Шляхи та поведінка посилань

- **Типи атрибутів target**: Керування поведінкою посилань.
- **Абсолютні та відносні шляхи**: Навігація по каталогах.
- **Синтаксис шляху**: Розуміння `/`, `./`, `../` для навігації файлами.
- **Стани посилань**: Керування різними взаємодіями з посиланнями (hover, active).
- **Внутрішні посилання**: Посилання на інший розділ сторінки за допомогою `href="#id"` на елементі `a` та призначення цільовому елементу того самого `id`.

## Важливість семантичного HTML

- **Структурна ієрархія для заголовків**: Важливо використовувати правильний елемент заголовка, щоб підтримувати структурну ієрархію вмісту. Елемент `h1` є найвищим рівнем заголовка, а `h6` — найнижчим.
- **Презентаційні HTML-елементи**: Елементи, що визначають вигляд вмісту, наприклад застарілі `center`, `big` та `font`.
- **Семантичні HTML-елементи**: Ці елементи надають значення структурі вмісту. Приклади:
  - `<header>`: Представляє вступний вміст.
  - `<nav>`: Містить навігаційні посилання.
  - `<article>`: Представляє автономний вміст.
  - `<aside>`: Використовується для бічних панелей або пов’язаного вмісту.
  - `<section>`: Групує пов’язаний вміст у документі.
  - `<footer>`: Визначає нижній колонтитул для розділу або документа.

## Семантичні HTML-елементи

- **Елемент виділення (`em`)**: Позначає текст із наголосом.
- **Елемент ідіоматичного тексту (`i`)**: Використовується для виділення альтернативного голосу або настрою, ідіоматичних термінів іншою мовою, технічних термінів і думок.
- **Елемент сильної важливості (`strong`)**: Позначає текст із сильною важливістю.
- **Елемент привернення уваги (`b`)**: Використовується для привернення уваги до тексту, який не є важливим для значення вмісту.
- **Елемент списку описів (`dl`)**: Використовується для представлення списку пар термін-опис.
- **Елемент терміна опису (`dt`)**: Використовується для представлення визначеного терміна.
- **Елемент деталей опису (`dd`)**: Використовується для представлення опису терміна.
- **Елемент блочного цитування (`blockquote`)**: Використовується для представлення розділу, процитованого з іншого джерела.
- **Елемент рядкового цитування (`q`)**: Використовується для представлення короткої рядкової цитати.
- **Елемент абревіатури (`abbr`)**: Використовується для представлення абревіатури або акроніма.
- **Елемент контактної адреси (`address`)**: Використовується для представлення контактної інформації.
- **Елемент дати та часу (`time`)**: Використовується для представлення дати та/або часу.
- **Елемент верхнього індексу (`sup`)**: Використовується для представлення тексту у верхньому індексі.
- **Елемент нижнього індексу (`sub`)**: Використовується для представлення тексту у нижньому індексі.
- **Елемент рядкового коду (`code`)**: Використовується для представлення фрагмента комп’ютерного коду.
- **Елемент неартикульованої анотації (`u`)**: Використовується для представлення ділянки рядкового тексту, який має бути відтворений так, щоб вказувати на не текстову анотацію.
- **Елемент рубі-анотації (`ruby`)**: Використовується для представлення тексту рубі-анотації.
- **Елемент закреслення (`s`)**: Використовується для представлення вмісту, який більше не є точним або актуальним.

## Елементи та атрибути форм HTML

### Форми

- **Елемент `form`**: Використовується для створення HTML-форми для введення користувачем.
- **Загальні типи вхідних даних**: 
  - `text`, `email`, `password`, `radio`, `checkbox`, `number`, `date`.
- **Атрибут `action`**: використовується для вказівки URL, куди слід відправляти дані форми.
- **Атрибут `method`**: використовується для вказівки HTTP-методу при відправленні даних форми. Найпоширеніші методи — `GET` та `POST`.

```html
<form method="value-goes-here" action="url-goes-here">
  <!-- inputs go inside here -->
</form>
```

- **Елемент `input`**: використовується для створення поля введення для користувача.
- **Атрибут `type`**: використовується для вказівки типу поля введення, наприклад `text`, `email`, `number`, `radio`, `checkbox`.
- **Атрибут `placeholder`**: використовується для показу підказки користувачу, що вводити у поле введення.
- **Атрибут `value`**: використовується для вказівки значення поля введення. Якщо поле має тип `button`, атрибут `value` можна використати для встановлення тексту кнопки.
- **Атрибут `name`**: використовується для призначення імені полю введення, яке слугує ключем при відправленні даних форми. Для радіокнопок, якщо їм призначити однаковий `name`, вони групуються, і можна вибрати лише одну опцію з групи.
- **Атрибут `size`**: використовується для визначення кількості символів, які мають бути видимі під час введення.
- **Атрибут `min`**: можна використовувати з типами введення, такими як `number`, щоб вказати мінімальне допустиме значення.
- **Атрибут `max`**: можна використовувати з типами введення, такими як `number`, щоб вказати максимальне допустиме значення.
- **Атрибут `minlength`**: використовується для вказівки мінімальної кількості символів, необхідних у полі введення.
- **Атрибут `maxlength`**: використовується для вказівки максимальної кількості символів, дозволених у полі введення.
- **Атрибут `required`**: використовується для вказівки, що поле введення має бути заповнене перед відправленням форми.
- **Атрибут `disabled`**: використовується для вказівки, що поле введення має бути вимкнене.
- **Атрибут `readonly`**: використовується для вказівки, що поле введення є лише для читання.

:::interactive_editor

```html
<!-- Text input -->
<input 
  type="text"
  id="name"
  name="name"
  placeholder="e.g. Quincy Larson"
  size="20"
  minlength="5"
  maxlength="30"
  required
/>

<!-- Number input -->
<input 
  type="number"
  id="quantity"
  name="quantity"
  min="2"
  max="10"
  disabled
/>

<!-- Button -->
<input type="button" value="Show Alert" />
```

:::

- **Елемент `label`**: використовується для створення мітки для поля введення.
- **Атрибут `for`**: використовується для вказівки, для якого поля введення призначена мітка.
- **Неявне зв’язування форми**: поля введення можна пов’язати з мітками, обгорнувши поле введення всередині елемента `label`.

:::interactive_editor

```html
<form action="">
  <label>
    Full Name:
    <input type="text" />
  </label>
</form>
```

:::

- **Явне зв’язування форми**: поля введення можна пов’язати з мітками, використовуючи атрибут `for` на елементі `label`.

:::interactive_editor

```html
<form action="">
  <label for="email">Email Address: </label>
  <input type="email" id="email" />
</form>
```

:::

- **Елемент `button`**: використовується для створення активної кнопки. Кнопка також може мати атрибут `type`, який керує поведінкою кнопки при її активації, наприклад `submit`, `reset`, `button`.

:::interactive_editor

```html
<button type="button">Show Form</button>
<button type="submit">Submit Form</button>
<button type="reset">Reset Form</button>
```

:::

- **Елемент `fieldset`**: використовується для групування пов’язаних полів введення.
- **Елемент `legend`**: використовується для додавання підпису, що описує групу полів введення.

:::interactive_editor

```html
<!-- Radio group -->
<fieldset>
  <legend>Was this your first time at our hotel?</legend>

  <input id="yes-option" type="radio" name="hotel-stay" value="yes" />
  <label for="yes-option">Yes</label>

  <input id="no-option" type="radio" name="hotel-stay" value="no" />
  <label for="no-option">No</label>
</fieldset>

<!-- Checkbox group -->
<fieldset>
  <legend>
    Why did you choose to stay at our hotel? (Check all that apply)
  </legend>

  <input type="checkbox" id="location" name="location" value="location" />
  <label for="location">Location</label>

  <input type="checkbox" id="price" name="price" value="price" />
  <label for="price">Price</label>
</fieldset>
```

:::

- **Стан фокусу**: це стан поля введення, коли воно вибране користувачем.

## Робота з елементами та атрибутами таблиць HTML

- **Елемент таблиці**: використовується для створення HTML-таблиці.
- **Елемент заголовка таблиці (`thead`)**: використовується для групування заголовкового вмісту в HTML-таблиці.
- **Елемент рядка таблиці (`tr`)**: використовується для створення рядка в HTML-таблиці.
- **Елемент заголовної клітинки таблиці (`th`)**: використовується для створення заголовної клітинки в HTML-таблиці.
- **Елемент тіла таблиці (`tbody`)**: використовується для групування основного вмісту в HTML-таблиці.
- **Елемент клітинки даних таблиці (`td`)**: використовується для створення клітинки з даними в HTML-таблиці.
- **Елемент нижнього колонтитула таблиці (`tfoot`)**: використовується для групування нижнього колонтитула в HTML-таблиці.
- **Елемент `caption`**: використовується для додавання заголовка HTML-таблиці.
- **Атрибут `colspan`**: використовується для вказівки кількості стовпців, які має охоплювати клітинка таблиці.

:::interactive_editor

```html
<table>
  <caption>Exam Grades</caption>

  <thead>
    <tr>
      <th>Last Name</th>
      <th>First Name</th>
      <th>Grade</th>
    </tr>
  </thead>

  <tbody>
    <tr>
      <td>Davis</td>
      <td>Alex</td>
      <td>54</td>
    </tr>

    <tr>
      <td>Doe</td>
      <td>Samantha</td>
      <td>92</td>
    </tr>

    <tr>
      <td>Rodriguez</td>
      <td>Marcus</td>
      <td>88</td>
    </tr>
  </tbody>

  <tfoot>
    <tr>
      <td colspan="2">Average Grade</td>
      <td>78</td>
    </tr>
  </tfoot>
</table>
```

:::

## Інструменти HTML

- **Валідатор HTML**: інструмент, який перевіряє синтаксис HTML-коду, щоб переконатися, що він є дійсним.
- **Інспектор DOM**: інструмент, який дозволяє переглядати та змінювати структуру HTML вебсторінки.
- **Інструменти розробника**: набір інструментів веброзробника, вбудованих безпосередньо в браузер, які допомагають налагоджувати, профілювати та аналізувати вебсторінки.

## Вступ до доступності

- **Керівництво з доступності вебконтенту**: Керівництво з доступності вебконтенту (WCAG) — це набір принципів для підвищення доступності вебконтенту для людей з інвалідністю. Чотири принципи WCAG — POUR, що означає сприйнятливий, керований, зрозумілий і надійний.

## Допоміжні технології для доступності

- **Читачі екрана**: Програмне забезпечення, яке озвучує вміст екрану комп’ютера. Використовуються людьми з вадами зору для доступу до вебу.
- **Клавіатури з великим шрифтом або Брайлем**: Використовуються людьми з порушеннями зору для доступу до вебу.
- **Збільшувачі екрану**: Програми, які збільшують вміст екрану комп’ютера. Використовуються людьми з низьким зором для доступу до вебу.
- **Альтернативні пристрої вказівки**: Використовуються людьми з порушеннями рухливості для доступу до вебу. Включають пристрої, такі як джойстики, трекболи та тачпади.
- **Розпізнавання голосу**: Використовується людьми з порушеннями рухливості для керування комп’ютером за допомогою голосу.

## Інструменти аудиту доступності

- **Загальні інструменти доступності**: Google Lighthouse, Wave, IBM Equal Accessibility Checker та axe DevTools — це деякі з поширених інструментів для аудиту доступності вебсайтів.

## Найкращі практики доступності

- **Правильна структура рівнів заголовків**: Використовуйте правильні рівні заголовків для створення логічної структури вмісту. Це допомагає допоміжним технологіям розуміти вміст вашого сайту.
- **Доступність і таблиці**: При використанні таблиць слід використовувати елемент `th` для визначення заголовних клітинок і `td` для визначення клітинок з даними. Це допомагає допоміжним технологіям розуміти структуру таблиці.
- **Важливість наявності мітки для полів введення**: Використовуйте елемент `label` для зв’язування міток із полями форми. Це допомагає допоміжним технологіям розуміти призначення поля введення.
- **Важливість якісного альтернативного тексту**: Використовуйте атрибут `alt` для надання текстової альтернативи зображенням. Це допомагає допоміжним технологіям розуміти вміст зображення.
- **Важливість якісного тексту посилань**: Використовуйте описовий текст посилань, щоб допомогти користувачам зрозуміти призначення посилання.
- **Найкращі практики для доступності аудіо- та відеоконтенту**: Надавайте підписи та транскрипти для аудіо- та відеоконтенту, щоб зробити його доступним для людей з порушеннями слуху. Також надавайте аудіоописи для відеоконтенту, щоб зробити його доступним для людей з порушеннями зору.
- **Атрибут `tabindex`**: Використовується для того, щоб зробити елементи фокусованими та визначити відносний порядок навігації за допомогою клавіатури. Важливо ніколи не використовувати атрибут `tabindex` зі значенням більше 0. Замість цього слід використовувати значення 0 або -1.
- **Атрибут `accesskey`**: Використовується для визначення клавіатурного скорочення для елемента. Це може допомогти користувачам з порушеннями рухливості легше орієнтуватися на сайті.

## WAI-ARIA, ролі та атрибути

- **WAI-ARIA**: Це ініціатива Web Accessibility Initiative — Accessible Rich Internet Applications. Це набір атрибутів, які можна додати до HTML-елементів для покращення доступності. Вони надають допоміжним технологіям додаткову інформацію про призначення та структуру вмісту.
- **Ролі ARIA**: Набір заздалегідь визначених ролей, які можна додати до HTML-елементів для визначення їх призначення. Це допомагає допоміжним технологіям розуміти вміст сайту. Приклади включають `role="tab"`, `role="menu"` та `role="alert"`.
- **Атрибути `aria-label` та `aria-labelledby`**: Ці атрибути використовуються для надання елементу програмної (або доступної) назви, що допомагає допоміжним технологіям (наприклад, читачам екрана) розуміти призначення елемента. Вони часто використовуються, коли візуальна мітка елемента є зображенням або символом, а не текстом. `aria-label` дозволяє визначити назву безпосередньо в атрибуті, тоді як `aria-labelledby` дозволяє посилатися на існуючий текст на сторінці.
- **Атрибут `aria-hidden`**: Використовується для приховування елемента від допоміжних технологій, таких як читачі екрана. Наприклад, це можна використовувати для приховування декоративних зображень, які не містять значущого вмісту.
- **Атрибут `aria-describedby`**: Використовується для надання додаткової інформації про елемент шляхом пов’язання його з іншим елементом, що містить цю інформацію. Це допомагає допоміжним технологіям розуміти призначення елемента.
