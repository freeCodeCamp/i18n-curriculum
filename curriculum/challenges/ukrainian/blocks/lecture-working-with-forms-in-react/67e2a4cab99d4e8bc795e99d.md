# --description--

React 19 представив дві помітні нові функції: серверні компоненти та серверні дії.

Від цієї версії серверні компоненти стали за замовчуванням у фреймворках, таких як Next.js, які їх підтримують.

Серверні дії, з іншого боку, — це функції, які виконуються на сервері, щоб дозволити обробляти форми безпосередньо на сервері без потреби в API кінцевих точках.

Серверна дія виглядає так:

```js
"use server";

async function submitForm(formData) {
  const name = formData.get("name");
  return { message: `Hello, ${name}!` };
}
```

Ця серверна дія витягує поле `name` з форми і повертає рядок із привітанням цього імені.

Щоб спростити керування станом для серверних дій і позбавити від необхідності використовувати JavaScript на стороні клієнта для простих форм, команда React у версії 19 представила хук `useActionState`.

Давайте детальніше розглянемо цей хук і подивимося, як він працює.

Документація React описує хук `useActionState` як хук, який «дозволяє оновлювати стан на основі результату дії форми».

Але це не означає, що ви можете використовувати хук `useActionState` лише з формами. Ви також можете використовувати його для керування натисканнями кнопок та іншими подіями, якщо у вас є відповідна дія.

І пам’ятайте, що оскільки `useActionState` — це хук, ви не можете використовувати його всередині серверного компонента.

Ось базовий синтаксис хука `useActionState`:

```js
const [state, action, isPending] = useActionState(actionFunction, initialState, permalink);
```

- `state` — це поточний стан, який повертає дія.

- `action` — це функція, яка запускає серверну дію.

- `isPending` — булевий параметр, що вказує, чи виконується дія зараз.

- Параметр `actionFunction` — це сама серверна дія.

- `initialState` — параметр, який представляє початковий стан перед запуском дії.

- `permalink` — необов’язковий рядок, що містить унікальну URL-адресу сторінки, яку змінює форма.

Щоб використовувати хук `useActionState`, спочатку переконайтеся, що у вас є дія. Використаємо дію з попереднього прикладу, але з невеликим доповненням:

```js
"use server";

export async function submitForm(_, formData) {
  const name = formData.get("name");

  const hour = new Date().getHours();
  let greeting;

  if (hour < 12) {
    greeting = "Good morning";
  } else if (hour < 18) {
    greeting = "Good afternoon";
  } else {
    greeting = "Good evening";
  }

  return { message: `${greeting}, ${name}` };
}
```

У вашому компоненті потрібно імпортувати хук `useActionState` і викликати його на верхньому рівні тіла компонента (перед оператором return), як і інші хуки. Також імпортуйте дію:

```jsx
"use client";

// Import the useActionState hook
import { useActionState } from "react";

// Import the submitForm action
import { submitForm } from "./actions/submitForm";

const Greeter = () => {

 // Initialize the hook
 const [state, submit, isPending] = useActionState(submitForm, {
   message: "",
 });

  return (
    <div className="flex flex-col items-center justify-center min-h-screen bg-gray-100 p-6">
      {/* Rest of component */}
    </div>
  );
};

export default Greeter;
```

Ось як виглядає повний код із невеликим стилізуванням:

```jsx
"use client";

import { useActionState } from "react";
import { submitForm } from "./actions/submitForm";

const Greeter = () => {
  const [state, submit, isPending] = useActionState(submitForm, {
    message: "",
  });

  return (
    <div className="flex flex-col items-center justify-center min-h-screen bg-gray-100 p-6">
      <form
        action={submit}
        className="bg-white p-6 rounded-2xl shadow-md w-full max-w-md"
      >
        <h2 className="text-2xl text-center font-semibold text-gray-700 mb-4">
          Greet Someone
        </h2>

        <input
          type="text"
          name="name"
          placeholder="Enter your name"
          required
          className="w-full p-3 border border-gray-300 rounded-lg focus:outline-none focus:ring-2 focus:ring-green-400"
        />

        <button
          type="submit"
          disabled={isPending}
          className="w-full mt-4 p-3 bg-green-500 text-white font-semibold rounded-lg hover:bg-green-600 disabled:bg-gray-400 transition-all"
        >
          {isPending ? "Greeting..." : "Greet"}
        </button>

        {state.message && (
          <p className="mt-4 text-green-600 text-center font-medium">
            {state.message}
          </p>
        )}
      </form>
    </div>
  );
};

export default Greeter;
```

У браузері ви побачите, як кнопка форми змінюється з `Greet` на `Greeting...`, поки дія `isPending` виконується — а привітання відображатиметься як `Good morning, {name}`, `Good afternoon, {name}` або `Good evening, {name}` залежно від часу доби, коли форму було відправлено.

Пам’ятаєте, ми згадували, що хук `useActionState` можна використовувати не лише у формі?

У цьому прикладі ми отримаємо п’ять користувачів із JSONPlaceholder за натисканням кнопки:

```js
"use server";

export async function getUsers() {
  const res = await fetch(
    "https://jsonplaceholder.typicode.com/users?_start=0&_limit=5/"
  );
  return await res.json();
}
```

Ось стилізований інтерфейс:

```jsx
"use client";

import { useActionState } from "react";
import { getUsers } from "./actions/getUsers";

export default function FetchUsers() {
  const [users, fetchAction, isPending] = useActionState(getUsers, []);

  return (
    <div className="p-6 max-w-lg mx-auto">
      <button
        onClick={fetchAction}
        disabled={isPending}
        className="px-4 py-2 cursor-pointer bg-green-500 text-white rounded-lg hover:bg-green-600 disabled:bg-gray-400 font-bold"
      >
        {isPending ? "Fetching Users..." : "Fetch Users"}
      </button>

      <ul className="mt-4 space-y-2">
        {users.map((user) => (
          <li key={user.id} className="p-3 bg-gray-100 rounded-lg">
            <p className="font-semibold">{user.name}</p>
            <p className="text-sm text-gray-600">{user.email}</p>
          </li>
        ))}
      </ul>
    </div>
  );
}
```

У браузері ви помітите, що текст кнопки ніколи не оновлюється на `Fetching Users...` після натискання.

Це відбувається тому, що React вважає отримання даних і відтворення пріоритетнішими за стан `isPending`, що блокує `isPending` і викликає помилку.

Щоб виправити цю проблему, потрібно обгорнути дію в `startTransition`:

```jsx
"use client";

// import startTransition from React
import { useActionState, startTransition } from "react";
import { getUsers } from "./actions/getUsers";

export default function FetchUsers() {
  const [users, fetchAction, isPending] = useActionState(getUsers, []);

  return (
    <div className="p-6 max-w-lg mx-auto">
      <button
        {/* wrap fetchAction in startTransition */}
        onClick={() => startTransition(() => fetchAction())}
        disabled={isPending}
        className="px-4 py-2 bg-green-500 font-bold cursor-pointer text-white rounded-lg hover:bg-green-600 disabled:bg-gray-400"
      >
        {isPending ? 'Fetching Users...' : 'Fetch Users'}
      </button>

      <ul className="mt-4 space-y-2">
        {users.map((user) => (
          <li key={user.id} className="p-3 bg-gray-100 rounded-lg">
            <p className="font-semibold">{user.name}</p>
            <p className="text-sm text-gray-600">{user.email}</p>
          </li>
        ))}
      </ul>
    </div>
  );
}
```

Якщо ви цікавитесь, що таке `startTransition`, це функція, яка повідомляє React, що оновлення стану має низький пріоритет і може бути перерване. Це дозволяє інтерфейсу залишатися адаптивним під час обробки асинхронних оновлень, таких як серверні дії.

Ось як використовувати хук `useActionState` всередині та поза формою.

Чому не можна використовувати хук `useActionState` всередині серверного компонента?

## --answers--

Тому що серверні компоненти не підтримують керування станом, через що `useActionState` несумісний.

### --feedback--

Хуки працюють лише в певному типі компонентів.

---

Тому що `useActionState` вимагає підключення до бази даних, яке доступне лише в клієнтських компонентах.

### --feedback--

---

Тому що `useActionState` — це React-хук, а хуки можна використовувати лише всередині клієнтських компонентів.

---

Тому що `useActionState` залежить від браузерних API, які недоступні на сервері.
