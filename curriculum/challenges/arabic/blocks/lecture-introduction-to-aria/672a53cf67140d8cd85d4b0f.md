# --interactive--

جعل المحتوى الثابت متاحًا يمكن أن يكون بسيطًا نسبيًا، لكن المحتوى الديناميكي قد يكون أكثر تحديًا. هنا يأتي دور WAI-ARIA.

لننظر إلى ما هو WAI-ARIA، غرضه، كيف يعمل، وبعض الأمثلة.

WAI-ARIA تعني مبادرة إمكانية الوصول إلى الويب - تطبيقات الإنترنت الغنية القابلة للوصول. إنها مواصفة تعزز إمكانية الوصول للمحتوى الديناميكي ومكوّنات واجهة المستخدم.

لاحظ أن WCAG وWAI-ARIA ليسا نفس الشيء. تقدم WCAG إرشادات عامة لإمكانية الوصول إلى الويب، بينما توفر WAI-ARIA قواعد محددة لجعل المحتوى الديناميكي والتفاعلي متاحًا لمستخدمي تقنيات المساعدة.

لذا، الغرض الأساسي من WAI-ARIA هو تحسين إمكانية الوصول للمحتوى الديناميكي ومكوّنات واجهة المستخدم التي لا تملك مكافئات أصلية في HTML.

يعمل WAI-ARIA عن طريق إدخال مجموعة من الخصائص التي يمكنك إضافتها إلى عناصر HTML لتوفير معلومات دلالية إضافية. تُصنف هذه الخصائص إلى أدوار، حالات، وخصائص.

دور ARIA يحدد غرض العنصر داخل موقع ويب أو تطبيق ويب. إليك مثالًا على تعيين الدور إلى `button` لعنصر `div`.

```html
<div role="button">Click Me</div>
```

عند القيام بذلك، تشير إلى تقنية مساعدة أن العنصر هو زر. مع ذلك، الأدوار لا تضيف أي وظيفة. مجرد إعطاء هذا العنصر من نوع `div` دور `role` بقيمة `button` لن يجعله يتصرف كزر. لجعله يبدو ويتصرف كزر، تحتاج إلى استخدام CSS وJavaScript لتحقيق النتيجة المطلوبة.

إليك مثالًا على استخدام HTML وCSS وJavaScript لإنشاء عنصر `button` مخصص.

**ملاحظة**: لا تقلق بشأن محاولة فهم شفرة CSS وJavaScript. ستتعلم هذه اللغات في الوحدات القادمة.

:::محرر_تفاعلي

```html
<link href="styles.css" rel="stylesheet">
<div id="custom-btn" role="button">Click Me</div>
<script src="index.js"></script>
```

```css
#custom-btn {
  display: inline-block;
  padding: 0.4em 1em;
  font-size: 1rem;
  font-family: sans-serif;
  color: buttontext;
  background-color: buttonface;
  border: 1px solid #ccc;
  border-radius: 4px;
  cursor: pointer;
  user-select: none;
  text-align: center;
}

#custom-btn:focus {
  outline: 2px solid Highlight;
  outline-offset: 2px;
}

#custom-btn:active {
  background-color: #ddd;
}
```

```js
const button = document.getElementById("custom-btn");

button.addEventListener("click", () => {
  alert("Button clicked!");
});

button.addEventListener("keydown", (e) => {
  if (e.key === "Enter" || e.key === " ") {
    e.preventDefault(); 
    button.click();
  }
});
```

:::

من الأفضل دائمًا استخدام العنصر الأصلي `button` أو `input` مع `type="button"` بدلاً من ذلك.

توفر خصائص ARIA تفاصيل إضافية عن العناصر. على سبيل المثال، خاصية `aria-labelledby` تتيح لك ربط عنصر بمسمى محدد:

```html
<h2 id="header-id">About freeCodeCamp</h2>
<button id="button-id" aria-labelledby="header-id button-id">Learn More</button>
```

هذا سيجعل العناصر مفهومة وقابلة للتنقل لمستخدمي تقنيات المساعدة.

للحصول على أفضل استفادة من WAI-ARIA، حاول الالتزام بـ HTML الأصلي كلما أمكن لأنه يوفر عادةً إمكانية وصول أفضل بشكل افتراضي.

استخدم WAI-ARIA فقط عندما لا يكفي HTML، ولا تنس اختبار عملك مع تقنيات المساعدة مثل قارئ الشاشة، أو اجعل أشخاصًا من ذوي الإعاقة يختبرون عملك. تأكد أيضًا من تحديث حالات وخصائص WAI-ARIA مع المحتوى في الوقت الحقيقي. تجنب الإفراط في استخدام ARIA لأنه قد يسبب ارتباكًا.
