# --description--

Cookie 是网站保存在你设备上的小型文本文件，用来记住关于你的一些信息。

接收和储存 Cookie 的过程如下：

- 你访问一个网站
- 网站服务器在响应中发送一个 `Set-Cookie` 头部
- 你的浏览器接收该头部并将 Cookie 文件保存到你的计算机硬盘上
- 每次你再次访问该网站时，浏览器会自动将 Cookie 发送回服务器

这个过程让网站能够记住你的登录会话、购物车商品、偏好和设置以及其他重要信息。

这种储存方式带来了重大安全风险。如果黑客获得了你设备的物理访问权限，或者使用恶意软件读取你的本地文件，他们就能窃取你的会话 Cookie。

拥有被盗的会话 Cookie 后，攻击者可以在完全不同的计算机上登录你的账户，而无需知道你的密码。

为了保护用户并防止 Cookie 被盗，开发者在设置 Cookie 时会使用特定的安全标记：

`HttpOnly` 标记阻止 JavaScript 读取 Cookie 文件。这一限制防止攻击者通过跨站脚本攻击窃取你的会话令牌。

`Secure` 标记确保浏览器只通过加密的 HTTPS 连接传输 Cookie。这防止坏人通过公共 Wi-Fi 等场景嗅探 Cookie。

`SameSite` 标记限制浏览器在跨站请求时发送 Cookie，从而阻止跨站请求伪造攻击。它有两个常见取值：

- `SameSite=Strict` 阻止浏览器在任何跨站请求中发送 Cookie。
- `SameSite=Lax` 允许在安全的顶层导航（如点击链接）时发送 Cookie，但阻止跨站表单提交和 API 请求时发送。

Cookie 还带来了巨大的隐私影响。第一方 Cookie 只跟踪你在特定访问网站上的行为，而第三方 Cookie 则会跨不同网站跟踪你。

广告网络会在成千上万个热门网站上放置这些 Cookie。当你浏览网络时，广告网络会反复读取同一个第三方 Cookie，以构建你个人兴趣、搜索历史和在线习惯的详细档案。

这种隐形跟踪带来了巨大的隐私问题，因为它允许企业在你不知情的情况下监控你的数字生活。

鉴于这些威胁，像 GDPR 这样的隐私法律要求网站显示 Cookie 同意横幅，让你知道网站如何使用它设置的 Cookie。这些法律赋予你拒绝跟踪 Cookie 的合法权利。

此外，现代浏览器现在默认阻止第三方跟踪 Cookie，为用户提供自动化的隐私保护。
