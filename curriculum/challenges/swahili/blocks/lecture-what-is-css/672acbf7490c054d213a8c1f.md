# --interactive--

CSS inaweza kutumika kwenye ukurasa wa mtandao kwa njia kuu tatu: ndani ya mstari, ya ndani, au ya nje.

Kila njia ina matumizi yake, faida, na vikwazo vyake, na kujua ni lini utumie kila moja ni muhimu kwa kuandika msimbo safi, wenye ufanisi, na rahisi kudumishwa.

Tuchambue aina tatu za CSS na ni lini unapaswa kuzitumia.

CSS ya ndani ya mstari inaandikwa moja kwa moja ndani ya kipengele cha HTML kwa kutumia sifa ya kipengele `style`. Inatumika kuweka mitindo kwa kipengele maalum.

Hapa kuna mfano wa kutumia CSS ya ndani ya mstari:

:::interactive_editor

```html
<p style="color: green;">This is an inline-styled paragraph.</p>
```

:::

Katika mfano huu, tunatumia sifa ya kipengele `style` kuweka maandishi ya aya kuwa `green`.

CSS ya ndani ya mstari kwa kawaida hutumika kwa mitindo ya haraka, ya mara moja au kubadili mitindo mingine kwa kipengele maalum.

Hata hivyo, inapaswa kuepukwa katika hali nyingi kwa sababu inaweza kuchafua HTML na kufanya msimbo kuwa mgumu kudumishwa.

Mara nyingi, ni bora kutumia CSS ya ndani au ya nje ili kuweka mitindo yako imepangwa na iwe rahisi kudumishwa.

CSS ya ndani inaandikwa ndani ya lebo za `style` ndani ya sehemu ya `head` ya hati ya HTML. Inatumika kuweka mitindo kwa ukurasa mzima na ni muhimu unapohitaji kupamba hati moja tu.

Hapa kuna mfano wa CSS ya ndani:

:::interactive_editor

```html
<head>
  <style>
    p {
      color: blue;
    }
  </style>
</head>
<body>
  <p>This paragraph is styled using internal CSS.</p>
</body>
```

:::

Katika mfano hapo juu, CSS ya ndani inaweka maandishi ya buluu kwa vipengele vyote vya `p` kwenye ukurasa.

CSS ya ndani ni bora kutumika unapohitaji kuweka mitindo kwa ukurasa maalum badala ya kurusha kwenye kurasa nyingi. Ni muhimu kwa tovuti za ukurasa mmoja au wakati mitindo haipaswi kutumika tena mahali pengine.

Hata hivyo, kuna hasara fulani, kama vile kutosaidia matumizi tena kwenye kurasa nyingi. Zaidi ya hayo, kama CSS ya ndani ya mstari, inachanganya HTML na CSS, na kufanya msimbo kuwa mgumu kudumishwa katika miradi mikubwa.

CSS ya nje inaandikwa katika faili la `.css` tofauti na kuunganishwa na hati ya HTML kwa kutumia kipengele cha `link` katika sehemu ya `head`.

Inakuwezesha kupamba kurasa nyingi kwa mtindo unaoendelea na ni njia inayopendekezwa katika ukuzaji wa mtandao wa kitaalamu.

Hapa kuna mfano wa kupamba vipengele vyote vya aya:

:::interactive_editor

```html
<head>
  <link rel="stylesheet" href="styles.css">
</head>
<body>
  <p>This paragraph is styled using external CSS.</p>
</body>
```

```css
p {
  color: red;
}
```

:::

Katika funzo la awali, ulijifunza kuwa kipengele cha `link` kina sifa ya kipengele `rel` inayobainisha uhusiano kati ya hati ya sasa na rasilimali iliyounganishwa, kama vile kuunganisha na karatasi ya mitindo au rasilimali ya nje.

Kwa upande mwingine, sifa ya kipengele `href` inaonyesha URL ya rasilimali iliyounganishwa, ikibainisha mahali rasilimali inapaswa kupatikana.

CSS ya nje ni bora kwa miradi mikubwa ambapo unataka kudumisha mtindo unaoendelea kwenye kurasa nyingi.

Inakuza mgawanyiko wa majukumu kwa kuweka HTML kushughulikia muundo na CSS kushughulikia mitindo, jambo linalofanya msimbo kuwa rahisi kudumishwa na kuongezeka.

Kuelewa ni lini utumie kila aina ya CSS ni muhimu kwa ukuzaji wa mtandao wenye ufanisi na mzuri.

Katika hali nyingi, CSS ya nje inapaswa kuwa njia yako ya kawaida, hasa kwa miradi mikubwa na yenye changamoto zaidi.
