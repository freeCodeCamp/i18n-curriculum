# --description--

これまでのレッスンでは、`def` キーワードを使って次のように関数を定義してきました。

```py
def square(num):
    return num ** 2

print(square(4)) # 16
```

しかし、`map()` や `filter()` のような高階関数を扱う場合は、無名のインライン関数を使うことができます。ここでラムダ関数が登場します。

以下は、`square()` 関数をラムダ関数に書き換えた例です。

```py
lambda num: num ** 2
```

前述の通り、ラムダ関数は無名なので、この関数にはもはや `square` という名前はありません。ラムダ関数は、このように高階関数で使う場合にとても便利です。

```py
numbers = [1, 2, 3, 4, 5]

even_numbers = list(filter(lambda x: x % 2 == 0, numbers))
print(even_numbers)  # [2, 4]
```

この例では、数値のリストがあり、偶数だけの新しいリストを作りたいとします。そこで、ラムダ関数を `filter()` 関数の引数の一つとして渡し、`2` と `4` の数値を含む新しいリストを取得しています。

ラムダ関数を使う際にはベストプラクティスを意識することが重要です。例えば、ラムダ関数を次のように変数に代入するのは良くない使い方です。

```py
numbers = [1, 2, 3, 4, 5]

square = lambda x: x ** 2
squared_numbers = list(map(square, numbers))
print(squared_numbers) # [1, 4, 9, 16, 25]
```

これは無名関数を使う意味を損ないます。この場合は、次のように通常の関数を使うべきです。

```py
numbers = [1, 2, 3, 4, 5]

def square(num):
    return num ** 2

squared_numbers = list(map(square, numbers))
print(squared_numbers) # [1, 4, 9, 16, 25]
```

また、次のように読みづらく不必要に複雑なラムダ関数を作るのも避けてください。

```py
result = (lambda x: (x**2 + 2*x - 1) if x > 0 else (x**3 - x + 4))(3)
print(result)  # 14
```

この関数は正しく動作し、`14` という正しい結果を出しますが、読みづらく見づらいです。この場合は、`if/else` 文を使った別の関数を作成し、その関数を呼び出す方が良いでしょう。

```py
def calculate_expression(x):
    if x > 0:
        return x**2 + 2*x - 1
    else:
        return x**3 - x + 4

print(calculate_expression(3))  # 14
```

通常の関数とラムダ関数はどちらもPythonプログラムで使い分けられます。インラインの単一式を扱う場合はラムダ関数を検討し、それ以外は通常の関数を使うのが良いでしょう。
