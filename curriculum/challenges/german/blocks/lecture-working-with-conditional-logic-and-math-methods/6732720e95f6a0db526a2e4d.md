# --interactive--

Binäre logische Operatoren helfen Ihnen dabei, zwei Ausdrücke zu bewerten und ein Ergebnis basierend auf deren Wahrheitswert zurückzugeben. Schauen wir uns die drei gebräuchlichsten binären logischen Operatoren an: logisches UND, logisches ODER und den nullish coalescing Operator.

Der logische UND-Operator wird durch ein doppeltes kaufmännisches Und-Zeichen (`&&`) dargestellt. Er prüft, ob beide Operanden wahr sind, und gibt ein Ergebnis zurück. Wenn beide Operanden truthy sind, gibt er den zweiten Wert zurück, also den rechten:

:::interactive_editor

```js
const result = true && 'hello';

console.log(result); // hello
```

:::

Im obigen Beispiel wird der Text `hello` in der Konsole ausgegeben, weil beide Operanden `true` sind. Wenn einer der Operanden falsy ist, gibt er den falsy-Wert zurück:

:::interactive_editor

```js
const result = 0 && 3;

console.log(result); // 0
```

:::

Da `0` ein falsy-Wert ist, wird die Zahl `0` in der Konsole ausgegeben. Und wenn beide Operanden falsy sind, gibt er den ersten falsy-Wert zurück:

:::interactive_editor

```js
const result = false && 0;

console.log(result); // false
```

:::

Da `false` ein falsy-Wert ist, wird `false` in der Konsole ausgegeben. Der logische UND-Operator ist nützlich, wenn Sie mehrere Bedingungen prüfen und sicherstellen möchten, dass alle wahr sind, bevor Sie fortfahren. Hier ist ein Beispiel:

:::interactive_editor

```js
if (2 < 3 && 3 < 4) {
 console.log('The if block runs'); 
} else {
 console.log('The else block runs');
} 
```

:::

In der Bedingung wird, da `2` kleiner als `3` UND `3` kleiner als `4` ist, der Satz `The if block runs` in der Konsole ausgegeben.

Der logische ODER-Operator prüft, ob mindestens einer der Operanden truthy ist. Wenn der erste Operand truthy ist, gibt er diesen Wert zurück:

:::interactive_editor

```js
const result = 'This is truthy' || false;

console.log(result); // This is truthy
```

:::

Wenn der erste Operand falsy ist, aber der zweite truthy, wird der zweite Wert in der Konsole ausgegeben:

:::interactive_editor

```js
const result = 0 || 'This is truthy';

console.log(result); // This is truthy
```

:::

Es ist üblich, den logischen ODER-Operator in `if/else`-Anweisungen wie dieser zu verwenden:

:::interactive_editor

```js
let userInput;

if (userInput || 'Guest') {
 console.log('A user is present');
} else {
 console.log('No user detected');
}
```

:::

Da wir der Variablen `userInput` keinen Wert zugewiesen haben, ist sie aktuell `undefined`. Die Bedingung in der `if`-Anweisung prüft, ob entweder die Variable `userInput` oder der String `Guest` truthy sind. Da der String `Guest` in einem booleschen Kontext wie diesem wahr ist, wird der String `A user is present` in der Konsole ausgegeben.

Der nullish coalescing Operator ist ausgefeilter als logisches ODER und logisches UND. Er wird durch ein doppeltes Fragezeichen (`??`) dargestellt und hilft in Szenarien, in denen Sie einen Wert nur zurückgeben möchten, wenn der erste Wert `null` oder `undefined` ist. Hier ist ein Beispiel für die Arbeit mit dem nullish coalescing Operator:

:::interactive_editor

```ts
const result = null ?? 'default';

console.log(result); // default
```

:::

Da `null` ein nullish-Wert ist, würde der String `default` in der Konsole ausgegeben. Der nullish coalescing Operator ist äußerst nützlich in Situationen, in denen `null` oder `undefined` die einzigen Werte sind, die einen Fallback- oder Standardwert auslösen sollten. Hier ist ein Beispiel für den Umgang mit den Präferenzeinstellungen eines Benutzers:


:::interactive_editor

```ts
const userSettings = {
 theme: null,
 volume: 0,
 notifications: false,
};

let theme = userSettings.theme ?? 'light';
console.log(theme); // light
```

:::

Im obigen Beispiel haben wir ein Objekt namens `userSettings`, das die Eigenschaften `theme`, `volume` und `notifications` enthält. Wir greifen auf `theme` mit Punktnotation wie `userSettings.theme` zu. Sie werden in einer zukünftigen Lektion mehr darüber lernen, wie man mit Objekten arbeitet. Da die `theme` des Benutzers aktuell auf `null` gesetzt ist, wird der String `light` in der Konsole ausgegeben.
