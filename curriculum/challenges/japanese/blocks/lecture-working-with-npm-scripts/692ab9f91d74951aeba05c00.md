# --description--

これからnpmスクリプトについて学びます。

その前に、いくつか重要な概念を復習しましょう：

* **npm** はJavaScriptパッケージをインストールおよび管理するためのコマンドラインツールです。
    
* **パッケージ** は関連するコードを論理的にまとめたディレクトリです。
    
* **スクリプト** はスクリプト言語で書かれた一連の命令やコマンドのことです。
    

**npmスクリプト** は、アプリケーションの`package.json`ファイル内で定義できるカスタムコマンドです。

これらは、プロジェクトの開発、テスト、デプロイの各フェーズで繰り返し使うタスクやワークフローを自動化するのに役立ちます。

例えば、npmスクリプトの使い方の一例として、開発環境全体を起動するために`npm start`をカスタマイズすることが挙げられます。開発環境を起動するたびにすべてのコマンドを一つずつ手動で書く代わりに、`npm start`を実行するだけで全プロセスが実行されます。

また、コードベース全体のリンティングやテストスイートの実行など、さまざまな用途でnpmスクリプトを使って自動化できます。

非常に強力なので、基本を見ていきましょう。

## package.jsonファイルの作成方法

カスタムnpmスクリプトを作成・実行するには、`package.json`ファイルが必要です。

`package.json`ファイルはNode.jsプロジェクトに不可欠です。このファイルにはプロジェクトのメタデータ、依存関係、スクリプトが含まれています。

復習ですが、`package.json`ファイルを作成するには、プロジェクトのルートディレクトリで`npm init`コマンドを実行します。すると、パッケージ名、バージョン、説明などを手動で入力するか、空欄のままデフォルト値を使うことができます。

この手順を終えると、プロジェクトのルートディレクトリに`package.json`ファイルが作成されます。

すべてデフォルト値のままにしておくと、`package.json`をコードエディターで開いたときに以下のキーと値のペアが見られます：

```json
{
  "name": "<project_name>",
  "version": "1.0.0",
  "main": "index.js",
  "scripts": {
    "test": "echo \"Error: no test specified\" && exit 1"
  },
  "author": "",
  "license": "ISC",
  "description": ""
}
```

ここに注目してください：

```json
"scripts": {
  "test": "echo \"Error: no test specified\" && exit 1"
}
```

ここでnpmスクリプトを、`scripts`キーの下に指定します。

デフォルトで1つのスクリプト、testスクリプトが見られます：

```json
"test": "echo \"Error: no test specified\" && exit 1"
```

これは`package.json`に定義されたnpmスクリプトの例です。

npmスクリプトを定義する基本的な構文は以下の通りです：

```json
"<script_name>": "<commands>"
```

コロンの左側にはスクリプト名があります。よく使われる名前には`run`、`start`、`test`、`build`、`dev`、`lint`があります。これらの名前は右側のコマンドの「ショートカット」として機能します。

コロンの右側には実際のコマンドが書かれています。

例えば：

```json
"scripts": {
  "start": "node app.js"
}
```

スクリプトを実行するには、コマンドラインで以下を入力します：

```bash
npm run <script_name>
```

これで`package.json`に定義された任意のカスタムスクリプトを実行できます。

時には、`run`という単語を省略してこの短縮形でスクリプトを実行することもできます：

```bash
npm <script_name>
```

ただし、これは`start`、`test`、`stop`、`restart`という特定のnpmスクリプト名にのみ有効です。

例に戻ると、`package.json`に以下がある場合：

```json
"scripts": {
  "start": "node app.js"
}
```

npmスクリプトを実行するには、コマンドラインで`npm run start`を入力します。そうすると実際に実行されるコマンドは`node app.js`です。

このnpmスクリプトには短縮形があるため、`npm start`を使っても同じ結果になります。

この場合、単純なコマンド`node app.js`を`npm start`に置き換えているだけですが、複雑なコマンドのシーケンスを順番または並行して実行する必要がある場合に、`npm start`を使ってすべてを実行できるとしたら、かなりの時間を節約できますよね。これがnpmスクリプトの真の力です。

ここで、よく使われるnpmスクリプトの簡単な概要を示します：

* `npm run`：カスタムnpmスクリプトを実行するために使います。
    
* `npm start`：メインのアプリケーションや開発サーバーを起動するために使います。
    
* `npm lint`：コードスタイルチェッカーを実行し、コードがコードスタイルの基準やベストプラクティスに合っているかをチェックします。
    
* `npm test`：プロジェクトのテストスイートを実行します。
    
他にもさまざまな目的のnpmスクリプトがありますが、これらが最も一般的に見られるものです。

また、慣例として、コロンを使って関連するスクリプトを同じ名前空間にグループ化することがよくあります。例えば、`start:server`や`start:ui`などです。

## コマンドを順番に実行する方法

複数のコマンドを順番に実行する必要がある場合、2つの方法があります。論理AND演算子（`&&`）かセミコロン（`;`）を使う方法です。それぞれの違いを見てみましょう。

論理AND演算子は、最初のコマンドが成功した場合にのみ次のコマンドを実行します。

例えば：

```json
"scripts": {
  "build": "webpack --mode production",
  "start": "npm run build && node server.js"
}
```

`start`コマンドに論理AND演算子があることに注目してください：

```json
"start": "npm run build && node server.js"
```

ここでは、`npm start`がまず`npm run build`を実行しようとし、このスクリプトが成功した場合にのみ`node server.js`が実行されます。

これは非常に一般的な方法で、信頼性を確保し、前のコマンドが失敗した場合に後続のコマンドが失敗やエラーを起こして時間やリソースを無駄にするのを防ぎます。

一方、セミコロン演算子を使うと、前のコマンドが成功したかどうかに関係なくコマンドを順番に実行します。

例では、この単純な変更がコマンドの実行に重要な影響を与えます：

```json
"scripts": {
  "build": "webpack --mode production",
  "start": "npm run build; node server.js"
}
```

再び`start`コマンドに注目してください：

```json
"start": "npm run build; node server.js"
```

ここでは`&&`演算子の代わりにセミコロンを使っているため、`node server.js`コマンドは`npm run build`の後に必ず実行されます。たとえ`node server.js`が成功しなかったとしてもです。

```json
"start": "npm run build; node server.js"
```

## コマンドを並行して実行する方法

コマンドを並行（パラレル）で実行する必要がある場合、`&`演算子を使えます。

```json
"scripts": {
  "start:server": "node server/index.js",
  "start:client": "npm run build:watch",
  "dev": "npm run start:server & npm run start:client"
}
```

ただし、この方法はプロセスをバックグラウンドで実行することを意味します。

つまり、同じターミナルセッションで実行すると、複数の出力が混ざって表示され、理解しにくくなる場合があります。

コマンドを並行して実行する他の方法には、`concurrently`や`npm-run-all`のようなツールがあります。

## npmスクリプトに引数を渡す方法

npmスクリプト内で実行されるコマンドに直接引数を渡す必要がある場合、コマンドにハイフン2つ（`--`）を書き、その後に引数を指定します。

`--`は、npmに対してそれ以降のすべてがnpmスクリプトが実行する基のコマンドへの引数であることを伝えます。

一般的な構文は以下の通りです：

```bash
npm run <script_name> -- <argument_1> <argument_2> <argument_3>
```

例えば、次のような`npm start`スクリプトを定義したとします：

```json
{
  "name": "project-example",
  "scripts": {
    "start": "node app.js"
  }
}
```

`npm start`を使う場合、サーバーを実行したいポートをどう指定しますか？

コマンドライン引数として直接ポートを渡したい場合、次のように書きます：

```bash
npm start -- 8000
```

コード内では、`process.argv`配列のインデックスを使ってこれらの引数にアクセスできます：

```bash
process.argv[0] # The path to the node executable.
process.argv[1] # The script file being executed.
process.argv[2] # The first argument (8000).
```

したがって、`process.argv[2]`を使って必要な場所でポート番号を取得し利用できます。これがコマンドライン引数とnpmスクリプトを組み合わせた力です。

このように、npmスクリプトは開発ワークフローを簡素化するための必須ツールです。複雑で再利用可能なコマンドのシーケンスを、日常的に使えるシンプルで一貫したショートカットに自動化・変換するのに役立ちます。
