## クライアントサイドの JavaScript

- **クライアントサイドの JavaScript**：ウェブアプリケーションのクライアント側、通常はウェブブラウザ上で動作するコードです。
- **クライアントサイドの JavaScript の制限**：強力ですが、クライアントサイドの JavaScript にはいくつかの制限があります：
  - ローカルファイルへのアクセスが非常に制限されている。
  - 複雑なアプリケーションロジックの処理には設計されていない。
  - セキュリティ上の懸念がある（例：データベースの資格情報などの機密情報が露出する可能性）。
- **制限の理由**：これらの制限は、クライアントサイドの JavaScript がもともとウェブブラウザ上でのみ動作するように設計されたためです。

## ブラウザ環境

- **JavaScript エンジン**：ウェブブラウザは JavaScript コードを実行するための環境を提供し、その中には JavaScript エンジンも含まれます。
- **ドキュメントオブジェクトモデル（DOM）**：ブラウザは DOM へのアクセスを提供し、コード内で HTML 要素にアクセスできます。
- **歴史的背景**：以前は JavaScript はブラウザの外で動作できませんでしたが、現在は Node.js によって可能になっています。

## Node.js とは？

- **Node.js**：JavaScript のランタイム環境です。公式には「オープンソースでクロスプラットフォームの JavaScript ランタイム環境」と定義されています。
- **オープンソース**：Node.js のコードは公開されており、多くの開発者コミュニティによってメンテナンスされています。
- **クロスプラットフォーム**：Windows、macOS、Linux など、どのオペレーティングシステムでも動作します。
- **JavaScript ランタイム環境**：ブラウザの外で JavaScript コードを実行できる環境を提供します。

## Node.js の利用例

- **一般的な用途**：Node.js はさまざまな業界で以下の用途に使われています：
  - HTTP リクエストを処理するウェブサーバーや API の構築。
  - ウェブおよびモバイルアプリケーションの開発。
  - 複雑なデータやデータベースの操作の処理。
- **業界での採用**：データベースと連携したり複雑なデータを扱うウェブサイトやウェブアプリケーションを使ったり開発したりする場合、Node.js が裏側で使われている可能性が高いです。
- **主な特徴**：Node.js は効率的でスケーラブル、世界中に多くの開発者とメンテナがいます。

## ブラウザ環境と Node ランタイム環境の違い

- **ブラウザ環境**：主にフロントエンドのウェブ開発向けで、クライアントサイドの JavaScript を実行します。
- **Node ランタイム環境**：主にバックエンドのウェブ開発向けで、サーバーサイドの JavaScript を実行します。
- **API の違い**：環境によって利用できる API が異なります：
  - **ブラウザ**：DOM API へのアクセスが可能ですが、ローカルファイルシステムへのアクセスは制限されています。
  - **Node**：ファイルシステムを含むほぼすべてのシステムリソースにアクセスできますが、DOM にはアクセスできません。

## グローバルオブジェクト

- **ブラウザのグローバルオブジェクト (`window`)**：ブラウザ関連の機能にアクセスできます：
  - DOM を操作するメソッド。
  - クッキーの管理。
  - ブラウザのイベント処理。

- **Node.js のグローバルオブジェクト (`global`)**：Node.js 固有の機能にアクセスできます：
  - ローカルファイル操作用の組み込みモジュール。
  - ネットワーク機能。
  - オペレーティングシステムとの連携。

## バージョン管理

- **ブラウザのバージョン**：ユーザーがウェブサイトを訪れる際に使うブラウザ環境のバージョンは制御できません。
- **Node.js のバージョン**：サーバーで使う Node.js のバージョンは選択できます。

## Node.js の重要性

- **フルスタック開発**：Node.js により、開発者は JavaScript を使ってフロントエンドとバックエンドの両方を構築できるようになり、ウェブ開発が変革されました。
- **効率性**：バックエンド開発のために新しいプログラミング言語を学ぶ必要がなくなり、開発プロセスが効率的になります。
- **影響力**：Node.js は強力でスケーラブル、多用途なツールとして世界中の開発者にとって非常に重要です。

## Node.js の利点

- **フルスタックで単一言語**：ウェブアプリケーションのフロントエンドとバックエンドの両方を JavaScript で実装できます。
  - 初期の学習コストが減ります。
  - 言語を切り替える必要がありません。
  - 効率と生産性が向上します。

- **ノンブロッキングでイベント駆動型のアーキテクチャ**：レスポンスの良さと効率が重要なリアルタイムアプリケーションの開発に最適です。

- **シングルスレッドとイベントループ**：多数の同時リクエストや入出力操作を効果的に処理できます。
  - 複数のリクエストを同時に処理する必要があるアプリケーションに最適です。

- **大規模なコミュニティ**：世界中の何千人もの開発者が Node.js を継続的にメンテナンスし、新機能を追加しています。
  - 多くの学習リソースがあります。
  - 質問の答えを見つけやすいです。

- **npm（Node Package Manager）**：プロジェクトのパッケージやモジュールをインストール・管理できる強力なツールです。
  - HTTP リクエスト処理、データベース操作、日時フォーマット、コードテストなど多目的なパッケージがあります。
  - 他の開発者が書いてテストし共有したコードを再利用できます。
  - ワークフローが速く効率的になります。

- **コスト効率**：Node.js は無料でオープンソースです。
  - パッケージやライブラリ、関連ツールも主に無料です。
  - 予算を最適化する際に重要な要素です。

## Node.js の欠点

- **シングルスレッドの制限**：一度に1つのスレッドしか動作しないため、1回に1つの操作しか処理できません。
  - アーキテクチャは同時リクエストを効率的に処理できますが、CPU 集約型のタスクはメインスレッドをブロックし、パフォーマンス問題を引き起こす場合があります。
  - CPU 集約型タスクの例：複雑な数学演算、画像・動画処理、暗号処理。
  - これを克服する方法はありますが、通常はアプリケーションの複雑さが増します。

- **非同期プログラミング**：Node.js は非同期プログラミングに依存しています。
  - 長時間かかる可能性のあるタスクを開始し、完了を待たずにメインプログラムは並行して動作します。
  - タスク完了時に結果を処理します。
  - 多くの場合「コールバック」と呼ばれる、非同期処理完了時に実行される関数を使います。
  - コードの読みやすさ、理解、デバッグが難しくなる場合があります。

- **パッケージの品質問題**：npm から選ぶパッケージの中には、継続的にメンテナンスされていなかったり、脆弱性をもたらすものがあります。
  - 開発者は各パッケージを慎重に評価する必要があります。
  - 品質とセキュリティのベストプラクティスに従っているか確認してください。

## スレッド

- **スレッド**：プロセス内の実行経路で、現在動作しているコンピュータプログラムのようなものです。

## NVM（Node Version Manager）

- **NVM**：同じコンピュータ上で複数の Node.js バージョンを管理するツールです。
- **目的**：異なるプロジェクトで異なる Node.js バージョンを簡単に切り替えられます。
- **推奨**：将来の柔軟性のために、Node.js をインストールする前に NVM をインストールすることをお勧めします。

## macOS/Linux での NVM インストール

- **Xcode コマンドラインツール（macOS）**：macOS で NVM インストールスクリプトを実行する前に手動でインストールする必要があります。

```bash
xcode-select --install
```

- **Xcode インストールの確認**：

```bash
xcode-select --version
```

- **NVM インストールスクリプトのダウンロードと実行**：以下のいずれかのコマンドを実行してください：

```bash
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.3/install.sh | bash
```

```bash
wget -qO- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.3/install.sh | bash
```

- **ターミナルの再起動**：処理完了後、新しいコマンドと設定を読み込むためにターミナルを再起動する必要があります。

- **NVM インストールの確認**：

```bash
nvm -v
```

## macOS/Linux での NVM を使った Node.js インストール

- **最新の LTS バージョンをインストール**：最新の長期サポート（LTS）バージョンをインストールするには：

```bash
nvm install --lts
```

- **LTS（長期サポート）**：このバージョンは通常、安定性、信頼性、セキュリティを優先し、新機能や実験的機能よりも重視されます。
  - 十分にテストされ、本番環境向けです。
  - バグ修正やセキュリティパッチに重点を置いた長期サポートが保証されています。

- **特定バージョンのインストール**：バージョン番号を指定して特定の Node.js バージョンをインストールできます：

```bash
nvm install <version>
```

例：

```bash
nvm install 20
```

- **バージョンの切り替え**：

```bash
nvm use <version>
```

例：

```bash
nvm use 20
```

- **現在の Node.js バージョンの確認**：

```bash
node -v
```

## Windows での NVM インストール

- **公式 NVM のサポート**：公式の NVM は特定のケースでのみ Windows をサポートしています。
  - Windows Subsystem for Linux (WSL) がある場合は、上記の macOS/Linux ガイドに従えます。

- **nvm-windows**：Windows 用の Node.js バージョン管理ユーティリティです。
  - 公式 NVM プロジェクトとは完全に別のプロジェクトです。
  - 目的は同じで、同じデバイス上で複数の Node.js バージョンを扱いやすくします。

- **インストール前の注意**：Windows 用 NVM をインストールする前に、既存の Node.js インストールをアンインストールすることをお勧めします。

- **インストール手順**：
  - 公式リポジトリの Releases セクションにアクセス：`https://github.com/coreybutler/nvm-windows/releases`。
  - 最新のインストーラーをダウンロード。
  - 指示に従ってインストール。

- **ターミナルの再起動**：`nvm` コマンドがすぐに使えない場合は、ターミナルを再起動してください。

- **インストールの確認**：

```powershell
nvm -v
```

## Windows 用 NVM での Node.js インストール

- **最新の LTS バージョンをインストール**：

```powershell
nvm install lts
```

- **特定バージョンのインストール**：

```powershell
nvm install <version>
```

例：

```powershell
nvm install 20
```

- **バージョンの切り替え**：

```powershell
nvm use <version>
```

例：

```powershell
nvm use 20
```

- **現在の Node.js バージョンの確認**：

```bash
node -v
```

## 基本的な NVM と Node.js のコマンド

- **コマンドについての注意**：すべてのコマンドは macOS/Linux および WSL にインストールされた NVM で動作します。`nvm-windows` では動作が異なるか、動作しない場合があります。`nvm-windows` 用の代替コマンドが可能な場合は提供されています。

### PowerShell の実行ポリシー（Windows）

- **デフォルトの実行ポリシー**：PowerShell のデフォルト実行ポリシーは `Restricted` で、セキュリティ対策としてスクリプトの実行を防止しています。

- **PowerShell を管理者として実行**：PowerShell を右クリックし、「管理者として実行」をクリックしてください。

- **現在の実行ポリシーの確認**：

```powershell
Get-ExecutionPolicy
```

- **実行ポリシーを RemoteSigned に設定**：ローカルスクリプトと `npm` の実行を許可します：

```powershell
Set-ExecutionPolicy RemoteSigned
```

- **設定後**：管理者権限の PowerShell ウィンドウを閉じ、管理者権限なしの PowerShell ウィンドウを開いてください。

### インストール済みバージョンの一覧表示

- **インストール済みバージョンの一覧表示**：インストールされている Node.js のバージョンをすべて表示し、現在使用中のバージョンを確認します：

```bash
nvm ls
```

### デフォルトの Node.js バージョン設定（macOS/Linux の NVM のみ）

- **デフォルトバージョンの設定**：新しいターミナルセッションで使うデフォルトの Node.js バージョンを設定します：

```bash
nvm alias default <version>
```

例：

```bash
nvm alias default 22.20.0
```

- **カスタムエイリアスの作成**：バージョンにより具体的な名前を付けられます：

```bash
nvm alias <new_name> <version>
```

例：

```bash
nvm alias new-api 18.20.8
```

- **カスタムエイリアスの使用**：

```bash
nvm use <new_name>
```

例：

```bash
nvm use new-api
```

### Node.js バージョンの削除

- **バージョンのアンインストール**：

```bash
nvm uninstall <version>
```

例：

```bash
nvm uninstall 20.19.5
```

- **メジャーバージョンでの削除（macOS/Linux の NVM）**：`uninstall` コマンドにメジャーバージョン（例：`22` や `20`）を指定して削除できます：

```bash
nvm uninstall 20
```

複数のリリースがインストールされている場合（例：`20.14.0` と `20.12.2`）、最新バージョンから順に削除されます。`nvm uninstall 20` を実行すると `20.14.0` が最初に削除されます。

### Node.js バージョンの確認

- **現在のバージョンの確認**：

```bash
node -v
```

### npm バージョンの確認

- **npm**：Node.js 用のパッケージマネージャーで、ソフトウェアパッケージのインストール、公開、管理に使われます。Node.js インストール時に自動的にインストールされます。

- **npm バージョンの確認**：

```bash
npm -v
```

### JavaScript ファイルの実行

- **JavaScript ファイルの実行**：現在アクティブな Node.js ランタイムで JavaScript ファイルを実行します：

```bash
node <file.js>
```

例：

```bash
node app.js
```

### Node.js プロジェクトの作成

- **プロジェクトの初期化**：プロジェクトの詳細や依存関係を管理する `package.json` ファイルを作成して新しい Node.js プロジェクトを作成します：

```bash
npm init
```

- **対話型セットアップ**：`npm init` コマンドは `package.json` ファイルのセットアップを対話形式で案内します。
  - プロジェクト名、説明、バージョンなどの質問があります。

- **デフォルトセットアップ**：デフォルトの `package.json` ファイルで開始するには：

```bash
npm init -y
```

または

```bash
npm init --yes
```

### パッケージのインストール

- **特定のパッケージをインストール**：

```bash
npm install <package>
```

例：

```bash
npm install express
```

### すべての依存関係のインストール

- **依存関係のインストール**：プロジェクトの `package.json` ファイルに記載されたすべての依存関係をインストールします：

```bash
npm install
```
