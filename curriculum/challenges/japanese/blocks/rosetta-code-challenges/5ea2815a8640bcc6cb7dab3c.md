# --description--

<ol>
  <li>0より大きい整数 <code>n₀</code> を取ります。</li>
  <li><code>n₀</code> を逆順にして <code>n₀</code> に加えることで、次の数 <code>n</code> を作ります。</li>
  <li><code>n</code> が回文、つまり <code>n</code> の桁を逆にしたものが <code>n</code> と等しくなったら終了します。</li>
</ol>

上記の漸化式は、ほとんどの初期値 `n` = 1, 2, ... に対して、比較的早く回文で終了します。

例えば `n₀` = 12 の場合は次のようになります：

```md
12
12 + 21 = 33,  a palindrome!
```

また `n₀` = 55 の場合は：

```md
55
55 + 55 = 110
110 + 011 = 121,  a palindrome!
```

回文のチェックは加算の*後*に行われることに注意してください。

一部の初期値は永遠に続くように見えます。196の漸化式は何百万回も計算され、何百万桁もの数が生成されましたが、回文にはなりませんでした。このように回文で終わらない数は**ライクリル数**と呼ばれます。

この課題では、500回（またはそれ以上）の繰り返しで回文を作らない初期値をライクリル数と定義します。

**シードと関連するライクリル数：**

ライクリル数のシーケンスで生成される任意の整数もまたライクリル数です。

一般に、あるライクリル数から始まるシーケンスは、以前のライクリル数候補のシーケンスに収束して合流する*場合があります*。例えば、196と689の数列は次のように始まります：

```md
    196
    196 + 691 = 887
    887 + 788 = 1675
    1675 + 5761 = 7436
    7436 + 6347 = 13783
    13783 + 38731 = 52514
    52514 + 41525 = 94039
    ...
    689
    689 + 986 = 1675
    1675 + 5761 = 7436
    ...
```

このように、689から始まるシーケンスは196のシーケンスに収束し、同じ数列を続けます。

このため、ライクリル数は真の**シード**ライクリル数候補と、回文を作らないものの、より小さいライクリル数のシーケンスに含まれる整数を生成する**関連**数に分けられます。

```js
function isLychrel(n) {
  function reverse(num) {
    return parseInt(
      num
        .toString()
        .split('')
        .reverse()
        .join('')
    );
  }

  var i;
  for (i = 0; i < 500; i++) {
    n = n + reverse(n);
    if (n == reverse(n)) break;
  }

  return i == 500;
}
```
