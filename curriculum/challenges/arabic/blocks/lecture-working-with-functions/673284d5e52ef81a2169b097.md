# --interactive--

في الدرس السابق، تعلمت كيف تعمل مع الدوال، وهي قطع قابلة لإعادة الاستخدام من الشفرة تساعد على جعل شفرتك أكثر تنظيمًا وأسهل في الصيانة وأكثر كفاءة. جميع الأمثلة السابقة استخدمت صيغة الدالة العادية، مثل هذا:

```js
function greetings(name) {
  console.log("Hello, " + name + "!");
}
```

لكن هناك طريقة أخرى لكتابة الدوال في JavaScript وهي إنشاء تعبير دالة سهمية. إليك كيف يمكنك إعادة صياغة المثال السابق لاستخدام صيغة الدالة السهمية بدلاً من ذلك:

```js
const greetings = (name) => {
  console.log("Hello, " + name + "!");
};
```

في هذا المثال المعدل، نُعرّف متغيرًا باستخدام `const` اسمه `greetings` ونعين له دالة مجهولة. ستبدو معظم الصيغة النحوية مألوفة لك باستثناء غياب كلمة `function` وإضافة السهم (`=>`) بين معلمة `name` وجسم الدالة. إذا كانت قائمة المعلمات تحتوي على معلمة واحدة فقط، يمكنك إزالة الأقواس العادية هكذا:

```js
const greetings = name => {
  console.log("Hello, " + name + "!");
};
```

إذا كانت دالتك السهمية لا تحتوي على معلمات، فيجب عليك استخدام الأقواس العادية هكذا:

```js
const greetings = () => {
  console.log("Hello");
};
```

عند تعلم الدوال لأول مرة، كان عليك تغليف جسم الدالة بأقواس متعرجة. لكن إذا كان جسم الدالة يحتوي على سطر واحد فقط من الشفرة، يمكنك إزالة الأقواس المتعرجة هكذا:

```js
const greetings = name => console.log("Hello, " + name + "!");
```

من المهم أن تعرف أن إزالة الأقواس العادية والمتعرجة في صيغة الدالة العادية لن تنجح. ستحصل على أخطاء إذا حاولت فعل شيء مثل هذا:

```js
// This will produce syntax errors 
function greetings name console.log("Hello, " + name + "!");
```

هذه الأنواع من الدوال ذات السطر الواحد تعمل فقط إذا كنت تستخدم صيغة الدالة السهمية. مفهوم رئيسي آخر هو جملة `return`. إليك مثالًا على استخدام صيغة الدالة السهمية لحساب المساحة:

:::interactive_editor

```js
const calculateArea = (width, height) => {
  const area = width * height;
  return area;
};

console.log(calculateArea(5, 3)); // 15
```

:::

ننشئ متغيرًا داخل الدالة اسمه `area` ثم نُرجع هذا المتغير. لكن يمكننا تبسيط الشفرة قليلاً وإرجاع الحساب نفسه:

:::interactive_editor

```js
const calculateArea = (width, height) => {
  return width * height;
}; 

console.log(calculateArea(5, 3)); // 15
```

:::

إذا حاولت إزالة الأقواس المتعرجة ووضع الحساب في نفس السطر، فستحصل على رسالة `Uncaught SyntaxError: Unexpected token 'return'`:

```js
const calculateArea = (width, height) => return width * height;
```

سبب ظهور هذا الخطأ هو حاجتك لإزالة جملة `return`. عندما تزيل جملة `return`، سيختفي الخطأ وستظل الدالة تُرجع الحساب ضمنيًا.

```js
const calculateArea = (width, height) => width * height;
```

فمتى يجب عليك استخدام صيغة الدالة السهمية؟ الأمر يعتمد. كثير من المطورين يستخدمونها باستمرار في مشاريعهم الشخصية. لكن عند العمل ضمن فريق، يعتمد الاختيار عادةً على ما إذا كان قاعدة الشفرة الحالية تستخدم الدوال العادية أو الدوال السهمية. في الدروس القادمة، سنغطي متى تستخدم الدوال السهمية ومتى تتجنبها.
