# --interactive--

Tujifunze kuhusu kusudi la njia ya `preventDefault()` kwenye matukio.

Kila tukio linalotokea katika DOM lina tabia fulani ya chaguo-msingi. Tukio la `click` kwenye kisanduku cha kuchagua hubadilisha hali ya kisanduku hicho, kwa chaguo-msingi. Kubonyeza kitufe cha nafasi (space bar) kwenye kitufe kilicho na makini huchochea kitufe hicho. Njia ya `preventDefault()` kwenye vitu hivi vya tukio inazuia tabia hiyo isitokee.

Tuchukulie mfano. Tufafanue kipengele cha `input` kwa mtumizi kuandika ndani yake:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<label>Enter some characters:
  <input type="text">
</label>
```

```css
  margin-bottom: 12px;

```

Kisha, tunahitaji kushikilia tukio la `keydown` ili kusikiliza herufi inayoandikwa kwenye kibodi. Kumbuka hatutaki matukio ya `change` au `input` hapa, kwa sababu tunahitaji taarifa ya kibodi.

```js
const input = document.querySelector("input");

input.addEventListener("keydown", (e) => {
    
})
```

`e.key` inakupa thamani ya kitufe kilichobonyezwa, kama `a` kwa kitufe cha `a` au `Enter` kwa kitufe cha `Enter`.

Kwa msimbo ulio hapo juu, unapoandika katika `input`, herufi unayoandika itaonyeshwa katika kipengele cha `p`.

Hii ni nzuri, lakini hatutaki kuonyesha herufi hizo pia katika `input`. Hapa ndipo njia yetu ya `preventDefault()` inapoingia. Tabia ya chaguo-msingi ya `keydown` ni kuonyesha herufi katika sehemu ya ingizo. Tuzuie hilo kwa kuita `e.preventDefault()`:

```js

input.addEventListener("keydown", (e) => {
  e.preventDefault();
```

Na kama hivyo, umezuia tabia ya chaguo-msingi ili uweze kutekeleza usindikaji wako wa tukio ulio maalum.

Mfano mwingine wa kawaida wa kutumia njia ya `e.preventDefault()` unahusiana na uwasilishaji wa fomu. Kwa chaguo-msingi, kuwasilisha fomu hutoa data kwa seva na kurudisha ukurasa upya. Kutumia `e.preventDefault()` kunazuia hili kutokea.

```js
const form = document.querySelector("form");

form.addEventListener("submit", (e) => {
  e.preventDefault();
  // rest of code goes here
});
```

Kuzuia tabia ya chaguo-msingi ni nzuri wakati unahitaji udhibiti zaidi juu ya jinsi mtumizi anavyoshirikiana na ukurasa, lakini ni muhimu kuzingatia mambo kama Ufikikaji: tabia yako maalum inapaswa kutoa vipengele sawa na tabia ya chaguo-msingi.
