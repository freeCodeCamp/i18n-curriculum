# --description--

在构建网页应用时，考虑你的文件和文件夹的组织结构非常重要。这将有助于保持你的代码有序且更易于维护。

让我们来看一个用于超文本标记语言和 CSS 项目的示例文件夹结构：

```bash
.
├── /assets
│   ├── /images
│   │   ├── logo.png
│   │   ├── banner.jpg
│   │   └── icons.svg
│   ├── /fonts
│   │   ├── custom-font.woff
│   │   └── custom-font.woff2
├── /css
│   ├── main.css
│   ├── about.css
│   └── contact.css
├── index.html
├── about.html
├── contact.html
└── README.md
```

在示例顶部，有一个单点，表现根目录。

根目录是文件系统中的顶层目录，作为所有文件路径的起点，并包含所有其他目录和文件。目录是文件夹的另一种称呼。

在根目录内，显示的第一个目录是 `assets` 目录。术语 `assets` 指的是项目中使用的任何文件，例如图像、字体或其他资源。

在此示例中，`assets` 目录内有两个目录，分别称为 `images` 和 `fonts`。

下一个目录是 `css` 目录。有时这也被称为样式表目录。这是你存放所有 CSS 文件的地方。

在 `css` 目录下是超文本标记语言文件。`index.html` 文件表现主页，而 `about.html` 和 `contact.html` 分别表现关于和联系页面。

最后，有一个 `README.md` 文件。`README` 文件通常用于提供有关项目的有用信息，例如它的功能、如何使用以及其他相关细节。

`.md` 扩展名代表 Markdown，这是一种轻量级标记语言，通常用于编写文档。

通过将你的 `project` 组织成资产、CSS 和 `HTML` 文件，可以轻松查找你需要的内容。这也使其他开发者更容易为你的 `project` 做出贡献。

重要的是要注意，这只是你如何组织你的文件和文件夹的一个示例。结构化一个项目有许多不同的方法。
