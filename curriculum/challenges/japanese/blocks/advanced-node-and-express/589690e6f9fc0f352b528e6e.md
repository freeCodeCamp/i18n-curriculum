# --description--

今はすべてが`server.js`ファイルに入っています。これだと管理が難しく、拡張性の低いコードになりがちです。`routes.js`と`auth.js`の2つの新しいファイルを作成してください。

どちらのファイルも、次のコードから始めてください。

```js
module.exports = function (app, myDataBase) {

}
```

次に、サーバーファイルの先頭でこれらのファイルを次のようにrequireしてください：`const routes = require('./routes.js');` データベースとの接続が成功した直後に、それぞれを次のようにインスタンス化します：`routes(app, myDataBase)`

最後に、サーバー内のすべてのルートを新しいファイルに貼り付け、サーバーファイルからは削除してください。また、ルーティング専用に作成された`ensureAuthenticated`関数も移動します。`const passport = require('passport');`ファイルのエクスポート行の上に、`routes.js`のような使用している依存関係を正しく追加してください。

エラーがなくなり、サーバーファイルにルーティングが（キャッチブロック内のルートを除いて）なくなるまで、依存関係の追加を続けてください。

`auth.js`ファイルでも、シリアライズやローカルストラテジーの設定など認証に関するすべてのものを同様に移動し、サーバーファイルから削除してください。依存関係を追加し、サーバー内の同じ場所で`auth(app, myDataBase)`を呼び出すことを忘れないでください。

正しくできたと思ったらページを提出してください。エラーが出る場合は、<a href="https://forum.freecodecamp.org/t/advanced-node-and-express/567135#clean-up-your-project-with-modules-2" target="_blank" rel="noopener noreferrer nofollow">完成したプロジェクトの例を確認できます</a>。
