# --interactive--

## Uthibitisho wa fomu kwa JavaScript

- **API ya uthibitisho wa vizingiti**: Vipengele fulani vya HTML, kama vipengele vya `textarea` na `input`, vinaweka wazi API ya uthibitisho wa vizingiti. API hii inakuwezesha kudhibitisha kwamba thamani iliyotolewa na mtumizi kwa kipengele hicho inakidhi uthibitisho wowote wa ngazi ya HTML uliyoandika, kama vile urefu wa chini au kulingana na muundo fulani.
- **Njia ya `checkValidity()`**: Njia hii hurudisha `true` ikiwa kipengele kinakidhi uthibitisho wote wa HTML (kulingana na sifa zake), na `false` ikiwa hakikidhi.

:::interactive_editor

```html
<form>
  <label>
    Email:
    <input
      id="email"
      type="email"
      required
      pattern=".+\.com$"
      placeholder="example@site.com"
    />
  </label>
</form>

<script>
  const input = document.getElementById("email");

  input.addEventListener("input", (e) => {
    if (!e.target.checkValidity()) {
      e.target.setCustomValidity("You must use a .com email.");
    } else {
      e.target.setCustomValidity("");
    }
  });
</script>
```

:::

- **Njia ya `reportValidity()`**: Njia hii inamwambia kivinjari kwamba `input` si halali.

:::interactive_editor

```html
<form>
  <label>
    Email:
    <input
      id="email2"
      type="email"
      required
      pattern=".+\.com$"
      placeholder="example@site.com"
    />
  </label>
</form>

<script>
  const input = document.getElementById("email2");

  input.addEventListener("input", (e) => {
    if (!e.target.checkValidity()) {
      e.target.reportValidity();
    }
  });
</script>
```

:::

- **Sifa ya `validity`**: Sifa hii hutumika kupata au kuweka hali ya uhalali wa udhibiti wa fomu (kama `<input>`, `<select>`, n.k.) na hutoa taarifa kuhusu kama ingizo la mtumizi linakidhi vizingiti vilivyowekwa kwa kipengele hicho (mfano, maeneo ya `required`, vizingiti vya muundo, urefu wa juu, n.k.).

:::interactive_editor

```html
<input
  id="age"
  type="number"
  min="18"
  placeholder="Enter age (18+)"
/>

<script>
  const input = document.getElementById("age");

  input.addEventListener("input", (e) => {
    console.log(e.target.validity);
  });
</script>
```

:::

- **Sifa ya `patternMismatch`**: Hii itakuwa `true` ikiwa thamani haikidhi muundo wa usemi wa kawaida uliobainishwa.

## Njia ya `preventDefault()`

- **Maelezo**: Kila tukio linalotokea katika DOM lina tabia fulani ya chaguo-msingi. Tukio la bofya kwenye kisanduku cha kuchagua hubadilisha hali ya kisanduku hicho, kwa chaguo-msingi. Kubonyeza kitufe cha nafasi kwenye kitufe kilicho na makini hufanya kitufe hicho kitumike. Njia ya `preventDefault()` kwenye vitu hivi vya `Event` inazuia tabia hiyo isitokee.

:::interactive_editor

```html
<form id="form">
  <input type="text" placeholder="Try to submit" />
  <button type="submit">Submit</button>
</form>

<p id="status"></p>

<script>
  const form = document.getElementById("form");
  const status = document.getElementById("status");

  form.addEventListener("submit", (event) => {
    event.preventDefault();
    status.textContent = "Form submission prevented.";
  });
</script>
```

:::

## Kuwasilisha fomu

- **Maelezo**: Kuna njia tatu za fomu kuwasilishwa. Ya kwanza ni wakati mtumizi anabofya kitufe kilicho kwenye fomu ambacho kina sifa ya `type` iliyowekwa kuwa `submit`. Ya pili ni wakati mtumizi anabonyeza kitufe cha `Enter` kwenye eneo lolote la `input` linaloweza kuhaririwa ndani ya fomu. Ya tatu ni kupitia wito wa JavaScript kwa njia za `requestSubmit()` au `submit()` za kipengele cha `form`.
- **Sifa ya `action`**: Sifa ya `action` inapaswa kuwa na URL au njia ya jamaa kwa kikoa cha sasa. Thamani hii inaamua wapi fomu itajaribu kutuma data - ikiwa hutaweka sifa ya `action`, fomu itatuma data kwa URL ya ukurasa wa sasa.

```html
<form action="https://freecodecamp.org" method="GET">
  <input
    type="number"
    name="number"
    placeholder="Enter a number"
  />
  <button type="submit">Submit</button>
</form>
```

- **Sifa ya `method`**: Sifa hii inakubali njia ya kawaida ya `HTTP`, kama `GET` au `POST`, na hutumia njia hiyo wakati wa kutuma ombi kwa URL ya kitendo. Wakati njia haijawekwa, fomu itatumia ombi la `GET` kwa chaguo-msingi. Data katika fomu itakuwa imeandikwa kama jozi za `name=value` za URL na kuongezwa kwenye URL ya kitendo kama vigezo vya hoja ya utafutaji.

```html
<form action="/data" method="POST">
  <input
    type="number"
    id="input"
    placeholder="Enter a number"
    name="number"
  />
  <button type="submit">Submit</button>
</form>
```

- **Sifa ya `enctype`**: Kipengele cha `form` kinakubali sifa ya `enctype`, inayowakilisha aina ya usimbaji wa data itakayotumika. Sifa hii inakubali thamani tatu tu: `application/x-www-form-urlencoded` (ambayo ni chaguo-msingi, ikituma data kama mwili wa fomu uliosimbwa kwa URL), `text/plain` (inayotuma data kwa maandishi wazi, katika jozi za `name=value` zilizotengwa kwa mistari mipya), au `multipart/form-data`, ambayo ni maalum kwa kushughulikia fomu zilizo na upakiaji wa faili.
