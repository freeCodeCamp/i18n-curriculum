# --description--

`Object-oriented programming` (OOP)는 필드(종종 속성 또는 프로프라고도 함)의 형태로 데이터를 포함할 수 있고, 절차(종종 메서드라고도 함)의 형태로 코드를 포함할 수 있는 `objects` 개념에 기반한 프로그래밍 패러다임입니다.

`OOP`의 일반적인 예는 Java나 Python 같은 프로그래밍 언어에서 클래스를 만드는 것입니다. 이 클래스는 객체의 구조를 정의하며, 여기에는 속성과 메서드가 포함됩니다. 예를 들어, `Car` 클래스는 `color`와 `model` 같은 속성, 그리고 `drive()`와 `brake()` 같은 메서드를 포함할 수 있습니다.

이 문맥에서 `oriented`라는 용어는 특정 목적이나 원칙에 맞춰져 있거나 지향된다는 의미입니다.

예를 들어, freeCodeCamp의 반응형 웹 디자인 커리큘럼은 프로젝트 완료를 중심으로 구성되어 있기 때문에 `project-oriented`로 설명되며, 이는 이론적 지식뿐 아니라 실용적인 애플리케이션에 포커스를 맞춘 교육 방식임을 뜻합니다.
