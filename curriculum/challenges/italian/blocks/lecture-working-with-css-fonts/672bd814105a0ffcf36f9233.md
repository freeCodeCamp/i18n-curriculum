# --interactive--

Una famiglia di caratteri tipografici è un gruppo di caratteri che condividono un design comune. Tutti i caratteri appartenenti alla stessa famiglia si basano sullo stesso tipo di carattere principale, ma presentano anche variazioni nello stile, nel peso e nella larghezza. Puoi pensarli come fratelli che condividono caratteristiche simili ma hanno anche alcune differenze. Per esempio, Arial è una famiglia di caratteri che include variazioni come Arial Bold e Arial Italic.

In CSS, puoi impostare la famiglia di caratteri con la proprietà `font-family`. Ecco un esempio di come impostare il carattere per il paragrafo su `Arial`:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<p id="arial-font">Example paragraph using Arial font.</p>
<p>Paragraph not using Arial font.</p>
```

```css
#arial-font {
  font-family: Arial;
}
```

:::

Ma cosa succede se la famiglia di caratteri non viene trovata? Puoi specificare più famiglie di caratteri in ordine di priorità, dalla più alta alla più bassa, separandole con delle virgole. Questi caratteri alternativi agiranno come opzioni di fallback. In questo esempio, Arial è il carattere principale e Lato è un carattere alternativo:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<p id="specified-font">Example paragraph using specified fonts.</p>
<p>Paragraph not using specified fonts.</p>
```

```css
#specified-font {
  font-family: Arial, Lato;
}
```

:::

Il browser renderizzerà il primo carattere (Arial) se è presente sul dispositivo. In caso contrario, verrà renderizzato il secondo carattere (Lato).

Devi anche sapere come funziona questo processo dietro le quinte. La selezione non si ferma se il primo carattere è disponibile. La famiglia di caratteri viene scelta un carattere alla volta, quindi se un carattere manca in un font, il browser lo cerca nei font con priorità più bassa. Interessante, vero?

Nel contesto dello sviluppo web, troverai anche famiglie di caratteri generiche. Sono molto importanti. Una famiglia di caratteri generica è un carattere predefinito che i browser web renderizzeranno quando i caratteri con priorità più alta non sono disponibili. Per assicurare che il contenuto sia comunque leggibile, il browser sostituisce il carattere originale con il carattere più appropriato trovato, basandosi sulla famiglia di caratteri generica specificata. Alcune delle più comuni sono:

- serif
- sans-serif
- monospace
- cursive
- fantasy

Ma ci sono altre opzioni tra cui scegliere. Nel codice qui sotto, abbiamo tre famiglie di caratteri alternative:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<p id="specified-font">Example paragraph using specified fonts.</p>
<p>Paragraph not using specified fonts.</p>
```

```css
#specified-font {
  font-family: Arial, Lato, sans-serif;
}
```

:::

Arial ha la priorità più alta. Se Arial non viene trovato, il browser proverà a renderizzare Lato. Se nessuno dei due viene trovato, il browser userà la famiglia di caratteri generica sans-serif, selezionando un carattere con queste caratteristiche tra quelli installati sul sistema dell’utente.

Dovresti sempre includere una famiglia di caratteri generica alla fine della lista di font-family. Tieni presente che il carattere generico potrebbe apparire diverso dal carattere che avevi immaginato nel tuo design, quindi è sempre utile verificare come appaiono questi font di fallback su diversi browser.

Per rendere l’esperienza utente il più coerente possibile, è anche consigliato usare caratteri web-safe. Queste famiglie di caratteri sono solitamente installate sulla maggior parte dei dispositivi, quindi è molto probabile che vengano trovate e renderizzate correttamente per la maggior parte degli utenti. Ne imparerai di più nelle prossime lezioni.

Le famiglie di caratteri sono essenziali nel mondo del design web. Usando la proprietà font-family, puoi trasformare istantaneamente l’aspetto e la sensazione del tuo testo.
