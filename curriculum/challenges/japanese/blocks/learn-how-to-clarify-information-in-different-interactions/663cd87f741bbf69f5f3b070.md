# --description--

マリア と ブライアン は以前このプロジェクトについて会話をしました。その中で、ブライアン は マリア に `timeline` を渡しました。`timeline` とは、起こるイベントを順番に並べたリストのことです。例えば、起きる瞬間から寝る瞬間までの一日の `timeline` を作ることができます。これにより、何が最初に起こり、次に何が起こり、最後に何が起こるかがわかります。プロジェクトでは、特定のタスクがいつ、どのシーケンスで起こるかを理解するために `timeline` を持つことが重要です。

`To match` は、ある情報が別の情報と同じであることを意味します。もし二つのものが `don't match` であれば、それらは同じではありません。例えば、友達の名前が二つのリストで綴りが違う場合、最初のリストの名前は二番目のリストの名前と `doesn't match` です。つまり、それらは異なるということです。

プロジェクトの `documentation` は、そのプロジェクトに関する重要な情報を含む書類やファイルの集まりです。プロジェクトの内容、完了までのステップ、守るべきルールやガイドラインなどの詳細が含まれています。これにより、プロジェクトに関わる全員が何をどうすればよいか理解できます。
