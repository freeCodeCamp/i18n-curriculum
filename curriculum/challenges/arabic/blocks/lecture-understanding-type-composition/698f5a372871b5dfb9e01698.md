# --description--

في الدروس السابقة، تعلمت كيفية العمل مع الأنواع الأولية مثل `string` و`boolean`. لكن TypeScript يسمح لك ببناء أنواع جديدة من الأنواع الموجودة. أحد الأمثلة على ذلك هو نوع الاتحاد.

نوع الاتحاد هو نوع يجمع بين نوعين أو أكثر من الأنواع الأخرى.

إليك الصيغة النحوية الأساسية:

```ts
type1 | type2 | type3
```

لنلقِ نظرة على بعض الأمثلة لفهم المفهوم بشكل أفضل.

هنا متغير يسمى `id` بنوع اتحاد لـ `string` أو `number`:

```ts
let id: string | number = "user_783";
```

يمكن تحديث القيمة إلى رقم وهذا لا يزال صالحًا لنوع الاتحاد:

```ts
let id: string | number = 1024;
```

إذا حاولت تعيين قيمة أخرى ليست `string` أو `number`، فسيعرض TypeScript رسالة خطأ:

```ts
// Type 'string[]' is not assignable to type 'string | number'.
let id: string | number = ["Java", "Python"];
```

عند العمل مع أنواع الاتحاد، من المهم استخدام الطرق التي تعمل مع جميع الأنواع، وإلا سيُرمي TypeScript خطأً مثل هذا:

```ts
function getId(id: string | number) {
  // Property 'toUpperCase' does not exist on type 'string | number'.
  return id.toUpperCase();
}
```

هناك حل لهذا يتضمن تضييق نوع الاتحاد. لكنك ستتعلم المزيد عن ذلك وعن كيفية عمل أنواع الاتحاد مع الأسماء المستعارة للأنواع في الدروس القادمة.

أنواع الاتحاد مفيدة أيضًا عند العمل مع القيم `null` و`undefined`.

إليك مثال حيث قد يحتوي متغير على قيمة `string`، أو قد يكون `null` إذا لم يتم تعيين قيمة بعد:

```ts
let username: string | null;

// Type 'number' is not assignable to type 'string | null'.
username = 42;
```

في هذه الحالة، يمكن أن يكون `username` إما `string` أو `null`، ولا شيء آخر. قد تستخدم `null` عندما تريد تمثيل غياب القيمة عن قصد.

إليك مثال آخر باستخدام `undefined`:

```ts
let score: number | undefined;

score = undefined; 

// Type 'string' is not assignable to type 'number | undefined'.
score = "high";
```

في هذا المثال، يمكن أن يكون `score` إما `number` أو `undefined`. قد تستخدم `undefined` عندما لم يتم تعيين قيمة بعد.

استخدام أنواع الاتحاد مع `null` و`undefined` يوضح متى قد تكون القيمة مفقودة، مع السماح لـ TypeScript بالكشف عن التعيينات غير الصالحة.
