# --description--

JavaScript — це мова з динамічною типізацією, що означає, що вам не потрібно вказувати тип даних змінної під час її оголошення. Замість цього тип визначається на основі значення, призначеного змінній під час виконання програми. Це дозволяє змінювати тип змінної протягом роботи програми.

Розглянемо приклад:

```js
let example = "Hello";
example = 42;
```

У цьому прикладі у нас є змінна з ім’ям `example` з типом даних рядок. Але потім ми оновлюємо значення, щоб воно стало числом.

Гнучкість динамічної типізації робить JavaScript більш поблажливим і зручним для швидкого написання скриптів, але це також може призвести до помилок, які важче виявити, особливо коли ваша програма стає більшою.

У статично типізованих мовах, таких як C# або C++, ви повинні оголосити тип даних змінної під час її створення, і цей тип не може змінюватися.

Наприклад, якщо ви оголосите змінну як `integer`, ви можете призначати їй лише цілі числа. Якщо спробуєте призначити інший тип, програма видасть помилку.

Ось приклад на мові C#:

```csharp
int data = 42; // data must always be an integer
data = "Hello"; // This would cause an error in C#
```

Різниця між динамічною та статичною типізацією полягає у гнучкості проти безпеки вашого коду. Мови з динамічною типізацією пропонують гнучкість, але ціною потенційних помилок виконання.

Статично типізовані мови застосовують суворіші правила, які можуть запобігти певним помилкам, але вимагають більше оголошень наперед і пропонують менше гнучкості у зміні типів.

Ось ще один приклад створення змінної з типом `number`, а потім зміни її на тип `string`:

```js
let data = 100;  // Initially a number
data = "New data";  // Dynamically changes to a string
```

У статично типізованій мові така зміна не була б дозволена, оскільки тип даних був би фіксованим.

Підсумовуючи, динамічна типізація в JavaScript дозволяє змінним вільно змінювати типи, що дає гнучкість, але може призвести до несподіваних помилок під час виконання.

Статично типізовані мови, як Java, вимагають вказувати типи змінних наперед, що допомагає виявляти помилки до запуску програми, але пропонує менше гнучкості.
