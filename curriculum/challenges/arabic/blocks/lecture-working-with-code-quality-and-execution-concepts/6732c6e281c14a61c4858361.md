# --interactive--

الإغلاقات من أقوى الميزات في JavaScript وغالبًا ما يُساء فهمها. في جوهرها، الإغلاق هو دالة لها وصول إلى المتغيرات في نطاقها الخارجي المحيط حتى بعد انتهاء تنفيذ الدالة الخارجية. قد يبدو هذا معقدًا لكنه مفهوم أساسي يمكّن العديد من أنماط البرمجة المتقدمة في JavaScript.

لفهم الإغلاقات، لنبدأ بمثال:

:::interactive_editor

```js
function outerFunction(x) {
    let y = 10;
    function innerFunction(){
        console.log(x + y);
    }
    return innerFunction;
}

let closure = outerFunction(5);
console.log(closure()); // 15
```

:::

في هذا المثال، تأخذ `outerFunction` معلمة `x` وتُعرّف متغيرًا محليًا `y`. ثم تُعرّف `innerFunction` التي تستخدم كلًا من `x` و `y`. في النهاية تُعيد `innerFunction`. عندما نستدعي `outerFunction(5)` تُعيد `innerFunction` التي نُعيّنها للمتغير `closure`. وعندما نستدعي لاحقًا `closure()`، لا تزال لديها وصول إلى `x` و `y` من `outerFunction` رغم انتهاء تنفيذ `outerFunction`. هذه هي جوهر الإغلاق.

تحافظ الدالة الداخلية على مرجع لبيئة العمل الخارجية الخاصة بها، مما يحفظ الوصول إلى المتغيرات في تلك البيئة حتى بعد انتهاء الدالة الخارجية.

الإغلاقات مفيدة بشكل خاص لإنشاء متغيرات ودوال خاصة. انظر هذا المثال:

:::interactive_editor

```js
function createCounter() {
    let count = 0;
    return function () {
        count++;
        return count;
    };
}

let counter = createCounter();
console.log(counter()); // 1
console.log(counter()); // 2
```

:::

في هذه الحالة، تُعيد `createCounter` دالة تزيد وتُعيد متغير `count`. متغير `count` غير متاح مباشرة من خارج `createCounter`، لكن الدالة المعادة (الإغلاق) لديها وصول إليه. في كل مرة نستدعي `counter()`، تزيد وتُعيد قيمة `count`.

يمكن للإغلاقات أيضًا التقاط عدة متغيرات من نطاقها الخارجي. على سبيل المثال:

:::interactive_editor

```js
function multiply(x) {
    return function (y) {
        return x * y;
    };
}

let double = multiply(2);
console.log(double(5)); // 10
```

:::

هنا تلتقط الدالة الداخلية المعلمة `x` من `multiply`. عندما ننشئ `double` باستدعاء `multiply(2)` فإنها تُرجع دالة تضرب دائمًا معلمتها في `2`.

أمر مهم يجب ملاحظته عن الإغلاقات هو أنها تلتقط المتغيرات بالمرجع وليس بالقيمة. هذا يعني أنه إذا تغيرت قيمة المتغير الملتقط، سيرى الإغلاق القيمة الجديدة. على سبيل المثال:

:::interactive_editor

```js
function createIncrementer() {
    let count = 0;
    return function () {
        count++;
        console.log(count);
    };
}

let increment = createIncrementer();
increment(); // 1
increment(); // 2
```

:::

في كل مرة نستدعي `increment`، يعمل مع نفس متغير `count`، وليس نسخة من قيمته الأولية. الإغلاقات أداة قوية في JavaScript. مع استمرارك في العمل مع JavaScript، ستجد أن فهم واستخدام الإغلاقات بفعالية يعزز قدرتك على كتابة شفرة نظيفة وفعالة وقوية.
