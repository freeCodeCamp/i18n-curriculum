## Costruttore stringa e metodo `toString()`

- **Definizione**: Un oggetto stringa viene usato per rappresentare una sequenza di caratteri. Gli oggetti stringa si creano usando la funzione costruttore `String`, che incapsula il valore primitivo in un oggetto.

:::interactive_editor

```js
const greetingObject = new String("Hello, world!");

console.log(typeof greetingObject); // "object"
```

:::

- **Metodo `toString()`**: Questo metodo converte un valore nella sua rappresentazione stringa. È un metodo che puoi usare per numeri, booleani, array e oggetti.

:::interactive_editor

```js
const num = 10;
console.log(num.toString()); // "10"

const arr = [1, 2, 3];
console.log(arr.toString()); // "1,2,3"
```

:::

Questo metodo accetta una base opzionale, che è un numero da 2 a 36. Questa base rappresenta la base numerica, come base 2 per il binario o base 8 per l’ottale. Se la base non è specificata, il valore predefinito è base 10, cioè decimale.


:::interactive_editor

```js
const num = 10;
console.log(num.toString(2)); // "1010"(binary)
```

:::

## Costruttore Number

- **Definizione**: Il costruttore `Number` serve a creare un oggetto numero. L’oggetto numero contiene alcune proprietà e metodi utili come il metodo `isNaN` e `toFixed`. La maggior parte delle volte userai il costruttore `Number` per convertire altri tipi di dati nel tipo numero.

:::interactive_editor

```js
const myNum = new Number("34");
console.log(typeof myNum); // "object"

const num = Number('100');
console.log(num); // 100

console.log(typeof num); // number
```

:::

## Buone pratiche per nominare variabili e funzioni

- **camelCasing**: Per convenzione, gli sviluppatori JavaScript usano il camel casing per nominare variabili e funzioni. Il camel casing consiste nel mettere la prima parola tutta in minuscolo e le parole successive con la prima lettera maiuscola. Es. `isLoading`.

- **Nominare booleani**: Per le variabili booleane, è pratica comune usare prefissi come "is", "has" o "can". 

```js
let isLoading = true;
let hasPermission = false;
let canEdit = true;
```

- **Nominare funzioni**: Per le funzioni, il nome dovrebbe indicare chiaramente cosa fa la funzione. Per le funzioni che restituiscono un booleano (spesso chiamate predicati), puoi usare gli stessi prefissi "is", "has" o "can". Quando hai funzioni che recuperano dati, è comune iniziare con la parola "get". Quando hai funzioni che impostano dati, è comune iniziare con la parola "set". Per le funzioni gestore di eventi, potresti usare il prefisso "handle" o il suffisso "Handler". 

```js
function getUserData() { /* ... */ }

function isValidEmail(email) { /* ... */ }

function getProductDetails(productId) { /* ... */ }

function setUserPreferences(preferences) { /* ... */ }

function handleClick() { /* ... */ }
```

- **Nominare variabili dentro i loop**: Quando nomini le variabili iteratrici nei loop, è comune usare lettere singole come `i`, `j` o `k`.

```js
for (let i = 0; i < array.length; i++) { /* ... */ }
```

## Lavorare con array sparsi

- **Definizione**: È possibile avere array con slot vuoti. Gli slot vuoti sono definiti come slot senza nulla dentro. Questo è diverso dagli slot dell’array con valore `undefined`. Questi tipi di array sono noti come array sparsi. 

:::interactive_editor

```js
const sparseArray = [1, , , 4];
console.log(sparseArray.length); // 4
```

:::

## Linters e formatter

- **Linters**: Un linter è uno strumento di analisi statica del codice che segnala errori di programmazione, bug, errori di stile e costrutti sospetti. Un esempio comune di linter è ESLint. 
- **Formatter**: I formatter sono strumenti che formattano automaticamente il codice per aderire a una guida di stile specifica. Un esempio comune di formatter è Prettier.

## Gestione della memoria

- **Definizione**: La gestione della memoria è il processo di controllo della memoria, assegnandola quando serve e liberandola quando non serve più. JavaScript usa la gestione automatica della memoria. Questo significa che JavaScript (più precisamente, il motore JavaScript nel tuo browser web) si occupa dell’assegnazione e della deallocazione della memoria per te. Non devi liberare esplicitamente la memoria nel tuo codice. Questo processo automatico è spesso chiamato "garbage collection".

## Closure

- **Definizione**: Una closure è una funzione che ha accesso alle variabili nel suo ambito lessicale esterno (inclusivo), anche dopo che la funzione esterna ha restituito il controllo. 

:::interactive_editor

```js
function outerFunction(x) {
  let y = 10;
  function innerFunction() {
    console.log(x + y);
  }
  return innerFunction;
}

let closure = outerFunction(5);
closure(); // 15
```

:::

## Parola chiave `var` e hoisting

- **Definizione**: `var` era il modo originale per dichiarare variabili prima del 2015. Ma c’erano alcuni problemi con `var` riguardo all’ambito, alla ridichiarazione e altro. Per questo motivo la programmazione moderna in JavaScript usa `let` e `const` invece.
- **Ridichiarare variabili con `var`**: Se provi a ridichiarare una variabile usando `let`, otterrai un `SyntaxError`. Ma con `var` puoi ridichiarare una variabile. 

```js
// Uncaught SyntaxError: Identifier 'num' has already been declared 
let num = 19;
let num = 18;

var myNum = 5;
var myNum = 10; // This is allowed and doesn't throw an error

console.log(myNum) // 10
```

- **`var` e ambito**: Le variabili dichiarate con `var` dentro un blocco (come un’istruzione `if` o un loop `for`) sono comunque accessibili fuori da quel blocco. 

:::interactive_editor

```js
if (true) {
  var num = 5;
}
console.log(num); // 5
```

:::

- **Hoisting**: Questo è il comportamento predefinito di JavaScript che sposta le dichiarazioni in cima ai rispettivi ambiti durante la fase di compilazione prima che il codice venga eseguito. Quando dichiari una variabile usando la parola chiave `var`, JavaScript sposta la dichiarazione in cima al suo ambito. 

:::interactive_editor

```js
console.log(num); // undefined
var num = 5;
console.log(num); // 5
```

:::

Quando dichiari una funzione usando la sintassi di dichiarazione della funzione, sia il nome della funzione sia il corpo della funzione vengono spostati in cima. Questo significa che puoi chiamare una funzione prima di averla dichiarata nel codice.

:::interactive_editor

```js
sayHello(); // "Hello, World!"

function sayHello() {
  console.log("Hello, World!");
}
```

:::

Le dichiarazioni di variabili fatte con `let` o `const` vengono spostate in cima, ma non sono inizializzate, e non puoi consultarle prima della dichiarazione effettiva nel codice. Questo comportamento è spesso chiamato "zona morta temporale".

:::interactive_editor

```js
console.log(num); // Throws a ReferenceError
let num = 10;
```

:::

## Lavorare con import, export e moduli

- **Modulo**: È un’unità autonoma di codice che incapsula funzioni, classi o variabili correlate. Per creare un modulo, scrivi il tuo codice JavaScript in un file separato.
- **Export**: Qualsiasi variabile, funzione o classe che vuoi rendere disponibile ad altre parti della tua applicazione deve essere esportata esplicitamente usando la parola chiave `export`. Ci sono due tipi di export: export nominato e export predefinito.
- **Import**: Per usare gli elementi esportati in un’altra parte della tua applicazione, devi importarli usando la parola chiave `import`. I tipi possono essere import nominato, import predefinito e import namespace.

```js
// Within a file called math.js, we export the following functions:

// Named export
export function add(num1, num2) {
  return num1 + num2;
}

// Default export
export default function subtract(num1, num2) {
  return num1 - num2;
}

// Within another file, we can import the functions from math.js.

// Named import - This line imports the add function.
// The name of the function must exactly match the one exported from math.js.
import { add } from './math.js';

// Default import - This line imports the subtract function.
// The name of the function can be anything.
import subtractFunc from './math.js';

// Namespace import - This line imports everything from the file.
import * as Math from './math.js';

console.log(add(5, 3)); // 8
console.log(subtractFunc(5, 3)); // 2
console.log(Math.add(5, 3)); // 8
console.log(Math.subtract(5, 3)); // 2
```
