# --interactive--

在上一課中，我們學習了像素等絕對長度單位。雖然絕對單位在某些情況下很有幫助，但有時你會想使用相對單位。

在本課程中，我們將學習兩個相對單位：`em`s 和 `rem`s。

`em` 單位是相對於該元素的字型大小。如果你在 `font-size` 屬性中使用 em，文字的大小將相對於父元素的字型大小。

為了更好地理解這個運作方式，讓我們來看一個範例：

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css" />
<p class="para">I am a paragraph element</p>

<div class="blue-box"></div>
```

```css
.para {
  font-size: 20px;
  margin-bottom: 1.5em;
  border: 2px solid red;
}

.blue-box {
  background-color: blue;
  color: white;
  padding: 10px;
  width: 100px;
  height: 100px;
}
```

:::

對於 HTML，我們有一個段落和一個 `div` 元素。段落元素有一個 `para` 類別，而 `div` 元素有一個 `blue-box` 類別。

對於 `para` 類別，我們將 `font-size` 設定為 `20px`，並將 `margin-bottom` 設定為 `1.5em`。這表示邊距將是段落元素字型大小的 1.5 倍。`1.5em` 會導致段落底部的邊距為 30 像素。我們也設定了 `2px solid red` 的 `border`，以便你能更清楚地看到邊距。 

對於 `blue-box` 類別，我們將背景顏色設為 `blue`，文字顏色設為 `white`，並將 `padding` 設為四邊各 10px。

從範例中，可以看到段落元素底部與藍色框之間會有明顯的空白。

那麼如果我們從 `para` 類別中移除 `font-size` 屬性會發生什麼？

底部邊距將相對於父元素的字型大小。在這個案例中，父元素是 body 元素，其字型大小的預設值為 `16px`。

`em` 的良好使用案例是在你處理像是按鈕或卡片這類模組化組件時。透過使用 `em` 單位，你可以確保組件的所有面向（例如內距、外距和邊框）會隨著字型大小成比例縮放，保持一致的比例。

所以，到目前為止，我們一直使用像素來設定元素的字型大小。但這確實會為使用者帶來一些挑戰。

在你的瀏覽器設定中，你可以控制預設的字型大小。

對於視覺障礙者，他們可能會增加字型大小以便更容易閱讀。但如果你在網頁設計中為字型大小設定像素，文字將不會與其他內容成比例縮放。

解決此問題的一種方法是為排版使用 `rem` 單位。`rem` 單位是相對於根元素的字型大小，也就是 `html` 元素。 

預設值中，`html` 元素的字型大小為 `16px`。如果使用者在瀏覽器設定中增加字型大小，`html` 元素的字型大小將會增加，且所有 rem 單位將會按比例縮放。

以下是使用 `rem` 單位作為字型大小而非像素的範例：

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css" />
<p>This is regular text.</p>
<p class="para">This text is slightly larger.</p>
```

```css
.para {
  font-size: 1.2rem;
  margin-bottom: 1.5em;
  border: 2px solid red;
}
```

:::

透過將字型大小設定為 `1.2rem`，段落元素的字型大小將是根元素字型大小的 1.2 倍。如果使用者尚未更改預設字型大小，段落元素的字型大小將是 `19.2px`，因為它是 `16px` 的 1.2 倍。

那麼你應該在什麼時候使用 `rem` 單位？`rem` 單位比像素更適合用於排版，因為它們會隨著使用者瀏覽器設定成比例縮放。這使你的內容對視覺障礙使用者更具可及性。

`rem` 單位也有助於維護不同元素之間的一致間距和版面配置。
