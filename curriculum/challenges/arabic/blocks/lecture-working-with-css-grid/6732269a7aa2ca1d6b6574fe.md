# --interactive--

تشير الشبكة الضمنية إلى الصفوف والأعمدة التي ينشئها المتصفح تلقائيًا عند وضع العناصر في تخطيط شبكة – تلك التي لم تُحدد صراحة باستخدام `grid-template-rows` أو `grid-template-columns`.

الخصائص التي تتحكم في الأعمدة والصفوف التي ينشئها المتصفح ضمنيًا هي `grid-auto-columns` و`grid-auto-rows`.

تشير الشبكة الضمنية أيضًا إلى الصفوف والأعمدة الإضافية التي يولدها المتصفح تلقائيًا عندما تضع عنصرًا خارج الصفوف والأعمدة المحددة صراحة.

على سبيل المثال، لنفترض أنك تحدد عمودين صريحين فقط في تخطيط الشبكة بهذه الطريقة:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<div class="grid-container">
  <div class="grid-item">Item 1</div>
  <div class="grid-item">Item 2</div>
  <div class="grid-item">Item 3</div>
  <div class="grid-item">Item 4</div>
  <div class="grid-item">Item 5</div>
  <div class="grid-item">Item 6</div>
</div>
```

```css
.grid-container {
  display: grid;
  grid-template-columns: 100px 100px; /* Only 2 explicit columns */
}

.grid-item {
  background-color: burlywood;
  border: 1px solid orangered;
  padding: 0.5rem;
  margin: 0.5rem;
}
```

:::

ملأت عنصران الصف الأول باستخدام العمودين الصريحين – `Item 1` في العمود الأول و`Item 2` في العمود الثاني.

تبدأ العناصر التالية صفًا جديدًا مع وضع `Item 3` في العمود الأول من الصف الثاني و`Item 4` في العمود الثاني من الصف الثاني، وهكذا.

كما رأيت بالفعل، الشبكات الصريحة هي مناطق الشبكة التي تنشئها عمدًا. أي الصفوف والأعمدة التي تحددها صراحة لتخطيط الشبكة باستخدام خصائص `grid-template-rows` و`grid-template-columns`.

إليك جدول يلخص الفروقات بين الشبكات الضمنية والصريحة:

| **الميزة**          | **الشبكة الصريحة**                                                                  | **الشبكة الضمنية**                                                                          |
|----------------------|------------------------------------------------------------------------------------|--------------------------------------------------------------------------------------------|
| **التحكم في الحجم**     | قابل للتخصيص بالكامل باستخدام  ` grid-template-rows `  و  ` grid-template-columns ` . | يتحكم به  ` grid-auto-rows `  و  ` grid-auto-columns ` ، أو الافتراضي هو  ` auto ` . |
| **السلوك الافتراضي** | لا يتغير إلا إذا تم تعريفه صراحة.                                         | يتكيف تلقائيًا مع العناصر الموضوعة خارج الشبكة الصريحة.                            |
| **التعقيد**       | يتطلب تخطيطًا أكثر لبنية التخطيط.                                       | أسهل في التنفيذ للمحتوى غير المنظم أو المتغير.                                  |
| **المرونة**      | بنية جامدة مع صفوف وأعمدة محددة.                                     | مرن ويتكيف مع المحتوى الموضوع ديناميكيًا.                                         |
| **الأداء**      | قد يكون أكثر كفاءة لأن التخطيط محدد مسبقًا.                           | قد يتطلب حسابات إضافية من المتصفح لتوليد الشبكة الضمنية.                  |
| **حالة الاستخدام**         | مفيد عندما تكون بنية الشبكة متوقعة ومحددة مسبقًا.                 | مفيد للتخطيطات الديناميكية حيث المحتوى غير معروف أو يتغير كثيرًا.                 |
