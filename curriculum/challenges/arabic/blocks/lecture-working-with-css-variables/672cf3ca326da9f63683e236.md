# --interactive--

قاعدة `@property` هي ميزة قوية في CSS تتيح للمطوّرين تعريف خصائص مخصصة مع تحكم أكبر في سلوكها، بما في ذلك كيفية تحريكها وقيمها الأولية.

تقدم هذه القاعدة طريقة لتعزيز وظيفة الخصائص المخصصة في CSS وتوفر مرونة أكبر في تطبيقها.

الصيغة الأساسية لقاعدة `@property` هي كما يلي:

```css
@property --property-name {
  syntax: '<type>';
  inherits: true | false;
  initial-value: <value>;
}
```

`--property-name` هو اسم الخاصية المخصصة التي تعرفها. مثل جميع الخصائص المخصصة، يجب أن يبدأ بشرطتين.

`syntax` يحدد نوع الخاصية، والذي يمكن أن يكون مثل `<color>`، `<length>`، `<number>`، `<percentage>`، أو أنواع أكثر تعقيدًا.

`inherits` يحدد ما إذا كانت الخاصية يجب أن ترث قيمتها من العنصر الأب.

`initial-value` يحدد القيمة الافتراضية للخاصية.

إليك مثال عملي على استخدام قاعدة `@property`:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<button class="button">Click Me</button>
```

```css
@property --main-color {
  syntax: '<color>';
  inherits: false;
  initial-value: #3498db;
}

.button {
  background-color: var(--main-color);
}
```

:::

في هذا المثال، نُعرّف خاصية مخصصة `--main-color` كقيمة `color`، نحدد أنها لا ترث، ونعطيها قيمة أولية `#3498db`.

واحدة من الفوائد الرئيسية لقاعدة `@property` هي أنها تسمح بتحريك الخصائص المخصصة، وهو أمر لم يكن ممكنًا من قبل:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<div class="gradient-box"></div>
```

```css
@property --gradient-angle {
  syntax: '<angle>';
  inherits: false;
  initial-value: 0deg;
}

.gradient-box {
  width: 100px;
  height: 100px;
  background: linear-gradient(var(--gradient-angle), red, blue);
  transition: --gradient-angle 0.5s;
}

.gradient-box:hover {
  --gradient-angle: 90deg;
}
```

:::

الكود أعلاه ينشئ تدرجًا لونيًا يتحرك بسلاسة عند تحويم المؤشر فوق العنصر، وهو أمر لم يكن ممكنًا مع الخصائص المخصصة القياسية.

الآن، دعنا نناقش كيف تعمل قاعدة `@property` مع الأنظمة البديلة.

الأنظمة البديلة ضرورية في CSS لضمان تدهور الأنماط بسلاسة في المتصفحات التي لا تدعم ميزات معينة. مع `@property`، تعمل الأنظمة البديلة على مستويين: للقاعدة نفسها وللاستخدام الخاص بالخاصية المخصصة.

للمتصفحات التي لا تدعم قاعدة `@property`، يمكنك توفير نظام بديل بإعلان الخاصية المخصصة بالطريقة التقليدية:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
```

```css
:root {
  --main-color: #3498db;
}

@property --main-color {
  syntax: '<color>';
  inherits: false;
  initial-value: #3498db;
}

body {
  background-color: var(--main-color);
}
```

:::

في هذه الحالة، المتصفحات التي تدعم `@property` ستستخدم النسخة المعرفة بشكل أكثر دقة، بينما ستعود المتصفحات الأخرى إلى إعلان الخاصية المخصصة القياسي.

عند استخدام الخاصية المخصصة، يمكنك توفير قيمة بديلة باستخدام دالة `var()`، تمامًا كما تفعل مع الخصائص المخصصة القياسية:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<button class="button">Click Me</button>
```

```css
.button {
  background-color: var(--main-color, #3498db);
}
```

:::

هذا يضمن أنه حتى إذا لم تُعرف خاصية `--main-color` أو كانت غير صالحة، سيظل الزر يحتوي على لون خلفية.

قاعدة `@property` تسمح أيضًا بسيناريوهات أنظمة بديلة أكثر تعقيدًا. على سبيل المثال، يمكنك استخدامها لتوفير أنظمة بديلة آمنة من حيث النوع:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<div class="box">Colored box</div>
```

```css
@property --padding {
  syntax: '<length>';
  inherits: false;
  initial-value: 10px;
}

.box {
  width: 100px;
  height: 100px;
  background-color: darkred;
  color: white;
  padding: var(--padding);
}
```

:::

في هذه الحالة، إذا حاول أحدهم تعيين `--padding` إلى قيمة غير صالحة، مثل لون، سيعود المتصفح إلى القيمة الأولية `10px`، محافظًا على سلامة النوع.

قاعدة CSS `@property` تعتبر متاحة على نطاق واسع عبر جميع محركات المتصفحات الحديثة الرئيسية (مثل Chrome وEdge وOpera وFirefox وSafari) منذ أوائل 2024. إذا أردت دعم متصفحات أقدم، يجب توفير CSS بديل كما هو موضح أعلاه.

في الختام، تمثل قاعدة `@property` تقدمًا كبيرًا في كيفية التعامل مع الخصائص المخصصة في CSS. توفر تحكمًا أكبر وتمكن إمكانيات جديدة مثل تحريك الخصائص المخصصة. مع ذلك، من المهم استخدامها بحكمة وتوفير أنظمة بديلة مناسبة لضمان تجربة جيدة عبر جميع المتصفحات. كما هو الحال مع أي تقنية ويب متقدمة، المفتاح هو تحسين التجربة للمتصفحات التي تدعمها مع ضمان الوظائف الأساسية لمن لا تدعمها.
