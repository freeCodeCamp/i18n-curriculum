## الأخطاء الشائعة في بايثون

- **SyntaxError**: هذا هو الخطأ الذي يظهر في بايثون عندما لا تتبع الشيفرة قواعد الصياغة النحوية الخاصة بها. على سبيل المثال، الشيفرة `print("Hello there"` ستؤدي إلى ظهور خطأ نحوي مع الرسالة `SyntaxError: '(' was never closed`، لأن الشيفرة تفتقد القوس المغلق.
- **NameError**: يظهر خطأ `NameError` في بايثون عندما تحاول الوصول إلى متغير أو دالة لم تقم بتعريفها. على سبيل المثال، إذا كتبت السطر `print(username)` في الشيفرة دون أن تعرّف متغير باسم `username` أولًا، ستحصل على خطأ باسم `NameError: name 'username' is not defined`.
- **TypeError**: هذا الخطأ يظهر عندما تحاول تنفيذ عملية بين نوعين من البيانات غير المتوافقة. مثلًا، إذا حاولت جمع سلسلة نصية مع رقم، ستحصل على الخطأ `TypeError: can only concatenate str (not "int") to str`.
- **IndexError**: ستحصل على خطأ `IndexError` إذا حاولت الوصول إلى فهرس غير موجود في قائمة أو متتالية أخرى مثل الصف المرتب أو السلسلة النصية. على سبيل المثال، في سلسلة نصية `Hello world`، فهرس آخر حرف هو `11`. إذا حاولت الوصول إلى حرف بهذا الشكل: `greet = "hello world"; print(greet[12])`، سيظهر الخطأ `IndexError: string index out of range`.
- **AttributeError**: يظهر هذا الخطأ عندما تحاول استخدام طريقة أو خاصية غير موجودة في كائن من هذا النوع. مثلًا، إذا استدعيت `.append()` على سلسلة نصية بهذا الشكل: `"hello".append("!")`، سيظهر الخطأ `AttributeError: 'str' object has no attribute 'append'`.

## تقنيات جيدة لتصحيح الأخطاء في Python

- **استخدام دالة `print`**: إدراج عبارات `print()` في نقاط مختلفة من شفرتك أثناء تصحيح الأخطاء يساعدك على رؤية قيم المتغيرات وكيفية تدفق الشفرة.
- **استخدام مصحح الأخطاء المدمج في بايثون (`pdb`)**: توفر بايثون وحدة `pdb` لتصحيح الأخطاء. هي جزء من مكتبة بايثون القياسية، لذا فهي متاحة دائمًا للاستخدام. مع `pdb`، يمكنك تعيين نقطة تتبع باستخدام الدالة `set_trace()` لتبدأ في تنفيذ الشفرة خطوة بخطوة وفحص المتغيرات بطريقة تفاعلية.
- **الاستفادة من أدوات تصحيح الأخطاء في بيئات التطوير المتكاملة**: تقدم العديد من بيئات التطوير المتكاملة (IDEs) ومحررات الشفرة مثل Pycharm وVS Code أدوات تصحيح أخطاء تشمل نقاط توقف، تنفيذ خطوة بخطوة، فحص المتغيرات، وميزات تصحيح أخرى.

## التعامل مع الاستثناءات

- **`try...except`**: تُستخدم هذه البنية لتنفيذ كتلة من الشفرة قد تثير استثناءً. كتلة `try` هي المكان الذي تتوقع فيه حدوث خطأ، بينما كتلة `except` تتعامل مع استثناء محدد وتُنفذ إذا حدث هذا الخطأ المحدد. إليك مثالاً:

  ```py
  try:
    print(22 / 0)
  except ZeroDivisionError:
    print('You can\'t divide by zero!')
    # You can't divide by zero!
  ```

يمكنك أيضًا ربط عدة كتل `except` لتتعامل مع أنواع أكثر من الاستثناءات:

  ```py
  try:
    number = int(input('Enter a number: '))
    print(22 / number)
  except ZeroDivisionError:
    print('You cannot divide by zero!')
    # You cannot divide by zero! prints when you enter 0
  except ValueError:
    print('Please enter a valid number!')
    # Please enter a valid number! prints when you enter a string  
  ```

- **`else` و `finally`**: توسع هذه الكتل عمل `try...except`. إذا لم يحدث استثناء، ينفذ كتلة `else`. كتلة `finally` تنفذ دائمًا بغض النظر عن وجود أخطاء.

  ```py
  try:
    result = 100 / 4
  except ZeroDivisionError:
    print('You cannot divide by zero!') # This will not run
  else:
    print(f'Result is {result}') # Result is 25.0
  finally:
    print('Execution complete!') # Execution complete!
  ```

- **كائن الاستثناء**: يتيح لك هذا الوصول إلى الاستثناء نفسه لتحسين تصحيح الأخطاء وطباعة رسالة الخطأ المباشرة. للوصول إلى كائن الاستثناء، يجب استخدام الكلمة المفتاحية `as`. إليك مثالاً:

  ```py
  try:
      value = int('This will raise an error')
  except ValueError as e:
      print(f'Caught an error: {e}')
      # Caught an error: invalid literal for int() with base 10: 'This will raise an error'
  ```

- **عبارة `raise`**: تتيح لك رفع استثناء يدويًا. يمكنك استخدامها لرمي استثناء عند تحقق شرط معين. إليك مثالًا:

  ```py
  def divide(a, b):
      if b == 0:
          raise ZeroDivisionError('You cannot divide by zero')
      return a / b
  ```
  
## الإشارة إلى الاستثناءات
  
تُستخدم عبارة `raise` أيضًا عند إنشاء استثناءات مخصصة خاصة بك، إذ يمكنك استخدامها لرمي استثناء مع رسالة مخصصة. إليك مثالًا على ذلك:

  ```py
  class InvalidCredentialsError(Exception):
      def __init__(self, message="Invalid username or password"):
          self.message = message
          super().__init__(self.message)

  def login(username, password):
      stored_username = "admin"
      stored_password = "password123"
      
      if username != stored_username or password != stored_password:
          raise InvalidCredentialsError()
      
      return f"Welcome, {username}!"
  ```  

إليك كيفية استخدام دالة `login` مع استثناء `InvalidCredentialsError`:

  ```py
  # failed login attempt
  try:
      message = login("user", "wrongpassword")
  except InvalidCredentialsError as e:
      print(f"Login failed: {e}")
  else:
      print(message)

  # successful login attempt
  try:
      message = login("admin", "password123")
  except InvalidCredentialsError as e:
      # This block is not executed because the login was successful
      print(f"Login failed: {e}")
  else:
      # The else block runs if the 'try' block completes without an exception
      print(message)
  ```

يمكن أيضًا استخدام جملة `raise` مع الكلمة المفتاحية `from` لربط الاستثناءات، مما يوضح العلاقة بين الأخطاء المختلفة:

  ```py
  def parse_config(filename):
    try:
        with open(filename, 'r') as file:
            data = file.read()
            return int(data)
    except FileNotFoundError:
        raise ValueError('Configuration file is missing') from None
    except ValueError as e:
        raise ValueError('Invalid configuration format') from e

  config = parse_config('config.txt')
  ```
