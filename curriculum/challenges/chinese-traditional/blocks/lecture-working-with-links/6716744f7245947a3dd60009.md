# --interactive--

你可能看過錨點元素或連結上的 `target` 屬性。這個重要的屬性告訴瀏覽器在哪裡開啟錨點元素的 URL。

啟用互動式編輯器，點擊該聯結，你將在新的瀏覽器分頁中被導向 freeCodeCamp 首頁。

:::interactive_editor

```html
<a href="https://freecodecamp.org" target="_blank">Visit freeCodeCamp</a>
```

:::

這個屬性有四個重要的可能值。請注意，每個值前面都有一個底線。

第一個值是 `_self`，這是預設值。這會在目前的瀏覽背景關係中開啟聯結。在大多數情況下，這將是目前的分頁或視窗。

第二個值是 `_blank`，會在新的瀏覽背景關係中開啟聯結。通常，這會在新的分頁中開啟。但有些使用者可能會組態他們的瀏覽器改為開啟新的視窗。

第三個值是 `_parent`，它會在目前背景關係的父層中開啟聯結。 例如，如果你的網站有一個 `iframe`，該 `iframe` 中的 `_parent` 值會在你的網站分頁/視窗中開啟，而不是在嵌入的影格中。

第四個值是 `_top`，它會在最頂層的瀏覽背景關係中開啟聯結——想像成「父層的父層」。這與 `_parent` 類似，但聯結將始終在完整的瀏覽器分頁/視窗中開啟，即使是巢狀嵌入的訊框。

還有第五個值，稱為 `_unfencedTop`，目前用於實驗性的 FencedFrame API。在本課程進行時，你可能還沒有理由使用這個值。

選擇正確的 `target` 值來控制你的使用者最終所在位置，是建立網站時一項重要的考量。
