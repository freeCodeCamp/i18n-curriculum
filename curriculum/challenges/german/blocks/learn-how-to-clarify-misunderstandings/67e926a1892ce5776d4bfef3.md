# --explanation--

`Reducing complexity` bedeutet, ein System leichter verständlich und handhabbar zu machen.
  
`Showing only what's necessary` bedeutet, dass der Benutzer nur die Teile sieht, mit denen er interagieren muss.
  
`Keeping the details hidden` bezieht sich darauf, internen Code oder Logik zu verbergen, die der Benutzer nicht benötigt.

Zusammengefasst schafft Abstraktion eine klare, einfache Schnittstelle, während die komplexeren inneren Abläufe im Hintergrund verborgen bleiben.
