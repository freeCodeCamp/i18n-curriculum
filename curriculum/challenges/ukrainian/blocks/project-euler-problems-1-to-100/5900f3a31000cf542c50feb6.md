# --description--

Якщо взяти 47, перевернути та додати, 47 + 74 = 121, що є паліндромом.

Не всі числа так швидко утворюють паліндроми. Наприклад,

<div style="margin-left: 4em;">
  349 + 943 = 1292,<br>
  1292 + 2921 = 4213<br>
  4213 + 3124 = 7337<br>
</div>

Тобто, 349 знадобилося три ітерації, щоб отримати паліндром.

Хоча ніхто ще не довів це, вважається, що деякі числа, як-от 196, ніколи не утворюють паліндром. Число, яке ніколи не утворює паліндром через процес перевертання і додавання, називають числом Лічрел. Через теоретичний характер цих чисел, для цілей цього завдання ми припускаємо, що число є Лічрелом, доки не доведено протилежне. Крім того, вам відомо, що для кожного числа менше за десять тисяч воно або (i) стане паліндромом менш ніж за п’ятдесят ітерацій, або (ii) ніхто, навіть із усією наявною обчислювальною потужністю, досі не зміг спроєктувати його у паліндром. Насправді, 10677 — перше число, для якого показано, що потрібно понад п’ятдесят ітерацій, щоб отримати паліндром: 4668731596684224866951378664 (53 ітерації, 28 цифр).

Дивовижно, але існують паліндромні числа, які самі є числами Лічрел; перший приклад — 4994.

Скільки чисел Лічрел існує менше за `num`?

**Примітка:** Формулювання було трохи змінене 24 квітня 2007 року, щоб підкреслити теоретичний характер чисел Лічрел.

```js
const countLychrelNumbers = (size) => {
  const numReverse = (num) => {
    return Number(num.toString().split('').reverse().join(''));
  };
  const isPalin = (num) => {
    if (numReverse(num) === num) {
      return true;
    }
    return false;
  };
  let total = 0;
  for (let i = 1; i < size; i++) {
    let loopCount = 1;
    let sum = i;
    while (loopCount < 50) {
      sum = sum + numReverse(sum);
      if (isPalin(sum)) {
        break;
      } else {
        loopCount++;
      }
    }
    if (loopCount === 50) {
      total++;
    }
  }
  return total;
}
```
