# --description--

Una fila que mide `n` unidades de longitud tiene bloques rojos con una longitud mínima de `m` unidades colocados sobre ella, de manera que cualquier par de bloques rojos (que pueden tener longitudes diferentes) están separados por al menos un cuadrado negro.

Deja que la función fill-count, $F(m, n)$, represente el número de formas en que se puede llenar una fila.

Por ejemplo, $F(3, 29) = 673135$ y $F(3, 30) = 1089155$.

Es decir, para m = 3, se puede ver que n = 30 es el valor más pequeño para el cual la función fill-count supera por primera vez un millón.

De la misma manera, para m = 10, se puede verificar que $F(10, 56) = 880711$ y $F(10, 57) = 1148904$, por lo que n = 57 es el valor mínimo para el cual la función fill-count supera por primera vez un millón.

Para m = 50, encuentra el valor mínimo de `n` para el cual la función fill-count supera por primera vez un millón.

**Nota:** Esta es una versión más difícil del Problema 114.
