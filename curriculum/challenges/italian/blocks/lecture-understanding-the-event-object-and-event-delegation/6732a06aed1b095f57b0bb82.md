# --interactive--

L’evento `change` è un evento speciale che viene attivato quando l’utente modifica il valore di certi elementi di input. Più precisamente:

- Quando una casella di spunta viene selezionata o deselezionata.

- Quando un pulsante di opzione viene selezionato.

- Quando l’utente fa una selezione da qualcosa come un selettore di date o un menu a discesa.

- Quando un input perde il focus (l’utente passa al campo successivo con il tab o clicca fuori dal modulo) dopo aver cambiato il valore.

- Quando l’utente conferma altrimenti il valore, ad esempio premendo invio dopo aver digitato del testo.

Nota che l’evento `change` NON si attiva mentre l’utente digita in un input. L’evento `change` si attiverà solo dopo che l’utente ha messo il focus su un altro elemento.

Ecco un esempio di utilizzo dell’evento `change` con un menu a discesa:

:::interactive_editor

```html
<select id="select-menu">
  <option value="option1">Option 1</option>
  <option value="option2">Option 2</option>
  <option value="option3">Option 3</option>
</select>
<script src="index.js"></script>
```

```js
const selectMenu = document.getElementById("select-menu");
selectMenu.addEventListener("change", (event) => {
  console.log(`You selected: ${event.target.value}`);
});
```

:::

Ogni volta che l’utente seleziona un’opzione diversa dal menu a discesa, l’evento `change` si attiverà e il valore selezionato verrà registrato nella console.

L’evento `change` genera comunque un oggetto `Event`, ma a differenza della maggior parte degli altri eventi non genera un’implementazione personalizzata – le uniche proprietà e metodi a cui potrai accedere sono quelli dell’oggetto base `Event`.

Questo differisce dall’evento `input`, che genera un oggetto `InputEvent` dedicato. L’evento change differisce anche in altri modi. Un evento `input` SI attiverà, per esempio, quando un utente digita contenuto in un campo.

Queste differenze sono importanti da ricordare, perché potresti confonderti con il momento in cui questi eventi si attivano.
