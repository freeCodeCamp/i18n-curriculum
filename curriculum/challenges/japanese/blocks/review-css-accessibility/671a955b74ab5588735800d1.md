## カラーコントラストツール

- **WebAIMのColor Contrast Checker**：このオンラインツールでは、デザインの前景色と背景色を入力すると、Webコンテンツアクセシビリティガイドライン（WCAG）の基準を満たしているかどうかをすぐに確認できます。
- **TPGi Colour Contrast Analyzer**：これは無料のカラーコントラストチェッカーで、ウェブサイトやアプリがWCAGの基準を満たしているかどうかをチェックできます。このツールには色覚異常シミュレーター機能もあり、色覚に問題がある人がウェブサイトやアプリをどのように見ているかを確認できます。

## アクセシビリティツリー

アクセシビリティツリーは、スクリーンリーダーなどの支援技術がウェブページのコンテンツを解釈し操作するために使う構造です。

## `max()` 関数

`max()` 関数は、カンマ区切りの値の中で最大の値を返します：

:::interactive_editor

```html
<head>
  <link rel="stylesheet" href="styles.css">
</head>
<body>
  <img src="https://cdn.freecodecamp.org/curriculum/css-photo-gallery/9.jpg" alt="A cat sitting on a table">
</body>
```

```css
img {
  width: max(250px, 25vw);
}
```

:::

上の例では、ビューポートの幅が1000ピクセル未満の場合、画像の幅は250pxになります。ビューポートの幅が1000ピクセルより大きい場合、画像の幅は25vwになります。これは25vwがビューポート幅の25％に相当するためです。

## CSSとアクセシビリティのベストプラクティス

- **`display: none;`**：`display: none;` を使うと、スクリーンリーダーやその他の支援技術はこのコンテンツにアクセスできなくなります。なぜならアクセシビリティツリーに含まれないためです。したがって、視覚的な表示とアクセシビリティの両方からコンテンツを完全に除外したい場合にのみ、この方法を使うことが重要です。
- **`visibility: hidden;`**：このプロパティと値は、コンテンツを視覚的に隠しますが、ドキュメントのレイアウト上はスペースを占有し続けます。これらの要素はアクセシビリティツリーからも除外されるため、スクリーンリーダーによる読み上げもされなくなります。
- **`.sr-only` CSSクラス**：これは、コンテンツを視覚的に隠しつつスクリーンリーダーにはアクセス可能にする一般的な手法です。

:::interactive_editor

```html
<head>
  <link rel="stylesheet" href="styles.css">
</head>
<body>
  <span>This span element can be read</span>
  <span class="sr-only">This span element can only be read by screen readers</span>
  <p style="display: none">
    This text is hidden with display: none and is NOT accessible to screen readers.
  </p>
  <p style="visibility: hidden">
    This text is hidden with visibility: hidden and is NOT read by screen readers,
    but it still takes up space.
  </p>
</body>

```

```css
.sr-only {
  position: absolute;
  width: 1px;
  height: 1px;
  padding: 0;
  margin: -1px;
  overflow: hidden;
  clip: rect(0, 0, 0, 0);
  white-space: nowrap;
  border: 0;
}

```

:::

- **`scroll-behavior: smooth;`**：このプロパティと値はスムーズスクロールの動作を有効にします。
- **`prefers-reduced-motion` 機能**：これはユーザーのアニメーション設定の好みを検出するために使えるメディア機能です。

## HTML属性によるコンテンツの非表示

- **`aria-hidden` 属性**：スクリーンリーダーなどの支援技術を使う人から要素を隠すために使います。例えば、意味のない装飾画像を隠す場合に使えます。
- **`hidden` 属性**：この属性はほとんどの最新ブラウザでサポートされており、コンテンツを視覚的にもアクセシビリティツリーからも隠します。JavaScriptで簡単に切り替えられます。

:::interactive_editor

```html
<p aria-hidden>This paragraph is visible for sighted users, but is hidden from assistive technology.</p>
<p hidden>This paragraph is hidden from both sighted users and assistive technology.</p>
<p>This is a normal paragraph</p>

```

:::

## `placeholder` 属性のアクセシビリティ問題

プレースホルダーテキストの使用はアクセシビリティにあまり良くありません。ユーザーがプレースホルダーテキストを実際の入力値と混同し、すでに値が入力されていると思い込むことがよくあります。
