# --description--

عرض القوائم مهمة أساسية في تطبيقات الويب باستخدام React، وتُستخدم لعرض البيانات للمستخدمين. في React، تُستخدم طريقة `map` لتحويل مصفوفة من البيانات إلى مصفوفة من عناصر JSX يمكن عرضها في واجهة المستخدم.

إليك مثالًا على مكوِّن يسمى `FruitList` يعرض قائمة من الفواكه:

```jsx
function FruitList() {
  const fruits = ['Apple', 'Banana', 'Cherry', 'Date'];
  return (
    <ul>
      {fruits.map(fruit => <li>{fruit}</li>)}
    </ul>
  );
}
```

في هذا المثال، تقوم دالة `map` بالتكرار على كل عنصر في مصفوفة `fruits`. لكل ثمرة، تُنشئ عنصر `li` جديد يحتوي على اسم الثمرة. ثم تُعرض مصفوفة عناصر `li` الجديدة داخل وسم الأب `ul`.

مع ذلك، عند عرض القوائم في React، من المهم ألا تنسى خاصية `key` لكل عنصر في القائمة. يجب أن تكون القيمة فريدة دائمًا، وتساعد React على تحديد العناصر التي تغيرت أو أُضيفت أو أُزيلت، وهذا ضروري لعرض القائمة وتحديثها بكفاءة.

إذا نسيت خاصية key، ستظهر تحذيرات في وحدة التحكم، لكنها لن تسبب خطأً قاتلًا. قد يستمر التطبيق في العرض والعمل، لكن قد تواجه أخطاء برمجية دقيقة، خاصة عند تغير القائمة. هذه الأخطاء قد يصعب معالجتها لأن واجهة المستخدم قد تبدو صحيحة في البداية.

خطأ شائع هو استخدام فهرس المصفوفة كـ key، مثل هذا:

```jsx
{fruits.map((fruit, index) => <li key={index}>{fruit}</li>)}
```

رغم أن هذا يُخمد التحذير، إلا أنه يُعتبر نمطًا خاطئًا عمومًا. استخدام الفهرس كـ key قد يسبب مشاكل عند إعادة ترتيب القائمة أو فرزها أو تصفيتها. تستخدم React الخاصية key لتتبع العناصر. إذا تغير ترتيب القائمة، قد تعيد React استخدام حالة المكوِّن بشكل خاطئ أو تفشل في تحديث DOM بكفاءة لأن المفاتيح (الفهارس) تبقى نفسها حتى لو تغير المحتوى في ذلك الفهرس.

أفضل الممارسات هي استخدام معرف فريد ومستقر لكل عنصر. عادةً يكون هذا معرفًا من قاعدة البيانات، مثل UUID أو معرف قاعدة بيانات.

إذا لم يكن لديك معرف فريد في بياناتك، يمكنك توليد واحد عند إنشاء البيانات (مثلًا باستخدام `crypto.randomUUID()` أو مكتبة مثل `uuid`)، أو استخدام تركيبة من الحقول التي تضمن التفرد. مع ذلك، يجب تجنب توليد المفاتيح أثناء العرض (مثلًا `key={Math.random()}`)، لأن ذلك يجعل React يعيد إنشاء عناصر DOM في كل عرض ويعيد تعيين حالتها.

لنُعدّل مثالنا ليشمل المفاتيح:

```jsx
function FruitList() {
  const fruits = ["Apple", "Banana", "Cherry", "Date"];
  return (
    <ul>
      {fruits.map((fruit, index) => (
        <li key={`${fruit}-${index}`}>{fruit}</li>
      ))}
    </ul>
  );
}
```

في هذا المثال المعاد هيكلته، نُنشئ مفتاحًا فريدًا لكل عنصر في القائمة بدمج اسم الثمرة مع فهرسها. هذا يضمن أن لكل عنصر مفتاح مميز، مما يساعد React على إدارة القائمة وتحديثها بكفاءة عند إضافة أو إزالة أو إعادة ترتيب العناصر.

تسمح React أيضًا بعرض هياكل أكثر تعقيدًا. على سبيل المثال، قد يكون لديك مصفوفة من الكائنات تمثل المستخدمين، كل منهم له خصائص متعددة تريد عرضها:

```jsx
function UserList() {
  const users = [
    { id: "user-001-employee", name: "Alice", email: "alice@example.com" },
    { id: "user-002-employee", name: "Bob", email: "bob@example.com" },
    { id: "user-003-employee", name: "John", email: "john@example.com" },
  ];
  return (
    <div>
      {users.map((user) => (
        <div key={user.id}>
          <h3>{user.name}</h3>
          <p>{user.email}</p>
        </div>
      ))}
    </div>
  );
}
```

في هذا المثال، نُنشئ هيكل JSX أكثر تعقيدًا لكل مستخدم، نعرض اسمه وعنوان بريده الإلكتروني. نستخدم معرف المستخدم `id` كـ `key`، وهذا يُعد ممارسة جيدة.

في الختام، عرض القوائم في React يتطلب تحويل مصفوفات البيانات إلى عناصر JSX، عادةً باستخدام دالة `map`.
