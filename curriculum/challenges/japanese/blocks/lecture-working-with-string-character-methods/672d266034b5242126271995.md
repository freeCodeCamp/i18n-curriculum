# --interactive--

プログラミングでは、文字がどのように数字として表されるかを理解することが基本です。ここで登場するのがASCIIです。ASCIIはAmerican Standard Code for Information Interchangeの略で、コンピューターでテキストを表すために使われる文字エンコーディングの標準です。各文字に数値が割り当てられ、機械が共通して認識できるようになっています。

このレッスンでは、ASCIIとは何か、どのように機能するのか、そしてJavaScriptの`charCodeAt()`や`fromCharCode()`のようなメソッドが文字エンコーディングとどのように関係しているかを探ります。JavaScriptの文字列は内部的にUnicode（UTF-16）を使っていますが、ASCIIの値は最初の128文字のUnicodeと一致するため、ASCIIベースの例がJavaScriptで動作します。

ASCIIは、文字（アルファベット、数字、記号など）を数値にエンコードするシステムです。各文字は特定の数値に対応しています。

例えば、大文字の`A`はASCIIで数値`65`で表され、小文字の`a`は`97`で表されます。このエンコーディングにより、コンピューターはテキストを保存・操作できます。

ASCII標準は128文字をカバーしており、以下を含みます：

- 英語の大文字と小文字（A-Z、a-z）。
- 数字（0-9）。  
- 一般的な句読点や記号（!、@、#など）。
- 制御文字（改行やタブなど）。  

JavaScriptでは、`charCodeAt()`メソッドを使って文字の数値コードにアクセスできます。このメソッドは指定したインデックスの文字のUTF-16コードユニットを返します。最初の128文字については、この値はASCIIコードと一致します。

例を見てみましょう：

:::interactive_editor

```js
let letter = "A";
console.log(letter.charCodeAt(0));  // 65
```

:::

この例では、`A`は文字列の最初の文字で、`charCodeAt(0)`を呼び出すとその数値コード（基本ラテン文字の場合はASCII値と一致）、`65`が返されます。

他の文字でもこのメソッドを使って数値コードを調べられます：

:::interactive_editor

```js
let symbol = "!";
console.log(symbol.charCodeAt(0));  // 33
```

:::

ここでは、感嘆符`!`の数値コードが`33`として返されます（ASCII値と一致）。

`charCodeAt()`は文字の数値コードを取得するのに役立ちますが、`fromCharCode()`メソッドは逆のことができます。すなわち、UTF-16コードユニット（基本文字の場合はASCIIと一致）を対応する文字に変換します。

実際に見てみましょう：

:::interactive_editor

```js
let char = String.fromCharCode(65);
console.log(char);  //  A
```

:::

この例では、`fromCharCode(65)`が数値コード`65`（`A`のASCII値と一致）を文字`A`に変換しています。

別の例として、数値`97`を対応する小文字に変換する場合：

:::interactive_editor

```js
let char = String.fromCharCode(97);
console.log(char);  // a
```

:::

これらのメソッドは、文字の数値コードに基づいて文字を操作したり比較したりする必要がある場合に特に便利です。

例えば、`charCodeAt()`を使って文字が大文字、小文字、数字のどれかをASCII値で判定できます。

一方、`fromCharCode()`はASCIIコードから動的に文字を生成するのに使えます。
