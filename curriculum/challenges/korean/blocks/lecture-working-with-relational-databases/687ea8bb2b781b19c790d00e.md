# --description--

SQL에서 기본 키와 외래 키는 표 간의 관계를 설정하는 데 사용됩니다.

먼저 기본 키부터 시작해 보겠습니다.

기본 키는 표에서 각 레코드(행)를 고유하게 식별하는 열 또는 열 집합입니다. 이 제약 조건은 표 내에 기본 키 값이 중복되는 레코드가 없도록 보장합니다.

이 때문에 표에는 기본 키가 하나만 있을 수 있습니다.

기본 키 열의 값은 `NULL`일 수 없으므로 항상 유효한 값을 가집니다.

PostgreSQL에서는 열을 표의 기본 키로 만들려면 데이터 유형 뒤에 `PRIMARY KEY`를 추가하면 됩니다:

```sql
column_name data_type PRIMARY KEY
```

다음은 `students` 표의 예입니다:

```sql
CREATE TABLE students (
  student_id SERIAL PRIMARY KEY,
  name VARCHAR(100)
);
```

각 학생은 고유한 `student_id`를 가지며, 이 ID가 표의 기본 키가 됩니다. `SERIAL` 유형은 `student_id`가 항상 고유한 값을 갖도록 보장하기 때문에 여기서 유용합니다.

복합 기본 키는 표에 행을 식별할 단일 고유 열이 없을 때 사용합니다. 이 경우 두 개 이상의 열을 조합하여 기본 키로 사용할 수 있으며, 이 조합이 고유합니다.

이렇게 하려면 표를 생성할 때 어떤 열이 복합 기본 키인지 다음과 같이 지정하세요:

```sql
CREATE TABLE table_name (
  column1 data_type column_constraint,
  column2 data_type column_constraint,
  column3 data_type column_constraint,
  ...
  PRIMARY KEY (column1, column2)
);
```

학생들이 여러 수업에 등록된 정보를 담은 `student_id`와 `course_id` 열이 있는 표가 있다고 상상해 보세요.

각 학생은 여러 수업에 등록하고, 각 수업에는 여러 학생이 등록합니다. 따라서 어느 열도 고유하지 않지만, 같은 학생이 같은 수업에 두 번 이상 등록되는 일은 없습니다. 그래서 두 열의 조합을 기본 키로 사용할 수 있습니다:

```sql
CREATE TABLE course_enrollments (
  student_id INT,
  course_id INT,
  ...
  PRIMARY KEY (student_id, course_id),
);
```

기본 키는 데이터의 고유성과 무결성을 보장하고, 데이터 검색을 최적화하며, 관계형 데이터베이스에서 다른 표와의 관계를 설정하는 데 필수적입니다.

다음으로 이 수업에서 다룰 키 유형인 외래 키로 넘어가겠습니다.

외래 키는 한 표의 열(또는 열 집합)로, 다른 표의 기본 키를 참조합니다. 한 표에는 여러 개의 외래 키가 있을 수 있습니다.

이 예에서는 `customers`와 `orders` 두 표가 있습니다:

```sql
CREATE TABLE customers (
  customer_id SERIAL PRIMARY KEY,
  first_name VARCHAR(100) NOT NULL,
  ...
);

CREATE TABLE orders (
  order_id SERIAL PRIMARY KEY,
  customer_id INTEGER,
  ...
  FOREIGN KEY (customer_id) REFERENCES customers(customer_id)
);
```

`customers` 표의 기본 키는 `customer_id`이고, `orders` 표의 기본 키는 `order_id`입니다.

하지만 `customer_id` 표에 `orders` 열도 있는 것을 주목하세요.

왜 `orders` 표에 고객 정보를 저장할까요?

이는 `customers`와 `orders` 표 간의 관계를 만들기 위해서입니다. 각 주문은 주문한 고객의 ID를 저장합니다.

표 간의 관계를 만들려면 `customer_id` 표의 `orders` 열을 `customer_id` 표의 `customers` 열을 참조하는 외래 키로 만듭니다:

```sql
FOREIGN KEY (customer_id) REFERENCES customers(customer_id)
```

외래 키 열의 값은 참조된 표의 기본 키 값과 일치하거나 `NULL`이어야 합니다(외래 키가 `NULL`를 허용하는 경우).

이것이 왜 유용할까요?

두 표의 이 열들이 정확히 일치하도록 보장함으로써, 외래 키와 기본 키는 다른 표에 존재하지 않는 레코드를 참조하는 "고아" 레코드 생성을 방지합니다.

기본 키와 외래 키는 관계형 데이터베이스의 핵심입니다. 이 키들이 관계형 데이터베이스의 "관계" 측면을 가능하게 하여 현실 세계 데이터를 정확하게 모델링할 수 있게 합니다.
