# --description--

在 React 中，渲染是组件出现在用户界面（UI）中的过程，通常是在浏览器中。

React 会获取你所有的 JavaScript、JSX 和 CSS 代码，确定它应该如何显示，然后将其显示在用户界面中。

渲染进程由三个阶段组成：触发器、渲染和提交。让我们更详细地看看这些阶段。

当 React 检测到某些内容已更改且用户界面可能需要更新时，会发生触发器阶段。此更改通常是由于状态或 props 的更新引起的。

例如，注意到是吃晚饭的时间可以触发你进入厨房开始做饭。

在下面的 `Counter` 示例中，点击增加或减少按钮会触发 React 显示新的 `count` 值：

```jsx
import { useState } from "react";

function Counter() {
  const [count, setCount] = useState(0);

  return (
    <div>
      <h1>{count}</h1>
      <button onClick={() => setCount(count - 1)}>Decrement</button>
      <button onClick={() => setCount(count + 1)}>Increment</button>
    </div>
  );
}

export default Counter;
```

一旦触发器发生，React 就进入渲染阶段。在这里，React 会重新评估你的组件并确定显示内容。

为此，React 使用了“真实” DOM 的轻量级拷贝。这称为虚拟 DOM。通过虚拟 DOM，React 可以快速查看组件中需要更改的内容。

将此阶段视为你在厨房，已经准备好食材，正在烹饪晚餐的时刻。

对于 `Counter` 组件，渲染阶段是 React 使用新的 `count` 值再次运行函数的时刻。React 根据更新后的 `count` 值重新计算组件中 `<h1>{count}</h1>` 部分的显示内容，但你不会看到屏幕上的任何变化，直到下一个阶段——提交。

提交阶段是 React 从虚拟 DOM 获取准备好的更改并将其应用到真实 DOM 的阶段。换句话说，这个阶段是你在屏幕上看到最终结果的阶段。

为了实现这一点，React 会将虚拟 DOM 与实际 DOM 进行比较，只识别需要更新的部分，并将这些更改应用到真实 DOM 以更新用户界面。

你可以将此阶段视为你上菜的时刻，使其可见，就像 React 在提交更新到实际 DOM 时所做的那样。

对于 `Counter` 组件，提交阶段是将新的 `count` 值应用到 `h1` 元素的时刻，你可以在页面上看到变化。

这三个进程非常快，因为 React 通过先计算虚拟 DOM 中的变化来最小化对真实 DOM 的直接操作，然后只更新真实 DOM 中需要更改的部分。
