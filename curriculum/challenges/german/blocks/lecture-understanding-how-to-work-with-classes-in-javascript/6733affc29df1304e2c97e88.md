# --interactive--

In modernem JavaScript sind Klassen wie Baupläne, die Sie im Code definieren können, um mehrere Objekte desselben Typs zu erstellen.

Wenn Sie beispielsweise ein Inventarsystem modellieren, müssen Sie für jede Produktart mehrere Objekte erstellen.

Jedes einzelne Produkt hat seine eigenen Eigenschaften und Aktionen, aber Produkte derselben Art können aus demselben „Bauplan“ erstellt werden. Genau dafür werden Klassen verwendet.

Klassen können entweder als Klassen-Ausdruck oder als Klassendeklaration definiert werden.

Hier ist die grundlegende Syntax für eine Klassendeklaration:

```js
class MyClassName {
  // Class Methods
  constructor() { ... }
  method1() { ... }
  method2() { ... }
  ...
}
```

Sie beginnen mit dem `class`-Schlüsselwort, gefolgt vom Namen der Klasse. Beim Benennen von Klassen sollten Sie die `PascalCase`-Benennungskonvention anstelle von `camelCase` verwenden.

Der Inhalt innerhalb der geschweiften Klammern wird als Klassenkörper bezeichnet. Hier kommen der Konstruktor und die Methoden hinein.

Der Konstruktor ist eine spezielle Methode, die automatisch aufgerufen wird, wenn ein neues Objekt aus der Klasse erstellt wird. Er wird typischerweise verwendet, um die Eigenschaften des Objekts zu initialisieren.

In diesem Beispiel sehen Sie eine `Dog`-Klasse mit einem Konstruktor.

```js
class Dog {
  constructor(name) {
    this.name = name;
  }
}
```

Das `this`-Schlüsselwort wird hier verwendet, um auf die aktuelle Instanz der `Dog`-Klasse zuzugreifen, sodass Sie die `name`-Eigenschaft dem gerade erstellten Objekt zuweisen können. Mehr über das `this`-Schlüsselwort lernen Sie in einer zukünftigen Lektion.

Unterhalb des Konstruktors können Sie Methoden definieren. Methoden sind Funktionen, die innerhalb einer Klasse definiert sind und Aktionen oder Operationen ausführen. Sie dienen dazu, Verhaltensweisen zu definieren, die Instanzen der Klasse ausführen können.

Hier ist ein Beispiel, wie Sie der `bark`-Klasse eine Methode namens `Dog` hinzufügen.

```js
class Dog {
  constructor(name) {
    this.name = name;
  }

  bark() {
    console.log(`${this.name} says woof!`);
  }
}
```

Wenn die `bark`-Methode aufgerufen wird, wird die Nachricht mit dem tatsächlich übergebenen Namen in der Konsole ausgegeben. Wenn Sie eine `Dog`-Instanz mit dem Namen `"Buddy"` erstellen, wird durch den Aufruf der `bark()`-Methode dieser Instanz `"Buddy says woof!"` in der Konsole ausgegeben.

Sobald Sie Ihre Klasse definiert haben, können Sie eine Instanz der Klasse mit dem `new`-Schlüsselwort, dem Namen der Klasse und deren Argumenten erstellen.

In diesem Beispiel erstellen wir eine neue `Dog`-Instanz mit dem Namen `Gino` und weisen sie dann der Variablen `dog` zu.

```js
const dog = new Dog("Gino");
```

Wenn Sie `new Dog("Gino")` in Ihrem Code schreiben, wird das neue Objekt erstellt und der Konstruktor automatisch mit den angegebenen Argumenten aufgerufen.

Sie können dann auf diese Eigenschaft mit der Punktnotation zugreifen, wie Sie in der folgenden Codezeile sehen können:

```js
console.log(dog.name);
```

Hier ist das vollständige Beispiel:

:::interactive_editor

```js
class Dog {
  constructor(name) {
    this.name = name;
  }

  bark() {
    console.log(`${this.name} says woof!`);
  }
}

const dog = new Dog("Gino");
console.log(dog.name);
```

:::

Sie können auch Methoden aufrufen. In diesem Beispiel rufen wir die `bark()`-Methode auf der `dog`-Instanz auf.

```js
dog.bark();
```

Hier ist das vollständige Beispiel mit dem Methodenaufruf:

:::interactive_editor

```js
class Dog {
  constructor(name) {
    this.name = name;
  } 
  bark() {
    console.log(`${this.name} says woof!`);
  }
}

const dog = new Dog("Gino");
console.log(dog.name);
dog.bark();
```

:::

Wie bereits erwähnt, können Sie Klassen auch als Klassen-Ausdruck definieren. Dabei ist die Klasse anonym und wird einer Variablen zugewiesen.

So sieht das vorherige Beispiel als Klassen-Ausdruck aus:

```js
const Dog = class {
  constructor(name) {
    this.name = name;
  }

  bark() {
    console.log(`${this.name} says woof!`);
  }
};
```

Meistens verwenden Sie Klassendeklarationen, aber das Verständnis von Klassen-Ausdrücken ist ebenfalls wertvoll, da sie mehr Flexibilität bieten können, besonders in dynamischen Programmier-Situationen.

Durch die Verwendung von Klassen und das Verständnis ihrer grundlegenden Syntax können Sie wiederverwendbaren Code erstellen, ihn in logische Einheiten organisieren und die Modularität des Codes fördern.
