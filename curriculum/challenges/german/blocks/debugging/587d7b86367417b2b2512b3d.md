# --description--

Das letzte Thema sind die gefürchteten Endlosschleifen. Schleifen sind großartige Werkzeuge, wenn Ihr Programm einen Codeblock eine bestimmte Anzahl von Malen oder bis zum Erreichen einer Bedingung ausführen soll, aber sie benötigen eine Abbruchbedingung, die die Schleife beendet. Endlosschleifen führen wahrscheinlich dazu, dass der Browser einfriert oder abstürzt und verursachen allgemeines Chaos bei der Programmausführung, was niemand möchte.

In der Einführung zu diesem Abschnitt gab es ein Beispiel für eine Endlosschleife – sie hatte keine Abbruchbedingung, um aus der `while`-Schleife innerhalb von `loopy()` auszubrechen. Rufen Sie diese Funktion NICHT auf!

```js
function loopy() {
  while(true) {
    console.log("Hello, world!");
  }
}
```

Es ist die Aufgabe der Programmierenden sicherzustellen, dass die Abbruchbedingung, die dem Programm sagt, wann es aus dem Schleifencode ausbrechen soll, schließlich erreicht wird. Ein Fehler ist, eine Zählvariable in die falsche Richtung von der Abbruchbedingung aus zu erhöhen oder zu verringern. Ein anderer Fehler ist, versehentlich eine Zähl- oder Indexvariable innerhalb des Schleifencodes zurückzusetzen, anstatt sie zu erhöhen oder zu verringern.

# --instructions--

Die `myFunc()`-Funktion enthält eine Endlosschleife, weil die Abbruchbedingung `i != 4` niemals auf `false` ausgewertet wird (und die Schleife beendet) – `i` wird bei jedem Durchlauf um 2 erhöht und überspringt 4, da `i` ungerade ist. Korrigieren Sie den Vergleichsoperator in der Abbruchbedingung, sodass die Schleife nur läuft, wenn `i` kleiner oder gleich 4 ist.

# --hints--

Ihr Code sollte den Vergleichsoperator in der Abbruchbedingung (dem mittleren Teil) der `for`-Schleife ändern.

```js
assert(__helpers.removeJSComments(code).match(/i\s*?<=\s*?4;/g).length == 1);
```

Ihr Code sollte den Vergleichsoperator in der Abbruchbedingung der Schleife korrigieren.

```js
assert(!__helpers.removeJSComments(code).match(/i\s*?!=\s*?4;/g));
```
