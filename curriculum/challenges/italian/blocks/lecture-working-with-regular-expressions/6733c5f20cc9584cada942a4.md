# --interactive--

Un gruppo di cattura ti permette di "catturare" una porzione della stringa corrispondente per usarla come preferisci. I gruppi di cattura si definiscono con parentesi che contengono il pattern da catturare, senza caratteri iniziali come un lookahead.

Catturiamo il `code` dalla nostra espressione regolare `freeCodeCamp`. Per farlo, racchiudiamo `code` tra parentesi e lo definiamo come gruppo di cattura:

```js
const regex = /free(code)camp/i;
```

Per confermare il comportamento, possiamo testarlo su una stringa `freecodecamp`:

:::interactive_editor

```js
const regex = /free(code)camp/i;
console.log(regex.test("freecodecamp")); // true
```

:::

Ma questo non sfrutta realmente il nostro gruppo catturato. Invece, diamo un’occhiata al risultato dell’uso di `match`:

```js
const regex = /free(code)camp/i;
console.log("freecodecamp".match(regex));
// [
//   'freecodecamp',
//   'code', <--
//   index: 0,
//   input: 'freecodecamp',
//   groups: undefined
// ]
```

Qui vediamo che il nostro array `match` ha un secondo elemento, che è la porzione della stringa catturata dal nostro gruppo di cattura.

Nota come il gruppo di cattura corrisponda esattamente al pattern `code`, dove una classe di caratteri corrisponderebbe a un singolo carattere tra `c`, `o`, `d` e `e`.

Ma come possiamo usarlo davvero? I gruppi di cattura si usano spesso quando si sostituisce il contenuto di una stringa. Prepariamo un codice per farlo. Trasformeremo `freecodecamp` in `paidcodeworld`:

:::interactive_editor

```js
const regex = /free(code)camp/i;
console.log("freecodecamp".replace(regex, "paidcodeworld"));
```

:::

Questo funziona da solo, ma cosa succede se non sappiamo quanti `o` ci sono in `code`? Se ci serve un quantificatore per uno o più `o`:

:::interactive_editor

```js
const regex = /free(co+de)camp/i;
console.log("freecoooooooodecamp".replace(regex, "paidcodeworld"));
```

:::

Otteniamo `paidcodeworld` come risultato. Vogliamo preservare il numero di `o`, quindi dobbiamo riutilizzare ciò che è stato catturato dall’espressione regolare.

Qui entra in gioco un riferimento indietro. Invece di inserire direttamente la porzione `code` nella stringa di sostituzione, possiamo fare riferimento direttamente al gruppo catturato.

In una chiamata `replace`, ottieni un riferimento indietro usando un simbolo di dollaro (`$`) seguito dal numero del gruppo di cattura da usare. Nel nostro caso, sarebbe `$1`, dato che `code` è catturato nel primo gruppo di cattura:

:::interactive_editor

```js
const regex = /free(co+de)camp/i;
console.log("freecoooooooodecamp".replace(regex, "paid$1world")); // paidcoooooooodeworld
```

:::

Abbiamo così preservato con successo un numero sconosciuto di caratteri `o` convertendo `freecodecamp` in `paidcodeworld`. Ma i riferimenti indietro non si limitano alla chiamata replace. Puoi usarli direttamente in un’espressione regolare.

Questo ti permetterebbe di corrispondere a un pattern precedentemente catturato più avanti nell’espressione regolare.

Supponiamo di voler corrispondere a `freecodecamp` due volte, con lo stesso numero di `o`, ma in qualsiasi punto della stringa.

Prima, dobbiamo separarli con il nostro carattere jolly e permettere che qualsiasi numero di caratteri corrisponda a quel jolly:

```js
const regex = /free(co+de)camp.*free(co+de)camp/i;
```

Questa espressione attuale non garantisce però che il numero di caratteri `o` sia lo stesso. Per farlo, dobbiamo sostituire il secondo gruppo di cattura con un riferimento al primo.

All’interno di un’espressione regolare, un riferimento indietro si indica con una barra rovesciata seguita dal numero del gruppo di cattura:

:::interactive_editor

```js
const regex = /free(co+de)camp.*free\1camp/i;
console.log(regex.test("freecooooodecamp is great i love freecooooodecamp")); // true
console.log(regex.test("freecooooodecamp is great i love freecodecamp")); // false
```

:::

E con questo vediamo che una stringa con il numero corretto di `o` corrisponde, mentre una stringa con due numeri diversi di `o` no.

Questa sintassi è ottima, ma può diventare rapidamente confusa quando fai riferimento a più gruppi di cattura. Per fortuna, invece di usare numeri, puoi dare un nome ai tuoi gruppi.

Per definire un gruppo di cattura nominato, aggiungi un punto interrogativo (`?`) seguito dal nome racchiuso tra segni di minore e maggiore all’inizio del gruppo. Chiamiamo il nostro gruppo di cattura `code`:

```js
const regex = /free(?<code>co+de)camp.*free\1camp/i;
```

Ora possiamo aggiornare il nostro riferimento indietro nell’espressione regolare per riferirci a questo gruppo. Un riferimento indietro nominato inizia con una barra rovesciata seguita dalla lettera `k` in JavaScript. Poi aggiungi il nome, di nuovo racchiuso tra segni di minore (`<`) e maggiore (`>`). Vediamolo:

```js
const regex = /free(?<code>co+de)camp.*free\k<code>camp/i;
```

Ora se controlliamo la nostra chiamata `test()`, vediamo che ancora passa:

:::interactive_editor

```js
const regex = /free(?<code>co+de)camp.*free\k<code>camp/i;
console.log(regex.test("freecooooodecamp is freecooooodecamp")); // true
```

:::

Per usare il nostro gruppo di cattura nominato in una chiamata `replace()`, inseriamo un simbolo di dollaro nella stringa, seguito dal nome racchiuso tra segni di minore e maggiore:

:::interactive_editor

```js
const regex = /free(?<code>co+de)camp/i;
console.log("freecooooodecamp".replace(regex, "paid$<code>camp")); // paidcooooodecamp
```

:::

Infine, a volte vuoi creare un gruppo di caratteri, ma non ti serve il risultato catturato.

Supponiamo di voler corrispondere a `freecodecamp` o `freecandycamp`. Potresti creare due pattern separati da un’asserzione OR:

```js
const regex = /freecodecamp|freecandycamp/i;
```

Ma questo può diventare piuttosto lungo per espressioni regolari più complesse. Invece, puoi creare un gruppo non catturante attorno ai caratteri che vuoi OR:

```js
const regex = /free(?:code|candy)camp/i;
```

Un gruppo non catturante non archivia la corrispondenza `code|candy` separatamente in memoria. Ma può essere utile per creare pattern alternativi senza sacrificare leggibilità o prestazioni.
