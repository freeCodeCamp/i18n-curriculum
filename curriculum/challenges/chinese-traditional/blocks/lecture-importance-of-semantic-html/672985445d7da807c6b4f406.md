# --interactive--

建立結構階層體系最重要的方面是正確使用標題元素。標題元素以 `h1`、`h2`、`h3` 等編號。這些數字表現該元素的標題層級。

就像文字文件一樣，你不想以錯誤的順序使用標題層級。你的 `h1` 元素是你的頂層標題。你在一個頁面上很少會有多於一個的 `h1`，而且它通常應該出現在你所有內容之前。

你的 `h2` 元素是你的副標題。它應該總是在你的 `h1` 之後，並且可能在一些介紹文字之後。與 `h1` 元素不同，你的頁面上可以有多個 `h2` 元素。你經常會這樣做來劃分不同的內容區段。 

依照這個樣式，你的 `h3` 元素應該總是在 `h2` 元素之後。也就是說，你不應該直接從 `h1` 跳到 `h3`。不過，你可以有多個相同層級的標題元素。舉例來說，這段程式碼是正確的：

```html
<section>
  <h1>freeCodeCamp</h1>
  <h2>Learn Front-End Development</h2>
  <h2>Learn Back-End Development</h2>
</section>
```

但這段程式碼不正確，因為 `h3` 在 `h2` 之前。 

將 `h2` 元素移到 `h3` 之上，使其語意正確。若要與程式碼互動，你需要啟用互動式編輯器。

:::interactive_editor

```html
<section>
  <h1>freeCodeCamp</h1>
  <h3>Introduction to HTML</h3>
  <h2>Learn Frontend Development</h2>
</section>
```

:::

可能會因為某個標題元素的樣式而想使用它，例如用 `h1` 來呈現大字體：

:::interactive_editor

```html
<article>
  <p>
    Here is some
    <h1>Large Text</h1>
  </p>
</article>
```

:::

相反地，你應該為你的文件結構選擇適當的元素，並使用 CSS 來達成你想要的樣式。

使用正確的階層體系對於存取性非常重要。輔助技術，例如螢幕閱讀器，依賴網頁的結構來判斷如何解析並向使用者宣告該網頁。在 `h1` 之後使用 `h3` 元素，可能會讓螢幕閱讀器使用者誤以為他們不小心跳過了重要內容，因為缺少了 `h2` 元素。

適當的結構對 SEO 也很重要。搜尋引擎使用自動化來解析你網頁的內容，並判斷何時及何處應該顯示在結果中。如果你的結構有誤，搜尋引擎可能無法在相關搜尋結果中給你很好的等級。

最後，根據你的結構有多錯誤，你的 HTML 甚至可能在技術上不合法。當發生這種情況時，網頁瀏覽器必須有效地猜測你想要做什麼。而它猜測的結果可能根本不是你想要的。 

正如你今天所學，使用適當的結構階層體系來設計你的頁面有許多原因。當你建置新專案時，請記住這一點。
