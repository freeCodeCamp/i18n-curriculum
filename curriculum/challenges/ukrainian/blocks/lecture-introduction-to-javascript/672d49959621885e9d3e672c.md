# --interactive--

Працюючи з JavaScript, ви часто оголошуєте змінні, щоб зберігати дані, які плануєте використовувати у своєму застосунку.

У сучасному JavaScript `let` і `const` — це рекомендовані способи оголошення змінних, але вони відрізняються тим, як обробляють присвоєння та повторне присвоєння значень.

У цьому уроці ми розглянемо, чим `let` і `const` відрізняються в оголошенні змінних, присвоєнні та повторному присвоєнні.

Ключове слово `let` дозволяє оголошувати змінні, які можна оновлювати або повторно присвоювати пізніше. Можна уявити `let` як гнучкий контейнер — після того, як ви зберегли в ньому значення, ви можете змінювати це значення за потреби протягом роботи застосунку.

Ось приклад оголошення та присвоєння змінної з `let`:

```js
let score = 10;
```

У цьому випадку змінна `score` оголошена і їй присвоєне значення `10`. Якщо ви хочете оновити значення пізніше, це легко зробити:

:::interactive_editor

```js
let score = 10;
console.log(score); // 10
score = 20;
console.log(score); // 20
```

:::

Тепер `score` містить значення `20`. Це робить `let` особливо корисним, коли ви знаєте, що значення змінної буде змінюватися під час виконання застосунку.

З іншого боку, `const` використовується для оголошення константних змінних. Після того, як ви присвоїли значення змінній, оголошеній з `const`, ви не можете повторно присвоїти їй інше значення.

Це робить `const` ідеальним для значень, які ви не хочете випадково змінити під час виконання застосунку.

Ось приклад оголошення та присвоєння змінної з `const`:

:::interactive_editor

```js
const maxScore = 100;
console.log(maxScore); // 100
```

:::

Після того, як `maxScore` отримала значення `100`, його не можна змінити:

```js
maxScore = 200; // This will result in an error
```

Спроба повторного присвоєння значення змінній `const` викличе помилку у вашій JavaScript консолі, оскільки змінні `const` є незмінними після присвоєння.

Ви можете оголосити змінну `let` без негайного присвоєння значення, і присвоїти їй значення пізніше:

:::interactive_editor

```js
let age;
console.log(age); // undefined
age = 25;
console.log(age); // 25
```

:::

Змінні, оголошені з `const`, повинні отримувати значення під час оголошення. Якщо ви спробуєте оголосити змінну `const` без присвоєння значення, отримаєте помилку:

```js
const age; // Error: Missing initializer in const declaration
```

Використовуйте `let`, коли потрібно оголосити змінні, які будуть повторно присвоюватися пізніше. Наприклад, для відстеження змінного рахунку або оновлення значення з часом у вашому застосунку.

Використовуйте `const`, коли хочете оголосити змінні, які мають залишатися константами, наприклад, значення конфігурації або налаштування, які не повинні випадково змінюватися.

Ви також можете використовувати ключове слово `var`, але це вже не так рекомендовано. `var` схоже на `let`, але має ширшу область видимості, що частіше призводить до проблем у вашому застосунку.
