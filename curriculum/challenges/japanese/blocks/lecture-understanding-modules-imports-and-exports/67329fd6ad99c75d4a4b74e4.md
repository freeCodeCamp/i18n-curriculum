# --description--

JavaScriptにおけるモジュールとは、関連する関数、クラス、または変数をまとめた自己完結型のコード単位です。

モジュールはアプリケーションの構成要素のようなもので、本の章のように考えてください。各モジュールは特定の機能に焦点を当てているため、コードがより整理され、保守しやすく、再利用しやすくなります。モジュールは名前の衝突を防ぎ、アプリケーションを別々の連結された部分に構造化するのに役立ちます。

JavaScriptのモジュールの概念は時間とともに進化してきましたが、最も広く使われている標準的な方法はES6（ECMAScript 2015）モジュールシステムです。このシステムは、異なるJavaScript環境間でモジュールを定義し使用するための標準化された方法を提供します。

モジュールを作成するには、JavaScriptのコードを別のファイルに書きます。アプリケーションの他の部分で使いたい変数、関数、またはクラスは明示的にエクスポートする必要があります。これは`export`キーワードを使って行います。

例えば、`math.js`というファイルにいくつかの数学関数があるとしましょう：

```js
export function add(a, b) {
    return a + b;
}

export function subtract(a, b) {
    return a - b;
}

const PI = 3.14159;
export { PI };
```

この例では、2つの関数（`add`と`subtract`）と定数（`PI`）をエクスポートしています。1つのモジュールから必要なだけ多くの項目をエクスポートできます。

これらのエクスポートされた項目をアプリケーションの別の部分で使うには、インポートする必要があります。これは`import`キーワードを使って行います。

例えば、`app.js`というファイルでこれらの数学関数を使いたいとします：

```js
import { add, subtract, PI } from './math.js';

console.log(add(5, 3));        // Outputs: 8
console.log(subtract(10, 4));  // Outputs: 6
console.log(PI);               // Outputs: 3.14159
```

ここでは、`math.js`モジュールから必要な関数と定数を特定してインポートしています。`'./math.js'`の部分は、現在のファイルから見たモジュールファイルの場所をJavaScriptに伝えています。

時には、モジュールがエクスポートするすべてをインポートしたい場合があります。これはアスタリスク（`*`）構文を使って行えます：

```js
import * as Math from './math.js';

console.log(Math.add(5, 3));        // Outputs: 8
console.log(Math.subtract(10, 4));  // Outputs: 6
console.log(Math.PI);               // Outputs: 3.14159
```

この場合、`math.js`からのすべてのエクスポートが`Math`というオブジェクトのプロパティとしてインポートされます。

もう一つよくあるパターンは、モジュールにデフォルトエクスポートを持つことです。これは通常、モジュールが主に1つの関数をエクスポートするときに使われます。モジュールごとにデフォルトエクスポートは1つだけです。

以下のようになります：

```js
// In math.js
export default function multiply(a, b) {
    return a * b;
}

// In app.js
import multiply from './math.js';

console.log(multiply(4, 5));  // Outputs: 20
```

デフォルトエクスポートをインポートするときは、波括弧を使う必要がなく、インポート名は自由に付けられます。

ブラウザでES6モジュールを使うには、`type`タグの`module`属性を`script`に指定する必要があることに注意してください：

```html
<script type="module" src="app.js"></script>
```

モジュールはJavaScriptコードを整理し構造化する強力な方法を提供します。アプリケーションをより小さく管理しやすい部分に分割し、コードの再利用を促進し、関心の分離をきれいに保つのに役立ちます。大きなアプリケーションを作るとき、モジュールはJavaScript開発の必須ツールになるでしょう。
