# --description--

이전 수업에서 Bash와 자주 사용하는 몇 가지 명령어에 대해 배웠습니다. 하지만 Bash는 더 복잡한 자동화 작업을 수행할 수 있는 완전한 스크립팅 언어를 지원합니다.

Bash 스크립팅은 Bash 명령어들을 파일에 순서대로 작성한 후, Bash로 실행하여 파일 내용을 실행하는 것을 말합니다.

Bash 스크립트가 어떻게 생겼는지 살펴보겠습니다:

```bash
#!/bin/bash

servers=("prod" "dev")

for server in "${servers[@]}"
do
  echo "Pulling $server"
  rsync --archive --verbose $server:/etc/nginx/conf.d/server.conf configs/$server.conf
done
```

이 스크립트는 원격 서버 목록에서 NGINX 구성 파일을 가져오는데, 이는 해당 구성을 로컬 저장소에 백업할 때 유용합니다. NGINX는 인기 있는 웹 서버이며, 앞으로의 모듈에서 서버 작업에 대해 더 배우게 됩니다.

첫 번째 줄은 "셰뱅"(`#!`)이라고 하며, 시스템에 어떤 인터프리터를 사용해 스크립트를 실행할지 알려줍니다. 여기서 흔히 쓰이는 값은 `/usr/bin/bash`, `/bin/bash`, 또는 `/bin/sh`입니다.

첫 번째 줄인 `servers=("prod" "dev")`는 서버 이름을 나타내는 문자열 목록을 생성합니다. `"prod"`는 보통 운영 사이트를 가리키고, `"dev"`는 개발 사이트를 나타내는데, 이는 보통 라이브 운영 환경에 배포하기 전에 새로운 기능과 변경 사항을 테스트하는 데 사용됩니다.

그다음, `for` 반복문으로 목록을 반복합니다 - `servers[@]` 구문은 목록을 개별 요소로 확장합니다. 각 요소는 반복할 때마다 server 변수에 할당됩니다.

`do` 문장은 반복문의 논리 블록, 즉 각 반복마다 실행할 코드를 시작함을 나타냅니다.

`echo "Pulling $server"`은 현재 server 변수의 값을 보간하여, 이 경우 prod 또는 dev가 되어 터미널에 출력합니다.

그다음 `rsync`은 서버에서 실제 파일을 가져와 로컬에 저장하는 데 사용되며, 이 명령어를 관리하기 위해 보간을 더 활용합니다.

`done`는 반복문의 논리 블록이 끝났음을 나타냅니다.

Bash 스크립팅을 이해하고 활용하는 데는 여러 중요한 장점이 있습니다.

첫째, 거의 모든 Unix 환경에 Bash가 기본으로 설치되어 있어, Node.js나 Python 같은 언어 런타임을 설정하지 않고도 스크립트를 그대로 실행할 수 있습니다.

둘째, 시스템에 있는 모든 바이너리 애플리케이션에 최소한의 노력으로 접근할 수 있습니다. 시스템에 doctl이 설치되어 있다면, 스크립트에서 해당 바이너리를 실행할 수 있습니다. 바이너리는 컴퓨터가 실행하는 컴파일된 프로그램입니다. Bash 스크립트가 있으면, 프로그램을 실행하는 바이너리를 실행할 수 있습니다.

마지막으로, 스크립트에 사용하는 구문은 터미널에서 직접 실행할 수 있습니다. 이는 스크립트의 작은 부분을 테스트할 때 훨씬 간단해진다는 뜻입니다 - 그 부분을 복사해 터미널에 붙여넣고 결과를 바로 확인할 수 있습니다!

Bash를 이해하면 스크립팅 실력이 향상될 뿐 아니라, 명령줄에서 마법사가 될 수 있습니다!
