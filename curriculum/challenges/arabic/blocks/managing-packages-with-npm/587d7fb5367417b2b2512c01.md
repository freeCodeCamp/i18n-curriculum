# --description--

تتبع `Versions` لحزم npm في قسم التبعيات في ملف package.json الخاص بك ما يُسمى بالإصدار الدلالي (Semantic Versioning - SemVer)، وهو معيار صناعي لإصدار البرمجيات يهدف إلى تسهيل إدارة التبعيات. يجب أن تستخدم المكتبات، وأطر العمل، أو الأدوات الأخرى المنشورة على npm الإصدار الدلالي لتوضيح نوع التغييرات التي يمكن أن تتوقعها المشاريع عند التحديث.

يمكن أن يكون فهم الإصدار الدلالي مفيدًا عند تطوير برمجيات تستخدم تبعيات خارجية (وهو الحال دائمًا تقريبًا). يومًا ما، سينقذك فهمك لهذه الأرقام من إدخال تغييرات كاسرة على مشروعك عن طريق الخطأ دون أن تفهم لماذا توقفت الأشياء التي كانت تعمل بالأمس عن العمل اليوم. هكذا يعمل الإصدار الدلالي حسب الموقع الرسمي:

```json
"package": "MAJOR.MINOR.PATCH"
```

يجب زيادة الإصدار MAJOR عند إجراء تغييرات غير متوافقة في واجهة برمجة التطبيقات. يجب زيادة الإصدار MINOR عند إضافة وظائف بطريقة متوافقة مع الإصدارات السابقة. يجب زيادة الإصدار PATCH عند إصلاح أخطاء برمجية بطريقة متوافقة مع الإصدارات السابقة. هذا يعني أن PATCHes هي إصلاحات أخطاء وMINORs تضيف ميزات جديدة لكن لا تكسر ما كان يعمل سابقًا. وأخيرًا، تضيف MAJORs تغييرات لا تعمل مع الإصدارات السابقة.

# --instructions--

في قسم التبعيات في ملف `package.json` الخاص بك، غيّر إصدار `@freecodecamp/example` ليطابق الإصدار MAJOR 1، والإصدار MINOR 2، والإصدار PATCH 13

```js
fetch(code + '/_api/package.json')
  .then(response => response.json())
  .then(
    data => {
      assert.property(
        data.dependencies,
        '@freecodecamp/example',
        '"dependencies" does not include "@freecodecamp/example"'
      );
    },
    error => {
      throw new Error(error.message || error.responseText);
    }
  );
```
