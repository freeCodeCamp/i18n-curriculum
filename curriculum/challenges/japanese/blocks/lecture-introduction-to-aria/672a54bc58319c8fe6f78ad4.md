# --interactive--

もし支援技術（スクリーンリーダーなど）を使う人からはコンテンツを隠しつつ、画面上には表示したい場合は、`aria-hidden`属性を使うことができます。

隠したいHTML要素にこの属性を追加し、その値を`true`に設定するだけです。こちらの例をご覧ください：`aria-hidden="true"`。

この属性は、その要素とその子要素すべてをアクセシビリティツリーから隠しますが、ページ上には表示されたままにします。よくある使い方は以下の通りです：

- 装飾目的だけのアイコンや画像。
- 重複したコンテンツ。

重要なのは、`aria-hidden`はスクリーンリーダーなどの支援技術からのみコンテンツを隠すということです。もしすべての人から隠す必要がある場合は、`aria-hidden`を使って隠すべきではありません。例えば、現在折りたたまれているハンバーガーメニューは、スクリーンリーダーユーザーだけでなく、すべてのキーボードユーザーから隠す必要があります。この場合は、メニューにCSSの`display`プロパティを`none`に設定して、折りたたまれているときにDOMから削除します。

キーボードでフォーカス可能な要素を隠すために`aria-hidden`を使うのは避けてください。`aria-hidden`属性は要素をアクセシビリティツリーから除外するだけで、DOMからは削除しません。したがって、スクリーンリーダーユーザーはその要素にタブ移動できますが、スクリーンリーダーはその要素を読み上げないため、実質的に「何もない」場所にフォーカスが当たることになります。

ここでは、`aria-hidden`属性に`true`を設定して、アイコンをアクセシビリティツリーから隠す例を示します。

アイコンとテキストの両方が同じ目的で存在する冗長性による混乱を避けるため、テキストだけを支援技術に公開しています。

**注意**：このインタラクティブな例にはFont AwesomeのCDNが含まれているため、プレビューウィンドウに歯車アイコンが表示されます。プレビューを見るにはインタラクティブエディターを有効にしてください。

:::interactive_editor

```html
<head>
  <!-- Font Awesome CDN -->
  <link
    rel="stylesheet"
    href="https://cdn.jsdelivr.net/npm/@fortawesome/fontawesome-free@6.5.0/css/all.min.css"
  />
</head>

<button>
  <i class="fa-solid fa-gear" aria-hidden="true"></i>
  <span class="label">Settings</span>
</button>
```

:::

以下の場合は`aria-hidden`を使う必要はありません：

- HTML要素にすでに`hidden`属性がある場合。
- 要素またはその祖先要素がすでに`display: none`で隠されている場合。
- 要素またはその祖先要素がすでに`visibility: hidden`で隠されている場合。

これら3つの場合、要素はすでにDOMから削除されているためアクセシビリティツリーからも隠されており、`aria-hidden`属性は不要です。

ARIA属性を使う際は、必ず支援技術でテストし、できれば障害のある方にテストしてもらい、隠された要素があっても理解しやすいか確認してください。

また、`aria-hidden`を`false`に設定しても、親要素のいずれかにこの属性が`true`に設定されている場合は、要素は支援技術に公開されません。

`aria-hidden`属性は、スクリーンリーダーユーザーなど支援技術を使う人から要素を隠すために使われます。

純粋に装飾的な要素や重複したコンテンツを隠すのに役立ちますが、アクセシビリティを妨げないように控えめに使うべきです。

一般的に、ページ上のすべてのコンテンツと機能は支援技術を使う人にも利用可能であるべきです。`aria-hidden`の使用は非常に限定的で、主にスクリーンリーダーユーザーの体験をすっきりさせるために、純粋に装飾的または重複した情報を除去する目的に限るべきです。スクリーンリーダーユーザーが興味を持たないと思うコンテンツを隠すために`aria-hidden`を使わないでください。スクリーンリーダーユーザーもページ上のすべての情報にアクセスする権利があります。

これらのベストプラクティスに従い、ユーザー体験をテストすることで、誰にとっても包括的なオンライン体験を作ることができます。
