# --interactive--

약어는 단어나 구의 축약형입니다. 예를 들어, 마침표가 뒤따르는 "Dr"는 "doctor"라는 단어의 약어입니다.

약어에는 두 가지 일반적인 형태가 있습니다: 두문자어와 머리글자어입니다.

두문자어는 구의 각 단어 첫 글자를 모아 만든 단어로, 각 글자가 그 구의 단어 첫 글자를 나타냅니다.

GUI는 두문자어의 예입니다. Graphical User Interface의 약자로, 각 단어의 첫 글자인 G, U, I를 따서 두문자어 GUI를 만듭니다.

머리글자어는 구의 각 단어 첫 글자를 모아 만든 것으로, 각 글자가 그 구의 단어 첫 글자를 나타냅니다.

예를 들어, HTML은 머리글자어입니다; HyperText Markup Language의 약자로, 각 글자를 하나씩 H, T, M, L로 발음합니다.

두문자어와 머리글자어는 모두 약어의 유형입니다. 차이점은 두문자어는 단어처럼 발음하고, 머리글자어는 각 글자를 개별적으로 발음한다는 점입니다.

이들은 특히 잘 알려져 있고 특정 문맥에서 이해하기 쉬울 때 더 간결한 글쓰기에 매우 유용합니다.

HTML에서 두문자어나 머리글자어 같은 약어를 표시해야 한다면, abbreviation 요소가 바로 적합합니다. 처음 사용할 때는 항상 전체 의미를 설명해야 합니다. 그런 다음 abbreviation 요소를 사용해 강조하고 더 자세한 정보를 제공할 수 있습니다.

다음은 `HTML is the foundation of the web` 문장이 포함된 단락 예시입니다:

:::interactive_editor

```html
<p><abbr>HTML</abbr> is the foundation of the web.</p>
```

:::

머리글자어 HTML이 abbreviation 요소 안에 있습니다. 브라우저에서는 실제로 아무것도 변하지 않은 것처럼 보입니다. 여전히 일반 텍스트처럼 보입니다. abbreviation 요소는 배경에서 유용한 문맥을 제공하지만, 사용자는 머리글자어를 일반 텍스트로 봅니다.

사용자가 이 머리글자어가 무엇을 의미하는지 이해하도록 돕고 싶다면 `title` 속성으로 전체 형태를 보여줄 수 있습니다.

`title` 속성은 선택 사항이지만 포함하기로 결정하면, 약어, 두문자어, 머리글자어의 사람이 읽을 수 있는 설명이어야 합니다.

앞서 예시와 동일한 문장에 `title` 속성을 추가해 보겠습니다. `HyperText Markup Language`는 머리글자어의 확장된 형태입니다:

:::interactive_editor

```html
<p><abbr title="HyperText Markup Language">HTML</abbr> is the foundation of the web.</p>
```

:::

보통 이 속성을 추가하면 abbreviation 요소의 스타일이 바뀝니다. 구체적인 스타일은 브라우저마다 다릅니다. 어떤 브라우저는 점선 밑줄을 표시하고, 다른 브라우저는 내용을 소문자 대문자 형태로 바꿀 수 있습니다. 사용자가 약어 위에 마우스를 올리면 전체 형태가 툴팁으로 표시됩니다.

웹 페이지의 모든 약어에 abbreviation 요소를 꼭 사용해야 하는 것은 아니지만, 불명확할 수 있거나 추가 문맥이 필요한 경우에는 권장합니다.

정보와 표현 사이에서 적절한 균형을 찾아 텍스트가 복잡해지지 않으면서도 명확하고 간결하게 유지하세요.
