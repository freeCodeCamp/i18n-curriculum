# --description--

In JavaScript ist `Set` ein eingebautes Objekt zur Verwaltung von Datensammlungen. Es ermöglicht Ihnen, einzigartige Werte beliebigen Typs zu speichern, egal ob primitive Werte oder Objekt-Referenzen. `Set` stellt sicher, dass jeder Wert darin nur einmal vorkommt, was es nützlich macht, um Duplikate aus einem Array zu entfernen oder Sammlungen von unterschiedlichen Werten zu verwalten.

Was `WeakSet` betrifft, so ist es eine spezielle Art von `Set` mit weniger Funktionen, die es Ihnen erlaubt, schwach gehaltene Objekt-Referenzen und Symbole zu speichern. Im Gegensatz zu `Set` unterstützt `WeakSet` keine Primitiven wie Zahlen oder Strings.

Im Unterschied zu einem normalen `Set` speichert ein `WeakSet` nur Objekte, und die Referenzen auf diese Objekte sind „schwach“, was bedeutet, dass WeakSets nicht verhindern, dass die gespeicherten Objekte vom Garbage Collector entfernt werden, wenn es keine anderen Referenzen auf sie gibt. Einfacher gesagt: Wenn das Objekt sonst nirgendwo in Ihrem Code verwendet wird, wird es automatisch entfernt, um Speicher freizugeben.

Um ein `Set` zu erstellen, verwenden Sie den `Set`-Konstruktor und weisen ihn einer Variablen zu:

```js
const myFirstSet = new Set();
```

Sie können das `Set` auch mit Werten initialisieren:

```js
const treeSet = new Set(["Baobab", "Jackalberry", "Mopane Tree", "Breadfruit"]);
```

Wenn Sie das `Set` in der Konsole ausgeben, sieht die Ausgabe so aus:

```js
/*
Set(4) {"Baobab", "Jackalberry", "Mopane Tree", "Breadfruit"}
  [[Entries]]
  0: "Baobab"
    value: "Baobab"
  1: "Jackalberry"
    value: "Jackalberry"
  2: "Mopane Tree"
    value: "Mopane Tree"
  3: "Breadfruit"
    value: "Breadfruit"
  size: 4
  [[Prototype]]: Set
*/
```

Wenn Sie das `Set` nicht mit Werten initialisiert haben, können Sie die `add()`-Methode verwenden, um ein Element zum `Set` hinzuzufügen:

```js
const treeSet = new Set();

// Add items to the treeSet
treeSet.add("Baobab");
treeSet.add("Jackalberry");
treeSet.add("Mopane Tree");
treeSet.add("Breadfruit");
```

Das Ergebnis und die Darstellung in der Konsole bleiben gleich.

Vergessen Sie nicht, dass doppelte Elemente im `Set` ignoriert werden:

```js
const treeSet = new Set();

// Add items to the treeSet
treeSet.add("Baobab");
treeSet.add("Jackalberry");
treeSet.add("Mopane Tree");
treeSet.add("Breadfruit");
treeSet.add("Baobab"); //duplicate item will be ignored

console.log(treeSet);
// Set(4) {"Baobab", "Jackalberry", "Mopane Tree", "Breadfruit"}
```

Die anderen Methoden, die Sie verwenden können, um ein `Set` zu manipulieren, sind:

- `delete()`  
- `clear()`
- `has()`
- `entries()`
- `forEach()`
- `keys()`
- `values()`

Sehen wir uns an, wie diese Methoden einzeln funktionieren. `delete()` entfernt ein angegebenes Element aus dem `Set`:

```js
const treeSet = new Set();

// Add items to the treeSet
treeSet.add("Baobab");
treeSet.add("Jackalberry");
treeSet.add("Mopane Tree");
treeSet.add("Breadfruit");

treeSet.delete("Breadfruit");

console.log(treeSet); // Set(3) {"Baobab", "Jackalberry", "Mopane Tree"}
```

`has()` prüft, ob ein bestimmter Wert im `Set` vorhanden ist:

```js
const treeSet = new Set();

// Add items to the treeSet
treeSet.add("Baobab");
treeSet.add("Jackalberry");
treeSet.add("Mopane Tree");
treeSet.add("Breadfruit");

treeSet.delete("Breadfruit");

console.log(treeSet.has("Breadfruit")); // false
```

`entries()` gibt einen `Set`-Iterator zurück, der ein Array der Werte im `[value, value]`-Format enthält:

```js
const treeSet = new Set();

// Add items to the treeSet
treeSet.add("Baobab");
treeSet.add("Jackalberry");
treeSet.add("Mopane Tree");
treeSet.add("Breadfruit");

treeSet.delete("Breadfruit");

console.log(treeSet.entries());
// SetIterator {"Baobab" => "Baobab", "Jackalberry" => "Jackalberry", "Mopane Tree" => "Mopane Tree"}
```

`keys()` und `values()` zeigen die Werte im `Set`. `keys()` ist nur ein Alias für die `values()`-Methode:

```js
const treeSet = new Set();

// Add items to the treeSet
treeSet.add("Baobab");
treeSet.add("Jackalberry");
treeSet.add("Mopane Tree");
treeSet.add("Breadfruit");

treeSet.delete("Breadfruit");

console.log("Keys: ", treeSet.keys());
console.log("Values: ", treeSet.values());
// Keys: SetIterator {"Baobab", "Jackalberry", "Mopane Tree"}
// Values: SetIterator {"Baobab", "Jackalberry", "Mopane Tree"}
```

`forEach()` ermöglicht es Ihnen, das `Set` zu durchlaufen:

```js
const treeSet = new Set();

// Add items to the treeSet
treeSet.add("Baobab");
treeSet.add("Jackalberry");
treeSet.add("Mopane Tree");
treeSet.add("Breadfruit");

treeSet.delete("Breadfruit");

treeSet.forEach((tree) => console.log(tree));
/*
Baobab
Jackalberry
Mopane Tree
*/
```

`clear()` entfernt alle Elemente aus dem Array:

```js
const treeSet = new Set();

// Add items to the treeSet
treeSet.add("Baobab");
treeSet.add("Jackalberry");
treeSet.add("Mopane Tree");
treeSet.add("Breadfruit");

treeSet.delete("Breadfruit");

treeSet.clear();

console.log(treeSet); // Set(0) {size: 0}
```

Es ist auch erwähnenswert, dass es eine `size`-Eigenschaft gibt, die die Anzahl der Elemente im `Set` zurückgibt:

```js
const treeSet = new Set();

// Add items to the treeSet
treeSet.add("Baobab");
treeSet.add("Jackalberry");
treeSet.add("Mopane Tree");
treeSet.add("Breadfruit");

treeSet.delete("Breadfruit");

console.log(treeSet.size); // 3
```

Genau wie `Set` gibt es auch einen `WeakSet`-Konstruktor, den Sie verwenden können, um ein `WeakSet` zu erstellen:

```javascript
const treeWeakSet = new WeakSet();
```

`WeakSet` hat ebenfalls die `add()`, `delete()` und `has()`-Methoden:

```javascript
const treeWeakSet = new WeakSet();

const baobab = { name: "Baobab" };
const jackalberry = { name: "Jackalberry" };
const mopaneTree = { name: "Mopane Tree" };
const breadfruit = { name: "Breadfruit" };

treeWeakSet.add(baobab);
treeWeakSet.add(jackalberry);
treeWeakSet.add(mopaneTree);
treeWeakSet.add(breadfruit);

treeWeakSet.delete(jackalberry);
console.log(treeWeakSet.has(jackalberry)); // false

console.log(treeWeakSet);
```

In der Ausgabe erscheinen die Inhalte des `WeakSet` so:

```javascript
/*
WeakSet {{…}, {…}, {…}}
  [[Entries]]
    0: value: {name: "Mopane Tree"}
    1: value: {name: "Baobab"}
    2: value: {name: "Breadfruit"}
  [[Prototype]]: WeakSet
*/
```

Die Inhalte sind hier sichtbar, weil die Objektvariablen noch starke Referenzen halten, die diese Objekte im Speicher behalten. Beachten Sie, dass WeakSets nicht iterierbar sind, sodass es keine Möglichkeit gibt, ihre Einträge zu durchlaufen oder programmgesteuert darauf zuzugreifen. Was Sie in der Ausgabe sehen, ist nur eine Hilfe beim Debuggen.

Vergessen Sie nicht, dass nur Symbole und Objekte mit klar definierten Schlüsseln und Werten unterstützt werden. Das Hinzufügen eines Primitivs wie Zahlen oder Strings führt zu einem Fehler:

```js
treeWeakSet.add("Alan Smith");

console.log(treeWeakSet); // Invalid value used in weak set
//    at WeakSet.add (<anonymous>)
```

Der entscheidende Unterschied zwischen einem `Set` und einem `WeakSet` ist, dass ein `Set` beliebige Werte speichert, während ein `WeakSet` nur Objekte speichern kann.

Hier sind einige weitere auffällige Unterschiede zwischen einem `Set` und einem `WeakSet`:

| Funktion | Set | WeakSet |
| --- | --- | --- |
| Art der gespeicherten Werte | Speichert beliebige Datentypen | Speichert nur Objekte |
| Referenzierung | Starke Referenzierung | Schwache Referenzierung |
| Iteration | Unterstützt Iteration mit `forEach` und Schleifen | Unterstützt keine Iteration |
| Methoden und Eigenschaften | `add()`, `delete()`, `has()`, `keys()`, `values()`, `size` und mehr | Nur `add()`, `delete()` und `has()` |
| Anwendungsfall | Allgemeine Sammlung einzigartiger Werte und Entfernen von Duplikaten aus Arrays | Effiziente Speicherverwaltung von Objekt-Referenzen |

Sie können die Unterschiede in den Arten der Werte sehen, die die beiden Arten von Sets speichern können, ihre Unterstützung für das Durchlaufen der gespeicherten Objekte und ihre idealen Anwendungsfälle. Bitte nehmen Sie sich einen Moment Zeit, um den Inhalt dieser Tabelle zu lesen.
