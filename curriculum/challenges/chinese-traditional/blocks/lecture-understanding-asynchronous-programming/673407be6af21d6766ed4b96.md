# --description--

在上一課中，我們了解了 Fetch API 是什麼以及如何使用它。在本課中，我們將討論 Fetch API 的 `GET`、`POST`、`PUT` 和 `DELETE` HTTP 方法。

讓我們從最常見的 HTTP 方法開始，那就是 `GET` 方法。它用於從伺服器取得資料。當你使用 `fetch()` 而未指定方法時，它預設為 `GET`。

```js
fetch('https://api.example.com/data')
```

在這段程式碼中，我們對 `GET` 發出了一個 `https://api.example.com/data` 請求。請注意，你不能直接使用這些資料；你必須先將回應轉換成 JSON 格式。只有這樣，你才能在專案中任何地方使用它。以下是如何做到的範例：

```js
fetch('https://api.example.com/data')
  .then(response => response.json())
  .then(data => console.log(data))
```

在這段程式碼中，來自 Fetch API 的回應是一個 promise，我們使用 `.then` 處理常式將回應轉換成 JSON 格式。在之前的課程中，你已經學過 `Promise` 是一個物件，代表非同步程序最終完成（或失敗）及其值。

當 promise 被創建時，其值是未知的。只有當非同步程序完成後，才會知道值。當我們將兩個 `.then` 處理常式串鏈到 fetch 呼叫時，這就是所謂的 promise 串鏈，這會在下一課中教你。

到目前為止，我們一直在從伺服器取得資源。但你知道我們也可以將資料送到伺服器嗎？`POST` 方法用於將資料送到伺服器以創建資源。以下是一個使用 `POST` 方法在伺服器上創建資源的範例：

```js
fetch('https://api.example.com/users', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({
    name: 'John Doe',
    email: 'john@example.com'
  })
})
```

在這個範例中，我們發出一個 `POST` 請求來創建一個新使用者。我們指定方法為 `POST`，設定適當的標頭，並包含一個 `body`，裡面是我們想要送出的資料。body 需要是字串，因此我們使用 `JSON.stringify()` 將物件轉換成 JSON 字串。

`PUT` 方法用於更新伺服器上已存在的資源。以下是一個範例：

```js
fetch('https://api.example.com/users/45', {
  method: 'PUT',
  headers: {
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({
    name: 'John Smith',
    email: 'john@example.com'
  })
})
```

在這個範例中，仔細看 URL，你會看到最後有一個 `45`。這通常用作唯一 ID 來識別我們想要更新的資料。我們在程式碼中使用了 `PUT` 方法，並且指定了作為 `body` 的資料，將用來更新該識別的資料。

`DELETE` 方法用於從伺服器刪除資源。以下是一個範例：

```js
fetch('https://api.example.com/users/45', {
  method: 'DELETE',
})
```

在這段程式碼中，我們包含了 `DELETE` 方法，並在 URL 結尾加上 ID 來識別需要刪除的資料。
