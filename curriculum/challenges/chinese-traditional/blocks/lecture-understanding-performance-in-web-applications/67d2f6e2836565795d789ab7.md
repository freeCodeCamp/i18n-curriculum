# --description--

你是否曾經想過，當你輸入 URL 並按下 Enter 鍵時，幕後發生了什麼？瀏覽器如何將程式碼轉換成我們看到的互動頁面？

了解瀏覽器如何呈現網頁，對於希望最佳化效率並提升使用者體驗的開發者來說至關重要。

在本課程中，我們將揭開 `rendering process` 的神秘面紗，將其拆解成清晰且易於理解的步驟。

首先，瀏覽器解析 HTML 並建置 DOM。

該處理程序從瀏覽器擷取頁面的 HTML 內容開始。它解析此 HTML 以構建文件物件模型（DOM）——一個表述頁面內容的樹狀結構。

請考慮這個簡單的 HTML 片段：

```html
<html>
  <body>
    <h1>Hello, World!</h1>
    <p>Welcome to our website.</p>
  </body>
</html>
```

瀏覽器解析此內容以創建一個以 `html` 為根的 DOM 樹，該樹包含 `body`，而 `body` 又包含 `h1` 和 `p` 元素。

接著，瀏覽器處理 CSS，構建 CSS 物件模型，或稱 CSSOM。這是另一個決定元素應如何被樣式化的樹狀結構。

給定這個 CSS：

```css
h1 {
  color: blue;
}
p {
  font-size: 16px;
}
```

CSS 物件模型指定 `h1` 元素為藍色，且 `p` 元素的字型大小為 16 像素。

瀏覽器接著將 DOM 和 CSS 物件模型結合成呈現樹。此樹僅包含呈現頁面所需的節點，每個節點都與其對應的樣式配對。

以我們的範例來說，呈現樹由 `h1` 和 `p` 元素組成，並適當地套用樣式。

有了呈現樹後，瀏覽器會在一個稱為 layout 或 reflow 的處理程序中計算每個元素的精確位置和大小。這可確保元素在螢幕上出現在正確的位置和尺寸。

最後，瀏覽器將像素繪製到螢幕上，根據計算出的樣式和佈局呈現每個元素。在複雜的頁面中，這可能涉及多個圖層的合成，以形成最終的視覺輸出。

總結來說，從解析 HTML 和 CSS 到在螢幕上繪製像素，瀏覽器的呈現過程是一連串複雜的步驟。
