# --interactive--

JavaScript에서 `null`와 `undefined`는 값이 없음을 나타내는 두 가지 구별되는 데이터 타입이지만, 비교할 때 다르게 동작합니다. 이러한 타입들이 다양한 비교 상황에서 어떻게 상호작용하는지 이해하는 것은 견고하고 버그 없는 코드를 작성하는 데 매우 중요합니다.

먼저 `undefined` 타입부터 시작해 보겠습니다. 변수가 선언되었지만 값이 할당되지 않은 경우 `undefined` 상태입니다. 이는 초기화되지 않은 변수와 인수가 제공되지 않은 함수 매개변수의 기본값입니다.

반면 `null` 타입은 의도적으로 값이 없음을 나타내는 할당된 값입니다. 변수에 의도적으로 값이 없음을 표시할 때 자주 사용합니다.

`null`와 `undefined`를 동등 연산자(`==`)로 비교할 때, JavaScript는 타입 강제 변환을 수행합니다. 즉, 비교 전에 피연산자들을 같은 타입으로 변환하려고 시도합니다. 이 경우 `null`와 `undefined`는 동등하다고 간주됩니다:

:::interactive_editor

```js
console.log(null == undefined); // true
```

:::

하지만 엄격한 동등 연산자(`===`)를 사용할 때는 타입 강제 변환 없이 값과 타입 모두를 검사하므로 `null`와 `undefined`는 같지 않습니다:

:::interactive_editor

```js
console.log(null === undefined); // false
```

:::

이 차이는 조건문을 작성하거나 동등성 검사를 할 때 꼭 기억해야 합니다. `null` 또는 `undefined`를 동등 연산자(`==`)로 다른 값과 비교할 때는 예상치 못한 동작이 발생할 수 있습니다. 예를 들어:

:::interactive_editor

```js
console.log(null == 0);  // false
console.log(null == ''); // false
console.log(undefined == 0); // false
console.log(undefined == ''); // false
```

:::

이 비교들은 `false`를 반환하는데, `null`와 `undefined`는 동등 연산자를 사용할 때 서로(그리고 자기 자신)만 같기 때문입니다. `null`의 다른 비교에서의 동작은 특히 까다롭습니다:

:::interactive_editor

```js
console.log(null > 0);  // false
console.log(null == 0); // false
console.log(null >= 0); // true
```

:::

반면 `undefined`는 숫자 문맥에서 항상 `NaN`로 변환되어, `undefined`와의 모든 숫자 비교가 `false`를 반환합니다:

:::interactive_editor

```js
console.log(undefined > 0);  // false
console.log(undefined < 0);  // false
console.log(undefined == 0); // false
```

:::

이러한 미묘한 차이 때문에, 특히 `null`와 `undefined`를 다룰 때는 값을 비교할 때 엄격한 동등 연산자를 사용하는 것이 일반적으로 권장됩니다. 이 방법은 예상치 못한 타입 강제 변환을 피하고 코드 동작을 더 예측 가능하게 만듭니다.

요약하자면, `null`와 `undefined`는 모두 값이 없음을 나타내지만, 비교할 때 다르게 동작합니다. 이러한 차이를 이해하는 것이 명확하고 오류 없는 JavaScript 코드를 작성하는 데 핵심입니다.
