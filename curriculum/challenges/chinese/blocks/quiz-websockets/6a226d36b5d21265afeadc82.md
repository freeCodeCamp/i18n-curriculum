### --question--

#### --text--

HTTP 通信与 WebSocket 通信有何不同？

#### --distractors--

HTTP 是双向的，而 WebSocket 是单向的。

---

HTTP 保持永久连接，而 WebSocket 每次交换后关闭。

---

HTTP 和 WebSocket 都遵循相同的请求-响应模式。

#### --answer--

HTTP 遵循 `Client --> Request --> Server --> Response`，而 WebSocket 是 `Client <--> Server`。

### --question--

#### --text--

Pub/Sub 消息流是什么？

#### --distractors--

发布者发送给订阅者 --> 订阅者转发给主题 --> 主题保存消息。

---

主题发送消息 --> 发布者接收 --> 发布者转发给订阅者。

---

订阅者连接发布者 --> 发布者发送数据 --> 订阅者确认接收。

#### --answer--

订阅者订阅主题 --> 发布者向该主题发送消息 --> 消息系统将消息传递给该主题的所有订阅者。

以下哪个正确描述了 WebSocket 连接流程？

#### --distractors--

客户端通过 TCP 连接 --> 服务器发送数据 --> 客户端接收 --> 连接关闭。

---

客户端发送数据 --> 服务器请求升级 --> 连接打开 --> 客户端确认。
