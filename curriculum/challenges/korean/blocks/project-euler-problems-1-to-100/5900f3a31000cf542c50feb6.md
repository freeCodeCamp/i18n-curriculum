# --description--

47을 뒤집어서 더하면, 47 + 74 = 121이 되어 회문의 수가 됩니다.

모든 수가 이렇게 빨리 회문을 만들지는 않습니다. 예를 들어,

<div style="margin-left: 4em;">
  349 + 943 = 1292,<br>
  1292 + 2921 = 4213<br>
  4213 + 3124 = 7337<br>
</div>

즉, 349는 회문에 도달하기까지 세 번의 반복이 필요했습니다.

아직 증명된 바는 없지만, 196 같은 일부 수는 절대 회문을 만들지 않는다고 생각됩니다. 뒤집어서 더하는 과정을 통해 회문을 절대 만들지 않는 수를 라이크렐 수라고 합니다. 이 수들의 이론적 특성 때문에, 이 문제에서는 증명되지 않는 한 수가 라이크렐 수라고 가정합니다. 또한 만 개 미만의 모든 수는 (i) 50회 미만의 반복으로 회문이 되거나, (ii) 지금까지 존재하는 모든 컴퓨팅 파워로도 회문으로 만들지 못한 수라고 알려져 있습니다. 실제로 10677은 회문을 만들기 위해 50회 이상의 반복이 필요한 첫 번째 수로 밝혀졌으며, 그 결과는 4668731596684224866951378664 (53회 반복, 28자리)입니다.

놀랍게도, 회문인 수 중에도 라이크렐 수가 있습니다; 첫 번째 예는 4994입니다.

`num` 미만의 수 중 라이크렐 수는 몇 개일까요?

**참고:** 2007년 4월 24일에 라이크렐 수의 이론적 성격을 강조하기 위해 문구가 약간 수정되었습니다.

```js
const countLychrelNumbers = (size) => {
  const numReverse = (num) => {
    return Number(num.toString().split('').reverse().join(''));
  };
  const isPalin = (num) => {
    if (numReverse(num) === num) {
      return true;
    }
    return false;
  };
  let total = 0;
  for (let i = 1; i < size; i++) {
    let loopCount = 1;
    let sum = i;
    while (loopCount < 50) {
      sum = sum + numReverse(sum);
      if (isPalin(sum)) {
        break;
      } else {
        loopCount++;
      }
    }
    if (loopCount === 50) {
      total++;
    }
  }
  return total;
}
```
