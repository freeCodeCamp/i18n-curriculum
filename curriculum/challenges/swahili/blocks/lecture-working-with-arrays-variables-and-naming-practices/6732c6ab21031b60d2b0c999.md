# --description--

Kuitia majina vigezo na vitendakazi ni sehemu muhimu ya kuandika msimbo safi, unaoeleweka na rahisi kudumishwa. Mikataba mizuri ya kuitia majina hufanya msimbo wako uwe na maelezo yenyewe, kupunguza haja ya maoni mengi na kufanya iwe rahisi kwa wasanidi programu wengine ikiwa ni pamoja na wewe mwenyewe baadaye kuelewa msimbo wako.

Tuanze na mikataba ya jumla ya kuitia majina katika JavaScript. Katika mafundisho ya awali ulijifunza kuhusu kutumia camel case kwa majina ya vigezo. Kwa vigezo vya thamani ya kweli au siyo kweli, ni mkataba wa kawaida kutumia viambishi awali kama `is`, `has`, au `can`. Hii mara moja inamwambia msomaji kuwa kigezo ni thamani ya kweli au siyo kweli:

```js
let isLoading = true;
let hasPermission = false;
let canEdit = true;
```

Kwa vitendakazi jina linapaswa kuonyesha wazi kinachofanywa na kitendakazi hicho. Mara nyingi ni msaada kuanza na kitenzi:

```js
function getUserData(){
  /* ... */
}

function calculateTotal(){
  /* ... */
}

function validateInput(){
  /* ... */
}
```

Kwa vitendakazi vinavyorejesha thamani ya kweli au siyo kweli vinavyoitwa mara nyingi predicates, unaweza kutumia viambishi awali sawa vya `is`, `has`, au `can`:

```js
function isValidEmail(email) {
  /* ... */
}

function hasRequiredFields(form) {
  /* ... */
}
```

Unapokuwa na vitendakazi vinavyopata data ni kawaida kuanza na neno `get`:

```js
function getProductDetails(productId) {
  /* ... */
}

function getUserProfile(userId) {
  /* ... */
}
```

Unapokuwa na vitendakazi vinavyoweka data ni kawaida kuanza na neno `set`:

```js
function setUserPreferences(preferences) {
  /* ... */
}

function setPageTitle(title) {
  /* ... */
}
```

Kwa vitendakazi vya mshughulikiaji wa tukio, unaweza kuweka kiambishi awali `handle` au kiambishi tamati `handler`:

```js
function handleClick(){
  /* ... */
}

function onSubmit(){
  /* ... */
}

function keyPressHandler(){
  /* ... */
}
```

Mshughulikiaji wa tukio ni kitendo kinachotokea baada ya tukio kama kubofya kitufe. Utajifunza kuhusu hilo katika mafundisho yajayo.

Unapotia majina vigezo vya mzunguko na mizunguko, ni kawaida kutumia herufi moja kama `i`, `j`, au `k`, lakini kwa mizunguko iliyopangwa ndani au mizunguko tata zaidi majina yenye maelezo zaidi yanaweza kusaidia:

```js
for (let i = 0; i < array.length; i++) {
  /* ... */
}

for (let studentIndex = 0; studentIndex < students.length; studentIndex++) {
  /* ... */
}
```

Kwa majina ya safu ya data fikiria kutumia nomino za wingi kuonyesha kuwa kigezo kina vitu vingi:

```js
const colors = ['red', 'green', 'blue'];
const userNames = ['Alice', 'Bob', 'Charlie'];
```

Kumbuka lengo ni kufanya msimbo wako uwe na maelezo yenyewe iwezekanavyo. Kanuni nzuri ni kwamba kama unahitaji kuongeza maoni kuelezea kinachofanywa na kigezo au kitendakazi, huenda ukahitaji kufikiria kuitia jina lingine lenye maelezo zaidi.

Mwishowe, kuwa na mkataba thabiti katika msimbo wako au timu. Ikiwa timu yako ina mikataba ya kuitia majina iliyowekwa, zingatia hiyo. Ulinganifu hufanya msimbo uwe rahisi kusoma na kudumishwa kwa kila mtu anayehusika.
