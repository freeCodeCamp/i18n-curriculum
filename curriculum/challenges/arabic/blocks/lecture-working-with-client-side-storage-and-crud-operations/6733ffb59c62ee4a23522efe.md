# --interactive--

`IndexedDB` هو أداة لتخزين البيانات المهيكلة في المتصفح. هذه الأداة مدمجة في متصفحات الويب الحديثة، مما يسمح لتطبيقات الويب بتخزين واسترجاع كائنات JavaScript بكفاءة.

على عكس آليات التخزين الأخرى مثل `localStorage`، التي تقتصر على تخزين السلاسل النصية، يمكن لـ `IndexedDB` تخزين كائنات JavaScript، والملفات، وأي نوع آخر من البيانات تقريبًا. هذا يجعل من السهل على تطبيقات الويب التي تحتاج إلى التعامل مع هياكل بيانات كبيرة ومعقدة.

الآن، دعنا نلقي نظرة على كيفية عمل `IndexedDB`.

الخطوة الأولى هي فتح قاعدة بيانات. إليك مثالًا:

:::interactive_editor

```js
let request = indexedDB.open("Sample DB", 1);

request.onerror = function(event) {
  console.log("Error opening database");
};

request.onsuccess = function(event) {
  let db = event.target.result;
  console.log("Database opened successfully");
};
```

:::

في هذا الشفرة، نفتح قاعدة بيانات باسم `"Sample DB"` بالإصدار 1. نوفر دالتين استدعاء راجع: واحدة لمعالجة الأخطاء، وأخرى عند فتح قاعدة البيانات بنجاح. الكائن `db` الذي نحصل عليه في دالة الاستدعاء الناجحة هو ما سنستخدمه للتفاعل مع قاعدة البيانات.

إذا تحققت من واجهة تطبيق أدوات المطور في المتصفح، سترى قاعدة البيانات `Sample DB` في قسم `IndexedDb` قد تم فتحها. بمجرد فتح قاعدة البيانات، يمكنك البدء بالعمل مع مخازن الكائنات.

مخازن الكائنات في IndexedDB تشبه الجداول في قواعد البيانات التقليدية. فهي تحتفظ بالبيانات الفعلية التي تريد تخزينها. إليك كيفية إنشاء مخزن كائنات:

```js
let request = indexedDB.open("Sample DB", 1);

request.onupgradeneeded = function(event) {
  let db = event.target.result;
  let objectStore = db.createObjectStore("customers", { keyPath: "id" });
};
```

هذا الشفرة تنشئ مخزن كائنات باسم `"customers"` مع `"id"` كمسار المفتاح. مسار المفتاح يشبه المفتاح الأساسي في قاعدة البيانات التقليدية - يُستخدم لتمييز كل سجل بشكل فريد.

لإضافة بيانات إلى مخزن الكائنات، نفعل شيئًا مثل هذا. الكائن `db` في هذا المثال يمثل نسخة قاعدة بيانات `IndexedDB`.

```js
let transaction = db.transaction(["customers"], "readwrite");
let objectStore = transaction.objectStore("customers");
let request = objectStore.add({ id: 1, name: "John Doe", email: "john@example.com" });

request.onerror = function(event) {
  console.log("Error adding data");
};

request.onsuccess = function(event) {
  console.log("Data added successfully");
};
```

هذا الشفرة تضيف زبونًا جديدًا إلى مخزن الكائنات `"customers"`. نبدأ معاملة (وهي الطريقة التي نجمع بها عمليات قاعدة البيانات)، نحصل على مرجع لمخزن الكائنات، ثم نضيف بياناتنا.

استرجاع البيانات يعمل بطريقة مشابهة. نبدأ معاملة، نحصل على مخزن الكائنات، ثم نستخدم طرقًا مثل `get` لاسترجاع البيانات:

```js
let transaction = db.transaction(["customers"]);
let objectStore = transaction.objectStore("customers");
let request = objectStore.get(1);

request.onerror = function(event) {
  console.log("Error retrieving data");
};

request.onsuccess = function(event) {
  console.log("Customer:", request.result);
};
```

هذا الشفرة تسترجع الزبون الذي له `id` يساوي `1` من مخزن الكائنات `"customers"`.

واحدة من الميزات الرئيسية لـ `IndexedDB` هي أنها غير متزامنة.

هذا يعني أنه عند التفاعل مع `IndexedDB`، لا تحجب العمليات الخيط الرئيسي لتطبيق الويب. هذا يضمن بقاء تطبيق الويب متجاوبًا حتى عند التعامل مع كميات كبيرة من البيانات.

بينما يوفر `IndexedDB` قدرات قوية، فإن منحنى تعلمه أكثر حدة مقارنة بخيارات واجهات برمجة التطبيقات للتخزين الأبسط، مما قد يكون تحديًا للمبتدئين. مع ذلك، للتطبيقات التي تحتاج إلى التعامل مع كميات كبيرة من البيانات المهيكلة على جانب العميل، يقدم `IndexedDB` قدرات لا مثيل لها.
