# --interactive--

Unapofanya kazi na CSS, mara nyingi unakutana na aina tatu tofauti za tabia za kuonyesha vipengele: `inline`, `block`, na `inline-block`.

Kila moja ya sifa hizi huathiri jinsi vipengele vinavyopangwa na jinsi vinavyoshirikiana na vipengele vingine kwenye ukurasa.

Katika funzo hili, tutazingatia jinsi sifa ya `inline-block` inavyofanya kazi na jinsi inavyotofautiana na vipengele vya ndani ya mstari (inline) na vipengele vya kiwango cha kizuizi (block-level).

Vipengele vya kiwango cha kizuizi hufanya kazi kama makontena makubwa au "vibao" vinavyochukua upana mzima wa kontena lao la mzazi. Daima huanza kwenye mstari mpya, na urefu na upana wake vinaweza kubadilishwa.

Vipengele vya ndani ya mstari (inline) vinachukua nafasi tu kadri inavyohitajika. Vinatiririka ndani ya maudhui yanayozunguka na havivunji mstari mpya.

Sifa ya `inline-block` ni mchanganyiko wa tabia hizi mbili. Kama vipengele vya ndani ya mstari, vipengele vya `inline-block` hubaki katika mtiririko wa maandishi bila kuanza mstari mpya.

Hata hivyo, tofauti na vipengele vya ndani ya mstari, unaweza kubadilisha upana na urefu wa kipengele cha `inline-block`, kama unavyofanya kwa vipengele vya kiwango cha kizuizi.

Kwa kifupi, tofauti kuu kati ya `inline` na `inline-block` ni kwamba vipengele vya `inline` haviwezi kudhibitiwa ukubwa wake, wakati vipengele vya `inline-block` vinaruhusu udhibiti kamili wa vipimo huku vikibaki ndani ya mstari pamoja na maudhui mengine.

Tuchunguze mfano mmoja.

:::interactive_editor

```html
<link href="styles.css" rel="stylesheet">

<div class="container">
  <span class="inline-block-element element1">Inline-Block</span>
  <span class="inline-block-element element2">Inline-Block</span>
</div>
```

```css
.inline-block-element {
  display: inline-block;
  width: 150px;
  height: 100px;
}

.element1 {
  background-color: lightblue;
}

.element2 {
  background-color: lightgreen;
}
```

:::

Katika mfano huu wa juu, tuna `div` yenye darasa la `container`. Ndani ya kipengele hicho cha `div`, tuna vipengele viwili vya `span`.

Kila kipengele cha span kimewekwa kuwa `display:inline-block` na kina upana na urefu uliowekwa pia.

Vipengele vya inline-block vitaonekana pembeni kwa pembeni kwa sababu hufanya kazi kama vipengele vya ndani ya mstari, lakini pia vina upana na urefu uliobainishwa, jambo linalowapa sifa za kipengele cha kizuizi.

Lakini, ukiondoa sifa ya `display: inline-block`, wala `height` wala `width` haviwezi kutekelezwa ingawa uliibainisha wazi.

Hapa kuna CSS iliyorekebishwa:

:::interactive_editor

```html
<link href="styles.css" rel="stylesheet">

<div class="container">
  <span class="inline-block-element element1">Span element</span>
  <span class="inline-block-element element2">Span element</span>
</div>
```

```css
.inline-block-element {
  width: 150px;
  height: 100px;
}

.element1 {
  background-color: lightblue;
}

.element2 {
  background-color: lightgreen;
}
```

:::

Katika msimbo huu, tumeondoa sifa ya `display: inline-block;` lakini tumebaki na kila kitu kingine kama kilivyo. Hapa, vipengele vya `span` vinarudi kwenye tabia yao ya kawaida kama vipengele vya ndani ya mstari.

Matokeo yake, upana na urefu uliobainishwa havizingatiwi, na vipengele vinachukua nafasi tu inayohitajika kwa maudhui yao.

Kuelewa jinsi `inline-block` inavyofanya kazi ni muhimu kwa sababu unaweza kuitumia kuunda mipangilio inayohitaji ulinganifu na udhibiti wa vipimo ndani ya mtiririko wa maandishi unaoendelea.
