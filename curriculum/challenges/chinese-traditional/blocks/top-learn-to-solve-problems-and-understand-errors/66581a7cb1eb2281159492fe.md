# --description--

假設你已經撰寫了以下程式碼：

```javascript
const a = "Hello";
const b = "World";

console.log(c);
```

這段程式碼會執行，但會產生錯誤。在技術術語中，這稱為 <dfn>發出</dfn> 錯誤。錯誤的第一部分會顯示錯誤的型別。這提供了你正在處理的問題的第一個線索。你將在課程後面學習更多不同的錯誤型別。在這個範例中，你有一個 `ReferenceError`。

<img src="https://cdn.statically.io/gh/TheOdinProject/curriculum/175b5ef2a1b4758a7b75f4ef43d7e27203e5707b/foundations/javascript_basics/understanding_errors/imgs/00.png" width="100%" alt="An error being displayed in the developer console"/>

當有人取用在目前範圍內未宣告和／或初始化的變數時，會發出 `ReferenceError`。在我們的例子中，錯誤訊息說明錯誤發生是因為 `c is not defined`。

此類型的不同錯誤會根據造成 `ReferenceError` 的原因顯示不同的訊息。<dfn>例如</dfn>，你可能會遇到另一個訊息是 `ReferenceError: can't access lexical declaration 'X' before initialization`。

錯誤的下一個零件會告訴我們你可以找到錯誤的檔案名稱（在這個例子中是我們的 `script.js`），以及行號。

這讓你可以輕鬆地導覽到程式碼中有問題的列。這裡，錯誤來自 `script.js` 的第四列，該列以聯結形式顯示在錯誤訊息下方，文字為 `script.js:4`。如果你點擊此聯結，大多數瀏覽器會在開發者工具的 Sources 分頁中導覽到該精確的程式碼列及其後的程式碼。

有時你的瀏覽器主控台也會顯示錯誤發生的行中行（或字元）。在我們的範例中，這會是在 `script.js:4:13`。
