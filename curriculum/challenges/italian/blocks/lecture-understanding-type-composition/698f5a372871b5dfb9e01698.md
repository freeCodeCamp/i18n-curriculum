# --description--

Nelle lezioni precedenti, hai imparato a lavorare con tipi primitivi come `string` e `boolean`. Ma TypeScript ti permette di creare nuovi tipi a partire da quelli esistenti. Un esempio di questo è il tipo union.

Un tipo union è un tipo che combina due o più tipi diversi.

Ecco la sintassi base:

```ts
type1 | type2 | type3
```

Diamo un’occhiata ad alcuni esempi per capire meglio il concetto.

Ecco una variabile chiamata `id` con un tipo union per `string` o `number`:

```ts
let id: string | number = "user_783";
```

Il valore può essere aggiornato a un numero, che è comunque valido per il tipo union:

```ts
let id: string | number = 1024;
```

Se provi ad assegnare un valore che non è né `string` né `number`, TypeScript mostrerà un messaggio di errore:

```ts
// Type 'string[]' is not assignable to type 'string | number'.
let id: string | number = ["Java", "Python"];
```

Quando lavori con tipi union, è importante usare metodi che funzionano per tutti i tipi coinvolti, altrimenti TypeScript lancerà un errore come questo:

```ts
function getId(id: string | number) {
  // Property 'toUpperCase' does not exist on type 'string | number'.
  return id.toUpperCase();
}
```

Esiste una soluzione che consiste nel restringere il tipo union. Ma imparerai di più su questo e su come i tipi union funzionano con gli alias di tipo nelle prossime lezioni.

I tipi union sono utili anche quando lavori con valori `null` e `undefined`.

Ecco un esempio in cui una variabile potrebbe contenere un valore `string`, oppure potrebbe essere `null` se non è stato ancora impostato alcun valore:

```ts
let username: string | null;

// Type 'number' is not assignable to type 'string | null'.
username = 42;
```

In questo caso, `username` può essere o un `string` o un `null`, ma nient’altro. Potresti usare `null` quando vuoi rappresentare intenzionalmente l’assenza di un valore.

Ecco un altro esempio che usa `undefined`:

```ts
let score: number | undefined;

score = undefined; 

// Type 'string' is not assignable to type 'number | undefined'.
score = "high";
```

In questo esempio, `score` può essere o un `number` o un `undefined`. Potresti usare `undefined` quando un valore non è ancora stato assegnato.

Usare tipi union con `null` e `undefined` rende chiaro quando un valore potrebbe mancare, permettendo comunque a TypeScript di intercettare assegnazioni non valide.
