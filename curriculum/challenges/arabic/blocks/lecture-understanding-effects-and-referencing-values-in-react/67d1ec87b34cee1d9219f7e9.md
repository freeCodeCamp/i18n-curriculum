# --description--

يوفر React العديد من الهوكس المدمجة التي تتيح لك تنفيذ ميزات مختلفة في مشاريعك. تشمل هذه `useState` و`useEffect` و`useContext` وغيرها.

لكن أحيانًا، تحتاج إلى إضافة ميزة لا يمكن لأي من الهوكس المدمجة مساعدتك بها. لحسن الحظ، يمكنك إنشاء هوكس مخصصة خاصة بك في React.

الهوكس المخصصة ليست معقدة كما قد تبدو. هي مجرد دوال قابلة لإعادة الاستخدام تتيح لك مشاركة المنطق عبر مكوّنات متعددة. هذا يعني أن قابلية إعادة الاستخدام سبب آخر يدفعك لبناء هوك خاص بك.

مع هوك مخصصة، يمكنك استخراج المنطق بعيدًا عن أي مكوّنات تستخدمها، مثل جلب البيانات، إدارة الحالة، التبديل، التأثيرات الجانبية مثل التحقق من حالة الاتصال بالإنترنت أو عدمه، وهكذا.

يمكنك بعد ذلك استيراد الهوك لاستخدامها في أي مكوّن، فتتمكن من التركيز على العرض والتقديم داخل تلك المكوّنات. هذا يعني تكرارًا أقل وازدواجية أقل، مما يعني أماكن أقل لإجراء تغييرات عند الرغبة في تحديثات.

الآن، لنلقِ نظرة على كيفية إنشاء هوك مخصصة خاصة بك.

في React، تبدأ جميع الهوكس المدمجة بكلمة `use`، لذا يجب أن تتبع هوكك المخصصة نفس الاصطلاح. يجب أن يعبر اسم هوكك المخصصة بوضوح عن وظيفتها.

لذا، إذا كانت هوكك المخصصة…

- تجلب بيانات، يمكنك تسميتها `useFetch`

- تبدّل شيئًا ما تشغيلًا وإيقافًا، يمكنك تسميتها `useToggle`

- أو إذا كانت تنفذ تقنية debouncing، فإن `useDebounce` اسم جيد

لنفترض أنك تريد بناء هوك مخصصة لإضافة تقنية debouncing إلى تطبيقك.

تقنية debouncing هي تقنية برمجية تحد من عدد مرات تشغيل دالة. تعمل عن طريق الانتظار حتى يتوقف المستخدم عن أداء فعل معين لفترة زمنية محددة قبل تنفيذ الدالة. على سبيل المثال، في مربع البحث، بدلًا من إجراء استعلام API مع كل ضغطة مفتاح، تنتظر تقنية debouncing حتى يتوقف المستخدم عن الكتابة لمدة، لنقل، 500 مللي ثانية.

لإنشاء هوك مخصصة لتقنية debouncing، تحتاج أولًا إلى إنشاء ملف `useDebounce.jsx` أو `useDebounce.js`. تقليديًا، تُحفظ ملفات الهوكس المخصصة التي تنشئها في مجلد `hooks`.

يمكنك استخدام بعض الهوكس المدمجة داخل هوكك المخصصة. لتقنية debouncing، تحتاج إلى هوك `useState` و`useEffect`، لذا استوردهما في أعلى ملفك:

```js
import { useState, useEffect } from "react";
```

بعد ذلك، أنشئ دالة `useDebounce` التي تأخذ `value` و`delay` كمعلمات. `value` هو المورد الذي تريد الانتظار له، و`delay` هي الفترة الزمنية التي تريد الانتظار خلالها. بما أنك تريد الانتظار لفترة زمنية، ستكون دالتي `setTimeout` و`clearTimeout` مفيدتين:

```js
function useDebounce(value, delay) {
  const [debouncedValue, setDebouncedValue] = useState(value);

  useEffect(() => {
    const handler = setTimeout(() => {
      setDebouncedValue(value);
    }, delay);

    return () => {
      clearTimeout(handler);
    };
  }, [value, delay]);

  return debouncedValue;
}

export { useDebounce };
```

حالة `debouncedValue` تحتفظ بالقيمة المؤجلة وتُرجعها، والتي تتحدث فقط بعد انتهاء فترة المهلة المحددة.

هنا يحدث السحر في `useEffect`. إذا تذكرت من الدرس السابق، أي شيء خارج دورة عرض React، مثل تعيين ومسح مؤقت، هو تأثير جانبي، ويجب استخدام هوك `useEffect` للتعامل معه.

داخل هوك `useEffect` هنا، تستخدم `setTimeout` لتعيين `debouncedValue`. ثم تُرجع دالة تنظيف تستخدم `clearTimeout` لمسح المهلة السابقة كلما تغيرت `value` أو `delay`، أو عند إزالة المكوّن.

وهنا مكوّن `FootballerSearch` يستخدم هوك `useDebounce` لتأخير البحث لمدة ثانية واحدة بعد توقف المستخدم عن الكتابة:

```jsx
import { useState, useEffect } from "react";
import { useDebounce } from "./hooks/useDebounce";
import footballers from "./footballers";

const FootballerSearch = () => {
  const [query, setQuery] = useState("");
  const debouncedQuery = useDebounce(query, 1000); // Start searching 1 second after the user stops typing

  useEffect(() => {
    if (debouncedQuery) {
      const results = footballers.filter((footballer) =>
        footballer.toLowerCase().includes(debouncedQuery.toLowerCase()),
      );
      console.log("Search results:", results);
    } else {
      console.log("Search results: []");
    }
  }, [debouncedQuery]);

  return (
    <>
      <h1 style={{ textAlign: "center" }}>Footballer Search App</h1>
      <div style={{ textAlign: "center" }}>
        <input
          style={{ padding: "0.5rem", width: "30%" }}
          type="text"
          value={query}
          onChange={(e) => setQuery(e.target.value)}
          placeholder="Search for a footballer..."
        />
      </div>
    </>
  );
};

export default FootballerSearch;
```

كما ترى، المتغير `debouncedQuery` هو الذي يهيئ هوك `useDebounce` بحالة الاستعلام (ما يكتبه المستخدم)، والتأخير لمدة 1000 مللي ثانية، أو ثانية واحدة. يتم التعامل مع البحث داخل هوك `useEffect`، وتُسجل نتائج البحث في وحدة التحكم.
