# --interactive--

En la lección anterior, aprendiste cómo trabajar con el objeto `arguments`, que es un objeto similar a un arreglo que contiene los valores de los argumentos pasados a la función.

:::interactive_editor

```js
function logArgs() {
  for (const arg of arguments) {
    console.log(arg);
  }
}

logArgs(1, 2, 3);
// result:
// 1
// 2
// 3
```

:::

Aunque esta es una forma válida de acceder y trabajar con un conjunto variable de argumentos de una función, las aplicaciones modernas de JavaScript usarán en su lugar la sintaxis de parámetro rest.

Aquí tienes un ejemplo actualizado usando la sintaxis de parámetro rest en lugar del objeto `arguments`:

:::interactive_editor

```js
function logArgs(...args) {
  for (const arg of args) {
    console.log(arg);
  }
}

logArgs(1, 2, 3);
// result:
// 1
// 2
// 3
```

:::

Este ejemplo actualizado ya no hace referencia directa al objeto `arguments`. En cambio, el último parámetro de la definición de la función tiene tres puntos delante. Esto hace que este parámetro rest se coloque dentro de un objeto `Array`. Puedes nombrar este parámetro rest como quieras. Solo asegúrate de que sea el último parámetro en la definición de la función, así:

```js
function exampleFunction(a, b, ...restOfArgs) {
  // some code here
}

function anotherFunction(x, y, ...theArgs) {
  // some code here
}
```

Hay algunas restricciones más al trabajar con la sintaxis de parámetro rest. Una restricción es que las definiciones de función solo pueden tener un parámetro rest. Así que el siguiente ejemplo aquí se consideraría inválido.

```js
// Won't work

function badFunction(...args, ...moreArgs) {
  // some code here
}
```

La siguiente restricción es que no se permiten comas finales después del parámetro rest: `function exampleFunction(a, b, ...restOfArgs, )`.

Otra restricción es que el parámetro rest no puede tener un valor por defecto. De lo contrario, se lanzará un `SyntaxError`.

```js
function badFunction(...args = [1,2]){
  // some code here
}
```

Entonces, ¿cuáles son algunas diferencias entre el objeto `arguments` y los parámetros `rest`?

Una diferencia clave es que el objeto `arguments` no es un arreglo real, por lo que no soporta métodos como `includes`, `pop` y `push`. Pero el parámetro rest es una instancia de `Array`. Así que puedes usar métodos válidos incorporados de arreglos sin necesidad de convertirlo primero en un arreglo real.

:::interactive_editor

```js
function hasCat(...args) {
  return args.includes("cat");
}

console.log(hasCat("dog", "chicken", "cat")); // true
console.log(hasCat("dog", "chicken", "horse")); // false
```

:::
