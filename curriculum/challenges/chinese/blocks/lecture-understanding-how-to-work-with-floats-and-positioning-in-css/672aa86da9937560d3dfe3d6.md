# --interactive--

CSS 中的浮动是一种技术，最初是为了让文本围绕图像等元素而设计的。 不过，随着时间的推移，开发人员发现了使用浮动的新方法，并以创造性的方式将其应用到布局设计中。 虽然 Flexbox 和 Grid 等现代布局方法现在更常用，但了解浮动仍很重要，尤其是在处理旧代码或需要实现特定布局效果时。

当一个元素被设置为浮动时，它会脱离正常的文档流，被推到其容器的左边或右边。 随后的内容将环绕浮动元素，填充剩余空间。 一个经典的用法是在图片周围包裹文字，即图片浮动到一边，文字包裹在图片周围。 这种技术仍在广泛使用，尤其是在文章和博客中，图片需要与文字放在一起。 这是代码示例：

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<div class="container">
  <img src="https://placehold.co/150x150" alt="Placeholder Image">
</div>
```

```css
.container {
  border: 1px solid black;
  padding: 10px;
}

img {
  float: left;
  margin-right: 20px;
}
```

:::

在上面的示例中，容器并没有包围浮动图像。 图像脱离了正常的文档流，而容器由于没有“看到”浮动的子元素，因此高度被坍缩为零。 

在 Flexbox 和 Grid 技术普及之前，浮动也是创建多栏布局的常用工具。 通过浮动相邻的元素，开发人员可以创建水平排列的列。 不过，在使用浮动元素时，重要的是要处理父元素在其子元素浮动时的坍缩问题。  The clearfix technique solution is applied to the class `container` element to fix this issue.

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<div class="container">
  <img src="https://placehold.co/150x150" alt="Placeholder Image">
  <p> This is an example of text flowing around a floated image.</p>
</div>
```

```css
.container {
  border: 1px solid black;
}

/* Clearfix CSS */
.container::after {
  content: "";  
  display: block;
  clear: both;
}

img {
  float: left;
  margin-right: 20px;
}
```

:::

- `::after` 是一个伪元素，用于在容器内容后添加一个不可见的块。
- `content: ""` 确保伪元素存在，但不显示任何内容。
- `display: block` 使伪元素成为块级元素。
- `clear: both` 确保伪元素清除其上方的任何左右浮动元素。

在上面的示例中，我们为你的文档添加了一个新建的段落元素，以使折叠更明显。因为段落仍然处于正常的文档流中，容器会刚好扩展以包裹它。然后，我们应用 clearfix 技术来修复折叠并使容器的边框正确显示。你可以尝试移除 clearfix 规则以再次看到容器折叠。

Clearfix 技术可确保父元素正确地环绕浮动子元素。 Clearfix 通过在浮动内容后添加 clear 属性，强制父容器“看到”浮动的子元素。

虽然由于 Flexbox 和 Grid 等更现代的技术的出现，浮动不再是复杂布局的首选方法，但在某些情况下，浮动仍然发挥着重要作用。 无论是在图像周围包装文字，还是在传统项目中工作，了解浮动的工作原理以及如何正确清除浮动，都是保持结构合理、响应式的布局的关键。
