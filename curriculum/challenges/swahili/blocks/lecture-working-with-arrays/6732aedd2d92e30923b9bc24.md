# --interactive--

Ufumbuzi wa muundo wa safu ya data ni kipengele katika JavaScript kinachokuwezesha kutoa thamani kutoka kwa safu za data na kuziweka katika vigezo kwa njia fupi na inayosomeka kwa urahisi zaidi. Hutoa sintaksia rahisi ya kufungua vipengele vya safu ya data na kuviweka katika vigezo tofauti.

Mbinu hii ni muhimu sana unapotumia safu za data na vitendakazi vinavyorejesha thamani nyingi. Hapa kuna mfano wa kutumia ufumbuzi wa muundo wa safu ya data:

:::interactive_editor

```js
let fruits = ["apple", "banana", "orange"];

let [first, second, third] = fruits;

console.log(first);  // "apple"
console.log(second); // "banana"
console.log(third);  // "orange"
```

:::

Katika mfano huu, tuna safu ya data iitwayo `fruits` yenye vipengele vitatu. Kwa kutumia ufumbuzi wa muundo wa safu ya data, tunaweka kipengele cha kwanza katika kigezo `first`, kipengele cha pili katika `second`, na kipengele cha tatu katika `third`. Hii inatuwezesha kufikia vipengele binafsi vya safu bila kutumia kielezo cha nambari.

Hivi ndivyo itakavyokuwa kama ungetumia kielezo kufikia kila kipengele badala ya kutumia ufumbuzi wa muundo wa safu ya data:

:::interactive_editor

```js
const fruits = ["apple", "banana", "orange"];

const first = fruits[0];
const second = fruits[1];
const third = fruits[2];

console.log(first); // "apple"
console.log(second); // "banana"
console.log(third); // "orange"
```

:::

Ufumbuzi wa muundo wa safu ya data pia unakuwezesha kuruka vipengele ambavyo huna nia navyo kwa kutumia koma. Kwa mfano:

:::interactive_editor

```js
let colors = ["red", "green", "blue", "yellow"];
let [firstColor, , thirdColor] = colors;

console.log(firstColor); // "red"
console.log(thirdColor); // "blue"
```

:::

Katika mfano huu, tunaruka kipengele cha pili cha safu ya `colors` kwa kutumia koma ya ziada. Hii inafanya `red` kuwekwa katika `firstColor` na `blue` kuwekwa katika `thirdColor`, huku `green` ikipuuzwa kabisa.

Kipengele kingine chenye nguvu cha ufumbuzi wa muundo wa safu ya data ni uwezo wa kutumia thamani za msingi. Ikiwa safu ina vipengele vichache kuliko vigezo unavyotaka kuweka, unaweza kutoa thamani za msingi:

:::interactive_editor

```js
let numbers = [1, 2];
let [a, b, c = 3] = numbers;

console.log(a); // 1
console.log(b); // 2
console.log(c); // 3
```

:::

Hapa, tunaweka thamani ya msingi `3` kwa `c` kwa sababu safu ya `numbers` haina kipengele cha tatu.

Sasa, tuchambue sintaksia ya REST, inayotambulika kwa nukta tatu (`...`). Inakuwezesha kukamata vipengele vilivyobaki vya safu ambavyo havijafumbuliwa na kuviweka katika safu mpya. Hivi ndivyo inavyofanya kazi:

:::interactive_editor

```js
let fruits = ["apple", "banana", "orange", "mango", "kiwi"];
let [first, second, ...rest] = fruits;

console.log(first);  // "apple"
console.log(second); // "banana"
console.log(rest);   // ["orange", "mango", "kiwi"]
```

:::

Katika mfano huu, `first` na `second` zinakamata vipengele viwili vya kwanza vya safu ya `fruits`, na `rest` inakamata vipengele vyote vilivyobaki kama safu mpya. Sintaksia ya REST lazima iwe kipengele cha mwisho katika muundo wa ufumbuzi.

Ufumbuzi wa muundo wa safu ya data ni kipengele chenye nguvu kinachoweza kufanya msimbo wako kuwa mfupi na rahisi kusoma. Ni muhimu hasa unapotumia safu za data, na unapotaka kutoa vipengele maalum kutoka kwa safu.
