# --description--

In Express können Sie **verkettbare Route-Handler** mit der `app.route()`-Methode erstellen. Das ist hilfreich, wenn Sie mehrere HTTP-Methoden wie GET, POST oder PUT für denselben Routenpfad behandeln möchten, ohne diesen Pfad jedes Mal zu wiederholen.

Anstatt separate Routendefinitionen wie `app.get("/user")`, `app.post("/user")` und `app.put("/user")` zu schreiben, ermöglicht Ihnen `app.route()`, diese in einem einzigen, übersichtlichen Codeblock zusammenzufassen.

Die `app.route()`-Methode fungiert als Abkürzung, um mehrere Handler für denselben Endpunkt zu definieren. Das ist besonders nützlich bei RESTful-APIs, bei denen eine einzelne Ressource (wie `/user`) oft mehrere Operationen unterstützt.

Hier ist ein Beispiel, das eine RESTful-API zur Verwaltung von Benutzern simuliert:

```javascript
const express = require("express");
const app = express();
const port = 3000;

app.use(express.json()); // Parse JSON request bodies

// Use app.route() to group handlers for the same path
app
  .route("/user")
  .get((req, res) => {
    res.send("Fetching user data");
  })
  .post((req, res) => {
    const { name } = req.body;
    res.send(`Creating a user with name: ${name}`);
  })
  .put((req, res) => {
    const { name } = req.body;
    res.send(`Updating the user to: ${name}`);
  })
  .delete((req, res) => {
    res.send("Deleting the user");
  });

app.listen(port, () => {
  console.log(`Server running at http://localhost:${port}`);
});
```

Im obigen Beispiel definiert `app.route("/user")` eine einzelne Route für den `/user`-Pfad. Dann verketten wir mehrere HTTP-Methoden, `.get()`, `.post()`, `.put()` und `.delete()`, um verschiedene Arten von Anfragen für dieselbe Route zu behandeln.

Hier eine Aufschlüsselung der verschiedenen Routen und wie sie in einer einfachen RESTful-Anwendung funktionieren könnten:

* GET `/user`: Ruft Benutzerdaten ab und gibt sie zurück.
    
* POST `/user`: Erstellt einen neuen Benutzer mit Daten im Anfrage-Body.
    
* PUT `/user`: Aktualisiert einen bestehenden Benutzer mit Daten im Anfrage-Body.
    
* DELETE `/user`: Löscht einen Benutzer.
    

Die Verwendung von `app.route()` hilft, Ihren Code organisiert zu halten, wenn Sie mehrere HTTP-Methoden für denselben Pfad behandeln. Anstatt die Route jedes Mal zu wiederholen, fassen Sie verwandte Handler an einem Ort zusammen.

Das reduziert Redundanz und macht Ihren Code leichter lesbar und wartbar. Wenn alle Methoden für eine Route in einem einzigen Block definiert sind, ist klarer ersichtlich, wie sich dieser Endpunkt verhält.

Wenn Ihre Anwendung wächst und die Routen komplexer werden, bietet `app.route()` eine einfache Struktur, die zusammengehörige Logik zusammenhält. Das ist besonders nützlich beim Entwerfen von RESTful-APIs, bei denen eine einzelne Ressource oft mehrere Operationen wie GET, POST, PUT und DELETE unterstützt.
