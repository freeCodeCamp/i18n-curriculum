# --interactive--

## أساسيات الكائن

- **التعريف**: الكائن هو هيكل بيانات يتكون من خصائص. تتألف الخاصية من مفتاح وقيمة. للوصول إلى البيانات من كائن يمكنك استخدام ترميز النقطة أو ترميز الأقواس.

:::interactive_editor

```js
const person = {
  name: "Alice",
  age: 30,
  city: "New York"
};

console.log(person.name);  // Alice
console.log(person["name"]); // Alice
```

:::

لتعيين خاصية لكائن موجود يمكنك استخدام ترميز النقطة أو ترميز الأقواس مع معامل التعيين.

:::interactive_editor

```js
const person = {
  name: "Alice",
  age: 30
};

person.job = "Engineer"
person["hobby"] = "Knitting"
console.log(person);  // {name: 'Alice', age: 30, job: 'Engineer', hobby: 'Knitting'}
```

:::

## حذف الخصائص من الكائن

- **معامل `delete`**: يُستخدم هذا المعامل لحذف خاصية من كائن.

:::interactive_editor

```js
const person = {
  name: "Alice",
  age: 30,
  job: "Engineer"
};

delete person.job;

console.log(person.job); // undefined
```

:::

## التحقق مما إذا كان للكائن خاصية

- **طريقة `hasOwnProperty()`**: تُرجع هذه الطريقة قيمة منطقية تشير إلى ما إذا كان للكائن الخاصية المحددة كخاصية خاصة به.

:::interactive_editor

```js
const person = {
  name: "Alice",
  age: 30
};

console.log(person.hasOwnProperty("name")); // true
console.log(person.hasOwnProperty("job")); // false
```

:::

- **طريقة `Object.hasOwn()`**: هذه هي الطريقة الحديثة والمُوصى بها للتحقق مما إذا كان للكائن خاصية خاصة به (ليست موروثة). الصيغة هي `Object.hasOwn(object, propertyName)`. تُرجع `true` إذا كانت الخاصية موجودة في الكائن و`false` إذا لم تكن موجودة، بغض النظر عن قيمة الخاصية. هذا يجعلها أكثر أمانًا من `hasOwnProperty()` وأكثر موثوقية من `if (obj.prop)` عندما تكون القيم `0` أو `false` أو `null` أو `undefined`.

:::interactive_editor

```js
const person = {
  name: "Alice",
  age: 30
};

console.log(Object.hasOwn(person, "name")); // true
console.log(Object.hasOwn(person, "job")); // false
```

:::

إليك مثالًا أكثر تفصيلًا يوضح لماذا `Object.hasOwn()` أكثر موثوقية من التحقق من القيمة مباشرة:

:::interactive_editor

```js
const settings = {
  darkMode: false,
  fontSize: 0,
  language: null
};

// Object.hasOwn() correctly sees these properties exist
console.log(Object.hasOwn(settings, "darkMode")); // true (value is false, but exists)
console.log(Object.hasOwn(settings, "fontSize")); // true (value is 0, but exists)
console.log(Object.hasOwn(settings, "theme"));    // false (property was never added)

// Using if() directly is unsafe for falsy values!
if (settings.darkMode) {
  console.log("Dark mode on"); // Does NOT print — misleading!
}

// Object.hasOwn() is the safe way
if (Object.hasOwn(settings, "darkMode")) {
  console.log("darkMode exists, value is:", settings.darkMode); // darkMode exists, value is: false
}
```

:::

- **معامل `in`**: يُرجع هذا المعامل `true` إذا كانت الخاصية موجودة في الكائن.

:::interactive_editor

```js
const person = {
  name: "Bob",
  age: 25
};

console.log("name" in person);  // true
```

:::

## الوصول إلى الخصائص من الكائنات المسكنة

- **الوصول إلى البيانات**: يتضمن الوصول إلى الخصائص من الكائنات المسكنة استخدام ترميز النقطة أو ترميز الأقواس، تمامًا كما في الكائنات البسيطة. لكنك تحتاج إلى تسلسل هذه الطرق للوصول إلى البنية المسكنة.

:::interactive_editor

```js
const person = {
  name: "Alice",
  age: 30,
  contact: {
    email: "alice@example.com",
    phone: {
      home: "123-456-7890",
      work: "098-765-4321"
    }
  }
};

console.log(person.contact.phone.work); // "098-765-4321"
```

:::

## أنواع البيانات الأولية وغير الأولية

- **أنواع البيانات الأولية**: تشمل هذه الأنواع الأرقام، والسلاسل النصية، والقيم المنطقية، و`null`، و`undefined`، والرموز. تسمى "أولية" لأنها تمثل قيمًا مفردة وليست كائنات. القيم الأولية غير قابلة للتغيير، مما يعني أنه بمجرد إنشائها لا يمكن تغيير قيمتها.
- **أنواع البيانات غير الأولية**: في JavaScript، هذه هي الكائنات، التي تشمل الكائنات العادية، والمصفوفات، والدوال. على عكس الأوليات، يمكن للأنواع غير الأولية أن تحتوي على قيم متعددة كخصائص أو عناصر.

## طرق الكائن

- **التعريف**: طرق الكائن هي دوال مرتبطة بكائن. تُعرف كخصائص للكائن ويمكنها الوصول إلى بيانات الكائن والتعامل معها. تشير الكلمة المفتاحية `this` داخل الطريقة إلى الكائن نفسه، مما يتيح الوصول إلى خصائصه.

:::interactive_editor

```js
const person = {
  name: "Bob",
  age: 30,
  sayHello: function() {
    return "Hello, my name is " + this.name;
  }
};

console.log(person.sayHello()); // "Hello, my name is Bob"
```

:::

## دالة البناء للكائن

- **التعريف**: في JavaScript، دالة البناء هي نوع خاص من الدوال تُستخدم لإنشاء الكائنات وتهيئتها. تُستدعى باستخدام الكلمة المفتاحية `new` ويمكنها تهيئة الخصائص والطرق على الكائن الجديد. دالة البناء `Object()` تنشئ كائنًا فارغًا جديدًا.

```js
new Object()
```

## العمل مع معامل التسلسل الاختياري (`?.`)

- **التعريف**: يتيح لك هذا المعامل الوصول الآمن إلى خصائص الكائن أو استدعاء الطرق دون القلق من وجودها.

:::interactive_editor

```ts
const user = {
  name: "John",
  profile: {
    email: "john@example.com",
    address: {
      street: "123 Main St",
      city: "Somewhere"
    }
  }
};

console.log(user.profile?.address?.street); // "123 Main St"
console.log(user.profile?.phone?.number);   // undefined
```

:::

## التفكيك في الكائن

- **التعريف**: يسمح التفكيك في الكائن باستخراج القيم من الكائنات وتعيينها إلى متغيرات بطريقة أكثر إيجازًا ووضوحًا.

:::interactive_editor

```js
const person = { name: "Alice", age: 30, city: "New York" };

const { name, age } = person;

console.log(name); // Alice
console.log(age);  // 30
```

:::

## العمل مع JSON

- **التعريف**: JSON تعني JavaScript Object Notation. هو تنسيق بيانات خفيف الوزن يعتمد على النص ويُستخدم عادة لتبادل البيانات بين الخادم وتطبيق الويب.

```js
{
  "name": "Alice",
  "age": 30,
  "isStudent": false,
  "list of courses": ["Mathematics", "Physics", "Computer Science"]
}
```

- **`JSON.stringify()`**: تُستخدم هذه الطريقة لتحويل كائن JavaScript إلى سلسلة JSON. هذا مفيد عند الرغبة في تخزين أو نقل البيانات بتنسيق يمكن مشاركته أو نقله بسهولة بين الأنظمة.

:::interactive_editor

```js
const user = {
  name: "John",
  age: 30,
  isAdmin: true
};

const jsonString = JSON.stringify(user);
console.log(jsonString); // '{"name":"John","age":30,"isAdmin":true}'
```

:::

- **`JSON.parse()`**: تُحول هذه الطريقة سلسلة JSON مرة أخرى إلى كائن JavaScript. هذا مفيد عند استرجاع بيانات JSON من خادم ويب أو من localStorage وتحتاج إلى التعامل مع البيانات في تطبيقك.

:::interactive_editor

```js
const jsonString = '{"name":"John","age":30,"isAdmin":true}';
const userObject = JSON.parse(jsonString);

// result: { name: 'John', age: 30, isAdmin: true }
console.log(userObject);
```

:::
