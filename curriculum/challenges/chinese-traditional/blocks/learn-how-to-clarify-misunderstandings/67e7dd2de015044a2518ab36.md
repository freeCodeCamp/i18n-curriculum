# --explanation--

`Think of it as` 用來幫助某人透過將概念與他們已熟悉的事物比較來理解該概念。它鼓勵你以更容易理解的方式想像該概念。 例如：

`Think of a function as a recipe.`－這有助於你將函式（程式）想像成一組指令，類似於食譜提供製作一道菜的步驟。

`Wrapping code` 是指以一種控制存取的方式將程式碼包裹或封裝起來。這將會 `limit` 對程式碼的存取。例如：

`We're wrapping the database access code inside a function to hide the complexity from the rest of the program.`－這表示資料庫的互動被隱藏在外部，而該函式提供一個簡化的介面來操作。
