# --description--

Ukiwa umekuwa ukitengeneza programu kwa JavaScript, bila shaka umewahi kukutana na ujumbe wa hitilafu. Kuelewa ujumbe wa hitilafu za kawaida kutakusaidia kutafuta makosa kwa ufanisi zaidi na kukuza ujuzi wako kama msanidi programu mwenye nguvu zaidi. Aina nne za kawaida za ujumbe wa hitilafu ni `SyntaxError`, `ReferenceError`, `TypeError`, na `RangeError`.

`SyntaxError` hutokea unapokuwa umeandika kitu vibaya katika msimbo wako, kama kukosa mabano ya mzunguko, au mabano ya mraba. Fikiria kama ni kosa la sarufi katika sentensi. Hapa kuna kosa la kawaida ambalo wasanidi programu hufanya wanapotengeneza safu za data:

```js
const arr = ["Beau", "Quincy" "Tom"]
```

Kila kipengele cha safu ya data kinahitaji kutenganishwa na koma vinginevyo kitasababisha ujumbe wa hitilafu.

Hitilafu ya kawaida ya pili ya JavaScript ni `ReferenceError`. Kuna aina kadhaa za `ReferenceError`, zinazochochewa kwa njia tofauti. Aina ya kwanza ya `ReferenceError` ni zile za vigezo ambavyo havijatangazwa.

```js
console.log(price);
```

Katika mfano huu, tunajaribu kuonyesha thamani ya vigezo `price` kwenye konsoli lakini haijatajwa. Hii itasababisha `ReferenceError`.

Mfano mwingine wa `ReferenceError` ni kujaribu kupata thamani ya kigezo kilichotangazwa kwa `let` au `const` kabla hajatajwa:

```js
console.log(b);
const b = 50;
```

Mfano hapo juu utasababisha hitilafu ya `Cannot access 'b' before initialization`.

Hitilafu ya tatu ya kawaida ni `TypeError`. Hitilafu hizi hutokea unapojaribu kufanya operesheni kwa aina isiyo sahihi. Hapa kuna mfano wa kujaribu kutumia njia ya `map` kwenye kitu:

```js
const developerObj = {
  name: "Jessica",
  country: "USA",
  isEmployed: true
};

developerObj.map()
```

Mfano huu utasababisha hitilafu ya `developerObj.map is not a function` kwa sababu njia ya `map` hutumika kwa safu za data na si vitu.

Hitilafu ya mwisho ya kawaida tutakayochunguza ni `RangeError`.

`RangeError` hutokea wakati msimbo wako unajaribu kutumia thamani ambayo iko nje ya kiwango ambacho JavaScript inaweza kushughulikia. Hapa kuna mfano wa kuweka thamani isiyo halali kwa `length` ya safu ya data:

```js
const arr = [];
arr.length = -1; 
```

Kwa kuwa `length` ya safu ya data lazima iwe nambari kamili isiyo na alama hasi, kuipanga kuwa `-1` kunasababisha `RangeError`.

Unapoendelea kutengeneza programu kwa JavaScript, tafadhali fahamu aina hizi tofauti za hitilafu utakazokutana nazo na kwa nini zinatokea.
