# --description--

이번 수업에서는 맵, 해시 맵, 그리고 세트에 대해 살펴보겠습니다. 그 전에 추상 자료형(Abstract Data Types)을 정의해 보겠습니다.

추상 자료형(ADT)은 데이터 유형을 개념적으로 나타내며, 데이터에 수행할 수 있는 연산과 그 데이터의 속성을 포함합니다.

추상 자료형은 **무엇**을 할 수 있는지에 대한 청사진과 같으며, **어떻게** 수행하는지는 포함하지 않습니다. 인터페이스와 실제 연산 구현을 분리합니다.

**맵**은 키-값 쌍의 컬렉션과 그 연산을 매우 구체적이고 효율적으로 관리하는 추상 자료형입니다.

맵에서는 모든 값이 특정 키와 연관되어 있습니다.

맵의 주요 특징 중 하나는 모든 키가 고유해야 한다는 점입니다. 이 고유성 덕분에 직접 조회가 가능해져 정보를 검색하는 과정이 훨씬 효율적입니다.

키만 고유해야 하며, 값은 중복될 수 있습니다.

맵 추상 자료형은 키-값 쌍 삽입, 키에 연관된 값 가져오기, 키에 연관된 값 업데이트, 키-값 쌍 제거, 그리고 키 존재 여부 확인과 같은 중요한 연산도 정의합니다.

이 연산들이 어떻게 수행되어야 하는지는 명시하지 않고, 단지 데이터 유형에서 사용할 수 있는 연산으로 나열할 뿐입니다.

**해시 맵**(hash map) 또는 해시 테이블(hash table)은 맵 추상 자료형의 구체적인 구현입니다.

해시 맵은 "해싱"이라는 기법을 사용해 일반적인 연산을 매우 효율적으로 수행합니다.

해싱은 해시 함수를 사용해 각 요소에 대한 해시 값을 생성하는 방식으로 작동합니다.

해시 값은 키-값 쌍의 키를 기반으로 생성되며, 이 값은 키-값 쌍이 저장되는 실제 자료 구조인 기본 배열에서 인덱스를 계산하는 데 사용됩니다.

그런데 두 키가 같은 인덱스를 갖게 되면 어떻게 될까요?

해시 맵은 이러한 충돌을 영리한 전략으로 해결합니다.

한 가지 방법은 "체이닝" 전략을 사용하는 것으로, 각 배열 인덱스가 연결 리스트(또 다른 자료 구조)를 가리키며, 같은 인덱스를 가진 모든 요소가 그 리스트에 저장됩니다.

또 다른 전략은 "오픈 어드레싱"으로, 미리 정의된 검색 수열에 따라 배열에서 다음 사용 가능한 인덱스를 찾는 방식입니다.

해시 맵의 평균 시간 복잡도는 키-값 쌍 삽입, 검색, 삭제에 대해 "상수 시간" `O(1)`입니다.

이 연산들의 최악 시간 복잡도는 선형 시간 `O(n)`으로, 해시 충돌이 많아 충돌 해결 전략을 여러 번 적용해야 할 때 발생합니다.

해시 맵에 삽입할 때의 공간 복잡도는 평균적으로 상수 `O(1)`이며, 새 쌍을 저장하는 데 일정한 메모리가 필요합니다. 그러나 최악의 경우 기본 배열 크기 조정 작업으로 인해 선형 공간 복잡도 `O(n)`가 될 수 있습니다. 일반적으로 요소를 제거할 때는 상수 공간 복잡도 `O(1)`입니다.

이로 인해 해시 테이블은 대상 키를 찾기 위해 `n`개의 요소를 스캔해야 하는 선형 자료 구조와 비슷해집니다. 하지만 해시 맵이 제대로 구현되면 이런 경우는 드뭅니다.

Python의 **딕셔너리**는 내부적으로 해시 맵으로 구현되어 있습니다.

Python 딕셔너리를 만들려면 중괄호 안에 키-값 쌍을 작성하고 쉼표로 구분하면 됩니다. 각 키와 값은 콜론으로 구분하세요.

```python
my_dictionary = {
  'A': 1,
  'B': 2, 
  'C': 3
}
```

이 코드에서 `'A'`는 키이고 `1`는 값입니다:

```python
'A': 1
```

또는 `dict()`를 사용할 수도 있습니다:

```python
my_dictionary = dict(A=1, B=2, C=3)
```

키에 해당하는 값을 가져올 수 있습니다:

```python
my_dictionary['A']  # 1
```

키에 연관된 값을 업데이트할 수도 있습니다:

```python
my_dictionary['A'] = 4
```

키-값 쌍을 제거할 수도 있습니다:

```python
del my_dictionary['A']  
```

딕셔너리에 키가 있는지(또는 없는지) 확인할 수도 있습니다:

```python
'C' in my_dictionary
```

그리고 각각 딕셔너리의 키, 값, 항목을 가져오는 메서드를 호출할 수 있습니다.

```python
my_dictionary.keys()
my_dictionary.values()
my_dictionary.items()
```

좋습니다. 이제 맵과 해시 맵에 대해 더 알게 되었으니, 세트에 대해 이야기해 보겠습니다.

**세트**는 고유한 요소들의 순서 없는 컬렉션입니다.

이 개념을 주요 구성 요소로 나누어 보겠습니다:

* 세트는 순서가 없습니다. 세트의 요소들은 특정한 순서로 저장되지 않으므로 인덱스로 접근할 수 없습니다.
    
* 세트는 고유한 요소만 포함합니다. 같은 값을 두 번 추가하려 하면, 값은 한 번만 저장됩니다.
    

세트는 수학에서의 집합과 유사하며, 교집합, 합집합, 차집합 같은 집합 연산을 구현합니다.

세트의 주요 장점 중 하나는 요소가 고유함을 보장한다는 점입니다(중복 없음). 그래서 리스트나 다른 자료 구조에서 중복을 제거할 때 자주 사용됩니다.

또한 세트는 동적입니다. 현재 저장된 요소 수에 맞게 크기를 조절할 수 있어 매우 강력합니다.

세트에 요소를 추가, 제거, 길이 구하기, 포함 여부 확인하는 평균 시간 복잡도는 "상수 시간" `O(1)`으로 매우 효율적입니다.

세트는 해시 테이블로 구현되므로, 추가, 제거, 멤버십 확인의 최악 시간 복잡도는 "선형 시간" `O(n)`입니다. 이는 해시 충돌이 여러 번 발생해 해시 테이블이 선형 자료 구조와 비슷해져 `n`번 스캔해야 할 때 발생할 수 있습니다.

공간 복잡도 측면에서 평균적으로 요소 삽입은 상수 복잡도 `O(1)`이며, 새 고유 요소는 일정한 메모리를 필요로 합니다. 그러나 최악의 경우 기본 배열 크기 조정 작업으로 인해 선형 공간 복잡도 `O(n)`가 될 수 있습니다. 일반적으로 요소 제거는 상수 공간 복잡도 `O(1)`입니다.

Python에는 세트를 다룰 때 사용하는 내장 `set` 자료 구조가 있습니다.

내부적으로 Python 세트는 키만 저장하는 해시 테이블로 구현되며, 값은 저장하지 않습니다.

세트는 해시 값이 항상 일정한 불변 데이터 유형의 객체만 저장할 수 있습니다. 반면, 가변 객체는 변경 시 해시 값이 바뀔 수 있어 세트에 포함될 수 없습니다. 세트에 저장된 객체의 해시 값이 바뀌면 프로그램이 해당 객체를 찾을 수 없게 됩니다.

Python에서 세트를 정의하려면 요소를 중괄호로 감싸고 쉼표로 구분하면 됩니다:

```python
numbers = {1, 2, 3, 4}
```

빈 세트를 만들려면 `set()`를 호출하세요:

```python
numbers = set()
```

빈 중괄호를 사용하면 자동으로 Python 딕셔너리가 생성되므로, 빈 세트를 만들려면 반드시 `set()` 함수를 호출해야 합니다.

세트에 요소를 추가하려면 `.add()` 메서드를 사용하세요:

```python
numbers.add(5)
```

세트에서 요소를 제거하려면 `.remove()` 메서드를 사용하세요:

```python
numbers.remove(5)
```

이 메서드는 요소가 없으면 `KeyError`를 발생시킵니다. 하지만 오류를 발생시키고 싶지 않으면 `.discard()` 메서드를 대신 사용할 수 있습니다.

`.pop()` 메서드는 세트에서 임의의 요소를 반환하며, `.clear()` 메서드는 세트의 모든 요소를 제거합니다.

요소가 세트에 있는지 테스트하려면 `in` 연산자를 사용하세요:

```python
5 in numbers
```

Python은 합집합, 차집합, 대칭 차집합, 교집합 같은 집합 연산도 지원하며, 다음 메서드로 수행할 수 있습니다:

```python
set_a = {1, 2, 3, 4}
set_b = {2, 3, 4, 5, 6}

set_a.union(set_b)
set_a.intersection(set_b)
set_a.symmetric_difference(set_b)
set_a.difference(set_b)
```

또는 이에 상응하는 연산자를 사용할 수도 있습니다:

```python
set_a | set_b
set_a & set_b
set_a ^ set_b
set_a - set_b 
```

추가, 제거, 멤버십 테스트의 평균 시간 복잡도는 "상수 시간" `O(1)`입니다.

이 연산들의 최악 시간 복잡도는 해시 맵의 최악 충돌 시나리오 때문에 "선형 시간" `O(n)`입니다.

또한 한 세트가 다른 세트의 부분집합인지 또는 상위집합인지 확인할 수도 있습니다:

```python
set_a.issubset(set_b)
set_a.issuperset(set_b)
```

일반적으로 고유한 항목 컬렉션을 저장하고 항목 존재 여부를 자주 확인해야 할 때 세트를 사용하세요.

맵, 해시 맵, 세트는 효율적인 데이터 조직과 검색을 위해 설계된 강력한 자료 구조입니다. 각각 고유한 특징과 사용 사례가 있으니, 개발자로서 프로젝트에 가장 적합한 것을 선택해야 합니다.
