# --interactive--

تُستخدم محدِّدات CSS المركبة لتعريف العلاقة بين المحدِّدات في CSS. تساعد في اختيار العناصر بناءً على علاقتها بعناصر أخرى، مما يسمح بتنسيق أكثر دقة وكفاءة.

سنناقش بعض محدِّدات CSS المركبة الأساسية وحالات استخدامها، بدءًا بمحدِّد العنصر التابع.

يُستخدم محدِّد العنصر التابع لاستهداف العناصر التي يطابقها المحدِّد الثاني إذا كانت مسكنة داخل عنصر أصل يطابق المحدِّد الأول. يمكن أن يكون الأصل أبًا أو جدًا أو أي عنصر أصل آخر أعلى في شجرة المستند.

لفهم كيفية عمل ذلك بشكل أفضل، دعنا نلقي نظرة على مثال.

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<figure>
  <img src="https://cdn.freecodecamp.org/curriculum/cat-photo-app/relaxing-cat.jpg" alt="Relaxing Cat">
  <figcaption>A relaxing cat with a border</figcaption>
</figure>
```

```css
figure img {
  border: 4px solid red;
}
```

:::

في المثال أعلاه، نستخدم محدِّد العنصر التابع لاختيار جميع عناصر الصورة داخل عناصر `figure` وتطبيق حدود `solid red` على جميع الجوانب الأربعة.

لاحظ أنه يُستخدم `space` بين محدِّد الأصل والابن.

في هذه الحالة، سيكون `figure` هو الأب و`img` هو التابع.

إذا كان لديك عدة صور داخل عنصر `figure`، فإن استخدام محدِّد العنصر التابع سيكون طريقة جيدة لتطبيق حدود سوداء صلبة حول كل صورة من تلك الصور.

نوع آخر من المحدِّدات هو محدِّد الطفل.

محدِّد الطفل (`>`) في CSS يُستخدم لاختيار العناصر التي هي أبناء مباشرين لعنصر أب محدد.

يستهدف هذا المحدِّد فقط العناصر التي لها أب محدد، مما يجعل قواعد CSS أكثر دقة ويمنع التنسيق غير المقصود للعناصر المتداخلة الأعمق.

لنلقِ نظرة على المثال التالي في HTML:

:::interactive_editor

```html
<div class="container">
  <p>First</p>
  <div>
    <p>Second</p>
  </div>
  <div>
    <p>Third</p>
  </div>
</div>
```

:::

في هيكل HTML أعلاه، يُطبّق صنف `container` على عنصر `div`.

داخل هذه الحاوية، يوجد عنصر `p` تابع مباشر ("First"), يليه عنصران إضافيان من نوع `div`، يحتوي كل منهما على عنصر `p` ("Second" و"Third").

العنصر `p` الأول هو تابع مباشر لعنصر `.container`، بينما العنصران الآخران `p` مسكنان داخل عناصر `div` أخرى، مما يجعلهما عناصر تابعة أعمق.

لتطبيق التنسيقات على التابع المباشر فقط لصنف `container`، يمكنك استخدام محدِّد الطفل كما يلي:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<div class="container">
  <p>First</p>
  <div>
    <p>Second</p>
  </div>
  <div>
    <p>Third</p>
  </div>
</div>
```

```css
.container > p {
  color: blue;
}
```

:::

في المثال أعلاه، نستهدف فقط الطفل المباشر لصنف `container`. هذا سيُعطي الطفل المباشر لون نص `blue`.

لأن العنصرين الآخرين من نوع الفقرة مسكنان داخل عناصر `div`، فلا يُعتبران أبناء مباشرين لصنف `container` ولن يحصلوا على لون النص الأزرق.

نوع آخر شائع من المحدِّدات هو محدِّد الأخ التالي.

محدِّد الأخ التالي (`+`) في CSS يختار عنصرًا يتبع مباشرة عنصر أخ محدد. هذا المحدِّد مفيد عندما تريد تطبيق تنسيقات على عنصر يتبع مباشرة عنصرًا آخر، مما يسمح بالتنسيق المستهدف بناءً على موقع العنصر بالنسبة لإخوته.

لنلقِ نظرة على المثال التالي في HTML:

:::interactive_editor

```html
<figure>
  <img
    src="https://cdn.freecodecamp.org/curriculum/cat-photo-app/relaxing-cat.jpg"
    alt="A cute orange cat lying on its back."
  />
  <figcaption>A cute orange cat lying on its back.</figcaption>
</figure>
```

:::

هنا، لدينا عنصر `figure` يحتوي على عنصر `img` يليه عنصر `figcaption`. عنصر `figcaption` هو الأخ المباشر لعنصر `img`.

إذا أردت تطبيق حدود سوداء حول عنصر `figcaption`، يمكنك استخدام محدِّد الأخ التالي كما يلي:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<figure>
  <img
    src="https://cdn.freecodecamp.org/curriculum/cat-photo-app/relaxing-cat.jpg"
    alt="A cute orange cat lying on its back."
  />
  <figcaption>A cute orange cat lying on its back.</figcaption>
</figure>
```

```css
img + figcaption {
  border: 4px solid black;
}
```

:::

في هذا المثال، يختار المجمّع التالي للأخوة (`+`) عنصر `figcaption` الذي يلي مباشرة عنصر `img`. تضيف قاعدة CSS المطبقة `4px solid black border` حول `figcaption`.

نوع آخر شائع من المحدِّدات هو محدِّد الأخ اللاحق.

محدِّد الأخ اللاحق (`~`) في CSS يختار جميع الإخوة لعنصر محدد الذين يأتون بعده.

على عكس محدِّد الأخ التالي الذي يستهدف الأخ الذي يلي مباشرة فقط، يمكن لمحدِّد الأخ اللاحق (`~`) استهداف أي إخوة يتبعون العنصر المحدد، مما يوفر مرونة أكبر في التنسيق.

لنلقِ نظرة على المثال التالي في HTML:

:::interactive_editor

```html
<div class="container">
  <h2>Subheading</h2>
  <p>First paragraph.</p>
  <p>Second paragraph.</p>
  <p>Third paragraph.</p>
  <p>Another paragraph element</p>
</div>
```

:::

في هذا الهيكل HTML، لدينا عنصر `h2` يليه أربعة عناصر فقرة. عناصر الفقرة هي إخوة لعنصر `h2`.

إذا أردت تنسيق جميع عناصر الفقرة التي تأتي بعد عنصر `h2`، يمكنك استخدام محدِّد الأخ اللاحق كما يلي:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<div class="container">
  <h2>Subheading</h2>
  <p>First paragraph.</p>
  <p>Second paragraph.</p>
  <p>Third paragraph.</p>
  <p>Another paragraph element</p>
</div>
```

```css
h2 ~ p {
  color: green;
}
```

:::

في هذا المثال، سيكون لجميع عناصر الفقرة التي تلي عنصر `h2` لون نص أخضر.

يستهدف محدِّد الأخ اللاحق (`~`) جميع إخوة الفقرة الذين يظهرون بعد عنصر `h2`، بغض النظر عما إذا كانوا إخوة مباشرين أم لا.

في الختام، فهم واستخدام محدِّدات CSS المركبة المختلفة يسمح لك بتطبيق تنسيقات دقيقة على عناصر HTML، مما يعزز التحكم ومرونة التصميم.

بإتقان هذه المحدِّدات، يمكنك إنشاء قواعد تنسيق أكثر تطورًا واستهدافًا تحسن مظهر ووظائف صفحات الويب الخاصة بك.
