# --interactive--

في الدرس السابق، تعلمت كيف ترمي الاستثناءات في برامجك. في هذا الدرس، سنلقي نظرة على كيفية التعامل مع هذه الأخطاء بسلاسة داخل كتلة `try...catch...finally`.

تُستخدم كتلة `try` لتغليف الشفرة التي قد ترمي خطأً. تعمل كمساحة آمنة لتجربة شيء قد يفشل.

تلتقط كتلة `catch` الأخطاء التي تحدث في كتلة `try` وتعالجها. يمكنك استخدام كائن `Error` داخل `catch` لفحص ما حدث من خطأ.

تُنفذ كتلة `finally` بعد كتلتي `try` و `catch`، بغض النظر عما إذا حدث خطأ أم لا. تُستخدم عادةً لمهام التنظيف، مثل إغلاق الملفات أو تحرير الموارد.

إليك مثالًا على استخدام كتلة `try...catch`:

:::interactive_editor

```js
function processInput(input) {
  if (typeof input !== "string") {
    throw new TypeError("Input must be a string.");
  }

  return input.toUpperCase();
}

try {
  console.log("Starting to process input...");
  const result = processInput(9);
  console.log("Processed result:", result);
} catch (error) {
  console.error("Error occurred:", error.message);
} 
```

:::

في هذا المثال، لدينا دالة تسمى `processInput` تتحقق أولاً مما إذا كان `input` ليس من نوع `string`. إذا كان الأمر كذلك، نرمي خطأً. وإلا، نُرجع نتيجة استخدام طريقة `toUpperCase` على الـ `input`.

نستدعي الدالة داخل كتلة `try`. بما أن استدعاء الدالة يرمي خطأً، فسيتم التقاطه داخل كتلة `catch` وسيُعرض رسالة الخطأ في وحدة التحكم.

الخطأ الممرر إلى كتلة `catch` هو كائن `Error` يحتوي على معلومات عن ذلك الخطأ. في هذه الحالة، نستخدم الخاصية `message` التي تعرض معلومات مفهومة للمستخدم.

نستخدم `console.error` لأنه مصمم خصيصًا لتسجيل الأخطاء. في العديد من المتصفحات الحديثة، يظهر ناتج `console.error()` باللون الأحمر في وحدة التحكم.

تُنفذ جملة `finally` بغض النظر عما إذا تم رمي استثناء أم لا.

```js
try {
  // Code that might throw an error
} catch (error) {
  // Code to handle the error
} finally {
  // Code that runs regardless of whether an error occurred or not
}
```

حالة استخدام جيدة لجملة `finally` هي عند العمل مع الملفات. في JavaScript، يمكنك فتح ملف، واستخدام كتلة `try` لكتابة بيانات في الملف. إذا حدثت أخطاء، يمكنك استخدام `catch` لالتقاط تلك الأخطاء. ثم تستخدم جملة `finally` لإغلاق الملف.
