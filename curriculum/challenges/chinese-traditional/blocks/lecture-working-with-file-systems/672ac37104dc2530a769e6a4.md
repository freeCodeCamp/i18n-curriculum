# --description--

當你開始建置你自己的網頁應用程式時，你需要注意你為你的檔案命名。

你會想要以易於理解和維護的方式為你的檔案命名。

讓我們來看看一些 HTML 檔案的好與壞檔名範例。請注意，這些範例中的 `.html` 擴充套件告訴我們這是 HTML 檔案。

這裡是一個不良檔案名稱的範例： 

```md
index1234.html
```

僅從名稱無法清楚知道該檔案的內容。這個檔案可能是用於產品頁面、部落格頁面或任何其他型別的頁面，但我們無法從檔案名稱得知。

這是一個更佳 HTML 檔案名稱的範例：

```md
about-us.html
```

透過使用像 `about-us.html` 這樣更具描述性的名稱，可以清楚知道該檔案的內容。

你也會注意到這個檔案名稱中沒有空格。取而代之的是，你使用連字號 `-` 來分隔字、單字。 

有時你可能會看到一個名為 `index.html` 的檔案名稱。這是一個用來表述網站主頁的特殊檔案名稱。當你造訪網站時，`index.html` 檔案是第一個被載入的檔案。

命名檔案時另一個重要的考量是特殊字元的使用。

使用特殊字元、數字和字母的混合可能會讓人難以理解該檔案的內容。

這裡是一個不良檔案名稱的範例：

```md
file-1!@.html
```

這個名稱看起來不必要地複雜，且沒有提供任何關於該檔案內容的資訊。

因此，最好只在你的檔案名稱中使用字母和連字號。

當你開始學習更多語言時，你會發現這些語言中對檔案命名也有共同的慣例。 

在這些情況下，最好始終參考官方文件或其他資源，以了解該語言中命名檔案的最佳做法。

你也會遇到在團隊中工作，並且需要遵循團隊為命名檔案所建立的慣例的情況。 

請務必參考團隊的風格指南，以確保你的檔案名稱與團隊其他成員保持一致。
