# --interactive--

이전 수업에서는 함수에 전달된 인수들의 값을 포함하는 배열과 유사한 `arguments` 객체를 다루는 방법을 배웠습니다.

:::interactive_editor

```js
function logArgs() {
  for (const arg of arguments) {
    console.log(arg);
  }
}

logArgs(1, 2, 3);
// result:
// 1
// 2
// 3
```

:::

이 방법도 함수에서 가변 인수를 접근하고 다루는 유효한 방법이지만, 최신 JavaScript 애플리케이션에서는 대신 rest 매개변수 구문을 사용합니다.

다음은 `arguments` 객체 대신 rest 매개변수 구문을 사용한 업데이트된 예제입니다:

:::interactive_editor

```js
function logArgs(...args) {
  for (const arg of args) {
    console.log(arg);
  }
}

logArgs(1, 2, 3);
// result:
// 1
// 2
// 3
```

:::

이 업데이트된 예제는 더 이상 `arguments` 객체를 직접 참조하지 않습니다. 대신 함수 정의의 마지막 매개변수 앞에 점 세 개가 붙어 있습니다. 이것이 이 rest 매개변수를 `Array` 객체 안에 넣도록 만듭니다. 이 rest 매개변수 이름은 원하는 대로 지을 수 있습니다. 단, 함수 정의에서 항상 마지막 매개변수여야 합니다:

```js
function exampleFunction(a, b, ...restOfArgs) {
  // some code here
}

function anotherFunction(x, y, ...theArgs) {
  // some code here
}
```

rest 매개변수 구문을 사용할 때 몇 가지 제한 사항이 더 있습니다. 그중 하나는 함수 정의에 rest 매개변수를 하나만 가질 수 있다는 점입니다. 따라서 다음 예제는 유효하지 않습니다.

```js
// Won't work

function badFunction(...args, ...moreArgs) {
  // some code here
}
```

다음 제한은 rest 매개변수 뒤에 쉼표가 올 수 없다는 것입니다: `function exampleFunction(a, b, ...restOfArgs, )`.

또 다른 제한은 rest 매개변수에 기본값을 지정할 수 없다는 점입니다. 그렇지 않으면 `SyntaxError`가 발생합니다.

```js
function badFunction(...args = [1,2]){
  // some code here
}
```

그렇다면 `arguments` 객체와 `rest` 매개변수 사이에는 어떤 차이가 있을까요?

가장 큰 차이점은 `arguments` 객체는 진짜 배열이 아니어서 `includes`, `pop`, `push` 같은 메서드를 지원하지 않는다는 점입니다. 하지만 rest 매개변수는 `Array` 인스턴스입니다. 따라서 먼저 진짜 배열로 변환하지 않고도 유효한 내장 배열 메서드를 사용할 수 있습니다.

:::interactive_editor

```js
function hasCat(...args) {
  return args.includes("cat");
}

console.log(hasCat("dog", "chicken", "cat")); // true
console.log(hasCat("dog", "chicken", "horse")); // false
```

:::
