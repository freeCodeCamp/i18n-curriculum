# --interactive--

메서드 체이닝은 여러 메서드를 연속해서 호출하는 기법입니다. JavaScript에서는 문자열, 배열, 객체 등 다양한 값에 메서드 체이닝을 사용할 수 있습니다. 문자열은 원시 값이지만, 문자열 메서드를 사용할 때 JavaScript가 임시로 문자열을 `String` 객체로 감쌉니다.

다양한 문자열 메서드를 사용한 예제를 살펴보겠습니다:

:::interactive_editor

```js
const result = "  Hello, World!  "
  .trim()
  .toLowerCase()
  .replace("world", "JavaScript");

console.log(result); // "hello, JavaScript!"
```

:::

이 예제에서는 문자열을 시작으로 세 가지 작업을 순서대로 수행합니다: 공백을 지우고, 소문자로 변환한 다음, 문자열 `world`을 `JavaScript`로 바꿉니다. 각 메서드는 새로운 문자열을 반환하며, 이 문자열이 다음 메서드 호출의 대상이 됩니다.

메서드 체이닝은 복잡한 작업을 할 때 코드 가독성을 크게 향상시킬 수 있습니다.

예를 들어, `filter`, `map`, `reduce` 메서드를 사용한 다음 예제를 보세요:

:::interactive_editor

```js
const transactions = [
  { amount: 100, type: "credit" },
  { amount: 20, type: "cash" },
  { amount: 150, type: "credit" },
  { amount: 50, type: "cash" },
  { amount: 75, type: "credit" }
];

const totalCreditWithBonus = transactions
  .filter((transaction) => transaction.type === "credit")
  .map((transaction) => transaction.amount * 1.1)
  .reduce((sum, amount) => sum + amount, 0);

console.log(totalCreditWithBonus); // 357.5
```

:::

이 예제에서는 각 객체가 금액과 신용카드 또는 현금 유형을 가진 거래 배열이 있습니다.

먼저 거래를 필터링하여 신용카드 거래만 포함하는 새 배열을 만듭니다. 그런 다음 필터링된 결과에 `map` 메서드를 체이닝하고, 각 거래 금액에 `1.1`를 곱하는데, 이는 `10%` 보너스를 나타냅니다.

그 결과에 `reduce` 메서드를 체이닝하여 각 금액을 합산하면 `357.5`이 됩니다.

메서드 체이닝은 코드를 더 간결하고 읽기 쉽게 만들 수 있지만, 신중하게 사용하는 것이 중요합니다.

너무 긴 체인은 디버깅하기 어려울 수 있는데, 체인 중 어느 단계에서 문제가 발생했는지 바로 알기 어렵기 때문입니다. 명확성과 디버깅 편의를 위해 매우 긴 체인은 여러 단계로 나누는 것이 좋은 습관입니다.

객체에서도 메서드 체이닝을 할 수 있습니다. 이 경우 각 메서드는 현재 객체를 가리키는 `this`를 반환하여 체인에서 다음 메서드가 이 객체에서 실행되도록 합니다.

:::interactive_editor

```js
const calculator = {
  total: 0,
  add(n) {
    this.total += n;
    return this;
  },
  multiply(n) {
    this.total *= n;
    return this;
  },
  subtract(n) {
    this.total -= n;
    return this;
  },
  getResult() {
    return this.total;
  }
};

const result = calculator.add(5).multiply(2).subtract(3).getResult();
console.log(result); // 7
```

:::

이렇게 하면 각 메서드가 같은 객체를 반환하므로 연속해서 메서드 호출을 계속할 수 있습니다.
