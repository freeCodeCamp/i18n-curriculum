# --description--

Se prendiamo 47, invertiamo e sommiamo, 47 + 74 = 121, che è palindromo.

Non tutti i numeri producono palindromi così rapidamente. Per esempio,

<div style="margin-left: 4em;">
  349 + 943 = 1292,<br>
  1292 + 2921 = 4213<br>
  4213 + 3124 = 7337<br>
</div>

Cioè, 349 ha impiegato tre iterazioni per arrivare a un palindromo.

Anche se nessuno l'ha ancora dimostrato, si pensa che alcuni numeri, come 196, non producano mai un palindromo. Un numero che non forma mai un palindromo attraverso il processo di invertire e sommare si chiama numero Lychrel. A causa della natura teorica di questi numeri, e per lo scopo di questo problema, assumiamo che un numero sia Lychrel finché non si dimostra il contrario. Inoltre ti viene detto che per ogni numero sotto diecimila, esso o (i) diventerà un palindromo in meno di cinquanta iterazioni, oppure (ii) nessuno, con tutta la potenza di calcolo esistente, è riuscito finora a mappare quel numero a un palindromo. Infatti, 10677 è il primo numero per cui si è dimostrato che servono più di cinquanta iterazioni prima di produrre un palindromo: 4668731596684224866951378664 (53 iterazioni, 28 cifre).

Sorprendentemente, ci sono numeri palindromi che sono essi stessi numeri Lychrel; il primo esempio è 4994.

Quanti numeri Lychrel ci sono sotto `num`?

**Nota:** La formulazione è stata leggermente modificata il 24 aprile 2007 per sottolineare la natura teorica dei numeri Lychrel.

```js
const countLychrelNumbers = (size) => {
  const numReverse = (num) => {
    return Number(num.toString().split('').reverse().join(''));
  };
  const isPalin = (num) => {
    if (numReverse(num) === num) {
      return true;
    }
    return false;
  };
  let total = 0;
  for (let i = 1; i < size; i++) {
    let loopCount = 1;
    let sum = i;
    while (loopCount < 50) {
      sum = sum + numReverse(sum);
      if (isPalin(sum)) {
        break;
      } else {
        loopCount++;
      }
    }
    if (loopCount === 50) {
      total++;
    }
  }
  return total;
}
```
