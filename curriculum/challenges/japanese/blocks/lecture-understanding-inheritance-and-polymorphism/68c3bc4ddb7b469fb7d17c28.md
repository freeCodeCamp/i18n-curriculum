# --description--

ポリモーフィズムは、オブジェクト指向プログラミング（OOP）で次に学ぶ重要な概念です。

ポリモーフィズムを使うと、同じ種類の多くのオブジェクトとやり取りできるインターフェースにアクセスできます。

ポリモーフィズムを詳しく見ていき、それがどのようにコードの再利用を可能にするかを理解しましょう。

ポリモーフィズムは、異なるクラスのメソッドが同じ名前を共有しながら、異なる処理を行うことを可能にします。同じメソッド名を異なるオブジェクトで呼び出すと、それぞれが独自の方法で応答します。

ポリモーフィズムの基本的な例を示します：

```py
class A:
   def action(self): ...

class B:
   def action(self): ...

class C:
   def action(self): ...

Class().method()  # Works for A, B, or C
```

異なる動物の鳴き声を使ってポリモーフィズムを表現した例はこちらです：

```py
class Cat:
   def speak(self):
       return "A cat meow"

class Bird:
   def speak(self):
       return "A bird tweet"
  
class Monkey:
   def speak(self):
       return "A monkey ooh ooh aah aah ooh ooh aah aah"

def animal_sound(animal):
   print(animal.speak())

animal_sound(Cat())
animal_sound(Bird())
animal_sound(Monkey())
```

この例では、`animal_sound()` は `speak()` メソッドを持つ任意のオブジェクトを受け取る関数です。

`Cat`、`Bird`、または `Monkey` を渡すと、そのオブジェクトの `speak()` メソッドを呼び出して結果を出力します。各クラスが `speak()` を異なる方法で定義しているため、同じ関数から異なる出力が得られます。これがポリモーフィズムの動作例です。

次は、インスタンスと属性を使った別の例です：

```py
class Twitter:
   def __init__(self, content):
       self.content = content

   def post(self):
       return f"🐦 Tweet: '{self.content}' (280 chars max)"

class Instagram:
   def __init__(self, content):
       self.content = content

   def post(self):
       return f"📸 Instagram Post: '{self.content}' + ✨ filters"

class LinkedIn:
   def __init__(self, content):
       self.content = content

   def post(self):
       return f"💼 LinkedIn Article: '{self.content}' (Professional Mode)"

def start(social_media):
   print(social_media.post())  # Calls .post() on any object

# Instances
tweet = Twitter('Just learned Python polymorphism!')
photo = Instagram('Sunset vibes 🌅')
article = LinkedIn('Why OOP matters in 2024')

# The polymorphic calls - same function, different outputs
start(tweet) # 🐦 Tweet: 'Just learned Python polymorphism!' (280 chars max)
start(photo) # 📸 Instagram Post: 'Sunset vibes 🌅' + ✨ filters
start(article) # 💼 LinkedIn Article: 'Why OOP matters in 2024' (Professional Mode)
```

また、**継承ベースのポリモーフィズム**と呼ばれる種類のポリモーフィズムもあります。

継承ベースのポリモーフィズムでは、親クラスがメソッドを定義し、複数の子クラスがそのメソッドを独自の方法でオーバーライドします。どの子オブジェクトでも同じメソッドを呼び出せますが、子クラスによって動作が異なります。

例を示します：

```py
class Animal:
   def speak(self):
       return 'Some generic sound'

class Cat(Animal):
   def speak(self):
       return 'A cat meow'

class Dog(Animal):
   def speak(self):
       return 'A dog barks woof woof'

class Monkey(Animal):
   def speak(self):
       return 'A monkey ooh ooh aah aah ooh ooh aah aah'
  
print(Cat().speak()) # A cat meow
print(Dog().speak()) # A dog barks woof woof
print(Monkey().speak()) # A monkey ooh ooh aah aah ooh ooh aah aah
print(Animal().speak()) # Some generic sound
```

親の `Animal` クラスの各子クラスが `speak()` メソッドをオーバーライドして独自の実装を提供しているのがわかります。したがって、各サブクラスのインスタンスで `speak()` メソッドを呼び出すと、その動物に特有の鳴き声が返されます。

さらに進めて、リスト内で呼び出しを行い、リストをループして各オブジェクトの `speak()` メソッドの戻り値を表示することもできます：

```py
animals = [Cat(), Dog(), Monkey()]

for animal in animals:
   print(animal.speak())

# Output:
# A cat meow
# A dog barks woof woof
# A monkey ooh ooh aah aah ooh ooh aah aah
```
