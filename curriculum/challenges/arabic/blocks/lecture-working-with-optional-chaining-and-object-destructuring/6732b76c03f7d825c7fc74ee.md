# --interactive--

عامل التسلسل الاختياري (`?.`) أداة مفيدة في JavaScript تتيح لك الوصول بأمان إلى خصائص الكائنات أو استدعاء الطرق دون القلق من وجودها. إنه مثل شبكة أمان للعمل مع الكائنات التي قد تحتوي على أجزاء مفقودة.

:::interactive_editor

```js
const person = {
  name: "Alice",
  age: 30
};

console.log(person.name); // "Alice"
console.log(person.job); // undefined
```

:::

في هذا المثال، `person.name` موجود، لذا يسجل `Alice`. لكن `person.job` غير موجود، لذلك يعطينا `undefined`.

الآن، لنفترض أننا نريد الوصول إلى خاصية في كائن قد لا تكون موجودة:

```js
const person = {
  name: "Alice",
  age: 30
};

console.log(person.address.street); // This will throw an error!
```

هذا المثال سيرمي `Uncaught TypeError`. بما أن `person.address` هو `undefined`، لا يمكننا الوصول إلى الخاصية `street`. هنا يأتي دور عامل التسلسل الاختياري. إليك مثالًا على استخدام عامل التسلسل الاختياري:

:::interactive_editor

```ts
const user = {
  name: "John",
  profile: {
    email: "john@example.com",
    address: {
      street: "123 Main St",
      city: "Somewhere"
    }
  }
};

console.log(user?.profile?.address?.street); // "123 Main St"
console.log(user?.profile?.phone?.number);   // undefined
```

:::

باستخدام عامل التسلسل الاختياري، نخبر JavaScript بالاستمرار في العملية فقط إذا كان الكائن (أو القيمة قبل `?.`) موجودًا وليس `null` أو `undefined`.

إذا كانت القيمة قبل `?.` هي `null` أو `undefined`، تعيد JavaScript `undefined` بدلاً من محاولة متابعة العملية ورمي خطأ.

تذكر، عامل التسلسل الاختياري مفيد جدًا عندما لا تكون متأكدًا من وجود خاصية أو طريقة. يساعد في منع الأخطاء ويجعل شفرتك أكثر متانة.
