# --description--

データ構造やアルゴリズムを扱い始めると、よく行う必要がある操作の一つが各ノードを訪問することだと気づきます。

このプロセスは「データ構造の走査」と呼ばれます。

走査は、データ構造内のすべてのノードに対して何かを行うために使われます。例えば、値を出力したり、特定の値を見つけたり、ノードに対して特定の操作を実行したりします。

各ノードを体系的に訪問することで、どのノードも見落とされないようにします。

しかし、どの順番でデータ構造を走査すればよいのでしょうか？どこから始めて、次のノードはどう選べばよいでしょうか？

走査の明確な方法がなければ、特定の道筋を持たずに迷路を歩くようなものです。

そこで、幅優先探索（BFS）や深さ優先探索（DFS）のようなアルゴリズムが非常に重要になります。これらはグラフの走査や、2つのノード間の経路探索によく使われます。

データ構造を走査する際には、これらのアルゴリズムがノードを訪問する順序を定義し、どのノードも見落とされないようにします。

まずは幅優先探索（BFS）から始めましょう。

## 幅優先探索（BFS）

**幅優先探索（BFS）** は、グラフの隣接するすべてのノードを訪問してから次のレベルに進むアルゴリズムです。

重みのないグラフで2つのノード間の最短経路を見つけるのに使えます。なぜなら、各レベルのノードをすべて調べるため、最も少ないエッジ数の経路を最初に見つけるからです。

このアルゴリズムは、訪問したノードを管理するためにキューというデータ構造を使って実装されることが多いです。キューはFIFO（先入れ先出し）方式で、最初に追加されたノードが最初に取り出されます。

アルゴリズムの流れは次の通りです：

* 特定のノードから始めます。
    
* そのノードを訪問済みとしてマークし、キューに追加します。
    
* キューが空でない間、現在のノードをキューから取り出します（デキュー）。その後、そのノードの隣接ノードのうち未訪問のものを訪問済みとしてマークし、キューに追加します。
    

重要な点は、幅優先探索（BFS）はキューをメモリに保持する必要があり、このキューに多くのノードが入る場合があるため、空間的な要求が大きくなることがあることです。特に同じレベルに多くのノードがあるグラフでは顕著です。

ここで、木と呼ばれる特定の種類のグラフに幅優先探索（BFS）を適用した例を見てみましょう。

木については今後のレッスンで詳しく学びますが、基本的にはサイクルのないグラフで、ノードが階層的に整理されています。サイクルとは、同じノードから始まり同じノードに戻る経路のことです。

この木に幅優先探索（BFS）を適用してみましょう：

<img src="https://cdn.freecodecamp.org/curriculum/lecture-transcripts/how-do-depth-first-and-breadth-first-search-work-1.png" alt="A tree diagram showing nodes A through G arranged in a hierarchy. Node A is at the root, with children B and C. Node B has children D and E, and node C has children F and G.">

**ステップ 1:**

木のルートであるノードAから始めます。Aをキューに追加し、すぐに訪問済みとしてマークします。

* **キュー:** `[A]`
    
* **訪問済み:** `{A}`
    

**ステップ 2:**

ノードAをキューから取り出します。未訪問の子ノード（ノードB、次にノードC）をキューに追加し、訪問済みとしてマークします。

* **キュー:** `[B, C]`
    
* **訪問済み:** `{A, B, C}`
    

同じレベルのノードがキューに追加される順序は、データ構造の実装やグラフ表現でエッジ（接続）が保存されている順序によって決まります。

実装が一貫していれば、同じレベルのノードの具体的な走査順序はアルゴリズムの正しさに影響しません。すべてのノードをレベルごとに訪問します。

**ステップ 3:**

ノードBをキューから取り出します。未訪問の子ノード（ノードD、次にノードE）をキューに追加し、訪問済みとしてマークします。

* **キュー:** `[C, D, E]`
    
* **訪問済み:** `{A, B, C, D, E}`
    

**ステップ 4:**

ノードCをキューから取り出します。未訪問の子ノード（ノードF、次にノードG）をキューに追加し、訪問済みとしてマークします。

* **キュー:** `[D, E, F, G]`
    
* **訪問済み:** `{A, B, C, D, E, F, G}`
    

**ステップ 5:**

ノードDをキューから取り出します。このノードには未訪問の子ノードがないため、訪問済みセットは変わりません。

* **キュー:** `[E, F, G]`
    
* **訪問済み:** `{A, B, C, D, E, F, G}`
    

**ステップ 6:**

ノードEをキューから取り出します。このノードには未訪問の子ノードがないため、訪問済みセットは変わりません。

* **キュー:** `[F, G]`
    
* **訪問済み:** `{A, B, C, D, E, F, G}`
    

**ステップ 7:**

ノードFをキューから取り出します。このノードには未訪問の子ノードがないため、訪問済みセットは変わりません。

* **キュー:** `[G]`
    
* **訪問済み:** `{A, B, C, D, E, F, G}`
    

**ステップ 8:**

ノードGをキューから取り出します。このノードには未訪問の子ノードがないため、訪問済みセットは変わりません。

* **キュー:** `[]`
    
* **訪問済み:** `{A, B, C, D, E, F, G}`
    

キューが空になると、走査は完了です。

ノードは次の順序で走査されました：

**A → B → C → D → E → F → G**

<img src="https://cdn.freecodecamp.org/curriculum/lecture-transcripts/how-do-depth-first-and-breadth-first-search-work-2.png" alt="The same tree diagram with arrows showing the BFS traversal order: A to B to C to D to E to F to G, demonstrating level-by-level visitation.">

アルゴリズムがレベルごとにノードを訪問していることに注目してください。

ノードAから始めて、次のレベルでノードBとCを訪問し、その次のレベルでノードD、E、F、Gを訪問します。これが幅優先探索（BFS）の基本原理です。

## 深さ優先探索（DFS）

幅優先探索（BFS）が同じレベルの隣接ノードをすべて訪問するのに対し、**深さ優先探索（DFS）** は各枝をできるだけ深くたどってから戻ります。

このアルゴリズムは、迷路を特定の道筋に沿って進み、行き止まりや出口に達するまで進みます。行き止まりに達したら戻って別の道を選びます。

深さ優先探索（DFS）は、解が一つだけのパズルの解決、グラフのサイクル検出、連結成分の発見に使われます。

このアルゴリズムは、再帰やスタックというデータ構造を使って実装されることが多いです。

スタックはLIFO（後入れ先出し）方式で、最後に追加されたノードが最初に取り出されます。

アルゴリズムの流れは次の通りです：

* 特定のノードから始めます。
    
* そのノードを訪問済みとしてマークし、スタックに追加します。
    
* スタックが空でない間、現在のノードをポップ（取り出し）します。これが「訪問」や処理（例えば値の出力）にあたります。その後、そのノードの未訪問の隣接ノードをすべて訪問済みとしてマークし、スタックに追加します。
    

このアルゴリズムの制限の一つは、重みのないグラフで2つのノード間の最短経路を必ずしも見つけられるとは限らないことです。

深さ優先探索（DFS）を先ほどの木の例に適用した例を見てみましょう。

<img src="https://cdn.freecodecamp.org/curriculum/lecture-transcripts/how-do-depth-first-and-breadth-first-search-work-3.png" alt="The same tree diagram as before, showing nodes A through G in their hierarchical structure, ready for DFS demonstration.">

**ステップ 1:**

ルートノードAから始めます。訪問済みとしてマークし、スタックに追加します。

* **スタック:** `[A]`
    
* **訪問済み:** `{A}`
    

**ステップ 2:**

ノードAをスタックからポップします。

次に、未訪問の子ノードであるノードBとノードCを逆順（`C`、次に`B`）でスタックに追加します。こうすることで`B`がスタックのトップ（LIFO）になり、次に処理されます。これらも訪問済みとしてマークします。

* **スタック:** `[C, B]`
    
* **訪問済み:** `{A, B, C}`
    

**ステップ 3:**

ノードBをスタックからポップします。

次に、未訪問の子ノードであるノードDとノードEを逆順（`E`、次に`D`）でスタックに追加します。これらも訪問済みとしてマークします。

* **スタック:** `[C, E, D]`
    
* **訪問済み:** `{A, B, C, D, E}`
    

**ステップ 4:**

ノードDをスタックからポップします。このノードにはスタックに追加する子ノードがありません。

* **スタック:** `[C, E]`
    
* **訪問済み:** `{A, B, C, D, E}`
    

**ステップ 5:**

ノードEをスタックからポップします。このノードにはスタックに追加する子ノードがありません。

* **スタック:** `[C]`
    
* **訪問済み:** `{A, B, C, D, E}`
    

**ステップ 6:**

ノードCをポップします。

次に、子ノードであるノードFとノードGを逆順（ノードG、次にノードF）でスタックに追加し、訪問済みとしてマークします。

* **スタック:** `[G, F]`
    
* **訪問済み:** `{A, B, C, D, E, F, G}`
    

**ステップ 7:**

ノードFをスタックからポップします。このノードにはスタックに追加する子ノードがありません。

* **スタック:** `[G]`
    
* **訪問済み:** `{A, B, C, D, E, F, G}`
    

**ステップ 8:**

ノードGをポップします。このノードにはスタックに追加する子ノードがありません。

* **スタック:** `[]`
    
* **訪問済み:** `{A, B, C, D, E, F, G}`
    

スタックが空になると、走査は完了し、すべてのノードが訪問されました。

ノードは次の順序で訪問されました：

**A → B → D → E → C → F → G**

<img src="https://cdn.freecodecamp.org/curriculum/lecture-transcripts/how-do-depth-first-and-breadth-first-search-work-4.png" alt="The tree diagram with numbers showing the DFS traversal order: A(1), B(2), D(3), E(4), C(5), F(6), G(7), demonstrating depth-first exploration of branches.">

ノードAから始めて、木の枝を下にたどり、ノードB、ノードD、ノードEまで進んでから、また上に戻ってノードC、ノードF、ノードGに進む様子がわかります。これが深さ優先探索（DFS）の基本原理で、枝を深くたどってから戻り、別の枝を探します。

この例ではスタックを使って解きましたが、深さ優先探索（DFS）は再帰を使って実装することもできます。再帰では関数が現在のノードを処理し、未訪問の隣接ノードごとに自分自身を呼び出します。関数呼び出しのスタックがLIFOの順序を暗黙的に管理します。

幅優先探索（BFS）と深さ優先探索（DFS）は、グラフや木の走査に欠かせないアルゴリズムです。幅優先探索（BFS）はレベルごとにノードを探索し、重みのないグラフで最短経路を見つけるのに最適です。一方、深さ優先探索（DFS）は枝をできるだけ深くたどってから戻るため、迷路の解決やサイクル検出に適しています。それぞれの長所と短所を理解することで、問題に応じて適切なアルゴリズムを選べるようになります。
