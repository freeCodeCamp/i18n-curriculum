# --description--

قبل أن تدفع المستودع الخاص بك إلى GitHub، يجب أن تكون مُوثَّقًا. تستخدم مفاتيح SSH و GPG زوجًا من المفتاح العام والخاص. هذا يعني أنك تحتفظ بمفتاح خاص على جهازك المحلي، تستخدمه للمصادقة على نفسك. تشارك مفتاحًا عامًا مع الأشخاص أو الخدمات التي تحتاج إلى التحقق من مصادقتك. بعبارة أخرى، المفتاح الخاص مطلوب لأداء إجراء، والمفتاح العام يُستخدم للتحقق من هذا الإجراء.

مفاتيح GPG، أو GNU Privacy Guard، تُستخدم عادةً لتوقيع الملفات أو الاستدعاءات. يمكن لأي شخص بعد ذلك استخدام مفتاح GPG العام الخاص بك للتحقق من أن توقيع الملف صادر عن مفتاحك وأن محتويات الملف لم تُعدّل أو تُعبث بها. مفاتيح SSH، أو Secure SHell، تُستخدم عادةً للمصادقة على اتصال بعيد بخادم عبر أداة `ssh`. ومع ذلك، كما ستتعلم في هذا الدرس، يمكنك أيضًا استخدام مفتاح SSH لتوقيع الاستدعاءات.

لتوليد مفتاح GPG، ستحتاج إلى تشغيل:

```sh
gpg --full-generate-key
```

لإنشاء مفتاح SSH، ستشغل:

```sh
ssh-keygen -t ed25519 -C "your_email@example.com"
```

تأكد من تقديم بريدك الإلكتروني الحقيقي.

يجب أن تطلب كلا الأمرين عبارة مرور لتأمين المفتاح. هذه العبارة اختيارية، لكنها موصى بها بشدة.

بالنسبة لمفاتيح SSH، ستحصل على ملفين - واحد لمفتاحك الخاص، وآخر لمفتاحك العام. أسماء ملفات المفتاح العام تنتهي بـ `.pub`، حتى لا تشارك مفتاحك الخاص عن طريق الخطأ. ستكون هذه الملفات في `~/.ssh`.

أما مفاتيح GPG فستكون في مجلد مخفي ومشفر. بدلاً من ذلك، لعرض مفاتيحك العامة، ستشغل:

```sh
gpg --list-secret-keys --keyid-format=long
```

بعد ذلك، للحصول على المفتاح العام، استخدم:

```sh
gpg --armor --export "<key id>"
```

لكن ماذا يمكنك أن تفعل بهذه المفاتيح الجديدة؟ حسنًا، تحدثنا عن توقيع الالتزامات، وهذا شيء يمكنك استخدام أي من هذه المفاتيح من أجله.

لتوقيع الالتزامات باستخدام مفتاح GPG، ستحتاج إلى رفع المفتاح العام، وليس الخاص، إلى حساب GitHub الخاص بك. ثم، خذ المعرف القصير الذي حصلت عليه من عرض المفاتيح سابقًا، وشغل هذا الأمر لتعيينه كمفتاح توقيع git الخاص بك:

```sh
git config --global user.signingkey <your_gpg_key_id>
```

بعد ذلك، يمكنك تمرير الخيار `-S` إلى أمر `git commit` لتوقيع التزام معين - ستحتاج إلى إدخال عبارة المرور الخاصة بك. بدلاً من ذلك، إذا أردت توقيع كل الالتزامات تلقائيًا، يمكنك تعيين إعداد autosign إلى `true`:

```sh
git config --global commit.gpgsign true
```

لتوقيع باستخدام مفتاح SSH، وهي ميزة جديدة نسبيًا على GitHub، ستحتاج أولًا إلى رفع المفتاح إلى حساب GitHub الخاص بك. ثم ستحتاج إلى تعيين وضع التوقيع لـ git لاستخدام SSH:

```sh
git config --global gpg.format ssh
```

ثم، لتعيين مفتاح التوقيع، ستمرر مسار الملف بدلًا من المعرف:

```sh
git config --global user.signingkey <path_to_your_ssh_keys>
```

أخيرًا، يمكنك تفعيل التوقيع التلقائي كما تفعل مع مفتاح GPG.

مع إعداد أي من هذين الأسلوبين، ستكون جاهزًا الآن لالتزامات مُوثَّقة ستحصل على شارة خاصة على GitHub.
