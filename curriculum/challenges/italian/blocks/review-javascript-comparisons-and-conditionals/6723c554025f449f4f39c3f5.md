## Confronti e i tipi di dati `null` e `undefined`

- **Confronti e `undefined`**: Una variabile è `undefined` quando è stata dichiarata ma non le è stato assegnato un valore. È il valore predefinito delle variabili non inizializzate e dei parametri di funzione a cui non è stato passato un argomento. `undefined` si converte in `NaN` nei contesti numerici, il che fa sì che tutti i confronti numerici con `undefined` restituiscano `false`.

:::interactive_editor

```js
console.log(undefined < 0); // false (NaN < 0 is false)
console.log(undefined >= 0); // false (NaN >= 0 is false)
```

:::

- **Confronti e `null`**: Il tipo `null` rappresenta l'assenza intenzionale di un valore. `null` si converte in `0` nei contesti numerici, il che può causare comportamenti inattesi nei confronti numerici:

:::interactive_editor

```js
console.log(null < 0); // false (0 < 0 is false)
console.log(null >= 0); // true (0 >= 0 is true)
```

:::

- Quando si usa l'operatore di uguaglianza (`==`), `null` e `undefined` sono uguali solo tra loro e a se stessi:

:::interactive_editor

```js
console.log(null == undefined); // true
console.log(null == 0); // false
console.log(undefined == NaN); // false
```

:::

- Tuttavia, usando l'operatore di uguaglianza stretta (`===`), che verifica sia il valore sia il tipo senza eseguire conversioni implicite, `null` e `undefined` non sono uguali:

:::interactive_editor

```js
console.log(null === undefined); // false
```

:::

## Istruzioni `switch`

- **Definizione**: Un'istruzione `switch` valuta un'espressione e confronta il suo valore con una serie di clausole `case`. Quando viene trovata una corrispondenza, viene eseguito il blocco di codice associato a quel caso. Un'istruzione `break` dovrebbe essere posta alla fine di ogni caso, per terminare la sua esecuzione e passare al successivo. Il caso `default` è un caso opzionale e viene eseguito solo se nessuno degli altri casi corrisponde. Il caso `default` è posto alla fine di un'istruzione `switch`.

:::interactive_editor

```js
const dayOfWeek = 3; 

switch (dayOfWeek) {
  case 1:
    console.log("It's Monday! Time to start the week strong.");
    break;
  case 2:
    console.log("It's Tuesday! Keep the momentum going.");
    break;
  case 3:
    console.log("It's Wednesday! We're halfway there.");
    break;
  case 4:
    console.log("It's Thursday! Almost the weekend.");
    break;
  case 5:
    console.log("It's Friday! The weekend is near.");
    break;
  case 6:
    console.log("It's Saturday! Enjoy your weekend.");
    break;
  case 7:
    console.log("It's Sunday! Rest and recharge.");
    break;
  default:
    console.log("Invalid day! Please enter a number between 1 and 7.");
}
```

:::
