# --description--

```css
/* rule 1 */
.class.second-class {
  font-size: 12px;
}

/* rule 2 */
.class .second-class {
  font-size: 24px;
}
```

여기서 규칙 1과 규칙 2는 동일한 특이성을 가집니다. 규칙 1은 체이닝 선택자(공백 없음)를 사용하고, 규칙 2는 자손 결합자(빈 공간)를 사용합니다. 하지만 두 규칙 모두 두 개의 클래스를 가지고 있으며, 결합자 기호 자체는 특이성에 영향을 주지 않습니다.

```css
/* rule 1 */
.class.second-class {
  font-size: 12px;
}
/* rule 2 */
.class > .second-class {
  font-size: 24px;
}
```

이 예제도 같은 내용을 보여줍니다. 규칙 2가 자식 결합자(`>`)를 사용하더라도 특이성 값은 변하지 않습니다. 두 규칙 모두 두 개의 클래스를 가지고 있어 동일한 특이성 값을 가집니다.

*참고:* 모든 것이 특이성에 영향을 주는 것은 아닙니다

선택자를 비교할 때, 범용 선택자(`*`)와 결합자(`+`, `~`, `>`, 그리고 빈 공간)와 같은 특수 기호를 볼 수 있습니다. 이 기호들은 그 자체로는 특이성을 추가하지 않습니다.
