# --description--

Дещо заплутано, але будь-який елемент може бути одночасно і гнучким контейнером, і гнучким елементом. Іншими словами, ви також можете застосувати `display: flex` до гнучкого елемента, а потім використати гнучкий контейнер, щоб розташувати його дочірні елементи.

<img src="https://cdn.freecodecamp.org/curriculum/odin-project/flex-box/flexbox-03.png" style="width: 95%; height: 95%;" alt="a flex container that has multiple flex items, within those flex items are nested flex items as well. Making the parent of those nested flex items also a flex container." />

Створення та вкладення кількох гнучких контейнерів і елементів — це основний спосіб, яким ви будете будувати складні макети. Наступне зображення було отримане, використовуючи лише гнучкий контейнер для розташування, розмірів і розміщення різних елементів. Гнучкий контейнер — це дуже потужний інструмент.

<img src="https://cdn.freecodecamp.org/curriculum/odin-project/flex-box/flexbox-02.png" alt="a complex layout of flex items and flex containers. There are multiple flex containers nested into each other, thus making them flex items as well"/>

Безперечно, на зображенні представлено макет CSS Flexbox із вкладеними гнучкими контейнерами. Зовнішній контейнер позначений як "ТАКОЖ гнучкий контейнер", виділений синім, а всередині нього є інший контейнер, позначений як "з гнучкими елементами" червоним кольором. Усередині червоного контейнера розташовані три елементи з міткою "гнучкі елементи" персикового кольору. Це демонструє, що гнучкий контейнер може бути вкладений у інший гнучкий контейнер і містити власні гнучкі елементи, що ілюструє рекурсивну природу структур макетів Flexbox.
