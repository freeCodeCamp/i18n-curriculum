# --description--

Якщо ви ще не зрозуміли, проблема в попередньому завданні була у виклику `splice` у функції `tabClose()`. На жаль, `splice` змінює оригінальний масив, на якому його викликають, тому другий виклик використовував змінений масив і дав несподівані результати.

Це невеликий приклад набагато більшого шаблону — ви викликаєте функцію для змінної, масиву або об’єкта, і функція змінює цю змінну або щось в об’єкті.

Один із основних принципів функціонального програмування — не змінювати речі. Зміни призводять до помилок. Легше запобігти помилкам, знаючи, що ваші функції нічого не змінюють, включно з аргументами функції або будь-якою глобальною змінною.

У попередньому прикладі не було складних операцій, але метод `splice` змінив оригінальний масив і спричинив помилку.

Нагадаємо, що у функціональному програмуванні зміна або модифікація чогось називається <dfn>мутацією</dfn>, а результат — <dfn>побічним ефектом</dfn>. Ідеальною функція має бути <dfn>чистою функцією</dfn>, тобто такою, що не спричиняє жодних побічних ефектів.

Давайте навчимося дотримуватися цього принципу і не змінювати жодну змінну чи об’єкт у нашому коді.

# --instructions--

Заповніть код функції `incrementer`, щоб вона повертала значення глобальної змінної `fixedValue`, збільшене на одиницю.

# --hints--

Ваша функція `incrementer` не повинна змінювати значення `fixedValue` (яка є `4`).

```js
incrementer();
assert(fixedValue === 4);
```

Ваша функція `incrementer` має повертати значення, яке на одиницю більше за значення `fixedValue`.

```js
const __newValue = incrementer();
assert(__newValue === 5);
```

Ваша функція `incrementer` має повертати значення, засноване на значенні глобальної змінної `fixedValue`.

```js
(function () {
  fixedValue = 10;
  const newValue = incrementer();
  assert(fixedValue === 10 && newValue === 11);
  fixedValue = 4;
})();
```
