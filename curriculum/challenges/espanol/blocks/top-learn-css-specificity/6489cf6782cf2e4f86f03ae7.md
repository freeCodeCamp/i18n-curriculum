# --description--

```css
/* rule 1 */
.class.second-class {
  font-size: 12px;
}

/* rule 2 */
.class .second-class {
  font-size: 24px;
}
```

Aquí tanto la regla 1 como la regla 2 tienen la misma especificidad. La regla 1 usa un selector encadenado (sin espacio) y la regla 2 usa un combinador descendiente (el espacio vacío). Pero ambas reglas tienen dos clases y el símbolo del combinador en sí no añade a la especificidad.

```css
/* rule 1 */
.class.second-class {
  font-size: 12px;
}
/* rule 2 */
.class > .second-class {
  font-size: 24px;
}
```

Este ejemplo muestra lo mismo. Aunque la regla 2 está usando un combinador hijo (`>`), esto no cambia el valor de especificidad. Ambas reglas todavía tienen dos clases, por lo que tienen los mismos valores de especificidad.

*Nota:* No todo contribuye a la especificidad

Al comparar selectores, puedes encontrar símbolos especiales para el selector universal (`*`) así como combinadores (`+`, `~`, `>`, y un espacio vacío). Estos símbolos no añaden ninguna especificidad por sí mismos.
