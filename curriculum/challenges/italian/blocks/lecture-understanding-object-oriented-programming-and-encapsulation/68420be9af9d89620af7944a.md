# --description--

La programmazione orientata agli oggetti, nota anche come OOP, è uno stile di programmazione in cui gli sviluppatori trattano tutto nel loro codice come un oggetto del mondo reale.

Una classe è come un modello per creare oggetti. Ogni singolo oggetto creato da una classe ha attributi che definiscono i dati e metodi che definiscono i comportamenti degli oggetti.

In una lezione precedente, hai imparato come creare classi. Ecco un promemoria della sintassi:

```py
class ClassName:
   def __init__(self, parameters):
       attribute = value

   def method_name(self):
       # method logic
```

Ecco un esempio di una classe che usa il metodo speciale `__init__` per inizializzare gli attributi brand e color ogni volta che un oggetto viene creato usando la classe:

```py
class Car:
   def __init__(self, brand, color):
       self.brand = brand
       self.color = color

# create two objects from the Car class
car1 = Car('Toyota', 'red')
car2 = Car('Lambo', 'green')

print('Car 1 Brand:', car1.brand) # Car 1 Brand: Toyota
print('Car 1 Color:', car1.color) # Car 1 Color: red

print('Car 2 Brand:', car2.brand) # Car 2 Brand: Lambo
print('Car 2 Color:', car2.color) # Car 2 Color: green
```

La programmazione orientata agli oggetti ha quattro principi chiave che ti aiutano a organizzare e gestire il codice in modo efficace. Sono incapsulamento, ereditarietà, polimorfismo e astrazione.

Il resto di questa lezione si concentrerà su come funziona l'incapsulamento.

L'incapsulamento è il raggruppamento degli attributi e dei metodi di un oggetto in un'unica unità, la classe.

Con l'incapsulamento, puoi nascondere lo stato interno dell'oggetto dietro un semplice insieme di metodi e attributi pubblici che funzionano come porte. Dietro quelle porte ci sono attributi e metodi privati che controllano come i dati cambiano e chi può vederli.

Supponiamo che tu voglia tenere traccia del saldo di un portafoglio. Vuoi permettere alle persone di depositare o prelevare denaro dal portafoglio, ma nessuno dovrebbe poter manomettere direttamente il saldo.

In quel caso, puoi rendere `deposit()` e `withdraw()` metodi pubblici, e nascondere il saldo sotto l'attributo `_balance`:

```py
class Wallet:
   def __init__(self, balance):
       self._balance = balance # For internal use by convention

   def deposit(self, amount):
       if amount > 0:
           self._balance += amount # Add to the balance safely

   def withdraw(self, amount):
       if 0 < amount <= self._balance:
           self._balance -= amount # Remove from the balance safely
```

Per convenzione, anteporre un singolo underscore agli attributi e ai metodi significa che sono destinati all'uso interno. Nessuno dovrebbe consultarli direttamente dall'esterno della classe perché ciò va contro i principi dell'incapsulamento, il che può portare a bug.

Mentre il prefisso con un singolo underscore è solo una convenzione, anteporre un doppio underscore agli attributi e ai metodi impedisce effettivamente che vengano consultati dall'esterno della loro classe, rendendo quegli attributi e metodi privati.

```py
class Wallet:
   def __init__(self, balance):
       self.__balance = balance # Private attribute

   def deposit(self, amount):
       if amount > 0:
           self.__balance += amount # Add to the balance safely

   def withdraw(self, amount):
       if 0 < amount <= self.__balance:
           self.__balance -= amount # Remove from the balance safely

account = Wallet(500)
print(account.__balance) # AttributeError: 'Wallet' object has no attribute '__balance'
```

Per ottenere il valore corrente di `__balance`, puoi definire un metodo get_balance. Per esempio:

```py
class Wallet:
   def __init__(self, balance):
       self.__balance = balance

   def deposit(self, amount):
       if amount > 0:
           self.__balance += amount

   def withdraw(self, amount):
       if 0 < amount <= self.__balance:
           self.__balance -= amount
  
   def get_balance(self):
       return self.__balance


acct_one = Wallet(100)
acct_one.deposit(50)
print(acct_one.get_balance()) # 150

acct_two = Wallet(450)
acct_two.withdraw(28)
print(acct_two.get_balance()) # 422

acct_two.deposit(150)
print(acct_two.get_balance()) # 572
```

Puoi anche definire un metodo privato `__validate` per verificare se ogni importo di deposito o prelievo è un numero positivo:

```py
class Wallet:
   def __init__(self):
       self.__balance = 0

   def __validate(self, amount):
       if amount < 0:
           raise ValueError('Amount must be positive')

   def deposit(self, amount):
       self.__validate(amount)
       self.__balance += amount

   def withdraw(self, amount):
       self.__validate(amount)
       if amount > self.__balance:
           raise ValueError('Insufficient funds')
       self.__balance -= amount

   def get_balance(self):
       return self.__balance

acct_one = Wallet()
acct_one.deposit(3)
print(acct_one.get_balance()) # 3

acct_one.deposit(50)
print(acct_one.get_balance()) # 53

acct_one.deposit(-4)  # ValueError: Amount must be positive
acct_one.withdraw(-8) # ValueError: Amount must be positive
acct_one.withdraw(58) # ValueError: Insufficient funds
```

Come puoi vedere, il metodo `__validate` è privato e viene eseguito dietro le quinte nei metodi pubblici `deposit()` e `withdraw()` per assicurarsi che l'importo sia sempre valido.

In una prossima lezione, imparerai di più su come funzionano gli attributi con prefisso doppio underscore.

In sintesi, l'incapsulamento blocca i dati interni dietro metodi pubblici chiari. È così che mantieni le tue classi al sicuro da manomissioni e centralizzi la validazione in un unico posto. Puoi aggiornare o estendere il tuo codice liberamente, sapendo che il codice esterno tocca solo le interfacce che esponi.
