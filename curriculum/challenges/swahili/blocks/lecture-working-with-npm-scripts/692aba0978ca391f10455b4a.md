# --description--

Tujifunze kuhusu hifadhidata ya npm, ambapo unaweza kushiriki vifurushi vyako vya JavaScript na ulimwengu mzima.

**Hifadhidata ya npm** ni hifadhidata ya vifurushi vya umma na vya binafsi.

Wasanidi programu na mashirika wanaweza kuchapisha vifurushi kwenye hifadhidata hii ili kushiriki zana zao na jamii zao na wanachama wa timu, na katika matukio mengi, na wasanidi programu wengine wote wa Node.js.

Vifurushi vinaweza kuwa vya umma au vya binafsi. Vifurushi vya umma vinaweza kupatikana na mtu yeyote. Vifurushi vya binafsi vinaweza kupatikana tu na watumizi maalum au wanachama wa shirika.

Katika hifadhidata ya npm, unaweza kushiriki kile ulichojenga na ulimwengu kwa kubofya mara chache tu. Inasikika vizuri, sivyo?

Tuweze kuona jinsi unavyoweza kuchapisha kifurushi hatua kwa hatua.

## Hatua ya 1: Unda Akaunti

Hatua ya kwanza ni kuunda akaunti ya bure ya npm. Ili kuunda akaunti yako:

* Nenda [https://www.npmjs.com/signup](https://www.npmjs.com/signup).
    
* Jaza fomu. Utahitaji kuingiza jina la mtumiaji, barua pepe, na nenosiri.
    

Tafadhali fahamu kwamba, kulingana na ukurasa wa usajili, "anwani yako ya barua pepe itaongezwa kwenye metadata ya vifurushi unavyovichapisha, hivyo inaweza kuonekana hadharani."

## Hatua ya 2: **Ingia**

Mara baada ya kuwa na akaunti ya npm, utahitaji kuingia kwenye terminal yako kwa kuendesha amri hii:

```bash
npm login
```

Kisha utahitaji kuthibitisha kwa taarifa za akaunti yako.

## **Hatua ya 3: Unda faili la** `package.json`

Kabla ya kuchapisha kifurushi chako, utahitaji kuunda faili la `package.json` kwa amri hii:

```bash
npm init
```

(Ikiwa mradi wako tayari una faili hili, unaweza kwenda moja kwa moja kwenye **hatua ya 4**).

Utahitaji kuingiza taarifa hizi kwa faili lako la `package.json`:

* **name**: Hii kawaida ni jina la mradi/direktori yako. Ikiwa unapanga kufanya kifurushi chako kuwa cha umma, jina hili lazima liwe la kipekee, isipokuwa ukitumia wigo (tutazungumzia hili hivi karibuni).
    
* **version**: Unapaswa kuanza na `"1.0.0"`, thamani ya chaguo-msingi. Hii inawakilisha toleo la awali la kifurushi chako na unaweza kulisasisha unavyofanya mabadiliko kwenye kifurushi.
    
* **main**: Sehemu ya kuingilia ya mradi wako. Hii ni `index.js` kwa chaguo-msingi.
    
* **author**: Unaweza kuandika jina lako au jina unalotaka kutumia.
    
* **license**: Chagua leseni maalum. Mifano ni MIT na ISC.
    
* **description**: Muhtasari mfupi wa kifurushi chako.
    

Unapochagua jina la kifurushi chako, hakikisha halijatumiwa tayari katika hifadhidata ya npm. Ili kufanya hivyo, unaweza kwenda kwenye URL ifuatayo, ukibadilisha `<your-package-name>` na jina la kifurushi chako:

```bash
https://www.npmjs.com/package/<your-package-name>
```

Kama hakuna kifurushi kwenye URL hiyo, jina la kifurushi linapatikana na unaweza kulitumia.

Hata hivyo, kama jina limechukuliwa tayari na unataka kulihifadhi, unaweza kutumia kile kinachojulikana kama **wigo** kuchapisha kifurushi chako.

### **Wigo**

Unapounda akaunti ya mtumiaji au shirika katika hifadhidata ya npm, unapata wigo lenye jina la mtumiaji wako au jina la shirika lako.

Wigo husaidia kuepuka migongano ya majina kwa sababu unaweza kuchapisha kifurushi chako chini ya wigo wako, hata kama kuna kifurushi kingine chenye jina lile lile tayari.

Jina la kifurushi chenye wigo huanza na jina la wigo. Kwa mfano, katika `package.json`, utapata jozi hii ya ufunguo-thamani:

```bash
"name": "@<scope-name>/<your-package-name>"
```

Unaweza kufanya mabadiliko haya moja kwa moja kwenye faili ikiwa unahitaji, au unaweza kuunda faili la `package.json` na wigo tangu mwanzo kwa kuongeza `--scope=@<scope-name>` kwenye amri ya `npm init`.

```bash
npm init --scope=@<scope-name>
```

Jina la wigo linaweza kuwa jina lako la mtumiaji ikiwa unatumia akaunti binafsi, au jina la shirika lako ikiwa unatumia akaunti ya shirika.

## Hatua ya 4: Unda README.md

README ya kifurushi chako ni faili la markdown ambapo unaelezea kile kifurushi chako kinachofanya na matumizi yake yaliyokusudiwa.

Mifano ya maudhui muhimu ambayo unaweza kutaka kujumuisha ni:

* Kusudi lake.
    
* Jinsi ya kukisakinisha.
    
* Vitegemezi muhimu na mambo ya kuzingatia.
    
* Jinsi ya kuanza.
    
* Vipande vya mfano vya msimbo.
    
* Miongozo ya michango (ikiwa unakubali michango).
    
* Unganisho kwa faili ya mabadiliko ambapo unafuatilia mabadiliko yaliyofanywa katika matoleo tofauti.
    

Muundo wa README ni rahisi kubadilika. Unaweza kuubadilisha ili uendane na mahitaji yako.

## **Hatua ya 5:** `.npmignore`

Faili la `.npmignore` linaambia hifadhidata ya npm ni faili gani ziingizwe na ni faili gani zisizojumuishwe wakati wa kuchapisha kifurushi chako. Ni muhimu sana kuzuia kushiriki taarifa nyeti kama vile funguo binafsi, nywila, na taarifa za kibinafsi.

Katika faili hili, unaweza kuandika mifumo ya kuonyesha ni faili gani, folda gani, na aina gani za faili zisizojumuishwe. Inafuata sheria sawa za kulinganisha mifumo kama faili za `.gitignore`.

Kama umefanya kazi na Git, unaweza kufikiria faili hili linafanana sana na `.gitignore`, sivyo? Hii ni kwa sababu ni hivyo. Kulingana na nyaraka za npm, "Kama hakuna faili la `.npmignore`, lakini kuna faili la `.gitignore`, basi npm itapuuzia vitu vinavyolingana na faili la `.gitignore`."

Pia unapaswa kujua kwamba npm inatafuta faili za `.npmignore` na `.gitignore` katika folda ndogo zote za kifurushi chako, si tu kwenye folda kuu.

Kwa chaguo-msingi, njia hizi hupuuziwa moja kwa moja, hivyo huna haja ya kuziingiza kwenye `.npmignore`:

* `.*.swp`
    
* `._*`
    
* `.DS_Store`
    
* `.git`
    
* `.gitignore`
    
* `.hg`
    
* `.npmignore`
    
* `.npmrc`
    
* `.lock-wscript`
    
* `.svn`
    
* `.wafpickle-*`
    
* `config.gypi`
    
* `CVS`
    
* `npm-debug.log`
    

Folda ya `node_modules` pia hupuuziwa, isipokuwa dependencies zilizojumuishwa.

## **Hatua ya 6: Chapisha kifurushi chako**

Kabla ya kuchapisha kifurushi chako kwenye hifadhidata ya npm, unapaswa kukijaribu kwa kuendesha amri hii:

```bash
npm install <full-path-to-package-directory>
```

Mara utakapokuwa tayari kuchapisha, amri itategemea aina ya kifurushi: chenye wigo dhidi ya kisichokuwa na wigo.

Kama kifurushi chako hakina wigo, endesha amri hii:

```bash
npm publish
```

Kama kifurushi chako kina wigo, kitachapishwa kama kifurushi binafsi kwa chaguo-msingi (ikiwa kipengele hiki kinapatikana kwenye akaunti yako). Ikiwa unataka kufanya kifurushi kuwa cha umma, utahitaji kuongeza `--access public` kwenye amri.

```bash
npm publish --access public
```

Ili kuona kifurushi chako cha umma, nenda kwenye URL hii:

```bash
https://npmjs.com/package/<your-package-name>
```

Unapaswa kuona dalili ya uonekano wa kifurushi chako (cha umma au binafsi) chini ya jina la kifurushi.

## **Hatua ya 7 (Hiari): Sakinisha kifurushi chako**

Sasa umejiandaa vyote, unaweza kusakinisha kifurushi chako katika mradi wa Node.js kwa amri hii:

```bash
npm install <your-package-name>
```

## Kusasisha kifurushi chako

Kama unahitaji kufanya mabadiliko kwenye kifurushi chako baada ya kuchapisha, unaweza kusasisha toleo la kifurushi katika `package.json`.

Huu ni mkataba wa kawaida wa utoaji wa toleo la kimaelezo:

* **Toleo la kwanza:** `1.0.0`.
    
* **Toleo la marekebisho:** Ongeza nambari ya tatu. Kwa mfano, kusasisha kutoka `1.0.0` hadi `1.0.1`.
    
* **Toleo dogo:** Ongeza nambari ya katikati na urejee nambari ya mwisho kuwa sifuri. Kwa mfano, kusasisha `1.0.0` hadi `1.1.0`.
    
* **Toleo kubwa:** Ongeza nambari ya kwanza na urejee nambari ya katikati na ya mwisho kuwa sifuri. Kwa mfano, kusasisha `1.1.0` hadi `2.0.0`.
    
Kazi nzuri. Sasa unajua jinsi ya kuchapisha kifurushi chako kwenye hifadhidata ya npm na jinsi ya kukisasisha kurekebisha hitilafu na kuongeza vipengele vipya.
