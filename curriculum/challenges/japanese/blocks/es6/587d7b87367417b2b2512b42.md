# --description--

`const`に不慣れな場合は、<a href="/learn/javascript-algorithms-and-data-structures/basic-javascript/declare-a-read-only-variable-with-the-const-keyword" target="_blank" rel="noopener noreferrer nofollow">この<code>const</code>キーワードに関するチャレンジ</a>を確認してください。

`const`宣言は、現代のJavaScriptで多くの用途があります。

一部の開発者は、値を再代入する必要があるとわかっている場合を除き、デフォルトではすべての変数を`const`で代入することを好みます。その場合にのみ`let`を使用します。

しかし、`const`を使って変数に代入されたオブジェクト（配列や関数を含む）は依然として変更可能であることを理解することが重要です。`const`宣言は、変数識別子の再代入のみを防ぎます。

```js
const s = [5, 6, 7];
s = [1, 2, 3];
s[2] = 45;
console.log(s);
```

`s = [1, 2, 3]`はエラーになります。その行をコメントアウトすると、`console.log`は値`[5, 6, 45]`を表示します。

ご覧の通り、オブジェクト`[5, 6, 7]`自体を変更でき、変数`s`は変更された配列`[5, 6, 45]`を指し続けます。すべての配列と同様に、`s`の配列要素は変更可能ですが、`const`が使われているため、代入演算子を使って変数識別子`s`を別の配列に指すことはできません。

# --instructions--

配列が`const s = [5, 7, 2]`として宣言されています。さまざまな要素の代入を使って配列を`[2, 5, 7]`に変更してください。

# --hints--

`const`キーワードを置き換えないでください。

```js
assert(__helpers.removeJSComments(code).match(/const/g));
```

`s`は定数変数（`const`を使って）であるべきです。

```js
assert(__helpers.removeJSComments(code).match(/const\s+s/g));
```

元の配列宣言を変更しないでください。

```js
assert(__helpers.removeJSComments(code).match(
/const\s+s\s*=\s*\[\s*5\s*,\s*7\s*,\s*2\s*\]\s*;?/g

```

`s`は`[2, 5, 7]`と等しくなるべきです。

```js
assert.deepEqual(s, [2, 5, 7]);
```
