# --description--

Servidores são essenciais para a web moderna. Eles alimentam as experiências online das quais dependemos todos os dias como usuários e alimentarão a maioria das experiências que você criará como desenvolvedor.

Vamos aprender mais sobre eles.

Como você pode saber, a internet é um sistema global de redes de computadores interconectadas. Os servidores são os componentes centrais que possibilitam essa comunicação ao gerenciar o processo de troca de dados.

Um **server** é um computador ou programa de computador que recebe solicitações de outros computadores e envia dados ou serviços para eles pela rede como resposta.

Servidor é um conceito duplo porque pode se referir a:

* O hardware físico real e poderoso que é projetado para funcionar de forma eficiente e contínua para fornecer esses serviços. Essas máquinas são construídas e projetadas para confiabilidade.
    
* Ou para o **software** real do computador que está rodando naquele hardware físico para fornecer os serviços.
    
Os computadores físicos que chamamos de “servidores” geralmente são armazenados em data centers, que são instalações grandes e seguras com ambientes controlados.

Servidores são uma parte essencial de **O Modelo Cliente-Servidor**.

O Modelo Cliente-Servidor é uma arquitetura de sistema distribuído onde os componentes são clientes ou servidores.

* O **cliente** é o processo ou aplicação que envia solicitações para servidores em busca de dados ou serviços. Por exemplo, um navegador web.
    
* O **servidor** é o dispositivo físico ou programa de computador que lida com essas requisições e retorna uma resposta.

Essencialmente, o cliente e o servidor fazem parte de um ciclo de requisição-resposta, onde o cliente envia uma requisição para o servidor e o servidor retorna uma resposta para o cliente.

Por exemplo, quando você usa seu navegador (o cliente) para autenticar em uma aplicação com seu nome de usuário e senha, suas informações são enviadas com segurança para um servidor. O servidor então verifica se o nome de usuário e a senha que você digitou correspondem às informações que a plataforma possui atualmente em seu banco de dados. Se houver correspondência, o servidor retorna uma resposta bem-sucedida para o cliente (seu navegador) e você é autenticado com sucesso no site.

Se nenhuma correspondência for encontrada, o servidor retorna uma resposta de falha para o cliente e você normalmente verá um erro ou algum tipo de feedback explicando por que sua tentativa de login não foi bem-sucedida.

Toda essa lógica de autenticação é executada no servidor. O servidor é responsável por validar as credenciais, interagir com o banco de dados e decidir como responder a cada solicitação.

Cada resposta do servidor inclui um código de status HTTP, que informa ao cliente o que aconteceu. Esses códigos de status permitem que o cliente saiba se uma solicitação foi bem-sucedida ou falhou e por quê.

Por exemplo:

* 200 significa que tudo estava OK.
    
* 404 significa que o recurso não foi encontrado no servidor.
    
* 500 significa que houve um erro interno do servidor.
    
* 503 significa que o servidor está temporariamente indisponível.
    
* E assim por diante…
    

Cada código de status tem seu próprio significado único.

Exemplos de aplicações do mundo real que usam servidores incluem comércio eletrônico, motores de busca, jogos, streaming de vídeo, armazenamento em nuvem e plataformas de colaboração, plataformas de mídia social, aplicações financeiras e mais.

Se os dados são atualizados dinamicamente ou a aplicação precisa de algum tipo de comunicação com um banco de dados, servidores são muito provavelmente componentes-chave do processo de comunicação.

Estes são alguns dos principais tipos de servidores:

* **Servidores web** entregam conteúdo web, como páginas HTML, imagens e arquivos CSS, para navegadores.
    
* **Servidores de aplicação** lidam com a lógica de negócios, processam a entrada do usuário e coordenam as respostas.
    
* **Servidores de banco de dados** executam sistemas de gerenciamento de banco de dados e armazenam, recuperam e gerenciam dados.
    
* **Servidores de email** enviam, recebem e armazenam mensagens de email.
    
* **Servidores de arquivos** gerenciam o acesso a arquivos e permitem que eles sejam armazenados e compartilhados em uma rede.
    
* **Servidores proxy** atuam como intermediários entre clientes e outros servidores, frequentemente usados para cache, segurança e controle de tráfego.

Os servidores são a espinha dorsal das poderosas plataformas online que usamos todos os dias. Eles possibilitam executar lógica complexa, armazenar e processar grandes quantidades de dados e suportar milhões de usuários ao mesmo tempo. Sem servidores, o mundo digital moderno no qual confiamos não existiria.
