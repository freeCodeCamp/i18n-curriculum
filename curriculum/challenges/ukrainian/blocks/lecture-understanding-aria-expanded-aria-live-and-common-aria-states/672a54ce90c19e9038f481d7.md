# --interactive--

Атрибут `aria-expanded` використовується для забезпечення доступності, щоб вказати, чи елемент керування розгорнутий чи згорнутий. Його застосовують разом із згортальними віджетами, такими як меню, акордеони та інші віджети розкриття, які керують видимістю вмісту. Атрибут `aria-expanded` встановлюється в `true`, якщо елемент керування розгорнутий, або в `false`, якщо він згорнутий.

Інформація, яку надає `aria-expanded`, дозволяє користувачам, які використовують читачі екрана, розуміти поточний стан елемента керування (чи він розгорнутий, чи згорнутий).

Атрибут `aria-expanded` застосовується до інтерактивного елемента, який перемикає видимість згортального віджета. Наприклад, якщо кнопка перемикає розгорнуте меню, атрибут `aria-expanded` розміщується на цій кнопці.

Коли меню розгорнуте, атрибут `aria-expanded` має бути встановлений у `true`, як у цьому прикладі:

```html
<button aria-expanded="true">Menu</button>
```

Коли меню згорнуте, він має бути встановлений у `false`.

```html
<button aria-expanded="false">Menu</button>
```

Завжди потрібно мати атрибут `aria-expanded`, встановлений або в `true`, або в `false` на елементі керування. Наприклад, якщо кнопка перемикає видимість меню, значення за замовчуванням для `aria-expanded` має відповідати початковій видимості меню.

Якщо меню за замовчуванням розгорнуте, `aria-expanded` спочатку має бути встановлений у `true`. Якщо меню за замовчуванням згорнуте, `aria-expanded` спочатку має бути встановлений у `false`.

Значення `aria-expanded` має динамічно оновлюватися за допомогою JavaScript під час взаємодії користувача з елементом.

Крім того, властивості `aria-controls` і `aria-owns` можна використовувати разом із `aria-expanded` для встановлення програмного зв’язку між елементом керування та елементом, яким він керує.

Почнемо з `aria-controls`. Коли його використовують із `aria-expanded`, атрибут `aria-controls` вказує елемент, яким керують. Наприклад, кнопка може розгортати або згортати список, що виконує роль меню. Значення `aria-controls` буде ідентифікатором `id` керованого елемента (у цьому прикладі — списку меню).

const menu = document.getElementById(button.getAttribute('aria-controls'));

  button.setAttribute('aria-expanded', String(!expanded));
  menu.hidden = expanded;

Зверніть увагу, що список йде безпосередньо після кнопки керування. Для розгорнутих елементів керування найкраще, щоб розгорнутий вміст йшов одразу після елемента, який ним керує, у DOM. Це запобігає необхідності користувачам читачів екрана шукати розгорнутий вміст і полегшує навігацію клавіатурою через інтерактивні елементи в розгорнутому вмісті.

Якщо неможливо розмістити розгорнутий вміст одразу після елемента керування, атрибут `aria-owns` дозволяє віртуально перемістити його після елемента керування в дереві доступності. Це дозволяє допоміжним технологіям, таким як читачі екрана, вважати, що розгорнутий вміст розташований безпосередньо після елемента керування в DOM.

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<button aria-owns="menu1" aria-expanded="true">Menu</button>
<main>
  <!-- an entire page worth of content --->
</main>
<ul id="menu1">
  <li>...</li>
  <li>...</li>
</ul>

<script src="index.js"></script>
```

```css

  color: white;
  padding: 8px 12px;
  border: none;
  border-radius: 4px;
  cursor: pointer;
  margin-bottom: 10px;
}

  background-color: #005ea2;
}

ul {
  list-style: none;
  padding: 0;
  margin: 8px 0;
  border: 1px solid #ccc;
  border-radius: 4px;
  width: 150px;

  z-index: 10;
}

li {
  padding: 8px;

}

}

```

```js
const button = document.querySelector('button');
const menu = document.getElementById(button.getAttribute('aria-owns'));

button.addEventListener('click', () => {
  const expanded = button.getAttribute('aria-expanded') === 'true';
  button.setAttribute('aria-expanded', String(!expanded));
  menu.hidden = expanded;
});

// Close the menu if user clicks outside
document.addEventListener('click', (e) => {
  if (!button.contains(e.target) && !menu.contains(e.target)) {
    button.setAttribute('aria-expanded', 'false');
    menu.hidden = true;
  }
});

```

Використання атрибута `aria-owns` має недоліки. Він створює зайву багатослівність для користувачів читачів екрана, оскільки більшість читачів автоматично озвучують увесь вміст розгорнутого елемента при першому розгортанні. Також він не змінює порядок переходу по табуляції, що може змусити користувачів клавіатури проходити через усі інші інтерактивні елементи на сторінці, перш ніж дістатися до розгорнутого вмісту, якщо не керувати порядком табуляції за допомогою JavaScript.

Ідеально, щоб розгорнутий вміст розміщувався після елемента керування, а атрибут `aria-owns` слід використовувати лише у найгіршому випадку, коли це неможливо. Якщо його потрібно застосувати, необхідно ретельно тестувати з широким спектром читачів екрана та браузерів, щоб переконатися, що ваша реалізація доступна для всіх.

І нагадаємо, що коли ви використовуєте `aria-controls` або `aria-owns`, значення `aria-expanded` має продовжувати оновлюватися під час розгортання та згортання елемента керування.

Атрибут `aria-expanded` вказує, чи елемент керування розгорнутий чи згорнутий. Ця інформація є важливою для користувачів читачів екрана, допомагаючи їм розуміти поточний стан згортальних елементів, таких як меню, акордеони та інші подібні віджети розкриття.

Правильне використання `aria-expanded` допоможе створити інтуїтивний користувацький досвід для всіх.
