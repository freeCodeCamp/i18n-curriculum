# --interactive--

再帰は、ベースケースに到達するまで関数を繰り返し呼び出すことができる複雑な機能です。従来のループとは異なり、再帰は深くネストされたオブジェクトや配列、またはファイルツリーのような未知の深さを持つものを扱うことができます。しかし、与えられた数字からカウントダウンするなど、より基本的なタスクにも使えます。

まさにそれを行う関数を作ってみましょう。関数名は`recursiveCountdown`とし、数字を受け取る必要があります。この数字をコンソールに出力するようにします。

:::interactive_editor

```js
const recursiveCountdown = (number) => {
  console.log(number);
};

recursiveCountdown(5);
```

:::

これを呼び出して数字5を渡すと、その数字がターミナルに出力されます。しかし、それだけで何も起こりませんし、数字5はカウントダウンではありません。

再帰部分を作り始める前に、まずベースケースを設定する必要があります。ベースケースが設定されていないと、コードはメモリ割り当てを超えてクラッシュするまで実行され続けます。

:::interactive_editor

```js
const recursiveCountdown = (number) => {
  if (number < 1) {
    return;
  }
  console.log(number);
};

recursiveCountdown(5);
```

:::

ベースケースとして、数字が1未満になったらカウントダウンを停止したいです。ベースケースに到達したら、returnで関数の実行を終了できます。

ベースケースを安全に準備できたので、再帰を設定しましょう。関数が再帰的になるポイントは、自分自身を実行中に呼び出すことです。この場合、数字を出力した後に関数を呼び出したいです。ただし、カウントダウンするためには、新しい数字は1つ小さくする必要があります。

:::interactive_editor

```js
const recursiveCountdown = (number) => {
  if (number < 1) {
    return;
  }
  console.log(number);
  recursiveCountdown(number - 1);
};

recursiveCountdown(5); 
```

:::

これでコンソールに5、4、3、2、1の数字が出力されます。

5つの数字が出ましたね！では、代わりにカウントアップしたい場合はどうでしょう？まったく新しい関数を書く代わりに、ログの順序と再帰呼び出しの順序を入れ替えればいいのです。

:::interactive_editor

```js
const recursiveCountdown = (number) => {
    if (number < 1) {
        return;
    }
    recursiveCountdown(number - 1);
    console.log(number);
  };

recursiveCountdown(5);
```

:::

これでコンソールに1、2、3、4、5の数字が出力されます。

なぜこれが動くのでしょう？これを理解するには、コールスタックを理解する必要があります。コールスタックはJavaScriptが関数呼び出しを追跡し解決する仕組みです。スタックは後入れ先出しのキューのように機能します。これをよりよく理解するために、関数にログを追加してみましょう。

:::interactive_editor

```js
const recursiveCountdown = (number) => {
  console.log(`Function execution started for number: ${number}`);
  if (number < 1) {
    console.log(`Base case reached, begin resolving stack`);
    return;
  }
  console.log(`Calling recursiveCountdown with number: ${number - 1}`);
  recursiveCountdown(number - 1);
  console.log(`Function execution completed for number: ${number}`);
};

recursiveCountdown(5);
```

:::

ここでは4つの重要なログ文を追加しました。最初のログは関数呼び出しが実行を開始したときに動きます。3番目のログは再帰関数が呼び出される直前に動きます。4番目のログは関数の実行が終了したときに動きます。結果は以下の通りです。

```md
Function execution started for number: 5
Calling recursiveCountdown with number: 4
Function execution started for number: 4
Calling recursiveCountdown with number: 3
Function execution started for number: 3
Calling recursiveCountdown with number: 2
Function execution started for number: 2
Calling recursiveCountdown with number: 1
Function execution started for number: 1
Calling recursiveCountdown with number: 0
Function execution started for number: 0
Base case reached, begin resolving stack
Function execution completed for number: 1
Function execution completed for number: 2
Function execution completed for number: 3
Function execution completed for number: 4
Function execution completed for number: 5
```

これはどうして起こるのでしょう？ここでコールスタックの出番です。`recursiveCountdown(5)`を呼び出すと、その関数呼び出しがコールスタックに追加されます。

その関数呼び出しが`recursiveCountdown(4)`を呼び出す必要があるところに達すると、結果を待つために停止します。その間に`recursiveCountdown(4)`が`recursiveCountdown(5)`の上にコールスタックに追加されます。

その関数呼び出しが`recursiveCountdown(3)`を呼び出す必要があるところに達すると、結果を待つために停止します。その間に`recursiveCountdown(3)`が`recursiveCountdown(4)`の上にコールスタックに追加されます。

その関数呼び出しが`recursiveCountdown(2)`を呼び出す必要があるところに達すると、結果を待つために停止します。その間に`recursiveCountdown(2)`が`recursiveCountdown(3)`の上にコールスタックに追加されます。

その関数呼び出しが`recursiveCountdown(1)`を呼び出す必要があるところに達すると、結果を待つために停止します。その間に`recursiveCountdown(1)`が`recursiveCountdown(2)`の上にコールスタックに追加されます。

そして最後に、その関数呼び出しが`recursiveCountdown(0)`を呼び出す必要があるところに達すると、結果を待つために停止します。その間に`recursiveCountdown(0)`が`recursiveCountdown(1)`の上にコールスタックに追加されます。

しかし`recursiveCountdown(0)`は別の関数を呼び出さず、ベースケースに到達して早期にreturnします。その関数の実行が終了したため、その関数呼び出しは「解決済み」と見なされます。解決されると、スタックから削除されます。

これで`recursiveCountdown(1)`はその呼び出しを待っていないため、スタックの一番上にあり実行を再開できます。

`recursiveCountdown(1)`は解決され、スタックから削除され、`recursiveCountdown(2)`の実行が再開されます。

`recursiveCountdown(2)`は解決され、スタックから削除され、`recursiveCountdown(3)`の実行が再開されます。

`recursiveCountdown(3)`は解決され、スタックから削除され、`recursiveCountdown(4)`の実行が再開されます。

`recursiveCountdown(4)`は解決され、スタックから削除され、`recursiveCountdown(5)`の実行が再開されます。

そして`recursiveCountdown(5)`は解決され、スタックから削除されます。これでコールスタックは空になり、再帰は完了です！

これはJavaScriptにおける再帰の基本的な概要です。複雑な概念なので、コードやログ文を試しながらコールスタックの動作に慣れてください。

ちなみに、ベースケースがないとメモリ不足でコードがクラッシュすることがあると話しました。これは再帰がどんどん関数呼び出しをコールスタックに積み重ねていき、スタックオーバーフローを起こすためです。人気のあるプログラミングコミュニティの名前もここから来ています。
