# --interactive--

Es ist wichtig sicherzustellen, dass alle Nutzer, einschließlich Menschen mit Behinderungen, problemlos auf Websites zugreifen können.

Für Menschen, die Screenreader verwenden, liefern die `aria-label` und `aria-labelledby` Attribute entscheidende Informationen über Seitenelemente, die unklar oder unsichtbar sein könnten.

Schauen wir uns an, was die `aria-label` und `aria-labelledby` Attribute sind und welche Rollen sie bei der Barrierefreiheit des Webs für Menschen mit Sehbeeinträchtigungen und verwandten Behinderungen spielen.

Sie werden bemerken, dass sowohl `aria-label` als auch `aria-labelledby` mit `aria-` beginnen. Was bedeutet das? ARIA steht für Accessible Rich Internet Applications. Es ist eine Reihe von Attributen mit dem Präfix `aria-`, die Entwickler nutzen können, um assistiven Technologien den Zweck von Elementen zu vermitteln. Das Attribut `aria-label` ist eine unsichtbare Bezeichnung für interaktive Elemente. Es fügt einem Element eine Textbezeichnung hinzu, die Screenreader vorlesen können.

`aria-label` ist besonders nützlich für Elemente, die keinen sichtbaren Text haben, aber dennoch von Screenreadern beschrieben werden müssen. Zum Beispiel benötigen Schaltflächen, die nur Symbole enthalten, oft `aria-label`, um ihren Zweck zu vermitteln.

Hier ist ein Beispiel:

```html
<button aria-label="Search">
  <i class="fa-solid fa-magnifying-glass"></i>
</button>
```

In diesem Fall könnte ein Screenreader diese Schaltfläche als `Search, button` ankündigen, obwohl die Schaltfläche nur ein Symbol enthält. Das `aria-label` Attribut sagt Screenreadern, welchen Text sie anstelle des Symbols verwenden sollen.

Wenn die Schaltfläche statt eines Symbols den Text „Search“ enthielte, wäre das `aria-label` Attribut nicht nötig, da der Text die Bezeichnung für die Schaltfläche liefert.

Für Eingabeelemente liefert das `aria-label` Attribut direkt eine Bezeichnung, falls keine sichtbare Bezeichnung mit der Eingabe verknüpft ist.

Das `aria-labelledby` Attribut macht genau dasselbe wie das `aria-label` Attribut, aber anstatt den Text direkt im Attribut zu definieren, verwenden Sie eine Referenz auf einen Text, der bereits auf der Seite existiert. Der vorhandene Text muss ein `id` Attribut haben, das als Referenzwert im `aria-labelledby` Attribut verwendet wird.

Hier ist ein Beispiel:

:::interactive_editor

```html
<input type="text" aria-labelledby="search-btn">
<button type="button" id="search-btn">Search</button>
```

:::

In diesem Fall wird der Text des Buttons als Bezeichnung für das Suchfeld verwendet. Screenreader geben die Eingabe etwa als `Search, edit` aus. Wenn Sie später den Button-Text in `Find` ändern, wird die Bezeichnung für das Eingabefeld automatisch auf den neuen Text aktualisiert, da sie sich auf den Button-Text bezieht.

Es ist auch möglich, mehrere `id`-Werte zu einem einzigen Wert im `aria-labelledby`-Attribut zu kombinieren. So funktioniert das:

:::interactive_editor

```html
<div>
  <span id="volume-label">Volume</span>
  <span id="volume-details">Adjust the volume level</span>
  <input
    type="range"
    min="0"
    max="100"
    value="30"
    aria-labelledby="volume-label volume-details">
</div>
```

:::

Für den Schieberegler sucht der Screenreader nach dem Inhalt der `volume-label` und `volume-details` Elemente und kündigt `Volume Adjust the volume level` an.

Sie haben gesehen, dass sowohl `aria-label` als auch `aria-labelledby` Attributen Screenreadern helfen, zu verstehen, was Elemente tun. Welches sollten Sie also verwenden? Da beide dieselbe Funktionalität bieten, kann entweder verwendet werden, aber es gibt einige Vorteile, `aria-labelledby` gegenüber `aria-label` zu nutzen:

- Wenn jemand einen Übersetzungsdienst verwendet, um den Inhalt Ihrer Seite zu übersetzen, wird der Text in einem `aria-label` Attribut möglicherweise nicht immer übersetzt.
- Die Verwendung von `aria-labelledby` kann auch helfen, eine Diskrepanz zwischen dem sichtbaren Bezeichnungstext und dem unsichtbaren Label für Screenreader-Nutzer zu vermeiden, da eine Aktualisierung des sichtbaren Texts automatisch das unsichtbare Label aktualisiert.
- `aria-labelledby` kann es viel einfacher machen, komplexe unsichtbare Labels programmatisch zu erstellen, die aus mehreren Textquellen bestehen.

Ein letzter Hinweis: Verwenden Sie `aria-label` und `aria-labelledby` nicht gleichzeitig an einem Element. In diesem Fall wird das unsichtbare Label für Screenreader immer durch `aria-labelledby` bestimmt, und `aria-label` wird vollständig ignoriert.
