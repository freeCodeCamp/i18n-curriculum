# --interactive--

Scopriamo cos’è il DOM e perché è così importante per lo sviluppo web. DOM sta per Document Object Model. È un’interfaccia di programmazione che ci permette di interagire con i documenti HTML.

Con il DOM puoi aggiungere, modificare o eliminare elementi su una pagina web. Puoi persino rendere il tuo sito interattivo facendo in modo che gli elementi ascoltino e rispondano agli eventi.

Nel DOM, un documento HTML è rappresentato come un albero di nodi. Ogni nodo rappresenta un elemento HTML del documento HTML:

```html
<!DOCTYPE html>
<html lang="en">
  <head>
    <title>DOM Example</title>
  </head>
  <body>
    <h1>What is the DOM?</h1>
    <h2>Let's learn about the DOM</h2>
  </body>
</html>
```

Questo è un diagramma che rappresenta la struttura base del DOM del nostro esempio:

```md
Document
========

HTML
-----

- Head
  - Title
- Body
  - H1
  - H2   
```

Naturalmente, questa può essere più dettagliata e complessa in base alla struttura del markup HTML di una pagina web.

Il documento HTML è il nodo radice nella gerarchia del DOM. Ha un solo nodo figlio, l’elemento `html`. Questo è l’elemento radice del documento HTML, dato che tutti gli altri nodi discendono da esso.

L’elemento `html` ha due figli: `head` e `body`.

L’elemento `head` contiene i metadati sul documento. Fornisce informazioni essenziali sulla pagina web.

L’elemento `body` contiene il contenuto visibile della pagina web.

Puoi consultare questi elementi con JavaScript, lavorare con loro nel tuo codice e persino modificarli dinamicamente. Questa è la potenza del DOM e delle web API.

Per consultare questi elementi in JavaScript, puoi usare i metodi `getElementById()` e `querySelector()`. Questi metodi sono web API perché forniscono modi standardizzati per interagire con il DOM usando JavaScript.

Con `getElementById()` puoi ottenere un oggetto che rappresenta l’elemento HTML con il `id` specificato. Ricorda che gli `id` devono essere unici in ogni documento HTML, quindi questo metodo restituirà solo un oggetto `Element`. Ecco un esempio:

```js
const container = document.getElementById("container");
```

Questa riga di codice JavaScript ottiene un elemento con il valore `id` di `container` e assegna quell’oggetto a una costante JavaScript. Devi passare il `id` tra virgolette come argomento. Se stampi questo oggetto nella console, lo vedrai nell’output:

:::interactive_editor

```html
<div id="container">
  <h1>Hello, World!</h1>
  <p>Welcome to learning about the DOM.</p>
</div>
<script src="index.js"></script>
```

```js
const container = document.getElementById("container");
console.log(container); 
```

:::

`querySelector()` è più ampio di `getElementById()`. Con `querySelector()` puoi ottenere il primo elemento nel documento HTML che corrisponde al selettore CSS passato come argomento.

In questo esempio, otterrai l’elemento che corrisponde al selettore `section`:

:::interactive_editor

```html
<section>
  <h2>Section Title</h2>
  <p>This is a section of the webpage.</p>
</section>
<script src="index.js"></script>  
```

```js
const sectionEl = document.querySelector("section");
console.log(sectionEl);
```

:::

Se vuoi selezionare un elemento tramite il suo nome di classe usando `querySelector()`, devi anteporre al nome della classe un punto (`.`). Ecco un esempio:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<div class="highlight">
  <p>This content is highlighted.</p>
</div>
<script src="index.js"></script>
```

```css
body {
  background-color: lightgray;
}

.highlight {
  background-color: yellow;
}
```

```js
const highlightEl = document.querySelector(".highlight");
console.log(highlightEl);
```

:::

Hai anche altri metodi per selezionare elementi multipli, come `getElementsByClassName()` e `querySelectorAll()`. Ne imparerai di più nelle lezioni future.

Capendo come usare le web API per manipolare il DOM in modo efficiente, puoi creare applicazioni web potenti e interattive.
