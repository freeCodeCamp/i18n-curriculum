# --description--

加号 `+` 存在两种形式：你在上一个挑战中使用的二元形式和一元形式。

一元加号，或者换句话说，应用于单个值的一元操作符 `+`，对数字没有任何作用。但如果操作对象不是数字，一元加号会将其转换为数字。

例如：

```js
// No effect on numbers
let x = 1;
console.log( +x ); // 1

let y = -2;
console.log( +y ); // -2

// Converts non-numbers
console.log( +true ); // 1
console.log( +"" );   //
```

将字串转换为数字的需求非常常见。例如，如果你从 `HTML` 表单字段获取值，它们通常是字串。如果你想将它们相加怎么办？

二进制加号会将它们作为字串添加：

```js
let apples = "2";
let oranges = "3";

alert( apples + oranges ); // "23", the binary plus concatenates strings
```

如果你想将它们视为数字，你需要先转换然后求和：

```js
let apples = "2";
let oranges = "3";

// both values converted to numbers before the binary plus
alert( +apples + +oranges ); // 5

// the longer variant
// alert( Number(apples) + Number(oranges) ); // 5
```

从数学家的角度来看，大量的加号可能显得奇怪。但从程序员的角度来看，这没什么特别的：一元加号先应用，它们将字串转换为数字，然后二元加号将它们相加。

为什么一元加号会先于二元加号应用于值？正如你将看到的，那是因为它们具有*更高*的优先级。

# --questions--

## --text--
鉴于 JavaScript 中一元和二元 `+` 操作符的解释，如何使用一元 `+` 正确地将字串值 `'2'` 和 `'3'` 相加为数字？

## --answers--

`console.log('2' + '3'); // Outputs: '23'` 

---

`console.log(+2 + +3); // Outputs: 5`

---

`console.log(+'2' + +'3'); // Outputs: 5`

---

`console.log(Number('2') + Number('3')); // Outputs: '23'`


## --video-solution--

3
