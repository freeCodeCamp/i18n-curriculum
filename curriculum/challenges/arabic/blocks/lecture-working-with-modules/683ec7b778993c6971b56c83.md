# --description--

في تطوير البرمجيات، المكتبة تشبه صندوق أدوات للمطوّرين.

بدلًا من تنفيذ كل جزء من الشفرة بنفسك من الصفر، توفر لك المكتبة شفرة مكتوبة مسبقًا وقابلة لإعادة الاستخدام، مثل الدوال، الأصناف، وهياكل البيانات التي يمكنك استخدامها في مشاريعك.

تمتلك بايثون مكتبة قياسية واسعة تحتوي على العديد من الوحدات المدمجة المختلفة. كلها حلول موحدة ومُختبرة جيدًا للعديد من المشاكل والمهام التي ستواجهها يوميًا كمبرمج، مثل:

* التفاعل مع نظام التشغيل.
    
* العمل مع الملفات.
    
* الشبكات.
    
* التعامل مع التاريخ والوقت.
    
* إجراء العمليات الرياضية.
    
* استخدام التعبيرات النمطية.
    
* اختبار وتصحيح الأخطاء في الشفرة.
    
* وأكثر من ذلك بكثير!
    

بعض الأمثلة على الوحدات المدمجة الشهيرة هي `math`، `random`، `re` (اختصار لـ "التعبيرات النمطية"), و `datetime`.

تحتوي وحدة `math` على دوال مفيدة لأداء عمليات رياضية أكثر تعقيدًا.

وحدة `random` مفيدة لتوليد أرقام عشوائية.

تُستخدم وحدة `re` للعمل مع التعبيرات النمطية.

ووحدة `datetime` مفيدة للعمل مع التواريخ والأوقات في بايثون.

لكن كيف يمكنك الوصول إلى المتغيرات، الثوابت، الدوال، والأصناف المعرفة في هذه الوحدات المدمجة؟

تستخدم عبارة استيراد. تتيح لك هذه العبارات استيراد الوحدات إلى سكريبت بايثون الخاص بك. عادةً ما تُكتب عبارات الاستيراد في أعلى الملف. يمكنك أيضًا تخصيصها حسب حاجتك. أولًا، تستخدم عبارة `import`، تليها اسم الوحدة:

```python
import module_name
```

لنفترض أنك تريد استيراد وحدة `math`. في هذه الحالة، تكتب هذا في أعلى ملفك:

```python
import math
```

بعدها، إذا احتجت لاستدعاء دالة من تلك الوحدة في سكريبت بايثون، تستخدم ترميز النقطة، مع اسم الوحدة متبوعًا باسم الدالة:

```python
module_name.function_name() 
```

على سبيل المثال، للحصول على الجذر التربيعي للعدد 36، تكتب `math` متبوعًا بنقطة ثم `sqrt`، وهو اختصار للجذر التربيعي، وداخل الأقواس العادية تمرر المعلمات اللازمة. في هذه الحالة، نحتاج فقط لتمرير الرقم الذي نريد الجذر التربيعي له:

```python
math.sqrt(36)
```

هذه أبسط نسخة من عبارة الاستيراد، لكن هناك بدائل أخرى.

إذا احتجت لاستيراد الوحدة باسم مختلف (المعروف أيضًا بـ "الاسم المستعار"), يمكنك استخدام هذه الصيغة، مع `as` متبوعة بالاسم المستعار في نهاية عبارة الاستيراد:

```python
import module_name as module_alias
```

يُستخدم هذا غالبًا لتقصير أسماء الوحدات الطويلة، أو لتجنب تعارض الأسماء.

على سبيل المثال، للإشارة إلى وحدة `math` باسم `m` في شفرتك، يمكنك تعيين اسم مستعار لها، هكذا:

```python
import math as m 
```

بعدها، يمكنك الوصول إلى عناصر الوحدة باستخدام الاسم المستعار:

```python
m.sqrt(36)
```

لكن أحيانًا لا تحتاج لاستيراد كل شيء من وحدة. ربما تحتاج فقط إلى دالة أو صنفين محددين. بايثون توفر لك ما تحتاجه في هذه الحالة.

الآن تبدأ عبارة الاستيراد بـ `from`، تليها اسم الوحدة، ثم الكلمة المفتاحية `import` متبوعة بأسماء العناصر التي تريد استيرادها:

```python
from module_name import name1, name2
```

بعدها، يمكنك استخدام هذه الأسماء بدون بادئة اسم الوحدة في سكريبت بايثون.

إذا أردت تعيين أسماء مستعارة لهذه الأسماء، يمكنك فعل ذلك باستخدام الكلمة المفتاحية `as` بعد كل اسم، متبوعة بالاسم المستعار الذي تريد استخدامه:

```python
from module_name import name1 as alias1, name2 as alias2
```

لنفترض أنك تريد فقط استيراد دوال radians و sine و cosine من وحدة `math`. تكتب:

```python
from math import radians, sin, cos
```

الآن يمكنك استدعاء هذه الدوال مباشرة في شفرتك، بدون استخدام بادئة وحدة `math`.

هنا مثال أكثر تفصيلًا:

لإيجاد الجيب وجيب التمام لزاوية معينة معبر عنها أولًا بالدرجات، يمكننا استدعاء دالة `radians` لتحويلها إلى راديان، ثم استدعاء دالتي الجيب وجيب التمام، مع تمرير الزاوية بالراديان:

```python
from math import radians, sin, cos

angle_degrees = 40
angle_radians = radians(angle_degrees)

sine_value = sin(angle_radians)
cos_value = cos(angle_radians)

print(sine_value) # 0.6427876096865393
print(cos_value)  # 0.766044443118978
```

لاحظ كيف نستدعي الدوال مباشرة، بدون اسم الوحدة كبادئة. هذا لأننا استوردنا الدوال بهذه الصيغة البديلة.

هذا مفيد، لكنه قد يسبب تعارضات في الأسماء إذا كان لديك دوال أو متغيرات بنفس الاسم معرفة في سكريبت بايثون نفسه. لذا يجب أن تأخذ هذا في الاعتبار عند اختيار نوع عبارة الاستيراد التي تريد استخدامها.

وأخيرًا، نجد عبارة استيراد تنتهي بعلامة النجمة. النجمة تخبر بايثون أنك تريد استيراد كل شيء في تلك الوحدة، لكنك تريد استيرادها بحيث لا تحتاج لاستخدام اسم الوحدة كبادئة:

```python
from module_name import *
```

على سبيل المثال، إذا فعلت هذا أثناء استيراد وحدة `math`، ستتمكن من استدعاء أي دالة معرفة في تلك الوحدة بدون تحديد اسم الوحدة كبادئة. إليك بعض الأمثلة:

```python
from math import *
print(sqrt(36))  # 6.0
print(pow(5, 2)) # 25.0
print(exp(1))    # 2.718281828459045
```

مع ذلك، يُنصح عمومًا بتجنب هذا لأنه قد يؤدي إلى تصادم في فضاء الأسماء، ويجعل من الصعب معرفة مصدر بعض الأسماء.

تعمل عبارات الاستيراد بنفس الطريقة تمامًا مع الدوال، الأصناف، الثوابت، المتغيرات، وأي عناصر أخرى معرفة في الوحدة.

إليك مثال على ثابت من وحدة `math`، وهو الرقم `pi`:

```python
import math
print(math.pi)
```

وهنا مثال على صنف من وحدة `datetime`. ننشئ كائن تاريخ يمثل 15 يوليو 1959. ثم نعيّن ذلك الكائن `date` إلى متغير ونصل إلى اليوم، الشهر، والسنة بشكل منفصل باستخدام ترميز النقطة:

```python
import datetime
birthday = datetime.date(1959, 7, 15)
print(birthday.day)    # 15
print(birthday.month)  # 7
print(birthday.year)   # 1959
```

يمكنك العثور على مزيد من المعلومات حول محتوى الوحدة في الوثائق الرسمية لبايثون الخاصة بتلك الوحدة.

رائع. الآن بعد أن عرفت المزيد عن الوحدات، يجب أن تعرف أيضًا عن هذا التعبير المهم جدًا في سكريبتات بايثون، لأنه مرتبط ارتباطًا وثيقًا:

```python
if __name__ == '__main__': 
    # Code
```

`__name__` هو متغير مدمج خاص في بايثون.

عندما يُنفذ ملف بايثون مباشرة، يعيّن بايثون قيمة هذا المتغير إلى السلسلة النصية `"__main__"`.

لكن إذا تم استيراد ملف بايثون كوحدة في سكريبت بايثون آخر، تُعيّن قيمة المتغير `__name__` إلى اسم تلك الوحدة (عادة اسم الملف بدون امتداد `.py`).

لهذا السبب غالبًا ما تجد هذا الشرط في سكريبتات بايثون. يحتوي على الشفرة التي تريد تشغيلها **فقط إذا** كان سكريبت بايثون يعمل كبرنامج رئيسي:

```python
if __name__ == '__main__': 
    # Code
```

لكن إذا تم استيراد السكريبت كوحدة، لا تُنفذ الشفرة داخل ذلك القسم.

هذا مفيد لأنه يسمح لسكريبتات بايثون بأن يكون لها غرضان. يمكن تشغيلها مباشرة لتنفيذ المنطق الرئيسي، أو يمكن استيرادها في وحدة أخرى بدون تنفيذ المنطق الرئيسي.
