# --description--

サーバーは現代のウェブに欠かせない存在です。ユーザーとして私たちが毎日利用するオンライン体験を支え、開発者としてあなたが作るほとんどの体験も支えることになります。

それでは、サーバーについてもっと学んでいきましょう。

ご存知のように、インターネットは相互に接続されたコンピュータネットワークのグローバルなシステムです。サーバーはこの通信を可能にする中心的なコンポーネントであり、データ交換のプロセスを処理します。

**サーバー**とは、他のコンピュータからのリクエストを受け取り、ネットワークを通じてデータやサービスをレスポンスとして送信するコンピュータまたはコンピュータプログラムのことです。

サーバーは二重の概念であり、以下のどちらかを指します：

* これらのサービスを効率的かつ継続的に提供するために設計された、実際の物理的で強力なコンピュータ**ハードウェア**。これらの機械は信頼性を重視して構築・設計されています。
    
* または、その物理ハードウェア上で動作し、サービスを提供する実際のコンピュータ**ソフトウェア**。
    
「サーバー」と呼ばれる物理的なコンピュータは通常、データセンターに保管されています。データセンターは大規模で安全な施設で、環境が管理されています。

サーバーは**クライアント・サーバーモデル**の重要な一部です。

クライアント・サーバーモデルは、コンポーネントがクライアントかサーバーのいずれかである分散システムのアーキテクチャです。

* **クライアント**は、データやサービスを求めてサーバーにリクエストを送るプロセスやアプリケーションです。例えば、ウェブブラウザです。
    
* **サーバー**は、そのリクエストを処理し、レスポンスを返す物理的なデバイスやコンピュータプログラムです。

基本的に、クライアントとサーバーはリクエスト・レスポンスのサイクルの一部であり、クライアントがサーバーにリクエストを送り、サーバーがクライアントにレスポンスを返します。

例えば、ブラウザ（クライアント）を使ってユーザー名とパスワードでアプリケーションに認証するとき、あなたの情報は安全にサーバーに送信されます。サーバーは入力されたユーザー名とパスワードがプラットフォームのデータベースにある情報と一致するかどうかを確認します。一致すれば、サーバーはクライアント（あなたのブラウザ）に成功のレスポンスを返し、サイトへの認証が成功します。

一致しなければ、サーバーは失敗のレスポンスをクライアントに返し、通常はログイン試行が失敗した理由を説明するエラーやフィードバックが表示されます。

この認証のロジックはすべてサーバー上で実行されます。サーバーは資格情報の検証、データベースとのやり取り、各リクエストへのレスポンスの決定を担当します。

各サーバーレスポンスにはHTTPステータスコードが含まれており、クライアントに何が起こったかを伝えます。これらのステータスコードは、リクエストが成功したか失敗したか、そしてその理由をクライアントに知らせます。

例えば：

* 200はすべてが正常であったことを意味します。
    
* 404はサーバー上にリソースが見つからなかったことを意味します。
    
* 500は内部サーバーエラーが発生したことを意味します。
    
* 503はサーバーが一時的に利用できないことを意味します。
    
* その他もあります…
    

各ステータスコードには固有の意味があります。

サーバーを利用する実際のアプリケーションの例には、電子商取引、検索エンジン、ゲーム、ビデオストリーミング、クラウドストレージやコラボレーションプラットフォーム、ソーシャルメディアプラットフォーム、金融アプリケーションなどがあります。

データが動的に更新されたり、アプリケーションがデータベースと何らかの通信を必要とする場合、サーバーは通信プロセスの重要なコンポーネントである可能性が高いです。

主なサーバーの種類には以下があります：

* **ウェブサーバー**はHTMLページ、画像、CSSファイルなどのウェブコンテンツをブラウザに配信します。
    
* **アプリケーションサーバー**はビジネスロジックを処理し、ユーザーの入力を処理し、レスポンスを調整します。
    
* **データベースサーバー**はデータベース管理システムを実行し、データの保存、取得、管理を行います。
    
* **メールサーバー**はメールメッセージの送受信と保存を行います。
    
* **ファイルサーバー**はファイルへのアクセスを管理し、ネットワーク上での保存や共有を可能にします。
    
* **プロキシサーバー**はクライアントと他のサーバーの間の仲介役を果たし、キャッシュ、セキュリティ、トラフィック制御などに使われます。

サーバーは私たちが毎日使う強力なオンラインプラットフォームの基盤です。複雑なロジックを実行し、大量のデータを保存・処理し、同時に何百万ものユーザーを支えることを可能にします。サーバーがなければ、私たちが頼っている現代のデジタル世界は存在しません。
