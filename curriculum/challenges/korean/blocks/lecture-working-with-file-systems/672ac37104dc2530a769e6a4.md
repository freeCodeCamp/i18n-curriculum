# --description--

자신만의 웹 애플리케이션을 만들기 시작할 때, 파일 이름을 어떻게 정할지 신경 써야 합니다.

파일 이름은 이해하기 쉽고 유지 관리하기 편한 방식으로 정해야 합니다.

HTML 파일 이름의 좋은 예와 나쁜 예를 살펴보겠습니다. 여기서 `.html` 확장자는 이 파일이 HTML 파일임을 알려줍니다.

나쁜 파일 이름 예시는 다음과 같습니다:

```md
index1234.html
```

이 이름만으로는 파일 내용이 무엇인지 알기 어렵습니다. 이 파일이 제품 페이지, 블로그 페이지, 또는 다른 어떤 페이지인지 파일 이름만으로는 알 수 없습니다.

HTML 파일에 더 적합한 파일 이름 예시는 다음과 같습니다:

```md
about-us.html
```

`about-us.html` 같은 더 설명적인 이름을 사용하면 파일 내용이 명확해집니다.

또한 이 파일 이름에는 공백이 없고, 대신 단어를 구분하기 위해 하이픈 `-`을 사용한 것을 알 수 있습니다.

가끔 `index.html`이라는 파일 이름을 볼 수 있는데, 이것은 웹사이트의 메인 페이지를 나타내는 특별한 파일 이름입니다. 웹사이트를 방문할 때 `index.html` 파일이 가장 먼저 로드됩니다.

파일 이름을 정할 때 또 하나 중요한 점은 특수 문자의 사용입니다.

특수 문자, 숫자, 문자들이 혼합되면 파일 내용이 무엇인지 이해하기 어려워질 수 있습니다.

나쁜 파일 이름 예시는 다음과 같습니다:

```md
file-1!@.html
```

이 이름은 불필요하게 복잡해 보이고 파일 내용에 대한 정보를 제공하지 않습니다.

따라서 파일 이름에는 문자와 하이픈만 사용하는 것이 가장 좋습니다.

더 많은 언어를 배우면서, 그 언어들에도 파일 이름을 정하는 공통 관례가 있다는 것을 알게 될 것입니다.

이럴 때는 공식 문서나 다른 자료를 참고하여 해당 언어에서 파일 이름을 정하는 모범 사례를 이해하는 것이 좋습니다.

또한 팀에서 작업할 때는 팀이 정한 파일 이름 관례를 따라야 하는 경우가 있습니다.

항상 팀의 스타일 가이드를 참고하여 파일 이름이 팀 내 다른 파일들과 일관되도록 하세요.
