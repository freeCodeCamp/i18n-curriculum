# --description--

Um eine Tabelle in SQL zu definieren, müssen Sie den Datentyp jeder Spalte angeben.

Denken Sie daran, dass dies die grundlegende Syntax zum Erstellen einer Tabelle in SQL ist. Jede Spalte hat ihren eigenen Datentyp.

```sql
CREATE TABLE table_name(
  column1 data_type column_constraint,
  column2 data_type column_constraint,
  column3 data_type column_constraint,
  ... etc
);
```

Als Entwickler ist es Ihr Ziel, diese Datentypen so genau wie möglich auszuwählen.

Es gibt sechs beliebte Kategorien von Datentypen in SQL:

- Numerisch, wie `INTEGER`, `FLOAT`, `SERIAL` und `DECIMAL`.
- Datum und Uhrzeit, wie `TIMESTAMP`, `DATE` und `TIME`.
- Zeichen und Strings. Dazu gehören `CHAR`, `VARCHAR` und `TEXT`.
- Unicode, einschließlich `NTEXT` und `NVARCHAR`. Diese werden verwendet, um sicherzustellen, dass Text korrekt gespeichert und abgerufen wird, unabhängig von der Herkunft der Zeichen.
- Binär, verwendet zum Speichern von nicht-textuellen Daten wie Bildern, Audio- und Videodateien.
- Und andere verschiedene Datentypen, wie `XML` und `TABLE`.

In dieser Lektion behandeln wir einige der am häufigsten verwendeten mit PostgreSQL. Datentypen variieren je nach Datenbankmanagementsystem, sind aber im Allgemeinen ziemlich ähnlich. Daher sollten Sie die vollständige Liste in der Dokumentation nachschlagen.

Wir beginnen mit numerischen Werten.

Zuerst haben wir den Datentyp `INTEGER`:

```sql
units_sold INTEGER
```

In diesem Beispiel weisen wir diesen Datentyp einer `units_sold`-Spalte zu.

`INTEGER`-Typen benötigen 4 Bytes in der Datenbank und können Werte von -2.147.483.648 bis 2.147.483.647 annehmen. Die offizielle PostgreSQL-Dokumentation beschreibt ihn als die „typische Wahl für ganze Zahlen“.

Wir haben auch `SMALLINT` und `BIGINT`, die im Grunde wie `INTEGER` sind, aber aufgrund der ihnen in der Datenbank zugewiesenen Größe einen kleineren bzw. größeren Zahlenbereich haben.

Eine nützliche Funktion in PostgreSQL ist die Erstellung einer Spalte mit dem Schlüsselwort `SERIAL`. Hier wird eine `id`-Spalte mit `SERIAL` erstellt:

```sql
id SERIAL
```

Obwohl es kein echter Datentyp ist, ist es sehr hilfreich, um eindeutige Identifikatorspalten zu erstellen, da die Spalte einen `INTEGER`-Typ hat, keine `NULL`-Werte zulässt und automatisch inkrementiert wird, wenn Zeilen hinzugefügt werden.

Zum Beispiel hat die erste Zeile automatisch eine `id` von `1`, die zweite Zeile eine `id` von `2` und so weiter, wodurch für jeden Datensatz eine eindeutige ID erstellt wird.

In MySQL entspricht `SERIAL` dem Attribut `AUTO_INCREMENT`, das verwendet wird, um automatisch fortlaufende ganze Zahlen zu generieren:

```sql
id INT AUTO_INCREMENT
```

Sie sehen, dass verschiedene Datenbankmanagementsysteme unterschiedliche Wege haben können, um genau dieselbe Funktionalität in Ihrer Datenbank zu erreichen.

Dies sind die am häufigsten verwendeten numerischen Datentypen, aber manchmal müssen Sie Text oder Zeichenfolgen darstellen.

Dafür gibt es den Datentyp `VARCHAR`. Dieser Datentyp wird für Strings variabler Länge verwendet. Sie können die maximale Zeichenlänge in runden Klammern angeben:

```sql
name VARCHAR(50)
```

In diesem Beispiel definieren wir eine `name`-Spalte, bei der die Werte bis zu 50 Zeichen lang sein können.

Dieser Datentyp legt eine maximale Länge für die Strings fest, aber wenn Sie Strings beliebiger Länge speichern müssen, können Sie stattdessen `TEXT` verwenden:

```sql
name TEXT
```

Neben Zahlen und Strings ist es auch üblich, Daten und Uhrzeiten zu speichern.

Wenn Sie beispielsweise eine Tabelle zur Speicherung von Ereignissen erstellen, müssen Sie möglicherweise das Datum jedes Ereignisses in einem bestimmten Format speichern. Dafür gibt es den Datentyp `DATE`:

```sql
event_date DATE
```

Um die Uhrzeit zu speichern, können Sie auch den Datentyp `TIME` verwenden. Zum Beispiel könnten Sie die Startzeit eines Ereignisses speichern:

```sql
start_time TIME
```

Der Datentyp `TIMESTAMP` kombiniert beides. Er enthält sowohl Datum als auch Uhrzeit. Er kann auch die Zeitzone enthalten, wenn Sie `TIMESTAMP WITH TIME ZONE` schreiben:

```sql
event_timestamp TIMESTAMP
```

```sql
event_timestamp TIMESTAMP WITH TIME ZONE
```

Und schließlich, wenn Sie einen Boolean-Wert speichern müssen, `TRUE` oder `FALSE`, können Sie den Datentyp `BOOLEAN` verwenden:

```sql
is_active BOOLEAN
```

Jetzt kennen Sie einige der gebräuchlichsten Datentypen in SQL, aber es gibt noch viele mehr. Sie können die Dokumentation Ihres Datenbankmanagementsystems konsultieren, um weitere Informationen über die verfügbaren Datentypen zu erhalten.
