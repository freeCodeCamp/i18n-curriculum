# --interactive--

In einer früheren Lektion wurden Sie erstmals mit dem Konzept der Booleans vertraut gemacht, aber in dieser Lektion werden wir tiefer darauf eingehen, wie Booleans funktionieren und wie die Gleichheits- und Ungleichheitsoperatoren arbeiten.

Booleans sind ein Datentyp mit nur `true` und `false` Werten. Sie sind nützlich, weil sie es Ihnen ermöglichen, etwas basierend auf bestimmten Bedingungen zu tun. Booleans sind essenziell, wenn Sie bewerten möchten, ob etwas passieren soll oder nicht, zum Beispiel um zu entscheiden, ob jemand auf eine bestimmte Funktion in Ihrer App zugreifen kann. Hier ist ein Beispiel, wie der Wert `true` einer Variablen namens `isOldEnoughToDrive` zugewiesen wird:

:::interactive_editor

```js
let isOldEnoughToDrive = true;

console.log(isOldEnoughToDrive); // true
```

:::

Sie können diese Variable in einer Bedingung wie folgt verwenden:

:::interactive_editor

```js
let isOldEnoughToDrive = true;

if (isOldEnoughToDrive) {
 console.log("You're old enough to drive"); // You're old enough to drive
} else {
 console.log("Sorry, you are not old enough to drive");
}
```

:::

Eine Bedingung hilft Ihnen, Entscheidungen in Ihrem Code basierend auf einer Bedingung zu treffen. Dieses Beispiel verwendet das, was man eine `if/else` Anweisung nennt.

Wenn `isOldEnoughToDrive` `true` ist, wird der Satz `You're old enough to drive` in der Konsole ausgegeben. Andernfalls, wenn `isOldEnoughToDrive` `false` ist, wird der Satz `Sorry, you are not old enough to drive` in der Konsole ausgegeben. Da die Variable `isOldEnoughToDrive` auf `true` gesetzt ist, wird der erste Satz in der Konsole ausgegeben. Mehr über `if/else` Anweisungen lernen Sie in einer zukünftigen Lektion.

Um zwei Werte zu vergleichen, können Sie entweder den Gleichheitsoperator oder den strikten Gleichheitsoperator verwenden. Das Ergebnis des Vergleichs ist ein Boolean mit dem Wert `true` oder `false`. Hier ist ein Beispiel für die Verwendung des Gleichheitsoperators, um einen String und eine Zahl zu vergleichen. Der Gleichheitsoperator wird durch ein doppeltes Gleichheitszeichen (`==`) dargestellt.

:::interactive_editor

```js
console.log(5 == "5"); // true
```

:::

In diesem Beispiel wandelt JavaScript den String `"5"` in die Zahl `5` um und prüft dann, ob sie gleich sind. Da beide Werte jetzt gleich sind, ist das Ergebnis true. Der Gleichheitsoperator verwendet vor dem Vergleich eine Typumwandlung.

Dies unterscheidet sich vom strikten Gleichheitsoperator, der keine Typumwandlung durchführt. Der strikte Gleichheitsoperator prüft, ob die Typen gleich sind und ob die Werte gleich sind. Hier ist ein Beispiel für die Verwendung des strikten Gleichheitsoperators, um eine Zahl und einen String zu vergleichen. Dieser Operator wird durch ein dreifaches Gleichheitszeichen (`===`) dargestellt.

:::interactive_editor

```js
console.log(5 === '5'); // false
```

:::

Der folgende Vergleich ergibt `false`, weil ein String-Datentyp nicht derselbe ist wie ein Zahl-Datentyp. Wenn Sie prüfen möchten, ob etwas ungleich einem anderen Wert ist, können Sie den Ungleichheitsoperator oder den strikten Ungleichheitsoperator verwenden. Hier ist ein Beispiel für die Verwendung des Ungleichheitsoperators (`!=`), um eine Zahl mit einem String zu vergleichen.

:::interactive_editor

```js
console.log(5 != "5"); // false
```

:::

In diesem Beispiel wäre das Ergebnis `false`, weil der Ungleichheitsoperator zuerst den String-Wert in eine Zahl umwandelt und dann die Werte vergleicht. Da die Werte gleich wären, gibt er `false` zurück. Wenn Sie den strikten Ungleichheitsoperator verwenden würden, erhalten Sie ein anderes Ergebnis. Der strikte Ungleichheitsoperator wird durch ein Ausrufezeichen gefolgt von zwei Gleichheitszeichen (`!==`) dargestellt.

:::interactive_editor

```js
console.log(5 !== "5"); // true
```

:::

Das Ergebnis wäre `true`, weil der strikte Ungleichheitsoperator keine Typumwandlung durchführt. Da die Zahl `5` nicht gleich dem String `"5"` ist, ist das Ergebnis `true`.

Es gilt als bewährte Vorgehensweise, wann immer möglich strikte Ungleichheits- und Gleichheitsoperatoren zu verwenden, da sie keine Typumwandlung durchführen. In professionellen Projekten werden Sie meistens Codebasen sehen, die diese beiden Operatoren den Ungleichheits- und Gleichheitsoperatoren vorziehen.
