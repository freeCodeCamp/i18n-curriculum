# --description--

النماذج أساسية في كل تطبيق ويب لأنها تتيح لك التعامل مع مُدخلات المستخدم، جمع البيانات، وتنفيذ الإجراءات.

في React، تُدار النماذج باستخدام الحالة أو المراجع، مما يمنحك تحكمًا كاملاً في سلوكها والتحقق منها. هاتان الطريقتان لإدارة النماذج تُسمى "المُدخلات المُتحكم بها" و"المُدخلات غير المُتحكم بها".

لنلقِ نظرة على ماهية المُدخلات المُتحكم بها وغير المُتحكم بها.

المُدخل المُتحكم به هو الطريقة الأكثر "شبهًا بـ React" للتعامل مع مُدخلات النموذج. مع المُدخلات المُتحكم بها، تخزن قيمة حقل الإدخال في الحالة وتُحدّثها عبر أحداث `onChange`. هذا يمنحك تحكمًا كاملاً في بيانات النموذج ويسمح بالتحقق الفوري والعرض الشرطي.

تعمل العملية كالتالي: يحتفظ React بحالة النموذج باستخدام الخطاف `useState`، وتُحدّثها عند كل تغيير. عندما يكتب المستخدم في حقل الإدخال، يُطلق حدث `onChange`، يُحدّث الحالة، ويُعيد React عرض المكوِّن بالقيمة الجديدة.

```jsx
import { useState } from "react";

function App() {
  const [name, setName] = useState("");

  const handleChange = (e) => {
    setName(e.target.value);
  };

  const handleSubmit = (e) => {
    e.preventDefault();
    console.log(name);
  };

  return (
    <>
      <form onSubmit={handleSubmit}>
        <label htmlFor="name">Your name</label> <br />
        <input value={name} id="name" onChange={handleChange} type="text" />
        <button type="submit">Submit</button>
      </form> 
    </>
  );
}

export default App;
```

تشمل فوائد المُدخلات المُتحكم بها ما يلي:

- وصول فوري إلى بيانات النموذج.

- يمكنك تنفيذ التحقق الفوري.

- يمكنك تعطيل زر الإرسال بشكل شرطي.

- يمكنك التحكم في قيمة الإدخال برمجيًا.

أما المُدخلات غير المُتحكم بها فتُرى أكثر في نماذج HTML التقليدية. بدلاً من التعامل مع المُدخلات عبر الخطاف `useState`، تحتفظ المُدخلات غير المُتحكم بها في HTML بحالتها الداخلية بمساعدة DOM.

بما أن DOM يتحكم في قيم الإدخال، ما تحتاجه هو سحب قيم حقول الإدخال باستخدام المرجع (ref). هذا الأسلوب يتطلب شفرة أقل ويؤدي أداءً أفضل لأن المراجع لا تجعل React يُعيد العرض.

إليك مثال على المُدخلات غير المُتحكم بها:

```jsx
import { useRef } from "react";

function App() {
  const nameRef = useRef();

  const handleSubmit = (e) => {
    e.preventDefault();
    console.log(nameRef.current.value);
  };

  return (
    <form onSubmit={handleSubmit}>
      <label htmlFor="name">Your</label>{" "}
      <input type="text" ref={nameRef} id="name" />
      <button type="submit">Submit</button>
    </form>
  );
}

export default App;
```

ميزة واضحة جدًا للمُدخلات غير المُتحكم بها هي أنها تتطلب شفرة أقل. كما أنها تؤدي أداءً أفضل وتشعر بأنها أكثر طبيعية للمبتدئين في React الذين يعرفون HTML.

فأيهما يجب أن تستخدم بين المُدخلات المُتحكم بها وغير المُتحكم بها؟

استخدم المُدخلات المُتحكم بها عندما تحتاج إلى تحديثات ديناميكية للنموذج، تحقق في الوقت الحقيقي، أو عندما تريد مزامنة قيم الإدخال مع الحالة. توفر تحكمًا أفضل لكنها تتطلب إعادة عرض أكثر.

استخدم المُدخلات غير المُتحكم بها عندما تحتاج إلى نماذج أبسط، تريد الوصول إلى القيم فقط عند الإرسال، أو عندما تعمل مع شفرة غير React.

بغض النظر عن اختيارك بين المُدخلات المُتحكم بها وغير المُتحكم بها، إليك بعض أفضل الممارسات التي يجب الالتزام بها عند إنشاء النماذج في React:

- دائمًا امنع الإرسال الافتراضي للنموذج.

- تأكد من التحقق من المُدخلات قبل الإرسال.

- قدم دائمًا ملاحظات واضحة للمستخدمين مع حالات التحميل، أخطاء التحقق أو الحالات ذات الصلة الأخرى.
