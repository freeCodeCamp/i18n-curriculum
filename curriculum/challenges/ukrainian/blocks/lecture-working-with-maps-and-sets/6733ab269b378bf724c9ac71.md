# --description--

У JavaScript `Set` — це вбудований об’єкт для керування колекціями даних. Він дозволяє зберігати унікальні значення будь-якого типу, як примітивні, так і референси на об’єкти. `Set` гарантує, що кожне значення в ньому з’являється лише один раз, що робить його корисним для видалення дублікатів із масиву або роботи з колекціями унікальних значень.

Щодо `WeakSet`, це спеціальний тип `Set` з обмеженими функціями, який дозволяє зберігати слабкі посилання на об’єкти та символи. На відміну від `Set`, `WeakSet` не підтримує примітиви, такі як числа або рядки.

На відміну від звичайного `Set`, `WeakSet` зберігає лише об’єкти, а посилання на ці об’єкти є «слабкими», тобто WeakSets не перешкоджають збору сміття для збережених об’єктів, якщо на них немає інших посилань. Простими словами, якщо об’єкт ніде більше не використовується у вашому коді, він автоматично видаляється для звільнення пам’яті.

Щоб створити `Set`, використовуйте конструктор `Set` і призначте його змінній:

```js
const myFirstSet = new Set();
```

Також можна ініціалізувати `Set` значеннями:

```js
const treeSet = new Set(['Baobab', 'Jackalberry', 'Mopane Tree', 'Breadfruit']);
```

Якщо вивести `Set` у консоль, вихідні дані виглядають так:

```js
/*
Set(4) {'Baobab', 'Jackalberry', 'Mopane Tree', 'Breadfruit'}
  [[Entries]]
  0: "Baobab"
    value: "Baobab"
  1: "Jackalberry"
    value: "Jackalberry"
  2: "Mopane Tree"
    value: "Mopane Tree"
  3: "Breadfruit"
    value: "Breadfruit"
  size: 4
  [[Prototype]]: Set
*/
```

Якщо ви не ініціалізували `Set` значеннями, можна використати метод `add()`, щоб додати елемент до `Set`:

```js
const treeSet = new Set();

// Add items to the treeSet
treeSet.add('Baobab');
treeSet.add('Jackalberry');
treeSet.add('Mopane Tree');
treeSet.add('Breadfruit');
```

Результат і вигляд у консолі залишаються такими ж.

Не забувайте, що дублікати ігноруються в `Set`:

```js
const treeSet = new Set();

// Add items to the treeSet
treeSet.add('Baobab');
treeSet.add('Jackalberry');
treeSet.add('Mopane Tree');
treeSet.add('Breadfruit');
treeSet.add('Baobab'); //duplicate item will be ignored

console.log(treeSet);
// Set(4) {'Baobab', 'Jackalberry', 'Mopane Tree', 'Breadfruit'}
```

Інші методи, які можна використовувати для маніпуляції `Set`, це:

- `delete()`  
- `clear()`
- `has()`
- `entries()`
- `forEach()`
- `keys()`
- `values()`

Розглянемо, як працюють ці методи по черзі. `delete()` видаляє вказаний елемент із `Set`:

```js
const treeSet = new Set();

// Add items to the treeSet
treeSet.add('Baobab');
treeSet.add('Jackalberry');
treeSet.add('Mopane Tree');
treeSet.add('Breadfruit');

treeSet.delete('Breadfruit');

console.log(treeSet); // Set(3) {'Baobab', 'Jackalberry', 'Mopane Tree'}
```

`has()` перевіряє, чи існує вказане значення в `Set`:

```js
const treeSet = new Set();

// Add items to the treeSet
treeSet.add('Baobab');
treeSet.add('Jackalberry');
treeSet.add('Mopane Tree');
treeSet.add('Breadfruit');

treeSet.delete('Breadfruit');

console.log(treeSet.has('Breadfruit')); // false
```

`entries()` повертає ітератор `Set`, що містить масив значень у форматі `[value, value]`:

```js
const treeSet = new Set();

// Add items to the treeSet
treeSet.add('Baobab');
treeSet.add('Jackalberry');
treeSet.add('Mopane Tree');
treeSet.add('Breadfruit');

treeSet.delete('Breadfruit');

console.log(treeSet.entries());
// SetIterator {'Baobab' => 'Baobab', 'Jackalberry' => 'Jackalberry', 'Mopane Tree' => 'Mopane Tree'}
```

`keys()` і `values()` показують значення в `Set`. `keys()` — це просто псевдонім для методу `values()`:

```js
const treeSet = new Set();

// Add items to the treeSet
treeSet.add('Baobab');
treeSet.add('Jackalberry');
treeSet.add('Mopane Tree');
treeSet.add('Breadfruit');

treeSet.delete('Breadfruit');

console.log('Keys: ', treeSet.keys());
console.log('Values: ', treeSet.values());
// Keys: SetIterator {'Baobab', 'Jackalberry', 'Mopane Tree'}
// Values: SetIterator {'Baobab', 'Jackalberry', 'Mopane Tree'}
```

`forEach()` дозволяє ітерувати по `Set`:

```js
const treeSet = new Set();

// Add items to the treeSet
treeSet.add('Baobab');
treeSet.add('Jackalberry');
treeSet.add('Mopane Tree');
treeSet.add('Breadfruit');

treeSet.delete('Breadfruit');

treeSet.forEach((tree) => console.log(tree));
/*
Baobab
Jackalberry
Mopane Tree
*/
```

`clear()` видаляє всі елементи масиву:

```js
const treeSet = new Set();

// Add items to the treeSet
treeSet.add('Baobab');
treeSet.add('Jackalberry');
treeSet.add('Mopane Tree');
treeSet.add('Breadfruit');

treeSet.delete('Breadfruit');

treeSet.clear();

console.log(treeSet); // Set(0) {size: 0}
```

Також варто згадати, що є властивість `size`, яка повертає кількість елементів у `Set`:

```js
const treeSet = new Set();

// Add items to the treeSet
treeSet.add('Baobab');
treeSet.add('Jackalberry');
treeSet.add('Mopane Tree');
treeSet.add('Breadfruit');

treeSet.delete('Breadfruit');

console.log(treeSet.size); // 3
```

Як і `Set`, існує також конструктор `WeakSet`, який можна використати для створення `WeakSet`:

```javascript
const treeWeakSet = new WeakSet();
```

`WeakSet` також має методи `add()`, `delete()` і `has()`:

```javascript
const treeWeakSet = new WeakSet();

treeWeakSet.add({ name: 'Baobab' });
treeWeakSet.add({ name: 'Jackalberry' });
treeWeakSet.add({ name: 'Mopane Tree' });
treeWeakSet.add({ name: 'Breadfruit' });

treeWeakSet.delete('Jackalberry');
console.log(treeWeakSet.has('Jackalberry')); // false

console.log(treeWeakSet);
```

У вихідних даних вміст `WeakSet` виглядає так:

```javascript
/*
WeakSet {{…}, {…}, {…}, {…}}
  [[Entries]]
    No properties
  [[Prototype]]: WeakSet
    .
    .
    .
*/
```

Вміст здається порожнім, тому що WeakSets не є ітерованими і не відкривають свій вміст безпосередньо.

Не забувайте, що підтримуються лише символи та об’єкти з чітко визначеними ключами і значеннями. Додавання примітивів, таких як числа або рядки, призведе до помилки:

```js
treeWeakSet.add('Alan Smith');

console.log(treeWeakSet); // Invalid value used in weak set
//    at WeakSet.add (<anonymous>)
```

Ключова відмінність між `Set` і `WeakSet` полягає в тому, що `Set` зберігає будь-яке значення, тоді як `WeakSet` може зберігати лише об’єкти.

Ось ще деякі помітні відмінності між `Set` і `WeakSet`:

| Функція | Множина | WeakSet |
| --- | --- | --- |
| Тип збережених значень | Зберігає будь-який тип даних | Зберігає лише об’єкти |
| Посилання | Сильні посилання | Слабкі посилання |
| Ітерація | Підтримує ітерацію з `forEach` і циклами | Не підтримує ітерацію |
| Методи та властивості | `add()`, `delete()`, `has()`, `keys()`, `values()`, `size` та інші | Лише `add()`, `delete()` і `has()` |
| Використання | Загальна колекція унікальних значень і видалення дублікатів із масивів | Ефективне відстеження пам’яті для посилань на об’єкти |

Ви можете побачити відмінності в типах значень, які можуть зберігати ці два види множин, їх підтримку ітерації по збережених об’єктах і їх ідеальні випадки використання. Приділіть хвилину, щоб ознайомитися з вмістом цієї таблиці.
