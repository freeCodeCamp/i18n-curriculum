# --description--

Configurar autenticação personalizada exige escrever uma quantidade significativa de lógica manual. Você precisa lidar com a leitura dos cabeçalhos da requisição, extrair cookies, verificar assinaturas e gerenciar erros.

Embora construir isso por conta própria seja ótimo para aprender, fazer isso para toda aplicação em produção consome muito tempo e é propenso a falhas de segurança. É por isso que os desenvolvedores usam o Passport.js.

Passport.js é um middleware de autenticação para Node.js. Ele funciona como um framework organizado que padroniza como você autentica requisições. Ele cuida dos detalhes de autenticação de baixo nível, permitindo que você foque na lógica de negócio da sua aplicação.

A ideia central por trás do Passport.js é que toda aplicação requer diferentes formas de login, mas o fluxo de autenticação subjacente deve permanecer o mesmo. O Passport alcança essa flexibilidade por meio de plugins modulares chamados **Strategies**.

Em vez de escrever código personalizado para cada método de autenticação, você instala a strategy específica que precisa. Existem mais de 500 strategies disponíveis, cobrindo quase todos os métodos de autenticação usados hoje. Algumas delas são:

- **passport-local:** Lida com autenticação tradicional de nome de usuário e senha contra seu próprio banco de dados.

- **passport-jwt:** Protege endpoints de API extraindo e validando JSON Web Tokens das requisições recebidas.

- **passport-google-oauth20:** Permite que usuários façam login instantaneamente usando suas contas Google existentes.

- **passport-facebook:** Login com contas do Facebook via OAuth 2.0.

- **passport-github2:** Permite que usuários façam login instantaneamente usando suas contas do GitHub.

Ao dividir a autenticação em strategies distintas, o Passport permite que sua aplicação cresça sem problemas. Se você começar hoje com um sistema de nome de usuário e senha e decidir adicionar "Login com Google" no próximo mês, não precisa reescrever todo o código de autenticação. Basta instalar a strategy do Google e conectá-la ao Passport.

Para usar o Passport em uma aplicação Express, você deve instalar o pacote core do Passport junto com a strategy específica que planeja usar. Para uma configuração padrão de nome de usuário e senha, execute o seguinte comando no seu terminal:

```bash
npm install passport passport-local
```

Depois de instalado, você precisa integrar o Passport na configuração da sua aplicação Express. Isso deve acontecer antes de definir suas rotas:

```js
const express = require('express');
const passport = require('passport');
const LocalStrategy = require('passport-local').Strategy;

const app = express();

// initialize Passport middleware
app.use(passport.initialize());

// configure Passport to use the Local Strategy
passport.use(new LocalStrategy((username, password, done) => {
    // your database look-up and password verification logic goes here
}));
```

O Passport se integra suavemente ao sistema padrão de rotas do Express como middleware. Quando um usuário envia suas credenciais para sua rota de login, o Passport intercepta a requisição, passa os dados para a strategy configurada e avalia o resultado.

```js
app.post('/login', passport.authenticate('local'), (req, res) => {
  // if this function runs, then authentication was successful
  res.send(`Welcome back, ${req.user.username}!`);
});
```

Se a strategy verifica o usuário com sucesso, o Passport executa duas tarefas críticas automaticamente:

1. **Gerenciamento de sessão:** Aciona funções auxiliares para armazenar um pequeno pedaço de dados do usuário dentro do cookie de sessão.

2. **Decoração da requisição:** Anexa o objeto do usuário autenticado diretamente à requisição recebida como `req.user`.

Uma vez que `req.user` está preenchido, qualquer middleware ou manipulador de rota subsequente na sua aplicação pode ver instantaneamente quem está logado e tomar decisões de autorização baseadas na identidade.

O Passport fornece uma base consistente, segura e pronta para produção para gerenciar identidades de usuários em suas aplicações Node.js.
