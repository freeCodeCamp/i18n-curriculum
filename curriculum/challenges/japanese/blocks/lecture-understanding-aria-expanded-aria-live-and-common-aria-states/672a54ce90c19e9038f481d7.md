# --interactive--

`aria-expanded` 属性はアクセシビリティの目的で使用され、コントロールが展開されているか折りたたまれているかを示します。これはメニュー、アコーディオン、その他のコンテンツの可視性を制御する折りたたみ可能なウィジェットと組み合わせて使われます。`aria-expanded` 属性は、コントロールが展開されている場合は `true` に、折りたたまれている場合は `false` に設定されます。

`aria-expanded` によって提供される情報は、スクリーンリーダーを使う人がコントロールの現在の状態（展開されているか折りたたまれているか）を理解するのに役立ちます。

`aria-expanded` 属性は、折りたたみ可能なウィジェットの可視性を切り替えるインタラクティブな要素に適用されます。例えば、ボタンが展開可能なメニューを切り替える場合、`aria-expanded` 属性はそのボタンに付けられます。

メニューが展開されているときは、`aria-expanded` 属性を `true` に設定する必要があります。例は以下の通りです。

```html
<button aria-expanded="true">Menu</button>
```

メニューが折りたたまれているときは、代わりに `false` に設定します。

```html
<button aria-expanded="false">Menu</button>
```

コントロール要素には常に `aria-expanded` 属性が `true` または `false` のいずれかに設定されている必要があります。例えば、ボタンがメニューの可視性を切り替える場合、`aria-expanded` のデフォルト値はメニューのデフォルトの可視性に基づいて設定されるべきです。

メニューがデフォルトで展開されている場合、`aria-expanded` は最初に `true` に設定されるべきです。メニューがデフォルトで折りたたまれている場合、`aria-expanded` は最初に `false` に設定されるべきです。

`aria-expanded` の値は、ユーザーが要素とやり取りするたびに JavaScript を使って動的に更新される必要があります。

さらに、`aria-controls` と `aria-owns` のプロパティは、`aria-expanded` と組み合わせて使うことで、コントロール要素と制御される要素の間にプログラム的な接続を確立できます。

まず `aria-controls` から始めましょう。`aria-expanded` と一緒に使うと、`aria-controls` 属性は制御される要素を指定するために使われます。例えば、ボタンがメニューとして機能するリストを展開または折りたたむ場合、`aria-controls` の値は制御される要素（この例ではメニューリスト）の `id` になります。

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<button aria-expanded="false" aria-controls="menu1">Menu</button>
<ul id="menu1" hidden>
  <li>...</li>
  <li>...</li>
</ul>

<script src="index.js"></script>
```

button {

```js
const button = document.querySelector('button');
const menu = document.getElementById(button.getAttribute('aria-controls'));

button.addEventListener('click', () => {
  const expanded = button.getAttribute('aria-expanded') === 'true';
  button.setAttribute('aria-expanded', String(!expanded));
  menu.hidden = expanded;
});

```

:::

リストがコントロールボタンの直後に続いていることに注目してください。このような展開可能なコントロールでは、展開されたコンテンツが DOM 内でそれを制御する要素の直後に配置されているのが最適です。これにより、スクリーンリーダー利用者が展開されたコンテンツを探す手間が省け、キーボードユーザーが展開されたコンテンツ内のインタラクティブなコントロールを簡単にナビゲートできます。

展開されたコンテンツをコントロール要素の直後に配置できない場合、`aria-owns` 属性を使うとアクセシビリティツリー上で仮想的にコントロールの後に移動させることができます。これにより、スクリーンリーダーなどの支援技術は、展開されたコンテンツが DOM 内でコントロールの直後にあるかのように扱えます。

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<button aria-owns="menu1" aria-expanded="true">Menu</button>
<main>
  <!-- an entire page worth of content --->
</main>
<ul id="menu1">
  <li>...</li>
  <li>...</li>
</ul>

<script src="index.js"></script>
```

```css

  color: white;
  padding: 8px 12px;
  border: none;
  border-radius: 4px;
  cursor: pointer;
  margin-bottom: 10px;
}

button:hover {
  background-color: #005ea2;
}

ul {
  list-style: none;
  padding: 0;
  margin: 8px 0;
  border: 1px solid #ccc;
  border-radius: 4px;
  width: 150px;
  background: #fff;
  position: absolute;
  z-index: 10;
}

li {
  padding: 8px;
  cursor: pointer;
}

}

```

```js
const button = document.querySelector('button');
const menu = document.getElementById(button.getAttribute('aria-owns'));

button.addEventListener('click', () => {
  const expanded = button.getAttribute('aria-expanded') === 'true';
  button.setAttribute('aria-expanded', String(!expanded));
  menu.hidden = expanded;
});

// Close the menu if user clicks outside
document.addEventListener('click', (e) => {
  if (!button.contains(e.target) && !menu.contains(e.target)) {
    button.setAttribute('aria-expanded', 'false');
    menu.hidden = true;
  }
});

```

:::

`aria-owns` 属性を使うことには欠点もあります。ほとんどのスクリーンリーダーは展開時に展開された要素の内容全体を自動的に読み上げるため、スクリーンリーダー利用者にとって不要な冗長さが生じます。また、タブ順は変わらないため、JavaScript でタブ順を管理しない限り、キーボードユーザーは展開されたコンテンツに到達する前にページ上の他のすべてのインタラクティブなコントロールをタブで通過しなければならない場合があります。

理想的には、展開可能なコンテンツはコントロール要素の後に配置し、`aria-owns` 属性はそれが不可能な最悪のケースでのみ使うべきです。使う場合は、幅広いスクリーンリーダーとブラウザで十分にテストして、すべての人にとってアクセシブルな実装であることを確認してください。

また、`aria-controls` または `aria-owns` のいずれかを使う場合は、コントロールが展開および折りたたまれるたびに `aria-expanded` の値を更新し続ける必要があることを忘れないでください。

`aria-expanded` 属性は、コントロールが展開されているか折りたたまれているかを示します。この情報はスクリーンリーダー利用者にとって重要で、メニューやアコーディオン、その他の類似した開閉ウィジェットの現在の状態を理解するのに役立ちます。

`aria-expanded` を正しく使うことで、誰にとっても直感的なユーザー体験を作り出せます。
