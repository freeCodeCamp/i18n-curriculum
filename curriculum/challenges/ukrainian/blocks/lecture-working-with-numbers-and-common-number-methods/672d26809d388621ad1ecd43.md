# --interactive--

У JavaScript `NaN` означає «не число». Це спеціальне значення, яке представляє неможливий або невизначений числовий результат. `NaN` — це властивість глобального об’єкта, і водночас його вважають типом числа в JavaScript, що спочатку може здатися нелогічним.

`NaN` зазвичай є результатом операцій, які мають повертати число, але не можуть дати осмислене числове значення. Наприклад:

:::interactive_editor

```js
let result = 0 / 0;
console.log(result); // NaN
```

:::

У цьому випадку ділення нуля на нуль математично невизначене, тому JavaScript повертає `NaN`. Однією з особливостей `NaN` є те, що воно не дорівнює нічому, навіть самому собі:

:::interactive_editor

```js
console.log(NaN === NaN); // false
```

:::

Ця унікальна поведінка ускладнює перевірку, чи є значення `NaN` за допомогою стандартних операторів порівняння. Щоб це вирішити, JavaScript надає функцію `isNaN()`. Властивість функції `isNaN()` використовується для визначення, чи є значення `NaN` чи ні. Однак важливо розуміти, як працює `isNaN()`, оскільки іноді вона може давати несподівані результати. Ось як поводиться `isNaN()`:

:::interactive_editor

```js
console.log(isNaN(NaN));       // true
console.log(isNaN(undefined)); // true
console.log(isNaN({}));        // true

console.log(isNaN(true));      // false
console.log(isNaN(null));      // false
console.log(isNaN(37));        // false

console.log(isNaN("37"));      // false: "37" is converted to 37
console.log(isNaN("37.37"));   // false: "37.37" is converted to 37.37
console.log(isNaN(""));        // false: empty string is converted to 0
console.log(isNaN(" "));       // false: string with a space is converted to 0

console.log(isNaN("blabla"));  // true: "blabla" is not a number
```

:::

Як бачите, `isNaN()` спочатку намагається перетворити параметр на число. Якщо це не вдається, вона повертає `true`. Така поведінка може призводити до несподіваних результатів, особливо при роботі з рядками, які можна перетворити на числа.

Через ці потенційні непослідовності, у ES6 (шосте видання JavaScript, випущене 2015 року) було введено `Number.isNaN()`. Цей метод не намагається перетворити параметр на число перед перевіркою. Він повертає `true` лише якщо значення точно дорівнює `NaN`:

:::interactive_editor

```js
console.log(Number.isNaN(NaN));        // true
console.log(Number.isNaN(Number.NaN)); // true
console.log(Number.isNaN(0 / 0));      // true

console.log(Number.isNaN("NaN"));      // false
console.log(Number.isNaN(undefined));  // false
console.log(Number.isNaN({}));         // false
console.log(Number.isNaN("blabla"));   // false
```

:::

`Number.isNaN()` забезпечує надійніший спосіб перевірки на `NaN`, особливо у випадках, коли перетворення типів може призвести до несподіваних результатів із глобальною функцією `isNaN()`. На практиці, коли працюєте з числовими операціями або вхідними даними користувача, які мають бути числами, часто потрібно перевіряти на `NaN`, щоб коректно обробляти помилки або несподівані вхідні дані. Наприклад:

:::interactive_editor

```js
let a = 0;
let b = 0;
let result = a / b;

if (Number.isNaN(result)) {
  result = "Error: Division resulted in NaN";
}

console.log(result); // "Error: Division resulted in NaN"
```

:::

У цьому прикладі ми використовуємо `Number.isNaN()`, щоб відловити випадки, коли операція ділення дає `NaN`, що дозволяє нам належним чином обробити цю ситуацію. Розуміння `NaN` і правильна перевірка на нього є ключовими для написання надійного JavaScript-коду, особливо при роботі з математичними операціями або парсингом вхідних даних користувача.
