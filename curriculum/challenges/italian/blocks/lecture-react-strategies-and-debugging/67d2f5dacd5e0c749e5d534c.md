# --description--

Il browser ha strumenti per sviluppatori integrati che puoi usare per eseguire il debug di HTML, CSS e JavaScript.

Tuttavia, non sono molto efficaci per trovare e risolvere bug nelle app React. Perciò il team di React ha sviluppato uno strumento chiamato "React Developer Tools" (noto anche come React DevTools) per ispezionare, eseguire il debug e profilare le app React.

React DevTools è disponibile come componente aggiuntivo per browser Chrome, Edge e Firefox. Se usi Chrome o Edge, vai al Chrome web store, cerca "React Developer Tools" e aggiungilo al tuo browser.

Se usi Firefox, vai alla pagina dei componenti aggiuntivi di Firefox, cerca lo strumento e aggiungilo al tuo browser.

Se usi Safari, puoi installare React DevTools da npm eseguendo `npm install -g react-devtools` oppure `yarn global add react-devtools`.

Dopo aver installato e abilitato React DevTools, se apri un'app React nel browser e poi apri gli strumenti per sviluppatori del browser, dovresti vedere due schede extra: Componenti e Profiler.

La scheda Componenti mostra ogni componente in un formato ad albero. Con essa puoi:

- Visualizzare la gerarchia dei componenti dell'app.
- Controllare e modificare props, stati e valori di contesto in tempo reale.
- Controllare il codice sorgente di ogni componente selezionato.
- Registrare i dati del componente nella console.
- Ispezionare gli elementi DOM del componente.

La scheda Profiler, invece, ti aiuta a registrare e analizzare le prestazioni dei componenti così puoi identificare i render inutili, vedere la durata dei commit e individuare cosa ottimizzare.

Ecco un'app semplice per mostrarti come ispezionare i componenti e le props e lo stato che hanno. È simile al codice usato in una lezione precedente sul prop drilling:

```jsx
import { useState } from "react";

export default function App() {
  const greeting = "Hello, Prop Drilling!";
  const response = "I'm not here to play!";

  return <Parent greeting={greeting} response={response} />;
}

const Parent = ({ greeting, response }) => {
  return <Child greeting={greeting} response={response} />;
};

const Child = ({ greeting, response }) => {
  return <Grandchild greeting={greeting} response={response} />;
};

const Grandchild = ({ greeting, response }) => {
  const [count, setCount] = useState(0);

  return (
    <>
      <h1>{greeting}</h1>
      <h2>{response}</h2>

      <button onClick={() => setCount(count + 1)}>Increase Count</button>
      <h2>Count: {count}</h2>
      <button onClick={() => setCount(count - 1)}>Decrease Count</button>
    </>
  );
};
```

Se guardi nella scheda Componenti di React DevTools, puoi vedere la vista ad albero dei componenti. Il componente `App` è in cima, seguito dai componenti `Parent`, `Child` e `Grandchild`.

Se selezioni uno di questi componenti, puoi vedere le props e lo stato in essi. Se selezioni il componente `Parent`, puoi vedere le props `greeting` e `response`, che sono rispettivamente `Hello, Prop Drilling!` e `I'm not here to play!`.

Puoi vedere le props e aggiornare lo stato in tempo reale, e modificarli se necessario. Per esempio, puoi selezionare il componente `Grandchild` e cambiare la prop `greeting` da `Hello, Prop Drilling!` a `Hello, Welcome to Prop Drilling!`, e vederlo riflesso immediatamente sulla pagina.

Per registrare i dati di un componente nella console, ispeziona gli elementi DOM corrispondenti e visualizza il codice sorgente del componente. Le icone in alto a destra ti permettono di farlo. Se selezioni il componente `Grandchild` e clicchi il pulsante `Log the component data to the console`, verranno registrate nella console le props, lo stato, gli hook, i nodi e altri dati.

Un bug comune che potresti incontrare in React si chiama props mismatch.

Per esempio, supponiamo che per il componente `Child` tu passi per errore `reply` come prop invece di `response`:

```jsx
const Child = ({ greeting, response }) => {
  return <Grandchild greeting={greeting} reply={response} />;
};
```

Ricorda che `Grandchild` si aspetta una prop `response`. Poiché il componente riceve una prop diversa, non può mostrare quel testo sulla pagina e aggiunge semplicemente un `h2` vuoto al DOM. Invece, vedrai solo l'elemento `h1` con il testo `Hello, Prop Drilling!`, insieme agli altri pulsanti e testi già presenti sulla pagina. L'elemento `h2` vuoto è ancora lì, ma poiché è vuoto, non lo vedi senza ispezionare il DOM.

Per risolvere questo, puoi ispezionare la progressione delle props dal componente `Parent` fino a `Child` e modificare direttamente il nome della prop. Se vai alla scheda Componenti, selezioni il componente `Child` e cambi la prop `reply` in `response`, vedrai l'elemento `h2` sulla pagina con il testo `I'm not here to play!`.
