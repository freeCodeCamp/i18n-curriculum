# --description--

백만 명의 사용자가 있는 바쁜 전화 시스템의 기록은 다음과 같습니다:

| RecNr | 발신자 | 수신자 |
|-------|--------|--------|
|   1   | 200007 | 100053 |
|   2   | 600183 | 500439 |
|   3   | 600863 | 701497 |
|  ...  |  ...   |  ...   |

기록 $n$에서 발신자 전화번호와 수신자 번호는 각각 $Caller(n) = S_{2n - 1}$ 와 $Called(n) = S_{2n}$ 이며, 여기서 ${S}_{1,2,3,\ldots}$ 는 "지연 피보나치 생성기"에서 나옵니다:

$1 ≤ k ≤ 55$ 일 때, $S_k = [100003 - 200003k + 300007{k}^3]\\;(\text{나머지 연산}\\;1000000)$

$56 ≤ k$ 일 때, $S_k = [S_{k - 24} + S_{k - 55}]\\;(\text{나머지 연산}\\;1000000)$

만약 $Caller(n) = Called(n)$ 이면 사용자가 잘못 걸었고 통화가 실패한 것으로 간주합니다; 그렇지 않으면 통화가 성공한 것입니다.

기록이 시작된 시점부터, 사용자 $X$ 와 $Y$ 가 서로 전화를 걸거나 받으면 친구라고 합니다. 마찬가지로, $X$ 가 $Y$ 의 친구이고 $Y$ 가 $Z$ 의 친구라면 $X$ 는 $Z$ 의 친구의 친구가 됩니다; 이와 같은 방식으로 더 긴 연결도 가능합니다.

총리의 전화번호는 524287입니다. 잘못 걸린 통화를 제외하고 성공한 통화가 몇 번 이루어진 후에, 총리를 포함한 사용자 중 99%가 총리의 친구이거나 친구의 친구 등이 될까요?
