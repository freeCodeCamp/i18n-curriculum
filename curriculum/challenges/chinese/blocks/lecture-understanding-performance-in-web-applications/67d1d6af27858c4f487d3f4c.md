# --description--

当我们谈论性能时，这个术语本身可能感觉有点主观，对吧？什么是好的性能？什么是不好的性能？

有没有一种标准的方法来衡量性能，还是完全取决于用户的个人偏好和感知？让我们来看一看。 

我们可以将网站的性能分为两类：感知性能和真实性能。

感知性能是用户如何感知网站的性能。它是用户根据响应速度和可靠性对网站进行的评估。这是一种主观的衡量标准，因此很难量化，但非常重要，因为用户体验决定了网站的成败。

相比之下，真实性能是网站的客观且可测量的性能。它通过页面装载时间、服务器响应时间和渲染时间等指标来衡量。这些测量受多个与网络和代码本身相关的因素影响。

即使是具有出色感知性能的网站，也可以进一步优化以获得更好的用户体验。

有几种技术可以改善感知性能。

通过在后台装载非必要资源尽可能减少初始装载时间是至关重要的。这种技术被称为`"lazy loading"`。第一次印象可以决定用户对你的网站的看法。

惰性装载也会影响你网站的真实性能。通过使用这种技术，你可以减少必须预先装载的内容量，从而使页面加载更快，这是真实的性能指标。

为用户交互提供快速的响应和反馈也非常重要。 

例如，当用户点击一个元素时立即显示一个加载指示器，可以帮助用户感受到与进程的连接和参与，使等待时间感觉更短——或者至少他们会感觉更投入。

保持用户积极参与你的网站对于改善他们的时间感知非常重要。即使其他资源，如图像，还未装载，文本一到达就显示出来，有助于从一开始就保持用户的参与度。

如果 `user` 正在等待一个长时间运行的 `Task` 完成，请保持他们对进度的了解并定期更新。

另一个建议是避免内容重排和内容跳动。例如，当广告或图像正在装载时，如果版型中尚未为这些新资源预留空间，网站可能会跳动或重新调整以为它们腾出空间。 

这些突发变化可能导致糟糕的用户体验，因为用户可能会感觉网站仍在 `load`。为避免这种情况，请提前规划并从一开始就为这些 `elements` 赋值空间。

如果你的网站有自定义字体，你还应该尽量减少字体装载延迟，因为这可能导致闪烁——或者在自定义字体装载时显示备用字体。一个建议是使用与自定义字体相似的备用字体，这样如果发生这种情况，变化会更加细微。

此外，请确保接口元素处于激活状态。用户应能够以最小的延迟与它们交互。

我们将在接下来的几节课中更详细地讲解这些技术。

虽然真实的性能很重要，但感知性能对用户体验有着巨大的影响。通过同时优化两者，你可以创建既快速又感觉快速的网站，打造流畅且引人入胜的用户体验。
