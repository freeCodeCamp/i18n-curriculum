# --description--

始める前に、CSSにおけるアットルールとは何かを知っておく必要があります。アットルールはブラウザーに指示を与える文です。メディアクエリ、キーフレーム、フォントフェイスなど、スタイルシートのさまざまな側面を定義するために使えます。メディアクエリやキーフレームのような概念は今後のレッスンで詳しく学びます。今回は`@font-face`アットルールについて学びます。

`@font-face`を使うと、フォントファイル、フォーマット、ウェイトやスタイルなどの重要なプロパティを指定してカスタムフォントを定義できます。これが基本的な構文です。`@font-face`の後に中括弧で囲まれたブロックが続きます。

```css
@font-face {
  /* Descriptors */
}
```

中括弧の中には、フォントフェイスをカスタマイズするためのディスクリプターを含める必要があります。よく使われるものをいくつか見てみましょう。font-familyディスクリプターは、スタイルシート全体でそのフォントを参照するために使う名前を指定します。例えば、次の`@font-face rule`を定義したとします。`font-family`ディスクリプターが定義され、その値は`MyCustomFont`です。

```css
@font-face {
  font-family: "MyCustomFont";
}
```

この場合、スタイルシート内でそのカスタムフォントファミリーを割り当てたいすべての要素に対して、その名前を使う必要があります。例えば、`font-family`の値として使います。

```css
body { 
  font-family: "MyCustomFont"; 
}
```

しかし、`@font-face`アットルールを有効にするには、`src`も指定する必要があります。これはフォントリソースへの参照を含みます。基本的には、カンマで区切られた外部参照やローカルにインストールされたフォントフェイス名のリストです。フォントリソースのフォーマットや技術に関するヒントも含めることができます。

以下のコードでは、`src`が定義されています。値はカンマで区切られ、読みやすくするために別々の行に配置されたURLのリストです。

```css
@font-face {
  font-family: "MyCustomFont"; 
  src: url("path/to/font.woff2"),
    url("path/to/font.woff"),
    url("path/to/font.otf");
}
```

`url()`関数を呼び出してファイルをスタイルシートに含めることができます。この場合、フォントファイルを含めています。ファイルパスは丸括弧と引用符の中に書き、ファイル拡張子も含める必要があります。読みやすくするために、各リソースを別の行に書くことができますが、最後の行はセミコロンで終わらせてください。

各フォントリソースに対してフォーマットを指定することもできます。これは任意です。ブラウザーにフォントフォーマットのヒントを与えます。フォーマットを省略すると、リソースはダウンロードされ、ダウンロード後にフォーマットが検出されます。フォーマットが無効な場合、リソースはダウンロードされません。

可能なフォントフォーマットには`collection`、`embedded-opentype`、`opentype`、`svg`、`truetype`、`woff`、`woff2`があります。

フォントフォーマットを指定した例を示します。特定のフォーマットを丸括弧と引用符の中に書く方法に注目してください。

```css
@font-face {
  font-family: "MyCustomFont"; 
  src: url("path/to/font.woff2") format("woff2"),
    url("path/to/font.otf") format("opentype"),
    url("path/to/font.woff") format("woff");
}
```

この例では、WOFF2フォーマット、OpenTypeフォーマット、WOFFフォーマットを指定しています。

woffは「Web Open Font Format」の略です。WOFFとWOFF2の違いはデータ圧縮に使われるアルゴリズムです。OpenTypeはMicrosoftとAdobeが開発したスケーラブルなコンピュータフォントのフォーマットで、フォントの追加機能にアクセスできます。主要なOSで広く使われています。

フォーマットの指定に加えて、フォントリソースの技術も指定できます。これも任意です。以下は2番目のフォントリソースの技術を指定した例です。

```css
@font-face {
  font-family: "MyCustomFont"; 
  src: url("path/to/font.woff2") format("woff2"),
    url("path/to/font.otf") format("opentype") tech(color-COLRv1),
    url("path/to/font.woff") format("woff");
}
```

これが`@font-face`ルールの基本です。このアットルールを使うと、独自のデザインのためにカスタムフォントを定義するフォントファイルを指定できます。
