# --description--

En las lecciones anteriores, aprendiste cómo trabajar con tipos primitivos como `string` y `boolean`. Pero TypeScript te permite crear nuevos tipos a partir de los existentes. Un ejemplo de esto sería el tipo unión.

Un tipo unión es un tipo que combina dos o más tipos.

Esta es la sintáxis básica:

```ts
type1 | type2 | type3
```

Veamos algunos ejemplos para entender mejor el concepto. 

Aquí hay una variable llamada `id` con un tipo unión para `string` o `number`:

```ts
let id: string | number = "user_783";
```

El valor puede actualizarse a un número que aún sea válido para el tipo unión:

```ts
let id: string | number = 1024;
```

Si intentaste asignar otro valor que no sea un `string` o `number`, entonces TypeScript mostrará un mensaje de error:

```ts
// Type 'string[]' is not assignable to type 'string | number'.
let id: string | number = ["Java", "Python"];
```

Al trabajar con tipos unión, es importante usar métodos que funcionen para todos los tipos, de lo contrario TypeScript generará un error como este:

```ts
function getId(id: string | number) {
  // Property 'toUpperCase' does not exist on type 'string | number'.
  return id.toUpperCase();
}
```

Hay una solución para esto que implica restringir la unión. Pero aprenderás más sobre eso, así como sobre cómo funcionan los tipos unión con alias de tipo en lecciones futuras.

Los tipos unión también son útiles cuando trabajas con valores `null` y `undefined`. 

Aquí hay un ejemplo donde una variable podría contener un valor de `string`, o podría ser `null` si aún no se ha establecido ningún valor:

```ts
let username: string | null;

// Type 'number' is not assignable to type 'string | null'.
username = 42;
```

En este caso, `username` puede ser un `string` o `null`, pero nada más. Podrías usar `null` cuando quieras representar intencionalmente la ausencia de un valor.

Aquí hay otro ejemplo usando `undefined`:

```ts
let score: number | undefined;

score = undefined; 

// Type 'string' is not assignable to type 'number | undefined'.
score = "high";
```

En este ejemplo, `score` puede ser un `number` o `undefined`. Podrías usar `undefined` cuando un valor aún no ha sido asignado.

Usar tipos unión con `null` y `undefined` deja claro cuándo un valor podría faltar, mientras que TypeScript sigue detectando asignaciones inválidas.
