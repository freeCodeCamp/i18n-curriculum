# --description--

많은 언어와 프레임워크가 파일과 폴더를 다루는 방법을 제공합니다. Node.js는 `fs` 또는 파일 시스템 모듈을 통해 이를 할 수 있게 해줍니다.

이 수업에서는 Node `fs` 모듈, 작동 방식, 그리고 이 모듈이 제공하는 메서드를 사용할 수 있는 세 가지 방법에 집중합니다.

Node `fs` 모듈은 파일과 폴더를 다루는 메서드를 제공합니다. 여기에는 열기와 닫기, 읽기와 쓰기, 삭제 작업이 포함됩니다.

`fs` 모듈은 Node.js 표준 모듈이므로, 환경에 Node.js가 설치되어 있으면 언제든 사용할 수 있습니다. 모듈을 사용하려면 이렇게 가져오세요:

```js
const fs = require("fs");
```

`fs` 모듈이 제공하는 유연성은 메서드를 동기식과 비동기식 모두로 사용할 수 있다는 점입니다. `fs` 모듈 메서드는 기본적으로 비동기이지만, 모든 메서드에 동기식 형태가 있습니다:

```js
fs.writeFile() // Asynchronous file writing
fs.writeFileSync() // Synchronous file writing

fs.readFile() // Asynchronous file reading
fs.readFileSync() // Synchronous file reading

fs.open() // Opens a file
fs.openAsBlob() // Opens as blob
fs.openSync() // Synchronous open
fs.opendir() // Opens directory
fs.opendirSync() // Synchronous directory open
```

여기서 끝나지 않습니다. 필요에 따라 메서드를 세 가지 방식으로 사용할 수 있습니다:

- 콜백과 함께, 예를 들어 `fs.writeFile()` (비동기)
- 프로미스와 함께, `async/await` 구문을 선호한다면, 예를 들어 `fs.promises.writeFile()`
- 동기식으로, 예를 들어 `fs.writeFileSync()`

다음은 메서드 비동기 사용의 기본 구문입니다:

```js
fs.writeFile("filePath", "content", "utf8", (err) => {
  if (err) {
    throw err;
  }
  console.log("File written to!");
});
```

프로미스 버전은 `fs/promises`에서 가져오거나 `promises`을 `fs` 가져오기 뒤에 연결할 수 있습니다. 구문은 다음과 같습니다:

```js
async function promisesExample() {
  try {
    await fs.promises.writeFile("filePath", "content", "utf8");
    console.log("File written to!");
  } catch (err) {
    console.error("Error:", err);
  }
}

promisesExample();
```

그리고 동기식 버전은 다음과 같습니다:

```js
try {
  fs.writeFileSync("filePath", "content", "utf8");
  console.log("File written to!");
} catch (err) {
  console.error("Error:", err);
}
```

메서드를 동기식으로 사용하는 것은 차단(blocking)임을 유의하세요. 즉, 프로그램이 멈추고 작업이 끝날 때까지 기다린 후에야 다음 코드 줄로 넘어갑니다.

간단한 스크립트, 작은 프로젝트, 일회성 작업에는 동기식 메서드가 괜찮을 수 있습니다. 하지만 실제 애플리케이션에서는 차단 I/O(입출력)가 문제입니다. Node가 파일 시스템 작업을 기다리는 동안 앱의 다른 부분이 멈출 수 있기 때문입니다. 이는 성능과 사용자 경험을 저하시킵니다.

그래서 비동기 메서드가 존재합니다. 이들은 Node가 백그라운드에서 파일 작업을 처리하는 동안 프로그램의 나머지 부분이 계속 실행되도록 합니다. 전통적으로는 콜백을 사용했지만, 앱이 커질수록 복잡해질 수 있습니다.

프로미스와 `async/await`가 이 문제를 해결합니다. 여전히 논블로킹이지만, 코드는 일반 동기식 코드처럼 읽히고 유지 관리가 훨씬 쉽습니다.

이 때문에 오늘날에는 프로미스 기반 접근법이 일반적으로 선호되며, 이 수업의 나머지 부분에서도 이를 사용할 것입니다.

`writeFile()` 메서드는 기존 파일에 쓰기 작업을 할 수 있습니다. 파일이 없으면 현재 디렉터리에 생성한 후 지정한 내용을 씁니다.

`writeFile()` 메서드를 사용하는 방법은 다음과 같습니다:

```js
const fs = require("fs/promises");

async function writeToFile() {
  try {
    await fs.writeFile(
      "article.md",
      "## Node `fs` Module: The Complete Guide",
      "utf8",
    );
    console.log("File written to!");
  } catch (err) {
    console.error("Error writing to file:", err);
  }
}

writeToFile(); // File written to!
```

그 후 article.md 파일의 내용은 다음과 같습니다:

```md
## Node `fs` Module: The Complete Guide
```

`appendFile()` 메서드는 기존 파일의 내용에 추가할 수 있습니다. 사용 방법은 다음과 같습니다:

```js
const fs = require("fs/promises");

async function appendToFile() {
  try {
    await fs.appendFile(
      "article.md",
      "\nIn this article, you will learn all there is to know about the Node fs module...",
      "utf8",
    );
    console.log("File appended to!");
  } catch (err) {
    console.log("Error appending to file:", err);
  }
}

appendToFile(); // File appended to!
```

`readFile()` 메서드는 파일의 내용을 확인할 수 있게 해줍니다:

```js
const fs = require("fs/promises");

async function readFileContent() {
  try {
    const fileContent = await fs.readFile("article.md", "utf8");
    console.log("File content:", fileContent);
  } catch (err) {
    console.log("File read successfully");
  }
}

readFileContent();

/*
File content: ## Node `fs` Module: The Complete Guide

In this article, you will learn all there is to know about the Node fs module...
*/
```

`utf8` 문자 인코딩을 지정하지 않으면 파일 내용을 버퍼로 받게 됩니다:

```js
async function readFileContent() {
  try {
    const fileContent = await fs.readFile("article.md");
    console.log("File content:", fileContent);
  } catch (err) {
    console.log("File read successfully");
  }
}

readFileContent();

/*
File content: <Buffer 23 23 20 4e 6f 64 65 20 60 66 73 60 20 4c 69 62 72 61 72 79 3a 20
 54 68 65 20 43 6f 6d 70 6c 65 74 65 20 47 75 69 64 65 0a 0a 49 6e 20 74 68 69 73 20 ... 
72 more bytes>
*/
```

다음 수업에서는 Node.js Buffer 모듈로 버퍼를 다루는 방법을 배우게 됩니다.

마지막으로 `unlink()` 메서드는 파일을 삭제할 수 있습니다:

```js
const fs = require("fs/promises");

async function deleteFile() {
  try {
    await fs.unlink("article.ts");
    console.log("File deleted successfully!");
  } catch (err) {
    console.error("Error deleting file:", err);
  }
}

deleteFile();
```

파일이 없으면 다음과 비슷한 오류 메시지가 나타납니다:

```bash
Error deleting file: Error: ENOENT: no such file or directory, unlink 'article.ts'
```

파일이 있으면 삭제되고, 설정한 성공 메시지를 받게 됩니다.
