# --description--

非常に単純化した形で、タンパク質は疎水性（H）と極性（P）の要素からなる文字列と考えられます。例えば HHPPHHHPHHPH のようなものです。

この問題では、タンパク質の向きが重要です。例えば HPP は PPH と区別されます。したがって、$n$ 個の要素からなるタンパク質は $2^n$ 通り存在します。

自然界でこれらの文字列が現れるときは、エネルギー的に有利なため、H-H の接触点の数ができるだけ多くなるように折りたたまれます。

その結果、H 要素は内側に集まり、P 要素は外側に配置される傾向があります。

自然のタンパク質はもちろん三次元で折りたたまれますが、ここでは<u>二次元</u>でのタンパク質の折りたたみのみを考えます。

下の図は、例のタンパク質が折りたたまれる2つの可能な方法を示しています（H-H の接触点は赤い点で示されています）。

<img alt="two possible ways to fold example protein" src="https://cdn.freecodecamp.org/curriculum/project-euler/protein-folding.gif" style="background-color: white; padding: 10px; display: block; margin-right: auto; margin-left: auto; margin-bottom: 1.2rem;">

左側の折りたたみは H-H の接触点が6つしかなく、自然には起こりません。一方、右側の折りたたみは H-H の接触点が9つあり、この文字列にとって最適です。

H と P の要素が文字列のどの位置にも同じ確率で現れると仮定すると、長さ8のランダムなタンパク質文字列の最適な折りたたみにおける平均の H-H 接触点数は $\frac{850}{2^8} = 3.3203125$ となります。

長さ15のランダムなタンパク質文字列の最適な折りたたみにおける平均の H-H 接触点数はいくつでしょうか？正確な結果が得られるまで必要な小数点以下の桁数で答えてください。
