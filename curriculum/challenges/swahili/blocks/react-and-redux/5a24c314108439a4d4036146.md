# --description--

Kitendakazi cha `mapDispatchToProps()` hutumika kutoa watengenezaji wa vitendo maalum kwa sehemu zako za React ili waweze kutuma vitendo dhidi ya hifadhi ya Redux. Ni sawa na muundo wa kitendakazi cha `mapStateToProps()` ulichokiandika katika changamoto iliyopita. Hurejesha kitu kinachoonyesha ramani ya vitendo vya dispatch kwa majina ya sifa, ambayo yanakuwa `props` za sehemu. Hata hivyo, badala ya kurejesha kipande cha `state`, kila sifa hurejesha kitendakazi kinachofanya wito wa `dispatch` kwa mtengenezaji wa kitendo na data yoyote muhimu ya kitendo. Una upatikanaji wa `dispatch` hii kwa sababu hupitishwa kwa `mapDispatchToProps()` kama kigezo unapoeleza kitendakazi, kama vile ulivyopitisha `state` kwa `mapStateToProps()`. Chini ya pazia, React Redux inatumia `store.dispatch()` ya Redux kufanya dispatch hizi kwa `mapDispatchToProps()`. Hii ni sawa na jinsi inavyotumia `store.subscribe()` kwa sehemu zinazopangwa kwa `state`.

Kwa mfano, una mtengenezaji wa kitendo cha `loginUser()` anayechukua `username` kama mzigo wa kitendo. Kitu kinachorejeshwa kutoka `mapDispatchToProps()` kwa mtengenezaji huyu wa kitendo kingekuwa kama ifuatavyo:

```jsx
{
  submitLoginUser: function(username) {
    dispatch(loginUser(username));
  }
}
```

# --instructions--

Mhariri wa msimbo unatoa mtengenezaji wa kitendo anayeitwa `addMessage()`. Andika kitendakazi cha `mapDispatchToProps()` kinachochukua `dispatch` kama hoja, kisha kirejeshe kitu. Kitu hicho kinapaswa kuwa na sifa ya `submitNewMessage` iliyowekwa kwa kitendakazi cha dispatch, ambacho kinachukua kigezo cha ujumbe mpya wa kuongeza wakati kinapotuma `addMessage()`.
