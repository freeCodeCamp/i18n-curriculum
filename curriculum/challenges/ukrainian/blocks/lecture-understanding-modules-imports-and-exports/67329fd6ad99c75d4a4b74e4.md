# --description--

У JavaScript модуль — це автономна одиниця коду, яка інкапсулює пов’язані функції, класи або змінні.

Уявіть модуль як будівельний блок вашого застосунку, подібно до розділу в книзі. Кожен модуль зосереджений на конкретній функціональності, що робить ваш код більш організованим, підтримуваним і повторно використовуваним. Модулі допомагають уникнути конфліктів імен і дозволяють структурувати застосунок у окремі, взаємопов’язані частини.

Концепція модулів у JavaScript розвивалась з часом, але найпоширенішим і підтримуваним підходом є система модулів ES6 (ECMAScript 2015). Ця система забезпечує стандартизований спосіб визначення та використання модулів у різних середовищах JavaScript.

Щоб створити модуль, ви пишете свій JavaScript-код у окремому файлі. Будь-які змінні, функції або класи, які ви хочете зробити доступними для інших частин застосунку, потрібно явно експортувати. Це можна зробити за допомогою ключового слова `export`.

Наприклад, припустимо, у вас є файл під назвою `math.js` з деякими математичними функціями:

```js
export function add(a, b) {
    return a + b;
}

export function subtract(a, b) {
    return a - b;
}

const PI = 3.14159;
export { PI };
```

У цьому прикладі ми експортуємо дві функції (`add` і `subtract`) та константу (`PI`). Ви можете експортувати стільки елементів, скільки потрібно, з одного модуля.

Щоб використати ці експортовані елементи в іншій частині застосунку, їх потрібно імпортувати. Це робиться за допомогою ключового слова `import`.

Припустимо, ви хочете використати ці математичні функції у файлі під назвою `app.js`:

```js
import { add, subtract, PI } from './math.js';

console.log(add(5, 3));        // Outputs: 8
console.log(subtract(10, 4));  // Outputs: 6
console.log(PI);               // Outputs: 3.14159
```

Тут ми імпортуємо конкретні функції та константу, які нам потрібні, з модуля `math.js`. Частина `'./math.js'` вказує JavaScript, де знайти файл модуля відносно поточного файлу.

Іноді ви можете захотіти імпортувати все, що експортує модуль. Це можна зробити за допомогою синтаксису зірочки (`*`):

```js
import * as Math from './math.js';

console.log(Math.add(5, 3));        // Outputs: 8
console.log(Math.subtract(10, 4));  // Outputs: 6
console.log(Math.PI);               // Outputs: 3.14159
```

У цьому випадку всі експорти з `math.js` імпортуються як властивості об’єкта з ім’ям `Math`.

Ще один поширений шаблон — мати експорт за замовчуванням у модулі. Зазвичай це використовується, коли модуль переважно експортує одну функцію. У модулі може бути лише один експорт за замовчуванням.

Ось як це виглядає:

```js
// In math.js
export default function multiply(a, b) {
    return a * b;
}

// In app.js
import multiply from './math.js';

console.log(multiply(4, 5));  // Outputs: 20
```

Зверніть увагу, що при імпорті експорту за замовчуванням не потрібно використовувати фігурні дужки, і імпорт можна назвати як завгодно.

Важливо зазначити, що щоб використовувати модулі ES6 у браузері, потрібно вказати `type` як `module` у вашому `script` тегу:

```html
<script type="module" src="app.js"></script>
```

Модулі надають потужний спосіб організувати та структурувати ваш JavaScript-код. Вони дозволяють розбивати застосунок на менші, керовані частини, сприяють повторному використанню коду і допомагають підтримувати чітке розділення обов’язків. Коли ви створюватимете більші застосунки, ви побачите, що модулі стануть незамінним інструментом у вашому наборі для розробки JavaScript.
