# --interactive--

Method chaining ist eine Technik, bei der Sie mehrere Methoden nacheinander aufrufen. Sie können method chaining bei vielen Werttypen in JavaScript verwenden, darunter Strings, Arrays und Objekte. Obwohl Strings primitive Werte sind, umhüllt JavaScript sie vorübergehend mit einem `String`-Objekt, wenn Sie eine String-Methode verwenden.

Schauen wir uns ein Beispiel mit verschiedenen String-Methoden an:

:::interactive_editor

```js
const result = "  Hello, World!  "
  .trim()
  .toLowerCase()
  .replace("world", "JavaScript");

console.log(result); // "hello, JavaScript!"
```

:::

In diesem Beispiel starten wir mit einem String und führen drei Operationen in Folge aus: Leerzeichen trimmen, in Kleinbuchstaben umwandeln und den String `world` durch `JavaScript` ersetzen. Jede Methode gibt einen neuen String zurück, der zum Ziel des nächsten Methodenaufrufs wird.

Method chaining kann die Lesbarkeit des Codes erheblich verbessern, wenn Sie mit komplexen Operationen arbeiten.

Betrachten Sie zum Beispiel dieses Beispiel mit den Methoden `filter`, `map` und `reduce`:

:::interactive_editor

```js
const transactions = [
  { amount: 100, type: "credit" },
  { amount: 20, type: "cash" },
  { amount: 150, type: "credit" },
  { amount: 50, type: "cash" },
  { amount: 75, type: "credit" }
];

const totalCreditWithBonus = transactions
  .filter((transaction) => transaction.type === "credit")
  .map((transaction) => transaction.amount * 1.1)
  .reduce((sum, amount) => sum + amount, 0);

console.log(totalCreditWithBonus); // 357.5
```

:::

In diesem Beispiel haben wir ein Array von Transaktionen, bei denen jedes Objekt einen Betrag und einen Typ (Kreditkarte oder Bargeld) hat.

Zuerst filtern wir die Transaktionen und erstellen ein neues Array nur mit Kreditkartentransaktionen. Dann verketten wir die `map`-Methode an das gefilterte Ergebnis und multiplizieren für jeden Transaktionsbetrag diesen mit `1.1`, was einen `10%`-Bonus darstellt.

Anschließend nehmen wir dieses Ergebnis und verketten die `reduce`-Methode, um alle Beträge zusammenzuzählen, was zu `357.5` führt.

Obwohl method chaining den Code kürzer und lesbarer machen kann, ist es wichtig, es mit Bedacht einzusetzen.

Sehr lange Verkettungen können schwer zu debuggen sein, da nicht sofort klar ist, welcher Schritt in der Kette ein Problem verursacht. Es ist oft eine gute Praxis, sehr lange Ketten in mehrere Schritte aufzuteilen, um die Übersicht zu verbessern und das Debugging zu erleichtern.

Sie können Methoden auch an einem Objekt verketten. In diesem Fall gibt jede Methode `this` zurück, was sich auf das aktuelle Objekt bezieht und es ermöglicht, dass die nächste Methode in der Kette darauf ausgeführt wird.

:::interactive_editor

```js
const calculator = {
  total: 0,
  add(n) {
    this.total += n;
    return this;
  },
  multiply(n) {
    this.total *= n;
    return this;
  },
  subtract(n) {
    this.total -= n;
    return this;
  },
  getResult() {
    return this.total;
  }
};

const result = calculator.add(5).multiply(2).subtract(3).getResult();
console.log(result); // 7
```

:::

So gibt jede Methode dasselbe Objekt zurück, sodass Sie Aufrufe nacheinander verketten können.
