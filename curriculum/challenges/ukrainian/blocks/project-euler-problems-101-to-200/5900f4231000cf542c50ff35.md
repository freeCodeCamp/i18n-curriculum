# --description--

Шифрування RSA базується на такій процедурі:

Згенерувати два різних простих числа `p` та `q`.
Обчислити `n=p*q` та `φ=(p-1)(q-1)`.
Знайти ціле число `e`, `1 < e < φ`, таке що `gcd(e,φ) = 1`

Повідомлення в цій системі — це число в інтервалі `[0,n-1]`.
Текст, який потрібно зашифрувати, потім якимось чином перетворюють на повідомлення (числа в інтервалі `[0,n-1]`).
Щоб зашифрувати текст, для кожного повідомлення `m`, обчислюють c=m<sup>e</sup> mod n.

Щоб розшифрувати текст, потрібна така процедура: обчислити `d`, таке що `ed=1 mod φ`, потім для кожного зашифрованого повідомлення `c`, обчислити m=c<sup>d</sup> mod n.

Існують значення `e` та `m`, для яких m<sup>e</sup> mod n = m.
Повідомлення `m`, для яких m<sup>e</sup> mod n=m, називають незашифрованими повідомленнями.

Проблема при виборі `e` полягає в тому, що не повинно бути забагато незашифрованих повідомлень.
Наприклад, нехай `p=19` та `q=37`.
Тоді `n=19*37=703` та `φ=18*36=648`.
Якщо вибрати `e=181`, то, хоча `gcd(181,648)=1`, виявляється, що всі можливі повідомлення
m `(0≤m≤n-1)` є незашифрованими при обчисленні m<sup>e</sup> mod n.
Для будь-якого дійсного вибору `e` існують деякі незашифровані повідомлення.
Важливо, щоб кількість незашифрованих повідомлень була мінімальною.

Для заданих `p` та `q` знайдіть суму всіх значень `e`, `1 < e < φ(p,q)` та `gcd(e,φ)=1`, так щоб кількість незашифрованих повідомлень для цього значення `e` була мінімальною.

# --solutions--

```js
function gcd(a, b) {
    if (b)
        return gcd(b, a % b);
    else
        return a;
}

function RSAEncryption(p, q) {
    let phi = (p - 1) * (q - 1);

    let best = Number.MAX_SAFE_INTEGER;
    let sum = 0;

    for (let e = 0; e < phi; ++e) {
        if (!(gcd(e, phi) == 1))
            continue;

        let msg = (gcd(p - 1, e - 1) + 1) * (gcd(q - 1, e - 1) + 1);

        if (best == msg) {
            sum += e;
        } else if (best > msg) {
            best = msg;
            sum = e;
        }
    }

    return sum;
}
```
