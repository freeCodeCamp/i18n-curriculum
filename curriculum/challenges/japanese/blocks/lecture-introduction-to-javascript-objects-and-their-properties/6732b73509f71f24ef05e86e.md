# --interactive--

JavaScriptを使うとき、ネストされたオブジェクトやオブジェクト内の配列を含む複雑なデータ構造に出会うことがよくあります。これらの構造は豊かで階層的なデータを表しますが、その中のデータにアクセスし操作する方法をしっかり理解する必要があります。ネストされた構造を効果的に操作する方法を見ていきましょう。

ネストされたオブジェクトからプロパティにアクセスするには、単純なオブジェクトのプロパティにアクセスするのと同様に、ドット記法やブラケット記法を使います。ただし、ネストされた構造の奥深くまで掘り下げるために、これらのアクセス方法を連鎖させる必要があります。

例えば、連絡先情報を持つ人物を表すネストされたオブジェクトを考えてみましょう。

```js
const person = {
  name: "Alice",
  age: 30,
  contact: {
    email: "alice@example.com",
    phone: {
      home: "123-456-7890",
      work: "098-765-4321"
    }
  }
};
```

`Alice`の勤務先電話番号にアクセスするには、次のようにプロパティアクセスを連鎖させます。

:::interactive_editor

```js
const person = {
  name: "Alice",
  age: 30,
  contact: {
    email: "alice@example.com",
    phone: {
      home: "123-456-7890",
      work: "098-765-4321"
    }
  }
};

console.log(person.contact.phone.work); // "098-765-4321"
```

:::

また、プロパティ名にスペースや特殊文字が含まれている場合や、変数を使ってプロパティにアクセスするときには、ブラケット記法が特に便利です。

:::interactive_editor

```js
const person = {
  name: "Alice",
  age: 30,
  contact: {
    email: "alice@example.com",
    phone: {
      home: "123-456-7890",
      work: "098-765-4321"
    }
  }
};

console.log(person['contact']['phone']['work']); // "098-765-4321"
```

:::

次に、オブジェクトのプロパティの一つが配列の値を持つ場合のデータへのアクセス方法を見てみましょう。こちらは住所の配列を含む`person`オブジェクトの修正版です。

```js
const person = {
  name: "Alice",
  age: 30,
  addresses: [
    { type: "home", street: "123 Main St", city: "Anytown" },
    { type: "work", street: "456 Market St", city: "Workville" }
  ]
};
```

`Alice`の勤務先住所の市にアクセスする例はこちらです。

:::interactive_editor

```js
const person = {
  name: "Alice",
  age: 30,
  addresses: [
    { type: "home", street: "123 Main St", city: "Anytown" },
    { type: "work", street: "456 Market St", city: "Workville" }
  ]
};

console.log(person.addresses[1].city); // "Workville"
```

:::

この例では、`person.addresses`が住所の配列を指します。その配列の2番目の住所にアクセスするには、ブラケット記法とインデックス`1`を使います。次に、その住所オブジェクトからドット記法で`city`にアクセスします。

ネストされたオブジェクトや配列のプロパティにアクセスする方法を理解することは、複雑なデータ構造を扱う際に不可欠です。今後のワークショップやラボで、これらのデータ構造を扱う練習をする機会があります。
