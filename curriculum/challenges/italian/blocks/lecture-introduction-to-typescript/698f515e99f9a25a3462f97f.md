# --description--

In questa lezione imparerai come lavorare con i tipi di funzione. Iniziamo con le annotazioni dei tipi dei parametri.

Ecco un esempio di una funzione di base:

```ts
function circleArea(radius) {
  return Math.PI * radius * radius;
}
```

Al momento, TypeScript mostrerebbe un messaggio di `Parameter 'radius' implicitly has an 'any' type`. Il tipo `any` indica che un valore può avere qualsiasi tipo. Imparerai di più su questo tipo in una lezione futura.

Per eliminare l'errore e tipizzare esplicitamente il parametro, puoi tipizzarlo così:

```ts
function circleArea(radius: number) {
  return Math.PI * radius * radius;
}
```

Se provi a chiamare la funzione `circleArea` con un tipo diverso da number, TypeScript mostrerà un errore come questo:

```ts
function circleArea(radius: number) {
  return Math.PI * radius * radius;
}

// Argument of type 'string' is not assignable to parameter of type 'number'
circleArea("3");
```

Se provi a fornire più argomenti di quanti la funzione si aspetta, TypeScript mostrerà un errore:

```ts
function circleArea(radius: number) {
  return Math.PI * radius * radius;
}

// Expected 1 arguments, but got 2.
circleArea(3, 4);
```

Se fornisci troppo pochi argomenti, anche TypeScript mostrerà un errore:

```ts
function circleArea(radius: number) {
  return Math.PI * radius * radius;
}

// Expected 1 arguments, but got 0.
// An argument for 'radius' was not provided.
circleArea();
```

Puoi rendere opzionali i parametri usando il simbolo `?` vicino al parametro così:

```ts
function area(radius: number, height?: number): number {
  const baseArea = Math.PI * radius * radius;

  if (height !== undefined) {
    const sideArea = 2 * Math.PI * radius * height;
    return baseArea + sideArea;
  }

  return baseArea;
}
```

Il controllo condizionale all'interno della funzione è presente nel caso in cui l'argomento `height` non venga fornito.

Se vuoi tipizzare i valori di ritorno, puoi aggiungere annotazioni del tipo di ritorno così:

```ts
function circleArea(radius: number): number {
  return Math.PI * radius * radius;
}
```

L'annotazione del tipo di ritorno va dopo la lista dei parametri. Simile alle annotazioni dei tipi delle variabili, la maggior parte delle volte TypeScript sarà abbastanza intelligente da inferire il tipo di ritorno. Quindi usa un'annotazione esplicita del tipo di ritorno quando vuoi rendere più chiara l'intenzione della funzione o prevenire modifiche involontarie al valore di ritorno.

Il prossimo esempio che vedremo riguarda le funzioni asincrone e le promise. Se la tua funzione deve restituire una promise, puoi usare il tipo `Promise` così:

```ts
async function getCityTemperature(city: string): Promise<number> {
  const response = await fetch(`https://api.example.com/weather?city=${city}`);
  const data = await response.json(); 
  return data.temperature;
}
```

In questa situazione, se provi a rimuovere il tipo di ritorno, TypeScript potrebbe inferire il ritorno come `Promise<any>`. Quindi questa sarebbe una buona situazione per tipizzare esplicitamente il valore di ritorno.

Finora abbiamo visto solo esempi di funzioni nominate. Ma come funzionano i tipi con le funzioni anonime?

Ecco un esempio che usa una funzione anonima come callback passata a un `forEach`:

```ts
const languages = ["JavaScript", "Python", "TypeScript"];

languages.forEach((lang) => {
  console.log(`${lang.toUpperCase()} has ${lang.length} characters`);
});
```

Il parametro `lang` non ha un'annotazione di tipo. Invece, TypeScript usa il typing contestuale, che significa che inferisce il tipo dal contesto in cui appare.
