# --description--

In Python sind Variablen wie eine beschriftete Box zum Speichern und Referenzieren von Daten verschiedener Typen. Um Variablen in Python zu deklarieren, weisen Sie einem Bezeichner mit dem Zuweisungsoperator (`=`) einen Wert zu. Sie müssen keine speziellen Schlüsselwörter wie `let` oder `const` in JavaScript oder `char` in C# verwenden.

In Python schreiben Sie einfach den Namen der Variable links, gefolgt vom Zuweisungsoperator, und rechts den Wert, den Sie der Variable zuweisen möchten. Hier ist ein Beispiel, wie man `name`- und `age`-Variablen deklariert:

```python
name = 'John Doe'
age = 25
```

Im obigen Beispiel hält die Variable `name` den Wert `'John Doe'`. Dieser Wert ist ein String, also eine Folge von Zeichen, die verwendet wird, um Text darzustellen. Strings werden mit einfachen oder doppelten Anführungszeichen geschrieben, zum Beispiel `'Hello'` oder `"Hello"`. In zukünftigen Lektionen lernen Sie mehr über die Arbeit mit Strings in Python.

Beim Benennen von Variablen in Python sollten Sie einige wichtige Regeln beachten:

- Variablennamen dürfen nur mit einem Buchstaben oder einem Unterstrich (`_`) beginnen, nicht mit einer Zahl.
- Variablennamen dürfen nur alphanumerische Zeichen (`a-z`, `A-Z`, `0-9`) und Unterstriche (`_`) enthalten.
- Variablennamen sind case-sensitive – `age`, `Age` und `AGE` gelten alle als unterschiedlich.
- Variablennamen dürfen keine der reservierten Schlüsselwörter von Python sein, wie `if`, `class` oder `def`.

Wenn Sie eine dieser Regeln verletzen, wirft Ihr Python-Programm einen `SyntaxError`:

```bash
    5variable_name = 5
     ^
SyntaxError: invalid syntax
```

Kommen wir nun zu einigen gängigen Benennungskonventionen für Variablen in Python.

Zuerst sollten Variablennamen in Kleinbuchstaben geschrieben werden, wobei einzelne Wörter durch einen Unterstrich getrennt sind. Das nennt man snake case:

```python
my_variable_name = 'freeCodeCamp'
```

Als Nächstes sollten Sie beschreibende Namen für Variablen verwenden. Wenn Sie zum Beispiel das Alter eines Benutzers als Variable speichern möchten, ist `user_age` besser als `age` oder eine Abkürzung wie `ua`:

```python
user_age = 30
```

So können Sie in einem großen Codebestand leicht den Zweck einer Variable anderen Teammitgliedern (oder Ihrem zukünftigen Ich) kommunizieren.

Eine weitere Konvention ist, keine Variablennamen mit nur einem Buchstaben zu verwenden. Das ist in Python zwar sehr verbreitet, sollte aber vermieden werden, da Variablennamen mit nur einem Buchstaben keinen Zweck oder keine Bedeutung vermitteln:

```python
x = 56 # What do you mean by x?
```

Das Pfund-Symbol (`#`) und der Text, der im obigen Beispiel folgt, werden Kommentar genannt. Vielleicht sind Ihnen Kommentare schon vertraut, deshalb gehen wir sie kurz durch und erklären, wie sie funktionieren.

In Python beginnen Kommentare mit einem Pfund-Symbol (`#`), und die Sprache ignoriert alles nach dem `#`-Symbol in dieser Zeile:

```python
# This is a single-line comment
```

Mehrzeilige Kommentare können durch aufeinanderfolgende einzeilige Kommentare erstellt werden:

```python
# This is a
# multi-line
# comment
```

Sie können Kommentare verwenden, um Ihren Code zu erklären, sich selbst Erinnerungen zu hinterlassen oder zu verdeutlichen, warum eine Zeile existiert. Kommentare sind besonders hilfreich, wenn Sie lernen oder im Team arbeiten.

Sie sollten jedoch keine Kommentare verwenden, um zu erklären, was Ihre Variablennamen bedeuten. Stattdessen sollten die Namen, die Sie für Ihre Variablen wählen, beschreibend sein und kommunizieren, wofür sie da sind, und die anderen oben genannten Benennungsregeln befolgen, um Syntaxfehler zu vermeiden.
