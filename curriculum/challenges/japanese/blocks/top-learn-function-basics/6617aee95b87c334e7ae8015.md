# --description--

関数が呼び出されたときに引数が提供されない場合、パラメータには `undefined` の値が代入されます。関数がこのケースに対応していないと、予期しない結果になることがあります。

```js
function favoriteAnimal(animal) {
  return animal + " is my favorite animal!"
}

console.log(favoriteAnimal()); // Output: undefined is my favorite animal!
```

上の例では、関数 `favoriteAnimal` が引数なしで呼び出されています。パラメータ `animal` には `undefined` の値が代入され、それが文字列 `" is my favorite animal!"` と連結されます。

値が提供されない場合は、デフォルトパラメータを利用できます。デフォルトパラメータを使うと、関数が呼び出されたときに値が渡されなかった場合に、パラメータにあらかじめ設定した値を代入できます。

```js
function favoriteAnimal(animal = 'Giant Panda') {
  return animal + " is my favorite animal!"
}
```

上の例では、パラメータ `animal` のデフォルト値が `'Giant Panda'` に設定されています。関数 `favoriteAnimal` が引数なしで呼び出されると、パラメータ `animal` には `'Giant Panda'` の値が代入されます。

## --answers--

引数が提供されない場合に関数がエラーをスローするようにします。

---

---

引数が渡されなかったことを示すために関数が `null` を返すようにします。

---

あらかじめ定義された値で関数を実行できるようにし、`undefined` に関連する問題を回避します。
