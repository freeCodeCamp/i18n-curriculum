# --description--

シリアライズとデシリアライズは認証に関して重要な概念です。オブジェクトをシリアライズするとは、その内容を小さな*キー*に変換し、それを元のオブジェクトにデシリアライズできるようにすることを意味します。これにより、ユーザー名やパスワードなどの認証データを新しいページのリクエストごとに送信しなくても、誰がサーバーと通信したかを知ることができます。

これを正しく設定するには、シリアライズ関数とデシリアライズ関数が必要です。Passportでは、これらを次のように作成できます。

```javascript
passport.serializeUser(cb);
passport.deserializeUser(cb);
```

`serializeUser`に渡されるコールバック関数は2つの引数で呼び出されます：完全なユーザーオブジェクトと、passportが使用するコールバックです。

このコールバックは2つの引数を期待します：エラーがあればそのエラー、そしてコールバックで返すべきユーザーを識別するための一意のキーです。ユーザーのオブジェクト内の`_id`を使用します。これはMongoDBによって生成されるため、一意であることが保証されています。

同様に、`deserializeUser`は2つの引数で呼び出されます：一意のキーとコールバック関数です。

このコールバックは2つの引数を期待します：エラーがあればそのエラー、そして完全なユーザーオブジェクトです。完全なユーザーオブジェクトを取得するには、以下のようにMongoの`_id`を検索するクエリを実行します。

```javascript
passport.serializeUser((user, done) => {
  done(null, user._id);
});

passport.deserializeUser((id, done) => {
  myDataBase.findOne({ _id: new ObjectID(id) }, (err, doc) => {
    done(null, null);
  });
});
```

上記の2つの関数をサーバーに追加してください。`ObjectID`クラスは`mongodb`パッケージから来ています。`mongodb@~3.6.0`はすでに依存関係として追加されています。このクラスは次のように宣言してください。

```javascript
const { ObjectID } = require('mongodb');
```

`deserializeUser`はデータベース接続を設定するまでエラーをスローします。ですので、今は`myDatabase.findOne`の呼び出しをコメントアウトし、`done(null, null)`のコールバック関数内で`deserializeUser`だけを呼び出してください。

正しくできたと思ったらページを提出してください。エラーが発生する場合は、<a href="https://forum.freecodecamp.org/t/advanced-node-and-express/567135#serialization-of-a-user-object-4" target="_blank" rel="noopener noreferrer nofollow">ここまで完成したプロジェクトを確認できます</a>。
