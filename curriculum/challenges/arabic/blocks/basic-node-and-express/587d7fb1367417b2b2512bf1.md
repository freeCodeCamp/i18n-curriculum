# --description--

بينما يقدم خادم HTML صفحات HTML، يقدم API بيانات. يسمح API <dfn>REST</dfn> (نقل الحالة التمثيلية) بتبادل البيانات بطريقة بسيطة، دون حاجة العملاء لمعرفة أي تفاصيل عن الخادم. يحتاج العميل فقط إلى معرفة مكان المورد (عنوان URL)، والفعل الذي يريد تنفيذه عليه (الفعل). يُستخدم الفعل GET عند جلب بعض المعلومات دون تعديل أي شيء. في الوقت الحالي، التنسيق المفضل لنقل المعلومات عبر الويب هو JSON. ببساطة، JSON هو طريقة ملائمة لتمثيل كائن JavaScript كسلسلة نصية، ليُرسل بسهولة.

لننشئ API بسيطًا بإنشاء مسار يستجيب بـ JSON على المسار `/json`. يمكنك فعل ذلك كالمعتاد باستخدام طريقة `app.get()`. داخل معالِج المسار، استخدم الطريقة `res.json()`، مرر كائنًا كمعلمة. تغلق هذه الطريقة سلسلة الطلب و الإستجابة، معيدة البيانات. خلف الكواليس، تُحوّل كائن JavaScript صالح إلى سلسلة نصية، ثم تضبط الأقسام العلوية المناسبة لتخبر المتصفح أنك تقدم JSON، وترسل البيانات مرة أخرى. الكائن الصالح له البنية المعتادة `{key: data}`. يمكن أن تكون `data` رقمًا، أو سلسلة نصية، أو كائنًا مسكنًا، أو مصفوفة. يمكن أن تكون `data` أيضًا متغيرًا أو نتيجة استدعاء دالة، وفي هذه الحالة تُقيّم قبل تحويلها إلى سلسلة نصية.

# --instructions--

قدّم الكائن `{"message": "Hello json"}` كاستجابة، بتنسيق JSON، لطلبات GET على مسار `/json`. ثم وجه متصفحك إلى `your-app-url/json`، يجب أن ترى الرسالة على الشاشة.

# --hints--

يجب أن تقدم نقطة النهاية `/json` كائن JSON `{"message": "Hello json"}`

```js
  const response = await fetch(code + '/json');
  if (!response.ok) {
    throw new Error(await response.text());
  }
  const data = await response.json();
  assert.equal(
    data.message,
    'Hello json',
    "The '/json' endpoint does not serve the right data"
  );
```
