# --interactive--

メソッドチェーンは、複数のメソッドを連続して呼び出す手法です。JavaScriptでは、文字列、配列、オブジェクトなど多くの値の種類でメソッドチェーンを使うことができます。文字列はプリミティブ値ですが、文字列メソッドを使うときにJavaScriptは一時的に`String`オブジェクトでラップします。

異なる文字列メソッドを使った例を見てみましょう。

:::interactive_editor

```js
const result = "  Hello, World!  "
  .trim()
  .toLowerCase()
  .replace("world", "JavaScript");

console.log(result); // "hello, JavaScript!"
```

:::

この例では、文字列から始めて3つの操作を順番に行います：空白をトリムし、小文字に変換し、文字列`world`を`JavaScript`に置き換えます。各メソッドは新しい文字列を返し、それが次のメソッド呼び出しの対象になります。

メソッドチェーンは複雑な操作を扱うときにコードの可読性を大幅に向上させることができます。

例えば、`filter`、`map`、`reduce`メソッドを使ったこの例を考えてみましょう。

:::interactive_editor

```js
const transactions = [
  { amount: 100, type: "credit" },
  { amount: 20, type: "cash" },
  { amount: 150, type: "credit" },
  { amount: 50, type: "cash" },
  { amount: 75, type: "credit" }
];

const totalCreditWithBonus = transactions
  .filter((transaction) => transaction.type === "credit")
  .map((transaction) => transaction.amount * 1.1)
  .reduce((sum, amount) => sum + amount, 0);

console.log(totalCreditWithBonus); // 357.5
```

:::

この例では、各オブジェクトが金額とクレジットカードまたは現金のタイプを持つ取引の配列があります。

まず取引をフィルターしてクレジットカード取引だけの新しい配列を作成します。次に、フィルター結果に`map`メソッドをチェーンし、各取引の金額に`1.1`を乗算します。これは`10%`ボーナスを表しています。

その結果に対して`reduce`メソッドをチェーンし、各金額を合計して`357.5`を得ます。

メソッドチェーンはコードを簡潔で読みやすくできますが、使いすぎには注意が必要です。

非常に長いチェーンはどのステップで問題が起きているかすぐにわからず、デバッグが難しくなります。明確さとデバッグのしやすさのために、長いチェーンは複数のステップに分けることをお勧めします。

オブジェクトのメソッドでもチェーンが可能です。この場合、各メソッドは現在のオブジェクトを指す`this`を返すため、チェーン内の次のメソッドがそのオブジェクト上で実行されます。

:::interactive_editor

```js
const calculator = {
  total: 0,
  add(n) {
    this.total += n;
    return this;
  },
  multiply(n) {
    this.total *= n;
    return this;
  },
  subtract(n) {
    this.total -= n;
    return this;
  },
  getResult() {
    return this.total;
  }
};

const result = calculator.add(5).multiply(2).subtract(3).getResult();
console.log(result); // 7
```

:::

このように、各メソッドは同じオブジェクトを返すので、連続してメソッド呼び出しをチェーンできます。
