## Nozioni di base su HTML

- **Ruolo di HTML**: HTML rappresenta il contenuto e la struttura della pagina web. 
- **Elementi HTML**: Gli elementi sono i mattoni fondamentali di un documento HTML. Rappresentano intestazioni, paragrafi, collegamenti, immagini e altro. La maggior parte degli elementi HTML è composta da un tag di apertura (`<elementName>`) e un tag di chiusura (`</elementName>`).

Ecco la sintassi di base:

```html
<elementName>Content goes here</elementName>
```

- **Elementi void**: Gli elementi void non possono avere contenuto e hanno solo un tag di apertura. Esempi includono gli elementi `img` e `meta`.

```html
<img>
<meta>
```

È comune vedere in alcuni codici una barra avanti `/` all’interno dell’elemento void. Entrambe queste forme sono accettabili:

```html
<img>
<img/>
```

- **Attributi**: Un attributo è un valore inserito all’interno del tag di apertura di un elemento HTML. Gli attributi forniscono informazioni aggiuntive sull’elemento o specificano come l’elemento deve comportarsi. Ecco la sintassi di base per un attributo:

```html
<element attribute="value"></element>
```

Un attributo booleano è un attributo che può essere presente o assente in un tag HTML. Se è presente, il valore è true, altrimenti è false. Esempi di attributi booleani includono `disabled`, `readonly` e `required`. 

- **Commenti**: I commenti si usano nella programmazione per lasciare note a te stesso e ad altri sviluppatori nel codice. Ecco la sintassi per un commento in HTML:

```html
<!--This is an HTML comment.-->
```

## Elementi HTML comuni

- **Elementi intestazione**: Ci sono sei elementi intestazione in HTML. Gli elementi intestazione da `h1` a `h6` servono a indicare l’importanza del contenuto che segue. Più basso è il numero, maggiore è l’importanza, quindi gli elementi `h2` hanno meno importanza rispetto agli elementi `h1`.

:::interactive_editor

```html
<h1>most important heading element</h1>
<h2>second most important heading element</h2>
<h3>third most important heading element</h3>
<h4>fourth most important heading element</h4>
<h5>fifth most important heading element</h5>
<h6>least important heading element</h6>
```

:::

- **Elementi paragrafo**: Si usano per i paragrafi in una pagina web.

:::interactive_editor

```html
<p>This is a paragraph element.</p>
```

:::

- **Elementi `img`**: L’elemento `img` serve per aggiungere immagini alla pagina web. L’attributo `src` specifica la posizione di quell’immagine. Per gli elementi immagine, è buona pratica includere un altro attributo chiamato attributo `alt`. Ecco un esempio di un elemento `img` con gli attributi `src` e `alt`:

:::interactive_editor

```html
<img src="https://cdn.freecodecamp.org/curriculum/cat-photo-app/lasagna.jpg" alt="A slice of lasagna on a plate.">
```

:::

- **Elemento `body`**: Questo elemento rappresenta il contenuto del documento HTML.

:::interactive_editor

```html
<body>
  <h1>CatPhotoApp</h1>
  <p>This is a paragraph element.</p>
</body>    
```

:::

- **Elementi `section`**: Questo elemento serve a suddividere il contenuto in sezioni più piccole.

:::interactive_editor

```html
<section>
  <h2>About Me</h2>
  <p>Hi, I am Jane Doe and I am a web developer.</p>
</section>
```

:::

- **Elementi `div`**: Questo elemento è un elemento HTML generico che non ha significato semantico. Si usa come contenitore generico per altri elementi HTML.

:::interactive_editor

```html
<div>
  <h1>I am a heading</h1>
  <p>I am a paragraph</p>
</div>
```

:::

- **Elementi àncora (`a`)**: Questi elementi si usano per applicare collegamenti a una pagina web. L’attributo `href` specifica dove deve andare il collegamento quando l’utente ci clicca sopra.

:::interactive_editor

```html
<a href="https://cdn.freecodecamp.org/curriculum/cat-photo-app/running-cats.jpg">cute cats</a>
```

:::

- **Elementi lista non ordinata (`ul`) e lista ordinata (`ol`)**: Per creare una lista puntata di elementi devi usare l’elemento `ul` con uno o più elementi `li` annidati dentro, così:

:::interactive_editor

```html
<ul>
  <li>catnip</li>
  <li>laser pointers</li>
  <li>lasagna</li>
</ul>
```

:::

Per creare una lista ordinata di elementi devi usare l’elemento `ol`:

:::interactive_editor

```html
<ol>
  <li>flea treatment</li>
  <li>thunder</li>
  <li>other cats</li>
</ol>
```

:::

- **Elemento enfasi (`em`)**: Si usa per mettere enfasi su un pezzo di testo. 

:::interactive_editor

```html
<p>Cats <em>love</em> lasagna.</p>  
```

:::

- **Elemento forte importanza (`strong`)**: Questo elemento serve a mettere forte enfasi sul testo per indicare urgenza e serietà. 

:::interactive_editor

```html
<p>
  <strong>Important:</strong> Before proceeding, make sure to wear your safety goggles. 
</p>
```

:::

- **Elementi `figure` e `figcaption`**: L’elemento `figure` serve a raggruppare contenuti come immagini e diagrammi. L’elemento `figcaption` rappresenta un sottotitolo per quel contenuto dentro l’elemento `figure`.

:::interactive_editor

```html
<figure>
  <img src="https://cdn.freecodecamp.org/curriculum/cat-photo-app/cats.jpg" alt="Two tabby kittens sleeping together on a couch.">
  <figcaption>Cats <strong>hate</strong> other cats.</figcaption>  
</figure>
```

:::

- **Elemento `main`**: Questo elemento rappresenta il contenuto principale di una pagina web.
- **Elemento `footer`**: Questo elemento si trova in fondo al documento HTML e di solito contiene informazioni sul copyright e altri collegamenti importanti della pagina.

:::interactive_editor

```html
<footer>
  <p>
    No Copyright - <a href="https://www.freecodecamp.org">freeCodeCamp.org</a>
  </p>
</footer>
```

:::

## Identificatori e raggruppamenti

- **ID**: Identificatori unici per elementi HTML. I nomi ID devono essere usati una sola volta per documento HTML. 

```html
<h1 id="title">Movie Review Page</h1>
```

I nomi ID non possono contenere spazi. Se il nome ID contiene più parole, puoi usare i trattini tra le parole così:

```html
<div id="red-box"></div>
```

- **Classi**: Le classi si usano per raggruppare elementi per definire lo stile e il comportamento. 

```html
<div class="box"></div>
```

A differenza degli ID, puoi riutilizzare lo stesso nome di classe in tutto il documento HTML. Il valore `class` può anche contenere spazi così:

```html
<div class="box red-box"></div>
<div class="box blue-box"></div>
```

## Caratteri speciali e collegamenti

- **Entità HTML**: Un’entità HTML, o riferimento a carattere, è un insieme di caratteri usati per rappresentare un carattere riservato in HTML. Esempi includono il simbolo ampersand (`&amp;`) e il simbolo minore di (`&lt;`).

```html
<p>This is an &lt;img /&gt; element</p>
```

- **Elemento `link`**: Questo elemento si usa per collegare risorse esterne come fogli di stile e icone del sito. Ecco la sintassi base per usare l’elemento `link` per un file CSS esterno: 

```html
<link rel="stylesheet" href="./styles.css" />
```

L’attributo `rel` specifica la relazione tra la risorsa collegata e il documento HTML. L’attributo `href` specifica la posizione URL della risorsa esterna.

- **Elemento `script`**: Questo elemento serve per incorporare codice eseguibile.

```html
<body>
  <script>
    alert("Welcome to freeCodeCamp");
  </script>
</body>
```

Anche se tecnicamente puoi scrivere tutto il codice JavaScript dentro i tag `script`, è considerata buona pratica collegare un file JavaScript esterno. Ecco un esempio di uso dell’elemento `script` per collegare un file JavaScript esterno:

```html
<script src="path-to-javascript-file.js"></script>
```

L’attributo `src` specifica la posizione di quel file JavaScript esterno.

## Codice boilerplate e codifica

- **Codice boilerplate HTML**: Questo è un codice boilerplate che include la struttura base e gli elementi essenziali di ogni documento HTML.

:::interactive_editor

```html
<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="utf-8" />
    <title>freeCodeCamp</title>
    <link rel="stylesheet" href="./styles.css" />
  </head>
  <body>
    <!--Headings, paragraphs, images, etc. go inside here-->
  </body>
</html>
```

:::

- **`DOCTYPE`**: Serve a indicare ai browser quale versione di HTML stai usando.
- **Elemento `html`**: Rappresenta l’elemento di livello superiore o radice di un documento HTML. Per specificare la lingua del documento, devi usare l’attributo `lang`.
- **Elemento `head`**: La sezione `head` contiene metadati importanti, informazioni dietro le quinte necessarie per browser e motori di ricerca.
- **Elementi `meta`**: Questi elementi rappresentano i metadati del sito. Contengono dettagli come la codifica dei caratteri e come siti come Twitter devono mostrare l’anteprima del collegamento della pagina e altro.
- **Elemento `title`**: Serve a impostare il testo che appare nella scheda o finestra del browser.
- **Codifica caratteri UTF-8**: UTF-8, o UCS Transformation Format 8, è una codifica caratteri standardizzata molto usata sul web. La proprietà `charset` si usa dentro un elemento `meta` per impostare la codifica caratteri su UTF-8.

## SEO e condivisione social

- **SEO**: L’ottimizzazione per i motori di ricerca è una pratica che ottimizza le pagine web per renderle più visibili e posizionarle più in alto nei motori di ricerca.  
- **Elemento meta (`description`)**: Serve a fornire una breve descrizione della pagina web che influisce sulla SEO.

```html
<meta
  name="description"
  content="Discover expert tips and techniques for gardening in small spaces, choosing the right plants, and maintaining a thriving garden."
/>
```

- **Tag Open Graph**: Il protocollo open graph ti permette di controllare come il contenuto del tuo sito appare su varie piattaforme social, come Facebook, LinkedIn e molte altre. 
Impostando queste proprietà open graph, puoi invogliare gli utenti a cliccare e interagire con il tuo contenuto. 
Puoi impostare queste proprietà tramite una raccolta di elementi `meta` dentro la sezione `head` del tuo HTML.
- **Proprietà `og:title`**: Serve a impostare il titolo che appare nei post sui social media.

```html
<meta content="freeCodeCamp.org" property="og:title" />
```

- **Proprietà `og:type`**: La proprietà `type` serve a rappresentare il tipo di contenuto condiviso sui social media. Esempi di contenuti includono articoli, siti web, video o musica.

```html
<meta property="og:type" content="website" />
``` 

- **Proprietà `og:image`**: Serve a impostare l’immagine mostrata nei post sui social media.

```html
<meta
  content="https://cdn.freecodecamp.org/platform/universal/fcc_meta_1920X1080-indigo.png"
  property="og:image"
/>
```

- **Proprietà `og:url`**: Serve a impostare l’URL su cui gli utenti cliccheranno nei post social.

```html
<meta property="og:url" content="https://www.freecodecamp.org" />
```

## Elementi multimediali e ottimizzazione

- **Elementi sostituiti**: Un elemento sostituito è un elemento il cui contenuto è determinato da una risorsa esterna invece che dal CSS stesso. Un esempio è un elemento `iframe`. `iframe` indica un frame in linea. È un elemento in linea usato per incorporare altri contenuti HTML direttamente nella pagina HTML. 

```html
<iframe src="https://www.example.com" title="Example Site"></iframe>
```

Puoi includere l’attributo `allowfullscreen` che permette all’utente di visualizzare l’iframe a schermo intero. 

```html
<iframe
  src="video-url"
  width="width-value"
  height="height-value"
  allowfullscreen
></iframe>
```

Per incorporare un video dentro un `iframe` puoi copiarlo direttamente da servizi video popolari come YouTube e Vimeo, oppure definirlo tu stesso con l’attributo `src` che punta all’URL di quel video. Ecco un esempio di incorporamento di un corso freeCodeCamp popolare da YouTube:

:::interactive_editor

```html
<h1>A freeCodeCamp YouTube Video Embedded with the iframe Element</h1>

<iframe
  width="560"
  height="315"
  src="https://www.youtube.com/embed/PkZNo7MFNFg?si=-UBVIUNM3csdeiWF"
  title="YouTube video player"
  allow="accelerometer; autoplay; clipboard-write; encrypted-media; gyroscope; picture-in-picture; web-share"
  referrerpolicy="strict-origin-when-cross-origin"
  allowfullscreen
></iframe>
```

:::

Ci sono altri elementi sostituiti, come `video` e `embed`. Alcuni elementi si comportano come elementi sostituiti in circostanze specifiche. Ecco un esempio di un elemento `input` con l’attributo `type` impostato su `image`:

```html
<input type="image" alt="Descriptive text goes here" src="example-img-url">
```

- **Ottimizzazione dei media**: Ci sono tre aspetti da considerare quando usi media, come immagini, nelle tue pagine web: la dimensione, il formato e la compressione. Un algoritmo di compressione serve a ridurre la dimensione di file o dati.
- **Formati immagine**: Due dei formati file più comuni sono PNG e JPG, ma non sono più i formati ideali per servire immagini. A meno che tu non debba supportare browser più vecchi, dovresti considerare formati più ottimizzati, come WEBP o AVIF.
- **Licenze immagini**: Un’immagine nel pubblico dominio non ha copyright e può essere usata liberamente senza restrizioni. Le immagini con licenza Creative Commons 0 sono considerate pubblico dominio. Alcune immagini possono essere rilasciate con licenze permissive, come Creative Commons o la licenza BSD usata da freeCodeCamp. 
- **SVG**: Scalable Vector Graphics traccia dati basati su percorsi ed equazioni per disegnare punti, linee e curve. Questo significa che un grafico vettoriale, come un SVG, può essere scalato a qualsiasi dimensione senza perdere qualità.

## Integrazione multimediale

- **Elementi `audio` e `video`**: Gli elementi `audio` e `video` ti permettono di aggiungere contenuti audio e video ai tuoi documenti HTML. L’elemento `audio` supporta formati audio popolari come mp3, wav e ogg. L’elemento `video` supporta formati mp4, ogg e webm.

```html
<audio src="CrystalizeThatInnerChild.mp3"></audio>
```

Se vuoi vedere il lettore audio sulla pagina, puoi aggiungere l’elemento `audio` con l’attributo `controls`:

:::interactive_editor

```html
<audio src="CrystalizeThatInnerChild.mp3" controls></audio>
```

:::

L’attributo `controls` permette agli utenti di gestire la riproduzione audio, inclusi volume, pausa e ripresa. L’attributo `controls` è un attributo booleano che puoi aggiungere a un elemento per abilitare i controlli di riproduzione integrati. Se omesso, non verranno mostrati controlli.

- **Attributo `loop`**: L’attributo `loop` è un attributo booleano che fa riprodurre l’audio in modo continuo.  

:::interactive_editor

```html
<audio
  src="https://cdn.freecodecamp.org/curriculum/js-music-player/can't-stay-down.mp3"
  loop
  controls
></audio>
```

:::

- **Attributo `muted`**: Quando è presente nell’elemento `audio`, l’attributo booleano `muted` fa partire l’audio in stato muto. 

:::interactive_editor

```html
<audio
  src="https://cdn.freecodecamp.org/curriculum/js-music-player/can't-stay-down.mp3"
  loop
  controls
  muted
></audio>
```

:::

- **Elemento `source`**: Per i tipi di file audio, ci sono differenze nel supporto tra browser. Per questo puoi usare elementi `source` dentro l’elemento `audio` e il browser selezionerà la prima sorgente che capisce. Ecco un esempio di uso di più elementi `source` per un elemento `audio`:

```html
<audio controls>
  <source src="audio.ogg" type="audio/ogg" />
  <source src="audio.wav" type="audio/wav" />
  <source src="audio.mp3" type="audio/mpeg" />
</audio>
```

Tutti gli attributi che abbiamo imparato finora sono supportati anche nell’elemento `video`. Ecco un esempio di uso di un elemento `video` con gli attributi `loop`, `controls` e `muted`:

:::interactive_editor

```html
<video
  src="https://archive.org/download/BigBuckBunny_124/Content/big_buck_bunny_720p_surround.mp4"
  loop
  controls
  muted
></video>
```

:::

- **Attributo `poster`**: Se vuoi mostrare un’immagine mentre il video si sta scaricando, puoi usare l’attributo `poster`. Questo attributo non è disponibile per gli elementi `audio` ed è unico per l’elemento `video`. 

:::interactive_editor

```html
<video
  src="https://archive.org/download/BigBuckBunny_124/Content/big_buck_bunny_720p_surround.mp4"
  loop
  controls
  muted
  poster="https://peach.blender.org/wp-content/uploads/title_anouncement.jpg?x11217"
  width="620"
></video>
```

:::

## Tipi di attributo target

- **Attributo `target`**: Questo attributo dice al browser dove aprire l’URL per l’elemento àncora. Ci sono quattro valori importanti per questo attributo: `_self`, `_blank`, `_parent` e `_top`. C’è un quinto valore, chiamato `_unfencedTop`, usato attualmente per l’API sperimentale `FencedFrame`. Probabilmente non avrai motivo di usarlo ancora.
- **Valore `_self`**: Questo è il valore predefinito per l’attributo `target`. Apre il collegamento nel contesto di navigazione corrente. Nella maggior parte dei casi sarà la scheda o finestra attuale.

```html
<a href="https://freecodecamp.org" target="_self">Visit freeCodeCamp</a>
```

- **Valore `_blank`**: Apre il collegamento in un nuovo contesto di navigazione. Tipicamente apre una nuova scheda, ma alcuni utenti possono configurare il browser per aprire una nuova finestra.

```html
<a href="https://freecodecamp.org" target="_blank">Visit freeCodeCamp</a>
```

- **Valore `_parent`**: Apre il collegamento nel genitore del contesto corrente. Per esempio, se il tuo sito ha un iframe, un valore `_parent` in quell’iframe aprirebbe nella scheda/finestra del sito, non nel frame incorporato.

```html
<a href="https://freecodecamp.org" target="_parent">Visit freeCodeCamp</a>
```

- **Valore `_top`**: Apre il collegamento nel contesto di navigazione più alto — pensa al “genitore del genitore”. È simile a `_parent`, ma il collegamento si aprirà sempre nella scheda/finestra completa del browser, anche per frame annidati.

```html
<a href="https://freecodecamp.org" target="_top">Visit freeCodeCamp</a>
```

## Percorsi assoluti vs relativi

- **Definizione di percorso**: Un percorso è una stringa che specifica la posizione di un file o directory in un file system. Nel web development, i percorsi permettono agli sviluppatori di collegare risorse come immagini, fogli di stile, script e altre pagine web.
- **Sintassi del percorso**: Ci sono tre sintassi chiave da conoscere. La prima è la barra, che può essere una barra rovesciata (`\`) o una barra avanti (`/`) a seconda del sistema operativo. La seconda è il punto singolo (`.`). Infine, abbiamo il doppio punto (`..`). La barra è il “separatore di percorso”. Serve a indicare una divisione tra nomi di cartelle o file. Un punto singolo indica la directory corrente, due punti indicano la directory genitore. 

```md
public/index.html
./favicon.ico
../src/index.css
```

- **Percorso assoluto**: Un percorso assoluto è un collegamento completo a una risorsa. Parte dalla directory radice, include tutte le altre directory e infine il nome file e l’estensione. La “directory radice” indica la cartella di livello più alto in una gerarchia. Un percorso assoluto include anche il protocollo — che può essere `http`, `https` o `file` — e il nome dominio se la risorsa è sul web. Ecco un esempio di percorso assoluto che collega al logo di freeCodeCamp:

:::interactive_editor

```html
<a href="https://design-style-guide.freecodecamp.org/img/fcc_secondary_small.svg">
  View fCC Logo
</a>
```

:::

- **Percorso relativo**: Un percorso relativo specifica la posizione di un file rispetto alla directory del file corrente. Non include protocollo o nome dominio, quindi è più corto e flessibile per collegamenti interni allo stesso sito. Ecco un esempio di collegamento alla pagina `about.html` dalla pagina `contact.html`, entrambe nella stessa cartella:

```html
<p>
  Read more on the
  <a href="about.html">About Page</a>
</p>
```

## Stati dei collegamenti

- **`:link`**: Questo è lo stato predefinito. Rappresenta un collegamento che l’utente non ha ancora visitato, cliccato o con cui non ha interagito. Puoi pensare a questo stato come base per tutti gli stili dei collegamenti nella pagina. Gli altri stati si basano su questo.
- **`:visited`**: Si applica quando un utente ha già visitato la pagina collegata. Per impostazione predefinita, questo rende il collegamento viola — ma puoi sfruttare il CSS per fornire un’indicazione visiva diversa all’utente. 
- **`:hover`**: Questo stato si applica quando un utente passa il cursore sopra un collegamento. Serve a dare maggiore attenzione al collegamento, per assicurarsi che l’utente intenda davvero cliccarlo.
- **`:focus`**: Questo stato si applica quando il collegamento ha il focus.
- **`:active`**: Questo stato si applica ai collegamenti che l’utente sta attivando. Di solito significa cliccare il collegamento con il pulsante principale del mouse, cioè il clic sinistro nella maggior parte dei casi.
