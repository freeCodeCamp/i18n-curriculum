# --interactive--

Katika JavaScript, kitu ni muundo wa data wa msingi unaokuwezesha kuhifadhi na kupanga data na utendaji unaohusiana.

Unaweza kufikiria kitu kama kontena linaloshikilia vipande mbalimbali vya taarifa, kama vile kabati la faili linavyoshikilia folda na nyaraka tofauti.

Vipande hivi vya taarifa huitwa sifa, na vina jina (au ufunguo) na thamani.

```js
const exampleObject = {
  propertyName: value,
}
```

Vitu ni vya aina nyingi sana na ni msingi wa JavaScript. Kwa kweli, karibu kila kitu katika JavaScript ni kitu au kinaweza kutendewa kama kitu. Hii ni pamoja na safu za data, vitendakazi, na hata aina za data za msingi kama mfuatano wa herufi na nambari zinapotumika kwa njia fulani.

Asili hii ya JavaScript inayozingatia vitu ni mojawapo ya sababu inavyokuwa lugha yenye kubadilika na yenye nguvu. Hebu tazama jinsi unavyoweza kuunda kitu:

```js
const person = {
  name: "Alice",
  age: 30,
  city: "New York"
};
```

Katika mfano huu, tumeunda kitu kinachoitwa `person` chenye sifa tatu: `name`, `age`, na `city`. Kila sifa ina jina na thamani, zimetenganishwa na nukta mbili.

Sasa, tuchunguze jinsi unavyoweza kupata sifa hizi. Kuna njia kuu mbili za kupata sifa za kitu katika JavaScript: uandishi wa nukta na uandishi wa mabano.

Uandishi wa nukta ni njia inayotumika zaidi na rahisi kupata sifa za kitu. Hii ndiyo sintaksia ya msingi ya uandishi wa nukta:

```js
objectName.propertyName
```

Hivi ndivyo unavyotumia uandishi wa nukta na kitu chetu cha `person`:

:::interactive_editor

```js
const person = {
  name: "Alice",
  age: 30,
  city: "New York"
};

console.log(person.name);  // Alice
console.log(person.age);   // 30
```

:::

Uandishi wa nukta ni mfupi na rahisi kusoma, na ndiyo chaguo linalopendekezwa unapojua jina halisi la sifa unayotaka kupata na jina hilo ni kitambulisho halali cha JavaScript (yaani halianzi na nambari wala halina herufi maalum au nafasi).

Kwa upande mwingine, uandishi wa mabano unakuwezesha kupata sifa za kitu kwa kutumia mfuatano wa herufi ndani ya mabano ya mraba. Hivi ndivyo unavyotumia uandishi wa mabano:

:::interactive_editor

```js
const person = {
  name: "Alice",
  age: 30,
  city: "New York"
};

console.log(person["name"]); // Alice
console.log(person["age"]); //  30
```

:::

Uandishi wa mabano ni rahisi zaidi kuliko uandishi wa nukta kwa sababu unakuwezesha kutumia majina ya sifa ambayo si vitambulisho halali vya JavaScript. Kwa mfano, kama ungekuwa na jina la sifa lenye nafasi au lianzi na nambari, ungetakiwa kutumia uandishi wa mabano:

:::interactive_editor

```js
const oddObject = {
  "1stProperty": "Hello",
  "property with spaces": "World"
};

console.log(oddObject["1stProperty"]);  // Hello
console.log(oddObject["property with spaces"]);  // World
```

:::

Faida nyingine ya uandishi wa mabano ni kwamba unakuwezesha kutumia vigezo kupata sifa kwa njia ya mabadiliko:

:::interactive_editor

```js
const person = {
  name: "Alice",
  age: 30,
  city: "Wonderland"
};

let propertyName = "city";
console.log(person[propertyName]); // Wonderland
```

:::

Urahisi huu hufanya uandishi wa mabano kuwa muhimu hasa wakati hujui jina halisi la sifa unapotengeneza msimbo, au unapotumia majina ya sifa yanayotoka kwa ingizo la mtumizi au chanzo kingine cha mabadiliko.

Ni vyema kutambua kwamba vitu katika JavaScript ni vyenye nguvu na vya aina nyingi. Haviwezi tu kuhifadhi thamani rahisi kama mfuatano wa herufi na nambari, bali pia safu za data au vitu vingine.

Kuelewa vitu na jinsi ya kufanya kazi navyo ni muhimu sana katika JavaScript kwa sababu hutumika sana katika lugha na katika maktaba na mifumo ya kazi ya JavaScript.

Unapoendelea kujifunza na kufanya kazi na JavaScript, utagundua kwamba kumiliki vitu kunafungua fursa nyingi za kuunda programu tata na zenye nguvu.
