# --description--

In Python, l'istruzione `raise` è uno strumento potente che ti permette di attivare manualmente eccezioni nel tuo codice. Ti dà il controllo su quando e come vengono generate le eccezioni, permettendoti di creare condizioni di errore personalizzate e far rispettare comportamenti specifici del programma.

L'istruzione `raise` viene usata per lanciare esplicitamente un'eccezione in qualsiasi punto del programma, consentendoti di segnalare che si è verificata una condizione di errore o che alcuni requisiti non sono stati soddisfatti.

L'istruzione `raise` di Python può essere usata in diversi modi per attivare eccezioni. Nella sua forma più semplice, puoi sollevare eccezioni integrate o creare messaggi di errore personalizzati. Ecco un esempio semplice:

```py
def check_age(age):
    if age < 0:
        raise ValueError('Age cannot be negative')
    return age

try:
    check_age(-5)
except ValueError as e:
    print(f'Error: {e}') # Error: Age cannot be negative
```

Qui puoi vedere che `raise` è la parola chiave che attiva un'eccezione.

In questo esempio, stiamo sollevando un `ValueError` con un messaggio personalizzato quando viene fornita un'età non valida.

L'istruzione `raise` può anche essere usata per rilanciare l'eccezione corrente, cosa particolarmente utile nella gestione delle eccezioni:

```py
def process_data(data):
    try:
        result = int(data)
        return result * 2
    except ValueError:
        print('Logging: Invalid data received')
        raise  # Re-raises the same ValueError

try:
    process_data('abc')
except ValueError:
    print('Handled at higher level')
```

Qui la parola chiave `raise` (senza argomenti), rilancia l'eccezione corrente che viene gestita.

Questo ti permette di registrare o eseguire operazioni di pulizia mantenendo la propagazione dell'errore lungo la catena delle chiamate.

Puoi creare e sollevare eccezioni personalizzate definendo le tue classi di eccezioni:

```py
class InsufficientFundsError(Exception):
    def __init__(self, balance, amount):
        self.balance = balance
        self.amount = amount
        super().__init__(f'Insufficient funds: ${balance} available, ${amount} requested')

def withdraw(balance, amount):
    if amount > balance:
        raise InsufficientFundsError(balance, amount)
    return balance - amount

try:
    new_balance = withdraw(100, 150)
except InsufficientFundsError as e:
    print(f'Transaction failed: {e}')
```

Qui puoi vedere che le classi di eccezioni personalizzate ereditano da `Exception` o dalle sue sottoclassi.

Imparerai di più su classi ed ereditarietà nelle lezioni future. Per ora, sappi che questo è un modo per creare le tue eccezioni con logica personalizzata.

L'istruzione `raise` può anche essere usata con la parola chiave `from` per concatenare eccezioni, mostrando la relazione tra diversi errori:

```py
def parse_config(filename):
    try:
        with open(filename, 'r') as file:
            data = file.read()
            return int(data)
    except FileNotFoundError:
        raise ValueError('Configuration file is missing') from None
    except ValueError as e:
        raise ValueError('Invalid configuration format') from e

config = parse_config('config.txt')
```

Qui puoi vedere che `raise ... from None` sopprime il contesto dell'eccezione originale:

```bash
Traceback (most recent call last):
  File "main.py", line 12, in <module>
    config = parse_config('config.txt')
             ^^^^^^^^^^^^^^^^^^^^^^^^^^
  File "main.py", line 7, in parse_config
    raise ValueError('Configuration file is missing') from None
ValueError: Configuration file is missing
```

E `raise ... from e` concatena la nuova eccezione a quella originale, preservando la traccia dell'errore.

```bash
Traceback (most recent call last):
  File "main.py", line 5, in parse_config
    return int(data)
           ^^^^^^^^^
ValueError: invalid literal for int() with base 10: ''

The above exception was the direct cause of the following exception:

Traceback (most recent call last):
  File "main.py", line 12, in <module>
    config = parse_config('config.txt')
             ^^^^^^^^^^^^^^^^^^^^^^^^^^
  File "main.py", line 9, in parse_config
    raise ValueError('Invalid configuration format') from e
ValueError: Invalid configuration format
```

Puoi anche sollevare eccezioni in modo condizionale usando istruzioni `assert`, che sono essenzialmente una forma abbreviata di `raise` con `AssertionError`:

```py
def calculate_square_root(number):
    assert number >= 0, 'Cannot calculate square root of negative number'
    return number ** 0.5

try:
    result = calculate_square_root(-4)
except AssertionError as e:
    print(f'Assertion failed: {e}')
```

L'istruzione `raise` è essenziale per creare applicazioni robuste dove devi far rispettare regole di business, convalidare input e fornire messaggi di errore significativi. Usando strategicamente `raise`, puoi rendere il tuo codice più prevedibile e più facile da eseguire il debug, offrendo agli utenti un feedback chiaro su cosa è andato storto.
