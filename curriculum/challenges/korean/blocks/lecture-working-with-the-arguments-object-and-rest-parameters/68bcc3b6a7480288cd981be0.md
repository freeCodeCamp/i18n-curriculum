# --interactive--

이전 수업과 워크숍에서 기억하셨겠지만, 여러 매개변수를 가진 함수를 만들고 그 함수에 인수를 전달해 호출할 수 있습니다.

다음은 복습입니다:

```js
// function definition
function getSum(num1, num2) {
  return num1 + num2;
}

// function call
getSum(3, 4); // 7
```

그런데 정의된 매개변수보다 더 많은 인수로 함수를 호출하면 어떻게 될까요?

:::interactive_editor

```js
// function definition
function getSum(num1, num2) {
  return num1 + num2;
}

// function call with extra argument
console.log(getSum(3, 4, 5)); // 7
```

:::

JavaScript는 이 경우 오류를 발생시키지 않습니다. 대신 추가 인수를 무시하고 `3`와 `4`를 더할 뿐입니다. 가변 인수를 받는 함수는 <dfn>가변 함수</dfn>라고 합니다.

가변 함수 작업 시 `arguments` 객체를 활용할 수 있습니다. 이 배열과 유사한 객체는 함수에 전달된 인수 값을 담고 있습니다.

예를 들어:

:::interactive_editor

```js
function logArgs() {
  for (const arg of arguments) {
    console.log(arg);
  }
}

logArgs(1, 2, 3);
// result:
// 1
// 2
// 3

logArgs("example"); // "example"
```

:::

`arguments` 객체가 배열과 유사하므로 특정 인덱스의 인수에 이렇게 접근할 수 있습니다:

:::interactive_editor

```js
function getArg() {
  return arguments[1];
}

console.log(getArg(2, 4, 6)); // 4
```

:::

또한 `length` 속성을 사용해 함수가 호출된 인수의 개수를 이렇게 알 수 있습니다:

:::interactive_editor

```js
function getArgs() {
  return arguments.length;
}

console.log(getArgs("Example")); // 1
console.log(getArgs("Another", "Example")); // 2
```

:::

`arguments` 객체가 실제 배열처럼 보이지만 `Array` 메서드인 `includes`이나 `push` 같은 내장 메서드는 없습니다. 이런 메서드를 사용하려면 `arguments` 객체를 `slice`, `Array.from()` 또는 전개 연산자 같은 방법으로 실제 배열로 변환해야 합니다:

:::interactive_editor

```js
function hasCat() {
  return [...arguments].includes("cat");
}

console.log(hasCat("dog", "chicken", "cat")); // true
console.log(hasCat("dog", "chicken", "horse")); // false
```

:::

가변 함수에 `arguments` 객체를 사용하는 것도 가능하지만, 최신 JavaScript 애플리케이션에서는 보통 REST 매개변수 구문을 사용합니다. 이 부분은 앞으로 수업에서 더 배우게 됩니다.
