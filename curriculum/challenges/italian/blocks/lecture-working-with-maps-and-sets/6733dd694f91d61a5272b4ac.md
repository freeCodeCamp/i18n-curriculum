# --description--

In JavaScript, un `Map` è un oggetto integrato che archivia coppie chiave-valore, simile a un oggetto. Tuttavia, si differenzia dagli oggetti JavaScript standard perché permette chiavi di qualsiasi tipo, inclusi oggetti e funzioni.

Una `WeakMap` è una collezione di coppie chiave-valore, simile a una `Map`, ma usa riferimenti deboli per le sue chiavi. Le chiavi devono essere oggetti, mentre i valori possono essere di qualsiasi tipo.

Pensa alla relazione tra una `WeakMap` e una `Map` come simile alla relazione tra una `WeakSet` e una `Set`, come hai imparato nella lezione precedente.

Per creare una `Map`, usi il costruttore `Map()` preceduto dalla parola chiave `new`:

```js
const myFirstMap = new Map();
```

Puoi inizializzare la Map con dei valori:

```js
const myTreesMap = new Map([
 [{ type: 'deciduous' }, 'Maple tree'],
 [['forest', 'grove'], 'Pine tree'],
 [42, 'Oak tree'],
 [true, 'Birch tree'],
 [function() { return 'I am a function key'; }, 'Willow tree'],
]);
```

Se non hai inizializzato la `Map` con valori, puoi usare il metodo `set()` per aggiungerli:

```js
const myTreesMap = new Map();
myTreesMap.set({ type: 'deciduous' }, 'Maple tree');
myTreesMap.set([1, 2], 'Pine tree');
myTreesMap.set(42, 'Oak tree');
myTreesMap.set(true, 'Birch tree');
myTreesMap.set(function() { return "I'm a function key"; }, 'Willow tree');

console.log(myTreesMap);
```

Altri metodi che puoi usare per lavorare con una `Map` sono:

- `get(key)` per recuperare il valore associato alla `key` specificata.
- `has(key)` per verificare se una `key` esiste nella `Map`.
- `delete(key)` per rimuovere una coppia chiave-valore dalla `Map`.
- `clear()` per rimuovere tutte le coppie chiave-valore.
- `entries()` per controllare gli elementi della `Map` (restituisce gli elementi in una `MapIterator`).
- `forEach()` per iterare sugli elementi della `Map`.
- `size` per indicare il numero di coppie chiave-valore nella `Map`.
    

Esiste un costruttore `WeakMap()` che puoi usare per creare una `WeakMap`:

```js
const myFirstWeakMap = new WeakMap();
```

I metodi `set()`, `get()`, `has()` e `delete()` sono tutti disponibili anche per l’uso con una `WeakMap`. Per esempio, ecco come puoi assegnare elementi alla `Map` con il metodo `set()`:

```js
const myTreeWeakMap = new WeakMap();

myTreeWeakMap.set({ id: 1 }, 'Maple tree');
myTreeWeakMap.set({ id: 2 }, 'Pine tree');
myTreeWeakMap.set({ id: 3 }, 'Oak tree');
myTreeWeakMap.set({ id: 4 }, 'Birch tree');
myTreeWeakMap.set({ id: 5 }, 'Willow tree');

console.log(myTreeWeakMap);
```

Ecco come appare una `WeakMap` nella console:

```js
/*
WeakMap {{…} => 'Willow tree', {…} => 'Maple tree', {…} => 'Pine tree', {…} => 'Oak tree'}
  [[Entries]]
    No properties
  [[Prototype]]: WeakMap
*/
```

Ecco le differenze tra una `Map` e una `WeakMap`:

| **Funzionalità** | **Map** | **WeakMap** |
| --- | --- | --- |
| **Tipo di chiave** | Le chiavi possono essere di qualsiasi tipo di dato, incluse stringhe, numeri, oggetti o anche funzioni. | Le chiavi devono essere oggetti. |
| **Caso d’uso** | Usa una `Map` quando devi associare dati a qualsiasi tipo di chiave. | Usa una `WeakMap` quando devi associare dati solo a oggetti. |
| **Iterazione** | Puoi iterare su una `Map` usando `forEach()`, `keys()`, `values()` o `entries()`. | Una `WeakMap` non è iterabile. |
| **Proprietà size** | `Map` ha una proprietà `size` per ottenere il numero di coppie chiave-valore. | `WeakMap` non ha una proprietà `size`. |

## --text--

Quale di queste differenze distingue una `Map` da una `WeakMap` in JavaScript?

## --answers--

Una `Map` permette solo oggetti come chiavi, mentre una `WeakMap` permette chiavi di qualsiasi tipo.
