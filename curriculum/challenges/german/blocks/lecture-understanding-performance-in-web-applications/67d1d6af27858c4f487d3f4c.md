# --description--

Wenn wir über Leistung sprechen, kann der Begriff selbst etwas subjektiv wirken, oder? Was ist gute Leistung? Was ist schlechte Leistung?

Gibt es eine standardisierte Methode, Leistung zu messen, oder hängt das komplett von den persönlichen Vorlieben und der Wahrnehmung des Nutzers ab? Schauen wir mal.

Wir können die Leistung einer Website in zwei verschiedene Kategorien einteilen: wahrgenommene Leistung und reale Leistung.

Die wahrgenommene Leistung beschreibt, wie Nutzer die Leistung einer Website wahrnehmen. Es geht darum, wie sie sie in Bezug auf Reaktionsfähigkeit und Zuverlässigkeit bewerten. Das ist eine subjektive Messung, daher schwer zu quantifizieren, aber sehr wichtig, denn die Nutzererfahrung bestimmt den Erfolg oder Misserfolg einer Website.

Im Gegensatz dazu ist die reale Leistung die objektive und messbare Leistung der Website. Sie wird mit Metriken wie Seitenladezeit, Server-Antwortzeit und Renderzeit gemessen. Diese Messwerte werden von verschiedenen Faktoren beeinflusst, die mit dem Netzwerk und dem Code selbst zusammenhängen.

Selbst eine Website mit ausgezeichneter wahrgenommener Leistung kann noch weiter optimiert werden, um die Nutzererfahrung noch besser zu machen.

Es gibt verschiedene Techniken, um die wahrgenommene Leistung zu verbessern.

Die anfängliche Ladezeit so weit wie möglich zu reduzieren, indem nicht essentielle Ressourcen im Hintergrund geladen werden, ist entscheidend. Diese Technik nennt man „lazy loading“. Dieser erste Eindruck kann bestimmen, was Nutzer von Ihrer Website halten.

Lazy loading wirkt sich auch auf die reale Leistung Ihrer Website aus. Durch diese Technik können Sie die Menge an Inhalten reduzieren, die sofort geladen werden müssen, sodass die Seite schneller lädt – was eine reale Leistungsmetrik ist.

Es ist auch sehr wichtig, schnelle Antworten und Feedback auf Nutzerinteraktionen zu geben.

Zum Beispiel kann das Anzeigen eines Ladeindikators für einen lang laufenden Prozess, sobald der Nutzer auf ein Element klickt, dem Nutzer helfen, sich verbunden und eingebunden zu fühlen, wodurch die Wartezeit kürzer erscheint – oder zumindest fühlt er sich mehr involviert.

Nutzer aktiv mit Ihrer Website zu beschäftigen, ist sehr wichtig, um ihre Zeitwahrnehmung zu verbessern. Das Anzeigen von Text, sobald er eintrifft, hilft dabei, Nutzer von Anfang an zu binden, auch wenn andere Ressourcen wie Bilder noch nicht geladen sind.

Wenn ein Nutzer auf die Fertigstellung einer lang laufenden Aufgabe wartet, halten Sie ihn über den Fortschritt auf dem Laufenden und aktualisieren Sie diesen regelmäßig.

Ein weiterer Tipp ist, Inhaltsverschiebungen und springende Inhalte zu vermeiden. Wenn zum Beispiel Anzeigen oder Bilder geladen werden, kann die Website springen oder sich neu anordnen, um Platz für diese neuen Ressourcen zu schaffen, falls diese noch keinen Platz im Layout haben.

Diese plötzlichen Änderungen können zu einer schlechten Nutzererfahrung führen, weil Nutzer das Gefühl haben, die Website lädt noch. Um das zu vermeiden, planen Sie im Voraus und weisen Sie diesen Elementen von Anfang an Platz zu.

Wenn Ihre Website benutzerdefinierte Schriftarten verwendet, sollten Sie auch versuchen, Ladeverzögerungen der Schriftart zu minimieren, da dies zu Flackern führen kann – oder dazu, dass die Fallback-Schriftart angezeigt wird, während die benutzerdefinierte Schriftart geladen wird. Ein Vorschlag ist, eine Fallback-Schriftart zu verwenden, die der benutzerdefinierten Schriftart ähnlich ist, sodass die Änderung in diesem Fall viel subtiler ausfällt.

Stellen Sie außerdem sicher, dass die Interface-Elemente aktiv sind. Der Nutzer sollte mit ihnen mit minimaler Verzögerung interagieren können.

Wir werden diese Techniken in den nächsten Lektionen noch genauer betrachten.

Während die reale Leistung wichtig ist, kann die wahrgenommene Leistung einen enormen Einfluss auf die Nutzererfahrung haben. Wenn Sie beide optimieren, können Sie Websites erstellen, die schneller sind und sich auch so anfühlen – für eine flüssige und ansprechende Nutzererfahrung.
