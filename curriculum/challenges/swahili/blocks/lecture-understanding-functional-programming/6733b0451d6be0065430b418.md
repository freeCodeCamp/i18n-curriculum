# --interactive--

Programu utendaji ni kuhusu kuandika msimbo ambao ni safi, unaotabirika, na rahisi kujaribu na kutafuta makosa. Dhana mbili muhimu tutazozichunguza leo ni vitendakazi safi na kuepuka athari za pembeni.

Tuanze na vitendakazi safi.

Kitendakazi safi ni kitendakazi ambacho, kikipewa ingizo sawa, daima hurejesha matokeo sawa, na hakibadilishi chochote nje ya nafsi yake.

Hapa kuna mfano wa kitendakazi safi:

```js
function add(a, b) {
  return a + b;
}
```

Kitendakazi hiki ni safi kwa sababu daima hurejesha matokeo sawa kwa ingizo sawa, na hakibadilishi chochote nje ya nafsi yake. Haijalishi unakitumia mara ngapi `add(2, 3)`, daima kitarejesha `5`, na hakitaathiri sehemu nyingine yoyote ya programu yako.

Sasa, tukilinganisha na kitendakazi kisicho safi:

```js
let total = 0;
function addToTotal(value) {
  total += value;
  return total;
}
```

Kitendakazi hiki si safi kwa sababu kinabadilisha kigezo kilicho nje ya nafsi yake (`total`). Matokeo ya `addToTotal(5)` yatatofautiana kulingana na kile kilichokuwa `total` kabla kitendakazi kuitwa. Kutotabirika kwa hili kunaweza kufanya msimbo wetu kuwa mgumu kuelewa na kutafuta makosa.

Hii inatupeleka kwenye dhana yetu muhimu inayofuata, ambayo ni athari za pembeni.

Athari ya pembeni ni mabadiliko yoyote yanayotokea katika hali ya programu wakati kitendakazi kinapoitwa. Hii inaweza kujumuisha kubadilisha kigezo cha kimataifa, kuandika kwenye faili, au kufanya wito wa API.

Vitendakazi safi havina athari za pembeni lakini vitendakazi visivyo safi vina athari hizi.

Kuepuka athari za pembeni ni kanuni muhimu ya programu utendaji. Tunapopunguza athari za pembeni, msimbo wetu unakuwa na utabirika zaidi na rahisi kujaribu.

Wakati wa kuandika msimbo, lengo letu kuu linapaswa kuwa kufanya vitendakazi vitende jambo moja na vifanye hilo vizuri sana bila kubadilisha sehemu nyingine za programu yetu.

Hapa kuna mfano wa kitendakazi chenye athari ya pembeni:

:::interactive_editor

```js
function greet(name) {
  console.log(`Hello, ${name}!`);
}

greet("Alice");
```

:::

Kitendakazi hiki kina athari ya pembeni kwa sababu kinashirikiana na dunia ya nje kwa kuandika kwenye konsoli.

Katika programu utendaji, tunaweza kuandika upya hivi:

:::interactive_editor

```js
function greet(name) {
  return `Hello, ${name}!`;
}
console.log(greet("Alice"));
```

:::

Sasa `greet` ni kitendakazi safi kinachorejesha tu mfuatano wa herufi, na sisi tunashughulikia athari ya pembeni (kuandika kwenye konsoli) kando.

Programu utendaji inatuhimiza kuunda programu zetu kwa kutumia vitendakazi safi kwa kugawanya msimbo wetu katika vipande vidogo, vinavyotabirika. Na hivyo, tunaweza kujenga programu ambazo ni rahisi kuelewa, kujaribu, na kudumisha.
