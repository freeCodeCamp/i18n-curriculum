# --description--

Das Debuggen Ihrer JavaScript-Programme geht über die Verwendung von `console.log()`-Anweisungen im Code hinaus. Es gibt fortgeschrittenere Techniken, die das Debuggen zum Kinderspiel machen, sobald Sie sich daran gewöhnt haben. Lassen Sie uns diese fortgeschrittenen Techniken durchgehen, damit Sie gerne Ihr JavaScript-Programm debuggen, wenn Sie auf Fehler stoßen.

Das erste Konzept, das wir uns ansehen, ist das Arbeiten mit Breakpoints. Breakpoints ermöglichen es Ihnen, die Ausführung Ihres Codes an einer bestimmten von Ihnen gewählten Zeile anzuhalten. Nach der Pause können Sie Variablen inspizieren, Ausdrücke auswerten und den Aufrufstapel untersuchen.

Um in Chrome einen Breakpoint zu einer beliebigen Zeile Ihres Codes hinzuzufügen, öffnen Sie die Entwicklerwerkzeuge und navigieren Sie zum Tab Sources, öffnen Sie die JavaScript-Datei, die Sie debuggen möchten, und klicken Sie auf eine Zeilennummer, um einen Breakpoint zu setzen.

Nachdem die Ausführung den Breakpoint erreicht und angehalten hat, können Sie den Code mit den Symbolen oben rechts schrittweise durchgehen. Sie können auch einen bedingten Breakpoint hinzufügen, indem Sie mit der rechten Maustaste auf eine Zeile klicken und „Add conditional breakpoint…“ auswählen.

Kommen wir nun zu den Watch Expressions. Watch Expressions ermöglichen es Ihnen, die Werte von Variablen oder Ausdrücken während der Codeausführung zu überwachen, selbst wenn sie außerhalb des aktuellen Gültigkeitsbereichs liegen.

Um eine Watch Expression hinzuzufügen, navigieren Sie zum Tab Sources der Entwicklerwerkzeuge, suchen Sie das Watch-Panel rechts und klicken Sie auf das Pluszeichen (`+`), um eine hinzuzufügen.

Das nächste Konzept, das wir betrachten, ist das Profiling. Profiling hilft Ihnen, Leistungsengpässe zu identifizieren, indem Sie Screenshots aufnehmen und CPU-Auslastung, Funktionsaufrufe sowie Ausführungszeit aufzeichnen.

Um das für Ihren Code zu tun, öffnen Sie den Tab Performance, klicken Sie auf Aufnehmen und führen Sie die Aktion aus, die Sie profilieren möchten. Nachdem Sie die Aktion ausgeführt haben, stoppen Sie die Aufnahme, um die Ergebnisse zu analysieren.

Das kann nützlich sein, wenn Ihre Anwendung bei bestimmten Vorgängen verzögert reagiert. Mit dem Profiler können Sie sehen, welche Funktion oder andere Daten im Code die meisten Ressourcen verbrauchen.

Kommen wir nun zum Untersuchen von Netzwerk-Anfragen. Das Untersuchen von Netzwerk-Anfragen kann Ihnen helfen, Probleme im Zusammenhang mit API-Anfragen zu debuggen, wie Parameterfehler, Adressfehler oder Serverfehler.

Um den Tab Network zum Debuggen zu verwenden, öffnen Sie die Entwicklerwerkzeuge und wechseln Sie zum Tab Network, klicken Sie dann auf einzelne Anfragen, um Details wie Kopfzeilen, Antworten und Nutzlasten zu sehen.

Im letzten Teil dieser Lektion konzentrieren wir uns auf `console.table` und `console.dir`.

`console.table()` zeigt tabellarische Daten als Tabelle in der Konsole an. Es nimmt ein obligatorisches Argument, das ein Array oder ein Objekt sein muss, und ein optionales Argument, um anzugeben, welche Eigenschaften oder Spalten angezeigt werden sollen.

`console.dir()` hingegen ermöglicht es Ihnen, eine interaktive Liste der Eigenschaften eines angegebenen JavaScript-Objekts anzuzeigen. Es gibt eine hierarchische Auflistung aus, die erweitert werden kann, um alle verschachtelten Eigenschaften zu sehen.

Jetzt, da Sie über ein erweitertes Werkzeugset an Debugging-Techniken verfügen, setzen Sie diese neuen Fähigkeiten in Ihren kommenden JavaScript-Projekten ein. Durch die Anwendung dieser Strategien können Sie Probleme effizienter identifizieren und beheben, was zu saubererem und zuverlässigerem Code führt.
