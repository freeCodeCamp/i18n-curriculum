# --interactive--

プログラミングでデータをソートする方法はたくさんあります。このレッスンでは、JavaScriptの組み込み`sort`メソッドにフォーカスします。

`sort`メソッドは配列の要素を並べ替えるために使われ、ソートされた配列への参照を返します。コピーは作成されず、要素はその場でソートされます。

`sort`メソッドの基本的な構文は以下の通りです。

```js
array.sort(compareFunction);
```

`compareFunction`はオプションのパラメータで、ソート順を定義する関数を指定します。数値をソートするときに比較関数を使う方法は後で見ていきます。

## 文字列のソート

最初の例では、ランダムな順序の文字列の配列があります。

```js
const fruits = ["Banana", "Orange", "Apple", "Mango"];
```

この配列をアルファベット順にソートするのが目標です。`sort`配列に対して`fruits`メソッドを呼び出すことでこれができます。

:::interactive_editor

```js
const fruits = ["Banana", "Orange", "Apple", "Mango"];
fruits.sort();

console.log(fruits); // ["Apple", "Banana", "Mango", "Orange"]
```

:::

結果は、`Apple`の果物から始まるアルファベット順にソートされた果物の配列になります。

## 数値のソート

次の例では、次の数値の配列をソートしたいと思います。

```js
const numbers = [414, 200, 5, 10, 3];
```

この`sort`配列に`numbers`メソッドを使おうとすると、予期しない結果になります。

:::interactive_editor

```js
const numbers = [414, 200, 5, 10, 3];
numbers.sort();

console.log(numbers); // [10, 200, 3, 414, 5]
```

:::

期待した結果は`[3, 5, 10, 200, 414]`でしたが、実際には`[10, 200, 3, 414, 5]`が返されました。

これは、`sort` メソッドが要素を文字列に変換し、その UTF-16 コード単位のシーケンスを比較するためです。

UTF-16コード単位は文字列内の文字を表す数値です。UTF-16コード単位の例として、`65`、`66`、`67`があり、それぞれ文字`A`、`B`、`C`を表します。

つまり、数値`200`は数値`3`の前に現れます。なぜなら文字列`200`が文字列`3`よりUTF-16コード単位の比較で前に来るからです。

この問題の解決策は、`sort`メソッドに比較関数を渡すことです。

以下は比較関数を使って`numbers`配列をソートする例です。

:::interactive_editor

```js
const numbers = [414, 200, 5, 10, 3];

numbers.sort((a, b) => a - b);

console.log(numbers); // [3, 5, 10, 200, 414]
```

:::

パラメータ`a`と`b`は比較される2つの要素です。比較関数は、`a`が`b`より前に来るべき場合は負の値を返し、`a`が`b`より後に来るべき場合は正の値を返し、`a`と`b`が等しい場合は0を返すべきです。

最初の比較は数値`414`と`200`の間で行われます。`414 - 200`の結果は`214`で、これは正の値です。つまり`414`は`200`の後にソートされるべきです。

次の比較は数値`200`と`5`の間で行われます。`200 - 5`の結果は`195`で、これも正の値です。つまり`200`は`5`の後にソートされるべきです。

このプロセスを配列内のすべての要素に繰り返し、結果として数値のソートされた配列が得られます。

## `undefined` 値と空のスロットの扱い

配列内の `undefined` 値と空のスロットを `sort` メソッドがどのように扱うかを理解することも重要です。

`sort` メソッドが `undefined` 値に遭遇すると、その値は比較関数に渡されません。代わりに、`undefined` 値は自動的に配列の末尾に移動されます。

`undefined` 値を含む配列を `sort` がどのように扱うかを示す例はこちらです：

:::interactive_editor

```js
const fruits = ["Banana", undefined, "Apple", "Mango"];
fruits.sort();

console.log(fruits); // ["Apple", "Banana", "Mango", undefined]
```

:::

`undefined` が他のすべての要素の後、ソートされた配列の末尾に配置されていることに注意してください。

同様に、`sort` メソッドがスパース配列の空のスロットに遭遇すると、その空のスロットを保持し、配列の末尾に移動します。`undefined` 値と空のスロットの両方が存在する場合は、`undefined` 値が先に来て、その後に空のスロットが最後に配置されます。

この動作を示す例はこちらです：

:::interactive_editor

```js
const arr = [, undefined, "banana", "apple"];
arr.sort();

console.log(arr); // ["apple", "banana", undefined, empty]
```

:::

この例では、文字列の `"apple"` と `"banana"` が最初にアルファベット順にソートされます。次に `undefined` 値がその後に配置され、最後に空のスロットが配列の末尾に保持されています。

プログラミングでデータをソートする方法は他にもたくさんありますが、JavaScriptの`sort`メソッドは配列の要素をソートする必要がある多くのケースで便利かつ効率的です。
