# --description--

Katika funzo lililopita, tuliangalia API ya Fetch ni nini na jinsi ya kuitumia. Katika funzo hili, tutajadili njia za HTTP za `GET`, `POST`, `PUT`, na `DELETE` za API ya Fetch.

Tuanze na njia ya kawaida zaidi ya HTTP ambayo ni njia ya `GET`. Hii hutumika kupata data kutoka kwa seva. Unapotumia `fetch()` bila kubainisha njia, kwa kawaida hutumia `GET`.

```js
fetch('https://api.example.com/data')
```

Katika msimbo huu, tunafanya ombi la `GET` kwenda `https://api.example.com/data`. Sasa, tafadhali fahamu kuwa huwezi kutumia data hii moja kwa moja; lazima ubadilishe jibu kuwa muundo wa JSON. Ni baada ya hapo tu unaweza kuitumia mahali popote unapotaka katika mradi wako. Hapa kuna mfano wa jinsi ya kufanya hivyo:

```js
fetch('https://api.example.com/data')
  .then(response => response.json())
  .then(data => console.log(data))
```

Katika msimbo huu, jibu linalotoka kutoka API ya Fetch ni ahadi, na tunatumia mshughulikiaji wa `.then` kubadilisha jibu kuwa muundo wa JSON. Katika funzo lililopita, ulijifunza kuwa `Promise` ni kitu kinacho wakilisha kukamilika kwa mwisho (au kushindwa) kwa mchakato isiyo ya wakati mmoja na thamani yake.

Thamani ya ahadi haijulikani wakati ahadi inapotengenezwa. Inajulikana tu wakati mchakato isiyo ya wakati mmoja umekamilika. Tunapounganisha mshughulikiaji wawili wa `.then` kwa wito la fetch, hii ni kitu kinachoitwa uunganishaji wa mfululizo wa ahadi ambacho kitafundishwa katika funzo lijalo.

Hadi sasa tumekuwa tukipata rasilimali kutoka kwa seva. Lakini, je, unajua kuwa tunaweza pia kutuma data kwa seva? Njia ya `POST` hutumika kutuma data kwa seva ili kuunda rasilimali. Hapa kuna mfano wa njia ya `POST` ambayo hutumika kuunda rasilimali kwenye seva:

```js
fetch('https://api.example.com/users', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({
    name: 'John Doe',
    email: 'john@example.com'
  })
})
```

Katika mfano huu, tunatuma ombi la `POST` kuunda mtumiaji mpya. Tunabainisha njia kama `POST`, kuweka vichwa vinavyofaa, na kujumuisha `body` yenye data tunayotaka kutuma. body inahitaji kuwa mfuatano wa herufi, hivyo tunatumia `JSON.stringify()` kubadilisha kitu chetu kuwa mfuatano wa herufi wa JSON.

Njia ya `PUT` hutumika kusahihisha rasilimali zilizopo kwenye seva. Hapa kuna mfano:

```js
fetch('https://api.example.com/users/45', {
  method: 'PUT',
  headers: {
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({
    name: 'John Smith',
    email: 'john@example.com'
  })
})
```

Katika mfano huu, angalia kwa makini URL, unaweza kuona `45` mwishoni. Hii kwa kawaida hutumika kama kitambulisho cha kipekee kutambua data tunayotaka kusahihisha. Tulitumia njia ya `PUT` katika msimbo na pia kubainisha data kama `body` ambayo itatumika kusahihisha data iliyotambuliwa.

Njia ya `DELETE` hutumika kufuta rasilimali kutoka kwa seva. Hapa kuna mfano:

```js
fetch('https://api.example.com/users/45', {
  method: 'DELETE',
})
```

Katika msimbo huu, tunajumuisha njia ya `DELETE` na kitambulisho mwishoni mwa URL kutambua data inayohitaji kufutwa.
