# --interactive--

クラスはオブジェクトを作成するための設計図のようなものだと学びました。

これらの設計図には、コード内で作成、アクセス、または変更されている特定のオブジェクトを指す一般的な方法が必要です。まさにそれが `this` キーワードの役割です。これはコードが実行されるコンテキストを指します。メソッド内で使うと、`this` の値はそのメソッドに関連付けられたオブジェクトへの参照になります。こうすることで、クラス内のオブジェクトのプロパティやメソッドにアクセスでき、異なるオブジェクトでメソッドを再利用できます。

例えば、ここに `Dessert` クラスがあります。今回は `hasPeanuts` プロパティを定義しています。

```js
class Dessert {
  constructor(hasPeanuts) {
    this.hasPeanuts = hasPeanuts;
  }
}
```

コンストラクタ内で `this` キーワードを使っていることに注目してください。コンストラクタはクラスの新しいオブジェクトが作成されると自動的に呼び出されることを覚えておいてください。

`this` キーワードは、`hasPeanuts` が現在作成中のオブジェクトのプロパティであるべきだとプログラムに伝えています。

代入文の右辺は、`hasPeanuts` パラメータの値を新しいオブジェクトの `hasPeanuts` プロパティに代入しています。

```js
this.hasPeanuts = hasPeanuts;
```

これはコンストラクタのコンテキストですが、他のメソッド内でも `this` キーワードを使うことができます。

こちらは少し変更を加えた `Dessert` クラスです。今は `name` と `price` の2つのプロパティがあります。

```js
class Dessert {
  constructor(name, price) {
    this.name = name;
    this.price = price;
  }

  showPrice() {
    console.log(`The price of ${this.name} is $${this.price}.`);
  }
}
```

新しい `showPrice` メソッドもあります。このメソッドでは、デザートオブジェクトの名前と価格を使って説明的なメッセージをコンソールに出力します。

`showPrice` メソッド内でオブジェクトのプロパティにアクセスするために `this` キーワードを使っているのに注目してください。

このメソッドのコンテキストで `this` キーワードはどんな値になるでしょうか？これはメソッドが呼び出されているオブジェクトを指します。この場合、名前が `"Brownie"`、価格が `$5.99` の新しい Dessert インスタンスを作成しています：

```js
const brownie = new Dessert("Brownie", 5.99);
```

このオブジェクトで `.showPrice()` メソッドを呼び出すと：

```js
brownie.showPrice();
```

こちらが完全な例です：

:::interactive_editor

```js
class Dessert {
  constructor(name, price) {
    this.name = name;
    this.price = price;
  }

  showPrice() {
    console.log(`The price of ${this.name} is $${this.price}.`);
  }
}

const brownie = new Dessert("Brownie", 5.99);
brownie.showPrice();
console.log(brownie);
```

:::

現在のオブジェクト（`brownie`）の名前と価格が文字列内に置き換えられます。

これは `this` キーワードのおかげで、クラス内で現在のオブジェクトを一般的に指す方法を提供しています。これにより、異なるオブジェクトでメソッドを再利用できます。

これはオブジェクトメソッドのコンテキストでの話ですが、`this` キーワードは単独の関数やアロー関数内でも値を持ちます。

単独の関数内での値は通常、グローバルオブジェクト（非厳格モードの場合）または `undefined`（厳格モードの場合）を指します。

アロー関数内の `this` キーワードの値は少し異なります。アロー関数は定義された外側のスコープから `this` の値を継承します。自身の `this` バインディングを作成しません。つまり、`this` の値はアロー関数が呼び出される場所や方法ではなく、アロー関数が定義されたコンテキストに依存します。これにより、アロー関数はコールバックやコンテキストの保持に便利です。

しかし、このために `this` の値はオブジェクトメソッド内で現在のオブジェクトを指さないので、アロー関数を使う際にはこれを覚えておく必要があります。

他の関数内で定義されたアロー関数にも同じことが当てはまります。親スコープから `this` の値を継承します。

`this` キーワードはオブジェクトにアクセスし操作するための強力なツールです。その仕組みを理解することは、JavaScriptのオブジェクト指向プログラミングをマスターするために不可欠です。
