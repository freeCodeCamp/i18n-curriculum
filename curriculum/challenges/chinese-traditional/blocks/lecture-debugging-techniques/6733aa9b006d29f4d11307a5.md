# --description--

當你在使用 JavaScript 進行程式設計時，你不可避免地會遇到錯誤訊息。了解常見的錯誤訊息將幫助你更有效地偵錯，並成為更強的程式設計師。四種常見的錯誤訊息類型是 `SyntaxError`、`ReferenceError`、`TypeError` 和 `RangeError`。

當你在程式碼中寫錯東西時，例如遺漏括號或方括號，就會發生 `SyntaxError`。可以把它想像成句子中的文法錯誤。以下是開發者在建立陣列時常犯的一個錯誤：

```js
const arr = ["Beau", "Quincy" "Tom"]
```

每個陣列元素需要以逗號分隔，否則會導致錯誤訊息。

第二個常見的 JavaScript 錯誤是 `ReferenceError`。有幾種不同類型的 `ReferenceError`，會以不同方式被觸發。第一種 `ReferenceError` 類型是未定義的變數。

```js
console.log(price);
```

在此範例中，我們嘗試將 `price` 變數記錄到主控台，但它尚未被定義。這將導致 `ReferenceError`。

另一個 `ReferenceError` 的範例是嘗試在使用 `let` 或 `const` 宣告的變數被定義之前存取該變數：

```js
console.log(b);
const b = 50;
```

上述範例將導致 `Cannot access 'b' before initialization` 錯誤。 

第三種常見錯誤是 `TypeError`。當你嘗試對錯誤的型別執行操作時，會發生這些錯誤。以下是一個嘗試在物件上使用 `map` 方法的範例：

```js
const developerObj = {
  name: "Jessica",
  country: "USA",
  isEmployed: true
};

developerObj.map()
```

此範例將導致 `developerObj.map is not a function` 錯誤，因為 `map` 方法是用於陣列而非物件。 

我們將要檢視的最後一個常見錯誤是 `RangeError`。

當你的程式碼嘗試使用超出 JavaScript 可控制代碼範圍的值時，就會發生 `RangeError`。以下是一個將無效值指定給陣列長度的範例：

```js
const arr = [];
arr.length = -1; 
```

由於陣列的 `length` 必須是非負整數，將它設為 `-1` 會觸發 `RangeError`。

當你繼續使用 JavaScript 撰寫程式時，請注意你可能會遇到的這些不同型別的錯誤以及它們發生的原因。
