# --description--

HTTPS significa Hypertext Transfer Protocol Secure. É a versão segura do HTTP.

Como você aprendeu em algumas palestras anteriores, o HTTP transfere dados entre seu navegador e um site. O HTTP padrão envia esses dados em texto simples, então qualquer pessoa pode interceptar esses dados em texto simples.

Por isso, o HTTPS usa um protocolo criptográfico chamado Transport Layer Security (TLS) para proteger essa transferência de dados. O TLS criptografa o canal de comunicação entre dois pontos. Sistemas mais antigos usavam Secure Sockets Layer (SSL), que muitas plataformas de hospedagem web ainda usam hoje.

A criptografia embaralha esses dados em um código ilegível que só o navegador do usuário e o servidor web têm as chaves para decifrar.

Por exemplo, quando você digita uma senha em um site, o HTTPS a embaralha instantaneamente e protege informações de cartão de crédito, endereços pessoais, credenciais de login e outras informações sensíveis. Hackers que interceptam os dados veem apenas caracteres aleatórios, então não podem usar as informações roubadas.

Essa proteção é vital em redes Wi-Fi públicas. Redes Wi-Fi abertas em cafeterias, aeroportos e outros locais públicos não têm segurança básica, dando aos cibercriminosos a capacidade de monitorar o tráfego não criptografado nessas redes. O HTTPS cria um túnel privado pela rede pública para manter seu histórico de navegação seguro contra agentes mal-intencionados próximos.

O HTTPS também fornece autenticação, que prova que um site é legítimo. Para usar HTTPS, o dono do site deve obter um certificado digital emitido por uma organização confiável. O certificado verifica a verdadeira identidade do dono do site.

Quando você visita um site e o endereço começa apenas com "http", essa é uma forma de saber que o dono do site não obteve esse certificado, tornando o site inseguro. Navegadores modernos também avisam sobre esses sites.

Por fim, a integridade dos dados é outro benefício chave do HTTPS. Isso significa que ninguém pode alterar seus dados durante o trânsito. Um hacker não pode injetar código malicioso nos arquivos do site enquanto eles viajam para seu dispositivo. Você recebe exatamente os dados que o dono do site enviou.

Motores de busca como o Google priorizam sites HTTPS, pois sites seguros têm melhor posição nos resultados de busca. Navegadores modernos também exigem HTTPS para recursos avançados como geolocalização, notificações push e câmeras web.
