# --description--

Lernen wir einige häufige HTTP-Antwortcodes kennen.

HTTP-Antwortcodes sind dreistellige numerische Codes, die Webserver verwenden, um mit Clients zu kommunizieren und ihnen mitzuteilen, was mit ihren Anfragen passiert ist.

Sie geben Auskunft darüber, ob die Anfrage erfolgreich bearbeitet wurde oder ob bei der Bearbeitung der Anfrage ein Fehler aufgetreten ist.

Wir werden einige der häufigsten in dieser Vorlesung behandeln.

## Erfolgscodes

Beginnen wir mit den **Erfolgscodes**. Diese Codes beginnen mit der Zahl 2.

Der Code **200 OK** ist der häufigste HTTP-Antwortcode. Er bedeutet, dass die Anfrage erfolgreich war und der Server die vom Client angeforderten Informationen zurückgegeben hat.

Der Code **201 Created** wird vom Server zurückgegeben, wenn der Client die Erstellung einer Ressource anfordert und der Vorgang erfolgreich abgeschlossen wurde. Zum Beispiel, wenn ein neues Benutzerkonto erfolgreich erstellt wurde, gibt der Server diesen Antwortcode an den Client zurück.

Der Code **204 No Content** wird zurückgegeben, wenn die Anfrage erfolgreich bearbeitet wurde, der Server aber keinen Inhalt zurückzugeben hat. Dies kann für Vorgänge verwendet werden, die eine Ressource in einer Datenbank löschen oder aktualisieren, aber keine Daten zurückgeben.

## **Umleitungscodes**

Kommen wir nun zu den **Umleitungscodes**.

Diese Codes werden vom Server zurückgegeben, wenn die vom Client angeforderte Ressource an einen anderen Ort verschoben wurde oder wenn der Client zusätzliche Schritte unternehmen muss, um die Anfrage abzuschließen.

Diese Codes beginnen mit der Zahl 3.

Der Code **301 Moved Permanently** wird zurückgegeben, wenn die vom Client angeforderte Ressource dauerhaft an einen anderen Ort verschoben wurde. In diesem Fall sollte der Server eine URL zum neuen Ort in der Antwort enthalten, und sobald der Client die Antwort erhält, sollte der Benutzer zu dieser URL umgeleitet werden.

Der Code **302 Found** zeigt an, dass die vom Client angeforderte Ressource vorübergehend an einen anderen Ort verschoben wurde. Der Server sollte die neue URL in der Antwort bereitstellen, und der Client sollte den Benutzer zu dieser URL umleiten. Die ursprüngliche URL könnte jedoch in Zukunft wieder verfügbar sein, da die Änderung nur vorübergehend ist. Das ist der Hauptunterschied zwischen den Antwortcodes 301 und 302.

Der Code **304 Not Modified** wird vom Server zurückgegeben, um dem Client mitzuteilen, dass die angeforderte Ressource seit der letzten Anfrage nicht geändert wurde. Der Server sendet diesen Code, um zu vermeiden, bereits gesendete Daten erneut zu übertragen, wenn sich der Inhalt nicht geändert hat. Es ist eine Technik zur Leistungsoptimierung.

## Client-Fehlercodes

Jetzt tauchen wir in die **Fehlercodes** ein.

Fehler können durch den Client oder durch den Server verursacht werden. Wir sprechen zuerst über Client-Fehlercodes.

Client-Fehlercodes werden zurückgegeben, wenn die Anfrage aufgrund eines Problems verursacht durch den Client nicht abgeschlossen werden konnte.

Diese Fehlercodes beginnen mit der Zahl 4.

Der Code **400 Bad Request** wird zurückgegeben, wenn der Server die vom Client gesendete Anfrage nicht verstehen konnte. Dies kann passieren, wenn der Client eine Anfrage mit falscher Syntax, ungültigen Parametern oder ungültigen Daten sendet.

Der Code **401 Unauthorized** steht im Zusammenhang mit der Authentifizierung. Er zeigt an, dass der Client bestimmte Authentifizierungsdaten bereitstellen muss, um auf die angeforderte Ressource zugreifen zu können. Diese Daten können einen Benutzernamen und ein Passwort umfassen.

Der Code **403 Forbidden** wird zurückgegeben, wenn der Client nicht berechtigt ist, auf die angeforderte Ressource zuzugreifen. Der Unterschied zwischen 401 und 403 besteht darin, dass bei 403 der Client nicht die erforderlichen Berechtigungen für die angeforderte Ressource hat, selbst wenn er gültige Authentifizierungsdaten besitzt.

Der Code **404 Not Found** wird zurückgegeben, wenn die Ressource auf dem Server nicht gefunden werden konnte. Vielleicht wurde sie gelöscht oder es gibt einen Tippfehler in der URL, der den Server daran hindert, die Ressource zu finden. Der Server kann auch ein temporäres Problem haben, das zu diesem Fehlercode führt.

## Server-Fehlercodes

Als Nächstes haben wir die **Server-Fehlercodes**. Diese Codes werden vom Server zurückgegeben, wenn die Anfrage aufgrund eines Problems auf der Serverseite nicht verarbeitet werden konnte.

Der Fehlercode **500 Internal Server Error** zeigt an, dass auf dem Server ein unerwarteter Fehler aufgetreten ist. Dieser Fehler kann durch eine Vielzahl von Problemen verursacht werden, von Konfigurationsproblemen bis hin zu temporären Fehlern.

Der Fehlercode **502 Bad Gateway** wird zurückgegeben, wenn der Server versucht hat, mit einem anderen Server zu kommunizieren, aber eine ungültige Antwort von diesem Server erhalten hat. Es gibt viele mögliche Ursachen für diesen Fehler.

Der Fehlercode **503 Service Unavailable** zeigt an, dass der Server die Anfrage vorübergehend nicht bearbeiten kann, sodass der Client es später erneut versuchen sollte. Häufige Ursachen für diesen Fehler sind Wartungsarbeiten und ein hohes Anfragevolumen, das der Server derzeit nicht bewältigen kann.

HTTP-Antwortcodes sind entscheidend, um die Kommunikation zwischen Clients und Webservern zu verstehen. Wenn Sie diese Codes verstehen, können Sie Ihre Webanwendungen besser debuggen und Fehler beheben.
