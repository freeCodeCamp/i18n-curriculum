## JavaScript upande wa mteja

- **JavaScript upande wa mteja**: Msimbo unaoendeshwa upande wa mteja wa programu ya mtandao, kawaida katika kivinjari cha mtandao.
- **Mipaka ya JavaScript upande wa mteja**: Ingawa yenye nguvu, JavaScript upande wa mteja ina mipaka fulani:
  - Upatikanaji mdogo sana wa mafaili ya eneo la mteja.
  - Haikusudiwi kushughulikia mantiki tata za programu.
  - Masuala ya usalama yanaweza kutokea (mfano, kufichua taarifa nyeti kama nywila za hifadhidata).
- **Sababu ya mipaka**: Mipaka hii ipo kwa sababu JavaScript upande wa mteja ilibuniwa awali kuendeshwa pekee katika vivinjari vya mtandao.

## Mazingira ya kivinjari

- **Injini ya JavaScript**: Vivinjari vya mtandao hutoa mazingira yanayohitajika kuendesha msimbo wa JavaScript, ikiwa ni pamoja na Injini ya JavaScript.
- **Mfano wa Kielelezo cha Hati (DOM)**: Vivinjari hutoa upatikanaji wa DOM, ikikuruhusu kufikia vipengele vya HTML katika msimbo wako.
- **Muktadha wa kihistoria**: Hapo awali, JavaScript haikuweza kuendeshwa nje ya kivinjari, lakini sasa inaweza kwa kutumia Node.js.

## Node.js ni nini?

- **Node.js**: Mazingira ya kuendesha JavaScript. Kimsingi inafafanuliwa kama "mazingira ya kuendesha JavaScript ya chanzo huru na yenye kuendana na mifumo mingi."
- **Chanzo huru**: Msimbo wa Node.js upo wazi kwa umma na unadumishwa na jamii kubwa ya wasanidi programu.
- **Kuendana na mifumo mingi**: Inaweza kufanya kazi kwenye mifumo yoyote ya uendeshaji, ikiwa ni pamoja na Windows, macOS, na Linux.
- **Mazingira ya kuendesha JavaScript**: Inakuwezesha kuendesha msimbo wa JavaScript nje ya kivinjari.

## Matumizi ya Node.js

- **Programu za kawaida**: Node.js hutumika katika sekta mbalimbali kwa:
  - Kujenga seva za mtandao na API zinazoshughulikia maombi ya HTTP.
  - Kuendeleza programu za mtandao na simu.
  - Kushughulikia data tata na mwingiliano na hifadhidata.
- **Matumizi katika sekta**: Ikiwa unatumia au kuendeleza tovuti au programu ya mtandao inayoshirikiana na hifadhidata au kushughulikia data tata, kuna uwezekano mkubwa utatumia Node.js nyuma ya pazia.
- **Sifa kuu**: Node.js ni yenye ufanisi, inayoweza kupanuka, na ina jamii kubwa ya wasanidi programu na wadumishaji duniani kote.

## Mazingira ya kivinjari dhidi ya Node runtime environment

- **Mazingira ya kivinjari**: Yamebuniwa hasa kwa ukuzaji wa mtandao upande wa mbele, yanaendesha JavaScript upande wa mteja.
- **Node runtime environment**: Yamebuniwa hasa kwa ukuzaji wa mtandao upande wa nyuma, yanaendesha JavaScript upande wa seva.
- **Tofauti za API**: Mazingira huamua API zinazopatikana:
  - **Kivinjari**: Upatikanaji wa API ya DOM, lakini na vikwazo vya upatikanaji wa mfumo wa faili wa eneo la mteja.
  - **Node**: Upatikanaji wa karibu rasilimali zote za mfumo, ikiwa ni pamoja na mfumo wa faili, lakini si DOM.

## Vitu vya ulimwengu wote

- **Kitu cha ulimwengu wote cha kivinjari (`window`)**: Kinatoa upatikanaji wa huduma zinazohusiana na kivinjari:
  - Njia za kudhibiti DOM.
  - Kusimamia vidakuzi (cookies).
  - Kushughulikia matukio ya kivinjari.

- **Kitu cha ulimwengu wote cha Node.js (`global`)**: Kinatoa upatikanaji wa huduma maalum za Node.js:
  - Moduli zilizojengwa ndani za kufanya kazi na mafaili ya eneo la mteja.
  - Uwezo wa mtandao.
  - Kuingiliana na mfumo wa uendeshaji.

## Udhibiti wa toleo

- **Toleo la kivinjari**: Huna udhibiti juu ya toleo la mazingira ya kivinjari watakaotumia watumizi wako kutembelea tovuti yako.
- **Toleo la Node.js**: Unaweza kuchagua toleo la Node.js unalotaka kutumia kwenye seva yako.

## Umuhimu wa Node.js

- **Ukuzaji wa full-stack**: Node.js imebadilisha ukuzaji wa mtandao kwa kuruhusu wasanidi programu kutumia JavaScript kujenga sehemu zote za mbele na nyuma za programu za full-stack.
- **Ufanisi**: Mchakato wa ukuzaji ni mzuri zaidi kwa sababu wasanidi programu hawahitaji kujifunza lugha mpya ya programu kwa ajili ya ukuzaji wa sehemu ya nyuma.
- **Athari**: Node.js ni chombo chenye nguvu, kinachoweza kupanuka, na chenye matumizi mengi ambacho kimekuwa muhimu sana kwa wasanidi programu duniani kote.

## Faida za Node.js

- **Lugha moja kwa full-stack**: Unaweza kutekeleza sehemu zote za mbele na nyuma za programu ya mtandao kwa kutumia JavaScript.
  - Inapunguza ugumu wa kujifunza mwanzoni.
  - Hakuna haja ya kubadilisha kati ya lugha za programu.
  - Inaongeza ufanisi na uzalishaji.

- **Msingi usiozuia, unaoendeshwa na matukio**: Nzuri kwa kuendeleza programu za wakati halisi ambapo majibu na ufanisi ni muhimu kwa kuunda uzoefu mzuri wa mtumiaji.

- **Nyuzi moja na mzunguko wa matukio**: Inaweza kushughulikia kwa ufanisi idadi kubwa ya maombi na shughuli za Ingizo-Tokeo kwa wakati mmoja.
  - Inafaa kwa programu zinazohitaji kushughulikia maombi mengi kwa wakati mmoja.

- **Jamii kubwa**: Maelfu ya wasanidi programu duniani kote wanaendelea kudumisha Node.js na kuongeza vipengele vipya.
  - Rasilimali nyingi za kujifunza zinapatikana.
  - Rahisi kupata majibu ya maswali.

- **npm (Node Package Manager)**: Chombo chenye nguvu kinachokuwezesha kusakinisha na kusimamia vifurushi na moduli kwa miradi yako.
  - Vifurushi vinapatikana kwa madhumuni mengi: kushughulikia maombi ya HTTP, kufanya kazi na hifadhidata, kuunda tarehe na nyakati, kupima msimbo, na zaidi.
  - Tumia tena msimbo uliyoandikwa, kujaribiwa, na kushirikiwa na wasanidi programu wengine.
  - Hufanya mtiririko wa kazi kuwa haraka na mzuri zaidi.

- **Gharama nafuu**: Node.js ni bure na chanzo huru.
  - Vifurushi, maktaba, na zana zinazohusiana pia kwa kawaida ni bure.
  - Ni jambo muhimu wakati wa kuboresha bajeti.

## Hasara za Node.js

- **Kizuizi cha nyuzi moja**: Inaendesha nyuzi moja kwa wakati, ikimaanisha inaweza kushughulikia operesheni moja kwa wakati mmoja tu.
  - Ingawa usanifu unaruhusu kushughulikia maombi ya wakati mmoja kwa ufanisi, kazi zinazotumia nguvu nyingi za prosesa zinaweza kuzuia nyuzi kuu na kusababisha matatizo ya utendaji.
  - Mifano ya kazi zinazotumia nguvu nyingi za prosesa: operesheni za kihesabu tata, usindikaji wa picha na video, usimbaji fiche.
  - Njia za kushinda hili zipo, lakini kawaida huongeza ugumu wa programu.

- **Uprogramu usio wa wakati mmoja (asynchronous)**: Node.js inategemea uprogramu usio wa wakati mmoja.
  - Zoezi linaloweza kuchukua muda mrefu kuendesha linaanzishwa, lakini badala ya kusubiri litakapokamilika, programu kuu inaendelea kuendesha wakati zoezi hilo linaendeshwa sambamba.
  - Wakati zoezi linapokamilika, programu hushughulikia matokeo.
  - Mara nyingi linahusisha "wito wa kurudisha" (callbacks), ambazo ni kazi zinazobainisha kinachotokea wakati operesheni zisizo za wakati mmoja zinapokamilika.
  - Inaweza kufanya msimbo kuwa mgumu zaidi kusoma, kuelewa, na kutafuta makosa.

- **Masuala ya ubora wa vifurushi**: Unapochagua vifurushi kutoka npm, baadhi huenda havidumishwi mara kwa mara na vinaweza kuleta hatari katika programu yako.
  - Wasanidi programu wanapaswa kutathmini kila kifurushi kwa makini.
  - Angalia kama vifurushi vinazingatia mbinu bora za ubora na usalama.

## Nyuzi

- **Nyuzi**: Njia ya utekelezaji ndani ya mchakato, kama programu ya kompyuta inayotekelezwa kwa sasa.

## NVM (Node Version Manager)

- **NVM**: Chombo cha kusimamia matoleo mengi ya Node.js kwenye kompyuta moja.
- **Madhumuni**: Inakusaidia kubadilisha kati ya matoleo tofauti ya Node.js kwa miradi tofauti kwa urahisi.
- **Mapendekezo**: Kusakinisha NVM kabla ya Node.js kunapendekezwa kwa ajili ya kubadilika baadaye.

## Kusakinisha NVM kwenye macOS/Linux

- **Zana za mstari wa amri za Xcode (macOS)**: Lazima zisakinishwe kwa mkono kabla ya kuendesha script ya usakinishaji wa NVM kwenye macOS.

```bash
xcode-select --install
```

- **Thibitisha usakinishaji wa Xcode**:

```bash
xcode-select --version
```

- **Pakua na Endesha script ya usakinishaji wa NVM**: Endesha moja ya amri hizi:

```bash
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.3/install.sh | bash
```

```bash
wget -qO- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.3/install.sh | bash
```

- **Anzisha upya terminal**: Huenda ukahitaji kuanzisha upya terminal baada ya mchakato kukamilika ili kupakia amri na usanidi mpya.

- **Thibitisha usakinishaji wa NVM**:

```bash
nvm -v
```

## Kusakinisha Node.js kwa kutumia NVM kwenye macOS/Linux

- **Sakinisha toleo la LTS la hivi karibuni**: Kusakinisha toleo la hivi karibuni la Long-Term Support (LTS) la Node.js:

```bash
nvm install --lts
```

- **LTS (Long-Term Support)**: Toleo hili kwa kawaida linapendekezwa kwa sababu linaweka kipaumbele utulivu, kuaminika, na usalama badala ya vipengele vipya au vya majaribio.
  - Limejaribiwa kwa kina na lipo tayari kwa uzalishaji.
  - Linaahidiwa kuwa na vipindi virefu vya msaada kwa kuzingatia marekebisho ya hitilafu na vifuniko vya usalama.

- **Sakinisha toleo maalum**: Unaweza kusakinisha toleo maalum la Node.js kwa kubainisha nambari ya toleo:

```bash
nvm install <version>
```

Mfano:

```bash
nvm install 20
```

- **Badilisha kwenda toleo**:

```bash
nvm use <version>
```

Mfano:

```bash
nvm use 20
```

- **Angalia toleo la sasa la Node.js**:

```bash
node -v
```

## Kusakinisha NVM kwenye Windows

- **Msaada rasmi wa NVM**: Matoleo rasmi ya NVM yana msaada kwa Windows katika hali maalum tu.
  - Ikiwa una Windows Subsystem for Linux (WSL), unaweza kufuata mwongozo wa macOS/Linux ulio hapo juu.

- **nvm-windows**: Zana ya kusimamia matoleo ya Node.js kwa Windows.
  - Mradi tofauti kabisa na mradi rasmi wa NVM.
  - Madhumuni sawa: kusaidia kufanya kazi na matoleo mengi ya Node.js kwenye kifaa kimoja.

- **Kabla ya kusakinisha**: Inapendekezwa kuondoa usakinishaji wowote wa awali wa Node kabla ya kusakinisha NVM kwa Windows.

- **Hatua za usakinishaji**:
  - Nenda kwenye sehemu ya Releases ya hifadhidata rasmi: `https://github.com/coreybutler/nvm-windows/releases`.
  - Pakua msakinishaji wa hivi karibuni.
  - Fuata maelekezo.

- **Anzisha upya terminal**: Huenda ukahitaji kuanzisha upya terminal ikiwa amri ya `nvm` haipatikani mara moja.

- **Thibitisha usakinishaji**:

```powershell
nvm -v
```

## Amri za msingi za NVM na Node.js

- **Kumbuka kuhusu amri**: Amri zote zinafanya kazi kwa NVM iliyosakinishwa kwenye macOS/Linux, ikiwa ni pamoja na WSL. Huenda zikafanya kazi tofauti au zisifanye kazi kabisa kwa `nvm-windows`. Amri mbadala za `nvm-windows` zinatolewa pale inapowezekana.

### Sera ya utekelezaji ya PowerShell (Windows)

- **Sera ya utekelezaji ya chaguo-msingi**: Sera ya utekelezaji ya PowerShell kwa chaguo-msingi ni `Restricted`, ambayo inazuia script kuendeshwa kama hatua ya usalama.

- **Endesha PowerShell kama msimamizi**: Bofya kulia PowerShell, kisha bofya "Run as Administrator".

- **Angalia sera ya utekelezaji ya sasa**:

```powershell
Get-ExecutionPolicy
```

- **Weka sera ya utekelezaji kuwa RemoteSigned**: Inaruhusu script za eneo na `npm` kuendeshwa:

```powershell
Set-ExecutionPolicy RemoteSigned
```

- **Baada ya kuweka sera**: Funga dirisha la msimamizi wa PowerShell na ufungue dirisha la PowerShell lisilo la msimamizi.

### Orodhesha matoleo yote yaliyosakinishwa

- **Orodhesha matoleo yaliyosakinishwa**: Ili kuorodhesha matoleo yote ya Node.js yaliyosakinishwa na kuangalia toleo linalotumika sasa:

```bash
nvm ls
```

### Weka toleo la chaguo-msingi la Node.js (NVM kwenye macOS/Linux pekee)

- **Weka toleo la chaguo-msingi**: Inaweka toleo la chaguo-msingi la Node.js kwa vikao vipya vya terminal:

```bash
nvm alias default <version>
```

Mfano:

```bash
nvm alias default 22.20.0
```

- **Tengeneza majina maalum**: Weka jina maalum zaidi kwa toleo:

```bash
nvm alias <new_name> <version>
```

Mfano:

```bash
nvm alias new-api 18.20.8
```

- **Tumia jina maalum**:

```bash
nvm use <new_name>
```

Mfano:

```bash
nvm use new-api
```

### Ondoa toleo la Node.js

- **Ondoa toleo**:

```bash
nvm uninstall <version>
```

Mfano:

```bash
nvm uninstall 20.19.5
```

- **Ondoa kwa toleo kuu (NVM kwenye macOS/Linux)**: Unaweza kuondoa toleo kwa kuendesha amri ya `uninstall` na toleo kuu kama `22` au `20`:

```bash
nvm uninstall 20
```

Ikiwa una matoleo mengi ya toleo kuu hilo yaliyosakinishwa, kama `20.14.0` na `20.12.2`, toleo la hivi karibuni litaondolewa kwanza. Kuendesha `nvm uninstall 20` kutaondoa `20.14.0` kwanza.

### Angalia toleo la Node.js

- **Angalia toleo la sasa**:

```bash
node -v
```

### Angalia toleo la npm

- **npm**: Meneja wa vifurushi kwa Node.js unaotumika kusakinisha, kuchapisha, na kusimamia vifurushi vya programu. Huasakinishwa moja kwa moja unaposakinisha Node.js.

- **Angalia toleo la npm**:

```bash
npm -v
```

### Endesha faili la JavaScript

- **Endesha faili la JavaScript**: Inaendesha faili la JavaScript katika mazingira ya sasa ya Node.js:

```bash
node <file.js>
```

Mfano:

```bash
node app.js
```

### Tengeneza mradi wa Node.js

- **Anzisha mradi**: Hutoa mradi mpya wa Node.js kwa kuunda faili la `package.json` linalofuatilia maelezo na utegemezi wake:

```bash
npm init
```

- **Mpangilio wa mwingiliano**: Amri ya `npm init` inakuongoza kupitia mwongozo wa kiolesura cha mstari wa amri kwa ajili ya kuanzisha faili la `package.json`.
  - Huita maswali kuhusu jina la mradi, maelezo, toleo, n.k.

- **Mpangilio wa chaguo-msingi**: Kuanzia na faili la `package.json` la chaguo-msingi:

```bash
npm init -y
```

au

```bash
npm init --yes
```

### Sakinisha kifurushi

- **Sakinisha kifurushi maalum**:

```bash
npm install <package>
```

Mfano:

```bash
npm install express
```

### Sakinisha utegemezi wote

- **Sakinisha utegemezi**: Kusakinisha utegemezi wote uliotajwa katika faili la `package.json` la mradi:

```bash
npm install
```
