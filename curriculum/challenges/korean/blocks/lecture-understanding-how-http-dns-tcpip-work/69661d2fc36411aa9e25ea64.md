# --description--

일반적인 HTTP 응답 코드에 대해 배워보겠습니다.

HTTP 응답 코드는 웹 서버가 클라이언트와 소통할 때 요청에 대해 어떤 일이 있었는지 알려주기 위해 사용하는 세 자리 숫자 코드입니다.

이 코드는 요청이 성공적으로 처리되었는지, 아니면 처리 중 오류가 있었는지에 대한 정보를 제공합니다.

이번 강의에서는 가장 흔히 사용되는 코드들을 다루겠습니다.

## 성공 코드

먼저 **성공 코드**부터 시작하겠습니다. 이 코드는 숫자 2로 시작합니다.

코드 **200 OK**는 가장 일반적인 HTTP 응답 코드입니다. 요청이 성공적으로 처리되었고 서버가 클라이언트가 요청한 정보를 반환했다는 뜻입니다.

코드 **201 Created**는 클라이언트가 리소스 생성을 요청했고 그 과정이 성공적으로 완료되었을 때 서버가 반환합니다. 예를 들어, 새 사용자 계정이 성공적으로 생성되면 서버는 이 응답 코드를 클라이언트에게 반환합니다.

코드 **204 No Content**는 요청이 성공적으로 처리되었지만 서버가 반환할 내용이 없을 때 반환됩니다. 데이터베이스에서 리소스를 삭제하거나 업데이트하는 과정에서 반환할 데이터가 없을 때 사용할 수 있습니다.

## **리다이렉트 코드**

이제 **리다이렉트 코드**에 대해 이야기해보겠습니다.

이 코드는 클라이언트가 요청한 리소스가 다른 위치로 이동했거나 클라이언트가 요청을 완료하기 위해 추가 단계를 수행해야 할 때 서버가 반환합니다.

이 코드는 숫자 3으로 시작합니다.

코드 **301 Moved Permanently**는 클라이언트가 요청한 리소스가 영구적으로 다른 위치로 이동했을 때 반환됩니다. 이 경우 서버는 응답에 새 위치의 URL을 포함해야 하며, 클라이언트가 응답을 받으면 사용자를 해당 URL로 리다이렉트해야 합니다.

코드 **302 Found**는 클라이언트가 요청한 리소스가 일시적으로 다른 위치로 이동했음을 나타냅니다. 서버는 새 URL을 응답에 제공하고 클라이언트는 사용자를 이 URL로 리다이렉트해야 합니다. 하지만 변경이 일시적이므로 원래 URL이 미래에 다시 사용 가능할 수 있습니다. 이것이 301과 302 응답 코드의 주요 차이점입니다.

코드 **304 Not Modified**는 서버가 클라이언트에게 요청한 리소스가 마지막 요청 이후 변경되지 않았음을 알리기 위해 반환합니다. 서버는 내용이 변경되지 않았다면 이미 보낸 데이터를 다시 보내지 않기 위해 이 코드를 보냅니다. 이는 성능 최적화 기법입니다.

## 클라이언트 오류 코드

이제 **오류 코드**를 살펴보겠습니다.

오류는 클라이언트와 서버 모두에서 발생할 수 있습니다. 먼저 클라이언트 오류 코드를 다루겠습니다.

클라이언트 오류 코드는 클라이언트가 발생시킨 문제로 인해 요청을 완료할 수 없을 때 반환됩니다.

이 오류 코드는 숫자 4로 시작합니다.

**400 Bad Request** 코드는 서버가 클라이언트가 보낸 요청을 이해하지 못했을 때 반환됩니다. 이는 클라이언트가 잘못된 구문, 유효하지 않은 매개변수 또는 잘못된 데이터를 포함한 요청을 보냈을 때 발생할 수 있습니다.

**401 Unauthorized** 코드는 인증과 관련이 있습니다. 클라이언트가 요청한 리소스에 접근하기 위해 특정 인증 자격 증명을 제공해야 함을 나타냅니다. 이 자격 증명에는 사용자 이름과 비밀번호가 포함될 수 있습니다.

**403 Forbidden** 코드는 클라이언트가 요청한 리소스에 접근할 권한이 없을 때 반환됩니다. 401과 403의 차이는 403에서는 클라이언트가 유효한 인증 자격 증명을 가지고 있더라도 해당 리소스에 필요한 권한이 없다는 점입니다.

**404 Not Found** 코드는 서버에서 리소스를 찾을 수 없을 때 반환됩니다. 리소스가 삭제되었거나 URL에 오타가 있어 서버가 리소스를 찾지 못할 수 있습니다. 서버가 일시적인 문제를 겪고 있어 이 오류 코드가 발생할 수도 있습니다.

## 서버 오류 코드

다음은 **서버 오류 코드**입니다. 이 코드는 서버 쪽 문제로 인해 요청을 처리할 수 없을 때 서버가 반환합니다.

**500 Internal Server Error** 오류 코드는 서버에서 예상치 못한 오류가 발생했음을 나타냅니다. 이 오류는 구성 문제부터 일시적인 문제까지 다양한 원인으로 발생할 수 있습니다.

**502 Bad Gateway** 오류 코드는 서버가 다른 서버와 통신하려고 했으나 그 서버로부터 잘못된 응답을 받았을 때 반환됩니다. 이 오류 유형에는 다양한 원인이 있을 수 있습니다.

**503 Service Unavailable** 오류 코드는 서버가 일시적으로 요청을 처리할 수 없으므로 클라이언트가 나중에 다시 시도해야 함을 나타냅니다. 이 오류의 일반적인 원인으로는 유지 관리 작업과 현재 서버가 처리할 수 없는 높은 요청량이 있습니다.

HTTP 응답 코드는 클라이언트와 웹 서버 간 통신을 이해하는 데 필수적입니다. 이 코드를 이해하면 웹 애플리케이션을 문제 해결하고 디버그하는 데 도움이 됩니다.
