# --description--

Зі зростанням вашого застосунку керування потоком даних між компонентами може ставати складнішим.

На початку може вистачити `useState` хука React, але з додаванням функцій ви можете зіткнутися з проблемами:

- Передавання пропсів через компоненти, яким вони не потрібні, що називають проп-дрилінгом
- Підтримання синхронності даних у різних частинах застосунку
- Обробка складних оновлень, які одночасно впливають на кілька компонентів

Ці та інші виклики можуть виникнути, що призведе до коду, який важче підтримувати, налагоджувати та тестувати. Саме тут на допомогу приходять бібліотеки для керування станом — вони забезпечують централізоване місце, де компоненти можуть отримувати або оновлювати потрібні їм дані.

Давайте розглянемо кілька варіантів керування станом і коли їх слід використовувати.

Context API — це менеджер стану, вбудований в React, який дозволяє ділитися станом між компонентами без використання сторонньої бібліотеки. Це добре відомий апгрейд над `useState` хуком, тому він ідеально підходить для випадків, як-от перемикання теми або статус автентифікації користувача.

Однак Context API не дуже добре справляється з частими оновленнями і може викликати зайві повторні відтворення, тому він менш підходить для складних потреб у стані в застосунках, таких як eCommerce або соціальні мережі.

Ось фішка лічильника, яка демонструє базове використання Context API:

```jsx
import { useState, createContext } from 'react';

const CounterContext = createContext();

const CounterProvider = ({ children }) => {
  const [count, setCount] = useState(0);

  return (
    <CounterContext.Provider value={{ count, setCount }}>
      {children}
    </CounterContext.Provider>
  );
};

export { CounterContext, CounterProvider };
```

Цей код створює контекст і провайдера, щоб ділитися `count` станом по всьому застосунку.

`CounterProvider` використовує хук `useState` для ініціалізації та керування `count` станом і його функцією оновлення. Обидва потім передаються дочірнім елементам через `Provider`.

Отже, коли ви обгортаєте весь застосунок `CounterProvider`, `count` стан стає доступним у будь-якій частині вашого застосунку.

Ось як можна обгорнути `CounterProvider` навколо вашого застосунку:

```jsx
import { CounterProvider } from './context/CounterContext';

function App() {
  return (
    <CounterProvider>
        {/* App components */}
    </CounterProvider>
  );
}

export default App;
```

А ось як можна використовувати `count` стан:

```jsx
import React, { useContext } from 'react';
import { CounterContext } from '../context/CounterContext';

const Counter = () => {
  const { count, setCount } = useContext(CounterContext);

  return (
    <>
      <div style={{ textAlign: 'center' }}>
        <h1>Context API Counter</h1>
        <button style={{ marginRight: '5px' }} onClick={() => setCount(count - 1)}>
          Decrease
        </button>
        <span>{count}</span>
        <button style={{ marginLeft: '5px' }} onClick={() => setCount(count + 1)}>
          Increase
        </button>
      </div>
    </>
  );
};

export default Counter;
```

Як бачите, `count` і його функція оновлення `setCount` ініціалізуються через функцію `useContext`.

Поточний `count` стан відображається, а `setCount` використовується для збільшення та зменшення `count` стану, коли користувач натискає відповідні кнопки.

Ще одна популярна бібліотека для керування станом — Redux, одна з найпопулярніших бібліотек для React. Вона існує вже давно і ідеально підходить для великих застосунків, таких як eCommerce, соціальні мережі, форуми тощо.

Redux керує станом, надаючи центральне сховище і суворий контроль над оновленнями стану. Він використовує передбачуваний шаблон з діями, редюсерами та підпрограмним забезпеченням.

Дії — це корисне навантаження інформації, які надсилають дані з вашого застосунку до Redux-сховища, часто викликані взаємодією користувача.

Редюсери — це функції, які визначають, як стан має змінюватися у відповідь на ці дії, забезпечуючи оновлення стану в незмінний спосіб.

Підпрограмне забезпечення, навпаки, виступає мостом між відправленням дій і редюсером, дозволяючи розширювати функціональність Redux (наприклад, логування, обробка асинхронних операцій) без зміни основного потоку.

Найпоширеніша скарга на Redux — це велика кількість шаблонного коду, необхідного для початку роботи. У відповідь команда Redux створила "Redux Toolkit" і "RTK Query", які значно спрощують процес налаштування.

Зазвичай ви визначаєте і дії, і редюсери в одному файлі за допомогою функції `createSlice()`. Зазвичай файл називають так, щоб він закінчувався словом `Slice`, наприклад, `productSlice`, `userSlice`, `counterSlice` тощо.

Ось `counterSlice` файл, який показує основи:

```jsx
import { createSlice } from '@reduxjs/toolkit';

const counterSlice = createSlice({
  name: 'counter',

  initialState: { count: 0 },

  reducers: {
    increment: (state) => {
      state.count += 1;
    },
    decrement: (state) => {
      state.count -= 1;
    },
  },
});

export const { increment, decrement } = counterSlice.actions;

export default counterSlice.reducer;
```

Далі потрібно обгорнути весь застосунок `Provider`, вибрати частину стану зі слайсу за допомогою `useSelector()`, а потім використати `useDispatch()`, щоб зробити стан активним.

Ще один варіант для розгляду — Zustand.

Zustand — це легка бібліотека для керування станом із простим API. Вона базується на хуках, тому має менше шаблонного коду порівняно з Redux, що робить її простішою і швидшою для налаштування.

Zustand ідеально підходить для застосунків малого та середнього масштабу. Вона працює, використовуючи `useStore` хук для прямого доступу до стану в компонентах і сторінках. Це дозволяє змінювати і отримувати дані без потреби в діях, редюсерах або провайдері.

Ось `useCounterStore`, який реалізує іншу функціональність лічильника:

```jsx
import { create } from 'zustand';

const useCounterStore = create((set) => ({
  count: 0,
  increment: () => set((state) => ({ count: state.count + 1 })),
  decrement: () => set((state) => ({ count: state.count - 1 })),
}));

export default useCounterStore;
```

А ось як ініціалізувати і використовувати стани у вашому застосунку:

```jsx
// Import the useCounterStore (it's just a hook)
import useCounterStore from '../useCounterStore';

const Counter = () => {
  // Initialize the states with the useCounterStore hook
  const { count, increment, decrement } = useCounterStore();

  return (
    <>
      <div style={{ textAlign: 'center' }}>
        <h1>Zustand Counter</h1>
        <button style={{ marginRight: '5px' }} onClick={() => decrement()}>
          Decrease
        </button>
        <span>{count}</span>
        <button style={{ marginLeft: '5px' }} onClick={() => increment()}>
          Increase
        </button>
      </div>
    </>
  );
};

export default Counter;
```

Хоча екосистема фронтенду постійно розвивається і регулярно з’являються нові бібліотеки для керування станом, ті, що ми розглянули, широко використовуються в індустрії.
