# --description--

Die `.env`-Datei und die `dotenv`-Bibliothek sind Werkzeuge, die in der modernen Softwareentwicklung verwendet werden, um sensible Konfigurationsdaten vom Anwendungscode zu trennen.

Eine `.env`-Datei ist eine Textdatei, die im Stammverzeichnis eines Projekts abgelegt wird und Konfigurationseinstellungen sowie Geheimnisse als Schlüssel-Wert-Paare speichert.

Anstatt API-Schlüssel, Datenbankpasswörter oder Portnummern direkt im Quellcode fest zu codieren und damit das Risiko einzugehen, sie offenzulegen, wenn der Code in ein öffentliches Repository wie GitHub hochgeladen wird, speichern Entwickler diese lokal in dieser Datei.

Die Struktur einer `.env`-Datei ist einfach:

```bash
PORT=3000
DB_HOST=localhost
API_KEY=xyz123secretkey
```

Da diese Datei hochsensible Schlüssel enthält, ist es üblich, `.env` in die `.gitignore`-Datei des Projekts aufzunehmen, damit sie niemals von Git verfolgt wird.

Das `dotenv`-Paket ist ein leichtgewichtiges Modul, das die `.env`-Datei mit Ihrer Anwendung verbindet.

Wenn Ihr Programm startet, liest `dotenv` die `.env`-Datei, parst die Schlüssel und injiziert sie in die Laufzeitumgebung. In einer Node.js-Anwendung initialisieren Sie das Paket beispielsweise ganz oben in Ihrer Einstiegspunktdatei. Nach dem Laden sind die Variablen global über das Objekt `process.env` zugänglich.

```javascript
const express = require('express');
const app = express();

// initialize and configure dotenv
require('dotenv').config();

// access the variables securely from the environment
const port = process.env.PORT || 5000;
const apiKey = process.env.API_KEY;

app.get('/', (req, res) => {
  res.send(`Connected successfully using API Key: ${apiKey}`);
});

app.listen(port, () => {
  console.log(`Server running on port ${port}`);
});
```

Diese Einrichtung bietet zwei wesentliche Vorteile. Erstens verbessert sie die Sicherheit, indem private Zugangsdaten aus dem Quellcode herausgehalten werden. Zweitens bietet sie Flexibilität, da dieselbe Codebasis sich in Entwicklungs-, Test- und Produktionsumgebungen unterschiedlich verhalten kann, indem einfach die lokale `.env`-Datei auf dem jeweiligen Hosting-Server ausgetauscht wird.
