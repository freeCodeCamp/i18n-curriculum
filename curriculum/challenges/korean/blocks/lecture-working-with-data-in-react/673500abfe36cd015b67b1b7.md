# --description--

React의 조건부 렌더링은 동적인 사용자 인터페이스를 만들 수 있게 해줍니다. 애플리케이션 내 특정 조건이나 상태에 따라 다른 내용을 보여줄 때 사용합니다.

조건부 렌더링을 사용하는 가장 일반적인 방법은 `if` 문, 삼항(`?:`) 연산자, 그리고 논리 AND (`&&`) 연산자를 사용하는 것입니다.

조건부 렌더링의 가장 간단한 형태는 `if` 문을 사용하는 것입니다. 예시는 다음과 같습니다:

```jsx
function Greeting({ isLoggedIn }) {
  if (isLoggedIn) {
    return <h1>Welcome back!</h1>;
  }
  return <h1>Please sign in</h1>;
}
```

이 예제에서 `Greeting` 컴포넌트는 `isLoggedIn` 프로프를 확인합니다. 만약 `true`라면 환영 메시지를 반환하고, 그렇지 않으면 사용자가 로그인하도록 요청합니다.

다음은 부모 `Greeting` 컴포넌트 안에 `App` 컴포넌트를 사용하는 예입니다:

```jsx
function App() {
  return (
    <div className="App">
      <Greeting isLoggedIn={false} />
    </div>
  );
}
```

더 간단한 조건에는 삼항 연산자(`?:`)가 JSX 내에서 직접 자주 사용됩니다. 인라인 조건부 렌더링을 가능하게 하여 코드를 더 간결하게 만들 수 있습니다:

```jsx
function Greeting({ isLoggedIn }) {
  return <h1>{isLoggedIn ? "Welcome back!" : "Please sign in."}</h1>;
}
```

이 코드는 이전 예제와 같은 결과를 내지만 더 압축된 형태입니다. 삼항 연산자는 `isLoggedIn`을 확인하고 적절한 메시지를 렌더링합니다.

조건부 렌더링의 또 다른 일반적인 패턴은 논리 AND (`&&`) 연산자를 사용하는 것입니다. 조건에 따라 무언가를 렌더링하거나 아무것도 렌더링하지 않을 때 특히 유용합니다:

```jsx
function Notification({ message }) {
  return (
    <div>
      {message && <p>{message}</p>}
    </div>
  );
}
```

이 예제에서 메시지가 담긴 단락 요소는 `message` 프로프가 참일 때만 렌더링됩니다. `message`이 거짓 같은 값, 즉 빈 문자열, `null`, 또는 `undefined`라면 화면에 아무것도 렌더링되지 않습니다.

이러한 조건부 렌더링 기법을 익히면 변화하는 데이터와 사용자 상호작용에 맞춰 더 인터랙티브하고 사용자 친화적인 애플리케이션을 만들 수 있습니다.
