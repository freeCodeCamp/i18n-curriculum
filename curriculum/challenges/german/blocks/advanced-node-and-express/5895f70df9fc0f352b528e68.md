# --description--

Eine Strategie ist eine Methode, um einen Benutzer zu authentifizieren. Sie können eine Strategie verwenden, um Benutzern die Authentifizierung basierend auf lokal gespeicherten Informationen zu ermöglichen (wenn Sie sie zuerst registrieren lassen) oder von verschiedenen Anbietern wie Google oder GitHub. Für dieses Projekt verwenden wir die Passport-Middleware. Passport bietet eine umfassende Sammlung von Strategien, die die Authentifizierung mit Benutzernamen und Passwort, GitHub, Google und mehr unterstützen.

`passport-local@~1.0.0` wurde bereits als Abhängigkeit hinzugefügt. Fügen Sie es Ihrem Server wie folgt hinzu:

```javascript
const LocalStrategy = require('passport-local');
```

Sagen Sie Passport, dass es ein instanziiertes `LocalStrategy`-Objekt mit einigen definierten Einstellungen **verwenden** soll. Stellen Sie sicher, dass dies (ebenso wie alles ab diesem Punkt) in der Datenbankverbindung gekapselt ist, da es darauf angewiesen ist!:

```javascript
passport.use(new LocalStrategy((username, password, done) => {
  myDataBase.findOne({ username: username }, (err, user) => {
    console.log(`User ${username} attempted to log in.`);
    if (err) return done(err);
    if (!user) return done(null, false);
    if (password !== user.password) return done(null, false);
    return done(null, user);
  });
}));
```

Dies definiert den Prozess, der verwendet wird, wenn Sie versuchen, jemanden lokal zu authentifizieren. Zuerst wird versucht, einen Benutzer in Ihrer Datenbank mit dem eingegebenen Benutzernamen zu finden. Dann wird überprüft, ob das Passwort übereinstimmt. Schließlich, wenn keine Fehler aufgetreten sind, die Sie geprüft haben (z. B. ein falsches Passwort), wird das `user`-Objekt zurückgegeben und der Benutzer ist authentifiziert.

Viele Strategien werden mit unterschiedlichen Einstellungen eingerichtet. Im Allgemeinen ist es einfach, sie basierend auf der README im Repository dieser Strategie einzurichten. Ein gutes Beispiel dafür ist die GitHub-Strategie, bei der Sie sich nicht um einen Benutzernamen oder ein Passwort kümmern müssen, weil der Benutzer zur Authentifizierungsseite von GitHub weitergeleitet wird. Solange er eingeloggt ist und zustimmt, gibt GitHub sein Profil zurück, das Sie verwenden können.

Im nächsten Schritt richten Sie ein, wie Sie die Authentifizierungsstrategie tatsächlich aufrufen, um einen Benutzer basierend auf Formulardaten zu validieren.

Senden Sie Ihre Seite ab, wenn Sie glauben, dass alles richtig ist.
