# --description--

До цього моменту ви бачили, чим JSX відрізняється від HTML у ключовому аспекті — використанням `className` замість `class` для визначення класів HTML.

Ще одна важлива відмінність JSX від HTML — це концепція самозакривного тегу.

В HTML майже всі теги мають і відкриваючий, і кінцевий тег: `<div></div>`; кінцевий тег завжди містить косу риску перед ім’ям тегу, який закривається. Однак існують особливі випадки в HTML, які називаються <dfn>порожні елементи</dfn>, або елементи, що не потребують і відкриваючого, і кінцевого тегу перед початком іншого елемента.

Наприклад, тег переносу рядка можна записати як `<br>` або як `<br />`, але ніколи не слід писати як `<br></br>`, оскільки він не містить жодного вмісту.

У JSX правила трохи інші. Будь-який JSX елемент можна записати як самозакривний тег, і кожен елемент повинен бути закритий. Тег переносу рядка, наприклад, завжди має бути записаний як `<br />`, щоб бути дійсним JSX, який можна транспілювати. `<div>`, натомість, можна записати як `<div />` або `<div></div>`. Різниця в тому, що у першій версії синтаксису неможливо додати щось у `<div />`. Ви побачите у наступних завданнях, що цей синтаксис корисний при відтворенні React компонентів.

# --instructions--

Виправте помилки в редакторі коду, щоб він став дійсним JSX і успішно транспілювався. Переконайтеся, що ви не змінюєте жодного вмісту — потрібно лише закрити теги там, де це необхідно.

# --hints--

Константа `JSX` має повертати елемент `div`.

```js
assert.strictEqual(JSX.type, 'div');
```

`div` має містити тег `br`.

```js
assert(Enzyme.shallow(JSX).find('br').length === 1);
```

`div` має містити тег `hr`.

```js
assert(Enzyme.shallow(JSX).find('hr').length === 1);
```
