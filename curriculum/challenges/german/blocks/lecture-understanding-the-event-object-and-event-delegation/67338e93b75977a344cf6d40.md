# --description--

Event-Bubbling oder Propagation beschreibt, wie ein Ereignis beim Auslösen zu übergeordneten Objekten „hochblubbert“. Betrachten Sie zum Beispiel diesen Code:

```html
<p>
  <span>Click me~!</span>
</p>
```

Das `p`-Element hier würde als übergeordnetes Element des `span`-Elements betrachtet werden.

Wenn Sie auf das `span`-Element klicken, wie Sie angewiesen werden, wird das `span`-Element zum Ziel eines `click`-Ereignisses. Dieses Ereignis blubbert jedoch auch zum übergeordneten Element hoch – das `p`-Element kann dieses Ereignis bei Bedarf empfangen und verarbeiten.

Aber was bedeutet das eigentlich? Nun, Sie könnten einen Ereignis-Listener am `p`-Element anbringen:

```js
const p = document.querySelector("p");
p.addEventListener("click", (event) => {
  console.log(event.target);
});
```

Wenn Sie dann auf das `span`-Element klicken, sehen Sie den Text `Click me~!` in der Konsole protokolliert.

Das Ereignis propagiert zum übergeordneten `p`-Element, das es in einem Ereignis-Listener verarbeitet, um das Ziel des Ereignisses anzuzeigen.

Beachten Sie, dass das Ziel immer noch das `span`-Element ist. Das liegt daran, dass das `span`-Element den ursprünglichen Klick erhalten hat.

Um sicherzugehen, wie die Dinge funktionieren, erweitern wir unseren Code:

```js
const p = document.querySelector("p");
const span = document.querySelector("span");
p.addEventListener("click", (event) => {
  console.log("P listener: ");
  console.log(event.target);
});
span.addEventListener("click", (event) => {
  console.log("Span listener: ");
  console.log(event.target);
});
```

Um Ihnen eine Vorstellung davon zu geben, wie das Ereignis hochblubbert, sehen Sie hier, was nach einem Klick auf das `span`-Element in der Konsole angezeigt wird:

```html
"Span listener: "
<span>Click me~!</span>
"P listener: "
<span>Click me~!</span>
```

Sehen wir uns nun an, was passiert, wenn Sie die Propagation eines Ereignisses mit `stopPropagation()` verhindern. Wir rufen es im Ereignis-Listener unseres `span` auf:

```js
const p = document.querySelector("p");
const span = document.querySelector("span");
p.addEventListener("click", (event) => {
  console.log("P listener: ");
  console.log(event.target);
});
span.addEventListener("click", (event) => {
  console.log("Span listener: ");
  console.log(event.target);
  event.stopPropagation();
});
```

Und klicken dann erneut auf unser `span`:

```html
"Span listener: "
<span>Click me~!</span>
```

Diesmal sehen wir nicht, dass unser `p`-Listener ausgelöst wird. Das Ereignis wird für das `p`-Element nie ausgelöst, weil wir ihm während der Verarbeitung für das untergeordnete `span`-Element gesagt haben, die Propagation zu stoppen.

Event-Delegation kann als das Gegenteil betrachtet werden. Dabei wird ein erfasstes Ereignis genommen und an ein anderes Element delegiert.

Zurück zu unserem Code: Aktualisieren wir ihn so, dass ein Klick auf ein `span`-Element dieses rot färbt:

```js
const p = document.querySelector("p");
const span = document.querySelector("span");
p.addEventListener("click", (event) => {});
span.addEventListener("click", (event) => {
  event.target.style.color = "red";
});
```

Aber was, wenn Sie zwanzig `span`-Elemente haben? Oder vielleicht verwenden Sie JavaScript, um weitere `span`-Elemente dynamisch zu erstellen?

Anstatt für jedes einzelne `span`-Element einen Ereignis-Listener anzubringen, können Sie tatsächlich den Listener am `p`-Element für alle verwenden. Mit anderen Worten, Sie delegieren die Behandlung der `span`-Klicks an das übergeordnete `p`-Element.

Unser Code könnte jetzt so aussehen:

```js
const p = document.querySelector("p");
p.addEventListener("click", (event) => {
  event.target.style.color = "red";
});
```

Beachten Sie, dass wir am `span`-Element überhaupt keinen Listener mehr angebracht haben. Sie haben die Ereignisbehandlung korrekt an das `p`-Element delegiert. Aber funktioniert das auch?

Lassen Sie uns ein paar zusätzliche `span`-Elemente generieren und sehen:

```html
<p>
  <span>Click me~!</span>
  <span>Click me~!</span>
  <span>Click me~!</span>
  <span>Click me~!</span>
</p>
```

Jetzt wird jedes Mal, wenn wir auf ein `span` klicken, der Text dieses Elements rot.

Und so haben wir mit nur einem Ereignis-Listener ein `click`-Ereignis korrekt von `span`-Elementen zum übergeordneten `p` hochblubbern lassen und die Logik für dieses `click`-Ereignis an das `p`-Element delegiert.

Event-Propagation und Delegation können ein komplexes Thema sein, besonders wenn Sie stark verschachtelte Elemente wie Tabellen betrachten. Sie sind eingeladen, dies weiter zu erforschen und mit dem hier geschriebenen Code zu experimentieren.
