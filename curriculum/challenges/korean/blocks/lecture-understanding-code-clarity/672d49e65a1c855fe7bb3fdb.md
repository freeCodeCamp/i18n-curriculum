# --description--

프로그래밍에서 주석은 코드에 추가적인 맥락을 제공하거나 자신과 다른 사람을 위한 메모를 남길 때 사용합니다.

주석은 코드가 실행될 때 JavaScript 엔진이 무시하는 한 줄 또는 여러 줄의 텍스트입니다. 주석은 코드를 읽는 사람, 즉 자신이나 다른 사람을 위한 것입니다.

JavaScript는 코드에 주석을 추가하는 두 가지 방법을 제공합니다: 한 줄 주석과 여러 줄 주석입니다.

한 줄 주석은 두 개의 슬래시(`//`)로 만듭니다. 예시는 다음과 같습니다:

```js
// I am a single line comment in JavaScript
```

이 주석 유형은 간단한 설명이나 명확히 할 때 좋습니다.

다음은 freeCodeCamp 커리큘럼 프로젝트 파일에서 가져온 실제 예시입니다:

```js
// This is to allow English to build without having to download the i18n files.
// It fails when trying to resolve the i18n-curriculum path if they don't exist.
const curriculumLocale = process.env.CURRICULUM_LOCALE ?? 'english';
const I18N_CURRICULUM_DIR = path.resolve(
  __dirname,
  curriculumLocale === 'english' ? '.' : 'i18n-curriculum/curriculum'
);
```

코드가 실제로 무엇을 하는지 이해하려고 걱정하지 마세요. 지금까지 배운 것보다 더 고급 내용입니다. 대신 개발자가 남긴 주석에 집중하세요. 이 주석은 이 코드가 왜 존재하는지에 대한 중요한 맥락을 제공합니다.

이런 주석은 팀에서 작업할 때 두 가지 이유로 중요합니다:

1. 프로젝트에 참여하는 다른 개발자가 이 코드의 목적을 이해할 수 있습니다.

1. 팀과 상의하지 않고 불필요한 변경이나 삭제를 막아 버그나 문제 발생을 예방합니다.

또 다른 주석 유형은 여러 줄 주석입니다. 기본 구문은 다음과 같습니다:

```js
/*
 I am a multiline comment.
 This is helpful for longer explanations.
*/
```

여러 줄 주석은 코드에 더 긴 설명, 해설, 메모를 작성해야 할 때 유용합니다.

freeCodeCamp 커리큘럼 프로젝트 파일을 다시 살펴보며 여러 줄 주석이 실제로 어떻게 사용되는지 보겠습니다.

```js
/* Since there can be more than one way to complete a certification (using the
legacy curriculum or the new one, for instance), we need a certification
field to track which certification this belongs to. */
const dupeCertifications = [
  {
    certification: 'responsive-web-design',
    dupe: '2022/responsive-web-design'
  }
];
const hasDupe = dupeCertifications.find(
  cert => cert.dupe === meta.superBlock
);
```

앞과 마찬가지로 JavaScript 코드는 아직 배우지 않은 개념을 사용하므로 무시하세요. 대신 개발자가 남긴 주석에 집중하세요.

팀의 개발자나 프로젝트에 새로 참여한 기여자도 이 코드가 왜 여기에 있는지 이해하고 프로젝트의 이 영역을 작업하기 전에 전체 맥락을 파악할 수 있습니다.

주석은 프로그래밍에서 유용하지만, 과도한 주석은 피하는 것이 중요합니다. 특히 코드가 명확하고 스스로 설명이 되는 경우 모든 줄에 주석을 달 필요는 없습니다.

명백한 내용을 설명하기 위해 주석을 사용하는 예시는 다음과 같습니다:

```js
// This code uses the const keyword to create a new variable called price.
// We are assigning the number 10 to the price variable.
const price = 10;
```

이 경우 코드는 스스로 설명이 되므로 주석을 추가할 필요가 없습니다. 목표는 가독성을 높이는 것이지 불필요한 설명으로 코드를 복잡하게 만드는 것이 아닙니다.

코드를 배우면서 개인 프로젝트에 주석을 다는 것은 괜찮습니다. 하지만 실제 프로젝트에서 다른 개발자와 함께 작업할 때는 스스로 설명되는 코드에 주석을 다는 것을 피하는 것이 중요합니다.

또한 혼란스럽거나 지나치게 복잡하거나 잘못 작성된 코드를 주석으로 설명하려고 해서는 안 됩니다. 그런 경우에는 코드를 리팩터링하거나 변경하여 다른 개발자가 더 잘 이해할 수 있도록 하는 것이 좋습니다.

주석은 코드를 문서화하고 이해하기 쉽게 만드는 강력한 도구입니다. 주석은 맥락을 제공하거나 자신과 다른 사람을 위한 메모를 남길 때 사용하세요.
