## Grundlagen der Arbeit mit Klassen

- **Definition**: Klassen in JavaScript werden verwendet, um Vorlagen für die Erstellung von Objekten zu definieren und Daten zu kapseln. Klassen enthalten einen Konstruktor, eine spezielle Methode, die automatisch aufgerufen wird, wenn ein neues Objekt aus der Klasse erstellt wird. Er dient dazu, die Eigenschaften des Objekts zu initialisieren. Das `this`-Schlüsselwort wird hier verwendet, um auf die aktuelle Instanz der Klasse zuzugreifen. Unter dem Konstruktor können Sie Methoden haben. Methoden sind Funktionen, die innerhalb einer Klasse definiert sind und Aktionen oder Operationen auf den Daten oder dem Zustand der Klasse ausführen. Sie dienen dazu, Verhaltensweisen zu definieren, die Instanzen der Klasse ausführen können.

:::interactive_editor

```js
class Dog {
  constructor(name) {
    this.name = name;
  }

  bark() {
    console.log(`${this.name} says woof!`);
  }
}

const dog = new Dog("Gino");
console.log(dog.name);  // Gino
```

:::

Um eine neue Instanz der Klasse zu erstellen, verwenden Sie das `new`-Schlüsselwort gefolgt vom Klassennamen:

```js
const dog = new Dog("Gino");
```

Sie können Klassen auch als Klassen-Ausdrücke erstellen. Dabei ist die Klasse anonym und wird einer Variablen zugewiesen.

:::interactive_editor

```js
const Dog = class {
  constructor(name) {
    this.name = name;
  }

  bark() {
    console.log(`${this.name} says woof!`);
  }
};

const dog = new Dog("Gino");
console.log(dog.name);  // Gino
dog.bark();  // Gino says woof!
```

:::

## Klassenvererbung

- **Definition**: In der Programmierung ermöglicht Vererbung, Klassen zu definieren, die Eigenschaften und Methoden von Elternklassen übernehmen. Dies fördert die Wiederverwendung von Code und stellt eine hierarchische Beziehung zwischen Klassen her. Eine Elternklasse ist eine Klasse, die als Vorlage für andere Klassen dient. Sie definiert Eigenschaften und Methoden, die von anderen Klassen geerbt werden. Eine Kindklasse ist eine Klasse, die die Eigenschaften und Methoden einer anderen Klasse erbt. Kindklassen können die Funktionalität ihrer Elternklassen erweitern, indem sie neue Eigenschaften und Methoden hinzufügen. In JavaScript verwenden wir das `extends`-Schlüsselwort, um Vererbung zu implementieren. Dieses Schlüsselwort zeigt an, dass eine Klasse die Kindklasse einer anderen Klasse ist.

:::interactive_editor

```js
class Vehicle {
  constructor(brand, year) {
    this.brand = brand;
    this.year = year;
  }
}

class Car extends Vehicle {
  honk() {
    console.log("Honk! Honk!");
  }
}

const myCar = new Car("freeCodeCamp Motors", 2019);
console.log(myCar.brand);  // freeCodeCamp Motors
console.log(myCar.year);  // 2019
myCar.honk();  // Honk! Honk!
```

:::

Das `super`-Schlüsselwort wird verwendet, um auf die Methoden, Konstruktoren und Felder der Elternklasse zuzugreifen.

:::interactive_editor

```js
class Vehicle {
  constructor(brand, year) {
    this.brand = brand;
    this.year = year;
  }
}

class Car extends Vehicle {
  constructor(brand, year, numDoors) {
    super(brand, year);
    this.numDoors = numDoors;
  }
}

const myCar = new Car("freeCodeCamp Motors", 2019, 4);
console.log(myCar.brand);  // freeCodeCamp Motors
console.log(myCar.year);  // 2019
console.log(myCar.numDoors);  // 4
```

:::

## Arbeiten mit statischen Methoden und statischen Eigenschaften

- **Statische Methoden**: Diese Methoden werden oft für Hilfsfunktionen verwendet, die keinen Zugriff auf den spezifischen Zustand eines Objekts benötigen. Sie werden innerhalb von Klassen definiert, um zusammengehörige Funktionalität zu kapseln.

:::interactive_editor

```js
class Movie {
  constructor(title, rating) {
    this.title = title;
    this.rating = rating;
  }

  static compareMovies(movieA, movieB) {
    if (movieA.rating > movieB.rating) {
      console.log(`${movieA.title} has a higher rating.`);
    } else if (movieA.rating < movieB.rating) {
      console.log(`${movieB.title} has a higher rating.`);
    } else {
      console.log("These movies have the same rating.");
    }
  }
}

let movieA = new Movie("Movie A", 80);
let movieB = new Movie("Movie B", 45);

Movie.compareMovies(movieA, movieB);  // Movie A has a higher rating.
```

:::

Statische Methoden sind auch hilfreich, um „Factory“-Methoden zu implementieren. Eine Factory-Methode ist eine Methode, die Sie zusätzlich zum Konstruktor definieren, um Objekte basierend auf bestimmten Kriterien zu erstellen.

```js
class Pizza {
  constructor(type, price) {
    this.type = type;
    this.price = price;
  }

  static createMargherita() {
    return new this("Margherita", 6.99);
  }
}

let myPizza = Pizza.createMargherita();
console.log(myPizza);  // Pizza { type: "Margherita", price: 6.99 }
console.log(myPizza.type);  // Margherita
```

- **Statische Eigenschaften**: Diese Eigenschaften werden verwendet, um Werte oder Attribute zu definieren, die mit der Klasse selbst verbunden sind, nicht mit Instanzen der Klasse. Statische Eigenschaften werden von allen Instanzen der Klasse geteilt und können ohne Erstellung einer Instanz der Klasse aufgerufen werden.

:::interactive_editor

```ts
class Car {
  // Static property
  static numberOfWheels = 4;

  constructor(make, model) {
    this.make = make;
    this.model = model;
  }

  // Instance method
  getCarInfo() {
    return `${this.make} ${this.model}`;
  }

  // Static method
  static getNumberOfWheels() {
    return Car.numberOfWheels;
  }
}

// Accessing static property directly from the class
console.log(Car.numberOfWheels);  // 4
```

:::
