# --description--

I selettori di classe selezionano tutti gli elementi con il `class` specificato, che è semplicemente un attributo che aggiungi a un elemento HTML. Ecco come aggiungere una classe a un tag HTML e selezionarla in CSS:

```html
<!-- index.html -->

<div class="alert-text">
  Please agree to our terms of service.
</div>
```

```css
/* styles.css */

.alert-text {
  color: red;
}
```

Nota la sintassi per i selettori `class`: un punto seguito immediatamente dal valore case-sensitive dell'attributo classe. Le classi non devono essere uniche, quindi puoi usare lo stesso `class` su quanti elementi vuoi.

Un'altra cosa che puoi fare con l'attributo `class` è aggiungere più classi a un singolo elemento come una lista separata da spazi, come `class="alert-text severe-alert"`. Poiché gli spazi separano i nomi `class` in questo modo, non dovresti mai usare spazi per nomi composti da più parole e dovresti invece usare un trattino.

## Selettori ID
I selettori ID sono simili ai selettori `class`. Selezionano un elemento con il `id` specificato, che è un altro attributo che aggiungi a un elemento HTML:

```html
<!-- index.html -->

<div id="title">My Awesome 90's Page</div>
```

```css
/* styles.css */

#title {
  background-color: red;
}
```

Invece del punto, usi un cancelletto seguito immediatamente dal valore case-sensitive dell'attributo `id`. Un errore comune è l'uso eccessivo dell'attributo `id` quando non è strettamente necessario e quando le classi sarebbero sufficienti. Anche se ci sono casi in cui usare un `id` ha senso o è necessario, come per sfruttare la specificità o per avere collegamenti che reindirizzano a una sezione della pagina corrente, dovresti usare gli `id` con parsimonia (se non evitarli del tutto).

La differenza principale tra classi e ID è che un elemento può avere solo un `id`. Un `id` non può essere ripetuto su una singola pagina e l'attributo `id` non dovrebbe contenere spazi.
