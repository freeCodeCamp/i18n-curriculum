# --description--

Чи траплялося у вас, що ваш застосунок Express зламався без чіткого повідомлення про помилку? Налагодження та логування допомагають побачити, що саме відбувається "під капотом", щоб ви могли швидше знайти й виправити проблеми.

У цьому уроці ми розглянемо, як Express обробляє налагодження та логування — і як ви можете використати це для покращення свого процесу розробки.

## Вбудоване налагодження в Express

Express вже має вбудований інструмент для налагодження під назвою `debug`. Вам не потрібно нічого встановлювати, щоб його використовувати.

Щоб увімкнути налагодження, достатньо встановити змінну середовища `DEBUG` під час запуску застосунку.

Ось приклад:

```js
DEBUG=express:* node index.js
```

Ця команда увімкне всі повідомлення налагодження в просторі імен `express:*`. Ви почнете бачити вихідні дані на кшталт:

```js
express:router:route dispatching GET /
express:router:layer processing GET /
```

Ці повідомлення допомагають відстежувати маршрутизацію, виклики підпрограмного забезпечення та інше. Це чудовий спосіб зрозуміти потік запитів у вашому застосунку.

Хочете бути конкретнішими? Можна звузити простір імен:

```js
DEBUG=express:router node index.js
```

Це покаже лише логи, пов’язані з логікою маршрутизатора Express.

У Windows синтаксис трохи відрізняється для PowerShell:

```js
$env:DEBUG = "express:*"; node index.js
```

А для CMD це виглядає так:

```js
set DEBUG=express:* && node index.js
```

Ідея та сама, просто інший спосіб встановлення змінних середовища.

Тепер подивімося на логування в Express.

Для логування HTTP-запитів багато розробників використовують підпрограмне забезпечення на кшталт `morgan`.

Ось як його використовувати:

```bash
npm install morgan
```

Потім додайте його до свого застосунку:

```js
const express = require('express')
const morgan = require('morgan')
const app = express()

app.use(morgan('dev'))s
```

Тепер кожен вхідний запит буде логуватися в терміналі з деталями, як-от метод, URL, статус і час відповіді.

І чому це важливо?

Налагодження дає вам уявлення про те, як Express обробляє запити, підпрограмне забезпечення та маршрутизацію, не засмічуючи ваші продакшн-логи.

Логування дає запис того, що відбувається під час запитів, що є критично важливим для налагодження помилок, розуміння трафіку та раннього виявлення проблем.

Разом вони дають вам контроль і прозорість у процесі розробки.
