# --description--

Nell'ultima lezione, hai imparato la memoizzazione e come funziona il hook `useMemo`.

In questa lezione, scoprirai come funzionano il hook `useCallback` e `React.memo`.

Nell'ultima lezione, abbiamo anche menzionato che `useCallback` serve per memorizzare i riferimenti alle funzioni.

Per `React.memo`, ti permette di memorizzare un componente per evitare render inutili quando la sua prop non è cambiata.

Ecco la sintassi base del hook `useCallback`:

```js
const handleClick = useCallback(() => {
  // code goes here
}, [dependency]);
```

Ecco la sintassi base di `React.memo`:

```jsx
const MemoizedComponent = React.memo(({ prop }) => {
  return (
    <>
      {/* Presentation */}
    </>
  )
});
```

Vediamo un esempio del hook `useCallback`:

```jsx
import { useState, useEffect } from "react";

function Counter() {
  const [count, setCount] = useState(0);

  const handleClick = () => {
    setCount((prevCount) => prevCount + 1);
  };

  useEffect(() => {
    console.log("useEffect runs");
  }, [handleClick]);

  return (
    <div>
      <p>Count: {count}</p>
      <button onClick={handleClick}>Increment</button>
    </div>
  );
}

export default Counter;
```

Nel componente, l'effetto viene eseguito ogni volta che `handleClick` cambia perché la funzione `handleClick` viene ricreata a ogni render.

Per risolvere questo, devi dire a React di trattare la funzione `handleClick` come la stessa cosa tra i render memorizzandola con il hook `useCallback`, così non viene ricreata:

```jsx
import { useState, useEffect, useCallback } from "react";

function Counter() {
  const [count, setCount] = useState(0);

  // Memoize the handleClick function with useCallback
  const handleClick = useCallback(() => {
    setCount((prevCount) => prevCount + 1);
  }, []);

  useEffect(() => {
    console.log("useEffect runs");
  }, [handleClick]);

  return (
    <div>
      <p>Count: {count}</p>
      <button onClick={handleClick}>Increment</button>
    </div>
  );
}

export default Counter;
```

Ora la funzione `handleClick` non viene più ricreata a ogni render.

Per mostrarti come funziona la funzione di ordine superiore `React.memo` (o `memo`) e come lavorano insieme al hook `useCallback`, ecco un componente `Counter` con una funzione `handleClick` che ha bisogno di `useCallback` ma attualmente non la usa:

```jsx
import { useState, useEffect, useCallback } from "react";
import CounterChild from "./CounterChild";

function Counter() {
  const [count, setCount] = useState(0);
  const [timer, setTimer] = useState(new Date().toLocaleTimeString());

  const handleClick = () => {
    setCount(count + 1);
  };

  useEffect(() => {
    const interval = setInterval(() => {
      setTimer(new Date().toLocaleTimeString());
    }, 1000);

    return () => clearInterval(interval);
  }, []);

  return (
    <div>
      <h1>Time: {timer}</h1>
      <p>Count: {count}</p>
      <button onClick={handleClick}>Increment</button>
      <CounterChild onClick={handleClick} />
    </div>
  );
}

export default Counter;
```

Questa funzione ha anche un timer nello stato che si aggiorna ogni secondo. Questo fa sì che il componente venga renderizzato di nuovo ogni volta che `timer` cambia, facendo ricreare la funzione `handleClick` a ogni render.

Ecco perché la funzione `handleClick` deve essere memorizzata con `useCallback`.

Ecco il componente `CounterChild`:

```jsx
const CounterChild = ({ onClick }) => {
  console.log("CounterChild component rendered");
  return <button onClick={onClick}>Increment from Child</button>;
};

export default CounterChild;
```

Questo componente `CounterChild` prende una prop `onClick`, dandoti la possibilità di incrementare anche il contatore da lì.

Poiché il componente `CounterChild` è un figlio del componente `Counter`, verrà renderizzato di nuovo ogni volta che `Counter` si renderizza di nuovo a causa del timer che cambia. Quindi anche `CounterChild` deve essere memorizzato.

Senza memoizzazione, poiché il componente si renderizza di nuovo a causa dell'aggiornamento del timer ogni secondo, anche il componente `CounterChild` viene renderizzato di nuovo.

Per evitare questo, devi memorizzare il componente `CounterChild` con `React.memo`:

```jsx
import React from "react";

const CounterChild = React.memo(({ onClick }) => {
  console.log("CounterChild component rendered");
  return <button onClick={onClick}>Increment from Child</button>;
});

export default CounterChild;
```

Le cose non funzionano ancora in modo ottimale anche dopo aver memorizzato `CounterChild` con `React.memo`.

Questo succede perché la funzione `handleClick` viene ricreata a ogni render, quindi deve essere memorizzata con `useCallback` per dire a React che vuoi che la funzione rimanga la stessa tra i render:

```js
const handleClick = useCallback(() => {
  setCount((prevCount) => prevCount + 1);
}, [count]);
```

Ora, il componente si renderizza di nuovo solo quando cambia lo stato `count`.
