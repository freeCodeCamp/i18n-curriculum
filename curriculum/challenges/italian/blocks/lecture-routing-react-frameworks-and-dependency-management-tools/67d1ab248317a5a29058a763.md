# --description--

Nel software, una dipendenza è quando un componente o modulo in un’applicazione si basa su un altro per funzionare correttamente. Le dipendenze sono comuni nelle applicazioni software perché permettono agli sviluppatori di usare funzioni o strumenti predefiniti creati da altri.

Se vuoi creare un’app React, dovrai installare le dipendenze di React. Senza queste dipendenze, nessuno del tuo codice funzionerà correttamente e l’applicazione mostrerà una lista di errori.

Quando lavori su progetti più complessi, spesso dovrai fare affidamento su molte dipendenze. Se un insieme di dipendenze non è ben gestito o definito in un progetto, si arriva a quella che viene chiamata dependency hell.

Per gestire le dipendenze software in un progetto, devi usare un package manager. Un package manager è uno strumento usato per l’installazione, gli aggiornamenti e la rimozione delle dipendenze. Molti linguaggi di programmazione popolari come JavaScript, Python, Ruby e Java usano tutti package manager.

In una lezione precedente, hai avuto una breve introduzione a un package manager popolare chiamato npm.

Per creare un nuovo progetto React usando Vite e npm puoi eseguire il seguente comando:

```sh
npm create vite@latest my-react-app -- --template react
```

Come ricorderai dalle lezioni precedenti, questo creerà tutto il codice boilerplate necessario per avviare una nuova applicazione React. Prima di poter avviare l’applicazione, dovrai installare le dipendenze eseguendo `npm install` o `npm i` per abbreviare.

Puoi vedere tutte le dipendenze nel file `package.json` che si trova nella directory principale del tuo progetto.

Il file `package.json` è un file di configurazione chiave nei progetti che contiene metadati sul tuo progetto, inclusi nome, versione e dipendenze. Definisce anche script, informazioni sulla licenza e altre impostazioni che aiutano a gestire il progetto e le sue dipendenze.

Quando installi le dipendenze, una cartella `node_modules` verrà aggiunta al tuo progetto.

La cartella `node_modules` è dove sono archiviati tutti i pacchetti e le librerie richiesti dal tuo progetto. Questa cartella contiene il codice effettivo per le dipendenze elencate nel file `package.json`, incluse sia le dipendenze dirette del tuo progetto sia le dipendenze di quelle dipendenze.

Le due dipendenze principali necessarie per un progetto React saranno i pacchetti `react` e `react-dom`:

```json
"dependencies": {
  "react": "^18.3.1",
  "react-dom": "^18.3.1"
}
```

Il `package.json` elencherà le versioni correnti che stai usando per quei pacchetti. Se devi aggiornare qualche pacchetto localmente, puoi eseguire il comando `npm update`. Oppure puoi aggiornare tutti i pacchetti globalmente eseguendo il comando `npm update -g`.

Oltre al file `package.json`, avrai anche un file `package-lock.json`. Questo file blocca le versioni esatte di tutti i pacchetti che il tuo progetto sta usando. Quando aggiorni un pacchetto, anche le nuove versioni verranno aggiornate nel file di blocco.

Un altro aspetto importante del file `package.json` sono le dev dependencies:

```json
"devDependencies": {
  "@eslint/js": "^9.17.0",
  "@types/react": "^18.3.18",
  "@types/react-dom": "^18.3.5",
  "@vitejs/plugin-react": "^4.3.4",
  "eslint": "^9.17.0",
  "eslint-plugin-react": "^7.37.2",
  "eslint-plugin-react-hooks": "^5.0.0",
  "eslint-plugin-react-refresh": "^0.4.16",
  "globals": "^15.14.0",
  "vite": "^6.0.5"
}
```

Le dev dependencies sono pacchetti usati solo per lo sviluppo e non in produzione. Un esempio è una libreria di test come Jest. Installeresti Jest come dev dependency perché lo usi solo per testare il progetto localmente, e non è necessario per far funzionare l’applicazione in produzione.

Per la maggior parte di questa lezione, ci siamo concentrati su npm. Ma ci sono altri package manager come yarn e pnpm. Quindi quale package manager dovresti usare per il tuo progetto?

Beh, la risposta breve è: dipende.

Se ti unisci a un progetto esistente con un team, tutte queste decisioni saranno già prese e userai il package manager esistente. Se stai creando un progetto da zero, dovrai valutare i pro e i contro di ogni manager e decidere quale si adatta meglio alle tue esigenze.
