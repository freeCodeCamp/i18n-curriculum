# --description--

広い型を持つ値を、より具体的な型に絞り込む必要がある場合があります。たとえば、オブジェクトが定義したインターフェースに合致していることを確認したり、文字列が特定の値のリスト内にあることを保証したりする場合です。これを実現する方法はいくつかあります。

最初は真値による絞り込みです。前のレッスンの例を考えてみましょう。

```js
const email = document.querySelector<HTMLInputElement>("#email");
console.log(email.value);
```

`value` が `email` の可能性があるため、`email` の `null` プロパティにアクセスしようとするとコンパイラーエラーが発生します。しかし、条件文を使って `email` が真値であることを確認してからプロパティにアクセスできます。

```js
const email = document.querySelector<HTMLInputElement>("#email");
if (email) {
  console.log(email.value);
}
```

この更新された例では、`null` が真値でないため、TypeScript は条件ブロック内で `email` が必ず `input` の要素であると推論できます。したがって、もはやコンパイラーエラーは発生しません。

真値チェックは逆方向でも機能します。

```js
const email = document.querySelector<HTMLInputElement>("#email");
if (!email) {
  throw new ReferenceError("Could not find email element!")
}
console.log(email.value);
```

この方法では、`email` が偽値の場合にエラーをスローします。`null` は偽値です。エラーをスローするとこのコードの論理的な実行が終了するため、`console.log()` の呼び出しに到達した時点で TypeScript は `email` が `null` でありえないことを知っています。

オプショナルチェーンも、`email` の値が `null` の場合はプロパティアクセスができないという前提のもと、型の絞り込みの一種です。

```js
const email = document.querySelector<HTMLInputElement>("#email");
console.log(email?.value);
```

では、他の型はどうでしょうか？ `typeof` 演算子を使って型を絞り込むこともできます。文字列または数値の可能性がある変数の例を見てみましょう。

```js
const myVal = Math.random() > 0.5 ? 222 : "222";
console.log(myVal / 10)
```

この例では、文字列に対して算術演算を行えないためコンパイラーエラーが発生します。しかし、`typeof` 変数の `myVal` を条件でチェックできます。

```js
const myVal = Math.random() > 0.5 ? 222 : "222";
if (typeof myVal === "number") {
  console.log(myVal / 10);
}
```

`typeof` キーワードを使ったため、TypeScript は `myVal` が数値であると認識し、安全に算術演算を行えます。

では、より複雑なオブジェクト型はどうでしょうか？ 対象のオブジェクトがクラスから来ている場合、`instanceof` キーワードを使って型を絞り込むことができます。`querySelector()` の例に戻りましょう。

```js
const email = document.querySelector("#email");
```

ジェネリック型を渡して TypeScript に要素の型を伝える代わりに、`instanceof` を使って型を絞り込み、より安全なコードを書けます。

```js
const email = document.querySelector("#email");

if (email instanceof HTMLInputElement) {
    console.log(email.value);
}
```

この方法は前のものと似ていますが、`instanceof` はランタイムのバリデーションです。つまり、もし TypeScript の型が間違っていても、JavaScript のコードが `email` が `input` の要素であることを確認します。

次に、API から `User` オブジェクトを取得して情報を出力しようとする例を見てみましょう。

```js
interface User {
    name: string;
    age: number;
}

const printAge = (user: User) => 
  console.log(`${user.name} is ${user.age} years old!`)

const request = await fetch("url")
const myUser = await request.json();
printAge(myUser);
```

API が正しいオブジェクトを返すことはわかっていても、TypeScript はそれを認識していないため、`myUser` を関数に渡そうとするとコンパイラーエラーが発生します。また、`.json()` メソッドはジェネリック型を受け取りません。

この問題を「簡単に」解決する方法は型キャストです。

```js
interface User {
    name: string;
    age: number;
}

const printAge = (user: User) => 
  console.log(`${user.name} is ${user.age} years old!`)

const request = await fetch("url")
const myUser = await request.json() as User;
printAge(myUser);
```

しかし、型をキャストすると、TypeScript の潜在的なエラー検出能力が弱まります。したがって、型をキャストする代わりに型ガードを書くことができます。

```js
interface User {
    name: string;
    age: number;
}

const isValidUser = (user: unknown): user is User => {
  return !!user && 
    typeof user === "object" &&
    "name" in user &&
    "age" in user;
}
```

この関数定義の重要な部分は戻り値の型です。`user is User` 構文は、この関数がブール値を返し、その値が `true` の場合、`user` が `User` インターフェースを満たしていることを示します。次に、`user` オブジェクトの構造が合っているか基本的なチェックを行います。ここでは真値（`!!user`）による絞り込みと `typeof` による絞り込みを使っています。これは `typeof null` が `"object"` を返すため必要です。

```js
interface User {
    name: string;
    age: number;
}

const isValidUser = (user: unknown): user is User => {
  return !!user && 
    typeof user === "object" &&
    "name" in user &&
    "age" in user;
}

const printAge = (user: User) => 
  console.log(`${user.name} is ${user.age} years old!`)

const request = await fetch("url")
const myUser = await request.json() as User;
if (isValidUser(myUser)) {
    printAge(myUser);
}
```

これらのロジックをすべて組み合わせると、もはやコンパイラーエラーは発生せず、コードを正常にビルドできます。

型の絞り込みは、安全でエラーの少ないコードを書くのに役立つ強力な機能ですが、TypeScript の型は完全に厳密ではないため、型を絞り込まずにキャストだけを行うような方法は避けてください。
