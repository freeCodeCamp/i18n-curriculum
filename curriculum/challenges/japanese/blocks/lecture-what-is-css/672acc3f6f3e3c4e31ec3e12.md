# --interactive--

CSSを使うとき、要素の表示動作には主に`inline`、`block`、`inline-block`の3種類があります。

これらのプロパティは、それぞれ要素の配置方法やページ上の他の要素との関わり方に影響を与えます。

このレッスンでは、`inline-block`プロパティの動作と、インライン要素やブロックレベル要素との違いに焦点を当てます。

ブロックレベル要素は親コンテナの幅いっぱいに広がる大きなコンテナや「ブロック」のように振る舞います。常に新しい行から始まり、高さや幅を調整できます。

インライン要素は必要なだけのスペースだけを使います。周囲のコンテンツの流れの中に入り込み、新しい行には移りません。

`inline-block`プロパティはこれら2つの動作のハイブリッドです。インライン要素のように、`inline-block`要素は新しい行を始めずにテキストの流れの中にとどまります。

しかし、インライン要素とは異なり、`inline-block`要素はブロックレベル要素のように幅や高さを調整できます。

つまり、`inline`と`inline-block`の主な違いは、`inline`要素はサイズを制御できませんが、`inline-block`要素は他のコンテンツとインラインのままでありながらサイズを完全に制御できる点です。

例を見てみましょう。

:::interactive_editor

```html
<link href="styles.css" rel="stylesheet">

<div class="container">
  <span class="inline-block-element element1">Inline-Block</span>
  <span class="inline-block-element element2">Inline-Block</span>
</div>
```

```css
.inline-block-element {
  display: inline-block;
  width: 150px;
  height: 100px;
}

.element1 {
  background-color: lightblue;
}

.element2 {
  background-color: lightgreen;
}
```

:::

上の例では、`div`に`container`というクラスが付いています。その`div`要素の中に2つの`span`要素があります。

それぞれのspan要素は`display:inline-block`に設定されており、幅と高さも指定されています。

インラインブロック要素はインライン要素のように横並びになりますが、幅と高さが指定されているためブロックのような性質も持ちます。

しかし、`display: inline-block`プロパティを削除すると、`height`も`width`も明確に定義していても適用されません。

修正したCSSはこちらです。

:::interactive_editor

```html
<link href="styles.css" rel="stylesheet">

<div class="container">
  <span class="inline-block-element element1">Span element</span>
  <span class="inline-block-element element2">Span element</span>
</div>
```

```css
.inline-block-element {
  width: 150px;
  height: 100px;
}

.element1 {
  background-color: lightblue;
}

.element2 {
  background-color: lightgreen;
}
```

:::

このコードでは`display: inline-block;`プロパティを削除し、それ以外はそのままにしています。ここで`span`要素はデフォルトのインライン要素の動作に戻ります。

その結果、指定した幅や高さは無視され、要素はコンテンツに必要なスペースだけを使います。

`inline-block`の動作を理解すると、連続したテキストの流れの中で整列とサイズ制御の両方が必要なレイアウトを作るのに役立ちます。
