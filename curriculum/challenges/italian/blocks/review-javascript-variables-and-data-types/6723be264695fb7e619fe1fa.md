## Lavorare con HTML, CSS e JavaScript

Mentre HTML e CSS forniscono la struttura del sito web, JavaScript aggiunge interattività ai siti permettendo funzionalità complesse, come gestire l'input dell'utente, animare elementi e persino costruire applicazioni web complete.

## Tipi di dati in JavaScript

I tipi di dati aiutano il programma a capire che tipo di dati sta gestendo, che sia un numero, un testo o altro.

- **Number**: Un numero rappresenta sia numeri interi sia valori a virgola mobile. Esempi di numeri interi sono 7, 19 e 90. 
- **Floating point**: Un numero a virgola mobile è un numero con un punto decimale. Esempi includono 3.14, 0.5 e 0.0001.
- **String**: Una stringa è una sequenza di caratteri, o testo, racchiusa tra virgolette. `"I like coding"` e `'JavaScript is fun'` sono esempi di stringhe.
- **Boolean**: Un booleano rappresenta uno dei due valori possibili: `true` o `false`. Puoi usare un booleano per rappresentare una condizione, come `isLoggedIn = true`.
- **Undefined e Null**: Un valore `undefined` è una variabile che è stata dichiarata ma non le è stato assegnato un valore. Un valore `null` è un valore vuoto, o una variabile a cui è stato intenzionalmente assegnato il valore `null`.
- **Object**: Un oggetto è una raccolta di coppie chiave-valore. La chiave è il nome della proprietà e il valore è il valore della proprietà.

Qui, l'oggetto `pet` ha tre proprietà o chiavi: `name`, `age` e `type`. I valori sono rispettivamente `Fluffy`, `3` e `dog`.

```js
let pet = {
  name: "Fluffy",
  age: 3,
  type: "dog"
};
```

- **Symbol**: Il tipo di dato Symbol è un valore unico e immutabile che può essere usato come identificatore per le proprietà degli oggetti.

Nell'esempio qui sotto, vengono creati due simboli con la stessa descrizione, ma non sono uguali.

:::interactive_editor

```js
const crypticKey1= Symbol("saltNpepper");
const crypticKey2= Symbol("saltNpepper");
console.log(crypticKey1 === crypticKey2); // false
```

:::

- **BigInt**: Quando un numero è troppo grande per il tipo di dato `Number`, puoi usare il tipo di dato BigInt per rappresentare numeri interi di lunghezza arbitraria.

Aggiungendo un `n` alla fine del numero, puoi creare un BigInt.

```js
const veryBigNumber = 1234567890123456789012345678901234567890n;
```

## Variabili in JavaScript

- Le variabili possono essere dichiarate usando la parola chiave `let`. 

```js
let cityName;
```

- Per assegnare un valore a una variabile, puoi usare l'operatore di assegnazione `=`.

```js
cityName = "New York";
```

- Le variabili dichiarate usando `let` possono essere riassegnate a un nuovo valore.

:::interactive_editor

```js
let cityName = "New York";
cityName = "Los Angeles";
console.log(cityName); // Los Angeles
```

:::

- Oltre a `let`, puoi anche usare `const` per dichiarare una variabile. Tuttavia, una variabile `const` non può essere riassegnata a un nuovo valore.

```js
const cityName = "New York";
cityName = "Los Angeles"; // TypeError: Assignment to constant variable.
```

- Le variabili dichiarate usando `const` sono usate per dichiarare costanti, che non possono cambiare durante il codice, come `PI` o `MAX_SIZE`.

## Convenzioni per i nomi delle variabili

- I nomi delle variabili dovrebbero essere descrittivi e significativi.
- I nomi delle variabili dovrebbero essere in camelCase come `cityName`, `isLoggedIn` e `veryBigNumber`.
- I nomi delle variabili non dovrebbero iniziare con un numero. Devono iniziare con una lettera, `_` o `$`.
- I nomi delle variabili non dovrebbero contenere spazi o caratteri speciali, eccetto `_` e `$`.
- I nomi delle variabili non devono essere parole chiave riservate.
- I nomi delle variabili sono case-sensitive. `age` e `Age` sono variabili diverse.

## Stringhe e immutabilità delle stringhe in JavaScript

- Le stringhe sono sequenze di caratteri racchiuse tra virgolette. Possono essere create usando virgolette singole e virgolette doppie.

```js
let correctWay = 'This is a string';
let alsoCorrect = "This is also a string";
```

- Le stringhe sono immutabili in JavaScript. Questo significa che una volta creata una stringa, non puoi cambiare i caratteri all'interno della stringa. Tuttavia, puoi riassegnare una stringa a un nuovo valore.

```js
let firstName = "John";
firstName = "Jane"; // Reassigning the string to a new value
```

## Concatenazione di stringhe in JavaScript

- La concatenazione è il processo di unire più stringhe o combinare stringhe con variabili che contengono testo. L'operatore `+` è uno dei metodi più semplici e usati frequentemente per concatenare stringhe.

:::interactive_editor

```js
let studentName = "Asad";
let studentAge = 25;
let studentInfo = studentName + " is " + studentAge + " years old.";
console.log(studentInfo); // Asad is 25 years old.
```

:::

- Se hai bisogno di aggiungere o concatenare a una stringa esistente, puoi usare l'operatore `+=`. Questo è utile quando vuoi costruire una stringa aggiungendo più testo nel tempo.

:::interactive_editor

```js
let message = "Welcome to programming, ";
message += "Asad!";
console.log(message); // Welcome to programming, Asad!
```

:::

- Un altro modo per concatenare stringhe è usare il metodo `concat()`. Questo metodo unisce due o più stringhe insieme.

:::interactive_editor

```js
let firstName = "John";
let lastName = "Doe";
let fullName = firstName.concat(" ", lastName);
console.log(fullName); // John Doe
```

:::

## Registrare messaggi con `console.log()`

- Il metodo `console.log()` è usato per registrare messaggi nella console. È uno strumento utile per eseguire il debug e testare il codice.

:::interactive_editor

```js
console.log("Hello, World!");
// Output: Hello, World!
```

:::

## I punti e virgola in JavaScript

- I punti e virgola sono usati principalmente per segnare la fine di un'istruzione. Questo aiuta il motore JavaScript a capire la separazione delle singole istruzioni, fondamentale per l'esecuzione corretta.

```js
let message = "Hello, World!"; // first statement ends here
let number = 42; // second statement starts here
```

- I punti e virgola aiutano a prevenire ambiguità nell'esecuzione del codice e assicurano che le istruzioni siano terminate correttamente.

## Commenti in JavaScript

- Qualsiasi riga di codice commentata viene ignorata dal motore JavaScript. I commenti servono a spiegare il codice, fare annotazioni o disabilitare temporaneamente il codice.
- I commenti su singola riga si creano usando `//`.

```js
// This is a single-line comment and will be ignored by the JavaScript engine
```

- I commenti su più righe si creano usando `/*` per iniziare il commento e `*/` per terminarlo.

```js
/*
This is a multi-line comment.
It can span multiple lines.
*/
```

## JavaScript come linguaggio a tipizzazione dinamica

- JavaScript è un linguaggio a tipizzazione dinamica, il che significa che non devi specificare il tipo di dato di una variabile quando la dichiari. Il motore JavaScript determina automaticamente il tipo di dato in base al valore assegnato alla variabile.

```js
let error = 404; // JavaScript treats error as a number
error = "Not Found"; // JavaScript now treats error as a string
```

- Altri linguaggi, come C#, che non sono a tipizzazione dinamica, genererebbero un errore:

```csharp
int error = 404; // value must always be an integer
error = "Not Found"; // This would cause an error in C#
```

## Usare l'operatore `typeof`

- L'operatore `typeof` serve a controllare il tipo di dato di una variabile. Restituisce una stringa che indica il tipo della variabile.

:::interactive_editor

```js
let age = 25;
console.log(typeof age); // "number"

let isLoggedIn = true;
console.log(typeof isLoggedIn); // "boolean"
```

:::

- Tuttavia, c'è una nota famosa in JavaScript riguardo `null`. L'operatore `typeof` restituisce `"object"` per valori `null`.

:::interactive_editor

```js
let user = null;
console.log(typeof user); // "object"
```
