# --description--

في هذا الدرس، ستتعلم كيفية العمل مع أنواع الدوال. لنبدأ بتعليقات نوع المعلمات.

إليك مثال على دالة أساسية:

```ts
function circleArea(radius) {
  return Math.PI * radius * radius;
}
```

في الوقت الحالي، سيعرض TypeScript رسالة تقول `Parameter 'radius' implicitly has an 'any' type`. يشير نوع `any` إلى أن القيمة يمكن أن تكون من أي نوع. ستتعلم المزيد عن هذا النوع في درس لاحق.

للتخلص من الخطأ وتعيين نوع المعلمة صراحةً، يمكنك تعيينه هكذا:

```ts
function circleArea(radius: number) {
  return Math.PI * radius * radius;
}
```

إذا حاولت استدعاء الدالة `circleArea` بنوع مختلف غير الرقم، سيعرض TypeScript خطأً مثل هذا:

```ts
function circleArea(radius: number) {
  return Math.PI * radius * radius;
}

// Argument of type 'string' is not assignable to parameter of type 'number'
circleArea("3");
```

إذا حاولت تقديم معلمات أكثر مما تتوقع الدالة، سيعرض TypeScript خطأً:

```ts
function circleArea(radius: number) {
  return Math.PI * radius * radius;
}

// Expected 1 arguments, but got 2.
circleArea(3, 4);
```

إذا قدمت معلمات أقل من المطلوبة، سيعرض TypeScript أيضًا خطأً:

```ts
function circleArea(radius: number) {
  return Math.PI * radius * radius;
}

// Expected 1 arguments, but got 0.
// An argument for 'radius' was not provided.
circleArea();
```

يمكنك جعل المعلمات اختيارية باستخدام رمز `?` بجانب المعلمة هكذا:

```ts
function area(radius: number, height?: number): number {
  const baseArea = Math.PI * radius * radius;

  if (height !== undefined) {
    const sideArea = 2 * Math.PI * radius * height;
    return baseArea + sideArea;
  }

  return baseArea;
}
```

تذكر أن المعلمات الاختيارية يجب أن تأتي دائمًا بعد المعلمات المطلوبة. وضع معلمة مطلوبة بعد اختيارية سيؤدي إلى خطأ في TypeScript.

التحقق الشرطي داخل الدالة موجود في حال لم يتم تقديم المعلمة `height`.

إذا أردت تعيين نوع لقيم الإرجاع، يمكنك إضافة تعليقات نوع الإرجاع هكذا:

```ts
function circleArea(radius: number): number {
  return Math.PI * radius * radius;
}
```

تعليق نوع الإرجاع يأتي بعد قائمة المعلمات. مثل تعليقات نوع المتغيرات، غالبًا ما يكون TypeScript ذكيًا بما يكفي لاستنتاج نوع الإرجاع. استخدم تعليق نوع إرجاع صريح عندما تريد توضيح نية الدالة أو منع تغييرات غير مقصودة في قيمة الإرجاع.

المثال التالي الذي سنراه يتعامل مع الدوال غير المتزامنة والوعود. إذا كانت دالتك سترجع وعدًا، يمكنك استخدام نوع `Promise` هكذا:

```ts
async function getCityTemperature(city: string): Promise<number> {
  const response = await fetch(`https://api.example.com/weather?city=${city}`);
  const data = await response.json(); 
  return data.temperature;
}
```

في هذه الحالة، إذا حاولت إزالة نوع الإرجاع، قد يستنتج TypeScript الإرجاع كـ `Promise<any>`. لذا سيكون من الأفضل تعيين نوع الإرجاع صراحةً.

حتى هذه النقطة، نظرنا فقط إلى أمثلة الدوال المسماة. لكن كيف تعمل الأنواع مع الدوال المجهولة؟

إليك مثالًا يستخدم دالة مجهولة في الاستدعاء الراجع الممرر إلى `forEach`:

```ts
const languages = ["JavaScript", "Python", "TypeScript"];

languages.forEach((lang) => {
  console.log(`${lang.toUpperCase()} has ${lang.length} characters`);
});
```

المعلمة `lang` لا تحتوي على تعليق نوع. بدلاً من ذلك، يستخدم TypeScript التعيين السياقي، مما يعني أنه يستنتج النوع من السياق الذي تظهر فيه.
