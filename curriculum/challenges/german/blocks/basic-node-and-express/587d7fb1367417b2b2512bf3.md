# --description--

Früher haben Sie die `express.static()`-Middleware-Funktion kennengelernt. Jetzt ist es an der Zeit, Middleware genauer zu betrachten. Middleware-Funktionen sind Funktionen, die 3 Argumente entgegennehmen: das Anfrageobjekt, das Antwortobjekt und die nächste Funktion im Anfrage-Antwort-Zyklus der Anwendung. Diese Funktionen führen Code aus, der Nebenwirkungen auf die Anwendung haben kann, und fügen normalerweise Informationen zu den Anfrage- oder Antwortobjekten hinzu. Sie können den Zyklus auch beenden, indem sie eine Antwort senden, wenn eine bestimmte Bedingung erfüllt ist. Wenn sie keine Antwort senden, starten sie die Ausführung der nächsten Funktion im Stapel. Dies löst den Aufruf des 3. Arguments, `next()`, aus.

Sehen Sie sich folgendes Beispiel an:

```js
function(req, res, next) {
  console.log("I'm a middleware...");
  next();
}
```

Angenommen, Sie haben diese Funktion auf einer Route eingebunden. Wenn eine Anfrage mit der Route übereinstimmt, zeigt sie den String „I’m a middleware…“ an und führt dann die nächste Funktion im Stapel aus. In dieser Aufgabe werden Sie eine Middleware auf Root-Ebene erstellen. Wie Sie in Aufgabe 4 gesehen haben, können Sie eine Middleware-Funktion auf Root-Ebene mit der `app.use(<mware-function>)`-Methode einbinden. In diesem Fall wird die Funktion für alle Anfragen ausgeführt, aber Sie können auch spezifischere Bedingungen festlegen. Wenn Sie beispielsweise möchten, dass eine Funktion nur für POST-Anfragen ausgeführt wird, könnten Sie `app.post(<mware-function>)` verwenden. Für alle HTTP-Verben (GET, DELETE, PUT, …) gibt es entsprechende Methoden.

# --instructions--

Erstellen Sie einen einfachen Logger. Für jede Anfrage soll er in der Konsole einen String im folgenden Format protokollieren: `method path - ip`. Ein Beispiel könnte so aussehen: `GET /json - ::ffff:127.0.0.1`. Beachten Sie, dass zwischen `method` und `path` ein Leerzeichen steht und dass der Bindestrich, der `path` und `ip` trennt, auf beiden Seiten von einem Leerzeichen umgeben ist. Sie können die Anfragemethode (HTTP-Verb), den relativen Routenpfad und die IP des Aufrufers aus dem Anfrageobjekt mit `req.method`, `req.path` und `req.ip` erhalten. Denken Sie daran, `next()` aufzurufen, wenn Sie fertig sind, sonst bleibt Ihr Server für immer hängen. Öffnen Sie unbedingt die „Logs“ und beobachten Sie, was passiert, wenn eine Anfrage eingeht.

**Hinweis:** Express wertet Funktionen in der Reihenfolge aus, in der sie im Code erscheinen. Das gilt auch für Middleware. Wenn sie für alle Routen funktionieren soll, muss sie vor diesen eingebunden werden.
