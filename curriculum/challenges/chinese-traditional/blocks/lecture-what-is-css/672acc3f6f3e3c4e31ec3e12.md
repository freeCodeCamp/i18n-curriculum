# --interactive--

在使用 CSS 時，你經常會遇到三種不同的元素顯示行為型別：`inline`、`block` 和 `inline-block`。

這些屬性中的每一個都會影響元素的位置以及它們與頁面上其他元素的互動方式。

在本課程中，我們將專注於 `inline-block` 屬性如何運作，以及它與行內和區塊層級元素的不同之處。

區塊層級元素的行為就像大型容器或「區塊」，會佔據其父容器的整個寬度。它們總是從新的一行開始，且其高度和寬度可以調整。

行內元素只佔用它們所需的空間。它們在周圍內容中流動，且不會換行。

`inline-block` 屬性是這兩種行為的混合體。像行內元素一樣，`inline-block` 元素保持在文字流程中，且不會另起新行。

然而，與行內元素不同，你可以像調整區塊層級元素一樣調整 `inline-block` 元素的寬度和高度。

簡而言之，`inline` 和 `inline-block` 之間的主要差異在於 `inline` 元素無法控制其大小，而 `inline-block` 元素則允許完全控制尺寸，同時仍與其他內容保持行內排列。

讓我們來看一個範例。

:::interactive_editor

```html
<link href="styles.css" rel="stylesheet">

<div class="container">
  <span class="inline-block-element element1">Inline-Block</span>
  <span class="inline-block-element element2">Inline-Block</span>
</div>
```

```css
.inline-block-element {
  display: inline-block;
  width: 150px;
  height: 100px;
}

.element1 {
  background-color: lightblue;
}

.element2 {
  background-color: lightgreen;
}
```

:::

在上述範例中，我們有一個 `div`，其類別為 `container`。在該 `div` 元素內，我們有兩個 `span` 元素。

每個 span 元素都設定為 `display:inline-block`，並且也設定了寬度和高度。

行內區塊元素會並排出現，因為它們的行為像行內元素，但它們也有指定的寬度和高度，這使它們具有區塊般的特性。

但是，如果你移除 `display: inline-block` 屬性，即使你明確定義了，`height` 和 `width` 也都不會被套用。

這是修訂後的 CSS：

:::interactive_editor

```html
<link href="styles.css" rel="stylesheet">

<div class="container">
  <span class="inline-block-element element1">Span element</span>
  <span class="inline-block-element element2">Span element</span>
</div>
```

```css
.inline-block-element {
  width: 150px;
  height: 100px;
}

.element1 {
  background-color: lightblue;
}

.element2 {
  background-color: lightgreen;
}
```

:::

在這段程式碼中，我們移除了 `display: inline-block;` 屬性，但其他部分保持不變。在這裡，`span` 元素恢復為其預設的行內元素行為。

因此，指定的寬度和高度會被忽略，元素僅佔用其內容所需的空間。

了解 `inline-block` 的運作方式很有用，因為你可以用它來建立需要在連續文字流程中同時控制對齊和尺寸的版面配置。
