# --description--

Ще один спосіб представити граф — це розмістити його у <dfn>матриці суміжності</dfn>. <dfn>Матриця суміжності</dfn> — це двовимірний (2D) масив, де кожен вкладений масив має таку ж кількість елементів, як і зовнішній масив. Іншими словами, це матриця або сітка чисел, де числа представляють ребра.

**Примітка**: Числа зверху та зліва від матриці — це просто мітки для вузлів. Усередині матриці одиниці означають, що існує ребро між вершинами (вузлами), які представляють рядок і стовпець. Нарешті, нулі означають, що ребра або зв’язку немає.

<pre>
    1 2 3
  \------
1 | 0 1 1
2 | 1 0 0
3 | 1 0 0
</pre>

Вгорі наведено дуже простий неорієнтований граф, де є три вузли, причому перший вузол з’єднаний із другим і третім вузлом. Нижче наведена реалізація того самого на JavaScript.

```js
var adjMat = [
  [0, 1, 1],
  [1, 0, 0],
  [1, 0, 0]
];
```

На відміну від списку суміжності, кожен «рядок» матриці має містити таку ж кількість елементів, як і вузлів у графі. Тут у нас матриця 3 на 3, що означає, що в нашому графі три вузли. Орієнтований граф виглядав би подібно. Нижче наведено граф, де перший вузол має ребро, спрямоване до другого вузла, а другий вузол має ребро, спрямоване до третього вузла.

```js
var adjMatDirected = [
  [0, 1, 0],
  [0, 0, 1],
  [0, 0, 0]
];
```

Графи також можуть мати <dfn>ваги</dfn> на своїх ребрах. Поки що ми розглядали <dfn>невзвешені</dfn> ребра, де наявність або відсутність ребра є бінарною (`0` або `1`). Ви можете мати різні ваги залежно від вашого застосунку.

# --instructions--

Створіть матрицю суміжності неорієнтованого графа з п’ятьма вузлами. Ця матриця має бути у вигляді багатовимірного масиву. Ці п’ять вузлів мають зв’язки між першим і четвертим вузлом, першим і третім вузлом, третім і п’ятим вузлом, а також четвертим і п’ятим вузлом. Всі ваги ребер дорівнюють одному.
