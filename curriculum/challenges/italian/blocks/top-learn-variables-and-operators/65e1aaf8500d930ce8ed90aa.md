# --description--

Il più `+` esiste in due forme: la forma binaria che hai usato nell’ultima sfida e la forma unaria.

Il più unario o, detto in altri termini, l’operatore più `+` applicato a un singolo valore, non fa nulla ai numeri. Ma se l’operando non è un numero, il più unario lo converte in numero.

Per esempio:

```js
// No effect on numbers
let x = 1;
console.log( +x ); // 1

let y = -2;
console.log( +y ); // -2

// Converts non-numbers
console.log( +true ); // 1
console.log( +"" );   //
```

La necessità di convertire stringhe in numeri si presenta molto spesso. Per esempio, se prendi valori da campi modulo HTML, di solito sono stringhe. Cosa succede se vuoi sommarli?

Il più binario li aggiungerebbe come stringhe:

```js
let apples = "2";
let oranges = "3";

alert( apples + oranges ); // "23", the binary plus concatenates strings
```

Se vuoi trattarli come numeri, devi convertirli e poi sommarli:

```js
let apples = "2";
let oranges = "3";

// both values converted to numbers before the binary plus
alert( +apples + +oranges ); // 5

// the longer variant
// alert( Number(apples) + Number(oranges) ); // 5
```

Dal punto di vista di un matematico, l’abbondanza di più può sembrare strana. Ma dal punto di vista di un programmatore, non c’è nulla di speciale: i più unari si applicano prima, convertono le stringhe in numeri, e poi il più binario li somma.

Perché i più unari si applicano ai valori prima di quelli binari? Come vedrai, è a causa della loro precedenza _più alta_.

# --questions--

## --text--
Dato quanto spiegato sugli operatori `+` unari e binari in JavaScript, come puoi sommare correttamente i valori stringa `'2'` e `'3'` come numeri usando il più `+` unario?

## --answers--

`console.log('2' + '3'); // Outputs: '23'` 

---

`console.log(+2 + +3); // Outputs: 5`

---

`console.log(+'2' + +'3'); // Outputs: 5`

---

`console.log(Number('2') + Number('3')); // Outputs: '23'`


## --video-solution--

3
