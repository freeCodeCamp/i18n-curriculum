# --interactive--

JavaScript는 때때로 놀랍거나 심지어 이상하게 작동하는 언어입니다. 그런 놀라움 중 하나는 숫자와 문자열을 계산에 섞을 때 발생합니다. 아마도 가장 먼저 시도할 것은 숫자와 문자열을 더하는 것입니다. JavaScript에서 `+` 연산자는 두 가지 역할을 합니다. 덧셈과 문자열 연결을 모두 처리하는데, 문자열 연결은 두 문자열을 합치는 방법입니다.

숫자와 문자열에 `+`을 사용하면 JavaScript는 둘 다 문자열로 취급하여 합칩니다. `typeof` 연산자로 결과의 타입을 확인하면 실제로 문자열임을 알 수 있습니다:

:::interactive_editor

```js
const result = 5 + '10';

console.log(result); // 510
console.log(typeof result); // string
```

:::

`5`과 `'10'`의 순서를 바꾸면 어떤 일이 일어날까요?

:::interactive_editor

```js
const result = '10' + 5;

console.log(result); // 105
console.log(typeof result); // string
```

:::

같은 일이 일어나는 것을 볼 수 있습니다. JavaScript는 `'10'`에서 문자열을, `5`에서 숫자를 보고 숫자를 문자열로 변환하여 연결합니다. 이것을 타입 강제 변환이라고 합니다. 타입 강제 변환은 한 데이터 타입의 값을 다른 타입으로 변환하는 것입니다.

문자열과 숫자로 뺄셈, 곱셈, 나눗셈 같은 다른 산술 연산을 시도하면 상황이 더 흥미로워집니다. 이런 경우 JavaScript는 수학 연산을 하기 전에 문자열을 숫자로 변환하려고 합니다 — 이것도 타입 강제 변환입니다! 결과는 다음과 같습니다:

:::interactive_editor

```js
const subtractionResult = '10' - 5;
console.log(subtractionResult); // 5
console.log(typeof subtractionResult); // number

const multiplicationResult = '10' * 2;
console.log(multiplicationResult); // 20
console.log(typeof multiplicationResult); // number

const divisionResult = '20' / 2;
console.log(divisionResult); // 10
console.log(typeof divisionResult); // number
```

:::

위 예제에서 JavaScript는 문자열 `'10'` 또는 `'20'`을 숫자로 성공적으로 변환한 뒤 계산을 수행합니다. 결과적으로 `'10' - 5`는 `5`을, `'10' * 2`은 `20`를, `'20' / 2`은 `10`를 산출합니다.

그런데 문자열이 숫자처럼 보이지 않는다면 어떻게 될까요? 그 경우를 살펴봅시다:

:::interactive_editor

```js
const subtractionResult = 'abc' - 5;
console.log(subtractionResult); // NaN
console.log(typeof subtractionResult); // number

const multiplicationResult = 'abc' * 2;
console.log(multiplicationResult); // NaN
console.log(typeof multiplicationResult); // number

const divisionResult = 'abc' / 2;
console.log(divisionResult); // NaN
console.log(typeof divisionResult); // number
```

:::

위 예제에서 문자열 `'abc'`는 유효한 숫자 값을 나타내지 않으므로 JavaScript가 의미 있는 숫자로 변환할 수 없습니다. 변환에 실패하면 JavaScript는 `NaN`을 반환하는데, 이는 "숫자가 아님"을 뜻합니다. `NaN`은 유효하지 않거나 표현할 수 없는 숫자를 나타내는 `Number` 타입의 특별한 값입니다.

불리언(`true` 또는 `false`)으로 산술 연산을 하면 어떻게 될까요? 살펴봅시다. JavaScript는 수학 연산에서 불리언을 숫자로 취급합니다: `true`은 `1`가 되고, `false`은 `0`가 됩니다.

:::interactive_editor

```js
const result1 = true + 1;
console.log(result1); // 2
console.log(typeof result1); // number

const result2 = false + 1;
console.log(result2); // 1
console.log(typeof result2); // number

const result3 = 'Hello' + true;
console.log(result3); // "Hellotrue"
console.log(typeof result3); // string
```

:::

처음 두 예제에서 `true + 1`는 `2`이 되었고, `false + 1`는 `1`이 되었습니다. 세 번째 예제 `'Hello' + true`에서는 JavaScript가 `true`을 문자열로 변환하여 `'Hello'`과 연결해 `'Hellotrue'`라는 문자열을 만들었습니다.

`null`과 `undefined`에 대해 JavaScript는 수학 연산에서 `null`를 `0`로, `undefined`을 `NaN`로 취급합니다:

:::interactive_editor

```js
const result1 = null + 5;
console.log(result1); // 5
console.log(typeof result1); // number

const result2 = undefined + 5;
console.log(result2); // NaN
console.log(typeof result2); // number
```

:::

JavaScript는 종종 타입 강제 변환을 수행하여 숫자, 문자열, 불리언 같은 데이터 타입을 때로는 예상치 못한 방식으로 자동 변환합니다. 이러한 변환을 이해하는 것은 버그를 피하고 프로젝트에서 견고한 코드를 작성하는 데 매우 중요합니다.
