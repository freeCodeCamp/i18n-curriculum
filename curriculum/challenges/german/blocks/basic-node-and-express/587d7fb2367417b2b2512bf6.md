# --description--

Eine weitere gängige Methode, um Eingaben vom Client zu erhalten, besteht darin, die Daten nach dem Routenpfad in einem Query-String zu kodieren. Der Query-String wird durch ein Fragezeichen (?) abgegrenzt und enthält Feld=Wert-Paare. Jedes Paar wird durch ein kaufmännisches Und (&) getrennt. Express kann die Daten aus dem Query-String auslesen und das Objekt `req.query` befüllen. Einige Zeichen, wie das Prozentzeichen (%), dürfen nicht in URLs vorkommen und müssen vor dem Senden in einem anderen Format kodiert werden. Wenn Sie die API von JavaScript verwenden, können Sie spezielle Methoden nutzen, um diese Zeichen zu kodieren oder zu dekodieren.

<blockquote>route_path: '/library'<br>actual_request_URL: '/library?userId=546&#x26;bookId=6754' <br>req.query: {userId: '546', bookId: '6754'}</blockquote>

# --instructions--

Erstellen Sie einen API-Endpunkt, der unter `GET /name` eingebunden ist. Antworten Sie mit einem JSON-Dokument, das die Struktur `{ name: 'firstname lastname'}` hat. Die Parameter für Vor- und Nachname sollten in einem Query-String kodiert sein, z. B. `?first=firstname&last=lastname`.

**Hinweis:** In der folgenden Aufgabe erhalten Sie Daten von einer POST-Anfrage unter demselben Routenpfad `/name`. Falls Sie möchten, können Sie die Methode `app.route(path).get(handler).post(handler)` verwenden. Diese Syntax erlaubt es Ihnen, verschiedene Verb-Handler auf demselben Routenpfad zu verketten. So sparen Sie etwas Tipparbeit und erhalten saubereren Code.

# --hints--

Test 1 : Ihr API-Endpunkt sollte mit `{ "name": "Mick Jagger" }` antworten, wenn der `/name` Endpunkt mit `?first=Mick&last=Jagger` aufgerufen wird

```js
  const response = await fetch(code + '/name?first=Mick&last=Jagger');
  if (!response.ok) {
    throw new Error(await response.text());
  }
  const data = await response.json();
  assert.equal(
    data.name,
    'Mick Jagger',
    'Test 1: "GET /name" route does not behave as expected'
  );
```
