# --description--

在 Python 中，异常处理是编写健壮且容错程序的核心部分。它允许你以结构化的方式预见、捕获和响应误差。

异常处理是在程序执行过程中捕获和管理发生的误差的过程，以防止你的代码意外崩溃。

Python 提供了 `try`、`except`、`else` 和 `finally` 块来优雅地处理误差。以下是一个基本示例：

```py
try:
    x = 10 / 0
except ZeroDivisionError:
    print("You can't divide by zero!")
```

- `try`：你预期可能发生误差的代码块。

- `except`：如果在 try 内引起指定类型的误差，则此块运行。

- 在这个分支中，除以零会引起 `ZeroDivisionError`，然后被捕获并处理。

下面是一个示例，展示如何使用 `else` 和 `finally` 块：

```py
try:
    x = 10 / 2
except ZeroDivisionError:
    print("You can't divide by zero!")
else:
    print('Division successful:', x)
finally:
    print('This block always runs.')
```

- `else`：如果在 try 块中没有引起异常，则运行。

- `finally`：无论是否发生异常，都会运行。适用于关闭文件或释放资源等清理任务。

你也可以使用多个单独的 `except` 块来捕获多个异常：

```py
try:
    number = int('abc')
    result = 10 / number
except ValueError:
    print('That was not a valid number.')
except ZeroDivisionError:
    print("Can't divide by zero.")
```

通过使用单独的 `except` 子句，你可以使你的错误响应更加具体和有用。

你也可以使用异常对象，通常使用 `as` 关键字将其别名为另一个名称。这里我们使用 `e` 作为误差对象的别名：

```py
try:
    x = 1 / 0
except ZeroDivisionError as e:
    print(f'Error occurred: {e}')
```

使用 `e` 让你访问实际的错误信息或对象以进行日志记录或调试。

你也可以通过将异常指定为一个元组，在单个 `except` 子句中捕获多个异常：

```py
try:
    number = int(input('Enter a number: '))
    result = 10 / number
except (ValueError, ZeroDivisionError) as e:
    print(f'Error occurred: {e}')
```

异常处理允许你的程序优雅地从误差中恢复。通过使用 `try`、`except`、`else` 和 `finally`，你可以预见潜在问题并构建更具弹性的应用。
