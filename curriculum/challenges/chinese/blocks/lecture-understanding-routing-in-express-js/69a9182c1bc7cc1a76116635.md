# --description--

在 Express 中，**路由路径** 是决定服务器如何响应传入请求的 URL 模式。当用户请求特定 URL 时，Express 会将该 URL 与路由路径匹配，并运行相应的处理程序。

路由路径定义在诸如 `app.get()`、`app.post()` 等路由方法中。这些路径可以是静态的，比如 `/home`，也可以是动态的，使用参数从 URL 中捕捉值，比如 `/user/:id`。

本课我们将重点讲解静态路由。下面是一个示例：

```javascript
// Static route
app.get("/home", (req, res) => {
  res.send("Welcome to the homepage!");
});
```

在这个例子中，服务器监听对 `/home` 的 GET 请求，并返回一个简单的消息。

`app.get()` 方法接受两个参数：

* 第一个参数 `"/home"` 是服务器监听的静态路由路径（或 URL）。
    
* 第二个参数是一个回调函数，当对该路径发起 GET 请求时运行。
    

在回调函数内部，`res.send()` 向客户发送响应。在此例中，它发送了文本 `Welcome to the homepage!`，该文本会显示在浏览器中。

这是另一个静态路由的例子：

```javascript
// Static route
app.get("/menu/drinks", (req, res) => {
  res.send("Welcome to the drinks menu!");
});
```

在这种情况下，服务器监听对 `/menu/drinks` URL 的 GET 请求，并在请求该路由时发送消息。

当你的网站有多个相关页面归类在一个公共路径下时，可能会使用这样的路由。例如，一个餐厅网站可以这样组织它的菜单页面：

* `/menu/food`
    
* `/menu/drinks`
    
* `/menu/desserts`
    

这种方式有助于保持你的 URL 清晰且有意义，同时也让你的路由更容易组织和维护，随着你的应用增长。

下一课我们将学习带参数的动态路由路径。
