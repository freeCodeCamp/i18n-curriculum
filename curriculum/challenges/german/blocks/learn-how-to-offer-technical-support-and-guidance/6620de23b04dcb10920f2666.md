## --text--

Warum betont Sarah die Einbindung von Kommentaren im Code?

## --answers--

Weil es den Code schneller ausführen lässt

### --feedback--

Kommentare im Code beeinflussen nicht die Geschwindigkeit, mit der der Code ausgeführt wird. Sie dienen dem besseren Verständnis für Menschen.

---

Um zukünftigen Entwicklern zu helfen, die Logik zu verstehen

---

Weil Kommentare in allen Programmiersprachen vorgeschrieben sind

### --feedback--

Obwohl Kommentare hilfreich sind, sind sie nicht in allen Programmiersprachen eine strikte Vorgabe.

---

Um zu verhindern, dass der Code Fehler enthält

### --feedback--

Kommentare helfen beim Verstehen des Codes, verhindern aber nicht direkt Fehler im Code.
