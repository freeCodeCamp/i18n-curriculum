# --description--

ウェブサイトを訪れたりAPIを使ったりするとき、ブラウザーとサーバーは常にやり取りをしています。

その会話の中で最も重要な部分の一つがHTTPレスポンスステータスコードです。

これは200、404、500のような短い数字のメッセージで、クライアント（ノートパソコンやスマホなどのエンドユーザーのデバイス）にリクエストの結果を伝えます。

すでにいくつか学びましたが、カテゴリーごとに分けて、それぞれの意味や使いどころを正確に思い出せるようにしましょう。

## 1xx – 情報レスポンス

1で始まるコードは日常の開発ではあまり見かけませんが、リクエストが受け取られ処理が続いていることを示します。例えばファイルをアップロードしているときなどです。

よくある例：

* 100 Continue – サーバーがクライアントに「リクエストの残りを送っていいですよ」と伝えます。
* 101 Switching Protocols – クライアントが`Upgrade`ヘッダーを送ったため、サーバーがプロトコルを切り替えています。Websocketsを使うときによく見られます。

## 2xx – 成功レスポンス

リクエストが正常に受け取られ、理解され、処理されたことを意味します。

* 200 OK – リクエストが成功しました。最も一般的な成功レスポンスです。
* 201 Created – 新しいリソースが正常に作成されました。例えば新規ユーザー登録後などです。
* 204 No Content – リクエストは成功しましたが、返すレスポンスボディがありません。`DELETE`リクエストでよく使われます。

## 3xx – リダイレクトメッセージ

クライアントに「探しているリソースは別の場所にあります」と伝えます。

* 301 Moved Permanently – リソースのURLが恒久的に変更されました。
* 302 Found – リソースは一時的に別のURLにあります。
* 304 Not Modified – キャッシュ用です。クライアントに変更がないのでキャッシュを使っていいと伝えます。

## 4xx – クライアントエラー

クライアントが無効なリクエストを送ったときに起こります。

* 400 Bad Request – リクエストが不正または不適切です。
* 401 Unauthorized – 認証が必要です（ログインが必要）。
* 403 Forbidden – 認証は成功しましたが、権限がありません。
* 404 Not Found – リクエストされたリソースは存在しません。ウェブで最も象徴的なエラーです。

## 5xx – サーバーエラー

サーバー側で何か問題が発生したことを意味します。

* 500 Internal Server Error – 一般的な「何か問題が起きました」というメッセージです。
* 502 Bad Gateway – プロキシとして動作するサーバーが別のサーバーから無効なレスポンスを受け取りました。
* 503 Service Unavailable – サーバーが一時的に過負荷かメンテナンス中です。

これで一般的なHTTPステータスコードに慣れたので、Expressでの実用例を見てみましょう。

APIを作っているとします。ユーザーが無効なデータを送った場合、400ステータスコード「Bad Request」で応答できます。クライアントに400の意味がわかるように文字列「Bad Request」も返せます。

```js
res.status(400).send("Bad Request");
```

すべてがうまくいった場合は200レスポンスを送れます。メッセージを含めることもできます。ここではシンプルな「Success」メッセージをJSONで送っています。

```js
res.status(200).json({ message: "Success" });
```

サーバーで何かがクラッシュした場合は500ステータスコードを送り、内部サーバーエラーが起きたことを明確に伝えるメッセージも送れます。

```js
res.status(500).send("Internal Server Error");
```

これらのステータスコードは機械だけでなく、開発者がバグをデバッグしたり、ユーザーや他の開発者が状況を理解したりするのに役立ちます。
