# --description--

값의 타입이 넓을 때 더 구체적인 타입으로 좁혀야 하는 경우가 있습니다. 예를 들어, 객체가 정의한 인터페이스와 일치하는지 확인하거나 문자열이 특정 값 목록에 포함되는지 확인해야 할 수 있습니다. 이를 달성하는 방법은 여러 가지가 있습니다.

첫 번째는 참 같은 값으로 좁히기입니다. 지난 수업의 예제를 살펴보세요:

```js
const email = document.querySelector<HTMLInputElement>("#email");
console.log(email.value);
```

`value`의 `email` 속성에 접근하려고 하면 `email`가 `null`일 수 있기 때문에 컴파일러 오류가 발생합니다. 하지만 조건문을 사용해 `email`가 참 같은 값인지 확인한 후 속성에 접근할 수 있습니다:

```js
const email = document.querySelector<HTMLInputElement>("#email");
if (email) {
  console.log(email.value);
}
```

이 업데이트된 예제에서 `null`가 참 같은 값이 아니므로 TypeScript는 조건문 블록 내에서 `email`가 반드시 `input` 요소임을 추론할 수 있습니다. 그래서 더 이상 컴파일러 오류가 발생하지 않습니다.

참 같은 값 확인은 반대 방향으로도 작동할 수 있습니다:

```js
const email = document.querySelector<HTMLInputElement>("#email");
if (!email) {
  throw new ReferenceError("Could not find email element!")
}
console.log(email.value);
```

이 방법에서는 `email`가 거짓 같은 값이면 오류를 발생시킵니다. `null`는 거짓 같은 값입니다. 오류를 발생시키면 이 코드의 논리 실행이 종료되므로 `console.log()` 호출에 도달했을 때 TypeScript는 `email`가 `null`일 수 없다는 것을 알게 됩니다.

선택적 체이닝도 `email` 값이 `null`일 경우 속성 접근이 불가능하다는 전제 하에 타입 좁히기의 한 형태입니다.

```js
const email = document.querySelector<HTMLInputElement>("#email");
console.log(email?.value);
```

그렇다면 다른 타입은 어떻게 할까요? `typeof` 연산자를 사용해 타입을 좁힐 수도 있습니다. 문자열 또는 숫자일 수 있다고 표시한 변수 예제를 살펴보겠습니다:

```js
const myVal = Math.random() > 0.5 ? 222 : "222";
console.log(myVal / 10)
```

이 예제에서는 문자열 값에 산술 연산을 수행할 수 없기 때문에 컴파일러 오류가 발생합니다. 하지만 `typeof` 변수의 `myVal`을 조건문으로 확인할 수 있습니다:

```js
const myVal = Math.random() > 0.5 ? 222 : "222";
if (typeof myVal === "number") {
  console.log(myVal / 10);
}
```

`typeof` 키워드를 사용했기 때문에 TypeScript는 `myVal`가 숫자임을 알게 되어 안전하게 산술 연산을 수행할 수 있습니다.

그렇다면 더 복잡한 객체 타입은 어떻게 할까요? 해당 객체가 클래스에서 온 경우 `instanceof` 키워드를 사용해 타입을 좁힐 수 있습니다. `querySelector()` 예제로 돌아가 보겠습니다:

```js
const email = document.querySelector("#email");
```

일반 타입을 전달해 TypeScript에 요소가 무엇인지 알려주는 대신 `instanceof`를 사용해 타입을 좁히고 더 안전한 코드를 작성할 수 있습니다:

```js
const email = document.querySelector("#email");

if (email instanceof HTMLInputElement) {
    console.log(email.value);
}
```

이 방법은 이전 방법과 비슷해 보이지만 `instanceof`는 런타임 유효성 검사입니다. 즉, TypeScript 타입을 잘못 지정했더라도 JavaScript 코드가 `email`가 `input` 요소임을 확인합니다.

다음으로 API에서 `User` 객체를 가져와 정보를 출력하려는 예제를 살펴보겠습니다:

```js
interface User {
    name: string;
    age: number;
}

const printAge = (user: User) => 
  console.log(`${user.name} is ${user.age} years old!`)

const request = await fetch("url")
const myUser = await request.json();
printAge(myUser);
```

API가 올바른 객체를 반환한다는 것을 알더라도 `myUser`를 함수에 전달하려고 하면 컴파일러 오류가 발생합니다. 그리고 `.json()` 메서드는 제네릭 타입을 받지 않습니다.

이 문제를 해결하는 "쉬운" 방법은 타입을 캐스팅하는 것입니다:

```js
interface User {
    name: string;
    age: number;
}

const printAge = (user: User) => 
  console.log(`${user.name} is ${user.age} years old!`)

const request = await fetch("url")
const myUser = await request.json() as User;
printAge(myUser);
```

하지만 타입을 캐스팅하면 TypeScript가 잠재적 오류를 잡는 능력이 약해집니다. 그래서 타입을 캐스팅하는 대신 타입 가드를 작성할 수 있습니다:

```js
interface User {
    name: string;
    age: number;
}

const isValidUser = (user: unknown): user is User => {
  return !!user && 
    typeof user === "object" &&
    "name" in user &&
    "age" in user;
}
```

이 함수 정의에서 반환 타입이 핵심 요소입니다. `user is User` 구문은 함수가 불리언 값을 반환하며, `true`일 때 `user` 값이 `User` 인터페이스를 만족한다는 것을 나타냅니다. 그런 다음 `user` 객체 구조가 일치하는지 기본 검사를 수행합니다 — 참 같은 값(`!!user`) 좁히기와 `typeof` 좁히기를 사용한 점에 주목하세요. `typeof null`가 `"object"`를 반환하기 때문에 반드시 이렇게 해야 합니다:

```js
interface User {
    name: string;
    age: number;
}

const isValidUser = (user: unknown): user is User => {
  return !!user && 
    typeof user === "object" &&
    "name" in user &&
    "age" in user;
}

const printAge = (user: User) => 
  console.log(`${user.name} is ${user.age} years old!`)

const request = await fetch("url")
const myUser = await request.json() as User;
if (isValidUser(myUser)) {
    printAge(myUser);
}
```

이제 모든 논리를 결합하면 더 이상 컴파일러 오류가 발생하지 않고 코드를 성공적으로 빌드할 수 있습니다.

타입 좁히기는 더 안전하고 오류가 적은 코드를 작성하는 데 도움이 되는 강력한 기능입니다. 하지만 TypeScript 타입이 완전히 엄격하지 않으므로 타입을 좁히지 않고 값의 타입을 캐스팅하는 등의 관행은 피하세요.
