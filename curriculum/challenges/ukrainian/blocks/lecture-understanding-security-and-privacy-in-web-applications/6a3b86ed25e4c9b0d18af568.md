# --description--

Файл `.env` і бібліотека `dotenv` — це інструменти, які використовують у сучасній розробці програмного забезпечення для відокремлення конфіденційних налаштувань від коду застосунку.

Файл `.env` — це текстовий файл, розміщений у корені директорії проєкту, який зберігає налаштування конфігурації та секрети у вигляді пар ключ-значення.

Замість того, щоб жорстко прописувати API-ключі, паролі від бази даних або номери портів безпосередньо у вихідному коді і ризикувати їх розкриттям, якщо код буде завантажено у публічний репозиторій, наприклад GitHub, розробники зберігають їх локально у цьому файлі.

Структура файлу `.env` дуже проста:

```bash
PORT=3000
DB_HOST=localhost
API_KEY=xyz123secretkey
```

Оскільки цей файл містить дуже конфіденційні ключі, звично додавати `.env` до файлу `.gitignore` проєкту, щоб Git ніколи не відслідковував його.

Пакет `dotenv` — це легкий модуль, який з’єднує файл `.env` із вашим застосунком.

Коли ваша програма запускається, `dotenv` читає файл `.env`, аналізує ключі і вводить їх у середовище виконання. Наприклад, у застосунку Node.js ви ініціалізуєте цей пакет на самому початку вхідного файлу. Після завантаження змінні стають доступними глобально через об’єкт `process.env`.

```javascript
const express = require('express');
const app = express();

// initialize and configure dotenv
require('dotenv').config();

// access the variables securely from the environment
const port = process.env.PORT || 5000;
const apiKey = process.env.API_KEY;

app.get('/', (req, res) => {
  res.send(`Connected successfully using API Key: ${apiKey}`);
});

app.listen(port, () => {
  console.log(`Server running on port ${port}`);
});
```

Таке налаштування дає дві основні переваги. По-перше, воно підвищує безпеку, тримаючи приватні облікові дані поза вихідним кодом. По-друге, забезпечує гнучкість, дозволяючи одному й тому самому коду поводитися по-різному у середовищах розробки, тестування та продакшену просто шляхом заміни локального файлу `.env` на відповідному хості.
