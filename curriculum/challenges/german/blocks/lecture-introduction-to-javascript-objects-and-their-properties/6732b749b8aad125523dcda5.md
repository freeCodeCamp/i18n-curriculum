# --interactive--

In JavaScript sind Funktionen und Objektmethoden beide Möglichkeiten, wiederverwendbaren Code zu kapseln, aber sie unterscheiden sich in der Art und Weise, wie sie definiert, verwendet werden und in welchem Kontext sie arbeiten. Diese Unterschiede zu verstehen, ist entscheidend, um effektiven und gut organisierten JavaScript-Code zu schreiben.

Wie Sie in früheren Modulen gelernt haben, sind Funktionen wiederverwendbare Codeblöcke, die eine bestimmte Aufgabe ausführen:

:::interactive_editor

```js
function greet(name) {
    return "Hello, " + name + "!";
}
console.log(greet("Alice")); // "Hello, Alice!"
```

:::

Objektmethoden hingegen sind Funktionen, die mit einem Objekt verknüpft sind. Sie werden als Eigenschaften eines Objekts definiert und können auf die Daten des Objekts zugreifen und diese manipulieren. Hier ist ein Beispiel für ein Objekt mit einer Methode:

:::interactive_editor

```js
const person = {
    name: "Bob",
    age: 30,
    sayHello: function() {
        return "Hello, my name is " + this.name;
    }
};

console.log(person.sayHello()); // "Hello, my name is Bob"
```

:::

In diesem Beispiel ist `sayHello` eine Methode des `person`-Objekts. Das Schlüsselwort `this` ermöglicht es der `sayHello`-Methode, auf die Eigenschaften des Objekts mit dem Namen `person` zuzugreifen. Mehr über das Schlüsselwort `this` lernen Sie in zukünftigen Lektionen.

Ein Unterschied zwischen Funktionen und Methoden ist, wie sie aufgerufen werden. Funktionen werden durch ihren Namen aufgerufen, während Methoden mit Punktnotation auf dem Objekt aufgerufen werden, zu dem sie gehören. Zum Beispiel rufen wir die `greet`-Funktion als `greet("Alice")` auf, aber die `sayHello`-Methode als `person.sayHello()`.

Ein weiterer wichtiger Unterschied ist der Kontext, in dem sie arbeiten. Reguläre Funktionen haben ihren eigenen Gültigkeitsbereich, aber sie haben keine eingebaute Referenz auf ein bestimmtes Objekt. Methoden hingegen sind an ihr Objekt gebunden und können mit dem Schlüsselwort `this` auf dessen Eigenschaften und andere Methoden zugreifen.

Ein wichtiger Punkt ist, dass Methoden helfen, Code in logische Objekte zu organisieren, während Funktionen für allgemeineren, wiederverwendbaren Code verwendet werden.

Das Verständnis des Unterschieds zwischen Funktionen und Objektmethoden ist wichtig, wenn Sie in Ihrer JavaScript-Reise vorankommen. Obwohl sie anfangs ähnlich erscheinen, hilft Ihnen die Unterscheidung, wann Sie welche verwenden, dabei, organisierteren, effizienteren und besseren Code zu schreiben.
