# --interactive--

在 JavaScript 中，建構式是一種用來創建和初始化物件的特殊函式。它是使用 `new` 關鍵字喚起，並且可以初始化新建立物件上的屬性和方法。

在本課程中，我們將探討如何使用 `Object()` 建構式。`Object()` 建構式會創建一個新的空物件。以下是一個範例：

```js
new Object()
```

當你呼叫 `new Object()`，它會傳回一個可用來儲存值的新物件。

`Object()` 建構式可以搭配或不搭配 `new` 關鍵字使用。當以函式形式呼叫且不使用 `new` 時，其行為會依傳入值的型別而有所不同。以下是未使用 `new` 關鍵字呼叫 `Object()` 建構式的範例：

:::interactive_editor

```js
const num = 42;
const numObj = Object(num); // Creates an object wrapper for the number

console.log(numObj);
console.log(typeof numObj); // "object"
```

:::

如你在第二個 `console.log` 中所見，`numObj` 是一個物件。這是因為我們使用了 `Object()` 建構式將該數字輸入轉換成物件。 

如果我們嘗試將 `null` 或 `undefined` 傳遞給 `Object()` 建構式，會發生什麼情況？

:::interactive_editor

```js
const newObj = new Object(undefined);
console.log(newObj); // {}
```

:::

結果將會是一個空的物件。另一個使用 `Object()` 建構式的情境是當你處理一個未知型別的值，並且你需要確保它是物件。讓我們來看看以下範例：

:::interactive_editor

```js
function toObject(value) {
  if (value === null || value === undefined) {
    return {};
  }

  if (typeof value === "object") {
    return value;
  }

  return Object(value);
}

console.log(toObject(null));

console.log(toObject(true));

console.log(toObject([1, 2, 3]));
```

:::

在這個範例中，我們有一個名為 `toObject` 的函式。第二個條件會檢查該值是否為物件型別，且如果條件為 `true`，將傳回該值。此條件會同時檢查物件以及陣列，因為陣列是特殊型別的物件。

如果兩個條件都不成立，該函式會傳回 `Object(value)`，它會將輸入轉換成物件。這適用於數字、字串和布林值等值。

大多數時候你不會使用 `Object()` 建構式來建立新的物件，因為你會改用物件字面值語法（例如，`const objectLiteral = { name: "Beau" }`）。但了解使用 Object 建構式的基本概念仍然很重要。
