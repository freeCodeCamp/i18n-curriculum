# --description--

المكوِّنات هي اللبنات الأساسية لتطبيقات React التي تتيح للمطوِّرين تقسيم واجهات المستخدم المعقدة إلى أجزاء أصغر يمكن إدارتها، مما يسهل تطوير وصيانة التطبيقات واسعة النطاق.

نوعا المكوِّنات في React هما المكوِّنات الوظيفية والمكوِّنات القائمة على الأصناف. في React الحديثة، يستخدم المطوِّرون المكوِّنات الوظيفية، وجميع الأمثلة التي سنراها اليوم ستكون مكوِّنات وظيفية.

على مستوى أعلى، يمكنك التفكير في المكوِّنات كدوال JavaScript تُعيد عناصر تصف واجهة المستخدم.

تُوصف هذه الواجهة باستخدام JSX، وهو امتداد صيغة نحوية لـ JavaScript يشبه HTML لكنه يسمح لك بكتابة عناصر واجهة المستخدم بطريقة أكثر تعبيرًا.

لننظر إلى مثال على مكوِّن React:

```jsx
function Greeting() {
  const name = "John"
  {/* The result will be Hello John*/}
  return <h1 className="title">Hello {name}</h1>;
}
```

في هذا المثال، عرّفنا مكوِّنًا يسمى `Greeting`. الأقواس المتعرجة `{}` داخل وسم `h1` تتيح لنا تضمين تعبيرات JavaScript، مما يسمح لنا بالوصول إلى المتغير `name` داخل عنصر `h1`.

نحن أيضًا نطبّق `className` يسمى `title` على عنصر `h1`.

لكن لماذا نستخدم `className` بدلًا من `class` كما في عناصر HTML العادية؟

ذلك لأن `class` في JavaScript هو اسم محجوز. لذلك، نحتاج إلى استخدام `className` بدلًا منه.

نستخدم أيضًا تعليقًا في JSX يوضح النتيجة المتوقعة. من المهم ملاحظة أنه يمكنك استخدام صيغة التعليق العادية هكذا، لكن يجب أن تُغلفها بالأقواس المتعرجة لكي تعمل:

```jsx
{/* Block Comments */}
```

شيء آخر قد لاحظته هو أننا نستخدم حرفًا كبيرًا في بداية اسم المكوِّن. لكن لماذا لا يمكننا استخدام أحرف صغيرة فقط؟

ذلك لأن React يعامل المكوِّنات التي تبدأ بحرف كبير كمكوِّنات مخصصة، بينما تُعتبر العناصر التي تبدأ بأحرف صغيرة عناصر HTML مدمجة.

عندما يصادف React وسمًا بحروف صغيرة، مثل `<div>` أو `<span>`، يفترض أنه عنصر HTML قياسي. أما إذا بدأ اسم المكوِّن بحرف كبير، فسيعامله React كمكوِّن معرف من المستخدم ويعرضه وفقًا لذلك. هذا التمييز يساعد React على التفريق بين وسوم HTML الأصلية والمكوِّنات التي تنشئها.

لاستخدام هذا المكوِّن `Greeting` في تطبيقنا، نكتب شيئًا مثل هذا:

```jsx
<Greeting />
```

سيعرض هذا عنصر `h1` مع النص `Hello John` على الصفحة. لكن ألقِ نظرة أقرب على الصيغة هنا. عند استخدام المكوِّن، ينتهي بشرطة مائلة ثم رمز أكبر من.

عند العمل مع JSX، يجب إغلاق جميع الوسوم واستخدامات المكوِّنات صراحةً. لذلك، إذا لم يكن للمكوِّن أو الوسم أي عناصر فرعية، يجب إغلاقه صراحةً كما هو موضح هنا:

```jsx
<Greeting /> {/* /> is required */}
```

حتى الآن، كنا ننظر فقط إلى كيفية عرض عنصر `h1` واحد. لكن يمكنك فعليًا عرض عدة عناصر.

لنلقِ نظرة على شفرة المثال التالية هنا:

```jsx
function Greeting() {
  const name = "John";
  {/* This will throw an error */}
  return <h1>Hello {name}</h1> 
  <p>Nice to meet you.</p>
}
```

نحاول إضافة جملة أخرى `Nice to meet you` لكنها لا تُعرض بشكل صحيح على الصفحة.

يبدو أن هناك رسالة خطأ بدلاً من ذلك. تقول رسالة الخطأ `Adjacent JSX elements must be wrapped in an enclosing tag.`

سبب ظهور رسالة الخطأ هذه هو أن العناصر الشقيقة المتعددة يجب أن تُغلف في عنصر أب. بينما يمكنك تغليف عناصر `h1` و `p` في `div` بسيط، هناك طريقة أخرى لإسكات الخطأ.

تُستخدم أجزاء React لتجميع العناصر معًا. هذا هو شكل المثال المعدل:

```jsx
function Greeting() {
  const name = "John";
  return (
    <Fragment>
      <h1>Hello {name}</h1>
      <p>Nice to meet you.</p>
    </Fragment>
  );
}
```

يمكنك أيضًا اختيار استخدام وسوم JSX الفارغة التي يمكن أن تكون اختصارًا لأجزاء React:

```jsx
function Greeting() {
  const name = "John";
  return (
    <>
      <h1>Hello {name}</h1>
      <p>Nice to meet you.</p>
    </>
  );
}
```

في الدروس القادمة، سنواصل تعلم المزيد عن كيفية العمل مع المكوِّنات وJSX. لكن الآن، حصلت على مقدمة قوية لبناء واجهات المستخدم باستخدام المكوِّنات، مما يؤسس قاعدة متينة لما هو قادم.
