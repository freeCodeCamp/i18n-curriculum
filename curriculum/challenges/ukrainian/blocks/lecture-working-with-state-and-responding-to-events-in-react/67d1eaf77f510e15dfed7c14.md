# --description--

Стан — це одна з найважливіших основ React та інших фронтенд-фреймворків. Це ніби мозок компонента, тобто він зберігає інформацію, яка може змінюватися з часом, і контролює, як компоненти поводяться та виглядають.

Давайте розглянемо, що таке стан і як хук `useState` дозволяє працювати з ним.

Стан представляє динамічні дані у вашому React-компоненті, наприклад, значення з введення користувача, дані, отримані з API, або елемент у списку справ.

Щоразу, коли стан змінюється, React повторно відтворює компонент без перезавантаження сторінки, щоб відобразити ці зміни в інтерфейсі користувача. Ця реактивність робить ваш застосунок інтерактивним.

Хук `useState` — це функція, яка дозволяє оголосити змінні стану у функціональних компонентах.

До появи хуків стан можна було використовувати лише у класових компонентах. Але з введенням хуків, починаючи з React 16.8, ви можете використовувати стан у функціональних компонентах за допомогою хука `useState`.

Щоб використати хук `useState`, потрібно імпортувати його з React:

```js
import { useState } from "react";
```

Також можна імпортувати сам React і отримати доступ до хука `useState` як властивості:

```js
import React from "react";
```

Ось як можна оголосити змінну стану, коли ви імпортуєте `useState`:

```js
const [stateVariable, setStateFunction] = useState(initialValue);
```

А ось як оголосити змінну стану, коли ви імпортуєте React:

```js
const [stateVariable, setStateFunction] = React.useState(initialValue);
```

У змінній стану у вас є:

- `stateVariable` зберігає поточне значення стану
- `setStateFunction` (функція встановлення) оновлює змінну стану
- `initialValue` задає ініціальне значення стану

Зверніть увагу, що стан у React-компоненті є приватним і ізольованим для кожного екземпляра компонента. Це означає, що якщо ви відтворюєте один і той самий компонент двічі, стан одного компонента не впливає на інший. Це також означає, що якщо ви хочете поділитися станом між компонентами, вам потрібно підняти стан до спільного батьківського елемента і передати його вниз через пропси.

Ще одна річ — хуки потрібно викликати на верхньому рівні компонента, безпосередньо перед ключовим словом `return`, щоб зберегти стан і ефекти послідовними під час повторних відтворень. Це означає, що ви не можете використовувати стан всередині циклів, умов або вкладених функцій.

Ось приклад керування станом за допомогою хука `useState` у функціональному компоненті `Counter`:

```jsx
// Importing the useState hook
import { useState } from "react";

function Counter() {
  const initialValue = 0;

  // The state variable and setter function
  const [count, setCount] = useState(initialValue);

  return (
    <div>
      {/* Display current state value */}
      <h2>{count}</h2>

      <button onClick={() => setCount(count - 1)}>Decrement</button>
      <button onClick={() => setCount(count + 1)}>Increment</button>
    </div>
  );
}

export default Counter;
```

У наведеному коді ми імпортували хук `useState` з React. У компоненті `Counter` `count` представляє поточний стан, а `setCount` — функція встановлення, яка відповідає за оновлення стану. Поточне значення стану — `0`. Інструкція `return` містить лічильник і дві кнопки для зменшення та збільшення лічильника на `1`.

Ви можете керувати кількома частинами стану, викликаючи хук `useState` кілька разів. Це особливо важливо, коли у вас є несуміжні змінні стану:

```jsx
function UserProfile() {
  const [isOnline, setIsOnline] = useState(false);
  const [notifications, setNotifications] = useState(0);

  // The rest of the component logic
}
```

Ви також можете викликати хук `useState` кілька разів, коли керуєте кількома станами, які оновлюються окремо, наприклад, поля форми:

```jsx
function SignUpForm() {
  const [name, setName] = useState("");
  const [username, setUsername] = useState("");
  const [email, setEmail] = useState("");

  // The rest of the component logic
}
```

Але в цьому випадку краще об’єднати стани, оскільки вони всі є частиною однієї форми:

```jsx
function SignUpForm() {
  const [formData, setFormData] = useState({
    name: "",
    username: "",
    email: "",
  });

  // The rest of the component logic
}
```

Ось що таке стан і як ви можете використовувати хук `useState`.
