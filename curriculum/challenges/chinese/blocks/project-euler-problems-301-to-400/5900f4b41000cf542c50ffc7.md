# --description--

我们正试图通过提问来查找从整数集合 {1, 2, ..., $n$} 中选出的隐藏数字。我们每提出一个数字（问题），其<u>花费等于所提数字</u>，并且我们会得到三种可能的答案之一：

- 你的猜测比隐藏数字低
- “是的，就是这样！”
- 你的猜测比隐藏数字大。

给定 $n$ 的值，最优策略会最小化 <u>最坏分支</u> 下的总成本（即所有提问的总和）。例如。

如果 $n = 3$，我们显然能做的最好的是询问数字 "<strong>2</strong>"。答案将立即引导我们查找隐藏的数字（总成本 = 2）。

如果 $n = 8$，我们可能决定使用一种 `"binary search"` 类型的策略：我们的第一个问题将是`"<strong>4</strong>"`，如果隐藏的数字大于 4，我们将需要一到两个额外的问题。让我们的第二个问题是`"<strong>6</strong>"`。如果隐藏的数字仍然大于 6，我们将需要第三个问题来区分 7 和 8。因此，我们的第三个问题将是`"<strong>7</strong>"`，这个最坏情况下的总成本将是 $4 + 6 + 7 = \mathbf{\color{red}{17}}$。

我们可以通过将“<strong>5</strong>”作为第一个问题，显著改善 $n = 8$ 时的最坏分支成本。如果告诉我们隐藏的数字大于 5，我们的第二个问题将是“<strong>7</strong>”，然后我们就能确定隐藏的数字是什么（总成本为 $5 + 7 = \mathbf{\color{blue}{12}}$）。如果告诉我们隐藏的数字小于 5，我们的第二个问题将是“<strong>3</strong>”，如果隐藏的数字小于 3，我们的第三个问题将是“<strong>1</strong>”，总成本为 $5 + 3 + 1 = \mathbf{\color{blue}{9}}$。由于 $\mathbf{\color{blue}{12 > 9}}$，该策略的最坏分支成本是 <strong><span style="color: red;">12</span></strong>。这比我们之前使用“二分查找”策略所达到的更好；它也优于或等于任何其他策略。因此，实际上，我们刚刚描述了一个针对 $n = 8$ 的最优策略。

令 $C(n)$ 为上述描述的针对 $n$ 的最优策略所达到的最坏分支代价。因此 $C(1) = 0$，$C(2) = 1$，$C(3) = 2$ 和 $C(8) = 12$。

同样，$C(100) = 400$ 且 $\displaystyle\sum_{n = 1}^{100} C(n) = 17575$。

查找 $\displaystyle\sum_{n = 1}^{200\\,000} C(n)$。
