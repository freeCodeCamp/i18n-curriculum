# --interactive--

Unapojenga programu zako, mara nyingi utataka kitu kifanyike baada ya kuchelewa kidogo, au kurudiarudia kwa vipindi vya kawaida. Hapo ndipo njia za `setTimeout()` na `setInterval()` zinapokuwa msaada. Njia zote mbili zinapokea vigezo viwili: kitendakazi na ucheleweshaji.

Tuchunguze kila njia kwa undani zaidi ili uanze kuzitumia katika miradi yako ya mtandao.

`setTimeout()` inakuwezesha kuchelewesha kitendo kwa muda uliobainishwa. Hapa kuna sintaksia ya msingi ya njia ya `setTimeout()`:

```js
setTimeout(functionToRun, delay);
```

`functionToRun` ni msimbo unaotaka utekelezwe baada ya ucheleweshaji uliobainishwa, na `delay` ni muda (kwa millisekunde) wa kusubiri kabla ya kuendesha kitendakazi. Hapa kuna mfano:

:::interactive_editor

```js
setTimeout(function () {
 console.log("This runs after 3 seconds");
}, 3000);
```

:::

Unaweza pia kutumia kitendakazi cha mshale kwa wito wa kurudisha, kama hii:

:::interactive_editor

```js
setTimeout(() => {
 console.log("This runs after 3 seconds"); 
}, 3000);
```

:::

`setInterval()` inaendelea kurudia kitendakazi kwa kipindi kilichowekwa. Ni kama kuwa na kengele inayopiga kila sekunde chache. `setInterval()` ni kamili unapotaka kitu kifanyike mara kwa mara, kama kuonyesha upya data au kuunda michoro inayosogea bila kukoma.

Kama `setTimeout()`, inachukua kitendakazi na ucheleweshaji:

```js
setInterval(functionToRun, delay);
```

`functionToRun` ni msimbo unaotaka kuendeshwa mara kwa mara, wakati `delay` ni muda kwa millisekunde kati ya kila utekelezaji wa kitendakazi hicho. Hapa kuna mfano:

:::interactive_editor

```js
setInterval(() => {
 console.log("This runs every 2 seconds");
}, 2000);

// Result:
// This runs every 2 seconds
// This runs every 2 seconds
// This runs every 2 seconds
// ...
```

:::

Kwa kuwa `setInterval()` inaendelea kuendesha kitendakazi kilichotolewa kwa kipindi kilichobainishwa, unaweza kutaka kuikomesha. Ili kufanya hivyo, lazima utumie njia ya `clearInterval()`.

Ili kufanya kazi hiyo, `clearInterval()` inachukua kitambulisho cha `setInterval()` unayotaka kuikomesha. Hii inaweza kuwa ni kigezo ulichoweka kipindi hicho. Njia moja ya kuzuia kipindi ni ndani ya `setTimeout()`, kwani hiyo itakomesha kipindi baada ya muda fulani:

:::interactive_editor

```js
const intervalID = setInterval(() => {
 console.log("This will stop after 5 seconds");
}, 1000);

setTimeout(() => {
 clearInterval(intervalID);
}, 5000);
```

:::

Unaweza pia kuzuia timeout kwa njia ya `clearTimeout()`:

:::interactive_editor

```js
let timeoutID = setTimeout(() => {
 console.log("This will not run");
}, 5000);

clearTimeout(timeoutID);
```

:::

Kwa msimbo huo, `setTimeout()` haitatekelezwa kabisa. Hivyo, njia nzuri zaidi ya kufanya hivyo ni kupitia DOM, kwa kuongeza msikilizaji wa tukio la bofya kwenye kitufe ili kuzuia timeout:

:::interactive_editor

```html
<h1>Cancel Timeout Example</h1>
<button id="cancelButton">Cancel Timeout</button>
<script src="index.js"></script>
```

```js
let timeoutID = setTimeout(() => {
  console.log("This will run if not canceled");
}, 5000);

document
.querySelector("#cancelButton")
.addEventListener("click", () => {
  clearTimeout(timeoutID);
  console.log("Timeout canceled!");
});
```

:::
