# --description--

Überschriften (`h1` bis `h6` Elemente) sind Arbeitspferd-Tags, die dabei helfen, Struktur und Bezeichnungen für Ihre Inhalte bereitzustellen. Screenreader können so eingestellt werden, dass sie nur die Überschriften auf einer Seite vorlesen, damit der Benutzer eine Zusammenfassung erhält. Das bedeutet, dass die Überschriften-Tags in Ihrer Auszeichnung eine semantische Bedeutung haben und zueinander in Beziehung stehen sollten, nicht nur nach ihrer Größenangabe ausgewählt werden.

*Semantische Bedeutung* bedeutet, dass das Tag, das Sie um Inhalte verwenden, den Typ der enthaltenen Information anzeigt.

Wenn Sie eine Arbeit mit einer Einleitung, einem Hauptteil und einem Fazit schreiben würden, wäre es wenig sinnvoll, das Fazit als Unterabschnitt des Hauptteils in Ihrer Gliederung zu setzen. Es sollte ein eigener Abschnitt sein. Ebenso müssen die Überschriften-Tags auf einer Webseite in der Reihenfolge stehen und die hierarchischen Beziehungen Ihrer Inhalte anzeigen.

Überschriften mit gleichem (oder höherem) Rang beginnen neue implizite Abschnitte, Überschriften mit niedrigerem Rang beginnen Unterabschnitte des vorherigen.

Als Beispiel würde eine Seite mit einem `h2` Element, gefolgt von mehreren Unterabschnitten, die mit `h4` Elementen bezeichnet sind, einen Screenreader-Benutzer verwirren. Bei sechs Auswahlmöglichkeiten ist es verlockend, ein Tag zu verwenden, weil es im Browser besser aussieht, aber Sie können CSS verwenden, um die relative Größe zu bearbeiten.

Ein letzter Punkt: Jede Seite sollte immer genau ein `h1` Element haben, das das Hauptthema Ihres Inhalts ist. Dieses und die anderen Überschriften werden teilweise von Suchmaschinen verwendet, um das Thema der Seite zu verstehen.

# --instructions--

Camper Cat möchte eine Seite auf seiner Website, die dem Thema Ninja werden gewidmet ist. Helfen Sie ihm, die Überschriften zu korrigieren, damit seine Auszeichnung semantische Bedeutung für den Inhalt liefert und die richtigen Eltern-Kind-Beziehungen seiner Abschnitte zeigt. Ändern Sie alle `h5` Tags in die richtige Überschriftenebene, um anzuzeigen, dass sie Unterabschnitte der `h2` sind. Verwenden Sie `h3` Tags dafür.

# --hints--

Ihr Code sollte 6 `h3` Elemente enthalten.

```js
assert.lengthOf(document.querySelectorAll('h3') , 6);
```

Ihr Code sollte 6 `h3` schließende Tags enthalten.

```js
assert.lengthOf((code.match(/\/h3/g) || []) ,6);
```

Ihr Code sollte keine `h5` Elemente enthalten.

```js
assert.isEmpty(document.querySelectorAll('h5'));
```

Ihr Code sollte keine `h5` schließenden Tags enthalten.

```js
assert.notMatch(code, /\/h5/);
```
