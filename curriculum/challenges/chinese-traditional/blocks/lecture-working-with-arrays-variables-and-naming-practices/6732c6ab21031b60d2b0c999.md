# --description--

命名變數和函式是撰寫乾淨、易讀且易於維護的程式碼的重要面向。良好的命名慣例使你的程式碼具備自我說明的特性，減少大量註解的需求，並讓其他開發者包括未來的你更容易理解你的程式碼。

讓我們從 JavaScript 的一般命名慣例開始。在之前的課程中，你學到了對變數名稱使用駝峰式命名。對於布林值變數，常見的做法是使用像 `is`、`has` 或 `can` 這樣的前置式。這會立即告訴讀者該變數是布林值：

```js
let isLoading = true;
let hasPermission = false;
let canEdit = true;
```

對於函式（程式），名稱應該清楚顯示該函式的功能。通常以動詞開頭會很有幫助：

```js
function getUserData(){
  /* ... */
}

function calculateTotal(){
  /* ... */
}

function validateInput(){
  /* ... */
}
```

對於傳回布林值且通常稱為謂詞的函式，你可以使用相同的 `is`、`has` 或 `can` 前置式：

```js
function isValidEmail(email) {
  /* ... */
}

function hasRequiredFields(form) {
  /* ... */
}
```

當你有用來擷取資料的函式（程式）時，通常會以字 `get` 開頭：

```js
function getProductDetails(productId) {
  /* ... */
}

function getUserProfile(userId) {
  /* ... */
}
```

當你有設定資料的函式（程式）時，通常會以字 `set` 開頭：

```js
function setUserPreferences(preferences) {
  /* ... */
}

function setPageTitle(title) {
  /* ... */
}
```

對於事件處理常式函式（程式），你可以在前面加上 `handle` 或在後面加上 `handler`：

```js
function handleClick(){
  /* ... */
}

function onSubmit(){
  /* ... */
}

function keyPressHandler(){
  /* ... */
}
```

事件處理常式是在事件發生後發生的動作，例如按鈕點擊。你將在未來的課程中學習這個內容。

在命名迭代器變數和迴圈時，通常會使用像 `i`、`j` 或 `k` 這樣的單一字母，但對於巢狀迴圈或較複雜的迭代，使用更具描述性的名稱會更有幫助：

```js
for (let i = 0; i < array.length; i++) {
  /* ... */
}

for (let studentIndex = 0; studentIndex < students.length; studentIndex++) {
  /* ... */
}
```

對於陣列名稱，請考慮使用複數名詞來顯示該變數包含多個條款：

```js
const colors = ['red', 'green', 'blue'];
const userNames = ['Alice', 'Bob', 'Charlie'];
```

請記住目標是讓你的程式碼盡可能自我說明。一個好的經驗法則是，如果你需要添加註解來解釋變數或函式的作用，你可能應該考慮將它重新命名為更具描述性的名稱。

最後，請在你的程式碼庫或團隊中保持一致性。如果你的團隊已經建立了命名慣例，請遵守它們。一致性讓程式碼對所有參與者來說更易讀且更易維護。
