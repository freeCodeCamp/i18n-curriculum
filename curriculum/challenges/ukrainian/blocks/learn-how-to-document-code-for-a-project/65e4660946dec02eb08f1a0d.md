# --description--

Файл `README` є важливою частиною багатьох проєктів, особливо в розробці програмного забезпечення. Це часто перший документ, який люди бачать, коли дивляться на проєкт. Файл `README` пояснює, про що цей проєкт, як його використовувати і іноді як робити внесок у нього. Розуміння того, для чого використовується файл `README`, є ключовим у контексті документації проєкту.

# --questions--

## --text--

Для чого зазвичай використовується файл `README` у проєкті?

## --answers--

Зберігання коду

### --feedback--

Хоча код є невід’ємною частиною проєкту, файл `README` не використовується для зберігання коду, а для надання інформації про проєкт.

---

Інструкції щодо проєкту

---

Контактна інформація

### --feedback--

Хоча контактна інформація може бути включена, основна мета файлу `README` — надати інструкції та інформацію про проєкт, а не лише контактні дані.

---

Журнал змін

### --feedback--

Журнал змін, часто званий changelog, зазвичай є окремим від файлу `README`. Файл `README` зосереджений на представленні та поясненні проєкту.
