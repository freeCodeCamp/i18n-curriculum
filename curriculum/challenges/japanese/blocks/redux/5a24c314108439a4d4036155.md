# --description--

これまでにReduxストアにアクションをディスパッチする方法を学びましたが、これらのアクションには`type`以外の情報は含まれていませんでした。アクションと一緒に特定のデータを送ることもできます。実際、これは非常に一般的です。なぜならアクションは通常ユーザーの操作から発生し、何らかのデータを伴うことが多いからです。Reduxストアはこのデータを知る必要がある場合が多いです。

# --instructions--

コードエディターには基本的な`notesReducer()`と`addNoteText()`のアクションクリエーターが定義されています。`addNoteText()`関数の本体を完成させて、`action`オブジェクトを返すようにしてください。そのオブジェクトには`type`プロパティが`ADD_NOTE`の値で含まれている必要があり、さらに`text`プロパティがアクションクリエーターに渡された`note`データに設定されている必要があります。アクションクリエーターを呼び出すときに、特定のノート情報を渡し、その情報をオブジェクトでアクセスできます。

次に、`switch`内の`notesReducer()`文を完成させてください。`addNoteText()`アクションを処理するケースを追加する必要があります。このケースは`ADD_NOTE`タイプのアクションがあるたびにトリガーされ、入ってくる`text`の`action`プロパティを新しい`state`として返すべきです。

アクションはコードの最後でディスパッチされています。完了したらコードを実行してコンソールを確認してください。これでアクション固有のデータをストアに送信し、ストアの`state`を更新するときに使うことができます。

# --hints--

アクションクリエーター`addNoteText`は`type`と`text`というキーを持つオブジェクトを返すべきです。

```js
assert(
  (function () {
    const addNoteFn = addNoteText('__TEST__NOTE');
    return addNoteFn.type === ADD_NOTE && addNoteFn.text === '__TEST__NOTE';
  })()
);
```

`ADD_NOTE`タイプのアクションを`addNoteText`アクションクリエーターでディスパッチすると、`state`がアクションクリエーターに渡された文字列に更新されるはずです。

```js
assert(
  (function () {
    const initialState = store.getState();
    store.dispatch(addNoteText('__TEST__NOTE'));
    const newState = store.getState();
    return initialState !== newState && newState === '__TEST__NOTE';
  })()
);
```
