# --description--

Quando costruisci progetti in JavaScript, spesso hai bisogno di strumenti extra per aiutarti a completare il lavoro. Anche le app più semplici possono richiedere funzionalità come salvare dati, effettuare il login, gestire il routing per spostarsi tra le pagine e così via.

Invece di scrivere tutto questo codice da zero, puoi usare pacchetti open source che altri sviluppatori hanno già creato e condiviso.

Qui entra in gioco npm. npm è un enorme registro online di software open source. Pensalo come un app store per il codice — puoi cercare ciò di cui hai bisogno, installarlo e iniziare a usarlo nel tuo progetto.

Molti pensano che npm stia per "Node Package Manager", ma non è il suo significato ufficiale. Nella documentazione più vecchia di npm, veniva descritto come un'abbreviazione ricorsiva bacronimica per "npm is not an acronym". In altre parole, npm non ha un significato ufficiale.

Oggi, npm si riferisce a tre parti correlate:

* Il sito web npmjs.com, dove puoi cercare i pacchetti nel registro.
    
* Il registro npm, che è un grande database pubblico che conserva tutti i pacchetti.
    
* Lo strumento CLI di npm che viene installato con Node.js e ti permette di installare e gestire i pacchetti.
    

Per esempio, se hai bisogno di gestire lo stato, lavorare con le date, inviare email o fare richieste HTTP nel tuo progetto, probabilmente esiste già un pacchetto npm che fa tutto questo. Con un semplice comando `npm install`, puoi iniziare a usare questo software già pronto invece di fare tutto da solo.

npm ti aiuta anche a gestire le dipendenze del tuo progetto. Questo significa che tiene traccia dei pacchetti che usi nei tuoi progetti e della versione di ciascuno di essi. Lo fa con i file `package.json` e `package-lock.json`, che imparerai a conoscere nelle lezioni future.

Yarn, PNPM e Bun sono alcune alternative a npm che sono diventate popolari nel tempo. Anche se ognuna offre vantaggi in aree specifiche come la velocità di installazione e l’efficienza dello storage, npm è ancora la più usata.

Nelle prossime lezioni imparerai vari comandi npm, come installare pacchetti, i file `package.json` e `package-lock.json` e persino come pubblicare il tuo pacchetto nel registro npm.
