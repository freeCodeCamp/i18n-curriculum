## npm이란 무엇인가요?

- **정의**: npm은 오픈소스 소프트웨어를 위한 온라인 저장소로, 코드용 "앱 스토어"처럼 작동합니다.
- **공식 의미**: npm은 공식적으로 "Node Package Manager"의 약자가 아니며, "npm is not an acronym"이라는 재귀적 약어입니다.
- **세 가지 관련 부분**:

  - **웹사이트 (npmjs.com)**: 저장소에서 패키지를 검색하는 곳입니다.
  - **저장소**: 모든 패키지를 저장하는 대규모 공개 데이터베이스입니다.
  - **명령줄 인터페이스(CLI)**: Node.js와 함께 설치되어 프로젝트 의존성을 관리하는 도구입니다.

- **대안**: npm이 가장 널리 사용되지만, 인기 있는 대안으로는 **Yarn**, **PNPM**, **Bun**이 있습니다.

## package.json 파일

- **목적**: Node.js 프로젝트의 메타데이터, 의존성, 스크립트를 저장하는 구성 파일입니다.
- **주요 필드**:

  - `name`와 `version`: 프로젝트를 식별합니다.
  - `main`: 진입점 파일을 지정하며, 보통 `index.js`입니다.
  - `license`: 법적 조건을 정의하며 기본값은 `ISC`입니다.
  - `dependencies`: 프로덕션에서 앱 실행에 필요한 패키지입니다.
  - `devDependencies`: 테스트 프레임워크 같은 개발 중에만 필요한 도구입니다.

- **생성**:

  - `npm init`: 대화형 설정 과정을 시작합니다.
  - `npm init -y`: 기본값으로 `package.json`를 즉시 설치합니다.

## 의미 있는 버전 관리 (SemVer)

- **형식**: `MAJOR.MINOR.PATCH`입니다.
- **규칙**:

  - **MAJOR**: 기존 코드를 깨뜨릴 수 있는 변경입니다.
  - **MINOR**: 이전 버전과 호환되는 새로운 기능입니다.
  - **PATCH**: 작은 버그 수정과 개선입니다.

- **범위 기호**:

  - `^` (캐럿): 마이너 및 패치 업데이트를 허용합니다 (예: `^1.2.3`는 `1.x.x`를 허용).
  - `~` (물결표): 패치 업데이트만 허용합니다 (예: `~1.2.3`는 `1.2.x`를 허용).
  - `*` (별표): 모든 버전과 일치하며 주로 테스트에 사용됩니다.

## 의존성 관리

- **설치**:

  - `npm install <name>`: 프로덕션 의존성을 추가합니다.
  - `npm install <name> -D`: 개발 의존성을 추가합니다.
  - `npm install <name>@<version>`: 특정 버전을 설치합니다.

- **노드 모듈과 잠금 파일**:

  - `node_modules/`: 설치된 패키지의 실제 코드를 포함합니다.
  - `package-lock.json`: 모든 의존성과 하위 의존성의 정확한 버전을 잠가 환경 간 일관성을 보장합니다.

- **제거**: `npm uninstall <name>`는 패키지를 제거하고 `package.json`를 업데이트합니다.
- **유지 관리**:

  - `npm outdated`: 새 버전이 있는 패키지를 식별합니다.
  - `npm update`: 허용된 SemVer 범위 내에서 패키지를 업데이트합니다.

## npm 스크립트

- **기능**: `scripts`의 `package.json` 섹션에 정의된 사용자 지정 명령으로 작업을 자동화합니다.
- **실행**: `npm run <script_name>`를 통해 실행합니다.
- `start`, `test`, `stop`, `restart`에 대한 축약형이 있으며 (예: `npm start`)가 있습니다.
- **순차 실행**:
- `&&`: 첫 번째 명령이 성공할 때만 두 번째 명령을 실행합니다.
- `;`: 성공 여부와 상관없이 명령을 차례대로 실행합니다.
- **동시 실행**: `&` 연산자를 사용해 프로세스를 백그라운드에서 실행합니다.
- **인수 전달**: `--` 구분자를 사용해 스크립트에 인수를 전달합니다 (예: `npm start -- 8000`).

## npm 저장소에 게시하기

- **저장소 접근**: 공개 및 비공개 패키지의 데이터베이스입니다.
- **게시 단계**:

1. `npmjs.com`에서 무료 계정을 만드세요.
2. 터미널에서 `npm login`로 로그인하세요.
3. `name`에 고유한 `version`와 유효한 `package.json`가 있는지 확인하세요.
4. 민감한 파일을 제외하는 `README.md`와 `.npmignore`를 만드세요.
5. `npm publish` (또는 범위가 지정된 패키지의 경우 `npm publish --access public`)를 실행하세요.

- **범위**: 이름 충돌을 피하기 위해 `@username/package-name` 아래에 게시할 때 사용합니다.

## Node.js의 모듈 시스템

### CommonJS (CJS)

- **기본 시스템**: Node.js가 모듈을 처리하는 원래 방식이며 동기적으로 로드합니다.
- **구문**:
- **가져오기**: `const module = require('./path')`.
- **내보내기**: `module.exports = { functionName }`.
- **식별**: `.cjs` 파일이나 `.js` 파일에 `type`에 `package.json`가 지정되지 않은 경우 사용합니다.

### ES 모듈 (ESM)

- **표준화된 형식**: 최신 JavaScript 표준이며 효율성을 위해 비동기적으로 로드합니다.
- **구문**:

  - **내보내기**: `export { name }` 또는 `export default function`.
  - **가져오기**: `import { name } from './path.mjs'`.
  - **식별**: `.mjs` 파일이나 `.js` 파일에 `package.json`에 `"type": "module"`가 포함된 경우 사용합니다.
