# --description--

让我们了解一下终端，以及它与命令行的区别。

命令行是一种基本的文本输入接口，允许用户输入“commands”，通常以一串字符的形式，并通过按下“Enter”键提交或执行它们。你最常见的命令行接口是在终端中看到的。

终端是一个特殊的应用，提供命令行接口以执行超出基本读写操作的系统级命令。你也可能听说过“终端模拟器”，它们本质上是将基本终端接口封装起来以提供额外特色或功能性的应用。但对于大多数通用的用途，你也可以安全地将它们称为“终端”。

最后，我们还需要讨论“shell”。shell 是包装命令行并将你的输入解释为命令，返回输出的软件。

但是你如何访问你系统上的终端？

好吧，如果你正在运行现代版本的 Windows，你很可能有两个不同的选项。Windows 开箱即用提供两个不同的 shell 应用程序：Powershell 和 Command Prompt。

这两个 shell 都随附各自的终端应用程序（名称与各自的 shell 相同），因此你可以使用你的应用菜单启动 Powershell 终端或 Command Prompt 终端。

Windows 还提供了一个相对新建且现代化的终端模拟器，恰当地命名为 Microsoft Terminal。

你可以从 Windows Store 安装此应用，然后像启动你的 Powershell 或 Command Prompt 终端一样启动它。Microsoft Terminal 的优势在于它允许你从同一个应用访问 Powershell 和 Command Prompt shell，以及如果你配置了 Windows Subsystems for Linux，还可以访问相应的 Linux shell。

MacOS 提供了一个恰如其名的终端，叫做 `Terminal`。你可以通过你的聚焦查找或应用启动器访问它。

MacOS 还支持第三方终端模拟器，例如 iTerm 或 Ghostty。你可以从它们的主页安装这些应用。

在 Linux 上，默认终端（以及你如何启动它）高度依赖于你使用的发行版和桌面环境。第三方终端模拟器也有很好的支持。例如，你可以在 Arch Linux 中设置 `kitty` —— 并且你可以通过诸如 `wofi` 这样的应用管理器启动它。

Linux 可以说拥有最多样化的终端应用可供选择。大多数这些应用都可以通过你的发行版的包管理器安装。

“terminal”、“shell”和“command line”之间的区别有时看起来相当拘泥细节。在大多数情况下，这些术语通常可以互换使用。但了解它们具有特定含义以及每个含义是什么是很重要的。
