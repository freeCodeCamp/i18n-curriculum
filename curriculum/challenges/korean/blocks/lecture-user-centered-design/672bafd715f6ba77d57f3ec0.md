# --description--

사용자 조사는 제품을 사용하는 사람들을 체계적으로 연구하는 것입니다. 목표는 사용자 요구, 행동, 그리고 문제점을 측정하는 데 있습니다.

사용자 조사는 여러 형태로 이루어집니다. 그중 가장 흔한 것 중 하나가 넷 프로모터 점수(Net Promoter Score, NPS)입니다. NPS는 사용자가 친구에게 제품을 추천할 가능성을 측정합니다. NPS는 사용자의 여정 중 주요 시점, 예를 들어 7일, 30일, 90일 후에 제공되는 설문조사를 통해 측정합니다. NPS는 0부터 10까지의 척도로 측정하며, 9와 10은 사이트의 적극적인 추천자를 나타냅니다.

또 다른 조사 방법은 종료 인터뷰입니다. 이는 사용자가 구독을 취소하거나 계정을 삭제할 때 보여주는 설문조사입니다. 이 설문조사에서 얻은 데이터는 사용자 이탈 원인을 파악하는 데 도움을 주어 문제를 해결할 수 있게 합니다.

반면 사용자 테스트는 사용자가 애플리케이션과 상호작용하는 동안 데이터를 수집하는 행위를 말합니다. 예를 들어, 비디오 게임이 베타 테스트를 거치는 것이 사용자 테스트의 한 형태입니다. 웹 개발자가 접할 수 있는 예로는 A/B 테스트가 있습니다. A/B 테스트는 사용자 기반의 무작위 부분집합에 새로운 기능을 배포하는 것을 포함합니다. 그런 다음 분석 데이터를 활용해 해당 기능이 유용한지 판단할 수 있습니다.

마지막으로 사용자 요구사항은 애플리케이션이 따라야 할 스토리나 기준을 의미합니다. 이는 개발 과정을 안내할 수 있습니다. 사용자 요구사항은 사용자 조사나 업계 표준에 의해 정의될 수 있으며, 이해관계자의 의견에 의해 정의되기도 합니다.

이 요구사항은 기능적일 수도 있고, 즉 애플리케이션이 어떻게 작동해야 하는지를 규정하거나, 비기능적일 수도 있는데 이는 애플리케이션이 어떻게 행동해야 하는지를 정의합니다. 사용자 요구사항은 고정된 것이 아니며, 사용자 테스트와 사용자 조사에서 얻은 정보가 요구사항에 영향을 미치고 사용자 기반이 변함에 따라 요구사항도 변합니다.

이 차이를 이해하는 것은 가장 정확한 데이터를 수집하여 최종 사용자에게 최고의 경험을 제공하는 데 필수적입니다.
