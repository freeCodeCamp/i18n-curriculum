# --description--

CSS розглядає кожен HTML-елемент як окремий блок, який зазвичай називають <dfn>CSS Box Model</dfn>. Блокові елементи автоматично починаються з нового рядка (наприклад, заголовки, абзаци та div), тоді як рядкові елементи розміщуються всередині навколишнього вмісту (наприклад, зображення або span). За замовчуванням макет елементів у такий спосіб називають <dfn>нормальним потоком</dfn> документа, але CSS пропонує властивість position, щоб заміщати його.

Коли положення елемента встановлено в `relative`, це дозволяє вказати, як CSS має переміщувати його *відносно* його поточного положення в нормальному потоці сторінки. Це поєднується з CSS-властивостями зміщення `left` або `right`, а також `top` або `bottom`. Вони вказують, на скільки пікселів, відсотків або em потрібно перемістити елемент *від* його звичайного положення. У наступному прикладі абзац переміщується на 10 пікселів від нижнього краю:

```css
p {
  position: relative;
  bottom: 10px;
}
```

Зміна положення елемента на relative не видаляє його з нормального потоку — інші елементи навколо нього поводяться так, ніби цей елемент знаходиться на своєму стандартному місці.

**Примітка:** Позиціонування дає багато гнучкості та контролю над візуальним макетом сторінки. Важливо пам’ятати, що незалежно від положення елементів, базова HTML-розмітка має бути організованою і логічною при читанні зверху вниз. Саме так користувачі з порушеннями зору (які користуються допоміжними пристроями, як-от читачі екрана) отримують доступ до вашого вмісту.
