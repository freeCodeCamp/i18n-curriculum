# --interactive--

설명 목록은 용어와 정의를 정리되고 읽기 쉬운 형식으로 제시하는 데 완벽합니다. 예를 들어, 용어집이나 실제 사전에서 단어와 그에 해당하는 정의를 찾을 수 있는 것처럼요.

다음은 두 개의 용어와 그에 해당하는 세부 정보를 포함한 HTML 설명 목록 예시입니다.

인터랙티브 편집기를 활성화하고 코드를 주석 해제하여 새 세부 항목이 미리보기 창에 나타나는 것을 확인하세요.

:::interactive_editor

```html
<dl>
  <dt>HTML</dt>
  <dd>HyperText Markup Language</dd>
  <dt>CSS</dt>
  <dd>Cascading Style Sheets</dd>
  <!-- <dt>JS</dt>
  <dd>JavaScript</dd> -->
</dl>
```

:::

이 경우 용어는 HTML과 CSS라는 약어이고, 세부 정보는 그 확장형입니다. 세부 정보는 정의나 용어와 관련된 다른 정보일 수도 있습니다.

설명 목록을 정의하려면 세 가지 HTML 요소가 필요합니다. 먼저, 설명 목록 요소인 `dl`가 있는데, 이는 전체 목록의 컨테이너입니다. 예시에서 설명 목록의 다른 모든 요소를 감싸고 있는 것을 볼 수 있습니다.

그다음, 각 용어마다 하나씩 설명 용어 요소인 `dt`가 있습니다. 이 설명 목록에는 HTML과 CSS 두 개의 용어가 있으므로 이 요소도 두 개가 있습니다.

마지막으로, 각 용어 뒤에는 해당 용어와 관련된 설명이나 세부 정보를 담는 설명 세부 요소인 `dd`가 있습니다. 이 예시에서는 Hypertext Markup Language와 Cascading Style Sheets입니다.

브라우저에서는 각 용어 뒤에 해당 설명이 따라오는 것을 볼 수 있습니다. 기본적으로 설명은 시각적으로 구분하기 위해 약간 오른쪽으로 들여쓰기 되어 있습니다.

하지만 설명 목록은 용어와 정의에만 국한되지 않습니다. 훨씬 더 다양하게 활용할 수 있습니다. 여기 두 가지 재료가 있는 레시피가 있습니다.

인터랙티브 편집기를 활성화하고 코드를 주석 해제하여 새 세부 항목이 미리보기 창에 나타나는 것을 확인하세요.

:::interactive_editor

```html
<dl>
  <dt>Flour</dt>
  <dd>2 cups</dd>
  <dt>Sugar</dt>
  <dd>1/2 cup</dd>
  <!-- <dt>Vegetable Oil</dt>
  <dd>2 tablespoons</dd> -->
</dl>
```

:::

전체 설명 목록은 설명 목록 요소 안에 있습니다. 첫 번째 재료인 `Flour`는 설명 용어 요소 안에 있습니다. 그리고 이 재료가 얼마나 필요한지 `2 cups`에서 볼 수 있습니다. 이는 해당 재료 바로 뒤에 있는 설명 세부 요소 안에 있습니다.

그리고 같은 구조가 `Sugar`에 대해서도 반복됩니다. 이 경우 레시피에는 두 가지 재료만 있지만, 더 많다면 같은 구조가 설명 목록 전체에 반복될 수 있습니다.

브라우저에서는 재료가 왼쪽에 정렬되고, 양은 시각적으로 구분하기 위해 들여쓰기 되어 나타납니다.

설명 목록의 다른 활용 사례로는 제품 사양, 자주 묻는 질문, 연락처 정보, 메타데이터 등이 있습니다. 기본적으로 두 개의 관련 정보가 키-값 쌍 형식으로 있을 때, 하나는 레이블인 키 역할을 하고 다른 하나는 추가 관련 정보인 값 역할을 할 때 설명 목록을 사용할 수 있습니다.
