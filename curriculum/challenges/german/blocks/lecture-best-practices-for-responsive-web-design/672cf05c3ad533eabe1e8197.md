# --interactive--

Media Queries sind eine grundlegende Komponente des responsiven Webdesigns, die es Entwickler*innen ermöglichen, unterschiedliche Styles basierend auf den Eigenschaften des Geräts oder Browsers des Nutzers anzuwenden. Sie bieten eine Möglichkeit, die Darstellung von Inhalten an verschiedene Geräte anzupassen, ohne den Inhalt selbst zu verändern.

Im Kern besteht eine Media Query aus einem Medientyp und einem oder mehreren Ausdrücken, die bestimmte Bedingungen prüfen. Wenn diese Bedingungen erfüllt sind, werden die entsprechenden Styles angewendet. Dieser Mechanismus ermöglicht die Erstellung responsiver Designs, die sich an verschiedene Bildschirmgrößen, Auflösungen und Gerätefähigkeiten anpassen.

Die grundlegende Syntax einer Media Query in CSS sieht so aus:

```css
@media mediatype and (feature: value) {
  /* CSS rules go here */
}
```

In dieser Struktur gibt mediatype den Medientyp an, auf den sich die Abfrage bezieht, und das feature: value-Paar definiert die Bedingung, die erfüllt sein muss, damit die Styles angewendet werden.

Medientypen beschreiben die allgemeine Kategorie eines Geräts. Sprechen wir über die am häufigsten verwendeten Medientypen:

- `all` ist für alle Geräte geeignet. Dies ist der Standard, wenn kein Medientyp angegeben ist.

- `print` ist für paginierte Materialien und Dokumente gedacht, die im Druckvorschau-Modus auf einem Bildschirm angezeigt werden.

- `screen` ist hauptsächlich für Bildschirme vorgesehen.

Früher gab es mehr Medientypen, wie `handheld` und `tv`, aber die meisten davon wurden zugunsten der Verwendung von Funktionen, um Geräte präziser anzusprechen, veraltet.

Media Features beschreiben spezifische Eigenschaften des User Agents, Ausgabegeräts oder der Umgebung. Sprechen wir über einige der am häufigsten verwendeten Media Features. `width` und `height` beziehen sich auf die Breite und Höhe des Viewports und werden oft mit `min-` oder `max-` Präfixen für Bereichsabfragen verwendet. Hier ist ein Beispiel:

```css
@media screen and (min-width: 768px) {
  /* Styles for screens at least 768px wide */
}
```

`aspect-ratio` beschreibt das Verhältnis zwischen Breite und Höhe des Viewports. Hier ist ein Beispiel:

```css
@media screen and (aspect-ratio: 16/9) {
  /* Styles for screens with a 16:9 aspect ratio */
}
```

Das `orientation`-Feature gibt an, ob sich das Gerät im Quer- oder Hochformat befindet. Hier ist ein Beispiel:

```css
@media screen and (orientation: landscape) {
  /* Styles for landscape orientation */
}
```

Das `resolution`-Feature beschreibt die Auflösung des Ausgabegeräts in Punkten pro Zoll (dpi) oder Punkten pro Zentimeter (dpcm). Hier ist ein Beispiel:

```css
@media screen and (min-resolution: 300dpi) {
  /* Styles for high-resolution screens */
}
```

Das `hover`-Feature prüft, ob der primäre Eingabemechanismus über Elemente schweben kann. Hier ist ein Beispiel:

```css
@media (hover: hover) {
  /* Styles for devices that support hover */
}
```

Das `prefers-color-scheme`-Feature erkennt, ob der Nutzer ein helles oder dunkles Farbschema angefordert hat. Hier ist ein Beispiel:

```css
@media (prefers-color-scheme: dark) {
  /* Styles for dark mode */
}
```

Media Queries können auch mehrere Bedingungen mit logischen Operatoren kombinieren. Der `and`-Operator wird verwendet, um mehrere Media Features zu verbinden, während `not` und `only` verwendet werden können, um Media Queries zu negieren oder zu isolieren. Hier ist ein Beispiel, das mehrere Features kombiniert:

```css
@media screen and (min-width: 768px) and (orientation: landscape) {
  /* Styles for landscape screens at least 768px wide */
}
```

Es ist auch möglich, mehrere Abfragen in einer durch Kommas getrennten Liste anzusprechen, was wie ein „oder“-Operator funktioniert:

```css
@media screen and (min-width: 768px), print {
  /* Styles for screens at least 768px wide OR for print */
}
```

Beim Arbeiten mit Media Queries ist es wichtig, die Kaskade zu berücksichtigen. Media Queries erhöhen nicht die Spezifität – sie gruppieren nur bedingte Regeln. Die normalen Regeln der CSS-Kaskade gelten weiterhin innerhalb jeder Media Query.

In der Praxis werden Media Queries oft verwendet, um responsive Layouts zu erstellen. Ein gängiges Muster ist, einen Basisstil für mobile Geräte zu definieren und dann Media Queries zu verwenden, um das Layout für größere Bildschirme zu verbessern:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<div class="container">
  <p>Lorem ipsum dolor sit amet, consectetur adipiscing elit. Integer nec odio. Praesent libero. Sed cursus ante dapibus diam.</p>
  <p>Sed nisi. Nulla quis sem at nibh elementum imperdiet. Duis sagittis ipsum. Praesent mauris. Fusce nec tellus sed augue semper porta.</p>
  <p>Mauris massa. Vestibulum lacinia arcu eget nulla. Class aptent taciti sociosqu ad litora torquent per conubia nostra, per inceptos himenaeos.</p>
</div>
```

```css
/* Base styles for mobile */
.container {
  width: 100%;
  padding: 15px;
}

/* Styles for tablets */
@media screen and (min-width: 768px) {
  .container {
    width: 750px;
    margin: 0 auto;
  }
}

/* Styles for desktops */
@media screen and (min-width: 1024px) {
  .container {
    width: 960px;
  }
}
```

:::

Dieser Ansatz, bekannt als „mobile-first“ responsives Design, stellt sicher, dass die Basisstile für mobile Geräte geeignet sind und für größere Bildschirme Verbesserungen hinzugefügt werden.

Zusammenfassend sind Media Queries ein mächtiges Werkzeug in CSS, das die Erstellung responsiver, anpassungsfähiger Webdesigns ermöglicht. Durch das Verständnis, wie man verschiedene Medientypen und Funktionen verwendet, können Entwickler*innen Websites erstellen, die optimale Nutzererfahrungen über eine breite Palette von Geräten und Präferenzen bieten. Da sich Webtechnologien ständig weiterentwickeln, hilft das Aktualisieren mit neuen Media Features dabei, noch differenziertere und benutzerfreundlichere responsive Designs zu erstellen.
