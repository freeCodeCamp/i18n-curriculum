# --description--

Nelle lezioni precedenti, hai imparato a lavorare con importazioni ed esportazioni in JavaScript. In questa lezione, vedremo come importare ed esportare componenti in React.

Un'esportazione rende un componente disponibile ad altri file. Un'importazione permette a un file di usare un componente esportato altrove. Nei progetti React, è così che i componenti vengono riutilizzati e combinati per costruire interfacce utente.

In questa lezione, imparerai come esportare componenti React usando esportazioni predefinite ed esportazioni nominate, e come importarli dove servono.

In questo esempio, abbiamo un componente `Cat` che appartiene a un file chiamato `Cat.jsx`. Per l'estensione del file, usiamo l'estensione `.jsx` perché questo file lavora principalmente con JSX.

Questo componente `Cat` restituisce un markup JSX con un titolo e un'immagine di un gatto chiamato Mr. Whiskers:

```jsx
function Cat() {
  return (
    <div className="card">
      <h2>Mr. Whiskers</h2>
      <img
        src="https://cdn.freecodecamp.org/curriculum/cat-photo-app/running-cats.jpg"
        alt="Tuxedo cats running on dirt ground."
      />
    </div>
  );
}
```

Se vogliamo usare il nostro componente `Cat` in un altro file, dobbiamo prima esportarlo così:

```jsx
function Cat() {
  return (
    <div className="card">
      <h2>Mr. Whiskers</h2>
      <img
        src="https://cdn.freecodecamp.org/curriculum/cat-photo-app/running-cats.jpg"
        alt="Tuxedo cats running on dirt ground."
      />
    </div>
  );
}

export default Cat;
```

Usiamo la parola chiave `default` perché questa sarà l'esportazione predefinita dal modulo. Puoi anche scegliere di esportare il componente sulla stessa riga della definizione del componente così:

```jsx
export default function Cat() {
  return (
    <div className="card">
      <h2>Mr. Whiskers</h2>
      <img
        src="https://cdn.freecodecamp.org/curriculum/cat-photo-app/running-cats.jpg"
        alt="Tuxedo cats running on dirt ground."
      />
    </div>
  );
}
```

Puoi scegliere di importare componenti figli in altri file di componenti genitori, oppure importarli nel file del componente radice. Per questo esempio, importeremo il componente `Cat` all'interno del file del componente radice.

Ogni progetto React avrà un componente di livello superiore, tipicamente chiamato `App.jsx`:

```jsx
export default function App() {
  return (
    // return component here
  );
}
```

Questo file si trova solitamente nella directory `src` del tuo progetto. Imparerai di più sulle strutture comuni dei progetti in una lezione futura.

Per usare il componente `Cat` all'interno del componente radice `App`, devi importarlo così:

```jsx
import Cat from "./Cat";

export default function App() {
  return (
    // return component here
  );
}
```

Ora puoi restituire il componente `Cat` all'interno del componente `App` così:

```jsx
import Cat from "./Cat";

export default function App() {
  return (
    <Cat />
  );
}
```

Questo approccio funziona bene quando un file è responsabile di esportare un singolo componente principale, mantenendo la sintassi di importazione semplice e facile da leggere. Ricorda, un file può avere solo un'esportazione predefinita, il che lo rende ideale per un file che contiene principalmente un solo componente.

Ora che sai come usare le esportazioni predefinite, vediamo le esportazioni nominate. Le esportazioni nominate permettono a un file di condividere più componenti o funzioni. A differenza delle esportazioni predefinite, queste devono essere importate usando esattamente il nome con cui sono state esportate (a meno che non le rinomini usando il termine `as`).

Questo è utile quando un file serve come libreria di componenti. Per esempio, ecco un file che contiene due componenti, Cat e Dog:

```jsx
// Animals.jsx
export function Cat() {
  return <h2>Mr. Whiskers</h2>;
}

export function Dog() {
  return <h2>Fido</h2>;
}
```

Usiamo la parola chiave export individualmente su ogni componente. Per usarli in un altro file, li importiamo usando le parentesi graffe:

```jsx
import { Cat, Dog } from "./Animals";

export default function App() {
  return (
    <div>
      <Cat />
      <Dog />
    </div>
  );
}
```

La sintassi `{ Cat, Dog }` dice a JavaScript di importare le esportazioni nominate specifiche dal file `Animals.jsx`. I nomi dentro le parentesi graffe devono corrispondere esattamente ai nomi esportati. Se hai bisogno di evitare conflitti di nomi, o preferisci un nome diverso nel file corrente, puoi rinominare un componente durante l'importazione usando la parola chiave as.

```jsx
import { Cat as Kitty } from "./Animals";

export default function App() {
  return <Kitty />;
}
```

Infine, un file può avere un'esportazione predefinita e più esportazioni nominate.

```jsx
// Animals.jsx
export default function Cat() {
  return <h2>Mr. Whiskers</h2>;
}

export function Dog() {
  return <h2>Fido</h2>;
}
```

Quando importi esportazioni miste, l'esportazione predefinita viene prima (senza parentesi), seguita dalle esportazioni nominate (dentro parentesi):

```jsx
// App.jsx
import Cat, { Dog } from "./Animals";

export default function App() {
  return (
    <div>
      <Cat />
      <Dog />
    </div>
  );
}
```

In questo esempio, `Cat` è l'esportazione predefinita, quindi non usa parentesi graffe. `Dog` è un'esportazione nominata, quindi le richiede.

Capendo come funzionano le esportazioni predefinite e nominate, puoi organizzare i componenti React su più file e riutilizzarli dove serve. Man mano che costruisci applicazioni più grandi, scegliere lo stile di esportazione giusto aiuterà a mantenere il codice chiaro e manutenibile, e renderà più facile lavorare con componenti creati da altri.
