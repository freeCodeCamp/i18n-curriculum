# --description--

Python에서 예외 처리는 견고하고 오류에 강한 프로그램을 작성하는 데 핵심적인 부분입니다. 예외 처리를 통해 오류를 예상하고, 잡아내며, 구조화된 방식으로 대응할 수 있습니다.

예외 처리는 프로그램 실행 중 발생하는 오류를 잡아내고 관리하는 과정으로, 코드가 예기치 않게 중단되지 않도록 합니다.

Python은 오류를 우아하게 처리하기 위해 `try`, `except`, `else`, 그리고 `finally` 블록을 제공합니다. 기본 예시는 다음과 같습니다:

```py
try:
    x = 10 / 0
except ZeroDivisionError:
    print("You can't divide by zero!")
```

- `try`: 오류가 발생할 수 있다고 예상하는 코드 블록입니다.

- `except`: try 내부에서 지정한 유형의 오류가 발생하면 이 블록이 실행됩니다.

- 이 경우, 0으로 나누면 `ZeroDivisionError`가 발생하고, 이 오류를 잡아 처리합니다.

다음은 `else`와 `finally` 블록 사용법도 보여주는 예시입니다:

```py
try:
    x = 10 / 2
except ZeroDivisionError:
    print("You can't divide by zero!")
else:
    print('Division successful:', x)
finally:
    print('This block always runs.')
```

- `else`: try 블록에서 예외가 발생하지 않으면 실행됩니다.

- `finally`: 예외 발생 여부와 상관없이 항상 실행됩니다. 파일 닫기나 자원 해제 같은 정리 작업에 유용합니다.

여러 예외를 별도의 `except` 블록으로 잡을 수도 있습니다:

```py
try:
    number = int('abc')
    result = 10 / number
except ValueError:
    print('That was not a valid number.')
except ZeroDivisionError:
    print("Can't divide by zero.")
```

별도의 `except` 절을 사용하면 오류에 더 구체적이고 유용하게 대응할 수 있습니다.

또한 예외 객체를 `as` 키워드로 다른 이름으로 별칭(alias)하여 사용할 수 있습니다. 여기서는 오류 객체를 `e`로 별칭했습니다:

```py
try:
    x = 1 / 0
except ZeroDivisionError as e:
    print(f'Error occurred: {e}')
```

`e`를 사용하면 실제 오류 메시지나 객체에 접근해 로그를 남기거나 디버깅할 수 있습니다.

튜플로 예외들을 지정해 하나의 `except` 절에서 여러 예외를 잡을 수도 있습니다:

```py
try:
    number = int(input('Enter a number: '))
    result = 10 / number
except (ValueError, ZeroDivisionError) as e:
    print(f'Error occurred: {e}')
```

예외 처리를 통해 프로그램이 오류에서 우아하게 복구할 수 있습니다. `try`, `except`, `else`, 그리고 `finally`를 사용해 잠재적 문제를 예상하고 더 견고한 애플리케이션을 만들 수 있습니다.
