# --description--

木はコンピューターサイエンスの世界で非常に重要です。

**木**とは、特定の種類のグラフです。

グラフが木として分類されるためには、以下の条件を満たす必要があります。

* ループやサイクル（開始ノードと終了ノードが同じ経路）がないこと。
    
* 連結していること（すべてのノードが他のすべてのノードから到達可能であること）。
    

木は非線形のデータ構造で、ノードを階層的に整理します。ノードは子ノード、兄弟ノード、親ノードを持つ場合があります。

ルートノードは木の最上部に位置します。親ノードを持たない唯一のノードです。通常、幅優先探索（BFS）や深さ優先探索（DFS）などのアルゴリズムで、このノードからデータ構造全体の走査を開始します。

これは木のグラフィカルな例です：

<img src="https://cdn.freecodecamp.org/curriculum/lecture-transcripts/what-are-trees-and-tries-and-how-do-they-work-1.png" alt="A tree diagram showing a hierarchical structure with node A at the root, nodes B and C as children of A, and nodes D and E as children of C. Node B is a leaf node.">

ノードは階層的に整理されているため、ノード間には関係があります。

**親ノード**とは、直下に他のノードが接続されているノードのことです。図では、ノードAがノードBとCの親ノードです。

**子ノード**とは、直上のノードに接続されているノードのことです。図では、ノードDとEがノードCの子ノードです。

ノードDとEは**葉ノード**にも分類されます。葉ノードとは子ノードを持たないノードのことです。木の「枝」の終わりと考えることができます。

木のノードには重要なプロパティもあります：

* **深さ**：ルートからそのノードまでの経路の長さ。例えば図では、ノードDの深さは2です。ルートから2つのエッジを通って到達するためです。
    
* **高さ**：そのノードから葉ノードまでの経路の長さ。例えばノードCの高さは1です。葉ノードの1つ上のレベルにあるためです。
    
* **次数**：各ノードが持つ子ノードの数。図では、ノードBは葉ノードなので次数は0です。ノードCは2つの子ノードを持つため次数は2です。
    

木全体にも**高さ**があります。木の高さはルートノードの高さと同じです。

木にはさまざまな種類があり、二分木、二分探索木、AVL木、赤黒木、B木などがあります。

## 二分木と二分探索木

これらは最もよく使われる木の種類の2つです。

**二分木**とは、各ノードが最大で2つの子ノード（左の子ノードと右の子ノード）を持つ木のことです。つまり、これまで見てきた例は二分木です！

<img src="https://cdn.freecodecamp.org/curriculum/lecture-transcripts/what-are-trees-and-tries-and-how-do-they-work-1.png" alt="The same binary tree example highlighting that it's a binary tree, with each node having at most two children.">

**二分探索木**は、二分木のより特定のバージョンで、非常に特別な順序付けのルールがあります。

理解するために、まずサブツリーを理解する必要があります。**サブツリー**とは、木の一部であり、それ自体が木である部分のことです。

例の木では、ノードC、D、Eが単独で木を形成しているため、サブツリーとみなされます。

<img src="https://cdn.freecodecamp.org/curriculum/lecture-transcripts/what-are-trees-and-tries-and-how-do-they-work-3.png" alt="A diagram highlighting a subtree within the main tree, showing nodes C, D, and E forming their own tree structure.">

二分探索木（BST）の順序付けのルールは、すべてのノードに対して、その左のサブツリーの値はノードの値より小さく、右のサブツリーの値はノードの値より大きいことを定めています。

左と右のサブツリーもそれぞれ二分探索木でなければなりません。

この順序付けにより、木がバランスしていれば、検索、挿入、削除の操作が非常に効率的になります。

バランスの取れた木とは、任意のノードの左と右のサブツリーの高さが非常に近く、操作が効率的に行える木のことです。

## トライ木

木と二分探索木について理解したところで、トライ木について見ていきましょう。

**トライ木**は、文字列の集合を格納するために使われる木のデータ構造です。

トライ木は**接頭辞木**とも呼ばれ、接頭辞に基づく文字列の検索に非常に効率的です。

トライ木の各ノードは文字列の1文字を表します。

ルートノードは特定の文字を表さないため、空文字列を表すと考えられます。

ルートから下に向かってトライ木をたどると、ノードへの経路が特定の接頭辞を定義します。単語を見つけるには、その接頭辞に従って目的の単語のノードに到達します。

完全な単語を表すノードには単語の終端マーカーが代入されます。

これは「top」「tea」「ten」という単語を持つトライ木の例です。

「tea」と「ten」は同じ接頭辞「te」を共有しているため、データ構造は最後の文字まで同じ経路をたどり、そのノードが単語の終端であることを赤いボーダーで示しています。

<img src="https://cdn.freecodecamp.org/curriculum/lecture-transcripts/what-are-trees-and-tries-and-how-do-they-work-4.png" alt="A trie data structure showing the words 'top', 'tea', and 'ten'. The root node branches to 't', which then branches to 'o' (leading to 'top') and 'e' (leading to 'tea' and 'ten'). End-of-word nodes are marked with red borders.">

検索操作の最悪ケースの時間計算量はO(L)で、Lは検索する文字列の長さです。

挿入も効率的です。この操作は、トライ木にまだ存在しない文字のノードを新たに作成するだけで済みます。

このデータ構造の大きな利点は、複数の文字列が同じ接頭辞を共有すると、その経路が重複し、接頭辞自体は一度だけ格納されることです。

この効率性により、トライ木はオートコンプリートやスペルチェッカーの機能を実装するのに最適です。

ただし、トライ木はすべての文字列集合に対して効率的とは限りません。文字列集合に多くのユニークな文字が含まれる場合、個別のノードとして多くのユニークな文字を格納する必要があり、単語を見つけるために多くのノードをたどる必要が出てきて効率的ではなくなります。

これでさまざまな種類の木とその用途について理解できたので、実際のシナリオで使い始めることができます。適切な木を選ぶ方法を知ることは、日々の仕事でチャレンジに取り組む際に役立つ貴重なスキルです。
