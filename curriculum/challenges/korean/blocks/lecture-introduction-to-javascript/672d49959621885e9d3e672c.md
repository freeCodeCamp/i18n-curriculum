# --interactive--

JavaScript로 작업할 때, 프로그램 전반에서 사용할 데이터를 저장하기 위해 변수를 선언하는 경우가 많습니다.

현대 JavaScript에서는 `let`와 `const`가 변수를 선언하는 선호되는 방법이지만, 값 할당과 재할당을 처리하는 방식에서 차이가 있습니다.

이번 수업에서는 `let`와 `const`가 변수 선언, 할당, 재할당에서 어떻게 다른지 살펴보겠습니다.

`let` 키워드는 나중에 업데이트하거나 재할당할 수 있는 변수를 선언할 수 있게 해줍니다. `let`를 유연한 컨테이너라고 생각할 수 있습니다 – 한 번 값을 저장하면 프로그램 전반에서 필요에 따라 그 값을 변경할 수 있습니다.

다음은 `let`로 변수를 선언하고 할당하는 예시입니다:

```js
let score = 10;
```

이 경우, 변수 `score`가 선언되고 값 `10`가 할당되었습니다. 나중에 값을 업데이트하고 싶다면 쉽게 할 수 있습니다:

:::interactive_editor

```js
let score = 10;
console.log(score); // 10
score = 20;
console.log(score); // 20
```

:::

이제 `score`는 값 `20`를 갖고 있습니다. 이는 프로그램 실행 중 변수의 값이 변경될 것임을 알 때 `let`가 특히 유용하다는 뜻입니다.

반면에, `const`는 상수를 선언할 때 사용합니다. `const`로 선언된 변수에 값을 할당하면, 재할당할 수 없습니다.

이것은 프로그램 실행 중에 실수로 변경하지 않으려는 값에 `const`를 사용하는 것이 이상적임을 의미합니다.

다음은 `const`로 변수를 선언하고 할당하는 예시입니다:

:::interactive_editor

```js
const maxScore = 100;
console.log(maxScore); // 100
```

:::

`maxScore`에 값 `100`가 할당되면 변경할 수 없습니다:

```js
maxScore = 200; // This will result in an error
```

`const` 변수에 값을 재할당하려 하면 JavaScript 콘솔에서 오류가 발생합니다. `const` 변수는 할당 후 변경 불가능하기 때문입니다.

값을 즉시 할당하지 않고 `let` 변수를 선언할 수 있으며, 나중에 값을 할당할 수도 있습니다:

:::interactive_editor

```js
let age;
console.log(age); // undefined
age = 25;
console.log(age); // 25
```

:::

`const`로 선언된 변수는 선언 시 반드시 값을 할당해야 합니다. 값을 할당하지 않고 `const` 변수를 선언하려 하면 오류가 발생합니다:

```js
const age; // Error: Missing initializer in const declaration
```

나중에 재할당할 변수를 선언해야 할 때는 `let`를 사용하세요. 예를 들어, 변하는 점수를 추적하거나 프로그램에서 시간이 지남에 따라 값을 업데이트할 때입니다.

변경하지 않아야 하는 구성 값이나 설정처럼 상수로 유지해야 할 변수를 선언할 때는 `const`를 사용하세요.

`var` 키워드도 사용할 수 있지만, 더 이상 권장되지 않습니다. `var`는 `let`와 비슷하지만 범위가 더 넓어 프로그램에서 문제를 일으킬 가능성이 더 큽니다.
