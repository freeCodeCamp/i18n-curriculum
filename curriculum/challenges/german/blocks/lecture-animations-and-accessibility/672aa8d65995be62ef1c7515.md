# --interactive--

CSS-Animationen ermöglichen es Ihnen, dynamische, visuell ansprechende Effekte auf Webseiten zu erstellen, ohne JavaScript oder komplexe Programmierung zu benötigen. Sie bieten eine Möglichkeit, Elemente über eine festgelegte Dauer hinweg sanft zwischen verschiedenen Stilen zu überblenden.

Im Kern besteht eine CSS-Animation aus zwei Hauptkomponenten: der `@keyframes`-Regel und der Animationseigenschaft.

Die `@keyframes`-Regel definiert die Phasen und Stile der Animation. Sie legt fest, welche Stile das Element zu verschiedenen Zeitpunkten während der Animation haben soll.

Hier ist ein Beispiel:

```css
@keyframes slide-in {
  0% {
    transform: translateX(-100%);
  }
  100% {
    transform: translateX(0);
  }
}
```

Diese `@keyframes`-Regel mit dem Namen `slide-in` definiert eine Animation, die ein Element von links nach rechts bewegt. Die Prozentsätze geben den Fortschritt der Animation an, wobei `0%` den Anfang und `100%` das Ende darstellt.

Die `translateX`-Funktion in Ihrer `@keyframes`-Animation steuert die horizontale Position eines Elements, während es animiert eingeblendet wird.

Um diese Animation auf ein Element anzuwenden, verwenden Sie die `animation`-Eigenschaft. Dieses Beispiel wiederholt die Animation unendlich oft, damit Sie sie in Aktion sehen können (Sie müssen den interaktiven Editor aktivieren, um Vorschauen zu sehen):

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<div class="sliding-element">Hello, I slide in!</div>
```

```css
@keyframes slide-in {
  0% {
    transform: translateX(-100%);
  }
  100% {
    transform: translateX(0);
  }
}

.sliding-element {
  animation: slide-in 2s ease-out infinite;
}
```

:::

Dies wendet die `slide-in`-Animation auf das Element mit einer Dauer von 2 Sekunden und einer `ease-out`-Timing-Funktion an.

Die `animation`-Eigenschaft ist tatsächlich eine Kurzschreibweise für mehrere einzelne Eigenschaften:

`animation-name`, die angibt, welche `@keyframes`-Regel verwendet werden soll.

`animation-duration`, die festlegt, wie lange die Animation dauern soll.

`animation-timing-function`, die definiert, wie die Animation im Zeitverlauf abläuft – zum Beispiel `ease`, `linear`, `ease-in-out`.

`animation-delay`, die eine Verzögerung vor dem Start der Animation angibt.

`animation-iteration-count`, die bestimmt, wie oft die Animation wiederholt werden soll.

`animation-direction`, die festlegt, ob die Animation vorwärts, rückwärts oder abwechselnd abgespielt wird.

`animation-fill-mode`, die angibt, wie das Element vor und nach der Animation gestylt sein soll.

`animation-play-state`, die es ermöglicht, die Animation zu pausieren und fortzusetzen.

Sie können diese Eigenschaften auch einzeln verwenden, um eine genauere Steuerung zu erhalten:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<div class="complex-animation">Watch my colors change!</div>
```

```css
.complex-animation {
  animation-name: color-change;
  animation-duration: 3s;
  animation-timing-function: linear;
  animation-delay: 1s;
  animation-iteration-count: infinite;
  animation-direction: alternate;
}

@keyframes color-change {
  0% {
    background-color: red;
  }
  50% {
    background-color: blue;
  }
  100% {
    background-color: green;
  }
}
```

:::

Dies erzeugt eine Animation, die kontinuierlich die Hintergrundfarbe eines Elements zwischen Rot, Blau und Grün wechselt.

CSS-Animationen können durch verschiedene Ereignisse ausgelöst werden, wie zum Beispiel das Überfahren eines Elements mit der Maus:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<button class="button">Hover over me!</button>
```

```css
.button {
  background-color: blue;
  transition: background-color 0.3s;
}

.button:hover {
  background-color: red;
}
```

:::

Während dieses Beispiel die `transition`-Eigenschaft verwendet, die für einfache Effekte leichter ist, zeigt es, wie CSS interaktive, animierte Elemente erstellen kann.

Es ist wichtig zu beachten, dass CSS-Animationen zwar mächtig sind, aber sparsam eingesetzt werden sollten. Übermäßiger Einsatz von Animationen kann die Leistung beeinträchtigen und für Nutzer mit bestimmten Barrierefreiheitsbedürfnissen ablenkend oder problematisch sein. Bieten Sie immer Optionen an, um Animationen für Nutzer, die weniger Bewegung bevorzugen, zu reduzieren oder zu deaktivieren.

CSS-Animationen bieten eine Möglichkeit, ansprechende, interaktive Web-Erlebnisse zu schaffen, ohne auf JavaScript angewiesen zu sein. Wenn Sie die Grundsätze von `@keyframes` und `animation`-Eigenschaften verstehen, können Sie Ihre Webdesigns performant und barrierefrei zum Leben erwecken.
