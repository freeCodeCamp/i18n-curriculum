# --description--

`flex-shrink` 类似于 `flex-grow`，但设置了弹性项的“收缩因子”。只有当所有弹性项的大小总和大于其父容器时，`flex-shrink` 才会被应用。例如，如果我们上面提到的 3 个 div 有一个宽度声明为：`width: 100px`，且 `.flex-container` 小于 `300px`，我们的 div 就必须收缩以适应。

默认收缩因子是 `flex-shrink: 1`，这意味着所有项都会均匀收缩。如果你不希望某项收缩，则可以指定 `flex-shrink: 0;`。你也可以指定更高的数字，使某些项的收缩速度高于默认水平。

这是一个示例。如果你缩小你的浏览器窗口，你会注意到 `.two` 的宽度永远不会小于给定的 `250px`，即使 `flex-grow` 规则本应指定每个元素应具有相等的大小。

这里需要注意的一个重要含义是，当你指定了 `flex-grow` 或 `flex-shrink` 时，弹性项不一定会遵守你为 `width` 指定的值。在上面的例子中，所有 3 个 div 都被赋予了 `width` 为 `250px`，但当它们的父元素足够大时，它们会增长以填满它。同样，当父元素太小时，默认行为是它们缩小以适应。这不是一个错误，但如果你没有预料到，可能会感到困惑。
