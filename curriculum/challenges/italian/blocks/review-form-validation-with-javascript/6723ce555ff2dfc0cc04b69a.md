# --interactive--

## Validazione dei moduli con JavaScript

- **Constraint Validation API**: Alcuni elementi HTML, come gli elementi `textarea` e `input`, espongono un'API di validazione dei vincoli. Questa API ti permette di verificare che il valore fornito dall'utente per quell'elemento superi qualsiasi validazione a livello HTML che hai scritto, come la lunghezza minima o la corrispondenza del pattern.
- **Metodo `checkValidity()`**: Questo metodo restituisce `true` se l'elemento soddisfa tutta la validazione HTML (basata sui suoi attributi), e `false` se non la soddisfa.

:::interactive_editor

```html
<form>
  <label>
    Email:
    <input
      id="email"
      type="email"
      required
      pattern=".+\.com$"
      placeholder="example@site.com"
    />
  </label>
</form>

<script>
  const input = document.getElementById("email");

  input.addEventListener("input", (e) => {
    if (!e.target.checkValidity()) {
      e.target.setCustomValidity("You must use a .com email.");
    } else {
      e.target.setCustomValidity("");
    }
  });
</script>
```

:::

- **Metodo `reportValidity()`**: Questo metodo comunica al browser che il `input` non è valido.

:::interactive_editor

```html
<form>
  <label>
    Email:
    <input
      id="email2"
      type="email"
      required
      pattern=".+\.com$"
      placeholder="example@site.com"
    />
  </label>
</form>

<script>
  const input = document.getElementById("email2");

  input.addEventListener("input", (e) => {
    if (!e.target.checkValidity()) {
      e.target.reportValidity();
    }
  });
</script>
```

:::

- **Proprietà `validity`**: Questa proprietà serve per ottenere o impostare lo stato di validità dei controlli del modulo (come `<input>`, `<select>`, ecc.) e fornisce informazioni sul fatto che l'input dell'utente rispetti i vincoli definiti per quell'elemento (ad esempio, campi `required`, vincoli di pattern, lunghezza massima, ecc.).

:::interactive_editor

```html
<input
  id="age"
  type="number"
  min="18"
  placeholder="Enter age (18+)"
/>

<script>
  const input = document.getElementById("age");

  input.addEventListener("input", (e) => {
    console.log(e.target.validity);
  });
</script>
```

:::

- **Proprietà `patternMismatch`**: Sarà `true` se il valore non corrisponde al pattern di espressione regolare specificato.

## Metodo `preventDefault()`

- **Definizione**: Ogni evento che si attiva nel DOM ha un comportamento predefinito. L'evento click su una casella di spunta cambia lo stato di quella casella, per impostazione predefinita. Premere la barra spaziatrice su un pulsante con focus attiva il pulsante. Il metodo `preventDefault()` su questi oggetti `Event` impedisce che quel comportamento avvenga.

:::interactive_editor

```html
<form id="form">
  <input type="text" placeholder="Try to submit" />
  <button type="submit">Submit</button>
</form>

<p id="status"></p>

<script>
  const form = document.getElementById("form");
  const status = document.getElementById("status");

  form.addEventListener("submit", (event) => {
    event.preventDefault();
    status.textContent = "Form submission prevented.";
  });
</script>
```

:::

## Invio dei moduli

- **Definizione**: Ci sono tre modi in cui un modulo può essere inviato. Il primo è quando l'utente clicca un pulsante nel modulo che ha l'attributo `type` impostato su `submit`. Il secondo è quando l'utente preme il tasto `Enter` su qualsiasi campo `input` modificabile nel modulo. Il terzo è tramite una chiamata JavaScript ai metodi `requestSubmit()` o `submit()` dell'elemento `form`.
- **Attributo `action`**: L'attributo `action` dovrebbe contenere un URL o un percorso relativo per il dominio corrente. Questo valore determina dove il modulo tenta di inviare i dati: se non imposti un attributo `action`, il modulo invierà i dati all'URL della pagina corrente.

```html
<form action="https://freecodecamp.org" method="GET">
  <input
    type="number"
    name="number"
    placeholder="Enter a number"
  />
  <button type="submit">Submit</button>
</form>
```

- **Attributo `method`**: Questo attributo accetta un metodo `HTTP` standard, come `GET` o `POST`, e usa quel metodo per effettuare la richiesta all'URL di azione. Quando non viene impostato un metodo, il modulo userà per impostazione predefinita una richiesta `GET`. I dati nel modulo saranno codificati URL come coppie `name=value` e aggiunti all'URL di azione come parametri di query.

```html
<form action="/data" method="POST">
  <input
    type="number"
    id="input"
    placeholder="Enter a number"
    name="number"
  />
  <button type="submit">Submit</button>
</form>
```

- **Attributo `enctype`**: L'elemento `form` accetta un attributo `enctype`, che rappresenta il tipo di codifica da usare per i dati. Questo attributo accetta solo tre valori: `application/x-www-form-urlencoded` (che è il predefinito, inviando i dati come corpo del modulo codificato URL), `text/plain` (che invia i dati in forma di testo semplice, in coppie `name=value` separate da nuove righe), o `multipart/form-data`, che è specifico per gestire moduli con caricamento di file.
