# --description--

Під час створення веб-застосунків важливо продумати організацію своїх файлів і папок. Це допоможе тримати код у порядку та полегшить його підтримку.

Давайте розглянемо приклад структури папок для проєкту з HTML і CSS:

```bash
.
├── /assets
│   ├── /images
│   │   ├── logo.png
│   │   ├── banner.jpg
│   │   └── icons.svg
│   ├── /fonts
│   │   ├── custom-font.woff
│   │   └── custom-font.woff2
├── /css
│   ├── main.css
│   ├── about.css
│   └── contact.css
├── index.html
├── about.html
├── contact.html
└── README.md
```

На початку прикладу є одна крапка, яка позначає кореневу директорію.

Коренева директорія — це директорія найвищого рівня в файловій системі, яка слугує відправною точкою для всіх шляхів до файлів і містить усі інші директорії та файли. Директорія — це інша назва для папки.

Всередині кореневої директорії першою показана директорія `assets`. Термін `assets` означає будь-які файли, які використовуються в проєкті, наприклад зображення, шрифти або інші ресурси.

У цьому прикладі всередині директорії `assets` є дві папки з назвами `images` і `fonts`.

Наступною директорією буде `css`. Іноді її також називають директорією стилів. Тут зберігаються всі ваші CSS-файли.

Під директорією `css` розташовані HTML-файли. Файл `index.html` представляє домашню сторінку, а `about.html` і `contact.html` — сторінки «Про нас» та «Контакти» відповідно.

Нарешті, є файл `README.md`. Файли `README` часто використовують, щоб надати корисну інформацію про проєкт, наприклад, що він робить, як ним користуватися та інші важливі деталі.

Розширення `.md` означає Markdown — легку мову розмітки, яку часто використовують для написання документації.

Організовуючи свій проєкт у папки assets, CSS і HTML-файли, ви полегшуєте пошук потрібного. Це також спрощує іншим розробникам робити внесок у ваш проєкт.

Важливо зазначити, що це лише один із прикладів організації файлів і папок. Існує багато різних способів структурувати проєкт.
