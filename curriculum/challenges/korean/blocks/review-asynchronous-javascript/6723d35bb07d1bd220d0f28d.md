# --description--

- **동기 JavaScript**는 순차적으로 실행되며 이전 작업이 끝날 때까지 다음 작업으로 넘어가지 않고 기다립니다.
- **비동기 JavaScript**는 메인 스레드를 막지 않고 여러 작업을 백그라운드에서 동시에 실행할 수 있게 합니다.
- **스레드**는 메인 프로그램 흐름과 독립적으로 실행할 수 있는 명령어 수열입니다.
- **콜백 함수**는 다른 함수에 인수로 전달되어 작업이 완료되거나 이벤트가 발생한 후 실행되는 함수입니다.

## JavaScript 엔진과 JavaScript 런타임

- **JavaScript 엔진**은 웹 브라우저에서 JavaScript 코드를 실행하는 프로그램입니다. 코드를 컴퓨터가 이해하고 처리할 수 있는 명령어로 변환하는 변환기 역할을 합니다.
- V8은 구글에서 개발한 JavaScript 엔진의 예입니다.
- **JavaScript 런타임**은 JavaScript 코드가 실행되는 환경으로, 코드를 처리하고 실행하는 JavaScript 엔진과 웹 브라우저 또는 Node.js 같은 추가 기능을 포함합니다.

## Fetch API

- Fetch API는 웹 앱이 네트워크 요청을 하도록 해주며, 주로 서버에서 데이터를 가져오거나 보내는 데 사용합니다. 이 요청을 만들 때 사용할 수 있는 `fetch()` 메서드를 제공합니다.
- Fetch API를 사용해 텍스트, 이미지, 오디오, JSON 및 기타 여러 유형의 데이터를 가져올 수 있습니다.

## Fetch API의 HTTP 메서드

Fetch API는 서버와 상호작용하기 위해 다양한 HTTP 메서드를 지원합니다. 가장 일반적인 메서드는 다음과 같습니다:

- **GET**: 서버에서 데이터를 가져올 때 사용합니다. 기본적으로 Fetch API는 데이터를 가져오기 위해 `GET` 메서드를 사용합니다.

```js
fetch('https://api.example.com/data')
```

가져온 데이터를 사용하려면 `.json()` 메서드를 사용해 JSON 형식으로 변환해야 합니다:

```js
fetch('https://api.example.com/data')
  .then(response => response.json())
  .then(data => console.log(data))
```

이 코드에서 `fetch()`는 프로미스를 반환하며, `.then()` 핸들러가 이행된 응답을 JSON 형식으로 변환합니다.

- **POST**: 서버에 데이터를 보낼 때 사용합니다. `POST` 메서드는 서버에 새 리소스를 생성할 때 사용합니다.

```js
fetch('https://api.example.com/users', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({
    name: 'John Doe',
    email: 'john@example.com'
  })
})
```

이 예제에서는 새 사용자를 생성하기 위해 `POST` 요청을 보내고 있습니다. 메서드를 `POST`로 지정하고, 적절한 헤더를 설정했으며, 전송할 데이터를 포함한 본문을 추가했습니다. 본문은 문자열이어야 하므로, 객체를 JSON 문자열로 변환하기 위해 `JSON.stringify()`를 사용합니다.

- **PUT**: 서버의 데이터를 업데이트할 때 사용합니다. `PUT` 메서드는 서버의 기존 리소스를 업데이트할 때 사용합니다.

```js
fetch('https://api.example.com/users/45', {
  method: 'PUT',
  headers: {
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({
    name: 'John Smith',
    email: 'john@example.com'
  })
})
```

이 예제에서는 URL 끝에 지정된 ID `45`를 업데이트하고 있습니다. 코드에서 `PUT` 메서드를 사용했으며, 업데이트할 데이터를 본문으로 지정했습니다.

- **DELETE**: 서버의 데이터를 삭제할 때 사용합니다. `DELETE` 메서드는 서버의 리소스를 삭제할 때 사용합니다.

```js
fetch('https://api.example.com/users/45', {
  method: 'DELETE'
})
```

이 예제에서는 ID `45`를 가진 사용자를 삭제하기 위해 `DELETE` 요청을 보내고 있습니다.

## 프로미스와 프로미스 체이닝

- **프로미스**는 비동기 작업의 최종 완료 또는 실패와 그 결과 값을 나타내는 객체입니다. 프로미스의 값은 비동기 작업이 완료되어야만 알 수 있습니다.
- 간단한 프로미스를 만드는 예는 다음과 같습니다:

```js
const promise = new Promise((resolve, reject) => {
  setTimeout(() => {
    resolve('Data received successfully');
  }, 2000);
});
```

- `.then()` 메서드는 프로미스가 이행되었을 때 실행할 내용을 지정하는 데 사용되고, `.catch()`은 발생하는 오류를 처리하는 데 사용됩니다.
- 프로미스에서 `.then()`과 `.catch()`를 사용하는 예시는 다음과 같습니다:

```js
promise
  .then(data => {
    console.log(data);
  })
  .catch(error => {
    console.error(error);
  });
```

위 예시에서 `.then()` 메서드는 프로미스에서 받은 데이터를 기록하는 데 사용되고, `.catch()` 메서드는 발생하는 오류를 기록하는 데 사용됩니다.

- **프로미스 체이닝**: 프로미스의 강력한 기능 중 하나는 여러 비동기 작업을 연속으로 연결할 수 있다는 점입니다. 각 `.then()`는 새로운 프로미스를 반환할 수 있어, 비동기 작업을 순차적으로 수행할 수 있습니다.
- 프로미스 체이닝 예시는 다음과 같습니다:

```js
fetch('https://api.example.com/data')
  .then(response => response.json())
  .then(data => {
    console.log(data);
    return fetch('https://api.example.com/other-data');
  })
  .then(response => response.json())
  .then(otherData => {
    console.log(otherData);
  })
  .catch(error => {
    console.error(error);
  });
```

위 예시에서는 먼저 한 URL에서 데이터를 가져오고, 첫 번째 응답을 바탕으로 다른 URL에서 데이터를 가져온 후, 마지막으로 두 번째로 받은 데이터를 기록합니다.

`catch` 메서드는 이 과정에서 발생하는 모든 오류를 처리합니다. 따라서 각 단계마다 오류 처리를 추가할 필요가 없어 코드가 훨씬 간단해집니다.

## 프로미스를 처리하기 위해 `async/await` 사용하기

`async`/`await`는 프로미스를 기반으로 하여 비동기 코드를 더 쉽게 작성하고 읽을 수 있게 합니다.

- **async**: `async` 키워드는 비동기 함수를 정의할 때 사용합니다. `async` 함수는 프로미스를 반환하며, 이 프로미스는 `async` 함수가 반환한 값으로 해결됩니다.
- **await**: `await` 키워드는 `async` 함수 내부에서 프로미스가 해결될 때까지 함수 실행을 일시 중지하는 데 사용합니다. `async` 함수 내부에서만 사용할 수 있습니다.
- `async/await` 사용 예시는 다음과 같습니다:

```js
async function delayedGreeting(name) {
  console.log("A Messenger entered the chat...");
  await new Promise(resolve => setTimeout(resolve, 2000));
  console.log(`Hello, ${name}!`);
}

delayedGreeting("Alice");
console.log("First Printed Message!");
```

위 예시에서 `delayedGreeting` 함수는 2초 동안 일시 중지한 후 인사 메시지를 출력하는 `async` 함수입니다. `await` 키워드는 `Promise`가 해결될 때까지 함수 실행을 일시 중지하는 데 사용됩니다.

- `async/await`의 가장 큰 장점 중 하나는 `try/catch` 블록을 통한 오류 처리입니다. 예시는 다음과 같습니다:

```js
async function fetchData() {
  try {
    const response = await fetch('https://api.example.com/data');
    const data = await response.json();
    console.log(data);
  } catch (error) {
    console.error(error);
  }
}

fetchData();
```

위 예시에서 `try` 블록은 오류를 발생시킬 수 있는 코드를 포함하고, `catch` 블록은 오류가 발생할 경우 이를 처리합니다. 이로 인해 오류 처리가 더 명확하고 읽기 쉬워집니다.

## `async` 속성

- `async` 속성은 브라우저가 HTML 문서 파싱을 계속하는 동안 스크립트 파일을 비동기적으로 다운로드하도록 지시합니다.
- 스크립트가 다운로드되면 HTML 파싱이 일시 중지되고 스크립트가 실행된 후 다시 HTML 파싱이 재개됩니다.
- 실행 순서가 중요하지 않은 독립적인 스크립트에는 `async`를 사용하세요.

## `defer` 속성

- `defer` 속성도 스크립트를 비동기적으로 다운로드하지만, HTML 문서가 완전히 파싱된 후에 스크립트 실행을 연기합니다.
- `defer` 스크립트는 HTML 문서에 나타난 순서대로 실행 순서를 유지합니다.

- `async`와 `defer` 속성은 인라인 스크립트에는 무시되고 외부 스크립트 파일에만 작동한다는 점이 중요합니다.

- `async`와 `defer` 속성이 모두 있을 때는 `async` 속성이 우선합니다.

## Geolocation API

- 위치 정보 API는 웹사이트가 사용자의 위치를 요청할 수 있는 방법을 제공합니다.

- 아래 예제는 사용자의 현재 위치를 얻기 위해 API의 `getCurrentPosition()` 메서드를 사용하는 방법을 보여줍니다.

```js
navigator.geolocation.getCurrentPosition(
  (position) => {
    console.log("Latitude: " + position.coords.latitude);
    console.log("Longitude: " + position.coords.longitude);
  },
  (error) => {
    console.log("Error: " + error.message);
  }
);
```

이 코드에서는 `getCurrentPosition`을 호출하고, 위치가 성공적으로 얻어지면 호출될 함수를 전달합니다.

`position` 객체에는 다양한 정보가 포함되어 있지만 여기서는 `latitude`과 `longitude`만 선택했습니다.

`position`를 가져오는 데 문제가 있으면 오류가 콘솔에 기록됩니다.

- 사용자의 프라이버시를 존중하여 필요한 경우에만 위치를 요청하는 것이 중요합니다.
