# --description--

Cercare in una lista di elementi è un'operazione comune in informatica. Ci sono due algoritmi chiave che dovresti conoscere quando si tratta di ricerca: la ricerca lineare e la ricerca binaria.

La ricerca lineare inizia dall'inizio di una lista e itera su ogni elemento finché non trova il valore obiettivo che sta cercando.

Se il valore obiettivo viene trovato, viene restituito l'indice in cui si trova nella lista. Se il valore obiettivo non viene trovato, viene restituito `-1`. Restituiamo `-1` perché non è un indice valido nella maggior parte dei linguaggi di programmazione.

Ecco come appare il codice per la ricerca lineare:

```python
def linear_search(arr, target):
    for i in range(len(arr)):
        if arr[i] == target:
            return i
    return -1
```

Se la lista su cui cercheremo è `[13, 4, 7, 9, 10]` e il valore obiettivo è `9`, la funzione restituirebbe `3` perché `9` si trova all'indice `3`.

Se cambiassimo il valore obiettivo in `5`, la funzione restituirebbe `-1` perché `5` non è nella lista.

Anche se questo è un algoritmo relativamente semplice, non è il più efficiente. Se hai una lista grande di elementi, la ricerca lineare può richiedere molto tempo per trovare il valore obiettivo.

La complessità temporale della ricerca lineare è `O(n)` perché il tempo necessario per cercare nella lista cresce linearmente con la dimensione della lista.

La complessità spaziale della ricerca lineare è `O(1)` perché non richiede spazio aggiuntivo per cercare nella lista.

La ricerca binaria è un algoritmo più efficiente per cercare in una lista grande di elementi. La condizione è che la lista deve essere ordinata in ordine crescente.

La ricerca binaria funziona dividendo la lista a metà e controllando se il valore obiettivo si trova nel mezzo della lista. Se il valore obiettivo è nel mezzo della lista, viene restituito l'indice del valore obiettivo. Altrimenti, l'algoritmo verifica se il valore obiettivo si trova nella metà sinistra o destra della lista.

Continua a dividere le parti rimanenti della lista in metà finché il valore obiettivo non viene trovato. Se il valore obiettivo non è nella lista, restituisce `-1`.

Ecco come appare il codice per la ricerca binaria:

```python
def binary_search(arr, target):
    low = 0
    high = len(arr) - 1

    while low <= high:
        mid = (low + high) // 2  

        if arr[mid] == target:
            return mid
        elif arr[mid] < target:
            low = mid + 1
        else:
            high = mid - 1

    return -1
```

Iniziamo identificando un indice `low` e un indice `high`. Questo rappresenta l'intervallo della lista in cui stiamo cercando.

Poi controlliamo la condizione che `low` sia minore o uguale a `high`. Se `low` è maggiore di `high`, abbiamo cercato in tutta la lista e il valore obiettivo non è stato trovato. In quel caso fermiamo la ricerca e restituiamo `-1`.

Se l'indice `low` è minore o uguale all'indice `high`, calcoliamo l'indice centrale della lista, `mid`. Poi controlliamo se il valore obiettivo si trova all'indice centrale. Se sì, restituiamo l'indice centrale.

Altrimenti, verifichiamo se il valore al punto medio è minore del valore obiettivo. Se lo è, aggiorniamo l'indice basso a essere l'indice centrale più uno. Questo significa che cercheremo nella metà destra della lista.

Infine, se nessuna delle altre condizioni è `True`, aggiorniamo l'indice `high` a essere l'indice centrale meno uno. Questo significa che cercheremo nella metà sinistra della lista.

Continuiamo a ripetere questo processo finché non troviamo il valore obiettivo o determiniamo che non è nella lista.

La complessità temporale della ricerca binaria è `O(log n)` perché il tempo necessario per cercare nella lista cresce in modo logaritmico con la dimensione della lista.

La complessità spaziale della ricerca binaria è `O(1)` perché non richiede spazio aggiuntivo per cercare nella lista.

La ricerca binaria e la ricerca lineare possono essere usate per una varietà di problemi che incontrerai in informatica. È importante capire le differenze tra i due algoritmi e quando usare ciascuno.
