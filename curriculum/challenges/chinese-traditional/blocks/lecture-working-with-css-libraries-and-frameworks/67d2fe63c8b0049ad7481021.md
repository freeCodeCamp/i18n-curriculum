# --description--

CSS 前處理器是一種工具，可透過變數、混入、巢狀結構和選擇器繼承等強大特性擴充標準 CSS。 

這些特性中的一些，例如變數和巢狀，現在已在原生的 CSS 中被支援或獲得更多支援，且這個趨勢很可能會持續。然而，早期版本的原生 CSS 並不支援這些特性。

這就是為什麼 CSS 預處理器最初會被如此廣泛使用的原因。它們會將你撰寫的 CSS 規則和樣式，以及其擴充語法，編譯成瀏覽器能理解的原生 CSS 檔案。

透過使用 CSS 前處理器，你可以以更具重複使用性和邏輯性的方式來組織你的 CSS 程式碼。你也將能存取強大的特性，例如 CSS 本身不直接支援的 mixins。

一些最受歡迎的 CSS 前處理器有 Sass、Less 和 Stylus。

讓我們來談談 Sass。

Sass 代表「語法超棒的樣式表」。它與所有版本的 CSS 相容，並由一個龐大的開發者社群維護。

Sass 支援（某功能）像是：

- 變數，讓你可以在整個試算表中儲存並重複使用數值。

- 巢狀 CSS 規則，讓你能在你的檔案中創建視覺階層體系。

- 模組，讓你可以將樣式拆分成多個樣式表。

- Mixins，讓你可以在整個網站中重複使用 CSS 宣告。

- 繼承，因此多個 CSS 選擇器可以共用屬性。

- 和運算子，用於基本數學操作。

Sass 也支援兩種語法。

SCSS 語法代表 Sassy CSS，它是 CSS 的超集。這表示 SCSS 擴充了 CSS 的基本語法。

SCSS 是你在使用 Sass 時最常見且會遇到的語法。它需要在 CSS 屬性周圍使用大括號（`{` 和 `}`），並且在 CSS 宣告式結尾使用分號（`;`），就像原生的 CSS 一樣。

這些檔案具有 `.scss` 擴充套件。以下是一個範例：

```scss
$primary-color: #3498eb;

header {
  background-color: $primary-color;
}
```

你在上方看到的是在 SCSS 中定義的變數。這個變數用在下面的 CSS 規則中。

讓我們比較 SCSS 與較少使用的語法——縮排語法。這也被稱為「Sass 語法」，因為它是 Sass 的原始語法。

這個語法依賴縮排來定義規則。以下是一個範例：

```scss
$primary-color: #3498eb

header
  background-color: $primary-color
```

請注意，CSS 規則周圍沒有大括號，CSS 宣告式結尾也沒有分號。在最上方，你還可以看到一個變數。請注意，在這個語法中，它的名稱以美元符號（`$`）開頭，這與原生的 CSS 變數不同，但運作方式非常相似。

Sass 也支援一個強大的特性，稱為 mixins。 

Mixin 允許你將多個 CSS 屬性及其值群組在名稱下，並在你的樣式表中重複使用該區塊的 CSS 程式碼。

這使你的 CSS 程式碼較不重複且較容易維護，因為如果你在該區塊中更改了某些內容，該更改會套用到你使用該 mixin 的所有地方。

由於 mixins 可以有訂製且具描述性的名稱，它們有助於理解每個 CSS 程式碼區塊的作用，從而使你的樣式表更容易理解。

而且 mixins 也透過套用統一的樣式來促進樣式表的一致性。

這是 SCSS 語法中 mixin 的範例：

```scss
@mixin center-flex {
  display: flex;
  justify-content: center;
  align-items: center;
}
```

要定義一個 mixin，首先撰寫 `@mixin` at-rule，接著是 mixin 的名稱。在這個例子中，mixin 名為 `center-flex`。它有三個 CSS 屬性，用來使用 Flexbox 將元素置中。

然後，一旦你定義了你的 mixin，就會使用 `@include` at-rule 將那些屬性包含在 CSS 規則中。你只需要寫 `@include`，接著是 mixin 的名稱。在這個例子中，名稱是 `center-flex`：

```scss
section {
  @include center-flex;
  height: 500px;
  background-color: #3289a8;
}
```

這是包含 mixin 和 CSS 規則的完整程式碼：

```scss
@mixin center-flex {
  display: flex;
  justify-content: center;
  align-items: center;
}

section {
  @include center-flex;
  height: 500px;
  background-color: #3289a8;
}
```

你可以在任意多個 CSS 規則中包含該 mixin。如果你需要做任何更改，你只需要更改 mixin 本身，變更將會自動套用到所有地方。此範例使用 SCSS 語法。請注意它有大括號和分號。

這是縮排語法中的等價寫法，也稱為 Sass 語法，沒有大括號或分號：

```scss
@mixin center-flex
  display: flex
  justify-content: center
  align-items: center

section
  @include center-flex
  height: 500px
  background-color: #3289a8
```

這是編譯後的 CSS 程式碼，瀏覽器在檔案編譯成原生 CSS 後實際會解讀的內容：

```css
section {
  display: flex;
  justify-content: center;
  align-items: center;
  height: 500px;
  background-color: #3289a8;
}
```

請注意，它包含了最上方的三個 mixin 屬性：`display`、`justify-content` 和 `align-items`。

這只是 CSS 前處理器能夠做到的簡單範例。它們有許多強大的特性，值得你學習並應用在你的日常工作流程中。

然而，它們確實有你應該注意的潛在缺點。

首先，CSS 預處理器必須將 CSS 規則編譯成標準 CSS。雖然這個編譯處理程序的潛在額外負擔在時間和資源方面通常是最小的，但你應該在實際專案中考慮這一點。

接著，他們可以創建偵錯挑戰，因為瀏覽器直接使用編譯後的 `CSS`。與標準 `CSS` 相比，找出是什麼從擴展語法產生了有問題的樣式可能需要多幾個步驟。

然而，CSS 前處理器的優點通常超過其缺點，尤其是對於複雜的專案。它們對於撰寫更乾淨、可重複使用、較少重複且可擴充的 CSS 非常有幫助。
