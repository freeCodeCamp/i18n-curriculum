# --description--

Manchmal können Regeln miteinander in Konflikt geraten, und wir erhalten unerwartete Ergebnisse. „Aber ich wollte, dass diese Absätze blau sind, warum sind sie rot wie diese anderen Absätze?“ So frustrierend das auch sein kann, ist es wichtig zu verstehen, dass CSS nicht einfach gegen Ihre Wünsche handelt. CSS macht nur das, was wir ihm sagen. Eine Ausnahme davon sind die Standardstile, die von einem Browser bereitgestellt werden. Diese Standardstile unterscheiden sich von Browser zu Browser, und sie sind der Grund, warum manche Elemente einen großen „Abstand“ zu anderen Elementen erzeugen oder warum Schaltflächen so aussehen, wie sie aussehen, obwohl wir keine CSS-Regeln geschrieben haben, um sie so zu gestalten.

Wenn Sie also auf ein unerwartetes Verhalten stoßen, liegt das entweder an diesen Standardstilen, daran, dass Sie nicht verstehen, wie eine Eigenschaft funktioniert, oder daran, dass Sie diese kleine Sache namens Kaskade nicht verstehen.

Die Kaskade bestimmt, welche Regeln tatsächlich auf Ihr HTML angewendet werden. Es gibt verschiedene Faktoren, die die Kaskade verwendet, um dies zu bestimmen. Wir werden drei dieser Faktoren untersuchen, die Ihnen hoffentlich helfen, diese frustrierenden „Ich hasse CSS“-Momente zu vermeiden.
