# --description--

`Event` 对象是一个负载，当用户以某种方式与你的网页交互时触发。 这些交互可以是点击按钮、聚焦 `input` 或摇动他们的 `Mobile` 设备。

像所有 JavaScript 对象一样，`Event` 对象具有许多可能有用的属性。可用的属性取决于触发此负载的事件。

所有 `Event` 对象都将具有 `type` 属性。该属性显示触发负载的事件类型，例如 `"keydown"` 或 `"click"`。这些值将对应于你可能传递给 `addEventListener()` 的相同值，在那里你可以捕捉并使用 `Event` 对象。

`Event` 对象将始终具有 `target` 属性。`target` 属性是对触发该事件的对象的引用。最常见的是某种 `HTMLElement` 对象，或者是 `Document` 或 `Window` 对象。但它也可以是更具体的东西，比如 `AudioContext`。

事件也有方法，这些方法是作为属性暴露在对象上的函数。一个常用的方法是 `preventDefault()`，调用它可以阻止事件的默认行为。

如果你想自己处理表单提交，例如，你可能会调用 `preventDefault()` 来阻止浏览器尝试将表单数据作为 `POST` 请求提交。你将在后续课程中学习更多关于 `POST` 请求的内容。

你也很可能会遇到 `stopPropagation()` 方法。此方法阻止事件向上冒泡或传播到父元素。你将在后续课程中了解这意味着什么。

还有大量特定于某些 `Event` 对象实现的属性。例如，`FetchEvent` 会有一个 request 属性，用于包含触发该事件的请求。 

如果你不确定有哪些属性可用，你可以记录相关的 `Event` 对象，甚至查看文档。
