# --description--

服务器对于现代网络至关重要。它们为我们作为用户每天依赖的在线体验提供支持，并且将为你作为开发者创建的大部分体验提供支持。

让我们更多地了解它们。

正如你所知，互联网是一个由相互连接的计算机网络组成的全局系统。服务器是启用这种通信的核心组件，通过处理交换数据的进程来实现。

**服务器** 是一种计算机或计算机程序，它接收来自其他计算机的请求，并通过网络将数据或服务作为响应发送给它们。

服务器是一个双重概念，因为它可以参考：

* 实际的物理、强大的计算机**硬件**，旨在高效且持续地`Run`以提供这些服务。这些机器是为可靠性而构建和设计的。
    
* 或者运行在该物理硬件上的实际计算机**软件**以提供服务。
    
我们所说的“服务器”是物理的计算机，通常存放在数据中心，数据中心是具有受控环境的大型安全设施。

服务器是**The Client-Server Model**的重要组成部分。

客户端-服务器模型是一种分布式系统架构，其中组件要么是客户，要么是服务器。

* **客户** 是发送请求到服务器以获取数据或服务的进程或应用。例如，网页浏览器。
    
* **服务器**是处理那些请求并返回响应的物理设备或计算机程序。

本质上，客户和服务器是请求-响应周期的一部分，其中客户向服务器发送请求，服务器向客户返回响应。

例如，当你使用浏览器（客户）通过用户名和密码在应用上进行身份验证时，你的信息会被安全地发送到服务器。服务器随后会查看你输入的用户名和密码是否与平台当前数据库中的信息匹配。如果匹配，服务器会向客户（你的浏览器）返回成功的响应，你就成功地在该网站上完成了身份验证。

如果未找到匹配，服务器会向客户返回失败响应，且你通常会看到解释登录尝试失败原因的误差或某种反馈。

所有这些认证逻辑都在服务器上运行。服务器负责验证凭证、与数据库交互以及决定如何响应每个请求。

每个服务器响应都包含一个超文本传输协议状态码，告诉客户发生了什么。这些状态码让客户知道请求是成功还是失败以及原因。

例如：

* 200 表示一切正常。
    
* 404 表示服务器上未找到资源。
    
* 500 表示发生了内部服务器误差。
    
* 503 表示服务器暂时不可用。
    
* 等等……
    

每个状态码都有其独特的含义。

使用服务器的真实世界应用示例包括电子商务、搜索引擎、游戏、视频流、云存储与协作平台、社交媒体平台、金融应用等。

如果数据是动态更新的，或者应用需要与数据库进行某种通信，服务器很可能是通信进程中的关键组件。

这些是一些主要的服务器类型：

* **网络服务器** 提供网页内容，例如 HTML 页面、图像 和 CSS 文件，给浏览器。
    
* **应用服务器** 处理业务逻辑，处理用户输入，并协调响应。
    
* **数据库服务器** 运行数据库管理系统并保存、检索和管理数据。
    
* **邮件服务器** 发送、接收和保存电子邮件消息。
    
* **文件服务器**管理对文件的访问，并允许它们在网络中被保存和共享。
    
* **代理服务器** 作为客户与其他服务器之间的中介，通常用于缓存、安全性或流量控制。

服务器是我们每天使用的强大在线平台的支柱。它们使运行复杂逻辑、保存和处理大量数据以及同时支持数百万用户成为可能。没有服务器，我们依赖的现代数字世界将不存在。
