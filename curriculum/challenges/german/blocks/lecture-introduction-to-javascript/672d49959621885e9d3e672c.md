# --interactive--

Wenn Sie mit JavaScript arbeiten, deklarieren Sie häufig Variablen, um Daten zu speichern, die Sie im gesamten Programm verwenden möchten.

In modernem JavaScript sind `let` und `const` die bevorzugten Methoden, um Variablen zu deklarieren, aber sie unterscheiden sich darin, wie sie Wertzuweisungen und erneute Zuweisungen handhaben.

In dieser Lektion werden wir untersuchen, wie sich `let` und `const` bei der Deklaration, Zuweisung und erneuten Zuweisung von Variablen unterscheiden.

Das Schlüsselwort `let` ermöglicht es Ihnen, Variablen zu deklarieren, die später aktualisiert oder erneut zugewiesen werden können. Sie können `let` als einen flexiblen Container betrachten – sobald Sie einen Wert darin gespeichert haben, können Sie diesen Wert im Verlauf Ihres Programms nach Bedarf ändern.

Hier ist ein Beispiel für die Deklaration und Zuweisung einer Variable mit `let`:

```js
let score = 10;
```

In diesem Fall wird die Variable `score` deklariert und der Wert `10` zugewiesen. Wenn Sie den Wert später aktualisieren möchten, können Sie das ganz einfach tun:

:::interactive_editor

```js
let score = 10;
console.log(score); // 10
score = 20;
console.log(score); // 20
```

:::

Jetzt hält `score` den Wert `20`. Das macht `let` besonders nützlich, wenn Sie wissen, dass sich der Wert einer Variable während der Programmausführung ändern wird.

Andererseits wird `const` verwendet, um Konstanten zu deklarieren. Sobald Sie einer mit `const` deklarierten Variable einen Wert zugewiesen haben, können Sie diesen nicht mehr ändern.

Das macht `const` ideal für Werte, die Sie während der Programmausführung nicht versehentlich ändern möchten.

Hier ist ein Beispiel für die Deklaration und Zuweisung einer Variable mit `const`:

:::interactive_editor

```js
const maxScore = 100;
console.log(maxScore); // 100
```

:::

Sobald `maxScore` der Wert `100` zugewiesen wurde, kann dieser nicht mehr geändert werden:

```js
maxScore = 200; // This will result in an error
```

Der Versuch, einer `const`-Variable einen neuen Wert zuzuweisen, führt zu einem Fehler in Ihrer JavaScript-Konsole, da `const`-Variablen nach der Zuweisung unveränderlich sind.

Sie können eine `let`-Variable deklarieren, ohne ihr sofort einen Wert zuzuweisen, und später einen Wert zuweisen:

:::interactive_editor

```js
let age;
console.log(age); // undefined
age = 25;
console.log(age); // 25
```

:::

Variablen, die mit `const` deklariert wurden, müssen zum Zeitpunkt der Deklaration einen Wert zugewiesen bekommen. Wenn Sie versuchen, eine `const`-Variable ohne Wertzuweisung zu deklarieren, erhalten Sie einen Fehler:

```js
const age; // Error: Missing initializer in const declaration
```

Sie sollten `let` verwenden, wenn Sie Variablen deklarieren müssen, die später erneut zugewiesen werden. Zum Beispiel, um einen sich ändernden Punktestand zu verfolgen oder einen Wert im Verlauf Ihres Programms zu aktualisieren.

Verwenden Sie `const`, wenn Sie Variablen deklarieren möchten, die konstant bleiben sollen, wie Konfigurationswerte oder Einstellungen, die nicht versehentlich geändert werden dürfen.

Sie können auch das Schlüsselwort `var` verwenden, aber es wird nicht mehr so empfohlen. `var` ist so etwas wie `let`, hat aber einen größeren Gültigkeitsbereich, was eher zu Problemen in Ihrem Programm führen kann.
