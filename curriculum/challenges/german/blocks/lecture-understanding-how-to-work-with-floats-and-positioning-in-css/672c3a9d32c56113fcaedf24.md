# --interactive--

Fixed- und sticky-Positionierung sind zwei wichtige CSS-Positionierungsstrategien, die jeweils unterschiedliche Verhaltensweisen im Vergleich zur absoluten Positionierung bieten. Wenn ein Element mit `position: fixed` positioniert wird, wird es aus dem normalen Dokumentenfluss entfernt und relativ zum Viewport platziert, was bedeutet, dass es an derselben Stelle bleibt, selbst wenn der Benutzer scrollt. Dies wird oft für Elemente wie Kopfzeilen oder Navigationsleisten verwendet, die jederzeit sichtbar bleiben müssen. Wenn Sie beispielsweise möchten, dass eine Überschrift oben auf der Seite fixiert bleibt, können Sie folgenden Code verwenden:

```html

<h1>Fixed Header</h1>
```

```css
body {
  margin: 0;
  padding-top: 60px;
  font-family: Arial, sans-serif;
  line-height: 1.6;
}

h1 {
  position: fixed;
  top: 0;
  width: 500px;
  background: white;
  padding: 10px;
  border-bottom: 2px solid #ccc;
}

p {
  max-width: 600px;
  margin: 20px auto;
}
```

:::

In diesem Beispiel haftet das `h1`-Element oben am Viewport und bleibt auch beim Scrollen an dieser Position. Dies ist besonders nützlich, um dauerhafte UI-Elemente wie sticky headers oder immer sichtbare Navigation zu erstellen.

`position: sticky` verhält sich als Hybrid zwischen relativer und fester Positionierung. Anfangs verhält sich das Element so, als wäre es relativ positioniert und bleibt im Fluss des Dokuments. Sobald der Benutzer das Element jedoch über einen bestimmten Punkt hinaus scrollt, „klebt“ es am Viewport (meistens oben) und verhält sich, als wäre es fixiert. Dies ist ideal, um Elemente wie sticky Navigationsleisten zu erstellen, die erst fixiert werden, wenn der Benutzer zu einer bestimmten Position scrollt. So wenden Sie sticky-Positionierung an:

```css
h1 {
  position: sticky;
  top: 30px;
  left: 30px;
}
```

:::

In diesem Beispiel verhält sich das `h1`-Element normal, bis der Benutzer es `30px` vom oberen Rand weg scrollt, woraufhin es am oberen Viewport „klebt“.

Hier sind einige wichtige Unterschiede zwischen absoluter, fester und sticky-Positionierung:

- Absolute Positionierung entfernt ein Element aus dem Dokumentenfluss und positioniert es relativ zum nächsten positionierten Vorfahren oder zum initialen Containing Block, falls keiner existiert. Das Element bleibt an dieser Position, unabhängig vom Scrollen.
- Fixed-Positionierung entfernt das Element ebenfalls aus dem Dokumentenfluss, fixiert es jedoch relativ zum Viewport, sodass es beim Scrollen immer sichtbar an derselben Stelle bleibt.
- Sticky-Positionierung hält das Element zunächst im normalen Fluss, erlaubt ihm aber, nach dem Überschreiten eines definierten Schwellenwerts an Ort und Stelle zu „kleben“.

Diese Strategien ermöglichen es Ihnen, Layouts mit schwebenden Elementen, sticky headers und dynamischeren UI-Komponenten zu erstellen, die die Benutzererfahrung verbessern, indem wichtige Informationen immer zugänglich bleiben.
