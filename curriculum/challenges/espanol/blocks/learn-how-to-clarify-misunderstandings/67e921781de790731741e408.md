# --explanation--

`Clarifying` significa explicar algo de una manera que elimina la confusión. Ayuda a alguien a entender mejor un concepto o situación. Por ejemplo:

`Thanks for clarifying how the API works.` - El hablante está agradecido porque la explicación hizo que la API fuera más fácil de entender.

`Abstraction` se refiere a simplificar algo mostrando solo las características esenciales y ocultando el funcionamiento interno complejo. Por ejemplo:

`Using a login function is an example of abstraction - you don't need to know how the password is checked, just that it works.` - Esto muestra cómo los usuarios interactúan con interfaces simples mientras la lógica técnica permanece oculta.
