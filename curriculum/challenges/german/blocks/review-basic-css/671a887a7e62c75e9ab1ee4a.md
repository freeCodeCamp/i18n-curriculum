## CSS-Grundlagen

- **Was ist CSS?**: Cascading Style Sheets (CSS) ist eine Stylesheet-Sprache, die verwendet wird, um HTML-Elementen Stile zuzuweisen. CSS wird für Farben, Hintergrundbilder, Layouts und mehr verwendet.
- **Grundaufbau einer CSS-Regel**: Eine CSS-Regel besteht aus zwei Hauptteilen: einem Selektor und einem Deklarationsblock. Ein Selektor ist ein Muster, das in CSS verwendet wird, um bestimmte HTML-Elemente für die Gestaltung auszuwählen und anzusprechen. Ein Deklarationsblock wendet eine Reihe von Stilen für einen oder mehrere Selektoren an.

Hier ist die allgemeine Syntax einer CSS-Regel:

```css
selector {
    property: value;
}
```

- **`meta name="viewport"` Element**: Dieses `meta` Element gibt dem Browser Anweisungen, wie die Abmessungen und Skalierung der Seite auf verschiedenen Geräten, insbesondere auf Mobiltelefonen und Tablets, gesteuert werden sollen.
- **Standard-Browserstile**: Jedem HTML-Element werden standardmäßig Browserstile zugewiesen. Dies umfasst normalerweise Dinge wie Standard-Außenabstände und Innenabstände.

## Inline-, interne und externe CSS

- **Inline-CSS**: Diese Stile werden direkt innerhalb eines HTML-Elements mit dem `style` Attribut geschrieben. Meistens werden Sie kein Inline-CSS verwenden, um die Trennung der Verantwortlichkeiten einzuhalten.

Hier ist ein Beispiel für Inline-CSS:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<p style="color: red;">This is a red paragraph.</p>
```

:::

- **Internes CSS**: Diese Stile werden innerhalb der `<style>` Tags im `head` Abschnitt eines HTML-Dokuments geschrieben. Dies kann nützlich sein, um kurze Codebeispiele zu erstellen, aber normalerweise werden Sie kein internes CSS verwenden.
- **Externes CSS**: Diese Stile werden in einer separaten CSS-Datei geschrieben und mit dem `link` Element im `head` Abschnitt mit dem HTML-Dokument verknüpft. Für die meisten Projekte verwenden Sie eine externe CSS-Datei anstelle von internem oder Inline-CSS.

## Arbeiten mit den `width` und `height` Eigenschaften

- **`width` Eigenschaft**: Diese Eigenschaft legt die Breite eines Elements fest. Wenn Sie keine Breite angeben, wird standardmäßig `auto` verwendet. Dadurch bestimmt der Browser die Breite des Elements basierend auf seinem Inhalt, dem übergeordneten Element und dem Anzeige-Typ.
- **`min-width` Eigenschaft**: Diese Eigenschaft legt die Mindestbreite eines Elements fest.
- **`max-width` Eigenschaft**: Diese Eigenschaft legt die Maximalbreite eines Elements fest.
- **`height` Eigenschaft**: Diese Eigenschaft legt die Höhe eines Elements fest. Ebenso ist die Höhe standardmäßig `auto`, was bedeutet, dass sie sich an den Inhalt anpasst.
- **`min-height` Eigenschaft**: Diese Eigenschaft legt die Mindesthöhe eines Elements fest.
- **`max-height` Eigenschaft**: Diese Eigenschaft legt die Maximalhöhe eines Elements fest.

## Verschiedene Arten von CSS-Kombinatoren

- **Nachfahre-Kombinator**: Dieser Kombinator wird verwendet, um Elemente anzusprechen, die Nachfahren eines bestimmten übergeordneten Elements sind. Das folgende Beispiel spricht alle `li` Elemente innerhalb von `ul` Elementen an.

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<ul>
    <li>Example item one</li>
    <li>Example item two</li>
    <li>Example item three</li>
</ul>
```

```css
ul li {
    background-color: yellow;
}
```

:::

- **Kind-Kombinator (`>`)**: Dieser Kombinator wird verwendet, um Elemente auszuwählen, die direkte Kinder eines bestimmten übergeordneten Elements sind. Das folgende Beispiel spricht alle `p` Elemente an, die direkte Kinder der `container` Klasse sind.

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<div class="container">
  <p>This will get styled.</p>

  <div>
    <p>This will not get styled.</p>
  </div>
</div>
```

```css
.container > p {
  background-color: black;
  color: white;
}
```

:::

- **Nächstes Geschwister-Kombinator (`+`)**: Dieser Kombinator wählt ein Element aus, das unmittelbar auf ein bestimmtes Geschwisterelement folgt. Das folgende Beispiel wählt das Absatz-Element aus, das unmittelbar auf das `h2` Element folgt.

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<h2>I am a sub heading</h2>

<p>This paragraph element will get a red background.</p>
```

```css
h2 + p {
  background-color: red;
}
```

:::

- **Nachfolgendes Geschwister-Kombinator (`~`)**: Dieser Kombinator wählt alle Geschwister eines bestimmten Elements aus, die nach ihm kommen. Das folgende Beispiel gestaltet nur das zweite Absatz-Element, da es das einzige ist, das Geschwister des `ul` Elements ist und denselben übergeordneten Container teilt.

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<div class="container">
  <p>This will not get styled.</p>
  <ul>
    <li>Example item one</li>
    <li>Example item two</li>
    <li>Example item three</li>
  </ul>
  <p>This will get styled.</p>
</div>
<p>This will not get styled.</p>
```

```css
ul ~ p {
  background-color: green;
}
```

:::

## Inline-, Block- und Inline-Block-Level-Elemente

- **Inline-Level-Elemente**: Inline-Elemente nehmen nur so viel Breite ein, wie sie benötigen, und beginnen nicht auf einer neuen Zeile. Diese Elemente fließen innerhalb des Inhalts, sodass Text und andere Inline-Elemente neben ihnen erscheinen können. Häufige Inline-Elemente sind `span`, `a` und `img` Elemente.
- **Block-Level-Elemente**: Block-Level-Elemente beginnen auf einer neuen Zeile und nehmen standardmäßig die volle verfügbare Breite ein, die sich über die Breite ihres Containers erstreckt. Einige häufige Block-Level-Elemente sind `div`, `p` und `section` Elemente.
- **Inline-Block-Level-Elemente**: Sie können ein Element mit der `inline-block` Eigenschaft auf `display` setzen. Diese Elemente verhalten sich wie Inline-Elemente, können aber eine `width` und `height` wie Block-Level-Elemente haben.

## Außenabstand und Innenabstand

- **`margin` Eigenschaft**: Diese Eigenschaft wird verwendet, um den Abstand außerhalb des Elements anzuwenden, zwischen dem Rahmen des Elements und den umgebenden Elementen.
- **`padding` Eigenschaft**: Diese Eigenschaft wird verwendet, um den Abstand innerhalb des Elements anzuwenden, zwischen dem Inhalt und seinem Rahmen.
- **`margin` Kurzschreibweise**: Sie können 1–4 Werte angeben, um die Außenabstände festzulegen. Ein Wert gilt für alle vier Seiten; zwei Werte setzen `top` und `bottom`, dann `right` und `left`; drei Werte setzen `top`, horizontal (`right` und `left`), dann `bottom`; vier Werte setzen `top`, `right`, `bottom`, `left`.
- **`padding` Kurzschreibweise**: Sie können 1–4 Werte angeben, um die Innenabstände festzulegen. Ein Wert gilt für alle vier Seiten; zwei Werte setzen `top` und `bottom`, dann `right` und `left`; drei Werte setzen `top`, horizontal (`right` und `left`), dann `bottom`; vier Werte setzen `top`, `right`, `bottom`, `left`.

## CSS-Spezifität

- **Spezifität von Inline-CSS**: Inline-CSS hat die höchste Spezifität, weil es direkt auf das Element angewendet wird. Es überschreibt jedes interne oder externe CSS. Der Spezifitätswert für Inline-Stile ist (1, 0, 0, 0).
- **Spezifität von internem CSS**: Internes CSS wird innerhalb eines `style` Elements im `head` Abschnitt des HTML-Dokuments definiert. Es hat eine geringere Spezifität als Inline-Stile. Sowohl internes als auch externes CSS haben denselben Spezifitätsgrad, der durch ihre Selektoren bestimmt wird, nicht durch ihren Ort.
- **Spezifität von externem CSS**: Externes CSS wird über ein `link` Element im `head` Abschnitt verknüpft und in separaten `.css` Dateien geschrieben. Wie internes CSS wird seine Spezifität durch die verwendeten Selektoren bestimmt. Wenn zwei Regeln dieselbe Spezifität haben, entscheidet die Reihenfolge im Quelltext: die Regel, die später im Dokument erscheint, hat Vorrang. Externes CSS bietet die beste Wartbarkeit für größere Projekte.
- **Universal-Selektor (`*`)**: Ein spezieller CSS-Selektor, der jedes Element im Dokument auswählt. Er wird oft verwendet, um einen Stil auf alle Elemente der Seite anzuwenden, was nützlich sein kann, um Stile zwischen verschiedenen Browsern zurückzusetzen oder zu normalisieren. Der Universal-Selektor hat den niedrigsten Spezifitätswert aller Selektoren. Er trägt 0 zu allen Teilen des Spezifitätswerts bei (0, 0, 0, 0).
- **Typselektoren**: Diese Selektoren sprechen Elemente basierend auf ihrem Tag-Namen an. Typselektoren haben eine relativ niedrige Spezifität im Vergleich zu anderen Selektoren. Der Spezifitätswert für einen Typselektor ist (0, 0, 0, 1).
- **Klassenselektoren**: Diese Selektoren werden durch einen Punkt (`.`) gefolgt vom Klassennamen definiert. Der Spezifitätswert für einen Klassenselektor ist (0, 0, 1, 0). Das bedeutet, dass Klassenselektoren Typselektoren überschreiben können, aber von ID-Selektoren und Inline-Stilen überschrieben werden können.
- **ID-Selektoren**: ID-Selektoren werden durch ein Rautezeichen (`#`) gefolgt vom ID-Namen definiert. ID-Selektoren haben eine sehr hohe Spezifität, höher als Typ- und Klassenselektoren, aber niedriger als Inline-Stile. Der Spezifitätswert für einen ID-Selektor ist (0, 1, 0, 0).
- **`!important` Schlüsselwort**: Wird verwendet, um einer Stilregel die höchste Priorität zu geben, sodass sie alle anderen Deklarationen für eine Eigenschaft überschreibt. Wenn es verwendet wird, zwingt es den Browser, den angegebenen Stil anzuwenden, unabhängig von der Spezifität anderer Selektoren. Sie sollten vorsichtig sein, wenn Sie `!important` verwenden, da es Ihr CSS schwerer wartbar und debugbar machen kann.
- **Cascade-Algorithmus**: Ein Algorithmus, der entscheidet, welche CSS-Regeln angewendet werden, wenn mehrere Stile dasselbe Element ansprechen. Er stellt sicher, dass die passendsten Stile verwendet werden, basierend auf einem Satz klar definierter Regeln.
- **CSS-Vererbung**: Der Prozess, bei dem Stile von übergeordneten Elementen an ihre Kinder weitergegeben werden. Vererbung ermöglicht es Ihnen, Stile auf einer höheren Ebene im Dokumentbaum zu definieren und sie auf mehrere Elemente anzuwenden, ohne sie explizit für jedes Element anzugeben.
