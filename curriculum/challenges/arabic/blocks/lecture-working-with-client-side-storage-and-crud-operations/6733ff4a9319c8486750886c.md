# --description--

توفّر واجهة برمجة تطبيقات التخزين على الويب (Web Storage API) آلية للمتصفحات لتخزين أزواج المفتاح-القيمة داخل المتصفح نفسه، مما يسمح للمطوّرين بتخزين معلومات يمكن استخدامها عبر إعادة تحميل الصفحات والجلسات المختلفة.

المكوّنان الرئيسيان في واجهة برمجة تطبيقات التخزين على الويب هما `localStorage` و`sessionStorage`.

`localStorage` هو الجزء من واجهة برمجة تطبيقات التخزين على الويب الذي يسمح ببقاء البيانات حتى بعد إغلاق نافذة المتصفح أو تحديث الصفحة. تبقى هذه البيانات متاحة حتى يتم إزالتها صراحةً بواسطة التطبيق أو المستخدم.

`sessionStorage` هو جزء آخر من واجهة برمجة تطبيقات التخزين على الويب يخزن البيانات طوال مدة جلسة الصفحة، أي أن البيانات متاحة طالما أن تبويب أو نافذة المتصفح مفتوحة. لكن، على عكس `localStorage`، تُمسح البيانات في `sessionStorage` عند إغلاق التبويب أو النافذة. ستتعلم المزيد عن `sessionStorage` في الدرس التالي.

تشمل الاستخدامات الشائعة لـ `localStorage` تخزين إعدادات المستخدم، مثل السمات أو تفضيلات اللغة، وتذكر بيانات النماذج عبر جلسات المتصفح، وتخزين قطع صغيرة من المعلومات مؤقتًا لتحسين أداء تطبيقات الويب.

التخزين المؤقت يعني تخزين البيانات التي يتم الوصول إليها بشكل متكرر في موقع تخزين مؤقت، يُعرف باسم الكاش، بحيث يمكن تلبية الطلبات اللاحقة لتلك البيانات بسرعة أكبر دون الحاجة إلى إعادة حسابها أو جلبها من مصدر بيانات أبطأ، مثل قاعدة بيانات أو خادم خارجي.

تشمل بعض الطرق الشائعة في `localStorage` الطرق `setItem` و`getItem` و`removeItem` و`clear`.

إليك مثالًا على استخدام الطريقة `setItem()` التي تخزن زوج مفتاح-قيمة في `localStorage`.

```js
localStorage.setItem('username', 'JaneDoe');
```

إذا أردنا بعد ذلك استرجاع قيمة مفتاح معين من `localStorage`، يمكننا استخدام الطريقة `getItem()`.

```js
let username = localStorage.getItem('username');
console.log(username); // JaneDoe
```

لإزالة عنصر من `localStorage` باستخدام مفتاحه، يمكنك استخدام الطريقة `removeItem()`.

```js
localStorage.removeItem('username');
```

لمسح كل البيانات في `localStorage`، يمكنك استخدام الطريقة `clear()`.

```js
localStorage.clear();
```

الآن، لنلق نظرة على مثال نستخدم فيه `localStorage` لتخزين السمة المفضلة للمستخدم.

```js
// Store the user's theme preference
localStorage.setItem('theme', 'dark');

// Retrieve the stored theme preference
const userTheme = localStorage.getItem('theme');
console.log(userTheme); // 'dark'

// Remove the theme preference
localStorage.removeItem('theme');

// Clear all localStorage data
localStorage.clear();
```

في هذا المثال:

- نخزن أولًا اختيار السمة (`dark`) للمستخدم.

- ثم نسترجع تلك السمة ونطبعها في وحدة التحكم.

- وأخيرًا، نوضح كيفية إزالة عنصر محدد أو مسح كل البيانات المخزنة.

يُعد `localStorage` مفيدًا جدًا لتخزين قطع صغيرة من البيانات التي تحتاج إلى البقاء بين الجلسات، لكن من المهم ملاحظة أنه لا ينبغي استخدام `localStorage` لتخزين المعلومات الحساسة، مثل كلمات المرور، لأنه قد يشكل مخاطر أمنية.
