# --description--

이전 수업에서 의존성이 무엇인지 배웠습니다. 여기서는 의존성에 대해 더 배우고 프로젝트에 사용할 외부 패키지를 선택할 때 고려할 점을 알아봅니다.

의존성은 프로젝트가 작동하기 위해 필요한 다른 코드 조각입니다. 보통 다른 개발자가 만들어 npm 레지스트리에 공유한 외부 패키지입니다.

의존성을 사용하면 많은 시간을 절약할 수 있습니다. 모든 것을 직접 만들지 않고도 웹 서버 생성, 라우팅 처리, 데이터 가져오기, 인증 관리 등 흔한 문제를 이미 해결한 패키지를 설치할 수 있습니다.

`package.json` 파일에서 프로젝트가 의존하는 모든 패키지는 `dependencies` 키 아래에 나열되어 있습니다.

```json
{
  "name": "package-dependencies",
  "version": "1.0.0",
  "description": "a demo on how package dependencies work",
  "main": "index.js",
  "scripts": {
    "test": "echo \"Error: no test specified\" && exit 1"
  },
  "keywords": [
    "Node",
    "NPM"
  ],
  "author": "Kolade Chris",
  "license": "ISC",
  "dependencies": {
    "chalk": "^5.6.2",
    "lodash": "^4.17.21"
  }
}
```

위 예시 `package.json` 파일에서 `chalk`와 `lodash`는 프로젝트의 의존성으로 나열되어 있습니다. 이는 프로젝트가 이 패키지들에 의존한다는 뜻입니다. 만약 이들이 설치되어 있지 않고 `package.json` 파일에 나열되어 있지 않으면, 이 패키지를 사용하는 기능들이 작동하지 않습니다.

또한 `devDependencies`라고 하는 다른 의존성 종류가 있습니다. 이들은 프로젝트를 빌드하거나 테스트할 때만 필요하며, 실제 운영 환경에서는 필요하지 않습니다. 일반적인 예는 다음과 같습니다:

* 프로젝트에 변경 사항이 있을 때 자동으로 재시작하는 Nodemon
    
* 테스트용 Jest 또는 Mocha
    
* 코드 린팅용 ESLint
    

예상대로 `devDependencies`는 `devDependencies` 섹션에 `package.json` 파일에 나타납니다.

```json
{
  "name": "package-dependencies",
  "version": "1.0.0",
  "description": "a demo on how package dependencies work",
  "main": "index.js",
  "scripts": {
    "test": "echo \"Error: no test specified\" && exit 1"
  },
  "keywords": [
    "Node",
    "NPM"
  ],
  "author": "Kolade Chris",
  "license": "ISC",
  "dependencies": {
    "chalk": "^5.6.2",
    "lodash": "^4.17.21"
  },
  "devDependencies": {
    "jest": "^30.2.0",
    "nodemon": "^3.1.10"
  }
}
```

프로젝트에서 사용할 패키지를 선택할 때 몇 가지 중요한 점을 기억하세요.

보안이 항상 최우선이어야 합니다. 단 하나의 안전하지 않은 패키지라도 전체 프로젝트에 문제를 일으킬 수 있습니다. 또한 큰 패키지나 최적화가 잘 안 된 패키지는 프로젝트 속도를 느리게 할 수 있으므로 성능도 고려해야 합니다.

인기 있고 잘 지원되는 패키지를 선택하는 것도 도움이 됩니다. 인기 있는 패키지는 보통 활발한 커뮤니티, 좋은 문서, 자주 업데이트되는 특징이 있습니다. 인기도를 쉽게 확인하는 방법 중 하나는 npm에서 주간 다운로드 수를 보는 것입니다. 예를 들어 `chalk` 패키지는 매주 30만 건 이상의 다운로드를 기록합니다.

<img src="https://cdn.freecodecamp.org/curriculum/lecture-transcripts/what-are-package-dependencies-and-what-are-some-considerations-for-choosing-an-external-package-1.webp" alt="screenshot from npm of chalk package statistics">

유지 관리도 중요합니다. 잘 관리되는 패키지는 정기적으로 업데이트되고 다른 소프트웨어의 새 버전과도 잘 작동합니다.

마지막으로 문서를 간과하지 마세요. 명확하고 자세한 문서는 패키지 사용법을 이해하기 쉽게 해주고 프로젝트에 통합할 때 많은 시간을 절약해 줍니다.
