# --description--

要在 SQL 中定義資料表，你需要指定每個行的資料型別。

請記住，這是建立資料表的基本語法。每個行都有自己的資料型別。

```sql
CREATE TABLE table_name(
  column1 data_type column_constraint,
  column2 data_type column_constraint,
  column3 data_type column_constraint,
  ... etc
);
```

作為開發者，你的目標是盡可能準確地選擇這些資料型別。

SQL 中有六種常見的資料型別類別：

- 數值型別，例如 `INTEGER`、`FLOAT`、`SERIAL` 和 `DECIMAL`。
- 日期和時間，例如 `TIMESTAMP`、`DATE` 和 `TIME`。
- 字元和字串。這些包括 `CHAR`、`VARCHAR` 和 `TEXT`。
- Unicode，包括 `NTEXT` 和 `NVARCHAR`。這些用於確保文字無論字元來源為何，都能正確儲存和擷取。
- 二進位用於儲存非文字資料，如影像、音訊和影片、視訊檔案。
- 以及其他雜項資料型別，例如 `XML` 和 `TABLE`。

在本課程中，我們將涵蓋一些在 PostgreSQL 中最廣泛使用的型別。資料型別會因資料庫管理系統而異，但通常相當相似。所以你需要查看文件中的完整列表。

我們將從數值開始。

首先，我們有 `INTEGER` 資料型別：

```sql
units_sold INTEGER
```

在此範例中，我們將此資料型別指定給 `units_sold` 行。

`INTEGER` 型別在資料庫中佔用 4 位元組，且其值的範圍從 -2,147,483,648 到 2,147,483,647。官方 PostgreSQL 文件將其描述為「整數的典型選擇」。

我們也有 `SMALLINT` 和 `BIGINT`，它們基本上與 `INTEGER` 相同，只是因為在資料庫中分配的大小不同，分別具有較小和較大的數字範圍。

PostgreSQL 中一個有用的特性是使用 `SERIAL` 關鍵字來創建行。這裡，使用 `SERIAL` 創建了一個 `id` 行：

```sql
id SERIAL
```

雖然它不是真正的資料型別，但它對於建立唯一識別符行非常有幫助，因為該行將具有 `INTEGER` 型別、不允許 `NULL` 值，且在新增列時會自動遞增。

例如，第一列會自動擁有 `id` 為 `1`，第二列會擁有 `id` 為 `2`，依此類推，為每筆記錄建立唯一的 ID。

在 MySQL 中，`SERIAL` 的等價屬性是 `AUTO_INCREMENT`，用於自動產生連續整數：

```sql
id INT AUTO_INCREMENT
```

你可以看到不同的資料庫管理系統可能有不同的方法來實現你資料庫中完全相同的功能。

這些是最常用的數值資料型別，但有時你可能需要表述文字或字元序列。

為此，你有 `VARCHAR` 資料型別。此資料型別用於可變字串長度。你可以在小括弧內設定最大字元長度：

```sql
name VARCHAR(50)
```

在此範例中，我們定義了一個 `name` 行，其值最多可達 50 個字元長。

此資料型別為字串設定最大長度，但如果你需要儲存任意長度的字串，可以改用 `TEXT`：

```sql
name TEXT
```

除了數字和字串之外，儲存日期和時間也是很常見的。

例如，如果你創建一個資料表來儲存事件，你可能需要以特定格式儲存每個事件的日期。為此，你有 `DATE` 資料型別：

```sql
event_date DATE
```

要儲存時間，你也可以使用 `TIME` 資料型別。<dfn>例如</dfn>，你可以儲存事件開始的時間：

```sql
start_time TIME
```

`TIMESTAMP` 資料型別結合了兩者。它包含日期和時間。如果你寫 `TIMESTAMP WITH TIME ZONE`，它也可能包含時區：

```sql
event_timestamp TIMESTAMP
```

```sql
event_timestamp TIMESTAMP WITH TIME ZONE
```

最後，如果你需要儲存布林值 `TRUE` 或 `FALSE`，你可以使用 `BOOLEAN` 資料型別：

```sql
is_active BOOLEAN
```

現在你已經知道 SQL 中一些最常見的資料型別，但還有更多。你可以查看你的資料庫管理系統的文件，以找到更多關於你可以使用的資料型別的資訊。
