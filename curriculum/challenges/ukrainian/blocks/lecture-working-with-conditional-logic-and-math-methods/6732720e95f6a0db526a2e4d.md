# --interactive--

Бінарні логічні оператори допомагають оцінити два вирази та повернути результат на основі їхньої істинності. Розглянемо три найпоширеніші бінарні логічні оператори: логічне AND, логічне OR і оператор nullish coalescing.

Логічний оператор AND позначається подвійним амперсандом (`&&`). Він перевіряє, чи обидва операнди є істинними, і повертає результат. Якщо обидва операнди істинні, він повертає друге значення, тобто те, що справа:

:::interactive_editor

```js
const result = true && 'hello';

console.log(result); // hello
```

:::

У наведеному вище прикладі текст `hello` виводиться в консоль, бо обидва операнди є `true`. Якщо один із операндів є хибним значенням, він повертає це хибне значення:

:::interactive_editor

```js
const result = 0 && 3;

console.log(result); // 0
```

:::

Оскільки `0` є хибним значенням, у консоль буде виведено число `0`. Якщо ж обидва операнди є хибними значеннями, повертається перше хибне значення:

:::interactive_editor

```js
const result = false && 0;

console.log(result); // false
```

:::

Оскільки `false` є хибним значенням, у консоль буде виведено `false`. Логічний оператор AND корисний, коли потрібно перевірити кілька умов і переконатися, що всі вони істинні, перш ніж продовжувати. Ось приклад:

:::interactive_editor

```js
if (2 < 3 && 3 < 4) {
 console.log('The if block runs'); 
} else {
 console.log('The else block runs');
} 
```

:::

У цій умові, оскільки `2` менше за `3` І `3` менше за `4`, у консоль буде виведено речення `The if block runs`.

Логічний оператор OR перевіряє, чи принаймні один із операндів є істинним. Якщо перший операнд істинний, він повертає це значення:

:::interactive_editor

```js
const result = 'This is truthy' || false;

console.log(result); // This is truthy
```

:::

Якщо перший операнд є хибним, але другий істинний, у консоль буде виведено друге значення:

:::interactive_editor

```js
const result = 0 || 'This is truthy';

console.log(result); // This is truthy
```

:::

Загалом логічний оператор OR часто використовують у інструкціях `if/else` так:

:::interactive_editor

```js
let userInput;

if (userInput || 'Guest') {
 console.log('A user is present');
} else {
 console.log('No user detected');
}
```

:::

Оскільки ми не призначили значення змінній `userInput`, вона наразі є `undefined`. Умова в інструкції `if` перевіряє, чи істинна змінна `userInput` або рядок `Guest`. Оскільки рядок `Guest` у булевому контексті є істинним, у консоль буде виведено рядок `A user is present`.

Оператор nullish coalescing складніший за логічні OR і AND. Позначений подвійним знаком питання (`??`), він допомагає в ситуаціях, коли потрібно повернути значення лише якщо перше є `null` або `undefined`. Ось приклад роботи з оператором nullish coalescing:

:::interactive_editor

```ts
const result = null ?? 'default';

console.log(result); // default
```

:::

Оскільки `null` є nullish-значенням, у консоль буде виведено рядок `default`. Оператор nullish coalescing надзвичайно корисний у випадках, коли лише `null` або `undefined` мають спричинити запасний або значення за замовчуванням. Ось приклад роботи з налаштуваннями переваг користувача:


:::interactive_editor

```ts
const userSettings = {
 theme: null,
 volume: 0,
 notifications: false,
};

let theme = userSettings.theme ?? 'light';
console.log(theme); // light
```

:::

У наведеному вище прикладі ми маємо об’єкт під назвою `userSettings`, який містить пропи `theme`, `volume` і `notifications`. Ми отримуємо доступ до `theme` за допомогою точкової нотації, як `userSettings.theme`. Ви дізнаєтеся більше про роботу з об’єктами в майбутньому уроці. Оскільки поточне значення `theme` користувача встановлено на `null`, у консоль буде виведено рядок `light`.
