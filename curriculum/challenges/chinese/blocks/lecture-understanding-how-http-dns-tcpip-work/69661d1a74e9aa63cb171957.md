# --description--

当你访问一个网站时，首先做的事情是在你的浏览器中输入一个网页地址。

例如，要访问 freeCodeCamp，你需要在浏览器的地址栏输入 `freecodecamp.org`。

这被称为域名。它是在互联网中该资源的唯一可读标识符。

域名与 DNS 的概念密切相关，DNS 是本课的主要主题。让我们先简单介绍一下域名层次结构，这样你就能理解 DNS 在幕后是如何工作的。

域名层次结构是从右向左读取的，从最通用的类别移动到最具体的类别。

域名的主要结构是从右到左：

* **顶层域 (TLD)：** 域名的最后一段，位于最右侧。在 `freecodecamp.org` 的分支中，那就是 `.org`。其他例子包括 `.com` 和 `.edu`。
    
* **二级域名 (SLD)：** 由个人或组织注册的名称。在 `freecodecamp.org` 中是 `freecodecamp`。这通常是你在看到一个网页地址时首先注意到的部分。
    
* **三级域（子域）：** 用于识别主域的特定部分。例如，`www`，代表“World Wide Web”。如今，现代网络服务器会自动处理域路由，因此你可以访问网站时输入或不输入 `www`。其他子域的例子包括 `blog.`、`shop.`、`support.` 和 `api.`。
    

域名对我们人类非常有帮助，因为它们易于记忆和输入到浏览器中。然而，正如你所知，计算机处理的是数字。你的浏览器必须将你输入的域名转换成称为 IP 地址的东西，以查找你要访问的网站。

**IP 地址** 是一个唯一的数字序列，用于标识网络（例如互联网）上的计算机。

IP 地址对于允许客户将请求发送到正确的服务器并获得适当的响应至关重要。IPv4 地址的示例是 `127.0.0.1`。在一种称为 IPv6 的新标准中，相应的 IP 地址是 `0:0:0:0:0:0:0:1`。

关于 IP `addresses` 有很多内容需要学习，随着你继续学习，你可能想更深入地探索它们。但现在，你只需要理解它们是在 `network` 上识别计算机的唯一数字 `addresses`。

从域名和 IP 地址的概念，现在我们可以达到 DNS 的概念。

DNS 代表 **域名系统**。这通常被称为互联网的“电话簿”。

DNS 将你在浏览器中输入的域名翻译为其对应的 IP 地址。它是一个分层且去中心化的命名系统，用于连接到互联网或特定网络的计算机和设备。

当用户在浏览器中输入一个域名，例如 `freecodecamp.org`，DNS 会将该名称映射到其对应服务器的 IP 地址。一旦服务器被识别，客户端的请求就会发送到服务器。服务器根据其编程方式处理该请求，最终向客户端发送响应。这完成了请求-响应周期。

这一切都得益于执行了最初转换到目标 IP 地址的 DNS。

让我们深入了解 DNS 分辨率进程的细节。

## DNS 分辨率进程

* DNS 分辨率通过将查询发送到一系列 DNS 服务器来工作。

* 当你（用户）在浏览器中输入一个域名时，**客户**会向局部的 DNS 服务器发送请求。这个局部的 DNS 服务器被称为**递归的解析器**。这通常由你的互联网服务提供商（ISP）管理。
    
* 递归解析器将该请求转发到**根服务器**。这些是通用的服务器，没有关于 IP 地址的直接信息。它们通过将递归解析器指向更具体的服务器来工作，特别是指向请求的特定顶层域的顶层域（TLD）服务器。例如，负责所有 `.org` 域的服务器。
    
* 递归解析器随后查询**顶层域（TLD）服务器**。该服务器返回持有所请求域（例如，`freecodecamp.org`）记录的特定权威名称服务器的地址。
    
* 递归解析器启动对相应的**权威名称服务器**的最终查询。如果找到**DNS 记录**，则将其返回给递归解析器。
    
* 当递归解析器接收到 IP 地址时，它会将其传递给客户，这样你就可以开始浏览网站。

权威名称服务器返回的是 **DNS Records**，而不仅仅是 IP 地址。DNS Records 是包含有关域的重要信息的结构化文本条目，例如子域信息、信息应缓存的时长以及执行所请求服务所需的信息。

为了优化进程，递归解析器会临时缓存 DNS 记录以加快进程。这样，你就不会在短时间内重复发出相同的请求。这使得整体进程更加高效。

正如你所见，DNS 服务器至关重要，因为它们提供了将域名转换为 IP 地址的初始关键步骤。这个转换过程使每个客户能够定位并连接到正确的服务器。没有这个目录系统，作为现代网络支柱的客户-服务器模型将无法运行。
