ジェシカはマークに安全を確保するために`regular security audits`を実施するよう伝えています。

`security audit`はシステムの脆弱性をチェックするプロセスです。例えば：

`After the security audit, the team made improvements to the system.` - レビュー後に安全性を高めるための変更が行われました。

`Regular`は継続的に行われることを意味します。例えば：

`Regular testing helps find problems before they get worse.` - 頻繁にテストすることで問題を早期に発見できます。
