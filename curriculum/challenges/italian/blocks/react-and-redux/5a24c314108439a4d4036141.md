# --description--

Questa serie di sfide introduce come usare Redux con React. Prima, ecco una revisione di alcuni dei principi chiave di ciascuna tecnologia. React è una libreria per la vista a cui fornisci dati, poi renderizza la vista in modo efficiente e prevedibile. Redux è un framework per la gestione dello stato che puoi usare per semplificare la gestione dello stato della tua applicazione. Tipicamente, in un'app React Redux, crei un unico archivio Redux che gestisce lo stato dell'intera app. I tuoi componenti React si iscrivono solo alle parti di dati nell'archivio rilevanti per il loro ruolo. Poi, mandi azioni direttamente dai componenti React, che a loro volta attivano aggiornamenti nell'archivio.

Anche se i componenti React possono gestire il proprio stato localmente, quando hai un'app complessa, è generalmente meglio mantenere lo stato dell'app in un unico posto con Redux. Ci sono eccezioni quando singoli componenti possono avere uno stato locale specifico solo per loro. Infine, poiché Redux non è progettato per funzionare con React subito, devi usare il pacchetto `react-redux`. Questo fornisce un modo per passare `state` e `dispatch` di Redux ai tuoi componenti React come `props`.

Nelle prossime sfide, prima creerai un semplice componente React che ti permette di inserire nuovi messaggi di testo. Questi vengono aggiunti a un array che viene mostrato nella vista. Questa dovrebbe essere una buona revisione di ciò che hai imparato nelle lezioni di React. Poi, creerai un archivio Redux e azioni che gestiscono lo stato dell'array dei messaggi. Infine, userai `react-redux` per collegare l'archivio Redux con il tuo componente, estraendo così lo stato locale nell'archivio Redux.
