# --description--

In SQL werden Primärschlüssel und Fremdschlüssel verwendet, um Beziehungen zwischen Tabellen herzustellen.

Beginnen wir mit den Primärschlüsseln.

Ein Primärschlüssel ist eine Spalte oder eine Gruppe von Spalten, die jeden Datensatz (Zeile) in einer Tabelle eindeutig identifiziert. Diese Einschränkung stellt sicher, dass keine Datensätze in der Tabelle denselben Wert für den Primärschlüssel haben.

Deshalb kann eine Tabelle nur einen Primärschlüssel haben.

Werte in der Primärschlüsselspalte können auch nicht `NULL` sein, sie haben also immer einen gültigen Wert.

In PostgreSQL müssen Sie nur `PRIMARY KEY` nach dem Datentyp hinzufügen, um eine Spalte zum Primärschlüssel Ihrer Tabelle zu machen:

```sql
column_name data_type PRIMARY KEY
```

Dies ist ein Beispiel für eine `students`-Tabelle:

```sql
CREATE TABLE students (
  student_id SERIAL PRIMARY KEY,
  name VARCHAR(100)
);
```

Jeder Student hat seine eigene `student_id`, und diese ID wird der Primärschlüssel der Tabelle sein. Der `SERIAL`-Typ ist hier nützlich, weil er sicherstellt, dass die `student_id` immer einen eindeutigen Wert hat.

Ein zusammengesetzter Primärschlüssel wird verwendet, wenn eine Tabelle keine einzelne eindeutige Spalte hat, um die Zeile zu identifizieren. In diesem Fall können Sie eine Kombination aus zwei oder mehr Spalten als Primärschlüssel verwenden, die zusammen eindeutig sind.

Dazu geben Sie beim Erstellen der Tabelle an, welche Spalten den zusammengesetzten Primärschlüssel bilden, so:

```sql
CREATE TABLE table_name (
  column1 data_type column_constraint,
  column2 data_type column_constraint,
  column3 data_type column_constraint,
  ...
  PRIMARY KEY (column1, column2)
);
```

Stellen Sie sich vor, Sie haben eine Tabelle mit `student_id`- und `course_id`-Spalten, die anzeigt, welche Studenten in welchen Kursen eingeschrieben sind.

Jeder Student ist in mehreren Kursen eingeschrieben, und jeder Kurs hat mehrere eingeschriebene Studenten. Daher ist keine der beiden Spalten eindeutig – aber Sie werden nie denselben Studenten mehr als einmal in demselben Kurs eingeschrieben haben. Deshalb können Sie die Kombination der beiden Spalten als Primärschlüssel verwenden:

```sql
CREATE TABLE course_enrollments (
  student_id INT,
  course_id INT,
  ...
  PRIMARY KEY (student_id, course_id),
);
```

Primärschlüssel sind entscheidend, um die Einzigartigkeit und Integrität der Daten sicherzustellen, die Datenabfrage zu optimieren und Beziehungen zu anderen Tabellen in einer relationalen Datenbank herzustellen.

Damit kommen wir zum nächsten Schlüsseltyp, den wir in dieser Lektion behandeln: Fremdschlüssel.

Ein Fremdschlüssel ist eine Spalte (oder eine Gruppe von Spalten) in einer Tabelle, die auf den Primärschlüssel einer anderen Tabelle verweist. Eine Tabelle kann mehrere Fremdschlüssel haben.

In diesem Beispiel haben wir zwei Tabellen, `customers` und `orders`:

```sql
CREATE TABLE customers (
  customer_id SERIAL PRIMARY KEY,
  first_name VARCHAR(100) NOT NULL,
  ...
);

CREATE TABLE orders (
  order_id SERIAL PRIMARY KEY,
  customer_id INTEGER,
  ...
  FOREIGN KEY (customer_id) REFERENCES customers(customer_id)
);
```

Der Primärschlüssel der `customers`-Tabelle ist `customer_id` und der Primärschlüssel der `orders`-Tabelle ist `order_id`.

Aber beachten Sie, dass es auch eine `customer_id`-Spalte in der `orders`-Tabelle gibt.

Warum speichern wir Kundendaten in der `orders`-Tabelle?

Wir tun dies, um eine Beziehung zwischen den `customers`- und `orders`-Tabellen herzustellen. Jede Bestellung speichert die ID des Kunden, der sie aufgegeben hat.

Um die Beziehung zwischen den Tabellen herzustellen, machen wir die `customer_id`-Spalte in der `orders`-Tabelle zu einem Fremdschlüssel, der auf die `customer_id`-Spalte in der `customers`-Tabelle verweist:

```sql
FOREIGN KEY (customer_id) REFERENCES customers(customer_id)
```

Werte in einer Fremdschlüsselspalte müssen entweder mit den Primärschlüsselwerten in der referenzierten Tabelle übereinstimmen oder `NULL` sein (wenn der Fremdschlüssel `NULL`s erlaubt).

Warum ist das hilfreich?

Indem sichergestellt wird, dass diese Spalten in beiden Tabellen genau übereinstimmen, verhindern Fremd- und Primärschlüssel die Entstehung von „verwaisten“ Datensätzen, also Datensätzen, die auf einen nicht existierenden Datensatz in einer anderen Tabelle verweisen.

Primär- und Fremdschlüssel sind grundlegend für relationale Datenbanken. Sie ermöglichen den „relationalen“ Aspekt dieser Datenbanken und erlauben es Ihnen, reale Daten auf genaue Weise abzubilden.
