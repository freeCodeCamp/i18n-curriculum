# --description--

Reactにおけるエフェクトとは、コンポーネントのレンダリング処理の外で起こるすべてのことを指します。つまり、ReactがUIのレンダリングの一部として直接扱わないものです。

よくある例としては、データの取得、ブラウザのタブタイトルの更新、ブラウザのローカルストレージの読み書き、ユーザーの位置情報の取得などがあります。これらの操作は外部の世界とやり取りを行い、副作用（サイドエフェクト）と呼ばれます。

Reactはこれらの副作用を扱うために`useEffect`フックを提供しています。`useEffect`を使うと、コンポーネントがレンダリングまたは更新された後に関数を実行できます。

`useEffect`フックがどのように動作し、なぜ現代のReact開発に不可欠なのかを見てみましょう。

`useEffect`フックを使うには、まずインポートする必要があります。

```js
import { useEffect } from "react";
```

次に、このように関数として使います。

```js
useEffect(() => {
  // Your side effect logic (usually a function) here
}, [dependencies]);
```

エフェクト関数はコンポーネントのレンダリング後に実行され、オプションの`dependencies`引数がエフェクトの実行タイミングを制御します。

`dependencies`は「リアクティブな値」（state、props、関数、変数など）の配列、空の配列、または完全に省略することができます。これらのオプションが`useEffect`の動作をどのように制御するかは以下の通りです。

- `dependencies`が1つ以上のリアクティブな値を含む配列の場合、それらが変化するたびにエフェクトが実行されます。

- `dependencies`が空の配列の場合、`useEffect`はコンポーネントが最初にレンダリングされたときに一度だけ実行されます。

- `dependencies`を省略すると、コンポーネントがレンダリングまたは更新されるたびにエフェクトが実行されます。

例えば、この`Counter`アプリケーションでは`dependencies`引数を渡していないため、コンポーネントのレンダリング時と更新のたびにエフェクトが実行されます。

```jsx
import { useState, useEffect } from "react";

const Counter = () => {
  const [count, setCount] = useState(0);

  useEffect(() => {
    console.log("Component renders");
  });

  return (
    <div
      style={{
        display: "flex",
        alignItems: "center",
        flexDirection: "column",
      }}
    >
      <h2>{count}</h2>
      <div>
        <button onClick={() => setCount(count + 1)}>Increase</button>
        <button onClick={() => setCount(count - 1)}>Decrease</button>
      </div>
    </div>
  );
};

export default Counter;
```

しかし、依存関係として空の配列を渡すと、エフェクトは最初のレンダリング時にのみ実行されます。

```js
useEffect(() => {
  console.log('Component renders');
}, []);
```

`count`のstateを依存関係として渡すと、コンポーネントの最初のレンダリング時と`count`が変化したときにエフェクトが実行されます。

```js
useEffect(() => {
  document.title = `The current count is ${count}`;
  console.log('component renders');
}, [count]);
```

設定したエフェクトがコンポーネントのレンダリングライフサイクルを超えて持続する場合、コンポーネントのレンダリングや更新後にその関数を「クリーンアップ」するための別の関数が必要になる場合があります。

例えば、エフェクト関数が`setInterval()`を使ったり、`window.addEventListener()`のようなイベントリスナーを設定したり、サーバーに接続したりする場合、それぞれ`clearInterval()`、`window.removeEventListener()`を実行し、サーバーから切断するためのクリーンアップ関数が必要です。

`useEffect`フックからクリーンアップ関数を返す構文は以下の通りです。

```js
useEffect(() => {
  // Your side effect logic here
  return () => {
    // Cleanup logic here (optional)
  };
}, [dependencies]);
```

例えば、スクロールイベントリスナーを追加した場合、クリーンアップ関数でそれを削除して整理できます。

```js
useEffect(() => {
  const handleScroll = () => {
    // Handle scroll logic
  };
  window.addEventListener("scroll", handleScroll);

  return () => {
    window.removeEventListener("scroll", handleScroll);
  };
}, []);
```
