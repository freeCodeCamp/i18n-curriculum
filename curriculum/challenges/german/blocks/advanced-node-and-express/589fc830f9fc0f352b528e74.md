# --description--

Die folgenden Aufgaben verwenden die `chat.pug`-Datei. Fügen Sie also in Ihrer `routes.js`-Datei eine GET-Route hinzu, die auf `/chat` zeigt, welche `ensureAuthenticated` verwendet und `chat.pug` rendert, wobei `{ user: req.user }` als Argument an die Antwort übergeben wird. Ändern Sie nun Ihre bestehende `/auth/github/callback`-Route, um das `req.session.user_id = req.user.id` zu setzen und zu `/chat` umzuleiten.

`socket.io@~2.3.0` wurde bereits als Abhängigkeit hinzugefügt, also fordern Sie es in Ihrem Server wie folgt mit `http` an (ist in Node.js eingebaut):

```javascript
const http = require('http').createServer(app);
const io = require('socket.io')(http);
```

Da der *http*-Server jetzt auf der *express app* montiert ist, müssen Sie vom *http*-Server aus lauschen. Ändern Sie die Zeile mit `app.listen` zu `http.listen`.

Das Erste, was behandelt werden muss, ist das Lauschen auf eine neue Verbindung vom Client. Das Schlüsselwort <dfn>on</dfn> macht genau das – es lauscht auf ein bestimmtes Ereignis. Es benötigt 2 Argumente: einen String mit dem Titel des ausgelösten Ereignisses und eine Funktion, durch die die Daten übergeben werden. Im Fall unseres Verbindungs-Listeners verwenden Sie `socket`, um die Daten im zweiten Argument zu definieren. Ein Socket ist ein einzelner Client, der verbunden ist.

Um Verbindungen zu Ihrem Server zu lauschen, fügen Sie Folgendes innerhalb Ihrer Datenbankverbindung hinzu:

```javascript
io.on('connection', socket => {
  console.log('A user has connected');
});
```

Damit sich der Client verbinden kann, müssen Sie Folgendes zu Ihrem `client.js` hinzufügen, das von der Seite geladen wird, nachdem Sie authentifiziert sind:

```js
/*global io*/
let socket = io();
```

Der Kommentar unterdrückt den Fehler, den Sie normalerweise sehen würden, da 'io' in der Datei nicht definiert ist. Sie haben bereits eine zuverlässige CDN zur Socket.IO-Bibliothek auf der Seite in `chat.pug` hinzugefügt.

Versuchen Sie nun, Ihre App zu laden und sich zu authentifizieren, und Sie sollten in Ihrer Serverkonsole `A user has connected` sehen.

**Hinweis:** `io()` funktioniert nur, wenn Sie sich mit einem Socket verbinden, der auf derselben URL/dem selben Server gehostet wird. Für die Verbindung zu einem externen Socket, der anderswo gehostet wird, würden Sie `io.connect('URL');` verwenden.

Senden Sie Ihre Seite ab, wenn Sie denken, dass alles richtig ist. Wenn Sie auf Fehler stoßen, können Sie <a href="https://forum.freecodecamp.org/t/advanced-node-and-express/567135#set-up-the-environment-6" target="_blank" rel="noopener noreferrer nofollow">das bis zu diesem Punkt abgeschlossene Projekt ansehen</a>.
