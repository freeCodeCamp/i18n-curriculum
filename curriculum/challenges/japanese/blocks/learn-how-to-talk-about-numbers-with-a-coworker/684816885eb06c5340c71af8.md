# --explanation--

`Initial unit tests` はテストの最初のステップです。開発者はコードの小さな部分（`units`）をチェックして、正しく動作するか確認します。例えば：

`We wrote initial unit tests to check the main functions.` — チームはコードの最も重要な部分が正しく動作することを確認するために初期テストを作成しました。

`Quality assurance (QA) checks` は品質保証チームが行う最終テストで、ソフトウェアが信頼でき、使いやすく、リリース準備が整っていることを確認します。例えば：

`Regular QA checks help prevent bugs from reaching users.` — 品質保証による頻繁なテストは早期にエラーを発見し、ユーザーが問題に直面しないようにします。

アリスは、テスト予算が初期のコードチェックからリリース前の最終テストまでのすべてのステップをカバーしていることを説明しています。
