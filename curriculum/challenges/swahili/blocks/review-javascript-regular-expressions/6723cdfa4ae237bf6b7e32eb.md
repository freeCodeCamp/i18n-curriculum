## Usemi wa kawaida na njia za kawaida

- **Ufafanuzi**: Usemi wa kawaida, au Regex, hutumika kuunda "muundo", ambao unaweza kutumia kukagua dhidi ya mfuatano wa herufi, kutoa maandishi, na zaidi.

```js
const regex = /freeCodeCamp/;
```

- **Njia ya `test()`**: Njia hii inakubali mfuatano wa herufi, ambao ndio mfuatano wa herufi wa kupima kama unalingana na usemi wa kawaida. Njia hii itarudisha thamani ya kweli au siyo kweli ikiwa mfuatano wa herufi unalingana na usemi wa kawaida.

:::interactive_editor

```js
const regex = /freeCodeCamp/;
const test = regex.test("e");
console.log(test); // false
```

:::

- **Njia ya `match()`**: Njia hii inakubali usemi wa kawaida, ingawa pia unaweza kupitisha mfuatano wa herufi ambao utajengwa kuwa usemi wa kawaida. Njia ya `match` inarudisha safu ya data ya mechi kwa mfuatano wa herufi.

```js
const regex = /freeCodeCamp/;
const match = "freeCodeCamp".match(regex);
console.log(match); // ['freeCodeCamp', index: 0, input: 'freeCodeCamp', groups: undefined]
```

- **Njia ya `replace()`**: Njia hii inakubali hoja mbili: usemi wa kawaida wa kulinganisha (au mfuatano wa herufi), na mfuatano wa herufi wa kuchukua nafasi ya mechi (au kitendakazi cha kuendesha kwa kila mechi).

:::interactive_editor

```js
const regex = /Jessica/;
const str = "Jessica is rly kewl";
const replaced = str.replace(regex, "freeCodeCamp");
console.log(replaced); // "freeCodeCamp is rly kewl"
```

:::

- **Njia ya `replaceAll`**: Njia hii hutumika kuchukua nafasi ya matukio yote ya muundo uliobainishwa na mfuatano mpya wa herufi. Njia hii itatupa kosa ikiwa utampa usemi wa kawaida bila kihimarisha cha globali.

:::interactive_editor

```js
const text = "I hate JavaScript! I hate programming!";
const newText = text.replaceAll("hate", "love");
console.log(newText);  // "I love JavaScript! I love programming!"
```

:::

- **Njia ya `matchAll`**: Njia hii hutumika kupata mechi zote za usemi wa kawaida uliotolewa katika mfuatano wa herufi, ikiwa ni pamoja na vikundi vya kunasa, na huzirudisha kama iterator. Iterator ni kitu kinachokuruhusu kupitia (au "kupita juu ya") mkusanyiko wa vitu.

:::interactive_editor

```js
const str = "JavaScript, Python, JavaScript, Swift, JavaScript";
const regex = /JavaScript/g;

const iterator = str.matchAll(regex);

for (let match of iterator) {
  console.log(match[0]); // "JavaScript" for each match
}
```

:::

## Vihimarisha vya usemi wa kawaida

- **Ufafanuzi**: Vihimarisha, ambavyo mara nyingi huitwa "bendera", hubadilisha tabia ya usemi wa kawaida.
- **Bendera ya `i`**: Bendera hii hufanya usemi wa kawaida kupuuza ukubwa wa herufi.

:::interactive_editor

```js
const regex = /freeCodeCamp/i;
console.log(regex.test("freecodecamp")); // true
console.log(regex.test("FREECODECAMP")); // true
```

:::

- **Bendera ya `g`**: Bendera hii, au kihimarisha cha globali, huruhusu usemi wako wa kawaida kulinganisha muundo zaidi ya mara moja.

:::interactive_editor

```js
const regex = /freeCodeCamp/gi;
console.log(regex.test("freeCodeCamp")); // true
console.log(regex.test("freeCodeCamp is great")); // false
```

:::

- **Ufafanuzi wa tagi ya nanga**: Tagi ya nanga ya `^`, mwanzoni mwa usemi wa kawaida, inasema "linganisha mwanzo wa mfuatano wa herufi". Tagi ya nanga ya `$`, mwishoni mwa usemi wa kawaida, inasema "linganisha mwisho wa mfuatano wa herufi".

:::interactive_editor

```js
const start = /^freeCodeCamp/i;
const end = /freeCodeCamp$/i;
console.log(start.test("freecodecamp")); // true
console.log(end.test("freecodecamp")); // true
```

:::

- **Bendera ya `m`**: Tagi za nanga hutafuta mwanzo na mwisho wa mfuatano mzima wa herufi. Lakini unaweza kufanya usemi wa kawaida ushughulikie mistari mingi kwa kutumia bendera ya `m`, au kihimarisha cha mistari mingi.

:::interactive_editor

```js
const start = /^freecodecamp/im;
const end = /freecodecamp$/im;
const str = `I love 
freecodecamp
it's my favorite
`;
console.log(start.test(str)); // true
console.log(end.test(str)); // true
```

:::

- **Bendera ya `d`**: Bendera hii huongeza taarifa unazopata kwenye kitu cha mechi.

```js
const regex = /freecodecamp/di;
const string = "we love freecodecamp isn't freecodecamp great?";
console.log(string.match(regex));
```

Na matokeo ni:

```js
// [
//   'freecodecamp',
//   index: 8,
//   input: "we love freecodecamp isn't freecodecamp great?",
//   groups: undefined,
//   indices: [
//     0: [8, 20],
//     groups: undefined
//   ]
// ]
```

- **`u` Flag**: Hii inaongeza utendaji wa usemi wa kawaida ili kuruhusu kulinganisha herufi maalum za unicode. Bendera ya `u` inakupa upatikanaji wa madarasa maalum kama `Extended_Pictographic` ili kulinganisha emoji nyingi. Pia kuna bendera ya `v`, ambayo inaongeza zaidi utendaji wa kulinganisha unicode.
- **`y` Flag**: Modifaya ya sticky hufanya kazi kwa njia inayofanana sana na modifaya ya global, lakini kwa tofauti chache. Tofauti kubwa ni kwamba usemi wa kawaida wa global utaanza kutoka lastIndex na kutafuta sehemu iliyobaki ya mfuatano wa herufi kwa kulinganisha nyingine, lakini usemi wa kawaida wa sticky utarudisha null na kusahisha lastIndex kuwa 0 ikiwa hakuna kulinganisha mara moja kwenye lastIndex ya awali.
- **`s` Flag**: Modifaya ya mstari mmoja inaruhusu tabia ya wildcard, inayowakilishwa na `.` katika regex, kulinganisha mapumziko ya mistari - kwa ufanisi kutibu mfuatano wa herufi kama mstari mmoja wa maandishi.

## Madarasa ya herufi

- **Wildcard `.`**: Madarasa ya herufi ni sintaksia maalum unayoweza kutumia kulinganisha seti au seti ndogo za herufi. Darasa la kwanza la herufi unalopaswa kujifunza ni darasa la wildcard. Wildcard inawakilishwa na nukta, au dot, na inalinganisha HERUFI YOYOTE moja ISIPOKUWA mapumziko ya mstari. Ili kuruhusu darasa la wildcard kulinganisha mapumziko ya mstari, kumbuka kwamba utahitaji kutumia bendera ya `s`.

```js
const regex = /a./;
```

- **`\d`**: Hii italinganisha tarakimu zote (`0-9`) kwenye mfuatano wa herufi.

```js
const regex = /\d/;
```

- **`\w`**: Hii hutumika kupatanisha herufi yoyote ya neno (`a-zA-Z0-9_`) katika mfuatano wa herufi. Herufi ya neno hufafanuliwa kama herufi yoyote, kutoka a hadi z au A hadi Z, au nambari kutoka 0 hadi 9, au alama ya underscore.

```js
const regex = /\w/;
```

- **`\s`**: Darasa la herufi za nafasi `\s` linaonyeshwa na mkwaju wa nyuma ukifuatiwa na `s`. Darasa hili la herufi linalinganisha nafasi yoyote, ikiwa ni pamoja na mistari mipya, nafasi, tab, na herufi maalum za nafasi za unicode.
- **Kukanusha Madarasa Maalum ya Herufi**: Ili kukanusha mojawapo ya madarasa haya ya herufi, badala ya kutumia herufi ndogo baada ya mkwaju wa nyuma, unaweza kutumia sawa yake ya herufi kubwa. Mfano ufuatao haulinganishi herufi ya nambari. Badala yake, unalinganisha herufi yoyote moja ambayo SI herufi ya nambari.

```js
const regex = /\D/;
```

- **Madarasa Maalum ya Herufi**: Unaweza kuunda madarasa maalum ya herufi kwa kuweka herufi unayotaka kulinganisha ndani ya mabano ya mraba.

```js
const regex = /[abcdf]/;
```

## Uthibitisho wa angalizo la mbele na nyuma

- **Ufafanuzi**: Uthibitisho wa angalizo la mbele na nyuma hukuruhusu kulinganisha miundo maalum kulingana na kuwepo au kutokuwepo kwa miundo inayozunguka.
- **Uthibitisho chanya wa angalizo la mbele**: Uthibitisho huu utalinganisha muundo pale ambapo muundo unafuatwa na muundo mwingine. Ili kujenga angalizo chanya la mbele, unahitaji kuanza na muundo unaotaka kulinganisha. Kisha, tumia mabano ya kawaida kufunika muundo unaotaka kutumia kama sharti lako. Baada ya kufungua bano, tumia `?=` kufafanua muundo huo kama angalizo chanya la mbele.

```js
const regex = /free(?=code)/i;
```

- **Uthibitisho hasi wa angalizo la mbele**: Hii ni aina ya sharti inayotumika kwenye usemi wa kawaida kukagua kwamba muundo fulani hautokei mbele kwenye mfuatano wa herufi.

```js
const regex = /free(?!code)/i;
```

- **Uthibitisho chanya wa angalizo la nyuma**: Uthibitisho huu utalinganisha muundo tu ikiwa umefuatwa na muundo mwingine maalum, bila kujumuisha muundo uliotangulia kwenye mechi.

```js
const regex = /(?<=free)code/i;
```

- **Uthibitisho hasi wa angalizo la nyuma**: Uthibitisho huu unahakikisha kwamba muundo haujafuatwa na muundo mwingine maalum. Unalinganisha tu ikiwa muundo uliobainishwa haujafuatwa mara moja na mfululizo uliotolewa, bila kujumuisha mfululizo uliotangulia kwenye mechi.

```js
const regex = /(?<!free)code/i;
```

## Vihimarisha vya usemi wa kawaida

- **Ufafanuzi**: Vihimarisha kwenye usemi wa kawaida vinafafanua ni mara ngapi muundo (au sehemu ya muundo) unapaswa kujitokeza. Vinasaidia kudhibiti idadi ya matukio ya herufi au vikundi kwenye mechi. Mfano ufuatao unatumika kulinganisha herufi iliyotangulia mara nne kamili.

```js
const regex = /^\d{4}$/;
```

- **`*`** : Inalinganisha matukio 0 au zaidi ya kipengele kilichotangulia.
- **`+`**: Inalinganisha tukio 1 au zaidi la kipengele kilichotangulia.
- **`?`**: Inalinganisha tukio 0 au 1 la kipengele kilichotangulia.
- **`{n}`**: Inalinganisha matukio n kamili ya kipengele kilichotangulia.
- **`{n,}`**: Inalinganisha matukio n au zaidi ya kipengele kilichotangulia.
- **`{n,m}`**: Inalinganisha matukio kati ya n na m ya kipengele kilichotangulia.

## Vikundi vya kunasa na rejeleo la nyuma

- **Vikundi vya kunasa**: Kikundi cha kunasa kinakuruhusu "kunasa" sehemu ya mfuatano wa herufi uliolinganishwa ili kuitumia utakavyo. Vikundi vya kunasa hufafanuliwa na mabano ya kawaida yanayobeba muundo wa kunasa, bila herufi za mwanzo kama kwenye angalizo la mbele.

```js
const regex = /free(code)camp/i;
```

- **Makundi yasiyokunasa**: Kundi lisilokunasa ni sawa na kundi la kunasa lakini halihifadhi sehemu ya mfuatano uliolingana kwa matumizi ya baadaye. Makundi yasiyokunasa huainishwa na `(?:...)`.

```js
const regex = /free(?:code)camp/i;
```

- **Rejeleo la nyuma**: Rejeleo la nyuma kwenye usemi wa kawaida linamaanisha njia ya kutumia tena sehemu ya muundo uliolinganishwa awali kwenye usemi huo huo. Inakuruhusu kurejelea kikundi kilichonaswa (sehemu ya muundo kwenye mabano) kwa nambari yake. Kwa mfano, `$1` inarejelea kikundi cha kwanza kilichonaswa.

:::interactive_editor

```js
const regex = /free(co+de)camp/i;
console.log("freecoooooooodecamp".replace(regex, "paid$1world"));
```

:::

- Unaweza kutumia rejea za nyuma ndani ya regex yenyewe kulinganisha maandishi yale yale yaliyokunwa na kundi lililotangulia kwa kutumia mkwaju wa nyuma na nambari ya kundi la kunasa. Kwa mfano:

:::interactive_editor

```js
const regex = /(hello) \1/i;
console.log(regex.test("hello hello")); // true
console.log(regex.test("hello world")); // false
```

:::
