# --description--

An diesem Punkt fragen Sie sich vielleicht, wie Sie diese Fehler beheben können.

1. Sie können damit beginnen zu verstehen, dass die Fehlermeldung Ihr Freund ist – nicht Ihr Feind. Fehlermeldungen sagen Ihnen genau, was an Ihrem Code falsch ist und welche Zeilen Sie untersuchen sollten, um die Fehlerquelle zu finden. Ohne Fehlermeldungen wäre das Debuggen unseres Codes ein Albtraum – denn er würde immer noch nicht funktionieren, Sie wüssten nur nicht warum!

1. Sie sollten Ihre Fehlermeldung im Web suchen, denn meistens finden Sie eine Lösung oder Erklärung auf StackOverflow oder in der Dokumentation. Wenn nichts anderes, erhalten Sie so mehr Klarheit darüber, warum Sie diesen Fehler bekommen.

1. Eine weitere Möglichkeit ist die Verwendung von `console.log()` — das ist eine beliebte Wahl für schnelles Debugging. Für aufwändigere Fehlersuche ist der Debugger möglicherweise besser geeignet, aber `console.log()` ist großartig, um sofortiges Feedback zu erhalten, ohne Ihre Funktionen Schritt für Schritt durchgehen zu müssen. Es gibt auch andere nützliche Methoden wie `console.table()`, `console.trace()` und mehr!
