# --description--

假设你已经编写了以下代码：

```javascript
const a = "Hello";
const b = "World";

console.log(c);
```

这段代码会运行，但会生成一个误差。从技术角度来说，这称为<dfn>抛出</dfn>误差。误差的第一部分显示误差的类型。这为你提供了正在处理内容的第一个线索。你将在本课后面学习更多不同的误差类型。在此示例中，你有一个 `ReferenceError`。

<img src="https://cdn.statically.io/gh/TheOdinProject/curriculum/175b5ef2a1b4758a7b75f4ef43d7e27203e5707b/foundations/javascript_basics/understanding_errors/imgs/00.png" width="100%" alt="An error being displayed in the developer console"/>

当引用当前作用域内未声明和/或未初始化的变量时，会抛出 `ReferenceError`。在我们的分支中，错误信息解释了错误发生的原因是因为 `c is not defined`。

这种类型的不同误差根据导致 `ReferenceError` 的原因有不同的消息。例如，你可能遇到的另一个消息是 `ReferenceError: can't access lexical declaration 'X' before initialization`。

误差的下一部分告诉我们可以找到误差的文件名（在此分支中为 `script.js`），以及行号。

这使你可以轻松导航到代码中有问题的行。这里，误差源自 `script.js` 的第四行，该行作为一个链接显示在错误信息下，文本为 `script.js:4`。如果你点击此链接，大多数浏览器会在开发者工具的 Sources 标签（页）中导航到代码的确切行及脚本的其余部分。

有时你的浏览器控制台还会显示发生错误的行中的列（或字符）。在我们的示例中，这将是 `script.js:4:13`。
