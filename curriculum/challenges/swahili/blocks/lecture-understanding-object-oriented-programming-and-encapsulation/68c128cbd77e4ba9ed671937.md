# --description--

Getters na setters ni njia zinazokuwezesha kudhibiti jinsi sifa za darasa la vitu zinavyopatikana na kubadilishwa. Kwa getters unapata thamani, na kwa setters unaweka thamani.

Hizi hufanyika kupitia kile kinachojulikana kama sifa. Hizo ndizo zinazounganisha getters na setters, na kuruhusu upatikanaji wa data.

Sifa hufanya kazi kama sifa za kawaida lakini hutenda kama njia chini ya uso. Fikiria kama data unayoeleza kama njia, lakini hufanya kazi kama sifa. Hii inamaanisha unaweza kupata sifa kwa kutumia uandishi wa nukta badala ya mabano ya kawaida.

Kitu kikuu ambacho sifa hufanya ni kuendesha mantiki ya ziada nyuma ya pazia wakati unapata, unaweka, au unafuta thamani kwa kutumia sifa hizo. Hii inawafanya chaguo bora wakati unataka kupata au kuendesha data ndani ya vitu.

Kwa nini basi kutumia sifa badala ya njia? Hii ni zaidi kuhusu urahisi wa kusoma na mkataba. Zinafanya msimbo wako kuwa safi na rahisi kusoma.

Unapotumia njia, lazima daima uitaje kwa mabano ya kawaida. Lakini kwa sifa, unaweza kuipata kama sifa ya kawaida kwa kutumia uandishi wa nukta. Hii inafanya msimbo wako uonekane rahisi hata ukiendesha kazi za ziada nyuma ya pazia.

Kwa mfano, unaweza kutaka kuhesabu thamani au kuangalia kama thamani mpya ni halali kabla ya kuihifadhi. Badala ya kuita njia kwa ajili ya hilo, unaweza kutumia njia ya sifa kama sifa kufanya hivyo.

Katika Python, dekoreta ni kitendakazi kinachobadilisha utendaji wa vitendakazi vingine, au madarasa, bila kubadilisha msimbo wao wa awali.

Ingawa inawezekana kuunda dekoreta maalum, hii ni zaidi ya wigo wa funzo hili. Jua tu kwamba kuunda sifa, unaeleza njia na kuweka dekoreta `@property` juu yake. Hii hubadilisha njia kuwa sifa, hivyo inaweza kupatikana kama sifa huku ndani ikipiga wito kwa njia iliyodekorewa.

Hii inatupeleka kwa getters. Hapa ni jinsi ya kuunda moja kwa kutumia dekoreta `@property`:

```py
class Circle:
    def __init__(self, radius):
        self._radius = radius

    @property
    def radius(self): # A getter to get the radius
        return self._radius
  
    @property
    def area(self):  # A getter to calculate area
        return 3.14 * (self._radius ** 2)

my_circle = Circle(3)

print(my_circle.radius) # 3
print(my_circle.area) # 28.26
```

Mfano huu unapata radius na eneo la mduara.

Angalia jinsi tulivyotumia `_radius` badala ya radius ndani ya darasa. Alama ya mstari chini ni mkataba wa kawaida wa Python kuonyesha kuwa sifa ni ya faragha. Kwa maneno mengine, inaashiria kuwa ni kwa matumizi ya ndani na haipaswi kufikiliwa moja kwa moja kutoka nje ya darasa.

Ili kuunda setter ya radius, kwa mfano, lazima ueleze njia nyingine yenye jina lile lile na utumie `@<property_name>.setter` juu yake:

Kutumia `self.radius` ndani ya `__init__` huhakikisha setter inafanya kazi wakati wa kuunda kitu, hivyo thamani zisizo halali za radius zinakamatwa mara moja.

```py
class Circle:
    def __init__(self, radius):
        self.radius = radius # Calling the setter

    @property
    def radius(self):  # A getter to get the radius
        return self._radius

    @radius.setter
    def radius(self, value):  # A setter to set the radius
        if value <= 0:
            raise ValueError('Radius must be positive')
        self._radius = value

my_circle = Circle(3)
print('Initial radius:', my_circle.radius) # Initial radius: 3

my_circle.radius = 8
print('After modifying the radius:', my_circle.radius) # After modifying the radius: 8
```

Katika mfano huu, setter ya radius haijiweka tu radius ya mduara, pia inaendesha uthibitisho unaohakikisha radius si nambari chanya.

Mara tu unapobainisha getters na setters, Python huwaita moja kwa moja chini ya pazia kila unapotumia sintaksia ya sifa ya kawaida:

```py
my_circle.radius # This will call the getter
my_circle.radius = 4 # This will call the setter
```

Kumbuka kuwa ndani ya setter, huwezi kutumia jina lile lile la sifa wakati wa kuweka thamani mpya. Hii ni kwa sababu `self.radius = value` itaita setter ndani ya njia ya setter yenyewe, na kusababisha mzunguko usioisha na `RecursionError`. Kwa hivyo lazima daima utumie fomu yenye kiambishi awali cha mstari chini `self._radius = value`.

Kama unavyoweza kudhibiti jinsi sifa inavyopatikana kupitia getter na jinsi inavyobadilishwa na setter, pia unaweza kudhibiti jinsi inavyofutwa kwa kutumia deleter.

Deleter huendesha mantiki maalum unapotumia kauli del kwenye sifa. Kuunda deleter, tumia dekoreta `@<property_name>.deleter`:

```py
class Circle:
    def __init__(self, radius):
        self.radius = radius

    # Getter
    @property
    def radius(self):
        return self._radius

    # Setter
    @radius.setter
    def radius(self, value):
        if value <= 0:
            raise ValueError("Radius must be positive")
        self._radius = value

    # Deleter
    @radius.deleter
    def radius(self):
        print("Deleting radius...")
        del self._radius
```

Hapa ni jinsi deleter anavyoweza kutumika:

```py
# Create circle object with a radius
my_circle = Circle(33)
print("Initial radius:", my_circle.radius)  # 33

# Delete the radius
# This calls the deleter
del my_circle.radius # Deleting radius...
print("Radius deleted!") # Radius deleted!

# Try to access radius after deletion
try:
    print(my_circle.radius)
except AttributeError as e:
    print("Error:", e) # Error: 'Circle' object has no attribute '_radius'
```

Jambo kuu unalopata kutoka hapa ni:

- Getters hukuruhusu kupata thamani au hata kuhesabu thamani papo hapo.
- Setters hukuruhusu kubadilisha thamani kwa usalama kwa kuendesha ukaguzi kabla ya kuweka thamani.
- Sifa ndizo zinazounganisha getters na setters ili uweze kuandika mantiki huku bado ukitumia uandishi wa nukta.
- Deleters hukuruhusu kufafanua kinachotokea wakati sifa inafutwa.
