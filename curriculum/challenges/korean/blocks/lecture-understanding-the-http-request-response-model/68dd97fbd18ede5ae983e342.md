# --description--

지난 수업에서 서버가 HTTP 요청을 처리하고 응답을 보내는 과정을 배웠습니다. 그 응답에는 웹에서 볼 수 있는 다양한 자산이 포함되어 있습니다.

HTTP 응답에서 반환되는 자산에는 텍스트 기반 자산, 미디어 자산, 폰트 등이 있습니다. 각 자산에는 브라우저가 자산을 올바르게 처리하고 표시하는 방법을 알려주는 **Content-Type** 헤더가 함께 제공됩니다.

텍스트 기반 자산은 항상 웹의 기본 요소였습니다. 사실 웹은 순수한 텍스트로 시작했습니다. 오늘날 텍스트 기반 자산에는 다음이 포함됩니다:

* **HTML (Hypertext Markup Language)**: 모든 웹페이지의 기본 구조입니다. HTML 없이는 웹페이지가 존재할 수 없습니다.
    
* **CSS (Cascading Stylesheet)**: 웹페이지의 외형을 제어하는 자산입니다. CSS를 사용해 HTML에 색상, 폰트, 레이아웃 같은 스타일을 적용합니다.
    
* **JavaScript**: HTML과 CSS에 상호작용과 동적 콘텐츠를 추가합니다.
    
* **JSON과 XML**: 이 두 형식은 서버와 브라우저 간에 구조화된 데이터를 전송할 수 있게 해줍니다.
    
미디어 자산에는 이미지, 오디오, 비디오가 포함됩니다. 이들은 시각과 소리로 웹에 생명을 불어넣습니다.

많은 웹사이트가 시각 정보를 전달하기 위해 이미지를 사용합니다. 이미지는 `JPG`, `JPEG`, `PNG`, `WEBP`, `GIF` 같은 다양한 형식으로 제공됩니다.

YouTube와 Vimeo 같은 웹사이트는 `MP4`, `MOV`, `AVI`, `WebM` 등 다양한 형식의 비디오 콘텐츠를 제공합니다.

오디오는 음악, 팟캐스트, 기타 음향 콘텐츠를 스트리밍할 수 있게 해 많은 플랫폼에서 중요한 역할을 합니다. 일반적인 오디오 형식에는 `MP3`, `AAC`, Ogg, `WAV`이 포함됩니다. Spotify와 YouTube Music 같은 플랫폼은 이러한 형식을 사용해 고품질 오디오 경험을 제공합니다.

HTTP 응답에서 반환되는 다른 자산으로는 `PDF`, `DOCX`, `PAGES` 같은 문서, `ZIP`와 `RAR` 같은 압축 아카이브, `WOFF`, `WOFF2`, `TTF` 같은 폰트 파일, `EXE` 같은 실행 파일이 있습니다.

다음은 이러한 일반적인 자산과 그 Content-Type 헤더를 정리한 표입니다:

| 자산 종류 | Content-Type 헤더 |
| --- | --- |
| HTML | text/html |
| CSS | text/css |
| JavaScript | application/javascript |
| JPEG | image/jpeg |
| PNG | image/png |
| GIF | image/gif |
| SVG | image/svg+xml |
| WebP | image/webp |
| WOFF | font/woff |
| WOFF2 | font/woff2 |
| TTF | font/ttf |
| OTF | font/otf |
| MP4 | video/mp4 |
| WebM | video/webm |
| Ogg (비디오) | video/ogg |
| MP3 | audio/mpeg |
| AAC | audio/aac |
| WAV | audio/wav |
| Ogg (오디오) | audio/ogg |
| JSON | application/json |
| XML | application/xml |
| PDF | application/pdf |
| ZIP | application/zip |
