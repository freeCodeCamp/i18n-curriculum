# --description--

`Promise` — це об’єкт, який представляє собою майбутнє завершення або невдачу асинхронної операції. Спочатку він перебуває у стані очікування. Потім він може перейти або у виконаний стан, коли операція успішна, або у відхилений стан, коли операція зазнає невдачі. Ось приклад створення `promise`:

```js
const aPromise = new Promise((resolve, reject) => {
  setTimeout(() => {
    resolve("Operation successful!");
  }, 1000);
});
```

У цьому прикладі ми створюємо проміс, який імітує асинхронну операцію за допомогою `setTimeout`. Через одну секунду проміс виконується з повідомленням `Operation successful!`.

Інший спосіб роботи з промісами — це використання методів `.then` та `.catch`.

Метод `.then()` використовується в промісі, щоб вказати, що має відбутися, коли проміс виконується, тоді як `.catch()` використовується для обробки будь-яких помилок, які виникають. Давайте подивимося, як ми можемо використати `.then` та `.catch` з нашим промісом:

```js
aPromise.then((result) => {
  console.log(result);  // Outputs: "Operation successful!"
}).catch((error) => {
  console.error(error);
});
```

У цьому коді, або інструкціях, що виконуються, коли проміс виконується, функція, передана в `.then()`, буде викликана з виконаним значенням промісу. Якщо виникне помилка, замість цього буде викликана функція, передана в `.catch()`.

Тепер поговоримо про ланцюжок промісів. Одна з потужних функцій промісів полягає в тому, що ми можемо ланцюжком поєднувати кілька асинхронних операцій. Кожен `.then()` може повертати новий проміс, що дозволяє виконувати послідовність асинхронних операцій одна за одною. Ось приклад:

```js
fetch('https://api.example.com/data')
  .then(response => response.json())
  .then(data => {
    console.log(data);
    return fetch('https://api.example.com/data2');
  })
  .then(response => response.json())
  .then(data2 => console.log(data2))
  .catch(error => console.error('Error:', error));
```

У цьому прикладі ми робимо два виклики API послідовно. Перший `.then()` аналізує відповідь як JSON. Другий `.then()` виводить дані в консоль і робить ще один виклик API. Третій `.then()` аналізує відповідь другого виклику API, а четвертий `.then()` виводить ці дані. Якщо в будь-який момент у цьому ланцюжку виникне помилка, її буде перехоплено `.catch()` у кінці.

Важливо зазначити, що `.catch()` перехопить помилки з будь-якого з попередніх кроків у ланцюжку. Це означає, що вам не потрібно додавати обробку помилок для кожного окремого кроку, що значно спрощує ваш код.
