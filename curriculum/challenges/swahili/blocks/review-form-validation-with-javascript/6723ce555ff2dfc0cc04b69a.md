# --interactive--

## Kuhakiki fomu kwa JavaScript

- **API ya uthibitisho wa vigezo**: Baadhi ya vipengele vya HTML, kama vile vipengele vya `textarea` na `input`, vinaweka wazi API ya uthibitisho wa vigezo. API hii inakuruhusu kuthibitisha kwamba thamani aliyoingiza mtumiaji kwa kipengele hicho inapita uthibitisho wowote wa HTML uliouandika, kama vile urefu wa chini au kulinganisha muundo.
- **Njia ya `checkValidity()`**: Njia hii inarudisha `true` ikiwa kipengele kinapita uthibitisho wote wa HTML (kulingana na sifa zake), na `false` ikiwa kinashindwa.

:::interactive_editor

```html
<form>
  <label>
    Email:
    <input
      id="email"
      type="email"
      required
      pattern=".+\.com$"
      placeholder="example@site.com"
    />
  </label>
</form>

<script>
  const input = document.getElementById("email");

  input.addEventListener("input", (e) => {
    if (!e.target.checkValidity()) {
      e.target.setCustomValidity("You must use a .com email.");
    } else {
      e.target.setCustomValidity("");
    }
  });
</script>
```

:::

- **Njia ya `reportValidity()`**: Njia hii inaiambia kivinjari kwamba `input` si sahihi.

:::interactive_editor

```html
<form>
  <label>
    Email:
    <input
      id="email2"
      type="email"
      required
      pattern=".+\.com$"
      placeholder="example@site.com"
    />
  </label>
</form>

<script>
  const input = document.getElementById("email2");

  input.addEventListener("input", (e) => {
    if (!e.target.checkValidity()) {
      e.target.reportValidity();
    }
  });
</script>
```

:::

- **Sifa ya `validity`**: Sifa hii hutumika kupata hali ya uhalali wa udhibiti wa fomu (kama `<input>`, `<select>`, n.k.) na hutoa taarifa kuhusu kama ingizo la mtumizi linakidhi vizingiti vilivyowekwa kwa kipengele hicho (mfano, sehemu zinazohitajika `required`, vizingiti vya muundo, urefu wa juu zaidi, n.k.).

:::interactive_editor

```html
<input
  id="age"
  type="number"
  min="18"
  placeholder="Enter age (18+)"
/>

<script>
  const input = document.getElementById("age");

  input.addEventListener("input", (e) => {
    console.log(e.target.validity);
  });
</script>
```

:::

- **Sifa ya `patternMismatch`**: Hii itakuwa `true` ikiwa thamani haifanani na muundo wa usemi wa kawaida uliobainishwa.

## Njia ya `preventDefault()`

- **Ufafanuzi**: Kila tukio linalotokea kwenye DOM lina tabia fulani ya chaguo-msingi. Tukio la kubofya kwenye kisanduku cha kuchagua hubadilisha hali ya kisanduku hicho, kwa chaguo-msingi. Kubonyeza space bar kwenye kitufe kilicho kwenye makini huamsha kitufe hicho. Njia ya `preventDefault()` kwenye vitu hivi vya `Event` husimamisha tabia hiyo isitokee.

:::interactive_editor

```html
<form id="form">
  <input type="text" placeholder="Try to submit" />
  <button type="submit">Submit</button>
</form>

<p id="status"></p>

<script>
  const form = document.getElementById("form");
  const status = document.getElementById("status");

  form.addEventListener("submit", (event) => {
    event.preventDefault();
    status.textContent = "Form submission prevented.";
  });
</script>
```

:::

## Kuwasilisha fomu

- **Ufafanuzi**: Kuna njia tatu ambazo fomu inaweza kuwasilishwa. Ya kwanza ni pale mtumiaji anapobofya kitufe kwenye fomu ambacho kina sifa ya `type` iliyowekwa kuwa `submit`. Ya pili ni pale mtumiaji anapobonyeza kitufe cha `Enter` kwenye sehemu yoyote ya `input` inayoweza kuhaririwa kwenye fomu. Ya tatu ni kupitia wito wa JavaScript kwa njia za `requestSubmit()` au `submit()` za kipengele cha `form`.
- **Sifa ya `action`**: Sifa ya `action` inapaswa kuwa na URL au njia ya jamaa kwa domain ya sasa. Thamani hii inaamua fomu itajaribu kutuma data wapi - ikiwa hutoi sifa ya `action`, fomu itatuma data kwenye URL ya ukurasa wa sasa.

```html
<form action="https://freecodecamp.org" method="GET">
  <input
    type="number"
    name="number"
    placeholder="Enter a number"
  />
  <button type="submit">Submit</button>
</form>
```

- **Sifa ya `method`**: Sifa hii inakubali njia ya kawaida ya `HTTP`, kama vile `GET` au `POST`, na hutumia njia hiyo wakati wa kutuma ombi kwenye action URL. Wakati njia haijawekwa, fomu itatumia ombi la `GET` kwa chaguo-msingi. Data kwenye fomu itasimbwa kwenye URL kama jozi za `name=value` na kuongezwa kwenye action URL kama hoja za utafutaji.

```html
<form action="/data" method="POST">
  <input
    type="number"
    id="input"
    placeholder="Enter a number"
    name="number"
  />
  <button type="submit">Submit</button>
</form>
```

- **Sifa ya `enctype`**: Kipengele cha `form` kinakubali sifa ya `enctype`, ambayo inawakilisha aina ya usimbaji wa kutumia kwa data. Sifa hii inakubali tu thamani tatu: `application/x-www-form-urlencoded` (ambayo ni chaguo-msingi, hutuma data kama mwili wa fomu uliosimbwa kwenye URL), `text/plain` (ambayo hutuma data kwa maandishi wazi, katika jozi za `name=value` zilizotenganishwa na mistari mipya), au `multipart/form-data`, ambayo ni maalum kwa kushughulikia fomu zenye upakiaji wa faili.
