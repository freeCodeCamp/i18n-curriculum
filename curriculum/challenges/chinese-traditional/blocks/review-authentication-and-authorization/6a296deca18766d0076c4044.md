## 認證與授權的差異

**認證** 是驗證使用者身份的過程。它回答「這個人真的是他們聲稱的那個人嗎？」當使用者登入時，後端伺服器會將他們的憑證與註冊時儲存的記錄進行比對。

後端開發者使用三大類認證因素：

- **你知道的東西：** 密碼、PIN 碼或安全問題答案。
- **你擁有的東西：** 智慧型手機上的簡訊驗證碼，或驗證器應用程式產生的臨時語彙單元。
- **你是什麼：** 生物特徵資料，如指紋或臉部辨識掃描。

**授權** 在認證之後接手，決定已認證的使用者被允許執行什麼操作。它回答「這個身份擁有哪些權限？」常見角色包括：

- **標準使用者：** 可以編輯自己的個人資料，但無法查看其他使用者的私人資料。
- **編輯者：** 可以撰寫和發布內容，但無法刪除程式碼庫。
- **管理員：** 擁有完整權限修改資料庫、刪除使用者和更改系統整體設定。

**認證必須先進行，然後才是授權。** 在知道使用者是誰之前，無法決定要授予哪些權限。

## JSON Web Tokens（JWT）

JWT 是伺服器在使用者成功登入後發給使用者的數位鑰匙卡。使用者在每次後續請求時出示它，以證明身份而無需重新輸入密碼。

JWT 的四步流程：

1. **登入：** 使用者將憑證傳送給伺服器。
2. **發行：** 伺服器驗證憑證並生成 JWT，然後回傳給使用者。
3. **儲存：** 客戶端通常將 JWT 儲存在瀏覽器的 cookie 或區域儲存中。
4. **請求：** 對受保護路由的每次未來請求，客戶端都會帶上 JWT，伺服器則進行驗證。

JWT 由三個部分組成，彼此以兩個點分隔：

- **標頭：** 編碼的 JSON，宣告語彙單元型別（JWT）和使用的雜湊演算法，如 HMAC SHA256。
- **有效載荷：** 編碼的 JSON，包含關於使用者的聲明，如使用者 ID、使用者名稱或角色。標頭和有效載荷是 Base64 編碼，非加密，因此切勿在此存放敏感資料如密碼。
- **簽名：** 由伺服器專有的祕密金鑰將編碼後的標頭和有效載荷一起雜湊產生。如果有效載荷被竄改，簽名將不匹配，伺服器會以 `401 Unauthorized` 回應拒絕請求。

JWT 使得**無狀態認證**成為可能。伺服器不需要在資料庫中儲存會話資料，因為簽名證明了它是語彙單元的創建者。

## 跨站請求偽造（CSRF）

CSRF 是一種攻擊，強迫已認證的使用者在不知情的情況下執行網頁應用程式上的不當操作。惡意網站誘使受害者瀏覽器向後端伺服器發送偽造請求，利用瀏覽器會自動附加 cookie 到任何發往目標伺服器的請求的特性。

**攻擊範例流程：**

1. 你登入，例如 `trusted-bank.com`。銀行在你的瀏覽器 cookie 中儲存會話語彙單元。
2. 未登出情況下，你造訪 `evil-hacker-site.com`。
3. 惡意網站暗中向 `trusted-bank.com/transfer?amount=1000&to=hacker` 提交請求。
4. 你的瀏覽器自動附加銀行會話 cookie，伺服器因此處理該請求。

惡意網站無法讀取你的 cookie。它只是利用瀏覽器會自動附加 cookie 到任何發往目標伺服器的請求這一事實。

**防禦 CSRF 的方法：**

- **`SameSite=Strict`：** 如果請求來自第三方網站，瀏覽器絕不會送出 cookie。
- **`SameSite=Lax`：** 允許在安全瀏覽（點擊連結）時送出 cookie，但阻擋跨站表單提交和 API 請求時送出 cookie。
- **CSRF 語彙單元：** 使用者載入頁面時，伺服器會產生隨機、唯一且不可預測的語彙單元，並附加到前端表單或會話中。客戶端必須在請求標頭或表單資料中包含它。因為惡意網站無法存取前端來複製語彙單元，偽造請求會驗證失敗。

## Passport.js

Passport.js 是 Node.js 的認證中介層，透過稱為 **Strategies** 的模組化外掛標準化你如何認證請求。擁有超過 500 種策略可用，你可以透過安裝相關軟體包添加任何認證方法：

- `passport-local`：使用者名稱和密碼對照你自己的資料庫。
- `passport-jwt`：驗證受保護 API 路由上的 JSON Web Tokens。
- `passport-google-oauth20`：使用 Google 帳號登入。
- `passport-github2`：使用 GitHub 帳號登入。

安裝核心軟體包和一個策略：

```bash
npm install passport passport-local
```

在定義路由前將 Passport 接入你的 Express 應用程式：

```js
const passport = require('passport');
const LocalStrategy = require('passport-local').Strategy;

app.use(passport.initialize());

passport.use(new LocalStrategy((username, password, done) => {
  // database look-up and password verification logic
}));
```

用 Passport 作為中介層保護登入路由：

```js
app.post('/login', passport.authenticate('local'), (req, res) => {
  res.send(`Welcome back, ${req.user.username}!`);
});
```

認證成功後，Passport 會自動處理**會話管理**（將使用者資料儲存在會話 cookie 中）和**請求裝飾**（將使用者物件附加到 `req.user`，供後續路由處理常式使用）。

## Helmet.js

Helmet.js 是 Node.js 的安全中介層集合，透過自動設定、修改或移除 Express 預設暴露的 HTTP 回應標頭來保護你的應用程式。

一步安裝並套用：

```bash
npm install helmet
```

```js
const helmet = require('helmet');
app.use(helmet());
```

將 `app.use(helmet())` 放在中介層堆疊頂端，對每個回應套用這些保護：

- **移除 X-Powered-By：** Express 預設會以 `X-Powered-By: Express` 宣告自身存在。Helmet 移除此標頭，防止攻擊者針對特定版本漏洞發動攻擊。
- **設定 Content-Security-Policy：** 限制瀏覽器載入腳本、圖片和樣式的來源，防禦跨站指令碼（XSS）攻擊。
- **X-Frame-Options 設為 SAMEORIGIN：** 防止你的網站被嵌入外部 iframe，阻擋點擊劫持攻擊。
- **設定 Strict-Transport-Security：** 強制瀏覽器僅透過 HTTPS 與你的伺服器通訊，保護資料不被不受信任網路攔截。
