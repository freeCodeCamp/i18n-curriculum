# --description--

前のチャレンジで学んだチルダが依存関係の最新のパッチをインストールできるのと同様に、キャレット (`^`) は将来のアップデートもインストールできるようにします。違いは、キャレットはマイナーアップデートとパッチの両方を許可することです。

現在の`@freecodecamp/example`のバージョンは`~1.2.13`であるべきで、これによりnpmは最新の`1.2.x`バージョンをインストールできます。もしバージョン接頭辞としてキャレット (^) を使った場合、npmは任意の`1.x.x`バージョンにアップデートできるようになります。

```json
"package": "^1.3.8"
```

これにより、パッケージの任意の`1.x.x`バージョンへのアップデートが可能になります。

# --instructions--

依存関係の`^`のバージョンにキャレット (`@freecodecamp/example`) を接頭辞として付けて、npmが新しいマイナーリリースにアップデートできるようにしてください。

**注意:** バージョン番号自体は変更しないでください。

# --hints--

`"dependencies"`には`"@freecodecamp/example"`を含めるべきです。

```js
fetch(code + '/_api/package.json')
  .then(response => response.json())
  .then(
    data => {
      assert.property(
        data.dependencies,
        '@freecodecamp/example',
        '"dependencies" does not include "@freecodecamp/example"'
      );
    },
    error => {
      throw new Error(error.message || error.responseText);
    }
  );
```

`"@freecodecamp/example"`のバージョンは`"^1.x.x"`と一致するべきです。

```js
fetch(code + '/_api/package.json')
  .then(response => response.json())
  .then(
    data => {
      assert.match(
        data.dependencies['@freecodecamp/example'],
        /^\^1\./,
        'Wrong version of "@freecodecamp/example". It should be ^1.2.13'
      );
    },
    error => {
      throw new Error(error.message || error.responseText);
    }
  );
```
