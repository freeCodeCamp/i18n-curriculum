# --description--

<dfn>Рядкові</dfn> значення в JavaScript можуть бути записані з одинарними або подвійними лапками, за умови, що ви починаєте і закінчуєте рядок однаковим типом лапок. На відміну від деяких інших мов програмування, одинарні та подвійні лапки працюють однаково в JavaScript.

```js
const doubleQuoteStr = "This is a string"; 
const singleQuoteStr = 'This is also a string';
```

Причина, чому ви можете захотіти використати один тип лапок замість іншого, полягає в тому, якщо ви хочете використати обидва в одному рядку. Це може статися, якщо ви хочете зберегти розмову в рядку і мати розмову в лапках. Інше застосування — збереження `<a>` тегу з різними атрибутами в лапках, все це всередині рядка.

```js
const conversation = 'Finn exclaims to Jake, "Algebraic!"';
```

Однак це стає проблемою, якщо вам потрібно використати зовнішні лапки всередині рядка. Пам’ятайте, рядок має однаковий тип лапок на початку і в кінці. Але якщо у вас є такі ж лапки десь посередині, рядок закінчиться раніше і виникне помилка.

```js
const goodStr = 'Jake asks Finn, "Hey, let\'s go on an adventure?"'; 
const badStr = 'Finn responds, "Let's go!"';
```

Тут `badStr` викличе помилку.

У <dfn>goodStr</dfn> вище ви можете безпечно використати обидва типи лапок, використовуючи зворотну скісну риску `\` як символ екранування.

**Примітка:** Зворотна скісна риска `\` не повинна плутатися з прямою скісною рискою `/`. Вони виконують різні функції.

# --instructions--

Змініть наданий рядок на рядок з одинарними лапками на початку і в кінці без символів екранування.

Зараз `<a>` тег у рядку використовує подвійні лапки скрізь. Вам потрібно змінити зовнішні лапки на одинарні, щоб ви могли видалити символи екранування.

# --hints--

Ви повинні видалити всі зворотні скісні риски (`\`).

```js
assert(
  !/\\/g.test(__helpers.removeJSComments(code)) &&
    myStr.match(
      '\\s*<a href\\s*=\\s*"http://www.example.com"\\s*target\\s*=\\s*"_blank">\\s*Link\\s*</a>\\s*'
    )
);
```

У вас має бути дві одинарні лапки `'` і чотири подвійні лапки `"`.

```js
assert(__helpers.removeJSComments(code).match(/"/g).length === 4 && __helpers.removeJSComments(code).match(/'/g).length === 2);
```
