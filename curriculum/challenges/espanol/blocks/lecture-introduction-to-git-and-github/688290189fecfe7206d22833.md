# --description--

En una lección anterior, mencionamos brevemente GitHub. Tanto GitHub como Gitlab son proveedores de control de versiones. Esto significa que son soluciones basadas en la nube que ofrecen almacenamiento de proyectos con control de versiones en algo llamado "repositorios", y permiten funciones de colaboración para usar con esos proyectos.

Antes de profundizar en las características que ofrecen estas plataformas, necesitamos hablar sobre los diferentes tipos de proyectos. Muchos proyectos se consideran "open-source", lo que significa que las personas pueden ver el código que publicas, proponer cambios, reportar problemas e incluso ejecutar una versión modificada.

Pero a veces puede que solo quieras que tus compañeros vean y trabajen en el código. También puedes publicar un proyecto como "closed-source", lo que significa que las únicas personas que pueden ver e interactuar con el proyecto son las que autorizas explícitamente.

Independientemente del enfoque que tome tu proyecto, sitios como GitHub y Gitlab pueden mejorar enormemente la colaboración de tu equipo.

Por ejemplo, un compañero trabajando en una característica podría estar atascado y tener preguntas. En lugar de tener que copiar y pegar el código en Discord para que lo leas, o hacer una llamada para compartir pantalla con su trabajo, puede subir los cambios directamente a GitHub. Desde ahí, puedes ver los cambios en tu navegador o incluso descargar los cambios para ejecutarlos y depurarlos localmente.

Cuando un compañero ha completado una función, puede proponer los cambios a través de estos sitios web. Luego tú puedes revisar los cambios, dejando comentarios en archivos o líneas específicas si tienes opiniones o preguntas. ¡Incluso puedes proponer ediciones sugeridas que ellos pueden aplicar a los cambios directamente en el navegador!

Si estás satisfecho con los cambios, puedes aceptarlos y fusionarlos en tu rama principal. Esto significa que todo el equipo podrá descargar los últimos cambios localmente para tener el código más actualizado cuando comiencen su próxima tarea.

Hablando de tareas, estos sitios también ofrecen potentes rastreadores de incidencias. Puedes usarlos para rastrear y asignar tickets a los miembros de tu equipo, aceptar reportes de errores del público e incluso planificar sprints completos mediante elementos como los tableros de proyectos.

Hay muchas más características que ofrecen estos sitios. En las próximas lecciones, cubriremos los aspectos clave para que puedas comenzar de inmediato y empezar a colaborar.
