# --interactive--

У вебдизайні фони відіграють важливу роль у формуванні загального вигляду та відчуття сторінки.

Однак при створенні дизайну з фонами дуже важливо враховувати доступність, щоб ваш вміст був зручним і читабельним для всіх користувачів, включно з тими, хто має порушення зору.

Однією з основних проблем доступності, пов’язаних із фонами, є забезпечення достатнього коефіцієнта контрастності між фоном і текстом.

Без належного контрасту користувачам із порушеннями зору, зокрема з низьким зором або дальтонізмом, може бути важко прочитати вміст на вашій сторінці.

Контрастність означає різницю у світлості або темності між двома кольорами. Достатній коефіцієнт контрастності між кольором фону та кольором тексту є необхідним для читабельності.

Рекомендації Web Content Accessibility Guidelines (WCAG) радять мінімальний коефіцієнт контрастності 4.5:1 для звичайного тексту і 3:1 для великого тексту.

Наприклад, розміщення білого тексту на світло-сірому фоні призведе до поганого контрасту, через що текст буде важко читати. Натомість білий текст на темно-синьому фоні забезпечить хороший контраст, покращуючи читабельність для всіх користувачів.

Ось приклад поганого контрасту:

:::interactive_editor

```html
<p style="color: lightgray; background-color: whitesmoke;">
   This is an example of poor contrast.
</p>
```

:::

А тепер приклад хорошого контрасту:

:::interactive_editor

```html
<p style="color: white; background-color: darkslategray;">
  This is an example of good contrast.
</p>
```

:::

Ще одна важлива порада — уникати розміщення тексту на складних або насичених фонах, таких як зображення або градієнти з кількома кольорами. Насичені фони ускладнюють розрізнення тексту від фону, навіть якщо контрастність достатня.

При створенні фонів уникайте використання кольору як єдиного способу передачі інформації. Наприклад, використання лише кольору для позначення помилки або успіху (наприклад, червоний для помилки або зелений для успіху) може створити проблеми для користувачів із дальтонізмом.

Окрім кольору, слід використовувати символи або текст для передачі інформації. Наприклад, поряд із червоним повідомленням про помилку можна додати іконку або жирний текст, щоб чітко показати, що сталася помилка.

Хоча це трапляється рідше, фоновий звук або відео також можуть впливати на доступність.

Фонова музика або відео з автоматичним відтворенням можуть відволікати деяких користувачів, особливо тих, хто має когнітивні порушення. Якщо ви додаєте фоновий звук, завжди надавайте можливість вимкнути або призупинити його.

Враховуючи ці особливості доступності, ви зможете створювати більш інклюзивні дизайни, які гарантують, що ваш вміст буде читабельним і зручним для всіх користувачів, незалежно від їхніх можливостей.
