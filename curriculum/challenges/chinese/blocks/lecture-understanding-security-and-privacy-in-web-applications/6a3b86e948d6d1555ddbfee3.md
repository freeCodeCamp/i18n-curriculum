# --description--

同源策略是你网页浏览器内置的一项安全措施，它阻止一个网站读取另一个网站的私有数据。这条规则保护你的个人账户免受黑客攻击。

一个源由三个具体部分组成：网络协议、域名和端口号。

例如，在地址 `https://example.com:443` 中，协议是 HTTPS，域名是 `example.com`，端口号是 `443`。

如果端口号是标准端口，浏览器通常会隐藏它。HTTPS 默认端口是 443，HTTP 默认端口是 80。即使隐藏了，浏览器仍然会检查它。只有当这三个部分完全匹配时，两个网站才具有相同的源。

在这条严格规则下，浏览器会阻止一个网站窥探另一个网站。想象你在一个浏览器标签页打开了你的在线银行账户，在另一个标签页打开了一个恶意网站。同源策略会阻止恶意网站窃取你的银行信息。

然而，现代网站经常需要跨不同源共享数据。例如，前端网页可能需要从独立的后端服务器拉取数据。浏览器中的同源规则会阻止这些安全的请求，因此开发者需要一种安全共享数据的方法。这就是 CORS 的作用。

CORS 是一个系统，它允许服务器通过浏览器规则开放访问。它使用特殊的头部来授予受信任网站的数据访问权限。

当你的浏览器向另一个网站请求数据时，发生以下情况：

* 它包含一个 `Origin` 头部，标明发起请求的网站
* 接收服务器随后检查这个名称
* 如果服务器信任该网站，它会回复一个特殊的权限头部，告诉浏览器该网站被允许读取传入的数据。

如果服务器没有发送这个权限，浏览器会立即阻止数据。前端网站随后会收到错误信息，而不是私有文件。

在使用 Express 的 Node.js 应用中，CORS 是通过 `cors` 包实现的：

```javascript
const express = require('express');
const cors = require('cors');
const app = express();

// this enables CORS for all origins and all routes for development
app.use(cors());
```

在生产环境中，你可能需要传入一个配置对象来限制对特定域的访问：

```javascript
const corsOptions = {
  origin: ['https://yourapp.com', 'https://admin.yourapp.com'],
  credentials: true
};

app.use(cors(corsOptions));
```

如果你使用的是原生 Node.js，你需要手动处理，在每个请求上设置 CORS 响应头：

```javascript
res.setHeader('Access-Control-Allow-Origin', 'https://example.com');
res.setHeader('Access-Control-Allow-Methods', 'GET, POST, PUT, DELETE, OPTIONS');
res.setHeader('Access-Control-Allow-Headers', 'Content-Type, Authorization');
```
