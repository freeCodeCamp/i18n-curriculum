# --description--

في درس سابق، تعلمت أن الخصائص هي المتغيرات التي تنتمي إلى كائن. هذا يعني أنها تحمل بيانات تصف حالة أو سلوك الكائن.

على سبيل المثال، عادةً ما يكون للسيارة علامة تجارية وطراز. يمكن أن تكون العلامة والطراز خصائص لصنف `Car`:

```python
class Car: 
    def __init__(self, brand, model): 
        self.brand = brand 
        self.model = model 

my_car = Car('Lamborghini', 'Gallardo') 
print(my_car.brand) # Lamborghini 
print(my_car.model) # Gallardo 
```

لكن أحيانًا، قد لا تعرف الخصائص التي تحتاجها حتى يعمل برنامجك. تخيل أنك تكتب سكربت يستقبل أسماء الخصائص من المستخدم أو من ملف إعدادات. هذه ليست خصائص يمكنك ترميزها مسبقًا.

هنا يأتي دور التعامل مع الخصائص بشكل ديناميكي. بهذه الطريقة، يمكنك الوصول إلى الخصائص أو تعديلها أو التحقق منها أو حتى حذفها باستخدام أسمائها كمتغيرات، وليس كأسماء ثابتة في شفرتك. هذا يمنح برنامجك المرونة للاستجابة لبيانات أو مدخلات المستخدم المختلفة أثناء التشغيل.

يوفر لك بايثون أربع دوال مدمجة مفيدة للعمل ديناميكيًا مع خصائص الكائنات. وهي `getattr()`, `setattr()`, `hasattr()`, و `delattr()`.

تتيح لك هذه الدوال الوصول إلى الخصائص، إنشاؤها، التحقق منها، وحذفها باستخدام أسماء متغيرة. دعنا نلقي نظرة على كل واحدة منها أثناء العمل.

تجعل `getattr()` من الممكن قراءة خاصية من كائن عندما لا تعرف اسمها حتى وقت التشغيل. إذا لم تكن الخاصية موجودة، ترفع استثناء `AttributeError`، إلا إذا وفرت قيمة افتراضية.

لاستخدامها، تمرر الكائن، اسم الخاصية، وقيمة افتراضية اختيارية:

```python
getattr(object, attribute_name, default_value) 
```

إليك مثالاً:

```python
class Person: 
    def __init__(self, name, age): 
        self.name = name 
        self.age = age 

person = Person('John Doe', 30) 
 
print(getattr(person, 'name')) # John Doe 
print(getattr(person, 'age')) # 30 
print(getattr(person, 'city', 'Milano')) # Milano
```

في المثال أعلاه، `Milano` هي القيمة الافتراضية لأن `city` غير موجودة في صنف `Person`.

كما قلنا سابقًا، القوة الحقيقية لـ `getattr()` تظهر عندما يأتي اسم الخاصية من متغير، مثل مدخلات المستخدم أو ملف ما.

في هذه الحالة، لا يمكنك استخدام الصيغة العادية `object.attribute_name` لأن اسم الخاصية غير ثابت.

```python
class Person: 
    def __init__(self, name, age): 
        self.name = name 
        self.age = age 

person = Person('John Doe', 30)

attr_name = input('Enter the attribute you want to see: ')
print(getattr(person, attr_name, 'Attribute not found'))
```

في هذا المثال، إذا كتب المستخدم `name`، سيرى `John Doe`، وإذا كتب `age`، سيرى `30`. وإذا كتب شيئًا غير موجود في الصنف مثل `email`، سيرى `Attribute not found`.

هنا بالضبط يبرز التعامل الديناميكي مع الخصائص. يسمح لشفرتك بالاستجابة لمدخلات وبيانات لم ترها من قبل.

بالإضافة إلى ذلك، قد ترغب في استعراض كل الخصائص التي يمتلكها كائن، وليس فقط التي تعرفها مسبقًا. الدالة المدمجة `dir()` تتيح لك ذلك. تعيد قائمة بكل أسماء الخصائص على الكائن. إليك كيفية استخدامها:

```python
class Person: 
    def __init__(self, name, age): 
        self.name = name 
        self.age = age 

person = Person('John Doe', 30)

# Loop through all attributes of the person object with dir() function
for attr in dir(person):
    # Ignore dunder methods like __init__ or __str__ and regular methods
    if not attr.startswith('__') and not callable(getattr(person, attr)): 
        value = getattr(person, attr)
        print(f'{attr}: {value}')

# Output
# age: 30
# name: John Doe
```

في الحلقة أعلاه، `callable()` هي دالة مدمجة تعيد `True` إذا كان الكائن الممرر إليها يمكن استدعاؤه كدالة أو طريقة، و `False` خلاف ذلك. بفحص `not callable(getattr(person, attr))`، تتخطى الحلقة الطرق وتطبع فقط خصائص البيانات مثل `name` و `age`.

تتيح لك دالة `setattr()` إنشاء خاصية جديدة أو تحديث خاصية موجودة بشكل ديناميكي. الصيغة تبدو هكذا:

```python
setattr(object, attribute_name, value) 
```

إليك مثالًا يضبط خصائص الإعدادات بناءً على بيانات من ملف إعدادات أو متغير بيئة:

```python
class Configuration:
    pass

# Data loaded at runtime (like from a config or env file)
settings_data = {
    'server_url': 'https://api.example.com',
    'timeout_sec': 30,
    'max_retries': 5
}

config_obj = Configuration()

# Dynamically set attributes using dictionary keys and values
for attr_name, attr_value in settings_data.items():
    setattr(config_obj, attr_name, attr_value)

print(config_obj.server_url) # https://api.example.com
print(config_obj.timeout_sec) # 30
```

هناك أيضًا `hasattr()`. قبل أن تفعل شيئًا مع خاصية أو تحذفها، من الممارسات الجيدة التحقق من وجودها. هذا ما تفعله `hasattr()`. تتحقق من وجود الخاصية وتعيد `True` أو `False` بناءً على النتيجة.

إليك الصيغة الأساسية:

```python
hasattr(object, attribute_name)  
```

وهنا مثال يتحقق ديناميكيًا من وجود خصائص في نسخة من صنف `Product`:

```python
class Product:
    def __init__(self, name, price):
        self.name = name
        self.price = price

product_a = Product('T-Shirt', 25)

required_attributes = ['name', 'price', 'inventory_id']

for attr in required_attributes:
    if not hasattr(product_a, attr):
        print(f"ERROR: Product is missing the required attribute: '{attr}'")
    else:
        # Access the attributes dynamically once their existence is confirmed
        print(f'{attr}: {getattr(product_a, attr)}')

# Output:
# name: T-Shirt
# price: 25
# ERROR: Product is missing the required attribute: 'inventory_id'
```

حدث خطأ ERROR لأن `inventory_id` مفقودة من صنف `Product` ومن نسخته.

أخيرًا، تتيح لك `delattr()` حذف خاصية بشكل ديناميكي:

```python
delattr(object, attribute_name) 
```

على سبيل المثال، تخيل أن كائنًا قد تم معالجته بالكامل، ثم قررت تنظيف أي خصائص حساسة أو مؤقتة قد تكون موجودة قبل حفظ النسخة النهائية. بعد ذلك، يمكنك استخدام `dir()` للتكرار على الخصائص المتبقية:

```python
class UserSession:
    def __init__(self, user_id, token):
        self.user_id = user_id
        self.auth_token = token # sensitive
        self.temp_counter = 0 # temporary

session = UserSession(101, 'a1b2c3d4e5')

# List of attributes to remove dynamically before "saving" the session
attributes_to_clean = ['auth_token', 'temp_counter']

# Dynamically remove specified attributes
for attr in attributes_to_clean:
    if hasattr(session, attr):
        delattr(session, attr)
        print(f'Removed attribute: {attr}')

print('\nFinal attributes remaining:')

# Loop through the remaining attributes with dir()
for attr in dir(session):
    # Ignore dunder methods like __init__ or __str__ and regular methods
    if not attr.startswith('__') and not callable(getattr(session, attr)):
        print(f' - {attr}: {getattr(session, attr)}')

# Output:
# Removed attribute: auth_token
# Removed attribute: temp_counter

# Final attributes remaining:
#  - user_id: 101
```

وهكذا يمكنك التعامل مع الخصائص بشكل ديناميكي!
