# --description--

У попередньому уроці ви створили папку проєкту та встановили Express:

```bash
mkdir my-app
cd my-app
npm init -y
npm install express
```

Нагадаємо, ці команди створюють нову папку проєкту, `cd` переходять у папку проєкту, ініціалізують проєкт і створюють базовий файл `package.json`, а також встановлюють Express.

Далі відкрийте редактор коду, створіть файл з іменем `index.js` і напишіть свій перший сервер Express:

```javascript
const express = require("express");
const app = express();
const port = 3000;

app.get("/", (req, res) => {
  res.send("Hello World!");
});

app.listen(port, () => {
  console.log(`Example app listening on port ${port}`);
});
```

Ви можете запустити свій код за допомогою такої команди в терміналі:

```bash
node index.js
```

Відкрийте браузер за адресою [http://localhost:3000](http://localhost:3000) і ви побачите повідомлення **"Hello World!"**

Ось як швидко можна почати працювати з Express! Тепер пройдемося по коду крок за кроком.

У першому рядку коду ви завантажуєте бібліотеку Express і призначаєте її змінній з назвою `express`:

```javascript
const express = require("express");
```

Змінна `express` — це функція-фабрика. Коли ви її викликаєте, вона створює і повертає новий екземпляр застосунку Express:

```javascript
const app = express();
```

`app` тепер має доступ до всіх методів, які надає Express, таких як `.get()`, `.post()`, `.listen()` тощо. Це ваш основний серверний об’єкт.

Наступний рядок визначає змінну `port`. Порт — це мережна кінцева точка, на якій ваш застосунок слухає запити, а `3000` — це загальновживаний порт для локальної розробки:

```javascript
const port = 3000;
```

Тепер подивімося на наступну частину:

```javascript
app.get("/", (req, res) => {
  res.send("Hello World!");
});
```

Цей блок коду визначає маршрут і вказує серверу, як відповідати на запити. `app.get()` реєструє обробник для HTTP GET-запитів. GET-запит — це загальновживаний HTTP-метод для отримання даних.

Аргумент `"/"` всередині `app.get()` представляє шлях маршруту. У цьому випадку він вказує на кореневий URL застосунку.

Частина `(req, res) => { ... }` — це функція зворотного виклику, яка виконується, коли запит відповідає цьому маршруту. Об’єкт `req` містить інформацію про вхідний запит, а об’єкт `res` використовується для відправлення відповіді клієнту.

Нарешті, `res.send("Hello World!")` відправляє відповідь клієнту. У цьому прикладі відповідь — це простий текст `"Hello World!"`.

Тепер подивімося на останню частину коду:

```javascript
app.listen(port, () => {
  console.log(`Example app listening on port ${port}`);
});
```

Метод `app.listen()` запускає застосунок Express і вказує Node слухати вхідні HTTP-запити на вказаному порті, у цьому випадку `3000`.

Коли сервер запускається, виконується функція зворотного виклику, і в консолі з’являється повідомлення `Example app listening on port 3000`, що підтверджує успішний запуск застосунку Express і готовність приймати запити.
