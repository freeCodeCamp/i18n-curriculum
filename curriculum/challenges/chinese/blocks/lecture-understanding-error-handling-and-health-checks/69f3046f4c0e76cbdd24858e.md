# --description--

你有没有想过，当你的 Express 应用出现问题时会发生什么？

如果数据库连接失败怎么办？或者用户访问了一个不存在的路由呢？

理解 Express 中错误处理的工作原理对于构建安全、可靠的网页应用至关重要。

在本课中，我们将讲解 Express 如何处理错误，以及如何使用真实的代码示例捕获和响应这些错误。

## 如何在 Express 中处理简单错误

在 Express 中，错误处理是基于中间件的。

就像普通的中间件函数一样，错误处理的中间件可以访问 `req`、`res` 和 `next`。但它在开头多了一个参数：`err`。

语法如下所示：

```js
app.use((err, req, res, next) => {
  console.error(err.message);
  res.status(500).send("Internal Server Error");
});
```

你需要在中间件栈的末尾定义这个中间件。

Express 之所以知道它是错误处理器，是因为它有四个参数——除了典型的 `err`、`req` 和 `res` 参数外，还有新的 `next` 参数。

在后续课程中，你将学习更多关于错误处理中间件以及它如何融入更广泛的 Express 中间件栈。

## 如何处理异步错误

处理异步错误取决于你使用的 Express 版本。

请注意，无论你使用哪个版本的 Express，都需要在所有路由之后，在栈的末尾定义你的错误处理中间件。

错误处理器如下：

```js
// App routes
// ...

// Error-handling middleware
app.use((err, req, res, next) => {
  console.error(err.message);
  res.status(500).send("Internal Server Error");
});
```

## Express 4

在 Express 4 及以下版本（许多项目仍在使用）中，异步错误**不会被自动捕获**。

如果你使用 `async`/`await`，并且在 `async` 路由处理器中抛出错误，Express 不会捕获它，除非你手动将其传递给 `next()`。

在 Express 4 中正确处理异步错误的方法如下：

```js
app.get("/user", async (req, res, next) => {
  try {
    const user = await getUserFromDatabase();
    res.send(user);
  } catch (err) {
    next(err); // Manually pass the error to the error handler
  }
});
```

如果你忘记了 `try/catch` 块和 `next(err)`，错误可能会：

* 导致你的应用崩溃
* 或产生未处理的 promise 拒绝警告

这就是为什么在 Express 4 中将异步逻辑包裹在 `try`/`catch` 中非常重要。

有些开发者甚至创建了辅助函数来避免到处重复 `try/catch`：

```js
function asyncHandler(fn) {
  return function (req, res, next) {
    Promise.resolve(fn(req, res, next)).catch(next);
  };
}
```

然后这样使用它：

```js
app.get("/user", asyncHandler(async (req, res) => {
  const user = await getUserFromDatabase();
  res.send(user);
}));
```

这样你就不需要在所有 `try/catch` 路由中使用 `async` 块，同时还能正确转发错误。

## Express 5

当前版本的 Express 改进了这一行为。

从 Express 5 开始，如果 `async` 路由抛出错误或返回拒绝的 promise，Express 会自动将其转发到错误处理中间件。

这意味着下面的示例可以在没有 `try/catch` 的情况下处理错误：

```js
app.get("/user", async (req, res) => {
  const user = await getUserFromDatabase();
  res.send(user);
});
```

如果 `getUserFromDatabase()` 抛出错误，Express 5 会自动捕获并传递给你的错误处理器。

无需手动调用 `next(err)`。

## 如何处理 404 错误

404 错误有点不同。它们发生在没有路由匹配请求时。

你可以用这个中间件来处理它们：

```js
app.use((req, res, next) => {
  res.status(404).send("Sorry, that route does not exist.");
});
```

**重要**：这应该放在你的路由之后，但在错误处理中间件之前。

最后，这里有一些 Express 中错误处理的最佳实践：

* 始终将错误处理中间件放在最后。
* 单独处理 404 错误。
* 在 Express 4 及以下版本中，使用 `next(err)` 从路由或 `async` 逻辑中冒泡错误。

在生产环境中，不要向用户发送堆栈追踪。应在内部记录详细信息，并返回用户友好的消息。

Express 中的错误处理就是设置监听问题并以清晰、安全的方式响应的中间件。无论是代码中的错误还是缺失的路由，你的应用都不应该在没有备份方案的情况下崩溃。
