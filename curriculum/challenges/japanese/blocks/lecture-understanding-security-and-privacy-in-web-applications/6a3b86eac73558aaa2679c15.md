# --description--

クッキーは、ウェブサイトがあなたのデバイスに保存する小さなテキストファイルで、あなたに関する情報を記憶します。

クッキーを受け取り保存する流れは次の通りです：

- ウェブサイトを訪問する
- ウェブサイトのサーバーがレスポンスに `Set-Cookie` ヘッダーを送る
- ブラウザがそのヘッダーを受け取り、クッキーをコンピューターのハードドライブに保存する
- そのウェブサイトを訪問するたびに、ブラウザが自動的にクッキーをサーバーに送り返す

この仕組みにより、ウェブサイトはログインセッション、ショッピングカートの中身、設定や好みなど重要な情報を記憶できます。

この保存方法には重大なセキュリティリスクがあります。ハッカーが物理的にデバイスにアクセスしたり、マルウェアでローカルファイルを読み取ったりすると、セッション用クッキーを盗まれる可能性があります。

盗まれたセッション用クッキーがあれば、攻撃者はパスワードを知らなくても全く別のコンピューターからあなたのアカウントにログインできます。

ユーザーを守りクッキーの盗難を防ぐために、開発者はクッキー設定時に特定のセキュリティフラグを使います：

`HttpOnly` フラグはJavaScriptによるクッキーの読み取りをブロックします。この制限により、クロスサイトスクリプティング攻撃でセッショントークンを盗まれるのを防ぎます。

`Secure` フラグはブラウザがクッキーを暗号化されたHTTPS接続でのみ送信するようにします。これにより公共Wi-Fiなどでのクッキー盗聴を防げます。

`SameSite` フラグはクロスサイトリクエスト時のクッキー送信を制限し、クロスサイトリクエストフォージェリ攻撃を防ぎます。主に2つの値があります：

- `SameSite=Strict` はどんなクロスサイトリクエストでもクッキー送信を防ぎます。
- `SameSite=Lax` はリンククリックなど安全なトップレベルのナビゲーションではクッキーを許可しますが、クロスサイトのフォーム送信やAPIリクエストではブロックします。

クッキーはプライバシーにも大きな影響があります。ファーストパーティクッキーは訪問した特定のウェブサイト上の行動だけを追跡しますが、サードパーティクッキーは全く異なる複数のウェブサイトにわたって追跡します。

広告ネットワークはこれらのクッキーを何千もの人気サイトに設置します。ネットを閲覧するたびに、広告ネットワークは同じサードパーティクッキーを繰り返し読み取り、あなたの興味、検索履歴、オンライン習慣の詳細なプロファイルを作成します。

この見えない追跡は、企業があなたのデジタルライフをあなたの積極的な認識なしに監視できるため、重大なプライバシー問題を引き起こします。

こうした脅威により、GDPRのようなプライバシー法はウェブサイトにクッキー同意バナーの表示を義務付け、クッキーの利用内容をユーザーに知らせます。これらの法律は追跡クッキーを拒否する法的権利を与えます。

さらに、最新のブラウザはデフォルトでサードパーティの追跡クッキーをブロックし、ユーザーに自動的なプライバシー保護を提供しています。
