# --interactive--

Елемент `form` в HTML використовується для збору інформації від користувача, такої як імена та адреси е-пошти. Ось приклад елемента `form`:

```html
<form action="url-goes-here">
  <!-- input elements go here -->
</form>
```

Атрибут `action` визначає, куди будуть надіслані дані форми після відправлення. Щоб збирати конкретну інформацію, наприклад імена та адреси е-пошти, ви використовуєте елемент `input`. Ось приклад використання елемента `input`.

Увімкніть інтерактивний редактор і взаємодійте з елементом `input` у вікні попереднього перегляду, ввівши своє ім’я у поле.

:::interactive_editor

```html
<form action="">
  <input type="text" />
</form>
```

:::

Елементи `input` є порожніми елементами і не мають кінцевих тегів. Атрибут `type` визначає тип даних, який очікується від користувача. У цьому випадку дані будуть у вигляді простого тексту. Щоб додати мітку для поля введення, ви використовуєте елемент `label`. Ось приклад використання елемента `label` з текстом `Full Name:`.

Натисніть на текст `Full Name:` у вікні попереднього перегляду, і ви побачите, що поле введення отримає фокус. Щоб взаємодіяти з прикладом, потрібно увімкнути інтерактивний редактор.

:::interactive_editor

```html
<form action="">
  <label>
    Full Name:
    <input type="text" />
  </label>
</form>
```

:::

Вклавши `input` всередину елемента `label`, ви створюєте неявний зв’язок між `label` і полем `input`. Термін «неявний» означає те, що розуміється або виводиться без необхідності явно вказувати або визначати додатковими атрибутами чи елементами. Щоб явно пов’язати `label` з `input`, можна використати атрибут `for`. Ось приклад використання атрибута `for` для мітки адреси е-пошти.

Взаємодійте з елементом `input` у вікні попереднього перегляду, ввівши фальшиву адресу е-пошти, наприклад `jane@example.com`. Щоб взаємодіяти з прикладом, потрібно увімкнути інтерактивний редактор.

:::interactive_editor

```html
<form action="">
  <label for="email"> Email Address: </label>
  <input type="email" id="email" />
</form>
```

:::

При використанні явного зв’язку значення атрибута `for` і `id` мають бути однаковими. У цьому випадку обидва значення встановлені як `email`. Тип `email` у полі введення забезпечує базову валідацію для правильно відформатованих адрес е-пошти. Якщо ви хочете показати користувачам додаткові підказки щодо очікуваного введення, можна використати атрибут `placeholder`. Ось приклад використання атрибута `placeholder` всередині поля введення е-пошти.

Увімкніть інтерактивний редактор, натисніть на поле введення е-пошти і почніть вводити адресу, і ви побачите, що текст заповнювача зникне.

:::interactive_editor

```html
<form action="">
  <label for="email"> Email Address: </label>
  <input type="email" id="email" placeholder="example@email.com" />
</form>
```

:::

Для тексту заповнювача бажано надати короткий корисний текст, який показує формат і тип даних, які ви очікуєте від користувачів. У цьому випадку текст заповнювача `example@email.com` показує користувачу, що потрібно ввести правильно відформатовану адресу е-пошти.
