# --interactive--

프로그래밍에서 배열은 요소 모음을 저장하는 기본적인 자료 구조입니다. 1차원 배열과 2차원 배열의 차이를 이해하는 것은 데이터를 효과적으로 정리하고 조작하는 데 매우 중요합니다. 초보자도 쉽게 이해할 수 있도록 이 개념들을 살펴보겠습니다.

1차원 배열, 흔히 배열이라고 부르는 것은 한 줄로 된 상자들의 행과 같습니다. 기차역에 일렬로 늘어선 사물함을 상상해 보세요. 각 사물함은 하나의 물건을 담을 수 있고, 번호만 알면 어느 사물함이든 직접 접근할 수 있습니다.

프로그래밍 용어로 1차원 배열의 각 항목은 하나의 인덱스를 사용해 접근합니다. 예를 들어 JavaScript에서는 다음과 같이 1차원 배열을 만들고 사용할 수 있습니다:

:::interactive_editor

```js
let fruits = ["apple", "banana", "cherry", "date"];
console.log(fruits[2]); // "cherry"
```

:::

여기서 `fruits`는 1차원 배열입니다. 과일 이름이 한 줄로 나열된 것처럼 생각할 수 있습니다. 과일 하나에 접근하려면 대괄호 안에 하나의 숫자(인덱스)를 사용합니다.

이제 2차원 배열로 넘어가 보겠습니다. 1차원 배열이 한 줄의 사물함이라면, 2차원 배열은 여러 행과 열로 이루어진 사물함 격자와 같습니다. 프로그래밍에서 2차원 배열은 본질적으로 배열들의 배열입니다. 체스판, 스프레드시트, 이미지의 픽셀처럼 자연스러운 격자 구조를 나타낼 때 사용합니다.

2차원 배열의 요소에 접근하려면 두 개의 인덱스가 필요합니다: 하나는 행, 다른 하나는 열입니다. JavaScript에서 2차원 배열을 만들고 사용하는 예는 다음과 같습니다:

:::interactive_editor

```js
let chessboard = [
    ["R", "N", "B", "Q", "K", "B", "N", "R"],
    ["P", "P", "P", "P", "P", "P", "P", "P"],
    [" ", " ", " ", " ", " ", " ", " ", " "],
    [" ", " ", " ", " ", " ", " ", " ", " "],
    [" ", " ", " ", " ", " ", " ", " ", " "],
    [" ", " ", " ", " ", " ", " ", " ", " "],
    ["p", "p", "p", "p", "p", "p", "p", "p"],
    ["r", "n", "b", "q", "k", "b", "n", "r"]
];

console.log(chessboard[0][3]); // "Q"
```

:::

이 예에서 `chessboard`는 체스 게임의 초기 배치를 나타내는 2차원 배열입니다. 맨 위 행의 여왕(`Q`)에 접근하려면 두 개의 인덱스 `[0][3]`를 사용합니다. 첫 번째 인덱스 `0`는 첫 번째 행을 선택하고, 두 번째 인덱스 `3`는 그 행의 네 번째 열을 선택합니다.

1차원 배열과 2차원 배열의 주요 차이는 데이터를 접근하고 정리하는 방식에 있습니다. 1차원 배열은 하나의 인덱스를 사용하며, 리스트나 수열 같은 선형 데이터에 적합합니다. 2차원 배열은 두 개의 인덱스를 사용하며, 격자 형태의 자료 구조에 이상적입니다.

JavaScript에서 2차원 배열은 실제로 배열들의 배열이라는 점도 주목할 만합니다. 즉, 외부 배열의 각 요소가 또 다른 배열입니다. 이런 중첩 구조는 매우 유연하지만 오류를 피하려면 주의 깊게 다뤄야 합니다.

프로그래밍을 배우면서 1차원 배열과 2차원 배열 중 어떤 것을 선택할지는 데이터의 성격과 조작 방식에 따라 달라집니다.

1차원 배열은 더 간단하고 많은 작업에 충분하지만, 2차원 배열은 더 복잡하고 구조화된 데이터를 다룰 때 매우 유용합니다.
