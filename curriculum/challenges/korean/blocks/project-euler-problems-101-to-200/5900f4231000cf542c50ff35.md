# --description--

RSA 암호화는 다음 절차를 기반으로 합니다:

서로 다른 두 소수 `p`와 `q`를 생성하세요.
`n=p*q`와 `φ=(p-1)(q-1)`를 계산하세요.
정수 `e`, `1 < e < φ`를 찾아 `gcd(e,φ) = 1`가 되도록 하세요.

이 시스템에서 메시지는 구간 `[0,n-1]`에 속하는 숫자입니다.
암호화할 텍스트는 어떤 방식으로든 메시지(구간 `[0,n-1]`에 속하는 숫자)로 변환됩니다.
텍스트를 암호화하려면 각 메시지에 대해 `m`, c=m<sup>e</sup> mod n를 계산하세요.

텍스트를 복호화하려면 다음 절차가 필요합니다: `d`를 계산하여 `ed=1 mod φ`가 되도록 한 다음, 각 암호화된 메시지에 대해 `c`, m=c<sup>d</sup> mod n를 계산하세요.

`e`와 `m` 값이 존재하여 m<sup>e</sup> mod n = m가 성립합니다.
m<sup>e</sup> mod n=m인 메시지 `m`를 무차폐 메시지라고 부릅니다.

`e`를 선택할 때 문제는 무차폐 메시지가 너무 많아서는 안 된다는 점입니다.
예를 들어, `p=19`와 `q=37`를 생각해 보세요.
그런 다음 `n=19*37=703`와 `φ=18*36=648`가 있습니다.
`e=181`를 선택하면, `gcd(181,648)=1`에도 불구하고 모든 가능한 메시지
m `(0≤m≤n-1)`가 m<sup>e</sup> mod n를 계산할 때 무차폐가 됨을 알 수 있습니다.
유효한 `e` 선택에 대해서는 일부 무차폐 메시지가 항상 존재합니다.
무차폐 메시지의 수가 최솟값이 되는 것이 중요합니다.

주어진 `p`와 `q`에 대해, 무차폐 메시지 수가 최솟값이 되는 `e` 값에 대해 모든 `1 < e < φ(p,q)`, `gcd(e,φ)=1`, `e` 값을 합산하세요.

# --solutions--

```js
function gcd(a, b) {
    if (b)
        return gcd(b, a % b);
    else
        return a;
}

function RSAEncryption(p, q) {
    let phi = (p - 1) * (q - 1);

    let best = Number.MAX_SAFE_INTEGER;
    let sum = 0;

    for (let e = 0; e < phi; ++e) {
        if (!(gcd(e, phi) == 1))
            continue;

        let msg = (gcd(p - 1, e - 1) + 1) * (gcd(q - 1, e - 1) + 1);

        if (best == msg) {
            sum += e;
        } else if (best > msg) {
            best = msg;
            sum = e;
        }
    }

    return sum;
}
```
