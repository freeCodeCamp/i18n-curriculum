# --interactive--

Die `@property`-Regel ist eine leistungsstarke CSS-Funktion, die Entwickler:innen ermöglicht, benutzerdefinierte Eigenschaften mit größerer Kontrolle über ihr Verhalten zu definieren, einschließlich ihrer Animationen und Anfangswerte.

Diese Regel bietet eine Möglichkeit, die Funktionalität von CSS-Custom-Properties zu erweitern und mehr Flexibilität bei ihrer Anwendung zu bieten.

Die grundlegende Syntax der `@property`-Regel sieht wie folgt aus:

```css
@property --property-name {
  syntax: '<type>';
  inherits: true | false;
  initial-value: <value>;
}
```

`--property-name` ist der Name der benutzerdefinierten Eigenschaft, die Sie definieren. Wie alle benutzerdefinierten Eigenschaften muss sie mit zwei Bindestrichen beginnen.

`syntax` definiert den Typ der Eigenschaft, der Dinge wie `<color>`, `<length>`, `<number>`, `<percentage>` oder komplexere Typen sein kann.

`inherits` gibt an, ob die Eigenschaft ihren Wert vom Elternelement erben soll.

`initial-value` legt den Standardwert der Eigenschaft fest.

Hier ist ein praktisches Beispiel für die Verwendung der `@property`-Regel:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<button class="button">Click Me</button>
```

```css
@property --main-color {
  syntax: '<color>';
  inherits: false;
  initial-value: #3498db;
}

.button {
  background-color: var(--main-color);
}
```

:::

In diesem Beispiel definieren wir eine benutzerdefinierte Eigenschaft `--main-color` als `color`-Wert, setzen sie auf kein Erben und geben ihr einen Anfangswert von `#3498db`.

Einer der wichtigsten Vorteile der `@property`-Regel ist, dass sie die Animation von benutzerdefinierten Eigenschaften ermöglicht, was zuvor nicht möglich war:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<div class="gradient-box"></div>
```

```css
@property --gradient-angle {
  syntax: '<angle>';
  inherits: false;
  initial-value: 0deg;
}

.gradient-box {
  width: 100px;
  height: 100px;
  background: linear-gradient(var(--gradient-angle), red, blue);
  transition: --gradient-angle 0.5s;
}

.gradient-box:hover {
  --gradient-angle: 90deg;
}
```

:::

Der obige Code erzeugt einen Farbverlauf, der beim Überfahren des Elements sanft animiert wird – etwas, das mit Standard-Custom-Properties nicht erreichbar war.

Kommen wir nun dazu, wie die `@property`-Regel mit Fallbacks funktioniert.

Fallbacks sind in CSS entscheidend, um sicherzustellen, dass Styles in Browsern, die bestimmte Funktionen nicht unterstützen, elegant abfallen. Mit der `@property` funktionieren Fallbacks auf zwei Ebenen: für die Regel selbst und für die Verwendung der benutzerdefinierten Eigenschaft.

Für Browser, die die `@property`-Regel nicht unterstützen, können Sie einen Fallback bereitstellen, indem Sie die benutzerdefinierte Eigenschaft auf herkömmliche Weise deklarieren:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
```

```css
:root {
  --main-color: #3498db;
}

@property --main-color {
  syntax: '<color>';
  inherits: false;
  initial-value: #3498db;
}

body {
  background-color: var(--main-color);
}
```

:::

In diesem Fall verwenden Browser, die `@property` unterstützen, die strengere Version, während andere auf die Standarddeklaration der benutzerdefinierten Eigenschaft zurückgreifen.

Beim Verwenden der benutzerdefinierten Eigenschaft können Sie einen Fallback-Wert mit der `var()`-Funktion angeben, genau wie bei Standard-Custom-Properties:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<button class="button">Click Me</button>
```

```css
.button {
  background-color: var(--main-color, #3498db);
}
```

:::

So wird sichergestellt, dass der Button auch dann eine Hintergrundfarbe hat, wenn die `--main-color`-Eigenschaft nicht definiert oder ungültig ist.

Die `@property`-Regel ermöglicht auch komplexere Fallback-Szenarien. Zum Beispiel können Sie sie verwenden, um typsichere Fallbacks bereitzustellen:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<div class="box">Colored box</div>
```

```css
@property --padding {
  syntax: '<length>';
  inherits: false;
  initial-value: 10px;
}

.box {
  width: 100px;
  height: 100px;
  background-color: darkred;
  color: white;
  padding: var(--padding);
}
```

:::

In diesem Fall, wenn jemand versucht, `--padding` auf einen ungültigen Wert wie eine Farbe zu setzen, fällt der Browser auf den Anfangswert von `10px` zurück und bewahrt so die Typsicherheit.

Die CSS-`@property`-Regel gilt seit Anfang 2024 als breit verfügbar in allen wichtigen modernen Browser-Engines (wie Chrome, Edge, Opera, Firefox und Safari). Wenn Sie frühere Browser unterstützen möchten, müssen Sie wie oben erklärt Fallback-CSS bereitstellen.

Zusammenfassend stellt die `@property`-Regel einen bedeutenden Fortschritt dar, wie wir mit benutzerdefinierten Eigenschaften in CSS arbeiten. Sie bietet mehr Kontrolle und ermöglicht neue Möglichkeiten wie die Animation von benutzerdefinierten Eigenschaften. Es ist jedoch wichtig, sie mit Bedacht einzusetzen und immer geeignete Fallbacks bereitzustellen, um eine gute Erfahrung in allen Browsern zu gewährleisten. Wie bei jeder modernen Webtechnologie ist der Schlüssel, die Erfahrung für unterstützende Browser zu verbessern und gleichzeitig die Grundfunktionalität für andere sicherzustellen.
