# --description--

En lecciones anteriores, aprendiste cómo trabajar con imports y exports en JavaScript. En esta lección, veremos cómo importar y exportar componentes en React.

Una exportación hace que un componente esté disponible para otros archivos. Una importación permite que un archivo use un componente que fue exportado en otro lugar. En proyectos de React, así es como se reutilizan y combinan los componentes para construir interfaces de usuario.

En esta lección, aprenderás cómo exportar componentes de React usando exportaciones por defecto y exportaciones nombradas, y cómo importarlos donde se necesiten.

En este ejemplo, tenemos un componente `Cat` que pertenece a un archivo llamado `Cat.jsx`. Para la extensión del archivo, estamos usando la extensión `.jsx` porque este archivo trabaja principalmente con JSX.

Este componente `Cat` devuelve un marcado JSX con un título e imagen para un gato llamado Mr. Whiskers:

```jsx
function Cat() {
  return (
    <div className="card">
      <h2>Mr. Whiskers</h2>
      <img
        src="https://cdn.freecodecamp.org/curriculum/cat-photo-app/running-cats.jpg"
        alt="Tuxedo cats running on dirt ground."
      />
    </div>
  );
}
```

Si queremos utilizar nuestro componente `Cat` en otro archivo, primero debemos exportarlo así:

```jsx
function Cat() {
  return (
    <div className="card">
      <h2>Mr. Whiskers</h2>
      <img
        src="https://cdn.freecodecamp.org/curriculum/cat-photo-app/running-cats.jpg"
        alt="Tuxedo cats running on dirt ground."
      />
    </div>
  );
}

export default Cat;
```

Estamos utilizando la palabra clave `default` porque este será el exportador predeterminado del módulo. También puedes elegir exportar el componente en la misma línea que la definición del componente de esta manera:

```jsx
export default function Cat() {
  return (
    <div className="card">
      <h2>Mr. Whiskers</h2>
      <img
        src="https://cdn.freecodecamp.org/curriculum/cat-photo-app/running-cats.jpg"
        alt="Tuxedo cats running on dirt ground."
      />
    </div>
  );
}
```

Puedes optar por importar componentes hijos en otros archivos de componentes padres o importarlos en el archivo del componente raíz. Para este ejemplo, importaremos el componente `Cat` dentro del archivo del componente raíz.

Cada proyecto de React tendrá un componente de nivel superior, típicamente llamado `App.jsx`:

```jsx
export default function App() {
  return (
    // return component here
  );
}
```

Este archivo generalmente se encuentra en el directorio `src` de tu proyecto. Aprenderás más sobre las estructuras comunes de proyectos en una lección futura.

Para utilizar el componente `Cat` dentro del componente raíz `App`, necesitarás importarlo así:

```jsx
import Cat from "./Cat";

export default function App() {
  return (
    // return component here
  );
}
```

Ahora, puedes retornar el componente `Cat` dentro del componente `App` así:

```jsx
import Cat from "./Cat";

export default function App() {
  return (
    <Cat />
  );
}
```

Este enfoque funciona bien cuando un archivo es responsable de exportar un solo componente principal, manteniendo la sintaxis de importación simple y fácil de leer. Recuerda, un archivo solo puede tener una exportación por defecto, lo que lo hace ideal para un archivo que contiene principalmente un solo componente.


Ahora que sabes cómo usar las exportaciones por defecto, veamos las exportaciones con nombre. Las exportaciones con nombre permiten que un archivo comparta múltiples componentes o funciones. A diferencia de las exportaciones por defecto, estas deben importarse usando el nombre exacto con el que fueron exportadas (a menos que las renombres usando el término `as`). 

Esto es útil cuando un archivo sirve como una biblioteca de componentes. Por ejemplo, aquí hay un archivo que contiene dos componentes, Cat y Dog:

```jsx
// Animals.jsx
export function Cat() {
  return <h2>Mr. Whiskers</h2>;
}

export function Dog() {
  return <h2>Fido</h2>;
}
```

Usamos la palabra clave export individualmente en cada componente. Para usarlos en otro archivo, los importamos usando llaves:


```jsx
import { Cat, Dog } from "./Animals";

export default function App() {
  return (
    <div>
      <Cat />
      <Dog />
    </div>
  );
}
```

La sintaxis `{ Cat, Dog }` le indica a JavaScript que importe las exportaciones nombradas específicas del archivo `Animals.jsx`. Los nombres dentro de las llaves deben coincidir exactamente con los nombres exportados. Si necesitas evitar conflictos de nombres, o si prefieres un nombre diferente en el archivo actual, puedes renombrar un componente durante la importación usando la palabra clave as.

```jsx
import { Cat as Kitty } from "./Animals";

export default function App() {
  return <Kitty />;
}
```

Finalmente, un archivo puede tener una exportación predeterminada y múltiples exportaciones nombradas.

```jsx
// Animals.jsx
export default function Cat() {
  return <h2>Mr. Whiskers</h2>;
}

export function Dog() {
  return <h2>Fido</h2>;
}
```

Al importar exportaciones mixtas, la exportación por defecto viene primero (sin llaves), seguida de las exportaciones nombradas (dentro de llaves):

```jsx
// App.jsx
import Cat, { Dog } from "./Animals";

export default function App() {
  return (
    <div>
      <Cat />
      <Dog />
    </div>
  );
}
```

En este ejemplo, `Cat` es la exportación por defecto, por lo que no usa llaves. `Dog` es una exportación nombrada, por lo que las requiere.

Al entender cómo funcionan las exportaciones por defecto y nombradas, puedes organizar los componentes de React en varios archivos y reutilizarlos donde sea necesario. A medida que construyes aplicaciones más grandes, elegir el estilo de exportación correcto ayudará a mantener tu código claro y mantenible, y facilitará trabajar con componentes creados por otros.
