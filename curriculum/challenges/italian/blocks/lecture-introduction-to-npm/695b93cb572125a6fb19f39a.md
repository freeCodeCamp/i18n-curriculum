# --description--

In una lezione precedente, hai imparato cosa sono le dipendenze. Qui, approfondirai l’argomento e vedrai cosa considerare quando scegli un pacchetto esterno da usare nel tuo progetto.

Una dipendenza è un altro pezzo di codice di cui il tuo progetto ha bisogno per funzionare. Di solito si tratta di pacchetti esterni creati e condivisi da altri sviluppatori nel registro npm.

Usare dipendenze ti fa risparmiare molto tempo. Invece di costruire tutto da zero, puoi installare un pacchetto che risolve già un problema comune — come creare un server web, gestire il routing, recuperare dati, gestire l’autenticazione e altro ancora.

Nel tuo file `package.json`, tutti i pacchetti da cui dipende il progetto sono elencati sotto la chiave `dependencies`:

```json
{
  "name": "package-dependencies",
  "version": "1.0.0",
  "description": "a demo on how package dependencies work",
  "main": "index.js",
  "scripts": {
    "test": "echo \"Error: no test specified\" && exit 1"
  },
  "keywords": [
    "Node",
    "NPM"
  ],
  "author": "Kolade Chris",
  "license": "ISC",
  "dependencies": {
    "chalk": "^5.6.2",
    "lodash": "^4.17.21"
  }
}
```

Nel file `package.json` di esempio sopra, `chalk` e `lodash` sono elencati come dipendenze del progetto. Questo significa che il progetto si basa su questi pacchetti per funzionare. Se non sono installati — e non sono elencati nel file `package.json` — qualsiasi funzionalità che li usa si romperà.

Esiste anche un’altra categoria di dipendenze chiamata `devDependencies`. Sono pacchetti necessari solo durante la build o il test del progetto, non quando è in esecuzione in produzione. Esempi comuni includono:

* Nodemon per riavviare automaticamente il progetto quando ci sono aggiornamenti
    
* Jest o Mocha per i test
    
* ESLint per il linting del codice
    

Come ti aspetti, `devDependencies` compaiono nella sezione `devDependencies` del file `package.json`:

```json
{
  "name": "package-dependencies",
  "version": "1.0.0",
  "description": "a demo on how package dependencies work",
  "main": "index.js",
  "scripts": {
    "test": "echo \"Error: no test specified\" && exit 1"
  },
  "keywords": [
    "Node",
    "NPM"
  ],
  "author": "Kolade Chris",
  "license": "ISC",
  "dependencies": {
    "chalk": "^5.6.2",
    "lodash": "^4.17.21"
  },
  "devDependencies": {
    "jest": "^30.2.0",
    "nodemon": "^3.1.10"
  }
}
```

Quando scegli quali pacchetti usare nel tuo progetto, ci sono alcune cose importanti da tenere a mente.

La sicurezza deve sempre venire prima di tutto. Anche un solo pacchetto non sicuro può creare problemi all’intero progetto. Devi anche considerare le prestazioni, perché un pacchetto grande o poco ottimizzato può rallentare il progetto.

È utile anche scegliere pacchetti popolari e ben supportati. I pacchetti popolari di solito hanno comunità attive, buona documentazione e aggiornamenti frequenti. Un modo semplice per verificare la popolarità è guardare il numero di download settimanali su npm. Per esempio, il pacchetto `chalk` ha oltre 300.000 download ogni settimana:

<img src="https://cdn.freecodecamp.org/curriculum/lecture-transcripts/what-are-package-dependencies-and-what-are-some-considerations-for-choosing-an-external-package-1.webp" alt="screenshot from npm of chalk package statistics">

Anche la manutenzione è importante. I pacchetti ben mantenuti vengono aggiornati regolarmente e funzionano meglio con le nuove versioni di altri software.

Infine, non trascurare la documentazione. Documenti chiari e completi rendono molto più facile capire come usare un pacchetto e ti fanno risparmiare molto tempo nell’integrarlo nel progetto.
