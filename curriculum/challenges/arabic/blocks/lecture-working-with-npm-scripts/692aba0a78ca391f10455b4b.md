# --description--

في تطوير الويب الحديث، يمكنك تقسيم شفرتك إلى ملفات متعددة تُسمى "وحدات".

يمكنك التفكير في الوحدة على أنها كتلة شفرة مستقلة وقابلة لإعادة الاستخدام.

عن طريق استيراد الوحدات إلى وحدات أخرى، يمكنك إعادة استخدام وظائفها مع الحفاظ على الشفرة في ملفات منفصلة، مما يجعلها أكثر قابلية للصيانة، خاصةً لتطبيقات الويب الكبيرة والمعقدة.

في Node.js، يُعتبر كل ملف وحدة.

لدى Node نظامان مختلفان للعمل مع الوحدات: CommonJS و ES.

CommonJS هو الطريقة الأصلية والافتراضية للعمل مع الحزم في Node.js. يقوم بتحميل الوحدات بشكل متزامن، مما يعني أنه يجب الانتظار حتى يتم تحميل الوحدة بالكامل قبل أن يبدأ في تحميل الوحدة التالية أو متابعة تنفيذ الشفرة.

لا يزال يُستخدم على نطاق واسع، لكنه يُستبدل تدريجيًا بصيغة ES الأحدث.

يستخدم CommonJS الدالة `require()` لاستيراد وحدة، حيث يأخذ مسارها النسبي كمعلمة.

على سبيل المثال، لنفترض أن لدينا وحدة تُسمى `math.js`، حيث نُعرّف العمليات الرياضية الأساسية.

سنستخدم دالة الضرب فقط هنا لجعل هذا المثال بسيطًا ومركزًا:

```js
function multiply(a, b) {
  return a * b;
}
```

كيف يمكننا إعادة استخدام هذه الدالة خارج وحدتنا الحالية؟ افتراضيًا، الدوال والمتغيرات المعرفة في وحدة Node.js تكون **خاصة** — لا يمكن للوحدات الأخرى الوصول إليها إلا إذا قمنا بتصديرها صراحةً. هناك عدة طرق للقيام بذلك.

طريقة شائعة هي تعيين قيمة إلى `module.exports`. هذا الكائن يمثل ما سترجعه الوحدة عند استيرادها في مكان آخر، مما يتيح للوحدات الأخرى الوصول إلى الدوال أو المتغيرات أو الكائنات التي اخترت إتاحتها.

في هذا المثال، نُعيّن كائنًا يحتوي على خاصية `multiply` إلى `module.exports`. قيمة هذه الخاصية هي مرجع إلى دالة `multiply` التي عرّفناها سابقًا. هذا سيصدر الدالة، حتى نتمكن من استدعائها من وحدات أخرى إذا احتجنا.

```javascript
function multiply(a, b) {
  return a * b;
}

module.exports = {
  multiply: multiply
};
```

لنفترض أننا نحاول استيراد وحدة `math.js` إلى وحدة أخرى. نستدعي الدالة `require()`، ونمرر المسار النسبي كمعلمة. ثم نُعيّن الكائن الذي تُرجعه `require()` إلى المتغير `math`.

```javascript
const math = require('./math');
```

دعنا نتحقق من قيمة `math` في وحدة التحكم بطباعتها باستخدام `console.log()`:

```javascript
console.log(math);
```

هذا هو الناتج. إنه كائن يحتوي على خاصية `multiply` ودالة كقيمة:

```javascript
{ multiply: [Function: multiply] }
```

إذا استدعينا هذه الدالة باستخدام `math.multiply()`، مع تمرير الرقمين `6` و `7` كمعلمات، يكون الناتج `42`. الدالة تعمل بنجاح ومع ذلك، هي في ملف آخر! إذن، جملة الاستيراد الخاصة بنا عملت بشكل صحيح.

```javascript
const math = require('./math');

console.log(math.multiply(6, 7)); // 42
```

لكن هناك طرق أخرى لتصدير دالة أو متغير أو كائن.

على سبيل المثال، إذا كنت تحتاج فقط إلى تصدير دالة أو متغير أو كائن واحد، يمكنك تعيينه مباشرة كقيمة لـ `module.exports`:

```javascript
function multiply(a, b) {
  return a * b;
}

module.exports = multiply;
```

إذا كنت تحتاج إلى تصدير عدة دوال، فقط أضفها إلى كائن `module.exports`، مفصولة بفواصل.

هنا، يمكنك أن ترى أن الكائن المصدر يحتوي على خاصيتين: `subtract` و `multiply`. قيمهما هي مراجع إلى دوالهما المقابلة، لذا سيكون لديك وصول إلى كلتيهما في أي وحدة تستورد `math.js`:

```javascript
function subtract(a, b) {
  return a - b;
}

function multiply(a, b) {
  return a * b;
}

module.exports = {
  subtract: subtract,
  multiply: multiply
};
```

عندما يحتوي كائن `module.exports` على خصائص متعددة، سيكون لديك خيار اختيار استيراد كل شيء أو استيراد الخصائص بشكل فردي.

في هذا المثال، نستورد الكائن الكامل الذي صدرته `math.js`، لذا سيكون لدينا وصول إلى كل من `subtract` و `multiply`:

```javascript
const math = require('./math');

console.log(math.subtract(6, 7));  // -1
console.log(math.multiply(6, 7));  // 42
```

إذا أردنا استيراد `multiply` فقط، نكتب اسمه داخل أقواس متعرجة، بعد كلمة `const`. في الأساس، تستخدم صيغة تعيين بالتفكيك للحصول على قيمته من الكائن المصدر. ثم يمكننا الإشارة إليه مباشرة باسمه في شفرتنا:

```javascript
const { multiply } = require('./math');

console.log(multiply(6, 7));
```

هذا يعمل لعناصر متعددة أيضًا. فقط افصل بينها بفواصل داخل الأقواس المتعرجة. هنا، نستورد كل من دالتي `subtract` و `multiply`:

```javascript
const { subtract, multiply } = require('./math');
```

افتراضيًا، سيعامل Node.js الملفات التالية كوحدات CommonJS:

* الملفات التي تحمل امتداد `.cjs`.
    
* الملفات التي تحمل امتداد `.js` عندما يحتوي أقرب ملف `package.json` أبوي على حقل من المستوى الأعلى `"type"` بقيمة `"commonjs"`.
    
* الملفات التي تحمل امتداد `.js` أو بدون امتداد، عندما لا يحتوي أقرب ملف `package.json` أبوي على حقل من المستوى الأعلى `"type"`، أو لا يوجد ملف `package.json` في أي مجلد أبوي.
    
* الملفات التي تحمل امتدادًا ليس من بين `.mjs`، `.cjs`، `.json`، `.node`، أو `.js`.
    

بينما تستبدل وحدات ES تدريجيًا وحدات CommonJS، فإن فهمها لا يزال ضروريًا للمطورين الذين يعملون مع قواعد شفرة Node.js الحالية والعديد من حزم npm.
