# --description--

У цьому уроці ми розглянемо мапи, хеш-мапи та множини. Але перед цим давайте визначимо абстрактні типи даних.

Абстрактний тип даних (ADT) — це концептуальне представлення типу даних, яке включає операції, що можна виконувати над даними, та властивості цих даних.

Абстрактні типи даних — це як креслення, які описують **які** операції можна виконувати, а не **як** їх виконувати. Вони відокремлюють інтерфейс від фактичної реалізації операцій.

**Мапа** — це ADT, який керує колекціями пар ключ-значення та їх операціями дуже специфічним і ефективним способом.

У мапі кожне значення пов’язане з певним ключем.

Однією з ключових характеристик мап є те, що кожен ключ має бути унікальним. Ця унікальність дозволяє здійснювати прямий пошук, що робить процес отримання інформації набагато ефективнішим.

Унікальними мають бути лише ключі, значення можуть повторюватися.

Абстрактний тип даних мапи також визначає важливі операції, такі як вставка пар ключ-значення, отримання значення за ключем, оновлення значення за ключем, видалення пари ключ-значення та перевірка наявності ключа в мапі.

Він не визначає, як саме ці операції мають виконуватися, а лише перераховує їх як частину доступних операцій типу даних.

**Хеш-мапа**, також відома як хеш-таблиця, — це конкретна реалізація абстрактного типу даних мапи.

Хеш-мапи використовують техніку, яка називається «хешування», щоб дуже ефективно виконувати загальні операції.

Хешування працює шляхом генерації хеш-значення для кожного елемента за допомогою хеш-функції.

Хеш-значення генерується на основі ключа пари ключ-значення і використовується для обчислення індексу в базовому масиві — фактичній структурі даних, де зберігаються пари ключ-значення.

Але ви можете запитати: що станеться, якщо два ключі дадуть однаковий індекс?

Хеш-мапи розв’язують ці колізії за допомогою хитрих стратегій.

Один із варіантів — використання стратегії «ланцюжків», коли кожен індекс масиву вказує на зв’язаний список (інша структура даних), де зберігаються всі елементи з однаковим індексом.

Інша стратегія — «відкрите адресування», що передбачає пошук наступного доступного індексу в масиві за заздалегідь визначеною послідовністю пошуку.

Середня складність часу виконання операцій вставки, отримання та видалення пар ключ-значення у хеш-мапах — «постійний час» `O(1)`.

Найгірша складність часу виконання цих операцій — лінійний час `O(n)`, що трапляється, коли виникає багато хеш-колізій, і стратегія розв’язання колізій застосовується багаторазово.

Просторова складність вставки в хеш-мапу у середньому — постійна `O(1)`, тобто постійний обсяг пам’яті для збереження нової пари. Однак у найгіршому випадку вона може бути лінійною `O(n)` через операцію зміни розміру базового масиву. Загалом видалення елемента має постійну просторова складність `O(1)`.

Це перетворює хеш-таблицю на щось подібне до лінійної структури даних, де `n` елементів потрібно просканувати, щоб знайти цільовий ключ. Проте це відносно рідко, якщо хеш-мапа реалізована правильно.

У Python **словники** реалізовані як хеш-мапи під капотом.

Щоб створити словник у Python, достатньо записати пари ключ-значення в фігурних дужках, розділяючи їх комами. Кожен ключ має бути відокремлений від відповідного значення двокрапкою.

```python
my_dictionary = {
  'A': 1,
  'B': 2, 
  'C': 3
}
```

У цьому коді `'A'` — це ключ, а `1` — значення:

```python
'A': 1
```

Альтернативно можна використати `dict()`:

```python
my_dictionary = dict(A=1, B=2, C=3)
```

Ви можете отримати значення за відповідним ключем:

```python
my_dictionary['A']  # 1
```

Також можна оновити значення, пов’язане з ключем:

```python
my_dictionary['A'] = 4
```

І видалити пару ключ-значення:

```python
del my_dictionary['A']  
```

Ви також можете перевірити, чи є ключ у словнику (або ні):

```python
'C' in my_dictionary
```

І викликати ці методи, щоб отримати ключі, значення та елементи словника відповідно.

```python
my_dictionary.keys()
my_dictionary.values()
my_dictionary.items()
```

Чудово. Тепер, коли ви знаєте більше про мапи та хеш-мапи, поговоримо про множини.

**Множини** — це невпорядковані колекції унікальних елементів.

Розберемо це поняття на ключові складові:

* Множини невпорядковані. Елементи множини не зберігаються у певному порядку, тому до них не можна звертатися за індексами.
    
* Множини містять лише унікальні елементи. Якщо ви спробуєте додати те саме значення двічі, збережеться лише одна копія.
    

Вони аналогічні множинам у математиці і реалізують ті ж операції над множинами, як-от перетин, об’єднання та різниця.

Однією з головних переваг множин є гарантія унікальності елементів (без дублікатів). Саме тому їх часто використовують для видалення дублікатів зі списків та інших структур даних.

Вони також динамічні. Множини можуть підлаштовуватися під кількість елементів, які зараз зберігаються. Це робить їх досить потужними.

Середня складність часу виконання операцій додавання, видалення, отримання довжини множини та перевірки наявності елемента — «постійний час» `O(1)`, що дуже ефективно.

Оскільки множини реалізовані як хеш-таблиці, найгірша складність часу виконання додавання, видалення та перевірки членства — «лінійний час» `O(n)`. Це може статися, коли виникає багато хеш-колізій, перетворюючи хеш-таблицю на щось подібне до лінійної структури даних, де потрібно `n` сканувань, щоб знайти ключ.

Щодо просторової складності, у середньому вставка елемента має постійну складність `O(1)`, оскільки новий унікальний елемент потребує постійного обсягу пам’яті. Однак у найгіршому випадку може відбутися операція зміни розміру базового масиву, що призведе до лінійної просторової складності `O(n)`. Загалом видалення елемента має постійну просторову складність `O(1)`.

У Python є вбудована `set` структура даних, яку ви використовуєте для роботи з множинами у своїх програмах.

Під капотом множини Python реалізовані за допомогою хеш-таблиці, де зберігаються лише ключі без будь-яких пов’язаних значень.

Множини можуть зберігати лише об’єкти незмінних типів даних, оскільки їх хеш-значення завжди залишаються однаковими. Натомість хеш-значення змінних об’єктів можуть змінюватися при їх зміні. Саме тому вони не можуть бути частиною множин. Якщо хеш-значення об’єкта, що зберігається в множині, зміниться, програма більше не зможе його знайти.

Щоб визначити множину в Python, достатньо оточити елементи фігурними дужками і розділити їх комами:

```python
numbers = {1, 2, 3, 4}
```

Щоб створити порожню множину, можна викликати `set()`:

```python
numbers = set()
```

Зверніть увагу, що якщо ви використаєте порожні фігурні дужки, це автоматично створить словник Python, а не множину, тому потрібно викликати функцію `set()`, щоб створити порожню множину.

Ви можете додати елемент до множини за допомогою методу `.add()`:

```python
numbers.add(5)
```

Також можна видалити елементи з множини за допомогою методу `.remove()`:

```python
numbers.remove(5)
```

Це викличе `KeyError`, якщо елемент не знайдено. Але якщо ви не хочете викликати помилку в такому випадку, можна замість цього використати метод `.discard()`.

Метод `.pop()` повертає довільний елемент із множини, а метод `.clear()` видаляє всі елементи з множини.

Ви можете перевірити, чи є елемент у множині, за допомогою оператора `in`:

```python
5 in numbers
```

Python також підтримує операції над множинами, включно з об’єднанням, різницею, симетричною різницею та перетином, які можна виконувати за допомогою цих методів:

```python
set_a = {1, 2, 3, 4}
set_b = {2, 3, 4, 5, 6}

set_a.union(set_b)
set_a.intersection(set_b)
set_a.symmetric_difference(set_b)
set_a.difference(set_b)
```

Або за допомогою їх еквівалентних операторів:

```python
set_a | set_b
set_a & set_b
set_a ^ set_b
set_a - set_b 
```

Середня складність часу виконання додавання, видалення та перевірки членства — «постійний час» `O(1)`.

Найгірша складність часу виконання цих операцій — «лінійний час» `O(n)` через найгірший сценарій колізій у хеш-мапі.

Ви також можете перевірити, чи є множина підмножиною або надмножиною іншої:

```python
set_a.issubset(set_b)
set_a.issuperset(set_b)
```

Загалом, множини варто використовувати, коли потрібно зберігати колекцію унікальних елементів і часто перевіряти наявність елемента.

Мапи, хеш-мапи та множини — це потужні структури даних, створені для ефективної організації та отримання даних. Кожна з них має свої унікальні характеристики та випадки використання. Як розробник, ви повинні обрати найкращу для свого проєкту.
