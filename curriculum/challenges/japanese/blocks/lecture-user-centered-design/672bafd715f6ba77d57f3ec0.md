# --description--

ユーザーリサーチとは、あなたの製品を使う人々を体系的に調査することです。目的はユーザーのニーズ、行動、問題点を測定することです。

ユーザーリサーチにはさまざまな形態があります。最も一般的なものの一つがネットプロモータースコア（NPS）です。NPSはユーザーがあなたの製品を友人にどれだけ勧めたいかを測定します。NPSはユーザーの旅路の重要な節目、例えば7日後、30日後、90日後に提供されるアンケートで測定されます。NPSは0から10のスケールで評価され、9と10は積極的な推奨者を示します。

もう一つのリサーチ手法はエグジットインタビューです。これはユーザーがサブスクリプションをキャンセルしたりアカウントを削除したときに表示されるアンケートです。このアンケートのデータはユーザー離脱の原因を把握するのに役立ち、対処することができます。

一方、ユーザーテストとは、ユーザーがあなたのアプリケーションとやり取りする際のデータを取得する実践を指します。例えば、ベータテスト中のビデオゲームはユーザーテストの一形態です。ウェブ開発者として遭遇するかもしれないのはA/Bテストです。A/Bテストは新機能をユーザーの部分集合にランダムに提供し、分析データを活用してその機能が有益かどうかを判断します。

最後に、ユーザー要件とはあなたのアプリケーションが従うべきストーリーや基準のことです。これは開発プロセスに情報を提供します。ユーザー要件はユーザーリサーチや業界標準、さらにはステークホルダーの意見によって定義される場合もあります。

これらの要件は機能的なもの、つまりアプリケーションがどのように動作すべきかを規定するものか、非機能的なもの、つまりアプリケーションがどのように振る舞うべきかを定義するものかに分けられます。ユーザー要件は静的なものではありません。ユーザーテストやユーザーリサーチの情報は要件に影響を与え、ユーザーベースの変化に伴って変わっていきます。

違いを理解することは、最も正確なデータを収集し、エンドユーザーに最高の体験を提供するために不可欠です。
