# --description--

このレッスンでは、マップ、ハッシュマップ、セットについて説明します。その前に、抽象データ型について定義しましょう。

抽象データ型（ADT）とは、データ型の概念的な表現であり、そのデータに対してどのような操作ができるかや、そのデータの性質を含みます。

抽象データ型は、**どのように**操作が行われるかではなく、**何を**操作できるかを示す設計図のようなものです。インターフェースと実際の操作の実装を分離しています。

**マップ**は、キーと値のペアの集合とその操作を非常に特定かつ効率的に管理する抽象データ型です。

マップでは、すべての値が特定のキーに関連付けられています。

マップの重要な特徴の一つは、すべてのキーが一意でなければならないことです。この一意性により、直接的な検索が可能になり、情報の取得が非常に効率的になります。

キーだけが一意であればよく、値は重複してもかまいません。

マップの抽象データ型は、キーと値のペアの挿入、キーに関連付けられた値の取得、キーに関連付けられた値の更新、キーと値のペアの削除、キーがマップに存在するかの確認などの重要な操作も定義しています。

これらの操作がどのように実装されるかは指定しておらず、利用可能な操作の一覧として示しています。

**ハッシュマップ**（ハッシュテーブルとも呼ばれます）は、マップ抽象データ型の具体的な実装です。

ハッシュマップは「ハッシュ化」と呼ばれる手法を使って、一般的な操作を非常に効率的に行います。

ハッシュ化は、ハッシュ関数を使って各要素のハッシュ値を生成する仕組みです。

ハッシュ値はキーに基づいて生成され、キーと値のペアが格納される基盤となる配列のインデックスを計算するために使われます。

ここで疑問に思うかもしれません：もし2つのキーが同じインデックスになるとどうなるのでしょうか？

ハッシュマップは巧妙な戦略でこれらの衝突を解決します。

一つの方法は「チェーン」戦略で、各配列のインデックスが連結リスト（別のデータ構造）を指し、同じインデックスのすべての要素がそこに格納されます。

もう一つの戦略は「オープンアドレッシング」で、あらかじめ定められた検索シーケンスに基づいて配列内の次の空きインデックスを探します。

ハッシュマップの平均的な時間計算量は、キーと値のペアの挿入、取得、削除において「定数時間」`O(1)`です。

これらの操作の最悪の場合の時間計算量は「線形時間」`O(n)`で、多くのハッシュ衝突が発生し、衝突解決戦略を何度も適用しなければならない場合に起こります。

ハッシュマップへの挿入の空間計算量は平均的には定数`O(1)`で、新しいペアを格納するための一定量のメモリが必要です。しかし最悪の場合、基盤となる配列のリサイズ操作により線形の空間計算量`O(n)`になることがあります。一般的に、要素の削除は定数の空間計算量`O(1)`です。

この場合、ハッシュテーブルは線形データ構造に似たものになり、ターゲットのキーを見つけるために`n`個の要素をスキャンする必要があります。ただし、これはハッシュマップが適切に実装されていれば比較的まれです。

Pythonの**辞書**は内部的にハッシュマップとして実装されています。

Pythonの辞書を作成するには、中括弧内にキーと値のペアを書き、カンマで区切るだけです。各キーは対応する値とコロンで区切ります。

```python
my_dictionary = {
  'A': 1,
  'B': 2, 
  'C': 3
}
```

このコードでは、`'A'`がキーで、`1`が値です：

```python
'A': 1
```

あるいは、`dict()`を使うこともできます：

```python
my_dictionary = dict(A=1, B=2, C=3)
```

対応するキーを使って値を取得できます：

```python
my_dictionary['A']  # 1
```

キーに関連付けられた値を更新することもできます：

```python
my_dictionary['A'] = 4
```

キーと値のペアを削除することもできます：

```python
del my_dictionary['A']  
```

辞書にキーが存在するかどうかも確認できます：

```python
'C' in my_dictionary
```

また、これらのメソッドを呼び出して、辞書のキー、値、アイテムをそれぞれ取得できます。

```python
my_dictionary.keys()
my_dictionary.values()
my_dictionary.items()
```

さて、マップとハッシュマップについて理解できたので、次はセットについて話しましょう。

**セット**は、順序のない一意の要素の集合です。

この概念を重要なポイントに分解しましょう：

* セットは順序がありません。セットの要素は特定の順序で格納されていないため、インデックスでアクセスできません。
    
* セットは一意の要素のみを含みます。同じ値を2回追加しようとすると、値は1つだけ保持されます。
    

数学の集合に似ており、交差、和集合、差集合などの集合演算を実装しています。

セットの主な利点の一つは、要素が一意であることを保証することです（重複なし）。そのため、リストや他のデータ構造から重複を取り除くためによく使われます。

また、セットは動的です。現在格納されている要素数に応じて調整されます。これにより非常に強力です。

セットの要素の追加、削除、長さの取得、要素の存在確認の平均的な時間計算量は「定数時間」`O(1)`で、とても効率的です。

セットはハッシュテーブルとして実装されているため、追加、削除、メンバーシップの最悪の場合の時間計算量は「線形時間」`O(n)`です。これは複数のハッシュ衝突が発生し、ハッシュテーブルが線形データ構造に似たものになり、`n`回のスキャンが必要になる場合に起こります。

空間計算量に関しては、平均的には要素の挿入は定数の計算量`O(1)`で、新しい一意の要素は一定量のメモリを必要とします。しかし最悪の場合、基盤となる配列のリサイズ操作があり、線形の空間計算量`O(n)`になることがあります。一般的に、要素の削除は定数の空間計算量`O(1)`です。

Pythonには組み込みの`set`データ構造があり、プログラムでセットを扱うために使います。

内部的に、Pythonのセットはキーだけを格納し、値は持たないハッシュテーブルで実装されています。

セットはハッシュ値が常に変わらない不変のデータ型のオブジェクトのみを格納できます。対照的に、可変オブジェクトのハッシュ値は変更されることがあり、そのためセットの一部にはできません。セットに格納されたオブジェクトのハッシュ値が変わると、プログラムはそのオブジェクトを見つけられなくなります。

Pythonでセットを定義するには、要素を中括弧で囲み、カンマで区切るだけです：

```python
numbers = {1, 2, 3, 4}
```

空のセットを作成するには、`set()`を呼び出します：

```python
numbers = set()
```

空の中括弧を使うと自動的にPythonの辞書が作成されるため、空のセットを作るには`set()`関数を呼び出す必要があります。

セットに要素を追加するには、`.add()`メソッドを使います：

```python
numbers.add(5)
```

セットから要素を削除するには、`.remove()`メソッドを使います：

```python
numbers.remove(5)
```

この操作は要素が見つからない場合に`KeyError`をスローします。しかし、エラーをスローしたくない場合は、代わりに`.discard()`メソッドを使うことができます。

`.pop()`メソッドはセットから任意の要素を返し、`.clear()`メソッドはセットのすべての要素を削除します。

要素がセットに含まれているかどうかは、`in`演算子でテストできます：

```python
5 in numbers
```

Pythonは和集合、差集合、対称差集合、交差集合などの集合演算もサポートしており、これらのメソッドで実行できます：

```python
set_a = {1, 2, 3, 4}
set_b = {2, 3, 4, 5, 6}

set_a.union(set_b)
set_a.intersection(set_b)
set_a.symmetric_difference(set_b)
set_a.difference(set_b)
```

または、それらに対応する演算子で実行できます：

```python
set_a | set_b
set_a & set_b
set_a ^ set_b
set_a - set_b 
```

要素の追加、削除、メンバーシップの平均的な時間計算量は「定数時間」`O(1)`です。

これらの操作の最悪の場合の時間計算量は、ハッシュマップの最悪の衝突シナリオのため「線形時間」`O(n)`です。

また、あるセットが別のセットの部分集合か上位集合かを確認することもできます：

```python
set_a.issubset(set_b)
set_a.issuperset(set_b)
```

一般的に、一意のアイテムの集合を格納し、頻繁にアイテムの存在を確認する必要がある場合はセットを使うべきです。

マップ、ハッシュマップ、セットは効率的なデータの整理と取得のために設計された強力なデータ構造です。それぞれに独自の特徴と用途があります。開発者として、プロジェクトに最適なものを選ぶ必要があります。
