# --description--

Expressでは、`express.Router()`はルートをより小さくモジュール化された単位に整理するのに役立つ強力な機能です。これはアプリケーションが成長するにつれて特に便利です。

すべてのルートを1つの大きなファイルに定義する代わりに、ルーターを使って別々のファイルに分割できます。これによりコードが読みやすく、保守しやすく、拡張しやすくなります。

`express.Router()`メソッドは新しいルーターオブジェクトを作成します。

ルーターはミニExpressアプリケーションのように動作します。独自のルートやミドルウェアを処理できますが、完全なExpressアプリケーションではありません。ルーターにルートを定義したら、`app.use()`を使ってメインのアプリにマウントできます。

つまり、ルーターは関連するルートをまとめてグループ化し、後でメインのアプリケーションに統合できるようにします。

ルーターを使う利点はいくつかあります：

**関心の分離**

ルートを別々のファイルに定義できるため、アプリケーションの整理と可読性が向上します。

**コードの再利用性**  
ルーターはアプリケーションの異なる部分や複数のプロジェクトで再利用できます。

**メインファイルがすっきりする**  
メインの`app.js`や`server.js`ファイルに多くのルート定義を詰め込む代わりに、ルーターに移動してメインファイルは設定やセットアップに集中できます。

`express.Router()`を使って`/users`パスのルートを定義する簡単な例を見てみましょう。

まず、`app.js`ファイルはこちらです：

```javascript
const express = require("express");
const app = express();
const port = 3000;

// Importing the user routes
const userRoutes = require("./userRoutes");

// Using the userRoutes router for /users path
app.use("/users", userRoutes);

app.listen(port, () => {
  console.log(`Server running at http://localhost:${port}`);
});
```

次に`userRoutes.js`はこちらです：

```javascript
const express = require("express");
const router = express.Router();

// GET request for /users
router.get("/", (req, res) => {
  res.send("List of users");
});

// POST request for /users
router.post("/", (req, res) => {
  res.send("Creating a new user");
});

// GET request for /users/:id
router.get("/:id", (req, res) => {
  const userId = req.params.id;
  res.send(`User details for user with id: ${userId}`);
});

module.exports = router;
```

では、全体の動作を詳しく見ていきましょう。

`app.js`では、`express`をインポートし、Expressアプリケーションのインスタンスを作成して`app`という変数に代入しています。

次に`userRoutes`から`userRoutes.js`ルーターをインポートし、`app.use("/users", userRoutes)`を使ってルーターをマウントしています。これは、リクエストが`/users`で始まる場合、`userRoutes`ルーターに渡すようExpressに指示しています。

そして`userRoutes.js`では、`express.Router()`を使って新しいルーターを作成し、`router.get()`、`router.post()`、`router.get("/:id")`を使ってルートを定義しています。

ここで定義されたルートは、ルーターがマウントされている`/users`に対して相対的であることに注意してください。つまり：

*   `router.get("/")` → `GET /users`：ユーザーの一覧を返します
    
*   `router.post("/")` → `POST /users`：新しいユーザー作成に関するメッセージを返します
    
*   `router.get("/:id")` → `GET /users/:id`：指定されたユーザーIDの詳細を返します
    

`express.Router()`を使うことでメインファイルがすっきりし、設定に集中できます。ユーザー関連のルートはすべて`userRoutes.js`にまとめられています。

アプリケーションが成長するにつれて、`/products`、`/orders`、`/auth`など他のセクション用に追加のルーターを作成できます。

各セクションはそれぞれのファイルで整理され、アプリケーションの保守と拡張がしやすくなります。
