# --description--

In der vorherigen Aufgabe hatte `bird` eine öffentliche Eigenschaft `name`. Sie gilt als öffentlich, weil sie außerhalb der Definition von `bird` aufgerufen und geändert werden kann.

```js
bird.name = "Duffy";
```

Deshalb kann jeder Teil Ihres Codes den Namen von `bird` ganz einfach auf einen beliebigen Wert ändern. Denken Sie an Dinge wie Passwörter und Bankkonten, die von jedem Teil Ihres Codes leicht änderbar wären. Das könnte viele Probleme verursachen.

Der einfachste Weg, diese öffentliche Eigenschaft privat zu machen, besteht darin, innerhalb der Konstruktorfunktion eine Variable zu erstellen. Dadurch wird der Gültigkeitsbereich dieser Variable auf die Konstruktorfunktion beschränkt und nicht global verfügbar. So kann nur noch auf die Variable zugegriffen und sie geändert werden, wenn sich die Methoden ebenfalls innerhalb der Konstruktorfunktion befinden.

```js
function Bird() {
  let hatchedEgg = 10;

  this.getHatchedEggCount = function() { 
    return hatchedEgg;
  };
}
let ducky = new Bird();
ducky.getHatchedEggCount();
```

Hier ist `getHatchedEggCount` eine privilegierte Methode, weil sie Zugriff auf die private Variable `hatchedEgg` hat. Das ist möglich, weil `hatchedEgg` im selben Kontext wie `getHatchedEggCount` deklariert ist. In JavaScript hat eine Funktion immer Zugriff auf den Kontext, in dem sie erstellt wurde. Das nennt man `closure`.
