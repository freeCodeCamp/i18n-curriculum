# --description--

React-Apps verlassen sich oft auf externe APIs und Datenbanken für dynamische Inhalte. Um auf die Daten dieser APIs und Datenbanken zuzugreifen, müssen Sie einige Techniken zum Datenabrufen verwenden.

Schauen wir uns an, wie das Datenabrufen in React funktioniert und welche verschiedenen Möglichkeiten Ihnen dafür zur Verfügung stehen.

React gibt keine Vorgaben, wie Sie Ihre Daten abrufen sollen. Das bedeutet, dass Sie auf einer grundlegenden Ebene die eingebaute Fetch API verwenden können, die alle modernen Browser unterstützen.

Sie können auch Axios und SWR verwenden. Axios ist eine auf Promises basierende HTTP-Anfragebibliothek, die auf dem XMLHttpRequest-Objekt aufbaut, und SWR ist ein React-Hook zum Datenabrufen, der vom Vercel-Team entwickelt wurde.

Beginnen wir mit einem Beispiel. Zuerst müssen Sie die `useState` und `useEffect` Hooks importieren:

```js
import { useState, useEffect } from "react";
```

Dann müssen Sie drei Zustandsvariablen namens `loading`, `data` und `error` erstellen:

```js
const [data, setData] = useState(null);
const [loading, setLoading] = useState(true);
const [error, setError] = useState(null);
```

Die Variable `loading` verfolgt, ob die Daten noch abgerufen werden. Die Variable `data` stellt die Daten selbst dar, und die Variable `error` erfasst mögliche Fehler, die während des Datenabrufprozesses auftreten können.

Da das Datenabrufen ein Nebeneffekt ist, ist es am besten, die Fetch API innerhalb eines `useEffect` Hooks zu verwenden.

Hier ist ein Beispiel dafür:

```js
useEffect(() => {
  fetch("https://jsonplaceholder.typicode.com/posts")
    .then((res) => res.json())
    .then((data) => {
      setData(data);
      setLoading(false);
    })
    .catch((err) => {
      setError(err);
      setLoading(false);
    });
}, []);
```

Dieser `useEffect` ruft die Daten mit der Fetch API ab und setzt alle Zustände.

Sie können es verbessern, indem Sie `async`/`await` anstelle der `.then()` Syntax verwenden. Das bedeutet, dass Sie eine separate Funktion innerhalb des `useEffect` benötigen, da Sie `useEffect` nicht mit dem `async` Schlüsselwort präfixieren können:

```js
useEffect(() => {
  const fetchData = async () => {
    try {
      const res = await fetch("https://jsonplaceholder.typicode.com/posts");
       
      if (!res.ok) {
        throw new Error("Network response was not ok");
      }

      const data = await res.json();
      setData(data);
    } catch (err) {
      setError(err);
    } finally {
      setLoading(false);
    }
  };

  fetchData();
}, []);
```

Sie können dann alle diese Zustände verwenden, um die Daten aus der API zu rendern.

Hier ist der vollständige Code:

```jsx
import { useState, useEffect } from "react";

const FetchPosts = () => {
  const [data, setData] = useState(null);
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState(null);

  useEffect(() => {
    const fetchData = async () => {
      try {
        const res = await fetch("https://jsonplaceholder.typicode.com/posts");
       
        if (!res.ok) {
          throw new Error("Network response was not ok");
        }

        const data = await res.json();
        setData(data);
      } catch (err) {
        setError(err);
      } finally {
        setLoading(false);
      }
    };

    fetchData();
  }, []);

  if (loading) {
    return <p>Loading...</p>;
  }

  if (error) {
    return <p>{error.message}</p>;
  }

  return (
    <ul>
      {data.map((post) => (
        <li key={post.id}>{post.title}</li>
      ))}
    </ul>
  );
};

export default FetchPosts;
```

In der Benutzeroberfläche würden Sie `Loading...` auf dem Bildschirm sehen, während die Daten abgerufen werden, und anschließend würden die Daten oder ein Fehler angezeigt, je nachdem, ob der Datenabruf erfolgreich war.

Erinnern Sie sich, dass wir auch über das Datenabrufen mit Axios und SWR gesprochen haben. Schauen wir uns ein Beispiel mit Axios an.

Zuerst müssen Sie Axios über die Befehlszeile so installieren:

```sh
npm i axios
```

Dann müssen Sie Axios so importieren:

```js
import axios from "axios";
```

Anschließend können Sie dieselben Zustandsvariablen wie zuvor verwenden und Daten von der API mit `axios.get` abrufen:

```js
const [data, setData] = useState(null);
const [loading, setLoading] = useState(true);
const [error, setError] = useState(null);

useEffect(() => {
  const fetchData = async () => {
    try {
      const res = await axios.get(
        "https://jsonplaceholder.typicode.com/users"
      );
      setData(res.data);
    } catch (err) {
      setError(err);
    } finally {
      setLoading(false);
    }
  };

  fetchData();
}, []);
```

Vielleicht ist Ihnen aufgefallen, dass es in diesem Beispiel keine `await res.json()` Zeile gibt. Das liegt daran, dass Axios JSON automatisch parst, sodass das nicht nötig ist.

Das letzte Beispiel, das wir uns ansehen, ist die Verwendung des `useSWR` Hooks zum Datenabrufen.

Genau wie bei Axios müssen Sie SWR so installieren:

```sh
npm install swr
```

Dann müssen Sie den `useSWR` Hook in die Datei importieren:

```js
import useSWR from "swr";
```

Im Vergleich zu den vorherigen Beispielen ist die SWR-Syntax viel kürzer. Sie müssen eine Fetcher-Funktion erstellen und diese als zweiten Parameter an den `useSWR` Hook übergeben (der Endpunkt ist der erste Parameter).

Sie können außerdem sowohl die Daten- als auch die Fehlerzustände aus dem `useSWR` Hook destrukturieren, sodass Sie den `useState` Hook nicht benötigen.

Hier ist die Syntax:

```js
const fetcher = (url) => fetch(url).then((res) => res.json());
const { data, error } = useSWR(endpoint, fetcher);
```

Beachten Sie, dass der Name „fetcher“ hier nur eine Konvention ist, Sie können die Variable also beliebig benennen.

Hier ist eine Komponente, die Todos von der JSON Placeholder API abruft:

```jsx
import useSWR from "swr";

const fetcher = (url) => fetch(url).then((res) => res.json());

const FetchTodos = () => {
  const { data, error } = useSWR(
    "https://jsonplaceholder.typicode.com/todos",
    fetcher
  );

  if (!data) {
    return <h2>Loading...</h2>;
  }
  if (error) {
    return <h2>Error: {error.message}</h2>;
  }

  return (
    <>
      <h2>Todos</h2>
      <div>
        {data.map((todo) => (
          <h3 key={todo.id}>{todo.title}</h3>
        ))}
      </div>
    </>
  );
};

export default FetchTodos;
```

Wie Sie in einer vorherigen Lektion über benutzerdefinierte Hooks gelernt haben, ist das Datenabrufen eine Logik, die Sie in einen benutzerdefinierten Hook auslagern können. Wenn Sie also Daten in mehreren Komponenten und Seiten abrufen, ist es am besten, einen `useFetch` Hook zu erstellen.

Hier ist ein `useFetch` Hook, der SWR für das Datenabrufen verwendet:

```jsx
import useSWR from "swr";

const fetcher = (url) => fetch(url).then((res) => res.json());

const useFetch = (url) => {
  const { data, error } = useSWR(url, fetcher);

  return {
    data,
    loading: !data && !error,
    error,
  };
};

export default useFetch;
```

Und so verwenden Sie den `useFetch` Hook, um das erste Beispiel umzuschreiben, das Beiträge von der JSON Placeholder API abruft:

```jsx
import useFetch from "./useFetch";

const FetchPosts = () => {
  const { data, loading, error } = useFetch(
    "https://jsonplaceholder.typicode.com/posts"
  );

  if (loading) {
    return <h2>Loading...</h2>;
  }

  if (error) {
    return <h2>{error.message}</h2>;
  }

  return (
    <>
      <h2>Posts</h2>
      <ul>
        {data.map((post) => (
          <li key={post.id}>{post.title}</li>
        ))}
      </ul>
    </>
  );
};

export default FetchPosts;
```
