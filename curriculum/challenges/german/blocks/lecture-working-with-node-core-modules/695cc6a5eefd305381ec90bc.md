# --description--

Viele Programmiersprachen und Frameworks bieten Möglichkeiten, mit Dateien und Ordnern zu arbeiten. Node.js ermöglicht dies mit dem `fs` oder Dateisystem-Modul.

Diese Lektion konzentriert sich auf das Node `fs`-Modul, wie es funktioniert und die drei verschiedenen Arten, wie Sie die von ihm bereitgestellten Methoden verwenden können.

Das Node `fs`-Modul stellt Ihnen Methoden zur Verfügung, um mit Dateien und Ordnern zu arbeiten, einschließlich Öffnen und Schließen, Lesen und Schreiben sowie Löschen.

Das `fs`-Modul ist ein Standardmodul von Node.js, daher steht es Ihnen zur Verfügung, solange Sie Node.js in Ihrer Umgebung installiert haben. Um das Modul zu verwenden, importieren Sie es so:

```js
const fs = require("fs");
```

Die Flexibilität, die das `fs`-Modul bietet, besteht darin, dass Sie seine Methoden sowohl synchron als auch asynchron verwenden können. Die `fs`-Modulmethoden sind standardmäßig asynchron, aber für jede Methode gibt es eine synchrone Form:

```js
fs.writeFile() // Asynchronous file writing
fs.writeFileSync() // Synchronous file writing

fs.readFile() // Asynchronous file reading
fs.readFileSync() // Synchronous file reading

fs.open() // Opens a file
fs.openAsBlob() // Opens as blob
fs.openSync() // Synchronous open
fs.opendir() // Opens directory
fs.opendirSync() // Synchronous directory open
```

Das ist noch nicht alles. Je nach Ihren Bedürfnissen können Sie die Methoden auf drei Arten verwenden:

- mit Callbacks, zum Beispiel `fs.writeFile()` (asynchron)
- mit Promises, wenn Sie die `async/await`-Syntax bevorzugen, zum Beispiel `fs.promises.writeFile()`
- synchron, zum Beispiel `fs.writeFileSync()`

Hier ist die grundlegende Syntax für die asynchrone Verwendung der Methoden:

```js
fs.writeFile("filePath", "content", "utf8", (err) => {
  if (err) {
    throw err;
  }
  console.log("File written to!");
});
```

Für die Promise-Version können Sie diese aus `fs/promises` importieren oder `promises` an Ihren `fs`-Import anhängen. Hier ist die Syntax:

```js
async function promisesExample() {
  try {
    await fs.promises.writeFile("filePath", "content", "utf8");
    console.log("File written to!");
  } catch (err) {
    console.error("Error:", err);
  }
}

promisesExample();
```

Und hier ist die synchrone Version:

```js
try {
  fs.writeFileSync("filePath", "content", "utf8");
  console.log("File written to!");
} catch (err) {
  console.error("Error:", err);
}
```

Beachten Sie, dass die Verwendung einer der Methoden synchron blockierend ist. Das bedeutet, Ihr Programm stoppt die Ausführung und wartet, bis die Operation abgeschlossen ist, bevor es mit der nächsten Codezeile fortfährt.

Für schnelle Skripte, kleine Projekte oder einmalige Aufgaben sind synchrone Methoden wahrscheinlich in Ordnung. Aber in realen Anwendungen wird blockierendes I/O (Eingabe/Ausgabe) zum Problem, weil es andere Teile Ihrer Anwendung einfrieren kann, während Node auf den Abschluss der Dateisystemoperationen wartet. Das verlangsamt alles und beeinträchtigt Leistung und Benutzererfahrung.

Deshalb gibt es asynchrone Methoden. Sie ermöglichen es dem Rest Ihres Programms, weiterzulaufen, während Node die Dateioperationen im Hintergrund erledigt. Traditionell wurden dafür Callbacks verwendet, aber die können schnell unübersichtlich werden, wenn Ihre Anwendung wächst.

Promises und `async/await` lösen dieses Problem. Sie sind weiterhin nicht blockierend, aber der Code liest sich wie normaler synchroner Code und ist viel leichter zu pflegen.

Aus diesem Grund wird heute meist der Promise-basierte Ansatz bevorzugt, und den werden wir im Rest dieser Lektion verwenden.

Die `writeFile()`-Methode ermöglicht es Ihnen, in eine bestehende Datei zu schreiben. Wenn die Datei nicht existiert, wird sie im aktuellen Verzeichnis erstellt und der angegebene Inhalt hineingeschrieben.

So verwenden Sie die `writeFile()`-Methode:

```js
const fs = require("fs/promises");

async function writeToFile() {
  try {
    await fs.writeFile(
      "article.md",
      "## Node `fs` Module: The Complete Guide",
      "utf8",
    );
    console.log("File written to!");
  } catch (err) {
    console.error("Error writing to file:", err);
  }
}

writeToFile(); // File written to!
```

Hier ist der Inhalt der Datei article.md danach:

```md
## Node `fs` Module: The Complete Guide
```

Die `appendFile()`-Methode ermöglicht es Ihnen, den Inhalt einer bestehenden Datei zu erweitern. So verwenden Sie sie:

```js
const fs = require("fs/promises");

async function appendToFile() {
  try {
    await fs.appendFile(
      "article.md",
      "\nIn this article, you will learn all there is to know about the Node fs module...",
      "utf8",
    );
    console.log("File appended to!");
  } catch (err) {
    console.log("Error appending to file:", err);
  }
}

appendToFile(); // File appended to!
```

Die `readFile()`-Methode zeigt Ihnen den Inhalt einer Datei an:

```js
const fs = require("fs/promises");

async function readFileContent() {
  try {
    const fileContent = await fs.readFile("article.md", "utf8");
    console.log("File content:", fileContent);
  } catch (err) {
    console.log("File read successfully");
  }
}

readFileContent();

/*
File content: ## Node `fs` Module: The Complete Guide

In this article, you will learn all there is to know about the Node fs module...
*/
```

Wenn Sie die `utf8`-Zeichenkodierung nicht angeben, erhalten Sie den Inhalt der Datei als Puffer:

```js
async function readFileContent() {
  try {
    const fileContent = await fs.readFile("article.md");
    console.log("File content:", fileContent);
  } catch (err) {
    console.log("File read successfully");
  }
}

readFileContent();

/*
File content: <Buffer 23 23 20 4e 6f 64 65 20 60 66 73 60 20 4c 69 62 72 61 72 79 3a 20
 54 68 65 20 43 6f 6d 70 6c 65 74 65 20 47 75 69 64 65 0a 0a 49 6e 20 74 68 69 73 20 ... 
72 more bytes>
*/
```

In der nächsten Lektion lernen Sie, wie Sie mit einem solchen Puffer mit dem Node.js Buffer-Modul umgehen.

Schließlich ermöglicht die `unlink()`-Methode das Löschen einer Datei:

```js
const fs = require("fs/promises");

async function deleteFile() {
  try {
    await fs.unlink("article.ts");
    console.log("File deleted successfully!");
  } catch (err) {
    console.error("Error deleting file:", err);
  }
}

deleteFile();
```

Wenn die Datei nicht existiert, erhalten Sie eine Fehlermeldung wie diese:

```bash
Error deleting file: Error: ENOENT: no such file or directory, unlink 'article.ts'
```

Wenn sie existiert, wird die Datei gelöscht und Sie erhalten die Erfolgsmeldung, die Sie festgelegt haben.
