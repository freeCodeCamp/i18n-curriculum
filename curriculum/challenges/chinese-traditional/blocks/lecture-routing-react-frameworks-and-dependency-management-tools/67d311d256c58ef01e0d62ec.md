# --description--

到目前為止，你一直在使用 React 來建置使用者介面。如果你需要像路由這樣的額外特性，那麼你必須匯入第三方函式庫，例如 React Router，才能在不同的視圖之間切換。

但是當你需要建置一個完整的堆疊網頁應用程式時會發生什麼？如果你只想使用 JavaScript，你可以為前端使用 React，並為後端邏輯使用 Node 和 Express。或者你也可以為後端使用其他語言，例如 Go、Python 或 Java。

雖然這些都是可行的選項，可是在某些時候你可能會想要使用 React 框架。React 框架提供像是路由、影像優化、資料擷取、身份驗證等特性。這表示在某些使用情境下，你可能不需要設定獨立的前端和後端應用程式。

讓我們更仔細地看看 Next.js，這是一個流行的 React 框架。Next.js 的主要特性之一是基於檔案系統的路由器。這個路由系統包括對動態路由、平行路由、路由處理常式、重新導向、國際化等的支援。

以下是建立訂製請求處理常式的範例：

```js
export async function GET() {
  const res = await fetch("https://example-api.com");
  const data = await res.json();

  return Response.json({ data });
}
```

你可以在 `app/api` 目錄內名為 `route.js` 的檔案中定義像 GET 或 POST 請求的路由處理常式。

Next.js 的另一個特性是自動的影像和字型優化。

這是一個在 `page.js` 檔案中使用 `Image` 組件的範例：

```jsx
import Image from "next/image";

export default function Page() {
  return (
    <Image src="link-to-image-goes-here" alt="descriptive-title-goes-here" />
  );
}
```

`Image` 組件擴充了原生的 HTML `img` 元素，並允許更快速的頁面載入和大小優化。這表示影像只有在進入視窗時才會載入，且 `Image` 組件會自動為每個裝置提供正確大小的影像。

雖然這些只是 Next.js 的部分特性，但你還可以使用更多特性來建置強固的全端網頁應用程式。還有其他像 Remix 這樣的 React 框架，也能提供相同的能力來建置現代化的全端網頁應用程式。

即使 JavaScript 函式庫和框架的生態系統不斷變化，了解你可用的選項以及學習每個框架的優缺點仍然很重要。
