# --interactive--

Давайте дізнаємося про призначення методу `preventDefault()` для подій.

Кожна подія, що відбувається в DOM, має певну поведінку за замовчуванням. Подія `click` на прапорці за замовчуванням змінює стан цього прапорця. Натискання пробілу на сфокусованій кнопці активує цю кнопку. Метод `preventDefault()` на цих об’єктах подій зупиняє цю поведінку.

Розглянемо приклад. Визначимо елемент `input`, у який користувач може вводити текст:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<label>Enter some characters:
  <input type="text">
</label>
```

```css
label {
  display: flex;
  flex-direction: column;
  font-family: Arial, sans-serif;
  font-size: 14px;
  color: #333;
  margin-bottom: 12px;
}

input[type="text"] {
  padding: 8px 12px;
  border: 1px solid #ccc;
  border-radius: 4px;
  font-size: 14px;
  margin-top: 4px;
  width: 100%;
  box-sizing: border-box;
  transition: border-color 0.2s, box-shadow 0.2s;
}

input[type="text"]:focus {
  border-color: #0078d4;
  box-shadow: 0 0 3px rgba(0, 120, 212, 0.5);
  outline: none;
}

```

:::

Якщо подивитися на результат, ми можемо вводити текст у поле введення, як і очікувалося. Але можливо, ми цього не хочемо. Можливо, натомість ми хочемо показувати символ, який вводить користувач, в окремому елементі. Спершу визначимо цей елемент:

:::interactive_editor

```html

  <input type="text">

```

```css
  color: #333;
  margin-bottom: 1rem;
  display: flex;
  flex-direction: column;

  font-size: 1rem;
  border: 2px solid #ccc;
  border-radius: 5px;
  margin-top: 0.5rem;
  outline: none;
}

  border-color: #007BFF;
}

  margin-top: 1rem;
  font-size: 1.1rem;
  color: #555;
}

```

Потім нам потрібно підписатися на подію `keydown`, щоб слухати, коли на клавіатурі вводиться символ. Зверніть увагу, що тут нам не потрібні події `change` або `input`, бо нам потрібна інформація саме з клавіатури.

```js
const input = document.querySelector("input");

input.addEventListener("keydown", (e) => {
    
})
```

Подія `keydown` спрацьовує, коли ви натискаєте клавішу на клавіатурі. Коли це відбувається, давайте відобразимо символ у нашому елементі `p`.


`e.key` дає вам значення натиснутої клавіші, наприклад `a` для клавіші `a` або `Enter` для клавіші `Enter`.

З наведеним кодом, коли ви вводите в `input`, символ, який ви вводите, відображатиметься в елементі `p`.

Це чудово, але ми не хочемо, щоб символи також відображалися в `input`. Саме тут нам допомагає метод `preventDefault()`. Поведінка за замовчуванням для `keydown` — відтворити символ у полі введення. Уникаємо цього, викликавши `e.preventDefault()`:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<label>Enter some characters:
  <input type="text">
</label>
<p id="output"></p>
<script src="index.js"></script>
```

```css
body {
  font-family: Arial, sans-serif;

  display: flex;
  flex-direction: column;
  align-items: center;
  justify-content: center;
  height: 100vh;
  margin: 0;
}

label {
  font-size: 1.2rem;
  color: #333;
  margin-bottom: 1rem;
  display: flex;
  flex-direction: column;
}

input[type="text"] {
  padding: 0.5rem;
  font-size: 1rem;
  border: 2px solid #ccc;
  border-radius: 5px;
  margin-top: 0.5rem;
  outline: none;
}

input[type="text"]:focus {
  border-color: #007BFF;
}

#output {
  margin-top: 1rem;
  font-size: 1.1rem;
  color: #555;
}

```

```js
const input = document.querySelector("input");
const output = document.getElementById("output");

input.addEventListener("keydown", (e) => {
  e.preventDefault();
  output.innerText = `You pressed the ${e.key} key`;
});
```

:::

І от так ви запобігаєте поведінці за замовчуванням, щоб реалізувати власну кастомну обробку подій.

Ще один поширений приклад використання методу `e.preventDefault` пов’язаний із відправленням форм. За замовчуванням відправлення форми надсилає дані на сервер і перезавантажує сторінку. Використання `e.preventDefault()` запобігає цьому.

```js
const form = document.querySelector("form");

form.addEventListener("submit", (e) => {
  e.preventDefault();
  // rest of code goes here
});
```

Запобігання поведінці за замовчуванням корисне, коли потрібен більший контроль над тим, як користувач взаємодіє зі сторінкою, але важливо пам’ятати про такі речі, як доступність — ваша кастомна поведінка має забезпечувати ті ж функції, що й поведінка за замовчуванням.
