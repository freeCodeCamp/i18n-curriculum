# --interactive--

دعنا نتعلم عن الغرض من طريقة `preventDefault()` على الأحداث.

كل حدث يحدث في DOM له سلوك افتراضي معين. حدث `click` على مربع اختيار يبدل حالة ذلك المربع، افتراضيًا. الضغط على شريط المسافة على زر مركّز يُفعّل الزر. طريقة `preventDefault()` على كائنات الأحداث هذه توقف حدوث ذلك السلوك.

لنلقِ نظرة على مثال. لنعرّف عنصر `input` ليكتب المستخدم فيه:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<label>Enter some characters:
  <input type="text">
</label>
```

```css
label {
  display: flex;
  flex-direction: column;
  font-family: Arial, sans-serif;
  font-size: 14px;
  color: #333;
  margin-bottom: 12px;
}

input[type="text"] {
  padding: 8px 12px;
  border: 1px solid #ccc;
  border-radius: 4px;
  font-size: 14px;
  margin-top: 4px;
  width: 100%;
  box-sizing: border-box;
  transition: border-color 0.2s, box-shadow 0.2s;
}

input[type="text"]:focus {
  border-color: #0078d4;
  box-shadow: 0 0 3px rgba(0, 120, 212, 0.5);
  outline: none;
}

```

:::

وإذا نظرنا إلى النتيجة، يمكننا الكتابة في حقل الإدخال كما هو متوقع. لكن ربما لا نريد ذلك. ربما نرغب بدلاً من ذلك في عرض الحرف الذي يكتبه المستخدم في عنصر منفصل. أولًا، لنعرّف عنصرنا لذلك:

ثم، نحتاج إلى الربط بحدث `keydown` للاستماع إلى كتابة حرف على لوحة المفاتيح. لاحظ أننا لا نريد أحداث `change` أو `input` هنا، لأننا نحتاج إلى معلومات لوحة المفاتيح.

```js
const input = document.querySelector("input");

input.addEventListener("keydown", (e) => {
    
})
```

يحدث حدث `keydown` عند الضغط على مفتاح في لوحة المفاتيح. عندما يحدث ذلك، دعنا نعرض الحرف في عنصر `p` الخاص بنا.

```js

  output.innerText = `You pressed the ${e.key} key`;

```

`e.key` يعطيك قيمة المفتاح المضغوط، مثل `a` لمفتاح `a` أو `Enter` لمفتاح `Enter`.

مع الشفرة أعلاه، عندما تكتب في `input`، سيُعرض الحرف الذي تكتبه في عنصر `p`.

هذا رائع، لكننا لا نريد عرض الأحرف في `input` أيضًا. هنا تأتي طريقة `preventDefault()` الخاصة بنا. السلوك الافتراضي لحدث `keydown` هو عرض الحرف في حقل الإدخال. دعنا نتجنب ذلك باستدعاء `e.preventDefault()`:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<label>Enter some characters:
  <input type="text">
</label>
<p id="output"></p>
<script src="index.js"></script>
```

```css
body {
  font-family: Arial, sans-serif;
  background-color: #f0f0f0;
  display: flex;
  flex-direction: column;
  align-items: center;
  justify-content: center;
  height: 100vh;
  margin: 0;
}

label {
  font-size: 1.2rem;
  color: #333;
  margin-bottom: 1rem;
  display: flex;
  flex-direction: column;
}

input[type="text"] {
  padding: 0.5rem;
  font-size: 1rem;
  border: 2px solid #ccc;
  border-radius: 5px;
  margin-top: 0.5rem;
  outline: none;
}

input[type="text"]:focus {
  border-color: #007BFF;
}

#output {
  margin-top: 1rem;
  font-size: 1.1rem;
  color: #555;
}

```

```js
const input = document.querySelector("input");
const output = document.getElementById("output");

input.addEventListener("keydown", (e) => {
  e.preventDefault();
  output.innerText = `You pressed the ${e.key} key`;
});
```

:::

وهكذا، تكون قد منعت السلوك الافتراضي لتتمكن من تنفيذ معالجة أحداث مخصصة خاصة بك.

مثال شائع آخر لاستخدام طريقة `e.preventDefault()` يتعلق بإرسال النماذج. افتراضيًا، يؤدي إرسال النموذج إلى إرسال البيانات إلى الخادم وإعادة تحميل الصفحة. استخدام `e.preventDefault()` يمنع حدوث ذلك.

```js
const form = document.querySelector("form");

form.addEventListener("submit", (e) => {
  e.preventDefault();
  // rest of code goes here
});
```

منع السلوك الافتراضي مفيد عندما تحتاج إلى مزيد من التحكم في كيفية تفاعل المستخدم مع الصفحة، لكن من المهم مراعاة أمور مثل إمكانية الوصول: يجب أن يوفر سلوكك المخصص نفس الميزات التي يوفرها السلوك الافتراضي.
