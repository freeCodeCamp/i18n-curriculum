# --description--

Werfen wir einen Blick auf die Vorteile, gute Linktexte im Kontext der Barrierefreiheit zu schreiben, und einige Beispiele für gute Linktexte.

Der erste sichtbare Vorteil eines guten Linktexts ist, dass er es allen erleichtert, Informationen schnell zu finden. Beschreibende Links helfen Nutzenden zu wissen, wohin sie gelangen und was sie aufrufen werden. Das sorgt dafür, dass sich die Nutzenden nicht verloren fühlen und verbessert die gesamte Nutzererfahrung.

Für diejenigen, die Screenreader verwenden, ist ein klarer und beschreibender Linktext ein Muss. Screenreader lesen den Linktext laut vor, daher ist ein Text wie „Read our accessibility guide“ viel besser als „Click here“.

Klare und beschreibende Linktexte zu erstellen, ist nicht nur für Menschen mit Sehbeeinträchtigungen vorteilhaft. Beschreibende Links helfen auch Menschen mit kognitiven Beeinträchtigungen, indem sie klaren Kontext bieten.

Hier sind einige bewährte Vorgehensweisen, die Sie beim Schreiben von Linktexten beachten sollten:

- Stellen Sie sicher, dass Links visuell deutlich erkennbar sind, indem Sie Unterstreichungen und andere visuelle Hinweise verwenden, damit Nutzende sie leicht identifizieren und navigieren können.
- Vermeiden Sie generische Linktexte wie „here“, „click here“ und „more-info“, da sie keine nützlichen Informationen liefern.
- Streben Sie kurze und beschreibende Linktexte an, idealerweise zwischen 2 und 5 Wörtern, die den Zweck des Links vermitteln.
- Vermeiden Sie Fachjargon und Abkürzungen, die Nutzende möglicherweise nicht verstehen.
- Konzentrieren Sie sich auf das Ziel, nicht auf die Aktion. Zum Beispiel „user behavior results“ statt „click here to read more“.
- Verwenden Sie nicht denselben Linktext für verschiedene Ziele.
- Platzieren Sie Links so, dass sie im umgebenden Text sinnvoll sind. Zum Beispiel „for more details, visit our events page“ statt „Click here for more“.

Hier sind einige Beispiele für gute Linktexte für bestimmte Anwendungsfälle im Vergleich zu weniger hilfreichen.
Angenommen, Sie möchten auf eine Seite verlinken, die Details zu einer Veranstaltung wie dieser bereitstellt:

```html
<a href="webinar-details-link">Details</a>
```

`Details` ist vage und liefert keine spezifischen Informationen darüber, was die Nutzenden finden, wenn sie auf den Link klicken. Ohne zusätzlichen Kontext könnten Nutzende nicht wissen, ob der Link zu Details über ein Webinar, ein Produkt, eine Richtlinie oder etwas anderes führt.

Hier ist ein Beispiel für einen guten Linktext:

```html
<a href="webinar-details-link">
  Get details about our upcoming webinar
</a>
```

Dieser Linktext gibt den Nutzenden Kontext über den Inhalt, den sie finden werden, und erleichtert die Entscheidung, ob sie darauf klicken möchten. Er reduziert Unklarheiten, indem er angibt, dass der Link mit einem Webinar zu tun hat.

Hier ist ein weiteres Beispiel, das auf einen Beitrag in einem Blog verlinkt:

```html
<a href="/blog-post-link">Read more</a>
```

Der Linktext `Read more` ist in Bezug auf Barrierefreiheit nicht ideal, da er keinen Kontext bietet.

Hier ist ein besseres Beispiel für einen Linktext:

```html
<a href="/blog-post-link">
   Read our latest blog post on web accessibility
</a>
```

Dieser aktualisierte Linktext gibt den Nutzenden eine klarere Vorstellung davon, was sie erwartet und warum sie auf den Link klicken sollten, was besonders für diejenigen wichtig ist, die Screenreader verwenden.

Werfen wir einen weiteren Blick auf ein Beispiel. Angenommen, Sie möchten mehr Informationen zu einem bestimmten Thema bereitstellen:

```html
<a href="/link-to-topic">More info</a>
```

`More info` ist ebenfalls vage und kann für die Barrierefreiheit problematisch sein. Er liefert keine spezifischen Details über das Ziel des Links, was für Nutzende, die auf Screenreader angewiesen sind oder die Seite nicht linear navigieren, verwirrend sein kann.

Hier ist ein besseres Beispiel für einen Linktext:

```html
<a href="/link-to-topic">
   Learn more about our accessibility efforts
</a>
```

Dieser aktualisierte Linktext ist besser, weil er klar beschreibt, was die Nutzenden finden, wenn sie auf den Link klicken. Nutzende können sofort verstehen, dass der Link zu Informationen über Ihre Barrierefreiheitsbemühungen führt.
