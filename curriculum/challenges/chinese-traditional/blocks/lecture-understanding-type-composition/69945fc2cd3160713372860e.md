# --description--

在前一課中，你學會了如何使用聯集（數學）型別。使用聯集（數學）型別的一種常見方法是使用型別別名。 

型別別名是型別的名稱。以下是為聯集（數學）型別提供名稱的範例：

```ts
type ID = number | string;
```

在你的專案中，你經常會創建型別別名並在整個程式中像這樣使用它們：

```ts
type ID = number | string;

type User = {
  id: ID;
  name: string;
};

function getUserId(user: User): ID {
  return user.id;
}
```

此範例使用兩個名為 `ID` 和 `User` 的型別。`ID` 將會是 `number` 或 `string` 的聯集型別，而 `User` 是具有 `id` 和 `name` 屬性的物件型別。`id` 指定為 `ID` 型別，而 `name` 則是 `string` 型別。 

`getUserId` 函式（程式）有一個名為 `user` 的參數，其型別為 `User`。`getUserId` 函式（程式）的返回類型為 `ID`。 

型別別名是描述物件結構的絕佳方式。通常會創建物件型別並像這樣在陣列中使用它們：

```ts
type User = {
  id: number | string;
  name: string;
};

const users: User[] = [
  { id: 101, name: "Alice" },
  { id: "A102", name: "Bob" },
  { id: 103, name: "Charlie" }
];
```

在此範例中，`users` 陣列必須只包含 `User` 物件。如果你忘記在其中一個物件中填寫某個屬性，TypeScript 就會發出類似這樣的錯誤：

```ts
type User = {
  id: number | string;
  name: string;
};

const users: User[] = [
  // Property 'name' is missing in type '{ id: number; }' but required in type 'User'.
  { id: 101 },
  { id: "A102", name: "Bob" },
  { id: 103, name: "Charlie" }
];
```

你可能已經注意到在命名型別別名時使用了 PascalCase。這是因為慣例上使用 PascalCase 有助於區分型別和變數，也有助於可讀性。當你看到 `User[]` 時，你會知道它是一個 `User` 型別物件的陣列。
