# --interactive--

La regola `@property` è una potente funzionalità CSS che permette agli sviluppatori di definire proprietà personalizzate con un controllo maggiore sul loro comportamento, incluso come animarle e i loro valori iniziali.

Questa regola offre un modo per migliorare la funzionalità delle proprietà personalizzate CSS e garantisce più flessibilità nel loro utilizzo.

La sintassi base della regola `@property` è la seguente:

```css
@property --property-name {
  syntax: '<type>';
  inherits: true | false;
  initial-value: <value>;
}
```

`--property-name` è il nome della proprietà personalizzata che stai definendo. Come tutte le proprietà personalizzate, deve iniziare con due trattini.

`syntax` definisce il tipo della proprietà, che può essere ad esempio `<color>`, `<length>`, `<number>`, `<percentage>` o tipi più complessi.

`inherits` specifica se la proprietà deve ereditare il valore dal suo elemento genitore.

`initial-value` imposta il valore predefinito della proprietà.

Ecco un esempio pratico di utilizzo della regola `@property`:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<button class="button">Click Me</button>
```

```css
@property --main-color {
  syntax: '<color>';
  inherits: false;
  initial-value: #3498db;
}

.button {
  background-color: var(--main-color);
}
```

:::

In questo esempio, definiamo una proprietà personalizzata `--main-color` come un valore `color`, impostiamo che non erediti e le assegniamo un valore iniziale di `#3498db`.

Uno dei vantaggi principali della regola `@property` è che permette di animare le proprietà personalizzate, cosa che prima non era possibile:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<div class="gradient-box"></div>
```

```css
@property --gradient-angle {
  syntax: '<angle>';
  inherits: false;
  initial-value: 0deg;
}

.gradient-box {
  width: 100px;
  height: 100px;
  background: linear-gradient(var(--gradient-angle), red, blue);
  transition: --gradient-angle 0.5s;
}

.gradient-box:hover {
  --gradient-angle: 90deg;
}
```

:::

Il codice sopra crea una sfumatura che si anima dolcemente quando l’elemento viene passato con il mouse, qualcosa che non si poteva ottenere con le proprietà personalizzate standard.

Ora, parliamo di come la regola `@property` funziona con i fallback.

I fallback sono fondamentali in CSS per garantire che gli stili degradino in modo elegante nei browser che non supportano certe funzionalità. Con `@property`, i fallback funzionano su due livelli: per la regola stessa e per l’uso della proprietà personalizzata.

Per i browser che non supportano la regola `@property`, puoi fornire un fallback dichiarando la proprietà personalizzata nel modo tradizionale:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
```

```css
:root {
  --main-color: #3498db;
}

@property --main-color {
  syntax: '<color>';
  inherits: false;
  initial-value: #3498db;
}

body {
  background-color: var(--main-color);
}
```

:::

In questo caso, i browser che supportano `@property` useranno la versione più rigorosamente definita, mentre gli altri ricadranno sulla dichiarazione standard della proprietà personalizzata.

Quando usi la proprietà personalizzata, puoi fornire un valore di fallback usando la funzione `var()`, proprio come faresti con le proprietà personalizzate standard:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<button class="button">Click Me</button>
```

```css
.button {
  background-color: var(--main-color, #3498db);
}
```

:::

Questo garantisce che anche se la proprietà `--main-color` non è definita o è invalida, il pulsante avrà comunque un colore di sfondo.

La regola `@property` permette anche scenari di fallback più complessi. Per esempio, puoi usarla per fornire fallback sicuri per tipo:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<div class="box">Colored box</div>
```

```css
@property --padding {
  syntax: '<length>';
  inherits: false;
  initial-value: 10px;
}

.box {
  width: 100px;
  height: 100px;
  background-color: darkred;
  color: white;
  padding: var(--padding);
}
```

:::

In questo caso, se qualcuno prova a impostare `--padding` su un valore invalido, come un colore, il browser ricadrà sul valore iniziale di `10px`, mantenendo la sicurezza del tipo.

La regola CSS `@property` è considerata ampiamente disponibile su tutti i principali motori browser moderni (come Chrome, Edge, Opera, Firefox e Safari) dalla prima metà del 2024. Se vuoi supportare browser più vecchi devi fornire CSS fallback come spiegato sopra.

In conclusione, la regola `@property` rappresenta un importante passo avanti nel modo in cui lavoriamo con le proprietà personalizzate in CSS. Offre più controllo e abilita nuove possibilità come l’animazione delle proprietà personalizzate. Tuttavia, è importante usarla con giudizio e fornire sempre fallback appropriati per garantire una buona esperienza su tutti i browser. Come per ogni tecnologia web all’avanguardia, la chiave è migliorare l’esperienza per i browser che la supportano assicurando al contempo la funzionalità base per quelli che non la supportano.
