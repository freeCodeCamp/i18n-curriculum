# --description--

React에서 상태에 있는 객체를 업데이트하는 것은 객체 속성 값을 직접 변경하는 데 익숙하다면 까다로울 수 있습니다.

React는 상태를 불변으로 취급하므로 직접 수정하면 안 됩니다.

React 상태의 객체를 직접 변경하려고 하면 어떤 일이 일어나는지 살펴보고, 올바른 방법을 알아봅시다.

컴포넌트 상태에 사용자 프로필을 나타내는 객체가 있고, 사용자가 나이를 업데이트하고 싶다고 가정해 보겠습니다:

```jsx
import { useState } from "react";

function Profile() {
  const [user, setUser] = useState({
    name: "John Doe",
    age: 31,
    city: "LA",
  });

  // Change user age directly
  const handleAgeChange = (e) => {
    user.age = e.target.value;
    console.log(user);
  };

  return (
    <div>
      <h1>User Profile</h1>
      <p>Name: {user.name}</p>
      <p>Age: {user.age}</p>
      <p>City: {user.city}</p>

      <h2>Update User Age </h2>
      <input type="number" value={user.age} onChange={handleAgeChange} />
    </div>
  );
}

export default Profile;
```

이 코드는 `user.age` 속성을 직접 수정하기 때문에 작동하지 않습니다.

콘솔에서는 `console.log(user)`이 새 나이를 보여주지만, setter 함수인 `setUser`를 사용하지 않았기 때문에 React가 컴포넌트를 다시 렌더링하지 않아 사용자 인터페이스에 반영되지 않습니다.

상태의 객체를 직접 업데이트하려면 setter 함수를 사용해 업데이트된 값이 포함된 새 객체를 만들어야 합니다. 예를 들어:

```js
const handleAgeChange = (e) => {
  setUser({
    age: e.target.value,
  });
};
```

이 방법은 작동합니다. 하지만 페이지를 보면 사용자의 나이는 업데이트되지만 이름과 도시 값은 사라집니다.

이는 setter 함수에 전달한 새 객체가 `age`에 대한 키/값 쌍만 포함했기 때문입니다.

이런 일이 발생하지 않도록 기존 객체를 먼저 복사한 다음, 업데이트하려는 속성만 변경할 수 있습니다. 여기서는 `age`입니다.

이를 위해 setter 함수인 `setUser`에 업데이트 함수라는 특별한 함수를 전달할 수 있습니다. 업데이트 함수는 대기 중인 상태를 인수로 받아 여기서는 `prevUser`라고 하며, 다음 상태를 반환해야 합니다:

```js
const handleAgeChange = (e) => {
  setUser((prevUser) => {
    const updatedUser = { ...prevUser, age: e.target.value };
    console.log('Previous State:', prevUser);
    console.log('Updated State:', updatedUser);
    return updatedUser;
  });
};
```

보시다시피, 전개 구문을 사용해 대기 중인 사용자 객체인 `updatedUser`를 복사하여 `...prevUser`라는 새 사용자 객체를 만듭니다. 그런 다음 폼 입력을 기반으로 나이를 업데이트하고 함수 하단에서 `updatedUser`를 다음 상태로 반환합니다.

이제 프로젝트가 예상대로 작동하며, 나이 입력 업데이트가 사용자의 이름이나 도시 이름에 영향을 주지 않습니다. 콘솔에서 이전 상태와 업데이트된 상태도 확인할 수 있습니다.

이것이 상태에서 객체를 업데이트하는 이상적인 방법이며, 특히 모든 속성을 업데이트하지 않을 때 유용합니다.

나머지 `name`과 `city` 속성을 업데이트하려면 각각 별도의 setter 함수를 작성해 해당 입력과 연결할 수 있습니다:

```js
const handleNameChange = (e) => {
  setUser((prevUser) => ({
    ...prevUser,
    name: e.target.value,
  }));
};

const handleCityChange = (e) => {
  setUser((prevUser) => ({
    ...prevUser,
    city: e.target.value,
  }));
};
```

또는 다음과 같이 하나의 setter 함수로 합칠 수도 있습니다:

```js
const handleChange = (e) => {
  const { name, value } = e.target;
  setUser((prevUser) => ({
    ...prevUser,
    [name]: value,
  }));
};
```

이 방법을 사용하려면 각 입력 필드에 `name` 속성이 있어야 합니다.

전체 코드는 다음과 같습니다:

```jsx
import { useState } from "react";

function Profile() {
  const [user, setUser] = useState({ name: "John Doe", age: 31, city: "LA" });

  const handleChange = (e) => {
    const { name, value } = e.target;

    setUser((prevUser) => ({...prevUser, [name]: value}));
  };

  return (
    <div>
      <h1>User Profile</h1>
      <p>Name: {user.name}</p>
      <p>Age: {user.age}</p>
      <p>City: {user.city}</p>

      <h2>Update User Age </h2>
      <input type="number" name="age" value={user.age} onChange={handleChange} />

      <h2>Update User Name </h2>
      <input type="text" name="name" value={user.name} onChange={handleChange} />

      <h2>Update User City </h2>
      <input type="text" name="city" value={user.city} onChange={handleChange} />
    </div>
  );
}

export default Profile;
```
