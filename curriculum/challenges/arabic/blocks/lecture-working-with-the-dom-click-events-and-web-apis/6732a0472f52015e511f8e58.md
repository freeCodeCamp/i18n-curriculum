# --description--

دعنا نتعلم عن APIs ولماذا هي ضرورية لتطوير الويب. API تعني واجهة برمجة التطبيقات. تحدد APIs مجموعة من القواعد والبروتوكولات التي تسمح لتطبيقات البرمجيات بالتواصل مع بعضها البعض وتبادل البيانات بكفاءة.

يمكنك التفكير فيها كتركيبات تسمح للمطوّرين بإنشاء وظائف أكثر تعقيدًا بناءً على وحدات بناء أبسط تم تنفيذها بالفعل.

هناك أنواع مختلفة من APIs. تم تصميم Web APIs خصيصًا لتطبيقات الويب. هناك أنواع مختلفة من Web APIs لتطوير JavaScript على جانب العميل. وهي ليست جزءًا من JavaScript نفسها.

غالبًا ما تُقسم هذه الأنواع من APIs إلى فئتين رئيسيتين: Browser APIs وThird-party APIs.

تُتيح Browser APIs البيانات من المتصفح. كمطوّر ويب، يمكنك الوصول إلى هذه البيانات والتعامل معها باستخدام JavaScript.

توفر أيضًا وصولًا إلى وظائف مختلفة، مثل تعديل هيكل الموقع، والتعامل مع الأحداث، والعمل مع التخزين، والتواصل مع الشبكة.

بعض الأمثلة على Browser APIs المستخدمة بشكل شائع تشمل:

DOM API، التي يمكنك استخدامها للتعامل مع عناصر HTML وأنماطها وخصائصها. ستتعلم المزيد عن DOM في الدروس القادمة. إنه مفهوم أساسي في تطوير الويب.

Storage API، لتخزين البيانات محليًا على جهاز المستخدم.

والمزيد.

هناك العديد من Browser APIs التي يمكنك استخدامها للتعامل مع تفاعلات المستخدم وللحصول على مزيد من المعلومات من المتصفح، مثل الموقع الحالي للمستخدم.

يمكنك أيضًا العمل مع أنواع أخرى من العناصر، مثل الفيديو والصوت.

تُعد Browser APIs أدوات مفيدة لبناء تطبيقات ويب قوية. فهي مدمجة بالفعل في المتصفح.

لكن يمكنك أيضًا استخدام Third-party APIs، التي لا تكون مدمجة في المتصفح افتراضيًا. لاستخدامها، تحتاج إلى استدعاء شفرتها بطريقة ما. عادةً ما تأتي مع توثيق مفصل يشرح كيفية استخدام خدماتها.

مثال على ذلك هو Google Maps API، الذي يمكنك استخدامه لعرض خرائط تفاعلية على موقعك.

هناك أنواع مختلفة من Third-party Web APIs، بما في ذلك APIs الطقس، وAPIs وسائل التواصل الاجتماعي، وAPIs الدفع، وAPIs البيانات، وAPIs الترجمة، والمزيد. فقط عليك إيجاد API التي تناسب احتياجاتك.

تُعد Web APIs أدوات أساسية لبناء تطبيقات ويب حديثة يمكنها التفاعل مع خدمات ومصادر بيانات مختلفة. بتعلم كيفية الاستفادة من قوتها، ستوسع مجموعة أدواتك لإنشاء مواقع ديناميكية وجذابة.
