# --description--

Kurudi kwenye sehemu ya usalama wa habari, unaweza kukumbuka kuwa kuhifadhi nywila za maandishi wazi *sio sahihi kamwe*. Sasa ni wakati wa kutekeleza BCrypt kutatua tatizo hili.

`bcrypt@~5.0.0` tayari imeongezwa kama utegemezi, hivyo itoe kwenye seva yako. Utahitaji kushughulikia kuchanganya nywila katika maeneo mawili muhimu: pale unapojiandikisha/kuhifadhi akaunti mpya, na wakati unapoangalia kama nywila ni sahihi wakati wa kuingia.

Kwa sasa kwenye njia yako ya usajili, unaingiza nywila ya mtumizi kama maandishi wazi kwenye hifadhidata kama ifuatavyo: `password: req.body.password`. Badala yake, changanya nywila kwa kuongeza yafuatayo kabla ya mantiki ya hifadhidata: `const hash = bcrypt.hashSync(req.body.password, 12);`, na badilisha `req.body.password` katika kuhifadhi hifadhidata na `password: hash` tu.

Katika mkakati wako wa uthibitisho wa utambulisho, unakagua yafuatayo katika msimbo wako kabla ya kumaliza mchakato: `if (password !== user.password) return done(null, false);`. Baada ya kufanya mabadiliko ya awali, sasa `user.password` ni mchanganyiko. Kabla ya kubadilisha msimbo uliopo, angalia jinsi kauli hiyo inavyokagua kama nywila **si** sawa kisha rudisha hali ya kutothibitishwa. Kwa kuzingatia hili, badilisha msimbo huo uonekane kama ifuatavyo ili kukagua ipasavyo nywila iliyowekwa dhidi ya mchanganyiko:

```js
if (!bcrypt.compareSync(password, user.password)) { 
  return done(null, false);
}
```

Hiyo ndiyo yote inahitajika kutekeleza mojawapo ya vipengele muhimu vya usalama unapo-hifadhi nywila.

Wasilisha ukurasa wako unapoona umefanya vizuri. Ikiwa unakutana na makosa, unaweza <a href="https://forum.freecodecamp.org/t/advanced-node-and-express/567135#hashing-your-passwords-1" target="_blank" rel="noopener noreferrer nofollow">kuangalia mradi uliofanyika hadi sasa</a>.
