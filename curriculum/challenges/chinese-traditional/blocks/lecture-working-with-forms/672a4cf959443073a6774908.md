# --interactive--

在 HTML 中，表單控制元件，例如輸入欄位，可以處於不同階段或狀態，例如 `focused` 狀態、`readonly` 狀態或 `disabled` 狀態。

第一個狀態會被視為 `default` 狀態。電子郵件位址輸入欄位的預設狀態是空白輸入。這就是電子郵件輸入欄位在頁面上第一次呈現時的樣子。 

:::interactive_editor

```html
<input type="email" name="email" id="email" />
```

:::

當使用者點擊表單控制元件或使用鍵盤的 tab 鍵選取它時，表示該控制元件處於 `focused` 狀態。當輸入欄位處於 `focused` 狀態時，大多數瀏覽器會在輸入欄位周圍顯示藍色的凸顯標示邊框。但你可以選擇在 CSS 中添加額外的樣式。

點擊預覽視窗中空白區域的任何零件，然後按下 `tab` 鍵以查看焦點狀態。若要查看預覽，你需要啟用互動式編輯器。

:::interactive_editor

```html
<input type="email" name="email" id="email" />
```

:::

另一種表單狀態是 `disabled` 狀態。此狀態向使用者顯示該輸入無法聚焦或作用。 

啟用互動式編輯器並嘗試點擊電子郵件輸入欄，你會注意到它將不再獲得焦點。

:::interactive_editor

```html
<input disabled type="email" name="email" id="email" />
```

:::

類似於 `focused` 狀態，你可以選擇為 `disabled` 狀態使用 CSS 添加額外的樣式。

另一種表單狀態是 `readonly` 狀態。當表單控制元件，例如輸入欄，無法被使用者編輯時，即為此狀態。以下是將電子郵件輸入欄設為唯讀的範例。`value` 屬性用於設定顯示在輸入欄內的值。

啟用互動式編輯器並嘗試在預覽視窗中編輯 `example@email.com` 的當前值，你會注意到這是不可能的。

:::interactive_editor

```html
<input
  readonly
  type="email"
  name="email"
  id="email"
  value="example@email.com"
/>
```

:::

`disabled` 狀態和 `readonly` 狀態之間的一個主要差異是 `readonly` 可以被聚焦，而 `disabled` 狀態則不行。

了解不同的表單狀態很重要，因為它們透過提供清晰的回授和指引來確保順暢的使用者體驗，同時處理錯誤。
