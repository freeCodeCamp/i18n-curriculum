# --instructions--

У цьому завданні ви створите алгоритм рекомендації книг за допомогою **K-найближчих сусідів**.

У цьому проєкті ви використаєте набір даних Book-Crossings, який містить 1,1 мільйона оцінок (за шкалою від 1 до 10) для 270 000 книг від 90 000 користувачів. Набір даних уже імпортовано в блокнот, тому додаткове скачування не потрібне.

Використайте `NearestNeighbors` з `sklearn.neighbors` для розробки моделі, яка показує книги, схожі на задану книгу. Алгоритм найближчих сусідів вимірює відстань, щоб визначити «близькість» екземплярів.

Створіть функцію з назвою `get_recommends`, яка приймає назву книги (з набору даних) як аргумент і повертає список із 5 схожих книг із відстанями від книги-аргументу.

Цей код:

```py
get_recommends("The Queen of the Damned (Vampire Chronicles (Paperback))")
```

повинен повернути:

```py
[
  'The Queen of the Damned (Vampire Chronicles (Paperback))',
  [
    ['Catch 22', 0.793983519077301], 
    ['The Witching Hour (Lives of the Mayfair Witches)', 0.7448656558990479], 
    ['Interview with the Vampire', 0.7345068454742432],
    ['The Tale of the Body Thief (Vampire Chronicles (Paperback))', 0.5376338362693787],
    ['The Vampire Lestat (Vampire Chronicles, Book II)', 0.5178412199020386]
  ]
]
```

Зверніть увагу, що дані, які повертає `get_recommends()`, — це список. Перший елемент списку — це назва книги, переданої у функцію. Другий елемент списку — це список із п’яти інших списків. Кожен із цих п’яти списків містить рекомендовану книгу та відстань від рекомендованої книги до книги, переданої у функцію.

Якщо ви побудуєте графік набору даних (необов’язково), помітите, що більшість книг оцінюють нечасто. Щоб забезпечити статистичну значущість, видаліть із набору даних користувачів із менше ніж 200 оцінками та книги з менше ніж 100 оцінками.

Перші три клітинки імпортують бібліотеки, які можуть знадобитися, та дані для використання. Остання клітинка призначена для тестування. Пишіть увесь свій код між цими клітинками.
