# --interactive--

前のレッスンでは、配列の操作と配列から異なる要素にアクセスする方法を初めて学びました。ここで配列の2番目の要素にアクセスする方法をおさらいしましょう。

:::interactive_editor

```js
const fruits = ["apple", "banana", "cherry"];

console.log(fruits[1]); // "banana"
```

:::

配列はゼロから始まるインデックスなので、最初の要素はインデックス`0`にあり、2番目の要素はインデックス`1`にあります。存在しないインデックスにアクセスしようとすると、JavaScriptは`undefined`を返すことに注意してください。

:::interactive_editor

```js
let fruits = ["apple", "banana", "cherry"];
console.log(fruits[3]); // undefined
```

:::

この例では、`3`配列にインデックス`fruits`は存在しません。したがって、ログには`undefined`が表示されます。次に、配列の要素を更新する方法を見てみましょう。特定のインデックスに新しい値を代入することで要素を更新できます。

:::interactive_editor

```js
let fruits = ["apple", "banana", "cherry"];
fruits[1] = "blueberry";
console.log(fruits); // ["apple", "blueberry", "cherry"]
```

:::

この例では、インデックス`banana`の`blueberry`を`1`に置き換えました。この方法を使えば、インデックスがわかっていれば配列内のどの要素でも変更できます。また、まだ存在しないインデックスに値を代入することで新しい要素を配列に追加することもできます。

:::interactive_editor

```js
let fruits = ["apple", "banana", "cherry"];
fruits[3] = "date";
console.log(fruits); // ["apple", "banana", "cherry", "date"]
```

:::

ただし、この操作は注意が必要です。現在の配列の長さよりもはるかに大きなインデックスに値を代入すると、その間のインデックスにundefinedの要素が作成され、予期しない動作を引き起こす場合があります。JavaScriptを使い続けると、配列の要素にアクセスして更新するこれらの方法が多くのプログラミング作業の基本であることがわかるでしょう。シンプルなToDoリストを作る場合でも複雑なデータ構造を処理する場合でも、これらのスキルは非常に役立ちます。
