# --description--

웹 스토리지 API는 브라우저 내에서 키-값 쌍을 저장할 수 있는 메커니즘을 제공하여, 개발자가 페이지 새로 고침과 세션 간에 사용할 수 있는 정보를 저장할 수 있게 합니다.

웹 스토리지 API의 두 가지 주요 컴포넌트는 `localStorage`와 `sessionStorage`입니다.

`localStorage`는 브라우저 창이 닫히거나 페이지가 새로 고침된 후에도 데이터가 유지되도록 하는 웹 스토리지 API의 부분입니다. 이 데이터는 애플리케이션이나 사용자가 명시적으로 제거할 때까지 계속 사용할 수 있습니다.

`sessionStorage`는 웹 스토리지 API의 또 다른 부분으로, 페이지 세션 동안 데이터를 저장합니다. 즉, 브라우저 탭이나 창이 열려 있는 동안 데이터가 유지됩니다. 하지만 `localStorage`와 달리 `sessionStorage`의 데이터는 탭이나 창이 닫히면 지워집니다. `sessionStorage`에 대해서는 다음 수업에서 더 배우게 됩니다.

`localStorage`의 일반적인 사용 사례에는 테마나 언어 설정 같은 사용자 설정 저장, 브라우저 세션 간 폼 데이터 기억, 웹 애플리케이션 성능 향상을 위한 작은 정보 캐싱 등이 포함됩니다.

캐싱은 자주 접근하는 데이터를 캐시라는 임시 저장소에 저장하여, 이후 해당 데이터 요청 시 데이터베이스나 외부 서버 같은 느린 데이터 소스에서 다시 계산하거나 가져오지 않고 더 빠르게 제공할 수 있게 하는 것을 의미합니다.

일부 자주 사용하는 `localStorage` 메서드에는 `setItem`, `getItem`, `removeItem` 그리고 `clear` 메서드가 있습니다.

다음은 `setItem()` 메서드를 사용해 `localStorage`에 키-값 쌍을 저장하는 예시입니다.

```js
localStorage.setItem('username', 'JaneDoe');
```

그런 다음 `localStorage`에서 주어진 키의 값을 가져오려면 `getItem()` 메서드를 사용할 수 있습니다.

```js
let username = localStorage.getItem('username');
console.log(username); // JaneDoe
```

키를 사용해 `localStorage`에서 항목을 제거하려면 `removeItem()` 메서드를 사용하세요.

```js
localStorage.removeItem('username');
```

`localStorage`의 모든 데이터를 지우려면 `clear()` 메서드를 사용하세요.

```js
localStorage.clear();
```

이제 `localStorage`을 사용해 사용자의 선호 테마를 저장하는 예시를 살펴보겠습니다.

```js
// Store the user's theme preference
localStorage.setItem('theme', 'dark');

// Retrieve the stored theme preference
const userTheme = localStorage.getItem('theme');
console.log(userTheme); // 'dark'

// Remove the theme preference
localStorage.removeItem('theme');

// Clear all localStorage data
localStorage.clear();
```

이 예시에서는:

- 먼저 사용자를 위해 테마 선택(`dark`)을 저장합니다.

- 그런 다음 그 테마를 가져와 콘솔에 출력합니다.

- 마지막으로 특정 항목을 제거하거나 저장된 모든 데이터를 지우는 방법을 보여줍니다.

`localStorage`는 세션 간에 유지해야 하는 작은 데이터를 저장하는 데 매우 유용하지만, 비밀번호 같은 민감한 정보를 저장하는 데는 보안 위험이 있으므로 `localStorage`를 사용하지 않는 것이 중요합니다.
