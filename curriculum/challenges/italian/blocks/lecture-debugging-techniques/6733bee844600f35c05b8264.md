# --interactive--

Nella lezione precedente, hai imparato come lanciare eccezioni nei tuoi programmi. In questa lezione, vedremo come gestire con eleganza questi errori in un blocco `try…catch…finally`.

Il blocco `try` serve a racchiudere codice che potrebbe generare un errore. Funziona come uno spazio sicuro per provare qualcosa che potrebbe fallire.

Il blocco `catch` cattura e gestisce gli errori che si verificano nel blocco `try`. Puoi usare l’oggetto `Error` all’interno di `catch` per ispezionare cosa è andato storto.

Il blocco `finally` viene eseguito dopo i blocchi `try` e `catch`, indipendentemente dal fatto che si sia verificato un errore o meno. Viene comunemente usato per attività di pulizia, come chiudere file o liberare risorse.

Ecco un esempio di utilizzo di un blocco `try…catch`:

:::interactive_editor

```js
function processInput(input) {
  if (typeof input !== "string") {
    throw new TypeError("Input must be a string.");
  }

  return input.toUpperCase();
}

try {
  console.log("Starting to process input...");
  const result = processInput(9);
  console.log("Processed result:", result);
} catch (error) {
  console.error("Error occurred:", error.message);
} 
```

:::

In questo esempio, abbiamo una funzione chiamata `processInput` che prima verifica se `input` non è di tipo `string`. Se è così, allora lanciamo un errore. Altrimenti, restituiamo il risultato dell’uso del metodo `toUpperCase` su `input`.

Chiamiamo la funzione all’interno di un blocco `try`. Poiché la chiamata alla funzione genera un errore, questo verrà catturato all’interno del blocco `catch` e il messaggio di errore verrà mostrato nella console.

L’errore passato nel blocco `catch` è un oggetto `Error` che contiene informazioni su quell’errore. In questo caso, stiamo usando la proprietà `message` che mostra informazioni leggibili dall’utente.

Stiamo usando `console.error` perché è progettato specificamente per registrare errori. In molti browser moderni, l’output di `console.error()` appare in rosso nella console.

L’istruzione `finally` viene eseguita indipendentemente dal fatto che sia stata lanciata un’eccezione o meno.

```js
try {
  // Code that might throw an error
} catch (error) {
  // Code to handle the error
} finally {
  // Code that runs regardless of whether an error occurred or not
}
```

Un buon caso d’uso per l’istruzione `finally` è se stai lavorando con file. In JavaScript, puoi aprire un file, usare un blocco `try` per scrivere dati nel file. Se ci sono errori, puoi usare `catch` per catturarli. Poi usa l’istruzione `finally` per chiudere il file.
