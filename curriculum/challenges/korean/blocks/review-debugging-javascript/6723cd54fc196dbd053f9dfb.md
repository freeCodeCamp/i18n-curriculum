## 흔한 오류 메시지 유형

- **SyntaxError**: 코드에서 괄호나 대괄호를 빼먹는 등 잘못 작성했을 때 발생하는 오류입니다. 문장의 문법 오류와 비슷하게 생각하세요.

```js
const arr = ["Beau", "Quincy" "Tom"]
```

- **ReferenceError**: 여러 가지 ReferenceError 유형이 있으며, 각각 다른 방식으로 발생합니다. 첫 번째 유형은 정의되지 않은 변수를 사용하는 경우입니다. 또 다른 ReferenceError 예는 `let` 또는 `const`로 선언된 변수를 정의되기 전에 접근하려 할 때입니다.

:::interactive_editor

```js
console.log(num);
const num = 50;
```

:::

- **TypeError**: 잘못된 타입에 대해 연산을 시도할 때 발생하는 오류입니다.

```js
const developerObj = {
  name: "Jessica",
  country: "USA",
  isEmployed: true
};

developerObj.map()
```

- **RangeError**: JavaScript가 처리할 수 있는 범위를 벗어난 값을 사용하려 할 때 발생하는 오류입니다.

```js
const arr = [];
arr.length = -1; 
```

## `throw` 문장

- **정의**: JavaScript에서 `throw` 문장은 사용자 정의 예외를 발생시키는 데 사용합니다. 프로그래밍에서 예외란 예상치 못한 사건이 발생해 프로그램의 정상 흐름을 방해하는 경우를 말합니다.

:::interactive_editor

```js
function validateNumber(input) {
  if (typeof input !== "number") {
    throw new TypeError("Expected a number, but received " + typeof input);
  }
  return input * 2;
}

console.log(validateNumber("Naomi"));   // TypeError: Expected a number, but received string
```

:::

## `try...catch...finally`

- **정의**: `try` 블록은 오류가 발생할 수 있는 코드를 감싸는 데 사용합니다. 실패할 수 있는 작업을 시도하는 안전한 공간 역할을 합니다. `catch` 블록은 try 블록에서 발생한 오류를 포착하고 처리합니다. catch 내부의 error 객체를 사용해 무엇이 잘못됐는지 확인할 수 있습니다. `finally` 블록은 오류 발생 여부와 상관없이 try와 catch 블록 뒤에 실행됩니다. 주로 파일 닫기나 자원 해제 같은 정리 작업에 사용합니다.

:::interactive_editor

```js
function processInput(input) {
  if (typeof input !== "string") {
    throw new TypeError("Input must be a string.");
  }

  return input.toUpperCase();
}

try {
  console.log("Starting to process input...");
  const result1 = processInput("hello");
  console.log("Processed result:", result1);  // HELLO
  const result2 = processInput(9);  // throws TypeError
  console.log("Processed result:", result2);  // not executed
} catch (error) {
  console.error("Error occurred:", error.message);
} 
```

:::

## 디버깅 기법

- **`debugger` 문장**: 이 문장은 코드의 특정 줄에서 실행을 일시 중지해 프로그램 상태를 조사할 수 있게 합니다.

:::interactive_editor

```js
let firstNumber = 5;
let secondNumber = 10;

debugger; // Code execution pauses here
let sum = firstNumber + secondNumber;

console.log(sum);
```

:::

- **중단점**: 중단점은 원하는 코드 줄에서 실행을 멈추게 합니다. 멈춘 후 변수 검사, 표현식 평가, 호출 스택 확인이 가능합니다.
- **감시자**: 감시 표현식은 현재 범위 밖에 있어도 코드 실행 중 변수나 표현식 값을 모니터링할 수 있게 합니다.
- **프로파일링**: 프로파일링은 CPU 사용량, 함수 호출, 실행 시간을 기록하고 스크린샷을 캡처해 성능 병목 현상을 찾는 데 도움을 줍니다.
- **`console.dir()`**: 이 메서드는 지정한 JavaScript 객체의 프로퍼티를 대화형 목록으로 표시합니다. 중첩된 모든 프로퍼티를 확장해 볼 수 있는 계층적 목록을 출력합니다.

```js
console.dir(document);
```

- **`console.table()`**: 이 메서드는 콘솔에 표 형식으로 데이터를 표시합니다. 필수 인수로 배열이나 객체를 받고, 선택 인수로 표시할 프로퍼티(열)를 지정할 수 있습니다.
