# --interactive--

تُعد ملاحظات المستخدم عنصرًا أساسيًا في تصميم الويب. على سبيل المثال، من المهم أن يتلقى المستخدمون إشارات بصرية عند تفاعلهم مع عناصر على الموقع، مثل التمرير فوق زر أو النقر على رابط. تساعد هذه الملاحظات المستخدمين على فهم حالة العناصر التفاعلية، مثل الإشارة إلى ما إذا تم زيارة الرابط أم لا.

فئات المزيفة لأفعال المستخدم في CSS هي كلمات مفتاحية خاصة تتيح لك تقديم هذا النوع من الملاحظات دون الحاجة إلى JavaScript أو لغات برمجة أخرى.

تشمل هذه الفئات المزيفة `:hover`، و`:active`، و`:focus`، و`:visited`، وغيرها. تتيح لك تغيير مظهر العناصر بناءً على تفاعلات المستخدم، مما يحسن تجربة المستخدم بشكل عام.

لنغص في بعض فئات المزيفة لأفعال المستخدم التي لدينا ونرى كيف تعمل.

تطبق الفئة المزيفة `:active` الأنماط عندما يُفعّل المستخدم عنصرًا. على سبيل المثال، عندما ينقر المستخدم على زر أو رابط، توفر رد فعل بصري فوري، يظهر للمستخدمين أن أفعالهم معترف بها.

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css" />
<a href="#">Example link</a>
```

```css
a:active {
  color: crimson;
}
```

:::

تُفعّل الفئة المزيفة `:hover` عندما يمرر المستخدم مؤشر الفأرة أو جهاز التوجيه فوق عنصر. يستخدم المطوّرون هذه الفئة غالبًا لإنشاء ردود فعل بصرية للأزرار أو الروابط أو أي عنصر يجب أن يستجيب لانتباه المستخدم. إليك زرًا يمرر المستخدم فوقه قبل النقر:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css" />
<button class="btn">Hover Over Me</button>
```

```css
.btn:hover {
  background-color: darkgreen;
  color: white;
  cursor: pointer;
}
```

:::

تطبق الفئة المزيفة `:focus` الأنماط عندما يكتسب عنصر التركيز، عادةً عبر التنقل باستخدام لوحة المفاتيح أو عند نقر المستخدم داخل حقل إدخال في نموذج. هذا ليس فقط لتقديم ملاحظات، بل هو ضروري أيضًا لإمكانية الوصول. يضمن ذلك أن المستخدمين الذين يعتمدون بشكل كبير على لوحة المفاتيح يمكنهم بسهولة تحديد العنصر الذي يتفاعلون معه.

إليك مثالًا على حقل إدخال يكتسب التركيز عند النقر عليه أو التنقل إليه عبر لوحة المفاتيح:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css" />
<form>
  <input type="text" />
</form>
```

```css
input:focus {
  outline: 2px solid darkgreen;
  border-radius: 4px;
}
```

:::

تستهدف الفئة المزيفة `:visited` الرابط الذي زاره المستخدم. يمكن أن يكون هذا مفيدًا لمساعدة المستخدمين على التمييز بين الصفحات التي زاروها بالفعل وتلك التي لم يزوروها بعد. إليك مثالًا على تغيير لون نص وسم الرابط إلى اللون السماوي عند زيارة الرابط:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css" />
<a href="https://www.example.com" target="_blank">Visit Example.com</a>
```

```css
a:visited {
  color: cyan;
}
```

:::

تتيح الفئة المزيفة `:checked` في CSS تنسيق عناصر النموذج مثل مربعات الاختيار وأزرار الاختيار عندما تكون مختارة (محددة). هذه الفئة المزيفة مفيدة لتخصيص مظهر هذه العناصر لتعزيز تجربة المستخدم، رغم أن المتصفحات توفر أنماطًا افتراضية لها.

إليك مثالًا مع مربع اختيار للموافقة على الشروط في موقع ويب.

**ملاحظة**: بعض خصائص CSS في هذا المثال لم تُغطَّ بعد. هذا فقط لإعطائك فكرة عن كيفية إنشاء مربع اختيار مخصص. ستتعلم كيف يعمل كل هذا في الدروس وورش العمل القادمة.

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css" />
<form>
  <label>
  Agree <input class="checkbox" type="checkbox" />
  </label>
</form>
```

```css
.checkbox {
  appearance: none;
  width: 18px;
  height: 18px;
  border: 2px solid #ccc;
  border-radius: 4px;
  display: inline-block;
  position: relative;
  cursor: pointer;
  transition: all 0.25s ease;
  vertical-align: middle; 
}

.checkbox:hover {
  border-color: #888;
}

.checkbox:checked {
  background-color: #4caf50;
  border-color: #4caf50;
}

.checkbox:checked::after {
  content: "";
  position: absolute;
  left: 4px;
  top: 0px;
  width: 5px;
  height: 10px;
  border: solid white;
  border-width: 0 2px 2px 0;
  transform: rotate(45deg);
}

.checkbox:focus {
  outline: 2px solid #90caf9;
  outline-offset: 2px;
}

```

:::

في هذا المثال، نستخدم الخاصية `appearance` مضبوطة على `none` لإزالة التنسيق الافتراضي الذي يطبقه المتصفح على مربعات الاختيار. عندما يحدد المستخدم المربع، سيكون له لون خلفية `green`.

أمثلة أخرى على فئات المزيفة لأفعال المستخدم هي:

- `:focus-within`: لتطبيق الأنماط على عنصر عندما يكون هو أو أي من عناصره التابعة في حالة تركيز.
- `:enabled`: لاستهداف أزرار النموذج أو العناصر الأخرى التي تكون مفعلة حاليًا.
- `:disabled`: لاستهداف أزرار النموذج أو العناصر الأخرى التي تكون معطلة.
- `:target`: لتطبيق الأنماط على عنصر يكون هدفًا لجزء من عنوان URL (الجزء بعد رمز `#`).
