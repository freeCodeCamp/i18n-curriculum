## العمل مع الأحداث في React

- **نظام الأحداث التركيبية**: هذه هي طريقة React في التعامل مع الأحداث. يعمل كغلاف حول الأحداث الأصلية مثل أحداث `click` و`keydown` و`submit`. تستخدم معالجات الأحداث في React اصطلاح التسمية بنمط الجمل المجمعة (camel casing). (مثال: `onClick`، `onSubmit`، إلخ)

إليك مثالًا على استخدام الخاصية `onClick` لعنصر `button` في React:

```jsx
function handleClick() {
  console.log("Button clicked!");
}

<button onClick={handleClick}>Click Me</button>;
```

في React، عادةً ما تبدأ دوال معالجات الأحداث بالبادئة `handle` للدلالة على مسؤوليتها عن التعامل مع الأحداث، مثل `handleClick` أو `handleSubmit`.

عندما يُفعّل المستخدم حدثًا، يمرر React كائن حدث تركيبي (Synthetic Event) إلى معالجك. يتصرف هذا الكائن بشكل مشابه لكائن الحدث الأصلي في جافاسكريبت العادية، ويوفر خصائص مثل `type` و`target` و`currentTarget`.

لمنع السلوكيات الافتراضية مثل تحديث المتصفح أثناء حدث `onSubmit`، يمكنك استدعاء الدالة `preventDefault()`:

```jsx
function handleSubmit(event) {
  event.preventDefault();
  console.log("Form submitted!");
}

<form onSubmit={handleSubmit}>
  <input type="text" />
  <button>Submit</button>
</form>;
```

يمكنك أيضًا تغليف دالة المعالجة داخل دالة سهمية هكذا:

```jsx
function handleDelete(id) {
  console.log("Deleting item:", id);
}

<button onClick={() => handleDelete(1)}>Delete Item</button>;
```

## العمل مع الحالة والدالة المساعدة `useState`

- **تعريف الحالة**: في React، الحالة هي كائن يحتوي على بيانات لمكوِّن. عند تحديث الحالة، يُعاد عرض المكوِّن. يعامل React الحالة كشيء غير قابل للتغيير، مما يعني أنه لا يجب تعديلها مباشرة.  
- **الدالة المساعدة `useState()`**: دالة `useState` تتيح لك إعلان متغيرات الحالة في المكوِّنات الوظيفية. هذه هي الصيغة الأساسية:

```js
const [stateVariable, setStateFunction] = useState(initialValue);
```

في متغير الحالة لديك ما يلي:

- `stateVariable` يحمل القيمة الحالية للحالة
- `setStateFunction` (دالة التعيين) تحدّث متغير الحالة
- `initialValue` يحدد الحالة الابتدائية

إليك مثالًا كاملاً لمكوِّن `Counter`:

```jsx
import { useState } from "react";

function Counter() {
  const [count, setCount] = useState(0);

  return (
    <div>
      <h2>{count}</h2>

      <button onClick={() => setCount(count - 1)}>Decrement</button>
      <button onClick={() => setCount(count + 1)}>Increment</button>
    </div>
  );
}

export default Counter;
```

## العرض ومكوِّنات React

- **التعريف**: في React، العرض هو العملية التي تظهر بها المكوِّنات في واجهة المستخدم (UI)، عادةً في المتصفح. تتكون عملية العرض من ثلاث مراحل: التفعيل، العرض، والتثبيت.

تحدث مرحلة التفعيل عندما يكتشف React أن شيئًا ما قد تغيّر وقد تحتاج واجهة المستخدم إلى تحديث. غالبًا ما يكون هذا التغيير بسبب تحديث في الحالة أو الخصائص.

بمجرد حدوث التفعيل، يدخل React مرحلة العرض. هنا، يعيد React تقييم مكوِّناتك ويحدد ما يجب عرضه. للقيام بذلك، يستخدم React نسخة خفيفة من DOM الحقيقي تُسمى DOM الافتراضي. باستخدام DOM الافتراضي، يمكن لـ React بسرعة التحقق مما يحتاج إلى تغيير في المكوِّن.

مرحلة التثبيت هي حيث يأخذ React التغييرات المُعدة من DOM الافتراضي ويطبقها على DOM الحقيقي. بعبارة أخرى، هذه هي المرحلة التي ترى فيها النتيجة النهائية على الشاشة.

## تحديث الكائنات والمصفوفات في الحالة

- **تحديث الكائنات في الحالة**: إذا كنت بحاجة إلى تحديث كائن في الحالة، يجب عليك أولًا إنشاء كائن جديد أو نسخ كائن موجود، ثم تعيين الحالة لذلك الكائن الجديد. يجب اعتبار أي كائن موضوع في الحالة للقراءة فقط. إليك مثالًا على تعيين اسم المستخدم، العمر، والمدينة. تُستخدم دالة `handleChange` لمعالجة تحديثات معلومات المستخدم:

```jsx
import { useState } from "react";

function Profile() {
  const [user, setUser] = useState({ name: "John Doe", age: 31, city: "LA" });

  const handleChange = (e) => {
    const { name, value } = e.target;

    setUser((prevUser) => ({...prevUser, [name]: value}));
  };

  return (
    <div>
      <h1>User Profile</h1>
      <p>Name: {user.name}</p>
      <p>Age: {user.age}</p>
      <p>City: {user.city}</p>

      <h2>Update User Age </h2>
      <input type="number" name="age" value={user.age} onChange={handleChange} />

      <h2>Update User Name </h2>
      <input type="text" name="name" value={user.name} onChange={handleChange} />

      <h2>Update User City </h2>
      <input type="text" name="city" value={user.city} onChange={handleChange} />
    </div>
  );
}

export default Profile;
```

- **تحديث المصفوفات في الحالة**: عند تحديث المصفوفات في الحالة، من المهم عدم تعديل المصفوفة مباشرة باستخدام طرق مثل `push()` أو `pop()`. بدلاً من ذلك، يجب إنشاء مصفوفة جديدة عند تحديث الحالة:

```js
const addItem = () => {
  const newItem = {
    id: items.length + 1,
    name: `Item ${items.length + 1}`,
  };

  // Creates a new array
  setItems((prevItems) => [...prevItems, newItem]);
};
```

إذا أردت إزالة عناصر من مصفوفة، يجب استخدام طريقة `filter()`، التي تُرجع مصفوفة جديدة بعد تصفية العناصر التي تريد إزالتها:

```js
const removeItem = (id) => {
  setItems((prevItems) => prevItems.filter((item) => item.id !== id));
};
```

## الإشارة إلى القيم باستخدام المراجع (Refs)

- **خاصية `ref`**: يمكنك الوصول إلى عقدة DOM في React باستخدام خاصية `ref`. هنا مثال يوضح استخدام `ref` لتركيز عنصر `input`. تُستخدم خاصية `current` للوصول إلى القيمة الحالية لذلك الـ `ref`:

```jsx
import { useRef } from "react";

const Focus = () => {
  const inputRef = useRef(null);

  const handleFocus = () => {
    if (inputRef.current) {
      inputRef.current.focus();
    }
  };

  return (
    <div>
      <input ref={inputRef} type="text" placeholder="Enter text" />
      <button onClick={handleFocus}>Focus Input</button>
    </div>
  );
};

export default Focus;
```

## العمل مع الدالة المساعدة `useEffect`

- **الدالة المساعدة `useEffect()`**: في React، التأثير هو أي شيء يحدث خارج عملية عرض المكوِّن. أي شيء لا يتعامل معه React مباشرة كجزء من عرض واجهة المستخدم. أمثلة شائعة تشمل جلب البيانات، تحديث عنوان تبويب المتصفح، القراءة من أو الكتابة إلى التخزين المحلي للمتصفح، الحصول على موقع المستخدم، وأكثر. هذه العمليات تتفاعل مع العالم الخارجي وتُعرف بالتأثيرات الجانبية. يوفر React الدالة المساعدة `useEffect` لتتعامل مع هذه التأثيرات. تتيح لك `useEffect` تشغيل دالة بعد عرض المكوِّن أو تحديثه.

```js
import { useEffect } from "react";

useEffect(() => {
 // Your side effect logic (usually a function) goes here
}, [dependencies]);
```

تعمل دالة التأثير بعد عرض المكوِّن، بينما يتحكم المعامل الاختياري `dependencies` في توقيت تشغيل التأثير.

لاحظ أن `dependencies` يمكن أن تكون مصفوفة من "القيم التفاعلية" (الحالة، الخصائص، الدوال، المتغيرات، وهكذا)، أو مصفوفة فارغة، أو تُحذف تمامًا. إليك كيف تتحكم كل هذه الخيارات في عمل `useEffect`:

- إذا كانت `dependencies` مصفوفة تحتوي على قيمة تفاعلية واحدة أو أكثر، فسيعمل التأثير كلما تغيرت تلك القيم.

- إذا كانت `dependencies` مصفوفة فارغة، يعمل `useEffect` مرة واحدة فقط عند العرض الأول للمكوِّن.

- إذا حذفت `dependencies`، يعمل التأثير في كل مرة يُعرض أو يُحدّث فيها المكوِّن.

## كيفية إنشاء الدوال المساعدة المخصصة

- **الدوال المساعدة المخصصة**: تتيح لك الدالة المساعدة المخصصة استخراج منطق قابل لإعادة الاستخدام من المكوِّنات، مثل جلب البيانات، إدارة الحالة، التبديل، والتأثيرات الجانبية مثل تتبع حالة الاتصال بالإنترنت. في React، تبدأ جميع الدوال المساعدة المدمجة بالكلمة `use`، لذا يجب أن تتبع دالتك المساعدة المخصصة نفس الاصطلاح.

إليك مثالًا على إنشاء دالة مساعدة `useDebounce`:

```js
function useDebounce(value, delay) {
  const [debouncedValue, setDebouncedValue] = useState(value);

  useEffect(() => {
    const handler = setTimeout(() => {
      setDebouncedValue(value);
    }, delay);

    return () => {
      clearTimeout(handler);
    };
  }, [value, delay]);

  return debouncedValue;
}

export { useDebounce };
```
