# --description--

Warum wirken manche Websites schnell und responsiv, während andere träge erscheinen? Die Antwort liegt in den wichtigsten Performance-Konzepten, die beeinflussen, wie eine Seite lädt und gerendert wird. Das Verständnis der wichtigsten Web-Performance-Konzepte ist entscheidend, um schnelle, flüssige und benutzerfreundliche Websites zu erstellen.

Schauen wir uns die Reihenfolge im Quelltext, den kritischen Rendering-Pfad, die Latenz und mehr an.

Beginnen wir mit der Reihenfolge im Quelltext.

Die Reihenfolge im Quelltext bezieht sich darauf, wie HTML-Elemente im Dokument strukturiert sind. Dies bestimmt, was zuerst lädt und kann die Performance und Barrierefreiheit erheblich beeinflussen.

Einige bewährte Vorgehensweisen dafür sind:

- Kritische Inhalte wie Überschriften, Navigation oder Haupttext höher in der HTML-Struktur platzieren.

- Nicht wesentliche Skripte wie solche für Analytics oder Drittanbieter-Widgets verzögern, damit sie das Rendern nicht blockieren.

- Progressive Enhancement verwenden, um sicherzustellen, dass das Grundgerüst der Seite funktioniert, noch bevor Styles und Skripte geladen sind. Progressive Enhancement ist eine Methode, Websites und Anwendungen so zu bauen, dass die Seite zuerst mit HTML funktioniert.

Hier ist ein Beispiel für eine gute Reihenfolge im Quelltext, basierend auf den gerade besprochenen bewährten Vorgehensweisen.

```html
<!-- Good source order: Essential content first -->
<h1>Welcome to FastSite!</h1>
<p>Critical information loads first.</p>
<script src="slow-script.js" defer></script>
```

Durch die Optimierung der Reihenfolge im Quelltext stellen wir sicher, dass Nutzer wichtige Inhalte so schnell wie möglich sehen.

Kommen wir nun zum kritischen Rendering-Pfad.

Der kritische Rendering-Pfad ist die Folge von Schritten, die der Browser durchläuft, um Code in Pixel auf dem Bildschirm umzuwandeln.

Hier sind die wichtigsten Schritte, auf die wir später noch genauer eingehen:

1. HTML parsen: Baut das DOM (Document Object Model) auf

2. CSS parsen: Baut das CSSOM (CSS Object Model) auf

3. JavaScript-Ausführung: Kann DOM und CSSOM verändern

4. Render-Baum-Konstruktion: Kombiniert DOM und CSSOM

5. Layout & Painting: Bestimmt Elementgrößen und zeichnet Pixel

6. Optimierungen:

    - Render-blockierende Ressourcen minimieren (z. B. große CSS-Dateien, ungenutztes JS).

    - Async- und Defer-Attribute für Skripte verwenden.

    - Zuerst nur essentielle Styles laden; nicht-kritisches CSS verzögern.

Insgesamt bedeutet ein kürzerer kritischer Rendering-Pfad eine schnellere wahrgenommene Performance. Darauf gehen wir später noch ausführlicher ein.

Und schließlich betrachten wir die Latenz.

Latenz ist die Zeit, die eine Anfrage benötigt, um zwischen Browser und Server hin- und zurückzureisen. Anders gesagt: Hohe Latenz bedeutet langsame Seiten.

Einige Möglichkeiten, die Latenz zu reduzieren, sind:

- Verwendung von CDNs, also Content Delivery Networks, um Dateien von näher gelegenen Standorten auszuliefern.

- Aktivierung von Kompression, zum Beispiel mit Gzip, um Dateigrößen zu verringern.

- Optimierung von Bildern und Einsatz von Lazy Loading – auch darauf gehen wir später noch ein.

Durch die Reduzierung der Latenz wirken Interaktionen sofortiger.

Zusammenfassend lässt sich sagen: Durch die Optimierung der Reihenfolge im Quelltext, die Verkürzung des kritischen Pfads und die Verringerung der Latenz können Sie Ihre Website schneller wirken lassen.
