# --description--

**優先キュー**は、キューやスタックと似た動作をする抽象データ型（ADT）ですが、重要な違いがあります。

ご存じの通り、標準的なキューはFIFO（先入れ先出し）方式で動作し、キューに最初に追加された要素が最初に取り出されます。

スタックはLIFO（後入れ先出し）方式で動作し、スタックに最後に追加された要素が最初に取り出されます。

キューとスタックは要素の挿入順序だけを考慮します。

しかし、優先キューは要素の「優先度」を考慮します。優先度は、どの要素を次に取り出すかを決めるために使われます。

通常は、最も優先度の高い要素が最初に取り出されますが、実装によっては最も優先度の低い要素を先に取り出す場合もあります。これはプログラムの要件によります。

優先キューは、2地点間の最短経路の検索、オペレーティングシステムのタスクスケジューリング、交通シミュレーション、データ圧縮、ネットワーク管理などの実用的な用途で非常に役立ちます。

実際には、優先キューはヒープというデータ構造を使って実装されることが多いです。

**ヒープ**は、**ヒープ特性**と呼ばれる非常に特定のプロパティを持つ木構造のデータ構造です。このプロパティは、ヒープの種類に基づいて各ノードとその子ノードの関係を決定します。

ヒープには主に2つの種類があります：

* 最大ヒープ（Max-heap）
    
* 最小ヒープ（Min-heap）
    

**最大ヒープ**では、各ノードの値はその子ノードの値以上です。

この例では、8、7、5、2、1のノードを持つ木構造が示されています。ノード7はノード2とノード1より大きく、ヒープ特性を満たしています。これは他のすべてのノードにも当てはまります。

<img src="https://cdn.freecodecamp.org/curriculum/lecture-transcripts/how-do-priority-queues-and-heaps-work-1.png" alt="A max-heap tree structure showing nodes with values 8 at the root, 7 and 5 as children of 8, and 2 and 1 as children of 7, demonstrating that each parent node is greater than its children.">

一方、**最小ヒープ**では、各ノードの値はその子ノードの値以下です。

この例では、4、7、9、12、15のノードがあります。例えば、ノード7はノード12とノード15より小さく、ヒープ特性を満たしています。これも他のすべてのノードに当てはまります。

<img src="https://cdn.freecodecamp.org/curriculum/lecture-transcripts/how-do-priority-queues-and-heaps-work-2.png" alt="A min-heap tree structure showing nodes with values 4 at the root, 7 and 9 as children of 4, and 12 and 15 as children of 7, demonstrating that each parent node is less than its children.">

ヒープ特性は重要で、最大（または最小）要素が常にトップに位置することを保証し、取り出しを非常に簡単にします。

実際には、ヒープは親ノードと子ノードに効率的にアクセスできるように配列として実装されることが多いです。

配列を使うことで、ヒープが完全二分木の構造を維持している場合、要素がメモリ上のどこにあるかをインデックスに基づく単純な数学的操作で見つけられるため、これらの値や「ノード」へのアクセスのロジックが簡単になります。

Pythonには`heapq`という組み込みモジュールがあり、最小ヒープの実装を扱うことができます。

このモジュールはPythonのリストに直接操作を行い、リストをヒープとして扱いながらヒープ特性を維持する特定のステップに従って動作します。

このモジュールを使うには、インポートしてください：

```python
import heapq
```

次に、空のリストを定義します。これがヒープの基盤となるデータ構造です：

```python
my_heap = []
```

ヒープに要素を追加するには、`heappush()`を呼び出し、ヒープの名前と追加したい要素を引数として渡します。これにより、ヒープ特性を保つために要素が適切な位置に自動的に追加されます：

```python
heapq.heappush(my_heap, 9)
```

最も優先度の高い要素（この場合は最小値）を取得するには、`heappop()`を呼び出します：

```python
heapq.heappop(my_heap)
```

`heappushpop()`は両方の操作を1回の呼び出しで行います。

これは、特にヒープが大きい場合に、リストをヒープとして再構成する「ヒープ化」操作を1回だけ行うため、個別に呼び出すより効率的です：

```python
heapq.heappushpop(my_heap, 15)
```

すでにリストがある場合、それをヒープに変換したいなら、`heapify()`を呼び出し、リストを引数として渡せます：

```python
heapq.heapify(my_heap)
```

しかし、今は要素を値でソートしていますよね？

もし「優先度」でソートしたい場合はどうしますか？

このような構造のタプルを格納することで実現できます：`(priority, element)`。

タプルは左から右へ要素ごとに比較されるため、優先度が最初に比較され、それに基づいて判断されます。

この場合、値が小さいほど優先度が高いことに注意してください。つまり、優先度が1のタプルは優先度が3のタプルより**高い**優先度を持ちます：

```python
my_heap = []

heapq.heappush(my_heap, (3, "A"))
heapq.heappush(my_heap, (2, "B"))
heapq.heappush(my_heap, (1, "C"))
```

同じ優先度の要素を挿入された順に取り出したい場合は、タプルの2番目の要素に一意のカウンターを含めて、同順位を区別することを検討してください。例えば、`(priority, counter, element)`のようにします。

では、ヒープの効率について話しましょう。

ヒープ（種類によって最小値または最大値）の挿入と抽出の平均および最悪ケースの時間計算量は対数時間で、`O(log n)`、これは必要な交換回数がおおよそヒープの高さ（log(n)）に比例するためです。

「peek」操作（最小値または最大値を取り出さずに取得する操作）の平均および最悪ケースの時間計算量は定数時間、O(1)です。

未ソートのリストからヒープを構築する「ヒープ化」操作は、平均および最悪ケースで線形時間、O(n)の計算量です。

同様に、任意の要素の検索や削除も平均および最悪ケースで線形時間、O(n)の計算量です。これはヒープ全体を走査する必要があるためです。

では、必要な空間はどのくらいでしょうか？

ヒープの空間計算量は線形、O(n)で、`n`は含まれる要素数です。要素とリストオブジェクト自体のわずかな追加オーバーヘッドだけを保存します。

優先キューとヒープはコンピュータサイエンスで非常に重要です。リストから最も重要な要素を素早く見つけて使うことができ、この効率性は地図上の最速ルート検索など、現実世界の重要なタスクを実行する多くのプログラムに不可欠です。
