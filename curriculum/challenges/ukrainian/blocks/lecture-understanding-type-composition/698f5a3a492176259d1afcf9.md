# --description--

У попередньому уроці ви дізналися, як працювати з псевдонімами типів і побачили приклади, як називати типи об’єктів.

Як ви пам’ятаєте, один зі способів назвати тип об’єкта — це використати ключове слово `type` ось так:

```ts
type ID = number | string;

type User = {
  id: ID;
  name: string;
};
```

Інший спосіб назвати тип об’єкта — це використати оголошення `interface` ось так:

```ts
type ID = number | string;

interface User {
  id: ID;
  name: string;
}
```

`type` і `interface` схожі, але одна ключова відмінність полягає в тому, що інтерфейси використовують ключове слово `extends` для розширення інших інтерфейсів, тоді як псевдоніми типів `type` використовують перетини типів (`&`) для комбінування типів.

Давайте розглянемо кілька прикладів:

```ts
interface User {
  id: number | string;
  name: string;
}

interface Admin extends User {
  role: string;
}

const adminUser: Admin = {
  id: 101,
  name: "Alice",
  role: "superadmin"
};
```

У цьому прикладі `User` починається з `id` і `name`. Інтерфейс `Admin` розширює `User`, що означає, що він включає `id` і `name` та додає нову властивість під назвою `role`.

Якщо ви хочете розширити `type`, ви б використали перетин типів (`&`) ось так:

```ts
type User = {
  id: number | string;
  name: string;
};

type Admin = User & {
  role: string;
};

const adminUser: Admin = {
  id: 101,
  name: "Alice",
  role: "superadmin"
};
```

`Admin` поєднує тип `User` з новим типом об’єкта, який включає `role`. Результат має таку ж форму, як і приклад з інтерфейсом — це об’єкт з `id`, `name` і `role`.

І `type`, і `interface` можуть описувати форму об’єкта. Якщо ви не впевнені, що використовувати, пам’ятайте, що `interface` спеціально призначений для форм об’єктів, тоді як `type` є більш гнучким і може також представляти об’єднання типів, примітиви та більш складні комбінації типів.
