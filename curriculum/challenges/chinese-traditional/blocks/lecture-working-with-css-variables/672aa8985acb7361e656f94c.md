# --interactive--

CSS 訂製屬性，也稱為 CSS 變數，是由 CSS 作者定義的實體，包含特定值以便在整個文件中重複使用。它們是一項強大的功能，允許更高效、易於維護且靈活的樣式表。

宣告訂製屬性的語法很簡單。它以兩個破折號（`--`）開頭，後面接屬性名稱：

```css
:root {
  --main-color: #3498db;
}
```

在這個範例中，我們宣告了一個名為 `--main-color` 的訂製屬性，值為 `#3498db`。`:root` 偽類通常用來宣告全域的訂製屬性，因為它表現 DOM 樹中最高層級的父元素。

要使用訂製屬性，你必須使用 `var()` 函式（程式）：

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<button class="button">Click Me</button>
```

```css
:root {
  --main-color: #3498db;
}

.button {
  background-color: var(--main-color);
}
```

:::

這會將類別為 `button` 的元素的背景顏色設定為儲存在 `--main-color` 的值。

訂製屬性的一個關鍵特性是它們遵循 CSS 層疊規則。這表示你可以為特定元素或背景關係重新定義它們：

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<div class="alert">This is an alert message.</div>
```

```css
.alert {
  --main-color: #e74c3c;
  background-color: var(--main-color);
}
```

:::

在這種情況下，具有 `alert` 類別的元素將使用不同的 `--main-color` 值，覆蓋全域的定義。

訂製屬性也支援備用值。如果訂製屬性未定義或無效，你可以提供備用值：

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<div class="text">This is some text.</div>
```

```css
:root {
  --text-color: green;
}

.text {
  color: var(--text-color, green);
}
```

:::

在這裡，如果沒有定義 `--text-color`，`color` 將預設為 `green`。

訂製屬性在建立可主題化設計時特別有用。你可以為不同主題定義一組屬性：

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<div class="light-theme">
  <p>This is some light themed text.</p>
</div>
```

```css
:root {
  --bg-color: black;
}

body {
  background-color: var(--bg-color);
}

.light-theme {
  --bg-color: white;
  --text-color: black;
  background-color: var(--bg-color);
  color: var(--text-color);
  padding: 10px;
}

```

:::

切換主題變得和為 `body` 元素添加或移除類別一樣簡單。

訂製屬性也可以與媒體查詢一起使用，以創建響應式設計：

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<div class="card">
  <h2>Responsive Design</h2>
  <p>Resize the window to see the card adapt!</p>
</div>

```

```css
:root {
  --card-width: 90%;
  --card-bg: #f0f0f0;
  --card-padding: 1rem;
  --text-color: #333;
}

/* Tablet screens and up */
@media (min-width: 600px) {
  :root {
    --card-width: 70%;
    --card-bg: #e8f5e9;
    --card-padding: 1.5rem;
  }
}

/* Desktop screens and up */
@media (min-width: 1024px) {
  :root {
    --card-width: 50%;
    --card-bg: #d0f0ff;
    --card-padding: 2rem;
  }
}

body {
  font-family: system-ui, sans-serif;
  display: flex;
  justify-content: center;
  align-items: center;
  min-height: 100vh;
  background-color: #fafafa;
}

.card {
  width: var(--card-width);
  background-color: var(--card-bg);
  padding: var(--card-padding);
  color: var(--text-color);
  border-radius: 8px;
  box-shadow: 0 2px 6px rgba(0,0,0,0.1);
  text-align: center;
  transition: all 0.3s ease;
}

```

:::

這種方法允許更易於維護的響應式佈局，因為你可以在一處更改數值，而不是在整個樣式表中更改。這可以大幅減少樣式表中的重複，特別是對於經常使用的複雜數值。

訂製屬性也可以引用其他訂製屬性：

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<button class="button">Click Me</button>
```

```css
:root {
  --primary-color: #3498db;
  --button-bg: var(--primary-color);
}

button {
  background-color: var(--button-bg);
}

```

:::

這允許在不同的樣式面向之間建立關係，使你更容易維護網站整體一致的主題。

雖然訂製屬性提供許多優點，但了解它們的瀏覽器支援情況非常重要。它們在現代瀏覽器中獲得良好支援，但舊版瀏覽器可能無法識別它們。當使用尖端功能時，務必準備備用方案。

總結來說，CSS 訂製屬性提供了一種強大的方式來產生更具動態性、彈性和易於維護的樣式表。它們允許產生可主題化的設計、簡化響應式佈局，並啟用樣式的執行期操作。 

隨著網頁開發持續演進，訂製屬性正成為開發者工具組中越來越重要的工具，為創建可調整且高效的 CSS 提供了新可能性。
