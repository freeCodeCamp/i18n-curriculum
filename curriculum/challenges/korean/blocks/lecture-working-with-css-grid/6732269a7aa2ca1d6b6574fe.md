# --interactive--

암시적 그리드는 그리드 레이아웃에서 아이템을 배치할 때 브라우저가 자동으로 생성하는 행과 열을 의미합니다 – `grid-template-rows` 또는 `grid-template-columns`을 사용해 명시적으로 정의하지 않은 부분입니다.

브라우저가 암시적으로 생성하는 열과 행을 제어하는 속성은 `grid-auto-columns`와 `grid-auto-rows`입니다.

암시적 그리드는 또한 명시적으로 정의된 행과 열 밖에 아이템을 배치할 때 브라우저가 자동으로 생성하는 추가 행과 열을 의미합니다.

예를 들어, 다음과 같이 그리드 레이아웃에서 두 개의 명시적 열만 정의했다고 가정해 보겠습니다:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<div class="grid-container">
  <div class="grid-item">Item 1</div>
  <div class="grid-item">Item 2</div>
  <div class="grid-item">Item 3</div>
  <div class="grid-item">Item 4</div>
  <div class="grid-item">Item 5</div>
  <div class="grid-item">Item 6</div>
</div>
```

```css
.grid-container {
  display: grid;
  grid-template-columns: 100px 100px; /* Only 2 explicit columns */
}

.grid-item {
  background-color: burlywood;
  border: 1px solid orangered;
  padding: 0.5rem;
  margin: 0.5rem;
}
```

:::

두 개의 아이템이 첫 번째 행을 두 개의 명시적 열을 사용해 채웠습니다 – `Item 1`는 첫 번째 열에, `Item 2`는 두 번째 열에 있습니다.

다음 아이템들은 두 번째 행의 첫 번째 열에 `Item 3`, 두 번째 열에 `Item 4`가 배치되면서 새 행을 시작합니다. 이런 식으로 계속됩니다.

이미 보았듯이, 명시적 그리드는 의도적으로 설정한 그리드 영역입니다. 즉, `grid-template-rows`와 `grid-template-columns` 속성을 사용해 그리드 레이아웃에 명시적으로 정의한 행과 열입니다.

다음은 암시적 그리드와 명시적 그리드의 차이를 요약한 표입니다:

| **기능**              | **명시적 그리드**                                                                   | **암시적 그리드**                                                                          |
|----------------------|------------------------------------------------------------------------------------|--------------------------------------------------------------------------------------------|
| **크기 제어**         | ` grid-template-rows `와 ` grid-template-columns `를 사용해 완전히 사용자 지정 가능.                          | ` grid-auto-rows `와 ` grid-auto-columns `에 의해 제어되거나 기본값인 ` auto `를 사용.                      |
| **기본 동작**         | 명시적으로 정의하지 않으면 변경되지 않음.                                          | 명시적 그리드 밖에 아이템이 배치되면 자동으로 적응함.                                      |
| **복잡성**            | 레이아웃 구조를 더 많이 계획해야 함.                                               | 구조화되지 않았거나 가변적인 콘텐츠에 구현하기 더 쉬움.                                    |
| **유연성**            | 정의된 행과 열로 엄격한 구조.                                                      | 동적으로 배치된 콘텐츠에 유연하게 적응함.                                                  |
| **성능**              | 레이아웃이 미리 정의되어 있어 잠재적으로 더 높은 성능.                             | 암시적 그리드 생성을 위해 추가적인 브라우저 계산이 필요할 수 있음.                         |
| **사용 사례**         | 그리드 구조가 예측 가능하고 미리 정의된 경우에 유용.                              | 콘텐츠가 알 수 없거나 자주 변경되는 동적 레이아웃에 유용.                                 |
