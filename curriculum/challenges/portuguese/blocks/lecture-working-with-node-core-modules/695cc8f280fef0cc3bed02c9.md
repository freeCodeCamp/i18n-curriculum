# --description--

O módulo OS é outro módulo padrão que vem incorporado no Node.js.

Ele permite que você interaja com o sistema operacional atual em que o Node está sendo executado para que você possa acessar informações vitais como o tipo de SO, detalhes da CPU, memória disponível, memória total, interfaces de rede e mais.

Para usar o módulo OS, você o importa desta forma:

```js
const os = require("os");
```

O módulo OS fornece vários métodos que você pode usar para tornar sua aplicação mais consciente do sistema. Vamos ver alguns dos mais comuns.

O método `platform()` recupera o sistema operacional no qual o Node está atualmente em execução:

```js
const os = require("os");
console.log(os.platform()); // darwin
```

Pode ser útil para implementar scripts multiplataforma:

```js
if (os.platform() === 'win32') {
 // Windows specific code
} else {
 // Non-Windows specific code
}
```

O método `arch()` mostra uma string que representa a arquitetura da CPU para a qual o Node.js foi compilado:

```js
const os = require("os");
console.log(os.arch()); // arm64
```

Isso pode ser útil se você quiser que os usuários baixem os binários e dependências corretos para uma determinada arquitetura.

Outros valores possíveis são `'x64'`, `'arm'`, `'arm64'`, `'ia32'`, `'mips'`, `'ppc'` e outros.

`type()` obtém o nome oficial do SO, para que você possa identificar programaticamente os sistemas operacionais:

```js
const os = require("os");
console.log(os.type()); // Darwin (the core OS for macOS, iOS, and other Apple products)
```

`release()` mostra a versão do kernel do sistema operacional, a parte central do sistema operacional que gerencia os recursos do sistema e a comunicação entre os componentes de hardware e software. Este método pode ser útil para acompanhar a compatibilidade entre as versões do kernel do sistema operacional e os requisitos do servidor.

Ele retorna isso como uma string como `'20.6.0'`:

```js
const os = require("os");
console.log(os.release()); // 25.0.0
```

`version()` retorna a versão específica do sistema operacional com mais detalhes do que o método `release()`:

```js
const os = require("os");
console.log(os.version());
// Darwin Kernel Version 25.0.0: Wed Sep 17 21:41:39 PDT 2025;
// root:xnu-12377.1.9~141/RELEASE_ARM64_T8103
```

`cpus()` retorna um array de objetos com detalhes sobre cada núcleo lógico da CPU. Isso pode ajudar a monitorar a carga da CPU:


`uptime()` mostra o tempo desde que o sistema foi iniciado. Ele pode ajudar a determinar há quanto tempo os servidores estão em execução:

```js
const os = require("os");
console.log(os.uptime()); // 23047
```

`totalmem()` e `freemem()` mostram a quantidade total de memória do sistema em bytes e a memória livre do sistema em bytes, respectivamente:

```js
const os = require("os");
console.log(os.totalmem()); // 8589934592 (8 GB)
console.log(os.freemem()); // 93585408 (87 MB)
```

`userInfo()` retorna um objeto contendo informações sobre o usuário atual do sistema:

```js
const os = require("os");
console.log(os.userInfo());

/*
[Object: null prototype] {

 gid: 20,

 homedir: '/Users/user',
 shell: '/bin/zsh'
}
*/
```
