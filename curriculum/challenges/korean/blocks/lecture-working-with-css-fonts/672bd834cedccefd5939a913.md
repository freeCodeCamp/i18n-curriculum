# --description--

시작하기 전에 CSS에서 @규칙이 무엇인지 알아야 합니다. @규칙은 브라우저에 지침을 제공하는 문장입니다. 미디어 쿼리, 키프레임, 폰트 페이스 등 스타일시트의 다양한 측면을 정의하는 데 사용할 수 있습니다. 미디어 쿼리와 키프레임 같은 개념은 앞으로 수업에서 더 배우게 됩니다. 이번에는 `@font-face` @규칙에 대해 배웁니다.

`@font-face`을 사용하면 폰트 파일, 형식, 그리고 굵기와 스타일 같은 중요한 프로프를 지정하여 사용자 정의 폰트를 정의할 수 있습니다. 기본 구문은 다음과 같습니다. `@font-face` 다음에 중괄호로 묶인 블록이 옵니다:

```css
@font-face {
  /* Descriptors */
}
```

중괄호 안에는 폰트 페이스를 사용자 정의하기 위한 설명자를 포함해야 합니다. 가장 자주 사용하는 설명자 몇 가지를 살펴보겠습니다. font-family 설명자는 스타일시트 전체에서 해당 폰트를 참조할 때 사용할 이름을 지정합니다. 예를 들어, 다음과 같이 `@font-face rule`을 정의했다고 가정해 봅시다. `font-family` 설명자가 정의되어 있고 그 값은 `MyCustomFont`입니다:

```css
@font-face {
  font-family: "MyCustomFont";
}
```

이 경우, CSS 선택자에 일치하는 모든 요소에 이 사용자 정의 폰트 패밀리를 할당하려면 스타일시트에서 해당 이름을 사용해야 합니다. 예를 들어 `font-family`의 값으로 사용합니다:

```css
body { 
  font-family: "MyCustomFont"; 
}
```

하지만 `@font-face` @규칙이 유효하려면 `src`도 지정해야 합니다. 이 프로프는 폰트 리소스에 대한 참조를 포함합니다. 기본적으로 쉼표로 구분된 외부 참조 또는 로컬에 설치된 폰트 페이스 이름 목록입니다. 또한 폰트 리소스의 형식과 기술에 관한 힌트를 포함할 수 있습니다.

아래 코드에서는 `src`이 정의되어 있습니다. 값은 쉼표로 구분된 URL 목록이며 가독성을 위해 각 줄에 나누어 작성했습니다:

```css
@font-face {
  font-family: "MyCustomFont"; 
  src: url("path/to/font.woff2"),
    url("path/to/font.woff"),
    url("path/to/font.otf");
}
```

스타일시트에 파일을 포함하려면 `url()` 함수를 호출할 수 있습니다. 여기서는 폰트 파일을 포함하고 있습니다. 소괄호와 따옴표 안에 파일 경로와 확장자를 작성해야 합니다. 가독성을 위해 각 리소스를 다른 줄에 작성할 수 있지만 마지막 줄은 세미콜론으로 끝나야 합니다.

각 폰트 리소스에 대해 형식을 지정할 수도 있습니다. 이는 선택 사항이며, 브라우저에 폰트 형식에 관한 힌트를 제공합니다. 형식을 생략하면 리소스가 다운로드된 후 형식이 감지됩니다. 형식이 유효하지 않으면 리소스가 다운로드되지 않습니다.

가능한 폰트 형식에는 `collection`, `embedded-opentype`, `opentype`, `svg`, `truetype`, `woff`, 그리고 `woff2`이 있습니다.

다음은 폰트 형식을 지정한 예입니다. 소괄호와 따옴표 안에 특정 형식을 작성하는 방식을 확인하세요:

```css
@font-face {
  font-family: "MyCustomFont"; 
  src: url("path/to/font.woff2") format("woff2"),
    url("path/to/font.otf") format("opentype"),
    url("path/to/font.woff") format("woff");
}
```

이 예에서는 WOFF2 형식, OpenType 형식, 그리고 WOFF 형식을 지정하고 있습니다.

woff는 "Web Open Font Format"의 약자입니다. WOFF와 WOFF2의 차이는 데이터를 압축하는 알고리즘에 있습니다. OpenType은 Microsoft와 Adobe가 개발한 확장 가능한 컴퓨터 폰트 형식으로, 사용자가 폰트의 추가 기능에 접근할 수 있게 합니다. 주요 운영체제에서 널리 사용됩니다.

형식을 지정하는 것 외에도 폰트 리소스의 기술을 지정할 수 있습니다. 이것도 선택 사항입니다. 두 번째 폰트 리소스의 기술을 지정한 예는 다음과 같습니다.

```css
@font-face {
  font-family: "MyCustomFont"; 
  src: url("path/to/font.woff2") format("woff2"),
    url("path/to/font.otf") format("opentype") tech(color-COLRv1),
    url("path/to/font.woff") format("woff");
}
```

이것이 `@font-face` 규칙의 기본입니다. 이 @규칙을 사용하면 고유한 디자인을 위한 사용자 정의 폰트를 정의할 때 폰트 파일을 지정할 수 있습니다.
