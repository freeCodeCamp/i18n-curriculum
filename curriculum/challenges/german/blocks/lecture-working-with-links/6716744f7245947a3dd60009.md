# --interactive--

Sie haben vielleicht das `target`-Attribut bei Ankerelementen oder Links gesehen. Dieses wichtige Attribut sagt dem Browser, wo die URL für das Ankerelement geöffnet werden soll.

Aktivieren Sie den interaktiven Editor, klicken Sie auf den Link, und Sie werden auf der freeCodeCamp-Startseite in einem neuen Browser-Tab weitergeleitet.

:::interactive_editor

```html
<a href="https://freecodecamp.org" target="_blank">Visit freeCodeCamp</a>
```

:::

Es gibt vier wichtige mögliche Werte für dieses Attribut. Beachten Sie, dass jedem Wert ein Unterstrich vorangestellt ist.

Der erste Wert ist `_self`, der Standardwert. Dieser öffnet den Link im aktuellen Browsing-Kontext. In den meisten Fällen ist das der aktuelle Tab oder das aktuelle Fenster.

Der zweite Wert ist `_blank`, der den Link in einem neuen Browsing-Kontext öffnet. Typischerweise wird dies in einem neuen Tab geöffnet. Manche Nutzer konfigurieren ihre Browser jedoch so, dass stattdessen ein neues Fenster geöffnet wird.

Der dritte Wert ist `_parent`, der den Link im übergeordneten Kontext des aktuellen öffnet. Zum Beispiel, wenn Ihre Website ein `iframe` hat, würde ein `_parent`-Wert in diesem `iframe` in dem Tab/Fenster Ihrer Website geöffnet werden, nicht im eingebetteten Frame.

Der vierte Wert ist `_top`, der den Link im obersten Browsing-Kontext öffnet – denken Sie an „der Elternteil des Elternteils“. Dies ist ähnlich wie `_parent`, aber der Link wird immer im vollständigen Browser-Tab/Fenster geöffnet, auch bei verschachtelten eingebetteten Frames.

Es gibt einen fünften Wert, genannt `_unfencedTop`, der derzeit für die experimentelle FencedFrame-API verwendet wird. Zum Zeitpunkt dieser Lektion werden Sie diesen wahrscheinlich noch nicht verwenden müssen.

Die Auswahl des richtigen `target`-Werts, um zu steuern, wo Ihre Nutzer landen, ist eine wichtige Überlegung beim Erstellen einer Website.
