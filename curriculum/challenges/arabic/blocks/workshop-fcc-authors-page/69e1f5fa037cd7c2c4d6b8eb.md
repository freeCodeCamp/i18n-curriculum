# --description--

أخيرًا، عليك التعامل مع طريقة `catch`. أولًا، قم بتسكين جملة `fetch`، وجملة `json`، واستدعاء displayAuthors داخل كتلة `try` ضمن `initialFetch`. ثم عدّل طريقة `catch` المتسلسلة لتصبح كتلة catch كما في المثال الموضح أدناه.

```js
try {
  // code that can cause or throw an error goes in this block
} catch (error) {
  // error handling code here
}
```

إذا فعلت ذلك بشكل صحيح، ستحتوي كتلة `try` على ثلاثة أسطر من الشفرة، وستحتوي كتلة `catch` على سطر واحد فقط. كما سيُحل خطأ `SyntaxError` ويجب أن يعمل التطبيق بشكل طبيعي.

أعطِ نفسك تحية على إنجازك! لقد تعلمت كيف تجعل شفرتك غير المتزامنة أكثر قابلية للقراءة باستخدام العاملين `async` و `await`! الآن صفحة المؤلفين الخاصة بك مكتملة.
