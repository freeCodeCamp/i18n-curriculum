# --interactive--

## Funzioni JavaScript

- Le funzioni sono blocchi di codice riutilizzabili che eseguono un'attività specifica.
- Le funzioni possono essere definite usando la parola chiave `function` seguita da un nome, una lista di parametri e un blocco di codice che esegue l'attività.

:::interactive_editor

```js
function addNumbers(x, y, z) {
  return x + y + z;
}

console.log(addNumbers(5, 3, 8)); // Output: 16
```

:::

- Gli argomenti sono valori passati a una funzione quando viene chiamata.
- Una chiamata di funzione è il processo di eseguire una funzione in un programma specificando il nome della funzione seguito da parentesi tonde, includendo opzionalmente argomenti all'interno delle parentesi.
- Quando una funzione termina la sua esecuzione, restituisce sempre un valore.
- Per impostazione predefinita, il valore di ritorno di una funzione è `undefined`.
- La parola chiave `return` viene usata per specificare il valore da restituire dalla funzione e termina l'esecuzione della funzione.
- I parametri predefiniti permettono alle funzioni di avere valori già impostati che verranno usati se un argomento non viene fornito quando la funzione viene chiamata. Questo rende le funzioni più flessibili e previene errori nei casi in cui certi argomenti potrebbero essere omessi.

:::interactive_editor

```js
const calculateTotal = (amount, taxRate = 0.05) => {
  return amount + (amount * taxRate);
};

console.log(calculateTotal(100)); // Output: 105
```

:::

- Le funzioni anonime sono funzioni senza nome che possono essere assegnate a variabili. Assegnandole a variabili, puoi riutilizzarle ovunque la variabile sia accessibile.

:::interactive_editor

```js
const multiplyNumbers = function(firstNumber, secondNumber) {
  return firstNumber * secondNumber;
};

console.log(multiplyNumbers(4, 5)); // Output: 20
```

:::

## Funzioni freccia

- Le funzioni freccia sono un modo più conciso per scrivere funzioni in JavaScript.

:::interactive_editor

```js
const calculateArea = (length, width) => {
  const area = length * width;
  return `The area of the rectangle is ${area} square units.`;
};

console.log(calculateArea(5, 10)); // Output: "The area of the rectangle is 50 square units."
```

:::

- Quando definisci una funzione freccia, non serve la parola chiave `function`.
- Se usi un solo parametro, puoi omettere le parentesi tonde intorno alla lista dei parametri.

:::interactive_editor

```js
const cube = x => {
  return x * x * x;
};

console.log(cube(3)); // Output: 27
```

:::

- Se il corpo della funzione consiste in una singola espressione, puoi omettere le parentesi graffe e la parola chiave `return`.

:::interactive_editor

```js
const square = number => number * number;

console.log(square(5)); // Output: 25
```

:::

## Ambito nella programmazione

- **Ambito globale**: è l'ambito più esterno in JavaScript. Le variabili dichiarate nell'ambito globale sono accessibili da qualsiasi punto del codice e si chiamano variabili globali.
- **Ambito locale**: si riferisce alle variabili dichiarate all'interno di una funzione. Queste variabili sono accessibili solo all'interno della funzione in cui sono dichiarate e si chiamano variabili locali.
- **Ambito di blocco**: un blocco è un insieme di istruzioni racchiuse tra parentesi graffe `{}` come nelle istruzioni `if` o nei loop.
- L'ambito di blocco con `let` e `const` offre un controllo ancora più preciso sull'accessibilità delle variabili, aiutando a prevenire errori e rendere il codice più prevedibile.
