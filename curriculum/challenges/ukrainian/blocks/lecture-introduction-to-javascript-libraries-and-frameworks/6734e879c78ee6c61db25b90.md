# --description--

Компоненти — це будівельні блоки застосунків React, які дозволяють розробникам розбивати складні інтерфейси користувача на менші, керовані частини, що полегшує розробку та підтримку масштабних застосунків.

Існують два типи компонентів у React: функціональні та класові компоненти. У сучасному React розробники використовують функціональні компоненти, і всі приклади, які ми розглянемо сьогодні, будуть функціональними компонентами.

На більш загальному рівні можна уявити компоненти як JavaScript-функції, які повертають елементи, що описують інтерфейс користувача.

Цей інтерфейс описується за допомогою JSX — розширення синтаксису для JavaScript, яке виглядає схожим на HTML, але дозволяє писати елементи інтерфейсу більш декларативно.

Давайте розглянемо приклад компонента React:

```jsx
function Greeting() {
  const name = "John"
  {/* The result will be Hello John*/}
  return <h1 className="title">Hello {name}</h1>;
}
```

У цьому прикладі ми визначили компонент з назвою `Greeting`. Фігурні дужки `{}`, розташовані всередині тегів `h1`, дозволяють вставляти JavaScript-вирази, що дає змогу отримати доступ до змінної `name` всередині елемента `h1`.

Ми також застосовуємо `className` під назвою `title` до елемента `h1`.

Але чому ми використовуємо `className` замість `class`, як у звичайних HTML-елементах?

Це тому, що в JavaScript `class` — зарезервоване ім’я. Тож нам потрібно використовувати `className` замість нього.

Ми також використовуємо коментар у JSX, який показує, який буде результат. Важливо зазначити, що ви можете використовувати звичайний синтаксис коментарів, але його потрібно обгорнути у фігурні дужки, щоб він працював:

```jsx
{/* Block Comments */}
```

Ще одна річ, яку ви могли помітити, — це те, що ми використовуємо велику літеру на початку назви компонента. Але чому ми не можемо використовувати всі літери в нижньому регістрі?

Це тому, що React розглядає компоненти з великої літери як кастомні компоненти, тоді як елементи з маленьких літер вважаються вбудованими HTML-елементами.

Коли React зустрічає тег у нижньому регістрі, наприклад `<div>` або `<span>`, він припускає, що це стандартний HTML-елемент. Однак якщо назва компонента починається з великої літери, React розглядатиме його як користувацький компонент і відтворюватиме відповідно. Це розмежування допомагає React відрізняти нативні HTML-теги від компонентів, які ви створюєте.

Щоб використати цей компонент `Greeting` у нашому застосунку, ми написали б щось на кшталт цього:

```jsx
<Greeting />
```

Це відтворить елемент `h1` зі текстом `Hello John` на сторінці. Але придивіться уважніше до синтаксису. Коли ми використовуємо компонент, він закінчується косою рискою і символом більше.

При роботі з JSX усі теги та використання компонентів потрібно явно закривати. Тож якщо компонент або тег не має дочірніх елементів, його потрібно явно закрити, як показано тут:

```jsx
<Greeting /> {/* /> is required */}
```

Поки що ми розглядали, як відтворити один елемент `h1`. Але насправді можна відтворювати кілька елементів.

Давайте подивимося на наступний приклад коду:

```jsx
function Greeting() {
  const name = "John";
  {/* This will throw an error */}
  return <h1>Hello {name}</h1> 
  <p>Nice to meet you.</p>
}
```

Ми намагаємося додати ще одне речення `Nice to meet you`, але воно не відтворюється на сторінці правильно.

Замість цього з’являється повідомлення про помилку. Повідомлення каже `Adjacent JSX elements must be wrapped in an enclosing tag.`

Причина цієї помилки в тому, що кілька братських елементів потрібно обгорнути в батьківський елемент. Хоча ви могли б обгорнути елементи `h1` і `p` у простий `div`, існує інший спосіб уникнути цієї помилки.

React-фрагменти використовуються для групування елементів разом. Ось як виглядатиме виправлений приклад:

```jsx
function Greeting() {
  const name = "John";
  return (
    <Fragment>
      <h1>Hello {name}</h1>
      <p>Nice to meet you.</p>
    </Fragment>
  );
}
```

Ви також можете використати порожні JSX-теги, які слугують скороченням для фрагментів:

```jsx
function Greeting() {
  const name = "John";
  return (
    <>
      <h1>Hello {name}</h1>
      <p>Nice to meet you.</p>
    </>
  );
}
```

У майбутніх уроках ми продовжимо вивчати, як працювати з компонентами та JSX. А поки що ви отримали міцне введення у створення інтерфейсів користувача за допомогою компонентів, заклавши міцний фундамент для подальшого навчання.
