# --description--

Коли ви встановлюєте залежності за допомогою npm, особливо в існуючому проєкті, npm дивиться у ваш файл `package.json`, щоб дізнатися, які пакети потрібні вашому проєкту.

Кожна залежність у `package.json` містить номер версії, наприклад:

```bash
"lodash": "^4.17.21"
```

Як ви вже дізналися раніше, **карет (^)** дозволяє npm встановлювати новіші мінорні та патч-версії, як `4.18.0` або `4.17.22`. Така гнучкість корисна, але може також призводити до проблем.

Якщо двоє розробників запускають `npm install` у різний час, npm може встановити трохи різні версії одного й того самого пакета. Навіть невеликі відмінності можуть спричинити помилки або непередбачену поведінку.

Файл `package-lock.json` вирішує цю проблему. Він створюється автоматично вперше, коли ви запускаєте `npm install`, і фіксує точні версії кожної встановленої залежності — включно з усіма дочірніми залежностями, тобто залежностями від залежностей.

Це гарантує, що всі, хто працює над проєктом, отримують однакові версії пакетів щоразу.

Ось простий файл `package.json` з `lodash` як залежністю:

```bash
{
  "name": "package-lock-json",
  "version": "1.0.0",
  "description": "",
  "main": "index.js",
  "scripts": {
    "test": "echo \"Error: no test specified\" && exit 1"
  },
  "keywords": [],
  "author": "",
  "license": "ISC",
  "type": "commonjs",
  "dependencies": {
    "lodash": "^4.17.21"
  }
}
```

А ось відповідний розділ з `package-lock.json`:

```bash
"node_modules/lodash": {
  "version": "4.17.21",
  "resolved": "https://registry.npmjs.org/lodash/-/lodash-4.17.21.tgz",
  "integrity": "sha512-v2kDEe57..."
}
```

Зверніть увагу на відмінність:

* У `package.json` `lodash` вказаний як `^4.17.21` — гнучке правило.
    
* У `package-lock.json` `lodash` зафіксований на версії `4.17.21`.
    

Незалежно від того, коли хтось запускає `npm install`, він отримає саме цю версію.

Таким чином, незалежно від того, коли хтось запускає `npm install`, він щоразу отримає ту саму версію Lodash.

Зверніть увагу, що це також стосується дочірніх залежностей, тобто залежностей будь-яких залежностей, які ви використовуєте у своєму проєкті.

Ви можете побачити, що окрім самого Express, є інші пакети, як `accepts`, `mime-types` і `negotiator`. Це дочірні залежності — пакети, від яких внутрішньо залежить Express для своєї роботи.

Також зверніть увагу, що хоча `dependencies` у `node_modules/express` використовують символи карет для діапазонів, якщо ви знайдете запис для кожної дочірньої залежності, як `node_modules/etags`, вони зафіксовані на конкретній версії.

Коротко кажучи, файл `package-lock.json` тримає всіх на одній хвилі і забезпечує послідовність між машинами розробників, CI-серверами, розгортаннями та іншими середовищами.

Важливо завжди комітити свій файл `package-lock.json` у Git, щоб ваша команда мала однакове налаштування. Також не редагуйте файл `package-lock.json` вручну — npm оновлюватиме його автоматично щоразу, коли змінюються залежності.
