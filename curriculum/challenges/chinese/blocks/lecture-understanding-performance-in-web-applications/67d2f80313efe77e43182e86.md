# --description--

让我们回顾一些专门用于提升 JavaScript 性能的最重要技术。

你可能会感到惊讶，但第一个技巧是在可能的情况下尽量少用 JavaScript。

如果你正在开发一个简单的静态网站，你可能能够在不依赖 JavaScript 的框架的情况下做到完全相同。

如果可能，你还应该尝试删除任何未使用的 JavaScript 代码并使用更简单的方案。

例如，你可以考虑使用内置浏览器特色，而不是自定义特色。这些内置浏览器特色包括表单验证、浏览器内置视频播放器以及 CSS 动画，而不是自定义 JavaScript 动画。

尽可能少使用动画。

减少你的 JavaScript 代码中的 DOM 操作，因为这在计算上代价高昂。

将你的 JavaScript 代码拆分为执行关键任务和非关键任务的模块也很有帮助。这样，你就能尽快预装载关键模块，并推迟非关键模块，以尽可能快速地渲染页面。

预先装载文件并不保证你使用时它们可用，但下载会更早开始，因此整体等待时间应更短。

一旦你的代码被拆分成多个文件，你可以在构建进程中自动压缩它们，以减小体积并优化装载时间。

如果某个事件监听器不再需要，你应该将其移除：

```js
element.removeEventListener("mousemove", handleMouseMove);
```

在此示例中，我们移除对 `mousemove` 事件的监听器，因此当事件被触发时，不再调用 `handleMouseMove` 函数。如果你保留事件监听器，程序将继续执行不再必要的计算，影响整体性能。如果你同时有多个活动事件监听器，这种效果会更加明显。

既然我们正在讨论事件，建议尽可能使用事件委托。

这意味着你不需要在单个子元素上设置事件监听器。你可以直接将其设置在父元素上，事件会冒泡直到找到可以处理它的父元素。

优化 JavaScript 对于开发快速且响应迅速的网页应用至关重要。在真实的项目中，你将不断分析你的网站性能，以查找需要改进的地方。
