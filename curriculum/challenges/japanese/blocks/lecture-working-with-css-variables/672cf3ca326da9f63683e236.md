# --interactive--

`@property` ルールは、開発者がカスタムプロパティの動作をより細かく制御できる強力な CSS 機能で、アニメーションの仕方や初期値の設定も含まれます。

このルールは CSS カスタムプロパティの機能を拡張し、より柔軟な適用を可能にします。

`@property` ルールの基本的な構文は以下の通りです。

```css
@property --property-name {
  syntax: '<type>';
  inherits: true | false;
  initial-value: <value>;
}
```

`--property-name` は定義するカスタムプロパティの名前です。すべてのカスタムプロパティと同様に、必ず二つのダッシュで始まらなければなりません。

`syntax` はプロパティの型を定義し、`<color>`、`<length>`、`<number>`、`<percentage>` などや、より複雑な型を指定できます。

`inherits` はプロパティが親要素から値を継承するかどうかを指定します。

`initial-value` はプロパティのデフォルト値を設定します。

`@property` ルールを使った実用的な例を見てみましょう。

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<button class="button">Click Me</button>
```

```css
@property --main-color {
  syntax: '<color>';
  inherits: false;
  initial-value: #3498db;
}

.button {
  background-color: var(--main-color);
}
```

:::

この例では、カスタムプロパティ `--main-color` を `color` 型として定義し、継承しない設定にして、初期値を `#3498db` にしています。

`@property` ルールの主な利点の一つは、カスタムプロパティのアニメーションが可能になることです。これは以前はできませんでした。

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<div class="gradient-box"></div>
```

```css
@property --gradient-angle {
  syntax: '<angle>';
  inherits: false;
  initial-value: 0deg;
}

.gradient-box {
  width: 100px;
  height: 100px;
  background: linear-gradient(var(--gradient-angle), red, blue);
  transition: --gradient-angle 0.5s;
}

.gradient-box:hover {
  --gradient-angle: 90deg;
}
```

:::

上記のコードは、要素にホバーしたときに滑らかにアニメーションするグラデーションを作成します。これは標準のカスタムプロパティでは実現できませんでした。

次に、`@property` ルールがフォールバックとどのように連携するかを説明します。

フォールバックは、特定の機能をサポートしないブラウザでもスタイルが適切に表示されるようにするために重要です。`@property` では、ルール自体とカスタムプロパティの使用の両方でフォールバックが機能します。

`@property` ルールをサポートしないブラウザ向けには、従来の方法でカスタムプロパティを宣言してフォールバックを提供できます。

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
```

```css
:root {
  --main-color: #3498db;
}

@property --main-color {
  syntax: '<color>';
  inherits: false;
  initial-value: #3498db;
}

body {
  background-color: var(--main-color);
}
```

:::

この場合、`@property` をサポートするブラウザはより厳密に定義されたバージョンを使用し、それ以外は標準のカスタムプロパティ宣言にフォールバックします。

カスタムプロパティを使用するときは、標準のカスタムプロパティと同様に `var()` 関数を使ってフォールバック値を指定できます。

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<button class="button">Click Me</button>
```

```css
.button {
  background-color: var(--main-color, #3498db);
}
```

:::

これにより、`--main-color` プロパティが定義されていないか無効な場合でも、ボタンに背景色が設定されます。

`@property` ルールは、より複雑なフォールバックのシナリオも可能にします。例えば、型安全なフォールバックを提供することもできます。

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<div class="box">Colored box</div>
```

```css
@property --padding {
  syntax: '<length>';
  inherits: false;
  initial-value: 10px;
}

.box {
  width: 100px;
  height: 100px;
  background-color: darkred;
  color: white;
  padding: var(--padding);
}
```

:::

この場合、`--padding` に色のような無効な値を設定しようとすると、ブラウザは `10px` の初期値にフォールバックし、型の安全性を保ちます。

CSS `@property` ルールは、2024年初頭から Chrome、Edge、Opera、Firefox、Safari など主要なモダンブラウザのすべてで広く利用可能とされています。より古いブラウザをサポートする場合は、上記のようにフォールバック CSS を用意する必要があります。

まとめると、`@property` ルールは CSS のカスタムプロパティの扱いにおいて大きな進歩を示しています。より細かい制御が可能になり、カスタムプロパティのアニメーションなど新しい可能性を開きます。ただし、適切なフォールバックを必ず用意して、すべてのブラウザで良好な体験を提供することが重要です。最先端のウェブ技術では、対応ブラウザの体験を向上させつつ、対応していないブラウザでも基本機能を確保することが鍵となります。
