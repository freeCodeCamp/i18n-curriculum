# --description--

어떤 웹사이트는 빠르고 반응형으로 느껴지는데, 어떤 사이트는 느리게 느껴지는 이유가 무엇일까요? 그 답은 페이지가 로드되고 렌더링되는 방식에 영향을 주는 주요 성능 개념에 있습니다. 빠르고 부드러우며 사용자 친화적인 웹사이트를 만들려면 주요 웹 성능 개념을 이해하는 것이 필수입니다.

소스 순서, 중요 렌더링 경로, 지연 시간 등을 살펴보겠습니다.

먼저 소스 순서부터 살펴보겠습니다.

소스 순서는 HTML 요소가 문서 내에서 구조화된 방식을 의미합니다. 이것이 무엇이 먼저 로드되는지를 결정하며 성능과 접근성에 큰 영향을 미칩니다.

몇 가지 모범 사례는 다음과 같습니다:

- 제목, 내비게이션 또는 주요 텍스트 같은 중요한 콘텐츠를 HTML 구조에서 상단에 배치하세요.

- 분석용 스크립트나 서드파티 위젯 같은 필수적이지 않은 스크립트는 렌더링을 차단하지 않도록 지연시키세요.

- 점진적 향상(progressive enhancement)을 사용해 스타일과 스크립트가 로드되기 전에도 핵심 경험이 작동하도록 하세요. 점진적 향상은 HTML을 먼저 작동시키는 것을 기반으로 웹사이트와 애플리케이션을 구축하는 방법입니다.

다음은 방금 살펴본 모범 사례를 적용한 좋은 소스 순서 예시입니다.

```html
<!-- Good source order: Essential content first -->
<h1>Welcome to FastSite!</h1>
<p>Critical information loads first.</p>
<script src="slow-script.js" defer></script>
```

소스 순서를 최적화하면 사용자가 중요한 콘텐츠를 가능한 빨리 볼 수 있게 됩니다.

이제 중요 렌더링 경로를 살펴보겠습니다.

중요 렌더링 경로는 브라우저가 코드를 화면의 픽셀로 변환하기 위해 따르는 단계의 수열입니다.

나중에 자세히 다룰 주요 단계는 다음과 같습니다:

1. HTML 파싱: DOM(Document Object Model)을 구축합니다.

2. CSS 파싱: CSSOM(CSS Object Model)을 구축합니다.

3. JavaScript 실행: DOM과 CSSOM을 수정할 수 있습니다.

4. 렌더 트리 구성: DOM과 CSSOM을 결합합니다.

5. 레이아웃 및 페인팅: 요소 크기를 결정하고 픽셀을 그립니다.

6. 최적화:

    - 렌더링을 차단하는 리소스(예: 큰 CSS 파일, 사용하지 않는 JS)를 최소화하세요.

    - 스크립트에 async와 defer 속성을 사용하세요.

    - 필수 스타일만 먼저 로드하고, 중요하지 않은 CSS는 지연시키세요.

전반적으로, 중요 렌더링 경로가 짧을수록 체감 성능이 빨라집니다. 이 부분은 나중에 더 자세히 다루겠습니다.

마지막으로 지연 시간을 살펴보겠습니다.

지연 시간은 브라우저와 서버 간 요청이 이동하는 데 걸리는 시간입니다. 즉, 지연 시간이 높으면 페이지가 느려집니다.

지연 시간을 줄이는 방법은 다음과 같습니다:

- CDN(콘텐츠 전송 네트워크)을 사용해 파일을 더 가까운 위치에서 제공하세요.

- Gzip 같은 압축을 활성화해 파일 크기를 줄이세요.

- 이미지를 최적화하고 지연 로딩(lazy loading)을 사용하세요 — 이 부분도 나중에 다룰 예정입니다.

지연 시간을 줄이면 상호작용이 즉각적으로 느껴집니다.

결론적으로, 소스 순서를 최적화하고, 중요 경로를 줄이며, 지연 시간을 단축하면 웹사이트가 빠르게 느껴지도록 만들 수 있습니다.
