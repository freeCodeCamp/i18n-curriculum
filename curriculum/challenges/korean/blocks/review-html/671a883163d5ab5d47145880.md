## HTML 기본

- **HTML의 역할**: HTML(Hypertext Markup Language)은 웹 구조의 기초로, 웹 페이지의 요소를 정의합니다.
- **HTML 요소**: 페이지의 내용을 나타내는 데 사용됩니다. 대부분은 여는 태그와 닫는 태그로 구성됩니다(예: `<h1></h1>`, `<p></p>`).
- **HTML 구조**: HTML은 `head`와 `body`로 구성되며, 메타데이터, 스타일, 콘텐츠가 구조화됩니다.
- **일반적인 HTML 요소**: 제목(`<h1>` - `<h6>`), 단락(`<p>`), 그리고 div 컨테이너(`<div>`).
- **`div` 요소**: `div` 요소는 의미론적 의미가 없는 일반적인 HTML 요소입니다. 다른 HTML 요소를 담는 일반 컨테이너로 사용됩니다.
- **Void 요소**: 닫는 태그가 없는 요소(예: `<img>`).
- **속성**: 요소에 메타데이터와 동작을 추가합니다.

## 식별자와 그룹화

- **ID**: 고유한 요소 식별자입니다.
- **클래스**: 스타일과 동작을 위해 요소를 그룹화합니다.

## 특수 문자와 링크

- **HTML 엔티티**: `&amp;`, `&lt;` 같은 특수 문자를 사용할 때.
- **`link` 요소**: 외부 스타일시트를 연결할 때 사용합니다.
- **`script` 요소**: 외부 JavaScript 파일을 임베드할 때 사용합니다.

## 보일러플레이트와 인코딩

- **HTML 보일러플레이트**: 웹 페이지의 기본 구조로, `DOCTYPE`, `html`, `head`, `body` 요소를 포함합니다. HTML 문서의 시작점으로 사용해야 합니다.
- **UTF-8 문자 인코딩**: 모든 문자가 올바르게 표시되도록 보장합니다.

## SEO와 소셜 공유

- **메타 태그(`description`)**: 웹 페이지에 대한 간단한 설명을 제공하며 SEO에 영향을 줍니다.
- **Open Graph 태그**: 소셜 미디어 공유를 향상시킵니다.

## 미디어 요소와 최적화

- **교체된 요소**: 이미지, iframe 같은 임베드된 콘텐츠입니다.
- **미디어 최적화**: 미디어 성능을 향상시키는 기법입니다.
- **이미지 포맷과 라이선스**: 사용 권한과 유형을 이해합니다.
- **SVG**: 선명한 시각 효과를 위한 확장 가능한 벡터 그래픽입니다.

## 멀티미디어 통합

- **HTML 오디오 및 비디오 요소**: 멀티미디어를 임베드합니다.
- **`<iframe>`로 임베드하기**: 외부 비디오 콘텐츠를 통합합니다.

## 경로와 링크 동작

- **target 속성 유형**: 링크 동작을 제어합니다.
- **절대 경로와 상대 경로**: 디렉터리 탐색 방법입니다.
- **경로 구문**: 파일 탐색을 위한 `/`, `./`, `../` 이해하기.
- **링크 상태**: 다양한 링크 상호작용(호버, 활성 등)을 관리합니다.
- **내부 링크**: `href="#id"` 요소에 `a`를 사용해 페이지 내 다른 섹션으로 연결하고, 대상 요소에 같은 `id`를 부여합니다.

## 의미론적 HTML의 중요성

- **제목 요소의 구조적 계층**: 콘텐츠의 구조적 계층을 유지하려면 올바른 제목 요소를 사용해야 합니다. `h1` 요소가 가장 높은 수준의 제목이고 `h6` 요소가 가장 낮은 수준의 제목입니다.
- **표현적 HTML 요소**: 콘텐츠의 외관을 정의하는 요소로, 더 이상 권장되지 않는 `center`, `big`, `font` 요소가 있습니다.
- **의미론적 HTML 요소**: 콘텐츠 구조에 의미를 부여하는 요소입니다. 예시는 다음과 같습니다:
  - `<header>`: 소개 콘텐츠를 나타냅니다.
  - `<nav>`: 내비게이션 링크를 포함합니다.
  - `<article>`: 독립적인 콘텐츠를 나타냅니다.
  - `<aside>`: 사이드바나 관련 콘텐츠에 사용됩니다.
  - `<section>`: 문서 내 관련 콘텐츠를 그룹화합니다.
  - `<footer>`: 섹션이나 문서의 푸터를 정의합니다.

## 의미론적 HTML 요소

- **강조(`em`) 요소**: 강조된 텍스트를 표시합니다.
- **관용구 텍스트(`i`) 요소**: 대체 음성이나 기분, 다른 언어의 관용구, 기술 용어, 생각을 강조할 때 사용합니다.
- **강한 중요성(`strong`) 요소**: 강한 중요성을 가진 텍스트를 표시합니다.
- **주의 환기(`b`) 요소**: 콘텐츠 의미에 중요하지 않은 텍스트에 주의를 끌 때 사용합니다.
- **설명 목록(`dl`) 요소**: 용어-설명 그룹 목록을 나타냅니다.
- **설명 용어(`dt`) 요소**: 정의되는 용어를 나타냅니다.
- **설명 세부사항(`dd`) 요소**: 용어에 대한 설명을 나타냅니다.
- **블록 인용(`blockquote`) 요소**: 다른 출처에서 인용한 구간을 나타냅니다.
- **인라인 인용(`q`) 요소**: 짧은 인라인 인용을 나타냅니다.
- **약어(`abbr`) 요소**: 약어나 두문자어를 나타냅니다.
- **연락처 주소(`address`) 요소**: 연락처 정보를 나타냅니다.
- **(날짜) 시간(`time`) 요소**: 날짜 및/또는 시간을 나타냅니다.
- **위 첨자(`sup`) 요소**: 위 첨자 텍스트를 나타냅니다.
- **아래 첨자(`sub`) 요소**: 아래 첨자 텍스트를 나타냅니다.
- **인라인 코드(`code`) 요소**: 컴퓨터 코드 조각을 나타냅니다.
- **비표현 주석(`u`) 요소**: 비텍스트 주석이 있음을 나타내는 인라인 텍스트 구간을 나타냅니다.
- **루비 주석(`ruby`) 요소**: 루비 주석 텍스트를 나타냅니다.
- **취소선(`s`) 요소**: 더 이상 정확하지 않거나 관련 없는 콘텐츠를 나타냅니다.

## HTML 폼 요소와 속성

### 폼

- **`form` 요소**: 사용자 입력을 위한 HTML 폼을 만듭니다.
- **일반 입력 유형**: `text`, `email`, `password`, `radio`, `checkbox`, `number`, `date`.
- **`action` 속성**: 폼 데이터를 보낼 URL을 지정합니다.
- **`method` 속성**: 폼 데이터를 보낼 때 사용할 HTTP 메서드를 지정합니다. 가장 일반적인 메서드는 `GET`와 `POST`입니다.

```html
<form method="value-goes-here" action="url-goes-here">
  <!-- inputs go inside here -->
</form>
```

- **`input` 요소**: 사용자 입력을 위한 입력 필드를 만듭니다.
- **`type` 속성**: 입력 필드 유형을 지정합니다(예: `text`, `email`, `number`, `radio`, `checkbox`).
- **`placeholder` 속성**: 입력 필드에 무엇을 입력해야 하는지 사용자에게 힌트를 보여줍니다.
- **`value` 속성**: 입력 값(value)을 지정합니다. 입력 유형이 `button`인 경우, `value` 속성으로 버튼 텍스트를 설정할 수 있습니다.
- **`name` 속성**: 입력 필드에 이름을 할당하며, 폼 제출 시 키 역할을 합니다. 라디오 버튼의 경우 같은 `name`를 주면 그룹화되어 한 번에 하나만 선택할 수 있습니다.
- **`size` 속성**: 사용자가 입력할 때 표시할 문자 수를 정의합니다.
- **`min` 속성**: `number` 같은 입력 유형과 함께 최솟값을 지정할 수 있습니다.
- **`max` 속성**: `number` 같은 입력 유형과 함께 최댓값을 지정할 수 있습니다.
- **`minlength` 속성**: 입력 필드에 필요한 최소 문자 수를 지정합니다.
- **`maxlength` 속성**: 입력 필드에 허용되는 최대 문자 수를 지정합니다.
- **`required` 속성**: 폼 제출 전에 입력 필드를 반드시 채워야 함을 지정합니다.
- **`disabled` 속성**: 입력 필드를 비활성화합니다.
- **`readonly` 속성**: 입력 필드를 읽기 전용으로 지정합니다.

:::interactive_editor

```html
<!-- Text input -->
<input 
  type="text"
  id="name"
  name="name"
  placeholder="e.g. Quincy Larson"
  size="20"
  minlength="5"
  maxlength="30"
  required
/>

<!-- Number input -->
<input 
  type="number"
  id="quantity"
  name="quantity"
  min="2"
  max="10"
  disabled
/>

<!-- Button -->
<input type="button" value="Show Alert" />
```

:::

- **`label` 요소**: 입력 필드에 대한 레이블을 만듭니다.
- **`for` 속성**: 레이블이 어떤 입력 필드에 해당하는지 지정합니다.
- **암묵적 폼 연관**: 입력 필드를 `label` 요소 안에 감싸 레이블과 연관시킬 수 있습니다.

:::interactive_editor

```html
<form action="">
  <label>
    Full Name:
    <input type="text" />
  </label>
</form>
```

:::

- **명시적 폼 연관**: `for` 요소에 `label` 속성을 사용해 입력과 레이블을 연관시킬 수 있습니다.

:::interactive_editor

```html
<form action="">
  <label for="email">Email Address: </label>
  <input type="email" id="email" />
</form>
```

:::

- **`button` 요소**: 클릭 가능한 버튼을 만듭니다. 버튼은 `type` 속성을 가질 수 있으며, 활성화 시 동작을 제어합니다(예: `submit`, `reset`, `button`).

:::interactive_editor

```html
<button type="button">Show Form</button>
<button type="submit">Submit Form</button>
<button type="reset">Reset Form</button>
```

:::

- **`fieldset` 요소**: 관련 입력들을 그룹화합니다.
- **`legend` 요소**: 입력 그룹을 설명하는 캡션을 추가합니다.

:::interactive_editor

```html
<!-- Radio group -->
<fieldset>
  <legend>Was this your first time at our hotel?</legend>

  <input id="yes-option" type="radio" name="hotel-stay" value="yes" />
  <label for="yes-option">Yes</label>

  <input id="no-option" type="radio" name="hotel-stay" value="no" />
  <label for="no-option">No</label>
</fieldset>

<!-- Checkbox group -->
<fieldset>
  <legend>
    Why did you choose to stay at our hotel? (Check all that apply)
  </legend>

  <input type="checkbox" id="location" name="location" value="location" />
  <label for="location">Location</label>

  <input type="checkbox" id="price" name="price" value="price" />
  <label for="price">Price</label>
</fieldset>
```

:::

- **포커스 상태**: 사용자가 선택한 입력 필드의 상태입니다.

## HTML 표 요소와 속성 다루기

- **표 요소**: HTML 표를 만듭니다.
- **표 헤드(`thead`) 요소**: 표의 헤더 콘텐츠를 그룹화합니다.
- **표 행(`tr`) 요소**: 표에 행을 만듭니다.
- **표 헤더 셀(`th`) 요소**: 표의 헤더 셀을 만듭니다.
- **표 본문(`tbody`) 요소**: 표의 본문 콘텐츠를 그룹화합니다.
- **표 데이터 셀(`td`) 요소**: 표의 데이터 셀을 만듭니다.
- **표 푸터(`tfoot`) 요소**: 표의 푸터 콘텐츠를 그룹화합니다.
- **`caption` 요소**: HTML 표의 제목을 추가합니다.
- **`colspan` 속성**: 표 셀이 차지할 열 수를 지정합니다.

:::interactive_editor

```html
<table>
  <caption>Exam Grades</caption>

  <thead>
    <tr>
      <th>Last Name</th>
      <th>First Name</th>
      <th>Grade</th>
    </tr>
  </thead>

  <tbody>
    <tr>
      <td>Davis</td>
      <td>Alex</td>
      <td>54</td>
    </tr>

    <tr>
      <td>Doe</td>
      <td>Samantha</td>
      <td>92</td>
    </tr>

    <tr>
      <td>Rodriguez</td>
      <td>Marcus</td>
      <td>88</td>
    </tr>
  </tbody>

  <tfoot>
    <tr>
      <td colspan="2">Average Grade</td>
      <td>78</td>
    </tr>
  </tfoot>
</table>
```

:::

## HTML 도구

- **HTML 검증기**: HTML 코드의 구문이 올바른지 검사하는 도구입니다.
- **DOM 검사기**: 웹 페이지의 HTML 구조를 검사하고 수정할 수 있는 도구입니다.
- **개발자 도구**: 브라우저에 내장된 웹 개발자 도구 모음으로, 웹 페이지를 디버그, 프로파일링, 분석하는 데 도움을 줍니다.

## 접근성 소개

- **웹 콘텐츠 접근성 지침**: 웹 콘텐츠 접근성 지침(WCAG)은 장애가 있는 사람이 웹 콘텐츠에 더 쉽게 접근할 수 있도록 만든 지침 모음입니다. WCAG의 네 가지 원칙은 인지 가능(Perceivable), 조작 가능(Operable), 이해 가능(Understandable), 견고함(Robust)을 뜻하는 POUR입니다.

## 접근성을 위한 보조 기술

- **화면 읽기 프로그램**: 컴퓨터 화면의 내용을 소리 내어 읽어주는 소프트웨어로, 시각 장애인이나 저시력자가 웹에 접근할 때 사용합니다.
- **큰 글자 또는 점자 키보드**: 시각 장애인이 웹에 접근할 때 사용합니다.
- **화면 확대기**: 컴퓨터 화면 내용을 확대하는 소프트웨어로, 저시력자가 웹에 접근할 때 사용합니다.
- **대체 포인팅 장치**: 이동성 장애인이 웹에 접근할 때 사용하는 조이스틱, 트랙볼, 터치패드 같은 장치입니다.
- **음성 인식**: 이동성 장애인이 음성으로 컴퓨터를 제어할 수 있게 해 웹에 접근할 때 사용합니다.

## 접근성 감사 도구

- **일반적인 접근성 도구**: Google Lighthouse, Wave, IBM Equal Accessibility Checker, axe DevTools 등이 웹사이트 접근성 감사를 위해 자주 사용됩니다.

## 접근성 모범 사례

- **적절한 제목 수준 구조**: 콘텐츠에 논리적 구조를 만들기 위해 적절한 제목 수준을 사용해야 합니다. 이는 보조 기술이 웹사이트 콘텐츠를 이해하는 데 도움을 줍니다.
- **접근성과 표**: 표를 사용할 때는 헤더 셀을 정의하는 `th` 요소와 데이터 셀을 정의하는 `td` 요소를 사용해야 합니다. 이는 보조 기술이 표 구조를 이해하는 데 도움을 줍니다.
- **입력에 레이블이 반드시 있어야 하는 중요성**: 레이블과 폼 입력을 연결하는 `label` 요소를 사용해야 합니다. 이는 보조 기술이 입력 목적을 이해하는 데 도움을 줍니다.
- **좋은 `alt` 텍스트의 중요성**: 이미지에 대한 텍스트 대체물을 제공하는 `alt` 속성을 사용해야 합니다. 이는 보조 기술이 이미지 내용을 이해하는 데 도움을 줍니다.
- **좋은 링크 텍스트의 중요성**: 링크 목적을 이해할 수 있도록 설명적인 링크 텍스트를 사용해야 합니다.
- **오디오 및 비디오 콘텐츠 접근성 모범 사례**: 청각 장애인을 위해 캡션과 대본을 제공해야 하며, 시각 장애인을 위해 비디오에 대한 오디오 설명도 제공해야 합니다.
- **`tabindex` 속성**: 요소를 포커스 가능하게 만들고 키보드 탐색 순서를 정의합니다. 값이 0보다 큰 `tabindex` 속성은 절대 사용하지 말고, 0 또는 -1 값을 사용해야 합니다.
- **`accesskey` 속성**: 요소에 대한 키보드 단축키를 정의합니다. 이는 이동성 장애인이 웹사이트를 더 쉽게 탐색하는 데 도움을 줍니다.

## WAI-ARIA, 역할, 속성

- **WAI-ARIA**: 웹 접근성 이니셔티브 - 접근 가능한 리치 인터넷 애플리케이션의 약자로, HTML 요소에 추가해 접근성을 향상시키는 속성 모음입니다. 보조 기술에 콘텐츠 목적과 구조에 대한 추가 정보를 제공합니다.
- **ARIA 역할**: HTML 요소에 추가해 목적을 정의하는 미리 정의된 역할 모음입니다. 보조 기술이 웹사이트 콘텐츠를 이해하는 데 도움을 줍니다. 예로 `role="tab"`, `role="menu"`, `role="alert"`가 있습니다.
- **`aria-label`와 `aria-labelledby` 속성**: 요소에 프로그램적(또는 접근 가능한) 이름을 부여해 화면 읽기 프로그램 같은 보조 기술이 요소 목적을 이해하도록 돕습니다. 시각적 레이블이 이미지나 기호일 때 자주 사용합니다. `aria-label`는 이름을 속성에 직접 정의하고, `aria-labelledby`는 페이지 내 기존 텍스트를 참조합니다.
- **`aria-hidden` 속성**: 화면 읽기 프로그램 같은 보조 기술에서 요소를 숨길 때 사용합니다. 예를 들어 의미 없는 장식용 이미지를 숨길 때 씁니다.
- **`aria-describedby` 속성**: 요소에 대한 추가 정보를 포함하는 다른 요소와 연관시켜 보조 기술이 요소 목적을 이해하도록 돕습니다.
