# --interactive--

`minmax()` 関数はグリッドトラックのサイズ範囲を定義し、行や列が占めることができるスペースの量を指定します。

トラックサイズは `px`（ピクセル）、`rem`、あるいは `em` のような単位や、分数単位（`fr`）で設定できることを覚えておいてください。

`minmax()` 関数はさらに進んで、グリッドトラックの最小サイズと最大サイズを設定できるようにします。

`minmax()` 関数の構文は次の通りです：

```css
minmax(min, max)
```

`min` はグリッドトラックの最小サイズで、ピクセル、パーセンテージ、または `auto` で設定できます。そして `max` はグリッドトラックの最大サイズで、同じ単位で設定可能です。

この2つの値は次のように連携します：

- `min` の値はグリッドトラックが設定したサイズより小さくならないことを保証します。

- `max` の値はグリッドトラックがどれだけ大きくなるかの上限を制限します。

グリッドトラックのサイズはコンテンツとコンテナのサイズに応じて `min` と `max` の間で動的に調整されます。

実際の例を見てみましょう：

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<div class="grid-container">
  <div>
    <h2>Item 1</h2>
  </div>
  <div>
    <h2>Item 2</h2>
  </div>
</div>
```

```css
.grid-container {
  display: grid;
  grid-template-columns: minmax(150px, 300px) 1fr;
  gap: 20px;
}

.grid-container > div {
  background: crimson;
  padding: 20px;
  text-align: center;
}
```

:::

ここで何が起きているでしょうか？

最初の列、`minmax(150px, 300px)` は利用可能なスペースに応じて常に `150px` 以上、最大で `300px` になります。

一方、2番目の列、`1fr` は他に共有する列がないため、グリッドコンテナ内の残りの利用可能なスペースをすべて占めます。

`minmax()` 関数が固定サイズや `fr` 単位より優れている点は、より柔軟で適応性やレスポンシブ性に優れていることです。
