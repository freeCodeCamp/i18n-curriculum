# --description--

<dfn>String</dfn>-Werte in JavaScript können mit einfachen oder doppelten Anführungszeichen geschrieben werden, solange Sie mit derselben Art von Anführungszeichen beginnen und enden. Im Gegensatz zu einigen anderen Programmiersprachen funktionieren einfache und doppelte Anführungszeichen in JavaScript gleich.

```js
const doubleQuoteStr = "This is a string"; 
const singleQuoteStr = 'This is also a string';
```

Der Grund, warum Sie eine Art von Anführungszeichen der anderen vorziehen könnten, ist, wenn Sie beide in einem String verwenden möchten. Das kann passieren, wenn Sie ein Gespräch in einem String speichern und das Gespräch in Anführungszeichen setzen wollen. Eine weitere Verwendung wäre, einen `<a>` Tag mit verschiedenen Attributen in Anführungszeichen zu speichern, alles innerhalb eines Strings.

```js
const conversation = 'Finn exclaims to Jake, "Algebraic!"';
```

Dies wird jedoch zum Problem, wenn Sie die äußersten Anführungszeichen innerhalb des Strings verwenden müssen. Denken Sie daran, ein String hat am Anfang und Ende dieselbe Art von Anführungszeichen. Wenn Sie aber dasselbe Anführungszeichen irgendwo in der Mitte haben, endet der String zu früh und es wird ein Fehler geworfen.

```js
const goodStr = 'Jake asks Finn, "Hey, let\'s go on an adventure?"'; 
const badStr = 'Finn responds, "Let's go!"';
```

Hier wirft `badStr` einen Fehler.

Im oben gezeigten <dfn>goodStr</dfn> können Sie beide Arten von Anführungszeichen sicher verwenden, indem Sie den Backslash `\` als Escape-Zeichen verwenden.

**Hinweis:** Der Backslash `\` darf nicht mit dem Vorwärtsschrägstrich `/` verwechselt werden. Sie erfüllen nicht dieselbe Funktion.

# --instructions--

Ändern Sie den vorgegebenen String so, dass er am Anfang und Ende einfache Anführungszeichen hat und keine Escape-Zeichen mehr verwendet.

Im Moment verwendet der `<a>` Tag im String überall doppelte Anführungszeichen. Sie müssen die äußeren Anführungszeichen in einfache ändern, damit Sie die Escape-Zeichen entfernen können.

# --hints--

Sie sollten alle Backslashes (`\`) entfernen.

```js
assert(
  !/\\/g.test(__helpers.removeJSComments(code)) &&
    myStr.match(
      '\\s*<a href\\s*=\\s*"http://www.example.com"\\s*target\\s*=\\s*"_blank">\\s*Link\\s*</a>\\s*'
    )
);
```

Sie sollten zwei einfache Anführungszeichen `'` und vier doppelte Anführungszeichen `"` haben.

```js
assert(__helpers.removeJSComments(code).match(/"/g).length === 4 && __helpers.removeJSComments(code).match(/'/g).length === 2);
```
