# --description--

React Server Components (RSCs) sind ein neuer Trend, der die Herangehensweise von React-Entwicklern verändert hat. Mit RSCs verlagert sich mehr Arbeit auf den Server, was viele Vorteile mit sich bringt.

Schauen wir uns an, was Server Components sind, wie sie funktionieren und was zur Einführung von Server Components geführt hat.

React Server Components sind React-Komponenten, die ausschließlich auf dem Server rendern und nur das fertige HTML an den Client senden. Das bedeutet, dass diese Komponenten direkt auf serverseitige Ressourcen zugreifen können und die Menge an JavaScript, die an den Browser gesendet wird, drastisch reduziert wird.

React-Apps haben traditionell ein „Client-Komponenten“-System verwendet, das alles in einer typischen React-App übernimmt, wie Rendering, Interaktivität und Nebeneffekte. Der Begriff „Client-Komponente“ wurde erst mit der Einführung von React Server Components kürzlich häufiger verwendet.

Das Client-Komponenten-System hat jedoch einige Nachteile, wie große JavaScript-Bündel und langsamere Ladezeiten beim ersten Laden.

React-Frameworks wie Next.js und Gatsby fanden Workarounds, um einige Prozesse auf den Server auszulagern, um diese Probleme zu beheben, aber keiner davon war standardisiert. Wenn Sie eines der Frameworks verwendet haben, haben Sie wahrscheinlich von `getServerSideProps` und `getServerData` gehört.

Dann kamen React Server Components, die es ermöglichen, einige Komponenten vollständig auf dem Server auszuführen, sodass Sie Dinge wie Datenabruf und Berechnungen erledigen können, bevor irgendein Code im Browser des Nutzers ausgeführt wird.

Server Components wurden zuerst durch Next.js populär gemacht und sind dort sofort verfügbar. Andere Frameworks wie Remix und Gatsby holen auf, und es gibt ein experimentelles Plugin für Vite namens `vite-plugin-react-server`, mit dem Sie Server Components erstellen können.

Wie funktionieren Server Components also?

Eine der besten Möglichkeiten, React Server Components zu demonstrieren, ist der Datenabruf.

In traditionellen React-Client-Komponenten lassen Sie den Browser API-Anfragen ausführen. Da Datenabruf ein Nebeneffekt ist, machen Sie diesen API-Aufruf in einem `useEffect`-Hook.

Es ist auch gute Praxis, Zustandsvariablen wie loading, data und error zu setzen, damit Sie anzeigen können, dass die Daten geladen werden, die Daten anzeigen, wenn sie bereit sind, oder einen Fehler in Ihrer App anzeigen.

Mit React Server Components können Sie die gesamte Komponente auf den Server verschieben und dort Daten abrufen, ohne `useState` oder `useEffect` verwenden zu müssen:

```jsx
const Users = async () => {
  const res = await fetch("https://jsonplaceholder.typicode.com/users");
  const users = await res.json();

  return (
    <>
      <h1 className="text-4xl text-center mt-6">Users</h1>
      <ul className="text-center mt-3">
        {users.map((user) => (
          <li key={user.id}>{user.name}</li>
        ))}
      </ul>
    </>
  );
};

export default Users;
```

Da React Server Components nur auf dem Server ausgeführt werden, können Sie einfach Daten von einer API abrufen und nur einmal rendern. Außerdem, da der Datenabruf auf dem Server näher an der Quelle stattfindet, kann Ihre App besser performen, besonders für Menschen mit langsamen Netzwerkverbindungen.

Ein wichtiger Punkt ist, dass der gesamte Code für Server Components auf dem Server bleibt und nicht an den Browser gesendet wird. Das bedeutet, dass Sie keine React-Hooks mit ihnen verwenden können und sie keinen Zugriff auf Web-APIs oder Browser-Ereignislistener haben. Wie können Sie also Interaktivität hinzufügen?

Im Next.js-App-Router sind standardmäßig alle Komponenten Server Components. Wenn Sie Interaktivität hinzufügen möchten, müssen Sie die Komponente mit der `"use client"`-Direktive als Client-Komponente markieren.

Angenommen, Sie möchten das vorherige Beispiel zu einer Client-Komponente machen. So können Sie das tun:

```jsx
"use client";

import { useState, useEffect } from "react";

const Users2 = () => {
  const [status, setStatus] = useState({
    users: [],
    loading: true,
    error: null,
  });

  async function fetchUsers() {
    try {
      const res = await fetch("https://jsonplaceholder.typicode.com/users");
      const data = await res.json();
      setStatus((prevStatus) => ({
        ...prevStatus,
        users: data,
        loading: false,
      }));
    } catch (err) {
      setStatus((prevStatus) => ({
        ...prevStatus,
        error: err.message,
        loading: false,
      }));
    }
  }

  useEffect(() => {
    fetchUsers();
  }, []);

  if (status.loading) {
    return <p>Loading Users...</p>;
  }
  if (status.error) {
    return <p>Error getting users: {status.error}</p>;
  }

  return (
    <>
      <h1 className="text-4xl text-center mt-6">Users</h1>
      <ul className="text-center mt-3">
        {status.users.map((user) => (
          <li key={user.id}>{user.name}</li>
        ))}
      </ul>
    </>
  );
};

export default Users2;
```

Wenn Sie Interaktivität wie Klick-Ereignisse hinzufügen möchten, muss die Komponente ebenfalls als Client-Komponente markiert sein:

```jsx
"use client";

import { useState } from "react";

const Counter = () => {
  const [count, setCount] = useState(0);

  return (
    <>
      <h1>Counter</h1>
      <button onClick={() => setCount(count + 1)}>Increment</button>
      <h2>{count}</h2>
      <button onClick={() => setCount(count - 1)}>Decrement</button>
    </>
  );
};

export default Counter;
```

Wenn Sie die use client-Direktive nicht zur Komponente hinzufügen, erhalten Sie einen Fehler mit der Meldung „You're importing a server component that needs `useState`. This React hook only works in a client component. To fix, mark the file (or its parent) with the `"use client"` directive.“

Die Hauptvorteile von React Server Components sind, dass der Datenabruf einfacher wird, der Code leichter zu lesen ist und die Komplexität auf der Client-Seite reduziert wird.
