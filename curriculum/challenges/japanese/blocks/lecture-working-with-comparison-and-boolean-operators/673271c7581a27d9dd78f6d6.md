# --interactive--

以前のレッスンでブール値の概念を初めて学びましたが、このレッスンではブール値の動作や等値演算子と不等値演算子の仕組みについてさらに深く掘り下げます。

ブール値は `true` と `false` のみを持つデータ型です。条件に基づいて何かを実行できるため便利です。ブール値は、アプリの特定の機能にアクセスできるかどうかを判断するなど、何かが起こるべきかどうかを評価したいときに欠かせません。以下は、`true` の値を `isOldEnoughToDrive` という変数に設定する例です。

:::interactive_editor

```js
let isOldEnoughToDrive = true;

console.log(isOldEnoughToDrive); // true
```

:::

この変数は次のように条件文の中で使えます。

:::interactive_editor

```js
let isOldEnoughToDrive = true;

if (isOldEnoughToDrive) {
 console.log("You're old enough to drive"); // You're old enough to drive
} else {
 console.log("Sorry, you are not old enough to drive");
}
```

:::

条件文は、条件に基づいてコード内で判断を行うのに役立ちます。この例では `if/else` 文と呼ばれるものを使っています。

もし `isOldEnoughToDrive` が `true` なら、文 `You're old enough to drive` がコンソールに表示されます。そうでなくて `isOldEnoughToDrive` が `false` なら、文 `Sorry, you are not old enough to drive` がコンソールに表示されます。`isOldEnoughToDrive` 変数は `true` に設定されているため、最初の文がコンソールに表示されます。`if/else` 文については今後のレッスンで詳しく学びます。

2つの値を比較するには、等値演算子か厳密等値演算子のどちらかを使えます。比較の結果は `true` か `false` のブール値になります。以下は文字列と数値を比較するために等値演算子を使った例です。等値演算子は等号2つ（`==`）で表されます。

:::interactive_editor

```js
console.log(5 == "5"); // true
```

:::

この例では、JavaScriptが文字列 `"5"` を数値 `5` に変換してから等しいかどうかをチェックします。両方の値が同じになったため、結果は true です。等値演算子は比較の前に型変換を行います。

これは型変換を行わない厳密等値演算子とは異なります。厳密等値演算子は型が同じかどうか、値が同じかどうかをチェックします。以下は数値と文字列を比較するために厳密等値演算子を使った例です。この演算子は等号3つ（`===`）で表されます。

:::interactive_editor

```js
console.log(5 === '5'); // false
```

:::

次の比較は `false` になります。なぜなら文字列のデータ型は数値のデータ型と同じではないからです。もし何かが別の値と等しくないかどうかをチェックしたい場合は、不等値演算子か厳密不等値演算子を使えます。以下は不等値演算子（`!=`）を使って数値と文字列を比較する例です。

:::interactive_editor

```js
console.log(5 != "5"); // false
```

:::

この例では、不等値演算子が最初に文字列の値を数値に変換してから比較するため、結果は `false` になります。値が同じになるので `false` を返します。厳密不等値演算子を使うと異なる結果になります。厳密不等値演算子は感嘆符に続けて等号2つ（`!==`）で表されます。

:::interactive_editor

```js
console.log(5 !== "5"); // true
```

:::

結果は `true` になります。なぜなら厳密不等値演算子は型変換を行わないためです。数値 `5` は文字列 `"5"` と等しくないので、結果は `true` になります。

型変換を行わないため、可能な限り厳密不等値演算子と厳密等値演算子を使うことがベストプラクティスとされています。プロのプロジェクトでは、ほとんどの場合、これら2つの演算子が不等値演算子や等値演算子より好まれて使われています。
