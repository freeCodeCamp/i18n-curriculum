# --description--

`debugger` 문장은 프로그램에서 무슨 일이 일어나고 있는지 조사하기 위해 특정 줄에서 코드를 일시 중지할 수 있게 해주는 강력한 JavaScript 도구입니다. 올바르게 사용하면 `debugger` 문장은 무언가가 제대로 작동하지 않는 이유를 알아내는 데 많은 시간을 절약할 수 있습니다.

JavaScript는 코드를 위에서 아래로 실행합니다. JavaScript가 코드를 실행하다가 `debugger` 문장을 만나면 즉시 해당 줄에서 실행을 일시 중지합니다. 이를 통해 변수, 함수, 그리고 전반적인 코드 흐름을 검사할 기회를 제공합니다.

이렇게 하면 무엇이 제대로 되고 있고 무엇이 잘못되고 있는지 정확히 알 수 있습니다. 다만, 이 기능은 브라우저의 개발자 도구가 열려 있을 때만 작동합니다. 그렇지 않으면 `debugger` 문장은 무시되고 코드는 평소처럼 계속 실행됩니다.

또한 `debugger` 문장을 사용하면 최신 브라우저는 지정한 줄에서 코드 실행을 일시 중지합니다. 그리고 재생 버튼을 클릭해 실행을 다시 시작할 수 있지만, 페이지 자체는 수동으로 다시 로드하지 않는 한 자동으로 새로 고쳐지지 않습니다.

`debugger` 문장을 사용하려면 코드 실행을 일시 중지하고 싶은 줄에 배치하세요. 예시는 다음과 같습니다:

```js
let firstNumber = 5;
let secondNumber = 10;
debugger; // Code execution pauses here
let sum = firstNumber + secondNumber;
console.log(sum);
```

이 예제에서, 코드가 실행되기 전에 콘솔이 열려 있지 않다가 나중에 열면 `15`가 콘솔에 표시됩니다. 콘솔이 열려 있는 상태에서 페이지를 새로 고치면 실행이 일시 중지되어 코드를 검사할 수 있습니다.

더 복잡한 예시는 다음과 같습니다:

```js
function calculateTotalPrice(price, discountPercentage) {
 debugger
 let discountAmount = (price * discountPercentage) / 100
 let totalPrice = price - discountAmount

 console.log(`Original Price: ${price}`)
 console.log(`Discount Amount: ${discountAmount}`)
 console.log(`Total Price after Discount: ${totalPrice}`)

 return totalPrice
}

let price = 100
let discount = 15

let finalPrice = calculateTotalPrice(price, discount)
console.log(`Final Price: ${finalPrice}`);
```

이 예제에서는 `debugger` 문장이 `calculateTotalPrice` 맨 위에 배치되어 있습니다. 함수가 호출되면 함수 실행이 일시 중지됩니다.

콘솔이 열린 상태에서 페이지를 새로 고치면 페이지가 계속 새로 고쳐지는 것을 볼 수 있는데, 이는 코드 실행이 중지된 것이 아니라 일시 중지된 것임을 확인시켜 줍니다. 실행을 계속하려면 재생 버튼을 클릭하면 됩니다.

JavaScript 프로그램을 계속 작성하면서 `debugger` 문장을 코드에 사용해 실행 흐름을 더 잘 이해하고, 실시간으로 변수를 검사하며, 어디서 문제가 발생하는지 빠르게 찾아보세요.
