# --description--

In Express, i **percorsi delle rotte** sono i pattern URL che determinano come il server risponde alle richieste in arrivo. Quando un utente richiede un URL specifico, Express abbina quell’URL a un percorso di rotta ed esegue il gestore corrispondente.

I percorsi delle rotte sono definiti all’interno di metodi di rotta come `app.get()`, `app.post()` e così via. Questi percorsi possono essere statici, come `/home`, oppure dinamici, dove si usano parametri per catturare valori dall’URL, come `/user/:id`.

In questa lezione ci concentreremo sulle rotte statiche. Ecco un esempio:

```javascript
// Static route
app.get("/home", (req, res) => {
  res.send("Welcome to the homepage!");
});
```

In questo esempio, il server ascolta le richieste GET a `/home` e risponde con un messaggio semplice.

Il metodo `app.get()` prende due argomenti:

* Il primo argomento, `"/home"`, è il percorso di rotta statico (o URL) a cui il server ascolta.
    
* Il secondo argomento è una funzione di callback che viene eseguita quando viene fatta una richiesta GET a quel percorso.
    

All’interno della callback, `res.send()` invia una risposta al client. In questo caso, invia il testo `Welcome to the homepage!`, che viene mostrato nel browser.

Ecco un altro esempio di rotta statica:

```javascript
// Static route
app.get("/menu/drinks", (req, res) => {
  res.send("Welcome to the drinks menu!");
});
```

In questo caso, il server ascolta le richieste GET all’URL `/menu/drinks` e invia un messaggio quando quella rotta viene richiesta.

Potresti usare rotte come questa quando il tuo sito ha più pagine correlate raggruppate sotto un percorso comune. Per esempio, un sito web di un ristorante potrebbe organizzare le pagine del menu così:

* `/menu/food`
    
* `/menu/drinks`
    
* `/menu/desserts`
    

Questo approccio aiuta a mantenere gli URL chiari e significativi, e rende anche le rotte più facili da organizzare e mantenere man mano che la tua applicazione cresce.

Nella prossima lezione vedremo i percorsi di rotta dinamici con parametri.
