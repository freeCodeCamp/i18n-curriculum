# --interactive--

Das `aria-describedby`-Attribut wird verwendet, um Screenreader-Nutzenden zusätzliche Informationen über ein Element bereitzustellen, indem auf vorhandene Inhalte auf der Seite verwiesen wird. Es schafft eine programmatische Verbindung zwischen dem Element und dem Inhalt (technisch als zugängliche Beschreibung bezeichnet), die Screenreader nutzen können, um Nutzende über die zusätzlichen Informationen zu informieren, wenn sie mit dem Element interagieren.

Die häufigste Verwendung von `aria-describedby` besteht darin, Anweisungen und Fehlermeldungen mit Formulareingaben zu verknüpfen. Aufgrund der verschiedenen Methoden, mit denen Screenreader-Nutzende eine Seite navigieren, können sie diese Meldungen beim Wechseln zwischen Eingabefeldern übersehen. Die Verwendung von `aria-describedby` hilft sicherzustellen, dass sie diese hören.

Schauen wir uns einige Beispiele an, um zu verstehen, wie es funktioniert. In diesem ersten Beispiel haben wir ein `form`-Element, das ein Passwort akzeptiert.

Aktivieren Sie den interaktiven Editor und geben Sie einige Zeichen in das Passwort-Eingabefeld ein. Sie werden sehen, dass das Passwort im Vorschaufenster maskiert wird. Sie sollten auch sehen, dass der `password-help`-Text rot bleibt, bis Sie 8 oder mehr Zeichen in das Eingabefeld eingeben.

HINWEIS: Dieses interaktive Beispiel verwendet CSS und JavaScript, um die Farbe des `password-help`-Textes dynamisch zu aktualisieren. Machen Sie sich keine Sorgen, den JavaScript-Code zu verstehen, denn Sie werden JavaScript in zukünftigen Modulen lernen.

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<form>
  <label for="password">Password:</label>
  <input type="password" id="password" aria-describedby="password-help" />
  <p id="password-help">Your password must be at least 8 characters long.</p>
</form>

<script src="index.js"></script>
```

```css
#password-help {
  color: red;
}
```

```js
const passwordEl = document.getElementById("password");
const passwordHelpText = document.getElementById("password-help");

passwordEl.addEventListener("input", (e) => { 
  const userInput = e.target.value;
  passwordHelpText.style.color = userInput.length >= 8 ? "green" : "red";
});
```

:::

Wir verwenden ein `label`-Element für den `Password`-Text und verknüpfen diesen mit dem Passwort-`input`-Feld.

Wir haben auch ein Absatz-Element, das die Passwortanforderungen beschreibt. Wir verwenden das `aria-describedby`-Attribut, um das Passwort-`input`-Feld mit den Passwortanforderungen im Absatz-Element zu verknüpfen. Wenn ein Screenreader-Nutzender mit diesem `input` interagiert, wird sein Screenreader den Namen des `input` (`Password`) ansagen und möglicherweise auch die Passwortanforderungen. Dies ist jedoch keine absolute Garantie, da einige Screenreader die zusätzlichen Inhalte nicht automatisch ansagen oder dies nur unter bestimmten Umständen tun. Das ist nicht ungewöhnlich. Jeder Screenreader ist anders und behandelt ARIA-Attribute auf seine eigene Weise. Dies schließt die Verwendung von `aria-describedby` nicht aus, da es Screenreader-Nutzenden im Allgemeinen zugutekommt.

Ein weiterer guter Anwendungsfall für das `aria-describedby`-Attribut ist, wenn Sie eine Lösch-`button` haben. Hier ist ein Beispiel für eine Lösch-`button`, gefolgt von einer Nachricht, die beschreibt, was passiert, wenn die Schaltfläche geklickt wird:

:::interactive_editor

```html
<button aria-describedby="delete-message">Delete</button>

<p id="delete-message">Warning! All deletions are permanent.</p>
```

:::

Wie im vorherigen Beispiel verknüpfen wir die Lösch-Schaltfläche mit der Nachricht mithilfe des `aria-describedby`-Attributs. Der Wert von `id` und der Wert des `aria-describedby`-Attributs müssen übereinstimmen.

Das `aria-describedby`-Attribut ist ein mächtiges Attribut, das dazu verwendet werden kann, sicherzustellen, dass Screenreader-Nutzenden zusätzliche Informationen über ein Element bereitgestellt werden, wenn sie mit dem Element interagieren. Es wird am häufigsten verwendet, um Anweisungen und Fehlermeldungen mit Formulareingaben zu verknüpfen, um die Wahrscheinlichkeit zu verringern, dass Screenreader-Nutzende diese Meldungen beim Navigieren durch das Formular übersehen.
