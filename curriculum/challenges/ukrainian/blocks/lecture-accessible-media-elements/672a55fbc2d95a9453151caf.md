# --interactive--

Багато користувачів користуються клавіатурою замість миші через фізичні обмеження, травми від повторюваних рухів або особисті вподобання. Це включає користувачів читачів екрана та тих, у кого немає робочої миші.
Доступність для клавіатури гарантує, що ці користувачі можуть ефективно орієнтуватися у веб-застосунках без перешкод.

Розглянемо кілька практичних технік, які ви можете застосувати, щоб зробити веб-застосунки доступними для клавіатури.

Багато користувачів покладаються на клавішу `Tab`, щоб переходити між інтерактивними елементами на сторінці. За замовчуванням браузери дозволяють користувачам переходити табуляцією між елементами, такими як посилання, кнопки та поля форм, у порядку їх появи в HTML. Це називається природним порядком табуляції.

Іноді ви можете захотіти змінити, які елементи можна сфокусувати, або змінити порядок їх фокусу. Атрибут `tabindex` дозволяє це зробити.

Ось базовий синтаксис:

```html
<element tabindex="number">Element Text</element>
```

Значення `tabindex` визначає, як елемент поводиться при навігації клавіатурою:

`tabindex="0"` додає елемент до природного порядку табуляції.

Табуляція переміщує фокус від `button` до `div`, а потім до посилання, відповідно до їх порядку в HTML.

Натисніть будь-де у вільному просторі вікна попереднього перегляду. Потім використайте клавішу `Tab`, щоб побачити, як фокус переходить між різними елементами. Щоб взаємодіяти з прикладом, потрібно увімкнути інтерактивний редактор.

:::interactive_editor

```html
<button>First</button>
<div tabindex="0">Second</div>
<a href="#">Third</a>
```

:::

`tabindex="-1"` робить елемент програмно фокусованим. Це корисно для керування фокусом у елементах, які зазвичай не можна сфокусувати, таких як заголовки, контейнери, діалоги або повідомлення про помилки:

```html
<p tabindex="-1">Sorry, there was an error with your submission.</p>
```

У цьому прикладі абзац не є частиною нормального порядку табуляції, тому користувачі не можуть дістатися до нього, натискаючи клавішу `Tab`. Однак, якщо встановити фокус на цей елемент через скрипт, повідомлення приверне увагу користувача. Ви дізнаєтеся більше про цю техніку в уроках JavaScript.

Коли `tabindex` більше за `0`, це встановлює кастомний порядок табуляції. Тож елементи з меншими позитивними значеннями отримують фокус першими.

У цьому прикладі табуляція сфокусується спочатку на `input` з `tabindex="1"`, потім на `2`, а потім на `3`, незалежно від їх порядку в HTML.

```html
<input tabindex="2">
<input tabindex="1">
<input tabindex="3">
```

Кастомні позитивні значення іноді використовують у складних віджетах, наприклад, у панелі інструментів, де потрібен певний порядок навігації. Однак цей підхід не рекомендується, бо він може ускладнити навігацію та ускладнити підтримку, особливо коли сторінка зростає або змінюється.

`accesskey` — це ще один атрибут, який ви можете використати, щоб зробити ваш веб-проєкт доступним для клавіатури. Він дозволяє визначити клавішу, яка фокусується або активує певний елемент.

Ось інтерактивний приклад, який ви можете спробувати, дотримуючись наведених нижче рекомендацій (спочатку увімкніть інтерактивний редактор):

- `accesskey="s"` призначає клавішу `S` кнопці `Save`. У більшості браузерів натискання `ALT + S` (у Windows) і `CTRL + Option + S` (на Mac) активує цю кнопку.
- `accesskey="c"` встановлює клавішу `C` для кнопки `Cancel`, дозволяючи користувачам активувати її за допомогою `ALT + C` (Windows) і `CTRL + Option + C` (Mac).
- `accesskey="h"` призначає клавішу `H` посиланню `Home`, дозволяючи користувачам переходити на домашню сторінку за допомогою `ALT + H` (Windows) і `CTRL + Option + H` (Mac).

Зверніть увагу, що точна комбінація клавіш для активації accesskey може відрізнятися залежно від браузера та операційної системи. Зазвичай це `ALT + Specified Key` у Windows і `CTRL + Option + Specified Key` на Mac.

:::interactive_editor

```html
<button accesskey="s">Save</button>
<button accesskey="c">Cancel</button>
<a href="index.html" accesskey="h">Home</a>
```

:::

Ще один спосіб зробити клавіатуру доступною у ваших застосунках — це забезпечити чіткі індикатори фокусу. Якщо вам здається, що індикатор фокусу за замовчуванням у браузері недостатній, ви можете замістити його, націлившись на стан фокусу елемента.

Ось приклад стилізації стану фокусу для елемента `button`. Натисніть будь-де у вільному просторі вікна попереднього перегляду, а потім натисніть клавішу `Tab`, щоб сфокусувати кнопку. Щоб взаємодіяти з прикладом, потрібно увімкнути інтерактивний редактор.

:::interactive_editor

```html
<link href="styles.css" rel="stylesheet">

<button>Example button</button>
```

```css
button:focus {
  outline: 2px solid #005fcc;
}
```

:::

Властивість `outline` використовується для визначення обрисів навколо елемента. У цьому прикладі обриси встановлені як суцільна синя лінія товщиною 2 пікселі. Індикатор фокусу має бути стилізований так, щоб було очевидно, який елемент наразі має фокус. Щоб бути доступним, індикатор повинен мати мінімальну контрастність кольору не менше 3:1 із кольором фону, який він покриває.

Також слід уникати клавіатурних пасток, які виникають, коли користувач не може перемістити фокус з певного елемента у компонентах, таких як модальні вікна та спливаючі повідомлення.
