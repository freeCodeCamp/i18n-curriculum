# --interactive--

如你從先前的課程和工作坊所回想，你可以創建帶有多個參數的函式，並以引數呼叫該函式。

這裡有一個提醒：

```js
// function definition
function getSum(num1, num2) {
  return num1 + num2;
}

// function call
getSum(3, 4); // 7
```

但是如果你有一個函式被呼叫時帶入的引數比它定義時接受的還多，該怎麼辦？

:::interactive_editor

```js
// function definition
function getSum(num1, num2) {
  return num1 + num2;
}

// function call with extra argument
console.log(getSum(3, 4, 5)); // 7
```

:::

JavaScript 在這種情況下不會發出錯誤。它會忽略多餘的引數，並且只將數字 `3` 和 `4` 相加。接受可變數量引數的函式稱為 <dfn>variadic functions</dfn>。

如果你正在使用可變引數函式（variadic functions），那麼你可以利用 `arguments` 物件。這個類陣列物件包含傳入函式的引數值。

這裡有一個範例：

:::interactive_editor

```js
function logArgs() {
  for (const arg of arguments) {
    console.log(arg);
  }
}

logArgs(1, 2, 3);
// result:
// 1
// 2
// 3

logArgs("example"); // "example"
```

:::

由於 `arguments` 物件類似陣列，你可以像這樣存取特定索引的引數：

:::interactive_editor

```js
function getArg() {
  return arguments[1];
}

console.log(getArg(2, 4, 6)); // 4
```

:::

你也可以像這樣使用 `length` 屬性來取得函式被呼叫時的引數數量：

:::interactive_editor

```js
function getArgs() {
  return arguments.length;
}

console.log(getArgs("Example")); // 1
console.log(getArgs("Another", "Example")); // 2
```

:::

儘管 `arguments` 物件看起來像是真正的陣列，但它沒有內建像 `includes` 或 `push` 這樣的 `Array` 方法。若要存取那些方法，你需要先使用像是 `slice`、`Array.from()` 或展開運算子將 `arguments` 物件轉換成真正的陣列：

:::interactive_editor

```js
function hasCat() {
  return [...arguments].includes("cat");
}

console.log(hasCat("dog", "chicken", "cat")); // true
console.log(hasCat("dog", "chicken", "horse")); // false
```

:::

雖然可以使用 `arguments` 物件來處理可變引數函式，但現代 JavaScript 應用程式通常會使用剩餘參數語法。你將在未來的課程中學習更多相關內容。
