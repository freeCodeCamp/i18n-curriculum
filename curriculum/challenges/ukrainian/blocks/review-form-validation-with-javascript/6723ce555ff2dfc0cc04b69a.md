# --interactive--

## Валідація форм за допомогою JavaScript

- **Constraint Validation API**: певні HTML-елементи, такі як `textarea` і `input`, відкривають доступ до constraint validation API. Цей API дозволяє переконатися, що значення, яке користувач ввів для цього елемента, проходить будь-яку валідацію на рівні HTML, яку ви задали (наприклад, мінімальну довжину або відповідність шаблону).
- **Метод `checkValidity()`**: цей метод повертає `true`, якщо елемент відповідає всім HTML-валідаціям (на основі його атрибутів), і `false`, якщо ні.

:::interactive_editor

```html
<form>
  <label>
    Email:
    <input
      id="email"
      type="email"
      required
      pattern=".+\.com$"
      placeholder="example@site.com"
    />
  </label>
</form>

<script>
  const input = document.getElementById("email");

  input.addEventListener("input", (e) => {
    if (!e.target.checkValidity()) {
      e.target.setCustomValidity("You must use a .com email.");
    } else {
      e.target.setCustomValidity("");
    }
  });
</script>
```

:::

- **Метод `reportValidity()`**: цей метод повідомляє браузеру, що `input` є недійсним.

:::interactive_editor

```html
<form>
  <label>
    Email:
    <input
      id="email2"
      type="email"
      required
      pattern=".+\.com$"
      placeholder="example@site.com"
    />
  </label>
</form>

<script>
  const input = document.getElementById("email2");

  input.addEventListener("input", (e) => {
    if (!e.target.checkValidity()) {
      e.target.reportValidity();
    }
  });
</script>
```

:::

- **Властивість `validity`**: Ця властивість використовується для отримання стану валідності елементів керування формою (таких як `<input>`, `<select>` тощо) і надає інформацію про те, чи відповідають введені користувачем дані обмеженням, визначеним для цього елемента (наприклад, полям з `required`, обмеженням за шаблоном, максимальною довжиною тощо).

:::interactive_editor

```html
<input
  id="age"
  type="number"
  min="18"
  placeholder="Enter age (18+)"
/>

<script>
  const input = document.getElementById("age");

  input.addEventListener("input", (e) => {
    console.log(e.target.validity);
  });
</script>
```

:::

- **Властивість `patternMismatch`**: вона буде `true`, якщо значення не відповідає заданому регулярному виразу.

## `preventDefault()` метод

- **Визначення**: кожна подія, що відбувається в DOM, має певну поведінку за замовчуванням. Подія click на прапорці за замовчуванням змінює стан цього прапорця. Натискання пробілу на сфокусованій кнопці активує цю кнопку. Метод `preventDefault()` на цих об’єктах `Event` зупиняє цю поведінку.

:::interactive_editor

```html
<form id="form">
  <input type="text" placeholder="Try to submit" />
  <button type="submit">Submit</button>
</form>

<p id="status"></p>

<script>
  const form = document.getElementById("form");
  const status = document.getElementById("status");

  form.addEventListener("submit", (event) => {
    event.preventDefault();
    status.textContent = "Form submission prevented.";
  });
</script>
```

:::

## Відправлення форм

- **Визначення**: існує три способи відправлення форми. Перший — коли користувач натискає на кнопку у формі, у якої атрибут `type` має значення `submit`. Другий — коли користувач натискає клавішу `Enter` у будь-якому редагованому полі `input` у формі. Третій — через виклик JavaScript методів `requestSubmit()` або `submit()` елемента `form`.
- **Атрибут `action`**: атрибут `action` повинен містити або посилання, або відносний шлях для поточного домену. Це значення визначає, куди форма намагається надіслати дані — якщо не встановити атрибут `action`, форма надішле дані на посилання поточної сторінки.

```html
<form action="https://freecodecamp.org" method="GET">
  <input
    type="number"
    name="number"
    placeholder="Enter a number"
  />
  <button type="submit">Submit</button>
</form>
```

- **Атрибут `method`**: цей атрибут приймає стандартний метод `HTTP`, наприклад `GET` або `POST`, і використовує цей метод при виконанні запиту до посилання дії. Якщо метод не встановлено, форма за замовчуванням виконає запит `GET`. Дані у формі будуть кодуватися у форматі `name=value` пар і додаватися до посилання дії як параметри запиту.

```html
<form action="/data" method="POST">
  <input
    type="number"
    id="input"
    placeholder="Enter a number"
    name="number"
  />
  <button type="submit">Submit</button>
</form>
```

- **Атрибут `enctype`**: елемент `form` приймає атрибут `enctype`, який визначає тип кодування для даних. Цей атрибут приймає лише три значення: `application/x-www-form-urlencoded` (за замовчуванням, надсилає дані у вигляді URL-кодованого тіла форми), `text/plain` (надсилає дані у вигляді простого тексту, у парах `name=value`, розділених новими рядками) або `multipart/form-data`, який спеціально призначений для обробки форм із завантаженням файлів.
