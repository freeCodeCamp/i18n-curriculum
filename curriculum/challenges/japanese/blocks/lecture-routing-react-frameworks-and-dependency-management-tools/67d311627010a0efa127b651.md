# --description--

これまでのレッスンで、Reactはシングルページアプリケーションであることを学びました。シングルページアプリケーションとは、1つのHTMLファイルを持ち、JavaScriptを使ってページ上のコンテンツを動的に更新するアプリケーションのことです。

では、Reactアプリケーションに複数の「ページ」を追加したい場合はどうすればよいでしょうか？それらの異なるビューにどうやって移動しますか？

ここで登場するのがReact Routerです。

React Routerは、Reactアプリケーションにルーティングを追加できるサードパーティのライブラリです。まず、既存のReactプロジェクトにReact Routerを次のようにインストールする必要があります。

```sh
npm i react-router
```

`package.json`ファイルを確認すると、`react-router`が依存関係のリストに追加されているのがわかります。

```json
"dependencies": {
  "react": "^18.3.1",
  "react-dom": "^18.3.1",
  "react-router": "^7.2.0"
}
```

次に、`main.jsx`または`index.jsx`ファイルの中で、`BrowserRouter`をインポートし、`BrowserRouter`コンポーネントの周りに`App`をレンダリングする必要があります。

```jsx
import { StrictMode } from "react";
import { createRoot } from "react-dom/client";
import { BrowserRouter } from "react-router";
import App from "./App.jsx";

import "./index.css";

createRoot(document.getElementById("root")).render(
  <StrictMode>
    <BrowserRouter>
      <App />
    </BrowserRouter>
  </StrictMode>
);
```

アプリケーションでルートを有効にするには、`import`文を更新して`Routes`と`Route`コンポーネントを含める必要があります。このようにします。

```js
import { BrowserRouter, Routes, Route } from "react-router";
```

次に、`BrowserRouter`の中に`Routes`と`Route`コンポーネントを追加します。

```jsx
createRoot(document.getElementById("root")).render(
  <StrictMode>
    <BrowserRouter>
      <Routes>
        <Route path="/" element={<App />} />
      </Routes>
    </BrowserRouter>
  </StrictMode>
);
```

`path`と`element`はURLとUIコンポーネントを結びつけるために使われます。この場合、ホームページのルートを`App`コンポーネントに設定しています。

大規模なアプリケーションでは、このように複数のビューとルートを設定することが一般的です。

```jsx
<Routes>
  <Route index element={<Home />} />
  <Route path="about" element={<About />} />

  <Route path="products">
    <Route index element={<ProductsHome />} />
    <Route path=":category" element={<Category />} />
    <Route path=":category/:productId" element={<ProductDetail />} />
    <Route path="trending" element={<Trending />} />
  </Route>
</Routes>
```

これらの例の`index`プロップは、特定のパスセグメントのデフォルトルートを表しています。つまり、`Home`コンポーネントはルートパス（`/`）で表示され、`ProductsHome`コンポーネントは`products`パスで表示されます。

また、いくつかのルートを別のルートの中にネストしているのにも気づいたかもしれません。

```jsx
<Route path="products">
  <Route path="trending" element={<Trending />} />
</Route>
```

これは、子ルートのパスが親ルートのパスに追加されることを意味します。したがって、この例では、トレンド商品の`path`は`products/trending`になります。

もし`path`がコロン（`:`）で始まる場合、それはルート内の動的セグメントを表します。

```jsx
<Route path=":category" element={<Category />} />
```

この例では、`category`という動的セグメントがあります。ユーザーが`products/brass-instruments`のようなURLに移動すると、ビューは`Category`コンポーネントに切り替わり、そのセグメントに基づいて適切なデータを動的に取得できます。

子コンポーネントの中で`useParams`フックを使うことで、動的セグメントの値にアクセスできます。

```jsx
import { useParams } from "react-router";

export default function Category() {
  let params = useParams();
   {/* Accessing the category param: params.category */}
   {/* rest of code goes here */}
}
```

動的ルートは、URLのパラメータに基づいて異なるコンテンツをレンダリングできる柔軟で再利用可能なコンポーネントを作成できるため便利です。すべての可能なルートに対して固定のパスリストを定義する代わりに、動的セグメントを使ってユーザーのリクエストに応じたさまざまなコンテンツをレンダリングできます。
