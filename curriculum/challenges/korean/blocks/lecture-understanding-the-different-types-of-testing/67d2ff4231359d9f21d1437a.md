# --description--

엔드투엔드 테스트, 줄여서 E2E 테스트는 사용자의 관점에서 실제 상황을 테스트합니다. E2E 테스트 시나리오의 예로는 회원가입과 로그인 과정 테스트나 전자상거래 사이트의 결제 과정 테스트가 있습니다.

엔드투엔드 테스트가 어떻게 작동하는지 더 잘 이해하려면 freeCodeCamp 코드베이스의 실제 예제를 살펴보겠습니다. 이 예제는 Microsoft에서 개발한 인기 있는 엔드투엔드 테스트 프레임워크인 Playwright를 사용합니다.

freeCodeCamp 코드베이스 안에는 기부, 인증, 시험 등 테스트를 담고 있는 `e2e`라는 디렉터리가 있습니다. 이 디렉터리 안에는 인증된 기부자 지원자 페이지 테스트가 들어 있는 `donate-page-donor.spec.ts` 파일이 있습니다.

첫 번째 단계는 이미 지원자인 데모 사용자로서 기부 페이지로 이동하는 것입니다:

```js
test.beforeEach(async ({ page }) => {
  execSync("node ./tools/scripts/seed/seed-demo-user --set-true isDonating");
  await page.goto("/donate");
});
```

이 파일에는 여러 테스트가 있으므로 `beforeEach` 훅이 각 테스트 전에 실행됩니다.

사용자가 기부 페이지에 도착하면 freeCodeCamp를 더 지원하는 방법에 대한 제안과 함께 감사 메시지를 보게 됩니다. 다음은 기부 페이지에서 발견되는 일부 텍스트를 확인하는 몇 가지 테스트입니다:

```js
test("should render the donate page correctly", async ({ page }) => {
  await expect(
    page.getByText("Thank you for your continued support")
  ).toBeVisible();

  await expect(
    page.getByText(
      "Your contributions are crucial in creating resources that empower millions of people to learn new skills and support their families."
    )
  ).toBeVisible();

  ...
});
```

기부자가 메뉴 바에 지원자 링크를 가지고 있는지, 그리고 지원자임을 나타내는 아바타 주위에 특별한 스타일의 테두리가 있는지도 확인하는 테스트도 있습니다:

```js
test("The menu should have a supporters link", async ({ page }) => {
  const menuButton = page.getByTestId("header-menu-button");
  const menu = page.getByTestId("header-menu");

  await expect(menuButton).toBeVisible();
  await menuButton.click();

  await expect(menu).toBeVisible();

  await expect(page.getByRole("link", { name: "Supporters" })).toBeVisible();
});

test("The Avatar should have a special border for donors", async ({ page }) => {
  const container = page.locator(".avatar-container");
  await expect(container).toHaveClass("avatar-container gold-border");
});
```

이러한 상세한 테스트를 통해 애플리케이션의 테스트 범위를 단위 테스트를 넘어 확장할 수 있습니다. 엔드투엔드 테스트는 애플리케이션이 올바르게 작동하고 사용자에게 예측 가능하도록 돕습니다. 다른 엔드투엔드 테스트 도구로는 Cypress, Selenium, Puppeteer가 있습니다.

엔드투엔드 테스트가 중요하지만, 설정, 설계, 유지 관리에 시간이 많이 들기 때문에 비용이 많이 듭니다. E2E 테스트는 중요한 사용자 흐름에 적합하며, 단위 테스트는 애플리케이션 내 작은 단위를 테스트하는 데 적합합니다.
