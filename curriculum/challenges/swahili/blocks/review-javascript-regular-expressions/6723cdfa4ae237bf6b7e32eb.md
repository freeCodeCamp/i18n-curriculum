## Usemi wa kawaida na njia za kawaida

- **Ufafanuzi**: Usemi wa kawaida, au Regex, hutumika kuunda "mufano", ambao unaweza kisha kutumia kukagua mfuatano wa herufi, kutoa maandishi, na zaidi.

```js
const regex = /freeCodeCamp/;
```

- **Njia ya `test()`**: Njia hii inakubali mfuatano wa herufi, ambao ni mfuatano wa herufi wa kujaribu kulinganisha na usemi wa kawaida. Njia hii itarudisha thamani ya kweli au siyo kweli ikiwa mfuatano wa herufi unalingana na usemi wa kawaida.

:::interactive_editor

```js
const regex = /freeCodeCamp/;
const test = regex.test("e");
console.log(test); // false
```

:::

- **Njia ya `match()`**: Njia hii inakubali usemi wa kawaida, ingawa pia unaweza kupitisha mfuatano wa herufi ambao utaundwa kuwa usemi wa kawaida. Njia ya `match` inarudisha safu ya matokeo ya kulinganisha kwa mfuatano wa herufi.

```js
const regex = /freeCodeCamp/;
const match = "freeCodeCamp".match(regex);
console.log(match); // ['freeCodeCamp', index: 0, input: 'freeCodeCamp', groups: undefined]
```

- **Njia ya `replace()`**: Njia hii inakubali hoja mbili: usemi wa kawaida wa kulinganisha (au mfuatano wa herufi), na mfuatano wa herufi wa kuchukua nafasi ya kulinganisha (au kitendakazi cha kuendesha kwa kila kulinganisha).

:::interactive_editor

```js
const regex = /Jessica/;
const str = "Jessica is rly kewl";
const replaced = str.replace(regex, "freeCodeCamp");
console.log(replaced); // "freeCodeCamp is rly kewl"
```

:::

- **Njia ya `replaceAll`**: Njia hii hutumika kubadilisha matukio yote ya mufano uliotajwa na mfuatano mpya wa herufi. Njia hii itatupa kosa ikiwa utampa usemi wa kawaida bila kihimarisha cha global.

:::interactive_editor

```js
const text = "I hate JavaScript! I hate programming!";
const newText = text.replaceAll("hate", "love");
console.log(newText);  // "I love JavaScript! I love programming!"
```

:::

- **Njia ya `matchAll`**: Njia hii hutumika kupata matokeo yote ya usemi wa kawaida uliotolewa katika mfuatano wa herufi, ikiwa ni pamoja na makundi ya kunasa, na kuyarudisha kama kielezi. Kielezi ni kitu kinachokuwezesha kupita juu ya mkusanyiko wa vitu.

:::interactive_editor

```js
const str = "JavaScript, Python, JavaScript, Swift, JavaScript";
const regex = /JavaScript/g;

const iterator = str.matchAll(regex);

for (let match of iterator) {
  console.log(match[0]); // "JavaScript" for each match
}
```

:::

## Kihimarisha cha usemi wa kawaida

- **Ufafanuzi**: Kihimarisha, kinachojulikana pia kama "bendera", hubadilisha tabia ya usemi wa kawaida.
- **Bendera ya `i`**: Bendera hii hufanya regex isizingatie tofauti za herufi.

:::interactive_editor

```js
const regex = /freeCodeCamp/i;
console.log(regex.test("freecodecamp")); // true
console.log(regex.test("FREECODECAMP")); // true
```

:::

- **Bendera ya `g`**: Bendera hii, au kihimarisha cha global, huruhusu usemi wako wa kawaida kulinganisha mufano zaidi ya mara moja.

:::interactive_editor

```js
const regex = /freeCodeCamp/gi;
console.log(regex.test("freeCodeCamp")); // true
console.log(regex.test("freeCodeCamp is great")); // false
```

:::

- **Ufafanuzi wa tagi ya nanga**: Tagi ya nanga ya `^`, mwanzoni mwa usemi wa kawaida, inasema "linganisha mwanzo wa mfuatano wa herufi". Tagi ya nanga ya `$`, mwishoni mwa usemi wa kawaida, inasema "linganisha mwisho wa mfuatano wa herufi".

:::interactive_editor

```js
const start = /^freeCodeCamp/i;
const end = /freeCodeCamp$/i;
console.log(start.test("freecodecamp")); // true
console.log(end.test("freecodecamp")); // true
```

:::

- **Bendera ya `m`**: Tagi za nanga hutafuta mwanzo na mwisho wa mfuatano mzima wa herufi. Lakini unaweza kufanya regex kushughulikia mistari mingi kwa bendera ya `m`, au kihimarisha cha mistari mingi.

:::interactive_editor

```js
const start = /^freecodecamp/im;
const end = /freecodecamp$/im;
const str = `I love 
freecodecamp
it's my favorite
`;
console.log(start.test(str)); // true
console.log(end.test(str)); // true
```

:::

- **Bendera ya `d`**: Bendera hii huongeza taarifa unazopata katika kitu cha matokeo ya kulinganisha.

```js
const regex = /freecodecamp/di;
const string = "we love freecodecamp isn't freecodecamp great?";
console.log(string.match(regex));
```

Na matokeo ni:

```js
// [
//   'freecodecamp',
//   index: 8,
//   input: "we love freecodecamp isn't freecodecamp great?",
//   groups: undefined,
//   indices: [
//     0: [8, 20],
//     groups: undefined
//   ]
// ]
```

- **Bendera ya `u`**: Hii huongeza uwezo wa usemi wa kawaida kuruhusu kulinganisha herufi maalum za unicode. Bendera ya `u` inakupa upatikanaji wa madarasa maalum kama `Extended_Pictographic` ya kulinganisha emoji nyingi. Pia kuna bendera ya `v`, ambayo huongeza zaidi uwezo wa kulinganisha unicode.
- **Bendera ya `y`**: Kihimarisha cha sticky hufanya kazi kama kihimarisha cha global, lakini kwa tofauti chache. Tofauti kubwa ni kwamba usemi wa kawaida wa global utaanza kutoka lastIndex na kutafuta sehemu iliyobaki ya mfuatano kwa kulinganisha nyingine, lakini usemi wa kawaida wa sticky utarudisha null na kuweka lastIndex kuwa 0 ikiwa hakuna kulinganisha mara moja kwenye lastIndex iliyopita.
- **Bendera ya `s`**: Kihimarisha cha single-line huruhusu herufi ya wildcard, inayowakilishwa na `.` katika regex, kulinganisha mapumziko ya mistari - kwa maana mfuatano unachukuliwa kama mstari mmoja wa maandishi.

## Madarasa ya herufi

- **Wildcard `.`**: Madarasa ya herufi ni sintaksia maalum unayoweza kutumia kulinganisha seti au seti ndogo za herufi. Darasa la kwanza la herufi unalopaswa kujifunza ni darasa la wildcard. Wildcard huwakilishwa na nukta, au dot, na hulinganisha herufi YOYOTE moja ISIPOKUWA mapumziko ya mistari. Ili kuruhusu darasa la wildcard kulinganisha mapumziko ya mistari, kumbuka unahitaji kutumia bendera ya `s`.

```js
const regex = /a./;
```

- **`\d`**: Hii italinganisha tarakimu zote (`0-9`) katika mfuatano wa herufi.

```js
const regex = /\d/;
```

- **`\w`**: Hii hutumika kulinganisha herufi yoyote ya neno (`a-z0-9_`) katika mfuatano wa herufi. Herufi ya neno inafafanuliwa kama herufi yoyote, kutoka a hadi z, au nambari kutoka 0 hadi 9, au herufi ya underscore.

```js
const regex = /\w/;
```

- **`\s`**: Darasa la nafasi nyeupe `\s`, linalowakilishwa na mkwaju wa nyuma ukifuatiwa na `s`. Darasa hili la herufi litalinganisha nafasi yoyote nyeupe, ikiwa ni pamoja na mistari mipya, nafasi, tabo, na herufi maalum za nafasi za unicode.
- **Kukanusha madarasa maalum ya herufi**: Ili kukanusha mojawapo ya madarasa haya ya herufi, badala ya kutumia herufi ndogo baada ya mkwaju wa nyuma, unaweza kutumia sawa yake ya herufi kubwa. Mfano ufuatao hauzilingani na herufi ya nambari. Badala yake, unalinganisha herufi yoyote moja ambayo SI herufi ya nambari.

```js
const regex = /\D/;
```

- **Madarasa maalum ya herufi**: Unaweza kuunda madarasa maalum ya herufi kwa kuweka herufi unayotaka kulinganisha ndani ya mabano ya mraba.

```js
const regex = /[abcdf]/;
```

## Uthibitisho wa angalizo la mbele na nyuma

- **Ufafanuzi**: Uthibitisho wa angalizo la mbele na nyuma hukuruhusu kulinganisha mifano maalum kulingana na uwepo au ukosefu wa mifano inayozunguka.
- **Uthibitisho chanya wa angalizo la mbele**: Uthibitisho huu utalinganisha mufano wakati mufano unafuatiwa na mufano mwingine. Ili kuunda angalizo chanya la mbele, unahitaji kuanza na mufano unayotaka kulinganisha. Kisha, tumia mabano ya kawaida kufunika mufano unayotaka kutumia kama sharti lako. Baada ya mabano ya kufungua, tumia `?=` kufafanua mufano huo kama angalizo chanya la mbele.

```js
const regex = /free(?=code)/i;
```

- **Uthibitisho hasi wa angalizo la mbele**: Huu ni aina ya sharti inayotumika katika usemi wa kawaida kukagua kwamba mufano fulani haujatokea mbele katika mfuatano wa herufi.

```js
const regex = /free(?!code)/i;
```

- **Uthibitisho chanya wa angalizo la nyuma**: Uthibitisho huu utalinganisha mufano tu ikiwa umeongozwa na mufano mwingine maalum, bila kujumuisha mufano uliotangulia katika kulinganisha.

```js
const regex = /(?<=free)code/i;
```

- **Uthibitisho hasi wa angalizo la nyuma**: Uthibitisho huu unahakikisha kwamba mufano hauongozwi na mufano mwingine maalum. Unalinganisha tu ikiwa mufano uliotajwa hauongozwi mara moja na mfululizo uliotolewa, bila kujumuisha mfululizo uliotangulia katika kulinganisha.

```js
const regex = /(?<!free)code/i;
```

## Kihimarisha cha regex

- **Ufafanuzi**: Kihimarisha katika usemi wa kawaida huainisha ni mara ngapi mufano (au sehemu ya mufano) unapaswa kuonekana. Husaidia kudhibiti idadi ya matukio ya herufi au makundi katika kulinganisha. Mfano ufuatao hutumika kulinganisha herufi iliyopita mara nne hasa.

```js
const regex = /^\d{4}$/;
```

- **`*`** : Linalinganisha matukio 0 au zaidi ya kipengele kilichotangulia.
- **`+`**: Linalinganisha matukio 1 au zaidi ya kipengele kilichotangulia.
- **`?`**: Linalinganisha tukio 0 au 1 la kipengele kilichotangulia.
- **`{n}`**: Linalinganisha matukio n hasa ya kipengele kilichotangulia.
- **`{n,}`**: Linalinganisha matukio n au zaidi ya kipengele kilichotangulia.
- **`{n,m}`**: Linalinganisha kati ya matukio n na m ya kipengele kilichotangulia.

## Makundi ya kunasa na rejea za nyuma

- **Makundi ya kunasa**: Kundi la kunasa linakuwezesha "kunasa" sehemu ya mfuatano uliolingana ili kuitumia kwa namna yoyote unayohitaji. Makundi ya kunasa huainishwa na mabano ya kawaida yanayojumuisha mufano wa kunasa, bila herufi za kuongoza kama angalizo la mbele.

```js
const regex = /free(code)camp/i;
```

- **Makundi yasiyokunasa**: Kundi lisilokunasa ni sawa na kundi la kunasa lakini halihifadhi sehemu ya mfuatano uliolingana kwa matumizi ya baadaye. Makundi yasiyokunasa huainishwa na `(?:...)`.

```js
const regex = /free(?:code)camp/i;
```

- **Rejea za nyuma**: Rejea ya nyuma katika usemi wa kawaida inarejelea njia ya kutumia tena sehemu ya mufano uliolingana awali katika usemi uleule. Inakuwezesha kurejea kundi lililonaswa (sehemu ya mufano katika mabano) kwa nambari yake. Kwa mfano, `$1` inarejelea kundi la kwanza lililonaswa.

:::interactive_editor

```js
const regex = /free(co+de)camp/i;
console.log("freecoooooooodecamp".replace(regex, "paid$1world"));
```

:::

- Unaweza kutumia rejea za nyuma ndani ya regex yenyewe kulinganisha maandishi yale yale yaliyokunwa na kundi lililotangulia kwa kutumia mkwaju wa nyuma na nambari ya kundi la kunasa. Kwa mfano:

:::interactive_editor

```js
const regex = /(hello) \1/i;
console.log(regex.test("hello hello")); // true
console.log(regex.test("hello world")); // false
```

:::
