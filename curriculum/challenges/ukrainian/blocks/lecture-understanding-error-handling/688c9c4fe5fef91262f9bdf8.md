# --description--

У Python інструкція `raise` — це потужний інструмент, який дозволяє вручну викликати виключення у вашому коді. Вона дає вам контроль над тим, коли і як виникають помилки, дозволяючи створювати кастомні умови помилок і забезпечувати певну поведінку програми.

Інструкція `raise` використовується для явного викидання виключення в будь-якій точці вашої програми, що дозволяє сигналізувати про те, що сталася помилка або що певні вимоги не були виконані.

Інструкція `raise` у Python може використовуватися кількома способами для виклику виключень. У найпростішому випадку ви можете підняти вбудовані виключення або створити кастомні повідомлення про помилки. Ось простий приклад:

```py
def check_age(age):
    if age < 0:
        raise ValueError('Age cannot be negative')
    return age

try:
    check_age(-5)
except ValueError as e:
    print(f'Error: {e}') # Error: Age cannot be negative
```

Тут ви бачите, що `raise` — це ключове слово, яке викликає виключення.

У цьому прикладі ми піднімаємо `ValueError` з кастомним повідомленням, коли вказано недійсний вік.

Інструкція `raise` також може використовуватися для повторного підняття поточного виключення, що особливо корисно при обробці виключень:

```py
def process_data(data):
    try:
        result = int(data)
        return result * 2
    except ValueError:
        print('Logging: Invalid data received')
        raise  # Re-raises the same ValueError

try:
    process_data('abc')
except ValueError:
    print('Handled at higher level')
```

Тут ключове слово `raise` (без аргументів) повторно піднімає поточне виключення, яке обробляється.

Це дозволяє вам вести логування або виконувати очищення, одночасно передаючи помилку далі по стеку викликів.

Ви можете створювати і піднімати кастомні виключення, визначаючи власні класи виключень:

```py
class InsufficientFundsError(Exception):
    def __init__(self, balance, amount):
        self.balance = balance
        self.amount = amount
        super().__init__(f'Insufficient funds: ${balance} available, ${amount} requested')

def withdraw(balance, amount):
    if amount > balance:
        raise InsufficientFundsError(balance, amount)
    return balance - amount

try:
    new_balance = withdraw(100, 150)
except InsufficientFundsError as e:
    print(f'Transaction failed: {e}')
```

Тут видно, що кастомні класи виключень успадковуються від `Exception` або його підкласів.

Ви дізнаєтеся більше про класи та успадкування у майбутніх уроках. Поки що знайте, що це спосіб створити власні виключення з кастомною логікою.

Інструкція `raise` також може використовуватися з ключовим словом `from` для ланцюжка виключень, що показує зв’язок між різними помилками:

```py
def parse_config(filename):
    try:
        with open(filename, 'r') as file:
            data = file.read()
            return int(data)
    except FileNotFoundError:
        raise ValueError('Configuration file is missing') from None
    except ValueError as e:
        raise ValueError('Invalid configuration format') from e

config = parse_config('config.txt')
```

Тут видно, що `raise ... from None` пригнічує контекст оригінального виключення:

```bash
Traceback (most recent call last):
  File "main.py", line 12, in <module>
    config = parse_config('config.txt')
             ^^^^^^^^^^^^^^^^^^^^^^^^^^
  File "main.py", line 7, in parse_config
    raise ValueError('Configuration file is missing') from None
ValueError: Configuration file is missing
```

А `raise ... from e` ланцюжить нове виключення з оригінальним, зберігаючи слід помилки.

```bash
Traceback (most recent call last):
  File "main.py", line 5, in parse_config
    return int(data)
           ^^^^^^^^^
ValueError: invalid literal for int() with base 10: ''

The above exception was the direct cause of the following exception:

Traceback (most recent call last):
  File "main.py", line 12, in <module>
    config = parse_config('config.txt')
             ^^^^^^^^^^^^^^^^^^^^^^^^^^
  File "main.py", line 9, in parse_config
    raise ValueError('Invalid configuration format') from e
ValueError: Invalid configuration format
```

Ви також можете піднімати виключення умовно, використовуючи інструкції `assert`, які фактично є скороченням для `raise` з `AssertionError`:

```py
def calculate_square_root(number):
    assert number >= 0, 'Cannot calculate square root of negative number'
    return number ** 0.5

try:
    result = calculate_square_root(-4)
except AssertionError as e:
    print(f'Assertion failed: {e}')
```

Інструкція `raise` є необхідною для створення надійних застосунків, де потрібно забезпечувати бізнес-правила, перевіряти вхідні дані та надавати зрозумілі повідомлення про помилки. Стратегічно використовуючи `raise`, ви зробите свій код більш передбачуваним і легшим для налагодження, одночасно даючи користувачам чіткий зворотний зв’язок про те, що пішло не так.
