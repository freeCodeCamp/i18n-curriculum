# --description--

API와 그것이 웹 개발에서 왜 중요한지 알아보겠습니다. API는 애플리케이션 프로그래밍 인터페이스(Application Programming Interface)의 약자입니다. API는 소프트웨어 애플리케이션들이 서로 효율적으로 소통하고 데이터를 교환할 수 있도록 규칙과 프로토콜을 정합니다.

API는 이미 구현된 간단한 구성 요소를 바탕으로 개발자가 더 복잡한 기능을 만들 수 있게 해주는 구조라고 생각할 수 있습니다.

API에는 여러 종류가 있습니다. 웹 API는 웹 애플리케이션을 위해 특별히 설계된 API입니다. 클라이언트 측 JavaScript 개발을 위한 다양한 웹 API가 있지만, 이들은 JavaScript 자체의 일부가 아닙니다.

이러한 API는 보통 두 가지 주요 범주로 나뉩니다: 브라우저 API와 서드파티 API입니다.

브라우저 API는 브라우저에서 데이터를 노출합니다. 웹 개발자는 JavaScript를 사용해 이 데이터에 접근하고 조작할 수 있습니다.

또한 웹사이트 구조 조작, 이벤트 처리, 저장소 작업, 네트워크 통신 등 다양한 기능에 접근할 수 있게 해줍니다.

자주 사용되는 브라우저 API의 예로는 다음이 있습니다:

HTML 요소, 스타일, 속성을 조작할 수 있는 DOM API. 앞으로 수업에서 DOM에 대해 더 많이 배우게 될 것입니다. 웹 개발의 핵심 개념입니다.

사용자 기기에 데이터를 로컬로 저장하는 Storage API.

그리고 더 많은 것들이 있습니다.

사용자 상호작용을 처리하고 사용자의 현재 위치 같은 브라우저 정보를 얻기 위해 사용할 수 있는 다양한 브라우저 API가 있습니다.

비디오나 오디오 같은 다른 유형의 요소도 다룰 수 있습니다.

브라우저 API는 강력한 웹 애플리케이션을 만드는 데 유용한 도구입니다. 이들은 기본적으로 브라우저에 내장되어 있습니다.

하지만 기본적으로 브라우저에 내장되어 있지 않은 서드파티 API도 사용할 수 있습니다. 이를 사용하려면 어떤 방식으로든 해당 코드를 가져와야 합니다. 보통 서비스 사용법을 설명하는 자세한 문서가 함께 제공됩니다.

예를 들어, 웹사이트에 인터랙티브한 지도를 표시할 수 있는 Google Maps API가 있습니다.

서드파티 웹 API에는 날씨 API, 소셜 미디어 API, 결제 API, 데이터 API, 번역 API 등 다양한 종류가 있습니다. 필요에 맞는 API를 찾기만 하면 됩니다.

웹 API는 다양한 서비스와 데이터 소스와 상호작용할 수 있는 현대적인 웹 애플리케이션을 만드는 데 필수적인 도구입니다. 이들의 기능을 활용하는 방법을 배우면 동적이고 매력적인 웹사이트를 만드는 도구가 확장됩니다.
