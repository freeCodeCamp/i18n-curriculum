# --description--

الآن بعد أن أنشأت تطبيق Express أساسي، حان الوقت لتلقي نظرة أعمق على التوجيه وكيف يعمل.

في Express، التوجيه هو الطريقة التي تحدد بها نقاط النهاية لتطبيق الويب. يمكنك التفكير في المسارات كقواعد تخبر التطبيق كيف يستجيب عندما يزور المستخدم عنوان URL معين.

لننظر مرة أخرى إلى تطبيق Express البسيط الخاص بنا:

```javascript
const express = require("express");
const app = express();
const port = 3000;

app.get("/", (req, res) => {
  res.send("Hello World!");
});

app.listen(port, () => {
  console.log(`Example app listening on port ${port}`);
});
```

في هذا المثال، يحدد التطبيق مسارًا لعنوان URL الجذر (`/`). عندما يرسل العميل طلب GET إلى هذا المسار، يستجيب الخادم بالنص `Hello World!`. ثم يبدأ التطبيق بالاستماع للطلبات الواردة على المنفذ `3000`.

لكن هناك طرق مسار أخرى.

يتيح لك Express التعامل مع أنواع مختلفة من طلبات HTTP باستخدام طرق توجيه مختلفة. الأكثر شيوعًا هي:

* `app.get()` – يسترجع البيانات
    
* `app.post()` – ينشئ بيانات جديدة
    
* `app.put()` – يحدث البيانات الموجودة
    
* `app.delete()` – يحذف البيانات
    

كل طريقة تحدد كيف يجب أن يستجيب تطبيقك لطلب HTTP معين على مسار محدد.

إليك بعض الأمثلة البسيطة:

```javascript
app.post("/submit", (req, res) => {
  res.send("Form submitted!");
});

app.put("/update", (req, res) => {
  res.send("Item updated!");
});

app.delete("/delete", (req, res) => {
  res.send("Item deleted!");
});
```

في كل حالة، يستمع Express لطلب بطريقة HTTP ومسار URL معين، ثم يشغل دالة المعالِج المقابلة لإرسال استجابة إلى العميل.

يمكنك اختبار هذه المسارات باستخدام أدوات مثل Postman أو `curl`، التي تتيح لك إرسال أنواع مختلفة من طلبات HTTP إلى الخادم.

وهذا هو التوجيه في Express - ببساطة يتعلق بتحديد كيف يستجيب تطبيقك للطلبات المختلفة.
