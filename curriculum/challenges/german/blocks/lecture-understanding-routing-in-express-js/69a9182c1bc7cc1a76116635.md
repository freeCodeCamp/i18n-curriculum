# --description--

In Express sind **Route-Pfade** die URL-Muster, die bestimmen, wie der Server auf eingehende Anfragen reagiert. Wenn ein Benutzer eine bestimmte URL anfragt, vergleicht Express diese URL mit einem Route-Pfad und führt den entsprechenden Handler aus.

Route-Pfade werden innerhalb von Routemethoden wie `app.get()`, `app.post()` und so weiter definiert. Diese Pfade können statisch sein, wie `/home`, oder dynamisch, wobei Parameter verwendet werden, um Werte aus der URL zu erfassen, wie `/user/:id`.

In dieser Lektion konzentrieren wir uns auf statische Routen. Hier ist ein Beispiel:

```javascript
// Static route
app.get("/home", (req, res) => {
  res.send("Welcome to the homepage!");
});
```

In diesem Beispiel hört der Server auf GET-Anfragen an `/home` und antwortet mit einer einfachen Nachricht.

Die `app.get()`-Methode nimmt zwei Argumente entgegen:

* Das erste Argument, `"/home"`, ist der statische Route-Pfad (oder die URL), auf den der Server hört.
    
* Das zweite Argument ist eine Callback-Funktion, die ausgeführt wird, wenn eine GET-Anfrage an diesen Pfad gesendet wird.
    

Innerhalb des Callbacks sendet `res.send()` eine Antwort zurück an den Client. In diesem Fall sendet sie den Text `Welcome to the homepage!`, der im Browser angezeigt wird.

Hier ist ein weiteres Beispiel für eine statische Route:

```javascript
// Static route
app.get("/menu/drinks", (req, res) => {
  res.send("Welcome to the drinks menu!");
});
```

In diesem Fall hört der Server auf GET-Anfragen an der URL `/menu/drinks` und sendet eine Nachricht zurück, wenn diese Route angefragt wird.

Solche Routen können Sie verwenden, wenn Ihre Website mehrere zusammengehörige Seiten unter einem gemeinsamen Pfad gruppiert. Zum Beispiel könnte eine Restaurant-Website ihre Menüseiten so organisieren:

* `/menu/food`
    
* `/menu/drinks`
    
* `/menu/desserts`
    

Dieser Ansatz hilft dabei, Ihre URLs klar und aussagekräftig zu halten, und macht es außerdem einfacher, Ihre Routen zu organisieren und zu pflegen, wenn Ihre Anwendung wächst.

In der nächsten Lektion schauen wir uns dynamische Route-Pfade mit Parametern an.
