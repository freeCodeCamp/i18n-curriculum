# --description--

前のレッスンでは、属性にシングルアンダースコアとダブルアンダースコアを付けることについて学びました。

その違いを思い出すために説明すると、シングルアンダースコアはクラス内部での使用を意図しているという慣例であり、クラスの外部から直接アクセスすべきではないことを意味します。一方、ダブルアンダースコアは、その属性がクラスの外部から直接アクセスされるのを防ぎます。

以下は、その両者の動作を示す例です：

```py
class Example:
    def __init__(self):
        self._internal = 'I can be accessed from outside the class, but should not'
        self.__private = 'You cannot access me directly from outside the class'

obj = Example()

print(obj._internal) # I can be accessed from outside the class, but should not
print(obj.__private)  # AttributeError: 'Example' object has no attribute '__private'
```

属性にダブルアンダースコアを付けると、Pythonの名前マングリング処理が起動し、Pythonは内部的にその属性名を変更して、アンダースコアとクラス名を接頭辞として追加し、`__attribute`を`_ClassName__attribute`に変えます。

これを実際に見るために、クラスのインスタンスを作成し、そのインスタンスの`__dict__`という特別な属性を使います。これはオブジェクトの属性を含む辞書です：

```py
class Example:
    def __init__(self, internal, private):
        self._internal = internal
        self.__private = private

example1 = Example(
    'I can be accessed from outside the class, but should not',
    'I cannot be accessed directly from outside the class'
)

print(example1.__dict__)
```

結果は次のようになります：

```py
{
  '_internal': 'I can be accessed from outside the class, but should not',
  '_Example__private': 'I cannot be accessed directly from outside the class'
}
```

ご覧の通り、`__private`属性は`_Example__private`として保存されています。つまり、クラスの外部からでも次のようにしてその属性にアクセスできます：

```py
class Example:
    def __init__(self, internal, private):
        self._internal = internal
        self.__private = private

example1 = Example(
    'I can be accessed from outside the class, but should not',
    'I cannot be accessed directly from outside the class'
)
example2 = Example(
    'I should not be accessed from outside the class',
    'But I can be accessed from outside the class with name mangling'
)

print(example1._Example__private) # I cannot be accessed directly from outside the class
print(example2._Example__private) # But I can be accessed from outside the class with name mangling
```

では、なぜPythonは名前マングリングを行うのでしょうか？

名前マングリングの主な目的は、継承を使う際に属性やメソッドの誤ったオーバーライドを防ぐことです。以下の例でそれが明確になります：

```py
class Parent:
    def __init__(self):
        self.__data = 'Parent data'

class Child(Parent):
    def __init__(self):
        super().__init__()
        self.__data = 'Child data'

c = Child()
print(c.__dict__) # {'_Parent__data': 'Parent data', '_Child__data': 'Child data'}
```

`Parent`クラスとそれを継承する`Child`の両方に、それぞれ別々の`_class__data`属性があることがわかります。これは名前マングリングによって可能になっています。そうでなければ、`Child`が誤ってParentのデータを上書きしてしまいます。

もし両方のクラスの属性にダブルアンダースコアを付けず、名前マングリングを許可しなかった場合は次のようになっていたでしょう：

```py
class Parent:
   def __init__(self):
       self.data = 'Parent data'

class Child(Parent):
   def __init__(self):
       super().__init__()
       self.data = 'Child data'

c = Child()
print(c.__dict__)  # {'data': 'Child data'}
```

では、属性の接頭辞にシングルアンダースコア（`_`）とダブルアンダースコア（`__`）のどちらを使うべきでしょうか？状況によります。属性がクラス内部だけで使われる場合はシングルアンダースコアを使いましょう。

しかし、継承されるクラスを扱う場合は、親クラスの属性が上書きされないようにダブルアンダースコアを使うべきです。
