# --description--


通常，你将创建两种链接：

- 指向互联网其他网站页面的链接

- 指向你自己网站上页面的链接


## 绝对链接
指向互联网其他网站页面的链接称为绝对链接。典型的绝对链接由以下部分组成：`protocol://domain/path`。绝对链接始终包含目标的协议和域。

你已经见过绝对链接的实际应用。你之前为 The Odin Project 的 About 页面创建的链接是一个绝对链接，因为它包含了协议和域。

`https://www.theodinproject.com/about`

## 相对链接
指向你自己网站内其他页面的链接称为相对链接。相对链接不包含域名，因为它是同一站点上的另一个页面，默认域名与创建链接的页面相同。

相对链接只包含相对于你创建链接的页面的另一页面的文件路径。这是相当抽象的，让我们通过一个示例来演示。

在 `odin-links-and-images` 目录中，创建另一个名为 `about.html` 的超文本标记语言文件，并将以下代码粘贴到其中：

```html
<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8">
    <title>Odin Links and Images</title>
  </head>

  <body>
    <h1>About Page</h1>
  </body>
</html>
```

回到 `index` 页面，添加以下锚点元素以创建到 `about` 页面的链接：

```html
<body>
  <h1>Homepage</h1>
	<a href="https://www.theodinproject.com/about">click me</a>

	<a href="about.html">About</a>
</body>
```

在浏览器中打开 `index.html` 文件并点击关于链接，以确保所有内容正确连接。点击该链接应跳转到你刚刚创建的 `about` 页面。

这是可行的，因为 `index` 和 `about` 页面在同一个目录中。这意味着你可以直接使用它的名称（`about.html`）作为链接的 `href` 值。

但你通常会想更好地组织你的网站目录。通常你只会在根目录下有 `index.html`，所有其他超文本标记语言文件都放在它们自己的目录中。

在 `odin-links-and-images` 目录内创建一个名为 `pages` 的目录，并将 `about.html` 文件移动到此新目录中。

刷新浏览器中的 `index` 页面，然后点击 `about` 链接。现在它将无法使用。这是因为 `about` 页面文件的位置已更改。

为了解决这个问题，你只需更新 `about` 链接的 `href` 值，将 `pages/` 目录包含进去，因为这是相对于 `index.html` 文件的 `about.html` 文件的新位置。

```html
<body>
  <h1>Homepage</h1>
  <a href="pages/about.html">About</a>
</body>
```

刷新浏览器中的 `index` 页面，然后再次尝试点击 `about` 链接，它现在应该可以正常工作了。

在许多分支中，这种方法通常可以正常工作；然而，你仍然可能遇到意外的问题。在链接前添加 `./` 在大多数情况下可以防止此类问题。通过添加 `./`，你是在向你的代码指定它应从**当前**目录开始查找文件/目录。

```html
<body>
  <h1>Homepage</h1>
  <a href="./pages/about.html">About</a>
</body>
```

## --text--

绝对链接和相对链接有什么区别？

## --answers--

绝对链接是指向当前网站上另一个页面的链接。相对链接是指向另一个网站的链接。

---

绝对链接是指向其他网站的链接。相对链接是指向当前网站上另一个页面的链接。

---

绝对 `link` 和相对 `link` 之间没有区别。
