# --description--

الـ Getters و Setters هي طرق تتيح لك التحكم في كيفية وصول وتعديل خصائص الصنف. مع getters تسترجع قيمة، ومع setters تعيّن قيمة.

تتم هذه العمليات من خلال ما يُعرف بالخصائص. هي التي تربط بين getters و setters، وتسمح بالوصول إلى البيانات.

تتصرف الخصائص مثل الخصائص العادية لكنها تعمل كطرق تحت الغطاء. فكر فيها كبيانات تعرفها كطرق، لكنها تعمل كخصائص. هذا يعني أنه يمكنك الوصول إلى الخصائص باستخدام ترميز النقطة بدلاً من الأقواس العادية أو العادية.

الشيء الأساسي الذي تفعله الخصائص هو أنها تنفذ منطقًا إضافيًا خلف الكواليس عند الحصول على القيم أو تعيينها أو حذفها. هذا يجعلها الخيار المثالي عندما تريد الوصول إلى البيانات أو التلاعب بها داخل الكائنات.

فلماذا تستخدم الخصائص بدلاً من الطرق؟ الأمر يتعلق أساسًا بالقراءة والاصطلاح. تجعل شفرتك أنظف وأسهل في القراءة.

عندما تستخدم طريقة، عليك دائمًا استدعاؤها بالأقواس العادية. لكن مع الخاصية، يمكنك الوصول إليها مثل خاصية عادية باستخدام ترميز النقطة. هذا يجعل شفرتك تبدو بسيطة حتى عندما تقوم بعمل إضافي خلف الكواليس.

على سبيل المثال، قد تريد حساب قيمة أو التحقق من صحة قيمة جديدة قبل حفظها. بدلًا من استدعاء طريقة لذلك، يمكنك استخدام طريقة تشبه الخاصية للقيام بذلك.

في Python، الديكوريتور هو دالة تعدل وظائف دوال أو أصناف أخرى دون تغيير شفرتها الأصلية.

بينما يمكن إنشاء ديكوريتورات مخصصة، هذا يتجاوز نطاق هذا الدرس. فقط اعرف أنه لإنشاء خاصية، تعرف طريقة وتضع فوقها الديكوريتور `@property`. هذا يحول الطريقة إلى خاصية، بحيث يمكن الوصول إليها كخاصية بينما تستدعي داخليًا الطريقة المزينة.

هذا يقودنا إلى getters. إليك كيفية إنشاء واحد باستخدام الديكوريتور `@property`:

```py
class Circle:
    def __init__(self, radius):
        self._radius = radius

    @property
    def radius(self): # A getter to get the radius
        return self._radius
  
    @property
    def area(self):  # A getter to calculate area
        return 3.14 * (self._radius ** 2)

my_circle = Circle(3)

print(my_circle.radius) # 3
print(my_circle.area) # 28.26
```

هذا المثال يحصل على نصف القطر ومساحة الدائرة.

لاحظ كيف استخدمنا `_radius` بدلًا من radius داخل الصنف. الشرطة السفلية هي اصطلاح شائع في Python للدلالة على أن الخاصية خاصة. بمعنى آخر، تشير إلى أنها للاستخدام الداخلي ولا يجب الوصول إليها مباشرة من خارج الصنف.

لإنشاء setter لتعيين نصف القطر، على سبيل المثال، عليك تعريف طريقة أخرى بنفس الاسم واستخدام الديكوريتور `@<property_name>.setter` فوقها:

استخدام `self.radius` داخل `__init__` يضمن تشغيل setter أثناء إنشاء الكائن، لذا يتم التقاط قيم نصف القطر غير الصالحة فورًا.

```py
class Circle:
    def __init__(self, radius):
        self.radius = radius # Calling the setter

    @property
    def radius(self):  # A getter to get the radius
        return self._radius

    @radius.setter
    def radius(self, value):  # A setter to set the radius
        if value <= 0:
            raise ValueError('Radius must be positive')
        self._radius = value

my_circle = Circle(3)
print('Initial radius:', my_circle.radius) # Initial radius: 3

my_circle.radius = 8
print('After modifying the radius:', my_circle.radius) # After modifying the radius: 8
```

في هذا المثال، setter نصف القطر لا يعيّن نصف القطر فقط، بل ينفذ تحققًا للتأكد من أن نصف القطر ليس رقمًا سالبًا.

بمجرد تعريف getters و setters، تستدعيها Python تلقائيًا خلف الكواليس كلما استخدمت صيغة الخاصية العادية:

```py
my_circle.radius # This will call the getter
my_circle.radius = 4 # This will call the setter
```

لاحظ أنه داخل setter، لا يمكنك استخدام نفس اسم الخاصية عند تعيين قيمة جديدة. ذلك لأن `self.radius = value` سيستدعي setter داخل نفسه، مما يؤدي إلى تكرار لا نهائي وخطأ `RecursionError`. لذلك يجب دائمًا استخدام الشكل الذي يبدأ بشرطة سفلية `self._radius = value`.

تمامًا كما يمكنك التحكم في كيفية وصول الخاصية عبر getter وكيفية تعديلها عبر setter، يمكنك أيضًا التحكم في كيفية حذفها باستخدام deleter.

يُشغّل deleter منطقًا مخصصًا عند استخدام عبارة del على خاصية. لإنشاء واحد، تستخدم الديكوريتور `@<property_name>.deleter`:

```py
class Circle:
    def __init__(self, radius):
        self.radius = radius

    # Getter
    @property
    def radius(self):
        return self._radius

    # Setter
    @radius.setter
    def radius(self, value):
        if value <= 0:
            raise ValueError("Radius must be positive")
        self._radius = value

    # Deleter
    @radius.deleter
    def radius(self):
        print("Deleting radius...")
        del self._radius
```

إليك كيف يمكن استخدام deleter:

```py
# Create circle object with a radius
my_circle = Circle(33)
print("Initial radius:", my_circle.radius)  # 33

# Delete the radius
# This calls the deleter
del my_circle.radius # Deleting radius...
print("Radius deleted!") # Radius deleted!

# Try to access radius after deletion
try:
    print(my_circle.radius)
except AttributeError as e:
    print("Error:", e) # Error: 'Circle' object has no attribute '_radius'
```

النتيجة المستفادة من هذا هي:

- الـ Getters تتيح لك استرجاع قيمة أو حتى حساب قيمة في الوقت الفعلي.
- الـ Setters تتيح لك تعديل القيم بأمان عبر تنفيذ فحوصات قبل التعيين.
- الخصائص هي التي تربط بين getters و setters بحيث يمكنك كتابة منطق مع الاستمرار في استخدام ترميز النقطة.
- الـ Deleters تتيح لك تحديد ما يحدث عند حذف خاصية.
