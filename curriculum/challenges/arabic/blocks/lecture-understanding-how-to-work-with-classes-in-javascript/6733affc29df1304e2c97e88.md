# --interactive--

في JavaScript الحديثة، الأصناف تشبه المخططات التي يمكنك تعريفها في الشفرة لإنشاء عدة كائنات من نفس النوع.

على سبيل المثال، إذا كنت تصمم نظام جرد، ستحتاج إلى إنشاء عدة كائنات لكل نوع من المنتجات.

كل منتج فردي سيكون له خصائصه وإجراءاته الخاصة، لكن يمكن إنشاء منتجات من نفس النوع باستخدام نفس "المخطط". هذا هو الغرض من استخدام الأصناف.

يمكن تعريف الأصناف إما كتعبير صنف أو إعلان صنف.

إليك الصيغة النحوية الأساسية لإعلان صنف:

```js
class MyClassName {
  // Class Methods
  constructor() { ... }
  method1() { ... }
  method2() { ... }
  ...
}
```

تبدأ بكتابة الكلمة المفتاحية `class` متبوعة باسم الصنف. عند تسمية الأصناف، يجب استخدام اصطلاح التسمية `PascalCase` بدلاً من `camelCase`.

المحتوى داخل الأقواس المتعرجة يُعرف بجسم الصنف. هنا توضع دالة البناء والطرق.

دالة البناء هي طريقة خاصة تُستدعى تلقائيًا عند إنشاء كائن جديد من الصنف. تُستخدم عادةً لتهيئة خصائص الكائن.

في هذا المثال، يمكنك رؤية صنف `Dog` مع دالة بناء.

```js
class Dog {
  constructor(name) {
    this.name = name;
  }
}
```

تُستخدم الكلمة المفتاحية `this` هنا للإشارة إلى النسخة الحالية من صنف `Dog`، مما يسمح لك بتعيين خاصية `name` للكائن الجاري إنشاؤه. ستتعلم المزيد عن الكلمة المفتاحية `this` في درس لاحق.

أسفل دالة البناء، يمكنك إضافة ما يسمى بالطرق. الطرق هي دوال معرفة داخل الصنف تؤدي إجراءات أو عمليات. تُستخدم لتعريف السلوكيات التي يمكن لنسخ الصنف تنفيذها.

إليك مثالًا على إضافة طريقة تسمى `bark` إلى صنف `Dog`.

```js
class Dog {
  constructor(name) {
    this.name = name;
  }

  bark() {
    console.log(`${this.name} says woof!`);
  }
}
```

عند استدعاء طريقة `bark`، ستُسجل الرسالة في وحدة التحكم مع الاسم الفعلي الذي تم تمريره إليها. إذا أنشأت نسخة من `Dog` بالاسم `"Buddy"`، فإن استدعاء طريقة `bark()` على تلك النسخة سيطبع `"Buddy says woof!"` في وحدة التحكم.

بمجرد تعريف الصنف، يمكنك إنشاء نسخة منه باستخدام الكلمة المفتاحية `new`، واسم الصنف، والمعلمات الخاصة به.

في هذا المثال، ننشئ نسخة جديدة من `Dog` بالاسم `Gino` ثم نُعيّنها إلى المتغير `dog`.

```js
const dog = new Dog("Gino");
```

عندما تكتب `new Dog("Gino")` في الشفرة، يُنشأ الكائن الجديد وتُستدعى دالة البناء تلقائيًا بالمعلمات المعطاة.

يمكنك بعد ذلك الوصول إلى هذه الخاصية باستخدام ترميز النقطة، كما ترى في السطر التالي من الشفرة:

```js
console.log(dog.name);
```

إليك المثال الكامل:

:::interactive_editor

```js
class Dog {
  constructor(name) {
    this.name = name;
  }

  bark() {
    console.log(`${this.name} says woof!`);
  }
}

const dog = new Dog("Gino");
console.log(dog.name);
```

:::

يمكنك أيضًا استدعاء الطرق. في هذا المثال، نستدعي طريقة `bark()` على نسخة `dog`.

```js
dog.bark();
```

إليك المثال الكامل مع تضمين استدعاء الطريقة:

:::interactive_editor

```js
class Dog {
  constructor(name) {
    this.name = name;
  } 
  bark() {
    console.log(`${this.name} says woof!`);
  }
}

const dog = new Dog("Gino");
console.log(dog.name);
dog.bark();
```

:::

كما ذُكر سابقًا، يمكنك أيضًا تعريف الأصناف كتعبير صنف. هنا يكون الصنف مجهولًا ويتم تعيينه إلى متغير.

إليك كيف يبدو المثال السابق كتعبير صنف:

```js
const Dog = class {
  constructor(name) {
    this.name = name;
  }

  bark() {
    console.log(`${this.name} says woof!`);
  }
};
```

في معظم الأحيان، ستستخدم إعلانات الأصناف، لكن فهم تعبيرات الأصناف مهم أيضًا لأنها توفر مرونة أكبر، خاصة في حالات البرمجة الديناميكية.

باستخدام الأصناف وفهم صياغتها الأساسية، يمكنك إنشاء شفرة قابلة لإعادة الاستخدام، وتنظيمها في وحدات منطقية، وتعزيز تجزئة الشفرة.
