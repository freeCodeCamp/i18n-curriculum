# --description--

I grafi possono essere rappresentati in modi diversi. Qui descriviamo un modo, chiamato <dfn>lista di adiacenza</dfn>. Una lista di adiacenza è essenzialmente un elenco puntato dove a sinistra c’è il nodo e a destra sono elencati tutti gli altri nodi a cui è collegato. Di seguito è mostrata una rappresentazione di una lista di adiacenza.

<blockquote>Nodo1: Nodo2, Nodo3<br>Nodo2: Nodo1<br>Nodo3: Nodo1</blockquote>

Quello sopra è un grafo non orientato perché `Node1` è collegato a `Node2` e `Node3`, e questa informazione è coerente con le connessioni che mostrano `Node2` e `Node3`. Una lista di adiacenza per un grafo orientato significherebbe che ogni riga della lista mostra la direzione. Se quello sopra fosse orientato, allora `Node2: Node1` indicherebbe che il bordo orientato punta da `Node2` verso `Node1`. Possiamo rappresentare il grafo non orientato sopra come una lista di adiacenza inserendolo in un oggetto JavaScript.

```js
var undirectedG = {
  Node1: ["Node2", "Node3"],
  Node2: ["Node1"],
  Node3: ["Node1"]
};
```

Questo può anche essere rappresentato più semplicemente come un array dove i nodi hanno solo numeri invece di etichette stringa.

```js
var undirectedGArr = [
  [1, 2], // Node1
  [0],    // Node2
  [0]     // Node3
];
```

# --instructions--

assert(Object.keys(undirectedAdjList).length === 4);

assert(
