# --description--

Aprendamos sobre las opciones y flags de comando.

En una lección anterior, aprendiste sobre cómo pasar argumentos a comandos como `touch readme.md`. Las opciones, o flags, son argumentos especiales que puedes pasar a un comando y que afectan la forma en que se comporta. Los dos términos se usan indistintamente, aunque "flags" tiende a usarse más específicamente para opciones que funcionan como un interruptor de encendido/apagado.

Las opciones suelen tener como prefijo uno o dos símbolos de guion (`-`), lo que ayuda a proporcionar una distinción visual entre una opción y un argumento. Primero, veamos la sintaxis de dos guiones, o "forma larga".

Muchas aplicaciones, como `ls`, aceptan un indicador `--version`, que imprime la versión actual de la aplicación en el terminal en lugar de ejecutar los comandos de la aplicación.

Muchas aplicaciones también ofrecen una bandera `--help`, que muestra instrucciones sobre cómo usar la aplicación.

La sintaxis de un solo guion, o "forma corta", normalmente usa opciones que son una sola letra. Por ejemplo, la bandera `-a` con `ls` lista todos los archivos, incluidos los archivos ocultos que comienzan con un punto (`.`), como `.env`.

La ventaja de estas opciones cortas es que puedes encadenar múltiples flags juntos. En lugar de `ls --all --human-readable --size` puedes usar las letras individuales todas a la vez con `ls -ahs`.

Algunas opciones esperan que se les pase un valor.

Al usar opciones de forma larga, normalmente necesitas usar un signo igual. En esta sintaxis, el valor se concatena directamente a la opción con el símbolo igual (`=`). Aquí tienes un ejemplo de cómo modificar el comportamiento de `ls` para incluir o excluir colores:

```bash
ls --color=never
```

Al usar opciones en forma corta, normalmente separas el valor con un espacio. Por ejemplo, aquí están las opciones en forma larga y corta para establecer el `width` del resultado de `ls`:

```bash
ls -w 50
```

```bash
ls --width=50
```

Observa cómo la forma corta usa `-w 50`, mientras que la forma larga usa `--width=50`. Esta distinción es importante para saberla y así evitar pasar un valor de opción como un argumento posicional.

Pero si alguna vez tienes dudas, recuerda que normalmente puedes usar el flag `--help` para ver la sintaxis esperada de las opciones.
