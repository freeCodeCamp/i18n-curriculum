# --description--

ウェブサービスとREST APIは、あなたが毎日使うほとんどのアプリやウェブサイトの動力源です。これらは異なるアプリケーション同士が通信し、ネットワーク上、主にインターネットを介してデータを共有できるようにします。

API（アプリケーションプログラミングインターフェース）は、ソフトウェアが他のソフトウェアと通信するための一般的な方法です。ウェブサービスは、HTTPなどのウェブ技術を使ってネットワーク上、通常はインターネットを介してデータを交換する特定のタイプのAPIです。このため、すべてのウェブサービスはAPIですが、すべてのAPIがウェブサービスであるわけではありません。REST APIは、データのリクエストとレスポンスの方法に関する特定のルールに従う一般的なタイプのウェブサービスです。

このレッスンでは、ウェブサービスが何であるかを大まかに説明し、次のレッスンでREST APIの動作について詳しく説明します。

ウェブサービスはソフトウェア間の橋渡しのようなものと考えてください。ユーザーインターフェースはなく、直接使うことを目的としていません。代わりに、アプリケーション間でデータを交換することに完全に集中しています。

システムがウェブサービスと見なされるためには、以下の特徴を持っている必要があります：

* **サーバー上で動作する**：ウェブサービスはウェブサイトのようにオンラインでホストされますが、ユーザーが見るページの代わりにデータや機能を公開します
    
* **機械間通信**：ウェブサービスは人間が直接見るためのものではなく、モバイルアプリやウェブアプリが利用するためのもので、最終的にユーザーが見ることができるデータや機能を公開します
    
* **言語非依存**：ウェブサービスはJavaやC#などどんな言語で書かれていてもよく、異なる言語で書かれていてもどのアプリからでもやり取りできます
    
* **プラットフォーム非依存**：マシンがどのプラットフォームで動いているかは関係ありません。例えば、Linux上で動くウェブサービスがWindows上のデバイスとの通信を妨げることはありません
    
* **標準的なウェブプロトコルを使う**：ウェブサービスは通常、インターネット上でデータを転送するためにHTTP/HTTPSを使い、ファイル転送にはFTP、メールにはSMTPを使います
    

**REST**と**SOAP**は最もよく知られたウェブサービスのタイプですが、**XML-RPC**のような他の方法もあります。

**REST**は**表現状態転送**の略で、現在最も広く使われています。多くの最新アプリケーションはRESTfulウェブサービスに依存しています。なぜなら、シンプルで柔軟、そしてウェブと自然に連携できるからです。

REST APIはリソースを公開します。リソースは通常、`/users`や`/users/20`のようなURLパスで表されます。クライアントがリソースをリクエストすると、サーバーはそのリソースの表現を返します。この表現は多くの場合JSONですが、XMLやプレーンテキストの形式の場合もあります。

クライアントはHTTPヘッダーを使って希望するフォーマットを指定できます。例えば、`Accept`ヘッダーを使ってJSONレスポンスをリクエストすることができます。適切な`Accept`の値を設定すればよいのです。

```bash
Accept: application/json
```

また、XMLを指定する場合は次のようにします：

```bash
Accept: application/xml
```

返されるリソースがJSONの場合はこのように見えます：

```json
{
  "id": 20,
  "name": "John Doe",
  "email": "john@example.com",
  "role": "admin",
  "createdAt": "2025-09-22T10:30:00Z"
}
```

XMLの場合はこのようになります：

```xml
<user>
  <id>20</id>
  <name>John Doe</name>
  <email>john@example.com</email>
  <role>admin</role>
  <createdAt>2025-09-22T10:30:00Z</createdAt>
</user>
```

SOAPは**Simple Object Access Protocol**の略です。XMLデータ形式のみを使い、より厳格な基準とセキュリティが必要な企業レベルのシステムで最も一般的です。

SOAPデータの例は次の通りです：

```xml
<soapenv:Envelope xmlns:soapenv="http://schemas.xmlsoap.org/soap/envelope/">
   <soapenv:Body>
      <getUser>
         <userId>20</userId>
      </getUser>
   </soapenv:Body>
</soapenv:Envelope>
```

XML-RPCはリクエストとレスポンスのエンコードにXMLを使います。これにより軽量ですが、RESTより柔軟性は低くなります。

UDDIはデータ交換サービスではなく、企業が利用可能なウェブサービスをリストアップ、発見、説明するためのXMLベースのディレクトリです。
