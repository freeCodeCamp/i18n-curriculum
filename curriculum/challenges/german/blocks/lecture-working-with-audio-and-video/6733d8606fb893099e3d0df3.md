# --description--

Sie haben bereits etwas über die `HTMLMediaElement` und MediaStreams APIs gelernt, aber es gibt noch einige weitere APIs, die bei der Arbeit mit Video und Audio sehr hilfreich sein können.

Die erste ist die Screen Capture API. Wie der Name schon vermuten lässt, ermöglicht diese API, den Bildschirm eines Nutzers aufzunehmen. Diese API wird verfügbar gemacht, indem Sie die `getDisplayMedia()`-Methode des `mediaDevices`-Objekts aufrufen und den zurückgegebenen MediaStream verwenden.

Dann gibt es die MediaStream Recording API. Diese API arbeitet zusammen mit den MediaStreams APIs und erlaubt es Ihnen, einen MediaStream (oder sogar direkt einen `HTMLMediaElement`) aufzunehmen. Sie löst dann `dataavailable`-Ereignisse mit Blob-Nutzlasten aus, die Sie im lokalen Dateispeicher ablegen können.

Die Grundlage all dieser Technologien bildet die Media Source Extensions API. Die Media Source Extensions API ermöglicht es Ihnen beispielsweise, den Webcam-Feed eines Nutzers direkt mit der `srcObject`-Eigenschaft an ein Videoelement zu übergeben.

Und schließlich die Web Audio API, die alles hörbare im Web antreibt. Diese API umfasst wichtige Objekte wie einen `AudioBuffer` (der speziell einen Puffer mit Audiodaten darstellt) oder den `AudioContext`.

Damit sind unsere Lektionen zu Audio und Video abgeschlossen!
