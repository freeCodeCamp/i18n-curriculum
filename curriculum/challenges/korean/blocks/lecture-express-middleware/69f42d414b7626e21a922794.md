# --description--

Express의 에러 핸들링 미들웨어는 요청-응답 주기 중에 발생하는 에러를 처리하는 특별한 유형의 미들웨어입니다. 예기치 않은 에러를 잡아내고, 기록하며, 클라이언트에 적절한 응답을 제공하는 데 필수적입니다.

이전 수업에서 에러 핸들링에 대해 배운 것처럼, Express 에러 핸들링 미들웨어는 네 개의 매개변수 `err`, `req`, `res`, `next`를 가집니다. 이 미들웨어는 에러가 `next()` 함수에 전달될 때 호출됩니다.

다시 한 번 구문을 상기해 보세요:

```js
app.use((err, req, res, next) => {
  console.error(err.message);
  res.status(500).send("Internal Server Error");
});
```

기본 Express 애플리케이션에서 어떻게 보이는지 예시입니다:

```js
const express = require('express')
const app = express()

// A simple route that throws an error
app.get('/', (req, res) => {
  throw new Error('Something went wrong!')
})

// Error-handling middleware
app.use((err, req, res, next) => {
  console.error(err.stack); // Logs the error stack
  res.status(500).send('Something went wrong!')
})

app.listen(3000, () => {
  console.log('Server running on http://localhost:3000')
})
```

* 라우트 핸들러: 문제를 시뮬레이션하기 위해 의도적으로 에러를 발생시킵니다.
* 에러 핸들링 미들웨어: `app.use()`로 등록된 이 미들웨어는 `next(err)`를 통해 전달된 에러를 잡아내고, 클라이언트에 500 HTTP 응답과 일반적인 에러 메시지를 보냅니다.
* `err`: 미들웨어에 전달되는 에러 객체입니다.

에러 핸들링 미들웨어는 애플리케이션에서 문제가 발생해도 사용자가 적절한 응답을 받을 수 있도록 보장합니다. 항상 미들웨어 스택의 마지막에 이 미들웨어를 포함해 모든 에러를 잡아내는 것이 중요합니다.
