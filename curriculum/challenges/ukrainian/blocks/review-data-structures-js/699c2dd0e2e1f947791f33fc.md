## Алгоритми та нотація Big O

- **Алгоритми**: Набір однозначних інструкцій для розв’язання задачі або виконання завдання. Алгоритми мають завершуватися за скінченну кількість кроків, і кожен крок має бути точним і однозначним.

- **Нотація Big O**: Описує найгірший випадок продуктивності або швидкість зростання алгоритму зі збільшенням розміру вхідних даних. Зосереджується на тому, як зростає використання ресурсів із розміром вхідних даних, ігноруючи сталий множник і члени нижчого порядку.

### Загальні часові складності

- **O(1) - Константний час**: Алгоритм виконується за однаковий час незалежно від розміру вхідних даних.

:::interactive_editor

```js
function checkEvenOrOdd(number) {
  if (number % 2 === 0) {
    return "Even";
  } else {
    return "Odd";
  }
}

console.log(checkEvenOrOdd(4)) // "Even"
console.log(checkEvenOrOdd(5)) // "Odd"
```

:::

- **O(log n) - Логарифмічний час**: Час зростає повільно зі збільшенням вхідних даних. Зустрічається в алгоритмах, які багаторазово зменшують розмір задачі на частку (наприклад, бінарний пошук).

- **O(n) - Лінійний час**: Час виконання зростає пропорційно розміру вхідних даних.

```js
for (const grade of grades) { // grades is an array
  console.log(grade);
}
```

- **O(n log n) - Лог-лінійний час**: Загальна складність ефективних алгоритмів сортування, таких як Merge Sort і Quick Sort.

- **O(n²) - Квадратичний час**: Час виконання зростає квадратично. Часто зустрічається у вкладених циклах.

:::interactive_editor

```js
const n = 3;

for (let i = 0; i < n; i++) {
  for (let j = 0; j < n; j++) {
    console.log("Hello, World!");
  }
}
```

:::

### Просторова складність

- **O(1) - Константний простір**: Алгоритм використовує однакову кількість пам’яті незалежно від розміру вхідних даних.
- **O(n) - Лінійний простір**: Використання пам’яті зростає пропорційно розміру вхідних даних.
- **O(n²) - Квадратичний простір**: Використання пам’яті зростає квадратично зі збільшенням розміру вхідних даних.

## Техніки розв’язання задач

- **Розуміння задачі**: Прочитайте умову задачі кілька разів. Визначте вхідні дані, очікуваний вихід і як перетворити вхідні дані у вихідні.

- **Псевдокод**: Опис логіки алгоритму на високому рівні, незалежний від мови програмування. Використовує загальновживану мову, змішану з програмними конструкціями, такими як `IF`, `ELSE`, `FOR`, `WHILE`.

```md
GET original_string
SET reversed_string = ""
FOR EACH character IN original_string:
  ADD character TO THE BEGINNING OF reversed_string
DISPLAY reversed_string
```

- **Крайні випадки**: Специфічні, дійсні вхідні дані, що знаходяться на межах того, що алгоритм повинен обробляти. Завжди враховуйте та тестуйте крайні випадки.

## Масиви

- **Статичні масиви**: Мають фіксований розмір, визначений під час ініціалізації. Елементи зберігаються у суміжних ділянках пам’яті. Розмір не можна змінити під час виконання програми.

- **Динамічні масиви**: Можуть автоматично збільшуватися або зменшуватися під час виконання програми. Зміна розміру відбувається через автоматичне копіювання у більші масиви за потреби.

### Реалізація динамічних масивів у JavaScript

:::interactive_editor

```js
let numbers = [3, 4, 5, 6];
// Access elements
console.log(numbers[0])  // 3

// Update elements
numbers[2] = 16
console.log([...numbers]);  // [3, 4, 16, 6]

// Add elements
numbers.push(7)
console.log([...numbers]);  // [3, 4, 16, 6, 7] 

// Add at a specific index
numbers.splice(3, 0, 15);
console.log([...numbers]);  // [3, 4, 16, 15, 6, 7]

// Remove elements
numbers.splice(2, 1);  // Remove at specific index
console.log([...numbers]);  // [3, 4, 15, 6, 7]

numbers.pop()   // Remove last element
console.log([...numbers]); // [3, 4, 15, 6]
```

:::

### Часові складності для динамічних масивів

- **Доступ**: O(1)
- **Вставка в кінець**: O(1) у середньому, O(n) при необхідності зміни розміру
- **Вставка посередині**: O(n)
- **Видалення**: O(n) для середини, O(1) для кінця

## Стек

- **Стек**: Структура даних типу LIFO (останній зайшов — перший вийшов). Елементи додаються і видаляються лише з вершини.

- **Операція push**: Додавання елемента на вершину стеку. Часова складність: O(1).

- **Операція pop**: Видалення елемента з вершини стеку. Часова складність: O(1).

:::interactive_editor

```js
// Using JavaScript array as a stack

let stack = [];

// Push operations
stack.push(1);
stack.push(2);
stack.push(3);
console.log([...stack]);  // [1, 2, 3]

// Pop operations
let topElement = stack.pop();
console.log(topElement);  // 3
console.log([...stack]);  // [1, 2]
```

:::

## Черги

- **Черги**: Структура даних типу FIFO (перший зайшов — перший вийшов). Елементи додаються в кінець і видаляються з початку.

- **Операція enqueue**: Додавання елемента в кінець черги. Часова складність: O(1).

- **Операція dequeue**: Видалення елемента з початку черги. Часова складність: O(1).

:::interactive_editor

```js
// Using JavaScript array as a queue
let queue = [];

// Enqueue operations
queue.push(1);
queue.push(2);
queue.push(3);
console.log([...queue]);  // [1, 2, 3]

// Dequeue operations
let firstElement = queue.shift();
console.log(firstElement);  // 1
console.log([...queue]);  // [2, 3]
```

:::

## Зв’язані списки

- **Зв’язані списки**: Лінійна структура даних, де кожен вузол містить дані та референс на наступний вузол. Вузли з’єднані як ланцюг.

### Однозв’язні списки

- **Структура**: Кожен вузол має дані і один референс на наступний вузол.
- **Перебір**: Можна рухатися лише вперед від голови до хвоста.
- **Головний вузол**: Перший вузол у списку, зазвичай єдиний, до якого є прямий доступ.
- **Хвостовий вузол**: Останній вузол у списку, вказує на `null`.

### Операції та часові складності

- **Вставка на початок**: O(1)
- **Вставка в кінець**: O(n) — потрібно пройти до кінця
- **Вставка посередині**: O(n) — потрібно пройти до позиції
- **Видалення з початку**: O(1)
- **Видалення з кінця**: O(n) — потрібно знайти попередній вузол
- **Видалення посередині**: O(n) — потрібно знайти вузол

### Двозв’язні списки

- **Структура**: Кожен вузол має дані і два референси: на наступний і попередній вузол.
- **Перебір**: Можна рухатися в обох напрямках.
- **Пам’ять**: Потребує більше пам’яті, ніж однозв’язні списки через додатковий референс.

## Коли використовувати кожну структуру даних

- **Масиви (динамічні масиви)**: Коли потрібен впорядкований, індексований доступ і розмір невідомий заздалегідь
- **Стек**: Для операцій LIFO (функціонал скасування, оцінка виразів, повернення назад)
- **Черги**: Для операцій FIFO (планування завдань, пошук у ширину)
- **Зв’язані списки**: Коли часто потрібно вставляти/видаляти на початку, розмір невідомий, випадковий доступ не потрібен
