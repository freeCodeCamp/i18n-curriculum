# --description--

Redux ist ein Framework zur Zustandsverwaltung, das mit verschiedenen Webtechnologien verwendet werden kann, darunter React.

In Redux gibt es ein einzelnes Zustandsobjekt, das für den gesamten Zustand Ihrer Anwendung verantwortlich ist. Das bedeutet, wenn Sie eine React-Anwendung mit zehn Komponenten hätten und jede Komponente ihren eigenen lokalen Zustand hätte, würde der gesamte Zustand Ihrer Anwendung durch ein einziges Zustandsobjekt definiert, das im Redux `store` gespeichert ist. Dies ist das erste wichtige Prinzip, das Sie beim Lernen von Redux verstehen sollten: Der Redux-Speicher ist die einzige Quelle der Wahrheit, wenn es um den Anwendungszustand geht.

Das bedeutet auch, dass jedes Mal, wenn ein Teil Ihrer Anwendung den Zustand aktualisieren möchte, dies **unbedingt** über den Redux-Speicher erfolgen muss. Der unidirektionale Datenfluss erleichtert es, die Zustandsverwaltung in Ihrer Anwendung nachzuvollziehen.

# --instructions--

Der Redux `store` ist ein Objekt, das Anwendungs-`state` hält und verwaltet. Es gibt eine Methode namens `createStore()` auf dem Redux-Objekt, mit der Sie den Redux `store` erstellen. Diese Methode nimmt eine `reducer`-Funktion als erforderliches Argument entgegen. Die `reducer`-Funktion wird in einer späteren Aufgabe behandelt und ist im Code-Editor bereits für Sie definiert. Sie nimmt einfach `state` als Argument und gibt `state` zurück.

Deklarieren Sie eine `store`-Variable und weisen Sie ihr die `createStore()`-Methode zu, wobei Sie `reducer` als Argument übergeben.

**Hinweis:** Der Code im Editor verwendet die ES6-Standardargument-Syntax, um diesen Zustand mit dem Wert `5` zu initialisieren. Wenn Sie mit Standardargumenten nicht vertraut sind, können Sie sich den <a href="https://www.freecodecamp.org/learn/javascript-algorithms-and-data-structures/es6/set-default-parameters-for-your-functions" target="_blank" rel="noopener noreferrer nofollow">ES6-Abschnitt im Curriculum</a> ansehen, der dieses Thema behandelt.
