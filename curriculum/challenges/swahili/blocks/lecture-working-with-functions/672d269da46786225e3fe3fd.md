# --interactive--

Vitendakazi ni vipande vya msimbo vinavyoweza kutumika tena ambavyo hufanya kazi maalum au kuhesabu thamani. Fikiria vitendakazi kama mashine inayochukua ingizo fulani, kufanya baadhi ya operesheni juu yake, kisha kutoa matokeo. Hapa kuna mfano wa kutangaza kitendakazi:

```js
function greet() {
  console.log("Hello, Jessica!");
}
```

Katika mfano huu, tumetangaza kitendakazi kinachoitwa `greet`. Ndani ya kitendakazi hicho, tuna `console.log` inayochapisha ujumbe `Hello, Jessica!`. Ikiwa tungejaribu kuendesha msimbo huu, hatutaona ujumbe ukaonekana kwenye konsoli. Hii ni kwa sababu tunahitaji kuita kitendakazi.

Wito wa kitendakazi, au uanzishaji, ni wakati tunapotumia au kuendesha kitendakazi. Ili kuita kitendakazi, utahitaji kurejea jina la kitendakazi ikifuatiwa na mabano ya kawaida:

:::interactive_editor

```js
function greet() {
  console.log("Hello, Jessica!");
}

greet(); // "Hello, Jessica!"
```

:::

Sasa ujumbe wa `Hello, Jessica!` utaandikwa kwenye konsoli. Lakini vipi kama tungetaka ujumbe useme `Hello, Nick!` au `Hello, Anna!`? Hatutaki kuandika kitendakazi kipya kila tunapomkaribisha mtumizi tofauti. Badala yake, tunaweza kuunda kitendakazi kinachoweza kutumika tena kinachotumia vigezo na hoja za kitendakazi.

Vigezo hufanya kazi kama viwakilishi vya thamani zitakazopitishwa kwa kitendakazi wakati kinapoitwa. Huwezesha vitendakazi kupokea ingizo na kufanya kazi na ingizo hilo. Hoja ni thamani halisi zinazopitishwa kwa kitendakazi wakati kinapoitwa. Hapa kuna toleo lililosasishwa la kitendakazi cha `greet` kinachotumia vigezo na hoja:

:::interactive_editor

```js
function greet(name) {
  console.log("Hello, " + name + "!");
}

greet("Alice"); // Hello, Alice!
greet("Nick"); // Hello, Nick!
```

:::

`name` hutumika kama kigezo wakati mfuatano wa herufi `Alice` na `Nick` hutumika kama hoja. Sasa tuna kitendakazi kinachoweza kutumika tena ambacho kinaweza kutumika mara nyingi katika msimbo wetu kwa hoja tofauti.

Kitendakazi kinapomaliza utekelezaji wake, daima hurejesha thamani. Kwa chaguo-msingi, thamani ya return itakuwa `undefined`. Hapa kuna mfano:

:::interactive_editor

```js
function doSomething() {
  console.log("Doing something...");
}

let result = doSomething();
console.log(result); // undefined
```

:::

Ikiwa unahitaji kitendakazi chako kurudisha thamani maalum, basi utahitaji kutumia tamko la `return`. Hapa kuna mfano wa kutumia tamko la `return` kurudisha jumla ya thamani mbili:

:::interactive_editor

```js
function calculateSum(num1, num2) {
  return num1 + num2;
}

console.log(calculateSum(3, 4)); // 7
```

:::

Mara nyingi utatumia tamko la `return`, kwa sababu unaweza kutumia thamani hiyo iliyotolewa na kitendakazi baadaye katika msimbo wako.

Hadi sasa, tumekuwa tukifanya kazi na vitendakazi vyenye majina, lakini pia unaweza kuunda kile kinachoitwa kitendakazi kisicho na jina. Kitendakazi kisicho na jina ni kitendakazi kisicho na jina ambacho kinaweza kupewa thamani ya kigezo kama hiki:

:::interactive_editor

```js
const sum = function (num1, num2) {
  return num1 + num2;
};

console.log(sum(3, 4)); // 7
```

:::

Katika mfano huu, tuna kigezo cha `const` kinachoitwa `sum` na tunakiweka thamani ya kitendakazi kisicho na jina kinachorudisha jumla ya `num1` na `num2`. Kisha tunaweza kuita `sum` na kupitisha nambari `3` na `4` kupata matokeo ya `7`.

Vitendakazi vinaunga mkono vigezo vya chaguo-msingi, vinavyokuwezesha kuweka thamani za chaguo-msingi kwa vigezo. Thamani hizi za chaguo-msingi hutumika ikiwa kitendakazi kitaitwa bila hoja kwa kigezo hicho. Hapa kuna mfano:

:::interactive_editor

```js
function greetings(name = "Guest") {
  console.log("Hello, " + name + "!");
}

greetings(); // Hello, Guest!
greetings("Anna"); // Hello, Anna!
```

:::

Katika mfano huu, ikiwa hakuna hoja itakayotolewa kwa `name`, itakuwa na thamani ya chaguo-msingi `Guest`.

Kwa muhtasari, vitendakazi vinakuwezesha kuandika msimbo unaoweza kutumika tena na uliopangwa. Vinaweza kuchukua ingizo (vigezo), kufanya vitendo, na kurudisha matokeo.
