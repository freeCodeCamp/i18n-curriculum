# --description--

Ein `plugin` ist eine Komponente, die einer bestehenden Computeranwendung eine bestimmte Funktion hinzufügt. Wenn ein Programm `plugins` unterstützt, ermöglicht es Anpassungen. 
Im Kontext einer IDE kann ein `plugins` zusätzliche Funktionen bereitstellen, wie neue Werkzeuge zum Programmieren, Debuggen oder zur Verbesserung der Benutzeroberfläche.

Zum Beispiel ist ein Code-Formatter in einer IDE oft ein `plugin`.

# --questions--

## --text--

Was bedeutet `plugin` im Programmierkontext?

## --answers--

Es ist die Hauptsoftware, die auf einem Computer läuft.

### --feedback--

Ein `plugin` ist nicht die Hauptsoftware; es ist ein Add-on, das die Fähigkeiten der Hauptsoftware erweitert oder ergänzt.

---

Es ist eine Art Computervirus.

### --feedback--

Ein `plugin` ist eine legitime Softwarekomponente, kein Virus. Es soll einer Software zusätzliche Funktionen hinzufügen.

---

Es ist eine Softwarekomponente, die einer bestehenden Anwendung Funktionen hinzufügt.

---

Es ist eine eigenständige Anwendung, die keine andere Software zum Funktionieren benötigt.

### --feedback--

Ein `plugin` ist nicht eigenständig; es benötigt ein anderes Programm, um zu funktionieren, und fügt diesem Funktionen hinzu.
