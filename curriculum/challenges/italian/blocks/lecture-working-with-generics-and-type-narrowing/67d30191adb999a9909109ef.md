# --description--

Hai già visto un tipo generico in una lezione precedente: `Array<string>`. Ma di solito userai i tipi generici nelle funzioni. Infatti, possono essere considerati come un parametro speciale che fornisci a una funzione per controllare il comportamento della definizione del tipo della funzione.

Torniamo al nostro esempio precedente di una funzione per ottenere un valore casuale da un array:

```js
const getRandomValue = (array: string[]): string => {
  return array[Math.floor(Math.random() * array.length)];
}
```

La nostra funzione accetta un array di stringhe e restituisce una stringa. Ma cosa succede se volessimo passare un array di numeri e restituire un numero?

Potresti usare un tipo union:

```js
const getRandomValue = (array: (string|number)[]): (string|number) => {
  return array[Math.floor(Math.random() * array.length)];
}
const val = getRandomValue([1, 2, 4])
```

Ma con questo approccio, anche se abbiamo passato un array solo di numeri, TypeScript determina che la variabile `val` potrebbe essere un numero O una stringa.

Invece, puoi definire un tipo generico per la funzione:

```js
const getRandomValue = <T>(array: T[]): T => {
  return array[Math.floor(Math.random() * array.length)];
}
const val = getRandomValue([1, 2, 4])
```

Ci sono un paio di componenti importanti nel nostro nuovo esempio. Prima di tutto, la sintassi `<T>` dice a TypeScript che stai definendo un tipo generico `T` per la funzione. `T` è un nome comune per i tipi generici, ma in realtà puoi chiamarlo come vuoi (rispettando le convenzioni delle variabili di JavaScript).

Poi, diciamo a TypeScript che il parametro array è un array di valori che corrispondono al tipo generico, e che il valore restituito è un singolo elemento di quel tipo. Questo permette a TypeScript di rilevare correttamente che `val` sarà un numero, perché l'array che abbiamo passato è composto solo da numeri.

Ma per le funzioni che non controlli? Diamo un’occhiata al metodo `document.querySelector()`:

```js
const email = document.querySelector("#email");
console.log(email.value);
```

Poiché stiamo facendo una query con un selettore ID, TypeScript non sa che tipo di elemento stiamo interrogando. Sa che è un `Element` di qualche tipo, ma il nostro tentativo di consultare la proprietà `value` lancerà un errore perché l’interfaccia dell’elemento di livello superiore non ha quella proprietà.

Per fortuna, possiamo passare un argomento di tipo alla chiamata della funzione:

```js
const email = document.querySelector<HTMLInputElement>("#email");
console.log(email.value);
```

Nota come stiamo usando di nuovo la sintassi con parentesi angolari (`<HTMLInputElement>`), ma questa volta in una chiamata di funzione. Questo dice a TypeScript che l’elemento che ci aspettiamo di trovare sarà un elemento `input`, e non otteniamo più un errore sulla proprietà value perché gli elementi input hanno quella proprietà. Ovviamente, `querySelector` può restituire `null` se un elemento non viene trovato, e TypeScript lancerà un errore di compilazione qui perché non gestiamo quel caso.

Ma ne parlerai nella prossima lezione.
