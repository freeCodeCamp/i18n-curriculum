# --description--

Атрибути `async` і `defer` у HTML-елементах `script` відіграють ключову роль у тому, як JavaScript-файли завантажуються та виконуються на вебсторінках. Розуміння їх допоможе покращити продуктивність вашого сайту та досвід користувача. Коли ви додаєте `script` у свій HTML-файл, він виглядає так:

```js
<script src="example.js"></script>
```

Коли браузер знаходить цей тег `script`, він припиняє аналіз HTML, завантажує скрипт, виконує його, а потім продовжує аналіз HTML. Це може уповільнити завантаження вашої вебсторінки, особливо якщо скрипти великі.

Саме тут на допомогу приходять атрибути `async` і `defer`. Вони дають змогу завантажувати скрипти ефективніше.

Почнемо з атрибута `async`:

```js
<script src="example.js" async></script>
```

Додавши атрибут `async` до тегу `script`, браузер продовжить аналіз HTML, поки скрипт завантажується. Коли скрипт повністю завантажиться, браузер призупинить аналіз HTML, виконає скрипт, а потім відновить аналіз HTML. Це може суттєво прискорити завантаження сторінки.

Важливо пам’ятати, що async-скрипти виконуються одразу після завантаження, тому вони можуть запускатися не в тому порядку, який нам потрібен. Саме тут на допомогу приходить атрибут `defer`. Ось як виглядає атрибут `defer`:

```js
<script src="example.js" defer></script>
```

Атрибут `defer` схожий на атрибут `async`. Однак defer-скрипти не виконуються одразу після завантаження. Натомість вони чекають, поки аналіз HTML завершиться. Крім того, defer-скрипти виконуються у тому порядку, в якому вони з’являються в коді HTML.

Коротко: використовуйте `async` для скриптів, де порядок виконання не має значення, а `defer` — коли потрібно гарантувати правильний порядок запуску скриптів. Обидва атрибути можуть значно покращити час завантаження сторінки, дозволяючи браузеру продовжувати аналіз HTML, поки скрипти завантажуються у фоновому режимі.
