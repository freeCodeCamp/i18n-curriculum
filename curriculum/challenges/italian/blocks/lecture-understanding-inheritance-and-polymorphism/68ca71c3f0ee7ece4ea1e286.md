# --description--

In una lezione precedente, hai imparato a prefissare gli attributi con un singolo underscore e un doppio underscore.

Per ricordarti la differenza tra i due, un singolo underscore è una convenzione che indica che l’attributo è destinato all’uso interno nella classe e non dovrebbe essere consultato direttamente dall’esterno della classe. Il doppio underscore, invece, impedisce che quell’attributo venga consultato direttamente dall’esterno della classe.

Ecco un esempio che mostra come funzionano entrambi:

```py
class Example:
    def __init__(self):
        self._internal = 'I can be accessed from outside the class, but should not'
        self.__private = 'You cannot access me directly from outside the class'

obj = Example()

print(obj._internal) # I can be accessed from outside the class, but should not
print(obj.__private)  # AttributeError: 'Example' object has no attribute '__private'
```

Prefissare un attributo con un doppio underscore attiva il processo di name mangling di Python, in cui Python rinomina internamente l’attributo aggiungendo un underscore e il nome della classe come prefisso, trasformando `__attribute` in `_ClassName__attribute`.

Per vedere questo in azione, crei un’istanza della classe e usi il `__dict__` attributo speciale di quell’istanza, che è un dizionario contenente gli attributi dell’oggetto:

```py
class Example:
    def __init__(self, internal, private):
        self._internal = internal
        self.__private = private

example1 = Example(
    'I can be accessed from outside the class, but should not',
    'I cannot be accessed directly from outside the class'
)

print(example1.__dict__)
```

Il risultato sarebbe:

```py
{
  '_internal': 'I can be accessed from outside the class, but should not',
  '_Example__private': 'I cannot be accessed directly from outside the class'
}
```

Come puoi vedere, l’attributo `__private` è archiviato come `_Example__private`. Questo significa che puoi ancora consultare quell’attributo dall’esterno della classe in questo modo:

```py
class Example:
    def __init__(self, internal, private):
        self._internal = internal
        self.__private = private

example1 = Example(
    'I can be accessed from outside the class, but should not',
    'I cannot be accessed directly from outside the class'
)
example2 = Example(
    'I should not be accessed from outside the class',
    'But I can be accessed from outside the class with name mangling'
)

print(example1._Example__private) # I cannot be accessed directly from outside the class
print(example2._Example__private) # But I can be accessed from outside the class with name mangling
```

Allora, perché Python fa il name mangling?

Lo scopo principale del name mangling è prevenire la sovrascrittura accidentale di attributi e metodi quando usi l’ereditarietà. Ecco un esempio che lo rende chiaro:

```py
class Parent:
    def __init__(self):
        self.__data = 'Parent data'

class Child(Parent):
    def __init__(self):
        super().__init__()
        self.__data = 'Child data'

c = Child()
print(c.__dict__) # {'_Parent__data': 'Parent data', '_Child__data': 'Child data'}
```

Puoi vedere che sia la classe `Parent` sia la `Child` che eredita da essa hanno i loro attributi `_class__data` separati. Questo è possibile grazie al name mangling. Altrimenti, il `Child` avrebbe sovrascritto accidentalmente i dati del genitore.

Ecco cosa sarebbe successo senza permettere a Python di fare il name mangling, cioè se non prefissi gli attributi in entrambe le classi con un doppio underscore:

```py
class Parent:
   def __init__(self):
       self.data = 'Parent data'

class Child(Parent):
   def __init__(self):
       super().__init__()
       self.data = 'Child data'

c = Child()
print(c.__dict__)  # {'data': 'Child data'}
```

Quindi, quale dovresti usare per prefissare gli attributi tra singolo underscore (`_`) e doppio underscore (`__`)? Dipende. Se un attributo è destinato solo all’uso interno nella classe, usa il singolo underscore.

Ma se stai lavorando con una classe che sarà ereditata, dovresti usare il doppio underscore così l’attributo del genitore non viene sovrascritto.
