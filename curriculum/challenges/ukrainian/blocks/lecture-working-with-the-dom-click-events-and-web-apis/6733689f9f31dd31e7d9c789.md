# --description--

Давайте дізнаємося про вузли DOM та їхні взаємовідносини в дереві DOM.

Так само, як у справжнього дерева є великі й малі гілки, з’єднані ієрархічною структурою, вузли DOM також мають прямі та непрямі зв’язки між собою. Ми використаємо цей приклад, щоб проілюструвати ці взаємовідносини:

```html
<!DOCTYPE html>
<html>
  <head>
    <title>DOM Tree Example</title>
  </head>
  <body>
    <h1>Heading 1</h1>
    <p>Paragraph 1</p>
    <ul>
      <li>List item 1</li>
      <li>List item 2</li>
    </ul>
  </body>
</html>
```

Почнемо зверху. Коренем дерева DOM є елемент `html`. Це контейнер найвищого рівня для всього вмісту HTML-документа. Усі інші вузли є нащадками цього кореневого вузла.

Потім, нижче кореневого вузла, ми знаходимо інші вузли в ієрархії. Батьківський вузол — це елемент, який містить інші елементи. Дочірній вузол — це елемент, який міститься всередині іншого елемента.

У цьому прикладі елемент `body` є батьківським для елемента абзацу, тоді як елемент абзацу є дочірнім для елемента `body`. Ці елементи представлені як вузли в дереві DOM із такими ж взаємовідносинами.

Так само, як у нас є батьківські та дочірні вузли, у нас також є братські вузли. Братські вузли — це елементи, які мають спільного батьківського елемента. У нашому прикладі два елементи списку мають спільного батьківського елемента — невпорядкований список, тому вони є братськими.

Аналогічно, елементи `h1` і абзац є братськими, бо мають спільного батьківського елемента `body`.

Також існують непрямі взаємовідносини на різних рівнях ієрархії. Нащадки — це елементи, які містяться всередині іншого елемента, безпосередньо або опосередковано. Елемент вважається нащадком іншого, якщо до нього можна дістатися, рухаючись вниз по дереву DOM від його предка.

У нашому прикладі елементи списку `li` є нащадками елемента `body`, оскільки вони опосередковано містяться в ньому.

Предок — це елемент, який знаходиться вище в ієрархії дерева DOM відносно іншого елемента. У нашому прикладі елемент `body` є предком елементів списку `li`.

Розуміння цих взаємовідносин є важливим для маніпулювання та навігації деревом DOM за допомогою JavaScript.
