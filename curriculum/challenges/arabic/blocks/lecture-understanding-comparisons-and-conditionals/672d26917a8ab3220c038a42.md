# --interactive--

في JavaScript، `null` و `undefined` هما نوعان مختلفان من البيانات تمثلان غياب القيمة، لكنهما يتصرفان بشكل مختلف في المقارنات. فهم كيفية تفاعل هذه الأنواع في سيناريوهات المقارنة المختلفة ضروري لكتابة شفرة قوية وخالية من الأخطاء البرمجية.

لنبدأ بنوع `undefined`. المتغير يكون `undefined` عندما يُعلن عنه دون تعيين قيمة له. هذه هي القيمة الافتراضية للمتغيرات غير المهيأة والمعلمات التي لم تُمرر لها معلمة.

أما نوع `null`، فهو قيمة تعيين تمثل عدم وجود قيمة مقصود. يُستخدم غالبًا للدلالة على أن المتغير لا يحتوي على قيمة عن قصد.

عند مقارنة `null` و `undefined` باستخدام معامل المساواة (`==`)، تقوم JavaScript بإجبار النوع. هذا يعني أنها تحاول تحويل القيم إلى نفس النوع قبل إجراء المقارنة. في هذه الحالة، يُعتبر `null` و `undefined` متساويين:

:::interactive_editor

```js
console.log(null == undefined); // true
```

:::

ومع ذلك، عند استخدام معامل المساواة الصارمة (`===`)، الذي يتحقق من القيمة والنوع دون إجراء إجبار النوع، لا يكون `null` و `undefined` متساويين:

:::interactive_editor

```js
console.log(null === undefined); // false
```

:::

هذا الاختلاف مهم أن تضعه في اعتبارك عند كتابة عبارات شرطية أو إجراء فحوصات المساواة في شفرتك. عند مقارنة `null` أو `undefined` مع قيم أخرى باستخدام معامل المساواة (`==`)، قد يكون السلوك غير متوقع. على سبيل المثال:

:::interactive_editor

```js
console.log(null == 0);  // false
console.log(null == ''); // false
console.log(undefined == 0); // false
console.log(undefined == ''); // false
```

:::

تعيد هذه المقارنات القيمة `false` لأن `null` و `undefined` متساويان فقط مع بعضهما البعض (ومع نفسيهما) عند استخدام معامل المساواة. سلوك `null` في المقارنات الأخرى معقد بشكل خاص:

:::interactive_editor

```js
console.log(null > 0);  // false
console.log(null == 0); // false
console.log(null >= 0); // true
```

:::

أما `undefined`، فإنه يتحول دائمًا إلى `NaN` في السياقات الرقمية، مما يجعل جميع المقارنات الرقمية مع `undefined` تعيد `false`:

:::interactive_editor

```js
console.log(undefined > 0);  // false
console.log(undefined < 0);  // false
console.log(undefined == 0); // false
```

:::

نظرًا لهذه الفروق الدقيقة، يُنصح عمومًا باستخدام معامل المساواة الصارمة عند مقارنة القيم، خاصة عند التعامل مع `null` و `undefined`. تساعد هذه الطريقة على تجنب إجبار النوع غير المتوقع وتجعل سلوك شفرتك أكثر قابلية للتنبؤ.

باختصار، بينما يُستخدم كل من `null` و `undefined` لتمثيل غياب القيمة، فإنهما يتصرفان بشكل مختلف في المقارنات. فهم هذه الاختلافات هو مفتاح لكتابة شفرة JavaScript واضحة وخالية من الأخطاء.
