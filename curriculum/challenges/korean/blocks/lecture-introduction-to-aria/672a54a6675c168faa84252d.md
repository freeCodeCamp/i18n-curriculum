# --interactive--

장애가 있는 사람을 포함해 모든 사용자가 문제없이 웹사이트에 접근할 수 있도록 하는 것이 중요합니다.

화면 읽기 프로그램을 사용하는 사람들에게 `aria-label`와 `aria-labelledby` 속성은 불분명하거나 보이지 않는 페이지 요소에 대한 중요한 정보를 제공합니다.

`aria-label`와 `aria-labelledby` 속성이 무엇인지, 그리고 시각 장애 및 관련 장애가 있는 사람들이 웹에 접근할 수 있도록 하는 데 어떤 역할을 하는지 살펴보겠습니다.

`aria-label`와 `aria-labelledby` 모두 aria 접두사가 붙어 있는 것을 알 수 있습니다. 그렇다면 이게 무슨 뜻일까요? ARIA는 Accessible Rich Internet Applications의 약자입니다. 이는 `aria-` 접두사가 붙은 속성 집합으로, 개발자가 보조 기술에 요소의 목적을 전달할 수 있게 해줍니다. `aria-label` 속성은 인터랙티브 요소에 보이지 않는 레이블을 추가합니다. 화면 읽기 프로그램이 읽을 수 있는 텍스트 레이블을 요소에 추가하는 것입니다.

`aria-label`는 특히 눈에 보이는 텍스트가 없지만 화면 읽기 프로그램으로 설명해야 하는 요소에 유용합니다. 예를 들어, 아이콘만 있는 버튼은 종종 `aria-label`가 필요해 그 목적을 전달합니다.

예를 들어 보겠습니다:

```html
<button aria-label="Search">
  <i class="fa-solid fa-magnifying-glass"></i>
</button>
```

이 경우, 버튼에 아이콘만 있어도 화면 읽기 프로그램은 이 버튼을 `Search, button`라고 알릴 수 있습니다. `aria-label` 속성은 화면 읽기 프로그램에 아이콘 대신 사용할 텍스트를 알려줍니다.

버튼에 아이콘 대신 "Search"라는 텍스트가 있다면, 텍스트가 버튼의 레이블 역할을 하므로 `aria-label` 속성은 필요하지 않습니다.

입력 요소의 경우, `aria-label` 속성은 입력과 연결된 눈에 보이는 레이블이 없을 때 직접 레이블을 제공합니다.

`aria-labelledby` 속성은 `aria-label` 속성과 똑같은 역할을 하지만, 속성에 직접 텍스트를 정의하는 대신 페이지에 이미 존재하는 텍스트를 참조하는 데 사용합니다. 기존 텍스트는 `id` 속성을 가져야 하며, `aria-labelledby` 속성의 참조 값으로 사용됩니다.

예를 들어 보겠습니다:

:::interactive_editor

```html
<input type="text" aria-labelledby="search-btn">
<button type="button" id="search-btn">Search</button>
```

:::

이 경우, 버튼의 텍스트가 검색 입력의 레이블로 사용됩니다. 화면 읽기 프로그램은 입력을 `Search, edit`처럼 알릴 것입니다. 나중에 버튼 텍스트를 `Find`로 변경하면, 입력의 레이블도 버튼 텍스트를 참조하기 때문에 자동으로 새 텍스트로 업데이트됩니다.
여러 `id` 값을 하나의 `aria-labelledby` 속성 값으로 결합하는 것도 가능합니다. 작동 방식은 다음과 같습니다:

:::interactive_editor

```html
<div>
  <span id="volume-label">Volume</span>
  <span id="volume-details">Adjust the volume level</span>
  <input
    type="range"
    min="0"
    max="100"
    value="30"
    aria-labelledby="volume-label volume-details">
</div>
```

:::

슬라이더의 경우, 화면 읽기 프로그램은 `volume-label`와 `volume-details` 요소의 내용을 찾아 `Volume Adjust the volume level`를 알릴 것입니다.

`aria-label`와 `aria-labelledby` 속성 모두 화면 읽기 프로그램이 요소의 역할을 이해하도록 돕는다는 것을 보았습니다. 그렇다면 어느 것을 사용해야 할까요? 둘 다 같은 기능을 제공하므로 어느 쪽을 사용해도 되지만, `aria-labelledby`를 `aria-label`보다 사용하는 데 몇 가지 장점이 있을 수 있습니다:

- 페이지 내용을 번역하는 번역 서비스를 사용하는 경우, `aria-label` 속성의 텍스트가 항상 번역되지 않을 수 있습니다.
- `aria-labelledby`를 사용하면 눈에 보이는 레이블 텍스트와 화면 읽기 프로그램용 보이지 않는 레이블 간 불일치를 방지할 수 있습니다. 눈에 보이는 텍스트를 업데이트하면 보이지 않는 레이블도 자동으로 업데이트되기 때문입니다.
- `aria-labelledby`는 여러 텍스트 출처로 구성된 복잡한 보이지 않는 레이블을 프로그래밍 방식으로 쉽게 만들 수 있게 합니다.

마지막으로, 한 요소에 `aria-label`와 `aria-labelledby`를 동시에 사용하지 마세요. 이 경우 화면 읽기 프로그램용 보이지 않는 레이블은 항상 `aria-labelledby`에 의해 결정되며 `aria-label`는 완전히 무시됩니다.
