# --description--

提升「time to usable」，換句話說，就是從使用者請求頁面到他們能夠有意義地互動之間的時間間隔，對於增強使用者體驗至關重要。

讓我們探索有效策略，以達成更快的可用時間。

首先專注於使用者最先看到的內容。這表示要立即載入必要的元素，並延緩非關鍵組件。

例如，你可以嘗試實作 lazy loading。

我們已經涵蓋了延遲載入，但這裡有一個延遲載入影像的範例，供你複習：

```html
<img src="image.jpg" loading="lazy" alt="Description">
```

你只需要為 `loading="lazy"` 屬性添加到影像標記。這會延緩影像的載入直到它們被需要，通常是在它們進入使用者的視窗時。透過只在影像和影片進入視窗時載入，我們不僅節省了頻寬，也加快了初始呈現的速度。

通常，網頁會一次載入所有影像，即使那些未顯示在螢幕上的影像也是如此。使用延遲載入時，位於摺疊線以下，換句話說，螢幕外的影像，只有當使用者向下捲動到它們時才會被載入。

以下是延遲載入 `iframe` 的範例：

```html
<iframe src="video.html" loading="lazy"></iframe>
```

你可以將之前使用過的相同屬性套用到 `iframes`，相同的概念也會適用。

你也可以使用 `defer` 屬性來延緩非關鍵 JavaScript，直到初始頁面載入之後。以下是一個我們這麼做的範例：

```html
<script src="non-critical.js" defer></script>
```

你也可以考慮將阻礙呈現的資源最小化。

會阻擋呈現的資源會延遲頁面變為可互動。

為避免這種情況，你可以嘗試非同步載入 CSS。對於非關鍵 CSS，使用 `media` 屬性有條件地載入樣式表。以下是我們這樣做的範例：

```html
<link rel="stylesheet" href="print.css" media="print">
```

透過將媒體值指定為 `print`，`print.css` 只會在頁面被列印或在螢幕上預覽時載入。在下一個範例中，透過將 `min-width: 800px` 作為 `media` 屬性的值，我們表示只想讓 `desktop.css` 在寬度大於 800px 的螢幕上載入：

```html
<!-- This stylesheet is only loaded on screens wider than 800px -->
<link rel="stylesheet" href="desktop.css" media="(min-width: 800px)">
```

這會透過跳過不必要的樣式來節省頻寬。

總結來說，透過實作這些策略，你可以大幅減少網站的「可用時間」，確保使用者能夠即時與你的內容互動。
