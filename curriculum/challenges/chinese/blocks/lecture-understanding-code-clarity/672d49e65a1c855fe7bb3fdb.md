# --description--

程序设计中的注释用于为代码提供额外的上下文，或为自己和他人留下备注。

注释是文本行或文本块， JavaScript 引擎在执行代码时会将其忽略。 它们的存在完全是为了方便阅读代码的人，无论是你自己还是其他人。

JavaScript 提供了两种在代码中添加注释的方法：单行注释和多行注释。

单行注释使用两个正斜杠（`//`）创建。 这是一个示例：

```js
// I am a single line comment in JavaScript
```

这种类型的注释适用于简短的解释或说明。

下面是 freeCodeCamp 课程项目文件中的一个真实案例：

```js
// This is to allow English to build without having to download the i18n files.
// It fails when trying to resolve the i18n-curriculum path if they don't exist.
const curriculumLocale = process.env.CURRICULUM_LOCALE ?? 'english';
const I18N_CURRICULUM_DIR = path.resolve(
  __dirname,
  curriculumLocale === 'english' ? '.' : 'i18n-curriculum/curriculum'
);
```

如果你不理解代码的实际作用，不要担心，因为你还没有学到这些知识。 相反，请关注开发人员留下的注释。 这一注释为该代码的存在提供了重要背景。

这样的注释对于团队工作的人来说非常重要，原因有二：

1. 其他参与项目的开发人员会理解这段代码的目的。

1. 这有助于防止在未征求团队意见的情况下进行不必要的更改或删除，从而导致错误或问题。

另一种类型的注释是多行注释。 这是基本语法：

```js
/*
 I am a multiline comment.
 This is helpful for longer explanations.
*/
```

当你需要在代码中编写较长的说明、解释或备注时，多行注释非常有用。

让我们再来看看 freeCodeCamp 课程项目文件，看看在现实世界中如何使用多行注释。

```js
/* Since there can be more than one way to complete a certification (using the
legacy curriculum or the new one, for instance), we need a certification
field to track which certification this belongs to. */
const dupeCertifications = [
  {
    certification: 'responsive-web-design',
    dupe: '2022/responsive-web-design'
  }
];
const hasDupe = dupeCertifications.find(
  cert => cert.dupe === meta.superBlock
);
```

和前面一样，忽略所有 JavaScript 代码，因为它使用的概念还没有讲过。 相反，请关注开发人员留下的注释。

团队中的开发人员，甚至是项目中的新贡献者，都能理解为什么会出现这段代码，并在处理项目的这一领域之前掌握完整的上下文。

虽然注释在编程中很有用，但避免过度注释也很重要。 你不需要对每一行代码都进行注释，尤其是在代码简单明了的情况下。

下面就是一个用注释来解释显而易见的代码的例子：

```js
// This code uses the const keyword to create a new variable called price.
// We are assigning the number 10 to the price variable.
const price = 10;
```

在这种情况下，无需在此处添加任何注释，因为代码的内容不言自明。 注释的目的是提高代码的可读性，而不是用不必要的解释来干扰代码。

如果你想在学习代码的过程中为个人项目添加注释，是没有问题的。 但是，一旦你开始与其他开发人员一起开发实际项目，就不要在不言自明的代码中使用注释，这一点很重要。

同样重要的是，不要使用注释来帮助解释混乱、过于复杂或写得不好的代码。 在这种情况下，最好重构或更改代码，以便其他开发人员更好地理解发生了什么。

注释是记录代码并使其更易于理解的强大工具。 你应使用注释来提供上下文，或为自己和他人留下备注。
