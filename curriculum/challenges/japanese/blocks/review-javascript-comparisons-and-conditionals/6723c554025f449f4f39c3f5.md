## 比較と `null` および `undefined` データ型

- **比較と `undefined`**: 変数が宣言されたが値が代入されていない場合、その変数は `undefined` です。これは初期化されていない変数や引数が渡されなかった関数のパラメータのデフォルト値です。`undefined` は数値コンテキストで `NaN` に変換されるため、`undefined` とのすべての数値比較は `false` を返します。

:::interactive_editor

```js
console.log(undefined < 0); // false (NaN < 0 is false)
console.log(undefined >= 0); // false (NaN >= 0 is false)
```

:::

- **比較と `null`**: `null` 型は値が意図的に存在しないことを表します。`null` は数値コンテキストで `0` に変換されるため、数値比較で予期しない動作を引き起こす場合があります。

:::interactive_editor

```js
console.log(null < 0); // false (0 < 0 is false)
console.log(null >= 0); // true (0 >= 0 is true)
```

:::

- 等値演算子（`==`）を使う場合、`null` と `undefined` はお互いと自分自身にのみ等しいです。

:::interactive_editor

```js
console.log(null == undefined); // true
console.log(null == 0); // false
console.log(undefined == NaN); // false
```

:::

- しかし、型変換を行わず値と型の両方をチェックする厳密等値演算子（`===`）を使う場合、`null` と `undefined` は等しくありません。

:::interactive_editor

```js
console.log(null === undefined); // false
```

:::

## `switch` 文

- **定義**: `switch` 文は式を評価し、その値を一連の `case` 節と照合します。マッチが見つかると、そのケースに関連付けられたコードブロックが実行されます。各ケースの最後には `break` 文を置いて、その実行を終了し次に進む必要があります。`default` ケースは任意のケースで、他のケースにマッチしなかった場合のみ実行されます。`default` ケースは `switch` 文の最後に置かれます。

:::interactive_editor

```js
const dayOfWeek = 3; 

switch (dayOfWeek) {
  case 1:
    console.log("It's Monday! Time to start the week strong.");
    break;
  case 2:
    console.log("It's Tuesday! Keep the momentum going.");
    break;
  case 3:
    console.log("It's Wednesday! We're halfway there.");
    break;
  case 4:
    console.log("It's Thursday! Almost the weekend.");
    break;
  case 5:
    console.log("It's Friday! The weekend is near.");
    break;
  case 6:
    console.log("It's Saturday! Enjoy your weekend.");
    break;
  case 7:
    console.log("It's Sunday! Rest and recharge.");
    break;
  default:
    console.log("Invalid day! Please enter a number between 1 and 7.");
}
```

:::
