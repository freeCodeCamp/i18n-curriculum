# --description--

在 Express 中，`express.Router()` 是一个强大的特色，帮助你将路由组织成更小、更模块化的部分。这在你的应用不断增长时尤其有用。

你可以不用把所有路由都定义在一个大文件里，而是通过路由器拆分到不同的文件中。这让你的代码更易读、易维护且更易扩展。

`express.Router()` 方法创建一个新的路由器对象。

路由器就像一个迷你 Express 应用。它可以处理自己的路由和中间件，但它不是一个完整的 Express 应用。一旦你在路由器上定义了路由，就可以用 `app.use()` 将它挂载到你的主应用中。

换句话说，路由器让你把相关的路由分组，然后再集成到主应用中。

使用路由器有几个好处：

**关注点分离**

你可以在不同文件中定义路由，这提升了应用的组织性和可读性。

**代码复用**  
路由器可以在应用的不同部分甚至不同项目中复用。

**主文件更简洁**  
你不用把主 `app.js` 或 `server.js` 文件塞满各种路由定义，可以把它们放到路由器里，让主文件专注于配置和设置。

我们来看一个简单示例，使用 `express.Router()` 为 `/users` 路径定义路由。

首先，这是 `app.js` 文件：

```javascript
const express = require("express");
const app = express();
const port = 3000;

// Importing the user routes
const userRoutes = require("./userRoutes");

// Using the userRoutes router for /users path
app.use("/users", userRoutes);

app.listen(port, () => {
  console.log(`Server running at http://localhost:${port}`);
});
```

接着是 `userRoutes.js`：

```javascript
const express = require("express");
const router = express.Router();

// GET request for /users
router.get("/", (req, res) => {
  res.send("List of users");
});

// POST request for /users
router.post("/", (req, res) => {
  res.send("Creating a new user");
});

// GET request for /users/:id
router.get("/:id", (req, res) => {
  const userId = req.params.id;
  res.send(`User details for user with id: ${userId}`);
});

module.exports = router;
```

现在让我们仔细看看它们是如何工作的。

在 `app.js` 中，我们导入 `express`，创建一个 Express 应用实例并赋值给变量 `app`。

然后，我们从 `userRoutes` 导入 `userRoutes.js` 路由器，并用 `app.use("/users", userRoutes)` 挂载它。这告诉 Express，如果请求以 `/users` 开头，就交给 `userRoutes` 路由器处理。

接着在 `userRoutes.js` 中，我们用 `express.Router()` 创建一个新路由器，并用 `router.get()`、`router.post()` 和 `router.get("/:id")` 定义路由。

注意这里定义的路由是相对于 `/users` 的，因为路由器就是挂载在那里。所以：

*   `router.get("/")` → `GET /users`：响应用户列表
    
*   `router.post("/")` → `POST /users`：响应关于创建新用户的消息
    
*   `router.get("/:id")` → `GET /users/:id`：响应给定用户 ID 的用户详情
    

使用 `express.Router()` 让你的主文件保持简洁，专注于配置。所有用户相关路由都集中在 `userRoutes.js` 中。

随着应用增长，你可以为其他部分创建额外的路由器，比如 `/products`、`/orders` 或 `/auth`。

每个部分都保持在自己的文件中，使应用更易维护和扩展。
