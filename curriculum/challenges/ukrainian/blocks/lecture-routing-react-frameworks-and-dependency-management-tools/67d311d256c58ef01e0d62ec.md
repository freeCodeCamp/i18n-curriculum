# --description--

До цього моменту ви використовували React для створення інтерфейсів користувача. Якщо вам потрібні були додаткові функції, як-от маршрутизація, то доводилося імпортувати сторонню бібліотеку, наприклад React Router, щоб мати змогу перемикатися між різними виглядами.

Але що робити, коли потрібно створити повноцінний веб-застосунок з фронтендом і бекендом? Ви можете використати React для фронтенду і Node з Express для бекенд-логіки, якщо хочете залишатися в межах JavaScript. Або ж можна застосувати інші мови, як Go, Python чи Java для бекенду.

Хоча всі ці варіанти є життєздатними, іноді краще використати React-фреймворк. React-фреймворки надають функції, як маршрутизація, оптимізація зображень, отримання даних, автентифікація та інші. Це означає, що для певних випадків вам може не знадобитися налаштовувати окремі фронтенд і бекенд застосунки.

Давайте ближче розглянемо Next.js — популярний React-фреймворк. Одна з головних функцій Next.js — маршрутизатор, що базується на файловій системі. Ця система маршрутизації підтримує динамічні маршрути, паралельні маршрути, обробники маршрутів, перенаправлення, інтернаціоналізацію та інше.

Ось приклад створення кастомного обробника запитів:

```js
export async function GET() {
  const res = await fetch("https://example-api.com");
  const data = await res.json();

  return Response.json({ data });
}
```

Ви можете визначати обробники маршрутів, як GET або POST запити, у файлі `route.js` всередині директорії `app/api`.

Ще одна функція Next.js — автоматична оптимізація зображень і шрифтів.

Ось приклад роботи з компонентом `Image` у файлі `page.js`:

```jsx
import Image from "next/image";

export default function Page() {
  return (
    <Image src="link-to-image-goes-here" alt="descriptive-title-goes-here" />
  );
}
```

Компонент `Image` розширює нативний HTML-елемент `img` і дозволяє швидше завантажувати сторінки та оптимізувати розмір. Це означає, що зображення завантажуються лише тоді, коли вони потрапляють у вікно перегляду, а компонент `Image` автоматично подає зображення правильного розміру для кожного пристрою.

Хоча це лише кілька функцій Next.js, існує багато інших можливостей для створення надійних повноцінних веб-застосунків. Існують також інші React-фреймворки, як Remix, які надають такі ж можливості для створення сучасних повноцінних веб-застосунків.

Навіть якщо ландшафт бібліотек і фреймворків JavaScript постійно змінюється, важливо знати доступні варіанти та вивчати переваги й недоліки кожного фреймворку.
