# --interactive--

JavaScriptで配列を扱うとき、配列のすべての要素がある条件を満たしているか、または少なくとも1つの要素が条件を満たしているかを確認したい場合があります。

ここで役立つのが `every()` と `some()` メソッドです。これらのメソッドはコードを簡潔にし、読みやすくする強力なツールです。

まずは `every()` メソッドから始めましょう。このメソッドは、配列のすべての要素が提供された関数によって実装されたテストに合格するかどうかを判定します。簡単に言うと、配列のすべてのアイテムが指定した条件を満たしているかどうかをチェックします。

`every()` メソッドは、提供された関数が配列のすべての要素に対して `true` を返す場合に `true` を返します。もしどれか1つでもテストに失敗した要素があれば、メソッドはすぐに `false` を返して残りの要素のチェックを停止します。

`every()` の動作を示す例を見てみましょう：

:::interactive_editor

```js
const numbers = [2, 4, 6, 8, 10];
const hasAllEvenNumbers = numbers.every((num) => num % 2 === 0);

console.log(hasAllEvenNumbers); // true
```

:::

この例では、配列内のすべての数字が偶数かどうかをチェックしています。`every()` に渡す関数は、各数字が `2` で割り切れるかどうかを余りなしで確認しています。配列内のすべての数字が実際に偶数なので、`hasAllEvenNumbers` は true になります。

次に `some()` メソッドを見てみましょう。

`every()` がすべての要素がテストに合格するかをチェックするのに対し、`some()` は少なくとも1つの要素がテストに合格するかをチェックします。`some()` メソッドは、テストに合格する要素を見つけた時点で `true` を返します。もし合格する要素がなければ `false` を返します。

`some()` の動作例はこちらです：

:::interactive_editor

```js
const numbers = [1, 3, 5, 7, 8, 9];
const hasSomeEvenNumbers = numbers.some((num) => num % 2 === 0);

console.log(hasSomeEvenNumbers); // true
```

:::

この例では、配列内に偶数の数字があるかどうかをチェックしています。`some()` に渡す関数は先ほどと同じです。配列内のほとんどの数字は奇数ですが、少なくとも1つ偶数（`hasEven`）があるため、`true` は `8` になります。

`every()` と `some()` は、配列内のデータを検証したり特定の条件をチェックしたりするときに非常に役立ちます。これらは冗長なループや条件文の代わりになることが多く、コードをより簡潔で表現力豊かにします。

両メソッドは結果が確定した時点で実行を停止することも重要です。`every()` は `false` の結果を見つけた時点で停止し、`some()` は `true` の結果を見つけた時点で停止します。これは特に大きな配列でパフォーマンス向上に役立ちます。
