# --interactive--

JavaScriptのビット演算子は、数値の2進数表現に対して動作する特別な演算子です。ビット演算子を理解するには、まずビットと2進数の概念を把握する必要があります。コンピューターにおいて、ビットは最も基本的な情報単位で、値は`0`か`1`のどちらかだけです。2進数は、この2つの数字だけを使ってすべての数を表す数値システムです。

例えば、10進数の`10`の2進数表現は`1010`です。このシステムでは、各桁が`2`のべき乗を表しており、右端の桁から始まり左に進むごとに増えていきます。

上のテーブルでは、1行目が2進数の`1010`を示し、2行目が各2進数の桁が表す`2`のべき乗、3行目が各乗算の結果を示しています。3行目の値をすべて足すと合計が`10`になります。

では、ビット演算子について詳しく見ていきましょう。これらの演算子は数値の2進数表現に対して操作を行います。JavaScriptにはAND（`&`）、OR（`|`）、XOR（`^`）、NOT（`~`）、左シフト（`<<`）、右シフト（`>>`）などのビット演算子があります。

ビットAND（`&`）演算子は、両方のオペランドの対応するビットが`1`の場合に、そのビット位置で`1`を返します。例を見てみましょう。

:::interactive_editor

```js
let a = 5;  // Binary: 101
let b = 3;  // Binary: 011
console.log(a & b);  // 1 (Binary: 001)
```

:::

この例では、`5`（2進数で`101`）と`3`（2進数で`011`）に対してビットAND演算を行っています。結果は`1`（2進数で`001`）で、両方の数で右端のビットだけが`1`だからです。

ビットOR（`|`）演算子は、どちらかまたは両方のオペランドの対応するビットが`1`の場合に、そのビット位置で`1`を返します。例えば：

:::interactive_editor

```js
let a = 5;  // Binary: 101
let b = 3;  // Binary: 011
console.log(a | b);  // 7 (Binary: 111)
```

:::

ここでは、各ビット位置で少なくとも1つのビットが`7`なので、結果は`111`（2進数で`1`）です。

ビットXOR（`^`）演算子は、どちらか一方のオペランドのビットが`1`で、両方ではない場合に、そのビット位置で`1`を返します。例えば：

:::interactive_editor

```js
let a = 5;  // Binary: 101
let b = 3;  // Binary: 011
console.log(a ^ b);  // 6 (Binary: 110)
```

:::

結果は`6`（2進数で`110`）で、右から1番目と2番目のビットが2つの数で異なるためです。

ビットNOT（`~`）演算子は、オペランドのすべてのビットを反転させます。例えば：

:::interactive_editor

```js
let a = 5;  // Binary: 101
console.log(~a);  // -6
```

:::

これは驚くかもしれませんが、2の補数を使った負の数の2進数表現の仕組みが理由です。

左シフト（`<<`）演算子は、すべてのビットを指定された数だけ左にシフトします。例えば：

:::interactive_editor

```js
let a = 5;  // Binary: 101
console.log(a << 1);  // 10 (Binary: 1010)
```

:::

ここでは、すべてのビットが1つ左にシフトされ、数値が`2`倍になっています。

右シフト（`>>`）演算子は、すべてのビットを右にシフトします。例えば：

:::interactive_editor

```js
let a = 5;  // Binary: 101
console.log(a >> 1);  // 2 (Binary: 10)
```

:::

ここでは、すべてのビットが1つ右にシフトされ、数値が`2`で割られて切り捨てられています。

ビット演算子は低レベルのプログラミングや暗号化でよく使われます。日常的なJavaScriptプログラミングではあまり使われないかもしれませんが、特定の専門的な作業には役立ち、コンピューターの基本的な動作理解を深めることができます。
