# --description--

عند كتابة شفرة Python، من الشائع مواجهة أخطاء. فهم هذه الأخطاء أساسي لتصحيح الأخطاء في شفرتك بسرعة وكفاءة. تخبرك هذه الرسائل بالضبط بما حدث خطأً إذا عرفت كيف تقرأها.

تشمل أخطاء Python الشائعة `SyntaxError`، و`NameError`، و`TypeError`، و`IndexError`، و`AttributeError`. تحدث هذه عندما لا يفهم Python شفرتك، أو عندما لا تتطابق منطقك مع البيانات التي تعمل معها.

إليك مثال على `SyntaxError`:

```py
print("Hello, world!"
# SyntaxError: unexpected EOF while parsing
```

هذا السطر يفتقد قوسًا إغلاقًا. يرفع Python `SyntaxError` لأن الشفرة لا تتبع قواعد الصيغة النحوية الصحيحة.

إليك مثال على `NameError`:

```py
print(name)
# NameError: name 'name' is not defined
```

تحاول طباعة متغير لم يُعرّف بعد. يرفع Python `NameError` عندما لا يجد متغيرًا بهذا الاسم.

إليك مثال على `TypeError`:

```py
5 + "5"
# TypeError: unsupported operand type(s) for +: 'int' and 'str'
```

لا يمكنك جمع عدد صحيح مع سلسلة نصية معًا. يرفع Python `TypeError` عندما تحاول تنفيذ عملية على أنواع بيانات غير متوافقة.

إليك مثال على `IndexError`:

```py
my_list = [1, 2, 3]
print(my_list[5])
# IndexError: list index out of range
```

تحاول الوصول إلى فهرس غير موجود في القائمة. يرفع Python `IndexError` عندما تتجاوز الحدود.

إليك مثال على `AttributeError`:

```py
num = 42
num.append(5)
# AttributeError: 'int' object has no attribute 'append'
```

كائن `int` لا يحتوي على طريقة `append()`. يرفع Python `AttributeError` عندما تحاول استخدام طريقة أو خاصية غير موجودة لهذا النوع من البيانات.

التعرف على رسائل الخطأ الشائعة في Python يساعدك على إصلاح المشاكل بسرعة أكبر. بدلًا من التخمين، اقرأ رسالة الخطأ بعناية. غالبًا ما تخبرك بالضبط بما حدث خطأً وأين تبحث.
