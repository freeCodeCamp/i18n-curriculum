# --description--

التعدد الشكلي هو المفهوم الرئيسي التالي في البرمجة الكائنية التوجه (OOP) الذي سنتحدث عنه.

مع التعدد الشكلي، لديك وصول إلى واجهة يمكنك من خلالها التفاعل مع العديد من الكائنات من نفس النوع.

لنلق نظرة أعمق على التعدد الشكلي وكيف يسمح لك بإعادة استخدام الشفرة.

يسمح التعدد الشكلي للطرق في أصناف مختلفة بمشاركة نفس الاسم ولكن أداء مهام مختلفة. تستدعي نفس اسم الطريقة على كائنات مختلفة، وكل منها يستجيب بطريقته الخاصة.

إليك المثال الأساسي للتعدد الشكلي:

```py
class A:
   def action(self): ...

class B:
   def action(self): ...

class C:
   def action(self): ...

Class().method()  # Works for A, B, or C
```

إليك مثالًا يستخدم أصوات حيوانات مختلفة لتوضيح التعدد الشكلي:

```py
class Cat:
   def speak(self):
       return "A cat meow"

class Bird:
   def speak(self):
       return "A bird tweet"
  
class Monkey:
   def speak(self):
       return "A monkey ooh ooh aah aah ooh ooh aah aah"

def animal_sound(animal):
   print(animal.speak())

animal_sound(Cat())
animal_sound(Bird())
animal_sound(Monkey())
```

في هذا المثال، `animal_sound()` هي دالة تأخذ أي كائن يحتوي على طريقة `speak()`.

عندما تمرر `Cat` أو `Bird` أو `Monkey`، تستدعي طريقة `speak()` الخاصة بالكائن وتطبع النتيجة. بما أن كل صنف يعرف `speak()` بطريقة مختلفة، تحصل على مخرجات مختلفة من نفس الدالة. هذا هو التعدد الشكلي في العمل.

إليك مثالًا آخر، هذه المرة مع حالات وخاصية:

```py
class Twitter:
   def __init__(self, content):
       self.content = content

   def post(self):
       return f"🐦 Tweet: '{self.content}' (280 chars max)"

class Instagram:
   def __init__(self, content):
       self.content = content

   def post(self):
       return f"📸 Instagram Post: '{self.content}' + ✨ filters"

class LinkedIn:
   def __init__(self, content):
       self.content = content

   def post(self):
       return f"💼 LinkedIn Article: '{self.content}' (Professional Mode)"

def start(social_media):
   print(social_media.post())  # Calls .post() on any object

# Instances
tweet = Twitter('Just learned Python polymorphism!')
photo = Instagram('Sunset vibes 🌅')
article = LinkedIn('Why OOP matters in 2024')

# The polymorphic calls - same function, different outputs
start(tweet) # 🐦 Tweet: 'Just learned Python polymorphism!' (280 chars max)
start(photo) # 📸 Instagram Post: 'Sunset vibes 🌅' + ✨ filters
start(article) # 💼 LinkedIn Article: 'Why OOP matters in 2024' (Professional Mode)
```

هناك أيضًا نوع من التعدد الشكلي يسمى **التعدد الشكلي القائم على الوراثة**.

في التعدد الشكلي القائم على الوراثة، يعلن الصنف الأب طريقة، وتتجاوز عدة أصناف فرعية تلك الطريقة بطريقتها الخاصة. يمكنك بعد ذلك استدعاء نفس الطريقة على أي كائن فرعي، وتتصرف بشكل مختلف حسب الصنف الفرعي الذي ينتمي إليه.

إليك مثالًا:

```py
class Animal:
   def speak(self):
       return 'Some generic sound'

class Cat(Animal):
   def speak(self):
       return 'A cat meow'

class Dog(Animal):
   def speak(self):
       return 'A dog barks woof woof'

class Monkey(Animal):
   def speak(self):
       return 'A monkey ooh ooh aah aah ooh ooh aah aah'
  
print(Cat().speak()) # A cat meow
print(Dog().speak()) # A dog barks woof woof
print(Monkey().speak()) # A monkey ooh ooh aah aah ooh ooh aah aah
print(Animal().speak()) # Some generic sound
```

يمكنك أن ترى أن كل صنف فرعي من الصنف الأب `Animal` يتجاوز طريقة `speak()` ليقدم تنفيذًا خاصًا به. لذلك عندما تستدعي طريقة `speak()` على نسخة من كل صنف فرعي، تعيد الصوت المحدد المرتبط بذلك الحيوان.

يمكنك أيضًا التقدم أكثر وتنفيذ الاستدعاء في قائمة، ثم التكرار عبر القائمة لعرض ما تعيده طريقة `speak()` لكل منها:

```py
animals = [Cat(), Dog(), Monkey()]

for animal in animals:
   print(animal.speak())

# Output:
# A cat meow
# A dog barks woof woof
# A monkey ooh ooh aah aah ooh ooh aah aah
```
