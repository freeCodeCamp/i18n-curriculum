# --description--

Останнє завдання було на крок ближче до принципів функціонального програмування, але все ще чогось бракує.

Ми не змінювали значення глобальної змінної, але функція `incrementer` не працювала б без наявності глобальної змінної `fixedValue`.

Ще один принцип функціонального програмування — завжди явно оголошувати свої залежності. Це означає, що якщо функція залежить від наявності змінної або об’єкта, то передавайте цю змінну або об’єкт безпосередньо у функцію як аргумент.

Цей принцип має кілька хороших наслідків. Функцію легше тестувати, ви точно знаєте, які вхідні дані вона приймає, і вона не залежить від нічого іншого у вашій програмі.

Це дає більше впевненості, коли ви змінюєте, видаляєте або додаєте новий код. Ви будете знати, що можна змінювати, а що ні, і зможете побачити потенційні пастки.

Нарешті, функція завжди повертатиме однаковий вихід для однакової множини вхідних даних, незалежно від того, яка частина коду її виконує.

# --instructions--

Давайте оновимо функцію `incrementer`, щоб вона чітко оголошувала свої залежності.

Напишіть функцію `incrementer`, щоб вона приймала аргумент, а потім повертала результат, збільшений на одиницю.

# --hints--

Ваша функція `incrementer` не повинна змінювати значення `fixedValue` (яка є `4`).

```js
assert(fixedValue === 4);
```

Ваша функція `incrementer` повинна приймати аргумент.

```js
assert(incrementer.length === 1);
```

Ваша функція `incrementer` повинна повертати значення, яке на одиницю більше за значення `fixedValue`.

```js
const __newValue = incrementer(fixedValue);
assert(__newValue === 5);
```
