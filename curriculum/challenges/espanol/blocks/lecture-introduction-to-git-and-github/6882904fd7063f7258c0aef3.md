# --description--

Antes de que puedas subir tu repositorio a GitHub, necesitarás estar autenticado. Tanto las claves SSH como las GPG usan un par público-privado. Esto significa que mantienes una clave privada en tu máquina local, que usas para autenticarte. Compartes una clave pública con las personas o servicios que necesitan validar tu autenticación. En otras palabras, la clave privada es necesaria para realizar una acción, y la clave pública se usa para verificar la acción.

Las claves GPG, o Gnu Privacy Guard, se usan típicamente para firmar archivos o commits. Alguien puede usar entonces tu clave pública GPG para verificar que la firma del archivo proviene de tu clave y que el contenido del archivo no ha sido modificado ni alterado. Las claves SSH, o Secure SHell, se usan típicamente para autenticar una conexión remota a un servidor, a través de la utilidad `ssh`. Sin embargo, como aprenderás en esta lección, también puedes usar una clave SSH para firmar commits.

Para generar una clave GPG, necesitarás ejecutar:

```sh
gpg --full-generate-key
```

Para una clave SSH, ejecutarás:

```sh
ssh-keygen -t ed25519 -C "your_email@example.com"
```

Asegúrate de proporcionar tu correo electrónico real.

Ambos comandos deberían solicitar una frase de paso para proteger la clave. Son opcionales, pero altamente recomendados.

Para las claves SSH, obtendrás dos archivos: uno para tu clave privada y otro para tu clave pública. Los nombres de los archivos de la clave pública terminarán en `.pub`, para que no compartas accidentalmente tu clave privada. Estos archivos estarán en `~/.ssh`.

Las claves GPG, sin embargo, estarán en un directorio oculto y cifrado. En cambio, para listar tus claves públicas, ejecutarás:

```sh
gpg --list-secret-keys --keyid-format=long
```

Luego, para obtener la clave pública, usa:

```sh
gpg --armor --export "<key id>"
```

¿Pero qué puedes hacer con estas nuevas claves? Bueno, hemos hablado de firmar tus commits, y eso es algo para lo que puedes usar cualquiera de estas claves.

Para firmar tus commits con tu clave GPG, necesitarás subir tu clave pública, no la clave privada, a tu cuenta de GitHub. Luego, toma el ID corto que obtuviste al listar las claves anteriormente y ejecuta este comando para configurarlo como tu clave de firma de git:

```sh
git config --global user.signingkey <your_gpg_key_id>
```

Luego, puedes pasar la bandera `-S` a tu comando `git commit` para firmar un commit específico; necesitarás proporcionar tu frase de contraseña. Alternativamente, si quieres firmar cada commit automáticamente, puedes configurar autosign en `true`:

```sh
git config --global commit.gpgsign true
```

Para firmar con una clave SSH, que es una función relativamente nueva en GitHub, primero necesitarás subir la clave a tu cuenta de GitHub. Luego deberás configurar el modo de firma para que git use SSH:

```sh
git config --global gpg.format ssh
```

Luego, para establecer la clave de firma, pasarás la ruta del archivo en lugar de un ID:

```sh
git config --global user.signingkey <path_to_your_ssh_keys>
```

Finalmente, puedes habilitar la firma automática como lo harías para una clave GPG.

Con cualquiera de estas estructuras configuradas, ahora estás listo para tener commits verificados que obtendrán una insignia especial en GitHub.
