# --description--

在本课中，你将学习如何为对象和数组添加类型。

让我们先来看一下数组。

这是一个编程语言数组的示例： 

```js
const programmingLanguages = ["Java", "C++", "Python"];
```

如果你想将那个数组显式地指定为字符串数组，那么你可以将它指定为 `string[]`：

```ts
const programmingLanguages:string[] = ["Java", "C++", "Python"];
```

正如你之前所学，如果你只是这样写，TypeScript 会足够智能地推导出类型为字符串数组：

```js
const programmingLanguages = ["Java", "C++", "Python"];
```

但显式类型指派将在你尝试像这样为 `array` 添加另一种类型的情况下提供帮助：

```ts
// Type 'number' is not assignable to type 'string'.(2322)
const programmingLanguages:string[] = ["Java", 9, "Python"];
```

另一种输入数组的方式是使用这种语法：

```ts
const programmingLanguages:Array<string> = ["Java", "C++", "Python"];
```

从根本上讲，这两种语法是相同的，选择哪一种通常是个人偏好问题。

现在，让我们来看一下 `typing` `objects`。

这是一个对象字面的示例：

```js
const developer = {
  firstName: "Jessica",
  isEmployed: true
}
```

要显式地为对象属性指定类型，你可以使用这种语法：

```ts
const developer: { firstName: string, isEmployed: boolean } = {
  firstName: "Jessica",
  isEmployed: true
}
```

如果你尝试从该对象添加或删除属性，那么你将得到如下误差：

```ts
// Property 'isEmployed' is missing in type '{ firstName: string; }' but required in type '{ firstName: string; isEmployed: boolean; }'
const developer: { firstName: string, isEmployed: boolean } = {
  firstName: "Jessica",
}
```

你可以通过在属性名后使用 `?` 来使对象属性变为可选，如下所示：

```ts
const developer: { firstName: string, isEmployed?: boolean } = {
  firstName: "Jessica",
}
```

TypeScript 不会再显示误差，因为 `isEmployed` 属性已被标记为可选的。

到目前为止，所有的对象示例都有严格的属性。这意味着我们显式列出了对象允许拥有的每个属性：

```ts
const developer: { firstName: string, isEmployed: boolean } = {
  firstName: "Jessica",
  isEmployed: true
}
```

但是如果你事先不知道所有的属性名称怎么办？如果你想要一个可以拥有任意数量属性但所有值都必须遵循特定类型的对象怎么办？

好吧，有两种方法可以实现这一点。第一种方法是使用 `Record`，它是 TypeScript 中的一个内置通用实用类型。

这是基本语法：

```ts
Record<Keys, ValueType>
```

第一个 `type` 表现密钥的 `type`，而第二个 `type` 表现值的 `type`。

这是使用 `Record` 为 `userRoles` 对象的示例：

```ts
const userRoles: Record<string, string> = {
  admin: "full-access",
  editor: "limited-access",
  viewer: "read-only"
};
```

这个 `userRoles` 对象可以有任意的字串密钥，并且所有的值必须是字串。如果你尝试赋值非字串的值，你将得到一个误差：

```ts
// Type 'number' is not assignable to type 'string'
const userRoles: Record<string, string> = {
  admin: 1
};
```

第二种表达相同想法的方式是使用索引签名：

```ts
const obj: { [key: string]: string } = {};
```

上面的语法表示任何具有字符串密钥的属性必须有一个字符串值。下面是一个 `settings` 对象的示例：

```ts
const settings: { [key: string]: string } = {
  theme: "dark",
  language: "en",
  layout: "grid"
};
```

就像之前一样，如果你有错误的值 `type`，TypeScript 会显示错误信息：

```ts
// Type 'boolean' is not assignable to type 'string'
const settings: { [key: string]: string } = {
  darkMode: true
};
```

`Record` 和索引签名在本质上是相同的。所以在它们之间选择主要是个人偏好问题。
