# --description--

これで、Node.jsの世界に足を踏み入れる準備ができました。まずはコンピューターにNode.jsをインストールしましょう。ただし、Node.jsをインストールする前に、NVM（Node Version Manager）というツールをインストールすることをお勧めします。名前の通り、NVMは同じコンピューター上で複数のNode.jsのバージョンを管理するためのツールです。

最初からこのツールを設定しておくと、将来的に異なるプロジェクトで異なるNode.jsのバージョンを切り替える必要が出てきたときに役立ちます。NVMを使えば、それがとても簡単にできます。

macOS/LinuxとWindowsでのNVMのインストール方法と、Node.jsのインストール方法を見ていきましょう。

## macOS/LinuxでのNVMのインストール方法

macOSとLinuxでは、シェルスクリプトを使って直接NVMをインストールできます。

ただしmacOSの場合、NVMの公式ドキュメントには「インストールスクリプトを実行する前にXcodeコマンドラインツールを手動でインストールする必要があり、そうしないと失敗する」と記載されています。

macOSでは、ターミナルで次のコマンドを実行してXcodeコマンドラインツールをインストールできます。

```bash
xcode-select --install
```

インストールが完了したら、次のコマンドでバージョン番号を確認し、インストールが成功したかを検証してください。

```bash
xcode-select --version
```

次に、NVMをインストールします。次のいずれかのコマンドを実行して、NVMのインストールスクリプトをダウンロードして実行できます。

```bash
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.3/install.sh | bash
```

```bash
wget -qO- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.3/install.sh | bash
```

この記事を読んでいる時点での最新のNVMバージョンを知りたい場合は、公式NVMドキュメントを参照してください：`https://github.com/nvm-sh/nvm?tab=readme-ov-file#install--update-script`。そこに最新バージョン番号を含むコマンドが記載されています。

処理が完了したら、新しいコマンドと設定を読み込むためにターミナルを再起動する必要がある場合があります。

次に、このコマンドを実行してNVMが正しくインストールされたか確認し、現在のバージョンをチェックしてください。

```bash
nvm -v
```

出力にNVMのバージョンが表示されるはずです。

## macOS/LinuxでNVMを使ってNode.jsをインストールする方法

macOS/LinuxにNVMがインストールされたので、Node.jsをインストールできます。

最新のLong-Term Support（LTS）バージョンのNode.jsをインストールするには、次のコマンドを実行してください。

```bash
nvm install --lts
```

`--lts`フラグに注目してください。このフラグは、現在のLong-Term Support（LTS）バージョンのNode.jsをインストールしたいことを示しています。このバージョンは通常、安定性、信頼性、セキュリティを優先し、新機能や実験的な機能よりも重視されます。十分にテストされており、本番環境での使用に適しています。また、バグ修正やセキュリティパッチに重点を置いた長期サポート期間が保証されています。

または、コマンドでバージョン番号を指定して特定のNode.jsバージョンをインストールすることもできます。

```bash
nvm install <version>
```

例えば、Node.js v20をインストールするには次のようにします。

```bash
nvm install 20
```

Node.jsのバージョンをインストールしたら、次のコマンドでそのバージョンに切り替えられます。

```bash
nvm use <version>
```

例えば、バージョン20に切り替えるには次のようにします。

```bash
nvm use 20
```

最後に、次のコマンドで現在のNode.jsのバージョンを確認できます。

```bash
node -v
```

## WindowsでのNVMのインストール方法

次に、WindowsでNVMをインストールして使う方法を見てみましょう。

公式のNVMは特定のケースでのみWindowsをサポートしています。もしWindows Subsystem for Linux（WSL）がセットアップされている場合は、上記のmacOS/Linuxのガイドに従ってNVMをインストールして使えます。

しかし、Windows自体にはいくつかの代替手段があります。そのうちの一つが`nvm-windows`です。これはWindows用のNode.jsバージョン管理ユーティリティで、公式NVMプロジェクトとは完全に別のプロジェクトです。ただし、これらのツールの目的は同じで、同じデバイス上で複数のNode.jsバージョンを扱いやすくすることです。

`nvm-windows`をインストールする前に、公式リポジトリではNVM for Windowsをインストールする前に既存のNodeのインストールをアンインストールすることを推奨しています。初めてNode.jsをインストールする場合は問題ありませんが、すでに他のNode.jsバージョンがインストールされている場合は先にアンインストールすることをお勧めします。

`nvm-windows`をインストールするには：

- 公式リポジトリのReleasesセクションにアクセスしてください：`https://github.com/coreybutler/nvm-windows/releases`。
- 最新のインストーラーをダウンロードします。
- 指示に従ってください。

インストールが完了したら、`nvm`コマンドがすぐに使えない場合はターミナルを再起動してください。

次のコマンドを実行して、インストールが成功したかどうかと現在の`nvm-windows`のバージョンを確認できます。

```powershell
nvm -v
```

## WindowsでNVMを使ってNode.jsをインストールする方法

Windowsに`nvm-windows`がインストールされたので、Node.jsをインストールする準備ができました。

最新のLong-Term Support（LTS）バージョンのNode.jsをインストールするには、次のコマンドを実行してください。

```powershell
nvm install lts
```

Long-Term Support（LTS）バージョンが通常推奨されますが、コマンドでバージョン番号を指定して別のバージョンをインストールすることもできます。

```powershell
nvm install <version>
```

例えば：

```powershell
nvm install 20
```

インストールが完了したら、次のコマンドでそのNode.jsバージョンに切り替えられます。

```powershell
nvm use <version>
```

例えば：

```powershell
nvm use 20
```

次のコマンドで現在のNode.jsバージョンを確認できます。

```bash
node -v
```

## 基本コマンド

NVMとNode.jsのインストール方法がわかったので、Node.jsとNVMを使う際に役立つ基本的なコマンドを見てみましょう。

また、これらのコマンドはmacOS/LinuxにインストールされたNVM（WSLを含む）ではすべて動作しますが、`nvm-windows`では動作が異なるか、まったく動作しない場合があります。可能な限り`nvm-windows`用の代替コマンドも紹介します。

### Powershellの実行ポリシーを調整する方法

WindowsでPowershellと`nvm-windows`を使っている場合、Powershellの実行ポリシーを調整する必要があるかもしれません。Powershellのデフォルトの実行ポリシーは`Restricted`で、これはスクリプトの実行を防ぐセキュリティ対策です。

まず、Powershellを管理者として起動します。Powershellを右クリックして「管理者として実行」をクリックしてください。

次に、現在のPowershellセッションの実行ポリシーを確認するには、次のコマンドを実行します。

```powershell
Get-ExecutionPolicy
```

最後に、ローカルスクリプトと`RemoteSigned`の実行を許可する`npm`にPowershellの実行ポリシーを設定するには、次のコマンドを使います。

```powershell
Set-ExecutionPolicy RemoteSigned
```

上記のコマンドを実行したら、管理者権限のPowershellウィンドウを閉じ、管理者権限なしのPowershellウィンドウを開いてください。

### すべてのバージョンを一覧表示する

インストールされているすべてのNode.jsバージョンを一覧表示し、現在使用中のバージョンを確認するには、次のコマンドを実行します。

```bash
nvm ls
```

### デフォルトのNode.jsバージョンを設定する（macOS/LinuxのNVMのみ）

このコマンドは、新しいターミナルセッションで使用されるNode.jsのデフォルトバージョンを設定します。

```bash
nvm alias default <version>
```

例えば：

```bash
nvm alias default 22.20.0
```

また、特定の名前をバージョンに割り当てるカスタムエイリアスを作成することもできます。

```bash
nvm alias <new_name> <version>
```

例えば：

```bash
nvm alias new-api 18.20.8
```

エイリアスを設定したら、この新しい名前を使ってそのバージョンに切り替えられます。

```bash
nvm use <new_name>
```

例えば：

```bash
nvm use new-api
```

### Node.jsのバージョンを削除する

Node.jsのバージョンを削除するには、次のコマンドを実行します。

```bash
nvm uninstall <version>
```

例えば：

```bash
nvm uninstall 20.19.5
```

macOS/LinuxのNVMでは、`uninstall`コマンドをメジャーバージョンの`22`や`20`と一緒に実行してバージョンを削除できます。例えば：

```bash
nvm uninstall 20
```

もし`20.14.0`や`20.12.2`のようにそのメジャーバージョンの複数のリリースがインストールされている場合は、最新バージョンが最初に削除されます。この場合、`nvm uninstall 20`を実行すると最初に`20.14.0`が削除されます。

### Node.jsのバージョンを確認する

現在のNode.jsのバージョンを確認するには、次を使います。

```bash
node -v
```

### npmのバージョンを確認する

`npm`はNode.jsのパッケージマネージャーです。ソフトウェアパッケージのインストール、公開、管理に使えます。Node.jsをインストールすると自動的にインストールされます。

現在の`npm`のバージョンを確認するには、次のコマンドを実行してください。

```bash
npm -v
```

### JavaScriptファイルを実行する

このコマンドは、現在アクティブなNode.jsランタイムで指定したJavaScriptファイルを実行します。

```bash
node <file.js>
```

例えば：

```bash
node app.js
```

### Node.jsプロジェクトを作成する

このコマンドは、プロジェクトの詳細や依存関係を追跡する`package.json`ファイルを作成して、新しいNode.jsプロジェクトを作成します。

```bash
npm init
```

`npm init`コマンドを実行すると、`package.json`ファイルの設定のための対話型CLIガイドが開始されます。プロジェクト名、説明、バージョンなどの質問がされます。デフォルトの`package.json`ファイルから始めたい場合は、`npm init -y`または`npm init --yes`コマンドを使えます。

### パッケージをインストールする

特定のパッケージをインストールするには、次を実行します。

```bash
npm install <package>
```

例えば：

```bash
npm install express
```

### 依存関係をインストールする

プロジェクトの`package.json`ファイルにリストされているすべての依存関係をインストールするには、次を実行します。

```bash
npm install
```
