# --interactive--

Reguläre Ausdrücke, oder Regex, sind eine Funktion, die von vielen verschiedenen Programmiersprachen unterstützt wird.

Ein regulärer Ausdruck ist eine spezielle Syntax, um ein „Muster“ zu erstellen, mit dem Sie dann einen String überprüfen, Text extrahieren und mehr machen können.

Schauen wir uns einen einfachen regulären Ausdruck an:

```js
const regex = /freeCodeCamp/;
```

Beachten Sie, dass Sie in JavaScript einen regulären Ausdruck definieren, indem Sie Ihr Muster zwischen zwei Schrägstrichen (`/`) setzen. Verwechseln Sie das nicht mit einem Kommentar, bei dem der Text nach beiden Schrägstrichen steht.

Dieser spezielle reguläre Ausdruck findet den Text `freeCodeCamp` mit großem `C` irgendwo in einem String. Aber wie können Sie das tatsächlich machen?

Das führt uns zu unserer ersten Methode – der `test()`-Methode. Die `test()`-Methode ist auf `RegExp`-Objekten vorhanden, das sind Objekte, die einen regulären Ausdruck darstellen (wie den, den wir gerade definiert haben).

Die `test()`-Methode akzeptiert einen String, der der zu überprüfende String auf Übereinstimmungen mit dem regulären Ausdruck ist. Versuchen wir zum Beispiel, den String `e` zu testen:

:::interactive_editor

```js
const regex = /freeCodeCamp/;
const test = regex.test("e");
console.log(test);
```

:::

Sie sehen, dass wir die `test()`-Methode auf unserem neuen Regex aufgerufen und den String `e` als Argument übergeben haben.

Die `test()`-Methode gab `false` zurück, weil der String `e` nicht mit dem Muster `freeCodeCamp` übereinstimmt. Obwohl das Muster `freeCodeCamp` den Buchstaben `e` enthält, ist das die entgegengesetzte Richtung davon, wie reguläre Ausdrücke funktionieren.

Schauen wir uns noch ein paar weitere Beispiele an. Nehmen Sie sich einen Moment Zeit, um diese zu betrachten:

:::interactive_editor

```js
const regex = /freeCodeCamp/;
console.log(regex.test("freeCodeCamp"));
console.log(regex.test("freeCodeCamp is great"));
console.log(regex.test("I love freeCodeCamp"));
console.log(regex.test("freecodecamp"));
console.log(regex.test("FREECODECAMP"));
console.log(regex.test("free"));
console.log(regex.test("code"));
console.log(regex.test("camp"));
```

:::

Beachten Sie, dass die ersten drei Strings `true` zurückgegeben haben. Diese Strings enthalten alle genau den Text `freeCodeCamp` irgendwo im String.

Zeilen 5 und 6 geben `false` zurück. Obwohl sie den Text `freecodecamp` enthalten, stimmt die Groß- und Kleinschreibung nicht überein. Reguläre Ausdrücke sind standardmäßig case-sensitiv.

Schließlich enthalten die letzten drei zwar einen Teil des Musters, aber die Strings enthalten nicht das gesamte Muster.

Die `test()`-Methode gibt einen `boolean` zurück, der angibt, ob der String überhaupt mit dem regulären Ausdruck übereinstimmt.

Aber was, wenn Sie mehr Informationen als das möchten? Nun, Strings haben eine `match()`-Methode. Diese Methode akzeptiert einen regulären Ausdruck, obwohl Sie auch einen String übergeben können, der dann in einen regulären Ausdruck umgewandelt wird.

`match()` gibt das Match-Array für den String zurück. Was ist ein Match-Array? Schauen wir es uns an:

```js
const regex = /freeCodeCamp/;
const match = "freeCodeCamp".match(regex);
console.log(match);
```

Wenn wir das ausführen, erhalten wir ein Array zurück! Aber es ist ein seltsam aussehendes Array. Es hat einige zusätzliche Eigenschaften:

```js
// [
//   'freeCodeCamp',
//   index: 0,
//   input: 'freeCodeCamp',
//   groups: undefined
// ]
```

Die `groups`-Eigenschaft würde alle erfassten Gruppen anzeigen. Was das bedeutet, lernen Sie in einer zukünftigen Lektion.

Die `index`-Eigenschaft sagt Ihnen, an welchem Zeichen im String die Übereinstimmung gefunden wurde. In unserem Fall wurde sie am Anfang des Strings gefunden.

Die `input`-Eigenschaft sagt Ihnen den String, auf dem die `match()`-Methode aufgerufen wurde.

Versuchen wir noch ein paar weitere und sehen, wie sich das Ergebnis ändert:

```js
const regex = /freeCodeCamp/;
console.log("freeCodeCamp".match(regex)); // ['freeCodeCamp', index: 0, input: 'freeCodeCamp', groups: undefined]
console.log("freeCodeCamp is great".match(regex)); // ['freeCodeCamp', index: 0, input: 'freeCodeCamp is great', groups: undefined]
console.log("I love freeCodeCamp".match(regex)); // ['freeCodeCamp', index: 7, input: 'I love freeCodeCamp', groups: undefined]
console.log("freecodecamp".match(regex)); // null
console.log("FREECODECAMP".match(regex)); // null
console.log("free".match(regex)); // null
```

Wir wissen bereits, dass die ersten drei Strings eine Übereinstimmung ergeben sollten, also schauen wir uns diese an:

```js
// ['freeCodeCamp', index: 0, input: 'freeCodeCamp', groups: undefined]
// ['freeCodeCamp', index: 0, input: 'freeCodeCamp is great', groups: undefined]
// ['freeCodeCamp', index: 7, input: 'I love freeCodeCamp', groups: undefined]
```

Ist das das, was Sie erwartet haben? Sie können sehen, wie sich die `input` und `index` je nach bereitgestelltem String und der Position der Übereinstimmung im String geändert haben.

Die anderen drei Zeilen, die nicht übereinstimmen, geben stattdessen `null` zurück.

Jetzt, wo wir Strings mit unserem regulären Ausdruck testen und abgleichen können, was ist, wenn wir den Inhalt eines Strings ersetzen wollen? Vielleicht hat jemand `freecodecamp` komplett in Kleinbuchstaben geschrieben, und wir wollen die Groß- und Kleinschreibung automatisch korrigieren.

Zuerst müssen wir unseren regulären Ausdruck aktualisieren, um die Kleinbuchstabenform von `freecodecamp` zu finden, und unseren Teststring erstellen:

```js
const regex = /freecodecamp/;
const str = "freecodecamp is rly kewl";
```

Strings haben jetzt eine `replace()`-Methode, die zwei Argumente akzeptiert: den regulären Ausdruck, der gefunden werden soll (oder einen String, wenn Sie nicht alle Funktionen von Regex benötigen), und den String, durch den die Übereinstimmung ersetzt werden soll (oder eine Funktion, die für jede Übereinstimmung ausgeführt wird).

Wenn wir also unseren `freecodecamp` durch die richtige Groß- und Kleinschreibung ersetzen wollen:

:::interactive_editor

```js
const regex = /freecodecamp/;
const str = "freecodecamp is rly kewl";
const replaced = str.replace(regex, "freeCodeCamp");
console.log(replaced);
```

:::

Sie sehen, dass `replace()` den aktualisierten String mit dem passenden Muster `replaced` zurückgibt.

Reguläre Ausdrücke und alle damit verbundenen Methoden können komplex und überwältigend wirken. Aber Sie werden die Gelegenheit haben, sie in den nächsten Lektionen weiter zu erkunden.
