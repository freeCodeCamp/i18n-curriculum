# --description--

Lazy registration 是一種 UI 設計樣式，允許使用者在不必註冊的情況下瀏覽並互動你的應用程式。一個很好的例子是電子商務網站。使用者應該能夠瀏覽產品並將幾個項目添加到購物車中。然後，如果他們有興趣購買，就需要註冊。

原因是使用者需要先看到你的網站所提供的價值，才願意提供他們的資訊並進行註冊。當你設計你的應用程式時，使用者應該能夠看到價值並感覺應用程式是安全的，才會願意提供他們的資訊。否則，他們將不願意註冊，而你將失去潛在客戶。

你需要確保溝通使用者的敏感資料將會被保護且安全。在後續的模組中，我們將討論如何保護你的應用程式並保障使用者的資料安全。

另一個懶人註冊的好例子是 YouTube。YouTube 是一個影片分享平台，擁有數百萬支涵蓋科技、流行文化和遊戲等各種主題的影片。如果你造訪 YouTube，使用者可以在不需要登入或註冊的情況下觀看任意多的影片。然而，如果他們想要按讚、留言或訂閱頻道，就需要註冊。

如果使用者喜歡他們正在觀看的內容或想參與對話，那麼他們會更願意進行註冊。延遲註冊是一種有用的設計樣式，允許使用者在願意提供他們的資訊之前先看到你的應用程式的價值。

下次你設計應用程式時，請考慮使用 lazy registration 來提升使用者參與度和留存率。
