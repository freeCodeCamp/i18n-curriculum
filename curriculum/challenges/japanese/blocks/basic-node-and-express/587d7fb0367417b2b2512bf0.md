# --description--

HTMLサーバーは通常、ユーザーがアクセスできる1つ以上のディレクトリを持っています。そこにアプリケーションで必要な静的アセット（スタイルシート、スクリプト、画像）を配置できます。

Expressでは、この機能をミドルウェア`express.static(path)`を使って実装できます。`path`パラメータはアセットを含むフォルダーの絶対パスです。

ミドルウェアが何かわからなくても心配いりません。後で詳しく説明します。基本的にミドルウェアはルートハンドラを横取りして、何らかの情報を追加する関数です。ミドルウェアは`app.use(path, middlewareFunction)`メソッドを使ってマウントする必要があります。最初の`path`引数は省略可能で、渡さなければすべてのリクエストに対してミドルウェアが実行されます。

# --instructions--

`express.static()`ミドルウェアを`/public`パスに`app.use()`でマウントしてください。アセットフォルダーの絶対パスは`__dirname + /public`です。

これでアプリはCSSスタイルシートを配信できるはずです。`/public/style.css`ファイルはプロジェクトのボイラープレート内の`/views/index.html`で参照されていることに注意してください。フロントページが少し良く見えるはずです！

# --hints--

アプリは`/public`ディレクトリから`/public`パスへアセットファイルを配信する必要があります

```js
  const response = await fetch(code + '/public/style.css');
  if (!response.ok) {
    throw new Error(await response.text());
  }
  const data = await response.text();
  assert.match(
    data,
    /body\s*\{[^\}]*\}/,
    'Your app does not serve static assets'
  );
```

アプリは`/public`ディレクトリ以外のフォルダーからファイルを配信してはいけません

```js
  const response = await fetch(code + '/server.js');
  if (response.ok) {
    const data = await response.text();
    assert.equal(
      response.status + '',
      404 + '',
      'Your app must serve files only from "public" directory'
    );
  } else {
    assert.equal(
      response.status + '',
      404 + '',
      'Your app must serve files only from "public" directory'
    );
  }
```
