# --description--

Middleware é um conceito essencial no Express que nos permite manipular requisições e respostas entre o cliente e o servidor. Funções middleware têm acesso ao objeto de requisição, ao objeto de resposta e à função next no ciclo de requisição-resposta da aplicação.

Vamos detalhar:

## O que é middleware?

Middleware é basicamente uma função que é executada durante o ciclo de vida de uma requisição ao servidor. Ela realiza várias tarefas, como registrar requisições, tratar erros, analisar requisições recebidas ou servir arquivos estáticos.

O Express oferece uma forma prática de definir middleware que processa requisições antes que elas cheguem aos manipuladores de rota.

Como o middleware funciona:

Quando uma requisição é feita a uma aplicação Express, o middleware é executado na ordem em que foi adicionado. O fluxo é assim:

* A requisição é enviada ao servidor.
* Funções middleware processam a requisição.
* Se o middleware chama `next()`, ele passa o controle para o próximo middleware ou manipulador de rota.
* Se o middleware não chama `next()`, o ciclo de requisição-resposta para.

```js
const express = require('express')
const app = express()

// Simple middleware that logs request info
app.use((req, res, next) => {
  console.log('Request URL:', req.url)
  console.log('Request Method:', req.method)
  next() // Passes control to the next middleware
})

app.get('/', (req, res) => {
  res.send('Hello World!')
})

app.listen(3000, () => {
  console.log('Server running on http://localhost:3000')
})
```

`app.use()`: Adiciona middleware globalmente à sua aplicação.

Neste exemplo, o middleware registra detalhes da requisição e chama `next()` para continuar para o próximo middleware ou manipulador de rota.

Middleware é um recurso poderoso no Express que permite interceptar e modificar requisições em vários pontos. Você pode usar middleware para lidar com coisas como registro, autenticação ou até modificar os objetos de requisição e resposta antes que eles cheguem aos seus manipuladores de rota.
