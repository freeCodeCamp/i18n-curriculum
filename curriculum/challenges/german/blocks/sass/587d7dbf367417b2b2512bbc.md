# --description--

<dfn>Partials</dfn> in Sass sind separate Dateien, die Abschnitte von CSS-Code enthalten. Diese werden in andere Sass-Dateien importiert und verwendet. Das ist eine großartige Möglichkeit, ähnlichen Code in einem Modul zu gruppieren, um ihn organisiert zu halten.

Die Namen von Partials beginnen mit dem Unterstrich (`_`), der Sass signalisiert, dass es sich um einen kleinen CSS-Abschnitt handelt und dieser nicht in eine CSS-Datei umgewandelt werden soll. Außerdem enden Sass-Dateien mit der Dateiendung `.scss`. Um den Code aus dem Partial in eine andere Sass-Datei zu übernehmen, verwenden Sie die `@import`-Direktive.

Wenn zum Beispiel alle Ihre Mixins in einem Partial namens „\_mixins.scss“ gespeichert sind und diese in der Datei „main.scss“ benötigt werden, verwenden Sie sie so in der Hauptdatei:

```scss
@import 'mixins'
```

Beachten Sie, dass der Unterstrich und die Dateiendung in der `import`-Anweisung nicht benötigt werden – Sass erkennt, dass es sich um ein Partial handelt. Sobald ein Partial in eine Datei importiert wurde, stehen alle Variablen, Mixins und anderer Code zur Verwendung bereit.

# --instructions--

Schreiben Sie eine `@import`-Anweisung, um ein Partial namens `_variables.scss` in die Datei main.scss zu importieren.

# --hints--

Ihr Code sollte die `@import`-Direktive verwenden und den Unterstrich im Dateinamen nicht enthalten.

```js
assert(code.match(/@import\s+?('|")variables\1/gi));
```
