# --description--

Configurar una autenticación personalizada requiere escribir una cantidad significativa de lógica manual. Tienes que manejar la lectura de los encabezados de la solicitud, extraer cookies, verificar firmas y gestionar errores.

Aunque construir esto tú mismo es excelente para aprender, hacerlo para cada aplicación en producción consume mucho tiempo y es propenso a errores de seguridad. Por eso los desarrolladores usan Passport.js.

Passport.js es un middleware de autenticación para Node.js. Actúa como un framework organizado que estandariza cómo autenticas las solicitudes. Maneja los detalles de autenticación a bajo nivel, permitiéndote enfocarte en la lógica de negocio de tu aplicación.

La idea principal detrás de Passport.js es que cada aplicación requiere diferentes formas de iniciar sesión, pero el flujo subyacente de autenticación debe permanecer igual. Passport logra esta flexibilidad mediante plugins modulares llamados **Strategies**.

En lugar de escribir código personalizado para cada método de autenticación, instalas la estrategia específica que necesitas. Hay más de 500 estrategias disponibles, cubriendo casi todos los métodos de autenticación usados hoy en día. Algunas de ellas son:

- **passport-local:** Maneja la autenticación tradicional con nombre de usuario y contraseña contra tu propia base de datos.

- **passport-jwt:** Protege endpoints de API extrayendo y validando JSON Web Tokens de las solicitudes entrantes.

- **passport-google-oauth20:** Permite a los usuarios iniciar sesión instantáneamente usando sus cuentas de Google existentes.

- **passport-facebook:** Iniciar sesión con cuentas de Facebook vía OAuth 2.0.

- **passport-github2:** Permite a los usuarios iniciar sesión instantáneamente usando sus cuentas de GitHub.

Al dividir la autenticación en estrategias distintas, Passport permite que tu aplicación crezca sin problemas. Si comienzas hoy con un sistema de usuario/contraseña y decides agregar "Iniciar sesión con Google" el próximo mes, no necesitas reescribir todo tu código de autenticación. Solo tienes que instalar la estrategia de Google y conectarla a Passport.

Para usar Passport en una aplicación Express, debes instalar el paquete core de Passport junto con la estrategia específica que planeas usar. Para una configuración estándar de usuario y contraseña, ejecuta el siguiente comando en tu terminal:

```bash
npm install passport passport-local
```

Una vez instalado, necesitas integrar Passport en la configuración de tu aplicación Express. Esto debe hacerse antes de definir tus rutas:

```js
const express = require('express');
const passport = require('passport');
const LocalStrategy = require('passport-local').Strategy;

const app = express();

// initialize Passport middleware
app.use(passport.initialize());

// configure Passport to use the Local Strategy
passport.use(new LocalStrategy((username, password, done) => {
    // your database look-up and password verification logic goes here
}));
```

Passport se integra sin problemas en el sistema estándar de rutas de Express como middleware. Cuando un usuario envía sus credenciales a tu ruta de inicio de sesión, Passport intercepta la solicitud, pasa los datos a la estrategia configurada y evalúa el resultado.

```js
app.post('/login', passport.authenticate('local'), (req, res) => {
  // if this function runs, then authentication was successful
  res.send(`Welcome back, ${req.user.username}!`);
});
```

Si la estrategia verifica exitosamente al usuario, Passport maneja automáticamente dos tareas críticas:

1. **Gestión de sesión:** Activa funciones auxiliares para almacenar un pequeño fragmento de datos del usuario dentro de la cookie de sesión.

2. **Decoración de la solicitud:** Adjunta el objeto del usuario autenticado directamente a la solicitud entrante como `req.user`.

Una vez que `req.user` está poblado, cualquier middleware o manejador de ruta posterior en tu aplicación puede ver instantáneamente quién está conectado y tomar decisiones de autorización basadas en su identidad.

Passport proporciona una base consistente, segura y lista para producción para gestionar identidades de usuario en tus aplicaciones Node.js.
