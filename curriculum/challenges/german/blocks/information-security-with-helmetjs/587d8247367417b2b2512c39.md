# --description--

Zur Erinnerung: Dieses Projekt baut auf dem folgenden Starter-Projekt auf, das von <a href="https://github.com/freeCodeCamp/boilerplate-infosec/" target="_blank" rel="noopener noreferrer nofollow">GitHub</a> geklont wurde.

Cross-Site-Scripting (XSS) ist eine häufige Angriffsart, bei der bösartige Skripte in verwundbare Seiten eingeschleust werden, um sensible Daten wie Sitzungscookies oder Passwörter zu stehlen.

Die Grundregel zur Verringerung des Risikos eines XSS-Angriffs ist einfach: „Vertrauen Sie niemals der Eingabe des Benutzers“. Als Entwickler sollten Sie immer alle von außen kommenden Eingaben bereinigen. Dazu gehören Daten aus Formularen, GET-Abfrage-URLs und sogar aus POST-Bodies. Bereinigen bedeutet, dass Sie die potenziell gefährlichen Zeichen wie z. B. &lt;, > finden und kodieren sollten.

Moderne Browser können helfen, das Risiko durch bessere Softwarestrategien abzuschwächen. Diese sind oft über HTTP-Kopfzeilen konfigurierbar.

Der HTTP-Header X-XSS-Protection bietet einen grundlegenden Schutz. Der Browser erkennt potenziell eingeschleuste Skripte mithilfe eines heuristischen Filters. Wenn der Header aktiviert ist, verändert der Browser den Skriptcode und neutralisiert ihn. Die Unterstützung ist jedoch noch begrenzt.

# --instructions--

Verwenden Sie `helmet.xssFilter()`, um Eingaben zu bereinigen, die an Ihren Server gesendet werden.

# --hints--

Die Middleware helmet.xssFilter() sollte korrekt eingebunden sein

```js
const response = await fetch(code + '/_api/app-info');
if (!response.ok) {
  throw Error(await response.text());
}
const data = await response.json();
assert.include(data.appStack, 'xXssProtection');
assert.property(data.headers, 'x-xss-protection');
```
