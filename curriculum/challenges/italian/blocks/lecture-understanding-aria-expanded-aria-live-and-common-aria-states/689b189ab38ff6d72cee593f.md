# --interactive--

Hai già imparato come il markup semantico e gli attributi ARIA rendano accessibile il contenuto. Nelle app reali, il contenuto è raramente statico e le pagine spesso si aggiornano dinamicamente tramite JavaScript. Quando succede, è importante assicurarsi che questi cambiamenti siano riflessi anche nell’HTML. Questo permette ai lettori di schermo di trasmettere correttamente agli utenti lo stato aggiornato. Senza questo, le persone che usano strumenti assistivi potrebbero non sapere mai che qualcosa è cambiato o potrebbero ricevere informazioni obsolete o fuorvianti.

Vediamo uno scenario comune in cui JavaScript cambia lo stato di un elemento e come puoi mantenere accessibile il tuo HTML aggiornando gli attributi ARIA.

Supponiamo che tu abbia un pulsante che apre e chiude un menu a discesa:

:::interactive_editor

```html
<button id="menuButton" aria-expanded="false" aria-controls="menuList">
  Menu
</button>

<ul id="menuList" hidden>
  <li>List item 1</li>
  <li>List item 2</li>
</ul>
<script src="index.js"></script>
```

```js
const button = document.getElementById("menuButton");
const menu = document.getElementById("menuList");

button.addEventListener("click", () => {
  const expanded = button.getAttribute("aria-expanded") === "true";
  button.setAttribute("aria-expanded", String(!expanded));
  menu.hidden = expanded;
});
```

:::

L’HTML crea un pulsante che controlla un menu a discesa. Il pulsante usa `aria-expanded` per indicare se il menu è aperto e `aria-controls` per collegarlo al menu facendo riferimento all’`id` del menu.

Il JavaScript ascolta i clic sul pulsante del menu. Quando viene cliccato, alterna il valore di `aria-expanded` e aggiorna la proprietà `hidden` sul menu. Questo mantiene sincronizzata la visibilità del menu e le informazioni di accessibilità, così sia gli utenti vedenti sia quelli che usano lettori di schermo sanno se il menu è aperto o chiuso.

Imparerai di più su questi attributi ARIA nelle prossime lezioni. Per ora, ricorda solo che quando il tuo JavaScript cambia lo stato di un elemento, dovresti anche aggiornare gli attributi ARIA rilevanti affinché le tecnologie assistive riflettano accuratamente ciò che è effettivamente sullo schermo.
