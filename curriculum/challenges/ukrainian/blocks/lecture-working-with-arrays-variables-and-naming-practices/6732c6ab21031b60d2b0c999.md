# --description--

Іменування змінних і функцій — важливий аспект написання чистого, читабельного та підтримуваного коду. Хороші практики іменування роблять ваш код самодокументованим, зменшуючи потребу в численних коментарях і полегшуючи розуміння коду іншими розробниками, включно з вашим майбутнім «я».

Почнемо з загальних конвенцій іменування в JavaScript. У попередніх уроках ви вивчали використання camel case для імен змінних. Для булевих змінних загальною практикою є використання префіксів, таких як `is`, `has` або `can`. Це одразу дає читачеві зрозуміти, що змінна є булевою:

```js
let isLoading = true;
let hasPermission = false;
let canEdit = true;
```

Для функцій ім’я має чітко вказувати, що робить функція. Часто корисно починати з дієслова:

```js
function getUserData(){
  /* ... */
}

function calculateTotal(){
  /* ... */
}

function validateInput(){
  /* ... */
}
```

Для функцій, які повертають булеве значення і часто називаються предикатами, можна використовувати ті ж `is`, `has` або `can` префікси:

```js
function isValidEmail(email) {
  /* ... */
}

function hasRequiredFields(form) {
  /* ... */
}
```

Коли у вас є функції, які отримують дані, зазвичай починають з слова `get`:

```js
function getProductDetails(productId) {
  /* ... */
}

function getUserProfile(userId) {
  /* ... */
}
```

Коли у вас є функції, які встановлюють дані, зазвичай починають з слова `set`:

```js
function setUserPreferences(preferences) {
  /* ... */
}

function setPageTitle(title) {
  /* ... */
}
```

Для функцій-обробників подій можна додати префікс `handle` або суфікс `handler`:

```js
function handleClick(){
  /* ... */
}

function onSubmit(){
  /* ... */
}

function keyPressHandler(){
  /* ... */
}
```

Обробник подій — це дія, яка відбувається після того, як сталася подія, наприклад, натискання кнопки. Ви дізнаєтеся про це в майбутніх уроках.

При іменуванні змінних і циклів ітерації часто використовують поодинокі літери, такі як `i`, `j` або `k`, але для вкладених циклів або складніших ітерацій корисно використовувати більш описові імена:

```js
for (let i = 0; i < array.length; i++) {
  /* ... */
}

for (let studentIndex = 0; studentIndex < students.length; studentIndex++) {
  /* ... */
}
```

Для імен масивів розгляньте можливість використання множини іменників, щоб показати, що змінна містить кілька елементів:

```js
const colors = ['red', 'green', 'blue'];
const userNames = ['Alice', 'Bob', 'Charlie'];
```

Пам’ятайте, що мета — зробити ваш код максимально самодокументованим. Хороше правило: якщо вам потрібно додати коментар, щоб пояснити, що робить змінна або функція, можливо, варто перейменувати її на щось більш описове.

Нарешті, будьте послідовними у своєму коді або команді. Якщо у вашій команді вже встановлені конвенції іменування, дотримуйтеся їх. Послідовність робить код більш читабельним і підтримуваним для всіх учасників.
