# --description--

伺服器是現代網路的核心。它們驅動著我們每天作為使用者所依賴的線上體驗，並且將驅動你作為開發者所創建的大部分體驗。

讓我們更深入了解它們。

如你所知，網際網路是一個由相互連接的全域的電腦網路組成的系統。伺服器是啟用此通訊的核心組件，負責處理資料交換的行程。

**伺服器** 是一台電腦或電腦程式，接收來自其他電腦的請求，並透過網路回應傳送資料或服務給它們。

伺服端是一個雙重概念，因為它可以指涉：

* 實際的、強大的實體的 **hardware**，設計用來高效且持續地執行，以提供這些服務。這些機器是為了可靠性而建造和設計的。
    
* 或者是執行在該實體硬體上的實際電腦 **軟體**，以提供服務。
    
我們所指稱為「伺服器」的實體電腦通常存放在資料中心，這些資料中心是大型且安全的設施，具有受控的環境。

伺服器是**客戶端/伺服器架構**的重要零件。

客戶端/伺服器模型是一種分散式系統架構，其中組件要麼是客戶端，要麼是伺端。

* **客戶端** 是向伺服器發送請求以取得資料或服務的處理程序或應用程式。 例如，網頁瀏覽器。
    
* **伺服器** 是處理那些請求並傳回回應的實體的裝置或電腦程式。

基本上，客戶端和伺服端是請求-回應週期的部分，其中客戶端向伺服端發送請求，伺服端傳回回應給客戶端。

例如，當你使用瀏覽器（客端）以你的使用者名稱和密碼在應用程式上進行驗證時，你的資訊會安全地傳送到伺服器。伺服器接著會檢查你輸入的使用者名稱和密碼是否與平台目前資料庫中的資料相符。如果相符，伺服器會傳回成功的回應給客端（你的瀏覽器），你便成功地在該網站上完成驗證。

如果找不到符合條件，伺服器會傳回失敗回應給客戶端，並且你通常會看到錯誤或某種回授說明為什麼你的登入嘗試未成功。

所有這些驗證邏輯都在伺服端執行。伺服端負責驗證憑證、與資料庫互動，以及決定如何回應每個請求。

每個伺服端回應都包含一個 HTTP 狀態碼，告訴客戶端發生了什麼。這些狀態碼讓客戶端知道請求是成功還是失敗，以及原因。

例如：

* 200 表示一切正常。
    
* 404 表示在伺服器上找不到該資源。
    
* 500 表示發生了內部伺服器錯誤。
    
* 503 表示伺服器暫時無法使用。
    
* 等等……
    

每個狀態碼都有其獨特的意義。

使用伺服器的實際應用範例包括電子商務、搜尋引擎、遊戲、影片串流、雲端儲存與協作平台、社交媒體平台、金融應用程式等。

如果資料是動態更新的，或者應用程式需要與資料庫進行某種通訊，伺服器很可能是通訊處理程序中的關鍵組件。

這些是一些主要的伺服器型別：

* **網頁伺服器** 傳送網頁內容，例如 HTML 頁面、影像、圖片 和 CSS 檔案，給瀏覽器。
    
* **應用程式伺服器** 控制代碼商務邏輯、處理程序使用者輸入，並協調回應。
    
* **資料庫伺服器** 執行資料庫管理系統並儲存、擷取及管理資料。
    
* **郵件伺服器** 傳送、接收和儲存電子郵件訊息。
    
* **檔案伺服器** 管理對檔案的存取，並允許它們在網路中被儲存和分享。
    
* **代理伺服器** 作為客戶端與其他伺服器之間的中介，常用於快取、安全性或流量控制。

伺服器是我們每天使用的強大線上平台的骨幹。它們使得執行複雜邏輯、儲存和處理大量資料，以及同時支援數百萬使用者成為可能。若沒有伺服器，我們依賴的現代數位世界將不復存在。
