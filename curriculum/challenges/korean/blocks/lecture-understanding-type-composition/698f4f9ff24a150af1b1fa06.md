# --description--

이전 수업에서 원시 타입, 유니언 타입, 그리고 인터페이스를 다루는 방법을 배웠습니다. 

하지만 TypeScript에는 알아두어야 할 특별한 타입들이 있습니다. `any` 타입부터 살펴보겠습니다. 

`any` 타입은 모든 종류의 값을 나타내는 데 사용됩니다. 

```ts
let randomValue: any;

randomValue = 42;           

randomValue = "Hello";     
 
randomValue = true;        

randomValue = { name: "Alice" }; 
```

이 타입은 쉽게 과도하게 사용되거나 잘못 사용될 수 있으니 주의해야 합니다. TypeScript 오류 메시지를 없애기 위해 모든 것을 `any` 타입으로 지정하고 싶지는 않을 것입니다. 그렇게 하면 TypeScript를 사용하고 타입 안전성을 추가하는 본래 목적이 무색해집니다. 

`any` 타입보다 더 안전한 대응 타입은 `unknown` 타입입니다. `unknown`는 `any`와 비슷하지만, `unknown` 타입을 사용할 때는 변수를 사용하기 전에 타입 검사를 해야 합니다.

예를 들어 보겠습니다:

```ts
function doubleValue(value: unknown) {
  if (typeof value === "number") {
    console.log(value * 2);
  } else if (typeof value === "string") {
    console.log(value + value);
  }
}

doubleValue(10);    
doubleValue("Hi "); 
doubleValue(true);   
```

이 예제에서 타입 검사는 올바른 타입의 값에만 연산이 수행되도록 보장합니다. 예상 타입과 일치하지 않는 값은 무시됩니다.

다음으로 살펴볼 타입은 `never` 타입입니다. 이 타입은 절대 발생하지 않는 것을 나타냅니다.

대부분의 상황에서 `never`를 직접 타입 주석으로 작성하지는 않습니다. 대신 다음과 같은 오류 메시지에서 `never` 타입을 보게 될 가능성이 큽니다:

```ts
function processValue(value: string | number) {
  if (typeof value === "string") {
    console.log("String value:", value.toUpperCase());
  } else if (typeof value === "number") {
    console.log("Number value:", value * 2);
  } else {
    console.log(value); 
  }
}
```

위 예제의 `else` 절 내부에서 `value`는 `never` 타입을 가집니다. 이는 `else` 분기가 이제 도달할 수 없고 남은 타입이 없기 때문입니다. 만약 `string`나 `number` 이외의 값으로 함수를 호출하려 하면 다음과 같은 오류가 나타납니다:

```ts
function processValue(value: string | number) {
  if (typeof value === "string") {
    console.log("String value:", value.toUpperCase());
  } else if (typeof value === "number") {
    console.log("Number value:", value * 2);
  } else {
    console.log(value); 
  }
}

// Argument of type 'boolean' is not assignable to parameter of type 'string | number'.
processValue(true); 
```

마지막으로 살펴볼 타입은 `void` 타입입니다. 반환값이 없는 함수가 있다면, 그 함수는 `void` 타입을 가집니다. 

예를 들어 보겠습니다:

```ts
type Status = "loading" | "success" | "error";

type Handler = {
  status: Status;
  onChange: (newStatus: Status) => void; 
};
```

이 예제에서 `onChange` 타입의 `Handler` 속성은 아무것도 반환하지 않기 때문에 `void`이 사용되고 있습니다.
