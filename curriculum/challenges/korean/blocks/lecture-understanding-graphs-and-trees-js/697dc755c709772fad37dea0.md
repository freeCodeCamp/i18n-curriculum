# --description--

**우선순위 큐**는 큐나 스택과 비슷하게 작동하는 추상 자료형(ADT)이지만, 한 가지 중요한 차이가 있습니다.

이미 알고 있듯이, 일반 큐는 FIFO(선입선출) 방식을 따르며, 큐에 먼저 추가된 요소가 가장 먼저 제거됩니다.

스택은 LIFO(후입선출) 방식을 따르며, 스택에 마지막으로 추가된 요소가 가장 먼저 제거됩니다.

큐와 스택은 요소가 삽입된 순서만 고려합니다.

하지만 우선순위 큐는 요소의 "우선순위"를 고려합니다. 우선순위는 어떤 요소를 다음에 제거할지 결정하는 데 사용됩니다.

보통은 우선순위가 가장 높은 요소가 먼저 제거되지만, 일부 구현에서는 우선순위가 가장 낮은 요소를 먼저 제거하기도 합니다. 이는 프로그램의 요구사항에 따라 달라집니다.

우선순위 큐는 두 위치 사이의 최단 경로 찾기, 운영체제에서 작업 스케줄링, 교통 시뮬레이션, 데이터 압축, 네트워크 관리 등 실용적인 애플리케이션에 매우 유용합니다.

실제로 우선순위 큐는 보통 힙 자료 구조를 사용해 구현합니다.

**힙**은 매우 특정한 속성인 **힙 속성**을 가진 트리 자료 구조입니다. 이 속성은 힙의 종류에 따라 각 노드와 자식 노드 간의 관계를 결정합니다.

힙에는 두 가지 주요 유형이 있습니다:

* 최대 힙
    
* 최소 힙
    

**최대 힙**에서는 각 노드의 값이 자식 노드의 값보다 크거나 같습니다.

이 예시에서는 8, 7, 5, 2, 1 노드가 있는 트리 구조를 볼 수 있습니다. 노드 7이 노드 2와 노드 1보다 크다는 점에서 힙 속성이 지켜지고 있습니다. 다른 모든 노드도 마찬가지입니다.

<img src="https://cdn.freecodecamp.org/curriculum/lecture-transcripts/how-do-priority-queues-and-heaps-work-1.png" alt="A max-heap tree structure showing nodes with values 8 at the root, 7 and 5 as children of 8, and 2 and 1 as children of 7, demonstrating that each parent node is greater than its children.">

반면, **최소 힙**에서는 각 노드의 값이 자식 노드의 값보다 작거나 같습니다.

이 예시에서는 4, 7, 9, 12, 15 노드가 있습니다. 예를 들어, 노드 7은 노드 12와 노드 15보다 작아 힙 속성이 지켜지고 있습니다. 다른 모든 노드도 마찬가지입니다.

<img src="https://cdn.freecodecamp.org/curriculum/lecture-transcripts/how-do-priority-queues-and-heaps-work-2.png" alt="A min-heap tree structure showing nodes with values 4 at the root, 7 and 9 as children of 4, and 12 and 15 as children of 7, demonstrating that each parent node is less than its children.">

힙 속성은 최대값(또는 최소값)이 항상 최상단에 위치하도록 보장하기 때문에 매우 중요하며, 이로 인해 제거가 매우 쉽습니다.

실제로 힙은 부모 노드와 자식 노드에 효율적으로 접근하기 위해 보통 배열로 구현합니다.

배열을 사용하면 이러한 값이나 "노드"에 접근하는 로직이 단순해집니다. 힙이 완전 이진 트리 구조를 유지한다면, 배열 구현은 인덱스를 기반으로 한 간단한 수학 연산만으로 메모리 내 요소 위치를 찾을 수 있기 때문입니다.

JavaScript에는 내장 힙 모듈이 없지만, 배열을 사용해 최소 힙을 구현할 수 있습니다.

다음은 JavaScript로 구현한 기본 최소 힙입니다:

```js
class MinHeap {
  constructor(compare = (a, b) => a - b) {
    this.data = [];
    this.compare = compare;
  }

  peek() {
    return this.data[0];
  }

  push(value) {
    this.data.push(value);
    this.#bubbleUp(this.data.length - 1);
  }

  pop() {
    if (this.data.length === 0) return undefined;

    const top = this.data[0];
    const last = this.data.pop();

    if (this.data.length > 0) {
      this.data[0] = last;
      this.#bubbleDown(0);
    }

    return top;
  }

  pushPop(value) {
    if (this.data.length === 0) return value;

    if (this.compare(this.data[0], value) < 0) {
      const top = this.data[0];
      this.data[0] = value;
      this.#bubbleDown(0);
      return top;
    }

    return value;
  }

  heapify(arr) {
    this.data = arr.slice();
    for (let i = Math.floor(this.data.length / 2) - 1; i >= 0; i--) {
      this.#bubbleDown(i);
    }
  }

  #bubbleUp(i) {
    while (i > 0) {
      const p = Math.floor((i - 1) / 2);
      if (this.compare(this.data[i], this.data[p]) >= 0) break;
      [this.data[i], this.data[p]] = [this.data[p], this.data[i]];
      i = p;
    }
  }

  #bubbleDown(i) {
    const n = this.data.length;

    while (true) {
      let smallest = i;
      const l = 2 * i + 1;
      const r = 2 * i + 2;

      if (l < n && this.compare(this.data[l], this.data[smallest]) < 0) smallest = l;
      if (r < n && this.compare(this.data[r], this.data[smallest]) < 0) smallest = r;

      if (smallest === i) break;

      [this.data[i], this.data[smallest]] = [this.data[smallest], this.data[i]];
      i = smallest;
    }
  }
}
````

이 힙을 사용하려면 빈 힙을 생성하세요. 이것이 힙의 기본 자료 구조가 됩니다:

```js
const myHeap = new MinHeap();
```

힙에 요소를 추가하려면 `push()`를 호출하세요. 힙 속성을 유지하기 위해 요소가 자동으로 적절한 위치에 추가됩니다:

```js
myHeap.push(9);
```

가장 높은 우선순위(이 경우 가장 작은 값)를 가진 요소를 얻으려면 `pop()`를 호출하세요:

```js
myHeap.pop();
```

`pushPop()`는 두 작업을 하나로 합친 호출입니다.

힙이 클 때는 두 작업을 따로 호출하는 것보다 한 번에 처리하는 것이 더 효율적입니다. 왜냐하면 재정렬 작업이 한 번만 수행되기 때문입니다:

```js
myHeap.pushPop(15);
```

이미 배열이 있고 이를 힙으로 변환하려면 `heapify()`를 호출할 수 있습니다:

```js
myHeap.heapify([9, 2, 7, 1]);
```

지금까지는 요소를 값으로 정렬하고 있죠?

그렇다면 "우선순위"로 정렬하려면 어떻게 해야 할까요?

이렇게 하면 됩니다: `[priority, element]` 구조의 배열을 저장하세요.

JavaScript에서는 배열을 자동으로 요소별로 비교하지 않으므로, 우선순위를 먼저 비교하는 비교 함수를 힙에 전달해야 합니다.

이 경우, 낮은 값이 더 높은 우선순위를 나타냅니다. 즉, 우선순위가 1인 항목이 우선순위가 3인 항목보다 **더 높은** 우선순위를 갖습니다:

```js
const myHeap = new MinHeap((a, b) => a[0] - b[0]);

myHeap.push([3, "A"]);
myHeap.push([2, "B"]);
myHeap.push([1, "C"]);
```

동일한 우선순위를 가진 요소가 삽입된 순서대로 제거되도록 하려면, 항목의 두 번째 요소로 고유한 카운터를 포함해 우선순위가 같을 때 순서를 구분할 수 있습니다. 예를 들어 `[priority, counter, element]`처럼 할 수 있습니다.

예를 들어, 우선순위를 먼저 비교하고, 그다음 카운터를 비교할 수 있습니다:

```js
let counter = 0;
const myHeap = new MinHeap((a, b) => (a[0] - b[0]) || (a[1] - b[1]));

myHeap.push([3, counter++, "A"]);
myHeap.push([2, counter++, "B"]);
myHeap.push([2, counter++, "C"]);
```

이제 힙의 효율성에 대해 이야기해 보겠습니다.

힙에서 최솟값 또는 최댓값을 삽입하거나 추출하는 평균 및 최악의 경우 시간 복잡도는 로그 시간, `O(log n)`입니다. 이는 보통 교환 횟수가 힙의 높이(log(n))에 비례하기 때문입니다.

"peek" 연산의 평균 및 최악의 경우 시간 복잡도는 상수 시간, O(1)입니다. peek은 최솟값 또는 최댓값을 제거하지 않고 얻는 작업입니다.

힙을 정렬되지 않은 리스트에서 만드는 "heapify" 연산은 평균 및 최악의 경우 선형 시간 복잡도, O(n)을 가집니다.

마찬가지로 임의 요소를 검색하거나 삭제하는 작업도 평균 및 최악의 경우 선형 시간 복잡도 O(n)을 가지는데, 이는 전체 힙을 탐색해야 할 수도 있기 때문입니다.

공간 복잡도는 선형, O(n)이며, `n`는 포함된 요소 수입니다. 요소와 배열 객체 자체에 대한 약간의 추가 오버헤드만 필요합니다.

우선순위 큐와 힙은 컴퓨터 과학에서 매우 중요합니다. 리스트에서 가장 중요한 요소를 빠르게 찾고 사용할 수 있게 해 주기 때문입니다. 이러한 효율성은 지도에서 가장 빠른 경로를 찾는 등 실제 중요한 작업을 수행하는 많은 컴퓨터 프로그램에 필수적입니다.
