# --description--

في درس سابق، تعلمت ما هي التبعيات. هنا، ستتعلم المزيد عنها والأمور التي يجب مراعاتها عند اختيار حزمة خارجية لاستخدامها في مشروعك.

التبعية هي قطعة شفرة أخرى يحتاجها مشروعك لكي يعمل. عادةً ما تكون هذه حزمًا خارجية أنشأها مطوّرون آخرون وشاركوا بها في سجل npm.

استخدام التبعيات يوفر عليك الكثير من الوقت. بدلًا من بناء كل شيء بنفسك، يمكنك تثبيت حزمة تحل مشكلة شائعة بالفعل — مثل إنشاء خادم ويب، التعامل مع التوجيه، جلب البيانات، إدارة التوثيق، والمزيد.

في ملف `package.json` الخاص بك، تُدرج كل الحزم التي يعتمد عليها مشروعك تحت المفتاح `dependencies`:

```json
{
  "name": "package-dependencies",
  "version": "1.0.0",
  "description": "a demo on how package dependencies work",
  "main": "index.js",
  "scripts": {
    "test": "echo \"Error: no test specified\" && exit 1"
  },
  "keywords": [
    "Node",
    "NPM"
  ],
  "author": "Kolade Chris",
  "license": "ISC",
  "dependencies": {
    "chalk": "^5.6.2",
    "lodash": "^4.17.21"
  }
}
```

في مثال ملف `package.json` أعلاه، تم إدراج `chalk` و `lodash` كتبعيات للمشروع. هذا يعني أن المشروع يعتمد على هذه الحزم لكي يعمل. إذا لم تكن مثبتة — وغير مدرجة في ملف `package.json` — ستتوقف أي ميزات تستخدمها عن العمل.

هناك أيضًا فئة أخرى من التبعيات تسمى `devDependencies`. هذه حزم تحتاجها فقط أثناء بناء أو اختبار مشروعك، وليس أثناء تشغيله في بيئة الإنتاج. أمثلة شائعة تشمل:

* Nodemon لإعادة تشغيل المشروع تلقائيًا عند وجود تحديثات
    
* Jest أو Mocha للاختبار
    
* ESLint لفحص الشفرة
    

كما هو متوقع، تظهر `devDependencies` تحت قسم `devDependencies` في ملف `package.json`:

```json
{
  "name": "package-dependencies",
  "version": "1.0.0",
  "description": "a demo on how package dependencies work",
  "main": "index.js",
  "scripts": {
    "test": "echo \"Error: no test specified\" && exit 1"
  },
  "keywords": [
    "Node",
    "NPM"
  ],
  "author": "Kolade Chris",
  "license": "ISC",
  "dependencies": {
    "chalk": "^5.6.2",
    "lodash": "^4.17.21"
  },
  "devDependencies": {
    "jest": "^30.2.0",
    "nodemon": "^3.1.10"
  }
}
```

عند اختيار الحزم لاستخدامها في مشروعك، هناك بعض الأمور المهمة التي يجب وضعها في الاعتبار.

يجب أن تكون الأمان دائمًا في المقام الأول. حتى حزمة واحدة غير آمنة يمكن أن تسبب مشاكل لمشروعك بأكمله. يجب أيضًا مراعاة الأداء، لأن الحزمة الكبيرة أو غير المحسنة جيدًا قد تبطئ مشروعك.

من المفيد أيضًا اختيار الحزم الشائعة والمدعومة جيدًا. الحزم الشائعة عادةً ما يكون لها مجتمعات نشطة، توثيق جيد، وتحديثات متكررة. طريقة سهلة للتحقق من الشعبية هي النظر إلى عدد التنزيلات الأسبوعية على npm. على سبيل المثال، تحصل حزمة `chalk` على أكثر من 300,000 تنزيل أسبوعيًا:

<img src="https://cdn.freecodecamp.org/curriculum/lecture-transcripts/what-are-package-dependencies-and-what-are-some-considerations-for-choosing-an-external-package-1.webp" alt="screenshot from npm of chalk package statistics">

الصيانة مهمة أيضًا. الحزم التي تُصان جيدًا تُحدَّث بانتظام وتعمل بشكل أفضل مع الإصدارات الجديدة من البرامج الأخرى.

أخيرًا، لا تتجاهل التوثيق. الوثائق الواضحة والشاملة تجعل فهم كيفية استخدام الحزمة أسهل بكثير وتوفر عليك الكثير من الوقت عند دمجها في مشروعك.
