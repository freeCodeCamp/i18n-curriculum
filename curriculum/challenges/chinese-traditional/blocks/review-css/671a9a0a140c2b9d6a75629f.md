## CSS 基礎知識

- **什麼是 CSS？**：階層式樣式表（CSS）是一種用於套用樣式到 HTML 元素的標記語言。CSS 用於顏色、背景影像、版面配置等。
- **CSS 規則的基本結構**：CSS 規則由兩個主要部分組成：選擇器和宣告區塊。選擇器是在 CSS 中用來識別並標的特定 HTML 元素以進行樣式設定的樣式。宣告區塊則是為特定選擇器或多個選擇器套用一組樣式。
- **`meta name="viewport"` 元素**：此 `meta` 元素提供瀏覽器指示，說明如何在不同裝置上控制頁面的尺寸和縮放，特別是在手機和平板電腦上。
- **預設瀏覽器樣式**：每個 HTML 元素都會套用預設瀏覽器樣式。這通常包括像是預設邊距和內距的條款。

## 行內、內部 和 外部 CSS

- **行內 CSS**：這些樣式是直接寫在 HTML 元素內，使用 `style` 屬性。大多數時候你不會使用行內 CSS，因為要遵守關注點分離原則。
- **內部 CSS**：這些樣式寫在 HTML 文件的 `head` 區段內的 `<style>` 頁籤中。這對於建立簡短的程式碼範例很有用，但通常你不會使用內部 CSS。
- **外部 CSS**：這些樣式寫在獨立的 CSS 檔案中，並使用 `head` 區段的 `link` 元素連結到 HTML 文件。對大多數專案來說，你會使用外部 CSS 檔案，而非內部或行內 CSS。

## 操作 `width` 和 `height` 屬性

- **`width` 屬性**：此屬性指定元素的寬度。如果你沒有指定寬度，則預設值為 `auto`。這讓瀏覽器根據元素的內容、父元素和顯示型別來決定元素的寬度。
- **`min-width` 屬性**：此屬性指定元素的最小寬度。
- **`max-width` 屬性**：此屬性指定元素的最大寬度。
- **`height` 屬性**：此屬性指定元素的高度。同樣地，高度預設值為 `auto`，表示它會根據內部內容調整。
- **`min-height` 屬性**：此屬性指定元素的最小高度。
- **`max-height` 屬性**：此屬性指定元素的最大高度。

## 不同型別的 CSS 組合器

- **後代組合器 (` `)&nbsp;\*選擇器之間的單一空格**：此組合器用於選取指定父元素的後代元素。
- **子代組合器 (`>`)**：此組合器用於選取指定父元素的直接子元素。
- **下一個兄弟組合器 (`+`)**：此組合器會選取緊接在指定兄弟元素後方的元素。

- **後續兄弟組合器 (`~`)**：此組合器會選取指定元素之後的所有兄弟元素。

## 行內、區塊和行內區塊層級元素

- **行內層級元素**：行內元素只佔用它們所需的寬度，且不會另起新行。這些元素會在內容中流動，允許文字和其他行內元素與它們並排出現。常見的行內元素有 `span`、`anchor` 和 `img` 元素。
- **區塊層級元素**：區塊層級元素預設會佔據其容器可用的全部寬度，延伸至容器的整個寬度。一些常見的區塊層級元素有 `div`、`p` 和 `section` 元素。
- **行內-區塊層級元素**：你可以使用 `display` 屬性將元素設定為 `inline-block`。這些元素的行為類似行內元素，但可以像區塊層級元素一樣設定 `width` 和 `height`。

## 外距和內距

- **`margin` 屬性**：此屬性用於在元素的邊框與周圍元素之間套用外部間距。
- **`margin` 簡寫**：你可以指定 1–4 個值來設定邊界的各邊。一個值套用到所有四個邊；兩個值設定 `top` 和 `bottom`，然後是 `right` 和 `left`；三個值設定 `top`、水平（`right` 和 `left`），然後是 `bottom`；四個值設定 `top`、`right`、`bottom`、`left`。
- **`padding` 屬性**：此屬性用於在元素內部、內容與其邊框之間應用間距。
- **`padding` 簡寫**：你可以指定 1–4 個值來設定內距的各邊。一個值套用於所有四邊；兩個值設定 `top` 和 `bottom`，然後是 `right` 和 `left`；三個值設定 `top`、水平（`right` 和 `left`），然後是 `bottom`；四個值設定 `top`、`right`、`bottom`、`left`。

## CSS 特異性

- **行內 CSS 特異性**：行內 CSS 擁有最高的特異性，因為它是直接套用到元素上。它會改寫任何內部或外部的 CSS。行內樣式的特異性值為 (1, 0, 0, 0)。
- **內部 CSS 特異性**：內部 CSS 是在 HTML 文件的 `head` 區段內的 `style` 元素中定義。它的特異性低於行內樣式，但可以改寫外部樣式。
- **外部 CSS 特異性**：外部 CSS 是透過 `head` 區段中的 `link` 元素連結，並寫在獨立的 `.css` 檔案中。它具有最低的特異性，但為較大型專案提供最佳的維護性。
- **通用選擇器 (`*`)**：這是一種特殊型別的 CSS 選擇器，可匹配文件中的任何元素。它常用於為頁面上的所有元素套用樣式，這對於重設或標準化不同瀏覽器之間的樣式非常有用。通用選擇器具有所有選擇器中最低的特異性值。它對特異性值的所有零件均貢獻 0（0、0、0、0）。
- **型別選擇器**：這些選擇器根據元素的頁籤名稱來標的元素。型別選擇器的特異性相較於其他選擇器較低。型別選擇器的特異性值為 (0, 0, 0, 1)。
- **類別選擇器**：這些選擇器由一個句點（`.`）後接類別名稱所定義。類別選擇器的特異性值為 (0, 0, 1, 0)。這表示類別選擇器可以改寫型別選擇器，但會被 ID 選擇器和行內樣式所改寫。
- **ID 選擇器**：ID 選擇器由井字號符號（`#`）後接 ID 名稱所定義。ID 選擇器具有非常高的特異性，特異性高於型別選擇器和類別選擇器，但低於行內樣式。ID 選擇器的特異性值為 (0, 1, 0, 0)。
- **`!important` 關鍵字**：用來賦予樣式規則最高的優先權，使其能夠改寫該屬性的任何其他宣告。使用時，它會強制瀏覽器套用指定的樣式，無論其他選擇器的特異性為何。你應該謹慎使用 `!important`，因為它可能會讓你的 CSS 更難維護和偵錯。
- **Cascade Algorithm**：一種用來決定當多個樣式針對同一元素時應該套用哪些 CSS 規則的演算法。它確保根據一組明確定義的規則使用最適合的樣式。
- **CSS 繼承**：樣式從父元素傳遞到子元素的處理程序。繼承允許你在文件樹的較高層級定義樣式，並將它們套用到多個元素，而無需為每個元素明確指定。

## 設計術語

- **版面配置**：這是視覺元素在頁面或螢幕上排列的方式，用以傳達訊息。這些元素可能包括文字、影像和空白區域。
- **對齊**：這是元素彼此之間的擺放方式。正確使用對齊有助於讓設計看起來乾淨且有條理。
- **複合**：這是排列元素以創建和諧設計的行為。它決定像 `images`、`text` 和形狀等元素如何相互關聯並以藝術的方式為設計做出貢獻。
- **平衡**：這是視覺重量在複合中如何分佈。設計師旨在透過對稱或非對稱的排列來創造一種平衡。
- **Scale**：這是指將一個元素的尺寸或大小與另一個元素進行比較。
- **階層體系**：這建立設計中元素的重要性順序。這是為了確保最重要的資訊先被注意到。
- **對比**：這是建立元素之間明顯區別的處理程序。你可以透過顏色、大小、形狀、質地或任何其他視覺特徵的變化來達成。強烈的對比也有助於提升可讀性。
- **留白（負空間）**：這是設計中的空白區域。它是環繞元素的空間。
- **UI (User Interface)**：UI 包含使用者在螢幕上可以看到的視覺和互動元素，如圖示、影像、文字、選單、聯結和按鈕。
- **UX (使用者體驗)**：UX 是關於使用者在使用產品或服務時的感受。具有良好設計使用者體驗的應用程式直覺、易用、高效、可存取且令人愉悅。
- **設計簡報**：這是一份概述專案目標、宗旨及需求的文件。它是指引設計流程的路線圖，確保最終產品符合客戶的需求。
- **向量基礎設計**：這涉及使用數學公式來定義線條、形狀和顏色，以創建數位藝術。
- **原型設計**：這是指建立產品或使用者介面的互動模型的處理程序。

## 使用者介面設計基礎

- **背景和前景顏色的良好對比**：確保背景和前景顏色具有良好對比以使 `text` 可讀非常重要。網頁內容無障礙指引（WCAG）建議一般 `text` 的最低對比度為 4.5:1，大字 `text` 則為 3:1。
- **設計中的良好視覺階層體系**：強大的視覺階層體系可以為眼睛提供清晰的路徑，確保你傳達的資訊依照你預期的順序被接收。
- **響應式影像**：響應式影像是會依照所觀看的螢幕大小縮放的影像。這很重要，因為它確保你的影像在所有裝置上看起來都很漂亮，從桌上型電腦到手機。
- **漸進式增強**：這是一種設計方法，確保所有使用者，不論瀏覽器或裝置，皆能存取應用程式的基本內容和功能。
- **以使用者為中心的設計**：這是一種優先考量終端用戶，從他們的需求到偏好和限制的方法。以使用者為中心設計的目標是打造一個直覺、高效且讓你的用戶樂於互動的網頁。
- **使用者研究**：這是對使用你產品的人的系統性研究。目標是衡量使用者需求、行為和痛點。
- **使用者測試**：這是指在使用者與你的應用程式介面互動時擷取資料的做法。
- **A/B 測試**：這是將新特性發佈給隨機選取的部分使用者群的行程。你可以利用分析資料來判斷該特性是否有益。
- **使用者需求**：這是指你的應用程式需要遵循的故事或評分標準。使用者需求可能由使用者研究或產業標準定義。它們甚至可以由利害關係人輸入定義。
- **漸進式揭露**：這是一種設計樣式，用於根據用戶當前的活動只顯示相關內容並隱藏其餘部分。這樣做是為了減少認知負擔並使用戶體驗更直覺。
- **延緩/懶惰註冊**：這是一種 UI 設計樣式，允許使用者瀏覽並互動你的應用程式，而不必註冊。

## 設計最佳實務

- **深色模式**：這是在網頁應用程式上的一項特殊功能，你可以將預設的淺色組織綱目更改為深色組織綱目。在深色模式中，你應該使用去飽和色。去飽和色是指強度較低且飽和度較低的顏色。
- **Breadcrumbs**：這是一種導覽輔助工具，顯示使用者在網站階層體系中的位置。最好將 breadcrumbs 放置在頁面頂部，讓使用者能輕鬆找到它。此外，你要確保 breadcrumbs 足夠大以便輕鬆閱讀，但又不能太大以免佔用頁面過多空間。
- **卡片組件**：你的卡片組件設計應該簡潔，不要視覺上過於雜亂或顯示過多資訊。對於媒體，請確保選擇高品質的影像和影片以提升使用者體驗。
- **無限滾動**：這是一種設計樣式，會在使用者向下捲動頁面時載入更多內容。你應該考慮使用「載入更多」按鈕，因為它讓使用者能控制何時想要看到更多內容。你也可以為使用者添加返回按鈕，讓他們能夠回到前一頁，而不必捲動回頁面頂端。
- **模態對話盒**：這是一種會顯示在現有頁面內容之上的快顯視窗。通常背景內容會有暗色覆蓋，以幫助使用者更專注於模態內容。此外，允許使用者點擊模態外部以關閉它，總是個好主意。當你使用 HTML 的 `dialog` 元素時，你將獲得許多內建的功能和存取利益。
- **表單註冊的進度指示**：這是一種向使用者顯示他們在處理程序中進度的方法。它可以用於表單、註冊和設定處理程序。你的設計應該簡單、容易找到，並且能夠返回到先前的步驟。
- **購物車**：購物車是使用者在電子商務平台上查看已選擇條款的地方。你的購物車應該始終對使用者可見，使用常見的圖示如購物車、袋子或籃子，並且有明確的呼叫行動按鈕，讓使用者能繼續結帳。

## 絕對單位

- **`px` (像素)**：這個絕對單位是在 CSS 中固定大小的度量單位，提供對尺寸的精確控制。這表示 1px 永遠相等於 1/96 英吋。
- **`in` (Inch)**：此絕對單位相當於 96px。
- **`cm` (公分)**：此絕對單位相當於 25.2/64 英吋。
- **`mm` (毫米)**：此絕對單位相當於 1/10 公分。
- **`q` (四分之一毫米)**：這個絕對單位相當於 1/40 公分。
- **`pc` (Picas)**：這個絕對單位相當於 1/6 英吋。
- **`pt` (Points)**：此絕對單位相當於 1/72 英吋。

## 相對單位

- **百分比**：這些相對單位允許你將大小、尺寸和其他屬性定義為其父元素的比例。<dfn>例如</dfn>，如果你在元素上設定 `width: 50%;`，它將佔據其父容器寬度的一半。
- **`em` 單位**：這些單位是相對於元素的字型大小。如果你在 `font-size` 屬性中使用 `ems`，文字的大小將相對於父元素的字型大小。
- **`rem` 單位**：這些單位是相對於根元素的字型大小，也就是 `html` 元素。
- **`vh` 單位**：`vh` 代表「viewport height」，`1vh` 相當於視窗高度的 1%。
- **`vw` 單位**：`vw` 代表「viewport width」，`1vw` 相當於視窗寬度的 1%。

## `calc` 函式（程式）

- **`calc()` 函式（程式）**：使用 `calc()` 函式（程式），你可以直接在你的樣式表中執行計算，以動態決定屬性值。這表示你可以透過根據視窗大小或其他元素計算尺寸，來創建彈性且響應式的使用者介面。

## 使用者動作偽類

- **偽類別定義**：這些是特殊的 CSS 關鍵字，允許你根據元素的特定狀態或位置來選取該元素。
- **使用者操作偽類**：這些是特殊關鍵字，允許你根據使用者互動改變元素的外觀，提升整體使用者體驗。
- **`:active` 偽類別**：此偽類別讓你選取元素的作用中狀態，例如點擊按鈕時。
- **`:hover` 偽類**：此偽類定義元素的暫留狀態。
- **`:focus` 偽類**：當元素獲得焦點時，此偽類會套用樣式，通常是透過鍵盤導覽或使用者點擊進入表單輸入時。
- **`:focus-within` 偽類別**：當元素本身或其任何子孫擁有焦點時，使用此偽類別來套用樣式。

## 輸入偽類別

- **輸入偽類**：這些偽類用於根據使用者互動前後的狀態來標的 HTML `input` 元素。
- **`:enabled` 偽類**：此偽類用於標的目前已啟用的表單按鈕或其他元素。
- **`:disabled` 偽類別**：此偽類別讓你為作用中的互動元素設定停用模式的樣式。
- **`:checked` 偽類**：此偽類用於向使用者顯示它已被選取。
- **`:valid` 偽類**：此偽類標的符合驗證條件的輸入欄。
- **`:invalid` 偽類別**：此偽類別標的未符合驗證條件的輸入欄。
- **`:in-range` 和 `:out-of-range` 偽類別**：這些偽類別根據元素的值是否在指定的範圍條件內或外，套用樣式。
- **`:required` 偽類別**：此偽類別標的具有 `required` 屬性的 `input` 元素。它向使用者表示必須填寫該欄位才能提交表單。
- **`:optional` 偽類**：此偽類將樣式套用到非必填且可以留空的輸入元素。
- **`:autofill` 偽類別**：此偽類別會將樣式套用到瀏覽器自動填入已儲存資料的輸入欄位。

## 位置偽類

- **位置偽類**：這些偽類用於為目前文件中被標的的聯結和元素設定樣式。
- **`:any-link` 偽類**：此偽類是 `:link` 和 `:visited` 偽類的組合。因此，它會匹配任何具有 href 屬性的錨點元素，無論是否已造訪。
- **`:link` 偽類別**：此偽類別允許你標的所有尚未造訪的連結。你可以用它在使用者點擊之前，為連結設定不同的樣式。
- **`:local-link` 偽類別**：此偽類別標的指向相同文件的連結。當你想區分內部連結與外部連結時，它會很有用。
- **`:visited` 偽類別**：此偽類別標的使用者已造訪的聯結。
- **`:target` 偽類別**：此偽類別用於將樣式套用到作為 URL 片段標的的元素。

## 樹狀結構偽類別

- **樹狀結構偽類**：這些偽類允許你根據元素在文件樹中的位置來標的並設定樣式。
- **`:root` 偽類別**：此偽類別通常是根 `html` 元素。它可幫助你標的文件中的最高層級，從而為整個文件套用共用樣式。
- **`:empty` 偽類別**：空元素，也就是除了空白字元外沒有其他子元素的元素，也包含在文件樹中。這就是為什麼有 `:empty` 偽類別來標的空元素。
- **`:nth-child(n)` 偽類**：此偽類允許你根據元素在父元素中的位置來選取元素。
- **`:nth-last-child(n)` 偽類別**：此偽類別可讓你從結尾開始計數以選取元素。
- **`:first-child` 偽類**：此偽類會選取父元素或文件中的第一個元素。
- **`:last-child` 偽類別**：此偽類別會選取父元素或文件中的最後一個元素。
- **`:only-child` 偽類**：此偽類會選取父元素或文件中唯一的元素。
- **`:first-of-type` 偽類別**：此偽類別會選取其父元素中指定元素型別的第一個出現。
- **`:last-of-type` 偽類別**：此偽類別會選取其父元素中指定元素型別的最後一個出現。
- **`:nth-of-type(n)` 偽類別**：此偽類別允許你根據同類型兄弟元素中的位置，選取其父元素內的特定元素。
- **`:only-of-type` 偽類別**：如果元素是在其父元素中唯一的該型別，則此偽類別會選取該元素。


## 功能性偽類別

- **功能性偽類別**：功能性偽類別允許你根據更複雜的條件或關係來選取元素。與根據狀態（例如，`:hover`、`:focus`）來標的元素的一般偽類別不同，功能性偽類別接受引數。
- **`:is()` 偽類**：此偽類接受一個選擇器列表（例如 `ol`、`ul`），並選取符合列表中任一選擇器的元素。
- **`:where()` 偽類別**：此偽類別接受一個選擇器列表（例如 `ol`、`ul`），並選擇符合列表中任一選擇器的元素。` :is` 與 `:where` 的差別在於後者的特異性為 0。
- **`:has()` 偽類別**：這個偽類別常被稱為「`parent`」選擇器，因為它允許你為包含選擇器列表中指定子元素的元素設定樣式。

## 偽元素

- **`::before` 偽元素**：此偽元素使用 `content` 屬性在元素之前插入像圖示這樣的裝飾性內容。
- **`::after` 偽元素**：此偽元素使用 `content` 屬性在元素之後插入像圖示這樣的裝飾性內容。
- **`::first-letter` 偽元素**：此偽元素會標的元素內容的第一個字母，讓你可以為它設定樣式。
- **`::marker` 偽元素**：此偽元素讓你選取列表項目的標記（項目符號或編號）以進行樣式設定。


## 色彩理論

- **色彩理論定義**：這是研究顏色如何相互作用以及它們如何影響我們的感知。它涵蓋色彩關係、色彩和諧以及色彩的心理影響。
- **主要顏色**：這些顏色為黃色、藍色和紅色，是所有其他顏色的基本色調。
- **次要色彩**：這些色彩是由混合相等量的兩種主要色彩所產生。綠色、橘色和紫色是次要色彩的範例。
- **第三色**：這些顏色是由原色與相鄰的次色混合而成。黃綠、藍綠和藍紫是第三色的例子。
- **暖色系**：這些顏色包括紅色、橘色和黃色，喚起舒適、溫暖和溫馨的感覺。
- **冷色調**：這些顏色包括藍色、綠色和紫色，喚起平靜、安詳和專業的感覺。
- **色彩輪**：色彩輪是一個圓形圖表，顯示顏色之間的關係。它是設計師的重要工具，因為它幫助他們選擇顏色組合。
- **類似色結構綱目**：這些色彩結構綱目創建連貫且舒緩的體驗。它們具有類似色，這些色彩在色輪上彼此相鄰。
- **互補色結構綱目**：這些色彩結構綱目產生高度對比和視覺衝擊。它們的顏色位於色輪的相對兩端。
- **三元色結構綱目**：這種結構綱目具有鮮豔的顏色。它們由彼此大約等距的顏色組成。如果將它們連接起來，會在色輪上形成一個等邊三角形。
- **單色結構綱目**：對於此結構綱目，所有顏色皆由相同的基色透過調整其明度、暗度和飽和度而產生。這喚起統一與和諧的感覺，同時仍創造對比。

## 在 CSS 中使用顏色的不同方式

- **命名顏色**：這些顏色是瀏覽器識別的預定義顏色名稱。範例包括 `blue`、`darkred`、`lightgreen`。
- **`rgb()` 函式（程式）**：RGB 代表紅色、綠色和藍色——光的三原色。這三種顏色以不同強度組合，產生廣泛的顏色範圍。`rgb()` 函式（程式）允許你使用 RGB 色彩模型定義顏色。
- **`rgba()` 函式（程式）**：此函式（程式）新增第四個數值 — alpha — 用來控制顏色的透明度。
- **`hsl()` 函式（程式）**：HSL 代表色相、飽和度和亮度——定義顏色的三個關鍵組件。
- **`hsla()` 函式（程式）**：此函式（程式）新增第四個數值 — alpha — 用來控制顏色的不透明度。
- **十六進位**：十六進位碼（hex code 的縮寫）是一個六字元字串，用來表述 RGB 色彩模型中的顏色。「hex」指的是使用數字 0 到 9 及字母 A 到 F 的 16 進位制系統。

## `box-shadow` 屬性

- **定義**：`box-shadow` 屬性會在元素周圍套用一個或多個陰影。
- **位移值**：你必須指定水平 (`offset-x`) 和垂直 (`offset-y`) 值。正的 `offset-x` 會將陰影向右移動，負值則向左移動。正的 `offset-y` 會將陰影向下移動，負值則向上移動。如果值為 `0`，則不需要包含單位。
- **模糊半徑**：此選用值控制陰影的模糊程度。如果未包含，預設為 `0`，會產生銳利的邊緣。數值越高，陰影越柔和。
- **Spread Radius**：這個選用值控制陰影擴展或縮小的程度。如果未包含，則預設為 `0`。
- **陰影顏色**：你可以使用命名顏色、十六進位值、`rgb()`、`rgba()`、`hsl()` 或 `hsla()` 函式指定顏色。
- **`inset` 關鍵字**：添加 `inset` 關鍵字會將陰影放置在元素內部，而非外部。
- **套用多重盒子陰影**：你可以用逗號分隔來套用多個陰影。陰影會從前到後分層。

### 語法

```css
box-shadow: offset-x offset-y blur-radius spread-radius color;
```

## 線性與放射漸層

- **線性漸層**：這些漸層會沿著直線在顏色之間產生漸進混合。你可以控制這條線的方向和使用的顏色。
- **放射狀漸層**：這些漸層會從中心點輻射出圓形或橢圓形的漸層。

## 輸入樣式的最佳實踐

- **輸入欄位樣式**：如同所有文字元素，你需要確保套用到文字輸入欄位的樣式是可存取的。這表示字型需要有足夠大小，且顏色需要與背景有足夠對比。輸入元素也可以被聚焦。當你編輯樣式時，應該注意保留明顯的指示器，當元素被聚焦時，例如粗體邊框。

## 為輸入使用 `appearance: none`

- **`appearance: none`**：瀏覽器會對許多元素套用預設樣式。`appearance: none` CSS 屬性讓你能完全控制樣式，但也有一些注意事項。當為輸入元素建置訂製樣式時，你需要確保焦點和錯誤指示仍然存在。

## 常見問題：為 `datetime-local` 和 `color` 屬性設計樣式

- **常見問題**：這些特殊類型的輸入依賴複雜的偽元素來創建像是日期和顏色選擇器之類的功能。這對於這些輸入的樣式設計帶來了重大挑戰。一個挑戰是預設樣式完全依賴瀏覽器，因此你撰寫的 CSS 來讓選擇器呈現你想要的樣子，在另一個瀏覽器上可能會完全不同。

## CSS 上限溢位(相對於 underflow) 屬性

- **定義**：上限溢位指的是元素控制代碼超出或「溢出」包含元素大小的內容的方式。上限溢位是二維的。
- **`overflow-x`**：x 軸決定水平上限溢位(相對於 underflow)。
- **`overflow-y`**：y 軸決定垂直上限溢位(相對於 underflow)。

## CSS 轉換屬性

- **定義**：此屬性可讓你對元素套用各種轉換，例如旋轉、縮放、傾斜或在 2D 或 3D 空間中平移（移動）它們。
- **`translate()` 函式（程式）**：此函式用於將元素從其目前位置移動。
- **`scale()` 函式（程式）**：此函式允許你改變元素的大小。
- **轉換與存取性**：如果你使用 transform 來隱藏或顯示內容，請確保內容仍可被螢幕閱讀器和鍵盤導覽存取。隱藏的內容應該是真正隱藏，例如使用 `display: none` 或 `visibility: hidden`，而不是僅僅視覺上移出螢幕。

## 盒模型

- **定義**：在 CSS 盒模型中，每個元素都被一個盒子包圍。這個盒子由四個組件組成：內容區域、`padding`、`border`、`margin`。
- **內容區域**：內容區域是盒子最內層的部分。它是包含元素實際內容（例如文字或影像）的空間。
- **`padding`**：`padding` 是緊接內容區域之後的區域。它是內容區域與元素邊框之間的空間。
- **`border`**：`border` 是 CSS 盒模型中元素的外緣或輪廓。它是元素的視覺邊界。
- **`margin`**：`margin` 是元素邊框外的空間。它決定元素與周圍其他元素之間的距離。

## 外邊距合併

- **定義**：當相鄰元素的垂直外邊距重疊時，會發生此行為，導致單一外邊距等於兩者中較大的那一個。此行為僅適用於垂直外邊距（上方和下方），不適用於水平外邊距（左方和右方）。

## `content-box` 和 `border-box` 屬性值

- **`box-sizing` 屬性**：此屬性用於決定 HTML 元素的最終 `width` 和 `height` 如何計算。
- **`content-box` 值**：在 `content-box` 模型中，你為元素設定的 `width` 和 `height` 決定內容區域的尺寸，但不包含 `padding`、`border` 或 `margin`。
- **`border-box` 值**：使用 `border-box` 時，元素的 `width` 和 `height` 包含內容區域、`padding` 和 `border`，但不包含 `margin`。

## CSS 重置

- **定義**：CSS 重置是一種樣式表，用來移除網頁瀏覽器套用於 HTML 元素的全部或部分預設格式。第三方的 CSS 重置選項包括 `sanitize.css` 和 `normalize.css`。

## CSS 濾鏡屬性

- **定義**：此屬性可用於創建各種效果，例如模糊、色彩偏移和對比度調整。
- **`blur()` 函式（程式）**：此函式（程式）對元素套用高斯模糊。數值以像素為單位，表述模糊的半徑。
- **`brightness()` 函式（程式）**：此函式調整元素的亮度。值為 0% 時，元素會變成完全黑色，而超過 100% 的值會增加亮度。
- **`grayscale()` 函式（程式）**：此函式（程式）將元素轉換為灰階。數值以百分比定義，100％ 為完全灰階，0％ 則保持影像不變。
- **`sepia()` 函式（程式）**：此函式（程式）會將元素套用棕褐色調。與灰階類似，它使用百分比數值。
- **`hue-rotate()` 函式（程式）**：此函式（程式）對元素套用色相旋轉。該值以度數定義，並表述繞色彩圓的旋轉。

## CSS Flexbox 與 Flex 模型簡介

- **定義**：CSS flexbox 是一種一維佈局模型，允許你在容器內以列和行排列元素。
- **Flex Model**：此模型定義了彈性項目在彈性容器內的排列方式。每個彈性容器都有兩個軸：主軸和交叉軸。

## `flex-direction` 屬性

- **定義**：此屬性設定主軸的方向。`flex-direction` 的預設值為 `row`，會將所有的彈性項目放在同一列，方向依照你的瀏覽器預設語言（由左至右或由右至左）。
- **`flex-direction: row-reverse;`**：這會反轉列中的項目。
- **`flex-direction: column;`**：這將使彈性項目垂直排列，而非水平排列。
- **`flex-direction: column-reverse;`**：這會垂直反轉彈性項目的順序。

## `flex-wrap` 屬性

- **定義**：此屬性決定彈性項目如何在彈性容器內換行以適應可用空間。`flex-wrap` 可以採用三種可能的值：`nowrap`、`wrap` 和 `wrap-reverse`。
- **`flex-wrap: nowrap;`**：這是預設值。彈性項目不會換行，即使它們的寬度超過容器的寬度。
- **`flex-wrap: wrap;`**：當項目超過其容器的寬度時，這個屬性會將項目換行。
- **`flex-wrap: wrap-reverse;`**：此屬性會以反向順序換行彈性項目。
- **`flex-flow` 屬性**：此屬性是 `flex-direction` 和 `flex-wrap` 的縮寫屬性。

## `justify-content` 屬性

- **定義**：此屬性沿著彈性容器的主軸對齊子元素。
- **`justify-content: flex-start;`**：在這種情況下，彈性項目會對齊到主軸的起點。這可以是水平或垂直方向。
- **`justify-content: flex-end;`**：在這種情況下，彈性項目會沿主軸的末端對齊，無論是水平還是垂直方向。
- **`justify-content: center;`**：這會將彈性項目沿主軸置中。
- **`justify-content: space-between;`**：這會沿主軸均勻分配元素。
- **`justify-content: space-around;`**：這會在主軸內均勻分配彈性項目，並在第一個項目前及最後一個項目後添加空間。
- **`justify-content: space-evenly;`**：這會沿主軸平均分配項目。
  
## `align-items` 屬性

- **定義**：此屬性用於沿交叉軸分配項目。請記住交叉軸垂直於主軸。
- **`align-items: center;`**：這用於沿交叉軸置中項目。
- **`align-items: flex-start;`**：這會將項目對齊到交叉軸的起點。
- **`align-items: stretch;`**：這用於沿交叉軸拉伸彈性項目。

## 排版介紹

- **定義**：排版是選擇合適字型和格式，使文字在視覺上吸引人且易於閱讀的藝術。 「字型」指的是個別字元的設計和排列方式。
- **字型定義**：字型是一組字元、數字和符號的整體設計與風格。它就像是一個字體家族的藍圖。
- **字型定義**：字型是字體的特定變體，具有特定的特徵，例如大小、粗細、樣式和寬度。
- **襯線字型**：這種字型具有經典風格，字元末端帶有小線條。襯線字型通常用於列印材料，如書籍。
- **無襯線字型**：這種字型看起來較為現代，字元末端沒有小線條。無襯線字型常用於數位設計，因為它們在螢幕上易於閱讀。一些範例包括 Helvetica、Arial 和 Roboto。
- **字型粗細**：這是字元的粗細度，包括細體、常規、粗體和黑體。
- **字型樣式**：這是字元的傾斜和方向，例如斜體和傾斜體。
- **字型寬度**：這是字元所佔用的水平空間，例如壓縮和擴展。
- **Baseline**：這是大多數字元所依附的虛擬線。
- **大寫字母高度**：這是大寫字母的高度，從基線量測到頂部。
- **X-height**：這是小寫字母的平均高度，不包括升部和降部。
- **Ascenders**：這些是小寫字母中超出 x-height 的零件，例如字母 `h`、`b`、`d` 和 `f` 的頂部。
- **Descenders**：這些是小寫字母中延伸到基線以下的零件，例如 'y'、'g'、'p' 和 'q' 的尾巴。
- **字距調整**：這是如何調整特定字元對組之間的間距，以提升它們的可讀性和美觀度。例如，減少字母 A 和 V 之間的間距。
- **Tracking**：這是調整一個區塊中所有字元之間間距的方式。它本質上是字元之間的距離。它會影響文字的密集度與開闊度。
- **行距**：這是文字行之間的垂直間距。它是從一行的基線測量到下一行的基線。
- **字型的最佳實務**：你應該選擇清晰且簡單的字型，使你的設計易於理解。這對於你網站的主要文字尤其重要。如果字型易於閱讀，使用者更有可能與你的內容互動。你應該最多使用兩到三種字型，以建立視覺一致性。使用過多字型會使文字更難閱讀，並削弱你的品牌識別度。這也可能影響使用者體驗，因為會增加網站的載入時間。你可以使用字型大小來為標題、副標題、段落及其他元素創建視覺階層體系。例如，網頁上的主標題應該使用較大的字型，接著是字型大小較小的副標題。這將為階層體系中的每個元素賦予特定的字型大小，幫助使用者透過視覺導覽結構。

- **常見字型家族**：以下是 CSS 中常用的一些字型家族：serif、sans-serif、monospace、cursive、fantasy

## 網頁安全字型

- **定義**：這些字型是非常可能安裝在電腦或裝置上的字型子集。當瀏覽器必須算繪字型時，它會嘗試在使用者系統中尋找字型檔案。但如果找不到該字型，通常會回退到預設系統字型。

- **網頁安全字型**：
  - 無襯線字型常用於網頁開發，因為它們的字元末端沒有小「腳」或線條，因此在螢幕上易於閱讀。一些網頁安全的無襯線字型範例有：Arial、Verdana 和 Trebuchet MS。

## At-rules 與 `@font-face` At-rule

- **定義**：At-rules 是提供瀏覽器指令的述語。你可以使用它們來定義樣式表的各種面向，例如媒體查詢、keyframes、字型臉（font faces）等。
- **`@font-face`**：這允許你透過指定字型檔案、格式及其他重要屬性，如粗細和樣式，來定義訂製字型。為了使 `@font-face` at-rule 有效，你還需要指定 `src` 屬性。此屬性包含對字型資源的引用。
- **字型格式**：對於每個字型資源，你也可以指定格式。這是選用的。它是瀏覽器關於字型格式的提示。如果省略格式，資源將會被下載，並在下載後偵測格式。如果格式無效，資源將不會被下載。可能的字型格式包括 `collection`、`embedded-opentype`、`opentype`、`svg`、`truetype`、`woff` 和 `woff2`。
- **`woff` 和 `woff2`**：`woff` 代表「Web Open Font Format」。它是一種由 Mozilla 與 Type Supply、LettError 及其他組織合作開發的廣泛使用字型格式。`woff` 和 `woff2` 之間的差異在於用來壓縮資料的演算法。
- **OpenType**：這是一種由 Microsoft 和 Adobe 開發的可縮放電腦字型格式，允許使用者存取字型中的額外功能。它廣泛應用於主要作業系統。
- **`tech()`**：這用於指定字型資源的技術。這是選用的。

## `text-shadow` 屬性

- **定義**：此屬性用於為文字添加陰影。你需要指定 X 和 Y 偏移量，分別表述陰影相對於文字的水平和垂直距離。這些數值為必填。X 偏移量和 Y 偏移量的正值會分別將陰影向右和向下移動，而負值則會將陰影向左和向上移動。
- **陰影顏色**：你也可以透過在偏移量之前或之後指定此值來自訂陰影的顏色。如果未指定顏色，瀏覽器會自動決定顏色，因此通常最好明確設定。
- **模糊半徑**：模糊半徑為選用，但會讓陰影看起來更平滑且更細緻。模糊半徑的預設值為零。數值越高，模糊越大，代表陰影變得越淡。
- **套用多重文字陰影**：文字可以有多個陰影。陰影會以層疊方式套用，從前到後，第一個陰影位於最上層。

## 無障礙樹狀結構

無障礙樹是一種結構，用於輔助技術，例如螢幕閱讀器，來解讀和互動網頁上的內容。

## `max()` 函式（程式）

`max()` 函式傳回一組以逗號分隔的值中最大的值：

```css
img {
  width: max(250px, 25vw);
}
```

在上述範例中，如果視窗寬度小於 1000 像素，影像的寬度將會是 250px（因為 250px 大於該視窗寬度的 25％）。如果視窗寬度大於 1000 像素，影像的寬度將會是 25vw。這是因為 25vw 相等於視窗寬度的 25％。

## 使用 CSS 和無障礙性的最佳實踐

- **`display: none;`**：使用 `display: none;` 意味著螢幕閱讀器和其他輔助技術將無法存取此內容，因為它不包含在無障礙樹中。因此，只有當你想要完全從視覺呈現和無障礙中移除內容時，才應該使用此方法。
- **`visibility: hidden;`**：此屬性和值會將內容視覺上隱藏，但仍保留在文件流程中，表示它仍佔據頁面上的空間。這些元素也不會被螢幕閱讀器讀取，因為它們已從存取樹中移除。
- **`.sr-only` CSS 類別**：這是一種常見技術，用來視覺上隱藏內容，同時保持其對螢幕閱讀器的可存取性。

```css
.sr-only {
  position: absolute;
  width: 1px;
  height: 1px;
  padding: 0;
  margin: -1px;
  overflow: hidden;
  clip: rect(0, 0, 0, 0);
  white-space: nowrap;
  border: 0;
}
```

- **`scroll-behavior: smooth;`**：此屬性和值啟用平滑滾動行為。
- **`prefers-reduced-motion` 特性**：這是一個媒體特性，可用來偵測使用者的動畫偏好。

```css
@media (prefers-reduced-motion: no-preference) {
  * {
    scroll-behavior: smooth;
  }
}
```

在上述範例中，如果使用者的裝置沒有設定動畫偏好，則會啟用平滑捲動。

- **`hidden` 屬性**：此屬性被大多數現代瀏覽器支援（某功能），並且會在視覺上及無障礙樹中隱藏內容。它可以透過 JavaScript 輕鬆切換。

```html
<p aria-hidden>This paragraph is visible for sighted users, but is hidden from assistive technology.</p>
<p hidden>This paragraph is hidden from both sighted users and assistive technology.</p>
```

:::

## 使用浮點數

- **定義**：浮動用於將元素從頁面上的正常流程中移除，並將其定位在其容器的左側或右側。當發生這種情況時，文字會環繞該浮動內容。
- **清除浮動**：`clear` 屬性用於判斷元素是否需要移動到浮動內容的下方。當你有多個浮動元素並排堆疊時，版面配置可能會出現重疊和塌陷的問題。因此，產生了 `clearfix` 技巧來解決這個問題。

## 靜態、相對和絕對定位

- **靜態定位**：這是文件的正常流程。元素依序從上到下、從左到右定位。
- **相對定位**：這允許你使用 `top`、`left`、`right` 和 `bottom` 屬性在正常文件流程中定位元素。你也可以使用相對定位讓元素與頁面上的其他元素重疊。
- **絕對定位**：這讓你可以將元素從正常的文件流程中取出，使其行為獨立於其他元素。

## 固定與黏性定位

- **固定定位**：當元素使用 `position: fixed` 定位時，它會從正常的文件流程中移除，並相對於視窗定位，這表示即使使用者滾動，該元素仍會保持在相同位置。這通常用於像標頭或導覽列這類需要始終保持可見的元素。
- **Sticky Positioning**：這種類型的定位會作為相對定位和固定定位的混合體。最初，元素的行為就像是相對定位，保持在文件的流程中。然而，一旦使用者將元素捲動過某個點，它就會「黏」在視窗（通常是頂部），並且行為就像是固定定位。這非常適合用來建立像是黏性導覽列這類元素，只有當使用者捲動到某個位置時才會變成固定。

- **定義**：CSS 中的 `z-index` 屬性用於控制頁面上重疊定位元素的垂直堆疊順序。

## CSS 訂製屬性（CSS 變數）

- **定義**：CSS 訂製屬性，也稱為 CSS 變數，是由 CSS 作者定義的實體，包含特定值以便在整個文件中重複使用。它們是一項強大的功能，允許更高效、可維護且靈活的樣式表。訂製屬性在建立可主題化設計時特別有用。你可以為不同主題定義一組屬性。

## `@property` 規則

- **定義**：`@property` 規則是一個強大的 CSS 功能，允許開發者訂製屬性，並對其行為擁有更高的控制元件，包括它們如何動畫以及初始值。

```css
@property --property-name {
  syntax: '<type>';
  inherits: true | false;
  initial-value: <value>;
}
```

- **`--property-name`**：這是你正在定義的訂製屬性名稱。像所有訂製屬性一樣，它必須以兩個破折號開頭。  
**`syntax`**：這定義了屬性的型別，可以是像 `<color>`、`<length>`、`<number>`、`<percentage>` 或更複雜的型別。
- **`inherits`**：這指定該屬性是否應該從其父元素繼承值。
- **`initial-value`**：這會設定該屬性的預設值。
- **後備方案**：當使用訂製屬性時，你可以使用 `var()` 函式（程式）提供後備值，就像使用標準訂製屬性一樣。

## CSS Grid 基礎

- **定義**：CSS Grid 是一種用於在網頁中創建複雜版面配置的二維版面系統。格線會有列和行，且它們之間會有間隙。若要定義格線版面配置，你需要將 `display` 屬性設為 `grid`。
- **`fr`（分數）單位**：此單位表述網格容器內空間的一部分。你可以使用 `fr` 單位來創建彈性網格。
- **在軌道之間創建間隙**：在 CSS grid 中有三種方法可以在軌道之間創建間隙。你可以使用 `column-gap` 屬性在行之間創建間隙。你可以使用 `row-gap` 屬性在列之間創建間隙。或者你可以使用 `gap` 簡寫屬性在列和行之間同時創建間隙。
- **`repeat()` 函式（程式）**：此函式用於重複軌道列表中的區段。你可以使用 `repeat()` 函式來取代撰寫 `grid-template-columns: 1fr 1fr 1fr;`。
- **明確的網格**：你可以使用 `grid-template-columns` 或 `grid-template-rows` 屬性來指定行數或軌道數。
- **隱式網格**：當項目被放置在網格外時，系統會自動為那些外部元素創建列和行。
- **`minmax()` 函式（程式）**：此函式用於設定軌道的最小和最大尺寸。
- **Grid Auto-Placement**：瀏覽器會使用自動配置演算法自動放置項目。`grid-auto-flow` 屬性控制此行為，使用像是 `row`、`column` 或 `dense` 的值來影響放置方向以及是否回填空白間隙。
- **基於行的配置**：所有網格都有行。若要指定條款在某行的開始位置，你可以使用 `grid-column-start` 和 `grid-row-start` 屬性。若要指定條款在該行的結束位置，你可以使用 `grid-column-end` 和 `grid-row-end` 屬性。你也可以選擇使用 `grid-column` 或 `grid-row` 簡寫屬性。
- **`grid-template-areas`**：此屬性用於為你將要在網格上定位的項目提供名稱。
- **對齊：** 使用 `justify-items` 和 `align-items` 來控制項目在其網格區域內的對齊方式。對單一網格項目使用 `justify-self` 和 `align-self` 來改寫容器上設定的對齊方式。

## CSS 動畫基礎

- **定義**：CSS 動畫允許你在網頁上創建動態且視覺吸引人的效果，而無需使用 JavaScript 或複雜的程式設計。它們提供了一種方法，可以在指定的持續時間內平滑地將元素從一種樣式過渡到另一種樣式。
- **`@keyframes` 規則**：此規則定義動畫的階段和樣式。它指定元素在動畫過程中各個時間點應該擁有的樣式。
- **`animation` 屬性**：這是用來套用動畫的縮寫屬性。
- **`animation-name`**：這指定要使用的 `@keyframes` 規則名稱。
- **`animation-duration`**：這設定動畫應該花費多長時間完成。
- **`animation-timing-function`**：這定義動畫隨時間的進展方式（例如 ease、linear、ease-in-out）。
- **`animation-delay`**：這指定動畫開始前的延遲時間。
- **`animation-iteration-count`**：這設定動畫應該重複的次數。
- **`animation-direction`**：這決定動畫應該向前播放、向後播放，或交替播放。
- **`animation-fill-mode`**：這指定元素在動畫開始前和結束後應該如何被樣式化。
- **`animation-play-state`**：這讓你可以暫停和恢復動畫。

## 無障礙功能與 `prefers-reduced-motion` 媒體查詢

- **`prefers-reduced-motion` 媒體查詢**：動畫的主要無障礙性問題之一是它們可能會造成某些使用者不適甚至實體上的傷害。患有前庭疾病或動作敏感的使用者，在螢幕上看到某些類型的移動時，可能會感到頭暈、噁心或頭痛。`prefers-reduced-motion` 媒體查詢允許網頁開發者偵測使用者是否在系統層級要求最小化動畫或動作效果。
