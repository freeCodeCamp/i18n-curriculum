## ¿Qué es npm?

- **Definición**: npm es un registro en línea de software de código abierto, que funciona como una "tienda de aplicaciones" para código.
- **Significado oficial**: npm no significa oficialmente "Node Package Manager"; es una abreviatura recursiva que significa "npm no es un acrónimo".
- **Tres partes relacionadas**:

  - **Sitio web (npmjs.com)**: Para buscar paquetes en el registro.
  - **Registro**: Una gran base de datos pública que almacena todos los paquetes.
  - **CLI (interfaz de línea de comandos)**: Una herramienta instalada con Node.js para gestionar las dependencias del proyecto.

- **Alternativas**: Aunque npm es el más usado, las alternativas populares incluyen **Yarn**, **PNPM** y **Bun**.

## El archivo package.json

- **Propósito**: Un archivo de configuración que almacena metadatos, dependencias y scripts para un proyecto Node.js.
- **Campos clave**:

  - `name` y `version`: Identifican el proyecto.
  - `main`: Especifica el archivo de entrada, a menudo `index.js`.
  - `license`: Define términos legales (por defecto es `ISC`).
  - `dependencies`: Paquetes necesarios para ejecutar la app en producción.
  - `devDependencies`: Herramientas necesarias solo durante el desarrollo, como frameworks de pruebas.

- **Creación**:

  - `npm init`: Inicia un proceso interactivo de configuración.
  - `npm init -y`: Instala un `package.json` con valores predeterminados al instante.

## Versionado semántico (SemVer)

- **Formato**: `MAJOR.MINOR.PATCH`.
- **Reglas**:

  - **MAYOR**: Cambios incompatibles que pueden afectar código existente.
  - **MENOR**: Nuevas funcionalidades compatibles hacia atrás.
  - **PARCHE**: Correcciones pequeñas y mejoras.

- **Símbolos de rango**:

  - `^` (Caret): Permite actualizaciones menores y de parche (por ejemplo, `^1.2.3` permite `1.x.x`).
  - `~` (Tilde): Permite solo actualizaciones de parche (por ejemplo, `~1.2.3` permite `1.2.x`).
  - `*` (Asterisco): Coincide con cualquier versión; se usa principalmente para pruebas.

## Gestión de dependencias

- **Instalación**:

  - `npm install <name>`: Añade una dependencia para producción.
  - `npm install <name> -D`: Añade una dependencia para desarrollo.
  - `npm install <name>@<version>`: Instala una versión específica.

- **Node modules y locks**:

  - `node_modules/`: Contiene el código real de los paquetes instalados.
  - `package-lock.json`: Bloquea las versiones exactas de todas las dependencias y "dependencias hijas" para asegurar consistencia entre entornos.

- **Eliminación**: `npm uninstall <name>` elimina el paquete y actualiza `package.json`.
- **Mantenimiento**:

  - `npm outdated`: Identifica qué paquetes tienen versiones más recientes disponibles.
  - `npm update`: Actualiza paquetes dentro del rango permitido por SemVer.

## Scripts de npm

- **Función**: Comandos personalizados definidos en la sección `scripts` de `package.json` para automatizar tareas.
- **Ejecución**: Se ejecutan mediante `npm run <script_name>`.
- Existen atajos para `start`, `test`, `stop` y `restart` (por ejemplo, `npm start`).
- **Ejecución secuencial**:
- `&&`: Ejecuta el segundo comando solo si el primero tiene éxito.
- `;`: Ejecuta comandos uno tras otro sin importar el resultado.
- **Ejecución concurrente**: Usa el operador `&` para ejecutar procesos en segundo plano.
- **Argumentos**: Pasa argumentos a los scripts usando el separador `--` (por ejemplo, `npm start -- 8000`).

## Publicación en el registro npm

- **Acceso al registro**: Una base de datos de paquetes públicos y privados.
- **Pasos para publicar**:

1. Crea una cuenta gratuita en `npmjs.com`.
2. Inicia sesión desde la terminal con `npm login`.
3. Asegúrate de tener un `name` único y un `version` válido en `package.json`.
4. Crea un `README.md` y un `.npmignore` (para excluir archivos sensibles).
5. Ejecuta `npm publish` (o `npm publish --access public` para paquetes con scope).

- **Scopes**: Se usan para evitar conflictos de nombres publicando bajo `@username/package-name`.

## Sistemas de módulos en Node.js

### CommonJS (CJS)

- **Sistema predeterminado**: La forma original en que Node.js maneja módulos; carga de forma síncrona.
- **Sintaxis**:
- **Importar**: `const module = require('./path')`.
- **Exportar**: `module.exports = { functionName }`.
- **Identificación**: Se usa para archivos `.cjs` o archivos `.js` cuando no se especifica `type` en `package.json`.

### Módulos ES (ESM)

- **Formato estandarizado**: Estándar moderno de JavaScript; carga de forma asíncrona para mayor eficiencia.
- **Sintaxis**:

  - **Exportar**: `export { name }` o `export default function`.
  - **Importar**: `import { name } from './path.mjs'`.
  - **Identificación**: Se usa para archivos `.mjs` o archivos `.js` cuando `package.json` contiene `"type": "module"`.
