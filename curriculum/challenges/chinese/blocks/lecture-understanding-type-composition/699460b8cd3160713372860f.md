# --description--

有时你会比 TypeScript 更了解某个 `type`。这时，`type` 断言就非常有用。

下面是一个使用 `querySelector()` 方法访问 `id` 为 `submit` 的元素的示例：

```ts
const submitBtn = document.querySelector("#submit");
```

现在，TypeScript 只知道这个 `submitBtn` 是某种 `Element` 或可能是 `null`。TypeScript 不会查看对应的超文本标记语言 来确定它的具体类型。

这是使用类型断言告诉 TypeScript 该元素是什么类型的好地方。下面是一个示例：

```ts
const submitBtn = document.querySelector("#submit") as HTMLButtonElement;
```

现在，TypeScript 会因为那个类型断言将其视为 `button` 元素。 

使用 `as` 关键字有一定的限制。例如，如果没有 TypeScript 抛出误差，你不能这样做：

```ts
// Conversion of type 'string' to type 'number' may be a mistake because neither type sufficiently overlaps with the other. If this was intentional, convert the expression to 'unknown' first.
const age = "awesome" as number;
```

在此示例中，尝试断言字串 `"awesome"` 应被视为数字类型没有多大意义。TypeScript 仅允许在兼容或重叠的类型之间进行类型断言。

一种消除该误差的方法是使用如下的双重断言：

```ts
const age = "awesome" as unknown as number;
```

尽管这在 TypeScript 中可行，但仍不推荐这样做，因为从逻辑上讲这没有太大意义。

另一种编写类型断言的方法是使用尖括弧语法，如下所示：

```ts
const submitBtn = <HTMLButtonElement>document.querySelector("#submit");
```

当使用 `tsx` 文件时，你需要小心这种语法，因为 TypeScript 会认为你正在尝试渲染一个组件并抛出如下误差：

```ts
// file: index.tsx
// This JSX tag requires 'React' to be in scope, but it could not be found.
// JSX element 'HTMLButtonElement' has no corresponding closing tag.
const submitBtn =  <HTMLButtonElement>document.querySelector("#submit");
```

在大多数情况下，你会看到使用 `as` 关键字的第一种语法被优先使用，而不是第二种选项。
