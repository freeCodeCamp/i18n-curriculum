# --description--

In der `index.pug` Datei, die bereitgestellt wird, gibt es ein Login-Formular. Es ist versteckt wegen des inline JavaScript `if showLogin`, mit dem das Formular eingerückt danach steht.

Fügen Sie im `res.render` für diese Seite eine neue Variable zum Objekt `showLogin: true` hinzu. Wenn Sie Ihre Seite aktualisieren, sollten Sie dann das Formular sehen! Dieses Formular ist so eingerichtet, dass es **POST** an `/login` sendet. Hier sollten Sie also die Annahme der POST-Anfrage einrichten und den Benutzer authentifizieren.

Für diese Aufgabe sollten Sie die Route `/login` hinzufügen, um eine POST-Anfrage zu akzeptieren. Um auf dieser Route zu authentifizieren, müssen Sie eine Middleware hinzufügen, die dies übernimmt, bevor Sie eine Antwort senden. Das geschieht, indem Sie einfach ein weiteres Argument mit der Middleware vor Ihrer Antwort übergeben. Die Middleware, die Sie verwenden sollen, ist `passport.authenticate('local')`.

`passport.authenticate` kann auch einige Optionen als Argument annehmen, wie zum Beispiel `{ failureRedirect: '/' }`, was unglaublich nützlich ist, also fügen Sie das unbedingt auch hinzu. Fügen Sie nach der Verwendung der Middleware eine Antwort hinzu (die nur aufgerufen wird, wenn die Authentifizierungs-Middleware erfolgreich ist), die den Benutzer zu `/profile` umleitet. Fügen Sie diese Route ebenfalls hinzu und lassen Sie sie die Ansicht `profile.pug` rendern.

Wenn die Authentifizierung erfolgreich war, wird das Benutzerobjekt in `req.user` gespeichert.

An diesem Punkt, wenn Sie einen Benutzernamen und ein Passwort im Formular eingeben, sollte es zur Startseite `/` umleiten, und die Konsole Ihres Servers sollte `'User {USERNAME} attempted to log in.'` anzeigen, da wir derzeit keinen Benutzer anmelden können, der nicht registriert ist.

Reichen Sie Ihre Seite ein, wenn Sie denken, dass alles richtig ist. Wenn Sie auf Fehler stoßen, können Sie <a href="https://forum.freecodecamp.org/t/advanced-node-and-express/567135#how-to-use-passport-strategies-7" target="_blank" rel="noopener noreferrer nofollow">das bis zu diesem Punkt abgeschlossene Projekt ansehen</a>.
