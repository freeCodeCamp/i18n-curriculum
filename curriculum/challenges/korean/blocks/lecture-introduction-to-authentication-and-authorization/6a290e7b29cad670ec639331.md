# --description--

맞춤 인증을 설정하려면 상당한 양의 수동 로직을 작성해야 합니다. 요청 헤더를 읽고, 쿠키를 추출하며, 서명을 검증하고, 오류를 관리하는 작업을 처리해야 합니다.

직접 구축하는 것은 학습에 좋지만, 모든 프로덕션 애플리케이션마다 이 작업을 하는 것은 시간이 많이 걸리고 보안 실수가 발생하기 쉽습니다. 그래서 개발자들이 Passport.js를 사용합니다.

Passport.js는 Node.js용 인증 미들웨어입니다. 요청 인증 방식을 표준화하는 체계적인 프레임워크 역할을 합니다. 저수준 인증 세부사항을 처리해 애플리케이션의 비즈니스 로직에 집중할 수 있게 합니다.

Passport.js의 핵심 아이디어는 모든 애플리케이션이 로그인하는 방식은 다르지만, 기본 인증 흐름은 동일해야 한다는 것입니다. Passport는 **Strategies**라는 모듈식 플러그인을 통해 이 유연성을 구현합니다.

모든 인증 방법마다 맞춤 코드를 작성하는 대신 필요한 특정 전략을 설치하세요. 현재 500개가 넘는 전략이 있어 거의 모든 인증 방식을 지원합니다. 그중 일부는 다음과 같습니다:

- **passport-local:** 자체 데이터베이스를 대상으로 전통적인 사용자 이름과 비밀번호 인증을 처리합니다.

- **passport-jwt:** 들어오는 요청에서 JSON Web Token을 추출하고 검증하여 API 엔드포인트를 보호합니다.

- **passport-google-oauth20:** 사용자가 기존 Google 계정으로 즉시 로그인할 수 있게 합니다.

- **passport-facebook:** OAuth 2.0을 통해 Facebook 계정으로 로그인합니다.

- **passport-github2:** 사용자가 GitHub 계정으로 즉시 로그인할 수 있게 합니다.

인증을 개별 전략으로 분리함으로써 Passport는 애플리케이션이 원활하게 성장하도록 돕습니다. 오늘 사용자 이름/비밀번호 시스템으로 시작하고 다음 달에 "Google로 로그인"을 추가해도 전체 인증 코드를 다시 작성할 필요가 없습니다. Google 전략만 설치해 Passport에 연결하면 됩니다.

Express 애플리케이션에서 Passport를 사용하려면 핵심 Passport 패키지와 사용할 특정 전략을 함께 설치해야 합니다. 표준 사용자 이름과 비밀번호 설정의 경우 터미널에서 다음 명령어를 실행하세요:

```bash
npm install passport passport-local
```

설치가 완료되면 Passport를 Express 애플리케이션 설정에 연결해야 합니다. 이 작업은 라우트를 정의하기 전에 이루어져야 합니다:

```js
const express = require('express');
const passport = require('passport');
const LocalStrategy = require('passport-local').Strategy;

const app = express();

// initialize Passport middleware
app.use(passport.initialize());

// configure Passport to use the Local Strategy
passport.use(new LocalStrategy((username, password, done) => {
    // your database look-up and password verification logic goes here
}));
```

Passport는 미들웨어로서 표준 Express 라우팅 시스템에 원활하게 통합됩니다. 사용자가 로그인 라우트에 자격 증명을 제출하면 Passport가 요청을 가로채 구성한 전략에 데이터를 전달하고 결과를 평가합니다.

```js
app.post('/login', passport.authenticate('local'), (req, res) => {
  // if this function runs, then authentication was successful
  res.send(`Welcome back, ${req.user.username}!`);
});
```

전략이 사용자를 성공적으로 검증하면 Passport는 두 가지 중요한 작업을 자동으로 처리합니다:

1. **세션 관리:** 세션 쿠키 안에 사용자 데이터의 작은 조각을 저장하는 도우미 함수를 실행합니다.

2. **요청 장식:** 인증된 사용자 객체를 `req.user`로서 들어오는 요청에 직접 첨부합니다.

`req.user`가 채워지면 애플리케이션의 이후 미들웨어나 라우트 핸들러가 즉시 로그인한 사용자를 확인하고 그 신원을 기반으로 권한 결정을 내릴 수 있습니다.

Passport는 Node.js 애플리케이션에서 사용자 신원을 관리하기 위한 일관되고 안전하며 프로덕션 준비가 된 기반을 제공합니다.
