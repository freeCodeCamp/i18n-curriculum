# --description--

SQL 代表结构化查询语言。它诞生于 1970 年代，最初被称为结构化英文查询语言（SEQUEL）。后来该术语被缩写为 SQL。

SQL 是一种用于在关系数据库中存储和管理数据的编程语言。

正如你在之前的课程中学到的，关系数据库将数据组织到表格中。每个表格都有一组行和列，其中表现数据属性及其关系。每行表现一个记录，而每列表现该记录的特定属性或字段。

使用 SQL，你可以添加、更改、删除、查找和检索关系数据库中的数据。它提供了一整套用于查询、过滤、排序和聚合数据的命令。

SQL 可以通过仅查询必要的数据来帮助优化数据库性能，并且它可以轻松地与各种编程语言集成，因此你可以使用它直接从你的应用与数据库交互。

它也已被采纳为行业标准。它迫使数据完整性并包含许多安全性特色，例如支持用户认证和数据加密。

它还具有可扩展性、可移植性，并且兼容广泛的数据库管理系统。

SQL 基于命令，通常称为 SQL 语句或 SQL 查询。

作为开发者，你将使用特定的 SQL 语言元素或关键字编写这些语句。它们允许你对你的数据库执行必要的操作。

在你输入 SQL 命令之前，你需要打开 `psql` shell，这样你才能直接与 PostgreSQL 数据库交互。在命令提示符或终端中输入：

```bash
psql -U <username> -d <database_name>
```

将 `username` 替换为你的用户名，将 `database_name` 替换为你想连接的数据库。

如果你还没有创建你自己的数据库，PostgreSQL 在安装时会自动包含一个名为 `postgres` 的默认数据库。你可以使用这个 `postgres` 数据库进行操作。

连接后，你将看到提示符变为：

```bash
postgres=#
```

它显示你连接的数据库名称并等待 SQL 命令。

让我们回顾一些你应该熟悉的最基本的 SQL 命令。你会注意到，按照惯例，SQL 命令以分号（`;`）结尾。

首先，你可以使用此命令创建一个名为 `my_database` 的数据库：

```sql
CREATE DATABASE my_database;
```

你可以输入 `CREATE`，后跟 `DATABASE`，以及你想要创建的数据库名称。请注意，命令的 SQL 关键字是用大写字母书写的。

然后，一旦你有了你的数据库，你就可以连接到它。

你如何连接到数据库实际上取决于你使用的环境和工具。

在命令提示符或终端中，你可以使用前面展示的方法。在交互式 `psql` shell 中，你可以使用 `\c` 快捷命令加上数据库名称来切换数据库，格式如下：

```bash
\c my_database
```

请注意，` \c` 命令和其他快捷命令（前面带有斜线的命令）是 psql shell 的一部分，而不是 SQL 语言本身的一部分，因此它们不需要用分号来结束命令。

现在你已连接到你的新建数据库，你将看到提示符已更改：

```bash
my_database=#
```

一旦你创建并连接到一个数据库，你就可以使用 `CREATE TABLE` 关键字创建一个表格：

```sql
CREATE TABLE products (
  id SERIAL,
  name VARCHAR(255)
);
```

在命令中，你编写 `CREATE TABLE`，后面跟着你想要创建的表格名称。在这个分支中，它将创建一个名为 `products` 的表格。

然后，在括号内，写出表格应包含的列名，并指定将存储在每个列中的值的数据类型。

在示例中，表格将有两列，一列用于产品 ID，另一列用于产品名称。

表格和列名称的标准命名约定是蛇形命名法，使用小写字母书写单词，并用下划线（`_`）分隔。例如：`delivery_orders`

这些是你应该知道的几个入门必备的 SQL 命令。

使用 `SQL`，你可以轻松地查询、操作和分析数据，以做出明智的决策并解决现实问题。
