# --description--

Fetch API дозволяє веб-застосункам робити мережеві запити, зазвичай для отримання або відправлення даних на сервер. Цей API надає `fetch()` метод, який ви можете використовувати для виконання цих запитів. Давайте розглянемо базовий приклад використання `fetch`:

```js
fetch('https://api.example.com/data')
```

У цьому прикладі ми робимо `GET` запит до `https://api.example.com/data`. Це поверне нам деякі дані, які потрібно конвертувати у формат JSON і можна використовувати будь-де.

За замовчуванням Fetch API використовує метод `GET` для отримання даних. Це буде розглянуто в наступному уроці разом з іншими загальними HTTP методами запитів.

Тепер обговоримо деякі загальні типи ресурсів, які отримують із мережі.

У наших веб-застосунках нам потрібні деякі загальні дані, як-от дані про погоду, список професій, список назв країн, коди країн або іконки/зображення прапорців країн. Використовуючи ці дані, ми можемо зробити наш застосунок більш інформативним і інтерактивним. Завдяки Fetch API ми можемо отримувати ці ресурси з мережі.

Зображення — це одні з найчастіше отримуваних ресурсів. Ви можете використовувати fetch для завантаження зображень статично або динамічно залежно від дій користувача і відображати їх у своєму веб-застосунку.

Текстові файли — ще один тип ресурсів, які часто отримують. Це можуть бути конфігураційні файли, журнали або навіть цілі документи, які ви хочете показати на своїй веб-сторінці.

В деяких випадках ви можете отримувати аудіо- або відеофайли. Fetch API також може працювати з такими типами ресурсів, що дозволяє вам працювати з широким спектром типів даних.
