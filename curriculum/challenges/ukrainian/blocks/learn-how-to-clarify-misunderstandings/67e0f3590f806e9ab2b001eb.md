# --explanation--

`Sit in on` означає відвідувати щось (наприклад, заняття чи зустріч) без активної участі. Наприклад:

`She asked if she could sit in on our design session.` – Вона хотіла спостерігати за сесією, але не вести її і не багато говорити.

`Code review` — це процес, коли розробники перевіряють код один одного на помилки та можливості для покращення. Наприклад:

`We found a small bug during the code review.` – Команда перевірила код і помітила проблему.
