# --description--

Perché alcuni siti web sembrano reattivi e veloci, mentre altri appaiono lenti? La risposta sta in concetti chiave delle prestazioni che influenzano come una pagina viene caricata e renderizzata. Comprendere i concetti fondamentali delle prestazioni web è essenziale per costruire siti veloci, fluidi e facili da usare.

Analizziamo l’ordine di sorgente, il percorso critico di rendering, la latenza e altro ancora.

Iniziamo dall’ordine di sorgente.

L’ordine di sorgente si riferisce al modo in cui gli elementi HTML sono strutturati nel documento. Questo determina cosa viene caricato per primo e può influenzare significativamente le prestazioni e l’accessibilità.

Alcune buone pratiche includono:

- Posizionare contenuti critici come intestazioni, navigazione o testo principale più in alto nella struttura HTML.

- Rinviare gli script non essenziali, come quelli per l’analisi o widget di terze parti, in modo che non blocchino il rendering.

- Usare il progressive enhancement, per garantire che l’esperienza base funzioni anche prima che stili e script vengano caricati. Il progressive enhancement è un modo di costruire siti e applicazioni basato sull’idea che la pagina debba funzionare prima con solo HTML.

Ecco un esempio di buon ordine di sorgente, usando le buone pratiche appena descritte.

```html
<!-- Good source order: Essential content first -->
<h1>Welcome to FastSite!</h1>
<p>Critical information loads first.</p>
<script src="slow-script.js" defer></script>
```

Ottimizzando l’ordine di sorgente, facciamo in modo che gli utenti vedano i contenuti importanti il prima possibile.

Ora vediamo il percorso critico di rendering.

Il percorso critico di rendering è la sequenza di passaggi che il browser segue per trasformare il codice in pixel sullo schermo.

Ecco i passaggi chiave che approfondiremo più avanti:

1. Parsing HTML: costruisce il DOM (Document Object Model)

2. Parsing CSS: costruisce il CSSOM (CSS Object Model)

3. Esecuzione JavaScript: può modificare il DOM e il CSSOM

4. Costruzione dell’albero di rendering: combina DOM e CSSOM

5. Layout e pittura: determina le dimensioni degli elementi e disegna i pixel

6. Ottimizzazioni:

    - Minimizzare le risorse che bloccano il rendering (es. grandi file CSS, JS inutilizzati).

    - Usare gli attributi async e defer per gli script.

    - Caricare prima solo gli stili essenziali; rinviare il CSS non critico.

In generale, un percorso critico di rendering più breve significa prestazioni percepite più veloci. Ne parleremo più nel dettaglio più avanti.

Infine, vediamo la latenza.

La latenza è il tempo che impiega una richiesta a viaggiare tra il browser e il server. In altre parole, alta latenza significa pagine lente.

Alcuni modi per ridurre la latenza includono:

- Usare CDN, cioè Content Delivery Network, per servire i file da posizioni più vicine.

- Abilitare la compressione con strumenti come Gzip per ridurre la dimensione dei file.

- Ottimizzare le immagini e usare il lazy loading — di cui parleremo anche più avanti.

Riducendo la latenza, rendiamo le interazioni più immediate.

In conclusione, ottimizzando l’ordine di sorgente, riducendo il percorso critico e abbattendo la latenza, puoi far sembrare il tuo sito veloce.
