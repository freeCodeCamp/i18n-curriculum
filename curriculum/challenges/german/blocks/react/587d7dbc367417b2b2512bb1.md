# --description--

React ist eine Open Source View-Bibliothek, die von Facebook entwickelt und gepflegt wird. Sie ist ein großartiges Werkzeug, um die Benutzeroberfläche (UI) moderner Webanwendungen zu rendern.

React verwendet eine Syntaxerweiterung von JavaScript namens JSX, die es Ihnen erlaubt, HTML direkt innerhalb von JavaScript zu schreiben. Das hat mehrere Vorteile. Es ermöglicht Ihnen, die volle programmatische Kraft von JavaScript innerhalb von HTML zu nutzen und hilft dabei, Ihren Code lesbar zu halten. Größtenteils ähnelt JSX dem HTML, das Sie bereits gelernt haben, jedoch gibt es einige wichtige Unterschiede, die in diesen Aufgaben behandelt werden.

Zum Beispiel können Sie, da JSX eine syntaktische Erweiterung von JavaScript ist, tatsächlich JavaScript direkt innerhalb von JSX schreiben. Dazu fügen Sie einfach den Code, der als JavaScript behandelt werden soll, in geschweifte Klammern ein: `{ 'this is treated as JavaScript code' }`. Behalten Sie das im Hinterkopf, da es in mehreren zukünftigen Aufgaben verwendet wird.

Da JSX jedoch kein gültiges JavaScript ist, muss JSX-Code in JavaScript kompiliert werden. Der Transpiler Babel ist ein beliebtes Werkzeug für diesen Prozess. Zum Komfort ist er für diese Aufgaben bereits im Hintergrund hinzugefügt. Wenn Sie syntaktisch ungültiges JSX schreiben, wird der erste Test in diesen Aufgaben fehlschlagen.

Es ist erwähnenswert, dass unter der Haube die Aufgaben `ReactDOM.render(JSX, document.getElementById('root'))` aufrufen. Dieser Funktionsaufruf platziert Ihr JSX in Reacts eigene, leichtgewichtige Darstellung des DOM. React verwendet dann Schnappschüsse seines eigenen DOM, um die Aktualisierung nur bestimmter Teile des tatsächlichen DOM zu optimieren.

# --instructions--

Der aktuelle Code verwendet JSX, um ein `div`-Element der Konstanten `JSX` zuzuweisen. Ersetzen Sie das `div` durch ein `h1`-Element und fügen Sie den Text `Hello JSX!` darin ein.
