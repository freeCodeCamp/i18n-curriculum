# --description--

У попередніх уроках ви дізналися, як працювати з примітивними типами, такими як `string` і `boolean`. Але TypeScript дозволяє створювати нові типи на основі існуючих. Одним із таких прикладів є об’єднаний тип.

Об’єднаний тип — це тип, який поєднує два або більше інших типів.

Ось базовий синтаксис:

```ts
type1 | type2 | type3
```

Давайте розглянемо кілька прикладів, щоб краще зрозуміти цю концепцію.

Ось змінна з назвою `id` з об’єднаним типом для `string` або `number`:

```ts
let id: string | number = "user_783";
```

Значення можна оновити на число, що також є дійсним для об’єднаного типу:

```ts
let id: string | number = 1024;
```

Якщо ви спробуєте призначити інше значення, яке не є `string` або `number`, TypeScript покаже повідомлення про помилку:

```ts
// Type 'string[]' is not assignable to type 'string | number'.
let id: string | number = ["Java", "Python"];
```

При роботі з об’єднаними типами важливо використовувати методи, які працюють для всіх типів, інакше TypeScript видасть помилку такого виду:

```ts
function getId(id: string | number) {
  // Property 'toUpperCase' does not exist on type 'string | number'.
  return id.toUpperCase();
}
```

Існує рішення, яке полягає у звуженні об’єднання. Але ви дізнаєтеся про це більше, а також про те, як об’єднані типи працюють з псевдонімами типів, у майбутніх уроках.

Об’єднані типи також корисні при роботі з `null` і `undefined` значеннями.

Ось приклад, де змінна може містити значення `string`, або це може бути `null`, якщо значення ще не встановлено:

```ts
let username: string | null;

// Type 'number' is not assignable to type 'string | null'.
username = 42;
```

У цьому випадку `username` може бути або `string`, або `null`, але нічим іншим. Ви можете використовувати `null`, коли хочете навмисно представити відсутність значення.

Ось ще один приклад із використанням `undefined`:

```ts
let score: number | undefined;

score = undefined; 

// Type 'string' is not assignable to type 'number | undefined'.
score = "high";
```

У цьому прикладі `score` може бути або `number`, або `undefined`. Ви можете використовувати `undefined`, коли значення ще не призначене.

Використання об’єднаних типів із `null` і `undefined` допомагає чітко показати, коли значення може бути відсутнім, одночасно дозволяючи TypeScript виявляти недійсні присвоєння.
