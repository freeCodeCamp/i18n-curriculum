# --description--

성능에 대해 이야기할 때, 그 자체가 다소 주관적으로 느껴질 수 있습니다. 좋은 성능이란 무엇일까요? 나쁜 성능은 무엇일까요?

성능을 측정하는 표준 방법이 있나요, 아니면 전적으로 사용자의 개인적인 선호와 인식에 달려 있나요? 한번 살펴보겠습니다.

웹사이트의 성능은 체감 성능과 실제 성능 두 가지 범주로 나눌 수 있습니다.

체감 성능은 사용자가 웹사이트의 성능을 어떻게 인식하는지를 말합니다. 반응성이나 신뢰성 측면에서 어떻게 평가하는지입니다. 이는 주관적인 측정이기 때문에 수치화하기 어렵지만, 사용자 경험이 웹사이트의 성공과 실패를 결정하므로 매우 중요합니다.

반면 실제 성능은 웹사이트의 객관적이고 측정 가능한 성능입니다. 페이지 로드 시간, 서버 응답 시간, 렌더링 시간 같은 지표로 측정합니다. 이러한 측정값은 네트워크와 코드 자체와 관련된 여러 요인의 영향을 받습니다.

체감 성능이 뛰어난 웹사이트라도 더 나은 사용자 경험을 위해 추가 최적화할 수 있습니다.

체감 성능을 개선하는 여러 기법이 있습니다.

비필수 리소스를 백그라운드에서 로드하여 초기 로드 시간을 최대한 줄이는 것이 필수적입니다. 이 기법을 "지연 로딩(lazy loading)"이라고 합니다. 첫인상이 사용자가 웹사이트에 대해 갖는 인상을 결정할 수 있습니다.

지연 로딩은 웹사이트의 실제 성능에도 영향을 미칩니다. 이 기법을 사용하면 처음에 로드해야 하는 콘텐츠 양을 줄여 페이지가 더 빨리 로드되므로, 이는 실제 성능 지표에 해당합니다.

사용자 상호작용에 빠른 응답과 피드백을 제공하는 것도 매우 중요합니다.

예를 들어, 사용자가 요소를 클릭하자마자 긴 작업에 대해 로딩 표시기를 보여주면 사용자가 프로세스에 연결되어 있다고 느끼고 참여도가 높아져 대기 시간이 더 짧게 느껴지거나 적어도 더 몰입하게 됩니다.

사용자가 웹사이트에 적극적으로 참여하도록 유지하는 것은 시간 인식을 개선하는 데 매우 중요합니다. 텍스트가 도착하는 즉시 표시하면 이미지 같은 다른 리소스가 아직 로드되지 않았더라도 처음부터 사용자의 참여를 유지하는 데 도움이 됩니다.

사용자가 긴 작업 완료를 기다리는 경우 진행 상황을 알려주고 정기적으로 업데이트하세요.

또 다른 팁은 콘텐츠 재배치와 점프 현상을 피하는 것입니다. 예를 들어 광고나 이미지가 로드될 때, 레이아웃에 공간이 미리 할당되어 있지 않으면 웹사이트가 점프하거나 재조정될 수 있습니다.

이러한 갑작스러운 변화는 사용자가 웹사이트가 아직 로드 중이라고 느껴 나쁜 사용자 경험을 초래할 수 있습니다. 이를 피하려면 처음부터 이러한 요소에 공간을 할당해 두세요.

웹사이트에 사용자 지정 폰트가 있다면 폰트 로딩 지연을 최소화해야 합니다. 그렇지 않으면 깜빡임 현상이나 사용자 지정 폰트가 로드되는 동안 대체 폰트가 표시될 수 있습니다. 이때 사용자 지정 폰트와 비슷한 대체 폰트를 사용하는 것이 좋습니다. 이렇게 하면 변경이 훨씬 더 미묘하게 느껴집니다.

또한 인터페이스 요소가 활성 상태인지 확인하세요. 사용자가 최소한의 지연으로 상호작용할 수 있어야 합니다.

다음 수업에서 이러한 기법을 더 자세히 다루겠습니다.

실제 성능도 중요하지만, 체감 성능은 사용자 경험에 엄청난 영향을 미칠 수 있습니다. 두 가지를 모두 최적화하면 더 빠르고 부드러우며 몰입감 있는 사용자 경험을 제공하는 웹사이트를 만들 수 있습니다.
