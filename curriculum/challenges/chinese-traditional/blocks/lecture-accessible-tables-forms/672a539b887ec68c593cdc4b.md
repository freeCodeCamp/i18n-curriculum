# --interactive--

當我們看到表格時，我們會立即開始在資料和標題之間建立視覺聯想。

例如，假設我們有一個表格表述我們的寵物。我們有兩隻狗和兩隻貓，表格正在顯示牠們的名字和年齡。雖然有視力的人可能能理解這個表格中的關係，但對於使用螢幕閱讀器來瀏覽表格的人來說，將數值和標題之間建立連接要困難得多。

作為一名網頁開發者，你負責建立這些關聯並以易於螢幕閱讀器使用者理解的方式結構化你的 HTML 標記。

那麼，讓我們看看你如何創建每個人都能理解的可存取表格。我們將涵蓋的第一個最佳實踐是使用表格標題。使用 `caption` 元素，你可以撰寫表格的標題（或名稱），讓使用者，特別是使用輔助技術的使用者，能夠快速了解表格的目的和內容。你應該將 `caption` 元素放置在 `table` 元素的開啟標籤之後。這樣，螢幕閱讀器和其他輔助技術就能在讀取內容之前，透過宣告標題來提供更多的背景關係。

```html
<table>
  <caption>Our Pets</caption>
  <!-- Table Rows and Columns -->
</table>
```

現在讓我們來談談列和行標題。標題是特殊的儲存格，通常位於列或行的開頭，用來描述該列或行中儲存的資料型別。你可以使用表格標題元素 `th` 來定義列或行標題。

例如，下面的程式碼創建了一個包含兩隻寵物的表格。每一列都有一個列標題（寵物的名字），每一行都有一個行標題，該行標題描述該行資料所表現的內容（年齡和型別）。

:::interactive_editor

```html
<table>
  <caption>Our Pets</caption>
  <thead>
    <tr>
      <!-- Column Headers -->
      <th>Name</th>
      <th>Age</th>
      <th>Type</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <th>Nora</th> <!-- Row Header -->
      <td>5</td>
      <td>Dog</td>
    </tr>
    <tr>
      <th>Gino</th> <!-- Row Header -->
      <td>2</td>
      <td>Cat</td>
    </tr>
  </tbody>
</table>
```

:::

請注意，上述程式碼在開啟的 `table` 元素後緊接著有一個 `caption` 元素。接著，在表格標題 (`thead`) 元素內，有行標題（`Name`、`Age` 和 `Type`）。在第二列和第三列中，在表格主體 (`tbody`) 元素內，我們找到每隻寵物的資料。寵物的名稱是列標題，因為它們位於表格標題元素 (`th`) 內。

將資料儲存格與其對應的標題關聯起來對螢幕閱讀器也非常重要。`scope` 屬性決定標題是列標題還是行標題。螢幕閱讀器可能會從表格的結構中正確猜測，但通常建議明確指出 `scope` 以確保清晰。

`scope` 屬性有四種可能的值。你最常使用的兩個是 `col` 代表行和 `row` 代表列。  
在下面的程式碼中，你可以看到我們將 `scope` 屬性添加到行和列標題。三個行標題（`Name`、`Age` 和 `Type`）的 `scope` 是 `col`，代表行。

這兩個列標題（`Nora` 和 `Gino`）的 `scope` 是 `row`。

:::interactive_editor

```html
<table>
  <caption>Our Pets</caption>
  <thead>
    <tr>
      <!-- Now they have scope -->
      <th scope="col">Name</th>
      <th scope="col">Age</th>
      <th scope="col">Type</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <th scope="row">Nora</th>
      <td>5</td>
      <td>Dog</td>
    </tr>
    <tr>
      <th scope="row">Gino</th>
      <td>2</td>
      <td>Cat</td>
    </tr>
  </tbody>
</table>
```

:::

如果一個行或列標題跨越多個儲存格，`scope` 也會個別套用到每一個儲存格。以下是一個範例：

:::interactive_editor

```html
<table>
  <tbody>
    <tr>
      <td></td>
      <th scope="col">Name</th>
      <th scope="col">Age</th>
    </tr>
    <tr>
      <th rowspan="2" scope="row">Dogs</th>
      <th scope="row">Nora</th>
      <td>5</td>
    </tr>
    <tr>
      <th scope="row">Gino</th>
      <td>2</td>
    </tr>
    <tr>
      <th rowspan="2" scope="row">Cats</th>
      <th scope="row">Lulu</th>
      <td>10</td>
    </tr>
    <tr>
      <th scope="row">Elizabeth</th>
      <td>6</td>
    </tr>
  </tbody>
</table>
```

:::

在此表格中，`Nora` 的年齡（`5`）將有一個行標題（`Age`）和兩個列標題（`Dogs` 和 `Nora`）。`Gino` 的年齡（`2`）也將有一個行標題（`Age`）和兩個列標題（`Dogs` 和 `Gino`）。

然而，有些螢幕閱讀器可能無法解析具有複雜結構的表格，因此你也應該盡可能將表格扁平化，以避免跨越多個儲存格的列和行標題。

你的目標應該始終是確保使用者能夠存取這些資訊，即使他們的螢幕閱讀器能控制代碼複雜的表格結構。

現在，對於儲存格寬度，建議避免使用固定值。你應該使用相對值，例如百分比。此外，盡量避免定義儲存格高度。這將允許使用者調整文字大小以符合他們的需求。

最後，應該盡可能讓瀏覽器決定表格寬度，以減少水平捲動的需求。

HTML 表格對於以可存取且易於理解的格式呈現結構化資料至關重要。遵循這些無障礙指引，你可以創建所有人都容易理解的表格。
