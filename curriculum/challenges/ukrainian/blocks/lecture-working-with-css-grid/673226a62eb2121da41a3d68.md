# --interactive--

Функція `minmax()` визначає діапазон розміру треку сітки, вказуючи, скільки простору може займати рядок або стовпець.

Пам’ятайте, що ви можете встановити розмір треку за допомогою одиниць, таких як `px` (пікселі), `rem` або навіть `em`, а також дробових одиниць (`fr`).

Функція `minmax()` йде далі, дозволяючи встановити мінімальний і максимальний розмір треку сітки.

Ось синтаксис функції `minmax()`:

```css
minmax(min, max)
```

`min` — це мінімальний розмір треку сітки, який можна задати у пікселях, відсотках або `auto`. А `max` — максимальний розмір треку сітки, який ви можете встановити тими ж одиницями.

Ці два значення працюють разом так:

- Значення `min` гарантує, що трек сітки ніколи не зменшиться менше встановленого розміру.

- Значення `max` обмежує, наскільки великим може стати трек сітки.

Розмір треку сітки динамічно регулюється між значеннями `min` і `max` залежно від вмісту та розміру контейнера.

Розглянемо практичний приклад:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<div class="grid-container">
  <div>
    <h2>Item 1</h2>
  </div>
  <div>
    <h2>Item 2</h2>
  </div>
</div>
```

```css
.grid-container {
  display: grid;
  grid-template-columns: minmax(150px, 300px) 1fr;
  gap: 20px;
}

.grid-container > div {
  background: crimson;
  padding: 20px;
  text-align: center;
}
```

:::

Що тут відбувається?

Перший стовпець, `minmax(150px, 300px)`, завжди буде мати мінімум `150px` і максимум `300px`, залежно від доступного простору.

З іншого боку, другий стовпець, `1fr`, займе весь доступний залишковий простір у контейнері сітки, оскільки немає інших стовпців, з якими можна було б поділити простір.

Перевага функції `minmax()` над фіксованими розмірами і навіть одиницями `fr` у тому, що вона більш гнучка, що робить її ідеальною для адаптивності та відзивчивості.
