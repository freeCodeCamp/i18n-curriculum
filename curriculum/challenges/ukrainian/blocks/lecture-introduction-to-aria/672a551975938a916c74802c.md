# --interactive--

Атрибут `aria-describedby` використовується для надання додаткової інформації про елемент користувачам читачів екрана шляхом посилання на існуючий вміст на сторінці. Він створює програмний зв’язок між елементом і вмістом (технічно це називається доступним описом), який читачі екрана можуть використовувати, щоб повідомити користувачам додаткову інформацію під час взаємодії з елементом.

Найпоширеніше використання `aria-describedby` — це пов’язати інструкції та повідомлення про помилки з вхідними полями форми. Через різні способи навігації користувачів читачів екрана по сторінці вони можуть пропустити ці повідомлення під час переходу між полями. Використання `aria-describedby` допомагає гарантувати, що вони їх почують.

Давайте розглянемо кілька прикладів, щоб зрозуміти, як це працює. У цьому першому прикладі у нас є `form` елемент, який приймає пароль.

Увімкніть інтерактивний редактор і введіть кілька символів у поле введення пароля. Ви побачите, що пароль приховано у вікні попереднього перегляду. Ви також повинні побачити, що текст `password-help` залишається червоним, доки ви не введете 8 або більше символів у поле.

ПРИМІТКА: Цей інтерактивний приклад використовує CSS і JavaScript для динамічного оновлення кольору тексту `password-help`. Не хвилюйтеся, якщо не розумієте JavaScript-код, бо ви вивчите JavaScript у майбутніх модулях.

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<form>
  <label for="password">Password:</label>
  <input type="password" id="password" aria-describedby="password-help" />
  <p id="password-help">Your password must be at least 8 characters long.</p>
</form>

<script src="index.js"></script>
```

```css
#password-help {
  color: red;
}
```

```js
const passwordEl = document.getElementById("password");
const passwordHelpText = document.getElementById("password-help");

passwordEl.addEventListener("input", (e) => { 
  const userInput = e.target.value;
  passwordHelpText.style.color = userInput.length >= 8 ? "green" : "red";
});
```

:::

Ми використовуємо `label` елемент для тексту `Password` і пов’язуємо його з полем введення пароля `input`.

Також у нас є абзац, який описує вимоги до пароля. Ми використовуємо атрибут `aria-describedby`, щоб пов’язати поле введення пароля `input` з вимогами до пароля в абзаці. Коли користувач читача екрана взаємодіє з цим `input`, його читач екрана оголосить назву `input` (`Password`) і може також оголосити вимоги до пароля. Однак це не абсолютна гарантія, оскільки деякі читачі екрана можуть не оголошувати додатковий вміст автоматично або робити це лише в певних випадках. Це не рідкість. Кожен читач екрана працює по-своєму з ARIA-атрибутами. Це не скасовує використання `aria-describedby`, оскільки це загалом допомагає користувачам читачів екрана.

Ще один хороший випадок використання атрибута `aria-describedby` — коли у вас є кнопка видалення `button`. Ось приклад кнопки видалення `button`, за якою йде повідомлення, що описує, що станеться після натискання на кнопку:

:::interactive_editor

```html
<button aria-describedby="delete-message">Delete</button>

<p id="delete-message">Warning! All deletions are permanent.</p>
```

:::

Як і в попередньому прикладі, ми пов’язуємо кнопку видалення з повідомленням за допомогою атрибута `aria-describedby`. Значення `id` і значення атрибута `aria-describedby` мають співпадати.

Атрибут `aria-describedby` — це потужний атрибут, який допомагає забезпечити надання додаткової інформації про елемент користувачам читачів екрана під час взаємодії з елементом. Найчастіше його використовують для пов’язання інструкцій і повідомлень про помилки з вхідними полями форми, щоб зменшити ймовірність того, що користувачі читачів екрана пропустять ці повідомлення під час навігації формою.
