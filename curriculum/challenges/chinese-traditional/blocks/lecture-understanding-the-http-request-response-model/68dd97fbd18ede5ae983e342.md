# --description--

在上一課中，你學到伺服端會處理 HTTP 請求並回傳回應。該回應包含你在網頁上看到的各種資源。

HTTP 回應傳回的資產包括基於文字的資產、多媒體資產、字型，以及其他資產。這些資產中的每一個都帶有 **Content-Type** 標頭，告訴瀏覽器如何正確地處理程序和顯示該資產。

基於文字的資產一直是網路的基本零件。事實上，網路最初是純文字。如今，基於文字的資產包括以下內容：

* **HTML（超文字標記語言）**：任何網頁的基本結構。沒有 HTML，網頁無法存在。
    
* **CSS (Cascading Stylesheet)**：控制網頁外觀的資產。使用 CSS，你可以將顏色、字型、版面配置等樣式套用到 HTML。
    
* **JavaScript**：JavaScript 為 HTML 和 CSS 添加互動性和動態內容。
    
* **JSON 和 XML**：這兩種格式讓你在伺服器與瀏覽器之間傳輸結構化資料。
    
媒體資產包括影像、音訊和影片。它們以視覺和聲音讓網頁更生動。

許多網站依賴影像來傳達視覺資訊。影像有不同的格式，如 `JPG`、`JPEG`、`PNG`、`WEBP` 和 `GIF`。

像 YouTube 和 Vimeo 這樣的網站提供 `MP4`、`MOV`、`AVI`、`WebM` 等格式的影片內容。

音訊是許多平台的重要零件，因為它讓使用者能串流音樂、播客和其他聲音內容。常見的音訊格式包括 `MP3`、`AAC`、Ogg 和 `WAV`。像 Spotify 和 YouTube Music 這樣的平台依賴這些格式來提供高品質的音訊體驗。

其他在 HTTP 回應中傳回的資產包括像 `PDF`、`DOCX` 和 `PAGES` 這類文件、像 `ZIP` 和 `RAR` 這類壓縮檔案、像 `WOFF`、`WOFF2` 和 `TTF` 這類字型檔案，以及像 `EXE` 這類可執行檔。

以下是這些常見資產及其內容型別的表格：

| Asset Type | Content-Type Header |
| --- | --- |
| HTML | text/html |
| CSS | text/css |
| JavaScript | application/javascript |
| JPEG | image/jpeg |
| PNG | image/png |
| GIF | image/gif |
| SVG | image/svg+xml |
| WebP | image/webp |
| WOFF | font/woff |
| WOFF2 | font/woff2 |
| TTF | font/ttf |
| OTF | font/otf |
| MP4 | video/mp4 |
| WebM | video/webm |
| Ogg (Video) | video/ogg |
| MP3 | audio/mpeg |
| AAC | audio/aac |
| WAV | audio/wav |
| Ogg (Audio) | audio/ogg |
| JSON | application/json |
| XML | application/xml |
| PDF | application/pdf |
| ZIP | application/zip |
