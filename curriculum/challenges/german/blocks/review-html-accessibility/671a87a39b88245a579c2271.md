## Einführung in die Barrierefreiheit

- **Web Content Accessibility Guidelines**: Die Web Content Accessibility Guidelines (WCAG) sind eine Reihe von Richtlinien, um Webinhalte für Menschen mit Behinderungen zugänglicher zu machen. Die vier Grundsätze der WCAG sind POUR, was für Wahrnehmbar, Bedienbar, Verständlich und Robust steht.

## Hilfstechnologien für Barrierefreiheit

- **Screenreader**: Softwareprogramme, die den Inhalt eines Computerbildschirms laut vorlesen. Sie werden von Menschen genutzt, die blind oder sehbeeinträchtigt sind, um auf das Web zuzugreifen.
- **Große Tastaturen oder Braille-Tastaturen**: Werden von Menschen mit Sehbeeinträchtigungen verwendet, um auf das Web zuzugreifen.
- **Bildschirmlupen**: Softwareprogramme, die den Inhalt eines Computerbildschirms vergrößern. Sie werden von Menschen mit eingeschränktem Sehvermögen genutzt, um auf das Web zuzugreifen.
- **Alternative Zeigegeräte**: Werden von Menschen mit Mobilitätseinschränkungen verwendet, um auf das Web zuzugreifen. Dazu gehören Geräte wie Joysticks, Trackballs und Touchpads.
- **Spracherkennung**: Wird von Menschen mit Mobilitätseinschränkungen genutzt, um auf das Web zuzugreifen. Sie ermöglicht es Nutzern, einen Computer mit ihrer Stimme zu steuern.

## Werkzeuge zur Überprüfung der Barrierefreiheit

- **Gängige Barrierefreiheitswerkzeuge**: Google Lighthouse, Wave, IBM Equal Accessibility Checker und axe DevTools sind einige der gängigen Werkzeuge, die verwendet werden, um die Barrierefreiheit einer Webseite zu überprüfen.

## Bewährte Vorgehensweisen für Barrierefreiheit

- **Richtige Struktur der Überschriftenebenen**: Sie sollten richtige Überschriftenebenen verwenden, um eine logische Struktur für Ihre Inhalte zu schaffen. Das hilft Menschen, die Hilfstechnologien nutzen, den Inhalt Ihrer Webseite besser zu verstehen.
- **Barrierefreiheit und Tabellen**: Beim Verwenden von Tabellen sollten Sie das `th`-Element verwenden, um Kopfzellen zu definieren, und das `td`-Element, um Datenzellen zu definieren. Das hilft Menschen, die Hilfstechnologien nutzen, die Struktur der Tabelle zu verstehen. Mit dem `caption`-Element können Sie die Bildunterschrift (oder den Titel) einer Tabelle schreiben, sodass Nutzer, insbesondere solche mit Hilfstechnologien, schnell den Zweck und Inhalt der Tabelle erfassen können. Sie sollten das `caption`-Element unmittelbar nach dem öffnenden Tag des `table`-Elements platzieren. So können Screenreader und andere Hilfstechnologien mehr Kontext bieten, indem sie die Bildunterschrift vor dem Vorlesen des Inhalts ankündigen.
- **Wichtigkeit, dass Eingaben eine zugeordnete Bezeichnung haben**: Sie sollten das `label`-Element verwenden, um Bezeichnungen mit Formulareingaben zu verknüpfen. Das hilft Menschen, die Hilfstechnologien nutzen, den Zweck der Eingabe zu verstehen.
- **Wichtigkeit guter `alt`-Texte**: Sie sollten das `alt`-Attribut verwenden, um eine Textalternative für Bilder bereitzustellen. Das hilft Menschen, die Hilfstechnologien nutzen, den Inhalt des Bildes zu verstehen.
- **Wichtigkeit guter Linktexte**: Sie sollten beschreibende Linktexte verwenden, um Nutzern den Zweck des Links verständlich zu machen. Das hilft Menschen, die Hilfstechnologien nutzen, den Zweck des Links zu verstehen.
- **Bewährte Vorgehensweisen zur Barrierefreiheit von Audio- und Videoinhalten**: Sie sollten Bildunterschriften und Transkripte für Audio- und Videoinhalte bereitstellen, um sie für Menschen mit Hörbeeinträchtigungen zugänglich zu machen. Außerdem sollten Sie Audiodeskriptionen für Videoinhalte bereitstellen, um sie für Menschen mit Sehbeeinträchtigungen zugänglich zu machen.
- **`tabindex`-Attribut**: Wird verwendet, um Elemente fokussierbar zu machen und die relative Reihenfolge zu definieren, in der sie mit der Tastatur navigiert werden sollen. Es ist wichtig, niemals das `tabindex`-Attribut mit einem Wert größer als 0 zu verwenden. Stattdessen sollten Sie entweder den Wert 0 oder -1 verwenden.

:::interactive_editor

```html
<p tabindex="-1">Sorry, there was an error with your submission.</p>
```

:::

- **`accesskey`-Attribut**: Wird verwendet, um eine Tastenkombination für ein Element zu definieren. Das kann Nutzern mit Mobilitätseinschränkungen helfen, die Webseite leichter zu navigieren.

:::interactive_editor

```html
<button accesskey="s">Save</button>
<button accesskey="c">Cancel</button>
<a href="index.html" accesskey="h">Home</a>
```

:::

## WAI-ARIA, Rollen und Attribute

- **WAI-ARIA**: Steht für Web Accessibility Initiative – Accessible Rich Internet Applications. Es ist eine Sammlung von Attributen, die zu HTML-Elementen hinzugefügt werden können, um die Barrierefreiheit zu verbessern. Es liefert Hilfstechnologien zusätzliche Informationen über Zweck und Struktur des Inhalts.
- **ARIA-Rollen**: Eine Reihe vordefinierter Rollen, die zu HTML-Elementen hinzugefügt werden können, um deren Zweck zu definieren. Das hilft Menschen, die Hilfstechnologien nutzen, den Inhalt der Webseite zu verstehen. Beispiele sind `role="tab"`, `role="menu"` und `role="alert"`.

Es gibt sechs Hauptkategorien von ARIA-Rollen:

- **Rollen für Dokumentstruktur**: Diese Rollen definieren die Gesamtstruktur der Webseite. Mit diesen Rollen können Hilfstechnologien die Beziehungen zwischen verschiedenen Abschnitten verstehen und Nutzern helfen, den Inhalt zu navigieren.
- **Widget-Rollen**: Diese Rollen definieren Zweck und Funktionalität interaktiver Elemente, wie Scrollbalken.
- **Landmark-Rollen**: Diese Rollen klassifizieren und kennzeichnen die Hauptabschnitte einer Webseite. Screenreader verwenden sie, um eine bequeme Navigation zu wichtigen Abschnitten einer Seite zu ermöglichen.
- **Live-Region-Rollen**: Diese Rollen definieren Elemente mit dynamisch änderndem Inhalt. So können Screenreader und andere Hilfstechnologien Änderungen für Nutzer mit Sehbehinderungen ankündigen.
- **Fenster-Rollen**: Diese Rollen definieren Unterfenster, wie Pop-up-Modaldialoge. Zu diesen Rollen gehören `alertdialog` und `dialog`.
- **Abstrakte Rollen**: Diese Rollen helfen bei der Organisation des Dokuments. Sie sind nur für die interne Nutzung durch den Browser gedacht, nicht für Entwickler, daher sollten Sie wissen, dass sie existieren, sie aber nicht auf Ihren Webseiten oder Webanwendungen verwenden.

- **`aria-label`- und `aria-labelledby`-Attribute**: Diese Attribute werden verwendet, um einem Element einen programmatischen (oder zugänglichen) Namen zu geben, der Menschen, die Hilfstechnologien (wie Screenreader) nutzen, hilft, den Zweck des Elements zu verstehen. Sie werden oft verwendet, wenn die visuelle Bezeichnung eines Elements ein Bild oder Symbol statt Text ist. `aria-label` erlaubt es, den Namen direkt im Attribut zu definieren, während `aria-labelledby` es ermöglicht, auf vorhandenen Text auf der Seite zu verweisen.

```html
<button aria-label="Search">
    <i class="fas fa-search"></i>
</button>
```

:::interactive_editor

```html
<input type="text" aria-labelledby="search-btn">
<button type="button" id="search-btn">Search</button>
```

:::

- **`aria-hidden`-Attribut**: Wird verwendet, um ein Element für Hilfstechnologien wie Screenreader zu verbergen. Zum Beispiel kann dies verwendet werden, um dekorative Bilder auszublenden, die keinen sinnvollen Inhalt bieten.

:::interactive_editor

```html
<button>
    <i class="fa-solid fa-gear" aria-hidden="true"></i>
    <span class="label">Settings</span>
</button>
```

:::

- **`aria-describedby`-Attribut**: Wird verwendet, um zusätzliche Informationen über ein Element bereitzustellen, indem es mit einem anderen Element verknüpft wird, das diese Informationen enthält. Das gibt Menschen, die Screenreader nutzen, sofortigen Zugriff auf die zusätzlichen Informationen, wenn sie zum Element navigieren. Häufige Verwendung ist die Verknüpfung von Formatierungsanweisungen mit einer Texteingabe oder einer Fehlermeldung mit einer Eingabe nach einer ungültigen Formulareinsendung.

:::interactive_editor

```html
<form>
    <label for="password">Password:</label>
    <input type="password" id="password" aria-describedby="password-help" />
    <p id="password-help">Your password must be at least 8 characters long.</p>
</form>
```

:::
