# --description--

때때로 서로 충돌하는 규칙이 있어서 예상치 못한 결과가 나올 수 있습니다. “이 단락들은 파란색이었으면 하는데, 왜 다른 단락들처럼 빨간색일까요?!” 이렇게 답답할 수 있지만, CSS가 마음대로 동작하는 것이 아니라는 점을 이해하는 것이 중요합니다. CSS는 우리가 지시한 대로만 동작합니다. 예외는 브라우저가 제공하는 기본 스타일입니다. 이 기본 스타일은 브라우저마다 다르며, 어떤 요소가 다른 요소와 큰 “간격”을 만드는 이유나, 버튼이 우리가 CSS 규칙을 작성하지 않았는데도 특정 모양으로 보이는 이유가 바로 이것 때문입니다.

따라서 이런 예상치 못한 동작이 발생한다면, 기본 스타일 때문이거나, 속성이 어떻게 동작하는지 이해하지 못했거나, 캐스케이드라는 개념을 이해하지 못했기 때문일 수 있습니다.

캐스케이드는 어떤 규칙이 실제로 HTML에 적용될지를 결정합니다. 캐스케이드가 이를 결정하는 데 사용하는 여러 요소가 있습니다. 이 중 세 가지 요소를 살펴보면서, “CSS가 너무 싫어!”라는 답답한 순간을 피하는 데 도움이 되길 바랍니다.
