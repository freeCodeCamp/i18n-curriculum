# --interactive--

클로저는 JavaScript에서 가장 강력하면서도 자주 오해받는 기능 중 하나입니다. 본질적으로 클로저는 외부 함수가 반환된 후에도 외부 렉시컬 범위에 있는 변수에 접근할 수 있는 함수입니다. 복잡하게 들릴 수 있지만, JavaScript에서 많은 고급 프로그래밍 패턴을 가능하게 하는 기본 개념입니다.

클로저를 이해하려면 예제부터 시작해 보세요:

:::interactive_editor

```js
function outerFunction(x) {
    let y = 10;
    function innerFunction(){
        console.log(x + y);
    }
    return innerFunction;
}

let closure = outerFunction(5);
console.log(closure()); // 15
```

:::

이 예제에서 `outerFunction`는 매개변수 `x`를 받고 지역 변수 `y`를 정의합니다. 그런 다음 `innerFunction`와 `x`를 모두 사용하는 `y`를 정의합니다. 마지막으로 `innerFunction`를 반환합니다. `outerFunction(5)`를 호출하면 `innerFunction`를 반환하는데, 이를 변수 `closure`에 할당합니다. 이후 `closure()`를 호출해도 `x`가 이미 실행을 마쳤음에도 `y`의 `outerFunction`과 `outerFunction`에 여전히 접근할 수 있습니다. 이것이 클로저의 핵심입니다.

내부 함수는 외부 렉시컬 환경에 대한 참조를 유지하여 외부 함수가 완료된 후에도 그 환경의 변수에 접근할 수 있게 합니다.

클로저는 특히 비공개 변수와 함수를 만들 때 유용합니다. 다음 예제를 보세요:

:::interactive_editor

```js
function createCounter() {
    let count = 0;
    return function () {
        count++;
        return count;
    };
}

let counter = createCounter();
console.log(counter()); // 1
console.log(counter()); // 2
```

:::

이 경우 `createCounter`는 `count` 변수를 증가시키고 반환하는 함수를 반환합니다. `count` 변수는 `createCounter` 외부에서 직접 접근할 수 없지만, 반환된 함수(클로저)는 접근할 수 있습니다. `counter()`를 호출할 때마다 `count`가 증가하고 반환됩니다.

클로저는 외부 범위에서 여러 변수를 캡처할 수도 있습니다. 예를 들어:

:::interactive_editor

```js
function multiply(x) {
    return function (y) {
        return x * y;
    };
}

let double = multiply(2);
console.log(double(5)); // 10
```

:::

여기서 내부 함수는 `x`의 매개변수 `multiply`를 캡처합니다. `double`를 호출해 `multiply(2)`를 만들면, 항상 인수를 `2`와 곱하는 함수를 반환합니다.

클로저에 대해 중요한 점은 변수 값을 복사하는 것이 아니라 참조로 캡처한다는 것입니다. 즉, 캡처된 변수의 값이 변경되면 클로저는 새 값을 보게 됩니다. 예를 들어:

:::interactive_editor

```js
function createIncrementer() {
    let count = 0;
    return function () {
        count++;
        console.log(count);
    };
}

let increment = createIncrementer();
increment(); // 1
increment(); // 2
```

:::

`increment`를 호출할 때마다 초기 값의 복사본이 아니라 같은 `count` 변수를 사용합니다. 클로저는 JavaScript에서 강력한 도구입니다. JavaScript를 계속 사용하면서 클로저를 이해하고 효과적으로 사용하면 깔끔하고 효율적이며 강력한 코드를 작성하는 능력이 크게 향상됩니다.
