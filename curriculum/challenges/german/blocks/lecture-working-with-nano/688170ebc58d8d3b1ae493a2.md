# --description--

Wenn Sie im Terminal arbeiten, sind Standard-Texteditoren nicht immer verfügbar, aber es gibt Programme, die dafür entwickelt wurden, Dateien direkt im Terminal zu bearbeiten.

Linux-Umgebungen enthalten typischerweise einen Texteditor. Vi oder Vim sind meist die gebräuchlichsten. Emacs und Nano müssen eventuell manuell installiert werden, sind aber für fast jede Distribution leicht verfügbar.

Das Problem bei Vim ist die steile Lernkurve. Wenn Sie damit nicht vertraut sind, wissen Sie vielleicht nicht einmal, wie Sie die Anwendung schließen! Emacs ist vermutlich etwas benutzerfreundlicher, hat aber ebenfalls eine Vielzahl an Tastenkombinationen.

Nano ist deutlich schlanker und benutzerfreundlicher, besonders für Neueinsteiger. Es geht viel schneller, mit Nano eine Datei zu öffnen und zu bearbeiten, und die Anwendung bietet sogar eine Bildschirmhilfe für die grundlegenden Tastenkombinationen.

Der Nachteil dabei ist jedoch die fehlende Erweiterbarkeit. Im Gegensatz zu Vim oder Emacs, die mit Plugins zu vollständigen Entwicklungsumgebungen ausgebaut werden können, ist Nano eher minimalistisch.

Werfen wir einen Blick darauf, was Nano kann. Um eine Datei zu öffnen, können Sie `nano <filename>` im Terminal eingeben.

Sobald eine Datei geöffnet ist, sehen Sie oben auf dem Bildschirm die Version von Nano, die Sie verwenden, und den Namen der geöffneten Datei.

Darunter sehen Sie den Inhalt der Datei. Sie können die Pfeiltasten benutzen, um den Cursor zu bewegen, und mit der Tastatur Text einfügen oder löschen. Die Datei scrollt bei Bedarf mit.

Unten sehen Sie dann eine Liste der verfügbaren Tastenkombinationen, die ungefähr so aussieht:

```bash
^G Help   ^O Write Out    ^F Where Is   ^K Cut      ^T Execute  ^C Location     M-U Undo    M-A Set Mark    M-] To Bracket    M-B Previous
^X Exit   ^R Read File    ^\ Replace    ^U Paste    ^J Justify  ^/ Go To Line   M-E Redo    M-6 Copy        ^B Where Was      M-F Next
```

Tastenkombinationen, die mit dem Caret-Zeichen (`^`) beginnen, bedeuten, dass Sie die Strg-Taste gedrückt halten und den angegebenen Buchstaben drücken müssen. Kombinationen, die mit `M-` beginnen, erfordern, dass Sie die „Meta-Taste“ gedrückt halten, die in den meisten Setups die Alt-Taste sein sollte, und den Buchstaben drücken.

Um eine Datei in Nano zu bearbeiten, müssen Sie nur wenige der Tastenkombinationen kennen. Nachdem Sie Änderungen an einer Datei vorgenommen haben, können Sie `Ctrl + O` drücken, um die Datei zu „Write Out“ (speichern). Nach Eingabe des Befehls ändert sich das Menü unten zu diesem:

```bash
File Name to write : <filename>
^G Get Help                 ^T  To Files
^C Cancel                   TAB Complete
```

Drücken Sie Enter, um die Datei zu speichern und zur Bearbeitung zurückzukehren, oder `Ctrl + C`, um abzubrechen und zur Bearbeitung zurückzukehren.

Während Sie im Editor sind, können Sie `Ctrl + X` drücken, um Nano zu beenden und zum Terminal zurückzukehren. Wenn Sie beim Versuch zu beenden ungespeicherte Änderungen haben, sehen Sie dies im unteren Menü:

```bash
Save modified buffer (ANSWERING "No" WILL DESTROY CHANGES) ?
                            Y Yes
^C Cancel                   N No
```

Drücken Sie `Y`, um Ihre ungespeicherten Änderungen zu speichern, oder `N`, um sie zu verwerfen.

Mit diesem Wissen sind Sie besser vorbereitet, wenn Sie das nächste Mal schnell eine Datei auf einem entfernten Server bearbeiten müssen, auf dem Sie möglicherweise keinen Zugriff auf eine vollständige grafische Umgebung haben.
