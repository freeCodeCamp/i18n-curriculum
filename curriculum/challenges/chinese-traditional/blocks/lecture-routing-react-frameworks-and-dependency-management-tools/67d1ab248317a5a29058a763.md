# --description--

在軟體中，相依性是指應用程式中的一個組件或模組依賴另一個組件或模組才能正常運作。相依性在軟體應用程式中很常見，因為它允許開發者使用其他人創建的預先建立的函式（程式）或工具。

如果你想建置一個 React 應用程式，你需要安裝 React 依賴項。沒有這些依賴項，你的程式碼將無法正確運作，且應用程式會顯示錯誤列表。

當你處理較複雜的專案時，你經常需要依賴許多相依性。如果一組相依性在專案中沒有被妥善管理或定義，則會導致所謂的相依性地獄。

要管理專案中的軟體依賴性，你需要使用軟體包管理工具。軟體包管理工具是用於安裝、更新和移除依賴性的工具。許多流行的程式設計語言，如 JavaScript、Python、Ruby 和 Java，都使用軟體包管理工具。

在之前的課程中，你曾簡要介紹過一個流行的軟體包管理工具，叫做 npm。

要使用 Vite 和 npm 創建一個新的 React 專案，你可以執行以下指令：

```sh
npm create vite@latest my-react-app -- --template react
```

如你從之前的課程中回想，這將創建啟動新 React 應用程式所需的所有樣板程式碼。在你能啟動應用程式之前，你需要透過執行 `npm install` 或簡寫為 `npm i` 來安裝相依套件。

你可以在位於你的專案根目錄中的 `package.json` 檔案查看所有相依性。

`package.json` 檔案是在專案中包含關於你的專案的詮釋資料的關鍵設定檔案，包括其名稱、版本和相依性。它也定義了腳本、授權資訊以及其他有助於管理專案及其相依性的設定。

當你安裝相依套件時，`node_modules` 資料夾將會被添加到你的專案。

`node_modules` 資料夾是儲存你的專案所需所有軟體包和函式庫的地方。此資料夾包含 `package.json` 檔案中列出的相依性實際程式碼，包括你的專案直接相依性以及這些相依性的任何相依性。

React 專案所需的兩個核心依賴將是 `react` 和 `react-dom` 軟體包：

```json
"dependencies": {
  "react": "^18.3.1",
  "react-dom": "^18.3.1"
}
```

`package.json` 會列出你目前使用的那些軟體包版本。如果你需要在區域的更新任何軟體包，你可以執行 `npm update` 指令。或者你可以透過執行 `npm update -g` 指令來全域的更新所有軟體包。

除了 `package.json` 檔案之外，你還會有一個 `package-lock.json` 檔案。這個檔案會鎖定你的專案正在使用的所有軟體包的確切版本。當你更新軟體包時，新的版本也會在鎖定檔案中更新。

`package.json` 檔案的另一個重要面向是開發相依性：

```json
"devDependencies": {
  "@eslint/js": "^9.17.0",
  "@types/react": "^18.3.18",
  "@types/react-dom": "^18.3.5",
  "@vitejs/plugin-react": "^4.3.4",
  "eslint": "^9.17.0",
  "eslint-plugin-react": "^7.37.2",
  "eslint-plugin-react-hooks": "^5.0.0",
  "eslint-plugin-react-refresh": "^0.4.16",
  "globals": "^15.14.0",
  "vite": "^6.0.5"
}
```

開發相依性是只用於開發而非生產環境的軟體包。舉例來說，像 Jest 這樣的測試函式庫就是一個例子。你會將 Jest 安裝為開發相依性，因為你只在本機測試你的專案時使用它，且在應用程式於生產環境執行時不需要它。

在本課程的大部分時間裡，我們一直專注於 npm。但還有其他軟體包管理工具，例如 yarn 和 pnpm。那麼你應該為你的專案使用哪一個軟體包管理工具呢？

嗯，簡短的回答是視情況而定。

如果你是與團隊一起加入現有的專案，則所有這些決定都會為你做出，你將使用現有的軟體包管理工具。如果你是從零開始建置專案，則你需要研究每個管理工具的優缺點，並決定哪一個更適合你的需求。
