# --description--

لا يحتوي Node.js على خادم WebSocket مدمج. عادةً ما يستخدم المطوّرون حزمًا خارجية مثل `ws` لتنفيذ بروتوكول WebSocket والتعامل مع الاتصال في الوقت الحقيقي بين العملاء والخوادم.

مكتبة `ws` خفيفة الوزن، سريعة، وسهلة الاستخدام. تنفذ بروتوكول WebSocket لتتمكن من إنشاء وإدارة اتصالات WebSocket.

يمكنك تثبيتها باستخدام npm:

```bash
npm install ws
```

بعد التثبيت، أنشئ خادم WebSocket عن طريق استيراد مكتبة `ws`، واستدعاء طريقتها `Server`، وتمرير منفذ:

```js
const WebSocket = require("ws");

const server = new WebSocket.Server({
  port: 3000,
});
```

هذا يبدأ خادم WebSocket يستمع للاتصالات على المنفذ `3000`.

عندما يتصل عميل، يصدر الخادم حدث `connection`.

```js
server.on("connection", (socket) => {
  console.log("Client connected");
});
```

فكر في هذا الحدث كطريقة الخادم لإبلاغك بأن "عميل جديد انضم".

تمثل معلمة `socket` الاتصال بين الخادم وهذا العميل. تستخدمها لإرسال واستقبال الرسائل.

عندما يرسل عميل متصل بيانات، يصدر الخادم حدث `message`.

```js
server.on("connection", (socket) => {
  socket.on("message", (message) => {
    console.log(message.toString());
  });
});
```

في هذا المثال، يستمع الخادم للرسائل الواردة ويسجلها في وحدة التحكم.

يمكنك إرسال بيانات إلى العميل باستخدام الطريقة `send()`:

```js
socket.send("Hello from the server");
```

هذا يرسل رسالة عبر اتصال WebSocket المفتوح.

طالما بقي الاتصال مفتوحًا، يمكن لكل من العميل والخادم الاستمرار في تبادل الرسائل في الوقت الحقيقي.

بجانب `connection` و`message`، تصدر اتصالات WebSocket أيضًا أحداث `close` و`error`.

يُفعّل حدث `close` عند إغلاق الاتصال:

```js
socket.on("close", () => {
  console.log("Disconnected");
});
```

يُفعّل حدث `error` عند حدوث خطأ:

```js
socket.on("error", (error) => {
  console.error(error);
});
```

تشكل هذه الأحداث معًا أساس معظم تطبيقات WebSocket.

متطلب شائع في تطبيقات الدردشة هو البث. البث يعني إرسال رسالة إلى عدة عملاء متصلين بدلاً من واحد فقط.

تخيل غرفة دردشة بها ثلاثة مستخدمين يُدعون **John Doe** و**Jane Doe** و**Wes Smith**.

إذا أرسل John Doe رسالة، قد يحتاج الخادم إلى إعادة توجيه تلك الرسالة إلى Jane Doe وWes Smith.

للقيام بذلك، يحتفظ الخادم بسجل العملاء المتصلين ويتكرر عليهم عند وصول رسالة.

قبل إرسال رسالة، يتحقق الخادم مما إذا كان اتصال العميل لا يزال مفتوحًا:

```js
if (client.readyState === WebSocket.OPEN) {
  // do something
}
```

كل اتصال WebSocket له خاصية `readyState` تصف حالته الحالية.

`WebSocket.OPEN` تعني أن الاتصال نشط وجاهز للإرسال والاستقبال.

من خلال التحقق من ذلك، يتجنب الخادم إرسال الرسائل إلى العملاء الذين فصلوا بالفعل أو الذين لا تزال اتصالاتهم قيد الإنشاء:

```js
server.on("connection", (socket) => {
  socket.on("message", (message) => {
    server.clients.forEach((client) => {
      if (client.readyState === WebSocket.OPEN) {
        client.send(message.toString());
      }
    });
  });
});
```

في هذا المثال، يتلقى كل عميل متصل الرسالة الواردة. هذا ما يسمى بالبث.

تستخدم العديد من التطبيقات في الوقت الحقيقي البث لتوصيل بيانات مثل رسائل الدردشة، الإشعارات الحية، تحديثات الرياضة، وأحداث الألعاب متعددة اللاعبين.

مع مكتبة `ws`، من السهل بناء خوادم WebSocket في Node.js.

كل ما تحتاجه هو إنشاء خادم، الاستماع للأحداث، تبادل الرسائل، وبث التحديثات إلى العملاء المتصلين.

تشكل هذه اللبنات الأساسية أساس العديد من التطبيقات في الوقت الحقيقي.
