# --description--

У цьому уроці ви навчитеся працювати з базовими примітивними типами та дізнаєтеся, як працюють анотації типів для змінних.

Для примітивних типів даних `string`, `null`, `undefined`, `number`, `boolean` і `bigint` TypeScript пропонує відповідні ключові слова типів.

У цьому уроці ми зосередимося на загальновживаних примітивах і не будемо заглиблюватися в деталі `bigint`. Тип даних `bigint` — це числовий тип у JavaScript, який використовується для представлення чисел більших за стандартний цілий тип.

Давайте розглянемо кожен із цих примітивних типів, починаючи з `string`. Ось як можна задати тип рядка в TypeScript:

```ts
let firstName: string = "Angie";
```

На відміну від звичайного JavaScript, змінна `firstName` очікує тип `string`. Тож якщо ви спробуєте призначити їй значення типу число, отримаєте помилку такого вигляду:

```ts
let firstName: string = "Angie";
firstName = 9 // Type 'number' is not assignable to type 'string'.
```

Наступний тип, який ми розглянемо, — це тип `number`. Ось приклад:

```ts
let age: number = 16;
```

Ви можете призначати цілі числа та числа з плаваючою комою типу `number`. Але якщо спробуєте призначити значення типу `bigint`, отримаєте помилку такого вигляду:

```ts
let age: number = 16;

// Type 'bigint' is not assignable to type 'number'.
age = 123n;
```

Наступний тип, який ми розглянемо, — це тип `boolean`. Ось кілька прикладів:

```ts
let isLoggedIn: boolean = true;
let isAdmin: boolean = false;
```

Останні два типи — це типи `null` і `undefined`.

Як ви пам’ятаєте з попередніх уроків JavaScript, `null` представляє навмисну відсутність значення. `undefined` означає, що змінну оголосили, але їй не призначили значення.

Приклади роботи з цими типами ви побачите пізніше, коли вивчатимете типи функцій і об’єднані типи.

Тепер, коли ви побачили, як працюють анотації типів для змінних, виникає питання, чи потрібно завжди явно вказувати типи змінних.

Якщо у вас проста анотація типу, як ця, то, можливо, не потрібно використовувати явний тип.

```ts
let firstName: string = "Angie";
```

Причина в тому, що TypeScript намагається автоматично вивести типи на основі значення. Тож якщо ви напишете так, TypeScript зрозуміє, що це рядок.

```js
let firstName = "Angie";
```

Ви повинні явно вказувати типи змінних, коли TypeScript не може чітко вивести тип або коли хочете забезпечити конкретний тип для ситуацій, наприклад, параметрів функцій або складних об’єктів.
