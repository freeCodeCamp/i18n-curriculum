# --description--

Окрім GET, існує ще один поширений HTTP-метод — POST. POST — це метод за замовчуванням, який використовується для надсилання даних клієнта через HTML-форми. У конвенції REST POST використовується для надсилання даних з метою створення нових елементів у базі даних (новий користувач або новий блог-пост). У цьому проєкті у вас немає бази даних, але ви все одно навчитеся обробляти POST-запити.

У таких запитах дані не відображаються в URL, вони приховані в тілі запиту. Тіло — це частина HTTP-запиту, яку також називають корисним навантаженням. Хоча дані не видно в URL, це не означає, що вони приватні. Щоб зрозуміти чому, подивіться на сирий вміст HTTP POST-запиту:

```http
POST /path/subpath HTTP/1.0
From: john@example.com
User-Agent: someBrowser/1.0
Content-Type: application/x-www-form-urlencoded
Content-Length: 20

name=John+Doe&age=25
```

Як бачите, тіло кодується так само, як рядок запиту. Це формат за замовчуванням, який використовують HTML-форми. За допомогою Ajax можна також використовувати JSON для обробки даних із більш складною структурою. Існує ще один тип кодування: multipart/form-data. Він використовується для завантаження бінарних файлів. У цій вправі ви будете використовувати URL-кодоване тіло. Щоб аналізувати дані, що надходять із POST-запитів, потрібно використовувати пакет `body-parser`. Цей пакет дозволяє застосовувати низку підпрограмного забезпечення, які можуть декодувати дані у різних форматах.

# --instructions--

`body-parser` вже встановлено і він знаходиться у вашому `package.json` файлі проєкту. `require` його на початку `myApp.js` файлу і збережіть у змінній з ім’ям `bodyParser`. Підпрограмне забезпечення для обробки URL-кодованих даних повертається `bodyParser.urlencoded({extended: false})`. Передайте функцію, що повертається попереднім викликом методу, `app.use()`. Як завжди, підпрограмне забезпечення має бути підключене перед усіма маршрутами, які від нього залежать.

**Примітка:** `extended` — це параметр конфігурації, який вказує `body-parser`, який тип аналізу потрібно використовувати. Коли `extended=false`, він використовує класичне кодування `querystring` бібліотеки. Коли `extended=true`, він використовує `qs` бібліотеку для аналізу.

При використанні `extended=false` значення можуть бути лише рядками або масивами. Об’єкт, що повертається при використанні `querystring`, не наслідує прототипно від стандартного JavaScript `Object`, отже такі функції, як `hasOwnProperty`, `toString`, будуть недоступні. Розширена версія дозволяє більшу гнучкість даних, але її перевершує JSON.

Підпрограмне забезпечення 'body-parser' має бути підключене

```js
  const response = await fetch(code + '/_api/add-body-parser');
  if (!response.ok) {
    throw new Error(await response.text());
  }
  const data = await response.json();
  assert.isAbove(
    data.mountedAt,
    0,
    '"body-parser" is not mounted correctly'
  );
```
