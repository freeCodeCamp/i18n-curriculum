# --interactive--

تُستخدم دالة البناء `Number()` لإنشاء كائن رقم. يحتوي كائن الرقم على بعض الخصائص والطرق المفيدة مثل `isNaN` والطريقة `toFixed`. إليك مثالًا يستخدم دالة البناء `Number()` مع الكلمة المفتاحية `new`:

:::interactive_editor

```js
const myNum = new Number("34");
console.log(typeof myNum); // "object" 
```

:::

في هذا المثال نمرر سلسلة نصية حرفية إلى دالة البناء `Number()` ونوع القيمة المرجعة يكون من نوع `object` بدلًا من `string`.

عندما تُستدعى دالة البناء `Number()` كدالة بدون الكلمة المفتاحية `new`، تكون قيمة الإرجاع من النوع البدائي للرقم. في معظم الأحيان ستستخدم دالة البناء `Number()` لتحويل أنواع بيانات أخرى إلى نوع الرقم. إليك مثالًا لتحويل سلسلة نصية إلى رقم:

:::interactive_editor

```js
const myNum = Number("100");
console.log(myNum); // 100

console.log(typeof myNum); // number
```

:::

هذا مفيد عندما تحصل على مُدخَل من المستخدم وتحتاج إلى تحويل تلك السلسلة النصية إلى رقم لتتمكن من إجراء العمليات الحسابية.

إذا حاولت استدعاء دالة البناء `Number()` مع سلسلة فارغة، ستكون النتيجة الرقم `0`:

:::interactive_editor

```js
const num = Number("");
console.log(num); // 0
```

:::

ذلك لأن جافاسكريبت تحاول تحليل السلسلة النصية وبما أنها لا تحتوي على أرقام، تكون النتيجة صفرًا.

إذا حاولت تمرير سلسلة تحتوي على حروف عشوائية، ستكون النتيجة `NaN` أو "ليس رقمًا".

:::interactive_editor

```js
const num = Number("random");
console.log(num); // NaN
```

:::

عند العمل مع القيم المنطقية، تُرجع `true` القيمة `1` لأن `true` تتحول إلى واحد، وتُرجع `false` القيمة `0` لأن `false` تتحول إلى صفر.

:::interactive_editor

```js
const boolTrue = Number(true);
const boolFalse = Number(false);

console.log(boolTrue); // 1
console.log(boolFalse); // 0
```

:::

إذا مررت `null`، تكون النتيجة `0`، وإذا مررت `undefined`، تكون النتيجة `NaN`.

:::interactive_editor

```js
const undefinedNum = Number(undefined);
const nullNum = Number(null);

console.log(undefinedNum); // NaN
console.log(nullNum); // 0
```

:::

عند العمل مع المصفوفات هناك بعض الأمور التي يجب مراعاتها.

المصفوفة الفارغة تُرجع `0`. المصفوفة التي تحتوي على رقم واحد تُرجع ذلك الرقم. المصفوفة التي تحتوي على أرقام متعددة تُرجع `NaN`. والمصفوفة التي تحتوي على سلسلة/سلاسل نصية تُرجع أيضًا `NaN`.

:::interactive_editor

```js
const emptyArr = Number([]);
const arrOneNum = Number([7]);
const arrMultiNum = Number([7, 36, 12]);
const arrStr = Number(["str1"]);
const arrMultiStr = Number(["str1", "str2"]);

console.log(emptyArr); // 0
console.log(arrOneNum); // 7
console.log(arrMultiNum); // NaN
console.log(arrStr); // NaN
console.log(arrMultiStr); // NaN
```

:::

عند العمل مع الكائنات، تكون النتيجة دائمًا `NaN`.

:::interactive_editor

```js
const obj1 = Number({});
const obj2 = Number({2: 2});
const obj3 = Number({key: "val"});
const obj4 = Number({key: true});

console.log(obj1); // NaN
console.log(obj2); // NaN
console.log(obj3); // NaN
console.log(obj4); // NaN
```

:::

في الختام، ستستخدم دالة البناء `Number()` غالبًا لتحويل النوع أكثر من استخدامها لإنشاء رقم أو كائن رقم.
