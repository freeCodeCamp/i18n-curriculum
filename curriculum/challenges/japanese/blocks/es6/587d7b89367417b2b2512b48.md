# --description--

ES6では、複数のパラメータや要素が期待される場所で配列や他の式を展開できる<dfn>スプレッド演算子</dfn>が導入されました。

以下のES5コードは、配列の最大値を計算するために`apply()`を使っています。

```js
var arr = [6, 89, 3, 45];
var maximus = Math.max.apply(null, arr);
```

`maximus`は`89`の値になります。

`Math.max.apply(null, arr)`を使わなければならなかったのは、`Math.max(arr)`が`NaN`を返すからです。`Math.max()`はカンマ区切りの引数を期待しており、配列は受け付けません。スプレッド演算子を使うと、この構文はずっと読みやすく保守しやすくなります。

```js
const arr = [6, 89, 3, 45];
const maximus = Math.max(...arr);
```

`maximus`は`89`の値になります。

`...arr`は展開された配列を返します。つまり、配列をスプレッドします。ただし、スプレッド演算子は関数の引数や配列リテラルのように、その場でのみ機能します。例えば：

```js
const spreaded = [...arr];
```

しかし、次のコードは動作しません：

```js
const spreaded = ...arr;
```

# --instructions--

スプレッド演算子を使って、`arr1`のすべての内容を別の配列`arr2`にコピーしてください。
