# --description--

В React оновлення масивів у стані досить просте, але легко припуститися помилки, особливо якщо ви звикли до vanilla JavaScript, де можна безпосередньо змінювати масиви.

У React стан вважається незмінним, щоб він міг розпізнавати зміни і правильно оновлювати інтерфейс користувача.

Давайте розглянемо, як можна оновлювати масиви, що зберігаються в стані React.

Однією з найпоширеніших помилок при оновленні масивів у стані React є безпосередня зміна масиву за допомогою методів, таких як `push()`, `pop()` або `splice()`. Ці методи змінюють оригінальний масив, а React цього не дозволяє.

React покладається на новий референс масиву, щоб виявити зміни, тому безпосередня зміна масиву може завадити компоненту повторно відтворитися, як очікується.

Ось приклад використання методу `push()` для додавання до масиву в стані, який не спрацює:

```jsx
import { useState } from "react";

function ItemsList() {
  const [items, setItems] = useState([
    { id: 0, name: "Item 1" },
    { id: 1, name: "Item 2" },
    { id: 2, name: "Item 3" },
  ]);

  const addItem = () => {
    const newItem = { id: items.length + 1, name: `Item ${items.length + 1}` };
    items.push(newItem); // This modifies the original array
    setItems(items); // React will not detect this change
  };

  return (
    <div>
      <button onClick={addItem}>Add Item</button>
      <ul>
        {items.map((item) => (
          <li key={item.id}>{item.name}</li>
        ))}
      </ul>
    </div>
  );
}

export default ItemsList;
```

Якщо ви натиснете кнопку `Add Item`, у інтерфейсі користувача нічого не станеться.

Також може виникнути спокуса видалити елементи з масиву за допомогою методу `pop()`:

```js
const removeItem = () => {
  items.pop(); // Modifies the original array
  setItems(items); // React will not detect this change, either
};
```

Щоб оновити масив у стані, ключове — створити новий масив, виконати операції і передати його React, замість того, щоб змінювати існуючий масив.

Оскільки це новий масив, React зрозуміє, що стан змінився, і виконає повторне відтворення.

Ось як можна додати до масиву `items` за допомогою оператора розширення:

```js
const addItem = () => {
  const newItem = {
    id: items.length + 1,
    name: `Item ${items.length + 1}`,
  };

  // Creates a new array
  setItems((prevItems) => [...prevItems, newItem]);
};
```

`[...prevItems, newItem]` створює новий масив, копіюючи всі елементи існуючого масиву `items`, що зберігається в стані, а потім додає `newItem` у кінець, який збільшує `id` і номер елемента.

Якщо ви хочете щось видалити з масиву, можна використати метод `filter()`, який повертає новий масив після фільтрації того, що потрібно видалити:

```js
const removeItem = (id) => {
  setItems((prevItems) => prevItems.filter((item) => item.id !== id));
};
```

Ось повний код:

```jsx
import { useState } from "react";

function ItemsList() {
  const [items, setItems] = useState([
    { id: 0, name: "Item 1" },
    { id: 1, name: "Item 2" },
    { id: 2, name: "Item 3" },
  ]);

  const addItem = () => {
    const newItem = { id: items.length + 1, name: `Item ${items.length + 1}` };
    setItems((prevItems) => [...prevItems, newItem]); // Creates a new array
  };

  const removeItem = (id) => {
    setItems((prevItems) => prevItems.filter((item) => item.id !== id)); // Creates a new array
  };

  return (
    <div>
      <button onClick={addItem}>Add Item</button>
      <ul>
        {items.map((item) => (
          <li key={item.id}>
            {item.name}{" "}
            <button onClick={() => removeItem(item.id)}>Remove</button>
          </li>
        ))}
      </ul>
    </div>
  );
}

export default ItemsList;
```

Це деякі загальні способи оновлення масиву в стані.
