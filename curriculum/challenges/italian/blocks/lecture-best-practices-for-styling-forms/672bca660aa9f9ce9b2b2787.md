# --interactive--

Scopriamo alcuni dei problemi comuni quando si cerca di definire lo stile di input speciali come gli input `datetime-local` e `color`.

Questi tipi speciali di input si basano su pseudo-elementi complessi per creare elementi come i selettori di data e colore. Questo rappresenta una sfida significativa per definire lo stile di questi input. Una difficoltà è che, poiché lo stile predefinito dipende interamente dal browser, il CSS che fa apparire correttamente il selettore in un browser può produrre un risultato molto diverso in un altro.

Ecco un esempio di input colore:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<form>
  <label for="favorite-color">Pick your favorite color:</label>
  <input type="color" id="favorite-color" name="favorite-color">
</form>
```

```css
input {
  padding: 8px 12px;
  margin: 8px 0;
  border-radius: 6px;
  border: 1px solid #ccc;
}

input[type="color"] {
  width: 60px;
  height: 40px;
  padding: 0;
  border: 2px solid #555;
  border-radius: 4px;
  cursor: pointer;
}

```

:::

Un altro problema può essere la complessità dello pseudo-elemento. Considera il selettore di data; ci sono molte parti in movimento e la struttura complessa dello pseudo-elemento potrebbe rappresentare una sfida significativa nell'applicare lo stile alle aree giuste.

Ecco un esempio di input data:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<form>
  <label for="birthdate">Select your birthdate:</label>
  <input type="date" id="birthdate" name="birthdate">
</form>
```

```css
input {
  padding: 8px 12px;
  margin: 8px 0;
  border-radius: 6px;
  border: 1px solid #ccc;
}

input[type="date"] {
  padding: 6px 10px;
  border: 2px solid #555;
  border-radius: 4px;
  font-size: 14px;
  cursor: pointer;
}

input[type="date"]::-webkit-calendar-picker-indicator {
  background-color: #4CAF50;
  color: white;
  border-radius: 4px;
  cursor: pointer;
}

```

:::

Naturalmente, con questi elementi complessi, corri anche il rischio di perdere accidentalmente funzionalità importanti quando li stili manualmente. Non solo potresti perdere indicatori importanti come lo stato di focus o l'elemento selezionato, ma potresti potenzialmente rompere completamente il selettore.

Per questi motivi molti sviluppatori si affidano a librerie JavaScript o componenti personalizzati invece di usare i componenti integrati del browser.
