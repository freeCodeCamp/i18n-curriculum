# --description--

Das OS-Modul ist ein weiteres Standardmodul, das in Node.js eingebaut ist.

Es ermöglicht Ihnen, mit dem aktuellen Betriebssystem zu interagieren, auf dem Node läuft, sodass Sie auf wichtige Informationen wie den OS-Typ, CPU-Details, verfügbaren Speicher, Gesamtspeicher, Netzwerkschnittstellen und mehr zugreifen können.

Um das OS-Modul zu verwenden, importieren Sie es so:

```js
const os = require("os");
```

Das OS-Modul stellt mehrere Methoden bereit, mit denen Sie Ihre Anwendung systembewusster machen können. Schauen wir uns einige gängige an.

Die `platform()`-Methode ruft das Betriebssystem ab, auf dem Node gerade läuft:

```js
const os = require("os");
console.log(os.platform()); // darwin
```

Dies kann nützlich sein, um plattformübergreifende Skripte zu implementieren:

```js
if (os.platform() === 'win32') {
 // Windows specific code
} else {
 // Non-Windows specific code
}
```

Die `arch()`-Methode zeigt einen String an, der die CPU-Architektur darstellt, für die Node.js kompiliert wurde:

```js
const os = require("os");
console.log(os.arch()); // arm64
```

Dies kann hilfreich sein, wenn Sie möchten, dass Benutzer die richtigen Binärdateien und Abhängigkeiten für eine bestimmte Architektur herunterladen.

Andere mögliche Werte sind `'x64'`, `'arm'`, `'arm64'`, `'ia32'`, `'mips'`, `'ppc'` und weitere.

`type()` gibt den offiziellen OS-Namen zurück, sodass Sie Betriebssysteme programmatisch identifizieren können:

```js
const os = require("os");
console.log(os.type()); // Darwin (the core OS for macOS, iOS, and other Apple products)
```

`release()` zeigt die Kernel-Version des Betriebssystems an, den Kernteil des Betriebssystems, der Systemressourcen und die Kommunikation zwischen Hardware- und Softwarekomponenten verwaltet. Diese Methode kann nützlich sein, um die Kompatibilität zwischen OS-Kernel-Versionen und Serveranforderungen zu verfolgen.

Sie gibt dies als String wie `'20.6.0'` zurück:

```js
const os = require("os");
console.log(os.release()); // 25.0.0
```

`version()` gibt die spezifische Betriebssystemversion mit mehr Details als die `release()`-Methode zurück:

```js
const os = require("os");
console.log(os.version());
// Darwin Kernel Version 25.0.0: Wed Sep 17 21:41:39 PDT 2025;
// root:xnu-12377.1.9~141/RELEASE_ARM64_T8103
```

`cpus()` gibt ein Array von Objekten mit Details zu jedem logischen CPU-Kern zurück. Dies kann helfen, die CPU-Auslastung zu überwachen:

`uptime()` zeigt die Zeit seit dem Systemstart an. Dies kann helfen zu bestimmen, wie lange Server bereits laufen:

```js
const os = require("os");
console.log(os.uptime()); // 23047
```

`totalmem()` und `freemem()` zeigen die Gesamtmenge des Systemspeichers in Bytes bzw. den freien Systemspeicher in Bytes an:

```js
const os = require("os");
console.log(os.totalmem()); // 8589934592 (8 GB)
console.log(os.freemem()); // 93585408 (87 MB)
```

`userInfo()` gibt ein Objekt mit Informationen über den aktuellen Systembenutzer zurück:

```js
const os = require("os");
console.log(os.userInfo());

/*
[Object: null prototype] {
 uid: 502,
 gid: 20,
 username: 'user',
 homedir: '/Users/user',
 shell: '/bin/zsh'
}
*/
```
