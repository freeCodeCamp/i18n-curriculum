# --interactive--

Приховування вмісту на вебсторінці — це загальна практика в розробці, але важливо робити це так, щоб не порушувати доступність. Різні техніки приховування можуть по-різному впливати на те, як допоміжні технології інтерпретують і представляють вміст користувачам. Давайте розглянемо найкращі практики приховування вмісту, зберігаючи доступність. Одним із загальних методів приховування вмісту є використання `display: none`. Ось приклад:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<p class="hidden">Hidden text</p>
<p>Visible text</p>
```

```css
.hidden {
  display: none;
}
```

:::

Хоча це ефективно приховує вміст візуально, він також видаляється з дерева доступності. Дерево доступності — це структура, яку використовують допоміжні технології, такі як читачі екрана, щоб інтерпретувати і взаємодіяти з вмістом на вебсторінці. Воно представляє вміст і його семантичне значення у вигляді, зрозумілому для допоміжних технологій і користувача.

Використання `display: none` означає, що читачі екрана та інші допоміжні технології не зможуть отримати доступ до цього вмісту, оскільки він не включений у дерево доступності. Тому важливо застосовувати цей метод лише тоді, коли ви хочете повністю видалити вміст як із візуального представлення, так і з доступності.

Інший підхід до приховування вмісту — використання `visibility: hidden`:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<p class="hidden">Hidden text</p>
<p>Visible text</p>
```

```css
.hidden {
  visibility: hidden;
}
```

:::

`visibility: hidden` приховує вміст візуально, але залишає його в потоці документа, тобто він все ще займає місце на сторінці. Як і `display: none`, `visibility: hidden` також видаляє вміст із дерева доступності. Це означає, що допоміжні технології, такі як читачі екрана, не зможуть отримати доступ до прихованого вмісту. Використовуйте `visibility: hidden` лише тоді, коли хочете приховати вміст від усіх, включно з користувачами допоміжних технологій.

Для вмісту, який має бути прихований візуально, але залишатися доступним для читачів екрана, можна застосувати техніку, яку часто називають «візуально прихований» або «тільки для читача екрана». Ось приклад із використанням CSS-класу `.sr-only`, який є загальною технікою для візуального приховування вмісту, зберігаючи його доступним для читачів екрана:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<p class="sr-only">Hidden text</p>
<p>Visible text</p>
```

```css
.sr-only {
  position: absolute;
  width: 1px;
  height: 1px;
  padding: 0;
  margin: -1px;
  overflow: hidden;
  clip: rect(0, 0, 0, 0);
  white-space: nowrap;
  border: 0;
}
```

:::

У цьому прикладі ми використовуємо властивості, такі як `position`, `clip` і `white-space`, про які ви дізнаєтеся в майбутніх уроках. Наразі просто знайте, що це CSS-правило ефективно приховує вміст візуально, зберігаючи його доступним для читачів екрана. Це корисно для надання додаткового контексту користувачам читачів екрана без впливу на візуальний макет.

Для перемикання видимості вмісту розгляньте використання атрибута `hidden`:

:::interactive_editor

```html
<p hidden>This content is hidden</p>
<p>This content is visible</p>
```

:::

Атрибут `hidden` підтримується більшістю сучасних браузерів і приховує вміст як візуально, так і з дерева доступності. Його легко перемикати за допомогою JavaScript. Нарешті, будьте обережні з приховуванням важливого вмісту. Якщо інформація є ключовою для розуміння або використання вебсайту, вона має бути видимою і доступною для всіх користувачів. Приховуйте вміст лише тоді, коли це справді покращує користувацький досвід. Дотримуючись цих найкращих практик, ви зможете забезпечити доступність свого вмісту для всіх користувачів, незалежно від того, як вони взаємодіють із вашим вебсайтом.
