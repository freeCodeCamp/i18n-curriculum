# --description--

在之前的课程中，你学习了如何使用像 `string` 和 `boolean` 这样的原生类型。但 TypeScript 允许你从现有类型构建新类型。一个例子就是并集类型。

并集 `type` 是一种组合两个或更多其他 `type` 的 `type`。

这是基本语法：

```ts
type1 | type2 | type3
```

让我们来看一些示例，以更好地理解这个概念。 

这是一个名为 `id` 的变量，其并集类型为 `string` 或 `number`：

```ts
let id: string | number = "user_783";
```

该值可以更新为仍然对该并集类型有效的数字：

```ts
let id: string | number = 1024;
```

如果你尝试赋值另一个不是 `string` 或 `number` 的值，TypeScript 将显示错误信息：

```ts
// Type 'string[]' is not assignable to type 'string | number'.
let id: string | number = ["Java", "Python"];
```

在处理并集类型时，重要的是使用适用于所有类型的方法，否则 TypeScript 会抛出类似这样的误差：

```ts
function getId(id: string | number) {
  // Property 'toUpperCase' does not exist on type 'string | number'.
  return id.toUpperCase();
}
```

有一个涉及缩小并集的方案。但你将在后续课程中了解更多关于这方面的内容，以及并集类型如何与类型别名一起工作。

当处理 `null` 和 `undefined` 值时，`union` 类型也很有用。 

下面是一个示例，其中变量可能包含 `string` 值，或者如果尚未设置值，则可能为 `null`：

```ts
let username: string | null;

// Type 'number' is not assignable to type 'string | null'.
username = 42;
```

在这个分支中，`username` 可以是 `string` 或 `null`，但不能是其他任何值。当你想有意表现值的缺失时，可以使用 `null`。

这里是另一个使用 `undefined` 的例子：

```ts
let score: number | undefined;

score = undefined; 

// Type 'string' is not assignable to type 'number | undefined'.
score = "high";
```

在此示例中，`score` 可以是 `number` 或 `undefined`。当值尚未被赋值时，你可能会使用 `undefined`。

使用包含 `null` 和 `undefined` 的并集类型可以明确表示值可能缺失，同时仍允许 TypeScript 捕获无效的赋值。
