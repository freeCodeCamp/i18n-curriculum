# --description--

你是否遇到过 Express 应用崩溃却没有明确错误信息的情况？调试和日志可以帮助你准确看到幕后发生了什么，从而更快地找到并修复问题。

在本课中，我们将探讨 Express 如何处理调试和日志——以及你如何利用它们来提升开发流程。

## Express 的内置调试功能

Express 已经包含了一个名为 `debug` 的调试工具。你无需安装任何东西即可使用它。

要启用调试，只需在运行应用时设置 `DEBUG` 环境变量。

示例如下：

```js
DEBUG=express:* node index.js
```

此命令启用了 `express:*` 命名空间下的所有调试消息。你将开始看到类似这样的输出：

```js
express:router:route dispatching GET /
express:router:layer processing GET /
```

这些消息帮助你追踪路由分派、中间件调用等。它是理解请求流程的绝佳方式。

想更具体一点？你可以缩小命名空间范围：

```js
DEBUG=express:router node index.js
```

这只会显示与 Express 路由器逻辑相关的日志。

在 Windows 上，PowerShell 的语法稍有不同：

```js
$env:DEBUG = "express:*"; node index.js
```

CMD 中则是这样：

```js
set DEBUG=express:* && node index.js
```

思路相同，只是设置环境变量的方式不同。

现在让我们看看 Express 中的日志功能。

对于 HTTP 请求日志，许多开发者使用像 `morgan` 这样的中间件。

用法如下：

```bash
npm install morgan
```

然后将它添加到你的应用中：

```js
const express = require('express')
const morgan = require('morgan')
const app = express()

app.use(morgan('dev'))s
```

现在，每个传入请求都会在终端记录，包含方法、URL、状态码和响应时间等详细信息。

为什么这很重要？

调试让你洞察 Express 如何处理请求、中间件和路由，而不会让生产日志变得杂乱。

日志则为你记录请求过程中的情况，这对于调试错误、理解流量和及早发现问题至关重要。

两者结合，让你在开发过程中拥有控制力和清晰度。
