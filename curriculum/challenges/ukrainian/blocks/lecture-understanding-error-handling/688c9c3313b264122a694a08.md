# --description--

В Python обробка виключень є ключовою частиною написання надійних і стійких до помилок програм. Вона дозволяє передбачати, ловити та реагувати на помилки структуровано.

Обробка виключень — це процес перехоплення та керування помилками, які виникають під час виконання програми, щоб ваш код не аварійно не завершувався.

Python надає блоки `try`, `except`, `else` і `finally` для плавного оброблення помилок. Ось базовий приклад:

```py
try:
    x = 10 / 0
except ZeroDivisionError:
    print("You can't divide by zero!")
```

- `try`: Блок коду, де ви очікуєте, що може виникнути помилка.

- `except`: Цей блок виконується, якщо в середині try виникає помилка вказаного типу.

- У цьому випадку ділення на нуль викликає `ZeroDivisionError`, який потім ловиться і обробляється.

А ось приклад, що також показує, як використовувати блоки `else` і `finally`:

```py
try:
    x = 10 / 2
except ZeroDivisionError:
    print("You can't divide by zero!")
else:
    print('Division successful:', x)
finally:
    print('This block always runs.')
```

- `else`: Виконується, якщо в блоці try не виникло виключення.

- `finally`: Виконується завжди — незалежно від того, чи сталася помилка. Корисний для завдань очищення, наприклад, закриття файлів або звільнення ресурсів.

Ви також можете ловити кілька виключень за допомогою окремих блоків `except`:

```py
try:
    number = int('abc')
    result = 10 / number
except ValueError:
    print('That was not a valid number.')
except ZeroDivisionError:
    print("Can't divide by zero.")
```

Використовуючи окремі умови `except`, ви можете зробити реакцію на помилки більш конкретною та корисною.

Також можна використовувати об’єкт виключення, який зазвичай присвоюють іншому імені за допомогою ключового слова `as`. Тут ми використовуємо `e` як псевдонім для об’єкта помилки:

```py
try:
    x = 1 / 0
except ZeroDivisionError as e:
    print(f'Error occurred: {e}')
```

Використання `e` дозволяє отримати доступ до фактичного повідомлення про помилку або об’єкта для логування чи налагодження.

Ви також можете ловити кілька виключень у одній умові `except`, вказуючи виключення у вигляді кортежу:

```py
try:
    number = int(input('Enter a number: '))
    result = 10 / number
except (ValueError, ZeroDivisionError) as e:
    print(f'Error occurred: {e}')
```

Обробка виключень дозволяє вашим програмам плавно відновлюватися після помилок. Використовуючи `try`, `except`, `else` і `finally`, ви можете передбачати потенційні проблеми і створювати більш стійкі застосунки.
