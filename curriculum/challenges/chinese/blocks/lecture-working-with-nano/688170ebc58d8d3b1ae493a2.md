# --description--

在终端中工作时，标准文本编辑器并不总是可用，但有些程序专为直接在终端内编辑文件而设计。

Linux 环境通常包含一个文本编辑器。Vi 或 Vim 往往是最常见的。Emacs 和 Nano 可能需要手动安装，但它们几乎在每个发行（版本）中都可以轻松获得。

Vim 的挑战在于它有一个较大的学习曲线。如果你还不熟悉它，你甚至可能不知道如何关闭该应用！Emacs 可以说对用户更友好一些，但仍然有大量的快捷键。

Nano 明显更加简洁且用户友好，尤其适合新手学习。使用 Nano 编辑文件可以更快上手，应用甚至提供了基本快捷键的屏幕帮助。

然而，这样做的代价是缺乏可扩展性。与可以通过插件扩展成完整开发环境的 Vim 或 Emacs 不同，Nano 相当简洁。

让我们来看看 Nano 能做什么。要打开一个文件，你可以在终端输入 `nano <filename>`。

一旦打开 `file`，你将在屏幕顶部看到你正在运行的 Nano 版本和你打开的 `file` 名称。

下面是文件的内容。你可以使用方向键来移动你的光标，并用键盘插入或删除文本。文件会根据需要滚动以跟随。

然后，在底部，你将看到一个可用键盘快捷的列表，看起来像这样：

```bash
^G Help   ^O Write Out    ^F Where Is   ^K Cut      ^T Execute  ^C Location     M-U Undo    M-A Set Mark    M-] To Bracket    M-B Previous
^X Exit   ^R Read File    ^\ Replace    ^U Paste    ^J Justify  ^/ Go To Line   M-E Redo    M-6 Copy        ^B Where Was      M-F Next
```

以插入符号（`^`）符号开头的快捷表示你需要按住 Ctrl 密钥并按下指定的字母。以 `M-` 开头的快捷表示你需要按住“meta 密钥”，在大多数设置中应为 Alt 密钥，然后按下该字母。

要在 Nano 中编辑文件，你只需要知道几个快捷命令。修改文件后，你可以按 `Ctrl + O` 来“Write Out”或储存文件。输入命令后，你会看到底部的菜单变成这样：

```bash
File Name to write : <filename>
^G Get Help                 ^T  To Files
^C Cancel                   TAB Complete
```

按回车键储存文件并返回编辑文件，或按 `Ctrl + C` 取消并返回编辑文件。

在编辑器中，你可以按 `Ctrl + X` 退出 Nano 并返回终端。如果你在尝试退出时有未储存的更改，你将在底部菜单中看到此内容：

```bash
Save modified buffer (ANSWERING "No" WILL DESTROY CHANGES) ?
                            Y Yes
^C Cancel                   N No
```

按 `Y` 储存你未储存的更改，或按 `N` 放弃它们。

有了这些知识，下次你需要快速编辑远程服务器上的文件时，你应该能更好地应对，即使你可能无法访问完整的图形环境。
