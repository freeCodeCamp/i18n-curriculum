# --description--

このレッスンでは、基本的なプリミティブ型の扱い方と変数の型注釈の仕組みについて学びます。

プリミティブデータ型の`string`、`null`、`undefined`、`number`、`boolean`、および`bigint`に対して、TypeScriptは対応する型キーワードを提供しています。

このレッスンでは、よく使われるプリミティブに焦点を当て、`bigint`についてはあまり詳しく扱いません。`bigint`データ型は、JavaScriptで標準の整数型より大きな数値を表す数値データ型です。

まず`string`から始めて、これらのプリミティブ型を見ていきましょう。TypeScriptで文字列に型を付ける方法は次の通りです。

```ts
let firstName: string = "Angie";
```

バニラのJavaScriptとは異なり、変数`firstName`は`string`型を期待しています。ですから、数値の値を再代入しようとすると、次のようなエラーが発生します。

```ts
let firstName: string = "Angie";
firstName = 9 // Type 'number' is not assignable to type 'string'.
```

次に見る型は`number`型です。例を示します。

```ts
let age: number = 16;
```

`number`型には整数や小数を代入できます。しかし、`bigint`型の値を再代入しようとすると、次のようなエラーが発生します。

```ts
let age: number = 16;

// Type 'bigint' is not assignable to type 'number'.
age = 123n;
```

次に見る型は`boolean`型です。いくつかの例を示します。

```ts
let isLoggedIn: boolean = true;
let isAdmin: boolean = false;
```

最後の2つの型は`null`型と`undefined`型です。

以前のJavaScriptレッスンで学んだように、`null`は意図的な値の不在を表します。`undefined`は変数が宣言されたが値が代入されていないことを意味します。

これらの型の扱い方は、後で関数型やユニオン型を学ぶ際に例を見ていきます。

変数に型注釈を付ける仕組みを見たところで、常に明示的に変数の型を指定する必要があるのかという疑問が出てきます。

もしこのような単純な型注釈であれば、明示的な型指定は必要ない場合があります。

```ts
let firstName: string = "Angie";
```

その理由は、TypeScriptが値に基づいて型を自動的に推論しようとするからです。ですから、これを書くとTypeScriptは文字列だと理解します。

```js
let firstName = "Angie";
```

TypeScriptが型を明確に推論できない場合や、関数のパラメータや複雑なオブジェクトのように特定の型を強制したい場合は、明示的に変数に型を付けることをお勧めします。
