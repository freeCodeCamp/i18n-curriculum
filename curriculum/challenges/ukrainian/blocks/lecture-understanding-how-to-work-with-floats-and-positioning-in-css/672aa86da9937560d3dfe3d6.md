# --interactive--

Float у CSS — це техніка, спочатку створена для того, щоб текст обтікав елемент, наприклад, зображення. З часом розробники знайшли нові способи використання float, застосовуючи їх для дизайну макетів у креативний спосіб. Хоча сучасні методи макетування, як Flexbox і Grid, зараз використовуються частіше, розуміння float все ще важливе, особливо при роботі зі старим кодом або коли потрібно досягти певних ефектів макету.

Коли елемент має float, його виводять з нормального потоку документа і розміщують ліворуч або праворуч у межах контейнера. Наступний вміст обтікає цей плаваючий елемент, заповнюючи залишковий простір. Класичне застосування — обтікання текстом зображень, коли зображення плаває з одного боку, а текст обтікає його. Ця техніка досі широко використовується, особливо в статтях і блогах, де зображення потрібно розмістити поруч із текстом. Ось приклад коду:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<div class="container">
  <img src="https://placehold.co/150x150" alt="Placeholder Image">
</div>
```

```css
.container {
  border: 1px solid black;
  padding: 10px;
}

img {
  float: left;
  margin-right: 20px;
}
```

:::

У наведеному прикладі контейнер не обтікає плаваюче зображення. Зображення виведене з нормального потоку документа, і контейнер зменшився до нульової висоти, бо він «не бачить» плаваючі дочірні елементи.

Float також були популярні для створення багатоколонкових макетів до того, як Flexbox і Grid стали поширеними. Плаваючи елементи поруч один з одним, розробники могли створювати колонки, вирівняні по горизонталі. Однак при використанні float важливо вирішувати проблему колапсу батьківських елементів, коли їхні дочірні елементи плавають. Техніка clearfix застосовується до класу `container`, щоб виправити цю проблему.

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<div class="container">
  <img src="https://placehold.co/150x150" alt="Placeholder Image">
  <p> This is an example of text flowing around a floated image.</p>
</div>
```

```css
.container {
  border: 1px solid black;
}

/* Clearfix CSS */
.container::after {
  content: "";  
  display: block;
  clear: both;
}

img {
  float: left;
  margin-right: 20px;
}
```

:::

- `::after` — це псевдоелемент, який додає невидимий блок після вмісту контейнера.
- `content: ""` забезпечує наявність псевдоелемента, але не відображає жодного вмісту.
- `display: block` робить псевдоелемент блочним елементом.
- `clear: both` забезпечує очищення псевдоелементом обох сторін від будь-яких плаваючих елементів вище.

У наведеному прикладі ми додали новий абзац, щоб колапс став більш помітним. Оскільки абзац залишається в нормальному потоці документа, контейнер розширюється достатньо, щоб його обгорнути. Потім ми застосовуємо техніку clearfix, щоб виправити колапс і правильно відобразити кордон контейнера. Ви можете спробувати видалити правило clearfix, щоб побачити, як контейнер знову колапсує.

Техніка clearfix гарантує, що батьківський елемент правильно обгортає свої плаваючі дочірні елементи. Clearfix змушує батьківський контейнер «бачити» плаваючі дочірні елементи, додаючи властивість clear після плаваючого вмісту.

Хоча float більше не є основним методом для складних макетів через появу сучасніших технік, таких як Flexbox і Grid, вони все ще відіграють важливу роль у певних випадках. Чи то обтікання текстом зображень, чи робота з застарілими проєктами — розуміння того, як працюють float і як їх правильно очищати, є ключем до підтримки добре структурованих і адаптивних макетів.
