# --description--

これまでのレッスンでは、JavaScriptにおけるインポートとエクスポートの使い方を学びました。このレッスンでは、Reactでコンポーネントをインポートおよびエクスポートする方法を見ていきます。

エクスポートはコンポーネントを他のファイルで使えるようにします。インポートは、別の場所でエクスポートされたコンポーネントをファイル内で使えるようにします。Reactのプロジェクトでは、コンポーネントを再利用し組み合わせてユーザーインターフェースを構築するために、この方法が使われます。

このレッスンでは、Reactコンポーネントをデフォルトエクスポートと名前付きエクスポートでエクスポートする方法と、それらを必要な場所でインポートする方法を学びます。

この例では、`Cat` コンポーネントがあり、`Cat.jsx` というファイルに属しています。ファイル拡張子には、主にJSXを扱うため`.jsx` 拡張子を使っています。

この`Cat` コンポーネントは、Mr. Whiskersという猫のタイトルと画像を含むJSXマークアップを返します：

```jsx
function Cat() {
  return (
    <div className="card">
      <h2>Mr. Whiskers</h2>
      <img
        src="https://cdn.freecodecamp.org/curriculum/cat-photo-app/running-cats.jpg"
        alt="Tuxedo cats running on dirt ground."
      />
    </div>
  );
}
```

別のファイルで`Cat` コンポーネントを使いたい場合、まずこのようにエクスポートする必要があります：

```jsx
function Cat() {
  return (
    <div className="card">
      <h2>Mr. Whiskers</h2>
      <img
        src="https://cdn.freecodecamp.org/curriculum/cat-photo-app/running-cats.jpg"
        alt="Tuxedo cats running on dirt ground."
      />
    </div>
  );
}

export default Cat;
```

このモジュールのデフォルトエクスポートになるため、`default` キーワードを使っています。コンポーネント定義と同じ行でエクスポートすることもできます：

```jsx
export default function Cat() {
  return (
    <div className="card">
      <h2>Mr. Whiskers</h2>
      <img
        src="https://cdn.freecodecamp.org/curriculum/cat-photo-app/running-cats.jpg"
        alt="Tuxedo cats running on dirt ground."
      />
    </div>
  );
}
```

子コンポーネントは他の親コンポーネントのファイルでインポートすることも、ルートコンポーネントのファイルでインポートすることもできます。この例では、ルートコンポーネントのファイル内で`Cat` コンポーネントをインポートします。

すべてのReactプロジェクトにはトップレベルのコンポーネントがあり、通常は`App.jsx` と呼ばれます：

```jsx
export default function App() {
  return (
    // return component here
  );
}
```

このファイルは通常、プロジェクトの`src` ディレクトリにあります。一般的なプロジェクト構成については、今後のレッスンで詳しく学びます。

ルートの`Cat` コンポーネント内で`App` コンポーネントを使うには、次のようにインポートします：

```jsx
import Cat from "./Cat";

export default function App() {
  return (
    // return component here
  );
}
```

そして、`Cat` コンポーネント内で`App` コンポーネントを返します：

```jsx
import Cat from "./Cat";

export default function App() {
  return (
    <Cat />
  );
}
```

この方法は、ファイルが単一の主要なコンポーネントをエクスポートする場合にうまく機能し、インポート構文をシンプルで読みやすく保ちます。ファイルはデフォルトエクスポートを1つだけ持てるため、主に1つのコンポーネントを含むファイルに最適です。

デフォルトエクスポートの使い方がわかったところで、名前付きエクスポートを見てみましょう。名前付きエクスポートは、ファイルが複数のコンポーネントや関数を共有できるようにします。デフォルトエクスポートとは異なり、エクスポートされた正確な名前でインポートする必要があります（`as` キーワードを使って名前を変更しない限り）。

これはファイルがコンポーネントのライブラリとして機能する場合に便利です。例えば、CatとDogという2つのコンポーネントを含むファイルは次のようになります：

```jsx
// Animals.jsx
export function Cat() {
  return <h2>Mr. Whiskers</h2>;
}

export function Dog() {
  return <h2>Fido</h2>;
}
```

それぞれのコンポーネントに個別にexportキーワードを使っています。別のファイルでこれらを使うには、波括弧を使ってインポートします：

```jsx
import { Cat, Dog } from "./Animals";

export default function App() {
  return (
    <div>
      <Cat />
      <Dog />
    </div>
  );
}
```

`{ Cat, Dog }` 構文は、JavaScriptに`Animals.jsx` ファイルから特定の名前付きエクスポートをインポートするよう指示しています。波括弧内の名前はエクスポートされた名前と完全に一致する必要があります。名前の衝突を避けたい場合や、現在のファイルで別の名前を使いたい場合は、asキーワードを使ってインポート時に名前を変更できます。

```jsx
import { Cat as Kitty } from "./Animals";

export default function App() {
  return <Kitty />;
}
```

最後に、ファイルは1つのデフォルトエクスポートと複数の名前付きエクスポートを持つことができます。

```jsx
// Animals.jsx
export default function Cat() {
  return <h2>Mr. Whiskers</h2>;
}

export function Dog() {
  return <h2>Fido</h2>;
}
```

混合エクスポートをインポートする場合、デフォルトエクスポートが最初に（波括弧なしで）、名前付きエクスポートが続きます（波括弧内）：

```jsx
// App.jsx
import Cat, { Dog } from "./Animals";

export default function App() {
  return (
    <div>
      <Cat />
      <Dog />
    </div>
  );
}
```

この例では、`Cat` がデフォルトエクスポートなので波括弧を使いません。`Dog` は名前付きエクスポートなので波括弧が必要です。

デフォルトエクスポートと名前付きエクスポートの仕組みを理解することで、Reactコンポーネントを複数のファイルに整理し、必要な場所で再利用できます。大規模なアプリケーションを構築する際には、適切なエクスポートスタイルを選ぶことでコードを明確かつ保守しやすくし、他の人が作成したコンポーネントを扱いやすくなります。
