# --description--

CSP（内容安全策略）和 Permissions-Policy 是两个强大的超文本传输协议安全头，通过限制可以加载的资源和页面可以访问的浏览器功能来保护 web 应用。

内容安全策略主要关注资源加载。它充当你前端脚本、样式和图像的安全过滤器。

恶意行为者经常试图向易受攻击的网站注入有害脚本。强大的 CSP 通过建立严格的可信源域白名单阻止这些跨站脚本（XSS）攻击。

当服务器发送 CSP 头时，浏览器会读取具体的指令。例如，`script-src` 指令告诉浏览器哪些域可以安全地在页面上运行 JavaScript。

如果黑客试图从未经批准的外部服务器注入恶意脚本，浏览器会检测到策略违规，立即丢弃该脚本，并记录安全误差。

CSP 还可以限制图像来源、表单目标和样式表来源。

Permissions-Policy 关注浏览器功能和设备硬件。它取代了旧的 Feature-Policy 头。

这种安全机制允许站点所有者显式启用或禁用自己页面及任何嵌入 iframe 的特定浏览器功能，并防止第三方代码滥用用户隐私。

使用 Permissions-Policy 头，你可以控制对摄像头、麦克风和地理位置跟踪等硬件的访问。

例如，如果你的网站不需要知道用户位置，你可以设置策略完全阻止地理位置 API。即使攻击者入侵了你页面上的脚本，浏览器也会拒绝激活跟踪功能。

Permissions-Policy 的实现使用清晰、结构化的头格式，命名功能并定义其允许的来源。

要严格限制硬件功能仅限你自己的域，你使用括号括起来的 `self` 关键字，比如 `camera=(self)`。如果你想对所有用户在整个页面禁用某个功能，则使用空列表，形式如 `geolocation=()`。

结合这两个头为 web 应用提供坚实防御。CSP 锁定数据传输并防止代码注入，Permissions-Policy 保护物理设备并确保隐私合规。

现代 web 开发者使用这两种工具来创建高度防护的浏览环境。
