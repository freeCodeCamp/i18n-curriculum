# --interactive--

プログラミングにおけるスコープとは、コードの異なる部分で変数の可視性とアクセス可能性を指します。スコープは変数がどこでアクセスまたは変更できるかを決定します。JavaScriptでは、スコープを理解することが、クリーンで効率的かつバグのないコードを書くために非常に重要です。スコープには主に3つの種類があります：グローバルスコープ、ローカルスコープ、そしてブロックスコープです。

グローバルスコープはJavaScriptプログラムの最も外側のスコープです。グローバルスコープで宣言された変数は、関数やブロック内を含めてコードのどこからでもアクセスできます。これらの変数はしばしばグローバル変数と呼ばれます。グローバル変数は便利な場合がありますが、名前の衝突を引き起こしやすく、コードの保守が難しくなるため、使用は控えめにすることをお勧めします。以下はグローバル変数の例です：

:::interactive_editor

```js
let globalVar = "I'm a global variable";

function printGlobalVar() {
    console.log(globalVar);
}

printGlobalVar(); // "I'm a global variable"
```

:::

この例では、`globalVar` はグローバルスコープで宣言されており、`printGlobalVar` 関数内でもアクセスできます。

一方、ローカルスコープは関数内でのみアクセス可能な変数を指します。以下はローカルスコープの例です：

:::interactive_editor

```js
function greet() {
    let message = "Hello, local scope!";
    console.log(message);
}

greet(); // "Hello, local scope!"
// console.log(message); // This will throw an error
```

:::

このコードでは、`message` は`greet` 関数内のローカル変数です。関数内で使用できますが、関数外でアクセスしようとするとエラーになります。

ブロックスコープはES6で`let` と `const` キーワードとともに導入された概念です。ブロックとは、波括弧（中括弧）`{}` 内の任意のコードセクションで、`if` 文、`for` ループ、または`while` ループなどが該当します。ループの概念は今後のレッスンで学習します。

ブロック内で`let` または `const` で宣言された変数は、そのブロック内でのみアクセス可能です。以下はブロックスコープの例です：

```js
if (true) {
    let blockVar = "I'm in a block";
    console.log(blockVar); // "I'm in a block"
}
console.log(blockVar); // This will throw an error
```

この例では、`blockVar` は`if` ブロック内でのみアクセス可能です。ブロック外でアクセスしようとするとエラーになります。これらの異なるスコープの種類を理解することは、変数のアクセス管理や意図しない副作用を避けるために不可欠です。

グローバル変数は名前の衝突を引き起こしやすく、コードの保守を難しくするため、控えめに使うべきです。ローカル変数はコードの異なる部分を分離して保つのに役立ち、特に大規模なプログラムで有用です。`let` と `const` によるブロックスコープは、変数のアクセス制御をさらに細かくし、エラーを防ぎコードの予測可能性を高めます。グローバル、ローカル、ブロックスコープの基本概念をマスターすることは、より高度なトピックを理解するためのしっかりした基盤となります。
