# --interactive--

客戶端儲存允許網站將資料儲存在使用者的裝置上。然而，像許多技術一樣，它可能會被濫用。 

讓我們探討一些與客戶端儲存相關的負面樣式，重點放在 cookie 及其被濫用於追蹤和指紋識別。

讓我們從 cookies 開始。 

在之前的課程中，你學到 cookies 是當使用者造訪網頁應用程式時儲存在使用者裝置上的資料。cookies 常見的誤用是過度追蹤。

網站使用 cookie 來追蹤使用者與網頁應用程式的互動，建立他們數位活動的歷史紀錄。這是為了標的廣告，但可能會引發重大的隱私疑慮。

例如，購物網站可能會使用像這樣的追蹤 cookie：

```js
document.cookie = "userID=123; path=/; expires=Thu, 18 Dec 2024 6:00:00 UTC";
```

這段程式碼設定了一個名為 `userID`、值為 `123` 的 cookie。該 cookie 將隨每個對網站的請求一起發送，使其能追蹤使用者的行為。雖然這看起來無害，但想像數百個網站共享這些資訊－這可能會產生你線上生活和選擇的非常詳細的圖像。

另一個令人擔憂的做法是瀏覽器指紋識別。 

此技術使用客戶端資訊來創建使用者瀏覽器的獨特「fingerprint」。網站能夠收集關於你瀏覽器版本、已安裝外掛程式、螢幕解析度及其他資料，以唯一識別你。 

這是一個網站如何為你創建指紋的簡單範例：

:::interactive_editor

```js
let fingerprintExample = navigator.userAgent + screen.width + screen.height;
console.log(fingerprintExample);
```

:::

這段程式碼將你的瀏覽器的 user agent 與你的螢幕尺寸結合起來。雖然這是一個基本範例，但真正的指紋辨識方法要複雜得多，且在識別使用者方面可以非常準確。

`localStorage` 也可能被誤用，因為有些網站會用它來不安全地儲存敏感資訊。例如：

```js
localStorage.setItem('userPassword', 'someonesPasswordHere');
```

這段程式碼將使用者的密碼儲存在 `localStorage`。這是一個嚴重的安全風險，因為 `localStorage` 中的資料未加密且容易被存取。

總結來說，雖然客戶端儲存提供許多利益，但負責任地使用它至關重要。當你繼續你的網頁開發旅程時，務必考慮使用者在客戶端資料的隱私與安全。
