# --description--

터미널에 대해 배우고, 커맨드 라인과 어떻게 다른지 알아보겠습니다.

커맨드 라인은 사용자가 보통 일련의 문자 형태로 된 "명령어"를 입력하고, 보통 "Enter" 키를 눌러 제출하거나 실행할 수 있는 기본 텍스트 입력 인터페이스입니다. 커맨드 라인 인터페이스는 대부분 터미널 내에서 볼 수 있습니다.

터미널은 기본적인 읽기/쓰기 작업을 넘어 시스템 수준 명령을 수행할 수 있는 커맨드 라인 인터페이스를 제공하는 특별한 애플리케이션입니다. "터미널 에뮬레이터"라는 용어도 들을 수 있는데, 이는 기본 터미널 인터페이스를 감싸 추가 기능이나 기능을 제공하는 애플리케이션입니다. 하지만 일반적인 용도에서는 이들도 그냥 "터미널"이라고 부르는 것이 무방합니다.

마지막으로 "셸"에 대해서도 이야기해야 합니다. 셸은 커맨드 라인을 감싸고 입력을 명령어로 해석하여 출력을 반환하는 소프트웨어입니다.

그렇다면 시스템에서 터미널에 어떻게 접근할 수 있을까요?

최신 버전의 Windows를 사용한다면 두 가지 옵션이 있을 가능성이 큽니다. Windows는 기본적으로 두 가지 셸 애플리케이션을 제공합니다: Powershell과 Command Prompt입니다.

이 두 셸은 각각의 이름을 가진 터미널 애플리케이션과 함께 제공되므로, 애플리케이션 메뉴에서 Powershell 터미널이나 Command Prompt 터미널 중 하나를 실행할 수 있습니다.

Windows는 또한 Microsoft Terminal이라는 비교적 새롭고 현대화된 터미널 에뮬레이터를 제공합니다.

이 애플리케이션은 Windows Store에서 설치할 수 있으며, Powershell이나 Command Prompt 터미널처럼 실행할 수 있습니다. Microsoft Terminal의 장점은 같은 애플리케이션에서 Powershell과 Command Prompt 셸 모두에 접근할 수 있고, Windows Subsystems for Linux를 설정했다면 해당 Linux 셸에도 접근할 수 있다는 점입니다.

MacOS는 "Terminal"이라는 이름의 터미널을 제공합니다. Spotlight 검색이나 애플리케이션 실행기에서 접근할 수 있습니다.

MacOS는 iTerm이나 Ghostty 같은 서드파티 터미널 에뮬레이터도 지원합니다. 이 애플리케이션들은 각자의 홈페이지에서 설치할 수 있습니다.

Linux에서는 기본 터미널과 실행 방법이 사용하는 배포판과 데스크톱 환경에 크게 좌우됩니다. 서드파티 터미널 에뮬레이터도 많이 지원합니다. 예를 들어 Arch Linux에서는 kitty를 설정할 수 있고, wofi 같은 애플리케이션 관리자를 통해 실행할 수 있습니다.

Linux는 선택할 수 있는 터미널 애플리케이션 종류가 가장 다양하다고 할 수 있습니다. 대부분 배포판의 패키지 관리자를 통해 설치할 수 있습니다.

"터미널", "셸", "커맨드 라인"의 구분은 때때로 지나치게 세세하게 느껴질 수 있습니다. 그리고 대부분의 경우 이 용어들은 서로 바꿔 써도 무방합니다. 하지만 각각이 가진 구체적인 의미와 그 의미가 무엇인지 아는 것이 중요합니다.
