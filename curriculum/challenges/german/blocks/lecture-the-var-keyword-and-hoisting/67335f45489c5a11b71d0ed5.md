# --interactive--

Hoisting ist ein Verhalten in JavaScript, das Anfänger oft verwirrt, aber das Verständnis davon kann Ihnen helfen, subtile Bugs in Ihrem Code zu vermeiden. Einfach ausgedrückt ist Hoisting das Standardverhalten von JavaScript, Deklarationen während der Kompilierungsphase vor der Ausführung des Codes an den Anfang ihres jeweiligen Gültigkeitsbereichs zu verschieben.

Um Hoisting zu verstehen, ist es wichtig zu wissen, dass JavaScript in zwei Phasen läuft: der Kompilierungsphase und der Ausführungsphase. Während der Kompilierungsphase geht die JavaScript-Engine Ihren Code durch und richtet Speicherplatz für Variablen und Funktionen ein. Hier kommt Hoisting ins Spiel.

Beginnen wir mit dem Hoisting von Variablen: Wenn Sie eine Variable mit dem `var`-Schlüsselwort deklarieren, hebt JavaScript die Deklaration an den Anfang ihres Gültigkeitsbereichs. Es ist jedoch entscheidend zu beachten, dass nur die Deklaration gehoben wird, nicht die Initialisierung. Das bedeutet, Sie können eine Variable in Ihrem Code verwenden, bevor Sie sie deklariert haben, aber ihr Wert wird `undefined` sein, bis Sie ihr tatsächlich einen Wert zuweisen.

:::interactive_editor

```js
console.log(x); // undefined
var x = 5;
console.log(x); // 5
```

:::

In diesem Code erhalten wir, obwohl wir `x` vor der Deklaration verwenden, keinen Fehler, sondern stattdessen `undefined`. Das liegt daran, dass JavaScript die Deklaration `var x` an den Anfang ihres Gültigkeitsbereichs hebt, aber nicht die Initialisierung `x = 5`. Es ist, als wäre der Code so umgeschrieben:

:::interactive_editor

```js
var x;
console.log(x); // undefined
x = 5;
console.log(x); // 5
```

:::

Das Hoisting von Funktionen funktioniert etwas anders. Wenn Sie eine Funktion mit der `function`-Deklarationssyntax deklarieren, werden sowohl der Funktionsname als auch der Funktionskörper gehoben. Das bedeutet, Sie können eine Funktion aufrufen, bevor Sie sie in Ihrem Code deklariert haben. Hier ist ein Beispiel für Funktions-Hoisting:

:::interactive_editor

```js
sayHello(); // "Hello, World!"

function sayHello(){
  console.log("Hello, World!");
}
```

:::

In diesem Fall können wir `sayHello()` vor seiner Deklaration aufrufen, weil die gesamte Funktion an den Anfang ihres Gültigkeitsbereichs gehoben wird.

Es ist wichtig zu beachten, dass Hoisting bei `let`- und `const`-Deklarationen, die in ES6 eingeführt wurden, anders funktioniert.

```js
console.log(y); // Throws a ReferenceError
let y = 10;
```

Diese Deklarationen werden zwar gehoben, aber sie sind nicht initialisiert, und Sie können nicht auf sie zugreifen, bevor die eigentliche Deklaration in Ihrem Code steht. Dies wird oft als temporäre Todzone bezeichnet.

Das Verständnis von Hoisting kann Ihnen helfen, saubereren und besser vorhersehbaren Code zu schreiben. Allerdings kann das Verlassen auf Hoisting Ihren Code schwerer lesbar und wartbar machen. Als bewährte Vorgehensweise wird empfohlen, Ihre Variablen am Anfang ihres Gültigkeitsbereichs und Ihre Funktionen vor ihrer Verwendung zu deklarieren, unabhängig vom Hoisting. Das macht das Verhalten Ihres Codes expliziter und für andere, einschließlich Ihres zukünftigen Ichs, leichter verständlich.
