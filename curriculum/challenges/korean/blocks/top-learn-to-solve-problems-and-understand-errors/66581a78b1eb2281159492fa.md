# --description--

멋진 JavaScript를 배우기 전에, 개발자에게 가장 중요한 기술인 문제 해결에 대해 이야기하기 시작해야 합니다.

문제 해결은 소프트웨어 개발자가 하는 핵심 작업입니다. 그들이 사용하는 프로그래밍 언어와 도구는 이 기본 기술에 비하면 부차적입니다.

V. Anton Spraul의 저서 "Think Like a Programmer"에서 문제 해결을 다음과 같이 정의합니다:

> 문제 해결은 특정 작업 집합을 수행하고 모든 명시된 제약 조건을 충족하는 독창적인 프로그램을 작성하는 것입니다.

작업 집합은 작은 코딩 연습 문제를 푸는 것부터 Facebook 같은 소셜 네트워크 사이트나 Google 같은 검색 엔진을 구축하는 것까지 다양합니다. 각 문제는 고유한 제약 조건을 가지는데, 예를 들어 고성능과 확장성은 코딩 연습 문제에서는 크게 중요하지 않을 수 있지만, 매일 수십억 건의 검색 쿼리를 처리해야 하는 Google 같은 애플리케이션에서는 필수적입니다.

새로운 프로그래머들은 종종 문제 해결을 가장 어려운 기술로 느낍니다. 구문과 프로그래밍 개념을 쉽게 배우면서도, 직접 코드를 작성하려 할 때 어디서부터 시작해야 할지 몰라 텍스트 편집기를 멍하니 바라보는 경우가 흔합니다.

문제 해결 능력을 향상하는 가장 좋은 방법은 많은 프로그램을 만들어 경험을 쌓는 것입니다. 연습을 많이 할수록 실제 문제를 해결할 준비가 더 잘 됩니다.
