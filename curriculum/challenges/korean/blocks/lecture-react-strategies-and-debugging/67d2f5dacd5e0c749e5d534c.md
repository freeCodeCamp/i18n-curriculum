# --description--

브라우저에는 HTML, CSS, JavaScript를 디버그할 수 있는 내장 개발자 도구가 있습니다.

하지만 React 앱의 버그를 찾고 고치기에는 적합하지 않습니다. 그래서 React 팀은 React 앱을 검사하고, 디버그하며, 프로파일링할 수 있는 "React Developer Tools"(React DevTools라고도 함)라는 도구를 개발했습니다.

React DevTools는 Chrome, Edge, Firefox용 브라우저 확장 프로그램으로 제공됩니다. Chrome이나 Edge를 사용한다면 Chrome 웹 스토어에 접속해 "React Developer Tools"를 검색하고 브라우저에 추가하세요.

Firefox를 사용한다면 Firefox 애드온 페이지에 접속해 도구를 검색하고 브라우저에 추가하세요.

Safari를 사용한다면 `npm install -g react-devtools` 또는 `yarn global add react-devtools`을 실행해 npm에서 React DevTools를 설치할 수 있습니다.

React DevTools를 설치하고 활성화한 후 React 앱을 브라우저에서 열고 개발자 도구를 열면, Components와 Profiler라는 두 개의 추가 탭이 보일 것입니다.

Components 탭은 각 컴포넌트를 트리 뷰 형식으로 보여줍니다. 이를 통해 다음을 할 수 있습니다:

- 앱의 컴포넌트 계층 구조를 확인하세요.
- 프로프스, 상태, 컨텍스트 값을 실시간으로 확인하고 수정하세요.
- 선택한 각 컴포넌트의 소스 코드를 확인하세요.
- 컴포넌트 데이터를 콘솔에 기록하세요.
- 컴포넌트의 DOM 요소를 검사하세요.

반면 Profiler 탭은 컴포넌트 성능을 기록하고 분석하는 데 도움을 줍니다. 이를 통해 불필요한 재렌더링을 식별하고, 커밋 소요 시간과 최적화할 부분을 확인할 수 있습니다.

다음은 컴포넌트와 그 안의 프로프스 및 상태를 검사하는 방법을 보여주는 간단한 앱입니다. 이전 수업에서 프로프 드릴링에 사용한 코드와 비슷합니다:

```jsx
import { useState } from "react";

export default function App() {
  const greeting = "Hello, Prop Drilling!";
  const response = "I'm not here to play!";

  return <Parent greeting={greeting} response={response} />;
}

const Parent = ({ greeting, response }) => {
  return <Child greeting={greeting} response={response} />;
};

const Child = ({ greeting, response }) => {
  return <Grandchild greeting={greeting} response={response} />;
};

const Grandchild = ({ greeting, response }) => {
  const [count, setCount] = useState(0);

  return (
    <>
      <h1>{greeting}</h1>
      <h2>{response}</h2>

      <button onClick={() => setCount(count + 1)}>Increase Count</button>
      <h2>Count: {count}</h2>
      <button onClick={() => setCount(count - 1)}>Decrease Count</button>
    </>
  );
};
```

React DevTools의 Components 탭을 보면 컴포넌트 트리 뷰를 볼 수 있습니다. `App` 컴포넌트가 최상단에 있고, 그 아래에 `Parent`, `Child`, `Grandchild` 컴포넌트가 있습니다.

이 컴포넌트들 중 하나를 선택하면 그 안의 프로프스와 상태를 볼 수 있습니다. `Parent` 컴포넌트를 선택하면 `greeting`와 `response` 프로프스가 각각 `Hello, Prop Drilling!`와 `I'm not here to play!`임을 확인할 수 있습니다.

프로프스와 상태를 실시간으로 보고 필요하면 변경할 수 있습니다. 예를 들어 `Grandchild` 컴포넌트를 선택하고 `greeting` 프로프스를 `Hello, Prop Drilling!`에서 `Hello, Welcome to Prop Drilling!`로 변경하면 페이지에 즉시 반영됩니다.

컴포넌트 데이터를 콘솔에 기록하려면 일치하는 DOM 요소를 검사하고 컴포넌트의 소스 코드를 확인하세요. 오른쪽 상단의 아이콘을 사용하면 됩니다. `Grandchild` 컴포넌트를 선택하고 `Log the component data to the console` 버튼을 클릭하면 프로프스, 상태, 훅, 노드 등 데이터를 콘솔에 기록합니다.

React에서 흔히 만나는 버그 중 하나는 프로프스 불일치입니다.

예를 들어 `Child` 컴포넌트에 `reply` 프로프스를 전달해야 하는데 실수로 `response`를 전달했다고 가정해 보세요:

```jsx
const Child = ({ greeting, response }) => {
  return <Grandchild greeting={greeting} reply={response} />;
};
```

`Grandchild`는 `response` 프로프스를 기대합니다. 컴포넌트가 다른 프로프스를 받으면 페이지에 해당 텍스트를 표시하지 못하고 빈 `h2`를 DOM에 추가합니다. 대신 `h1` 요소에 `Hello, Prop Drilling!` 텍스트가 표시되고, 이미 페이지에 있는 다른 버튼과 텍스트도 그대로 보입니다. 빈 `h2` 요소는 존재하지만 비어 있어서 DOM을 검사하지 않으면 보이지 않습니다.

이 문제를 고치려면 `Parent` 컴포넌트에서 `Child`까지 프로프스 전달 과정을 검사하고 프로프스 이름을 직접 수정하세요. Components 탭에서 `Child` 컴포넌트를 선택하고 `reply` 프로프스를 `response`으로 변경하면 페이지에 `h2` 요소가 `I'm not here to play!` 텍스트와 함께 표시됩니다.
