## 영구 저장소

- **정의**: 영구 저장소는 전원이 꺼지거나 장치가 재시작된 후에도 데이터가 계속 유지되도록 저장하는 방식을 말합니다.

## 생성, 읽기, 수정, 삭제 (CRUD)

- **생성**: 새로운 데이터를 만드는 과정을 말합니다. 예를 들어, 웹 애플리케이션에서 사용자가 블로그에 새 게시물을 추가할 때가 이에 해당합니다.
- **읽기**: 데이터베이스에서 데이터를 가져오는 작업입니다. 예를 들어, 블로그 게시물을 방문하거나 웹사이트에서 프로필을 볼 때 데이터베이스에 저장된 데이터를 가져와 표시하는 작업입니다.
- **수정**: 데이터베이스에 있는 기존 데이터를 변경하는 작업입니다. 예를 들어, 블로그 게시물을 편집하거나 프로필 정보를 업데이트하는 경우입니다.
- **삭제**: 데이터베이스에서 데이터를 제거하는 작업입니다. 예를 들어, 블로그 게시물이나 계정을 삭제할 때 수행하는 작업입니다.

## HTTP 메서드

- **정의**: HTTP는 하이퍼텍스트 전송 프로토콜로, 웹에서 데이터 통신의 기반이 됩니다. HTTP 메서드는 웹상의 자원에 대해 수행할 수 있는 동작을 정의합니다. 일반적인 메서드는 GET, POST, PUT, PATCH, DELETE입니다.
- **`GET` 메서드**: 서버에서 데이터를 가져오는 데 사용됩니다.
- **`POST` 메서드**: 서버에 데이터를 제출하여 새 자원을 생성하는 데 사용됩니다.
- **`PUT` 메서드**: 자원을 완전히 교체하여 업데이트하는 데 사용됩니다.
- **`PATCH` 메서드**: 자원을 부분적으로 업데이트하는 데 사용됩니다.
- **`DELETE` 메서드**: 데이터베이스에서 레코드를 삭제하는 데 사용됩니다.

## `localStorage`와 `sessionStorage` 속성

- **웹 저장소 API**: 이 API는 브라우저 내에서 키-값 쌍을 저장할 수 있는 메커니즘을 제공하여, 개발자가 페이지 새로 고침이나 세션 간에 사용할 수 있는 정보를 저장할 수 있게 합니다. 웹 저장소 API의 두 가지 주요 컴포넌트는 `localStorage`와 `sessionStorage` 속성입니다.
- **`localStorage` 속성**: `localStorage`은 브라우저 창이 닫히거나 페이지가 새로 고침되어도 데이터가 유지되도록 하는 웹 저장소 API의 일부입니다. 이 데이터는 애플리케이션이나 사용자가 명시적으로 삭제할 때까지 계속 유지됩니다.
- **`localStorage.setItem()` 메서드**: 이 메서드는 `localStorage`에 키-값 쌍을 저장하는 데 사용됩니다.

```js
localStorage.setItem('username', 'Jessica');
```

- **`localStorage.getItem()` 메서드**: 이 메서드는 `localStorage`에서 주어진 키의 값을 가져오는 데 사용됩니다.

```js
localStorage.setItem('username', 'codingRules');

let username = localStorage.getItem('username');
console.log(username); // codingRules
```

- **`localStorage.removeItem()` 메서드**: 이 메서드는 키를 사용하여 `localStorage`에서 특정 항목을 삭제하는 데 사용됩니다.

```js
localStorage.removeItem('username');
```

- **`localStorage.clear()` 메서드**: 이 메서드는 `localStorage`에 저장된 모든 데이터를 지우는 데 사용됩니다.

```js
localStorage.clear();
```

- **`sessionStorage` 속성**: 현재 세션 동안만 유지되며 브라우저 탭이나 창이 닫히면 지워지는 데이터를 저장합니다.
- **`sessionStorage.setItem()` 메서드**: 이 메서드는 `sessionStorage`에 키-값 쌍을 저장하는 데 사용됩니다.

```js
sessionStorage.setItem('cart', '3 items');
```

- **`sessionStorage.getItem()` 메서드**: 이 메서드는 `sessionStorage`에서 주어진 키의 값을 가져오는 데 사용됩니다.

```js
sessionStorage.setItem('cart', '3 items');

let cart = sessionStorage.getItem('cart');
console.log(cart); // '3 items'
```

- **`sessionStorage.removeItem()` 메서드**: 이 메서드는 키를 사용하여 `sessionStorage`에서 특정 항목을 삭제하는 데 사용됩니다.

```js
sessionStorage.removeItem('cart');
```

- **`sessionStorage.clear()` 메서드**: 이 메서드는 `sessionStorage`에 저장된 모든 데이터를 지우는 데 사용됩니다.

```js
sessionStorage.clear();
```

## 쿠키 다루기

- **정의**: 쿠키는 웹 쿠키 또는 브라우저 쿠키라고도 하며, 서버가 사용자의 웹 브라우저에 보내는 작은 데이터 조각입니다. 이 쿠키들은 사용자의 장치에 저장되고 이후 요청 시 서버로 다시 전송됩니다. 쿠키는 HTTP가 무상태 프로토콜인 상황에서 웹 애플리케이션이 상태를 유지하고 사용자 정보를 기억하는 데 필수적입니다.
- **세션 쿠키**: 이 쿠키는 사용자가 웹사이트에서 활동하는 세션 동안만 유지됩니다. 사용자가 브라우저나 탭을 닫으면 세션 쿠키는 삭제됩니다. 주로 방문 중 로그인 상태 유지 같은 작업에 사용됩니다.
- **보안 쿠키**: 이 쿠키는 HTTPS를 통해서만 전송되어 전송 중 공격자가 가로채지 못하도록 보장합니다.
- **HttpOnly 쿠키**: 이 쿠키는 브라우저에서 실행되는 JavaScript가 접근하거나 수정할 수 없어 교차 사이트 스크립팅(XSS) 공격에 더 안전합니다.
- **Set-Cookie 헤더**: 웹사이트를 방문할 때 서버가 HTTP 응답에 Set-Cookie 헤더를 보낼 수 있습니다. 이 헤더는 브라우저에 특정 정보를 가진 쿠키를 저장하도록 지시합니다. 예를 들어, 다음 방문 시 사용자를 인식하는 데 도움이 되는 고유 ID를 저장할 수 있습니다.
JavaScript에서 쿠키를 수동으로 설정하려면 `document.cookie`를 사용하세요:

```js
document.cookie = "organization=freeCodeCamp; expires=Fri, 31 Dec 2021 23:59:59 GMT; path=/";
```

쿠키를 삭제하려면 만료 날짜를 과거로 설정하면 됩니다.

```js
document.cookie = "username=; expires=Thu, 01 Jan 1970 00:00:00 GMT; path=/";
```

## 캐시 API

- **정의**: 캐싱은 파일 복사본을 임시 저장소에 저장하여 더 빠르게 접근할 수 있도록 하는 과정입니다. 캐시 API는 네트워크 요청과 응답을 저장하여 웹 애플리케이션이 더 효율적으로 작동하고 오프라인에서도 기능할 수 있게 합니다. 이는 서비스 워커 API의 일부이며, 불안정하거나 느린 네트워크 환경에서 작동하는 프로그레시브 웹 앱(PWA)을 만드는 데 중요합니다.
캐시 API는 Request와 Response 객체를 저장하는 저장소 메커니즘입니다. 서버에 요청을 보낼 때 응답을 저장해 두었다가 나중에 새 네트워크 요청 대신 캐시에서 가져올 수 있습니다. 이를 통해 로드 시간이 줄고 대역폭이 절약되며 사용자 경험이 향상됩니다.
- **캐시 저장소**: HTTP 요청과 해당 응답의 키-값 쌍을 저장하는 데 사용됩니다. 이를 통해 이전에 요청한 자원을 효율적으로 다시 가져올 수 있어, 이후 방문 시 네트워크에서 다시 가져올 필요를 줄이고 성능을 개선합니다.
- **Cache-Control**: 개발자가 캐시된 자원을 얼마나 오래 보관할지, 재검증할지 아니면 캐시에서 바로 제공할지 정의할 수 있습니다.
- **오프라인 지원**: 캐시 API를 사용하면 오프라인 우선 웹 애플리케이션을 만들 수 있습니다. 예를 들어, PWA는 사용자가 네트워크에 연결되어 있지 않을 때 캐시된 자산을 제공할 수 있습니다.

## 부정적 패턴과 클라이언트 측 저장소

- **과도한 추적**: 명확한 동의나 정당한 필요 없이 클라이언트 측 저장소(쿠키, 로컬 저장소, 세션 저장소 등)에 과도한 사용자 데이터를 수집하고 저장하는 행위를 말합니다. 이는 여러 사이트나 세션에 걸쳐 사용자 행동, 선호도, 상호작용을 추적하여 사용자 프라이버시를 침해할 수 있습니다.
- **브라우저 지문 추적**: 쿠키나 전통적 추적 방법 대신 사용자의 장치와 브라우저 고유 특성을 기반으로 개별 사용자를 추적하고 식별하는 기법입니다. 쿠키가 사용자의 장치에 로컬로 저장되는 반면, 지문 추적은 사용자의 브라우저 세션을 구별하는 독특한 "지문"을 만들기 위해 다양한 정보를 수집합니다.
- **로컬 저장소에 비밀번호 저장**: 명백한 부정적 패턴으로 보일 수 있지만, 비밀번호 같은 민감한 데이터를 로컬 저장소에 저장하는 것은 보안 위험이 큽니다. 로컬 저장소는 암호화되지 않고 쉽게 접근할 수 있으므로 민감한 데이터를 절대 저장하지 말아야 합니다.

## IndexedDB

- **정의**: IndexedDB는 브라우저에서 구조화된 데이터를 저장하는 데 사용됩니다. 최신 웹 브라우저에 내장되어 있어 웹 애플리케이션이 JavaScript 객체를 효율적으로 저장하고 가져올 수 있게 합니다.

## 캐시/서비스 워커

- **정의**: 서비스 워커는 웹 페이지와 별도로 백그라운드에서 실행되는 스크립트입니다. 네트워크 요청을 가로채고 캐시에 접근하며 웹 앱이 오프라인에서도 작동하도록 만듭니다. 이는 프로그레시브 웹 앱의 핵심 컴포넌트입니다.
