# --description--

HTML은 매우 관대한 언어입니다 – 닫는 태그를 빼먹는 등 실수를 해도 요소가 여전히 렌더링됩니다.

닫는 태그가 없는 `h2` 요소가 있다고 가정해 보세요:

```html
<h1>Article Topic</h1>
<h2>Subheading 1 </h2>
<h2>Subheading 2 </h2>

<!-- This h2 does not have a closing tag -->
<h2>Subheading 3
```

닫는 태그가 없는 `h2`도 여전히 잘 렌더링됩니다. 이는 브라우저가 일반적인 오류를 처리하고 작성자의 의도에 최대한 가깝게 HTML을 렌더링하려고 하는 파싱 알고리즘을 사용하기 때문입니다.

하지만 때로는 이런 점이 문제를 일으킬 수 있습니다. 코드에서 기존 제목 2 태그 아래에 몇 개의 문단을 추가해 보겠습니다:

```html
<h1>Article Topic</h1>
<p>Lorem ipsum dolor sit amet consectetur adipisicing elit. Maiores, nisi.</p>

<h2>Subheading 1 </h2>
<p>Lorem, ipsum dolor sit amet consectetur adipisicing elit. At, doloremque.</p>

<h2>Subheading 2 </h2>
<p>Lorem ipsum dolor sit amet consectetur adipisicing elit. Unde, placeat.</p>

<!-- This h2 does not have a closing tag -->
<h2>Subheading 3
<p>Lorem ipsum dolor, sit amet consectetur adipisicing elit. Tempore, illum.</p>
```

그 결과, 닫는 `h2` 태그가 없는 `h2` 아래의 문단 요소가 제목 2로 렌더링됩니다. 그래서 HTML 유효성 검사기가 필요합니다.

HTML 유효성 검사기는 HTML 코드를 표준 HTML 명세에 따라 검사하는 도구입니다. HTML 코드의 오류와 경고를 찾아내어 웹 페이지가 올바르게 구조화되고 웹 표준을 준수하도록 도와줍니다.

HTML 유효성 검사기를 사용하면 본인과 미래의 코드 검토자뿐 아니라 팀원이나 오픈소스 기여자 등 코드를 보는 모든 사람에게도 도움이 됩니다.

사용할 수 있는 여러 HTML 유효성 검사기가 있습니다. 가장 널리 인정받는 것은 `w3.org` 마크업 유효성 검사 서비스입니다.

사이트 [`validator.w3.org`](https://validator.w3.org/)에 방문하면 `Validate by Direct Input` 버튼을 클릭하고 HTML 코드를 붙여넣을 수 있습니다.

`Check` 버튼을 클릭하면 수정해야 할 오류 목록이 표시됩니다.

또 다른 HTML 유효성 검사기로는 [`jsonformatter.org`](https://jsonformatter.org/)이 있습니다.

첫 번째 편집기에 HTML 코드를 복사해 붙여넣고 `Validate` 버튼을 클릭하면 코드 내 오류를 확인할 수 있습니다.
