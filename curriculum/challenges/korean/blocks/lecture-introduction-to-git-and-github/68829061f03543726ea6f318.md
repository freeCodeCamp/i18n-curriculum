# --description--

Git과 GitHub에 대해 조금 이야기했으니 이제 이를 모두 연결할 수 있습니다. 이를 위해서는 저장소가 필요합니다. 그런데 저장소란 무엇일까요? 저장소는 프로젝트를 담는 컨테이너라고 생각할 수 있습니다. 앱을 작업한다면, 그 앱의 파일들을 저장소에 함께 보관합니다.

저장소는 컴퓨터에 로컬로 있거나 GitHub 같은 서비스에 원격으로 있을 수 있습니다. 저장소를 만들려면 로컬이나 원격 중 어느 쪽에서 시작해도 결과는 같습니다. 따라서 작업 흐름에 가장 적합한 방식을 선택하세요. 원격 저장소에서 시작하는 작업 흐름이라면, GitHub 웹사이트에서 직접 저장소를 만들 수 있습니다. 다음 지침을 따라주세요:

<a href="https://github.com" target="_blank">GitHub 웹사이트</a>에 방문해 로그인한 후, 네비게이션 바 오른쪽 상단의 더하기 아이콘을 클릭하세요. 여러 옵션이 보이지만, 여기서 중요한 것은 "New Repository"입니다. 이를 선택하면 새 저장소를 만드는 UI가 열립니다.

처음 보게 될 옵션은 템플릿을 선택하는 것입니다. 아직 템플릿을 설정하지 않았을 가능성이 높지만, 템플릿은 새 저장소를 시작할 때 활용할 수 있는 특별한 저장소 유형입니다. 모든 프로젝트에 문서나 이슈 템플릿을 포함할 때 유용합니다.

다음 옵션은 소유자를 선택하는 것입니다. 사용자 계정이나 소유하거나 접근 권한이 있는 조직을 선택할 수 있습니다. 그리고 저장소 이름을 정하세요. 일반적으로 프로젝트를 간결하게 설명하는 이름을 사용하는 것이 좋습니다. 모범 사례로 저장소 이름에 공백을 피하는 것이 좋으며, GitHub가 자동으로 하이픈으로 대체합니다.

그다음으로는 선택 사항으로 저장소 홈페이지에 표시될 설명을 추가할 수 있습니다.

저장소를 공개할지 비공개로 할지도 선택할 수 있습니다. 공개 저장소는 누구나 보고 다운로드할 수 있습니다. 비공개 저장소는 본인과 명시적으로 접근 권한을 부여한 사람만 접근할 수 있습니다.

마지막으로 템플릿을 선택하지 않으면 저장소에 자동으로 생성할 파일 옵션이 나타납니다. README 파일은 저장소 홈페이지에 표시되며 프로젝트 문서와 설명을 제공하는 좋은 방법입니다. `.gitignore` 파일은 Git이 "무시"할 파일을 지정할 수 있게 해줍니다. 이는 패키지 의존성, 환경 파일 등 저장하고 싶지 않은 항목에 적합합니다. GitHub는 지정한 언어나 런타임에 맞는 일반적인 항목을 제외하는 템플릿을 제공합니다. 라이선스는 소프트웨어를 누가 어떻게 사용할 수 있는지 법적으로 명시하는 문서입니다.

설정을 모두 마쳤으면 "Create Repository" 버튼을 클릭해 과정을 완료하세요. 그러면 새 저장소의 랜딩 페이지로 자동 리다이렉트됩니다!

여기서 "Code" 버튼을 클릭하면 저장소를 컴퓨터에 클론할 수 있는 옵션이 나타납니다.

이번 수업에서는 "Codespaces"와 "Copilot" 탭은 무시하세요. 대신 "Local" 탭의 세 가지 옵션에 집중할 것입니다. 첫 번째 두 가지인 HTTPS와 SSH는 같은 방식으로 사용합니다. `git clone` 명령어와 상자에 제공된 URL을 사용해 원격 저장소를 컴퓨터에 클론합니다. 두 가지의 차이는 연결 및 인증 방식입니다.

공개 저장소를 클론할 때는 실제로 인증이 필요하지 않습니다. 하지만 비공개 저장소를 클론하거나 공개 또는 비공개 저장소에 푸시하려면 인증이 필요합니다. HTTPS 원격 URL은 GitHub 사용자 이름과 비밀번호로 인증하지만, 보안 문제로 인해 더 이상 지원되지 않고 사용자 이름과 액세스 토큰을 사용해야 합니다. 그럼에도 SSH 방식이 훨씬 안전하며, 개인 키와 공개 키를 사용합니다. SSH는 매번 토큰을 제공할 필요 없이 자동으로 로그인되는 추가 이점이 있습니다. SSH 작동 방식에 대해 더 깊이 다루는 별도의 수업이 있으며, GitHub 문서에서 SSH 설정 방법도 확인할 수 있습니다.

저장소를 클론하려면 터미널에서 다음 명령어를 실행하세요:

```sh
git clone git@github.com:your-username/your-repository-name
```

이 명령어는 컴퓨터에 저장소 이름의 폴더를 만들고 그 안에 저장소의 모든 파일을 포함합니다. 그다음 `cd` (디렉터리 변경) 명령어로 클론한 저장소로 이동할 수 있습니다:

```sh
cd name-of-cloned-repo
```

또한 로컬 저장소 복사본에는 GitHub의 저장소를 가리키는 원격이 설정되어 있습니다. 원격을 확인하려면 다음 명령어를 실행하세요:

```sh
git remote -v
```

또 다른 클론 방법은 GitHub CLI를 사용하는 것입니다. 이 도구는 명령줄을 벗어나지 않고 GitHub 관련 작업을 할 수 있게 해줍니다. 설치되어 있지 않다면 GitHub 문서에서 설치 방법을 확인할 수 있으며, 시스템 패키지 관리자에서 사용할 수 있을 것입니다.

CLI 도구로 저장소를 클론하려면 다음 명령어를 사용하세요:

```sh
gh repo clone your-username/your-repository-name
```

이 명령어들의 결과는 동일합니다: 로컬에 저장소 복사본이 생깁니다.

GitHub CLI를 사용해 저장소를 만들 수도 있습니다:

```sh
gh repo create
```

인수를 전달하지 않으면 CLI가 대화형 마법사를 실행해 저장소 설정을 도와줍니다. 이 마법사는 웹 UI와 동일한 옵션을 제공합니다. 이 도구를 사용하면 새 저장소를 즉시 클론할 수도 있어 앞서 설명한 수동 단계를 거치지 않아도 됩니다.

이제 저장소가 어떻게 작동하는지, 그리고 작업을 제대로 버전 관리하기 위해 어떻게 저장소를 만드는지 기본적으로 이해했습니다. 앞으로 수업에서는 기존 로컬 저장소를 GitHub에 푸시하는 방법과 원격 저장소에 변경 사항을 푸시하는 방법을 배우게 됩니다.
