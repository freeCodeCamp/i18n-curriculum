# --description--

もしまだ気づいていなければ、前のチャレンジの問題は`splice`関数内の`tabClose()`呼び出しにありました。残念ながら、`splice`は呼び出された元の配列を変更するため、2回目の呼び出しでは変更された配列が使われ、予期しない結果になりました。

これはもっと大きなパターンの小さな例です。変数や配列、オブジェクトに対して関数を呼び出すと、その関数が変数やオブジェクトの何かを変更してしまうことがあります。

関数型プログラミングの核心的な原則の一つは、物事を変更しないことです。変更はバグにつながります。関数が引数やグローバル変数を含めて何も変更しないとわかっていれば、バグを防ぎやすくなります。

前の例では複雑な操作はありませんでしたが、`splice`メソッドが元の配列を変更し、バグが発生しました。

関数型プログラミングでは、物事を変更・変化させることを<dfn>ミューテーション</dfn>と呼び、その結果を<dfn>副作用</dfn>と呼びます。理想的には、関数は<dfn>純粋な関数</dfn>であり、副作用を起こさないことが求められます。

この規律を身につけて、コード内の変数やオブジェクトを変更しないようにしましょう。

# --instructions--

関数`incrementer`のコードを完成させて、グローバル変数`fixedValue`の値に1を足した値を返すようにしてください。

# --hints--

関数`incrementer`は`fixedValue`（これは`4`です）の値を変更してはいけません。

```js
incrementer();
assert(fixedValue === 4);
```

関数`incrementer`は`fixedValue`の値より1大きい値を返すべきです。

```js
const __newValue = incrementer();
assert(__newValue === 5);
```

関数`incrementer`はグローバル変数`fixedValue`の値に基づいた値を返すべきです。

```js
(function () {
  fixedValue = 10;
  const newValue = incrementer();
  assert(fixedValue === 10 && newValue === 11);
  fixedValue = 4;
})();
```
