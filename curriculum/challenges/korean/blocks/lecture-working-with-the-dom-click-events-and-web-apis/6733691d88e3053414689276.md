# --interactive--

이전 수업에서 `addEventListener()` 메서드를 사용해 이벤트를 다루는 방법을 배웠습니다. 하지만 JavaScript에서 이벤트를 다루는 또 다른 방법이 있는데, 이는 권장되지 않습니다.

인라인 이벤트 핸들러는 이벤트가 발생할 때 JavaScript 코드를 실행하기 위해 HTML 요소에 사용하는 특별한 속성입니다.

다음은 인라인 클릭 이벤트 핸들러가 있는 `button` 요소의 예입니다:

:::interactive_editor

```html
<button onclick="alert('Hello World!')">Show alert</button>
```

:::

사용자가 버튼을 클릭하면 `alert` 함수가 호출되고 메시지 `Hello World!`가 포함된 경고 대화상자가 표시됩니다.

인라인 이벤트 핸들러를 사용하는 또 다른 방법은 HTML 문서의 `script` 태그에 정의된 함수를 호출하는 것입니다.

다음은 `changeBgColor`라는 함수를 정의하고 인라인 클릭 이벤트 핸들러에서 호출하는 예입니다:

:::interactive_editor

```html
<script>
  function changeBgColor() {
    document.body.style.backgroundColor = "lightblue";
  }
</script>

<button onclick="changeBgColor()">Change background color</button>
```

:::

사용자가 버튼을 클릭하면 `changeBgColor` 함수가 호출되고 페이지의 배경색이 연한 파란색으로 변경됩니다. 이렇게 인라인 이벤트 핸들러를 사용할 수는 있지만, 모범 사례로 간주되지는 않습니다.

첫째, 인라인 이벤트 핸들러는 요소에 하나의 이벤트 리스너만 연결할 수 있습니다. 동일한 요소에 여러 이벤트 리스너를 연결하려면 `addEventListener()`를 사용해야 합니다. 또 다른 이유는 인라인 이벤트 핸들러가 HTML과 JavaScript 코드를 섞어 작성하기 때문에 코드가 읽기 어렵고 유지 관리하기 힘들어질 수 있습니다. HTML 코드와 JavaScript 코드를 분리하여 `addEventListener()`를 사용해 이벤트 리스너를 요소에 연결하는 것이 더 좋습니다.

인라인 이벤트 핸들러는 현대 JavaScript에서 권장되지 않습니다. 따라서 JavaScript에서 이벤트를 다룰 때는 `addEventListener()` 메서드를 사용하는 것이 모범 사례입니다.
