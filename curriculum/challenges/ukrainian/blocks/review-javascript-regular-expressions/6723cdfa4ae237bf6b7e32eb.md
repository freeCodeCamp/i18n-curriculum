# --interactive--

## Регулярні вирази та поширені методи

- **Визначення**: регулярні вирази, або regex, використовують для створення «шаблону», який потім можна застосувати для перевірки рядка, вилучення тексту та іншого.

```js
const regex = /freeCodeCamp/;
```

- **Метод `test()`**: цей метод приймає рядок, який потрібно перевірити на відповідність регулярному виразу. Метод поверне булеве значення, якщо рядок відповідає регулярному виразу.

:::interactive_editor

```js
const regex = /freeCodeCamp/;
const test = regex.test("e");
console.log(test); // false
```

:::

- **Метод `match()`**: цей метод приймає регулярний вираз, хоча можна також передати рядок, який буде перетворений на регулярний вираз. Метод `match` повертає масив збігів для рядка.

```js
const regex = /freeCodeCamp/;
const match = "freeCodeCamp".match(regex);
console.log(match); // ['freeCodeCamp', index: 0, input: 'freeCodeCamp', groups: undefined]
```

- **Метод `replace()`**: цей метод приймає два аргументи: регулярний вираз для пошуку (або рядок) і рядок для заміни збігу (або функцію, яку потрібно виконати для кожного збігу).

:::interactive_editor

```js
const regex = /Jessica/;
const str = "Jessica is rly kewl";
const replaced = str.replace(regex, "freeCodeCamp");
console.log(replaced); // "freeCodeCamp is rly kewl"
```

:::

- **Метод `replaceAll`**: цей метод використовується для заміни всіх входжень заданого шаблону на новий рядок. Якщо передати регулярний вираз без глобального модифікатора, то метод викличе помилку.

:::interactive_editor

```js
const text = "I hate JavaScript! I hate programming!";
const newText = text.replaceAll("hate", "love");
console.log(newText);  // "I love JavaScript! I love programming!"
```

:::

- **Метод `matchAll`**: цей метод використовується для отримання всіх збігів заданого регулярного виразу в рядку, включно з групами захоплення, і повертає їх у вигляді ітератора. Ітератор — це об’єкт, який дозволяє пройтися (або «ітерувати над») колекцією елементів.

:::interactive_editor

```js
const str = "JavaScript, Python, JavaScript, Swift, JavaScript";
const regex = /JavaScript/g;

const iterator = str.matchAll(regex);

for (let match of iterator) {
  console.log(match[0]); // "JavaScript" for each match
}
```

:::

## Модифікатори регулярних виразів

- **Визначення**: модифікатори, які часто називають «прапорцями», змінюють поведінку регулярного виразу.  
- **Прапорець `i`**: цей прапорець змушує регулярний вираз ігнорувати регістр.

:::interactive_editor

```js
const regex = /freeCodeCamp/i;
console.log(regex.test("freecodecamp")); // true
console.log(regex.test("FREECODECAMP")); // true
```

:::

- **Прапорець `g`**: цей прапорець, або глобальний модифікатор, дозволяє регулярному виразу знаходити шаблон більше одного разу.

:::interactive_editor

```js
const regex = /freeCodeCamp/gi;
console.log(regex.test("freeCodeCamp")); // true
console.log(regex.test("freeCodeCamp is great")); // false
```

:::

- **Визначення якоря**: якір `^` на початку регулярного виразу означає «збіг з початком рядка». Якір `$` в кінці регулярного виразу означає «збіг з кінцем рядка».

:::interactive_editor

```js
const start = /^freeCodeCamp/i;
const end = /freeCodeCamp$/i;
console.log(start.test("freecodecamp")); // true
console.log(end.test("freecodecamp")); // true
```

:::

- **Прапорець `m`**: Якорі шукають початок і кінець усього рядка. Але ви можете змусити регулярний вираз обробляти кілька рядків за допомогою прапорця `m`, або багаторядкового модифікатора.

:::interactive_editor

```js
const start = /^freecodecamp/im;
const end = /freecodecamp$/im;
const str = `I love 
freecodecamp
it's my favorite
`;
console.log(start.test(str)); // true
console.log(end.test(str)); // true
```

:::

- **Прапорець `d`**: цей прапорець розширює інформацію, яку ви отримуєте в об’єкті збігу.

```js
const regex = /freecodecamp/di;
const string = "we love freecodecamp isn't freecodecamp great?";
console.log(string.match(regex));
```

І результат:

```js
// [
//   'freecodecamp',
//   index: 8,
//   input: "we love freecodecamp isn't freecodecamp great?",
//   groups: undefined,
//   indices: [
//     0: [8, 20],
//     groups: undefined
//   ]
// ]
```

- **`u` прапорець**: розширює функціональність регулярного виразу, дозволяючи йому співпадати зі спеціальними символами Unicode. Прапорець `u` дає доступ до спеціальних класів, таких як `Extended_Pictographic`, для співпадіння з більшістю емодзі. Існує також прапорець `v`, який ще більше розширює можливості співпадіння Unicode.
- **`y` прапорець**: модифікатор sticky поводиться дуже схоже на глобальний модифікатор, але з кількома винятками. Найголовніший полягає в тому, що глобальний регулярний вираз починає пошук з lastIndex і шукає співпадіння в усьому залишку рядка, а sticky регулярний вираз повертає null і скидає lastIndex до 0, якщо одразу немає співпадіння на попередньому lastIndex.
- **`s` прапорець**: модифікатор single-line дозволяє символу підстановки, який у регулярних виразах позначається як `.`, співпадати з розривами рядків — фактично розглядаючи рядок як один єдиний рядок тексту.

## Класи символів

- **Універсальний символ `.`**: класи символів — це спеціальний синтаксис, який ви можете використовувати, щоб співпадати з множинами або підмножинами символів. Перший клас символів, який варто вивчити, — це універсальний символ. Універсальний символ позначається крапкою і співпадає з БУДЬ-ЯКИМ одним символом, ОКРІМ розривів рядків. Щоб дозволити універсальному символу співпадати з розривами рядків, пам’ятайте, що потрібно використовувати прапорець `s`.

```js
const regex = /a./;
```

- **`\d`**: це співпаде з усіма цифрами (`0-9`) у рядку.

```js
const regex = /\d/;
```

- **`\w`**: Це використовується для пошуку будь-якого символу слова (`a-zA-Z0-9_`) у рядку. Символ слова визначається як будь-яка літера від a до z або від A до Z, або число від 0 до 9, або символ підкреслення.

```js
const regex = /\w/;
```

- **`\s`**: Клас пробільних символів `\s` позначається зворотною скісною рискою, за якою йде `s`. Цей клас символів відповідає будь-якому пробільному символу, включно з новими рядками, пробілами, табуляціями та спеціальними пробілами Unicode.
- **Заперечення спеціальних класів символів**: Щоб заперечити один із цих класів символів, замість використання малої літери після зворотної скісної риски, можна використати відповідну велику літеру. Наступний приклад не відповідає числовому символу. Натомість він відповідає будь-якому одному символу, який НЕ є числовим.

```js
const regex = /\D/;
```

- **Користувацькі класи символів**: Ви можете створювати користувацькі класи символів, розміщуючи символи, які хочете співставити, всередині квадратних дужок.

```js
const regex = /[abcdf]/;
```

## Твердження випередження та відставання

- **Визначення**: твердження випередження та відставання дозволяють співставляти конкретні шаблони залежно від наявності або відсутності навколишніх шаблонів.
- **Позитивне твердження випередження**: це твердження співставить шаблон, якщо за ним слідує інший шаблон. Щоб створити позитивне твердження випередження, потрібно почати з шаблону, який ви хочете співставити. Потім використайте круглі дужки, щоб обгорнути шаблон, який ви хочете використати як умову. Після відкриваючої дужки використайте `?=`, щоб визначити цей шаблон як позитивне твердження випередження.

```js
const regex = /free(?=code)/i;
```

- **Негативне твердження випередження**: це тип умови, що використовується в регулярних виразах для перевірки, що певний шаблон не зустрічається попереду в рядку.

```js
const regex = /free(?!code)/i;
```

- **Позитивне твердження відставання**: це твердження співставить шаблон лише якщо перед ним стоїть інший конкретний шаблон, не включаючи цей попередній шаблон у співставлення.

```js
const regex = /(?<=free)code/i;
```

- **Негативне твердження відставання**: це твердження гарантує, що шаблон не передує іншому конкретному шаблону. Воно співставляє лише якщо вказаний шаблон не безпосередньо передує заданій послідовності, не включаючи цю послідовність у співставлення.

```js
const regex = /(?<!free)code/i;
```

## Квантори регулярних виразів

- **Визначення**: квантори в регулярних виразах вказують, скільки разів шаблон (або його частина) має з’явитися. Вони допомагають контролювати кількість входжень символів або груп у співпадінні. Наступний приклад використовується, щоб знайти попередній символ рівно чотири рази.

```js
const regex = /^\d{4}$/;
```

- **`*`** : збігається з 0 або більше входженнями попереднього елемента.
- **`+`**: збігається з 1 або більше входженнями попереднього елемента.
- **`?`**: збігається з 0 або 1 входженням попереднього елемента.
- **`{n}`**: збігається рівно з n входженнями попереднього елемента.
- **`{n,}`**: збігається з n або більше входженнями попереднього елемента.
- **`{n,m}`**: збігається з кількістю входжень попереднього елемента від n до m.

## Групи захоплення та референси

- **Групи захоплення**: група захоплення дозволяє «захопити» частину знайденого рядка, щоб використовувати її так, як вам потрібно. Групи захоплення визначаються круглими дужками, які містять шаблон для захоплення, без провідних символів, як-от випередження.

```js
const regex = /free(code)camp/i;
```

- **Некатчинг-групи**: Некатчинг-група схожа на групу захоплення, але не зберігає знайдену частину рядка для подальшого використання. Некатчинг-групи визначаються `(?:...)`.

```js
const regex = /free(?:code)camp/i;
```

- **Референси**: референс у регулярних виразах — це спосіб повторно використати частину шаблону, яка була знайдена раніше в тому ж виразі. Він дозволяє посилатися на групу захоплення (частину шаблону в круглих дужках) за її номером. Наприклад, `$1` посилається на першу групу захоплення.

:::interactive_editor

```js
const regex = /free(co+de)camp/i;
console.log("freecoooooooodecamp".replace(regex, "paid$1world"));
```

:::

- Ви можете використовувати зворотні посилання всередині самого regex, щоб знайти той самий текст, захоплений попередньою групою, за допомогою зворотної скісної риски та номера групи. Наприклад:

:::interactive_editor

```js
const regex = /(hello) \1/i;
console.log(regex.test("hello hello")); // true
console.log(regex.test("hello world")); // false
```

:::
