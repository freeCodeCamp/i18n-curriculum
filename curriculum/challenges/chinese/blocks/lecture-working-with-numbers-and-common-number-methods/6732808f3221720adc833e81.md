# --interactive--

`parseFloat()` 和 `parseInt()` 是 JavaScript 中两种将字符串转换为数字的基本方法。 这些方法在处理用户输入或以字符串格式提供但需要作为数值处理的数据时特别有用。

让我们从 `parseFloat()` 开始。 该方法解析字符串参数并返回浮点数。 它的设计目的是从字符串开头提取数字，即使字符串后面包含非数字字符。 请记住，浮点数是带有小数点的数字。 下面是 `parseFloat()` 的工作原理：

:::interactive_editor

```js
console.log(parseFloat("3.14"));     // 3.14
console.log(parseFloat("3.14 abc")); // 3.14
console.log(parseFloat("3.14.5"));   // 3.14
console.log(parseFloat("abc 3.14")); // NaN
```

:::

如你所见，`parseFloat()` 从字符串的开头开始解析，直到遇到不属于浮点数的字符为止。 如果在字符串开头找不到有效的数字，则返回 `NaN` （非数字）。

而 `parseInt()` 会解析一个字符串参数并返回一个整数。 与 `parseFloat()` 类似，它从字符串的开头开始，在第一个非数字字符处停止。 下面是 `parseInt()` 的工作原理：

:::interactive_editor

```js
console.log(parseInt("42"));       // 42
console.log(parseInt("42px"));     // 42
console.log(parseInt("3.14"));     // 3
console.log(parseInt("abc123"));   // NaN
```

:::

`parseInt()` 在遇到第一个非数字时停止解析。 对于浮点数，它只返回整数部分。 如果在字符串开头找不到有效整数，则返回 `NaN` 。

这两种方法都有一些值得注意的行为。 它们会忽略前导空白：

:::interactive_editor

```js
console.log(parseFloat("  3.14"));  // 3.14
console.log(parseInt("  42"));      // 42
```

:::

它们可以处理字符串开头的加号和减号：

:::interactive_editor

```js
console.log(parseFloat("+3.14"));  // 3.14
console.log(parseInt("-42"));      // -42
```

:::

值得注意的是，这些方法虽然功能强大，但也有一定的局限性。 例如，它们不能直接处理所有数字格式，如科学记数法。 对于更复杂的解析需求，可能需要使用其他技术或库。

总之，`parseFloat()` 和 `parseInt()` 是 JavaScript 中将字符串转换为数字的重要工具。 了解了它们的工作原理和具体行为，就能在应用程序中更有效地处理数字数据，尤其是在处理用户输入或外部数据源时。
