# --interactive--

在之前的课程中，你首次接触了布尔的概念，但在本课中，我们将更深入地探讨布尔的工作原理以及相等和不相等操作符的工作方式。

布尔是一种只有 `true` 和 `false` 值的数据类型。  They're useful because they allow you to do something based on some conditions. Booleans are essential when you want to evaluate whether something should happen or not, like deciding if someone can access a certain feature in your app. 下面是一个示例，说明如何为名为 `isOldEnoughToDrive` 的变量设置 `true` 值：

:::interactive_editor

```js
let isOldEnoughToDrive = true;

console.log(isOldEnoughToDrive); // true
```

:::

你可以像这样在条件中使用这个变量：

:::interactive_editor

```js
let isOldEnoughToDrive = true;

if (isOldEnoughToDrive) {
 console.log("You're old enough to drive"); // You're old enough to drive
} else {
 console.log("Sorry, you are not old enough to drive");
}
```

:::

条件语句帮助你在代码中根据条件做出决定。 此示例使用了的 `if/else` 语句。

如果 `isOldEnoughToDrive` 是 `true`，则 `You're old enough to drive` 将被记录到控制台。 相反，如果 `isOldEnoughToDrive` 是 `false`，那么 `Sorry, you are not old enough to drive` 将被记录到控制台。 由于 `isOldEnoughToDrive` 变量设置为 `true`，第一句话将被记录到控制台。  You will learn more about `if/else` statements in a future lesson.

要比较两个值，可以使用相等运算符或严格相等运算符。 比较结果将是布尔值 `true` 或 `false`。 下面是一个使用相等运算符比较字符串和数字的示例。 相等运算符用双等号表示（`==`）。

:::interactive_editor

```js
console.log(5 == "5"); // true
```

:::

在本例中，JavaScript 将字符串 `"5"` 转换为数字 `5`，然后检查它们是否相等。 由于现在两个值相同，结果为真。 在检查每个值是否相等之前，相等运算符会使用类型转换。

这与严格相等运算符不同，严格相等运算符不执行类型转换。 严格相等运算符将同时检查类型和值是否相同。 下面是一个使用严格相等运算符比较数字和字符串的示例。 该运算符用三个等号表示（`===`）。

:::interactive_editor

```js
console.log(5 === '5'); // false
```

:::

下面的比较结果将是 `false` ，因为字符串数据类型与数字数据类型不同。 如果需要检查某个值是否不等于另一个值，可以使用不相等或严格不相等运算符。 下面是一个使用不相等运算符（`!=`）比较数字和字符串的示例。 

:::interactive_editor

```js
console.log(5 != "5"); // false
```

:::

在这个示例中，结果将是 `false`，因为不相等运算符首先将字符串值转换为数字，然后再比较值。 由于值相同，因此将返回 `false` 。 如果尝试使用严格不相等运算符，则会得到不同的结果。 严格不相等运算符用感叹号表示，后面跟两个等号（`!==`）。

:::interactive_editor

```js
console.log(5 !== "5"); // true
```

:::

由于严格不相等操作符不执行任何类型转换，因此结果将是 `true` 。 因为数字 `5` 不等于字符串 `"5"`，那么结果就是 `true` 。

最佳实践是尽可能使用严格不相等和严格相等运算符，因为它们不执行类型转换。 在大多数专业项目中，你会发现代码库通常更喜欢这两个运算符，而不是不相等和相等运算符。
