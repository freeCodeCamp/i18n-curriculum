# --description--

웹 애플리케이션을 구축할 때, 백엔드 서버는 단순히 데이터를 보내는 것뿐만 아니라 HTTP 응답 헤더 형태로 브라우저에 숨겨진 정보도 함께 보냅니다.

기본적으로 Express 설정은 서버 기술에 대한 세부 정보를 노출합니다. 공격자는 이러한 기본 헤더를 이용해 애플리케이션의 취약점을 찾을 수 있습니다. 그래서 개발자들이 Helmet.js를 사용합니다.

Helmet.js는 Node.js 애플리케이션을 위한 보안 미들웨어 모음입니다. 특정 HTTP 헤더를 자동으로 설정, 수정 또는 제거하여 보호막 역할을 합니다.

이 헤더들을 관리함으로써 Helmet은 애플리케이션을 일반적인 웹 취약점, 악성 스크립트, 데이터 스니핑 공격으로부터 보호하는 데 도움을 줍니다.

수십 개의 보안 프로토콜을 수동으로 설정하는 대신, Helmet은 기본적으로 여러 작은 미들웨어 함수를 구성합니다. 설치는 단 한 줄의 터미널 명령어로 가능합니다:

```bash
npm install helmet
```

이 보안 보호막을 전체 Express 애플리케이션에 적용하려면 단 한 줄의 코드만 필요합니다:

```js
const express = require('express');
const helmet = require('helmet');

const app = express();

// apply Helmet security middleware
app.use(helmet());
```

미들웨어 스택 맨 위에 `app.use(helmet())`를 배치하면, Helmet은 모든 나가는 응답을 가로채서 HTTP 헤더를 안전하게 설정합니다.

Helmet을 초기화하면 여러 중요한 보안 작업을 자동으로 처리합니다. 여기 가장 중요한 설정들이 있습니다:

**서버 기술 숨기기 (X-Powered-By)**: 기본적으로 Express는 `X-Powered-By: Express` 헤더를 보내 자신을 광고합니다. 이는 해커에게 백엔드가 어떤 기술을 사용하는지 알려주어 버전별 버그를 노릴 수 있게 합니다. Helmet은 이 헤더를 완전히 제거해 공격자가 내부 구성을 추측하게 만듭니다.

**크로스 사이트 스크립팅 완화 (Content-Security-Policy)**: 콘텐츠 보안 정책(CSP)은 크로스 사이트 스크립팅(XSS) 공격에 대한 가장 강력한 방어 중 하나입니다. Helmet은 브라우저가 스크립트, 이미지, 스타일을 어디서 로드할 수 있는지 엄격한 규칙을 설정합니다. 이를 통해 악성 주입자가 사용자 브라우저에서 무단 스크립트를 실행하지 못하게 합니다.

**클릭재킹 방지 (X-Frame-Options)**: 클릭재킹은 악성 사이트가 사용자의 웹사이트를 보이지 않는 iframe 안에 임베드해 사용자가 의도하지 않은 버튼을 클릭하게 만드는 공격입니다. Helmet은 `X-Frame-Options` 헤더를 `SAMEORIGIN`으로 설정해 외부 신뢰할 수 없는 도메인에 웹사이트가 임베드되지 않도록 합니다.

**보안 연결 강제 (Strict-Transport-Security)**: Helmet은 브라우저가 서버와 통신할 때 안전하지 않은 HTTP 대신 오직 안전한 HTTPS 연결만 사용하도록 강제합니다. 이는 같은 네트워크 내 공격자가 데이터를 가로채는 것을 방지합니다.

요약하자면, Helmet은 프로덕션 애플리케이션에 필수적인 방어층을 제공합니다. 사용자 정의 코드의 버그를 고치지는 않지만, 자동 스캐너와 일반적인 공격 기법에 대해 서버 구성을 즉시 강화합니다.
