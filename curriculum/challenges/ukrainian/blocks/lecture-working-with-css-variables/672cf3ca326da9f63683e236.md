# --interactive--

Правило `@property` — це потужна функція CSS, яка дозволяє розробникам оголошувати кастомні властивості з більшим контролем над їхньою поведінкою, включно з тим, як вони анімуються, та їхніми ініціальними значеннями.

Це правило надає спосіб розширити функціональність кастомних властивостей CSS і пропонує більше гнучкості в їхньому застосуванні.

Базовий синтаксис правила `@property` виглядає так:

```css
@property --property-name {
  syntax: '<type>';
  inherits: true | false;
  initial-value: <value>;
}
```

`--property-name` — це ім’я кастомної властивості, яку ви оголошуєте. Як і всі кастомні властивості, вона має починатися з двох дефісів.

`syntax` визначає тип властивості, який може бути, наприклад, `<color>`, `<length>`, `<number>`, `<percentage>` або більш складними типами.

`inherits` вказує, чи має властивість наслідувати своє значення від батьківського елемента.

`initial-value` задає значення за замовчуванням для властивості.

Ось практичний приклад використання правила `@property`:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<button class="button">Click Me</button>
```

```css
@property --main-color {
  syntax: '<color>';
  inherits: false;
  initial-value: #3498db;
}

.button {
  background-color: var(--main-color);
}
```

:::

У цьому прикладі ми оголошуємо кастомну властивість `--main-color` як значення типу `color`, встановлюємо, що вона не наслідується, і задаємо їй ініціальне значення `#3498db`.

Однією з ключових переваг правила `@property` є те, що воно дозволяє анімувати кастомні властивості, чого раніше не було можливим:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<div class="gradient-box"></div>
```

```css
@property --gradient-angle {
  syntax: '<angle>';
  inherits: false;
  initial-value: 0deg;
}

.gradient-box {
  width: 100px;
  height: 100px;
  background: linear-gradient(var(--gradient-angle), red, blue);
  transition: --gradient-angle 0.5s;
}

.gradient-box:hover {
  --gradient-angle: 90deg;
}
```

:::

Код вище створює градієнт, який плавно анімується при наведенні на елемент, чого не можна було досягти за допомогою стандартних кастомних властивостей.

Тепер обговоримо, як правило `@property` працює із запасними варіантами.

Запасні варіанти дуже важливі в CSS, щоб забезпечити плавне погіршення стилів у браузерах, які не підтримують певні функції. У випадку з `@property` запасні варіанти працюють на двох рівнях: для самого правила і для використання кастомної властивості.

Для браузерів, які не підтримують правило `@property`, ви можете надати запасний варіант, оголосивши кастомну властивість традиційним способом:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
```

```css
:root {
  --main-color: #3498db;
}

@property --main-color {
  syntax: '<color>';
  inherits: false;
  initial-value: #3498db;
}

body {
  background-color: var(--main-color);
}
```

:::

У цьому випадку браузери, які підтримують `@property`, використовуватимуть більш строго визначену версію, а інші повернуться до стандартного оголошення кастомної властивості.

При використанні кастомної властивості ви можете надати запасне значення за допомогою функції `var()`, так само як і зі стандартними кастомними властивостями:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<button class="button">Click Me</button>
```

```css
.button {
  background-color: var(--main-color, #3498db);
}
```

:::

Це гарантує, що навіть якщо властивість `--main-color` не визначена або є недійсною, кнопка все одно матиме колір фону.

Правило `@property` також дозволяє складніші сценарії запасних варіантів. Наприклад, ви можете використовувати його для забезпечення типобезпечних запасних варіантів:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<div class="box">Colored box</div>
```

```css
@property --padding {
  syntax: '<length>';
  inherits: false;
  initial-value: 10px;
}

.box {
  width: 100px;
  height: 100px;
  background-color: darkred;
  color: white;
  padding: var(--padding);
}
```

:::

У цьому випадку, якщо хтось спробує встановити `--padding` у недійсне значення, наприклад колір, браузер повернеться до ініціального значення `10px`, зберігаючи типобезпечність.

Правило CSS `@property` вважається широко доступним у всіх основних сучасних браузерах (як-от Chrome, Edge, Opera, Firefox і Safari) з початку 2024 року. Якщо ви хочете підтримувати старіші браузери, потрібно надавати запасний CSS, як описано вище.

Підсумовуючи, правило `@property` є значним кроком уперед у роботі з кастомними властивостями в CSS. Воно дає більше контролю і відкриває нові можливості, як-от анімація кастомних властивостей. Проте важливо використовувати його розумно і завжди надавати відповідні запасні варіанти, щоб забезпечити хороший досвід у всіх браузерах. Як і з будь-якою передовою веб-технологією, ключове завдання — покращити досвід у браузерах, які її підтримують, і водночас гарантувати базову функціональність для тих, які не підтримують.
