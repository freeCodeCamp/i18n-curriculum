# --interactive--

CSS를 다룰 때, 요소에 대해 세 가지 다른 display 동작 유형인 `inline`, `block`, `inline-block`를 자주 접하게 됩니다.

이 속성들은 요소가 어떻게 배치되고 페이지 내 다른 요소와 어떻게 상호작용하는지에 영향을 줍니다.

이번 수업에서는 `inline-block` 속성이 어떻게 작동하는지, 그리고 인라인 요소와 블록 요소와 어떻게 다른지에 집중하겠습니다.

블록 요소는 부모 컨테이너의 전체 너비를 차지하는 큰 컨테이너 또는 "블록"처럼 동작합니다. 항상 새 줄에서 시작하며 높이와 너비를 조절할 수 있습니다.

인라인 요소는 필요한 만큼의 공간만 차지합니다. 주변 콘텐츠 내에서 흐르며 새 줄로 넘어가지 않습니다.

`inline-block` 속성은 이 두 동작의 혼합형입니다. 인라인 요소처럼 `inline-block` 요소는 새 줄에서 시작하지 않고 텍스트 흐름 내에 남아 있습니다.

하지만 인라인 요소와 달리, `inline-block` 요소는 블록 요소처럼 너비와 높이를 조절할 수 있습니다.

요약하자면, `inline`와 `inline-block`의 주요 차이는 `inline` 요소는 크기를 제어할 수 없지만 `inline-block` 요소는 다른 콘텐츠와 인라인 상태를 유지하면서도 크기를 완전히 조절할 수 있다는 점입니다.

예제를 살펴보겠습니다.

:::interactive_editor

```html
<link href="styles.css" rel="stylesheet">

<div class="container">
  <span class="inline-block-element element1">Inline-Block</span>
  <span class="inline-block-element element2">Inline-Block</span>
</div>
```

```css
.inline-block-element {
  display: inline-block;
  width: 150px;
  height: 100px;
}

.element1 {
  background-color: lightblue;
}

.element2 {
  background-color: lightgreen;
}
```

:::

위 예제에서는 `div`에 `container`라는 클래스가 있습니다. 그 `div` 요소 안에는 두 개의 `span` 요소가 있습니다.

각 span 요소는 `display:inline-block`로 설정되어 있고 너비와 높이도 지정되어 있습니다.

인라인-블록 요소는 인라인 요소처럼 나란히 나타나지만, 너비와 높이가 지정되어 있어 블록과 같은 속성을 가집니다.

하지만 `display: inline-block` 속성을 제거하면, 명확히 정의했음에도 `height`와 `width` 모두 적용되지 않습니다.

수정된 CSS는 다음과 같습니다.

:::interactive_editor

```html
<link href="styles.css" rel="stylesheet">

<div class="container">
  <span class="inline-block-element element1">Span element</span>
  <span class="inline-block-element element2">Span element</span>
</div>
```

```css
.inline-block-element {
  width: 150px;
  height: 100px;
}

.element1 {
  background-color: lightblue;
}

.element2 {
  background-color: lightgreen;
}
```

:::

이 코드에서는 `display: inline-block;` 속성을 제거하고 나머지는 그대로 유지했습니다. 여기서 `span` 요소는 기본 동작인 인라인 요소로 돌아갑니다.

그 결과 지정한 너비와 높이는 무시되고, 요소는 콘텐츠에 필요한 공간만 차지합니다.

`inline-block`가 어떻게 작동하는지 이해하는 것은 연속된 텍스트 흐름 내에서 정렬과 크기 제어가 모두 필요한 레이아웃을 만들 때 유용합니다.
