# --description--

在之前的課程中，你學會了如何使用像 `string` 和 `boolean` 這樣的原生型別。但 TypeScript 允許你從現有的型別建造新的型別。其中一個例子就是聯集（數學）型別。

聯集（數學）型別是結合兩個或多個其他型別的型別。

這是基本語法：

```ts
type1 | type2 | type3
```

讓我們來看看一些範例，以更好地理解這個概念。 

這裡有一個名為 `id` 的變數，其型別為 `string` 或 `number` 的聯集。

```ts
let id: string | number = "user_783";
```

該值可以更新為仍然對該聯集（數學）型別有效的數字：

```ts
let id: string | number = 1024;
```

如果你嘗試指定另一個不是 `string` 或 `number` 的值，TypeScript 就會顯示錯誤訊息：

```ts
// Type 'string[]' is not assignable to type 'string | number'.
let id: string | number = ["Java", "Python"];
```

當使用聯集（數學）型別時，重要的是要使用適用於所有型別的函式，否則 TypeScript 會發出像這樣的錯誤：

```ts
function getId(id: string | number) {
  // Property 'toUpperCase' does not exist on type 'string | number'.
  return id.toUpperCase();
}
```

這有一個解法，涉及縮小聯集（數學）。但你會在未來的課程中學到更多關於這個以及聯集（數學）型別如何與型別別名一起運作。

當處理 `null` 和 `undefined` 值時，聯集（數學）型別也很有幫助。 

這裡是一個範例，說明變數可能包含 `string` 值，或者如果尚未設定值則可能是 `null`：

```ts
let username: string | null;

// Type 'number' is not assignable to type 'string | null'.
username = 42;
```

在這種情況下，`username` 可以是 `string` 或 `null`，但不能是其他任何值。當你想要刻意表述沒有值時，可以使用 `null`。

這裡是另一個使用 `undefined` 的範例：

```ts
let score: number | undefined;

score = undefined; 

// Type 'string' is not assignable to type 'number | undefined'.
score = "high";
```

在此範例中，`score` 可以是 `number` 或 `undefined`。當值尚未被指定時，你可能會使用 `undefined`。

使用包含 `null` 和 `undefined` 的聯集（數學）型別可以清楚表示值可能遺失，同時仍允許 TypeScript 偵測無效的指派。
