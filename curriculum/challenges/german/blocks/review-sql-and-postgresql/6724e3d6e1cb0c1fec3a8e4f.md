## Einführung in relationale Datenbanken

- **Relationale Datenbanken**: Organisieren Daten in miteinander verbundenen Tabellen, die aus Zeilen und Spalten bestehen. Jede Zeile stellt einen Datensatz dar, und jede Spalte ein Attribut der Daten.
- **Vorteile relationaler Datenbanken**: Skalierbar, in vielen Bereichen anwendbar (z. B. Gesundheitswesen, Wirtschaft, Gaming) und strukturiert, um verlässliche Daten zu gewährleisten.
- **Häufige Anwendungsfälle**: Webentwicklung, Inventarsysteme, E-Commerce, Gesundheitswesen und Unternehmensanwendungen.

## Wichtige Konzepte

- **Schema**: Eine relationale Datenbank benötigt ein Schema, das ihre Struktur definiert – Tabellen, Spalten, Datentypen, Einschränkungen und Beziehungen.
- **Primärschlüssel**: Eindeutige Bezeichner für jede Zeile in einer Tabelle. Sie sind entscheidend für die Datenintegrität und werden verwendet, um Datensätze zwischen Tabellen über Fremdschlüssel zu verknüpfen.
- **Fremdschlüssel**: Verweise auf Primärschlüssel in einer anderen Tabelle, die verwendet werden, um verwandte Daten über Tabellen hinweg zu verbinden.
- **Beziehungen**: Durch das Verbinden von Tabellen über Primär- und Fremdschlüssel können Sie normalisierte Daten strukturieren und sinnvolle Abfragen durchführen.
- **Entity-Relationship-Diagramme (ERDs)**: Visualisieren, wie Entitäten (Tabellen) in einem Datenbankschema miteinander in Beziehung stehen.
- **Datenintegrität**: Wird durch Schlüssel und Datentypen sichergestellt. Gewährleistet Konsistenz und Genauigkeit der gespeicherten Daten.

## SQL-Grundlagen

- **Abfragen**: Anfragen, um bestimmte Daten aus der Datenbank abzurufen.

```sql
SELECT * FROM dogs WHERE age < 3;
```

- **WHERE-Klausel**: Filtert Ergebnisse basierend auf Bedingungen. Verwenden Sie Vergleichsoperatoren wie `<`, `=`, `>` usw.
- **SELECT mit ORDER BY**: Ruft Ergebnisse ab und sortiert sie basierend auf einer Spalte.

```sql
SELECT columns FROM table_name ORDER BY column_name;
```

## Tabellenoperationen

- **`CREATE TABLE`-Anweisung**: Diese Anweisung wird verwendet, um eine neue Tabelle in einer Datenbank zu erstellen.

```sql
CREATE TABLE first_table();
```

- **`ALTER TABLE ADD COLUMN`-Anweisung**: Diese Anweisung wird verwendet, um einer bestehenden Tabelle eine Spalte hinzuzufügen.

```sql
ALTER TABLE table_name ADD COLUMN column_name DATATYPE;
```

- **`ALTER TABLE DROP COLUMN`-Anweisung**: Diese Anweisung wird verwendet, um eine Spalte aus einer bestehenden Tabelle zu löschen.

```sql
ALTER TABLE table_name DROP COLUMN column_name;
```

- **`ALTER TABLE RENAME COLUMN`-Anweisung**: Diese Anweisung wird verwendet, um eine Spalte in einer Tabelle umzubenennen.

```sql
ALTER TABLE table_name RENAME COLUMN column_name TO new_name;
```

- **`DROP TABLE`-Anweisung**: Diese Anweisung wird verwendet, um eine gesamte Tabelle aus der Datenbank zu löschen.

```sql
DROP TABLE table_name;
```

- **`ALTER DATABASE RENAME`-Anweisung**: Diese Anweisung wird verwendet, um eine Datenbank umzubenennen.

```sql
ALTER DATABASE database_name RENAME TO new_database_name;
```

- **`DROP DATABASE`-Anweisung**: Diese Anweisung wird verwendet, um eine gesamte Datenbank zu löschen.

```sql
DROP DATABASE database_name;
```

## Einschränkungen und Datenintegrität

- **`ALTER TABLE ADD COLUMN` mit Einschränkung**: Diese Anweisung wird verwendet, um einer bestehenden Tabelle eine Spalte mit einer Einschränkung hinzuzufügen.

```sql
ALTER TABLE table_name ADD COLUMN column_name DATATYPE CONSTRAINT;
```

- **`NOT NULL`-Einschränkung**: Diese Einschränkung stellt sicher, dass eine Spalte keine NULL-Werte enthalten darf.

```sql
column_name VARCHAR(50) NOT NULL
```

- **`ALTER TABLE ADD PRIMARY KEY`-Anweisung**: Diese Anweisung wird verwendet, um einer Tabelle eine Primärschlüsseleinschränkung hinzuzufügen.

```sql
ALTER TABLE table_name ADD PRIMARY KEY(column_name);
```

- **`ALTER TABLE DROP CONSTRAINT`-Anweisung**: Diese Anweisung wird verwendet, um eine Einschränkung aus einer Tabelle zu entfernen.

```sql
ALTER TABLE table_name DROP CONSTRAINT constraint_name;
```

- **`ALTER TABLE ADD COLUMN` mit Fremdschlüssel**: Diese Anweisung wird verwendet, um einer Tabelle eine Fremdschlüsselspalte hinzuzufügen, die auf eine andere Tabelle verweist.

```sql
ALTER TABLE table_name ADD COLUMN column_name DATATYPE REFERENCES referenced_table_name(referenced_column_name);
```

- **`ALTER TABLE ADD UNIQUE`-Anweisung**: Diese Anweisung wird verwendet, um einer Spalte eine UNIQUE-Einschränkung hinzuzufügen.

```sql
ALTER TABLE table_name ADD UNIQUE(column_name);
```

- **`ALTER TABLE ALTER COLUMN SET NOT NULL`-Anweisung**: Diese Anweisung wird verwendet, um eine NOT NULL-Einschränkung auf eine bestehende Spalte zu setzen.

```sql
ALTER TABLE table_name ALTER COLUMN column_name SET NOT NULL;
```

- **`INSERT`-Anweisung mit NULL-Werten**: Diese Anweisung zeigt, wie NULL-Werte in eine Tabelle eingefügt werden.

```sql
INSERT INTO table_name(column_a) VALUES(NULL);
-- or
INSERT INTO table_name(column_b) VALUES('value'); -- if column_a allows nulls
```

- **Zusammengesetzter Primärschlüssel**: Diese Einschränkung definiert einen Primärschlüssel, der aus mehreren Spalten besteht.

```sql
CREATE TABLE course_enrollments (
    student_id INT,
    course_id INT,
    PRIMARY KEY (student_id, course_id)
);
```

## Datenmanipulation (CRUD)

- **`INSERT`-Anweisung**: Diese Anweisung wird verwendet, um eine einzelne Zeile in eine Tabelle einzufügen.

```sql
INSERT INTO table_name(column_1, column_2) VALUES(value1, value2);
```

- **`INSERT`-Anweisung mit ausgelassenen Spalten**: Diese Anweisung zeigt, wie Werte eingefügt werden, ohne die Spaltennamen explizit aufzulisten, wobei die Standardspaltenreihenfolge in der Tabelle verwendet wird.

```sql
INSERT INTO dogs VALUES ('Gino', 3);
```

- **`INSERT`-Anweisung mit mehreren Zeilen**: Diese Anweisung wird verwendet, um mehrere Zeilen in einer einzigen Operation in eine Tabelle einzufügen.

```sql
INSERT INTO dogs (name, age) VALUES 
('Gino', 3),
('Nora', 2);
```

- **`UPDATE`-Anweisung**: Diese Anweisung wird verwendet, um vorhandene Daten in einer Tabelle basierend auf einer Bedingung zu aktualisieren.

```sql
UPDATE table_name SET column_name=new_value WHERE condition;
```

- **`DELETE`-Anweisung**: Diese Anweisung wird verwendet, um Zeilen aus einer Tabelle basierend auf einer Bedingung zu löschen.

```sql
DELETE FROM table_name WHERE condition;
```

## Datentypen

- **`NUMERIC`-Datentyp**: Dieser Datentyp wird verwendet, um präzise Dezimalzahlen mit einer angegebenen Genauigkeit und Skalierung zu speichern.

```sql
price NUMERIC(10, 2)
```

- **`TEXT`-Datentyp**: Dieser Datentyp wird verwendet, um Zeichenketten variabler Länge ohne spezifische Längenbegrenzung zu speichern.

```sql
column_name TEXT
```

- **`INTEGER`-Datentyp**: Dieser Datentyp wird verwendet, um ganze Zahlen ohne Dezimalstellen zu speichern.

```sql
units_sold INTEGER
```

- **`SMALLINT` und `BIGINT`-Datentypen**: Dies sind Varianten von INTEGER mit kleineren bzw. größeren Wertebereichen.

- **`SERIAL`-Datentyp**: Dieser Datentyp wird verwendet, um in PostgreSQL automatisch inkrementierende Ganzzahlspalten zu erstellen.

```sql
id SERIAL
```

- **`AUTO_INCREMENT`-Attribut**: Dieses Attribut wird in MySQL verwendet, um automatisch inkrementierende Ganzzahlspalten zu erstellen.

```sql
id INT AUTO_INCREMENT
```

- **`VARCHAR`-Datentyp**: Dieser Datentyp wird verwendet, um Zeichenketten variabler Länge mit einer angegebenen maximalen Länge zu speichern.

```sql
name VARCHAR(50)
```

- **`DATE`-Datentyp**: Dieser Datentyp wird verwendet, um Datumswerte (Jahr, Monat, Tag) zu speichern.

```sql
event_date DATE
```

- **`TIME`-Datentyp**: Dieser Datentyp wird verwendet, um Zeitwerte (Stunde, Minute, Sekunde) zu speichern.

```sql
start_time TIME
```

- **`TIMESTAMP`-Datentyp**: Dieser Datentyp wird verwendet, um Datum- und Zeitwerte zu speichern, optional mit Zeitzoneninformationen.

```sql
event_timestamp TIMESTAMP
event_timestamp TIMESTAMP WITH TIME ZONE
```

- **`BOOLEAN`-Datentyp**: Dieser Datentyp wird verwendet, um Wahrheitswerte (true/false) zu speichern.

```sql
is_active BOOLEAN
```

## Datenbankbeziehungen

- **Arten von Beziehungen**: Dies sind die verschiedenen Arten, wie Tabellen in einer relationalen Datenbank miteinander verbunden sein können.
  - Eins-zu-eins
  - Eins-zu-viele
  - Viele-zu-eins
  - Viele-zu-viele
  - Selbstreferenzierend (rekursiv)

- **Eins-zu-eins-Beziehung**: Dieser Beziehungstyp bedeutet, dass jeder Datensatz in einer Tabelle genau einem Datensatz in einer anderen Tabelle entspricht.

```md
One employee is assigned exactly one vehicle.
Tables: employees, vehicles
```

- **Eins-zu-viele-Beziehung**: Dieser Beziehungstyp bedeutet, dass ein Datensatz in einer Tabelle mit mehreren Datensätzen in einer anderen Tabelle verbunden sein kann.

```md
One customer can have many orders.
Tables: customers → orders
```

- **Viele-zu-viele-Beziehung über Zwischentabelle**: Dieser Beziehungstyp wird mit einer Zwischentabelle implementiert, die Fremdschlüssel von beiden verbundenen Tabellen enthält.

```sql
CREATE TABLE books_authors (
    author_id INT REFERENCES authors(id),
    book_id INT REFERENCES books(id)
);
```

- **Selbstreferenzierende Beziehungen**: Dieser Beziehungstyp tritt auf, wenn eine Tabelle auf sich selbst verweist und so eine hierarchische Struktur bildet.

```md
An employee table where each employee may report to another employee.
```

## Fortgeschrittenes SQL (Joins)

- **`INNER JOIN`-Anweisung**: Dieser Join gibt nur die Zeilen zurück, die in beiden Tabellen übereinstimmende Werte haben.

```sql
SELECT *
FROM products
INNER JOIN sales ON products.product_id = sales.product_id;
```

- **`FULL OUTER JOIN`-Anweisung**: Dieser Join gibt alle Zeilen aus beiden Tabellen zurück, einschließlich nicht übereinstimmender Zeilen aus beiden Tabellen.

```sql
SELECT *
FROM products
FULL OUTER JOIN sales ON products.product_id = sales.product_id;
```

- **`LEFT OUTER JOIN`-Anweisung**: Dieser Join gibt alle Zeilen aus der linken Tabelle und die passenden Zeilen aus der rechten Tabelle zurück.

```sql
SELECT *
FROM products
LEFT JOIN sales ON products.product_id = sales.product_id;
```

- **`RIGHT OUTER JOIN`-Anweisung**: Dieser Join gibt alle Zeilen aus der rechten Tabelle und die passenden Zeilen aus der linken Tabelle zurück.

```sql
SELECT *
FROM products
RIGHT JOIN sales ON products.product_id = sales.product_id;
```

- **`SELF JOIN`-Anweisung**: Dieser Join wird verwendet, um eine Tabelle mit sich selbst zu verbinden, um Zeilen innerhalb derselben Tabelle zu vergleichen.

```sql
SELECT A.column_name, B.column_name
FROM table_name A
JOIN table_name B ON A.related_column = B.related_column;
```

- **`CROSS JOIN`-Anweisung**: Dieser Join gibt das kartesische Produkt von zwei Tabellen zurück, indem jede Zeile der ersten Tabelle mit jeder Zeile der zweiten Tabelle kombiniert wird.

```sql
SELECT *
FROM table1
CROSS JOIN table2;
```

## PostgreSQL-spezifische Befehle

- **`psql`-Login-Befehl**: Dieser Befehl wird verwendet, um sich mit einem bestimmten Benutzernamen und einer Datenbank bei PostgreSQL anzumelden.

```bash
psql --username=freecodecamp --dbname=postgres
```

- **`\l`-Befehl**: Dieser Befehl listet alle Datenbanken in der PostgreSQL-Instanz auf.

```sql
\l
```

- **`CREATE DATABASE` und `\c`-Befehle**: Diese Befehle werden verwendet, um eine neue Datenbank zu erstellen und sich mit ihr zu verbinden.

```sql
CREATE DATABASE database_name;
\c database_name
```

- **`\d`-Befehl**: Dieser Befehl listet alle Tabellen in der aktuellen Datenbank auf.

```sql
\d
```

- **`\d table_name`-Befehl**: Dieser Befehl zeigt das Schema/die Struktur einer bestimmten Tabelle an.

```sql
\d table_name
```

- **`\q`-Befehl**: Dieser Befehl beendet den PostgreSQL-Client.

```bash
\q
```

## Relationale vs. nicht-relationale Datenbanken

- **Nicht-relationale (NoSQL) Datenbanken**: Speichern unstrukturierte oder semi-strukturierte Daten. Benötigen kein starres Schema und sind flexibler für sich entwickelnde Datenmodelle.
- **Auswahl zwischen relational und nicht-relational**: Hängt von der Art Ihrer Daten und den Anforderungen der Anwendung ab.
- **Relational vs. nicht-relational**: Wählen Sie relational für strukturierte Daten und Konsistenz; NoSQL für Flexibilität und schnell wechselnde Daten.

## Beliebte RDBMS-Systeme

- **MySQL**: Open Source, zuverlässig, weit verbreitet in der Webentwicklung, unterstützt von einer großen Community.
- **PostgreSQL**: Open Source, fortschrittlich, erweiterbar. Unterstützt benutzerdefinierte Datentypen und serverseitige Programmierung.
- **SQLite**: Leichtgewichtig, dateibasiert, serverlos. Ideal für kleine Anwendungen.
- **Microsoft SQL Server**: Proprietäre, unternehmensgerechte Datenbank.
- **Oracle Database**: Kommerzielles RDBMS, bekannt für Leistung und Skalierbarkeit im großen Maßstab.
