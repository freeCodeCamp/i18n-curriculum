# --description--

저장소를 GitHub에 푸시하기 전에 인증이 필요합니다. SSH와 GPG 키 모두 공개-개인 키 쌍을 사용합니다. 즉, 로컬 컴퓨터에 개인 키를 보관하며 이를 사용해 자신을 인증합니다. 공개 키는 인증을 검증해야 하는 사람이나 서비스와 공유합니다. 다시 말해, 개인 키는 작업을 수행하는 데 필요하고 공개 키는 그 작업을 확인하는 데 사용됩니다.

GPG, 즉 Gnu Privacy Guard 키는 보통 파일이나 커밋에 서명하는 데 사용합니다. 다른 사람이 공개 GPG 키를 사용해 파일 서명이 본인의 키에서 왔고 파일 내용이 변경되거나 변조되지 않았음을 확인할 수 있습니다. SSH, 즉 Secure SHell 키는 보통 `ssh` 유틸리티를 통해 원격 서버 연결을 인증하는 데 사용합니다. 하지만 이 수업에서 배우겠지만, SSH 키로 커밋에 서명할 수도 있습니다.

GPG 키를 생성하려면 다음 명령을 실행하세요:

```sh
gpg --full-generate-key
```

SSH 키를 생성하려면 다음 명령을 실행하세요:

```sh
ssh-keygen -t ed25519 -C "your_email@example.com"
```

실제 이메일을 반드시 입력하세요.

이 두 명령 모두 키를 보호하기 위한 암호 구문(passphrase)을 요청할 것입니다. 암호 구문은 선택 사항이지만 강력히 권장합니다.

SSH 키의 경우 개인 키와 공개 키 두 개의 파일이 생성됩니다. 공개 키 파일 이름은 `.pub`로 끝나 개인 키를 실수로 공유하지 않도록 합니다. 이 파일들은 `~/.ssh`에 저장됩니다.

반면 GPG 키는 숨겨진 암호화된 디렉터리에 저장됩니다. 대신 공개 키 목록을 보려면 다음 명령을 실행하세요:

```sh
gpg --list-secret-keys --keyid-format=long
```

그다음 공개 키를 얻으려면 다음을 사용하세요:

```sh
gpg --armor --export "<key id>"
```

그렇다면 이 새 키들로 무엇을 할 수 있을까요? 커밋에 서명하는 방법에 대해 이야기했는데, 이 키들 중 어느 쪽으로도 서명할 수 있습니다.

GPG 키로 커밋에 서명하려면 개인 키가 아닌 공개 키를 GitHub 계정에 업로드해야 합니다. 그리고 앞서 키 목록에서 얻은 짧은 ID를 사용해 다음 명령으로 git 서명 키로 설정하세요:

```sh
git config --global user.signingkey <your_gpg_key_id>
```

그다음 `-S` 플래그를 `git commit` 명령에 전달해 특정 커밋에 서명할 수 있습니다. 이때 암호 구문을 입력해야 합니다. 또는 모든 커밋에 자동으로 서명하려면 autosign 설정을 `true`로 지정하세요:

```sh
git config --global commit.gpgsign true
```

SSH 키로 서명하는 것은 GitHub에서 비교적 새 기능입니다. 먼저 키를 GitHub 계정에 업로드해야 합니다. 그다음 git 서명 모드를 SSH로 설정하세요:

```sh
git config --global gpg.format ssh
```

서명 키를 설정할 때는 ID 대신 파일 경로를 전달합니다:

```sh
git config --global user.signingkey <path_to_your_ssh_keys>
```

마지막으로 GPG 키와 마찬가지로 자동 서명을 활성화할 수 있습니다.

이 두 방법 중 어느 하나를 설정하면, 이제 GitHub에서 특별 배지를 받는 검증된 커밋을 만들 준비가 된 것입니다.
