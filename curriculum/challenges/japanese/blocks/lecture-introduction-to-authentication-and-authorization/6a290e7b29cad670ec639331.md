# --description--

カスタム認証を設定するには、多くの手動ロジックを書く必要があります。リクエストヘッダーの読み取り、クッキーの抽出、署名の検証、エラーの管理を行わなければなりません。

自分で構築することは学習には最適ですが、すべての本番アプリケーションでこれを行うのは時間がかかり、セキュリティの見落としが起こりやすいです。だからこそ開発者はPassport.jsを使います。

Passport.jsはNode.js向けの認証ミドルウェアです。リクエストの認証方法を標準化する整理されたフレームワークとして機能します。低レベルの認証の詳細を処理し、アプリケーションのビジネスロジックに集中できるようにします。

Passport.jsの基本的な考え方は、すべてのアプリケーションで異なるログイン方法が必要ですが、認証の基本的な流れは同じであるべきだということです。Passportは**Strategies**と呼ばれるモジュール式プラグインによってこの柔軟性を実現しています。

すべての認証方法に対してカスタムコードを書く代わりに、必要な特定のストラテジーをインストールします。現在使われているほぼすべての認証方法をカバーする500以上のストラテジーがあります。その一部は以下の通りです：

- **passport-local:** 自分のデータベースに対する従来のユーザー名とパスワード認証を処理します。

- **passport-jwt:** 受信リクエストからJSON Web Tokenを抽出し検証してAPIエンドポイントを保護します。

- **passport-google-oauth20:** 既存のGoogleアカウントを使って即座にログインできます。

- **passport-facebook:** OAuth 2.0を使ってFacebookアカウントでログインします。

- **passport-github2:** GitHubアカウントを使って即座にログインできます。

認証を個別のストラテジーに分けることで、Passportはアプリケーションの成長をスムーズにします。今日ユーザー名/パスワードシステムで始めて、来月「Googleでログイン」を追加したくなっても、認証コード全体を書き直す必要はありません。GoogleのストラテジーをインストールしてPassportに組み込むだけです。

ExpressアプリケーションでPassportを使うには、コアのPassportパッケージと使用予定の特定のストラテジーをインストールする必要があります。標準的なユーザー名とパスワードのセットアップの場合、ターミナルで次のコマンドを実行してください：

```bash
npm install passport passport-local
```

インストール後、PassportをExpressアプリケーションのセットアップに組み込む必要があります。これはルートを定義する前に行わなければなりません：

```js
const express = require('express');
const passport = require('passport');
const LocalStrategy = require('passport-local').Strategy;

const app = express();

// initialize Passport middleware
app.use(passport.initialize());

// configure Passport to use the Local Strategy
passport.use(new LocalStrategy((username, password, done) => {
    // your database look-up and password verification logic goes here
}));
```

Passportはミドルウェアとして標準のExpressルーティングシステムにスムーズに統合されます。ユーザーがログインルートに資格情報を提出すると、Passportがリクエストを傍受し、設定されたストラテジーにデータを渡して結果を評価します。

```js
app.post('/login', passport.authenticate('local'), (req, res) => {
  // if this function runs, then authentication was successful
  res.send(`Welcome back, ${req.user.username}!`);
});
```

ストラテジーがユーザーを正常に検証すると、Passportは2つの重要なタスクを自動的に処理します：

1. **セッション管理：** セッションクッキー内にユーザーデータの小さな断片を保存するヘルパー関数を起動します。

2. **リクエスト装飾：** 認証されたユーザーオブジェクトを直接`req.user`として受信リクエストに添付します。

`req.user`が設定されると、アプリケーション内のその後のミドルウェアやルートハンドラは誰がログインしているかを即座に確認でき、その身元に基づいて認可の判断ができます。

PassportはNode.jsアプリケーションでユーザーの身元管理を一貫して安全に、そして本番環境に対応した基盤を提供します。
