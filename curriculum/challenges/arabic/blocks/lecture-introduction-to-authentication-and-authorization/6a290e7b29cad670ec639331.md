# --description--

إعداد توثيق مخصص يتطلب كتابة كمية كبيرة من المنطق اليدوي. عليك التعامل مع قراءة أقسام الطلب، استخراج الكوكيز، التحقق من التواقيع، وإدارة الأخطاء.

بينما بناء هذا بنفسك مفيد للتعلم، فإن القيام به لكل تطبيق إنتاجي يستغرق وقتًا طويلاً وعرضة لأخطاء أمنية. لهذا السبب يستخدم المطوّرون Passport.js.

Passport.js هو برمجيات وسيطة للتوثيق في Node.js. يعمل كإطار عمل منظم يموحد طريقة توثيق الطلبات. يتولى تفاصيل التوثيق منخفضة المستوى، مما يسمح لك بالتركيز على منطق العمل في تطبيقك.

الفكرة الأساسية وراء Passport.js هي أن كل تطبيق يحتاج إلى طرق مختلفة لتسجيل الدخول، لكن تدفق التوثيق الأساسي يجب أن يبقى نفسه. يحقق Passport هذه المرونة من خلال ملحقات معيارية تسمى **Strategies**.

بدلاً من كتابة شفرة مخصصة لكل طريقة توثيق، تثبت الاستراتيجية المحددة التي تحتاجها. هناك أكثر من 500 استراتيجية متاحة، تغطي تقريبًا كل طريقة توثيق مستخدمة اليوم. بعضها:

- **passport-local:** يتعامل مع توثيق اسم المستخدم وكلمة المرور التقليدي ضد قاعدة بياناتك الخاصة.

- **passport-jwt:** يحمي نقاط نهاية API باستخراج والتحقق من رموز JSON Web Tokens من الطلبات الواردة.

- **passport-google-oauth20:** يسمح للمستخدمين بتسجيل الدخول فورًا باستخدام حسابات Google الحالية لديهم.

- **passport-facebook:** تسجيل الدخول بحسابات Facebook عبر OAuth 2.0.

- **passport-github2:** يسمح للمستخدمين بتسجيل الدخول فورًا باستخدام حسابات GitHub الخاصة بهم.

بتقسيم التوثيق إلى استراتيجيات مميزة، يسمح Passport لتطبيقك بالنمو بسلاسة. إذا بدأت اليوم بنظام اسم مستخدم/كلمة مرور وقررت إضافة "تسجيل الدخول عبر Google" الشهر المقبل، لا تحتاج إلى إعادة كتابة شفرة التوثيق بالكامل. فقط ثبت استراتيجية Google واربطها بـ Passport.

لاستخدام Passport في تطبيق Express، يجب تثبيت حزمة Passport الأساسية مع الاستراتيجية المحددة التي تخطط لاستخدامها. لإعداد اسم مستخدم وكلمة مرور قياسي، نفذ الأمر التالي في الطرفية:

```bash
npm install passport passport-local
```

بعد التثبيت، تحتاج إلى ربط Passport في إعداد تطبيق Express الخاص بك. يجب أن يحدث هذا قبل تعريف المسارات:

```js
const express = require('express');
const passport = require('passport');
const LocalStrategy = require('passport-local').Strategy;

const app = express();

// initialize Passport middleware
app.use(passport.initialize());

// configure Passport to use the Local Strategy
passport.use(new LocalStrategy((username, password, done) => {
    // your database look-up and password verification logic goes here
}));
```

يدمج Passport بسلاسة في نظام التوجيه القياسي لـ Express كبرمجيات وسيطة. عندما يرسل المستخدم بيانات اعتماده إلى مسار تسجيل الدخول، يعترض Passport الطلب، يمرر البيانات إلى الاستراتيجية المكوّنة، ويقيّم النتيجة.

```js
app.post('/login', passport.authenticate('local'), (req, res) => {
  // if this function runs, then authentication was successful
  res.send(`Welcome back, ${req.user.username}!`);
});
```

إذا تحقق الاستراتيجية من المستخدم بنجاح، يتولى Passport مهمتين حاسمتين تلقائيًا:

1. **إدارة الجلسة:** يشغل دوال مساعدة لتخزين قطعة صغيرة من بيانات المستخدم داخل كوكي الجلسة.

2. **تزيين الطلب:** يربط كائن المستخدم المُوثَّق مباشرةً بالطلب الوارد كـ `req.user`.

بمجرد ملء `req.user`، يمكن لأي برمجيات وسيطة لاحقة أو معالج مسار في تطبيقك رؤية من هو المستخدم المُسجّل فورًا واتخاذ قرارات التفويض بناءً على هويته.

يوفر Passport أساسًا ثابتًا وآمنًا وجاهزًا للإنتاج لإدارة هويات المستخدمين في تطبيقات Node.js الخاصة بك.
