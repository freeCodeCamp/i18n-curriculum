# --description--

El middleware a nivel de enrutador es similar al middleware a nivel de aplicación, pero está vinculado a una instancia de `express.Router()` en lugar de a la instancia principal de la app. Se puede usar para modularizar tu aplicación Express creando módulos de rutas aislados.

El middleware a nivel de enrutador funciona de la misma manera que el middleware a nivel de aplicación, pero se aplica a instancias específicas de enrutadores. Esto facilita aplicar middleware solo a un conjunto de rutas en lugar de globalmente.

```js
const express = require('express')
const app = express()
const router = express.Router()

// Router-level middleware to log requests to '/menu' routes
router.use((req, res, next) => {
  console.log('Request made to /menu route')
  next() // Pass control to the next handler
})

// Define routes
router.get('/drinks', (req, res) => {
  res.send('Welcome to the drinks menu!')
})

app.use('/menu', router) // Apply the router

app.listen(3000, () => {
  console.log('Server running on http://localhost:3000')
})
```

* `router.use()`: Este es middleware a nivel de enrutador, que se aplica solo a las rutas manejadas por router.
* `app.use('/menu', router)`: El middleware se aplica solo a las rutas que comienzan con `/menu` (por ejemplo, `/menu/drinks`).

El middleware a nivel de enrutador te permite dividir tu app en componentes modulares y aplicar middleware selectivamente a un grupo de rutas. Esto ayuda con la organización y la claridad del código.
