# --description--

مكونات خادم React (RSCs) هي توجه جديد غيّر طريقة تعامل مطوّري React مع الأمور. مع RSCs، ينتقل المزيد من العمل إلى الخادم، وهذا له فوائد كثيرة.

لنلقِ نظرة على ماهية مكونات الخادم، كيف تعمل، وما الذي أدى إلى تقديم مكونات الخادم.

مكونات خادم React هي مكونات React تُعرض حصريًا على الخادم، الذي يرسل فقط HTML النهائي إلى العميل. هذا يعني أن تلك المكونات يمكنها الوصول مباشرة إلى موارد الخادم وتقليل كمية جافاسكريبت المرسلة إلى المتصفح بشكل كبير.

تقليديًا، تستخدم تطبيقات React نظام "مكونات العميل" الذي يتولى كل شيء في تطبيق React النموذجي، مثل العرض، التفاعل، والآثار الجانبية. لم يكن مصطلح "مكون العميل" يُستخدم كثيرًا حتى ظهور مكونات خادم React مؤخرًا.

لكن نظام مكونات العميل له بعض العيوب مثل حزم جافاسكريبت الكبيرة وأوقات تحميل أولية أبطأ.

وجدت أُطُر عمل React مثل Next.js وGatsby حلولًا لتفريغ بعض العمليات إلى الخادم لحل تلك المشاكل، لكن لم يكن أي منها موحدًا. إذا استخدمت أيًا من هذين الإطارين، فمن المحتمل أنك سمعت عن `getServerSideProps` و`getServerData`.

ثم ظهرت مكونات خادم React، التي تتيح لك تشغيل بعض المكونات بالكامل على الخادم حتى تتمكن من القيام بأشياء مثل جلب البيانات والحساب قبل تشغيل أي شفرة في متصفح المستخدم.

انتشرت مكونات الخادم أولًا وأصبحت متاحة بسهولة في Next.js. أُطُر أخرى مثل Remix وGatsby تلحق بالركب، وهناك إضافة تجريبية لـ Vite تسمى `vite-plugin-react-server` تتيح لك بناء مكونات خادم.

فكيف تعمل مكونات الخادم؟

واحدة من أفضل الطرق لشرح مكونات خادم React هي من خلال جلب البيانات.

في مكونات عميل React التقليدية، تترك المتصفح يتولى طلبات API. وبما أن جلب البيانات هو أثر جانبي، تقوم بإجراء هذا الطلب داخل خطاف `useEffect`.

من الممارسات الجيدة أيضًا تعيين متغيرات الحالة مثل loading وdata وerror لتتمكن من الإشارة إلى أن البيانات تُحمّل، عرض البيانات عند جاهزيتها، أو عرض خطأ في التطبيق.

مع مكونات خادم React، يمكنك نقل المكون بالكامل إلى الخادم وجلب البيانات هناك دون الحاجة لاستخدام `useState` أو `useEffect`:

```jsx
const Users = async () => {
  const res = await fetch("https://jsonplaceholder.typicode.com/users");
  const users = await res.json();

  return (
    <>
      <h1 className="text-4xl text-center mt-6">Users</h1>
      <ul className="text-center mt-3">
        {users.map((user) => (
          <li key={user.id}>{user.name}</li>
        ))}
      </ul>
    </>
  );
};

export default Users;
```

بما أن مكونات خادم React تعمل فقط على الخادم، يمكنك ببساطة جلب البيانات من API وعرضها مرة واحدة فقط. أيضًا، بما أن جلب البيانات يحدث على الخادم، أقرب إلى المصدر، قد يتحسن أداء تطبيقك، خاصة للأشخاص ذوي الاتصالات الشبكية البطيئة.

مشكلة كبيرة هي أن كل الشفرة الخاصة بمكونات الخادم تبقى على الخادم، ولا تُرسل إلى المتصفح. هذا يعني أنك لا تستطيع استخدام خطاطيف React معها، ولا يمكنها الوصول إلى واجهات برمجة تطبيقات الويب أو مستمعي أحداث المتصفح. فكيف تضيف التفاعل؟

في موجِّه تطبيق Next.js، كل المكونات هي مكونات خادم افتراضيًا. إذا أردت إضافة التفاعل، يجب أن تحدد المكون كمكون عميل باستخدام توجيه `"use client"`.

لنفترض أنك تريد تحويل المثال السابق إلى مكون عميل. إليك كيف تفعل ذلك:

```jsx
"use client";

import { useState, useEffect } from "react";

const Users2 = () => {
  const [status, setStatus] = useState({
    users: [],
    loading: true,
    error: null,
  });

  async function fetchUsers() {
    try {
      const res = await fetch("https://jsonplaceholder.typicode.com/users");
      const data = await res.json();
      setStatus((prevStatus) => ({
        ...prevStatus,
        users: data,
        loading: false,
      }));
    } catch (err) {
      setStatus((prevStatus) => ({
        ...prevStatus,
        error: err.message,
        loading: false,
      }));
    }
  }

  useEffect(() => {
    fetchUsers();
  }, []);

  if (status.loading) {
    return <p>Loading Users...</p>;
  }
  if (status.error) {
    return <p>Error getting users: {status.error}</p>;
  }

  return (
    <>
      <h1 className="text-4xl text-center mt-6">Users</h1>
      <ul className="text-center mt-3">
        {status.users.map((user) => (
          <li key={user.id}>{user.name}</li>
        ))}
      </ul>
    </>
  );
};

export default Users2;
```

إذا أردت إضافة تفاعل مثل أحداث النقر، يجب أيضًا تحديد المكون كمكون عميل:

```jsx
"use client";

import { useState } from "react";

const Counter = () => {
  const [count, setCount] = useState(0);

  return (
    <>
      <h1>Counter</h1>
      <button onClick={() => setCount(count + 1)}>Increment</button>
      <h2>{count}</h2>
      <button onClick={() => setCount(count - 1)}>Decrement</button>
    </>
  );
};

export default Counter;
```

إذا لم تضف توجيه use client إلى المكون، ستحصل على خطأ برسالة تقول "أنت تستورد مكون خادم يحتاج `useState`. هذا الخطاف في React يعمل فقط في مكون عميل. لإصلاح ذلك، علّم الملف (أو الأصل) بتوجيه `"use client"`."

الفوائد الرئيسية التي تأتي مع مكونات خادم React هي أن جلب البيانات يصبح أبسط، الشفرة أسهل للقراءة، وتعقيد العميل يقل.
