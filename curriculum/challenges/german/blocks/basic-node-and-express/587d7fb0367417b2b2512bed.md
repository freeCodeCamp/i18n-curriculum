# --description--

Bei diesen Aufgaben schreiben Sie Ihren Code mit einer der folgenden Methoden:

- Klonen Sie <a href="https://github.com/freeCodeCamp/boilerplate-express/" target="_blank" rel="noopener noreferrer nofollow">dieses GitHub-Repository</a> und bearbeiten Sie die Aufgaben lokal.
- Verwenden Sie einen Website-Builder Ihrer Wahl, um das Projekt abzuschließen. Stellen Sie sicher, dass Sie alle Dateien aus unserem GitHub-Repository einbinden.

Während der Entwicklung ist es wichtig, den Überblick über das Geschehen im Code zu behalten.

Node ist einfach eine JavaScript-Umgebung. Wie bei clientseitigem JavaScript können Sie die Konsole nutzen, um nützliche Debug-Informationen anzuzeigen. Auf Ihrem lokalen Rechner sehen Sie die Konsolenausgabe in einem Terminal.

Wir empfehlen, das Terminal während der Arbeit an diesen Aufgaben geöffnet zu lassen. Indem Sie die Ausgabe im Terminal lesen, können Sie eventuelle Fehler erkennen.

Der Server muss nach Änderungen an seinen Dateien neu gestartet werden.

Sie können den Server im Terminal mit `Ctrl + C` stoppen und ihn direkt mit Node (`node mainEntryFile.js`) oder über ein Ausführungsskript in der `package.json`-Datei mit `npm run` starten.

Zum Beispiel würde das `"start": "node server.js"`-Skript im Terminal mit `npm run start` ausgeführt.

Um das automatische Neustarten des Servers beim Speichern von Dateien zu implementieren, stellt Node das Flag `--watch` bereit, das Sie zu Ihrem Startskript `"start": "node --watch server.js"` hinzufügen können, oder Sie installieren ein npm-Paket wie `nodemon`. Wir überlassen Ihnen diese Aufgabe als Übung.
