# --description--

In einer vorherigen Lektion haben Sie gelernt, dass Attribute die Variablen sind, die zu einem Objekt gehören. Das bedeutet, sie enthalten Daten, die den Zustand oder das Verhalten des Objekts beschreiben.

Zum Beispiel hätte ein Auto normalerweise eine Marke und ein Modell. Die Marke und das Modell könnten Attribute für eine `Car` Klasse sein:

```python
class Car: 
    def __init__(self, brand, model): 
        self.brand = brand 
        self.model = model 

my_car = Car('Lamborghini', 'Gallardo') 
print(my_car.brand) # Lamborghini 
print(my_car.model) # Gallardo 
```

Manchmal wissen Sie jedoch nicht, welche Attribute Sie benötigen, bis Ihr Programm läuft. Stellen Sie sich vor, Sie schreiben ein Skript, das Attributnamen von einem Benutzer oder einer Konfigurationsdatei erhält. Das sind keine Attribute, die Sie im Voraus fest im Code verankern können.

Hier kommt das dynamische Handhaben von Attributen ins Spiel. So können Sie Attribute mit ihren Namen als Variablen zugreifen, ändern, überprüfen oder sogar löschen – und nicht als feste Namen im Code. Das gibt Ihrem Programm die Flexibilität, auf unterschiedliche Daten oder Benutzereingaben spontan zu reagieren.

Python stellt Ihnen vier praktische eingebaute Funktionen zur Verfügung, um dynamisch mit Objektattributen zu arbeiten. Diese sind `getattr()`, `setattr()`, `hasattr()` und `delattr()`.

Sie ermöglichen es Ihnen, Attribute mit variablen Namen zuzugreifen, zu erstellen, zu prüfen und zu entfernen. Schauen wir uns jede einzelne in Aktion an.

`getattr()` macht es möglich, ein Attribut von einem Objekt zu lesen, wenn Sie dessen Namen erst zur Laufzeit kennen. Wenn das Attribut nicht existiert, löst es eine `AttributeError` aus, es sei denn, Sie geben einen Standardwert an.

Um es zu verwenden, übergeben Sie das Objekt, den Attributnamen und optional einen Standardwert:

```python
getattr(object, attribute_name, default_value) 
```

Hier ein Beispiel:

```python
class Person: 
    def __init__(self, name, age): 
        self.name = name 
        self.age = age 

person = Person('John Doe', 30) 
 
print(getattr(person, 'name')) # John Doe 
print(getattr(person, 'age')) # 30 
print(getattr(person, 'city', 'Milano')) # Milano
```

Im obigen Beispiel ist `Milano` ein Standardwert, weil `city` in der `Person` Klasse nicht existiert.

Wie bereits erwähnt, zeigt sich die wahre Stärke von `getattr()`, wenn der Attributname aus einer Variablen stammt, etwa aus Benutzereingaben oder einer Datei.

In diesem Fall können Sie die reguläre `object.attribute_name` Syntax nicht verwenden, weil der Attributname nicht feststeht.

```python
class Person: 
    def __init__(self, name, age): 
        self.name = name 
        self.age = age 

person = Person('John Doe', 30)

attr_name = input('Enter the attribute you want to see: ')
print(getattr(person, attr_name, 'Attribute not found'))
```

In diesem Fall, wenn der Benutzer `name` eingibt, sieht er `John Doe`, und wenn er `age` eingibt, sieht er `30`. Und wenn er etwas eingibt, das in der Klasse nicht existiert, wie `email`, sieht er `Attribute not found`.

Genau hier zeigt sich der Vorteil des dynamischen Handhabens von Attributen. Es erlaubt Ihrem Code, auf Eingaben und Daten zu reagieren, die er vorher nicht kannte.

Außerdem möchten Sie vielleicht alle Attribute eines Objekts durchsehen, nicht nur die, die Sie bereits kennen. Die eingebaute `dir()` Funktion ermöglicht das. Sie gibt eine Liste aller Attributnamen des Objekts zurück. So verwenden Sie sie:

```python
class Person: 
    def __init__(self, name, age): 
        self.name = name 
        self.age = age 

person = Person('John Doe', 30)

# Loop through all attributes of the person object with dir() function
for attr in dir(person):
    # Ignore dunder methods like __init__ or __str__ and regular methods
    if not attr.startswith('__') and not callable(getattr(person, attr)): 
        value = getattr(person, attr)
        print(f'{attr}: {value}')

# Output
# age: 30
# name: John Doe
```

Im obigen Schleifendurchlauf ist `callable()` eine eingebaute Funktion, die `True` zurückgibt, wenn das übergebene Objekt wie eine Funktion oder Methode aufgerufen werden kann, und sonst `False`. Durch die Prüfung `not callable(getattr(person, attr))` überspringt die Schleife Methoden und gibt nur die Datenattribute wie `name` und `age` aus.

Die `setattr()` Funktion erlaubt es Ihnen, ein neues Attribut dynamisch zu erstellen oder ein bestehendes zu aktualisieren. Die Syntax sieht so aus:

```python
setattr(object, attribute_name, value) 
```

Hier ein Beispiel, das Konfigurationsattribute basierend auf Daten aus einer Konfigurations- oder Umgebungsvariablendatei setzt:

```python
class Configuration:
    pass

# Data loaded at runtime (like from a config or env file)
settings_data = {
    'server_url': 'https://api.example.com',
    'timeout_sec': 30,
    'max_retries': 5
}

config_obj = Configuration()

# Dynamically set attributes using dictionary keys and values
for attr_name, attr_value in settings_data.items():
    setattr(config_obj, attr_name, attr_value)

print(config_obj.server_url) # https://api.example.com
print(config_obj.timeout_sec) # 30
```

Es gibt auch `hasattr()`. Bevor Sie etwas mit einem Attribut machen oder es löschen, ist es gute Praxis zu prüfen, ob es existiert. Genau das ermöglicht `hasattr()`. Es prüft, ob ein Attribut existiert, und gibt `True` oder `False` zurück, je nach Ergebnis.

Hier die Grundsyntax:

```python
hasattr(object, attribute_name)  
```

Und hier ein Beispiel, das dynamisch die Existenz von Attributen in einer `Product` Klasseninstanz prüft:

```python
class Product:
    def __init__(self, name, price):
        self.name = name
        self.price = price

product_a = Product('T-Shirt', 25)

required_attributes = ['name', 'price', 'inventory_id']

for attr in required_attributes:
    if not hasattr(product_a, attr):
        print(f"ERROR: Product is missing the required attribute: '{attr}'")
    else:
        # Access the attributes dynamically once their existence is confirmed
        print(f'{attr}: {getattr(product_a, attr)}')

# Output:
# name: T-Shirt
# price: 25
# ERROR: Product is missing the required attribute: 'inventory_id'
```

Die ERROR-Ausgabe entstand, weil `inventory_id` in der `Product` Klasse und deren Instanz fehlt.

Zuletzt erlaubt `delattr()` Ihnen, ein Attribut dynamisch zu entfernen:

```python
delattr(object, attribute_name) 
```

Stellen Sie sich zum Beispiel vor, ein Objekt wurde vollständig verarbeitet, und Sie möchten vor dem Speichern der finalen Version alle sensiblen oder temporären Attribute entfernen, die eventuell existieren. Danach können Sie `dir()` verwenden, um die verbleibenden Attribute durchzugehen:

```python
class UserSession:
    def __init__(self, user_id, token):
        self.user_id = user_id
        self.auth_token = token # sensitive
        self.temp_counter = 0 # temporary

session = UserSession(101, 'a1b2c3d4e5')

# List of attributes to remove dynamically before "saving" the session
attributes_to_clean = ['auth_token', 'temp_counter']

# Dynamically remove specified attributes
for attr in attributes_to_clean:
    if hasattr(session, attr):
        delattr(session, attr)
        print(f'Removed attribute: {attr}')

print('\nFinal attributes remaining:')

# Loop through the remaining attributes with dir()
for attr in dir(session):
    # Ignore dunder methods like __init__ or __str__ and regular methods
    if not attr.startswith('__') and not callable(getattr(session, attr)):
        print(f' - {attr}: {getattr(session, attr)}')

# Output:
# Removed attribute: auth_token
# Removed attribute: temp_counter

# Final attributes remaining:
#  - user_id: 101
```

So können Sie Attribute dynamisch handhaben!
