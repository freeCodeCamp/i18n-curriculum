# --description--

让我们了解常见的 HTTP `response` 代码。

超文本传输协议响应代码是网络服务器用来与客户通信的三位数字代码，用以告知他们的请求发生了什么。

它们提供有关请求是否被成功处理或在处理请求时是否发生了误差的信息。

我们将在本讲中介绍一些最常见的。

## 成功代码

让我们从**成功代码**开始。这些代码以数字 2 开头。

代码 **200 OK** 是最常见的超文本传输协议响应代码。它表示请求成功，服务器返回了客户请求的信息。

当客户请求创建资源且进程成功完成时，服务器会返回代码 **201 Created**。例如，如果新建用户账户成功，服务器会向客户返回此响应代码。

当请求已成功被处理但服务器没有内容返回时，会返回代码 **204 No Content**。这可用于删除或更新数据库中资源但无需返回数据的进程。

## **重定向代码**

现在让我们谈谈**重定向代码**。

当资源请求被客户移动到不同位置或客户需要采取额外步骤来完成请求时，服务器会返回这些代码。

这些代码以数字 3 开头。

当客户请求的资源已永久移动到另一个位置时，会返回代码 **301 Moved Permanently**。在这种情况下，服务器应在响应中包含指向新位置的 URL，一旦客户收到响应，用户应被重定向到该 URL。

代码 **302 Found** 表示客户请求的资源已被临时移动到另一个位置。服务器应在响应中提供新的 URL，客户应将用户重定向到该 URL。然而，原始 URL 未来可能仍然可用，因为此更改是临时的。这是响应代码 301 和 302 之间的主要区别。

服务器返回代码 **304 Not Modified**，以告知客户所请求的资源自上次请求以来未被修改。服务器发送此代码以避免在内容未更改时重复发送已发送的数据。这是一种性能优化技术。

## 客户误差代码

现在我们将开始深入研究**误差代码**。

误差可能由客户和服务器引起。我们将首先讨论客户误差代码。

当请求因客户生成的问题而无法完成时，会返回客户误差代码。

这些误差代码以数字 4 开头。

当服务器无法理解客户发送的请求时，会返回 **400 Bad Request** 代码。这可能发生在客户发送了语法错误、参数无效或数据无效的请求时。

**401 Unauthorized** 状态码与认证相关。它表示客户必须提供某些认证凭据才能访问正在请求的资源。这些凭据可能包括用户名和密码。

当客户无权访问请求的资源时，会返回 **403 Forbidden** 代码。401 和 403 的区别在于，403 表示客户即使拥有有效的认证凭据，也没有请求特定资源所需的权限。

当服务器上找不到资源时，会返回 **404 Not Found** 代码。可能是资源被删除了，或者 URL 中有拼写错误，导致服务器无法查找该资源。服务器也可能遇到临时问题，导致出现此误差代码。

## 服务器误差代码

接下来，我们有**服务器误差代码**。当请求因服务器端的问题无法处理时，服务器会返回这些代码。

**500 内部服务器误差** 代码表示服务器上发生了意外的错误。此误差可能由多种问题引起，范围从配置问题到临时问题。

当服务器尝试与另一台服务器通信但从该服务器收到无效的响应时，会返回 **502 Bad Gateway** 误差代码。此类误差有多种原因。

**503 服务不可用** 误差代码表示服务器暂时无法处理该请求，因此客户应稍后再试。此误差的常见原因包括维护任务和服务器当前无法处理的大量请求。

超文本传输协议响应代码对于理解客户与网络服务器之间的通信至关重要。通过理解这些代码，你可以排查和调试你的网络应用。
