# --description--

L'ultima sfida ha introdotto un metodo all'oggetto `duck`. Ha usato la notazione a punto `duck.name` per consultare il valore della proprietà `name` all'interno dell'istruzione return:

```js
sayName: function() {return "The name of this duck is " + duck.name + ".";}
```

Anche se questo è un modo valido per consultare la proprietà dell'oggetto, c'è un problema. Se il nome della variabile cambia, qualsiasi codice che fa riferimento al nome originale dovrebbe essere aggiornato di conseguenza. In una definizione di oggetto breve non è un problema, ma se un oggetto ha molti riferimenti alle sue proprietà c'è una maggiore possibilità di errore.

Un modo per evitare questi problemi è usare la parola chiave `this`:

```js
let duck = {
  name: "Aflac",
  numLegs: 2,
  sayName: function() {return "The name of this duck is " + this.name + ".";}
};
```

`this` è un argomento complesso, e l'esempio sopra è solo uno dei modi per usarlo. Nel contesto attuale, `this` indica l'oggetto a cui il metodo è associato: `duck`. Se il nome dell'oggetto viene cambiato in `mallard`, non è necessario trovare tutti i riferimenti a `duck` nel codice. Questo rende il codice riutilizzabile e più facile da leggere.
