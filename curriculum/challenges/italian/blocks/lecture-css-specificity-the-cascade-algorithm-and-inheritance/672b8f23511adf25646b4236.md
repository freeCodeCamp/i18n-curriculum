# --interactive--

L'algoritmo della cascade è il processo che il browser usa per decidere quali regole CSS applicare quando ci sono più stili che puntano allo stesso elemento. Garantisce che vengano usati gli stili più appropriati, basandosi su un insieme di regole ben definite.

Il processo inizia con la **rilevanza**. Il browser filtra prima tutte le regole CSS per trovare quelle che effettivamente si applicano all'elemento in questione. Questo include il confronto dei selettori e la considerazione delle media query che potrebbero essere attive.

Una media query è una tecnica CSS usata per applicare stili basati sulle caratteristiche del dispositivo o della viewport, come la larghezza, l'altezza o l'orientamento.

Successivamente, l'algoritmo considera **origine e importanza**. Il CSS può provenire da fonti diverse: gli stili predefiniti del browser (user-agent), gli stili impostati dall'utente e gli stili scritti dall'autore (te).

Dopo aver considerato l'origine, l'algoritmo valuta l'importanza di ogni regola, dando priorità alle regole contrassegnate con `!important`, che sovrascrivono altre regole indipendentemente dalla loro origine.

Dopo il filtro per origine e importanza, l'algoritmo guarda alla **specificità**. Quando due regole della stessa origine e livello di importanza si applicano, viene applicata la regola con la specificità più alta.

La specificità è una misura di quanto un selettore sia mirato, con selettori più specifici che hanno la precedenza su quelli più generici.

Infine, se tutto il resto è uguale, entra in gioco **l'ordine di apparizione**. Quando due regole hanno la stessa specificità, viene applicata quella che appare per ultima nel CSS.

Ecco perché l'ordine in cui scrivi i tuoi stili può a volte influenzare il risultato.

Diamo un'occhiata a un esempio.

Nel file HTML c'è un singolo elemento paragrafo. Poi nel CSS abbiamo due regole, ciascuna che punta all'elemento paragrafo.

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<p>example paragraph</p>
```

```css
p {
  color: blue;
}

p {
  color: green; 
}
```

:::

La prima regola imposta il colore del testo di tutti gli elementi paragrafo su blu mentre la seconda regola imposta il colore del testo di tutti gli elementi paragrafo su verde.

Quindi quale colore verrà applicato? Il colore verde verrà applicato agli elementi paragrafo.

Considerando rilevanza, origine e importanza, specificità, ambito e ordine di apparizione, l'algoritmo della cascade garantisce che il tuo CSS si comporti in modo prevedibile, permettendoti di progettare pagine web più complesse e sfumate.
