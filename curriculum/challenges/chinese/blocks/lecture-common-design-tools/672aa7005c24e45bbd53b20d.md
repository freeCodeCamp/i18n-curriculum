# --description--

在设计新建特色或应用时，一个好的第一步是创建设计简介。

设计简报是一份概述项目目标、宗旨和需求的文档。它是指导设计进程的路线图，确保最终产品满足客户的需求。

通常客户会编写设计简报，并将其作为工作草稿。有时，设计师可能会编写设计简报并与客户协商，以确保它符合他们的需求。

设计简报中应包含几个关键元素。

第一个元素是项目和业务的概述。该概述应包括公司的详细信息、使命、价值观、独特卖点以及产品或服务。

下一个关键元素应是为项目记录目标和宗旨。这应包括项目的目的和期望的结果。

目标示例包括增加网站流量或将每月页面访问量提高 X％。

另一个关键元素是目标受众。设计简介应包括关于受众的目标人口统计、兴趣和需求的信息。

你还应该包括有关竞争的信息以及该项目将如何区别于竞争对手。

另一个关键元素是项目作用域。它应包括交付物、时间表和预算。交付物应包括作为项目一部分将产生的所有项的列表，例如模型和最终设计。

如果不清晰地定义项目作用域，事情可能会失控并超出预算。因此，最好尽可能详细地说明预期交付的内容以及交付时间。

项目设计中一个具有挑战性的方面是时间尺度和预算。对在给定时间范围和预算内可以实现的内容保持现实非常重要。因此，拥有一个概述这些约束的设计简报非常重要。

一旦所有这些细节都已讨论并记录，设计简报应在项目开始前由所有利益相关者审查并批准。到那时，设计师才能开始他们的工作。

那么，开发者在这一切中的角色是什么？开发者的角色是接受设计，理解项目需求，并将其转化为可运行的产品。

这涉及编写代码、测试和调试应用，以确保其符合设计简报中列出的要求。

开发者经常会在团队中工作，工作由多个开发者分担。

通常还会有一个项目经理，负责协调工作并确保项目按计划进行。

因此，虽然作为开发者你可能不会参与设计和初始决策进程，但理解设计简报及其如何影响你的工作仍然很重要。
