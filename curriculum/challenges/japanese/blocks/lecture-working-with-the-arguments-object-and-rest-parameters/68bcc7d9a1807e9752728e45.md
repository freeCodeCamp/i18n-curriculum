# --interactive--

前のレッスンでは、関数に渡された引数の値を含む配列のようなオブジェクトである`arguments`オブジェクトの扱い方を学びました。

:::interactive_editor

```js
function logArgs() {
  for (const arg of arguments) {
    console.log(arg);
  }
}

logArgs(1, 2, 3);
// result:
// 1
// 2
// 3
```

:::

これは関数から可変の引数をアクセスして扱う有効な方法ですが、現代のJavaScriptアプリケーションでは代わりにrestパラメータ構文を使います。

こちらは`arguments`オブジェクトの代わりにrestパラメータ構文を使った更新された例です：

:::interactive_editor

```js
function logArgs(...args) {
  for (const arg of args) {
    console.log(arg);
  }
}

logArgs(1, 2, 3);
// result:
// 1
// 2
// 3
```

:::

この更新された例ではもはや`arguments`オブジェクトを直接参照していません。代わりに関数定義の最後のパラメータの前に3つのドットが付いています。これにより、このrestパラメータは`Array`オブジェクトの中に格納されます。このrestパラメータは好きな名前を付けられます。ただし、関数定義の最後のパラメータであることを必ず守ってください：

```js
function exampleFunction(a, b, ...restOfArgs) {
  // some code here
}

function anotherFunction(x, y, ...theArgs) {
  // some code here
}
```

restパラメータ構文にはいくつかの制限があります。1つの制限は、関数定義にrestパラメータは1つだけしか持てないことです。次の例は無効とみなされます。

```js
// Won't work

function badFunction(...args, ...moreArgs) {
  // some code here
}
```

次の制限は、restパラメータの後に末尾のカンマは許されないことです：`function exampleFunction(a, b, ...restOfArgs, )`。

さらに、restパラメータにデフォルト値を設定することはできません。そうしないと`SyntaxError`がスローされます。

```js
function badFunction(...args = [1,2]){
  // some code here
}
```

では、`arguments`オブジェクトと`rest`パラメータの違いは何でしょうか？

大きな違いの1つは、`arguments`オブジェクトは本物の配列ではないため、`includes`、`pop`、`push`のようなメソッドをサポートしていません。しかしrestパラメータは`Array`のインスタンスです。したがって、本物の配列に変換しなくても有効な組み込み配列メソッドを使えます。

:::interactive_editor

```js
function hasCat(...args) {
  return args.includes("cat");
}

console.log(hasCat("dog", "chicken", "cat")); // true
console.log(hasCat("dog", "chicken", "horse")); // false
```
