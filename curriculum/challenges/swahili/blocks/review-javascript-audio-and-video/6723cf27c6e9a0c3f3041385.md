## `Audio` Kijenzi na Njia za Kawaida

- **Maelezo**: Kijenzi cha `Audio`, kama vijenzi vingine, ni kitendakazi maalum kinachoitwa kwa neno kuu la `new`. Hurejesha `HTMLAudioElement`, ambacho unaweza kisha kutumia kuchezesha sauti kwa mtumizi au kuiongeza kwenye DOM ili mtumizi aweze kuikagua mwenyewe. Unapoitisha kijenzi, unaweza hiari kupeleka URL kama hoja (ya pekee). URL hii inapaswa kuonyesha chanzo cha faili la sauti unalotaka kuichezesha. Au, ikiwa unahitaji kubadilisha chanzo kwa njia ya mabadiliko, unaweza weka thamani ya URL kwenye sifa ya `src` ya kipengele cha sauti kilichorejeshwa.
- **Njia ya `play()`**: Njia hii hutumika na vipengele vya `audio` au `video` kuanza kuchezesha media.

:::interactive_editor

```html
<audio id="audio" controls>
  <source src="https://www.w3schools.com/html/horse.mp3" type="audio/mpeg">
</audio>

<button id="playBtn">Play Audio</button>

<script>
  const audio = document.getElementById("audio");
  const playBtn = document.getElementById("playBtn");

  playBtn.addEventListener("click", () => {
    audio.play();
  });
</script>
```

:::

- **Njia ya `pause()`**: Njia hii hutumika na vipengele vya `audio` au `video` kusitisha kuchezesha media. Huhifadhi nafasi ya sasa, hivyo ikiwa `play()` itaitwa, huanza kutoka nafasi hiyo.

:::interactive_editor

```html
<audio id="myAudio" controls>
  <source src="https://www.w3schools.com/html/horse.mp3" type="audio/mpeg">
</audio>

<br>

<button id="pauseBtn">Pause Audio</button>

<script>
  document.addEventListener("DOMContentLoaded", () => {
    const audio = document.getElementById("myAudio");
    const pauseBtn = document.getElementById("pauseBtn");

    pauseBtn.addEventListener("click", () => {
      audio.pause();
    });
  });
</script>
```

:::

- **Njia ya `addTextTrack()`**: Njia hii inakuwezesha kubainisha njia ya maandishi kuhusishwa na kipengele cha media - ambayo ni msaada hasa kwa kuongeza manukuu kwenye video.
- **Njia ya `fastSeek()`**: Njia hii inakuwezesha kusogeza nafasi ya kuchezesha hadi wakati maalum ndani ya media.

Kijenzi cha `Audio()` kina sifa kama vile kina njia. Sifa hizo ni pamoja na:

- `currentTime`: kwa kupata wakati wa sasa wa kuchezesha sauti
- `loop`: kwa kufanya sauti ichezeendelee kwa kuanzisha upya moja kwa moja inapofika mwisho
- `muted`: kwa kutuliza sauti ya kipengele cha media bila kujali kiwango cha sauti kilichowekwa

## Aina tofauti za sauti na video

- **Aina ya MIME**: Aina ya MIME, inayomaanisha Multipurpose Internet Mail Extensions, ni njia iliyosanifiwa kuonyesha aina ya faili kwa njia ya programu. Aina ya MIME inaweza kumwambia programu, kama kivinjari chako, jinsi ya kushughulikia faili fulani. Katika kesi ya sauti na video, aina ya MIME inaonyesha kuwa ni muundo wa vyombo vya habari unaoweza kuingizwa kwenye ukurasa wa mtandao.
- **Kipengele cha `source`**: Hiki hutumika kubainisha aina ya faili na chanzo - na kinaweza kujumuisha aina nyingi tofauti kwa kutumia vipengele vingi vya chanzo. Unapofanya hivi, kivinjari kitaamua muundo bora wa kutumia kwa mazingira ya mtumizi kwa sasa.
- **MP3**: Hii ni aina ya muundo wa faili wa kidijitali unaotumika kuhifadhi muziki, sauti, au sauti nyingine. Ni toleo lililobanwa la kurekodi sauti ambalo hufanya ukubwa wa faili kuwa mdogo, hivyo ni rahisi kuhifadhi na kushiriki. MP3 ina msaada mpana zaidi kwenye vivinjari na aina ya MIME ya `audio/mpeg`.
- **MP4**: MP4 ni aina ya muundo wa faili wa kidijitali unaotumika kuhifadhi video na sauti. Hutumika kama kontena linaloshikilia video (picha) na sauti (muziki au hotuba) katika faili moja. MP4 inaweza kuwa na aina ya MIME `audio/mp4` AU `video/mp4`, kulingana na ikiwa ni faili ya video au sauti tu.
- Aina nyingine ni `WMV` inayohusiana na programu ya Windows Media Player, `OGG`, `MKV`, `AVI`, na nyingine zaidi.

## codecs

- **Maelezo**: Codec, kifupi cha "encoder/decoder", ni algoriti au programu inayoweza kubadilisha sauti na video kati ya muundo wa analogue na kidijitali. Codecs zinaweza kubainishwa kama sehemu ya aina ya MIME. Sintaksia ya msingi ya kubainisha codec ni kuongeza nukta-koma baada ya aina ya media, kisha `codecs=` na codec.

## API ya HTMLMediaElement

- **Maelezo**: API ya `HTMLMediaElement` hutumika kudhibiti tabia za vipengele vya sauti na video kwenye ukurasa wako. Inapanua kiolesura cha msingi cha `HTMLElement`, hivyo una upatikanaji wa sifa za msingi, njia pamoja na matukio haya ya msaada kama `waiting`, `ended`, `canplay`, `canplaythrough`, na mengine. Mifano ya njia ni pamoja na `play()`, `fastSeek()`, `pause()`, na `canPlayType()` kwa kuangalia kama kivinjari kinaweza kuchezesha faili la sauti.

## API ya Media Capture na Streams

- **Maelezo**: API ya Media Capture na Streams, au API ya MediaStream, hutumika kunasa sauti na video kutoka kwa kifaa chako. Ili kutumia API hii, unahitaji kuunda kitu cha `MediaStream`. Unaweza kufanya hivi kwa kijenzi, lakini kingehusishwa moja kwa moja na vifaa vya mtumizi. Badala yake, sifa ya `mediaDevices` ya kitu cha `global` navigator ina njia ya `getUserMedia()` ambayo unaweza kutumia.

```js
window.navigator.mediaDevices.getUserMedia({
  audio: true,
  video: {
    width: {
      min: 1280,
      ideal: 1920,
      max: 3840
    },
    height: {
      min: 720,
      ideal: 1080,
      max: 2160
    }
  }
});
```

## API ya Screen Capture

- **Maelezo**: API ya Screen Capture hutumika kurekodi skrini ya mtumizi. API hii inapatikana kwa kuitisha njia ya `getDisplayMedia()` ya kitu cha `mediaDevices` na kutumia mtiririko wa media uliorejeshwa.

## API ya MediaStream Recording

- **Maelezo**: API ya MediaStream Recording hufanya kazi sambamba na API za MediaStreams, ikikuruhusu kurekodi MediaStream (au hata `HTMLMediaElement` moja kwa moja).

## API ya Media Source Extensions

- **mada**: API ya Media Source Extensions ndiyo inayokuwezesha kupitisha moja kwa moja picha ya kamera ya mtumizi kwenye kipengele cha video kwa sifa ya `srcObject`.

## API ya Web Audio

- **Maelezo**: API ya Web Audio ndiyo inayosimamia kila kitu kinachosikika kwenye mtandao. API hii inajumuisha vitu muhimu kama `AudioBuffer` (wakilisha Kihifadhi cha Muda kinachoshikilia data ya sauti) au `AudioContext`, inayotumika kuwakilisha mchoro wa usindikaji wa sauti.
