# --description--

지금까지는 JavaScript로 클라이언트 측 코드를 작성해 왔습니다. 클라이언트 측 코드는 보통 웹 브라우저 같은 웹 애플리케이션의 클라이언트 쪽에서 실행됩니다.

클라이언트 측 JavaScript는 매우 강력할 수 있지만, 더 복잡한 작업에는 몇 가지 제한이 있습니다.

예를 들어:

- 로컬 파일에 대한 접근이 매우 제한적입니다.
- 복잡한 애플리케이션 로직을 처리하도록 설계되지 않았습니다.
- 보안 문제도 있을 수 있습니다. 예를 들어, 데이터베이스 자격 증명 같은 민감한 정보를 클라이언트 측 코드에 노출하고 싶지 않습니다.
    
클라이언트 측 JavaScript가 이런 제한을 가지는 이유는 처음에 웹 브라우저에서만 실행되도록 설계되었기 때문입니다.

웹 브라우저는 JavaScript 코드를 실행하는 데 필요한 환경을 제공합니다. 여기에는 JavaScript 엔진도 포함됩니다. 또한 문서 객체 모델(DOM)에 접근할 수 있어 코드에서 HTML 요소에 접근할 수 있습니다.

이전에는 브라우저 외부에서 JavaScript를 실행할 수 없었지만, 이제는 Node.js로 가능합니다.

Node.js는 JavaScript 런타임 환경입니다. 공식적으로는 “오픈 소스이자 크로스 플랫폼 JavaScript 런타임 환경”으로 정의됩니다.

이것이 의미하는 바를 살펴보겠습니다:

- Node.js는 코드가 공개되어 있고 많은 개발자 커뮤니티가 유지 관리하기 때문에 오픈 소스입니다.
- Windows, macOS, Linux 등 어떤 운영체제에서도 작동하기 때문에 크로스 플랫폼입니다.
- 브라우저 외부에서 JavaScript 코드를 실행할 수 있게 해주기 때문에 JavaScript 런타임 환경입니다.
    
Node.js는 다양한 산업 분야에서 사용됩니다. 개발자들은 HTTP 요청을 처리하는 웹 서버와 API, 그리고 웹 및 모바일 애플리케이션을 구축하는 데 사용합니다.

데이터베이스와 상호작용하거나 복잡한 데이터를 처리하는 웹사이트나 웹 애플리케이션을 사용하거나 개발한다면, 언젠가 Node.js를 백엔드에서 사용하게 될 가능성이 높습니다.

Node.js는 효율적이고 확장 가능하며, 전 세계에 많은 개발자와 유지 관리자가 있습니다.

Node.js가 무엇인지 알았으니, 이제 브라우저와 Node 런타임 환경 간의 차이점을 살펴보겠습니다.

브라우저 환경은 주로 프런트엔드 웹 개발을 위해 설계되어 클라이언트 측 JavaScript를 실행합니다. 반면 Node 런타임 환경은 주로 백엔드 웹 개발을 위해 설계되어 서버 측 JavaScript를 실행합니다. 이것이 제공하는 API도 결정합니다.

예를 들어, 브라우저에서는 DOM API에 접근할 수 있지만 로컬 파일 시스템 접근에는 제한이 있습니다. 반면 Node 런타임 환경에서는 파일 시스템을 포함한 거의 모든 시스템 자원에 접근할 수 있지만 DOM에는 접근할 수 없습니다.

두 JavaScript 환경 모두 전역 객체를 제공합니다. 브라우저 JavaScript 런타임에서는 이것을 `window`라고 부릅니다. 이 `window` 객체는 DOM 조작, 쿠키 관리, 브라우저 이벤트 처리 같은 브라우저 관련 기능에 접근할 수 있게 해줍니다. Node.js 환경에서는 이 전역 객체를 `global`라고 부릅니다. `global` 객체는 로컬 파일 작업, 네트워킹, 운영체제와 상호작용하는 내장 모듈 같은 Node.js 고유 기능에 접근할 수 있게 해줍니다.

마지막으로 중요한 차이점은 서버에서 사용할 Node.js 버전을 선택할 수 있지만, 사용자가 웹사이트를 방문할 때 사용하는 브라우저 환경 버전은 제어할 수 없다는 점입니다.

Node.js는 전 세계 개발자에게 매우 중요해졌습니다. 강력하고 확장 가능하며 다재다능한 도구로, 개발자가 JavaScript를 사용해 프런트엔드와 백엔드 모두를 구축할 수 있게 하여 웹 개발을 혁신했습니다. 덕분에 백엔드 개발을 위해 새로운 프로그래밍 언어를 배울 필요가 없어 개발 과정이 더 효율적입니다.
