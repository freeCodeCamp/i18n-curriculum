# --description--

I siti web e le app devono raccogliere dati e elaborarli, oppure generare qualcosa di cui l’utente ha bisogno, come i risultati di una ricerca. Quindi, proprio come il testo, anche i moduli sono una parte fondamentale del web.

Con i moduli, gli utenti possono registrarsi e accedere a un sito, inviare messaggi e cercare elementi. Per esempio, accedi alla pagina dei risultati di ricerca di Google digitando la tua query nel modulo di ricerca più popolare sul web, il campo di input di ricerca di Google.

Vediamo cosa compone un modulo, come i campi di input, l’attributo `action`, i metodi del modulo e, soprattutto, cosa succede quando invii un modulo.

I moduli sono incorporati in una pagina web con l’elemento `form`, ma per far apparire gli input, hai anche bisogno degli elementi `input`, a seconda delle tue necessità. Alcuni di questi elementi di input sono testo, email, password, selezione, file, input nascosti e altro.

Per usare questi campi di input, devi racchiuderli nell’elemento `form`. Ognuno degli input ha bisogno di un attributo `name` così il server sa a quali dati corrisponde ogni campo.

Ecco un esempio:

```html
<form>
   <input type="text" name="username" />
   <input type="email" name="email" />
   <input type="file" name="profilePicture" />
   <input type="radio" name="gender" />
   <input type="checkbox" name="subscribe" />
   <input type="hidden" name="userId" />
</form>
```

I campi di input devono anche essere validati per assicurarsi che l’utente inserisca le informazioni corrette.

Questa validazione può avvenire sia lato client sia lato server. HTML ti permette di fare una validazione base lato client con gli attributi `required` e `pattern`. Tuttavia, dovresti anche validare lato server per una maggiore sicurezza, dato che la validazione HTML serve soprattutto per l’esperienza utente e un feedback immediato.

L’attributo `required` specifica che il campo di input deve essere compilato, mentre l’attributo `pattern` ti permette di inserire espressioni regolari che obbligano l’utente a digitare le informazioni nel modo che ti serve.

Ecco un esempio di questo per un campo username:

```html
<input type="text" name="username" required pattern="[A-Za-z0-9]{3,}" />
```

Dopo che i campi di input sono stati validati, deve avvenire l’invio. Ma cosa succede quando il modulo viene inviato?

Prima di tutto, per far avvenire l’invio, il modulo deve avere un attributo `action` che specifica dove il browser deve inviare i valori dei campi di input e un attributo `method` che specifica come vengono inviati i dati del modulo. Approfondiremo l’attributo `method` più avanti.

Ecco come potrebbe apparire un attributo action:

```html
<form action="/register">
	<!-- input fields -->
</form>
```

L’attributo `method` può essere `GET`, `POST`, `PUT`, `PATCH` o `DELETE`:

* `GET` recupera qualcosa dal server, per esempio i risultati di una ricerca.
    
* `POST` invia qualcosa al server, per esempio le informazioni di login o registrazione inserite nei campi di input.
    
* `PUT` modifica qualcosa sul server.
    
* `PATCH` modifica parzialmente qualcosa sul server.
    
* `DELETE` elimina qualcosa dal server.
    
Nota che HTML non supporta direttamente i metodi `PUT`, `PATCH` e `DELETE`, ma esistono soluzioni alternative. Se lavori con JavaScript vanilla, puoi usare la Fetch API per specificare il metodo, e in Node.js e Express puoi usare il pacchetto `method-override`.

Ecco come potrebbe apparire un modulo che fa una richiesta `GET`:

```html
<form action="/search" method="GET">
   <input type="text" name="q">
   <button type="submit">Search</button>
</form>
```

Ecco come potrebbe apparire un modulo che fa una richiesta `POST`:

```html
<form action="/submit" method="POST">
   <input type="text" name="username" required pattern="[A-Za-z0-9]{3,}" />
   <input type="password" name="password" />
   <button type="submit">Submit</button>
</form>
```

Dopo la richiesta `GET` o `POST`, il server verifica la validazione lato server e o elabora o rifiuta i valori di input.

Se la validazione ha successo, il server invia le informazioni richieste o lancia un errore se la validazione fallisce.

Se la validazione ha successo, la risposta del server potrebbe essere un reindirizzamento in caso di autenticazione o ricerca, oppure un messaggio di successo che conferma l’azione.

Ecco un riepilogo di cosa succede quando un modulo viene inviato:

* L’utente compila i campi del modulo.
    
* Il browser verifica la validazione lato client (se presente) e procede se la validazione ha successo.
    
* Il browser invia i dati del modulo al server in base agli attributi action e method.
    
* Il server riceve i dati e li elabora (salvandoli in un database, registrando un utente, effettuando il login, inviando la query di ricerca e così via).
    
* Se c’è una validazione lato server, il server ricontrolla i dati prima di elaborarli.
    
* Il server risponde con un messaggio di successo, un errore o reindirizza l’utente in base allo stato della validazione (successo o fallimento).
    
* L’utente vede un feedback basato sulla risposta del server.
