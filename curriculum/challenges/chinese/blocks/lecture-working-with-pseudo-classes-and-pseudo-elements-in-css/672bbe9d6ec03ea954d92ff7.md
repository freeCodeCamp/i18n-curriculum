# --interactive--

输入元素的外观和行为在创建网页表单时非常重要。具有响应用户操作的输入的表单在提升用户体验方面效果显著，且不应引起混淆或挫败感。

CSS 提供了几种输入伪类，可以使你的表单更直观、更易用。让我们详细了解这些伪类。

`:focus` 伪类允许你在用户选择或点击输入元素时定位该元素，从而突出显示活动输入字段。这有助于用户轻松识别他们当前正在输入的位置：

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css" />
<form>
  <input type="text" />
</form>
```

```css
input:focus {
  border: 2px solid crimson;
  outline: none;
}
```

:::

顾名思义，`:hover` 伪类在用户将光标放置在元素上时触发。它在用户与输入交互之前提供视觉反馈，有效地提醒他们输入已准备好操作。

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css" />
<form>
  <input type="text" />
</form>
```

```css
input:hover {
  background-color: orange;
}
```

:::

在上面的示例中，当用户将鼠标悬停在输入元素上时，背景会变成橙色，提示他们该字段已准备好进行交互。

`:checked` 伪类允许你在用户选择单选按钮或复选框时以不同的样式显示。这样，用户可以轻松查看他们选择了哪个选项。

这是一个带有复选框以同意网站条款的示例。 

**注意**：此示例中的部分 CSS 使用了尚未讲解的属性。这只是为了让你了解如何创建一个自定义复选框。你将在后续课程和研讨会中学习所有这些内容的工作原理。 

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css" />
<form>
  <label>
  Agree <input class="checkbox" type="checkbox" />
  </label>
</form>
```

```css
.checkbox {
  appearance: none;
  width: 18px;
  height: 18px;
  border: 2px solid #ccc;
  border-radius: 4px;
  display: inline-block;
  position: relative;
  cursor: pointer;
  transition: all 0.25s ease;
  vertical-align: middle; 
}

.checkbox:hover {
  border-color: #888;
}

.checkbox:checked {
  background-color: #4caf50;
  border-color: #4caf50;
}

.checkbox:checked::after {
  content: "";
  position: absolute;
  left: 4px;
  top: 0px;
  width: 5px;
  height: 10px;
  border: solid white;
  border-width: 0 2px 2px 0;
  transform: rotate(45deg);
}

.checkbox:focus {
  outline: 2px solid #90caf9;
  outline-offset: 2px;
}

```

:::

`:required` 选择具有 `required` 属性的输入元素。它向用户表示必须填写该字段才能提交形式。

这是一个包含一些必填输入字段的形式示例：

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css" />
<form action="#">
  <label for="name">Name:</label>
  <input type="text" id="name" name="name" required />

  <label for="email">Email:</label>
  <input type="email" id="email" name="email" required />

  <label for="phone">Phone Number:</label>
  <input type="tel" id="phone" name="phone" />

  <button type="submit">Submit</button>
</form>
```

```css
input:required {
  border: 2px solid orange;
}
```

:::

在验证形式时，你可以使用 `:valid` 伪类来为符合验证条件的输入字段设置样式，使用 `:invalid` 来为不符合条件的输入字段设置样式。通常，你会为有效输入使用绿色，为无效输入使用红色。 

这是一个使用 `:valid` 伪类的示例：

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css" />
<form>
  <label for="email">Email:</label>
  <input type="email" id="email" name="email" />
</form> 
```

```css
input:valid {
  border-color: green;
}
```

:::

以下是使用 `:invalid` 伪类的示例：

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css" />
<form>  
  <label for="email">Email:</label>
  <input type="email" id="email" name="email" />
</form>
```

```css
input:invalid {
  border-color: crimson;
}
```

:::

`:disabled` 伪类允许你选择和样式化被禁用的输入元素。这些元素具有 `disabled` 属性，阻止用户与它们交互。当输入被禁用时，无法点击、聚焦或编辑。

这是一个标签和禁用 `input` 元素的示例。

在 CSS 中，我们针对那个禁用的输入，给它设置了 `lightgray` 的背景颜色，并将光标指针更改为 `not-allowed`。 

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css" />
<form>
  <label for="name">Name:</label>
  <input class="text-input" type="text" id="name" name="name" disabled />
</form>
```

```css
.text-input:disabled {
  background-color: lightgray;
  cursor: not-allowed;
}
```

:::

`cursor: not-allowed;` CSS 属性值改变光标的外观，以表示该操作不被允许。 

当应用于一个元素时，它会显示带有圆圈斜线符号（通常是带有对角线的圆圈）的光标，以向用户表示该元素不可交互或无法点击或操作。

以下是一些其他 `input` 伪类及其作用的示例：

- `:autofill`：将样式应用于浏览器自动填充已储存数据的输入字段。
- `:optional`：为非必填且可以留空的输入元素设置样式。
- `:in-range` 和 `:out-of-range`：根据元素的值是否在指定的范围约束内或外，对元素进行样式设置。
