# --interactive--

이전 수업에서, 서버에서 데이터를 가져오거나 파일에서 데이터를 읽는 등 시간이 오래 걸리는 작업을 기다리는 동안 다른 코드가 실행될 수 있게 하는 비동기 프로그래밍에 대해 배웠습니다.

`async`/`await`는 프로미스를 기반으로 하여 비동기 코드를 작성하고 읽기 쉽게 만듭니다. `async` 키워드를 함수 앞에 붙이면, 그 함수는 항상 `Promise`를 반환한다는 뜻입니다. `await` 키워드는 `Promise`가 해결될 때까지 기다렸다가 다음 코드 줄로 넘어가게 하는데, 비동기 함수 안이나 모듈의 최상위 본문에서만 사용할 수 있습니다. `async`/`await`가 어떻게 작동하는지 보여주는 예제를 살펴보세요:

:::interactive_editor

```js
async function delayedGreeting(name) {
  console.log("A Messenger entered the chat...");
  await new Promise(resolve => setTimeout(resolve, 2000));
  console.log(`Hello, ${name}!`);
}

delayedGreeting("Alice");
console.log("First Printed Message!");
```

:::

이 코드에서는 `async` 함수인 `delayedGreeting`을 정의했습니다. 이 함수 안에서 `await`를 사용해 실행을 2초 동안 멈춥니다. 지연 후 인사말을 출력합니다.

이 함수를 호출하면 인사말 앞에 `First Printed Message!`가 나타나는 것을 볼 수 있습니다. 이는 함수가 비동기이기 때문에 나머지 코드 실행을 막지 않기 때문입니다.

`async`/`await`의 가장 큰 장점 중 하나는 오류 처리입니다. 프로미스에서는 오류를 처리하기 위해 `.catch()` 메서드를 자주 사용했지만, `async`/`await`에서는 `try`/`catch` 블록을 사용할 수 있습니다. 다음 예제를 보세요:

```js
async function fetchUserData() {
  try {
    let response = await fetch(`https://api.example.com/users`);
    let userData = await response.json();
    console.log(userData);
  } catch (error) {
    console.log("Error fetching user data:", error);
  }
}

fetchUserData();
```

이 예제에서는 `async`/`await`를 사용해 API에서 사용자 데이터를 가져옵니다. `await` 키워드는 두 번 사용되었는데, 한 번은 fetch 작업이 완료될 때까지 기다리고, 또 한 번은 JSON 파싱이 끝날 때까지 기다립니다. 이 과정에서 오류가 발생하면 `catch` 블록에서 잡힙니다.
