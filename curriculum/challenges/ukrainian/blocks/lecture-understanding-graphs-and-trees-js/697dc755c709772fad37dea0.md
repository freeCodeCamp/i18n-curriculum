# --description--

**Пріоритетна черга** — це абстрактний тип даних (ADT), який працює подібно до черги або стеку, але з однією ключовою відмінністю.

Як ви, можливо, вже знаєте, стандартні черги працюють за принципом FIFO (перший прийшов — перший пішов), де перший доданий елемент черги є першим, який буде видалений.

Стек працює за принципом LIFO (останній прийшов — перший пішов), де останній доданий елемент стеку є першим, який буде видалений.

Черги та стеки враховують лише порядок додавання елементів.

Однак пріоритетні черги враховують «пріоритет» елементів. Пріоритет використовується для визначення, який елемент слід видалити наступним.

Зазвичай елемент з найвищим пріоритетом видаляється першим, але деякі реалізації можуть вибирати видаляти елемент з найнижчим пріоритетом першим. Це залежить від вимог вашої програми.

Пріоритетні черги дуже корисні для практичних застосувань, таких як пошук найкоротшого шляху між двома локаціями, планування завдань в операційних системах, моделювання трафіку, стиснення даних і керування мережами.

На практиці пріоритетні черги зазвичай реалізують за допомогою структури даних купи.

**Купа** — це дерево структура даних із дуже специфічною властивістю, яку називають **властивістю купи**. Ця властивість визначає відношення між кожним вузлом і його дочірніми елементами, залежно від типу купи.

Існують два основні типи куп:

* Макс-купа
    
* Мін-купа
    

У **макс-купі** значення кожного вузла є більшим або рівним значенню його дочірніх елементів.

У цьому прикладі ви бачите структуру дерева з вузлами 8, 7, 5, 2 та 1. Зверніть увагу, що вузол 7 більший за вузли 2 і 1, що відповідає властивості купи. Це справедливо і для всіх інших вузлів.

<img src="https://cdn.freecodecamp.org/curriculum/lecture-transcripts/how-do-priority-queues-and-heaps-work-1.png" alt="A max-heap tree structure showing nodes with values 8 at the root, 7 and 5 as children of 8, and 2 and 1 as children of 7, demonstrating that each parent node is greater than its children.">

Навпаки, у **мін-купі** значення кожного вузла є меншим або рівним значенню його дочірніх елементів.

У цьому прикладі ми маємо вузли зі значеннями 4, 7, 9, 12 та 15. Наприклад, вузол 7 менший за вузли 12 і 15, що відповідає властивості купи. Це також справедливо для всіх інших вузлів.

<img src="https://cdn.freecodecamp.org/curriculum/lecture-transcripts/how-do-priority-queues-and-heaps-work-2.png" alt="A min-heap tree structure showing nodes with values 4 at the root, 7 and 9 as children of 4, and 12 and 15 as children of 7, demonstrating that each parent node is less than its children.">

Властивість купи є ключовою, бо вона гарантує, що максимальний (або мінімальний) елемент (залежно від типу купи) завжди знаходиться на верху, що дуже полегшує його видалення.

На практиці купи зазвичай реалізують як масиви для ефективного доступу до батьківських і дочірніх вузлів.

Використання масивів спрощує логіку доступу до цих значень або «вузлів», бо за лаштунками, якщо купа підтримує структуру повного двійкового дерева, реалізація у вигляді масиву вимагає лише простих математичних операцій на основі індексів, щоб знайти, де елементи розташовані в пам’яті.

У JavaScript немає вбудованого модуля купи, але ви можете реалізувати мін-купу за допомогою масиву.

Ось базова реалізація мін-купи на JavaScript:

```js
class MinHeap {
  constructor(compare = (a, b) => a - b) {
    this.data = [];
    this.compare = compare;
  }

  peek() {
    return this.data[0];
  }

  push(value) {
    this.data.push(value);
    this.#bubbleUp(this.data.length - 1);
  }

  pop() {
    if (this.data.length === 0) return undefined;

    const top = this.data[0];
    const last = this.data.pop();

    if (this.data.length > 0) {
      this.data[0] = last;
      this.#bubbleDown(0);
    }

    return top;
  }

  pushPop(value) {
    if (this.data.length === 0) return value;

    if (this.compare(this.data[0], value) < 0) {
      const top = this.data[0];
      this.data[0] = value;
      this.#bubbleDown(0);
      return top;
    }

    return value;
  }

  heapify(arr) {
    this.data = arr.slice();
    for (let i = Math.floor(this.data.length / 2) - 1; i >= 0; i--) {
      this.#bubbleDown(i);
    }
  }

  #bubbleUp(i) {
    while (i > 0) {
      const p = Math.floor((i - 1) / 2);
      if (this.compare(this.data[i], this.data[p]) >= 0) break;
      [this.data[i], this.data[p]] = [this.data[p], this.data[i]];
      i = p;
    }
  }

  #bubbleDown(i) {
    const n = this.data.length;

    while (true) {
      let smallest = i;
      const l = 2 * i + 1;
      const r = 2 * i + 2;

      if (l < n && this.compare(this.data[l], this.data[smallest]) < 0) smallest = l;
      if (r < n && this.compare(this.data[r], this.data[smallest]) < 0) smallest = r;

      if (smallest === i) break;

      [this.data[i], this.data[smallest]] = [this.data[smallest], this.data[i]];
      i = smallest;
    }
  }
}
````

Щоб використати цю купу, ви можете створити порожню купу. Це буде базова структура даних для купи:

```js
const myHeap = new MinHeap();
```

Щоб додати елементи до купи, ви викликаєте `push()`. Це автоматично додасть елемент на потрібне місце, щоб зберегти властивість купи:

```js
myHeap.push(9);
```

Щоб отримати елемент з найвищим пріоритетом (у цьому випадку найменше значення), ви викликаєте `pop()`:

```js
myHeap.pop();
```

`pushPop()` поєднує обидві операції в один виклик.

Це ефективніше, ніж викликати їх послідовно окремо, особливо коли купа велика, бо виконується лише одна операція перестановки:

```js
myHeap.pushPop(15);
```

Якщо у вас вже є масив і ви хочете перетворити його на купу, ви можете викликати `heapify()`:

```js
myHeap.heapify([9, 2, 7, 1]);
```

Але наразі ми сортуємо елементи за їхніми значеннями, так?

Що, якби ми хотіли сортувати їх за «пріоритетом» замість цього?

Ви можете зробити це, зберігаючи масиви з такою структурою: `[priority, element]`.

У JavaScript масиви не порівнюються автоматично елемент за елементом для сортування, тому ви передаєте функцію порівняння до купи, яка спочатку порівнює пріоритети.

Зверніть увагу, що в цьому випадку нижчі значення означають вищий пріоритет. Це означає, що елемент з пріоритетом 1 матиме **вищий** пріоритет, ніж елемент з пріоритетом 3:

```js
const myHeap = new MinHeap((a, b) => a[0] - b[0]);

myHeap.push([3, "A"]);
myHeap.push([2, "B"]);
myHeap.push([1, "C"]);
```

Якщо вам потрібно, щоб елементи з однаковим пріоритетом видалялися в порядку їх додавання, можна додати унікальний лічильник як другий елемент вашого елемента, щоб розв’язати рівність, ось так `[priority, counter, element]`.

Наприклад, ви можете спочатку порівнювати за пріоритетом, а потім за лічильником:

```js
let counter = 0;
const myHeap = new MinHeap((a, b) => (a[0] - b[0]) || (a[1] - b[1]));

myHeap.push([3, counter++, "A"]);
myHeap.push([2, counter++, "B"]);
myHeap.push([2, counter++, "C"]);
```

Тепер поговоримо про ефективність куп.

Середня та гірша складність за часом для вставки та вилучення мінімального або максимального значення з купи (залежно від типу купи) є логарифмічною, `O(log n)`, бо кількість перестановок зазвичай пропорційна висоті купи, яка дорівнює log(n).

Середня та гірша складність за часом для операції «peek» є сталою, O(1). Операція peek полягає у отриманні мінімального або максимального значення (залежно від типу купи) без його видалення.

Операція «heapify», коли купа будується з не відсортованого списку, має лінійну складність за часом, O(n), у середньому та гіршому випадках.

Аналогічно, пошук і видалення довільного елемента мають лінійну середню та гіршу складність за часом O(n), оскільки потенційно вимагають обходу всієї купи.

А скільки вони займають пам’яті?

Складність за простором купи є лінійною, O(n), де `n` — це кількість елементів, які вона містить. Потрібно зберігати лише елементи та невеликий додатковий наклад для самого об’єкта масиву.

Пріоритетні черги та купи дуже важливі в інформатиці. Вони дозволяють швидко знаходити та використовувати найважливіші елементи зі списку. Ця ефективність є критичною для багатьох комп’ютерних програм, які виконують важливі завдання у реальному світі, наприклад, пошук найшвидшого маршруту на карті.
