# --description--

Katika funzo la awali, ulijifunza jinsi ya kuunda hifadhidata mpya kwenye GitHub. Lakini vipi kama ulikuwa na mradi uliopo ndani ya kompyuta unayotaka kuongeza kwenye GitHub? Katika funzo hili, utajifunza jinsi ya kusukuma hifadhidata za ndani zilizopo kwenye GitHub na kuvuta mabadiliko kutoka kwenye hifadhidata za mbali.

Anza kwa kuunda hifadhidata mpya kwenye GitHub bila mafaili yoyote. Ukimaliza kwa usahihi, unapaswa kuelekezwa kwenye ukurasa unaojumuisha URL inayohitajika kwa kunakili hifadhidata yako.

Sasa nenda kwenye terminal yako na chagua mradi uliopo unayotaka kuongeza kwenye GitHub. Katika funzo hili, ninachagua mradi wa ndani uitwao `super-awesome-game` wenye mafaili yafuatayo ndani:

```sh
jessicawilkins super-awesome-game >> ls
README.md  index.html script.js  styles.css
```

Ili kuanzisha udhibiti wa toleo kwa mradi huu kwa kutumia Git, utahitaji kuendesha amri ya `git init` katika saraka ya mradi. Hii itaanzisha hifadhidata tupu ya Git ili Git ianze kufuatilia mabadiliko ya mradi huu. Ukimaliza kwa usahihi unapaswa kuona matokeo yanayofanana kwenye terminal:

```sh
jessicawilkins super-awesome-game >> git init
Initialized empty Git repository in /Users/jessicawilkins/workspace/freeCodeCamp/super-awesome-game/.git/
```

Unapoanzisha hifadhidata tupu ya Git kwa mradi, saraka mpya ya siri ya `.git` itaongezwa. Ili kuona ongezeko la saraka hii unaweza kuendesha amri ya `ls -a` inayoorodhesha mafaili yote na saraka, ikiwa ni pamoja na zile zilizofichwa:

```sh
jessicawilkins super-awesome-game >> ls -a
.          ..         .git       README.md  index.html script.js  styles.css
```

Ni muhimu usifute saraka ya `.git`, vinginevyo historia yako yote ya Git itapotea.

Sasa baada ya hifadhidata tupu ya Git kuanzishwa, unaweza kuendesha amri ya `git status`. Amri hii hutumika kuonyesha hali ya sasa ya saraka yako ya kazi - utatumia amri hii mara nyingi katika mtiririko wako wa kazi. Unapaswa kuona matokeo yanayofanana na haya:

```sh
jessicawilkins super-awesome-game >> git status
On branch main

No commits yet

Untracked files:
  (use "git add <file>..." to include in what will be committed)
	README.md
	index.html
	script.js
	styles.css

nothing added to commit but untracked files present (use "git add" to track)
```

`On branch main` ina maana uko kwenye tawi la `main`. Tawi katika Git ni eneo tofauti la kazi ambapo unaweza kufanya mabadiliko. Tawi la `main` mara nyingi litaonyesha tawi kuu au la uzalishaji katika programu halisi. Timu za wasanidi programu hutoa matawi mengi kwa ajili ya vipengele vipya na kurekebisha hitilafu kisha kuunganisha mabadiliko hayo tena kwenye tawi la `main`. Utajifunza zaidi kuhusu mchakato huo katika funzo lijalo.

`Untracked files` ina maana Git imetambua mafaili ambayo kwa sasa haifuatilii.

Kuna hali tano ambazo faili inaweza kuwa nazo:

1. "Haifuatiliwi": Hii ina maana faili ni mpya kwenye hifadhidata, na Git haijawahi "kuiona" hapo awali.
2. "Imebadilishwa": Faili hii ilikuwepo katika ahadi iliyopita, na ina mabadiliko ambayo hayajafungiwa.
3. "Imepuuzwa": Huenda usiona mafaili yaliyepuuzwa kwenye Git, lakini IDE yako inaweza kuwa na kiashiria kwao. Mafaili yaliyepuuzwa hayajumuishwi katika shughuli za Git, kawaida kwa sababu zimetajwa katika faili la `.gitignore`.
4. "Imefutwa": Faili iliyofutwa ni kinyume cha faili isiyofuatiliwa - ni faili iliyokuwepo hapo awali, na imeondolewa.
5. "Imebadilishwa jina": Faili iliyobadilishwa jina ni faili ambayo yaliyomo hayjakubadilika, lakini jina au mahali pa faili limebadilishwa. Katika baadhi ya matukio, faili inaweza kuchukuliwa kuwa imebadilishwa jina hata kama ina mabadiliko madogo.

Unapokuwa na mabadiliko ambayo hayjafungiwa katika hifadhidata yako, utahitaji kwanza "kuziweka kwenye hatua ya kuandaa". "Eneo la kuandaa" ni faili maalum katika saraka yako ya `.git` ambayo ina taarifa zitakazowekwa katika ahadi yako inayofuata. Ahadi ni kwa maana ya picha ya hali ya sasa ya hifadhidata yako.

Kama unataka kuweka faili moja kwa moja, basi unaweza kutumia amri ifuatayo:

```sh
git add name-of-file
```

Kama unataka kuweka mabadiliko yote ambayo hayajawa kwenye hatua ya kuandaa, basi unaweza kutumia `git add .` Nukta (`.`) ni jina mbadala la saraka ya sasa unayopo. Wasanidi programu wengine wataonya dhidi ya kutumia `git add .` kwa sababu unaweza kuweka mafaili ambayo hukusudia. Ikiwa hilo litafanyika, unaweza "kutoa kwenye hatua ya kuandaa" kwa kutumia amri ifuatayo:

```sh
git reset name-of-file
```

Kwa ujumla, mradi tu unafuatilia mafaili yanayoongezwa kwa kutumia `git status` kabla ya kuyaweka kwenye hatua ya kuandaa, utakuwa sawa.

Mara tu unapoweka mafaili yako kwenye hatua ya kuandaa, basi utahitaji kuyafunga. Unaweza kufunga mabadiliko yako kwa kuendesha amri ya `git commit`. Hii itafungua mhariri unaopendelea ulioweka katika usanidi wa Git. Mara mhariri anapofunguka, unaweza kutoa ujumbe wa kina kuhusu mabadiliko yako. Pia unaweza kuchagua kutoa ujumbe mfupi kwa kutumia amri ya `git commit -m` kama ifuatavyo:

```sh
git commit -m "short message goes here"
```

Kama utatumia bendera ya `-m`, bado utataka kutoa ujumbe wa kuelezea ili kutoa muktadha wa mabadiliko. Hapa kuna mifano ya ujumbe mzuri unaweza kutumia:

```sh
git commit -m "feat(api): implement JWT-based authentication"
git commit -m "fix(auth): correct token expiration time"
git commit -m "refactor: simplify login flow logic"
```

`feat(api)` na `fix(auth)` zinafuata mtindo wa Conventional Commits ambao hutumika kutoa muktadha juu ya aina za mabadiliko katika ahadi hiyo.

Baada ya kufunga kazi yako, na kuendesha `git status` tena, unapaswa kuona matokeo yafuatayo kwenye terminal:

```sh
On branch main
nothing to commit, working tree clean
```

Ili kuona ahadi zako zote za awali kwa mradi huu unaweza kuendesha amri ya `git log`. Hii itaorodhesha ahadi zote za awali na taarifa muhimu kama mwandishi, tarehe ya ahadi, ujumbe wa ahadi na hash ya ahadi. Hash ya ahadi ni mfuatano mrefu unaotumika kama kitambulisho cha kipekee kwa ahadi.

Mara tu unapomaliza kufunga kazi yako, sasa unaweza kusukuma mabadiliko yako kwenye GitHub. Kwanza utahitaji kuanzisha muunganisho wa mbali kwa hifadhidata yako ya mbali. Ulijifunza kuhusu hili katika funzo la awali, lakini hapa ni ukumbusho kwa kutumia SSH:

```sh
git remote add origin git@github.com:your-github-username/name-of-repo.git
```

Hatua ya mwisho ni kusukuma mabadiliko yako kwenye GitHub kwa kutumia amri hii:

```sh
git push -u origin main
```

Bendera ya `-u` ni kifupi cha `--set-upstream`. Inaunda muunganisho kati ya tawi lako la ndani la `main` na tawi la mbali la `main` kwenye `origin`. Unahitaji kuitumia mara moja tu kwa kila tawi jipya. Baada ya hapo, unaweza tu kuendesha `git push` bila kutaja jina la tawi.

Ukimaliza kwa usahihi, unapaswa kuona matokeo yanayofanana kwenye terminal:

```sh
Enumerating objects: 3, done.
Counting objects: 100% (3/3), done.
Delta compression using up to 8 threads
Compressing objects: 100% (2/2), done.
Writing objects: 100% (3/3), 253 bytes | 253.00 KiB/s, done.
Total 3 (delta 0), reused 0 (delta 0), pack-reused 0 (from 0)
To https://github.com/jdwilkin4/super-awesome-game.git
 * [new branch]      main -> main
branch 'main' set up to track 'origin/main'.
```

Pia unapaswa kuona mabadiliko mapya kwenye ukurasa wako wa hifadhidata ya GitHub unapoonyesha upya ukurasa.

Kama unashirikiana na wengine kwenye mradi na unahitaji kuvuta mabadiliko mapya kutoka kwenye hifadhidata, unaweza kutumia amri ya `git pull`. Utajifunza zaidi kuhusu kufanya kazi kwa timu na ushirikiano katika funzo lijalo.
