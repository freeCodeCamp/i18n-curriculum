# --instructions--

Le password non dovrebbero mai essere archiviate in testo semplice. Devono essere archiviate come hash, nel caso in cui la lista delle password venga scoperta. Tuttavia, non tutti gli hash sono uguali.

Per questo progetto imparerai l'importanza di una buona sicurezza creando un cracker di password per scoprire password che sono state hashate usando SHA-1.

Crea una funzione che prende in input un hash SHA-1 di una password e restituisce la password se è una delle prime 10.000 password più usate. Se l'hash SHA-1 NON corrisponde a una password nel database, restituisci "PASSWORD NOT IN DATABASE".

La funzione deve hashare ogni password da `top-10000-passwords.txt` e confrontarla con l'hash passato alla funzione.

La funzione deve prendere un secondo argomento opzionale chiamato `use_salts`. Se impostato su true, ogni stringa di sale dal file `known-salts.txt` deve essere aggiunta in coda E in testa a ogni password da `top-10000-passwords.txt` prima di hashare e confrontare con l'hash passato alla funzione.

Ecco alcune password hashate per testare la funzione:

- `b305921a3723cd5d70a375cd21a61e60aabb84ec` dovrebbe restituire "sammy123"
- `c7ab388a5ebefbf4d550652f1eb4d833e5316e3e` dovrebbe restituire "abacab"
- `5baa61e4c9b93f3f0682250b6cf8331b7ee68fd8` dovrebbe restituire "password"

Ecco alcune password hashate per testare la funzione quando `use_salts` è impostato su `True`:

- `53d8b3dc9d39f0184144674e310185e41a87ffd5` dovrebbe restituire "superman"
- `da5a4e8cf89539e66097acd2f8af128acae2f8ae` dovrebbe restituire "q1w2e3r4t5"
- `ea3f62d498e3b98557f9f9cd0d905028b3b019e1` dovrebbe restituire "bubbles1"

La libreria `hashlib` è stata importata per te. Dovresti considerare di usarla nel tuo codice. <a href="https://docs.python.org/3/library/hashlib.html" target="_blank" rel="noopener noreferrer nofollow">Scopri di più su "hashlib" qui</a>.
