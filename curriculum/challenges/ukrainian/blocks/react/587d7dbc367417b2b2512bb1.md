# --description--

React — це бібліотека для відтворення інтерфейсу користувача з відкритим кодом, створена та підтримувана Facebook. Це чудовий інструмент для відтворення інтерфейсу користувача (UI) сучасних веб-застосунків.

React використовує розширення синтаксису JavaScript під назвою JSX, яке дозволяє писати HTML безпосередньо всередині JavaScript. Це має кілька переваг. Воно дає змогу використовувати повну програмну потужність JavaScript у HTML і допомагає зберігати код читабельним. Загалом JSX схожий на HTML, який ви вже вивчили, але є кілька ключових відмінностей, які будуть розглянуті в цих завданнях.

Наприклад, оскільки JSX є синтаксичним розширенням JavaScript, ви можете писати JavaScript безпосередньо всередині JSX. Для цього просто включіть код, який хочете трактувати як JavaScript, у фігурні дужки: `{ 'this is treated as JavaScript code' }`. Запам’ятайте це, оскільки це використовується в кількох наступних завданнях.

Однак, оскільки JSX не є дійсним JavaScript, код JSX потрібно скомпілювати у JavaScript. Транспайлер Babel — популярний інструмент для цього процесу. Для вашої зручності він уже доданий за лаштунками для цих завдань. Якщо ви випадково напишете синтаксично некоректний JSX, ви побачите, що перший тест у цих завданнях не пройде.

Варто зазначити, що під капотом завдання викликають `ReactDOM.render(JSX, document.getElementById('root'))`. Цей виклик функції розміщує ваш JSX у власному легковаговому представленні DOM від React. React потім використовує знімки свого DOM, щоб оптимізувати оновлення лише певних частин фактичного DOM.


```js
assert(Enzyme.shallow(JSX).contains('Hello JSX!'));
```
