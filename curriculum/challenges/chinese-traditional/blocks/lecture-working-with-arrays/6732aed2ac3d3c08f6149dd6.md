# --interactive--

在程式設計中，陣列是用來儲存元素群集的基本資料結構。了解一維陣列和二維陣列之間的差異，對於有效地組織和操作資料至關重要。讓我們以簡單易懂的方式來探討這些概念。

一維陣列，通常稱為陣列，就像是一列箱子。想像你在火車站有一排置物櫃。每個置物櫃可以放一個項目，如果你知道它的編號，就可以直接存取任何置物櫃。

在程式設計術語中，一維陣列中的每個條款都是使用單一索引來存取的。 例如，在 JavaScript 中，你可能會這樣創建並使用一維陣列：

:::interactive_editor

```js
let fruits = ["apple", "banana", "cherry", "date"];
console.log(fruits[2]); // "cherry"
```

:::

這裡，`fruits` 是一個一維陣列。你可以將它視為一列水果名稱。要存取任何水果，你使用一個數字（索引）放在方括號內。

現在，讓我們繼續討論二維陣列。如果一維陣列就像是一排鎖櫃，二維陣列就像是一個鎖櫃網格－多列多行。在程式設計中，二維陣列本質上是陣列的陣列。它用來表述具有自然網格狀結構的資料，例如棋盤、試算表或影像中的像素。

要存取二維陣列中的元素，你需要兩個索引：一個用於列，一個用於行。以下是如何在 JavaScript 中創建並使用二維陣列的範例：

:::interactive_editor

```js
let chessboard = [
    ["R", "N", "B", "Q", "K", "B", "N", "R"],
    ["P", "P", "P", "P", "P", "P", "P", "P"],
    [" ", " ", " ", " ", " ", " ", " ", " "],
    [" ", " ", " ", " ", " ", " ", " ", " "],
    [" ", " ", " ", " ", " ", " ", " ", " "],
    [" ", " ", " ", " ", " ", " ", " ", " "],
    ["p", "p", "p", "p", "p", "p", "p", "p"],
    ["r", "n", "b", "q", "k", "b", "n", "r"]
];

console.log(chessboard[0][3]); // "Q"
```

:::

在這個範例中，`chessboard` 是一個二維陣列，表述一個西洋棋遊戲的初始佈局。要存取位於頂列的皇后（`Q`），我們使用兩個索引：`[0][3]`。第一個索引 `0` 選擇第一列，第二個索引 `3` 選擇該列中的第四行。

一維陣列和二維陣列之間的主要差異在於你如何存取和組織資料。一維陣列使用單一索引，適用於像列表或序列這類線性資料。二維陣列使用兩個索引，非常適合格狀資料結構。

值得注意的是，在 JavaScript 中，二維陣列實際上是陣列的陣列。這表示外層陣列的每個元素本身就是一個陣列。這種巢狀結構提供了極大的彈性，但也需要謹慎處理以避免錯誤。

隨著你在程式設計旅程中的進展，你會發現選擇一維陣列或二維陣列取決於你的資料性質以及你需要如何操作它。 

一維陣列較為簡單且足以應付許多任務，而二維陣列在處理更複雜、有結構的資料時則變得非常重要。
