# --description--

A veces habrá situaciones en las que sabrás más sobre un tipo que TypeScript. Aquí es donde las aserciones de tipo son útiles.

Aquí hay un ejemplo usando el método `querySelector()` para acceder a un elemento con el `id` de `submit`:

```ts
const submitBtn = document.querySelector("#submit");
```

En este momento, TypeScript solo sabe que este `submitBtn` es algún tipo de `Element` o posiblemente `null`. TypeScript no revisará el HTML correspondiente para averiguarlo.

Este es un buen lugar para usar una estructura de tipo para decirle a TypeScript qué tipo de elemento es este. Aquí hay un ejemplo:

```ts
const submitBtn = document.querySelector("#submit") as HTMLButtonElement;
```

Ahora, TypeScript tratará esto como un elemento `button` debido a esa aserción de tipo. 

Hay límites al usar la palabra clave `as`. Por ejemplo, no puedes hacer cosas como esta sin que TypeScript lance un error:

```ts
// Conversion of type 'string' to type 'number' may be a mistake because neither type sufficiently overlaps with the other. If this was intentional, convert the expression to 'unknown' first.
const age = "awesome" as number;
```

En este ejemplo, no tiene mucho sentido intentar afirmar que la cadena `"awesome"` debe ser vista como un tipo numérico. TypeScript solo permite afirmaciones de tipo entre tipos compatibles o que se superponen.

Una forma de eliminar ese error sería usar una doble aserción como esta:

```ts
const age = "awesome" as unknown as number;
```

Aunque esto funciona en TypeScript, todavía no se recomienda porque lógicamente no tiene mucho sentido.

Otra forma de escribir aserciones de tipo es usar la sintaxis de corchetes angulares así:

```ts
const submitBtn = <HTMLButtonElement>document.querySelector("#submit");
```

Debes tener cuidado con esta sintaxis al usar archivos `tsx` porque TypeScript pensará que estás intentando renderizar un componente y lanzará un error como este:

```ts
// file: index.tsx
// This JSX tag requires 'React' to be in scope, but it could not be found.
// JSX element 'HTMLButtonElement' has no corresponding closing tag.
const submitBtn =  <HTMLButtonElement>document.querySelector("#submit");
```

En la mayoría de las situaciones, verás que se usa la primera sintaxis con la palabra clave `as` en lugar de la segunda opción.
