# --description--

`README` 파일은 많은 프로젝트, 특히 소프트웨어 개발에서 중요한 부분입니다. 프로젝트를 처음 볼 때 사람들이 가장 먼저 접하는 문서인 경우가 많습니다. `README` 파일은 프로젝트가 무엇인지, 어떻게 사용하는지, 때로는 어떻게 기여하는지 설명합니다. 프로젝트 문서에서 `README` 파일이 어떤 용도로 사용되는지 이해하는 것이 매우 중요합니다.

# --questions--

## --text--

프로젝트에서 `README` 파일은 일반적으로 어떤 용도로 사용되나요?

## --answers--

코드를 저장하는 용도

### --feedback--

코드는 프로젝트의 필수 요소이지만, `README` 파일은 코드를 저장하는 데 사용되지 않고 프로젝트에 대한 정보를 제공하는 데 사용됩니다.

---

프로젝트에 관한 지침을 제공하는 용도

---

연락처 정보를 제공하는 용도

### --feedback--

연락처 정보가 포함될 수는 있지만, `README` 파일의 주요 목적은 단순한 연락처 정보가 아니라 프로젝트에 관한 지침과 정보를 제공하는 것입니다.

---

변경 내역 기록

### --feedback--

변경 내역 기록(changelog)은 보통 `README` 파일과 별개입니다. `README` 파일은 프로젝트를 소개하고 설명하는 데 중점을 둡니다.
