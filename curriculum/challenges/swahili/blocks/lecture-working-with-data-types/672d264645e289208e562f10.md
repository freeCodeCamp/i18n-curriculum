# --description--

JavaScript ni lugha yenye aina zinazobadilika (dynamically typed), maana yake huna haja ya kubainisha aina ya data ya kigezo unapotangaza. Badala yake, aina huamuliwa kulingana na thamani iliyopewa kigezo wakati programu inatekelezwa. Hii inakuwezesha kubadilisha aina ya kigezo katika programu nzima.

Tuchukulie mfano:

```js
let example = "Hello";
example = 42;
```

Katika mfano huu, tuna kigezo kinachoitwa `example` chenye aina ya data ya mfuatano wa herufi (string). Lakini kisha tunasasisha thamani kuwa nambari badala yake.

Uwezo wa dynamic typing hufanya JavaScript kuwa rahisi na yenye msamaha zaidi kwa ajili ya kuandika msimbo wa haraka, lakini pia inaweza kuleta hitilafu ambazo zinaweza kuwa ngumu kugundua, hasa programu inapokua kubwa zaidi.

Katika lugha zenye aina zilizowekwa kabla kama C# au C++, lazima utangaze aina ya data ya kigezo unapoanzisha, na aina hiyo haiwezi kubadilika.

Kwa mfano, ukitangaza kigezo kama `integer`, unaweza tu kupewa thamani za nambari kamili. Ukijaribu kupewa aina tofauti, programu itatupa kosa.

Hapa kuna mfano katika lugha ya C#:

```csharp
int data = 42; // data must always be an integer
data = "Hello"; // This would cause an error in C#
```

Tofauti kati ya dynamic typing na static typing iko katika ufanisi dhidi ya usalama wa msimbo wako. Lugha zenye dynamic typing hutoa ufanisi lakini kwa gharama ya hitilafu zinazoweza kutokea wakati wa utekelezaji.

Lugha zenye static typing zinaweka sheria kali zinazoweza kuzuia hitilafu fulani, lakini zinahitaji matangazo zaidi kabla na hutoa ufanisi mdogo wa kubadilisha aina.

Hapa kuna mfano mwingine wa kuunda kigezo chenye aina `number` kisha kubadilisha kuwa aina `string` baadaye:

```js
let data = 100;  // Initially a number
data = "New data";  // Dynamically changes to a string
```

Katika lugha yenye static typing, mabadiliko kama haya hayaruhusiwi, kwa sababu aina ya data itakuwa imedhamiriwa.

Kwa kumalizia, dynamic typing ya JavaScript inaruhusu vigezo kubadilisha aina kwa uhuru, ambayo hutoa ufanisi lakini inaweza kusababisha hitilafu zisizotarajiwa wakati wa utekelezaji.

Lugha zenye static typing kama Java zinahitaji ubainishaji wa aina za vigezo mapema, jambo linalosaidia kugundua hitilafu kabla programu kuanza lakini hutoa ufanisi mdogo.
