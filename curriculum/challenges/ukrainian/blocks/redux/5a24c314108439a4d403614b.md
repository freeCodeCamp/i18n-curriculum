# --description--

Redux — це фреймворк для керування станом, який можна використовувати з різними веб-технологіями, включно з React.

У Redux існує єдиний об’єкт стану, який відповідає за весь стан вашого застосунку. Це означає, що якщо у вас є React-застосунок із десятьма компонентами, і кожен компонент має свій локальний стан, то весь стан вашого застосунку визначається одним об’єктом стану, розміщеним у Redux `store`. Це перший важливий принцип, який потрібно зрозуміти при вивченні Redux: Redux store — це єдине джерело правди щодо стану застосунку.

Це також означає, що кожного разу, коли будь-яка частина вашого застосунку хоче оновити стан, вона **повинна** робити це через Redux store. Односпрямований потік даних полегшує відстеження керування станом у вашому застосунку.

# --instructions--

Redux `store` — це об’єкт, який зберігає та керує станом застосунку `state`. Існує метод під назвою `createStore()` на Redux-об’єкті, який ви використовуєте для створення Redux `store`. Цей метод приймає `reducer` функцію як обов’язковий аргумент. `reducer` функція розглядається в одному з наступних завдань і вже визначена для вас у редакторі коду. Вона просто приймає `state` як аргумент і повертає `state`.

Оголосіть змінну `store` і призначте їй результат виклику методу `createStore()`, передаючи `reducer` як аргумент.

**Примітка:** Код у редакторі використовує синтаксис ES6 для аргументів за замовчуванням, щоб ініціалізувати цей стан значенням `5`. Якщо ви не знайомі з аргументами за замовчуванням, можете звернутися до <a href="https://www.freecodecamp.org/learn/javascript-algorithms-and-data-structures/es6/set-default-parameters-for-your-functions" target="_blank" rel="noopener noreferrer nofollow">розділу ES6 у навчальній програмі</a>, де ця тема розглядається.
