# --interactive--

CSSのコンビネーターは、CSSのセレクター間の関係を定義するために使われます。これにより、他の要素との関係に基づいて要素を選択でき、より正確で効率的なスタイリングが可能になります。

ここでは、主なCSSコンビネーターとその使い方について説明します。まずは子孫コンビネーターから始めましょう。

子孫コンビネーターは、最初のセレクターにマッチする祖先要素の中にネストされた、2番目のセレクターにマッチする要素をターゲットにします。祖先要素は親要素や親の親要素などが該当します。

この仕組みを理解するために、例を見てみましょう。

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<figure>
  <img src="https://cdn.freecodecamp.org/curriculum/cat-photo-app/relaxing-cat.jpg" alt="Relaxing Cat">
  <figcaption>A relaxing cat with a border</figcaption>
</figure>
```

```css
figure img {
  border: 4px solid red;
}
```

:::

上の例では、子孫コンビネーターを使って`figure`要素の中にあるすべての画像要素を選択し、4辺すべてに`solid red`のボーダーを適用しています。

親セレクターと子セレクターの間には`space`が使われていることに注意してください。

この場合、`figure`が親で、`img`が子になります。

もし`figure`要素の中に複数の画像があれば、子孫コンビネーターを使うことで、それぞれの画像に黒の実線ボーダーを適用するのに適しています。

次に紹介するのは子コンビネーターです。

CSSの子コンビネーター（`>`）は、指定した親要素の直接の子要素だけを選択するために使われます。

このコンビネーターは特定の親を持つ要素だけをターゲットにするため、CSSルールをより正確にし、より深くネストされた要素への意図しないスタイル適用を防ぎます。

次のHTML例を見てみましょう。

:::interactive_editor

```html
<div class="container">
  <p>First</p>
  <div>
    <p>Second</p>
  </div>
  <div>
    <p>Third</p>
  </div>
</div>
```

:::

上のHTML構造では、`container`クラスが`div`要素に適用されています。

このコンテナの中には、直接の子である`p`要素（"First"）があり、その後に2つの`div`要素が続き、それぞれに`p`要素（"Second"と"Third"）が含まれています。

最初の`p`要素は`.container`要素の直接の子ですが、他の2つの`p`要素は他の`div`要素の中にネストされているため、より深い子孫要素です。

`container`クラスの直接の子だけにスタイルを適用したい場合は、子コンビネーターを次のように使います。

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<div class="container">
  <p>First</p>
  <div>
    <p>Second</p>
  </div>
  <div>
    <p>Third</p>
  </div>
</div>
```

```css
.container > p {
  color: blue;
}
```

:::

上の例では、`container`クラスの直接の子だけをターゲットにしています。これにより、直接の子に`blue`のテキストカラーが適用されます。

他の2つの段落要素は`div`要素の中にネストされているため、`container`クラスの直接の子とはみなされず、青色のテキストカラーは適用されません。

次に紹介する一般的なコンビネーターは隣接兄弟コンビネーターです。

CSSの隣接兄弟コンビネーター（`+`）は、指定した兄弟要素の直後に続く要素を選択します。このコンビネーターは、ある要素のすぐ後に続く要素にスタイルを適用したい場合に便利で、兄弟要素の位置に基づいたターゲットスタイリングが可能です。

次のHTML例を見てみましょう。

:::interactive_editor

```html
<figure>
  <img
    src="https://cdn.freecodecamp.org/curriculum/cat-photo-app/relaxing-cat.jpg"
    alt="A cute orange cat lying on its back."
  />
  <figcaption>A cute orange cat lying on its back.</figcaption>
</figure>
```

:::

ここでは、`figure`要素の中に`img`要素があり、その直後に`figcaption`要素が続いています。`figcaption`要素は`img`要素の直後の兄弟要素です。

`figcaption`要素に黒いボーダーを適用したい場合は、隣接兄弟コンビネーターを次のように使います。

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<figure>
  <img
    src="https://cdn.freecodecamp.org/curriculum/cat-photo-app/relaxing-cat.jpg"
    alt="A cute orange cat lying on its back."
  />
  <figcaption>A cute orange cat lying on its back.</figcaption>
</figure>
```

```css
img + figcaption {
  border: 4px solid black;
}
```

:::

この例では、隣接兄弟コンビネーター（`+`）が`figcaption`要素の直後に続く`img`要素を選択しています。適用されたCSSルールは`4px solid black border`の周りに`figcaption`を追加します。

もう一つの一般的なコンビネーターは後続兄弟コンビネーターです。

CSSの後続兄弟コンビネーター（`~`）は、指定した要素の後に続くすべての兄弟要素を選択します。

隣接兄弟コンビネーターが直後の兄弟だけをターゲットにするのに対し、後続兄弟コンビネーター（`~`）は指定した要素の後に続く任意の兄弟要素をターゲットにでき、より柔軟なスタイリングが可能です。

次のHTML例を見てみましょう。

:::interactive_editor

```html
<div class="container">
  <h2>Subheading</h2>
  <p>First paragraph.</p>
  <p>Second paragraph.</p>
  <p>Third paragraph.</p>
  <p>Another paragraph element</p>
</div>
```

:::

このHTML構造では、`h2`要素の後に4つの段落要素が続いています。段落要素は`h2`要素の兄弟要素です。

`h2`要素の後に続くすべての段落要素にスタイルを適用したい場合は、後続兄弟コンビネーターを次のように使います。

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<div class="container">
  <h2>Subheading</h2>
  <p>First paragraph.</p>
  <p>Second paragraph.</p>
  <p>Third paragraph.</p>
  <p>Another paragraph element</p>
</div>
```

```css
h2 ~ p {
  color: green;
}
```

:::

この例では、`h2`要素の後に続くすべての段落要素に緑色のテキストカラーが適用されます。

後続兄弟コンビネーター（`~`）は、`h2`要素の後に現れるすべての段落兄弟要素をターゲットにします。直後の兄弟であるかどうかは問いません。

まとめると、さまざまなCSSコンビネーターを理解し使いこなすことで、HTML要素に正確なスタイルを適用でき、デザインのコントロールと柔軟性が向上します。

これらのセレクターをマスターすれば、より洗練されターゲットを絞ったスタイルルールを作成でき、ウェブページの見た目と機能性の両方を改善できます。

## --text--

どのCSSルールが、どれだけ深くネストされていても、`span`要素が`div`要素の中にあるすべての場合にスタイルを適用しますか？

## --answers--

```css
div > span {
  color: red;
}
```
