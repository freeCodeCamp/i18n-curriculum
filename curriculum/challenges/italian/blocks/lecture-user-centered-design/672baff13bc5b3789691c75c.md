# --interactive--

**NOTA**: Alcuni esempi interattivi potrebbero usare CSS che non hai ancora imparato. Non preoccuparti di capire tutto il codice. Lo scopo degli esempi è mostrarti anteprime di questi concetti di design per farti capire meglio come funzionano. Per vedere le anteprime, devi abilitare l'editor interattivo.

Quando si parla di design web, ci sono molti tipi di aiuti alla navigazione che puoi usare. Gli esempi includono barre di navigazione superiori, barre laterali e piè di pagina. Ma se il tuo sito è più complesso con livelli di navigazione più profondi, potresti voler considerare l’uso dei breadcrumb.

I breadcrumb sono un aiuto alla navigazione che mostra all’utente dove si trova nella gerarchia del sito. Ecco un esempio di come appaiono i breadcrumb per un sito di elettronica fittizio:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<div class="breadcrumbs">
  Home / Electronics / Phones / Smartphone XYZ
</div>
```

```css
.breadcrumbs {
  font-size: 0.9rem;
  color: #555;
  margin: 1rem 0;
}
```

:::

Nella maggior parte dei siti, i breadcrumb sono mostrati in cima alla pagina, indicando all’utente il percorso seguito per arrivare alla pagina corrente. Partendo da `Homepage`, l’utente ha navigato nella categoria `Electronics`, poi nella categoria `Phones` e infine nel prodotto `Smartphone XYZ`. Probabilmente hai già interagito con i breadcrumb su un sito mentre cercavi un prodotto o un’informazione specifica.

L’uso dei breadcrumb è utile perché aiuta gli utenti a capire dove si trovano nella gerarchia del sito e come tornare alle pagine precedenti. Questo è particolarmente utile quando un utente arriva da un risultato di ricerca o da un collegamento esterno e deve comprendere il contesto della pagina su cui si trova.

Quando progetti i breadcrumb, ci sono alcune considerazioni da tenere a mente. La prima è decidere quale sarà il separatore. Il separatore è il carattere che divide i diversi livelli della gerarchia. I separatori comuni includono il segno maggiore di (`>`), le virgolette angolari (`»`) e la barra obliqua (`/`).

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<div class="breadcrumbs">
  Home &gt; Electronics &gt; Phones &gt; Smartphone XYZ
</div>
```

```css
.breadcrumbs {
  font-size: 0.9rem;
  color: #555;
  margin: 1rem 0;
}
```

:::

La seconda considerazione è la posizione dei breadcrumb. Di solito sono posizionati in cima alla pagina, sopra o sotto la barra di navigazione principale. Gli utenti non dovrebbero faticare a trovare i breadcrumb, quindi assicurati che siano visibili e facili da individuare.

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<nav class="main-nav">
  <a href="#">Home</a>
  <a href="#">Electronics</a>
  <a href="#">Phones</a>
</nav>
<div class="breadcrumbs">
  Home &gt; Electronics &gt; Phones &gt; Smartphone XYZ
</div>
```

```css
.main-nav {
  background-color: #333;
  padding: 1rem;
}

.main-nav a {
  color: white;
  margin-right: 1rem;
  text-decoration: none;
}

.breadcrumbs {
  font-size: 0.9rem;
  color: #555;
  margin: 1rem 0;
}
```

:::

Un’altra considerazione è la dimensione dei breadcrumb. Devono essere abbastanza grandi da essere facilmente leggibili, ma non così grandi da occupare troppo spazio nella pagina. Ricorda, i breadcrumb non sono pensati come strumento di navigazione principale, ma come aiuto secondario alla navigazione. Nei siti con molte informazioni in una pagina, gli utenti possono facilmente vedere dove si trovano nella gerarchia e tornare alle pagine precedenti usando i breadcrumb.

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<div class="breadcrumbs">
  Home &gt; Electronics &gt; Phones &gt; Smartphone XYZ
</div>
<p>
  Lorem ipsum dolor sit amet, consectetur adipiscing elit. Sed do eiusmod tempor incididunt ut labore et dolore magna aliqua.
</p>
```

```css
.breadcrumbs {
  font-size: 1rem;
  color: #555;
  margin: 1rem 0;
}   

p {
  font-size: 1.2rem;
  line-height: 1.6;
}
```

:::

Quindi, quando dovresti usare i breadcrumb sul tuo sito? Se il sito è semplice e ha una gerarchia poco profonda, i breadcrumb potrebbero non essere necessari. Una gerarchia poco profonda significa che il sito ha solo pochi livelli di navigazione, quindi i breadcrumb potrebbero non aggiungere molto valore. Di solito, vedrai i breadcrumb su siti di e-commerce, siti di notizie e altri siti con una gerarchia profonda come i siti di documentazione tecnica.
