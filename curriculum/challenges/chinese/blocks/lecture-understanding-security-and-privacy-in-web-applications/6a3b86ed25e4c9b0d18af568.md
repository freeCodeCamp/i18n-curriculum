# --description--

`.env` 文件和 `dotenv` 库是现代软件开发中用来将敏感配置数据与应用代码分离的工具。

`.env` 文件是放置在项目目录根部的纯文本文件，用于以键值对形式存储配置设置和秘密信息。

开发者不会将 API 密钥、数据库密码或端口号硬编码到源代码中，以免代码推送到像 GitHub 这样的公共仓库时暴露这些信息，而是将它们本地保存在此文件中。

`.env` 文件的结构非常简单：

```bash
PORT=3000
DB_HOST=localhost
API_KEY=xyz123secretkey
```

由于该文件包含高度敏感的密钥，通常会将 `.env` 添加到项目的 `.gitignore` 文件中，以确保它永远不会被 Git 跟踪。

`dotenv` 包是一个轻量级模块，用于将 `.env` 文件连接到你的应用中。

当程序启动时，`dotenv` 会读取 `.env` 文件，解析密钥，并将它们注入到运行期环境中。例如，在 Node.js 应用中，你会在入口文件的最顶部初始化该包。加载后，这些变量通过 `process.env` 对象在全局范围内可访问。

```javascript
const express = require('express');
const app = express();

// initialize and configure dotenv
require('dotenv').config();

// access the variables securely from the environment
const port = process.env.PORT || 5000;
const apiKey = process.env.API_KEY;

app.get('/', (req, res) => {
  res.send(`Connected successfully using API Key: ${apiKey}`);
});

app.listen(port, () => {
  console.log(`Server running on port ${port}`);
});
```

这种设置带来两个主要好处。首先，它通过将私密凭证从源代码中分离出来提升了安全性。其次，它提供了灵活性，使同一代码库能在开发、测试和生产环境中表现不同，只需在各自的托管服务器上替换本地 `.env` 文件即可。
