# --description--

اختبار نهاية إلى نهاية، أو E2E اختصارًا، يختبر السيناريوهات الواقعية من منظور المستخدم. تشمل أمثلة سيناريوهات اختبار E2E اختبار عملية التسجيل وتسجيل الدخول أو اختبار إتمام الشراء لموقع تجارة إلكترونية.

لفهم أفضل لكيفية عمل اختبار نهاية إلى نهاية، دعنا نلقي نظرة على مثال واقعي من شفرة freeCodeCamp. يستخدم هذا المثال Playwright، وهو إطار عمل شائع لاختبار نهاية إلى نهاية طورته شركة Microsoft.

داخل شفرة freeCodeCamp، يوجد مجلد يسمى `e2e` يحتوي على اختبارات للتبرعات، والشهادات، والاختبارات، والمزيد. داخل هذا المجلد، يوجد ملف `donate-page-donor.spec.ts` يحتوي على اختبارات لصفحة الداعم المتبرع المُوثَّق.

الخطوة الأولى هي الانتقال إلى صفحة التبرع كمستخدم تجريبي هو بالفعل داعم:

```js
test.beforeEach(async ({ page }) => {
  execSync("node ./tools/scripts/seed/seed-demo-user --set-true isDonating");
  await page.goto("/donate");
});
```

بما أن هناك اختبارات متعددة في هذا الملف، فإن الخطاف `beforeEach` سيُنفذ قبل كل اختبار من هذه الاختبارات.

بمجرد أن يكون المستخدم في صفحة التبرع، سيرى رسالة شكر مع اقتراحات حول كيفية دعم freeCodeCamp أكثر. إليك بعض الاختبارات للتحقق من بعض النصوص الموجودة في صفحة التبرع:

```js
test("should render the donate page correctly", async ({ page }) => {
  await expect(
    page.getByText("Thank you for your continued support")
  ).toBeVisible();

  await expect(
    page.getByText(
      "Your contributions are crucial in creating resources that empower millions of people to learn new skills and support their families."
    )
  ).toBeVisible();

  ...
});
```

هناك أيضًا اختبارات للتحقق من أن المتبرع لديه رابط داعم في شريط القائمة، بالإضافة إلى حدود مميزة خاصة حول صورته الرمزية للدلالة على أنه داعم:

```js
test("The menu should have a supporters link", async ({ page }) => {
  const menuButton = page.getByTestId("header-menu-button");
  const menu = page.getByTestId("header-menu");

  await expect(menuButton).toBeVisible();
  await menuButton.click();

  await expect(menu).toBeVisible();

  await expect(page.getByRole("link", { name: "Supporters" })).toBeVisible();
});

test("The Avatar should have a special border for donors", async ({ page }) => {
  const container = page.locator(".avatar-container");
  await expect(container).toHaveClass("avatar-container gold-border");
});
```

بوجود هذه الأنواع من الاختبارات التفصيلية، يمكنك زيادة تغطية الاختبارات لتطبيقك إلى ما هو أبعد من اختبارات الوحدة فقط. تساعد اختبارات نهاية إلى نهاية أيضًا في ضمان أن يتصرف تطبيقك بشكل صحيح، وأن يكون متوقعًا للمستخدمين. تشمل أمثلة أدوات اختبار نهاية إلى نهاية الأخرى Cypress وSelenium وPuppeteer.

بينما اختبار نهاية إلى نهاية مهم، فهو مكلف لأنه يستغرق وقتًا طويلاً للإعداد، والتصميم، والصيانة. اختبارات E2E رائعة لتدفقات المستخدم الحرجة، بينما اختبارات الوحدة ممتازة لاختبار الوحدات الصغيرة في التطبيق.
