# --description--

Sobald Sie den Anfangszustand einer Komponente definiert haben, können Sie jeden Teil davon in der gerenderten Benutzeroberfläche anzeigen. Wenn eine Komponente zustandsbehaftet ist, hat sie immer Zugriff auf die Daten in `state` in ihrer `render()`-Methode. Sie können auf die Daten mit `this.state` zugreifen.

Wenn Sie innerhalb des `return` der Render-Methode auf einen Zustandswert zugreifen möchten, müssen Sie den Wert in geschweifte Klammern einschließen.

`state` ist eine der mächtigsten Funktionen von Komponenten in React. Sie ermöglicht es Ihnen, wichtige Daten in Ihrer App zu verfolgen und eine Benutzeroberfläche als Antwort auf Änderungen dieser Daten zu rendern. Wenn sich Ihre Daten ändern, ändert sich Ihre Benutzeroberfläche. React verwendet das sogenannte virtuelle DOM, um Änderungen im Hintergrund zu verfolgen. Wenn sich Zustandsdaten aktualisieren, löst das ein erneutes Rendern der Komponenten aus, die diese Daten verwenden – einschließlich der Kindkomponenten, die die Daten als Prop erhalten haben. React aktualisiert das tatsächliche DOM, aber nur dort, wo es notwendig ist. Das bedeutet, Sie müssen sich nicht darum kümmern, das DOM zu ändern. Sie deklarieren einfach, wie die Benutzeroberfläche aussehen soll.

Beachten Sie, dass wenn Sie eine Komponente zustandsbehaftet machen, keine anderen Komponenten von ihrem `state` wissen. Ihr `state` ist vollständig gekapselt oder lokal für diese Komponente, es sei denn, Sie geben Zustandsdaten an eine Kindkomponente als `props` weiter. Dieses Konzept der gekapselten `state` ist sehr wichtig, weil es Ihnen erlaubt, bestimmte Logik zu schreiben und diese Logik dann an einem Ort in Ihrem Code enthalten und isoliert zu haben.
