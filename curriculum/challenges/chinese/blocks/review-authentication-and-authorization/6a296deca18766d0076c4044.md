## 认证与授权

**认证** 是验证用户身份的过程。它回答了*“这个人真的是他们声称的那个人吗？”* 当用户登录时，后端服务器会将他们的凭据与注册时存储的记录进行比较。

后端开发者使用三类主要的认证因素：

- **你知道的东西：** 密码、PIN 码或安全问题答案。
- **你拥有的东西：** 智能手机上的短信验证码，或认证器应用生成的临时令牌。
- **你本身的东西：** 生物识别数据，如指纹或面部识别扫描。

**授权** 在认证之后进行，决定经过认证的用户被允许做什么。它回答了*“这个身份拥有什么权限？”* 常见角色包括：

- **标准用户：** 可以编辑自己的个人资料，但不能查看其他用户的私人数据。
- **编辑者：** 可以撰写和发布内容，但不能删除代码库。
- **管理员：** 拥有完全权限，可以修改数据库、删除用户和更改系统范围的设置。

**认证必须始终先发生，然后才是授权。** 在知道用户是谁之前，你无法决定授予什么权限。

## JSON Web 令牌（JWT）

JWT 是服务器在用户成功登录后发放的数字钥匙卡。用户在每次后续请求时出示它，以证明身份，无需重新输入密码。

JWT 的四步流程：

1. **登录：** 用户向服务器发送凭据。
2. **发放：** 服务器验证凭据并生成 JWT，然后返回给用户。
3. **存储：** 客户端存储 JWT，通常在浏览器 cookie 或本地存储中。
4. **请求：** 对受保护路由的每个后续请求，客户端都会发送 JWT，服务器进行验证。

JWT 由三个部分组成，中间用两个点分隔：

- **头部：** 编码的 JSON，声明令牌类型（JWT）和使用的哈希算法，如 HMAC SHA256。
- **负载：** 编码的 JSON，包含关于用户的声明，如用户 ID、用户名或角色。头部和负载是 Base64 编码的，不是加密的，因此绝不要在这里存储密码等敏感数据。
- **签名：** 通过将编码后的头部和负载与服务器独有的密钥一起哈希生成。如果负载被篡改，签名将不匹配，服务器会以 `401 Unauthorized` 响应拒绝请求。

JWT 支持**无状态认证**。服务器无需在数据库中保存会话数据，因为签名证明了令牌是服务器创建的。

## 跨站请求伪造（CSRF）

CSRF 是一种攻击，迫使经过认证的用户在不知情的情况下执行网页应用上的不良操作。恶意网站欺骗受害者浏览器向后端服务器发送伪造请求，利用浏览器会自动附加 cookie 到任何发往目标服务器的请求这一事实。

**攻击示例流程：**

1. 你登录了，比如 `trusted-bank.com`。银行在你的浏览器 cookie 中存储了会话令牌。
2. 你没有注销，访问了 `evil-hacker-site.com`。
3. 恶意网站悄悄向 `trusted-bank.com/transfer?amount=1000&to=hacker` 提交请求。
4. 你的浏览器自动附加了银行会话 cookie，服务器因此处理了该请求。

恶意网站无法读取你的 cookie。它只是利用了浏览器自动附加 cookie 到任何发往目标服务器请求的事实。

**防御 CSRF 的方法：**

- **`SameSite=Strict`：** 如果请求来自第三方网站，浏览器绝不发送 cookie。
- **`SameSite=Lax`：** 允许在安全导航（点击链接）时发送 cookie，但阻止跨站表单提交和 API 请求时发送。
- **CSRF 令牌：** 当用户加载页面时，服务器生成一个随机、唯一且不可预测的令牌，并附加到前端表单或会话中。客户端必须在请求头或表单数据中包含它。由于恶意网站无法访问前端复制令牌，伪造请求会验证失败。

## Passport.js

Passport.js 是 Node.js 的认证中间件，通过称为 **策略** 的模块化插件标准化认证请求的方式。拥有超过 500 种策略，你可以通过安装相关包添加任何认证方法：

- `passport-local`：使用用户名和密码对你的数据库进行认证。
- `passport-jwt`：验证受保护 API 路由上的 JSON Web 令牌。
- `passport-google-oauth20`：使用 Google 账户登录。
- `passport-github2`：使用 GitHub 账户登录。

安装核心包和一个策略：

```bash
npm install passport passport-local
```

在定义路由之前，将 Passport 集成到你的 Express 应用中：

```js
const passport = require('passport');
const LocalStrategy = require('passport-local').Strategy;

app.use(passport.initialize());

passport.use(new LocalStrategy((username, password, done) => {
  // database look-up and password verification logic
}));
```

用 Passport 作为中间件保护登录路由：

```js
app.post('/login', passport.authenticate('local'), (req, res) => {
  res.send(`Welcome back, ${req.user.username}!`);
});
```

认证成功后，Passport 会自动处理**会话管理**（将用户数据存储在会话 cookie 中）和**请求装饰**（将用户对象附加到 `req.user`，供后续路由处理器使用）。

## Helmet.js

Helmet.js 是 Node.js 的安全中间件集合，通过自动设置、修改或移除 Express 默认暴露的 HTTP 响应头来保护你的应用。

一步安装并应用它：

```bash
npm install helmet
```

```js
const helmet = require('helmet');
app.use(helmet());
```

将 `app.use(helmet())` 放在中间件栈顶部，应用这些保护到每个响应：

- **移除 X-Powered-By：** Express 默认通过 `X-Powered-By: Express` 广告其存在。Helmet 移除此头部，防止攻击者针对特定版本漏洞进行攻击。
- **设置内容安全策略（Content-Security-Policy）：** 限制浏览器加载脚本、图像和样式的来源，防御跨站脚本（XSS）攻击。
- **设置 X-Frame-Options 为 SAMEORIGIN：** 防止你的网站被嵌入外部 iframe，阻止点击劫持攻击。
- **设置严格传输安全（Strict-Transport-Security）：** 强制浏览器仅通过 HTTPS 与服务器通信，保护数据免受不可信网络上的拦截。
