# --description--

Файл `.env` — це прихований файл, який використовується для передачі змінних середовища у ваш застосунок. Цей файл є секретним, ніхто, крім вас, не має до нього доступу, і його можна використовувати для зберігання даних, які ви хочете тримати приватними або прихованими. Наприклад, ви можете зберігати API-ключі від зовнішніх сервісів або URI вашої бази даних. Також його можна використовувати для зберігання параметрів конфігурації. Встановлюючи параметри конфігурації, ви можете змінювати поведінку вашого застосунку без необхідності переписувати частину коду.

Змінні середовища доступні у застосунку як `process.env.VAR_NAME`. Об’єкт `process.env` — це глобальний об’єкт Node, а змінні передаються у вигляді рядків. За конвенцією, імена змінних пишуться всіма великими літерами, а слова розділяються підкресленням. `.env` — це shell-файл, тому вам не потрібно брати імена чи значення в лапки. Також важливо пам’ятати, що навколо знака рівності при призначенні значень змінним не повинно бути пробілів, наприклад `VAR_NAME=value`. Зазвичай кожне визначення змінної розміщується на окремому рядку.

# --instructions--

Давайте додамо змінну середовища як параметр конфігурації.

Створіть файл `.env` у корені каталогу вашого проєкту та збережіть у ньому змінну `MESSAGE_STYLE=uppercase`.

Потім у обробнику маршруту GET `/json`, який ви створили в останньому завданні, отримайте доступ до `process.env.MESSAGE_STYLE` і перетворіть значення властивості `message` об’єкта відповіді на верхній регістр, якщо змінна дорівнює `uppercase`. Об’єкт відповіді має бути або `{"message": "Hello json"}`, або `{"message": "HELLO JSON"}`, залежно від значення `MESSAGE_STYLE`. Зверніть увагу, що ви повинні читати значення `process.env.MESSAGE_STYLE` **всередині** обробника маршруту, а не поза ним, через спосіб запуску наших тестів.

Вам потрібно буде використати пакет `dotenv`. Він завантажує змінні середовища з вашого файлу `.env` у `process.env`. Пакет `dotenv` вже встановлений і знаходиться у файлі `package.json` вашого проєкту. На початку файлу `myApp.js` додайте `require('dotenv').config()` для завантаження змінних середовища.

# --hints--

Відповідь кінцевої точки `/json` має змінюватися залежно від змінної середовища `MESSAGE_STYLE`

```js
  const response = await fetch(code + '/_api/use-env-vars');
  if (!response.ok) {
    throw new Error(await response.text());
  }
  const data = await response.json();
  assert.isTrue(
    data.passed,
    'The response of "/json" does not change according to MESSAGE_STYLE'
  );
```
