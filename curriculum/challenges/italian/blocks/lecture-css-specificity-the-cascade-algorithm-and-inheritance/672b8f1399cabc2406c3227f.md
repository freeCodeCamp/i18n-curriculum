# --interactive--

La parola chiave `!important` in CSS serve a dare a una regola di stile la priorità più alta, permettendo di sovrascrivere qualsiasi altra dichiarazione per una proprietà.

Quando viene usata, forza il browser ad applicare lo stile specificato, indipendentemente dalla specificità degli altri selettori.

Supponiamo di avere un elemento paragrafo HTML con stili in linea come questo:

:::interactive_editor

```html
<p class="para" style="background-color: lightblue; color: black;">
  This is a paragraph.
</p>
```

:::

In questo esempio, l’elemento paragrafo ha un colore di sfondo impostato su `lightblue` e un colore del testo impostato su `black`.

Il file CSS applica i seguenti stili all’elemento paragrafo:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<p class="para" style="background-color: lightblue; color: black;">
  This is a paragraph.
</p>
```

```css
.para {
  background-color: black;
  color: white;
}
```

:::

Poiché gli stili in linea hanno una precedenza più alta rispetto ai selettori di classe, ID e tipo, il colore di sfondo nero e il colore del testo bianco non verranno applicati a quell’elemento paragrafo.

Per sovrascrivere quegli stili in linea, puoi usare la parola chiave `!important` così:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<p class="para" style="background-color: lightblue; color: black;">
  This is a paragraph.
</p>
```

```css
.para {
  background-color: black !important;
  color: white !important;
}
```

:::

La parola chiave `!important` si usa dopo il valore CSS e prima del punto e virgola.

Ora l’elemento paragrafo avrà quei colori nero e bianco applicati perché gli stili in linea vengono sovrascritti usando la parola chiave `!important`.

La parola chiave `!important` in CSS serve a dare a una regola di stile la priorità più alta, sovrascrivendo efficacemente altre dichiarazioni, incluse quelle con specificità più alta e gli stili in linea.

Tuttavia, la parola chiave `!important` non cambia la specificità del selettore CSS stesso. Garantisce semplicemente che la regola con `!important` venga applicata, anche se ci sono altre regole in conflitto con specificità più alta.

Un altro caso d’uso appropriato per la parola chiave `!important` è sovrascrivere gli stili di librerie o framework di terze parti quando non hai controllo sul CSS originale.

Tuttavia, un uso eccessivo della parola chiave `!important` può rendere difficile mantenere ed eseguire il debugging del CSS, perché rompe la naturale cascata degli stili e può portare a conseguenze indesiderate.

Quindi, è meglio usare la parola chiave `!important` con parsimonia.
