# --interactive--

HTML 표는 예전만큼 자주 사용되지는 않습니다. 하지만 프론트엔드 개발자라면 여전히 익숙해야 합니다. 표는 1990년대 초반 개발자들이 브라우저에서 데이터를 표시하기 위해 사용한 가장 초기 방법 중 하나입니다.

다음은 미국 노동통계국에서 표를 생성하는 데 사용한 코드 예시입니다:

:::interactive_editor

```html
<table id="quickfacts">
  <thead>
    <tr>
      <th colspan="2">Quick Facts: Software Developers, Quality Assurance Analysts, and Testers</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <th>2023 Median Pay</th>
      <td>
        $130,160 per year
        <br>$62.58 per hour
      </td>
    </tr>
    <tr>
      <th>Typical Entry-Level Education</th>
      <td>Bachelor's degree</td>
    </tr>
    <tr>
      <th>Work Experience in a Related Occupation</th>
      <td>None</td>
    </tr>
    <tr>
      <th>On-the-job Training</th>
      <td>None</td>
    </tr>
    <tr>
      <th>Number of Jobs, 2022</th>
      <td>1,795,300</td>
    </tr>
    <tr>
      <th>Job Outlook, 2022-32</th>
      <td>25% (Much faster than average)</td>
    </tr>
    <tr>
      <th>Employment Change, 2022-32</th>
      <td>451,200</td>
    </tr>
  </tbody>
  <tfoot>
    <tr>
      <th>If this table had a footer it would be here.</th>
    </tr>
  </tfoot>
</table>
```

:::

보시다시피, `table` 요소가 있고 그 안에 `id`가 `quickfacts`로 설정되어 있습니다. 표 안에는 표 헤드 요소인 `thead`, 표 본문 요소인 `tbody`, 그리고 표 푸터 요소인 `tfoot`가 있습니다.

표의 헤드, 본문, 푸터 요소 각각은 여러 개의 표 행 `tr`을 포함할 수 있습니다. 그리고 각 표 행은 그 행의 데이터를 레이블하는 표 헤더 `th`를 포함할 수 있습니다. 또한 여러 개의 개별 데이터 셀인 표 데이터 `td`도 포함할 수 있습니다.

이렇게 많은 HTML 요소가 있지만 너무 겁먹지 마세요 — 이들은 간단한 계층 구조를 따릅니다.

다음은 이 모든 요소를 포함하는 가장 간단한 표입니다:

:::interactive_editor

```html
<table>
  <thead>
    <tr>
      <th>The title of this table</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <th>First Row</th>
      <td>
        First Data Cell
      </td>
    </tr>
    <tr>
      <th>Second Row</th>
      <td>
        Second Data Cell
      </td>
    </tr>
  </tbody>
  <tfoot>
    <tr>
      <th>The footer of this table, which might contain date of publication, author credits, or other meta information.</th>
    </tr>
  </tfoot>
</table>
```

:::

보시다시피, 데이터 자체는 항상 `tr` 안에 있습니다 — 그리고 그 `tr` 요소 안에는 헤더가 있는 `th` 요소와 데이터가 있는 `td` 요소가 있습니다.

일부 웹사이트는 더 적절한 `div` 요소 대신 `table`를 사용해 자신만의 표를 만들기도 합니다.

일반 `div` 요소를 사용해 표 형식 데이터를 표시하는 것도 가능하지만, `table` 요소를 사용하는 것이 더 좋습니다.

오래전에는 개발자들이 웹페이지에서 데이터가 아닌 요소를 배치하기 위해 `table`를 사용하기도 했습니다. 이는 모범 사례로 간주된 적이 없습니다. 하지만 아직도 표를 이렇게 사용하는 코드베이스를 만날 수 있습니다.

요즘 개발자들은 CSS flexbox와 grid를 사용해 레이아웃을 만듭니다. freeCodeCamp에서 이 도구들을 나중에 자세히 다룰 예정입니다.

지금은 HTML 표를 원래 의도한 목적, 즉 표 형식 데이터를 표시하는 데만 사용하세요.
