### --question--

#### --text--

Worin unterscheidet sich die HTTP-Kommunikation von der WebSocket-Kommunikation?

#### --distractors--

HTTP ist bidirektional, während WebSocket unidirektional ist.

---

HTTP hält eine permanente Verbindung, während WebSocket nach jedem Austausch schließt.

---

HTTP und WebSocket folgen beide dem gleichen Anfrage-Antwort-Muster.

#### --answer--

HTTP folgt `Client --> Request --> Server --> Response`, während WebSocket `Client <--> Server` ist.

#### --text--

Wie verläuft der Nachrichtenfluss bei Pub/Sub?

#### --distractors--

Ein Publisher sendet an einen Abonnenten --> der Abonnent leitet an ein Topic weiter --> das Topic speichert die Nachricht.

---

Ein Topic sendet eine Nachricht --> Publisher empfangen sie --> Publisher leiten sie an Abonnenten weiter.

---

Ein Abonnent verbindet sich mit einem Publisher --> der Publisher sendet Daten --> der Abonnent bestätigt den Empfang.

#### --answer--

Ein Abonnent abonniert ein Topic --> ein Publisher sendet eine Nachricht an dieses Topic --> das Nachrichtensystem liefert sie an alle Abonnenten dieses Topics.
