# --interactive--

프로그래밍에서 범위는 코드의 여러 부분에서 변수의 가시성과 접근성을 의미합니다. 변수에 어디서 접근하거나 수정할 수 있는지를 결정합니다. JavaScript에서 범위를 이해하는 것은 깔끔하고 효율적이며 버그 없는 코드를 작성하는 데 매우 중요합니다. 범위에는 전역 범위, 지역 범위, 블록 범위의 세 가지 주요 유형이 있습니다.

전역 범위는 JavaScript 프로그램에서 가장 바깥쪽 범위입니다. 전역 범위에서 선언된 변수는 함수나 블록 내를 포함해 코드 어디서든 접근할 수 있습니다. 이러한 변수는 종종 전역 변수라고 부릅니다. 전역 변수는 편리할 수 있지만, 이름 충돌을 일으키고 코드를 유지보수하기 어렵게 만들 수 있으므로 자주 사용하지 않는 것이 좋습니다. 전역 변수의 예는 다음과 같습니다:

:::interactive_editor

```js
let globalVar = "I'm a global variable";

function printGlobalVar() {
    console.log(globalVar);
}

printGlobalVar(); // "I'm a global variable"
```

:::

이 예제에서 `globalVar`는 전역 범위에서 선언되어 `printGlobalVar` 함수 내부에서 접근할 수 있습니다.

반면 지역 범위는 함수 내에서만 접근할 수 있는 변수를 의미합니다. 지역 범위의 예는 다음과 같습니다:

:::interactive_editor

```js
function greet() {
    let message = "Hello, local scope!";
    console.log(message);
}

greet(); // "Hello, local scope!"
// console.log(message); // This will throw an error
```

:::

이 코드에서 `message`는 `greet` 함수 내의 지역 변수입니다. 함수 내부에서 사용할 수 있지만 함수 외부에서 접근하려 하면 오류가 발생합니다.

블록 범위는 ES6에서 `let`와 `const` 키워드와 함께 도입된 개념입니다. 블록은 중괄호 `{}` 안에 있는 코드 섹션으로, `if` 문, `for` 반복문, `while` 반복문 등이 이에 해당합니다. 반복문 개념은 다음 수업에서 다룰 예정입니다.

블록 내에서 `let` 또는 `const`로 선언된 변수는 해당 블록 내에서만 접근할 수 있습니다. 블록 범위의 예는 다음과 같습니다:

```js
if (true) {
    let blockVar = "I'm in a block";
    console.log(blockVar); // "I'm in a block"
}
console.log(blockVar); // This will throw an error
```

이 예제에서 `blockVar`는 `if` 블록 내에서만 접근할 수 있습니다. 블록 외부에서 접근하려 하면 오류가 발생합니다. 이러한 다양한 범위 유형을 이해하는 것은 변수 접근성을 관리하고 코드에서 의도하지 않은 부작용을 방지하는 데 필수적입니다.

전역 변수는 이름 충돌을 일으키고 코드를 유지보수하기 어렵게 만들 수 있으므로 자주 사용하지 않는 것이 좋습니다. 지역 변수는 코드의 서로 다른 부분을 격리하는 데 도움을 주어 특히 큰 프로그램에서 유용합니다. `let`와 `const`를 사용한 블록 범위는 변수 접근성을 더욱 세밀하게 제어하여 오류를 방지하고 코드를 더 예측 가능하게 만듭니다. 전역, 지역, 블록 범위의 기본 개념을 숙달하면 더 고급 주제를 이해하는 데 탄탄한 기반이 됩니다.
