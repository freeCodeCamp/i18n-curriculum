# --description--

Huduma za mtandao na REST APIs zinaendesha programu nyingi na tovuti unazotumia kila siku. Zinawawezesha programu tofauti kuwasiliana na kushirikiana data kupitia mtandao, mara nyingi mtandao wa intaneti.

API (Application Programming Interface) ni njia ya jumla kwa programu kuwasiliana na programu nyingine. Huduma ya mtandao ni aina maalum ya API inayotumia teknolojia za mtandao, kama HTTP, kubadilishana data kupitia mtandao, kawaida intaneti. Kwa sababu hii, kila huduma ya mtandao ni API—lakini si kila API ni huduma ya mtandao. REST APIs ni aina ya kawaida ya huduma za mtandao zinazofuata seti maalum ya sheria za jinsi data inavyotakiwa kuombwa na kurudishwa.

Funzo hili litazingatia huduma za mtandao kwa kiwango cha juu, na funzo lijalo litaelezea kwa undani zaidi jinsi REST APIs zinavyofanya kazi.

Unaweza kufikiria huduma ya mtandao kama daraja kati ya programu na programu. Haina kiolesura cha mtumiaji na haikusudiwi kutumika moja kwa moja. Badala yake, inalenga kabisa kubadilishana data kati ya programu.

Ili mfumo wowote uchukuliwe kama huduma ya mtandao, lazima uwe na sifa zifuatazo:

* **Inaendesha kwenye seva**: huduma ya mtandao lazima iwe mwenyeji mtandaoni kama tovuti, lakini badala ya kurasa ambazo watumiaji wanaweza kuona, inafichua data au kipengele fulani
    
* **Mawasiliano kati ya mashine**: huduma ya mtandao haijatengenezwa kwa ajili ya watu kuiona moja kwa moja, bali kwa programu za simu na programu za mtandao kuitumia, ili ziweze kufichua data au kipengele ambacho mtumiaji anaweza kuona hatimaye
    
* **Haitegemei lugha yoyote**: huduma ya mtandao inaweza kuandikwa kwa lugha yoyote kama Java au C#, na programu yoyote inapaswa kuweza kuingiliana nayo, hata kama imeandikwa kwa lugha tofauti
    
* **Haitegemei jukwaa lolote**: haijalishi ni jukwaa gani mashine zinaendesha. Kwa mfano, huduma ya mtandao inayotumia Linux haipaswi kuzuia kifaa kinachotumia Windows kuwasiliana nayo
    
* **Tumia itifaki za kawaida za mtandao**: huduma ya mtandao kawaida inapaswa kutumia HTTP/HTTPS kubadilishana data mtandaoni, FTP kwa uhamishaji wa faili, na SMTP kwa barua pepe
    

**REST** na **SOAP** ni aina mbili maarufu za huduma za mtandao, ingawa kuna mbinu nyingine kama **XML-RPC**.

**REST**, inayojulikana kama **uhamishaji wa hali ya uwakilishi (REST)**, ni inayotumika zaidi leo. Programu nyingi za kisasa hutegemea huduma za mtandao za RESTful kwa sababu ni rahisi, zina ufanisi, na zinafanya kazi kwa asili na mtandao.

REST APIs huweka wazi rasilimali, ambazo kawaida huwakilishwa na njia za URL kama `/users` au `/users/20`. Wakati mteja anaomba rasilimali, seva hurudisha uwakilishi wa rasilimali hiyo. Uwawakilishi huu mara nyingi huwa katika JSON, lakini pia unaweza kuwa katika muundo kama XML au hata maandishi ya kawaida.

Wateja wanaweza kubainisha muundo wanayotaka kwa kutumia vichwa vya HTTP, kama kichwa cha `Accept`. Kwa mfano, mteja anaweza kuomba jibu la JSON kwa kuweka thamani inayofaa ya `Accept`.

```bash
Accept: application/json
```

Na unaweza kubainisha unataka XML kwa njia hii:

```bash
Accept: application/xml
```

Rasilimali unayopata inaweza kuonekana hivi ikiwa ni JSON:

```json
{
  "id": 20,
  "name": "John Doe",
  "email": "john@example.com",
  "role": "admin",
  "createdAt": "2025-09-22T10:30:00Z"
}
```

Na hivi, ikiwa ni XML:

```xml
<user>
  <id>20</id>
  <name>John Doe</name>
  <email>john@example.com</email>
  <role>admin</role>
  <createdAt>2025-09-22T10:30:00Z</createdAt>
</user>
```

SOAP inasimama kwa **Simple Object Access Protocol**. Inatumia tu muundo wa data wa XML, na ni maarufu zaidi katika mifumo ya ngazi ya biashara inayohitaji viwango na usalama mkali zaidi.

Hapa ni mfano wa data ya SOAP:

```xml
<soapenv:Envelope xmlns:soapenv="http://schemas.xmlsoap.org/soap/envelope/">
   <soapenv:Body>
      <getUser>
         <userId>20</userId>
      </getUser>
   </soapenv:Body>
</soapenv:Envelope>
```

XML-RPC hutumia XML kuandika maombi na majibu yake. Hii inafanya iwe nyepesi lakini isiyo na ufanisi kama REST.

Kuhusu UDDI, si huduma ya kubadilishana data bali ni saraka inayotegemea XML ambapo biashara zinaweza kuorodhesha, kugundua, na kuelezea huduma zao za mtandao.
