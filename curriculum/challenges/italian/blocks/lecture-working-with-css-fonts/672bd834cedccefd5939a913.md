# --description--

Prima di iniziare, dovresti sapere cos’è una regola at in CSS. Le regole at sono istruzioni che forniscono indicazioni al browser. Puoi usarle per definire vari aspetti del foglio di stile, come media query, fotogrammi chiave, font face e altro ancora. Imparerai di più su concetti come media query e fotogrammi chiave nelle lezioni future. Questa volta, imparerai la regola at `@font-face`.

Con `@font-face` puoi definire un carattere tipografico personalizzato specificando il file del carattere, il formato e altre proprietà importanti, come peso e stile. Questa è la sintassi di base. Puoi vedere `@font-face` seguito da un blocco racchiuso tra parentesi graffe:

```css
@font-face {
  /* Descriptors */
}
```

All’interno delle parentesi graffe, dovrai includere dei descrittori per personalizzare il tuo font face. Vediamo alcuni dei più comunemente usati. Il descrittore font-family specifica il nome che userai in tutto il foglio di stile per riferirti a quel carattere. Per esempio, supponiamo che tu definisca questo `@font-face rule`. Ha il descrittore `font-family` definito e il suo valore è `MyCustomFont`:

```css
@font-face {
  font-family: "MyCustomFont";
}
```

In questo caso, dovresti usare quel nome nel tuo foglio di stile ovunque tu voglia assegnare quella famiglia di caratteri personalizzata a tutti gli elementi selezionati da un selettore CSS. Per esempio, lo useresti come valore di `font-family`:

```css
body { 
  font-family: "MyCustomFont"; 
}
```

Ma perché la regola at `@font-face` sia valida, devi anche specificare `src`. Questo contiene i riferimenti alle risorse del carattere. È fondamentalmente una lista di riferimenti esterni o nomi di font face installati localmente separati da virgole. Può anche includere suggerimenti sul formato e sulla tecnologia delle risorse del carattere.

Nel codice qui sotto, `src` è definito. Il valore è una lista di URL separati da virgole e scritti su righe separate per migliorare la leggibilità:

```css
@font-face {
  font-family: "MyCustomFont"; 
  src: url("path/to/font.woff2"),
    url("path/to/font.woff"),
    url("path/to/font.otf");
}
```

Puoi chiamare la funzione `url()` per includere un file nel tuo foglio di stile. In questo caso, stiamo includendo i file dei caratteri. Devi scrivere il percorso del file tra parentesi tonde e virgolette, includendo l’estensione del file. Per migliorare la leggibilità, puoi scrivere ogni risorsa su una riga diversa, ma l’ultima deve terminare con un punto e virgola.

Per ogni risorsa del carattere, puoi anche specificare il formato. Questo è opzionale. È un suggerimento per il browser sul formato del carattere. Se il formato viene omesso, la risorsa verrà scaricata e il formato sarà rilevato dopo il download. Se il formato è invalido, la risorsa non verrà scaricata.

I formati di font possibili includono `collection`, `embedded-opentype`, `opentype`, `svg`, `truetype`, `woff` e `woff2`.

Ecco un esempio con i formati di font. Nota come scriviamo il formato specifico tra parentesi e virgolette:

```css
@font-face {
  font-family: "MyCustomFont"; 
  src: url("path/to/font.woff2") format("woff2"),
    url("path/to/font.otf") format("opentype"),
    url("path/to/font.woff") format("woff");
}
```

In questo esempio, puoi vedere che stiamo specificando il formato WOFF2, il formato OpenType e il formato WOFF.

woff sta per "Web Open Font Format". La differenza tra WOFF e WOFF2 è l’algoritmo usato per comprimere i dati. OpenType è un formato per caratteri scalabili per computer sviluppato da Microsoft e Adobe che permette agli utenti di accedere a funzionalità aggiuntive in un carattere. È ampiamente usato nei principali sistemi operativi.

Oltre a specificare il formato, puoi anche specificare la tecnologia della risorsa del carattere. Anche questo è opzionale. Ecco un esempio in cui specifichiamo la tecnologia della seconda risorsa del carattere.

```css
@font-face {
  font-family: "MyCustomFont"; 
  src: url("path/to/font.woff2") format("woff2"),
    url("path/to/font.otf") format("opentype") tech(color-COLRv1),
    url("path/to/font.woff") format("woff");
}
```

Questi sono i fondamenti della regola `@font-face`. Con questa regola at puoi specificare il file del carattere per definire font personalizzati per i tuoi design unici.
