# --description--

讓我們來了解非同步 JavaScript。非同步事件會在不同時間發生，彼此獨立。

在軟體開發的背景關係中，「非同步」指的是在背景中執行、獨立於程式主流程的任務。非同步處理程序的主要優點是它們不會阻塞主程式的執行。

這對於可能需要很長時間來執行的任務特別有幫助，例如從遠端伺服器擷取資料、處理大型檔案、處理使用者輸入，以及執行複雜計算。這就是我們所稱的非同步程式設計。

這種方法與你迄今為止使用的傳統同步程式設計技術形成對比。當你撰寫多行程式碼時，你通常可以預測會發生什麼以及何時發生。第一行會被執行，然後是第二行，依此類推，按照你撰寫的順序。

在此範例中，第一行定義 `variable` 的程式碼會先執行，接著是第二行，最後是第三行將 `string` 記錄到 `console` 的程式碼：

```js
const topic = "JavaScript";
const learning = `I'm learning ${topic}!`;
console.log(learning);
```

一切都是依序且可預測的。每一行在下一行開始之前都會完成。這種類型的 JavaScript 程式稱為單執行緒。

執行緒的概念非常重要。`thread` 是處理程序中的一個執行單位。它就像程式中的一個獨立控制元件流程。

在同步程式設計中，執行緒依序執行，一個接著一個。如果執行緒被鎖住，例如等待使用者輸入，整個處理程序會被鎖住直到該執行緒完成。

在非同步程式設計中，執行緒可以並行執行，同時執行多個執行緒。這樣，即使其中一個執行緒被區段阻塞，程式仍然可以繼續同時執行多個任務，而不會讓主程式無回應。

非同步程式設計通常涉及回呼、promises 或 async/await 來控制非阻塞操作。

如你在先前課程中所學，回呼是你傳遞給另一個函式以便稍後呼叫的函式。事件處理常式是你之前使用過的一種特定型別的回呼。它們曾經是實作 JavaScript 非同步函式最常見的方式。然而，如果回呼函式也接受其他回呼函式，程式碼和邏輯可能會非常快速地變得相當複雜。 

目前，在 JavaScript 中實作非同步程式設計最常用的技術是 promise。`Promise` 是一個物件，表述非同步處理程序最終的完成（或失敗）及其值。

當 `promise` 被創建時，其值尚未知曉。只有當非同步行程完成時，才會知道其值。例如，從遠端 API 為你的網頁應用程式擷取資料的行程可能需要一些時間。與此同時，你希望你的使用者擁有良好的使用體驗，對吧？

為了實作這個，你可以創建一個 promise，以保持使用者介面作用中且可互動，同時非同步處理程序正在執行。當處理程序完成時，promise 將包含由 API 傳回的資料，因此應用程式可以在資料可用時適當地控制代碼或呈現它。

非同步程式設計是建造高效 JavaScript 應用程式的強大工具。透過了解同步與非同步程式設計之間的差異，你可以為你的應用程式選擇正確的方法，並撰寫更高效的程式碼。
