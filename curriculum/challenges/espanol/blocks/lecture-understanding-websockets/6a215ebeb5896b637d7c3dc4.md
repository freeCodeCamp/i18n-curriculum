# --description--

Node.js no tiene un servidor WebSocket integrado. Los desarrolladores suelen usar paquetes externos como `ws` para implementar el protocolo WebSocket y manejar la comunicación en tiempo real entre clientes y servidores.

La biblioteca `ws` es ligera, rápida y fácil de usar. Implementa el protocolo WebSocket para que puedas crear y gestionar conexiones WebSocket.

Puedes instalarla con npm:

```bash
npm install ws
```

Una vez instalada, crea un servidor WebSocket importando la biblioteca `ws`, invocando su método `Server` y pasando un puerto:

```js
const WebSocket = require("ws");

const server = new WebSocket.Server({
  port: 3000,
});
```

Esto inicia un servidor WebSocket que escucha conexiones en el puerto `3000`.

Cuando un cliente se conecta, el servidor emite un evento `connection`.

```js
server.on("connection", (socket) => {
  console.log("Client connected");
});
```

Piensa en este evento como la forma en que el servidor dice "un nuevo cliente se ha unido".

El parámetro `socket` representa la conexión entre el servidor y ese cliente. Lo usas para enviar y recibir mensajes.

Cuando un cliente conectado envía datos, el servidor emite un evento `message`.

```js
server.on("connection", (socket) => {
  socket.on("message", (message) => {
    console.log(message.toString());
  });
});
```

En este ejemplo, el servidor escucha mensajes entrantes y los registra en la consola.

Puedes enviar datos de vuelta a un cliente usando el método `send()`:

```js
socket.send("Hello from the server");
```

Esto envía un mensaje a través de la conexión WebSocket abierta.

Mientras la conexión permanezca abierta, tanto el cliente como el servidor pueden seguir intercambiando mensajes en tiempo real.

Además de `connection` y `message`, las conexiones WebSocket también emiten los eventos `close` y `error`.

`close` se activa cuando una conexión se cierra:

```js
socket.on("close", () => {
  console.log("Disconnected");
});
```

`error` se activa cuando algo sale mal:

```js
socket.on("error", (error) => {
  console.error(error);
});
```

Juntos, estos eventos forman la base de la mayoría de las aplicaciones WebSocket.

Un requisito común en aplicaciones de chat es la transmisión masiva. Transmitir significa enviar un mensaje a varios clientes conectados en lugar de solo uno.

Imagina una sala de chat con tres usuarios llamados **John Doe**, **Jane Doe** y **Wes Smith**.

Si John Doe envía un mensaje, el servidor puede necesitar reenviar ese mensaje a Jane Doe y Wes Smith.

Para hacer esto, el servidor mantiene un registro de los clientes conectados y recorre cada uno cuando llega un mensaje.

Antes de enviar un mensaje, el servidor verifica si la conexión del cliente sigue abierta:

```js
if (client.readyState === WebSocket.OPEN) {
  // do something
}
```

Cada conexión WebSocket tiene una propiedad `readyState` que describe su estado actual.

`WebSocket.OPEN` significa que la conexión está activa y lista para enviar y recibir datos.

Al verificar esto, el servidor evita enviar mensajes a clientes que ya se desconectaron o cuyas conexiones aún se están estableciendo:

```js
server.on("connection", (socket) => {
  socket.on("message", (message) => {
    server.clients.forEach((client) => {
      if (client.readyState === WebSocket.OPEN) {
        client.send(message.toString());
      }
    });
  });
});
```

En este ejemplo, cada cliente conectado recibe el mensaje entrante. Esto se llama transmisión masiva.

Muchas aplicaciones en tiempo real usan la transmisión masiva para entregar datos como mensajes de chat, notificaciones en vivo, actualizaciones deportivas y eventos de juegos multijugador.

Con la biblioteca `ws` es sencillo construir servidores WebSocket en Node.js.

Solo necesitas crear un servidor, escuchar eventos, intercambiar mensajes y transmitir actualizaciones a los clientes conectados.

Estos bloques básicos forman la base de muchas aplicaciones en tiempo real.
