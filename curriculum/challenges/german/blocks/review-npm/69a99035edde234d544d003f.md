## Was ist npm?

- **Definition**: npm ist ein Online-Register für Open-Source-Software und funktioniert wie ein „App Store“ für Code.
- **Offizielle Bedeutung**: npm steht offiziell nicht für „Node Package Manager“; es ist ein rekursives Backronym für „npm is not an acronym“.
- **Drei verwandte Teile**:

  - **Website (npmjs.com)**: Zum Nachschlagen von Paketen im Register.
  - **Registry**: Eine große öffentliche Datenbank, die alle Pakete speichert.
  - **CLI (Befehlszeilenschnittstelle)**: Ein mit Node.js installiertes Tool zur Verwaltung von Projektabhängigkeiten.

- **Alternativen**: Während npm am weitesten verbreitet ist, sind beliebte Alternativen **Yarn**, **PNPM** und **Bun**.

## Die package.json-Datei

- **Zweck**: Eine Konfigurationsdatei, die Metadaten, Abhängigkeiten und Skripte für ein Node.js-Projekt speichert.
- **Wichtige Felder**:

  - `name` und `version`: Identifizieren das Projekt.
  - `main`: Gibt die Einstiegsdatei an, oft `index.js`.
  - `license`: Definiert rechtliche Bedingungen (Standard ist `ISC`).
  - `dependencies`: Pakete, die zum Ausführen der App in der Produktion benötigt werden.
  - `devDependencies`: Werkzeuge, die nur während der Entwicklung benötigt werden, wie Test-Frameworks.


- **Erstellung**:

  - `npm init`: Startet einen interaktiven Einrichtungsprozess.
  - `npm init -y`: Installiert ein `package.json` mit Standardwerten sofort.

## Semantische Versionierung (SemVer)

- **Format**: `MAJOR.MINOR.PATCH`.
- **Regeln**:

  - **MAJOR**: Brechende Änderungen, die bestehenden Code beeinflussen können.
  - **MINOR**: Neue Funktionen, die abwärtskompatibel sind.
  - **PATCH**: Kleine Bugfixes und Verbesserungen.

- **Bereichssymbole**:

  - `^` (Caret): Erlaubt Minor- und Patch-Updates (z. B. `^1.2.3` erlaubt `1.x.x`).
  - `~` (Tilde): Erlaubt nur Patch-Updates (z. B. `~1.2.3` erlaubt `1.2.x`).
  - `*` (Sternchen): Passt auf jede Version; wird hauptsächlich zum Testen verwendet.

## Abhängigkeitsverwaltung

- **Installation**:

  - `npm install <name>`: Fügt eine Produktionsabhängigkeit hinzu.
  - `npm install <name> -D`: Fügt eine Entwicklungsabhängigkeit hinzu.
  - `npm install <name>@<version>`: Installiert eine bestimmte Version.

- **Node Modules & Locks**:

  - `node_modules/`: Enthält den tatsächlichen Code der installierten Pakete.
  - `package-lock.json`: Sperrt die genauen Versionen aller Abhängigkeiten und „Kind-Abhängigkeiten“, um Konsistenz über Umgebungen hinweg sicherzustellen.

- **Entfernung**: `npm uninstall <name>` entfernt das Paket und aktualisiert `package.json`.
- **Wartung**:

  - `npm outdated`: Erkennt, welche Pakete neuere Versionen verfügbar haben.
  - `npm update`: Aktualisiert Pakete innerhalb des erlaubten SemVer-Bereichs.

## npm-Skripte

- **Funktion**: Benutzerdefinierte Befehle, die im `scripts`-Abschnitt von `package.json` definiert sind, um Aufgaben zu automatisieren.
- **Ausführung**: Läuft über `npm run <script_name>`.
- Es gibt Kurzbefehle für `start`, `test`, `stop` und `restart` (z. B. `npm start`).
- **Sequenzielle Ausführung**:
- `&&`: Führt den zweiten Befehl nur aus, wenn der erste erfolgreich ist.
- `;`: Führt Befehle nacheinander aus, unabhängig vom Erfolg.
- **Parallele Ausführung**: Verwenden Sie den `&`-Operator, um Prozesse im Hintergrund laufen zu lassen.
- **Argumente**: Übergeben Sie Argumente an Skripte mit dem `--`-Trennzeichen (z. B. `npm start -- 8000`).

## Veröffentlichung im npm-Register

- **Registerzugriff**: Eine Datenbank mit öffentlichen und privaten Paketen.
- **Veröffentlichungsschritte**:

1. Erstellen Sie ein kostenloses Konto bei `npmjs.com`.
2. Melden Sie sich über das Terminal mit `npm login` an.
3. Stellen Sie sicher, dass ein eindeutiger `name` und eine gültige `version` in `package.json` vorhanden sind.
4. Erstellen Sie eine `README.md` und eine `.npmignore` (um sensible Dateien auszuschließen).
5. Führen Sie `npm publish` aus (oder `npm publish --access public` für Scoped-Pakete).

- **Scopes**: Werden verwendet, um Namenskonflikte zu vermeiden, indem unter `@username/package-name` veröffentlicht wird.

## Modulsysteme in Node.js

### CommonJS (CJS)

- **Standard-System**: Die ursprüngliche Methode, wie Node.js Module handhabt; lädt synchron.
- **Syntax**:
- **Import**: `const module = require('./path')`.
- **Export**: `module.exports = { functionName }`.
- **Identifikation**: Wird für `.cjs`-Dateien oder `.js`-Dateien verwendet, wenn kein `type` in `package.json` angegeben ist.

### ES-Module (ESM)

- **Standardisiertes Format**: Moderner JavaScript-Standard; lädt asynchron für mehr Effizienz.
- **Syntax**:

  - **Export**: `export { name }` oder `export default function`.
  - **Import**: `import { name } from './path.mjs'`.
  - **Identifikation**: Wird für `.mjs`-Dateien oder `.js`-Dateien verwendet, wenn `package.json` `"type": "module"` enthält.
