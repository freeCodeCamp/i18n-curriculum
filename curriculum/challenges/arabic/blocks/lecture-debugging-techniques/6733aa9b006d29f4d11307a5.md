# --description--

أثناء برمجتك بلغة JavaScript، لا بد أنك واجهت رسائل خطأ. فهم رسائل الخطأ الشائعة يساعدك على معالجة الأخطاء بشكل أكثر فعالية ويطور مهاراتك كمطوِّر. الأنواع الأربعة الشائعة لرسائل الخطأ هي `SyntaxError`، و`ReferenceError`، و`TypeError`، و`RangeError`.

يحدث `SyntaxError` عندما تكتب شيئًا بشكل غير صحيح في شفرتك، مثل فقدان قوس أو قوس معقوف. فكر فيه كأنه خطأ نحوي في جملة. إليك خطأ شائع يرتكبه المطوّرون عند إنشاء المصفوفات:

```js
const arr = ["Beau", "Quincy" "Tom"]
```

يجب فصل كل عنصر في المصفوفة بفاصلة، وإلا سينتج عن ذلك رسالة خطأ.

الخطأ الشائع الثاني في JavaScript هو `ReferenceError`. هناك عدة أنواع من `ReferenceError`، تُثار بطرق مختلفة. النوع الأول من `ReferenceError` هو المتغيرات غير المعرفة.

```js
console.log(price);
```

في هذا المثال، نحاول تسجيل المتغير `price` في وحدة التحكم لكنه لم يُعلن بعد. سينتج عن ذلك `ReferenceError`.

مثال آخر على `ReferenceError` هو محاولة الوصول إلى متغير مُعلن باستخدام `let` أو `const` قبل تهيئته:

```js
console.log(b);
const b = 50;
```

المثال أعلاه سينتج عنه خطأ `Cannot access 'b' before initialization`.

الخطأ الشائع الثالث هو `TypeError`. تحدث هذه الأخطاء عندما تحاول تنفيذ عملية على نوع غير صحيح. إليك مثالًا على محاولة استخدام طريقة `map` على كائن:

```js
const developerObj = {
  name: "Jessica",
  country: "USA",
  isEmployed: true
};

developerObj.map()
```

هذا المثال سينتج عنه خطأ `developerObj.map is not a function` لأن طريقة `map` تُستخدم للمصفوفات وليس للكائنات.

آخر خطأ شائع سنناقشه هو `RangeError`.

يحدث `RangeError` عندما تحاول شفرتك استخدام قيمة خارج النطاق الذي يمكن لـ JavaScript التعامل معه. إليك مثالًا على تعيين قيمة غير صالحة لطول المصفوفة:

```js
const arr = [];
arr.length = -1; 
```

بما أن طول المصفوفة `length` يجب أن يكون عددًا صحيحًا غير سالب، فإن تعيينه إلى `-1` يثير `RangeError`.

أثناء استمرارك في البرمجة بلغة JavaScript، كن على وعي بهذه الأنواع المختلفة من الأخطاء التي قد تواجهها ولماذا تحدث.
