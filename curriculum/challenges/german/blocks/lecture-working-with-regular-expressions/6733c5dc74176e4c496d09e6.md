# --interactive--

Lookahead- und Lookbehind-Zusicherungen ermöglichen es Ihnen, bestimmte Muster basierend auf dem Vorhandensein oder Fehlen umliegender Muster zu erkennen. Es gibt vier Varianten dieser Zusicherungen.

Zuerst gibt es die positive Lookahead-Zusicherung. Diese Zusicherung erkennt ein Muster, wenn das Muster von einem anderen Muster gefolgt wird.

Um eine positive Lookahead-Zusicherung zu erstellen, beginnen Sie mit dem Muster, das Sie erkennen möchten. Dann verwenden Sie runde Klammern, um das Muster einzuschließen, das Sie als Bedingung verwenden wollen. Nach der öffnenden Klammer verwenden Sie `?=`, um dieses Muster als positive Lookahead zu definieren.

Dieses Muster erkennt nur das Wort `free`, wenn es von `code` gefolgt wird:

```js
const regex = /free(?=code)/i;
```

Testen wir das Verhalten unseres Musters:

:::interactive_editor

```js
const regex = /free(?=code)/i;
console.log(regex.test("freeCodeCamp")); // true
console.log(regex.test("free code camp")); // false
console.log(
  regex.test("I need someone for free to write code for me")
); // false
```

:::

Beachten Sie, wie nur der String, bei dem `free` unmittelbar von `code` gefolgt wird, den Test besteht.

Aber was, wenn Sie das Vorhandensein von `free` erkennen wollen, wenn es NICHT von `code` gefolgt wird? Sie können Ihre positive Lookahead in eine negative Lookahead umwandeln, um das Verhalten umzukehren. Ändern Sie dazu Ihr `?=` in `?!`:

```js
const regex = /free(?!code)/i;
```

Testen wir das an denselben Strings:

:::interactive_editor

```js
const regex = /free(?!code)/i;
console.log(regex.test("freeCodeCamp")); // false
console.log(regex.test("free code camp")); // true
console.log(
  regex.test("I need someone for free to write code for me")
); // true
```

:::

Wie erwartet sind die Ergebnisse umgekehrt. Der einzige String, der nicht besteht, ist der erste, bei dem `free` unmittelbar von `code` gefolgt wird.

Lookbehind-Zusicherungen funktionieren ähnlich wie Lookahead-Zusicherungen, außer dass sie bedingt basierend auf einem vorhergehenden Muster erkennen, anstatt auf einem folgenden Muster. Schauen wir uns eine positive Lookbehind an.

Eine positive Lookbehind wird mit `?<=` statt `?=` bezeichnet. Lassen Sie unseren regulären Ausdruck `code` erkennen, wenn es von `free` vorausgeht:

```js
const regex = /(?<=free)code/i;
```

Genau wie bei unserer positiven Lookahead erkennt unsere positive Lookbehind den ersten String, weil `code` unmittelbar von `free` vorausgeht:

:::interactive_editor

```js
const regex = /(?<=free)code/i;
console.log(regex.test("freeCodeCamp")); // true
console.log(regex.test("free code camp")); // false
console.log(
  regex.test("I need someone for free to write code for me")
); // false
```

:::

Um `code` zu erkennen, wenn es NICHT von `free` vorausgeht, können wir eine negative Lookbehind verwenden. Eine negative Lookbehind wird definiert, indem `?<=` durch `?<!` ersetzt wird:

```js
const regex = /(?<!free)code/i;
```

Dies würde jede Vorkommnis von `code` erkennen, das NICHT unmittelbar von `free` vorausgeht:

:::interactive_editor

```js
const regex = /(?<!free)code/i;
console.log(regex.test("freeCodeCamp")); // false
console.log(regex.test("free code camp")); // true
console.log(
  regex.test("I need someone for free to write code for me")
); // true
```

:::

Denken Sie daran, dass `Regex.prototype.test` nur bestätigt, ob ein String dem regulären Ausdruck entspricht. Verwenden wir unsere negative Lookbehind mit `String.prototype.match`, um zu sehen, wie Zusicherungen das beeinflussen:

```js
const regex = /(?<!free)code/i;
console.log("freeCodeCamp".match(regex)); // null
console.log("free code camp".match(regex)); // ['code', index: 5, input: 'free code camp', groups: undefined]
console.log(
  "I need someone for free to write code for me".match(regex)
); // ['code', index: 33, input: 'I need someone for free to write code for me', groups: undefined]
```

Beachten Sie, dass obwohl unser regulärer Ausdruck eine Lookbehind verwendet, um das Vorhandensein von `free` zu prüfen, `free` nicht erkannt wird. Der einzige Text, der in der Übereinstimmung enthalten ist, ist `code`.

Lookaheads und Lookbehinds sind unglaublich nützlich, um Text bedingt zu erkennen, ohne den zurückgegebenen Wert Ihrer Übereinstimmung zu beeinflussen.
