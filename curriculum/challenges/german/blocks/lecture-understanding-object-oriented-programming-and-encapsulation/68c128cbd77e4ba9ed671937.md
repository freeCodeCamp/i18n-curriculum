# --description--

Getter und Setter sind Methoden, mit denen Sie steuern können, wie auf die Attribute einer Klasse zugegriffen und wie sie verändert werden. Mit Gettern rufen Sie einen Wert ab, und mit Settern legen Sie einen Wert fest.

Diese Aktionen erfolgen über sogenannte Eigenschaften. Sie verbinden Getter und Setter und ermöglichen den Zugriff auf Daten.

Eigenschaften verhalten sich wie Attribute, funktionieren unter der Haube aber wie Methoden. Man kann sie sich als Daten vorstellen, die Sie wie Methoden definieren, die aber wie Attribute arbeiten. Das bedeutet, Sie können auf Eigenschaften mit Punktnotation zugreifen, anstatt mit Klammern oder runden Klammern.

Das Wichtigste, was Eigenschaften tun, ist, dass sie hinter den Kulissen zusätzliche Logik ausführen, wenn Sie Werte mit ihnen abrufen, setzen oder löschen. Das macht sie zur perfekten Wahl, wenn Sie Daten innerhalb von Objekten zugreifen oder manipulieren möchten.

Warum sollten Sie dafür Eigenschaften statt Methoden verwenden? Es geht hauptsächlich um Lesbarkeit und Konvention. Sie machen Ihren Code sauberer und leichter lesbar.

Wenn Sie eine Methode verwenden, müssen Sie sie immer mit runden Klammern aufrufen. Mit einer Eigenschaft können Sie jedoch einfach wie bei einem normalen Attribut mit Punktnotation darauf zugreifen. Das lässt Ihren Code einfach aussehen, auch wenn im Hintergrund zusätzliche Arbeit erledigt wird.

Zum Beispiel möchten Sie vielleicht einen Wert berechnen oder prüfen, ob ein neuer Wert gültig ist, bevor Sie ihn speichern. Statt dafür eine Methode aufzurufen, können Sie das auf eine attributähnliche Weise erledigen.

In Python ist ein Decorator eine Funktion, die die Funktionalitäten anderer Funktionen oder Klassen verändert, ohne deren ursprünglichen Code zu ändern.

Obwohl es möglich ist, eigene Decorators zu erstellen, geht das über den Umfang dieser Lektion hinaus. Wissen Sie einfach, dass Sie zum Erstellen einer Eigenschaft eine Methode definieren und den `@property`-Decorator darüber setzen. Dadurch wird die Methode zu einer Eigenschaft, sodass Sie wie auf ein Attribut zugreifen können, während intern die dekorierte Methode aufgerufen wird.

Kommen wir zu den Gettern. So erstellen Sie einen mit dem `@property`-Decorator:

```py
class Circle:
    def __init__(self, radius):
        self._radius = radius

    @property
    def radius(self): # A getter to get the radius
        return self._radius
  
    @property
    def area(self):  # A getter to calculate area
        return 3.14 * (self._radius ** 2)

my_circle = Circle(3)

print(my_circle.radius) # 3
print(my_circle.area) # 28.26
```

Dieses Beispiel erhält den Radius und die Fläche eines Kreises.

Beachten Sie, wie wir innerhalb der Klasse `_radius` anstelle von radius verwendet haben. Der Unterstrich ist eine gängige Python-Konvention, um zu zeigen, dass ein Attribut privat sein soll. Mit anderen Worten, es signalisiert, dass es für den internen Gebrauch gedacht ist und nicht direkt von außerhalb der Klasse aufgerufen werden sollte.

Um zum Beispiel einen Setter für den Radius zu erstellen, müssen Sie eine weitere Methode mit demselben Namen definieren und darüber `@<property_name>.setter` verwenden:

Die Verwendung von `self.radius` innerhalb von `__init__` stellt sicher, dass der Setter während der Objekterstellung ausgeführt wird, sodass ungültige Radiuswerte sofort erkannt werden.

```py
class Circle:
    def __init__(self, radius):
        self.radius = radius # Calling the setter

    @property
    def radius(self):  # A getter to get the radius
        return self._radius

    @radius.setter
    def radius(self, value):  # A setter to set the radius
        if value <= 0:
            raise ValueError('Radius must be positive')
        self._radius = value

my_circle = Circle(3)
print('Initial radius:', my_circle.radius) # Initial radius: 3

my_circle.radius = 8
print('After modifying the radius:', my_circle.radius) # After modifying the radius: 8
```

In diesem Beispiel setzt der Radius-Setter nicht nur den Radius für den Kreis, sondern führt auch eine Validierung durch, die sicherstellt, dass der Radius keine negative Zahl ist.

Sobald Sie Getter und Setter definiert haben, ruft Python sie automatisch im Hintergrund auf, wann immer Sie die normale Attributsyntax verwenden:

```py
my_circle.radius # This will call the getter
my_circle.radius = 4 # This will call the setter
```

Beachten Sie, dass Sie innerhalb des Setters nicht denselben Namen der Eigenschaft verwenden können, wenn Sie einen neuen Wert zuweisen. Das liegt daran, dass `self.radius = value` den Setter innerhalb der Setter-Methode selbst aufrufen würde, was zu einer unendlichen Rekursion und einem `RecursionError` führt. Deshalb müssen Sie immer die mit einem Unterstrich versehene Form `self._radius = value` verwenden.

So wie Sie steuern können, wie auf ein Attribut über einen Getter zugegriffen wird und wie es mit einem Setter verändert wird, können Sie auch steuern, wie es mit einem Deleter gelöscht wird.

Ein Deleter führt benutzerdefinierte Logik aus, wenn Sie die del-Anweisung für eine Eigenschaft verwenden. Um einen zu erstellen, verwenden Sie den `@<property_name>.deleter`-Decorator:

```py
class Circle:
    def __init__(self, radius):
        self.radius = radius

    # Getter
    @property
    def radius(self):
        return self._radius

    # Setter
    @radius.setter
    def radius(self, value):
        if value <= 0:
            raise ValueError("Radius must be positive")
        self._radius = value

    # Deleter
    @radius.deleter
    def radius(self):
        print("Deleting radius...")
        del self._radius
```

So kann der Deleter verwendet werden:

```py
# Create circle object with a radius
my_circle = Circle(33)
print("Initial radius:", my_circle.radius)  # 33

# Delete the radius
# This calls the deleter
del my_circle.radius # Deleting radius...
print("Radius deleted!") # Radius deleted!

# Try to access radius after deletion
try:
    print(my_circle.radius)
except AttributeError as e:
    print("Error:", e) # Error: 'Circle' object has no attribute '_radius'
```

Die wichtigsten Erkenntnisse daraus sind:

- Getter ermöglichen es Ihnen, einen Wert abzurufen oder sogar einen Wert dynamisch zu berechnen.
- Setter erlauben es Ihnen, Werte sicher zu ändern, indem sie vor der Zuweisung Prüfungen durchführen.
- Eigenschaften verbinden diese Getter und Setter, sodass Sie Logik schreiben können und trotzdem die Punktnotation verwenden.
- Deleter definieren, was passiert, wenn ein Attribut gelöscht wird.
