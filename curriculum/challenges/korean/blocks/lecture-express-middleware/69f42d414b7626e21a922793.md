# --description--

라우터 수준 미들웨어는 애플리케이션 수준 미들웨어와 비슷하지만, 메인 앱 인스턴스 대신 `express.Router()` 인스턴스에 바인딩됩니다. 이를 통해 Express 애플리케이션을 분리된 라우트 모듈로 모듈화할 수 있습니다.

라우터 수준 미들웨어는 애플리케이션 수준 미들웨어와 동일한 방식으로 작동하지만 특정 라우터 인스턴스에만 적용됩니다. 이로 인해 미들웨어를 전역이 아닌 특정 라우트 집합에만 적용하기가 더 쉬워집니다.

```js
const express = require('express')
const app = express()
const router = express.Router()

// Router-level middleware to log requests to '/menu' routes
router.use((req, res, next) => {
  console.log('Request made to /menu route')
  next() // Pass control to the next handler
})

// Define routes
router.get('/drinks', (req, res) => {
  res.send('Welcome to the drinks menu!')
})

app.use('/menu', router) // Apply the router

app.listen(3000, () => {
  console.log('Server running on http://localhost:3000')
})
```

* `router.use()`: 이것은 라우터 수준 미들웨어로, 라우터가 처리하는 라우트에만 적용됩니다.
* `app.use('/menu', router)`: 미들웨어는 `/menu`로 시작하는 라우트에만 적용됩니다(예: `/menu/drinks`).

라우터 수준 미들웨어를 사용하면 애플리케이션을 모듈형 컴포넌트로 나누고 미들웨어를 특정 라우트 그룹에 선택적으로 적용할 수 있습니다. 이는 구조화와 코드 명확성에 도움이 됩니다.
