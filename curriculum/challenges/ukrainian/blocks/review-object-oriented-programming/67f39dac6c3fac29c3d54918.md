## Що таке об’єктно-орієнтоване програмування?

- **Об’єктно-орієнтоване програмування**: стиль програмування, у якому розробники розглядають усе у своєму коді як реальний об’єкт. Його часто називають OOP. Чотири ключові принципи, які допомагають ефективно організовувати та керувати кодом, — це **інкапсуляція**, **успадкування**, **поліморфізм** та **абстракція**.
- **Класи**: шаблон для створення об’єктів. Кожен об’єкт, створений із класу, має атрибути, які визначають дані, і методи, що визначають поведінку об’єктів.

## Що таке інкапсуляція?

- **Інкапсуляція**: об’єднання атрибутів і методів об’єкта в єдину одиницю. Вона дозволяє приховати внутрішній стан об’єкта за простим набором публічних методів і атрибутів, які діють як двері. За цими дверима — приватні атрибути і методи, які контролюють, як змінюються дані і хто може їх бачити.
- **Приклад інкапсуляції**: якщо ви хочете відстежувати баланс гаманця, ви дозволите внесення і зняття коштів, але не захочете, щоб хтось міг змінювати сам баланс гаманця:

```py
class Wallet:
   def __init__(self, balance):
       self.__balance = balance # Private attribute

   def deposit(self, amount):
       if amount > 0:
           self.__balance += amount # Add to the balance safely

   def withdraw(self, amount):
       if 0 < amount <= self.__balance:
           self.__balance -= amount # Remove from the balance safely

account = Wallet(500)
print(account.__balance) # AttributeError: 'Wallet' object has no attribute '__balance'
```

- **Різниця між префіксами атрибутів з одним і подвійним підкресленням**: префікс з одним підкресленням означає, що атрибути і методи призначені для внутрішнього використання. Це конвенція, яка не забороняє доступ до атрибутів ззовні. Префікс з подвійним підкресленням ефективно забороняє доступ до них поза межами класу.

## Що таке геттери і сеттери?

- **Геттери і сеттери**: методи, які дозволяють контролювати, як атрибути класу отримуються і змінюються. Ви отримуєте значення за допомогою геттерів і призначаєте значення за допомогою сеттерів.
- **Властивості**: вони поєднують геттери і сеттери, дозволяючи доступ до даних. Вони виконують додаткову логіку за лаштунками, коли ви отримуєте, призначаєте або видаляєте значення.
- **Чому властивості замість методів**: властивості використовують замість методів для кращої читабельності і чистішого коду. Вони дозволяють отримувати значення через точкову нотацію, як звичайні атрибути, без круглих дужок.
- **Створення геттера**: щоб створити геттер, використовуйте декоратор `@property`. Ось геттер, який отримує радіус кола:

```py
class Circle:
    def __init__(self, radius):
        self._radius = radius

    @property
    def radius(self): # A getter to get the radius
        return self._radius
  
    @property
    def area(self):  # A getter to calculate area
        return 3.14 * (self._radius ** 2)

my_circle = Circle(3)

print(my_circle.radius) # 3
print(my_circle.area) # 28.26
```

- **Створення сеттера**: щоб створити сеттер, який призначатиме радіус, потрібно визначити інший метод з тим самим ім’ям і використати `@<property_name>.setter` над ним:

```py
class Circle:
    def __init__(self, radius):
        self._radius = radius

    @property
    def radius(self):  # A getter to get the radius
        return self._radius

    @radius.setter
    def radius(self, value):  # A setter to set the radius
        if value <= 0:
            raise ValueError('Radius must be positive')
        self._radius = value

my_circle = Circle(3)
print('Initial radius:', my_circle.radius) # Initial radius: 3

my_circle.radius = 8
print('After modifying the radius:', my_circle.radius) # After modifying the radius: 8
```

- **Як Python обробляє геттери і сеттери**: після визначення геттерів і сеттерів Python автоматично викликає їх під капотом щоразу, коли ви використовуєте звичайний синтаксис атрибутів ось так:

```py
my_circle.radius # This will call the getter
my_circle.radius = 4 # This will call the setter
```

 При призначенні значення не слід призначати властивості безпосередньо, бо це спричинить `RecursionError`. Використовуйте окреме внутрішнє ім’я, часто з підкресленням, щоб зберігати значення.

- **Делетер**: після встановлення і отримання значення за допомогою сеттера і геттера, ви можете контролювати, як його видалити, за допомогою `deleter`. Делетер виконує кастомну логіку, коли ви використовуєте інструкцію `del` для властивості. Щоб створити делетер, використовуйте декоратор `@<property_name>.deleter`.

```py
  # Deleter
    @radius.deleter
    def radius(self):
        print("Deleting radius...")
        del self._radius
```

## Що таке успадкування і як воно сприяє повторному використанню коду?

- **Успадкування**: процес, за якого дочірній клас використовує атрибути і методи батьківського класу. Успадкування сприяє повторному використанню коду, забезпечує чітку ієрархію і дозволяє налаштовувати поведінку без переписування всього. Щоб реалізувати успадкування, дочірній клас приймає ім’я батьківського класу:

```py
class Parent:
    # Parent attributes and methods

class Child(Parent):
    # Child inherits, extends, and/or overrides where necessary
```

- **Одинарне і множинне успадкування**: коли дочірній клас успадковує властивості і методи від одного батька, як показано вище, це називається **одинарним успадкуванням**. Коли дочірній клас успадковує властивості і методи від кількох батьків, це **множинне успадкування**. Ось синтаксис для цього:

```py
class Parent:
    # Attributes and methods for Parent

class Child:
    # Attributes and methods for Child

class GrandChild(Parent, Child):
    # GrandChild inherits from both Parent and Child
    # GrandChild can combine or override behavior from each
```

- **Функція `super()`**: функція, яка дозволяє **викликати** метод батьківського класу, коли клас має іншу реалізацію цього методу або розширює його, без дублювання коду.

## Що таке поліморфізм і як він сприяє повторному використанню коду?

- **Поліморфізм**: принцип OOP, який дозволяє різним класам використовувати однакову назву методу, але кожен клас реалізує його по-своєму при виклику. Ось синтаксис для цього:

```py
class A:
   def action(self): ...

class B:
   def action(self): ...

class C:
   def action(self): ...

Class().method()  # Works for A, B, or C
```

- **Поліморфізм на основі успадкування**: батьківський клас задає метод, а кожен дочірній клас адаптує його під свої потреби.

## Що таке name mangling і як він працює?

- **Name mangling**: процес, у якому Python внутрішньо перейменовує атрибут з подвійним підкресленням, додаючи підкреслення і ім’я класу як префікс, перетворюючи `__attribute` на `_ClassName__attribute`.
- **Мета name mangling**: основна мета name mangling — запобігти випадковому перекриттю атрибутів і методів при використанні успадкування. Ось код, який робить це більш зрозумілим:

```py
class Parent:
    def __init__(self):
        self.__data = 'Parent data'

class Child(Parent):
    def __init__(self):
        super().__init__()
        self.__data = 'Child data'

c = Child()
print(c.__dict__) # {'_Parent__data': 'Parent data', '_Child__data': 'Child data'}
```

## Що таке абстракція і як вона допомагає тримати складні системи організованими?

- **Абстракція**: концепція програмування, у якій приховують складні деталі реалізації об’єкта або системи і показують лише суттєві функції. У Python та інших мовах програмування абстракція спрощує складні системи, підвищуючи повторне використання.
- **Приклад абстракції**: хороший приклад абстракції у повсякденному житті — це автомобіль, який дозволяє користуватися кермом, педалями і важелем перемикання передач, не знаючи, як працює двигун чи гальма.
- **Як Python реалізує абстракцію**: Python реалізує абстракцію через модуль `abc`. Модуль надає клас `ABC` (абстрактний базовий клас) і декоратор `@abstractmethod`. Абстрактний базовий клас (ABC) визначає загальні методи і властивості, які мають реалізувати підкласи. Його не можна інстанціювати.
- **Як визначається абстрактний метод**: абстрактний метод визначається за допомогою `@abstractmethod` і має бути перевизначений у підкласах, навіть якщо має реалізацію за замовчуванням. Основний синтаксис абстракції виглядає так:

```py
from abc import ABC, abstractmethod

# Define an abstract base class
class AbstractClass(ABC):
    @abstractmethod
    def abstract_method(self):
        pass

# Concrete subclass that implements the abstract method
class ConcreteClassOne(AbstractClass):
    def abstract_method(self):
        print('Implementation in ConcreteClassOne')

# Another concrete subclass
class ConcreteClassTwo(AbstractClass):
    def abstract_method(self):
        print('Implementation in ConcreteClassTwo')
```
