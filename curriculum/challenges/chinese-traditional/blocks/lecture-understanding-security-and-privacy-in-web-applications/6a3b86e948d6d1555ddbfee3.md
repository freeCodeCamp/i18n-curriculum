# --description--

同源政策是你網頁瀏覽器內建的安全機制，防止一個網站讀取另一個網站的私人資料。這條規則保護你的個人帳戶免受駭客攻擊。

一個來源包含三個特定部分：網路協定、網域名稱和埠號。

例如，在位址 `https://example.com:443` 中，協定是 HTTPS，網域名稱是 `example.com`，埠號是 `443`。

瀏覽器通常會隱藏標準的埠號。HTTPS 預設為埠 443，HTTP 預設為埠 80。即使隱藏，瀏覽器仍會檢查它。只有當這三個部分完全相符時，兩個網站才算是同源。

在這條嚴格規則下，瀏覽器會阻止網站窺探不同網站。想像你在一個瀏覽器分頁打開線上銀行帳戶，另一個分頁打開惡意網站。同源政策會阻止惡意網站竊取你的銀行資訊。

然而，現代網站經常需要跨不同來源分享資料。例如，前端網頁可能需要從獨立的後端伺服器拉取資料。瀏覽器的同源規則會阻擋這些安全的請求，因此開發者需要一種安全分享資料的方法。這就是 CORS 的用武之地。

CORS 是一套讓伺服器透過瀏覽器規則開放存取的系統。它使用特殊標頭來授權受信任網站存取資料。

當你的瀏覽器向另一個網站請求資料時，會發生以下情況：

* 它會包含一個 `Origin` 標頭，指出發出請求的網站名稱
* 接收伺服器會檢查這個名稱
* 如果伺服器信任該網站，會回覆一個特殊的授權標頭，告訴瀏覽器該網站被允許讀取傳入的資料。

如果伺服器沒有發送這個授權，瀏覽器會立即阻擋資料。前端網站會收到錯誤訊息，而非私人檔案。

在使用 Express 的 Node.js 應用程式中，CORS 是透過 `cors` 軟體包來實作的：

```javascript
const express = require('express');
const cors = require('cors');
const app = express();

// this enables CORS for all origins and all routes for development
app.use(cors());
```

在生產環境中，你可能需要傳入一個設定物件來限制特定網域的存取：

```javascript
const corsOptions = {
  origin: ['https://yourapp.com', 'https://admin.yourapp.com'],
  credentials: true
};

app.use(cors(corsOptions));
```

如果你使用原生 Node.js，必須手動在每個請求上設定 CORS 回應標頭：

```javascript
res.setHeader('Access-Control-Allow-Origin', 'https://example.com');
res.setHeader('Access-Control-Allow-Methods', 'GET, POST, PUT, DELETE, OPTIONS');
res.setHeader('Access-Control-Allow-Headers', 'Content-Type, Authorization');
```
