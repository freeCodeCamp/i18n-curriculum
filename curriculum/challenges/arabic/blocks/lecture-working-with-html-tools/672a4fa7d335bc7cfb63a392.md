# --description--

HTML لغة متسامحة جدًا – العناصر تُعرض حتى عندما ترتكب أخطاء، مثل نسيان تضمين وسم الإغلاق.

لنفترض أن لديك عنصر `h2` بدون وسم إغلاق:

```html
<h1>Article Topic</h1>
<h2>Subheading 1 </h2>
<h2>Subheading 2 </h2>

<!-- This h2 does not have a closing tag -->
<h2>Subheading 3
```

سيظل عنصر `h2` بدون وسم إغلاق يُعرض بشكل صحيح. يحدث هذا لأن المتصفحات تستخدم خوارزمية تحليل تتعامل مع الأخطاء الشائعة وتحاول عرض HTML بأقرب شكل ممكن إلى نية المؤلف.

لكن هذا قد يسبب مشاكل أحيانًا. لنضف بعض الفقرات تحت وسوم العنوان 2 الموجودة في الشفرة:

```html
<h1>Article Topic</h1>
<p>Lorem ipsum dolor sit amet consectetur adipisicing elit. Maiores, nisi.</p>

<h2>Subheading 1 </h2>
<p>Lorem, ipsum dolor sit amet consectetur adipisicing elit. At, doloremque.</p>

<h2>Subheading 2 </h2>
<p>Lorem ipsum dolor sit amet consectetur adipisicing elit. Unde, placeat.</p>

<!-- This h2 does not have a closing tag -->
<h2>Subheading 3
<p>Lorem ipsum dolor, sit amet consectetur adipisicing elit. Tempore, illum.</p>
```

نتيجة لذلك، يتم عرض عنصر الفقرة تحت `h2` الذي لا يحتوي على وسم إغلاق `h2` كعنوان من المستوى الثاني. لهذا السبب تحتاج إلى مدقق HTML.

مدقق HTML هو أداة تتحقق من صحة شفرة HTML الخاصة بك مقابل مواصفات HTML القياسية. يساعدك على تحديد الأخطاء والتحذيرات في شفرتك، مما يضمن أن صفحات الويب منظمة بشكل صحيح ومتوافقة مع معايير الويب.

استخدام مدقق HTML يفيدك أنت ومراجعات الشفرة المستقبلية، وكذلك أي شخص آخر يراجع شفرتك، مثل زملائك في الفريق والمساهمين في المصادر المفتوحة.

هناك عدة مدققات HTML يمكنك استخدامها. الأكثر قبولًا هو خدمة التحقق من الترميز في `w3.org`.

عند زيارة الموقع [`validator.w3.org`](https://validator.w3.org/)، يمكنك النقر على زر `Validate by Direct Input` ولصق شفرة HTML الخاصة بك.

عند النقر على زر `Check`، ستظهر قائمة بالنتائج مع الأخطاء التي تحتاج إلى إصلاح.

مدقق HTML آخر يمكنك استخدامه هو [`jsonformatter.org`](https://jsonformatter.org/).

يمكنك نسخ ولصق شفرة HTML داخل المحرر الأول، وعند النقر على زر `Validate`، سيعرض لك أي أخطاء في شفرتك.
