# --description--

이전 수업에서 `string`, 그리고 `boolean` 같은 원시 타입을 다루는 방법을 배웠습니다. 하지만 TypeScript는 기존 타입을 조합해 새로운 타입을 만들 수 있게 해줍니다. 그중 하나가 유니언 타입입니다.

유니언 타입은 두 개 이상의 다른 타입을 결합한 타입입니다.

기본 구문은 다음과 같습니다:

```ts
type1 | type2 | type3
```

개념을 더 잘 이해하기 위해 몇 가지 예제를 살펴보겠습니다.

다음은 `id` 또는 `string` 타입의 유니언 타입을 가진 `number`라는 변수입니다:

```ts
let id: string | number = "user_783";
```

값을 숫자로 업데이트해도 유니언 타입에 여전히 유효합니다:

```ts
let id: string | number = 1024;
```

만약 `string` 또는 `number`가 아닌 다른 값을 할당하려 하면 TypeScript가 오류 메시지를 표시합니다:

```ts
// Type 'string[]' is not assignable to type 'string | number'.
let id: string | number = ["Java", "Python"];
```

유니언 타입을 사용할 때는 모든 타입에 대해 작동하는 메서드를 사용하는 것이 중요합니다. 그렇지 않으면 TypeScript가 다음과 같은 오류를 발생시킵니다:

```ts
function getId(id: string | number) {
  // Property 'toUpperCase' does not exist on type 'string | number'.
  return id.toUpperCase();
}
```

이 문제를 해결하는 방법으로 유니언을 좁히는(narrowing) 방법이 있습니다. 이 방법과 유니언 타입이 타입 별칭과 어떻게 작동하는지는 앞으로 수업에서 더 배우게 됩니다.

유니언 타입은 `null`과 `undefined` 값을 다룰 때도 유용합니다.

다음은 변수가 `string` 값을 가질 수도 있고, 아직 값이 설정되지 않았다면 `null`일 수도 있는 예입니다:

```ts
let username: string | null;

// Type 'number' is not assignable to type 'string | null'.
username = 42;
```

이 경우 `username`은 `string` 또는 `null` 중 하나일 수 있지만 그 외의 값은 가질 수 없습니다. 값이 없음을 의도적으로 나타내고 싶을 때 `null`를 사용할 수 있습니다.

다음은 `undefined`를 사용한 또 다른 예입니다:

```ts
let score: number | undefined;

score = undefined; 

// Type 'string' is not assignable to type 'number | undefined'.
score = "high";
```

이 예제에서 `score`은 `number` 또는 `undefined` 중 하나일 수 있습니다. 값이 아직 할당되지 않은 경우 `undefined`를 사용할 수 있습니다.

`null`과 `undefined`을 유니언 타입으로 사용하면 값이 없을 수도 있음을 명확히 하면서도 TypeScript가 잘못된 할당을 잡아낼 수 있습니다.
