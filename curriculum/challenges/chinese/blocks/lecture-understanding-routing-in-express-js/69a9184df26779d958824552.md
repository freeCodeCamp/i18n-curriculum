# --description--

在 Web 应用中，提供静态文件是一个常见需求。静态文件包括图像、CSS 文件、客户端 JavaScript、字体以及其他不会动态变化的资源。

Express 通过其内置的 `express.static()` 中间件，使得提供这些文件变得简单。

在 Express 中，静态文件服务意味着直接从服务器的文件系统将文件传送到客户端浏览器，而无需任何额外的服务器端处理。

例如，当浏览器请求 `/styles.css`、`/script.js` 或 `/images/logo.png` 这样的文件时，Express 可以在指定的文件夹中找到这些文件并直接提供。如果找不到文件，Express 会自动返回 404。由于这些文件不需要在服务器上生成或修改，它们被称为*静态*文件。

这让提供样式表和图像等资源变得简单，无需为每个文件手动定义路由。

下面是如何在 Express 中设置静态文件服务：

```javascript
const express = require("express");
const path = require("path");
const app = express();
const port = 3000;

// Serve static files from the "public" directory
app.use(express.static(path.join(__dirname, "public")));

app.get("/", (req, res) => {
  res.send("Welcome to the static file demo!");
});

app.listen(port, () => {
  console.log(`Server running at http://localhost:${port}`);
});
```

首先，我们导入 `express` 来创建服务器，导入 `path` 以便在不同操作系统间安全地构造文件路径。

接着，`app.use(express.static(path.join(__dirname, "public")))` 告诉 Express 从 `public` 文件夹提供文件。`express.static()` 中间件告诉 Express 从 `public` 文件夹提供文件。该目录中的任何文件都可以被浏览器访问。

我们还定义了一个简单的 `GET` 路由，路径为 `/`，用于发送欢迎消息。这表明静态文件服务可以与动态路由共存。

最后，`app.listen()` 在端口 3000 启动服务器。服务器运行后，对 `public` 文件夹中静态资源的请求会被自动处理。

要实现这一点，你需要一个名为 `public` 的文件夹来保存静态文件。一个典型的项目结构可能如下：

```md
my-app/
│
├── app.js
└── public/
    ├── index.html
    ├── styles.css
    └── image.png
```

当用户发出请求，例如访问 `http://localhost:3000/index.html` 时，Express 会检查请求的文件是否存在于 `public` 目录中。如果文件存在，Express 会直接发送给浏览器。如果文件不存在，Express 会继续执行下一个中间件或返回 404 响应。

记住，你不需要为每个文件定义路由，因此请求 `http://localhost:3000/styles.css` 时，Express 会自动查找 `public/styles.css` 并返回它。

你还可以自定义静态文件的服务方式。例如，你可以从多个目录提供文件，或者配置缓存行为。

假设你的应用有如下文件结构：

```md
my-app/
│
├── app.js
│
├── public/
│   ├── index.html
│   └── styles.css
│
└── images/
    └── image.png
```

下面是如何注册多个目录来提供静态文件：

```javascript
app.use(express.static(path.join(__dirname, "public")));
app.use("/images", express.static(path.join(__dirname, "images")));
```

这里我们像之前一样使用 `app.use(express.static(path.join(__dirname, "public")));`，这意味着 `public` 目录中的文件从根路径提供，就像之前一样。

但如果你看下一行，会看到我们将 `"/images"` 作为第一个参数传给 `app.use()`，后面跟着 `express.static(path.join(__dirname, "images"))`。在这种情况下，`"/images"` 是一个挂载路径，它告诉 Express 只有当收到 `/images` 路径的请求时才使用这个中间件。此时，`express.static()` 被触发，查找 `images` 目录中的文件。

所以根据这个配置，以下请求的处理方式如下：

*   GET `http://localhost:3000/styles.css`：Express 在 `public` 目录查找 `styles.css`，找到文件后发送给客户端。
    
*   GET `http://localhost:3000/images/image.png`：由于请求以 `/images` 开头，Express 在 `images` 目录查找 `image.png`，找到文件后发送给客户端。
    

使用 Express 提供静态文件既简单又强大。通过使用 `express.static()`，你可以轻松提供图像、样式表和客户端 JavaScript 等资源。

将静态文件保存在专门的 `public` 文件夹，或者使用多个文件目录和挂载路径，有助于保持应用的组织性、可维护性和易扩展性。
