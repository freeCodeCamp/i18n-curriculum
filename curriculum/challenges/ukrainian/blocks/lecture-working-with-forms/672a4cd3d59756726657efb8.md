# --interactive--

Елемент `button` використовується для виконання певної дії при його активації. Ось приклад елемента `button` з текстом кнопки `Start Game`.

```html
<button>Start Game</button>
```

Інші приклади використання елемента `button` включають відправлення форми, показ модального вікна або відкриття та закриття бічного меню. Елемент `button` має атрибут `type`, який контролює поведінку кнопки при її активації. Першим можливим значенням атрибута `type` буде тип `button`. Ось приклад використання елемента `button` з типом `button` і текстом `Show Alert`:

```html
<button type="button">Show Alert</button>
```

За замовчуванням кнопка нічого не робить при активації. Однак ви можете додати JavaScript-код, щоб зробити кнопку інтерактивною, наприклад, показати alert у цьому випадку.

Натисніть на кнопку `Show Alert`, щоб побачити, як на екрані з’явиться alert. Щоб взаємодіяти з прикладом, потрібно увімкнути інтерактивний редактор.


**ПРИМІТКА**: Цей інтерактивний приклад використовує JavaScript, але вам не потрібно розуміти JavaScript-код. Ви вивчите JavaScript у майбутніх модулях.

:::interactive_editor

```html
<button type="button">Show Alert</button>
<script src="index.js"></script>
```

```js
const btn = document.querySelector("button");
btn.addEventListener("click", () => alert("You clicked on the alert button"));
```

:::

Іншим можливим значенням атрибута `type` є значення `submit`. Ось приклад використання елемента `button` з типом `submit`.

:::interactive_editor

```html
<form action="">
  <label for="email">Email address:</label>
  <input type="email" id="email" name="email" />
  <button type="submit">Submit form</button>
</form>
```

:::

Всередині цього елемента `form` є елементи `label` і `input` для введення е-пошти користувача. Коли користувач натискає кнопку відправки, його дані будуть надіслані на сервер і оброблені. Третім можливим значенням атрибута `type` є значення `reset`. Ось приклад елемента `form` з кнопками скидання та відправки.

Увімкніть інтерактивний редактор і взаємодійте з полем введення е-пошти у вікні попереднього перегляду, ввівши фальшиву адресу е-пошти. Потім натисніть кнопку скидання, щоб побачити, як ваша е-пошта зникає з поля.

:::interactive_editor

```html
<form action="">
  <label for="email">Email address:</label>
  <input type="email" id="email" name="email" />
  <button type="reset">Reset form</button>
  <button type="submit">Submit form</button>
</form>
```

:::

У цьому зміненому прикладі використовуються елементи `label` і `input` для збору адреси е-пошти користувача. Коли користувач натискає кнопку скидання, усі його введені дані будуть очищені. Важливо зазначити, що кнопки скидання зазвичай не найкраща ідея, бо вони можуть призвести до випадкового скидання даних користувачем. Також кілька кнопок у вашій формі можуть захарастити інтерфейс користувача.

Ще один спосіб створити кнопки в HTML — це використовувати елемент `input`. Елемент `input` також має атрибут `type` з можливими значеннями `submit`, `reset` і `button`. Ось приклад використання елемента `input` з множиною `type`, встановленою в `button`:

:::interactive_editor

```html
<input class="start-btn" type="button" value="Start Game" />
<script src="index.js"></script>
```

```js
document.addEventListener("DOMContentLoaded", () => {
  const btn = document.querySelector(".start-btn");
  btn.addEventListener("click", () => {
    const paraEl = document.createElement("p");
    const bodyEl = document.querySelector("body");

    bodyEl.appendChild(paraEl);
    paraEl.textContent = "The game has started!!!";
  });
});
```

:::

Атрибут `value` використовується для відображення тексту кнопки. Отже, у чому різниця між використанням елементів `input` і `button`? Елементи `input` — це порожні елементи, тобто вони не можуть мати дочірніх вузлів, таких як текст, і можуть мати лише початковий тег. З іншого боку, елемент `button` пропонує більше гнучкості, бо ви можете вкладати всередину нього текст, зображення та іконки.
