# --interactive--

الكلمة المفتاحية `var` في JavaScript هي واحدة من الطرق الأصلية لإعلان المتغيرات. كانت جزءًا من اللغة منذ نشأتها ولعدة سنوات ظلت الطريقة الأساسية لإنشاء المتغيرات. لكن مع تطور JavaScript واكتساب المطورين خبرة أكبر في اللغة، ظهرت بعض العيوب في استخدام `var` مما أدى إلى إدخال `let` و`const` في عام 2015.

عندما تعلن عن متغير باستخدام `var`، يصبح نطاقه إما نطاق دالة أو نطاق عام. هذا يعني أنه إذا أعلنت عن متغير داخل دالة باستخدام `var`، يكون الوصول إليه محصورًا داخل تلك الدالة فقط. أما إذا أعلنته خارج أي دالة، فيصبح متغيرًا عامًا يمكن الوصول إليه في جميع أنحاء الشفرة الخاصة بك. هذا السلوك قد يؤدي أحيانًا إلى نتائج غير متوقعة ويجعل شفرتك أصعب في الفهم.

مشكلة `var` هي أنه يسمح لك بإعادة إعلان نفس المتغير عدة مرات دون إصدار خطأ. هذا قد يؤدي إلى الكتابة فوق المتغير عن طريق الخطأ ويجعل تصحيح الأخطاء أكثر صعوبة.

:::interactive_editor

```js
var num = 5;
console.log(num); // 5

// This is allowed and doesn't throw an error
var num = 10;
console.log(num); // 10
```

:::

أكبر مشكلة في `var` هي افتقاره إلى نطاق الكتلة. المتغيرات المعلنة بـ `var` داخل كتلة مثل جملة `if` أو حلقة `for` تظل قابلة للوصول خارج تلك الكتلة.

:::interactive_editor

```js
if (true) {
  var num = 5;
}
console.log(num); // 5
```

:::

هذا السلوك قد يؤدي إلى تسرب متغيرات غير مقصود ويجعل شفرتك أكثر عرضة للأخطاء البرمجية.

بسبب هذه المشاكل، تحولت عملية تطوير JavaScript الحديثة إلى الابتعاد عن `var` لصالح `let` و`const`. هذه الكلمات المفتاحية توفر نطاق كتلة يتوافق بشكل أفضل مع كيفية عمل النطاق في العديد من لغات البرمجة الأخرى.

كما أنها لا تسمح بإعادة الإعلان داخل نفس النطاق، مما يساعد على منع التجاوزات العرضية.

بينما لا تزال `var` جزءًا من JavaScript وتعمل في جميع المتصفحات، يُنصح عمومًا باستخدام `let` و`const` في تطوير JavaScript الحديث. فهي توفر قواعد نطاق واضحة، وتساعد على تجنب الأخطاء الشائعة، وتجعل سلوك شفرتك أكثر قابلية للتنبؤ.
