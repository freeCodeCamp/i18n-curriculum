# --interactive--

ウェブフォームを作成するとき、入力要素の見た目や動作は重要です。ユーザーの操作に反応する入力があるフォームはユーザー体験を大きく向上させ、混乱やフラストレーションを引き起こしません。

CSSは、フォームをより直感的で使いやすくするためのいくつかの入力疑似クラスを提供しています。これらの疑似クラスを詳しく見ていきましょう。

`:focus` 疑似クラスは、ユーザーが入力要素を選択またはクリックしたときに対象にでき、アクティブな入力フィールドに注目を集めます。これにより、ユーザーは現在どこに入力しているかを簡単に識別できます。

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css" />
<form>
  <input type="text" />
</form>
```

```css
input:focus {
  border: 2px solid crimson;
  outline: none;
}
```

:::

名前の通り、`:hover` 疑似クラスはユーザーが要素にカーソルを置いたときに発動します。ユーザーが入力とやり取りする前に視覚的なフィードバックを提供し、入力が操作可能であることを効果的に知らせます。

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css" />
<form>
  <input type="text" />
</form>
```

```css
input:hover {
  background-color: orange;
}
```

:::

上の例では、ユーザーが入力要素にカーソルを置くと背景色がオレンジ色に変わり、そのフィールドが操作可能であることを知らせています。

`:checked` 疑似クラスは、ユーザーがラジオボタンやチェックボックスを選択したときに異なるスタイルを適用できます。これにより、ユーザーはどのオプションを選んだかを簡単に確認できます。

以下は、ウェブサイトの利用規約に同意するチェックボックスの例です。

**注意**：この例のCSSにはまだ学習していないプロパティが含まれています。これはカスタムチェックボックスの作り方のイメージをつかむためのもので、今後のレッスンやワークショップで詳しく学びます。

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css" />
<form>
  <label>
  Agree <input class="checkbox" type="checkbox" />
  </label>
</form>
```

```css
.checkbox {
  appearance: none;
  width: 18px;
  height: 18px;
  border: 2px solid #ccc;
  border-radius: 4px;
  display: inline-block;
  position: relative;
  cursor: pointer;
  transition: all 0.25s ease;
  vertical-align: middle; 
}

.checkbox:hover {
  border-color: #888;
}

.checkbox:checked {
  background-color: #4caf50;
  border-color: #4caf50;
}

.checkbox:checked::after {
  content: "";
  position: absolute;
  left: 4px;
  top: 0px;
  width: 5px;
  height: 10px;
  border: solid white;
  border-width: 0 2px 2px 0;
  transform: rotate(45deg);
}

.checkbox:focus {
  outline: 2px solid #90caf9;
  outline-offset: 2px;
}

```

:::

`:required` は `required` 属性を持つ入力要素を対象にします。これは、フォームを提出するためにそのフィールドを必ず入力しなければならないことをユーザーに示します。

以下は必須入力フィールドがあるフォームの例です。

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css" />
<form action="#">
  <label for="name">Name:</label>
  <input type="text" id="name" name="name" required />

  <label for="email">Email:</label>
  <input type="email" id="email" name="email" required />

  <label for="phone">Phone Number:</label>
  <input type="tel" id="phone" name="phone" />

  <button type="submit">Submit</button>
</form>
```

```css
input:required {
  border: 2px solid orange;
}
```

:::

フォームのバリデーション時には、`:valid` 疑似クラスを使ってバリデーション条件を満たす入力フィールドにスタイルを適用し、`:invalid` 疑似クラスで条件を満たさない入力フィールドにスタイルを適用できます。通常、有効な入力には緑色、無効な入力には赤色を使います。

以下は `:valid` 疑似クラスを使った例です。

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css" />
<form>
  <label for="email">Email:</label>
  <input type="email" id="email" name="email" />
</form> 
```

```css
input:valid {
  border-color: green;
}
```

:::

`:invalid` 疑似クラスを使った例はこちらです。

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css" />
<form>  
  <label for="email">Email:</label>
  <input type="email" id="email" name="email" />
</form>
```

```css
input:invalid {
  border-color: crimson;
}
```

:::

`:disabled` 疑似クラスは無効化された入力要素を選択してスタイルを適用できます。これらの要素は `disabled` 属性を持ち、ユーザーが操作できないようにします。無効化された入力はクリック、フォーカス、編集ができません。

以下はラベルと無効化された入力要素の例です。

CSSでは、その無効化された入力を対象に背景色を `lightgray` にし、カーソルを `not-allowed` に変更しています。

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css" />
<form>
  <label for="name">Name:</label>
  <input class="text-input" type="text" id="name" name="name" disabled />
</form>
```

```css
.text-input:disabled {
  background-color: lightgray;
  cursor: not-allowed;
}
```

:::

`cursor: not-allowed;` CSSプロパティ値は、操作が許可されていないことを示すためにカーソルの見た目を変えます。

このプロパティを要素に適用すると、通常は斜線の入った円のシンボルが付いたカーソルが表示され、ユーザーにその要素が操作不可またはクリック不可であることを知らせます。

他の入力疑似クラスとその機能の例は以下の通りです：

- `:autofill`：ブラウザが自動的に保存データで入力フィールドを埋めた場合にスタイルを適用します。
- `:optional`：必須でなく空のままにできる入力要素にスタイルを適用します。
- `:in-range` と `:out-of-range`：値が指定された範囲内か範囲外かに基づいて要素にスタイルを適用します。
