# --description--

Wir haben schon ein wenig über Git und GitHub gesprochen, jetzt können wir alles zusammenfügen. Dafür brauchen Sie ein Repository. Aber was ist ein Repository? Sie können sich ein Repository als Container für ein Projekt vorstellen – wenn Sie an einer App arbeiten, würden Sie die Dateien für diese App in einem Repository zusammenhalten.

Repositories können lokal auf Ihrem Computer oder remote auf einem Dienst wie GitHub liegen. Um ein Repository zu erstellen, können Sie entweder lokal oder remote starten – das Ergebnis ist dasselbe, wählen Sie also die Methode, die am besten zu Ihrem Arbeitsablauf passt. Für einen Arbeitsablauf, bei dem Sie mit einem Remote-Repository starten, können Sie eines direkt über die GitHub-Website erstellen, indem Sie diesen Anweisungen folgen:

Besuchen Sie <a href="https://github.com" target="_blank">die GitHub-Website</a>, stellen Sie sicher, dass Sie eingeloggt sind, und klicken Sie dann auf das Plus-Symbol oben rechts in der Navigationsleiste. Sie sehen einige Optionen, aber die, die Sie interessiert, ist „New Repository“. Wählen Sie diese aus, um die Benutzeroberfläche zum Erstellen eines neuen Repositorys zu öffnen.

Die erste Option, die Sie dort sehen, ist die Möglichkeit, eine Vorlage auszuwählen. Wahrscheinlich haben Sie noch keine eingerichtet, aber eine Vorlage ist eine spezielle Art von Repository, die Sie als Ausgangspunkt für Ihre neuen Repositories verwenden können. Diese sind hilfreich, um Dokumentationen oder Issue-Vorlagen in all Ihren Projekten einzuschließen.

Als Nächstes können Sie einen Besitzer auswählen, entweder Ihr Benutzerkonto oder eine Organisation, die Sie besitzen oder auf die Sie Zugriff haben. Dann können Sie das Repository benennen – im Allgemeinen sollten Sie einen Namen wählen, der das Projekt, das Sie erstellen, prägnant beschreibt. Als bewährte Vorgehensweise sollten Sie Leerzeichen im Repository-Namen vermeiden. GitHub ersetzt diese automatisch durch Bindestriche.

Optional können Sie eine Beschreibung hinzufügen, die auf der Startseite des Repositorys angezeigt wird.

Sie können auch wählen, ob das Repository öffentlich oder privat sein soll. Ein öffentliches Repository kann von jedem angesehen und heruntergeladen werden. Ein privates Repository kann nur von Ihnen und den Personen, denen Sie explizit Zugriff gewähren, aufgerufen werden.

Wenn Sie keine Vorlage auswählen, sehen Sie Optionen, um Dateien im Repository automatisch zu generieren. Eine README-Datei erscheint auf der Startseite des Repositorys und ist eine großartige Möglichkeit, Dokumentation und Beschreibungen Ihres Projekts bereitzustellen. Eine `.gitignore`-Datei erlaubt es Ihnen, Dateien anzugeben, die Git „ignorieren“ soll – das ist ideal für Dinge wie Paketabhängigkeiten, Umgebungsdateien und andere Dinge, die nicht gespeichert werden sollen. GitHub bietet Vorlagen für diese Datei an, die die gängigen Elemente für die angegebene Sprache oder Laufzeit ausschließen. Und eine Lizenz ist ein rechtliches Dokument, das festlegt, wer Ihre Software nutzen darf und was damit gemacht werden darf.

Wenn Sie alle Einstellungen nach Ihren Wünschen konfiguriert haben, können Sie auf die Schaltfläche „Create Repository“ klicken, um den Vorgang abzuschließen. Sie werden automatisch zur Zielseite Ihres neuen Repositorys weitergeleitet!

Von hier aus können Sie auf die Schaltfläche „Code“ klicken, um Optionen zu erhalten, Ihr Repository auf Ihren Computer zu klonen.

Die Registerkarten „Codespaces“ und „Copilot“ können Sie für diese Lektion ignorieren. Stattdessen konzentrieren wir uns auf die drei Optionen im Tab „Local“. Die ersten beiden, HTTPS und SSH, werden auf dieselbe Weise verwendet. Sie verwenden den `git clone`-Befehl mit der im Feld angegebenen URL, um das Remote-Repository auf Ihren Computer zu klonen. Der Unterschied zwischen diesen beiden liegt in den Verbindungs- und Authentifizierungsmethoden.

Zum Klonen eines öffentlichen Repositorys müssen Sie sich tatsächlich nicht authentifizieren. Aber um ein privates Repository zu klonen oder um auf öffentliche oder private Repositorys zu pushen, müssen Sie authentifiziert sein. Eine HTTPS-Remote-URL authentifiziert Sie über Ihren GitHub-Benutzernamen und Ihr Passwort. Aufgrund von Sicherheitsbedenken wird dies jedoch nicht mehr unterstützt, und Sie müssen stattdessen einen Benutzernamen und einen Zugriffstoken verwenden. Selbst dann ist es eine viel weniger sichere Methode als SSH, das Ihre privaten und öffentlichen Schlüssel verwendet. Das hat den zusätzlichen Vorteil, dass Sie automatisch angemeldet sind, ohne jedes Mal einen Token eingeben zu müssen. Es wird eine separate Lektion geben, die tiefer in SSH und seine Funktionsweise eintaucht, aber Sie können auch die GitHub-Dokumentation zum Einrichten von SSH lesen.

Um das Repository zu klonen, können Sie dies im Terminal ausführen:

```sh
git clone git@github.com:your-username/your-repository-name
```

Es wird ein Ordner auf Ihrem Computer mit dem Namen Ihres Repositorys erstellt, der alle Dateien Ihres Repositorys enthält. Von dort aus können Sie mit `cd` (Verzeichnis wechseln) in Ihr geklontes Repository wechseln mit:

```sh
cd name-of-cloned-repo
```

Außerdem hat die lokale Kopie Ihres Repositories eine Remote-Verbindung, die auf die Kopie auf GitHub zeigt. Um Ihre Remotes zu überprüfen, können Sie den folgenden Befehl ausführen:

```sh
git remote -v
```

Eine weitere Möglichkeit, Ihr Repository zu klonen, ist die Verwendung der GitHub CLI. Dieses Tool wird verwendet, um GitHub-spezifische Aufgaben auszuführen, ohne die Befehlszeile zu verlassen. Wenn Sie es nicht installiert haben, finden Sie Anweisungen dazu in der GitHub-Dokumentation – aber es sollte in Ihrem Paketmanager verfügbar sein.

Um ein Repository mit dem CLI-Tool zu klonen, verwenden Sie:

```sh
gh repo clone your-username/your-repository-name
```

Das Endergebnis dieser Befehle ist dasselbe: Sie haben eine lokale Kopie Ihres Repositorys.

Sie können die GitHub CLI auch verwenden, um ein Repository zu erstellen:

```sh
gh repo create
```

Wenn Sie keine Argumente übergeben, bietet Ihnen die CLI einen interaktiven Assistenten zur Einrichtung Ihres Repositorys. Dieser Assistent bietet Ihnen alle Optionen der Web-Benutzeroberfläche. Mit diesem Tool haben Sie auch die Möglichkeit, das neue Repository sofort zu klonen, sodass Sie die manuellen Schritte, die zuvor beschrieben wurden, nicht durchlaufen müssen.

Sie haben jetzt ein grundlegendes Verständnis davon, wie Repositories funktionieren und wie Sie eines erstellen können, um Ihre Arbeit richtig zu versionieren. In zukünftigen Lektionen lernen Sie, wie Sie ein bestehendes lokales Repository zu GitHub pushen und wie Sie Änderungen an Remote-Repositories pushen.
