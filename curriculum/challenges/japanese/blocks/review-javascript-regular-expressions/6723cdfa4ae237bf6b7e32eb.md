# --interactive--

## 正規表現と一般的なメソッド

- **定義**: 正規表現（Regex）は「パターン」を作成するために使われ、そのパターンを使って文字列をチェックしたり、テキストを抽出したりできます。

```js
const regex = /freeCodeCamp/;
```

- **`test()` メソッド**: このメソッドは文字列を受け取り、その文字列が正規表現にマッチするかどうかを判定します。マッチすればブール値を返します。

:::interactive_editor

```js
const regex = /freeCodeCamp/;
const test = regex.test("e");
console.log(test); // false
```

:::

- **`match()` メソッド**: このメソッドは正規表現を受け取りますが、文字列を渡すと正規表現に変換されます。`match` メソッドは文字列のマッチ配列を返します。

```js
const regex = /freeCodeCamp/;
const match = "freeCodeCamp".match(regex);
console.log(match); // ['freeCodeCamp', index: 0, input: 'freeCodeCamp', groups: undefined]
```

- **`replace()` メソッド**: このメソッドは2つの引数を受け取ります。1つ目はマッチさせる正規表現（または文字列）、2つ目はマッチした部分を置き換える文字列（または各マッチに対して実行する関数）です。

:::interactive_editor

```js
const regex = /Jessica/;
const str = "Jessica is rly kewl";
const replaced = str.replace(regex, "freeCodeCamp");
console.log(replaced); // "freeCodeCamp is rly kewl"
```

:::

- **`replaceAll` メソッド**: このメソッドは指定したパターンのすべての出現箇所を新しい文字列に置き換えます。グローバル修飾子なしの正規表現を渡すとエラーが発生します。

:::interactive_editor

```js
const text = "I hate JavaScript! I hate programming!";
const newText = text.replaceAll("hate", "love");
console.log(newText);  // "I love JavaScript! I love programming!"
```

:::

- **`matchAll` メソッド**: このメソッドは文字列内の正規表現にマッチするすべての結果（キャプチャグループを含む）を取得し、イテレータとして返します。イテレータはコレクションの要素を順に処理できるオブジェクトです。

:::interactive_editor

```js
const str = "JavaScript, Python, JavaScript, Swift, JavaScript";
const regex = /JavaScript/g;

const iterator = str.matchAll(regex);

for (let match of iterator) {
  console.log(match[0]); // "JavaScript" for each match
}
```

:::

## 正規表現の修飾子

- **定義**: 修飾子（フラグとも呼ばれます）は正規表現の動作を変更します。
- **`i` フラグ**: このフラグは正規表現を大文字小文字を区別しないようにします。

:::interactive_editor

```js
const regex = /freeCodeCamp/i;
console.log(regex.test("freecodecamp")); // true
console.log(regex.test("FREECODECAMP")); // true
```

:::

- **`g` フラグ**: このフラグ（グローバル修飾子）は正規表現がパターンに複数回マッチすることを可能にします。

:::interactive_editor

```js
const regex = /freeCodeCamp/gi;
console.log(regex.test("freeCodeCamp")); // true
console.log(regex.test("freeCodeCamp is great")); // false
```

:::

- **アンカーの定義**: `^` アンカーは正規表現の先頭に置き、「文字列の先頭にマッチする」ことを示します。`$` アンカーは正規表現の末尾に置き、「文字列の末尾にマッチする」ことを示します。

:::interactive_editor

```js
const start = /^freeCodeCamp/i;
const end = /freeCodeCamp$/i;
console.log(start.test("freecodecamp")); // true
console.log(end.test("freecodecamp")); // true
```

:::

- **`m` フラグ**: アンカーは文字列全体の先頭と末尾を探しますが、`m` フラグ（マルチライン修飾子）を使うと複数行に対応できます。

:::interactive_editor

```js
const start = /^freecodecamp/im;
const end = /freecodecamp$/im;
const str = `I love 
freecodecamp
it's my favorite
`;
console.log(start.test(str)); // true
console.log(end.test(str)); // true
```

:::

- **`d` フラグ**: このフラグはマッチオブジェクトで得られる情報を拡張します。

```js
const regex = /freecodecamp/di;
const string = "we love freecodecamp isn't freecodecamp great?";
console.log(string.match(regex));
```

結果は以下の通りです：

```js
// [
//   'freecodecamp',
//   index: 8,
//   input: "we love freecodecamp isn't freecodecamp great?",
//   groups: undefined,
//   indices: [
//     0: [8, 20],
//     groups: undefined
//   ]
// ]
```

- **`u` フラグ**: このフラグは正規表現の機能を拡張し、特殊なユニコード文字にマッチできるようにします。`u` フラグは`Extended_Pictographic` のような特殊クラスにアクセスでき、ほとんどの絵文字にマッチします。さらに機能を拡張する`v` フラグもあります。
- **`y` フラグ**: スティッキーフラグはグローバルフラグに似ていますが、いくつか例外があります。最大の違いは、グローバル正規表現はlastIndexから文字列の残り全体を検索しますが、スティッキー正規表現は直前のlastIndexで即座にマッチしなければnullを返し、lastIndexを0にリセットします。
- **`s` フラグ**: シングルライン修飾子は、正規表現のワイルドカード文字（`.`）が改行にもマッチできるようにし、文字列を単一行として扱います。

## 文字クラス

- **ワイルドカード `.`**: 文字クラスは文字の集合や部分集合にマッチさせるための特別な構文です。最初に覚えるべき文字クラスはワイルドカードクラスです。ワイルドカードはピリオド（ドット）で表され、改行を除く任意の1文字にマッチします。改行にもマッチさせるには`s` フラグを使う必要があります。

```js
const regex = /a./;
```

- **`\d`**: これは文字列内のすべての数字（`0-9`）にマッチします。

```js
const regex = /\d/;
```

- **`\w`**: これは文字列内の任意の単語文字（`a-zA-Z0-9_`）にマッチするために使います。単語文字とは、aからzまたはAからZの任意の文字、0から9の数字、またはアンダースコア文字のことです。

```js
const regex = /\w/;
```

- **`\s`**: 空白文字クラス`\s`はバックスラッシュに続く`s`で表されます。この文字クラスは改行、スペース、タブ、特殊なユニコード空白文字を含む任意の空白にマッチします。
- **特殊文字クラスの否定**: これらの文字クラスを否定するには、バックスラッシュの後に小文字ではなく大文字を使います。以下の例は数字文字にはマッチせず、数字以外の任意の1文字にマッチします。

```js
const regex = /\D/;
```

- **カスタム文字クラス**: マッチさせたい文字を大括弧で囲むことでカスタム文字クラスを作成できます。

```js
const regex = /[abcdf]/;
```

## 先読みと後読みアサーション

- **定義**: 先読みと後読みアサーションは、周囲のパターンの有無に基づいて特定のパターンにマッチさせることを可能にします。
- **肯定先読みアサーション**: このアサーションは、あるパターンの後に別のパターンが続く場合にマッチします。肯定先読みを作るには、まずマッチさせたいパターンを書き、その後に条件として使いたいパターンを丸括弧で囲み、開き括弧の直後に`?=`を使います。

```js
const regex = /free(?=code)/i;
```

- **否定先読みアサーション**: これは正規表現で、特定のパターンが文字列の先に存在しないことを確認する条件です。

```js
const regex = /free(?!code)/i;
```

- **肯定後読みアサーション**: このアサーションは、特定のパターンの直前に別のパターンがある場合にのみマッチし、直前のパターンはマッチ結果に含めません。

```js
const regex = /(?<=free)code/i;
```

- **否定後読みアサーション**: このアサーションは、特定のパターンの直前に別のパターンがないことを保証します。指定されたパターンが直前にない場合にのみマッチし、直前のパターンはマッチ結果に含めません。

```js
const regex = /(?<!free)code/i;
```

## 正規表現の量指定子

- **定義**: 量指定子は、パターン（またはパターンの一部）が何回現れるかを指定します。マッチ内の文字やグループの出現回数を制御します。以下の例は直前の文字をちょうど4回マッチさせます。

```js
const regex = /^\d{4}$/;
```

- **`*`** : 直前の要素が0回以上繰り返されることにマッチします。
- **`+`**: 直前の要素が1回以上繰り返されることにマッチします。
- **`?`**: 直前の要素が0回または1回現れることにマッチします。
- **`{n}`**: 直前の要素がちょうどn回現れることにマッチします。
- **`{n,}`**: 直前の要素がn回以上現れることにマッチします。
- **`{n,m}`**: 直前の要素がn回からm回の間で現れることにマッチします。

## キャプチャグループとバックリファレンス

- **キャプチャグループ**: キャプチャグループはマッチした文字列の一部を「キャプチャ」し、必要に応じて利用できるようにします。キャプチャグループは丸括弧で囲まれたパターンで定義され、先読みのような先頭文字はありません。

```js
const regex = /free(code)camp/i;
```

- **非キャプチャグループ**: 非キャプチャグループはキャプチャグループに似ていますが、マッチした部分を保存しません。非キャプチャグループは `(?:...)` で定義されます。

```js
const regex = /free(?:code)camp/i;
```

- **バックリファレンス**: 正規表現のバックリファレンスは、同じ式内で以前にマッチした部分を再利用する方法です。キャプチャグループ（丸括弧内のパターン）を番号で参照できます。例えば、`$1` は最初のキャプチャグループを指します。

:::interactive_editor

```js
const regex = /free(co+de)camp/i;
console.log("freecoooooooodecamp".replace(regex, "paid$1world"));
```

:::

- バックリファレンスは正規表現内で、バックスラッシュとキャプチャグループ番号を使って、以前にキャプチャしたテキストと同じものにマッチさせることができます。例えば：

:::interactive_editor

```js
const regex = /(hello) \1/i;
console.log(regex.test("hello hello")); // true
console.log(regex.test("hello world")); // false
```

:::
