# --description--

디지털 세계는 공격자가 데이터를 훔치고, 시스템을 방해하며, 의심 없는 사용자와 플랫폼에서 금전을 갈취하기 위해 약점을 계속 찾으면서 끊임없는 보안 위협에 직면하고 있습니다.

이러한 보안 위협 몇 가지와 이를 완화하는 방법을 살펴보겠습니다.

**피싱**은 가장 흔한 보안 위협 중 하나입니다. 피싱 공격에서는 범죄자가 신뢰받는 기업을 가장한 사기성 이메일을 보내 사용자가 악성 링크를 클릭하거나 계정 비밀번호 및 기타 민감한 정보를 공유하도록 속입니다.

피싱을 줄이기 위해 회사는 다음을 해야 합니다:

- 사용자 대상 정기적인 보안 인식 교육을 실시하세요
- 직원은 의심스러운 발신자 주소와 긴급한 형식을 식별하는 법을 배워야 합니다
- 소프트웨어 팀은 악성 메시지가 사용자 받은편지함에 도달하기 전에 차단할 수 있도록 이메일 인증 프로토콜을 배포해야 합니다

**멀웨어**는 사용자와 개발자에게 또 다른 심각한 위협입니다. 이 범주에는 바이러스, 스파이웨어, 랜섬웨어가 포함됩니다.

악의적인 행위자는 랜섬웨어를 사용해 피해자의 컴퓨터 파일을 암호화한 뒤, 데이터를 잠금 해제하기 위해 큰 금액을 요구합니다.

멀웨어를 방지하려면 소프트웨어 팀이 보통 자동 업데이트를 실행하고 알려진 소프트웨어 취약점을 패치합니다. 보안은 다층적이므로 사용자는 즉시 업데이트를 설치하고 장치에 안티멀웨어 소프트웨어를 설치하는 등 자신의 역할도 해야 합니다.

**자격 증명 채우기 공격**은 사용자 인증 시스템을 목표로 합니다. 공격자는 자동화 도구를 사용해 수백만 개의 도난당한 사용자 이름과 비밀번호 조합을 여러 웹사이트에서 테스트합니다.

이 공격이 성공하는 이유는 많은 사람이 여러 플랫폼에서 동일한 비밀번호를 재사용하기 때문입니다.

회사는 다중 요소 인증을 시행해 이 위협을 완화할 수 있습니다. 다중 요소 인증은 모바일 인증 앱 코드, 일회용 비밀번호(OTP), 이메일 인증 같은 두 번째 검증 단계를 요구합니다.

해커가 비밀번호를 훔쳐도 이 2차 토큰 없이는 계정에 접근할 수 없습니다.

**분산 서비스 거부 공격**은 시스템 가용성을 목표로 합니다. 공격자는 대상 웹사이트에 대량의 가짜 인터넷 트래픽을 쏟아 부어 서버를 갑자기 과부하시키고 다운시키며 정상 고객이 서비스에 접근하지 못하게 막습니다.

이 공격을 방어하려면 기업은 Cloudflare, Akamai, AWS Shield 같은 클라우드 기반 트래픽 필터링 서비스를 사용합니다.

이 완화 플랫폼은 들어오는 트래픽 급증을 분석합니다. 악성 데이터 패킷을 걸러내고 정상 사용자 트래픽은 통과시킵니다.

마지막으로, **내부자 위협**은 회사 내부 인물을 포함합니다. 여기에는 고의로 데이터를 훔치는 불만 직원이나 실수로 정보를 유출하는 부주의한 직원이 포함됩니다.

이 내부 위험에 대응하려면 회사는 최소 권한 원칙을 시행합니다. 이 보안 기준은 직원이 일상 업무를 수행하는 데 필요한 특정 파일에만 접근하도록 보장합니다.

보안 팀은 또한 시스템 로그를 모니터링해 비정상적인 데이터 다운로드나 무단 파일 접근을 감지합니다.
