# --interactive--

在网页上隐藏内容是网页开发中的常见做法，但关键是要做到不影响无障碍访问。 不同的隐藏技术会对辅助技术如何向用户解释和展示内容产生不同的影响。 让我们来探讨一些既能隐藏内容又能保持可访问性的最佳做法。 隐藏内容的常用方法之一是使用 `display: none`。 这是一个示例：

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<p class="hidden">Hidden text</p>
<p>Visible text</p>
```

```css
.hidden {
  display: none;
}
```

:::

虽然这在视觉上有效地隐藏了内容，但也将其从可访问性树中移除。 可访问性树是屏幕阅读器等辅助技术用来解释网页内容并与之交互的结构。 它以辅助技术可以理解并呈现给用户的方式来表示内容及其语义。

使用 `display: none` 意味着屏幕阅读器和其他辅助技术将无法访问此内容，因为它不包含在辅助功能树中。 因此，只有当你想从视觉呈现和可访问性两方面完全删除内容时，才必须使用这种方法。 

另一种隐藏内容的方法是使用 `visibility: hidden`：

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<p class="hidden">Hidden text</p>
<p>Visible text</p>
```

```css
.hidden {
  visibility: hidden;
}
```

:::

`visibility: hidden` 在视觉上隐藏内容，但将其保留在文档流中，这意味着它仍占据页面空间。 和 `display: none` 类似，`visibility: hidden` 也会从辅助功能树中删除内容。 这意味着屏幕阅读器等辅助技术将无法访问隐藏内容。 只有当你想对所有人（包括使用辅助技术的人员）隐藏内容时，才使用 `visibility: hidden`。

对于应在视觉上隐藏但屏幕阅读器仍可访问的内容，可以使用一种通常称为“视觉隐藏”或“仅限屏幕阅读器”的技术。 下面是一个使用 `.sr-only` CSS 类的示例，这是一种常用的技术，用于在视觉上隐藏内容，同时保持屏幕阅读器可以访问这些内容：

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<p class="sr-only">Hidden text</p>
<p>Visible text</p>
```

```css
.sr-only {
  position: absolute;
  width: 1px;
  height: 1px;
  padding: 0;
  margin: -1px;
  overflow: hidden;
  clip: rect(0, 0, 0, 0);
  white-space: nowrap;
  border: 0;
}
```

:::

In this example, we are using properties like `position`, `clip`, and `white-space`, which you will learn about in future lessons. 现在，你只需知道这条 CSS 规则能有效地隐藏视觉内容，同时保持屏幕阅读器的可访问性。 它可以在不影响视觉布局的情况下，为屏幕阅读器用户提供额外的上下文。 

要切换内容的可见性，请考虑使用 `hidden` 属性：

:::interactive_editor

```html
<p hidden>This content is hidden</p>
<p>This content is visible</p>
```

:::

大多数现代浏览器都支持 `hidden` 属性，它既隐藏内容的视觉显示，也从可访问性树中将其移除。 它可以通过 JavaScript 轻松切换。 最后，谨慎隐藏重要内容。 如果信息对理解或使用网站至关重要，则应让所有用户都能看到并访问。 只有当隐藏内容能真正提升用户体验时，才这样做。 通过遵循这些最佳实践，你可以确保所有用户都能访问内容，无论他们如何与网站互动。
