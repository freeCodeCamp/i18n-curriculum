# --explanation--

`Think of it as`는 이미 익숙한 것에 비유하여 개념을 이해하도록 돕는 데 사용됩니다. 개념을 더 쉽게 이해할 수 있도록 상상하게 합니다. 예를 들어:

`Think of a function as a recipe.` - 이것은 함수를 요리법처럼 단계별 지침 집합으로 생각하게 도와줍니다.

`Wrapping code`는 코드를 감싸거나 포장하여 접근을 제어하는 것을 의미합니다. 이렇게 하면 코드에 대한 접근을 `limit`합니다. 예를 들어:

`We're wrapping the database access code inside a function to hide the complexity from the rest of the program.` - 이는 데이터베이스 상호작용이 외부에 숨겨지고 함수가 간단한 인터페이스를 제공한다는 뜻입니다.
