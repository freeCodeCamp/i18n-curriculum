# --interactive--

配列の分割代入は、JavaScriptの機能で、配列から値を取り出して変数により簡潔で読みやすい形で代入することができます。配列の要素を個別の変数に展開する便利な構文を提供します。

この手法は、配列や複数の値を返す関数を扱うときに特に役立ちます。以下は配列の分割代入を使った例です：

:::interactive_editor

```js
let fruits = ["apple", "banana", "orange"];

let [first, second, third] = fruits;

console.log(first);  // "apple"
console.log(second); // "banana"
console.log(third);  // "orange"
```

:::

この例では、`fruits` という3つの要素を持つ配列があります。配列の分割代入を使って、最初の要素を変数`first`に、2番目の要素を`second`に、3番目の要素を`third`に代入しています。これにより、インデックス表記を使わずに配列の個々の要素に簡単にアクセスできます。

配列の分割代入を使わずに、インデックスでそれぞれの要素にアクセスした場合は次のようになります：

:::interactive_editor

```js
const fruits = ["apple", "banana", "orange"];

const first = fruits[0];
const second = fruits[1];
const third = fruits[2];

console.log(first); // "apple"
console.log(second); // "banana"
console.log(third); // "orange"
```

:::

配列の分割代入では、カンマを使って興味のない要素をスキップすることもできます。例えば：

:::interactive_editor

```js
let colors = ["red", "green", "blue", "yellow"];
let [firstColor, , thirdColor] = colors;

console.log(firstColor); // "red"
console.log(thirdColor); // "blue"
```

:::

この例では、`colors` 配列の2番目の要素を余分なカンマでスキップしています。これにより、`red` が`firstColor` に、`blue` が`thirdColor` に代入され、`green` は無視されます。

配列の分割代入のもう一つの強力な機能は、デフォルト値を使えることです。配列の要素数が代入しようとしている変数の数より少ない場合、デフォルト値を指定できます：

:::interactive_editor

```js
let numbers = [1, 2];
let [a, b, c = 3] = numbers;

console.log(a); // 1
console.log(b); // 2
console.log(c); // 3
```

:::

ここでは、`3` 配列に3番目の要素がないため、`c` にデフォルト値`numbers` が代入されます。

次に、3つのドット（`...`）で表されるrest構文について説明します。これは、分割代入されなかった配列の残りの要素を新しい配列としてまとめて取得することができます。使い方は次の通りです：

:::interactive_editor

```js
let fruits = ["apple", "banana", "orange", "mango", "kiwi"];
let [first, second, ...rest] = fruits;

console.log(first);  // "apple"
console.log(second); // "banana"
console.log(rest);   // ["orange", "mango", "kiwi"]
```

:::

この例では、`first` と `second` が `fruits` 配列の最初の2つの要素を取得し、`rest` が残りのすべての要素を新しい配列として取得します。rest構文は分割代入パターンの最後の要素でなければなりません。

配列の分割代入は、コードをより簡潔で読みやすくする強力な機能です。配列を扱うときや、配列から特定の要素を取り出す必要があるときに特に役立ちます。
