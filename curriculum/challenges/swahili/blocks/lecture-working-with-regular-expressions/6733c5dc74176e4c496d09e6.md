# --interactive--

Uthibitisho wa angalizo la mbele na angalizo la nyuma hukuruhusu kulinganisha mifumo maalum kulingana na uwepo au ukosefu wa mifumo inayozunguka. Kuna aina nne za uthibitisho hizi.

Ya kwanza ni uthibitisho chanya wa angalizo la mbele. Uthibitisho huu utalinganisha muundo wakati muundo unafuatiwa na muundo mwingine.

Ili kuunda angalizo la mbele chanya, unahitaji kuanza na muundo unaotaka kulinganisha. Kisha, tumia mabano ya kawaida kufunika muundo unaotaka kutumia kama sharti lako. Baada ya mabano ya kawaida ya kufungua, tumia `?=` kuainisha muundo huo kama angalizo la mbele chanya.

Muundo huu utalinganisha neno `free` tu ikiwa unafuatiwa na `code`:

```js
const regex = /free(?=code)/i;
```

Hebu jaribu tabia ya muundo wetu:

:::interactive_editor

```js
const regex = /free(?=code)/i;
console.log(regex.test("freeCodeCamp")); // true
console.log(regex.test("free code camp")); // false
console.log(
  regex.test("I need someone for free to write code for me")
); // false
```

:::

Angalia jinsi mfuatano wa herufi ambapo `free` unafuatiwa mara moja na `code` unavyopitisha mtihani.

Lakini vipi kama unataka kulinganisha uwepo wa `free` wakati HAUFUATIWI na `code`? Unaweza kubadilisha angalizo lako la mbele chanya kuwa angalizo hasi la mbele ili kubadilisha tabia. Ili kufanya hivyo, badilisha `?=` kuwa `?!`:

```js
const regex = /free(?!code)/i;
```

Hebu jaribu hii dhidi ya mfuatano wetu uleule wa herufi:

:::interactive_editor

```js
const regex = /free(?!code)/i;
console.log(regex.test("freeCodeCamp")); // false
console.log(regex.test("free code camp")); // true
console.log(
  regex.test("I need someone for free to write code for me")
); // true
```

:::

Kama ilivyotarajiwa, matokeo yamegeuzwa. Mfuatano pekee wa herufi unaoshindwa ni ule wa kwanza, ambapo `free` unafuatiwa mara moja na `code`.

Uthibitisho wa angalizo la nyuma hufanya kazi kwa njia sawa na uthibitisho wa angalizo la mbele, isipokuwa badala ya kulinganisha kwa sharti kulingana na muundo unaofuata, hulinganisha kwa sharti kulingana na muundo uliotangulia. Hebu tuchunguze angalizo la nyuma chanya.

Angalizo la nyuma chanya linaonyeshwa kwa `?<=` badala ya `?=`. Hebu tufanye usemi wetu wa kawaida ulinganishe `code` wakati unatanguliwa na `free`:

```js
const regex = /(?<=free)code/i;
```

Kama ilivyo kwa angalizo letu la mbele chanya, angalizo letu la nyuma chanya linalinganisha mfuatano wa kwanza kwa sababu `code` unatanguliwa mara moja na `free`:

:::interactive_editor

```js
const regex = /(?<=free)code/i;
console.log(regex.test("freeCodeCamp")); // true
console.log(regex.test("free code camp")); // false
console.log(
  regex.test("I need someone for free to write code for me")
); // false
```

:::

Ili kulinganisha `code` wakati HAUTANGULIWI na `free`, tunaweza kutumia angalizo hasi la nyuma. Angalizo hasi la nyuma linafafanuliwa kwa kubadilisha `?<=` na `?<!`:

```js
const regex = /(?<!free)code/i;
```

Hii italinganisha tukio lolote la `code` ambalo HALITANGULIWI mara moja na `free`:

:::interactive_editor

```js
const regex = /(?<!free)code/i;
console.log(regex.test("freeCodeCamp")); // false
console.log(regex.test("free code camp")); // true
console.log(
  regex.test("I need someone for free to write code for me")
); // true
```

:::

Kumbuka kuwa `RegExp.prototype.test` inathibitisha tu kama mfuatano wa herufi unalingana na usemi wa kawaida. Tuchukue negative lookbehind yetu tukitumia `String.prototype.match` kuona jinsi uthibitisho unavyoathiri hilo:

```js
const regex = /(?<!free)code/i;
console.log("freeCodeCamp".match(regex)); // null
console.log("free code camp".match(regex)); // ['code', index: 5, input: 'free code camp', groups: undefined]
console.log(
  "I need someone for free to write code for me".match(regex)
); // ['code', index: 33, input: 'I need someone for free to write code for me', groups: undefined]
```

Angalia jinsi ingawa usemi wetu wa kawaida unatumia angalizo la nyuma kuangalia uwepo wa `free`, haulingani na `free`. Maandishi pekee yaliyomo katika kulinganisha ni `code`.

Angalizo la mbele na angalizo la nyuma ni muhimu sana kwa kulinganisha maandishi kwa sharti bila kuathiri thamani inayorejeshwa ya kulinganisha kwako.
