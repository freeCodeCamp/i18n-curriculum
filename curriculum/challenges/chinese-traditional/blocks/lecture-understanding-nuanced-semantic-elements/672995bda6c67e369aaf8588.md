# --interactive--

定義列表非常適合以有組織且易於閱讀的格式呈現字詞和定義，就像在詞彙表或真正的字典中，你可以找到字詞及其對應的定義。

這是一個 HTML 中描述性列表的範例，包含兩個術語及其對應的詳細資訊。

啟用互動式編輯器，並嘗試取消註解程式碼，以查看新的詳細條款出現在預覽視窗中。

:::interactive_editor

```html
<dl>
  <dt>HTML</dt>
  <dd>HyperText Markup Language</dd>
  <dt>CSS</dt>
  <dd>Cascading Style Sheets</dd>
  <!-- <dt>JS</dt>
  <dd>JavaScript</dd> -->
</dl>
```

:::

在這種情況下，術語是縮寫 `HTML` 和 `CSS`，而細節是它們的完整寫法。細節也可以是定義，或與術語相關的其他資訊。

你將需要三個 HTML 元素來定義描述列表。首先是描述列表元素 `dl`，它是整個列表的容器。你可以在範例中看到它包裹著描述列表的所有其他元素。

然後，為每個術語使用一個描述術語元素 `dt`。在此情況下，描述列表有兩個術語，HTML 和 CSS，因此它有兩個這樣的元素。

最後，在每個術語之後你會找到一個描述細節元素 `dd`，用於該術語的描述或相關細節。在此範例中，它們是超文字標記語言和階層式樣式表。

在瀏覽器中，你會看到每個術語後面跟著其對應的描述。預設情況下，描述會稍微向右縮排，以便視覺上區分。

但描述列表不限於只有術語和定義。它們的用途遠比這更廣泛。這裡有一個包含兩種食材的食譜。

啟用互動式編輯器，並嘗試取消註解程式碼，以查看新的詳細條款出現在預覽視窗中。

:::interactive_editor

```html
<dl>
  <dt>Flour</dt>
  <dd>2 cups</dd>
  <dt>Sugar</dt>
  <dd>1/2 cup</dd>
  <!-- <dt>Vegetable Oil</dt>
  <dd>2 tablespoons</dd> -->
</dl>
```

:::

整個描述列表位於描述列表元素中。第一個成分 `Flour` 位於描述項目元素中。接著，你可以看到你需要多少這個成分：`2 cups`。這位於其對應成分後方的描述細節元素中。

相同的結構也會重複用於 `Sugar`。在這種情況下，配方只有兩種成分，但如果有更多，則相同的結構可以在描述列表中重複出現。

在瀏覽器中，你會看到材料靠左對齊，而份量則縮排以視覺上區分它們。

描述列表的其他使用案例包括產品規格、常見問題、聯絡資訊和詮釋資料。基本上，當你有兩個以鍵值配對格式相關的資訊，其中一個作為標籤、鍵，另一個作為額外相關資訊、值時，你可以使用描述列表。
