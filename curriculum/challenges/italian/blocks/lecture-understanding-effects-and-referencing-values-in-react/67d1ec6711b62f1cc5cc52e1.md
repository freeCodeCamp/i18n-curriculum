# --description--

In React, un effetto è tutto ciò che accade al di fuori del processo di rendering del componente. Cioè, tutto ciò che React non gestisce direttamente come parte del rendering dell’interfaccia utente.

Esempi comuni includono il recupero di dati, l’aggiornamento del titolo della scheda del browser, la lettura o la scrittura nella memoria locale del browser, l’ottenimento della posizione dell’utente e molto altro. Queste operazioni interagiscono con il mondo esterno e sono note come effetti collaterali.

React fornisce il hook `useEffect` per permetterti di gestire questi effetti collaterali. `useEffect` ti consente di eseguire una funzione dopo che il componente è stato renderizzato o aggiornato.

Vediamo come funziona il hook `useEffect` e perché è essenziale per lo sviluppo moderno con React.

Per usare il hook `useEffect`, devi prima importarlo:

```js
import { useEffect } from "react";
```

Poi lo usi come una funzione, così:

```js
useEffect(() => {
  // Your side effect logic (usually a function) here
}, [dependencies]);
```

La funzione effetto viene eseguita dopo il rendering del componente, mentre l’argomento opzionale `dependencies` controlla quando l’effetto viene eseguito.

Nota che `dependencies` può essere un array di “valori reattivi” (stato, props, funzioni, variabili e così via), un array vuoto o essere completamente omesso. Ecco come tutte queste opzioni controllano il funzionamento di `useEffect`:

- Se `dependencies` è un array che include uno o più valori reattivi, l’effetto verrà eseguito ogni volta che cambiano.

- Se `dependencies` è un array vuoto, `useEffect` viene eseguito solo una volta, quando il componente viene renderizzato per la prima volta.

- Se ometti `dependencies`, l’effetto viene eseguito ogni volta che il componente viene renderizzato o aggiornato.

Per esempio, in questa applicazione `Counter`, non passiamo un argomento `dependencies`, quindi l’effetto viene eseguito quando il componente viene renderizzato e ogni volta che si aggiorna:

```jsx
import { useState, useEffect } from "react";

const Counter = () => {
  const [count, setCount] = useState(0);

  useEffect(() => {
    console.log("Component renders");
  });

  return (
    <div
      style={{
        display: "flex",
        alignItems: "center",
        flexDirection: "column",
      }}
    >
      <h2>{count}</h2>
      <div>
        <button onClick={() => setCount(count + 1)}>Increase</button>
        <button onClick={() => setCount(count - 1)}>Decrease</button>
      </div>
    </div>
  );
};

export default Counter;
```

Ma se passiamo un array vuoto come dipendenza, l’effetto viene eseguito solo al primo rendering:

```js
useEffect(() => {
  console.log('Component renders');
}, []);
```

Se passi lo stato `count` come dipendenza, l’effetto viene eseguito quando il componente viene renderizzato per la prima volta e quando `count` cambia:

```js
useEffect(() => {
  document.title = `The current count is ${count}`;
  console.log('component renders');
}, [count]);
```

Nota che, se l’effetto che imposti persiste oltre il ciclo di vita del rendering del componente, potresti aver bisogno di un’altra funzione per “pulire” quell’effetto dopo che il componente è stato renderizzato o aggiornato.

Per esempio, se la tua funzione effetto usa `setInterval()`, imposta un listener per eventi come `window.addEventListener()` o si connette a un server, avrai bisogno di una funzione di pulizia per rimuovere `clearInterval()`, `window.removeEventListener()` e disconnetterti dal server, rispettivamente.

Ecco la sintassi per restituire una funzione di pulizia dal hook `useEffect`:

```js
useEffect(() => {
  // Your side effect logic here
  return () => {
    // Cleanup logic here (optional)
  };
}, [dependencies]);
```

Per esempio, se aggiungi un listener per l’evento di scroll, puoi pulirlo rimuovendolo nella tua funzione di pulizia:

```js
useEffect(() => {
  const handleScroll = () => {
    // Handle scroll logic
  };
  window.addEventListener("scroll", handleScroll);

  return () => {
    window.removeEventListener("scroll", handleScroll);
  };
}, []);
```
