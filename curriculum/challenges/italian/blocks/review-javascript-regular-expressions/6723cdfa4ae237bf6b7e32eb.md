## Espressioni regolari e metodi comuni

- **Definizione**: Le espressioni regolari, o regex, servono a creare un "pattern" che puoi poi usare per verificare una stringa, estrarre testo e altro.

```js
const regex = /freeCodeCamp/;
```

- **Metodo `test()`**: Questo metodo accetta una stringa, che è la stringa da testare per verificare se corrisponde all'espressione regolare. Restituisce un booleano se la stringa corrisponde alla regex.

:::interactive_editor

```js
const regex = /freeCodeCamp/;
const test = regex.test("e");
console.log(test); // false
```

:::

- **Metodo `match()`**: Questo metodo accetta un'espressione regolare, anche se puoi passare una stringa che verrà convertita in un'espressione regolare. Il metodo `match` restituisce l'array delle corrispondenze per la stringa.

```js
const regex = /freeCodeCamp/;
const match = "freeCodeCamp".match(regex);
console.log(match); // ['freeCodeCamp', index: 0, input: 'freeCodeCamp', groups: undefined]
```

- **Metodo `replace()`**: Questo metodo accetta due argomenti: l'espressione regolare da cercare (o una stringa) e la stringa con cui sostituire la corrispondenza (o una funzione da eseguire su ogni corrispondenza).

:::interactive_editor

```js
const regex = /Jessica/;
const str = "Jessica is rly kewl";
const replaced = str.replace(regex, "freeCodeCamp");
console.log(replaced); // "freeCodeCamp is rly kewl"
```

:::

- **Metodo `replaceAll`**: Questo metodo serve a sostituire tutte le occorrenze di un pattern specificato con una nuova stringa. Lancia un errore se gli passi un'espressione regolare senza il modificatore globale.

:::interactive_editor

```js
const text = "I hate JavaScript! I hate programming!";
const newText = text.replaceAll("hate", "love");
console.log(newText);  // "I love JavaScript! I love programming!"
```

:::

- **Metodo `matchAll`**: Questo metodo serve a recuperare tutte le corrispondenze di una data espressione regolare in una stringa, inclusi i gruppi di cattura, e le restituisce come un iteratore. Un iteratore è un oggetto che ti permette di scorrere (o "iterare su") una collezione di elementi.

:::interactive_editor

```js
const str = "JavaScript, Python, JavaScript, Swift, JavaScript";
const regex = /JavaScript/g;

const iterator = str.matchAll(regex);

for (let match of iterator) {
  console.log(match[0]); // "JavaScript" for each match
}
```

:::

## Modificatori delle espressioni regolari

- **Definizione**: I modificatori, spesso chiamati "flag", modificano il comportamento di un'espressione regolare.
- **Flag `i`**: Questo flag fa sì che una regex ignori le differenze tra maiuscole e minuscole.

:::interactive_editor

```js
const regex = /freeCodeCamp/i;
console.log(regex.test("freecodecamp")); // true
console.log(regex.test("FREECODECAMP")); // true
```

:::

- **Flag `g`**: Questo flag, o modificatore globale, permette alla tua espressione regolare di trovare più corrispondenze di un pattern.

:::interactive_editor

```js
const regex = /freeCodeCamp/gi;
console.log(regex.test("freeCodeCamp")); // true
console.log(regex.test("freeCodeCamp is great")); // false
```

:::

- **Definizione di àncora**: L'àncora `^` all'inizio dell'espressione regolare indica "corrispondi all'inizio della stringa". L'àncora `$` alla fine dell'espressione regolare indica "corrispondi alla fine della stringa".

:::interactive_editor

```js
const start = /^freeCodeCamp/i;
const end = /freeCodeCamp$/i;
console.log(start.test("freecodecamp")); // true
console.log(end.test("freecodecamp")); // true
```

:::

- **Flag `m`**: Le àncore cercano l'inizio e la fine dell'intera stringa. Ma puoi far gestire a una regex più righe con il flag `m`, o modificatore multi-linea.

:::interactive_editor

```js
const start = /^freecodecamp/im;
const end = /freecodecamp$/im;
const str = `I love 
freecodecamp
it's my favorite
`;
console.log(start.test(str)); // true
console.log(end.test(str)); // true
```

:::

- **Flag `d`**: Questo flag amplia le informazioni che ottieni in un oggetto corrispondenza.

```js
const regex = /freecodecamp/di;
const string = "we love freecodecamp isn't freecodecamp great?";
console.log(string.match(regex));
```

E il risultato è:

```js
// [
//   'freecodecamp',
//   index: 8,
//   input: "we love freecodecamp isn't freecodecamp great?",
//   groups: undefined,
//   indices: [
//     0: [8, 20],
//     groups: undefined
//   ]
// ]
```

- **Flag `u`**: Questo amplia la funzionalità di un'espressione regolare per permetterle di corrispondere a caratteri unicode speciali. Il flag `u` ti dà accesso a classi speciali come `Extended_Pictographic` per corrispondere alla maggior parte delle emoji. Esiste anche un flag `v` che espande ulteriormente la funzionalità del matching unicode.
- **Flag `y`**: Il modificatore sticky si comporta in modo molto simile al modificatore globale, ma con alcune eccezioni. La più importante è che una regex globale parte da lastIndex e cerca nell'intero resto della stringa un'altra corrispondenza, mentre una regex sticky restituisce null e resetta lastIndex a 0 se non c'è una corrispondenza immediata all'ultimo lastIndex.
- **Flag `s`**: Il modificatore single-line permette a un carattere jolly, rappresentato da `.` nella regex, di corrispondere anche ai ritorni a capo — trattando di fatto la stringa come una singola riga di testo.

## Classi di caratteri

- **Jolly `.`**: Le classi di caratteri sono una sintassi speciale che puoi usare per corrispondere a insiemi o sottoinsiemi di caratteri. La prima classe di caratteri che dovresti imparare è la classe jolly. Il jolly è rappresentato da un punto e corrisponde a QUALSIASI singolo carattere TRANNE i ritorni a capo. Per permettere alla classe jolly di corrispondere anche ai ritorni a capo, ricorda che devi usare il flag `s`.

```js
const regex = /a./;
```

- **`\d`**: Questo corrisponde a tutte le cifre (`0-9`) in una stringa.

```js
const regex = /\d/;
```

- **`\w`**: Serve a corrispondere a qualsiasi carattere di parola (`a-z0-9_`) in una stringa. Un carattere di parola è definito come qualsiasi lettera da a a z, o un numero da 0 a 9, o il carattere underscore.

```js
const regex = /\w/;
```

- **`\s`**: La classe di spazi bianchi `\s`, rappresentata da una barra rovesciata seguita da `s`. Questa classe di caratteri corrisponde a qualsiasi spazio bianco, inclusi ritorni a capo, spazi, tabulazioni e caratteri di spazio unicode speciali.
- **Negare le classi di caratteri speciali**: Per negare una di queste classi di caratteri, invece di usare una lettera minuscola dopo la barra rovesciata, puoi usare la corrispondente maiuscola. L'esempio seguente non corrisponde a un carattere numerico. Invece, corrisponde a qualsiasi singolo carattere che NON sia numerico.

```js
const regex = /\D/;
```

- **Classi di caratteri personalizzate**: Puoi creare classi di caratteri personalizzate mettendo i caratteri che vuoi far corrispondere all'interno di un insieme di parentesi quadre.

```js
const regex = /[abcdf]/;
```

## Asserzioni lookahead e lookbehind

- **Definizione**: Le asserzioni lookahead e lookbehind ti permettono di corrispondere a pattern specifici basandoti sulla presenza o assenza di pattern circostanti.
- **Asserzione lookahead positiva**: Questa asserzione corrisponde a un pattern quando è seguito da un altro pattern. Per costruire un lookahead positivo, devi partire dal pattern che vuoi corrispondere. Poi, usa le parentesi per racchiudere il pattern che vuoi usare come condizione. Dopo la parentesi aperta, usa `?=` per definire quel pattern come lookahead positivo.

```js
const regex = /free(?=code)/i;
```

- **Asserzione lookahead negativa**: Questo è un tipo di condizione usata nelle espressioni regolari per verificare che un certo pattern non si presenti più avanti nella stringa.

```js
const regex = /free(?!code)/i;
```

- **Asserzione lookbehind positiva**: Questa asserzione corrisponde a un pattern solo se è preceduto da un altro pattern specifico, senza includere il pattern precedente nella corrispondenza.

```js
const regex = /(?<=free)code/i;
```

- **Asserzione lookbehind negativa**: Questa asserzione assicura che un pattern non sia preceduto da un altro pattern specifico. Corrisponde solo se il pattern specificato non è immediatamente preceduto dalla sequenza data, senza includere la sequenza precedente nella corrispondenza.

```js
const regex = /(?<!free)code/i;
```

## Quantificatori nelle regex

- **Definizione**: I quantificatori nelle espressioni regolari specificano quante volte un pattern (o parte di un pattern) deve comparire. Aiutano a controllare il numero di occorrenze di caratteri o gruppi in una corrispondenza. L'esempio seguente serve a corrispondere esattamente quattro volte il carattere precedente.

```js
const regex = /^\d{4}$/;
```

- **`*`**: Corrisponde a 0 o più occorrenze dell'elemento precedente.
- **`+`**: Corrisponde a 1 o più occorrenze dell'elemento precedente.
- **`?`**: Corrisponde a 0 o 1 occorrenza dell'elemento precedente.
- **`{n}`**: Corrisponde esattamente a n occorrenze dell'elemento precedente.
- **`{n,}`**: Corrisponde a n o più occorrenze dell'elemento precedente.
- **`{n,m}`**: Corrisponde a un numero di occorrenze dell'elemento precedente compreso tra n e m.

## Gruppi di cattura e backreference

- **Gruppi di cattura**: Un gruppo di cattura ti permette di "catturare" una porzione della stringa corrispondente per usarla come vuoi. I gruppi di cattura sono definiti da parentesi che contengono il pattern da catturare, senza caratteri iniziali come un lookahead.

```js
const regex = /free(code)camp/i;
```

- **Gruppi non di cattura**: Un gruppo non di cattura è simile a un gruppo di cattura ma non memorizza la porzione corrispondente della stringa per un uso successivo. I gruppi non di cattura sono definiti da `(?:...)`.

```js
const regex = /free(?:code)camp/i;
```

- **Backreference**: Una backreference nelle espressioni regolari è un modo per riutilizzare una parte del pattern che è stata catturata prima nella stessa espressione. Ti permette di riferirti a un gruppo catturato (una parte del pattern tra parentesi) tramite il suo numero. Per esempio, `$1` si riferisce al primo gruppo catturato.

:::interactive_editor

```js
const regex = /free(co+de)camp/i;
console.log("freecoooooooodecamp".replace(regex, "paid$1world"));
```

:::

- Puoi usare le backreference all'interno della regex stessa per corrispondere allo stesso testo catturato da un gruppo precedente con una barra rovesciata e il numero del gruppo di cattura. Per esempio:

:::interactive_editor

```js
const regex = /(hello) \1/i;
console.log(regex.test("hello hello")); // true
console.log(regex.test("hello world")); // false
```

:::
