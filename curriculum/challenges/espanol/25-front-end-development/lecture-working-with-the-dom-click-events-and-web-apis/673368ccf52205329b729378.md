# --transcript--

¿Cuál es la diferencia entre `innerText`, `textContent` y `innerHTML`?

Aprendamos sobre `innerText`, `textContent` y `innerHTML`.

Estas son propiedades que puedes acceder en JavaScript para obtener o cambiar el contenido de un elemento HTML. Aunque al principio pueden parecer muy similares, realmente tienen diferencias clave. Elegir la correcta depende de tu caso de uso específico, así que profundicemos.

Comencemos con `innerText`.

`innerText` representa el contenido de texto visible del elemento HTML y sus descendientes. Esta propiedad no incluye texto oculto ni etiquetas HTML, sólo texto renderizado.

Por ejemplo, aquí puedes ver un elemento `div` que contiene dos párrafos:

```html
<div id="container">
  <p>Hello, World!</p>
  <p>I'm learning JavaScript</p>
</div>
```

Si obtenemos una referencia a este elemento HTML en nuestro código JavaScript usando `getElementById()`, podemos acceder a la propiedad `innerText` de este elemento:

```js
const container = document.getElementById("container");
console.log(container.innerText);
```

Este es el texto interior de este elemento:

```md
Hello, World!
I'm learning JavaScript
```

La propiedad devuelve una cadena con el texto contenido dentro del elemento, incluyendo texto de sus descendientes.

Debes saber que `innerText` sólo devuelve el texto que está visible en el momento particular cuando se solicita la cadena. Si un elemento secundario está oculto, su texto no será visible.

Este es un ejemplo donde el segundo párrafo está oculto:

```html
<div id="container">
  <p>Hello, World!</p>
  <p hidden>I'm learning JavaScript</p>
</div>
```

Si intentamos registrar el `innerText` de nuevo, ahora la salida no tendrá el texto del segundo párrafo:

```js
console.log(container.innerText);
```

Esta será la salida:

```md
Hello, World!
```

Puedes establecer el `innerText` de un elemento HTML de esta manera, pero esto reemplazará el texto existente y añadirá un elemento de salto de línea (`br`) por cada salto de línea:

```js
container.innerText = "JavaScript is awesome!";
```

Ya que `innerText` toma en cuenta la visibilidad, obtener su valor desencadena un proceso llamado "reflow", que recalcula la posición de ciertos elementos en el sitio web. Este proceso puede ser computacionalmente intensivo, así que deberías evitar desencadenarlo si es posible.

Genial. Ahora hablemos de `textContent`.

`textContent` devuelve el contenido de texto simple de un elemento, incluyendo todo el texto dentro de sus descendientes.

La diferencia más importante entre `innerText` y `textContent` es que `textContent` siempre devuelve el contenido completo de texto de un elemento HTML y sus descendientes, independientemente de si es visible o está oculto.

Aquí tenemos el mismo ejemplo en HTML:

```html
<div id="container">
  <p>Hello, World!</p>
  <p>I'm learning JavaScript</p>
</div>
```

Si intentas acceder a esta propiedad, verás el texto del elemento y sus descendientes como salida, manteniendo la sangría y el espaciado:

```js
const container = document.getElementById("container");
console.log(container.textContent);
```

```md
Hello, World!
I'm learning JavaScript
```

Si un elemento HTML no es visible, como puedes ver aquí, donde hemos ocultado el segundo párrafo, su texto aún estará incluido en esta propiedad:

```html
<div id="container">
  <p>Hello, World!</p>
  <p hidden>I'm learning JavaScript</p>
</div>
```

Verás la misma salida:

```md
Hello, World!
I'm learning JavaScript
```

`textContent` también incluirá el contenido de elementos como `script` y `style`.

Si intentas reemplazar el valor de `textContent` en un nodo, eliminará todos sus nodos secundarios y los reemplazará con un único nodo de texto que contiene la nueva cadena:

```js
const container = document.getElementById("container");
container.textContent = "Hello, World!";
```

Y finalmente, hablemos de cómo `textContent` e `innerText` difieren de `innerHTML`.

Recuerda que con `innerHTML` puedes establecer el contenido interno HTML de un elemento. Esto es útil para inyectar nuevo HTML en el DOM dinámicamente.

Sin embargo, recuerda que esto plantea un riesgo de seguridad si no tienes control sobre la cadena, como las cadenas que contienen datos ingresados por el usuario. Si esos datos son maliciosos, pueden ocasionar serios problemas de seguridad.

Para evitar esto, se recomienda usar la propiedad `textContent` para insertar texto simple.

Las propiedades `innerText`, `textContent` e `innerHTML` en JavaScript ofrecen diferentes maneras de acceder y manipular el contenido de los elementos HTML. Debes entender las diferencias entre estas propiedades si tu objetivo es trabajar eficientemente con el contenido HTML en JavaScript.
