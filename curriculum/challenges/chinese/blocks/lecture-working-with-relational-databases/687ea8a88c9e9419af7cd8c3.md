# --description--

让我们来看一些你可以用来插入和查看 SQL 中数据的基本命令。

首先，要查看数据，你需要插入它。所以我们将从插入开始。

本课将涵盖的示例将基于这张狗狗记录表格：

```sql
CREATE TABLE dogs(
  id SERIAL,
  name VARCHAR(100),
  age INTEGER
);
```

该表格将保存各种狗的名字和年龄。

如果我们假设正在使用 PostgreSQL，可以为 `id` 使用 `SERIAL` 数据类型。由于 `id` 是使用 `SERIAL` 创建的，其值将是一个 `INTEGER`，从 `1` 开始，并且在插入新记录时会自动递增。因此，在插入记录时你不需要为它传递值。

让我们先使用 `INSERT INTO` 语句向我们的 `dogs` 表格插入记录。有很多方法可以做到这一点。第一种方法是插入单行并指定将要插入值的列。

你编写 `INSERT INTO`，后跟表格名称（在此分支中为 `dogs`），然后是括号内的列，接着是 `VALUES`，以及括号内对应的值。

这是一个示例，我们正在插入一个 `name` 为 `'Gino'`、`age` 为 `3` 的狗记录，记住 `id` 会自动赋值：

```sql
INSERT INTO dogs (name, age)
VALUES ('Gino', 3);
```

这通常是最安全的选项，因为你显式指定了列，所以值将按顺序赋值给那些列。

注意 `'Gino'` 需要用单引号括起来，因为它是 `VARCHAR` 类型。

另一个备选是插入记录时不指定列，如下所示：

```sql
INSERT INTO dogs
VALUES ('Gino', 3);
```

此语法是有效的 SQL，但容易出错。

由于值是根据列顺序赋值的，PostgreSQL 会尝试将 `'Gino'` 插入到期望整数的 `id` 列中。这会导致诸如 `invalid input syntax for type integer` 的误差。

因此，在插入数据时显式指定列名更安全。

你也可以通过用逗号分隔，在同一个 SQL `command` 中插入多个 `record`。

这里，我们插入了两条狗的记录，一条是 `'Gino'`，另一条是 `'Nora'`。

```sql
INSERT INTO dogs (name, age)
VALUES
  ('Gino', 3),
  ('Nora', 2);
```

正如你可以插入记录一样，你也可以从数据库中查询信息。在数据库的上下文中，查询是对数据的请求。

在 SQL 中，`SELECT` 语句用于从一个或多个表格中查询数据。你可以自定义查询以获取你需要的准确信息。

如果你需要查询 `dogs` 表格中的所有信息，只需编写 `SELECT`，后跟星号（`*`），然后是 `FROM`，以及表格名称 `dogs`：

```sql
SELECT *
FROM dogs;
```

在此命令中，星号是一个通配符字符，表现为“所有列”。

你将获得表格中所有记录的所有数据：

```sql
 id | name | age 
----+------+-----
  1 | Gino |   3
  2 | Nora |   2
```

要查询特定的列，你可以在命令中 `SELECT` 后面写出这些列的名称，用逗号分隔。在此示例中，我们查询 `dogs` 表格的 `name` 和 `age` 列：

```sql
SELECT name, age
FROM dogs;
```

结果将仅包含 `name` 和 `age` 列：

```sql
 name | age 
------+-----
 Gino |   3
 Nora |   2
```

有时，你可能需要根据特定条件查询数据。例如，要获取所有小于 3 岁的狗，你可以使用 `WHERE` 关键字和小于 (`<`) 比较操作符：

```sql
SELECT *
FROM dogs
WHERE age < 3;
```

结果将仅包含年龄小于 3 的狗：

```sql
 id | name | age 
----+------+-----
  2 | Nora |   2
```

如果你只是想查找 `'Gino'` 的 `age`，你可以使用相等（`=`）比较操作符：

```sql
SELECT age
FROM dogs
WHERE name = 'Gino';
```

结果如下：

```sql
 age 
-----
  3
```

这些是在 SQL 中插入和查看数据的常见且简单的方法，但你可以选择许多不同的选项，其中一些是高级的。你将在接下来的课程中学习它们。
