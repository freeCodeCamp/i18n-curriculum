# --description--

In der vorherigen Lektion haben Sie gelernt, wie man Python-Skripte lokal ausführt. Aber es gibt Situationen, in denen Sie keine vollständigen Python-Programme erstellen möchten und nur etwas Python-Code testen wollen. Hier kommt die Python-Interactive-Shell ins Spiel.

Aber zuerst wiederholen wir, was ein Terminal ist.

Ein Terminal ist eine textbasierte Schnittstelle, mit der Sie durch Eingabe von Befehlen mit Ihrem Computer interagieren können. Jedes Betriebssystem bringt eine Standard-Terminal-App mit. Auf macOS können Sie die Terminal-App verwenden. Unter Windows können Sie die Eingabeaufforderung oder PowerShell nutzen. Unter Linux hat jede Desktop-Umgebung ihre eigene Standard-Terminal-App, wie GNOME Terminal oder Konsole.

Öffnen Sie eine Terminal-App und geben Sie `python` ein, dann drücken Sie `Enter`. Dadurch starten Sie eine Python-Interactive-Shell. Eine Interactive-Shell ist ein Programm, das es Ihnen ermöglicht, Befehle einzeln einzugeben und die Ergebnisse sofort zu sehen.

Wenn Sie eine neue Sitzung starten, sehen Sie anfangs möglicherweise diese Ausgabe:

```bash
Python 3.12.2 (main, Mar 21 2024, 22:48:26) [Clang 14.0.3 (clang-1403.0.22.14.1)] on darwin
Type "help", "copyright", "credits" or "license" for more information.
>>> 
```

Das `>>>`-Symbol bedeutet, dass Python darauf wartet, dass Sie einen Befehl eingeben. Versuchen Sie, `"Hello, world!"` im Terminal auszugeben:

```bash
print("Hello, world!")
```

Sie sollten den Text wie folgt sehen:

```bash
>>> print("Hello, world!")
Hello, world!
```

Nachdem der Text ausgegeben wurde, kehrt Python zu folgendem zurück:

```bash
>>>
```

Das bedeutet, Sie können einen weiteren Befehl eingeben.

Der Python-Interpreter folgt dem sogenannten Read, Evaluate, Print, Loop-Zyklus. Oder kurz REPL. Immer wenn Sie Befehle eingeben, liest der Interpreter die Eingabe, wertet sie aus, gibt das Ergebnis aus und kehrt zurück, um das `>>>` anzuzeigen, damit Sie weitere Befehle eingeben können.

Was passiert, wenn Sie versuchen, einen ungültigen Befehl wie diesen einzugeben?

```bash
>>> something random
```

Der Python-Interpreter folgt trotzdem dem gleichen REPL-Prozess. In diesem Fall erhalten Sie eine Fehlermeldung:

```bash
>>> something random
  File "<stdin>", line 1
    something random
              ^^^^^^
SyntaxError: invalid syntax
```

Wenn Sie die Interactive-Shell verlassen möchten, können Sie `exit()` eingeben oder `Ctrl + D` (Mac/Linux) bzw. `Ctrl + Z + Enter` (Windows) drücken.

Die Verwendung der Python-Interactive-Shell ist ideal für kleine Experimente mit Code oder grundlegende Erkundungen. Während Sie die restlichen Lektionen im Kurs durchgehen, können Sie einige der neuen Methoden, die Sie gelernt haben, in einer Interactive-Shell ausprobieren.

Wenn Sie vorhaben, mit längeren Programmen zu arbeiten oder über mehrere Dateien hinweg, ist es besser, in einem Code-Editor oder einer IDE zu arbeiten.
