# --description--

イベント処理は、すべてのインタラクティブなウェブサイトに欠かせない要素です。

Reactは、Synthetic Event Systemという仕組みを通じて、イベントを強力かつ一貫して処理する方法を提供しています。これは、以前のレッスンで学んだ`click`、`keydown`、`submit`のようなネイティブイベントのラッパーです。

このクロスブラウザ対応のラッパーにより、すべてのブラウザでイベントが同じように動作し、不整合が起きません。

Reactでイベントがどのように動作するかを見て、プロジェクトで使い始めましょう。

Reactでは、イベントハンドラはネイティブブラウザのイベントと似ていますが、いくつかの調整があります。

小文字のイベント属性名（`onclick`や`onsubmit`）の代わりに、Reactではキャメルケース（`onClick`や`onSubmit`）を使います。

さらに、イベントの種類を文字列で指定する代わりに、Reactはイベントハンドラとして関数を期待します。

イベントハンドラ関数は要素にプロップとして渡され、`onClick`や`onSubmit`のようなイベントタイプはJSXの属性として使われます。

通常のHTMLでクリックイベントを扱う方法をおさらいしましょう。

```html
<button onclick="alert('Button clicked!')">Click Me</button>
```

そしてReactで同じことをする方法はこちらです。

```jsx
function handleClick() {
  console.log("Button clicked!");
}

<button onClick={handleClick}>Click Me</button>;
```

この例では、`handleClick`がユーザーがボタンをクリックしたときにコンソールにメッセージを記録します。

`handleClick`属性内の`onClick`の後に丸括弧が不要なのは、関数を呼び出すのではなく参照を渡しているからです。

Reactでは、イベントハンドラ関数は通常、イベントを処理することを示す接頭辞`handle`で始まります。例えば`handleClick`や`handleSubmit`です。

ユーザーの操作でイベントが発生すると、ReactはSynthetic Eventオブジェクトをハンドラに渡します。このオブジェクトは、バニラJavaScriptのネイティブイベントオブジェクトと似ており、`type`、`target`、`currentTarget`のようなプロパティを提供します。

ハンドラ関数に`event`をパラメータとして渡し、その構造を確認するためにコンソールに記録してみましょう。

```js
function handleClick(event) {
  console.log(event);
}
```

例えば、`onSubmit`イベント中にブラウザの更新などのデフォルト動作を防ぎたい場合は、`preventDefault()`メソッドを呼び出します。

```jsx
function handleSubmit(event) {
  event.preventDefault();
  console.log("Form submitted!");
}

<form onSubmit={handleSubmit}>
  <input type="text" />
  <button>Submit</button>
</form>;
```

また、イベントが親要素にバブリングするのを防ぐには、`event.stopPropagation()`を呼び出します。

削除や編集機能のような特別なケースを処理するときに、イベントハンドラに追加のデータを渡したい場合があります。その場合は、インラインのアロー関数でハンドラをラップしてください。

```jsx
function handleDelete(id) {
  console.log("Deleting item:", id);
}

<button onClick={() => handleDelete(1)}>Delete Item</button>;
```

Reactではインラインイベントハンドラを使っても問題ありません。Reactは効率的に再レンダリングを管理し、パフォーマンスの問題を回避します。

一方、バニラJavaScriptでは、インラインイベントハンドラは毎回新しい関数を作成するため、仮想DOMがない分パフォーマンスに悪影響を与える場合があります。
