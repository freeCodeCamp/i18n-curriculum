## 정규 표현식과 자주 쓰이는 메서드

- **정의**: 정규 표현식, 또는 정규식은 "패턴"을 만들 때 사용하며, 이 패턴을 이용해 문자열을 검사하거나 텍스트를 추출하는 등 다양한 작업을 할 수 있습니다.

```js
const regex = /freeCodeCamp/;
```

- **`test()` 메서드**: 이 메서드는 문자열을 인수로 받아 정규 표현식과 일치하는지 검사합니다. 문자열이 정규식과 일치하면 불리언 값을 반환합니다.

:::interactive_editor

```js
const regex = /freeCodeCamp/;
const test = regex.test("e");
console.log(test); // false
```

:::

- **`match()` 메서드**: 이 메서드는 정규 표현식을 인수로 받으며, 문자열을 전달하면 정규 표현식으로 변환됩니다. `match` 메서드는 문자열에 대한 일치 배열을 반환합니다.

```js
const regex = /freeCodeCamp/;
const match = "freeCodeCamp".match(regex);
console.log(match); // ['freeCodeCamp', index: 0, input: 'freeCodeCamp', groups: undefined]
```

- **`replace()` 메서드**: 이 메서드는 두 개의 인수를 받습니다: 일치시킬 정규 표현식(또는 문자열)과 일치한 부분을 대체할 문자열(또는 각 일치에 대해 실행할 함수)입니다.

:::interactive_editor

```js
const regex = /Jessica/;
const str = "Jessica is rly kewl";
const replaced = str.replace(regex, "freeCodeCamp");
console.log(replaced); // "freeCodeCamp is rly kewl"
```

:::

- **`replaceAll` 메서드**: 이 메서드는 지정한 패턴의 모든 발생을 새 문자열로 대체할 때 사용합니다. 글로벌 수정자가 없는 정규식을 전달하면 오류가 발생합니다.

:::interactive_editor

```js
const text = "I hate JavaScript! I hate programming!";
const newText = text.replaceAll("hate", "love");
console.log(newText);  // "I love JavaScript! I love programming!"
```

:::

- **`matchAll` 메서드**: 이 메서드는 문자열에서 주어진 정규식과 일치하는 모든 결과(캡처 그룹 포함)를 가져와 반복자로 반환합니다. 반복자는 항목 모음을 순회할 수 있는 객체입니다.

:::interactive_editor

```js
const str = "JavaScript, Python, JavaScript, Swift, JavaScript";
const regex = /JavaScript/g;

const iterator = str.matchAll(regex);

for (let match of iterator) {
  console.log(match[0]); // "JavaScript" for each match
}
```

:::

## 정규 표현식 수정자

- **정의**: 수정자, 흔히 "플래그"라고도 하며, 정규 표현식의 동작 방식을 변경합니다.  
- **`i` 플래그**: 이 플래그는 정규식이 대소문자를 구분하지 않도록 만듭니다.

:::interactive_editor

```js
const regex = /freeCodeCamp/i;
console.log(regex.test("freecodecamp")); // true
console.log(regex.test("FREECODECAMP")); // true
```

:::

- **`g` 플래그**: 이 플래그, 또는 글로벌 수정자는 정규 표현식이 패턴을 여러 번 일치시킬 수 있게 합니다.

:::interactive_editor

```js
const regex = /freeCodeCamp/gi;
console.log(regex.test("freeCodeCamp")); // true
console.log(regex.test("freeCodeCamp is great")); // false
```

:::

- **앵커 정의**: 정규 표현식의 시작 부분에 있는 `^` 앵커는 "문자열의 시작과 일치"를 의미합니다. 정규 표현식의 끝 부분에 있는 `$` 앵커는 "문자열의 끝과 일치"를 의미합니다.

:::interactive_editor

```js
const start = /^freeCodeCamp/i;
const end = /freeCodeCamp$/i;
console.log(start.test("freecodecamp")); // true
console.log(end.test("freecodecamp")); // true
```

:::

- **`m` 플래그**: 앵커는 전체 문자열의 시작과 끝을 찾습니다. 하지만 `m` 플래그, 즉 여러 줄 수정자를 사용하면 정규 표현식이 여러 줄을 처리하도록 만들 수 있습니다.

:::interactive_editor

```js
const start = /^freecodecamp/im;
const end = /freecodecamp$/im;
const str = `I love 
freecodecamp
it's my favorite
`;
console.log(start.test(str)); // true
console.log(end.test(str)); // true
```

:::

- **`d` 플래그**: 이 플래그는 매치 객체에서 얻는 정보를 확장합니다.

```js
const regex = /freecodecamp/di;
const string = "we love freecodecamp isn't freecodecamp great?";
console.log(string.match(regex));
```

결과는 다음과 같습니다:

```js
// [
//   'freecodecamp',
//   index: 8,
//   input: "we love freecodecamp isn't freecodecamp great?",
//   groups: undefined,
//   indices: [
//     0: [8, 20],
//     groups: undefined
//   ]
// ]
```

- **`u` 플래그**: 이 플래그는 정규 표현식의 기능을 확장하여 특수 유니코드 문자를 매치할 수 있게 합니다. `u` 플래그는 대부분의 이모지를 매치하는 `Extended_Pictographic` 같은 특수 클래스를 사용할 수 있게 해줍니다. 또한 유니코드 매칭 기능을 더 확장하는 `v` 플래그도 있습니다.  
- **`y` 플래그**: 스티키 수정자는 글로벌 수정자와 매우 비슷하게 동작하지만 몇 가지 예외가 있습니다. 가장 큰 차이는 글로벌 정규식은 lastIndex부터 시작해 문자열의 나머지 전체를 검색하지만, 스티키 정규식은 이전 lastIndex에서 즉시 매치가 없으면 null을 반환하고 lastIndex를 0으로 재설정한다는 점입니다.  
- **`s` 플래그**: 싱글라인 수정자는 정규식에서 `.`로 나타내는 와일드카드 문자가 줄바꿈 문자도 매치할 수 있게 하여, 문자열을 한 줄의 텍스트처럼 처리합니다.

## 문자 클래스

- **와일드카드 `.`**: 문자 클래스는 문자 집합이나 부분집합과 일치시키기 위해 사용할 수 있는 특별한 구문입니다. 가장 먼저 배워야 할 문자 클래스는 와일드카드 클래스입니다. 와일드카드는 마침표, 즉 점으로 나타내며 줄 바꿈을 제외한 모든 단일 문자와 일치합니다. 와일드카드 클래스가 줄 바꿈과도 일치하게 하려면 `s` 플래그를 사용해야 한다는 점을 기억하세요.

```js
const regex = /a./;
```

- **`\d`**: 이 클래스는 문자열 내 모든 숫자(`0-9`)와 일치합니다.

```js
const regex = /\d/;
```

- **`\w`**: 문자열에서 모든 단어 문자를 매치하는 데 사용합니다(`a-zA-Z0-9_`). 단어 문자는 a부터 z 또는 A부터 Z까지의 모든 문자, 0부터 9까지의 숫자, 또는 밑줄 문자로 정의됩니다.

```js
const regex = /\w/;
```

- **`\s`**: 공백 클래스 `\s`는 역슬래시 다음에 `s`가 오는 형태로 나타냅니다. 이 문자 클래스는 줄 바꿈, 공백, 탭, 특수 유니코드 공백 문자를 포함한 모든 공백과 일치합니다.
- **특수 문자 클래스 부정하기**: 이러한 문자 클래스 중 하나를 부정하려면 역슬래시 다음에 소문자 대신 대문자를 사용하면 됩니다. 다음 예시는 숫자 문자와 일치하지 않고, 숫자 문자가 아닌 모든 단일 문자와 일치합니다.

```js
const regex = /\D/;
```

- **사용자 정의 문자 클래스**: 일치시키려는 문자를 대괄호 안에 넣어 사용자 정의 문자 클래스를 만들 수 있습니다.

```js
const regex = /[abcdf]/;
```

## 전방 탐색과 후방 탐색 어서션

- **정의**: 전방 탐색과 후방 탐색 어서션은 주변 패턴의 존재 여부에 따라 특정 패턴을 매칭할 수 있게 합니다.
- **긍정 전방 탐색 어서션**: 이 어서션은 특정 패턴 뒤에 다른 패턴이 올 때 매칭합니다. 긍정 전방 탐색을 만들려면, 먼저 매칭하려는 패턴을 작성하세요. 그런 다음, 조건으로 사용할 패턴을 소괄호로 감쌉니다. 소괄호 뒤에 `?=`를 사용해 해당 패턴을 긍정 전방 탐색으로 정의합니다.

```js
const regex = /free(?=code)/i;
```

- **부정 전방 탐색 어서션**: 정규 표현식에서 특정 패턴이 문자열 앞에 나타나지 않는지 확인하는 조건 유형입니다.

```js
const regex = /free(?!code)/i;
```

- **긍정 후방 탐색 어서션**: 이 어서션은 특정 패턴 앞에 다른 특정 패턴이 있을 때만 매칭하며, 앞선 패턴은 매칭 결과에 포함하지 않습니다.

```js
const regex = /(?<=free)code/i;
```

- **부정 후방 탐색 어서션**: 이 어서션은 특정 패턴 앞에 다른 특정 패턴이 없음을 보장합니다. 지정한 패턴이 바로 앞선 수열에 의해 선행되지 않을 때만 매칭하며, 앞선 수열은 매칭 결과에 포함하지 않습니다.

```js
const regex = /(?<!free)code/i;
```

## 정규식 수량자

- **정의**: 정규 표현식에서 수량자는 패턴(또는 패턴의 일부)이 몇 번 나타나야 하는지를 지정합니다. 수량자는 일치하는 문자나 그룹의 발생 횟수를 제어하는 데 도움을 줍니다. 다음 예시는 바로 앞의 문자가 정확히 네 번 나타나는 경우를 찾는 데 사용됩니다.

```js
const regex = /^\d{4}$/;
```

- **`*`** : 바로 앞 요소가 0번 이상 나타나는 경우와 일치합니다.
- **`+`**: 바로 앞 요소가 1번 이상 나타나는 경우와 일치합니다.
- **`?`**: 바로 앞 요소가 0번 또는 1번 나타나는 경우와 일치합니다.
- **`{n}`**: 바로 앞 요소가 정확히 n번 나타나는 경우와 일치합니다.
- **`{n,}`**: 바로 앞 요소가 n번 이상 나타나는 경우와 일치합니다.
- **`{n,m}`**: 바로 앞 요소가 n번에서 m번 사이에 나타나는 경우와 일치합니다.

## 캡처 그룹과 백참조

- **캡처 그룹**: 캡처 그룹은 일치한 문자열의 일부를 "캡처"하여 필요에 따라 사용할 수 있게 합니다. 캡처 그룹은 캡처할 패턴을 포함하는 소괄호로 정의하며, 전방 탐색 같은 선행 문자가 없습니다.

```js
const regex = /free(code)camp/i;
```

- **비캡처 그룹**: 비캡처 그룹은 캡처 그룹과 비슷하지만, 나중에 사용할 수 있도록 일치한 문자열을 저장하지 않습니다. 비캡처 그룹은 `(?:...)`로 정의합니다.

```js
const regex = /free(?:code)camp/i;
```

- **백참조**: 정규 표현식에서 백참조는 같은 표현식 내에서 이전에 일치한 패턴의 일부를 재사용하는 방법을 말합니다. 이는 소괄호 안에 있는 캡처 그룹을 번호로 참조할 수 있게 합니다. 예를 들어, `$1`는 첫 번째 캡처 그룹을 가리킵니다.

:::interactive_editor

```js
const regex = /free(co+de)camp/i;
console.log("freecoooooooodecamp".replace(regex, "paid$1world"));
```

:::

- 역참조는 정규식 내에서 백슬래시와 캡처 그룹 번호를 사용해 이전 그룹이 캡처한 동일한 텍스트와 일치시킬 수 있습니다. 예를 들어:

:::interactive_editor

```js
const regex = /(hello) \1/i;
console.log(regex.test("hello hello")); // true
console.log(regex.test("hello world")); // false
```

:::
