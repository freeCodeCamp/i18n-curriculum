# --description--

Плюс `+` існує у двох формах: бінарній, яку ви використовували в останньому завданні, і у унарній формі.

Унарний плюс, або, іншими словами, оператор плюс `+`, застосований до одного значення, нічого не робить з числами. Але якщо операнд не є числом, унарний плюс перетворює його на число.

Наприклад:

```js
// No effect on numbers
let x = 1;
console.log( +x ); // 1

let y = -2;
console.log( +y ); // -2

// Converts non-numbers
console.log( +true ); // 1
console.log( +"" );   //
```

Потреба перетворювати рядки на числа виникає дуже часто. Наприклад, якщо ви отримуєте значення з полів HTML-форми, вони зазвичай є рядками. Що робити, якщо ви хочете їх додати?

Бінарний плюс додасть їх як рядки:

```js
let apples = "2";
let oranges = "3";

alert( apples + oranges ); // "23", the binary plus concatenates strings
```

Якщо ви хочете розглядати їх як числа, потрібно спочатку перетворити, а потім додати:

```js
let apples = "2";
let oranges = "3";

// both values converted to numbers before the binary plus
alert( +apples + +oranges ); // 5

// the longer variant
// alert( Number(apples) + Number(oranges) ); // 5
```

З точки зору математика, надмірність плюсів може здатися дивною. Але з точки зору програміста немає нічого особливого: унарні плюси застосовуються першими, вони перетворюють рядки на числа, а потім бінарний плюс їх додає.

Чому унарні плюси застосовуються до значень раніше за бінарні? Як ви побачите, це через їх _вищий_ пріоритет.

# --questions--

## --text--
Враховуючи пояснення унарних і бінарних операторів `+` у JavaScript, як правильно додати рядкові значення `'2'` і `'3'` як числа, використовуючи унарний `+`?

## --answers--

`console.log('2' + '3'); // Outputs: '23'` 

---

`console.log(+2 + +3); // Outputs: 5`

---

`console.log(+'2' + +'3'); // Outputs: 5`

---

`console.log(Number('2') + Number('3')); // Outputs: '23'`


## --video-solution--

3
