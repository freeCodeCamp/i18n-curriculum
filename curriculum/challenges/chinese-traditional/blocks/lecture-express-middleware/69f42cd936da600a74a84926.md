# --description--

中介層是 Express 中一個重要的概念，它讓我們能夠在客戶端和伺服端之間處理請求和回應。中介層函式可以存取請求物件、回應物件，以及應用程式請求-回應週期中的下一個函式。

讓我們來拆解說明：

## 什麼是中介層？

中介層本質上是一個在伺服器請求生命週期中執行的函式。它會執行各種任務，例如記錄請求、處理錯誤、解析進來的請求，或提供靜態檔案。

Express 提供了一個方便的方式來定義中介層，讓它能在請求到達路由處理常式之前先進行處理。

中介層如何運作：

當對 Express 應用程式發出請求時，中介層會依照加入的順序執行。流程如下：

* 請求被送到伺服器。
* 中介層函式處理該請求。
* 如果中介層呼叫 `next()`，它會將控制權傳遞給下一個中介層或路由處理常式。
* 如果中介層沒有呼叫 `next()`，請求-回應週期就會停止。

```js
const express = require('express')
const app = express()

// Simple middleware that logs request info
app.use((req, res, next) => {
  console.log('Request URL:', req.url)
  console.log('Request Method:', req.method)
  next() // Passes control to the next middleware
})

app.get('/', (req, res) => {
  res.send('Hello World!')
})

app.listen(3000, () => {
  console.log('Server running on http://localhost:3000')
})
```

`app.use()`：將中介層全域地添加到你的應用程式中。

在這個範例中，中介層會記錄請求細節，並呼叫 `next()` 以繼續執行下一個中介層或路由處理常式。

中介層是 Express 中一個強大的功能，讓你能在不同階段攔截並修改請求。你可以使用中介層來處理像是記錄、驗證，甚至在請求到達路由處理常式之前修改請求和回應物件。
