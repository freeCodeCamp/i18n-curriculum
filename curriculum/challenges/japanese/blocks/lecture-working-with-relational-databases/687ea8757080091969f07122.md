# --description--

SQLはStructured Query Languageの略です。1970年代に開発され、当初はstructured English query language（SEQUEL）として知られていました。この用語は後にSQLに短縮されました。

SQLはリレーショナルデータベース内のデータを保存および管理するためのプログラミング言語です。

前のレッスンで学んだように、リレーショナルデータベースはデータをテーブルに整理します。各テーブルには行と列のセットがあり、データの属性とその関係が表されています。各行はレコードを表し、各列はそのレコードの特定の属性またはフィールドを表します。

SQLを使うと、リレーショナルデータベースからデータを追加、変更、削除、検索、取得できます。データのクエリ、フィルタリング、並べ替え、集計のための包括的なコマンドセットを提供します。

SQLは必要なデータだけをクエリすることでデータベースのパフォーマンスを最適化でき、さまざまなプログラミング言語と簡単に統合できるため、アプリケーションから直接データベースとやり取りするのに使えます。

また、業界標準として採用されています。データの整合性を強制し、ユーザー認証やデータ暗号化のサポートなど、多くのセキュリティ機能を含みます。

さらに、スケーラブルでポータブル、そして多くのデータベース管理システムと互換性があります。

SQLはコマンドに基づいており、一般にSQL文またはSQLクエリとして知られています。

開発者として、特定のSQL言語要素やキーワードを使ってこれらの文を書きます。これにより、データベースに必要な操作を実行できます。

SQLコマンドを入力する前に、psqlシェルを開いてPostgreSQLデータベースと直接やり取りできるようにする必要があります。コマンドプロンプトやターミナルで次のように入力してください：

```bash
psql -U <username> -d <database_name>
```

`username`をユーザー名に、`database_name`を接続したいデータベース名に置き換えてください。

まだ自分のデータベースを作成していない場合、PostgreSQLはインストール時に`postgres`というデフォルトのデータベースを自動的に含みます。この`postgres`データベースを使って進めることができます。

接続すると、プロンプトが次のように変わります：

```bash
postgres=#
```

接続しているデータベース名が表示され、SQLコマンドの入力を待っています。

ここで、知っておくべき基本的なSQLコマンドをいくつか見ていきましょう。慣例として、SQLコマンドはセミコロン（`;`）で終わることに注意してください。

まず、`my_database`という名前のデータベースを作成するには、このコマンドを使います：

```sql
CREATE DATABASE my_database;
```

`CREATE`に続けて`DATABASE`、そして作成したいデータベース名を入力します。コマンドのSQLキーワードは大文字で書かれていることに注意してください。

次に、データベースを作成したら、それに接続できます。

どのようにデータベースに接続するかは、使用している環境やツールによって異なります。

コマンドプロンプトやターミナルでは、先ほど示した方法を使えます。対話型のpsqlシェルでは、次のように`\c`ショートカットコマンドに続けてデータベース名を入力して切り替えられます：

```bash
\c my_database
```

`\c`コマンドやその他のショートカットコマンド（バックスラッシュで始まるもの）はpsqlシェルの一部であり、SQL言語自体の一部ではないため、コマンドの終了にセミコロンは不要です。

新しいデータベースに接続すると、プロンプトが次のように変わります：

```bash
my_database=#
```

データベースを作成して接続したら、`CREATE TABLE`キーワードを使ってテーブルを作成できます：

```sql
CREATE TABLE products (
  id SERIAL,
  name VARCHAR(255)
);
```

コマンドでは、`CREATE TABLE`に続けて作成したいテーブル名を書きます。この例では、`products`という名前のテーブルが作成されます。

次に丸括弧内に、テーブルが持つべき列の名前を書き、それぞれの列に格納される値のデータ型を指定します。

例では、テーブルは2つの列を持ち、1つは製品ID用、もう1つは製品名用の列です。

テーブル名や列名の標準的な命名慣例はスネークケースで、小文字で単語を記述し、アンダースコアで区切ります（`_`）。例えば：`delivery_orders`

これらはSQLを始めるために知っておくべき基本的なコマンドのいくつかです。

SQLを使えば、データを簡単にクエリ、操作、分析して、情報に基づいた意思決定や現実の問題解決ができます。
