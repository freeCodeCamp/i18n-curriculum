## Python-Klassen und Objekte

- **Klassendefinition**: Eine Klasse ist eine Vorlage zur Erstellung von Objekten. Sie definiert das Verhalten, das ein Objekt durch seine Attribute und Methoden haben wird. Hier ist ein einfaches Beispiel für eine Klassendefinition in Python:

```python
class Dog:
    def __init__(self, name, age):
        self.name = name
        self.age = age

    def bark(self):
        print(f'{self.name.upper()} says woof woof!')
```

- **Objekte erstellen**: Objekte sind Instanzen einer Klasse. Sie werden erzeugt, indem man die Klasse mit den notwendigen Argumenten aufruft.

```python
dog1 = Dog('Jack', 3)
dog2 = Dog('Thatcher', 5)

dog1.bark()  # JACK says woof woof!
dog2.bark()  # THATCHER says woof woof!
```

- **Methoden auf Objekten aufrufen**: Sie können Methoden auf Objekten aufrufen, um Aktionen auszuführen oder Informationen abzurufen.

```python
object_name1.method_name()
object_name2.method_name()
```

- **Unterschied zwischen Klasse und Objekt**: Eine Klasse ist eine wiederverwendbare Vorlage, während ein Objekt eine konkrete Instanz dieser Klasse mit tatsächlichen Daten ist.

## Attribute

- **Instanzattribute**: Werden in `__init__()` mit `self` definiert und sind für jedes Objekt einzigartig.
- **Klassenattribute**: Werden direkt innerhalb der Klasse definiert und von allen Instanzen geteilt.

```python
class Dog:
    species = 'French Bulldog'  # Class attribute

    def __init__(self, name):
        self.name = name  # Instance attribute

print(Dog.species) # French Bulldog

jack = Dog('Jack')
print(jack.name)     # Jack
print(jack.species)  # French Bulldog
```

## Methoden

- **Methoden**: Funktionen, die innerhalb einer Klasse definiert sind und auf die Attribute des Objekts zugreifen.

```python
class Car:
    def __init__(self, color, model):
        self.color = color
        self.model = model

    def describe(self):
        return f'This car is a {self.color} {self.model}'

my_car_1 = Car('red', 'Tesla Model S')
print(my_car_1.describe())  # This car is a red Tesla Model S
```

- **Methoden aufrufen**: Rufen Sie Methoden auf Objekten mit der Punktnotation auf. Hier ist ein Beispiel, wie die `describe`-Methode bei zwei verschiedenen Auto-Objekten aufgerufen wird:

```python
class Car:
    def __init__(self, color, model):
        self.color = color  
        self.model = model  

    def describe(self):
        return f'This car is a {self.color} {self.model}'

my_car_1 = Car('red', 'Tesla Model S')
my_car_2 = Car('green', 'Lamborghini Revuelto')

print(my_car_1.describe()) # Calling method using the dot notation

print(my_car_2.describe()) # Calling method using the dot notation
```

## Dunder- (Magic-)Methoden

- **Definition**: Spezielle Methoden, die mit einem doppelten Unterstrich beginnen und enden (z. B. `__init__`, `__len__`, `__str__`, `__eq__`). Python verwendet sie intern für eingebaute Operationen.

```python
class Book:
    def __init__(self, title, pages):
        self.title = title
        self.pages = pages

    def __len__(self):
        return self.pages

    def __str__(self):
        return f"'{self.title}' has {self.pages} pages"

    def __eq__(self, other):
        return self.pages == other.pages

book1 = Book('Built Wealth Like a Boss', 420)
print(len(book1))        # 420
print(str(book1))        # 'Built Wealth Like a Boss' has 420 pages
```

- **Dunder-Methoden indirekt aufrufen**: Sie müssen Dunder-Methoden nicht direkt aufrufen. Stattdessen ruft Python sie automatisch auf, wenn bestimmte Aktionen stattfinden. Diese Operationen umfassen:

  - **arithmetische Operationen wie Addition, Subtraktion, Multiplikation, Division und andere**. Zusätzlich wird `__add__()` aufgerufen, `__sub__()` für Subtraktion, `__mul__()` für Multiplikation und `__truediv__()` für Division.
        
  - **String-Operationen wie Verkettung, Wiederholung, Formatierung und Umwandlung in Text**. `__add__()` wird für Verkettung aufgerufen, `__mul__()` für Wiederholung, `__format__()` für Formatierung, `__str__()` und `__repr__()` für Textumwandlung und so weiter.
        
  - **Vergleichsoperationen wie Gleichheit, Kleiner-als, Größer-als und andere**. `__eq__()` wird für Gleichheitsprüfungen aufgerufen, `__lt__()` für Kleiner-als, `__gt__()` für Größer-als und so weiter.
        
  - **Iterierbare Operationen wie das Iterierbar-Machen eines Objekts und das Voranschreiten durch Elemente**. `__iter__()` wird aufgerufen, um einen Iterator zurückzugeben, und `__next__()`, um das nächste Element abzurufen.

## Praxisbeispiel: Einkaufswagen

- **Warenkorb-Klasse mit Dunder-Methoden**: Ermöglicht das Hinzufügen, Entfernen, Iterieren und Überprüfen des Inhalts mit eingebautem Verhalten.

```python
class Cart:
    def __init__(self):
        self.items = []

    def add(self, item):
        self.items.append(item)

    def remove(self, item):
        if item in self.items:
            self.items.remove(item)
        else:
            print(f'{item} is not in cart')

    def list_items(self):
        return self.items

    def __len__(self):
        return len(self.items)

    def __getitem__(self, index):
        return self.items[index]

    def __contains__(self, item):
        return item in self.items

    def __iter__(self):
        return iter(self.items)

cart = Cart()
cart.add('Laptop')
print(len(cart))        # 1
print('Laptop' in cart) # True
```
