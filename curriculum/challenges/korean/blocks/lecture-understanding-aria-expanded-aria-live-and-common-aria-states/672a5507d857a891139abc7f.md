# --interactive--

`aria-controls` 속성은 페이지에서 다른 요소의 존재를 제어하는 요소와 제어되는 요소 사이에 프로그래밍적 관계를 만드는데 사용됩니다. 이 관계는 화면 읽기 프로그램 사용자가 제어 방식에 대해 더 잘 이해하도록 도와줍니다. 일반적인 사용 예로는 서로 다른 콘텐츠 섹션의 가시성을 제어하는 탭 세트나 메뉴의 가시성을 전환하는 버튼이 있습니다.

어떻게 작동하는지 보기 위해 예제를 살펴보겠습니다. 이 탭 인터페이스 예제에서는 세 개의 버튼이 있는 `div` 요소가 있습니다:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<div role="tablist">
  <button role="tab" id="tab1" aria-controls="section1" aria-selected="true">
    Tab 1
  </button>
  <button role="tab" id="tab2" aria-controls="section2" aria-selected="false">
    Tab 2
  </button>
  <button role="tab" id="tab3" aria-controls="section3" aria-selected="false">
    Tab 3
  </button>
</div>
```


`div`는 `role="tablist"`를 사용해 탭 그룹의 컨테이너 요소 역할을 한다고 나타냅니다.

각 버튼은 탭을 나타내며 `role` 속성이 `tab`로 설정되어 있습니다. 대부분의 탭 인터페이스에서 첫 번째 탭 패널은 기본적으로 보이므로, 첫 번째 탭 버튼에는 `aria-selected` 속성이 `true`로 설정되어 있어 해당 콘텐츠 섹션이 현재 보인다는 것을 나타냅니다. `aria-controls` 속성은 각 버튼을 제어하는 콘텐츠 섹션과 연결하는 데 사용됩니다.

다음은 탭에 대응하는 세 개의 콘텐츠 섹션입니다:

:::interactive_editor


```js
const tabs = document.querySelectorAll('[role="tab"]');
const tabList = document.querySelector('[role="tablist"]');

tabs.forEach(tab => {
  tab.addEventListener('click', () => {
    activateTab(tab);
  });

  tab.addEventListener('keydown', (e) => {
    const key = e.key;
    const currentIndex = Array.from(tabs).indexOf(tab);
    let newIndex = null;

    if (key === 'ArrowRight') {
      newIndex = (currentIndex + 1) % tabs.length;
    } else if (key === 'ArrowLeft') {
      newIndex = (currentIndex - 1 + tabs.length) % tabs.length;
    } else if (key === 'Home') {
      newIndex = 0;
    } else if (key === 'End') {
      newIndex = tabs.length - 1;
    }

    if (newIndex !== null) {
      tabs[newIndex].focus();
      activateTab(tabs[newIndex]);
    }
  });
});

function activateTab(tab) {
  const tabPanels = document.querySelectorAll('[role="tabpanel"]');

  tabs.forEach(t => {
    t.setAttribute('aria-selected', 'false');
    t.setAttribute('tabindex', '-1');
  });

  tabPanels.forEach(panel => panel.hidden = true);

  tab.setAttribute('aria-selected', 'true');
  tab.removeAttribute('tabindex');

  const panelId = tab.getAttribute('aria-controls');
  const panel = document.getElementById(panelId);
  panel.hidden = false;
  tab.focus();
}

```

:::

각 콘텐츠 섹션에는 `role` 속성이 `tabpanel`로 설정되어 있고, 접근 가능한 이름을 부여하기 위해 해당 탭을 가리키는 `aria-labelledby` 속성이 있습니다. `hidden` 속성은 현재 선택되지 않은 콘텐츠 섹션을 숨기는 데 사용됩니다. 각 섹션 ID는 해당 버튼의 `aria-controls` 속성 값과 일치합니다. `section1` ID는 첫 번째 버튼의 `aria-controls` 속성과, `section2`는 두 번째 버튼의 `aria-controls` 속성과, `section3`는 세 번째 버튼의 `aria-controls` 속성과 일치합니다. 이렇게 해서 탭과 콘텐츠 섹션 간의 연결이 설정됩니다.

다른 섹션으로 전환하려면 JavaScript를 사용해 현재 보이는 섹션에 따라 섹션의 `hidden` 속성과 버튼의 `aria-selected` 속성을 업데이트해야 합니다. 한 번에 하나의 섹션만 보일 수 있으며, 이 섹션에는 `hidden` 속성이 없어야 하고 버튼에는 `aria-selected`가 `true`로 설정되어야 합니다. 나머지 숨겨진 섹션들은 모두 `hidden` 속성이 있어야 하며, 버튼에는 `aria-selected`가 `false`로 설정되어야 합니다.

탭은 `aria-controls` 속성의 일반적인 사용 사례지만, 한 요소가 다른 요소의 가시성이나 동작을 제어하는 다른 상황에서도 사용할 수 있습니다. 예를 들어 아코디언, 드롭다운 메뉴, 모달 등이 있습니다.

다음에 요소의 가시성을 제어하는 사용자 인터페이스를 설계할 때는 `aria-controls` 속성을 사용해 제어하는 요소와 제어되는 요소 사이의 관계를 설정하는 것을 고려하세요.
