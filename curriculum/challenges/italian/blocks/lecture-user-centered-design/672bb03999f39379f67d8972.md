# --interactive--

**NOTA**: Alcuni esempi interattivi potrebbero usare CSS che non hai ancora imparato. Non preoccuparti di capire tutto il codice. Lo scopo degli esempi è mostrarti anteprime di questi concetti di design per farti capire meglio come funzionano. Per vedere le anteprime, devi abilitare l’editor interattivo.

La progressive disclosure è un pattern di design usato per mostrare agli utenti solo il contenuto rilevante in base alla loro attività corrente e nascondere il resto. Questo serve a ridurre il carico cognitivo e rendere l’esperienza utente più intuitiva.

:::interactive_editor

```html
<div class="progressive-disclosure">

    Show More Details
  </button>
  <div id="extra-content" class="hidden" tabindex="-1">

      Here are additional details that are revealed only when you want to see them.
```

```css
  }
```

Per esempio, quando usi la pagina di ricerca di Google, la maggior parte delle tue ricerche sarà semplice. Quindi ha senso avere un’interfaccia utente semplice con pochissime opzioni. Ma per le poche volte in cui ti serve una ricerca più avanzata, Google offre un’opzione di ricerca avanzata che rivela più opzioni.

Puoi trovare l’opzione di ricerca avanzata di Google dal menu delle impostazioni nella homepage di Google. Oppure puoi andare a `google.com/advanced_search`.

Un altro esempio di progressive disclosure è il pulsante “Più dettagli” sui prodotti in un sito di e-commerce. Quando visiti un sito come Amazon, vedrai una lista di immagini di prodotti in diverse categorie. Questo permette agli utenti di fare acquisti con facilità senza essere sopraffatti da troppe informazioni.

Se l’utente è interessato a un prodotto in particolare, può cliccare sull’immagine del prodotto per vedere più dettagli. Mostrare tutte le informazioni subito potrebbe essere troppo per l’utente. Quindi, usando la progressive disclosure, l’utente può scegliere di vedere più informazioni quando è pronto.

```js
document.querySelectorAll('.details-btn').forEach(button => {
  button.addEventListener('click', () => {
    const product = button.closest('.product');
    const details = product.querySelector('.product-details');
    const isHidden = details.classList.toggle('hidden');
    product.setAttribute('aria-expanded', !isHidden);
    if (!isHidden) {
      details.focus();
      button.textContent = 'Hide details';
    } else {
      button.textContent = 'More details';
    }
  });
});
```

Allora, quali sono alcune buone pratiche per progettare con la tecnica della progressive disclosure? La prima considerazione è mantenere tutte le informazioni importanti visibili in ogni momento. Gli utenti non dovrebbero faticare a trovare informazioni chiave perché sono nascoste da funzionalità avanzate extra.

Un’altra considerazione è fornire un unico punto di accesso per gli utenti per consultare funzionalità o informazioni aggiuntive. Questo può essere un pulsante o un collegamento sempre visibile sulla pagina. Aggiungere più punti di accesso può confondere gli utenti e portare a una cattiva esperienza utente.
