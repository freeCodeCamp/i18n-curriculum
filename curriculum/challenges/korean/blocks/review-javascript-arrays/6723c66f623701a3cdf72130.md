## JavaScript 배열 기본

- **정의**: JavaScript 배열은 숫자 인덱스로 식별되는 값들의 순서 있는 모음입니다. JavaScript 배열의 값들은 숫자, 문자열, 불리언, 객체, 심지어 다른 배열 등 다양한 데이터 유형일 수 있습니다. 배열은 메모리에서 연속적으로 저장되므로, 모든 요소가 하나의 연속된 메모리 블록에 저장되어 인덱싱이 효율적이고 인덱스를 통한 요소 접근이 빠릅니다.

```js
const developers = ["Jessica", "Naomi", "Tom"];
```

- **배열 요소 접근**: 배열에서 요소에 접근하려면 배열 이름 뒤에 대괄호 안에 인덱스 번호를 적어 참조해야 합니다. JavaScript 배열은 0부터 시작하는 인덱스를 사용하므로 첫 번째 요소는 인덱스 0, 두 번째 요소는 인덱스 1에 있습니다. 존재하지 않는 인덱스에 접근하면 JavaScript는 `undefined`를 반환합니다.

:::interactive_editor

```js
const developers = ["Jessica", "Naomi", "Tom"];
console.log(developers[0]) // "Jessica"
console.log(developers[1]) // "Naomi"

console.log(developers[10]) // undefined
```

:::

- **`length` 속성**: 이 속성은 배열에 들어있는 항목 수를 반환하는 데 사용됩니다.

:::interactive_editor

```js
const developers = ["Jessica", "Naomi", "Tom"];
console.log(developers.length) // 3
```

:::

- **배열 요소 업데이트**: 배열의 요소를 업데이트하려면 대입 연산자(`=`)를 사용해 특정 인덱스의 요소에 새 값을 할당합니다.

:::interactive_editor

```js
const fruits = ['apple', 'banana', 'cherry'];
fruits[1] = 'blueberry';

console.log(fruits); // ['apple', 'blueberry', 'cherry']
```

:::

## 2차원 배열  

- **정의**: 2차원 배열은 본질적으로 배열의 배열입니다. 체스판, 스프레드시트, 이미지의 픽셀처럼 자연스러운 격자 구조를 나타내는 데 사용됩니다. 2차원 배열의 요소에 접근하려면 행과 열을 나타내는 두 개의 인덱스가 필요합니다.

:::interactive_editor

```js
const chessboard = [
    ['R', 'N', 'B', 'Q', 'K', 'B', 'N', 'R'],
    ['P', 'P', 'P', 'P', 'P', 'P', 'P', 'P'],
    [' ', ' ', ' ', ' ', ' ', ' ', ' ', ' '],
    [' ', ' ', ' ', ' ', ' ', ' ', ' ', ' '],
    [' ', ' ', ' ', ' ', ' ', ' ', ' ', ' '],
    [' ', ' ', ' ', ' ', ' ', ' ', ' ', ' '],
    ['p', 'p', 'p', 'p', 'p', 'p', 'p', 'p'],
    ['r', 'n', 'b', 'q', 'k', 'b', 'n', 'r']
];

console.log(chessboard[0][3]); // "Q"
```

:::

## 배열 구조 분해  

- **정의**: 배열 구조 분해는 JavaScript에서 배열의 값을 추출해 더 간결하고 읽기 쉬운 방식으로 변수에 할당할 수 있게 하는 기능입니다. 배열 요소를 개별 변수로 풀어내는 편리한 구문을 제공합니다.

:::interactive_editor

```js
const fruits = ["apple", "banana", "orange"];

const [first, second, third] = fruits;

console.log(first); // "apple"
console.log(second); // "banana"
console.log(third); // "orange"
```

:::

- **나머지 구문**: 구조 분해되지 않은 배열의 나머지 요소들을 새 배열로 캡처할 수 있게 합니다.

:::interactive_editor

```js
const fruits = ["apple", "banana", "orange", "mango", "kiwi"];
const [first, second, ...rest] = fruits;

console.log(first); // "apple"
console.log(second); // "banana"
console.log(rest); // ["orange", "mango", "kiwi"]
```

:::

## 자주 쓰이는 배열 메서드 

- **`push()` 메서드**: 이 메서드는 배열 끝에 요소를 추가하며 새 배열 길이를 반환합니다.

:::interactive_editor

```js
const desserts = ["cake", "cookies", "pie"];
desserts.push("ice cream");

console.log(desserts); // ["cake", "cookies", "pie", "ice cream"];
```

:::

- **`pop()` 메서드**: 이 메서드는 배열에서 마지막 요소를 제거하고 제거된 요소를 반환합니다. 배열이 비어 있으면 반환값은 `undefined`입니다.

:::interactive_editor

```js
const desserts = ["cake", "cookies", "pie"];
desserts.pop();

console.log(desserts); // ["cake", "cookies"];
```

:::

- **`shift()` 메서드**: 이 메서드는 배열에서 첫 번째 요소를 제거하고 제거된 요소를 반환합니다. 배열이 비어 있으면 반환값은 `undefined`입니다.

:::interactive_editor

```js
const desserts = ["cake", "cookies", "pie"];
desserts.shift();

console.log(desserts); // ["cookies", "pie"];
```

:::

- **`unshift()` 메서드**: 이 메서드는 배열 앞에 요소를 추가하며 새 배열 길이를 반환합니다.

:::interactive_editor

```js
const desserts = ["cake", "cookies", "pie"];
desserts.unshift("ice cream");

console.log(desserts); // ["ice cream", "cake", "cookies", "pie"];
```

:::

- **`indexOf()` 메서드**: 이 메서드는 배열 내 특정 요소의 첫 번째 인덱스를 찾는 데 유용합니다. 요소를 찾지 못하면 `-1`을 반환합니다.

:::interactive_editor

```js
const fruits = ["apple", "banana", "orange", "banana"];
const index = fruits.indexOf("banana");

console.log(index); // 1
console.log(fruits.indexOf("not found")); // -1
```

:::

- **`splice()` 메서드**: 이 메서드는 배열의 어느 위치에서든 요소를 추가하거나 제거하는 데 사용됩니다. `splice()` 메서드의 반환값은 배열에서 제거된 항목들의 배열입니다. 아무것도 제거하지 않으면 빈 배열을 반환합니다. 이 메서드는 원본 배열을 직접 수정하며 새 배열을 만들지 않습니다. 첫 번째 인수는 배열 수정 시작 인덱스, 두 번째 인수는 제거할 요소 수, 그 다음 인수들은 추가할 요소들입니다.

:::interactive_editor

```js
const colors = ["red", "green", "blue"];
colors.splice(1, 0, "yellow", "purple");

console.log(colors); // ["red", "yellow", "purple", "green", "blue"]
```

:::

- **`includes()` 메서드**: 이 메서드는 배열에 특정 값이 포함되어 있는지 확인합니다. 배열에 해당 요소가 있으면 `true`를, 없으면 `false`를 반환합니다.

:::interactive_editor

```js
const programmingLanguages = ["JavaScript", "Python", "C++"];

console.log(programmingLanguages.includes("Python")); // true
console.log(programmingLanguages.includes("Perl")); // false
```

:::

- **`concat()` 메서드**: 이 메서드는 두 개 이상의 배열을 병합해 새 배열을 만듭니다.

:::interactive_editor

```js
const programmingLanguages = ["JavaScript", "Python", "C++"];
const newList = programmingLanguages.concat("Perl");

console.log(newList); // ["JavaScript", "Python", "C++", "Perl"]
```

:::

- **`slice()` 메서드**: 이 메서드는 원본 배열의 일부 구간을 시작과 끝 인덱스로 지정해 얕은 복사본을 포함하는 새 배열을 반환합니다. 새 배열은 원본 배열과 같은 요소를 참조합니다(복제본이 아님). 즉, 요소가 원시값(숫자나 문자열)일 경우 값이 복사되고, 객체나 배열일 경우 참조가 복사됩니다.

:::interactive_editor

```js
const programmingLanguages = ["JavaScript", "Python", "C++"];
const newList = programmingLanguages.slice(1);

console.log(newList); // ["Python", "C++"]
```

:::

- **전개 구문**: 전개 구문은 배열의 얕은 복사본을 만드는 데 사용됩니다.

:::interactive_editor

```js
const originalArray = [1, 2, 3];
const shallowCopiedArray = [...originalArray];

shallowCopiedArray.push(4);

console.log(originalArray); // [1, 2, 3]
console.log(shallowCopiedArray); // [1, 2, 3, 4]
```

:::

- **`split()` 메서드**: 이 메서드는 문자열을 구분자를 기준으로 나누어 부분 문자열 배열을 만듭니다. 구분자가 없으면 원본 문자열을 단일 요소로 포함하는 배열을 반환합니다.

:::interactive_editor

```js
const str = "hello";
const charArray = str.split("");

console.log(charArray); // ["h", "e", "l", "l", "o"]
```

:::

- **`reverse()` 메서드**: 이 메서드는 배열을 제자리에서 뒤집습니다.

:::interactive_editor

```js
const desserts = ["cake", "cookies", "pie"];
console.log(desserts.reverse()); // ["pie", "cookies", "cake"]
```

:::

- **`join()` 메서드**: 이 메서드는 배열의 모든 요소를 지정한 구분자로 연결해 하나의 문자열로 만듭니다. 구분자가 없거나 빈 문자열(`""`)이면 구분자 없이 연결합니다.

:::interactive_editor

```js
const reversedArray = ["o", "l", "l", "e", "h"];
const reversedString = reversedArray.join("");

console.log(reversedString); // "olleh"
```

:::
