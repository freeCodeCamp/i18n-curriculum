# --description--

Abbiamo parlato un po’ di Git e GitHub, quindi ora possiamo iniziare a mettere tutto insieme. Per farlo, ti serve un repository. Ma cos’è un repository? Puoi pensare a un repository come a un contenitore per un progetto: se stai lavorando a un’app, conserverai i file di quell’app insieme in un repository.

I repository possono essere locali sul tuo computer oppure remoti su un servizio come GitHub. Per creare un repository, puoi partire sia da locale sia da remoto: il risultato sarà lo stesso, quindi scegli l’approccio che si adatta meglio al tuo flusso di lavoro. Se inizi con un repository remoto, puoi crearne uno direttamente dal sito di GitHub seguendo queste istruzioni:

Visita <a href="https://github.com" target="_blank">il sito di GitHub</a>, assicurati di aver effettuato l’accesso, poi clicca sull’icona del più in alto a destra nella barra di navigazione. Vedrai diverse opzioni, ma quella che ti interessa è "New Repository". Selezionala per aprire l’interfaccia per creare un nuovo repository.

La prima opzione che vedrai è la possibilità di scegliere un template. Probabilmente non ne hai ancora uno configurato, ma un template è un tipo speciale di repository che puoi usare come base per i tuoi nuovi repository. Sono utili per includere documentazione o modelli di issue in tutti i tuoi progetti.

La tua opzione successiva è scegliere il proprietario: puoi selezionare il tuo account utente o qualsiasi organizzazione di cui fai parte o a cui hai accesso. Poi puoi dare un nome al repository: in generale, è meglio usare un nome che descriva sinteticamente il progetto che stai costruendo. Come buona pratica, evita gli spazi nel nome del repository. GitHub li sostituirà automaticamente con trattini.

Poi puoi opzionalmente aggiungere una descrizione che apparirà nella pagina principale del repository.

Puoi anche scegliere se il repository sarà pubblico o privato. Un repository pubblico può essere visualizzato e scaricato da chiunque. Un repository privato può essere consultato solo da te e da chi gli concedi esplicitamente l’accesso.

Infine, se non selezioni un template, vedrai le opzioni per generare automaticamente dei file nel repository. Un file README apparirà nella homepage del repository ed è un ottimo modo per fornire documentazione e descrizioni del progetto. Un file `.gitignore` ti permette di specificare i file che Git "ignorerà" — è utile per cose come dipendenze di pacchetti, file di ambiente e altre cose che non vuoi salvare. GitHub offre dei template per questo file, che escludono gli elementi comuni per il linguaggio o runtime specificato. E una licenza è un documento legale che definisce chi può usare il tuo software e cosa può farne.

Quando hai configurato tutte le impostazioni come preferisci, puoi cliccare il pulsante "Create Repository" per completare il processo. Verrai automaticamente reindirizzato alla pagina di destinazione del tuo nuovo repository!

Da qui, puoi cliccare il pulsante "Code" per ottenere le opzioni per clonare il repository sul tuo computer.

Puoi ignorare le schede "Codespaces" e "Copilot" per questa lezione. Invece, ci concentreremo sulle tre opzioni nella scheda "Local". Le prime due, HTTPS e SSH, si usano allo stesso modo. Usi il comando `git clone` con l’URL fornito nel riquadro per clonare il repository remoto sul tuo computer. La differenza tra queste due è nei metodi di connessione e autenticazione.

Per clonare un repository pubblico, in realtà non serve autenticarsi. Ma per clonare un repository privato, o per fare push sia su repository pubblici sia privati, devi essere autenticato. Un URL remoto HTTPS ti autentica tramite il tuo nome utente e password GitHub. Tuttavia, per motivi di sicurezza questo non è più supportato, e ora devi usare un nome utente e un token di accesso. Anche così, è un metodo molto meno sicuro rispetto a SSH, che usa le tue chiavi private e pubbliche. Questo ha il vantaggio aggiuntivo di farti accedere automaticamente, senza dover fornire un token ogni volta. Ci sarà una lezione separata che approfondirà SSH e come funziona, ma puoi anche leggere la documentazione di GitHub su come configurare SSH.

Per clonare il repository, puoi eseguire questo comando nel terminale:

```sh
git clone git@github.com:your-username/your-repository-name
```

Questo creerà una cartella sul tuo computer con il nome del repository e conterrà tutti i file del repository al suo interno. Da lì puoi `cd` (cambiare directory) al repository clonato con:

```sh
cd name-of-cloned-repo
```

Inoltre, la copia locale del tuo repository avrà un remoto che punta alla copia su GitHub. Per controllare i remoti puoi eseguire questo comando:

```sh
git remote -v
```

Un’altra opzione per clonare il repository è usare la CLI di GitHub. Questo strumento serve per fare attività specifiche di GitHub senza uscire dalla riga di comando. Se non l’hai installata, puoi trovare le istruzioni nella documentazione di GitHub — ma dovrebbe essere disponibile nel gestore di pacchetti del tuo sistema.

Per clonare un repository con la CLI, useresti:

```sh
gh repo clone your-username/your-repository-name
```

Il risultato finale di questi comandi è lo stesso: avrai una copia del repository localmente.

Puoi anche usare la CLI di GitHub per creare un repository:

```sh
gh repo create
```

Se non passi argomenti, la CLI ti offrirà una procedura guidata interattiva per configurare il repository. Questa procedura ti darà tutte le stesse opzioni dell’interfaccia web. Con questo strumento, avrai anche l’opzione di clonare subito il nuovo repository, così non dovrai seguire manualmente i passaggi descritti prima.

Ora hai una comprensione di base di come funzionano i repository e di come crearne uno per iniziare a versionare correttamente il tuo lavoro. Nelle lezioni future imparerai come fare push di un repository locale esistente su GitHub e come inviare modifiche ai repository remoti.
