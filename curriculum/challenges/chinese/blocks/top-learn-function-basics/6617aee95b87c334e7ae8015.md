# --description--

如果调用函数时未提供参数，则该参数将被赋值为 `undefined`。如果函数未设计为处理此分支，可能会导致意外结果。

```js
function favoriteAnimal(animal) {
  return animal + " is my favorite animal!"
}

console.log(favoriteAnimal()); // Output: undefined is my favorite animal!
```

在上面的示例中，函数 `favoriteAnimal` 在没有参数的情况下被调用。参数 `animal` 被赋值为 `undefined`，然后与字串 `" is my favorite animal!"` 连接。

如果未提供值，你可以使用默认参数。默认参数允许你为参数赋值一个默认值，如果调用函数时未提供该值。

```js
function favoriteAnimal(animal = 'Giant Panda') {
  return animal + " is my favorite animal!"
}
```

在上面的示例中，参数 `animal` 的默认值被设置为 `'Giant Panda'`。如果调用函数 `favoriteAnimal` 时没有传入参数，参数 `animal` 将被赋值为 `'Giant Panda'`。

## --text--

JavaScript 函数中的默认参数如何增强像 `favoriteAnimal` 这样的函数在没有任何参数调用时的行为？

## --answers--

如果未提供参数，它们会导致函数抛出误差。

---

它们防止函数在没有所需数量参数的情况下被调用。

---

它们确保函数返回 `null`，以表示没有传递参数。

---

它们允许函数使用预定义值执行，避免与 `undefined` 相关的问题。
