# --description--

الحفاظ على تحديث التبعيات ومعرفة كيفية تثبيت إصدار محدد مهم لاستقرار المشروع. يساعدك ذلك على تجنب الأخطاء البرمجية غير المتوقعة ويمنحك تحكمًا أكبر عند صدور إصدارات جديدة.

لتثبيت إصدار محدد من حزمة، استخدم الصيغة التالية:

```bash
npm install package@version-number
```

على سبيل المثال، إذا كان مشروعك يحتاج إلى إصدار أقدم من Express، مثل `4.21.2`، يمكنك تثبيته بهذه الطريقة:

```bash
npm install express@4.21.2
```

إذا لم تُدرج رقم الإصدار، سيقوم npm تلقائيًا بتثبيت أحدث إصدار متاح.

إذا لم تكن متأكدًا من الإصدارات المتوفرة، يمكنك التحقق منها باستخدام هذا الأمر:

```bash
npm view package-name versions
```

على سبيل المثال:

```bash
npm view express versions
```

لتحديث الحزم، يمكنك تحديثها بشكل فردي أو كلها مرة واحدة. قبل ذلك، من المفيد أولًا معرفة أيها قديم:

```bash
npm outdated
```

سيُظهر هذا أي تبعيات لديها إصدارات أحدث متاحة، ويجب أن ترى مخرجات مشابهة لهذا:

```bash
user@Kolade ~/Desktop/fCC/script-code/node/npm/install-spec-version  % npm outdated
Package  Current  Wanted  Latest  Location              Depended by
chalk      5.5.0   5.5.0   5.6.2  node_modules/chalk    install-spec-version
cleave     1.0.1   1.0.1   1.0.2  node_modules/cleave   install-spec-version
express   4.21.2  4.21.2   5.2.1  node_modules/express  install-spec-version
```

بعد التحقق من التبعيات القديمة، يمكنك البدء بتحديثها.

لتحديث تبعية واحدة، استخدم الأمر التالي:

```bash
npm update package-name
```

هذا يحدث الحزمة فقط ضمن نطاق الإصدار المسموح به في ملف `package.json`. على سبيل المثال، لتحديث Express، يمكنك استخدام الأمر التالي:

```bash
npm update express
```

إذا أردت تخطي نطاق الإصدار وتثبيت أحدث إصدار من حزمة، استخدم الأمر التالي:

```bash
npm install package-name@latest
```

لتحديث كل التبعيات التي تقع ضمن نطاقات الإصدارات المسموح بها، نفذ الأمر:

```bash
npm update
```

لكن لاحظ أن `npm update` لا يرقّي الحزمة إلى إصدار رئيسي جديد إذا لم يسمح به ملف `package.json`. الإصدارات الرئيسية غالبًا ما تحتوي تغييرات جذرية، لذا npm لا يطبقها تلقائيًا.

إذا أردت تحديدًا أحدث إصدار رئيسي، يجب أن تنفذ الأمر:

```bash
npm install package-name@latest
```

هذا يجبر npm على تثبيت أحدث إصدار، بغض النظر عن نطاق الإصدار في ملف `package.json`.
