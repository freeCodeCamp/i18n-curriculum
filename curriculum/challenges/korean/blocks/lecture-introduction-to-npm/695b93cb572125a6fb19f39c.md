# --description--

이전 수업에서 의존성이 무엇인지, 그리고 프로젝트에 적합한 의존성을 어떻게 선택하는지 배웠습니다.

이번 수업에서는 npm을 사용해 의존성을 설치하고 삭제하는 방법을 배우세요.

의존성을 설치하려면 다음과 같이 `npm install` 명령어를 사용하세요:

```bash
npm install dependencyName
```

또한 `install`을 `i`로 줄여서 사용할 수도 있습니다:

```bash
npm i dependencyName
```

패키지를 설치하면 npm이 `dependencies` 파일에 새로운 `package.json` 섹션을 추가하고(이미 없을 경우) 설치한 패키지를 목록에 표시합니다. 예를 들어 `chalk` 패키지를 설치하면 다음과 같습니다:

```json
{
  "name": "install-uninstall-deps",
  "version": "1.0.0",
  "description": "a simple demo on how to install and uninstall dependencies",
  "main": "index.js",
  "scripts": {
    "test": "echo \"Error: no test specified\" && exit 1"
  },
  "keywords": ["npm", "deps"],
  "author": "Kolade Chris",
  "license": "ISC",
  "dependencies": {
    "chalk": "^5.6.2"
  }
}
```

의존성을 설치하면 프로젝트 루트 폴더에 두 가지 새 항목이 생깁니다:

* `node_modules/`: 설치한 패키지의 실제 코드를 담고 있는 폴더
    
* `package-lock.json`: 해당 패키지들의 정확한 버전과 자체 의존성 정보를 저장하는 파일 (이 파일에 대해서는 나중 수업에서 더 자세히 배웁니다)
    

<img src="https://cdn.freecodecamp.org/curriculum/lecture-transcripts/how-can-you-install-and-remove-dependencies-1.png" alt="">

대부분 프로젝트는 한 개 이상의 의존성이 필요합니다. 하나씩 설치하는 대신, npm은 공백으로 구분된 패키지 이름 목록을 입력해 여러 패키지를 한꺼번에 설치할 수 있게 해줍니다:

```bash
npm install express lodash bcrypt
```

앞과 마찬가지로 여러 의존성을 한꺼번에 설치하면 npm이 `package.json`과 `package-lock.json` 파일, 그리고 `node_modules`을 업데이트합니다. `package.json`은 다음과 같습니다:

```json
{
  "name": "install-uninstall-deps",
  "version": "1.0.0",
  "description": "a simple demo on how to install and uninstall dependencies",
  "main": "index.js",
  "scripts": {
    "test": "echo \"Error: no test specified\" && exit 1"
  },
  "keywords": ["npm", "deps"],
  "author": "Kolade Chris",
  "license": "ISC",
  "dependencies": {
    "bcrypt": "^6.0.0",
    "chalk": "^5.6.2",
    "express": "^5.1.0",
    "lodash": "^4.17.21"
  }
}
```

`devDependency`를 설치하려면 `--save-dev` 명령어에 `install` 플래그를 추가하세요. 더 짧은 `-D` 버전도 사용할 수 있습니다:

```bash
npm i nodemon -D
```

이 명령어를 실행하면 npm이 `package.json` 파일을 업데이트하고 `devDependencies` 섹션 아래에 패키지를 추가합니다. 예시는 다음과 같습니다:

```json
{
  "name": "install-uninstall-deps",
  "version": "1.0.0",
  "description": "a simple demo on how to install and uninstall dependencies",
  "main": "index.js",
  "scripts": {
    "test": "echo \"Error: no test specified\" && exit 1"
  },
  "keywords": ["npm", "deps"],
  "author": "Kolade Chris",
  "license": "ISC",
  "dependencies": {
    "bcrypt": "^6.0.0",
    "chalk": "^5.6.2",
    "express": "^5.1.0",
    "lodash": "^4.17.21"
  },
  "devDependencies": {
    "nodemon": "^3.1.10"
  }
}
```

`npm install` 명령어를 실행할 때마다 npm은 설치 성공 여부를 알려주는 메시지를 표시합니다. 성공 메시지는 보통 다음과 같습니다:


패키지를 삭제하거나 제거하려면 `npm uninstall` 명령어를 사용하세요:

```bash
npm uninstall dependencyName
```

예를 들어 프로젝트에서 `chalk`를 삭제하려면 다음 명령어를 실행합니다:

```bash
npm uninstall chalk
```

여러 패키지를 한꺼번에 삭제할 때도 각 패키지 이름 사이에 공백을 넣어 나열하면 됩니다:

```bash
npm uninstall express lodash bcrypt
```

삭제 명령어가 성공했는지 실패했는지 알려주는 메시지도 받게 됩니다:

```bash

```

그리고 `package.json` 파일도 자동으로 업데이트됩니다:

```json
{
  "name": "install-uninstall-deps",
  "version": "1.0.0",
  "description": "a simple demo on how to install and uninstall dependencies",
  "main": "index.js",
  "scripts": {
    "test": "echo \"Error: no test specified\" && exit 1"
  },
  "keywords": ["npm", "deps"],
  "author": "Kolade Chris",
  "license": "ISC",
  "devDependencies": {
    "nodemon": "^3.1.10"
  }
}
```

`devDependency`를 삭제하려면 다음 명령어를 실행하세요:

```bash
npm uninstall nodemon
```

삭제할 때는 `-D` 플래그가 필요 없다는 점을 기억하세요. npm이 자동으로 올바른 섹션, 이 경우 `devDependencies`에서 패키지를 제거합니다.

`devDependency`를 삭제한 후에는 `package.json`이 다시 업데이트됩니다:

```json
{
  "name": "install-uninstall-deps",
  "version": "1.0.0",
  "description": "a simple demo on how to install and uninstall dependencies",
  "main": "index.js",
  "scripts": {
    "test": "echo \"Error: no test specified\" && exit 1"
  },
  "keywords": ["npm", "deps"],
  "author": "Kolade Chris",
  "license": "ISC"
}
```
