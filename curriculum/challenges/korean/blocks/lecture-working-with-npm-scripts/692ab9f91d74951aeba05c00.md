# --description--

이제 npm 스크립트에 대해 배워보겠습니다.

본격적으로 시작하기 전에 몇 가지 중요한 개념을 복습해 보겠습니다:

* **npm**은 JavaScript 패키지를 설치하고 관리할 수 있는 명령줄 도구입니다.
    
* **패키지**는 관련된 코드를 논리적으로 묶은 디렉터리입니다.
    
* **스크립트**는 스크립팅 프로그래밍 언어로 작성된 명령어나 지시문의 수열입니다.
    

**npm 스크립트**는 애플리케이션의 `package.json` 파일에 정의할 수 있는 사용자 지정 명령어입니다.

이들은 프로젝트의 개발, 테스트, 배포 단계에서 반복적으로 사용할 작업과 워크플로를 자동화하는 데 유용합니다.

예를 들어, npm 스크립트의 한 가지 사용 사례는 전체 개발 환경을 시작하기 위해 `npm start`을 사용자 지정하는 것입니다. 개발 환경을 시작할 때마다 모든 명령어를 하나씩 수동으로 작성하는 대신, `npm start`를 실행하면 전체 과정이 자동으로 수행됩니다.

또한 npm 스크립트를 사용해 코드베이스 전반에 걸쳐 린팅을 자동화하거나 테스트 스위트를 실행하는 등 다양한 용도로 활용할 수 있습니다.

매우 강력하니, 기본 개념부터 살펴보겠습니다.

## package.json 파일 생성 방법

사용자 지정 npm 스크립트를 만들고 실행하려면 `package.json` 파일이 필요합니다.

`package.json` 파일은 Node.js 프로젝트에 필수적입니다. 이 파일에는 프로젝트의 메타데이터, 의존성, 스크립트가 포함되어 있습니다.

참고로, `package.json` 파일을 생성하려면 프로젝트 루트 디렉터리에서 `npm init` 명령어를 실행해야 합니다. 그런 다음 패키지 이름, 버전, 설명 등 세부 정보를 직접 입력하거나 빈칸으로 두어 기본값을 사용할 수 있습니다.

이 과정을 마치면 프로젝트 루트 디렉터리에 `package.json` 파일이 생성됩니다.

기본값을 모두 유지했다면, `package.json`을 코드 편집기로 열었을 때 다음과 같은 키-값 쌍을 볼 수 있습니다:

```json
{
  "name": "<project_name>",
  "version": "1.0.0",
  "main": "index.js",
  "scripts": {
    "test": "echo \"Error: no test specified\" && exit 1"
  },
  "author": "",
  "license": "ISC",
  "description": ""
}
```

여기 이 키-값 쌍을 주목하세요:

```json
"scripts": {
  "test": "echo \"Error: no test specified\" && exit 1"
}
```

여기에 `scripts` 키 아래에 npm 스크립트를 지정할 수 있습니다.

기본적으로 하나의 스크립트가 있는데, 바로 테스트 스크립트입니다:

```json
"test": "echo \"Error: no test specified\" && exit 1"
```

이것은 `package.json`에 정의된 npm 스크립트 예시입니다.

npm 스크립트를 정의하는 기본 구문은 다음과 같습니다:

```json
"<script_name>": "<commands>"
```

콜론 왼쪽에는 스크립트 이름이 있습니다. 일반적인 이름으로는 `run`, `start`, `test`, `build`, `dev`, `lint` 등이 있습니다. 이 이름들은 오른쪽에 있는 실제 명령어의 "단축키" 역할을 합니다.

콜론 오른쪽에는 실제 명령어가 있습니다.

예를 들어:

```json
"scripts": {
  "start": "node app.js"
}
```

스크립트를 실행하려면 명령줄에 다음을 입력하세요:

```bash
npm run <script_name>
```

이렇게 하면 `package.json`에 정의된 모든 사용자 지정 스크립트를 실행할 수 있습니다.

때때로 `run`라는 단어 없이 이 축약형을 사용해 스크립트를 실행할 수도 있습니다:

```bash
npm <script_name>
```

하지만 이 방법은 `start`, `test`, `stop`, `restart`라는 특정 npm 스크립트에만 적용됩니다.

예제로 돌아가서, `package.json`에 다음이 있다면:

```json
"scripts": {
  "start": "node app.js"
}
```

npm 스크립트를 실행하려면 명령줄에 `npm run start`를 입력해야 합니다. 이렇게 하면 실제로 실행되는 명령어는 `node app.js`입니다.

이 npm 스크립트에 축약형이 있으므로 `npm start`를 사용해도 결과는 같습니다.

이 경우 단순히 `node app.js` 명령어를 `npm start`로 바꾸는 것뿐입니다. 하지만 복잡한 명령어 수열을 순차적 또는 병렬로 실행해야 할 때 `npm start` 하나로 모두 실행할 수 있다면 시간을 크게 절약할 수 있겠죠? 바로 npm 스크립트의 진가가 발휘되는 부분입니다.

가장 자주 사용하는 npm 스크립트 몇 가지를 간단히 살펴보겠습니다:

* `npm run`: 사용자 지정 npm 스크립트를 실행할 때 사용합니다.
    
* `npm start`: 메인 애플리케이션이나 개발 서버를 시작할 때 사용합니다.
    
* `npm lint`: 코드 스타일 검사기를 실행해 코드가 스타일 표준과 모범 사례를 준수하는지 확인할 때 사용합니다.
    
* `npm test`: 프로젝트의 테스트 스위트를 실행할 때 사용합니다.
    
다양한 용도의 다른 npm 스크립트도 많지만, 이들이 가장 흔히 볼 수 있는 것들입니다.

또한, 관련된 스크립트를 같은 네임스페이스 아래 콜론으로 그룹화하는 관례가 있습니다. 예를 들어 `start:server`와 `start:ui`가 있습니다.

## 명령어를 순차적으로 실행하는 방법

여러 명령어를 순서대로 실행해야 할 때는 두 가지 방법이 있습니다. 논리 AND 연산자(`&&`)를 사용하거나 세미콜론(`;`)을 사용할 수 있습니다. 차이점을 살펴보겠습니다.

논리 AND 연산자는 첫 번째 명령어가 성공적으로 실행될 때만 다음 명령어를 실행합니다.

예를 들어:

```json
"scripts": {
  "build": "webpack --mode production",
  "start": "npm run build && node server.js"
}
```

`start` 명령어에 논리 AND 연산자가 있는 것을 확인하세요:

```json
"start": "npm run build && node server.js"
```

여기서 `npm start`는 먼저 `npm run build`를 실행하려 시도하고, 이 스크립트가 성공하면 `node server.js`가 실행됩니다.

이 방법은 신뢰성을 보장하고, 이전 명령어가 실패하거나 오류를 발생시키면 이후 명령어 실행에 시간과 자원을 낭비하지 않도록 하기 때문에 매우 흔히 사용됩니다.

반면, 세미콜론 연산자를 사용하면 이전 명령어 성공 여부와 상관없이 명령어를 순차적으로 실행합니다.

예제에서 이 간단한 변경은 명령어 실행 방식에 중요한 영향을 미칩니다:

```json
"scripts": {
  "build": "webpack --mode production",
  "start": "npm run build; node server.js"
}
```

다시 `start` 명령어에 주목하세요:

```json
"start": "npm run build; node server.js"
```

이제 `&&` 연산자 대신 세미콜론을 사용했으므로, `node server.js` 명령어는 `npm run build` 다음에 항상 실행됩니다. `node server.js`가 성공하지 못했더라도 마찬가지입니다.

```json
"start": "npm run build; node server.js"
```

## 명령어를 병렬로 실행하는 방법

명령어를 병렬로 실행해야 할 때는 `&` 연산자를 사용할 수 있습니다.

```json
"scripts": {
  "start:server": "node server/index.js",
  "start:client": "npm run build:watch",
  "dev": "npm run start:server & npm run start:client"
}
```

하지만 이 옵션은 프로세스를 백그라운드에서 실행한다는 점을 알아두세요.

즉, 같은 터미널 세션에서 실행하면 출력이 섞여서 이해하기 어려울 수 있습니다.

명령어를 병렬로 실행하는 다른 도구로는 `concurrently`와 `npm-run-all`가 있습니다.

## npm 스크립트에 인수 전달하는 방법

npm 스크립트 내에서 실행되는 명령어에 인수를 직접 전달해야 할 때는, 두 개의 하이픈(`--`) 뒤에 인수를 작성하면 됩니다.

`--`는 npm에게 그 뒤에 오는 모든 것이 npm 스크립트가 실행할 실제 명령어에 전달될 인수임을 알려줍니다.

일반 구문은 다음과 같습니다:

```bash
npm run <script_name> -- <argument_1> <argument_2> <argument_3>
```

예를 들어, `npm start` 스크립트를 정의했다고 가정해 보겠습니다:

```json
{
  "name": "project-example",
  "scripts": {
    "start": "node app.js"
  }
}
```

`npm start`를 사용할 때 서버를 실행할 포트를 어떻게 지정할 수 있을까요?

포트를 명령줄 인수로 직접 전달한다고 하면 이렇게 작성합니다:

```bash
npm start -- 8000
```

코드에서는 `process.argv` 배열의 인덱스를 사용해 이 인수에 접근할 수 있습니다:

```bash
process.argv[0] # The path to the node executable.
process.argv[1] # The script file being executed.
process.argv[2] # The first argument (8000).
```

따라서 `process.argv[2]`를 사용해 필요한 곳 어디서든 포트 번호를 접근하고 사용할 수 있습니다. 이것이 명령줄 인수와 npm 스크립트가 결합된 강력한 기능입니다.

보시다시피, npm 스크립트는 개발 워크플로를 단순화하는 데 필수적인 도구입니다. 복잡하고 재사용 가능한 명령어 수열을 간단하고 일관된 단축키로 자동화하고 변환해 일상 업무에서 활용할 수 있게 도와줍니다.
