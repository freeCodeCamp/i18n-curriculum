# --description--

Una **coda di priorità** è un tipo astratto di struttura dati (ADT) che funziona in modo simile a una coda o a uno stack, ma con una differenza fondamentale.

Come forse già sai, le code standard seguono un approccio FIFO (First-in, First-out), dove il primo elemento aggiunto alla coda è il primo a essere rimosso dalla coda.

Gli stack seguono un approccio LIFO (Last-in, First-out), dove l’ultimo elemento aggiunto allo stack è il primo a essere rimosso dallo stack.

Code e stack considerano solo l’ordine di inserimento degli elementi.

Tuttavia, le code di priorità tengono conto della "priorità" degli elementi. La priorità viene usata per determinare quale elemento deve essere rimosso per primo.

Di solito, l’elemento con la priorità più alta viene rimosso per primo, ma alcune implementazioni possono scegliere di rimuovere per prime le priorità più basse. Questo dipende dai requisiti del tuo programma.

Le code di priorità sono molto utili per applicazioni pratiche come trovare il percorso più breve tra due luoghi, pianificare attività nei sistemi operativi, simulare il traffico, comprimere dati e gestire reti.

In pratica, le code di priorità sono comunemente implementate usando una struttura dati heap.

Un **heap** è una struttura dati ad albero con una proprietà molto specifica chiamata **proprietà di heap**. Questa proprietà determina la relazione tra ogni nodo e i suoi figli, in base al tipo di heap.

Esistono due tipi principali di heap:

* Max-heap
    
* Min-heap
    

In un **max-heap**, il valore di ogni nodo è maggiore o uguale al valore dei suoi figli.

In questo esempio, puoi vedere una struttura ad albero con i nodi 8, 7, 5, 2 e 1. Nota che il nodo 7 è maggiore del nodo 2 e del nodo 1, seguendo la proprietà di heap. Questo vale per tutti gli altri nodi.

<img src="https://cdn.freecodecamp.org/curriculum/lecture-transcripts/how-do-priority-queues-and-heaps-work-1.png" alt="A max-heap tree structure showing nodes with values 8 at the root, 7 and 5 as children of 8, and 2 and 1 as children of 7, demonstrating that each parent node is greater than its children.">

Al contrario, in un **min-heap**, il valore di ogni nodo è minore o uguale al valore dei suoi figli.

In questo esempio, abbiamo nodi con valori 4, 7, 9, 12 e 15. Per esempio, il nodo 7 è minore del nodo 12 e del nodo 15, seguendo la proprietà di heap. Questo vale anche per tutti gli altri nodi.

<img src="https://cdn.freecodecamp.org/curriculum/lecture-transcripts/how-do-priority-queues-and-heaps-work-2.png" alt="A min-heap tree structure showing nodes with values 4 at the root, 7 and 9 as children of 4, and 12 and 15 as children of 7, demonstrating that each parent node is less than its children.">

La proprietà di heap è fondamentale perché garantisce che l’elemento massimo (o minimo) (a seconda del tipo di heap) rimanga sempre in cima, rendendo molto facile la sua rimozione.

In pratica, gli heap sono tipicamente implementati come array per consultare efficientemente i nodi genitori e figli.

Usare gli array semplifica la logica per consultare questi valori o "nodi" perché, dietro le quinte, se l’heap mantiene la struttura di un albero binario completo, l’implementazione con array richiede solo semplici operazioni matematiche basate sugli indici per trovare dove si trovano gli elementi in memoria.

Python ha un modulo `heapq` integrato che puoi usare per lavorare con un’implementazione di un min-heap.

Funziona operando direttamente sulle liste Python, seguendo passaggi specifici che trattano gli elementi come se la lista fosse un heap, preservando la proprietà di heap.

Per usare questo modulo, devi solo importarlo:

```python
import heapq
```

Poi, devi definire una lista vuota. Questa sarà la struttura dati sottostante per l’heap:

```python
my_heap = []
```

Per aggiungere elementi all’heap, chiameresti `heappush()`, passando il nome dell’heap e l’elemento che vuoi aggiungere come argomenti. Questo aggiungerà automaticamente l’elemento nella lista nel punto giusto, per preservare la proprietà di heap:

```python
heapq.heappush(my_heap, 9)
```

Per ottenere l’elemento con la priorità più alta (in questo caso, il valore più piccolo), chiameresti `heappop()`:

```python
heapq.heappop(my_heap)
```

`heappushpop()` combina entrambe le operazioni in una sola chiamata.

Questo è più efficiente che chiamarle separatamente in sequenza, specialmente quando l’heap è grande, perché esegue una sola operazione di "heapify" per riordinare la lista come heap:

```python
heapq.heappushpop(my_heap, 15)
```

Se hai già una lista e vuoi trasformarla in un heap, puoi chiamare `heapify()`, passando la lista come argomento:

```python
heapq.heapify(my_heap)
```

Ma attualmente stiamo ordinando gli elementi in base ai loro valori, giusto?

E se volessimo ordinarli invece in base alla loro "priorità"?

Potresti farlo archiviando tuple con questa struttura: `(priority, element)`.

Poiché le tuple vengono confrontate elemento per elemento da sinistra a destra, le priorità saranno confrontate per prime e le decisioni saranno basate su di esse.

Nota che, in questo caso, valori più bassi rappresentano priorità più alte. Questo significa che una tupla con priorità 1 avrà una priorità **più alta** di una tupla con priorità 3:

```python
my_heap = []

heapq.heappush(my_heap, (3, "A"))
heapq.heappush(my_heap, (2, "B"))
heapq.heappush(my_heap, (1, "C"))
```

Se hai bisogno che gli elementi con la stessa priorità vengano rimossi nell’ordine in cui sono stati inseriti, potresti considerare di includere un contatore univoco come secondo elemento della tupla per risolvere i casi di parità, così: `(priority, counter, element)`.

Ora parliamo dell’efficienza degli heap.

Le complessità temporali medie e nel peggior caso per inserire ed estrarre il valore minimo o massimo da un heap (a seconda del tipo di heap) sono logaritmiche, `O(log n)`, perché il numero di scambi richiesti è solitamente proporzionale all’altezza dell’heap, che è log(n).

La complessità temporale media e nel peggior caso per l’operazione di "peek" è costante, O(1). Peek significa ottenere il valore minimo o massimo (a seconda del tipo di heap) senza rimuoverlo.

L’operazione di "heapify", dove l’heap viene costruito da una lista non ordinata, ha complessità temporale lineare, O(n), nei casi medi e peggiori.

Allo stesso modo, sia la ricerca che la cancellazione di un elemento arbitrario hanno complessità temporali lineari medie e nel peggior caso di O(n), perché potenzialmente richiedono di attraversare tutto l’heap.

E quanta memoria richiedono?

La complessità spaziale dell’heap è lineare, O(n), dove `n` è il numero di elementi che contiene. Serve solo per archiviare gli elementi e un piccolo overhead aggiuntivo per l’oggetto lista stesso.

Code di priorità e heap sono molto importanti nell’informatica. Ti permettono di trovare e usare rapidamente gli elementi più importanti di una lista. Questa efficienza è cruciale per molti programmi che svolgono compiti reali critici, come trovare il percorso più veloce su una mappa.
