# --description--

Die `debugger`-Anweisung ist ein mächtiges JavaScript-Werkzeug, mit dem Sie Ihren Code an einer bestimmten Zeile anhalten können, um zu untersuchen, was im Programm vor sich geht. Wenn die `debugger`-Anweisung richtig verwendet wird, kann sie Ihnen viel Zeit sparen, um herauszufinden, warum etwas nicht so funktioniert, wie es sollte.

JavaScript führt Ihren Code von oben nach unten aus. Wenn JavaScript Ihren Code ausführt und auf eine `debugger`-Anweisung trifft, wird die Ausführung sofort an dieser Zeile angehalten. So haben Sie die Möglichkeit, Variablen zu überprüfen, Funktionen zu kontrollieren und den Ablauf des Codes im Allgemeinen zu untersuchen.

Dadurch können Sie genau sehen, was richtig oder falsch läuft. Dies geschieht jedoch nur, wenn die Entwicklerwerkzeuge Ihres Browsers geöffnet sind. Andernfalls wird die `debugger`-Anweisung ignoriert, und der Code läuft wie gewohnt weiter.

Außerdem pausieren moderne Browser bei Verwendung der `debugger`-Anweisung die Codeausführung an der angegebenen Zeile. Sie ermöglichen auch, die Ausführung durch Klicken auf die Wiedergabetaste fortzusetzen, aber die Seite selbst wird nicht automatisch neu geladen, es sei denn, dies wird manuell ausgelöst.

Um die `debugger`-Anweisung zu verwenden, platzieren Sie sie in der Zeile, an der die Codeausführung pausieren soll. Hier ist ein Beispiel:

```js
let firstNumber = 5;
let secondNumber = 10;
debugger; // Code execution pauses here
let sum = firstNumber + secondNumber;
console.log(sum);
```

Bei diesem Beispiel sehen Sie, wenn Sie die Konsole vor dem Ausführen des Codes nicht geöffnet haben und sie später öffnen, `15` darin. Während die Konsole geöffnet ist und Sie die Seite neu laden, wird die Ausführung für Sie pausiert, damit Sie den Code untersuchen können.

Hier ist ein komplexeres Beispiel:

```js
function calculateTotalPrice(price, discountPercentage) {
 debugger
 let discountAmount = (price * discountPercentage) / 100
 let totalPrice = price - discountAmount

 console.log(`Original Price: ${price}`)
 console.log(`Discount Amount: ${discountAmount}`)
 console.log(`Total Price after Discount: ${totalPrice}`)

 return totalPrice
}

let price = 100
let discount = 15

let finalPrice = calculateTotalPrice(price, discount)
console.log(`Final Price: ${finalPrice}`);
```

In diesem Beispiel wird die `debugger`-Anweisung am Anfang der `calculateTotalPrice`-Funktion platziert. Wenn die Funktion aufgerufen wird, wird die Ausführung der Funktion pausiert.

Wenn Sie die Seite mit geöffneter Konsole neu laden, sehen Sie, dass die Seite weiterhin neu lädt, was bestätigt, dass die Ausführung des Codes nur pausiert, aber nicht beendet wird. Wenn Sie möchten, dass die Ausführung fortgesetzt wird, können Sie auf die Wiedergabetaste klicken.

Während Sie Ihre JavaScript-Programme weiterentwickeln, testen Sie die `debugger`-Anweisung in Ihrem Code, um zu sehen, wie sie Ihnen helfen kann, den Ablauf der Ausführung besser zu verstehen, Variablen in Echtzeit zu überprüfen und schnell zu erkennen, wo etwas schiefgehen könnte.
