# --description--

미들웨어는 Express에서 필수적인 개념으로, 클라이언트와 서버 사이에서 요청과 응답을 처리할 수 있게 해줍니다. 미들웨어 함수는 요청 객체, 응답 객체, 그리고 애플리케이션의 요청-응답 주기에서 다음 함수에 접근할 수 있습니다.

자세히 살펴보겠습니다:

## 미들웨어란 무엇인가요?

미들웨어는 본질적으로 서버에 대한 요청의 생명주기 동안 실행되는 함수입니다. 요청 기록, 오류 처리, 들어오는 요청 구문 분석, 정적 파일 제공 등 다양한 작업을 수행합니다.

Express는 라우트 핸들러에 도달하기 전에 들어오는 요청을 처리하는 미들웨어를 정의하는 편리한 방법을 제공합니다.

미들웨어 작동 방식:

Express 애플리케이션에 요청이 들어오면, 미들웨어는 추가된 순서대로 실행됩니다. 흐름은 다음과 같습니다:

* 요청이 서버로 전송됩니다.
* 미들웨어 함수가 요청을 처리합니다.
* 미들웨어가 `next()`를 호출하면, 제어가 다음 미들웨어나 라우트 핸들러로 넘어갑니다.
* 미들웨어가 `next()`를 호출하지 않으면, 요청-응답 주기가 정지합니다.

```js
const express = require('express')
const app = express()

// Simple middleware that logs request info
app.use((req, res, next) => {
  console.log('Request URL:', req.url)
  console.log('Request Method:', req.method)
  next() // Passes control to the next middleware
})

app.get('/', (req, res) => {
  res.send('Hello World!')
})

app.listen(3000, () => {
  console.log('Server running on http://localhost:3000')
})
```

`app.use()`: 애플리케이션에 전역적으로 미들웨어를 추가합니다.

이 예제에서 미들웨어는 요청 세부 정보를 기록하고 `next()`를 호출하여 다음 미들웨어나 라우트 핸들러로 진행합니다.

미들웨어는 Express에서 요청을 가로채고 다양한 지점에서 수정할 수 있는 강력한 기능입니다. 미들웨어를 사용해 기록, 인증 처리, 또는 라우트 핸들러에 도달하기 전에 요청과 응답 객체를 수정할 수 있습니다.
