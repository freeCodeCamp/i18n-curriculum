# --description--

Парадигма розділяй і володарюй у комп’ютерних науках — це техніка рекурсивного розбиття задач на менші підзадачі. Одним із ключових аспектів цієї техніки є рекурсія, яка відбувається, коли функція викликає сама себе повторно, доки не буде досягнуто базового випадку. У цьому уроці ми розглянемо алгоритм сортування злиттям, щоб краще зрозуміти, як працює техніка розділяй і володарюй.

Припустимо, у нас є цей список чисел:

```md
42 37 53 17
```

Мета — відсортувати цей список від найменшого до найбільшого, використовуючи алгоритм сортування злиттям. Перший крок — розділити цей список навпіл:

```md
42 37 | 53 17
```

Потім потрібно подивитися на ліву частину списку:

```md
42 37
```

Ми беремо цей підсписок і знову ділимо навпіл, доки в кожному підсписку не залишиться лише один елемент:

```md
42 | 37
```

Список, що містить лише один елемент, за замовчуванням відсортований. Далі потрібно об’єднати кожен із цих підсписків з одним елементом у відсортований список:

```md
37 42
```

Потім повторюємо той самий процес для правої частини початкового списку:

```md
// right side of original list
53 17

// divide the list in half
53 | 17

// merge the lists in sorted order
17 53
```

Тепер, коли обидві половини початкового списку відсортовані, ми об’єднуємо ці дві половини разом і сортуємо елементи:

```markdown
17 37 42 53
```

Ось як виглядає цей алгоритм у коді:

```javascript
function mergeSort(arr) {
    if (arr.length <= 1) {
        return arr;
    }

    const mid = Math.floor(arr.length / 2);
    const left = mergeSort(arr.slice(0, mid));
    const right = mergeSort(arr.slice(mid));

    const sorted = [];
    let i = 0;
    let j = 0;

    while (i < left.length && j < right.length) {
        if (left[i] <= right[j]) {
            sorted.push(left[i]);
            i += 1;
        } else {
            sorted.push(right[j]);
            j += 1;
        }
    }

    return sorted.concat(left.slice(i)).concat(right.slice(j));
}
```

Часова складність алгоритму merge sort становить `O(n log n)`, оскільки список безперервно ділиться навпіл (`log n`), а потім об’єднується (`O(n)`). На відміну від інших алгоритмів сортування, таких як bubble sort, merge sort не є алгоритмом сортування на місці і має просторову складність `O(n)`.
