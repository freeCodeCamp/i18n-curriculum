## الأنواع الشائعة لرسائل الخطأ

- **SyntaxError**: تحدث هذه الأخطاء عندما تكتب شيئًا بشكل غير صحيح في الشفرة، مثل فقدان قوس أو قوس معقوف. فكر فيها كخطأ نحوي في جملة.

```js
const arr = ["Beau", "Quincy" "Tom"]
```

- **ReferenceError**: هناك عدة أنواع من أخطاء المرجع، تُثار بطرق مختلفة. النوع الأول من أخطاء المرجع هو المتغيرات غير المعرفة. مثال آخر على ReferenceError هو محاولة الوصول إلى متغير معلن بـ `let` أو `const` قبل تعريفه.

:::interactive_editor

```js
console.log(num);
const num = 50;
```

:::

- **TypeError**: تحدث هذه الأخطاء عندما تحاول تنفيذ عملية على نوع خاطئ.

```js
const developerObj = {
  name: "Jessica",
  country: "USA",
  isEmployed: true
};

developerObj.map()
```

- **RangeError**: تحدث هذه الأخطاء عندما تحاول الشفرة استخدام قيمة خارج النطاق الذي يمكن لـ JavaScript التعامل معه.

```js
const arr = [];
arr.length = -1; 
```

## جملة `throw`

- **التعريف**: تُستخدم جملة `throw` في JavaScript لرمي استثناء معرف من قبل المستخدم. الاستثناء في البرمجة هو حدث غير متوقع يحدث ويعطل التدفق الطبيعي للبرنامج.

:::interactive_editor

```js
function validateNumber(input) {
  if (typeof input !== "number") {
    throw new TypeError("Expected a number, but received " + typeof input);
  }
  return input * 2;
}

console.log(validateNumber("Naomi"));   // TypeError: Expected a number, but received string
```

:::

## `try...catch...finally`

- **التعريف**: يُستخدم كتلة `try` لتغليف الشفرة التي قد ترمي خطأً. تعمل كمساحة آمنة لتجربة شيء قد يفشل. تلتقط كتلة `catch` الأخطاء التي تحدث في كتلة try وتعالجها. يمكنك استخدام كائن الخطأ داخل catch لفحص ما حدث خطأً. تُنفذ كتلة `finally` بعد كتلتي try و catch، بغض النظر عما إذا حدث خطأ أم لا. تُستخدم عادةً لمهام التنظيف، مثل إغلاق الملفات أو تحرير الموارد.

:::interactive_editor

```js
function processInput(input) {
  if (typeof input !== "string") {
    throw new TypeError("Input must be a string.");
  }

  return input.toUpperCase();
}

try {
  console.log("Starting to process input...");
  const result1 = processInput("hello");
  console.log("Processed result:", result1);  // HELLO
  const result2 = processInput(9);  // throws TypeError
  console.log("Processed result:", result2);  // not executed
} catch (error) {
  console.error("Error occurred:", error.message);
} 
```

:::

## تقنيات تصحيح الأخطاء

- **جملة `debugger`**: تتيح لك هذه الجملة إيقاف الشفرة عند سطر معين للتحقيق فيما يحدث في البرنامج.

:::interactive_editor

```js
let firstNumber = 5;
let secondNumber = 10;

debugger; // Code execution pauses here
let sum = firstNumber + secondNumber;

console.log(sum);
```

:::

- **نقاط التوقف**: تتيح نقاط التوقف إيقاف تنفيذ الشفرة عند سطر تختاره. بعد الإيقاف، يمكنك فحص المتغيرات، تقييم التعبيرات، وفحص مكدس الاستدعاءات.
- **المراقبون**: تتيح مراقبة التعبيرات متابعة قيم المتغيرات أو التعبيرات أثناء تشغيل الشفرة حتى لو كانت خارج النطاق الحالي.
- **التحليل الزمني**: يساعد التحليل الزمني في تحديد عنق الزجاجة في الأداء من خلال التقاط لقطات شاشة وتسجيل استخدام وحدة المعالجة المركزية، واستدعاءات الدوال، ووقت التنفيذ.
- **`console.dir()`**: تُستخدم هذه الطريقة لعرض قائمة تفاعلية بخواص كائن JavaScript محدد. تعرض قائمة هرمية يمكن توسيعها لرؤية كل الخصائص المسكنة.

```js
console.dir(document);
```

- **`console.table()`**: تعرض هذه الطريقة بيانات جدولية كجدول في وحدة التحكم. تأخذ معلمة إلزامية واحدة يجب أن تكون مصفوفة أو كائن، ومعلمة اختيارية لتحديد الخصائص (الأعمدة) التي تُعرض.
