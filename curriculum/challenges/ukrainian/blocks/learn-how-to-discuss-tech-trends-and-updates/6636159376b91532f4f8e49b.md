# --questions--

## --text--

Для якого типу проєкту або команди низькокодовий розвиток особливо корисний, згідно з Сарою?

## --answers--

Для великих команд зі спеціалізованими навичками кодування

### --feedback--

Сара зазначає, що низькокодовий розвиток більше підходить для команд з обмеженими ресурсами кодування, а не для великих і високоспеціалізованих.

---

Для проєктів, які потребують великої кількості ручного кодування

### --feedback--

Це вводить в оману; Сара вказує, що низькокодовий розвиток корисний саме тому, що зменшує потребу у великій кількості ручного кодування.

---

Для проєктів з багатьма завданнями та командами, які мають обмежені ресурси кодування

---

Для малих, простих проєктів, які потребують мало або зовсім не потребують кодування

### --feedback--

Хоча низькокодовий розвиток можна використовувати в малих проєктах, Сара конкретно згадує його корисність у керуванні багатьма завданнями з обмеженим персоналом для кодування, що передбачає більшу складність.
