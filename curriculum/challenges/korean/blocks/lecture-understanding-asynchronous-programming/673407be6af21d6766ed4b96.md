# --description--

이전 수업에서는 Fetch API가 무엇이고 어떻게 사용하는지 살펴보았습니다. 이번 수업에서는 Fetch API의 `GET`, `POST`, `PUT` 그리고 `DELETE` HTTP 메서드에 대해 알아보겠습니다.

가장 흔히 사용하는 HTTP 메서드인 `GET` 메서드부터 시작하겠습니다. 이 메서드는 서버에서 데이터를 가져오는 데 사용됩니다. `fetch()`를 사용할 때 메서드를 지정하지 않으면 기본값으로 `GET`가 설정됩니다.

```js
fetch('https://api.example.com/data')
```

이 코드에서는 `GET` 요청을 `https://api.example.com/data`에 보내고 있습니다. 여기서 주의할 점은 이 데이터를 바로 사용할 수 없고, 응답을 JSON 형식으로 변환해야 한다는 것입니다. 그래야 프로젝트 어디서든 사용할 수 있습니다. 다음은 그 방법의 예입니다:

```js
fetch('https://api.example.com/data')
  .then(response => response.json())
  .then(data => console.log(data))
```

이 코드에서 Fetch API로부터 오는 응답은 프로미스이며, 우리는 `.then` 핸들러를 사용해 응답을 JSON 형식으로 변환하고 있습니다. 이전 수업에서 `Promise`가 비동기 프로세스의 최종 완료(또는 실패)와 그 값을 나타내는 객체라는 것을 배웠습니다.

프로미스의 값은 프로미스가 생성될 때 알 수 없고, 비동기 프로세스가 완료되어야만 알 수 있습니다. fetch 호출에 두 개의 `.then` 핸들러를 체이닝할 때, 이것을 프로미스 체이닝이라고 하며 다음 수업에서 배울 예정입니다.

지금까지는 서버에서 리소스를 가져오는 작업을 했습니다. 그런데 서버로 데이터를 보낼 수도 있다는 사실을 알고 있나요? `POST` 메서드는 서버에 데이터를 보내 리소스를 생성하는 데 사용됩니다. 다음은 서버에 데이터를 생성하는 데 사용하는 `POST` 메서드 예시입니다:

```js
fetch('https://api.example.com/users', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({
    name: 'John Doe',
    email: 'john@example.com'
  })
})
```

이 예시에서는 새로운 사용자를 생성하기 위해 `POST` 요청을 보내고 있습니다. 메서드를 `POST`로 지정하고, 적절한 헤더를 설정하며, 보내고자 하는 데이터를 포함한 `body`를 추가했습니다. 본문은 문자열이어야 하므로, 객체를 JSON 문자열로 변환하기 위해 `JSON.stringify()`를 사용합니다.

`PUT` 메서드는 서버의 기존 리소스를 업데이트하는 데 사용됩니다. 다음은 그 예시입니다:

```js
fetch('https://api.example.com/users/45', {
  method: 'PUT',
  headers: {
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({
    name: 'John Smith',
    email: 'john@example.com'
  })
})
```

이 예시에서 URL 끝에 `45`가 있는 것을 주의 깊게 보세요. 이는 보통 업데이트하려는 데이터를 식별하는 고유 ID로 사용됩니다. 코드에서는 `PUT` 메서드를 사용했고, 업데이트할 데이터를 `body`로 지정했습니다.

`DELETE` 메서드는 서버에서 리소스를 삭제하는 데 사용됩니다. 다음은 그 예시입니다:

```js
fetch('https://api.example.com/users/45', {
  method: 'DELETE',
})
```

이 코드에서는 `DELETE` 메서드와 삭제할 데이터를 식별하기 위한 ID를 URL 끝에 포함하고 있습니다.
