# --description--

路由層級中介層類似於應用程式層級中介層，但它綁定於 `express.Router()` 的實體，而非主要的應用程式實體。它可以用來將你的 Express 應用程式模組化，透過建立獨立的路由模組。

路由層級中介層的運作方式與應用程式層級中介層相同，但它只套用於特定的路由實體。這讓你更容易只對一組路由套用中介層，而非全域套用。

```js
const express = require('express')
const app = express()
const router = express.Router()

// Router-level middleware to log requests to '/menu' routes
router.use((req, res, next) => {
  console.log('Request made to /menu route')
  next() // Pass control to the next handler
})

// Define routes
router.get('/drinks', (req, res) => {
  res.send('Welcome to the drinks menu!')
})

app.use('/menu', router) // Apply the router

app.listen(3000, () => {
  console.log('Server running on http://localhost:3000')
})
```

* `router.use()`：這是路由層級中介層，只套用於由 router 處理的路由。
* `app.use('/menu', router)`：中介層只套用於以 `/menu` 開頭的路由（例如 `/menu/drinks`）。

路由層級中介層讓你能將應用程式拆分成模組化組件，並選擇性地對一組路由套用中介層，有助於組織和程式碼清晰度。
