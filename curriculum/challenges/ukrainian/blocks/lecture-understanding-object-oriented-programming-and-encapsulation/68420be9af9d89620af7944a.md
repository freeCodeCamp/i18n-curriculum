# --description--

Об’єктно-орієнтоване програмування, також відоме як OOP, — це стиль програмування, у якому розробники ставляться до всього в своєму коді як до реального об’єкта.

Клас — це як креслення для створення об’єктів. Кожен об’єкт, створений із класу, має атрибути, які визначають дані, і методи, які визначають поведінку об’єктів.

У попередньому уроці ви навчились створювати класи. Ось нагадування синтаксису:

```py
class ClassName:
   def __init__(self, parameters):
       attribute = value

   def method_name(self):
       # method logic
```

Ось приклад класу, який використовує спеціальний метод `__init__` для ініціалізації атрибутів brand і color щоразу, коли створюється об’єкт за допомогою класу:

```py
class Car:
   def __init__(self, brand, color):
       self.brand = brand
       self.color = color

# create two objects from the Car class
car1 = Car('Toyota', 'red')
car2 = Car('Lambo', 'green')

print('Car 1 Brand:', car1.brand) # Car 1 Brand: Toyota
print('Car 1 Color:', car1.color) # Car 1 Color: red

print('Car 2 Brand:', car2.brand) # Car 2 Brand: Lambo
print('Car 2 Color:', car2.color) # Car 2 Color: green
```

Об’єктно-орієнтоване програмування має чотири ключові принципи, які допомагають ефективно організовувати та керувати кодом. Це інкапсуляція, успадкування, поліморфізм і абстракція.

Решта цього уроку буде зосереджена на тому, як працює інкапсуляція.

Інкапсуляція — це об’єднання атрибутів і методів об’єкта в єдину одиницю — клас.

З інкапсуляцією ви можете приховати внутрішній стан об’єкта за простим набором публічних методів і атрибутів, які діють як двері. За цими дверима знаходяться приватні атрибути і методи, які контролюють, як змінюються дані і хто може їх бачити.

Припустимо, ви хочете відстежувати баланс гаманця. Ви хочете дозволити людям вносити або знімати гроші з гаманця, але ніхто не повинен мати змогу безпосередньо змінювати баланс.

У такому випадку ви можете зробити `deposit()` і `withdraw()` публічними методами, а баланс приховати під атрибутом `_balance`:

```py
class Wallet:
   def __init__(self, balance):
       self._balance = balance # For internal use by convention

   def deposit(self, amount):
       if amount > 0:
           self._balance += amount # Add to the balance safely

   def withdraw(self, amount):
       if 0 < amount <= self._balance:
           self._balance -= amount # Remove from the balance safely
```

За конвенцією, префікс атрибутів і методів одним підкресленням означає, що вони призначені для внутрішнього використання. Ніхто не повинен безпосередньо звертатися до них поза межами класу, оскільки це суперечить принципам інкапсуляції і може призвести до помилок.

Хоча префікс одним підкресленням — це лише конвенція, префіксування атрибутів і методів подвійним підкресленням ефективно забороняє доступ до них ззовні класу, роблячи ці атрибути і методи приватними.

```py
class Wallet:
   def __init__(self, balance):
       self.__balance = balance # Private attribute

   def deposit(self, amount):
       if amount > 0:
           self.__balance += amount # Add to the balance safely

   def withdraw(self, amount):
       if 0 < amount <= self.__balance:
           self.__balance -= amount # Remove from the balance safely

account = Wallet(500)
print(account.__balance) # AttributeError: 'Wallet' object has no attribute '__balance'
```

Щоб отримати поточне значення `__balance`, ви можете визначити метод get_balance. Наприклад:

```py
class Wallet:
   def __init__(self, balance):
       self.__balance = balance

   def deposit(self, amount):
       if amount > 0:
           self.__balance += amount

   def withdraw(self, amount):
       if 0 < amount <= self.__balance:
           self.__balance -= amount
  
   def get_balance(self):
       return self.__balance


acct_one = Wallet(100)
acct_one.deposit(50)
print(acct_one.get_balance()) # 150

acct_two = Wallet(450)
acct_two.withdraw(28)
print(acct_two.get_balance()) # 422

acct_two.deposit(150)
print(acct_two.get_balance()) # 572
```

Ви також можете визначити приватний метод `__validate` для перевірки, чи є кожна сума внеску або зняття додатним числом:

```py
class Wallet:
   def __init__(self):
       self.__balance = 0

   def __validate(self, amount):
       if amount < 0:
           raise ValueError('Amount must be positive')

   def deposit(self, amount):
       self.__validate(amount)
       self.__balance += amount

   def withdraw(self, amount):
       self.__validate(amount)
       if amount > self.__balance:
           raise ValueError('Insufficient funds')
       self.__balance -= amount

   def get_balance(self):
       return self.__balance

acct_one = Wallet()
acct_one.deposit(3)
print(acct_one.get_balance()) # 3

acct_one.deposit(50)
print(acct_one.get_balance()) # 53

acct_one.deposit(-4)  # ValueError: Amount must be positive
acct_one.withdraw(-8) # ValueError: Amount must be positive
acct_one.withdraw(58) # ValueError: Insufficient funds
```

Як бачите, метод `__validate` є приватним і виконується за лаштунками у публічних методах `deposit()` і `withdraw()`, щоб переконатися, що сума завжди є дійсною.

У наступному уроці ви дізнаєтесь більше про те, як працюють атрибути з префіксом подвійного підкреслення.

Підсумовуючи, інкапсуляція захищає внутрішні дані за чіткими публічними методами. Саме так ви захищаєте свої класи від несанкціонованих змін і централізуєте валідацію в одному місці. Ви можете вільно оновлювати або розширювати свій код, знаючи, що зовнішній код взаємодіє лише з тими інтерфейсами, які ви відкриваєте.
