# --description--

حقن SQL هو ثغرة أمنية في الويب يستغلها المهاجمون لتغيير أو استغلال الاستعلامات التي يُجريها تطبيق الويب على قاعدة البيانات. على سبيل المثال، يمكن للمهاجمين إدخال شفرة SQL خبيثة في الاستعلام لتشغيل هذه الشفرة الخبيثة على الخادم الخلفي.

يحدث هذا عندما تسمح للمستخدمين بإدخال بيانات عبر نماذج تسجيل الدخول أو أشرطة البحث أو معلمات URL، ثم تستخدم تلك المُدخَلات في استيفاء سلسلة نصية أو ربطها لتوليد أمر SQL النهائي.

اعتمادًا على هدف المهاجم، يمكن للشفرة الخبيثة أن تسبب اضطرابات كبيرة في قاعدة البيانات وتطبيق الويب، مثل تجاوز التوثيق، سرقة البيانات من قاعدة البيانات، أو تعديل أو حذف سجلات قاعدة البيانات. لذلك، يجب أن تأخذ هذا التهديد على محمل الجد.

على سبيل المثال، لنفترض أن تطبيق الويب الخاص بك يستخدم استعلام SQL مثل هذا لتوثيق المستخدمين، حيث يضمّن اسم المستخدم وكلمة المرور التي أدخلها المستخدم مباشرة:

```sql
'SELECT * FROM users WHERE username = ' + username_input + ' AND password = ' + password_input + ';'
```

إذا أدخل المهاجم شيئًا مثل هذا في حقل `username`:

```sql
'" " OR "1"="1" --'
```

وأي شيء آخر في حقل `password`، بما أن `"1"="1"` دائمًا `true` و`--` يحول باقي الاستعلام إلى تعليق، يصبح بند `WHERE` كما يلي:

```sql
WHERE username = " " OR TRUE
```

والذي يكون دائمًا `true`.

اعتمادًا على تنفيذ قاعدة البيانات، قد يُرجع هذا بيانات أول مستخدم أو أي مستخدم في قاعدة البيانات ويسمح للمهاجم بتسجيل الدخول بدون بيانات اعتماد صحيحة.

فكيف تمنع هذا؟ هناك طريقتان رئيسيتان لمنع حقن SQL:

- لا تكتب استعلامات ديناميكية باستخدام ربط السلاسل النصية.
- وامنع إدخال شفرة SQL خبيثة من أن تُدرج في الاستعلامات التي ستُنفذ في الخادم الخلفي.

عمليًا، يمكنك استخدام العبارات المعلمة، المعروفة أيضًا بالاستعلامات المعلمة. تفصل هذه العبارات بنية SQL عن البيانات التي يدخلها المستخدم، مما يمنع قاعدة البيانات من تفسير البيانات المُدخلة ككود قابل للتنفيذ.

تقنية مهمة أخرى هي التحقق من المُدخَل. يُنصح بها كإجراء أمني ثانوي في جميع الحالات، لكنها مهمة بشكل خاص لأجزاء استعلامات SQL المتعلقة بأسماء الجداول، أسماء الأعمدة، وترتيب الفرز. إذا أمكن، يُفضل تعيين أسماء الجداول والأعمدة مباشرة في شفرتك، وليس من مُدخَل المستخدم.

إذا حدث هجوم حقن SQL رغم كل التدابير الوقائية، يمكنك تقليل تأثيره على قاعدة البيانات بتقييد صلاحيات كل حساب قاعدة بيانات. امنح كل مستخدم قاعدة بيانات فقط الصلاحيات اللازمة لمهامه، ولا شيء أكثر. إذا كان الحساب يحتاج فقط لقراءة البيانات من قاعدة البيانات، امنحه صلاحية القراءة فقط. وكإجراء وقائي، لا تعطي أبدًا صلاحيات المدير لحسابات التطبيق. هذا خطر جدًا لأن المهاجم قد يحصل على وصول كامل وتحكم كامل في قاعدة البيانات.

حقن SQL هجوم شائع جدًا يجب أن تكون على دراية به كمطور وأن تمنعه عند تنفيذ تطبيق الويب الخاص بك. باستخدام هذه التقنيات، يمكنك حماية قاعدة بياناتك والحفاظ على أمان بيانات المستخدمين.
