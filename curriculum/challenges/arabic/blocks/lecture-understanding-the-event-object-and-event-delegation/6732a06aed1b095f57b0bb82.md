# --interactive--

حدث `change` هو حدث خاص يُطلق عندما يغير المستخدم قيمة بعض عناصر الإدخال. وبشكل أكثر تحديدًا:

- عند تحديد أو إلغاء تحديد مربع اختيار.

- عند تحديد زر اختيار.

- عند قيام المستخدم بالاختيار من شيء مثل محدد التاريخ أو قائمة منسدلة.

- عند فقدان حقل الإدخال للتركيز (عندما ينتقل المستخدم إلى الحقل التالي باستخدام التبويب، أو ينقر خارج النموذج) بعد أن يغير المستخدم القيمة.

- عندما يؤكد المستخدم القيمة بطريقة أخرى، مثل الضغط على Enter بعد كتابة نص.

لاحظ أن حدث `change` لا يُطلق عند كتابة المستخدم داخل حقل الإدخال. سيُطلق حدث `change` فقط بعد أن ينتقل التركيز إلى عنصر آخر.

إليك مثالًا على استخدام حدث `change` مع قائمة منسدلة:

:::interactive_editor

```html
<select id="select-menu">
  <option value="option1">Option 1</option>
  <option value="option2">Option 2</option>
  <option value="option3">Option 3</option>
</select>
<script src="index.js"></script>
```

```js
const selectMenu = document.getElementById("select-menu");
selectMenu.addEventListener("change", (event) => {
  console.log(`You selected: ${event.target.value}`);
});
```

:::

في كل مرة يختار المستخدم خيارًا مختلفًا من القائمة المنسدلة، يُطلق حدث `change`، ويتم تسجيل القيمة المختارة في وحدة التحكم.

يُولد حدث `change` كائن `Event`، لكنه لا يُولد تنفيذًا مخصصًا كما في معظم الأحداث الأخرى – الخصائص والطرق التي يمكنك الوصول إليها هي فقط تلك الموجودة في كائن `Event` الأساسي.

هذا يختلف عن حدث `input`، الذي يُولد كائن `InputEvent` مخصص. كما يختلف حدث change في عدة نواحٍ. على سبيل المثال، حدث `input` سيُطلق عندما يكتب المستخدم محتوى في حقل.

هذه الاختلافات مهمة لتتذكرها، إذ قد تواجه صعوبة في توقيت إطلاق هذه الأحداث.
