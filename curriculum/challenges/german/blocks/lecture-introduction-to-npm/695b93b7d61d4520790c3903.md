# --description--

Wenn Sie Projekte in JavaScript erstellen, benötigen Sie oft zusätzliche Werkzeuge, die Ihnen helfen, Aufgaben zu erledigen. Selbst einfache Anwendungen brauchen Funktionen wie das Speichern von Daten, das Einloggen, Routing zum Wechseln zwischen Seiten und so weiter.

Anstatt all diesen Code von Grund auf neu zu schreiben, können Sie Open-Source-Pakete verwenden, die andere Entwickler bereits erstellt und geteilt haben.

Hier kommt npm ins Spiel. npm ist ein riesiges Online-Register für Open-Source-Software. Man kann es sich wie einen App-Store für Code vorstellen – Sie können nach dem suchen, was Sie brauchen, es installieren und in Ihrem Projekt verwenden.

Viele denken, npm steht für „Node Package Manager“, aber das ist nicht die offizielle Bedeutung. In älteren npm-Dokumentationen wurde es als „rekursives bakronymisches Kürzel für ‚npm is not an acronym‘“ beschrieben. Mit anderen Worten: npm steht offiziell für nichts.

Heute bezieht sich npm auf drei zusammenhängende Teile:

* Die Website npmjs.com, auf der Sie Pakete im Register nachschlagen können.
    
* Das npm-Register, eine große öffentliche Datenbank, die alle Pakete speichert.
    
* Das npm-CLI-Tool, das mit Node.js installiert wird und mit dem Sie Pakete installieren und verwalten können.
    

Wenn Sie zum Beispiel in Ihrem Projekt den Zustand verwalten, Daten verarbeiten, E-Mails senden oder HTTP-Anfragen stellen müssen, gibt es wahrscheinlich schon ein npm-Paket, das das erledigt. Mit einem einfachen `npm install`-Befehl können Sie diese fertige Software nutzen, anstatt alles selbst zu programmieren.

npm hilft Ihnen auch, Ihre Projektabhängigkeiten zu verwalten. Das bedeutet, es verfolgt die Pakete, die Sie in Ihren Projekten verwenden, und die Version jedes einzelnen Pakets. Das geschieht über die `package.json`- und `package-lock.json`-Dateien, die Sie in zukünftigen Lektionen kennenlernen werden.

Yarn, PNPM und Bun sind einige Alternativen zu npm, die im Laufe der Zeit populär geworden sind. Während jede von ihnen Vorteile in bestimmten Bereichen wie Installationsgeschwindigkeit und Speicherplatz bietet, ist npm immer noch am weitesten verbreitet.

In den kommenden Lektionen lernen Sie verschiedene npm-Befehle, wie man Pakete installiert, über die `package.json`- und `package-lock.json`-Dateien und sogar, wie Sie Ihr eigenes Paket im npm-Register veröffentlichen können.
