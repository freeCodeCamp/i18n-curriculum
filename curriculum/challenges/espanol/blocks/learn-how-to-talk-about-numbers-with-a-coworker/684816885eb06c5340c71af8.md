# --explanation--

Las `Initial unit tests` son el primer paso en las pruebas. Los desarrolladores verifican pequeñas partes del código (`units`) para asegurarse de que hagan lo que se supone que deben hacer. Por ejemplo:

`We wrote initial unit tests to check the main functions.` — El equipo creó pruebas iniciales para asegurarse de que las partes más importantes del código funcionaran correctamente.

Las `Quality assurance (QA) checks` son pruebas finales realizadas por los equipos de QA para asegurarse de que el software sea confiable, fácil de usar y esté listo para el lanzamiento. Por ejemplo:

`Regular QA checks help prevent bugs from reaching users.` — Las pruebas frecuentes por parte de QA ayudan a detectar errores temprano, para que los usuarios no enfrenten problemas.
