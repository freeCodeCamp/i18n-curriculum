# --description--

Nello sviluppo web moderno, puoi dividere il tuo codice in più file chiamati "moduli".

Puoi pensare a un modulo come a un blocco di codice autonomo e riutilizzabile.

Importando moduli in altri moduli, puoi riutilizzare le loro funzionalità mantenendo il codice in file separati, il che è più gestibile, soprattutto per applicazioni web grandi e complesse.

In Node.js, ogni file è considerato un modulo.

Node ha due sistemi diversi per lavorare con i moduli: CommonJS e ES.

CommonJS è il modo originale e predefinito di lavorare con i pacchetti in Node.js. Carica i moduli in modo sincrono, il che significa che deve aspettare che un modulo sia completamente caricato prima di iniziare a caricare il successivo o continuare l'esecuzione del codice.

È ancora molto usato, ma viene gradualmente sostituito dalla sintassi ES più recente.

CommonJS usa la funzione `require()` per importare un modulo, prendendo come argomento il suo percorso relativo.

Per esempio, supponiamo di avere un modulo chiamato `math.js`, dove definiamo operazioni matematiche di base.

Useremo solo una funzione di moltiplicazione qui per mantenere questo esempio semplice e focalizzato:

```js
function multiply(a, b) {
  return a * b;
}
```

Come possiamo riutilizzare questa funzione al di fuori del nostro modulo attuale? Per impostazione predefinita, le funzioni e le variabili definite in un modulo Node.js sono **private** — altri moduli non possono consultarle a meno che non le esportiamo esplicitamente. Ci sono diversi modi per farlo.

Un modo comune è assegnare un valore a `module.exports`. Questo oggetto rappresenta ciò che il modulo restituirà quando viene importato altrove, dando ad altri moduli accesso alle funzioni, variabili o oggetti che hai scelto di esporre.

In questo esempio, assegniamo un oggetto con una proprietà `multiply` a `module.exports`. Il valore di quella proprietà è un riferimento alla funzione `multiply` che abbiamo definito prima. Questo esporterà la funzione, così potremo chiamarla da altri moduli se necessario.

```javascript
function multiply(a, b) {
  return a * b;
}

module.exports = {
  multiply: multiply
};
```

Supponiamo di provare a importare il modulo `math.js` in un altro modulo. Chiamiamo la funzione `require()` e passiamo il percorso relativo come argomento. Poi assegniamo l'oggetto restituito da `require()` alla variabile `math`.

```javascript
const math = require('./math');
```

Controlliamo il valore di `math` nella console stampandolo con `console.log()`:

```javascript
console.log(math);
```

Questo è l'output. È un oggetto con una proprietà `multiply` e una funzione come valore:

```javascript
{ multiply: [Function: multiply] }
```

Se chiamiamo questa funzione con `math.multiply()`, passando i numeri `6` e `7` come argomenti, l'output è `42`. La funzione viene eseguita correttamente eppure si trova in un altro file! Quindi la nostra istruzione di importazione ha funzionato correttamente.

```javascript
const math = require('./math');

console.log(math.multiply(6, 7)); // 42
```

Ma ci sono altri modi per esportare una funzione, variabile o oggetto.

Per esempio, se hai bisogno di esportare solo una singola funzione, variabile o oggetto, puoi assegnarla direttamente come valore di `module.exports`:

```javascript
function multiply(a, b) {
  return a * b;
}

module.exports = multiply;
```

Se devi esportare più funzioni, devi solo aggiungerle all'oggetto `module.exports`, separate da una virgola.

Qui puoi vedere che l'oggetto esportato ha due proprietà: `subtract` e `multiply`. I loro valori sono riferimenti alle rispettive funzioni, quindi avrai accesso a entrambe in qualsiasi modulo che importa `math.js`:

```javascript
function subtract(a, b) {
  return a - b;
}

function multiply(a, b) {
  return a * b;
}

module.exports = {
  subtract: subtract,
  multiply: multiply
};
```

Quando l'oggetto `module.exports` ha più proprietà, avrai la possibilità di scegliere se importare tutto o importare le proprietà singolarmente.

In questo esempio stiamo importando l'intero oggetto esportato da `math.js`, quindi avremo accesso sia a `subtract` che a `multiply`:

```javascript
const math = require('./math');

console.log(math.subtract(6, 7));  // -1
console.log(math.multiply(6, 7));  // 42
```

Se vogliamo importare solo `multiply`, dobbiamo scrivere il suo nome tra parentesi graffe, dopo la parola chiave `const`. In pratica, stai usando la sintassi di assegnazione destrutturante per ottenere il suo valore dall'oggetto esportato. Poi possiamo riferirci direttamente a esso per nome nel nostro codice:

```javascript
const { multiply } = require('./math');

console.log(multiply(6, 7));
```

Questo funziona anche per più elementi. Devi solo separarli con una virgola all'interno delle parentesi graffe. Qui stiamo importando sia le funzioni `subtract` che `multiply`:

```javascript
const { subtract, multiply } = require('./math');
```

Per impostazione predefinita, Node.js tratterà come moduli CommonJS i seguenti:

* File con estensione `.cjs`.
    
* File con estensione `.js` quando il file `package.json` genitore più vicino contiene un campo di primo livello `"type"` con valore `"commonjs"`.
    
* File con estensione `.js` o senza estensione, quando il file `package.json` genitore più vicino non contiene un campo di primo livello `"type"`, o non esiste alcun `package.json` in nessuna cartella genitore.
    
* File con un'estensione che non è `.mjs`, `.cjs`, `.json`, `.node` o `.js`.
    

Anche se i moduli ES stanno gradualmente sostituendo i moduli CommonJS, comprenderli è ancora essenziale per gli sviluppatori che lavorano con basi di codice Node.js esistenti e molti pacchetti npm.
