# --description--

في الدرس السابق، تعلمت كيفية العمل مع أنواع الاتحاد. إحدى الطرق الشائعة للعمل مع أنواع الاتحاد هي استخدام اسم مستعار للنوع.

الاسم المستعار للنوع هو اسم يُعطى لنوع معين. إليك مثالًا على إعطاء اسم لنوع اتحاد:

```ts
type ID = number | string;
```

غالبًا في مشاريعك، ستنشئ أسماء مستعارة للأنواع وتستخدمها في جميع أنحاء برنامجك بهذه الطريقة:

```ts
type ID = number | string;

type User = {
  id: ID;
  name: string;
};

function getUserId(user: User): ID {
  return user.id;
}
```

يستخدم هذا المثال نوعين اسمهما `ID` و`User`. سيكون `ID` نوع اتحاد لـ `number` أو `string` بينما `User` هو نوع كائن يحتوي على خاصيتي `id` و`name`. الخاصية `id` محددة بنوع `ID` بينما الخاصية `name` لها نوع `string`.

الدالة `getUserId` لها معلمة واحدة اسمها `user` ونوعها `User`. نوع الإرجاع للدالة `getUserId` سيكون `ID`.

الأسماء المستعارة للأنواع طريقة رائعة لوصف شكل الكائن. من الشائع إنشاء أنواع كائنات واستخدامها في المصفوفات بهذه الطريقة:

```ts
type User = {
  id: number | string;
  name: string;
};

const users: User[] = [
  { id: 101, name: "Alice" },
  { id: "A102", name: "Bob" },
  { id: 103, name: "Charlie" }
];
```

في هذا المثال، يجب أن تمتلئ مصفوفة `users` فقط بكائنات من نوع `User`. إذا نسيت خاصية في أحد الكائنات، فسيُصدر TypeScript خطأً مثل هذا:

```ts
type User = {
  id: number | string;
  name: string;
};

const users: User[] = [
  // Property 'name' is missing in type '{ id: number; }' but required in type 'User'.
  { id: 101 },
  { id: "A102", name: "Bob" },
  { id: 103, name: "Charlie" }
];
```

ربما لاحظت استخدام PascalCase عند تسمية الأسماء المستعارة للأنواع. ذلك لأن الاصطلاح يستخدم PascalCase لتمييز الأنواع عن المتغيرات. كما يساعد ذلك على تحسين قابلية القراءة. عندما ترى `User[]`، ستعرف أنها مصفوفة من كائنات من نوع `User`.
