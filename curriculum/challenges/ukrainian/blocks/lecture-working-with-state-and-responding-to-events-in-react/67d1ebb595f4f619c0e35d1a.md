# --description--

Оновлення об’єктів у стані в React може бути складним, якщо ви звикли змінювати значення властивостей об’єкта безпосередньо.

React розглядає стан як незмінний, тобто його не слід змінювати напряму.

Давайте подивимося, що станеться, якщо ви спробуєте змінити об’єкти в стані React напряму, а потім розглянемо правильний спосіб це зробити.

Припустімо, у вас є об’єкт у стані компонента, який представляє профіль користувача, і ви хочете, щоб користувач оновив свій вік:

```jsx
import { useState } from "react";

function Profile() {
  const [user, setUser] = useState({
    name: "John Doe",
    age: 31,
    city: "LA",
  });

  // Change user age directly
  const handleAgeChange = (e) => {
    user.age = e.target.value;
    console.log(user);
  };

  return (
    <div>
      <h1>User Profile</h1>
      <p>Name: {user.name}</p>
      <p>Age: {user.age}</p>
      <p>City: {user.city}</p>

      <h2>Update User Age </h2>
      <input type="number" value={user.age} onChange={handleAgeChange} />
    </div>
  );
}

export default Profile;
```

Цей код не спрацює, бо ви безпосередньо змінюєте властивість `user.age`.

Навіть якщо `console.log(user)` покаже новий вік у консолі, React не перерендерить компонент, щоб показати це в інтерфейсі користувача, бо ви не використали функцію встановлення, `setUser`.

Щоб оновити об’єкт напряму в стані, потрібно використати функцію встановлення, щоб створити новий об’єкт із оновленим значенням. Наприклад:

```js
const handleAgeChange = (e) => {
  setUser({
    age: e.target.value,
  });
};
```

Це працює. Але якщо подивитися на сторінку зараз, вік користувача оновлюється, але значення для імені та міста втрачаються.

Це тому, що новий об’єкт, який ви передали функції встановлення, містив лише пару ключ/значення для `age`.

Щоб уникнути цього, ви можете спочатку скопіювати існуючий об’єкт, а потім оновити лише ту властивість, яку хочете змінити, у цьому випадку `age`.

Для цього можна передати спеціальну функцію, яку називають функцією оновлення, у вашу функцію встановлення `setUser`. Функція оновлення приймає поточний стан як аргумент, тут названий `prevUser`, і має повернути наступний стан:

```js
const handleAgeChange = (e) => {
  setUser((prevUser) => {
    const updatedUser = { ...prevUser, age: e.target.value };
    console.log('Previous State:', prevUser);
    console.log('Updated State:', updatedUser);
    return updatedUser;
  });
};
```

Як бачите, ми створюємо новий об’єкт користувача `updatedUser`, використовуючи синтаксис розширення, щоб скопіювати поточний об’єкт користувача `...prevUser`. Потім оновлюємо вік на основі введених у формі даних і повертаємо `updatedUser` в кінці функції як наступний стан.

Тепер ваш проєкт працює як очікувалося, і оновлення поля віку не впливає на ім’я користувача чи назву міста. Ви також можете бачити попередні та оновлені стани в консолі.

Це ідеальний спосіб оновлення об’єкта в стані, особливо коли ви не оновлюєте всі властивості.

Щоб оновити решту властивостей `name` і `city`, ви можете написати окремі функції встановлення і підключити їх до відповідних полів введення:

```js
const handleNameChange = (e) => {
  setUser((prevUser) => ({
    ...prevUser,
    name: e.target.value,
  }));
};

const handleCityChange = (e) => {
  setUser((prevUser) => ({
    ...prevUser,
    city: e.target.value,
  }));
};
```

Або об’єднати їх в одну функцію встановлення так:

```js
const handleChange = (e) => {
  const { name, value } = e.target;
  setUser((prevUser) => ({
    ...prevUser,
    [name]: value,
  }));
};
```

Щоб це працювало, кожне поле введення має мати атрибут `name`.

Ось повний код зараз:

```jsx
import { useState } from "react";

function Profile() {
  const [user, setUser] = useState({ name: "John Doe", age: 31, city: "LA" });

  const handleChange = (e) => {
    const { name, value } = e.target;

    setUser((prevUser) => ({...prevUser, [name]: value}));
  };

  return (
    <div>
      <h1>User Profile</h1>
      <p>Name: {user.name}</p>
      <p>Age: {user.age}</p>
      <p>City: {user.city}</p>

      <h2>Update User Age </h2>
      <input type="number" name="age" value={user.age} onChange={handleChange} />

      <h2>Update User Name </h2>
      <input type="text" name="name" value={user.name} onChange={handleChange} />

      <h2>Update User City </h2>
      <input type="text" name="city" value={user.city} onChange={handleChange} />
    </div>
  );
}

export default Profile;
```
