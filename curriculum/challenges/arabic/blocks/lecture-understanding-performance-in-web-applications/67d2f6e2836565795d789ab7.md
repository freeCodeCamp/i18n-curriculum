# --description--

هل تساءلت يومًا ماذا يحدث خلف الكواليس عندما تدخل عنوان URL وتضغط Enter؟ كيف يحول المتصفح الشفرة إلى صفحات تفاعلية نراها؟

فهم كيفية عرض المتصفح لصفحة ويب أمر أساسي للمطوِّرين الذين يهدفون إلى تحسين الأداء وتعزيز تجربة المستخدم.

في هذا الدرس، سنوضح عملية العرض خطوة بخطوة بطريقة واضحة وسهلة الفهم.

أولًا، يحلل المتصفح شفرة HTML ويبني نموذج كائن المستند (DOM).

تبدأ العملية بجلب المتصفح لمحتوى HTML الخاص بالصفحة. ثم يحلل هذا الـ HTML لبناء نموذج كائن المستند، أو DOM — وهي بنية شجرية تمثل محتوى الصفحة.

انظر إلى هذا المقتطف البسيط من HTML:

```html
<html>
  <body>
    <h1>Hello, World!</h1>
    <p>Welcome to our website.</p>
  </body>
</html>
```

يقوم المتصفح بتحليل هذا لإنشاء شجرة DOM بجذر `html`، تحتوي على `body`، والتي بدورها تحتوي على عناصر `h1` و `p`.

بعد ذلك، يعالج المتصفح CSS، ويبني نموذج كائن CSS، أو CSSOM. هذه بنية شجرية أخرى تحدد كيف يجب تنسيق العناصر.

بالنظر إلى هذا الـ CSS:

```css
h1 {
  color: blue;
}
p {
  font-size: 16px;
}
```

يحدد نموذج كائن CSS أن عناصر `h1` باللون الأزرق وعناصر `p` بحجم خط 16 بكسل.

ثم يجمع المتصفح بين DOM ونموذج كائن CSS لتشكيل شجرة العرض. تتضمن هذه الشجرة فقط العقد اللازمة لعرض الصفحة، وكل منها مرتبط بأنماطه المقابلة.

في مثالنا، تتكون شجرة العرض من عناصر `h1` و `p`، منسقة بشكل مناسب.

مع وجود شجرة العرض، يحسب المتصفح الموقع والحجم الدقيق لكل عنصر في عملية تسمى التخطيط أو إعادة التدفق. هذا يضمن ظهور العناصر في المكان والحجم الصحيحين على الشاشة.

أخيرًا، يرسم المتصفح البكسلات على الشاشة، معروضًا كل عنصر بناءً على الأنماط والتخطيط المحسوبين. في الصفحات المعقدة، قد يشمل ذلك طبقات متعددة تُدمج معًا لتشكيل المخرجات البصرية النهائية.

في الختام، من تحليل HTML وCSS إلى رسم البكسلات على الشاشة، عملية عرض المتصفح هي متتالية معقدة من الخطوات.
