# --description--

React는 프로젝트에서 다양한 기능을 구현할 수 있도록 여러 내장 훅을 제공합니다. 여기에는 `useState`, `useEffect`, `useContext` 등이 포함됩니다.

하지만 때로는 내장 훅으로는 해결할 수 없는 기능을 추가해야 할 때가 있습니다. 다행히 React에서는 직접 커스텀 훅을 만들 수 있습니다.

커스텀 훅은 생각만큼 복잡하지 않습니다. 여러 컴포넌트에서 로직을 공유할 수 있게 해주는 재사용 가능한 함수일 뿐입니다. 즉, 재사용성도 커스텀 훅을 만드는 이유 중 하나입니다.

커스텀 훅을 사용하면 데이터 가져오기, 상태 관리, 토글링, 사용자 온라인/오프라인 상태 확인 같은 부수 효과 등 훅을 사용하는 컴포넌트에서 로직을 분리할 수 있습니다.

그 후 훅을 가져와서 어떤 컴포넌트에서든 사용할 수 있으므로, 컴포넌트 내에서는 렌더링과 표현에 집중할 수 있습니다. 이는 중복을 줄이고, 업데이트할 때 변경해야 할 부분도 줄어듭니다.

이제 직접 커스텀 훅을 만드는 방법을 살펴보겠습니다.

React에서 모든 내장 훅은 `use`로 시작하므로, 커스텀 훅도 같은 관례를 따라야 합니다. 커스텀 훅 이름은 그 기능을 명확히 전달해야 합니다.

예를 들어 커스텀 훅이…

- 데이터를 가져온다면 `useFetch`라고 부를 수 있고

- 어떤 것을 켜고 끄는 토글 기능이라면 `useToggle`라고 부를 수 있으며

- 디바운싱을 구현한다면 `useDebounce`가 좋은 이름입니다.

앱에 디바운싱을 추가하는 커스텀 훅을 만든다고 가정해 봅시다.

디바운싱은 함수가 실행되는 빈도를 제한하는 프로그래밍 기법입니다. 사용자가 특정 시간 동안 동작을 멈출 때까지 기다렸다가 함수를 실행합니다. 예를 들어 검색창에서 매 키 입력마다 API 호출을 하는 대신, 사용자가 500밀리초 동안 입력을 멈출 때까지 기다렸다가 호출합니다.

디바운싱 커스텀 훅을 만들려면 먼저 `useDebounce.jsx` 또는 `useDebounce.js` 파일을 만들어야 합니다. 관례상, 만든 커스텀 훅 파일은 `hooks` 폴더에 저장합니다.

커스텀 훅 내에서 내장 훅을 사용할 수 있습니다. 디바운싱에는 `useState`과 `useEffect` 훅이 필요하므로 파일 상단에 가져오세요:

```js
import { useState, useEffect } from "react";
```

다음으로 `useDebounce` 함수를 만들고 `value`과 `delay`를 매개변수로 받습니다. `value`는 기다릴 값이고, `delay`는 기다릴 시간입니다. 일정 시간 기다려야 하므로 `setTimeout`과 `clearTimeout` 함수가 유용합니다:

```js
function useDebounce(value, delay) {
  const [debouncedValue, setDebouncedValue] = useState(value);

  useEffect(() => {
    const handler = setTimeout(() => {
      setDebouncedValue(value);
    }, delay);

    return () => {
      clearTimeout(handler);
    };
  }, [value, delay]);

  return debouncedValue;
}

export { useDebounce };
```

`debouncedValue` 상태는 지연된 값을 저장하고 반환하며, 지정한 타임아웃 기간이 지나야 업데이트됩니다.

`useEffect`에서 실제 마법이 일어납니다. 이전 수업에서 배웠듯이, 타이머 설정과 해제 같은 React 렌더링 사이클 외부 작업은 부수 효과이므로 `useEffect` 훅을 사용해 처리해야 합니다.

여기 `useEffect` 훅 안에서 `setTimeout`을 사용해 `debouncedValue`를 설정합니다. 그리고 `clearTimeout`이나 `value`이 변경되거나 컴포넌트가 언마운트될 때 이전 타임아웃을 `delay`로 지우는 정리(cleanup) 함수를 반환합니다.

그리고 사용자가 입력을 멈춘 후 1초 동안 검색을 지연시키는 `FootballerSearch` 훅을 사용하는 `useDebounce` 컴포넌트가 있습니다:

```jsx
import { useState, useEffect } from "react";
import { useDebounce } from "./hooks/useDebounce";
import footballers from "./footballers";

const FootballerSearch = () => {
  const [query, setQuery] = useState("");
  const debouncedQuery = useDebounce(query, 1000); // Start searching 1 second after the user stops typing

  useEffect(() => {
    if (debouncedQuery) {
      const results = footballers.filter((footballer) =>
        footballer.toLowerCase().includes(debouncedQuery.toLowerCase()),
      );
      console.log("Search results:", results);
    } else {
      console.log("Search results: []");
    }
  }, [debouncedQuery]);

  return (
    <>
      <h1 style={{ textAlign: "center" }}>Footballer Search App</h1>
      <div style={{ textAlign: "center" }}>
        <input
          style={{ padding: "0.5rem", width: "30%" }}
          type="text"
          value={query}
          onChange={(e) => setQuery(e.target.value)}
          placeholder="Search for a footballer..."
        />
      </div>
    </>
  );
};

export default FootballerSearch;
```

보시다시피 `debouncedQuery` 변수는 쿼리 상태(사용자가 입력한 값)와 1,000밀리초, 즉 1초 지연을 `useDebounce` 훅에 초기화합니다. 검색 자체는 `useEffect` 훅 안에서 처리되며, 검색 결과는 콘솔에 기록됩니다.
