# --description--

في Express، تُعد `express.Router()` ميزة قوية تساعدك على تنظيم مساراتك إلى أجزاء أصغر ومودولية. هذا مفيد بشكل خاص مع نمو التطبيق.

بدلاً من تعريف جميع المسارات في ملف كبير واحد، يمكنك تقسيمها إلى ملفات منفصلة باستخدام الموجِّهات. هذا يجعل شفرتك أسهل في القراءة والصيانة والتوسع.

تُنشئ طريقة `express.Router()` كائن موجِّه جديد.

يعمل الموجِّه مثل تطبيق Express صغير. يمكنه التعامل مع مساراته وبرمجياته الوسيطة الخاصة، لكنه ليس تطبيق Express كامل. بمجرد تعريف المسارات على الموجِّه، يمكنك تركيبه في التطبيق الرئيسي باستخدام `app.use()`.

بمعنى آخر، يسمح لك الموجِّه بتجميع المسارات المرتبطة معًا ثم دمجها لاحقًا في التطبيق الرئيسي.

لاستخدام الموجِّهات عدة مزايا:

**فصل الاهتمامات**

يمكنك تعريف المسارات في ملفات منفصلة، مما يحسن تنظيم ووضوح التطبيق.

**إعادة استخدام الشفرة**  
يمكن إعادة استخدام الموجِّهات في أجزاء مختلفة من التطبيق أو حتى عبر مشاريع متعددة.

**تنظيف الملف الرئيسي**  
بدلاً من ملء ملف `app.js` أو `server.js` الرئيسي بالعديد من تعريفات المسارات، يمكنك نقلها إلى الموجِّهات والحفاظ على تركيز الملف الرئيسي على الإعداد والتكوين.

لننظر إلى مثال بسيط حيث نستخدم `express.Router()` لتعريف مسارات لمسار `/users`.

أولاً، هذا هو ملف `app.js`:

```javascript
const express = require("express");
const app = express();
const port = 3000;

// Importing the user routes
const userRoutes = require("./userRoutes");

// Using the userRoutes router for /users path
app.use("/users", userRoutes);

app.listen(port, () => {
  console.log(`Server running at http://localhost:${port}`);
});
```

وهذا هو `userRoutes.js`:

```javascript
const express = require("express");
const router = express.Router();

// GET request for /users
router.get("/", (req, res) => {
  res.send("List of users");
});

// POST request for /users
router.post("/", (req, res) => {
  res.send("Creating a new user");
});

// GET request for /users/:id
router.get("/:id", (req, res) => {
  const userId = req.params.id;
  res.send(`User details for user with id: ${userId}`);
});

module.exports = router;
```

الآن لنلق نظرة أقرب على كيفية عمل كل شيء.

في `app.js`، نستورد `express` ونُنشئ نسخة من تطبيق Express ونعينها إلى متغير اسمه `app`.

بعد ذلك، نستورد الموجِّه `userRoutes` من `userRoutes.js` ونستخدم `app.use("/users", userRoutes)` لتركيب الموجِّه. هذا يخبر Express أنه إذا بدأ الطلب بـ `/users`، فمرره إلى الموجِّه `userRoutes`.

ثم في `userRoutes.js`، ننشئ موجِّهًا جديدًا باستخدام `express.Router()` ونعرف المسارات باستخدام `router.get()`، `router.post()`، و `router.get("/:id")`.

لاحظ أن المسارات المعرفة هنا نسبية إلى `/users` لأن الموجِّه مركب هناك. إذن:

*   `router.get("/")` → `GET /users`: يرد بقائمة المستخدمين
    
*   `router.post("/")` → `POST /users`: يرد برسالة حول إنشاء مستخدم جديد
    
*   `router.get("/:id")` → `GET /users/:id`: يرد بتفاصيل المستخدم لرقم معرف معين
    

استخدام `express.Router()` يحافظ على نظافة الملف الرئيسي وتركيزه على التكوين. جميع المسارات المتعلقة بالمستخدمين مجمعة في `userRoutes.js`.

مع نمو التطبيق، يمكنك إنشاء موجِّهات إضافية لأقسام أخرى، مثل `/products`، `/orders`، أو `/auth`.

يبقى كل قسم منظمًا في ملفه الخاص، مما يجعل التطبيق أسهل في الصيانة والتوسع.
