# --description--

React 19 ha introdotto due nuove funzionalità importanti chiamate componenti server e azioni server.

Da quella versione in poi, i componenti server sono diventati il predefinito in framework come Next.js che li supportano nativamente.

Le azioni server, invece, sono funzioni che vengono eseguite sul server per permettere la gestione dei moduli direttamente sul server senza bisogno di endpoint API.

Un’azione server si presenta così:

```js
"use server";

async function submitForm(formData) {
  const name = formData.get("name");
  return { message: `Hello, ${name}!` };
}
```

Questa azione server estrae un campo `name` da un modulo e restituisce una stringa che saluta quel nome.

Per semplificare la gestione dello stato per le azioni server e rimuovere la necessità di JavaScript lato client per moduli semplici, il team di React ha introdotto il hook `useActionState` nella versione 19.

Diamo un’occhiata più da vicino a questo hook e vediamo come funziona.

La documentazione di React descrive il hook `useActionState` come un hook che "ti permette di aggiornare lo stato in base al risultato di un’azione su un modulo."

Ma questo non significa che puoi usare il hook `useActionState` solo con i moduli. Puoi anche usarlo per gestire clic su pulsanti e altri eventi, purché ci sia un’azione in atto.

E tieni presente che, dato che `useActionState` è un hook, non puoi usarlo all’interno di un componente server.

Ecco la sintassi base del hook `useActionState`:

```js
const [state, action, isPending] = useActionState(actionFunction, initialState, permalink);
```

- `state` è lo stato corrente che l’azione restituisce.

- `action` è la funzione che attiva l’azione server.

- `isPending` è un booleano che indica se l’azione è in esecuzione o meno.

- Il parametro `actionFunction` è l’azione server stessa.

- `initialState` è il parametro che rappresenta il punto di partenza per lo stato prima che l’azione venga eseguita.

- `permalink` è una stringa opzionale che contiene l’URL unico della pagina che il modulo modifica.

Per usare il hook `useActionState`, assicurati prima di avere un’azione in atto. Usiamo l’azione dell’esempio precedente, con una piccola modifica:

```js
"use server";

export async function submitForm(_, formData) {
  const name = formData.get("name");

  const hour = new Date().getHours();
  let greeting;

  if (hour < 12) {
    greeting = "Good morning";
  } else if (hour < 18) {
    greeting = "Good afternoon";
  } else {
    greeting = "Good evening";
  }

  return { message: `${greeting}, ${name}` };
}
```

Nel tuo componente, devi poi importare il hook `useActionState` e chiamarlo al livello superiore del corpo del componente (prima dell’istruzione return) proprio come gli altri hook. Dovresti anche importare l’azione:

```jsx
"use client";

// Import the useActionState hook
import { useActionState } from "react";

// Import the submitForm action
import { submitForm } from "./actions/submitForm";

const Greeter = () => {

 // Initialize the hook
 const [state, submit, isPending] = useActionState(submitForm, {
   message: "",
 });

  return (
    <div className="flex flex-col items-center justify-center min-h-screen bg-gray-100 p-6">
      {/* Rest of component */}
    </div>
  );
};

export default Greeter;
```

Ecco come appare il codice completo con un po’ di definizione dello stile:

```jsx
"use client";

import { useActionState } from "react";
import { submitForm } from "./actions/submitForm";

const Greeter = () => {
  const [state, submit, isPending] = useActionState(submitForm, {
    message: "",
  });

  return (
    <div className="flex flex-col items-center justify-center min-h-screen bg-gray-100 p-6">
      <form
        action={submit}
        className="bg-white p-6 rounded-2xl shadow-md w-full max-w-md"
      >
        <h2 className="text-2xl text-center font-semibold text-gray-700 mb-4">
          Greet Someone
        </h2>

        <input
          type="text"
          name="name"
          placeholder="Enter your name"
          required
          className="w-full p-3 border border-gray-300 rounded-lg focus:outline-none focus:ring-2 focus:ring-green-400"
        />

        <button
          type="submit"
          disabled={isPending}
          className="w-full mt-4 p-3 bg-green-500 text-white font-semibold rounded-lg hover:bg-green-600 disabled:bg-gray-400 transition-all"
        >
          {isPending ? "Greeting..." : "Greet"}
        </button>

        {state.message && (
          <p className="mt-4 text-green-600 text-center font-medium">
            {state.message}
          </p>
        )}
      </form>
    </div>
  );
};

export default Greeter;
```

Nel browser, vedresti il pulsante del modulo cambiare da `Greet` a `Greeting...` mentre l’azione `isPending` è in corso — e il saluto mostrerebbe `Good morning, {name}`, `Good afternoon, {name}` o `Good evening, {name}`, a seconda dell’ora del giorno in cui il modulo è stato inviato.

Ricordi quando abbiamo detto che puoi usare il hook `useActionState` anche fuori da un modulo?

In questo esempio, recuperiamo cinque utenti da JSONPlaceholder con un clic su un pulsante:

```js
"use server";

export async function getUsers() {
  const res = await fetch(
    "https://jsonplaceholder.typicode.com/users?_start=0&_limit=5/"
  );
  return await res.json();
}
```

Ecco l’interfaccia stilizzata:

```jsx
"use client";

import { useActionState } from "react";
import { getUsers } from "./actions/getUsers";

export default function FetchUsers() {
  const [users, fetchAction, isPending] = useActionState(getUsers, []);

  return (
    <div className="p-6 max-w-lg mx-auto">
      <button
        onClick={fetchAction}
        disabled={isPending}
        className="px-4 py-2 cursor-pointer bg-green-500 text-white rounded-lg hover:bg-green-600 disabled:bg-gray-400 font-bold"
      >
        {isPending ? "Fetching Users..." : "Fetch Users"}
      </button>

      <ul className="mt-4 space-y-2">
        {users.map((user) => (
          <li key={user.id} className="p-3 bg-gray-100 rounded-lg">
            <p className="font-semibold">{user.name}</p>
            <p className="text-sm text-gray-600">{user.email}</p>
          </li>
        ))}
      </ul>
    </div>
  );
}
```

Nel browser, vedresti che il testo del pulsante non viene mai aggiornato a `Fetching Users...` dopo il clic.

Questo succede perché React tratta il recupero dati e il rendering come una priorità più alta rispetto allo stato `isPending`, che blocca `isPending` nel processo e lancia un errore.

Per risolvere questo problema, devi avvolgere l’azione in `startTransition`:

```jsx
"use client";

// import startTransition from React
import { useActionState, startTransition } from "react";
import { getUsers } from "./actions/getUsers";

export default function FetchUsers() {
  const [users, fetchAction, isPending] = useActionState(getUsers, []);

  return (
    <div className="p-6 max-w-lg mx-auto">
      <button
        {/* wrap fetchAction in startTransition */}
        onClick={() => startTransition(() => fetchAction())}
        disabled={isPending}
        className="px-4 py-2 bg-green-500 font-bold cursor-pointer text-white rounded-lg hover:bg-green-600 disabled:bg-gray-400"
      >
        {isPending ? 'Fetching Users...' : 'Fetch Users'}
      </button>

      <ul className="mt-4 space-y-2">
        {users.map((user) => (
          <li key={user.id} className="p-3 bg-gray-100 rounded-lg">
            <p className="font-semibold">{user.name}</p>
            <p className="text-sm text-gray-600">{user.email}</p>
          </li>
        ))}
      </ul>
    </div>
  );
}
```

Se ti stai chiedendo cos’è `startTransition`, è una funzione che dice a React che un aggiornamento dello stato ha bassa priorità e può essere interrotto. Questo mantiene l’interfaccia responsiva mentre gestisce aggiornamenti asincroni come le azioni server.

## --answers--

Gestisce lo stato in base al risultato di un’azione, inclusi invii di moduli, clic su pulsanti e altri aggiornamenti guidati da eventi.
