# --description--

작은 바닐라 HTML, CSS, JavaScript 프로젝트와 달리, 새로운 React 프로젝트를 시작할 때는 모든 것이 제대로 작동하도록 몇 가지 단계를 더 거쳐야 합니다. 필요한 모든 설정을 직접 하려고 하기보다는, 이를 대신해주는 도구들이 있습니다.

프로젝트 설정에 가장 인기 있는 도구 중 하나가 Vite입니다. 프랑스어로 "빠르다"는 뜻의 Vite는 최신 웹 프로젝트를 위한 더 빠른 개발 환경을 제공하는 빌드 도구입니다. Vite는 React뿐만 아니라 Vue나 Svelte 같은 다른 라이브러리와 프레임워크와도 함께 사용할 수 있습니다. 심지어 바닐라 JavaScript 프로젝트에도 사용할 수 있습니다.

Vite로 새 프로젝트를 만들려면 명령줄을 사용해야 합니다. Windows를 사용한다면 명령 프롬프트나 Windows PowerShell을 사용할 수 있습니다. Mac을 사용한다면 터미널 앱을 사용하세요.

npm이 작동하려면 필요한 Node.js가 설치되어 있지 않다면, 공식 Node.js 웹페이지 https://nodejs.org 에서 설치 지침을 따라 설치한 후 여기로 돌아와 React 설치를 계속하세요.

명령줄을 열었으면 다음 명령어를 사용할 수 있습니다:

```bash
npm create vite@latest my-react-app -- --template react
```

이 명령어는 Vite의 React 템플릿을 사용해 `my-react-app`라는 이름의 새 React 프로젝트를 만듭니다.

그다음 새 프로젝트를 열어 Vite가 제공한 React 보일러플레이트 코드를 확인할 수 있습니다.

Vite의 좋은 점은 React 프로젝트를 시작하는 데 절대적으로 필요한 파일만 제공한다는 것입니다.

프로젝트를 실제로 실행하려면 명령줄에서 다음 명령어로 의존성을 설치해야 합니다:

```bash
cd my-react-app
npm install
```

`cd my-react-app` 명령어는 프로젝트가 위치한 올바른 디렉터리로 이동하는 것을 보장합니다.

`npm install` 명령어는 React 프로젝트를 제대로 실행하는 데 필요한 의존성을 설치하는 데 사용됩니다. 의존성은 React 자체, ReactDOM, 또는 추가 기능을 제공하는 다른 서드파티 패키지처럼 React 프로젝트가 제대로 작동하려면 필요한 라이브러리나 패키지입니다.

`npm install`을 실행하면 프로젝트 디렉터리 내 `package.json` 파일을 읽어 거기에 나열된 모든 의존성을 설치하여 필요한 구성 요소 없이 React 앱을 빌드하고 실행할 수 있게 합니다.

`package.json` 파일은 프로젝트에 대한 메타데이터(이름, 버전, 의존성 등)를 포함하는 핵심 구성 파일입니다. 또한 스크립트, 라이선스 정보, 기타 설정을 정의해 프로젝트와 의존성을 관리하는 데 도움을 줍니다.

의존성이 설치되면 프로젝트 내에 `node_modules`라는 새 폴더가 생긴 것을 볼 수 있습니다.

`node_modules` 폴더는 프로젝트에 필요한 모든 패키지와 라이브러리가 저장되는 곳입니다. 이 폴더에는 `package.json` 파일에 나열된 의존성들의 실제 코드가 포함되어 있으며, 프로젝트의 직접 의존성과 그 의존성들의 의존성 모두를 포함합니다.

프로젝트를 실행하려면 `npm run dev` 명령어를 실행하고 `http://localhost:5173/`에서 새 브라우저 탭을 열면 됩니다.

Vite가 제공한 시작 템플릿을 볼 수 있을 것입니다.

로컬 서버를 중지하려면 명령줄에서 `CTRL + C`를 누르세요.

이 시작 템플릿의 코드를 실제로 보려면 프로젝트 내 `src` 폴더로 들어가 `App.jsx` 파일을 확인하세요.

그다음 파일을 수정하고 저장하면 브라우저에서 변경된 내용을 바로 확인할 수 있습니다.

이제 React 애플리케이션 개발을 시작할 준비가 되었습니다!
