# --description--

在之前的课程中，你学习了如何使用类型别名，并看到了如何命名对象类型的示例。 

正如你所知，命名一个对象 `type` 的一种方法是使用 `type` 关键字，如下所示：

```ts
type ID = number | string;

type User = {
  id: ID;
  name: string;
};
```

另一种命名对象类型的方法是使用如下的 `interface` 声明：

```ts
type ID = number | string;

interface User {
  id: ID;
  name: string;
}
```

`type` 和 `interface` 类似，但一个关键区别是接口使用 `extends` 关键字来基于其他接口构建，而 `type` 别名使用交集类型（`&`）来组合类型。

让我们来看一些例子：

```ts
interface User {
  id: number | string;
  name: string;
}

interface Admin extends User {
  role: string;
}

const adminUser: Admin = {
  id: 101,
  name: "Alice",
  role: "superadmin"
};
```

在此示例中，`User` 从 `id` 和 `name` 开始。`Admin` 接口扩展了 `User`，这意味着它包含 `id` 和 `name`，并添加了一个名为 `role` 的新属性。

如果你想扩展一个 `type`，你会像这样使用交集 `type`（`&`）：

```ts
type User = {
  id: number | string;
  name: string;
};

type Admin = User & {
  role: string;
};

const adminUser: Admin = {
  id: 101,
  name: "Alice",
  role: "superadmin"
};
```

`Admin` 将 `User` 类型与包含 `role` 的新建对象类型组合。结果与接口示例的形状相同，即具有 `id`、`name` 和 `role` 的对象。

`type` 和 `interface` 都可以描述对象的形状。如果你不确定使用哪一个，请记住 `interface` 是专门为对象形状设计的，而 `type` 更加灵活，也可以表现并集类型、原语以及更复杂的类型组合。
