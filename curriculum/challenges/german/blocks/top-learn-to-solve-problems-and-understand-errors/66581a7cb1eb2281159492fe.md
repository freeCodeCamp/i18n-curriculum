# --description--

Nehmen wir an, Sie haben den folgenden Code geschrieben:

```javascript
const a = "Hello";
const b = "World";

console.log(c);
```

Dieser Code wird ausgeführt, aber er erzeugt einen Fehler. Technisch gesehen nennt man das, einen Fehler zu <dfn>werfen</dfn>. Der erste Teil eines Fehlers zeigt die Art des Fehlers an. Das gibt den ersten Hinweis darauf, womit Sie es zu tun haben. Sie werden später in der Lektion mehr über die verschiedenen Fehlertypen lernen. In diesem Beispiel haben Sie einen `ReferenceError`.

<img src="https://cdn.statically.io/gh/TheOdinProject/curriculum/175b5ef2a1b4758a7b75f4ef43d7e27203e5707b/foundations/javascript_basics/understanding_errors/imgs/00.png" width="100%" alt="Ein Fehler, der in der Entwicklerkonsole angezeigt wird"/>

Ein `ReferenceError` wird geworfen, wenn auf eine Variable verwiesen wird, die im aktuellen Gültigkeitsbereich nicht deklariert und/oder initialisiert wurde. In unserem Fall erklärt die Fehlermeldung, dass der Fehler aufgetreten ist, weil `c is not defined`.

Verschiedene Fehler dieses Typs haben unterschiedliche Meldungen, je nachdem, was den `ReferenceError` verursacht. Zum Beispiel könnte eine andere Meldung, auf die Sie stoßen, `ReferenceError: can't access lexical declaration 'X' before initialization` sein.

Der nächste Teil eines Fehlers gibt uns den Namen der Datei an, in der Sie den Fehler finden können (in diesem Fall unsere `script.js`), sowie die Zeilennummer.

Das ermöglicht es Ihnen, leicht zur problematischen Zeile in Ihrem Code zu navigieren. Hier stammt der Fehler aus der vierten Zeile von `script.js`, die als Link unter der Fehlermeldung mit dem Text bei `script.js:4` angezeigt wird. Wenn Sie auf diesen Link klicken, navigieren die meisten Browser zur genauen Codezeile und dem Rest Ihres Skripts im Quellen-Tab der Entwicklerwerkzeuge.

Manchmal zeigt die Konsole Ihres Browsers auch die Spalte (oder das Zeichen) in der Zeile an, an der der Fehler auftritt. In unserem Beispiel wäre das bei `script.js:4:13`.
