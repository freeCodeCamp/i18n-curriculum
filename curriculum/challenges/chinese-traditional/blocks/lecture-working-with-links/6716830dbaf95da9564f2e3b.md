# --description--

你可能看過像 `/public/logo.png`、`./script.js` 或 `../styles.css` 這樣的聯結。但這些特殊類型的聯結代表什麼意思？這些稱為檔案路徑。有三種主要的語法需要了解。第一種是斜線，根據你的作業系統，可以是反斜線（`\`）或正斜線（`/`）。第二種是單點（`.`）。最後，我們有雙點（`..`）。

斜線被稱為「路徑分隔符」。它用來顯示資料夾或檔案名稱之間文字的中斷。這就是你的電腦知道 `naomis-files/` 指向同名的目錄，而 `naomis/files/` 指向 `naomis` 目錄中的 `files` 目錄的原因。

單一個點表示目前目錄，兩個點表示上層目錄。你通常會看到使用單一個點來確保路徑被識別為相對路徑。請記得你之前的課程中已經學過相對路徑與絕對路徑的差異。

不過，雙點在存取目前工作目錄外的檔案時更為常見。

例如，給定這個檔案樹：

```sh
my-app/
├─ public/
│  ├─ favicon.ico
│  ├─ index.html
├─ src/
│  ├─ index.css
│  ├─ index.js
```

如果你的 `public/index.html` 檔案需要載入 `favicon.ico` 檔案，你會使用單點的相對路徑來存取目前目錄：`./favicon.ico`。如果你的 `public/index.html` 檔案需要載入 `index.css` 檔案，你會使用雙點的相對路徑先導覽到父目錄 `my-app`，然後到 `src` 目錄，最後到你的檔案：`../src/index.css`。
