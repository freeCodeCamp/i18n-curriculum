# --description--

Finora hai scritto codice client-side con JavaScript, codice che viene eseguito sul lato client di un'applicazione web, di solito un browser web.

Il JavaScript client-side può essere molto potente, ma ha alcune limitazioni per attività più complesse.

Per esempio:

- Ha un accesso molto limitato ai file locali.
- Non è progettato per gestire logiche applicative complesse.
- E presenta potenziali problemi di sicurezza. Per esempio, non vorresti esporre informazioni sensibili, come le credenziali del database, al codice client-side.
    
Il JavaScript client-side ha queste limitazioni perché è stato inizialmente progettato per essere eseguito esclusivamente nei browser web.

I browser web forniscono l'ambiente necessario per eseguire codice JavaScript, incluso il motore JavaScript. Offrono anche l'accesso al Document Object Model (DOM), così puoi consultare gli elementi HTML nel tuo codice.

In passato non potevamo eseguire JavaScript al di fuori di un browser, ma ora possiamo farlo con Node.js.

Node.js è un ambiente di esecuzione JavaScript. È definito ufficialmente come “un ambiente di esecuzione JavaScript open source e multipiattaforma”.

Vediamo cosa significa:

- Node.js è open source perché il suo codice è pubblicamente disponibile e mantenuto da una grande comunità di sviluppatori.
- È multipiattaforma perché funziona su qualsiasi sistema operativo, inclusi Windows, macOS e Linux.
- Ed è un ambiente di esecuzione JavaScript perché ti permette di eseguire codice JavaScript al di fuori di un browser.
    
Node.js è usato in molti settori diversi. Gli sviluppatori lo usano per costruire server web e API che gestiscono richieste HTTP, oltre ad applicazioni web e mobile.

Se stai usando o sviluppando un sito web o un'applicazione web che interagisce con un database o gestisce dati complessi, c'è una buona probabilità che userai Node.js dietro le quinte a un certo punto.

È efficiente, scalabile e ha una grande comunità di sviluppatori e manutentori in tutto il mondo.

Ora che sai cos'è Node.js, vediamo alcune differenze tra il browser e l'ambiente di esecuzione Node.

L'ambiente del browser è progettato principalmente per lo sviluppo front-end, quindi esegue JavaScript client-side. Al contrario, l'ambiente di esecuzione Node è progettato principalmente per lo sviluppo back-end, quindi esegue JavaScript server-side. Questo determina anche le API che offrono.

Per esempio, dal browser puoi consultare l'API DOM, ma ci sono restrizioni per l'accesso al file system locale. Al contrario, dall'ambiente di esecuzione Node puoi consultare quasi tutte le risorse di sistema, incluso il file system, ma non il DOM.

Entrambi gli ambienti JavaScript forniscono un oggetto globale. Nell'ambiente JavaScript del browser, questo si chiama `window`. Questo oggetto `window` fornisce accesso a funzionalità legate al browser, come metodi per manipolare il DOM, gestire i cookie e gestire eventi del browser. Nell'ambiente Node.js, questo oggetto globale si chiama `global`. L'oggetto `global` fornisce accesso a funzionalità specifiche di Node.js, come moduli integrati per lavorare con file locali, networking e interazione con il sistema operativo.

Infine, un'altra differenza importante è che puoi scegliere la versione di Node.js che vuoi usare nel tuo server. Tuttavia, non hai alcun controllo sulla versione dell'ambiente browser che i tuoi utenti useranno per visitare il tuo sito web.

Node.js è diventato molto importante per gli sviluppatori in tutto il mondo. È uno strumento potente, scalabile e versatile che ha trasformato lo sviluppo web permettendo agli sviluppatori di usare JavaScript per costruire sia il front-end sia il back-end di applicazioni full-stack. Questo ha reso il processo di sviluppo più efficiente perché gli sviluppatori non devono imparare un nuovo linguaggio di programmazione solo per sviluppare il back-end.
