# --interactive--

`reduce` メソッドは、JavaScriptの関数で、配列を処理して単一の値にまとめることができます。この単一の値は、数値、文字列、オブジェクト、あるいは別の配列である場合もあります。

これは配列を単一の出力に「縮小」するため、`reduce` と呼ばれています。最初は複雑に見えるかもしれませんが、`reduce` を理解すると多くの状況でコードを大幅に簡単にできます。

基本的に、`reduce` は配列の各要素に順番に関数を適用し、各計算の結果を次に渡していくことで動作します。この関数はしばしばリデューサー関数と呼ばれます。

リデューサー関数は主に2つのパラメータを取ります：アキュムレータと現在の値です。アキュムレータは処理の途中結果を保存する場所で、現在の値は処理中の配列の要素です。

`reduce` の動作を示す例を見てみましょう：

:::interactive_editor

```js
const numbers = [1, 2, 3, 4, 5];
const sum = numbers.reduce(
  (accumulator, currentValue) => accumulator + currentValue,
  0
);

console.log(sum); // 15
```

:::

この例では、`reduce` を使って配列内のすべての数の合計を求めています。

リデューサー関数はアキュムレータ（`0` で始まり、`reduce` の2番目の引数で指定されています）を取り、各数値を加算します。

各加算の結果が次のイテレーションの新しいアキュムレータになります。

`reduce` メソッドは2番目の引数として初期値を取ることもできます。これはアキュムレータが開始する値です。上の例では `0` に設定しています。

初期値を指定しない場合、`reduce` は配列の最初の要素を初期アキュムレータとして使い、2番目の要素から処理を開始します。

`reduce` の素晴らしい点の一つはその柔軟性です。リデューサー関数を自分で定義するため、配列の処理方法や生成したい結果の種類を完全にコントロールできます。これにより `reduce` は非常に強力ですが、最初は理解が少し難しい場合もあります。

練習すれば、`reduce` メソッドの使い方に慣れていきます。
