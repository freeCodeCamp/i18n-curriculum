# --interactive--

이진 논리 연산자는 두 표현식을 평가하고 그 진릿값에 따라 결과를 반환하는 데 도움을 줍니다. 가장 흔히 쓰이는 세 가지 이진 논리 연산자, 즉 논리 AND, 논리 OR, 그리고 nullish 병합 연산자를 살펴보겠습니다.

논리 AND 연산자는 이중 앰퍼샌드(`&&`)로 나타냅니다. 두 피연산자가 모두 참인지 확인하고 결과를 반환합니다. 두 피연산자가 모두 참이면 오른쪽에 있는 두 번째 값을 반환합니다:

:::interactive_editor

```js
const result = true && 'hello';

console.log(result); // hello
```

:::

위 예제에서 텍스트 `hello`가 콘솔에 출력되는 이유는 두 피연산자가 모두 `true`이기 때문입니다. 만약 어느 한 쪽 피연산자가 거짓 같은 값이면, 그 거짓 같은 값을 반환합니다:

:::interactive_editor

```js
const result = 0 && 3;

console.log(result); // 0
```

:::

`0`가 거짓 같은 값이므로 숫자 `0`가 콘솔에 출력됩니다. 그리고 두 피연산자가 모두 거짓 같은 값이면 첫 번째 거짓 같은 값을 반환합니다:

:::interactive_editor

```js
const result = false && 0;

console.log(result); // false
```

:::

`false`가 거짓 같은 값이므로 `false`가 콘솔에 출력됩니다. 논리 AND 연산자는 여러 조건을 확인하고 모두 참일 때만 진행하려 할 때 유용합니다. 다음은 예시입니다:

:::interactive_editor

```js
if (2 < 3 && 3 < 4) {
 console.log('The if block runs'); 
} else {
 console.log('The else block runs');
} 
```

:::

조건에서 `2`가 `3`보다 작고 `3`가 `4`보다 작으므로 문장 `The if block runs`이 콘솔에 출력됩니다.

논리 OR 연산자는 적어도 하나의 피연산자가 참인지 확인합니다. 첫 번째 피연산자가 참이면 그 값을 반환합니다:

:::interactive_editor

```js
const result = 'This is truthy' || false;

console.log(result); // This is truthy
```

:::

첫 번째 피연산자가 거짓 같은 값이지만 두 번째가 참이면 두 번째 값이 콘솔에 출력됩니다:

:::interactive_editor

```js
const result = 0 || 'This is truthy';

console.log(result); // This is truthy
```

:::

논리 OR 연산자는 `if/else` 문장 같은 곳에서 자주 사용됩니다:

:::interactive_editor

```js
let userInput;

if (userInput || 'Guest') {
 console.log('A user is present');
} else {
 console.log('No user detected');
}
```

:::

`userInput` 변수에 값을 할당하지 않았기 때문에 현재 `undefined`입니다. `if` 문장의 조건은 `userInput` 변수나 문자열 `Guest` 중 하나라도 참인지 확인합니다. 문자열 `Guest`는 불리언 문맥에서 참이므로 문자열 `A user is present`가 콘솔에 출력됩니다.

nullish 병합 연산자는 논리 OR와 논리 AND보다 더 정교합니다. 이중 물음표(`??`)로 나타내며, 첫 번째 값이 `null` 또는 `undefined`일 때만 값을 반환하는 상황에 유용합니다. nullish 병합 연산자를 사용하는 예는 다음과 같습니다:

:::interactive_editor

```ts
const result = null ?? 'default';

console.log(result); // default
```

:::

`null`가 nullish 값이므로 문자열 `default`가 콘솔에 출력됩니다. nullish 병합 연산자는 `null` 또는 `undefined`만 대체 수단이나 기본값을 트리거해야 하는 상황에서 매우 유용합니다. 다음은 사용자의 선호 설정을 다루는 예입니다:


:::interactive_editor

```ts
const userSettings = {
 theme: null,
 volume: 0,
 notifications: false,
};

let theme = userSettings.theme ?? 'light';
console.log(theme); // light
```

:::

위 예제에서 `userSettings`라는 객체가 `theme`, `volume`, `notifications` 프로퍼티를 포함하고 있습니다. 점 표기법을 사용해 `theme`에 `userSettings.theme`처럼 접근합니다. 객체 작업 방법은 이후 수업에서 더 배우게 됩니다. 사용자의 `theme`가 현재 `null`로 설정되어 있으므로 문자열 `light`가 콘솔에 출력됩니다.
