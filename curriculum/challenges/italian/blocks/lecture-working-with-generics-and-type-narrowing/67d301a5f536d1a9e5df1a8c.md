# --description--

Ci saranno momenti in cui hai un valore con un tipo generico e devi restringerlo a un tipo più specifico. Per esempio, potresti dover assicurarti che un oggetto corrisponda a un’interfaccia che hai definito. Oppure che una stringa sia all’interno di una lista specifica di valori. Ci sono diversi modi per ottenere questo risultato.

Il primo è il narrowing basato sulla veridicità (truthiness). Considera il nostro esempio della lezione precedente:

```js
const email = document.querySelector<HTMLInputElement>("#email");
console.log(email.value);
```

Otteniamo un errore del compilatore tentando di consultare la proprietà `value` di `email`, perché `email` potrebbe essere `null`. Tuttavia, possiamo usare un’istruzione condizionale per confermare che `email` è truthy prima di consultare la proprietà:

```js
const email = document.querySelector<HTMLInputElement>("#email");
if (email) {
  console.log(email.value);
}
```

In questo esempio aggiornato, poiché `null` non è un valore truthy, TypeScript riesce a dedurre che `email` DEVE essere un elemento `input` all’interno del blocco condizionale. Quindi non genera più un errore del compilatore.

I controlli di truthiness possono funzionare anche al contrario:

```js
const email = document.querySelector<HTMLInputElement>("#email");
if (!email) {
  throw new ReferenceError("Could not find email element!")
}
console.log(email.value);
```

Con questo approccio, lanciamo un errore se `email` è falsy. `null` è un valore falsy. Lanciare un errore termina l’esecuzione logica di questo codice, il che significa che quando arriviamo alla chiamata `console.log()` TypeScript sa che `email` non può essere `null`.

Il concatenamento opzionale è anche una forma di type narrowing, basata sullo stesso principio che la consultazione della proprietà non può avvenire se il valore `email` è `null`.

```js
const email = document.querySelector<HTMLInputElement>("#email");
console.log(email?.value);
```

Ma per altri tipi? Puoi anche restringere i tipi usando l’operatore `typeof`. Vediamo un esempio di una variabile che abbiamo indicato potrebbe essere una stringa O un numero:

```js
const myVal = Math.random() > 0.5 ? 222 : "222";
console.log(myVal / 10)
```

In questo esempio, vediamo un errore del compilatore perché non possiamo fare operazioni aritmetiche su una stringa. Ma possiamo usare un condizionale per controllare il `typeof` della variabile `myVal`:

```js
const myVal = Math.random() > 0.5 ? 222 : "222";
if (typeof myVal === "number") {
  console.log(myVal / 10);
}
```

Poiché abbiamo usato la parola chiave `typeof`, TypeScript ora sa che `myVal` deve essere un numero, e possiamo fare operazioni aritmetiche in sicurezza.

Ma per tipi di oggetti più complessi? Se l’oggetto in questione proviene da una classe, puoi effettivamente usare la parola chiave `instanceof` per restringere il tipo. Tornando al nostro esempio `querySelector()`:

```js
const email = document.querySelector("#email");
```

Invece di passare un tipo generico e dire a TypeScript cos’è l’elemento, possiamo usare `instanceof` per restringere il tipo e scrivere codice più sicuro:

```js
const email = document.querySelector("#email");

if (email instanceof HTMLInputElement) {
    console.log(email.value);
}
```

Questo approccio può sembrare uguale al precedente, ma `instanceof` è una validazione a runtime — il che significa che, se per qualche motivo abbiamo sbagliato il tipo TypeScript, il nostro codice JavaScript confermerà comunque che `email` è un elemento `input`.

Ora, vediamo un esempio in cui recuperiamo un oggetto `User` da un’API e proviamo a stampare le informazioni:

```js
interface User {
    name: string;
    age: number;
}

const printAge = (user: User) => 
  console.log(`${user.name} is ${user.age} years old!`)

const request = await fetch("url")
const myUser = await request.json();
printAge(myUser);
```

Otteniamo un errore del compilatore tentando di passare `myUser` alla funzione perché, anche se sappiamo che l’API restituisce l’oggetto corretto, TypeScript non lo sa. E il metodo `.json()` non accetta un tipo generico.

Il modo “facile” per risolvere questo problema sarebbe fare un cast del tipo:

```js
interface User {
    name: string;
    age: number;
}

const printAge = (user: User) => 
  console.log(`${user.name} is ${user.age} years old!`)

const request = await fetch("url")
const myUser = await request.json() as User;
printAge(myUser);
```

Ma ogni volta che fai un cast del tipo, stai di fatto indebolendo la capacità di TypeScript di intercettare potenziali errori. Quindi invece di fare un cast, puoi scrivere un type guard:

```js
interface User {
    name: string;
    age: number;
}

const isValidUser = (user: unknown): user is User => {
  return !!user && 
    typeof user === "object" &&
    "name" in user &&
    "age" in user;
}
```

Il tipo di ritorno qui è la componente chiave di questa definizione di funzione. La sintassi `user is User` indica che la nostra funzione restituisce un valore booleano, che quando `true` significa che il valore `user` soddisfa l’interfaccia `User`. Facciamo poi alcuni controlli di base per assicurarci che la struttura dell’oggetto `user` corrisponda — nota l’uso di un narrowing basato sulla truthiness (`!!user`) e un narrowing `typeof`. Dobbiamo farlo perché `typeof null` restituisce `"object"`:

```js
interface User {
    name: string;
    age: number;
}

const isValidUser = (user: unknown): user is User => {
  return !!user && 
    typeof user === "object" &&
    "name" in user &&
    "age" in user;
}

const printAge = (user: User) => 
  console.log(`${user.name} is ${user.age} years old!`)

const request = await fetch("url")
const myUser = await request.json() as User;
if (isValidUser(myUser)) {
    printAge(myUser);
}
```

Ora, se combiniamo tutta la nostra logica, non otteniamo più errori del compilatore e possiamo buildare con successo il nostro codice.

Il type narrowing è una funzionalità potente che ti aiuta a scrivere codice più sicuro e meno soggetto a errori — ma ricorda che i tipi di TypeScript non sono completamente rigidi, quindi evita pratiche come fare il cast del tipo di un valore senza restringerlo.
