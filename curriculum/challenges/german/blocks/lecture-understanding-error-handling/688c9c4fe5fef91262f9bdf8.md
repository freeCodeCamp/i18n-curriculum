# --description--

In Python ist die `raise`-Anweisung ein mächtiges Werkzeug, mit dem Sie Ausnahmen in Ihrem Code manuell auslösen können. Sie gibt Ihnen die Kontrolle darüber, wann und wie Fehler erzeugt werden, sodass Sie benutzerdefinierte Fehlerbedingungen erstellen und ein bestimmtes Programmverhalten erzwingen können.

Die `raise`-Anweisung wird verwendet, um an einer beliebigen Stelle in Ihrem Programm explizit eine Ausnahme zu werfen, wodurch Sie signalisieren können, dass ein Fehlerzustand eingetreten ist oder bestimmte Anforderungen nicht erfüllt wurden.

Die `raise`-Anweisung in Python kann auf verschiedene Weise verwendet werden, um Ausnahmen auszulösen. Im einfachsten Fall können Sie eingebaute Ausnahmen auslösen oder benutzerdefinierte Fehlermeldungen erstellen. Hier ist ein einfaches Beispiel:

```py
def check_age(age):
    if age < 0:
        raise ValueError('Age cannot be negative')
    return age

try:
    check_age(-5)
except ValueError as e:
    print(f'Error: {e}') # Error: Age cannot be negative
```

Sie sehen hier, dass `raise` das Schlüsselwort ist, das eine Ausnahme auslöst.

In diesem Beispiel lösen wir eine `ValueError` mit einer benutzerdefinierten Nachricht aus, wenn ein ungültiges Alter angegeben wird.

Die `raise`-Anweisung kann auch verwendet werden, um die aktuelle Ausnahme erneut zu werfen, was besonders nützlich bei der Ausnahmebehandlung ist:

```py
def process_data(data):
    try:
        result = int(data)
        return result * 2
    except ValueError:
        print('Logging: Invalid data received')
        raise  # Re-raises the same ValueError

try:
    process_data('abc')
except ValueError:
    print('Handled at higher level')
```

Hier wirft das Schlüsselwort `raise` (ohne Argumente) die gerade behandelte Ausnahme erneut.

Das ermöglicht es Ihnen, Protokolle zu erstellen oder Aufräumarbeiten durchzuführen, während der Fehler weiterhin im Aufrufstapel nach oben propagiert wird.

Sie können benutzerdefinierte Ausnahmen erstellen und auslösen, indem Sie eigene Ausnahme-Klassen definieren:

```py
class InsufficientFundsError(Exception):
    def __init__(self, balance, amount):
        self.balance = balance
        self.amount = amount
        super().__init__(f'Insufficient funds: ${balance} available, ${amount} requested')

def withdraw(balance, amount):
    if amount > balance:
        raise InsufficientFundsError(balance, amount)
    return balance - amount

try:
    new_balance = withdraw(100, 150)
except InsufficientFundsError as e:
    print(f'Transaction failed: {e}')
```

Hier sehen Sie, dass benutzerdefinierte Ausnahme-Klassen von `Exception` oder dessen Unterklassen erben.

Mehr über Klassen und Vererbung lernen Sie in zukünftigen Lektionen. Für den Moment wissen Sie, dass dies eine Möglichkeit ist, eigene Ausnahmen mit benutzerdefinierter Logik zu erstellen.

Die `raise`-Anweisung kann auch zusammen mit dem Schlüsselwort `from` verwendet werden, um Ausnahmen zu verketten und so die Beziehung zwischen verschiedenen Fehlern zu zeigen:

```py
def parse_config(filename):
    try:
        with open(filename, 'r') as file:
            data = file.read()
            return int(data)
    except FileNotFoundError:
        raise ValueError('Configuration file is missing') from None
    except ValueError as e:
        raise ValueError('Invalid configuration format') from e

config = parse_config('config.txt')
```

Hier sehen Sie, dass `raise ... from None` den ursprünglichen Ausnahme-Kontext unterdrückt:

```bash
Traceback (most recent call last):
  File "main.py", line 12, in <module>
    config = parse_config('config.txt')
             ^^^^^^^^^^^^^^^^^^^^^^^^^^
  File "main.py", line 7, in parse_config
    raise ValueError('Configuration file is missing') from None
ValueError: Configuration file is missing
```

Und `raise ... from e` verknüpft die neue Ausnahme mit der ursprünglichen, wodurch die Fehlerkette erhalten bleibt.

```bash
Traceback (most recent call last):
  File "main.py", line 5, in parse_config
    return int(data)
           ^^^^^^^^^
ValueError: invalid literal for int() with base 10: ''

The above exception was the direct cause of the following exception:

Traceback (most recent call last):
  File "main.py", line 12, in <module>
    config = parse_config('config.txt')
             ^^^^^^^^^^^^^^^^^^^^^^^^^^
  File "main.py", line 9, in parse_config
    raise ValueError('Invalid configuration format') from e
ValueError: Invalid configuration format
```

Sie können Ausnahmen auch bedingt mit `assert`-Anweisungen auslösen, die im Grunde eine Kurzform für `raise` mit `AssertionError` sind:

```py
def calculate_square_root(number):
    assert number >= 0, 'Cannot calculate square root of negative number'
    return number ** 0.5

try:
    result = calculate_square_root(-4)
except AssertionError as e:
    print(f'Assertion failed: {e}')
```

Die `raise`-Anweisung ist essenziell, um robuste Anwendungen zu erstellen, bei denen Sie Geschäftsregeln durchsetzen, Eingaben validieren und aussagekräftige Fehlermeldungen bereitstellen müssen. Durch den strategischen Einsatz von `raise` können Sie Ihren Code vorhersehbarer und leichter debugbar machen und den Nutzenden klare Rückmeldungen geben, was schiefgelaufen ist.
