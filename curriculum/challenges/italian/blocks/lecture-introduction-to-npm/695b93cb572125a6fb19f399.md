# --description--

npm ha bisogno di un modo per archiviare dettagli importanti sul tuo progetto – cose come quali pacchetti usa, la versione di quei pacchetti e quali comandi può eseguire. Lo fa con il file `package.json`.

Pensa al file `package.json` come a un file di configurazione npm per il progetto. È un semplice file JSON che descrive il tuo progetto usando coppie chiave-valore per tenere traccia di cose come:

* il nome e la versione del progetto
    
* l'autore
    
* il file principale che dovrebbe essere eseguito per primo
    
* la licenza
    
* il repository del codice, le parole chiave e le descrizioni
    
* eventuali dipendenze di cui il progetto ha bisogno
    

Ecco come potrebbe apparire un file `package.json`:

```json
{
  "name": "my-app",
  "version": "1.0.0",
  "description": "A simple NPM project",
  "main": "index.js",
  "scripts": {
    "start": "node index.js",
    "test": "echo \"Running tests...\""
  },
  "repository": {
    "type": "git",
    "url": "git+https://github.com/username/project.git"
  },
  "keywords": ["fCC", "js"],
  "author": "Kolade Chris",
  "license": "ISC",
  "dependencies": {
    "react": "^19.2.1"
  },
  "devDependencies": {
    "cypress": "^15.7.1"
  }
}
```

Il campo `main` è spesso impostato su `index.js` e indica a Node.js quale file eseguire per primo.

Il campo `license` serve a specificare i termini legali con cui altre persone possono usare, modificare e distribuire il codice nel tuo pacchetto. Esistono molte licenze diverse come `MIT`, `ISC` o `GPL`, che hanno tutte termini e permessi diversi. La predefinita è `ISC`, la licenza Internet Systems Consortium, che è una licenza open source molto permissiva.

Man mano che il tuo progetto cresce e installi pacchetti, npm li aggiunge sotto una di queste due sezioni:

* `dependencies`, che sono pacchetti di cui la tua app ha bisogno per funzionare in produzione
    
* `devDependencies`, che sono pacchetti di cui hai bisogno solo durante lo sviluppo
    

Per esempio, se il tuo progetto usa React per la sua interfaccia utente, quello va in `dependencies` perché è necessario per far funzionare il progetto una volta distribuito. Uno strumento come Cypress, usato solo per i test end-to-end prima della distribuzione in produzione, appartiene a `devDependencies`.

Ci sono un paio di modi semplici per creare un file `package.json`. Il modo più comune è eseguire questo comando:

```bash
npm init
```

Dovresti eseguirlo nella cartella principale del tuo progetto. Questo comando avvia un processo di configurazione interattivo in cui npm ti fa una serie di domande. Le tue risposte vengono usate per compilare campi come nome del progetto, versione, descrizione, script, parole chiave, autore e licenza.

Se preferisci saltare le domande e creare un file `package.json` con valori predefiniti, puoi eseguire:

```bash
npm init -y
```

o

```bash
npm init --yes
```

Questo crea istantaneamente un file `package.json` per te con valori predefiniti in molti campi.

Di solito è meglio usare `npm init` senza la flag `-y` o `--yes` così puoi passare in rassegna ogni campo e personalizzare le informazioni del tuo progetto. Se non rispondi a una domanda, npm usa semplicemente un valore predefinito. Per esempio, se lasci vuota la domanda sulla licenza, npm la imposta su `ISC` per impostazione predefinita.

Ecco un esempio di cosa vedrai eseguendo `npm init`:

```bash

  "name": "my-app",
  "version": "1.0.0",
  "description": "a description for my-app",
  "main": "index.js",
  "scripts": {
    "test": "echo \"Error: no test specified\" && exit 1"
  },
  "keywords": [
    "fCC",
    "JS"
  ],

  "license": "ISC"

```

Potresti notare che `dependencies` e `devDependencies` non sono ancora elencati. Questo perché queste sezioni vengono aggiunte solo dopo che installi il primo pacchetto.

Noterai anche che il campo `repository` manca. Anche se npm lo ha chiesto durante `npm init`, non lo compila automaticamente in seguito. Per esempio, quando esegui comandi come `git remote add origin` o `git push`, npm non aggiorna il campo repository per te — devi aggiungerlo manualmente se vuoi che sia incluso.

Nelle prossime lezioni imparerai di più sulle dipendenze in `package.json` e come installarle, aggiornarle e rimuoverle.
