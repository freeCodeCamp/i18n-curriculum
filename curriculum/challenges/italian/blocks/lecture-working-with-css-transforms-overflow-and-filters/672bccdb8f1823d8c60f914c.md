# --interactive--

Un reset CSS è un foglio di stile che rimuove tutta o parte della formattazione predefinita che i browser web applicano agli elementi HTML. Per esempio, potresti aver notato che alcuni elementi, come paragrafi e intestazioni, hanno già dei margini per impostazione predefinita anche se non li definisci esplicitamente nei tuoi fogli di stile personalizzati.

Vedrai questo comportamento anche con varie proprietà in un’ampia gamma di elementi HTML. Browser diversi possono inoltre applicare stili predefiniti diversi agli elementi HTML. Gli stili predefiniti in Google Chrome potrebbero non essere esattamente gli stessi di Mozilla Firefox o Microsoft Edge. Questo può causare stili incoerenti tra browser, cosa che dovresti evitare il più possibile.

Per gestire questo, un reset CSS rimuove gli stili predefiniti. Rimuovendo tutti o parte degli stili predefiniti, puoi ottenere una base coerente per il tuo design e ridurre al minimo le potenziali incoerenze tra browser e dispositivi. Rimuovere gli stili predefiniti può anche semplificare il processo di definizione dello stile perché vedrai solo gli stili che hai applicato esplicitamente nei tuoi fogli di stile personalizzati. Ci sono due approcci principali ai reset CSS: puoi definire reset CSS personalizzati oppure usare reset CSS di terze parti.

I reset CSS personalizzati sono fogli di stile che crei da zero per adattarli alle esigenze del tuo progetto. In questo modo puoi controllare gli stili specifici che verranno resettati con molta flessibilità. Tuttavia, devi anche investire tempo per sviluppare e mantenere i fogli di stile. Ecco un esempio di una regola CSS molto comune per resettare margine e spaziatura interna di tutti gli elementi HTML:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<h1>Example Heading</h1>
<p>This is a paragraph.</p>
```

```css
* {
  margin: 0;
  padding: 0;
}
```

:::

Solitamente viene scritta all’inizio del foglio di stile CSS. Il selettore asterisco è un selettore jolly che corrisponde a tutti gli elementi HTML, quindi avranno un margine predefinito di zero e una spaziatura interna predefinita di zero su tutti e quattro i lati. Questo ti dà un punto di partenza e poi puoi personalizzarli in selettori CSS più specifici più avanti nel foglio di stile.

Puoi usare questo approccio per selezionare qualsiasi elemento HTML e resettarne gli stili predefiniti. Basta creare un foglio di stile personalizzato e usare i selettori CSS appropriati per corrispondere agli elementi e definire gli stili.

Ma questo può richiedere molto tempo. Se vuoi risparmiare tempo, puoi anche usare un reset CSS di terze parti. Questi fogli di stile sono già predefiniti quindi puoi semplicemente scaricarli e aggiungerli direttamente al tuo progetto. Un ottimo esempio di reset CSS di terze parti è `Normalize.css`. Questo foglio di stile normalizza gli stili per un’ampia gamma di elementi HTML mantenendo comunque alcuni stili predefiniti utili, specialmente quelli importanti per l’accessibilità. Corregge anche bug comuni e incoerenze di stile.

Un’altra opzione è `sanitize.css`. Questa è una libreria CSS che puoi usare per assicurarti che gli stili predefiniti siano coerenti su tutti i principali browser moderni. Questa libreria è sviluppata insieme a `Normalize.css`, quindi evolvono insieme. Ha anche fogli di stile individuali che puoi scaricare per scopi specifici come normalizzare moduli e tipografia.

Ci sono molte opzioni disponibili ma dovresti scegliere quelle che meglio si adattano alle esigenze del tuo progetto. Puoi anche combinare entrambi gli approcci usando reset CSS di terze parti insieme a reset personalizzati.

Quando lavori con i reset CSS, è importante considerare anche l’accessibilità. La tua applicazione web dovrebbe essere accessibile a tutti. Quindi, non dovresti resettare stili che potrebbero essere utili per i lettori di schermo o altre tecnologie assistive. Dovresti anche considerare l’impatto che questi fogli di stile aggiuntivi possono avere sulle prestazioni della tua applicazione, perché devono essere scaricati prima che gli stili personalizzati vengano applicati.

Rimuovendo gli stili predefiniti, i reset CSS ti danno un punto di partenza vuoto per implementare il tuo design. Questo si traduce in un’esperienza utente più uniforme e coerente tra browser e dispositivi.
