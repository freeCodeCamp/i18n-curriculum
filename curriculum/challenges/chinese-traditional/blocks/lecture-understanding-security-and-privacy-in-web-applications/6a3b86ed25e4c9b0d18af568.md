# --description--

`.env` 檔案和 `dotenv` 函式庫是現代軟體開發中用來將敏感設定資料與應用程式程式碼分離的工具。

`.env` 檔案是一個純文字檔，放置在專案目錄的根目錄，裡面以鍵值對形式儲存設定和機密資訊。

開發者不會將 API 金鑰、資料庫密碼或埠號直接寫死在來源碼中，避免推送到像 GitHub 這樣的公開倉庫時暴露這些資訊，而是將它們本地儲存在這個檔案中。

`.env` 檔案的結構非常簡單：

```bash
PORT=3000
DB_HOST=localhost
API_KEY=xyz123secretkey
```

因為這個檔案包含高度敏感的金鑰，標準作法是將 `.env` 加入專案的 `.gitignore` 檔案，確保它永遠不會被 Git 追蹤。

`dotenv` 套件是一個輕量級模組，負責將 `.env` 檔案與你的應用程式連結起來。

當你的程式啟動時，`dotenv` 會讀取 `.env` 檔案，解析裡面的鍵值，並注入到執行期環境中。以 Node.js 應用程式為例，你會在入口檔案的最頂端初始化這個套件。載入後，這些變數就能透過 `process.env` 物件在全域範圍內存取。

```javascript
const express = require('express');
const app = express();

// initialize and configure dotenv
require('dotenv').config();

// access the variables securely from the environment
const port = process.env.PORT || 5000;
const apiKey = process.env.API_KEY;

app.get('/', (req, res) => {
  res.send(`Connected successfully using API Key: ${apiKey}`);
});

app.listen(port, () => {
  console.log(`Server running on port ${port}`);
});
```

使用這種設定有兩大利益。首先，它透過將私密憑證從來源碼中分離，提高安全性。其次，它提供彈性，讓同一份程式碼庫能在開發、測試和生產環境中表現不同，只需在各自的主機伺服器上替換本地的 `.env` 檔案即可。
