# --interactive--

継承はCSSの重要な概念で、親要素から子要素へスタイルがどのように受け継がれるかを決定します。

現実世界で子どもが親から特徴を受け継ぐのと同じように、CSSでも特定のプロパティが親要素から子要素に継承されます。

これにより、ドキュメント全体で一貫したスタイリングを効率的に適用できます。

CSSでは、すべてのプロパティがデフォルトでは継承されるわけではありません。例えば、`color`、`font-family`、`line-height`のようなプロパティは継承されます。つまり、親要素にテキストの色を設定すると、特にオーバーライドしない限り、その子要素すべてがその色を継承します。

例えば、親の`div`要素にインラインスタイルで色を設定し、子の`p`要素がその色を継承する次の例を考えてみましょう。

:::interactive_editor

```html
<div style="color: blue;">
  This is the parent element.
  <p>This is the child element inheriting the color.</p>
</div>
```

:::

この場合、親の`div`と子の`p`の両方が青色でテキストを表示します。これは色が継承されているためです。

一方で、`margin`、`padding`、`border`、`background`のようなプロパティはデフォルトでは継承されません。子要素にこれらのスタイルを継承させたい場合は、子要素に直接設定するか、`inherit`キーワードを使う必要があります。

`inherit`キーワードは、通常は継承されないプロパティでも親要素から強制的に継承させるために使えます。

例えば、特定の子要素に親と同じ`padding`を持たせたい場合、子要素に`padding: inherit`を設定できます。

:::interactive_editor

```html
<div style="padding: 20px;">
  This is the parent element with padding.
  <p style="padding: inherit;">This is the child element inheriting the padding.</p>
</div>
```

:::

この場合、子の`p`要素は親の`20px`要素からパディングの`div`を継承します。

継承は、スタイルの一貫性を保ち、冗長性を減らすのに特に役立ちます。

複数の要素に同じスタイルルールを書く代わりに、親要素に一度だけ定義すれば、子要素がそれを継承します。これによりCSSがより簡潔で管理しやすくなります。

ただし、継承は親から子への一方向であることを覚えておいてください。子要素でスタイルをオーバーライドしても、親要素には影響しません。
