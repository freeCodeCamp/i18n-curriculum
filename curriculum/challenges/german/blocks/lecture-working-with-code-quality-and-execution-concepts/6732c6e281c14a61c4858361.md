# --interactive--

Closures sind eine der mächtigsten und oft missverstandenen Funktionen in JavaScript. Im Kern ist ein Closure eine Funktion, die Zugriff auf Variablen in ihrem äußeren umschließenden lexikalischen Gültigkeitsbereich hat, selbst nachdem die äußere Funktion bereits zurückgegeben wurde. Das klingt vielleicht komplex, ist aber ein grundlegendes Konzept, das viele fortgeschrittene Programmiermuster in JavaScript ermöglicht.

Um Closures zu verstehen, beginnen wir mit einem Beispiel:

:::interactive_editor

```js
function outerFunction(x) {
    let y = 10;
    function innerFunction(){
        console.log(x + y);
    }
    return innerFunction;
}

let closure = outerFunction(5);
console.log(closure()); // 15
```

:::

In diesem Beispiel nimmt `outerFunction` einen Parameter `x` entgegen und definiert eine lokale Variable `y`. Dann definiert es eine `innerFunction`, die sowohl `x` als auch `y` verwendet. Schließlich gibt es `innerFunction` zurück. Wenn wir `outerFunction(5)` aufrufen, gibt es `innerFunction` zurück, das wir der Variablen `closure` zuweisen. Wenn wir später `closure()` aufrufen, hat es immer noch Zugriff auf `x` und `y` von `outerFunction`, obwohl `outerFunction` bereits ausgeführt wurde. Das ist das Wesentliche eines Closures.

Die innere Funktion behält eine Referenz auf ihre äußere lexikalische Umgebung bei und bewahrt so den Zugriff auf die Variablen in dieser Umgebung, selbst nachdem die äußere Funktion beendet ist.

Closures sind besonders nützlich, um private Variablen und Funktionen zu erstellen. Betrachten Sie dieses Beispiel:

:::interactive_editor

```js
function createCounter() {
    let count = 0;
    return function () {
        count++;
        return count;
    };
}

let counter = createCounter();
console.log(counter()); // 1
console.log(counter()); // 2
```

:::

In diesem Fall gibt `createCounter` eine Funktion zurück, die eine `count`-Variable inkrementiert und zurückgibt. Die Variable `count` ist von außerhalb von `createCounter` nicht direkt zugänglich, aber die zurückgegebene Funktion (unser Closure) hat Zugriff darauf. Jedes Mal, wenn wir `counter()` aufrufen, wird `count` inkrementiert und zurückgegeben.

Closures können auch mehrere Variablen aus ihrem äußeren Gültigkeitsbereich erfassen. Zum Beispiel:

:::interactive_editor

```js
function multiply(x) {
    return function (y) {
        return x * y;
    };
}

let double = multiply(2);
console.log(double(5)); // 10
```

:::

Hier erfasst die innere Funktion den Parameter `x` von `multiply`. Wenn wir `double` durch den Aufruf von `multiply(2)` erstellen, gibt es eine Funktion zurück, die ihr Argument immer mit `2` multipliziert.

Ein wichtiger Hinweis zu Closures ist, dass sie Variablen per Referenz und nicht per Wert erfassen. Das bedeutet, wenn sich der Wert einer erfassten Variable ändert, sieht das Closure den neuen Wert. Zum Beispiel:

:::interactive_editor

```js
function createIncrementer() {
    let count = 0;
    return function () {
        count++;
        console.log(count);
    };
}

let increment = createIncrementer();
increment(); // 1
increment(); // 2
```

:::

Jedes Mal, wenn wir `increment` aufrufen, arbeitet es mit derselben `count`-Variable, nicht mit einer Kopie ihres Anfangswerts. Closures sind ein mächtiges Werkzeug in JavaScript. Wenn Sie weiterhin mit JavaScript arbeiten, werden Sie feststellen, dass das Verständnis und die effektive Nutzung von Closures Ihre Fähigkeit, sauberen, effizienten und leistungsstarken Code zu schreiben, erheblich verbessern kann.
