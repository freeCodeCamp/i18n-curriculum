# --description--

이전 수업에서 배운 것처럼, 쿠키는 웹사이트가 사용자의 기기에 저장할 수 있는 간단한 데이터입니다.

하지만 실제로 쿠키에 더 복잡한 자료 구조를 저장할 수 있습니다. 흔한 방법 중 하나는 JSON을 사용해 객체나 배열을 저장하는 것입니다. 예를 들어 다음과 같습니다:

```js
const userData = {
  name: "John Doe",
  age: 30,
  role: "admin"
};

document.cookie = "userInfo=" + JSON.stringify(userData) + "; path=/";
```

이 예제에서는 사용자 데이터가 담긴 객체를 만들고, 이를 JSON 문자열로 변환한 다음 쿠키에 저장합니다. 이 데이터를 다시 가져올 때는 `JSON.parse()`를 사용해 JSON 문자열을 객체로 파싱할 수 있습니다.

이제 주제 중 "HTTP 헤더" 부분이 궁금할 수 있습니다. 보통 쿠키는 서버가 HTTP 헤더를 통해 설정합니다.

예를 들어, 서버가 다음과 같은 헤더를 보낼 수 있습니다:

```http
Set-Cookie: username=John Doe; expires=Thu, 31 Dec 2024 6:00:00 IST; path=/
```

이 헤더는 브라우저에 쿠키를 설정하라고 지시합니다.

또한 JavaScript를 사용해 브라우저에서 직접 쿠키를 설정할 수도 있습니다. 이는 서버로 즉시 보내지 않아도 되는 데이터를 저장할 때 유용합니다.

쿠키는 약 4KB 크기 제한이 있으니 너무 많은 데이터를 저장하면 웹 애플리케이션이 느려질 수 있습니다.

쿠키에 많은 데이터를 저장하면 모든 HTTP 요청에 쿠키가 포함되어 네트워크 트래픽이 증가할 수 있습니다.
