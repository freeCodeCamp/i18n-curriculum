## Порівняння та типи даних `null` і `undefined`

- **Порівняння та `undefined`**: Змінна є `undefined`, коли її оголосили, але їй не призначили значення. Це значення за замовчуванням для неініціалізованих змінних і параметрів функції, яким не передали аргумент. `undefined` перетворюється на `NaN` у числових контекстах, що призводить до того, що всі числові порівняння з `undefined` повертають `false`.

:::interactive_editor

```js
console.log(undefined < 0); // false (NaN < 0 is false)
console.log(undefined >= 0); // false (NaN >= 0 is false)
```

:::

- **Порівняння та `null`**: Тип `null` представляє навмисну відсутність значення. `null` перетворюється на `0` у числових контекстах, що може призвести до несподіваної поведінки в числових порівняннях:

:::interactive_editor

```js
console.log(null < 0); // false (0 < 0 is false)
console.log(null >= 0); // true (0 >= 0 is true)
```

:::

- При використанні оператора рівності (`==`) `null` і `undefined` рівні лише один одному та собі:

:::interactive_editor

```js
console.log(null == undefined); // true
console.log(null == 0); // false
console.log(undefined == NaN); // false
```

:::

- Однак при використанні оператора строгої рівності (`===`), який перевіряє і значення, і тип без перетворення типів, `null` і `undefined` не рівні:

:::interactive_editor

```js
console.log(null === undefined); // false
```

:::

## Інструкції `switch`

- **Визначення**: Інструкція `switch` оцінює вираз і порівнює його значення з низкою умов `case`. Коли знаходиться відповідність, виконується блок коду, пов’язаний із цим випадком. Інструкція `break` має бути розміщена в кінці кожного випадку, щоб завершити його виконання і перейти до наступного. `default` — це необов’язковий випадок, який виконується лише якщо жоден із інших випадків не співпадає. `default` розміщується в кінці інструкції `switch`.

:::interactive_editor

```js
const dayOfWeek = 3; 

switch (dayOfWeek) {
  case 1:
    console.log("It's Monday! Time to start the week strong.");
    break;
  case 2:
    console.log("It's Tuesday! Keep the momentum going.");
    break;
  case 3:
    console.log("It's Wednesday! We're halfway there.");
    break;
  case 4:
    console.log("It's Thursday! Almost the weekend.");
    break;
  case 5:
    console.log("It's Friday! The weekend is near.");
    break;
  case 6:
    console.log("It's Saturday! Enjoy your weekend.");
    break;
  case 7:
    console.log("It's Sunday! Rest and recharge.");
    break;
  default:
    console.log("Invalid day! Please enter a number between 1 and 7.");
}
```

:::
