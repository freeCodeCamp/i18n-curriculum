# --interactive--

Оператор `typeof` у JavaScript — це простий, але потужний інструмент, який дозволяє побачити тип даних змінної або значення. Він завжди повертає рядок, що вказує тип.

Давайте розглянемо кілька прикладів:

:::interactive_editor

```js
let num = 42;
console.log(typeof num); // "number"
```

:::

У цьому першому прикладі ми створили змінну з назвою `num` і призначили їй число `42`. Коли ви використовуєте оператор `typeof` для змінної з ім’ям `num`, він поверне рядок `number`.

Ось ще один приклад використання оператора `typeof` для змінної з назвою `isUserLoggedIn`:

:::interactive_editor

```js
let isUserLoggedIn = true;
console.log(typeof isUserLoggedIn); // "boolean"
```

:::

Коли ви використовуєте оператор `typeof` для змінної `isUserLoggedIn`, він поверне рядок `boolean`, тому що булеве значення `true` було призначене цій змінній.

Використання оператора `typeof` може бути особливо корисним, коли ви налагоджуєте код або намагаєтеся зрозуміти, з яким типом даних ви працюєте у своєму коді.

Однак у JavaScript є відома особливість, пов’язана з `null`.

Давайте розглянемо приклад:

:::interactive_editor

```js
let exampleVariable = null;
console.log(typeof exampleVariable); // "object"
```

:::

У цьому прикладі ми маємо змінну з назвою `exampleVariable` і призначили їй значення `null`. Але коли ми використовуємо оператор `typeof`, він повертає рядок `object`.

Це широко вважається помилкою у JavaScript, що сягає своїм корінням ранніх днів мови. Причина такої поведінки полягає у тому, як JavaScript був спочатку спроектований.

Коли мова тільки реалізовувалась, значення на кшталт `null` представлялися як особливий тип об’єкта, що й призводить до цього несподіваного результату.

На жаль, це стало частиною мови, і хоча це може збивати з пантелику, вам потрібно це знати.
