# --description--

Eine der größten Funktionen bei der Verwendung einer Template-Engine ist, dass Sie Variablen vom Server an die Template-Datei übergeben können, bevor diese in HTML gerendert wird.

In Ihrer Pug-Datei können Sie eine Variable verwenden, indem Sie den Variablennamen inline mit anderem Text auf einem Element referenzieren, wie `#{variable_name}`, oder indem Sie ein Gleichheitszeichen ohne Leerzeichen auf dem Element verwenden, wie `p=variable_name`, wodurch der Wert der Variable dem Text des p-Elements zugewiesen wird.

Pug basiert darauf, Leerzeichen und Tabs zu verwenden, um verschachtelte Elemente darzustellen und die Menge an Code zu reduzieren, die nötig ist, um eine schöne Website zu erstellen.

Betrachten Sie zum Beispiel den folgenden Pug-Code: 
 
```pug
head
  script(type='text/javascript').
    if (foo) bar(1 + 5);
body
  if youAreUsingPug
      p You are amazing
    else
      p Get on it!
```

Der obige Code erzeugt das folgende HTML:

```html
<head>
  <script type="text/javascript">
    if (foo) bar(1 + 5);
  </script>
</head>
<body>
  <p>You are amazing</p>
</body>
```

Ihre `index.pug`-Datei, die in Ihrem Projekt enthalten ist, verwendet die Variablen `title` und `message`.

Übergeben Sie diese vom Server an die Pug-Datei, indem Sie ein Objekt als zweiten Parameter zu Ihrem `res.render`-Aufruf hinzufügen, das die Variablen und ihre Werte enthält. Geben Sie `title` den Wert `Hello` und `message` den Wert `Please log in`.

Es sollte so aussehen:

```javascript
res.render('index', { title: 'Hello', message: 'Please log in' });
```

Aktualisieren Sie nun Ihre Seite, und Sie sollten diese Werte an der richtigen Stelle in Ihrer Ansicht sehen, wie es in Ihrer `index.pug`-Datei festgelegt ist!

Senden Sie Ihre Seite ab, wenn Sie denken, dass alles richtig ist. Wenn Sie auf Fehler stoßen, können Sie sich das <a href="https://forum.freecodecamp.org/t/advanced-node-and-express/567135#use-a-template-engines-power-2" target="_blank" rel="noopener noreferrer nofollow">Projekt bis zu diesem Punkt abgeschlossen</a> ansehen.
