# --interactive--

Il costruttore `Number()` viene usato per creare un oggetto number. L’oggetto number contiene alcune proprietà e metodi utili come `isNaN` e il metodo `toFixed`. Ecco un esempio che usa il costruttore `Number()` con la parola chiave `new`:

:::interactive_editor

```js
const myNum = new Number("34");
console.log(typeof myNum); // "object" 
```

:::

In questo esempio passiamo una stringa letterale al costruttore `Number()` e il tipo di ritorno è di tipo `object` invece che `string`. 

Quando il costruttore `Number()` viene chiamato come funzione senza la parola chiave `new`, allora il valore di ritorno sarà il tipo primitivo number. La maggior parte delle volte userai il costruttore `Number()` per convertire altri tipi di dati nel tipo number. Ecco un esempio di conversione di una stringa in un numero:

:::interactive_editor

```js
const myNum = Number("100");
console.log(myNum); // 100

console.log(typeof myNum); // number
```

:::

Questo è utile quando ricevi input dall’utente e devi convertire quella stringa in un numero per poter fare calcoli matematici. 

Se provi a chiamare il costruttore `Number()` con una stringa vuota, il risultato sarà il numero `0`: 

:::interactive_editor

```js
const num = Number("");
console.log(num); // 0
```

:::

Questo succede perché JavaScript cerca di analizzare la stringa e dato che non contiene cifre, il risultato sarà zero.

Se provi a passare una stringa con caratteri casuali, il risultato sarà `NaN` o “Not a Number”. 

:::interactive_editor

```js
const num = Number("random");
console.log(num); // NaN
```

:::

Quando lavori con booleani, `true` restituisce `1` perché `true` viene convertito in uno e `false` restituisce `0` perché `false` viene convertito in zero. 

:::interactive_editor

```js
const boolTrue = Number(true);
const boolFalse = Number(false);

console.log(boolTrue); // 1
console.log(boolFalse); // 0
```

:::

Se passi `null`, il risultato sarà `0` e se passi `undefined`, il risultato sarà `NaN`. 

:::interactive_editor

```js
const undefinedNum = Number(undefined);
const nullNum = Number(null);

console.log(undefinedNum); // NaN
console.log(nullNum); // 0
```

:::

Quando lavori con array ci sono alcune cose da considerare.

Un array vuoto restituisce `0`. Un array con un solo numero restituisce quel numero. Un array con numeri multipli restituisce `NaN`. E un array con stringhe restituisce `NaN`.

:::interactive_editor

```js
const emptyArr = Number([]);
const arrOneNum = Number([7]);
const arrMultiNum = Number([7, 36, 12]);
const arrStr = Number(["str1"]);
const arrMultiStr = Number(["str1", "str2"]);

console.log(emptyArr); // 0
console.log(arrOneNum); // 7
console.log(arrMultiNum); // NaN
console.log(arrStr); // NaN
console.log(arrMultiStr); // NaN
```

:::

Quando lavori con oggetti, il risultato è sempre `NaN`.

:::interactive_editor

```js
const obj1 = Number({});
const obj2 = Number({2: 2});
const obj3 = Number({key: "val"});
const obj4 = Number({key: true});

console.log(obj1); // NaN
console.log(obj2); // NaN
console.log(obj3); // NaN
console.log(obj4); // NaN
```

:::

In conclusione, userai soprattutto il costruttore `Number()` per la conversione di tipo più che per creare un numero o un oggetto number.
