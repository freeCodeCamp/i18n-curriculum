# --description--

Es gibt zwei Arten von Testansätzen, die Sie in der Softwareentwicklung kennen sollten. Diese Ansätze sind TDD (Test-Driven Development) und BDD (Behavior-Driven Development).

Schauen wir uns zuerst TDD an. Test-Driven Development ist eine Methodik, die betont, zuerst Tests zu schreiben, bevor die Funktionen für eine Anwendung entwickelt werden.

Hier ist der typische Ablauf für TDD:

1. Schreiben Sie einen Test

2. Stellen Sie sicher, dass er fehlschlägt

3. Schreiben Sie den Code für diesen Test

4. Stellen Sie sicher, dass der Test diesmal besteht

5. Nehmen Sie alle notwendigen Refaktorierungen am Code vor

Wiederholen Sie die Schritte 1–5.

Tests vor der Entwicklung von Funktionen zu schreiben, liefert Entwicklern während des Entwicklungsprozesses Echtzeit-Feedback. Es hilft auch, Bugs früher in der Entwicklungsphase zu erkennen.

Während einige Softwareteams sich für den TDD-Ansatz entscheiden, ist TDD ein polarisierendes Thema in der Branche. Sie werden viele Artikel und Videos finden, die TDD entweder befürworten oder kritisieren. Unabhängig davon, wie Sie zu diesem Ansatz stehen, ist es wichtig, ihn zu kennen, da Sie ihm in realen Szenarien begegnen könnten.

Der zweite Testansatz, den wir uns ansehen, ist Behavior-Driven Development (BDD). Behavior-Driven Development ist der Ansatz, eine Reihe von Tests mit Geschäftszielen in Einklang zu bringen. Diese Tests sollten in einer Sprache geschrieben sein, die sowohl von technischen als auch von nicht-technischen Personen verstanden wird.

Beim Testen spezifischer Verhaltensweisen in einer Softwareanwendung verwendet BDD häufig die Gherkin-Sprache, die in Unternehmen verwendet wird, um Verhaltensweisen und Szenarien zu beschreiben. Die Gherkin-Sprache verwendet die Aussagen Given, When und Then, um Szenarien zu beschreiben.

Hier ist ein Beispiel:

**Szenario:** Erstellen eines neuen Blogbeitrags

**Given** der Benutzer ist in seinem Konto angemeldet und befindet sich auf der Seite „Neuen Beitrag erstellen“,

**When** der Benutzer einen Titel und Inhalt für den Beitrag eingibt und auf die Schaltfläche „Veröffentlichen“ klickt,

**Then** sollte der Blogbeitrag gespeichert und auf der Startseite mit dem korrekten Titel, Inhalt und Zeitstempel angezeigt werden.

Zuerst definieren Sie ein Szenario. Beispielhafte Szenarien sind, wenn sich ein Benutzer in einer Anwendung anmeldet oder eine Bestellung auf einer Website aufgibt. In diesem Beispiel geht es um das Szenario, dass ein Benutzer einen neuen Blogbeitrag auf einer Website erstellt.

Nachdem das Szenario definiert ist, beginnen Sie mit einer Given-Aussage. Diese gibt den Kontext an, wo der Benutzer startet, bevor er eine Aktion ausführt. Die When-Aussage wird verwendet, um die vom Benutzer ausgeführte Aktion darzustellen.

Schließlich wird die Then-Aussage verwendet, um das gewünschte Ergebnis der vorherigen Aktion des Benutzers darzustellen. Sobald Sie alle Ihre Szenarien definiert haben, können Sie für jedes Szenario Tests schreiben.

Beispiele für BDD-Testframeworks sind Cucumber, JBehave und SpecFlow.
