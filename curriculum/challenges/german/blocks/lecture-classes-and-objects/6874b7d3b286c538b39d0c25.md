# --description--

Spezielle Methoden in Python, auch bekannt als „magische Methoden“ oder „dunder-Methoden“, sind besondere Python-Methoden, die mit doppelten Unterstrichen beginnen und enden (`__`). Das Wort „dunder“ selbst stammt von double underscores (**d** für double, **under** für underscores).

Sie haben wahrscheinlich schon spezielle Methoden verwendet, ohne es zu wissen. Jedes Mal, wenn Sie etwas wie `3 + 4` schreiben, führt Python im Hintergrund stillschweigend `3.__add__(4)` aus. Das ist eine spezielle Methode in Aktion. Obwohl Sie spezielle Methoden *direkt* aufrufen *können*, tun Sie das selten. Etwas wie `3 + 4` ist viel klarer und leichter zu lesen als selbst `3.__add__(4)` aufzurufen.

Abgesehen von `__add__` ist `__init__()` eine weitere spezielle Methode, die Sie am häufigsten sehen und verwenden, da sie ein Klasseninitialisierer ist. Es gibt auch andere wie `__len__()` und `__str__()`.

Betrachten Sie spezielle Methoden als die Regisseure der Abläufe zwischen einer programmierenden Person und dem Python-Sprachinterpreter selbst.

Denken Sie daran, Sie müssen spezielle Methoden nicht direkt aufrufen. Stattdessen ruft Python sie automatisch auf, wenn bestimmte Aktionen passieren. Diese Operationen umfassen:

- **Arithmetische Operationen wie Addition, Subtraktion, Multiplikation, Division und andere**. Für Addition wird `__add__()` aufgerufen, `__sub__()` für Subtraktion, `__mul__()` für Multiplikation und `__truediv__()` für Division.
    
- **String-Operationen wie Verkettung, Wiederholung, Formatierung und Umwandlung in Text**. `__add__()` wird für Verkettung aufgerufen, `__mul__()` für Wiederholung, `__format__()` für Formatierung, `__str__()` und `__repr__()` für Textumwandlung und so weiter.
    
- **Vergleichsoperationen wie Gleichheit, Kleiner-als, Größer-als und andere**. `__eq__()` wird für Gleichheitsprüfungen aufgerufen, `__lt__()` für Kleiner-als, `__gt__()` für Größer-als und so weiter.
    
- **Iterative Operationen wie das Iterierbar-Machen eines Objekts und das Voranschreiten durch Elemente**. `__iter__()` wird aufgerufen, um einen Iterator zurückzugeben, und `__next__()`, um das nächste Element abzurufen.
    

Normalerweise wissen Python-Datentypen wie Strings und Zahlen bereits, wie man Dinge addiert, verkettet, auf Gleichheit prüft, in Schleifen verwendet und mehr.

Wenn Sie jedoch Ihre eigene Klasse erstellen, weiß Python nicht automatisch, wie es damit umgehen soll.

Hier kommen spezielle Methoden ins Spiel – sie ermöglichen es Ihnen, das eingebaute Verhalten von Python anzupassen.

Angenommen, Sie möchten die Anzahl der Seiten in Buchobjekten ermitteln, die mit der untenstehenden Klasse erstellt wurden, oder sie vergleichen und eine lesbare Zeichenkette der Objekte erhalten. So sieht es ohne spezielle Methoden aus:

```python
class Book:
   def __init__(self, title, pages):
       self.title = title
       self.pages = pages

book1 = Book("Built Wealth Like a Boss", 420)
book2 = Book("Be Your Own Start", 420)

print(len(book1)) # TypeError: object of type 'Book' has no len()
print(str(book1)) # <__main__.Book object at 0x102ed2900>
print(book1 == book2) # False even though they have the same number of pages
```

Im Beispiel:

- `len(book1)` schlug fehl, weil Python nicht weiß, wie es die Länge Ihres Buchobjekts ohne `__len__()` ermitteln soll
    
- `str(book1)` gab etwas wie `<__main__.Book object at 0x102ed2900>` aus, weil das die Standarddarstellung ist, wenn Sie `__str__()` nicht verwenden
    
- `book1 == book2` ergab `False`, weil Python einfach prüft, ob beide Objekte im Speicher identisch sind, nicht nach Inhalt.
    

So können Sie Ihre eigenen speziellen Methoden `__len__()`, `__str__()` und `__eq__()` definieren, um die Arbeit mit Objekten der Klasse `Book` zu erleichtern:

```python
class Book:
   def __init__(self, title, pages):
       self.title = title
       self.pages = pages

   def __len__(self):
       return self.pages

   def __str__(self):
       return f"'{self.title}' has {self.pages} pages"

   def __eq__(self, other):
       return self.pages == other.pages
  
book1 = Book("Built Wealth Like a Boss", 420)
book2 = Book("Be Your Own Start", 420)

print(len(book1)) # 420
print(len(book2)) # 420
print(str(book1)) # 'Built Wealth Like a Boss' has 420 pages
print(str(book2)) # 'Be Your Own Start' has 420 pages
print(book1 == book2) # True
```

Ein weiteres Beispiel ist ein Einkaufswagen, bei dem Sie Folgendes tun:

- Artikel zum Wagen hinzufügen
    
- Artikel aus dem Wagen entfernen
    
- Die Anzahl der Artikel im Wagen ermitteln
    
- Prüfen, welche Artikel im Wagen sind
    
- Prüfen, ob ein bestimmter Artikel im Wagen ist
    
- Einen Artikel an einem bestimmten Index im Wagen zurückgeben oder anzeigen
    

Während Sie vielleicht eine Methode haben, die Artikel zum Wagen hinzufügt und bestimmte Artikel entfernt, können Sie spezielle Methoden für alle anderen Funktionen erstellen:

- `__len__()`, um die Länge der Artikel im Wagen zu ermitteln
    
- `__iter__()`, um durch die Artikel im Wagen zu iterieren, damit Sie sie sehen können
    
- `__contains__()`, um zu prüfen, ob ein bestimmter Artikel im Wagen ist
    
- `__getitem__()`, um einen Artikel an einem bestimmten Index im Wagen zurückzugeben oder anzuzeigen
    

Hier ist ein Beispiel für eine `Cart`-Klasse mit diesen benutzerdefinierten Methoden und speziellen Methoden:

```python
class Cart:
   def __init__(self):
       self.items = []

   def add(self, item):
       self.items.append(item)

   def remove(self, item):
       if item in self.items:
           self.items.remove(item)
       else:
           print(f'{item} is not in cart')

   def list_items(self):
       return self.items

   def __len__(self):
       return len(self.items)

   def __getitem__(self, index):
       return self.items[index]

   def __contains__(self, item):
       return item in self.items

   def __iter__(self):
       return iter(self.items)
```

Und so können Sie sie verwenden:

```python
cart = Cart()
cart.add('Laptop')
cart.add('Wireless mouse')
cart.add('Ergo keyboard')
cart.add('Monitor')

for item in cart:
   print(item, end=' ') # Laptop Wireless mouse Ergo keyboard Monitor

print(len(cart)) # 4
print(cart[3]) # Monitor

print('Monitor' in cart) # True
print('banana' in cart) # False

cart.remove('Ergo keyboard')

print(cart.list_items()) # ['Laptop', 'Wireless mouse', 'Monitor']

cart.remove('banana') # banana is not in cart
```
