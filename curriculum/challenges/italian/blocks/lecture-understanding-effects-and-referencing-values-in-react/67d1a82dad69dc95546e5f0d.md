# --description--

In React, possono esserci situazioni in cui hai bisogno di un accesso diretto a un elemento DOM. È qui che i "ref" diventano utili. I ref possono anche memorizzare valori mutabili, ma per questo lo stato è una scelta migliore.

In JavaScript vanilla, usavi i metodi `getElementById()` e `querySelector()` per consultare gli elementi DOM. Ma in React, usi i ref per consultare gli elementi nel DOM.

Una delle principali differenze è che, con i ref, non hai bisogno di identificatori come ID e classi per fare riferimento agli elementi.

Allora, come puoi creare e usare i ref? React fornisce un hook `useRef()` che ti permette proprio di farlo.

Il primo passaggio è importare l’hook da React:

```js
import { useRef } from "react";
```

Poi, devi creare una variabile che contiene il ref con il valore iniziale del ref dentro l’hook `useRef`, per esempio un `sectionRef` inizializzato a `null`:

```js
const sectionRef = useRef(null);
```

L’ultima cosa da fare è allegare la variabile ref all’elemento nel tuo JSX usando l’attributo `ref`:

```jsx
<section ref={sectionRef}>
  {/* Section content */}
</section>
```

Se registri il ref nella console, vedrai che è un oggetto con il valore current, in questo caso `null`:

```js
console.log(sectionRef); // { current: null }
```

Puoi anche registrare il valore current nella console con la proprietà `current` così puoi vedere direttamente il valore:

```js
console.log(sectionRef.current); // null
```

I valori successivi del ref dipendono dal ciclo di vita del componente.

Per esempio, il valore iniziale di `sectionRef` sarà sempre `null` perché è quello con cui è stato inizializzato. Dopo che il componente è montato, il valore del ref sarà l’elemento `section` a cui il ref è allegato.

Se il componente viene smontato, il valore del ref torna al valore iniziale `null`.

Un esempio tipico per mostrare un ref è mettere il focus su un elemento input al render o cliccando un pulsante.

Ecco come farlo quando clicchi un pulsante:

```jsx
import { useRef } from "react";

const Focus = () => {
  const inputRef = useRef(null);

  const handleFocus = () => {
    if (inputRef.current) {
      inputRef.current.focus();
    }
  };

  return (
    <div>
      <input ref={inputRef} type="text" placeholder="Enter text" />
      <button onClick={handleFocus}>Focus Input</button>
    </div>
  );
};

export default Focus;
```

Nel codice sopra, il `inputRef` è creato e allegato all’elemento `input`. C’è anche un pulsante con un evento `onClick` che chiama una funzione `handleFocus`.

Tutto quello che fa la funzione `handleFocus` è chiamare il metodo `focus()` sull’elemento `input`. Nota che, poiché `input` è un componente integrato che viene con React, l’effettivo elemento DOM `input` è impostato sulla proprietà `current` del ref. Quindi chiami il metodo `focus()` con `input.current.focus()`.

Ecco alcune buone pratiche da tenere a mente quando lavori con i ref:

- Usa i ref principalmente per interagire con il DOM. Puoi anche usarli per dati mutabili, ma lo stato è una scelta migliore per questo.

- Non usare i ref per la gestione base dello stato – per quello c’è `useState`.

- Assicurati di controllare che `ref.current` esista prima di consultare le sue proprietà. Ecco come farlo di nuovo:

```js
const handleFocus = () => {
  if (inputRef.current) {
    inputRef.current.focus();
  }
};
```

Questo previene errori nel caso in cui il ref venga consultato prima che sia allegato al DOM o dopo che è stato rimosso.
