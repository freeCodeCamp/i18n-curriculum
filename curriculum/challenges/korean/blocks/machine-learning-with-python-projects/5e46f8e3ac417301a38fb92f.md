# --instructions--

이번 도전 과제에서는 **K-최근접 이웃(K-Nearest Neighbors)** 을 사용해 도서 추천 알고리즘을 만듭니다.

이 프로젝트에서는 90,000명의 사용자가 270,000권의 책에 대해 1~10점 척도로 평가한 110만 개의 평점이 포함된 Book-Crossings 데이터셋을 사용합니다. 데이터셋은 이미 노트북에 가져와져 있으므로 별도의 다운로드는 필요 없습니다.

`NearestNeighbors`에서 `sklearn.neighbors`를 사용해 주어진 책과 비슷한 책을 보여주는 모델을 개발하세요. 최근접 이웃 알고리즘은 인스턴스 간 거리를 측정해 “가까움”을 판단합니다.

책 제목(데이터셋에서 가져온)을 인수로 받아 해당 책과 비슷한 5권의 책과 그 거리 정보를 리스트로 반환하는 `get_recommends`라는 함수를 만드세요.

이 코드는:

```py
get_recommends("The Queen of the Damned (Vampire Chronicles (Paperback))")
```

다음과 같이 반환해야 합니다:

```py
[
  'The Queen of the Damned (Vampire Chronicles (Paperback))',
  [
    ['Catch 22', 0.793983519077301], 
    ['The Witching Hour (Lives of the Mayfair Witches)', 0.7448656558990479], 
    ['Interview with the Vampire', 0.7345068454742432],
    ['The Tale of the Body Thief (Vampire Chronicles (Paperback))', 0.5376338362693787],
    ['The Vampire Lestat (Vampire Chronicles, Book II)', 0.5178412199020386]
  ]
]
```

`get_recommends()`에서 반환하는 데이터가 리스트임을 주목하세요. 리스트의 첫 번째 요소는 함수에 전달된 책 제목이고, 두 번째 요소는 다섯 개의 리스트로 이루어진 리스트입니다. 각 다섯 개 리스트는 추천 도서와 추천 도서와 함수 인수로 전달된 도서 간 거리를 포함합니다.

데이터셋을 그래프로 그려보면(선택 사항) 대부분의 책이 자주 평가되지 않았음을 알 수 있습니다. 통계적 유의성을 확보하려면 200개 미만 평점을 가진 사용자와 100개 미만 평점을 가진 책은 데이터셋에서 제거하세요.

처음 세 셀은 필요한 라이브러리와 데이터를 가져옵니다. 마지막 셀은 테스트용입니다. 모든 코드는 이 셀들 사이에 작성하세요.
