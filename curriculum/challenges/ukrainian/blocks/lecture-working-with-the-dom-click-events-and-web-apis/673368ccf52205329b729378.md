# --interactive--

Давайте дізнаємося про `innerText`, `textContent` і `innerHTML`.

Це властивості, до яких ви можете отримати доступ у JavaScript, щоб отримати або змінити вміст HTML-елемента. Хоча вони можуть виглядати дуже схожими на перший погляд, між ними є ключові відмінності. Вибір правильного залежить від вашого конкретного випадку, тож давайте розберемося.

Почнемо з `innerText`.

`innerText` представляє видимий текстовий вміст HTML-елемента та його нащадків. Ця властивість не включає прихований текст або HTML-теги, лише відтворений текст.

Наприклад, тут ви бачите `div` елемент, який містить два абзаци:

```html
<div id="container">
  <p>Hello, World!</p>
  <p>I'm learning JavaScript</p>
</div>
```

Якщо ми отримаємо референс на цей HTML-елемент у нашому JavaScript-коді за допомогою `getElementById()`, ми можемо отримати доступ до властивості `innerText` цього елемента:

:::interactive_editor

```html
<div id="container">
  <p>Hello, World!</p>
  <p>I'm learning JavaScript</p>
</div>
<script src="index.js"></script>
```

```js
const container = document.getElementById("container");
console.log(container.innerText);
```

:::

Це внутрішній текст цього елемента:

```md
Hello, World!
I'm learning JavaScript
```

Властивість повертає рядок із текстом, що міститься всередині елемента, включно з текстом його нащадків.

Варто знати, що `innerText` повертає лише текст, який видимий у конкретний момент, коли запитується рядок. Якщо дочірній елемент прихований, його текст не буде видимим.

Ось приклад, де другий абзац прихований:

:::interactive_editor

```html
<div id="container">
  <p>Hello, World!</p>
  <p hidden>I'm learning JavaScript</p>
</div>
```

:::

Якщо ми спробуємо знову вивести `innerText`, тепер вихідні дані не міститимуть текст другого абзацу:

:::interactive_editor

```html
<div id="container">
  <p>Hello, World!</p>
  <p hidden>I'm learning JavaScript</p>
</div>
<script src="index.js"></script>
```

```js
const container = document.getElementById("container");
console.log(container.innerText);
```

:::

Ви можете встановити `innerText` HTML-елемента таким чином, але це замінить існуючий текст і додасть елемент переносу рядка (`br`) для кожного переносу рядка:

:::interactive_editor

```html
<div id="container">
  <p>Hello, World!</p>
  <p>I'm learning JavaScript</p>
</div>
<script src="index.js"></script>
```

```js
const container = document.getElementById("container");
container.innerText = "JavaScript is awesome!";
```

:::

Оскільки `innerText` враховує видимість, отримання його значення запускає процес, який називається "перерахунок потоку" (reflow), що повторно обчислює позиції певних елементів на сайті. Цей процес може бути обчислювально затратним, тому варто уникати його виклику, якщо це можливо.

Чудово. Тепер поговоримо про `textContent`.

`textContent` повертає простий текстовий вміст елемента, включно з усім текстом у його нащадках.

Найважливіша відмінність між `innerText` і `textContent` полягає в тому, що `textContent` завжди повертає повний текстовий вміст HTML-елемента та його нащадків, незалежно від того, чи він видимий, чи прихований.

Ось той самий приклад у HTML:

```html
<div id="container">
  <p>Hello, World!</p>
  <p>I'm learning JavaScript</p>
</div>
```

Якщо ви спробуєте отримати доступ до цієї властивості, ви побачите текст елемента та його нащадків у вихідних даних, зберігаючи відступи та пробіли:

:::interactive_editor

```html
<div id="container">
  <p>Hello, World!</p>
  <p>I'm learning JavaScript</p>
</div>
<script src="index.js"></script>
```

```js
const container = document.getElementById("container");
console.log(container.textContent);
```

:::

Якщо HTML-елемент не видимий, як тут, де ми приховали другий абзац, його текст все одно буде включений у цю властивість:

:::interactive_editor

```html
<div id="container">
  <p>Hello, World!</p>
  <p hidden>I'm learning JavaScript</p>
</div>
<script src="index.js"></script>
```

```js
const container = document.getElementById("container");
console.log(container.textContent);
```

:::
  
`textContent` також включатиме вміст таких елементів, як `script` і `style`.

Якщо ви спробуєте замінити значення `textContent` у вузлі, це видалить усі його дочірні вузли і замінить їх одним текстовим вузлом із новим рядком:

:::interactive_editor

```html
<div id="container">
  <p>Hello, World!</p>
  <p hidden>I'm learning JavaScript</p>
</div>
<script src="index.js"></script>
```

```js
const container = document.getElementById("container");
container.textContent = "New content";
```

:::

І нарешті, поговоримо про те, чим `textContent` і `innerText` відрізняються від `innerHTML`.

Пам’ятайте, що за допомогою `innerHTML` ви можете встановити внутрішній HTML-вміст елемента. Це корисно для динамічного додавання нового HTML у DOM.

Однак пам’ятайте, що це створює ризик безпеки, якщо ви не контролюєте рядок, наприклад, якщо рядок містить дані, введені користувачем. Якщо ці дані шкідливі, це може призвести до серйозних проблем із безпекою.

Щоб уникнути цього, рекомендується використовувати властивість `textContent` для вставки простого тексту.

Властивості `innerText`, `textContent` і `innerHTML` у JavaScript надають різні способи доступу та маніпулювання вмістом HTML-елементів. Ви повинні розуміти відмінності між цими властивостями, якщо ваша мета — ефективно працювати з HTML-вмістом у JavaScript.
