# --description--

In der letzten Lektion haben Sie gelernt, dass der Server eine HTTP-Anfrage verarbeitet und eine Antwort zurücksendet. Diese Antwort enthält verschiedene Assets, die Sie im Web sehen.

Die von einer HTTP-Antwort zurückgegebenen Assets umfassen textbasierte Assets, Medien-Assets, Schriftarten und so weiter. Jedes dieser Assets wird mit einer **Content-Type**-Kopfzeile geliefert, die dem Browser mitteilt, wie das Asset korrekt verarbeitet und angezeigt wird.

Textbasierte Assets sind schon immer ein grundlegender Bestandteil des Webs gewesen. Tatsächlich begann das Web als reiner Text. Heutzutage umfassen textbasierte Assets Folgendes:

* **HTML (Hypertext Markup Language)**: Die Grundstruktur jeder Webseite. Ohne HTML kann eine Webseite nicht existieren.
    
* **CSS (Cascading Stylesheet)**: Das Asset, das steuert, wie eine Webseite aussieht. Mit CSS wenden Sie Stile wie Farben, Schriftarten und Layout auf das HTML an.
    
* **JavaScript**: JavaScript fügt HTML und CSS Interaktivität und dynamische Inhalte hinzu.
    
* **JSON und XML**: Diese beiden Formate ermöglichen den Transfer strukturierter Daten zwischen Servern und Browsern.
    
Medien-Assets umfassen Bilder, Audios und Videos. Sie erwecken das Web mit visuellen und akustischen Elementen zum Leben.

Viele Websites verlassen sich auf Bilder, um visuelle Informationen zu vermitteln. Bilder gibt es in verschiedenen Formaten wie `JPG`, `JPEG`, `PNG`, `WEBP` und `GIF`.

Websites wie YouTube und Vimeo liefern Videoinhalte in Formaten wie `MP4`, `MOV`, `AVI`, `WebM` und mehr.

Audio ist ein wichtiger Bestandteil vieler Plattformen, da es Nutzern ermöglicht, Musik, Podcasts und andere Klanginhalte zu streamen. Gängige Audioformate sind `MP3`, `AAC`, Ogg und `WAV`. Plattformen wie Spotify und YouTube Music nutzen diese Formate, um hochwertige Audioerlebnisse zu bieten.

Weitere Assets, die in einer HTTP-Antwort zurückgegeben werden, sind Dokumente wie `PDF`, `DOCX` und `PAGES`, komprimierte Archive wie `ZIP` und `RAR`, Schriftdateien wie `WOFF`, `WOFF2` und `TTF` sowie ausführbare Dateien wie `EXE`.

Hier ist eine Tabelle dieser gängigen Assets und ihrer Content-Type-Kopfzeilen:

| Asset-Typ | Content-Type-Kopfzeile |
| --- | --- |
| HTML | text/html |
| CSS | text/css |
| JavaScript | application/javascript |
| JPEG | image/jpeg |
| PNG | image/png |
| GIF | image/gif |
| SVG | image/svg+xml |
| WebP | image/webp |
| WOFF | font/woff |
| WOFF2 | font/woff2 |
| TTF | font/ttf |
| OTF | font/otf |
| MP4 | video/mp4 |
| WebM | video/webm |
| Ogg (Video) | video/ogg |
| MP3 | audio/mpeg |
| AAC | audio/aac |
| WAV | audio/wav |
| Ogg (Audio) | audio/ogg |
| JSON | application/json |
| XML | application/xml |
| PDF | application/pdf |
| ZIP | application/zip |
