# --description--

Se non l'hai ancora capito, il problema nella sfida precedente era nella chiamata `splice` nella funzione `tabClose()`. Purtroppo, `splice` modifica l'array originale su cui viene chiamata, quindi la seconda chiamata ha usato un array modificato e ha prodotto risultati inattesi.

Questo è un piccolo esempio di un pattern molto più ampio: chiami una funzione su una variabile, un array o un oggetto, e la funzione modifica la variabile o qualcosa nell'oggetto.

Uno dei principi fondamentali della programmazione funzionale è non modificare le cose. Le modifiche portano a bug. È più facile prevenire bug sapendo che le tue funzioni non cambiano nulla, inclusi gli argomenti della funzione o qualsiasi variabile globale.

L'esempio precedente non aveva operazioni complicate, ma il metodo `splice` ha modificato l'array originale, causando un bug.

Ricorda che nella programmazione funzionale, modificare o alterare le cose si chiama <dfn>mutazione</dfn>, e il risultato si chiama <dfn>effetto collaterale</dfn>. Una funzione, idealmente, dovrebbe essere una <dfn>funzione pura</dfn>, cioè che non provoca effetti collaterali.

Cerchiamo di padroneggiare questa disciplina e di non alterare nessuna variabile o oggetto nel nostro codice.

# --instructions--

Completa il codice della funzione `incrementer` in modo che restituisca il valore della variabile globale `fixedValue` aumentato di uno.

# --hints--

La tua funzione `incrementer` non dovrebbe modificare il valore di `fixedValue` (che è `4`).

```js
incrementer();
assert(fixedValue === 4);
```

La tua funzione `incrementer` dovrebbe restituire un valore che è uno in più del valore `fixedValue`.

```js
const __newValue = incrementer();
assert(__newValue === 5);
```

La tua funzione `incrementer` dovrebbe restituire un valore basato sul valore della variabile globale `fixedValue`.

```js
(function () {
  fixedValue = 10;
  const newValue = incrementer();
  assert(fixedValue === 10 && newValue === 11);
  fixedValue = 4;
})();
```
