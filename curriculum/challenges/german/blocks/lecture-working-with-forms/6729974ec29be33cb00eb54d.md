# --interactive--

Das `form`-Element in HTML wird verwendet, um Benutzerinformationen zu sammeln, wie Namen und E-Mail-Adressen. Hier ist ein Beispiel für ein `form`-Element:

```html
<form action="url-goes-here">
  <!-- input elements go here -->
</form>
```

Das `action`-Attribut gibt an, wohin die Formulardaten bei der Einsendung gesendet werden. Um bestimmte Informationen zu sammeln, wie Namen und E-Mail-Adressen, verwenden Sie das `input`-Element. Hier ist ein Beispiel für die Verwendung eines `input`-Elements.

Aktivieren Sie den interaktiven Editor und interagieren Sie mit dem `input`-Element im Vorschaufenster, indem Sie Ihren Namen in das Feld eingeben.

:::interactive_editor

```html
<form action="">
  <input type="text" />
</form>
```

:::

`input`-Elemente sind sogenannte Void-Elemente und haben kein schließendes Tag. Das `type`-Attribut definiert den vom Benutzer erwarteten Datentyp. In diesem Fall wären die Daten Klartext. Um eine Bezeichnung für die Eingabe hinzuzufügen, verwenden Sie ein `label`-Element. Hier ist ein Beispiel für die Verwendung eines `label`-Elements mit dem Text `Full Name:`.

Klicken Sie im Vorschaufenster auf den Text `Full Name:`, und Sie werden sehen, dass die Eingabe den Fokus erhält. Um mit dem Beispiel zu interagieren, müssen Sie den interaktiven Editor aktivieren.

:::interactive_editor

```html
<form action="">
  <label>
    Full Name:
    <input type="text" />
  </label>
</form>
```

:::

Indem Sie ein `input` innerhalb eines `label`-Elements verschachteln, erstellen Sie eine implizite Verbindung zwischen dem `label` und dem `input`-Feld. Der Begriff „implizit“ bedeutet, dass etwas verstanden oder abgeleitet wird, ohne dass es explizit mit zusätzlichen Attributen oder Elementen angegeben oder definiert werden muss. Um ein `label` explizit mit einem `input` zu verbinden, können Sie das `for`-Attribut verwenden. Hier ist ein Beispiel für die Verwendung des `for`-Attributs für eine E-Mail-Bezeichnung.

Interagieren Sie mit dem `input`-Element im Vorschaufenster, indem Sie eine gefälschte E-Mail-Adresse wie `jane@example.com` eingeben. Um mit dem Beispiel zu interagieren, müssen Sie den interaktiven Editor aktivieren.

:::interactive_editor

```html
<form action="">
  <label for="email"> Email Address: </label>
  <input type="email" id="email" />
</form>
```

:::

Bei der Verwendung einer expliziten Verbindung müssen die Werte für das `for`-Attribut und `id` identisch sein. In diesem Fall sind beide Werte auf `email` gesetzt. Der `email`-Typ in der Eingabe bietet eine grundlegende Validierung für korrekt formatierte E-Mail-Adressen. Wenn Sie den Benutzern zusätzliche Hinweise zur erwarteten Eingabe geben möchten, können Sie das `placeholder`-Attribut verwenden. Hier ist ein Beispiel für die Verwendung des `placeholder`-Attributs innerhalb der E-Mail-Eingabe.

Aktivieren Sie den interaktiven Editor, klicken Sie auf die E-Mail-Eingabe und beginnen Sie, eine E-Mail einzugeben. Sie werden sehen, dass der Platzhaltertext verschwindet.

:::interactive_editor

```html
<form action="">
  <label for="email"> Email Address: </label>
  <input type="email" id="email" placeholder="example@email.com" />
</form>
```

:::

Für den Platzhaltertext sollten Sie hilfreichen kurzen Text bereitstellen, der das Format und den Typ der Daten zeigt, die Sie von Ihren Benutzern erwarten. In diesem Fall zeigt der Platzhaltertext `example@email.com` dem Benutzer, dass er eine korrekt formatierte E-Mail-Adresse eingeben muss.
