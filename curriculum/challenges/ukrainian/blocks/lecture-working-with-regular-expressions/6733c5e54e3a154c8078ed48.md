# --interactive--

Давайте вивчимо квантори в регулярних виразах.

Уявіть ситуацію, коли потрібно знайти чотиризначний ідентифікаційний код. Ви знаєте, що можна використати клас символів `\d`, тож ви могли б написати його чотири рази. І щоб уникнути зайвих символів, ви додали б як початковий, так і кінцевий анкер:

```js
const regex = /^\d\d\d\d$/;
```

І це працює – воно знайде чотири числові символи. Але замість того, щоб писати один і той самий клас кілька разів, можна додати квантор.

Квантори визначаються фігурними дужками з одним або двома числами всередині. Давайте використаємо квантор у нашому шаблоні:

```js
const regex = /^\d{4}$/;
```

Зверніть увагу, що наш квантор містить лише число `4`. Цей синтаксис означає «знайти попередній символ рівно чотири рази». Подивімося, як це працює:

:::interactive_editor

```js
const regex = /^\d{4}$/;
console.log(regex.test("123")); // false
console.log(regex.test("1234")); // true
console.log(regex.test("12345")); // false
console.log(regex.test("123456")); // false
console.log(regex.test("1234567")); // false
```

:::

Шаблон співпадає лише з рядком, що містить рівно чотири цифри, бо ми використали анкери, а наш квантор дозволяє лише рівно чотири цифри. Але можливо, ідентифікаційний код повинен мати мінімум чотири цифри.

Щоб дозволити чотири або більше цифр, додайте кому після числа у кванторі:

```js
const regex = /^\d{4,}$/;
```

Тепер наш синтаксис дозволяє шаблону знаходити чотири або більше цифр. Перевіримо:

:::interactive_editor

```js
const regex = /^\d{4,}$/;
console.log(regex.test("123")); // false
console.log(regex.test("1234")); // true
console.log(regex.test("12345")); // true
console.log(regex.test("123456")); // true
console.log(regex.test("1234567")); // true
```

:::

Сім цифр — це досить довгий ідентифікатор. Ці ідентифікатори повинні мати максимум 6 цифр і мінімум 4 цифри. Щоб це зробити, можна додати друге число у квантор після коми:

```js
const regex = /^\d{4,6}$/;
```

І тепер наш шаблон більше не співпадає з ідентифікатором із сімома цифрами, бо він більший за наш максимум у шість цифр.

:::interactive_editor

```js
const regex = /^\d{4,6}$/;
console.log(regex.test("123")); // false
console.log(regex.test("1234")); // true
console.log(regex.test("12345")); // true
console.log(regex.test("123456")); // true
console.log(regex.test("1234567")); // false
```

:::

Зверніть увагу, що ви не можете використовувати цей синтаксис, щоб встановити лише максимум — завжди потрібно вказувати мінімум. Але якщо встановити мінімум у `1`, ви фактично отримаєте той самий результат.

Ми отримали оновлені вимоги від користувачів. Тепер ідентифікатори можуть необов’язково починатися з літери. Ми вже знаємо клас символів для цього, тож додамо його до нашого регулярного виразу:

```js
const regex = /^[a-zA-Z]\d{4,6}$/;
```

Але тепер ми вимагаємо наявність літери. Як зробити її необов’язковою?

Ви могли б використати синтаксис квантора з мінімумом `0` і максимумом `1`:

```js
const regex = /^[a-zA-Z]{0,1}\d{4,6}$/;
```

Але насправді існує спеціальний скорочений квантор для одного необов’язкового символу — знак питання (`?`). Замінімо наш квантор на знак питання:

```js
const regex = /^[a-zA-Z]?\d{4,6}$/;
```

Перевіримо результат:

:::interactive_editor

```js
const regex = /^[a-zA-Z]?\d{4,6}$/;
console.log(regex.test("123")); // false
console.log(regex.test("a1234")); // true
console.log(regex.test("12345")); // true
console.log(regex.test("az12345")); // false
console.log(regex.test("X123456")); // true
console.log(regex.test("1234567")); // false
```

:::

Тепер наш шаблон дозволяє один необов’язковий символ літери, за яким іде від чотирьох до шести цифр.

На жаль, ми щойно зрозуміли, що неправильно прочитали вимоги. Потрібно дозволити будь-яку кількість літер перед цифрами. Ми можемо використати квантор з мінімумом `0` і без визначеного максимуму:

```js
const regex = /^[a-zA-Z]{0,}\d{4,6}$/;
```

Але наш шаблон знову стає довгим. На щастя, є ще одне скорочення для «знайти попередній символ нуль або більше разів» — символ зірочки (`*`). Замінімо наш квантор на нього і перевіримо:

:::interactive_editor

```js
const regex = /^[a-zA-Z]*\d{4,6}$/;
console.log(regex.test("123")); // false
console.log(regex.test("a1234")); // true
console.log(regex.test("12345")); // true
console.log(regex.test("az12345")); // true
console.log(regex.test("X123456")); // true
console.log(regex.test("1234567")); // false
```

:::

Тепер ми успішно знаходимо будь-який ідентифікатор з нульовою або більшою кількістю літер, за якими йде від чотирьох до шести цифр. Але виявилося, що це викликає збій у системі — нам насправді потрібно вимагати принаймні одну літеру.

Знову ж таки, можна використати квантор з мінімумом один і без визначеного максимуму, або скористатися ще одним спеціальним синтаксисом — символом плюса (`+`):

:::interactive_editor

```js
const regex = /^[a-zA-Z]+\d{4,6}$/;
console.log(regex.test("123")); // false
console.log(regex.test("a1234")); // true
console.log(regex.test("12345")); // false
console.log(regex.test("az12345")); // true
console.log(regex.test("X123456")); // true
console.log(regex.test("1234567")); // false
```

:::

Тепер ідентифікатори, які не починаються принаймні з однієї літери, не проходять перевірку, незалежно від кількості цифр.

Ви можете використовувати квантори, щоб значно скоротити і зробити більш читабельними ваші регулярні вирази.
