# --description--

في البرمجيات، التبعية هي عندما يعتمد مكوِّن أو وحدة في تطبيق على مكوِّن أو وحدة أخرى ليعمل بشكل صحيح. التبعيات شائعة في تطبيقات البرمجيات لأنها تتيح للمطوِّرين استخدام دوال أو أدوات مُعدة مسبقًا أنشأها الآخرون.

إذا أردت بناء تطبيق React، ستحتاج إلى تثبيت تبعيات React. بدون هذه التبعيات، لن يعمل أي من شفرتك بشكل صحيح وسيعرض التطبيق قائمة من الأخطاء.

عندما تعمل على مشاريع أكثر تعقيدًا، غالبًا ما تحتاج إلى الاعتماد على العديد من التبعيات. إذا لم تُدار مجموعة التبعيات أو تُحدد بشكل جيد في مشروع، فسيؤدي ذلك إلى ما يُعرف بجحيم التبعيات.

لإدارة تبعيات البرمجيات في مشروع، ستحتاج إلى استخدام مدير حزم. مدير الحزم هو أداة تُستخدم للتثبيت والتحديث والإزالة للتبعيات. العديد من لغات البرمجة الشهيرة مثل JavaScript وPython وRuby وJava تستخدم مديري حزم.

في درس سابق، تم تقديم مدير الحزم الشهير npm بشكل موجز.

لإنشاء مشروع React جديد باستخدام Vite وnpm يمكنك تشغيل الأمر التالي:

```sh
npm create vite@latest my-react-app -- --template react
```

كما تتذكر من الدروس السابقة، هذا سينشئ كل شفرة القالب الجاهز اللازمة لإطلاق تطبيق React جديد. قبل أن تطلق التطبيق، ستحتاج إلى تثبيت التبعيات بتشغيل `npm install` أو اختصارًا `npm i`.

يمكنك عرض كل التبعيات في ملف `package.json` الموجود في الدليل الجذري لمشروعك.

ملف `package.json` هو ملف تكوين رئيسي في المشاريع يحتوي على بيانات وصفية عن مشروعك، بما في ذلك اسمه، وإصداره، وتبعياته. كما يحدد السكربتات، ومعلومات الترخيص، وإعدادات أخرى تساعد في إدارة المشروع وتبعياته.

عند تثبيت التبعيات، سيُضاف مجلد `node_modules` إلى مشروعك.

مجلد `node_modules` هو المكان الذي تُخزن فيه كل الحزم والمكتبات المطلوبة لمشروعك. يحتوي هذا المجلد على الشفرة الفعلية للتبعيات المدرجة في ملف `package.json`، بما في ذلك تبعيات مشروعك المباشرة وأي تبعيات لتلك التبعيات.

التبعيتان الأساسيتان اللازمتان لمشروع React هما حزمتا `react` و`react-dom`:

```json
"dependencies": {
  "react": "^18.3.1",
  "react-dom": "^18.3.1"
}
```

سيُدرج ملف `package.json` الإصدارات الحالية التي تستخدمها لتلك الحزم. إذا احتجت إلى تحديث أي حزمة محليًا، يمكنك تشغيل الأمر `npm update`. أو يمكنك تحديث كل الحزم عالميًا بتشغيل الأمر `npm update -g`.

بالإضافة إلى ملف `package.json`، سيكون لديك أيضًا ملف `package-lock.json`. هذا الملف يُثبت الإصدارات الدقيقة لكل الحزم التي يستخدمها مشروعك. عند تحديث حزمة، تُحدث الإصدارات الجديدة في ملف القفل أيضًا.

جانب مهم آخر في ملف `package.json` هو التبعيات الخاصة بالتطوير:

```json
"devDependencies": {
  "@eslint/js": "^9.17.0",
  "@types/react": "^18.3.18",
  "@types/react-dom": "^18.3.5",
  "@vitejs/plugin-react": "^4.3.4",
  "eslint": "^9.17.0",
  "eslint-plugin-react": "^7.37.2",
  "eslint-plugin-react-hooks": "^5.0.0",
  "eslint-plugin-react-refresh": "^0.4.16",
  "globals": "^15.14.0",
  "vite": "^6.0.5"
}
```

التبعيات الخاصة بالتطوير هي حزم تُستخدم فقط أثناء التطوير وليس في الإنتاج. مثال على ذلك مكتبة اختبار مثل Jest. تثبت Jest كتبعيات تطوير لأنك تستخدمها فقط لاختبار مشروعك محليًا، ولا تحتاجها لتشغيل التطبيق في الإنتاج.

لجزء كبير من هذا الدرس، ركزنا على npm. لكن هناك مديري حزم آخرين مثل yarn وpnpm. إذًا، أي مدير حزم يجب أن تستخدم لمشروعك؟
