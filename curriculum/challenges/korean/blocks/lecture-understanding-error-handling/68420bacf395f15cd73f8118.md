# --description--

파이썬 코드를 작성할 때 오류가 발생하는 경우가 많습니다. 이러한 오류를 이해하는 것은 코드를 빠르고 효율적으로 디버깅하는 데 중요합니다. 오류 메시지는 어떻게 읽느냐에 따라 무엇이 잘못되었는지 정확히 알려줍니다.

일반적인 파이썬 오류에는 `SyntaxError`, `NameError`, `TypeError`, `IndexError`, `AttributeError`가 있습니다. 이들은 파이썬이 코드를 이해하지 못하거나, 논리가 작업 중인 데이터와 맞지 않을 때 발생합니다.

다음은 `SyntaxError`의 예입니다:

```py
print("Hello, world!"
# SyntaxError: unexpected EOF while parsing
```

이 줄은 닫는 괄호가 빠져 있습니다. 파이썬은 코드가 올바른 구문 규칙을 따르지 않기 때문에 `SyntaxError`를 발생시킵니다.

다음은 `NameError`의 예입니다:

```py
print(name)
# NameError: name 'name' is not defined
```

아직 정의되지 않은 변수를 출력하려고 합니다. 파이썬은 해당 이름의 변수를 찾을 수 없을 때 `NameError`를 발생시킵니다.

다음은 `TypeError`의 예입니다:

```py
5 + "5"
# TypeError: unsupported operand type(s) for +: 'int' and 'str'
```

정수와 문자열을 더할 수 없습니다. 파이썬은 호환되지 않는 데이터 유형에 연산을 시도할 때 `TypeError`를 발생시킵니다.

다음은 `IndexError`의 예입니다:

```py
my_list = [1, 2, 3]
print(my_list[5])
# IndexError: list index out of range
```

리스트에 존재하지 않는 인덱스에 접근하려고 합니다. 파이썬은 범위를 벗어날 때 `IndexError`를 발생시킵니다.

다음은 `AttributeError`의 예입니다:

```py
num = 42
num.append(5)
# AttributeError: 'int' object has no attribute 'append'
```

`int` 객체에 `append()` 메서드가 없습니다. 파이썬은 해당 데이터 유형에 존재하지 않는 메서드나 속성을 사용하려고 할 때 `AttributeError`를 발생시킵니다.

일반적인 파이썬 오류 메시지를 인식하면 문제를 더 빨리 고칠 수 있습니다. 추측하지 말고 오류 메시지를 주의 깊게 읽으세요. 대부분 무엇이 잘못되었고 어디를 봐야 하는지 정확히 알려줍니다.
