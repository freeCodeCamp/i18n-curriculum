# --description--

Wenn Sie mit CSS arbeiten, werden Sie früher oder später auf Kompatibilitätsprobleme mit Browsern stoßen. Deshalb ist es wichtig, Browser-Fallbacks bereitzustellen, um potenzielle Probleme zu vermeiden.

Wenn Ihr Browser das CSS einer Webseite parst, ignoriert er alle Eigenschaften, die er nicht erkennt oder nicht unterstützt. Wenn Sie zum Beispiel eine CSS-Variable verwenden, um die Hintergrundfarbe einer Seite zuzuweisen, wird Internet Explorer die Hintergrundfarbe ignorieren, weil er CSS-Variablen nicht unterstützt. In diesem Fall verwendet der Browser den Wert, den er für diese Eigenschaft hat. Wenn er keinen anderen Wert für diese Eigenschaft findet, fällt er auf den Standardwert zurück, der normalerweise nicht ideal ist.

Das bedeutet, wenn Sie einen Browser-Fallback bereitstellen möchten, ist es ganz einfach, einen weiteren, weiter verbreiteten Wert direkt vor Ihrer Deklaration anzugeben. So hat ein älterer Browser eine Alternative, auf die er zurückgreifen kann, während ein neuerer Browser einfach die Deklaration interpretiert, die später in der Kaskade steht.

# --instructions--

Es sieht so aus, als würde eine Variable verwendet, um die Hintergrundfarbe der `.red-box`-Klasse festzulegen. Verbessern Sie unsere Browser-Kompatibilität, indem Sie direkt vor der bestehenden Deklaration eine weitere `background`-Deklaration hinzufügen und deren Wert auf `red` setzen.

# --hints--

Ihre `.red-box`-Regel sollte einen Fallback mit der `background` enthalten, der auf `red` gesetzt ist, und zwar direkt vor der bestehenden `background`-Deklaration.

```js
const spacelessCode = __helpers.removeWhiteSpace(__helpers.removeCssComments(code));
assert.match(
  spacelessCode,
  /\.red-box{background:(red|#ff0000|#f00|rgb\(255,0,0\)|rgb\(100%,0%,0%\)|hsl\(0,100%,50%\));background:var\(--red-color\);height:200px;width:200px;}/gi
);
```
