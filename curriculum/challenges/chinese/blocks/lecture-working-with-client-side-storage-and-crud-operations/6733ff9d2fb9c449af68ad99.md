# --interactive--

客户端保存允许网站将数据存储在用户设备上。然而，像许多技术一样，它可能被滥用。 

让我们探讨一些与客户端存储关联的负面模式，重点关注 cookie 及其在跟踪和指纹识别中的滥用。

让我们从 cookies 开始。 

在之前的课程中，你学习了 cookies 是当用户访问 web 应用时保存在用户设备上的数据。cookies 的一个常见误用是过度跟踪。

网站使用 cookie 来跟踪用户与 web 应用的交互，创建他们数字活动的历史记录。这是为了目标广告，但可能引起重大隐私问题。

例如，购物网站可能会使用这样的跟踪 cookie：

```js
document.cookie = "userID=123; path=/; expires=Thu, 18 Dec 2024 6:00:00 UTC";
```

这段代码设置了一个名为 `userID`、值为 `123` 的 cookie。该 cookie 会随每个对网站的请求一起发送，从而允许网站跟踪用户的行为。虽然这看起来无害，但想象一下数百个网站共享这些信息——这可能会形成你在线生活和选择的非常详细的画像。

另一个令人担忧的做法是浏览器指纹识别。 

该技术使用客户端信息来创建用户浏览器的唯一“指纹”。网站能够收集有关你浏览器版本、已安装插件、屏幕分辨率及其他数据的信息，以唯一识别你。 

这是一个网站如何创建你的指纹的简单示例：

:::interactive_editor

```js
let fingerprintExample = navigator.userAgent + screen.width + screen.height;
console.log(fingerprintExample);
```

:::

这段代码将你的浏览器的 user agent 与你的屏幕维度结合起来。虽然这是一个基本示例，但真实的指纹识别方法要复杂得多，并且在识别用户方面可以非常准确。

`localStorage` 也可能被滥用，因为某些网站使用它来不安全地保存敏感信息。例如：

```js
localStorage.setItem('userPassword', 'someonesPasswordHere');
```

这段代码将用户的密码保存到 `localStorage`。这是一个严重的安全性风险，因为 `localStorage` 中的数据未加密且可以轻松访问。

总之，虽然客户端存储提供了许多好处，但负责任地使用它至关重要。在你继续你的网页开发旅程时，始终考虑客户端用户数据的隐私和安全性。
