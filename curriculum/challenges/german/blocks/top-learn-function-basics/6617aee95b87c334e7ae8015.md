# --description--

Wenn eine Funktion aufgerufen wird, aber kein Argument übergeben wird, dann wird dem Parameter der Wert `undefined` zugewiesen. Dies kann zu unerwarteten Ergebnissen führen, wenn die Funktion nicht dafür ausgelegt ist, diesen Fall zu behandeln.

```js
function favoriteAnimal(animal) {
  return animal + " is my favorite animal!"
}

console.log(favoriteAnimal()); // Output: undefined is my favorite animal!
```

Im obigen Beispiel wird die Funktion `favoriteAnimal` ohne Argument aufgerufen. Dem Parameter `animal` wird der Wert `undefined` zugewiesen, der dann mit dem String `" is my favorite animal!"` verkettet wird.

Wenn kein Wert übergeben wird, können Sie Standardparameter verwenden. Standardparameter erlauben es, einem Parameter einen Standardwert zuzuweisen, falls beim Aufrufen der Funktion kein Wert übergeben wird.

```js
function favoriteAnimal(animal = 'Giant Panda') {
  return animal + " is my favorite animal!"
}
```

Im obigen Beispiel ist der Standardwert des Parameters `animal` auf `'Giant Panda'` gesetzt. Wenn die Funktion `favoriteAnimal` ohne Argument aufgerufen wird, erhält der Parameter `animal` den Wert `'Giant Panda'`.

Wie verbessern Standardparameter in JavaScript-Funktionen das Verhalten von Funktionen wie `favoriteAnimal`, wenn sie ohne Argumente aufgerufen werden?
