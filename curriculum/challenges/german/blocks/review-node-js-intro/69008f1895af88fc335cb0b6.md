## Clientseitiges JavaScript

- **Clientseitiges JavaScript**: Code, der auf der Clientseite einer Webanwendung ausgeführt wird, üblicherweise in einem Webbrowser.
- **Einschränkungen von clientseitigem JavaScript**: Obwohl leistungsfähig, hat clientseitiges JavaScript bestimmte Einschränkungen:
  - Sehr eingeschränkter Zugriff auf lokale Dateien.
  - Nicht dafür ausgelegt, komplexe Anwendungslogik zu verarbeiten.
  - Potenzielle Sicherheitsbedenken (z. B. das Offenlegen sensibler Informationen wie Datenbank-Zugangsdaten).
- **Grund für die Einschränkungen**: Diese Einschränkungen bestehen, weil clientseitiges JavaScript ursprünglich ausschließlich für die Ausführung in Webbrowsern entwickelt wurde.

## Browser-Umgebung

- **JavaScript-Engine**: Webbrowser stellen die Umgebung bereit, die benötigt wird, um JavaScript-Code auszuführen, einschließlich der JavaScript-Engine.
- **Document Object Model (DOM)**: Browser ermöglichen den Zugriff auf das DOM, sodass Sie HTML-Elemente in Ihrem Code ansprechen können.
- **Historischer Kontext**: Früher konnte JavaScript nicht außerhalb eines Browsers ausgeführt werden, heute ist das mit Node.js möglich.

## Was ist Node.js?

- **Node.js**: Eine JavaScript-Laufzeitumgebung. Offiziell definiert als „eine Open-Source- und plattformübergreifende JavaScript-Laufzeitumgebung“.
- **Open Source**: Der Node.js-Code ist öffentlich verfügbar und wird von einer großen Entwickler-Community gepflegt.
- **Plattformübergreifend**: Funktioniert auf jedem Betriebssystem, einschließlich Windows, macOS und Linux.
- **JavaScript-Laufzeitumgebung**: Ermöglicht die Ausführung von JavaScript-Code außerhalb eines Browsers.

## Anwendungsfälle von Node.js

- **Häufige Anwendungen**: Node.js wird in vielen Branchen verwendet für:
  - Aufbau von Webservern und APIs, die HTTP-Anfragen verarbeiten.
  - Entwicklung von Web- und mobilen Anwendungen.
  - Verarbeitung komplexer Daten und Datenbankinteraktionen.
- **Branchenakzeptanz**: Wenn Sie eine Website oder Webanwendung nutzen oder entwickeln, die mit einer Datenbank interagiert oder komplexe Daten verarbeitet, ist die Wahrscheinlichkeit hoch, dass Node.js im Hintergrund verwendet wird.
- **Wesentliche Merkmale**: Node.js ist effizient, skalierbar und verfügt über eine große Community von Entwicklern und Betreuern weltweit.

## Browser- vs. Node-Laufzeitumgebung

- **Browser-Umgebung**: Hauptsächlich für Frontend-Webentwicklung konzipiert, führt clientseitiges JavaScript aus.
- **Node-Laufzeitumgebung**: Hauptsächlich für Backend-Webentwicklung konzipiert, führt serverseitiges JavaScript aus.
- **API-Unterschiede**: Die Umgebung bestimmt die verfügbaren APIs:
  - **Browser**: Zugriff auf die DOM-API, aber Einschränkungen beim Zugriff auf das lokale Dateisystem.
  - **Node**: Zugriff auf fast alle Systemressourcen, einschließlich des Dateisystems, aber nicht auf das DOM.

## Globale Objekte

- **Browser-Globales Objekt (`window`)**: Bietet Zugriff auf browserbezogene Funktionen:
  - Methoden zur Manipulation des DOM.
  - Verwaltung von Cookies.
  - Verarbeitung von Browser-Ereignissen.

- **Node.js Globales Objekt (`global`)**: Bietet Zugriff auf Node.js-spezifische Funktionen:
  - Eingebaute Module für die Arbeit mit lokalen Dateien.
  - Netzwerkfähigkeiten.
  - Interaktion mit dem Betriebssystem.

## Versionskontrolle

- **Browser-Version**: Sie haben keine Kontrolle über die Version der Browser-Umgebung, die Ihre Nutzer zum Besuch Ihrer Website verwenden.
- **Node.js-Version**: Sie können die Version von Node.js auswählen, die Sie auf Ihrem Server verwenden möchten.

## Bedeutung von Node.js

- **Full-Stack-Entwicklung**: Node.js hat die Webentwicklung revolutioniert, indem es Entwicklern ermöglicht, JavaScript sowohl für Frontend als auch Backend von Full-Stack-Anwendungen zu verwenden.
- **Effizienz**: Der Entwicklungsprozess ist effizienter, weil Entwickler keine neue Programmiersprache nur für die Backend-Entwicklung lernen müssen.
- **Auswirkung**: Node.js ist ein leistungsstarkes, skalierbares und vielseitiges Werkzeug, das für Entwickler weltweit sehr wichtig geworden ist.

## Vorteile von Node.js

- **Eine Sprache für Full-Stack**: Sie können sowohl Frontend als auch Backend einer Webanwendung mit JavaScript implementieren.
  - Verringert die anfängliche Lernkurve.
  - Kein Wechsel zwischen Programmiersprachen nötig.
  - Steigert Effizienz und Produktivität.

- **Nicht blockierende, ereignisgesteuerte Architektur**: Ideal für die Entwicklung von Echtzeitanwendungen, bei denen Reaktionsfähigkeit und Effizienz für eine gute Benutzererfahrung entscheidend sind.

- **Einzelner Thread und Event Loop**: Kann effektiv eine große Anzahl gleichzeitiger Anfragen und Ein-/Ausgabeoperationen verarbeiten.
  - Perfekt für Anwendungen, die mehrere Anfragen gleichzeitig bearbeiten müssen.

- **Große Community**: Tausende Entwickler weltweit pflegen Node.js ständig und fügen neue Funktionen hinzu.
  - Viele Lernressourcen verfügbar.
  - Einfach, Antworten auf Fragen zu finden.

- **npm (Node Package Manager)**: Ein leistungsstarkes Tool, mit dem Sie Pakete und Module für Ihre Projekte installieren und verwalten können.
  - Pakete für viele Zwecke verfügbar: HTTP-Anfragen verarbeiten, mit Datenbanken arbeiten, Daten und Zeiten formatieren, Code testen und mehr.
  - Wiederverwendung von Code, der von anderen Entwicklern geschrieben, getestet und geteilt wurde.
  - Macht den Arbeitsablauf schneller und effizienter.

- **Kostenersparnis**: Node.js ist kostenlos und Open Source.
  - Pakete, Bibliotheken und zugehörige Tools sind ebenfalls überwiegend kostenlos.
  - Ein wichtiger Faktor bei der Budgetoptimierung.

## Nachteile von Node.js

- **Single-Thread-Limitierung**: Führt jeweils nur einen Thread aus, das heißt, es kann immer nur eine Operation gleichzeitig bearbeiten.
  - Obwohl die Architektur eine effiziente Verarbeitung gleichzeitiger (simultaner) Anfragen ermöglicht, können CPU-intensive Aufgaben den Hauptthread blockieren und zu Leistungsproblemen führen.
  - Beispiele für CPU-intensive Aufgaben: komplexe mathematische Operationen, Bild- und Videobearbeitung, Kryptografie.
  - Es gibt Möglichkeiten, dies zu umgehen, aber sie erhöhen meist die Komplexität der Anwendung.

- **Asynchrone Programmierung**: Node.js basiert auf asynchroner Programmierung.
  - Eine Aufgabe, die lange dauern kann, wird gestartet, aber anstatt zu warten, bis sie abgeschlossen ist, läuft das Hauptprogramm weiter, während die asynchrone Aufgabe parallel ausgeführt wird.
  - Wenn die Aufgabe abgeschlossen ist, verarbeitet das Programm das Ergebnis.
  - Oft sind „Callbacks“ beteiligt, das sind Funktionen, die definieren, was passiert, wenn asynchrone Operationen abgeschlossen sind.
  - Kann den Code potenziell schwerer lesbar, verständlich und debugbar machen.

- **Qualitätsbedenken bei Paketen**: Beim Auswählen von Paketen aus npm sind manche möglicherweise nicht ständig gepflegt und können Sicherheitslücken in Ihre Anwendung einführen.
  - Entwickler sollten jedes Paket sorgfältig bewerten.
  - Prüfen, ob Pakete bewährte Qualitäts- und Sicherheitspraktiken einhalten.

## Threads

- **Thread**: Ein Ausführungspfad innerhalb eines Prozesses, wie ein Computerprogramm, das gerade läuft.

## NVM (Node Version Manager)

- **NVM**: Ein Tool zur Verwaltung mehrerer Node.js-Versionen auf demselben Computer.
- **Zweck**: Hilft Ihnen, einfach zwischen verschiedenen Node.js-Versionen für unterschiedliche Projekte zu wechseln.
- **Empfehlung**: Es wird empfohlen, NVM vor Node.js zu installieren, um später flexibel zu bleiben.

## Installation von NVM auf macOS/Linux

- **Xcode-Befehlszeilentools (macOS)**: Müssen vor dem Ausführen des NVM-Installationsskripts auf macOS manuell installiert werden.

```bash
xcode-select --install
```

- **Xcode-Installation überprüfen**:

```bash
xcode-select --version
```

- **NVM-Installationsskript herunterladen und ausführen**: Führen Sie einen dieser Befehle aus:

```bash
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.3/install.sh | bash
```

```bash
wget -qO- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.3/install.sh | bash
```

- **Terminal neu starten**: Möglicherweise müssen Sie Ihr Terminal neu starten, wenn der Vorgang abgeschlossen ist, damit der neue Befehl und die Konfiguration geladen werden.

- **NVM-Installation überprüfen**:

```bash
nvm -v
```

## Installation von Node.js mit NVM auf macOS/Linux

- **Neueste LTS-Version installieren**: Um die neueste Long-Term-Support-(LTS-)Version von Node.js zu installieren:

```bash
nvm install --lts
```

- **LTS (Long-Term Support)**: Diese Version wird normalerweise empfohlen, da sie Stabilität, Zuverlässigkeit und Sicherheit über neue oder experimentelle Funktionen stellt.
  - Gründlich getestet und produktionsbereit.
  - Garantiert längere Supportzeiträume mit Fokus auf Bugfixes und Sicherheitspatches.

- **Bestimmte Version installieren**: Sie können eine bestimmte Node.js-Version installieren, indem Sie die Versionsnummer angeben:

```bash
nvm install <version>
```

Beispiel:

```bash
nvm install 20
```

- **Zu einer Version wechseln**:

```bash
nvm use <version>
```

Beispiel:

```bash
nvm use 20
```

- **Aktuelle Node.js-Version prüfen**:

```bash
node -v
```

## Installation von NVM auf Windows

- **Offizielle NVM-Unterstützung**: Offizielle Versionen von NVM unterstützen Windows nur in bestimmten Fällen.
  - Wenn Sie das Windows Subsystem for Linux (WSL) haben, können Sie der macOS/Linux-Anleitung oben folgen.

- **nvm-windows**: Ein Node.js-Versionenverwaltungs-Tool für Windows.
  - Vollständig separates Projekt vom offiziellen NVM-Projekt.
  - Gleicher Zweck: Ihnen helfen, mit mehreren Node.js-Versionen auf demselben Gerät zu arbeiten.

- **Vor der Installation**: Es wird empfohlen, vorherige Node-Installationen zu deinstallieren, bevor Sie NVM für Windows installieren.

- **Installationsschritte**:
  - Gehen Sie zum Releases-Bereich des offiziellen Repositories: `https://github.com/coreybutler/nvm-windows/releases`.
  - Laden Sie den neuesten Installer herunter.
  - Folgen Sie den Anweisungen.

- **Terminal neu starten**: Möglicherweise müssen Sie Ihr Terminal neu starten, wenn der `nvm`-Befehl nicht sofort verfügbar ist.

- **Installation überprüfen**:

```powershell
nvm -v
```

## Grundlegende NVM- und Node.js-Befehle

- **Hinweis zu Befehlen**: Alle Befehle funktionieren für NVM, das auf macOS/Linux installiert ist, einschließlich WSL. Sie können auf `nvm-windows` anders funktionieren oder gar nicht. Alternative Befehle für `nvm-windows` werden wenn möglich angegeben.

### PowerShell-Ausführungsrichtlinie (Windows)

- **Standard-Ausführungsrichtlinie**: Die Standard-Ausführungsrichtlinie von PowerShell ist `Restricted`, die aus Sicherheitsgründen das Ausführen von Skripten verhindert.

- **PowerShell als Administrator ausführen**: Rechtsklicken Sie auf PowerShell und wählen Sie „Als Administrator ausführen“.

- **Aktuelle Ausführungsrichtlinie prüfen**:

```powershell
Get-ExecutionPolicy
```

- **Ausführungsrichtlinie auf RemoteSigned setzen**: Erlaubt lokale Skripte und `npm` auszuführen:

```powershell
Set-ExecutionPolicy RemoteSigned
```

- **Nach dem Setzen der Richtlinie**: Schließen Sie das Administratorfenster von PowerShell und öffnen Sie ein PowerShell-Fenster ohne Administratorrechte.

### Alle installierten Versionen auflisten

- **Installierte Versionen auflisten**: Um alle installierten Node.js-Versionen aufzulisten und die aktuell verwendete zu prüfen:

```bash
nvm ls
```

### Standard-Node.js-Version festlegen (NVM nur auf macOS/Linux)

- **Standardversion festlegen**: Legt die Standardversion von Node.js für neue Terminal-Sitzungen fest:

```bash
nvm alias default <version>
```

Beispiel:

```bash
nvm alias default 22.20.0
```

- **Benutzerdefinierte Aliase erstellen**: Weist einer Version einen spezifischeren Namen zu:

```bash
nvm alias <new_name> <version>
```

Beispiel:

```bash
nvm alias new-api 18.20.8
```

- **Benutzerdefinierten Alias verwenden**:

```bash
nvm use <new_name>
```

Beispiel:

```bash
nvm use new-api
```

### Eine Node.js-Version entfernen

- **Version deinstallieren**:

```bash
nvm uninstall <version>
```

Beispiel:

```bash
nvm uninstall 20.19.5
```

- **Nach Hauptversion entfernen (NVM auf macOS/Linux)**: Sie können eine Version entfernen, indem Sie den `uninstall`-Befehl mit einer Hauptversionsnummer wie `22` oder `20` ausführen:

```bash
nvm uninstall 20
```

Wenn Sie mehrere Releases für diese Hauptversion installiert haben, wie `20.14.0` und `20.12.2`, wird zuerst die neueste Version entfernt. Das Ausführen von `nvm uninstall 20` entfernt zuerst `20.14.0`.

### Node.js-Version prüfen

- **Aktuelle Version prüfen**:

```bash
node -v
```

### npm-Version prüfen

- **npm**: Ein Paketmanager für Node.js, der zum Installieren, Veröffentlichen und Verwalten von Softwarepaketen verwendet wird. Wird automatisch mit Node.js installiert.

- **npm-Version prüfen**:

```bash
npm -v
```

### Eine JavaScript-Datei ausführen

- **JavaScript-Datei ausführen**: Führt die JavaScript-Datei in der aktuell aktiven Node.js-Laufzeitumgebung aus:

```bash
node <file.js>
```

Beispiel:

```bash
node app.js
```

### Ein Node.js-Projekt erstellen

- **Projekt initialisieren**: Erstellt ein neues Node.js-Projekt, indem eine `package.json`-Datei angelegt wird, die Details und Abhängigkeiten verfolgt:

```bash
npm init
```

- **Interaktive Einrichtung**: Der `npm init`-Befehl führt Sie durch eine interaktive CLI-Anleitung zur Einrichtung der `package.json`-Datei.
  - Stellt Fragen zu Projektname, Beschreibung, Version usw.

- **Standard-Einrichtung**: Um mit einer Standard-`package.json`-Datei zu starten:

```bash
npm init -y
```

oder

```bash
npm init --yes
```

### Ein Paket installieren

- **Bestimmtes Paket installieren**:

```bash
npm install <package>
```

Beispiel:

```bash
npm install express
```

### Alle Abhängigkeiten installieren

- **Abhängigkeiten installieren**: Um alle im `package.json`-Datei eines Projekts aufgeführten Abhängigkeiten zu installieren:

```bash
npm install
```
