# --interactive--

HTMLでは、引用された要素を使って引用文を周囲のコンテンツから区別します。これにより、引用文が識別しやすい形式になります。

別の情報源から引用したセクションを表すには、ブロック引用要素を使うべきです。これは主に長い引用文に使われます。引用元にアドレスがある場合は、`cite` 属性で引用元を示せます。この属性の値は有効なURLである必要があります。以下はブロック引用要素内の引用の例です。

:::interactive_editor

```html
<blockquote cite="https://www.freecodecamp.org/news/learn-to-code-book/">
  "Can you imagine what it would be like to be a successful developer? To have built software systems that people rely upon?"
</blockquote>
```

:::

この要素には`cite`属性があります。`cite`属性の値は引用元のURLです。この属性はブロック引用の見た目を変えませんが、スクリーンリーダーや検索エンジンに引用についての情報を提供するのに役立ちます。ブラウザでは、テキストが少しインデントされているのがわかります。

ブロック引用を引用符で始めて終わらせたい場合は、引用符をテキスト内に明示的に書く必要があります。先ほどのようにブロック引用要素内に直接テキストを書いてもいいですし、複数の段落がある場合は1つ以上の段落要素で囲むこともできます。これは複数段落のテキストを同じブロック引用内にまとめたいときに便利です。以下は4つの段落の例です。

:::interactive_editor

```html
<blockquote cite="https://www.freecodecamp.org/news/learn-to-code-book/">
  <p>Build your projects. Show them to your friends. Build projects for your friends.</p>
  <p>Build your network. Help the people you meet along the way. What goes around comes around. You'll get what's coming to you.</p>   
  <p>It is not too late. Life is long.</p>
  <p>You will look back on this moment years from now and be glad you made a move.</p>
</blockquote>
```

:::

すべての段落は同じブロック引用要素内に含まれているため、同じ引用の一部です。この要素には引用元のURLが入った`cite`属性があります。ブラウザでは4つの段落が互いに揃って表示されますが、コンテナに対してはインデントされています。

ここまで`cite`属性を使って引用元を示してきましたが、この属性はユーザーに引用元を見せるものではありません。裏側で機能するだけです。

引用元を視覚的に示したい場合は、ブロック引用要素の外に`cite`という引用要素を追加できます。これは`cite`属性とは異なります。引用要素は、本、記事、曲、映画、ウェブサイト、研究論文などの参照された創作物のタイトルをマークアップするためのHTML要素です。以下は例です。

:::interactive_editor

```html
<div>
  <blockquote cite="https://www.freecodecamp.org/news/learn-to-code-book/">
    Can you imagine what it would be like to be a successful developer? To have built software systems that people rely upon?
  </blockquote>
  <p>—Quincy Larson, <cite>How to Learn to Code and Get a Developer Job [Full Book].</cite></p>
</div>
```

:::

ブロック引用要素には引用文が含まれています。その下に著者名の段落要素があり、続いて引用要素があります。引用要素には引用元の本のタイトルが入っています。

ブラウザで見ると、引用元がはっきり表示され、引用がクインシー・ラーソン著の本からのものであることがわかります。この本のタイトルは`How to Learn to Code and Get a Developer Job`です。

他の情報源からの長い引用にはこのようなブロック引用を使うべきです。しかし、段落の中で数語だけ引用したい場合もあります。

そのためにインライン引用要素があります。これは他の情報源からの短いインライン引用に使います。ほとんどの最新ブラウザはこの要素を使うと自動的に引用符を付けます。以下は例です。

:::interactive_editor

```html
<p>
  As Quincy Larson said,
  <q cite="https://www.freecodecamp.org/news/learn-to-code-book/">
    Momentum is everything.
  </q>
</p>
```

:::

段落要素の中にテキストがあり、その一部がインライン引用要素内にあります。`cite`属性を追加して引用元を示すこともできます。

これはブロック引用要素の場合と同じように機能します。見た目は変わりませんが、スクリーンリーダーや検索エンジンに引用についての情報を提供します。

ブラウザでは、引用文が段落の一部として表示され、引用符で囲まれているのがわかります。ほとんどの最新ブラウザはこれらの引用符を自動的に付けます。

ブロック引用とインライン引用の違いは何でしょうか？他の情報源からの長い引用にはブロック引用を使い、既存の段落の一部として短い引用を含めたい場合はインライン引用を使うべきです。
