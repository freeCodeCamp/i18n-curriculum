# --description--

SQLでは、主キーと外部キーを使ってテーブル間の関係を確立します。

まずは主キーから始めましょう。

主キーとは、テーブル内の各レコード（行）を一意に識別する列または列の組み合わせです。この制約により、テーブル内のレコードが主キーの値で重複することはありません。

だからこそ、テーブルには主キーが1つだけ存在します。

主キーの列の値は`NULL`であってはいけないため、常に有効な値が入ります。

PostgreSQLでは、列をテーブルの主キーにするには、データ型の後に`PRIMARY KEY`を追加するだけです。

```sql
column_name data_type PRIMARY KEY
```

これは`students`テーブルの例です。

```sql
CREATE TABLE students (
  student_id SERIAL PRIMARY KEY,
  name VARCHAR(100)
);
```

各学生にはそれぞれの`student_id`があり、このIDがテーブルの主キーになります。`SERIAL`型は、`student_id`が常に一意の値を持つことを保証するために便利です。

複合主キーは、行を識別する単一の一意な列がない場合に使います。この場合、2つ以上の列の組み合わせを主キーとして使い、それらが一緒に一意になるようにします。

これを行うには、テーブル作成時にどの列が複合主キーかを次のように指定します。

```sql
CREATE TABLE table_name (
  column1 data_type column_constraint,
  column2 data_type column_constraint,
  column3 data_type column_constraint,
  ...
  PRIMARY KEY (column1, column2)
);
```

例えば、`student_id`と`course_id`の列を持つテーブルがあり、学生がどのクラスに登録されているかを示しているとします。

各学生は複数のクラスに登録され、各クラスには複数の学生が登録されます。したがって、どちらの列も一意ではありませんが、同じ学生が同じクラスに複数回登録されることはありません。だから、2つの列の組み合わせを主キーとして使えます。

```sql
CREATE TABLE course_enrollments (
  student_id INT,
  course_id INT,
  ...
  PRIMARY KEY (student_id, course_id),
);
```

主キーは、データの一意性と整合性を確保し、データ取得を最適化し、リレーショナルデータベース内の他のテーブルとの関係を確立するために不可欠です。

次に、このレッスンで扱うもう一つのキーの種類、外部キーについて説明します。

外部キーとは、あるテーブルの列（または列の組み合わせ）で、別のテーブルの主キーを参照するものです。テーブルは複数の外部キーを持つことができます。

この例では、`customers`と`orders`という2つのテーブルがあります。

```sql
CREATE TABLE customers (
  customer_id SERIAL PRIMARY KEY,
  first_name VARCHAR(100) NOT NULL,
  ...
);

CREATE TABLE orders (
  order_id SERIAL PRIMARY KEY,
  customer_id INTEGER,
  ...
  FOREIGN KEY (customer_id) REFERENCES customers(customer_id)
);
```

`customers`テーブルの主キーは`customer_id`で、`orders`テーブルの主キーは`order_id`です。

しかし、`customer_id`テーブルには`orders`列もあります。

なぜ`orders`テーブルに顧客情報を保存しているのでしょうか？

これは`customers`テーブルと`orders`テーブルの間に関係を作るためです。各注文は、それを提出した顧客のIDを保存します。

テーブル間の関係を作るために、`customer_id`テーブルの`orders`列を、`customer_id`テーブルの`customers`列を参照する外部キーにします。

```sql
FOREIGN KEY (customer_id) REFERENCES customers(customer_id)
```

外部キーの列の値は、参照先テーブルの主キーの値と一致するか、`NULL`でなければなりません（外部キーが`NULL`を許可している場合）。

なぜこれが役立つのでしょうか？

両方のテーブルのこれらの列が正確に一致することを保証することで、外部キーと主キーは「孤立した」レコード、つまり他のテーブルに存在しないレコードを参照するレコードの作成を防ぎます。

主キーと外部キーはリレーショナルデータベースの基本です。これらが「リレーショナル」な側面を支え、現実世界のデータを正確にモデル化できるようにします。
