# --description--

パスとは、ファイルシステム内のファイルやディレクトリの場所を指定する文字列です。ウェブ開発では、パスを使って画像、スタイルシート、スクリプト、その他のウェブページなどのリソースにリンクします。絶対パスと相対パスの2種類があり、どちらもファイルシステム内のファイルの場所を指定する際に重要です。ここでは両者を見て、どちらをいつ使うべきか理解しましょう。

絶対パスはリソースへの完全なリンクです。ルートディレクトリから始まり、すべてのディレクトリを含み、最後にファイル名と拡張子が続きます。「ルートディレクトリ」とは階層の最上位のディレクトリやフォルダーを指します。

ローカルマシン上のリソースにリンクする場合は、ファイルの完全なディレクトリの場所を含む絶対パスを使います。以下は絶対パスで`about.html`ファイルにリンクする例です。

```html
<p>
  Read more on the
  <a
    href="/Users/user/Desktop/fCC/script-code/absolute-vs-relative-paths/pages/about.html"
    >About Page</a
    >
</p>
```

これはルートから始まり、`Users`というフォルダーに入り、次に`user`、`Desktop`、`fCC`、`script-code`、`absolute-vs-relative-paths`、`pages`というフォルダーを順にたどって、最終的に`about.html`ファイルにたどり着くため、このような形になります。

絶対URLはリソースにアクセスするための完全なアドレスです。プロトコル（`http`、`https`、`file`など）と、リソースがウェブ上にある場合はドメイン名を含みます。以下はfreeCodeCampのロゴにリンクする絶対URLの例です。

```html
<a href="https://design-style-guide.freecodecamp.org/img/fcc_secondary_small.svg">
  View fCC Logo
</a>
```

この例では、プロトコルが`https`、ドメイン名が`design-style-guide.freecodecamp.org`、ファイル名が`fcc_secondary_small.svg`です。

ブラウザのアドレスバーに表示される絶対URLは次のようになります。

```sh
file:///Users/user/Desktop/fCC/script-code/absolute-vs-relative-paths/pages/about.html
```

URLにはプロトコル`file://`が含まれています。また、ファイルに至るフォルダーの系列を表す`/Users/user/Desktop/fCC/script-code/absolute-vs-relative-paths/pages/`というパスも含まれています。最後に、ファイル名と拡張子である`about.html`も含まれています。

絶対パスはファイルシステム内のファイルの完全な場所を示し、ローカルマシン上のリソースでよく使われます。絶対URLはアクセス情報（プロトコルやウェブリソースの場合はドメイン名など）を含み、ブラウザにリソースの取得方法と場所を伝えます。

次に相対パスを見てみましょう。相対パスは現在のファイルのディレクトリから見たファイルの場所を指定します。プロトコルやドメイン名は含まれず、同じウェブサイト内の内部リンクに対して短く柔軟に使えます。以下は同じフォルダー内にある`about.html`ページから`contact.html`ページにリンクする例です。

```html
<p>
  Read more on the
  <a href="about.html">About Page</a>
</p>
```

`contact.html`ページにいると想像してください。`about.html`ページが同じ場所にあるので、単にファイル名だけを指定しています。これは相対ファイルパスの例です。

では、絶対パス、絶対URL、相対パスのどれをいつ使うべきでしょうか？以下のルールを守ってください。

- サイトのルートやローカルマシン上の既知のディレクトリなど、固定された場所からリソースを参照したい場合は絶対パスを使います。

- 外部のウェブサイトにホストされているリソースにリンクする場合は絶対URLを使います。

- 同じウェブサイト内のリソースにリンクする場合は相対パスを使います。

- 開発中にコードをよりシンプルで管理しやすくしたい場合は相対パスを使うことをお勧めします。

- ローカルでテストするときは、インターネット接続なしでもリンクが機能するように相対パスを使います。
