# --description--

GitHub Pages 是部署靜態網站或不需要後端伺服器來控制邏輯的應用程式的非常穩定的選項。也就是說，完全在客戶端或使用者瀏覽器中執行的應用程式，可以完全部署在這個平台上。

此外，GitHub Pages 對於開放原始碼專案完全免費。有許多充分的理由讓你的專案成為開放原始碼，而 Pages 則增添了另一個理由。

但是你實際上要如何部署到 GitHub Pages 呢？這取決於你的應用程式如何運作。

讓我們從一個基本的 HTML 頁面開始。如果你願意，可以將這個頁面添加到你現有的 repo 中：

```html
<!DOCTYPE html>
<html>
  <head>
    <title>freeCodeCamp</title>
    <link rel="stylesheet" href="./style.css" />
  </head>
  <body>
    <h1>Welcome to freeCodeCamp's Lessons!</h1>
    <p>Let's have some fun learning about Git and GitHub!</p>
  </body>
</html>
```

現在我們需要造訪我們的儲存庫並點擊「Settings」標籤以進入儲存庫設定。接著我們需要從側邊欄選擇「Pages」。

一開始只有少數幾個選項，可是隨著我們開始設定，情況會改變。首先，我們需要告訴 GitHub 從哪裡部署。

我們會想要將此保留為「從分支部署」，因為我們沒有使用框架，也沒有組態任何 GitHub 動作。你將在即將到來的課程中學習更多關於這些的內容。

接著，將我們要部署的分支設定為 `main`，以部署該分支上的內容。

接著你會有選擇，從你的專案根目錄或從 `docs` 資料夾部署。這個特性被像 freeCodeCamp 這樣的專案使用－我們過去會將文件與應用程式打包在一起，因此我們可以透過 `docs` 資料夾只將文件部署到 GitHub 頁面。暫時請保持為 `/`（根目錄）資料夾，然後點擊此下拉選單旁的「存檔」按鈕以儲存你的變更。

一旦你存檔，你將會看到一個新的輸入欄位來組態訂製網域。如果你已經有一個，並且想要使用它，你可以跟隨聯結到文件中進行操作。然而，對於本課程，我們將保持此欄位為空白。

你也可以看到你的新 GitHub Pages 網站已部署到的網域。若沒有訂製網域，預設會是 `<username>.github.io`，其中 `<username>` 是你的帳號名稱（如果你是從組織擁有的儲存庫部署，則為該組織的帳號）。而儲存庫將會在與儲存庫名稱相符的路徑上提供服務，例如 `<username>.github.io/<repo-name>`。如果你造訪該 URL，就會看到你的網站！

現在，當你將新的變更推送到你的 `main` 分支時，GitHub Pages 將自動使用最新的變更更新我們已部署的網站。此過程可能需要幾分鐘，視伺服端載入情況而定－你可能需要進行強制重新整理以清除快取。

這樣一來，我們就成功將一個基本的網頁應用程式部署到 GitHub Pages。你也可以在你的 Pages 部署中包含 CSS 和 JavaScript 檔案，因此你可以建造美觀且互動性強的網站。而且，一旦你學會像 React 或 Angular 這樣的框架，你甚至可以將它們部署到 GitHub Pages！
