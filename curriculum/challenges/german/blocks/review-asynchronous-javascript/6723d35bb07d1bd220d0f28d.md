# --description--

- **Synchrones JavaScript** wird nacheinander ausgeführt und wartet, bis die vorherige Operation abgeschlossen ist, bevor es mit der nächsten fortfährt.
- **Asynchrones JavaScript** ermöglicht es, mehrere Operationen im Hintergrund auszuführen, ohne den Hauptthread zu blockieren.
- **Thread** ist eine Folge von Anweisungen, die unabhängig vom Hauptprogrammfluss ausgeführt werden kann.
- **Callback-Funktionen** sind Funktionen, die als Argumente an andere Funktionen übergeben werden und nach Abschluss der Operation oder als Ergebnis eines Ereignisses ausgeführt werden.

## Die JavaScript-Engine und die JavaScript-Laufzeitumgebung

- Die **JavaScript-Engine** ist ein Programm, das JavaScript-Code in einem Webbrowser ausführt. Sie funktioniert wie ein Konverter, der Ihren Code nimmt, ihn in Anweisungen umwandelt, die der Computer verstehen kann, und entsprechend arbeitet.
- V8 ist ein Beispiel für eine von Google entwickelte JavaScript-Engine.
- Die **JavaScript-Laufzeitumgebung** ist die Umgebung, in der JavaScript-Code ausgeführt wird. Sie umfasst die JavaScript-Engine, die den Code verarbeitet und ausführt, sowie zusätzliche Funktionen wie einen Webbrowser oder Node.js.

## Die Fetch-API

- Die Fetch-API ermöglicht Web-Apps, Netzwerk-Anfragen zu stellen, typischerweise um Daten vom Server abzurufen oder zu senden. Sie stellt eine `fetch()`-Methode bereit, mit der Sie diese Anfragen durchführen können.
- Sie können mit der Fetch-API Text, Bilder, Audio, JSON und andere Datentypen abrufen.

## HTTP-Methoden für die Fetch-API

Die Fetch-API unterstützt verschiedene HTTP-Methoden zur Interaktion mit dem Server. Die gebräuchlichsten Methoden sind:

- **GET**: Wird verwendet, um Daten vom Server abzurufen. Standardmäßig verwendet die Fetch-API die `GET`-Methode, um Daten abzurufen.

```js
fetch('https://api.example.com/data')
```

Um die abgerufenen Daten zu verwenden, müssen sie mit der `.json()`-Methode in das JSON-Format konvertiert werden:

```js
fetch('https://api.example.com/data')
  .then(response => response.json())
  .then(data => console.log(data))
```

In diesem Code ist die Antwort, die von der Fetch-API kommt, ein Promise, und der `.then`-Handler konvertiert die Antwort in ein JSON-Format.

- **POST**: Wird verwendet, um Daten an den Server zu senden. Die `POST`-Methode wird verwendet, um neue Ressourcen auf dem Server zu erstellen.

```js
fetch('https://api.example.com/users', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({
    name: 'John Doe',
    email: 'john@example.com'
  })
})
```

In diesem Beispiel senden wir eine `POST`-Anfrage, um einen neuen Benutzer zu erstellen. Wir haben die Methode als `POST` angegeben, die passenden Header gesetzt und einen Body mit den Daten eingefügt, die wir senden möchten. Der Body muss ein String sein, daher verwenden wir `JSON.stringify()`, um unser Objekt in einen JSON-String umzuwandeln.

- **PUT**: Wird verwendet, um Daten auf dem Server zu aktualisieren. Die `PUT`-Methode wird verwendet, um bestehende Ressourcen auf dem Server zu aktualisieren.

```js
fetch('https://api.example.com/users/45', {
  method: 'PUT',
  headers: {
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({
    name: 'John Smith',
    email: 'john@example.com'
  })
})
```

In diesem Beispiel aktualisieren wir die ID `45`, die am Ende der URL angegeben ist. Wir haben die `PUT`-Methode im Code verwendet und auch die Daten als Body angegeben, die zur Aktualisierung der identifizierten Daten verwendet werden.

- **DELETE**: Wird verwendet, um Daten auf dem Server zu löschen. Die `DELETE`-Methode wird verwendet, um Ressourcen auf dem Server zu löschen.

```js
fetch('https://api.example.com/users/45', {
  method: 'DELETE'
})
```

In diesem Beispiel senden wir eine `DELETE`-Anfrage, um einen Benutzer mit der ID `45` zu entfernen.

## Promise und Promise-Verkettung

- **Promises** sind Objekte, die den endgültigen Abschluss oder das Scheitern einer asynchronen Operation und deren Ergebniswert darstellen. Der Wert des Promise ist erst bekannt, wenn die `async`-Operation abgeschlossen ist.
- Hier ist ein Beispiel, um ein einfaches Promise zu erstellen:

```js
const promise = new Promise((resolve, reject) => {
  setTimeout(() => {
    resolve('Data received successfully');
  }, 2000);
});
```

- Die `.then()`-Methode wird in einem Promise verwendet, um anzugeben, was passieren soll, wenn das Promise erfüllt wird, während `.catch()` verwendet wird, um Fehler zu behandeln, die auftreten.
- Hier ist ein Beispiel für die Verwendung von `.then()` und `.catch()` mit einem Promise:

```js
promise
  .then(data => {
    console.log(data);
  })
  .catch(error => {
    console.error(error);
  });
```

Im obigen Beispiel wird die `.then()`-Methode verwendet, um die vom Promise empfangenen Daten zu protokollieren, während die `.catch()`-Methode verwendet wird, um auftretende Fehler zu protokollieren.

- **Promise-Verkettung**: Eine der leistungsstarken Funktionen von Promises ist, dass wir mehrere asynchrone Operationen miteinander verketten können. Jede `.then()` kann ein neues Promise zurückgeben, sodass Sie eine Folge von asynchronen Operationen nacheinander ausführen können.
- Hier ist ein Beispiel für Promise-Verkettung:

```js
fetch('https://api.example.com/data')
  .then(response => response.json())
  .then(data => {
    console.log(data);
    return fetch('https://api.example.com/other-data');
  })
  .then(response => response.json())
  .then(otherData => {
    console.log(otherData);
  })
  .catch(error => {
    console.error(error);
  });
```

Im obigen Beispiel rufen wir zuerst Daten von einer URL ab, dann von einer anderen URL basierend auf der ersten Antwort und protokollieren schließlich die zweite empfangene Daten.

Die `catch`-Methode würde alle während des Prozesses auftretenden Fehler behandeln. Das bedeutet, dass Sie nicht für jeden Schritt eine Fehlerbehandlung hinzufügen müssen, was Ihren Code erheblich vereinfacht.

## Verwendung von `async/await` zur Behandlung von Promises

Async/await macht das Schreiben und Lesen von asynchronem Code einfacher und basiert auf Promises.

- **async**: Das `async`-Schlüsselwort wird verwendet, um eine asynchrone Funktion zu definieren. Eine `async`-Funktion gibt ein Promise zurück, das mit dem Wert aufgelöst wird, den die `async`-Funktion zurückgibt.
- **await**: Das `await`-Schlüsselwort wird innerhalb einer `async`-Funktion verwendet, um die Ausführung der Funktion anzuhalten, bis das Promise aufgelöst ist. Es kann nur innerhalb einer `async`-Funktion verwendet werden.
- Hier ist ein Beispiel für die Verwendung von `async/await`:

```js
async function delayedGreeting(name) {
  console.log("A Messenger entered the chat...");
  await new Promise(resolve => setTimeout(resolve, 2000));
  console.log(`Hello, ${name}!`);
}

delayedGreeting("Alice");
console.log("First Printed Message!");
```

Im obigen Beispiel ist die `delayedGreeting`-Funktion eine `async`-Funktion, die 2 Sekunden wartet, bevor sie die Begrüßungsnachricht ausgibt. Das `await`-Schlüsselwort wird verwendet, um die Ausführung der Funktion anzuhalten, bis das `Promise` aufgelöst ist.

- Einer der größten Vorteile von `async/await` ist die Fehlerbehandlung über `try/catch`-Blöcke. Hier ist ein Beispiel:

```js
async function fetchData() {
  try {
    const response = await fetch('https://api.example.com/data');
    const data = await response.json();
    console.log(data);
  } catch (error) {
    console.error(error);
  }
}

fetchData();
```

Im obigen Beispiel enthält der `try`-Block den Code, der einen Fehler werfen könnte, und der `catch`-Block behandelt den Fehler, falls er auftritt. Das macht die Fehlerbehandlung einfacher und lesbarer.

## Das `async`-Attribut

- Das `async`-Attribut weist den Browser an, die Skriptdatei asynchron herunterzuladen, während das HTML-Dokument weiterhin geparst wird.
- Sobald das Skript heruntergeladen ist, wird das HTML-Parsen pausiert, das Skript ausgeführt und dann das HTML-Parsen fortgesetzt.
- Sie sollten `async` für unabhängige Skripte verwenden, bei denen die Ausführungsreihenfolge keine Rolle spielt.

## Das `defer`-Attribut

- Das `defer`-Attribut lädt das Skript ebenfalls asynchron herunter, verschiebt aber die Ausführung des Skripts, bis das HTML-Dokument vollständig geparst wurde.
- Die `defer`-Skripte behalten die Ausführungsreihenfolge bei, wie sie im HTML-Dokument erscheinen.

- Es ist wichtig zu beachten, dass sowohl `async` als auch `defer`-Attribute für inline-Skripte ignoriert werden und nur für externe Skriptdateien funktionieren.

- Wenn sowohl `async` als auch `defer`-Attribute vorhanden sind, hat das `async`-Attribut Vorrang.

## Geolocation-API

- Die Geolocation-API bietet Websites eine Möglichkeit, den Standort des Benutzers anzufragen.

- Das folgende Beispiel zeigt die `getCurrentPosition()`-Methode der API, die verwendet wird, um den aktuellen Standort des Benutzers zu erhalten.

```js
navigator.geolocation.getCurrentPosition(
  (position) => {
    console.log("Latitude: " + position.coords.latitude);
    console.log("Longitude: " + position.coords.longitude);
  },
  (error) => {
    console.log("Error: " + error.message);
  }
);
```

In diesem Code rufen wir `getCurrentPosition` auf und übergeben eine Funktion, die aufgerufen wird, wenn die Position erfolgreich ermittelt wurde.

Das `position`-Objekt enthält verschiedene Informationen, hier haben wir nur `latitude` und `longitude` ausgewählt.

Wenn es ein Problem bei der Ermittlung der `position` gibt, wird der Fehler in der Konsole protokolliert.

- Es ist wichtig, die Privatsphäre des Benutzers zu respektieren und den Standort nur dann anzufragen, wenn es notwendig ist.
