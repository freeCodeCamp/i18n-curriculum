# --description--

Poiché Redux è un framework per la gestione dello stato, aggiornare lo stato è una delle sue attività principali. In Redux, tutti gli aggiornamenti dello stato vengono attivati inviando azioni. Un'azione è semplicemente un oggetto JavaScript che contiene informazioni su un evento di azione che si è verificato. L'archivio Redux riceve questi oggetti azione e aggiorna di conseguenza il suo stato. A volte un'azione Redux trasporta anche dei dati. Per esempio, l'azione trasporta un nome utente dopo che un utente ha effettuato il login. Sebbene i dati siano opzionali, le azioni devono avere una proprietà `type` che specifica il 'tipo' di azione che si è verificata.

Pensa alle azioni Redux come messaggeri che consegnano informazioni sugli eventi che accadono nella tua app all'archivio Redux. L'archivio poi si occupa di aggiornare lo stato in base all'azione che è avvenuta.

# --instructions--

Scrivere un'azione Redux è semplice come dichiarare un oggetto con una proprietà type. Dichiara un oggetto `action` e dagli una proprietà `type` impostata sulla stringa `'LOGIN'`.

# --hints--

Un oggetto `action` dovrebbe esistere.

```js
assert(
  (function () {
    return typeof action === 'object';
  })()
);
```

L'oggetto `action` dovrebbe avere una chiave proprietà `type` con valore `LOGIN`.

```js
assert(
  (function () {
    return action.type === 'LOGIN';
  })()
);
```
