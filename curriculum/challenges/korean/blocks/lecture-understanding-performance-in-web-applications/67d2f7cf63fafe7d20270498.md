# --description--

Lighthouse나 Chrome 개발자 도구 같은 도구도 훌륭하지만, 때로는 사이트에 맞춘 맞춤형 인사이트가 필요할 때가 있습니다.

바로 그럴 때 성능 웹 API가 도움이 됩니다!

이번 수업에서는 웹사이트 속도를 측정하고 분석하는 데 사용할 수 있는 세 가지 핵심 웹 API를 자세히 살펴보겠습니다.

그런데 성능 웹 API가 무엇일까요?

성능 웹 API는 개발자가 코드에서 직접 웹페이지가 얼마나 효율적으로 로드되고 반응하는지 추적할 수 있게 해줍니다. 이 API들은 페이지 로드 시간 측정, 렌더링 및 상호작용 지연 추적, JavaScript 실행 시간 분석을 가능하게 합니다.

이 API들을 사용하면 타사 소프트웨어에 의존하지 않고도 나만의 성능 모니터링 도구를 만들 수 있습니다!

오늘 바로 사용할 수 있는 세 가지 강력한 웹 API를 살펴보겠습니다.

먼저 `performance.now()`입니다.

이 API는 사이트의 다양한 부분이 로드되는 데 걸리는 시간을 측정할 수 있도록 고정밀 타임스탬프(밀리초 단위)를 제공합니다.

예를 들어 함수가 얼마나 빨리 실행되는지 측정하고 싶다면:

```js
const start = performance.now();  
// Run some code here  
const end = performance.now();  

console.log(`Execution time: ${end - start}ms`);
```

이것은 `Date.now()`을 사용하는 것보다 더 정확한데, 밀리초 단위에 마이크로초 정밀도를 제공해 시계 오차 문제를 피할 수 있기 때문입니다. 스크립트 실행 시간, 이벤트 응답 지연, 애니메이션 성능 추적에 사용할 수 있습니다.

다음은 Performance Timing API입니다.

이 API는 DNS 조회부터 `DOMContentLoaded`까지 페이지 로드의 모든 단계를 세부적으로 알려줍니다.

페이지가 완전히 로드되는 데 걸리는 시간을 측정하고 싶나요?

```js
let [navigationTiming] = performance.getEntriesByType("navigation");

if (navigationTiming instanceof PerformanceNavigationTiming) {
  // Calculate time from navigation start to DOM content loaded
  const pageLoadTime =
    navigationTiming.domContentLoadedEventEnd - navigationTiming.startTime;

  console.log("DOM Content Loaded Time:", pageLoadTime, "ms");
}
```

이 API로 추적할 수 있는 주요 지표는 DNS 조회 시간, 즉 연결 속도, Time to First Byte(TTFB), 즉 서버 응답 속도, 그리고 `DOMContentLoaded`, 즉 페이지가 상호작용할 준비가 된 시점입니다.

페이지 로드 시간이 느리다면 이 API가 지연이 발생하는 정확한 위치를 알려줍니다!

마지막으로 `PerformanceObserver`에 대해 이야기해 보겠습니다.

이 API는 레이아웃 이동, 긴 작업, 사용자 상호작용 같은 성능 이벤트를 감지합니다.

오래 걸리는 JavaScript 작업을 모니터링하고 싶나요?

```js
const observer = new PerformanceObserver((list) => {  
  list.getEntries().forEach((entry) => {  
    console.log(`Long task detected: ${entry.duration}ms`);  
  });  
});  

observer.observe({ type: "longtask", buffered: true });
```

이 API가 추적할 수 있는 것은 긴 작업, 즉 렌더링을 차단하는 JavaScript, UI 지연을 감지하는 레이아웃 이동, 그리고 First Input Delay(FID), 즉 페이지가 사용자 입력에 얼마나 빨리 반응하는지입니다.

실시간 성능 추적이 필요하다면 이 API가 큰 도움이 됩니다.

그렇다면 어떤 API를 사용해야 할까요? 간단한 비교표입니다:

| Performance API        | 가장 적합한 용도                                  |
| :--------------------- | :---------------------------------------------- |
| `performance.now()`    | 함수와 스크립트의 정밀한 시간 측정               |
| Performance Timing API | 전체 페이지 로드 성능 측정                       |
| Performance Observer   | 상호작용과 렌더링의 실시간 모니터링             |

이 API들을 조합하면 나만의 성능 측정 도구를 만들고 사이트에 가장 중요한 부분을 정확히 추적할 수 있습니다.
