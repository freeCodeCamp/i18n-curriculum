## 用 JavaScript 驗證表單

- **條件驗證 API**：某些 HTML 元素（像是 `textarea` 和 `input` 元素）會提供條件驗證 API。這個 API 讓你可以確認使用者在該元素輸入的值是否通過你寫的 HTML 層級驗證（例如最小長度或樣式比對）。
- **`checkValidity()` 方法**：這個方法會在元素通過所有 HTML 驗證（根據其屬性）時傳回 `true`，驗證失敗時傳回 `false`。

:::interactive_editor

```html
<form>
  <label>
    Email:
    <input
      id="email"
      type="email"
      required
      pattern=".+\.com$"
      placeholder="example@site.com"
    />
  </label>
</form>

<script>
  const input = document.getElementById("email");

  input.addEventListener("input", (e) => {
    if (!e.target.checkValidity()) {
      e.target.setCustomValidity("You must use a .com email.");
    } else {
      e.target.setCustomValidity("");
    }
  });
</script>
```

:::

- **`reportValidity()` 方法**：這個方法會告訴瀏覽器 `input` 是無效的。

:::interactive_editor

```html
<form>
  <label>
    Email:
    <input
      id="email2"
      type="email"
      required
      pattern=".+\.com$"
      placeholder="example@site.com"
    />
  </label>
</form>

<script>
  const input = document.getElementById("email2");

  input.addEventListener("input", (e) => {
    if (!e.target.checkValidity()) {
      e.target.reportValidity();
    }
  });
</script>
```

:::

- **`validity` 屬性**：此屬性用來取得表單控制元件（如 `<input>`、`<select>` 等）的有效性狀態，並提供有關使用者輸入是否符合該元素所定義條件（例如 `required` 欄位、樣式條件、最大長度等）的資訊。

:::interactive_editor

```html
<input
  id="age"
  type="number"
  min="18"
  placeholder="Enter age (18+)"
/>

<script>
  const input = document.getElementById("age");

  input.addEventListener("input", (e) => {
    console.log(e.target.validity);
  });
</script>
```

:::

- **`patternMismatch` 屬性**：如果值不符合指定的正規表示式樣式，這個屬性會是 `true`。

## `preventDefault()` 方法

- **定義**：DOM 中每個事件觸發時都會有某種預設行為。例如，點擊核取方塊會預設切換該核取方塊的狀態；在聚焦的按鈕上按下空白鍵會啟動該按鈕。這些 `preventDefault()` 物件上的 `Event` 方法可以阻止這些預設行為發生。

:::interactive_editor

```html
<form id="form">
  <input type="text" placeholder="Try to submit" />
  <button type="submit">Submit</button>
</form>

<p id="status"></p>

<script>
  const form = document.getElementById("form");
  const status = document.getElementById("status");

  form.addEventListener("submit", (event) => {
    event.preventDefault();
    status.textContent = "Form submission prevented.";
  });
</script>
```

:::

## 提交表單

- **定義**：表單有三種提交方式。第一種是使用者點擊表單中 `type` 屬性設為 `submit` 的按鈕。第二種是使用者在表單中任何可編輯的 `Enter` 欄位按下 `input` 鍵。第三種是透過 JavaScript 呼叫 `requestSubmit()` 或 `submit()` 方法於 `form` 元素上。
- **`action` 屬性**：`action` 屬性應該包含一個 URL 或目前網域的相對路徑。這個值決定表單要將資料送往哪裡——如果你沒有設定 `action` 屬性，表單會將資料送到目前頁面的 URL。

```html
<form action="https://freecodecamp.org" method="GET">
  <input
    type="number"
    name="number"
    placeholder="Enter a number"
  />
  <button type="submit">Submit</button>
</form>
```

- **`method` 屬性**：這個屬性接受標準的 `HTTP` 方法，例如 `GET` 或 `POST`，並在向 action URL 發送請求時使用該方法。如果沒有設定方法，表單預設會用 `GET` 請求。表單中的資料會以 `name=value` 配對的 URL 編碼方式，附加在 action URL 作為查詢參數。

```html
<form action="/data" method="POST">
  <input
    type="number"
    id="input"
    placeholder="Enter a number"
    name="number"
  />
  <button type="submit">Submit</button>
</form>
```

- **`enctype` 屬性**：`form` 元素接受一個 `enctype` 屬性，代表要用哪種編碼方式傳送資料。這個屬性只接受三種值：`application/x-www-form-urlencoded`（預設，將資料以 URL 編碼的表單主體傳送）、`text/plain`（以純文字方式傳送資料，`name=value` 配對以新行分隔）、或 `multipart/form-data`，專門用於處理有檔案上傳的表單。
