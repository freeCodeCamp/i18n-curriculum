# --interactive--

`IndexedDB` è uno strumento per archiviare dati strutturati nel browser. È integrato nei browser web moderni, permettendo alle app web di archiviare e recuperare oggetti JavaScript in modo efficiente.

A differenza di altri meccanismi di archiviazione come `localStorage`, che è limitato a memorizzare stringhe, `IndexedDB` può archiviare oggetti JavaScript, file e praticamente qualsiasi altro tipo di dato. Questo rende semplice per le applicazioni web lavorare con strutture dati grandi e complesse.

Ora, vediamo come funziona `IndexedDB`.

Il primo passaggio è aprire un database. Ecco un esempio:

:::interactive_editor

```js
let request = indexedDB.open("Sample DB", 1);

request.onerror = function(event) {
  console.log("Error opening database");
};

request.onsuccess = function(event) {
  let db = event.target.result;
  console.log("Database opened successfully");
};
```

:::

In questo codice, stiamo aprendo un database chiamato `"Sample DB"` con versione 1. Forniamo due funzioni di callback: una per gestire gli errori e un’altra per quando il database viene aperto con successo. L’oggetto `db` che otteniamo nella callback di successo è quello che useremo per interagire con il database.

Se controlli l’interfaccia applicazione degli strumenti per sviluppatori del browser, vedrai che il tuo `Sample DB` nella sezione `IndexedDb` è stato aperto. Una volta aperto il database, puoi iniziare a lavorare con gli object store.

Gli object store in IndexedDB sono simili alle tabelle nei database tradizionali. Contengono i dati effettivi che vuoi archiviare. Ecco come creare un object store:

```js
let request = indexedDB.open("Sample DB", 1);

request.onupgradeneeded = function(event) {
  let db = event.target.result;
  let objectStore = db.createObjectStore("customers", { keyPath: "id" });
};
```

Questo codice crea un object store chiamato `"customers"` con `"id"` come percorso chiave. Il percorso chiave è come una chiave primaria in un database tradizionale: serve a identificare univocamente ogni record.

Per aggiungere dati al nostro object store, faremmo qualcosa del genere. Il `db` in questo esempio rappresenta l’istanza del database `IndexedDB`.

```js
let transaction = db.transaction(["customers"], "readwrite");
let objectStore = transaction.objectStore("customers");
let request = objectStore.add({ id: 1, name: "John Doe", email: "john@example.com" });

request.onerror = function(event) {
  console.log("Error adding data");
};

request.onsuccess = function(event) {
  console.log("Data added successfully");
};
```

Questo codice aggiunge un nuovo cliente al nostro object store `"customers"`. Avviamo una transazione (che è il modo in cui raggruppiamo le operazioni sul database), otteniamo un riferimento al nostro object store e poi aggiungiamo i dati.

Recuperare i dati funziona in modo simile. Avviamo una transazione, otteniamo il nostro object store e poi usiamo metodi come `get` per recuperare i dati:

```js
let transaction = db.transaction(["customers"]);
let objectStore = transaction.objectStore("customers");
let request = objectStore.get(1);

request.onerror = function(event) {
  console.log("Error retrieving data");
};

request.onsuccess = function(event) {
  console.log("Customer:", request.result);
};
```

Questo codice recupera il cliente con `id` di `1` dal nostro object store `"customers"`.

Una delle funzionalità chiave di `IndexedDB` è che è asincrono.

Questo significa che quando interagisci con `IndexedDB`, le operazioni non bloccano il thread principale dell’applicazione web. Questo garantisce che la tua applicazione web rimanga responsiva anche quando gestisce grandi quantità di dati.

Sebbene `IndexedDB` offra funzionalità potenti, ha una curva di apprendimento più ripida rispetto ad altre API di archiviazione più semplici, il che può essere una sfida per chi inizia. Tuttavia, per le applicazioni che devono gestire grandi quantità di dati strutturati sul client, `IndexedDB` offre capacità senza pari.
