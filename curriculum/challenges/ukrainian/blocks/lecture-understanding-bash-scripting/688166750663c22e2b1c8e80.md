# --description--

У попередніх уроках ви дізналися про Bash і деякі загальні команди, які можна використовувати. Але Bash підтримує повноцінну мову скриптів, за допомогою якої можна виконувати більш складні автоматизовані завдання.

Bash-скрипти полягають у написанні послідовності команд Bash у файлі, який потім можна виконати за допомогою Bash, щоб запустити вміст файлу.

Давайте подивимося, як може виглядати bash-скрипт:

```bash
#!/bin/bash

servers=("prod" "dev")

for server in "${servers[@]}"
do
  echo "Pulling $server"
  rsync --archive --verbose $server:/etc/nginx/conf.d/server.conf configs/$server.conf
done
```

Цей скрипт отримує конфігураційні файли NGINX зі списку віддалених серверів, що може бути корисним для резервного копіювання цих конфігів у локальне сховище. NGINX — популярний вебсервер, і ви дізнаєтеся більше про роботу з серверами у майбутньому модулі.

Перший рядок називається "shebang" (`#!`) і повідомляє системі, який інтерпретатор слід використовувати для запуску скрипта. Загальноприйнятими значеннями тут є `/usr/bin/bash`, `/bin/bash` або `/bin/sh`.

Перший рядок, `servers=("prod" "dev")`, створює список рядків, що представляють імена серверів. `"prod"` часто використовується для позначення продуктивного сайту, тоді як `"dev"` позначає сайт розробки, який зазвичай використовується для тестування нових функцій і змін перед їх розгортанням у живому продуктивному середовищі.

Потім список ітерується за допомогою циклу `for` — синтаксис `servers[@]` розгортає список на окремі елементи. Кожен елемент призначається змінній server під час кожної ітерації.

Інструкція `do` позначає початок логічного блоку циклу, тобто коду, який має виконуватися на кожній ітерації.

`echo "Pulling $server"` виконує інтерполяцію поточного значення змінної server, яке в цьому випадку буде prod або dev, і виводить його в термінал.

Потім `rsync` використовується для отримання фактичного файлу з сервера та збереження його локально, знову ж таки з інтерполяцією для керування цією командою.

`done` позначає кінець логічного блоку циклу.

Існує кілька суттєвих переваг розуміння та використання Bash-скриптів.

Перша полягає в тому, що майже в кожному Unix-середовищі Bash доступний за замовчуванням, що означає, що ви можете запускати свій скрипт без необхідності налаштовувати середовища виконання мов, таких як Node.js або Python.

Друга — це те, що ви маєте доступ до всіх бінарних застосунків, доступних у системі, з мінімальними зусиллями. Якщо у вашій системі встановлено doctl, ваш скрипт може виконати цей бінарний файл. Бінарний файл — це скомпільована програма, яку запускає комп’ютер. Коли у вас є bash-скрипт, ви можете виконати бінарний файл, який запустить програму.

І нарешті, синтаксис, який ви використовуєте у своєму скрипті, також можна виконувати безпосередньо в терміналі. Це означає, що тестування невеликої частини скрипта стає набагато простішим — ви можете скопіювати та вставити цю частину безпосередньо в термінал і побачити результат!

Розуміння Bash не лише підвищить ваш рівень у написанні скриптів, а й зробить вас майстром командного рядка!
