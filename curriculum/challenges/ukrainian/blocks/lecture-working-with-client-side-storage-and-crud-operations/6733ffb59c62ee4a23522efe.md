# --interactive--

`IndexedDB` — це інструмент для зберігання структурованих даних у браузері. Він вбудований у сучасні веб-браузери, що дозволяє веб-застосункам ефективно зберігати та отримувати JavaScript-об’єкти.

На відміну від інших механізмів зберігання, таких як `localStorage`, які обмежені зберіганням рядків, `IndexedDB` може зберігати JavaScript-об’єкти, файли та практично будь-який інший тип даних. Це полегшує роботу веб-застосунків, яким потрібно працювати з великими та складними структурами даних.

Тепер давайте подивимося, як працює `IndexedDB`.

Перший крок — відкрити базу даних. Ось приклад:

:::interactive_editor

```js
let request = indexedDB.open("Sample DB", 1);

request.onerror = function(event) {
  console.log("Error opening database");
};

request.onsuccess = function(event) {
  let db = event.target.result;
  console.log("Database opened successfully");
};
```

:::

У цьому коді ми відкриваємо базу даних з іменем `"Sample DB"` версії 1. Ми надаємо дві функції зворотного виклику: одну для обробки помилок, іншу — для випадку успішного відкриття бази даних. Об’єкт `db`, який ми отримуємо у функції зворотного виклику успіху, — це те, чим ми користуватимемося для взаємодії з базою даних.

Якщо перевірити інтерфейс застосунку у інструментах розробника браузера, ви побачите, що ваша `Sample DB` у розділі `IndexedDb` була відкрита. Після відкриття бази даних можна починати працювати з об’єктними сховищами.

Об’єктні сховища в IndexedDB схожі на таблиці в традиційних базах даних. Вони містять фактичні дані, які ви хочете зберегти. Ось як створити об’єктне сховище:

```js
let request = indexedDB.open("Sample DB", 1);

request.onupgradeneeded = function(event) {
  let db = event.target.result;
  let objectStore = db.createObjectStore("customers", { keyPath: "id" });
};
```

Цей код створює об’єктне сховище з іменем `"customers"` з `"id"` як шляхом ключа. Шлях ключа — це як первинний ключ у традиційній базі даних — він використовується для унікальної ідентифікації кожного запису.

Щоб додати дані до нашого об’єктного сховища, ми зробили б щось подібне. `db` у цьому прикладі представляє екземпляр бази даних `IndexedDB`.

```js
let transaction = db.transaction(["customers"], "readwrite");
let objectStore = transaction.objectStore("customers");
let request = objectStore.add({ id: 1, name: "John Doe", email: "john@example.com" });

request.onerror = function(event) {
  console.log("Error adding data");
};

request.onsuccess = function(event) {
  console.log("Data added successfully");
};
```

Цей код додає нового клієнта до нашого об’єктного сховища `"customers"`. Ми починаємо транзакцію (це спосіб згрупувати операції з базою даних), отримуємо референс на наше об’єктне сховище, а потім додаємо наші дані.

Отримання даних працює подібним чином. Ми починаємо транзакцію, отримуємо об’єктне сховище, а потім використовуємо методи, як `get`, щоб отримати дані:

```js
let transaction = db.transaction(["customers"]);
let objectStore = transaction.objectStore("customers");
let request = objectStore.get(1);

request.onerror = function(event) {
  console.log("Error retrieving data");
};

request.onsuccess = function(event) {
  console.log("Customer:", request.result);
};
```

Цей код отримує клієнта з `id` `1` з нашого об’єктного сховища `"customers"`.

Однією з ключових функцій `IndexedDB` є те, що він асинхронний.

Це означає, що коли ви взаємодієте з `IndexedDB`, операції не блокують головний потік веб-застосунку. Це гарантує, що ваш веб-застосунок залишається адаптивним навіть при роботі з великими обсягами даних.

Хоча `IndexedDB` надає потужні можливості, він має більш круту криву навчання порівняно з іншими простішими API для зберігання, що може бути складним для початківців. Проте для застосунків, яким потрібно обробляти великі обсяги структурованих даних на стороні клієнта, `IndexedDB` пропонує неперевершені можливості.
