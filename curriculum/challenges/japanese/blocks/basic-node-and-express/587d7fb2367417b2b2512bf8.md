# --description--

パス`/name`にPOSTハンドラを設置してください。前回と同じパスです。htmlのフロントページにフォームを用意しています。これは演習10（クエリ文字列）と同じデータを送信します。ボディパーサーが正しく設定されていれば、パラメータはオブジェクト`req.body`の中に見つかるはずです。いつものライブラリの例を見てみましょう：

<blockquote>ルート: POST '/library'<br>urlencoded_body: userId=546&#x26;bookId=6754 <br>req.body: {userId: '546', bookId: '6754'}</blockquote>

前回と同じJSONオブジェクト`{name: 'firstname lastname'}`で応答してください。アプリのフロントページに用意したhtmlフォームを使ってエンドポイントが動作するかテストしてください。

ヒント：GETやPOST以外にもいくつかのhttpメソッドがあります。そして慣例として、http動詞とサーバーで実行する操作には対応関係があります。慣例的な対応は以下の通りです：

POST（時にPUT） - リクエストで送信された情報を使って新しいリソースを作成する、

GET - 既存のリソースを変更せずに読み取る、

PUTまたはPATCH（時にPOST） - 送信されたデータを使ってリソースを更新する、

DELETE - リソースを削除する。

また、サーバーとの接続を交渉するために使われる他のメソッドもいくつかあります。GETを除き、上記の他のメソッドはペイロード（つまりリクエストボディ内のデータ）を持つことができます。ボディパーサーミドルウェアはこれらのメソッドでも動作します。
