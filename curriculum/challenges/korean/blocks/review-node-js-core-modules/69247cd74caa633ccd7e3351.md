## Node.js 코어 모듈 소개

- **Node.js 코어 모듈**: Node.js에 내장되어 기본 제공되는 모듈로, 파일 작업, 암호화, 운영 체제 상호작용 등 필수 기능을 제공합니다.
- **표준 모듈**: 별도 설치 없이 Node.js가 설치되어 있으면 바로 사용할 수 있습니다.
- **가져오기 구문**: `require()`를 사용해 코어 모듈을 애플리케이션에 가져오세요.

```javascript
const fs = require("fs");
const crypto = require("crypto");
const os = require("os");
```

## 파일 시스템(fs) 모듈

- **목적**: 파일과 폴더를 열고 닫고 읽고 쓰고 삭제하는 작업을 위한 메서드를 제공합니다.
- **가져오기**: `const fs = require("fs");`
- **유연성**: 콜백, 프로미스, 동기 방식 세 가지 형태로 메서드를 사용할 수 있습니다.

### 메서드 사용 패턴

- **콜백을 사용하는 비동기 방식**: 기본 동작이며, 블로킹하지 않습니다.

```javascript
fs.writeFile("filePath", "content", "utf8", (err) => {
  if (err) throw err;
  console.log("File written to!");
});
```

- **프로미스 기반**: 최신 애플리케이션에 권장되며 `async/await`를 사용합니다.

```javascript
const fs = require("fs/promises");

async function writeToFile() {
  try {
    await fs.writeFile("article.md", "## Node fs Module");
    console.log("File written to!");
  } catch (err) {
    console.error(err);
  }
}
```

- **동기 방식**: 블로킹 작업으로 간단한 스크립트에 적합합니다.

```javascript
try {
  fs.writeFileSync("article.md", "## Node fs Module", "utf8");
  console.log("File written to!");
} catch (err) {
  console.error(err);
}
```

### 주요 파일 작업

- **`writeFile()` 메서드**: 지정한 내용으로 파일을 생성하거나 덮어씁니다.

```javascript
await fs.writeFile("article.md", "## Node fs Module: The Complete Guide");
```

- **`appendFile()` 메서드**: 기존 파일 끝에 내용을 추가합니다.

```javascript
await fs.appendFile("article.md", "\n\nIn this article, you will learn...");
```

- **`readFile()` 메서드**: 파일 내용을 읽습니다.

```javascript
const content = await fs.readFile("article.md", "utf8");
console.log("File content:", content);
```

- **`unlink()` 메서드**: 파일 시스템에서 파일을 삭제합니다.

```javascript
await fs.unlink("article.md");
console.log("File deleted successfully");
```

### 문자 인코딩

- **UTF-8 인코딩**: `"utf8"`을 지정해 파일을 텍스트 문자열로 읽으세요.
- **버퍼 형식**: 인코딩을 지정하지 않으면 파일이 바이너리 버퍼 객체로 읽힙니다.

```javascript
// Returns string
const textContent = await fs.readFile("file.txt", "utf8");

// Returns Buffer
const bufferContent = await fs.readFile("file.txt");
```

## 버퍼 모듈

- **목적**: 파일, 이미지, 네트워크 스트림의 바이너리 데이터를 메모리에서 직접 처리합니다.
- **가져오기**: `const { Buffer } = require("buffer");`
- **사용 사례**: 파일 입출력, TCP 스트림, 이미지 처리, 바이너리 데이터 조작 등.

### 버퍼 생성

- **`Buffer.from()` 메서드**: 문자열, 배열 또는 기타 데이터로부터 버퍼를 생성합니다.

```javascript
// From string
const myStrBuffer = Buffer.from("freeCodeCamp");
console.log(myStrBuffer); // <Buffer 66 72 65 65 43 6f 64 65 43 61 6d 70>

// From array of numbers
const myNumBuffer = Buffer.from([70, 82, 69, 69, 67, 79, 68, 69, 67, 65, 77, 80]);
```

- **`Buffer.alloc()` 메서드**: 지정한 크기의 0으로 채워진 버퍼를 생성합니다.

```javascript
const someBuffer = Buffer.alloc(10);
console.log(someBuffer); // <Buffer 00 00 00 00 00 00 00 00 00 00>
```

### 버퍼 작업

- **배열과 같은 접근**: 인덱스 표기법으로 버퍼의 개별 요소에 접근할 수 있습니다.

```javascript
console.log(myStrBuffer[0]); // 102 (byte value for 'f')
```

- **`toString()` 메서드**: 버퍼를 읽을 수 있는 문자열로 변환합니다.

```javascript
console.log(myStrBuffer.toString()); // "freeCodeCamp"
```

- **`Buffer.write()` 메서드**: 할당된 버퍼에 데이터를 씁니다.

```javascript
someBuffer.write("Hello fCC");
console.log(someBuffer.toString()); // "Hello fCC"
```

- **`Buffer.byteLength()` 메서드**: 문자열을 저장하는 데 필요한 바이트 수를 반환합니다.

```javascript
console.log(Buffer.byteLength("Hello freeCodeCamp")); // 18
```

### 추가 버퍼 메서드

- **`Buffer.isBuffer()`**: 객체가 버퍼인지 확인합니다.
- **`Buffer.compare()`**: 두 버퍼를 비교해 정렬 순서를 반환합니다.
- **`Buffer.concat()`**: 여러 버퍼를 하나로 합칩니다.

## 암호화 모듈

- **목적**: 해싱, 암호화, 복호화, 디지털 서명 등 암호화 기능을 제공합니다.
- **가져오기**: `const crypto = require("crypto");`
- **보안 참고**: 프로덕션 인증에는 `bcrypt` 또는 `jsonwebtoken` 같은 검증된 라이브러리를 사용하세요.

### 해싱 메서드

- **`createHash()` 메서드**: SHA256, SHA512, MD5 같은 알고리즘으로 단방향 해시를 생성합니다.

```javascript
const hashedPassword = crypto
  .createHash("sha256")
  .update("myPassword123")
  .digest("hex");

console.log("createHash result:", hashedPassword);
```

- **`createHmac()` 메서드**: 인증과 데이터 무결성을 위해 비밀 키를 사용해 해시를 생성합니다.

```javascript
const hashedMessage = crypto
  .createHmac("sha256", "secret-key")
  .update("Hello World")
  .digest("hex");
```

### 암호화 및 복호화

- **`createCipheriv()` 및 `createDecipheriv()` 메서드**: 알고리즘, 키, 초기화 벡터를 사용해 데이터를 암호화하고 복호화합니다.

```javascript
const key = Buffer.from("12345678901234567890123456789012"); // 32 bytes for AES-256
const iv = Buffer.from("1234567890123456"); // 16 bytes for AES

// Encryption
const cipher = crypto.createCipheriv("aes-256-cbc", key, iv);
let encrypted = cipher.update("Hello campers!", "utf8", "hex");
encrypted += cipher.final("hex");

// Decryption
const decipher = crypto.createDecipheriv("aes-256-cbc", key, iv);
let decrypted = decipher.update(encrypted, "hex", "utf8");
decrypted += decipher.final("utf8");
```

### 난수 생성

- **`randomBytes()` 메서드**: 암호학적으로 안전한 난수 바이트를 생성합니다.

```javascript
console.log("Random Bytes:", crypto.randomBytes(16).toString("hex"));
// Output: a6154ef5a296fa176ad0f332bd94d712
```

- **`randomInt()` 메서드**: 지정한 범위 내에서 안전한 난수 정수를 생성합니다.

```javascript
console.log("Random Int:", crypto.randomInt(0, 100)); // Random number 0-99
```

### 키 관리

- **`createSecretKey()` 메서드**: 원시 바이트로부터 암호화 키 객체를 생성합니다.

```javascript
const secret = crypto.createSecretKey(crypto.randomBytes(32));
console.log(secret.export().toString('hex'));
```

### 디지털 서명

- **`sign()` 및 `verify()` 메서드**: 개인키/공개키 쌍을 사용해 디지털 서명을 생성하고 검증합니다.
- **사용 사례**: 데이터의 진위와 무결성을 증명합니다.

### 추가 암호화 메서드

- **`createPublicKey()` 및 `createPrivateKey()`**: 외부에서 생성된 키를 다룹니다.
- **`createDiffieHellman()`**: 안전한 공유 비밀 생성 기능을 활성화합니다.
- **`Certificate()`**: HTTPS 인증서 작업을 처리합니다.

## 운영 체제(OS) 모듈

- **목적**: 시스템 정보와 크로스 플랫폼 호환성을 위한 운영 체제 관련 유틸리티를 제공합니다.
- **가져오기**: `const os = require("os");`
- **사용 사례**: 시스템 모니터링, 크로스 플랫폼 스크립팅, 하드웨어 정보 조회.

### 시스템 정보 메서드

- **`platform()` 메서드**: 운영 체제 플랫폼을 반환합니다.

```javascript
console.log(os.platform()); // 'darwin', 'win32', 'linux', etc.

// Cross-platform usage
if (os.platform() === 'win32') {
  // Windows-specific code
} else {
  // Unix-like systems
}
```

- **`arch()` 메서드**: CPU 아키텍처를 반환합니다.

```javascript
console.log(os.arch()); // 'arm64', 'x64', 'arm', 'ia32', etc.
```

- **`type()` 메서드**: 공식 운영 체제 이름을 반환합니다.

```javascript
console.log(os.type()); // 'Darwin', 'Linux', 'Windows_NT'
```

### 버전 정보

- **`release()` 메서드**: OS 커널 버전을 표시합니다.

```javascript
console.log(os.release()); // '25.0.0'
```

- **`version()` 메서드**: 상세한 OS 버전 정보를 반환합니다.

```javascript
console.log(os.version());
// Darwin Kernel Version 25.0.0: Wed Sep 17 21:41:39 PDT 2025
```

### 하드웨어 정보

- **`cpus()` 메서드**: CPU 코어 정보 배열을 반환합니다.

```javascript
console.log(os.cpus()); // Array of CPU core objects with model, speed, times
```

- **`uptime()` 메서드**: 시스템 가동 시간을 초 단위로 반환합니다.

```javascript
console.log(os.uptime()); // 23047 (seconds since system boot)
```

### 메모리 정보

- **`totalmem()` 메서드**: 시스템 전체 메모리 용량을 바이트 단위로 반환합니다.
- **`freemem()` 메서드**: 사용 가능한 시스템 메모리를 바이트 단위로 반환합니다.
