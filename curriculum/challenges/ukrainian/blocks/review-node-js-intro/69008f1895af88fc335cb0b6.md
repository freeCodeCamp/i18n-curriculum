## JavaScript на стороні клієнта

- **JavaScript на стороні клієнта**: Код, який виконується на стороні клієнта веб-застосунку, зазвичай у веббраузері.
- **Обмеження JavaScript на стороні клієнта**: Хоча він потужний, JavaScript на стороні клієнта має певні обмеження:
  - Дуже обмежений доступ до локальних файлів.
  - Не призначений для обробки складної логіки застосунку.
  - Можливі проблеми з безпекою (наприклад, розкриття конфіденційної інформації, такої як облікові дані бази даних).
- **Причина обмежень**: Ці обмеження існують, бо JavaScript на стороні клієнта спочатку був розроблений для виключного запуску у веббраузерах.

## Середовище браузера

- **JavaScript-двигун**: Веббраузери надають середовище, необхідне для запуску JavaScript-коду, включно з JavaScript-двигуном.
- **Document Object Model (DOM)**: Браузери надають доступ до DOM, що дозволяє отримувати доступ до HTML-елементів у вашому коді.
- **Історичний контекст**: Раніше JavaScript не міг запускатися поза браузером, але тепер це можливо з Node.js.

## Що таке Node.js?

- **Node.js**: Середовище виконання JavaScript. Офіційно визначене як «відкрите та кросплатформне середовище виконання JavaScript».
- **Відкритий код**: Код Node.js є публічно доступним і підтримується великою спільнотою розробників.
- **Кросплатформність**: Працює на будь-якій операційній системі, включно з Windows, macOS і Linux.
- **Середовище виконання JavaScript**: Дозволяє запускати JavaScript-код поза браузером.

## Сфери застосування Node.js

- **Загальні застосунки**: Node.js використовується у багатьох галузях для:
  - Створення вебсерверів і API, які обробляють HTTP-запити.
  - Розробки веб- та мобільних застосунків.
  - Обробки складних даних і взаємодії з базами даних.
- **Впровадження в індустрії**: Якщо ви використовуєте або розробляєте вебсайт чи вебзастосунок, який взаємодіє з базою даних або обробляє складні дані, велика ймовірність, що Node.js працює за лаштунками.
- **Ключові характеристики**: Node.js є ефективним, масштабованим і має велику спільноту розробників і підтримувачів по всьому світу.

## Середовище браузера проти середовища виконання Node

- **Середовище браузера**: Переважно призначене для фронтенд-розробки, виконує JavaScript на стороні клієнта.
- **Середовище виконання Node**: Переважно призначене для бекенд-розробки, виконує JavaScript на стороні сервера.
- **Відмінності в API**: Середовище визначає доступні API:
  - **Браузер**: Доступ до DOM API, але обмеження на доступ до локальної файлової системи.
  - **Node**: Доступ майже до всіх системних ресурсів, включно з файловою системою, але без DOM.

## Глобальні об’єкти

- **Глобальний об’єкт браузера (`window`)**: Надає доступ до функцій, пов’язаних із браузером:
  - Методи для маніпуляції DOM.
  - Керування куками.
  - Обробка подій браузера.

- **Глобальний об’єкт Node.js (`global`)**: Надає доступ до специфічних функцій Node.js:
  - Вбудовані модулі для роботи з локальними файлами.
  - Можливості мережі.
  - Взаємодія з операційною системою.

## Контроль версій

- **Версія браузера**: Ви не контролюєте версію браузера, яку використовуватимуть ваші користувачі для відвідування сайту.
- **Версія Node.js**: Ви можете обрати версію Node.js, яку хочете використовувати на сервері.

## Важливість Node.js

- **Full-stack розробка**: Node.js змінив веброзробку, дозволивши розробникам використовувати JavaScript для створення як фронтенду, так і бекенду повноцінних застосунків.
- **Ефективність**: Процес розробки стає ефективнішим, бо розробникам не потрібно вивчати нову мову програмування лише для бекенду.
- **Вплив**: Node.js — потужний, масштабований і універсальний інструмент, який став дуже важливим для розробників у всьому світі.

## Переваги Node.js

- **Одна мова для full-stack**: Ви можете реалізувати і фронтенд, і бекенд вебзастосунку, використовуючи JavaScript.
  - Зменшує початкову криву навчання.
  - Не потрібно перемикатися між мовами програмування.
  - Підвищує ефективність і продуктивність.

- **Неблокуюча, подієво-орієнтована архітектура**: Ідеальна для розробки реального часу застосунків, де важливі швидкість реакції та ефективність для створення гарного користувацького досвіду.

- **Один потік і цикл подій**: Може ефективно обробляти велику кількість одночасних запитів і операцій вводу-виводу.
  - Ідеально підходить для застосунків, які потребують одночасної обробки багатьох запитів.

- **Велика спільнота**: Тисячі розробників по всьому світу постійно підтримують Node.js і додають нові функції.
  - Багато навчальних ресурсів.
  - Легко знайти відповіді на запитання.

- **npm (Node Package Manager)**: Потужний інструмент, який дозволяє встановлювати і керувати пакетами та модулями для ваших проєктів.
  - Пакети доступні для багатьох цілей: обробка HTTP-запитів, робота з базами даних, форматування дат і часу, тестування коду тощо.
  - Повторне використання коду, написаного, протестованого і поділеного іншими розробниками.
  - Прискорює і робить робочий процес ефективнішим.

- **Економічність**: Node.js безкоштовний і з відкритим кодом.
  - Пакети, бібліотеки та пов’язані інструменти також переважно безкоштовні.
  - Важливий фактор при оптимізації бюджету.

## Недоліки Node.js

- **Обмеження одно потоку**: Виконує лише один потік одночасно, тобто може обробляти лише одну операцію за раз.
  - Хоча архітектура дозволяє ефективно обробляти одночасні запити, задачі, що інтенсивно використовують центральний процесор, можуть блокувати головний потік і спричиняти проблеми з продуктивністю.
  - Приклади таких задач: складні математичні операції, обробка зображень і відео, криптографія.
  - Існують способи обійти це, але вони зазвичай ускладнюють застосунок.

- **Асинхронне програмування**: Node.js базується на асинхронному програмуванні.
  - Задача, яка може тривати довго, запускається, але замість очікування її завершення головна програма продовжує виконання, поки асинхронна задача працює паралельно.
  - Коли задача завершується, програма обробляє результат.
  - Часто це пов’язано з «функціями зворотного виклику» — функціями, які визначають, що відбувається після завершення асинхронної операції.
  - Може ускладнювати читання, розуміння і налагодження коду.

- **Проблеми з якістю пакетів**: При виборі пакетів з npm деякі можуть не підтримуватися постійно і можуть вносити уразливості у ваш застосунок.
  - Розробникам слід ретельно оцінювати кожен пакет.
  - Перевіряти, чи дотримуються пакети найкращих практик якості та безпеки.

## Потоки

- **Потік**: Шлях виконання всередині процесу, як комп’ютерна програма, що зараз працює.

## NVM (Node Version Manager)

- **NVM**: Інструмент для керування кількома версіями Node.js на одному комп’ютері.
- **Призначення**: Допомагає легко перемикатися між різними версіями Node.js для різних проєктів.
- **Рекомендація**: Рекомендується встановити NVM перед Node.js для майбутньої гнучкості.

## Встановлення NVM на macOS/Linux

- **Інструменти командного рядка Xcode (macOS)**: Потрібно вручну встановити перед запуском скрипта встановлення NVM на macOS.

```bash
xcode-select --install
```

- **Перевірка встановлення Xcode**:

```bash
xcode-select --version
```

- **Завантаження і запуск скрипта встановлення NVM**: Виконайте одну з цих команд:

```bash
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.3/install.sh | bash
```

```bash
wget -qO- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.3/install.sh | bash
```

- **Перезапуск термінала**: Можливо, доведеться перезапустити термінал після завершення процесу, щоб завантажити нові команди і конфігурацію.

- **Перевірка встановлення NVM**:

```bash
nvm -v
```

## Встановлення Node.js за допомогою NVM на macOS/Linux

- **Встановлення останньої LTS-версії**: Щоб встановити останню версію з довготривалою підтримкою (LTS) Node.js:

```bash
nvm install --lts
```

- **LTS (довготривала підтримка)**: Цю версію зазвичай рекомендують, бо вона пріоритетно орієнтована на стабільність, надійність і безпеку замість нових або експериментальних функцій.
  - Ретельно протестована і готова до продакшену.
  - Гарантовано має триваліший період підтримки з акцентом на виправлення помилок і патчі безпеки.

- **Встановлення конкретної версії**: Можна встановити конкретну версію Node.js, вказавши номер версії:

```bash
nvm install <version>
```

Приклад:

```bash
nvm install 20
```

- **Перемикання на версію**:

```bash
nvm use <version>
```

Приклад:

```bash
nvm use 20
```

- **Перевірка поточної версії Node.js**:

```bash
node -v
```

## Встановлення NVM на Windows

- **Офіційна підтримка NVM**: Офіційні версії NVM підтримують Windows лише в окремих випадках.
  - Якщо у вас є Windows Subsystem for Linux (WSL), ви можете слідувати інструкціям для macOS/Linux вище.

- **nvm-windows**: Утиліта для керування версіями Node.js на Windows.
  - Повністю окремий проєкт від офіційного NVM.
  - Мета та сама: допомогти працювати з кількома версіями Node.js на одному пристрої.

- **Перед встановленням**: Рекомендується видалити будь-яку попередню інсталяцію Node перед встановленням NVM для Windows.

- **Кроки встановлення**:
  - Перейдіть у розділ Releases офіційного репозиторію: `https://github.com/coreybutler/nvm-windows/releases`.
  - Скачайте останній інсталятор.
  - Дотримуйтесь інструкцій.

- **Перезапуск термінала**: Можливо, доведеться перезапустити термінал, якщо команда `nvm` не доступна одразу.

- **Перевірка встановлення**:

```powershell
nvm -v
```

## Основні команди NVM і Node.js

- **Примітка щодо команд**: Усі команди працюють для NVM, встановленого на macOS/Linux, включно з WSL. Вони можуть працювати інакше або не працювати взагалі для `nvm-windows`. Альтернативні команди для `nvm-windows` наведені, коли це можливо.

### Політика виконання PowerShell (Windows)

- **Політика виконання за замовчуванням**: Політика виконання PowerShell за замовчуванням — `Restricted`, що забороняє запуск скриптів з міркувань безпеки.

- **Запуск PowerShell від імені адміністратора**: Клікніть правою кнопкою на PowerShell, потім виберіть «Запустити від імені адміністратора».

- **Перевірка поточної політики виконання**:

```powershell
Get-ExecutionPolicy
```

- **Встановлення політики RemoteSigned**: Дозволяє запуск локальних скриптів і `npm`:

```powershell
Set-ExecutionPolicy RemoteSigned
```

- **Після встановлення політики**: Закрийте вікно PowerShell адміністратора і відкрийте звичайне вікно PowerShell.

### Перелік усіх встановлених версій

- **Перелік встановлених версій**: Щоб побачити всі встановлені версії Node.js і перевірити, яка зараз використовується:

```bash
nvm ls
```

### Встановлення версії Node.js за замовчуванням (тільки NVM на macOS/Linux)

- **Встановлення версії за замовчуванням**: Встановлює версію Node.js за замовчуванням для нових сесій термінала:

```bash
nvm alias default <version>
```

Приклад:

```bash
nvm alias default 22.20.0
```

- **Створення кастомних псевдонімів**: Призначте більш конкретну назву версії:

```bash
nvm alias <new_name> <version>
```

Приклад:

```bash
nvm alias new-api 18.20.8
```

- **Використання кастомного псевдоніма**:

```bash
nvm use <new_name>
```

Приклад:

```bash
nvm use new-api
```

### Видалення версії Node.js

- **Видалення версії**:

```bash
nvm uninstall <version>
```

Приклад:

```bash
nvm uninstall 20.19.5
```

- **Видалення за основною версією (NVM на macOS/Linux)**: Можна видалити версію, виконавши команду `uninstall` з основною версією, наприклад `22` або `20`:

```bash
nvm uninstall 20
```

Якщо у вас встановлено кілька релізів для цієї основної версії, наприклад `20.14.0` і `20.12.2`, спочатку буде видалена остання версія. Виконання `nvm uninstall 20` спочатку видалить `20.14.0`.

### Перевірка версії Node.js

- **Перевірка поточної версії**:

```bash
node -v
```

### Перевірка версії npm

- **npm**: Менеджер пакетів для Node.js, який використовується для встановлення, публікації та керування програмними пакетами. Автоматично встановлюється разом з Node.js.

- **Перевірка версії npm**:

```bash
npm -v
```

### Запуск JavaScript-файлу

- **Виконання JavaScript-файлу**: Запускає JavaScript-файл у поточному активному середовищі виконання Node.js:

```bash
node <file.js>
```

Приклад:

```bash
node app.js
```

### Створення проєкту Node.js

- **Ініціалізація проєкту**: Створює новий проєкт Node.js, створюючи файл `package.json`, який відстежує його деталі та залежності:

```bash
npm init
```

- **Інтерактивне налаштування**: Команда `npm init` проведе вас через інтерактивний інтерфейс командного рядка для налаштування файлу `package.json`.
  - Запитує назву проєкту, опис, версію тощо.

- **Налаштування за замовчуванням**: Щоб почати з файлу `package.json` за замовчуванням:

```bash
npm init -y
```

або

```bash
npm init --yes
```

### Встановлення пакета

- **Встановлення конкретного пакета**:

```bash
npm install <package>
```

Приклад:

```bash
npm install express
```

### Встановлення всіх залежностей

- **Встановлення залежностей**: Щоб встановити всі залежності, перелічені у файлі `package.json` проєкту:

```bash
npm install
```
