# --description--

Je, umewahi kupata programu yako ya Express ikivunjika bila ujumbe wa hitilafu ulio wazi? Kurekebisha makosa na kurekodi kunaweza kusaidia kuona hasa kinachoendelea nyuma ya pazia, ili upate na kurekebisha matatizo kwa haraka zaidi.

Katika funzo hili, tutaangalia jinsi Express inavyoshughulikia kurekebisha makosa na kurekodi—na jinsi unavyoweza kutumia ili kuboresha mchakato wako wa ukuzaji.

## Kurekebisha makosa iliyojengwa ndani na Express

Express tayari ina chombo cha kurekebisha makosa kinachoitwa `debug`. Huna haja ya kusakinisha chochote ili kuitumia.

Kuzima kurekebisha makosa, unachohitaji kufanya ni kuweka kigezo cha mazingira cha `DEBUG` unapoendesha programu yako.

Hapa kuna mfano:

```js
DEBUG=express:* node index.js
```

Amri hii inawasha ujumbe wote wa kurekebisha makosa chini ya nafasi ya majina ya `express:*`. Utaanza kuona matokeo kama:

```js
express:router:route dispatching GET /
express:router:layer processing GET /
```

Ujumbe huu unakusaidia kufuatilia usambazaji wa njia za data, wito wa programu ya kati, na zaidi. Ni njia nzuri ya kuelewa mtiririko wa maombi katika programu yako.

Unataka kuwa maalum zaidi? Unaweza kupunguza nafasi ya majina:

```js
DEBUG=express:router node index.js
```

Hii itaonyesha tu rekodi zinazohusiana na mantiki ya kipanga-njia cha Express.

Kwenye Windows, sintaksia ni tofauti kidogo kwa PowerShell:

```js
$env:DEBUG = "express:*"; node index.js
```

Na kwa CMD, inaonekana hivi:

```js
set DEBUG=express:* && node index.js
```

Wazo ni sawa, ni njia tofauti ya kuweka vigezo vya mazingira.

Sasa tuchunguze kurekodi katika Express.

Kwa kurekodi maombi ya HTTP, wasanidi programu wengi hutumia programu ya kati kama `morgan`.

Hivi ndivyo unavyotumia:

```bash
npm install morgan
```

Kisha iungeze kwenye programu yako:

```js
const express = require('express')
const morgan = require('morgan')
const app = express()

app.use(morgan('dev'))s
```

Sasa, kila ombi linalokuja litaandikwa kwenye terminal na maelezo kama njia, URL, nambari ya hali, na muda wa jibu.

Na kwa nini hili ni muhimu?

Kurekebisha makosa kunakupa ufahamu wa jinsi Express inavyoshughulikia maombi, programu ya kati, na kupanga njia bila kujaa rekodi zako za uzalishaji.

Kurekodi kunakupa kumbukumbu ya kinachoendelea katika maombi, ambayo ni muhimu kwa kutafuta hitilafu, kuelewa trafiki, na kugundua matatizo mapema.

Pamoja, vinakupa udhibiti na uwazi katika mchakato wako wa ukuzaji.
