# --description--

Спливання подій, або їх поширення, означає, як подія «спливає» до батьківських об’єктів, коли вона виконується. Наприклад, розглянемо цей код:

```html
<p>
  <span>Click me~!</span>
</p>
```

Елемент `p` тут вважається батьківським елементом для `span`.

Коли ви натискаєте на елемент `span`, як вам наказано, елемент `span` стає ціллю події `click`. Однак ця подія також спливає до батьківського елемента – `p` може отримати і обробити цю подію за потреби.

Але що це насправді означає? Ви можете приєднати слухача подій до елемента `p`:

```js
const p = document.querySelector("p");
p.addEventListener("click", (event) => {
  console.log(event.target);
});
```

Тоді, коли ви натиснете на елемент `span`, у консолі з’явиться текст `Click me~!`.

Подія поширюється до батьківського елемента `p`, який обробляє її у слухачі подій, щоб показати ціль події.

Зверніть увагу, що ціль все ще елемент `span`. Це тому, що початковий клік отримав елемент `span`.

Щоб краще зрозуміти, як це працює, розширимо наш код:

```js
const p = document.querySelector("p");
const span = document.querySelector("span");
p.addEventListener("click", (event) => {
  console.log("P listener: ");
  console.log(event.target);
});
span.addEventListener("click", (event) => {
  console.log("Span listener: ");
  console.log(event.target);
});
```

Щоб показати, як подія спливає, ось що ви побачите в консолі після натискання на елемент `span`:

```html
"Span listener: "
<span>Click me~!</span>
"P listener: "
<span>Click me~!</span>
```

Тепер подивимося, що станеться, якщо ви зупините поширення події за допомогою `stopPropagation()`. Викличемо його у слухачі подій `span`:

```js
const p = document.querySelector("p");
const span = document.querySelector("span");
p.addEventListener("click", (event) => {
  console.log("P listener: ");
  console.log(event.target);
});
span.addEventListener("click", (event) => {
  console.log("Span listener: ");
  console.log(event.target);
  event.stopPropagation();
});
```

Потім знову натиснемо на `span`:

```html
"Span listener: "
<span>Click me~!</span>
```

Цього разу ми не побачимо спрацьовування слухача `p`. Подія ніколи не доходить до елемента `p`, бо ми наказали зупинити поширення, поки вона оброблялася для дочірнього елемента `span`.

Делегування подій можна вважати протилежним процесом. Це процес взяття захопленої події і делегування її іншому елементу.

Повертаючись до нашого коду, оновимо його так, щоб натискання на елемент `span` змінювало його колір на червоний:

```js
const p = document.querySelector("p");
const span = document.querySelector("span");
p.addEventListener("click", (event) => {});
span.addEventListener("click", (event) => {
  event.target.style.color = "red";
});
```

Але що, якщо у вас є двадцять елементів `span`? Або ви використовуєте JavaScript, щоб динамічно створювати більше елементів `span`?

Замість того, щоб приєднувати слухача подій до кожного елемента `span`, ви можете використати слухача на батьківському елементі `p` для всіх них. Іншими словами, ви делегуєте обробку кліків `span` батьківському елементу `p`.

Наш код тепер може виглядати приблизно так:

```js
const p = document.querySelector("p");
p.addEventListener("click", (event) => {
  event.target.style.color = "red";
});
```

Зверніть увагу, що ми більше не маємо жодного слухача, приєднаного до елемента `span`. Ви правильно делегували обробку подій елементу `p`. Але чи це працює?

Давайте створимо кілька додаткових елементів `span` і перевіримо:

```html
<p>
  <span>Click me~!</span>
  <span>Click me~!</span>
  <span>Click me~!</span>
  <span>Click me~!</span>
</p>
```

Тепер кожного разу, коли ми натискаємо на `span`, текст цього елемента стане червоним.

І ось так, з одним слухачем подій ми правильно дозволили події `click` спливати від елементів `span` до батьківського `p` і делегували логіку обробки цієї події `click` елементу `p`.

Поширення та делегування подій може бути складною темою, особливо коли йдеться про сильно вкладені елементи, як-от таблиці. Рекомендуємо вам дослідити це глибше і поекспериментувати з кодом, який ми тут написали.
