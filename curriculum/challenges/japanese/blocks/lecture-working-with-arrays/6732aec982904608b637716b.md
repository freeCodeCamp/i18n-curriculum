# --interactive--

JavaScriptの配列は動的なので、簡単に要素を追加したり削除したりできます。配列の先頭と末尾に要素を追加・削除する主なメソッドは4つあります：`push()`、`pop()`、`shift()`、`unshift()`です。それぞれのメソッドを詳しく見ていきましょう。

`push()`メソッドは、配列の末尾に1つ以上の要素を追加するために使います。`push()`メソッドの戻り値は配列の新しい長さです。既存の`fruits`配列に新しい果物を追加する例を見てみましょう。

:::interactive_editor

```js
const fruits = ["apple", "banana"];
const newLength = fruits.push("orange");
console.log(newLength); // 3
console.log(fruits); // ["apple", "banana", "orange"]
```

:::

この例では、`fruits`という2つの要素を持つ配列から始めます。次に`push()`メソッドを使って文字列`orange`を配列の末尾に追加しています。

`fruits`配列を宣言するときに`const`を使っていることに気づいたかもしれません。でも、`fruits`が定数なのに、なぜこの`fruits`配列に要素を追加できるのでしょうか？これは、`const`キーワードで配列を宣言すると、その配列への参照が作成されるためです。配列自体は変更可能ですが、`fruits`定数に新しい値を再代入することはできません。例えば次のようにです。

```js
const fruits = ["apple", "banana"];
fruits = ["This", "will", "not", "work"];
console.log(fruits); // Uncaught TypeError: Assignment to constant variable. 
```

次に見ていくのは`pop()`メソッドです。`pop()`メソッドは配列の最後の要素を削除し、その要素を返します。元の配列も変更されます。使い方は次の通りです。

:::interactive_editor

```js
let fruits = ["apple", "banana", "orange"];
let lastFruit = fruits.pop();
console.log(fruits); // ["apple", "banana"]
console.log(lastFruit); // "orange"
```

:::

この例では、3つの果物が入った配列から始めます。`pop()`メソッドは配列の最後の要素（`orange`）を削除して返します。元の`fruits`配列は変更され、2つの要素だけになります。

`unshift()`メソッドは配列の先頭に1つ以上の要素を追加し、新しい長さを返します。`push()`と似ていますが、配列の末尾ではなく先頭を変更します。例を見てみましょう。

:::interactive_editor

```js
let numbers = [2, 3];
let newLength = numbers.unshift(1);
console.log(numbers); // [1, 2, 3]
console.log(newLength); // 3
```

:::

この例では、`unshift()`を使って`numbers`配列の先頭に数値`1`を追加しています。メソッドは配列の新しい長さ`3`を返します。

最後に`shift()`メソッドを見てみましょう。`shift()`は配列の最初の要素を削除し、その要素を返します。`pop()`と似ていますが、配列の末尾ではなく先頭で動作します。使い方は次の通りです。

:::interactive_editor

```js
let colors = ["red", "green", "blue"];
let firstColor = colors.shift();
console.log(colors); // ["green", "blue"]
console.log(firstColor); // "red"
```

:::

この例では、3つの色が入った配列から始めます。`shift()`メソッドは配列の最初の要素（`red`）を削除して返します。元の`colors`配列は2つの要素だけになります。

`push()`と`unshift()`は複数の要素を一度に追加できますが、`pop()`と`shift()`は一度に1つの要素しか削除しないことに注意してください。
