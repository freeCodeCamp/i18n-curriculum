# --description--

더하기 `+`는 두 가지 형태가 있습니다: 이전 도전 과제에서 사용한 이항 형태와 단항 형태입니다.

단항 더하기, 즉 단일 값에 적용되는 더하기 연산자 `+`는 숫자에는 아무런 영향을 주지 않습니다. 하지만 피연산자가 숫자가 아니면 단항 더하기는 그것을 숫자로 변환합니다.

예를 들어:

```js
// No effect on numbers
let x = 1;
console.log( +x ); // 1

let y = -2;
console.log( +y ); // -2

// Converts non-numbers
console.log( +true ); // 1
console.log( +"" );   //
```

문자열을 숫자로 변환해야 할 필요가 매우 자주 발생합니다. 예를 들어, HTML 폼 필드에서 값을 가져올 때 보통 문자열입니다. 이 값을 합산하려면 어떻게 해야 할까요?

이항 더하기는 문자열로 더합니다:

```js
let apples = "2";
let oranges = "3";

alert( apples + oranges ); // "23", the binary plus concatenates strings
```

숫자로 처리하려면 변환한 후 합산해야 합니다:

```js
let apples = "2";
let oranges = "3";

// both values converted to numbers before the binary plus
alert( +apples + +oranges ); // 5

// the longer variant
// alert( Number(apples) + Number(oranges) ); // 5
```

수학자의 관점에서는 더하기가 이렇게 많은 것이 이상해 보일 수 있지만, 프로그래머의 관점에서는 특별할 것 없습니다: 단항 더하기가 먼저 적용되어 문자열을 숫자로 변환하고, 그다음 이항 더하기가 합산합니다.

왜 단항 더하기가 이항 더하기보다 먼저 적용될까요? 곧 알게 되겠지만, 단항 더하기가 _더 높은_ 우선순위를 가지기 때문입니다.

# --questions--

## --text--
JavaScript에서 단항과 이항 `+` 연산자에 대한 설명을 바탕으로, 단항 `'2'`를 사용해 문자열 값 `'3'`과 `+`를 숫자로 올바르게 합산하려면 어떻게 해야 할까요?

## --answers--

`console.log('2' + '3'); // Outputs: '23'` 

---

`console.log(+2 + +3); // Outputs: 5`

---

`console.log(+'2' + +'3'); // Outputs: 5`

---

`console.log(Number('2') + Number('3')); // Outputs: '23'`


## --video-solution--

3
