# --description--

Cuando vemos una tabla, inmediatamente comenzamos a hacer asociaciones visuales entre los datos y los encabezados.

Por ejemplo, digamos que tenemos una tabla que representa nuestras mascotas. Tenemos dos perros y dos gatos y la tabla nos muestra sus nombres y edades. Mientras que una persona vidente puede comprender las relaciones en esta tabla, asociar los valores con sus encabezados resulta mucho más difícil para las personas que usan lectores de pantalla para navegar por la tabla.

Como desarrollador web, eres responsable de crear estas asociaciones y estructurar tu código HTML de una manera que sea fácil de interpretar para los usuarios de lectores de pantalla.

Así que veamos cómo puedes crear tablas accesibles que todos puedan entender. La primera mejor práctica que cubriremos es el uso de leyendas (`caption`) para las tablas. Con el elemento `caption`, puedes escribir la leyenda (o título) de una tabla, para que los usuarios, especialmente aquellos que utilizan tecnologías de asistencia, puedan comprender rápidamente el propósito y contenido de la tabla. Debes colocar el elemento `caption` inmediatamente después de la etiqueta de apertura del elemento `table`. De esta manera, los lectores de pantalla y otras tecnologías de asistencia pueden ofrecer más contexto anunciando primero la leyenda antes de leer el contenido.

```html
<table>
  <caption>Our Pets</caption>
  <!-- Table Rows and Columns -->
</table>
```

Ahora hablemos sobre los encabezados de filas y columnas. Los encabezados son celdas especiales, que típicamente se encuentran al inicio de una fila o columna, que describen el tipo de datos almacenados en la fila o columna. Puedes definir un encabezado de fila o columna con el elemento de encabezado de tabla, `th`.

Por ejemplo, el código a continuación crea una tabla para dos mascotas. Cada fila tiene un encabezado de fila (el nombre de la mascota) y cada columna tiene un encabezado de columna, que describe qué representa el dato en la columna (edad y tipo).

```html
<table>
  <caption>Our Pets</caption>
  <thead>
    <tr>
      <!-- Column Headers -->
      <th>Name</th>
      <th>Age</th>
      <th>Type</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <th>Nora</th> <!-- Row Header -->
      <td>5</td>
      <td>Dog</td>
    </tr>
    <tr>
      <th>Gino</th> <!-- Row Header -->
      <td>2</td>
      <td>Cat</td>
    </tr>
  </tbody>
</table>
```

Observa que el código anterior tiene un elemento `caption` inmediatamente después del elemento `table`. Luego, dentro del elemento `thead` del encabezado de la tabla, están los encabezados de columna (`Nombre`, `Edad`, y `Tipo` ). En la segunda y tercera filas, dentro del elemento `tbody`, encontramos los datos de cada una de nuestras mascotas. Los nombres de las mascotas son los encabezados de fila porque están dentro de los elementos de encabezado de tabla (`th`).

Asociar las celdas de datos con sus correspondientes encabezados también es muy importante para los lectores de pantalla. El atributo `scope` determina si un encabezado es un encabezado de fila o de columna. Los lectores de pantalla pueden adivinar esto correctamente a partir de la estructura de la tabla, pero generalmente se recomienda indicar explícitamente el `scope` para asegurar claridad.

El atributo `scope` tiene cuatro valores posibles. Los dos valores que usarás más a menudo son `col` para columna y `row` para fila. En el código a continuación, puedes ver que añadimos el atributo `scope` a los encabezados de columna y fila. Los tres encabezados de columna (`Nombre`, `Edad`, y `Tipo`) tienen un `scope` de `col`, columna.

Los dos encabezados de fila (`Nora` y `Gino`) tienen un `scope` de `row`.

```html
<table>
  <caption>Our Pets</caption>
  <thead>
    <tr>
      <!-- Now they have scope -->
      <th scope="col">Name</th>
      <th scope="col">Age</th>
      <th scope="col">Type</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <th scope="row">Nora</th>
      <td>5</td>
      <td>Dog</td>
    </tr>
    <tr>
      <th scope="row">Gino</th>
      <td>2</td>
      <td>Cat</td>
    </tr>
  </tbody>
</table>
```

Si un encabezado de columna o fila se extiende a través de múltiples celdas, el `scope` también se aplicará a cada una de las celdas individualmente. Aquí tienes un ejemplo de eso:

<table>
  <tbody>
    <tr>
      <td></td>
      <th scope="col">Nombre</th>
      <th scope="col">Edad</th>
    </tr>
    <tr>
      <th rowspan="2" scope="row">Perros</th>
      <th scope="row">Nora</th>
      <td>5</td>
    </tr>
    <tr>
      <th scope="row">Gino</th>
      <td>2</td>
    </tr>
    <tr>
      <th rowspan="2" scope="row">Gatos</th>
      <th scope="row">Lulu</th>
      <td>10</td>
    </tr>
    <tr>
      <th scope="row">Elizabeth</th>
      <td>6</td>
    </tr>
  </tbody>
</table>

En esta tabla, la celda con la edad de `Nora` (`5`) tendrá un encabezado de columna (`Edad`) y dos encabezados de fila (`Perros` y `Nora`). La edad de `Gino` (`2`) también tendrá un encabezado de columna (`Edad`) y dos encabezados de fila (`Perros` y `Gino`).

Sin embargo, algunos lectores de pantalla pueden no ser capaces de interpretar tablas con estructuras complejas, por lo que también deberías intentar simplificar la tabla tanto como sea posible para evitar encabezados de fila y columna que abarquen múltiples celdas.

Tu objetivo siempre debería ser asegurarte de que los usuarios puedan acceder a esta información, incluso si sus lectores de pantalla pueden manejar estructuras de tabla complejas.

Ahora, en cuanto al ancho de las celdas, se recomienda evitar usar valores fijos. Debes usar valores relativos en su lugar, como porcentajes. Además, intenta evitar definir la altura de las celdas. Esto permitirá a los usuarios ajustar el tamaño del texto para adaptarse a sus necesidades.

Y finalmente, deberías permitir que el navegador determine el ancho de la tabla siempre que sea posible, para reducir la necesidad de desplazamiento horizontal.

Las tablas HTML son esenciales para presentar datos estructurados en un formato accesible y comprensible. Siguiendo estas directrices de accesibilidad, puedes crear tablas que sean fáciles de entender para todos.
