# --description--

package.jsonファイルのdependenciesセクションにあるnpmパッケージの`Versions`は、依存関係の管理を簡単にすることを目的としたソフトウェアのバージョン管理の業界標準であるセマンティックバージョニング（SemVer）に従っています。npmで公開されるライブラリ、フレームワーク、その他のツールは、プロジェクトがアップデートした際にどのような変更があるかを明確に伝えるためにSemVerを使用するべきです。

SemVerを知っていると、外部依存関係を使ったソフトウェア開発（ほぼ常にそうですが）に役立ちます。いつか、この数字の意味を理解していることで、昨日は動いていたのに今日突然動かなくなった理由がわからずに、プロジェクトに壊れる変更を誤って導入することを防げます。公式サイトによると、セマンティックバージョニングは次のように機能します：

```json
"package": "MAJOR.MINOR.PATCH"
```

MAJORバージョンは互換性のないAPI変更を行ったときに増やします。MINORバージョンは後方互換性のある機能追加を行ったときに増やします。PATCHバージョンは後方互換性のあるバグ修正を行ったときに増やします。つまり、PATCHはバグ修正で、MINORは新機能の追加ですが、どちらも以前動いていたものを壊しません。最後に、MAJORは以前のバージョンと互換性のない変更を加えます。

# --instructions--

`package.json`ファイルのdependenciesセクションで、`@freecodecamp/example`のバージョンをMAJORバージョン1、MINORバージョン2、PATCHバージョン13に変更してください。
