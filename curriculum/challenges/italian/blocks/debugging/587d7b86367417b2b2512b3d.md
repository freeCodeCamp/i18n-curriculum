# --description--

L’ultimo argomento è il temuto loop infinito. I loop sono strumenti utili quando vuoi che il tuo programma esegua un blocco di codice un certo numero di volte o finché non si verifica una condizione, ma hanno bisogno di una condizione terminale che fermi il loop. I loop infiniti rischiano di bloccare o far crashare il browser e causare caos nell’esecuzione del programma, cosa che nessuno vuole.

C’è stato un esempio di loop infinito nell’introduzione di questa sezione: non aveva una condizione terminale per uscire dal loop `while` dentro `loopy()`. NON chiamare questa funzione!

```js
function loopy() {
  while(true) {
    console.log("Hello, world!");
  }
}
```

Spetta al programmatore assicurarsi che la condizione terminale, che dice al programma quando uscire dal codice del loop, venga raggiunta alla fine. Un errore comune è incrementare o decrementare una variabile contatore nella direzione sbagliata rispetto alla condizione terminale. Un altro errore è resettare accidentalmente una variabile contatore o indice dentro il codice del loop, invece di incrementarla o decrementarla.

# --instructions--

La funzione `myFunc()` contiene un loop infinito perché la condizione terminale `i != 4` non sarà mai `false` (e quindi non interromperà il loop) — `i` incrementerà di 2 a ogni passaggio, saltando proprio il 4 dato che `i` è dispari all’inizio. Correggi l’operatore di confronto nella condizione terminale in modo che il loop venga eseguito solo per `i` minore o uguale a 4.
