# --explanation--

`Clarifying` означає пояснювати щось так, щоб усунути плутанину. Це допомагає краще зрозуміти концепцію або ситуацію. Наприклад:

`Thanks for clarifying how the API works.` — мовник вдячний, бо пояснення зробило API легшим для розуміння.

`Abstraction` стосується спрощення чогось, показуючи лише основні функції та приховуючи складну внутрішню логіку. Наприклад:

`Using a login function is an example of abstraction - you don't need to know how the password is checked, just that it works.` — це показує, як користувачі взаємодіють із простими інтерфейсами, тоді як технічна логіка залишається прихованою.
