# --interactive--

當使用者在你的網站上填寫表單時，重要的是他們必須以正確的格式填寫所有必要的資訊。HTML 表單控制元件，例如輸入欄位，具有許多內建的驗證功能，你可以用來檢查無效的資料。這將有助於確保使用者在資訊提交並由伺服端處理程序處理之前修正這些錯誤。

「client-side」一詞指的是發生在使用者的電腦或裝置上的所有事物，例如你直接互動的網站或應用程式的那部分。這包括版面配置、設計以及任何互動功能。

「server-side」一詞指的是在主機網站或應用程式的伺服器、電腦或系統上發生的所有事情。這包括處理資料、執行應用程式，以及處理來自使用者裝置的請求。

雖然客戶端驗證很重要，但你也需要伺服端驗證以增強安全性。惡意使用者可能會繞過客戶端檢查，因此強固的伺服端措施是必要的。你將在後續的模組中學習更多相關內容。現在，讓我們來看看一些客戶端表單驗證的範例。

一個常見的內建表單驗證範例是使用輸入欄位中的 `required` 屬性。`required` 屬性指定使用者必須填寫該部分的表單，才能提交。以下是使用 `required` 屬性於電子郵件輸入欄位的範例。

點擊 `Submit Form` 按鈕而未提供電子郵件位址，你將會看到一則訊息彈出，告訴你要填寫該欄位。

:::interactive_editor

```html
<form action="">
  <label for="email">Email Address (Required field):</label>
  <input required type="email" name="email" id="email" />
  <button type="submit">Submit Form</button>
</form>
```

:::

每個瀏覽器都會有自己的一組樣式來顯示此警示訊息。

使用電子郵件輸入的另一個優點是電子郵件輸入具有一些基本驗證，以確保電子郵件位址格式正確。例如，如果你輸入隨機字並點擊提交，瀏覽器會顯示警告，指出缺少 `@` 符號。 

在電子郵件欄位中輸入 `abc`，然後點擊提交按鈕。你應該會看到一則訊息彈出，因為這不是有效的電子郵件位址。

:::interactive_editor

```html
<form action="">
  <label for="email">Email Address (Required field):</label>
  <input required type="email" name="email" id="email" />
  <button type="submit">Submit Form</button>
</form>
```

:::

重要的是要注意，瀏覽器只會對標準電子郵件位址進行基本驗證。額外的驗證層由你來添加，你將在後續的模組中學習相關內容。

電子郵件輸入的其他驗證方式是使用 `minlength` 和 `maxlength` 屬性。以下是使用額外驗證的範例。

在欄位中輸入 `b@m` 並點擊提交按鈕。你會看到一則訊息彈出，因為它未達到最低要求長度。

:::interactive_editor

```html
<form action="">
  <label for="email">Email Address (Required field):</label>
  <input
    required
    type="email"
    name="email"
    id="email"
    minlength="4"
    maxlength="64"
  />
  <button type="submit">Submit Form</button>
</form>
```

:::

`minlength` 和 `maxlength` 屬性用於設定電子郵件輸入的最小和最大字元長度。如果你沒有包含最小長度或超過最大字元長度，瀏覽器將顯示警示訊息。
