# --description--

Una strategia è un modo per autenticare un utente. Puoi usare una strategia per permettere agli utenti di autenticarsi basandosi su informazioni salvate localmente (se li fai registrare prima) o da diversi provider come Google o GitHub. Per questo progetto useremo il middleware Passport. Passport offre un insieme completo di strategie che supportano l'autenticazione usando nome utente e password, GitHub, Google e altro.

`passport-local@~1.0.0` è già stata aggiunta come dipendenza. Aggiungila al tuo server come segue:

```javascript
const LocalStrategy = require('passport-local');
```

Dì a passport di **usare** un oggetto `LocalStrategy` istanziato con alcune impostazioni definite. Assicurati che questo (così come tutto ciò che segue) sia racchiuso nella connessione al database, dato che ne dipende!:

```javascript
passport.use(new LocalStrategy((username, password, done) => {
  myDataBase.findOne({ username: username }, (err, user) => {
    console.log(`User ${username} attempted to log in.`);
    if (err) return done(err);
    if (!user) return done(null, false);
    if (password !== user.password) return done(null, false);
    return done(null, user);
  });
}));
```

Qui si definisce il processo da usare quando provi ad autenticare qualcuno localmente. Prima cerca un utente nel tuo database con il nome utente inserito. Poi verifica che la password corrisponda. Infine, se non sono emersi errori che hai controllato (ad esempio una password errata), viene restituito l'oggetto `user` e l'utente è autenticato.

Molte strategie si configurano usando impostazioni diverse. In generale, è facile configurarle seguendo il README del repository di quella strategia. Un buon esempio è la strategia GitHub, dove non devi preoccuparti di nome utente o password perché l'utente viene mandato alla pagina di autenticazione di GitHub. Finché è loggato e acconsente, GitHub restituisce il suo profilo da usare.

Nel passaggio successivo configurerai come chiamare effettivamente la strategia di autenticazione per convalidare un utente basandoti sui dati del modulo.
