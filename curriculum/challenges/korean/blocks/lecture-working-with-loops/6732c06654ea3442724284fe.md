# --interactive--

이전 수업에서 `for` 루프, `for...in` 루프, `for...of` 루프를 다루는 방법을 배웠습니다. 이번 수업에서는 `while` 루프와 `do...while` 루프에 대해 배웁니다.

`while` 루프는 조건이 참인 동안 코드 블록을 실행합니다. `while` 루프의 기본 구문은 다음과 같습니다:

```js
while (condition) {
  // code block to be executed
}
```

조건은 코드 블록이 실행되기 전에 검사됩니다. 조건이 거짓이면 코드 블록은 실행되지 않습니다.

`while` 루프는 코드 블록을 몇 번 실행해야 할지 모를 때 유용합니다. `while` 루프를 사용하는 예는 다음과 같습니다:

:::interactive_editor

```js
let counter = 0;
while(counter < 5) {
  console.log(counter);
  counter++;
}
```

:::

이 예제에서는 `counter`라는 변수가 `0`으로 초기화되어 있습니다. `while` 루프는 `counter`의 값이 `5`보다 작을 동안 계속 실행됩니다. 루프 안에서는 `counter`의 값을 콘솔에 출력하고 `counter`를 `1`만큼 증가시킵니다.

`while` 루프와 비슷한 또 다른 루프는 `do...while` 루프입니다. 기본 구문은 다음과 같습니다:

```js
do {
  // code block to be executed
} while (condition);
```

`do...while` 루프와 `while` 루프의 주요 차이점은 `do...while` 루프가 조건을 검사하기 전에 코드 블록을 최소 한 번 실행한다는 점입니다.

조건이 참이면 코드 블록이 계속 실행됩니다. 조건이 거짓이면 코드 블록 실행이 중단됩니다.

`do...while` 루프를 사용하는 예는 다음과 같습니다:

:::interactive_editor

```js
let counter = 0;
do {
  console.log(counter);
  counter++;
} while (counter < 5);  
```

:::

이 예제에서는 `counter`라는 변수가 `0`으로 초기화되어 있습니다. `do...while` 루프는 `counter`의 값을 콘솔에 출력하고 `counter`를 `1`만큼 증가시킵니다. 코드 블록을 실행한 후 `counter`의 값이 `5`보다 작은지 검사합니다. 작으면 루프가 계속 실행되고, 그렇지 않으면 루프가 중단됩니다.

대부분의 경우 `while` 루프를 `do...while` 루프보다 더 자주 사용할 것입니다. 하지만 두 종류의 루프 모두 알고 언제 사용하는지 아는 것이 좋습니다.
