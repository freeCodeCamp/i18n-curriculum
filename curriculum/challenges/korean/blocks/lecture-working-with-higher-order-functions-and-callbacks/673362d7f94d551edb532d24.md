# --interactive--

프로그래밍에서 데이터를 정렬하는 방법은 여러 가지가 있습니다. 이 수업에서는 JavaScript의 내장 `sort` 메서드에 집중하겠습니다.

`sort` 메서드는 배열의 요소를 정렬하는 데 사용하며, 정렬된 배열에 대한 참조를 반환합니다. 요소들은 제자리에서 정렬되기 때문에 복사본이 만들어지지 않습니다.

다음은 `sort` 메서드의 기본 구문입니다:

```js
array.sort(compareFunction);
```

`compareFunction`은 정렬 순서를 정의하는 함수를 지정하는 선택적 매개변수입니다. 숫자를 정렬할 때 비교 함수를 어떻게 사용하는지 나중에 살펴보겠습니다.

첫 번째 예제에서는 무작위 순서로 된 문자열 배열이 있습니다.

```js
const fruits = ["Banana", "Orange", "Apple", "Mango"];
```

목표는 배열을 알파벳 순서로 정렬하는 것입니다. `sort` 배열에서 `fruits` 메서드를 호출하여 이 작업을 할 수 있습니다.

:::interactive_editor

```js
const fruits = ["Banana", "Orange", "Apple", "Mango"];
fruits.sort();

console.log(fruits); // ["Apple", "Banana", "Mango", "Orange"]
```

:::

결과는 `Apple` 과일로 시작하는 알파벳 순서로 정렬된 과일 배열이 될 것입니다.

다음 예제에서는 다음 숫자 배열을 정렬하려고 합니다:

```js
const numbers = [414, 200, 5, 10, 3];
```

이 `sort` 배열에 `numbers` 메서드를 사용하면 예상치 못한 결과가 나옵니다.

:::interactive_editor

```js
const numbers = [414, 200, 5, 10, 3];
numbers.sort();

console.log(numbers); // [10, 200, 3, 414, 5]
```

:::

우리는 `[3, 5, 10, 200, 414]` 결과를 기대했지만 대신 `[10, 200, 3, 414, 5]` 결과가 나왔습니다.

이는 `sort` 메서드가 요소들을 문자열로 변환한 후 UTF-16 코드 단위 수열을 비교하기 때문입니다.

UTF-16 코드 단위는 문자열 내 문자를 나타내는 숫자 값입니다. UTF-16 코드 단위의 예로는 `65`, `66`, `67` 숫자가 있으며, 각각 `A`, `B`, `C` 문자를 나타냅니다.

따라서 `200` 숫자가 `3` 숫자보다 배열에서 앞에 나타나는데, 이는 문자열 `200`이 문자열 `3`보다 UTF-16 코드 단위를 비교할 때 앞서기 때문입니다.

이 문제의 해결책은 `sort` 메서드에 비교 함수를 제공하는 것입니다.

다음은 비교 함수를 사용하여 `numbers` 배열을 정렬하는 예입니다:

:::interactive_editor

```js
const numbers = [414, 200, 5, 10, 3];

numbers.sort((a, b) => a - b);

console.log(numbers); // [3, 5, 10, 200, 414]
```

:::

매개변수 `a`와 `b`는 비교되는 두 요소입니다. 비교 함수는 `a`가 `b`보다 앞에 와야 하면 음수를, `a`가 `b`보다 뒤에 와야 하면 양수를, `a`과 `b`가 같으면 0을 반환해야 합니다.

첫 번째 비교는 `414`와 `200` 숫자 사이에서 이루어집니다. `414 - 200`의 결과는 양수인 `214`입니다. 이는 `414`가 정렬된 배열에서 `200` 뒤에 와야 함을 의미합니다.

다음 비교는 `200`와 `5` 숫자 사이에서 이루어집니다. `200 - 5`의 결과는 양수인 `195`입니다. 이는 `200`가 정렬된 배열에서 `5` 뒤에 와야 함을 의미합니다.

이 과정을 배열의 모든 요소에 반복하면 숫자가 정렬된 배열이 결과로 나옵니다.

프로그래밍에서 데이터를 정렬하는 방법은 더 많지만, JavaScript의 `sort` 메서드는 배열 요소를 정렬해야 할 때 많은 경우에 유용하고 효율적일 수 있습니다.
