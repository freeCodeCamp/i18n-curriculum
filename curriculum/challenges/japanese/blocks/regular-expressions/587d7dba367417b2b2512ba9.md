# --description--

<dfn>先読み</dfn>は、JavaScriptに文字列の先を見てさらに先のパターンを確認するよう指示するパターンです。これは同じ文字列内で複数のパターンを検索したい場合に役立ちます。

先読みには、<dfn>ポジティブ先読み</dfn>と<dfn>ネガティブ先読み</dfn>の2種類があります。

ポジティブ先読みは、検索パターン内の要素が存在することを確認しますが、実際にはマッチしません。ポジティブ先読みは`(?=...)`のように使い、`...`がマッチしない必須部分です。

一方、ネガティブ先読みは検索パターン内の要素が存在しないことを確認します。ネガティブ先読みは`(?!...)`のように使い、`...`が存在してほしくないパターンです。ネガティブ先読み部分が存在しなければ、残りのパターンが返されます。

先読みは少し分かりにくいですが、いくつか例を見てみましょう。

```js
let quit = "qu";
let noquit = "qt";
let quRegex= /q(?=u)/;
let qRegex = /q(?!u)/;
quit.match(quRegex);
noquit.match(qRegex);
```

これら両方の`match`呼び出しは`["q"]`を返します。

先読みのより実用的な使い方は、1つの文字列内で2つ以上のパターンをチェックすることです。以下は3～6文字で少なくとも1つ数字が含まれる（単純な）パスワードチェッカーの例です。

```js
let password = "abc123";
let checkPass = /(?=\w{3,6})(?=\D*\d)/;
checkPass.test(password);
```

# --instructions--

`pwRegex`で先読みを使い、5文字より長く、連続した2つの数字を含むパスワードにマッチさせてください。

# --hints--

正規表現には2つのポジティブ`lookaheads`を使う必要があります。

```js
assert(pwRegex.source.match(/\(\?=.*?\)\(\?=.*?\)/) !== null);
```

正規表現は文字列`astronaut`にマッチしてはいけません。

```js
pwRegex.lastIndex = 0;
assert(!pwRegex.test('astronaut'));
```

正規表現は文字列`banan1`にマッチしてはいけません。

```js
pwRegex.lastIndex = 0;
assert(!pwRegex.test('banan1'));
```

正規表現は文字列`bana12`にマッチする必要があります。

```js
pwRegex.lastIndex = 0;
assert(pwRegex.test('bana12'));
```

正規表現は文字列`abc123`にマッチする必要があります。

```js
pwRegex.lastIndex = 0;
assert(pwRegex.test('abc123'));
```

正規表現は文字列`12345`にマッチしてはいけません。

```js
pwRegex.lastIndex = 0;
assert(!pwRegex.test('12345'));
```

正規表現は文字列`8pass99`にマッチする必要があります。

```js
pwRegex.lastIndex = 0;
assert(pwRegex.test('8pass99'));
```

```js
pwRegex.lastIndex = 0;
```

