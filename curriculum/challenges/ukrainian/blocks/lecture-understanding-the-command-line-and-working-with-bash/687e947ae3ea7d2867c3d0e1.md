# --description--

Давайте дізнаємося про опції та прапорці команд.

У попередньому уроці ви вивчали, як передавати аргументи командам, наприклад `touch readme.md`. Опції, або прапорці, — це спеціальні аргументи, які можна передати команді, щоб вплинути на її поведінку. Ці два терміни використовуються взаємозамінно, хоча "прапорці" частіше застосовуються для опцій, які виконують функцію вмикання або вимикання.

Опції зазвичай мають префікс з одного або двох дефісів (`-`), що допомагає візуально відрізнити опцію від аргументу. Спочатку розглянемо синтаксис з двома дефісами, або "довгу форму".

Багато застосунків, таких як `ls`, приймають прапорець `--version`, який виводить поточну версію застосунку в термінал замість запуску команд застосунку.

Також багато застосунків пропонують прапорець `--help`, який виводить інструкції щодо використання застосунку.

Синтаксис з одним дефісом, або "коротка форма", зазвичай використовує опції у вигляді однієї літери. Наприклад, прапорець `-a` з `ls` виводить список усіх файлів, включно з прихованими файлами, які починаються з крапки (`.`), як `.env`.

Перевага таких коротких опцій у тому, що можна об’єднувати кілька прапорців разом. Замість `ls --all --human-readable --size` можна використати всі літери одночасно з `ls -ahs`.

Деякі опції очікують, що їм передадуть значення.

При використанні довгої форми опцій зазвичай потрібно застосовувати знак «дорівнює». У цьому синтаксисі значення безпосередньо приєднується до опції через знак «дорівнює» (`=`). Ось приклад зміни поведінки `ls`, щоб включати або виключати кольори:

```bash
ls --color=never
```

При використанні короткої форми опцій значення зазвичай відокремлюють пробілом. Наприклад, ось довга та коротка форма опцій для встановлення `width` результату `ls`:

```bash
ls -w 50
```

```bash
ls --width=50
```

Зверніть увагу, що коротка форма використовує `-w 50`, а довга форма — `--width=50`. Ця відмінність важлива, щоб не передати значення опції як позиційний аргумент.

Але якщо ви сумніваєтеся, пам’ятайте, що зазвичай можна використати прапорець `--help`, щоб побачити очікуваний синтаксис опцій!
