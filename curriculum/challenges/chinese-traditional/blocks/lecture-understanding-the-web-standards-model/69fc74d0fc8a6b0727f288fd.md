# --description--

開放的網路不會自動運作。每當瀏覽器呈現頁面、播放影片或繪製 3D 場景時，它都是依照多年來由工程師和公司代表在工作小組中制定的指令來執行。

這些指令稱為網路標準。而負責制定這些標準的團體稱為標準組織。你可以把它們想像成網際網路的立法機關。

目前在網路標準工作中扮演主要角色的組織包括 **W3C**、**WHATWG**、**ECMA (TC39)** 和 **Khronos Group**。它們各自管理堆疊的不同層級，且運作方式也略有不同。

**Web 超文字應用技術工作小組**（WHATWG）維護 HTML 和 DOM，作為一個持續更新的「活標準」。它不發佈版本，而是即時演進，由一個擁有強大編輯權限的小組負責編輯。決策權掌握在主要瀏覽器引擎團隊，如 Chromium、Gecko 和 WebKit。

**萬維網聯盟**（W3C）是網路的主要標準組織，成員涵蓋瀏覽器廠商、政府和研究機構。

W3C 監督 CSS、SVG、無障礙指引和眾多網路 API。它透過一個緩慢且以共識為基礎的流程運作，提案需經過多階段審查才能成為正式標準。

**Ecma International**（前身為歐洲電腦製造商協會，ECMA）是一個通用標準組織，最初專注於計算和通訊技術。如今，它正式發佈 ECMAScript 標準，JavaScript 的實作即基於此標準。語言標準的技術開發由 Ecma 的技術委員會 39（**TC39**）負責，而 Ecma International 則提供組織架構、發佈流程和標準治理。

該委員會約每兩個月召開一次會議，成員來自 Google、Apple、Mozilla、Microsoft 等。新語言特性經過五階段提案流程，從第 0 階段的草案到第 4 階段的完整規範與實作。像 async/await、optional chaining 和 ES 模組等特性，都是經過此流程才納入 JavaScript 語言。

**Khronos Group** 是一個由成員資助的聯盟，專注於圖形、運算和媒體標準。成員包括 NVIDIA、AMD 等 GPU 製造商代表，以及瀏覽器廠商。對網路而言，其最重要的貢獻是 WebGL 和 WebGL 2，為瀏覽器帶來 GPU 加速的 3D 呈現。
