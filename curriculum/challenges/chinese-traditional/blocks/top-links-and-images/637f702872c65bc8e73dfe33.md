# --description--


一般來說，你會創建兩種聯結：

- 指向網際網路上其他網站頁面的聯結

- 指向你自己網站上頁面的聯結


## 絕對聯結
指向網際網路上其他網站頁面的連結稱為絕對連結。典型的絕對連結會由以下零件組成：`protocol://domain/path`。絕對連結將始終包含目的地的協定和網域。

你已經看過絕對連結的實際應用。你先前為 The Odin Project 的 About 頁面創建的連結是絕對連結，因為它包含了協定和網域。

`https://www.theodinproject.com/about`

## 相對聯結
你自己網站內其他頁面的聯結稱為相對聯結。相對聯結不包含網域名稱，因為它是同一網站上的另一個頁面，假設網域名稱會與你創建聯結的頁面相同。

相對連結只包含相對於你正在建立連結的頁面的其他頁面的檔案路徑。這相當抽象，讓我們透過範例來實際看看。

在 `odin-links-and-images` 目錄中，創建另一個名為 `about.html` 的 HTML 檔案，並將以下程式碼貼入其中：

```html
<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8">
    <title>Odin Links and Images</title>
  </head>

  <body>
    <h1>About Page</h1>
  </body>
</html>
```

回到 `index` 頁面，為你的 `about` 頁面添加以下錨點元素以創建連結：

```html
<body>
  <h1>Homepage</h1>
	<a href="https://www.theodinproject.com/about">click me</a>

	<a href="about.html">About</a>
</body>
```

在瀏覽器中開啟 `index.html` 檔案並點擊 about 聯結，以確保所有部分正確連接。點擊該聯結應該會前往你剛剛創建的 `about` 頁面。

這是可行的，因為 `index` 和 `about` 頁面位於相同的目錄。這表示你可以直接使用它的名稱（`about.html`）作為聯結的 `href` 值。

但你通常會想要更好地組織你的網站目錄。通常你只會在根目錄放置 `index.html`，而所有其他 HTML 檔案則放在它們自己的目錄中。

在 `odin-links-and-images` 目錄中創建一個名為 `pages` 的目錄，並將 `about.html` 檔案移動到這個新目錄中。

在瀏覽器中重新整理 `index` 頁面，然後點擊 `about` 聯結。它現在會壞掉。這是因為 `about` 頁面檔案的位置已經改變。

為了解決這個問題，你只需要更新 `about` 聯結的 `href` 值，將 `pages/` 目錄包含進去，因為相對於 `index.html` 檔案，`about.html` 檔案的新位置就在該目錄中。

```html
<body>
  <h1>Homepage</h1>
  <a href="pages/about.html">About</a>
</body>
```

在瀏覽器中重新整理 `index` 頁面，然後再試著點擊 `about` 聯結，現在它應該已恢復正常運作。

在許多情況下，這樣做通常沒問題；然而，你仍可能遇到此方法帶來的意外問題。在聯結前加上 `./` 在大多數情況下可以避免這類問題。透過添加 `./`，你是在告訴你的程式碼應該從**目前**目錄開始尋找該檔案／目錄。

```html
<body>
  <h1>Homepage</h1>
  <a href="./pages/about.html">About</a>
</body>
```

絕對聯結和相對聯結有什麼差別？

## --answers--

絕對聯結是指向當前網站上另一個頁面的聯結。相對聯結是指向另一個網站的聯結。
