# --description--

Sasa uko tayari kuchukua hatua zako za kwanza katika ulimwengu wa Node.js kwa kusakinisha kwenye kompyuta yako. Lakini kwanza, kabla hujasakinisha Node.js, tunapendekeza usakinishe chombo kinachoitwa NVM (Node Version Manager). Kama jina linavyosema, NVM ni chombo cha kusimamia matoleo mbalimbali ya Node.js kwenye kompyuta moja.

Kuweka chombo hiki tangu mwanzo kutakusaidia siku zijazo kwa sababu unaweza kuhitaji kubadilisha kati ya matoleo tofauti ya Node.js kwa miradi tofauti. Kwa NVM, unaweza kufanya hivyo kwa urahisi sana.

Tuchunguze jinsi unavyoweza kusakinisha NVM kwenye macOS/Linux na Windows, na jinsi unavyoweza kuitumia kusakinisha Node.js.

## Jinsi ya kusakinisha NVM kwenye macOS/Linux

Kwenye macOS na Linux, unaweza kusakinisha NVM moja kwa moja kupitia script ya shell.

Lakini kwa macOS, kumbuka kuwa nyaraka rasmi za NVM zinaeleza kwamba “unahitaji kusakinisha zana za mstari wa amri za Xcode kwa mikono kabla ya kuendesha script ya usakinishaji, vinginevyo, itashindwa.”

Kwenye macOS, unaweza kusakinisha zana za mstari wa amri za Xcode kwa kuendesha amri hii kwenye terminal yako:

```bash
xcode-select --install
```

Mara usakinishaji ukimalizika, unaweza kuangalia nambari ya toleo na kuthibitisha usakinishaji kwa amri hii:

```bash
xcode-select --version
```

Kisha, ni wakati wa kusakinisha NVM. Unaweza kupakua na kuendesha script ya usakinishaji ya NVM kwa kuendesha moja ya amri hizi:

```bash
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.3/install.sh | bash
```

```bash
wget -qO- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.3/install.sh | bash
```

Ili kupata toleo la hivi karibuni la NVM wakati unaposoma makala hii, tafadhali rejea nyaraka rasmi za NVM: `https://github.com/nvm-sh/nvm?tab=readme-ov-file#install--update-script`. Huko, utapata amri hizi zikiwa na nambari ya toleo la hivi karibuni.

Huenda ukahitaji kuanzisha upya terminal yako baada ya mchakato kukamilika ili kupakia amri mpya na usanidi mpya.

Kisha, endesha amri hii kuthibitisha kuwa NVM ilisakinishwa kwa usahihi na kuangalia toleo lako la sasa:

```bash
nvm -v
```

Unapaswa kuona toleo lako la NVM kwenye matokeo.

## Jinsi ya kusakinisha Node.js kwa kutumia NVM kwenye macOS/Linux

Sasa umeweka NVM kwenye macOS/Linux, unaweza kusakinisha Node.js.

Ili kusakinisha toleo la hivi karibuni la Long-Term Support (LTS) la Node.js, endesha amri hii:

```bash
nvm install --lts
```

Angalia bendera ya `--lts`. Bendera hii inaonyesha kuwa unataka kusakinisha toleo la sasa la Long-Term Support (LTS) la Node.js. Toleo hili kawaida linapendekezwa kwa sababu linaweka kipaumbele utulivu, kuaminika, na usalama zaidi kuliko vipengele vipya au vya majaribio. Limejaribiwa kwa kina na liko tayari kwa uzalishaji. Pia linahakikishiwa kuwa na vipindi virefu vya msaada kwa makini kwenye marekebisho ya hitilafu na vifuniko vya usalama.

Mbali na hilo, unaweza kusakinisha toleo maalum la Node.js kwa kubainisha nambari ya toleo kwenye amri:

```bash
nvm install <version>
```

Kwa mfano, hivi ndivyo unavyoweza kusakinisha Node.js v20:

```bash
nvm install 20
```

Mara baada ya kumaliza kusakinisha toleo la Node.js, unaweza kubadilisha kwenda toleo hilo kwa amri hii:

```bash
nvm use <version>
```

Kwa mfano, hivi ndivyo unavyobadilisha kwenda toleo 20:

```bash
nvm use 20
```

Mwishowe, unaweza kuangalia toleo la sasa la Node.js kwa amri hii:

```bash
node -v
```

## Jinsi ya kusakinisha NVM kwenye Windows

Sasa tuchunguze jinsi unavyoweza kusakinisha na kutumia NVM kwenye Windows.

Matoleo rasmi ya NVM yanaunga mkono Windows katika hali maalum tu. Ikiwa umeweka Windows Subsystem for Linux (WSL) kwenye kifaa chako, unaweza kufuata mwongozo wa macOS/Linux ulio hapo juu kusakinisha na kutumia NVM.

Hata hivyo, kwa Windows yenyewe, kuna mbadala kadhaa. Moja yao ni `nvm-windows`, chombo cha usimamizi wa matoleo ya Node.js kwa Windows. Huu ni mradi tofauti kabisa na mradi rasmi wa NVM. Lakini madhumuni ya zana hizi ni sawa — kukusaidia kufanya kazi na matoleo mengi ya Node.js kwenye kifaa kimoja.

Kabla ya kusakinisha `nvm-windows`, unapaswa kujua kuwa hifadhi rasmi inapendekeza kuondoa usakinishaji wowote wa awali wa Node kabla ya kusakinisha NVM kwa Windows. Ikiwa hii ni mara yako ya kwanza kusakinisha Node.js, utakuwa sawa. Ikiwa tayari una matoleo mengine ya Node.js yaliyosakinishwa, inashauriwa kuyaondoa kwanza.

Ili kusakinisha `nvm-windows`:

- Nenda kwenye sehemu ya Releases ya hifadhi rasmi: `https://github.com/coreybutler/nvm-windows/releases`.
- Pakua msakinishaji wa hivi karibuni.
- Fuata maelekezo.

Baada ya kukamilisha usakinishaji, huenda ukahitaji kuanzisha upya terminal yako ikiwa amri ya `nvm` haipatikani mara moja.

Unaweza kuangalia kama usakinishaji ulifanikiwa na kuona toleo lako la sasa la `nvm-windows` kwa kuendesha amri hii:

```powershell
nvm -v
```

## Jinsi ya kusakinisha Node.js kwa kutumia NVM kwa Windows

Sasa umeweka `nvm-windows` kwenye Windows, uko tayari kusakinisha Node.js.

Ili kusakinisha toleo la hivi karibuni la Long-Term Support (LTS) la Node.js, endesha amri hii:

```powershell
nvm install lts
```

Toleo la Long-Term Support (LTS) kawaida linapendekezwa, lakini unaweza kusakinisha toleo tofauti kwa kubainisha nambari ya toleo kwenye amri:

```powershell
nvm install <version>
```

Kwa mfano:

```powershell
nvm install 20
```

Mara usakinishaji ukimalizika, unaweza kubadilisha kwenda toleo hilo la Node.js kwa amri hii:

```powershell
nvm use <version>
```

Kwa mfano:

```powershell
nvm use 20
```

Pia unaweza kuangalia toleo lako la sasa la Node.js kwa amri hii:

```bash
node -v
```

## Amri za msingi

Sasa unajua jinsi ya kusakinisha NVM na Node.js, tuchunguze baadhi ya amri za msingi ambazo zitakusaidia unapotumia Node.js na NVM.

Pia, ingawa amri hizi zote zitafanya kazi kwa NVM iliyosakinishwa kwenye macOS/Linux, pamoja na WSL, huenda zikafanya kazi tofauti au zisifanye kazi kabisa kwa `nvm-windows`. Tutaacha amri mbadala za `nvm-windows` pale inapowezekana.

### Jinsi ya kubadilisha sera ya utekelezaji ya Powershell

Ikiwa unatumia Windows na unafuata pamoja na Powershell na `nvm-windows`, huenda ukahitaji kubadilisha sera ya utekelezaji ya Powershell. Sera ya utekelezaji ya chaguo-msingi kwa Powershell ni `Restricted`, ambayo ni hatua ya usalama kuzuia script zisifanye kazi.

Kwanza, endesha Powershell kama msimamizi kwa kubofya kulia Powershell, kisha bofya “Run as Administrator“.

Kisha, ili kuona sera ya sasa ya utekelezaji ya kikao cha sasa cha Powershell, endesha amri ifuatayo:

```powershell
Get-ExecutionPolicy
```

Mwishowe, kuweka sera ya utekelezaji ya Powershell kuwa `RemoteSigned`, ambayo inaruhusu script za ndani na `npm` kufanya kazi, tumia amri ifuatayo:

```powershell
Set-ExecutionPolicy RemoteSigned
```

Mara baada ya kuendesha amri hapo juu, unaweza kufunga dirisha la msimamizi wa Powershell, na kufungua dirisha la Powershell lisilo la msimamizi.

### Orodhesha matoleo yote

Ili kuorodhesha matoleo yote ya Node.js yaliyosakinishwa na kuangalia toleo linalotumika sasa, endesha amri hii:

```bash
nvm ls
```

### Weka toleo la chaguo-msingi la Node.js (NVM kwa macOS/Linux pekee)

Amri hii inaweka toleo la chaguo-msingi la Node.js litakalotumika kwa vikao vipya vya terminal:

```bash
nvm alias default <version>
```

Kwa mfano:

```bash
nvm alias default 22.20.0
```

Pia unaweza kuunda majina ya utani maalum kuweka jina maalum kwa toleo:

```bash
nvm alias <new_name> <version>
```

Kwa mfano:

```bash
nvm alias new-api 18.20.8
```

Baada ya kuweka jina la utani, unaweza kutumia jina hili jipya kubadilisha kwenda toleo hilo:

```bash
nvm use <new_name>
```

Kwa mfano:

```bash
nvm use new-api
```

### Ondoa toleo la Node.js

Ili kuondoa toleo la Node.js, endesha amri hii:

```bash
nvm uninstall <version>
```

Kwa mfano:

```bash
nvm uninstall 20.19.5
```

Kwa NVM kwa macOS/Linux, unaweza kuondoa toleo la Node.js kwa kuendesha amri ya `uninstall` na toleo kuu kama `22` au `20`. Kwa mfano:

```bash
nvm uninstall 20
```

Ikiwa una matoleo mengi ya toleo kuu hilo yaliyosakinishwa kama `20.14.0` na `20.12.2`, toleo la hivi karibuni litaondolewa kwanza. Katika kesi hii, kuendesha `nvm uninstall 20` kutaondoa `20.14.0` kwanza.

### Angalia toleo lako la Node.js

Ili kuangalia toleo lako la sasa la Node.js, unaweza kutumia ifuatayo:

```bash
node -v
```

### Angalia toleo lako la npm

`npm` ni meneja wa pakiti kwa Node.js. Unaweza kuitumia kusakinisha, kuchapisha, na kusimamia pakiti za programu. Inasakinishwa moja kwa moja unaposakinisha Node.js.

Ili kuangalia toleo lako la sasa la `npm`, endesha amri hii:

```bash
npm -v
```

### Endesha faili la JavaScript

Amri hii inaendesha faili la JavaScript ulilobainisha katika runtime ya sasa ya Node.js:

```bash
node <file.js>
```

Kwa mfano:

```bash
node app.js
```

### Unda mradi wa Node.js

Amri hii inaunda mradi mpya wa Node.js kwa kuunda faili la `package.json` linalofuatilia maelezo na utegemezi wake.

```bash
npm init
```

Unapoendesha amri ya `npm init`, itakuongoza kupitia mwongozo wa kiolesura cha mstari wa amri kwa kuanzisha faili la `package.json`. Itakuuliza maswali kuhusu jina la mradi, maelezo, toleo, n.k. Ikiwa unataka kuanza na faili la `package.json` la chaguo-msingi, unaweza kutumia amri ya `npm init -y` au `npm init --yes`.

### Sakinisha pakiti

Ili kusakinisha pakiti maalum, endesha:

```bash
npm install <package>
```

Kwa mfano:

```bash
npm install express
```

### Sakinisha utegemezi

Ili kusakinisha utegemezi wote uliorodheshwa katika faili la `package.json` la mradi, endesha:

```bash
npm install
```
