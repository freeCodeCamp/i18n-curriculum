# --description--

In Express, l'oggetto `res` (response) viene usato per inviare risposte al client dopo aver elaborato le richieste in arrivo. Express offre diversi metodi integrati per inviare tipi diversi di risposte. Per esempio, puoi inviare testo semplice, dati JSON, reindirizzare un utente a un altro URL o renderizzare un template.

Uno dei metodi di risposta più comunemente usati è `res.send()`:

```javascript
app.get("/text", (req, res) => {
  res.send("Hello, World!");
});
```

Il metodo `res.send()` invia una risposta al client. Può inviare testo semplice, HTML, JSON o anche dati binari. In questo esempio, quando un client visita `/text`, il server risponde con il testo `Hello, World!`.

Un altro metodo di risposta comunemente usato è `res.json()`.

Questo metodo invia una risposta JSON al client. È particolarmente utile quando si costruiscono API, dove spesso è necessario restituire dati strutturati come oggetti o array. Express converte automaticamente un oggetto JavaScript in una stringa formattata JSON:

```javascript
app.get("/user", (req, res) => {
  res.json({ name: "John Doe", age: 30 });
});
```

Se un client invia una richiesta a `/user`, il server risponde con quanto segue:

```json
{
  "name": "John Doe",
  "age": 30
}
```

Quando hai bisogno di controllare il codice di stato HTTP di una risposta, puoi usare `res.status()`. Questo metodo imposta il codice di stato ed è tipicamente concatenato con un altro metodo di risposta come `res.send()` o `res.json()`:

```javascript
app.get("/notfound", (req, res) => {
  res.status(404).send("Not Found");
});
```

Qui, il server imposta il codice di stato a `404` e invia il messaggio `Not Found` al client.

Se vuoi inviare l’utente a un URL diverso, puoi usare `res.redirect()`:

```javascript
app.get("/oldpage", (req, res) => {
  res.redirect("/newpage");
});
```

Quando un client visita `/oldpage`, viene automaticamente reindirizzato a `/newpage`.

Infine, `res.render()` si usa quando si lavora con motori di template come Pug, EJS o Handlebars. Renderizza un template di vista e invia l’HTML risultante al client:

```javascript
app.get("/profile", (req, res) => {
  res.render("profile", { username: "john_doe" });
});
```

In questo esempio, il server renderizza una vista `profile` e le passa una variabile `username`, che può poi essere mostrata nell’HTML.

Questi metodi di risposta ti permettono di restituire diversi tipi di dati in base alla natura della richiesta. Che tu stia inviando testo, dati JSON o reindirizzando il client, Express ha un set flessibile di strumenti per gestirlo.
