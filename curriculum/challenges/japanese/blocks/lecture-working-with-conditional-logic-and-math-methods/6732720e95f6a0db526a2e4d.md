# --interactive--

バイナリ論理演算子は、2つの式を評価し、それらの真偽値に基づいて結果を返すのに役立ちます。ここでは、最も一般的な3つのバイナリ論理演算子、論理AND、論理OR、そしてヌリッシュ合体演算子を見ていきましょう。

論理AND演算子は、二重アンパサンド（`&&`）で表されます。両方のオペランドが真かどうかをチェックし、結果を返します。両方のオペランドが真であれば、右側の2番目の値を返します。

:::interactive_editor

```js
const result = true && 'hello';

console.log(result); // hello
```

:::

上の例では、両方のオペランドが`hello`なので、テキスト`true`がコンソールに表示されます。どちらかのオペランドが偽値の場合は、その偽値を返します。

:::interactive_editor

```js
const result = 0 && 3;

console.log(result); // 0
```

:::

`0`は偽値なので、数値`0`がコンソールに表示されます。両方のオペランドが偽値の場合は、最初の偽値を返します。

:::interactive_editor

```js
const result = false && 0;

console.log(result); // false
```

:::

`false`は偽値なので、`false`がコンソールに表示されます。論理AND演算子は、複数の条件をチェックし、すべてが真であることを確認してから処理を進めたい場合に便利です。以下はその例です。

:::interactive_editor

```js
if (2 < 3 && 3 < 4) {
 console.log('The if block runs'); 
} else {
 console.log('The else block runs');
} 
```

:::

条件では、`2`が`3`より小さく、かつ`3`が`4`より小さいので、文`The if block runs`がコンソールに表示されます。

論理OR演算子は、少なくとも1つのオペランドが真かどうかをチェックします。最初のオペランドが真であれば、その値を返します。

:::interactive_editor

```js
const result = 'This is truthy' || false;

console.log(result); // This is truthy
```

:::

最初のオペランドが偽値で、2番目が真値の場合は、2番目の値がコンソールに表示されます。

:::interactive_editor

```js
const result = 0 || 'This is truthy';

console.log(result); // This is truthy
```

:::

論理OR演算子は、次のような`if/else`文で使われることがよくあります。

:::interactive_editor

```js
let userInput;

if (userInput || 'Guest') {
 console.log('A user is present');
} else {
 console.log('No user detected');
}
```

:::

`userInput`変数に値が代入されていないため、現在は`undefined`です。`if`文の条件は、`userInput`変数か文字列`Guest`のどちらかが真かどうかをチェックします。このようなブール文脈では文字列`Guest`が真なので、文字列`A user is present`がコンソールに表示されます。

ヌリッシュ合体演算子は、論理ORや論理ANDよりも高度です。二重疑問符（`??`）で表され、最初の値が`null`または`undefined`の場合にのみ値を返したいシナリオで役立ちます。ヌリッシュ合体演算子を使った例を見てみましょう。

:::interactive_editor

```ts
const result = null ?? 'default';

console.log(result); // default
```

:::

`null`はヌリッシュ値なので、文字列`default`がコンソールに表示されます。ヌリッシュ合体演算子は、`null`や`undefined`だけがフォールバックやデフォルト値をトリガーすべき場合に非常に便利です。以下はユーザーの設定を扱う例です。

:::interactive_editor

```ts
const userSettings = {
 theme: null,
 volume: 0,
 notifications: false,
};

let theme = userSettings.theme ?? 'light';
console.log(theme); // light
```

:::

上の例では、`userSettings`というオブジェクトがあり、`theme`、`volume`、`notifications`のプロパティを含んでいます。ドット記法で`theme`にアクセスしており、`userSettings.theme`のように書きます。オブジェクトの扱い方は今後のレッスンで詳しく学びます。ユーザーの`theme`が現在`null`に設定されているので、文字列`light`がコンソールに表示されます。
