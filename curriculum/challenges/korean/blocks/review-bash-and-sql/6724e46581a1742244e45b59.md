## 데이터베이스 정규화

데이터 중복을 줄이고 무결성을 향상하기 위해 관계형 데이터베이스를 체계적으로 구성하는 과정입니다.

이점은 다음과 같습니다:

- 중복 데이터를 최소화하여 저장 공간을 절약하고 불일치를 줄입니다.
- 기본 키와 외래 키를 사용해 데이터 무결성을 강화합니다.
- 데이터베이스를 더 쉽게 유지 관리하고 이해할 수 있게 합니다.

### 정규형

- **1NF (제1정규형)**
  - 각 셀은 단일(원자적) 값을 가집니다.
  - 각 레코드는 고유하며(기본 키로 강제) 중복되지 않습니다.
  - 행과 열의 순서는 중요하지 않습니다.
  - 예: `students` 표에 여러 전화번호가 있을 때 이를 별도의 `student_phones` 표로 분리합니다.

- **2NF (제2정규형)**
  - 1NF 조건을 충족합니다.
  - **부분 함수 종속성**이 없습니다: 모든 비키 속성은 복합 기본 키 전체에 의존해야 합니다.
  - 예: `orders` 표를 `order_header`과 `order_items`로 나누어 일부 키에만 의존하는 속성을 제거합니다.

- **3NF (제3정규형)**
  - 2NF 조건을 충족합니다.
  - **이행 함수 종속성**이 없습니다: 비키 속성이 다른 비키 속성에 의존하지 않습니다.
  - 예: `city_postal_code`를 모든 주문에 저장하는 대신 `cities` 표로 옮깁니다.

- **BCNF (보이스-코드 정규형)**
  - 3NF 조건을 충족합니다.
  - 모든 결정자(함수 종속성의 좌변)는 슈퍼키여야 합니다.

**팁**: 무결성과 성능의 균형을 위해 대부분 설계에서 3NF를 목표로 하세요.

## 주요 SQL 개념

- SQL은 관계형 데이터베이스와 소통하기 위한 구조화된 쿼리 언어입니다.
- **기본 명령어** → `SELECT`, `INSERT`, `UPDATE`, `DELETE`, `CREATE TABLE`, `ALTER TABLE` 등
- `Joins` → 여러 표에서 데이터를 결합합니다 (`INNER JOIN`, `LEFT JOIN`, `RIGHT JOIN`, `FULL JOIN`).

## Bash에서 SQL 명령 실행하기

PostgreSQL용 `psql` 명령줄 클라이언트나 다른 데이터베이스용 유사 도구를 사용해 명령줄에서 직접 SQL 명령을 실행할 수 있습니다.

예를 들어 PostgreSQL에서 SQL 파일을 실행하려면:

```bash
psql -U username -d database_name -c "SELECT * FROM students;"
```

MySQL 명령도 직접 실행할 수 있습니다:

```bash
mysql -u username -p database_name -e "SELECT * FROM students;"
```

### 파일에서 SQL 실행하기

```bash
# PostgreSQL
psql -U username -d database_name -f script.sql

# MySQL
mysql -u username -p database_name < script.sql
```

### Bash 스크립트에 SQL 임베드하기

```bash
#!/bin/bash
DB_USER="school_admin"
DB_NAME="school"

# Insert student data
psql -U "$DB_USER" -d "$DB_NAME" -c \
"INSERT INTO students (name, age, major) VALUES ('Alice', 20, 'CS');"
```

### SQL에서 변수 사용하기

```bash
#!/bin/bash
DB_USER="school_admin"
DB_NAME="school"
STUDENT_NAME="Bob"
AGE=21

psql -U "$DB_USER" -d "$DB_NAME" -c \
"INSERT INTO students (name, age) VALUES ('$STUDENT_NAME', $AGE);"
```

## Bash에서 SQL 쿼리 결과 가져오기 및 사용하기

`psql`를 통해 SQL 쿼리를 실행하면 반환된 값을 Bash 스크립트에서 **캡처**하고 **처리**할 수 있습니다.

### 단일 값 캡처하기

```bash
#!/bin/bash
DB_USER="school_admin"
DB_NAME="school"

# Get total student count
STUDENT_COUNT=$(psql -U "$DB_USER" -d "$DB_NAME" -t -A -c \
"SELECT COUNT(*) FROM students;")

echo "Total students: $STUDENT_COUNT"
```

출력 → 42

### 여러 열 가져오기

```bash
#!/bin/bash
DB_USER="school_admin"
DB_NAME="school"

# Get top 3 students' names and ages
RESULTS=$(psql -U "$DB_USER" -d "$DB_NAME" -t -A -F"," -c \
"SELECT name, age FROM students LIMIT 3;")

echo "Top 3 students:"
echo "$RESULTS"
```

출력

```bash
Alice,20
Bob,21
Charlie,22
```

### 쿼리 결과 반복 처리하기

```bash
#!/bin/bash
DB_USER="school_admin"
DB_NAME="school"

# Get student names and majors
psql -U "$DB_USER" -d "$DB_NAME" -t -A -F"," -c \
"SELECT name, major FROM students;" | while IFS="," read -r name major
do
    echo "Student: $name | Major: $major"
done
```

출력 형태

```bash
Student: Alice | Major: CS
Student: Bob   | Major: Math
Student: Carol | Major: Physics
```

## SQL 인젝션

입력 필드에 악의적인 SQL 코드를 삽입해 데이터베이스를 조작하는 웹 보안 취약점입니다.

이로 인해 다음과 같은 위험한 행동이 발생할 수 있습니다:

- 인증 우회
- 민감한 데이터 탈취
- 레코드 수정 또는 삭제

SQL 인젝션 공격 예시:

```sql
SELECT * FROM users WHERE username = ' " " OR "1"="1" -- ' AND password = 'anything';
```

이 쿼리는 `OR "1"="1"` 조건이 항상 참이므로 모든 사용자를 반환해 공격자가 로그인 검사를 우회할 수 있습니다.

### SQL 인젝션 방지법

1. **준비된 문장 사용**: SQL 코드와 데이터를 분리해 인젝션을 막습니다. 예시 (Node.js pg 사용):

    ```sql
    client.query('SELECT * FROM users WHERE username = $1 AND password = $2', [username, password]);
    ```

2. **입력 유효성 검사**: 모든 사용자 입력을 정제하고 예상 형식에 맞는지 검사하세요.

3. **최소 권한 원칙**: 애플리케이션에 필요한 최소 권한만 가진 데이터베이스 계정을 사용하세요.

**참고**: 애플리케이션 계정에 관리자 권한을 절대 부여하지 마세요.

## N+1 문제

애플리케이션이 항목 목록(N)을 가져오는 쿼리 하나를 실행한 뒤, 각 항목에 대해 관련 데이터를 가져오는 추가 쿼리를 N번 실행해 총 N+1개의 쿼리가 발생하는 문제입니다.

**왜 문제인가요**

- 각 쿼리는 네트워크와 처리 비용을 추가합니다.
- 여러 개의 작은 쿼리가 하나의 최적화된 쿼리보다 느립니다.

### N+1 패턴 예시

```sql
-- 1: Get list of orders
SELECT * FROM orders LIMIT 50;

-- N: For each order, get customer
SELECT * FROM customers WHERE customer_id = ...;
```

**해결책**: `JOINs` 또는 다른 집합 기반 연산을 사용하세요.

```sql
SELECT 
  orders.order_id,
  orders.product,
  orders.quantity,
  customers.customer_id,
  customers.name,
  customers.email,
  customers.address
FROM orders
JOIN customers 
  ON orders.customer_id = customers.customer_id
WHERE orders.order_id IN (SELECT order_id FROM orders LIMIT 50);
```

관련 데이터를 하나의 쿼리로 결합할 기회를 항상 찾아보세요.
