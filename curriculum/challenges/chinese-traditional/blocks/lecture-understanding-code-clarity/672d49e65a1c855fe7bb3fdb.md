# --description--

程式設計中的註解用於為程式碼提供額外的背景關係或為你自己和他人留下備註。

註解是當你的程式碼執行時，JavaScript 引擎會忽略的文字行或區段。它們純粹是為了閱讀程式碼的人提供利益，無論是你還是其他人。

JavaScript 提供兩種方式為你的程式碼添加註解：單行註解和多行註解。

單行註解是使用兩個斜線（`//`）創建的。以下是一個範例：

```js
// I am a single line comment in JavaScript
```

這種類型的註解非常適合用於簡短的說明或澄清。

這裡是一個來自 freeCodeCamp 課程專案檔案的真實範例：

```js
// This is to allow English to build without having to download the i18n files.
// It fails when trying to resolve the i18n-curriculum path if they don't exist.
const curriculumLocale = process.env.CURRICULUM_LOCALE ?? 'english';
const I18N_CURRICULUM_DIR = path.resolve(
  __dirname,
  curriculumLocale === 'english' ? '.' : 'i18n-curriculum/curriculum'
);
```

不必擔心試圖理解程式碼實際在做什麼，因為它比你目前學過的更進階。相反地，請專注於開發者留下的註解。這個註解提供了這段程式碼存在的重要背景關係。

像這樣的註解對於團隊合作的人來說很重要，有兩個原因：

1. 其他在專案中工作的開發者會理解這段程式碼的目的。

1. 它有助於防止在未諮詢團隊的情況下進行不必要的更改或刪除，這可能導致錯誤或問題。

另一種註解是多行註解。以下是基本語法：

```js
/*
 I am a multiline comment.
 This is helpful for longer explanations.
*/
```

多行註解在你需要在程式碼中撰寫較長的描述、說明或註記時非常有用。

讓我們再看看 freeCodeCamp 課程專案檔案，了解多行註解如何在現實世界中使用。

```js
/* Since there can be more than one way to complete a certification (using the
legacy curriculum or the new one, for instance), we need a certification
field to track which certification this belongs to. */
const dupeCertifications = [
  {
    certification: 'responsive-web-design',
    dupe: '2022/responsive-web-design'
  }
];
const hasDupe = dupeCertifications.find(
  cert => cert.dupe === meta.superBlock
);
```

就像之前一樣，忽略所有的 JavaScript 程式碼，因為它使用了尚未教過的概念。相反地，專注於開發者留下的註解。

團隊中的開發者，甚至是剛加入專案的新貢獻者，都能理解這段 `code` 為何存在，並在著手處理專案此區域之前，擁有完整的背景關係。

雖然註解在程式設計中很有用，但避免過度註解很重要。你不需要對每一行程式碼都加註解，特別是當程式碼簡單且不言自明時。

這裡是一個使用註解來說明明顯事項的範例：

```js
// This code uses the const keyword to create a new variable called price.
// We are assigning the number 10 to the price variable.
const price = 10;
```

在這種情況下，這裡不需要添加任何註解，因為程式碼本身已經很清楚。目標是提升可讀性，而不是用不必要的說明使程式碼變得混亂。

如果你想在學習撰寫程式碼時，為你的個人專案添加註解，那是可以的。但一旦你開始與其他開發者合作處理真實世界的專案，就很重要不要為自我解釋的程式碼使用註解。

同時，也很重要的是不要使用註解來解釋混亂、過於複雜或寫得不好的程式碼。在這些情況下，最好重構或更改你的程式碼，讓其他開發者能更好地理解發生了什麼。

註解是用來記錄你的程式碼並使其更容易理解的強大工具。你應該使用註解來提供背景關係或為自己和他人留下備註。
