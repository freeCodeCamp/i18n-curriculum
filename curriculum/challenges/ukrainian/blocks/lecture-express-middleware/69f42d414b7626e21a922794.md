# --description--

Підпрограмне забезпечення для обробки помилок в Express — це особливий тип підпрограмного забезпечення, який використовується для обробки помилок, що виникають під час циклу запит-відповідь. Воно необхідне для виявлення несподіваних помилок, їх логування та надання відповідних відповідей клієнту.

Як ви бачили в попередньому уроці про обробку помилок, підпрограмне забезпечення для обробки помилок в Express має чотири параметри: `err`, `req`, `res` і `next`. Це підпрограмне забезпечення викликається, коли помилка передається функції `next()`.

Ось нагадування синтаксису ще раз:

```js
app.use((err, req, res, next) => {
  console.error(err.message);
  res.status(500).send("Internal Server Error");
});
```

А ось як це виглядає у базовому застосунку Express:

```js
const express = require('express')
const app = express()

// A simple route that throws an error
app.get('/', (req, res) => {
  throw new Error('Something went wrong!')
})

// Error-handling middleware
app.use((err, req, res, next) => {
  console.error(err.stack); // Logs the error stack
  res.status(500).send('Something went wrong!')
})

app.listen(3000, () => {
  console.log('Server running on http://localhost:3000')
})
```

* Обробник маршруту: обробник маршруту навмисно викликає помилку, щоб змоделювати проблему.
* Підпрограмне забезпечення для обробки помилок: це підпрограмне забезпечення реєструється за допомогою `app.use()`. Воно ловить помилки, передані через `next(err)`, і надсилає клієнту HTTP-відповідь 500 з загальним повідомленням про помилку.
* `err`: це об’єкт помилки, який передається підпрограмному забезпеченню.

Підпрограмне забезпечення для обробки помилок гарантує, що коли у вашому застосунку щось іде не так, користувачі все одно отримують належну відповідь. Важливо завжди розміщувати це підпрограмне забезпечення в кінці стеку підпрограмного забезпечення, щоб ловити будь-які помилки.
