# --interactive--

تعلمت أن الأصناف تشبه المخططات التي يمكنك استخدامها لإنشاء الكائنات.

تحتاج هذه المخططات إلى طريقة عامة للإشارة إلى الكائن المحدد الذي يتم إنشاؤه أو الوصول إليه أو تعديله داخل الشفرة. هذا بالضبط ما تُستخدم له الكلمة المفتاحية `this`. تشير إلى السياق الذي من المفترض أن تُنفذ فيه الشفرة. إذا استخدمتها داخل طريقة، ستكون قيمة `this` مرجعًا إلى الكائن المرتبط بتلك الطريقة. بهذه الطريقة، يمكنك الوصول إلى الخصائص والطرق الخاصة بالكائن داخل الصنف وإعادة استخدام الطريقة على كائنات مختلفة.

على سبيل المثال، هنا لدينا صنف `Dessert`. هذه المرة، نُعرّف خاصية `hasPeanuts`.

```js
class Dessert {
  constructor(hasPeanuts) {
    this.hasPeanuts = hasPeanuts;
  }
}
```

لاحظ أننا نستخدم الكلمة المفتاحية `this` داخل دالة البناء. تذكر أن دالة البناء تُستدعى تلقائيًا عند إنشاء كائن جديد من الصنف.

الكلمة المفتاحية `this` تخبر البرنامج أن خاصية `hasPeanuts` يجب أن تكون خاصية للكائن الذي يتم إنشاؤه حاليًا.

الجانب الأيمن من جملة التعيين يُعيّن قيمة معلمة `hasPeanuts` إلى خاصية `hasPeanuts` الخاصة بالكائن الجديد.

```js
this.hasPeanuts = hasPeanuts;
```

هذا في سياق دالة البناء، لكن يمكنك أيضًا استخدام الكلمة المفتاحية `this` داخل طرق أخرى.

هذا هو صنف `Dessert` مع بعض التغييرات. الآن لدينا خاصيتان: `name` و `price`.

```js
class Dessert {
  constructor(name, price) {
    this.name = name;
    this.price = price;
  }

  showPrice() {
    console.log(`The price of ${this.name} is $${this.price}.`);
  }
}
```

لدينا أيضًا طريقة جديدة `showPrice`. في هذه الطريقة، نطبع رسالة وصفية إلى وحدة التحكم مع اسم وسعر كائن الحلوى.

لاحظ كيف نستخدم الكلمة المفتاحية `this` للوصول إلى خصائص الكائن داخل طريقة `showPrice`.

ما القيمة التي ستأخذها الكلمة المفتاحية `this` في سياق هذه الطريقة؟ ستشير إلى الكائن الذي تُستدعى عليه الطريقة. في هذه الحالة، ننشئ نسخة جديدة من Dessert بالاسم `"Brownie"` والسعر `$5.99`:

```js
const brownie = new Dessert("Brownie", 5.99);
```

إذا استدعينا الطريقة `.showPrice()` على هذا الكائن:

```js
brownie.showPrice();
```

إليك المثال الكامل:

:::interactive_editor

```js
class Dessert {
  constructor(name, price) {
    this.name = name;
    this.price = price;
  }

  showPrice() {
    console.log(`The price of ${this.name} is $${this.price}.`);
  }
}

const brownie = new Dessert("Brownie", 5.99);
brownie.showPrice();
console.log(brownie);
```

:::

يتم استبدال اسم وسعر الكائن الحالي (`brownie`) في السلسلة النصية.

كل هذا بفضل الكلمة المفتاحية `this`، التي تمنحك طريقة عامة للإشارة إلى الكائن الحالي داخل الصنف. هذا يسمح لك بإعادة استخدام الطريقة على كائنات مختلفة.

هذا في سياق طرق الكائنات، لكن الكلمة المفتاحية `this` سيكون لها أيضًا قيمة في الدوال المستقلة والدوال السهمية.

قيمتها داخل دالة مستقلة عادةً ما تشير إلى الكائن العالمي (في الوضع غير الصارم) أو `undefined` (في الوضع الصارم).

قيمة الكلمة المفتاحية `this` تختلف قليلاً في الدوال السهمية. الدوال السهمية ترث قيمة `this` من النطاق المحيط الذي تُعرّف فيه. لا تنشئ ربطًا خاصًا بها لـ `this`. هذا يعني أن قيمة `this` تعتمد على السياق الذي تُعرّف فيه الدالة السهمية، وليس على مكان أو كيفية استدعائها. هذا يجعل الدوال السهمية مفيدة للاستدعاءات الراجعة أو للحفاظ على السياق.

لكن بسبب هذا، لن تشير قيمة `this` إلى الكائن الحالي في طرق الكائنات، لذا يجب أن تضع هذا في اعتبارك عند العمل مع الدوال السهمية.

ينطبق نفس الأمر على الدوال السهمية المعرفة داخل دوال أخرى. سترث قيمة `this` من نطاقها الأب.

الكلمة المفتاحية `this` أداة قوية للوصول إلى الكائنات والتعامل معها. فهم كيفية عملها ضروري لإتقان البرمجة الكائنية التوجه في JavaScript.
