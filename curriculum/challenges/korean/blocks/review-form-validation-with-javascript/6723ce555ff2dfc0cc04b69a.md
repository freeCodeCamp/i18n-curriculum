## JavaScript로 폼 유효성 검사하기

- **제약 조건 유효성 검사 API**: `textarea`와 `input` 같은 특정 HTML 요소는 제약 조건 유효성 검사 API를 노출합니다. 이 API를 사용하면 사용자가 입력한 값이 최소 길이나 패턴 일치 같은 HTML 수준의 유효성 검사를 통과하는지 확인할 수 있습니다.
- **`checkValidity()` 메서드**: 이 메서드는 요소가 모든 HTML 유효성 검사(속성 기반)를 통과하면 `true`를 반환하고, 실패하면 `false`를 반환합니다.

:::interactive_editor

```html
<form>
  <label>
    Email:
    <input
      id="email"
      type="email"
      required
      pattern=".+\.com$"
      placeholder="example@site.com"
    />
  </label>
</form>

<script>
  const input = document.getElementById("email");

  input.addEventListener("input", (e) => {
    if (!e.target.checkValidity()) {
      e.target.setCustomValidity("You must use a .com email.");
    } else {
      e.target.setCustomValidity("");
    }
  });
</script>
```

:::

- **`reportValidity()` 메서드**: 이 메서드는 브라우저에 `input`가 유효하지 않다고 알립니다.

:::interactive_editor

```html
<form>
  <label>
    Email:
    <input
      id="email2"
      type="email"
      required
      pattern=".+\.com$"
      placeholder="example@site.com"
    />
  </label>
</form>

<script>
  const input = document.getElementById("email2");

  input.addEventListener("input", (e) => {
    if (!e.target.checkValidity()) {
      e.target.reportValidity();
    }
  });
</script>
```

:::

- **`validity` 속성**: 이 속성은 `<input>`, `<select>` 같은 폼 컨트롤의 유효성 상태를 가져오거나 설정하는 데 사용되며, 사용자 입력이 해당 요소에 정의된 제약 조건(예: `required` 필드, 패턴 제약, 최대 길이 등)을 충족하는지에 대한 정보를 제공합니다.

:::interactive_editor

```html
<input
  id="age"
  type="number"
  min="18"
  placeholder="Enter age (18+)"
/>

<script>
  const input = document.getElementById("age");

  input.addEventListener("input", (e) => {
    console.log(e.target.validity);
  });
</script>
```

:::

- **`patternMismatch` 속성**: 값이 지정된 정규 표현식 패턴과 일치하지 않으면 `true`가 됩니다.

## `preventDefault()` 메서드

- **정의**: DOM에서 발생하는 모든 이벤트는 기본 동작을 가지고 있습니다. 예를 들어 체크박스의 클릭 이벤트는 기본적으로 체크박스 상태를 전환합니다. 포커스된 버튼에서 스페이스 바를 누르면 버튼이 활성화됩니다. `preventDefault()` 메서드는 이러한 `Event` 객체에서 그 동작이 발생하지 않도록 정지시킵니다.

:::interactive_editor

```html
<form id="form">
  <input type="text" placeholder="Try to submit" />
  <button type="submit">Submit</button>
</form>

<p id="status"></p>

<script>
  const form = document.getElementById("form");
  const status = document.getElementById("status");

  form.addEventListener("submit", (event) => {
    event.preventDefault();
    status.textContent = "Form submission prevented.";
  });
</script>
```

:::

## 폼 제출하기

- **정의**: 폼을 제출하는 방법은 세 가지가 있습니다. 첫 번째는 사용자가 `type` 속성이 `submit`로 설정된 폼 내 버튼을 클릭할 때입니다. 두 번째는 사용자가 폼 내 편집 가능한 `Enter` 필드에서 `input` 키를 누를 때입니다. 세 번째는 JavaScript에서 `requestSubmit()` 요소의 `submit()` 또는 `form` 메서드를 호출할 때입니다.
- **`action` 속성**: `action` 속성은 URL 또는 현재 도메인에 대한 상대 경로를 포함해야 합니다. 이 값은 폼이 데이터를 전송하려는 위치를 결정합니다. `action` 속성을 설정하지 않으면 폼은 현재 페이지의 URL로 데이터를 전송합니다.

```html
<form action="https://freecodecamp.org" method="GET">
  <input
    type="number"
    name="number"
    placeholder="Enter a number"
  />
  <button type="submit">Submit</button>
</form>
```

- **`method` 속성**: 이 속성은 `HTTP` 메서드(예: `GET`, `POST`)를 받아 액션 URL에 요청할 때 해당 메서드를 사용합니다. 메서드가 설정되지 않으면 폼은 기본적으로 `GET` 요청을 사용합니다. 폼 데이터는 `name=value` 쌍으로 URL 인코딩되어 액션 URL에 쿼리 매개변수로 추가됩니다.

```html
<form action="/data" method="POST">
  <input
    type="number"
    id="input"
    placeholder="Enter a number"
    name="number"
  />
  <button type="submit">Submit</button>
</form>
```

- **`enctype` 속성**: `form` 요소는 `enctype` 속성을 받아 데이터에 사용할 인코딩 유형을 나타냅니다. 이 속성은 세 가지 값만 허용합니다: `application/x-www-form-urlencoded` (기본값, URL 인코딩된 폼 본문으로 데이터 전송), `text/plain` (데이터를 평문 형태로, `name=value` 쌍을 줄바꿈으로 구분하여 전송), 그리고 `multipart/form-data` (파일 업로드가 포함된 폼을 처리할 때 사용).
