# --interactive--

HTMLでは、入力のようなフォームコントロールは、`focused`状態、`readonly`状態、または`disabled`状態のような異なる段階や状態にあります。

最初の状態は`default`状態と考えられます。メールアドレス入力のデフォルト状態は空の入力です。これはページに最初にレンダリングされたときのメール入力の見た目です。

:::interactive_editor

```html
<input type="email" name="email" id="email" />
```

:::

ユーザーがフォームコントロールをクリックするか、キーボードのタブキーで選択すると、それは`focused`状態になります。入力が`focused`状態のとき、多くのブラウザは入力の周りに青いハイライトされたボーダーを表示します。しかし、CSSで追加のスタイルを設定することもできます。

プレビューウィンドウの空白部分のどこかをクリックしてから`tab`キーを押して、フォーカス状態を確認してください。プレビューを見るにはインタラクティブエディターを有効にする必要があります。

:::interactive_editor

```html
<input type="email" name="email" id="email" />
```

:::

別のフォーム状態は`disabled`状態です。この状態は、入力がフォーカスされたりアクティブにできないことをユーザーに示します。

インタラクティブエディターを有効にしてメール入力をクリックしてみると、もうフォーカスできないことに気づくでしょう。

:::interactive_editor

```html
<input disabled type="email" name="email" id="email" />
```

:::

`focused`状態と似ていますが、CSSで`disabled`状態に対して追加のスタイルを設定することもできます。

別のフォーム状態のタイプは`readonly`状態です。これは入力のようなフォームコントロールがユーザーによって編集できない場合です。以下はメール入力を読み取り専用に設定する例です。`value`属性は入力フィールド内に表示される値を設定するために使われます。

インタラクティブエディターを有効にしてプレビューウィンドウで`example@email.com`の現在の値を編集しようとすると、それができないことに気づくでしょう。

:::interactive_editor

```html
<input
  readonly
  type="email"
  name="email"
  id="email"
  value="example@email.com"
/>
```

:::

`disabled`状態と`readonly`状態の主な違いは、`readonly`はフォーカスできるのに対し、`disabled`状態はフォーカスできないことです。

異なるフォーム状態を理解することは、エラー処理中に明確なフィードバックと案内を提供してスムーズなユーザー体験を保証するために重要です。
