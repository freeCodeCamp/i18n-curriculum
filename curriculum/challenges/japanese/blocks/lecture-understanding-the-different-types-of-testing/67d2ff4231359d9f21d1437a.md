# --description--

エンドツーエンドテスト（E2Eテスト）は、ユーザーの視点から実際のシナリオをテストします。E2Eテストの例としては、登録とログインのプロセスのテストや、eコマースサイトのチェックアウトのテストがあります。

エンドツーエンドテストの仕組みをよりよく理解するために、freeCodeCampのコードベースから実際の例を見てみましょう。この例では、Microsoftが開発した人気のあるエンドツーエンドテストフレームワークであるPlaywrightを使用しています。

freeCodeCampのコードベース内には、寄付、認定、試験などのテストを保持する`e2e`というディレクトリがあります。そのディレクトリ内には、認証された寄付者サポーターページのテストがある`donate-page-donor.spec.ts`ファイルがあります。

最初のステップは、すでにサポーターであるデモユーザーとして寄付ページに移動することです。

```js
test.beforeEach(async ({ page }) => {
  execSync("node ./tools/scripts/seed/seed-demo-user --set-true isDonating");
  await page.goto("/donate");
});
```

このファイルには複数のテストがあるため、`beforeEach`フックが各テストの前に実行されます。

ユーザーが寄付ページにいると、freeCodeCampをさらに支援する方法の提案とともに感謝のメッセージが表示されます。寄付ページにあるテキストの一部を確認するためのテストの例をいくつか紹介します。

```js
test("should render the donate page correctly", async ({ page }) => {
  await expect(
    page.getByText("Thank you for your continued support")
  ).toBeVisible();

  await expect(
    page.getByText(
      "Your contributions are crucial in creating resources that empower millions of people to learn new skills and support their families."
    )
  ).toBeVisible();

  ...
});
```

また、寄付者がメニューバーにサポーターリンクを持っていることや、サポーターであることを示すためにアバターの周りに特別なスタイライズされたボーダーがあることを確認するテストもあります。

```js
test("The menu should have a supporters link", async ({ page }) => {
  const menuButton = page.getByTestId("header-menu-button");
  const menu = page.getByTestId("header-menu");

  await expect(menuButton).toBeVisible();
  await menuButton.click();

  await expect(menu).toBeVisible();

  await expect(page.getByRole("link", { name: "Supporters" })).toBeVisible();
});

test("The Avatar should have a special border for donors", async ({ page }) => {
  const container = page.locator(".avatar-container");
  await expect(container).toHaveClass("avatar-container gold-border");
});
```

このような詳細なテストを行うことで、アプリケーションのテストカバレッジを単体テストだけでなく広げることができます。エンドツーエンドテストは、アプリケーションが正しく動作し、ユーザーにとって予測可能であることを保証するのにも役立ちます。その他のエンドツーエンドテストツールの例には、Cypress、Selenium、Puppeteerがあります。

エンドツーエンドテストは重要ですが、セットアップ、設計、メンテナンスに時間がかかるためコストが高くなります。E2Eテストは重要なユーザーフローに適しており、単体テストはアプリケーションの小さな単位をテストするのに適しています。
