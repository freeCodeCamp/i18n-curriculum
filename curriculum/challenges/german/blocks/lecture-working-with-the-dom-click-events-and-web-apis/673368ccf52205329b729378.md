# --interactive--

Lernen wir etwas über `innerText`, `textContent` und `innerHTML`.

Dies sind Eigenschaften, auf die Sie in JavaScript zugreifen können, um den Inhalt eines HTML-Elements abzurufen oder zu ändern. Auch wenn sie auf den ersten Blick sehr ähnlich aussehen, gibt es wichtige Unterschiede. Die Wahl der richtigen Eigenschaft hängt von Ihrem konkreten Anwendungsfall ab, also steigen wir ein.

Beginnen wir mit `innerText`.

`innerText` stellt den sichtbaren Textinhalt des HTML-Elements und seiner Nachfahren dar. Diese Eigenschaft enthält keinen versteckten Text oder HTML-Tags, sondern nur den gerenderten Text.

Zum Beispiel sehen Sie hier ein `div`-Element, das zwei Absätze enthält:

```html
<div id="container">
  <p>Hello, World!</p>
  <p>I'm learning JavaScript</p>
</div>
```

Wenn wir in unserem JavaScript-Code eine Referenz auf dieses HTML-Element mit `getElementById()` erhalten, können wir auf die `innerText`-Eigenschaft dieses Elements zugreifen:

:::interactive_editor

```html
<div id="container">
  <p>Hello, World!</p>
  <p>I'm learning JavaScript</p>
</div>
<script src="index.js"></script>
```

```js
const container = document.getElementById("container");
console.log(container.innerText);
```

:::

Dies ist der innere Text dieses Elements:

```md
Hello, World!
I'm learning JavaScript
```

Die Eigenschaft gibt einen String mit dem Text zurück, der innerhalb des Elements enthalten ist, einschließlich des Texts seiner Nachfahren.

Sie sollten wissen, dass `innerText` nur den Text zurückgibt, der zum Zeitpunkt der Abfrage sichtbar ist. Wenn ein Kindknoten verborgen ist, wird dessen Text nicht sichtbar sein.

Hier ist ein Beispiel, bei dem der zweite Absatz verborgen ist:

:::interactive_editor

```html
<div id="container">
  <p>Hello, World!</p>
  <p hidden>I'm learning JavaScript</p>
</div>
```

:::

Wenn wir versuchen, den `innerText` erneut zu protokollieren, enthält die Ausgabe jetzt nicht mehr den Text des zweiten Absatzes:

:::interactive_editor

```html
<div id="container">
  <p>Hello, World!</p>
  <p hidden>I'm learning JavaScript</p>
</div>
<script src="index.js"></script>
```

```js
const container = document.getElementById("container");
console.log(container.innerText);
```

:::

Sie können die `innerText` eines HTML-Elements so setzen, aber dadurch wird der vorhandene Text ersetzt und für jeden Zeilenumbruch ein Zeilenumbruch-Element (`br`) hinzugefügt:

:::interactive_editor

```html
<div id="container">
  <p>Hello, World!</p>
  <p>I'm learning JavaScript</p>
</div>
<script src="index.js"></script>
```

```js
const container = document.getElementById("container");
container.innerText = "JavaScript is awesome!";
```

:::

Da `innerText` die Sichtbarkeit berücksichtigt, löst das Abrufen seines Werts einen Prozess namens „Reflow“ aus, der die Position bestimmter Elemente auf der Website neu berechnet. Dieser Prozess kann rechenintensiv sein, daher sollten Sie ihn möglichst vermeiden.

Gut. Nun sprechen wir über `textContent`.

`textContent` gibt den reinen Textinhalt eines Elements zurück, einschließlich aller Texte innerhalb seiner Nachfahren.

Der wichtigste Unterschied zwischen `innerText` und `textContent` ist, dass `textContent` immer den vollständigen Textinhalt eines HTML-Elements und seiner Nachfahren zurückgibt, unabhängig davon, ob er sichtbar oder verborgen ist.

Hier haben wir dasselbe Beispiel in HTML:

```html
<div id="container">
  <p>Hello, World!</p>
  <p>I'm learning JavaScript</p>
</div>
```

Wenn Sie versuchen, auf diese Eigenschaft zuzugreifen, sehen Sie den Text des Elements und seiner Nachfahren als Ausgabe, wobei Einrückungen und Abstände erhalten bleiben:

:::interactive_editor

```html
<div id="container">
  <p>Hello, World!</p>
  <p>I'm learning JavaScript</p>
</div>
<script src="index.js"></script>
```

```js
const container = document.getElementById("container");
console.log(container.textContent);
```

:::

Wenn ein HTML-Element nicht sichtbar ist, wie Sie hier sehen können, wo wir den zweiten Absatz verborgen haben, wird dessen Text trotzdem in dieser Eigenschaft enthalten sein:

:::interactive_editor

```html
<div id="container">
  <p>Hello, World!</p>
  <p hidden>I'm learning JavaScript</p>
</div>
<script src="index.js"></script>
```

```js
const container = document.getElementById("container");
console.log(container.textContent);
```

:::
  
`textContent` enthält auch den Inhalt von Elementen wie `script` und `style`.

Wenn Sie versuchen, den Wert von `textContent` an einem Knoten zu ersetzen, werden alle seine Kindknoten entfernt und durch einen einzelnen Textknoten mit dem neuen String ersetzt:

:::interactive_editor

```html
<div id="container">
  <p>Hello, World!</p>
  <p hidden>I'm learning JavaScript</p>
</div>
<script src="index.js"></script>
```

```js
const container = document.getElementById("container");
container.textContent = "New content";
```

:::

Und schließlich sprechen wir darüber, wie sich `textContent` und `innerText` von `innerHTML` unterscheiden.

Denken Sie daran, dass Sie mit `innerHTML` den inneren HTML-Inhalt eines Elements setzen können. Das ist hilfreich, um neuen HTML-Code dynamisch in den DOM einzufügen.

Beachten Sie jedoch, dass dies ein Sicherheitsrisiko darstellt, wenn Sie keine Kontrolle über den String haben, zum Beispiel bei Strings mit Benutzereingaben. Wenn diese Daten bösartig sind, kann das zu ernsthaften Sicherheitsproblemen führen.

Um dies zu vermeiden, wird empfohlen, die `textContent`-Eigenschaft zu verwenden, um reinen Text einzufügen.

Die `innerText`-, `textContent`- und `innerHTML`-Eigenschaften in JavaScript bieten verschiedene Möglichkeiten, auf den Inhalt von HTML-Elementen zuzugreifen und ihn zu manipulieren. Sie sollten die Unterschiede zwischen diesen Eigenschaften verstehen, wenn Sie effektiv mit HTML-Inhalten in JavaScript arbeiten möchten.
