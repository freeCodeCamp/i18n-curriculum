## Introducción a los Módulos Core de Node.js

- **Módulos Core de Node.js**: Módulos integrados que vienen preinstalados con Node.js, proporcionando funcionalidad esencial para operaciones de archivos, criptografía, interacciones con el sistema operativo y más.
- **Módulos estándar**: No es necesario instalarlos por separado; están disponibles mientras Node.js esté instalado.
- **Sintaxis de importación**: Usa `require()` para importar módulos principales en tu aplicación.

```javascript
const fs = require("fs");
const crypto = require("crypto");
const os = require("os");
```

## Módulo File System (fs)

- **Propósito**: Proporciona métodos para trabajar con archivos y carpetas, incluyendo operaciones de apertura, cierre, lectura, escritura y eliminación.
- **Importa**: `const fs = require("fs");`
- **Flexibilidad**: Métodos disponibles en tres formas: callbacks, promises y síncronos.

### Patrones de uso de métodos

- **Asíncrono con Callbacks**: comportamiento predeterminado, no bloqueante.

```javascript
fs.writeFile("filePath", "content", "utf8", (err) => {
  if (err) throw err;
  console.log("File written to!");
});
```

- **Basado en Promesas**: Preferido para aplicaciones modernas, usa `async/await`.

```javascript
const fs = require("fs/promises");

async function writeToFile() {
  try {
    await fs.writeFile("article.md", "## Node fs Module");
    console.log("File written to!");
  } catch (err) {
    console.error(err);
  }
}
```

- **Síncrono**: Operación bloqueante, adecuada para scripts simples.

```javascript
try {
  fs.writeFileSync("article.md", "## Node fs Module", "utf8");
  console.log("File written to!");
} catch (err) {
  console.error(err);
}
```

### Operaciones básicas de archivos

- **Método `writeFile()`**: Crea o sobrescribe un archivo con el contenido especificado.

```javascript
await fs.writeFile("article.md", "## Node fs Module: The Complete Guide");
```

- **Método `appendFile()`**: Añade contenido al final de un archivo existente.

```javascript
await fs.appendFile("article.md", "\n\nIn this article, you will learn...");
```

- **Método `readFile()`**: Lee el contenido de un archivo.

```javascript
const content = await fs.readFile("article.md", "utf8");
console.log("File content:", content);
```

- **Método `unlink()`**: Elimina un archivo del sistema de archivos.

```javascript
await fs.unlink("article.md");
console.log("File deleted successfully");
```

### Codificación de caracteres

- **Codificación UTF-8**: Especifica `"utf8"` para leer archivos como cadenas de texto.
- **Formato de Buffer**: Sin especificación de codificación, los archivos se leen como objetos buffer binarios.

```javascript
// Returns string
const textContent = await fs.readFile("file.txt", "utf8");

// Returns Buffer
const bufferContent = await fs.readFile("file.txt");
```

## Módulo Buffer

- **Propósito**: Maneja datos binarios directamente en memoria para archivos, imágenes y flujos de red.
- **Importa**: `const { Buffer } = require("buffer");`
- **Casos de uso**: Entrada/salida de archivos, flujos TCP, procesamiento de imágenes y manipulación de datos binarios.

### Creación de Buffer

- **Método `Buffer.from()`**: Crea un buffer a partir de cadenas, arreglos u otros datos.

```javascript
// From string
const myStrBuffer = Buffer.from("freeCodeCamp");
console.log(myStrBuffer); // <Buffer 66 72 65 65 43 6f 64 65 43 61 6d 70>

// From array of numbers
const myNumBuffer = Buffer.from([70, 82, 69, 69, 67, 79, 68, 69, 67, 65, 77, 80]);
```

- **Método `Buffer.alloc()`**: Crea un buffer de tamaño especificado lleno de ceros.

```javascript
const someBuffer = Buffer.alloc(10);
console.log(someBuffer); // <Buffer 00 00 00 00 00 00 00 00 00 00>
```

### Operaciones de Buffer

- **Acceso similar a un arreglo**: Accede a elementos individuales del buffer usando notación de índice.

```javascript
console.log(myStrBuffer[0]); // 102 (byte value for 'f')
```

- **Método `toString()`**: Convierte el buffer en una cadena legible.

```javascript
console.log(myStrBuffer.toString()); // "freeCodeCamp"
```

- **Método `Buffer.write()`**: Escribe datos en un buffer asignado.

```javascript
someBuffer.write("Hello fCC");
console.log(someBuffer.toString()); // "Hello fCC"
```

- **Método `Buffer.byteLength()`**: Devuelve el número de bytes necesarios para almacenar una cadena.

```javascript
console.log(Buffer.byteLength("Hello freeCodeCamp")); // 18
```

### Métodos adicionales de Buffer

- **`Buffer.isBuffer()`**: Verifica si un objeto es un buffer.
- **`Buffer.compare()`**: Compara dos buffers y devuelve el orden de clasificación.
- **`Buffer.concat()`**: Une múltiples buffers en uno solo.

## Módulo Cripto

- **Propósito**: Proporciona funcionalidad criptográfica, incluyendo hashing, cifrado, descifrado y firmas digitales.
- **Importa**: `const crypto = require("crypto");`
- **Nota de seguridad**: Usa bibliotecas bien probadas como `bcrypt` o `jsonwebtoken` para la autenticación en producción.

### Métodos de Hashing

- **Método `createHash()`**: Crea un hash unidireccional usando algoritmos como SHA256, SHA512 o MD5.

```javascript
const hashedPassword = crypto
  .createHash("sha256")
  .update("myPassword123")
  .digest("hex");

console.log("createHash result:", hashedPassword);
```

- **Método `createHmac()`**: Crea un hash con una clave secreta para autenticación e integridad de datos.

```javascript
const hashedMessage = crypto
  .createHmac("sha256", "secret-key")
  .update("Hello World")
  .digest("hex");
```

### Cifrado y Descifrado

- **Métodos `createCipheriv()` y `createDecipheriv()`**: Cifra y descifra datos usando algoritmos, claves y vectores de inicialización.

```javascript
const key = Buffer.from("12345678901234567890123456789012"); // 32 bytes for AES-256
const iv = Buffer.from("1234567890123456"); // 16 bytes for AES

// Encryption
const cipher = crypto.createCipheriv("aes-256-cbc", key, iv);
let encrypted = cipher.update("Hello campers!", "utf8", "hex");
encrypted += cipher.final("hex");

// Decryption
const decipher = crypto.createDecipheriv("aes-256-cbc", key, iv);
let decrypted = decipher.update(encrypted, "hex", "utf8");
decrypted += decipher.final("utf8");
```

### Generación de Valor Aleatorio

- **Método `randomBytes()`**: Genera bytes aleatorios criptográficamente seguros.

```javascript
console.log("Random Bytes:", crypto.randomBytes(16).toString("hex"));
// Output: a6154ef5a296fa176ad0f332bd94d712
```

- **Método `randomInt()`**: Genera enteros aleatorios seguros dentro del rango especificado.

```javascript
console.log("Random Int:", crypto.randomInt(0, 100)); // Random number 0-99
```

### Gestión de claves

- **Método `createSecretKey()`**: Crea objetos de clave criptográfica a partir de bytes sin procesar.

```javascript
const secret = crypto.createSecretKey(crypto.randomBytes(32));
console.log(secret.export().toString('hex'));
```

### Firmas Digitales

- **Métodos `sign()` y `verify()`**: Crea y valida firmas digitales usando pares de claves privadas/públicas.
- **Caso de uso**: Demuestra la autenticidad e integridad de los datos.

### Métodos Cripto Adicionales

- **`createPublicKey()` y `createPrivateKey()`**: Trabaja con claves generadas externamente.
- **`createDiffieHellman()`**: Permite la generación segura de secretos compartidos.
- **`Certificate()`**: Maneja operaciones de certificados HTTPS.

## Módulo OS

- **Propósito**: Proporciona utilidades relacionadas con el sistema operativo para información del sistema y compatibilidad multiplataforma.
- **Importa**: `const os = require("os");`
- **Casos de uso**: monitoreo del sistema, scripting multiplataforma, información de hardware.

### Métodos de Información del Sistema

- **Método `platform()`**: Devuelve la plataforma del sistema operativo.

```javascript
console.log(os.platform()); // 'darwin', 'win32', 'linux', etc.

// Cross-platform usage
if (os.platform() === 'win32') {
  // Windows-specific code
} else {
  // Unix-like systems
}
```

- **Método `arch()`**: Devuelve la arquitectura de la CPU.

```javascript
console.log(os.arch()); // 'arm64', 'x64', 'arm', 'ia32', etc.
```

- **Método `type()`**: Devuelve el nombre oficial del sistema operativo.

```javascript
console.log(os.type()); // 'Darwin', 'Linux', 'Windows_NT'
```

### Información de la versión

- **Método `release()`**: Muestra la versión del kernel del SO.

```javascript
console.log(os.release()); // '25.0.0'
```

- **Método `version()`**: Devuelve información detallada de la versión del SO.

```javascript
console.log(os.version());
// Darwin Kernel Version 25.0.0: Wed Sep 17 21:41:39 PDT 2025
```

### Información de hardware

- **Método `cpus()`**: Devuelve un arreglo con información de los núcleos de la CPU.

```javascript
console.log(os.cpus()); // Array of CPU core objects with model, speed, times
```

- **Método `uptime()`**: Devuelve el tiempo de actividad del sistema en segundos.

```javascript
console.log(os.uptime()); // 23047 (seconds since system boot)
```

### Información de Memoria

- **Método `totalmem()`**: Devuelve la memoria total del sistema en bytes.
- **Método `freemem()`**: Devuelve la memoria del sistema disponible en bytes.

## Mejores Prácticas

### Consideraciones de rendimiento

- **Evita los métodos síncronos**: Usa métodos asíncronos en aplicaciones de producción para evitar bloqueos.
- **Prefiere Promesas**: Usa `async/await` con API basada en promesas para un código más limpio y mantenible.
- **Manejo de errores**: Siempre implementa un manejo adecuado de errores con bloques try-catch o callbacks de error.

### Directrices de seguridad

- **Módulo Crypto**: Usa bibliotecas establecidas para la autenticación en lugar de crear soluciones criptográficas personalizadas.
- **Validación de Entrada**: Valida las rutas de archivos y la entrada del usuario para prevenir vulnerabilidades de seguridad.
- **Gestión de claves**: Almacena las claves criptográficas de forma segura, nunca las codifiques directamente en el código fuente.

### Desarrollo multiplataforma

- **Uso del Módulo OS**: Usa los métodos del módulo OS para compatibilidad multiplataforma.
- **Manejo de Rutas**: Usa el módulo `path` para operaciones de rutas de archivos multiplataforma.
- **Variables de Entorno**: Revisa las variables de entorno y configuraciones específicas de la plataforma.

## Casos de uso comunes

### Operaciones de archivo

- Gestión de archivos de configuración
- Creación y mantenimiento de archivos de registro
- Funcionalidad de exportación/importación de datos
- Procesamiento de plantillas

### Seguridad de Datos

- Hashing de contraseñas para la autenticación de usuarios
- Cifrado de datos para información sensible
- Generación aleatoria de tokens para sesiones
- Verificación de firma digital

### Integración del sistema

- Despliegue de aplicaciones multiplataforma
- Monitoreo de recursos del sistema
- Comprobaciones de compatibilidad de hardware
- Optimización del rendimiento basada en las capacidades del sistema
