## --quiz--

### --question--

#### --text--

Cos’è una funzione pura?

#### --distractors--

Una funzione che modifica variabili globali e restituisce `undefined` o `null`.

---

Una funzione che stampa un output sulla console.

---

Una funzione che cambia comportamento in base a fattori esterni e restituisce `null`.

#### --answer--

Una funzione che restituisce sempre lo stesso output per lo stesso input e non produce effetti collaterali.

### --question--

#### --text--

Cos’è un effetto collaterale nella programmazione funzionale?

#### --distractors--

Un `TypeError` inatteso nel tuo codice.

---

Una funzione che impiega troppo tempo per eseguire e blocca il programma.

---

Una chiamata ricorsiva che genera un loop infinito e blocca il programma.

#### --answer--

Una modifica allo stato del programma osservabile all’esterno della funzione.

### --question--

#### --text--

Cos’è il currying nella programmazione funzionale?

#### --distractors--

Una tecnica per progettare test che ti aiutano a garantire che il tuo codice sia privo di errori al 100%.

---

Un modo più ottimizzato per scrivere funzioni ricorsive in modo che siano due volte più veloci di altre funzioni.

---

Il processo di costruzione, progettazione e test del codice per assicurarsi che rispetti gli standard definiti da ECMAScript.

#### --answer--

Il processo di trasformare una funzione con più argomenti in una sequenza di funzioni, ciascuna con un solo argomento.

### --question--

#### --text--

Quale dei seguenti è un esempio di currying?

#### --distractors--

```js
const curriedAverage = (a, b, c) => a + b + c / 3
```

---

```js
const curriedAverage = (a, b, c) => a + b + c
```

---

```js
function curriedAverage(a) {
  return a 
}

function curried(c) {
  return c
}
```

#### --answer--

```js
function curriedAverage(a) {
  return function(b) {
    return function(c) {
      return (a + b + c) / 3;
    };
  };
}
```
