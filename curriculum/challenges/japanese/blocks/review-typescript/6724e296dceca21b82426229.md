## TypeScriptとは何か

- **JavaScript**: JavaScriptは動的型付け言語です。これは変数が実行時にどんな値でも受け取れることを意味します。動的型付け言語の課題は、型の安全性がないためにエラーが発生しやすいことです。

例えば、JavaScriptの関数が配列を期待していても、数値を渡して呼び出すことができます。

```javascript
const getRandomValue = (array) => {
  return array[Math.floor(Math.random() * array.length)];
}

console.log(getRandomValue(10));
```

上記の例の`console`の出力は`undefined`になります。

- **TypeScript**: TypeScriptはJavaScript言語を拡張し、静的型付けを導入しています。これにより、コードを実行する前に型の不一致によるエラーを検出できます。

例えば、`array`パラメータの型を次のように定義できます。

```typescript
const getRandomValue = (array: string[]) => {
  return array[Math.floor(Math.random() * array.length)];
}
```

この型定義は、`array`パラメータが文字列の配列でなければならないことをTypeScriptに伝えます。`getRandomValue`を呼び出して数値を渡すと、コンパイラーエラーが発生します。

- **コンパイラー**: TypeScriptコードはまず通常のJavaScriptにコンパイルする必要があります。コンパイラーを実行すると、TypeScriptはコードを評価し、型が一致しない問題があればエラーをスローします。

## TypeScriptのデータ型

- **TypeScriptのプリミティブデータ型**: プリミティブデータ型である`string`、`null`、`undefined`、`number`、`boolean`、`bigint`に対応する型キーワードがTypeScriptにはあります。

```typescript
let str: string = "Naomi";
let num: number = 42;
let bool: boolean = true;
let nope: null = null;
let nada: undefined = undefined;
```

- **配列**: 特定の型の配列は2つの異なる構文で定義できます。

```typescript
const arrOne: string[] = ["Naomi"];
const arrTwo: Array<string> = ["Camperchan"];
```

- **オブジェクト**: オブジェクトの正確な構造を定義できます。

```typescript
const obj: { a: string, b: number } = { a: "Naomi", b: 42 };
```

任意のキーを持ち、すべての値が文字列でなければならないオブジェクトを定義するには、2つの方法があります。

```typescript
const objOne: Record<string, string> = {};
const objTwo: { [key: string]: string } = {};
```

- **TypeScriptのその他の便利な型**:
  - **`any`**: `any`は値が任意の型を持つことを示します。コンパイラーにその変数の型を気にしないよう指示します。
  - **`unknown`**: `unknown`は値の型を気にしているが、実際には何の型か分からないことをTypeScriptに伝えます。`unknown`は一般的に`any`より好まれます。
  - **`void`**: これは通常関数を定義するときに使う特別な型です。戻り値がない関数は戻り値の型として`void`を使います。
  - **`never`**: 存在しない型を表します。

- **`type`キーワード**: このキーワードは`const`のようなものですが、変数を宣言する代わりに型を宣言できます。

これはユニオン型や特定の値だけを含む型などのカスタム型を宣言するときに便利です。

```typescript
type stringOrNumber = string | number;
type bot = "camperchan" | "camperbot" | "naomi";
```

- **`interface`**: インターフェースは型のためのクラスのようなものです。ほかのインターフェースを実装または拡張でき、特にオブジェクト型であり、`type`宣言の特定の機能が必要な場合を除き、一般的に好まれます。

```typescript
interface wowie {
  zowie: boolean;
  method: () => void;
}
```

- **戻り値の型定義**: 関数の戻り値の型も定義できます。

以下の例は戻り値を文字列として定義しています。もし他の値を返そうとすると、TypeScriptはコンパイラーエラーを出します。

```typescript
const getRandomValue = (array: string[]): string => {
  return array[Math.floor(Math.random() * array.length)];
}
```

## ジェネリクス

- **ジェネリック型の定義**: ジェネリック型を定義し、関数内で参照できます。これは関数の型定義の振る舞いを制御するために関数に渡す特別なパラメータのようなものです。

関数のジェネリック型を定義する例はこちらです。

```typescript
const getRandomValue = <T>(array: T[]): T => {
  return array[Math.floor(Math.random() * array.length)];
}
const val = getRandomValue([1, 2, 4])
```

`<T>`構文は、関数のジェネリック型`T`を定義していることをTypeScriptに伝えます。`T`はジェネリック型の一般的な名前ですが、任意の名前を使えます。

次に、`array`パラメータがジェネリック型に一致する値の配列であり、戻り値が同じ型の単一要素であることをTypeScriptに伝えます。

- **関数呼び出しでの型引数の指定**: 関数名とパラメータの間に山括弧を使って型引数を渡せます。

関数呼び出しで型引数を渡す例はこちらです。

```typescript
const email = document.querySelector<HTMLInputElement>("#email");
console.log(email.value);
```

これは、期待する要素が入力要素であることをTypeScriptに伝えます。

## 型の絞り込み

- **真偽値による絞り込み**: 以下の例では、`value`が`email`である可能性があるため、`email`の`null`プロパティにアクセスしようとするとコンパイラーエラーになります。

```typescript
const email = document.querySelector<HTMLInputElement>("#email");
console.log(email.value);
```

条件文を使って`email`が真であることを確認してからプロパティにアクセスできます。

```typescript
const email = document.querySelector<HTMLInputElement>("#email");
if (email) {
  console.log(email.value);
}
```

真偽値チェックは逆方向でも機能します。

```typescript
const email = document.querySelector<HTMLInputElement>("#email");
if (!email) {
  throw new ReferenceError("Could not find email element!")
}
console.log(email.value);
```

エラーをスローするとこのコードの論理的な実行が終了するため、`console.log()`の呼び出しに到達した時点で、TypeScriptは`email`が`null`でないことを認識します。

- **オプショナルチェーン**: オプショナルチェーン`?.`も型の絞り込みの一種で、`email`の値が`null`の場合はプロパティアクセスができないという前提に基づいています。

```typescript
const email = document.querySelector<HTMLInputElement>("#email");
console.log(email?.value);
```

- **`typeof`演算子**: `typeof`演算子を使って変数の型を条件でチェックできます。

```typescript
const myVal = Math.random() > 0.5 ? 222 : "222";
if (typeof myVal === "number") {
  console.log(myVal / 10);
}
```

- **`instanceof`キーワード**: オブジェクトがクラスから来ている場合、`instanceof`キーワードを使って型を絞り込めます。

```typescript
const email = document.querySelector("#email");

if (email instanceof HTMLInputElement) {
    console.log(email.value);
}
```

- **キャスト**: TypeScriptが値の型を自動的に判別できない場合（以下の例の`request.json()`メソッドの結果など）、コンパイラーエラーになります。これを解決する方法の一つが型キャストですが、これによりTypeScriptのエラー検出能力が弱まります。

```typescript
interface User {
    name: string;
    age: number;
}

const printAge = (user: User) => 
  console.log(`${user.name} is ${user.age} years old!`)

const request = await fetch("url")
const myUser = await request.json() as User;
printAge(myUser);
```

- **型ガード**: 型キャストの代わりに型ガードを書くことができます。

```typescript
interface User {
    name: string;
    age: number;
}

const isValidUser = (user: unknown): user is User => {
  return !!user && 
    typeof user === "object" &&
    "name" in user &&
    "age" in user;
}
```

`user is User`構文は、関数がブール値を返し、trueの場合は`user`の値が`User`インターフェースを満たしていることを示します。

## `tsconfig`ファイル

- **`tsconfig.json`**: TypeScriptのコンパイラー設定はプロジェクトのルートディレクトリにある`tsconfig.json`ファイルにあります。

```json
{
  "compilerOptions": {
    "rootDir": "./src",
    "outDir": "./prod",
    "lib": ["ES2023"],
    "target": "ES2023",
    "module": "ES2022",
    "moduleResolution": "Node",
    "esModuleInterop": true,
    "skipLibCheck": true,
    "strict": true
  },
  "exclude": ["test/"]
}
```

上記の例で使われているコンパイラーオプションの説明は以下の通りです。

- **`compilerOptions`**: `compilerOptions`プロパティはTypeScriptコンパイラーの動作を制御します。
- **`rootDir`と`outDir`**: `rootDir`と`outDir`はTypeScriptにソースファイルのディレクトリとトランスパイルされたJavaScriptコードのディレクトリを指定します。
- **`lib`**: `lib`プロパティはコンパイラーが使う型定義を決め、特定のESリリースやDOMなどのサポートを含めることができます。
- **`module`と`moduleResolution`**: `module`と`moduleResolution`はモジュールの使い方（CommonJSかECMAScriptか）を管理します。
- **`esModuleInterop`**: `esModuleInterop`はCommonJSとESモジュール間の相互運用性をスムーズにするため、インポートの名前空間オブジェクトを自動生成します。
- **`skipLibCheck`**: `skipLibCheck`オプションはコード内でインポートされていない`.d.ts`ファイルの検証をスキップします。
- **`strict`**: `strict`フラグはnullable型の適切な扱いや、TypeScriptが`any`にフォールバックするときの警告など、いくつかのチェックを有効にします。
- **`exclude`**: 最上位の`exclude`プロパティはコンパイル時にこれらのTypeScriptファイルを無視するようコンパイラーに指示します。
