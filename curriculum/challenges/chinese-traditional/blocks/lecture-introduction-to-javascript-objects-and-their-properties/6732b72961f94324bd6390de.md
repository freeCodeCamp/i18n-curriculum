# --interactive--

在 JavaScript 中，有幾種方法可以檢查物件是否具有特定的屬性。了解這些方法對於有效地操作物件非常重要，特別是當你處理來自外部來源的資料或需要確保某些屬性存在才能使用時。 

我們將探討一些常見的方法：`hasOwnProperty()` 方法、`Object.hasOwn()` 方法、`in` 運算子，以及檢查是否為 `undefined`。

讓我們從 `hasOwnProperty()` 方法開始。此方法傳回一個布林值，表示該物件是否擁有指定的屬性作為其自身的屬性。以下是一個範例：

:::interactive_editor

```js
const person = {
  name: "Alice",
  age: 30
};

console.log(person.hasOwnProperty("name")); // true
console.log(person.hasOwnProperty("job")); // false
```

:::

在這個範例中，我們有一個名為 `person` 的物件，具有兩個屬性：`name` 和 `age`。要檢查 `name` 是否為 `person` 物件中的屬性，我們使用 `hasOwnProperty()` 方法。因為 `name` 是屬性，所以它會傳回 `true`。但是當我們使用 `hasOwnProperty()` 來檢查 `job` 是否為屬性時，它會傳回 `false`，因為該屬性不存在於物件中。

`Object.hasOwn()` 是檢查物件是否擁有自身（非繼承）屬性的現代且推薦的方法。可以將它視為 `hasOwnProperty()` 的升級且更安全的版本。語法為 `Object.hasOwn(object, propertyName)` — 你將物件作為第一個引數，屬性名稱作為第二個引數。

這是一個基本範例：

:::interactive_editor

```js
const person = {
  name: "Alice",
  age: 30
};

console.log(Object.hasOwn(person, "name")); // true
console.log(Object.hasOwn(person, "job")); // false
```

:::

在此範例中，`Object.hasOwn(person, "name")` 傳回 `true`，因為 `name` 直接存在於 `person` 物件上。`Object.hasOwn(person, "job")` 傳回 `false`，因為 `job` 從未被添加到該物件。

一個非常重要的事情是要了解 `Object.hasOwn()` 僅檢查該屬性是否**存在**——它不在意該屬性的值。這表示即使該值是 `0`、`false`、`null` 或 `undefined`，它仍會傳回 `true`：

:::interactive_editor

```js
const user = {
  username: "coder123",
  score: 0,
  isActive: false,
  nickname: null
};

// Object.hasOwn() correctly reports these all exist
console.log(Object.hasOwn(user, "score"));    // true  (value is 0, but property exists)
console.log(Object.hasOwn(user, "isActive")); // true  (value is false, but property exists)
console.log(Object.hasOwn(user, "nickname")); // true  (value is null, but property exists)
console.log(Object.hasOwn(user, "email"));   // false (property was never added)

// Danger! Using if() directly gives wrong results for falsy values
if (user.score) {
  console.log("Has score"); // This will NOT print even though score exists!
}

// Safe! Object.hasOwn() gives correct result
if (Object.hasOwn(user, "score")) {
  console.log("Has score:", user.score); // Has score: 0
}
```

:::

檢查物件中是否存在屬性的另一種方法是使用 `in` 運算子。像 `hasOwnProperty()` 一樣，如果屬性存在於物件上，`in` 運算子會傳回 `true`。以下是你如何使用它：

:::interactive_editor

```js
const person = {
  name: "Bob",
  age: 25
};
console.log("name" in person);  // true
```

:::

在此範例中，`"name" in person` 傳回 `true`，因為 `name` 是 `person` 的一個屬性。

第三種方法涉及檢查屬性是否為 `undefined`。這種方法可能有用，但它有一些限制。以下是一個範例：

:::interactive_editor

```js
const car = {
  brand: "Toyota",
  model: "Corolla",
  year: 2020
};

console.log(car.brand !== undefined); // true
console.log(car.color !== undefined); // false
```

:::

在這段程式碼中，我們檢查 `car.brand` 和 `car.color` 是否不是 `undefined`。這是因為存取物件上不存在的屬性會傳回 `undefined`。然而，如果屬性明確地具有 `undefined` 值，這種方法可能會產生誤判。

在實務上，這些方法之間的選擇通常取決於你的程式碼的具體需求。理解它們之間的差異將幫助你在不同情境中做出正確的選擇。
