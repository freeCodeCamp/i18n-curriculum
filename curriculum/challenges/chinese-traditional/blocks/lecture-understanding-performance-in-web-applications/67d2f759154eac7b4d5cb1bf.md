# --description--

如我們所知，感知效率不僅僅是網站實際的速度——而是網站給使用者的速度感受。例如，如果你曾經使用過一個感覺很慢的網站，即使它實際上並不慢？那就是感知效率——網站給使用者的速度感受。

小幅調整就能大幅提升使用者體驗。讓我們來探討一些改善感知效率的實用方法！

旋轉指示器會讓使用者感覺等待時間比實際更長。相反地，試試骨架螢幕 — 灰色佔位符，模擬真實內容並逐漸載入：

```html
<div class="skeleton"></div>
<script>
  setTimeout(() => {
    document.querySelector(".skeleton").innerHTML = "✅ Content Loaded!";
  }, 2000);
</script>
```

使用者感覺頁面是在逐步載入，而不是在黑暗中等待。

除此之外，使用者不應該必須等到所有內容載入完成才能開始互動。先載入重要內容，然後在背景中載入其餘部分。我們之前已經探討過這個懶惰載入的做法：

```html
<img src="placeholder.jpg" data-src="real-image.jpg" class="lazy">
<script>
  document.addEventListener("DOMContentLoaded", function () {
    const lazyImages = document.querySelectorAll("img.lazy");
    lazyImages.forEach(img => {
      img.src = img.dataset.src;
    });
  });
</script>
```

這讓頁面感覺很有回應性，即使並非所有內容都已載入。

現在－如果你的网站能預測使用者接下來會點擊什麼呢？透過預先載入，你可以在他們請求之前取得下一頁：

```html
<link rel="preload" href="next-page.html" as="document">
```

當使用者導航時，這會將感知的載入時間降低到接近零。

最後，使用者不喜歡不確定性。當他們點擊按鈕時，立即確認他們的操作：

```html
<button onclick="this.innerText='⏳ Processing...'; setTimeout(() => this.innerText='✅ Done!', 2000);">
  Click Me
</button>
```

這表示，即使有延遲，使用者也會感覺到有事情正在發生。

總結來說，提升感知效率不僅僅是關於未經處理的速度，而是關於速度給使用者的感受。使用骨架螢幕、優先顯示重要內容、智慧地預先載入，並提供即時回授，使你的網站感覺更快。
