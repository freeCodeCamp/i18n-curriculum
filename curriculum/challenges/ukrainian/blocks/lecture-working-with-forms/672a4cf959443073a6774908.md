# --interactive--

У HTML елементи керування формою, як-от input, можуть перебувати в різних станах або умовах, наприклад `focused` стан, `readonly` стан або `disabled` стан.

Перший стан вважається `default` станом. За замовчуванням поле введення для е-пошти порожнє. Саме так виглядає поле е-пошти, коли його вперше відтворюють на сторінці.

:::interactive_editor

```html
<input type="email" name="email" id="email" />
```

:::

Коли користувач натискає на елемент керування формою або вибирає його за допомогою клавіші табуляції, це означає, що він перебуває в `focused` стані. Коли input у `focused` стані, більшість браузерів показують синій підсвічений кордон навколо поля введення. Але ви можете додати додаткові стилі в CSS.

Натисніть на будь-яку частину пробілу у вікні попереднього перегляду, а потім натисніть клавішу `tab`, щоб побачити стан фокусу. Щоб побачити попередні перегляди, потрібно увімкнути інтерактивний редактор.

:::interactive_editor

```html
<input type="email" name="email" id="email" />
```

:::

Ще один стан форми — це `disabled` стан. Цей стан показує користувачам, що поле введення не можна сфокусувати або активувати.

Увімкніть інтерактивний редактор і спробуйте натиснути на поле введення е-пошти — ви помітите, що воно більше не отримує фокус.

:::interactive_editor

```html
<input disabled type="email" name="email" id="email" />
```

:::

Подібно до `focused` стану, ви можете додати додаткові стилі для `disabled` стану за допомогою CSS.

Ще один тип стану форми — це `readonly` стан. Це коли елемент керування формою, як-от input, не редагується користувачем. Ось приклад, як зробити поле введення е-пошти лише для читання. Атрибут `value` використовується для встановлення значення, яке відображається всередині поля введення.

Увімкніть інтерактивний редактор і спробуйте змінити поточне значення `example@email.com` у вікні попереднього перегляду — ви помітите, що це неможливо.

:::interactive_editor

```html
<input
  readonly
  type="email"
  name="email"
  id="email"
  value="example@email.com"
/>
```

:::

Ключова відмінність між `disabled` станом і `readonly` станом полягає в тому, що `readonly` можна сфокусувати, а `disabled` — ні.

Розуміння різних станів форми важливе, бо вони забезпечують плавний користувацький досвід, надаючи чіткий зворотний зв’язок і підказки під час обробки помилок.
