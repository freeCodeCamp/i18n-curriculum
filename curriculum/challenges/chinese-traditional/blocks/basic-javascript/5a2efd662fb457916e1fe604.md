# --description--

你將學習的下一種迴圈稱為 `do...while` 迴圈。它被稱為 `do...while` 迴圈，是因為它會先無論如何先 `do` 一次迴圈內的程式碼，然後只要指定的條件評估為 `true`，就會繼續執行該迴圈。

```js
const ourArray = [];
let i = 0;

do {
  ourArray.push(i);
  i++;
} while (i < 5);
```

上述範例的行為類似其他型別的迴圈，結果陣列會看起來像 `[0, 1, 2, 3, 4]`。然而，使 `do...while` 與其他迴圈不同的是當條件在第一次檢查時失敗的行為。讓我們來看看實際運作。這裡有一個一般的 `while` 迴圈，只要 `i < 5` 就會執行迴圈中的程式碼：

```js
const ourArray = []; 
let i = 5;

while (i < 5) {
  ourArray.push(i);
  i++;
}
```

在這個範例中，我們將 `ourArray` 的值初始化為空陣列，並將 `i` 的值初始化為 5。當我們執行 `while` 迴圈時，條件會被計算為 `false`，因為 `i` 不小於 5，所以我們不會執行迴圈內的程式碼。結果是 `ourArray` 不會被加入任何值，並且在上述所有程式碼執行完畢後，它仍然看起來像 `[]`。現在，來看看 `do...while` 迴圈：

```js
const ourArray = []; 
let i = 5;

do {
  ourArray.push(i);
  i++;
} while (i < 5);
```

在這種情況下，我們將 `i` 的值初始化為 5，就像我們對 `while` 迴圈所做的那樣。當我們到達下一行時，沒有條件需要評估，所以我們進入大括號內的程式碼並執行它。我們會將單一元素添加到陣列中，然後在進行條件檢查之前將 `i` 遞增。當我們最終在最後一行評估條件 `i < 5` 時，我們看到 `i` 現在是 6，不符合條件檢查，因此我們離開迴圈並完成。在上述範例結束時，`ourArray` 的值是 `[5]`。本質上，`do...while` 迴圈確保迴圈內的程式碼至少會執行一次。讓我們嘗試透過將值推入陣列來讓 `do...while` 迴圈運作。
