# --description--

**連結リスト**は、各ノードがシーケンス内の次のノードに接続されている線形のデータ構造です。

これらの接続により、各ノードがデータと連結リスト内の次のノードへの参照を格納するノードの鎖のようなデータ構造が作られます。

<img src="https://cdn.freecodecamp.org/curriculum/lecture-transcripts/how-do-singly-linked-lists-work-and-how-do-they-differ-from-doubly-linked-list-1.png" alt="linked list basic structure visualization">

これらの参照を使って、最初のノードから次のノードへと順に移動します。

連結リストは、スタック、キュー、デックなどの他のデータ構造を実装するためによく使われます。また、深さ優先探索や幅優先探索などの重要なグラフアルゴリズムの実装にも使われます。

## 単方向連結リスト

**単方向連結リスト**は、各ノードがシーケンス内の次のノードに接続されている連結リストの一種です。

各ノードは、次のノードへの参照を格納して接続されています。

このノードごとの単一の参照により、連結リストを開始から終了まで一方向にたどることができます。

探索は前方にしか進めず、後ろには戻れません。

この例では、ヘッドノードであるノードAから開始します。

**ヘッド**ノードは連結リストの最初のノードです。

単方向連結リストでは、ヘッドノードが通常、直接アクセスできる唯一のノードです。特定のノードを探すときは、ここから探索が始まります。

<img src="https://cdn.freecodecamp.org/curriculum/lecture-transcripts/how-do-singly-linked-lists-work-and-how-do-they-differ-from-doubly-linked-list-2.webp" alt="singly linked list traversal visualization">

処理はノードAから始まり、次にノードB、ノードC、そして最後に**テイル**ノードであるノードDへと続きます。コードに特定のロジックを実装していれば、その途中で停止する場合もあります。

**テイル**ノードは最後のノードで、連結リストの終端に達したことを判断するために使われます。

### **ノードの挿入**

連結リストの素晴らしい点の一つは、固定サイズではないことです。ノード間の接続を更新するだけで、必要に応じて拡張や縮小ができます。

連結リストの先頭、途中、末尾にノードを**挿入**できます。

連結リストは必ずしもノードを特定の順序で格納する必要はありません。順序はノード間の接続によって決まります。

ただし、特定の用途でノードを特定の順序に保つ必要がある場合は、そのロジックをコードに実装すれば、挿入位置が先頭、途中、末尾のいずれかに決まります。

連結リストの先頭にノードを挿入するには、新しいノードと元のヘッドノードの間に接続を作り、新しいノードをヘッドノードにすればいいだけです。

これは、ノードEを先頭に挿入し、この新しいノードを連結リストのヘッドノードにする例です。

<img src="https://cdn.freecodecamp.org/curriculum/lecture-transcripts/how-do-singly-linked-lists-work-and-how-do-they-differ-from-doubly-linked-list-3.png" alt="singly linked list insert at beginning visualization">

連結リストの先頭にノードを挿入する操作は、ヘッドノードへの参照と新しいヘッドノードと次のノードの接続を更新するだけなので、定数時間の計算量 `O(1)` です。

この例では、ノードEを連結リストの先頭に挿入しています。これは正しく動作します。しかし、連結リストをアルファベット順にソートしておきたい場合は、ノードEは連結リストの末尾に挿入しなければなりません。

連結リストの末尾にノードを挿入するには、まず末尾までたどり着き、新しいノードへの接続を追加して新しいテイルノードにします。

この操作は、連結リスト内のノード数をnとすると線形時間の計算量 `O(n)` です。なぜなら、挿入のために連結リストの末尾まで一つずつノードをたどる必要があるからです。

<img src="https://cdn.freecodecamp.org/curriculum/lecture-transcripts/how-do-singly-linked-lists-work-and-how-do-they-differ-from-doubly-linked-list-4.png" alt="singly linked list insert at end visualization">

連結リストの途中にノードを挿入する場合は、ノード間の接続も更新する必要があります。シーケンスの前のノードを新しいノードに接続し、新しいノードを次のノードに接続します。以下の図のようになります。

<img src="https://cdn.freecodecamp.org/curriculum/lecture-transcripts/how-do-singly-linked-lists-work-and-how-do-they-differ-from-doubly-linked-list-5.webp" alt="singly linked list insert in middle visualization">

挿入操作は、ノードを作成しノード間の接続を更新するだけなので、定数空間の計算量 `O(1)` です。この操作は連結リストのサイズには依存しません。

### **ノードの削除**

ノードを挿入できるように、連結リストの先頭、途中、末尾からノードを削除することもできます。

先頭のノードを削除するには、ヘッドノードへの参照を更新し、シーケンスの次のノードにします。

この操作は、連結リストのヘッドノードへの参照を更新するだけなので、定数時間の計算量 `O(1)` です。

<img src="https://cdn.freecodecamp.org/curriculum/lecture-transcripts/how-do-singly-linked-lists-work-and-how-do-they-differ-from-doubly-linked-list-6.png" alt="singly linked list remove from beginning visualization">

連結リストの途中のノードを削除するには、前のノードの参照を更新して次のノードに接続し、ノード間の「橋渡し」を作ります。以下の図のようになります。

<img src="https://cdn.freecodecamp.org/curriculum/lecture-transcripts/how-do-singly-linked-lists-work-and-how-do-they-differ-from-doubly-linked-list-7.png" alt="singly linked list remove from middle visualization">

これにより、削除したいノード（この場合はノードB）が接続のシーケンスから外され、次回の探索でたどられなくなります。

連結リストの末尾のノードを削除するには、前のノードの接続を削除し、そのノードを新しいテイルノードにします。これで連結リストは新しいテイルノードで終わります。

この操作は、連結リストの末尾までたどる必要があるため、線形時間の計算量 `O(n)` です。

<img src="https://cdn.freecodecamp.org/curriculum/lecture-transcripts/how-do-singly-linked-lists-work-and-how-do-they-differ-from-doubly-linked-list-8.png" alt="singly linked list remove from end visualization">

削除操作は、ノードを削除するのに追加のメモリを必要としないため、定数空間の計算量 `O(1)` です。

## 双方向連結リスト

単方向連結リストについて理解したところで、双方向連結リストについて説明します。

**双方向連結リスト**では、各ノードが2つの参照を格納します。1つは次のノードへの参照、もう1つはシーケンス内の前のノードへの参照です。

これにより、双方向連結リストは両方向にたどることができます。

<img src="https://cdn.freecodecamp.org/curriculum/lecture-transcripts/how-do-singly-linked-lists-work-and-how-do-they-differ-from-doubly-linked-list-9.webp" alt="doubly linked list structure visualization">

このタイプの連結リストでは、必要に応じて末尾からの探索を開始できるように、連結リスト自体にテイルノードへの参照を保持することも一般的です。

これは便利ですよね？単方向連結リストより柔軟です。

しかし、双方向連結リストは各ノードが**2つ**の参照を格納するため、単方向連結リストよりもメモリを多く使います。

これは、プロジェクトに適したデータ構造を選ぶ際に考慮すべきポイントです。

トレードオフがあります。

挿入と削除の操作はまったく同じですが、今度はノードごとに2つの参照を更新し、双方向連結リストの末尾に要素を効率よく挿入し、必要に応じて末尾から探索を開始するためにテイルノードへの参照を管理する必要があります。

単方向連結リストと双方向連結リストは、要素を順序通りに格納・操作するための重要なデータ構造です。違いを理解することは、特定の用途に適したものを選ぶために不可欠です。
