# --description--

Die Fetch-API ermöglicht es Webanwendungen, Netzwerk-Anfragen zu stellen, typischerweise um Daten vom Server abzurufen oder dorthin zu senden. Diese API stellt eine `fetch()`-Methode bereit, mit der Sie diese Anfragen ausführen können. Schauen wir uns ein einfaches Beispiel an, wie man `fetch` verwendet:

```js
fetch('https://api.example.com/data')
```

In diesem Beispiel führen wir eine `GET`-Anfrage an `https://api.example.com/data` aus. Diese liefert uns dann Daten zurück, die wir in das JSON-Format umwandeln müssen und überall verwenden können.

Standardmäßig verwendet die Fetch-API die `GET`-Methode, um Daten abzurufen. Dies wird in der nächsten Lektion behandelt, zusammen mit anderen gängigen HTTP-Anfragemethoden.

Kommen wir nun zu einigen häufigen Ressourcentypen, die aus dem Netzwerk abgerufen werden.

In unseren Webanwendungen benötigen wir häufig Daten wie Wetterdaten, Listen von Berufen, Länderliste, Ländercodes oder Länderflaggen-Icons/Bilder. Mit diesen Daten können wir unsere Anwendung informativer und interaktiver gestalten. Dank der Fetch-API können wir diese Ressourcen aus dem Netzwerk abrufen.

Bilder sind häufig abgerufene Ressourcen. Sie könnten fetch verwenden, um Bilder statisch oder dynamisch basierend auf Benutzeraktionen zu laden und sie in Ihrer Webanwendung anzuzeigen.

Textdateien sind ein weiterer Ressourcentyp, der oft abgerufen wird. Dazu können Konfigurationsdateien, Protokolldateien oder sogar ganze Dokumente gehören, die Sie auf Ihrer Webseite anzeigen möchten.

In manchen Fällen könnten Sie Audio- oder Videodateien abrufen. Die Fetch-API kann auch mit diesen Ressourcentypen umgehen, sodass Sie mit einer Vielzahl von Datentypen arbeiten können.
