# --interactive--

`@property` 규칙은 개발자가 애니메이션 방식과 초기 값을 포함해 동작을 더 세밀하게 제어할 수 있는 사용자 정의 속성을 정의할 수 있게 하는 강력한 CSS 기능입니다.

이 규칙은 CSS 사용자 정의 속성의 기능을 확장하고 적용에 더 많은 유연성을 제공합니다.

`@property` 규칙의 기본 구문은 다음과 같습니다:

```css
@property --property-name {
  syntax: '<type>';
  inherits: true | false;
  initial-value: <value>;
}
```

`--property-name`는 정의하려는 사용자 정의 속성의 이름입니다. 모든 사용자 정의 속성처럼 두 개의 대시로 시작해야 합니다.

`syntax`는 속성의 유형을 정의하며, `<color>`, `<length>`, `<number>`, `<percentage>` 또는 더 복잡한 유형이 될 수 있습니다.

`inherits`는 속성이 부모 요소로부터 값을 상속받을지 여부를 지정합니다.

`initial-value`는 속성의 기본값을 설정합니다.

다음은 `@property` 규칙을 사용하는 실제 예시입니다:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<button class="button">Click Me</button>
```

```css
@property --main-color {
  syntax: '<color>';
  inherits: false;
  initial-value: #3498db;
}

.button {
  background-color: var(--main-color);
}
```

:::

이 예시에서는 사용자 정의 속성 `--main-color`을 `color` 값으로 정의하고, 상속하지 않도록 설정하며, 초기 값을 `#3498db`으로 지정했습니다.

`@property` 규칙의 주요 장점 중 하나는 이전에는 불가능했던 사용자 정의 속성의 애니메이션을 허용한다는 점입니다:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<div class="gradient-box"></div>
```

```css
@property --gradient-angle {
  syntax: '<angle>';
  inherits: false;
  initial-value: 0deg;
}

.gradient-box {
  width: 100px;
  height: 100px;
  background: linear-gradient(var(--gradient-angle), red, blue);
  transition: --gradient-angle 0.5s;
}

.gradient-box:hover {
  --gradient-angle: 90deg;
}
```

:::

위 코드는 요소에 마우스를 올렸을 때 부드럽게 애니메이션되는 그라디언트를 생성하는데, 이는 표준 사용자 정의 속성으로는 불가능했습니다.

이제 `@property` 규칙이 대체 수단과 어떻게 작동하는지 살펴보겠습니다.

대체 수단은 특정 기능을 지원하지 않는 브라우저에서 스타일이 점진적으로 저하되도록 보장하는 데 매우 중요합니다. `@property`에서는 대체 수단이 두 가지 수준에서 작동합니다: 규칙 자체에 대한 대체 수단과 사용자 정의 속성 사용에 대한 대체 수단입니다.

`@property` 규칙을 지원하지 않는 브라우저를 위해 전통적인 방식으로 사용자 정의 속성을 선언하여 대체 수단을 제공할 수 있습니다:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
```

```css
:root {
  --main-color: #3498db;
}

@property --main-color {
  syntax: '<color>';
  inherits: false;
  initial-value: #3498db;
}

body {
  background-color: var(--main-color);
}
```

:::

이 경우 `@property`을 지원하는 브라우저는 더 엄격하게 정의된 버전을 사용하고, 그렇지 않은 브라우저는 표준 사용자 정의 속성 선언으로 대체됩니다.

사용자 정의 속성을 사용할 때는 표준 사용자 정의 속성처럼 `var()` 함수를 사용해 대체 값을 제공할 수 있습니다:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<button class="button">Click Me</button>
```

```css
.button {
  background-color: var(--main-color, #3498db);
}
```

:::

이렇게 하면 `--main-color` 속성이 정의되지 않았거나 유효하지 않아도 버튼에 배경색이 유지됩니다.

`@property` 규칙은 더 복잡한 대체 시나리오도 허용합니다. 예를 들어, 유형 안전 대체 수단을 제공할 수 있습니다:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<div class="box">Colored box</div>
```

```css
@property --padding {
  syntax: '<length>';
  inherits: false;
  initial-value: 10px;
}

.box {
  width: 100px;
  height: 100px;
  background-color: darkred;
  color: white;
  padding: var(--padding);
}
```

:::

이 경우 누군가 `--padding`에 색상 같은 잘못된 값을 설정하려 하면 브라우저가 `10px`의 초기 값으로 대체하여 유형 안전성을 유지합니다.

CSS `@property` 규칙은 2024년 초부터 Chrome, Edge, Opera, Firefox, Safari 같은 주요 최신 브라우저 엔진에서 널리 지원되는 것으로 간주됩니다. 이전 브라우저를 지원하려면 위에서 설명한 대체 CSS를 제공해야 합니다.

결론적으로 `@property` 규칙은 CSS 사용자 정의 속성을 다루는 방식에 큰 발전을 의미합니다. 더 많은 제어를 제공하고 사용자 정의 속성 애니메이션 같은 새로운 가능성을 열어줍니다. 하지만 신중하게 사용하고 모든 브라우저에서 좋은 경험을 보장하려면 적절한 대체 수단을 항상 제공하는 것이 중요합니다. 최신 웹 기술과 마찬가지로, 지원하는 브라우저에는 경험을 향상시키고 지원하지 않는 브라우저에는 기본 기능을 보장하는 것이 핵심입니다.
