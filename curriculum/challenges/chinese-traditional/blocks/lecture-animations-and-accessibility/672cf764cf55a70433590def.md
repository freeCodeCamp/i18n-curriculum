# --description--

動畫可以大幅提升網站的視覺吸引力和使用者體驗。然而，它們也可能對某些使用者造成重大存取挑戰。了解這些問題並實作解決方案，以確保你的網站對所有使用者皆可存取，是非常重要的。

動畫的主要無障礙性問題之一是它們可能會導致某些使用者感到不適甚至身體上的傷害。患有前庭疾病或對動作敏感的使用者，在螢幕上看到某些類型的移動時，可能會感到頭暈、噁心或頭痛。

此外，動畫可能會讓有認知障礙或注意力障礙的使用者分心。快速閃爍或頻閃效果尤其成問題。它們可能會觸發光敏性癲癇患者的癲癇發作。一般規則是避免任何每秒閃爍超過 3 次的內容。

另一個問題是動畫可能會讓某些使用者難以專注於或閱讀內容。這對於視力低下或有閱讀困難的使用者尤其如此，他們可能難以追蹤移動的文字或變動的版面配置。

為了解決這些問題，CSS 提供了 `prefers-reduced-motion` 媒體查詢。這個特性允許網頁開發者偵測使用者是否在系統層級要求最小化動畫或動態效果。

以下是你如何使用 `prefers-reduced-motion`：

```css
@media (prefers-reduced-motion: reduce) {
  * {
    animation-duration: 0.01ms !important;
    animation-iteration-count: 1 !important;
    transition-duration: 0.01ms !important;
    scroll-behavior: auto !important;
  }
}
```

這段 CSS 程式碼有效地為已顯示偏好減少動作的使用者停用大部分動畫和過渡效果。讓我們來拆解說明：

`@media` 查詢規則會檢查使用者是否偏好減少動態效果。如果為真，則套用所包含的樣式。

在媒體查詢中，我們針對所有元素（`*`）並覆寫動畫和過渡屬性。

我們將 `animation-duration` 和 `transition-duration` 設定為極小的值（`0.01ms`）。這基本上關閉了動畫，但仍允許它們完成，這對某些功能非常重要。

`animation-iteration-count: 1` 確保任何迴圈動畫只播放一次。

`scroll-behavior: auto` 停用平滑捲動效果。

`!important` 宣告用於確保這些規則優先於其他動畫樣式。

重要的是要注意，雖然這個方法有效地減少了動作，但它是一種通用的做法。若要更精確地控制，你可能想為你的動畫定義特定的減少動作替代方案。

以下是一個更具標的性的方法範例：

```css
.animated-element {
  transition: transform 0.3s ease-in-out;
}

@media (prefers-reduced-motion: reduce) {
  .animated-element {
    transition: none;
  }
}
```

在這種情況下，我們只會在偏好減少動作時停用特定元素的 `transition`。這讓你能為需要的使用者提供替代的、較少動作強度的體驗。

請記住，目標不是完全移除你網站上的所有動態效果，而是提供選項，可讓所有使用者舒適地與你的內容互動。即使是偏好減少動態效果的使用者，有些動態仍然對可用性和回授有益。

在實作動畫時，請謹慎使用，而非僅作為裝飾。避免大型且意外的移動，並在可能的情況下提供控制元件以暫停、停止或隱藏動畫。重要的是，使用 `prefers-reduced-motion` 查詢來提供低動作替代方案，確保你的內容對所有使用者（包括對動作敏感者）皆可存取且舒適。

透過注意這些存取性問題並利用像是 `prefers-reduced-motion` 這類工具，你可以創建引人入勝且包含所有使用者的動畫體驗。
