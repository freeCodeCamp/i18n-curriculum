# --description--

在 JavaScript 中，`Set` 是一個內建的物件，用於管理資料群集。它讓你能夠存放任何型別的唯一值，無論是原始型別還是物件引用。`Set` 確保其中的每個值只出現一次，這使它在從陣列中消除重複項目或處理不同值的群集時非常有用。

至於 `WeakSet`，它是一種特殊類型的 `Set`，功能較少，允許你存放弱引用的物件和符號。與 `Set` 不同，`WeakSet` 不支援像數字或字串這類的原始型別。

與一般的 `Set` 不同，`WeakSet` 只存放物件，且對這些物件的引用是「弱引用」，這表示 WeakSet 不會阻止被存放的物件在沒有其他引用時被垃圾回收。簡單來說，如果物件在你的程式碼中沒有被其他地方使用，它會自動被移除以釋放記憶體。

要創建 `Set`，你可以使用 `Set` 建構式並將其指定給一個變數：

```js
const myFirstSet = new Set();
```

你也可以用值來初始化 `Set`：

```js
const treeSet = new Set(["Baobab", "Jackalberry", "Mopane Tree", "Breadfruit"]);
```

如果你將 `Set` 輸出到主控台，結果會是這樣：

```js
/*
Set(4) {"Baobab", "Jackalberry", "Mopane Tree", "Breadfruit"}
  [[Entries]]
  0: "Baobab"
    value: "Baobab"
  1: "Jackalberry"
    value: "Jackalberry"
  2: "Mopane Tree"
    value: "Mopane Tree"
  3: "Breadfruit"
    value: "Breadfruit"
  size: 4
  [[Prototype]]: Set
*/
```

如果你沒有用值初始化 `Set`，你可以使用 `add()` 方法來新增項目到 `Set`：

```js
const treeSet = new Set();

// Add items to the treeSet
treeSet.add("Baobab");
treeSet.add("Jackalberry");
treeSet.add("Mopane Tree");
treeSet.add("Breadfruit");
```

結果和在主控台中顯示的樣子保持不變。

別忘了，重複的項目在 `Set` 中會被忽略：

```js
const treeSet = new Set();

// Add items to the treeSet
treeSet.add("Baobab");
treeSet.add("Jackalberry");
treeSet.add("Mopane Tree");
treeSet.add("Breadfruit");
treeSet.add("Baobab"); //duplicate item will be ignored

console.log(treeSet);
// Set(4) {"Baobab", "Jackalberry", "Mopane Tree", "Breadfruit"}
```

你還可以使用以下方法來操作 `Set`：

- `delete()`  
- `clear()`
- `has()`
- `entries()`
- `forEach()`
- `keys()`
- `values()`

讓我們逐一看看這些方法的運作。`delete()` 會從 `Set` 中移除指定的項目：

```js
const treeSet = new Set();

// Add items to the treeSet
treeSet.add("Baobab");
treeSet.add("Jackalberry");
treeSet.add("Mopane Tree");
treeSet.add("Breadfruit");

treeSet.delete("Breadfruit");

console.log(treeSet); // Set(3) {"Baobab", "Jackalberry", "Mopane Tree"}
```

`has()` 用來檢查指定的值是否存在於 `Set` 中：

```js
const treeSet = new Set();

// Add items to the treeSet
treeSet.add("Baobab");
treeSet.add("Jackalberry");
treeSet.add("Mopane Tree");
treeSet.add("Breadfruit");

treeSet.delete("Breadfruit");

console.log(treeSet.has("Breadfruit")); // false
```

`entries()` 傳回一個 `Set` 迭代器，該迭代器包含 `[value, value]` 格式的值陣列：

```js
const treeSet = new Set();

// Add items to the treeSet
treeSet.add("Baobab");
treeSet.add("Jackalberry");
treeSet.add("Mopane Tree");
treeSet.add("Breadfruit");

treeSet.delete("Breadfruit");

console.log(treeSet.entries());
// SetIterator {"Baobab" => "Baobab", "Jackalberry" => "Jackalberry", "Mopane Tree" => "Mopane Tree"}
```

`keys()` 和 `values()` 顯示 `Set` 中的值。`keys()` 只是 `values()` 方法的別名：

```js
const treeSet = new Set();

// Add items to the treeSet
treeSet.add("Baobab");
treeSet.add("Jackalberry");
treeSet.add("Mopane Tree");
treeSet.add("Breadfruit");

treeSet.delete("Breadfruit");

console.log("Keys: ", treeSet.keys());
console.log("Values: ", treeSet.values());
// Keys: SetIterator {"Baobab", "Jackalberry", "Mopane Tree"}
// Values: SetIterator {"Baobab", "Jackalberry", "Mopane Tree"}
```

`forEach()` 讓你可以迭代 `Set`：

```js
const treeSet = new Set();

// Add items to the treeSet
treeSet.add("Baobab");
treeSet.add("Jackalberry");
treeSet.add("Mopane Tree");
treeSet.add("Breadfruit");

treeSet.delete("Breadfruit");

treeSet.forEach((tree) => console.log(tree));
/*
Baobab
Jackalberry
Mopane Tree
*/
```

`clear()` 會移除陣列中的所有項目：

```js
const treeSet = new Set();

// Add items to the treeSet
treeSet.add("Baobab");
treeSet.add("Jackalberry");
treeSet.add("Mopane Tree");
treeSet.add("Breadfruit");

treeSet.delete("Breadfruit");

treeSet.clear();

console.log(treeSet); // Set(0) {size: 0}
```

值得一提的是，`size` 屬性會傳回 `Set` 中項目的數量：

```js
const treeSet = new Set();

// Add items to the treeSet
treeSet.add("Baobab");
treeSet.add("Jackalberry");
treeSet.add("Mopane Tree");
treeSet.add("Breadfruit");

treeSet.delete("Breadfruit");

console.log(treeSet.size); // 3
```

就像 `Set` 一樣，也有 `WeakSet` 建構式可以用來創建 `WeakSet`：

```javascript
const treeWeakSet = new WeakSet();
```

`WeakSet` 也有 `add()`、`delete()` 和 `has()` 方法：

```javascript
const treeWeakSet = new WeakSet();

const baobab = { name: "Baobab" };
const jackalberry = { name: "Jackalberry" };
const mopaneTree = { name: "Mopane Tree" };
const breadfruit = { name: "Breadfruit" };

treeWeakSet.add(baobab);
treeWeakSet.add(jackalberry);
treeWeakSet.add(mopaneTree);
treeWeakSet.add(breadfruit);

treeWeakSet.delete(jackalberry);
console.log(treeWeakSet.has(jackalberry)); // false

console.log(treeWeakSet);
```

輸出中，`WeakSet` 的內容會顯示成這樣：

```javascript
/*
WeakSet {{…}, {…}, {…}}
  [[Entries]]
    0: value: {name: "Mopane Tree"}
    1: value: {name: "Baobab"}
    2: value: {name: "Breadfruit"}
  [[Prototype]]: WeakSet
*/
```

這些內容之所以可見，是因為物件變數仍持有強引用，讓這些物件保留在記憶體中。請記住，WeakSet 不可迭代，因此無法用迴圈遍歷其條目或以程式方式存取。你在輸出中看到的只是偵錯輔助。

別忘了，只有符號和具有明確鍵值的物件受到支援。加入原始型別（例如數字或字串）會導致錯誤：

```js
treeWeakSet.add("Alan Smith");

console.log(treeWeakSet); // Invalid value used in weak set
//    at WeakSet.add (<anonymous>)
```

`Set` 和 `WeakSet` 之間的主要差異是，`Set` 可以存放任何值，而 `WeakSet` 只能存放物件。

以下是 `Set` 和 `WeakSet` 之間其他明顯的差異：

| 特性 | Set | WeakSet |
| --- | --- | --- |
| 存放值的型別 | 可存放任何資料型別 | 只存放物件 |
| 引用方式 | 強引用 | 弱引用 |
| 迭代支援 | 支援使用 `forEach` 和迴圈迭代 | 不支援迭代 |
| 方法和屬性 | `add()`、`delete()`、`has()`、`keys()`、`values()`、`size` 等 | 只有 `add()`、`delete()` 和 `has()` |
| 使用情境 | 一般用途的唯一值群集與陣列去重 | 物件引用的高效記憶體追蹤 |

你可以看到這兩種集合能存放的值型別差異、它們對迭代的支援，以及理想的使用情境。請花點時間閱讀這張表格的內容。
