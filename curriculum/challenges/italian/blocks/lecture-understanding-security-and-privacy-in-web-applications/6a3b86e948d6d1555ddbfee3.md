# --description--

La Same-Origin Policy è una misura di sicurezza integrata nei tuoi browser web che impedisce a un sito di leggere dati privati da un altro sito. Questa regola protegge i tuoi account personali dagli hacker.

Un'origine ha tre parti specifiche: il protocollo web, il nome di dominio e il numero di porta.

Per esempio, nell'indirizzo `https://example.com:443`, il protocollo è HTTPS, il nome di dominio è `example.com` e il numero di porta è `443`.

I browser spesso nascondono il numero di porta se è quello standard. HTTPS usa per impostazione predefinita la porta 443, e HTTP la porta 80. Anche se nascosta, il browser la controlla comunque. Due siti hanno la stessa origine solo quando tutte e tre queste parti corrispondono esattamente.

Secondo questa regola rigorosa, un browser blocca un sito che cerca di spiare un altro sito. Immagina di aprire il tuo conto bancario online in una scheda del browser e di aprire un sito malevolo in una seconda scheda. La Same-Origin Policy impedisce al sito malevolo di rubare le tue informazioni bancarie.

Tuttavia, i siti moderni spesso devono condividere dati tra origini diverse. Per esempio, una pagina frontend potrebbe dover prendere dati da un server backend separato. Le regole della Same-Origin nei browser bloccano queste richieste sicure, quindi gli sviluppatori avevano bisogno di un modo per condividere dati in modo sicuro. Qui entra in gioco CORS.

CORS è un sistema che permette ai server di aprire l'accesso attraverso le regole del browser. Usa intestazioni speciali per concedere l'accesso ai dati a siti affidabili.

Ecco cosa succede quando il tuo browser chiede dati a un altro sito:

* Include un'intestazione `Origin` che indica il sito che fa la richiesta
* Il server ricevente controlla questo nome
* Se il server si fida del sito, risponde con un'intestazione di permesso speciale che dice al browser che il sito può leggere i dati in arrivo.

Se il server non invia questo permesso, il browser blocca immediatamente i dati. Il sito frontend riceve allora un messaggio di errore invece dei file privati.

Nelle applicazioni Node.js che usano Express, CORS si implementa con il pacchetto `cors`:

```javascript
const express = require('express');
const cors = require('cors');
const app = express();

// this enables CORS for all origins and all routes for development
app.use(cors());
```

In produzione, potresti dover passare un oggetto di configurazione per limitare l'accesso a un dominio specifico:

```javascript
const corsOptions = {
  origin: ['https://yourapp.com', 'https://admin.yourapp.com'],
  credentials: true
};

app.use(cors(corsOptions));
```

Se usi Node.js puro, devi gestirlo manualmente impostando tu stesso le intestazioni di risposta CORS su ogni richiesta:

```javascript
res.setHeader('Access-Control-Allow-Origin', 'https://example.com');
res.setHeader('Access-Control-Allow-Methods', 'GET, POST, PUT, DELETE, OPTIONS');
res.setHeader('Access-Control-Allow-Headers', 'Content-Type, Authorization');
```
