# --description--

アプリケーションレベルのミドルウェアは、Expressアプリのインスタンスにバインドされるミドルウェアです。通常、複数のルートにまたがる処理が必要なリクエストを扱うために使われます。グローバルに適用することも、特定のルートにだけ適用することもできます。

このミドルウェアは`app.use()`や`app.get()`、`app.post()`などの特定のルートメソッドを使って登録されます。特定のルートに制限されていない限り、すべての受信リクエストに対して実行されます。

```js
const express = require('express')
const app = express()

// Application-level middleware that logs all incoming requests
app.use((req, res, next) => {
  console.log('Request received at:', new Date())
  next(); // Passes control to the next middleware
})

app.get('/', (req, res) => {
  res.send('Hello World!')
})

app.listen(3000, () => {
  console.log('Server running on http://localhost:3000')
})
```

* `app.use()`: これはアプリケーションレベルのミドルウェアです。ルートに関係なくサーバーへのすべてのリクエストをログに記録します。
* このミドルウェアはすべての受信リクエストに対して実行され、現在のタイムスタンプなどの詳細をサーバーに提供します。

アプリケーションレベルのミドルウェアは、アプリ内の多くまたはすべてのルートに適用する必要があるタスクに便利です。これにはログ記録、認証、エラー処理などが含まれます。
