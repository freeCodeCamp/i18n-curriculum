# --description--

Same-Origin Policyは、ウェブブラウザに組み込まれた安全対策で、あるウェブサイトが別のウェブサイトのプライベートデータを読み取るのを防ぎます。このルールは個人のアカウントをハッカーから守ります。

オリジンは3つの特定の部分で構成されます：ウェブプロトコル、ドメイン名、そしてポート番号です。

例えば、アドレス`https://example.com:443`では、プロトコルはHTTPS、ドメイン名は`example.com`、ポート番号は`443`です。

ブラウザは標準のポート番号の場合、それを隠すことがよくあります。HTTPSはデフォルトでポート443、HTTPはポート80です。隠されていてもブラウザはそれを確認します。3つの部分すべてが完全に一致した場合にのみ、2つのウェブサイトは同じオリジンとみなされます。

この厳格なルールのもと、ブラウザは異なるウェブサイトがのぞき見するのをブロックします。例えば、1つのブラウザタブでオンライン銀行口座を開き、別のタブで悪意のあるウェブサイトを開いたとします。Same-Origin Policyは悪意のあるウェブサイトが銀行情報を盗むのを防ぎます。

しかし、現代のウェブサイトは異なるオリジン間でデータを共有する必要がよくあります。例えば、フロントエンドのウェブページが別のバックエンドサーバーからデータを取得する場合です。ブラウザのSame-Originルールはこれらの安全なリクエストをブロックするため、開発者は安全にデータを共有する方法が必要でした。ここでCORSが登場します。

CORSは、サーバーがブラウザのルールを通じてアクセスを開放できる仕組みです。信頼できるサイトにデータアクセスを許可する特別なヘッダーを使います。

ブラウザが別のサイトにデータを要求するときに起こることは次の通りです：

* リクエストを行うウェブサイトの名前を示す`Origin`ヘッダーを含めます
* 受け取ったサーバーはこの名前を確認します
* サーバーがウェブサイトを信頼すれば、ブラウザにそのサイトが受信データを読み取ることを許可する特別な許可ヘッダーで応答します。

サーバーがこの許可を送らない場合、ブラウザは即座にデータをブロックします。フロントエンドのウェブサイトはプライベートファイルの代わりにエラーメッセージを受け取ります。

Node.jsのExpressを使ったアプリケーションでは、CORSは`cors`パッケージで実装されます：

```javascript
const express = require('express');
const cors = require('cors');
const app = express();

// this enables CORS for all origins and all routes for development
app.use(cors());
```

本番環境では、特定のドメインへのアクセスを制限するために設定オブジェクトを渡す必要がある場合があります：

```javascript
const corsOptions = {
  origin: ['https://yourapp.com', 'https://admin.yourapp.com'],
  credentials: true
};

app.use(cors(corsOptions));
```

生のNode.jsを使う場合は、すべてのリクエストでCORSレスポンスヘッダーを自分で設定して手動で対応する必要があります：

```javascript
res.setHeader('Access-Control-Allow-Origin', 'https://example.com');
res.setHeader('Access-Control-Allow-Methods', 'GET, POST, PUT, DELETE, OPTIONS');
res.setHeader('Access-Control-Allow-Headers', 'Content-Type, Authorization');
```
