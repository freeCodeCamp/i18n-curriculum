# --interactive--

在 JavaScript 中，變數充當用於儲存你可以在整個程式中存取和修改的資料的容器。

你可以將變數想像成裝有值的盒子。使用變數，你可以追蹤像是數字或文字這類的東西，並在你的程式中需要時取用這些值。

在 JavaScript 中宣告變數的一種方法是使用 `let` 關鍵字。你將在未來的課程中學習更多關於 `let` 關鍵字以及其他宣告變數的方法。

以下是使用 `let` 宣告名為 `age` 的變數的範例：

```js
let age;
```

目前，`age` 變數尚未被指定值。如果你嘗試使用它，它會傳回 `undefined`，表示它沒有值。

這裡是一個範例。 

**注意**：`console.log()` 是一個將資訊輸出到主控台的函式（程式），主控台是你的網頁瀏覽器中用於偵錯程式碼的零件。你將在未來的課程中學習更多關於 `console.log()` 的內容。此外，`//` 符號用於在你的程式碼中添加註解。註解是給你自己或其他程式設計師的備註，當程式執行時會被忽略。

:::interactive_editor
 
```js
let age;
console.log(age); // undefined
```

:::

要將值指定給變數，你需要像這樣使用指定運算子：

```js
let age = 25;
```

現在當你使用 `age` 變數時，它會傳回 `25` 的值。

:::interactive_editor

```js
let age = 25;
console.log(age); // 25
```

:::

指定運算子看起來像等號（`=`），但它不會檢查相等性。你將在未來的課程中學習用於檢查相等性的正確運算子。

指定運算子用於將值指定給變數。將值指定給變數的這個過程稱為初始化。

使用 `let` 關鍵字宣告變數的一個優點是你可以重新指定它們的值。在程式設計中，重新指定是指給已經有值的變數賦予新值。

這裡是一個重新指定 `age` 變數值的範例。

:::interactive_editor

```js
let age = 25;
console.log(age); // 25
age = 30;
console.log(age); // 30
```

:::

現在 `age` 變數持有 `30` 的值。請注意，因為 `age` 變數已經宣告過，所以不需要再次使用 `let` 關鍵字宣告。

當使用重新指定時，你只需要取用變數名稱。重新指定很有用，因為它允許你在程式執行時更新並更改儲存在變數中的值。一個很好的例子是更新遊戲中的分數。

命名變數看似簡單，但有一些規則和最佳實踐可確保你的程式碼易讀且具功能性。

你的變數名稱應該描述資料表現的內容。例如，與其使用像 `x` 這樣的名稱，不如使用更具描述性的名稱，如 `age` 或 `points`，這會讓你的程式碼更容易理解。

```js
// Bad variable names
let x = 10;
let y = "John";

// Good variable names
let age = 10;
```

JavaScript 中的變數必須以字母、底線（`_`）或美元符號（`$`）開頭。它們不能以數字開頭。

```js
// Valid variable names
let age;
let _score;
let $total;

// Invalid variable names
let 1stPlace; // starts with a number
```

變數名稱區分大小寫，這表示全小寫的字 `age` 和首字母大寫的字 `Age` 被視為不同的變數。

:::interactive_editor

```js
let age = 25;
let Age = 30;
console.log(age); // 25
console.log(Age); // 30
```

:::

這就是為什麼堅持使用像 camelCase 這樣一致的命名慣例很重要。camelCase 是指第一個字全部小寫，後續每個字的第一個字母皆為大寫。

這裡是一個使用駝峰式命名法為變數命名的範例：

```js
let thisIsCamelCase;
let anotherExampleVariable;
let freeCodeCampStudents;
```

JavaScript 中有某些關鍵字你不能用作變數名稱，例如 `let`、`const`、`function` 或 `return`，因為它們是語言本身保留的。

你也應該避免在你的變數名稱中使用像驚嘆號（`!`）或 at（`@`）符號等特殊字元。最好使用字母、數字、底線或美元符號來保持變數名稱的可讀性。

遵循這些指引後，隨著程式碼複雜度的增加，你的程式碼將會更乾淨且更易於管理。
