## JavaScript do Lado do Cliente

- **JavaScript do Lado do Cliente**: Código que é executado no lado do cliente de uma aplicação web, geralmente em um navegador web.
- **Limitações do JavaScript do lado do cliente**: Embora poderoso, o JavaScript do lado do cliente tem certas limitações:
  - Acesso muito restrito a arquivos locais.
  - Não projetado para lidar com lógica complexa de aplicação.
  - Preocupações potenciais de segurança (por exemplo, expor informações sensíveis como credenciais de banco de dados).
- **Razão para as limitações**: Essas limitações existem porque o JavaScript do lado do cliente foi inicialmente projetado para rodar exclusivamente em navegadores web.

## Ambiente do Navegador

- **JavaScript Engine**: Navegadores web fornecem o ambiente necessário para executar código JavaScript, incluindo o JavaScript Engine.
- **Document Object Model (DOM)**: Os navegadores fornecem acesso ao DOM, permitindo que você acesse elementos HTML no seu código.
- **Contexto Histórico**: Anteriormente, JavaScript não podia ser executado fora de um navegador, mas agora pode com Node.js.

## O que é Node.js?

- **Node.js**: Um ambiente de execução JavaScript. Definido oficialmente como "um ambiente de execução JavaScript open-source e multiplataforma."
- **Código Aberto**: o código do Node.js é publicamente disponível e mantido por uma grande comunidade de desenvolvedores.
- **Cross-Platform**: Funciona em qualquer sistema operacional, incluindo Windows, macOS e Linux.
- **Ambiente de Execução JavaScript**: Permite que você execute código JavaScript fora de um navegador.

## Casos de Uso do Node.js

- **Aplicações Comuns**: Node.js é usado em uma ampla variedade de indústrias para:
  - Construindo servidores web e APIs que lidam com requisições HTTP.
  - Desenvolvendo aplicações web e móveis.
  - Manipulação de dados complexos e interações com banco de dados.
- **Adoção na Indústria**: Se você está usando ou desenvolvendo um site ou aplicação web que interage com um banco de dados ou lida com dados complexos, há uma grande chance de que você use Node.js nos bastidores.
- **Características Principais**: Node.js é eficiente, escalável e possui uma grande comunidade de desenvolvedores e mantenedores ao redor do mundo.

## Ambiente de Execução Browser vs. Node

- **Ambiente de Navegador**: Projetado principalmente para desenvolvimento web front-end, executa JavaScript do lado do cliente.
- **Node Runtime Environment**: Principalmente projetado para desenvolvimento web back-end, executa JavaScript no lado do servidor.
- **Diferenças de API**: O ambiente determina as APIs disponíveis:
  - **Browser**: Acesso à API DOM, mas com restrições para acessar o sistema de arquivos local.
  - **Node**: Acesso a quase todos os recursos do sistema, incluindo o sistema de arquivos, mas não ao DOM.

## Objetos Globais

- **Objeto Global do Navegador (`window`)**: Fornece acesso a funcionalidades relacionadas ao navegador:
  - Métodos para manipular o DOM.
  - Gerenciando cookies.
  - Manipulando eventos do navegador.

- **Objeto Global do Node.js (`global`)**: Fornece acesso a funcionalidades específicas do Node.js:
  - Módulos integrados para trabalhar com arquivos locais.
  - Capacidades de rede.
  - Interagindo com o sistema operacional.

## Controle de Versão

- **Versão do Navegador**: Você não tem controle sobre a versão do ambiente do navegador que seus usuários usarão para visitar seu site.
- **Versão do Node.js**: Você pode escolher a versão do Node.js que deseja usar no seu servidor.

## Importância do Node.js

- **Desenvolvimento Full-Stack**: Node.js transformou o desenvolvimento web ao permitir que desenvolvedores usem JavaScript para construir tanto o frontend e o backend de aplicações full-stack.
- **Eficiência**: O processo de desenvolvimento é mais eficiente porque os desenvolvedores não precisam aprender uma nova linguagem de programação apenas para desenvolver o backend.
- **Impacto**: Node.js é uma ferramenta poderosa, escalável e versátil que se tornou muito importante para desenvolvedores em todo o mundo.

## Vantagens do Node.js

- **Linguagem Única para Full-Stack**: Você pode implementar tanto o front-end quanto o back-end de uma aplicação web usando JavaScript.
  - Reduz a curva de aprendizado inicial.
  - Não é necessário alternar entre linguagens de programação.
  - Aumenta a eficiência e a produtividade.

- **Arquitetura Não Bloqueante e Orientada a Eventos**: Excelente para desenvolver aplicações em tempo real onde a capacidade de resposta e a eficiência são essenciais para criar uma boa experiência do usuário.

- **Single Thread e Event Loop**: Pode lidar efetivamente com um grande número de requisições simultâneas e operações de Entrada-Saída.
  - Perfeito para aplicações que exigem o tratamento de múltiplas requisições simultaneamente.

- **Grande Comunidade**: Milhares de desenvolvedores ao redor do mundo mantêm constantemente o Node.js e adicionam novos recursos.
  - Muitos recursos de aprendizado disponíveis.
  - Fácil encontrar respostas para perguntas.

- **npm (Node Package Manager)**: Uma ferramenta poderosa que permite instalar e gerenciar pacotes e módulos para seus projetos.
  - Pacotes disponíveis para muitos propósitos: manipulação de requisições HTTP, trabalho com bancos de dados, formatação de datas e horas, teste de código e mais.
  - Reutilize código escrito, testado e compartilhado por outros desenvolvedores.
  - Torna o fluxo de trabalho mais rápido e eficiente.

- **Custo-Benefício**: Node.js é gratuito e de código aberto.
  - Pacotes, bibliotecas e ferramentas associadas também são principalmente gratuitas.
  - Fator importante ao otimizar o orçamento.

## Desvantagens do Node.js

- **Limitação de Thread Única**: Executa apenas uma thread por vez, o que significa que só pode lidar com uma operação por vez.
  - Embora a arquitetura permita o manuseio eficiente de requisições concorrentes (simultâneas), tarefas intensivas em CPU podem bloquear a thread principal e resultar em problemas de desempenho.
  - Exemplos de tarefas intensivas para CPU: operações matemáticas complexas, processamento de imagem e vídeo e criptografia.
  - Existem maneiras de superar isso, mas geralmente aumentam a complexidade da aplicação.

- **Programação Assíncrona**: Node.js depende de programação assíncrona.
  - Uma tarefa que pode levar muito tempo para ser executada é iniciada, mas em vez de esperar até que seja concluída, o programa principal continua executando enquanto a tarefa assíncrona roda em paralelo.
  - Quando a tarefa é concluída, o programa processa o resultado.
  - Frequentemente envolve "callbacks", que são funções que definem o que acontece quando operações assíncronas são concluídas.
  - Pode potencialmente tornar o código mais difícil de ler, entender e depurar.

- **Preocupações com a Qualidade do Pacote**: Ao escolher pacotes do npm, alguns podem não ser constantemente mantidos e podem introduzir vulnerabilidades em sua aplicação.
  - Os desenvolvedores devem avaliar cada pacote cuidadosamente.
  - Verifique se os pacotes seguem as melhores práticas de qualidade e segurança.

## Threads

- **Thread**: Um caminho de execução dentro de um processo, como um programa de computador que está sendo executado no momento.

## NVM (Gerenciador de Versão do Node)

- **NVM**: Uma ferramenta para gerenciar múltiplas versões do Node.js no mesmo computador.
- **Propósito**: Ajuda você a alternar facilmente entre diferentes versões do Node.js para diferentes projetos.
- **Recomendação**: Recomenda-se instalar o NVM antes do Node.js para flexibilidade futura.

## Instalando o NVM no macOS/Linux

- **Xcode Command Line Tools (macOS)**: Deve ser instalado manualmente antes de executar o script de instalação do NVM no macOS.

```bash
xcode-select --install
```

- **Verifique a Instalação do Xcode**:

```bash
xcode-select --version
```

- **Baixar e Executar o Script de Instalação do NVM**: Execute um destes comandos:

```bash
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.3/install.sh | bash
```

```bash
wget -qO- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.3/install.sh | bash
```

- **Reinício do Terminal**: Pode ser necessário reiniciar seu terminal quando o processo for concluído para carregar o novo comando e a configuração.

- **Verifique a Instalação do NVM**:

```bash
nvm -v
```

## Instalando Node.js com NVM no macOS/Linux

- **Instalar a Versão LTS Mais Recente**: Para instalar a versão mais recente de Long-Term Support (LTS) do Node.js:

```bash
nvm install --lts
```

- **LTS (Long-Term Support)**: Esta versão geralmente é recomendada porque prioriza estabilidade, confiabilidade e segurança em vez de recursos novos ou experimentais.
  - Testado minuciosamente e pronto para produção.
  - Garantido ter períodos de suporte mais longos com foco em correções de bugs e patches de segurança.

- **Instalar Versão Específica**: Você pode instalar uma versão específica do Node.js especificando o número da versão:

```bash
nvm install <version>
```

Exemplo:

```bash
nvm install 20
```

- **Mudar para uma Versão**:

```bash
nvm use <version>
```

Exemplo:

```bash
nvm use 20
```

- **Verificar a Versão Atual do Node.js**:

```bash
node -v
```

## Instalando o NVM no Windows

- **Suporte Oficial do NVM**: Versões oficiais do NVM suportam Windows apenas em casos específicos.
  - Se você tem o Windows Subsystem for Linux (WSL), pode seguir o guia para macOS/Linux acima.

- **nvm-windows**: Uma ferramenta de gerenciamento de versões do Node.js para Windows.
  - Projeto completamente separado do projeto oficial NVM.
  - Mesmo propósito: ajudar você a trabalhar com múltiplas versões do Node.js no mesmo dispositivo.

- **Antes de Instalar**: Recomenda-se desinstalar qualquer instalação anterior do Node antes de instalar o NVM para Windows.

- **Passos de Instalação**:
  - Vá para a seção Releases do repositório oficial: `https://github.com/coreybutler/nvm-windows/releases`.
  - Baixe o instalador mais recente.
  - Siga as instruções.

- **Reinício do Terminal**: Pode ser necessário reiniciar seu terminal se o comando `nvm` não estiver disponível imediatamente.

- **Verificar Instalação**:

```powershell
nvm -v
```

## Comandos básicos do NVM e Node.js

- **Nota sobre Comandos**: Todos os comandos funcionam para NVM instalado no macOS/Linux, incluindo WSL. Eles podem funcionar de forma diferente ou podem não funcionar de forma alguma para `nvm-windows`. Comandos alternativos para `nvm-windows` são fornecidos quando possível.

### Política de Execução do PowerShell (Windows)

- **Política de Execução Padrão**: A política de execução padrão do PowerShell é `Restricted`, que impede a execução de scripts como uma medida de segurança.

- **Execute o PowerShell como Administrador**: Clique com o botão direito no PowerShell e depois clique em "Run as Administrator".

- **Verificar Política de Execução Atual**:

```powershell
Get-ExecutionPolicy
```

- **Definir Política de Execução para RemoteSigned**: Permite que scripts locais e `npm` sejam executados:

```powershell
Set-ExecutionPolicy RemoteSigned
```

- **Após definir a política**: Feche a janela do administrador do PowerShell e abra uma janela do PowerShell sem privilégios de administrador.

### Listar Todas as Versões Instaladas

- **Listar Versões Instaladas**: Para listar todas as versões instaladas do Node.js e verificar qual está sendo usada atualmente:

```bash
nvm ls
```

### Definir Versão Padrão do Node.js (NVM apenas no macOS/Linux)

- **Definir Versão Padrão**: Define a versão padrão do Node.js para novas sessões de terminal:

```bash
nvm alias default <version>
```

Exemplo:

```bash
nvm alias default 22.20.0
```

- **Criar Aliases Personalizados**: Atribua um nome mais específico a uma versão:

```bash
nvm alias <new_name> <version>
```

Exemplo:

```bash
nvm alias new-api 18.20.8
```

- **Usar Alias Personalizado**:

```bash
nvm use <new_name>
```

Exemplo:

```bash
nvm use new-api
```

### Remover uma Versão do Node.js

- **Desinstalar uma Versão**:

```bash
nvm uninstall <version>
```

Exemplo:

```bash
nvm uninstall 20.19.5
```

- **Remover por Versão Principal (NVM no macOS/Linux)**: Você pode remover uma versão executando o comando `uninstall` com uma versão principal como `22` ou `20`:

```bash
nvm uninstall 20
```

Se você tiver várias versões para essa versão principal instaladas, como `20.14.0` e `20.12.2`, a versão mais recente será removida primeiro. Executar `nvm uninstall 20` removerá `20.14.0` primeiro.

### Verificar Versão do Node.js

- **Verificar Versão Atual**:

```bash
node -v
```

### Verificar versão do npm

- **npm**: Um gerenciador de pacotes para Node.js usado para instalar, publicar e gerenciar pacotes de software. Instalado automaticamente quando você instala o Node.js.

- **Verificar Versão do npm**:

```bash
npm -v
```

### Executar um arquivo JavaScript

- **Executar Arquivo JavaScript**: Executa o arquivo JavaScript no runtime Node.js ativo no momento:

```bash
node <file.js>
```

Exemplo:

```bash
node app.js
```

### Criar um Projeto Node.js

- **Inicializar Projeto**: Cria um novo projeto Node.js criando um arquivo `package.json` que rastreia seus detalhes e dependências:

```bash
npm init
```

- **Configuração Interativa**: O comando `npm init` guia você por um assistente interativo de linha de comando para configurar o arquivo `package.json`.
  - Faz perguntas para nome do projeto, descrição, versão, etc.

- **Configuração Padrão**: Para começar com um arquivo `package.json` padrão:

```bash
npm init -y
```

ou

```bash
npm init --yes
```

### Instalar um Pacote

- **Instalar Pacote Específico**:

```bash
npm install <package>
```

Exemplo:

```bash
npm install express
```

### Instalar Todas as Dependências

- **Instalar Dependências**: Para instalar todas as dependências listadas no arquivo `package.json` de um projeto:

```bash
npm install
```
