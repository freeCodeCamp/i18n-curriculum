# --description--

React コンポーネントには、コンポーネントのライフサイクルの特定の時点でアクションを実行する機会を提供するいくつかの特別なメソッドがあります。これらはライフサイクルメソッド、またはライフサイクルフックと呼ばれ、コンポーネントを特定のタイミングで捕まえることができます。これはレンダリングされる前、更新される前、プロップスを受け取る前、アンマウントされる前などです。以下は主なライフサイクルメソッドのリストです： `componentWillMount()` `componentDidMount()` `shouldComponentUpdate()` `componentDidUpdate()` `componentWillUnmount()` 次のいくつかのレッスンでは、これらのライフサイクルメソッドの基本的な使用例を扱います。

**注意：** `componentWillMount` ライフサイクルメソッドは 16.X の将来のバージョンで非推奨となり、バージョン 17 で削除されます。詳細はこの<a href="https://www.freecodecamp.org/news/how-to-safely-use-reacts-life-cycles-with-fiber-s-async-rendering-fd4469ebbd8f/" target="_blank" rel="noopener noreferrer nofollow">記事</a>をご覧ください。

# --instructions--

`componentWillMount()` メソッドは、コンポーネントが DOM にマウントされる際に `render()` メソッドの前に呼び出されます。`componentWillMount()` 内で何かをコンソールにログ出力してください。出力を見るためにブラウザのコンソールを開いておくことをお勧めします。

# --hints--

`MyComponent` は `div` 要素をレンダリングする必要があります。

```js
assert(
  (function () {
    const mockedComponent = Enzyme.mount(React.createElement(MyComponent));
    return mockedComponent.find('div').length === 1;
  })()
);
```

`console.log` は `componentWillMount` 内で呼び出されるべきです。

```js
assert(
  (function () {
    const lifecycle = React.createElement(MyComponent)
      .type.prototype.componentWillMount.toString()
      .replace(/ /g, '');
    return lifecycle.includes('console.log(');
  })()
);
```
