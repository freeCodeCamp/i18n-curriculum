# --description--

HTTPS означає Hypertext Transfer Protocol Secure. Це захищена версія HTTP.

Як ви вже дізналися з кількох попередніх лекцій, HTTP передає дані між вашим браузером і вебсайтом. Стандартний HTTP надсилає ці дані у відкритому тексті, тому будь-хто може перехопити цей відкритий текст.

З цієї причини HTTPS використовує криптографічний протокол під назвою Transport Layer Security (TLS) для захисту цієї передачі даних. TLS шифрує канал зв’язку між двома точками. Старіші системи використовували Secure Sockets Layer (SSL), який досі застосовують багато платформ вебхостингу.

Шифрування перетворює ці дані на нерозбірливий код, який можуть розшифрувати лише браузер користувача та вебсервер, що мають ключі.

Наприклад, коли ви вводите пароль на вебсайті, HTTPS миттєво його шифрує і захищає інформацію про кредитні картки, особисті адреси, облікові дані для входу та іншу конфіденційну інформацію. Хакери, які перехоплюють ці дані, бачать лише випадкові символи, тому не можуть використати викрадену інформацію.

Цей захист життєво важливий у публічних Wi-Fi мережах. Відкриті Wi-Fi мережі в кав’ярнях, аеропортах та інших громадських місцях не мають базового захисту, що дає кіберзлочинцям можливість стежити за незашифрованим трафіком у цих мережах. HTTPS створює приватний тунель через публічну мережу, щоб ваша історія переглядів була захищена від зловмисників поруч.

HTTPS також забезпечує автентифікацію, що доводить легітимність вебсайту. Щоб використовувати HTTPS, власник сайту повинен отримати цифровий сертифікат, який видає довірена організація. Сертифікат підтверджує справжню особу власника сайту.

Коли ви заходите на сайт, і адреса починається лише з "http", це один із способів дізнатися, що власник сайту не отримав цей сертифікат, через що сайт небезпечний. Сучасні браузери також попереджають вас про такі сайти.

Нарешті, цілісність даних — ще одна ключова перевага HTTPS. Це означає, що ніхто не може змінити ваші дані під час передачі. Хакер не може впровадити шкідливий код у файли сайту, поки вони передаються на ваш пристрій. Ви отримуєте саме ті дані, які надіслав власник сайту.

Пошукові системи, як-от Google, надають пріоритет сайтам з HTTPS, оскільки безпечні сайти займають вищі позиції у результатах пошуку. Сучасні браузери також вимагають HTTPS для розширених функцій, таких як геолокація, push-повідомлення та вебкамери.
