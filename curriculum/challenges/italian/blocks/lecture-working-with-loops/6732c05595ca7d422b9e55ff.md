# --interactive--

Un loop `for...in` è più adatto quando devi iterare sulle proprietà di un oggetto. Questo loop itera su tutte le proprietà enumerabili di un oggetto, incluse le proprietà ereditate e quelle non numeriche.

Una proprietà ereditata è una proprietà che proviene dalla catena di prototipi dell’oggetto. Una proprietà non numerica è una proprietà che non è un numero o una stringa che può essere convertita in numero.

Ecco la sintassi base di un loop `for...in`:

```js
for (variable in object) {
  // code block to be executed
}
```

La variabile nell’esempio rappresenta la proprietà corrente dell’oggetto su cui si sta iterando.

Diamo un’occhiata a qualche esempio per capire meglio come funziona il loop `for...in`.

Nel primo esempio abbiamo un oggetto `fruit` e vogliamo iterare su ogni proprietà e stampare il valore nella console.

:::interactive_editor

```js
const fruit = {
  name: 'apple',
  color: 'red',
  price: 0.99
};

for (const prop in fruit) {
  console.log(fruit[prop]);
}
```

:::

La variabile `prop` rappresenta la proprietà corrente dell’oggetto. `fruit[prop]` viene usato per consultare il valore di ogni proprietà.

Alla prima iterazione, `prop` sarà `name`. Alla seconda iterazione, `prop` sarà `color`, e così via.

I risultati stampati nella console saranno `apple`, `red` e `0.99`.

Nel secondo esempio abbiamo un oggetto annidato e vogliamo iterare su ogni proprietà e stampare il valore nella console.

:::interactive_editor

```js
const person = {
  name: 'John',
  age: 30,
  address: {
    street: '123 Main St',
    city: 'Anytown',
    state: 'CA'
  }
};

for (const prop in person) {
  console.log(person[prop]);
}
```

:::

La proprietà `address` è essa stessa un oggetto. Il loop `for...in` itera anche sulle proprietà dell’oggetto `person` e stampa l’intero oggetto `address` nella console.

Ecco come appariranno i risultati nella console:

```md
John
30
{ street: '123 Main St', city: 'Anytown', state: 'CA' }
```

Se vuoi iterare sulle proprietà dell’oggetto `address`, puoi annidare un altro loop `for...in` dentro il primo.

:::interactive_editor

```js
const person = {
  name: 'John',
  age: 30,
  address: {
    street: '123 Main St',
    city: 'Anytown',
    state: 'CA'
  }
};

function isObject(obj) {
  return typeof obj === 'object' && !Array.isArray(obj) && obj !== null;
}

for (const prop in person) {
  if (isObject(person[prop])) {
    for (const nestedProp in person[prop]) {
      console.log(person[prop][nestedProp]);
    }
  } else {
    console.log(person[prop]);
  }
}
```

:::

In questo esempio abbiamo una funzione personalizzata `isObject` che verifica se il valore è un oggetto.

Il metodo `Array.isArray` viene usato per verificare se il valore è un array. Mettendo l’operatore logico NOT (`!`) davanti al metodo, stiamo controllando se il valore non è un array.

Il motivo per cui non possiamo semplicemente usare `typeof` uguale a `'object'` è che gli array sono considerati oggetti anche in JavaScript. Vogliamo escludere gli array dal controllo.

Inoltre, a causa di un bug storico in JavaScript, `typeof null` restituisce `'object'`. Quindi vogliamo anche escludere i valori `null` dal controllo.

Se la condizione è vera, annidiamo un altro loop `for...in` che itera sulle proprietà dell’oggetto annidato e stampa il valore nella console.

La variabile `nestedProp` rappresenta la proprietà corrente dell’oggetto annidato.

Ecco come appariranno i risultati modificati nella console:

```md
"John"
30
"123 Main St"
"Anytown"
"CA"
```

Un loop `for...in` è utile quando devi iterare sulle proprietà di un oggetto.

Non è consigliato usare un loop `for...in` per iterare sugli elementi di un array. Invece, usa un loop `for...of` o altri metodi per array come `forEach`, `map`, `filter` e `reduce`, che imparerai nelle lezioni future.
