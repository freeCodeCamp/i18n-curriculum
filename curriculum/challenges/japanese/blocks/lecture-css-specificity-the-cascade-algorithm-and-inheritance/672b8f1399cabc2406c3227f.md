# --interactive--

CSSの`!important`キーワードは、スタイルルールに最高の優先度を与え、プロパティの他の宣言をすべて上書きできるようにします。

これを使うと、他のセレクターの特異性に関係なく、指定したスタイルがブラウザーで適用されます。

例えば、次のようにインラインスタイルが設定されたHTMLの段落要素があるとします。

:::interactive_editor

```html
<p class="para" style="background-color: lightblue; color: black;">
  This is a paragraph.
</p>
```

:::

この例では、段落要素に背景色が`lightblue`、文字色が`black`に設定されています。

CSSファイルは段落要素に次のスタイルを適用しています。

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<p class="para" style="background-color: lightblue; color: black;">
  This is a paragraph.
</p>
```

```css
.para {
  background-color: black;
  color: white;
}
```

:::

インラインスタイルはクラス、ID、型セレクターよりも優先度が高いため、その段落要素には黒い背景色と白い文字色は適用されません。

これらのインラインスタイルを上書きするには、次のように`!important`キーワードを使います。

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<p class="para" style="background-color: lightblue; color: black;">
  This is a paragraph.
</p>
```

```css
.para {
  background-color: black !important;
  color: white !important;
}
```

:::

`!important`キーワードはCSSの値の後、セミコロンの前に使います。

これで段落要素には黒と白の色が適用されます。インラインスタイルが`!important`キーワードの使用で上書きされているためです。

CSSの`!important`キーワードは、スタイルルールに最高の優先度を与え、特異性が高いものやインラインスタイルを含む他の宣言を効果的に上書きします。

ただし、`!important`キーワードはCSSセレクター自体の特異性を変えません。単に`!important`が付いたルールが、特異性が高い他の競合ルールよりも適用されることを保証します。

`!important`キーワードのもう一つの適切な使い方は、元のCSSを制御できないサードパーティのライブラリやフレームワークのスタイルを上書きする場合です。

しかし、`!important`キーワードを多用すると、スタイルの自然なカスケードが壊れ、予期しない結果を招くため、CSSの保守やデバッグが難しくなります。

そのため、`!important`キーワードは控えめに使うのがベストです。
