# --description--

Tornando alla sezione sulla sicurezza informatica, ricorderai che memorizzare le password in chiaro non va *mai* bene. Ora è il momento di implementare BCrypt per risolvere questo problema.

`bcrypt@~5.0.0` è già stata aggiunta come dipendenza, quindi importala nel tuo server. Dovrai gestire l'hashing in 2 aree chiave: dove gestisci la registrazione/salvataggio di un nuovo account e quando verifichi che una password sia corretta al login.

Attualmente, nella tua rotta di registrazione, inserisci la password in chiaro dell'utente nel database così: `password: req.body.password`. Invece, esegui l'hashing delle password aggiungendo quanto segue prima della logica del database: `const hash = bcrypt.hashSync(req.body.password, 12);`, e sostituisci `req.body.password` nel salvataggio del database con solo `password: hash`.

Nella tua strategia di autenticazione, controlli quanto segue nel codice prima di completare il processo: `if (password !== user.password) return done(null, false);`. Dopo aver fatto le modifiche precedenti, ora `user.password` è un hash. Prima di modificare il codice esistente, nota come l’istruzione stia controllando se la password **non** è uguale e in tal caso ritorna non autenticato. Tenendo questo a mente, modifica quel codice in modo che appaia così per controllare correttamente la password inserita rispetto all’hash:

```js
if (!bcrypt.compareSync(password, user.password)) { 
  return done(null, false);
}
```

Questo è tutto ciò che serve per implementare una delle funzionalità di sicurezza più importanti quando devi memorizzare password.
