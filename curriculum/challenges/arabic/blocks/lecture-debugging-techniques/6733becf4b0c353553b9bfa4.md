# --description--

جملة `debugger` هي أداة قوية في JavaScript تتيح لك إيقاف شفرتك عند سطر معين للتحقق مما يحدث في البرنامج. عند استخدامها بشكل صحيح، توفر جملة `debugger` الكثير من الوقت لمحاولة معرفة سبب عدم عمل شيء كما ينبغي.

ينفذ JavaScript شفرتك من الأعلى إلى الأسفل. وعندما يصادف تنفيذ الشفرة جملة `debugger`، يتوقف التنفيذ فورًا عند ذلك السطر. هذا يمنحك فرصة لفحص المتغيرات، والتحقق من الدوال، وتتبع تدفق الشفرة بشكل عام.

بهذا يمكنك رؤية ما يحدث بشكل صحيح أو خاطئ بالضبط. لكن هذا يحدث فقط إذا كانت أدوات المطوِّر في المتصفح مفتوحة. وإلا، تُتجاهل جملة `debugger` ويستمر تنفيذ الشفرة كالمعتاد.

بالإضافة إلى ذلك، عند استخدام جملة `debugger`، توقف المتصفحات الحديثة تنفيذ الشفرة عند السطر المحدد. كما تسمح لك باستئناف التنفيذ بالنقر على زر التشغيل، لكن الصفحة نفسها لا تُعاد تحميلها تلقائيًا إلا إذا تم تفعيل ذلك يدويًا.

لاستخدام جملة `debugger`، ضعها في السطر الذي تريد إيقاف تنفيذ الشفرة عنده. إليك مثالًا:

```js
let firstNumber = 5;
let secondNumber = 10;
debugger; // Code execution pauses here
let sum = firstNumber + secondNumber;
console.log(sum);
```

مع هذا المثال، إذا لم تكن وحدة التحكم مفتوحة قبل تشغيل الشفرة وفتحتها لاحقًا، سترى الرقم `15` هناك. بينما وحدة التحكم مفتوحة وأعدت تحميل الصفحة، يتوقف التنفيذ لتتمكن من فحص الشفرة.

إليك مثالًا أكثر تعقيدًا:

```js
function calculateTotalPrice(price, discountPercentage) {
 debugger
 let discountAmount = (price * discountPercentage) / 100
 let totalPrice = price - discountAmount

 console.log(`Original Price: ${price}`)
 console.log(`Discount Amount: ${discountAmount}`)
 console.log(`Total Price after Discount: ${totalPrice}`)

 return totalPrice
}

let price = 100
let discount = 15

let finalPrice = calculateTotalPrice(price, discount)
console.log(`Final Price: ${finalPrice}`);
```

في هذا المثال، وُضعت جملة `debugger` في بداية دالة `calculateTotalPrice`. عند استدعاء الدالة، يتوقف تنفيذها مؤقتًا.

إذا أعدت تحميل الصفحة مع فتح وحدة التحكم، سترى أن الصفحة تستمر في إعادة التحميل، مما يؤكد أن تنفيذ الشفرة متوقف مؤقتًا فقط وليس متوقفًا نهائيًا. إذا أردت استئناف التنفيذ، يمكنك النقر على زر التشغيل.

بينما تواصل بناء برامج JavaScript الخاصة بك، جرّب جملة `debugger` في شفرتك لترى كيف تساعدك على فهم تدفق التنفيذ بشكل أفضل، وفحص المتغيرات في الوقت الحقيقي، وتحديد مكان الخطأ بسرعة.
