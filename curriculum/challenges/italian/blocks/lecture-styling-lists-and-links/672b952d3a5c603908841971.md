# --interactive--

Gli stili predefiniti dei collegamenti giocano un ruolo cruciale nel migliorare l'usabilità e l'accessibilità sul web.

Questi stili, tipicamente blu per i collegamenti non visitati e viola per quelli visitati, sono diventati uno standard a cui gli utenti si aspettano e si affidano quando navigano nei siti web.

Lo scopo principale degli stili predefiniti dei collegamenti è fornire segnali visivi chiari che aiutano gli utenti a distinguere tra elementi interattivi e non interattivi su una pagina web.

Questa distinzione è fondamentale per creare un'esperienza di navigazione intuitiva e facile da usare.

Consideriamo gli stili predefiniti di base per i collegamenti:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<a href="/">Example link</a>
```

```css
a:link {
  color: blue;
  text-decoration: underline;
}

a:visited {
  color: purple;
}
```

:::

Questi stili svolgono diverse funzioni importanti.

Innanzitutto, il colore blu per i collegamenti non visitati risalta rispetto alla maggior parte dei colori di sfondo e del testo, rendendo i collegamenti facilmente identificabili. Questo contrasto è fondamentale per permettere agli utenti di scansionare rapidamente una pagina e trovare elementi di navigazione o informazioni importanti.

La sottolineatura enfatizza ulteriormente che il testo è cliccabile, fornendo un segnale visivo aggiuntivo. Questo è particolarmente utile per gli utenti daltonici o che hanno difficoltà a distinguere i colori.

Il cambiamento di colore per i collegamenti visitati (tipicamente in viola) aiuta gli utenti a tenere traccia dei siti già visitati. Questa funzionalità è preziosa per navigare siti web grandi o fare ricerche, poiché evita che gli utenti visitino involontariamente le stesse pagine.

Considera questo esempio HTML:

:::interactive_editor

```html
<p>Learn more about <a href="https://www.example.com/cats">cats</a> and <a href="https://www.example.com/dogs">dogs</a>.</p>
```

:::

Senza alcun CSS personalizzato, la maggior parte dei browser renderizzerà questi collegamenti in blu con una sottolineatura. Dopo aver cliccato su uno dei collegamenti, il suo colore cambierà in viola, fornendo un feedback immediato all'utente sulla sua cronologia di navigazione.

Sebbene sia comune per i designer modificare questi stili predefiniti per adattarli all'estetica di un sito, è fondamentale mantenere i principi di base che li guidano.

Se scegli di cambiare gli stili predefiniti, assicurati che i collegamenti siano ancora chiaramente distinguibili dal testo normale, che ci sia una differenza visibile tra collegamenti visitati e non visitati e che i colori scelti offrano un contrasto sufficiente con lo sfondo.

Ad esempio, potresti usare uno stile personalizzato come questo:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<a href="/">Example link</a>
```

```css
a:link {
  color: blue;
  text-decoration: none;
  border-bottom: 1px solid blue;
}

a:visited {
  color: purple;
  border-bottom: 1px solid purple;
}
```

:::

Questo mantiene lo schema di colori blu e viola sostituendo la sottolineatura con un bordo inferiore per un aspetto più moderno.

È anche importante considerare i diversi stati dei collegamenti. Oltre agli stati predefinito e visitato, i collegamenti hanno tipicamente stati hover e active:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<a href="/">Example link</a>
```

```css
a:hover {
  color: red;
}

a:active {
  color: darkorange;
}
```

:::

Questi stati forniscono un feedback immediato agli utenti mentre interagiscono con i collegamenti, migliorando l'usabilità complessiva del sito.

In conclusione, anche se è possibile personalizzare gli stili dei collegamenti, i principi dietro gli stili predefiniti dovrebbero essere mantenuti.

Essi giocano un ruolo cruciale nel creare un'esperienza web usabile e accessibile, aiutando gli utenti a navigare e interagire efficacemente con i contenuti.

Dai sempre priorità alla chiarezza e all'esperienza utente quando progetti gli stili dei collegamenti per i tuoi siti web.
