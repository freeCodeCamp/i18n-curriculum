# --interactive--

정적인 콘텐츠를 접근 가능하게 만드는 것은 비교적 간단할 수 있지만, 동적인 콘텐츠는 더 어려울 수 있습니다. 바로 이 부분에서 WAI-ARIA가 필요합니다.

WAI-ARIA가 무엇인지, 그 목적, 작동 방식, 그리고 몇 가지 예제를 살펴보겠습니다.

WAI-ARIA는 Web Accessibility Initiative - Accessible Rich Internet Applications의 약자입니다. 동적 콘텐츠와 UI(사용자 인터페이스) 컴포넌트의 접근성을 향상시키는 명세입니다.

WCAG와 WAI-ARIA는 동일하지 않다는 점을 유의하세요. WCAG는 웹 접근성에 대한 일반적인 지침을 제공하는 반면, WAI-ARIA는 보조 기술 사용자를 위해 동적이고 인터랙티브한 콘텐츠를 접근 가능하게 만드는 구체적인 규칙을 제공합니다.

따라서 WAI-ARIA의 주요 목적은 네이티브 HTML에 해당하는 요소가 없는 동적 콘텐츠와 UI 컴포넌트의 접근성을 개선하는 것입니다.

WAI-ARIA는 HTML 요소에 추가 의미 정보를 제공하는 속성 집합을 도입하여 작동합니다. 이 속성들은 역할, 상태, 속성으로 분류됩니다.

ARIA 역할은 웹사이트나 웹 애플리케이션 내에서 요소의 목적을 정의합니다. 다음은 `button` 요소에 역할을 `div`로 설정하는 예입니다.

```html
<div role="button">Click Me</div>
```

이렇게 하면 보조 기술에 이 요소가 버튼임을 알리는 것입니다. 하지만 역할은 기능을 제공하지 않습니다. 단순히 이 `div`에 `role`를 `button`로 주는 것만으로는 버튼처럼 작동하지 않습니다. 버튼처럼 보이고 동작하게 하려면 CSS와 JavaScript를 사용해야 원하는 결과를 얻을 수 있습니다.

다음은 HTML, CSS, JavaScript를 사용해 커스텀 `button` 요소를 만드는 예입니다.

**참고**: CSS와 JavaScript 코드를 이해하려고 걱정하지 마세요. 이 언어들은 앞으로의 모듈에서 배우게 됩니다.

:::interactive_editor

```html
<link href="styles.css" rel="stylesheet">
<div id="custom-btn" role="button">Click Me</div>
<script src="index.js"></script>
```

```css
#custom-btn {
  display: inline-block;
  padding: 0.4em 1em;
  font-size: 1rem;
  font-family: sans-serif;
  color: buttontext;
  background-color: buttonface;
  border: 1px solid #ccc;
  border-radius: 4px;
  cursor: pointer;
  user-select: none;
  text-align: center;
}

#custom-btn:focus {
  outline: 2px solid Highlight;
  outline-offset: 2px;
}

#custom-btn:active {
  background-color: #ddd;
}
```

```js
const button = document.getElementById("custom-btn");

button.addEventListener("click", () => {
  alert("Button clicked!");
});

button.addEventListener("keydown", (e) => {
  if (e.key === "Enter" || e.key === " ") {
    e.preventDefault(); 
    button.click();
  }
});
```

:::

네이티브 `button` 또는 `input` 요소에 `type="button"`를 사용하는 것이 항상 더 좋습니다.

ARIA 속성은 요소에 대한 추가 세부 정보를 제공합니다. 예를 들어, `aria-labelledby` 속성은 요소를 특정 레이블에 연결할 수 있게 합니다:

```html
<h2 id="header-id">About freeCodeCamp</h2>
<button id="button-id" aria-labelledby="header-id button-id">Learn More</button>
```

이렇게 하면 보조 기술 사용자에게 요소가 이해 가능하고 탐색하기 쉬워집니다.

WAI-ARIA를 최대한 활용하려면 가능한 한 네이티브 HTML을 사용하는 것이 좋습니다. 네이티브 HTML이 일반적으로 기본적으로 더 많은 접근성을 제공하기 때문입니다.

HTML이 부족할 때만 WAI-ARIA를 사용하고, 화면 읽기 프로그램 같은 보조 기술로 테스트하거나 장애가 있는 분들에게 테스트를 부탁하세요. 또한 WAI-ARIA 상태와 속성이 콘텐츠와 실시간으로 업데이트되는지 확인하세요. ARIA를 과도하게 사용하면 혼란을 줄 수 있으니 주의하세요.
