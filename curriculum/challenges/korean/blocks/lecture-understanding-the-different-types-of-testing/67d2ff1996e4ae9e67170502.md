# --description--

애플리케이션을 설계할 때, 하나의 역할만 담당하는 작은 함수들이 여러 개 있을 때가 많습니다. 이것을 "단일 책임 원칙"이라고 합니다. 작은 함수들이 여러 개 있을 때는 이 함수들이 예상대로 작동하는지 확인하기 위해 테스트하는 것이 가장 좋습니다. 이번 수업에서는 인기 있는 Jest 테스트 프레임워크를 사용해 단위 테스트를 만드는 방법을 살펴보겠습니다.

이 예제에서는 새로 포맷된 문자열을 반환하는 함수 하나를 만들 것입니다:

```js
export function getFormattedWord(str) {
  if (!str) return '';
  return str.charAt(0).toUpperCase() + str.slice(1);
}
```

이 `getFormattedWord` 함수는 `str`라는 매개변수를 가지고 있으며, 먼저 `str`가 비어 있는지 확인합니다. 비어 있다면 빈 문자열을 반환합니다. 그렇지 않으면 첫 글자가 대문자인 새 문자열을 반환합니다. 이 함수는 테스트 파일에서 사용할 수 있도록 내보내고 있습니다.

별도의 `getFormattedWord.test.js` 파일에서 이 함수가 제대로 작동하는지 확인하는 테스트를 작성할 수 있습니다.

`getFormattedWord.test.js` 파일 상단에서는 먼저 다음과 같이 함수를 가져와야 합니다:

```js
import { getFormattedWord } from "./getFormattedWord.js";
```

그다음 `npm i jest`를 사용해 Jest 패키지를 설치해야 합니다. 그리고 소문자로 이루어진 단어 `hello`가 대문자 `Hello`로 시작하는 `H`로 제대로 포맷되는지 확인하는 테스트를 추가할 수 있습니다:

```js
test('capitalizes the first letter of a word', () => {
  expect(getFormattedWord('hello')).toBe('Hello');
});
```

`expect` 함수는 값을 테스트하는 데 사용됩니다. 이 함수는 값이 예상대로 작동하는지 확인하는 함수인 매처와 결합됩니다. 이 경우 매처는 `toBe()`입니다. Jest에는 진릿값, 문자열, 숫자 등 다양한 테스트를 도와주는 매처가 있습니다.

테스트를 실행하는 한 가지 방법은 `package.json` 파일에 npm 스크립트를 추가하는 것입니다. Jest 프레임워크용 예시 스크립트는 다음과 같습니다:

```js
"scripts": {
  "test": "jest"
},
```

그다음 터미널에서 `npm run test` 명령어를 실행해 테스트를 돌리세요. 테스트가 통과하면 터미널에 다음과 같은 내용이 표시됩니다:

```sh
PASS ./getFormattedWord.test.js
  ✓ capitalizes the first letter of a word (1 ms)

Test Suites: 1 passed, 1 total
Tests:       1 passed, 1 total
Snapshots:   0 total
Time:        0.103 s, estimated 1 s
Ran all test suites.
```

실패한 테스트가 어떻게 보이는지 확인하려면, 다음과 같이 함수를 일부러 수정해 테스트를 깨뜨릴 수 있습니다:

```js
export function getFormattedWord(str) {
  if (!str) return '';
  return "This is incorrect";
}
```

이제 `npm run test` 명령어를 실행하면 테스트가 다른 결과를 기대했기 때문에 오류 메시지가 나타납니다:

```sh
FAIL ./getFormattedWord.test.js
  ✕ capitalizes the first letter of a word (1 ms)

  ● capitalizes the first letter of a word

  expect(received).toBe(expected) // Object.is equality

  Expected: "Hello"
  Received: "This is incorrect"

    2 |
    3 | test('capitalizes the first letter of a word', () => {
    4 |   expect(getFormattedWord('hello')).toBe('Hello');
                                            ^
    5 | });
    6 |

    at Object.toBe (getFormattedWord.test.js:4:37)

Test Suites: 1 failed, 1 total
Tests:       1 failed, 1 total
Snapshots:   0 total
Time:        0.121 s, estimated 1 s
Ran all test suites.
```

다시 원래 함수로 수정해 봅시다:

```js
export function getFormattedWord(str) {
  if (!str) return '';
  return str.charAt(0).toUpperCase() + str.slice(1);
}
```

`getFormattedWord`에 인수가 제공되지 않은 경우를 확인하는 테스트도 하나 더 추가하세요:

```js
test("returns an empty string when no argument is provided", () => {
  expect(getFormattedWord()).toBe("");
});
```

더 견고하게 만들기 위해 추가할 수 있는 테스트가 있지만, 이 두 가지 테스트가 단위 테스트를 시작하기에 좋은 입문입니다.

JavaScript 애플리케이션 테스트에 자주 사용되는 프레임워크로는 Jest, Mocha, Vitest가 있습니다.

단위 테스트는 프로그램에서 더 많은 버그를 잡는 데 도움이 되고, 모든 것이 예상대로 작동하는지 확인하기 때문에 중요합니다. 또한 코드의 예상 동작을 나타내기 때문에 애플리케이션 문서 역할도 할 수 있습니다.
