## 什么是 npm？

- **定义**：npm 是一个开源软件的在线注册表，类似于代码的“应用商店”。
- **官方含义**：npm 并不正式代表“Node Package Manager”；它是一个递归的缩略语，意思是“npm 不是一个缩略语”。
- **三个相关部分**：

  - **网站（npmjs.com）**：用于查找注册表中的包。
  - **注册表**：存储所有包的大型公共数据库。
  - **CLI（命令行接口）**：随 Node.js 安装的工具，用于管理项目依赖。

- **备选方案**：虽然 npm 是最广泛使用的，但流行的备选方案包括 **Yarn**、**PNPM** 和 **Bun**。

## package.json 文件

- **用途**：一个配置文件，用于保存 Node.js 项目的元数据、依赖和脚本。
- **关键字段**：

  - `name` 和 `version`：标识项目。
  - `main`：指定入口文件，通常是 `index.js`。
  - `license`：定义法律条款（默认是 `ISC`）。
  - `dependencies`：运行应用生产环境所需的包。
  - `devDependencies`：仅在开发期间需要的工具，如测试框架。

- **创建**：

  - `npm init`：启动交互式设置流程。
  - `npm init -y`：立即安装一个带默认值的 `package.json`。

## 语义化版本控制（SemVer）

- **格式**：`MAJOR.MINOR.PATCH`。
- **规则**：

  - **MAJOR**：破坏性更改，可能影响现有代码。
  - **MINOR**：向后兼容的新功能。
  - **PATCH**：小的错误修复和改进。

- **范围符号**：

  - `^`（插入符号）：允许次要和补丁更新（例如 `^1.2.3` 允许 `1.x.x`）。
  - `~`（波浪号）：只允许补丁更新（例如 `~1.2.3` 允许 `1.2.x`）。
  - `*`（星号）：匹配任何版本；主要用于测试。

## 依赖管理

- **安装**：

  - `npm install <name>`：添加生产依赖。
  - `npm install <name> -D`：添加开发依赖。
  - `npm install <name>@<version>`：安装特定版本。

- **Node Modules 和锁定文件**：

  - `node_modules/`：包含已安装包的实际代码。
  - `package-lock.json`：锁定所有依赖和“子依赖”的确切版本，确保环境一致性。

- **移除**：`npm uninstall <name>` 移除包并更新 `package.json`。
- **维护**：

  - `npm outdated`：识别哪些包有新版本可用。
  - `npm update`：更新在允许的 SemVer 范围内的包。

## npm 脚本

- **功能**：在 `scripts` 的 `package.json` 部分定义的自定义命令，用于自动化任务。
- **执行**：通过 `npm run <script_name>` 运行。
- 有 `start`、`test`、`stop` 和 `restart` 的简写（例如 `npm start`）。
- **顺序执行**：
- `&&`：仅当第一个命令成功时运行第二个命令。
- `;`：无论成功与否，依次运行命令。
- **并发执行**：使用 `&` 操作符在后台运行进程。
- **参数**：使用 `--` 分隔符传递参数给脚本（例如 `npm start -- 8000`）。

## 发布到 npm 注册表

- **注册表访问**：公共和私有包的数据库。
- **发布步骤**：

1. 在 `npmjs.com` 创建免费账户。
2. 通过终端使用 `npm login` 登录。
3. 确保 `name` 中有唯一的 `version` 和有效的 `package.json`。
4. 创建 `README.md` 和 `.npmignore`（用于排除敏感文件）。
5. 运行 `npm publish`（或针对作用域包运行 `npm publish --access public`）。

- **作用域**：用于避免名称冲突，通过 `@username/package-name` 发布。

## Node.js 中的模块系统

### CommonJS（CJS）

- **默认系统**：Node.js 处理模块的原始方式；同步加载。
- **语法**：
- **导入**：`const module = require('./path')`。
- **导出**：`module.exports = { functionName }`。
- **识别**：用于 `.cjs` 文件，或当 `.js` 中未指定 `type` 时的 `package.json` 文件。

### ES 模块（ESM）

- **标准格式**：现代 JavaScript 标准；为提高效率异步加载。
- **语法**：

  - **导出**：`export { name }` 或 `export default function`。
  - **导入**：`import { name } from './path.mjs'`。
  - **识别**：用于 `.mjs` 文件，或当 `.js` 包含 `package.json` 时的 `"type": "module"` 文件。
