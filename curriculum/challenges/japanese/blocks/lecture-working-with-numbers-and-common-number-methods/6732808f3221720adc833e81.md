# --interactive--

`parseFloat()` と `parseInt()` は、文字列を数値に変換するための JavaScript の重要なメソッドです。これらのメソッドは、ユーザーの入力や文字列形式で受け取ったデータを数値として扱う必要がある場合に特に役立ちます。

まず `parseFloat()` から始めましょう。このメソッドは文字列の引数をパースして浮動小数点数を返します。文字列の先頭から数値を抽出するように設計されており、文字列の後半に数値以外の文字があっても問題ありません。浮動小数点数は小数点を含む数値であることを覚えておいてください。`parseFloat()` の動作は次の通りです。

:::interactive_editor

```js
console.log(parseFloat("3.14"));     // 3.14
console.log(parseFloat("3.14 abc")); // 3.14
console.log(parseFloat("3.14.5"));   // 3.14
console.log(parseFloat("abc 3.14")); // NaN
```

:::

ご覧の通り、`parseFloat()` は文字列の先頭からパースを始め、浮動小数点数の一部になり得ない文字に出会うまで続けます。文字列の先頭に有効な数値が見つからない場合は `NaN`（Not a Number）を返します。

一方、`parseInt()` は文字列の引数をパースして整数を返します。`parseFloat()` と同様に文字列の先頭から始めますが、最初の数字以外の文字に出会うとストップします。`parseInt()` の動作は次の通りです。

:::interactive_editor

```js
console.log(parseInt("42"));       // 42
console.log(parseInt("42px"));     // 42
console.log(parseInt("3.14"));     // 3
console.log(parseInt("abc123"));   // NaN
```

:::

`parseInt()` は最初に出会った数字以外の文字でパースをストップします。浮動小数点数の場合は整数部分のみを返します。文字列の先頭に有効な整数が見つからない場合は `NaN` を返します。

両方のメソッドには注目すべき動作があります。先頭の空白は無視されます。

:::interactive_editor

```js
console.log(parseFloat("  3.14"));  // 3.14
console.log(parseInt("  42"));      // 42
```

:::

文字列の先頭にあるプラスやマイナスの符号も処理します。

:::interactive_editor

```js
console.log(parseFloat("+3.14"));  // 3.14
console.log(parseInt("-42"));      // -42
```

:::

これらのメソッドは強力ですが、いくつかの制限もあります。例えば、科学的記数法のようなすべての数値形式を直接扱うわけではありません。より複雑なパースが必要な場合は、追加の手法やライブラリを使う必要があります。

まとめると、`parseFloat()` と `parseInt()` は JavaScript で文字列を数値に変換するための便利なツールです。これらの動作や特徴を理解することで、特にユーザー入力や外部データを扱う際に数値データをより効果的に処理できます。
