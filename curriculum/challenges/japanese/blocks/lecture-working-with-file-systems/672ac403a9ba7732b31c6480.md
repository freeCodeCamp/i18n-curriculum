# --description--

ファイルはその内容と構造に基づいて分類されます。ファイルタイプは、コンピューターのプログラムがファイルをどのように開き処理するかを決定します。

ファイルタイプは拡張子によって識別されます。ファイル拡張子とは、ファイル名の末尾に付く3文字または4文字の接尾辞です。ウェブアプリケーションでよく見かける主なファイルタイプはHTML、CSS、JavaScriptの3つです。

HTMLファイルはウェブアプリケーションの構造と内容を含みます。拡張子は`.html`です。

CSSファイルはスタイルを定義します。拡張子は`.css`です。

そしてJavaScriptファイルはより高度な機能やインタラクティブ性を追加します。拡張子は`.js`です。

ウェブアプリケーションを開発する際には画像も含める必要があります。以下は最も広く使われている画像ファイルタイプのいくつかです。

JPEGは「Joint Photographic Experts Group」の略で、効率的な圧縮で知られています。写真や画像に適しています。ただし圧縮はロスィー（不可逆）で、ファイルサイズを小さくするために一部の画質が失われます。

PNGは「Portable Network Graphics」の略で、圧縮後も画質を保持します。そのためファイルサイズは大きくなります。PNGは透過をサポートし、ロゴやアイコンのようなシャープなエッジの画像に適しています。

GIF画像形式はアニメーションと透過の両方をサポートしますが、カラーパレットが限られています。GIFは「Graphics Interchange Format」の略です。

最後にSVGは「Scalable Vector Graphics」の略で、ベクター画像形式です。画質を損なうことなく必要なだけ拡大縮小できます。

画像形式について理解できたところで、次は動画と音声の形式を見てみましょう。

MP3は効率的な圧縮で知られる音声形式です。ロスィー形式なので圧縮時に一部の音声データが失われ、ファイルサイズを小さくします。

MP4は最も一般的な動画形式の一つです。良好な圧縮を提供し、複数の音声・動画コーデック、字幕、メタデータをサポートします。

MOVはAppleが開発したQuickTimeのマルチメディアファイル形式で、主にQuickTime Playerと関連しています。

その他の人気のある音声・動画ファイルタイプには、元の音質を保持するロスレス音声形式のWAVや、高品質なオープンソース動画形式のWebMがあります。

ウェブアプリケーションのフォントをカスタマイズする必要がある場合は、フォント形式も扱います。以下は最も広く使われている3つのフォント形式です。

TTFは「TrueType Font」の略で、異なるOSに互換性のあるフォント形式です。

WOFFはウェブ開発向けに特化したモダンなフォント形式で、「Web Open Font Format」の略です。より良い圧縮によりファイルサイズが小さくなり、ライセンス情報を含むメタデータも保存できます。

WOFF2はWOFFの後継で、さらに効率的な圧縮とパフォーマンスを実現しています。

複数のフォルダーやファイルをまとめる必要がある場合は、アーカイブファイルを作成できます。最も広く使われているアーカイブ形式はZIPです。ZIPはロスレス圧縮を提供し、処理中にデータが失われることはありません。多くのOSやソフトウェアで広くサポートされています。

最後にドキュメントについて説明します。ウェブアプリケーションでは通常、`README`というファイルがあり、使い方、インストール方法、ライセンス、貢献方法などの情報が含まれています。

これらは通常、Markdownというファイル形式で書かれています。Markdownを使うと、見出し、小見出し、リンク、画像、リストなどを含む構造化された文書を作成できます。Markdownファイルの拡張子は`.md`または`.markdown`です。

こちらはfreeCodeCampのGitHubリポジトリの`README`ファイルです：https://github.com/freeCodeCamp/freeCodeCamp/blob/main/README.md

Markdownを使ってこのような詳細な構造とフォーマットを作成できます。

キャリアを通じて多くの`README`ファイルを読み書きすることになるでしょう。これらの一般的なファイルタイプを知ることはウェブ開発に不可欠です。

HTML、CSS、JavaScriptという基本的な構成要素から画像、動画、音声形式まで、すべてのファイルタイプがインタラクティブなウェブアプリケーションを作る上で重要な役割を果たしています。
