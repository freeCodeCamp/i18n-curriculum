# --description--

JavaScript 被认为是一种动态类型语言。这意味着变量可以在运行时接收任何值——将变量声明为数字类型并不妨碍你后来为它赋值为字串，函数参数也可以传递任何值。

动态类型语言的挑战在于，如果你不小心，缺乏类型安全可能会引入误差。TypeScript 扩展了 JavaScript 语言，加入了静态类型指派，这有助于在你部署代码之前捕捉这些误差。但它是如何工作的？

考虑这个 JavaScript 函数：

```js
const getRandomValue = (array) => {
  return array[Math.floor(Math.random() * array.length)];
}
```

我们的 `getRandomValue` 函数接受一个数组并从该数组中返回一个随机值。但由于 JavaScript 不验证类型，没有任何东西阻止你用数字调用该函数：

```js
console.log(getRandomValue(10));
```

当前示例的控制台输出将返回 `undefined`，因为它期望的是数组而不是数字。

但是使用 TypeScript 你可以为数组参数定义一个类型：

```js
const getRandomValue = (array: string[]) => {
  return array[Math.floor(Math.random() * array.length)];
}
```

此类型定义告诉 TypeScript，数组参数必须是字串数组。然后当你调用 `getRandomValue` 并传入一个数字时，你会得到另一种称为编译器误差的错误。

大多数 JavaScript 运行期环境，例如浏览器或 Node.js，不能本地运行 TypeScript。相反，你首先需要将 TypeScript 代码编译或转换成普通的 JavaScript。你可以使用 TypeScript 语言自带的内置编译器来完成这项工作。当你运行编译器时，TypeScript 会评估你的代码，并在类型不匹配的任何问题上抛出误差——例如将数字传递给期望数组的函数。

你将在接下来的几节课中了解这些类型的工作原理。但正是这种额外的安全性使得 TypeScript 成为许多开发者和组织青睐的语言。
