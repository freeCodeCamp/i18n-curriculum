# --description--

Das Einrichten einer benutzerdefinierten Authentifizierung erfordert das Schreiben einer beträchtlichen Menge manueller Logik. Sie müssen die Kopfzeilen der Anfrage lesen, Cookies extrahieren, Signaturen überprüfen und Fehler verwalten.

Während es großartig zum Lernen ist, dies selbst zu bauen, ist es zeitaufwendig und anfällig für Sicherheitslücken, wenn Sie es für jede Produktionsanwendung tun. Deshalb verwenden Entwickler Passport.js.

Passport.js ist eine Authentifizierungs-Middleware für Node.js. Es fungiert als organisiertes Framework, das standardisiert, wie Sie Anfragen authentifizieren. Es übernimmt die Details der niedrigen Ebene der Authentifizierung, sodass Sie sich auf die Geschäftslogik Ihrer Anwendung konzentrieren können.

Die Kernidee hinter Passport.js ist, dass jede Anwendung unterschiedliche Möglichkeiten zum Einloggen benötigt, der zugrunde liegende Authentifizierungsablauf jedoch gleich bleiben sollte. Passport erreicht diese Flexibilität durch modulare Plugins, die **Strategies** genannt werden.

Anstatt für jede Authentifizierungsmethode eigenen Code zu schreiben, installieren Sie die spezifische Strategy, die Sie benötigen. Es gibt über 500 Strategien, die fast jede heute verwendete Authentifizierungsmethode abdecken. Einige davon sind:

- **passport-local:** Handhabt die traditionelle Authentifizierung mit Benutzername und Passwort gegen Ihre eigene Datenbank.

- **passport-jwt:** Schützt API-Endpunkte, indem es JSON Web Tokens aus eingehenden Anfragen extrahiert und validiert.

- **passport-google-oauth20:** Ermöglicht Benutzern, sich sofort mit ihren bestehenden Google-Konten einzuloggen.

- **passport-facebook:** Einloggen mit Facebook-Konten über OAuth 2.0.

- **passport-github2:** Ermöglicht Benutzern, sich sofort mit ihren GitHub-Konten einzuloggen.

Indem die Authentifizierung in einzelne Strategien aufgeteilt wird, erlaubt Passport Ihrer Anwendung, nahtlos zu wachsen. Wenn Sie heute mit einem Benutzername/Passwort-System starten und nächsten Monat "Mit Google einloggen" hinzufügen möchten, müssen Sie Ihren gesamten Authentifizierungscode nicht neu schreiben. Sie müssen nur die Google-Strategy installieren und in Passport einbinden.

Um Passport in einer Express-Anwendung zu verwenden, müssen Sie das Kern-Passport-Paket zusammen mit der spezifischen Strategy installieren, die Sie verwenden möchten. Für eine Standard-Benutzername-und-Passwort-Konfiguration führen Sie den folgenden Befehl in Ihrem Terminal aus:

```bash
npm install passport passport-local
```

Nach der Installation müssen Sie Passport in Ihre Express-Anwendungs-Konfiguration einbinden. Dies muss geschehen, bevor Sie Ihre Routen definieren:

```js
const express = require('express');
const passport = require('passport');
const LocalStrategy = require('passport-local').Strategy;

const app = express();

// initialize Passport middleware
app.use(passport.initialize());

// configure Passport to use the Local Strategy
passport.use(new LocalStrategy((username, password, done) => {
    // your database look-up and password verification logic goes here
}));
```

Passport integriert sich nahtlos als Middleware in das Standard-Express-Routing-System. Wenn ein Benutzer seine Zugangsdaten an Ihre Login-Route absendet, fängt Passport die Anfrage ab, übergibt die Daten an Ihre konfigurierte Strategy und bewertet das Ergebnis.

```js
app.post('/login', passport.authenticate('local'), (req, res) => {
  // if this function runs, then authentication was successful
  res.send(`Welcome back, ${req.user.username}!`);
});
```

Wenn die Strategy den Benutzer erfolgreich verifiziert, übernimmt Passport automatisch zwei kritische Aufgaben:

1. **Sitzungsverwaltung:** Es ruft Hilfsfunktionen auf, um ein kleines Stück Benutzerdaten im Sitzungscookie zu speichern.

2. **Anfrage-Dekoration:** Es hängt das authentifizierte Benutzerobjekt direkt an die eingehende Anfrage als `req.user` an.

Sobald `req.user` befüllt ist, kann jede nachfolgende Middleware oder jeder Route-Handler in Ihrer Anwendung sofort sehen, wer eingeloggt ist, und basierend auf dessen Identität Autorisierungsentscheidungen treffen.

Passport bietet eine konsistente, sichere und produktionsreife Grundlage für die Verwaltung von Benutzeridentitäten in Ihren Node.js-Anwendungen.
