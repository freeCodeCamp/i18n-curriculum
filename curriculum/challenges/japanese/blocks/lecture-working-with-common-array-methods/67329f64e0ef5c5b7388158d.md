# --interactive--

JavaScriptでは、`indexOf()` メソッドは配列内の特定の要素の最初のインデックスを見つけるのに便利です。要素が見つからない場合は、`-1` を返します。基本的な構文は以下の通りです。

```js
array.indexOf(element, fromIndex)
```

`element` は配列内で検索したい値を表し、`fromIndex` パラメータは検索を開始する位置です。`fromIndex` パラメータは省略可能です。`fromIndex` が指定されない場合、検索は配列の先頭から始まります。例を見てみましょう。

:::interactive_editor

```js
let fruits = ["apple", "banana", "orange", "banana"];
let index = fruits.indexOf("banana");
console.log(index); // 1
```

:::

この例では、`fruits` という配列にさまざまな果物の名前が入っています。`indexOf()` メソッドを使って、`fruits` 配列内の文字列 `banana` のインデックスを探しています。`banana` はインデックス `1` に存在するため、メソッドは `1` を返し、それが `index` 変数に格納されコンソールに出力されます。

検索している要素が配列に見つからない場合、`indexOf()` は `-1` を返します。例えば：

:::interactive_editor

```js
let fruits = ["apple", "banana", "orange"];
let index = fruits.indexOf("grape");
console.log(index); // -1
```

:::

ここでは、fruits 配列内で文字列 `grape` を `indexOf()` を使って検索しています。`grape` は配列に存在しないため、メソッドは `-1` を返し、それが `index` 変数に格納されコンソールに出力されます。

特定のインデックス番号の後からアイテムを探し始めたい場合は、次の例のように2番目の引数を渡すことができます。

:::interactive_editor

```js
let colors = ["red", "green", "blue", "yellow", "green"];
let index = colors.indexOf("green", 3);
console.log(index); // 4
```

:::

この例では、検索は配列の先頭からではなく、インデックス番号 `3`（`yellow`）から始まり、`4` の結果が得られます。
