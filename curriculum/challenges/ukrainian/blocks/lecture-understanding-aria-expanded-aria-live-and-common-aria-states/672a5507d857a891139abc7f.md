# --interactive--

Атрибут `aria-controls` використовується для створення програмного зв’язку між елементом, який керує наявністю іншого елемента на сторінці, і самим керованим елементом. Цей зв’язок допомагає користувачам із читачами екрана краще розуміти, як працює керування. Загальні випадки використання включають набір вкладок, які керують видимістю різних розділів вмісту, або кнопку, що перемикає видимість меню.

Давайте розглянемо приклад, щоб побачити, як це працює. У цьому прикладі вкладкового інтерфейсу ми маємо елемент `div` із набором із трьох кнопок:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<div role="tablist">
  <button role="tab" id="tab1" aria-controls="section1" aria-selected="true">
    Tab 1
  </button>
  <button role="tab" id="tab2" aria-controls="section2" aria-selected="false">
    Tab 2
  </button>
  <button role="tab" id="tab3" aria-controls="section3" aria-selected="false">
    Tab 3
  </button>
</div>
```


`div` використовує `role="tablist"`, щоб вказати, що він слугує контейнером для групи вкладок.

Кожна кнопка представляє вкладку і має атрибут `role`, встановлений у `tab`. У більшості вкладкових інтерфейсів перша панель вкладки видима за замовчуванням, тому перша кнопка вкладки має атрибут `aria-selected`, встановлений у `true`, щоб показати, що пов’язаний із нею розділ вмісту наразі видимий. Атрибут `aria-controls` використовується, щоб асоціювати кожну кнопку з розділом вмісту, яким вона керує.

Ось три розділи вмісту, які відповідають вкладкам:

```js
const tabs = document.querySelectorAll('[role="tab"]');
const tabList = document.querySelector('[role="tablist"]');

tabs.forEach(tab => {
  tab.addEventListener('click', () => {
    activateTab(tab);
  });

  tab.addEventListener('keydown', (e) => {
    const key = e.key;
    const currentIndex = Array.from(tabs).indexOf(tab);
    let newIndex = null;

    if (key === 'ArrowRight') {
      newIndex = (currentIndex + 1) % tabs.length;
    } else if (key === 'ArrowLeft') {
      newIndex = (currentIndex - 1 + tabs.length) % tabs.length;
    } else if (key === 'Home') {
      newIndex = 0;
    } else if (key === 'End') {
      newIndex = tabs.length - 1;
    }

    if (newIndex !== null) {
      tabs[newIndex].focus();
      activateTab(tabs[newIndex]);
    }
  });
});

function activateTab(tab) {
  const tabPanels = document.querySelectorAll('[role="tabpanel"]');

  tabs.forEach(t => {
    t.setAttribute('aria-selected', 'false');
    t.setAttribute('tabindex', '-1');
  });

  tabPanels.forEach(panel => panel.hidden = true);

  tab.setAttribute('aria-selected', 'true');
  tab.removeAttribute('tabindex');

  const panelId = tab.getAttribute('aria-controls');
  const panel = document.getElementById(panelId);
  panel.hidden = false;
  tab.focus();
}

```

:::

Кожен розділ вмісту має атрибут `role`, встановлений у `tabpanel`, і атрибут `aria-labelledby`, який вказує на відповідну вкладку, щоб надати панелі доступну назву. Атрибут `hidden` використовується для приховування розділів вмісту, які наразі не вибрані. Ідентифікатор кожного розділу збігається зі значенням атрибута `aria-controls` на відповідній кнопці. Ідентифікатор `section1` збігається з атрибутом `aria-controls` на першій кнопці, `section2` — з атрибутом `aria-controls` на другій кнопці, а `section3` — з атрибутом `aria-controls` на третій кнопці. Саме так встановлюється асоціація між вкладками та їхніми розділами вмісту.

Щоб перемикатися між різними розділами, потрібно використовувати JavaScript для оновлення атрибута `hidden` на розділі та атрибута `aria-selected` на кнопці залежно від того, який розділ наразі видимий. Одночасно може бути видимий лише один розділ, і він не повинен мати атрибут `hidden`, а `aria-selected` має бути встановлений у `true` на його кнопці. Решта прихованих розділів повинні мати атрибут `hidden`, а `aria-selected` має бути встановлений у `false` на їхніх кнопках.

Вкладки — це загальний випадок використання атрибута `aria-controls`, але його можна застосовувати й у інших сценаріях, де один елемент керує видимістю або поведінкою іншого елемента. Інші приклади включають акордеони, випадаючі меню та модальні вікна.

Наступного разу, коли ви розроблятимете інтерфейс користувача, що передбачає керування видимістю елементів, розгляньте можливість використання атрибута `aria-controls` для встановлення зв’язку між керуючим елементом і керованим елементом.
