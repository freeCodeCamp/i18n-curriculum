# --description--

GitHub Pages ni chaguo thabiti sana kwa kusambaza tovuti zisizobadilika, au programu ambazo hazihitaji seva ya nyuma kusimamia mantiki. Yaani, programu zinazotumia kabisa upande wa mteja, au katika kivinjari cha mtumizi, zinaweza kusambazwa kikamilifu kwenye jukwaa hili.

Zaidi ya hayo, GitHub Pages ni bure kabisa kwa miradi ya chanzo huru. Kuna sababu nyingi nzuri za kufanya mradi wako kuwa chanzo huru, na Pages huongeza sababu nyingine zaidi.

Lakini unavyosambaza mradi kwenye GitHub Pages? Hiyo inategemea jinsi programu yako inavyofanya kazi.

Tuanze na ukurasa wa msingi wa HTML. Huu ni mfano unaweza kuuweka kwenye repo yako iliyopo ikiwa unataka:

```html
<!DOCTYPE html>
<html>
  <head>
    <title>freeCodeCamp</title>
    <link rel="stylesheet" href="./style.css" />
  </head>
  <body>
    <h1>Welcome to freeCodeCamp's Lessons!</h1>
    <p>Let's have some fun learning about Git and GitHub!</p>
  </body>
</html>
```

Sasa tunahitaji kutembelea hifadhidata yetu na kubofya kichupo cha "Settings" ili kwenda kwenye mipangilio ya hifadhidata. Kisha tunahitaji kuchagua "Pages" kutoka kwenye upau wa upande.

Kwanza, kuna chaguzi chache tu, lakini hiyo itabadilika tunapoanza kusanidi mipangilio. Kwanza, tunahitaji kusema GitHub tunapotaka kusambaza kutoka.

Tutapendelea kuacha hii kama "Deploy from a branch", kwa sababu hatutumi mfumo wa kazi na hatujasanidi vitendo vya GitHub kabisa. Utajifunza zaidi kuhusu hizo katika funzo lijalo.

Ifuatayo, weka tawi tunalosambazia kutoka `main` ili kusambaza kile kilicho hapo.

Kisha utapewa chaguo la kusambaza kutoka kwenye folda ya mzizi wa mradi wako au kutoka kwenye folda ya `docs`. Kipengele hiki hutumiwa na miradi kama freeCodeCamp - tulikuwa tunajumuisha nyaraka na programu yetu, ili tuweze kusambaza nyaraka tu kwenye GitHub Pages kupitia folda ya `docs`. Kwa sasa, acha iwe folda ya `/` (mzizi) na bonyeza kitufe cha "Save" kando na menyu hii ili kuhifadhi mabadiliko yako.

Ukipohifadhi, utaona ingizo jipya la kusanidi kikoa maalum. Ikiwa tayari una kikoa na ungependa kulitumia, unaweza kufuata kiungo cha nyaraka kufanya hivyo. Kwa funzo hili, hata hivyo, tutaacha wazi.

Pia unaweza kuona kikoa ambacho tovuti yako mpya ya GitHub Pages imesambazwa. Bila kikoa maalum, hii itakuwa `<username>.github.io`, ambapo `<username>` ni jina la akaunti yako (au akaunti ya shirika ikiwa unasambaza kutoka kwenye hifadhidata inayomilikiwa na shirika). Na hifadhidata itahudumiwa kwenye njia inayolingana na jina la hifadhidata, kwa mfano `<username>.github.io/<repo-name>`. Ukitembelea URL hiyo, utaona tovuti yako!

Sasa, unaposukuma mabadiliko mapya kwenye tawi la `main`, GitHub Pages itasasisha moja kwa moja tovuti yetu iliyosambazwa na mabadiliko ya hivi karibuni. Mchakato huu unaweza kuchukua dakika chache, kulingana na mzigo wa seva - na unaweza kuhitaji kufanya onyesho upya wa nguvu ili kufuta kumbukumbu ya zamani.

Na kwa hayo, tumesambaza kwa mafanikio programu ya msingi ya mtandao kwenye GitHub Pages. Pia unaweza kujumuisha mafaili ya CSS na JavaScript katika usambazaji wako wa Pages, hivyo unaweza kujenga tovuti nzuri na zinazoshirikiana. Na mara utakapojifunza mfumo wa kazi kama React au Angular, unaweza hata kusambaza hizo kwenye GitHub Pages pia!
