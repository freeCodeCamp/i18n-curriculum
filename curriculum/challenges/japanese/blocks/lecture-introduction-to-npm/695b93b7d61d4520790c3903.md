# --description--

JavaScriptでプロジェクトを作るとき、作業を助けるための追加ツールがよく必要になります。簡単なアプリでも、データの保存、ログイン、ページ間のルーティングなどの機能が必要になることがあります。

これらすべてのコードを最初から書く代わりに、他の開発者がすでに作って共有しているオープンソースのパッケージを使うことができます。

ここでnpmの登場です。npmは巨大なオンラインのオープンソースソフトウェアのレジストリです。コードのためのアプリストアのようなもので、必要なものを検索してインストールし、プロジェクトで使い始めることができます。

多くの人はnpmを「Node Package Manager」の略だと思っていますが、公式の意味ではありません。古いnpmのドキュメントでは「npmは頭字語ではない」という再帰的な略語として説明されていました。つまり、npmは公式には何の略でもありません。

現在、npmは次の3つの関連する部分を指します：

* npmjs.comというウェブサイトで、レジストリ内のパッケージを調べられます。
    
* すべてのパッケージを保存している大きな公開データベースであるnpmレジストリ。
    
* Node.jsと一緒にインストールされ、パッケージのインストールや管理を行うnpmのコマンドラインツール。
    

例えば、プロジェクトで状態管理、日付処理、メール送信、HTTPリクエストを扱う必要がある場合、すでにそれを行うnpmパッケージがあるかもしれません。簡単な`npm install`コマンドで、自分で全部作る代わりにこの既製のソフトウェアを使い始められます。

npmはプロジェクトの依存関係の管理も助けます。これは、プロジェクトで使うパッケージとそれぞれのバージョンを追跡することを意味します。これは`package.json`と`package-lock.json`ファイルで行われており、これらについては今後のレッスンで学びます。

Yarn、PNPM、Bunはnpmの代替として人気が出てきたものです。インストール速度やストレージ効率など特定の分野で利点がありますが、npmは依然として最も広く使われています。

今後のレッスンでは、さまざまなnpmコマンド、パッケージのインストール、`package.json`と`package-lock.json`ファイル、さらには自分のパッケージをnpmレジストリに公開する方法も学びます。
