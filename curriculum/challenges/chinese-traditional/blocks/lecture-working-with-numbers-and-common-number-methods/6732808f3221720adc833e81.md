# --interactive--

`parseFloat()` 和 `parseInt()` 是 JavaScript 中兩個將字串轉換為數字的基本方法。當處理使用者輸入或需要將以字串格式呈現但必須視為數值的資料時，這些方法特別有用。

讓我們從 `parseFloat()` 開始。這個方法會解析一個字串引數並傳回一個浮點數。它的設計目的是從字串的開頭擷取數字，即使字串後面包含非數字字元。請記住，浮點數是帶有小數點的數字。以下是 `parseFloat()` 的運作方式：

:::interactive_editor

```js
console.log(parseFloat("3.14"));     // 3.14
console.log(parseFloat("3.14 abc")); // 3.14
console.log(parseFloat("3.14.5"));   // 3.14
console.log(parseFloat("abc 3.14")); // NaN
```

:::

如你所見，`parseFloat()` 從字串的開頭開始解析，並持續直到遇到無法成為浮點數一部分的字元。如果它在字串開頭找不到有效的數字，則會傳回 `NaN`（非數字）。

另一方面，`parseInt()` 會解析字串引數並傳回整數。像 `parseFloat()` 一樣，它從字串的開頭開始，但會在遇到第一個非數字字元時停止。以下是 `parseInt()` 的運作方式：

:::interactive_editor

```js
console.log(parseInt("42"));       // 42
console.log(parseInt("42px"));     // 42
console.log(parseInt("3.14"));     // 3
console.log(parseInt("abc123"));   // NaN
```

:::

`parseInt()` 在遇到第一個非數字字元時停止解析。對於浮點數，它只傳回整數零件。如果它在字串開頭找不到有效的整數，則傳回 `NaN`。

這兩種方法都有一些值得注意的行為。它們會忽略前置空白：

:::interactive_editor

```js
console.log(parseFloat("  3.14"));  // 3.14
console.log(parseInt("  42"));      // 42
```

:::

它們處理字串開頭的加號和減號：

:::interactive_editor

```js
console.log(parseFloat("+3.14"));  // 3.14
console.log(parseInt("-42"));      // -42
```

:::

值得注意的是，雖然這些方法功能強大，但它們有一些限制。例如，它們不會直接控制所有數字格式，如科學記號。若有更複雜的解析需求，你可能需要使用額外的技術或函式庫。

總結來說，`parseFloat()` 和 `parseInt()` 是在 JavaScript 中將字串轉換為數字的寶貴工具。了解它們的運作方式及其特定行為，能讓你在應用程式中更有效地控制數值資料，特別是在處理使用者輸入或外部資料來源時。
