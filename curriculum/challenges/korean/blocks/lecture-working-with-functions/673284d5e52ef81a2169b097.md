# --interactive--

이전 수업에서는 함수에 대해 배웠습니다. 함수는 재사용 가능한 코드 조각으로, 코드를 더 모듈화하고 유지 관리하기 쉽고 효율적으로 만듭니다. 이전 모든 예제는 다음과 같은 일반 함수 구문을 사용했습니다:

```js
function greetings(name) {
  console.log("Hello, " + name + "!");
}
```

하지만 JavaScript에서 함수를 작성하는 또 다른 방법은 화살표 함수 표현식을 만드는 것입니다. 이전 예제를 화살표 함수 구문으로 리팩터링하는 방법은 다음과 같습니다:

```js
const greetings = (name) => {
  console.log("Hello, " + name + "!");
};
```

이 수정된 예제에서는 `const` 변수인 `greetings`을 만들고 익명 함수를 할당하고 있습니다. 대부분의 구문은 익숙할 텐데, `function` 키워드가 빠지고 `=>` 매개변수와 함수 본문 사이에 화살표(`name`)가 추가된 점이 다릅니다. 매개변수가 하나뿐이라면 소괄호를 이렇게 생략할 수 있습니다:

```js
const greetings = name => {
  console.log("Hello, " + name + "!");
};
```

화살표 함수에 매개변수가 없다면 반드시 소괄호를 이렇게 사용해야 합니다:

```js
const greetings = () => {
  console.log("Hello");
};
```

함수를 처음 배울 때는 함수 본문을 중괄호로 감싸야 했습니다. 하지만 함수 본문이 한 줄 코드만 포함한다면 중괄호를 이렇게 생략할 수 있습니다:

```js
const greetings = name => console.log("Hello, " + name + "!");
```

일반 함수 구문에서 소괄호와 중괄호를 생략하면 작동하지 않는다는 점을 꼭 기억하세요. 다음과 같이 하면 오류가 발생합니다:

```js
// This will produce syntax errors 
function greetings name console.log("Hello, " + name + "!");
```

이런 한 줄 함수는 화살표 함수 구문에서만 작동합니다. 또 다른 중요한 개념은 `return` 문입니다. 화살표 함수 구문을 사용해 영역을 계산하는 예는 다음과 같습니다:

:::interactive_editor

```js
const calculateArea = (width, height) => {
  const area = width * height;
  return area;
};

console.log(calculateArea(5, 3)); // 15
```

:::

함수 내부에 `area`라는 변수를 만들고 그 변수를 반환하고 있습니다. 하지만 계산식 자체를 반환하도록 코드를 좀 더 간결하게 만들 수 있습니다:

:::interactive_editor

```js
const calculateArea = (width, height) => {
  return width * height;
}; 

console.log(calculateArea(5, 3)); // 15
```

:::

중괄호를 제거하고 계산식을 같은 줄에 두면 `Uncaught SyntaxError: Unexpected token 'return'` 메시지가 나타납니다:

```js
const calculateArea = (width, height) => return width * height;
```

이 오류가 발생하는 이유는 `return` 문을 제거해야 하기 때문입니다. `return` 문을 제거하면 오류가 사라지고 함수가 계산식을 암묵적으로 반환합니다.

```js
const calculateArea = (width, height) => width * height;
```

그렇다면 언제 화살표 함수 구문을 사용해야 할까요? 상황에 따라 다릅니다. 많은 개발자가 개인 프로젝트에서는 일관되게 사용합니다. 하지만 팀 작업에서는 기존 코드베이스가 일반 함수인지 화살표 함수인지에 따라 선택하는 경우가 많습니다. 앞으로 수업에서 화살표 함수를 언제 사용하고 언제 피해야 하는지 다룰 예정입니다.
