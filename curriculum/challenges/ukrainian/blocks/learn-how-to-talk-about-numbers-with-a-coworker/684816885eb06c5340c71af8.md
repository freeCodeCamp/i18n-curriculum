# --explanation--

`Initial unit tests` — це перший крок у тестуванні. Розробники перевіряють невеликі частини коду (`units`), щоб упевнитися, що вони працюють так, як потрібно. Наприклад:

`We wrote initial unit tests to check the main functions.` — Команда створила ранні тести, щоб переконатися, що найважливіші частини коду працюють правильно.

`Quality assurance (QA) checks` — це фінальні тести, які виконують команди забезпечення якості, щоб упевнитися, що програмне забезпечення надійне, зручне для користувача і готове до запуску. Наприклад:

`Regular QA checks help prevent bugs from reaching users.` — Часте тестування командою забезпечення якості допомагає виявляти помилки на ранніх етапах, щоб користувачі не стикалися з проблемами.

Еліс описує, як бюджет на тестування охоплює всі кроки — від початкових перевірок коду до фінального тестування перед запуском.
