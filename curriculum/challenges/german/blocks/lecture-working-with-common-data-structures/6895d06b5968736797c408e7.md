# --description--

In dieser Lektion besprechen wir Maps, Hash Maps und Sets. Aber bevor wir dazu kommen, definieren wir abstrakte Datentypen.

Ein abstrakter Datentyp (ADT) ist eine konzeptionelle Darstellung eines Datentyps, einschließlich der Operationen, die auf den Daten ausgeführt werden können, und der Eigenschaften dieser Daten.

Abstrakte Datentypen sind wie Baupläne, die beschreiben, **welche** Operationen ausgeführt werden können, nicht **wie** sie ausgeführt werden. Sie trennen die Schnittstelle von der tatsächlichen Implementierung der Operationen.

Eine **map** ist ein ADT, das Sammlungen von Schlüssel-Wert-Paaren und deren Operationen auf eine sehr spezifische und effiziente Weise verwaltet.

In einer Map ist jeder Wert mit einem bestimmten Schlüssel verknüpft.

Eine der wichtigsten Eigenschaften von Maps ist, dass jeder Schlüssel eindeutig sein muss. Diese Einzigartigkeit ermöglicht direkte Nachschlagen, was den Prozess des Abrufens von Informationen viel effizienter macht.

Nur Schlüssel müssen eindeutig sein, Werte können sich wiederholen.

Der abstrakte Datentyp Map definiert auch wichtige Operationen, wie das Einfügen von Schlüssel-Wert-Paaren, das Abrufen des Werts, der mit einem Schlüssel verknüpft ist, das Aktualisieren des Werts, das Entfernen eines Schlüssel-Wert-Paares und das Überprüfen, ob ein Schlüssel in der Map existiert.

Er legt nicht fest, wie diese Operationen ausgeführt werden sollen, sondern listet sie nur als Teil der verfügbaren Operationen des Datentyps auf.

Eine **hash map**, auch bekannt als Hashtabelle, ist eine konkrete Implementierung des abstrakten Datentyps Map.

Hash Maps verwenden eine Technik namens „Hashing“, um häufige Operationen sehr effizient auszuführen.

Hashing funktioniert im Wesentlichen, indem für jedes Element ein Hashwert mit einer Hashfunktion erzeugt wird.

Der Hashwert wird basierend auf dem Schlüssel des Schlüssel-Wert-Paares generiert und dient dazu, einen Index in einem zugrundeliegenden Array zu berechnen, der tatsächlichen Datenstruktur, in der die Schlüssel-Wert-Paare gespeichert sind.

Aber Sie fragen sich vielleicht: Was passiert, wenn zwei Schlüssel zum gleichen Index führen?

Hash Maps lösen diese Kollisionen mit cleveren Strategien.

Eine Möglichkeit ist die „Verkettung“, bei der jeder Array-Index auf eine verkettete Liste (eine andere Datenstruktur) zeigt, in der alle Elemente mit demselben Index gespeichert sind.

Eine andere Strategie ist das „Open Addressing“, bei dem nach dem nächsten verfügbaren Index im Array basierend auf einer vordefinierten Suchfolge gesucht wird.

Die durchschnittliche Zeitkomplexität von Hash Maps ist „Konstante Zeit“ `O(1)` für das Einfügen, Abrufen und Löschen von Schlüssel-Wert-Paaren.

Die schlechteste Zeitkomplexität dieser Operationen ist lineare Zeit `O(n)`, was auftritt, wenn viele Hash-Kollisionen vorliegen und die Kollisionsbehandlungsstrategie mehrfach angewendet werden muss.

Die Speicherkomplexität beim Einfügen in eine Hash Map ist im Durchschnitt konstant `O(1)`, da eine konstante Menge Speicher zum Speichern des neuen Paares benötigt wird. Im schlimmsten Fall kann sie jedoch lineare Speicherkomplexität `O(n)` aufgrund einer Größenanpassung des zugrundeliegenden Arrays haben. Im Allgemeinen hat das Entfernen eines Elements eine konstante Speicherkomplexität `O(1)`.

Dies verwandelt die Hashtabelle in etwas Ähnliches wie eine lineare Datenstruktur, bei der `n` Elemente durchsucht werden müssen, um den Zielschlüssel zu finden. Dies ist jedoch relativ selten, wenn die Hash Map richtig implementiert ist.

Pythons **Dictionaries** sind hinter den Kulissen als Hash Maps implementiert.

Um ein Python-Dictionary zu erstellen, schreiben Sie einfach die Schlüssel-Wert-Paare in geschweifte Klammern und trennen sie mit einem Komma. Jeder Schlüssel sollte durch einen Doppelpunkt vom entsprechenden Wert getrennt sein.

```python
my_dictionary = {
  'A': 1,
  'B': 2, 
  'C': 3
}
```

In diesem Code ist `'A'` der Schlüssel und `1` der Wert:

```python
'A': 1
```

Alternativ können Sie `dict()` verwenden:

```python
my_dictionary = dict(A=1, B=2, C=3)
```

Sie können den Wert über den entsprechenden Schlüssel abrufen:

```python
my_dictionary['A']  # 1
```

Sie können auch den Wert aktualisieren, der mit einem Schlüssel verknüpft ist:

```python
my_dictionary['A'] = 4
```

Und Sie können ein Schlüssel-Wert-Paar entfernen:

```python
del my_dictionary['A']  
```

Sie können auch überprüfen, ob ein Schlüssel im Dictionary (oder nicht) vorhanden ist:

```python
'C' in my_dictionary
```

Und Sie können diese Methoden aufrufen, um die Schlüssel, Werte und Elemente des Dictionaries zu erhalten.

```python
my_dictionary.keys()
my_dictionary.values()
my_dictionary.items()
```

Super. Jetzt, wo Sie mehr über Maps und Hash Maps wissen, sprechen wir über Sets.

**Sets** sind ungeordnete Sammlungen von einzigartigen Elementen.

Lassen Sie uns dieses Konzept in seine wichtigsten Bestandteile zerlegen:

* Sets sind ungeordnet. Die Elemente eines Sets werden nicht in einer bestimmten Reihenfolge gespeichert, daher können Sie nicht über Indizes auf sie zugreifen.
    
* Sets enthalten nur einzigartige Elemente. Wenn Sie versuchen, denselben Wert zweimal hinzuzufügen, wird nur eine Kopie des Werts gespeichert.
    

Sie sind analog zu Mengen in der Mathematik und implementieren dieselben Mengenoperationen wie Schnittmenge, Vereinigung und Differenz.

Ein Hauptvorteil von Sets ist, dass sie garantieren, dass die Elemente einzigartig sind (keine Duplikate). Deshalb werden sie oft verwendet, um Duplikate aus Listen und anderen Datenstrukturen zu entfernen.

Sie sind auch dynamisch. Sie können sich an die Anzahl der aktuell gespeicherten Elemente anpassen. Das macht sie sehr leistungsfähig.

Die durchschnittliche Zeitkomplexität für das Hinzufügen, Entfernen, Ermitteln der Länge des Sets und Überprüfen, ob ein Element im Set ist, ist „Konstante Zeit“ `O(1)`, was sehr effizient ist.

Da Sets als Hashtabellen implementiert sind, ist die schlechteste Zeitkomplexität für das Hinzufügen, Entfernen und Überprüfen der Mitgliedschaft „Lineare Zeit“ `O(n)`. Dies kann auftreten, wenn es viele Hash-Kollisionen gibt, wodurch die Hashtabelle zu etwas Ähnlichem wie einer linearen Datenstruktur wird, bei der `n` Durchläufe erforderlich sind, um den Schlüssel zu finden.

Bezüglich der Speicherkomplexität hat das Einfügen eines Elements im Durchschnitt konstante Komplexität `O(1)`, da ein neues einzigartiges Element eine konstante Menge Speicher benötigt. Im schlimmsten Fall kann es jedoch eine Größenanpassung des zugrundeliegenden Arrays geben, die lineare Speicherkomplexität `O(n)` erfordert. Im Allgemeinen benötigt das Entfernen eines Elements konstante Speicherkomplexität `O(1)`.

Python hat eine eingebaute `set` Datenstruktur, die Sie verwenden, um mit Sets in Ihren Programmen zu arbeiten.

Hinter den Kulissen sind Python-Sets als Hashtabelle implementiert, in der nur die Schlüssel gespeichert werden, ohne zugehörige Werte.

Sets können nur Objekte unveränderlicher Datentypen speichern, weil deren Hashwerte immer gleich bleiben. Im Gegensatz dazu können sich die Hashwerte veränderlicher Objekte ändern, wenn sie verändert werden. Deshalb können sie nicht Teil von Sets sein. Wenn sich der Hashwert eines in einem Set gespeicherten Objekts ändert, könnte das Programm es nicht mehr finden.

Um ein Set in Python zu definieren, müssen Sie die Elemente einfach mit geschweiften Klammern umgeben und mit Kommas trennen:

```python
numbers = {1, 2, 3, 4}
```

Um ein leeres Set zu erstellen, können Sie `set()` aufrufen:

```python
numbers = set()
```

Beachten Sie, dass bei Verwendung leerer geschweifter Klammern automatisch ein Python-Dictionary erstellt wird, kein Set. Deshalb müssen Sie die `set()`-Funktion aufrufen, um ein leeres Set zu erstellen.

Sie können ein Element mit der `.add()`-Methode zu einem Set hinzufügen:

```python
numbers.add(5)
```

Sie können auch Elemente mit der `.remove()`-Methode aus dem Set entfernen:

```python
numbers.remove(5)
```

Dies wirft einen `KeyError`, wenn das Element nicht gefunden wird. Wenn Sie in diesem Fall keinen Fehler werfen möchten, können Sie stattdessen die `.discard()`-Methode verwenden.

Die `.pop()`-Methode gibt ein beliebiges Element aus dem Set zurück, während die `.clear()`-Methode alle Elemente aus dem Set entfernt.

Sie können testen, ob ein Element in einem Set ist, mit dem `in`-Operator:

```python
5 in numbers
```

Python unterstützt auch Mengenoperationen, einschließlich Vereinigung, Differenz, symmetrische Differenz und Schnittmenge, die Sie mit diesen Methoden ausführen können:

```python
set_a = {1, 2, 3, 4}
set_b = {2, 3, 4, 5, 6}

set_a.union(set_b)
set_a.intersection(set_b)
set_a.symmetric_difference(set_b)
set_a.difference(set_b)
```

Oder mit den entsprechenden Operatoren:

```python
set_a | set_b
set_a & set_b
set_a ^ set_b
set_a - set_b 
```

Die durchschnittliche Zeitkomplexität für das Hinzufügen, Entfernen und Testen der Mitgliedschaft ist „Konstante Zeit“ `O(1)`.

Die schlechteste Zeitkomplexität für diese Operationen ist „Lineare Zeit“ `O(n)` aufgrund des schlimmsten Kollisionsszenarios bei Hash Maps.

Sie können auch überprüfen, ob ein Set eine Teilmenge oder Obermenge eines anderen Sets ist:

```python
set_a.issubset(set_b)
set_a.issuperset(set_b)
```

Im Allgemeinen sollten Sie Sets verwenden, wenn Sie eine Sammlung einzigartiger Elemente speichern und häufig prüfen müssen, ob ein Element vorhanden ist.

Maps, Hash Maps und Sets sind leistungsfähige Datenstrukturen, die für eine effiziente Organisation und den Abruf von Daten entwickelt wurden. Jede von ihnen hat ihre eigenen einzigartigen Eigenschaften und Anwendungsfälle. Als Entwickler müssen Sie die beste für Ihr Projekt auswählen.
