# --description--

当你用 JavaScript 编程时，你不可避免地会遇到错误信息。理解常见的错误信息将帮助你更有效地调试，并成长为更优秀的程序员。四种常见的错误信息类型是 `SyntaxError`、`ReferenceError`、`TypeError` 和 `RangeError`。

当你在代码中写错东西时，比如缺少一个括号或方括弧，就会发生 `SyntaxError`。可以把它想象成句子中的语法错误。以下是开发者在创建数组时常犯的一个错误：

```js
const arr = ["Beau", "Quincy" "Tom"]
```

每个数组元素需要用逗号分隔，否则会导致错误信息。

第二种常见的 JavaScript 误差是 `ReferenceError`。`ReferenceError` 有多种类型，由不同方式触发。第一种 `ReferenceError` 类型是未定义的变量。

```js
console.log(price);
```

在此示例中，我们尝试将 `price` 变量记录到控制台，但它尚未被定义。这将导致一个 `ReferenceError`。

另一个 `ReferenceError` 的例子是尝试在用 `let` 或 `const` 声明的变量被定义之前访问该变量：

```js
console.log(b);
const b = 50;
```

上面的示例将导致 `Cannot access 'b' before initialization` 误差。 

第三种常见的误差是 `TypeError`。当你尝试对错误的类型执行操作时，会发生这些误差。下面是一个尝试在对象上使用 `map` 方法的例子：

```js
const developerObj = {
  name: "Jessica",
  country: "USA",
  isEmployed: true
};

developerObj.map()
```

此示例将导致 `developerObj.map is not a function` 误差，因为 `map` 方法用于数组而非对象。 

我们将要看的最后一个常见误差是 `RangeError`。

当你的代码尝试使用超出 JavaScript 可处理范围的值时，会发生 `RangeError`。下面是将无效值赋值给数组长度的示例：

```js
const arr = [];
arr.length = -1; 
```

由于数组的 `length` 必须是非负整数，将其设置为 `-1` 会触发 `RangeError`。

当你继续用 JavaScript 编程时，请注意你可能遇到的这些不同类型的误差以及它们发生的原因。
