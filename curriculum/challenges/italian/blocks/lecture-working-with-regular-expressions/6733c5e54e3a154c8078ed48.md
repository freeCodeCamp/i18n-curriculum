# --interactive--

Impariamo a conoscere i quantificatori nelle espressioni regolari.

Considera uno scenario in cui vuoi trovare un codice identificativo di quattro cifre. Sai che puoi usare la classe di caratteri `\d`, quindi potresti scriverla quattro volte. E per evitare caratteri estranei, includeresti sia l’àncora di inizio che quella di fine:

```js
const regex = /^\d\d\d\d$/;
```

E questo funziona – corrisponderà a quattro caratteri numerici. Ma invece di scrivere la stessa classe più volte, puoi usare un quantificatore.

I quantificatori si definiscono con parentesi graffe contenenti uno o due numeri. Usiamo un quantificatore nel nostro pattern:

```js
const regex = /^\d{4}$/;
```

Nota come il nostro quantificatore contiene solo il numero `4`. Questa sintassi significa "corrispondi esattamente quattro volte al carattere precedente". Vediamo come si comporta:

:::interactive_editor

```js
const regex = /^\d{4}$/;
console.log(regex.test("123")); // false
console.log(regex.test("1234")); // true
console.log(regex.test("12345")); // false
console.log(regex.test("123456")); // false
console.log(regex.test("1234567")); // false
```

:::

Il pattern corrisponde solo alla stringa con esattamente quattro cifre, perché abbiamo usato le àncore e il nostro quantificatore permette solo esattamente quattro cifre. Ma forse il codice identificativo deve essere almeno di quattro cifre.

Per permettere quattro o più cifre, aggiungi una virgola dopo il numero nel tuo quantificatore:

```js
const regex = /^\d{4,}$/;
```

Ora la nostra sintassi permette al pattern di corrispondere a quattro o più cifre. Proviamolo:

:::interactive_editor

```js
const regex = /^\d{4,}$/;
console.log(regex.test("123")); // false
console.log(regex.test("1234")); // true
console.log(regex.test("12345")); // true
console.log(regex.test("123456")); // true
console.log(regex.test("1234567")); // true
```

:::

Un identificatore di sette cifre è piuttosto lungo. Questi identificatori dovrebbero avere un massimo di 6 cifre e un minimo di 4 cifre. Per ottenere questo, puoi aggiungere un secondo numero al tuo quantificatore dopo la virgola:

```js
const regex = /^\d{4,6}$/;
```

E ora il nostro pattern non corrisponde più all’identificatore di sette cifre, perché supera il massimo di sei cifre.

:::interactive_editor

```js
const regex = /^\d{4,6}$/;
console.log(regex.test("123")); // false
console.log(regex.test("1234")); // true
console.log(regex.test("12345")); // true
console.log(regex.test("123456")); // true
console.log(regex.test("1234567")); // false
```

:::

Nota che non puoi usare questa sintassi per impostare solo un massimo – devi sempre impostare un minimo. Ma se imposti il minimo a `1`, puoi ottenere lo stesso risultato.

Abbiamo ricevuto requisiti aggiornati dagli utenti. Gli identificatori possono ora opzionalmente iniziare con una lettera. Conosciamo già la classe di caratteri per questo, quindi aggiungiamola alla nostra espressione regolare:

```js
const regex = /^[a-zA-Z]\d{4,6}$/;
```

Ma ora richiediamo la presenza di una lettera. Come possiamo renderla opzionale?

Potresti usare la sintassi del quantificatore con `0` come minimo e `1` come massimo:

```js
const regex = /^[a-zA-Z]{0,1}\d{4,6}$/;
```

Ma in realtà esiste un quantificatore speciale abbreviato per un singolo carattere opzionale – il punto interrogativo (`?`). Sostituiamo il nostro quantificatore con il punto interrogativo:

```js
const regex = /^[a-zA-Z]?\d{4,6}$/;
```

Dobbiamo validare il risultato:

:::interactive_editor

```js
const regex = /^[a-zA-Z]?\d{4,6}$/;
console.log(regex.test("123")); // false
console.log(regex.test("a1234")); // true
console.log(regex.test("12345")); // true
console.log(regex.test("az12345")); // false
console.log(regex.test("X123456")); // true
console.log(regex.test("1234567")); // false
```

:::

Il nostro pattern ora permette una singola lettera opzionale, seguita da quattro a sei cifre.

Sfortunatamente, ci siamo appena accorti di aver interpretato male i requisiti. Dobbiamo permettere un numero qualsiasi di lettere prima dei numeri. Possiamo usare il nostro quantificatore con un minimo `0` e nessun massimo definito:

```js
const regex = /^[a-zA-Z]{0,}\d{4,6}$/;
```

Ma il nostro pattern sta diventando di nuovo lungo. Per fortuna, esiste un altro modo abbreviato per "corrispondi al carattere precedente zero o più volte" – il simbolo asterisco (`*`). Sostituiamo il nostro quantificatore con questo nel pattern e testiamolo:

:::interactive_editor

```js
const regex = /^[a-zA-Z]*\d{4,6}$/;
console.log(regex.test("123")); // false
console.log(regex.test("a1234")); // true
console.log(regex.test("12345")); // true
console.log(regex.test("az12345")); // true
console.log(regex.test("X123456")); // true
console.log(regex.test("1234567")); // false
```

:::

Ora corrispondiamo con successo a qualsiasi identificatore con zero o più lettere seguite da quattro a sei numeri. Ma si scopre che questo sta causando un crash al nostro sistema – dobbiamo effettivamente richiedere almeno una lettera.

Ancora una volta, potremmo usare un quantificatore con minimo uno e nessun massimo definito, oppure potremmo usare un’altra sintassi speciale – il simbolo più (`+`):

:::interactive_editor

```js
const regex = /^[a-zA-Z]+\d{4,6}$/;
console.log(regex.test("123")); // false
console.log(regex.test("a1234")); // true
console.log(regex.test("12345")); // false
console.log(regex.test("az12345")); // true
console.log(regex.test("X123456")); // true
console.log(regex.test("1234567")); // false
```

:::

Ora gli identificatori che non iniziano con almeno una lettera falliscono, indipendentemente da quanti numeri ci siano.

Puoi usare i quantificatori per migliorare molto la brevità e la leggibilità delle tue espressioni regolari.
