# --interactive--

프로그래밍에서 루프는 코드 블록을 여러 번 반복하는 데 사용됩니다.

루프의 예로는 항목 목록을 출력해야 하는 프로그램을 설계할 때가 있습니다. 이때 루프를 사용해 목록의 각 항목을 출력할 수 있습니다.

또 다른 예는 게임을 설계할 때 캐릭터를 화면을 가로질러 움직이게 하는 경우입니다. 루프를 사용해 루프가 실행될 때마다 캐릭터를 일정 픽셀만큼 이동시킬 수 있습니다.

JavaScript에는 사용할 수 있는 여러 종류의 루프가 있습니다. 이 수업에서는 `for` 루프를 다룹니다. `for` 루프의 기본 구문은 다음과 같습니다:

```js
for (initialization; condition; increment or decrement) {
  // code block to be executed
}
```

초기화 문장은 루프가 시작되기 전에 실행됩니다. 보통 카운터 변수를 초기화하는 데 사용합니다. 카운터 변수는 루프가 몇 번 실행되었는지 추적하는 데 쓰이는 변수입니다.

조건문은 루프의 각 반복 전에 평가됩니다. 반복은 루프를 한 번 통과하는 것을 의미합니다.

조건이 참이면 루프 내부의 코드 블록이 실행됩니다. 조건이 거짓이면 루프가 정지하고 다음 코드 블록으로 넘어갑니다.

루프의 마지막 부분은 증가/감소 문장입니다. 이 문장은 루프의 각 반복 후에 실행됩니다. 보통 카운터 변수를 증가시키거나 감소시키는 데 사용합니다.

:::interactive_editor

```js
for (let i = 0; i < 5; i++) {
  console.log(i);
}
```

:::

위 예제의 첫 부분에서는 카운터 변수 `i`를 `0`으로 초기화합니다. `i` 루프에서 `for`를 카운터 변수로 사용하는 것이 일반적인 관례입니다.

다음은 조건을 확인하는 부분입니다. 이 경우 조건은 `i`가 `5`보다 작은지 확인합니다. `i`가 `0`이므로 조건이 참이고, 루프 내부의 코드 블록이 실행됩니다.

루프 내부의 코드 블록은 `i`의 값을 콘솔에 출력하는 것입니다. `i`의 값이 `0`이므로 콘솔에는 `0`의 값이 표시됩니다.

그다음 증가 문장이 실행됩니다. 이 경우 `i`를 `1`만큼 증가시킵니다. 그래서 `i`는 이제 `1`입니다.

다시 조건을 확인하는데, `i`가 `5`보다 작은지 검사합니다. `i`가 이제 `1`이므로 조건이 여전히 참이고, 루프 내부의 코드 블록이 다시 실행됩니다.

이 과정을 조건이 거짓이 될 때까지 반복합니다. 이 경우 `i`가 `5`이 되면 조건이 거짓이 되어 루프가 정지합니다.

루프를 사용할 때 항상 참인 조건을 만들지 않도록 주의해야 합니다. 그렇게 되면 루프가 무한히 실행되어 프로그램이 충돌할 수 있습니다. 이를 무한 루프라고 합니다.

중첩된 `for` 루프를 만드는 것도 가능합니다. 중첩 루프는 한 루프 안에 다른 루프를 넣는 것입니다. 나중에 이런 경우의 예제를 살펴보겠습니다.

루프는 코드 블록을 일정 횟수 반복해야 할 때 프로그래밍에서 유용합니다. `for` 루프를 처음 다루는 것은 까다로울 수 있지만, 연습하면 익숙해질 것입니다.
