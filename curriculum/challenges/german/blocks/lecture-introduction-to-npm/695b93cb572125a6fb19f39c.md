# --description--

In einer vorherigen Lektion haben Sie gelernt, was Abhängigkeiten sind und wie Sie die richtigen für Ihr Projekt auswählen.

In dieser Lektion lernen Sie, wie Sie Abhängigkeiten mit npm installieren und entfernen.

Um eine Abhängigkeit zu installieren, verwenden Sie den `npm install`-Befehl so:

```bash
npm install dependencyName
```

Sie können `install` auch auf `i` abkürzen, so:

```bash
npm i dependencyName
```

Nach der Installation eines Pakets fügt npm einen neuen `dependencies`-Abschnitt zu Ihrer `package.json`-Datei hinzu (falls dieser noch nicht vorhanden ist) und listet das installierte Paket auf. Zum Beispiel sieht es so aus, wenn Sie das `chalk`-Paket installieren:

```json
{
  "name": "install-uninstall-deps",
  "version": "1.0.0",
  "description": "a simple demo on how to install and uninstall dependencies",
  "main": "index.js",
  "scripts": {
    "test": "echo \"Error: no test specified\" && exit 1"
  },
  "keywords": ["npm", "deps"],
  "author": "Kolade Chris",
  "license": "ISC",
  "dependencies": {
    "chalk": "^5.6.2"
  }
}
```

Wenn Sie eine Abhängigkeit installieren, erscheinen zwei neue Dinge im Stammverzeichnis Ihres Projekts:

* `node_modules/`: ein Ordner, der den eigentlichen Code der installierten Pakete enthält.
    
* `package-lock.json`: eine Datei, die detaillierte Informationen über die genauen Versionen dieser Pakete speichert, einschließlich ihrer eigenen Abhängigkeiten. (Mehr zu dieser Datei lernen Sie in einer späteren Lektion.)
    

<img src="https://cdn.freecodecamp.org/curriculum/lecture-transcripts/how-can-you-install-and-remove-dependencies-1.png" alt="">

Die meisten Projekte benötigen mehr als eine Abhängigkeit. Statt sie einzeln zu installieren, können Sie mit npm mehrere Pakete gleichzeitig installieren, indem Sie eine durch Leerzeichen getrennte Liste von Paketnamen eingeben:

```bash
npm install express lodash bcrypt
```

Wie zuvor aktualisiert npm beim gleichzeitigen Installieren mehrerer Abhängigkeiten Ihre `package.json`- und `package-lock.json`-Dateien sowie Ihren `node_modules`, um sie einzuschließen. So würde die `package.json` aussehen:

```json
{
  "name": "install-uninstall-deps",
  "version": "1.0.0",
  "description": "a simple demo on how to install and uninstall dependencies",
  "main": "index.js",
  "scripts": {
    "test": "echo \"Error: no test specified\" && exit 1"
  },
  "keywords": ["npm", "deps"],
  "author": "Kolade Chris",
  "license": "ISC",
  "dependencies": {
    "bcrypt": "^6.0.0",
    "chalk": "^5.6.2",
    "express": "^5.1.0",
    "lodash": "^4.17.21"
  }
}
```

Um eine `devDependency` zu installieren, fügen Sie dem `--save-dev`-Befehl die `install`-Option hinzu. Sie können auch die kürzere `-D`-Version verwenden:

```bash
npm i nodemon -D
```

Nach Ausführung dieses Befehls aktualisiert npm Ihre `package.json`-Datei und fügt das Paket unter dem `devDependencies`-Abschnitt hinzu. So sieht das aus:

```json
{
  "name": "install-uninstall-deps",
  "version": "1.0.0",
  "description": "a simple demo on how to install and uninstall dependencies",
  "main": "index.js",
  "scripts": {
    "test": "echo \"Error: no test specified\" && exit 1"
  },
  "keywords": ["npm", "deps"],
  "author": "Kolade Chris",
  "license": "ISC",
  "dependencies": {
    "bcrypt": "^6.0.0",
    "chalk": "^5.6.2",
    "express": "^5.1.0",
    "lodash": "^4.17.21"
  },
  "devDependencies": {
    "nodemon": "^3.1.10"
  }
}
```

Wann immer Sie einen `npm install`-Befehl ausführen, zeigt npm eine Meldung an, die Ihnen mitteilt, ob die Installation erfolgreich war oder nicht. Eine Erfolgsmeldung sieht normalerweise so aus:

```bash
user@Kolade ~/Desktop/fCC/script-code/node/npm/install-uninstall-deps  % npm install express lodash bcrypt

added 72 packages, and audited 74 in 8s

17 packages are looking for funding
  run `npm fund` for details

found 0 vulnerabilities

user@Kolade ~/Desktop/fCC/script-code/node/npm/install-uninstall-deps  % npm install nodemon -D

added 27 packages, and audited 101 in 16s

21 packages are looking for funding
  run `npm fund` for details

found 0 vulnerabilities
```

Um ein Paket zu deinstallieren oder zu entfernen, verwenden Sie den `npm uninstall`-Befehl:

```bash
npm uninstall dependencyName
```

Zum Beispiel, um `chalk` aus Ihrem Projekt zu entfernen, führen Sie diesen Befehl aus:

```bash
npm uninstall chalk
```

Sie können auch mehrere Pakete gleichzeitig deinstallieren, indem Sie sie mit Leerzeichen getrennt auflisten:

```bash
npm uninstall express lodash bcrypt
```

Sie erhalten ebenfalls eine Meldung, die anzeigt, ob der Deinstallationsbefehl erfolgreich war oder fehlgeschlagen ist:

```bash
user@Kolade ~/Desktop/fCC/script-code/node/npm/install-uninstall-deps  % npm uninstall chalk

added 1 package, removed 1 package, and audited 101 packages in 1s

20 packages are looking for funding
  run `npm fund` for details

found 0 vulnerabilities

user@Kolade ~/Desktop/fCC/script-code/node/npm/install-uninstall-deps  % npm uninstall express lodash bcrypt

removed 70 packages, and audited 31 in 1s

4 packages are looking for funding
  run `npm fund` for details

found 0 vulnerabilities
```

Und Ihre `package.json`-Datei wird automatisch aktualisiert:

```json
{
  "name": "install-uninstall-deps",
  "version": "1.0.0",
  "description": "a simple demo on how to install and uninstall dependencies",
  "main": "index.js",
  "scripts": {
    "test": "echo \"Error: no test specified\" && exit 1"
  },
  "keywords": ["npm", "deps"],
  "author": "Kolade Chris",
  "license": "ISC",
  "devDependencies": {
    "nodemon": "^3.1.10"
  }
}
```

Um eine `devDependency` zu deinstallieren, können Sie einfach Folgendes ausführen:

```bash
npm uninstall nodemon
```

Beachten Sie, dass Sie beim Deinstallieren nicht die `-D`-Option benötigen. npm entfernt das Paket automatisch aus dem richtigen Abschnitt, in diesem Fall `devDependencies`.

Nach der Deinstallation der `devDependency` wird Ihre `package.json` erneut aktualisiert:

```json
{
  "name": "install-uninstall-deps",
  "version": "1.0.0",
  "description": "a simple demo on how to install and uninstall dependencies",
  "main": "index.js",
  "scripts": {
    "test": "echo \"Error: no test specified\" && exit 1"
  },
  "keywords": ["npm", "deps"],
  "author": "Kolade Chris",
  "license": "ISC"
}
```
