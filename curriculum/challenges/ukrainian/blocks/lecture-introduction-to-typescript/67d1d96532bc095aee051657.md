# --description--

JavaScript вважається мовою з динамічною типізацією. Це означає, що змінні можуть отримувати будь-які значення під час виконання — оголошення змінної як числа не забороняє призначити їй рядок пізніше, а параметри функції можуть приймати будь-які значення.

Проблема динамічно типізованої мови полягає в тому, що відсутність типобезпеки може призвести до помилок, якщо не бути обережним. TypeScript розширює мову JavaScript, додаючи статичну типізацію, яка допомагає виявляти ці помилки ще до розгортання коду. Але як це працює?

Розглянемо цю функцію JavaScript:

```js
const getRandomValue = (array) => {
  return array[Math.floor(Math.random() * array.length)];
}
```

Наша функція `getRandomValue` приймає масив і повертає випадкове значення з цього масиву. Але оскільки JavaScript не перевіряє типи, нічого не заважає викликати функцію з числом:

```js
console.log(getRandomValue(10));
```

Вихідні дані в консолі для цього прикладу будуть `undefined`, бо функція очікувала масив, а отримала число.

Але з TypeScript ви можете визначити тип для параметра масиву:

```js
const getRandomValue = (array: string[]) => {
  return array[Math.floor(Math.random() * array.length)];
}
```

Це визначення типу повідомляє TypeScript, що аргумент масиву має бути масивом рядків. Тоді, коли ви викликаєте `getRandomValue` і передаєте число, ви отримуєте інший тип помилки — помилку компілятора.

Більшість середовищ виконання JavaScript, таких як браузер або Node.js, не можуть запускати TypeScript напряму. Замість цього спочатку потрібно скомпілювати або транслювати код TypeScript у звичайний JavaScript. Ви можете зробити це за допомогою вбудованого компілятора, який постачається з мовою TypeScript. Коли ви запускаєте компілятор, TypeScript перевіряє ваш код і видає помилку для будь-яких проблем, де типи не співпадають — наприклад, якщо передати число у функцію, яка очікує масив.

Ви дізнаєтеся більше про те, як працюють ці типи, у наступних кількох уроках. Але саме ця додаткова безпека робить TypeScript привабливою мовою для багатьох розробників і організацій.
