# --description--

CRUD 是一个缩写，代表 `Create`、`Read`、`Update` 和 `Delete`。这四个是持久存储的四种操作。

持久存储是指以一种即使断电或设备重启后仍能使用的方式保存数据。

理解 CRUD 的基本操作如何工作对于网页开发至关重要，因为它构成了操作数据库和创建允许用户添加、查看、修改和删除数据的应用的基础。 

现在，让我们更仔细地看看 CRUD 的每个部分。

- 创建是指创建新数据的过程。例如，在一个网页应用中，这可能是当用户为博客添加新帖子时。

- 读取是从数据库中检索数据的操作。例如，当你访问博客文章或查看你的网站评测时，你正在执行读取操作以获取并显示存储在数据库中的数据。

- 更新涉及修改数据库中现有的数据。一个例子是编辑博客文章或更新你的评测信息。

- 删除是从数据库中移除数据的操作。例如，当你删除一篇博客文章或账户时，你正在执行删除操作。

CRUD 用于处理数据库、用户界面和 RESTful API。RESTful API 是一套用于创建网络服务的约定，允许客户通过标准的超文本传输协议方法执行 CRUD 操作，从而与数据库或后端系统交互。

HTTP 代表超文本传输协议，它是网络上数据通信的基础。存在 HTTP 方法，用于定义可以在网络资源上执行的操作。常见的方法有 `GET`、`POST`、`PUT`、`PATCH`、`DELETE`。

你将在后续课程中学习更多关于 RESTful APIs 和超文本传输协议的内容，但这里是 CRUD 如何映射到不同超文本传输协议方法的快速概述。

- `POST` 用于创建一个新资源。

- `GET` 用于检索或读取数据。

- `PUT` 用于通过完全替换来更新资源。

- `PATCH` 用于部分更新资源。

- `DELETE` 用于删除资源。

下面是一个示例，展示了如何使用 JavaScript 中的简单数组在代码中表现 CRUD 操作：

```js
let items = [];

// Create
function createItem(item) {
  items.push(item);
}

// Read
function readItems() {
  return items;
}

// Update
function updateItem(index, newItem) {
  items[index] = newItem;
}

// Delete
function deleteItem(index) {
  items.splice(index, 1);
}

// Example Usage
createItem('Book');
console.log(readItems()); // ['Book']
updateItem(0, 'Magazine');
console.log(readItems()); // ['Magazine']
deleteItem(0);
console.log(readItems()); // []
```

下面为例：

- 我们通过将项推入数组来创建项。

- 我们通过返回数组来读取项。

- 我们通过修改给定索引处的数组元素来更新一项。

- 我们通过使用 `splice()` 从数组中删除一项。

这是 CRUD 操作在概念层面如何工作的基本表现。
