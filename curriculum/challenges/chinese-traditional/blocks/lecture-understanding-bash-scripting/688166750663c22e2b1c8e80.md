# --description--

在之前的課程中，你學習了 Bash 和一些你可能會使用的常見指令。但 Bash 支援完整的腳本語言，你可以用來執行更複雜的自動化任務。

Bash 腳本撰寫涉及在一個檔案中編寫一連串的 Bash 指令，然後你可以使用 Bash 來執行該檔案的內容。

讓我們來看看 bash 腳本可能的樣子：

```bash
#!/bin/bash

servers=("prod" "dev")

for server in "${servers[@]}"
do
  echo "Pulling $server"
  rsync --archive --verbose $server:/etc/nginx/conf.d/server.conf configs/$server.conf
done
```

此腳本會從遠端伺服器列表中擷取 NGINX 設定檔案，這對於將這些設定備份到區域的資料庫非常有用。NGINX 是一個流行的網頁伺服器，你將在未來的模組中學習更多關於操作伺服器的內容。 

第一行稱為「shebang」（`#!`），用來告訴系統應該使用哪個直譯器來執行腳本。這裡常見的值有 `/usr/bin/bash`、`/bin/bash` 或 `/bin/sh`。

第一行，`servers=("prod" "dev")`，實例化一個字串列表，表述伺服器名稱。`"prod"` 通常用來指涉生產環境，而 `"dev"` 指涉開發環境，通常用於在部署到實際生產環境之前測試新特性和變更。

然後，使用 `for` 迴圈迭代該列表——`servers[@]` 語法將列表展開為單獨的元素。每次迭代時，將每個元素指定給 server。

`do` 述語顯示迴圈邏輯區塊的開始，或每次迭代時應該執行的程式碼。

`echo "Pulling $server"` 會插入目前 `server` 變數的值，在此情況下會是 prod 或 dev，並將其記錄到終端機。

然後使用 `rsync` 從伺服器拉取實際的檔案並區域地存檔，使用更多插值來管理該指令。

`done` 顯示迴圈邏輯區塊的結束。

了解並運用 Bash 腳本有許多重要的優勢。

第一點是幾乎每個 Unix 環境都預設提供 Bash，這表示你可以直接執行你的腳本，而不需要組態任何像 Node.js 或 Python 這類的語言執行期。

第二點是你可以輕鬆存取系統上所有可用的二進位應用程式。如果你的系統已安裝 doctl，你的腳本就能執行該二進位檔。二進位檔是電腦執行的已編譯程式。當你有 bash 腳本時，你可以執行該二進位檔，這將啟動程式。

最後，你用於腳本的語法也可以直接在終端機中執行。這表示測試腳本中的小部分變得更簡單－你可以將該部分直接複製並貼上到終端機中，然後查看結果！

理解 Bash 不僅會提升你在腳本撰寫的遊戲層級，還會讓你成為命令列的高手！
