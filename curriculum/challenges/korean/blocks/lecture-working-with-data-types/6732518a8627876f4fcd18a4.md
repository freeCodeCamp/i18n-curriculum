# --interactive--

JavaScript에서 `typeof` 연산자는 변수나 값의 데이터 유형을 확인할 수 있는 간단하지만 강력한 도구입니다. 항상 유형을 나타내는 문자열을 반환합니다.

몇 가지 예제를 살펴보겠습니다:

:::interactive_editor

```js
let num = 42;
console.log(typeof num); // "number"
```

:::

첫 번째 예제에서는 `num`라는 변수를 만들고 숫자 `42`를 할당했습니다. `typeof`라는 변수에 `num` 연산자를 사용하면 문자열 `number`를 반환합니다.

다음은 `typeof`라는 변수에 `isUserLoggedIn` 연산자를 사용한 또 다른 예입니다:

:::interactive_editor

```js
let isUserLoggedIn = true;
console.log(typeof isUserLoggedIn); // "boolean"
```

:::

`typeof` 변수에 `isUserLoggedIn` 연산자를 사용하면, 변수에 불리언 `boolean`가 할당되어 있기 때문에 문자열 `true`를 반환합니다.

`typeof` 연산자는 디버깅하거나 코드에서 어떤 종류의 데이터를 다루고 있는지 이해하려 할 때 특히 유용합니다.

하지만 JavaScript에서 `null`에 관한 잘 알려진 특이점이 있습니다.

예제를 살펴보겠습니다:

:::interactive_editor

```js
let exampleVariable = null;
console.log(typeof exampleVariable); // "object"
```

:::

이 예제에서는 `exampleVariable`라는 변수가 있고 `null` 값을 할당했습니다. 그런데 `typeof` 연산자를 사용하면 문자열 `object`를 반환합니다.

이것은 JavaScript 초창기부터 존재한 버그로 널리 알려져 있습니다. 이런 동작의 원인은 JavaScript가 처음 설계될 때의 방식에 뿌리를 두고 있습니다.

언어가 처음 구현될 때, `null` 같은 값들은 특별한 유형의 객체로 나타내져서 이런 예상치 못한 결과가 발생했습니다.

안타깝게도 이것은 언어의 일부가 되었고, 혼란스럽지만 반드시 알아야 할 부분입니다.
