# --description--

Crypto는 Node.js에 내장된 또 다른 핵심 모듈입니다. 해싱, 암호화, 복호화, 디지털 서명 생성 등 민감한 정보를 보호하고 앱을 안전하게 유지하는 데 사용되는 도구들을 포함합니다.

하지만 `crypto`는 플러그 앤 플레이 보안 기능이 아니라 저수준 빌딩 블록을 제공합니다. 직접 암호화나 인증 코드를 작성하는 것은 주의하지 않으면 안전하지 않을 수 있습니다. 대부분의 경우, 비밀번호 해싱에는 `bcrypt` 같은 잘 검증된 라이브러리를, 로그인과 토큰 처리에는 `jsonwebtoken` (JWT)를 사용하는 것이 가장 좋습니다.

그럼에도 불구하고 crypto 모듈의 일부 메서드가 어떻게 작동하는지 이해하는 것은 여전히 유용합니다.

이 메서드들을 사용하려면 `crypto` 모듈을 가져와야 합니다:

```js
const crypto = require("crypto");
```

일부 메서드는 해싱과 데이터 암호화 같은 데이터 변환 용도로 사용되고, 다른 메서드들은 난수 생성과 비밀 키 생성 같은 키 및 보안 관리에 사용됩니다.

먼저 데이터 해싱과 암호화에 관한 메서드를 살펴보겠습니다.

`createHash()` 메서드는 `sha256`, `sha512`, `md5` 같은 알고리즘을 받아 해시 객체를 생성합니다. 이 작업은 단방향이므로 역으로 되돌릴 수 없습니다.

`createHash()`는 비밀번호 해싱과 파일 지문 생성에 유용합니다. 사용하려면 알고리즘을 전달하고, `update()` 메서드로 데이터를 입력한 뒤, `digest()` 메서드에 인코딩을 지정해 해시 값을 얻습니다:

```js
const crypto = require("crypto");

const hashedPassword = crypto
  .createHash("sha256")
  .update("myStrongPassword")
  .digest("hex");

console.log("createHash result:", hashedPassword);
// createHash result: f92c9cfa0ead1bcec05ca75888a4074ba994ad237e5e2a8c7cc6a620378c061d
```

`createHmac()`은 `createHash()`와 거의 동일하지만, 비밀 키를 받아서 그 키를 가진 사람만 해시를 검증할 수 있게 하므로 인증과 데이터 무결성 검증에 이상적입니다:

```js
const crypto = require("crypto");

const hashedMessage = crypto
  .createHmac("sha256", "secretkey")
  .update("important-secret-message")
  .digest("hex");

console.log("createHmac result:", hashedMessage);
// createHmac result: da48d6f026b6036286b1fb872c63264130d5cc4271f3a213bb6ddca5a023e77e
```

`createCipheriv()`과 `createDecipheriv()` 메서드는 데이터를 암호화하고 복호화합니다. 둘 다 알고리즘, 키, 그리고 암호화 과정 시작 시 사용하는 난수 또는 고유 데이터 블록인 `iv`를 인수로 받습니다:

```js
createCipheriv(algorithm, key, iv);
createDecipheriv(algorithm, key, iv);
```

복호화하려면 키가 동일해야 하며, 그렇지 않으면 복호화가 실패해 오류가 발생하거나 원본 메시지 대신 읽을 수 없는 데이터가 나옵니다.

다음은 두 메서드의 실제 사용 예입니다:

```js
const crypto = require("crypto");

// A key must match the algorithm length. Here AES-256 is 32 bytes
const key = Buffer.from("12345678901234567890123456789012");

// A fixed IV, 16 bytes for AES
const iv = Buffer.from("1234567890123456");

const cipher = crypto.createCipheriv("aes-256-cbc", key, iv);

let encrypted = cipher.update("Hello campers!", "utf8", "hex");
encrypted += cipher.final("hex");

console.log("Encrypted data:", encrypted);
// Encrypted data: 4ee93aa398ab44e3540e4a67ca96bc8c

// Decrypt the "Hello campers!" message
const decipher = crypto.createDecipheriv("aes-256-cbc", key, iv);
let decrypted = decipher.update(encrypted, "hex", "utf8");
decrypted += decipher.final("utf8");

console.log("Decrypted data:", decrypted);
// Decrypted data: Hello campers!
```

또 다른 데이터 변환용 crypto 메서드는 `sign()`와 `verify()`입니다.

`sign()`는 개인 키를 사용해 데이터로부터 디지털 서명을 생성합니다. 이 서명은 데이터가 개인 키 소유자로부터 왔으며 변조되지 않았음을 증명합니다.

`verify()`는 그 서명을 검증하며, 데이터나 서명이 일치하지 않으면 실패합니다.

이제 난수 생성과 비밀 키 생성용 crypto 메서드를 살펴보겠습니다.

`randomBytes()`는 `size`를 받아 암호학적으로 안전한 토큰을 생성합니다. 이 때문에 UUID(범용 고유 식별자)를 생성하는 데 적합합니다. 또한 토큰과 키에 안전하지 않은 `Math.random()`를 대체할 수 있습니다.

```js
console.log("Random Bytes:", crypto.randomBytes(16));
// Random Bytes: <Buffer 01 88 aa 1e 2c 38 48 39 26 e1 6b a9 d8 c5 ed 49>
```

출력은 기본적으로 버퍼입니다. Buffer 모듈 수업에서 배운 것처럼, `toString()` 메서드로 그 버퍼를 문자열로 변환할 수 있습니다:

```js
console.log("Random Bytes:", crypto.randomBytes(16).toString("hex"));
// Random Bytes: a6154ef5a296fa176ad0f332bd94d712
```

여기서 `'hex'` 인수는 `toString('hex')` 메서드에서 버퍼의 이진 데이터를 16진수 문자열로 인코딩하도록 Node에 지시합니다.

`randomInt()` 메서드는 `min`와 `max` 값을 받아 그 사이의 안전한 난수 정수를 생성합니다. OTP와 무작위 선택에 유용합니다.

```js
console.log("Random Int:", crypto.randomInt(0, 100)); // 89
```

다시 말하지만, `Math.random()`와 `Math.floor()`보다 업그레이드된 점은 이 메서드가 내부적으로 암호학적으로 안전한 난수를 사용해 공격자가 결과 난수를 예측할 수 없다는 점입니다.

또 다른 메서드는 `createSecretKey()`입니다. 버퍼를 받아 `KeyObject`로 감싼 원시 바이트를 생성합니다:

```js
const crypto = require("crypto");

const secret = crypto.createSecretKey(crypto.randomBytes(32));
console.log(secret); // SecretKeyObject [KeyObject] { [Symbol(kKeyType)]: 'secret' }
```

그다음 `export` 메서드로 그 `KeyObject`를 전송할 수 있습니다:

```js
const secret = crypto.createSecretKey(crypto.randomBytes(32));
console.log(secret.export());
// <Buffer 53 06 a1 c7 75 69 8b 38 8b a4 b2 f7 1b bc b8
// ae e2 d1 bf 67 af 1a 6a 0a 6e a0 29 62 bb 52 52 32>
```

마지막으로, 버퍼에 `toString()`를 사용해 문자열 표현을 확인할 수 있습니다:

```js
const secret = crypto.createSecretKey(crypto.randomBytes(32));
console.log(secret.export().toString('hex'));
// 32dfe5917668580160986f1623bf8152913329c71163be9c3404a110cd78efd6
```

이 외에도 다음과 같은 메서드가 있습니다:

- `createPublicKey()`와 `createPrivateKey()`는 외부에서 생성된 키를 다룰 수 있게 합니다
- `createDiffieHellman()`는 두 당사자가 비밀을 직접 보내지 않고 공유 비밀을 생성할 수 있게 합니다
- `Certificate()`는 HTTPS에서 사용하는 키를 다루며, 인증서 내용을 파싱, 내보내기, 검증할 수 있습니다
