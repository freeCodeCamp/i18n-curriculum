# --description--

式 `first things first` は、何よりも先に重要なことを行うべきだと強調するために使われます。これはタスクの最初に言われることが多く、最も重要なステップにまずフォーカスすることを意味します。例えば、`First things first, let's review the project plan` は他のタスクに移る前に計画をレビューすることから始めるという意味です。

技術的な文脈での `environment` は、システムが動作するソフトウェアやハードウェアの条件のセットを指すことが多いです。例えば、`development environment` はプログラマーがコードを書いてテストする場所です。
