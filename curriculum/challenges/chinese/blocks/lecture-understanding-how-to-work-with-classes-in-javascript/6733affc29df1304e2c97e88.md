# --interactive--

在现代 JavaScript 中，`class` 就像蓝图，你可以在 `code` 中定义它来创建多个相同类型的 `object`。

例如，如果你正在为库存系统建模，你需要为每种产品类型创建多个对象。

每个单独的产品都会有它自己的属性和动作，但同一类的产品可以从相同的 `blueprint` 创建。这就是 `class` 的用途。

类可以定义为类表达式或类声明。

这是一个类声明的基本语法：

```js
class MyClassName {
  // Class Methods
  constructor() { ... }
  method1() { ... }
  method2() { ... }
  ...
}
```

你首先编写 `class` 关键字，后跟类的名称。命名类时，你应使用 `PascalCase` 命名约定，而不是 `camelCase`。

花括弧内的内容称为类体。这是构造函数和方法所在的位置。

构造函数是一个特殊的方法，当从类中新建一个对象时会自动调用。它通常用于初始化对象的属性。

在这个例子中，你可以看到一个带有构造函数的 `Dog` 类。

```js
class Dog {
  constructor(name) {
    this.name = name;
  }
}
```

这里使用 `this` 关键字来参考当前的 `Dog` 类实例，允许你为正在创建的对象赋值 `name` 属性。你将在后续课程中学习更多关于 `this` 关键字的内容。

在构造函数下面，你可以有称为方法的方法。方法是在类内部定义的函数，用于执行动作或操作。它们用于定义类的实例可以执行的行为。

这里是为你的 `Dog` 类添加一个名为 `bark` 的方法的示例。

```js
class Dog {
  constructor(name) {
    this.name = name;
  }

  bark() {
    console.log(`${this.name} says woof!`);
  }
}
```

当调用 `bark` 方法时，消息将使用传入的实际名称记录到控制台。如果你创建一个名为 `"Buddy"` 的 `Dog` 实例，调用该实例上的 `bark()` 方法将会在控制台打印 `"Buddy says woof!"`。

一旦你定义了你的类，你可以使用 `new` 关键字、类名及其参数来创建该类的实例。

在此示例中，我们创建一个名为 `Gino` 的新 `Dog` 实例，然后将其赋值给变量 `dog`。

```js
const dog = new Dog("Gino");
```

当你在代码中写 `new Dog("Gino")` 时，新的对象被创建，并且构造函数会自动使用给定的参数调用。

然后你可以使用点符号访问这个属性，就像你在下面这行代码中看到的那样：

```js
console.log(dog.name);
```

这是完整的示例：

:::interactive_editor

```js
class Dog {
  constructor(name) {
    this.name = name;
  }

  bark() {
    console.log(`${this.name} says woof!`);
  }
}

const dog = new Dog("Gino");
console.log(dog.name);
```

:::

你也可以调用方法。在这个例子中，我们正在对 `dog` 实例调用 `bark()` 方法。

```js
dog.bark();
```

这是包含方法调用的完整示例：

:::interactive_editor

```js
class Dog {
  constructor(name) {
    this.name = name;
  } 
  bark() {
    console.log(`${this.name} says woof!`);
  }
}

const dog = new Dog("Gino");
console.log(dog.name);
dog.bark();
```

:::

如前所述，你也可以将类定义为类表达式。这是指类是匿名的并赋值给一个变量。

下面是之前示例作为一个类表达式的样子：

```js
const Dog = class {
  constructor(name) {
    this.name = name;
  }

  bark() {
    console.log(`${this.name} says woof!`);
  }
};
```

大多数时候，你会使用类声明，但理解类表达式也很有价值，因为它们可以提供更多灵活性，尤其是在动态编程情况下。

通过使用 `class` 并理解其基本 `syntax`，你可以创建可重用的 `code`，将其组织成逻辑单元，并促进 `code` 的模块性。
