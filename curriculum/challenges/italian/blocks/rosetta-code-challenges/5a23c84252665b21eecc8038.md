# --description--

Subleq è un esempio di One-Instruction Set Computer (OISC).

Prende il nome dalla sua unica istruzione, che è **SU**btract e **B**ranch se **L**ess o **EQ**ual a zero.

La tua attività è creare un interprete che emuli una macchina del genere.

La memoria della macchina consiste in un array di numeri interi con segno. Qualsiasi dimensione di parola ragionevole va bene, ma la memoria deve poter contenere numeri sia negativi sia positivi.

L’esecuzione inizia con il puntatore all’istruzione puntato alla prima parola, cioè all’indirizzo 0. Procede come segue:

<ol>
  <li>Indichiamo con A, B e C i valori contenuti nelle tre parole consecutive in memoria a partire dal puntatore all’istruzione.</li>
  <li>Avanza il puntatore all’istruzione di 3 parole per puntare all’indirizzo successivo a quello contenente C.</li>
  <li>Se A è -1, allora viene letto un carattere dall’input standard e il suo codice viene archiviato all’indirizzo indicato da B. C non viene usato.</li>
  <li>Se B è -1, allora il numero contenuto all’indirizzo indicato da A viene interpretato come codice di un carattere e il carattere corrispondente viene prodotto in output. Anche in questo caso C non viene usato.</li>
  <li>Altrimenti, sia A sia B sono trattati come indirizzi di locazioni di memoria. Il numero contenuto all’indirizzo A viene sottratto dal numero all’indirizzo B (e il risultato viene archiviato di nuovo all’indirizzo B). Se il risultato è zero o negativo, il valore C diventa il nuovo puntatore all’istruzione.</li>
  <li>Se il puntatore all’istruzione diventa negativo, l’esecuzione si interrompe.</li>
</ol>

Altri indirizzi negativi oltre a -1 possono essere trattati come equivalenti a -1 o generare un errore, a tua discrezione.

La tua soluzione dovrebbe accettare un programma da eseguire sulla macchina, separatamente dall’input fornito al programma stesso.

Questo programma dovrebbe essere in codice macchina subleq "grezzo" — numeri decimali separati da spazi, senza nomi simbolici o altre estensioni a livello di assembly, da caricare in memoria a partire dall’indirizzo 0. Mostra l’output della tua soluzione quando esegue questo programma "Hello, world!". (Nota che l’esempio assume ASCII o un suo superset, come uno qualsiasi dei set di caratteri Latin-N o Unicode. Puoi tradurlo in un altro set di caratteri se la tua implementazione è in un ambiente non compatibile con ASCII.)

<pre>15 17 -1 17 -1 -1 16 1 -1 16 3 -1 15 15 0 0 -1 72 101 108 108 111 44 32 119 111 114 108 100 33 10 0</pre>

Che corrisponde a qualcosa del genere in un ipotetico linguaggio assembler:

<pre>start:
    zero, message, -1
    message, -1, -1
    neg1, start+1, -1
    neg1, start+3, -1
    zero, zero, start
zero: 0
neg1: -1
message: "Hello, world!\n\0"
</pre>

function Subleq(mem) {
  var out = '';
  var instructionPointer = 0;
  do {
    var a = mem[instructionPointer];
    var b = mem[instructionPointer + 1];
    if (a === -1) {
    } else if (b === -1) {
      out += String.fromCharCode(mem[a]);
    } else {
      mem[b] -= mem[a];
      if (mem[b] < 1) {
        instructionPointer = mem[instructionPointer + 2];
        continue;
      }
    }
    instructionPointer += 3;
  } while (instructionPointer >= 0);

  return out;
}
