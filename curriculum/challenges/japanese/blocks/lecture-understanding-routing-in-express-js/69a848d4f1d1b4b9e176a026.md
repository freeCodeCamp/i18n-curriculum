# --description--

基本的なExpressアプリを作成したので、次はルーティングとその仕組みを詳しく見ていきましょう。

Expressでは、ルーティングとはウェブアプリケーションのエンドポイントを定義する方法です。ルートは、ユーザーが特定のURLを訪れたときにアプリがどのように応答するかを指示するルールのようなものと考えてください。

もう一度、シンプルなExpressアプリを見てみましょう。

```javascript
const express = require("express");
const app = express();
const port = 3000;

app.get("/", (req, res) => {
  res.send("Hello World!");
});

app.listen(port, () => {
  console.log(`Example app listening on port ${port}`);
});
```

この例では、アプリはルートURL（`/`）のルートを定義しています。クライアントがこのパスにGETリクエストを送ると、サーバーはテキスト`Hello World!`で応答します。アプリはその後、ポート`3000`で受信リクエストの待機を開始します。

しかし、他にもルートメソッドがあります。

Expressでは、異なる種類のHTTPリクエストを異なるルーティングメソッドで処理できます。最も一般的なものは次の通りです。

* `app.get()` – データを取得する
    
* `app.post()` – 新しいデータを作成する
    
* `app.put()` – 既存のデータを更新する
    
* `app.delete()` – データを削除する
    

それぞれのメソッドは、特定のルートで特定のHTTPリクエストに対してアプリケーションがどのように応答すべきかを定義します。

いくつか簡単な例を見てみましょう。

```javascript
app.post("/submit", (req, res) => {
  res.send("Form submitted!");
});

app.put("/update", (req, res) => {
  res.send("Item updated!");
});

app.delete("/delete", (req, res) => {
  res.send("Item deleted!");
});
```

どの場合も、Expressは特定のHTTPメソッドとURLパスのリクエストを待ち受け、それに対応するハンドラ関数を実行してクライアントにレスポンスを返します。

これらのルートは、Postmanや`curl`のようなツールを使ってテストできます。これらのツールはサーバーに対して異なる種類のHTTPリクエストを送ることができます。

これがExpressのルーティングです。つまり、アプリケーションが異なるリクエストにどう応答するかを定義することです。
