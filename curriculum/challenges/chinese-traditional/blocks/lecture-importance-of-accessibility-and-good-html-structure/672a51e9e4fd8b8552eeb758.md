# --description--

無障礙設計涉及創造每個人都能使用的產品和服務。在網頁開發的背景關係中，這是指製作每個人都能理解和互動的網站，包括有視覺、聽覺、動作和認知障礙的人。

某些可能影響使用者線上體驗的殘障範例包括：

- 失明。
- 低視力。
- 色盲。
- 失聰。
- 使用鍵盤、滑鼠或觸控螢幕時的困難。
- 注意力障礙
- 記憶體問題。
- 說話或理解口語的困難。
- 對閃爍燈光的敏感度。

這只是眾多可能影響全球使用者的條件中的幾個。

為了幫助你創建可存取的網站，萬維網聯盟（World Wide Web Consortium，簡稱 W3C）制定了一套國際標準，你可以遵循這些標準，使你的网站對身心障礙者更易存取且更容易使用。

這些標準被稱為「網頁內容存取指引」（WCAG）。

這些指導方針是以四個核心原則設計，稱為 **POUR**。

- `P` 代表可感知。使用者必須能夠感知你所呈現的資訊。舉例來說，你可以為影像提供替代文字，讓使用螢幕閱讀器存取你網站的使用者能夠理解它們。
- `O` 代表可操作。使用者必須能夠與使用者介面互動。例如，你可以確保所有功能也能透過鍵盤存取，而不僅僅是滑鼠。
- `U` 代表可理解性。使用者必須能夠理解這些資訊。 例如，你可以避免複雜的句子並盡可能使用簡單的語言。
- `R` 代表強固。必須有廣泛的瀏覽器和其他工具，包括輔助技術，能夠解讀內容。

使用語意化的 HTML 對於讓你的網站與不同瀏覽器和輔助技術相容非常有幫助。

如果你的內容沒有遵循這些核心原則中的任何一項，並非所有人都能使用你的網站。

若你想確認是否正確遵循這些指引，你可以存取 World Wide Web Consortium 的 Quick Reference。在那裡，你會找到完整的標準與技術列表。

無障礙性對於網頁開發至關重要。透過以包容性為核心進行開發，你可以確保每個人都能存取並參與你的內容，促進平等，並為全球所有人創造更佳的使用者體驗。
