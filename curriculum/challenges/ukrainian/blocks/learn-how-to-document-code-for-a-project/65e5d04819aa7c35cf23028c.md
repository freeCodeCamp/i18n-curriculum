# --description--

Слово `so` є сполучником, який використовується для показу причини та наслідку або для пояснення причини чогось. У технологіях і програмуванні `so` часто з’єднує проблему або ситуацію з рішенням чи результатом.

# --questions--

## --text--

Виберіть речення, де `so` використано правильно для з’єднання причини та наслідку в технічному контексті.

## --answers--

`The program crashed, so we must reboot the computer.`

---

`We wrote the code, so but it didn't compile.`

### --feedback--

Слово `so` тут використано неправильно. Після `But` не слід використовувати `so`. Має бути або `The code didn't compile, so we rewrote it`, або `We wrote the code, but it didn't compile.`

---

`The server is online, so therefore we can't connect.`

### --feedback--

Використання `so` і `therefore` разом є надмірним. Має бути або `The server is online, so we can connect`, або `The server is online; therefore, we can connect.`

---

`We need more data, so we can ignore the results.`

### --feedback--

Це речення логічно не з’єднує причину і наслідок. Краще використати `so`, наприклад, `We need more data, so we will perform more tests.`
