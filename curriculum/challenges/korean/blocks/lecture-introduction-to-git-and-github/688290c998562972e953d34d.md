# --description--

이전에 변경 사항을 커밋하고 원격 저장소에 푸시하는 방법을 배웠지만, 모두 기본 `main` 브랜치에서 작업했습니다. 작은 프로젝트에는 괜찮지만, 다른 오픈 소스 프로젝트에 기여하거나 프로젝트가 커질수록 브랜치는 필수입니다.

예를 들어, 저장소를 포크하고 `main` 브랜치에 커밋하면 원본 저장소에서 받아들이지 않은 변경 사항이 포크에 생길 수 있습니다. 이렇게 되면 원본 저장소에서 받아들인 변경 사항으로 브랜치를 업데이트하기가 매우 어려워집니다. 또는 자신의 프로젝트에서 작업할 때 브랜치를 사용하지 않으면 서로 다른 작업 간 전환이 훨씬 어려워집니다. 대신 어떤 변경이 어떤 기능이나 버그 수정과 관련 있는지 수동으로 추적해야 합니다.

그렇다면 브랜치란 정확히 무엇일까요? `main` 브랜치를 고속도로라고 생각해 보세요. 각 커밋은 고속도로에 진입하는 자동차입니다. 새로운 기능을 만들고 싶어 잠시 고속도로에서 빠져나가야 할 수도 있습니다.

브랜치는 기본 기록에서 벗어난 분기점으로, 새로운 기능이나 버그 수정을 자유롭게 시도할 수 있는 공간입니다. 변경 사항이 만족스럽다면 브랜치를 `main`에 병합해 고속도로에 다시 진입할 수 있습니다. 만족스럽지 않으면 브랜치를 삭제하고 `main`로 돌아가 변경 사항을 받아들이지 않고 작업할 수 있습니다.

새 브랜치를 만들기 전에 `git branch` 명령어로 현재 브랜치를 확인하세요. 출력은 다음과 같을 수 있습니다:

```sh
* main
```

지금은 기본 `main` 브랜치 하나만 있습니다. 별표(`*`)는 현재 "체크아웃"한 브랜치를 나타냅니다. 이 부분은 곧 더 자세히 배울 것입니다.

`git branch` 명령어와 새 브랜치 이름 인수로 새 브랜치를 만들 수 있습니다:

```sh
git branch feature
```

이 명령어는 `feature`라는 새 브랜치를 만듭니다. `git branch`으로 다시 브랜치를 확인해 보겠습니다:

```sh
  feature
* main
```

새 브랜치가 보이지만 여전히 `main` 브랜치에 있습니다. 새 브랜치를 "체크아웃"해 전환해야 합니다. 체크아웃하면 Git이 해당 시점의 파일 상태를 불러옵니다.

다음 명령어로 새 브랜치를 체크아웃하세요:

```sh
git checkout feature
```

또는

```sh
git switch feature
```

그리고 다시 `git branch`를 실행해 보세요:

```sh
* feature
  main
```

별표가 현재 체크아웃한 브랜치로 이동한 것을 확인할 수 있습니다.

새 브랜치를 만들 때 보통 바로 체크아웃하고 싶을 때가 많습니다. 두 명령어로 브랜치를 만들고 체크아웃하는 대신, 한 명령어로 만들고 전환할 수 있습니다:

```sh
git checkout -b feature
```

또는

```sh
git switch -c feature
```

이제 새 브랜치를 만들고 체크아웃했습니다. 이 브랜치는 `main` 브랜치의 복사본으로, 브랜치를 만든 시점의 `main`와 동일한 코드를 가집니다. 이제 `main` 브랜치에 영향을 주지 않고 이 브랜치에 코드를 추가할 수 있습니다. 기능을 만들기 위해 고속도로에서 빠져나온 셈입니다. `git status`를 확인하면 다음과 비슷할 것입니다:

```sh
On branch feature
nothing to commit, working tree clean
```

다음 명령어로 새 `feature.md` 파일을 만드세요:

```sh
echo "This is our new feature" > feature.md
```

상태는 이제 이렇게 보일 것입니다:

```sh
On branch feature
Untracked files:
  (use "git add <file>..." to include in what will be committed)
	feature.md

nothing added to commit but untracked files present (use "git add" to track)
```

다음으로 새 기능을 커밋할 차례입니다. 먼저 파일을 스테이징에 추가하세요:

```sh
git add feature.md
```

그다음 변경 사항을 커밋합니다:

```sh
git commit -m "my new feature"
```

마지막으로 `feature` 브랜치를 저장소에 푸시하세요:

```sh
git push -u origin feature
```

`-u` 플래그는 로컬 `feature` 브랜치를 원격 브랜치와 연결해 이후 푸시를 `git push`만으로 할 수 있게 합니다.
