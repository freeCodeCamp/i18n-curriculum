# --description--

Invece di riprodurre audio e video, a volte potresti voler catturare audio o video. L'API Media Capture and Streams, o MediaStream API, ti permette di farlo.

Per usare l'API, però, devi creare l'oggetto `MediaStream`. Potresti farlo con il costruttore, ma non sarebbe collegato all'hardware dell'utente. Invece, la proprietà `mediaDevices` dell'oggetto globale `navigator` ha un metodo `getUserMedia()` che puoi usare.

Questo metodo accetta un singolo oggetto `constraints` che definisce il tipo di media che vuoi ricevere. Questo oggetto ha una proprietà `audio` e una `video`, che rappresentano rispettivamente i flussi audio e video. Queste proprietà possono essere `false` se non vuoi ricevere quel tipo di flusso, `true` se lo vuoi, oppure oggetti che definiscono vincoli aggiuntivi.

Per esempio, puoi richiedere una risoluzione specifica per il video in output:

```js
window.navigator.mediaDevices.getUserMedia({
  audio: true,
  video: {
    width: {
      min: 1280,
      ideal: 1920,
      max: 3840
    },
    height: {
      min: 720,
      ideal: 1080,
      max: 2160
    }
  }
});
```

Questo oggetto di vincoli specifica risoluzioni minime e massime per il flusso video. La proprietà `ideal` indica la risoluzione che preferiresti avere – e il flusso fornirà la risoluzione più vicina al tuo ideale.

Una volta creato il tuo MediaStream (supponendo che l'utente approvi la richiesta automatica di consultare il suo hardware), puoi usare i dati del flusso come ti serve.

Nota che `getUserMedia()` restituisce una Promise, il che significa che avrai bisogno di una funzione di callback per consumare il flusso, oppure di usare la sintassi async/await. Imparerai di più sulle Promise e sulla programmazione asincrona nelle lezioni future.

Ecco un esempio base che renderizza il flusso della webcam dell'utente sulla pagina.

```js
const video = document.querySelector("video");
const stream =
  await window.navigator.mediaDevices.getUserMedia({ video: true });
video.srcObject = stream;
await video.play();
```

Vale la pena notare che questa API non offre accesso alla cattura dello schermo. Imparerai quell'API nella prossima lezione.
