# --description--

Налаштування кастомної автентифікації вимагає написання значної кількості ручної логіки. Потрібно обробляти читання заголовків запитів, витягувати куки, перевіряти підписи та керувати помилками.

Хоча створювати це самому — чудовий спосіб навчитися, робити це для кожного продуктивного застосунку займає багато часу і схильне до помилок у безпеці. Саме тому розробники використовують Passport.js.

Passport.js — це підпрограмне забезпечення автентифікації для Node.js. Воно виступає як організований фреймворк, що стандартизує спосіб автентифікації запитів. Воно обробляє низькорівневі деталі автентифікації, дозволяючи вам зосередитися на бізнес-логіці вашого застосунку.

Основна ідея Passport.js полягає в тому, що кожен застосунок потребує різних способів входу, але базовий потік автентифікації має залишатися однаковим. Passport досягає цієї гнучкості через модульні плагіни, які називаються **стратегіями**.

Замість того, щоб писати кастомний код для кожного методу автентифікації, ви встановлюєте конкретну стратегію, яка вам потрібна. Існує понад 500 стратегій, що охоплюють майже всі сучасні методи автентифікації. Деякі з них:

- **passport-local:** Обробляє традиційну автентифікацію за ім’ям користувача та паролем у вашій власній базі даних.

- **passport-jwt:** Захищає кінцеві точки API, витягуючи та перевіряючи JSON Web Tokens із вхідних запитів.

- **passport-google-oauth20:** Дозволяє користувачам миттєво входити, використовуючи свої існуючі облікові записи Google.

- **passport-facebook:** Вхід через облікові записи Facebook за допомогою OAuth 2.0.

- **passport-github2:** Дозволяє користувачам миттєво входити, використовуючи свої облікові записи GitHub.

Розбивши автентифікацію на окремі стратегії, Passport дозволяє вашому застосунку безперешкодно розвиватися. Якщо ви починаєте сьогодні з системи ім’я користувача/пароля і вирішуєте додати «Вхід через Google» наступного місяця, вам не потрібно переписувати весь код автентифікації. Просто встановіть стратегію Google і підключіть її до Passport.

Щоб використовувати Passport у застосунку Express, потрібно встановити основний пакет Passport разом із конкретною стратегією, яку ви плануєте використовувати. Для стандартної системи ім’я користувача та пароля виконайте таку команду у вашому терміналі:

```bash
npm install passport passport-local
```

Після встановлення потрібно підключити Passport до налаштувань вашого застосунку Express. Це має відбутися до того, як ви визначите свої маршрути:

```js
const express = require('express');
const passport = require('passport');
const LocalStrategy = require('passport-local').Strategy;

const app = express();

// initialize Passport middleware
app.use(passport.initialize());

// configure Passport to use the Local Strategy
passport.use(new LocalStrategy((username, password, done) => {
    // your database look-up and password verification logic goes here
}));
```

Passport плавно інтегрується у стандартну систему маршрутизації Express як підпрограмне забезпечення. Коли користувач відправляє свої облікові дані на маршрут входу, Passport перехоплює запит, передає дані налаштованій стратегії і оцінює результат.

```js
app.post('/login', passport.authenticate('local'), (req, res) => {
  // if this function runs, then authentication was successful
  res.send(`Welcome back, ${req.user.username}!`);
});
```

Якщо стратегія успішно перевіряє користувача, Passport автоматично виконує дві критичні задачі:

1. **Управління сесією:** Воно викликає допоміжні функції для збереження невеликої частини даних користувача всередині сесійної куки.

2. **Декорування запиту:** Воно приєднує автентифікований об’єкт користувача безпосередньо до вхідного запиту як `req.user`.

Як тільки `req.user` заповнений, будь-яке наступне підпрограмне забезпечення або обробник маршруту у вашому застосунку може миттєво побачити, хто увійшов, і приймати рішення про авторизацію на основі їхньої ідентичності.

Passport забезпечує послідовну, безпечну та готову до продуктивного використання основу для керування ідентичностями користувачів у ваших застосунках Node.js.
