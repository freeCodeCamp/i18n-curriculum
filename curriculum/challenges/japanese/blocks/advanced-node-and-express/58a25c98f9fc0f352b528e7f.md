# --description--

情報セキュリティのセクションに戻ると、プレーンテキストのパスワードを保存するのは*絶対に*許されないことを覚えているかもしれません。ここでは、この問題を解決するためにBCryptを実装する時です。

`bcrypt@~5.0.0` はすでに依存関係として追加されているので、サーバーでrequireしてください。ハッシュ化は主に2つの重要な箇所で扱う必要があります。新しいアカウントの登録・保存時と、ログイン時にパスワードが正しいか確認する時です。

現在の登録ルートでは、ユーザーのプレーンテキストのパスワードを次のようにデータベースに挿入しています：`password: req.body.password`。代わりにパスワードをハッシュ化するために、データベースの処理の前に次のコードを追加してください：`const hash = bcrypt.hashSync(req.body.password, 12);`。そして、データベース保存時の`req.body.password`を`password: hash`に置き換えてください。

認証戦略では、処理を完了する前にコード内で次のようにチェックしています：`if (password !== user.password) return done(null, false);`。前述の変更を加えた後、`user.password`はハッシュ値になります。既存のコードを変更する前に、パスワードが**等しくない場合**に認証失敗を返すようにしていることに注意してください。この点を踏まえ、入力されたパスワードとハッシュを正しく比較するために、コードを次のように変更してください：

```js
if (!bcrypt.compareSync(password, user.password)) { 
  return done(null, false);
}
```

これで、パスワードを保存する際に最も重要なセキュリティ機能の1つを実装できます。
