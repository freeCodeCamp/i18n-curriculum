# --interactive--

사용자가 웹사이트에서 폼을 작성할 때, 필요한 모든 정보를 올바른 형식으로 입력하는 것이 중요합니다. 입력과 같은 HTML 폼 컨트롤에는 잘못된 데이터를 검사할 수 있는 많은 내장 유효성 검사가 있습니다. 이는 사용자가 정보를 제출하고 서버가 처리하기 전에 이러한 실수를 고치도록 도와줍니다.

"클라이언트 측"이라는 용어는 사용자의 컴퓨터나 기기에서 일어나는 모든 일을 의미하며, 사용자가 직접 상호작용하는 웹사이트나 애플리케이션의 부분을 포함합니다. 여기에는 레이아웃, 디자인, 그리고 모든 인터랙티브 기능이 포함됩니다.

"서버 측"이라는 용어는 웹사이트나 애플리케이션을 호스팅하는 서버, 컴퓨터, 또는 시스템에서 일어나는 모든 일을 의미합니다. 여기에는 데이터 처리, 애플리케이션 실행, 그리고 사용자의 기기에서 오는 요청 처리 등이 포함됩니다.

클라이언트 측 유효성 검사가 중요하지만, 보안을 위해 서버 측 유효성 검사도 필요합니다. 악의적인 사용자는 클라이언트 측 검사를 우회할 수 있으므로, 견고한 서버 측 조치가 필수적입니다. 이 부분은 이후 모듈에서 더 배우게 됩니다. 지금은 클라이언트 측 폼 유효성 검사 예시를 살펴보겠습니다.

내장 폼 유효성 검사의 일반적인 예는 입력에 `required` 속성을 사용하는 것입니다. `required` 속성은 사용자가 폼의 해당 부분을 제출하기 전에 반드시 작성해야 함을 지정합니다. 다음은 이메일 입력에 `required` 속성을 사용하는 예시입니다.

이메일 주소를 입력하지 않고 `Submit Form` 버튼을 클릭하면, 필드를 작성하라는 메시지가 팝업으로 나타납니다.

:::interactive_editor

```html
<form action="">
  <label for="email">Email Address (Required field):</label>
  <input required type="email" name="email" id="email" />
  <button type="submit">Submit Form</button>
</form>
```

:::

각 브라우저는 이 경고 메시지를 표시하는 고유한 스타일을 가지고 있습니다.

이메일 입력을 사용하는 또 다른 장점은 이메일 입력이 올바른 형식의 이메일 주소인지 기본적인 유효성 검사를 한다는 점입니다. 예를 들어, 무작위 단어를 입력하고 제출 버튼을 클릭하면, 브라우저가 `@` 기호가 없다는 경고를 표시합니다.

이메일 필드에 `abc`를 입력하고 제출 버튼을 클릭하세요. 유효한 이메일 주소가 아니므로 메시지가 팝업으로 나타납니다.

:::interactive_editor

```html
<form action="">
  <label for="email">Email Address (Required field):</label>
  <input required type="email" name="email" id="email" />
  <button type="submit">Submit Form</button>
</form>
```

:::

브라우저가 표준 이메일 주소에 대해 기본적인 유효성 검사만 수행한다는 점을 기억하세요. 추가 유효성 검사 레이어를 추가하는 것은 여러분의 몫이며, 이는 이후 모듈에서 배우게 됩니다.

이메일 입력에 대한 다른 유효성 검사 방법으로는 `minlength`와 `maxlength` 속성을 사용하는 것입니다. 다음은 추가 유효성 검사를 사용하는 예시입니다.

필드에 `b@m`를 입력하고 제출 버튼을 클릭하세요. 최소 요구 길이를 충족하지 않아 메시지가 팝업으로 나타납니다.

:::interactive_editor

```html
<form action="">
  <label for="email">Email Address (Required field):</label>
  <input
    required
    type="email"
    name="email"
    id="email"
    minlength="4"
    maxlength="64"
  />
  <button type="submit">Submit Form</button>
</form>
```

:::

`minlength`와 `maxlength` 속성은 이메일 입력의 최소 및 최대 문자 길이를 설정하는 데 사용됩니다. 최소 길이를 포함하지 않거나 최대 문자 길이를 초과하면 브라우저가 경고 메시지를 표시합니다.
