# --description--

Katika funzo la awali, ulijifunza jinsi ya kufanya kazi na type aliases na ukaona mifano ya jinsi ya kuitia jina aina za vitu.

Kama unavyokumbuka, njia moja ya kuitia jina aina ya kitu ni kutumia neno kuu la `type` kama hii:

```ts
type ID = number | string;

type User = {
  id: ID;
  name: string;
};
```

Njia nyingine ya kuitia jina aina ya kitu ni kutumia tamko la `interface` kama hii:

```ts
type ID = number | string;

interface User {
  id: ID;
  name: string;
}
```

`type` na `interface` ni sawa lakini tofauti kuu ni kwamba interfaces hutumia neno kuu la `extends` kujenga juu ya interfaces nyingine, wakati `type` aliases hutumia aina za intersection (`&`) kuunganisha aina.

Tuchunguze baadhi ya mifano:

```ts
interface User {
  id: number | string;
  name: string;
}

interface Admin extends User {
  role: string;
}

const adminUser: Admin = {
  id: 101,
  name: "Alice",
  role: "superadmin"
};
```

Katika mfano huu, `User` huanza na `id` na `name`. Interface ya `Admin` inaongeza `User`, ikimaanisha inajumuisha `id` na `name` na kuongeza sifa mpya inayoitwa `role`.

Ikiwa ungependa kuongeza `type`, utatumia aina ya intersection (`&`) kama hii:

```ts
type User = {
  id: number | string;
  name: string;
};

type Admin = User & {
  role: string;
};

const adminUser: Admin = {
  id: 101,
  name: "Alice",
  role: "superadmin"
};
```

`Admin` huunganisha aina ya `User` na aina mpya ya kitu inayojumuisha `role`. Matokeo ni umbo sawa na mfano wa interface ambao ni kitu chenye `id`, `name`, na `role`.

Zote `type` na `interface` zinaweza kuelezea umbo la kitu. Ikiwa huna uhakika ni ipi utumie, kumbuka kwamba `interface` imetengenezwa mahsusi kwa maumbo ya vitu, wakati `type` ni rahisi zaidi na inaweza pia kuwakilisha aina za muungano, primitives, na mchanganyiko tata zaidi wa aina.
