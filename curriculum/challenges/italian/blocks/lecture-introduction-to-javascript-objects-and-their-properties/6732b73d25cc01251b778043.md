# --interactive--

In JavaScript, capire la differenza tra tipi di dati primitivi e non primitivi è importante per scrivere codice efficiente e senza bug.

Queste due categorie di tipi di dati si comportano in modo diverso in termini di come vengono memorizzate in memoria e come vengono manipolate nei tuoi programmi.

I tipi di dati primitivi sono la forma più semplice di dati in JavaScript. Includono number, bigint, string, boolean, `null`, `undefined` e symbol. Questi tipi si chiamano "primitivi" perché rappresentano valori singoli e non sono oggetti.

Quando lavori con tipi di dati primitivi, gestisci direttamente i loro valori. Per esempio, quando crei una variabile con un valore primitivo, quel valore viene memorizzato direttamente nella variabile.

I valori primitivi sono immutabili, il che significa che una volta creati, il loro valore non può essere cambiato. Tuttavia, puoi riassegnare un nuovo valore alla variabile. Ecco un esempio di lavoro con tipi di dati primitivi:

:::interactive_editor

```js
let num1 = 5;
let num2 = num1;
num1 = 10;

console.log(num2); // 5
```

:::

In questo esempio, stiamo assegnando un valore primitivo (`5`) da `num1` a `num2`. Questo crea una copia indipendente del valore. Di conseguenza, qualsiasi modifica fatta alla variabile originale (`num1`) non influisce sulla copia (`num2`).

I tipi di dati non primitivi, invece, sono più complessi. In JavaScript, questi sono oggetti, che includono oggetti regolari, array e funzioni. A differenza dei primitivi, i tipi non primitivi possono contenere valori multipli come proprietà o elementi.

Quando crei una variabile con un valore non primitivo, ciò che viene memorizzato nella variabile è in realtà un riferimento alla posizione in memoria dove l’oggetto è memorizzato, non l’oggetto stesso. Questo porta a differenze importanti nel comportamento. Ecco un esempio con tipi non primitivi:

:::interactive_editor

```js
const originalPerson = { name: "John", age: 30 };
const copiedPerson = originalPerson;

originalPerson.age = 31;

console.log(copiedPerson.age); // 31
```

:::

In questo esempio abbiamo un oggetto chiamato `originalPerson` con due proprietà `name` e `age`. Poi assegniamo l’oggetto `originalPerson` a una variabile chiamata `copiedPerson`.

Successivamente aggiorniamo il valore `age` per l’oggetto `originalPerson`. Quando stampiamo la proprietà `age` dell’oggetto `copiedPerson`, mostra il valore aggiornato.

Ma perché succede questo? Succede perché sia `originalPerson` che `copiedPerson` fanno riferimento allo stesso oggetto in memoria.

In JavaScript, quando assegni un oggetto a un’altra variabile, stai copiando il riferimento all’oggetto, non l’oggetto stesso. Questo è noto come copia superficiale per riferimento. Di conseguenza, qualsiasi modifica fatta all’oggetto tramite un riferimento si riflette in tutti i riferimenti a quell’oggetto.

Man mano che continui a lavorare con JavaScript, incontrerai molte situazioni in cui capire la differenza tra tipi primitivi e non primitivi è importante. È un concetto fondamentale che sta alla base di molti aspetti del linguaggio ed è la chiave per scrivere codice efficiente e corretto.
