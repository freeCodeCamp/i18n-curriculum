# --description--

Шлях — це рядок, який вказує розташування файлу або каталогу у файловій системі. У веброзробці шляхи дозволяють розробникам направляти на ресурси, такі як зображення, таблиці стилів, скрипти та інші вебсторінки. Існують абсолютні та відносні шляхи — обидва є важливими при вказуванні розташування файлів у файловій системі. Розглянемо обидва, щоб ви могли вирішити, який із них і коли використовувати.

Абсолютний шлях — це повне посилання на ресурс. Він починається з кореневого каталогу, включає всі інші каталоги і нарешті ім’я файлу та розширення. «Кореневий каталог» — це каталог найвищого рівня або папка у ієрархії.

Якщо ви направляєте на ресурс на вашому локальному комп’ютері, використовуйте абсолютний шлях, який містить повне розташування файлу в каталозі. Ось як направити на файл `about.html` за допомогою абсолютного шляху:

```html
<p>
  Read more on the
  <a
    href="/Users/user/Desktop/fCC/script-code/absolute-vs-relative-paths/pages/about.html"
    >About Page</a
    >
</p>
```

Він виглядає так, тому що ми починаємо з кореня і переходимо у папку `Users`, потім у папку `user`, потім у папку `Desktop`, потім у папку `fCC`, потім у папку `script-code`, потім у папку `absolute-vs-relative-paths`, потім у папку `pages`, щоб нарешті дістатися до файлу `about.html`.

Абсолютний URL — це повна адреса, яка використовується для доступу до ресурсу. Він включає протокол — який може бути `http`, `https` або `file` — і доменне ім’я, якщо ресурс знаходиться в інтернеті. Ось приклад абсолютного URL, що направляє на логотип freeCodeCamp:

```html
<a href="https://design-style-guide.freecodecamp.org/img/fcc_secondary_small.svg">
  View fCC Logo
</a>
```

У цьому прикладі протокол — `https`, доменне ім’я — `design-style-guide.freecodecamp.org`, а ім’я файлу — `fcc_secondary_small.svg`.

Ось як абсолютний URL виглядає у рядку адреси браузера:

```sh
file:///Users/user/Desktop/fCC/script-code/absolute-vs-relative-paths/pages/about.html
```

URL включає протокол `file://`. Також він містить шлях, який виглядає так: `/Users/user/Desktop/fCC/script-code/absolute-vs-relative-paths/pages/`, і представляє послідовність папок, що ведуть до файлу. І нарешті, він також містить `about.html`, що є ім’ям файлу та розширенням.

Абсолютний шлях показує повне розташування файлу у файловій системі і зазвичай використовується для ресурсів на локальному комп’ютері. Абсолютний URL включає інформацію для доступу — таку як протокол і, для вебресурсів, доменне ім’я — що вказує браузеру, як і де отримати ресурс.

Тепер розглянемо відносний шлях. Відносний шлях вказує розташування файлу відносно каталогу поточного файлу. Він не містить протоколу чи доменного імені, що робить його коротшим і гнучкішим для внутрішніх посилань у межах одного сайту. Ось приклад направлення на сторінку `about.html` зі сторінки `contact.html`, обидві з яких знаходяться в одній папці:

```html
<p>
  Read more on the
  <a href="about.html">About Page</a>
</p>
```

Уявіть, що ви на сторінці `contact.html`, і оскільки сторінка `about.html` знаходиться в тому самому місці, ви просто вказуєте ім’я файлу. Це приклад використання відносного шляху до файлу.

Отже, який шлях використовувати і коли: абсолютний шлях, абсолютний URL чи відносний шлях? Ось правила, яких слід дотримуватися:

- Використовуйте абсолютні шляхи, коли хочете посилатися на ресурс із фіксованого розташування, наприклад, з кореня вашого сайту або відомого каталогу на локальному комп’ютері.

- Використовуйте абсолютний URL, коли направляєте на ресурс, розміщений на зовнішньому вебсайті.

- Використовуйте відносні шляхи, коли направляєте на ресурси в межах одного сайту.

- Використовуйте відносні шляхи, якщо хочете зберегти код чистішим і легшим для підтримки під час розробки.

- Використовуйте відносні шляхи під час локального тестування, щоб переконатися, що посилання працюють без підключення до інтернету.
