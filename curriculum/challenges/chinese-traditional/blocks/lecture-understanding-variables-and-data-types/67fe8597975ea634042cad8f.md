# --description--

在 Python 中，變數就像一個標籤盒子，用來儲存和取用不同型別的資料。要在 Python 中宣告變數，你需要使用指派（`=`）運算子將值指定給識別字。你不需要像 JavaScript 中使用 `let` 或 `const`，或 C# 中使用 `char` 這類特殊關鍵字。

在 Python 中，你只要在左邊寫變數名稱，接著是指定運算子，然後在右邊寫你想要指定給變數的值。以下是如何宣告 `name` 和 `age` 變數的範例：

```python
name = 'John Doe'
age = 25
```

在上述範例中，變數 `name` 儲存的值是 `'John Doe'`。這個值是一個字串，字串是一系列用來表述文字的字元。字串以單引號或雙引號書寫，例如 `'Hello'` 或 `"Hello"`。在未來的課程中，你將學習更多關於在 Python 中操作字串的方法。

在 Python 中命名變數時，有一些重要規則你應該記住：

- 變數名稱只能以字母或底線（`_`）開頭，不能以數字開頭。
- 變數名稱只能包含英數字元（`a-z`、`A-Z`、`0-9`）和底線（`_`）。
- 變數名稱區分大小寫 — `age`、`Age` 和 `AGE` 都被視為獨立的名稱。
- 變數名稱不能是 Python 的保留關鍵字之一，例如 `if`、`class` 或 `def`。

如果你違反任何這些規則，你的 Python 程式將會丟出 `SyntaxError`：

```bash
    5variable_name = 5
     ^
SyntaxError: invalid syntax
```

現在讓我們來看看 Python 中變數的一些常見命名慣例。

首先，變數名稱應該全部小寫，並且以底線分隔不同的字。這稱為 snake case：

```python
my_variable_name = 'freeCodeCamp'
```

接下來，你應該為變數使用描述性名稱。例如，如果你想將使用者的年齡存檔為變數，`user_age` 比 `age` 或像 `ua` 這樣的縮寫更好：

```python
user_age = 30
```

這樣，你就能在大型程式碼庫中輕鬆地向其他團隊成員（或未來的自己）傳達變數的用途。

另一個慣例是避免使用單字母的變數名稱。這在 Python 中非常常見，但應該避免，因為單字母的變數名稱不傳達任何目的或意義：

```python
x = 56 # What do you mean by x?
```

如果你在迴圈或類似情況中，這會有所不同，因為像 `i`、`j`、`k` 等變數名稱是常見且可接受的。

此外，上述範例中的井字號符號（`#`）及其後的文字稱為註解。你可能已經熟悉註解，因此我們快速瀏覽並說明它們的運作方式。

在 Python 中，註解以井字號符號（`#`）開始，語言會忽略該行中 `#` 符號後的所有內容：

```python
# This is a single-line comment
```

多行註解可以透過連續的單行註解來創建：

```python
# This is a
# multi-line
# comment
```

你可以使用註解來說明你的程式碼、為自己留下提醒，或釐清某行程式碼存在的原因。當你在學習或團隊合作時，註解特別有幫助。

但是，你不應該使用註解來解釋你的變數名稱的意義。相反地，你為變數選擇的名稱應該具有描述性並傳達它們的用途，且遵循先前提到的其他命名規則以避免語法錯誤。
