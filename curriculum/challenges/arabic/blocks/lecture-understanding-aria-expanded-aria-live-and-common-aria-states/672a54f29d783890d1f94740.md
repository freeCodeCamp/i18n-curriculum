# --interactive--

عناصر التحكم الدلالية في النماذج مثل `input` و`select` و`textarea` و`button` و`fieldset` لها حالات مدمجة تُنقل إلى تقنيات المساعدة.

على سبيل المثال، يمكنك استخدام الخاصية `disabled` لتعطيل زر أو الخاصية `checked` للإشارة إلى أن مربع اختيار محدد.

لكن إذا كنت تنشئ عنصر تحكم مخصص، تحتاج إلى استخدام خصائص ARIA لنقل حالة التحكم إلى تقنيات المساعدة.

في هذا الدرس، سنناقش بعض حالات ARIA الشائعة التي يمكنك استخدامها على عناصر التحكم المخصصة.

أول حالة ARIA سنناقشها هي `aria-selected`. تُستخدم هذه الحالة للإشارة إلى أن العنصر محدد. يمكنك استخدام هذه الحالة على عناصر تحكم مخصصة مثل واجهة التبويبات، صندوق القائمة، أو الشبكة.

إليك مثالًا على كيفية استخدام `aria-selected` على عنصر تحكم تبويب مخصص:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<div role="tablist">
  <button role="tab" aria-selected="true">Tab 1</button>
  <button role="tab" aria-selected="false">Tab 2</button>
  <button role="tab" aria-selected="false">Tab 3</button>
</div>
<script src="index.js"></script>
```

```css
[role="tablist"] {
  display: flex;
  border-bottom: 2px solid #ddd;
  gap: 0.25rem;
  font-family: system-ui, sans-serif;
}

[role="tab"] {
  appearance: none;
  border: none;
  background: none;
  padding: 0.5rem 1rem;
  cursor: pointer;
  font-size: 1rem;
  color: #444;
  border-radius: 4px 4px 0 0;
  transition: background-color 0.2s, color 0.2s;
}

[role="tab"]:hover {
  background-color: #f3f3f3;
}

[role="tab"][aria-selected="true"] {
  background-color: #fff;
  color: #0078d4;
  border: 2px solid #0078d4;
  border-bottom: 2px solid #fff; 
  font-weight: 600;
  position: relative;
  z-index: 1;
}

[role="tab"]:focus {
  outline: 2px solid #0078d4;
  outline-offset: 2px;
}

```

```js
document.addEventListener("click", (event) => {
  const clickedTab = event.target.closest('[role="tab"]');
  if (!clickedTab) return;

  const tablist = clickedTab.closest('[role="tablist"]');
  const tabs = tablist.querySelectorAll('[role="tab"]');

  tabs.forEach((tab) => {
    const isSelected = tab === clickedTab;
    tab.setAttribute("aria-selected", isSelected);
    tab.tabIndex = isSelected ? 0 : -1;
  });
});

document.addEventListener("keydown", (event) => {
  const activeTab = document.activeElement;
  if (activeTab.getAttribute("role") !== "tab") return;

  const tablist = activeTab.closest('[role="tablist"]');
  const tabs = Array.from(tablist.querySelectorAll('[role="tab"]'));
  const index = tabs.indexOf(activeTab);

  let newIndex = index;
  if (event.key === "ArrowRight") newIndex = (index + 1) % tabs.length;
  if (event.key === "ArrowLeft") newIndex = (index - 1 + tabs.length) % tabs.length;

  if (newIndex !== index) {
    tabs[newIndex].focus();
    tabs[newIndex].click();
  }
});

```

:::

تُستخدم التبويبات لعرض عدة لوحات محتوى في مساحة محدودة. تُستخدم حالة `aria-selected` للإشارة إلى التبويب المحدد حاليًا.

عندما يختار المستخدم تبويبًا، تُضبط حالة `aria-selected` للتبويب المحدد على `true`، وحالة `aria-selected` للتبويبات الأخرى على `false`.

حالة ARIA شائعة أخرى هي `aria-disabled`. تُستخدم هذه الحالة للإشارة إلى أن العنصر معطل فقط للأشخاص الذين يستخدمون تقنيات المساعدة، مثل قارئ الشاشة. من المهم ملاحظة أن `aria-disabled` لا يعطل العنصر فعليًا. يعود الأمر إليك، كمطور، لجعله يبدو ويتصرف كعنصر معطل. تُستخدم هذه الخاصية أيضًا غالبًا على عناصر HTML الأصلية بدلًا من الخاصية `disabled`. اختيار أيهما يعتمد على سياق استخدام الزر.

إليك مثالًا على كيفية استخدام `aria-disabled` على زر تحرير مخصص:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<div role="button" tabindex="-1" aria-disabled="true">Edit</div>
```

```css
[role="button"] {
  display: inline-block;
  background-color: #0078d4;
  color: #fff;
  padding: 0.5rem 1rem;
  border-radius: 6px;
  font-family: system-ui, sans-serif;
  font-size: 1rem;
  text-align: center;
  cursor: pointer;
  user-select: none;
  transition: background-color 0.2s, transform 0.1s, opacity 0.2s;
}

[role="button"]:not([aria-disabled="true"]):hover {
  background-color: #005fa3;
}

[role="button"]:not([aria-disabled="true"]):focus {
  outline: 2px solid #005fa3;
  outline-offset: 2px;
}

[role="button"]:not([aria-disabled="true"]):active {
  transform: scale(0.97);
}

[role="button"][aria-disabled="true"] {
  opacity: 0.5;
  pointer-events: none;
  cursor: not-allowed;
  background-color: #b0b0b0;
  color: #f2f2f2;
}
```

:::

تُستخدم الخاصية `aria-disabled` لإخبار مستخدمي قارئ الشاشة أن زر التحرير معطل ولا يمكن التفاعل معه. مرة أخرى، لا يعطل الزر فعليًا. عند استخدام `aria-disabled`، ستحتاج إلى تطبيق التنسيق وJavaScript لجعل التحكم يبدو ويتصرف كزر معطل.

في معظم الحالات، ستستخدم على الأرجح عنصر الزر الأصلي، لكن هناك حالات قد تحتاج فيها إلى استخدام عنصر تحكم مخصص. لذلك، من الضروري معرفة كيفية نقل حالة التحكم إلى تقنيات المساعدة.

الحالة التالية التي سنناقشها هي `aria-haspopup`. تُستخدم هذه الحالة للإشارة إلى أن عنصرًا تفاعليًا سيُفعّل عنصر منبثق عند التنشيط. يمكنك استخدام الخاصية `aria-haspopup` فقط عندما يكون للعنصر المنبثق أحد الأدوار التالية: `menu`، `listbox`، `tree`، `grid`، أو `dialog`. يجب أن تكون قيمة `aria-haspopup` إما أحد هذه الأدوار أو `true`، والتي تعني افتراضيًا دور `menu`.

إليك مثالًا لقائمة محرر ملفات تستخدم `aria-haspopup`:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<button id="menubutton" aria-haspopup="menu" aria-controls="filemenu" aria-expanded="false">File</button>
<ul id="filemenu" role="menu" aria-labelledby="menubutton" hidden>
  <li role="menuitem" tabindex="-1">Open</li>
  <li role="menuitem" tabindex="-1">New</li>
  <li role="menuitem" tabindex="-1">Save</li>
  <li role="menuitem" tabindex="-1">Delete</li>
</ul>
```

```css
#menubutton {

  border-radius: 4px;
  cursor: pointer;
  font-size: 14px;
  position: relative;
}

#menubutton:hover,
#menubutton:focus {
  background-color: #005ea2;
  outline: none;
}

#filemenu {
  list-style: none;
  padding: 4px 0;
  margin: 4px 0 0;
  border: 1px solid #ccc;
  border-radius: 4px;

  z-index: 1000;
  box-shadow: 0 4px 6px rgba(0,0,0,0.1);
}

#filemenu[hidden] {
  display: none;
}

#filemenu [role="menuitem"] {
  display: block;

}
```

تُستخدم حالة `aria-haspopup` للإشارة إلى أن زر قائمة `File` سيفتح قائمة منبثقة عند تفعيله. قد يسمع مستخدمو قارئ الشاشة هذه المعلومة الإضافية عند التنقل إلى الزر.

ستحتاج إلى استخدام JavaScript لإظهار وإخفاء القائمة المنبثقة، وتنفيذ دعم لوحة المفاتيح المناسب للتفاعل مع القائمة. أيضًا، لاحظ أن دور ARIA `menu` يشير إلى نوع محدد جدًا من القوائم. عادةً ما يشير إلى قائمة إجراءات يمكن للمستخدم تنفيذها، مشابهة لقائمة في تطبيق سطح مكتب. لا يشمل الاستخدامات الأكثر شيوعًا لما نطلق عليه عادةً "القوائم"، مثل قوائم التنقل. في الواقع، معظم "القوائم" التي تنشئها على الويب لن تكون قوائم ARIA ولن تستخدم معها `aria-haspopup`.

الحالة التالية التي سنناقشها هي `aria-required`. تُستخدم الخاصية `aria-required` للإشارة إلى أن الحقل يجب تعبئته قبل إرسال النموذج.

إليك مثالًا على العمل مع الخاصية `aria-required` لعنصر تحكم نموذج مخصص.

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<div id="name-label">Full Name*</div>
<div role="textbox" contenteditable aria-labelledby="name-label" aria-required="true" id="name"></div>
```

```css
#name-label {

  font-size: 0.95rem;
  font-weight: 600;
  margin-bottom: 0.25rem;
  color: #333;
}

[role="textbox"] {
  display: block;
  width: 100%;
  min-height: 2rem;
  padding: 0.5rem 0.75rem;
  border: 1.5px solid #ccc;
  border-radius: 4px;
  font-family: system-ui, sans-serif;
  font-size: 1rem;
  color: #222;
  background-color: #fff;
  line-height: 1.4;
  transition: border-color 0.2s, box-shadow 0.2s;
}

[role="textbox"]:hover {
  border-color: #999;
}

[role="textbox"]:focus {
  outline: none;
  border-color: #0078d4;
  box-shadow: 0 0 0 3px rgba(0, 120, 212, 0.25);
}

[role="textbox"]:empty::before {
  content: attr(data-placeholder);
  color: #aaa;
  pointer-events: none;
}

[role="textbox"][aria-required="true"] {
  border-left: 3px solid #e81123;
  padding-left: calc(0.75rem - 3px);
}
```

:::

نحتاج إلى استخدام الخاصية `contenteditable` حتى يتمكن المستخدمون من كتابة مُدخلاتهم. نستخدم أيضًا الخاصية `aria-required` مضبوطة على `true` للإشارة إلى أن عنصر التحكم المخصص هذا مطلوب.

لجعل عنصر التحكم يبدو كعنصر تحكم نموذج عادي، ستحتاج إلى إضافة CSS. كما ستحتاج إلى إضافة JavaScript لمنع إرسال النموذج بدون محتوى.

إذا كان المسمى يحتوي بالفعل على كلمة `required`، فيجب عليك حذف الخاصية `aria-required`. هذا يضمن أن قارئات الشاشة تعلن كلمة "مطلوب" مرة واحدة فقط.

في معظم الحالات، ستستخدم على الأرجح عناصر `label` و`form` الأصلية مع الخاصية `required`. لكن إذا كنت بحاجة إلى إنشاء عنصر تحكم نموذج مخصص، فمن المهم إضافة الخاصية `aria-required` عند الضرورة.

بالإضافة إلى ذلك، يمكن استخدام خاصية `aria-required` على مُدخلات النماذج الأصلية، مثل عناصر `input` و`textarea` و`select`. غالبًا ما يُفضل استخدامها بدلاً من الخاصية الأصلية required، لأن الخاصية required قد تثير مشكلات في سهولة الاستخدام وإمكانية الوصول، خاصة مع معالجة الأخطاء الافتراضية التي يوفرها المتصفح. في النهاية، ستحتاج إلى الاختبار لتحديد أي خاصية هي الأنسب لحالتك.

آخر حالة ARIA سنناقشها هي `aria-checked`. تُستخدم هذه الخاصية للإشارة إلى ما إذا كان العنصر في حالة محددة. تُستخدم عادةً عند إنشاء مربعات اختيار مخصصة، أزرار اختيار، مفاتيح، وصناديق قائمة.

إليك مثالًا على كيفية استخدام `aria-checked` على عنصر تحكم مربع اختيار مخصص:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<div role="checkbox" aria-checked="true" tabindex="0">Checkbox</div>
```

```css
[role="checkbox"] {

  align-items: center;

  font-family: system-ui, sans-serif;
  font-size: 1rem;
  cursor: pointer;
  user-select: none;
  color: #222;
}

[role="checkbox"]::before {
  content: "";
  display: inline-block;
  width: 1rem;
  height: 1rem;
  border: 2px solid #666;
  border-radius: 4px;
  background-color: #fff;
  transition: all 0.2s ease;
  box-sizing: border-box;
}

[role="checkbox"]:hover::before {
  border-color: #0078d4;
}

[role="checkbox"]:focus::before {
  outline: 2px solid #0078d4;
  outline-offset: 2px;
}

[role="checkbox"][aria-checked="true"]::before {
  background-color: #0078d4;
  border-color: #0078d4;
  background-image: url("data:image/svg+xml;utf8,<svg xmlns='http://www.w3.org/2000/svg' width='14' height='14' viewBox='0 0 14 14'><path fill='white' d='M5.2 10.4L2 7.2l1.1-1.1 2.1 2.1L10.9 2.5 12 3.6z'/></svg>");
  background-repeat: no-repeat;
  background-position: center;
}

[role="checkbox"][aria-disabled="true"] {
  opacity: 0.5;
  pointer-events: none;
  cursor: not-allowed;
}

```

:::

عناصر مربع الاختيار الأصلية لها حالة `checked` مدمجة تُنقل إلى تقنيات المساعدة. لكن إذا كنت تنشئ عنصر تحكم مربع اختيار مخصص، ستحتاج إلى استخدام الخاصية `aria-checked` للإشارة إلى حالته.

عندما يتفاعل المستخدم مع عنصر التحكم مربع الاختيار المخصص، ستحتاج إلى استخدام حالة `aria-checked` لتعكس الحالة الجديدة لمربع الاختيار. عندما يكون مربع الاختيار محددًا، تُضبط الخاصية `aria-checked` على `true`. وعندما لا يكون محددًا، تُضبط على `false`.

عادةً ما تتمتع العناصر الأصلية بدعم أفضل وميزات إمكانية وصول مدمجة.

ومع ذلك، إذا اضطررت إلى إنشاء عناصر تحكم مخصصة، فإن استخدام خصائص ARIA ضروري لنقل حالة هذه العناصر إلى تقنيات المساعدة بفعالية.

كما هو الحال دائمًا، اختبر عملك للتأكد من تطبيق خصائص ARIA بشكل صحيح وأن عنصر التحكم المخصص يعمل بطريقة متاحة وسهلة الاستخدام.
