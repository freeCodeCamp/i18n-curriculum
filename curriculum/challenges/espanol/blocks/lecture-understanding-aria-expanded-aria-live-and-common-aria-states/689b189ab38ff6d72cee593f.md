# --interactive--

Anteriormente aprendiste cómo el marcado semántico y los atributos ARIA hacen que el contenido sea accesible. En aplicaciones reales, el contenido rara vez es estático, y las páginas a menudo se actualizan dinámicamente mediante JavaScript. Cuando esto sucede, es importante asegurarse de que estos cambios también se reflejen en el HTML. Esto permite que los lectores de pantalla transmitan con precisión el estado actualizado a los usuarios. Sin esto, las personas que dependen de herramientas de asistencia podrían no saber nunca que algo cambió, o podrían recibir información desactualizada o engañosa.

Veamos un escenario común donde JavaScript cambia el estado de un elemento, y cómo puedes mantener tu HTML accesible actualizando los atributos ARIA.

Supongamos que tienes un botón que abre y cierra un menú desplegable:

:::interactive_editor

```html
<button id="menuButton" aria-expanded="false" aria-controls="menuList">
  Menu
</button>

<ul id="menuList" hidden>
  <li>List item 1</li>
  <li>List item 2</li>
</ul>
<script src="index.js"></script>
```

```js
const button = document.getElementById("menuButton");
const menu = document.getElementById("menuList");

button.addEventListener("click", () => {
  const expanded = button.getAttribute("aria-expanded") === "true";
  button.setAttribute("aria-expanded", String(!expanded));
  menu.hidden = expanded;
});
```

:::

El HTML configura un botón que controla un menú desplegable. El botón usa `aria-expanded` para indicar si el menú está abierto, y `aria-controls` para vincularlo al menú haciendo referencia al `id` del menú.

JavaScript escucha los clics en el botón del menú. Cuando se hace clic, cambia el valor de `aria-expanded` y actualiza la propiedad `hidden` en el menú. Esto mantiene la visibilidad del menú y la información de accesibilidad sincronizadas, para que tanto los usuarios con visión como los que usan lectores de pantalla sepan si el menú está abierto o cerrado.

Aprenderás más sobre esos atributos ARIA en las próximas lecciones. Por ahora, solo recuerda que cuando tu JavaScript cambia el estado de un elemento, también debes actualizar los atributos ARIA relevantes para que las tecnologías de asistencia reflejen con precisión lo que realmente está en la pantalla.
