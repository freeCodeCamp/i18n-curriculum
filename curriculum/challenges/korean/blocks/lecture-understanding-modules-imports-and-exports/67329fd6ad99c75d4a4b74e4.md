# --description--

JavaScript에서 모듈은 관련된 함수, 클래스 또는 변수를 캡슐화한 독립적인 코드 단위입니다.

모듈을 애플리케이션의 구성 요소, 즉 책의 한 챕터처럼 생각하세요. 각 모듈은 특정 기능에 집중하여 코드를 더 체계적이고 유지 관리하기 쉬우며 재사용 가능하게 만듭니다. 모듈은 이름 충돌을 방지하고 애플리케이션을 별도의 상호 연결된 조각으로 구조화할 수 있게 도와줍니다.

JavaScript에서 모듈 개념은 시간이 지나면서 발전했지만, 가장 널리 사용되고 지원되는 방식은 ES6(ECMAScript 2015) 모듈 시스템입니다. 이 시스템은 다양한 JavaScript 환경에서 모듈을 정의하고 사용하는 표준화된 방법을 제공합니다.

모듈을 만들려면 별도의 파일에 JavaScript 코드를 작성하세요. 애플리케이션의 다른 부분에서 사용할 변수, 함수 또는 클래스를 명시적으로 내보내야 합니다. 이는 `export` 키워드를 사용해 할 수 있습니다.

예를 들어, `math.js`라는 파일에 몇 가지 수학 함수가 있다고 가정해 봅시다:

```js
export function add(a, b) {
    return a + b;
}

export function subtract(a, b) {
    return a - b;
}

const PI = 3.14159;
export { PI };
```

이 예제에서는 두 개의 함수(`add`, `subtract`)와 상수(`PI`)를 내보내고 있습니다. 하나의 모듈에서 필요한 만큼 여러 항목을 내보낼 수 있습니다.

이 내보낸 항목들을 애플리케이션의 다른 부분에서 사용하려면 가져와야 합니다. 이는 `import` 키워드를 사용해 합니다.

예를 들어, `app.js`라는 파일에서 이 수학 함수를 사용하고 싶다면:

```js
import { add, subtract, PI } from './math.js';

console.log(add(5, 3));        // Outputs: 8
console.log(subtract(10, 4));  // Outputs: 6
console.log(PI);               // Outputs: 3.14159
```

여기서는 `math.js` 모듈에서 필요한 특정 함수와 상수를 가져오고 있습니다. `'./math.js'` 부분은 현재 파일을 기준으로 모듈 파일이 어디에 있는지 JavaScript에 알려줍니다.

때때로 모듈이 내보내는 모든 것을 가져오고 싶을 때가 있습니다. 이럴 때는 별표(`*`) 구문을 사용하세요:

```js
import * as Math from './math.js';

console.log(Math.add(5, 3));        // Outputs: 8
console.log(Math.subtract(10, 4));  // Outputs: 6
console.log(Math.PI);               // Outputs: 3.14159
```

이 경우 `math.js`에서 내보낸 모든 항목이 `Math`이라는 객체의 프로퍼티로 가져와집니다.

또 다른 일반적인 패턴은 모듈에 기본 내보내기(default export)를 사용하는 것입니다. 보통 모듈이 주로 하나의 함수를 내보낼 때 사용합니다. 모듈당 기본 내보내기는 하나만 가질 수 있습니다.

다음과 같이 작성합니다:

```js
// In math.js
export default function multiply(a, b) {
    return a * b;
}

// In app.js
import multiply from './math.js';

console.log(multiply(4, 5));  // Outputs: 20
```

기본 내보내기를 가져올 때는 중괄호를 사용하지 않고, 가져오는 이름을 원하는 대로 정할 수 있다는 점에 주의하세요.

브라우저에서 ES6 모듈을 사용하려면 `type` 태그에서 `module`를 `script`로 지정해야 한다는 점도 중요합니다:

```html
<script type="module" src="app.js"></script>
```

모듈은 JavaScript 코드를 조직하고 구조화하는 강력한 방법을 제공합니다. 애플리케이션을 더 작고 관리하기 쉬운 조각으로 나누고, 코드 재사용을 촉진하며, 관심사의 분리를 깔끔하게 유지할 수 있게 도와줍니다. 더 큰 애플리케이션을 만들수록 모듈이 JavaScript 개발 도구에서 필수적인 도구임을 알게 될 것입니다.
