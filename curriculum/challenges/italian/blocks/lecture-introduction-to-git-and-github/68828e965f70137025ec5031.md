# --description--

Se hai lavorato a qualcosa, come una tesi, dove hai salvato copie dopo aver fatto modifiche importanti e hai chiamato i file con nomi tipo `Thesis-Draft`, `Thesis-Final`, `Thesis-Final-For-Real` e `Thesis-Absolute-Final-DO-NOT-CHANGE`, hai usato un approccio molto rudimentale al controllo di versione. Ma se ti dicessi che esiste un modo migliore?

Potresti aver sentito parlare di strumenti come Git, SVN o Mercurial. Questi sono sistemi di controllo di versione e sono una componente chiave del flusso di lavoro di ogni sviluppatore. Un sistema di controllo di versione ti permette di tracciare e gestire le modifiche nel tuo progetto. Per le lezioni che seguiranno, ci concentreremo su Git.

Git offre strumenti potenti che ti permettono di gestire la cronologia delle modifiche di un progetto. Imparerai di più su come usare Git e sulle funzionalità che offre nelle prossime lezioni. Per ora, ecco cosa devi sapere davvero:

Git ti permette di fare cose come creare commit, che sono un’istantanea di uno specifico stato del tuo codice. Puoi poi confrontare i commit per vedere quali modifiche hai fatto, annullare un commit che contiene modifiche errate e altro ancora.

Puoi anche creare "branch", che puoi pensare come percorsi diversi nel tuo codice. Mentre un branch è la strada che percorri, i commit sono i punti di riferimento lungo il percorso. I branch ti permettono di lavorare su una nuova funzionalità in isolamento, in modo che se il tuo lavoro va storto puoi eliminare il branch senza influenzare il codice principale.

Queste funzionalità rendono il controllo di versione un elemento essenziale per la collaborazione con altri sviluppatori. Quando più persone lavorano sullo stesso progetto, i branch permettono a ciascuno di lavorare sul proprio codice senza sovrascrivere le modifiche degli altri.

Spesso abbinerai un sistema di controllo di versione come Git a un provider di controllo di versione come GitHub o Gitlab, che ti offre strumenti di collaborazione aggiuntivi e la possibilità di fare backup dei tuoi progetti nel cloud.

Ma anche senza una soluzione di archiviazione basata sul cloud, i sistemi di controllo di versione possono rivoluzionare il tuo flusso di lavoro. Niente più file `THIS-IS-THE-FINAL-COPY-FOREVER`.

Ci sono molte informazioni interessanti da scoprire su come usare questi sistemi, quindi facciamo un’immersione profonda nelle prossime lezioni!
