# --description--

Die Verbesserung der „Time to Usable“, also des Zeitintervalls von der Anforderung einer Seite durch den Benutzer bis zu dem Moment, in dem er sinnvoll mit ihr interagieren kann, ist entscheidend für eine bessere Benutzererfahrung.

Lassen Sie uns effektive Strategien erkunden, um eine schnellere Time to Usable zu erreichen.

Beginnen Sie damit, sich auf die Inhalte zu konzentrieren, die der Benutzer zuerst sieht. Das bedeutet, wesentliche Elemente sofort zu laden und nicht-kritische Komponenten aufzuschieben.

Zum Beispiel könnten Sie versuchen, Lazy Loading zu implementieren.

Wir haben Lazy Loading bereits behandelt, aber hier ist ein Beispiel für Lazy Load von Bildern zur Auffrischung:

```html
<img src="image.jpg" loading="lazy" alt="Description">
```

Sie müssen nur das `loading="lazy"`-Attribut zu den Bild-Tags hinzufügen. Dadurch wird das Laden der Bilder aufgeschoben, bis sie benötigt werden, typischerweise wenn sie in den Viewport des Benutzers gelangen. Indem wir Bilder und Videos nur laden, wenn sie in den Viewport kommen, sparen wir Bandbreite und beschleunigen das initiale Rendern.

Normalerweise lädt eine Webseite alle Bilder auf einmal, auch solche, die nicht auf dem Bildschirm sichtbar sind. Mit Lazy Loading werden Bilder unterhalb des sichtbaren Bereichs, also außerhalb des Bildschirms, erst geladen, wenn der Benutzer zu ihnen scrollt.

Und hier ist ein Beispiel für Lazy Load von iframes:

```html
<iframe src="video.html" loading="lazy"></iframe>
```

Sie können dasselbe Attribut wie zuvor anwenden, diesmal aber auf iframes, und dasselbe Konzept gilt.

Sie können auch das `defer`-Attribut verwenden, um nicht-kritisches JavaScript bis nach dem initialen Seitenladen zu verzögern. Hier ist ein Beispiel, wie wir genau das tun:

```html
<script src="non-critical.js" defer></script>
```

Sie könnten auch in Betracht ziehen, render-blockierende Ressourcen zu minimieren.

Render-blockierende Ressourcen verzögern, dass die Seite interaktiv wird.

Um dies zu vermeiden, könnten Sie versuchen, CSS asynchron zu laden. Für nicht-kritisches CSS verwenden Sie das `media`-Attribut, um Stylesheets bedingt zu laden. Hier ist ein Beispiel, wie wir das tun:

```html
<link rel="stylesheet" href="print.css" media="print">
```

Indem Sie den Media-Wert als `print` angeben, wird das `print.css` nur geladen, wenn die Seite gedruckt oder auf dem Bildschirm angezeigt wird. Im nächsten Beispiel sagen wir durch Übergabe von `min-width: 800px` als Wert für das `media`-Attribut, dass wir `desktop.css` nur für Bildschirme mit einer Breite von mehr als 800px laden möchten:

```html
<!-- This stylesheet is only loaded on screens wider than 800px -->
<link rel="stylesheet" href="desktop.css" media="(min-width: 800px)">
```

Das spart Bandbreite, indem unnötige Styles übersprungen werden.

Zusammenfassend können Sie durch die Implementierung dieser Strategien die „Time to Usable“ Ihrer Webseite deutlich reduzieren und sicherstellen, dass Benutzer schnell mit Ihren Inhalten interagieren können.
