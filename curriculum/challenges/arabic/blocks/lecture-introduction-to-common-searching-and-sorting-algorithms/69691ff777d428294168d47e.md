# --description--

نموذج تقسيم المشكلة وغزوها في علوم الحاسوب هو تقنية لتقسيم المشكلات بشكل متكرر إلى مشكلات فرعية أصغر. أحد الجوانب الأساسية لهذه التقنية هو العودية، التي تحدث عندما تستدعي الدالة نفسها مرارًا حتى الوصول إلى حالة أساسية. في هذا الدرس، سنلقي نظرة على خوارزمية دمج الفرز لفهم أفضل لكيفية عمل تقنية تقسيم المشكلة وغزوها.

لنفترض أن لدينا هذه القائمة من الأرقام:

```md
42 37 53 17
```

الهدف هو فرز تلك القائمة من الأصغر إلى الأكبر باستخدام خوارزمية دمج الفرز. الخطوة الأولى هي تقسيم تلك القائمة إلى نصفين:

```md
42 37 | 53 17
```

بعدها نحتاج إلى النظر إلى الجانب الأيسر من القائمة:

```md
42 37
```

نأخذ تلك القائمة الفرعية ونقسمها إلى نصفين مرة أخرى حتى تحتوي كل قائمة فرعية على عنصر واحد فقط:

```md
42 | 37
```

القائمة التي تحتوي على عنصر واحد فقط تكون مرتبة افتراضيًا. بعد ذلك نحتاج إلى دمج كل تلك القوائم الفرعية ذات العنصر الواحد في قائمة مرتبة:

```md
37 42
```

ثم نتبع نفس العملية للجانب الأيمن من القائمة الأصلية:

```md
// right side of original list
53 17

// divide the list in half
53 | 17

// merge the lists in sorted order
17 53
```

الآن بعد أن تم فرز كلا نصفي القائمة الأصلية، ندمج هذين النصفيْن معًا ونرتب العناصر:

```markdown
17 37 42 53
```

إليك كيف تبدو الخوارزمية في الشفرة:

```javascript
function mergeSort(arr) {
    if (arr.length <= 1) {
        return arr;
    }

    const mid = Math.floor(arr.length / 2);
    const left = mergeSort(arr.slice(0, mid));
    const right = mergeSort(arr.slice(mid));

    const sorted = [];
    let i = 0;
    let j = 0;

    while (i < left.length && j < right.length) {
        if (left[i] <= right[j]) {
            sorted.push(left[i]);
            i += 1;
        } else {
            sorted.push(right[j]);
            j += 1;
        }
    }

    return sorted.concat(left.slice(i)).concat(right.slice(j));
}
```

تعقيد الزمن لخوارزمية الدمج يكون `O(n log n)` لأن القائمة تُقسّم باستمرار إلى نصفين (`log n`) ثم تُدمج معًا (`O(n)`). على عكس خوارزميات الترتيب الأخرى مثل bubble sort، خوارزمية الدمج ليست خوارزمية ترتيب في المكان ولها تعقيد مساحة قدره `O(n)`.
