## CSS 基础

- **什么是 CSS？**：层叠样式表（CSS）是一种用于为超文本标记语言元素应用样式的标记语言。CSS 用于颜色、背景图像、版型等。
- **CSS 规则的基本剖析**： CSS 规则由两个主要部分组成：选择器和声明块。 选择器是 CSS 中使用的一种模式，用于识别和针对特定 HTML 元素设计样式。 声明块为给定的一个或多个选择器应用一组样式。
- **`meta name="viewport"` 元素**：此 `meta` 元素为浏览器提供了如何在不同设备（尤其是手机和平板电脑）上控制页面尺寸和缩放比例的说明。
- **默认浏览器样式**：每个 HTML 元素都将应用默认浏览器样式。 这通常包括默认外边距和内边距等项目。

## 内联、内部和外部 CSS

- **内联 CSS**：这些样式可使用 `style` 属性直接写入 HTML 元素中。 出于分离的考虑，大多数情况下你不会使用内联 CSS。
- **内部 CSS**：这些样式写在 HTML 文档 `head` 部分的 `<style>` 标签内。这对于创建简短的代码示例很有用，但通常你不会使用内部 CSS。
- **外部 CSS**：这些样式被写入一个单独的 CSS 文件，并使用 `head` 部分的 `link` 元素链接到 HTML 文档。 在大多数项目中，你都会使用外部 CSS 文件，而不是内部或内联 CSS。

## 使用 `width` 和 `height` 属性

- **`width` 属性**：该属性指定元素的宽度。 如果未指定宽度，则默认设置为 `auto`。  This lets the browser determine the element's width based on its content, parent, and display type.
- **`min-width` 属性**：该属性指定元素的最小宽度。
- **`max-width` 属性**：该属性指定元素的最大宽度。
- **`height` 属性**：该属性指定元素的高度。 同样，高度默认为 `auto`，这意味着它会根据内部内容进行调整。
- **`min-height` 属性**：该属性指定元素的最小高度。
- **`max-height` 属性**：该属性指定元素的最大高度。

## 不同类型的 CSS 组合选择器

- **后代组合器（` `）&nbsp;\*选择器之间的单个空格**：此组合器用于定位指定父元素的后代元素。
- **子代组合符 (`>`)**：此组合符用于选择指定父元素的直接子元素。
- **相邻兄弟选择器（`+`）**：该选择器选择紧跟指定同级元素的元素。

- **后续兄弟选择器（`~`）**：该选择器选择指定元素后面的所有同级元素。

## 内联、块和内联块级元素

- **内联元素**：内联元素只占用所需的宽度，并且不会从新行开始。 这些元素在内容中流动，允许文本和其他内联元素出现在它们旁边。 常见的内联元素有 `span`、`anchor` 和 `img` 元素。
- **块级元素**：块级元素默认占据其容器可用的全部宽度，横跨容器的宽度。一些常见的块级元素有 `div`、`p` 和 `section` 元素。
- **内联块级元素**：你可以使用 `display` 属性将元素设置为 `inline-block` 。 这些元素的行为类似于内联元素，但可以像块级元素一样设置 `width` 和 `height`。

## 外边距和内边距

- **`margin` 属性**：该属性用于在元素外部、元素边框和周围元素之间设置空间。
- **`margin` 简写**：你可以指定 1-4 个值来设置各个外边距。 一个值适用于所有四边；两个值设置的先是 `top` 和 `bottom`，然后是 `right` 和 `left`；三个值设置的先是 `top`、水平的边（`right` 和 `left`），然后是 `bottom`；四个值依次设置的是 `top`、`right`、`bottom`、`left`。
- **`padding` 属性**：该属性用于在元素内部的内容和边框之间设置空间。
- **`padding` 简写**：你可以指定 1-4 个值来设置各个内边距。 一个值适用于所有四边；两个值设置的先是 `top` 和 `bottom`，然后是 `right` 和 `left`；三个值设置的先是 `top`、水平的边（`right` 和 `left`），然后是 `bottom`；四个值依次设置的是 `top`、`right`、`bottom`、`left`。

## CSS 优先级

- **内联 CSS 优先级**：内联 CSS 的优先级最高，因为它直接应用于元素。 它会覆盖任何内部或外部 CSS。 内联样式的优先级值为 (1, 0, 0, 0)。
- **内部 CSS 优先级**：内部 CSS 是在 HTML 文档 `head` 部分的 `style` 元素中定义的。 它比内联样式的优先级低，但可以覆盖外部样式。
- **外部 CSS 优先级**：外部 CSS 通过 `head` 部分的 `link` 元素进行链接，并写入单独的 `.css` 文件中。 它的优先级最低，但却能为大型项目提供最佳的可维护性。
- **Universal Selector (`*`)**: This is a special type of CSS selector that matches any element in the document. 它通常用于将一种样式应用于页面上的所有元素，这对于在不同浏览器中重置或规范样式非常有用。 在所有选择器中，通用选择器的优先级值最低。 它对优先级值的所有部分 (0, 0, 0, 0) 的贡献均为 0。
- **类型选择器**：这些选择器根据元素的标签名称选择元素。 与其他选择器相比，类型选择器的优先级相对较低。 类型选择器的优先级值为 (0, 0, 0, 1)。
- **类选择器**：这些选择器由一个句点（`.`）后跟类名定。 类选择器的优先级值为 (0, 0, 1, 0)。 这意味着类选择器可以覆盖类型选择器，但它们可以被 ID 选择器和内联样式覆盖。
- **ID Selectors**: ID selectors are defined by a hash symbol (`#`) followed by the ID name. ID 选择器的优先级非常高，高于类型选择器和类选择器，但低于内联样式。 ID 选择器的优先级值为 (0, 1, 0, 0)。
- **`!important` 关键字**：用于赋予样式规则最高优先级，使其优先于某个属性的其他声明。 使用时，它会强制浏览器应用指定的样式，而不考虑其他选择器的优先级。 使用 `!important` 时应谨慎，因为它会使 CSS 更难维护和调试。
- **级联算法**： 当有多个样式针对同一元素时，用于决定应用哪些 CSS 规则的算法。 它确保根据一套明确定义的规则，使用最合适的样式。
- **CSS 继承**：样式从父元素向子元素传递的过程。 通过继承，可以在文档树的较高层次定义样式，并将其应用于多个元素，而无需为每个元素明确指定样式。

## 设计术语

- **布局**：视觉元素在页面或屏幕上的排列方式，用以传达信息。 这些元素可能包括文字、图像和留白。
- **对齐**：元素之间的位置关系。 正确使用对齐方式有助于使设计看起来整洁有序。
- **构图**：一种排列元素以创造和谐设计的行为。 它决定了图像、文字和形状等元素如何相互关联，并以艺术的方式为设计做出贡献。
- **平衡**：视觉权重在构图中的分布方式。 设计师的目标是通过对称或不对称的安排来创造平衡。
- **缩放**：将一个元素的尺寸或大小与另一个元素的尺寸或大小进行比较。
- **层次结构**：确定了设计中各元素的重要性顺序。 这是要确保最重要的信息首先得到关注。
- **对比**：在元素之间建立明确区别的过程。 你可以通过颜色、大小、形状、纹理或任何其他视觉特征的变化来实现这一点。 强烈的对比度也有助于提高可读性。
- **空白（负空间）**：这是设计中的空白区域。它是环绕元素的区域。
- **UI（用户界面）**：UI 包括用户可以在其屏幕上看到的视觉和交互元素，如图标、图像、文本、菜单、链接和按钮。
- **UX（用户体验）**：UX 是关于用户在使用产品或服务时的感受。具有良好设计用户体验的应用直观、易用、高效、可访问且令人愉悦。
- **设计概要**：一份概述项目目的、目标和要求的文件。 它是指导设计过程的路线图，确保最终产品满足客户的需求。
- **基于矢量的设计**：包括使用数学公式定义线条、形状和颜色来创作数字艺术。
- **原型设计**：指创建产品或用户界面交互模型的过程。

## UI 设计基础

- **背景色和前景色的良好对比度**：确保背景色和前景色具有良好的对比度对文字的可读性非常重要。 《网页内容无障碍指南》（WCAG）建议，普通文字的对比度最低为 4.5:1，大文字的对比度最低为 3:1。
- **设计中的良好视觉层次结构**： 强大的视觉层次结构可以为视觉提供清晰的路径，确保你传达的信息按照你所希望的顺序被阅读。
- **响应式图像**：响应式图像是指能够根据其显示屏幕的尺寸进行缩放的图像。 这一点很重要，因为它能确保你的图像在所有设备上，从台式机到手机，都看起来良好。
- **渐进式增强**：这是一种设计方法，它确保所有用户，无论使用何种浏览器或设备，都能访问应用程序的核心内容和功能。
- **以用户为中心的设计**：这是一种优先考虑最终用户的方法，从他们的需求到他们的偏好和限制。 以用户为中心的设计的目标是设计出直观、使用高效、让用户乐于互动的网页。
- **用户研究**：这是对使用产品的人进行的系统研究。 目标是衡量用户的需求、行为和痛点。
- **用户测试**：这是指在用户与应用程序交互时从他们那里获取数据的做法。
- **A/B 测试**：这是向随机选择的用户群子集发送新功能的过程。 然后，你可以利用分析数据来确定该功能是否有益。
- **用户要求**：这是指你的应用程序需要遵循的故事或标准。 用户需求可以通过用户研究或行业标准来确定。 它们甚至可以由利益相关者的意见来定义。
- **渐进式披露**：这是一种设计模式，用于仅根据用户当前的活动向其显示相关内容，而隐藏其他内容。 这样做是为了减少认知负荷，使用户体验更加直观。
- **延迟/懒注册**：这是一种用户界面设计模式，允许用户在无需注册的情况下浏览应用程序并与之交互。

## 设计最佳实践

- **深色模式**：这是网络应用程序的一项特殊功能，你可以将默认的浅色配色方案更改为深色配色方案。 在深色模式下应使用不饱和色。 不饱和色是指强度较低、饱和度较低的颜色。
- **面包屑**：这是一种导航辅助工具，用于显示用户在网站层次结构中的位置。 最好在页面顶部放置面包屑，以便用户轻松找到。 而且，要确保面包屑足够大，便于阅读，但又不会太大，以免占用页面太多空间。
- **卡片组件**：你的卡片组件应设计简洁，视觉上不杂乱或显示过多信息。 对于媒体内容，确保选择高质量的图像和视频，以增强用户体验。
- **无限滚动**：这是一种随着用户向下滚动页面而加载更多内容的设计模式。 你应该考虑使用“加载更多”按钮，因为它可以让用户控制何时想看更多内容。 你还可以添加一个返回按钮，这样用户就可以返回到上一页，而不必一直向上滚动。
- **模态对话框**：这是一种弹出式窗口，将显示在现有页面内容的顶部。 通常情况下，背景内容会覆盖一层暗淡的颜色，以帮助用户更好地关注模态内容。 此外，允许用户点击模态对话框外区域将其关闭是一个好主意。 当你使用 HTML `dialog` 元素时，你将获得许多内置功能和可访问性优势。
- **表单注册的进度指示**：这是一种向用户显示其在流程中所处位置的方法。 它可用于表单、注册和设置流程。 你的设计应该简洁、易于查找，并且可以返回到之前的步骤。
- **购物车**： 购物车是用户查看自己已在电子商务平台上选择的商品的地方。 用户应始终能看到购物车，使用购物车、购物袋或购物篮等常用图标，并有一个清晰的行动召唤按钮，以便用户去结账。

## 常用设计工具

- **Figma**：这款基于云的工具专门用于用户界面和用户体验（UI/UX）设计。 它使设计和开发团队能够随时随地进行协作，提供的内置功能包括基于矢量的设计、自动布局、评论和反馈系统等。
- **Sketch**：这是一款流行的设计工具，界面直观、操作简单，非常适合想要快速创建原型的开发人员使用。 它还被设计师广泛用于创建 UI、图标和网页布局等任务。
- **Adobe XD**：这是一款基于矢量的设计和原型工具，适用于 UI/UX 设计，因其与 Photoshop、Illustrator 和 After Effects 等其他 Adobe 应用程序的无缝集成而闻名。
- **Canva**：该工具可让你创建各种可视化内容，包括海报、封面照片、演示文稿、视频短片等。 它的设计简单易用，非常适合初学者使用。

## 绝对单位

- **`px`（像素）**：该绝对单位是 CSS 中的固定尺寸测量单位，可精确控制尺寸。 这意味着 1px 始终等于 1/96 英寸。
- **`in`（英寸）**：该绝对单位等于 96px。
- **`cm`（厘米）**：该绝对单位等于 25.2/64 英寸。
- **`mm`（毫米）**：该绝对单位等于 1/10 厘米。
- **`q`（1/4 毫米）**：该绝对单位等于 1/40 厘米。
- **`pc`（派卡）**：该绝对单位等于 1/6 英寸。
- **`pt`（点）**：该绝对单位等于 1/72 英寸。

## 相对单位

- **百分比**：使用这些相对单位，可以将大小、尺寸和其他属性定义为其父元素的一定比例。 例如，如果在一个元素上设置 `width: 50%;` ，它将占据其父容器宽度的一半。
- **`em` 单位**：该单位是相对于元素的字体大小而言的。 如果你使用 `ems` 作为 `font-size` 属性，文本的大小将相对于父元素的字体大小。
- **`rem` 单位**：该单位是相对于根元素，即 `html` 元素的字体大小而言的。
- **`vh` 单位**：`vh` 代表 `"viewport height"`，`1vh` 等于视口高度的 1%。
- **`vw` 单位**：`vw` 代表 `"viewport width"`，`1vw` 等于视口宽度的 1%。

## `calc` 函数

- **`calc()` 函数**：使用 `calc()` 函数，你可以直接在样式表中执行计算，动态确定属性值。 这意味着，你可以根据视口尺寸或其他元素计算尺寸，创建灵活的响应式用户界面。

## 用户行为伪类

- **伪类定义**：伪类是一种特殊的 CSS 关键字，可以根据元素的特定状态或位置来选择元素。
- **用户行为伪类**：这些特殊的关键字允许你根据用户交互改变元素的外观，从而改善整体用户体验。
- **`:active` 伪类**：该伪类允许你选择元素的激活状态，如点击按钮。
- **`:hover` 伪类**：该伪类定义了元素的悬停状态。
- **`:focus` 伪类**：该伪类在元素获得焦点时应用样式，通常是通过键盘导航或用户点击表单的输入框。
- **`:focus-within` 伪类**：该伪类用于在元素或其任何后代元素获得焦点时对其应用样式。

## Input 伪类

- **Input 伪类**：这些伪类用于根据 HTML `input` 元素在用户交互之前和之后的状态来定位它们。
- **`:enabled` 伪类**：该伪类用于定位当前已启用的表单按钮或其他元素。
- **`:disabled` 伪类**：该伪类允许你为禁用模式下的交互式元素设计样式。
- **`:checked` 伪类**：该伪类用于向用户表明它已被选中。
- **`:valid` 伪类**：该伪类的目标是符合验证标准的输入字段。
- **`:invalid` 伪类**：该伪类的目标是不符合验证标准的输入字段。
- **`:in-range` 和 `:out-of-range` 伪类**：这些伪类根据元素的值是否在指定的范围约束内或外，应用样式。
- **`:required` 伪类**：该伪类的目标是具有 `required` 属性的 `input` 元素。 它提示用户，他们必须填写该字段才能提交表单。
- **`:optional` 伪类**：此伪类将样式应用于非必填且可以为空的输入元素。
- **`:autofill` 伪类**：该伪类为浏览器自动填充的已保存数据的输入字段应用样式。

## 位置伪类

- **位置伪类**：这些伪类用于为当前文档中的链接和元素设计样式。
- **`:any-link` 伪类**：此伪类是 `:link` 和 `:visited` 伪类的组合。因此，它匹配任何带有 href 属性的锚点元素，无论是否已访问。
- **`:link` 伪类**：该伪类允许你选择网页上所有未访问的链接。 你可以用它在用户点击链接之前为链接设计不同的样式。
- **`:local-link` 伪类**：该伪类选择指向同一文档的链接。 当你想区分内部链接和外部链接时，它可能会很有用。
- **`:visited` 伪类**：该伪类的目标是用户访问过的链接。
- **`:target` 伪类**：此伪类用于应用样式到作为 URL 片段目标的元素。

## 树结构伪类

- **树结构伪类**：这些伪类允许你根据元素在文档树中的位置来选择并确定其样式。
- **`:root` 伪类**：该伪类通常是根 `html` 元素。 它可以帮助你选择文档中的最高级别，从而在整个文档中应用通用样式。
- **`:empty` 伪类**：空元素，即文档树中还包括的除空白区域外没有子元素的元素。 这就是为什么有一个 `:empty` 伪类来选择空元素。
- **`:nth-child(n)` 伪类**：该伪类允许你根据元素在父类中的位置选择元素。
- **`:nth-last-child(n)` 伪类**：该伪类允许你从末尾开始计数来选择元素。
- **`:first-child` 伪类**：该伪类选择父元素或文档中的第一个元素。
- **`:last-child` 伪类**：此伪类选择父元素或文档中的最后一个元素。
- **`:only-child` 伪类**：此伪类选择父元素或文档中唯一的元素。
- **`:first-of-type` 伪类**：该伪类选择父类中首次出现的特定元素类型。
- **`:last-of-type` 伪类**：该伪类选择父类中最后出现的特定元素类型。
- **`:nth-of-type(n)` 伪类**：该伪类允许你根据元素在同类型同级元素中的位置，选择父元素中的某个特定元素。
- **`:only-of-type` 伪类**：如果元素是父元素中唯一的同类型元素，则该伪类会选择该元素。


## 函数伪类

- **函数伪类**：函数伪类允许你基于更复杂的条件或关系选择元素。与基于状态选择元素的常规伪类（例如，`:hover`、`:focus`）不同，函数伪类接受参数。
- **`:is()` 伪类**：该伪类接收一个选择器列表（ 例如 `ol`、`ul`），并选择一个与列表中某个选择器相匹配的元素。
- **`:where()` 伪类**：该伪类接收一个选择器列表（ 例如 `ol`、`ul`），并选择一个与列表中某个选择器相匹配的元素。 `:is` 和 `:where` 的区别在于后者的优先级为 0。
- **`:has()` 伪类**：该伪类通常被称为 `"parent"` 选择器，因为它允许你为包含选择器列表中指定的子元素的元素设计样式。

## 伪元素

- **`::before` 伪元素**：该伪元素使用 `content` 属性在元素前插入图标等装饰内容。
- **`::after` 伪元素**：该伪元素使用 `content` 属性在元素后插入图标等装饰内容。
- **`::first-letter` 伪元素**：该伪元素的目标是元素内容的第一个字母，允许你对其进行样式设置。
- **`::marker` 伪元素**：该伪元素能让你选择列表项的标记（项目符号或编号）进行样式设置。


## 色彩理论

- **色彩理论定义**：这是关于色彩之间如何相互作用以及它们如何影响我们的感知的研究。 内容包括色彩关系、色彩和谐以及色彩的心理影响。
- **原色**：这些颜色包括黄色、蓝色和红色，是所有其他颜色的基本色调。
- **间色**：这些颜色是由两种原色等量混合而成。 绿色、橙色和紫色都是间色。
- **复色**：这些颜色由一种原色和一种邻近的间色组合而成。 黄绿、蓝绿和蓝紫都属于复色。
- **暖色**：这些颜色包括红色、橙色和黄色，能唤起舒适、温暖和惬意的感觉。
- **冷色**：这些颜色包括蓝色、绿色和紫色，能唤起平静、安详和专业的感觉。
- **色轮**：色轮是一个圆形图，表示颜色之间的关系。 它是设计师的必备工具，因为它可以帮助他们选择色彩组合。
- **邻近色配色方案**：这些配色方案能营造出和谐舒缓的体验。 它们有类似的颜色，在色轮中彼此相邻。
- **互补色配色方案**：这些配色方案能产生强烈的对比和视觉冲击。 相对于彼此，它们的颜色位于色轮的两端。
- **三色配色方案**：该配色方案色彩鲜艳。 它们由彼此距离大致相等的颜色组成。 如果把它们相连，就会在色轮上形成一个等边三角形。
- **单色配色方案**： 在这种配色方案中，所有颜色都是通过调整明度、暗度和饱和度从相同的基色中衍生出来的。 这既能唤起统一和谐的感觉，又能形成对比。

## 在 CSS 中使用颜色的不同方法

- **命名颜色**：这些颜色是浏览器识别的预定义颜色名称。 包括 `blue`、`darkred`、`lightgreen`。
- **`rgb()` 函数**：RGB 代表红、绿、蓝——光的三原色。 这三种颜色以不同的强度组合在一起，创造出多种颜色。  The `rgb()` function allows you to define colors using the RGB color model.
- **`rgba()` 函数**：该函数添加了第四个值——alpha——用于控制颜色的透明度。
- **`hsl()` 函数**：HSL 代表色相、饱和度和亮度——是定义颜色的三个关键要素。
- **`hsla()` 函数**：此函数添加了第四个值——alpha——用于控制颜色的不透明度。
- **十六进制**：hex 代码（十六进制代码的简称）是一个六个字符的字符串，用于表示 RGB 颜色模型中的颜色。 “hex”指的是十六进制计数系统，使用数字 0 至 9 和字母 A 至 F。

## `box-shadow` 属性

- **定义**：`box-shadow` 属性在一个元素周围应用一个或多个阴影。
- **偏移值**：你必须指定水平（`offset-x`）和垂直（`offset-y`）值。正的 `offset-x` 会将阴影向右移动，负值则向左移动。正的 `offset-y` 会将阴影向下移动，负值则向上移动。如果值为 `0`，则无需包含单位。
- **模糊半径**：此可选的值控制阴影的模糊程度。如果未包含，则默认为 `0`，这会创建犀利的边缘。数值越大，阴影越柔和。
- **扩散半径**：此可选的值控制阴影的扩展或收缩程度。如果未包含，则默认为 `0`。
- **阴影颜色**：你可以使用命名颜色、十六进制值、`rgb()`、`rgba()`、`hsl()` 或 `hsla()` 函数来指定颜色。
- **`inset` 关键字**：添加 `inset` 关键字会将阴影放置在元素内部，而不是外部。
- **应用多个盒子阴影**：你可以通过逗号分隔来应用多个阴影。阴影从前到后分层。

### 语法

box-shadow: offset-x offset-y blur-radius spread-radius color;

## 设置输入样式的最佳实践

- **设置输入框样式**： 与所有文本元素一样，你需要确保应用于文本输入的样式是可访问的。 这意味着字体大小要适当，颜色与背景要有足够的对比度。 输入元素也得是可聚焦的。 在编辑样式时，应注意在元素获得焦点时保留一个明显的视觉标识，如加粗边框。

## 对输入使用 `appearance: none`

- **`appearance: none`**：浏览器会对很多元素应用默认样式。 `appearance: none` CSS 属性可让你完全控制样式，但也有一些注意事项。 为输入元素创建自定义样式时，需要确保焦点和错误提示标识仍然存在。

## 设置 `datetime-local` 和 `color` 属性样式的常见问题

- **常见问题**：这些特殊类型的输入依赖于复杂的伪元素来创建日期和颜色选择器等内容。 这给设置这些输入的样式带来了巨大挑战。 其中一个挑战是默认样式完全取决于浏览器，因此你编写的用于实现选择器预期外观的 CSS，在另一款浏览器中可能呈现截然不同的效果。

## CSS 溢出属性

- **定义**：溢出是指元素在其内容超出或“溢出”其包含元素大小时的处理方式。 溢出是二维的。
- **`overflow-x`**：x 轴决定水平溢出。
- **`overflow-y`**：y 轴决定垂直溢出。

## CSS 变换属性

- **定义**：它可以对元素进行各种变换，如旋转、缩放、倾斜或在二维或三维空间中平移（移动）。
- **`translate()` 函数**：该函数用于从当前位置移动元素。
- **`scale()` 函数**：该函数能让你改变元素的大小。
- **变换和可访问性**：如果你使用变换隐藏或显示内容，请确保屏幕阅读器和键盘导航仍可访问内容。 隐藏内容应真正隐藏，例如使用 `display: none` 或 `visibility: hidden`，而不仅仅是在视觉上移到屏幕外。

## 盒模型

- **定义**：在 CSS 盒模型中，每个元素都被一个盒子包围。 这个盒子由四个部分组成：内容区域、`padding`、`border`、`margin`。
- **内容区域**：内容区域是盒子的最内层部分。 它是包含文本或图像等元素实际内容的空间。
- **`padding`**：内边距是紧邻内容区域的部分。 它是内容区域与元素边框之间的空间。
- **`border`**：边框是元素在盒模型中的最外层轮廓。 这是元素的视觉边界。
- **`margin`**：外边距是元素边框之外的空间。 用于控制该元素与周围其他元素之间的距离。

## 外边距折叠

- **定义**：这种行为发生在相邻元素的垂直外边距发生重叠时，结果是合并成一个外边距，等于两者中较大的那一个。 这种行为只适用于垂直外边距（上下），不适用于水平外边距（左右）。

## `content-box` 和 `border-box` 属性值

- **`box-sizing` 属性**：该属性用于确定如何计算 HTML 元素的最终 `width` 和 `height`。
- **`content-box` 值**：在 `content-box` 模式中，为元素设置的 `width` 和 `height` 决定了内容区域的尺寸，但不包括 `padding`、`border` 或 `margin`。
- **`border-box` 值**：在 `border-box` 模式中，元素的 `width` 和 `height` 包括内容区域、 `padding` 和 `border`，但不包括 `margin`。

## CSS 重置

- **定义**：CSS 重置是一种样式表，可移除网络浏览器应用于 HTML 元素的全部或部分默认格式。 CSS 重置的第三方选项包括 `sanitize.css` 和 `normalize.css`。

## CSS 滤镜属性

- **定义**：该属性可用于创建各种效果，如模糊、色彩偏移和对比度调整。
- **`blur()` 函数**：该函数对元素进行高斯模糊处理。 数值以像素为单位，表示模糊的半径。
- **`brightness()` 函数**：该函数调整元素亮度。 0% 的值将使元素完全变黑，而 100% 以上的值会增加亮度。
- **`grayscale()` 函数**：该函数将元素转换为灰度。 数值定义为百分比，100% 表示完全灰度，0% 则使图像保持不变。
- **`sepia()` 函数**：该函数为元素添加棕褐色调。 与灰度一样，它也使用百分比值。
- **`hue-rotate()` 函数**：该函数对元素应用色相旋转。 值以度为单位，表示围绕色环的旋转。

## CSS Flexbox 和 Flex 模型简介

- **定义**：CSS flexbox 是一种一维版型模型，允许你在容器内按行和列排列元素。
- **Flex Model**：该模型定义了弹性项在弹性容器内的排列方式。每个弹性容器都有两个轴：主轴和交叉轴。

## `flex-direction` 属性

- **定义**：此属性设置主轴的方向。`flex-direction` 的默认值是 `row`，它将所有的弹性项放置在同一行，方向为浏览器默认语言的方向（从左到右或从右到左）。
- **`flex-direction: row-reverse;`**：这会反转行中的项。
- **`flex-direction: column;`**：这将使弹性项垂直排列，而不是水平排列。
- **`flex-direction: column-reverse;`**：这会垂直反转弹性项的顺序。

## `flex-wrap` 属性

- **定义**：此属性决定了弹性项如何在弹性容器内换行以适应可用空间。`flex-wrap` 可以取三个可能的值：`nowrap`、`wrap` 和 `wrap-reverse`。
- **`flex-wrap: nowrap;`**：这是默认值。弹性项不会换行，即使它们的宽度超过容器的宽度。
- **`flex-wrap: wrap;`**：当项超出其容器宽度时，此属性将自动换行。
- **`flex-wrap: wrap-reverse;`**：此属性将以相反顺序换行 flex 项。
- **`flex-flow` 属性**：此属性是 `flex-direction` 和 `flex-wrap` 的简写属性。

## `justify-content` 属性

- **定义**：此属性沿着弹性容器的主轴对齐子元素。
- **`justify-content: flex-start;`**：在这种分支中，弹性项将对齐到主轴的起点。这可以是水平或垂直方向。
- **`justify-content: flex-end;`**：在这种情况下，弹性项沿主轴的末端对齐，水平或垂直方向。
- **`justify-content: center;`**：这会将弹性项沿主轴居中。
- **`justify-content: space-between;`**：这将沿主轴均匀分布元素。
- **`justify-content: space-around;`**：这将在主轴内均匀分布弹性项，在第一个项之前和最后一个项之后添加空白。
- **`justify-content: space-evenly;`**：这会沿主轴均匀分布这些项。
  
## `align-items` 属性

- **定义**：此属性用于沿交叉轴分布项。请记住，交叉轴垂直于主轴。
- **`align-items: center;`**：用于沿交叉轴居中这些项。
- **`align-items: flex-start;`**：这将项对齐到交叉轴的起点。
- **`align-items: stretch;`**：用于沿交叉轴拉伸弹性项。

## 排版简介

- **定义**：排版是选择合适字体和形式，使文本在视觉上吸引人且易于读取的艺术。 “Type” 指的是单个字符的设计和排列方式。
- **字型定义**：字型是一组字符、数字和符号的整体设计和风格。 它就像一个字体家族的蓝图。
- **字体定义**：字体是具有如大小、重量、风格和宽度等特定特征的字型的具体变体。
- **衬线体**：衬线体具有古典风格，字符末端有小线条。 衬线体通常用于书籍等印刷品。
- **无衬线体**：无衬线体的外观更为现代，字符末端没有小线条。 无衬线体通常用于数字设计，因为它们在屏幕上易于阅读。 例如，Helvetica、Arial 和 Roboto。
- **字体字重**：字符的粗细，包括细体、常规体、粗体和特粗体。
- **字体样式**：字符的倾斜方向与形态，如斜体和伪斜体。
- **字体宽度**：字符所占的水平空间，包括压缩和扩展。
- **基线**：是大多数字符所依附的一条假想线。
- **大写字母高度**：是指大写字母从基线到顶部的垂直高度。
- **X 高度**：是指小写字母的平均高度，不包括升部笔画和降部笔画。
- **升部**：是小写字母中超出 x 高度的部分，例如字母“h”、“b”、“d”和“f”的顶部。
- **降部**：是小写字母中向下延伸至基线以下的部分，例如字母“y”、“g”、“p”和“q”的尾部。
- **字距调整**：是指调整特定字符对之间的间距，以提升其可读性和美观度。 例如，减少字母 A 和 V 之间的间距。
- **字距微调**：是指调整整段文本中所有字符之间的间距。 本质上就是字符之间的距离。 它会影响文本的疏密程度。
- **行距**：是指文本行与行之间的垂直间距。 通常是从一行的基线到下一行基线的距离。
- **排版最佳实践**：应选择简洁明了的字体，使设计易于理解。 这对于网站正文尤为重要。 如果字体易于阅读，用户更可能与你内容互动。 为了创造视觉一致性，你应该最多使用两到三种字体。 使用过多字体会使文本更难阅读，并削弱你的品牌识别度。 这还可能因增加网站加载时间而影响用户体验。 你可以利用字体大小，为标题、副标题、段落和其他元素创建视觉层次。 例如，网页上的主标题应使用更大的字体，然后是较小字体的副标题。 这将为层次结构中的每个元素赋予特定的字体大小，帮助用户通过视觉层次浏览内容。

  
## CSS `font-family` 属性

- **定义**：字体族是一组共享通用设计的字体。 属于同一字体族的所有字体均基于同一核心字型，但它们的风格、粗细和宽度会有所变化。 你可以通过逗号分隔，按优先级从高到低指定多个字体族。 这些备用字体将作为回退选项。 你应始终在字体族列表末尾包含一个通用字体族。
- **常见字体族**：以下是 CSS 中使用的一些常见字体族：serif、sans-serif、monospace、cursive、fantasy。

## 网页安全字体

- **定义**：网页安全字体是极有可能已安装在计算机或设备上的一类字体子集。 当浏览器需要渲染字体时，它会尝试在用户系统中查找字体文件。 但若未找到该字体，通常会回退到默认系统字体。

- **网页安全字体**：
  - 无衬线字体因字符末端无“小脚”或细线，常用于网页开发，易于屏幕阅读。 网页安全无衬线字体的一些例子有 Arial、Verdana 和 Trebuchet MS。
  - 相比之下，衬线字体因字符末端有“小脚”，常用于传统印刷。 网页安全衬线字体包括：Times New Roman 和 Georgia。

## @ 规则和 `@font-face` 规则

- **定义**：@规则是向浏览器提供指令的语句。 你可以将其用于定义样式表的各个方面，例如媒体查询、关键帧、字体面等。
- **`@font-face`**：此规则能让你通过指定字体文件、格式以及其他重要属性来定义自定义字体，如粗细和样式。 要使 `@font-face` 规则有效，你还需要指定 `src` 属性。 该属性包含对字体资源的引用。
- **字体格式**：你还可以为每个字体资源指定格式。 这是可选项。 这是浏览器对字体格式的提示。 如果省略格式，将下载资源，并在下载后检测格式。 如果格式无效，将无法下载资源。 可能的字体格式包括 `collection`、`embedded-opentype`、`opentype`、`svg`、`truetype`、`woff` 和 `woff2`。
- **`woff` 和 `woff2`**：`woff` 代表 "Web Open Font Format"。它是一种由 Mozilla 与 Type Supply、LettError 及其他组织合作开发的广泛使用的字体格式。`woff` 和 `woff2` 之间的区别在于用于压缩数据的算法。
- **OpenType**：这是微软和 Adobe 开发的一种可扩展计算机字体格式，允许用户访问字体中的其他功能。 它广泛应用于各大操作系统。
- **`tech()`**：用于指定字体资源的技术。 这是可选项。

## 使用外部字体

- **Definition**: An external font is a font file that is not included directly within your project files. 它们通常托管在单独的服务器上。 要在项目中包含这些外部字体，可以在 CSS 中使用 `link` 元素或 `@import` 语句。
- **Google 字体**：这是一项 Google 服务，提供一系列字体，其中许多字体是专门为网页开发而设计的。
- **Font Squirrel**：这是另一种流行的资源，可用于为项目添加自定义外部字体。

## `text-shadow` 属性

- **定义**：此属性用于对文本应用阴影。 你需要指定 X 和 Y 偏移，它们分别代表阴影与文本的水平和垂直距离。 这些值是必需的。 X 偏移和 Y 偏移的正值将分别使阴影向右和向下移动，而负值将使阴影向左和向上移动。
- **阴影颜色**：你还可以在偏移之前或之后指定此值，从而自定义阴影的颜色。 如果未指定颜色，浏览器会自动确定颜色，因此通常最好明确设置颜色。
- **模糊半径**：模糊半径是可选项，会使阴影看起来更平滑、更细腻。 模糊半径的默认值为零。 数值越大，模糊程度越高，这意味着阴影会变浅。
- **应用多个文本阴影**：文本可以有多个阴影。 这些阴影将从前到后分层应用，第一个阴影位于顶层。

- **TPGi 颜色对比分析器**：这是一款免费的颜色对比度检查器，可让你检查网站和应用程序是否符合《网页内容可访问性指南》（WCAG）标准。 该工具还带有色盲模拟器功能，可让你查看网站或应用程序在色觉异常用户眼中是什么样子。

## 可访问性树

可访问性树是屏幕阅读器等辅助技术用来解释网页内容并与之交互的结构。

## `max()` 函数

`max()` 函数返回一组逗号分隔值中最大的值：

```css
img {
  width: max(250px, 25vw);
}
```

In the above example, the width of the image will be 250px if the viewport width is less than 1000 pixels (since 250px is greater than 25% of that viewport width). 如果视口宽度大于 1000 像素，图像宽度将为 25vw。 这是因为 25vw 相当于视口宽度的 25%。

## CSS 和可访问性最佳实践

- **`display: none;`**：使用 `display: none;` 意味着屏幕阅读器和其他辅助技术将无法访问此内容，因为它不包含在辅助功能树中。 因此，只有当你想从视觉呈现和可访问性两方面完全删除内容时，才必须使用这种方法。
- **`visibility: hidden;`**：该属性和值在视觉上隐藏内容，但将其保留在文档流中，这意味着它仍占据页面空间。 屏幕阅读器也无法读取这些元素，因为它们已经从辅助功能树中删除。
- **`.sr-only` CSS 类**：这是一种常用的技术，用于在视觉上隐藏内容，同时保持屏幕阅读器可以访问这些内容。

```css
.sr-only {
  position: absolute;
  width: 1px;
  height: 1px;
  padding: 0;
  margin: -1px;
  overflow: hidden;
  clip: rect(0, 0, 0, 0);
  white-space: nowrap;
  border: 0;
}
```

- **`scroll-behavior: smooth;`**：该属性和值可实现平滑滚动效果。
- **`prefers-reduced-motion` 特性**：该媒体特性可用于检测用户的动画偏好。

```css
@media (prefers-reduced-motion: no-preference) {
  * {
    scroll-behavior: smooth;
  }
}
```

在上面的示例中，如果用户的设备上没有设置动画偏好，就会启用平滑滚动。

## 使用 HTML 属性隐藏内容

- **`aria-hidden` 属性**：用于向使用屏幕阅读器等辅助技术的用户隐藏元素。 例如，这可用于隐藏不提供任何有意义内容的装饰图片。
- **`hidden` 属性**：大多数现代浏览器都支持该属性，它既隐藏内容的视觉显示，也从可访问性树中将其移除。 它可以通过 JavaScript 轻松切换。

:::interactive_editor

```html
<p aria-hidden>This paragraph is visible for sighted users, but is hidden from assistive technology.</p>
<p hidden>This paragraph is hidden from both sighted users and assistive technology.</p>
```

:::

## `placeholder` 属性的可访问性问题

使用占位符文本不利于可访问性。 用户经常会将占位符文本与实际输入值混淆，以为输入中已经有一个值。

## 使用不同的属性选择器和链接

- **定义**: 通过 `attribute` 选择器，你可以根据 HTML 元素的属性来定位这些元素，如 `href` 或 `title` 属性。
- **`title` 属性**：该属性提供有关元素的附加信息。

## 使用 `lang` 和 `data-lang` 属性定位元素

- **`lang` 属性**：该属性在 HTML 中用于指定元素内容的语言。 特别是在多语言网站上，你可能需要根据元素所使用的语言为其设计不同的样式。
- **`data-lang` 属性**：自定义数据属性如 `data-lang` 通常用于在元素中存储附加信息，例如指定特定文本部分所使用的语言。


## 使用属性选择器、有序列表元素和 `type` 属性

- **`type` Attribute**：在 HTML 中处理有序列表时，`type` 属性允许你指定所使用的编号样式，如数字、字母或罗马数字。

## 使用浮动

- **定义**：浮动用于将一个元素从页面的正常流中移除，并将其定位在其容器的左侧或右侧。当发生这种情况时，文本将环绕该浮动内容。
- **清除浮动**：`clear` 属性用于确定是否需要将元素移动到浮动内容的下方。 当多个浮动元素堆叠在一起时，可能会出现布局重叠和折叠的问题。 因此，我们创建了一个 `clearfix` 技术来解决这个问题。

## 静态、相对和绝对定位

- **静态定位**：这是文档的正常流。元素按从上到下、从左到右的顺序依次定位。
- **相对定位**：允许你使用 `top`、`left`、`right` 和 `bottom` 属性在正常文档流程中定位元素。 你还可以使用相对定位来使元素与页面上的其他元素重叠。
- **绝对定位**：允许你将一个元素从正常的文档流程中分离出来，使其行为独立于其他元素。

## 固定和粘性定位

- **固定定位**：当使用 `position: fixed` 定位元素时，它会从正常文档流中移除并相对于视口放置，这意味着即使用户滚动，它也会保持在相同的位置。 这通常用于标题或导航栏等需要始终保持可见的元素。
- **Sticky Positioning**: This type of positioning will act as a hybrid between relative and fixed positioning. 起初，元素的行为就像相对定位一样，保持在文档流程中。 但是，一旦用户滚动元素超过某个点，它就会“粘”在视口（通常是顶部）上，并表现得像是固定定位。 这非常适合创建粘性导航栏等元素，只有当用户滚动到某个位置时，导航栏才会固定下来。

## 使用 `z-index` 属性

- **定义**：CSS 中的 `z-index` 属性用于控制页面上堆叠的已定位元素的垂直顺序。

## 响应式网页设计

- **定义**：响应式设计的核心原则是适应性，即网站能够根据屏幕尺寸和浏览设备的功能调整布局和内容。
- **流动网格**：流动网格使用百分比等相对单位，而不是像素等固定单位，允许内容根据屏幕尺寸调整大小和回流。
- **弹性图像**：弹性图像可在其包含的元素内调整大小，确保在较小的屏幕上不会溢出容器。

## 媒体查询

- **定义**：媒体查询允许开发人员根据设备的特性，主要是视口宽度，应用不同的样式。
- **`all` 媒体类型**：适用于所有设备。 如果未指定媒体类型，则默认为该类型。
- **`print` 媒体类型**：适用于分页材料和在打印预览模式下在屏幕上查看的文档。
- **`screen` 媒体类型**：主要适用于屏幕。
- **`aspect-ratio`**：描述视口宽度和高度之间的比例。
- **`orientation`**：用于指示设备是横向还是纵向。
- **`resolution`**：用于描述输出设备的分辨率，单位是点/英寸（dpi）或点/厘米（dpcm）。
- **`hover`**：用于测试主输入机制是否可以将鼠标悬停在元素上。
- **`prefers-color-scheme`**：用于检测用户是否要求使用浅色或深色主题。
- **媒体查询和逻辑操作符**：`and` 操作符用于组合多个媒体特性，而 `not` 和 `only` 可用于否定或隔离媒体查询。

## 常见媒体断点

- **定义**：媒体断点是网站设计中的特定点，在这些点上，布局和内容会进行调整，以适应不同的屏幕尺寸。 你可以针对手机、平板电脑和桌面屏幕使用一些通用的断点。 但是，试图为不同设备寻找每一种可能的屏幕尺寸并不明智。
- **小型设备（智能手机）**：最大 640px
- **中型设备（平板电脑）**： 641px 至 1024px
- **大型设备（台式机）**： 1025px 或更大

## 移动优先方法

- **定义**：移动优先方法是一种响应式网页设计中的设计理念和开发策略，优先为移动设备创建网站，然后再为更大屏幕设计。

## CSS 自定义属性（CSS 变量）

- **定义**：CSS 自定义属性，也称为 CSS 变量，是由 CSS 作者定义的包含要在整个文档中重用的特定值的实体。 它们是一个强大的特性，可以实现更高效、可维护和灵活的样式表。 自定义属性在创建主题设计时特别有用。  You can define a set of properties for different themes.

## `@property` 规则

- **定义**：`@property` 规则是一个强大的 CSS 特性，它允许开发人员定义自定义属性，从而更好地控制它们的行为，包括它们的动画方式和初始值。

```css
@property --property-name {
  syntax: '<type>';
  inherits: true | false;
  initial-value: <value>;
}
```

- **`--property-name`**：是你定义的自定义属性的名称。 像所有自定义属性一样，它必须以两个连字符开始。 
**`syntax`**: This defines the type of the property, which can be things like `<color>`, `<length>`, `<number>`, `<percentage>`, or more complex types.
- **`inherits`**：指定属性是否应该从父元素继承它的值。
- **`initial-value`**：设置属性的默认值。
- **回退值**：使用自定义属性时，你可以像使用标准自定义属性一样，使用 `var()` 函数提供回退值。

## CSS Grid 基础

- **定义**： CSS Grid 是一种二维布局系统，用于在网页中创建复杂的布局。 Grid 有行和列，行与列之间有间隙。 要定义网格布局，你需要将 `display` 属性设置为 `grid`。
- ** `fr`（分数）单位**: 该单位代表网格容器内空间的一部分。 你可以使用 `fr` 单元创建弹性的网格。
- **在轨道间创建间隙**：在 CSS 网格中，有三种方法可以在轨道间创建间隙。 你可以使用 `column-gap` 属性在列之间创建间隙。 你可以使用 `row-gap` 属性在行与行之间创建间隙。 你也可以使用 `gap` 简写属性同时在行和列之间创建间隙。
- **`repeat()` 函数**: 此函数用于重复轨道列表中的部分。 你可以使用 `repeat()` 函数来代替 `grid-template-columns: 1fr 1fr 1fr;`。
- **显式网格**：你可以使用 `grid-template-columns` 或 `grid-template-rows` 属性指定行数或轨道数。
- **隐式网格**：当项目放置在网格外时，会自动为这些外部元素创建行和列。
- **`minmax()` 函数**：此函数用于设置轨道的最小和最大尺寸。
- **网格自动放置**：浏览器使用自动放置算法自动放置项。`grid-auto-flow` 属性控制此行为，使用诸如 `row`、`column` 或 `dense` 的值来影响放置方向以及是否回填空隙。
- **基于网格线的定位**：所有网格都有网格线。 要指定项目的起始网格线，你可以使用 `grid-column-start` 和 `grid-row-start` 属性。 要指定项目的结束网格线，你可以使用 `grid-column-end` 和 `grid-row-end` 属性。 你也可以使用 `grid-column` 和 `grid-row` 简写属性。
- **`grid-template-areas`**：此属性用于为你将在网格上定位的项提供名称。
- **对齐：** 使用 `justify-items` 和 `align-items` 控制项在其网格区域内的对齐方式。对单个网格项使用 `justify-self` 和 `align-self` 来覆写容器上设置的对齐方式。

## CSS 动画基础

- **定义**：通过 CSS 动画，你无需 JavaScript 或复杂的编程，就能在网页上创建动态的、具有视觉吸引力的效果。 它们提供了一种方法，可在指定的持续时间内在不同风格之间平滑过渡元素。
- **`@keyframes` 规则**：该规则定义了动画的阶段和样式。 它规定了元素在动画过程中的不同时间点应具有的样式。
- **`animation` 属性**：这是用于应用动画的简写属性。
- **`animation-name`**：指定要使用的 `@keyframes` 规则的名称。
- **`animation-duration`**：设置动画完成所需的时间。
- **`animation-timing-function`**：定义动画随时间变化的推进方式（如缓和、线性、缓进缓出）。
- **`animation-delay`**：指定动画开始前的延迟时间。
- **`animation-iteration-count`**：设置动画的重复次数。
- **`animation-direction`**：决定动画是向前播放、向后播放还是交替播放。
- **`animation-fill-mode`**：指定动画前后元素的样式。
- **`animation-play-state`**：允许暂停和恢复动画。

## 可访问性和 `prefers-reduced-motion` 媒体查询

- **`prefers-reduced-motion` 媒体查询**：动画的主要可访问性问题之一是它们可能会给某些用户带来不适，甚至是身体伤害。 患有前庭失调症或运动敏感症的人在屏幕上看到某些类型的运动时可能会感到头晕、恶心或头痛。 通过 `prefers-reduced-motion` 媒体查询，网络开发人员可以在系统级别检测用户是否要求最小化动画或运动效果。
