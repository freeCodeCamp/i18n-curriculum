# --description--

이제 컴퓨터에 Node.js를 설치하여 Node.js 세계에 첫발을 내딛을 준비가 되었습니다. 하지만 Node.js를 설치하기 전에 NVM(Node Version Manager)이라는 도구를 설치하는 것을 권장합니다. 이름에서 알 수 있듯이, NVM은 같은 컴퓨터에서 여러 버전의 Node.js를 관리하는 도구입니다.

처음부터 이 도구를 설정하면 앞으로 여러 프로젝트에서 서로 다른 Node.js 버전을 전환해야 할 때 큰 도움이 됩니다. NVM을 사용하면 이 작업을 매우 쉽게 할 수 있습니다.

macOS/Linux와 Windows에서 NVM을 설치하는 방법과 이를 사용해 Node.js를 설치하는 방법을 살펴보겠습니다.

## macOS/Linux에서 NVM 설치 방법

macOS와 Linux에서는 셸 스크립트를 통해 직접 NVM을 설치할 수 있습니다.

하지만 macOS의 경우, NVM 공식 문서에 “설치 스크립트를 실행하기 전에 Xcode 명령줄 도구를 수동으로 설치해야 하며, 그렇지 않으면 실패할 수 있다”고 명시되어 있습니다.

macOS에서는 터미널에서 다음 명령어를 실행해 Xcode 명령줄 도구를 설치할 수 있습니다:

```bash
xcode-select --install
```

설치가 완료되면 다음 명령어로 버전 번호를 확인하고 설치를 검증하세요:

```bash
xcode-select --version
```

그다음 NVM을 설치할 차례입니다. 다음 명령어 중 하나를 실행해 NVM 설치 스크립트를 다운로드하고 실행할 수 있습니다:

```bash
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.3/install.sh | bash
```

```bash
wget -qO- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.3/install.sh | bash
```

이 글을 읽는 시점에 최신 NVM 버전을 확인하려면 공식 NVM 문서 `https://github.com/nvm-sh/nvm?tab=readme-ov-file#install--update-script`를 참조하세요. 거기에서 최신 버전 번호가 포함된 명령어를 찾을 수 있습니다.

설치가 완료되면 새 명령과 설정을 적용하기 위해 터미널을 재시작해야 할 수도 있습니다.

그다음 다음 명령어를 실행해 NVM이 제대로 설치되었는지 확인하고 현재 버전을 확인하세요:

```bash
nvm -v
```

출력에 NVM 버전이 표시되어야 합니다.

## macOS/Linux에서 NVM으로 Node.js 설치하기

이제 macOS/Linux에 NVM이 설치되었으니 Node.js를 설치할 수 있습니다.

최신 장기 지원(LTS) 버전의 Node.js를 설치하려면 다음 명령어를 실행하세요:

```bash
nvm install --lts
```

여기서 `--lts` 플래그에 주목하세요. 이 플래그는 현재 장기 지원(LTS) 버전을 설치하겠다는 뜻입니다. 이 버전은 보통 안정성, 신뢰성, 보안을 우선시하며 새롭거나 실험적인 기능보다 우선합니다. 철저히 테스트되어 프로덕션 환경에 적합하며, 버그 수정과 보안 패치에 중점을 둔 더 긴 지원 기간이 보장됩니다.

또는 명령어에 버전 번호를 지정해 특정 버전의 Node.js를 설치할 수도 있습니다:

```bash
nvm install <version>
```

예를 들어, Node.js v20을 설치하는 방법은 다음과 같습니다:

```bash
nvm install 20
```

설치가 완료되면 다음 명령어로 해당 버전으로 전환할 수 있습니다:

```bash
nvm use <version>
```

예를 들어, 버전 20으로 전환하는 방법은 다음과 같습니다:

```bash
nvm use 20
```

마지막으로, 다음 명령어로 현재 Node.js 버전을 확인할 수 있습니다:

```bash
node -v
```

## Windows에서 NVM 설치 방법

이제 Windows에서 NVM을 설치하고 사용하는 방법을 살펴보겠습니다.

공식 NVM 버전은 특정 경우에만 Windows를 지원합니다. 만약 Windows Subsystem for Linux(WSL)를 설정했다면, 위 macOS/Linux 가이드를 따라 NVM을 설치하고 사용할 수 있습니다.

하지만 Windows 자체에서는 몇 가지 대안이 있습니다. 그중 하나가 `nvm-windows`로, Windows용 Node.js 버전 관리 유틸리티입니다. 이 도구는 공식 NVM 프로젝트와는 완전히 별개의 프로젝트입니다. 하지만 목적은 동일합니다 — 같은 장치에서 여러 Node.js 버전을 쉽게 관리할 수 있도록 돕는 것입니다.

`nvm-windows`를 설치하기 전에, 공식 저장소에서는 NVM for Windows를 설치하기 전에 기존 Node 설치를 제거할 것을 권장합니다. 만약 처음 Node.js를 설치하는 것이라면 문제없지만, 이미 다른 버전이 설치되어 있다면 먼저 제거하는 것이 좋습니다.

`nvm-windows`를 설치하려면:

- 공식 저장소의 Releases 섹션으로 이동하세요: `https://github.com/coreybutler/nvm-windows/releases`.
- 최신 설치 프로그램을 다운로드하세요.
- 안내에 따라 설치를 진행하세요.

설치가 완료되면 `nvm` 명령어가 바로 사용되지 않을 경우 터미널을 재시작해야 할 수도 있습니다.

다음 명령어를 실행해 설치가 성공했는지 확인하고 현재 `nvm-windows` 버전을 확인할 수 있습니다:

```powershell
nvm -v
```

## Windows에서 NVM으로 Node.js 설치하기

이제 Windows에 `nvm-windows`가 설치되었으니 Node.js를 설치할 준비가 되었습니다.

최신 장기 지원(LTS) 버전의 Node.js를 설치하려면 다음 명령어를 실행하세요:

```powershell
nvm install lts
```

장기 지원(LTS) 버전이 보통 권장되지만, 명령어에 버전 번호를 지정해 다른 버전을 설치할 수도 있습니다:

```powershell
nvm install <version>
```

예를 들어:

```powershell
nvm install 20
```

설치가 완료되면 다음 명령어로 해당 버전의 Node.js로 전환할 수 있습니다:

```powershell
nvm use <version>
```

예를 들어:

```powershell
nvm use 20
```

현재 Node.js 버전을 확인하려면 다음 명령어를 사용하세요:

```bash
node -v
```

## 기본 명령어

이제 NVM과 Node.js를 설치하는 방법을 알았으니, Node.js와 NVM을 사용할 때 유용한 기본 명령어를 살펴보겠습니다.

또한, 이 모든 명령어는 macOS/Linux에 설치된 NVM과 WSL에서는 작동하지만, `nvm-windows`에서는 다르게 작동하거나 전혀 작동하지 않을 수 있습니다. 가능한 경우 `nvm-windows`에 대한 대체 명령어도 알려드리겠습니다.

### Powershell 실행 정책 조정 방법

Windows에서 Powershell과 `nvm-windows`를 사용 중이라면 Powershell 실행 정책을 조정해야 할 수도 있습니다. Powershell의 기본 실행 정책은 `Restricted`로, 스크립트 실행을 막는 보안 조치입니다.

먼저, Powershell 아이콘을 마우스 오른쪽 버튼으로 클릭하고 “관리자 권한으로 실행”을 선택해 Powershell을 관리자 권한으로 실행하세요.

그다음 현재 Powershell 세션의 실행 정책을 확인하려면 다음 명령어를 실행하세요:

```powershell
Get-ExecutionPolicy
```

마지막으로, 로컬 스크립트와 `RemoteSigned` 실행을 허용하는 `npm` 실행 정책으로 설정하려면 다음 명령어를 사용하세요:

```powershell
Set-ExecutionPolicy RemoteSigned
```

위 명령어를 실행한 후에는 관리자 권한 Powershell 창을 닫고, 일반 권한 Powershell 창을 새로 열면 됩니다.

### 설치된 모든 버전 목록 보기

설치된 모든 Node.js 버전을 나열하고 현재 사용 중인 버전을 확인하려면 다음 명령어를 실행하세요:

```bash
nvm ls
```

### 기본 Node.js 버전 설정 (macOS/Linux의 NVM 전용)

이 명령어는 새 터미널 세션에서 사용할 기본 Node.js 버전을 설정합니다:

```bash
nvm alias default <version>
```

예를 들어:

```bash
nvm alias default 22.20.0
```

특정 버전에 더 구체적인 이름을 할당하는 사용자 지정 별칭도 만들 수 있습니다:

```bash
nvm alias <new_name> <version>
```

예를 들어:

```bash
nvm alias new-api 18.20.8
```

별칭을 설정한 후에는 이 새 이름을 사용해 해당 버전으로 전환할 수 있습니다:

```bash
nvm use <new_name>
```

예를 들어:

```bash
nvm use new-api
```

### Node.js 버전 제거하기

Node.js 버전을 제거하려면 다음 명령어를 실행하세요:

```bash
nvm uninstall <version>
```

예를 들어:

```bash
nvm uninstall 20.19.5
```

macOS/Linux용 NVM에서는 `uninstall` 명령어에 메이저 버전 번호인 `22` 또는 `20`을 사용해 버전을 제거할 수 있습니다. 예를 들어:

```bash
nvm uninstall 20
```

만약 `20.14.0`와 `20.12.2` 같은 여러 릴리스가 설치되어 있다면 최신 버전이 먼저 제거됩니다. 이 경우 `nvm uninstall 20` 명령어를 실행하면 `20.14.0`이 먼저 제거됩니다.

### 현재 Node.js 버전 확인하기

현재 Node.js 버전을 확인하려면 다음 명령어를 사용하세요:

```bash
node -v
```

### 현재 npm 버전 확인하기

`npm`는 Node.js용 패키지 관리자입니다. 소프트웨어 패키지를 설치, 게시, 관리하는 데 사용합니다. Node.js를 설치하면 자동으로 함께 설치됩니다.

현재 `npm` 버전을 확인하려면 다음 명령어를 실행하세요:

```bash
npm -v
```

### JavaScript 파일 실행하기

이 명령어는 현재 활성화된 Node.js 런타임에서 지정한 JavaScript 파일을 실행합니다:

```bash
node <file.js>
```

예를 들어:

```bash
node app.js
```

### Node.js 프로젝트 생성하기

이 명령어는 프로젝트 세부 정보와 의존성을 추적하는 `package.json` 파일을 생성해 새 Node.js 프로젝트를 만듭니다.

```bash
npm init
```

`npm init` 명령어를 실행하면 `package.json` 파일 설정을 위한 대화형 명령줄 가이드가 시작됩니다. 프로젝트 이름, 설명, 버전 등을 묻습니다. 기본 `package.json` 파일로 시작하려면 `npm init -y` 또는 `npm init --yes` 명령어를 사용할 수 있습니다.

### 패키지 설치하기

특정 패키지를 설치하려면 다음 명령어를 실행하세요:

```bash
npm install <package>
```

예를 들어:

```bash
npm install express
```

### 의존성 설치하기

프로젝트의 `package.json` 파일에 나열된 모든 의존성을 설치하려면 다음 명령어를 실행하세요:

```bash
npm install
```
