# --description--

Web Storage API 提供瀏覽器內建的機制來儲存鍵值對，讓開發者能夠儲存可跨不同頁面重新載入和工作階段使用的資訊。 

Web Storage API 中的兩個主要組件是 `localStorage` 和 `sessionStorage`。

`localStorage` 是 Web Storage API 的一部分，允許資料即使在瀏覽器視窗關閉或頁面重新整理後仍然持續存在。這些資料會保持可用，直到被應用程式或使用者明確地移除為止。 

`sessionStorage` 是 Web Storage API 的另一個零件，會在頁面工作階段期間儲存資料，這表示只要瀏覽器分頁或視窗開啟，資料就可用。然而，不同於 `localStorage`，當分頁或視窗關閉時，`sessionStorage` 中的資料會被清除。你將在下一課中學習更多關於 `sessionStorage` 的內容。 

`localStorage` 的常見使用案例包括儲存使用者設定，例如主題或語言偏好、跨瀏覽器工作階段記住表單資料，以及快取小量資訊以提升網頁應用程式的效率。

快取是指將經常存取的資料儲存在一個暫時的儲存位置，稱為快取，這樣後續對該資料的請求就能更快速地回應，而不必重新計算或從較慢的資料來源（例如資料庫或外部伺端）取用。

一些常見的 `localStorage` 方法包括 `setItem`、`getItem`、`removeItem` 和 `clear` 方法。 

這裡是一個使用 `setItem()` 方法將鍵值對組存入 `localStorage` 的範例。

```js
localStorage.setItem('username', 'JaneDoe');
```

然後如果我們想從 `localStorage` 取得給定鍵的值，我們可以使用 `getItem()` 方法。

```js
let username = localStorage.getItem('username');
console.log(username); // JaneDoe
```

要使用其鍵從 `localStorage` 移除一個項目， 你可以使用 `removeItem()` 方法。

```js
localStorage.removeItem('username');
```

要清除 `localStorage` 中的所有資料，你可以使用 `clear()` 方法。

```js
localStorage.clear();
```

現在，讓我們來看看一個使用 `localStorage` 來儲存使用者偏好主題的範例。

```js
// Store the user's theme preference
localStorage.setItem('theme', 'dark');

// Retrieve the stored theme preference
const userTheme = localStorage.getItem('theme');
console.log(userTheme); // 'dark'

// Remove the theme preference
localStorage.removeItem('theme');

// Clear all localStorage data
localStorage.clear();
```

在此範例中：

- 我們首先為使用者儲存一個主題選擇（`dark`）。

- 然後我們擷取該主題並將其輸出到主控台。

- 最後，我們示範如何移除特定的條款或清除所有已儲存的資料。

`localStorage` 非常適合用來儲存需要在多個工作階段之間持續存在的小量資料，但重要的是要注意，`localStorage` 不應用來儲存敏感資訊，例如密碼，因為這可能會帶來安全風險。
