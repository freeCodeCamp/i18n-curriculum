# --description--

Un percorso è una stringa che specifica la posizione di un file o di una directory in un file system. Nello sviluppo web, i percorsi permettono agli sviluppatori di collegarsi a risorse come immagini, fogli di stile, script e altre pagine web. Esistono percorsi assoluti e relativi: entrambi sono essenziali per specificare la posizione dei file all’interno di un file system. Vediamo i due tipi per capire quale usare e quando.

Un percorso assoluto è un collegamento completo a una risorsa. Inizia dalla directory radice, include tutte le altre directory e infine il nome del file e l’estensione. La "directory radice" indica la cartella o directory di livello più alto in una gerarchia.

Se stai collegando a una risorsa sulla tua macchina locale, usa un percorso assoluto, che include la posizione completa della directory del file. Ecco come collegarsi al file `about.html` con un percorso assoluto:

```html
<p>
  Read more on the
  <a
    href="/Users/user/Desktop/fCC/script-code/absolute-vs-relative-paths/pages/about.html"
    >About Page</a
    >
</p>
```

Appare così perché partiamo dalla radice e entriamo in una cartella chiamata `Users`, poi in una cartella chiamata `user`, poi in una cartella chiamata `Desktop`, poi in una cartella chiamata `fCC`, poi in una cartella chiamata `script-code`, poi in una cartella chiamata `absolute-vs-relative-paths`, poi in una cartella chiamata `pages` per arrivare infine al file `about.html`.

Un URL assoluto è un indirizzo completo usato per consultare una risorsa. Include il protocollo — che potrebbe essere `http`, `https` e `file` — e il nome di dominio se la risorsa è sul web. Ecco un esempio di URL assoluto che collega al logo di freeCodeCamp:

```html
<a href="https://design-style-guide.freecodecamp.org/img/fcc_secondary_small.svg">
  View fCC Logo
</a>
```

In questo esempio, il protocollo è `https`, il nome di dominio è `design-style-guide.freecodecamp.org` e il nome del file è `fcc_secondary_small.svg`.

Ecco come appare l’URL assoluto nella barra degli indirizzi del browser:

```sh
file:///Users/user/Desktop/fCC/script-code/absolute-vs-relative-paths/pages/about.html
```

L’URL include il protocollo, `file://`. Include anche il percorso, che appare così: `/Users/user/Desktop/fCC/script-code/absolute-vs-relative-paths/pages/`, e rappresenta la serie di cartelle che portano al file. Infine, include anche il `about.html`, che è il nome del file e l’estensione.

Un percorso assoluto mostra la posizione completa di un file all’interno di un file system ed è comunemente usato per risorse su una macchina locale. Un URL assoluto include informazioni di consultazione — come il protocollo e, per le risorse web, il nome di dominio — che indicano al browser come e dove recuperare la risorsa.

Ora vediamo il percorso relativo. Un percorso relativo specifica la posizione di un file rispetto alla directory del file corrente. Non include il protocollo né il nome di dominio, rendendolo più breve e più flessibile per collegamenti interni all’interno dello stesso sito web. Ecco un esempio di collegamento alla pagina `about.html` dalla pagina `contact.html`, entrambe nella stessa cartella:

```html
<p>
  Read more on the
  <a href="about.html">About Page</a>
</p>
```

Immagina quindi di essere sulla pagina `contact.html` e, dato che la pagina `about.html` si trova nello stesso posto, basta indicare il nome del file. Questo è un esempio di uso di un percorso relativo.

Quindi, quale dovresti usare e quando: un percorso assoluto, un URL assoluto o un percorso relativo? Ecco le regole da seguire:

- Usa percorsi assoluti quando vuoi fare riferimento a una risorsa da una posizione fissa, come dalla radice del tuo sito o da una directory nota sulla tua macchina locale.

- Usa URL assoluti quando colleghi a una risorsa ospitata su un sito esterno.

- Usa percorsi relativi quando colleghi a risorse all’interno dello stesso sito web.

- Usa percorsi relativi se vuoi mantenere il codice più pulito e più facile da mantenere durante lo sviluppo.

- Usa percorsi relativi durante i test locali per assicurarti che i collegamenti funzionino senza connessione a internet.
