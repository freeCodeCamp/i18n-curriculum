# --description--

Гілкування програм, тобто таких, що виконують різні дії за певних умов, базується на інструкціях `if`, `else if` та `else` у JavaScript. Умова іноді має вигляд перевірки, чи дорівнює результат певному значенню.

Цю логіку англійською мовою виражають як «if x equals y, then ...», що буквально може бути записано в коді з використанням `=`, або оператора присвоєння. Це призводить до несподіваного потоку керування у вашій програмі.

Як розглядалось у попередніх завданнях, оператор присвоєння (`=`) у JavaScript призначає значення імені змінної. А оператори `==` та `===` перевіряють рівність (потрійний `===` перевіряє сувору рівність, тобто і значення, і тип мають співпадати).

Код нижче призначає `x` значення 2, що оцінюється як `true`. Майже кожне значення саме по собі у JavaScript оцінюється як `true`, за винятком так званих «хибних» значень: `false`, `0`, `""` (порожній рядок), `NaN`, `undefined` та `null`.

```js
let x = 1;
let y = 2;
if (x = y) {

} else {

}
```

У цьому прикладі блок коду всередині інструкції `if` виконається для будь-якого значення `y`, якщо тільки `y` не є хибним. Блок `else`, який ми очікуємо виконати тут, фактично не виконається.

# --instructions--

Виправте умову, щоб програма виконувала правильну гілку, а відповідне значення було призначене `result`.

# --hints--

Ваш код має виправити умову так, щоб вона перевіряла рівність, а не використовувала присвоєння.

```js
assert(result == 'Not equal!');
```

Умова має використовувати або `==`, або `===` для перевірки рівності.

```js
assert(__helpers.removeJSComments(code).match(/x\s*?===?\s*?y/g));
```
