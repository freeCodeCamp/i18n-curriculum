# --description--

SQL означає Structured Query Language. Його розробили у 1970-х роках, спочатку він називався structured English query language (SEQUEL). Згодом цю назву скоротили до SQL.

SQL — це мова програмування, яка використовується для зберігання та керування даними в реляційних базах даних.

Як ви дізналися на попередньому уроці, реляційні бази даних організовують дані у таблиці. Кожна таблиця має набір рядків і стовпців, де представлені атрибути даних та їхні зв’язки. Кожен рядок представляє запис, а кожен стовпець — конкретний атрибут або поле цього запису.

За допомогою SQL ви можете додавати, змінювати, видаляти, знаходити та отримувати дані з реляційної бази даних. Він надає повний набір команд для запитів, фільтрації, сортування та агрегування даних.

SQL допомагає оптимізувати продуктивність бази даних, запитуючи лише необхідні дані, і його легко інтегрувати з багатьма мовами програмування, тож ви можете взаємодіяти з базами даних безпосередньо зі своїх застосунків.

Ця мова також стала галузевим стандартом. Вона забезпечує цілісність даних і включає багато функцій безпеки, таких як підтримка автентифікації користувачів і шифрування даних.

SQL масштабована, портативна і сумісна з широким спектром систем керування базами даних.

SQL базується на командах, які зазвичай називають SQL-інструкціями або SQL-запитами.

Як розробник, ви писатимете ці інструкції, використовуючи конкретні елементи мови SQL або ключові слова. Вони дозволяють виконувати необхідні операції з вашою базою даних.

Перед тим як вводити SQL-команди, потрібно відкрити оболонку psql, щоб безпосередньо взаємодіяти з базою даних PostgreSQL. У командному рядку або терміналі введіть:

```bash
psql -U <username> -d <database_name>
```

Замість `username` введіть своє ім’я користувача, а замість `database_name` — базу даних, до якої хочете підключитися.

Якщо ви ще не створили власну базу даних, PostgreSQL автоматично включає базу даних за замовчуванням під назвою `postgres` під час встановлення. Ви можете використовувати цю базу `postgres`, щоб слідувати за прикладами.

Після підключення ви побачите, що запрошення змінилося на:

```bash
postgres=#
```

Воно показує назву бази даних, до якої ви підключені, і чекає на SQL-команди.

Розглянемо деякі з найфундаментальніших SQL-команд, з якими варто ознайомитися. За конвенцією, SQL-команди закінчуються крапкою з комою (`;`).

Спершу ви можете використати цю команду, щоб створити базу даних з назвою `my_database`:

```sql
CREATE DATABASE my_database;
```

Ви можете ввести `CREATE` з наступним `DATABASE` і назвою бази даних, яку хочете створити. Зверніть увагу, що ключові слова SQL у команді написані великими літерами.

Після створення бази даних ви можете підключитися до неї.

Як саме підключатися до бази даних залежить від середовища та інструменту, який ви використовуєте.

У командному рядку або терміналі можна скористатися методом, показаним раніше. В інтерактивній оболонці psql можна переключатися між базами даних за допомогою команди-шорткату `\c`, після якої йде назва бази даних, наприклад:

```bash
\c my_database
```

Зверніть увагу, що команда `\c` та інші команди-шорткати (ті, що починаються зі зворотної скісної риски) є частиною оболонки psql, а не мови SQL, тому для них не потрібна крапка з комою в кінці.

Після підключення до нової бази даних ви побачите, що запрошення змінилося:

```bash
my_database=#
```

Після створення та підключення до бази даних ви можете створити таблицю за допомогою ключових слів `CREATE TABLE`:

```sql
CREATE TABLE products (
  id SERIAL,
  name VARCHAR(255)
);
```

У команді ви пишете `CREATE TABLE`, а потім назву таблиці, яку хочете створити. У цьому випадку буде створена таблиця з назвою `products`.

Потім у круглих дужках ви вказуєте назви стовпців, які має мати таблиця, і визначаєте тип даних значень, що зберігатимуться в кожному стовпці.

У прикладі таблиця матиме два стовпці: один для ідентифікатора продукту, інший — для назви продукту.

Стандартна конвенція найменування таблиць і стовпців — це snake case, тобто слова пишуться в нижньому регістрі і розділяються підкресленням (`_`). Наприклад: `delivery_orders`

Це кілька основних SQL-команд, які варто знати, щоб почати.

За допомогою SQL ви легко можете запитувати, маніпулювати та аналізувати дані, щоб приймати обґрунтовані рішення та розв’язувати реальні задачі.
