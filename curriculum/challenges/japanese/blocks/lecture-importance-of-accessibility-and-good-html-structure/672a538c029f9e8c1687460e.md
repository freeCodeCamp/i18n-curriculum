# --interactive--

以前に適切な見出しレベル構造について学びました。今回は、良い見出し構造がアクセシビリティにどのように影響するかを学びます。

見出しを正しく使うことで、ユーザーがウェブページをナビゲートし理解するための視覚的な階層が作られます。論理的な見出しの階層を使うことで、スクリーンリーダー利用者がコンテンツの構造を理解し、素早くナビゲートできるようになります。続く内容を正確に表す適切な見出しテキストを作成することで、誰もがサイト上で必要な情報を見つけやすくなります。さらに、適切に作られた見出しはサイトのSEO向上にも役立つ場合があります。

基本的な例を示します：

:::interactive_editor

```html
<h1> Music Store Page</h1>

<h2>Featured Albums</h2>
<p>Check out our featured albums below:</p>
```

:::

見出しはサイトの土台のようなものです。良い土台がなければ、コンテンツのアクセシビリティは損なわれます。

見出しを正しく使うことで、支援技術を使う人に優しいウェブプロジェクトにする方法を見ていきましょう。

`h1`から`h6`までの見出しは、スクリーンリーダー利用者のためのナビゲーション構造を作ります。スクリーンリーダーはページ上のすべての見出しを一覧表示できるため、ユーザーは必要なセクションに直接ジャンプできます。したがって、見出しを正しく配置することは、これらのユーザーが不要なコンテンツを避け、探している情報を素早く見つけるのに重要です。

:::interactive_editor

```html
<h1>I am an h1 heading</h1>
<h2>I am an h2 heading</h2>
<h3>I am an h3 heading</h3>
<h4>I am an h4 heading</h4>
<h5>I am an h5 heading</h5>
<h6>I am an h6 heading</h6>
```

:::

視覚に障害がある方や認知障害のある方は、認知負荷を減らすために情報を素早く簡単に処理する必要があります。

見出しの適切な配置は重要ですが、見出しテキストを明確で説明的にすることも必要です。

見出しを正しく使うための主なポイントは以下の通りです：

- 明確な構成を反映した階層で見出しを使う。例えば、ページタイトルは`h1`、主要なセクションは`h2`見出し、小セクションは`h3`、以下`h6`までといった具合です。
- `h1`から`h3`、または`h2`から`h4`のように見出しレベルを飛ばさない。
- 各見出しの後に続く内容を要約する、明確で説明的なテキストを使う。
- 見出しを単独で使わない。見出しの後には必ず何らかの内容が続く必要がある。
- 見出しのように見せるためにテキストを装飾するのではなく、必要に応じて適切な見出しタグを使う。
- 各ページには、メインのトピックやタイトルを表す単一の`h1`要素を使う。

以下は、ページで見出しを使う基本的なマークアップ例です：

:::interactive_editor

```html
<!-- Page title -->
<h1>What is HTML</h1>

<!-- First section -->
<section>
  <h2>Introduction to HTML</h2>
  <p>
    HTML stands for HyperText Markup Language. It is the standard language for
    creating web pages.
  </p>
</section>
<!-- Second section -->
<section>
  <h2>History of HTML</h2>
  <p>HTML began to take shape in the early 90s</p>
  <h3>Origins</h3>
  <p>
    HTML was created by Tim Berners-Lee in 1991. It has evolved significantly
    over the years.
  </p>
</section>
```

:::
