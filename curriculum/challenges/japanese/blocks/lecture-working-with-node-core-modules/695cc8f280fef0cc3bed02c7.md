# --description--

JavaScriptはもともとウェブブラウザー上で動作するように作られ、その主な役割はウェブページをインタラクティブにすることでした。そのため、初期のJavaScriptは主にフォーム内のテキスト処理やDocument Object Model（DOM）の操作に焦点を当てていました。

しかし、ウェブ上のすべてのデータがテキストというわけではありません。ファイル、画像、動画はバイナリデータであり、異なる処理方法が必要です。ブラウザーでは、これらの種類のデータは通常、JavaScript自体ではなくブラウザーの専門的なコンポーネントによって処理されます。

現代のブラウザーは、これらのタスクを管理するためにレンダリングエンジンとJavaScriptエンジンを使用しています。例えば、ChromeのBlink（V8搭載）、SafariのWebKit（JavaScriptCore搭載）、FirefoxのGecko（SpiderMonkey搭載）などです。

Node.jsはブラウザー上で動作しないため、特にファイルの入出力（I/O）やTCPストリームのようにデータがチャンク単位でやり取りされる場合に、バイナリデータを扱う独自の方法が必要でした。そこで登場するのが`Buffer`モジュールです。

Node.jsの`Buffer`モジュールを使うと、ファイルや画像、ネットワークストリームのようなバイナリデータを直接扱うことができます。これにより、バイナリをメモリ上で直接保存・操作できます。

`fs`モジュールと同様に、`Buffer`はNode.jsのコアモジュールの一つなので、使用前に別途インストールする必要はありません。

使うには、まず分割代入でモジュールをインポートしてください。

```js
const { Buffer } = require("buffer");
```

次に、提供されるメソッドを使って`Buffer`を呼び出します。例えば、`Buffer.from()`は文字列や配列、その他の生データからバッファを作成できます。

文字列と配列から`Buffer`を作成する例は以下の通りです。

```js
// Create a buffer from a string
const myStrBuffer = Buffer.from("freeCodeCamp");
console.log(myStrBuffer); // <Buffer 66 72 65 65 43 6f 64 65 43 61 6d 70>

// Create a buffer from an array of numbers
const myNumBuffer = Buffer.from([
  70, 82, 69, 69, 67, 79, 68, 69, 67, 65, 77, 80,
]);

console.log(myNumBuffer); // <Buffer 46 52 45 45 43 4f 44 45 43 41 4d 50>
```

`Buffer`モジュールの一部のメソッドはインポートしなくても使えますが、他のメソッドは明示的に`Buffer`をインポートしないと利用できません。ですので、プロジェクトで使う場合は常にBufferをインポートすることをお勧めします。

配列のように、個々の`buffer`要素にアクセスすることもできます。

```js
console.log(myNumBuffer[0]); // 70
console.log(myStrBuffer[0]); // 102
```

また、バッファの実際の中身を確認するために`toString()`メソッドを使うこともできます。

```js
console.log(myStrBuffer.toString()); // freeCodeCamp
console.log(myNumBuffer.toString()); // FREECODECAMP
```

`Buffer.alloc()`は指定したサイズ（バイト数）の新しいバッファを作成します。中のすべてのバイトは自動的に`0`で埋められます。

```js
const someBuffer = Buffer.alloc(10);
console.log(someBuffer); // <Buffer 00 00 00 00 00 00 00 00 00 00>
```

`alloc()`メソッドに渡したサイズに基づいて、バッファがゼロで初期化されていることがわかります。

このバッファに`Buffer.write()`メソッドを使って書き込むことができます。

```js
someBuffer.write("Hello fCC");

console.log(someBuffer); // <Buffer 48 65 6c 6c 6f 20 66 43 43 00>
console.log(someBuffer.toString()); // Hello fCC
```

バッファの容量を超えてデータを書き込むと、余分な部分は切り捨てられます。

```js
someBuffer.write("Hello freeCodeCamp");

console.log(someBuffer); // <Buffer 48 65 6c 6c 6f 20 66 72 65 65>
console.log(someBuffer.toString()); // Hello free
```

最後に、特定のエンコーディングで文字列を保存するのに必要なバイト数を表示するために`Buffer.byteLength()`を使うことができます。

```js
console.log(Buffer.byteLength("Hello freeCodeCamp")); // 18
```

その他の`Buffer`メソッドには以下があります：

- `Buffer.isBuffer()`：指定したオブジェクトがバッファかどうかをチェックする
- `Buffer.compare()`：2つのバッファを比較してソート順を返す
- `Buffer.concat()`：複数のバッファを1つに結合する
