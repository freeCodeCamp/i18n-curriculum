# --interactive--

상속은 CSS에서 부모 요소로부터 자식 요소로 스타일이 전달되는 방식을 결정하는 핵심 개념입니다.

현실 세계에서 자식이 부모로부터 특성을 물려받는 것처럼, CSS에서도 특정 속성이 부모 요소로부터 자식 요소에게 상속될 수 있습니다.

이 덕분에 문서 전체에 일관된 스타일을 더 효율적으로 적용할 수 있습니다.

CSS에서는 모든 속성이 기본적으로 상속되는 것은 아닙니다. 예를 들어, `color`, `font-family`, 그리고 `line-height` 같은 속성은 상속됩니다. 즉, 부모 요소에 텍스트 색상을 설정하면, 특별히 재정의하지 않는 한 모든 자식 요소가 그 색상을 상속받습니다.

예를 들어, 부모 `div` 요소에 인라인 스타일로 색상을 설정하고, 자식 `p` 요소가 그 색상을 상속받는 다음 예시를 살펴보세요:

:::interactive_editor

```html
<div style="color: blue;">
  This is the parent element.
  <p>This is the child element inheriting the color.</p>
</div>
```

:::

이 경우, 부모 `div`와 자식 `p` 모두 텍스트가 파란색으로 표시되는데, 이는 색상이 상속되었기 때문입니다.

반면에, `margin`, `padding`, `border`, 그리고 `background` 같은 속성은 기본적으로 상속되지 않습니다. 자식 요소가 이러한 스타일을 상속받게 하려면, 자식 요소에 직접 설정하거나 `inherit` 키워드를 사용해 명시적으로 지정해야 합니다.

`inherit` 키워드는 일반적으로 상속되지 않는 속성도 부모 요소로부터 강제로 상속받게 할 때 사용합니다.

예를 들어, 특정 자식 요소가 부모와 같은 `padding`을 갖게 하려면, 자식 요소에 `padding: inherit`를 설정할 수 있습니다:

:::interactive_editor

```html
<div style="padding: 20px;">
  This is the parent element with padding.
  <p style="padding: inherit;">This is the child element inheriting the padding.</p>
</div>
```

:::

이 경우, 자식 `p` 요소는 부모 `20px` 요소의 `div` 패딩을 상속받습니다.

상속은 스타일시트에서 일관성을 유지하고 중복을 줄이는 데 특히 유용합니다.

여러 요소에 같은 스타일 규칙을 반복해서 작성하는 대신, 부모 요소에 한 번만 정의하면 자식 요소들이 이를 상속받아 CSS를 더 간결하고 관리하기 쉽게 만듭니다.

하지만 상속은 한 방향, 즉 부모에서 자식으로만 작동한다는 점을 기억하세요. 자식 요소에서 스타일을 재정의해도 부모 요소에는 영향을 주지 않습니다.
