# --description--

Анімації можуть значно покращити візуальну привабливість і користувацький досвід на вебсайті. Однак вони також можуть створювати суттєві проблеми з доступністю для певних користувачів. Важливо розуміти ці проблеми та впроваджувати рішення, щоб ваш сайт залишався доступним для всіх.

Однією з основних проблем доступності, пов’язаних з анімаціями, є те, що вони можуть викликати дискомфорт або навіть фізичну шкоду деяким користувачам. Люди з вестибулярними розладами або чутливістю до руху можуть відчувати запаморочення, нудоту або головний біль при впливі певних типів руху на екрані.

Крім того, анімації можуть відволікати користувачів із когнітивними порушеннями або розладами уваги. Особливо проблемними є швидкі мерехтливі або стробоскопічні ефекти. Вони можуть викликати судоми у людей з фоточутливою епілепсією. Загалом уникайте будь-якого вмісту, який мерехтить більше трьох разів на секунду.

Ще одна проблема полягає в тому, що анімації можуть ускладнювати фокусування або читання вмісту для деяких користувачів. Це особливо стосується людей із низьким зором або труднощами читання, які можуть мати проблеми з відстеженням рухомого тексту або змінних макетів.

Щоб вирішити ці проблеми, CSS надає медіазапит `prefers-reduced-motion`. Ця функція дозволяє веброзробникам визначати, чи користувач на системному рівні запросив мінімальні анімації або ефекти руху.

Ось як ви можете використати `prefers-reduced-motion`:

```css
@media (prefers-reduced-motion: reduce) {
  * {
    animation-duration: 0.01ms !important;
    animation-iteration-count: 1 !important;
    transition-duration: 0.01ms !important;
    scroll-behavior: auto !important;
  }
}
```

Цей фрагмент коду CSS ефективно вимикає більшість анімацій і переходів для користувачів, які вказали перевагу скороченого руху. Розглянемо детальніше:

Правило медіазапиту `@media` перевіряє, чи користувач віддає перевагу скороченому руху. Якщо так, застосовуються стилі всередині.

Усередині медіазапиту ми націлюємося на всі елементи (`*`) і переозначаємо властивості анімації та переходів.

Ми встановлюємо `animation-duration` і `transition-duration` на надзвичайно мале значення (`0.01ms`). Це фактично вимикає анімації, але дозволяє їм завершитися, що може бути важливим для певної функціональності.

`animation-iteration-count: 1` гарантує, що будь-які циклічні анімації відтворюються лише один раз.

`scroll-behavior: auto` вимикає ефекти плавного прокручування.

Оголошення `!important` використовується, щоб ці правила мали пріоритет над іншими стилями анімації.

Важливо зазначити, що хоча цей метод ефективно зменшує рух, він є загальним підходом. Для більш точного контролю ви можете визначити конкретні альтернативи зі скороченим рухом для своїх анімацій.

Ось приклад більш цілеспрямованого підходу:

```css
.animated-element {
  transition: transform 0.3s ease-in-out;
}

@media (prefers-reduced-motion: reduce) {
  .animated-element {
    transition: none;
  }
}
```

У цьому випадку ми вимикаємо лише `transition` для конкретного елемента, коли віддається перевага скороченому руху. Це дозволяє надати альтернативний досвід із менш інтенсивним рухом для користувачів, які цього потребують.

Пам’ятайте, що мета не в тому, щоб повністю прибрати весь рух із сайту, а в тому, щоб надати варіанти, які дозволять усім користувачам комфортно взаємодіяти з вашим вмістом. Деякий рух все ще може бути корисним для зручності використання та зворотного зв’язку, навіть для користувачів, які віддають перевагу скороченому руху.

При впровадженні анімацій використовуйте їх обдумано, а не лише для декору. Уникайте великих несподіваних рухів і надавайте можливість призупиняти, зупиняти або приховувати анімації, коли це можливо. Важливо використовувати медіазапит `prefers-reduced-motion`, щоб запропонувати альтернативу з низьким рівнем руху, забезпечуючи доступність і комфорт для всіх користувачів, включно з тими, хто чутливий до руху.

Усвідомлюючи ці проблеми доступності та використовуючи інструменти на кшталт `prefers-reduced-motion`, ви можете створювати захопливі анімовані враження, які будуть інклюзивними та доступними для всіх користувачів.
