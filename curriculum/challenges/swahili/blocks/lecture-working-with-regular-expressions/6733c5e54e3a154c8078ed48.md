# --interactive--

Tujifunze kuhusu vihimarisha katika usemi wa kawaida.

Fikiria hali ambapo unataka kulinganisha msimbo wa utambulisho wenye tarakimu nne. Unajua unaweza kutumia darasa la tabia la `\d`, kwa hivyo unaweza kuandika mara nne. Na ili kuepuka herufi zozote zisizohitajika, utaweka tagi za nanga za mwanzo na mwisho:

```js
const regex = /^\d\d\d\d$/;
```

Na hii inafanya kazi – italinganisha herufi nne za nambari. Lakini badala ya kuandika darasa hilo mara nyingi, unaweza kulipa kihimarisha.

Vihimarisha huainishwa na mabano ya mviringo yenye nambari moja au mbili ndani. Tuchukue kihimarisha katika muundo wetu:

```js
const regex = /^\d{4}$/;
```

Angalia jinsi kihimarisha chetu kinavyokuwa na nambari `4` tu. Sintaksia hii inamaanisha "linganisha tabia iliyotangulia mara nne kabisa". Tuchunguze jinsi inavyofanya kazi:

:::interactive_editor

```js
const regex = /^\d{4}$/;
console.log(regex.test("123")); // false
console.log(regex.test("1234")); // true
console.log(regex.test("12345")); // false
console.log(regex.test("123456")); // false
console.log(regex.test("1234567")); // false
```

:::

Muundo huu unalinganisha mfuatano wenye tarakimu nne tu, kwa sababu tumetumia tagi za nanga na kihimarisha chetu kinaruhusu tarakimu nne tu kabisa. Lakini labda msimbo wa utambulisho unahitaji kuwa na kiwango cha chini cha tarakimu nne.

Ili kuruhusu tarakimu nne au zaidi, ongeza koma baada ya nambari katika kihimarisha chako:

```js
const regex = /^\d{4,}$/;
```

Sasa, sintaksia yetu inaruhusu muundo kulinganisha tarakimu nne au zaidi. Tujaribu:

:::interactive_editor

```js
const regex = /^\d{4,}$/;
console.log(regex.test("123")); // false
console.log(regex.test("1234")); // true
console.log(regex.test("12345")); // true
console.log(regex.test("123456")); // true
console.log(regex.test("1234567")); // true
```

:::

Kitambulisho chenye tarakimu saba ni kirefu sana. Vitambulisho hivi vinapaswa kuwa na kiwango cha juu cha tarakimu 6, na kiwango cha chini cha tarakimu 4. Ili kufanikisha hili, unaweza kuongeza nambari ya pili katika kihimarisha chako baada ya koma:

```js
const regex = /^\d{4,6}$/;
```

Na sasa muundo wetu hauzingatii kitambulisho chenye tarakimu saba, kwa sababu ni zaidi ya kiwango chetu cha juu cha tarakimu sita.

:::interactive_editor

```js
const regex = /^\d{4,6}$/;
console.log(regex.test("123")); // false
console.log(regex.test("1234")); // true
console.log(regex.test("12345")); // true
console.log(regex.test("123456")); // true
console.log(regex.test("1234567")); // false
```

:::

Kumbuka huwezi kutumia sintaksia hii kuweka kiwango cha juu pekee – lazima daima uweke kiwango cha chini. Lakini ukitenga kiwango cha chini kuwa `1`, unaweza kufanikisha matokeo sawa.

Tumepokea mahitaji yaliyosasishwa kutoka kwa watumizi wetu. Vitambulisho sasa vinaweza kuanza kwa hiari na herufi. Tayari tunajua darasa la tabia kwa hili, kwa hivyo tuongeze kwenye usemi wetu wa kawaida:

```js
const regex = /^[a-zA-Z]\d{4,6}$/;
```

Lakini sasa tunalazimisha kuwepo kwa herufi. Tunawezaje kuifanya kuwa hiari?

Unaweza kutumia sintaksia ya kihimarisha yenye `0` kama kiwango cha chini na `1` kama kiwango cha juu:

```js
const regex = /^[a-zA-Z]{0,1}\d{4,6}$/;
```

Lakini kwa kweli kuna kihimarisha kifupi maalum kwa tabia moja hiari – alama ya swali (`?`). Tuchukue kihimarisha chetu na kuibadilisha na alama ya swali:

```js
const regex = /^[a-zA-Z]?\d{4,6}$/;
```

Tunapaswa kuthibitisha matokeo:

:::interactive_editor

```js
const regex = /^[a-zA-Z]?\d{4,6}$/;
console.log(regex.test("123")); // false
console.log(regex.test("a1234")); // true
console.log(regex.test("12345")); // true
console.log(regex.test("az12345")); // false
console.log(regex.test("X123456")); // true
console.log(regex.test("1234567")); // false
```

:::

Muundo wetu sasa unaruhusu herufi moja hiari, ikifuatiwa na tarakimu nne hadi sita.

Kwa bahati mbaya, tumegundua tulisoma mahitaji vibaya. Tunahitaji kuruhusu herufi yoyote idadi yoyote kabla ya nambari. Tunaweza kutumia kihimarisha chenye kiwango cha chini cha `0` na kutokuwa na kiwango cha juu kilichowekwa:

```js
const regex = /^[a-zA-Z]{0,}\d{4,6}$/;
```

Lakini muundo wetu unakuwa mrefu tena. Kwa bahati nzuri, kuna kifupi kingine cha "linganisha tabia iliyotangulia mara sifuri au zaidi" – alama ya nyota (`*`). Tuchukue kihimarisha chetu na kuibadilisha na alama hiyo katika muundo, na tujaribu:

:::interactive_editor

```js
const regex = /^[a-zA-Z]*\d{4,6}$/;
console.log(regex.test("123")); // false
console.log(regex.test("a1234")); // true
console.log(regex.test("12345")); // true
console.log(regex.test("az12345")); // true
console.log(regex.test("X123456")); // true
console.log(regex.test("1234567")); // false
```

:::

Sasa tunalinganisha kwa mafanikio kitambulisho chochote chenye herufi sifuri au zaidi ikifuatiwa na nambari nne hadi sita. Lakini inajitokeza kuwa hii inasababisha mfumo wetu kuanguka – tunapaswa kweli kuhitaji angalau herufi moja.

Tena, tunaweza kutumia kihimarisha chenye kiwango cha chini cha moja na kutokuwa na kiwango cha juu kilichowekwa, au tunaweza kutumia sintaksia nyingine maalum – alama ya kuongeza (`+`):

:::interactive_editor

```js
const regex = /^[a-zA-Z]+\d{4,6}$/;
console.log(regex.test("123")); // false
console.log(regex.test("a1234")); // true
console.log(regex.test("12345")); // false
console.log(regex.test("az12345")); // true
console.log(regex.test("X123456")); // true
console.log(regex.test("1234567")); // false
```

:::

Sasa vitambulisho ambavyo havianzi na angalau herufi moja vinashindwa, bila kujali ni nambari ngapi zipo.

Unaweza kutumia vihimarisha kuboresha kwa kiasi kikubwa ufupi na usomaji wa usemi wako wa kawaida.
