# --interactive--

In JavaScript ist es wichtig, den Unterschied zwischen primitiven und nicht-primitiven Datentypen zu verstehen, um effizienten und fehlerfreien Code zu schreiben.

Diese beiden Kategorien von Datentypen verhalten sich unterschiedlich, wenn es darum geht, wie sie im Speicher abgelegt und in Ihren Programmen verarbeitet werden.

Primitive Datentypen sind die einfachste Form von Daten in JavaScript. Dazu gehören number, bigint, string, boolean, `null`, `undefined` und symbol. Diese Typen werden „primitiv“ genannt, weil sie einzelne Werte darstellen und keine Objekte sind.

Wenn Sie mit primitiven Datentypen arbeiten, arbeiten Sie direkt mit deren Werten. Zum Beispiel wird bei der Erstellung einer Variablen mit einem primitiven Wert dieser Wert direkt in der Variablen gespeichert.

Primitive Werte sind unveränderlich, das heißt, sobald sie erstellt sind, kann ihr Wert nicht mehr geändert werden. Sie können jedoch der Variablen einen neuen Wert zuweisen. Hier ist ein Beispiel für die Arbeit mit primitiven Datentypen:

:::interactive_editor

```js
let num1 = 5;
let num2 = num1;
num1 = 10;

console.log(num2); // 5
```

:::

In diesem Beispiel weisen wir einen primitiven Wert (`5`) von `num1` an `num2` zu. Dadurch wird eine unabhängige Kopie des Werts erstellt. Änderungen an der ursprünglichen Variablen (`num1`) wirken sich daher nicht auf die Kopie (`num2`) aus.

Nicht-primitiven Datentypen sind hingegen komplexer. In JavaScript sind das Objekte, zu denen reguläre Objekte, Arrays und Funktionen gehören. Im Gegensatz zu primitiven Typen können nicht-primitive Typen mehrere Werte als Eigenschaften oder Elemente enthalten.

Wenn Sie eine Variable mit einem nicht-primitiven Wert erstellen, wird in der Variablen tatsächlich eine Referenz auf den Speicherort gespeichert, an dem das Objekt liegt, nicht das Objekt selbst. Das führt zu wichtigen Verhaltensunterschieden. Hier ist ein Beispiel mit nicht-primitiven Typen:

:::interactive_editor

```js
const originalPerson = { name: "John", age: 30 };
const copiedPerson = originalPerson;

originalPerson.age = 31;

console.log(copiedPerson.age); // 31
```

:::

In diesem Beispiel haben wir ein Objekt namens `originalPerson` mit zwei Eigenschaften `name` und `age`. Dann weisen wir das `originalPerson`-Objekt einer Variablen namens `copiedPerson` zu.

Anschließend aktualisieren wir den `age`-Wert für das `originalPerson`-Objekt. Wenn wir die `age`-Eigenschaft des `copiedPerson`-Objekts ausgeben, zeigt sie den aktualisierten Wert an.

Aber warum passiert das? Das liegt daran, dass sowohl `originalPerson` als auch `copiedPerson` auf dasselbe Objekt im Speicher verweisen.

In JavaScript kopieren Sie beim Zuweisen eines Objekts an eine andere Variable die Referenz auf das Objekt, nicht das Objekt selbst. Das nennt man flaches Kopieren per Referenz. Änderungen am Objekt über eine Referenz wirken sich daher auf alle Referenzen dieses Objekts aus.

Wenn Sie weiterhin mit JavaScript arbeiten, werden Sie oft Situationen begegnen, in denen das Verständnis des Unterschieds zwischen primitiven und nicht-primitiven Typen wichtig ist. Es ist ein grundlegendes Konzept, das viele Aspekte der Sprache prägt und entscheidend für effizienten und korrekten Code ist.
