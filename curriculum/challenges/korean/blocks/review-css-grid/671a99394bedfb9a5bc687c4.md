## CSS 그리드 기본

- **정의**: CSS 그리드는 웹 페이지에서 복잡한 레이아웃을 만들기 위해 사용하는 2차원 레이아웃 시스템입니다. 그리드는 행과 열로 구성되며 그 사이에 간격이 있습니다. 그리드 레이아웃을 정의하려면 `display` 속성을 `grid`로 설정하세요.

```css
.container {
  display: grid;
}
```

- **`fr` (분수) 단위**: 이 단위는 그리드 컨테이너 내 공간의 분수를 나타냅니다. 유연한 그리드를 만들 때 `fr` 단위를 사용할 수 있습니다.
- **트랙 사이 간격 만들기**: CSS 그리드에서 트랙 사이 간격을 만드는 방법은 세 가지가 있습니다. 열 사이 간격을 만들려면 `column-gap` 속성을 사용하세요. 행 사이 간격을 만들려면 `row-gap` 속성을 사용하세요. 또는 행과 열 모두 사이 간격을 만들려면 `gap` 단축 속성을 사용할 수 있습니다.
- **`grid-template-columns`**: 그리드 트랙 열의 선 이름과 크기를 설정하는 데 사용합니다.

```css
.container {
  display: grid;
  width: 100%;
  grid-template-columns: 30px 1fr;
}
```

- **`grid-template-rows`**: 그리드 트랙 행의 선 이름과 크기를 설정하는 데 사용합니다.
- **`grid-auto-flow`**: 자동 배치된 항목이 그리드에 어떻게 맞춰지는지 결정합니다.

```css
.container {
  display: grid;
  width: 100%;
  grid-auto-flow: column;
}
```

- **`grid-auto-columns`**: 암시적으로 생성된 열의 크기를 설정하는 데 사용합니다.

```css
.container {
  display: grid;
  width: 100%;
  grid-auto-columns: auto;
}
```

- **`place-items`**: 블록 및 인라인 방향 모두에서 항목을 정렬하는 데 사용합니다.
- **`align-items`**: 그리드 컨테이너 내 항목의 정렬을 설정하는 데 사용합니다.
- **`repeat()` 함수**: 트랙 목록에서 섹션을 반복하는 데 사용합니다. `grid-template-columns: 1fr 1fr 1fr;`를 쓰는 대신 `repeat()` 함수를 사용할 수 있습니다.

```css
.container {
  display: grid;
  grid-template-columns: repeat(3, 1fr);
}
```

- **명시적 그리드**: `grid-template-columns` 또는 `grid-template-rows` 속성을 사용해 선이나 트랙 수를 지정할 수 있습니다.
- **암시적 그리드**: 항목이 그리드 밖에 배치되면, 그 밖의 요소를 위해 행과 열이 자동으로 생성됩니다.
- **`minmax()` 함수**: 트랙의 최솟값과 최댓값 크기를 설정하는 데 사용합니다.

```css
.container {
  display: grid;
  grid-template-columns: repeat(4, 1fr);
  grid-auto-rows: minmax(150px, auto);
}
```

- **선 기반 배치**: 모든 그리드에는 선이 있습니다. 항목이 어느 선에서 시작하는지 지정하려면 `grid-column-start`와 `grid-row-start` 속성을 사용하세요. 항목이 어느 선에서 끝나는지 지정하려면 `grid-column-end`과 `grid-row-end` 속성을 사용하세요. 또는 `grid-column`이나 `grid-row` 단축 속성을 사용할 수도 있습니다.

다음은 `grid-column` 속성을 사용해 요소가 모든 열에 걸쳐 늘어나도록 하는 예시입니다.

```css
.element {
  grid-column: 1 / -1;
}
```

- **`grid-template-areas`**: 그리드에서 위치를 지정할 항목에 이름을 부여하는 데 사용합니다.

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<div class="container">
  <div class="header">Header</div>
  <div class="sidebar">Sidebar</div>
  <div class="main">Main Content</div>
  <div class="footer">Footer</div>
</div>

```

```css
.container {
  display: grid;
  grid-template-columns: 200px 1fr; 
  grid-template-rows: auto 1fr auto; 
  grid-template-areas:
    "header header"
    "sidebar main"
    "footer footer"; 
  gap: 20px; 
}

.header {
  grid-area: header; 
  background-color: #4CAF50;
  padding: 10px;
  color: white;
}

.sidebar {
  grid-area: sidebar;
  background-color: #f4f4f4;
  padding: 10px;
}

.main {
  grid-area: main; 
  background-color: #e0e0e0;
  padding: 10px;
}

.footer {
  grid-area: footer; 
  background-color: #4CAF50;
  padding: 10px;
  color: white;
}
```

:::

## CSS 디버깅

- **개발자 도구(DevTools)**: 개발자 도구를 사용하면 CSS를 실시간으로 검사하고 수정할 수 있습니다. 스타일 창에는 선택한 요소에 적용된 모든 CSS 규칙이 표시되며, 상속된 스타일도 포함됩니다. 개별 속성을 켜고 끌 수 있고, 값을 편집하거나 브라우저에서 직접 새 규칙을 추가할 수도 있습니다. 이러한 즉각적인 피드백은 소스 코드를 수정하지 않고도 다양한 스타일을 실험하는 데 매우 유용합니다.
- **CSS 검증기**: 검증기는 CSS가 공식 명세에 맞는지 확인하고 오류나 경고를 보고합니다. 사용할 수 있는 인기 있는 검증기는 W3C CSS Validator입니다.
- **반응형 디자인 디버깅**: 개발자 도구에는 사이트가 다양한 화면 크기와 기기에서 어떻게 보이는지 시뮬레이션하는 옵션이 있습니다. 이를 통해 브레이크포인트 문제나 뷰포트 크기에 따라 잘 확장되지 않는 스타일을 식별할 수 있습니다.
