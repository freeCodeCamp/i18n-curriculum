# --description--

قبل أن نبدأ، يجب أن تعرف ما هي قاعدة عند في CSS. قواعد عند هي تعليمات تُعطى للمتصفح. يمكنك استخدامها لتعريف جوانب مختلفة من ورقة الأنماط، مثل استعلامات الوسائط، والإطارات المفتاحية، ووجوه الخطوط، والمزيد. ستتعلم المزيد عن مفاهيم مثل استعلامات الوسائط والإطارات المفتاحية في دروس لاحقة. هذه المرة، ستتعلم عن قاعدة عند `@font-face`.

مع `@font-face`، يمكنك تعريف خط مخصص بتحديد ملف الخط، والصيغة، وخصائص مهمة أخرى مثل الوزن والنمط. هذه هي الصيغة النحوية الأساسية. سترى `@font-face` متبوعة بكتلة محاطة بأقواس معقوفة:

```css
@font-face {
  /* Descriptors */
}
```

داخل الأقواس المعقوفة، ستحتاج إلى إضافة محددات لتخصيص وجه الخط الخاص بك. لنستعرض بعض المحددات الأكثر استخدامًا. يحدد محدد font-family الاسم الذي ستستخدمه في جميع أنحاء ورقة الأنماط للإشارة إلى ذلك الخط. على سبيل المثال، لنفترض أنك عرّفت هذه `@font-face rule`. تحتوي على محدد `font-family` وقيمته هي `MyCustomFont`:

```css
@font-face {
  font-family: "MyCustomFont";
}
```

في هذه الحالة، ستحتاج إلى استخدام هذا الاسم في ورقة الأنماط حيثما تريد تعيين عائلة الخط المخصصة لكل العناصر التي يطابقها محدد CSS. على سبيل المثال، ستستخدمه كقيمة لـ `font-family`:

```css
body { 
  font-family: "MyCustomFont"; 
}
```

لكن لكي تكون قاعدة عند `@font-face` صالحة، تحتاج أيضًا إلى تحديد `src`. يحتوي هذا على مراجع لموارد الخط. هو في الأساس قائمة بمراجع خارجية أو أسماء وجوه الخط المثبتة محليًا مفصولة بفواصل. يمكن أن تتضمن أيضًا تلميحات حول صيغة وتقنية موارد الخط.

في الشفرة أدناه، تم تعريف `src`. القيمة هي قائمة من عناوين URL مفصولة بفواصل وموضوعة على أسطر منفصلة لتحسين قابلية القراءة:

```css
@font-face {
  font-family: "MyCustomFont"; 
  src: url("path/to/font.woff2"),
    url("path/to/font.woff"),
    url("path/to/font.otf");
}
```

يمكنك استدعاء الدالة `url()` لتضمين ملف في ورقة الأنماط. في هذه الحالة، نضمّن ملفات الخطوط. ستحتاج إلى كتابة مسار الملف داخل أقواس عادية وعلامات اقتباس، بما في ذلك امتداد الملف. لتحسين قابلية القراءة، يمكنك كتابة كل مورد في سطر مختلف، لكن يجب أن ينتهي الأخير بفاصلة منقوطة.

لكل مورد خط، يمكنك أيضًا تحديد الصيغة. هذا اختياري. هو تلميح للمتصفح حول صيغة الخط. إذا تم حذف الصيغة، سيتم تنزيل المورد وسيتم اكتشاف الصيغة بعد التنزيل. إذا كانت الصيغة غير صالحة، فلن يتم تنزيل المورد.

الصيغ الممكنة للخطوط تشمل `collection`، `embedded-opentype`، `opentype`، `svg`، `truetype`، `woff`، و `woff2`.

إليك مثالًا مع صيغ الخطوط. لاحظ كيف نكتب الصيغة المحددة داخل أقواس عادية وعلامات اقتباس:

```css
@font-face {
  font-family: "MyCustomFont"; 
  src: url("path/to/font.woff2") format("woff2"),
    url("path/to/font.otf") format("opentype"),
    url("path/to/font.woff") format("woff");
}
```

في هذا المثال، يمكنك أن ترى أننا نحدد صيغة WOFF2، وصيغة OpenType، وصيغة WOFF.

woff تعني "صيغة الخط المفتوح للويب". الفرق بين WOFF و WOFF2 هو الخوارزمية المستخدمة لضغط البيانات. OpenType هي صيغة لخطوط الحاسوب القابلة للتوسع طورتها مايكروسوفت وأدوبي وتسمح للمستخدمين بالوصول إلى ميزات إضافية في الخط. تُستخدم على نطاق واسع عبر أنظمة التشغيل الرئيسية.

بالإضافة إلى تحديد الصيغة، يمكنك أيضًا تحديد تقنية مورد الخط. هذا اختياري أيضًا. إليك مثالًا حيث نحدد تقنية المورد الثاني للخط.

```css
@font-face {
  font-family: "MyCustomFont"; 
  src: url("path/to/font.woff2") format("woff2"),
    url("path/to/font.otf") format("opentype") tech(color-COLRv1),
    url("path/to/font.woff") format("woff");
}
```

هذه هي أساسيات قاعدة عند `@font-face`. مع هذه القاعدة، يمكنك تحديد ملف الخط لتعريف خطوط مخصصة لتصاميمك الفريدة.
