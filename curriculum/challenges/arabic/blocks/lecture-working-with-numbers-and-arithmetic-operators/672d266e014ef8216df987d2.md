# --interactive--

نوع `Number` هو واحد من أكثر أنواع البيانات استخدامًا في JavaScript ولغات البرمجة الأخرى. قد تبدو الأرقام بسيطة، لكن هناك الكثير لاستكشافه عندما يتعلق الأمر بالأرقام في JavaScript. لذا، دعنا نلقي نظرة أعمق. في JavaScript، يمثل نوع البيانات `Number` قيمة رقمية.

على عكس العديد من لغات البرمجة الأخرى التي تفصل بين الأعداد الصحيحة والأعداد العشرية في أنواع مختلفة، تستخدم JavaScript نوع `Number` موحدًا واحدًا لتمثيل الأرقام. هذا يعني أنه يمكنك العمل مع الأعداد الصحيحة، والأعداد العشرية، وحتى القيم الرقمية الخاصة كلها تحت مظلة نوع البيانات `Number` نفسها.

إليك مثالًا بسيطًا يوضح أن الأعداد الصحيحة، والأعداد العشرية، والأعداد السالبة كلها من نوع number:

:::interactive_editor

```js
const wholeNumber = 50;
const decimalNumber = 4.5;
const negativeNumber = -7;

console.log(typeof wholeNumber); // number
console.log(typeof decimalNumber); // number
console.log(typeof negativeNumber); // number
```

:::

يتضمن نوع `Number` في JavaScript أنواعًا مختلفة من القيم الرقمية، بدءًا من الأعداد الصحيحة البسيطة والأعداد العشرية إلى الحالات الخاصة مثل `Infinity` و `NaN`، أو "ليس رقمًا". دعنا نوضح الأنواع الرئيسية التي ستواجهها. الأعداد الصحيحة هي أعداد كاملة بدون أي جزء كسري أو عشري. يمكن أن تكون موجبة أو سالبة أو صفرًا. إليك بعض الأمثلة:

:::interactive_editor

```js
const positiveInteger = 100;
const negativeInteger = -25;
const zero = 0;

console.log(typeof positiveInteger); // number
console.log(typeof negativeInteger); // number
console.log(typeof zero); // number
```

:::

الأعداد العشرية هي أعداد تحتوي على نقاط عشرية. غالبًا ما يُشار إليها ببساطة بـ "floats" من قبل مطوّري JavaScript. الأعداد العشرية مفيدة عندما تحتاج إلى مزيد من الدقة، مثل التعامل مع القياسات أو العملات. إليك بعض الأمثلة:

:::interactive_editor

```js
const floatingPointNumber = 4.5;
const anotherFloat = 89.56;
const oneMoreFloat = 16.462;

console.log(typeof floatingPointNumber); // number
console.log(typeof anotherFloat); // number
console.log(typeof oneMoreFloat); // number
```

:::

يمكن لـ JavaScript تمثيل الأرقام التي تتجاوز الحد الأقصى باستخدام `Infinity`. ستواجه هذا عندما تحاول قسمة رقم على صفر أو في حالات نادرة، تتجاوز الحد الأعلى لنوع `Number`. إليك مثالًا:

:::interactive_editor

```js
const infiniteNumber = 1 / 0;
console.log(infiniteNumber); // Infinity
console.log(typeof infiniteNumber); // number
```

:::

أحيانًا في JavaScript، لا تؤدي بعض العمليات الحسابية إلى رقم صالح. على سبيل المثال، إذا حاولت إجراء عملية حسابية على شيء ليس رقمًا، ستحصل على `NaN`، والذي يعني "ليس رقمًا":

:::interactive_editor

```js
const notANumber = 'hello world' / 2;
console.log(notANumber); // NaN
```

:::

من المدهش أن نوع `NaN` هو أيضًا `Number`:

:::interactive_editor

```js
const notANumber = 'hello world' / 2;
console.log(typeof notANumber); // number
```

:::

بجانب النظام العشري القياسي (القاعدة 10)، تدعم JavaScript أيضًا الأرقام بأنظمة قواعد مختلفة مثل الثنائي، والثماني، والسداسي عشر. النظام الثنائي هو نظام قاعدة 2 يستخدم فقط الأرقام 1 و0. النظام الثماني هو نظام قاعدة 8 يستخدم الأرقام من 0 إلى 7 فقط. النظام السداسي عشر هو نظام قاعدة 16 يستخدم الأرقام من 0 إلى 9 والحروف من a إلى f، كما ترى في ألوان CSS السداسية.
