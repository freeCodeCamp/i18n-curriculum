# --description--

关系数据库是一个组织成表格的数据集合。这些表格由行和列组成。

行表现单个记录。例如，如果你正在存储用户数据，每行可以表现一个特定用户。

列表现描述每条记录的属性或字段。表格可能包含用于常见用户属性的列，例如姓名和身份证号码。

关系数据库的一个关键特征是它们允许我们通过共同的属性将表格链接起来，从而连接不同的信息，使我们更容易看到它们之间的关系。

关系数据库需要一个模式。该模式定义了数据库的整体结构，包括其表格、列、数据类型、关系和约束。

关系数据库中的所有表格都有一个主键，它是每行的唯一标识符。它们通过外键实现表格之间的关系。你将在接下来的课程中了解更多关于主键、外键及其工作原理的内容。目前，只需知道它们是跨表格链接记录的基础。

例如，假设你正在为一个运营濒危动物野生保护中心的组织创建一个数据库。

其中一个表格可以保存关于动物的通用信息，你可以称它为 `animals`。每一行可以表现一只动物，而每一列可以表现该动物的一个属性，比如物种、大小、权值、年龄以及其他独特的特征。

为了唯一标识每个动物，你通常会为表格添加一个 `id` 列。由于它对每个动物都是唯一的，因此它将是表格的主密钥。每个动物的 ID 可以用来将该表格中的记录链接到其他表格中的相关记录。

你也可以有一个 `species` 表格来保存关于每个物种的信息，例如它们的栖息地和保护状态。`animals` 表格中的每个动物都可以使用物种 ID 引用该表格中的一个物种。

你也可以创建一个表格来保存动物的兽医记录。这样，中心就能跟踪它们的健康状况、治疗和用药情况。

一旦你有了这个通用的模式，你就可以将这些表格链接到作为动物相关数据中心点的 `animals` 表格。

这样，你不必将关于动物的所有信息保存在同一个表格中。相反，你可以将相关数据存储在不同的表格中，并通过关系将它们链接起来。通过创建这些关系，你可以用单个查询获取所需的特定数据。

查询是从数据库中请求特定数据的操作。在我们的示例上下文中，你可以进行的查询示例包括查找所有需要兽医检查的动物、查找中心中最濒危的物种、查找在圈养中出生的动物数量等。你可以自定义查询以满足你的需求。

我们的示例与野生动物保护相关，但关系数据库在医疗、建筑、商业、游戏、教育、政府、电子商务、社交媒体等多个行业有广泛的应用。

它们提供了多个优势，包括可扩展性。它们可以处理大型数据集，因此非常适合复杂的现实应用。它们还通过主键、外键和数据类型来迫使数据完整性，从而确保数据保持一致且准确。

除了关系数据库，你还可以使用非关系数据库。

关系数据库和非关系数据库之间的主要区别在于数据的存储方式。非关系数据库，也称为 NoSQL 数据库，更加灵活。它们将数据存储在不相连的单个文件中。它们的数据模型也更加灵活。

虽然某些非关系数据库可能具有基本的模式或数据模型，但它通常比关系数据库中定义的模式更不严格。如果有必要，你可以添加、修改或删除数据字段。

使用关系数据库还是非关系数据库的决定取决于多种因素，包括数据的性质和应用的具体需求。你应评估这两种方法之间的权衡，以选择最适合你的应用的方案。
