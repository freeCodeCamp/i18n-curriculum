# --interactive--

Usemi wa kawaida, au regex, ni kipengele kinachoungwa mkono na lugha nyingi tofauti za programu.

Usemi wa kawaida ni sintaksia maalum ya kuunda "muundo", ambao unaweza kisha kutumia kukagua mfuatano wa herufi, kutoa maandishi, na zaidi.

Tuchunguze usemi wa kawaida wa msingi:

```js
const regex = /freeCodeCamp/;
```

Angalia jinsi, katika JavaScript, unavyofafanua usemi wa kawaida kwa kuunda muundo wako kati ya alama mbili za mstari wa mbele (`/`). Jaribu usichanganye hili na maoni, ambapo maandishi huja baada ya alama zote mbili za mstari wa mbele.

Usemi huu wa kawaida utaendana na maandishi `freeCodeCamp`, yenye herufi kubwa `C`, popote pale katika mfuatano wa herufi. Lakini unaweza kufanya hivyo vipi kweli?

Hii inatupeleka kwenye njia yetu ya kwanza – njia ya `test()`. Njia ya `test()` ipo kwenye vitu vya `RegExp`, ambavyo ni vitu vinavyowakilisha usemi wa kawaida (kama ule tulioufafanua tu).

Njia ya `test()` inakubali mfuatano wa herufi, ambao ni mfuatano wa herufi wa kupima kama unalingana na usemi wa kawaida. Kwa mfano, tujaribu kupima mfuatano wa herufi `e`:

:::interactive_editor

```js
const regex = /freeCodeCamp/;
const test = regex.test("e");
console.log(test);
```

:::

Unaweza kuona tumewaita njia ya `test()` kwenye regex yetu mpya, na kupitisha mfuatano wa herufi `e` kama hoja.

Njia ya `test()` ilirudisha `false` kwa sababu mfuatano wa herufi `e` hauendani na muundo `freeCodeCamp`. Ingawa muundo `freeCodeCamp` unajumuisha herufi `e`, hiyo ni mwelekeo tofauti na jinsi usemi wa kawaida unavyofanya kazi.

Tuchunguze mifano michache zaidi. Chukua muda kuzingatia hizi:

:::interactive_editor

```js
const regex = /freeCodeCamp/;
console.log(regex.test("freeCodeCamp"));
console.log(regex.test("freeCodeCamp is great"));
console.log(regex.test("I love freeCodeCamp"));
console.log(regex.test("freecodecamp"));
console.log(regex.test("FREECODECAMP"));
console.log(regex.test("free"));
console.log(regex.test("code"));
console.log(regex.test("camp"));
```

:::

Angalia jinsi mistari mitatu ya kwanza ilirudisha `true`. Mifuatano hii yote ina maandishi, `freeCodeCamp`, kabisa, mahali fulani ndani ya mfuatano wa herufi.

Mistari ya 5 na 6 inarudisha `false`. Ingawa ina maandishi `freecodecamp`, herufi hazilingani. Usemi wa kawaida unazingatia herufi kwa chaguo-msingi.

Mwishowe, ingawa mistari mitatu ya mwisho ina sehemu ya muundo, mfuatano wa herufi haujumuishi muundo mzima.

Njia ya `test()` inarudisha `boolean`, ikionyesha kama mfuatano wa herufi unalingana na usemi wa kawaida kabisa.

Lakini vipi kama ungetaka taarifa zaidi kuliko hiyo? Vizuri, mfuatano wa herufi una njia ya `match()`. Njia hii inakubali usemi wa kawaida, ingawa pia unaweza kupitisha mfuatano wa herufi ambao utaundwa kuwa usemi wa kawaida.

`match()` inarudisha safu ya mechi kwa mfuatano wa herufi. Safu ya mechi ni nini? Vizuri, tuchunguze:

```js
const regex = /freeCodeCamp/;
const match = "freeCodeCamp".match(regex);
console.log(match);
```

Ikiwa tutaendesha hii, tunapata safu kurudi! Lakini ni safu ya ajabu kuangalia. Ina sifa za ziada:

```js
// [
//   'freeCodeCamp',
//   index: 0,
//   input: 'freeCodeCamp',
//   groups: undefined
// ]
```

Sifa ya `groups` itaonyesha vikundi vyovyote vilivyoshikiliwa. Utajifunza maana yake katika funzo lijalo.

Sifa ya `index` inakuambia ni herufi gani ndani ya mfuatano wa herufi mechi ilipatikana. Katika kesi yetu, ilipatikana mwanzoni mwa mfuatano wa herufi.

Sifa ya `input` inakuambia mfuatano wa herufi ambao njia ya `match()` ilitumiwa juu yake.

Tujaribu tena mifano michache, na tuone jinsi matokeo yanavyobadilika:

```js
const regex = /freeCodeCamp/;
console.log("freeCodeCamp".match(regex)); // ['freeCodeCamp', index: 0, input: 'freeCodeCamp', groups: undefined]
console.log("freeCodeCamp is great".match(regex)); // ['freeCodeCamp', index: 0, input: 'freeCodeCamp is great', groups: undefined]
console.log("I love freeCodeCamp".match(regex)); // ['freeCodeCamp', index: 7, input: 'I love freeCodeCamp', groups: undefined]
console.log("freecodecamp".match(regex)); // null
console.log("FREECODECAMP".match(regex)); // null
console.log("free".match(regex)); // null
```

Tujue tayari kwamba mistari mitatu ya kwanza inapaswa kutoa mechi, kwa hivyo tuchunguze hizo:

```js
// ['freeCodeCamp', index: 0, input: 'freeCodeCamp', groups: undefined]
// ['freeCodeCamp', index: 0, input: 'freeCodeCamp is great', groups: undefined]
// ['freeCodeCamp', index: 7, input: 'I love freeCodeCamp', groups: undefined]
```

Je, hiyo ndiyo uliyotarajia? Unaweza kuona jinsi `input` na `index` zimebadilika kulingana na mfuatano wa herufi uliotolewa, na mahali mechi ilipatikana ndani ya mfuatano wa herufi.

Mistari mingine mitatu, ambayo haiko mechi, inarudisha `null` badala ya safu.

Sasa tunapoweza kupima na kupata mechi za mfuatano wa herufi kwa usemi wetu wa kawaida, vipi kama tunataka kubadilisha yaliyomo ndani ya mfuatano wa herufi? Labda mtu ameandika `freecodecamp` kwa herufi ndogo zote, na tunataka kurekebisha herufi moja kwa moja kwao.

Kwanza, tunahitaji kusahihisha usemi wetu wa kawaida ili ulingane na aina ya herufi ndogo ya `freecodecamp`, na kuunda mfuatano wetu wa kupima:

```js
const regex = /freecodecamp/;
const str = "freecodecamp is rly kewl";
```

Sasa, mfuatano wa herufi una njia ya `replace()` ambayo inakubali hoja mbili: usemi wa kawaida wa kulinganisha (au mfuatano wa herufi, ikiwa hutaki vipengele vyote vya regex), na mfuatano wa herufi wa kubadilisha mechi nayo (au kitendakazi cha kuendesha kwa kila mechi).

Kwa hivyo kama tungetaka kubadilisha `freecodecamp` yetu na herufi sahihi:

:::interactive_editor

```js
const regex = /freecodecamp/;
const str = "freecodecamp is rly kewl";
const replaced = str.replace(regex, "freeCodeCamp");
console.log(replaced);
```

:::

Unaweza kuona kwamba `replace()` inarudisha mfuatano wa herufi uliosasishwa na muundo unaolingana `replaced`.

Usemi wa kawaida, na njia zote zinazohusiana nazo, zinaweza kuonekana ngumu na kuzidi uwezo. Lakini utapata nafasi ya kuzichunguza zaidi katika mfululizo huu wa mafundisho.
