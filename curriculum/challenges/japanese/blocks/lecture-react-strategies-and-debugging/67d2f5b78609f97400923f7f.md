# --description--

アプリが成長するにつれて、コンポーネント間でデータがどのように流れるかを管理するのは複雑になってきます。

最初は、Reactの`useState`フックで十分かもしれませんが、機能を追加していくうちに以下のような問題に直面することがあります。

- 必要のないコンポーネントにまでプロップスを渡してしまう、いわゆるプロップドリリング
- アプリの異なる部分でデータの同期を保つこと
- 複数のコンポーネントに同時に影響を与える複雑な更新の処理

これらやその他の課題が発生し、コードベースの保守、デバッグ、テストが難しくなることがあります。そこで状態管理ライブラリの出番です。状態管理ライブラリは、コンポーネントが必要なデータを取得したり更新したりできる集中管理された場所を提供します。

ここでは、いくつかの異なる状態管理の選択肢と、それらを使うべきタイミングを見ていきましょう。

Context APIはReactに組み込まれた状態管理機能で、サードパーティのライブラリを使わずにコンポーネント間で状態を共有できます。`useState`フックのよく知られたアップグレード版であり、テーマの切り替えやユーザー認証の状態のようなケースに最適です。

しかし、Context APIは頻繁な更新にはあまり向いておらず、不必要な再レンダリングを引き起こすことがあるため、eコマースやソーシャルメディアプラットフォームのような複雑な状態管理にはあまり適していません。

以下はContext APIの基本的な使い方を示すカウンターコンポーネントです。

```jsx
import { useState, createContext } from 'react';

const CounterContext = createContext();

const CounterProvider = ({ children }) => {
  const [count, setCount] = useState(0);

  return (
    <CounterContext.Provider value={{ count, setCount }}>
      {children}
    </CounterContext.Provider>
  );
};

export { CounterContext, CounterProvider };
```

このコードは、アプリケーション全体で`count`状態を共有するためのコンテキストとプロバイダーを作成しています。

`CounterProvider`は`useState`フックを使って`count`状態とそのセッターを初期化および管理しています。これらは`Provider`を通じて子コンポーネントに渡されます。

つまり、アプリ全体を`CounterProvider`でラップすると、`count`状態がアプリケーションのどこでも利用可能になります。

アプリケーションを`CounterProvider`でラップする方法は以下の通りです。

```jsx
import { CounterProvider } from './context/CounterContext';

function App() {
  return (
    <CounterProvider>
        {/* App components */}
    </CounterProvider>
  );
}

export default App;
```

そして、`count`状態を使う方法は以下の通りです。

```jsx
import React, { useContext } from 'react';
import { CounterContext } from '../context/CounterContext';

const Counter = () => {
  const { count, setCount } = useContext(CounterContext);

  return (
    <>
      <div style={{ textAlign: 'center' }}>
        <h1>Context API Counter</h1>
        <button style={{ marginRight: '5px' }} onClick={() => setCount(count - 1)}>
          Decrease
        </button>
        <span>{count}</span>
        <button style={{ marginLeft: '5px' }} onClick={() => setCount(count + 1)}>
          Increase
        </button>
      </div>
    </>
  );
};

export default Counter;
```

ご覧の通り、`count`とそのセッター関数`setCount`は`useContext`関数を通じて初期化されています。

現在の`count`状態が表示され、ユーザーが減算ボタンと加算ボタンをクリックすると、それぞれ`setCount`が使われて`count`状態が増減されます。

もう一つの人気のある状態管理ライブラリはReduxで、Reactと一緒に使われる最も人気のある状態管理ライブラリの一つです。長い歴史があり、eコマースやソーシャルメディアプラットフォーム、フォーラムなどの大規模なアプリケーションに最適です。

Reduxは中央のストアと状態更新の厳格な管理を提供することで状態管理を行います。アクション、リデューサー、ミドルウェアという予測可能なパターンを使います。

アクションは、ユーザーの操作によってトリガーされることが多く、アプリケーションからReduxストアへデータを送る情報のペイロードです。

リデューサーは、これらのアクションに応じて状態がどのように変わるかを指定する関数で、状態を不変の方法で更新することを保証します。

一方、ミドルウェアはアクションのディスパッチとリデューサーの間の橋渡しをし、Reduxの機能を拡張（例えばログ記録や非同期処理の対応）できるようにします。コアの流れを変更せずに機能を追加できます。

Reduxに対する最も一般的な不満は、始めるのに必要なボイラープレートコードが多いことです。これに対応して、Reduxチームは「Redux Toolkit」と「RTK Query」を導入し、セットアップの手間をかなり簡単にしました。

通常、アクションとリデューサーは`createSlice()`関数を使って1つのファイルに定義します。ファイル名は`Slice`で終わることが多く、例えば`productSlice`、`userSlice`、`counterSlice`などがあります。

以下は基本を示す`counterSlice`ファイルです。

```jsx
import { createSlice } from '@reduxjs/toolkit';

const counterSlice = createSlice({
  name: 'counter',

  initialState: { count: 0 },

  reducers: {
    increment: (state) => {
      state.count += 1;
    },
    decrement: (state) => {
      state.count -= 1;
    },
  },
});

export const { increment, decrement } = counterSlice.actions;

export default counterSlice.reducer;
```

ここから、アプリ全体を`Provider`でラップし、スライスから`useSelector()`で状態の一部を選択し、`useDispatch()`を使って状態をアクティブにします。

もう一つの選択肢としてZustandがあります。

ZustandはシンプルなAPIを持つ軽量の状態管理ライブラリです。フックベースなのでReduxに比べてボイラープレートが少なく、セットアップが簡単で速いです。

Zustandは小規模から中規模のアプリケーションに最適です。`useStore`フックを使ってコンポーネントやページで直接状態にアクセスします。これにより、アクションやリデューサー、プロバイダーを必要とせずにデータの変更やアクセスが可能です。

以下は別のカウンター機能を実装した`useCounterStore`です。

```jsx
import { create } from 'zustand';

const useCounterStore = create((set) => ({
  count: 0,
  increment: () => set((state) => ({ count: state.count + 1 })),
  decrement: () => set((state) => ({ count: state.count - 1 })),
}));

export default useCounterStore;
```

そして、アプリで状態を初期化し使う方法は以下の通りです。

```jsx
// Import the useCounterStore (it's just a hook)
import useCounterStore from '../useCounterStore';

const Counter = () => {
  // Initialize the states with the useCounterStore hook
  const { count, increment, decrement } = useCounterStore();

  return (
    <>
      <div style={{ textAlign: 'center' }}>
        <h1>Zustand Counter</h1>
        <button style={{ marginRight: '5px' }} onClick={() => decrement()}>
          Decrease
        </button>
        <span>{count}</span>
        <button style={{ marginLeft: '5px' }} onClick={() => increment()}>
          Increase
        </button>
      </div>
    </>
  );
};

export default Counter;
```

フロントエンドのエコシステムは常に進化しており、新しい状態管理ライブラリも定期的に登場しますが、ここで紹介したものは業界で広く使われています。
