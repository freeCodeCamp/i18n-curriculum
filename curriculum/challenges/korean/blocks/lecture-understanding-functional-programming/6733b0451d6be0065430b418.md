# --interactive--

함수형 프로그래밍은 깔끔하고 예측 가능하며 테스트와 디버그가 더 쉬운 코드를 작성하는 것입니다. 오늘 살펴볼 두 가지 핵심 개념은 순수한 함수와 부작용을 피하는 것입니다.

먼저 순수한 함수부터 시작해 보겠습니다.

순수한 함수란 같은 입력이 주어지면 항상 같은 출력을 반환하고, 자신 외부의 어떤 것도 변경하지 않는 함수를 말합니다.

다음은 순수한 함수의 예입니다:

```js
function add(a, b) {
  return a + b;
}
```

이 함수는 같은 입력에 대해 항상 같은 결과를 반환하고 자신 외부를 변경하지 않기 때문에 순수한 함수입니다. `add(2, 3)`를 몇 번 호출해도 항상 `5`를 반환하며, 프로그램의 다른 부분에 영향을 주지 않습니다.

이제 이를 불순수 함수와 비교해 보겠습니다:

```js
let total = 0;
function addToTotal(value) {
  total += value;
  return total;
}
```

이 함수는 자신 외부의 변수(`total`)를 변경하기 때문에 불순수 함수입니다. `addToTotal(5)`의 결과는 함수 호출 전 `total`가 무엇이었는지에 따라 달라집니다. 이런 예측 불가능성은 코드를 이해하고 디버그하기 어렵게 만듭니다.

이제 다음 중요한 개념인 부작용에 대해 알아보겠습니다.

부작용이란 함수가 호출될 때 프로그램 상태에 발생하는 모든 변화를 말합니다. 전역 변수를 변경하거나, 파일에 쓰거나, API를 호출하는 것 등이 이에 해당합니다.

순수한 함수는 부작용이 없지만, 불순수 함수는 부작용이 있습니다.

부작용을 피하는 것은 함수형 프로그래밍의 핵심 원칙입니다. 부작용을 최소화하면 코드가 더 예측 가능해지고 테스트하기 쉬워집니다.

코드를 작성할 때 주요 목표는 함수가 한 가지 일을 아주 잘 하도록 하고, 프로그램의 다른 부분을 변경하지 않도록 하는 것입니다.

다음은 부작용이 있는 함수의 예입니다:

:::interactive_editor

```js
function greet(name) {
  console.log(`Hello, ${name}!`);
}

greet("Alice");
```

:::

이 함수는 콘솔에 로그를 출력하며 외부와 상호작용하기 때문에 부작용이 있습니다.

함수형 프로그래밍에서는 이를 다음과 같이 다시 작성할 수 있습니다:

:::interactive_editor

```js
function greet(name) {
  return `Hello, ${name}!`;
}
console.log(greet("Alice"));
```

:::

이제 `greet`는 문자열을 단순히 반환하는 순수한 함수가 되었고, 부작용(콘솔 로그 출력)은 별도로 처리합니다.

함수형 프로그래밍은 코드를 작고 예측 가능한 조각으로 나누어 순수한 함수로 프로그램을 구성하도록 권장합니다. 이렇게 하면 이해하고 테스트하며 유지보수하기 쉬운 프로그램을 만들 수 있습니다.
