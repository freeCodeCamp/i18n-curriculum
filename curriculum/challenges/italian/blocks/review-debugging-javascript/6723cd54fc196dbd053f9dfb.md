## Tipi comuni di messaggi di errore

- **SyntaxError**: Questi errori si verificano quando scrivi qualcosa in modo errato nel tuo codice, come dimenticare una parentesi o una graffa. Pensalo come un errore grammaticale in una frase.

```js
const arr = ["Beau", "Quincy" "Tom"]
```

- **ReferenceError**: Esistono diversi tipi di ReferenceError, attivati in modi diversi. Il primo tipo di ReferenceError è quello delle variabili non definite. Un altro esempio di ReferenceError è cercare di consultare una variabile, dichiarata con `let` o `const`, prima che sia stata definita.

:::interactive_editor

```js
console.log(num);
const num = 50;
```

:::

- **TypeError**: Questi errori si verificano quando provi a eseguire un'operazione su un tipo sbagliato.

```js
const developerObj = {
  name: "Jessica",
  country: "USA",
  isEmployed: true
};

developerObj.map()
```

- **RangeError**: Questi errori si verificano quando il tuo codice cerca di usare un valore che è fuori dall'intervallo che JavaScript può gestire.

```js
const arr = [];
arr.length = -1; 
```

## L'istruzione `throw` 

- **Definizione**: L'istruzione `throw` in JavaScript serve a lanciare un'eccezione definita dall'utente. Un'eccezione in programmazione è quando succede un evento imprevisto che interrompe il normale flusso del programma.

:::interactive_editor

```js
function validateNumber(input) {
  if (typeof input !== "number") {
    throw new TypeError("Expected a number, but received " + typeof input);
  }
  return input * 2;
}

console.log(validateNumber("Naomi"));   // TypeError: Expected a number, but received string
```

:::

## `try...catch...finally`

- **Definizione**: Il blocco `try` serve a racchiudere il codice che potrebbe lanciare un errore. Funziona come uno spazio sicuro per provare qualcosa che potrebbe fallire. Il blocco `catch` cattura e gestisce gli errori che si verificano nel blocco try. Puoi usare l'oggetto errore dentro catch per esaminare cosa è andato storto. Il blocco `finally` viene eseguito dopo i blocchi try e catch, indipendentemente dal fatto che si sia verificato un errore. Viene comunemente usato per attività di pulizia, come chiudere file o liberare risorse.

:::interactive_editor

```js
function processInput(input) {
  if (typeof input !== "string") {
    throw new TypeError("Input must be a string.");
  }

  return input.toUpperCase();
}

try {
  console.log("Starting to process input...");
  const result1 = processInput("hello");
  console.log("Processed result:", result1);  // HELLO
  const result2 = processInput(9);  // throws TypeError
  console.log("Processed result:", result2);  // not executed
} catch (error) {
  console.error("Error occurred:", error.message);
} 
```

:::

## Tecniche di debugging

- **Istruzione `debugger`**: Questa istruzione ti permette di mettere in pausa il codice su una riga specifica per indagare cosa sta succedendo nel programma.

:::interactive_editor

```js
let firstNumber = 5;
let secondNumber = 10;

debugger; // Code execution pauses here
let sum = firstNumber + secondNumber;

console.log(sum);
```

:::

- **Breakpoint**: I breakpoint ti permettono di mettere in pausa l'esecuzione del codice su una riga specifica a tua scelta. Dopo la pausa, puoi ispezionare variabili, valutare espressioni ed esaminare lo stack delle chiamate.
- **Watchers**: Le espressioni watch ti permettono di monitorare i valori di variabili o espressioni mentre il codice viene eseguito anche se sono fuori dall'ambito corrente.
- **Profiling**: Il profiling ti aiuta a identificare i colli di bottiglia delle prestazioni permettendoti di catturare screenshot e registrare l'uso della CPU, le chiamate alle funzioni e il tempo di esecuzione.
- **`console.dir()`**: Questo metodo serve a mostrare una lista interattiva delle proprietà di un oggetto JavaScript specificato. Produce un elenco gerarchico che può essere espanso per vedere tutte le proprietà annidate.

```js
console.dir(document);
```

- **`console.table()`**: Questo metodo mostra dati tabellari come una tabella nella console. Richiede un argomento obbligatorio, che deve essere un array o un oggetto, e un argomento opzionale per specificare quali proprietà (colonne) mostrare.
