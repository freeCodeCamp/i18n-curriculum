# --description--

`Versions` пакетів npm у розділі залежностей вашого файлу package.json дотримуються того, що називається семантичним версіонуванням (SemVer) — галузевого стандарту для версіонування програмного забезпечення, який покликаний полегшити керування залежностями. Бібліотеки, фреймворки або інші інструменти, опубліковані на npm, повинні використовувати SemVer, щоб чітко повідомляти, яких змін можуть очікувати проєкти при оновленні.

Знання SemVer може бути корисним, коли ви розробляєте програмне забезпечення, що використовує зовнішні залежності (а це майже завжди так). Колись ваше розуміння цих чисел врятує вас від випадкового внесення несумісних змін у проєкт без розуміння, чому те, що працювало вчора, сьогодні раптом не працює. Ось як працює семантичне версіонування згідно з офіційним сайтом:

```json
"package": "MAJOR.MINOR.PATCH"
```

ВЕРСІЯ MAJOR має збільшуватися, коли ви робите несумісні зміни в API. ВЕРСІЯ MINOR має збільшуватися, коли ви додаєте функціональність, сумісну з попередніми версіями. ВЕРСІЯ PATCH має збільшуватися, коли ви робите сумісні з попередніми версіями виправлення помилок. Це означає, що PATCH — це виправлення помилок, а MINOR додає нові функції, але жоден із них не порушує те, що працювало раніше. Нарешті, MAJOR додає зміни, які не працюватимуть із попередніми версіями.

# --instructions--

У розділі залежностей вашого файлу `package.json` змініть версію `@freecodecamp/example` так, щоб вона відповідала MAJOR версії 1, MINOR версії 2 і PATCH версії 13

# --hints--

`"dependencies"` має містити `"@freecodecamp/example"`.

```js
fetch(code + '/_api/package.json')
  .then(response => response.json())
  .then(
    data => {
      assert.property(
        data.dependencies,
        '@freecodecamp/example',
        '"dependencies" does not include "@freecodecamp/example"'
      );
    },
    error => {
      throw new Error(error.message || error.responseText);
    }
  );
```

Версія `"@freecodecamp/example"` має бути `"1.2.13"`.

```js
fetch(code + '/_api/package.json')
  .then(response => response.json())
  .then(
    data => {
      assert.equal(
        data.dependencies['@freecodecamp/example'],
        '1.2.13',
        'Wrong version of "@freecodecamp/example". It should be 1.2.13'
      );
    },
    error => {
      throw new Error(error.message || error.responseText);
    }
  );
```
