# --description--

Middleware no nível da aplicação é um middleware vinculado a uma instância de um app Express. Ele é normalmente usado para lidar com requisições que precisam de processamento em várias rotas. Pode ser aplicado globalmente ou a rotas específicas.

Esse middleware é registrado usando `app.use()` ou métodos de rota específicos como `app.get()`, `app.post()`, etc. Ele é executado para toda requisição recebida, a menos que seja restrito a rotas específicas.

```js
const express = require('express')
const app = express()

// Application-level middleware that logs all incoming requests
app.use((req, res, next) => {
  console.log('Request received at:', new Date())
  next(); // Passes control to the next middleware
})

app.get('/', (req, res) => {
  res.send('Hello World!')
})

app.listen(3000, () => {
  console.log('Server running on http://localhost:3000')
})
```

* `app.use()`: Este é um middleware no nível da aplicação. Ele registra toda requisição feita ao servidor, independentemente da rota.
* O middleware roda para toda requisição recebida, fornecendo ao servidor detalhes como o timestamp atual.

Middleware no nível da aplicação é útil para tarefas que precisam ser aplicadas em muitas ou todas as rotas do app. Isso pode incluir registro de logs, autenticação e tratamento de erros.
