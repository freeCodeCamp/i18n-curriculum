# --description--

Benutzerforschung ist die systematische Untersuchung der Personen, die Ihr Produkt nutzen. Das Ziel ist es, Benutzerbedürfnisse, Verhaltensweisen und Schmerzpunkte zu messen.

Benutzerforschung gibt es in vielen Formen. Eine der gebräuchlichsten ist der Net Promoter Score, oder NPS. Der NPS misst, wie wahrscheinlich es ist, dass Ihre Nutzer Ihr Produkt einem Freund empfehlen. Der NPS wird durch eine Umfrage ermittelt, die an wichtigen Meilensteinen entlang der Nutzerreise angeboten wird, zum Beispiel nach 7 Tagen, 30 Tagen und 90 Tagen. Der NPS wird auf einer Skala von 0 bis 10 gemessen, wobei 9 und 10 einen aktiven Fürsprecher Ihrer Website anzeigen.

Ein weiterer Forschungsansatz ist das Exit-Interview. Dabei handelt es sich um eine Umfrage, die Sie Ihren Nutzern zeigen, wenn sie ein Abonnement kündigen oder ein Konto löschen. Die Daten aus dieser Umfrage können Ihnen Einblicke in die Gründe für die Abwanderung von Nutzern geben, sodass Sie diese adressieren können.

Benutzertesting hingegen bezeichnet die Praxis, Daten von Nutzern zu erfassen, während sie mit Ihrer Anwendung interagieren. Ein Beispiel ist ein Videospiel, das sich in der Beta-Testphase befindet – das ist eine Form des Benutzertestings. Eine Methode, der Sie als Webentwickler begegnen könnten, ist das A/B-Testing. Dabei wird eine neue Funktion an eine zufällig ausgewählte Teilmenge Ihrer Nutzerbasis ausgeliefert. Anschließend können Sie Analyse-Daten nutzen, um zu bestimmen, ob die Funktion vorteilhaft ist.

Schließlich beziehen sich Benutzeranforderungen auf die User Stories oder das Regelwerk, dem Ihre Anwendung folgen muss. Dies kann den Entwicklungsprozess informieren. Benutzeranforderungen können durch Benutzerforschung oder Branchenstandards definiert werden. Sie können sogar durch Input von Stakeholdern festgelegt werden.

Diese Anforderungen können funktional sein, das heißt, sie bestimmen, wie Ihre Anwendung funktionieren soll, oder nicht-funktional, das heißt, sie definieren, wie sich Ihre Anwendung verhalten soll. Benutzeranforderungen sind auch nicht statisch. Die Informationen aus Benutzertests und Benutzerforschung können die Anforderungen beeinflussen, und sie ändern sich, wenn sich Ihre Nutzerbasis verändert.

Das Verständnis dieses Unterschieds ist entscheidend, um die genauesten Daten zu sammeln, damit Sie die beste Erfahrung für Ihre Endnutzer liefern können.
