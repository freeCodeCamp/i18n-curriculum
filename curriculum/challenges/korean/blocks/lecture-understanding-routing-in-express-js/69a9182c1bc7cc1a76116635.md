# --description--

Express에서 **라우트 경로**는 서버가 들어오는 요청에 어떻게 응답할지 결정하는 URL 패턴입니다. 사용자가 특정 URL을 요청하면 Express는 그 URL을 라우트 경로와 매칭하고 해당 핸들러를 실행합니다.

라우트 경로는 `app.get()`, `app.post()` 같은 라우트 메서드 안에 정의됩니다. 이 경로들은 `/home`처럼 정적일 수도 있고, `/user/:id`처럼 URL에서 값을 캡처하는 매개변수를 사용하는 동적 경로일 수도 있습니다.

이번 수업에서는 정적 라우트에 집중하겠습니다. 다음은 그 예입니다:

```javascript
// Static route
app.get("/home", (req, res) => {
  res.send("Welcome to the homepage!");
});
```

이 예제에서 서버는 `/home`에 대한 GET 요청을 듣고 간단한 메시지로 응답합니다.

`app.get()` 메서드는 두 개의 인수를 받습니다:

* 첫 번째 인수인 `"/home"`는 서버가 듣는 정적 라우트 경로(또는 URL)입니다.
    
* 두 번째 인수는 해당 경로에 GET 요청이 들어올 때 실행되는 콜백 함수입니다.
    

콜백 안에서 `res.send()`는 클라이언트에게 응답을 보냅니다. 이 경우, 브라우저에 표시되는 `Welcome to the homepage!` 텍스트를 보냅니다.

다음은 또 다른 정적 라우트 예입니다:

```javascript
// Static route
app.get("/menu/drinks", (req, res) => {
  res.send("Welcome to the drinks menu!");
});
```

이 경우 서버는 `/menu/drinks` URL에 대한 GET 요청을 듣고, 그 라우트가 요청되면 메시지를 보냅니다.

사이트에 공통 경로 아래 여러 관련 페이지가 있을 때 이런 라우트를 사용할 수 있습니다. 예를 들어, 식당 웹사이트는 메뉴 페이지를 다음과 같이 구성할 수 있습니다:

* `/menu/food`
    
* `/menu/drinks`
    
* `/menu/desserts`
    

이 방법은 URL을 명확하고 의미 있게 유지하는 데 도움이 되며, 애플리케이션이 커질수록 라우트를 더 쉽게 정리하고 관리할 수 있게 합니다.

다음 수업에서는 매개변수가 있는 동적 라우트 경로를 살펴보겠습니다.
