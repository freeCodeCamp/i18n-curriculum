# --interactive--

Властивість `z-index` у CSS використовується для керування вертикальним порядком накладання позиційованих елементів, які перекриваються на сторінці. Коли кілька елементів накладаються один на одного, значення `z-index` визначає, який елемент буде зверху. Чим вище значення `z-index`, тим ближче елемент до глядача, а нижчі значення розміщують елемент далі в стеку.

Однак `z-index` працює лише з елементами, які позиційовані, тобто елемент повинен мати значення властивості position, відмінне від `static`, наприклад `relative`, `absolute` або `fixed`. За замовчуванням значення `z-index` — це `auto`, що розміщує елемент у стандартному порядку накладання. Розглянемо приклад із трьома блоками:

```html
<div class="container">
  <div class="box1">Box 1</div>
  <div class="box2">Box 2</div>
  <div class="box3">Box 3</div>
</div>
```

Тепер ми можемо застосувати стилі, щоб позиціонувати блоки так, щоб вони перекривалися один з одним ось так:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<div class="container">
  <div class="box1">Box 1</div>
  <div class="box2">Box 2</div>
  <div class="box3">Box 3</div>
</div>
```

```css
.container {
  position: relative;
  width: 300px;
  height: 300px;
  border: 1px solid black;
}

.box1 {
  position: absolute;
  z-index: 1;
  background: lightcoral;
  top: 20px;
  left: 20px;
  width: 100px;
  height: 100px;
}

.box2 {
  position: absolute;
  z-index: 3;
  background: gold;
  top: 40px;
  left: 40px;
  width: 100px;
  height: 100px;
}

.box3 {
  position: absolute;
  z-index: 2;
  background: lightgreen;
  top: 60px;
  left: 60px;
  width: 100px;
  height: 100px;
}
```

:::

Для контейнера буде встановлено позицію `relative`, а всі блоки, вкладені всередину, отримають позицію `absolute`. Кожен блок має різне значення для `z-index`, що призводить до того, що блоки накладаються один на одного шарами.

Можна уявити `z-index` як спосіб створення шарів на вебсторінці, і елементи з вищими значеннями `z-index` розміщуються над тими, що мають нижчі значення. Це особливо корисно для керування поведінкою перекривних елементів у складних макетах, таких як модальні вікна, спливаючі підказки або підказки.
