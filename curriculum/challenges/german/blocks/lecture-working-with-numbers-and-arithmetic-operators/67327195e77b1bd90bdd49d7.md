# --interactive--

JavaScript ist eine Sprache, in der Dinge manchmal auf überraschende oder sogar merkwürdige Weise funktionieren. Eine solche Überraschung tritt auf, wenn Sie Zahlen und Strings in Berechnungen mischen. Das Erste, was Sie wahrscheinlich versuchen werden, ist, eine Zahl und einen String zu addieren. In JavaScript erfüllt der `+`-Operator eine doppelte Funktion. Er behandelt sowohl die Addition als auch die String-Konkatenation, also das Verbinden von zwei Strings.

Wenn Sie `+` mit einer Zahl und einem String verwenden, entscheidet JavaScript, beide als Strings zu behandeln und sie zusammenzufügen. Wenn Sie den Typ des Ergebnisses mit dem `typeof`-Operator überprüfen, sehen Sie, dass es tatsächlich ein String ist:

:::interactive_editor

```js
const result = 5 + '10';

console.log(result); // 510
console.log(typeof result); // string
```

:::

Was glauben Sie, passiert, wenn Sie die Reihenfolge von `5` und `'10'` vertauschen?

:::interactive_editor

```js
const result = '10' + 5;

console.log(result); // 105
console.log(typeof result); // string
```

:::

Sie sehen, dass dasselbe passiert ist. JavaScript sieht einen String in `'10'` und eine Zahl in `5`, also wandelt es die Zahl in einen String um und fügt sie zusammen. Dies ist als Typumwandlung bekannt. Typumwandlung bedeutet, dass ein Wert von einem Datentyp in einen anderen konvertiert wird.

Es wird interessanter, wenn Sie versuchen, andere arithmetische Operationen wie Subtraktion, Multiplikation oder Division mit einem String und einer Zahl durchzuführen. In diesen Fällen versucht JavaScript, den String vor der Berechnung in eine Zahl umzuwandeln – eine weitere Typumwandlung! So sieht das aus:

:::interactive_editor

```js
const subtractionResult = '10' - 5;
console.log(subtractionResult); // 5
console.log(typeof subtractionResult); // number

const multiplicationResult = '10' * 2;
console.log(multiplicationResult); // 20
console.log(typeof multiplicationResult); // number

const divisionResult = '20' / 2;
console.log(divisionResult); // 10
console.log(typeof divisionResult); // number
```

:::

In den obigen Beispielen wandelt JavaScript erfolgreich den String `'10'` oder `'20'` in eine Zahl um und führt dann die Berechnung durch. Dadurch ergibt `'10' - 5` `5`, `'10' * 2` ergibt `20` und `'20' / 2` führt zu `10`.

Aber was, wenn der String nicht wie eine Zahl aussieht? Sehen wir uns an, was in diesem Fall passiert:

:::interactive_editor

```js
const subtractionResult = 'abc' - 5;
console.log(subtractionResult); // NaN
console.log(typeof subtractionResult); // number

const multiplicationResult = 'abc' * 2;
console.log(multiplicationResult); // NaN
console.log(typeof multiplicationResult); // number

const divisionResult = 'abc' / 2;
console.log(divisionResult); // NaN
console.log(typeof divisionResult); // number
```

:::

In den obigen Beispielen stellt der String `'abc'` keinen gültigen numerischen Wert dar, daher kann JavaScript ihn nicht in eine sinnvolle Zahl umwandeln. Wenn eine solche Umwandlung fehlschlägt, gibt JavaScript `NaN` zurück, was für „Not a Number“ steht. `NaN` ist ein spezieller Wert des Typs `Number`, der eine ungültige oder nicht darstellbare Zahl repräsentiert.

Was passiert, wenn Sie arithmetische Operationen mit einem Boolean (`true` oder `false`) durchführen? Sehen wir uns an, was passiert. JavaScript behandelt Booleans in mathematischen Operationen als Zahlen: `true` wird zu `1` und `false` wird zu `0`.

:::interactive_editor

```js
const result1 = true + 1;
console.log(result1); // 2
console.log(typeof result1); // number

const result2 = false + 1;
console.log(result2); // 1
console.log(typeof result2); // number

const result3 = 'Hello' + true;
console.log(result3); // "Hellotrue"
console.log(typeof result3); // string
```

:::

In den ersten beiden Beispielen ergab `true + 1` `2`, und `false + 1` ergab `1`. Im dritten Beispiel, `'Hello' + true`, wandelte JavaScript `true` in einen String um und fügte ihn mit `'Hello'` zusammen, was zu `'Hellotrue'` führt, einem String.

Für `null` und `undefined` behandelt JavaScript `null` als `0` und `undefined` als `NaN` in mathematischen Operationen:

:::interactive_editor

```js
const result1 = null + 5;
console.log(result1); // 5
console.log(typeof result1); // number

const result2 = undefined + 5;
console.log(result2); // NaN
console.log(typeof result2); // number
```

:::

JavaScript führt oft Typumwandlungen durch und konvertiert automatisch Datentypen wie Zahlen, Strings und Booleans auf manchmal unerwartete Weise. Das Verständnis dieser Umwandlungen ist entscheidend, um Bugs zu vermeiden und robusten Code in Ihren Projekten zu schreiben.
