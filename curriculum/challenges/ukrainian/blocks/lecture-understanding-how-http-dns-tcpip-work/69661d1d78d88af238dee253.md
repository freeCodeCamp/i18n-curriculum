# --description--

Цифровий світ, який ми знаємо сьогодні, базується на ідеї, що пристрої можуть спілкуватися між собою, надсилаючи та отримуючи дані за потреби.

Але чи замислювалися ви коли-небудь, як цей процес працює за лаштунками?

Коли ви переглядаєте інтернет, ви постійно надсилаєте запити та отримуєте дані через мережу. Це можливо завдяки детальному процесу, який базується на стандартних правилах і процедурах, що гарантують правильну відправку та отримання інформації відповідними пристроями.

Цей набір правил і процедур називається **протоколом**.

Протоколи фактично дозволяють двом або більше пристроям спілкуватися між собою, забезпечуючи, що всі вони дотримуються певних стандартів, наприклад формату даних, які будуть обмінюватися.

Протоколи гарантують, що всі пристрої, залучені до процесу обміну даними, зможуть розуміти один одного. У переносному значенні, вони всі «говорять однією мовою».

У своїй основі TCP/IP — це протоколи, які роблять інтернет можливим.

**TCP/IP** — це набір протоколів зв’язку, які визначають, як дані пакуються, надсилаються, маршрутизуються та отримуються через мережу. Вони забезпечують надійну передачу інформації.

Зокрема, **IP**, де IP означає **Internet Protocol**, — це протокол, який гарантує, що дані йдуть від джерела до правильного пункту призначення в мережі.

Чому це важливо? Пам’ятайте, що інтернет — це глобальна мережа взаємопов’язаних пристроїв. Щоб ваш пристрій міг надіслати запит, він спочатку має знати, куди відправити ці дані. Саме для цього існують IP-адреси — основа Internet Protocol (IP). Вони унікально ідентифікують пристрої в мережі, щоб джерело знало, куди відправляти дані.

Сам по собі Internet Protocol є безз’єднувальним, що означає, що кожен блок даних (названий «пакетом») надсилається незалежно до пункту призначення через мережу, без попереднього встановлення з’єднання або гарантії, що він буде доставлений правильно чи у правильному порядку. Якщо пакети дійдуть до місця призначення, джерело про це зовсім не повідомляється.

Ось тут протоколи на кшталт TCP стають дуже важливими.

Протокол **TCP**, відомий як **Transmission Control Protocol**, розбиває дані на невеликі сегменти. Ці базові одиниці даних нумеруються і збираються назад, коли досягають пункту призначення, використовуючи спеціальні алгоритми, які визначають, чи є дані повними і правильними. Ці алгоритми також перевіряють помилки та втрачені пакети, тому можуть попросити джерело надіслати їх повторно, якщо потрібно, забезпечуючи правильне отримання інформації.

Разом Transmission Control Protocol і Internet Protocol встановлюють правила і стандарти, необхідні для надійної передачі даних через інтернет.

Окрім TCP/IP, існує ще один дуже важливий протокол для сучасного інтернету: HTTP.

HTTP працює на рівні застосунку.

**HTTP** означає **HyperText Transfer Protocol**. Він визначає правила і стандарти для комунікації між клієнтом (наприклад, вашим браузером) і сервером.

Коли ви намагаєтеся отримати доступ до вебсайту через браузер, браузер надсилає HTTP-запит до сервера, просячи конкретний ресурс і пов’язані з ним ресурси. Сервер отримує HTTP-запит, виконує свою логіку і повертає HTTP-відповідь клієнту з запитаним вмістом або інформацією про те, чому запит не був успішним.

HTTP-запити та HTTP-відповіді мають дуже схожу структуру, яка включає:

* Лінію запиту (початкова лінія)
    
* Заголовки
    
* Тіло (необов’язкове)

## **HTTP-запити**

**Лінія запиту** HTTP-запиту поділяється на три основні частини:

* Метод, який описує дію, що запитується. Наприклад, `GET` для отримання даних і `POST` для відправлення даних. Інші загальні методи включають `PUT`, `PATCH`, `DELETE` та `HEAD`.
    
* Шлях або URI, який описує конкретний ресурс, що запитується клієнтом.
    
* Версія HTTP-протоколу, яка використовується для процесу комунікації. Наприклад, `HTTP/1.1`.

Ось приклад із методом `GET` на кореневому шляху `/` і версією HTTP 1.1:

`GET / HTTP/1.1`

**Заголовок** містить пари ключ-значення з інформацією про запит.

Наприклад: `Content-Type: application/json`.

Під заголовком є порожній рядок, який вказує, що метадані (додаткова інформація про запит) завершені.

І нарешті, **тіло**. Воно необов’язкове. Його надсилають, коли клієнт має передати інформацію серверу, наприклад, при відправленні нових даних за допомогою методу `POST`.

Ось приклад HTTP-запиту:

```md
GET / HTTP/1.1
Host: example.com
User-Agent: Mozilla/5.0 (Windows NT 10.0; Win64; x64)
Accept: text/html
Connection: keep-alive
```

## **HTTP-відповіді**

Вони мають дуже схожу структуру:

* Лінія статусу
    
* Заголовок
    
* Тіло
    

Спочатку **лінія статусу** містить інформацію про результат запиту. Зазвичай ви знайдете тут:

* Версію HTTP, яка використовується для процесу комунікації. Наприклад, `HTTP/1.1`.
    
* Код статусу, який вказує результат запиту.
    
* Фразу причини, короткий опис коду статусу, зрозумілий людині. Це допомагає зрозуміти, що сталося.
    

Поширені коди статусу включають:

* `200 OK`
    
* `201 CREATED`
    
* `400 BAD REQUEST`
    
* `404 NOT FOUND`
    
* `403 FORBIDDEN`
    
* `500 INTERNAL SERVER ERROR`
    

Існує багато кодів статусу. Вони класифікуються за першою цифрою зліва направо.

* `1xx` для інформаційних відповідей.
    
* `2xx` для успішних відповідей.
    
* `3xx` для повідомлень про перенаправлення.
    
* `4xx` для відповідей з помилками клієнта.
    
* `5xx` для відповідей з помилками сервера.
    

Під лінією статусу HTTP-відповіді знаходиться **заголовок** з метаданими про відповідь. Наприклад, `Content-Type: text/html`. Тут також може бути довжина вмісту, дата, місце розташування та додаткова інформація, яка може бути корисною клієнту.

І нарешті, HTTP-відповідь має **тіло**, яке містить дані, які клієнт запитував, або пояснення того, що сталося, якщо запит не був успішним. Це необов’язково, але включено в більшість відповідей.

Ось приклад HTTP-відповіді:

```md
HTTP/1.1 200 OK
Content-Type: text/html
Content-Length: 105
Connection: keep-alive

<!DOCTYPE html>
<html>
<head><title>Home</title></head>
<body><h1>Welcome!</h1></body>
</html>
```

Тепер, коли ви знаєте про HTTP, можливо, вам цікаво дізнатися більше про HTTPS.

Можна уявити HTTPS як «захищену» версію HTTP, де дані, що надсилаються між браузером і сервером, шифруються для безпеки. Більшість сучасного веб-трафіку використовує HTTPS.

Протоколи на кшталт TCP/IP і HTTP зробили Всесвітню павутину та інтернет загалом незамінним інструментом, на який більшість із нас покладається щодня. Завдяки їм пристрої можуть правильно спілкуватися через мережу і обмінюватися даними по всьому світу.
