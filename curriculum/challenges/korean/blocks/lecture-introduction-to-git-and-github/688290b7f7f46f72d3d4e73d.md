# --description--

커밋을 만들고 이를 자신이 관리하는 저장소에 푸시하는 방법을 배웠습니다. 유지하지 않는 다른 프로젝트에도 같은 방식으로 기여할 수 있지만, 몇 가지 추가 단계가 있습니다.

무엇보다도, 일반적으로 다른 사람의 프로젝트에 "쓰기 접근" 권한이 없습니다. 즉, 커밋을 직접 푸시할 수 없다는 뜻입니다. 대신 작업할 저장소 복사본을 만들어야 합니다. 이 과정을 "포크"라고 합니다.

저장소를 포크하려면, GitHub에서 기여하려는 저장소로 이동하세요. 상단에 "Fork" 버튼이 보일 겁니다. 이 버튼을 클릭하면 포크를 생성하는 UI로 이동합니다.

자신의 저장소를 만들 때와 비슷한 화면이 나타나지만 옵션은 더 적습니다. 소유자를 개인 계정이나 접근 권한이 있는 조직 중에서 선택할 수 있고, 포크의 새 이름과 새 설명을 지정할 수 있습니다.

또한 `main` 브랜치(또는 저장소에 기본 설정된 브랜치)만 복사하는 옵션이 있습니다. 특정 기능 브랜치에 기여할 필요가 없다면 이 옵션을 선택하는 것이 일반적으로 좋습니다. 브랜치에 대해서는 다음 수업에서 더 배우게 됩니다. 지금은 그 박스를 체크 상태로 두세요.

"Create fork"를 클릭하면 새 저장소가 생성되고 자동으로 그 저장소로 이동합니다.

다음으로, SSH나 GitHub CLI를 사용해 자신의 저장소를 복제하듯이 로컬 컴퓨터에 복제하세요.

이제 로컬 복제본에 설정된 원격 저장소를 살펴보겠습니다. 로컬 저장소 디렉터리에서 터미널을 열고 `git remote -v` 명령어를 실행해 전체 목록을 확인하세요. 대략 다음과 비슷할 겁니다:

```sh
origin git@github.com:your-username/your-repo-name.git (fetch)
origin git@github.com:your-username/your-repo-name.git (pull)
```

GitHub에서 저장소를 복제하면 Git이 자동으로 포크를 `origin` 원격 저장소로 설정합니다. 하지만 포크에서 작업할 때는 원본 저장소의 변경 사항을 가져와 자신의 복사본을 업데이트해야 할 때가 많습니다.

이를 위해 원본 저장소를 또 다른 원격 저장소로 추가해 보겠습니다. 포크한 저장소 페이지로 돌아가서 "Code" 버튼을 클릭하고 SSH URL을 복사하세요. 그런 다음 다음 명령어로 새로운 upstream 원격 저장소를 만드세요:

```sh
git remote add upstream git@github.com:original-repo-username/repo-name.git
```

`upstream`는 원격 저장소 이름입니다. 다른 이름을 써도 되지만 `upstream`가 관례입니다. 원격 저장소 목록을 다시 보면 새로 추가한 upstream 원격 저장소가 보일 겁니다:

```sh
origin git@github.com:your-username/your-repo-name.git (fetch)
origin git@github.com:your-username/your-repo-name.git (pull)
upstream git@github.com:original-repo-username/your-repo-name.git (fetch)
upstream git@github.com:original-repo-username/your-repo-name.git (pull)
```

`upstream` 원격 저장소를 통해 `git fetch upstream`나 `git merge upstream/main` 같은 작업을 하여 원본 프로젝트의 최신 변경 사항을 로컬 포크에 가져올 수 있습니다.

포크도 여전히 저장소이며, 커밋을 만들고 변경 사항을 푸시하는 방법은 같습니다. 하지만 그 전에 다른 사람의 프로젝트에 변경 사항을 만들 때 따라야 할 특정 전략이 있다는 점을 이해해야 합니다. 이러한 전략은 앞으로 수업에서 더 배우게 됩니다.

소스 코드를 공개하고, 코드를 수정할 수 있게 하며, 코드 기여를 받는 프로젝트를 "오픈 소스" 프로젝트라고 합니다. 오픈 소스에는 커뮤니티가 버그를 찾아 고치는 데 도움을 주고, 사람들이 기여한 내용을 통해 배우며, 개발자 커뮤니티를 형성해 네트워크로 활용할 수 있다는 많은 이점이 있습니다.

다른 사람의 프로젝트에 변경 사항을 만들면 "오픈 소스 기여자"가 됩니다. 오픈 소스 프로젝트에 기여하는 것은 전문 개발자 팀에서 일하는 경험을 미리 체험하는 좋은 기회가 될 수 있습니다. 여기서 배우는 전략 중 일부는 그런 경험에 도움이 되고 첫 유급 역할에도 이어질 겁니다.

기여 방법에 대해 궁금한 점이 있으면 대부분의 오픈 소스 프로젝트는 필요한 모든 정보를 담은 기여 가이드라인을 제공합니다. 그래도 확실하지 않으면 저장소에 이슈를 열어 질문하거나, Discord 같은 플랫폼에 커뮤니티 공간이 있다면 가입해 질문하세요.
