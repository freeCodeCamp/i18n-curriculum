# --interactive--

アンカー要素、つまりリンクに `target` 属性を見たことがあるかもしれません。この重要な属性は、ブラウザーにアンカー要素のURLをどこで開くかを指示します。

インタラクティブエディターを有効にして、リンクをクリックすると、新しいブラウザータブでfreeCodeCampのホームページに移動します。

:::interactive_editor

```html
<a href="https://freecodecamp.org" target="_blank">Visit freeCodeCamp</a>
```

:::

この属性には4つの重要な値があります。各値の前にはアンダースコアが付いていることに注意してください。

最初の値は `_self` で、これはデフォルトの値です。リンクを現在の閲覧コンテキストで開きます。ほとんどの場合、これは現在のタブまたはウィンドウです。

2番目の値は `_blank` で、リンクを新しい閲覧コンテキストで開きます。通常は新しいタブで開きますが、一部のユーザーはブラウザーの設定で新しいウィンドウで開くようにしている場合もあります。

3番目の値は `_parent` で、リンクを現在のコンテキストの親で開きます。例えば、あなたのウェブサイトに `iframe` があり、その `_parent` に `iframe` の値がある場合、そのリンクは埋め込みフレームではなくウェブサイトのタブやウィンドウで開きます。

4番目の値は `_top` で、リンクを最上位の閲覧コンテキストで開きます。つまり「親の親」のようなものです。これは `_parent` に似ていますが、ネストされた埋め込みフレームでもリンクは常にブラウザーの完全なタブやウィンドウで開きます。

5番目の値として `_unfencedTop` がありますが、これは現在実験的なFencedFrame APIで使われています。このレッスンの時点では、まだ使う必要はほとんどないでしょう。

ユーザーがどこに移動するかを制御するために適切な `target` の値を選ぶことは、ウェブサイトを作る際に重要なポイントです。
