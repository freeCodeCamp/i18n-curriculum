# --description--

في الدروس السابقة، تعلمت أن React هو تطبيق صفحة واحدة. تطبيقات الصفحة الواحدة هي تطبيقات تحتوي على ملف HTML واحد وتستخدم JavaScript لتحديث أي محتوى على الصفحة بشكل ديناميكي.

فماذا يحدث عندما تحتاج إلى إضافة "صفحات" متعددة إلى تطبيق React الخاص بك؟ كيف تتنقل بين هذه العروض المختلفة؟

حسنًا، هنا يأتي دور React Router.

React Router هو مكتبة طرف ثالث تتيح لك إضافة التوجيه إلى تطبيقات React الخاصة بك. للبدء، ستحتاج إلى تثبيت React Router في مشروع React موجود كما يلي:

```sh
npm i react-router
```

إذا فتحت ملف `package.json`، سترى أن `react-router` أُضيف إلى قائمة التبعيات:

```json
"dependencies": {
  "react": "^18.3.1",
  "react-dom": "^18.3.1",
  "react-router": "^7.2.0"
}
```

ثم داخل ملف `main.jsx` أو `index.jsx`، ستحتاج إلى استيراد `BrowserRouter` وعرض `BrowserRouter` حول مكوِّن `App` الخاص بك:

```jsx
import { StrictMode } from "react";
import { createRoot } from "react-dom/client";
import { BrowserRouter } from "react-router";
import App from "./App.jsx";

import "./index.css";

createRoot(document.getElementById("root")).render(
  <StrictMode>
    <BrowserRouter>
      <App />
    </BrowserRouter>
  </StrictMode>
);
```

لتمكين المسارات في تطبيقك، ستحتاج إلى تحديث جملة `import` لتشمل مكوِّنات `Routes` و `Route` كما يلي:

```js
import { BrowserRouter, Routes, Route } from "react-router";
```

ثم داخل `BrowserRouter`، أضف مكوِّنات `Routes` و `Route`:

```jsx
createRoot(document.getElementById("root")).render(
  <StrictMode>
    <BrowserRouter>
      <Routes>
        <Route path="/" element={<App />} />
      </Routes>
    </BrowserRouter>
  </StrictMode>
);
```

تُستخدم الخاصيتان `path` و `element` لربط عنوان URL بمكوِّنات واجهة المستخدم معًا. في هذه الحالة، نُعد مسارًا للصفحة الرئيسية يشير إلى مكوِّن `App`.

من الشائع في التطبيقات الأكبر وجود عروض ومسارات متعددة مُعدة بهذه الطريقة:

```jsx
<Routes>
  <Route index element={<Home />} />
  <Route path="about" element={<About />} />

  <Route path="products">
    <Route index element={<ProductsHome />} />
    <Route path=":category" element={<Category />} />
    <Route path=":category/:productId" element={<ProductDetail />} />
    <Route path="trending" element={<Trending />} />
  </Route>
</Routes>
```

الخاصية `index` في هذه الأمثلة تمثل المسار الافتراضي لجزء معين من المسار. لذا، سيُعرض مكوِّن `Home` عند المسار الجذري (`/`) بينما سيُعرض مكوِّن `ProductsHome` عند مسار `products`.

قد تكون لاحظت أيضًا أننا نُسكن بعض المسارات داخل مسار آخر بهذه الطريقة:

```jsx
<Route path="products">
  <Route path="trending" element={<Trending />} />
</Route>
```

هذا يعني أن مسار المسار الفرعي سيُضاف إلى مسار المسار الرئيسي. لذا في هذا المثال، سيكون `path` للمنتجات الرائجة هو `products/trending`.

إذا بدأ `path` بنقطتين رأسيتين (`:`)، فهذا يمثل جزءًا ديناميكيًا في المسار:

```jsx
<Route path=":category" element={<Category />} />
```

في هذا المثال لدينا جزء ديناميكي يسمى `category`. عندما ينتقل المستخدم إلى عنوان URL مثل `products/brass-instruments`، سيتغير العرض إلى مكوِّن `Category` ويمكنك جلب البيانات المناسبة ديناميكيًا بناءً على الجزء.

يمكنك الوصول إلى قيمة الجزء الديناميكي باستخدام الخطاف `useParams` داخل المكوِّن الفرعي كما يلي:

```jsx
import { useParams } from "react-router";

export default function Category() {
  let params = useParams();
   {/* Accessing the category param: params.category */}
   {/* rest of code goes here */}
}
```

المسارات الديناميكية مفيدة لأنها تتيح لك إنشاء مكوِّنات مرنة وقابلة لإعادة الاستخدام يمكنها عرض محتوى مختلف بناءً على المعلمات في عنوان URL. بدلًا من تعريف قائمة ثابتة من المسارات لكل مسار ممكن، يمكنك استخدام الأجزاء الديناميكية لعرض محتوى متنوع بناءً على ما طلبه المستخدم.
