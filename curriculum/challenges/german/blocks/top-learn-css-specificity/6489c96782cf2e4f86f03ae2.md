# --description--

Eine CSS-Deklaration, die spezifischer ist, hat Vorrang vor weniger spezifischen. Inline-Stile, die Sie in einer vorherigen Lektion behandelt haben, besitzen die höchste Spezifität im Vergleich zu Selektoren, während jeder Selektortyp seine eigene Spezifitätsebene hat, die dazu beiträgt, wie spezifisch eine Deklaration ist. Andere Selektoren tragen ebenfalls zur Spezifität bei, aber Sie konzentrieren sich nur auf die, die Sie bisher gelernt haben:

1. ID-Selektoren (am spezifischsten)
2. Klassen-Selektoren
3. Typselektoren

Die Spezifität wird nur berücksichtigt, wenn ein Element mehrere widersprüchliche Deklarationen hat, die es ansprechen – ähnlich wie ein Entscheidungsfaktor. Ein ID-Selektor schlägt immer eine beliebige Anzahl von Klassen-Selektoren, ein Klassen-Selektor schlägt immer eine beliebige Anzahl von Typselektoren, und ein Typselektor schlägt immer eine beliebige Anzahl von weniger spezifischen Selektoren. Wenn keine Deklaration einen Selektor mit höherer Spezifität hat, schlägt eine größere Anzahl desselben Selektors eine kleinere Anzahl desselben Selektors.

# --questions--

## --text--

Welche der folgenden Reihenfolgen ist die korrekte Spezifitätsreihenfolge für CSS-Selektoren, vom spezifischsten bis zum am wenigsten spezifischen?

## --answers--

Typselektoren, Klassen-Selektoren, ID-Selektoren

---

ID-Selektoren, Klassen-Selektoren, Typselektoren

---

Klassen-Selektoren, Typselektoren, ID-Selektoren

---

ID-Selektoren, Typselektoren, Klassen-Selektoren
