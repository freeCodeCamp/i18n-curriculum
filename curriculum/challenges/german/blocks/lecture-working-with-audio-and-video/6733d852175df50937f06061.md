# --description--

Anstatt Audio und Video abzuspielen, möchten Sie manchmal Audio oder Video aufnehmen. Die Media Capture and Streams API, oder MediaStream API, ermöglicht Ihnen dies.

Um die API zu verwenden, müssen Sie jedoch das `MediaStream`-Objekt erstellen. Sie könnten dies mit dem Konstruktor tun, aber es wäre nicht an die Hardware des Benutzers gebunden. Stattdessen hat die `mediaDevices`-Eigenschaft des globalen `navigator`-Objekts eine `getUserMedia()`-Methode, die Sie verwenden können.

Diese Methode akzeptiert ein einzelnes `constraints`-Objekt, das den Typ der Medien definiert, die Sie empfangen möchten. Dieses Objekt hat eine `audio`- und `video`-Eigenschaft, die Audio- und Videostreams widerspiegeln. Diese Eigenschaften können `false` sein, wenn Sie diesen Streamtyp nicht empfangen möchten, `true`, wenn Sie ihn empfangen möchten, oder Objekte, die zusätzliche Einschränkungen definieren.

Zum Beispiel können Sie eine bestimmte Auflösung für die Videoausgabe verlangen:

```js
window.navigator.mediaDevices.getUserMedia({
  audio: true,
  video: {
    width: {
      min: 1280,
      ideal: 1920,
      max: 3840
    },
    height: {
      min: 720,
      ideal: 1080,
      max: 2160
    }
  }
});
```

Dieses Einschränkungsobjekt gibt minimale und maximale Auflösungen für den Videostream an. Die `ideal`-Eigenschaft gibt die Auflösung an, die Sie am liebsten hätten – und der Stream liefert die Auflösung, die Ihrer Idealvorstellung am nächsten kommt.

Sobald Sie Ihren MediaStream erstellt haben (vorausgesetzt, der Benutzer genehmigt die automatische Anfrage zum Zugriff auf seine Hardware), können Sie die Stream-Daten nach Bedarf verwenden.

Beachten Sie, dass `getUserMedia()` ein Promise zurückgibt, was bedeutet, dass Sie entweder eine Callback-Funktion benötigen, um den Stream zu verarbeiten, oder die async/await-Syntax verwenden müssen. Sie werden in zukünftigen Lektionen mehr über Promises und asynchrone Programmierung lernen.

Hier ist ein einfaches Beispiel, das den Webcam-Stream des Benutzers auf der Seite rendert.

```js
const video = document.querySelector("video");
const stream =
  await window.navigator.mediaDevices.getUserMedia({ video: true });
video.srcObject = stream;
await video.play();
```

Es ist erwähnenswert, dass diese API keinen Zugriff auf die Bildschirmaufnahme bietet. Über diese API lernen Sie in der nächsten Lektion.
