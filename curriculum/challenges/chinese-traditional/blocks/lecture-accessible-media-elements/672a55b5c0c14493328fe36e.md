# --description--

替代文字，通常縮寫為 `alt` 文字，是對影像的簡短文字描述。它為無法看到影像的使用者提供重要的資訊，例如使用螢幕閱讀器和其他輔助技術的人。

這對於讓網站對視覺障礙者可存取至關重要。

替代文字也被搜尋引擎用來理解影像。有些瀏覽器在影像未正確載入時也可能顯示它。當影像檔案遺失或使用者有連線問題時，可能會發生這種情況。因此，它對許多不同的用途都有幫助。

視覺障礙者無法知道影像中是什麼，除非在 alt text 中有描述。對於一張小狗影像來說，糟糕的 alt text 範例會是「A cute puppy」。

<img src="https://cdn.freecodecamp.org/curriculum/lecture-transcripts/puppy.png" alt="A cute puppy." />

這段文字不足以具體傳達影像的重要細節，例如小狗長什麼樣子？小狗在哪裡？小狗周圍有沒有任何重要的物件？

讓我們改進這個。一個好的替代文字範例是：「一隻戴著橘色項圈的黑白小狗趴在沙地上，側頭望向一旁。牠的前爪附近放著一顆鮮橘色的球。」

在這裡你可以看到使用 HTML 的影像元素、`alt` 屬性，以及更詳細的描述：

```html
<img src="puppy.png" alt="A black and white puppy with an orange collar lies on its belly in the sand, looking off to the side. A bright orange ball rests near its front paws." />
```

重要的是要注意，為影像撰寫替代文字沒有唯一正確的方法。你在描述中包含的內容將取決於圖片所使用的背景關係。舉例來說，如果小狗的圖片出現在關於狗品種的網站上，你可能會想要包含更多關於小狗實體描述的細節，並且可能忽略小狗在海灘上玩橘色球的事實。最終，你為影像使用的 `alt` 文字應該反映出將影像包含在頁面上的主要目的，而你提供的資訊應該讓無法看到影像的人獲得理解該目的所需的必要細節。

這裡有另一個美麗熱帶度假村的範例。讓我們來描述它。

<img src="https://cdn.freecodecamp.org/curriculum/lecture-transcripts/resort.jpg" alt="Resort.">

這張影像的壞 `alt` 文字範例是「Resort」。

這太短，且沒有提供足夠的影像資訊。為了改善這個描述，你可以將它擴充為包含影像中最重要的元素：

熱帶度假村，設有被棕櫚樹和平房環繞的游泳池。

你可以這樣使用 HTML 中的 alt 屬性：

```html
<img src="resort.png" alt="Tropical resort featuring a swimming pool surrounded by palm trees and bungalows." />
```

既然你知道什麼造就好的和壞的 `alt` 文字，讓我們來看看一些最佳實踐。

- 你應該嘗試保持 `alt` 文字簡短。它應該足夠詳細以理解影像，但又不應過長以致於造成混淆。
- 你不應該嘗試描述每一個細節。請專注於影像中最重要的部分。
- 通常，你不需要以「影像的」或「圖片的」開始。你可以直接開始描述。
- 此外，如果影像周圍有類似的文字，你不需要再重複撰寫。
- 通常建議以句號結束替代文字以保持一致性。
- 如果影像是連結到另一個頁面，`alt` 文字應該描述使用者點擊後會發生什麼，而不是描述影像本身。

例如，如果你的網站有一個右箭頭圖示，會帶使用者到下一頁，請不要只寫「right arrow」這樣的替代文字，就像這個範例中，你可以看到帶有這個描述的 alt 屬性：

```html
<a href="about.html">
  <img src="arrow-right.png" alt="Right arrow." />
</a>
```

你應該改寫成類似這樣，其中 `alt` 文字描述使用者點擊影像後會發生什麼事。他們將會前往下一頁。

```html
<a href="about.html">
  <img src="arrow-right.png" alt="Go to next page." />
</a>
```

只有傳達重要資訊的影像才應該有 `alt` 文字。如果影像僅用於裝飾目的，則應該有 `null`（空白）`alt` 文字，讓螢幕閱讀器和其他輔助技術可以忽略它。

這是一個空的 `alt` 屬性的範例：

```html
<img src="decorative_image.jpg" alt="" />
```

你網站上的每個影像都應該有一個 `alt` 屬性，即使它是空的。如果你完全省略 `alt` 屬性，有些螢幕閱讀器會讀取檔案名稱，這可能會讓使用輔助技術的人感到分心，因此不建議這麼做。

最後，在你網站上線之前，你應該仔細測試螢幕閱讀器是否能正確讀取 `alt` 文字。

撰寫有效的 `alt` 文字對於建立可存取的網頁內容至關重要。作為網頁開發者，透過為你的影像提供清晰的描述，你可以確保每個人都能以包容的線上體驗參與你的網站。
