# --description--

多型是我們接下來要討論的物件導向程式設計（OOP）的下一個關鍵概念。

透過多型，你可以存取一個介面，在那裡你可以與許多相同種類的物件互動。

讓我們更深入地了解多型以及它如何讓你重複使用程式碼。

多型允許不同類別中的方法使用相同名稱，但執行不同的任務。你對不同的物件呼叫相同的方法名稱，每個物件都會以自己的方式回應。

以下是多型的基本範例：

```py
class A:
   def action(self): ...

class B:
   def action(self): ...

class C:
   def action(self): ...

Class().method()  # Works for A, B, or C
```

以下是一個使用不同動物聲音來描述多型的範例：

```py
class Cat:
   def speak(self):
       return "A cat meow"

class Bird:
   def speak(self):
       return "A bird tweet"
  
class Monkey:
   def speak(self):
       return "A monkey ooh ooh aah aah ooh ooh aah aah"

def animal_sound(animal):
   print(animal.speak())

animal_sound(Cat())
animal_sound(Bird())
animal_sound(Monkey())
```

在這個範例中，`animal_sound()` 是一個函式（程式），它接受任何具有 `speak()` 方法的物件。 

當你傳入 `Cat`、`Bird` 或 `Monkey` 時，它會呼叫該物件的 `speak()` 方法並列印結果。因為每個類別對 `speak()` 的定義不同，所以你會從相同的函式得到不同的輸出。這就是多型的實際運作。

這裡有另一個範例，這次包含實體和一個屬性：

```py
class Twitter:
   def __init__(self, content):
       self.content = content

   def post(self):
       return f"🐦 Tweet: '{self.content}' (280 chars max)"

class Instagram:
   def __init__(self, content):
       self.content = content

   def post(self):
       return f"📸 Instagram Post: '{self.content}' + ✨ filters"

class LinkedIn:
   def __init__(self, content):
       self.content = content

   def post(self):
       return f"💼 LinkedIn Article: '{self.content}' (Professional Mode)"

def start(social_media):
   print(social_media.post())  # Calls .post() on any object

# Instances
tweet = Twitter('Just learned Python polymorphism!')
photo = Instagram('Sunset vibes 🌅')
article = LinkedIn('Why OOP matters in 2024')

# The polymorphic calls - same function, different outputs
start(tweet) # 🐦 Tweet: 'Just learned Python polymorphism!' (280 chars max)
start(photo) # 📸 Instagram Post: 'Sunset vibes 🌅' + ✨ filters
start(article) # 💼 LinkedIn Article: 'Why OOP matters in 2024' (Professional Mode)
```

還有一種稱為**基於繼承的多型**的多型。

在基於繼承的多型中，父類別定義一個方法，並且多個子類別以自己的方式改寫該方法。然後你可以對任何子物件呼叫相同的方法，而它會根據該子類別的不同而有不同的行為。

這裡有一個範例：

```py
class Animal:
   def speak(self):
       return 'Some generic sound'

class Cat(Animal):
   def speak(self):
       return 'A cat meow'

class Dog(Animal):
   def speak(self):
       return 'A dog barks woof woof'

class Monkey(Animal):
   def speak(self):
       return 'A monkey ooh ooh aah aah ooh ooh aah aah'
  
print(Cat().speak()) # A cat meow
print(Dog().speak()) # A dog barks woof woof
print(Monkey().speak()) # A monkey ooh ooh aah aah ooh ooh aah aah
print(Animal().speak()) # Some generic sound
```

你可以看到每個父類別 `Animal` 的子類別都改寫了 `speak()` 方法以提供自己的實作。因此，當你對每個子類別的實體呼叫 `speak()` 方法時，它會傳回該動物相關的特定聲音。

你也可以更進一步，在列表中進行呼叫，然後透過迴圈遍歷列表來顯示每個 `speak()` 方法傳回的內容：

```py
animals = [Cat(), Dog(), Monkey()]

for animal in animals:
   print(animal.speak())

# Output:
# A cat meow
# A dog barks woof woof
# A monkey ooh ooh aah aah ooh ooh aah aah
```
