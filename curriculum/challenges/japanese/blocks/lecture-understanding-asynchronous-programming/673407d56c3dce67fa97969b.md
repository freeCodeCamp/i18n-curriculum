# --interactive--

前のレッスンでは、サーバーからデータを取得したり、ファイルからデータを読み込んだりといった時間のかかる処理を待つ間に、他のコードを実行できる非同期プログラミングについて学びました。

`async`/`await`はPromiseの上に構築されており、非同期コードの記述と読み取りを簡単にします。`async`キーワードを関数の前に付けると、その関数は常に`Promise`を返すことを意味します。`await`キーワードは、`Promise`が解決されるのを待ってから次のコード行に進むことを可能にしますが、これは非同期関数の中かモジュールのトップレベルの本文の中でのみ使うことができます。`async`/`await`の動作を示す例を見てみましょう。

:::interactive_editor

```js
async function delayedGreeting(name) {
  console.log("A Messenger entered the chat...");
  await new Promise(resolve => setTimeout(resolve, 2000));
  console.log(`Hello, ${name}!`);
}

delayedGreeting("Alice");
console.log("First Printed Message!");
```

:::

このコードでは、`async`関数として`delayedGreeting`を定義しています。この関数の中で、`await`を使って2秒間処理を一時停止しています。遅延の後に挨拶を出力します。

この関数を呼び出すと、挨拶の前に`First Printed Message!`が表示されるのがわかります。これは関数が非同期であり、他のコードの実行をブロックしないためです。

`async`/`await`の最大の利点の一つはエラー処理です。Promiseではエラー処理に`.catch()`メソッドを使うことが多かったですが、`async`/`await`では`try`/`catch`ブロックを使うことができます。例を見てみましょう。

```js
async function fetchUserData() {
  try {
    let response = await fetch(`https://api.example.com/users`);
    let userData = await response.json();
    console.log(userData);
  } catch (error) {
    console.log("Error fetching user data:", error);
  }
}

fetchUserData();
```

この例では、`async`/`await`を使ってAPIからユーザーデータを取得しています。`await`キーワードは2回使われています。1回目はfetch操作の完了を待ち、2回目はJSONの解析が終わるのを待っています。この処理中にエラーが発生した場合は、`catch`ブロックで捕捉されます。
