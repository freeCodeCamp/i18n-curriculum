# --description--

Давайте дізнаємося про реєстр npm, де ви можете поділитися своїми JavaScript-пакетами з усім світом.

**Реєстр npm** — це база даних публічних і приватних пакетів.

Розробники та організації можуть публікувати пакети в цей реєстр, щоб ділитися своїми інструментами з спільнотами та членами команд, а в багатьох випадках — з усіма іншими розробниками Node.js.

Пакети можуть бути публічними або приватними. Публічні пакети доступні будь-кому. Приватні пакети доступні лише певним користувачам або членам організації.

У реєстрі npm ви можете поділитися тим, що створили, з усім світом всього за кілька кліків. Звучить чудово, правда?

Давайте подивимося, як ви можете опублікувати пакет крок за кроком.

## Крок 1: Створіть обліковий запис

Перший крок — створити безкоштовний обліковий запис npm. Щоб створити свій обліковий запис:

* Перейдіть за посиланням [https://www.npmjs.com/signup](https://www.npmjs.com/signup).
    
* Заповніть форму. Вам потрібно буде ввести своє ім’я користувача, е-пошту та пароль.
    

Зверніть увагу, що згідно зі сторінкою реєстрації, «ваша адреса е-пошти буде додана до метаданих пакетів, які ви публікуєте, тому вона може бути видимою публічно».

## Крок 2: **Увійдіть**

Після створення облікового запису npm вам потрібно буде увійти в терміналі, виконавши цю команду:

```bash
npm login
```

Потім потрібно буде пройти автентифікацію, використовуючи свої дані облікового запису.

## **Крок 3: Створіть файл** `package.json`

Перед тим як публікувати пакет, потрібно створити файл `package.json` за допомогою цієї команди:

```bash
npm init
```

(Якщо у вашому проєкті він уже є, можете переходити безпосередньо до **кроку 4**).

Вам потрібно буде вказати таку інформацію для файлу `package.json`:

* **name**: Зазвичай це назва вашого проєкту/папки. Якщо ви плануєте зробити пакет публічним, ця назва має бути унікальною, якщо ви не використовуєте scopes (ми розглянемо це трохи пізніше).
    
* **version**: Починайте з `"1.0.0"`, це значення за замовчуванням. Воно представляє ініціальну версію вашого пакета, і ви можете оновлювати його, коли вносите зміни.
    
* **main**: Точка входу вашого проєкту. За замовчуванням це `index.js`.
    
* **author**: Можете вказати своє ім’я або улюблений нікнейм.
    
* **license**: Оберіть конкретну ліцензію. Приклади: MIT, ISC.
    
* **description**: Короткий опис вашого пакета.
    

При виборі назви пакета переконайтеся, що вона ще не зайнята в реєстрі npm. Для цього перейдіть за URL, замінивши `<your-package-name>` на назву вашого пакета:

```bash
https://www.npmjs.com/package/<your-package-name>
```

Якщо за цією адресою немає пакета, назва доступна і ви можете її використовувати.

Якщо ж назва вже зайнята, але ви дуже хочете її зберегти, можна використати так званий **scope** для публікації.

### **Scopes**

Коли ви створюєте обліковий запис користувача або організації в реєстрі npm, вам надається scope з вашим ім’ям користувача або назвою організації.

Scopes допомагають уникнути конфліктів імен, бо ви можете публікувати пакет під своїм scope, навіть якщо інший пакет з такою ж назвою вже існує.

Назва пакета зі scope починається з імені scope. Наприклад, у `package.json` ви знайдете таку пару ключ-значення:

```bash
"name": "@<scope-name>/<your-package-name>"
```

Цю зміну можна внести безпосередньо у файл, якщо потрібно, або створити файл `package.json` зі scope з самого початку, додавши `--scope=@<scope-name>` до команди `npm init`.

```bash
npm init --scope=@<scope-name>
```

Ім’я scope може бути вашим ім’ям користувача, якщо ви використовуєте особистий обліковий запис, або назвою організації, якщо це організаційний обліковий запис.

## Крок 4: Створіть README.md

README вашого пакета — це markdown-файл, у якому ви описуєте, що робить ваш пакет і як його використовувати.

Приклади важливого вмісту, який варто включити:

* Його призначення.
    
* Як його встановити.
    
* Важливі залежності та особливості.
    
* Як почати роботу.
    
* Приклади фрагментів коду.
    
* Правила внесення внесків (якщо ви їх приймаєте).
    
* Посилання на файл changelog, де відстежуються зміни у різних версіях.
    

Структура README гнучка, ви можете адаптувати її під свої потреби.

## **Крок 5:** `.npmignore`

Файл `.npmignore` повідомляє реєстру npm, які файли включати, а які ігнорувати при публікації пакета. Це дуже важливо, щоб не поширювати конфіденційну інформацію, таку як приватні ключі, паролі та особисті дані.

У цьому файлі можна вказати шаблони, які визначають, які файли, папки та типи файлів ігноруються. Він використовує ті ж правила шаблонів, що й файли `.gitignore`.

Якщо ви працювали з Git, то цей файл може здатися дуже схожим на `.gitignore`, правда? І це так. Згідно з документацією npm, «якщо немає файлу `.npmignore`, але є файл `.gitignore`, тоді npm ігнорує те, що вказано у файлі `.gitignore`».

Вам також слід знати, що npm шукає файли `.npmignore` і `.gitignore` у всіх підпапках вашого пакета, а не лише в кореневій папці.

За замовчуванням ці шляхи ігноруються автоматично, тому вам не потрібно додавати їх у `.npmignore`:

* `.*.swp`
    
* `._*`
    
* `.DS_Store`
    
* `.git`
    
* `.gitignore`
    
* `.hg`
    
* `.npmignore`
    
* `.npmrc`
    
* `.lock-wscript`
    
* `.svn`
    
* `.wafpickle-*`
    
* `config.gypi`
    
* `CVS`
    
* `npm-debug.log`
    

Папка `node_modules` також ігнорується, крім bundled dependencies.

## **Крок 6: Опублікуйте свій пакет**

Перед тим як опублікувати пакет у реєстрі npm, варто протестувати його, виконавши цю команду:

```bash
npm install <full-path-to-package-directory>
```

Коли будете готові опублікувати, команда залежатиме від типу пакета: scoped чи unscoped.

Якщо ваш пакет unscoped, виконайте цю команду:

```bash
npm publish
```

Якщо пакет scoped, він за замовчуванням буде опублікований як приватний (якщо ця функція доступна у вашому обліковому записі). Щоб зробити його публічним, потрібно додати `--access public` до команди.

```bash
npm publish --access public
```

Щоб побачити свій публічний пакет, перейдіть за цим URL:

```bash
https://npmjs.com/package/<your-package-name>
```

Під назвою пакета ви побачите позначку видимості (публічний або приватний).

## **Крок 7 (необов’язково): Встановіть свій пакет**

Тепер, коли все налаштовано, ви можете встановити свій пакет у проєкт Node.js за допомогою цієї команди:

```bash
npm install <your-package-name>
```

## Оновлення пакета

Якщо після публікації потрібно внести зміни, оновіть версію пакета у `package.json`.

Ось стандартна конвенція семантичного версіонування:

* **Перший реліз:** `1.0.0`.
    
* **Патч-реліз:** Збільшуйте третю цифру. Наприклад, оновлення з `1.0.0` до `1.0.1`.
    
* **Мінорний реліз:** Збільшуйте середню цифру і скидайте останню до нуля. Наприклад, оновлення з `1.0.0` до `1.1.0`.
    
* **Мажорний реліз:** Збільшуйте першу цифру і скидайте середню та останню до нуля. Наприклад, оновлення з `1.1.0` до `2.0.0`.
    
Чудова робота. Тепер ви знаєте, як опублікувати пакет у реєстрі npm і як оновлювати його, щоб виправляти помилки та додавати нові функції.
