# --interactive--

이전 수업에서는 배열을 다루고 배열에서 다양한 요소에 접근하는 방법을 처음 배웠습니다. 배열에서 두 번째 요소에 접근하는 방법을 다시 알려드리겠습니다:

:::interactive_editor

```js
const fruits = ["apple", "banana", "cherry"];

console.log(fruits[1]); // "banana"
```

:::

배열은 0부터 시작하는 인덱스를 사용하므로, 첫 번째 요소는 인덱스 `0`에 있고, 두 번째 요소는 인덱스 `1`에 있습니다. 만약 배열에 존재하지 않는 인덱스에 접근하려고 하면, JavaScript는 `undefined`를 반환한다는 점을 기억하세요.

:::interactive_editor

```js
let fruits = ["apple", "banana", "cherry"];
console.log(fruits[3]); // undefined
```

:::

이 예제에서는 `3` 배열에 `fruits` 인덱스가 없습니다. 그래서 로그에는 `undefined`가 표시됩니다. 이제 배열 요소를 업데이트하는 방법을 살펴보겠습니다. 특정 인덱스에 새 값을 할당하여 요소를 업데이트할 수 있습니다.

:::interactive_editor

```js
let fruits = ["apple", "banana", "cherry"];
fruits[1] = "blueberry";
console.log(fruits); // ["apple", "blueberry", "cherry"]
```

:::

이 예제에서는 인덱스 `banana`에 있는 `blueberry`를 `1`로 교체했습니다. 이 방법을 사용하면 인덱스를 알고 있는 한 배열의 어떤 요소든 변경할 수 있습니다. 또한 아직 존재하지 않는 인덱스에 값을 할당하여 배열에 새 요소를 추가할 수도 있습니다:

:::interactive_editor

```js
let fruits = ["apple", "banana", "cherry"];
fruits[3] = "date";
console.log(fruits); // ["apple", "banana", "cherry", "date"]
```

:::

하지만 이 작업을 할 때는 주의하세요. 현재 배열 길이보다 훨씬 큰 인덱스에 값을 할당하면, 그 사이 인덱스들은 undefined 요소로 채워져 예상치 못한 동작이 발생할 수 있습니다. JavaScript를 계속 다루면서 배열 요소에 접근하고 업데이트하는 이 방법들이 많은 프로그래밍 작업의 기본임을 알게 될 것입니다. 간단한 할 일 목록을 만들든 복잡한 자료 구조를 처리하든, 이 기술들은 매우 유용할 것입니다.
