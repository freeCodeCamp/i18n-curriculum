# --description--

もし `let` に慣れていなければ、<a href="/learn/javascript-algorithms-and-data-structures/basic-javascript/explore-differences-between-the-var-and-let-keywords" target="_blank" rel="noopener noreferrer nofollow">こちらの <code>let</code> と <code>var</code> の違いに関するチャレンジ</a>を確認してください。

`var` キーワードで変数を宣言すると、グローバルに宣言されるか、関数内で宣言された場合はローカルになります。

`let` キーワードも似た動作をしますが、いくつかの追加機能があります。`let` キーワードでブロック、文、または式の中で変数を宣言すると、そのスコープはそのブロック、文、または式に限定されます。

例えば：

```js
var numArray = [];
for (var i = 0; i < 3; i++) {
  numArray.push(i);
}
console.log(numArray);
console.log(i);
```

ここでコンソールには `[0, 1, 2]` と `3` の値が表示されます。

`var` キーワードでは、`i` はグローバルに宣言されます。したがって `i++` が実行されると、グローバル変数が更新されます。このコードは次のように似ています：

```js
var numArray = [];
var i;
for (i = 0; i < 3; i++) {
  numArray.push(i);
}
console.log(numArray);
console.log(i);
```

ここでコンソールには `[0, 1, 2]` と `3` の値が表示されます。

この動作は、`for` ループ内で `i` 変数を使って関数を作成し後で使うために保存した場合に問題を引き起こします。これは保存された関数が常に更新されたグローバル `i` 変数の値を参照するためです。

```js
var printNumTwo;
for (var i = 0; i < 3; i++) {
  if (i === 2) {
    printNumTwo = function() {
      return i;
    };
  }
}
console.log(printNumTwo());
```

ここでコンソールには `3` の値が表示されます。

ご覧の通り、`printNumTwo()` は 3 を出力し、2 ではありません。これは `i` に代入された値が更新され、`printNumTwo()` が関数が for ループ内で作成された時の `i` の値ではなくグローバル `i` を返すためです。`let` キーワードはこの動作に従いません：

```js
let printNumTwo;
for (let i = 0; i < 3; i++) {
  if (i === 2) {
    printNumTwo = function() {
      return i;
    };
  }
}
console.log(printNumTwo());
console.log(i);
```

ここでコンソールには `2` の値と `i is not defined` というエラーが表示されます。

`i` はグローバルスコープで宣言されていないため定義されていません。これは `for` ループ文内でのみ宣言されています。`printNumTwo()` は正しい値を返しました。なぜなら `i` キーワードがループ文内で異なる値（0、1、2）を持つ3つの異なる `let` 変数を作成したからです。

# --instructions--

`i` が `if` 文内で宣言されている変数が、関数の最初の行で宣言された `i` とは別の変数になるようにコードを修正してください。コード内で `var` キーワードを使わないように注意してください。

この演習は `var` と `let` キーワードが宣言された変数にスコープを割り当てる方法の違いを示すために設計されています。この演習で使われているような関数をプログラミングする場合、混乱を避けるために異なる変数名を使うことをお勧めします。
