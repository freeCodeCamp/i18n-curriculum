# --description--

Перш ніж ви зможете запушити свій репозиторій на GitHub, потрібно пройти автентифікацію. І SSH, і GPG ключі використовують пару публічного та приватного ключів. Це означає, що ви зберігаєте приватний ключ на своєму локальному комп’ютері, який використовуєте для автентифікації. Публічний ключ ви передаєте людям або сервісам, які мають підтвердити вашу автентифікацію. Іншими словами, приватний ключ потрібен для виконання дії, а публічний ключ — для перевірки цієї дії.

Ключі GPG, або Gnu Privacy Guard, зазвичай використовують для підписування файлів або комітів. Хтось може використати ваш публічний GPG ключ, щоб перевірити, що підпис файлу зроблено вашим ключем і що вміст файлу не було змінено чи підроблено. Ключі SSH, або Secure SHell, зазвичай використовують для автентифікації віддаленого з’єднання з сервером — через утиліту `ssh`. Проте, як ви дізнаєтеся з цього уроку, SSH ключ можна також використовувати для підписування комітів.

Щоб згенерувати GPG ключ, потрібно запустити:

```sh
gpg --full-generate-key
```

Для SSH ключа запустіть:

```sh
ssh-keygen -t ed25519 -C "your_email@example.com"
```

Обов’язково вкажіть свою реальну е-пошту.

Обидві ці команди мають запросити парольну фразу для захисту ключа. Вони необов’язкові, але дуже рекомендовані.

Для SSH ключів ви отримаєте два файли — один для приватного ключа, інший для публічного. Імена файлів публічних ключів закінчуються `.pub`, щоб ви випадково не поділилися приватним ключем. Ці файли будуть у `~/.ssh`.

GPG ключі, натомість, зберігаються у прихованій і зашифрованій теці. Замість цього, щоб побачити список публічних ключів, запустіть:

```sh
gpg --list-secret-keys --keyid-format=long
```

Потім, щоб отримати публічний ключ, використайте:

```sh
gpg --armor --export "<key id>"
```

А що ж можна робити з цими новими ключами? Ми вже говорили про підписування комітів, і це те, для чого можна використовувати будь-який із цих ключів.

Щоб підписувати коміти своїм GPG ключем, потрібно завантажити публічний ключ, а не приватний, у свій акаунт GitHub. Потім візьміть короткий ідентифікатор, який отримали, переглянувши ключі раніше, і виконайте цю команду, щоб встановити його як ключ підписування git:

```sh
git config --global user.signingkey <your_gpg_key_id>
```

Потім можна додати прапорець `-S` до команди `git commit`, щоб підписати конкретний коміт — доведеться ввести парольну фразу. Або, якщо хочете підписувати всі коміти автоматично, можна встановити конфігурацію autosign у `true`:

```sh
git config --global commit.gpgsign true
```

Щоб підписувати за допомогою SSH ключа, що є відносно новою функцією на GitHub, спочатку потрібно завантажити ключ у свій акаунт GitHub. Потім потрібно встановити режим підписування git на використання SSH:

```sh
git config --global gpg.format ssh
```

Потім, щоб встановити ключ підписування, передайте шлях до файлу замість ідентифікатора:

```sh
git config --global user.signingkey <path_to_your_ssh_keys>
```

Нарешті, можна увімкнути автоматичне підписування так само, як і для GPG ключа.

З будь-яким із цих налаштувань у вас тепер будуть перевірені коміти, які отримають спеціальний значок на GitHub.
