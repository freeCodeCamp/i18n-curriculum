# --description--

Buscar en una lista de elementos es una ocurrencia común en la informática. Hay dos algoritmos clave que debes conocer cuando se trata de búsqueda: búsqueda lineal y búsqueda binaria.

La búsqueda lineal comienza al inicio de una lista y recorre cada elemento hasta encontrar el valor objetivo que está buscando.

Si se encuentra el valor objetivo, se devuelve el índice donde está ubicado en la lista. Si no se encuentra el valor objetivo, se devuelve `-1`. Devolvemos `-1` porque no es un índice válido en la mayoría de los lenguajes de programación.

Aquí está cómo se ve el código para la búsqueda lineal:

```python
def linear_search(arr, target):
    for i in range(len(arr)):
        if arr[i] == target:
            return i
    return -1
```

Si la lista en la que buscaremos es `[13, 4, 7, 9, 10]` y el valor objetivo es `9`, la función devolvería `3` porque `9` está en el índice `3`.

Si cambiamos el valor objetivo a `5`, la función devolvería `-1` porque `5` no está en la lista.

Aunque este es un algoritmo relativamente sencillo, no es el más eficiente. Si tienes una lista grande de elementos, la búsqueda lineal puede tardar mucho tiempo en encontrar el valor objetivo.

La complejidad temporal de la búsqueda lineal es `O(n)` porque el tiempo que tarda en buscar en la lista crece linealmente con el tamaño de la lista.

La complejidad espacial de la búsqueda lineal es `O(1)` porque no requiere espacio adicional para buscar en la lista.

La búsqueda binaria es un algoritmo más eficiente para buscar en una lista grande de elementos. La condición aquí es que la lista debe estar ordenada en orden ascendente.

La búsqueda binaria funciona dividiendo la lista por la mitad y verificando si el valor objetivo está en el medio de la lista. Si el valor objetivo está en el medio de la lista, se devuelve el índice del valor objetivo. De lo contrario, el algoritmo verifica si el valor objetivo está en la mitad izquierda o derecha de la lista.

Continúa dividiendo las partes restantes de la lista en mitades hasta que se encuentre el valor objetivo. Si el valor objetivo no está en la lista, devuelve `-1`

Aquí está cómo se ve el código para la búsqueda binaria:

```python
def binary_search(arr, target):
    low = 0
    high = len(arr) - 1

    while low <= high:
        mid = (low + high) // 2  

        if arr[mid] == target:
            return mid
        elif arr[mid] < target:
            low = mid + 1
        else:
            high = mid - 1

    return -1
```

Comenzamos identificando un índice `low` y un índice `high`. Esto representa el rango de la lista que estamos buscando.

Luego verificamos la condición de que `low` sea menor o igual a `high`. Si `low` es mayor que `high`, hemos buscado en toda la lista y no se encontró el valor objetivo. En ese caso, detenemos la búsqueda y devolvemos `-1`.

Si el índice `low` es menor o igual que el índice `high`, calculamos el índice medio de la lista, `mid`. Luego verificamos si el valor objetivo está en el índice medio. Si es así, devolvemos el índice medio.

De lo contrario, verificamos si el valor en el punto medio es menor que el objetivo. Si lo es, actualizamos el índice bajo para que sea el índice medio más uno. Esto significa que buscaremos en la mitad derecha de la lista.

Por último, si ninguna de las otras condiciones es `True`, actualizamos el índice `high` para que sea el índice medio menos uno. Esto significa que buscaremos en la mitad izquierda de la lista.

Continuamos repitiendo este proceso hasta encontrar el objetivo o determinar que el objetivo no está en la lista.

La complejidad temporal de la búsqueda binaria es `O(log n)` porque el tiempo que tarda en buscar en la lista crece logarítmicamente con el tamaño de la lista.

La complejidad espacial de la búsqueda binaria es `O(1)` porque no requiere espacio adicional para buscar en la lista.

La búsqueda binaria y la búsqueda lineal pueden usarse para una variedad de problemas que encontrarás en la informática. Es importante entender las diferencias entre los dos algoritmos y cuándo usar cada uno.
