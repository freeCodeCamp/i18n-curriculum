# --description--

En la última lección, aprendiste que el servidor procesa una solicitud HTTP y envía una respuesta. Esa respuesta incluye varios recursos que ves en la web.

Los recursos devueltos por una respuesta HTTP incluyen recursos basados en texto, recursos multimedia, fuentes, y así sucesivamente. Cada uno de estos recursos viene con un encabezado **Content-Type**, que le indica al navegador cómo procesar y mostrar correctamente el recurso.

Los recursos basados en texto siempre han sido una parte fundamental de la web. De hecho, la web comenzó como texto puro. Hoy en día, los recursos basados en texto incluyen lo siguiente:

* **HTML (Hypertext Markup Language)**: La estructura básica de cualquier página web. Sin HTML, una página web no puede existir.
    
* **CSS (Cascading Stylesheet)**: El recurso que controla cómo se ve una página web. Con CSS, aplicas estilos como colores, fuentes y diseño al HTML.
    
* **JavaScript**: JavaScript añade interactividad y contenido dinámico a HTML y CSS.
    
* **JSON y XML**: Estos dos formatos te permiten transferir datos estructurados entre servidores y navegadores.
    
Los recursos multimedia incluyen imágenes, audios y videos. Dan vida a la web con elementos visuales y sonido.

Muchos sitios web dependen de imágenes para comunicar información visual. Las imágenes vienen en diferentes formatos como `JPG`, `JPEG`, `PNG`, `WEBP` y `GIF`.

Sitios web como YouTube y Vimeo ofrecen contenido de video con formatos como `MP4`, `MOV`, `AVI`, `WebM` y más.

El audio es una parte clave de muchas plataformas ya que permite a los usuarios reproducir música, podcasts y otro contenido sonoro. Los formatos de audio comunes incluyen `MP3`, `AAC`, Ogg y `WAV`. Plataformas como Spotify y YouTube Music dependen de estos formatos para ofrecer experiencias de audio de alta calidad.

Otros recursos que se devuelven en una respuesta HTTP son documentos como `PDF`, `DOCX` y `PAGES`, archivos comprimidos como `ZIP` y `RAR`, archivos de fuentes como `WOFF`, `WOFF2` y `TTF`, y ejecutables como `EXE`.
