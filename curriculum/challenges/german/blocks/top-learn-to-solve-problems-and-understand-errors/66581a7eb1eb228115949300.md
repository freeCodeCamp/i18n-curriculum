# --description--

Angenommen, Sie haben zwei Strings, die Sie kombinieren möchten, um eine Nachricht zu erstellen, wie im Folgenden:

```js
const str1 = "Hello";
const str2 = "World!";
const message = str1.push(str2);
```

<img src="https://cdn.statically.io/gh/TheOdinProject/curriculum/4ed59981b4ce2c60b5b83bf7415d3127b61821f5/foundations/javascript_basics/understanding_errors/imgs/03.png" style="width:100%" alt="ein Beispiel für einen Type Error in der Chrome Entwicklerkonsole">

Hier erhalten Sie eine `TypeError` mit einer Meldung, die besagt, dass `str1.push is not a function`. Dies ist eine häufige Fehlermeldung, die Lernende verwirrt, weil Sie vielleicht wissen, dass `.push()` sicherlich eine Funktion ist (zum Beispiel, wenn Sie sie zuvor verwendet haben, um Elemente zu Arrays hinzuzufügen).

Aber das ist der Schlüssel – `.push()` ist keine String-Methode, sondern eine Array-Methode. Daher ist sie „keine Funktion“, die Sie als String-Methode finden können. Wenn Sie `.push()` in `.concat()` ändern, eine richtige String-Methode, läuft unser Code wie beabsichtigt!

Ein guter Hinweis, den Sie sich merken sollten, wenn Sie mit einem `TypeError` konfrontiert sind, ist, den Datentyp zu betrachten, auf den Sie eine Methode oder Operation anwenden möchten. Wahrscheinlich stellen Sie fest, dass es nicht das ist, was Sie denken, oder die Operation oder Methode nicht mit diesem Typ kompatibel ist.

# --questions-- 

## --text--

Warum führt der folgende JavaScript-Code zu einem `TypeError`?

```js
const str1 = "Hello";
const str2 = "World!";
const message = str1.push(str2);
```

## --answers--

Die `.push()`-Methode kann nur bei Objekten verwendet werden, nicht bei Strings.

---

Die `.push()`-Methode ist für Strings nicht verfügbar, weil es sich um eine Array-Methode handelt.

---

Die `.push()`-Methode ist falsch geschrieben und sollte .pusch() heißen.

---

Der `TypeError` ist das Ergebnis eines Syntaxfehlers im JavaScript-Code.
