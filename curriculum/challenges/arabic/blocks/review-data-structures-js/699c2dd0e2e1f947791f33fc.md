## الخوارزميات وترميز Big O

- **الخوارزميات**: مجموعة من التعليمات الواضحة لحل مشكلة أو تنفيذ مهمة. يجب أن تنتهي الخوارزميات بعدد محدود من الخطوات، ويجب أن تكون كل خطوة دقيقة وواضحة.

- **ترميز Big O**: يصف أسوأ أداء محتمل أو معدل نمو الخوارزمية مع زيادة حجم المُدخَل. يركز على كيفية زيادة استخدام الموارد مع حجم المُدخَل، متجاهلاً العوامل الثابتة والحدود ذات الرتبة الأقل.

### التعقيدات الزمنية الشائعة

- **O(1) - زمن ثابت**: تستغرق الخوارزمية نفس الوقت بغض النظر عن حجم المُدخَل.

:::interactive_editor

```js
function checkEvenOrOdd(number) {
  if (number % 2 === 0) {
    return "Even";
  } else {
    return "Odd";
  }
}

console.log(checkEvenOrOdd(4)) // "Even"
console.log(checkEvenOrOdd(5)) // "Odd"
```

:::

- **O(log n) - زمن لوغاريتمي**: يزداد الوقت ببطء مع نمو المُدخَل. شائع في الخوارزميات التي تقلل حجم المشكلة بشكل متكرر بنسبة معينة (مثل البحث الثنائي).

- **O(n) - زمن خطي**: يزداد وقت التنفيذ تناسبياً مع حجم المُدخَل.

```js
for (const grade of grades) { // grades is an array
  console.log(grade);
}
```

- **O(n log n) - زمن لوغاريتمي خطي**: تعقيد زمني شائع لخوارزميات الفرز الفعالة مثل دمج الفرز والفرز السريع.

- **O(n²) - زمن تربيعي**: يزداد وقت التنفيذ تربيعياً. يظهر غالباً في الحلقات المتداخلة.

:::interactive_editor

```js
const n = 3;

for (let i = 0; i < n; i++) {
  for (let j = 0; j < n; j++) {
    console.log("Hello, World!");
  }
}
```

:::

### التعقيد المكاني

- **O(1) - مساحة ثابتة**: تستخدم الخوارزمية نفس كمية الذاكرة بغض النظر عن حجم المُدخَل.
- **O(n) - مساحة خطية**: يزداد استخدام الذاكرة تناسبياً مع حجم المُدخَل.
- **O(n²) - مساحة تربيعية**: يزداد استخدام الذاكرة تربيعياً مع حجم المُدخَل.

## تقنيات حل المشكلات

- **فهم المشكلة**: اقرأ نص المشكلة عدة مرات. حدد المُدخَل، الناتج المتوقع، وكيفية تحويل المُدخَل إلى الناتج.

- **الرمز الزائف**: وصف عالي المستوى لمنطق الخوارزمية مستقل عن اللغة. يستخدم لغة مكتوبة شائعة مختلطة مع تراكيب برمجية مثل `IF`، `ELSE`، `FOR`، `WHILE`.

```md
GET original_string
SET reversed_string = ""
FOR EACH character IN original_string:
  ADD character TO THE BEGINNING OF reversed_string
DISPLAY reversed_string
```

- **حالات الحافة**: مُدخلات محددة وصحيحة تحدث عند حدود ما يجب أن تتعامل معه الخوارزمية. اعتبرها دائماً واختبرها.

## المصفوفات

- **المصفوفات الثابتة**: لها حجم ثابت يُحدد عند التهيئة. تُخزن العناصر في مواقع ذاكرة متجاورة. لا يمكن تغيير الحجم أثناء تنفيذ البرنامج.

- **المصفوفات الديناميكية**: يمكن أن تنمو أو تتقلص تلقائياً أثناء تنفيذ البرنامج. تتعامل مع تغيير الحجم عبر النسخ التلقائي إلى مصفوفات أكبر عند الحاجة.

### تنفيذ المصفوفات الديناميكية في JavaScript

:::interactive_editor

```js
let numbers = [3, 4, 5, 6];
// Access elements
console.log(numbers[0])  // 3

// Update elements
numbers[2] = 16
console.log([...numbers]);  // [3, 4, 16, 6]

// Add elements
numbers.push(7)
console.log([...numbers]);  // [3, 4, 16, 6, 7] 

// Add at a specific index
numbers.splice(3, 0, 15);
console.log([...numbers]);  // [3, 4, 16, 15, 6, 7]

// Remove elements
numbers.splice(2, 1);  // Remove at specific index
console.log([...numbers]);  // [3, 4, 15, 6, 7]

numbers.pop()   // Remove last element
console.log([...numbers]); // [3, 4, 15, 6]
```

:::

### التعقيدات الزمنية للمصفوفات الديناميكية

- **الوصول**: O(1)
- **الإدراج في النهاية**: O(1) في المتوسط، O(n) عند الحاجة لتغيير الحجم
- **الإدراج في الوسط**: O(n)
- **الحذف**: O(n) للوسط، O(1) للنهاية

## المكدسات

- **المكدسات**: هيكل بيانات من نوع آخر يدخل أولاً يخرج أخيراً (LIFO). تُضاف وتُزال العناصر من الأعلى فقط.

- **عملية الدفع (Push)**: إضافة عنصر إلى أعلى المكدس. التعقيد الزمني: O(1).

- **عملية السحب (Pop)**: إزالة عنصر من أعلى المكدس. التعقيد الزمني: O(1).

:::interactive_editor

```js
// Using JavaScript array as a stack

let stack = [];

// Push operations
stack.push(1);
stack.push(2);
stack.push(3);
console.log([...stack]);  // [1, 2, 3]

// Pop operations
let topElement = stack.pop();
console.log(topElement);  // 3
console.log([...stack]);  // [1, 2]
```

:::

## الطوابير

- **الطوابير**: هيكل بيانات من نوع أول يدخل أولاً يخرج أولاً (FIFO). تُضاف العناصر إلى الخلف وتُزال من الأمام.

- **عملية الإدخال (Enqueue)**: إضافة عنصر إلى نهاية الطابور. التعقيد الزمني: O(1).

- **عملية الإخراج (Dequeue)**: إزالة عنصر من بداية الطابور. التعقيد الزمني: O(1).

:::interactive_editor

```js
// Using JavaScript array as a queue
let queue = [];

// Enqueue operations
queue.push(1);
queue.push(2);
queue.push(3);
console.log([...queue]);  // [1, 2, 3]

// Dequeue operations
let firstElement = queue.shift();
console.log(firstElement);  // 1
console.log([...queue]);  // [2, 3]
```

:::

## القوائم المرتبطة

- **القوائم المرتبطة**: هيكل بيانات خطي حيث تحتوي كل عقدة على بيانات ومرجع للعقدة التالية. العقد مرتبطة كسلسلة.

### القوائم المرتبطة الأحادية

- **الهيكل**: كل عقدة تحتوي على بيانات ومرجع واحد للعقدة التالية.
- **التنقل**: يمكن التحرك فقط للأمام من الرأس إلى الذيل.
- **عقدة الرأس**: أول عقدة في القائمة، عادةً العقدة الوحيدة التي يمكن الوصول إليها مباشرة.
- **عقدة الذيل**: آخر عقدة في القائمة، تشير إلى `null`.

### العمليات والتعقيدات الزمنية

- **الإدراج في البداية**: O(1)
- **الإدراج في النهاية**: O(n) - يجب التنقل إلى النهاية
- **الإدراج في الوسط**: O(n) - يجب التنقل إلى الموضع
- **الحذف من البداية**: O(1)
- **الحذف من النهاية**: O(n) - يجب التنقل لإيجاد العقدة السابقة
- **الحذف من الوسط**: O(n) - يجب التنقل لإيجاد العقدة

### القوائم المرتبطة الثنائية

- **الهيكل**: كل عقدة تحتوي على بيانات ومرجعين: العقدة التالية والعقدة السابقة.
- **التنقل**: يمكن التحرك في كلا الاتجاهين.
- **الذاكرة**: تتطلب ذاكرة أكثر من القوائم المرتبطة الأحادية بسبب المرجع الإضافي.

## متى تستخدم كل هيكل بيانات

- **المصفوفات (المصفوفات الديناميكية)**: عندما تحتاج وصول مرتب ومفهرس ولا تعرف الحجم مسبقاً
- **المكدسات**: لعمليات LIFO (وظيفة التراجع، تقييم التعبيرات، التراجع)
- **الطوابير**: لعمليات FIFO (جدولة المهام، البحث بالاتساع أولاً)
- **القوائم المرتبطة**: عند الحاجة لإدراج/حذف متكرر في البداية، حجم غير معروف، لا حاجة للوصول العشوائي
