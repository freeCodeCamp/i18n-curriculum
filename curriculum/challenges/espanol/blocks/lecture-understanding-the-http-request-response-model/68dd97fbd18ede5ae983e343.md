# --description--

Los sitios web y las aplicaciones necesitan recopilar datos y procesarlos, o generar algo que el usuario necesite, como resultados de búsqueda. Así que, al igual que el texto, los formularios también son una parte fundamental de la web.

Con formularios, los usuarios pueden registrarse e iniciar sesión en un sitio, enviar mensajes y buscar elementos. Por ejemplo, accedes a la página de resultados de búsqueda de Google escribiendo tu consulta en el formulario más popular en la web, el campo de búsqueda de Google.

Veamos qué estructura un formulario, como los campos de entrada, el atributo `action`, los métodos del formulario y, lo más importante, qué sucede cuando envías un formulario.

Los formularios se insertan en una página web con el elemento `form`, pero para que los inputs aparezcan, también necesitas los elementos `input`, según tu necesidad. Algunos de esos elementos `input` son texto, correo electrónico, contraseña, selección, archivo, inputs ocultos y más.

Para usar estos campos de entrada, tienes que envolverlos con el elemento `form`. Cada uno de los inputs necesita un atributo `name` para que el servidor sepa qué datos corresponden a cada campo.

Aquí hay un ejemplo:

```html
<form>
   <input type="text" name="username" />
   <input type="email" name="email" />
   <input type="file" name="profilePicture" />
   <input type="radio" name="gender" />
   <input type="checkbox" name="subscribe" />
   <input type="hidden" name="userId" />
</form>
```

Los campos de entrada también necesitan ser validados para asegurar que el usuario ingrese la información correcta.

Esta validación puede ocurrir tanto del lado del cliente como del lado del servidor. HTML te permite hacer una validación básica del lado del cliente con los atributos `required` y `pattern`. Aun así, también debes validar del lado del servidor para mayor seguridad, ya que la validación en HTML es principalmente para la experiencia del usuario y retroalimentación inmediata.

El atributo `required` especifica que el campo de entrada debe ser completado, mientras que el atributo `pattern` te permite ingresar expresiones regulares que obligan al usuario a escribir la información de la manera que necesitas.

Aquí tienes un ejemplo de eso para un campo de nombre de usuario:

```html
<input type="text" name="username" required pattern="[A-Za-z0-9]{3,}" />
```

Después de que los campos de entrada sean validados, debe realizarse el envío. Entonces, ¿qué sucede cuando se envía el formulario?

Primero, para que se realice el envío, el formulario necesita tener un atributo `action` que especifique a dónde debe enviar el navegador los valores de los campos de entrada y un atributo `method` que especifique cómo se envían los datos del formulario. Más sobre el atributo `method` más adelante.

Así es como podría verse un atributo action:

```html
<form action="/register">
	<!-- input fields -->
</form>
```

El atributo `method` puede ser `GET`, `POST`, `PUT`, `PATCH` o `DELETE`:

* `GET` recupera algo del servidor, por ejemplo, resultados de búsqueda.
    
* `POST` envía algo al servidor, por ejemplo, información de inicio de sesión o registro ingresada a través de los campos de entrada.
    
* `PUT` modifica algo en el servidor.
    
* `PATCH` modifica parcialmente algo en el servidor.
    
* `DELETE` elimina algo del servidor.
    
Ten en cuenta que HTML no soporta directamente los métodos `PUT`, `PATCH` y `DELETE`, pero existen soluciones alternativas. Si trabajas con JavaScript puro, puedes usar la Fetch API para especificar el método, y en Node.js y Express, puedes usar el paquete `method-override`.

Así es como podría verse un formulario que realiza una solicitud `GET`:

```html
<form action="/search" method="GET">
   <input type="text" name="q">
   <button type="submit">Search</button>
</form>
```

Y así es como podría verse un formulario que hace una solicitud `POST`:

```html
<form action="/submit" method="POST">
   <input type="text" name="username" required pattern="[A-Za-z0-9]{3,}" />
   <input type="password" name="password" />
   <button type="submit">Submit</button>
</form>
```

Después de la solicitud `GET` o `POST`, el servidor verifica la validación del lado del servidor y procesa o rechaza los valores de entrada.

Si la validación es exitosa, el servidor envía la información solicitada o lanza un error si la validación no es exitosa. 

Si la validación es exitosa, la respuesta del servidor podría ser una redirección en caso de autenticación o búsqueda, o un mensaje de éxito que confirme la acción.

Aquí tienes un resumen de lo que sucede cuando se envía un formulario:

* El usuario llena los campos del formulario.
    
* El navegador verifica la validación del lado del cliente (si existe) y continúa si la validación es exitosa.
    
* El navegador envía los datos del formulario al servidor según los atributos action y method.
    
* El servidor recibe los datos y los procesa (almacenándolos en una base de datos, registrando a un usuario, iniciando sesión de un usuario, enviando una consulta de búsqueda, y así sucesivamente).
    
* Si hay una validación del lado del servidor, el servidor vuelve a verificar los datos antes de procesarlos.
    
* El servidor responde con un mensaje de éxito, error, o redirige al usuario según el estado de validación (éxito o fallo).
    
* El usuario ve retroalimentación basada en la respuesta del servidor.
