# --description--

Il middleware può essere montato su una rotta specifica usando `app.METHOD(path, middlewareFunction)`. Il middleware può anche essere collegato a catena all'interno di una definizione di rotta.

Guarda il seguente esempio:

```js
app.get('/user', function(req, res, next) {
  req.user = getTheUserSync();  // Hypothetical synchronous operation
  next();
}, function(req, res) {
  res.send(req.user);
});
```

Questo approccio è utile per dividere le operazioni del server in unità più piccole. Questo porta a una struttura dell'app migliore e alla possibilità di riutilizzare il codice in posti diversi. Questo approccio può anche essere usato per eseguire una validazione sui dati. In ogni punto dello stack di middleware puoi bloccare l'esecuzione della catena corrente e passare il controllo a funzioni progettate specificamente per gestire gli errori. Oppure puoi passare il controllo alla rotta corrispondente successiva, per gestire casi speciali. Vedremo come nella sezione avanzata di Express.

# --instructions--

Nella rotta `app.get('/now', ...)` collega una funzione middleware e il gestore finale. Nella funzione middleware dovresti aggiungere l'ora corrente all'oggetto richiesta nella chiave `req.time`. Puoi usare `new Date().toString()`. Nel gestore, rispondi con un oggetto JSON, prendendo la struttura `{time: req.time}`.

**Nota:** Il test non passerà se non colleghi il middleware. Se monti la funzione altrove, il test fallirà, anche se il risultato dell'output è corretto.

  const response = await fetch(code + '/_api/chain-middleware-time');

L'endpoint `/now` dovrebbe restituire l'ora corrente.

```js
  const response = await fetch(code + '/_api/chain-middleware-time');
  if (!response.ok) {
    throw new Error(await response.text());
  }
  const data = await response.json();
  var now = new Date();
  assert.isAtMost(
    Math.abs(new Date(data.time) - now),
    20000,
    'the returned time is not between +- 20 secs from now'
  );
```
