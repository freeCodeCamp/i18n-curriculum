# --description--

Давайте розглянемо деякі з найважливіших технік покращення продуктивності JavaScript саме.

Можливо, це здасться дивним, але першою технікою є використання меншої кількості JavaScript, коли це можливо.

Якщо ви розробляєте простий статичний вебсайт, ви можете зробити те саме без фреймворків, що залежать від JavaScript.

Також варто намагатися видаляти будь-який невикористаний JavaScript код і використовувати простіші рішення, якщо це можливо.

Наприклад, можна розглянути використання вбудованих функцій браузера замість кастомних. До таких вбудованих функцій браузера належать валідація форм, вбудований відеопрогравач браузера та CSS-анімації замість кастомних JavaScript-анімацій.

Використовуйте якомога менше анімацій.

Зменшуйте кількість маніпуляцій з DOM у вашому JavaScript коді, оскільки це обчислювально затратна операція.

Розбиття вашого JavaScript коду на модулі, які виконують критичні та некритичні завдання, також корисне. Таким чином, ви зможете завантажувати критичні модулі якомога раніше, а некритичні відкладати, щоб відтворити сторінку якомога швидше.

Попереднє завантаження файлів не гарантує, що вони будуть доступні в момент використання, але завантаження почнеться раніше, тож загальний час очікування має бути коротшим.

Коли ваш код розбитий на кілька файлів, ви можете автоматично мінімізувати їх під час процесу збірки, щоб зменшити розмір і оптимізувати час завантаження.

Якщо слухач подій більше не потрібен, його слід видалити:

```js
element.removeEventListener("mousemove", handleMouseMove);
```

У цьому прикладі ми видаляємо слухача події для `mousemove`, тому функція `handleMouseMove` більше не викликається при спрацьовуванні події. Якщо залишити слухача події, програма продовжить виконувати непотрібні обчислення, що вплине на загальну продуктивність. Цей ефект буде ще помітнішим, якщо одночасно активні кілька слухачів подій.

І оскільки ми торкнулися теми подій, рекомендується використовувати делегування подій, коли це можливо.

Це означає, що не потрібно встановлювати слухачів подій на окремі дочірні елементи. Ви можете встановити їх безпосередньо на батьківський елемент, і подія буде підніматися вгору по дереву, доки не знайде батька, який зможе її обробити.

Оптимізація JavaScript є ключовою для розробки швидких і адаптивних вебзастосунків. У реальному проєкті ви постійно аналізуватимете продуктивність свого сайту, щоб знаходити області для покращення.
