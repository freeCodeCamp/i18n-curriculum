# --description--

Das Bereitstellen statischer Dateien ist eine häufige Anforderung in Webanwendungen. Statische Dateien umfassen Ressourcen wie Bilder, CSS-Dateien, clientseitiges JavaScript, Schriftarten und andere Ressourcen, die sich nicht dynamisch ändern.

Express macht das Bereitstellen dieser Dateien mit seiner eingebauten `express.static()`-Middleware einfach.

In Express bedeutet das Bereitstellen statischer Dateien, Dateien direkt vom Dateisystem des Servers an den Browser des Clients zu liefern, ohne zusätzliche serverseitige Verarbeitung.

Wenn ein Browser beispielsweise Dateien wie `/styles.css`, `/script.js` oder `/images/logo.png` anfragt, kann Express diese Dateien in einem angegebenen Ordner finden und direkt bereitstellen. Wenn eine Datei nicht gefunden wird, antwortet Express automatisch mit einem 404. Da diese Dateien nicht auf dem Server generiert oder verändert werden müssen, nennt man sie *statisch*.

Das macht das Bereitstellen von Assets wie Stylesheets und Bildern einfach und erspart es, für jede Datei manuell Routen zu definieren.

So richten Sie das Bereitstellen statischer Dateien in Express ein:

```javascript
const express = require("express");
const path = require("path");
const app = express();
const port = 3000;

// Serve static files from the "public" directory
app.use(express.static(path.join(__dirname, "public")));

app.get("/", (req, res) => {
  res.send("Welcome to the static file demo!");
});

app.listen(port, () => {
  console.log(`Server running at http://localhost:${port}`);
});
```

Zuerst importieren wir `express`, um unseren Server zu erstellen, und `path`, um Dateipfade plattformübergreifend sicher zu konstruieren.

Als Nächstes sagt `app.use(express.static(path.join(__dirname, "public")))` Express, Dateien aus dem Ordner `public` bereitzustellen. Die `express.static()`-Middleware weist Express an, Dateien aus dem Ordner `public` bereitzustellen. Jede Datei in diesem Verzeichnis wird für den Browser zugänglich.

Wir definieren außerdem eine einfache `GET`-Route für `/`, die eine Willkommensnachricht sendet. Das zeigt, dass das Bereitstellen statischer Dateien mit dynamischen Routen koexistieren kann.

Schließlich startet `app.listen()` den Server auf Port 3000. Sobald der Server läuft, werden Anfragen für statische Assets im Ordner `public` automatisch bearbeitet.

Damit das funktioniert, benötigen Sie einen Ordner namens `public`, um Ihre statischen Dateien zu speichern. Eine typische Projektstruktur könnte so aussehen:

```md
my-app/
│
├── app.js
└── public/
    ├── index.html
    ├── styles.css
    └── image.png
```

Wenn ein Benutzer eine Anfrage stellt, zum Beispiel indem er `http://localhost:3000/index.html` besucht, prüft Express, ob die angeforderte Datei im Verzeichnis `public` existiert. Wenn die Datei existiert, sendet Express sie direkt an den Browser. Wenn die Datei nicht existiert, geht Express zur nächsten Middleware weiter oder gibt eine 404-Antwort zurück.

Denken Sie daran, dass Sie keine Routen für jede Datei definieren müssen. Eine Anfrage an `http://localhost:3000/styles.css` bedeutet, dass Express nach `public/styles.css` sucht und diese automatisch zurückgibt.

Sie können auch anpassen, wie statische Dateien bereitgestellt werden. Zum Beispiel können Sie Dateien aus mehreren Verzeichnissen bereitstellen oder das Caching-Verhalten konfigurieren.

Stellen Sie sich vor, Ihre Anwendung hat diese Dateistruktur:

```md
my-app/
│
├── app.js
│
├── public/
│   ├── index.html
│   └── styles.css
│
└── images/
    └── image.png
```

So können Sie mehrere Verzeichnisse registrieren, um statische Dateien bereitzustellen:

```javascript
app.use(express.static(path.join(__dirname, "public")));
app.use("/images", express.static(path.join(__dirname, "images")));
```

Hier verwenden wir `app.use(express.static(path.join(__dirname, "public")));` wie zuvor, was bedeutet, dass Dateien im Verzeichnis `public` vom Stamm-Pfad aus bereitgestellt werden, genau wie zuvor.

Wenn Sie sich jedoch die nächste Zeile ansehen, sehen Sie, dass wir `"/images"` als erstes Argument an `app.use()` übergeben, gefolgt von `express.static(path.join(__dirname, "images"))`. In diesem Fall ist `"/images"` ein Mount-Pfad, der Express anweist, diese Middleware nur zu verwenden, wenn eine Anfrage an `/images` eingeht. Wenn das passiert, wird `express.static()` ausgelöst und sucht nach Dateien im Verzeichnis `images`.

Mit dieser Konfiguration funktionieren die folgenden Anfragen so:

*   GET `http://localhost:3000/styles.css`: Express sucht im Verzeichnis `public` nach `styles.css`, findet die Datei und sendet sie an den Client.
    
*   GET `http://localhost:3000/images/image.png`: Da diese Anfrage mit `/images` beginnt, sucht Express im Verzeichnis `images` nach `image.png`, findet die Datei und sendet sie an den Client.
    

Das Bereitstellen statischer Dateien mit Express ist einfach und leistungsstark. Mit `express.static()` können Sie problemlos Assets wie Bilder, Stylesheets und clientseitiges JavaScript bereitstellen.

Das Speichern statischer Dateien in einem eigenen Ordner wie `public` oder die Verwendung mehrerer Verzeichnisse und Mount-Pfade hilft, Ihre Anwendung organisiert, wartbar und skalierbar zu halten.
