# --interactive--

## Загальні атрибути доступності ARIA

- **`aria-expanded` атрибут**: Використовується для передачі стану функції перемикання (або розкриття) користувачам читачів екрана.

:::interactive_editor

```html
<button id="menuBtn" aria-expanded="false">Menu</button>

<script>
  const btn = document.getElementById("menuBtn");

  btn.addEventListener("click", () => {
    const expanded = btn.getAttribute("aria-expanded") === "true";
    btn.setAttribute("aria-expanded", String(!expanded));
  });
</script>
```

:::

- **атрибут `aria-haspopup`**: Цей стан використовується, щоб вказати, що інтерактивний елемент активує спливаючий елемент. Ви можете використовувати атрибут `aria-haspopup` лише тоді, коли спливаючий елемент має одну з таких ролей: `menu`, `listbox`, `tree`, `grid` або `dialog`. Значення `aria-haspopup` має бути або однією з цих ролей, або `true`, що еквівалентно `menu`.

:::interactive_editor

```html
<button
  id="menubutton"
  aria-haspopup="menu"
  aria-controls="filemenu"
  aria-expanded="false"
>
  File
</button>

<ul
  id="filemenu"
  role="menu"
  aria-labelledby="menubutton"
  hidden
>
  <li role="menuitem" tabindex="-1">Open</li>
  <li role="menuitem" tabindex="-1">New</li>
  <li role="menuitem" tabindex="-1">Save</li>
  <li role="menuitem" tabindex="-1">Delete</li>
</ul>

<script>
  const button = document.getElementById("menubutton");
  const menu = document.getElementById("filemenu");

  button.addEventListener("click", () => {
    const expanded = button.getAttribute("aria-expanded") === "true";

    button.setAttribute("aria-expanded", String(!expanded));
    menu.hidden = expanded;
  });
</script>
```

:::

- **атрибут `aria-checked`**: Цей атрибут використовується, щоб вказати, чи перебуває елемент у стані вибраності. Найчастіше його застосовують при створенні кастомних прапорців, радіокнопок, перемикачів і listbox.

:::interactive_editor

```html
<div
  id="checkbox"
  role="checkbox"
  aria-checked="true"
  tabindex="0"
  style="
    display: inline-flex;
    align-items: center;
    gap: 6px;
    cursor: pointer;
  "
>
  <span
    id="box"
    aria-hidden="true"
    style="
      width: 16px;
      height: 16px;
      border: 2px solid blue;
      background: blue;
      display: inline-block;
    "
  ></span>
  Checkbox
</div>

<script>
  const checkbox = document.getElementById("checkbox");
  const box = document.getElementById("box");

  const toggle = () => {
    const checked = checkbox.getAttribute("aria-checked") === "true";
    checkbox.setAttribute("aria-checked", String(!checked));
    box.style.background = checked ? "white" : "black";
  };

  checkbox.addEventListener("click", toggle);

  checkbox.addEventListener("keydown", (e) => {
    if (e.key === " " || e.key === "Enter") {
      e.preventDefault();
      toggle();
    }
  });
</script>
```

:::

- **атрибут `aria-disabled`**: Цей стан використовується, щоб вказати, що елемент вимкнений лише для користувачів допоміжних технологій, таких як читачі екрана.

:::interactive_editor

```html
<div
  id="editBtn"
  role="button"
  tabindex="-1"
  aria-disabled="true"
  style="opacity: 0.5; cursor: not-allowed;"
>
  Edit
</div>

<button id="toggle">Toggle Disabled</button>

<script>
  const editBtn = document.getElementById("editBtn");
  const toggleBtn = document.getElementById("toggle");

  toggleBtn.addEventListener("click", () => {
    const disabled = editBtn.getAttribute("aria-disabled") === "true";

    editBtn.setAttribute("aria-disabled", String(!disabled));
    editBtn.tabIndex = disabled ? 0 : -1;
    editBtn.style.opacity = disabled ? "1" : "0.5";
    editBtn.style.cursor = disabled ? "pointer" : "not-allowed";
  });
</script>
```

:::

- **атрибут `aria-selected`**: Цей стан використовується, щоб вказати, що елемент вибраний. Ви можете застосовувати цей стан до кастомних елементів керування, таких як вкладковий інтерфейс, listbox або сітка.

:::interactive_editor

```html
<div role="tablist">
  <button role="tab" aria-selected="true">Tab 1</button>
  <button role="tab" aria-selected="false">Tab 2</button>
  <button role="tab" aria-selected="false">Tab 3</button>
</div>

<script>
  const tabs = document.querySelectorAll('[role="tab"]');

  tabs.forEach((tab) => {
    tab.addEventListener("click", () => {
      tabs.forEach(t => t.setAttribute("aria-selected", "false"));
      tab.setAttribute("aria-selected", "true");
    });
  });
</script>
```

:::

- **атрибут `aria-controls`**: Використовується для зв’язку елемента з іншим елементом, яким він керує. Це допомагає користувачам допоміжних технологій зрозуміти взаємозв’язок між елементами.

:::interactive_editor

```html
<div role="tablist">
  <button 
    role="tab"
    id="tab1"
    aria-controls="section1"
    aria-selected="true"
  >
    Tab 1
  </button>
  <button
    role="tab"
    id="tab2"
    aria-controls="section2"
    aria-selected="false"
  >
    Tab 2
  </button>
  <button
    role="tab"
    id="tab3"
    aria-controls="section3"
    aria-selected="false"
  >
    Tab 3
  </button>
</div>
```

:::

- **атрибут `hidden`**: Приховує неактивні панелі як від візуальних користувачів, так і від користувачів допоміжних технологій.

## Робота з живими регіонами та динамічним вмістом

- **`aria-live` атрибут**: Робить частину вебсторінки живим регіоном, тобто будь-які оновлення в цій області будуть оголошені читачем екрана, щоб користувачі не пропустили важливі зміни.
- **`polite` значення**: Більшість живих регіонів використовують це значення. Воно означає, що оновлення не є терміновим, тому читач екрана може почекати, поки завершить поточне оголошення або користувач завершить свою дію, перш ніж оголосити оновлення.

Ось приклад живого регіону, який динамічно оновлюється за допомогою JavaScript:

:::interactive_editor

```html
<div aria-live="polite" id="status"></div>

<button id="updateStatus">Update Status</button>

<script>
  const statusEl = document.getElementById("status");
  const btn = document.getElementById("updateStatus");

  btn.addEventListener("click", () => {
    statusEl.textContent = "Your file has been successfully uploaded.";
  });
</script>
```

:::

- **`contenteditable` атрибут**: Перетворює елемент на живий редактор, дозволяючи користувачам оновлювати його вміст, ніби це текстове поле. Якщо для області з contenteditable немає видимої мітки або заголовка, додайте доступну назву за допомогою атрибута `aria-label`, щоб допомогти користувачам читачів екрана зрозуміти призначення редагованої області.

:::interactive_editor

```html
<div
  contenteditable="true"
  aria-label="Note editor"
  id="editor"
  style="border: 1px solid #ccc; padding: 8px;"
>
  Editable content goes here
</div>

<p id="status" aria-live="polite"></p>

<script>
  const editor = document.getElementById("editor");
  const status = document.getElementById("status");

  editor.addEventListener("input", () => {
    status.textContent = "Content updated";
  });
</script>
```

:::

## Події `focus` і `blur`

- **`blur` подія**: Виникає, коли елемент втрачає фокус.

:::interactive_editor

```html
<input
  id="nameInput"
  type="text"
  placeholder="Type here and click outside"
  aria-label="Name input"
/>

<p id="status" aria-live="polite"></p>

<script>
  const input = document.getElementById("nameInput");
  const status = document.getElementById("status");

  input.addEventListener("blur", () => {
    status.textContent = "Input lost focus";
  });
</script>
```

:::

- **`focus` подія**: Виникає, коли елемент отримує фокус.

:::interactive_editor

```html
<input
  id="emailInput"
  type="email"
  placeholder="Click or tab into this field"
  aria-label="Email input"
/>

<p id="status" aria-live="polite"></p>

<script>
  const input = document.getElementById("emailInput");
  const status = document.getElementById("status");

  input.addEventListener("focus", () => {
    status.textContent = "Input received focus";
  });
</script>
```

:::
