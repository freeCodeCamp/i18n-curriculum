# --description--

```js
function favoriteAnimal(animal) {
  return animal + " is my favorite animal!"
}

console.log(favoriteAnimal('Goat'));
```

Вставляючи `animal` у круглі дужки функції `favoriteAnimal()`, ви повідомляєте JavaScript, що надішлете деяке значення у вашу функцію `favoriteAnimal`. Це означає, що animal — це просто заповнювач для майбутнього значення. Але яке значення ви надсилаєте?

Останній рядок, `favoriteAnimal('Goat')`, — це місце, де ви викликаєте вашу функцію `favoriteAnimal` і передаєте значення `'Goat'` у цю функцію. Тут `'Goat'` — це ваш аргумент. Ви кажете функції `favoriteAnimal`: «Будь ласка, надішли `'Goat'` у функцію `favoriteAnimal` і використай `'Goat'` там, де є заповнювач `'animal'`». Завдяки гнучкості, яку дає використання параметра, ви можете оголосити будь-яку тварину своєю улюбленою.

Ось діаграма, яка допоможе вам уявити, як параметри передаються у функцію і як з неї повертаються значення.

Зверніть увагу на те, що викликаючи `favoriteAnimal()` всередині `console.log()` з аргументом `'Goat'`, ви отримуєте повернене значення функції — рядок `"Goat is my favorite animal!"`, який виводиться у консоль. Ви передаєте виклик функції `favoriteAnimal('Goat')` як аргумент у іншому виклику функції — `log()`.
