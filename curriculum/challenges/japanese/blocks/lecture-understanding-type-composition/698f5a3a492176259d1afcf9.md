# --description--

前のレッスンでは、type aliases の使い方を学び、オブジェクト型の名前の付け方の例を見ました。

ご存じのように、オブジェクト型に名前を付ける方法の一つは、次のように `type` キーワードを使うことです。

```ts
type ID = number | string;

type User = {
  id: ID;
  name: string;
};
```

もう一つの方法は、次のように `interface` 宣言を使うことです。

```ts
type ID = number | string;

interface User {
  id: ID;
  name: string;
}
```

`type` と `interface` は似ていますが、重要な違いは、インターフェースは `extends` キーワードを使って他のインターフェースを拡張するのに対し、`type` エイリアスは交差型（`&`）を使って型を組み合わせることです。

いくつか例を見てみましょう。

```ts
interface User {
  id: number | string;
  name: string;
}

interface Admin extends User {
  role: string;
}

const adminUser: Admin = {
  id: 101,
  name: "Alice",
  role: "superadmin"
};
```

この例では、`User` は `id` と `name` から始まります。`Admin` インターフェースは `User` を拡張しているので、`id` と `name` を含み、新しいプロパティ `role` を追加しています。

もし `type` を拡張したい場合は、次のように交差型（`&`）を使います。

```ts
type User = {
  id: number | string;
  name: string;
};

type Admin = User & {
  role: string;
};

const adminUser: Admin = {
  id: 101,
  name: "Alice",
  role: "superadmin"
};
```

`Admin` は `User` 型と、新しいオブジェクト型で `role` を含む型を組み合わせています。結果はインターフェースの例と同じ形状で、`id`、`name`、`role` を持つオブジェクトです。

`type` と `interface` はどちらもオブジェクトの形状を表現できます。どちらを使うか迷ったら、`interface` はオブジェクトの形状に特化しているのに対し、`type` はより柔軟で、ユニオン型やプリミティブ、より複雑な型の組み合わせも表現できることを覚えておいてください。
