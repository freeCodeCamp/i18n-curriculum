# --explanation--

`Think of it as` é usado para ajudar alguém a entender um conceito comparando-o com algo com que já esteja familiarizado. Ele incentiva você a imaginar o conceito de uma forma que facilite o entendimento. Por exemplo:

`Think of a function as a recipe.` - Isso ajuda você a imaginar uma função como um conjunto de instruções, semelhante a como uma receita fornece passos para criar um prato.

`Wrapping code` significa envolver ou empacotar o código de uma forma que controle o acesso a ele. Isso vai `limit` o acesso ao código. Por exemplo:

`We're wrapping the database access code inside a function to hide the complexity from the rest of the program.` - Isso significa que as interações com o banco de dados estão ocultas do exterior e a função fornece uma interface simplificada para trabalhar.
