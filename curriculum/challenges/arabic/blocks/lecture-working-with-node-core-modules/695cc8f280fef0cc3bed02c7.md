# --description--

تم إنشاء JavaScript في الأصل لتعمل في متصفحات الويب، حيث كان دورها الرئيسي جعل صفحات الويب تفاعلية. لهذا السبب، ركزت JavaScript المبكرة بشكل أساسي على معالجة النصوص في النماذج والتعامل مع نموذج كائن المستند (DOM).

مع ذلك، ليست كل البيانات على الويب نصية. الملفات، الصور، والفيديوهات هي بيانات ثنائية، تتطلب آليات معالجة مختلفة. في المتصفحات، عادةً ما تعالج هذه الأنواع من البيانات بواسطة مكونات متخصصة في المتصفح بدلاً من JavaScript نفسها.

تستخدم المتصفحات الحديثة محركات عرض ومحركات JavaScript لإدارة هذه المهام — مثل Blink (مع V8) في Chrome، وWebKit (مع JavaScriptCore) في Safari، وGecko (مع SpiderMonkey) في Firefox.

لا يعمل Node.js داخل المتصفح، لذا احتاج إلى طريقة خاصة به لمعالجة البيانات الثنائية، خصوصًا عند التعامل مع إدخال وإخراج الملفات (I/O) وتدفقات TCP، حيث تأتي البيانات على شكل أجزاء. هنا يأتي دور وحدة `Buffer`.

تتيح لك وحدة `Buffer` في Node.js العمل مباشرة مع البيانات الثنائية مثل الملفات، الصور، أو تدفقات الشبكة. يمكنك من خلالها تخزين ومعالجة البيانات الثنائية مباشرة في الذاكرة.

تمامًا مثل وحدة `fs`، تُعد `Buffer` واحدة من وحدات Node.js الأساسية، لذلك لا تحتاج إلى تثبيتها بشكل منفصل قبل استخدامها.

لاستخدامها، استورد الوحدة أولًا باستخدام التفكيك:

```js
const { Buffer } = require("buffer");
```

ثم استدعِ `Buffer` مع الطرق التي توفرها. على سبيل المثال، تتيح لك `Buffer.from()` إنشاء مخزن مؤقت من سلسلة نصية أو مصفوفة أو بيانات خام أخرى.

إليك كيفية إنشاء `Buffer` من سلسلة نصية ومصفوفة:

```js
// Create a buffer from a string
const myStrBuffer = Buffer.from("freeCodeCamp");
console.log(myStrBuffer); // <Buffer 66 72 65 65 43 6f 64 65 43 61 6d 70>

// Create a buffer from an array of numbers
const myNumBuffer = Buffer.from([
  70, 82, 69, 69, 67, 79, 68, 69, 67, 65, 77, 80,
]);

console.log(myNumBuffer); // <Buffer 46 52 45 45 43 4f 44 45 43 41 4d 50>
```

بينما يمكن استخدام بعض طرق وحدة `Buffer` دون استيرادها أولًا، فإن طرقًا أخرى غير متاحة إلا إذا استوردت `Buffer` صراحةً. لذلك يُنصح دائمًا باستيراد Buffer عند استخدامه في مشاريعك.

يمكنك الوصول إلى عناصر `buffer` الفردية تمامًا مثل المصفوفة:

```js
console.log(myNumBuffer[0]); // 70
console.log(myStrBuffer[0]); // 102
```

يمكنك أيضًا استخدام طريقة `toString()` على المخازن المؤقتة لرؤية شكلها الحقيقي:

```js
console.log(myStrBuffer.toString()); // freeCodeCamp
console.log(myNumBuffer.toString()); // FREECODECAMP
```

تتيح لك `Buffer.alloc()` إنشاء مخزن مؤقت جديد بحجم معين (عدد البايتات). يتم ملء كل بايت بداخله تلقائيًا بـ `0`:

```js
const someBuffer = Buffer.alloc(10);
console.log(someBuffer); // <Buffer 00 00 00 00 00 00 00 00 00 00>
```

يمكنك أن ترى أن المخزن المؤقت مُهيأ بالأصفار، بناءً على الحجم الممرر إلى طريقة `alloc()`.

يمكنك المتابعة واستخدام طريقة `Buffer.write()` للكتابة إلى هذا المخزن المؤقت:

```js
someBuffer.write("Hello fCC");

console.log(someBuffer); // <Buffer 48 65 6c 6c 6f 20 66 43 43 00>
console.log(someBuffer.toString()); // Hello fCC
```

إذا كتبت بيانات أكثر مما يمكن للمخزن المؤقت استيعابه، فسيتم اقتطاعها:

```js
someBuffer.write("Hello freeCodeCamp");

console.log(someBuffer); // <Buffer 48 65 6c 6c 6f 20 66 72 65 65>
console.log(someBuffer.toString()); // Hello free
```

أخيرًا، يمكنك استخدام `Buffer.byteLength()` لإظهار عدد البايتات اللازمة لتخزين سلسلة نصية بترميز معين:

```js
console.log(Buffer.byteLength("Hello freeCodeCamp")); // 18
```

تشمل طرق `Buffer` الأخرى:

- `Buffer.isBuffer()`: يتحقق مما إذا كان الكائن المعطى مخزنًا مؤقتًا
- `Buffer.compare()`: يقارن بين مخزنَين مؤقتَين ويُرجع ترتيب فرزهما
- `Buffer.concat()`: يدمج عدة مخازن مؤقتة في واحد
