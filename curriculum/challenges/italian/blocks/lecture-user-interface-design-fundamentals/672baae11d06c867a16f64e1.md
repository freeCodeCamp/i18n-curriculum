# --description--

Il progressive enhancement è un approccio di progettazione che garantisce a tutti gli utenti, indipendentemente dal browser o dal dispositivo, di poter consultare il contenuto essenziale e le funzionalità di un’applicazione.

Si concentra nel fornire un’esperienza di base che funzioni per tutti, offrendo allo stesso tempo funzionalità aggiuntive e miglioramenti agli utenti con browser più avanzati o connessioni internet migliori.

L’approccio del progressive enhancement si basa su questi principi fondamentali:

- Tutto il contenuto principale e le funzionalità di base devono essere accessibili su tutti i browser
- Tutti i layout avanzati devono essere forniti tramite fogli di stile CSS esterni
- Tutte le funzionalità avanzate devono essere fornite tramite file JavaScript esterni
- Le preferenze del browser dell’utente devono essere rispettate

Usare un approccio di progressive enhancement rende le tue applicazioni più accessibili perché tutto il contenuto e le funzionalità principali non dovrebbero essere bloccati in ambienti non supportati.

In termini di velocità, un approccio di progressive enhancement può anche aiutare a migliorare le prestazioni delle tue applicazioni.

Gli utenti con connessioni internet più lente potranno comunque consultare il contenuto perché il browser scaricherà prima le risorse necessarie.

Per quanto riguarda la SEO, il progressive enhancement può anche aiutare a migliorare la visibilità delle tue applicazioni.

I motori di ricerca saranno in grado di scansionare il contenuto delle tue applicazioni perché il contenuto principale è disponibile nella risposta HTML iniziale.

Sebbene alcuni abbiano criticato questo approccio ritenendo che non sia sempre realistico per applicazioni che si basano molto su JavaScript per le loro funzionalità, rimane comunque una buona pratica da seguire quando si costruiscono applicazioni.
