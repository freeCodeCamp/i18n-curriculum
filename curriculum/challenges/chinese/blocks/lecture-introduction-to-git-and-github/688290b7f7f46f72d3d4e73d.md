# --description--

你已经了解了如何创建提交并将它们推送到你自己控制的仓库。你也可以对你不维护的其他项目执行相同操作，但需要一些额外步骤。

首先，通常你不会对别人的 `project` 拥有“写入访问”权限。这意味着你不能直接向他们推送 `Commit`。相反，你需要创建该仓库的一个拷贝来进行工作。这个过程称为“forking”。

要分叉一个仓库，请访问你想在 GitHub 上贡献的仓库。在顶部，你会看到一个“Fork”按钮。点击此按钮将带你进入创建分叉的界面。

你将看到一个类似于你创建自己仓库时的屏幕，但选项更少。你可以选择一个所有者，要么是你的个人账户，要么是你有访问权限的组织，为分支选择一个新名称，以及一个新的描述。

你也可以选择只拷贝 `main` 分支（或仓库配置的默认分支）。这通常是个好主意，除非你需要为特定的特色分支做贡献。你将在后续课程中了解更多关于分支的内容。现在，请保持该选项勾选。

点击 "Create fork"，你的新仓库将被创建，且你会自动跳转到该仓库。

接下来，像克隆你自己的仓库一样，使用 SSH 或 GitHub CLI 将它克隆到你的局部计算机。

现在，让我们看看你的局部克隆配置了哪些远程。打开局部仓库目录中的终端并运行 `git remote -v` 以获取完整的列表，它应该看起来像这样：

```sh
origin git@github.com:your-username/your-repo-name.git (fetch)
origin git@github.com:your-username/your-repo-name.git (pull)
```

通过从 GitHub 克隆仓库，Git 已自动将你的派生仓库配置为 `origin` 远程。然而，当你从派生仓库工作时，你通常需要更新你的拷贝以引入原始仓库的更改。

为此，让我们将原始仓库添加为另一个远程。返回你 fork 仓库的页面，点击 `Code` 按钮，获取 SSH URL。然后运行命令来创建一个新的 upstream 远程：

```sh
git remote add upstream git@github.com:original-repo-username/repo-name.git
```

`upstream` 是远程的名称——你可以将它命名为其他名称，但 `upstream` 通常是惯例。如果你再次查看你的远程，你将看到你新建的 upstream 远程：

```sh
origin git@github.com:your-username/your-repo-name.git (fetch)
origin git@github.com:your-username/your-repo-name.git (pull)
upstream git@github.com:original-repo-username/your-repo-name.git (fetch)
upstream git@github.com:original-repo-username/your-repo-name.git (pull)
```

使用 `upstream` 远程，你将能够执行诸如 `git fetch upstream` 和 `git merge upstream/main` 之类的操作，以将原始项目的最新更改合并到你的局部分叉中。

分支仍然是一个代码库，你以相同的方式创建提交并推送更改。然而，在深入了解之前，你应该明白在对别人的项目进行更改时应遵循某些策略。你将在后续课程中学习更多关于这些策略的内容。

将其源代码公开、允许你修改代码并接受代码贡献的项目称为“开源”项目。开源有许多好处，例如允许社区帮助你识别和修复错误、从他人对你工作的贡献中学习，以及培养一个你可以转化为网络的开发者社区。

当你对某人的 `project` 进行更改时，你被视为一个“开源贡献者”。为开源 `project` 贡献可以很好地预演在专业开发团队工作的体验。这里的一些策略将帮助你做到这一点，并延续到你的第一个有偿职位。

如果你对如何贡献有任何疑问，大多数开源项目都会有贡献指南，其中包含你需要了解的所有内容的概述。但如果你仍然不确定，打开仓库中的 issue 提出你的问题总是没有坏处——或者，如果他们在像 Discord 这样的平台上有社区空间，加入并在那里提问。
