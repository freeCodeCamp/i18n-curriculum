# --description--

在之前的課程中，你已經學會如何使用原生型別、聯集型別以及介面。 

但是 TypeScript 有一些你應該注意的特殊型別。讓我們先從查看 `any` 型別開始。 

`any` type 用於表述任何類型的值。 

```ts
let randomValue: any;

randomValue = 42;           

randomValue = "Hello";     
 
randomValue = true;        

randomValue = { name: "Alice" }; 
```

使用這種型別時，你必須小心，因為它很容易被過度使用和誤用。你不想為了消除 TypeScript 的錯誤訊息而將所有東西都打成 `any` 型別。那樣會違背使用 TypeScript 及增加型別安全性的初衷。 

比起 `any` 型別，更安全的對應是 `unknown` 型別。`unknown` 類似於 `any`，但使用 `unknown` 型別時，你必須先對該變數進行型別檢查，才能使用它。

這裡有一個範例：

```ts
function doubleValue(value: unknown) {
  if (typeof value === "number") {
    console.log(value * 2);
  } else if (typeof value === "string") {
    console.log(value + value);
  }
}

doubleValue(10);    
doubleValue("Hi "); 
doubleValue(true);   
```

在此範例中，型別檢查確保操作行為僅在正確型別的值上執行。不符合預期型別的值將被忽略。

下一個我們將要看的型別是 `never` 型別。這個型別表述某些永遠不會發生的事情。

在大多數情況下，你不會使用 `never` 來撰寫型別註解。相反地，你更可能會在像這樣的錯誤訊息中看到 `never` 型別出現：

```ts
function processValue(value: string | number) {
  if (typeof value === "string") {
    console.log("String value:", value.toUpperCase());
  } else if (typeof value === "number") {
    console.log("Number value:", value * 2);
  } else {
    console.log(value); 
  }
}
```

在上述範例的 `else` 語句中，`value` 具有 `never` 型別。這是因為 `else` 分支現在無法被到達，且沒有剩餘的型別。如果你嘗試用除了 `string` 或 `number` 以外的值來呼叫該函式，將會顯示類似以下的錯誤：

```ts
function processValue(value: string | number) {
  if (typeof value === "string") {
    console.log("String value:", value.toUpperCase());
  } else if (typeof value === "number") {
    console.log("Number value:", value * 2);
  } else {
    console.log(value); 
  }
}

// Argument of type 'boolean' is not assignable to parameter of type 'string | number'.
processValue(true); 
```

我們最後要看的型別是 `void` 型別。當你有一個不傳回值的函式時，那就是 `void` 型別。 

這裡有一個範例：

```ts
type Status = "loading" | "success" | "error";

type Handler = {
  status: Status;
  onChange: (newStatus: Status) => void; 
};
```

在此範例中，`Handler` 型別的 `onChange` 屬性沒有傳回任何東西，這就是為什麼使用 `void`。
