# --description--

In der Softwareentwicklung bezeichnet eine Abhängigkeit eine Situation, in der eine Komponente oder ein Modul in einer Anwendung auf eine andere angewiesen ist, um richtig zu funktionieren. Abhängigkeiten sind in Softwareanwendungen üblich, weil sie es Entwicklerinnen und Entwicklern ermöglichen, vorgefertigte Funktionen oder Werkzeuge zu nutzen, die von anderen erstellt wurden.

Wenn Sie eine React-Anwendung erstellen möchten, müssen Sie die React-Abhängigkeiten installieren. Ohne diese Abhängigkeiten funktioniert kein Teil Ihres Codes korrekt, und die Anwendung zeigt eine Liste von Fehlern an.

Bei komplexeren Projekten müssen Sie oft auf viele Abhängigkeiten zurückgreifen. Wenn ein Satz von Abhängigkeiten in einem Projekt nicht gut verwaltet oder definiert ist, führt das zu dem, was als Abhängigkeits-Hölle bekannt ist.

Um Software-Abhängigkeiten in einem Projekt zu verwalten, benötigen Sie einen Paketmanager. Ein Paketmanager ist ein Werkzeug, das für die Installation, Aktualisierung und Entfernung von Abhängigkeiten verwendet wird. Viele beliebte Programmiersprachen wie JavaScript, Python, Ruby und Java verwenden Paketmanager.

In einer früheren Lektion wurden Sie kurz mit einem beliebten Paketmanager namens npm vertraut gemacht.

Um ein neues React-Projekt mit Vite und npm zu erstellen, können Sie den folgenden Befehl ausführen:

```sh
npm create vite@latest my-react-app -- --template react
```

Wie Sie aus den vorherigen Lektionen wissen, erstellt dieser Befehl den gesamten notwendigen Boilerplate-Code, der zum Starten einer neuen React-Anwendung erforderlich ist. Bevor Sie die Anwendung starten können, müssen Sie die Abhängigkeiten installieren, indem Sie `npm install` oder kurz `npm i` ausführen.

Sie können alle Abhängigkeiten in der `package.json`-Datei einsehen, die sich im Stammverzeichnis Ihres Projekts befindet.

Die `package.json`-Datei ist eine wichtige Konfigurationsdatei in Projekten, die Metadaten über Ihr Projekt enthält, einschließlich Name, Version und Abhängigkeiten. Sie definiert außerdem Skripte, Lizenzinformationen und andere Einstellungen, die bei der Verwaltung des Projekts und seiner Abhängigkeiten helfen.

Wenn Sie Abhängigkeiten installieren, wird ein `node_modules`-Ordner zu Ihrem Projekt hinzugefügt.

Der `node_modules`-Ordner ist der Speicherort aller Pakete und Bibliotheken, die Ihr Projekt benötigt. Dieser Ordner enthält den eigentlichen Code für die in der `package.json`-Datei aufgeführten Abhängigkeiten, einschließlich der direkten Abhängigkeiten Ihres Projekts und aller Abhängigkeiten dieser Abhängigkeiten.

Die beiden Kernabhängigkeiten, die für ein React-Projekt benötigt werden, sind die Pakete `react` und `react-dom`:

```json
"dependencies": {
  "react": "^18.3.1",
  "react-dom": "^18.3.1"
}
```

Die `package.json` listet die aktuell verwendeten Versionen dieser Pakete auf. Wenn Sie Pakete lokal aktualisieren müssen, können Sie den Befehl `npm update` ausführen. Oder Sie aktualisieren alle Pakete global, indem Sie den Befehl `npm update -g` ausführen.

Zusätzlich zur `package.json`-Datei gibt es auch eine `package-lock.json`-Datei. Diese Datei sperrt die genauen Versionen aller Pakete, die Ihr Projekt verwendet. Wenn Sie ein Paket aktualisieren, werden die neuen Versionen auch in der Sperrdatei aktualisiert.

Ein weiterer wichtiger Aspekt der `package.json`-Datei sind die Dev-Abhängigkeiten:

```json
"devDependencies": {
  "@eslint/js": "^9.17.0",
  "@types/react": "^18.3.18",
  "@types/react-dom": "^18.3.5",
  "@vitejs/plugin-react": "^4.3.4",
  "eslint": "^9.17.0",
  "eslint-plugin-react": "^7.37.2",
  "eslint-plugin-react-hooks": "^5.0.0",
  "eslint-plugin-react-refresh": "^0.4.16",
  "globals": "^15.14.0",
  "vite": "^6.0.5"
}
```

Dev-Abhängigkeiten sind Pakete, die nur für die Entwicklung verwendet werden und nicht in der Produktion. Ein Beispiel dafür ist eine Testbibliothek wie Jest. Jest würden Sie als Dev-Abhängigkeit installieren, weil Sie sie nur zum Testen Ihres Projekts lokal verwenden und sie für den Betrieb der Anwendung in der Produktion nicht benötigt wird.

Im Großteil dieser Lektion lag der Fokus auf npm. Es gibt jedoch auch andere Paketmanager wie yarn und pnpm. Welchen Paketmanager sollten Sie also für Ihr Projekt verwenden?

Wenn Sie einem bestehenden Projekt in einem Team beitreten, werden all diese Entscheidungen für Sie getroffen, und Sie verwenden den vorhandenen Paketmanager. Wenn Sie ein Projekt von Grund auf neu erstellen, müssen Sie die Vor- und Nachteile der einzelnen Manager recherchieren und entscheiden, welcher besser zu Ihren Bedürfnissen passt.
