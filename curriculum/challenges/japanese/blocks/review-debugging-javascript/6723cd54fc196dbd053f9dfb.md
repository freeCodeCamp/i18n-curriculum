# --interactive--

## よくあるエラーメッセージの種類

- **SyntaxError**: これはコード内で何かを間違って書いたときに発生します。例えば、括弧やブラケットの閉じ忘れなどです。文の文法ミスのようなものだと考えてください。

```js
const arr = ["Beau", "Quincy" "Tom"]
```

- **ReferenceError**: ReferenceErrorにはいくつかの種類があり、発生する原因も異なります。最も一般的なReferenceErrorは未定義の変数を使おうとした場合です。もう一つの例は、`let`や`const`で宣言された変数を定義する前にアクセスしようとした場合です。

:::interactive_editor

```js
console.log(num);
const num = 50;
```

:::

- **TypeError**: これは間違った型に対して操作をしようとしたときに発生します。

```js
const developerObj = {
  name: "Jessica",
  country: "USA",
  isEmployed: true
};

developerObj.map()
```

- **RangeError**: JavaScriptが扱える範囲外の値を使おうとしたときに発生します。

```js
const arr = [];
arr.length = -1; 
```

## `throw` 文

- **定義**: JavaScriptの`throw`文は、ユーザー定義の例外をスローするために使います。プログラミングにおける例外とは、予期しない出来事が起きてプログラムの通常の流れが中断されることです。

:::interactive_editor

```js
function validateNumber(input) {
  if (typeof input !== "number") {
    throw new TypeError("Expected a number, but received " + typeof input);
  }
  return input * 2;
}

console.log(validateNumber("Naomi"));   // TypeError: Expected a number, but received string
```

:::

## `try...catch...finally`

- **定義**: `try`ブロックは、エラーが発生する可能性のあるコードを囲むために使います。失敗するかもしれない処理を試す安全な場所として機能します。`catch`ブロックはtryブロック内で発生したエラーをキャッチして処理します。catch内のエラーオブジェクトを使って何が問題だったか調べることができます。`finally`ブロックは、エラーの有無にかかわらずtryとcatchの後に必ず実行されます。ファイルを閉じたりリソースを解放したりするクリーンアップ処理に使われることが多いです。

:::interactive_editor

```js
function processInput(input) {
  if (typeof input !== "string") {
    throw new TypeError("Input must be a string.");
  }

  return input.toUpperCase();
}

try {
  console.log("Starting to process input...");
  const result1 = processInput("hello");
  console.log("Processed result:", result1);  // HELLO
  const result2 = processInput(9);  // throws TypeError
  console.log("Processed result:", result2);  // not executed
} catch (error) {
  console.error("Error occurred:", error.message);
} 
```

:::

## デバッグの手法

- **`debugger` 文**: この文を使うと、特定の行でコードの実行を一時停止してプログラムの状態を調べることができます。

:::interactive_editor

```js
let firstNumber = 5;
let secondNumber = 10;

debugger; // Code execution pauses here
let sum = firstNumber + secondNumber;

console.log(sum);
```

:::

- **ブレークポイント**: ブレークポイントは、任意の行でコードの実行を一時停止させることができます。停止後は変数の値を調べたり、式を評価したり、呼び出し履歴を確認したりできます。
- **ウォッチャー**: ウォッチ式は、現在のスコープ外であってもコードの実行中に変数や式の値を監視できます。
- **プロファイリング**: プロファイリングは、CPU使用率や関数呼び出し、実行時間を記録してパフォーマンスのボトルネックを特定するのに役立ちます。
- **`console.dir()`**: このメソッドは、指定したJavaScriptオブジェクトのプロパティをインタラクティブなリストで表示します。階層的に展開してネストされたすべてのプロパティを確認できます。

```js
console.dir(document);
```

- **`console.table()`**: このメソッドは、コンソールに表形式のデータをテーブルとして表示します。必須の引数は配列かオブジェクトで、オプションの引数で表示するプロパティ（列）を指定できます。
