# --description--

Katika React, kunaweza kuwa na hali ambapo unahitaji upatikanaji wa moja kwa moja kwa kipengele cha DOM. Hapo ndipo "refs" zinapokuja kuwa msaada. Refs pia zinaweza kuhifadhi thamani zinazobadilika, lakini state ni chaguo bora zaidi kwa hilo.

Katika JavaScript ya kawaida, ulitumia njia za `getElementById()` na `querySelector()` kufikia vipengele vya DOM. Lakini katika React, unatumia refs kufikia vipengele katika DOM.

Tofauti moja kuu ni kwamba, kwa kutumia refs, hauhitaji vitambulisho kama IDs na darasa la vitu kurejea vipengele.

Basi, unaweza kuunda na kutumia refs vipi? React hutoa kitufe cha `useRef()` kinachokuwezesha kufanya hivyo.

Hatua ya kwanza ni kuleta kitufe hicho kutoka React:

```js
import { useRef } from "react";
```

Ifuatayo, unahitaji kuunda kigezo kinachoshikilia ref na thamani ya awali ya ref ndani ya kitufe cha `useRef`, sema `sectionRef` iliyowekwa kuwa `null`:

```js
const sectionRef = useRef(null);
```

Jambo la mwisho kufanya ni kuambatanisha kigezo cha ref kwenye kipengele chako cha JSX kwa kutumia sifa ya `ref`:

```jsx
<section ref={sectionRef}>
  {/* Section content */}
</section>
```

Ukichapisha ref kwenye konsoli, utaona ni kitu chenye thamani ya sasa, katika kesi hii, `null`:

```js
console.log(sectionRef); // { current: null }
```

Pia unaweza kuchapisha thamani ya sasa kwenye konsoli kwa kutumia sifa ya `current` ili uone thamani moja kwa moja:

```js
console.log(sectionRef.current); // null
```

Thamani zinazofuata za ref zinategemea mzunguko wa maisha wa sehemu.

Kwa mfano, thamani ya awali ya `sectionRef` itakuwa daima `null` kwa sababu hiyo ndiyo iliyoanzishwa. Baada ya sehemu kuwekewa, thamani ya ref itakuwa kipengele cha `section` ambacho ref imeambatanishwa nacho.

Ikiwa sehemu itatolewa, thamani ya ref inarudi kwenye thamani ya awali ya `null`.

Mfano wa kawaida wa kuonyesha ref ni kuweka makini kwenye kipengele cha ingizo wakati wa onyesho, au kwa kubofya kitufe.

Hivi ndivyo unavyofanya hivyo unapobofya kitufe:

```jsx
import { useRef } from "react";

const Focus = () => {
  const inputRef = useRef(null);

  const handleFocus = () => {
    if (inputRef.current) {
      inputRef.current.focus();
    }
  };

  return (
    <div>
      <input ref={inputRef} type="text" placeholder="Enter text" />
      <button onClick={handleFocus}>Focus Input</button>
    </div>
  );
};

export default Focus;
```

Katika msimbo hapo juu, `inputRef` imeundwa na kuambatanishwa na kipengele cha `input`. Pia kuna kitufe chenye tukio la `onClick` linaloitisha kitendakazi cha `handleFocus`.

Kile kitendakazi cha `handleFocus` kinachofanya ni kuitisha njia ya `focus()` kwenye kipengele cha `input`. Kumbuka, kwa sababu `input` ni sehemu iliyojengwa ndani inayokuja na React, kipengele halisi cha `input` cha DOM kimewekwa kwenye sifa ya `current` ya ref. Hivyo unaitisha njia ya `focus()` kwa `input.current.focus()`.

Hapa kuna mbinu bora unazopaswa kuzifahamu unapotumia refs:

- Tumia refs hasa kuingiliana na DOM. Pia unaweza kuzitumia kwa data zinazobadilika, lakini state ni chaguo bora zaidi kwa hilo.

- Usitumie refs kwa usimamizi wa state wa kawaida – hilo ni jukumu la `useState`.

- Hakikisha unakagua kama `ref.current` ipo kabla ya kufikia sifa zake. Hapa ni jinsi ya kufanya hivyo tena:

```js
const handleFocus = () => {
  if (inputRef.current) {
    inputRef.current.focus();
  }
};
```

Hii inazuia makosa ikiwa ref itafikiwa kabla haijaambatanishwa na DOM au baada ya kuondolewa.
