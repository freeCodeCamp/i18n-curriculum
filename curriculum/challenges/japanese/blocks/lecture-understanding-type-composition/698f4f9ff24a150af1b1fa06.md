# --description--

これまでのレッスンでは、プリミティブ型やユニオン型、インターフェースの使い方を学びました。

しかし、TypeScriptには知っておくべき特別な型があります。まずは`any`型を見てみましょう。

`any`型は、あらゆる種類の値を表すために使われます。

```ts
let randomValue: any;

randomValue = 42;           

randomValue = "Hello";     
 
randomValue = true;        

randomValue = { name: "Alice" }; 
```

この型は簡単に多用・誤用されやすいので注意が必要です。TypeScriptのエラーメッセージを消すためだけにすべてを`any`型にするのは避けてください。それではTypeScriptを使い型安全性を高める意味がなくなってしまいます。

`any`型のより安全な代替は`unknown`型です。`unknown`は`any`に似ていますが、`unknown`型の場合は変数を使う前に型チェックを行う必要があります。

例を示します。

```ts
function doubleValue(value: unknown) {
  if (typeof value === "number") {
    console.log(value * 2);
  } else if (typeof value === "string") {
    console.log(value + value);
  }
}

doubleValue(10);    
doubleValue("Hi "); 
doubleValue(true);   
```

この例では、型チェックによって正しい型の値にのみ操作が行われます。期待される型に合わない値は無視されます。

次に見ていくのは`never`型です。この型は決して起こらないことを表します。

ほとんどの場合、`never`を使って型注釈を書くことはありません。代わりに、次のようなエラーメッセージで`never`型が現れることが多いです。

```ts
function processValue(value: string | number) {
  if (typeof value === "string") {
    console.log("String value:", value.toUpperCase());
  } else if (typeof value === "number") {
    console.log("Number value:", value * 2);
  } else {
    console.log(value); 
  }
}
```

上の例の`else`節の中で、`value`は`never`型を持っています。これは`else`の分岐が到達不可能になり、残る型がなくなったためです。もし`string`や`number`以外の値で関数を呼び出そうとすると、次のようなエラーが表示されます。

```ts
function processValue(value: string | number) {
  if (typeof value === "string") {
    console.log("String value:", value.toUpperCase());
  } else if (typeof value === "number") {
    console.log("Number value:", value * 2);
  } else {
    console.log(value); 
  }
}

// Argument of type 'boolean' is not assignable to parameter of type 'string | number'.
processValue(true); 
```

最後に見ていくのは`void`型です。戻り値がない関数は`void`型になります。

例を示します。

```ts
type Status = "loading" | "success" | "error";

type Handler = {
  status: Status;
  onChange: (newStatus: Status) => void; 
};
```

この例では、`onChange`型の`Handler`プロパティは何も返さないため、`void`が使われています。
