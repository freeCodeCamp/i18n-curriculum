# --description--

Subleq — це приклад комп’ютера з однією інструкцією (OISC).

Назва походить від єдиної інструкції, яка виконує **SU**бтракцію та **B**ранч, якщо результат **L**ише менший або **EQ**уальний нулю.

Ваше завдання — створити інтерпретатор, який емулюватиме таку машину.

Пам’ять машини складається з масиву знакових цілих чисел. Будь-який розумний розмір слова підходить, але пам’ять має вміщувати як від’ємні, так і додатні числа.

Виконання починається з покажчика команд, спрямованого на перше слово за адресою 0. Далі виконується таке:

<ol>
  <li>Нехай A, B і C — це значення, збережені у трьох послідовних словах пам’яті, починаючи з адреси покажчика команд.</li>
  <li>Зсунути покажчик команд на 3 слова вперед, щоб він вказував на адресу після тієї, що містить C.</li>
  <li>Якщо A дорівнює -1, зчитати символ зі стандартного вводу і зберегти його код у адресі, заданій B. C не використовується.</li>
  <li>Якщо B дорівнює -1, число за адресою A інтерпретується як код символу, який виводиться. C знову не використовується.</li>
  <li>Інакше A і B розглядаються як адреси пам’яті. Число за адресою A віднімається від числа за адресою B (результат зберігається назад за адресою B). Якщо результат нульовий або від’ємний, покажчик команд стає рівним C.</li>
  <li>Якщо покажчик команд стає від’ємним, виконання припиняється.</li>
</ol>

Інші від’ємні адреси, крім -1, можна вважати еквівалентними -1 або викликати помилку — на ваш розсуд.

Ваш розв’язок має приймати програму для виконання на машині окремо від вхідних даних самої програми.

Ця програма має бути у вигляді сирого сублек-коду — десяткові числа, розділені пробілами, без символічних імен чи інших розширень асемблера, які завантажуються в пам’ять починаючи з адреси 0. Покажіть вихід вашого розв’язку, коли він отримує цю програму "Hello, world!". (Зверніть увагу, що приклад припускає ASCII або його надмножину, наприклад будь-який з латинських наборів символів або Unicode. Ви можете перекласти це в інший набір символів, якщо ваша реалізація працює в середовищі, несумісному з ASCII.)

<pre>15 17 -1 17 -1 -1 16 1 -1 16 3 -1 15 15 0 0 -1 72 101 108 108 111 44 32 119 111 114 108 100 33 10 0</pre>

Що відповідає приблизно такому коду в гіпотетичній мові асемблера:

<pre>start:
    zero, message, -1
    message, -1, -1
    neg1, start+1, -1
    neg1, start+3, -1
    zero, zero, start
zero: 0
neg1: -1
message: "Hello, world!\n\0"
</pre>

function Subleq(mem) {
  var out = '';
  var instructionPointer = 0;
  do {
    var a = mem[instructionPointer];
    var b = mem[instructionPointer + 1];
    if (a === -1) {
    } else if (b === -1) {
      out += String.fromCharCode(mem[a]);
    } else {
      mem[b] -= mem[a];
      if (mem[b] < 1) {
        instructionPointer = mem[instructionPointer + 2];
        continue;
      }
    }
    instructionPointer += 3;
  } while (instructionPointer >= 0);

  return out;
}
