# --interactive--

前のレッスンでは、関数の使い方を学びました。関数は再利用可能なコードのかたまりで、コードをよりモジュール化し、保守しやすく、効率的にします。これまでの例はすべて、次のような通常の関数構文を使っていました。

```js
function greetings(name) {
  console.log("Hello, " + name + "!");
}
```

しかし、JavaScriptで関数を書くもう一つの方法として、アロー関数式を作成する方法があります。前の例をアロー関数構文に書き換えると次のようになります。

```js
const greetings = (name) => {
  console.log("Hello, " + name + "!");
};
```

この修正された例では、`const`という変数を作成し、無名関数を代入しています。構文のほとんどは見慣れたものですが、`function`キーワードがなくなり、`name`パラメータと関数本体の間に矢印（`=>`）が追加されている点が異なります。パラメータが1つだけの場合は、丸括弧を省略できます。

```js
const greetings = name => {
  console.log("Hello, " + name + "!");
};
```

アロー関数にパラメータがない場合は、丸括弧を必ず使う必要があります。

```js
const greetings = () => {
  console.log("Hello");
};
```

関数の本体を波括弧で囲む必要がありましたが、関数本体が1行だけの場合は波括弧を省略できます。

```js
const greetings = name => console.log("Hello, " + name + "!");
```

通常の関数構文で丸括弧や波括弧を省略するとエラーになります。例えば次のようなコードは動作しません。

```js
// This will produce syntax errors 
function greetings name console.log("Hello, " + name + "!");
```

このような1行の関数はアロー関数構文でのみ使えます。もう一つの重要な概念は`return`文です。アロー関数構文を使って面積を計算する例を示します。

:::interactive_editor

```js
const calculateArea = (width, height) => {
  const area = width * height;
  return area;
};

console.log(calculateArea(5, 3)); // 15
```

:::

関数内で`area`という変数を作成し、その変数を返しています。しかし計算自体を直接返すようにコードを簡潔にできます。

:::interactive_editor

```js
const calculateArea = (width, height) => {
  return width * height;
}; 

console.log(calculateArea(5, 3)); // 15
```

:::

波括弧を省略して計算を同じ行に書こうとすると、`Uncaught SyntaxError: Unexpected token 'return'`というエラーメッセージが表示されます。

```js
const calculateArea = (width, height) => return width * height;
```

このエラーが出る理由は、`return`文を削除する必要があるからです。`return`文を削除すればエラーは消え、関数は計算結果を暗黙的に返します。

```js
const calculateArea = (width, height) => width * height;
```

では、いつアロー関数構文を使うべきでしょうか？それは状況によります。多くの開発者は個人のプロジェクトで一貫して使っていますが、チームで作業する場合は既存のコードベースが通常の関数かアロー関数かによって選択されます。今後のレッスンで、アロー関数を使うべき場合と避けるべき場合を説明します。
