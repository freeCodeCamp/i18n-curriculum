# --description--

Aggiornare gli oggetti nello stato in React può essere complicato se sei abituato a modificare direttamente i valori delle proprietà degli oggetti.

React considera lo stato come immutabile, il che significa che non dovresti modificarlo direttamente.

Vediamo cosa succede se provi a cambiare direttamente gli oggetti nello stato di React, e poi approfondiamo il modo corretto di farlo.

Supponiamo che tu abbia un oggetto nello stato del tuo componente che rappresenta il profilo di un utente, e vuoi che l'utente aggiorni la sua età:

```jsx
import { useState } from "react";

function Profile() {
  const [user, setUser] = useState({
    name: "John Doe",
    age: 31,
    city: "LA",
  });

  // Change user age directly
  const handleAgeChange = (e) => {
    user.age = e.target.value;
    console.log(user);
  };

  return (
    <div>
      <h1>User Profile</h1>
      <p>Name: {user.name}</p>
      <p>Age: {user.age}</p>
      <p>City: {user.city}</p>

      <h2>Update User Age </h2>
      <input type="number" value={user.age} onChange={handleAgeChange} />
    </div>
  );
}

export default Profile;
```

Questo codice non funzionerà perché stai modificando direttamente la proprietà `user.age`.

Anche se `console.log(user)` mostrerà la nuova età nella console, React non renderizzerà di nuovo il componente per mostrarla nell'interfaccia utente perché non hai usato la funzione setter, `setUser`.

Per aggiornare direttamente un oggetto nello stato, devi usare la funzione setter per creare un nuovo oggetto con il valore aggiornato. Per esempio:

```js
const handleAgeChange = (e) => {
  setUser({
    age: e.target.value,
  });
};
```

Questo funziona. Ma se guardi la pagina ora, l'età dell'utente viene aggiornata, ma i valori per il nome e la città si perdono.

Questo succede perché il nuovo oggetto che hai passato alla funzione setter conteneva solo una coppia chiave/valore per `age`.

Per evitare che ciò accada, puoi prima copiare l'oggetto esistente, poi aggiornare solo la proprietà che vuoi modificare, in questo caso `age`.

Per farlo, puoi passare una funzione speciale chiamata funzione updater alla tua funzione setter, `setUser`. La funzione updater prende lo stato in sospeso come argomento, qui chiamato `prevUser`, e dovrebbe restituire il prossimo stato:

```js
const handleAgeChange = (e) => {
  setUser((prevUser) => {
    const updatedUser = { ...prevUser, age: e.target.value };
    console.log('Previous State:', prevUser);
    console.log('Updated State:', updatedUser);
    return updatedUser;
  });
};
```

Come vedi, creiamo un nuovo oggetto utente chiamato `updatedUser` usando la sintassi di espansione per copiare l'oggetto utente in sospeso, `...prevUser`. Poi aggiorniamo l'età in base all'input del modulo e restituiamo `updatedUser` in fondo alla funzione come prossimo stato.

Ora il tuo progetto funziona come previsto, e gli aggiornamenti all'input dell'età non influenzano il nome o la città dell'utente. Puoi anche vedere gli stati precedente e aggiornato nella console.

Questo è il modo ideale per aggiornare un oggetto nello stato, specialmente quando non stai aggiornando tutte le proprietà.

Per aggiornare le restanti proprietà `name` e `city`, puoi scriverle come funzioni setter separate e collegarle ai rispettivi input:

```js
const handleNameChange = (e) => {
  setUser((prevUser) => ({
    ...prevUser,
    name: e.target.value,
  }));
};

const handleCityChange = (e) => {
  setUser((prevUser) => ({
    ...prevUser,
    city: e.target.value,
  }));
};
```

Oppure puoi combinarle in un'unica funzione setter così:

```js
const handleChange = (e) => {
  const { name, value } = e.target;
  setUser((prevUser) => ({
    ...prevUser,
    [name]: value,
  }));
};
```

Per far funzionare questo, ogni campo di input deve avere un attributo `name`.

Ecco il codice completo ora:

```jsx
import { useState } from "react";

function Profile() {
  const [user, setUser] = useState({ name: "John Doe", age: 31, city: "LA" });

  const handleChange = (e) => {
    const { name, value } = e.target;

    setUser((prevUser) => ({...prevUser, [name]: value}));
  };

  return (
    <div>
      <h1>User Profile</h1>
      <p>Name: {user.name}</p>
      <p>Age: {user.age}</p>
      <p>City: {user.city}</p>

      <h2>Update User Age </h2>
      <input type="number" name="age" value={user.age} onChange={handleChange} />

      <h2>Update User Name </h2>
      <input type="text" name="name" value={user.name} onChange={handleChange} />

      <h2>Update User City </h2>
      <input type="text" name="city" value={user.city} onChange={handleChange} />
    </div>
  );
}

export default Profile;
```
