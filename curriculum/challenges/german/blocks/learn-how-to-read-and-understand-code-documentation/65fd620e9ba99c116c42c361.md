# --description--

Entwickler verwenden oft `once`, um den Zeitpunkt von Aufgaben innerhalb ihres Workflows anzugeben. Es bedeutet `as soon as` oder `when`.

Zum Beispiel `Once you receive the client's feedback, update the project documentation.` Dies bedeutet, dass die Aktualisierung der Dokumentation unmittelbar nach Erhalt des Feedbacks beginnen sollte und zeigt die Folge der Aktionen im Lebenszyklus eines Projekts.
