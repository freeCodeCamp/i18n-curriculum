# --description--

Eine weitere Möglichkeit, Selektoren zu verwenden, besteht darin, sie als Liste ohne Trennung zu verketten. Angenommen, Sie hätten folgendes HTML:

```html
<div>
  <div class="subsection header">Latest Posts</div>
  <p class="subsection preview">This is where a preview for a post might go.</p>
</div>
```

Sie haben zwei Elemente mit der `subsection`-Klasse, die jeweils eigene Stile haben. Aber was, wenn Sie nur eine separate Regel auf das Element anwenden möchten, das zusätzlich die `header`-Klasse als zweite Klasse besitzt? Nun, Sie könnten beide `class`-Selektoren in Ihrem CSS so verketten:

```css
.subsection.header {
  color: red;
}
```

Was `.subsection.header` bewirkt, ist, dass es jedes Element auswählt, das sowohl die `subsection`- als auch die `header`-Klasse besitzt. Beachten Sie, dass zwischen den `.subsection`- und `.header`-`class`-Selektoren kein Leerzeichen steht. Diese Syntax funktioniert grundsätzlich für die Verkettung beliebiger Kombinationen von Selektoren, außer für die Verkettung von mehr als einem Typselektor.

Dies kann auch verwendet werden, um eine Klasse und eine ID zu verketten, wie unten gezeigt:

```html
<div>
  <div class="subsection header">Latest Posts</div>
  <p class="subsection" id="preview">This is where a preview for a post might go.</p>
</div>
```

Sie können die beiden obigen Elemente nehmen und sie mit folgendem kombinieren:

```css
.subsection.header {
  color: red;
}

.subsection#preview {
  color: blue;
}
```

Im Allgemeinen können Sie nicht mehr als einen Typselektor verketten, da ein Element nicht gleichzeitig zwei verschiedene Typen sein kann. Zum Beispiel würde das Verketten von zwei Typselektoren wie `div` und `p` den Selektor `divp` ergeben, der nicht funktionieren würde, da der Selektor versuchen würde, ein tatsächliches `<divp>`-Element zu finden, das nicht existiert.

Angenommen, ein Element hat ein `id` von `title` und ein `class` von `primary`. Wie würden Sie beide Attribute für eine einzelne Regel verwenden?

## --answers--

```css
.title.primary {
  ...
}
```

```css
  ...

```

```css
#title.primary { 
  ...
}
```
