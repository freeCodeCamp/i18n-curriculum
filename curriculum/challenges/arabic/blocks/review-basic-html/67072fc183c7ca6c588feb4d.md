# --interactive--

## أساسيات HTML

- **دور HTML**: تمثل HTML محتوى وهيكل صفحة الويب.
- **عناصر HTML**: العناصر هي اللبنات الأساسية لوثيقة HTML. تمثل العناوين، الفقرات، الروابط، الصور والمزيد. معظم عناصر HTML تتكون من وسم بداية (`<elementName>`) ووسم إغلاق (`</elementName>`).

إليك الصيغة النحوية الأساسية:

```html
<elementName>Content goes here</elementName>
```

- **العناصر الفارغة**: لا يمكن للعناصر الفارغة أن تحتوي على محتوى ولها وسم بداية فقط. من الأمثلة عليها عناصر `img` و`meta`.

```html
<img>
<meta>
```

من الشائع رؤية بعض قواعد الشفرة التي تتضمن الشرطة المائلة الأمامية `/` داخل العنصر الفارغ. كلا الطريقتين مقبولتان:

```html
<img>
<img/>
```

- **الخصائص**: الخاصية هي قيمة توضع داخل وسم البداية لعنصر HTML. توفر الخصائص معلومات إضافية عن العنصر أو تحدد كيف يجب أن يتصرف العنصر. إليك الصيغة الأساسية لخاصية:

```html
<element attribute="value"></element>
```

الخاصية ذات القيمة المنطقية هي خاصية يمكن أن تكون موجودة أو غير موجودة في وسم HTML. إذا كانت موجودة، تكون القيمة true، وإلا فهي false. من أمثلة الخصائص ذات القيمة المنطقية `disabled`، `readonly`، و`required`.

- **التعليقات**: تُستخدم التعليقات في البرمجة لترك ملاحظات لنفسك وللمطورين الآخرين في الشفرة. إليك الصيغة لتعليق في HTML:

```html
<!--This is an HTML comment.-->
```

## عناصر HTML الشائعة

- **عناصر العناوين**: هناك ستة عناصر عناوين في HTML. تُستخدم عناصر العناوين من `h1` إلى `h6` للدلالة على أهمية المحتوى الذي يليها. كلما كان الرقم أقل، كانت الأهمية أكبر، لذا فإن عناصر `h2` أقل أهمية من عناصر `h1`.

:::interactive_editor

```html
<h1>most important heading element</h1>
<h2>second most important heading element</h2>
<h3>third most important heading element</h3>
<h4>fourth most important heading element</h4>
<h5>fifth most important heading element</h5>
<h6>least important heading element</h6>
```

:::

- **عناصر الفقرة**: تُستخدم للفقرات في صفحة الويب.

:::interactive_editor

```html
<p>This is a paragraph element.</p>
```

:::

- **عناصر `img`**: يُستخدم عنصر `img` لإضافة الصور إلى صفحة الويب. تُستخدم الخاصية `src` لتحديد موقع الصورة. من الممارسات الجيدة إضافة خاصية أخرى تسمى `alt`. إليك مثالًا على عنصر `img` مع خاصيتي `src` و`alt`:

:::interactive_editor

```html
<img src="https://cdn.freecodecamp.org/curriculum/cat-photo-app/lasagna.jpg" alt="A slice of lasagna on a plate.">
```

:::

- **عنصر `body`**: يُستخدم هذا العنصر لتمثيل محتوى وثيقة HTML.

:::interactive_editor

```html
<body>
  <h1>CatPhotoApp</h1>
  <p>This is a paragraph element.</p>
</body>
```

:::

- **عناصر `section`**: يُستخدم هذا العنصر لتقسيم المحتوى إلى أقسام أصغر.

:::interactive_editor

```html
<section>
  <h2>About Me</h2>
  <p>Hi, I am Jane Doe and I am a web developer.</p>
</section>
```

:::

- **عناصر `div`**: هذا عنصر HTML عام لا يحمل أي معنى دلالي. يُستخدم كحاوية عامة لاحتواء عناصر HTML أخرى.

:::interactive_editor

```html
<div>
  <h1>I am a heading</h1>
  <p>I am a paragraph</p>
</div>
```

:::

- **عناصر الوسم الرابط (`a`)**: تُستخدم هذه العناصر لتطبيق الروابط على صفحة الويب. تُستخدم الخاصية `href` لتحديد وجهة الرابط عند نقر المستخدم عليه.

:::interactive_editor

```html
<a href="https://cdn.freecodecamp.org/curriculum/cat-photo-app/running-cats.jpg">cute cats</a>
```

:::

- **عناصر القوائم غير المرتبة (`ul`) والمرتبة (`ol`)**: لإنشاء قائمة نقطية من العناصر، يجب استخدام عنصر `ul` مع عنصر أو أكثر من عناصر `li` متسكنة بداخله مثل هذا:

:::interactive_editor

```html
<ul>
  <li>catnip</li>
  <li>laser pointers</li>
  <li>lasagna</li>
</ul>
```

:::

لإنشاء قائمة مرتبة من العناصر، يجب استخدام عنصر `ol`:

:::interactive_editor

```html
<ol>
  <li>flea treatment</li>
  <li>thunder</li>
  <li>other cats</li>
</ol>
```

:::

- **عنصر التأكيد (`em`)**: يُستخدم لوضع تأكيد على جزء من النص.

:::interactive_editor

```html
<p>Cats <em>love</em> lasagna.</p>
```

:::

- **عنصر الأهمية القوية (`strong`)**: يُستخدم لوضع تأكيد قوي على النص للدلالة على الإلحاح والجدية.

:::interactive_editor

```html
<p>
  <strong>Important:</strong> Before proceeding, make sure to wear your safety goggles.
</p>
```

:::

- **عناصر `figure` و`figcaption`**: يُستخدم عنصر `figure` لتجميع محتوى مثل الصور والرسوم البيانية. يُستخدم عنصر `figcaption` لتمثيل تسمية توضيحية لذلك المحتوى داخل عنصر `figure`.

:::interactive_editor

```html
<figure>
  <img src="https://cdn.freecodecamp.org/curriculum/cat-photo-app/cats.jpg" alt="Two tabby kittens sleeping together on a couch.">
  <figcaption>Cats <strong>hate</strong> other cats.</figcaption>
</figure>
```

:::

- **عنصر `main`**: يُستخدم لتمثيل المحتوى الرئيسي لصفحة الويب.
- **عنصر `footer`**: يُوضع هذا العنصر في أسفل وثيقة HTML وعادةً يحتوي على معلومات حقوق النشر وروابط مهمة أخرى للصفحة.

:::interactive_editor

```html
<footer>
  <p>
    No Copyright - <a href="https://www.freecodecamp.org">freeCodeCamp.org</a>
  </p>
</footer>
```

:::

## المعرفات والتجميع

- **المعرفات (IDs)**: معرفات فريدة لعناصر HTML. يجب استخدام أسماء المعرف مرة واحدة فقط في وثيقة HTML.

```html
<h1 id="title">Movie Review Page</h1>
```

لا يمكن لأسماء المعرفات أن تحتوي على فراغات. إذا كان اسم المعرف يحتوي على كلمات متعددة، يمكنك استخدام الشرطات بين الكلمات مثل هذا:

```html
<div id="red-box"></div>
```

- **الأصناف (Classes)**: تُستخدم الأصناف لتجميع العناصر لأغراض التنسيق والسلوك.

```html
<div class="box"></div>
```

على عكس المعرفات، يمكنك إعادة استخدام نفس اسم الصنف في جميع أنحاء وثيقة HTML. يمكن أن تحتوي قيمة `class` أيضًا على فراغات مثل هذا:

```html
<div class="box red-box"></div>
<div class="box blue-box"></div>
```

## الأحرف الخاصة والربط

- **كيانات HTML**: كيان HTML، أو مرجع الحرف، هو مجموعة من الأحرف تُستخدم لتمثيل حرف محجوز في HTML. من الأمثلة على ذلك رمز العطف (`&amp;`) ورمز أقل من (`&lt;`).

```html
<p>This is an &lt;img /&gt; element</p>
```

- **عنصر `link`**: يُستخدم هذا العنصر للربط بالموارد الخارجية مثل أوراق الأنماط وأيقونات الموقع. إليك الصيغة الأساسية لاستخدام عنصر `link` لملف CSS خارجي:

```html
<link rel="stylesheet" href="./styles.css" />
```

تُستخدم الخاصية `rel` لتحديد العلاقة بين المورد المرتبط ووثيقة HTML. تُستخدم الخاصية `href` لتحديد موقع عنوان URL للمورد الخارجي.

- **عنصر `script`**: يُستخدم هذا العنصر لتضمين شفرة قابلة للتنفيذ.

```html
<body>
  <script>
    alert("Welcome to freeCodeCamp");
  </script>
</body>
```

بينما يمكنك كتابة كل شفرة JavaScript داخل وسم `script`، يُعتبر من أفضل الممارسات الربط بملف JavaScript خارجي بدلاً من ذلك. إليك مثالًا على استخدام عنصر `script` للربط بملف JavaScript خارجي:

```html
<script src="path-to-javascript-file.js"></script>
```

تُستخدم الخاصية `src` هنا لتحديد موقع ملف JavaScript الخارجي.

## القالب الجاهز والترميز

- **قالب HTML الجاهز**: هو قالب يتضمن الهيكل الأساسي والعناصر الضرورية التي تحتاجها كل وثيقة HTML.

:::interactive_editor

```html
<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="utf-8" />
    <title>freeCodeCamp</title>
    <link rel="stylesheet" href="./styles.css" />
  </head>
  <body>
    <!--Headings, paragraphs, images, etc. go inside here-->
  </body>
</html>
```

:::

- **`DOCTYPE`**: يُستخدم لإخبار المتصفحات بإصدار HTML الذي تستخدمه.
- **عنصر `html`**: يمثل العنصر الأعلى أو جذر وثيقة HTML. لتحديد لغة الوثيقة، يجب استخدام الخاصية `lang`.
- **عنصر `head`**: يحتوي قسم `head` على بيانات وصفية مهمة وهي معلومات خلف الكواليس يحتاجها المتصفحات ومحركات البحث.
- **عناصر `meta`**: تمثل بيانات وصفية لموقعك. تحتوي على تفاصيل مثل ترميز الأحرف، وكيف يجب أن يعرض تويتر رابط صفحتك، والمزيد.
- **عنصر `title`**: يُستخدم لتعيين النص الذي يظهر في تبويب أو نافذة المتصفح.
- **ترميز الأحرف UTF-8**: UTF-8، أو UCS Transformation Format 8، هو ترميز أحرف معياري يُستخدم على نطاق واسع في الويب. ترميز الأحرف هو الطريقة التي تستخدمها الحواسيب لتخزين الأحرف كبيانات. تُستخدم الخاصية `charset` داخل عنصر `meta` لتعيين ترميز الأحرف إلى UTF-8.

## تحسين محركات البحث والمشاركة الاجتماعية

- **تحسين محركات البحث (SEO)**: هو ممارسة تهدف إلى تحسين صفحات الويب لتصبح أكثر ظهورًا وترتيبًا أعلى في محركات البحث.
- **عنصر Meta (`description`)**: يُستخدم لتوفير وصف قصير لصفحة الويب ولتحسين SEO.

```html
<meta
  name="description"
  content="Discover expert tips and techniques for gardening in small spaces, choosing the right plants, and maintaining a thriving garden."
/>
```

- **وسوم Open Graph**: بروتوكول Open Graph يمكّنك من التحكم في كيفية ظهور محتوى موقعك عبر منصات التواصل الاجتماعي المختلفة مثل فيسبوك ولينكدإن وغيرها. من خلال تعيين هذه الخصائص، يمكنك جذب المستخدمين للنقر والتفاعل مع محتواك. يمكنك تعيين هذه الخصائص عبر مجموعة من عناصر `meta` داخل قسم `head` في HTML.
- **خاصية `og:title`**: تُستخدم لتعيين العنوان الذي يظهر في منشورات وسائل التواصل الاجتماعي.

```html
<meta content="freeCodeCamp.org" property="og:title" />
```

- **خاصية `og:type`**: تُستخدم خاصية `type` لتمثيل نوع المحتوى الذي يتم مشاركته على وسائل التواصل الاجتماعي. من أمثلة هذا المحتوى المقالات، المواقع، الفيديوهات، أو الموسيقى.

```html
<meta property="og:type" content="website" />
```

- **خاصية `og:image`**: تُستخدم لتعيين الصورة المعروضة في منشورات وسائل التواصل الاجتماعي.

```html
<meta
  content="https://cdn.freecodecamp.org/platform/universal/fcc_meta_1920X1080-indigo.png"
  property="og:image"
/>
```

- **خاصية `og:url`**: تُستخدم لتعيين عنوان URL الذي سينقر عليه المستخدمون في منشورات وسائل التواصل الاجتماعي.

```html
<meta property="og:url" content="https://www.freecodecamp.org" />
```

## عناصر الوسائط والتحسين

- **العناصر المستبدلة**: العنصر المستبدل هو عنصر يُحدد محتواه بواسطة مورد خارجي بدلاً من CSS نفسه. مثال على ذلك عنصر `iframe`. `iframe` تعني الإطار المضمّن. هو عنصر مضمّن يُستخدم لتضمين محتوى HTML آخر مباشرة داخل صفحة HTML.

```html
<iframe src="https://www.example.com" title="Example Site"></iframe>
```

يمكنك تضمين الخاصية `allowfullscreen` التي تسمح للمستخدم بعرض الإطار في وضع ملء الشاشة.

```html
<iframe
  src="video-url"
  width="width-value"
  height="height-value"
  allowfullscreen
></iframe>
```

لتضمين فيديو داخل `iframe` يمكنك نسخه مباشرة من خدمات الفيديو الشهيرة مثل YouTube وVimeo، أو تعريفه بنفسك باستخدام الخاصية `src` التي تشير إلى عنوان URL لذلك الفيديو. إليك مثالًا لتضمين دورة شهيرة من freeCodeCamp من YouTube:

:::interactive_editor

```html
<h1>A freeCodeCamp YouTube Video Embedded with the iframe Element</h1>

<iframe
  width="560"
  height="315"
  src="https://www.youtube.com/embed/PkZNo7MFNFg?si=-UBVIUNM3csdeiWF"
  title="YouTube video player"
  allow="accelerometer; autoplay; clipboard-write; encrypted-media; gyroscope; picture-in-picture; web-share"
  referrerpolicy="strict-origin-when-cross-origin"
  allowfullscreen
></iframe>
```

:::

هناك بعض العناصر المستبدلة الأخرى، مثل `video` و`embed`. وبعض العناصر تتصرف كعناصر مستبدلة في ظروف معينة. إليك مثالًا على عنصر `input` مع الخاصية `type` مضبوطة على `image`:

```html
<input type="image" alt="Descriptive text goes here" src="example-img-url">
```

- **تحسين الوسائط**: هناك ثلاث أدوات يجب مراعاتها عند استخدام الوسائط مثل الصور في صفحات الويب: الحجم، الصيغة، والضغط.
- **صيغ الصور**: من أكثر صيغ الملفات شيوعًا PNG وJPG، لكنها لم تعد الصيغ المثالية لخدمة الصور. ما لم تكن بحاجة لدعم المتصفحات القديمة، يجب أن تفكر في استخدام صيغة أكثر تحسينًا مثل WEBP أو AVIF.
- **تراخيص الصور**: الصورة التي تقع ضمن الملكية العامة لا تحمل حقوق نشر ويمكن استخدامها بحرية دون قيود. الصور المرخصة بموجب ترخيص Creative Commons 0 تُعتبر ملكية عامة. قد تُصدر بعض الصور تحت تراخيص متساهلة مثل ترخيص Creative Commons أو ترخيص BSD الذي يستخدمه freeCodeCamp.
- **رسومات SVG**: الرسومات المتجهة القابلة للتوسع تتبع البيانات بناءً على المسارات والمعادلات لرسم النقاط والخطوط والمنحنيات. هذا يعني أن الرسومات المتجهة، مثل SVG، يمكن تكبيرها لأي حجم دون التأثير على الجودة.

## التكامل متعدد الوسائط

- **عناصر `audio` و`video`**: تسمح عناصر `audio` و`video` بإضافة محتوى صوتي ومرئي إلى وثائق HTML. يدعم عنصر `audio` صيغ صوتية شائعة مثل mp3، wav، وogg. يدعم عنصر `video` صيغ mp4، ogg، وwebm.

```html
<audio src="CrystalizeThatInnerChild.mp3"></audio>
```

إذا أردت رؤية مشغل الصوت على الصفحة، يمكنك إضافة عنصر `audio` مع الخاصية `controls`:

:::interactive_editor

```html
<audio src="CrystalizeThatInnerChild.mp3" controls></audio>
```

:::

تمكّن الخاصية `controls` المستخدمين من التحكم في تشغيل الصوت، بما في ذلك ضبط الصوت، الإيقاف المؤقت، أو الاستئناف. الخاصية `controls` هي خاصية منطقية يمكن إضافتها لتمكين عناصر التحكم المدمجة. إذا تم حذفها، فلن تظهر عناصر التحكم.

- **خاصية `loop`**: خاصية `loop` هي خاصية منطقية تجعل الصوت يعاد تشغيله باستمرار.

:::interactive_editor

```html
<audio
  src="https://cdn.freecodecamp.org/curriculum/js-music-player/can't-stay-down.mp3"
  loop
  controls
></audio>
```

:::

- **خاصية `muted`**: عند وجودها في عنصر `audio`، تبدأ الخاصية المنطقية `muted` تشغيل الصوت في حالة كتم.

:::interactive_editor

```html
<audio
  src="https://cdn.freecodecamp.org/curriculum/js-music-player/can't-stay-down.mp3"
  loop
  controls
  muted
></audio>
```

:::

- **عنصر `source`**: بالنسبة لأنواع ملفات الصوت، هناك اختلافات في دعم المتصفحات لكل نوع. لتلبية ذلك، يمكنك استخدام عناصر `source` داخل عنصر `audio` وسيختار المتصفح أول مصدر يفهمه. إليك مثالًا على استخدام عدة عناصر `source` لعنصر `audio`:

```html
<audio controls>
  <source src="audio.ogg" type="audio/ogg" />
  <source src="audio.wav" type="audio/wav" />
  <source src="audio.mp3" type="audio/mpeg" />
</audio>
```

جميع الخصائص التي تعلمناها حتى الآن مدعومة أيضًا في عنصر `video`. إليك مثالًا على استخدام عنصر `video` مع الخصائص `loop`، `controls`، و`muted`:

:::interactive_editor

```html
<video
  src="https://archive.org/download/BigBuckBunny_124/Content/big_buck_bunny_720p_surround.mp4"
  loop
  controls
  muted
></video>
```

:::

- **خاصية `poster`**: إذا أردت عرض صورة أثناء تحميل الفيديو، يمكنك استخدام الخاصية `poster`. هذه الخاصية غير متاحة لعناصر `audio` وفريدة لعنصر `video`.

:::interactive_editor

```html
<video
  src="https://archive.org/download/BigBuckBunny_124/Content/big_buck_bunny_720p_surround.mp4"
  loop
  controls
  muted
  poster="https://peach.blender.org/wp-content/uploads/title_anouncement.jpg?x11217"
  width="620"
></video>
```

:::

## أنواع خاصية الهدف

- **خاصية `target`**: تخبر هذه الخاصية المتصفح أين يفتح عنوان URL لعنصر وسم الرابط. هناك أربع قيم مهمة ممكنة لهذه الخاصية: `_self`، `_blank`، `_parent`، و`_top`. هناك قيمة خامسة تسمى `_unfencedTop` تُستخدم حاليًا لتجربة واجهة برمجة التطبيقات `FencedFrame`. ربما لن تحتاج لاستخدامها بعد.
- **قيمة `_self`**: هذه هي القيمة الافتراضية لخاصية `target`. تفتح الرابط في سياق التصفح الحالي. في معظم الحالات، سيكون ذلك في التبويب أو النافذة الحالية.

```html
<a href="https://freecodecamp.org" target="_self">Visit freeCodeCamp</a>
```

- **قيمة `_blank`**: تفتح الرابط في سياق تصفح جديد. عادةً ما يفتح في تبويب جديد، لكن بعض المستخدمين قد يضبطون متصفحاتهم لفتح نافذة جديدة بدلاً من ذلك.

```html
<a href="https://freecodecamp.org" target="_blank">Visit freeCodeCamp</a>
```

- **قيمة `_parent`**: تفتح الرابط في الأصل (الأب) للسياق الحالي. على سبيل المثال، إذا كان موقعك يحتوي على iframe، فإن قيمة `_parent` في ذلك iframe ستفتح في تبويب/نافذة موقعك، وليس في الإطار المضمّن.

```html
<a href="https://freecodecamp.org" target="_parent">Visit freeCodeCamp</a>
```

- **قيمة `_top`**: تفتح الرابط في أعلى سياق تصفح، فكّر فيها كـ "الأب للأب". تشبه `_parent`، لكن الرابط سيفتح دائمًا في تبويب/نافذة المتصفح كاملة، حتى للأطر المضمّنة المتداخلة.

```html
<a href="https://freecodecamp.org" target="_top">Visit freeCodeCamp</a>
```

## المسارات المطلقة مقابل النسبية

- **تعريف المسار**: المسار هو سلسلة تحدد موقع ملف أو مجلد في نظام الملفات. في تطوير الويب، تتيح المسارات للمطورين الربط بالموارد مثل الصور، أوراق الأنماط، السكربتات، وصفحات الويب الأخرى.
- **صيغة المسار**: هناك ثلاث صيغ رئيسية يجب معرفتها. أولًا الشرطة المائلة، التي يمكن أن تكون شرطة مائلة عكسية (`\`) أو شرطة مائلة أمامية (`/`) حسب نظام التشغيل. ثانيًا النقطة المفردة (`.`). وأخيرًا النقطتان (`..`). تُعرف الشرطة المائلة بأنها "فاصل المسار". تُستخدم للدلالة على فاصل بين أسماء المجلدات أو الملفات. النقطة المفردة تشير إلى الدليل الحالي، والنقطتان تشير إلى الدليل الأب.

```md
public/index.html
./favicon.ico
../src/index.css
```

- **المسار المطلق**: المسار المطلق هو رابط كامل لمورد. يبدأ من الدليل الجذر، ويشمل كل الدلائل الأخرى، وأخيرًا اسم الملف والامتداد. يشير "الدليل الجذر" إلى الدليل الأعلى في التسلسل الهرمي. يتضمن المسار المطلق أيضًا البروتوكول، الذي قد يكون `http`، `https`، أو `file` واسم النطاق إذا كان المورد على الويب. إليك مثالًا لمسار مطلق يربط بشعار freeCodeCamp:

:::interactive_editor

```html
<a href="https://design-style-guide.freecodecamp.org/img/fcc_secondary_small.svg">
  View fCC Logo
</a>
```

:::

- **المسار النسبي**: يحدد المسار النسبي موقع ملف بالنسبة إلى دليل الملف الحالي. لا يتضمن البروتوكول أو اسم النطاق، مما يجعله أقصر وأكثر مرونة للروابط الداخلية ضمن نفس الموقع. إليك مثالًا للربط بصفحة `about.html` من صفحة `contact.html`، وكلاهما في نفس المجلد:

```html
<p>
  Read more on the
  <a href="about.html">About Page</a>
</p>
```

## حالات الرابط

- **`:link`**: هذه هي الحالة الافتراضية. تمثل رابطًا لم يزره المستخدم أو ينقر عليه أو يتفاعل معه بعد. يمكنك اعتبار هذه الحالة كأساس لأنماط جميع الروابط في صفحتك. الحالات الأخرى تبنى فوقها.
- **`:visited`**: تنطبق عندما يكون المستخدم قد زار الصفحة المرتبطة بها سابقًا. افتراضيًا، يتحول لون الرابط إلى البنفسجي، لكن يمكنك الاستفادة من CSS لتوفير مؤشر بصري مختلف للمستخدم.
- **`:hover`**: تنطبق هذه الحالة عندما يحوم المستخدم بمؤشر الفأرة فوق الرابط. تساعد هذه الحالة في جذب انتباه إضافي للرابط، لضمان نية المستخدم الفعلية للنقر عليه.
- **`:focus`**: تنطبق هذه الحالة عندما يكون التركيز على الرابط.
- **`:active`**: تنطبق على الروابط التي يتم تفعيلها بواسطة المستخدم. عادةً يعني ذلك النقر على الرابط بزر الفأرة الأساسي (النقر الأيسر) في معظم الحالات.
