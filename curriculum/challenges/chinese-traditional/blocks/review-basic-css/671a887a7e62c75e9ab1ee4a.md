## CSS 基礎知識

- **什麼是 CSS？**：階層式樣式表（CSS）是一種用於套用樣式到 HTML 元素的標記語言。CSS 用於顏色、背景影像、版面配置等。
- **CSS 規則的基本結構**：CSS 規則由兩個主要部分組成：選擇器和宣告區塊。選擇器是在 CSS 中用來識別並標的特定 HTML 元素以進行樣式設定的樣式。宣告區塊則是為特定選擇器或多個選擇器套用一組樣式。

以下是 CSS 規則的一般語法：

```css
selector {
    property: value;
}
```

- **`meta name="viewport"` 元素**：此 `meta` 元素提供瀏覽器指示，說明如何在不同裝置上控制頁面的尺寸和縮放，特別是在手機和平板電腦上。
- **預設瀏覽器樣式**：每個 HTML 元素都會套用預設瀏覽器樣式。這通常包括像是預設邊距和內距的條款。

## 行內、內部 和 外部 CSS

- **行內 CSS**：這些樣式是直接寫在 HTML 元素內，使用 `style` 屬性。大多數時候你不會使用行內 CSS，因為要遵守關注點分離原則。

這裡是一個行內 CSS 的範例：

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<p style="color: red;">This is a red paragraph.</p>
```

:::

- **內部 CSS**：這些樣式寫在 HTML 文件的 `head` 區段內的 `<style>` 頁籤中。這對於建立簡短的程式碼範例很有用，但通常你不需要使用內部 CSS。
- **外部 CSS**：這些樣式寫在獨立的 CSS 檔案中，並使用 `head` 區段的 `link` 元素連結到 HTML 文件。對大多數專案來說，你會使用外部 CSS 檔案，而非內部或行內 CSS。

## 操作 `width` 和 `height` 屬性

- **`width` 屬性**：此屬性指定元素的寬度。如果你沒有指定寬度，則預設值為 `auto`。這讓瀏覽器根據元素的內容、父元素和顯示型別來決定元素的寬度。
- **`min-width` 屬性**：此屬性指定元素的最小寬度。
- **`max-width` 屬性**：此屬性指定元素的最大寬度。
- **`height` 屬性**：此屬性指定元素的高度。同樣地，高度預設值為 `auto`，表示它會根據內部內容調整。
- **`min-height` 屬性**：此屬性指定元素的最小高度。
- **`max-height` 屬性**：此屬性指定元素的最大高度。

## 不同型別的 CSS 組合器

- **後代組合器**：此組合器用於標的為指定父元素的後代元素。以下範例將標的所有位於 `ul` 元素內的 `li` 項目。

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<ul>
    <li>Example item one</li>
    <li>Example item two</li>
    <li>Example item three</li>
</ul>
```

```css
ul li {
    background-color: yellow;
}
```

:::

- **子代組合器 (`>`)**：此組合器用於選取指定父元素的直接子元素。以下範例將標的所有屬於 `container` 類別的直接子代 `p` 元素。

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<div class="container">
  <p>This will get styled.</p>

  <div>
    <p>This will not get styled.</p>
  </div>
</div>
```

```css
.container > p {
  background-color: black;
  color: white;
}
```

:::

- **下一個兄弟組合器 (`+`)**：此組合器會選取緊接在指定兄弟元素後面的元素。以下範例將選取緊接在 `h2` 元素後面的段落元素。

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<h2>I am a sub heading</h2>

<p>This paragraph element will get a red background.</p>
```

```css
h2 + p {
  background-color: red;
}
```

:::

- **後續兄弟組合器 (`~`)**：此組合器會選取指定元素之後的所有兄弟元素。以下範例將只會為第二個段落元素套用樣式，因為它是唯一與 `ul` 元素為兄弟且擁有相同父元素的元素。

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<div class="container">
  <p>This will not get styled.</p>
  <ul>
    <li>Example item one</li>
    <li>Example item two</li>
    <li>Example item three</li>
  </ul>
  <p>This will get styled.</p>
</div>
<p>This will not get styled.</p>
```

```css
ul ~ p {
  background-color: green;
}
```

:::

## 行內、區塊和行內區塊層級元素

- **行內層級元素**：行內元素只佔用它們所需的寬度，且不會另起新行。這些元素會在內容中流動，允許文字和其他行內元素與它們並排出現。常見的行內元素有 `span`、`anchor` 和 `img` 元素。
- **區塊層級元素**：區塊層級元素預設會從新行開始，並佔據其容器可用的全部寬度，延伸至容器的寬度。一些常見的區塊層級元素有 `div`、`paragraph` 和 `section` 元素。
- **行內-區塊層級元素**：你可以使用 `display` 屬性將元素設定為 `inline-block`。這些元素的行為類似行內元素，但可以像區塊層級元素一樣設定 `width` 和 `height`。

## 外距和內距

- **`margin` 屬性**：此屬性用於在元素的邊框與周圍元素之間套用外部間距。
- **`padding` 屬性**：此屬性用於在元素內部、內容與其邊框之間應用間距。
- **`margin` 簡寫**：你可以指定 1–4 個值來設定邊界的各邊。一個值套用到所有四個邊；兩個值設定 `top` 和 `bottom`，然後是 `right` 和 `left`；三個值設定 `top`、水平（`right` 和 `left`），然後是 `bottom`；四個值設定 `top`、`right`、`bottom`、`left`。
- **`padding` 簡寫**：你可以指定 1–4 個值來設定內距的各邊。一個值套用於所有四邊；兩個值設定 `top` 和 `bottom`，然後是 `right` 和 `left`；三個值設定 `top`、水平（`right` 和 `left`），然後是 `bottom`；四個值設定 `top`、`right`、`bottom`、`left`。

## CSS 特異性

- **行內 CSS 特異性**：行內 CSS 擁有最高的特異性，因為它是直接套用到元素上。它會改寫任何內部或外部的 CSS。行內樣式的特異性值為 (1, 0, 0, 0)。
- **內部 CSS 特異性**：內部 CSS 是在 HTML 文件的 `head` 區段內的 `style` 元素中定義。它的特異性低於行內樣式，但可以改寫外部樣式。
- **外部 CSS 特異性**：外部 CSS 是透過 `head` 區段中的 `link` 元素連結，並寫在獨立的 `.css` 檔案中。它具有最低的特異性，但為較大型專案提供最佳的維護性。
- **通用選擇器 (`*`)**：一種特殊型別的 CSS 選擇器，可匹配文件中的任何元素。它常用於為頁面上的所有元素套用樣式，這對於重設或標準化不同瀏覽器之間的樣式非常有用。通用選擇器具有所有選擇器中最低的特異性值。它對特異性值的所有部分均貢獻 0（0、0、0、0）。
- **型別選擇器**：這些選擇器根據元素的頁籤名稱來標的元素。型別選擇器的特異性相較於其他選擇器較低。型別選擇器的特異性值為 (0, 0, 0, 1)。
- **類別選擇器**：這些選擇器由一個句點（`.`）後接類別名稱所定義。類別選擇器的特異性值為 (0, 0, 1, 0)。這表示類別選擇器可以改寫型別選擇器，但會被 ID 選擇器和行內樣式所改寫。
- **ID 選擇器**：ID 選擇器由井字號符號（`#`）後接 ID 名稱所定義。ID 選擇器具有非常高的特異性，特異性高於型別選擇器和類別選擇器，但低於行內樣式。ID 選擇器的特異性值為 (0, 1, 0, 0)。
- **`!important` 關鍵字**：用來賦予樣式規則最高的優先權，使其能夠改寫該屬性的任何其他宣告。使用時，它會強制瀏覽器套用指定的樣式，無論其他選擇器的特異性為何。你應該謹慎使用 `!important`，因為它可能會讓你的 CSS 更難維護和偵錯。
- **Cascade Algorithm**：一種用來決定當多個樣式針對同一元素時應該套用哪些 CSS 規則的演算法。它確保根據一組明確定義的規則使用最適合的樣式。
- **CSS 繼承**：樣式從父元素傳遞到子元素的處理程序。繼承允許你在文件樹的較高層級定義樣式，並將它們套用到多個元素，而無需為每個元素明確指定。
