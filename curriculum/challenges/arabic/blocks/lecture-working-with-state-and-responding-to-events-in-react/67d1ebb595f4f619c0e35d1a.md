# --description--

تحديث الكائنات في الحالة في React قد يكون معقدًا إذا اعتدت تغيير قيم خصائص الكائن مباشرة.

تعامل React الحالة على أنها غير قابلة للتغيير، مما يعني أنه لا يجب تعديلها مباشرة.

لننظر ماذا يحدث إذا حاولت تغيير الكائنات في حالة React مباشرة، ثم نغوص في الطريقة الصحيحة للقيام بذلك.

لنفترض أن لديك كائنًا في حالة المكوِّن يمثل ملف تعريف المستخدم، وتريد أن يُحدِّث المستخدم عمره:

```jsx
import { useState } from "react";

function Profile() {
  const [user, setUser] = useState({
    name: "John Doe",
    age: 31,
    city: "LA",
  });

  // Change user age directly
  const handleAgeChange = (e) => {
    user.age = e.target.value;
    console.log(user);
  };

  return (
    <div>
      <h1>User Profile</h1>
      <p>Name: {user.name}</p>
      <p>Age: {user.age}</p>
      <p>City: {user.city}</p>

      <h2>Update User Age </h2>
      <input type="number" value={user.age} onChange={handleAgeChange} />
    </div>
  );
}

export default Profile;
```

هذا الشفرة لن تعمل لأنك تعدل خاصية `user.age` مباشرة.

رغم أن `console.log(user)` سيُظهر العمر الجديد في وحدة التحكم، إلا أن React لن تعيد عرض المكوِّن لتظهره في واجهة المستخدم لأنك لم تستخدم دالة التعيين، `setUser`.

لتحديث كائن مباشرة في الحالة، تحتاج إلى استخدام دالة التعيين لإنشاء كائن جديد بالقيمة المحدثة. على سبيل المثال:

```js
const handleAgeChange = (e) => {
  setUser({
    age: e.target.value,
  });
};
```

هذا يعمل. لكن إذا نظرت إلى الصفحة الآن، يتم تحديث عمر المستخدم، لكن قيم الاسم والمدينة تُفقد.

هذا لأن الكائن الجديد الذي مررته إلى دالة التعيين احتوى فقط على زوج مفتاح/قيمة لـ `age`.

لمنع حدوث ذلك، يمكنك نسخ الكائن الموجود أولًا، ثم تحديث الخاصية التي تريد تحديثها فقط، في هذه الحالة، `age`.

للقيام بذلك، يمكنك تمرير دالة خاصة تُسمى دالة التحديث إلى دالة التعيين، `setUser`. تأخذ دالة التحديث الحالة المعلقة كمعلمة، هنا تُسمى `prevUser`، ويجب أن تُرجع الحالة التالية:

```js
const handleAgeChange = (e) => {
  setUser((prevUser) => {
    const updatedUser = { ...prevUser, age: e.target.value };
    console.log('Previous State:', prevUser);
    console.log('Updated State:', updatedUser);
    return updatedUser;
  });
};
```

كما ترى، ننشئ كائن مستخدم جديد يُسمى `updatedUser` باستخدام صيغة النشر لنسخ كائن المستخدم المعلق، `...prevUser`. ثم نُحدِّث العمر بناءً على مُدخل النموذج ونُرجع `updatedUser` في أسفل الدالة كالحالة التالية.

الآن يعمل مشروعك كما هو متوقع، والتحديثات على مُدخل العمر لا تؤثر على اسم المستخدم أو اسم المدينة. يمكنك أيضًا رؤية الحالات السابقة والمحدثة في وحدة التحكم.

هذه هي الطريقة المثالية لتحديث كائن في الحالة، خاصة عندما لا تُحدِّث كل الخصائص.

لتحديث الخصائص المتبقية `name` و `city`، يمكنك كتابتها كدوال تعيين منفصلة وربطها بمُدخلاتها الخاصة:

```js
const handleNameChange = (e) => {
  setUser((prevUser) => ({
    ...prevUser,
    name: e.target.value,
  }));
};

const handleCityChange = (e) => {
  setUser((prevUser) => ({
    ...prevUser,
    city: e.target.value,
  }));
};
```

أو يمكنك دمجها في دالة تعيين واحدة مثل هذا:

```js
const handleChange = (e) => {
  const { name, value } = e.target;
  setUser((prevUser) => ({
    ...prevUser,
    [name]: value,
  }));
};
```

لجعل هذا يعمل، يجب أن يحتوي كل حقل إدخال على خاصية `name`.

إليك الشفرة الكاملة الآن:

```jsx
import { useState } from "react";

function Profile() {
  const [user, setUser] = useState({ name: "John Doe", age: 31, city: "LA" });

  const handleChange = (e) => {
    const { name, value } = e.target;

    setUser((prevUser) => ({...prevUser, [name]: value}));
  };

  return (
    <div>
      <h1>User Profile</h1>
      <p>Name: {user.name}</p>
      <p>Age: {user.age}</p>
      <p>City: {user.city}</p>

      <h2>Update User Age </h2>
      <input type="number" name="age" value={user.age} onChange={handleChange} />

      <h2>Update User Name </h2>
      <input type="text" name="name" value={user.name} onChange={handleChange} />

      <h2>Update User City </h2>
      <input type="text" name="city" value={user.city} onChange={handleChange} />
    </div>
  );
}

export default Profile;
```
