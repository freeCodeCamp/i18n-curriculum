## Загальні помилки в Python

- **SyntaxError**: помилка, яку Python викликає, коли ваш код не відповідає його правилам синтаксису. Наприклад, код `print("Hello there"` призведе до синтаксичної помилки з повідомленням `SyntaxError: '(' was never closed`, бо в коді бракує закриваючої дужки.
- **NameError**: Python викликає `NameError`, коли ви намагаєтеся отримати доступ до змінної або функції, яку не визначили. Наприклад, якщо у вашому коді є рядок `print(username)` без попереднього визначення змінної `username`, ви отримаєте помилку імені з повідомленням `NameError: name 'username' is not defined`.
- **TypeError**: ця помилка виникає, коли ви виконуєте операцію над двома або більше несумісними типами даних. Наприклад, якщо ви спробуєте додати рядок до числа, отримаєте помилку `TypeError: can only concatenate str (not "int") to str`.
- **IndexError**: ви отримаєте `IndexError`, якщо звернетеся до індексу, якого немає у списку або інших послідовностях, таких як кортеж і рядок. Наприклад, у рядку `Hello world` індекс останнього символу — `11`. Якщо ви спробуєте звернутися до символу так: `greet = "hello world"; print(greet[12])`, отримаєте помилку з повідомленням `IndexError: string index out of range`.
- **AttributeError**: Python викликає цю помилку, коли ви намагаєтеся використати метод або властивість, якої немає в об’єкті цього типу. Наприклад, виклик `.append()` для рядка `"hello".append("!")` призведе до помилки з повідомленням `AttributeError: 'str' object has no attribute 'append'`.

## Хороші техніки налагодження в Python

- **Використання функції `print`**: вставлення `print()` інструкцій у різних місцях коду під час налагодження допомагає бачити значення змінних і як проходить потік коду.
- **Використання вбудованого налагоджувача Python (`pdb`)**: Python надає модуль `pdb` для налагодження. Це частина стандартної бібліотеки Python, тому він завжди доступний. За допомогою `pdb` можна встановити трасування методом `set_trace()` для початку покрокового виконання коду та перевірки змінних в інтерактивному режимі.
- **Використання інструментів налагодження IDE**: багато інтегрованих середовищ розробки (IDE) та редакторів коду, таких як Pycharm і VS Code, пропонують інструменти налагодження з точками зупинки, покроковим виконанням, перевіркою змінних та іншими функціями налагодження.

## Обробка виключень

- **`try...except`**: використовується для виконання блоку коду, який може викликати виключення. Блок `try` — це місце, де ви очікуєте, що може виникнути помилка, а блок `except` обробляє вказане виключення, якщо воно виникає. Ось приклад:

  ```py
  try:
    print(22 / 0)
  except ZeroDivisionError:
    print('You can\'t divide by zero!')
    # You can't divide by zero!
  ```

  Ви також можете ланцюжити кілька блоків `except`, щоб обробляти різні типи виключень:

  ```py
  try:
    number = int(input('Enter a number: '))
    print(22 / number)
  except ZeroDivisionError:
    print('You cannot divide by zero!')
    # You cannot divide by zero! prints when you enter 0
  except ValueError:
    print('Please enter a valid number!')
    # Please enter a valid number! prints when you enter a string  
  ```

- **`else` і `finally`**: ці блоки розширюють `try...except`. Якщо виключення не виникає, виконується блок `else`. Блок `finally` виконується завжди, незалежно від помилок.

  ```py
  try:
    result = 100 / 4
  except ZeroDivisionError:
    print('You cannot divide by zero!') # This will not run
  else:
    print(f'Result is {result}') # Result is 25.0
  finally:
    print('Execution complete!') # Execution complete!
  ```

- **Об’єкт виключення**: дозволяє отримати доступ до самого виключення для кращого налагодження та виведення прямого повідомлення про помилку. Щоб отримати об’єкт виключення, потрібно використати ключове слово `as`. Ось приклад:

  ```py
  try:
      value = int('This will raise an error')
  except ValueError as e:
      print(f'Caught an error: {e}')
      # Caught an error: invalid literal for int() with base 10: 'This will raise an error'
  ```

- **Інструкція `raise`**: дозволяє вручну викликати виключення. Ви можете використати її, щоб кинути виключення, коли виконується певна умова. Ось приклад:

  ```py
  def divide(a, b):
      if b == 0:
          raise ZeroDivisionError('You cannot divide by zero')
      return a / b
  ```
  
## Сигналізація виключень
  
  Інструкція `raise` також корисна, коли ви створюєте власні кастомні виключення, оскільки можна кинути виключення з власним повідомленням. Ось приклад:

  ```py
  class InvalidCredentialsError(Exception):
      def __init__(self, message="Invalid username or password"):
          self.message = message
          super().__init__(self.message)

  def login(username, password):
      stored_username = "admin"
      stored_password = "password123"
      
      if username != stored_username or password != stored_password:
          raise InvalidCredentialsError()
      
      return f"Welcome, {username}!"
  ```  

  Ось як можна використати функцію `login` з виключенням `InvalidCredentialsError`:

  ```py
  # failed login attempt
  try:
      message = login("user", "wrongpassword")
  except InvalidCredentialsError as e:
      print(f"Login failed: {e}")
  else:
      print(message)

  # successful login attempt
  try:
      message = login("admin", "password123")
  except InvalidCredentialsError as e:
      # This block is not executed because the login was successful
      print(f"Login failed: {e}")
  else:
      # The else block runs if the 'try' block completes without an exception
      print(message)
  ```

  Інструкція `raise` також може використовуватися з ключовим словом `from` для ланцюжка виключень, показуючи зв’язок між різними помилками:

  ```py
  def parse_config(filename):
    try:
        with open(filename, 'r') as file:
            data = file.read()
            return int(data)
    except FileNotFoundError:
        raise ValueError('Configuration file is missing') from None
    except ValueError as e:
        raise ValueError('Invalid configuration format') from e

  config = parse_config('config.txt')
  ```
