# --description--

У попередньому уроці ви дізналися, як працювати з об’єднаними типами. Один із поширених способів роботи з об’єднаними типами — це використання псевдоніма типу.

Псевдонім типу — це ім’я для типу. Ось приклад, як надати ім’я об’єднаному типу:

```ts
type ID = number | string;
```

Часто у своїх проєктах ви створюватимете псевдоніми типів і використовуватимете їх по всій програмі так:

```ts
type ID = number | string;

type User = {
  id: ID;
  name: string;
};

function getUserId(user: User): ID {
  return user.id;
}
```

У цьому прикладі використано два типи з іменами `ID` і `User`. `ID` буде об’єднаним типом для `number` або `string`, тоді як `User` — це тип об’єкта з властивостями `id` і `name`. `id` визначено з типом `ID`, а `name` має тип `string`.

Функція `getUserId` має один параметр із назвою `user`, і його тип — `User`. Тип повернення функції `getUserId` буде `ID`.

Псевдоніми типів — це чудовий спосіб описати форму об’єкта. Зазвичай створюють типи об’єктів і використовують їх у масивах так:

```ts
type User = {
  id: number | string;
  name: string;
};

const users: User[] = [
  { id: 101, name: "Alice" },
  { id: "A102", name: "Bob" },
  { id: 103, name: "Charlie" }
];
```

У цьому прикладі масив `users` повинен містити лише об’єкти типу `User`. Якщо ви забудете властивість в одному з об’єктів, TypeScript видасть помилку такого вигляду:

```ts
type User = {
  id: number | string;
  name: string;
};

const users: User[] = [
  // Property 'name' is missing in type '{ id: number; }' but required in type 'User'.
  { id: 101 },
  { id: "A102", name: "Bob" },
  { id: 103, name: "Charlie" }
];
```

Ви могли помітити використання PascalCase при іменуванні псевдонімів типів. Це тому, що за конвенцією використання PascalCase допомагає відрізняти типи від змінних. Це також покращує читабельність. Коли ви бачите `User[]`, ви знаєте, що це масив об’єктів типу `User`.
