# --description--

網路是一個充滿豐富資訊和活動的強大場所。但如果你不小心，它也可能是一個危險的地方。外面有許多駭客和網路犯罪分子正在尋找新的方法來入侵網站並危害資料。

你可能聽過一些影響 Meta 和 Microsoft 等數十億美元公司的高評測資料外洩事件。當你從事網頁應用程式開發時，你需要進行徹底的安全測試，以協助識別漏洞和弱點。

以下是在執行安全稽核和測試時需要注意的幾個原則：

- 機密性：這可防止敏感資訊洩漏給非預期的收件人。

- 完整性：這涉及防止惡意使用者修改使用者資訊。

- 驗證：這涉及確認使用者的身份，以確保他們被允許使用該系統。

- 授權：這是判斷已驗證使用者被允許執行哪些動作或被允許存取系統哪些零件的處理程序。

- 可用性：這確保資訊和服務在使用者需要時可供授權使用者使用。

- 不可否認性：這確保寄件者和收件者雙方都有交付證明及寄件者身份驗證。它防止寄件者否認曾經傳送過該資訊。

既然我們了解了安全測試背後的原因，讓我們來看看幾個常見的安全威脅：

- 跨站指令碼攻擊（XSS）：你在之前的課程中已經學過這個。但為了複習，XSS 攻擊發生在攻擊者將惡意指令碼注入到其他使用者瀏覽的網頁中。這些指令碼隨後可以在受害者瀏覽器的上下文中執行，可能會竊取 cookie 和會話資料，或在使用者不知情或未同意的情況下執行其他惡意行為。

- SQL 注入：SQL 是一種用於管理和儲存關聯式資料庫中資訊的語言。SQL 注入允許惡意用戶將惡意程式碼注入資料庫。你將在未來的模組中學習更多關於 SQL、關聯式資料庫和 SQL 注入的內容。

- 拒絕服務（DoS）攻擊：當惡意使用者以大量請求或流量淹沒網站時，會導致伺服器變慢甚至當機，使網站無法供使用者使用。

還有許多其他型別的安全威脅，但你可以使用許多工具來幫助保護你的網頁應用程式並使其更安全。以下是安全測試工具所屬的主要類別：

- 靜態應用程式安全測試：這些工具用於評估應用程式的原始碼，以識別安全漏洞。

- 動態應用程式安全測試（DAST）：這些工具與應用程式的前端介面互動，以發現潛在的安全弱點。與靜態應用程式安全測試不同，DAST 工具無法存取原始碼。

另一種安全測試是滲透測試（簡稱 pentest）。這涉及對應用程式進行模擬的網路攻擊，以識別系統中的任何漏洞。中大型公司可能會有一支專門的資安團隊，定期執行 pentest 作為其例行安全稽核的一部分。

網路安全的世界非常廣闊且細緻。但每個開發者都應該了解一些針對網頁應用程式的常見威脅以及防範方法。
