# --description--

Робота над цими завданнями передбачає написання коду одним із таких способів:

- Клонувати <a href="https://github.com/freeCodeCamp/boilerplate-npm/" target="_blank" rel="noopener noreferrer nofollow">цей репозиторій GitHub</a> і виконувати завдання локально, використовуючи <a href="https://www.freecodecamp.org/news/how-to-run-the-freecodecamp-backend-challenges-locally/" target="_blank" rel="noopener noreferrer nofollow">цей посібник</a> для налаштування Node.js, npm і запуску бекенд-проєктів на вашому комп’ютері.
- Використовувати будь-який конструктор сайтів на свій вибір для виконання проєкту. Обов’язково включіть усі файли з нашого репозиторію GitHub.

Файл `package.json` є центром будь-якого проєкту Node.js або пакета npm. Він зберігає інформацію про ваш проєкт. Складається з одного JSON-об’єкта, де інформація зберігається у парах ключ-значення. Є лише два обов’язкові поля: `name` і `version`, але рекомендується додавати додаткову інформацію.

Ви можете створити файл `package.json` з терміналу за допомогою команди `npm init`. Це запустить покрокове налаштування. Використання `npm init` з прапорцем `-y` згенерує файл без запитань, `npm init -y`.

Якщо подивитися на дерево файлів вашого проєкту, ви знайдете файл `package.json` на верхньому рівні дерева. Саме цей файл ви будете покращувати у наступних кількох завданнях.

Одним із найпоширеніших елементів інформації в цьому файлі є поле `author`. Воно вказує, хто створив проєкт, і може бути рядком або об’єктом із контактними чи іншими деталями. Для більших проєктів рекомендується об’єкт, але для цього проєкту підійде простий рядок, як у наведеному прикладі.

```json
"author": "Jane Doe",
```
