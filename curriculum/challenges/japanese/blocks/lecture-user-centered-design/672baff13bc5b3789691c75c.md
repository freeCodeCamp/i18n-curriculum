# --interactive--

**注意**: いくつかのインタラクティブな例では、まだ学んでいないCSSが使われている場合があります。コードのすべてを理解しようと心配しないでください。これらの例の目的は、デザインの概念をプレビューで示し、仕組みをよりよく理解してもらうことです。プレビューを見るには、インタラクティブエディターを有効にしてください。

ウェブデザインには、トップナビゲーションバー、サイドバー、フッターなど、さまざまな種類のナビゲーション補助があります。しかし、サイトが複雑で階層が深い場合は、パンくずリストの使用を検討するとよいでしょう。

パンくずリストは、ユーザーがサイトの階層のどこにいるかを示すナビゲーション補助です。以下は、架空の電子機器サイトのパンくずリストの例です。

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<div class="breadcrumbs">
  Home / Electronics / Phones / Smartphone XYZ
</div>
```

```css
.breadcrumbs {
  font-size: 0.9rem;
  color: #555;
  margin: 1rem 0;
}
```

:::

ほとんどのウェブサイトでは、パンくずリストはページの上部に表示され、ユーザーが現在のページにたどり着くまでの経路を示します。`Homepage`から始まり、ユーザーは`Electronics`カテゴリー、次に`Phones`カテゴリー、そして最後に`Smartphone XYZ`製品へと進みました。商品や特定の情報を検索しているときに、パンくずリストに触れたことがあるはずです。

パンくずリストの利用は、ユーザーがサイトの階層のどこにいるかを理解し、前のページに戻る方法を知るのに役立ちます。特に、検索結果や外部リンクから来たユーザーが、現在のページの文脈を理解するのに便利です。

パンくずリストを設計するときに考慮すべき点がいくつかあります。まず、区切り文字を決めることです。区切り文字は階層の異なるレベルを分ける記号です。一般的な区切り文字には、大なり記号（`>`）、右向きの引用符（`»`）、スラッシュ（`/`）があります。

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<div class="breadcrumbs">
  Home &gt; Electronics &gt; Phones &gt; Smartphone XYZ
</div>
```

```css
.breadcrumbs {
  font-size: 0.9rem;
  color: #555;
  margin: 1rem 0;
}
```

:::

次に考慮すべきはパンくずリストの配置です。パンくずリストは通常、ページの上部に配置され、メインナビゲーションバーの上か下に置かれます。ユーザーがパンくずリストを見つけるのに苦労しないよう、見やすく分かりやすい場所に配置してください。

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<nav class="main-nav">
  <a href="#">Home</a>
  <a href="#">Electronics</a>
  <a href="#">Phones</a>
</nav>
<div class="breadcrumbs">
  Home &gt; Electronics &gt; Phones &gt; Smartphone XYZ
</div>
```

```css
.main-nav {
  background-color: #333;
  padding: 1rem;
}

.main-nav a {
  color: white;
  margin-right: 1rem;
  text-decoration: none;
}

.breadcrumbs {
  font-size: 0.9rem;
  color: #555;
  margin: 1rem 0;
}
```

:::

もう一つの考慮点はパンくずリストのサイズです。パンくずリストは読みやすい大きさにしつつ、ページのスペースを取りすぎないようにします。パンくずリストは主要なナビゲーションツールではなく、あくまで補助的なナビゲーション補助であることを覚えておいてください。情報量が多いページでは、ユーザーが階層の位置を簡単に把握し、前のページに戻るのに役立ちます。

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<div class="breadcrumbs">
  Home &gt; Electronics &gt; Phones &gt; Smartphone XYZ
</div>
<p>
  Lorem ipsum dolor sit amet, consectetur adipiscing elit. Sed do eiusmod tempor incididunt ut labore et dolore magna aliqua.
</p>
```

```css
.breadcrumbs {
  font-size: 1rem;
  color: #555;
  margin: 1rem 0;
}   

p {
  font-size: 1.2rem;
  line-height: 1.6;
}
```

:::

では、いつパンくずリストをサイトで使うべきでしょうか？サイトがシンプルで階層が浅い場合は、パンくずリストは必要ないかもしれません。階層が浅いとは、ナビゲーションのレベルが少ないことを意味し、パンくずリストの価値があまりない場合があります。一般的に、パンくずリストはeコマースサイト、ニュースサイト、技術文書サイトのような階層が深いサイトでよく見られます。
