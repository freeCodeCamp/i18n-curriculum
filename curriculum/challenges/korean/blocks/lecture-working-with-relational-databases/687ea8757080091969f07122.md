# --description--

SQL은 Structured Query Language의 약자입니다. 1970년대에 처음 개발되었으며, 당시에는 structured English query language (SEQUEL)로 알려졌습니다. 이 용어는 나중에 SQL로 축약되었습니다.

SQL은 관계형 데이터베이스에서 데이터를 저장하고 관리하는 데 사용되는 프로그래밍 언어입니다.

이전 수업에서 배운 것처럼, 관계형 데이터베이스는 데이터를 표로 구성합니다. 각 표는 행과 열로 이루어져 있으며, 데이터 속성과 그 관계를 나타냅니다. 각 행은 하나의 레코드를 나타내고, 각 열은 해당 레코드의 특정 속성 또는 필드를 나타냅니다.

SQL을 사용하면 관계형 데이터베이스에서 데이터를 추가, 변경, 삭제, 검색 및 조회할 수 있습니다. 데이터 쿼리, 필터링, 정렬, 집계에 필요한 포괄적인 명령어 세트를 제공합니다.

SQL은 필요한 데이터만 쿼리하여 데이터베이스 성능을 최적화하는 데 도움을 주며, 다양한 프로그래밍 언어와 쉽게 통합되어 애플리케이션에서 데이터베이스와 직접 상호작용할 수 있습니다.

또한 산업 표준으로 채택되어 데이터 무결성을 강제하고, 사용자 인증과 데이터 암호화 같은 많은 보안 기능을 포함합니다.

확장 가능하고, 이식성이 뛰어나며, 다양한 데이터베이스 관리 시스템과 호환 가능합니다.

SQL은 일반적으로 SQL 문(statement) 또는 SQL 쿼리(query)라고 하는 명령어를 기반으로 합니다.

개발자는 특정 SQL 언어 요소나 키워드를 사용해 이러한 명령어를 작성합니다. 이를 통해 데이터베이스에서 필요한 작업을 수행할 수 있습니다.

SQL 명령어를 입력하기 전에, PostgreSQL 데이터베이스와 직접 상호작용할 수 있도록 psql 셸을 열어야 합니다. 명령 프롬프트나 터미널에 다음을 입력하세요:

```bash
psql -U <username> -d <database_name>
```

`username`를 사용자 이름으로, `database_name`을 연결하려는 데이터베이스 이름으로 바꾸세요.

아직 데이터베이스를 만들지 않았다면, PostgreSQL은 설치 시 기본 데이터베이스인 `postgres`를 자동으로 포함합니다. 이 `postgres` 데이터베이스를 따라 하기에 사용할 수 있습니다.

연결되면 프롬프트가 다음과 같이 바뀝니다:

```bash
postgres=#
```

연결된 데이터베이스 이름을 보여주며 SQL 명령어를 기다립니다.

가장 기본적인 SQL 명령어 몇 가지를 살펴보겠습니다. 관례상 SQL 명령어는 세미콜론(`;`)으로 끝난다는 점을 알게 될 것입니다.

먼저, `my_database`라는 이름의 데이터베이스를 생성하는 명령어는 다음과 같습니다:

```sql
CREATE DATABASE my_database;
```

`CREATE` 다음에 `DATABASE`과 생성하려는 데이터베이스 이름을 입력하세요. 명령어의 SQL 키워드는 대문자로 작성된다는 점에 주의하세요.

데이터베이스를 만든 후에는 연결할 수 있습니다.

데이터베이스에 연결하는 방법은 사용하는 환경과 도구에 따라 다릅니다.

명령 프롬프트나 터미널에서는 앞서 보여준 방법을 사용할 수 있습니다. 대화형 psql 셸에서는 `\c` 단축 명령어 다음에 데이터베이스 이름을 입력해 데이터베이스를 전환할 수 있습니다:

```bash
\c my_database
```

`\c` 명령어와 역슬래시가 앞에 붙은 다른 단축 명령어들은 psql 셸의 일부이며 SQL 언어 자체의 일부가 아니므로, 명령어를 완료할 때 세미콜론이 필요하지 않습니다.

새 데이터베이스에 연결하면 프롬프트가 다음과 같이 바뀝니다:

```bash
my_database=#
```

데이터베이스를 생성하고 연결한 후에는 `CREATE TABLE` 키워드를 사용해 테이블을 만들 수 있습니다:

```sql
CREATE TABLE products (
  id SERIAL,
  name VARCHAR(255)
);
```

명령어에서 `CREATE TABLE` 다음에 생성하려는 테이블 이름을 작성하세요. 이 예에서는 `products`라는 이름의 테이블을 생성합니다.

그다음 소괄호 안에 테이블이 가져야 할 열 이름을 작성하고, 각 열에 저장될 값의 데이터 유형을 지정합니다.

예제에서는 제품 ID용 열과 제품 이름용 열 두 개가 있습니다.

테이블과 열 이름의 표준 명명 관례는 스네이크 케이스로, 단어를 소문자로 작성하고 밑줄로 구분합니다(`_`). 예를 들어: `delivery_orders`

시작하는 데 알아야 할 필수 SQL 명령어 몇 가지입니다.

SQL을 사용하면 데이터를 쉽게 쿼리하고 조작하며 분석하여 정보에 기반한 결정을 내리고 실제 문제를 해결할 수 있습니다.
