# --interactive--

プログラミングにおいて、継承はあるクラスが他のクラスからプロパティやメソッドを受け継ぐことを可能にします。

例えば、`Car`は`Vehicle`の一種なので、すべての種類の車両に共通する最も一般的なプロパティやメソッドを持つ`Vehicle`クラスを定義し、その後`Car`クラスのプロパティやメソッドを継承する`Vehicle`クラスを定義できます。

この「～は～である」という関係が継承の特徴です。この例では、`Car`は`Vehicle`のより専門的な形態です。

継承を使い始めるために知っておくべき主な概念は、親クラスと子クラスの2つです。

親クラスは他のクラスの設計図のように機能するクラスです。親クラスは他のクラスに継承されるプロパティやメソッドを定義します。

子クラスは他のクラスのプロパティやメソッドを継承するクラスです。子クラスは新しいプロパティやメソッドを追加して親クラスの機能を拡張することもできます。

前の例では、`Vehicle`は`Car`クラスの親クラスであり、`Car`クラスは`Vehicle`の子クラスです。

親クラスは複数の子クラスを持つことができます。

JavaScriptでは、継承を実装するために`extends`キーワードを使います。このキーワードはあるクラスが別のクラスの子クラスであることを示します。

この例では、クラス定義の最初の行だけを示していますが、`Car`クラスが`Vehicle`クラスを拡張しているので、`Car`は`Vehicle`から継承しています。

```js
class Vehicle {
  // Implementation of Vehicle...
}

class Car extends Vehicle {
  // Implementation of Car...
}
```

`Vehicle`クラスには`brand`と`year`という2つのプロパティがあるとしましょう。更新されたコードでこれが確認できます。

`Car`には`honk`というメソッドがあります。ただし、`Car`クラスには独自のコンストラクタがありません。プロパティを継承しているか見てみましょう。

```js
class Vehicle {
  constructor(brand, year) {
    this.brand = brand;
    this.year = year;
  }
}

class Car extends Vehicle {
  honk() {
    console.log("Honk! Honk!");
  }
}
```

`Car`のインスタンスを定義するには、`brand`と`year`という2つの引数を渡す必要があります。この場合、ブランドは`freeCodeCamp Motors`で、年式は`2019`です。

```js
let myCar = new Car("freeCodeCamp Motors", 2019);
```

これらのプロパティにアクセスし、ドット記法を使ってメソッドを呼び出そうとすると：

```js
console.log(myCar.brand);
console.log(myCar.year);
myCar.honk();
```

完全な例はこちらです：

:::interactive_editor

```js
class Vehicle {
  constructor(brand, year) {
    this.brand = brand;
    this.year = year;
  }
}

class Car extends Vehicle {
  honk() {
    console.log("Honk! Honk!");
  }
}

let myCar = new Car("freeCodeCamp Motors", 2019);
console.log(myCar.brand);
console.log(myCar.year);
myCar.honk();
```

:::

確かに、出力は正しいので、これらのプロパティとメソッドは`Car`インスタンスに定義されており、`Vehicle`クラスからこれらのプロパティを継承することで多くのコードの繰り返しを省けました。

この例では、子クラスに追加のプロパティがなかったため、コンストラクタはなくメソッドだけがありました。

もし追加のプロパティを加える必要がある場合は、コンストラクタを定義する必要があります。

例を見てみましょう。


```js
class Vehicle {
  constructor(brand, year) {
    this.brand = brand;
    this.year = year;
  }
}

class Car extends Vehicle {
  constructor(brand, year, numDoors) {
    super(brand, year);
    this.numDoors = numDoors;
  }
}
```

今回は`numDoors`プロパティを`Car`サブクラスに追加します。

更新されたコードを見ると、このクラスにはコンストラクタがあることがわかります。コンストラクタ内では`super()`が呼び出され、`brand`と`year`という引数が渡されています。

`super()`はスーパークラスのコンストラクタを呼び出しているので、`super(brand, year)`を使うことでサブクラス内でスーパークラスのプロパティを定義していることになります。これは別の用語で、スーパークラスは親クラス、サブクラスは子クラスです。

次に、`this.numDoors = numDoors`行は`Car`クラスに排他的に属する新しいプロパティ、ドアの数（`numDoors`）を定義しています。

これは`Vehicle`クラスを拡張し、`Car`インスタンスに排他的なプロパティを追加する方法です。

これが継承の基本的な利点の一つである拡張性の例です。

今、`Car`クラスのインスタンスを作成すると、`brand`、`year`、`numDoors`という3つのプロパティを持っていることがわかります。インスタンスを作成するときに必要な引数も渡す必要があります。

```js
let myCar = new Car("freeCodeCamp Motors", 2019, 4);
```

これらのプロパティの値を`console.log()`で出力すると：

```js
console.log(myCar.brand);
console.log(myCar.year);
console.log(myCar.numDoors);
```

完全な例はこちらです：

:::interactive_editor

```js
class Vehicle {
  constructor(brand, year) {
    this.brand = brand;
    this.year = year;
  }
}

class Car extends Vehicle {
  constructor(brand, year, numDoors) {
    super(brand, year);
    this.numDoors = numDoors;
  }
}

let myCar = new Car("freeCodeCamp Motors", 2019, 4);
console.log(myCar.brand);
console.log(myCar.year);
console.log(myCar.numDoors);
```

:::

継承の主な利点は、コードの再利用性、モジュール性、拡張性、そしてコード構造の改善です。

階層構造を実装することで、親クラスに書いたコードを子クラスで再利用でき、繰り返しを避けられます。

継承はまた、複雑なシステムを階層内のより単純なコンポーネントに分割することでモジュール性を促進します。

さらに、親クラスの機能を拡張できることで、要件の変化に対応しやすくなり、開発の後半で新しい機能を追加しやすくなります。

最後に、階層構造はコードの理解や保守を容易にします。

これらが継承の基本です。継承の仕組みを理解することで、構造が良く保守しやすく拡張可能なオブジェクト指向プログラムを設計できます。
