## Nambari za hali za jibu la HTTP

- **Nambari za hali** ni ujumbe mfupi wa nambari unaomwambia mteja kilichotokea baada ya ombi kufanywa (kwa mfano, 200, 404, au 500).

### 1xx — Taarifa

- **100 Endelea** — seva inamwambia mteja atume sehemu iliyobaki ya ombi.
- **101 Kubadilisha itifaki** — seva inabadilisha itifaki (hii huonekana mara nyingi na WebSockets).

### 2xx — Mafanikio

- **200 Sawa** — ombi limefanikiwa.
- **201 Imetengenezwa** — rasilimali mpya imetengenezwa kwa mafanikio.
- **204 Hakuna maudhui** — ombi limefanikiwa lakini hakuna mwili wa jibu (hufanywa mara nyingi kwa `DELETE`).

### 3xx — Elekezo upya

- **301 Imesogezwa kwa kudumu** — rasilimali ina URL mpya ya kudumu.
- **302 Imepatikana** — rasilimali iko kwa muda katika URL tofauti.
- **304 Haijabadilika** — mteja anaweza kutumia toleo lake lililohifadhiwa; hakuna kilichobadilika.

### 4xx — Makosa ya mteja

- **400 Ombi baya** — ombi limeandikwa vibaya au halikubaliki.
- **401 Hauruhusiwi** — uthibitisho wa utambulisho unahitajika.
- **403 Imekwisha ruhusiwa** — imethibitishwa, lakini haina ruhusa.
- **404 Haipatikani** — rasilimali iliyotakiwa haipo.

### 5xx — Makosa ya seva

- **500 Hitilafu ya ndani ya seva** — hitilafu ya kawaida upande wa seva.
- **502 Mlango mbaya** — wakala alipokea jibu lisilo halali kutoka seva nyingine.
- **503 Huduma haipatikani** — seva imejaa au haifanyi kazi kwa muda.

### Kutumia nambari za hali katika Express

- `res.status(400).send("Bad Request")` — inatuma 400 na ujumbe wa maandishi rahisi.
- `res.status(200).json({ message: "Success" })` — inatuma 200 na mwili wa JSON.
- `res.status(500).send("Internal Server Error")` — inatuma 500 na ujumbe.

## Kushughulikia hitilafu katika Express

- **Programu ya kati ya kushughulikia hitilafu** ina vigezo vinne: `(err, req, res, next)`. Saini ya vigezo vinne ndio inayoashiria kwa Express kuwa hii ni mshughulikiaji wa hitilafu.
- Weka programu ya kati ya kushughulikia hitilafu **mwishoni** mwa safu ya programu za kati, baada ya njia zote.

```javascript
app.use((err, req, res, next) => {
  console.error(err.message);
  res.status(500).send("Internal Server Error");
});
```

### Kushughulikia hitilafu zisizo za wakati halisi katika Express 4

- Hitilafu zisizo za wakati halisi hazikamatwi moja kwa moja katika Express 4.
- Wazungushe mantiki zisizo za wakati halisi katika `try/catch` na fanya wito wa `next(err)` ili kupeleka hitilafu kwa mshughulikiaji wa hitilafu.

```javascript
app.get("/user", async (req, res, next) => {
  try {
    const user = await getUserFromDatabase();
    res.send(user);
  } catch (err) {
    next(err);
  }
});
```

- Kipakiaji cha `asyncHandler` kinaepuka kurudia `try/catch` katika kila njia:

```javascript
function asyncHandler(fn) {
  return function (req, res, next) {
    Promise.resolve(fn(req, res, next)).catch(next);
  };
}
```

### Kushughulikia hitilafu zisizo za wakati halisi katika Express 5

- Katika Express 5, hitilafu zisizo za wakati halisi na ahadi zilizokataliwa hupelekwa moja kwa moja kwa mshughulikiaji wa hitilafu — hakuna haja ya `next(err)` kwa mkono.

### Kushughulikia makosa ya 404

- Ongeza programu ya kati ya kukamata yote **baada ya njia lakini kabla ya mshughulikiaji wa hitilafu** kushughulikia njia zisizolingana:

```javascript
app.use((req, res, next) => {
  res.status(404).send("Sorry, that route does not exist.");
});
```

### Mbinu bora za kushughulikia hitilafu

- Daima weka programu ya kati ya kushughulikia hitilafu mwisho.
- Shughulikia makosa ya 404 tofauti na makosa mengine.
- Katika uzalishaji, andika maelezo ya hitilafu ndani — usitume kamwe mfuatiliaji wa makosa kwa watumiaji.

## Kurekebisha makosa na kuandika kumbukumbu katika Express

- **Kurekebisha makosa iliyojengwa ndani**: weka kigezo cha mazingira cha `DEBUG` kuwezesha matokeo ya kurekebisha makosa ya Express yaliyojengwa ndani.
  - `DEBUG=express:* node index.js` — inawasha ujumbe wote wa kurekebisha makosa wa Express.
  - `DEBUG=express:router node index.js` — inaonyesha tu matokeo ya kurekebisha makosa yanayohusiana na kipanga-njia.
  - PowerShell: `$env:DEBUG = "express:*"; node index.js`
  - CMD: `set DEBUG=express:* && node index.js`
- **`morgan`**: programu maarufu ya kati ya kuandika kumbukumbu za maombi ya HTTP (njia, URL, nambari ya hali, muda wa jibu).

```javascript
const morgan = require('morgan');
app.use(morgan('dev'));
```

- **Kurekebisha makosa** hufuata jinsi Express inavyoshughulikia maombi na programu za kati bila kuingiza kumbukumbu za uzalishaji.
- **Kuandika kumbukumbu** hurekodi kinachotokea katika maombi — ni muhimu kugundua hitilafu na kuelewa trafiki.

## Ukaguzi wa afya na kufunga kwa heshima

### Ukaguzi wa afya

- **Njia ya ukaguzi wa afya** inaripoti kama programu inaendeshwa vizuri.
- Mizigo ya kusawazisha na watendaji (kwa mfano, Kubernetes, AWS Elastic Beanstalk) hupiga njia hii kuamua kama huduma iko katika hali nzuri.
- Jibu la **200** lina maana ya afya nzuri; kutopata jibu husababisha kuanzisha upya moja kwa moja.

```javascript
app.get('/health', (req, res) => {
  res.status(200).send('OK');
});
```

### Kufunga kwa heshima

- **Kufunga kwa heshima** kunaruhusu programu kumaliza maombi yaliyopo kabla ya kusitisha, badala ya kukata muunganisho mara moja.
- Sikiliza ishara ya `SIGTERM` (inapotumwa wakati mchakato unapaswa kumalizika) na fanya wito wa `server.close()`:

```javascript
process.on('SIGTERM', () => {
  server.close(() => {
    console.log('Server closed. Cleaning up...');
  });
});
```

- Sikiliza ishara ya `SIGINT` (inapotumwa wakati msanidi programu anabonyeza Ctrl+C wakati wa ukuzaji wa ndani) na fanya mantiki ile ile ya kufunga kama `SIGTERM`.

### Kuweka programu kuwa haiko katika hali nzuri kabla ya kufunga

- Rudisha **503** kutoka kwa ukaguzi wa afya wakati wa kufunga ili mizigo ya kusawazisha isizuie trafiki mpya kwenda kwa programu.

```javascript
let isShuttingDown = false;

app.get('/health', (req, res) => {
  if (isShuttingDown) {
    res.status(503).send('Shutting down');
  } else {
    res.status(200).send('OK');
  }
});

process.on('SIGTERM', () => {
  isShuttingDown = true;
  server.close(() => {
    console.log('Graceful shutdown complete');
  });
});
```
