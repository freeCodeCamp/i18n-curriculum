## Archiviazione persistente

- **Definizione**: L'archiviazione persistente indica un modo di salvare i dati in modo che rimangano disponibili anche dopo che l'alimentazione è stata spenta o il dispositivo è stato riavviato.

## Creare, leggere, aggiornare, eliminare (CRUD)

- **Creare**: Si riferisce al processo di creare nuovi dati. Per esempio, in un'app web, questo può essere quando un utente aggiunge un nuovo post a un blog.
- **Leggere**: Questa è l'operazione in cui i dati vengono recuperati da un database. Per esempio, quando visiti un post del blog o visualizzi il tuo profilo su un sito web, stai eseguendo un'operazione di lettura per recuperare e mostrare i dati memorizzati nel database.
- **Aggiornare**: Questo comporta la modifica di dati esistenti nel database. Un esempio è modificare un post del blog o aggiornare le informazioni del profilo.
- **Eliminare**: Questa è l'operazione che rimuove dati da un database. Per esempio, quando elimini un post del blog o un account, stai eseguendo un'operazione di eliminazione.

## Metodi HTTP

- **Definizione**: HTTP sta per Hypertext Transfer Protocol ed è la base per la comunicazione dei dati sul web. Esistono metodi HTTP che definiscono le azioni che possono essere eseguite sulle risorse tramite il web. I metodi comuni sono GET, POST, PUT, PATCH, DELETE.
- **Metodo `GET`**: Serve per recuperare dati da un server.
- **Metodo `POST`**: Serve per inviare dati a un server che crea una nuova risorsa.
- **Metodo `PUT`**: Serve per aggiornare una risorsa sostituendola completamente.
- **Metodo `PATCH`**: Serve per aggiornare parzialmente una risorsa.
- **Metodo `DELETE`**: Serve per rimuovere record da un database.

## Proprietà `localStorage` e `sessionStorage`

- **Web Storage API**: Questa API fornisce un meccanismo per i browser per archiviare coppie chiave-valore direttamente nel browser, permettendo agli sviluppatori di conservare informazioni utilizzabili attraverso diversi ricaricamenti di pagina e sessioni. I due componenti principali della Web Storage API sono le proprietà `localStorage` e `sessionStorage`.
- **Proprietà `localStorage`**: `localStorage` è la parte della Web Storage API che permette ai dati di persistere anche dopo che la finestra del browser è stata chiusa o la pagina è stata aggiornata. Questi dati rimangono disponibili finché non vengono esplicitamente rimossi dall'applicazione o dall'utente.
- **Metodo `localStorage.setItem()`**: Questo metodo serve per memorizzare una coppia chiave-valore in `localStorage`.

```js
localStorage.setItem('username', 'Jessica');
```

- **Metodo `localStorage.getItem()`**: Questo metodo serve per recuperare il valore di una chiave data da `localStorage`.

```js
localStorage.setItem('username', 'codingRules');

let username = localStorage.getItem('username');
console.log(username); // codingRules
```

- **Metodo `localStorage.removeItem()`**: Questo metodo serve per rimuovere un elemento specifico da `localStorage` usando la sua chiave.

```js
localStorage.removeItem('username');
```

- **Metodo `localStorage.clear()`**: Questo metodo serve per cancellare tutti i dati memorizzati in `localStorage`.

```js
localStorage.clear();
```

- **Proprietà `sessionStorage`**: Memorizza dati che durano solo per la sessione corrente e vengono cancellati quando la scheda o la finestra del browser viene chiusa.
- **Metodo `sessionStorage.setItem()`**: Questo metodo serve per memorizzare una coppia chiave-valore in `sessionStorage`.

```js
sessionStorage.setItem('cart', '3 items');
```

- **Metodo `sessionStorage.getItem()`**: Questo metodo serve per recuperare il valore di una chiave data da `sessionStorage`.

```js
sessionStorage.setItem('cart', '3 items');

let cart = sessionStorage.getItem('cart');
console.log(cart); // '3 items'
```

- **Metodo `sessionStorage.removeItem()`**: Questo metodo serve per rimuovere un elemento specifico da `sessionStorage` usando la sua chiave.

```js
sessionStorage.removeItem('cart');
```

- **Metodo `sessionStorage.clear()`**: Questo metodo serve per cancellare tutti i dati memorizzati in `sessionStorage`.

```js
sessionStorage.clear();
```

## Lavorare con i cookie

- **Definizione**: I cookie, noti anche come cookie web o cookie del browser, sono piccoli pezzi di dati che un server invia al browser web di un utente. Questi cookie vengono memorizzati sul dispositivo dell'utente e inviati nuovamente al server con le richieste successive. I cookie sono essenziali per aiutare le applicazioni web a mantenere lo stato e ricordare le informazioni dell'utente, cosa particolarmente importante dato che HTTP è un protocollo senza stato.
- **Cookie di sessione**: Questi cookie durano solo per la durata della sessione dell'utente sul sito web. Una volta che l'utente chiude il browser o la scheda, il cookie di sessione viene eliminato. Questi cookie sono tipicamente usati per attività come mantenere un utente connesso durante la visita.
- **Cookie sicuri**: Questi cookie vengono inviati solo tramite HTTPS, garantendo che non possano essere intercettati da un attaccante durante il trasferimento.
- **Cookie HttpOnly**: Questi cookie non possono essere consultati o modificati da JavaScript in esecuzione nel browser, rendendoli più sicuri contro attacchi di cross-site scripting (XSS).
- **Intestazione Set-Cookie**: Quando visiti un sito web, il server può inviare un'intestazione Set-Cookie nella risposta HTTP. Questa intestazione dice al browser di salvare un cookie con informazioni specifiche. Per esempio, potrebbe memorizzare un ID univoco che aiuta il sito a riconoscerti la prossima volta che visiti.
Puoi impostare manualmente un cookie in JavaScript usando `document.cookie`:

```js
document.cookie = "organization=freeCodeCamp; expires=Fri, 31 Dec 2021 23:59:59 GMT; path=/";
```

Per eliminare un cookie, puoi impostare la sua data di scadenza a un momento nel passato.

```js
document.cookie = "username=; expires=Thu, 01 Jan 1970 00:00:00 GMT; path=/";
```

## Cache API

- **Definizione**: La cache è il processo di memorizzare copie di file in una posizione di archiviazione temporanea, così da poterli consultare più rapidamente. La Cache API serve per memorizzare richieste e risposte di rete, rendendo le applicazioni web più efficienti e capaci di funzionare anche offline. Fa parte della più ampia Service Worker API ed è fondamentale per creare Progressive Web App (PWA) che funzionano anche in condizioni di rete lente o inaffidabili.
La Cache API è un meccanismo di archiviazione che conserva oggetti Request e Response. Quando viene fatta una richiesta a un server, l'applicazione può memorizzare la risposta e poi recuperarla dalla cache invece di fare una nuova richiesta di rete. Questo riduce i tempi di caricamento, risparmia banda e migliora l'esperienza utente complessiva.
- **Archivio cache**: Serve per memorizzare coppie chiave-valore di richieste HTTP e delle loro risposte corrispondenti. Questo permette un recupero efficiente delle risorse già richieste, riducendo la necessità di scaricarle nuovamente dalla rete nelle visite successive e migliorando le prestazioni.
- **Cache-Control**: Gli sviluppatori possono definire per quanto tempo una risorsa in cache deve essere mantenuta e se deve essere riconsultata o servita direttamente dalla cache.
- **Supporto offline**: Usando la Cache API, puoi creare applicazioni web che funzionano prima offline. Per esempio, una PWA può servire risorse memorizzate in cache quando l'utente è disconnesso dalla rete.

## Pattern negativi e archiviazione lato client

- **Tracciamento eccessivo**: Si riferisce alla pratica di raccogliere e memorizzare una quantità eccessiva di dati utente nell'archiviazione lato client (come cookie, local storage o session storage) senza un consenso chiaro, informato o una necessità legittima. Spesso comporta il tracciamento del comportamento, delle preferenze e delle interazioni dell'utente su più siti o sessioni, il che può violare la privacy dell'utente.
- **Fingerprinting del browser**: Tecnica usata per tracciare e identificare utenti individuali basandosi su caratteristiche uniche del loro dispositivo e browser, invece di affidarsi a cookie o altri metodi tradizionali di tracciamento. A differenza dei cookie, che sono memorizzati localmente sul dispositivo dell'utente, il fingerprinting raccoglie una serie di informazioni che possono essere usate per creare un "impronta" distintiva della sessione del browser dell'utente.
- **Memorizzare password in LocalStorage**: Questo può sembrare un pattern negativo ovvio, ma memorizzare dati sensibili come password nel local storage rappresenta un rischio per la sicurezza. Il local storage non è criptato e può essere facilmente consultato. Quindi non dovresti mai memorizzare dati sensibili di alcun tipo lì.

## IndexedDB

- **Definizione**: IndexedDB serve per memorizzare dati strutturati nel browser. È integrato nei browser moderni, permettendo alle app web di archiviare e recuperare oggetti JavaScript in modo efficiente.

## Cache/Service Worker

- **Definizione**: Un Service Worker è uno script che gira in background separatamente dalla tua pagina web. Può intercettare richieste di rete, consultare la cache e far funzionare l'app web offline. È un componente chiave delle Progressive Web App.
