# --description--

I moduli sono fondamentali in ogni applicazione web perché ti permettono di gestire l'input dell'utente, raccogliere dati e attivare azioni.

In React, i moduli si gestiscono usando lo stato o i ref, dandoti il pieno controllo sul loro comportamento e sulla validazione. Questi due modi di gestire i moduli si chiamano input "controllato" e "non controllato".

Vediamo cosa sono gli input controllati e non controllati.

L'input controllato è il modo più "React-like" di gestire gli input dei moduli. Con gli input controllati, memorizzi il valore del campo di input nello stato e lo aggiorni tramite eventi `onChange`. Questo ti dà il controllo completo sui dati del modulo e permette una validazione istantanea e un rendering condizionale.

Il processo funziona così: React mantiene lo stato del modulo con il hook `useState` e lo aggiorni a ogni cambiamento. Quando un utente digita in un campo di input, si attiva l'evento `onChange`, aggiorna lo stato e React renderizza di nuovo il componente con il nuovo valore.

```jsx
import { useState } from "react";

function App() {
  const [name, setName] = useState("");

  const handleChange = (e) => {
    setName(e.target.value);
  };

  const handleSubmit = (e) => {
    e.preventDefault();
    console.log(name);
  };

  return (
    <>
      <form onSubmit={handleSubmit}>
        <label htmlFor="name">Your name</label> <br />
        <input value={name} id="name" onChange={handleChange} type="text" />
        <button type="submit">Submit</button>
      </form> 
    </>
  );
}

export default App;
```

I vantaggi degli input controllati includono:

- Accesso immediato ai dati del modulo.

- Puoi implementare una validazione istantanea.

- Puoi disabilitare condizionatamente il pulsante di invio.

- Puoi controllare il valore dell'input in modo programmatico.

Gli input non controllati invece si vedono più spesso nei moduli HTML tradizionali. Quindi, invece di gestire gli input tramite il hook `useState`, gli input non controllati in HTML mantengono il loro stato interno con l’aiuto del DOM.

Poiché il DOM controlla i valori degli input, ciò che serve è prelevare i valori dei campi di input con i ref. Questo approccio richiede meno codice e ha prestazioni migliori perché i ref non fanno renderizzare di nuovo React.

Ecco un esempio di input non controllati:

```jsx
import { useRef } from "react";

function App() {
  const nameRef = useRef();

  const handleSubmit = (e) => {
    e.preventDefault();
    console.log(nameRef.current.value);
  };

  return (
    <form onSubmit={handleSubmit}>
      <label htmlFor="name">Your</label>{" "}
      <input type="text" ref={nameRef} id="name" />
      <button type="submit">Submit</button>
    </form>
  );
}

export default App;
```

Un vantaggio molto evidente degli input non controllati è che richiedono meno codice. Inoltre hanno prestazioni migliori e risultano più naturali per chi inizia con React e conosce già l’HTML.

Allora, quale dovresti usare tra input controllati e non controllati?

Usa gli input controllati quando ti servono aggiornamenti dinamici del modulo, validazione in tempo reale o quando vuoi sincronizzare i valori degli input con lo stato. Offrono un controllo migliore ma richiedono più renderizzazioni.

Usa gli input non controllati quando ti servono moduli più semplici, vuoi consultare i valori solo all’invio o quando lavori con codice non React.

Indipendentemente da quale usi tra input controllati e non controllati, ecco alcune buone pratiche da seguire quando crei moduli in React:

- Previeni sempre l’invio predefinito del modulo.

- Assicurati di validare gli input prima dell’invio.

- Fornisci sempre un feedback chiaro agli utenti con stati di caricamento, errori di validazione o altri stati correlati.
