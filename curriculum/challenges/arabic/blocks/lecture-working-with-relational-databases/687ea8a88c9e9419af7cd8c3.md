# --description--

لنستعرض بعض الأوامر الأساسية التي يمكنك استخدامها لإدخال وعرض البيانات في SQL.

أولًا، لعرض البيانات، يجب إدخالها. لذا سنبدأ بالإدخالات.

الأمثلة التي سنغطيها في هذا الدرس ستعتمد على هذا الجدول الخاص بسجلات الكلاب:

```sql
CREATE TABLE dogs(
  id SERIAL,
  name VARCHAR(100),
  age INTEGER
);
```

سيخزن الجدول أسماء وأعمار كلاب مختلفة.

إذا افترضنا أننا نعمل مع PostgreSQL، يمكننا استخدام نوع البيانات `SERIAL` للعمود `id`. بما أن `id` تم إنشاؤه باستخدام `SERIAL`، ستكون قيمته من نوع `INTEGER`، تبدأ من `1`، وتزداد تلقائيًا عند إدخال سجل جديد. لذلك لن تحتاج إلى تمرير قيمة له عند إدخال سجل.

لنبدأ بإدخال سجلات في جدول `dogs` باستخدام جملة `INSERT INTO`. هناك عدة طرق للقيام بذلك. الأولى هي إدخال صف واحد وتحديد الأعمدة التي ستُدرج القيم فيها.

تكتب `INSERT INTO`، تليها اسم الجدول (في هذه الحالة، `dogs`)، ثم الأعمدة داخل أقواس عادية، ثم `VALUES`، والقيم المقابلة لها داخل أقواس عادية.

هذا مثال، نحن نُدرج سجل كلب باسم `name` هو `'Gino'`، وعمره `age` يساوي `3`، تذكر أن `id` سيُعيَّن تلقائيًا:

```sql
INSERT INTO dogs (name, age)
VALUES ('Gino', 3);
```

عادةً ما يكون هذا الخيار الأكثر أمانًا لأنك تحدد الأعمدة صراحةً، فتُعيَّن القيم لتلك الأعمدة بالترتيب.

لاحظ أن `'Gino'` يجب أن تكون بين علامات اقتباس مفردة لأنها من نوع `VARCHAR`.

بديل آخر هو إدخال السجل دون تحديد الأعمدة، مثل هذا:

```sql
INSERT INTO dogs
VALUES ('Gino', 3);
```

هذه الصيغة صحيحة في SQL، لكنها عرضة للأخطاء.

بما أن القيم تُعيَّن بناءً على ترتيب الأعمدة، سيحاول PostgreSQL إدخال `'Gino'` في عمود `id`، الذي يتوقع عددًا صحيحًا. هذا يؤدي إلى خطأ مثل `invalid input syntax for type integer`.

لهذا السبب، من الأفضل تحديد أسماء الأعمدة صراحةً عند إدخال البيانات.

يمكنك أيضًا إدخال سجلات متعددة في نفس أمر SQL بفصلها بفاصلة.

هنا، ندخل سجلين لكلب، واحد لـ `'Gino'` وآخر لـ `'Nora'`.

```sql
INSERT INTO dogs (name, age)
VALUES
  ('Gino', 3),
  ('Nora', 2);
```

وكما يمكنك إدخال سجلات، يمكنك استعلام المعلومات من قاعدة البيانات. في سياق قواعد البيانات، الاستعلام هو طلب بيانات.

في SQL، تُستخدم جملة `SELECT` لاستعلام البيانات من جدول أو أكثر. يمكنك تخصيص الاستعلام للحصول على المعلومات التي تحتاجها بالضبط.

إذا كنت تريد استعلام كل المعلومات من جدول `dogs`، فقط اكتب `SELECT`، متبوعًا بعلامة النجمة (`*`)، ثم `FROM`، واسم الجدول `dogs`:

```sql
SELECT *
FROM dogs;
```

في هذا الأمر، علامة النجمة هي رمز بدل تمثّل "كل الأعمدة".

ستحصل على كل بيانات كل السجلات في الجدول:

```sql
 id | name | age 
----+------+-----
  1 | Gino |   3
  2 | Nora |   2
```

لاستعلام أعمدة محددة، يمكنك كتابة أسماء تلك الأعمدة في الأمر، مباشرة بعد `SELECT`، مفصولة بفاصلة. في هذا المثال، نستعلم عن عمودي `name` و `age` من جدول `dogs`:

```sql
SELECT name, age
FROM dogs;
```

النتيجة ستشمل فقط عمودي `name` و `age`:

```sql
 name | age 
------+-----
 Gino |   3
 Nora |   2
```

أحيانًا قد تحتاج لاستعلام بيانات بناءً على شرط معين. على سبيل المثال، للحصول على كل الكلاب التي عمرها أقل من 3 سنوات، يمكنك استخدام الكلمة المفتاحية `WHERE` وعامل المقارنة أقل من (`<`):

```sql
SELECT *
FROM dogs
WHERE age < 3;
```

النتيجة ستشمل فقط الكلاب التي عمرها أقل من 3:

```sql
 id | name | age 
----+------+-----
  2 | Nora |   2
```

إذا كنت تحاول فقط معرفة `age` الخاص بـ `'Gino'`، يمكنك استخدام عامل المقارنة يساوي (`=`):

```sql
SELECT age
FROM dogs
WHERE name = 'Gino';
```

وهنا النتيجة:

```sql
 age 
-----
  3
```

هذه طرق شائعة وبسيطة لإدخال وعرض البيانات في SQL، لكن هناك خيارات كثيرة مختلفة يمكنك اختيارها وبعضها أكثر تقدمًا. ستتعلم عنها في الدروس القادمة.
