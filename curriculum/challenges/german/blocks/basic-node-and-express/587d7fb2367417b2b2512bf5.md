# --description--

Beim Erstellen einer API müssen wir den Nutzenden ermöglichen, uns mitzuteilen, was sie von unserem Service erhalten möchten. Wenn der Client beispielsweise Informationen über einen in der Datenbank gespeicherten Benutzer anfragt, braucht er eine Möglichkeit, uns mitzuteilen, an welchem Benutzer er interessiert ist. Eine mögliche Methode, dieses Ziel zu erreichen, ist die Verwendung von Route-Parametern. Route-Parameter sind benannte Segmente der URL, die durch Schrägstriche (/) getrennt sind. Jedes Segment erfasst den Wert des URL-Teils, der seiner Position entspricht. Die erfassten Werte finden Sie im `req.params`-Objekt.

<blockquote>route_path: '/user/:userId/book/:bookId'<br>actual_request_URL: '/user/546/book/6754' <br>req.params: {userId: '546', bookId: '6754'}</blockquote>

# --instructions--

Erstellen Sie einen Echo-Server, der an der Route `GET /:word/echo` angebunden ist. Antworten Sie mit einem JSON-Objekt, das die Struktur `{echo: word}` hat. Das Wort, das wiederholt werden soll, finden Sie bei `req.params.word`. Sie können Ihre Route im Adressfeld Ihres Browsers testen, indem Sie passende Routen besuchen, z. B. `your-app-rootpath/freecodecamp/echo`.

# --hints--

Test 1 : Ihr Echo-Server sollte Wörter korrekt wiederholen

```js
  const response = await fetch(code + '/eChOtEsT/echo');
  if (!response.ok) {
    throw new Error(await response.text());
  }
  const data = await response.json();
  assert.equal(
    data.echo,
    'eChOtEsT',
    'Test 1: the echo server is not working as expected'
  );
```
