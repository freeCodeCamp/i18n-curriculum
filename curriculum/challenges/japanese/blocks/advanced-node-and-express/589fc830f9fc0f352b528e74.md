# --description--

以下のチャレンジでは `chat.pug` ファイルを使用します。ですので、`routes.js` ファイルに `/chat` を指す GET ルートを追加してください。このルートは `ensureAuthenticated` を使い、`chat.pug` をレンダリングし、`{ user: req.user }` をレスポンスの引数として渡します。次に、既存の `/auth/github/callback` ルートを変更して `req.session.user_id = req.user.id` を設定し、`/chat` へリダイレクトしてください。

`socket.io@~2.3.0` はすでに依存関係として追加されているので、Node.js に組み込みの `http` とともにサーバーで require/インスタンス化してください。

```javascript
const http = require('http').createServer(app);
const io = require('socket.io')(http);
```

*http* サーバーが *express app* にマウントされたので、*http* サーバーからリッスンする必要があります。`app.listen` の行を `http.listen` に変更してください。

最初に処理すべきことは、クライアントからの新しい接続を待ち受けることです。<dfn>on</dfn> キーワードはまさにそれを行います — 特定のイベントを待ち受けます。2つの引数が必要です：発生するイベントの名前を含む文字列と、データを受け取る関数です。接続リスナーの場合、2番目の引数のデータを定義するために `socket` を使ってください。ソケットは接続された個々のクライアントを指します。

サーバーへの接続を待ち受けるには、データベース接続内に次のコードを追加してください：

```javascript
io.on('connection', socket => {
  console.log('A user has connected');
});
```

クライアントが接続するには、認証後にページで読み込まれる `client.js` に次のコードを追加してください：

```js
/*global io*/
let socket = io();
```

このコメントは、ファイル内で 'io' が定義されていないために通常表示されるエラーを抑制します。`chat.pug` でページに Socket.IO ライブラリの信頼できる CDN をすでに追加しています。

アプリを読み込み、認証するとサーバーのコンソールに `A user has connected` が表示されるはずです。

**注意：** `io()` は同じ URL/サーバーでホストされているソケットに接続する場合のみ動作します。別の場所でホストされている外部ソケットに接続する場合は `io.connect('URL');` を使います。

正しくできたと思ったらページを提出してください。エラーが出る場合は<a href="https://forum.freecodecamp.org/t/advanced-node-and-express/567135#set-up-the-environment-6" target="_blank" rel="noopener noreferrer nofollow">ここまで完成したプロジェクトを確認できます</a>。
