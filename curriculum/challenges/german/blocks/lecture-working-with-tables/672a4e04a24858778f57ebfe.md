# --interactive--

HTML-Tabellen werden heutzutage nicht mehr so häufig verwendet wie früher. Aber als Front-End-Entwickler sollten Sie trotzdem mit ihnen vertraut sein. Tabellen sind eine der frühesten Methoden, die Entwickler in den 1990er-Jahren hatten, um Daten in einem Browser darzustellen.

Hier ist ein Beispiel für Code, der verwendet wird, um eine Tabelle vom U.S. Bureau of Labor Statistics zu erzeugen:

:::interactive_editor

```html
<table id="quickfacts">
  <thead>
    <tr>
      <th colspan="2">Quick Facts: Software Developers, Quality Assurance Analysts, and Testers</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <th>2023 Median Pay</th>
      <td>
        $130,160 per year
        <br>$62.58 per hour
      </td>
    </tr>
    <tr>
      <th>Typical Entry-Level Education</th>
      <td>Bachelor's degree</td>
    </tr>
    <tr>
      <th>Work Experience in a Related Occupation</th>
      <td>None</td>
    </tr>
    <tr>
      <th>On-the-job Training</th>
      <td>None</td>
    </tr>
    <tr>
      <th>Number of Jobs, 2022</th>
      <td>1,795,300</td>
    </tr>
    <tr>
      <th>Job Outlook, 2022-32</th>
      <td>25% (Much faster than average)</td>
    </tr>
    <tr>
      <th>Employment Change, 2022-32</th>
      <td>451,200</td>
    </tr>
  </tbody>
  <tfoot>
    <tr>
      <th>If this table had a footer it would be here.</th>
    </tr>
  </tfoot>
</table>
```

:::

Wie Sie sehen, gibt es ein Haupt-`table`-Element mit einem `id`, das auf `quickfacts` gesetzt ist. Innerhalb davon hat die Tabelle ein Tabellenkopf-Element, `thead`, ein Tabellenkörper-Element, `tbody`, und ein Tabellenfuß-Element, `tfoot`.

Das Tabellenkopf-, Körper- und Fuß-Element kann jeweils eine beliebige Anzahl von Tabellenzeilen, `tr`, enthalten. Und jede Tabellenzeile kann eine Tabellenkopf-`th` enthalten, die die Daten in dieser Zeile bezeichnet. Außerdem kann sie eine beliebige Anzahl von einzelnen Datenzellen enthalten, die Tabellendaten genannt werden, `td`.

Das sind jetzt viele HTML-Elemente. Aber lassen Sie sich nicht einschüchtern – sie folgen einer einfachen Hierarchie.

Hier ist die einfachste Tabelle, die wir erstellen können und die all diese Elemente enthält:

:::interactive_editor

```html
<table>
  <thead>
    <tr>
      <th>The title of this table</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <th>First Row</th>
      <td>
        First Data Cell
      </td>
    </tr>
    <tr>
      <th>Second Row</th>
      <td>
        Second Data Cell
      </td>
    </tr>
  </tbody>
  <tfoot>
    <tr>
      <th>The footer of this table, which might contain date of publication, author credits, or other meta information.</th>
    </tr>
  </tfoot>
</table>
```

:::

Wie Sie sehen, befinden sich die Daten immer innerhalb eines `tr` – und innerhalb dieses `tr`-Elements befindet sich ein `th`-Element mit einer Kopfzeile und ein `td`-Element mit Daten.

Manche Websites entscheiden sich dafür, `div`-Elemente zu verwenden, um ihre eigenen Tabellen zu erstellen, anstatt das passendere `table`-Element zu nutzen.

Obwohl es möglich ist, tabellarische Daten mit generischen `div`-Elementen darzustellen, ist es dennoch besser, stattdessen das `table`-Element zu verwenden.

Vor vielen Jahren haben Entwickler vielleicht ein `table` verwendet, um nicht-datenbezogene Elemente auf einer Webseite zu positionieren. Das wurde nie als bewährte Vorgehensweise angesehen. Aber Sie könnten auf Codebasen stoßen, in denen Tabellen immer noch so verwendet werden.

Heutzutage nutzen Entwickler CSS Flexbox und Grid, um ihre Layouts zu gestalten. freeCodeCamp wird diese Werkzeuge später ausführlich behandeln.

Für den Moment verwenden Sie HTML-Tabellen nur für ihren ursprünglichen Zweck: die Darstellung tabellarischer Daten.
