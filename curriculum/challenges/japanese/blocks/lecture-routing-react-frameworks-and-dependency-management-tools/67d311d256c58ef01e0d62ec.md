# --description--

これまでのところ、Reactを使ってユーザーインターフェースを構築してきました。もしルーティングのような追加機能が必要な場合は、React Routerのようなサードパーティのライブラリをインポートして、異なるビュー間を切り替えられるようにしていました。

では、フルスタックのウェブアプリケーションを構築する必要がある場合はどうでしょうか？フロントエンドにはReactを使い、バックエンドのロジックにはJavaScriptだけで済ませたいならNodeとExpressを使うことができます。あるいは、Go、Python、Javaなど他の言語をバックエンドに使うこともできます。

これらはすべて有効な選択肢ですが、Reactフレームワークを使いたい場合もあります。Reactフレームワークはルーティング、画像の最適化、データ取得、認証などの機能を提供します。つまり、特定のユースケースではフロントエンドとバックエンドのアプリケーションを別々に設定する必要がない場合があります。

ここで、人気のあるReactフレームワークのNext.jsを詳しく見てみましょう。Next.jsの主な機能の一つはファイルシステムベースのルーターです。このルーティングシステムは動的ルート、並列ルート、ルートハンドラ、リダイレクト、国際化などをサポートしています。

カスタムのリクエストハンドラを作成する例を示します：

```js
export async function GET() {
  const res = await fetch("https://example-api.com");
  const data = await res.json();

  return Response.json({ data });
}
```

`route.js`ディレクトリ内の`app/api`というファイルで、GETやPOSTリクエストのようなルートハンドラを定義できます。

Next.jsのもう一つの機能は自動の画像とフォントの最適化です。

`Image`ファイル内で`page.js`コンポーネントを使う例を示します：

```jsx
import Image from "next/image";

export default function Page() {
  return (
    <Image src="link-to-image-goes-here" alt="descriptive-title-goes-here" />
  );
}
```

`Image`コンポーネントはネイティブのHTML `img`要素を拡張しており、ページの読み込み速度とサイズの最適化を可能にします。つまり、画像はビューポートに入ったときだけ読み込まれ、`Image`コンポーネントは各デバイスに適したサイズの画像を自動的に提供します。

これらはNext.jsの機能の一部にすぎませんが、堅牢なフルスタックウェブアプリケーションを構築するために使える多くの機能があります。Remixのような他のReactフレームワークも、モダンなフルスタックウェブアプリケーションを構築する同様の能力を提供します。

JavaScriptのライブラリやフレームワークの状況は常に変化していますが、利用可能な選択肢を知り、それぞれのフレームワークの長所と短所を学ぶことは重要です。
