# --description--

Lernen wir, wie man alle Vorkommen eines Musters in einem String findet oder ersetzt.

Sie haben bereits die `replace()`- und `match()`-Methoden sowie den globalen `g`-Modifikator kennengelernt. Nun können Sie dieses Wissen kombinieren, um alle Muster in einem String zu bearbeiten. Erinnern wir uns an unseren ursprünglichen Match-Code:

```js
const regex = /freeCodeCamp/;
const match = "freeCodeCamp".match(regex);
console.log(match);
```

Und unser resultierendes Match-Objekt:

```js
// [
//   'freeCodeCamp',
//   index: 0,
//   input: 'freeCodeCamp',
//   groups: undefined
// ]
```

Aber was, wenn wir einen String mit mehreren Vorkommen von `freecodecamp` zum Finden haben? Schauen wir uns an, wie `match()` sich dabei verhält. Wir werfen auch unser altes `replace()`-Beispiel ein, nur zum Vergleich:

```js
const regex = /freecodecamp/;
const str = "freecodecamp is the best we love freecodecamp";
const matched = str.match(regex);
const replaced = str.replace(regex, "freeCodeCamp");
console.log(matched);
console.log(replaced);
```

Und das Ergebnis ist dieses:

```js
// [
//   'freecodecamp',
//   index: 0,
//   input: 'freecodecamp is the best we love freecodecamp',
//   groups: undefined
// ]
// freeCodeCamp is the best we love freecodecamp
```

Oh nein! `match()` hat nur das erste Vorkommen zurückgegeben, und `replace()` hat nur das erste Vorkommen ersetzt. Das liegt daran, dass `match()` und `replace()` standardmäßig nur gegen das erste Muster-Vorkommen arbeiten.

Glücklicherweise können Sie das vermeiden, indem Sie den globalen Modifikator in Ihrem regulären Ausdruck verwenden. Fügen wir ihn unserem Ausdruck hinzu:

```js
const regex = /freecodecamp/g;
const str = "freecodecamp is the best we love freecodecamp";
const matched = str.match(regex);
const replaced = str.replace(regex, "freeCodeCamp");
console.log(matched);
console.log(replaced);
```

Und bestätigen das Ergebnis:

```js
// [ 'freecodecamp', 'freecodecamp' ]
// freeCodeCamp is the best we love freeCodeCamp
```

Das hat funktioniert! Unser Replace-Aufruf hat alle Kleinbuchstaben-`freecodecamp`-Strings ersetzt, und unsere `match()`-Methode hat beide gefunden.

Interessant ist hier, dass Sie beim Verwenden des globalen Modifikators mit `match()` die zusätzlichen Informationen über Capture-Gruppen und String-Indizes verlieren, die im Match-Array enthalten wären.

Glücklicherweise brachte uns das ECMAScript-Update von 2019 zwei neue Methoden: `matchAll()` und `replaceAll()`. Wie ihre einzelnen Gegenstücke akzeptieren diese Methoden einen String oder regulären Ausdruck, und `replaceAll()` akzeptiert auch ein zweites Argument als den Ersetzungsstring.

Aber im Gegensatz zu den vorherigen Methoden werfen `replaceAll()` und `matchAll()` einen Fehler, wenn Sie ihnen einen regulären Ausdruck ohne globalen Modifikator geben. Aktualisieren wir unseren Code, um diese neuen Methoden zu verwenden:

```js
const pattern = "freecodecamp";
const str = "freecodecamp is the best we love freecodecamp";
const matched = str.matchAll(pattern);
const replaced = str.replaceAll(pattern, "freeCodeCamp");
console.log(matched);
console.log(replaced);
```

Und unser Ergebnis:

```js
// {}
// freeCodeCamp is the best we love freeCodeCamp
```

Gute Nachrichten! Unser `replaceAll()` hat genau so funktioniert, wie wir es wollten – er hat alle Vorkommen des kleingeschriebenen `freecodecamp` durch die korrekt in CamelCase geschriebene Version ersetzt.

Aber was ist dieses leere Objekt? Nun, `matchAll()` gibt einen speziellen Objekttyp zurück, der `Iterator` heißt, den die freeCodeCamp-Konsole nicht verarbeiten kann.

Wenn wir in unsere Browser-Konsole schauen, hat der `Iterator` eine `next()`-Methode, die wir aufrufen können, um den nächsten Wert zu erhalten:

```js
// RegExpStringIterator {  }
//  <prototype>: RegExp String Iterator {
//    next: ƒ next(),
//    Symbol(Symbol.toStringTag): "RegExp String Iterator"
//    <prototype>: Object { ... }
//  }
```

Rufen wir `matched.next()` auf und protokollieren das Ergebnis:

```js
// {
//   "done": false,
//   "value": [
//     0: "freecodecamp"
//     groups: undefined
//     index: 0
//     input: "freecodecamp is the best we love freecodecamp"
//   ]
// }
```

Da ist unser Match-Array! `next()` gibt uns ein Objekt mit zwei Werten: `done`, das `false` ist, wenn noch weitere Elemente im Iterator verfügbar sind, und `value`, das der Wert ist, über den wir gerade iteriert haben. Wenn wir es also noch einmal aufrufen:

```js
const regex = /freecodecamp/g;
const str = "freecodecamp is the best we love freecodecamp";
const matched = str.matchAll(regex);
const replaced = str.replaceAll(regex, "freeCodeCamp");
console.log(matched);
console.log(replaced);
console.log(matched.next());
console.log(matched.next());
```

```js
// {
//   "done": false,
//   "value": Array [ "freecodecamp"]
// }
//
// {
//   "done": false,
//   "value": Array [ "freecodecamp"]
// }
```

Moment, warum sagt es, `done` ist immer noch `false`? Es sollten doch nur zwei Matches im Array sein, oder? Rufen wir es ein drittes Mal auf und sehen, was wir bekommen:

```js
// {
//   "done": false,
//   "value": Array [ "freecodecamp"]
// }
//
// {
//   "done": false,
//   "value": Array [ "freecodecamp"]
// }
//
// {
//   "done": true,
//   "value": undefined
// }
```

`done` ist endlich `true`, aber warum ist dieser Wert `undefined`? Nun, wie sich herausstellt, ist der `matchAll()`-Iterator faul. Er findet nicht alle Ihre Matches auf einmal. Er findet nur dann ein Match, wenn Sie ihn durch einen Aufruf von `next()` dazu auffordern.

Solange er ein Match findet, ist er nicht `done`. Sobald er kein Match mehr findet und `undefined` zurückgibt, ist er `done`. Das mag unpraktisch erscheinen, kann aber sehr hilfreich sein, wenn Ihr regulärer Ausdruck rechenintensiv ist.

Wenn Ihr Beispiel weniger rechenintensiv ist, wie unseres, können Sie diese Funktion überspringen und alle Matches auf einmal extrahieren, indem Sie den Iterator in ein Array umwandeln. Das erreichen Sie, indem Sie `Array.from()` aufrufen und Ihren Iterator als Argument übergeben.

Aktualisieren wir unseren Code, um das zu verwenden – wir bereinigen auch unsere `replaceAll`-Aufrufe, da wir wissen, dass das funktioniert:

```js
const regex = /freecodecamp/g;
const str = "freecodecamp is the best we love freecodecamp";
const matched = str.matchAll(regex);
console.log(Array.from(matched));
```

Und wir erhalten endlich unser Array mit den Matches:

```js
// [
//   'freecodecamp',
//   index: 0,
//   input: 'freecodecamp is the best we love freecodecamp',
//   groups: undefined
// ]
//
// [
//   'freecodecamp',
//   index: 33,
//   input: 'freecodecamp is the best we love freecodecamp',
//   groups: undefined
// ]
```

Diese leistungsstarken Methoden können Ihnen helfen, Daten aus Strings zu manipulieren und zu extrahieren, ohne Leistung oder Lesbarkeit opfern zu müssen.
