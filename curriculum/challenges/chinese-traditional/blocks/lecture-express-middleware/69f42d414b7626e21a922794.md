# --description--

Express 中的錯誤處理中介層是一種特殊類型的中介層，用來處理在請求-回應週期中發生的錯誤。它對於捕捉意外錯誤、記錄錯誤，以及向客戶端提供適當的回應非常重要。

如你在先前的錯誤處理課程中所見，Express 的錯誤處理中介層有四個參數：`err`、`req`、`res` 和 `next`。當錯誤被傳遞給 `next()` 函式時，這個中介層會被喚起。

這裡再提醒一次語法：

```js
app.use((err, req, res, next) => {
  console.error(err.message);
  res.status(500).send("Internal Server Error");
});
```

以下是在基本 Express 應用程式中的範例：

```js
const express = require('express')
const app = express()

// A simple route that throws an error
app.get('/', (req, res) => {
  throw new Error('Something went wrong!')
})

// Error-handling middleware
app.use((err, req, res, next) => {
  console.error(err.stack); // Logs the error stack
  res.status(500).send('Something went wrong!')
})

app.listen(3000, () => {
  console.log('Server running on http://localhost:3000')
})
```

* 路由處理常式：路由處理常式故意丟出錯誤以模擬問題。
* 錯誤處理中介層：這個中介層是用 `app.use()` 註冊的。它會捕捉透過 `next(err)` 傳遞的錯誤，並向客戶端發送 500 HTTP 回應，附帶一般錯誤訊息。
* `err`：這是傳遞給中介層的錯誤物件。

錯誤處理中介層確保當你的應用程式發生問題時，使用者仍能收到適當的回應。務必將此中介層放在中介層堆疊的最後，以捕捉所有錯誤。
