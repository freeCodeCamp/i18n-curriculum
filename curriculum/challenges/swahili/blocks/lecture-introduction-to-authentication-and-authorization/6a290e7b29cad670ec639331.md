# --description--

Kuweka uthibitisho wa utambulisho wa kawaida kunahitaji kuandika mantiki nyingi kwa mkono. Lazima ushughulikie kusoma vichwa vya ombi, kutoa vidakuzi, kuthibitisha saini, na kusimamia makosa.

Ingawa kujijengea mwenyewe ni nzuri kwa kujifunza, kufanya hivyo kwa kila programu ya uzalishaji kunachukua muda mwingi na kuna hatari ya makosa ya usalama. Hii ndiyo sababu wasanidi programu hutumia Passport.js.

Passport.js ni programu ya kati ya uthibitisho wa utambulisho kwa Node.js. Inafanya kazi kama mfumo wa kazi uliopangwa unaoleta utaratibu wa jinsi unavyothibitisha maombi. Inashughulikia maelezo ya chini ya uthibitisho wa utambulisho, ikikuruhusu kuzingatia mantiki ya biashara ya programu yako.

Wazo kuu nyuma ya Passport.js ni kwamba kila programu inahitaji njia tofauti za kuingia, lakini mtiririko wa msingi wa uthibitisho wa utambulisho unapaswa kubaki ule ule. Passport inafanikisha kubadilika hii kupitia programu-jalizi za moduli zinazojulikana kama **Strategies**.

Badala ya kuandika msimbo maalum kwa kila njia ya uthibitisho, unasakinisha strategy maalum unayohitaji. Kuna zaidi ya strategies 500 zinazopatikana, zikihusisha karibu kila njia ya uthibitisho inayotumika leo. Baadhi yao ni:

- **passport-local:** Inashughulikia uthibitisho wa kawaida wa jina la mtumiaji na nenosiri dhidi ya hifadhidata yako mwenyewe.

- **passport-jwt:** Inalinda kiishio cha API kwa kutoa na kuthibitisha Tokeni za JSON Web kutoka kwa maombi yanayoingia.

- **passport-google-oauth20:** Inaruhusu watumiaji kuingia mara moja kwa kutumia akaunti zao za Google zilizopo.

- **passport-facebook:** Ingia kwa akaunti za Facebook kupitia OAuth 2.0.

- **passport-github2:** Inaruhusu watumiaji kuingia mara moja kwa kutumia akaunti zao za GitHub.

Kwa kugawanya uthibitisho katika strategies tofauti, Passport inaruhusu programu yako kukua bila matatizo. Ukianza na mfumo wa jina la mtumiaji/nenosiri leo na ukaamua kuongeza "Ingia na Google" mwezi ujao, huna haja ya kuandika upya msimbo wako wote wa uthibitisho. Unahitaji tu kusakinisha strategy ya Google na kuiunganisha na Passport.

Ili kutumia Passport katika programu ya Express, lazima usakinishe kifurushi cha msingi cha Passport pamoja na strategy maalum unayopanga kutumia. Kwa usanidi wa kawaida wa jina la mtumiaji na nenosiri, endesha amri ifuatayo kwenye terminal yako:

```bash
npm install passport passport-local
```

Mara baada ya kusakinishwa, unahitaji kuunganisha Passport katika usanidi wa programu yako ya Express. Hii lazima ifanyike kabla hujaeleza njia zako za data:

```js
const express = require('express');
const passport = require('passport');
const LocalStrategy = require('passport-local').Strategy;

const app = express();

// initialize Passport middleware
app.use(passport.initialize());

// configure Passport to use the Local Strategy
passport.use(new LocalStrategy((username, password, done) => {
    // your database look-up and password verification logic goes here
}));
```

Passport inaunganishwa kwa urahisi katika mfumo wa kawaida wa njia za Express kama programu ya kati. Mtumiaji anapowasilisha taarifa zake kwenye njia yako ya kuingia, Passport hukamata ombi, hupitisha data kwa strategy uliyoipanga, na kutathmini matokeo.

```js
app.post('/login', passport.authenticate('local'), (req, res) => {
  // if this function runs, then authentication was successful
  res.send(`Welcome back, ${req.user.username}!`);
});
```

Ikiwa strategy inathibitisha mtumiaji kwa mafanikio, Passport hushughulikia kazi mbili muhimu moja kwa moja:

1. **Usimamizi wa kikao:** Inasababisha vitendakazi vya msaada kuhifadhi kipande kidogo cha data ya mtumiaji ndani ya kidakuzi cha kikao.

2. **Mapambo ya ombi:** Inaambatanisha kitu cha mtumiaji aliyethibitishwa moja kwa moja kwenye ombi linaloingia kama `req.user`.

Mara `req.user` itakapojazwa, programu yoyote ya kati au mshughulikiaji wa njia katika programu yako inaweza kuona mara moja ni nani aliyeingia na kufanya maamuzi ya idhini kulingana na utambulisho wake.

Passport hutoa msingi thabiti, salama, na tayari kwa uzalishaji wa kusimamia utambulisho wa watumiaji katika programu zako za Node.js.
