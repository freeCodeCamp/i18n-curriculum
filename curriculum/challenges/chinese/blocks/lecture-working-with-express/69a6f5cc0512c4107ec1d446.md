# --description--

Express.js，简称 Express，是一个基于 Node.js 构建的极简且灵活的网络框架。它简化了创建网络服务器和 API 的过程。

与其使用 Node 的本地 `http` 模块编写大量公式化代码，Express 让你只需几行代码就能创建健壮的网络应用。

你可以把它看作是让 Node.js 更简单、更强大的快速、无偏见的层。

以下是开发者喜欢 Express 的三个主要原因：

1. **极简且快速**——Express 只提供必要的功能，给予你完全的控制权，同时允许你通过中间件和插件按需添加特色。

2. **庞大的生态系统**——它拥有大量用于日志、认证、安全性、会话等的中间件和插件集合。

3. **非常适合 API**——它简单的路由和请求处理模型使 Express 特别适合构建 RESTful API。

使用 Express，你可以创建：

* 网络应用

* RESTful API

* 单页应用（SPA）后端

* 以中间件为主的服务

* 任何基于 HTTP 的服务器端内容


## 如何安装 Express

首先，确保你已经安装了 Node.js 和 npm。以下是检查 node 和 npm 是否安装的方法：

```bash
node -v
npm -v
```

如果你看到版本号，说明它们已安装。否则，你需要安装它们。你可以在之前的课程或官方 Node.js 网站找到安装说明。

接下来，在终端中输入以下命令：

```bash
mkdir my-app
cd my-app
npm init -y
npm install express
```

你将在后续课程中了解更多关于中间件和插件的内容，但简而言之，中间件是在接收请求和发送响应之间运行的代码。它通常用于误差处理、输入验证、修改请求或响应中的数据等。你可以将多个中间件串联起来，构建灵活且可复用的逻辑来处理请求。

插件通常是外部的第三方 Node.js 模块，用于扩展 Express 的功能。你可以通过 npm 单独安装它们，大多数插件以中间件形式实现，只需简单设置即可在你的应用中使用。你无需从零编写常见功能，可以使用插件来处理日志、认证或安全性最佳实践等。

Express 是 Node.js 服务器端开发中流行的框架。它为你提供了快速启动服务器、处理路由、管理中间件和构建 API 的基础模块——所有这些都用简洁且易读的代码完成。
