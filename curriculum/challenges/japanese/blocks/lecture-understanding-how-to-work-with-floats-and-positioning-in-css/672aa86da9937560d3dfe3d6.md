# --interactive--

CSSのフロートは、もともとテキストを画像などの要素の周りに回り込ませるために設計された手法です。しかし、時間が経つにつれて開発者はフロートを新しい方法で使い、レイアウトデザインに創造的に応用するようになりました。現在ではFlexboxやGridのようなモダンなレイアウト手法が主流ですが、特に古いコードを扱う場合や特定のレイアウト効果を実現したい場合には、フロートの理解が依然として重要です。

要素がフロートされると、その要素は通常の文書の流れから外れ、コンテナの左または右に寄せられます。その後に続くコンテンツはフロートされた要素の周りに回り込み、残りのスペースを埋めます。典型的な使い方は、画像を片側にフロートさせてテキストをその周りに回り込ませる方法です。この手法は特に記事やブログで、画像をテキストの横に配置する際に今でも広く使われています。以下はコード例です。

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<div class="container">
  <img src="https://placehold.co/150x150" alt="Placeholder Image">
</div>
```

```css
.container {
  border: 1px solid black;
  padding: 10px;
}

img {
  float: left;
  margin-right: 20px;
}
```

:::

上の例では、コンテナがフロートされた画像を包み込んでいません。画像は通常の文書の流れから外れているため、コンテナはフロートされた子要素を「認識」できず、高さがゼロに潰れています。

フロートはまた、FlexboxやGridが普及する前に複数列レイアウトを作るためにもよく使われていました。要素を横にフロートさせることで、横並びのカラムを作ることができました。ただし、フロートを使う場合は、子要素がフロートされることで親要素が潰れてしまう問題を対処する必要があります。この問題を解決するために、`container` クラスにclearfix手法を適用しています。

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<div class="container">
  <img src="https://placehold.co/150x150" alt="Placeholder Image">
  <p> This is an example of text flowing around a floated image.</p>
</div>
```

```css
.container {
  border: 1px solid black;
}

/* Clearfix CSS */
.container::after {
  content: "";  
  display: block;
  clear: both;
}

img {
  float: left;
  margin-right: 20px;
}
```

:::

- `::after` はコンテナの内容の後に見えないブロックを追加する疑似要素です。
- `content: ""` は疑似要素が存在することを保証しますが、内容は表示しません。
- `display: block` は疑似要素をブロックレベル要素にします。
- `clear: both` は疑似要素が上にあるフロートされた要素の両側をクリアするようにします。

上の例では、潰れがよりわかりやすくなるように新しい段落要素を追加しています。段落は通常の文書の流れに残るため、コンテナはそれを包み込むのに十分な高さまで拡張されます。次にclearfix手法を適用して潰れを修正し、コンテナのボーダーが正しく表示されるようにしています。clearfixルールを外してみると、再びコンテナが潰れるのを確認できます。

clearfix手法は、親要素がフロートされた子要素を正しく包み込むようにします。clearfixはフロートされた内容の後にclearプロパティを追加し、親コンテナがフロートされた子要素を「認識」するように強制します。

FlexboxやGridのようなモダンな手法があるため、複雑なレイアウトにフロートを使うことは減りましたが、特定のシナリオでは依然として重要な役割を果たしています。画像の周りにテキストを回り込ませる場合やレガシープロジェクトを扱う場合など、フロートの仕組みと正しいクリア方法を理解することは、構造が整ったレスポンシブなレイアウトを維持するために欠かせません。
