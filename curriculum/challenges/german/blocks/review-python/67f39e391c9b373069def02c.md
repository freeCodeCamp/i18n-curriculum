## Was ist Python?

- **Einführung**: Python ist eine universelle Programmiersprache, die für ihre Einfachheit und Benutzerfreundlichkeit bekannt ist. Python wird in vielen Bereichen eingesetzt, wie Datenwissenschaft und maschinelles Lernen, Webentwicklung, Skripting und Automatisierung, eingebettete Systeme und IoT und vieles mehr.
- **Häufige Anwendungsfälle**: Python wird in Datenwissenschaft, maschinellem Lernen, Webentwicklung, Cybersicherheit, Automatisierung und Mikrocomputern wie dem Raspberry Pi und MicroPython-kompatiblen Boards verwendet.

## Python in Ihrer lokalen Umgebung

- **Installation**: Der beste Weg, Python auf Windows, Mac und Linux zu installieren, ist der Download des Installationsprogramms von der offiziellen Python-Website (`https://www.python.org/`).
- **Terminal**: Eine textbasierte Schnittstelle, mit der Sie durch Eingabe von Befehlen mit Ihrem Computer interagieren können. Jedes Betriebssystem verfügt über eine Standard-Terminal-App. Auf macOS können Sie die Terminal-App verwenden. Unter Windows können Sie die Eingabeaufforderung oder PowerShell nutzen. Unter Linux hat jede Desktop-Umgebung ihre eigene Standard-Terminal-App, wie GNOME Terminal oder Konsole.
- **IDE**: IDE steht für Integrierte Entwicklungsumgebung und bietet Funktionen wie einen Code-Editor, Testwerkzeuge, ein Terminal und mehr.
- **Beliebte Code-Editoren und IDEs für Python**: VS Code, PyCharm und Spyder
- **Code lokal ausführen**: Um Python-Code auszuführen, haben Sie mehrere Möglichkeiten. Die erste Möglichkeit ist, auf die Schaltfläche „Ausführen“ in VS Code zu klicken, die wie ein Play-Button oben rechts aussieht. Dadurch wird ein Terminal geöffnet und Ihr Python-Skript dort ausgeführt.

Eine weitere Möglichkeit ist, ein Terminal zu öffnen und das Programm manuell auszuführen. Hier ist ein Beispiel, um ein Programm namens `main.py` auszuführen:

```bash
python main.py
```

Sie müssen diesen Befehl aus dem Ordner ausführen, in dem `main.py` gespeichert ist. Wenn sich `main.py` beispielsweise in einem Ordner namens `python-projects` befindet, verwenden Sie zuerst Folgendes im Terminal:

```bash
cd python-projects
```

Dann führen Sie aus:

```bash
python main.py
```

## Verwendung der Python-Interactive-Shell

- **Definition**: Eine interaktive Shell ist ein Programm, mit dem Sie Befehle einzeln eingeben und die Ergebnisse sehen können. Öffnen Sie eine Terminal-App, geben Sie `python` ein und drücken Sie `Enter`. Dadurch wird eine Python-Interactive-Shell gestartet. Das Symbol `>>>` bedeutet, dass Python darauf wartet, dass Sie einen Befehl eingeben. Versuchen Sie, `"Hello, world!"` in das Terminal auszugeben:

```bash
print("Hello, world!")
```

Sie sollten den Text wie folgt sehen:

```bash
>>> print("Hello, world!")
Hello, world!
```

Nachdem der Text ausgegeben wurde, kehrt Python zu folgendem zurück:

```bash
>>>
```

Das bedeutet, Sie können einen weiteren Befehl eingeben.

- **Read, Evaluate, Print, Loop-Zyklus (REPL)**: Immer wenn Sie Befehle eingeben, liest der Interpreter die Eingabe, wertet sie aus, gibt das Ergebnis aus und kehrt zurück, um das `>>>` anzuzeigen, damit Sie weitere Befehle eingeben können.

## Variablen

- **Variablen deklarieren**: Um eine Variable zu deklarieren, beginnen Sie mit dem Variablennamen, gefolgt vom Zuweisungsoperator (`=`) und dann dem Wert. Dies kann eine Zahl, ein String, ein Boolean usw. sein. Hier sind einige Beispiele:

```py
name = 'John Doe'
age = 25
```

- **Benennungskonventionen für Variablen**: Hier sind die Benennungskonventionen, die Sie für Variablen verwenden sollten:

- Variablennamen dürfen nur mit einem Buchstaben oder einem Unterstrich (_) beginnen, nicht mit einer Zahl.
  - Variablennamen dürfen nur alphanumerische Zeichen (a-z, A-Z, 0-9) und Unterstriche (_) enthalten.
  - Variablennamen sind case-sensitive — `age`, `Age` und `AGE` gelten alle als einzigartig.
  - Variablennamen dürfen keine der reservierten Schlüsselwörter von Python sein, wie `if`, `class` oder `def`.
  - Variablennamen mit mehreren Wörtern werden durch Unterstriche getrennt. Zum Beispiel `snake_case`.

## Häufige Datentypen in Python

- **Einführung**: Python ist eine dynamisch typisierte Sprache wie JavaScript, das heißt, Sie müssen die Typen von Variablen nicht explizit deklarieren. Die Sprache erkennt den Typ einer Variable basierend auf dem Wert, den Sie der Variable zuweisen.
- **Integer**: Eine ganze Zahl ohne Dezimalstellen:

```py
my_integer_var = 10
print('Integer:', my_integer_var) # Integer: 10
```

- **Float**: Eine Zahl mit Dezimalstellen:

```py
my_float_var = 4.50
print('Float:', my_float_var) # Float: 4.5
```

- **String**: Eine Folge von Zeichen, die in Anführungszeichen eingeschlossen sind:

```py
my_string_var = 'hello'
print('String:', my_string_var) # String: hello
```

- **Boolean**: Ein Wert, der entweder `True` oder `False` darstellt:

```py
my_boolean_var = True
print('Boolean:', my_boolean_var) # Boolean: True
```

- **Set**: Eine ungeordnete Sammlung einzigartiger Elemente:

```py
my_set_var = {7, 5, 8}
print('Set:', my_set_var) # Set: {7, 5, 8}
```

- **Dictionary**: Eine Sammlung von Schlüssel-Wert-Paaren, eingeschlossen in geschweifte Klammern:

```py
my_dictionary_var = {"name": "Alice", "age": 25}
print('Dictionary:', my_dictionary_var) # Dictionary: {'name': 'Alice', 'age': 25}
```

- **Tuple**: Eine unveränderliche, geordnete Sammlung, eingeschlossen in runde Klammern:

```py
my_tuple_var = (7, 5, 8)
print('Tuple:', my_tuple_var) # Tuple: (7, 5, 8)
```

- **Range**: Eine Zahlenfolge, die oft in Schleifen verwendet wird:

```py
my_range_var = range(5)
print(my_range_var) # range(0, 5)
```

- **List**: Eine geordnete Sammlung von Elementen, die verschiedene Datentypen unterstützen kann:

```py
my_list = [22, 'Hello world', 3.14, True]
print(my_list) # [22, 'Hello world', 3.14, True]
```

- **None**: Ein spezieller Wert, der das Fehlen eines Werts darstellt:

```py
my_none_var = None
print('None:', my_none_var) # None: None
```

## Unveränderliche und veränderliche Typen

- **Unveränderliche Typen**: Diese Typen können nach der Deklaration nicht geändert werden, obwohl Sie ihre Variablen auf etwas Neues zeigen lassen können, was als erneute Zuweisung bezeichnet wird. Dazu gehören Integer, Float, Complex, Boolean, String, Tuple, Range und `None`.
- **Veränderliche Typen**: Diese Typen können nach der Deklaration geändert werden. Sie können Elemente hinzufügen, entfernen oder aktualisieren. Dazu gehören Sammlungstypen wie List, Set und Dictionary.
- **`type()`-Funktion**: Um den Typ einer Variable zu sehen, können Sie die `type()`-Funktion wie folgt verwenden:

```py
greeting = 'Hello there!'
age = 21

print(type(greeting)) # <class 'str'>
print(type(age)) # <class 'int'>
```

- **`isinstance()`-Funktion**: Diese wird verwendet, um zu überprüfen, ob eine Variable einem bestimmten Datentyp entspricht:

```py
greeting = 'Hello world'
name = 'John Doe'

print(isinstance(greeting, str)) # True
print(isinstance(name, int)) # False
```

## Arbeiten mit Strings

- **Definition**: Wie Sie sich aus JavaScript erinnern, sind Strings unveränderlich, was bedeutet, dass Sie sie nach ihrer Erstellung nicht ändern können. In Python können Sie entweder einfache oder doppelte Anführungszeichen verwenden. Es wird empfohlen, eine Regel zu wählen und dabei zu bleiben:

```py
developer = 'Jessica'
city = "Los Angeles"
```

- **Zugriff auf Zeichen in Strings**: Sie können auf Zeichen in Strings mit der Klammernotation zugreifen, so:

```py
my_str = 'Hello world'

print(my_str[0])  # H
print(my_str[6])  # w

print(my_str[-1])  # d
print(my_str[-2]) # l
```

- **Escape-Sequenzen in Strings**: Sie können einen Backslash (`\`) verwenden, wenn Ihr String Anführungszeichen enthält, so:

```py
msg = 'It\'s a sunny day'
quote = "She said, \"Hello!\""
```

- **String-Verkettung**: Um Strings zu verketten, können Sie den `+`-Operator verwenden, so:

```py
developer = 'Jessica'
print('My name is ' + developer + '.') # My name is Jessica.
```

Eine andere Möglichkeit, Strings zu verketten, ist die Verwendung des `+=`-Operators. Dieser wird verwendet, um Verkettung und Zuweisung in einem Schritt durchzuführen, so:

```py
greeting = 'My name is '
developer = 'Jessica.'

greeting += developer
print(greeting) # My name is Jessica.
```

- **`f-strings`**: Dies steht für formatierte String-Literale. Damit können Sie Interpolation durchführen und auch Verkettungen mit einer kompakten und lesbaren Syntax erledigen:

```py
developer = 'Jessica'
greeting = f'My name is {developer}.'
print(greeting) # My name is Jessica.
```

- **String-Slicing**: Dabei können Sie Teile eines Strings extrahieren. Hier ist die Grundsyntax:

```py
str[start:stop:step]
```

Die Startposition gibt den Index an, an dem die Extraktion beginnen soll. Die Stopp-Position ist der Punkt, an dem der Ausschnitt enden soll. Diese Position ist nicht inklusive. Die Schritt-Position gibt das Intervall an, um das beim Slicing inkrementiert wird. Hier sind einige Beispiele:

```py
message = 'Python is fun!'

print(message[0:6])  # Python
print(message[7:])  # is fun!
print(message[::2])  # Pto sfn
```

- **Länge eines Strings ermitteln**: Die `len()`-Funktion wird verwendet, um die Anzahl der Zeichen im String zurückzugeben:

```py
developer = 'Jessica'

print(len(developer)) # 7
```

## Arbeiten mit dem `in`-Operator

- **`in`-Operator**: Dieser gibt einen Boolean zurück, der angibt, ob das Zeichen oder die Zeichen im String vorhanden sind oder nicht:

```py
my_str = 'Hello world'

print('Hello' in my_str)  # True
print('hey' in my_str)    # False
print('hi' in my_str)    # False
print('e' in my_str)  # True
print('f' in my_str)  # False
```

## Häufige String-Methoden

- **`upper()`**: Diese String-Methode gibt einen neuen String zurück, bei dem alle Zeichen in Großbuchstaben umgewandelt wurden:

```py
developer = 'Jessica'

print(developer.upper()) # JESSICA
```

- **`lower()`**: Diese String-Methode gibt einen neuen String zurück, bei dem alle Zeichen in Kleinbuchstaben umgewandelt wurden:

```py
developer = 'Jessica'

print(developer.lower()) # jessica
```

- **`strip()`**: Diese String-Methode gibt eine Kopie des Strings zurück, bei der angegebene führende und abschließende Zeichen entfernt wurden (wenn kein Argument übergeben wird, werden führende und abschließende Leerzeichen entfernt).

```py
greeting = '  hello world  '

trimmed_my_str = greeting.strip()
print(trimmed_my_str)  # 'hello world'
```

- **`replace()`**: Diese Methode gibt einen neuen String zurück, bei dem alle Vorkommen des alten Strings durch einen neuen ersetzt wurden.

```py
greeting = 'hello world'

replaced_my_str = greeting.replace('hello', 'hi')
print(replaced_my_str)  # 'hi world'
```

- **`split()`**: Diese Methode wird verwendet, um einen String anhand eines angegebenen Trennzeichens in eine Liste zu teilen. Ein Trennzeichen ist ein String, der angibt, wo die Teilung erfolgen soll.

```py
dashed_name = 'example-dashed-name'

split_words = dashed_name.split('-')
print(split_words)  # ['example', 'dashed', 'name']
```

- **`join()`**: Diese Methode wird verwendet, um Elemente eines iterierbaren Objekts mit einem Trennzeichen zu einem String zusammenzufügen. Ein iterierbares Objekt ist eine Sammlung von Elementen, über die man wie über eine Liste, einen String oder ein Tuple iterieren kann.

```py
example_list = ['example', 'dashed', 'name']

joined_str = ' '.join(example_list)
print(joined_str)  # example dashed name
```

- **`startswith(prefix)`**: Diese String-Methode gibt einen Boolean zurück, der angibt, ob ein String mit dem angegebenen Präfix beginnt:

```py
developer = 'Naomi'

result = developer.startswith('N')
print(result)  # True
```

- **`endswith(suffix)`**: Diese String-Methode gibt einen Boolean zurück, der angibt, ob ein String mit dem angegebenen Suffix endet:

```py
developer = 'Naomi'

result = developer.endswith('N')
print(result)  # False
```

- **`find()`**: Diese String-Methode gibt den Index des ersten Vorkommens eines Substrings zurück. Wenn keiner gefunden wird, wird `-1` zurückgegeben:

```py
developer = 'Naomi'

result = developer.find('N')
print(result)  # 0

city = 'Los Angeles'
print(city.find('New')) # -1
```

- **`count(substring)`**: Diese String-Methode zählt, wie oft ein Substring in einem String vorkommt:

```py
city = 'Los Angeles'
print(city.count('e')) # 2
```

- **`capitalize()`**: Diese String-Methode gibt einen neuen String zurück, bei dem das erste Zeichen großgeschrieben und die anderen Zeichen kleingeschrieben sind:

```py
dessert = 'chocolate cake'
print(dessert.capitalize()) # Chocolate cake
```

- **`isupper()`**: Diese String-Methode gibt `True` zurück, wenn alle Buchstaben im String Großbuchstaben sind, und `False`, wenn nicht:

```py
dessert = 'chocolate cake'
print(dessert.isupper()) # False
```

- **`islower()`**: Diese String-Methode gibt `True` zurück, wenn alle Buchstaben im String Kleinbuchstaben sind, und `False`, wenn nicht:

```py
dessert = 'chocolate cake'
print(dessert.islower()) # True
```

- **`title()`**: Diese String-Methode gibt einen neuen String zurück, bei dem der erste Buchstabe jedes Wortes großgeschrieben ist:

```py
city = 'los angeles'
print(city.title()) # Los Angeles
```

- **`maketrans()`**: Diese String-Methode wird verwendet, um eine Tabelle von 1-zu-1-Zeichenzuordnungen für Übersetzungen zu erstellen. Sie wird oft mit der `translate()`-Methode verwendet, die diese Tabelle auf einen String anwendet und das übersetzte Ergebnis zurückgibt.

```py
trans_table = str.maketrans('abc', '123')
print(trans_table) # {97: 49, 98: 50, 99: 51}

result = 'abcabc'.translate(trans_table)
print(result)  # 123123
```

## Häufige Operationen mit Integern und Floats

- **Grundlegende mathematische Operationen**: In Python können Sie grundlegende mathematische Operationen mit Integern und Floats durchführen, einschließlich Addition, Subtraktion, Multiplikation und Division:

```py
int_1 = 56
int_2 = 12
float_1 = 5.4
float_2 = 12.0

# Addition

print('Integer Addition:', int_1 + int_2) # Integer Addition: 68
print('Float Addition:', float_1 + float_2) # Float Addition: 17.4

# Subtraction

print('Int Subtraction:', int_1 - int_2) # Int Subtraction: 44
print('Float Subtraction:',  float_2 - float_1) # Float Subtraction: 6.6

# Multiplication

print('Int Multiplication:', int_1 * int_2) # Int Multiplication: 672
print('Float Multiplication:', float_2 * float_1) # Float Multiplication: 64.80000000000001

# Division

print('Division:', int_1 / int_2) # Division: 4.666666666666667
print('Float Division:', float_2 / float_1) # Float Division: 2.222222222222222
```

Wenn Sie einen Float und einen Integer addieren, wird das Ergebnis in einen Float umgewandelt, so:

```py
int_1 = 56
float_1 = 5.4

print(int_1 + float_1) # 61.4
```

- **Modulo-Operator (`%`)**: Dieser gibt den Rest zurück, wenn eine Zahl durch eine andere geteilt wird:

```py
int_1 = 56
int_2 = 12

print(int_1 % int_2) # 8
```

- **Ganzzahldivision (`//`)**: Dieser Operator wird verwendet, um zwei Zahlen zu dividieren und das Ergebnis auf die nächste ganze Zahl abzurunden:

```py
int_1 = 56
int_2 = 12

print(int_1 // int_2) # 4
```

- **Exponentiation-Operator (`**`)**: Dieser Operator wird verwendet, um eine Zahl mit der Potenz einer anderen zu potenzieren:

```py
int_1 = 4
int_2 = 2

print(int_1 ** int_2) # 16
```

- **`float()`-Funktion**: Sie können diese Funktion verwenden, um einen Integer in einen Float umzuwandeln.

```py
num = 4

print(float(num)) # 4.0
```

- **`int()`-Funktion**: Sie können diese Funktion verwenden, um einen Float in einen Integer umzuwandeln.

```py
num = 4.0

print(int(num)) # 4
```

- **`round()`-Funktion**: Diese wird verwendet, um eine Zahl auf die nächste ganze Zahl zu runden:

```py
num_1 = 3.4
num_2 = 7.7

print(round(num_1)) # 3
print(round(num_2)) # 8
```

- **`abs()`-Funktion**: Diese wird verwendet, um den Absolutwert einer Zahl zurückzugeben:

```py
num = -13

print(abs(num)) # 13
```

- **`pow()`-Funktion**: Diese wird verwendet, um eine Zahl mit der Potenz einer anderen zu potenzieren:

```py
result = pow(2, 3) 
print(result)  # 8
```

## Erweiterte Zuweisungen

- **Definition**: Erweiterte Zuweisungen kombinieren eine binäre Operation mit einer Zuweisung in einem Schritt. Sie nehmen eine Variable, wenden eine Operation mit einem anderen Wert darauf an und speichern das Ergebnis wieder in derselben Variable.

```py
# Addition assignment 
my_var = 10
my_var += 5

print(my_var) # 15

# Subtraction assignment
count = 14
count -= 3

print(count) # 11

# Multiplication assignment 
product = 65
product *= 7

print(product) # 455

# Division assignment 
price = 100
price /= 4

print(price) # 25.0

# Floor Division assignment 
total_pages = 23
total_pages //= 5

print(total_pages) # 4

# Modulo assignment 
bits = 35
bits %= 2

print(bits) # 1

# Exponentiation assignment 
power = 2
power **= 3

print(power) # 8
```

Es gibt auch andere erweiterte Zuweisungsoperatoren, wie die für bitweise Operatoren. Dazu gehören `&=`, `^=`, `>>=` und `<<=`.

## Arbeiten mit Funktionen

- **Definition**: Funktionen sind wiederverwendbare Codeabschnitte, die Eingaben (Argumente) entgegennehmen und eine Ausgabe zurückgeben. Um eine Funktion aufzurufen, müssen Sie den Funktionsnamen gefolgt von runden Klammern referenzieren:

```py
# Defining a function

def get_sum(num_1, num_2):
    return num_1 + num_2

result = get_sum(3, 4) # function call
print(result) # 7
```

Wenn eine Funktion keinen Wert explizit zurückgibt, ist der Standard-Rückgabewert `None`:

```py
def greet():
    print('hello') 

result = greet() # hello
print(result) # None
```

Sie können auch Standardwerte für Parameter angeben, so:

```py
def get_sum(num_1, num_2=2):
    return num_1 + num_2

result = get_sum(3) 
print(result) # 5
```

Wenn Sie die Funktion mit einer falschen Anzahl von Argumenten aufrufen, erhalten Sie einen `TypeError`:

```py
def calculate_sum(a, b):
    print(a + b)

calculate_sum()

# TypeError: calculate_sum() missing 2 required positional arguments: 'a' and 'b'
```

```py
name = input('What is your name?') # User types 'Kolade' and presses Enter
```

```py
print(int(True)) # 1
```

## Gültigkeitsbereich in Python

- **Lokaler Gültigkeitsbereich**: Dies ist, wenn eine Variable, die innerhalb einer Funktion oder Klasse deklariert wurde, nur innerhalb dieser Funktion oder Klasse aufgerufen werden kann.

```py
def my_func():
    num = 10
    print(num)
```

- **Einschließender Gültigkeitsbereich**: Dies ist, wenn eine Funktion, die in einer anderen Funktion verschachtelt ist, auf die Variablen der umschließenden Funktion zugreifen kann.

```py
def outer_func():
    msg = 'Hello there!'
    
    def inner_func():
        print(msg)
    inner_func()

print(outer_func()) # Hello there!
```

- **Globaler Gültigkeitsbereich**: Dies bezieht sich auf Variablen, die außerhalb von Funktionen oder Klassen deklariert wurden und von überall im Programm aufgerufen werden können.

```py
tax = 0.70 

def get_total(subtotal):
    total = subtotal + (subtotal * tax)
    return total

print(get_total(100))  # 170.0
```

- **Eingebauter Gültigkeitsbereich**: Reservierte Namen in Python für vordefinierte Funktionen, Module, Schlüsselwörter und Objekte.

```py
print(str(45)) # '45'
print(type(3.14)) # <class 'float'>
print(isinstance(3, str)) # False
```

## Vergleichsoperatoren

- **Gleich (`==`)**: Prüft, ob zwei Werte gleich sind:

```py
print(3 == 4) # False
```

- **Ungleich (`!=`)**: Prüft, ob zwei Werte ungleich sind:

```py
print(3 != 4) # True
```

- **Streng größer als (`>`)**: Prüft, ob ein Wert größer als ein anderer ist:

```py
print(3 > 4) # False
```

- **Streng kleiner als (`<`)**: Prüft, ob ein Wert kleiner als ein anderer ist:

```py
print(3 < 4) # True
```

- **Größer oder gleich (`>=`)**: Prüft, ob ein Wert größer oder gleich einem anderen ist:

```py
print(3 >= 4) # False
```

- **Kleiner oder gleich (`<=`)**: Prüft, ob ein Wert kleiner oder gleich einem anderen ist:

```py
print(3 <= 4) # True
```

## Arbeiten mit `if`, `elif` und `else`-Anweisungen

- **`if`-Anweisungen**: Dies sind Bedingungen, die verwendet werden, um zu bestimmen, ob etwas wahr ist oder nicht. Wenn die Bedingung zu `True` ausgewertet wird, wird dieser Codeblock ausgeführt.

```py
age = 18

if age >= 18:
    print('You are an adult') # You are an adult
```

- **`elif`-Klausel**: Dies sind Bedingungen, die nach einer `if`-Anweisung kommen. Wenn die `elif`-Bedingung zu `True` ausgewertet wird, wird dieser Codeblock ausgeführt.

```py
age = 16

if age >= 18:
    print('You are an adult')
elif age >= 13:
    print('You are a teenager')  # You are a teenager
```

- **`else`-Klausel**: Diese wird ausgeführt, wenn keine andere Bedingung zu `True` ausgewertet wird.

```py
age = 12

if age >= 18:
    print('You are an adult')
elif age >= 13:
    print('You are a teenager')
else:
    print('You are a child')  # You are a child
```

Sie können auch verschachtelte `if`-Anweisungen wie folgt verwenden:

```py
is_citizen = True
age = 25

if is_citizen:
    if age >= 18:
        print('You are eligible to vote') # You are eligible to vote
else:
    print('You are not eligible to vote')
```

## Truthy- und Falsy-Werte

- **Definition**: In Python hat jeder Wert einen inhärenten Boolean-Wert oder ein eingebautes Verständnis, ob er in einem logischen Kontext als `True` oder `False` behandelt werden soll. Viele Werte gelten als truthy, das heißt, sie werden in einem logischen Kontext als `True` ausgewertet. Andere sind falsy, was bedeutet, dass sie als `False` ausgewertet werden. Hier sind einige Beispiele für falsy-Werte:

```md
None
False
Integer 0
Float 0.0
Empty strings ''
```

Andere Werte wie Nicht-Null-Zahlen und nicht-leere Strings sind truthy.

## Arbeiten mit der `bool()`-Funktion

- **Definition**: Wenn Sie überprüfen möchten, ob ein Wert truthy oder falsy ist, können Sie die eingebaute `bool()`-Funktion verwenden. Sie wandelt einen Wert explizit in sein Boolean-Äquivalent um und gibt `True` für truthy-Werte und `False` für falsy-Werte zurück. Hier sind einige Beispiele:

```py
print(bool(False)) # False
print(bool(0))  # False
print(bool('')) # False

print(bool(True)) # True
print(bool(1)) # True
print(bool('Hello')) # True
```

## Boolean-Operatoren und Kurzschlussauswertung

- **Definition**: Dies sind spezielle Operatoren, mit denen Sie mehrere Ausdrücke kombinieren können, um komplexere Entscheidungslogiken in Ihrem Code zu erstellen. Es gibt drei Boolean-Operatoren in Python: `and`, `or` und `not`.
- **`and`-Operator**: Dieser Operator nimmt zwei Operanden und gibt den ersten Operand zurück, wenn er falsy ist, andernfalls gibt er den zweiten Operand zurück. Beide Operanden müssen truthy sein, damit ein Ausdruck einen truthy-Wert ergibt.

```py
is_citizen = True
age = 25

print(is_citizen and age) # 25
```

Sie können den `and`-Operator auch in Bedingungen wie folgt verwenden:

```py
is_citizen = True
age = 25

if is_citizen and age >= 18:
    print('You are eligible to vote') # You are eligible to vote
else:
    print('You are not eligible to vote')
```

- **`or`-Operator**: Dieser Operator gibt den ersten Operand zurück, wenn er truthy ist, andernfalls den zweiten Operand. Ein `or`-Ausdruck ergibt einen truthy-Wert, wenn mindestens ein Operand truthy ist. Hier ist ein Beispiel:

```py
age = 19
is_employed = False

print(age or is_employed) # 19
```

Wie beim `and`-Operator können Sie den `or`-Operator in Bedingungen wie folgt verwenden:

```py
age = 19
is_student = True

if age < 18 or is_student:
    print('You are eligible for a student discount') # You are eligible for a student discount
else:
    print('You are not eligible for a student discount')
```

- **Kurzschlussauswertung**: Die `and`- und `or`-Operatoren sind als Kurzschlussoperatoren bekannt. Kurzschlussauswertung bedeutet, dass Python Werte von links nach rechts prüft und stoppt, sobald das endgültige Ergebnis bestimmt ist.
- **`not`-Operator**: Dieser Operator nimmt einen einzelnen Operanden und invertiert dessen Boolean-Wert. Er wandelt truthy-Werte in `False` und falsy-Werte in `True` um. Im Gegensatz zu den vorherigen Operatoren gibt `not` immer `True` oder `False` zurück. Hier sind einige Beispiele:

```py
print(not '') # True, because empty string is falsy
print(not 'Hello') # False, because non-empty string is truthy
print(not 0) # True, because 0 is falsy
print(not 1) # False, because 1 is truthy
print(not False) # True, because False is falsy
print(not True) # False, because True is truthy
```

Hier ist ein Beispiel für den `not`-Operator in einer Bedingung:

```py
is_admin = False

if not is_admin:
    print('Access denied for non-administrators.') # Access denied for non-administrators.
else:
    print('Welcome, Administrator!')
```


## Python-Listen

- **Einführung**: In Python ist der List-Datentyp eine geordnete Folge von Elementen, die aus Strings, Zahlen oder sogar anderen Listen bestehen können. Listen sind veränderlich und nullbasiert indiziert.

```python
cities = ['Los Angeles', 'London', 'Tokyo']
```

- **Zugriff auf Elemente in einer Liste**: Um auf ein Element aus der `cities`-Liste zuzugreifen, können Sie dessen Indexnummer in der Folge referenzieren:

```python
cities = ['Los Angeles', 'London', 'Tokyo']
cities[0] # Los Angeles
```

- **Zugriff auf Elemente mit negativer Indizierung**: Um auf das letzte Element einer Liste zuzugreifen, können Sie `-1` als Indexnummer verwenden:

```python
cities = ['Los Angeles', 'London', 'Tokyo']
cities[-1] # Tokyo
```

- Negative Indizierung wird verwendet, um Elemente vom Ende der Liste aus zuzugreifen, anstatt vom Anfang bei Index `0`.

- **Listen mit dem `list()`-Konstruktor erstellen**: Listen können auch mit dem `list()`-Konstruktor erstellt werden. Der `list()`-Konstruktor wird verwendet, um ein iterierbares Objekt in eine Liste umzuwandeln:

```python
developer = 'Jessica'

print(list(developer)) 
# Result: ['J', 'e', 's', 's', 'i', 'c', 'a']
```

- **Länge einer Liste ermitteln**: Sie können die `len()`-Funktion verwenden, um die Länge einer Liste zu erhalten:

```python
numbers = [1, 2, 3, 4, 5]
len(numbers) # 5
```

- **Veränderlichkeit von Listen**: Listen sind veränderlich, das heißt, Sie können jedes Element in der Liste aktualisieren, solange Sie eine gültige Indexnummer angeben. Um Listen an einem bestimmten Index zu aktualisieren, können Sie diesem Index einen neuen Wert zuweisen:

```python
programming_languages = ['Python', 'Java', 'C++', 'Rust']
programming_languages[0] = 'JavaScript'
print(programming_languages) # ['JavaScript', 'Java', 'C++', 'Rust']
```

- **Index-Fehler außerhalb des Bereichs**: Wenn Sie einen Index (entweder positiv oder negativ) angeben, der außerhalb des Bereichs der Liste liegt, erhalten Sie einen `IndexError`:

```python
programming_languages = ['Python', 'Java', 'C++', 'Rust']
programming_languages[10] = 'JavaScript'

"""
Traceback (most recent call last):
  File "<stdin>", line 1, in <module>
IndexError: list assignment index out of range
"""
```

- **Elemente aus einer Liste entfernen**: Elemente können mit dem Schlüsselwort `del` aus einer Liste entfernt werden:

```python
developer = ['Jane Doe', 23, 'Python Developer']
del developer[1]
print(developer) # ['Jane Doe', 'Python Developer']
```

- **Überprüfen, ob ein Element in einer Liste existiert**: Das Schlüsselwort `in` kann verwendet werden, um zu überprüfen, ob ein Element in einer Liste vorhanden ist:

```py
programming_languages = ['Python', 'Java', 'C++', 'Rust']

'Rust' in programming_languages # True
'JavaScript' in programming_languages # False
```

- **Verschachtelung von Listen**: Listen können in anderen Listen verschachtelt sein:

```python
developer = ['Alice', 25, ['Python', 'Rust', 'C++']]
```

- Um auf die verschachtelte Liste zuzugreifen, müssen Sie sie mit dem Index `2` aufrufen, da Listen nullbasiert indiziert sind.

```python
developer = ['Alice', 25, ['Python', 'Rust', 'C++']]
developer[2] # ['Python', 'Rust', 'C++']
```

- Um weiter auf die zweite Sprache aus dieser verschachtelten Liste zuzugreifen, müssen Sie den Index `1` verwenden:

```python
developer = ['Alice', 25, ['Python', 'Rust', 'C++']]
developer[2][1] # Rust
```

- **Werte aus einer Liste entpacken:** Das Entpacken von Werten aus einer Liste ist eine Technik, um Werte aus einer Liste neuen Variablen zuzuweisen. Hier ist ein Beispiel, um die `developer`-Liste in neue Variablen namens `name`, `age` und `job` zu entpacken:

```py
developer = ['Alice', 34, 'Rust Developer']
name, age, job = developer
```

- Wenn die Anzahl der Variablen auf der linken Seite des Zuweisungsoperators nicht mit der Gesamtzahl der Elemente in der Liste übereinstimmt, erhalten Sie einen `ValueError`.

- **Verbleibende Elemente aus einer Liste sammeln**: Um verbleibende Elemente aus einer Liste zu sammeln, können Sie den Sternchenoperator (`*`) wie folgt verwenden:

```py
developer = ['Alice', 34, 'Rust Developer']
name, *rest = developer
```

- **Listen slicen**: Slicing ist das Konzept, einen Teil einer Liste mit dem Slice-Operator `:` zuzugreifen. Um eine Liste zu slicen, die bei Index `1` beginnt und vor Index `3` endet, können Sie folgende Syntax verwenden:

```python
desserts = ['Cake', 'Cookies', 'Ice Cream', 'Pie']
desserts[1:3] # ['Cookies', 'Ice Cream']
```

- **Schrittintervalle**: Es ist auch möglich, ein Schrittintervall anzugeben, das bestimmt, wie viel zwischen den Indizes inkrementiert wird. Hier ist ein Beispiel, wenn Sie eine Liste nur mit geraden Zahlen mithilfe von Slicing extrahieren möchten:

```python
numbers = [1, 2, 3, 4, 5, 6]
numbers[1::2] # [2, 4, 6]
```

## Listen-Methoden

- **append()**: Wird verwendet, um ein Element am Ende der Liste hinzuzufügen. Hier ist ein Beispiel für die Verwendung der `append()`-Methode, um die Zahl `6` zu dieser `numbers`-Liste hinzuzufügen:

```py
numbers = [1, 2, 3, 4, 5]
numbers.append(6)
print(numbers) # [1, 2, 3, 4, 5, 6]
```

- **Listen anhängen**: Die `append()`-Methode kann auch verwendet werden, um eine Liste am Ende einer anderen hinzuzufügen:

```py
numbers = [1, 2, 3, 4, 5]
even_numbers = [6, 8, 10]

numbers.append(even_numbers)
print(numbers) # [1, 2, 3, 4, 5, [6, 8, 10]]
```

- **extend()**: Wird verwendet, um mehrere Elemente am Ende einer Liste hinzuzufügen. Hier ist ein Beispiel, um die Zahlen `6`, `8` und `10` am Ende der `numbers`-Liste hinzuzufügen:

```py
numbers = [1, 2, 3, 4, 5]
even_numbers = [6, 8, 10]

numbers.extend(even_numbers)
print(numbers) # [1, 2, 3, 4, 5, 6, 8, 10]
```

- **insert()**: Wird verwendet, um ein Element an einem bestimmten Index in der Liste einzufügen. Hier ist ein Beispiel für die Verwendung der `insert()`-Methode:

```py
numbers = [1, 2, 3, 4, 5]
numbers.insert(2, 2.5)

print(numbers) # [1, 2, 2.5, 3, 4, 5]
```

- **remove()**: Wird verwendet, um ein Element aus der Liste zu entfernen. Die `remove()`-Methode entfernt nur das erste Vorkommen eines Elements in der Liste:

```py
numbers = [1, 2, 3, 4, 5, 5, 5]
numbers.remove(5)

print(numbers) # [1, 2, 3, 4, 5, 5]

```

- **pop()**: Wird verwendet, um ein bestimmtes Element aus der Liste zu entfernen und zurückzugeben:

```py
numbers = [1, 2, 3, 4, 5]
numbers.pop(1) # The number 2 is returned
```

- Wenn Sie kein Element für die `pop`-Methode angeben, wird das letzte Element entfernt.

```py
numbers = [1, 2, 3, 4, 5]
numbers.pop() # The number 5 is returned
```

- **clear()**: Wird verwendet, um alle Elemente aus der Liste zu entfernen:

```py
numbers = [1, 2, 3, 4, 5]
numbers.clear()

print(numbers) # []
```

- **sort()**: Die `sort()`-Methode wird verwendet, um die Elemente an Ort und Stelle zu sortieren. Hier ist ein Beispiel, um eine zufällige Liste von `numbers` an Ort und Stelle zu sortieren:

```py
numbers = [19, 2, 35, 1, 67, 41]
numbers.sort()

print(numbers) # [1, 2, 19, 35, 41, 67]
```

- **sorted()**: Eine eingebaute Funktion, die eine neue sortierte Liste zurückgibt, anstatt die ursprüngliche Liste zu verändern:

```py
numbers = [19, 2, 35, 1, 67, 41]
sorted_numbers = sorted(numbers)

print(sorted_numbers) # [1, 2, 19, 35, 41, 67]
print(numbers) # [19, 2, 35, 1, 67, 41]
```

- **reverse()**: Wird verwendet, um die Reihenfolge der Elemente in einer Liste umzukehren:

```py
numbers = [6, 5, 4, 3, 2, 1]
numbers.reverse()

print(numbers) # [1, 2, 3, 4, 5, 6]
```

- **index()**: Wird verwendet, um den ersten Index zu finden, an dem ein Element in einer Liste gefunden werden kann:

```python
programming_languages = ['Rust', 'Java', 'Python', 'C++']
programming_languages.index('Java') # 1
```

- Wenn das Element mit der `index()`-Methode nicht gefunden werden kann, ist das Ergebnis ein `ValueError`.

## Tupel in Python

- **Definition**: Ein Tuple ist ein Python-Datentyp, der verwendet wird, um eine geordnete Folge von Werten zu erstellen. Tuples können eine gemischte Menge von Datentypen enthalten:

```py
developer = ('Alice', 34, 'Rust Developer')
```

- Tuples sind unveränderlich, das heißt, die Elemente im Tuple können nach der Erstellung nicht geändert werden. Wenn Sie versuchen, eines der Elemente im Tuple zu aktualisieren, erhalten Sie einen `TypeError`:

```py
programming_languages = ('Python', 'Java', 'C++', 'Rust')
programming_languages[0] = 'JavaScript'

"""
Traceback (most recent call last):
  File "<stdin>", line 1, in <module>
TypeError: "tuple" object does not support item assignment
"""
```

- **Zugriff auf Elemente aus einem Tuple**: Um auf ein Element aus einem Tuple zuzugreifen, verwenden Sie die Klammernotation und die Indexnummer:

```py
developer = ('Alice', 34, 'Rust Developer')
developer[1] # 34
```

- Negative Indizierung kann verwendet werden, um Elemente vom Ende des Tuples aus zuzugreifen:

```py
numbers = (1, 2, 3, 4, 5)
numbers[-2] # 4
```

- Wenn Sie versuchen, eine Indexnummer anzugeben, die die Länge des Tuples überschreitet oder gleich ist, erhalten Sie einen `IndexError`:

```py
numbers = (1, 2, 3, 4, 5)
numbers[7]

"""
Traceback (most recent call last):
  File "<stdin>", line 1, in <module>
IndexError: tuple index out of range
"""
```

- Ein Tuple kann auch mit dem `tuple()`-Konstruktor erstellt werden. Innerhalb des Konstruktors können Sie verschiedene iterierbare Objekte wie Strings, Listen und sogar andere Tuples übergeben.

```python
developer = 'Jessica'

print(tuple(developer)) 
# Result: ('J', 'e', 's', 's', 'i', 'c', 'a')
```

- **Überprüfen von Elementen in einem Tuple**: Um zu prüfen, ob ein Element in einem Tuple enthalten ist, können Sie das Schlüsselwort `in` wie folgt verwenden:

```py
programming_languages = ('Python', 'Java', 'C++', 'Rust')

'Rust' in programming_languages # True
'JavaScript' in programming_languages # False
```

- **Tuples entpacken**: Elemente können aus einem Tuple wie folgt entpackt werden:

```py
developer = ('Alice', 34, 'Rust Developer')
name, age, job = developer
```

- Wenn Sie verbleibende Elemente aus einem Tuple sammeln müssen, können Sie den Sternchenoperator (`*`) wie folgt verwenden:

```py
developer = ('Alice', 34, 'Rust Developer')
name, *rest = developer
```

- **Tuples slicen**: Slicing kann verwendet werden, um einen Teil eines Tuples zu extrahieren. Zum Beispiel können die Elemente `pie` und `cookies` in ein separates Tuple gesliced werden:

```py
desserts = ('cake', 'pie', 'cookies', 'ice cream')
desserts[1:3] # ('pie', 'cookies')
```

- **Elemente aus Tuples entfernen**: Das Entfernen eines Elements aus einem Tuple führt zu einem `TypeError`, da Tuples unveränderlich sind:

```py
developer = ('Jane Doe', 23, 'Python Developer')
del developer[1]

"""
Traceback (most recent call last):
  File "<stdin>", line 1, in <module>
TypeError: "tuple" object doesn't support item deletion
"""
```

- **Wann sollte man ein Tuple statt einer Liste verwenden?**: Wenn Sie eine dynamische Sammlung von Elementen benötigen, bei der Sie Elemente hinzufügen, entfernen und aktualisieren können, sollten Sie eine Liste verwenden. Wenn Sie wissen, dass Sie mit einer festen und unveränderlichen Datensammlung arbeiten, sollten Sie ein Tuple verwenden.

## Häufige Tuple-Methoden

- **`count()`**: Wird verwendet, um zu bestimmen, wie oft ein Element in einem Tuple vorkommt. Zum Beispiel können Sie überprüfen, wie oft die Sprache `'Rust'` im Tuple vorkommt:

```py
programming_languages = ('Rust', 'Java', 'Python', 'C++', 'Rust')
programming_languages.count('Rust') # 2
```

- Wenn das angegebene Element in der `count()`-Methode überhaupt nicht im Tuple vorhanden ist, dann ist der Rückgabewert `0`:

```py
programming_languages = ('Rust', 'Java', 'Python', 'C++', 'Rust')
programming_languages.count('JavaScript') # 0
```

- Wenn keine Argumente an die `count()`-Methode übergeben werden, gibt Python standardmäßig einen `TypeError` zurück.

- **index()**: Wird verwendet, um den Index zu finden, an dem ein bestimmtes Element im Tuple vorhanden ist. Hier ist ein Beispiel für die Verwendung der `index()`-Methode, um den Index für die Sprache `'Java'` zu finden:

```py
programming_languages = ('Rust', 'Java', 'Python', 'C++', 'Rust')
programming_languages.index('Java') # 1
```

- Wenn das angegebene Element nicht gefunden werden kann, gibt Python einen `ValueError` zurück.

- Sie können einen optionalen Startindex an die `index()`-Methode übergeben, um anzugeben, wo die Suche nach dem Element im Tuple beginnen soll:

```py
programming_languages = ('Rust', 'Java', 'Python', 'C++', 'Rust', 'Python')
programming_languages.index('Python', 3) # 5
```

- Sie können auch einen optionalen Endindex an die `index()`-Methode übergeben, um anzugeben, wo die Suche nach dem Element im Tuple enden soll:

```py
programming_languages = ('Rust', 'Java', 'Python', 'C++', 'Rust', 'Python', 'JavaScript', 'Python')
programming_languages.index('Python', 2, 5) # 2
```

- **`sorted()`**: Wird verwendet, um die Elemente in einem iterierbaren Objekt zu sortieren und eine neue sortierte Liste zurückzugeben. Hier ist ein Beispiel, wie man mit der `sorted()`-Funktion eine neue Liste von Zahlen erstellt:

```py
numbers = (13, 2, 78, 3, 45, 67, 18, 7)
sorted(numbers) # [2, 3, 7, 13, 18, 45, 67, 78]
```

- **Sortierverhalten anpassen**: Sie können das Sortierverhalten für ein iterierbares Objekt mit den optionalen Argumenten `reverse` und `key` anpassen. Hier ist ein Beispiel, wie man das Argument `key` verwendet, um Elemente in einem Tuple nach Länge zu sortieren:

```py
programming_languages = ('Rust', 'Java', 'Python', 'C++', 'Rust', 'Python')
sorted(programming_languages, key=len)

# Result
# ['C++', 'Rust', 'Java', 'Rust', 'Python', 'Python']
```

- Sie können eine neue Liste von Werten in umgekehrter Reihenfolge erstellen, indem Sie das Argument `reverse` so verwenden:

```py
programming_languages = ('Rust', 'Java', 'Python', 'C++', 'Rust', 'Python')

print(sorted(programming_languages, reverse=True))

# Result
# ['Rust', 'Rust', 'Python', 'Python', 'Java', 'C++']
```

## Schleifen in Python

- **Definition**: Schleifen werden verwendet, um einen Codeblock eine festgelegte Anzahl von Malen zu wiederholen.

- **`for`-Schleife**: Wird verwendet, um über eine Folge (wie eine Liste, ein Tuple oder einen String) zu iterieren und für jedes Element in dieser Folge einen Codeblock auszuführen. Hier ist ein Beispiel für die Verwendung einer `for`-Schleife, um eine Liste zu durchlaufen und jede Sprache in der Konsole auszugeben:

```py
programming_languages = ['Rust', 'Java', 'Python', 'C++']

for language in programming_languages:
    print(language)

"""
Result 

Rust
Java
Python
C++
"""
```

- Hier ist ein Beispiel für die Verwendung einer `for`-Schleife, um den String `code` zu durchlaufen und jedes Zeichen auszugeben:

```py
for char in 'code':
    print(char)

"""
Result 

c
o
d
e
"""
```

- `for`-Schleifen können verschachtelt werden. Hier ist ein Beispiel für die Verwendung einer verschachtelten `for`-Schleife:

```py
categories = ['Fruit', 'Vegetable']
foods = ['Apple', 'Carrot', 'Banana']

for category in categories:
    for food in foods:
        print(category, food)

"""
Result

Fruit Apple
Fruit Carrot
Fruit Banana
Vegetable Apple
Vegetable Carrot
Vegetable Banana
"""

```

- **`while`-Schleife**: Wiederholt einen Codeblock, bis die Bedingung `False` ist. Hier ist ein Beispiel für die Verwendung einer `while`-Schleife für ein Ratespiel:

```py
secret_number = 3
guess = 0

while guess != secret_number:
    guess = int(input('Guess the number (1-5): '))
    if guess != secret_number:
        print('Wrong! Try again.')

print('You got it!')

"""
Result

Guess the number (1-5): 2
Wrong! Try again.
Guess the number (1-5): 1
Wrong! Try again.
Guess the number (1-5): 3
You got it!
"""

```

- **`break`- und `continue`-Anweisungen**: Werden in Schleifen verwendet, um die Ausführung der Schleife zu verändern.

- Die `break`-Anweisung wird verwendet, um die Schleife sofort zu verlassen, wenn eine bestimmte Bedingung erfüllt ist. Hier ist ein Beispiel für die Verwendung der `break`-Anweisung für eine Liste von `developer_names`:

```py
developer_names = ['Jess', 'Naomi', 'Tom']

for developer in developer_names:
    if developer == 'Naomi':
        break
    print(developer)
```

- Die `continue`-Anweisung wird verwendet, um die aktuelle Iteration zu überspringen und mit der nächsten Iteration der Schleife fortzufahren. Hier ist ein Beispiel, wie man die `continue`-Anweisung anstelle einer `break`-Anweisung verwendet:

```py
developer_names = ['Jess', 'Naomi', 'Tom']

for developer in developer_names:
    if developer == 'Naomi':
        continue
    print(developer)
```

- Sowohl `for`- als auch `while`-Schleifen können mit einer `else`-Klausel kombiniert werden, die nur ausgeführt wird, wenn die Schleife nicht durch eine `break` beendet wurde:

```python
words = ['sky', 'apple', 'rhythm', 'fly', 'orange']

for word in words:
    for letter in word:
        if letter.lower() in 'aeiou':
            print(f"'{word}' contains the vowel '{letter}'")
            break
    else:
        print(f"'{word}' has no vowels")
```

## Bereiche und ihre Verwendung in Schleifen

- **Die `range()`-Funktion**: Wird verwendet, um eine Folge von ganzen Zahlen zu erzeugen.

```py
range(start, stop, step)
```

- Das erforderliche Argument `stop` ist eine ganze Zahl (nicht eingeschlossen), die den Endpunkt für die erzeugte Zahlenfolge darstellt. Hier ist ein Beispiel für die Verwendung der `range()`-Funktion:

```py
for num in range(3):
    print(num)
```

- Wenn kein `start`-Argument angegeben wird, ist der Standardwert `0`. Standardmäßig erhöht sich die Folge ganzer Zahlen um `1`. Sie können das optionale Argument `step` verwenden, um den Standardinkrementwert zu ändern. Hier ist ein Beispiel, wie man eine Folge von geraden ganzen Zahlen von 2 bis einschließlich 10 erzeugt (also bis, aber nicht einschließlich 11):

```py
for num in range(2, 11, 2):
    print(num)
```

- Wenn Sie der `range()`-Funktion keine Argumente übergeben, erhalten Sie eine `TypeError`.

- Die `range()`-Funktion akzeptiert nur ganze Zahlen als Argumente, keine Fließkommazahlen. Die Verwendung von Fließkommazahlen führt ebenfalls zu einem `TypeError`:

```py
ERROR!
Traceback (most recent call last):
  File "<main.py>", line 1, in <module>
TypeError: 'float' object cannot be interpreted as an integer
```

- Sie können für das Argument `step` eine negative ganze Zahl verwenden, um eine Folge von ganzen Zahlen in absteigender Reihenfolge zu erzeugen:

```py
for num in range(40, 0, -10):
    print(num)
```

- Die `range()`-Funktion kann auch verwendet werden, um eine Liste von ganzen Zahlen zu erstellen, indem sie mit dem `list`-Konstruktor kombiniert wird. Der `list`-Konstruktor wird verwendet, um ein iterierbares Objekt in eine Liste umzuwandeln. Hier ist ein Beispiel, wie man eine Liste von geraden ganzen Zahlen zwischen 2 und 10 inklusive erzeugt:

```py
numbers = list(range(2, 11, 2))
print(numbers) # [2, 4, 6, 8, 10]
```

## `enumerate()`- und `zip()`-Funktionen in Python

- **`enumerate()`**: Wird verwendet, um über eine Folge zu iterieren und für jedes Element den Index zu verfolgen. Die `enumerate()`-Funktion nimmt ein iterierbares Objekt als Argument und gibt ein `enumerate`-Objekt zurück, das aus dem Index und dem Wert jedes Elements im iterierbaren Objekt besteht.

```python
languages = ['Spanish', 'English', 'Russian', 'Chinese']

for index, language in enumerate(languages):
    print(f'Index {index} and language {language}')

# Result
# Index 0 and language Spanish
# Index 1 and language English
# Index 2 and language Russian
# Index 3 and language Chinese

```

- Die `enumerate()`-Funktion kann auch außerhalb einer `for`-Schleife verwendet werden:

```python
languages = ['Spanish', 'English', 'Russian', 'Chinese']

print(list(enumerate(languages)))
# [(0, 'Spanish'), (1, 'English'), (2, 'Russian'), (3, 'Chinese')]
```

- Die `enumerate()`-Funktion akzeptiert auch ein optionales Argument `start`, das den Startwert für die Zählung angibt. Wenn dieses Argument weggelassen wird, beginnt die Zählung bei `0`.

- **`zip()`**: Wird verwendet, um parallel über mehrere iterierbare Objekte zu iterieren. Hier ist ein Beispiel für die Verwendung der `zip()`-Funktion, um über `developers` und `ids` zu iterieren:

```python
developers = ['Naomi', 'Dario', 'Jessica', 'Tom']
ids = [1, 2, 3, 4]

for name, dev_id in zip(developers, ids):
    print(f'Name: {name}')
    print(f'ID: {dev_id}')


"""
Result

Name: Naomi
ID: 1
Name: Dario
ID: 2
Name: Jessica
ID: 3
Name: Tom
ID: 4
"""

```

## Listen-Komprehensionen in Python

- **Definition**: List Comprehension ermöglicht es Ihnen, eine neue Liste in einer einzigen Zeile zu erstellen, indem Sie die Schleife und die Bedingung direkt innerhalb von eckigen Klammern kombinieren. Dadurch wird der Code kürzer und oft leichter lesbar.

```py
even_numbers = [num for num in range(21) if num % 2 == 0]
print(even_numbers)
```

## Iterable-Funktionen

- **`filter()`**: Wird verwendet, um Elemente aus einem iterierbaren Objekt basierend auf einer Bedingung zu filtern. Es gibt einen Iterator zurück, der nur die Elemente enthält, die die Bedingung erfüllen. Hier ist ein Beispiel, wie man eine neue Liste mit nur Wörtern erstellt, die länger als vier Zeichen sind:

```py
words = ['tree', 'sky', 'mountain', 'river', 'cloud', 'sun']

def is_long_word(word):
    return len(word) > 4

long_words = list(filter(is_long_word, words))
print(long_words) # ['mountain', 'river', 'cloud']
```

- **`map()`**: Wird verwendet, um eine Funktion auf jedes Element in einem iterierbaren Objekt anzuwenden und ein neues iterierbares Objekt mit den Ergebnissen zurückzugeben. Hier ist ein Beispiel für die Verwendung der `map()`-Funktion, um eine Liste von Celsius-Temperaturen in Fahrenheit umzuwandeln:

```py
celsius = [0, 10, 20, 30, 40]

def to_fahrenheit(temp):
    return (temp * 9/5) + 32

fahrenheit = list(map(to_fahrenheit, celsius))
print(fahrenheit) # [32.0, 50.0, 68.0, 86.0, 104.0]
```

- **`sum()`**: Wird verwendet, um die Summe aus einem iterierbaren Objekt wie einer Liste oder einem Tuple zu erhalten. Hier ist ein Beispiel für die Verwendung der `sum()`-Funktion:

```py
numbers = [5, 10, 15, 20]
total = sum(numbers)
print(total) # Result: 50
```

- Sie können auch ein optionales Argument `start` übergeben, das den Anfangswert für die Summierung festlegt. Hier ist ein aktualisiertes Beispiel, das das Argument `start` als Positionsargument verwendet:

```py
numbers = [5, 10, 15, 20]
total = sum(numbers, 10) # positional argument
print(total) # 60
```

- Sie können das Argument `start` auch als Schlüsselwortargument wie folgt verwenden:

```py
numbers = [5, 10, 15, 20]
total = sum(numbers, start=10) # keyword argument
print(total) # 60
```

## Lambda-Funktionen

- **Definition**: Eine Lambda-Funktion in Python ist eine kurze Möglichkeit, eine Funktion ohne Namen (eine anonyme Funktion) zu erstellen.  
- Lambda-Funktionen werden oft als Argument an eine andere Funktion übergeben. Hier ist ein Beispiel für eine Lambda-Funktion:

```py
numbers = [1, 2, 3, 4, 5]

even_numbers = list(filter(lambda x: x % 2 == 0, numbers))
print(even_numbers)  # [2, 4]
```

- Bewährte Vorgehensweisen für die Verwendung von Lambda-Funktionen sind, sie nicht einer Variablen zuzuweisen, sie einfach und lesbar zu halten und sie für kurze, einmalige Funktionen zu verwenden.

## Dictionaries

- **Dictionaries**: Dictionaries sind eingebaute Datenstrukturen, die Sammlungen von Schlüssel-Wert-Paaren speichern. Schlüssel müssen unveränderliche Datentypen sein. Dies ist die allgemeine Syntax eines Python-Dictionaries:

```python
dictionary = {
    key1: value1,
    key2: value2
}
```

- **`dict()`-Konstruktor**: Der `dict()`-Konstruktor ist eine alternative Möglichkeit, ein Dictionary zu erstellen. Sie übergeben eine Liste von Tuples als Argument an den `dict()`-Konstruktor. Diese Tuples enthalten den Schlüssel als erstes Element und den Wert als zweites Element.

```python
pizza = dict([('name', 'Margherita Pizza'), ('price', 8.9), ('calories_per_slice', 250), ('toppings', ['mozzarella', 'basil'])])
```

- **Klammernotation**: Um auf den Wert eines Schlüssel-Wert-Paares zuzugreifen, können Sie die Syntax verwenden, die als Klammernotation bekannt ist.

```python
dictionary[key]
```

## Häufige Dictionary-Methoden

- **`get()`-Methode**: Die `get()`-Methode ruft den Wert ab, der mit einem Schlüssel verknüpft ist. Sie ist ähnlich wie die Klammernotation, ermöglicht es Ihnen jedoch, einen Standardwert festzulegen, um Fehler zu vermeiden, falls der Schlüssel nicht existiert.

```python
dictionary.get(key, default)
```

- **`keys()`- und `values()`-Methoden**: Die `keys()`- und `values()`-Methoden geben jeweils ein View-Objekt mit allen Schlüsseln und Werten im Dictionary zurück. Ein View-Objekt ist eine Möglichkeit, den Inhalt eines Dictionaries zu sehen, ohne eine separate Kopie der Daten zu erstellen.

```python
pizza = {
    'name': 'Margherita Pizza',
    'price': 8.9,
    'calories_per_slice': 250
}

pizza.keys()
# dict_keys(['name', 'price', 'calories_per_slice'])

pizza.values()
# dict_values(['Margherita Pizza', 8.9, 250])
```

- **`items()`-Methode**: Die `items()`-Methode gibt ein View-Objekt mit allen Schlüssel-Wert-Paaren im Dictionary zurück, einschließlich der Schlüssel und Werte.

```python
pizza.items()
# dict_items([('name', 'Margherita Pizza'), ('price', 8.9), ('calories_per_slice', 250)])
```

- **`clear()`-Methode**: Die `clear()`-Methode entfernt alle Schlüssel-Wert-Paare aus dem Dictionary.

```python
pizza.clear()
```

- **`pop()`-Methode**: Die `pop()`-Methode entfernt das Schlüssel-Wert-Paar mit dem als erstem Argument angegebenen Schlüssel und gibt dessen Wert zurück. Wenn der Schlüssel nicht existiert, wird der als zweites Argument angegebene Standardwert zurückgegeben. Wenn der Schlüssel nicht existiert und kein Standardwert angegeben ist, wird ein `KeyError` ausgelöst.

```python
pizza.pop('price', 10)
pizza.pop('total_price') # KeyError
```

- **`popitem()`-Methode**: In Python 3.7 und höher entfernt die `popitem()`-Methode das zuletzt eingefügte Element.

```python
pizza.popitem()
```

- **`update()`-Methode**: Die `update()`-Methode aktualisiert die Schlüssel-Wert-Paare mit den Schlüssel-Wert-Paaren eines anderen Dictionaries. Wenn Schlüssel gemeinsam sind, werden deren Werte überschrieben. Neue Schlüssel werden als neue Schlüssel-Wert-Paare zum Dictionary hinzugefügt.

```python
pizza.update({ 'price': 15, 'total_time': 25 })
```

## Über ein Dictionary iterieren

- **Über Werte iterieren**: Wenn Sie über die Werte in einem Dictionary iterieren müssen, können Sie eine `for`-Schleife mit `values()` schreiben, um alle Werte eines Dictionaries zu erhalten.

```python
products = {
    'Laptop': 990,
    'Smartphone': 600,
    'Tablet': 250,
    'Headphones': 70,
}

for price in products.values():
    print(price)
```

Ausgabe:

```md
990
600
250
70
```

- **Über Schlüssel iterieren**: Wenn Sie über die Schlüssel im oben genannten `products`-Dictionary iterieren müssen, können Sie `products.keys()` oder `products` direkt schreiben.

```python
for product in products.keys():
    print(product)
    
# Or

for product in products:
    print(product)
```

Ausgabe:

```md
Laptop
Smartphone
Tablet
Headphones
```

- **Über Schlüssel-Wert-Paare iterieren**: Wenn Sie gleichzeitig über die Schlüssel und deren zugehörige Werte iterieren müssen, können Sie über `products.items()` iterieren. Sie erhalten einzelne Tuples mit den Schlüsseln und deren zugehörigen Werten.

```python
for product in products.items():
    print(product)
```

Ausgabe:

```md
('Laptop', 990)
('Smartphone', 600)
('Tablet', 250)
('Headphones', 70)
```

Um den Schlüssel und den Wert in separaten Schleifenvariablen zu speichern, müssen Sie sie mit einem Komma trennen. Die erste Variable speichert den Schlüssel, die zweite den Wert.

```python
for product, price in products.items():
    print(product, price)
```

Ausgabe:

```md
Laptop 990
Smartphone 600
Tablet 250
Headphones 70
```

- **`enumerate()`-Funktion**: Wenn Sie über ein Dictionary iterieren müssen und dabei einen Zähler verfolgen wollen, können Sie die `enumerate()`-Funktion aufrufen. Die Funktion gibt ein `enumerate`-Objekt zurück, das jedem Element eine ganze Zahl als Zähler zuweist. Sie können den Zähler bei jeder Zahl starten lassen, standardmäßig beginnt er bei 0.

Das Zuweisen von Index und Element zu separaten Schleifenvariablen ist die übliche Verwendung von `enumerate()`. Zum Beispiel können Sie mit `products.items()` das gesamte Schlüssel-Wert-Paar zusätzlich zum Index erhalten:

```python
for index, product in enumerate(products.items()):
    print(index, product)
```

Ausgabe:

```md
0 ('Laptop', 990)
1 ('Smartphone', 600)
2 ('Tablet', 250)
3 ('Headphones', 70)
```

Um den Anfangswert der Zählung anzupassen, können Sie der `enumerate()`-Funktion ein zweites Argument übergeben. Hier starten wir die Zählung bei 1.

```python
for index, product in enumerate(products.items(), 1):
    print(index, product)
```

Ausgabe:

```md
1 ('Laptop', 990)
2 ('Smartphone', 600)
3 ('Tablet', 250)
4 ('Headphones', 70)
```

## Sets

- **Sets**: Sets sind eingebaute Datenstrukturen in Python, die keine doppelten Werte zulassen. Sets sind veränderlich und ungeordnet, was bedeutet, dass ihre Elemente nicht in einer bestimmten Reihenfolge gespeichert werden, sodass Sie nicht mit Indizes oder Schlüsseln darauf zugreifen können. Außerdem können Sets nur Werte unveränderlicher Datentypen enthalten, wie Zahlen, Strings und Tuples.

- **Ein Set definieren**: Um ein Set zu definieren, schreiben Sie seine Elemente innerhalb geschweifter Klammern und trennen sie mit Kommas.

```python
my_set = {1, 2, 3, 4, 5}
```

- **Ein leeres Set definieren**: Wenn Sie ein leeres Set definieren müssen, müssen Sie die `set()`-Funktion verwenden. Nur leere geschweifte Klammern zu schreiben, erzeugt automatisch ein Dictionary.

```python
set() # Set
{}    # Dictionary
```

## Häufige Set-Methoden

- **`add()`-Methode**: Sie können ein Element mit der `add()`-Methode zu einem Set hinzufügen, indem Sie das neue Element als Argument übergeben.

```python
my_set.add(6)
```

- **`remove()`- und `discard()`-Methoden**: Um ein Element aus einem Set zu entfernen, können Sie entweder die `remove()`-Methode oder die `discard()`-Methode verwenden, wobei Sie das zu entfernende Element als Argument übergeben. Die `remove()`-Methode löst einen `KeyError` aus, wenn das Element nicht gefunden wird, während die `discard()`-Methode dies nicht tut.

```python
my_set.remove(4)
my_set.discard(4)
```

- **`clear()`-Methode**: Die `clear()`-Methode entfernt alle Elemente aus dem Set.

```python
my_set.clear()
```

## Mathematische Mengenoperationen

- **`issubset()`- und `issuperset()`-Methoden**: Die `issubset()`- und `issuperset()`-Methoden prüfen, ob ein Set eine Teilmenge oder Obermenge eines anderen Sets ist.

```python
my_set = {1, 2, 3, 4, 5}
your_set = {2, 3, 4, 5}

print(your_set.issubset(my_set)) # True
print(my_set.issuperset(your_set)) # True
```

- **`isdisjoint()`-Methode**: Die `isdisjoint()`-Methode prüft, ob zwei Sets disjunkt sind, also keine gemeinsamen Elemente haben.

```python
my_set = {1, 2, 3}
your_set = {4, 5, 6}

print(my_set.isdisjoint(your_set)) # True
```

- **Vereinigungsoperator (`|`)**: Der Vereinigungsoperator `|` gibt ein neues Set mit allen Elementen beider Sets zurück.

```python
my_set = {1, 2, 3}
your_set = {4, 5, 6}

my_set | your_set # {1, 2, 3, 4, 5, 6}
```

- **Schnittmengenoperator (`&`)**: Der Schnittmengenoperator `&` gibt ein neues Set mit nur den Elementen zurück, die beide Sets gemeinsam haben.

```python
my_set = {1, 2, 3, 4, 5}
your_set = {2, 3, 4, 6}

my_set & your_set # {2, 3, 4}
```

- **Differenzoperator (`-`)**: Der Differenzoperator `-` gibt ein neues Set mit den Elementen des ersten Sets zurück, die nicht im anderen Set sind.

```python
my_set = {1, 2, 3, 4, 5}
your_set = {2, 3, 4, 6}

my_set - your_set # {1, 5}
```

- **Symmetrischer Differenzoperator (`^`)**: Der symmetrische Differenzoperator `^` gibt ein neues Set mit den Elementen zurück, die entweder im ersten oder im zweiten Set, aber nicht in beiden sind.

```python
my_set = {1, 2, 3, 4, 5}
your_set = {2, 3, 4, 6}

my_set ^ your_set # {1, 5, 6}
```

- **`in`-Operator**: Sie können mit dem `in`-Operator prüfen, ob ein Element in einem Set enthalten ist oder nicht.

```python
print(5 in my_set)
```

## Python-Standardbibliothek

- **Python-Standardbibliothek**: Eine Bibliothek bietet Ihnen vorgefertigten und wiederverwendbaren Code, wie Funktionen, Klassen und Datenstrukturen, die Sie in Ihren Projekten verwenden können. Python verfügt über eine umfangreiche Standardbibliothek mit eingebauten Modulen, die standardisierte Lösungen für viele Probleme und Aufgaben implementieren. Einige Beispiele für beliebte eingebaute Module sind `math`, `random`, `re` (Abkürzung für „reguläre Ausdrücke“) und `datetime`.

## Import-Anweisung

- **Import-Anweisung**: Um auf die in eingebauten Modulen definierten Elemente zuzugreifen, verwenden Sie eine Import-Anweisung. Import-Anweisungen werden in der Regel am Anfang der Datei geschrieben. Import-Anweisungen funktionieren gleich für Funktionen, Klassen, Konstanten, Variablen und alle anderen im Modul definierten Elemente.

- **Einfache Import-Anweisung**: Sie können das Schlüsselwort `import` gefolgt vom Namen des Moduls verwenden:

```python
import module_name
```

Wenn Sie dann eine Funktion aus diesem Modul aufrufen müssen, verwenden Sie die Punktnotation mit dem Namen des Moduls, gefolgt vom Namen der Funktion.

```python
module_name.function_name()
```

Zum Beispiel würden Sie in Ihrem Code Folgendes schreiben, um das Modul `math` zu importieren und die Quadratwurzel von 36 zu erhalten:

```python
import math

math.sqrt(36)
```

- **Importieren eines Moduls unter einem anderen Namen**: Wenn Sie ein Modul unter einem anderen Namen importieren müssen (auch als „Alias“ bezeichnet), können Sie `as` gefolgt vom Alias am Ende der Import-Anweisung verwenden. Dies wird oft für lange Modulnamen oder zur Vermeidung von Namenskonflikten genutzt.

```python
import module_name as module_alias
```

Zum Beispiel können Sie das Modul `math` in Ihrem Code als `m` bezeichnen, indem Sie einen Alias so zuweisen:

```python
import math as m
```

Dann können Sie auf die Elemente des Moduls mit dem Alias zugreifen:

```python
m.sqrt(36)
```

- **Importieren bestimmter Elemente**: Wenn Sie nicht alles aus einem Modul benötigen, können Sie bestimmte Elemente mit `from` importieren. In diesem Fall beginnt die Import-Anweisung mit `from`, gefolgt vom Modulnamen, dann dem Schlüsselwort `import` und schließlich den Namen der zu importierenden Elemente.

```python
from module_name import name1, name2
```

Dann können Sie diese Namen ohne das Modulpräfix in Ihrem Python-Skript verwenden. Zum Beispiel:

```python
from math import radians, sin, cos

angle_degrees = 40
angle_radians = radians(angle_degrees)

sine_value = sin(angle_radians)
cos_value = cos(angle_radians)

print(sine_value) # 0.6427876096865393
print(cos_value)  # 0.766044443118978
```
 
Das ist hilfreich, kann aber zu Namenskonflikten führen, wenn Sie bereits Funktionen oder Variablen mit demselben Namen haben. Behalten Sie das im Hinterkopf, wenn Sie entscheiden, welche Art von Import-Anweisung Sie verwenden möchten.

Wenn Sie diesen Namen Aliase zuweisen müssen, können Sie dies ebenfalls tun, indem Sie das Schlüsselwort `as` gefolgt vom Alias verwenden.

```python
from module_name import name1 as alias1, name2 as alias2
```

- **Import-Anweisung mit Sternchen (`*`)**: Das Sternchen sagt Python, dass Sie alles in diesem Modul importieren möchten, aber so, dass Sie den Namen des Moduls nicht als Präfix verwenden müssen.

```python
from module_name import *
```

Zum Beispiel, wenn Sie dies verwenden, um das Modul `math` zu importieren, können Sie jede Funktion, die in diesem Modul definiert ist, aufrufen, ohne den Namen des Moduls als Präfix anzugeben.

```python
from math import *
print(sqrt(36))  # 6.0
```

Dies wird jedoch allgemein nicht empfohlen, da es zu Namensraumkollisionen führen und es schwieriger machen kann, zu wissen, woher Namen stammen.

## `if __name__ == '__main__'`

- **`__name__`-Variable**: `__name__` ist eine spezielle eingebaute Variable in Python. Wenn eine Python-Datei direkt ausgeführt wird, setzt Python den Wert dieser Variablen auf den String `"__main__"`. Wenn die Python-Datei jedoch als Modul in ein anderes Python-Skript importiert wird, wird der Wert der `__name__`-Variablen auf den Namen dieses Moduls gesetzt.

Deshalb finden Sie diese Bedingung oft in Python-Skripten. Sie enthält den Code, den Sie **nur** ausführen möchten, wenn das Python-Skript als Hauptprogramm läuft.

```python
if __name__ == '__main__': 
    # Code
```

## Häufige Fehler in Python

- **SyntaxError**: Der Fehler, den Python auslöst, wenn Ihr Code nicht den Syntaxregeln entspricht. Zum Beispiel führt der Code `print("Hello there"` zu einem Syntaxfehler mit der Meldung `SyntaxError: '(' was never closed`, weil eine schließende Klammer fehlt.
- **NameError**: Python löst einen `NameError` aus, wenn Sie versuchen, auf eine Variable oder Funktion zuzugreifen, die Sie nicht definiert haben. Wenn Sie zum Beispiel die Zeile `print(username)` in Ihrem Code haben, ohne vorher eine `username`-Variable definiert zu haben, erhalten Sie einen NameError mit der Meldung `NameError: name 'username' is not defined`.
- **TypeError**: Dies ist der Fehler, den Python auslöst, wenn Sie eine Operation mit zwei oder mehr inkompatiblen Datentypen ausführen. Wenn Sie zum Beispiel versuchen, einen String zu einer Zahl zu addieren, erhalten Sie den Fehler `TypeError: can only concatenate str (not "int") to str`.
- **IndexError**: Sie erhalten einen `IndexError`, wenn Sie auf einen Index zugreifen, der in einer Liste oder anderen Sequenzen wie Tuple und String nicht existiert. Zum Beispiel ist in einem `Hello world`-String der Index des letzten Zeichens `10`. Wenn Sie versuchen, ein Zeichen auf diese Weise zuzugreifen, `greet = "hello world"; print(greet[11])`, erhalten Sie einen Fehler mit der Meldung `IndexError: string index out of range`.
- **AttributeError**: Python löst diesen Fehler aus, wenn Sie versuchen, eine Methode oder Eigenschaft zu verwenden, die in einem Objekt dieses Typs nicht existiert. Zum Beispiel führt das Aufrufen von `.append()` auf einem String wie `"hello".append("!")` zu einem Fehler mit der Meldung `AttributeError: 'str' object has no attribute 'append'`.

- **Verwendung der `print`-Funktion**: Das Einfügen von `print()`-Anweisungen an verschiedenen Stellen im Code während des Debuggens hilft Ihnen, die Werte von Variablen zu sehen und wie Ihr Code abläuft.
- **Verwendung des eingebauten Debuggers von Python (`pdb`)**: Python stellt ein `pdb`-Modul zum Debuggen bereit. Es ist Teil der Standardbibliothek von Python und daher immer verfügbar. Mit `pdb` können Sie mit der Funktion `set_trace()` eine Spur setzen, sodass Sie interaktiv durch den Code gehen und Variablen inspizieren können.
- **Nutzung von Debugging-Tools in IDEs**: Viele integrierte Entwicklungsumgebungen (IDEs) und Code-Editoren wie Pycharm und VS Code bieten Debugging-Tools mit Haltepunkten, schrittweiser Ausführung, Variableninspektion und anderen Debugging-Funktionen.

## Ausnahmebehandlung

- **`try...except`**: Wird verwendet, um einen Codeblock auszuführen, der eine Ausnahme auslösen könnte. Der `try`-Block ist der Bereich, in dem Sie einen Fehler erwarten, während der `except`-Block eine bestimmte Ausnahme abfängt und ausgeführt wird, wenn dieser Fehler auftritt. Hier ist ein Beispiel:

  ```py
  try:
    print(22 / 0)
  except ZeroDivisionError:
    print('You can\'t divide by zero!')
    # You can't divide by zero!
  ```

Sie können auch mehrere `except`-Blöcke verketten, um verschiedene Arten von Ausnahmen zu behandeln:

  ```py
  try:
    number = int(input('Enter a number: '))
    print(22 / number)
  except ZeroDivisionError:
    print('You cannot divide by zero!')
    # You cannot divide by zero! prints when you enter 0
  except ValueError:
    print('Please enter a valid number!')
    # Please enter a valid number! prints when you don't enter an integer
  ```

- **`else`- und `finally`-Blöcke**: Diese Blöcke erweitern `try...except`. Wenn keine Ausnahme auftritt, wird der `else`-Block ausgeführt. Der `finally`-Block wird immer ausgeführt, unabhängig von Fehlern.

  ```py
  try:
    result = 100 / 4
  except ZeroDivisionError:
    print('You cannot divide by zero!') # This will not run
  else:
    print(f'Result is {result}') # Result is 25.0
  finally:
    print('Execution complete!') # Execution complete!
  ```

- **Ausnahmeobjekt**: Damit können Sie auf die Ausnahme selbst zugreifen, um besser zu debuggen und die direkte Fehlermeldung auszugeben. Um auf das Ausnahmeobjekt zuzugreifen, verwenden Sie das Schlüsselwort `as`. Hier ist ein Beispiel:

  ```py
  try:
      value = int('This will raise an error')
  except ValueError as e:
      print(f'Caught an error: {e}')
      # Caught an error: invalid literal for int() with base 10: 'This will raise an error'
  ```

- **Die `raise`-Anweisung**: Damit können Sie manuell eine Ausnahme auslösen. Sie können sie verwenden, um eine Ausnahme zu werfen, wenn eine bestimmte Bedingung erfüllt ist. Hier ist ein Beispiel:

  ```py
  def divide(a, b):
      if b == 0:
          raise ZeroDivisionError('You cannot divide by zero')
      return a / b
  ```
  
## Ausnahme-Signalisierung
  
Die `raise`-Anweisung ist auch nützlich, wenn Sie eigene benutzerdefinierte Ausnahmen erstellen, da Sie damit eine Ausnahme mit einer benutzerdefinierten Nachricht werfen können. Hier ist ein Beispiel dafür:

  ```py
  class InvalidCredentialsError(Exception):
      def __init__(self, message="Invalid username or password"):
          self.message = message
          super().__init__(self.message)

  def login(username, password):
      stored_username = "admin"
      stored_password = "password123"
      
      if username != stored_username or password != stored_password:
          raise InvalidCredentialsError()
      
      return f"Welcome, {username}!"
  ```  

Hier sehen Sie, wie Sie die `login`-Funktion mit der `InvalidCredentialsError`-Ausnahme verwenden können:

  ```py
  # failed login attempt
  try:
      message = login("user", "wrongpassword")
  except InvalidCredentialsError as e:
      print(f"Login failed: {e}")
  else:
      print(message)

  # successful login attempt
  try:
      message = login("admin", "password123")
  except InvalidCredentialsError as e:
      # This block is not executed because the login was successful
      print(f"Login failed: {e}")
  else:
      # The else block runs if the 'try' block completes without an exception
      print(message)
  ```

Die `raise`-Anweisung kann auch mit dem Schlüsselwort `from` verwendet werden, um Ausnahmen zu verketten und die Beziehung zwischen verschiedenen Fehlern anzuzeigen:

  ```py
  def parse_config(filename):
    try:
        with open(filename, 'r') as file:
            data = file.read()
            return int(data)
    except FileNotFoundError:
        raise ValueError('Configuration file is missing') from None
    except ValueError as e:
        raise ValueError('Invalid configuration format') from e

  config = parse_config('config.txt')
  ```

## Python-Klassen und Objekte

- **Klassendefinition**: Eine Klasse ist eine Vorlage zur Erstellung von Objekten. Sie definiert das Verhalten, das ein Objekt durch seine Attribute und Methoden haben wird. Hier ist ein einfaches Beispiel für eine Klassendefinition in Python:

```python
class Dog:
    def __init__(self, name, age):
        self.name = name
        self.age = age

    def bark(self):
        print(f'{self.name.upper()} says woof woof!')
```

- **Objekte erstellen**: Objekte sind Instanzen einer Klasse. Sie werden erstellt, indem die Klasse mit den notwendigen Argumenten aufgerufen wird.

```python
dog1 = Dog('Jack', 3)
dog2 = Dog('Thatcher', 5)

dog1.bark()  # JACK says woof woof!
dog2.bark()  # THATCHER says woof woof!
```

- **Methoden auf Objekten aufrufen**: Sie können Methoden auf Objekten aufrufen, um Aktionen auszuführen oder Informationen abzurufen.

```python
object_name1.method_name()
object_name2.method_name()
```

- **Unterschied zwischen Klasse und Objekt**: Eine Klasse ist eine wiederverwendbare Vorlage, während ein Objekt eine spezifische Instanz dieser Klasse mit tatsächlichen Daten ist.

## Attribute

- **Instanzattribute**: Werden in `__init__()` mit `self` definiert und sind für jedes Objekt einzigartig.
- **Klassenattribute**: Werden direkt innerhalb der Klasse definiert und von allen Instanzen geteilt.

```python
class Dog:

    def __init__(self, name):
```

- **Methoden**: Funktionen, die innerhalb einer Klasse definiert sind und auf die Attribute des Objekts zugreifen.

```python
class Car:
    def __init__(self, color, model):
```

## Dunder- (Magic-) Methoden

- **Definition**: Spezielle Methoden, die mit einem doppelten Unterstrich beginnen und enden (z. B. `__init__`, `__len__`, `__str__`, `__eq__`). Python verwendet sie intern für eingebaute Operationen.

```python
class Book:
    def __init__(self, title, pages):
        self.title = title
        self.pages = pages

    def __len__(self):
        return self.pages

    def __str__(self):
        return f"'{self.title}' has {self.pages} pages"

    def __eq__(self, other):
        return self.pages == other.pages

book1 = Book('Built Wealth Like a Boss', 420)
print(len(book1))        # 420
print(str(book1))        # 'Built Wealth Like a Boss' has 420 pages
```

- **Dunder-Methoden indirekt aufrufen**: Sie müssen Dunder-Methoden nicht direkt aufrufen. Python ruft sie automatisch auf, wenn bestimmte Aktionen stattfinden. Diese Operationen umfassen:

- **arithmetische Operationen wie Addition, Subtraktion, Multiplikation, Division und andere**. `__add__()` wird für Addition aufgerufen, `__sub__()` für Subtraktion, `__mul__()` für Multiplikation und `__truediv__()` für Division.
        
- **String-Operationen wie Verkettung, Wiederholung, Formatierung und Umwandlung in Text**. `__add__()` wird für Verkettung aufgerufen, `__mul__()` für Wiederholung, `__format__()` für Formatierung, `__str__()` und `__repr__()` für Textumwandlung und so weiter.
        
- **Vergleichsoperationen wie Gleichheit, Kleiner-als, Größer-als und andere**. `__eq__()` wird für Gleichheitsprüfungen aufgerufen, `__lt__()` für Kleiner-als, `__gt__()` für Größer-als und so weiter.
        
- **Iterative Operationen wie das Iterierbar-Machen eines Objekts und das Voranschreiten durch Elemente**. `__iter__()` wird aufgerufen, um einen Iterator zurückzugeben, und `__next__()`, um das nächste Element abzurufen.

## Praxisbeispiel: Einkaufswagen

- **Warenkorb-Klasse mit Dunder-Methoden**: Ermöglicht das Hinzufügen, Entfernen, Iterieren und Überprüfen von Inhalten mit eingebautem Verhalten.

```python
class Cart:
    def __init__(self):
        self.items = []

    def add(self, item):
        self.items.append(item)

    def remove(self, item):
        if item in self.items:
            self.items.remove(item)
        else:
            print(f'{item} is not in cart')

    def list_items(self):
        return self.items

    def __len__(self):
        return len(self.items)

    def __getitem__(self, index):
        return self.items[index]

    def __contains__(self, item):
        return item in self.items

    def __iter__(self):
        return iter(self.items)

cart = Cart()
cart.add('Laptop')
print(len(cart))        # 1
print('Laptop' in cart) # True
```

## Was ist objektorientierte Programmierung?

- **Objektorientierte Programmierung**: Ein Programmierstil, bei dem Entwickler alles im Code wie ein reales Objekt behandeln. Es wird oft OOP genannt. Die vier wichtigsten Grundsätze, die Ihnen helfen, Code effektiv zu organisieren und zu verwalten, sind **Kapselung**, **Vererbung**, **Polymorphismus** und **Abstraktion**.
- **Klassen**: Die Vorlage zur Erstellung von Objekten. Jedes einzelne Objekt, das aus einer Klasse erstellt wird, hat Attribute, die Daten definieren, und Methoden, die das Verhalten des Objekts bestimmen.

## Was ist Kapselung?

- **Kapselung**: Das Bündeln der Attribute und Methoden eines Objekts zu einer einzigen Einheit. Sie ermöglicht es, den internen Zustand des Objekts hinter einer einfachen Menge öffentlicher Methoden und Attribute zu verbergen, die wie Türen fungieren. Hinter diesen Türen befinden sich private Attribute und Methoden, die steuern, wie sich die Daten ändern und wer sie sehen kann.
- **Beispiel für Kapselung**: Wenn Sie den Kontostand einer Brieftasche verfolgen möchten, erlauben Sie Einzahlungen und Abhebungen, aber niemand soll den Kontostand selbst manipulieren können:

```py
class Wallet:
   def __init__(self, balance):
       self.__balance = balance # Private attribute

   def deposit(self, amount):
       if amount > 0:
           self.__balance += amount # Add to the balance safely

   def withdraw(self, amount):
       if 0 < amount <= self.__balance:
           self.__balance -= amount # Remove from the balance safely

account = Wallet(500)
print(account.__balance) # AttributeError: 'Wallet' object has no attribute '__balance'
```

- **Unterschied zwischen Präfix mit einfachem und doppeltem Unterstrich bei Attributen**: Das Präfix mit einem einfachen Unterstrich bedeutet, dass Attribute und Methoden für den internen Gebrauch bestimmt sind. Dies ist eine Konvention und erzwingt nicht den Zugriff von außen. Das Präfix mit einem doppelten Unterstrich verhindert effektiv, dass auf Attribute und Methoden von außerhalb ihrer Klasse zugegriffen wird.

## Was sind Getter und Setter?

- **Getter und Setter**: Methoden, mit denen Sie steuern, wie auf die Attribute einer Klasse zugegriffen und wie sie geändert werden. Sie rufen Werte mit Gettern ab und setzen Werte mit Settern.
- **Properties**: Sie verbinden Getter und Setter und ermöglichen den Zugriff auf Daten. Sie führen hinter den Kulissen zusätzliche Logik aus, wenn Sie Werte abrufen, setzen oder löschen.
- **Warum Properties statt Methoden**: Properties werden anstelle von Methoden verwendet, um die Lesbarkeit zu verbessern und den Code sauberer zu machen. Sie ermöglichen den Zugriff auf Werte mit Punktnotation, wie bei regulären Attributen, ohne Klammern.
- **Einen Getter erstellen**: Um einen Getter zu erstellen, verwenden Sie den `@property`-Decorator. Hier ist ein Getter, der den Radius eines Kreises abruft:

```py
class Circle:
    def __init__(self, radius):
        self.radius = radius # Calling the setter

    @property
    def radius(self): # A getter to get the radius
        return self._radius
  
    @property
    def area(self):  # A getter to calculate area
        return 3.14 * (self._radius ** 2)

my_circle = Circle(3)

print(my_circle.radius) # 3
print(my_circle.area) # 28.26
```

Die Verwendung von self.radius innerhalb von `__init__()` stellt sicher, dass der Setter während der Objekterstellung ausgeführt wird, sodass ungültige Radiuswerte sofort erkannt werden.

- **Einen Setter erstellen**: Um den Setter zu erstellen, der den Radius setzt, müssen Sie eine weitere Methode mit demselben Namen definieren und `@<property_name>.setter` darüber verwenden:

```py
class Circle:
    def __init__(self, radius):
        self.radius = radius # Calling the setter

    @property
    def radius(self):  # A getter to get the radius
        return self._radius

    @radius.setter
    def radius(self, value):  # A setter to set the radius
        if value <= 0:
            raise ValueError('Radius must be positive')
        self._radius = value

my_circle = Circle(3)
print('Initial radius:', my_circle.radius) # Initial radius: 3

my_circle.radius = 8
print('After modifying the radius:', my_circle.radius) # After modifying the radius: 8
```

- **Wie Python Getter und Setter behandelt**: Sobald Sie Getter und Setter definiert haben, ruft Python sie automatisch im Hintergrund auf, wann immer Sie die normale Attributsyntax auf diese Weise verwenden:

```py
my_circle.radius # This will call the getter
my_circle.radius = 4 # This will call the setter
```

Innerhalb des Setters sollten Sie nicht dem Eigenschaftsnamen selbst einen Wert zuweisen, da dies zu einem `RecursionError` führt.

- **Deleter**: Nach dem Setzen und Abrufen eines Werts mit Setter und Getter können Sie steuern, wie er mit einem `deleter` gelöscht wird. Ein Deleter führt benutzerdefinierte Logik aus, wenn Sie die `del`-Anweisung für eine Property verwenden. Um einen Deleter zu erstellen, verwenden Sie den `@<property_name>.deleter`-Decorator.

```py
  # Deleter
    @radius.deleter
    def radius(self):
        print("Deleting radius...")
        del self._radius
```

## Was ist Vererbung und wie fördert sie die Wiederverwendung von Code?

- **Vererbung**: Der Prozess, bei dem eine Kindklasse die Attribute und Methoden einer Elternklasse verwendet. Vererbung fördert die Wiederverwendung von Code, schafft klare Hierarchien und ermöglicht die Anpassung des Verhaltens, ohne alles neu schreiben zu müssen. Um Vererbung zu implementieren, nimmt eine Kindklasse den Namen einer Elternklasse an:

```py
class Parent:
    # Parent attributes and methods

class Child(Parent):
    # Child inherits, extends, and/or overrides where necessary
```

- **Einfache und Mehrfachvererbung**: Wenn eine Kindklasse Eigenschaften und Methoden von nur einer Elternklasse erbt, wie oben gezeigt, nennt man das **einfache Vererbung**. Wenn eine Kindklasse Eigenschaften und Methoden von mehr als einer Elternklasse erbt, spricht man von **Mehrfachvererbung**. Hier ist die Syntax dafür:

```py
class Parent:
    # Attributes and methods for Parent

class Child:
    # Attributes and methods for Child

class GrandChild(Parent, Child):
    # GrandChild inherits from both Parent and Child
    # GrandChild can combine or override behavior from each
```

- **`super()`-Funktion**: Eine Funktion, mit der Sie eine Methode einer Elternklasse **aufrufen** können, wenn eine Klasse eine andere Implementierung dieser Methode hat oder die Methode erweitert, ohne Code zu duplizieren.

## Was ist Polymorphismus und wie fördert er die Wiederverwendung von Code?

- **Polymorphismus**: Das OOP-Prinzip, das es verschiedenen Klassen erlaubt, denselben Methodennamen zu verwenden, wobei jede Klasse die Methode unterschiedlich implementiert, wenn sie aufgerufen wird. Hier ist die Syntax dafür:

```py
class A:
   def action(self): ...

class B:
   def action(self): ...

class C:
   def action(self): ...

Class().method()  # Works for A, B, or C
```

- **Vererbungsbasierter Polymorphismus**: Eine Elternklasse definiert eine Methode, und jede Kindklasse passt sie für ihre Zwecke an.

## Was ist Name Mangling und wie funktioniert es?

- **Name Mangling**: Ein Prozess, bei dem Python intern ein Attribut mit doppeltem Unterstrich umbenennt, indem es einen Unterstrich und den Klassennamen als Präfix hinzufügt, wodurch `__attribute` zu `_ClassName__attribute` wird.
- **Zweck des Name Mangling**: Der Hauptzweck des Name Mangling ist es, versehentliches Überschreiben von Attributen und Methoden bei Verwendung von Vererbung zu verhindern. Hier ist ein Code, der das verständlicher macht:

```py
class Parent:
    def __init__(self):
        self.__data = 'Parent data'

class Child(Parent):
    def __init__(self):
        super().__init__()
        self.__data = 'Child data'

c = Child()
print(c.__dict__) # {'_Parent__data': 'Parent data', '_Child__data': 'Child data'}
```

## Was ist Abstraktion und wie hilft sie, komplexe Systeme organisiert zu halten?

- **Abstraktion**: Ein Programmierkonzept, bei dem komplexe Implementierungsdetails eines Objekts oder Systems verborgen werden und nur die wesentlichen Funktionen gezeigt werden. In Python und anderen Programmiersprachen vereinfacht Abstraktion komplexe Systeme durch erhöhte Wiederverwendbarkeit.
- **Beispiel für Abstraktion**: Ein gutes Beispiel für Abstraktion im Alltag ist ein Auto, bei dem Sie nur das Lenkrad, die Pedale und den Schalthebel benutzen, ohne zu wissen, wie der Motor oder die Bremsen funktionieren.
- **Wie Python Abstraktion implementiert**: Python implementiert Abstraktion durch das `abc`-Modul. Das Modul stellt die `ABC`-Klasse (abstrakte Basisklasse) und den `@abstractmethod`-Decorator bereit. Eine abstrakte Basisklasse (ABC) definiert die gemeinsamen Methoden und Eigenschaften, die Unterklassen implementieren müssen. Sie kann nicht instanziiert werden.
- **Wie eine abstrakte Methode definiert wird**: Eine abstrakte Methode wird mit `@abstractmethod` definiert und muss in Unterklassen überschrieben werden, auch wenn sie eine Standardimplementierung hat. Die grundlegende Syntax der Abstraktion sieht so aus:

```py
from abc import ABC, abstractmethod

# Define an abstract base class
class AbstractClass(ABC):
    @abstractmethod
    def abstract_method(self):
        pass

# Concrete subclass that implements the abstract method
class ConcreteClassOne(AbstractClass):
    def abstract_method(self):
        print('Implementation in ConcreteClassOne')

# Another concrete subclass
class ConcreteClassTwo(AbstractClass):
    def abstract_method(self):
        print('Implementation in ConcreteClassTwo')
```

## Algorithmen und Big-O-Notation

- **Algorithmen**: Eine Reihe eindeutiger Anweisungen zur Lösung eines Problems oder zur Durchführung einer Aufgabe. Algorithmen müssen in einer endlichen Anzahl von Schritten abgeschlossen sein, und jeder Schritt muss präzise und eindeutig sein.

- **Big-O-Notation**: Beschreibt die Worst-Case-Leistung oder das Wachstum eines Algorithmus, wenn die Eingabegröße zunimmt. Sie konzentriert sich darauf, wie der Ressourcenverbrauch mit der Eingabegröße wächst, und ignoriert konstante Faktoren und niedrigere Ordnungstermen.

### Häufige Zeitkomplexitäten

- **O(1) – konstante Zeit**: Der Algorithmus benötigt unabhängig von der Eingabegröße immer die gleiche Zeit.

```python
def check_even_or_odd(number):
    if number % 2 == 0:
        return 'Even'
    else:
        return 'Odd'
```

- **O(log n) – logarithmische Zeit**: Die Zeit wächst langsam mit der Eingabegröße. Häufig bei Algorithmen, die die Problemgröße wiederholt um einen Bruchteil reduzieren (wie binäre Suche).

- **O(n) – lineare Zeit**: Die Laufzeit wächst proportional zur Eingabegröße.

```python
for grade in grades:
    print(grade)
```

- **O(n log n) – log-lineare Zeit**: Häufige Zeitkomplexität effizienter Sortieralgorithmen wie Merge Sort und Quick Sort.

- **O(n²) – quadratische Zeit**: Die Laufzeit wächst quadratisch. Oft bei verschachtelten Schleifen.

```python
for i in range(n):
    for j in range(n):
        print("Hello, World!")
```

### Speicherkomplexität

- **O(1) – konstanter Speicher**: Der Algorithmus verwendet unabhängig von der Eingabegröße immer denselben Speicher.
- **O(n) – linearer Speicher**: Der Speicherverbrauch wächst proportional zur Eingabegröße.
- **O(n²) – quadratischer Speicher**: Der Speicherverbrauch wächst quadratisch mit der Eingabegröße.

## Problemlösungstechniken

- **Problem verstehen**: Lesen Sie die Problemstellung mehrmals. Identifizieren Sie die Eingabe, die erwartete Ausgabe und wie die Eingabe in die Ausgabe umgewandelt wird.

- **Pseudocode**: Eine sprachunabhängige, hochrangige Beschreibung der Algorithmuslogik. Verwendet eine Mischung aus gebräuchlicher Schriftsprache und Programmierkonstrukten wie `IF`, `ELSE`, `FOR`, `WHILE`.

```md
GET original_string
SET reversed_string = ""
FOR EACH character IN original_string:
  ADD character TO THE BEGINNING OF reversed_string
DISPLAY reversed_string
```

- **Randfälle**: Spezifische, gültige Eingaben, die an den Grenzen dessen auftreten, was ein Algorithmus verarbeiten soll. Berücksichtigen und testen Sie immer Randfälle.

## Arrays

- **Statische Arrays**: Haben eine feste Größe, die bei der Initialisierung bestimmt wird. Elemente werden an benachbarten Speicherstellen gespeichert. Die Größe kann während der Programmausführung nicht geändert werden.

- **Dynamische Arrays**: Können während der Programmausführung automatisch wachsen oder schrumpfen. Die Größenänderung erfolgt durch automatisches Kopieren in größere Arrays, wenn nötig.

### Python-Listen (dynamische Arrays)

```python
numbers = [3, 4, 5, 6]

# Access elements
numbers[0]  # 3

# Update elements
numbers[2] = 16

# Add elements
numbers.append(7)
numbers.insert(3, 15)  # Insert at specific index

# Remove elements
numbers.pop(2)  # Remove at specific index
numbers.pop()   # Remove last element
```

### Zeitkomplexitäten für dynamische Arrays

- **Zugriff**: O(1)
- **Einfügen am Ende**: O(1) im Durchschnitt, O(n) bei Größenänderung
- **Einfügen in der Mitte**: O(n)
- **Löschen**: O(n) in der Mitte, O(1) am Ende

## Stacks

- **Stacks**: Last-In, First-Out (LIFO) Datenstruktur. Elemente werden nur oben hinzugefügt und entfernt.

- **Push-Operation**: Ein Element oben auf den Stack legen. Zeitkomplexität: O(1).

- **Pop-Operation**: Ein Element oben vom Stack entfernen. Zeitkomplexität: O(1).

```python
# Using Python list as stack
stack = []

# Push operations
stack.append(1)
stack.append(2)
stack.append(3)

# Pop operations
top_element = stack.pop()  # Returns 3
```

## Queues

- **Queues**: First-In, First-Out (FIFO) Datenstruktur. Elemente werden hinten hinzugefügt und vorne entfernt.

- **Enqueue-Operation**: Ein Element hinten in die Queue einfügen. Zeitkomplexität: O(1).

- **Dequeue-Operation**: Ein Element vorne aus der Queue entfernen. Zeitkomplexität: O(1).

```python
from collections import deque

# Using deque for efficient queue operations
queue = deque()

# Enqueue operations
queue.append(1)
queue.append(2)
queue.append(3)

# Dequeue operations
first_element = queue.popleft()  # Returns 1
```

## Verkettete Listen

- **Verkettete Listen**: Lineare Datenstruktur, bei der jeder Knoten Daten und eine Referenz auf den nächsten Knoten enthält. Knoten sind wie eine Kette verbunden.

### Einfach verkettete Listen

- **Struktur**: Jeder Knoten hat Daten und eine Referenz auf den nächsten Knoten.
- **Durchlaufen**: Kann nur vorwärts vom Kopf bis zum Ende gehen.
- **Kopf-Knoten**: Erster Knoten in der Liste, normalerweise der einzige direkt zugängliche Knoten.
- **End-Knoten**: Letzter Knoten in der Liste, zeigt auf `None`.

### Operationen und Zeitkomplexitäten

- **Einfügen am Anfang**: O(1)
- **Einfügen am Ende**: O(n) – muss bis zum Ende durchlaufen
- **Einfügen in der Mitte**: O(n) – muss bis zur Position durchlaufen
- **Löschen am Anfang**: O(1)
- **Löschen am Ende**: O(n) – muss vorherigen Knoten finden
- **Löschen in der Mitte**: O(n) – muss Knoten finden

### Doppelt verkettete Listen

- **Struktur**: Jeder Knoten hat Daten und zwei Referenzen: auf den nächsten und den vorherigen Knoten.
- **Durchlaufen**: Kann in beide Richtungen gehen.
- **Speicher**: Benötigt mehr Speicher als einfach verkettete Listen wegen der zusätzlichen Referenz.

## Hash-Maps und Sets

### Maps und Hash-Maps

- **Map (abstrakter Datentyp)**: Verwalten Sammlungen von Schlüssel-Wert-Paaren. Jeder Schlüssel muss eindeutig sein, Werte können wiederholt werden.

- **Hash-Map**: Konkrete Implementierung des Map-ADT mit Hashing-Technik. Verwendet eine Hashfunktion, um Hashwerte für Schlüssel zu erzeugen, die den Speicherort im zugrundeliegenden Array bestimmen.

### Python-Wörterbücher (Hash Maps)

```python
# Creating dictionaries
my_dictionary = {
    "A": 1,
    "B": 2, 
    "C": 3
}

# Alternative creation
my_dictionary = dict(A=1, B=2, C=3)

# Access and modify
value = my_dictionary["A"]  # 1
my_dictionary["A"] = 4      # Update value
del my_dictionary["A"]      # Remove key-value pair

# Check membership
"C" in my_dictionary

# Get keys, values, items
my_dictionary.keys()
my_dictionary.values()
my_dictionary.items()
```

### Zeitkomplexitäten für Hash-Maps

- **Durchschnittlicher Fall**: O(1) für Einfügen, Abrufen, Löschen
- **Schlimmster Fall**: O(n) bei vielen Hash-Kollisionen

### Sets

- **Sets**: Ungeordnete Sammlungen einzigartiger Elemente. Keine Duplikate erlaubt, keine bestimmte Reihenfolge.

- **Nur unveränderliche Elemente**: Sets können nur unveränderliche Datentypen enthalten (Zahlen, Strings, Tuple), da Hashwerte konstant bleiben müssen.

```python
# Creating sets
numbers = {1, 2, 3, 4}
empty_set = set()  # Must use set(), not {}

# Add and remove elements
numbers.add(5)
numbers.remove(4)      # Raises KeyError if not found
numbers.discard(4)     # No error if not found

# Set operations
set_a = {1, 2, 3, 4}
set_b = {2, 3, 4, 5, 6}

# Union, intersection, difference, symmetric difference
set_a.union(set_b)                    # or set_a | set_b
set_a.intersection(set_b)             # or set_a & set_b
set_a.difference(set_b)               # or set_a - set_b
set_a.symmetric_difference(set_b)     # or set_a ^ set_b

# Subset and superset checks
set_a.issubset(set_b)
```

### Zeitkomplexitäten für Sets

- **Durchschnittlicher Fall**: O(1) für Hinzufügen, Entfernen, Mitgliedschaftstest
- **Schlimmster Fall**: O(n) wegen Hash-Kollisionen

## Hash-Kollisionen

- **Hash-Kollision**: Tritt auf, wenn zwei verschiedene Schlüssel denselben Hashwert erzeugen.

- **Strategien zur Kollisionsbehandlung**:
  - **Verkettung**: Jeder Array-Index zeigt auf eine verkettete Liste, die alle Elemente mit demselben Hashwert speichert.
  - **Open Addressing**: Suche nach dem nächsten verfügbaren Index mit einer vordefinierten Sequenz.

## Wann welche Datenstruktur verwenden

- **Listen**: Wenn Sie geordneten, indizierten Zugriff benötigen und die Größe nicht im Voraus kennen.
- **Stacks**: Für LIFO-Operationen (Rückgängig-Funktion, Ausdrucksauswertung, Backtracking).
- **Queues**: Für FIFO-Operationen (Aufgabenplanung, Breitensuche).
- **Verkettete Listen**: Wenn häufiges Einfügen/Löschen am Anfang, unbekannte Größe, kein direkter Zugriff benötigt wird.
- **Hash-Maps**: Für schnellen Schlüssel-Wert-Zugriff, Zählen von Vorkommen, Caching.
- **Sets**: Für Einzigartigkeitsprüfung, mathematische Mengenoperationen, Entfernen von Duplikaten.

## Suchalgorithmen

Suchalgorithmen ermöglichen es Ihnen, in einer bestimmten Liste von Elementen nach einem Ziel zu suchen.

In der Informatik gibt es zwei Suchalgorithmen, die Sie kennen sollten. Das sind die **lineare Suche** und die **binäre Suche**. Es ist wichtig, die Unterschiede zwischen diesen beiden Algorithmen zu verstehen und zu wissen, wann welcher Algorithmus verwendet wird.

### Lineare Suche

- Die lineare Suche iteriert durch eine Liste von Elementen und überprüft jedes Element von Anfang an, bis das Ziel-Element gefunden wird.
- Wenn das Ziel-Element gefunden wird, wird der Index zurückgegeben, an dem es sich in der Liste befindet.
- Wenn das Ziel nicht gefunden wird, gibt sie `-1` zurück, was in den meisten Programmiersprachen einen **ungültigen Index** bedeutet.
- Da die lineare Suche jedes Element überprüft, bis das Ziel gefunden wird, ist sie für eine große Liste von Elementen nicht effizient.
- Die Zeitkomplexität der linearen Suche ist `O(n)`, weil die Zeit, die zum Durchsuchen der Liste benötigt wird, linear mit der Größe der Liste wächst.
- Die Speicherkomplexität der linearen Suche ist `O(1)`, weil kein zusätzlicher Speicherplatz zum Durchsuchen der Liste benötigt wird.

### Binäre Suche

- Die binäre Suche funktioniert, indem sie eine Liste von Elementen halbiert und prüft, ob sich der Zielwert in der Mitte der Liste befindet.
- Voraussetzung für die binäre Suche ist, dass die Elemente in der Liste aufsteigend sortiert sind.
- Die binäre Suche ist ein effizienterer Algorithmus für große Listen, da sie die Liste halbiert und die Hälfte ignoriert, in der das Ziel nicht gefunden wird.
- Wenn das Ziel in der Mitte der Liste gefunden wird, wird der Index des Zielwerts zurückgegeben.
- Wenn das Element nicht gefunden wird, prüft der Algorithmus, ob sich das Ziel in der linken oder rechten Hälfte der Liste befindet.
- Er teilt die verbleibenden Teile der Liste weiterhin in Hälften, bis das Ziel gefunden wird.
- Wenn das Ziel schließlich nicht gefunden wird, gibt sie `-1` zurück.
- Die Zeitkomplexität der binären Suche ist `O(log n)`, weil die Suchzeit logarithmisch mit der Größe der Liste wächst.
- Die Speicherkomplexität der binären Suche ist `O(1)`, da kein zusätzlicher Speicher benötigt wird.

### Wie sich die lineare Suche von der binären Suche unterscheidet

- Die binäre Suche ist für eine große Liste von Elementen besser geeignet als die lineare Suche.
- Die Zeitkomplexität der linearen Suche ist `O(n)`, weil die Zeit, die zum Durchsuchen der Liste benötigt wird, linear mit der Größe der Liste wächst.
- Die Zeitkomplexität der binären Suche ist `O(log n)`, weil die Zeit, die zum Durchsuchen der Liste benötigt wird, logarithmisch mit der Größe der Liste wächst.

## Sortieralgorithmen und Divide-and-Conquer

In der Informatik ist Divide-and-Conquer eine Technik, mit der ein Problem in kleinere Teilprobleme zerlegt wird, damit diese leichter zu lösen sind. Rekursion ist die Technik, die oft bei Divide-and-Conquer eingesetzt wird, und Divide-and-Conquer ist eine mächtige Strategie, die verwendet wird, um viele effiziente Sortieralgorithmen wie Merge Sort zu implementieren.

### Merge Sort

- Merge Sort ist ein Sortieralgorithmus, der dem Divide-and-Conquer-Ansatz folgt.
- Er funktioniert, indem er eine Liste rekursiv in kleinere Teil-Listen aufteilt, bis jede Teil-Liste nur noch ein Element enthält.
- Dann werden die Teil-Listen wiederholt in sortierter Reihenfolge zusammengeführt.
- Die Zeitkomplexität von Merge Sort ist `O(n log n)`, weil die Liste kontinuierlich halbiert `(log n)` und dann wieder zusammengeführt `(O(n))` wird.
- Die Speicherkomplexität von Merge Sort ist `O(n)`, weil es kein In-Place-Sortieralgorithmus ist.

## Überblick über Graphen

Ein Graph ist eine Menge von Knoten (Vertices), die durch Kanten (Verbindungen) verbunden sind. Jeder Knoten kann mit mehreren anderen Knoten verbunden sein und bildet so ein Netzwerk. Die verschiedenen Arten von Graphen umfassen:

- Gerichtet: Kanten haben eine Richtung (von einem Knoten zu einem anderen), oft mit Linien und Pfeilen dargestellt.
- Ungerichtet: Kanten haben keine Richtung, dargestellt durch einfache Linien.
- Vertex: Jeder Knoten ist mit einer Bezeichnung oder Kennung verbunden.
- Zyklisch: Enthält Zyklen (ein Pfad, der am selben Knoten beginnt und endet).
- Azyklisch (DAG): Enthält keine Zyklen.
- Kantenbeschriftet: Jede Kante hat eine Bezeichnung, die meist neben der entsprechenden Kante gezeichnet wird.
- Gewichtet: Kanten haben Gewichte (Werte), die für arithmetische Operationen verwendet werden können.
- Getrennt: Enthält zwei oder mehr Knoten, die durch keine Kanten verbunden sind.

Graphen werden in verschiedenen Anwendungen verwendet, z. B. Karten, Netzwerke, Empfehlungssysteme, Abhängigkeitsauflösung.

## Graphdurchläufe

Dabei werden alle Knoten eines Graphen besucht. Die zwei Hauptalgorithmen sind:

- **Breitensuche (BFS)**
  - Verwendet eine Queue.
  - Erkundet Ebene für Ebene.
  - Findet den kürzesten Weg in ungewichteten Graphen.

- **Tiefensuche (DFS)**
  - Verwendet einen Stack (oder Rekursion).
  - Erkundet einen Zweig vollständig, bevor zurückgegangen wird.
  - Nützlich für Zykluserkennung und Pfadsuche.

## Graphdarstellungen

Graphen können auf zwei Hauptarten dargestellt werden:

- **Adjazenzliste**
  - Jeder Knoten hat eine Liste seiner Nachbarn.
  - Speicherplatzsparend für dünn besetzte Graphen.
  - Einfach, Nachbarn zu durchlaufen.

- **Adjazenzmatrix**
  - Ein 2D-Array, bei dem Zeilen und Spalten Knoten repräsentieren.
  - Speicherintensiv für große Graphen.
  - Schnell zu prüfen, ob eine Kante zwischen zwei Knoten existiert.

## Bäume

Ein Baum ist eine spezielle Art von Graph, der azyklisch und zusammenhängend ist. Wichtige Eigenschaften sind:

- Keine Schleifen oder Zyklen (Pfad, bei dem Start- und Endknoten gleich sind).
- Muss zusammenhängend sein (jeder Knoten ist von jedem anderen erreichbar).

### Häufige Baumtypen

Die häufigsten Baumtypen sind:

- Binärbäume
  - Jeder Knoten hat höchstens zwei Kinder, ein linkes und ein rechtes Kind.

- Binäre Suchbäume (BST)
  - Ein Binärbaum, bei dem jedes linke Kind kleiner als sein Elternknoten und jedes rechte Kind größer als sein Elternknoten ist.


## Tries

Auch Präfixbäume genannt, werden verwendet, um Mengen von Strings zu speichern, wobei jeder Knoten ein Zeichen repräsentiert.

Gemeinsame Präfixe werden nur einmal gespeichert, was sie effizient für Aufgaben wie Autovervollständigung und Rechtschreibprüfung macht.

Such- und Einfügeoperationen haben eine Zeitkomplexität von O(L), wobei L die Länge des Strings ist.

## Prioritätswarteschlangen

Eine Prioritätswarteschlange ist ein abstrakter Datentyp, bei dem jedes Element eine Priorität hat.

Queues und Stacks berücksichtigen nur die Einfügereihenfolge, während Prioritätswarteschlangen die Priorität der Elemente berücksichtigen.

Standard-Queues folgen FIFO (First In First Out) und Stacks folgen LIFO (Last In First Out). In einer Prioritätswarteschlange werden Elemente mit höherer Priorität vor denen mit niedrigerer Priorität bedient, unabhängig von der Einfügereihenfolge.

## Heaps

Eine spezialisierte baumbasierte Datenstruktur mit einer sehr spezifischen Eigenschaft, der Heap-Eigenschaft.

Die Heap-Eigenschaft bestimmt die Beziehung zwischen Eltern- und Kindknoten. Es gibt zwei Arten von Heaps:

- Max-Heap
  - Der Wert jedes Elternknotens ist größer oder gleich den Werten seiner Kinder.
  - Das größte Element befindet sich an der Wurzel.

- Min-Heap
  - Der Wert jedes Elternknotens ist kleiner oder gleich den Werten seiner Kinder.
  - Das kleinste Element befindet sich an der Wurzel.

### Python-`heapq`-Modul-Beispiel

```py
import heapq

# Create empty heap
my_heap = []

# Insert elements
heapq.heappush(my_heap, 9)
heapq.heappush(my_heap, 3)
heapq.heappush(my_heap, 5)
print(my_heap) # [3, 9, 5]

# Remove smallest element
print(heapq.heappop(my_heap))  # 3
print(my_heap) # [5, 9]

# Push + Pop in one step
print(heapq.heappushpop(my_heap, 7)) # 5
print(my_heap) # [7, 9]

# Transform list into heap
nums = [5, 7, 3, 1]
heapq.heapify(nums)
```

### Verwendung von Prioritäten

```py
my_heap = []
heapq.heappush(my_heap, (3, "A"))
heapq.heappush(my_heap, (2, "B"))
heapq.heappush(my_heap, (1, "C"))

# Removes lowest number = highest priority
print(heapq.heappop(my_heap))  # (1, "C")
```

## Einführung in dynamische Programmierung

- **Definition**: Dynamische Programmierung ist eine algorithmische Technik, die komplexe Probleme löst, indem sie in einfachere Teilprobleme zerlegt werden und die Ergebnisse gespeichert werden, um redundante Berechnungen zu vermeiden.
- **Überlappende Teilprobleme**: Dieselben kleineren Probleme treten mehrfach auf, wenn das größere Problem gelöst wird. Statt diese Teilprobleme wiederholt zu berechnen, speichern wir ihre Lösungen.
- **Optimale Teilstruktur**: Die optimale Lösung des Problems enthält optimale Lösungen seiner Teilprobleme. Das bedeutet, wir können die beste Lösung aufbauen, indem wir die besten Lösungen für kleinere Teile kombinieren.

## Lösungen mit dynamischer Programmierung

- **Memoisierung (Top-Down-Ansatz)**: Memoisierung speichert die Ergebnisse teurer Funktionsaufrufe und gibt das zwischengespeicherte Ergebnis zurück, wenn dieselben Eingaben erneut auftreten.

```py
def climb_stairs_memo(n, memo={}):
    """Dynamic programming with memoization"""
    # Check if we've already calculated this value
    if n in memo:
        return memo[n]  # Return cached result - O(1) lookup!
    
    # Base cases
    if n <= 2:
        return n
    
    # Calculate once and store in memo for future use
    memo[n] = climb_stairs_memo(n-1, memo) + climb_stairs_memo(n-2, memo)
    return memo[n]
```

- **Tabellierung (Bottom-Up-Ansatz)**: Tabellierung baut die Lösung von unten nach oben auf und füllt eine Tabelle mit Lösungen für Teilprobleme.

```py
def climb_stairs_tabulation(n):
    """Dynamic programming with tabulation"""
    if n <= 2:
        return n
    
    # Create array to store results for all steps from 0 to n
    dp = [0] * (n + 1)
    dp[1] = 1  # 1 way to reach step 1
    dp[2] = 2  # 2 ways to reach step 2
    
    # Build up the solution iteratively
    for i in range(3, n + 1):
        # Ways to reach step i = ways to reach (i-1) + ways to reach (i-2)
        dp[i] = dp[i-1] + dp[i-2]
    
    return dp[n]
```

## Anwendungen aus der Praxis mit dynamischer Programmierung

- **Routenoptimierung**: GPS-Systeme verwenden dynamische Programmieralgorithmen, um die kürzesten Wege zwischen Orten zu finden.
- **Textverarbeitung**: Rechtschreibprüfungen und Autovervollständigungsfunktionen basieren oft auf dynamischer Programmierung, um Editierabstände zwischen Wörtern zu berechnen.
- **Finanzmodellierung**: Anlagestrategien und Portfoliooptimierung nutzen häufig Techniken der dynamischen Programmierung.
- **Ressourcenzuweisung**: Das Rucksackproblem und seine Varianten treten bei der Terminplanung, Budgetierung und Ressourcenverwaltung auf.

## Wann dynamische Programmierung verwenden

Sie sollten dynamische Programmierung in den folgenden Fällen in Betracht ziehen:

- Das Problem lässt sich in sich überschneidende Teilprobleme zerlegen.
- Das Problem zeigt eine optimale Teilstruktur.
- Eine naive rekursive Lösung würde wiederholte Berechnungen erfordern.
- Sie müssen die Zeitkomplexität auf Kosten der Speicherkomplexität optimieren.
