# --description--

フォームはユーザーの入力を扱い、データを収集し、アクションを起動できるため、すべてのウェブアプリケーションにとって基本的なものです。

Reactでは、フォームは状態やrefsを使って管理され、その動作やバリデーションを完全にコントロールできます。フォームを管理するこの2つの方法は「制御された入力」と「非制御入力」と呼ばれます。

制御された入力と非制御入力が何かを見てみましょう。

制御された入力はフォーム入力を扱う最も「Reactらしい」方法です。制御された入力では、入力フィールドの値を状態に保存し、`onChange` イベントを通じて更新します。これによりフォームデータを完全にコントロールでき、即時のバリデーションや条件付きレンダリングが可能になります。

このプロセスは次のように動作します：Reactは`useState` フックでフォームの状態を管理し、変更のたびに更新します。ユーザーが入力フィールドに文字を入力すると、`onChange` イベントが発火し、状態が更新され、Reactが新しい値でコンポーネントをレンダリングし直します。

```jsx
import { useState } from "react";

function App() {
  const [name, setName] = useState("");

  const handleChange = (e) => {
    setName(e.target.value);
  };

  const handleSubmit = (e) => {
    e.preventDefault();
    console.log(name);
  };

  return (
    <>
      <form onSubmit={handleSubmit}>
        <label htmlFor="name">Your name</label> <br />
        <input value={name} id="name" onChange={handleChange} type="text" />
        <button type="submit">Submit</button>
      </form> 
    </>
  );
}

export default App;
```

制御された入力の利点は以下の通りです：

- フォームデータに即座にアクセスできる。

- 即時のバリデーションを実装できる。

- 送信ボタンを条件付きで無効化できる。

- 入力値をプログラム的にコントロールできる。

一方、非制御入力は従来のHTMLフォームでよく見られます。つまり、`useState` フックを使って入力を扱う代わりに、非制御入力はDOMの助けを借りて独自の内部状態を維持します。

DOMが入力値をコントロールするため、入力フィールドの値をrefで取得する必要があります。この方法はコード量が少なく、refsはReactの再レンダリングを引き起こさないためパフォーマンスも良いです。

非制御入力の例を示します：

```jsx
import { useRef } from "react";

function App() {
  const nameRef = useRef();

  const handleSubmit = (e) => {
    e.preventDefault();
    console.log(nameRef.current.value);
  };

  return (
    <form onSubmit={handleSubmit}>
      <label htmlFor="name">Your</label>{" "}
      <input type="text" ref={nameRef} id="name" />
      <button type="submit">Submit</button>
    </form>
  );
}

export default App;
```

非制御入力の非常に目立つ利点はコード量が少ないことです。またパフォーマンスが良く、HTMLに慣れているReact初心者にはより自然に感じられます。

では、制御された入力と非制御入力のどちらを使うべきでしょうか？

動的なフォーム更新やリアルタイムバリデーションが必要な場合、または入力値を状態と同期させたい場合は制御された入力を使ってください。より良いコントロールが得られますが、再レンダリングが多くなります。

よりシンプルなフォームが必要な場合、送信時のみ値にアクセスしたい場合、または非Reactコードと連携する場合は非制御入力を使ってください。

制御された入力と非制御入力のどちらを使う場合でも、Reactでフォームを作る際に守るべきベストプラクティスは以下の通りです：

- 常にデフォルトのフォーム送信を防止する。

- 送信前に入力をバリデーションする。

- ローディングやバリデーションエラーなどの状態をユーザーに明確にフィードバックする。
