# --interactive--

У JavaScript `null` і `undefined` — це два різні типи даних, які представляють відсутність значення, але поводяться по-різному під час порівнянь. Розуміння того, як ці типи взаємодіють у різних сценаріях порівняння, є важливим для написання надійного та безпомилкового коду.

Почнемо з типу `undefined`. Змінна є `undefined`, коли її оголосили, але їй не призначили значення. Це значення за замовчуванням для неініціалізованих змінних і параметрів функції, яким не передали аргумент.

Тип `null`, навпаки, — це призначене значення, яке представляє навмисну відсутність значення. Його часто використовують, щоб показати, що змінна свідомо не має значення.

При порівнянні `null` і `undefined` за допомогою оператора рівності (`==`) JavaScript виконує перетворення типів. Це означає, що він намагається привести операнди до одного типу перед порівнянням. У цьому випадку `null` і `undefined` вважаються рівними:

:::interactive_editor

```js
console.log(null == undefined); // true
```

:::

Однак при використанні оператора строгої рівності (`===`), який перевіряє і значення, і тип без перетворення типів, `null` і `undefined` не рівні:

:::interactive_editor

```js
console.log(null === undefined); // false
```

:::

Ця відмінність важлива для врахування при написанні умовних інструкцій або перевірок рівності у вашому коді. При порівнянні `null` або `undefined` з іншими значеннями за допомогою оператора рівності (`==`) поведінка може бути несподіваною. Наприклад:

:::interactive_editor

```js
console.log(null == 0);  // false
console.log(null == ''); // false
console.log(undefined == 0); // false
console.log(undefined == ''); // false
```

:::

Ці порівняння повертають `false`, тому що `null` і `undefined` рівні лише між собою (і самим собі) при використанні оператора рівності. Поведінка `null` в інших порівняннях особливо хитка:

:::interactive_editor

```js
console.log(null > 0);  // false
console.log(null == 0); // false
console.log(null >= 0); // true
```

:::

`undefined`, натомість, завжди перетворюється на `NaN` у числових контекстах, що робить усі числові порівняння з `undefined` рівними `false`:

:::interactive_editor

```js
console.log(undefined > 0);  // false
console.log(undefined < 0);  // false
console.log(undefined == 0); // false
```

:::

Враховуючи ці нюанси, загалом рекомендується використовувати оператор строгої рівності при порівнянні значень, особливо коли маєте справу з `null` і `undefined`. Такий підхід допомагає уникнути несподіваного перетворення типів і робить поведінку вашого коду більш передбачуваною.

Підсумовуючи, хоча `null` і `undefined` обидва використовуються для представлення відсутності значення, вони поводяться по-різному під час порівнянь. Розуміння цих відмінностей є ключем до написання зрозумілого та безпомилкового JavaScript-коду.
