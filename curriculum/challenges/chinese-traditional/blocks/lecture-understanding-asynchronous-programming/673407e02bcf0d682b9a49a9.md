# --description--

JavaScript 引擎具有讀取、理解並執行你的程式碼的能力。它就像一個轉換器，將你的程式碼轉換成計算機能理解並依此運作的指令。 

最知名的 JavaScript 引擎之一是由 Google 開發的 V8，使用於 Chrome 和 Node.js。JavaScript 引擎的運作分為幾個步驟。首先，它會解析你的程式碼，逐行閱讀以確保 JavaScript 程式碼沒有錯誤。接著，它會將此程式碼轉換成位元組碼，這是你的程式碼較簡單且中介的版本，更容易讓電腦理解和執行。最後，它會執行這些位元組碼以執行你的程式指令。以下是一個 JavaScript 程式碼範例：

```js
const greeting = "Hello, World!";
console.log(greeting);
```

當你執行這段程式碼時，JavaScript 引擎會先解析它以檢查是否有任何語法錯誤。解析是指引擎讀取程式碼並將其拆解成它能理解的結構，並在過程中檢查錯誤。 

然後，它將程式碼編譯成中間格式（通常是 `bytecode` 或機器碼，取決於引擎）。編譯是將人類可讀的程式碼轉換成電腦能更快速執行的高效格式的處理程序。 

最後，引擎執行程式碼，將 `Hello, World!` 列印到主控台。

現在，讓我們來談談 JavaScript 執行期。JavaScript 執行期是你的 JavaScript 程式碼被執行的環境。它包含 JavaScript 引擎（例如 Chrome 中的 V8 或 Firefox 中的 SpiderMonkey），該引擎負責處理和執行程式碼，以及環境所提供的額外特性（例如網頁瀏覽器或 Node.js，你將在未來的課程中進一步了解）。

雖然核心的 JavaScript 語言處理像是變數、迴圈和函式（程式）、函數（數學）這類的事物，但執行期提供額外的工具，讓 JavaScript 能夠與語言本身以外的事物互動，例如 DOM（用於網頁）或 Fetch API（用於發出網路請求）。

簡而言之，執行期讓 JavaScript 不僅能完成基本的程式設計任務，例如與網頁互動或處理基於時間的動作，還能透過提供語言本身以外的這些額外特性來實現更多功能。

雖然你不需要了解引擎和執行期的每個細節來撰寫 JavaScript，但具備基本的理解可以幫助你撰寫更高效的程式碼並更有效地偵錯問題。
