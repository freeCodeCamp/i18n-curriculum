# --interactive--

JavaScriptでは、変数はプログラム全体でアクセスして変更できるデータを格納するコンテナの役割を果たします。

変数は値を入れる箱のようなものと考えてください。変数を使うことで、数値やテキストなどを管理し、プログラムの必要なときにそれらの値を参照できます。

JavaScriptで変数を宣言する方法の一つは、`let` キーワードを使うことです。`let` キーワードや他の変数宣言方法については、今後のレッスンで詳しく学びます。

ここでは、`let` を使って `age` という変数を宣言する例を示します。

```js
let age;
```

今のところ、`age` 変数には値が代入されていません。もし使おうとすると、値がないことを意味する `undefined` が返されます。

例を見てみましょう。

**注意**：`console.log()` はコンソールに情報を出力する関数で、コンソールはコードのデバッグに使うウェブブラウザの一部です。`console.log()` については今後のレッスンで詳しく学びます。また、`//` 記号はコード内にコメントを追加するために使います。コメントはコード実行時に無視される、自分や他のプログラマーへのメモです。

:::interactive_editor
 
```js
let age;
console.log(age); // undefined
```

:::

変数に値を代入するには、次のように代入演算子を使います。

```js
let age = 25;
```

これで `age` 変数を使うと、`25` の値が返されます。

:::interactive_editor

```js
let age = 25;
console.log(age); // 25
```

:::

代入演算子は等号（`=`）のように見えますが、等価性をチェックするものではありません。等価性をチェックする正しい演算子については今後のレッスンで学びます。

代入演算子は変数に値を代入するために使います。この変数に値を代入するプロセスは初期化と呼ばれます。

`let` キーワードを使って変数を宣言する利点の一つは、値を再代入できることです。プログラミングでは、再代入とはすでに値がある変数に新しい値を与えることを意味します。

`age` 変数の値を再代入する例を示します。

:::interactive_editor

```js
let age = 25;
console.log(age); // 25
age = 30;
console.log(age); // 30
```

:::

これで `age` 変数は `30` の値を持ちます。`let` キーワードは再度必要ありませんでした。なぜなら `age` 変数はすでに宣言されているため、二度宣言する必要がないからです。

再代入を使う場合は、変数名だけを参照すれば十分です。再代入はプログラムの実行中に変数に格納された値を更新・変更できるため便利です。ゲームのポイントを更新する例が良い例です。

変数名の付け方は単純に思えるかもしれませんが、コードの可読性と機能性を保つためにいくつかのルールとベストプラクティスがあります。

変数名はデータが何を表しているかを説明するべきです。例えば、`x` のような名前を使う代わりに、`age` や `points` のようなより説明的な名前を使うとコードが理解しやすくなります。

```js
// Bad variable names
let x = 10;
let y = "John";

// Good variable names
let age = 10;
```

JavaScriptの変数は、文字、アンダースコア（`_`）、またはドル記号（`$`）で始める必要があります。数字で始めることはできません。

```js
// Valid variable names
let age;
let _score;
let $total;

// Invalid variable names
let 1stPlace; // starts with a number
```

変数名は大文字と小文字を区別します。つまり、すべて小文字の `age` と、先頭が大文字の `Age` は別の変数とみなされます。

:::interactive_editor

```js
let age = 25;
let Age = 30;
console.log(age); // 25
console.log(Age); // 30
```

:::

だからこそ、キャメルケースのような一貫した命名規則を守ることが重要です。キャメルケースは最初の単語がすべて小文字で、続く単語の先頭が大文字になる書き方です。

キャメルケースの変数名の例を示します。

```js
let thisIsCamelCase;
let anotherExampleVariable;
let freeCodeCampStudents;
```

JavaScriptには `let`、`const`、`function`、`return` のように変数名として使えない予約語があります。

また、変数名に感嘆符（`!`）やアットマーク（`@`）のような特殊文字を使うのは避けてください。変数名は文字、数字、アンダースコア、ドル記号を使って読みやすくするのがベストです。

これらのガイドラインに従うことで、コードはよりきれいで管理しやすくなり、複雑さが増しても扱いやすくなります。
