# --explanation--

`Build tools` sind Softwareprogramme, die dabei helfen, sich wiederholende Aufgaben wie Kompilieren, Testen und Verpacken von Softwareprojekten zu automatisieren und zu verwalten. Zum Beispiel:

`Maven is a popular build tool for Java projects, helping automate the compilation process.` – Das bedeutet, dass Maven den Prozess des Erstellens von Java-Anwendungen vereinfacht, indem es Aufgaben wie das Kompilieren von Code und das Erstellen von Paketen automatisiert.

`Built-in` bezeichnet Funktionen, die bereits Teil der Software sind und keine zusätzliche Installation oder Konfiguration benötigen. Zum Beispiel:

`Visual Studio has built-in tools for debugging code.` – Das bedeutet, dass Debugging-Tools bereits in Visual Studio enthalten sind und Benutzer keine zusätzlichen Tools installieren müssen, um mit dem Debuggen zu beginnen.

`Version control` ist ein System, das Änderungen am Code verfolgt und verwaltet, sodass Teams am selben Projekt arbeiten können, ohne Arbeit zu verlieren oder Konflikte zu verursachen. Zum Beispiel:

`Git is a widely-used version control system for tracking code changes and collaborating with teammates.` – Das bedeutet, dass Git Entwicklern hilft, verschiedene Versionen des Codes zu verwalten und gemeinsam am selben Projekt zu arbeiten.
