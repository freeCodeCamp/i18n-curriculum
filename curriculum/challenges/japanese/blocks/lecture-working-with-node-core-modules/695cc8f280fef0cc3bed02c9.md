# --description--

OSモジュールはNode.jsに組み込まれている標準モジュールのひとつです。

これを使うと、Nodeが動作している現在のオペレーティングシステムとやり取りでき、OSの種類、CPUの詳細、利用可能なメモリ、総メモリ、ネットワークインターフェースなどの重要な情報にアクセスできます。

OSモジュールを使うには、次のようにインポートします。

```js
const os = require("os");
```

OSモジュールは、アプリケーションをよりシステムに対応させるために使えるいくつかのメソッドを提供しています。よく使われるものを見てみましょう。

`platform()` メソッドは、Nodeが現在動作しているオペレーティングシステムを取得します。

```js
const os = require("os");
console.log(os.platform()); // darwin
```

これはクロスプラットフォームのスクリプトを実装する際に役立ちます。

```js
if (os.platform() === 'win32') {
 // Windows specific code
} else {
 // Non-Windows specific code
}
```

`arch()` メソッドは、Node.jsがコンパイルされたCPUアーキテクチャを表す文字列を表示します。

```js
const os = require("os");
console.log(os.arch()); // arm64
```

これは、ユーザーが特定のアーキテクチャに合ったバイナリや依存関係をダウンロードする際に役立ちます。

他に考えられる値としては、`'x64'`、`'arm'`、`'arm64'`、`'ia32'`、`'mips'`、`'ppc'`などがあります。

`type()` は公式のOS名を取得し、プログラム的にオペレーティングシステムを識別できます。

```js
const os = require("os");
console.log(os.type()); // Darwin (the core OS for macOS, iOS, and other Apple products)
```

`release()` はシステムのOSカーネルのバージョンを表示します。OSカーネルはシステムリソースやハードウェアとソフトウェアの間の通信を管理するオペレーティングシステムの中核部分です。このメソッドはOSカーネルのバージョンとサーバー要件の互換性を追跡するのに役立ちます。

これは`'20.6.0'` のような文字列で返されます。

```js
const os = require("os");
console.log(os.release()); // 25.0.0
```

`version()` は`release()` メソッドよりも詳細な特定のオペレーティングシステムのバージョンを返します。

```js
const os = require("os");
console.log(os.version());
// Darwin Kernel Version 25.0.0: Wed Sep 17 21:41:39 PDT 2025;
// root:xnu-12377.1.9~141/RELEASE_ARM64_T8103
```

`cpus()` は各論理CPUコアの詳細を含むオブジェクトの配列を返します。CPU負荷の監視に役立ちます。


`uptime()` はシステムが起動してからの時間を表示します。サーバーがどのくらい稼働しているかを判断するのに役立ちます。

```js
const os = require("os");
console.log(os.uptime()); // 23047
```

`totalmem()` と `freemem()` はそれぞれシステムの総メモリ量（バイト単位）と空きメモリ量（バイト単位）を表示します。

```js
const os = require("os");
console.log(os.totalmem()); // 8589934592 (8 GB)
console.log(os.freemem()); // 93585408 (87 MB)
```

`userInfo()` は現在のシステムユーザーに関する情報を含むオブジェクトを返します。

```js
const os = require("os");
console.log(os.userInfo());

/*

}
*/
```
