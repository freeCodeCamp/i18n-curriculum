# --description--

`Versions` der npm-Pakete im Abschnitt dependencies Ihrer package.json-Datei folgen der sogenannten semantischen Versionierung (SemVer), einem Branchenstandard für Software-Versionierung, der das Verwalten von Abhängigkeiten erleichtern soll. Bibliotheken, Frameworks oder andere Werkzeuge, die auf npm veröffentlicht werden, sollten SemVer verwenden, um klar zu kommunizieren, welche Art von Änderungen Projekte erwarten können, wenn sie ein Update durchführen.

Das Wissen über SemVer kann nützlich sein, wenn Sie Software entwickeln, die externe Abhängigkeiten nutzt (was fast immer der Fall ist). Eines Tages wird Ihr Verständnis dieser Zahlen Sie davor bewahren, versehentlich Breaking Changes in Ihr Projekt einzuführen, ohne zu verstehen, warum Dinge, die gestern noch funktionierten, heute plötzlich nicht mehr funktionieren. So funktioniert die semantische Versionierung laut der offiziellen Webseite:

```json
"package": "MAJOR.MINOR.PATCH"
```

Die MAJOR-Version sollte erhöht werden, wenn Sie inkompatible API-Änderungen vornehmen. Die MINOR-Version sollte erhöht werden, wenn Sie Funktionen auf abwärtskompatible Weise hinzufügen. Die PATCH-Version sollte erhöht werden, wenn Sie abwärtskompatible Bugfixes vornehmen. Das bedeutet, dass PATCHes Bugfixes sind und MINORs neue Funktionen hinzufügen, aber keine der beiden etwas kaputt macht, was vorher funktionierte. Schließlich fügen MAJORs Änderungen hinzu, die mit früheren Versionen nicht kompatibel sind.

# --instructions--

Ändern Sie im Abschnitt dependencies Ihrer `package.json`-Datei die Version von `@freecodecamp/example` so, dass sie der MAJOR-Version 1, der MINOR-Version 2 und der PATCH-Version 13 entspricht.

# --hints--

`"dependencies"` sollte `"@freecodecamp/example"` enthalten.

```js
fetch(code + '/_api/package.json')
  .then(response => response.json())
  .then(
    data => {
      assert.property(
        data.dependencies,
        '@freecodecamp/example',
        '"dependencies" does not include "@freecodecamp/example"'
      );
    },
    error => {
      throw new Error(error.message || error.responseText);
    }
  );
```
