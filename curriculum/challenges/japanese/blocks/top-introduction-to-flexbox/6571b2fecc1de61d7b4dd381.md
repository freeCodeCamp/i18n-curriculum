# --description--

やや混乱しやすいですが、どんな要素でもフレックスコンテナとフレックスアイテムの両方になれます。言い換えれば、フレックスアイテムに`display: flex`を設定して、その子要素をフレックスボックスで配置することもできます。

<img src="https://cdn.freecodecamp.org/curriculum/odin-project/flex-box/flexbox-03.png" style="width: 95%; height: 95%;" alt="a flex container that has multiple flex items, within those flex items are nested flex items as well. Making the parent of those nested flex items also a flex container." />

複数のフレックスコンテナとアイテムを作成しネストすることが、複雑なレイアウトを構築する主な方法です。次の画像は、フレックスボックスだけを使ってさまざまな要素を配置、サイズ調整、配置したものです。フレックスボックスは非常に強力な機能です。

<img src="https://cdn.freecodecamp.org/curriculum/odin-project/flex-box/flexbox-02.png" alt="a complex layout of flex items and flex containers. There are multiple flex containers nested into each other, thus making them flex items as well"/>

この画像は、ネストされたフレックスコンテナを含むCSSフレックスボックスレイアウトを表しています。外側のコンテナは青色で「ALSO a flex container」と示され、その中に赤色で「with flex items」とマークされた別のコンテナがあります。赤いコンテナの中には、ピーチ色で「flex items」とラベル付けされた3つのアイテムがあります。これは、フレックスコンテナが別のフレックスコンテナ内にネストされ、自身のフレックスアイテムを含めることができることを示しており、フレックスボックスレイアウト構造の再帰的な性質を表しています。
