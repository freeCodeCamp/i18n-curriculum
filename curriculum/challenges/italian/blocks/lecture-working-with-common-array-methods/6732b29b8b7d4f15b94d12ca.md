# --interactive--

Una copia superficiale di un array è un nuovo array che contiene gli stessi elementi dell'originale. Se l'array contiene solo valori primitivi come numeri o stringhe, il nuovo array è completamente separato. Ma se l'array contiene altri array al suo interno, sia l'originale sia la copia hanno riferimenti agli stessi array interni. Questo significa che se modifichi qualcosa dentro un array interno condiviso, vedrai quella modifica in entrambi gli array.

Le copie superficiali sono utili quando devi modificare la struttura di primo livello, come aggiungere, rimuovere o riordinare elementi, senza modificare l'array originale o gli array interni.

Ci sono diversi metodi per creare copie superficiali di array, e ne esploreremo alcuni tra i più comuni: `concat()`, `slice()` e l'operatore di espansione.

Iniziamo con il metodo `concat()`. Questo metodo crea un nuovo array unendo due o più array. Quando usato con un solo array, crea effettivamente una copia superficiale. Ecco un esempio:

:::interactive_editor

```js
const originalArray = [1, 2, 3];
const copyArray = [].concat(originalArray);

console.log(copyArray); // [1, 2, 3]
console.log(copyArray === originalArray); // false
```

:::

In questo esempio, stiamo usando il metodo `concat()` per concatenare un array vuoto a `originalArray`. Questo creerà un nuovo array che è una copia superficiale di `originalArray`.

`copyArray` contiene gli stessi elementi di `originalArray`, ma è un oggetto array diverso, motivo per cui il controllo di uguaglianza stretta (`===`) restituisce `false`.

Un altro metodo per creare una copia superficiale è il metodo `slice()`. Quando chiamato senza argomenti, `slice()` restituisce una copia superficiale dell'intero array. Ecco come funziona:

:::interactive_editor

```js
const originalArray = [1, 2, 3];
const copyArray = originalArray.slice();

console.log(copyArray); // [1, 2, 3]
console.log(copyArray === originalArray); // false
```

:::

In questo caso, `originalArray.slice()` crea un nuovo array che è una copia superficiale di `originalArray`. Anche qui, `copyArray` contiene gli stessi elementi ma è un oggetto array diverso.

L'operatore di espansione (`...`), introdotto in ES6, offre un altro modo conciso per creare copie superficiali di array. Ecco un esempio:

:::interactive_editor

```js
const originalArray = [1, 2, 3];
const copyArray = [...originalArray];

console.log(copyArray); // [1, 2, 3]
console.log(copyArray === originalArray); // false
```

:::

L'operatore di espansione (`...`) espande gli elementi di `originalArray` in un nuovo array, creando effettivamente una copia superficiale. È importante notare che tutti questi metodi creano nuovi oggetti array, il che significa che puoi modificare la copia senza influenzare l'array originale. Per esempio:

:::interactive_editor

```js
const originalArray = [1, 2, 3];
const copyArray = [...originalArray];

copyArray.push(4);
console.log(originalArray); // [1, 2, 3]
console.log(copyArray);     // [1, 2, 3, 4]
```

:::

In questo esempio, aggiungere un elemento a `copyArray` non influisce su `originalArray`.

In sintesi, le copie superficiali di array possono essere facilmente create usando metodi come `concat()`, `slice()` o l'operatore di espansione. Questi metodi sono utili per creare nuovi array che possono essere manipolati indipendentemente dall'array originale.
