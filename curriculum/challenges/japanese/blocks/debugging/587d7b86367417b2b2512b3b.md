# --description--

<dfn>オフバイワンエラー</dfn>（時にOBOEとも呼ばれます）は、文字列や配列の特定のインデックスを対象にしようとしたとき（スライスやセグメントへのアクセス）、またはそれらのインデックスをループ処理するときに発生します。JavaScriptのインデックスは0から始まるため、最後のインデックスは常に要素数より1つ小さくなります。長さと同じインデックスにアクセスしようとすると、「インデックス範囲外」の参照エラーがスローされたり、`undefined`が出力されたりする場合があります。

インデックス範囲を引数に取る文字列や配列のメソッドを使う場合は、それらが包括的（指定したインデックスの要素が返される範囲に含まれる）かどうかをドキュメントで確認すると役立ちます。以下はオフバイワンエラーの例です。

```js
let alphabet = "abcdefghijklmnopqrstuvwxyz";
let len = alphabet.length;
for (let i = 0; i <= len; i++) {
  console.log(alphabet[i]);
}
for (let j = 1; j < len; j++) {
  console.log(alphabet[j]);
}
for (let k = 0; k < len; k++) {
  console.log(alphabet[k]);
}
```

ここでの最初の例はループが1回多く、2番目の例はループが1回少なく（最初のインデックス0を見逃している）、3番目の例は正しいです。

# --instructions--

以下の関数の2つのインデックスエラーを修正して、1から5までの数字がすべてコンソールに出力されるようにしてください。

# --hints--

ループの初期条件を設定して、最初のインデックスから開始するようにしてください。

```js
assert(__helpers.removeJSComments(code).match(/i\s*?=\s*?0\s*?;/g).length == 1);
```

ループの初期条件を修正して、インデックスが0から始まるようにしてください。

```js
assert(!__helpers.removeJSComments(code).match(/i\s?=\s*?1\s*?;/g));
```

ループの終了条件を設定して、最後のインデックスで停止するようにしてください。

```js
assert(__helpers.removeJSComments(code).match(/i\s*<\s*len\s*;|i\s*<=\s*len\s*-\s*1\s*;/g).length == 1);
```

ループの終了条件を修正して、長さの1つ前で停止するようにしてください。

```js
assert(!__helpers.removeJSComments(code).match(/i\s*?<=\s*?len;/g));
```
