# --description--

Ora che abbiamo visto l’incapsulamento, l’ereditarietà e il polimorfismo, parliamo del prossimo concetto chiave della programmazione orientata agli oggetti: l’astrazione.

L’astrazione è il processo di nascondere i dettagli complessi dell’implementazione e mostrare solo le funzionalità essenziali di un oggetto o sistema. Pensala come concentrarsi su cosa fa qualcosa piuttosto che su come lo fa.

L’astrazione non è limitata a Python. È un concetto di programmazione che può essere implementato in molti linguaggi che supportano la programmazione orientata agli oggetti.

Per illustrare l’astrazione, immagina di guidare un’auto. Quando sei al sedile del guidatore, interagisci con le parti essenziali come il volante, la leva del cambio e i pedali dell’acceleratore e del freno. Non hai bisogno di conoscere i dettagli intricati di come funziona il motore, come il cambio inserisce le marce o la fisica dietro il sistema frenante, perché tutti questi sono dettagli complessi dell’implementazione.

Questa è un’astrazione in azione! Ti fornisce un’interfaccia semplificata per interagire con un sistema complesso.

Nel caso dell’auto, l’interfaccia semplificata è il volante, i freni e l’acceleratore, mentre il sistema complesso è l’auto stessa.

Per quanto riguarda come Python implementa l’astrazione, lo fa tramite il modulo `abc`.

Questo modulo fornisce la classe `ABC` (che sta per “abstract base class”) e il decoratore `@abstractmethod`.

`ABC` è la classe da cui si deve ereditare, ma non puoi creare oggetti diretti da essa. Definisce un’interfaccia comune di metodi e proprietà che le sue sottoclassi devono implementare.

D’altra parte, un metodo astratto è un metodo dichiarato in una Abstract Base Class (ABC) usando il decoratore `@abstractmethod`. Può non avere implementazione o averne una predefinita di base. Tuttavia, ogni sottoclasse deve sovrascriverlo per essere considerata concreta e istanziabile, anche se è fornita un’implementazione predefinita.

Ecco la sintassi base di una classe astratta in Python:

```py
from abc import ABC, abstractmethod

# Define an abstract base class
class AbstractClass(ABC):
    @abstractmethod
    def abstract_method(self):
        pass

# Concrete subclass that implements the abstract method
class ConcreteClassOne(AbstractClass):
    def abstract_method(self):
        print('Implementation in ConcreteClassOne')

# Another concrete subclass
class ConcreteClassTwo(AbstractClass):
    def abstract_method(self):
        print('Implementation in ConcreteClassTwo')
```

Ecco un esempio base:

```py
from abc import ABC, abstractmethod

class Animal(ABC): # Inherits from abstract base class
   @abstractmethod # Abstract method decorator
   def make_sound(self):  # The method subclasses must override
       pass

# Concrete class that will override the abstract method
class Dog(Animal):
   def make_sound(self):
       print('Woof!')

# Another concrete class that will override the abstract method
class Cat(Animal):
   def make_sound(self):
       print('Meow!')

# Another concrete class that will override the abstract method
class Monkey(Animal):
   def make_sound(self):
       print('Ooh ooh aah aah!')

# Create instances of each concrete class
animals = [Dog(), Cat(), Monkey()]

# Loop through the instances to call the make_sound method
for animal in animals:
   animal.make_sound()

# Output:
# Woof!
# Meow!
# Ooh ooh aah aah!
```

In questo esempio:

- Importiamo la classe `ABC` e `abstractmethod` dal modulo `abc`.
- Creiamo una classe `Animal` che eredita da `ABC` e definiamo un metodo astratto `make_sound` che ogni sottoclasse di `Animal` deve sovrascrivere.
- Creiamo le classi concrete `Dog`, `Cat` e `Monkey`, che devono sovrascrivere il metodo astratto `make_sound`.
- Istanziamo le classi concrete e chiamiamo il loro metodo `make_sound` per mostrare come ognuna implementa il metodo astratto `make_sound` a modo suo.

Ricorda che non puoi creare un’istanza della classe Animal. Ecco cosa succede se provi a farlo:

```py
dog = Animal() 
# TypeError: Can't instantiate abstract class Animal 
# without an implementation for abstract method 'make_sound'
```

La stessa regola vale per le sottoclassi che non forniscono un’implementazione per il metodo astratto. Anche se definisci una sottoclasse di Animal, non puoi istanziarla finché non sovrascrive make_sound. Ecco un esempio che lo mostra:

```py
class Bird(Animal):
    pass

bird = Bird()
# TypeError: Can't instantiate abstract class Bird 
# without an implementation for abstract method 'make_sound'
```

Ecco un altro esempio, questa volta con un attributo di istanza che puoi passare alle istanze dei metodi concreti:

```py
from abc import ABC, abstractmethod

# The blueprint for any toy that can speak
class TalkingToy(ABC):
   def __init__(self, name):
       self.name = name
   @abstractmethod
   def speak(self):
       pass

class RobotToy(TalkingToy):
   def speak(self):
       print(f'{self.name} says beep boop! I am a robot!')

class TeddyBearToy(TalkingToy):
   def speak(self):
       print(f"{self.name} says hug me! I'm cuddly!")

class DinosaurToy(TalkingToy):
   def speak(self):
       print(f'{self.name} says ROOOOAR!')

# Create toys
rusty = RobotToy('Rusty')
fluffy = TeddyBearToy('Fluffy')
rex = DinosaurToy('Rex')

toys = [rusty, fluffy, rex]
for toy in toys:
   toy.speak()

# Output:
# Rusty says beep boop! I am a robot!
# Fluffy says hug me! I'm cuddly!
# Rex says ROOOOAR!
```

In questo esempio:

- Abbiamo una abstract base class `TalkingToy` che definisce un modello per qualsiasi giocattolo che può parlare.
- Le sottoclassi `RobotToy`, `TeddyBearToy` e `DinosaurToy` implementano il metodo `speak` a modo loro.
- Quando creiamo istanze di queste sottoclassi e chiamiamo il metodo `speak`, ogni giocattolo parla in modo unico.

In conclusione, l’astrazione in Python semplifica i sistemi complessi aumentando la riusabilità.

Hai visto come puoi riusare un singolo metodo da una classe astratta in più sottoclassi, costringendo ogni sottoclasse a fornire il proprio comportamento specifico.

Questo approccio mantiene il tuo codice organizzato, flessibile e più facile da mantenere, soprattutto man mano che la tua applicazione cresce.
