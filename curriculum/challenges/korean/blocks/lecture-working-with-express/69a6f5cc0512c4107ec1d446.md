# --description--

Express.js 또는 간단히 Express는 노드 위에 구축된 최소한이면서 유연한 웹 프레임워크입니다. 웹 서버와 API를 구축하는 과정을 단순화합니다.

노드의 기본 `http` 모듈로 보일러플레이트 코드를 잔뜩 작성하는 대신, Express는 몇 줄의 코드만으로 견고한 웹 애플리케이션을 만들 수 있게 해줍니다.

Node.js를 더 쉽고 강력하게 만들어 주는 빠르고 자유로운 계층이라고 생각할 수 있습니다.

개발자들이 Express를 좋아하는 세 가지 주요 이유는 다음과 같습니다:

1. **최소한이면서 빠름** – Express는 필수 기능만 제공하여 완전한 제어권을 주면서, 미들웨어와 플러그인을 통해 필요에 따라 기능을 추가할 수 있습니다.

2. **방대한 생태계** – 로깅, 인증, 보안, 세션 등 다양한 미들웨어와 플러그인이 풍부하게 모여 있습니다.

3. **API에 최적화** – 간단한 라우팅과 요청 처리 모델 덕분에 RESTful API 구축에 특히 적합합니다.

Express로 다음을 만들 수 있습니다:

* 웹 애플리케이션

* RESTful API

* 싱글 페이지 애플리케이션(SPA) 백엔드

* 미들웨어 중심 서비스

* HTTP를 사용하는 서버 사이드 모든 것


## Express 설치 방법

먼저, Node.js와 npm이 설치되어 있는지 확인하세요. node와 npm이 설치되어 있는지 확인하는 방법은 다음과 같습니다:

```bash
node -v
npm -v
```

버전 번호가 보이면 설치된 것입니다. 보이지 않는다면 설치해야 합니다. 이전 수업이나 공식 Node.js 웹사이트에서 설치 방법을 확인하세요.

다음으로 터미널에 아래 명령어를 입력하세요:

```bash
mkdir my-app
cd my-app
npm init -y
npm install express
```

미들웨어와 플러그인에 대해서는 앞으로 수업에서 더 배우겠지만, 간단히 말해 미들웨어는 요청을 받고 응답을 보내는 사이에 실행되는 코드입니다. 주로 오류 처리, 입력 유효성 검사, 요청이나 응답 데이터 수정 등에 사용됩니다. 미들웨어를 연결해 유연하고 재사용 가능한 요청 처리 로직을 만들 수 있습니다.

플러그인은 보통 Express 기능을 확장하는 외부 서드파티 노드 모듈입니다. npm으로 별도 설치하며, 대부분 미들웨어로 구현되어 약간의 설정만으로 애플리케이션에서 사용할 수 있습니다. 로깅, 인증, 보안 모범 사례 같은 공통 기능을 처음부터 작성하는 대신 플러그인을 활용할 수 있습니다.

Express는 Node.js 서버 사이드 개발에 인기 있는 프레임워크입니다. 서버를 빠르게 띄우고, 라우트를 처리하며, 미들웨어를 관리하고, API를 구축할 수 있는 빌딩 블록을 제공하며, 모두 깔끔하고 읽기 쉬운 코드로 가능합니다.
