# --description--

你是否曾經遇過 Express 應用程式崩潰卻沒有明確錯誤訊息的情況？偵錯和記錄可以幫助你清楚看到幕後發生了什麼，讓你能更快找到並修正問題。

在本課程中，我們將探討 Express 如何處理偵錯和記錄，以及你如何利用它們來改善開發流程。

## Express 內建的偵錯功能

Express 已經內建了一個名為 `debug` 的偵錯工具。你不需要安裝任何東西就能使用它。

要啟用偵錯，只要在執行應用程式時設定 `DEBUG` 環境變數即可。

以下是一個範例：

```js
DEBUG=express:* node index.js
```

這個指令會啟用 `express:*` 命名空間下的所有偵錯訊息。你會開始看到類似以下的輸出：

```js
express:router:route dispatching GET /
express:router:layer processing GET /
```

這些訊息能幫助你追蹤路由分派、中介層呼叫等，是理解應用程式請求流程的好方法。

想要更精確嗎？你可以縮小命名空間範圍：

```js
DEBUG=express:router node index.js
```

這樣只會顯示與 Express 路由器邏輯相關的日誌。

在 Windows 上，PowerShell 的語法稍有不同：

```js
$env:DEBUG = "express:*"; node index.js
```

而 CMD 則是這樣：

```js
set DEBUG=express:* && node index.js
```

概念相同，只是設定環境變數的方式不同。

接著我們來看看 Express 的記錄功能。

對於 HTTP 請求的記錄，許多開發者會使用像 `morgan` 這樣的中介層。

使用方式如下：

```bash
npm install morgan
```

然後將它加入你的應用程式：

```js
const express = require('express')
const morgan = require('morgan')
const app = express()

app.use(morgan('dev'))s
```

現在，每個進來的請求都會被記錄到終端機，包含方法、URL、狀態碼和回應時間等詳細資訊。

那為什麼這很重要？

偵錯讓你洞悉 Express 如何處理請求、中介層和路由，且不會讓生產環境的日誌變得混亂。

記錄則提供你跨請求的運作紀錄，對於偵錯錯誤、了解流量和及早發現問題至關重要。

兩者結合，讓你在開發過程中擁有掌控力和清晰度。
