## Kuweka na Kusanidi Git

- **Kuweka Git**: Ili kuangalia kama Git tayari imewekwa kwenye kompyuta yako unaweza kuendesha amri ifuatayo kwenye terminal:

```sh
git --version
```

Kama utaona nambari ya toleo, hiyo ina maana Git imewekwa. Ikiwa sivyo, basi utahitaji kuiweka.

Kwa mifumo ya Linux, Git mara nyingi huja imewekwa awali katika distros nyingi. Ikiwa huna Git imewekwa awali, unapaswa kuweza kuiweka kwa amri za meneja wa pakiti kama `sudo apt-get install git` au `sudo pacman -S git`.

Kwa watumiaji wa Mac, unaweza kuweka Git kupitia Homebrew kwa `brew install git`, au unaweza kupakua kifurushi cha kusakinisha kutoka kwenye tovuti ya Git.

Kwa Windows, unaweza kupakua kifurushi cha kusakinisha kutoka kwenye tovuti ya Git. Au, kama umeweka Chocolatey, unaweza kuendesha `choco install git.install` katika PowerShell. Kumbuka kwamba kwenye Windows, unaweza pia kutaka kupakua Git Bash ili uwe na mazingira ya shell yanayofanana na Unix.

Ili kuhakikisha usakinishaji umefanikiwa, endesha tena amri ya `git --version` kwenye terminal.

- **Masanidi wa Git**: `git config` hutumika kuweka vigezo vya usanidi vinavyohusika na jinsi Git inavyofanya kazi kwenye kompyuta yako. Ili kuona vigezo vyako vya sasa vya usanidi na mahali vinahifadhiwa kwenye mfumo wako, unaweza kuendesha amri ifuatayo:

```sh
git config --list --show-origin
```

Kwa sasa unapaswa kuona tu usanidi wa ngazi ya mfumo ikiwa umeweka Git kwa mara ya kwanza.

Ili kuweka jina lako la mtumiaji, unaweza kuendesha amri ifuatayo:

```sh
git config --global user.name "Jane Doe"
```

Bendera ya `--global` hutumika hapa kuweka jina la mtumiaji kwa miradi yote kwenye mfumo wako inayotumia Git. Ikiwa unahitaji ghairi jina la mtumiaji kwa mradi fulani, basi unaweza kuendesha amri hiyo katika saraka ya mradi husika bila bendera ya `--global`.

Ili kuweka anwani ya barua pepe ya mtumiaji, unaweza kuendesha amri ifuatayo:

```sh
git config --global user.email janedoe@example.com
```

Usanidi mwingine unaoweza kuweka ni mhariri unaopendelea Git atumie. Hapa kuna mfano wa jinsi ya kuweka mhariri unaopendelea kuwa Emacs:

```sh
git config --global core.editor emacs
```

Kama hutaki kuweka mhariri unaopendelea, basi Git itatumia mhariri wa chaguo-msingi wa mfumo wako.

## GitHub

- **Ufafanuzi**: GitHub ni suluhisho linalotegemea wingu linalotoa hifadhi ya miradi inayodhibitiwa kwa toleo katika kitu kinachoitwa "hifadhidata", na huruhusu vipengele vya ushirikiano kutumia na miradi hiyo.
- **GitHub CLI**: Zana hii hutumika kufanya kazi maalum za GitHub bila kuondoka kwenye mstari wa amri. Ikiwa huna imewekwa, unaweza kupata maelekezo ya jinsi ya kuifanya kutoka kwenye nyaraka za GitHub - lakini inapaswa kupatikana katika meneja wa pakiti wa mfumo wako.
- **GitHub Pages**: GitHub Pages ni chaguo la kuweka tovuti zisizobadilika, au programu ambazo hazihitaji seva ya nyuma kushughulikia mantiki. Yaani, programu zinazotumia upande wa mteja pekee, au katika kivinjari cha mtumiaji, zinaweza kuwekwa kikamilifu kwenye jukwaa hili.
- **GitHub Actions**: GitHub Actions ni kipengele kinachokuwezesha kuendesha otomatiki mfululizo wa kazi moja kwa moja katika hifadhidata yako ya GitHub ikiwa ni pamoja na kujenga, kujaribu, na kuweka msimbo wako.

## Amri za Kawaida za Git

- **`git init`**: Hii itaanzisha hifadhidata tupu ya Git ili Git ianze kufuatilia mabadiliko ya mradi huu. Unapoanzisha hifadhidata tupu ya Git kwa mradi, saraka mpya ya `.git` iliyofichwa itaongezwa. Saraka hii ya `.git` ina taarifa muhimu kwa Git kusimamia mradi wako.
- **`git status`**: Amri hii hutumika kuonyesha hali ya sasa ya saraka yako ya kazi - utatumia amri hii mara nyingi katika mchakato wako wa kazi.
- **`git add`**: Amri hii hutumika kuweka mabadiliko yako katika eneo la kuandaa. Kila kitu kilicho katika eneo la kuandaa kitaongezwa kwa mgawo wa thamani unaofuata. Ikiwa unataka kuweka mabadiliko yote yasiyokuwa yameandaliwa, basi unaweza kutumia `git add .` Nukta (`.`) ni jina mbadala la saraka ya sasa unayopo.
- **`git commit`**: Amri hii hutumika kufanya mgawo wa thamani wa mabadiliko yako. Mgawo wa thamani ni picha ya hali ya mradi wako wakati huo. Ukifanya `git commit`, itafungua mhariri unaopendelea ulioweka katika usanidi wa Git. Ukifungua mhariri, unaweza kutoa ujumbe wa kina kuhusu mabadiliko yako. Pia unaweza kutoa ujumbe mfupi kwa kutumia amri ya `git commit -m` kama ifuatavyo:

```sh
git commit -m "short message goes here"
```

- **`git log`**: Hii itaorodhesha migawo yote ya thamani iliyopita na taarifa muhimu kama mwandishi, tarehe ya mgawo, ujumbe wa mgawo na hash ya mgawo. Hash ya mgawo ni mfuatano mrefu unaotumika kama kitambulisho cha kipekee cha mgawo.
- **`git remote add`**: Amri hii hutumika kusanidi muunganisho wa mbali kwa hifadhidata yako ya mbali.
- **`git push`**: Amri hii hutumika kusukuma mabadiliko yako kwenda kwenye hifadhidata ya mbali.
- **`git pull`**: Amri hii hutumika kuvuta mabadiliko ya hivi karibuni kutoka kwenye hifadhidata yako ya mbali kwenda kwenye hifadhidata yako ya ndani.
- **`git clone`**: Amri hii itakamilisha nakala ya hifadhidata. Hii ina maana utakuwa na nakala ya hifadhidata. Nakala hii inajumuisha historia ya hifadhidata, mafaili/folda zote na migawo kwenye kifaa chako cha ndani.
- **`git remote -v`**: Amri hii itaonyesha orodha ya hifadhidata za mbali zinazohusiana na hifadhidata yako ya Git ya ndani.
- **`git branch`**: Amri hii itaorodhesha matawi yote ya ndani.
- **`git fetch upstream`**: Amri hii inawaambia Git kupata mabadiliko ya hivi karibuni yaliyopo kwenye remote yako ya juu (ambayo ni hifadhidata ya asili).
- **`git merge upstream/main`**: Amri hii inawaambia Git kuunganisha mabadiliko ya hivi karibuni kutoka tawi la `main` katika remote ya juu kwenda kwenye tawi lako la sasa.
- **`git reset`**: Amri hii inakuwezesha kuweka upya hali ya sasa ya tawi. Kupitisha bendera ya `--hard` inawaambia Git kulazimisha mafaili ya ndani kuendana na hali ya tawi. Hii inahakikisha unaanza na hali safi ya kazi.
- **`git rebase`**: Rebase katika Git ni njia ya kusogeza au kuunganisha mfululizo wa migawo kutoka tawi moja kwenda jingine.

## Kufanya kazi na Matawi

- **Ufafanuzi**: Tawi katika Git ni eneo tofauti la kazi ambapo unaweza kufanya mabadiliko. Tawi la `main` mara nyingi litaonyesha tawi kuu au tawi la uzalishaji katika programu halisi. Timu za wasanidi programu hutoa matawi mengi kwa ajili ya vipengele vipya na marekebisho ya hitilafu kisha kuunganisha mabadiliko hayo tena kwenye tawi la `main`.
- **Kuumba Tawi Jipya**: Ili kuunda tawi jipya unaweza kuendesha amri ifuatayo:

```sh
git branch feature
```

Ili kuhamia kwenye tawi hilo, unaweza kuendesha amri ifuatayo:

```sh
git checkout feature
```

Wasanidi wengi watatumia amri fupi ya kuunda na kuhamia kwenye tawi ambayo ni ifuatayo:

```sh
git checkout -b new-branch-name
```

Amri mpya na mbadala ni amri ya `git switch`. Hapa kuna mfano wa kuunda na kuhamia kwenye tawi jipya:

```sh
git switch -c new-branch-name
```

- **Mikakati ya Matawi**: Tawi lako la `main` ni tawi lako la chaguo-msingi na kawaida huwa thabiti. Hivyo ni bora kutoa matawi mapya kutoka hapo kwa ajili ya vitu kama marekebisho ya hitilafu, vipengele vipya, au kazi nyingine mbalimbali.
- **Migongano ya Uunganishaji**: Hii hutokea wakati Git inajaribu kuunganisha mabadiliko kutoka matawi tofauti lakini haijui ni mabadiliko gani ya kuweka. Hii kawaida hutokea wakati kuna mabadiliko yanayopingana kwa sehemu ile ile ya faili.

## Hali Tano za Faili Inayofuatiliwa na Git

- **"Isiyofuatiliwa"**: Hii ina maana faili ni mpya kwenye hifadhidata, na Git haijawahi "kuiona" kabla.
- **"Imebadilishwa"**: Faili hii ilikuwepo katika mgawo uliopita, na ina mabadiliko ambayo hayajafanywa mgawo wa thamani.
- **"Imepuuzwa"**: Huenda usione mafaili yaliyopuuzwa katika Git, lakini IDE yako inaweza kuwa na kiashiria kwao. Mafaili yaliyopuuzwa hayajumuishwi katika shughuli za Git, kawaida kwa sababu yameorodheshwa katika faili la `.gitignore`.
- **"Imefutwa"**: Faili iliyofutwa ni kinyume cha faili isiyofuatiliwa - ni faili ambayo hapo awali ilikuwepo, na imeondolewa.
- **"Imebadilishwa Jina"**: Faili iliyobadilishwa jina ni faili ambayo maudhui hayajabadilika, lakini jina au mahali pa faili limebadilishwa. Katika baadhi ya matukio, faili inaweza kuchukuliwa kuwa imebadilishwa jina hata kama ina mabadiliko madogo.

## Mafaili ya `.gitignore`

- **Ufafanuzi**: Faili la `.gitignore` ni aina maalum ya faili inayohusiana na shughuli za Git. Jina linaonyesha faili hili hutumika kuambia Git kuacha kufuatilia vitu, na hiyo ndiyo matumizi ya kawaida. Lakini kinachofanyika ni kuambia Git kuacha kufuatilia faili.

## Kufanya kazi na Hifadhidata

- **Ufafanuzi**: Hifadhidata ni kama kontena la mradi - kama unafanya kazi kwenye programu, unahifadhi mafaili ya programu hiyo pamoja katika hifadhidata. Hifadhidata zinaweza kuwa za ndani kwenye kompyuta yako, au za mbali kwenye huduma kama GitHub.
- **Hifadhidata za Umma dhidi ya Binafsi**: Hifadhidata ya umma inaweza kutazamwa na kupakuliwa na mtu yeyote. Hifadhidata binafsi inaweza kufikiwa na wewe pekee, na mtu yeyote uyeowaruhusu upatikanaji wazi.
- **Kuumba Hifadhidata kwenye GitHub**: Ili kuunda hifadhidata mpya kwenye GitHub, unaweza kubofya kitufe cha `"New Repository"` na kufuata mwongozo wa GitHub wa kusanidi hifadhidata mpya.
- **Kusukuma Hifadhidata za Ndani kwenda GitHub**: Ikiwa una mradi wa ndani kwenye kompyuta yako, unaweza kusukuma hifadhidata hiyo kwenda GitHub. Hapa kuna muhtasari wa hatua kwa hatua wa mchakato:

1. Anzisha hifadhidata tupu ya git katika saraka ya mradi (`git init`).
2. Fanya mabadiliko kwenye mradi wako.
3. Endesha amri ya `git status` kuona mabadiliko yote yaliyofuatiliwa na git.
4. Andaa mabadiliko yako (`git add`).
5. Fanya mgawo wa thamani wa mabadiliko yako (`git commit`).
6. Sanidi muunganisho wa mbali (`git remote add`).
7. Sukuma mabadiliko yako kwenda GitHub (`git push`).

## Maombi ya Kuvuta Mabadiliko

- **Maombi ya Kuvuta Mabadiliko**: Ombi la kuvuta mabadiliko ni ombi la kuvuta mabadiliko kutoka kwenye tawi lako kwenda kwenye tawi lengwa. Maombi ya kuvuta mabadiliko ni mtiririko unaotumika unapotaka kuchangia mabadiliko ya msimbo kwenye mradi. Njia hii inaruhusu wasimamizi wa mradi kukagua mabadiliko yako. Wanaweza kuacha maoni, kuuliza maswali, na kupendekeza marekebisho. Kisha baada ya mchakato wa ukaguzi kukamilika, inaweza kuidhinishwa na kuunganishwa kwenye tawi kuu.

## Kuchangia Miradi Mengine

- **Mchakato**: Kuna maelfu ya miradi ambayo unaweza kuchangia. Hapa ni mchakato wa msingi wa jinsi ya kuchangia kwenye hifadhidata nyingine:

1. Soma nyaraka za michango
2. Tafuta tatizo linalopatikana la kufanya kazi
3. Fanya fork ya hifadhidata
4. Nakili hifadhidata uliyoifork
5. Unda tawi jipya
6. Fanya mabadiliko kulingana na tatizo
7. Tengeneza PR (Ombi la kuvuta mabadiliko)
8. Subiri ukaguzi wa PR hiyo

## Kufanya kazi na Funguo za SSH na GPG

- **Funguo za GPG**: Funguo za GPG, au Gnu Privacy Guard, hutumika kawaida kusaini mafaili au migawo. Mtu anaweza kutumia funguo yako ya umma ya GPG kuthibitisha kuwa saini ya faili ni kutoka kwa funguo yako na kwamba maudhui ya faili hayajabadilishwa au kuingiliwa.

Ili kuunda funguo ya GPG, utahitaji kuendesha:

```sh
gpg --full-generate-key
```

- **Funguo za SSH**: Funguo za SSH, au Secure SHell, hutumika kawaida kuthibitisha muunganisho wa mbali kwenye seva - kupitia zana ya `ssh`. Pia unaweza kutumia funguo ya SSH kusaini migawo.

Kwa funguo ya SSH, utaendesha:

```sh
ssh-keygen -t ed25519 -C "your_email@example.com"
```

`ed25519` ni algoriti ya kisasa ya saini ya funguo ya umma.

- **Kusaini Migawo kwa Funguo za GPG**: Ili kusaini migawo yako kwa funguo yako ya GPG, utahitaji kupakia funguo yako ya umma, si funguo binafsi, kwenye akaunti yako ya GitHub. Ili kuorodhesha funguo zako za umma, utahitaji kuendesha amri ifuatayo:

```sh
gpg --list-secret-keys --keyid-format=long
```

Kisha, kupata funguo ya umma, tumia:

```sh
gpg --armor --export "<key id>"
```
  
Kisha, chukua ID fupi uliyopata kutoka kuorodhesha funguo na endesha amri hii kuiweka kama funguo yako ya kusaini git:

```sh
git config --global user.signingkey <your_gpg_key_id>
```

Kisha, unaweza kupitisha bendera ya `-S` kwa amri yako ya `git commit` kusaini mgawo maalum - utahitaji kutoa nenosiri lako. Vinginevyo, kama unataka kusaini kila mgawo moja kwa moja, unaweza kuweka usanidi wa autosign kuwa `true`:

```sh
git config --global commit.gpgsign true
```

- **Kusaini Migawo kwa Funguo za SSH**: Kusaini kwa funguo ya SSH, ambayo ni kipengele kipya kwenye GitHub, utahitaji kuanza kwa kupakia funguo kwenye akaunti yako ya GitHub. Kisha utahitaji kuweka hali ya kusaini kwa git kutumia SSH:

```sh
git config --global gpg.format ssh
```

Kisha, kuweka funguo ya kusaini, utapitisha njia ya faili badala ya ID:

```sh
git config --global user.signingkey <path_to_your_ssh_keys>
```
