# --interactive--

関数型プログラミングは、コードをきれいで予測可能にし、テストやデバッグをしやすく書くことに関するものです。今日探求する2つの重要な概念は、純粋な関数と副作用を避けることです。

まずは純粋な関数から始めましょう。

純粋な関数とは、同じ入力に対して常に同じ出力を返し、自分の外側の何も変更しない関数のことです。

純粋な関数の例を示します：

```js
function add(a, b) {
  return a + b;
}
```

この関数は、同じ入力に対して常に同じ結果を返し、自分の外側の何も変更しないため純粋です。何度`add(2, 3)`を呼び出しても、常に`5`を返し、プログラムの他の部分に影響を与えません。

次に、不純な関数と比較してみましょう：

```js
let total = 0;
function addToTotal(value) {
  total += value;
  return total;
}
```

この関数は、自分の外側の変数（`total`）を変更するため不純です。`addToTotal(5)`の結果は、関数が呼び出される前の`total`の状態によって異なります。この予測不可能さがコードの理解やデバッグを難しくします。

ここで次の重要な概念、副作用について説明します。

副作用とは、関数が呼び出されたときにプログラムの状態に起こるあらゆる変化のことです。グローバル変数の変更、ファイルへの書き込み、APIコールなどが含まれます。

純粋な関数には副作用がありませんが、不純な関数には副作用があります。

副作用を避けることは関数型プログラミングの重要な原則です。副作用を最小限にすると、コードがより予測可能でテストしやすくなります。

コードを書くときの主な目標は、関数に一つのことをさせ、それを非常にうまく行わせることです。他の部分を変更しないようにしてください。

副作用のある関数の例を示します：

:::interactive_editor

```js
function greet(name) {
  console.log(`Hello, ${name}!`);
}

greet("Alice");
```

:::

この関数はコンソールにログを出力することで外部とやり取りしているため、副作用があります。

関数型プログラミングでは、これを次のように書き換えることがあります：

:::interactive_editor

```js
function greet(name) {
  return `Hello, ${name}!`;
}
console.log(greet("Alice"));
```

:::

今、`greet`は単に文字列を返す純粋な関数で、副作用（コンソールへのログ出力）は別に処理します。

関数型プログラミングは、コードを小さく予測可能な部分に分けて純粋な関数を組み合わせることを推奨します。こうして、理解しやすくテストしやすく保守しやすいプログラムを作ることができます。
