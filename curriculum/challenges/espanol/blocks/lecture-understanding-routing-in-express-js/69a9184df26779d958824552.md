# --description--

Servir archivos estáticos es un requisito común en las aplicaciones web. Los archivos estáticos incluyen recursos como imágenes, archivos CSS, JavaScript del lado del cliente, fuentes y otros recursos que no cambian dinámicamente.

Express facilita servir estos archivos con su middleware integrado `express.static()`.

En Express, servir archivos estáticos significa entregar archivos directamente desde el sistema de archivos del servidor al navegador del cliente sin procesamiento adicional del lado del servidor.

Por ejemplo, cuando un navegador solicita archivos como `/styles.css`, `/script.js` o `/images/logo.png`, Express puede localizar esos archivos en una carpeta especificada y servirlos directamente. Si no se encuentra un archivo, Express responderá automáticamente con un 404. Como estos archivos no necesitan ser generados ni modificados en el servidor, se llaman *estáticos*.

Esto facilita servir recursos como hojas de estilo e imágenes, y elimina la necesidad de definir rutas manualmente para cada archivo.

Así es como puedes configurar el servicio de archivos estáticos en Express:

```javascript
const express = require("express");
const path = require("path");
const app = express();
const port = 3000;

// Serve static files from the "public" directory
app.use(express.static(path.join(__dirname, "public")));

app.get("/", (req, res) => {
  res.send("Welcome to the static file demo!");
});

app.listen(port, () => {
  console.log(`Server running at http://localhost:${port}`);
});
```

Primero, importamos `express` para crear nuestro servidor y `path` para construir rutas de archivos de forma segura en diferentes sistemas operativos.

Luego, `app.use(express.static(path.join(__dirname, "public")))` indica a Express que sirva archivos desde la carpeta `public`. El middleware `express.static()` indica a Express que sirva archivos desde la carpeta `public`. Cualquier archivo dentro de este directorio se vuelve accesible para el navegador.

También definimos una ruta `GET` simple para `/` que envía un mensaje de bienvenida. Esto muestra que el servicio de archivos estáticos puede coexistir con rutas dinámicas.

Finalmente, `app.listen()` inicia el servidor en el puerto 3000. Una vez que el servidor está en funcionamiento, las solicitudes de recursos estáticos en la carpeta `public` se manejan automáticamente.

Para que esto funcione, necesitas una carpeta llamada `public` para almacenar tus archivos estáticos. Una estructura típica de proyecto podría verse así:

```md
my-app/
│
├── app.js
└── public/
    ├── index.html
    ├── styles.css
    └── image.png
```

Cuando un usuario hace una solicitud, por ejemplo, visitando `http://localhost:3000/index.html`, Express verifica si el archivo solicitado existe dentro del directorio `public`. Si el archivo existe, Express lo envía directamente al navegador. Si el archivo no existe, Express pasa al siguiente middleware o devuelve una respuesta 404.

Recuerda que no necesitas definir rutas para cada archivo, así que una solicitud a `http://localhost:3000/styles.css` significa que Express busca `public/styles.css` y responde con él automáticamente.

También puedes personalizar cómo se sirven los archivos estáticos. Por ejemplo, podrías servir archivos desde múltiples directorios o configurar el comportamiento de caché.

Imagina que tu aplicación tiene esta estructura de archivos:

```md
my-app/
│
├── app.js
│
├── public/
│   ├── index.html
│   └── styles.css
│
└── images/
    └── image.png
```

Así es como puedes registrar múltiples directorios para servir archivos estáticos desde ellos:

```javascript
app.use(express.static(path.join(__dirname, "public")));
app.use("/images", express.static(path.join(__dirname, "images")));
```

Aquí usamos `app.use(express.static(path.join(__dirname, "public")));` como antes, lo que significa que los archivos dentro del directorio `public` se sirven desde la ruta raíz, igual que antes.

Pero si miras la siguiente línea, verás que pasamos `"/images"` como primer argumento a `app.use()`, seguido de `express.static(path.join(__dirname, "images"))`. En este caso, `"/images"` es una ruta de montaje, que indica a Express que solo use este middleware cuando reciba una solicitud a `/images`. Cuando eso sucede, `express.static()` se activa y busca archivos dentro del directorio `images`.

Así que con esta configuración, estas solicitudes funcionarían así:

*   GET `http://localhost:3000/styles.css`: Express busca en el directorio `public` el archivo `styles.css`, lo encuentra y lo envía al cliente.
    
*   GET `http://localhost:3000/images/image.png`: Como esta solicitud comienza con `/images`, Express busca en el directorio `images` el archivo `image.png`, lo encuentra y lo envía al cliente.
    

Servir archivos estáticos con Express es sencillo y potente. Usando `express.static()`, puedes servir fácilmente recursos como imágenes, hojas de estilo y JavaScript del lado del cliente.

Mantener los archivos estáticos en una carpeta dedicada como `public`, o usar múltiples directorios y rutas de montaje, ayuda a mantener tu aplicación organizada, mantenible y fácil de escalar.
