# --description--

جاء React 19 بميزتين جديدتين ملحوظتين تُسميان مكونات الخادم وإجراءات الخادم.

من ذلك الإصدار فصاعدًا، أصبحت مكونات الخادم هي الافتراضية في أُطُر العمل مثل Next.js التي تدعمها بسهولة.

أما إجراءات الخادم فهي دوال تُنفذ على الخادم للسماح بمعالجة النماذج مباشرة على الخادم دون الحاجة إلى نقاط نهاية API.

يبدو إجراء الخادم هكذا:

```js
"use server";

async function submitForm(formData) {
  const name = formData.get("name");
  return { message: `Hello, ${name}!` };
}
```

يستخرج هذا الإجراء حقل `name` من نموذج ويُرجع سلسلة نصية تُحيي ذلك الاسم.

لتبسيط إدارة الحالة لإجراءات الخادم وإزالة الحاجة إلى جافاسكريبت على جانب العميل للنماذج البسيطة، قدم فريق React الخطاف `useActionState` في الإصدار 19.

لنلق نظرة أقرب على هذا الخطاف ونرى كيف يعمل.

تصف وثائق React الخطاف `useActionState` بأنه خطاف "يسمح لك بتحديث الحالة بناءً على نتيجة إجراء نموذج."

لكن هذا لا يعني أنه يمكنك استخدام الخطاف `useActionState` فقط مع النماذج. يمكنك أيضًا استخدامه لإدارة نقرات الأزرار والأحداث الأخرى، طالما لديك إجراء مُعد.

وتذكر أنه، بما أن `useActionState` هو خطاف، فلا يمكنك استخدامه داخل مكون خادم.

إليك الصيغة الأساسية للخطاف `useActionState`:

```js
const [state, action, isPending] = useActionState(actionFunction, initialState, permalink);
```

- `state` هي الحالة الحالية التي يُرجعها الإجراء.

- `action` هي الدالة التي تُشغل إجراء الخادم.

- `isPending` هي قيمة منطقية تشير إلى ما إذا كان الإجراء قيد التنفيذ حاليًا أم لا.

- معلمة `actionFunction` هي إجراء الخادم نفسه.

- `initialState` هي المعلمة التي تمثل نقطة البداية للحالة قبل تشغيل الإجراء.

- `permalink` هو نص اختياري يحتوي على عنوان URL الفريد للصفحة التي يُعدلها النموذج.

لاستخدام الخطاف `useActionState`، تأكد أولًا من وجود إجراء مُعد. لنستخدم الإجراء من المثال السابق مع بعض التعديلات:

```js
"use server";

export async function submitForm(_, formData) {
  const name = formData.get("name");

  const hour = new Date().getHours();
  let greeting;

  if (hour < 12) {
    greeting = "Good morning";
  } else if (hour < 18) {
    greeting = "Good afternoon";
  } else {
    greeting = "Good evening";
  }

  return { message: `${greeting}, ${name}` };
}
```

في مكونك، تحتاج بعد ذلك إلى استيراد الخطاف `useActionState` واستدعائه في المستوى الأعلى من جسم المكون (قبل جملة الإرجاع) تمامًا مثل الخطافات الأخرى. يجب أيضًا استيراد الإجراء:

```jsx
"use client";

// Import the useActionState hook
import { useActionState } from "react";

// Import the submitForm action
import { submitForm } from "./actions/submitForm";

const Greeter = () => {

 // Initialize the hook
 const [state, submit, isPending] = useActionState(submitForm, {
   message: "",
 });

  return (
    <div className="flex flex-col items-center justify-center min-h-screen bg-gray-100 p-6">
      {/* Rest of component */}
    </div>
  );
};

export default Greeter;
```

إليك كيف تبدو الشفرة كاملة مع بعض التنسيق:

```jsx
"use client";

import { useActionState } from "react";
import { submitForm } from "./actions/submitForm";

const Greeter = () => {
  const [state, submit, isPending] = useActionState(submitForm, {
    message: "",
  });

  return (
    <div className="flex flex-col items-center justify-center min-h-screen bg-gray-100 p-6">
      <form
        action={submit}
        className="bg-white p-6 rounded-2xl shadow-md w-full max-w-md"
      >
        <h2 className="text-2xl text-center font-semibold text-gray-700 mb-4">
          Greet Someone
        </h2>

        <input
          type="text"
          name="name"
          placeholder="Enter your name"
          required
          className="w-full p-3 border border-gray-300 rounded-lg focus:outline-none focus:ring-2 focus:ring-green-400"
        />

        <button
          type="submit"
          disabled={isPending}
          className="w-full mt-4 p-3 bg-green-500 text-white font-semibold rounded-lg hover:bg-green-600 disabled:bg-gray-400 transition-all"
        >
          {isPending ? "Greeting..." : "Greet"}
        </button>

        {state.message && (
          <p className="mt-4 text-green-600 text-center font-medium">
            {state.message}
          </p>
        )}
      </form>
    </div>
  );
};

export default Greeter;
```

في المتصفح، سترى زر النموذج يتغير من `Greet` إلى `Greeting...` أثناء كون الإجراء `isPending` - وستظهر التحية كـ `Good morning, {name}` أو `Good afternoon, {name}` أو `Good evening, {name}`، حسب وقت تقديم النموذج.

تذكر كيف ذكرنا أنه يمكنك أيضًا استخدام الخطاف `useActionState` خارج النموذج؟

في هذا المثال، سنجلب خمسة مستخدمين من JSONPlaceholder بنقر زر:

```js
"use server";

export async function getUsers() {
  const res = await fetch(
    "https://jsonplaceholder.typicode.com/users?_start=0&_limit=5/"
  );
  return await res.json();
}
```

إليك واجهة المستخدم المنسقة:

```jsx
"use client";

import { useActionState } from "react";
import { getUsers } from "./actions/getUsers";

export default function FetchUsers() {
  const [users, fetchAction, isPending] = useActionState(getUsers, []);

  return (
    <div className="p-6 max-w-lg mx-auto">
      <button
        onClick={fetchAction}
        disabled={isPending}
        className="px-4 py-2 cursor-pointer bg-green-500 text-white rounded-lg hover:bg-green-600 disabled:bg-gray-400 font-bold"
      >
        {isPending ? "Fetching Users..." : "Fetch Users"}
      </button>

      <ul className="mt-4 space-y-2">
        {users.map((user) => (
          <li key={user.id} className="p-3 bg-gray-100 rounded-lg">
            <p className="font-semibold">{user.name}</p>
            <p className="text-sm text-gray-600">{user.email}</p>
          </li>
        ))}
      </ul>
    </div>
  );
}
```

في المتصفح، سترى أن نص الزر لا يتغير أبدًا إلى `Fetching Users...` بعد النقر عليه.

يحدث هذا لأن React يعامل جلب البيانات والعرض كأولوية أعلى من حالة `isPending`، مما يمنع `isPending` في العملية ويرمي خطأ.

لإصلاح هذه المشكلة، تحتاج إلى تغليف الإجراء بـ `startTransition`:

```jsx
"use client";

// import startTransition from React
import { useActionState, startTransition } from "react";
import { getUsers } from "./actions/getUsers";

export default function FetchUsers() {
  const [users, fetchAction, isPending] = useActionState(getUsers, []);

  return (
    <div className="p-6 max-w-lg mx-auto">
      <button
        {/* wrap fetchAction in startTransition */}
        onClick={() => startTransition(() => fetchAction())}
        disabled={isPending}
        className="px-4 py-2 bg-green-500 font-bold cursor-pointer text-white rounded-lg hover:bg-green-600 disabled:bg-gray-400"
      >
        {isPending ? 'Fetching Users...' : 'Fetch Users'}
      </button>

      <ul className="mt-4 space-y-2">
        {users.map((user) => (
          <li key={user.id} className="p-3 bg-gray-100 rounded-lg">
            <p className="font-semibold">{user.name}</p>
            <p className="text-sm text-gray-600">{user.email}</p>
          </li>
        ))}
      </ul>
    </div>
  );
}
```

إذا كنت تتساءل ما هو `startTransition`، فهو دالة تخبر React بأن تحديث الحالة ذو أولوية منخفضة ويمكن مقاطعته. هذا يحافظ على استجابة واجهة المستخدم أثناء التعامل مع التحديثات غير المتزامنة مثل إجراءات الخادم.

هكذا تستخدم الخطاف `useActionState` داخل النموذج وخارجه.
