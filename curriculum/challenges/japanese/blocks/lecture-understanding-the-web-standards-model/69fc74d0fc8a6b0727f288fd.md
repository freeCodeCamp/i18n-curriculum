# --description--

オープンウェブは自動で動いているわけではありません。ブラウザがページをレンダリングしたり、動画を再生したり、3Dシーンを描画したりするたびに、それは長年にわたり作業グループに参加するエンジニアや企業代表者たちによって設定された指示に従っています。

これらの指示はウェブ標準と呼ばれます。そして、それらを設定する責任を持つグループはスタンダード団体と呼ばれます。インターネットの立法者のような存在だと考えてください。

現在ウェブ標準の多くの作業を行っている組織には、**W3C**、**WHATWG**、**ECMA（TC39）**、および**Khronos Group**があります。それぞれがスタックの異なる層を管理し、運営方法も少しずつ異なります。

**Web Hypertext Application Technology Working Group**（WHATWG）は、HTMLとDOMを単一の継続的に更新される「リビングスタンダード」として維持しています。バージョンごとのリリースではなく、仕様はリアルタイムで進化し、強い編集権限を持つ小さなグループによって編集されます。意思決定権はChromium、Gecko、WebKitなどの主要なブラウザエンジンチームにあります。

**World Wide Web Consortium**（W3C）はウェブの主要なスタンダード団体です。ブラウザベンダー、政府、研究機関など幅広いメンバーシップを持っています。

W3CはCSS、SVG、アクセシビリティガイドライン、多数のウェブAPIを監督しています。提案は複数のレビュー段階を経て公式標準になるまで、合意に基づくゆっくりとしたプロセスで進められます。

**Ecma International**（旧ヨーロッパコンピュータ製造業者協会、ECMA）は、もともとコンピューティングと通信技術に焦点を当てた汎用のスタンダード団体です。現在はJavaScriptの実装の基盤となるECMAScript標準を正式に公開しています。言語標準の技術的開発はEcmaの技術委員会39（**TC39**）が担当し、Ecma Internationalは組織的枠組み、公開プロセス、標準のガバナンスを提供しています。

委員会はGoogle、Apple、Mozilla、Microsoftなどの代表者とともに約2か月ごとに会合を開きます。新しい言語機能は、ステージ0の大まかなスケッチからステージ4の完全に仕様化され実装されるまでの5段階の提案プロセスを経ます。async/await、optional chaining、ESモジュールなどの機能はすべてこのプロセスを経てJavaScript言語に導入されました。

**Khronos Group**はグラフィックス、コンピュート、メディア標準に焦点を当てたメンバー資金によるコンソーシアムです。メンバーにはNVIDIAやAMDなどのGPUメーカーの代表団とブラウザベンダーが含まれます。ウェブにおける最も重要な貢献はWebGLおよびWebGL 2で、これによりブラウザでGPUアクセラレーションされた3Dレンダリングが可能になります。
