# --description--

다양한 오디오와 비디오 형식에 대해 배워보겠습니다.

아마도 몇 가지 흔히 쓰이는 형식에 대해 들어봤거나 사용해본 적이 있을 거예요. MP3, MP4, MOV, WebM 같은 용어가 익숙하다면, 이들은 모두 비디오와 오디오의 서로 다른 형식입니다. 그리고 이 외에도 훨씬 더 많은 형식이 있습니다.

파일 형식에 대해 자세히 알아보기 전에 MIME 유형에 대해 이야기해야 합니다. MIME 유형은 Multipurpose Internet Mail Extensions의 약자로, 파일 형식을 프로그래밍적으로 표준화하여 나타내는 방법입니다.

거의 모든 파일 형식에는 MIME 유형이 있습니다. 예를 들어 HTML은 `text/html` 유형을 가지고 있습니다. JSON 객체는 `application/json` 유형을 가집니다. 심지어 Windows `.exe` 설치 파일도 MIME 유형이 있는데, `application/vnd.microsoft.portable-executable`입니다.

MIME 유형은 브라우저 같은 애플리케이션이 특정 파일을 어떻게 처리할지 알려줍니다. 오디오와 비디오의 경우 MIME 유형은 웹 페이지에 임베드할 수 있는 멀티미디어 형식임을 나타냅니다.

MP3 파일은 `audio/mpeg` MIME 유형을 가집니다. 반면 MP4는 비디오 파일인지 오디오 전용인지에 따라 `audio/mp4` 또는 `video/mp4` MIME 유형을 가질 수 있습니다. 이 구분은 브라우저가 파일을 어떻게 처리할지 결정하는 데 도움을 줍니다.

WAV라는 파형 형식, 다목적 OGG, Windows 미디어 플레이어용 WMV, 오픈 소스 MKV 등 다양한 다른 파일 형식도 많이 있습니다.

이 파일 형식들의 차이를 알면 사용자가 최상의 경험을 할 수 있도록 도울 수 있지만, 때로는 사용자의 컴퓨터가 어떤 형식을 지원할지 알 수 없습니다. 다행히도 `video`와 `audio` 요소는 모두 `source` 요소를 지원합니다.

`source` 요소를 사용하면 파일 형식을 지정할 수 있고 `source`하며 여러 `source` 요소를 사용해 여러 형식을 포함할 수 있습니다. 이렇게 하면 브라우저가 사용자의 현재 환경에 가장 적합한 형식을 결정합니다.

이로써 추측할 필요가 없어지고, 흥미로운 애플리케이션을 만드는 데 집중할 수 있습니다.
