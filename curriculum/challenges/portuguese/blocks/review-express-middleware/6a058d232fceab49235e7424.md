## O que é middleware?

- **Middleware** é uma função que executa durante o ciclo de vida de uma requisição ao servidor.
- Middleware tem acesso ao **objeto request** (`req`), ao **objeto response** (`res`) e à **função `next`**.
- Usos comuns: registrar requisições, tratar erros, analisar dados recebidos e servir arquivos estáticos.
- Middleware é executado na ordem em que é adicionado ao app.

### Como o middleware funciona

1. Uma requisição é enviada ao servidor.
2. Funções middleware processam a requisição.
3. Se o middleware chama `next()`, o controle passa para o próximo middleware ou manipulador de rota.
4. Se o middleware não chama `next()`, o ciclo requisição-resposta para.

```javascript
app.use((req, res, next) => {
  console.log('Request URL:', req.url);
  console.log('Request Method:', req.method);
  next();
});
```

- `app.use()` adiciona middleware globalmente à aplicação.

## Middleware em nível de aplicação

- **Middleware em nível de aplicação** está vinculado a uma instância do app Express.
- Registrado usando `app.use()` ou métodos específicos de rota como `app.get()`, `app.post()`, etc.
- Executa para toda requisição recebida, a menos que seja restrito a uma rota específica.
- Casos de uso: registro, autenticação e tratamento de erros em várias rotas.

```javascript
app.use((req, res, next) => {
  console.log('Request received at:', new Date());
  next();
});
```

## Middleware em nível de roteador

- **Middleware em nível de roteador** está vinculado a uma instância de `express.Router()` em vez do app principal.
- Aplicado usando `router.use()` e afeta apenas rotas gerenciadas por esse roteador.
- Ajuda a modularizar o app em módulos de rota isolados.
- Monte um roteador com `app.use('/path', router)` para limitar o middleware a rotas específicas.

```javascript
const router = express.Router();

router.use((req, res, next) => {
  console.log('Request made to /menu route');
  next();
});

router.get('/drinks', (req, res) => {
  res.send('Welcome to the drinks menu!');
});

app.use('/menu', router);
```

## Middleware para tratamento de erros

- **Middleware para tratamento de erros** é um middleware especial usado para capturar erros durante o ciclo requisição-resposta.
- Ele tem **quatro parâmetros**: `err`, `req`, `res` e `next`. A assinatura com quatro parâmetros indica ao Express que é um manipulador de erros.
- Invocado quando um erro é passado para `next(err)`.
- Deve ser colocado **no final** da pilha de middleware, após todos os outros middleware e rotas.

```javascript
app.use((err, req, res, next) => {
  console.error(err.stack);
  res.status(500).send('Something went wrong!');
});
```

## Middleware embutido e de terceiros

### Middleware embutido

- **`express.json()`**: analisa requisições recebidas com payloads JSON.

```javascript
app.use(express.json());
```

- **`express.static()`**: serve arquivos estáticos como imagens, CSS e JavaScript.

```javascript
app.use(express.static('public'));
```

- **`express.urlencoded()`**: analisa requisições recebidas com payloads codificados em URL (formato enviado por formulários HTML); disponibiliza os dados analisados em `req.body`.

```javascript
app.use(express.urlencoded({ extended: true }));
```

### Middleware de terceiros

- **`cors`**: habilita o compartilhamento de recursos entre origens diferentes (Cross-Origin Resource Sharing).

```javascript
app.use(cors());
```

- **`morgan`**: registra requisições HTTP.

```javascript
app.use(morgan('tiny'));
```

- Tanto funções middleware embutidas quanto de terceiros são adicionadas na ordem em que são chamadas com `app.use()`.
