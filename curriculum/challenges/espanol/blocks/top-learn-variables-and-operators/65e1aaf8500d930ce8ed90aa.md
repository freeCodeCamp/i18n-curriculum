# --description--

El signo más `+` existe en dos formas: la forma binaria que usaste en el último desafío y la forma unaria.

El más unario o, en otras palabras, el operador más `+` aplicado a un solo valor, no hace nada a los números. Pero si el operando no es un número, el más unario lo convierte en un número.

Por ejemplo:

```js
// No effect on numbers
let x = 1;
console.log( +x ); // 1

let y = -2;
console.log( +y ); // -2

// Converts non-numbers
console.log( +true ); // 1
console.log( +"" );   //
```

La necesidad de convertir cadenas a números surge muy a menudo. Por ejemplo, si estás obteniendo valores de campos de formulario HTML, usualmente son cadenas. ¿Qué pasa si quieres sumarlos?

El signo más binario los sumaría como cadenas:

```js
let apples = "2";
let oranges = "3";

alert( apples + oranges ); // "23", the binary plus concatenates strings
```

Si quieres tratarlos como números, necesitas convertirlos y luego sumarlos:

```js
let apples = "2";
let oranges = "3";

// both values converted to numbers before the binary plus
alert( +apples + +oranges ); // 5

// the longer variant
// alert( Number(apples) + Number(oranges) ); // 5
```

Desde el punto de vista de un matemático, la abundancia de signos más puede parecer extraña. Pero desde el punto de vista de un programador, no hay nada especial: los signos más unarios se aplican primero, convierten cadenas en números, y luego el signo más binario los suma.

¿Por qué se aplican los signos más unarios a los valores antes que los binarios? Como vas a ver, eso es debido a su *mayor* precedencia.

# --questions--

## --text--
Dada la explicación de los operadores unarios y binarios `+` en JavaScript, ¿cómo puedes sumar correctamente los valores de cadena `'2'` y `'3'` como números usando el operador unario `+`?

## --answers--

`console.log('2' + '3'); // Outputs: '23'` 

---

`console.log(+2 + +3); // Outputs: 5`

---

`console.log(+'2' + +'3'); // Outputs: 5`

---

`console.log(Number('2') + Number('3')); // Outputs: '23'`


## --video-solution--

3
