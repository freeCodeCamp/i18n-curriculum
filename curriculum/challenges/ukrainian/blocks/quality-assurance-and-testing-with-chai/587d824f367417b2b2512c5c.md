# --description--

Нагадаємо, цей проєкт будується на основі стартового проєкту, скопійованого з <a href="https://github.com/freeCodeCamp/boilerplate-mochachai/" target="_blank" rel="noopener noreferrer nofollow">GitHub</a>.

У наступних завданнях ви імітуватимете взаємодію людини зі сторінкою, використовуючи безголовий браузер.

Безголові браузери — це веббраузери без графічного інтерфейсу. Вони можуть відтворювати та інтерпретувати HTML, CSS і JavaScript так само, як звичайний браузер, що робить їх особливо корисними для тестування вебсторінок.

Для наступних завдань ви будете використовувати Zombie.js — легкий безголовий браузер, який не потребує встановлення додаткових бінарних файлів. Але існує багато інших, потужніших варіантів безголових браузерів.

Mocha дозволяє запускати код перед виконанням будь-яких тестів. Це корисно, наприклад, щоб додати записи до бази даних, які будуть використані в інших тестах.

З безголовим браузером перед запуском тестів потрібно **відвідати** сторінку, яку ви тестуватимете.

Хук `suiteSetup` виконується лише один раз на початку тестового набору.

Існують також інші типи хуків, які можуть виконувати код перед кожним тестом, після кожного тесту або в кінці тестового набору. Детальніше дивіться в документації Mocha.

# --instructions--

У межах `tests/2_functional-tests.js`, одразу після оголошення `Browser`, додайте URL вашого проєкту до властивості `site` змінної:

```js
Browser.site = 'http://0.0.0.0:3000'; // Your URL here
```

Потім на кореневому рівні набору `'Functional Tests with Zombie.js'` створіть новий екземпляр об’єкта `Browser` за допомогою такого коду:

```js
const browser = new Browser();
```

І використайте хук `suiteSetup`, щоб направити `browser` на маршрут `/` за допомогою такого коду. **Примітка**: `done` передається як функція зворотного виклику до `browser.visit`, її не слід викликати самостійно.

```js
suiteSetup(function(done) {
  return browser.visit('/', done);
});
```
