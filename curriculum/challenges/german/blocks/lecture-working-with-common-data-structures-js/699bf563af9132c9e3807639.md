# --description--

Arrays sind eine grundlegende Datenstruktur in der Informatik. Alle Arrays speichern geordnete Datensammlungen, aber je nach Typ können sie hinter den Kulissen unterschiedlich funktionieren.

Ihr zugrundeliegendes Verhalten kann einen wichtigen Einfluss auf die Effizienz des Programms haben. Lernen wir also dynamische und statische Arrays sowie ihre Unterschiede kennen, damit Sie das effizienteste für Ihr Programm auswählen können.

Wir beginnen mit statischen Arrays.

**Statische Arrays** haben eine feste Größe. Sie speichern Elemente an benachbarten Speicherstellen.

Die Größe eines statischen Arrays wird bei der Initialisierung des Arrays festgelegt. Sobald dieser bestimmte Speicherblock zugewiesen ist, ist er fix und kann während der Programmausführung nicht mehr geändert werden. Dies ist ein zentrales Merkmal statischer Arrays.

Das Speichern der Elemente an benachbarten Speicherstellen macht den Datenabruf effizienter, weil das Programm die Position des ersten Elements speichern und dann Indizes verwenden kann, um einfache Berechnungen durchzuführen und die anderen Elemente im Speicher zu finden.

Dank dessen erfolgt der Zugriff auf die Werte eines statischen Arrays in konstanter Zeit `O(1)`, was sehr effizient ist.

Sie können ein statisches Array verwenden, wenn Sie die Anzahl der zu speichernden Elemente im Voraus kennen. Es ist auch hilfreich, wenn die Werte sehr häufig abgerufen werden, da der Zugriff sehr effizient ist.

Diese Datenstruktur kann jedoch weder wachsen noch schrumpfen. Wenn sich die Anzahl der zu speichernden Elemente ändern kann, sollten Sie stattdessen ein dynamisches Array verwenden.

Der Versuch, die Größe eines statischen Arrays zu vergrößern, würde bedeuten, ein neues Array zu erstellen und alle Elemente vom alten Array in das neue zu kopieren, was ineffizient ist. In diesem Fall wäre ein dynamisches Array viel besser, weil es diesen Prozess automatisch handhabt.

JavaScript hat keine traditionellen, niedrigstufigen statischen Arrays wie C oder Java, aber Sie können ein Array mit fester Größe mit dem eingebauten `Array`-Objekt simulieren:

```js
let numbers = new Array(3); // fixed size of 3
numbers[0] = 10;
numbers[1] = 20;
numbers[2] = 30;
```

Obwohl JavaScript das Ändern der Größe erlaubt, stellt dies konzeptionell ein Array mit fester Länge dar.

Arrays in JavaScript sind standardmäßig dynamisch, schauen wir uns also diese an.

**Dynamische Arrays** sind flexibler, weil sie während der Programmausführung automatisch wachsen oder schrumpfen können.

Sie funktionieren durch einen automatischen Größenanpassungsmechanismus, der die Elemente in ein neues Array kopiert, wenn das ursprüngliche Array voll ist. Der Prozess wird effizient durchgeführt, weil die Größe des neuen Arrays so gewählt wird, dass diese rechenintensiven Operationen seltener auftreten.

Der Zugriff auf die Elemente eines dynamischen Arrays erfolgt in konstanter Zeit `O(1)`, daher ist diese Operation sehr effizient.

Das Einfügen eines Elements in die Mitte des Arrays benötigt lineare Zeit `O(n)`, weil die Elemente danach verschoben werden müssen.

Das Einfügen eines Elements am Ende des Arrays benötigt konstante Zeit `O(1)`, wenn im dynamischen Array noch Platz ist. Ist das Array jedoch voll und muss vergrößert werden, hat diese Operation eine `O(n)`-Komplexität.

Sie sollten dynamische Arrays verwenden, wenn Sie die Anzahl der Werte, die Sie speichern müssen, nicht im Voraus kennen. Sie sind auch hilfreich, wenn Sie häufig Elemente einfügen und löschen.

Um ein Array in JavaScript zu erstellen, schreiben Sie die Elemente in eckige Klammern, getrennt durch Kommas:

```js
let numbers = [3, 4, 5, 6];
```

Dies erstellt ein dynamisches Array mit vier Elementen.

Sie können auf ein Element im Array zugreifen, indem Sie den Variablennamen schreiben, gefolgt von eckigen Klammern mit dem Index.

Die Indizes beginnen bei 0 für das erste Element und erhöhen sich um 1 für jedes nächste Element.

```js
numbers[0]; // 3
numbers[1]; // 4
numbers[2]; // 5
numbers[3]; // 6
```

Um einen Wert zu aktualisieren, müssen Sie ihn einfach neu zuweisen:

```js
numbers[2] = 16
```

Jetzt wird das Array zu `[3, 4, 16, 6]`.

Um ein Element am Ende des Arrays hinzuzufügen, verwenden Sie die `.push()`-Methode:

```js
numbers.push(7)
```

Jetzt wird das Array zu `[3, 4, 16, 6, 7]`:

Um Elemente an einem bestimmten Index einzufügen, hat JavaScript keine `.insert()` wie Python, aber Sie können die `.splice()`-Methode verwenden. Die Syntax sieht so aus:

```js
array.splice(index, deleteCount, newElement)
```

Hier ein Beispiel:

```js
numbers.splice(3, 0, 15);
```

Dieses Beispiel bedeutet:

- Beginne bei Index 3
- Entferne 0 Elemente
- Füge 15 ein

Wenn das Array `[3, 4, 16, 6, 7]` war, wird es jetzt zu `[3, 4, 16, 15, 6, 7]`.

Sie können ein Element an einem bestimmten Index mit derselben `.splice`-Methode entfernen:

```js
numbers.splice(2, 1);
```

Das bedeutet, beginne bei Index 2 und entferne 1 Element.

Um das letzte Element zu entfernen, verwenden Sie die `.pop()`-Methode:

```js
numbers.pop();
```

Es gibt viele weitere eingebaute Array-Methoden, die Sie in der Dokumentation für die Arbeit mit Arrays in JavaScript nachschlagen können.

Das ist die Stärke dynamischer Arrays.

Im Allgemeinen sollten Sie statische Arrays verwenden, wenn Sie die Anzahl der Elemente im Voraus kennen und häufig darauf zugreifen müssen, und dynamische Arrays, wenn die Anzahl der Elemente unbekannt oder variabel ist.

Sie sollten immer den Kompromiss zwischen der Einfachheit statischer Arrays und der Flexibilität dynamischer Arrays bedenken. Beide sind für bestimmte Anwendungsfälle und Szenarien hilfreich. Die Fähigkeit, das beste für ein gegebenes Problem auszuwählen, gehört zu den Problemlösungsfähigkeiten, die Sie mit der Zeit durch Übung entwickeln werden.
