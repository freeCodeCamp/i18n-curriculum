# --description--

Jetzt, da Sie das CommonJS-Modulsystem verstehen, schauen wir uns das ES-Modulsystem an.

ES-Module, oder ECMAScript-Module, sind das offizielle, standardisierte Modulformat für JavaScript, definiert durch die ECMAScript-Spezifikation. Sie verwenden `import` und `export`-Anweisungen, und in Node.js werden sie asynchron geladen, was effizienter ist.

Node.js behandelt Dateien als ES-Module in folgenden Fällen:

* Dateien mit einer `.mjs`-Erweiterung.
    
* Und Dateien mit einer `.js`-Erweiterung, wenn die nächstgelegene übergeordnete `package.json`-Datei ein oberstes `"type"`-Feld mit dem Wert `"module"` enthält.
    

Hier haben wir dieselbe `multiply`-Funktion, die Sie schon zuvor gesehen haben, aber jetzt verwenden wir das ESM-System. Dafür weisen wir beiden Modulen eine `.mjs`-Erweiterung zu.

Um die `multiply`-Funktion aus dem `math.mjs`-Modul zu exportieren, verwenden Sie das Schlüsselwort `export` gefolgt vom Namen der Funktion, Variablen oder des Objekts, das Sie innerhalb geschweifter Klammern exportieren möchten:

```javascript
function multiply(a, b) {
  return a * b;
}

export { multiply };
```

Sie können die Funktion auch direkt exportieren, indem Sie das Schlüsselwort `export` vor dem Schlüsselwort `function` verwenden. Das ist als benannter Export bekannt. Pro Modul können Sie mehrere benannte Exporte haben:

```javascript
export function multiply(a, b) {
  return a * b;
}
```

Um dieses Modul dann zu importieren, verwenden Sie das Schlüsselwort `import` gefolgt von geschweiften Klammern. Beachten Sie, dass wir `import` anstelle von `require()` verwenden. Das ist einer der Hauptunterschiede zwischen CommonJS und ES-Modulen.

Innerhalb der geschweiften Klammern schreiben wir den Namen der Funktion, Variablen oder des Objekts, das wir importieren möchten. In diesem Fall ist es `multiply`. Dann verwenden Sie das Schlüsselwort `from` und den relativen Pfad zum Modul. Dieser Code wird erfolgreich ausgeführt, und die Ausgabe wird `42` sein.

```javascript
import { multiply } from './math.mjs';

console.log(multiply(6, 7)); // 42
```

Im ES-System können Sie pro Modul einen Standardexport festlegen. So müssen Sie keine geschweiften Klammern verwenden, um ihn zu importieren.

Hier ist ein Beispiel, bei dem wir die `multiply`-Funktion als Standardexport für das gesamte `math.mjs`-Modul festlegen:

```javascript
export default function multiply(a, b) {
  return a * b;
}
```

Jetzt brauchen wir keine geschweiften Klammern mehr, da die Funktion standardmäßig importiert und zugewiesen wird:

```javascript
import multiply from './math.mjs';
```

Wenn Sie keine Dateierweiterungen oder die `package.json`-Datei verwenden, um Node.js explizit mitzuteilen, welches Modulsystem Sie verwenden möchten, wird der Quellcode auf ES-Modul-Syntax untersucht. Wenn diese Syntax gefunden wird, wird der Code als ES-Modul ausgeführt; andernfalls als CommonJS-Modul.

Während CommonJS in den frühen Tagen von Node.js als Standard diente, ist es wichtig, die Unterschiede zu kennen und zu wissen, wie man beide verwendet, da Entwickler zunehmend ESM wegen seiner Effizienz und Kompatibilität mit modernem JavaScript übernehmen.
