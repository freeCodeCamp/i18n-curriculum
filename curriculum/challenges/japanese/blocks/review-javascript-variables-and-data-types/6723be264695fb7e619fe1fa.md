## HTML、CSS、JavaScriptの連携

HTMLとCSSはウェブサイトの構造を提供しますが、JavaScriptはユーザー入力の処理、要素のアニメーション、さらには完全なウェブアプリケーションの構築など、複雑な機能を可能にしてウェブサイトにインタラクティブ性をもたらします。

## JavaScriptのデータ型

データ型は、プログラムが扱うデータの種類を理解するのに役立ちます。数値、テキスト、その他の何かかどうかを判断します。

- **Number**: 数値は整数と浮動小数点の両方を表します。整数の例には7、19、90があります。 
- **Floating point**: 浮動小数点数は小数点を含む数値です。例として3.14、0.5、0.0001があります。
- **String**: 文字列は引用符で囲まれた文字のシーケンス、つまりテキストです。`"I like coding"` と `'JavaScript is fun'` は文字列の例です。
- **Boolean**: ブール値は2つの可能な値のいずれかを表します：`true` または `false`。条件を表すのに使えます。例えば `isLoggedIn = true` のように。
- **Undefined and Null**: `undefined` の値は宣言されたが値が代入されていない変数です。`null` の値は空の値、つまり意図的に `null` が代入された変数です。
- **Object**: オブジェクトはキーと値のペアの集合です。キーはプロパティ名、値はプロパティの値を表します。

ここで、`pet` オブジェクトは3つのプロパティまたはキーを持っています：`name`、`age`、`type`。それぞれの値は `Fluffy`、`3`、`dog` です。

```js
let pet = {
  name: "Fluffy",
  age: 3,
  type: "dog"
};
```

- **Symbol**: Symbolデータ型はオブジェクトのプロパティの識別子として使える、ユニークで不変の値です。

以下の例では、同じ説明を持つ2つのシンボルが作成されていますが、等しくはありません。

:::interactive_editor

```js
const crypticKey1= Symbol("saltNpepper");
const crypticKey2= Symbol("saltNpepper");
console.log(crypticKey1 === crypticKey2); // false
```

:::

- **BigInt**: 数値が`Number`データ型で扱えないほど大きい場合、BigIntデータ型を使って任意の長さの整数を表現できます。

数値の末尾に`n`を付けることで、BigIntを作成できます。

```js
const veryBigNumber = 1234567890123456789012345678901234567890n;
```

## JavaScriptの変数

- 変数は`let`キーワードを使って宣言できます。 

```js
let cityName;
```

- 変数に値を代入するには、代入演算子`=`を使います。

```js
cityName = "New York";
```

- `let`で宣言された変数は新しい値に再代入できます。

:::interactive_editor

```js
let cityName = "New York";
cityName = "Los Angeles";
console.log(cityName); // Los Angeles
```

:::

- `let`以外に`const`も変数宣言に使えます。ただし、`const`変数は新しい値に再代入できません。

```js
const cityName = "New York";
cityName = "Los Angeles"; // TypeError: Assignment to constant variable.
```

- `const`で宣言された変数は、コード全体で変更が許されない定数の宣言に使われます。例えば`PI`や`MAX_SIZE`のように。

## 変数名の慣例

- 変数名は説明的で意味のあるものにしてください。
- 変数名は`cityName`、`isLoggedIn`、`veryBigNumber`のようにキャメルケースにしてください。
- 変数名は数字で始めてはいけません。文字、`_`、または`$`で始める必要があります。
- 変数名にスペースや特殊文字は使えません。ただし`_`と`$`は例外です。
- 変数名は予約キーワードにしてはいけません。
- 変数名は大文字と小文字を区別します。`age`と`Age`は異なる変数です。

## JavaScriptの文字列と文字列の不変性

- 文字列は引用符で囲まれた文字のシーケンスです。シングルクォートと二重引用符の両方で作成できます。

```js
let correctWay = 'This is a string';
let alsoCorrect = "This is also a string";
```

- JavaScriptの文字列は不変です。つまり、一度作成された文字列の文字を変更できません。ただし、文字列を新しい値に再代入することは可能です。

```js
let firstName = "John";
firstName = "Jane"; // Reassigning the string to a new value
```

## JavaScriptの文字列連結

- 連結は複数の文字列を結合したり、文字列を保持する変数と組み合わせたりする処理です。`+`演算子は文字列を連結する最も簡単でよく使われる方法の一つです。

:::interactive_editor

```js
let studentName = "Asad";
let studentAge = 25;
let studentInfo = studentName + " is " + studentAge + " years old.";
console.log(studentInfo); // Asad is 25 years old.
```

:::

- 既存の文字列に追加や追記をしたい場合は、`+=`演算子を使えます。これは文字列に時間をかけてテキストを追加していく場合に便利です。

:::interactive_editor

```js
let message = "Welcome to programming, ";
message += "Asad!";
console.log(message); // Welcome to programming, Asad!
```

:::

- 文字列を連結するもう一つの方法は`concat()`メソッドを使うことです。このメソッドは2つ以上の文字列を結合します。

:::interactive_editor

```js
let firstName = "John";
let lastName = "Doe";
let fullName = firstName.concat(" ", lastName);
console.log(fullName); // John Doe
```

:::

## `console.log()`でのメッセージのログ出力

- `console.log()`メソッドはコンソールにメッセージをログ出力するために使います。コードのデバッグやテストに役立つツールです。

:::interactive_editor

```js
console.log("Hello, World!");
// Output: Hello, World!
```

:::

## JavaScriptのセミコロン

- セミコロンは主に文の終わりを示すために使われます。これによりJavaScriptエンジンは個々の命令の区切りを理解し、正しい実行が可能になります。

```js
let message = "Hello, World!"; // first statement ends here
let number = 42; // second statement starts here
```

- セミコロンはコードの実行における曖昧さを防ぎ、文が正しく終了していることを保証します。

## JavaScriptのコメント

- コメントアウトされたコードの行はJavaScriptエンジンに無視されます。コメントはコードの説明、メモ、または一時的にコードを無効にするために使います。
- 一行コメントは`//`を使って作成します。

```js
// This is a single-line comment and will be ignored by the JavaScript engine
```

- 複数行コメントは`/*`で開始し、`*/`で終了します。

```js
/*
This is a multi-line comment.
It can span multiple lines.
*/
```

## 動的型付け言語としてのJavaScript

- JavaScriptは動的型付け言語であり、変数を宣言するときにデータ型を指定する必要はありません。JavaScriptエンジンが代入された値に基づいて自動的にデータ型を判断します。

```js
let error = 404; // JavaScript treats error as a number
error = "Not Found"; // JavaScript now treats error as a string
```

- C#のような動的型付けでない言語では、エラーになります。

```csharp
int error = 404; // value must always be an integer
error = "Not Found"; // This would cause an error in C#
```

## `typeof`演算子の使用

- `typeof`演算子は変数のデータ型をチェックするために使います。変数の型を示す文字列を返します。

:::interactive_editor

```js
let age = 25;
console.log(typeof age); // "number"

let isLoggedIn = true;
console.log(typeof isLoggedIn); // "boolean"
```

:::

- ただし、JavaScriptには`null`に関してよく知られた特異点があります。`typeof`演算子は`"object"`の値に対して`null`を返します。

:::interactive_editor

```js
let user = null;
console.log(typeof user); // "object"
```

:::
