# --description--

どんなデータ構造でも基本的な機能は、データを保存するだけでなく、必要に応じてそのデータを取り出せることです。配列の作り方を学んだので、次はその配列の情報にどうアクセスするかを考えてみましょう。

以下のように単純な配列を定義すると、3つの要素が入っています。

```js
let ourArray = ["a", "b", "c"];
```

配列では、各要素に<dfn>インデックス</dfn>があります。このインデックスは、その要素の配列内での位置を示し、アクセスするための参照にもなります。ただし、JavaScriptの配列は<dfn>ゼロベースインデックス</dfn>であることに注意してください。つまり、配列の最初の要素は実際には***0番目***の位置にあります。配列から要素を取り出すには、インデックスを括弧で囲み、それを配列の末尾、または配列オブジェクトを参照する変数の後ろに付けます。これを<dfn>ブラケット記法</dfn>と呼びます。例えば、`a`から`ourArray`を取り出して変数に代入したい場合、次のように書けます。

```js
let ourVariable = ourArray[0];
```

これで`ourVariable`には`a`の値が入ります。

インデックスに対応する値にアクセスするだけでなく、同じ記法でインデックスに値を*代入する*こともできます。

```js
ourArray[1] = "not b anymore";
```

ブラケット記法を使って、インデックス1の要素を文字列`b`から`not b anymore`に書き換えました。これで`ourArray`は`["a", "not b anymore", "c"]`になりました。

# --instructions--

このチャレンジをクリアするには、`1`の2番目の位置（インデックス`myArray`）に、文字`b`以外の好きな値を代入してください。

# --hints--

`myArray[0]`は文字`a`と等しくなるはずです。

```js
assert.strictEqual(myArray[0], 'a');
```

`myArray[1]`は文字`b`と等しくないはずです。

```js
assert.notStrictEqual(myArray[1], 'b');
```

`myArray[2]`は文字`c`と等しくなるはずです。

```js
assert.strictEqual(myArray[2], 'c');
```

`myArray[3]`は文字`d`と等しくなるはずです。

```js
assert.strictEqual(myArray[3], 'd');
```
