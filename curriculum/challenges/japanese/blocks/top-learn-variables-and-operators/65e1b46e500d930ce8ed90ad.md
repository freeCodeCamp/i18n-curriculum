# --description--

数値を1ずつ増減させる操作は、最も一般的な数値操作の一つです。

そのため、これを行うための特別な演算子があります。

- インクリメント `++` は変数の値を1増やします。

```js
let counter = 2;

// works the same as counter = counter + 1, but is shorter
counter++;      

console.log(counter); // 3
```

- デクリメント `--` は変数の値を1減らします。

```js
let counter = 2;

// works the same as counter = counter - 1, but is shorter
counter--;

console.log(counter); // 1
```

演算子 `++` と `--` は変数の前後どちらにも置くことができます。

- 演算子が変数の後ろにある場合は「後置形式」と呼ばれます：`counter++`。

- 演算子が変数の前にある場合は「前置形式」と呼ばれます：`++counter`。

どちらの文も同じことをしています：`counter` を `1` だけ増やしています。

違いはありますか？はい、ただし `++/--` の返り値を使った場合にのみ違いが見えます。

説明しましょう。ご存じの通り、すべての演算子は値を返します。インクリメント／デクリメントも例外ではありません。前置形式は新しい値を返し、後置形式は増減前の古い値を返します。

違いを確認するために、例を示します。

```js
let counter = 1;
let a = ++counter; // (*)

console.log(a); // 2
```

まとめると：

- インクリメント／デクリメントの結果を使わない場合は、どちらの形式を使っても違いはありません。

```js
let counter = 0;
counter++;
++counter;

console.log( counter ); // 2, the lines above did the same
```

- 値を増やしてすぐに演算子の結果を使いたい場合は、前置形式を使う必要があります。

```js
let counter = 0;
console.log( ++counter ); // 1
```

- 値を増やすが、増やす前の値を使いたい場合は、後置形式を使う必要があります。

```js 
let counter = 0;
console.log( counter++ ); // 0
```

# --questions--

## --text--

以下のJavaScriptコードの2つの`console.log`文の出力は何ですか？

```js
let counter = 1;
console.log(2 * ++counter); // Statement A
```

```js
let counter = 1;
console.log(2 * counter++); // Statement B
```

## --answers--

違いはありません。どちらの`console.log`文も同じ結果を表示します。

---

文Aは`4`をログに出力します。なぜなら`++counter`が掛け算の前に値を増やすからです。文Bは`2`をログに出力します。なぜなら`counter++`が掛け算の後に値を増やすからです。

---

文Aは`2`をログに出力します。なぜなら`++counter`が構文エラーだからです。文Bは`4`をログに出力します。なぜなら`counter++`がカウンターを増やす正しい方法だからです。

---

どちらの文も`3`をログに出力します。なぜならどちらの場合もカウンターは`console.log`の前に増やされるからです。
