# --description--

Nachdem wir uns Encapsulation, Vererbung und Polymorphismus angesehen haben, besprechen wir nun das nächste Schlüsselkonzept der objektorientierten Programmierung – Abstraktion.

Abstraktion ist der Prozess, komplexe Implementierungsdetails zu verbergen und nur die wesentlichen Funktionen eines Objekts oder Systems anzuzeigen. Man kann es sich so vorstellen, dass man sich darauf konzentriert, was etwas tut, anstatt wie es das tut.

Abstraktion ist nicht auf Python beschränkt. Es ist ein Programmierkonzept, das in vielen Sprachen umgesetzt werden kann, die objektorientierte Programmierung unterstützen.

Um Abstraktion zu veranschaulichen, stellen Sie sich vor, Sie fahren ein Auto. Wenn Sie am Steuer sitzen, interagieren Sie mit wichtigen Teilen wie dem Lenkrad, dem Schalthebel sowie dem Gaspedal und der Bremse. Sie müssen nicht die komplizierten Details kennen, wie der Motor funktioniert, wie das Getriebe die Gänge wechselt oder die Physik hinter dem Bremssystem, denn all das sind komplexe Implementierungsdetails.

Das ist Abstraktion in Aktion! Sie erhalten eine vereinfachte Schnittstelle, um mit einem komplexen System zu interagieren.

Im Fall eines Autos ist die vereinfachte Schnittstelle das Lenkrad, die Bremsen und das Gaspedal, während das komplexe System das Auto selbst ist.

Wie Python Abstraktion umsetzt, geschieht über das `abc`-Modul.

Dieses Modul stellt die `ABC`-Klasse (steht für „abstract base class“) und den `@abstractmethod`-Decorator bereit.

`ABC` ist die Klasse, von der geerbt werden soll, aber von der Sie keine direkten Objekte erstellen können. Sie definiert eine gemeinsame Schnittstelle von Methoden und Eigenschaften, die ihre Unterklassen implementieren müssen.

Eine abstrakte Methode ist eine Methode, die in einer Abstract Base Class (ABC) mit dem `@abstractmethod`-Decorator deklariert wird. Sie kann keine oder eine grundlegende Standardimplementierung haben. Jede Unterklasse muss sie jedoch überschreiben, um als konkret und instanziierbar zu gelten, selbst wenn eine Standardimplementierung vorhanden ist.

Hier ist die grundlegende Syntax einer abstrakten Klasse in Python:

```py
from abc import ABC, abstractmethod

# Define an abstract base class
class AbstractClass(ABC):
    @abstractmethod
    def abstract_method(self):
        pass

# Concrete subclass that implements the abstract method
class ConcreteClassOne(AbstractClass):
    def abstract_method(self):
        print('Implementation in ConcreteClassOne')

# Another concrete subclass
class ConcreteClassTwo(AbstractClass):
    def abstract_method(self):
        print('Implementation in ConcreteClassTwo')
```

Hier ein einfaches Beispiel:

```py
from abc import ABC, abstractmethod

class Animal(ABC): # Inherits from abstract base class
   @abstractmethod # Abstract method decorator
   def make_sound(self):  # The method subclasses must override
       pass

# Concrete class that will override the abstract method
class Dog(Animal):
   def make_sound(self):
       print('Woof!')

# Another concrete class that will override the abstract method
class Cat(Animal):
   def make_sound(self):
       print('Meow!')

# Another concrete class that will override the abstract method
class Monkey(Animal):
   def make_sound(self):
       print('Ooh ooh aah aah!')

# Create instances of each concrete class
animals = [Dog(), Cat(), Monkey()]

# Loop through the instances to call the make_sound method
for animal in animals:
   animal.make_sound()

# Output:
# Woof!
# Meow!
# Ooh ooh aah aah!
```

In diesem Beispiel:

- Wir importieren die `ABC`-Klasse und `abstractmethod` aus dem `abc`-Modul.
- Dann erstellen wir eine `Animal`-Klasse, die von `ABC` erbt, und definieren darin eine abstrakte Methode `make_sound`, die jede Unterklasse von `Animal` überschreiben muss.
- Wir erstellen die konkreten Klassen `Dog`, `Cat` und `Monkey`, die die abstrakte Methode `make_sound` überschreiben müssen.
- Wir instanziieren die konkreten Klassen und rufen deren `make_sound`-Methode auf, um zu zeigen, wie jede von ihnen die abstrakte Methode `make_sound` auf ihre eigene Weise implementiert.

Denken Sie daran, dass Sie keine Instanz der Animal-Klasse erstellen können. So sieht es aus, wenn Sie es versuchen:

```py
dog = Animal() 
# TypeError: Can't instantiate abstract class Animal 
# without an implementation for abstract method 'make_sound'
```

Die gleiche Regel gilt für Unterklassen, die keine Implementierung für die abstrakte Methode bereitstellen. Selbst wenn Sie eine Unterklasse von Animal definieren, können Sie sie nicht instanziieren, bis sie make_sound überschreibt. Hier ein Beispiel dazu:

```py
class Bird(Animal):
    pass

bird = Bird()
# TypeError: Can't instantiate abstract class Bird 
# without an implementation for abstract method 'make_sound'
```

Hier ein weiteres Beispiel, diesmal mit einem Instanzattribut, das Sie den Instanzen der konkreten Methoden übergeben können:

```py
from abc import ABC, abstractmethod

# The blueprint for any toy that can speak
class TalkingToy(ABC):
   def __init__(self, name):
       self.name = name
   @abstractmethod
   def speak(self):
       pass

class RobotToy(TalkingToy):
   def speak(self):
       print(f'{self.name} says beep boop! I am a robot!')

class TeddyBearToy(TalkingToy):
   def speak(self):
       print(f"{self.name} says hug me! I'm cuddly!")

class DinosaurToy(TalkingToy):
   def speak(self):
       print(f'{self.name} says ROOOOAR!')

# Create toys
rusty = RobotToy('Rusty')
fluffy = TeddyBearToy('Fluffy')
rex = DinosaurToy('Rex')

toys = [rusty, fluffy, rex]
for toy in toys:
   toy.speak()

# Output:
# Rusty says beep boop! I am a robot!
# Fluffy says hug me! I'm cuddly!
# Rex says ROOOOAR!
```

In diesem Beispiel:

- Wir haben eine abstrakte Basisklasse `TalkingToy`, die eine Vorlage für jedes Spielzeug definiert, das sprechen kann.
- Die Unterklassen `RobotToy`, `TeddyBearToy` und `DinosaurToy` implementieren die `speak`-Methode auf ihre eigene Weise.
- Wenn wir Instanzen dieser Unterklassen erstellen und die `speak`-Methode aufrufen, spricht jedes Spielzeug auf seine einzigartige Weise.

Zusammenfassend vereinfacht Abstraktion in Python komplexe Systeme, indem sie die Wiederverwendbarkeit erhöht.

Sie haben gesehen, wie Sie eine einzelne Methode aus einer abstrakten Klasse in mehreren Unterklassen wiederverwenden können, während Sie jede Unterklasse zwingen, ihr spezifisches Verhalten bereitzustellen.

Dieser Ansatz hält Ihren Code organisiert, flexibel und leichter wartbar, besonders wenn Ihre Anwendung wächst.
