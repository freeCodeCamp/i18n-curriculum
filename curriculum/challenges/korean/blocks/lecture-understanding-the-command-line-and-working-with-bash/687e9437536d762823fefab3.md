# --description--

생산성을 높이기 위해 사용할 수 있는 명령줄 단축키에 대해 배워보세요.

터미널과 명령줄 단축키는 작업 흐름을 간소화하고 생산성을 극대화하는 데 도움이 됩니다. Linux와 macOS는 모두 Unix에서 유래했기 때문에 많은 단축키가 동일합니다. 하지만 Windows에서는 약간의 차이가 있을 수 있습니다.

첫 번째 단축키는 위쪽 화살표 키입니다. 이 키를 누르면 명령 기록을 한 명령씩 거꾸로 탐색할 수 있습니다. 아래쪽 화살표 키는 명령 기록을 앞으로 탐색하는 데 사용합니다. 이 두 키를 사용하면 이전에 실행한 명령을 빠르게 찾아 다시 실행할 수 있습니다.

많은 셸은 자동 완성 기능도 제공합니다. 입력을 시작하면 명령을 제안해 주는데, 탭 키를 눌러 제안된 나머지 부분을 채워 명령줄에 전체 구문을 빠르게 입력할 수 있습니다. 이러한 제안이 생성되는 방식은 셸마다 다릅니다. 예를 들어, zsh에서는 최근 명령 기록을 기반으로 제안하지만, PowerShell에서는 사용 가능한 명령, 변수, 인수를 기반으로 하며 최근 사용 빈도는 덜 반영합니다.

때로는 명령이 터미널에 많은 출력을 생성할 수 있습니다. 이런 추가 출력은 작업 흐름을 방해하거나 산만하게 만들 수 있습니다. Linux와 macOS 장치에서 사용하는 *nix 기반 터미널에서는 Control + L을 눌러 화면 전체를 지우고 새로 깨끗한 프롬프트로 시작할 수 있습니다. PowerShell에서는 `cls` 명령을 실행해야 하며, 이 명령을 Control + L 같은 키 조합에 바인딩할 수 있습니다.

명령 실행을 중단하려면 Control + C를 사용해 프로세스를 종료할 수 있습니다. 이 명령은 실행 중인 명령을 종료하고 새 프롬프트를 생성해 다른 작업을 계속할 수 있게 합니다. PowerShell 사용자에게 Control + C는 텍스트 복사에도 사용되며, 복사할 텍스트가 선택되지 않은 모호하지 않은 상황에서만 프로세스 종료에 사용됩니다.

다음 몇 가지 단축키는 *nix 기반 터미널에서만 사용할 수 있으며 PowerShell에는 대응하는 단축키가 없습니다.

때로는 멀티태스킹이 필요할 수 있습니다. 즉, 한 프로세스나 명령을 백그라운드에서 실행하면서 다른 작업을 진행하는 경우입니다. Control + Z를 누르면 현재 프로세스를 백그라운드 작업으로 전환하고 명령줄로 돌아와 작업을 계속할 수 있습니다. 백그라운드 작업으로 포커스를 다시 전환하려면 `fg`을 사용해 복원하세요.

마지막으로, 연속해서 두 번 실행해야 하는 명령이 있을 수 있습니다. 위쪽 화살표 키로 이전에 실행한 명령을 거꾸로 탐색할 수도 있지만, 명령줄에 느낌표 두 개(`!!`)를 입력하고 엔터를 누르면 마지막으로 실행한 명령이 자동으로 실행됩니다.

많은 터미널은 더 많은 단축키를 제공하니, 자신이 사용하는 환경의 문서를 참고해 가장 적합한 단축키를 찾아보세요. 하지만 이 기본적인 단축키들은 생산성을 극대화하고 터미널 마법사가 되는 데 훌륭한 출발점이 될 것입니다.
