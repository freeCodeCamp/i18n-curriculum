# --description--

이전 수업에서 JavaScript에서 가져오기와 내보내기를 다루는 방법을 배웠습니다. 이번 수업에서는 React에서 컴포넌트를 가져오고 내보내는 방법을 살펴보겠습니다.

내보내기는 컴포넌트를 다른 파일에서 사용할 수 있게 만듭니다. 가져오기는 다른 곳에서 내보낸 컴포넌트를 파일에서 사용할 수 있게 합니다. React 프로젝트에서는 이렇게 컴포넌트를 재사용하고 결합하여 사용자 인터페이스를 만듭니다.

이번 수업에서는 기본 내보내기와 이름 내보내기를 사용해 React 컴포넌트를 내보내고, 필요한 곳에서 가져오는 방법을 배웁니다.

이 예제에서는 `Cat` 컴포넌트가 `Cat.jsx`라는 파일에 있습니다. 파일 확장자는 JSX를 주로 다루기 때문에 `.jsx`를 사용합니다.

이 `Cat` 컴포넌트는 Mr. Whiskers라는 고양이의 제목과 이미지를 포함한 JSX 마크업을 반환합니다:

```jsx
function Cat() {
  return (
    <div className="card">
      <h2>Mr. Whiskers</h2>
      <img
        src="https://cdn.freecodecamp.org/curriculum/cat-photo-app/running-cats.jpg"
        alt="Tuxedo cats running on dirt ground."
      />
    </div>
  );
}
```

다른 파일에서 `Cat` 컴포넌트를 사용하려면 먼저 이렇게 내보내야 합니다:

```jsx
function Cat() {
  return (
    <div className="card">
      <h2>Mr. Whiskers</h2>
      <img
        src="https://cdn.freecodecamp.org/curriculum/cat-photo-app/running-cats.jpg"
        alt="Tuxedo cats running on dirt ground."
      />
    </div>
  );
}

export default Cat;
```

이 모듈에서 기본 내보내기가 될 것이므로 `default` 키워드를 사용합니다. 컴포넌트 정의와 같은 줄에서 내보내기를 선택할 수도 있습니다:

```jsx
export default function Cat() {
  return (
    <div className="card">
      <h2>Mr. Whiskers</h2>
      <img
        src="https://cdn.freecodecamp.org/curriculum/cat-photo-app/running-cats.jpg"
        alt="Tuxedo cats running on dirt ground."
      />
    </div>
  );
}
```

다른 부모 컴포넌트 파일에서 자식 컴포넌트를 가져오거나 루트 컴포넌트 파일에서 가져올 수 있습니다. 이 예제에서는 루트 컴포넌트 파일 안에서 `Cat` 컴포넌트를 가져오겠습니다.

모든 React 프로젝트에는 보통 `App.jsx`라는 최상위 컴포넌트가 있습니다:

```jsx
export default function App() {
  return (
    // return component here
  );
}
```

이 파일은 보통 프로젝트의 `src` 디렉터리에 있습니다. 일반적인 프로젝트 구조는 이후 수업에서 더 배우게 됩니다.

루트 `Cat` 컴포넌트 안에서 `App` 컴포넌트를 사용하려면 이렇게 가져와야 합니다:

```jsx
import Cat from "./Cat";

export default function App() {
  return (
    // return component here
  );
}
```

이제 `Cat` 컴포넌트 안에서 `App` 컴포넌트를 반환할 수 있습니다:

```jsx
import Cat from "./Cat";

export default function App() {
  return (
    <Cat />
  );
}
```

이 방법은 파일이 단일 주요 컴포넌트를 내보내는 역할을 할 때 잘 작동하며, 가져오기 구문을 간단하고 읽기 쉽게 유지합니다. 파일에는 기본 내보내기가 하나만 있을 수 있으므로, 주로 하나의 컴포넌트를 포함하는 파일에 적합합니다.

기본 내보내기를 사용하는 방법을 알았으니, 이름 내보내기를 살펴보겠습니다. 이름 내보내기는 파일이 여러 컴포넌트나 함수를 공유할 수 있게 합니다. 기본 내보내기와 달리, 내보낸 정확한 이름으로 가져와야 합니다(단, `as` 키워드를 사용해 이름을 바꿀 수 있습니다).

이것은 파일이 컴포넌트 라이브러리 역할을 할 때 유용합니다. 예를 들어, 두 개의 컴포넌트 Cat과 Dog가 있는 파일이 있습니다:

```jsx
// Animals.jsx
export function Cat() {
  return <h2>Mr. Whiskers</h2>;
}

export function Dog() {
  return <h2>Fido</h2>;
}
```

각 컴포넌트에 export 키워드를 개별적으로 사용합니다. 다른 파일에서 사용하려면 중괄호를 사용해 이렇게 가져옵니다:

```jsx
import { Cat, Dog } from "./Animals";

export default function App() {
  return (
    <div>
      <Cat />
      <Dog />
    </div>
  );
}
```

`{ Cat, Dog }` 구문은 JavaScript에 `Animals.jsx` 파일에서 특정 이름 내보내기를 가져오라고 알려줍니다. 중괄호 안의 이름은 내보낸 이름과 정확히 일치해야 합니다. 이름 충돌을 피하거나 현재 파일에서 다른 이름을 선호한다면, as 키워드를 사용해 가져올 때 이름을 바꿀 수 있습니다.

```jsx
import { Cat as Kitty } from "./Animals";

export default function App() {
  return <Kitty />;
}
```

마지막으로, 파일은 하나의 기본 내보내기와 여러 이름 내보내기를 가질 수 있습니다.

```jsx
// Animals.jsx
export default function Cat() {
  return <h2>Mr. Whiskers</h2>;
}

export function Dog() {
  return <h2>Fido</h2>;
}
```

혼합 내보내기를 가져올 때는 기본 내보내기가 먼저 오고(중괄호 없이), 그 다음에 이름 내보내기들이 옵니다(중괄호 안에):

```jsx
// App.jsx
import Cat, { Dog } from "./Animals";

export default function App() {
  return (
    <div>
      <Cat />
      <Dog />
    </div>
  );
}
```

이 예제에서 `Cat`는 기본 내보내기이므로 중괄호를 사용하지 않습니다. `Dog`는 이름 내보내기이므로 중괄호가 필요합니다.

기본 내보내기와 이름 내보내기가 어떻게 작동하는지 이해하면, 여러 파일에 걸쳐 React 컴포넌트를 체계적으로 구성하고 필요한 곳에서 재사용할 수 있습니다. 더 큰 애플리케이션을 만들 때 적절한 내보내기 방식을 선택하면 코드를 명확하고 유지보수하기 쉽게 만들고, 다른 사람이 만든 컴포넌트와 작업하기도 수월해집니다.
