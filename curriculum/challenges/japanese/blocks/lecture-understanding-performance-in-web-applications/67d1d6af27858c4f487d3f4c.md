# --description--

パフォーマンスについて話すとき、その言葉自体が少し主観的に感じられることがありますよね？良いパフォーマンスとは何でしょうか？悪いパフォーマンスとは何でしょうか？

パフォーマンスを測る標準的な方法はあるのでしょうか、それとも完全にユーザーの個人的な好みや知覚に依存するのでしょうか？見てみましょう。

ウェブサイトのパフォーマンスは、知覚されるパフォーマンスと実際のパフォーマンスの2つの異なるカテゴリーに分類できます。

知覚されるパフォーマンスとは、ユーザーがウェブサイトのパフォーマンスをどのように感じるかです。レスポンスの速さや信頼性の観点から評価する方法です。これは主観的な測定なので定量化は難しいですが、非常に重要です。なぜならユーザー体験がウェブサイトの成功や失敗を決めるからです。

一方、実際のパフォーマンスはウェブサイトの客観的かつ測定可能なパフォーマンスです。ページの読み込み時間、サーバーレスポンス時間、レンダリング時間などの指標で測定されます。これらの測定値はネットワークやコード自体に関連する複数の要因に影響されます。

優れた知覚パフォーマンスを持つウェブサイトでも、さらに最適化してより良いユーザー体験を提供できます。

知覚パフォーマンスを改善するためのいくつかの手法があります。

非必須のリソースをバックグラウンドで読み込むことで初期読み込み時間をできるだけ短くすることが重要です。この手法は「レイジーローディング」として知られています。最初の印象がユーザーのウェブサイトに対する評価を決めることがあります。

レイジーローディングはウェブサイトの実際のパフォーマンスにも影響します。この手法を使うことで、最初に読み込む必要のあるコンテンツの量を減らせるため、ページの読み込みが速くなり、これは実際のパフォーマンスの指標となります。

ユーザーの操作に対して迅速なレスポンスとフィードバックを提供することも非常に重要です。

例えば、ユーザーが要素をクリックしたらすぐに長時間かかる処理の読み込みインジケーターを表示すると、ユーザーは処理に関与していると感じ、待ち時間が短く感じられたり、少なくともより関わっている感覚を持てます。

ユーザーを積極的にウェブサイトに引き込むことは、時間の知覚を改善するために非常に重要です。テキストを到着したらすぐに表示することは、画像など他のリソースがまだ読み込まれていなくても、最初からユーザーの関心を保つのに役立ちます。

ユーザーが長時間かかるタスクの完了を待っている場合は、進捗状況を知らせて定期的に更新してください。

もう一つのポイントは、コンテンツの再配置やジャンプを避けることです。例えば、広告や画像が読み込まれるとき、レイアウトにスペースが確保されていない場合は、ウェブサイトがジャンプしたり再調整されたりすることがあります。

これらの突然の変化は、ユーザーにウェブサイトがまだ読み込み中だと感じさせるため、悪いユーザー体験につながります。これを避けるために、最初からこれらの要素のスペースを確保しておくことが重要です。

カスタムフォントを使っている場合は、フォントの読み込み遅延を最小限に抑えることも試みてください。遅延があると、カスタムフォントが読み込まれるまでフォールバックフォントが表示されてちらつきが起きることがあります。これを防ぐために、カスタムフォントに似たフォールバックフォントを使うと、もし起きても変化がより自然になります。

また、インターフェースの要素がアクティブであることを確認してください。ユーザーは最小限の遅延で操作できる必要があります。

これらの手法については、次のいくつかのレッスンでさらに詳しく説明します。

実際のパフォーマンスも重要ですが、知覚されるパフォーマンスはユーザー体験に大きな影響を与えます。両方を最適化することで、速く感じられ、スムーズで魅力的なユーザー体験を作り出せます。
