# --description--

在上一課中，我們學習了如何使用 `localStorage`，並簡要介紹了 `sessionStorage`。

請記住，`sessionStorage` 是指當使用者關閉執行網頁應用程式的分頁或視窗時，資料會立即被清除。它非常適合資料只需在單一會話期間持續存在的情況，例如在導覽過程中維護表單資料或在結帳流程中儲存臨時狀態資訊。

就像 `localStorage` 一樣，`sessionStorage` 使用鍵值配對來儲存和擷取資料。`sessionStorage` 使用的方法也和 `localStorage` 相同，唯一真正的差別在於資料儲存的時間長短。 

以下是使用不同方法的幾個範例：

- `sessionStorage.setItem()`：在 `sessionStorage` 中儲存一個鍵值配對。

```js
sessionStorage.setItem('cart', '3 items');
```

- `sessionStorage.getItem()`：從 `sessionStorage` 取得指定鍵的值。

```js
let cart = sessionStorage.getItem('cart');
console.log(cart); // Outputs: '3 items'
```

- `sessionStorage.removeItem()`：使用其鍵從 `sessionStorage` 中移除特定項目。

```js
sessionStorage.removeItem('cart');
```

- `sessionStorage.clear()`：清除儲存在 `sessionStorage` 中的所有資料。

```js
sessionStorage.clear();
```

讓我們來看一個範例，我們將資料儲存在 `sessionStorage` 中，該資料只會在瀏覽器分頁或視窗開啟期間存在：

```js
// Store data in sessionStorage
sessionStorage.setItem('currentUser', 'JohnDoe');

// Retrieve the stored data
const user = sessionStorage.getItem('currentUser');
console.log(user); // 'JohnDoe'

// Remove a specific key from sessionStorage
sessionStorage.removeItem('currentUser');

// Clear all sessionStorage data
sessionStorage.clear();
```

在這個範例中，我們：

1. 將目前使用者的名稱（`JohnDoe`）儲存在 `sessionStorage`。

2. 擷取並顯示它。

3. 移除與鍵 `currentUser` 相關的項目。

4. 清除所有 `sessionStorage` 資料。

與 `localStorage` 的主要差異在於，只要使用者關閉分頁，所有儲存的會話資料將會遺失。

`sessionStorage` 在以下情境中特別有用：

- 在多頁表單行程中儲存臨時資料，例如表單輸入。

- 儲存不需要跨會話持續的臨時選擇或偏好。

- 在單頁應用程式中維護狀態，且在分頁關閉後不需要被記住。

`sessionStorage` 確保使用者離開頁面後，會清除該會話的資料，這對於你不想保留超過當前會話的資訊的情境非常適合。
