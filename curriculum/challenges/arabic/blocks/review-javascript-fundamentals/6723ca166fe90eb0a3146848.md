## دالة البناء للسلسلة النصية وطريقة `toString()`

- **التعريف**: يُستخدم كائن السلسلة النصية لتمثيل متتالية من الأحرف. تُنشأ كائنات السلسلة النصية باستخدام دالة البناء `String`، التي تغلف القيمة الأولية داخل كائن.

:::interactive_editor

```js
const greetingObject = new String("Hello, world!");

console.log(typeof greetingObject); // "object"
```

:::

- **طريقة `toString()`**: هذه الطريقة تحول القيمة إلى تمثيلها كسلسلة نصية. يمكنك استخدامها للأرقام، والقيم المنطقية، والمصفوفات، والكائنات.

:::interactive_editor

```js
const num = 10;
console.log(num.toString()); // "10"

const arr = [1, 2, 3];
console.log(arr.toString()); // "1,2,3"
```

:::

تقبل هذه الطريقة معاملًا اختياريًا هو أساس العد (radix) وهو رقم من 2 إلى 36. يمثل هذا الأساس النظام العددي، مثل الأساس 2 للثنائي أو الأساس 8 للثماني. إذا لم يُحدد الأساس، فيُفترض الأساس 10، وهو النظام العشري.


:::interactive_editor

```js
const num = 10;
console.log(num.toString(2)); // "1010"(binary)
```

:::

## دالة البناء للعدد

- **التعريف**: تُستخدم دالة البناء `Number` لإنشاء كائن عدد. يحتوي كائن العدد على بعض الخصائص والطرق المفيدة مثل `isNaN` و`toFixed`. في الغالب، ستستخدم دالة البناء `Number` لتحويل أنواع بيانات أخرى إلى نوع البيانات العددية.

:::interactive_editor

```js
const myNum = new Number("34");
console.log(typeof myNum); // "object"

const num = Number('100');
console.log(num); // 100

console.log(typeof num); // number
```

:::

## أفضل الممارسات لتسمية المتغيرات والدوال

- **camelCasing**: حسب الاصطلاح، يستخدم مطورو JavaScript طريقة camel casing لتسمية المتغيرات والدوال. حيث تكون الكلمة الأولى كلها أحرف صغيرة والكلمات التالية تبدأ بحرف كبير. مثال: `isLoading`.

- **تسمية القيم المنطقية**: للمتغيرات التي تحمل قيمًا منطقية، من الممارسات الشائعة استخدام بادئات مثل "is"، "has"، أو "can".

```js
let isLoading = true;
let hasPermission = false;
let canEdit = true;
```

- **تسمية الدوال**: يجب أن يوضح اسم الدالة بوضوح ما تفعله. للدوال التي تُرجع قيمة منطقية (تُسمى غالبًا predicates)، يمكنك استخدام نفس البادئات "is"، "has"، أو "can". للدوال التي تسترجع بيانات، من الشائع البدء بكلمة "get". للدوال التي تضبط بيانات، من الشائع البدء بكلمة "set". لدوال معالجات الأحداث، قد تستخدم بادئة "handle" أو لاحقة "Handler".

```js
function getUserData() { /* ... */ }

function isValidEmail(email) { /* ... */ }

function getProductDetails(productId) { /* ... */ }

function setUserPreferences(preferences) { /* ... */ }

function handleClick() { /* ... */ }
```

- **تسمية المتغيرات داخل الحلقات**: عند تسمية متغيرات التكرار في الحلقات، من الشائع استخدام أحرف مفردة مثل `i`، `j`، أو `k`.

```js
for (let i = 0; i < array.length; i++) { /* ... */ }
```

## العمل مع المصفوفات المتفرقة

- **التعريف**: من الممكن وجود مصفوفات تحتوي على خانات فارغة. تُعرف الخانات الفارغة بأنها خانات لا تحتوي على أي شيء. هذا يختلف عن خانات المصفوفة التي تحتوي على القيمة `undefined`. تُعرف هذه الأنواع من المصفوفات بالمصفوفات المتفرقة.

:::interactive_editor

```js
const sparseArray = [1, , , 4];
console.log(sparseArray.length); // 4
```

:::

## أدوات التدقيق والتنسيق

- **أدوات التدقيق (Linters)**: أداة تدقيق الشفرة هي أداة تحليل ثابتة تُشير إلى أخطاء البرمجة، والأخطاء البرمجية، وأخطاء الأسلوب، والبنى المشبوهة. مثال شائع لأداة تدقيق هو ESLint.
- **أدوات التنسيق (Formatters)**: أدوات التنسيق هي أدوات تُنسق شفرتك تلقائيًا لتتوافق مع دليل أسلوب معين. مثال شائع لأداة تنسيق هو Prettier.

## إدارة الذاكرة

- **التعريف**: إدارة الذاكرة هي عملية التحكم في الذاكرة، تخصيصها عند الحاجة وتحريرها عند عدم الحاجة إليها. يستخدم JavaScript إدارة ذاكرة تلقائية. هذا يعني أن JavaScript (وبشكل أدق، محرك JavaScript في متصفح الويب) يتولى تخصيص الذاكرة وإلغاء تخصيصها نيابة عنك. لا تحتاج لتحرير الذاكرة صراحة في شفرتك. تُسمى هذه العملية التلقائية غالبًا "جمع القمامة".

## الإغلاقات (Closures)

- **التعريف**: الإغلاق هو دالة لها وصول إلى المتغيرات في نطاقها الخارجي (النطاق اللفظي المحيط)، حتى بعد أن تنتهي الدالة الخارجية من التنفيذ.

:::interactive_editor

```js
function outerFunction(x) {
  let y = 10;
  function innerFunction() {
    console.log(x + y);
  }
  return innerFunction;
}

let closure = outerFunction(5);
closure(); // 15
```

:::

## الكلمة المفتاحية `var` والرفع (Hoisting)

- **التعريف**: كانت `var` الطريقة الأصلية لإعلان المتغيرات قبل 2015. لكن ظهرت بعض المشاكل مع `var` فيما يتعلق بالنطاق، وإعادة الإعلان، وأكثر. لذلك يستخدم برمجة JavaScript الحديثة `let` و`const` بدلاً منها.
- **إعادة إعلان المتغيرات باستخدام `var`**: إذا حاولت إعادة إعلان متغير باستخدام `let`، فستحصل على `SyntaxError`. لكن مع `var`، يُسمح بإعادة إعلان المتغير.

```js
// Uncaught SyntaxError: Identifier 'num' has already been declared 
let num = 19;
let num = 18;

var myNum = 5;
var myNum = 10; // This is allowed and doesn't throw an error

console.log(myNum) // 10
```

- **`var` والنطاق**: المتغيرات المعلنة باستخدام `var` داخل كتلة (مثل جملة `if` أو حلقة `for`) تظل متاحة خارج تلك الكتلة.

:::interactive_editor

```js
if (true) {
  var num = 5;
}
console.log(num); // 5
```

:::

- **الرفع (Hoisting)**: هو سلوك JavaScript الافتراضي الذي ينقل الإعلانات إلى أعلى نطاقاتها أثناء مرحلة الترجمة قبل تنفيذ الشفرة. عند إعلان متغير باستخدام الكلمة المفتاحية `var`، يرفع JavaScript الإعلان إلى أعلى نطاقه.

:::interactive_editor

```js
console.log(num); // undefined
var num = 5;
console.log(num); // 5
```

:::

عندما تعلن دالة باستخدام صيغة إعلان الدالة، يُرفع كل من اسم الدالة وجسمها. هذا يعني أنه يمكنك استدعاء الدالة قبل إعلانها في شفرتك.

:::interactive_editor

```js
sayHello(); // "Hello, World!"

function sayHello() {
  console.log("Hello, World!");
}
```

:::

إعلانات المتغيرات التي تتم باستخدام `let` أو `const` تُرفع، لكنها غير مهيأة، ولا يمكنك الوصول إليها قبل الإعلان الفعلي في شفرتك. يُشار إلى هذا السلوك غالبًا باسم "منطقة الموت الزمنية".

:::interactive_editor

```js
console.log(num); // Throws a ReferenceError
let num = 10;
```

:::

## العمل مع الاستيراد، التصدير والوحدات

- **الوحدة**: هي وحدة شفرة مستقلة تغلف دوالًا، أصنافًا، أو متغيرات ذات صلة. لإنشاء وحدة، تكتب شفرة JavaScript في ملف منفصل.
- **التصدير**: أي متغيرات، دوال، أو أصناف تريد إتاحتها لأجزاء أخرى من التطبيق يجب تصديرها صراحة باستخدام الكلمة المفتاحية `export`. هناك نوعان من التصدير: التصدير المسمى والتصدير الافتراضي.
- **الاستيراد**: لاستخدام العناصر المصدرة في جزء آخر من التطبيق، تحتاج إلى استيرادها باستخدام الكلمة المفتاحية `import`. الأنواع تشمل الاستيراد المسمى، الاستيراد الافتراضي، واستيراد النطاق.

```js
// Within a file called math.js, we export the following functions:

// Named export
export function add(num1, num2) {
  return num1 + num2;
}

// Default export
export default function subtract(num1, num2) {
  return num1 - num2;
}

// Within another file, we can import the functions from math.js.

// Named import - This line imports the add function.
// The name of the function must exactly match the one exported from math.js.
import { add } from './math.js';

// Default import - This line imports the subtract function.
// The name of the function can be anything.
import subtractFunc from './math.js';

// Namespace import - This line imports everything from the file.
import * as Math from './math.js';

console.log(add(5, 3)); // 8
console.log(subtractFunc(5, 3)); // 2
console.log(Math.add(5, 3)); // 8
console.log(Math.subtract(5, 3)); // 2
```
