## Регулярні вирази та загальні методи

- **Визначення**: Регулярні вирази, або regex, використовуються для створення "шаблону", який ви можете застосувати для перевірки рядка, вилучення тексту та іншого.

```js
const regex = /freeCodeCamp/;
```

- **Метод `test()`**: Цей метод приймає рядок, який потрібно перевірити на відповідність регулярному виразу. Метод поверне булеве значення, якщо рядок відповідає regex.

:::interactive_editor

```js
const regex = /freeCodeCamp/;
const test = regex.test("e");
console.log(test); // false
```

:::

- **Метод `match()`**: Цей метод приймає регулярний вираз, хоча ви також можете передати рядок, який буде перетворений у регулярний вираз. Метод `match` повертає масив збігів для рядка.

```js
const regex = /freeCodeCamp/;
const match = "freeCodeCamp".match(regex);
console.log(match); // ['freeCodeCamp', index: 0, input: 'freeCodeCamp', groups: undefined]
```

- **Метод `replace()`**: Цей метод приймає два аргументи: регулярний вираз для пошуку (або рядок) і рядок для заміни збігу (або функцію, яка виконується для кожного збігу).

:::interactive_editor

```js
const regex = /Jessica/;
const str = "Jessica is rly kewl";
const replaced = str.replace(regex, "freeCodeCamp");
console.log(replaced); // "freeCodeCamp is rly kewl"
```

:::

- **Метод `replaceAll`**: Цей метод використовується для заміни всіх входжень заданого шаблону на новий рядок. Він викличе помилку, якщо передати регулярний вираз без глобального модифікатора.

:::interactive_editor

```js
const text = "I hate JavaScript! I hate programming!";
const newText = text.replaceAll("hate", "love");
console.log(newText);  // "I love JavaScript! I love programming!"
```

:::

- **Метод `matchAll`**: Цей метод використовується для отримання всіх збігів заданого регулярного виразу в рядку, включно з групами захоплення, і повертає їх у вигляді ітератора. Ітератор — це об’єкт, який дозволяє пройтися (або "ітерувати над") колекцією елементів.

:::interactive_editor

```js
const str = "JavaScript, Python, JavaScript, Swift, JavaScript";
const regex = /JavaScript/g;

const iterator = str.matchAll(regex);

for (let match of iterator) {
  console.log(match[0]); // "JavaScript" for each match
}
```

:::

## Модифікатори регулярних виразів

- **Визначення**: Модифікатори, часто звані "прапорцями", змінюють поведінку регулярного виразу.
- **Прапорець `i`**: Цей прапорець змушує regex ігнорувати регістр.

:::interactive_editor

```js
const regex = /freeCodeCamp/i;
console.log(regex.test("freecodecamp")); // true
console.log(regex.test("FREECODECAMP")); // true
```

:::

- **Прапорець `g`**: Цей прапорець, або глобальний модифікатор, дозволяє регулярному виразу знаходити шаблон більше одного разу.

:::interactive_editor

```js
const regex = /freeCodeCamp/gi;
console.log(regex.test("freeCodeCamp")); // true
console.log(regex.test("freeCodeCamp is great")); // false
```

:::

- **Визначення якорів**: Якір `^` на початку регулярного виразу означає "збіг з початком рядка". Якір `$` в кінці регулярного виразу означає "збіг з кінцем рядка".

:::interactive_editor

```js
const start = /^freeCodeCamp/i;
const end = /freeCodeCamp$/i;
console.log(start.test("freecodecamp")); // true
console.log(end.test("freecodecamp")); // true
```

:::

- **Прапорець `m`**: Якорі шукають початок і кінець усього рядка. Але ви можете змусити regex працювати з кількома рядками за допомогою прапорця `m`, або мульти-рядкового модифікатора.

:::interactive_editor

```js
const start = /^freecodecamp/im;
const end = /freecodecamp$/im;
const str = `I love 
freecodecamp
it's my favorite
`;
console.log(start.test(str)); // true
console.log(end.test(str)); // true
```

:::

- **Прапорець `d`**: Цей прапорець розширює інформацію, яку ви отримуєте в об’єкті збігу.

```js
const regex = /freecodecamp/di;
const string = "we love freecodecamp isn't freecodecamp great?";
console.log(string.match(regex));
```

І результат:

```js
// [
//   'freecodecamp',
//   index: 8,
//   input: "we love freecodecamp isn't freecodecamp great?",
//   groups: undefined,
//   indices: [
//     0: [8, 20],
//     groups: undefined
//   ]
// ]
```

- **Прапорець `u`**: Цей прапорець розширює функціональність регулярного виразу, дозволяючи йому знаходити спеціальні символи Unicode. Прапорець `u` дає доступ до спеціальних класів, таких як `Extended_Pictographic`, для пошуку більшості емодзі. Існує також прапорець `v`, який ще більше розширює можливості пошуку Unicode.
- **Прапорець `y`**: Модифікатор sticky працює дуже схоже на глобальний, але з кількома винятками. Найголовніший — глобальний регулярний вираз починає пошук з lastIndex і шукає по всій решті рядка, а sticky повертає null і скидає lastIndex у 0, якщо немає збігу саме на lastIndex.
- **Прапорець `s`**: Модифікатор single-line дозволяє символу підстановки, який у regex позначається `.`, відповідати також і розривам рядків — фактично розглядаючи рядок як один рядок тексту.

## Класи символів

- **Універсальний `.`**: Класи символів — це спеціальний синтаксис, який ви можете використовувати для пошуку множин або підмножин символів. Перший клас символів, який варто вивчити, — це універсальний клас. Універсальний клас позначається крапкою і відповідає будь-якому одному символу, крім розривів рядків. Щоб дозволити універсальному класу відповідати розривам рядків, пам’ятайте, що потрібно використовувати прапорець `s`.

```js
const regex = /a./;
```

- **`\d`**: Це відповідатиме всім цифрам (`0-9`) у рядку.

```js
const regex = /\d/;
```

- **`\w`**: Використовується для пошуку будь-якого символу слова (`a-z0-9_`) у рядку. Символ слова визначається як будь-яка літера від a до z, або число від 0 до 9, або символ підкреслення.

```js
const regex = /\w/;
```

- **`\s`**: Клас порожнього простору `\s`, який позначається зворотною скісною рискою та `s`. Цей клас символів відповідає будь-якому порожньому простору, включно з новими рядками, пробілами, табуляціями та спеціальними символами Unicode для пробілів.
- **Заперечення спеціальних класів символів**: Щоб заперечити один із цих класів символів, замість використання літери нижнього регістру після зворотної скісної риски, можна використати відповідний символ верхнього регістру. Наступний приклад не відповідає числовому символу. Замість цього він відповідає будь-якому одному символу, який НЕ є числовим.

```js
const regex = /\D/;
```

- **Кастомні класи символів**: Ви можете створювати кастомні класи символів, розміщуючи символи, які хочете знайти, всередині квадратних дужок.

```js
const regex = /[abcdf]/;
```

## Твердження випередження та відставання

- **Визначення**: Твердження випередження та відставання дозволяють знаходити конкретні шаблони залежно від наявності або відсутності навколишніх шаблонів.
- **Позитивне твердження випередження**: Це твердження знайде шаблон, якщо він слідує за іншим шаблоном. Щоб створити позитивне твердження випередження, почніть зі шаблону, який хочете знайти. Потім використайте круглі дужки, щоб обгорнути шаблон, який буде умовою. Після відкриваючої дужки використайте `?=`, щоб визначити цей шаблон як позитивне твердження випередження.

```js
const regex = /free(?=code)/i;
```

- **Негативне твердження випередження**: Це тип умови в регулярних виразах, який перевіряє, що певний шаблон не зустрічається попереду в рядку.

```js
const regex = /free(?!code)/i;
```

- **Позитивне твердження відставання**: Це твердження знайде шаблон лише якщо він передує іншому конкретному шаблону, не включаючи цей попередній шаблон у збіг.

```js
const regex = /(?<=free)code/i;
```

- **Негативне твердження відставання**: Це твердження гарантує, що шаблон не передує іншому конкретному шаблону. Воно відповідає лише якщо вказаний шаблон не йде безпосередньо після заданої послідовності, не включаючи цю послідовність у збіг.

```js
const regex = /(?<!free)code/i;
```

## Квантори регулярних виразів

- **Визначення**: Квантори в регулярних виразах визначають, скільки разів шаблон (або його частина) має з’явитися. Вони допомагають контролювати кількість входжень символів або груп у збігу. Наступний приклад використовується для пошуку попереднього символу рівно чотири рази.

```js
const regex = /^\d{4}$/;
```

- **`*`**: Відповідає 0 або більше входжень попереднього елемента.
- **`+`**: Відповідає 1 або більше входжень попереднього елемента.
- **`?`**: Відповідає 0 або 1 входженню попереднього елемента.
- **`{n}`**: Відповідає рівно n входженням попереднього елемента.
- **`{n,}`**: Відповідає n або більше входженням попереднього елемента.
- **`{n,m}`**: Відповідає від n до m входжень попереднього елемента.

## Групи захоплення та зворотні посилання

- **Групи захоплення**: Група захоплення дозволяє "захопити" частину знайденого рядка для подальшого використання. Групи захоплення визначаються круглими дужками, які містять шаблон для захоплення, без провідних символів, як у твердженнях випередження.

```js
const regex = /free(code)camp/i;
```

- **Некатчинг-групи**: Некатчинг-група схожа на групу захоплення, але не зберігає знайдену частину рядка для подальшого використання. Некатчинг-групи визначаються `(?:...)`.

```js
const regex = /free(?:code)camp/i;
```

- **Зворотні посилання**: Зворотне посилання в регулярних виразах — це спосіб повторно використати частину шаблону, яка була знайдена раніше в тому ж виразі. Воно дозволяє посилатися на захоплену групу (частину шаблону в дужках) за її номером. Наприклад, `$1` посилається на першу захоплену групу.

:::interactive_editor

```js
const regex = /free(co+de)camp/i;
console.log("freecoooooooodecamp".replace(regex, "paid$1world"));
```

:::

- Ви можете використовувати зворотні посилання всередині самого regex, щоб знайти той самий текст, захоплений попередньою групою, за допомогою зворотної скісної риски та номера групи. Наприклад:

:::interactive_editor

```js
const regex = /(hello) \1/i;
console.log(regex.test("hello hello")); // true
console.log(regex.test("hello world")); // false
```

:::
