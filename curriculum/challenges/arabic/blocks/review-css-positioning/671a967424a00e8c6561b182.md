# --interactive--

## العمل مع العناصر العائمة

- **التعريف**: تُستخدم العناصر العائمة لإخراج عنصر من تدفقه الطبيعي في الصفحة وتحديد موضعه إما على الجانب الأيسر أو الأيمن من الحاوية الخاصة به. عند حدوث ذلك، يلتف النص حول هذا المحتوى العائم.

```css
float: left;
float: right;
```

- **مسح العناصر العائمة**: تُستخدم خاصية `clear` لتحديد ما إذا كان يجب نقل عنصر إلى أسفل المحتوى العائم. عند وجود عدة عناصر عائمة متراصة بجانب بعضها، قد تحدث مشاكل في التداخل والانهيار في التخطيطات. لذلك تم إنشاء حيلة `clearfix` لحل هذه المشكلة.

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<div class="clearfix">
  <div class="box left">Left</div>
  <div class="box right">Right</div>
</div>
```

```css
.box {
  width: 100px;
  height: 60px;
  color: white;
  text-align: center;
  line-height: 60px;
}

.left {
  float: left;
  background: teal;
}

.right {
  float: right;
  background: purple;
}

.clearfix::after {
  content: "";
  display: block;
  clear: both;
}
```

:::

## التحديد الثابت، النسبي والمطلق

- **التحديد الثابت**: هذا هو التدفق الطبيعي للوثيقة. تُحدد مواضع العناصر من الأعلى إلى الأسفل ومن اليسار إلى اليمين واحدًا تلو الآخر.
- **التحديد النسبي**: يسمح لك باستخدام خصائص `top` و`left` و`right` و`bottom` لتحديد موضع العنصر ضمن التدفق الطبيعي للوثيقة. يمكنك أيضًا استخدام التحديد النسبي لجعل العناصر تتداخل مع عناصر أخرى في الصفحة.

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<div class="relative">Relative Element</div>
```

```css
.relative {
    position: relative;
    top: 30px;
    left: 30px;
}
```

:::

- **التحديد المطلق**: يسمح لك بإخراج عنصر من التدفق الطبيعي للوثيقة، مما يجعله يتصرف بشكل مستقل عن العناصر الأخرى.

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<div class="parent">
  Parent
  <div class="positioned">Absolute</div>
</div>
```

```css
.parent {
  position: relative;
  height: 120px;
  background: #eee;
}

.positioned {
  position: absolute;
  top: 30px;
  left: 30px;
  background-color: coral;
}
```

:::

## التحديد الثابت والتحديد اللاصق

- **التحديد الثابت**: عندما يُحدد عنصر باستخدام `position: fixed`، يُزال من التدفق الطبيعي للوثيقة ويُوضع بالنسبة إلى إطار العرض، مما يعني أنه يبقى في نفس الموضع حتى عند تمرير المستخدم للصفحة. يُستخدم هذا غالبًا لعناصر مثل الأقسام العلوية أو أشرطة التنقل التي يجب أن تبقى مرئية دائمًا.

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<div class="navbar">Fixed Navbar</div>
<div style="height: 600px;"></div>
```

```css
.navbar {
  position: fixed; 
  top: 0; 
  width: 100%;
  background: #0077ff;
  color: white;
  padding: 8px; 
}
```

:::

- **التحديد اللاصق**: هذا النوع من التحديد يعمل كخليط بين التحديد النسبي والثابت. في البداية، يتصرف العنصر كما لو أنه محدد نسبيًا، ويبقى ضمن تدفق الوثيقة. لكن بمجرد أن يمرر المستخدم العنصر إلى نقطة معينة، "يلتصق" بإطار العرض (عادة في الأعلى) ويتصرف كما لو أنه ثابت. هذا مفيد لإنشاء عناصر مثل أشرطة التنقل اللاصقة التي تصبح ثابتة فقط عند تمرير المستخدم إلى موضع معين.

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<p>Scroll down</p>
<div class="positioned">Sticky Element</div>
<div style="height: 500px;"></div>
```

```css
.positioned {
  position: sticky;
  top: 30px;
  left: 30px;
  background: #4caf50;
  color: white;
  padding: 6px;
}
```

:::

## العمل مع خاصية `z-index`

- **التعريف**: تُستخدم خاصية `z-index` في CSS للتحكم في ترتيب التكديس العمودي للعناصر المحددة الموضع التي تتداخل على الصفحة.

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<div class="container">
  <div class="box box1">1</div>
  <div class="box box2">2</div>
</div>
```

```css
.container {
  position: relative;
  width: 300px;
  height: 300px;
  border: 1px solid black;
}

.box {
  position: absolute;
  width: 100px;
  height: 100px;
  color: white;
  display: flex;
  align-items: center;
  justify-content: center;
}

.box1 {
  background: lightcoral;
  top: 20px;
  left: 20px;
  z-index: 1;
}

.box2 {
  background: steelblue;
  top: 40px;
  left: 40px;
  z-index: 2;
}
```
