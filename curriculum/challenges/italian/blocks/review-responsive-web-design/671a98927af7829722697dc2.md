# --interactive--

## Design web responsivo

- **Definizione**: Il principio fondamentale del design responsivo è l’adattabilità – la capacità di un sito web di modificare il layout e il contenuto in base alla dimensione dello schermo e alle capacità del dispositivo su cui viene visualizzato.
- **Griglie fluide**: Usano unità relative come percentuali invece di unità fisse come i pixel, permettendo al contenuto di ridimensionarsi e riorganizzarsi in base alla dimensione dello schermo.
- **Immagini flessibili**: Sono impostate per ridimensionarsi all’interno dei loro contenitori, assicurando che non escano dai contenitori su schermi più piccoli.

## Media query

- **Definizione**: Permette agli sviluppatori di applicare stili diversi in base alle caratteristiche del dispositivo, principalmente alla larghezza del viewport.

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<p class="mq-example">The background of this paragraph will change when the viewport is at least 768px wide.</p>
```

```css
.mq-example {
  padding: 10px;
  border: 1px solid #ccc;
  background-color: lightyellow;
}

@media screen and (min-width: 768px) {
  /* Styles for screens at least 768px wide */
  .mq-example {
    background-color: lightblue;
  }
}
```

:::

- **`all` tipo di media**: È adatto a tutti i dispositivi. Questo è il predefinito se non viene specificato alcun tipo di media.
- **`print` tipi di media**: È destinato a materiale impaginato e documenti visualizzati su schermo in modalità anteprima di stampa.
- **`screen` tipi di media**: È pensato principalmente per gli schermi.
- **`aspect-ratio`**: Descrive il rapporto tra la larghezza e l’altezza del viewport.

```css
@media screen and (aspect-ratio: 16/9) {
  /* Styles for screens with a 16:9 aspect ratio */
}
```

- **`orientation`**: Serve a indicare se il dispositivo è in orientamento orizzontale o verticale.

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<p class="orientation-example">This paragraph's background will change in landscape orientation.</p>
```

```css
.orientation-example {
  padding: 10px;
  border: 1px solid #ccc;
  background-color: lightyellow;
}

@media screen and (orientation: landscape) {
  /* Styles for landscape orientation */
  .orientation-example {
    background-color: lightgreen;
  }
}
```

:::

- **`resolution`**: Serve a descrivere la risoluzione del dispositivo di output in punti per pollice (dpi) o punti per centimetro (dpcm).

```css
@media screen and (min-resolution: 300dpi) {
  /* Styles for high-resolution screens */
}
```

- **`hover`**: Serve a verificare se il meccanismo di input principale può passare sopra gli elementi (hover).

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<button>Hover over me</button>
```

```css
button {
  padding: 10px 20px;
  font-size: 1rem;
  background-color: lightgray;
  border: 1px solid #999;
  cursor: pointer;
}

@media (hover: hover) {
  /* Styles for devices that support hover */
  button:hover {
    background-color: yellow;
  }
}
```

:::

- **`prefers-color-scheme`**: Serve a rilevare se l’utente ha richiesto un tema chiaro o scuro.
- **Media query e operatori logici**: L’operatore `and` si usa per combinare più funzionalità media, mentre `not` e `only` possono essere usati per negare o isolare media query.

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<p class="logical-example">This paragraph's background will change when the screen is at least 768px wide AND in landscape orientation.</p>
```

```css
.logical-example {
  padding: 10px;
  border: 1px solid #ccc;
  background-color: lightyellow;
}

@media screen and (min-width: 768px) and (orientation: landscape) {
  /* Styles for landscape screens at least 768px wide */
  .logical-example {
    background-color: lightcoral;
  }
}
```

:::

## Breakpoint comuni per i media

- **Definizione**: I breakpoint media sono punti specifici nel design di un sito web in cui il layout e il contenuto si adattano per accogliere diverse dimensioni dello schermo. Ci sono alcuni breakpoint generali che puoi usare per indirizzare telefoni, tablet e schermi desktop. Ma non è saggio cercare di coprire ogni singola possibile dimensione dello schermo per dispositivi diversi.

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<p class="breakpoint-example">This text will change size when the viewport width is at least 768px.</p>
```

```css
.breakpoint-example {
  font-size: 1rem;
  padding: 10px;
  border: 1px solid #ccc;
}

/* Styles for screens wider than 768px */
@media screen and (min-width: 768px) {
  .breakpoint-example {
    font-size: 1.125rem;
    background-color: lightblue;
  }
}
```

:::

- **Dispositivi piccoli (smartphone)**: fino a 640px
- **Dispositivi medi (tablet)**: da 641px a 1024px
- **Dispositivi grandi (desktop)**: 1025px e oltre

## Approccio mobile first

- **Definizione**: L’approccio mobile first è una filosofia di design e una strategia di sviluppo nel design web responsivo che dà priorità alla creazione di siti per dispositivi mobili prima di progettare per schermi più grandi.

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<div class="container">
  <p>The width of this container changes based on the viewport size.</p>
</div>
```

```css
/* Base styles for mobile */
.container {
  width: 100%;
  padding: 10px;
  background-color: lightblue;
}

/* Styles for larger screens */
@media screen and (min-width: 768px) {
  .container {
    width: 750px;
    margin: 0 auto;
    padding: 20px;
  }
}

@media screen and (min-width: 1024px) {
  .container {
    width: 960px;
  }
}
```
