# --interactive--

Інструкції `switch` і ланцюжки `if/else if/else` — це обидві структури керування потоком у програмуванні, які дозволяють виконувати різні блоки коду залежно від певних умов. Однак вони мають різні характеристики та сфери застосування.

Інструкція `switch` оцінює вираз і порівнює його значення з низкою умов `case`. Коли знаходиться відповідність, виконується блок коду, пов’язаний із цією умовою `case`. Ось базова структура інструкції `switch`:

```js
switch (expression) {
  case value1:
    // code to be executed if expression === value1
    break;
  case value2:
    // code to be executed if expression === value2
    break;
  default:
    // code to be executed if expression doesn't match any case
}
```

Інструкція `break` наприкінці кожної умови `case` є дуже важливою. Вона наказує програмі вийти з блоку switch після виконання відповідної умови `case`. Без неї програма продовжила б виконання наступних випадків, що називається «провалюванням» (fall-through).

Інструкції `switch` зазвичай використовують, коли порівнюють одну змінну з кількома можливими значеннями. Вони особливо корисні, коли потрібно перевірити багато потенційних умов для однієї змінної. Ось приклад використання інструкції `switch` для днів тижня:

:::interactive_editor

```js
let dayOfWeek = 3; 

switch (dayOfWeek) {
    case 1:
        console.log("It's Monday! Time to start the week strong.");
        break;
    case 2:
        console.log("It's Tuesday! Keep the momentum going.");
        break;
    case 3:
        console.log("It's Wednesday! We're halfway there.");
        break;
    case 4:
        console.log("It's Thursday! Almost the weekend.");
        break;
    case 5:
        console.log("It's Friday! The weekend is near.");
        break;
    case 6:
        console.log("It's Saturday! Enjoy your weekend.");
        break;
    case 7:
        console.log("It's Sunday! Rest and recharge.");
        break;
    default:
        console.log("Invalid day! Please enter a number between 1 and 7.");
}
```

:::

Інструкції `switch` можуть бути більш читабельними та лаконічними при роботі з багатьма можливими значеннями для однієї змінної.

Інструкції `if/else if`, навпаки, є більш гнучкими. Вони можуть оцінювати складні умови та різні змінні в кожній умові. Це робить їх придатними для ширшого кола сценаріїв. Ось приклад, коли варто використати інструкцію `if/else` замість інструкції `switch`:

:::interactive_editor

```js
let creditScore = 720; 
let annualIncome = 60000; 
let loanAmount = 200000; 

let eligibilityStatus;

if (creditScore >= 750 && annualIncome >= 80000) {
    eligibilityStatus = "Eligible for premium loan rates.";
} else if (creditScore >= 700 && annualIncome >= 50000) {
    eligibilityStatus = "Eligible for standard loan rates.";
} else if (creditScore >= 650 && annualIncome >= 40000) {
    eligibilityStatus = "Eligible for subprime loan rates.";
} else if (creditScore < 650) {
    eligibilityStatus = "Not eligible due to low credit score.";
} else {
    eligibilityStatus = "Not eligible due to insufficient income.";
}

console.log(eligibilityStatus);
```

:::

У цьому прикладі ми маємо річний дохід людини та кредитний рейтинг і перевіряємо, на які типи позик вона може претендувати. Оскільки ми маємо справу зі складнішими логічними оцінками та кількома змінними, тут краще використати інструкцію `if/else`, а не `switch`.

Варто зазначити, що інструкції `switch` у JavaScript використовують суворе порівняння (`===`), тобто вони не виконують перетворення типів. Це може бути перевагою з точки зору передбачуваності та уникнення тонких помилок.

Підсумовуючи, хоча і інструкції `switch`, і ланцюжки `if/else if` дозволяють реалізовувати багатогілкову логіку у вашому коді, вони мають різні сильні сторони. Інструкції `switch` чудово підходять для обробки багатьох можливих значень однієї змінної, тоді як ланцюжки `if/else if` пропонують більшу гнучкість для складних умов. Вибір між ними часто залежить від конкретних вимог вашого коду та особистих або командних уподобань у стилі кодування.
