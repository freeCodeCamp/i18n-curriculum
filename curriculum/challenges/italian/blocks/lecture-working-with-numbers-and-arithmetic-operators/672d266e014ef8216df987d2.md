# --interactive--

Il tipo `Number` è uno dei tipi di dato più usati in JavaScript e in altri linguaggi di programmazione. I numeri possono sembrare semplici, ma c’è molto da scoprire quando si tratta di numeri in JavaScript. Quindi, diamo un’occhiata più approfondita. In JavaScript, il tipo di dato `Number` rappresenta un valore numerico.

A differenza di molti altri linguaggi di programmazione che separano numeri interi e numeri a virgola mobile in tipi diversi, JavaScript usa un unico tipo `Number` per rappresentare i numeri. Questo significa che puoi lavorare con numeri interi, decimali e persino valori numerici speciali tutti sotto lo stesso tipo di dato `Number`.

Ecco un esempio base che mostra come numeri interi, numeri a virgola mobile e numeri negativi siano tutti di tipo number:

:::interactive_editor

```js
const wholeNumber = 50;
const decimalNumber = 4.5;
const negativeNumber = -7;

console.log(typeof wholeNumber); // number
console.log(typeof decimalNumber); // number
console.log(typeof negativeNumber); // number
```

:::

Il tipo `Number` di JavaScript include diversi tipi di valori numerici, che vanno da semplici numeri interi e numeri a virgola mobile a casi speciali come `Infinity` e `NaN`, o "Not a Number". Vediamo i principali tipi che incontrerai. I numeri interi sono numeri interi senza parte frazionaria o decimale. Possono essere positivi, negativi o zero. Ecco alcuni esempi:

:::interactive_editor

```js
const positiveInteger = 100;
const negativeInteger = -25;
const zero = 0;

console.log(typeof positiveInteger); // number
console.log(typeof negativeInteger); // number
console.log(typeof zero); // number
```

:::

I numeri a virgola mobile sono numeri con la virgola decimale. Gli sviluppatori JavaScript li chiamano spesso semplicemente "float". I float sono utili quando serve più precisione, ad esempio quando si trattano misurazioni o valute. Ecco alcuni esempi:

:::interactive_editor

```js
const floatingPointNumber = 4.5;
const anotherFloat = 89.56;
const oneMoreFloat = 16.462;

console.log(typeof floatingPointNumber); // number
console.log(typeof anotherFloat); // number
console.log(typeof oneMoreFloat); // number
```

:::

JavaScript può rappresentare numeri che superano il limite massimo con `Infinity`. Lo incontrerai quando provi a dividere un numero per zero o in rari casi, superi il limite superiore del tipo `Number`. Ecco un esempio:

:::interactive_editor

```js
const infiniteNumber = 1 / 0;
console.log(infiniteNumber); // Infinity
console.log(typeof infiniteNumber); // number
```

:::

A volte in JavaScript alcune operazioni matematiche non producono un numero valido. Per esempio, se provi a eseguire un’operazione matematica su qualcosa che non è un numero, otterrai `NaN`, che sta per "Not a Number":

:::interactive_editor

```js
const notANumber = 'hello world' / 2;
console.log(notANumber); // NaN
```

:::

Sorprendentemente, il tipo di `NaN` è anch’esso `Number`:

:::interactive_editor

```js
const notANumber = 'hello world' / 2;
console.log(typeof notANumber); // number
```

:::

Oltre al sistema decimale standard (base 10), JavaScript supporta anche numeri in basi diverse come binario, ottale ed esadecimale. Il binario è un sistema in base 2 che usa solo le cifre 1 e 0. L’ottale è un sistema in base 8 che usa solo le cifre da 0 a 7. L’esadecimale è un sistema in base 16 che usa le cifre da 0 a 9 e le lettere da a a f, come vedi nei colori esadecimali CSS.
