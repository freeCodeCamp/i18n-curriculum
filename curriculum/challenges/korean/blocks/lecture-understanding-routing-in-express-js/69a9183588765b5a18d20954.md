# --description--

Express에서 `res` (응답) 객체는 들어오는 요청을 처리한 후 클라이언트에 응답을 보내는 데 사용됩니다. Express는 다양한 유형의 응답을 보내기 위한 여러 내장 메서드를 제공합니다. 예를 들어, 일반 텍스트, JSON 데이터 전송, 사용자를 다른 URL로 리다이렉트하거나 템플릿을 렌더링할 수 있습니다.

가장 자주 사용되는 응답 메서드 중 하나는 `res.send()`입니다:

```javascript
app.get("/text", (req, res) => {
  res.send("Hello, World!");
});
```

`res.send()` 메서드는 클라이언트에 응답을 보냅니다. 일반 텍스트, HTML, JSON 또는 바이너리 데이터도 보낼 수 있습니다. 이 예제에서 클라이언트가 `/text`에 방문하면 서버는 `Hello, World!`라는 텍스트로 응답합니다.

또 다른 자주 사용되는 응답 메서드는 `res.json()`입니다.

이 메서드는 클라이언트에 JSON 응답을 보냅니다. API를 구축할 때 특히 유용하며, 객체나 배열 같은 구조화된 데이터를 반환해야 할 때 자주 사용합니다. Express는 JavaScript 객체를 자동으로 JSON 형식 문자열로 변환합니다:

```javascript
app.get("/user", (req, res) => {
  res.json({ name: "John Doe", age: 30 });
});
```

클라이언트가 `/user`에 요청을 보내면 서버는 다음과 같이 응답합니다:

```json
{
  "name": "John Doe",
  "age": 30
}
```

응답의 HTTP 상태 코드를 제어해야 할 때는 `res.status()`를 사용할 수 있습니다. 이 메서드는 상태 코드를 설정하며 보통 `res.send()` 또는 `res.json()` 같은 다른 응답 메서드와 체인으로 사용됩니다:

```javascript
app.get("/notfound", (req, res) => {
  res.status(404).send("Not Found");
});
```

여기서 서버는 상태 코드를 `404`으로 설정하고 `Not Found` 메시지를 클라이언트에 보냅니다.

사용자를 다른 URL로 보내고 싶다면 `res.redirect()`를 사용할 수 있습니다:

```javascript
app.get("/oldpage", (req, res) => {
  res.redirect("/newpage");
});
```

클라이언트가 `/oldpage`에 방문하면 자동으로 `/newpage`로 리다이렉트됩니다.

마지막으로, `res.render()`는 Pug, EJS, Handlebars 같은 템플릿 엔진을 사용할 때 쓰입니다. 뷰 템플릿을 렌더링하고 결과 HTML을 클라이언트에 보냅니다:

```javascript
app.get("/profile", (req, res) => {
  res.render("profile", { username: "john_doe" });
});
```

이 예제에서 서버는 `profile` 뷰를 렌더링하고 `username` 변수를 전달하여 HTML에 표시할 수 있습니다.

이러한 응답 메서드를 통해 요청의 성격에 따라 다양한 유형의 데이터를 반환할 수 있습니다. 텍스트, JSON 데이터 전송 또는 클라이언트 리다이렉트 등 어떤 경우든 Express는 유연한 도구 세트를 제공합니다.
