# --description--

在之前的課程中，我們學習了如何在 React 中建置像這樣的小型組件：

```jsx
function Greeting() {
  const developerName = "Jessica";
  return <h1>Hi {developerName}!</h1>;
}
```

我們可以選擇將此組件巢狀在另一個父組件或根組件內，如下所示：

```jsx
function App() {
  return <Greeting />;
}

function Greeting() {
  const developerName = "Jessica";
  return <h1>Hi {developerName}!</h1>;
}
```

雖然這段程式碼可以正常執行並在螢幕上顯示 `Hi Jessica!` 的結果，但它並不是那麼靈活的組件。 

如果我們想顯示不同的名字，例如 Naomi、Tom 或 Oliver，該怎麼辦？這就是 React props 派上用場的地方。Props 是 properties 的縮寫，是父組件將資料傳遞給子組件的方式。Props 可以是任何型別：字串、數字、布林值、物件或陣列。

讓我們更新之前的範例，現在接受一個 `name` prop：

```jsx
function App() {
  return <Greeting name="Jessica" />;
}

export default App;

function Greeting(props) {
  console.log(props);
  return <h1>Hi {props.name}!</h1>;
}
```

對於名為 `Greeting` 的子組件，我們現在使用 `props.name`，而不是硬編碼名稱 `"Jessica"`。我們也將 `props` 記錄到主控台，顯示為一個物件。

然後，在父層的 `App` 組件內，我們將值傳遞給 `name` 屬性，以便它可以傳遞給子層。結果在螢幕上將與之前相同，但現在我們已經創建了一個更靈活的組件。

現在我們可以重複使用子組件多次，並且每次傳入不同的名稱：

```jsx
function App() {
  return (
    <>
      <Greeting name="Naomi" />
      <Greeting name="Tom" />
      <Greeting name="Jessica" />
      <Greeting name="Oliver" />
    </>
  );
}
```

你也可以選擇在 props 中使用物件解構，使其更易讀。以下是你如何重寫 `Greeting` 組件：

```jsx
function Greeting({ name }) {
  return <h1>Hi {name}!</h1>;
}
```

這段程式碼達成相同的結果，但更清楚組件預期接收哪些 props。

React 也提供一個特殊的屬性叫做 `children`。你放在組件開啟和關閉頁籤之間的任何 JSX 都會作為 `children` 傳遞給該組件。

例如：

```jsx
function Card({ children }) {
  return <div className="card">{children}</div>;
}

function App() {
  return (
    <Card>
      <h2>Hello</h2>
    </Card>
  );
}
```

在這種情況下，`<Card>...</Card>` 內的 JSX 會成為 `children` 屬性。DOM 中的呈現輸出將會是：

```html
<div class="card">
  <h2>Hello</h2>
</div>
```

這個樣式經常用於組件複合，其中一個組件包裹其他 UI 元素。

有時候，你可能需要傳遞很多屬性作為 props。與其一一傳遞，你可以先將它們轉換成物件，然後使用展開運算子（`...`）。 

這是一個名為 `DeveloperCard` 的新子組件範例：

```jsx
function DeveloperCard({ name, age, country }) {
  return (
    <div className="developer-card">
      <h1>Developer: {name}</h1>
      <p>Age: {age}</p>
      <p>Country: {country}</p>
    </div>
  );
}
```

這個 `DeveloperCard` 組件接受三個 props：`name`、`age` 和 `country`。 

在父層的 `App` 組件中，我們可以使用展開語法將物件中的所有屬性作為獨立的 props 傳遞給子組件：

```jsx
function App() {
  const developerObj = {
    name: "Alice",
    age: 30,
    country: "USA",
  };

  return (
    <div className="App">
      <DeveloperCard {...developerObj} />
    </div>
  );
}
```

當處理物件陣列並將多組屬性傳遞給子組件時，這特別有用。例如，你可能有一個開發者列表，其中陣列中的每個物件具有相同的結構，但表現不同的人。

你將在未來的課程中學習更多關於如何在陣列中呈現列表。

在 React 中使用 props 使你的組件更具彈性且可重複使用，讓你能建置更複雜的使用者介面。然而，重要的是要注意 props 是不可變的，這表示一旦傳遞給組件就無法更改。如果你需要控制代碼使用者輸入並修改資料，應該改用 state。你將在未來的課程中學習更多關於管理 state 的內容。
