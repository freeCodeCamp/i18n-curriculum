# --interactive--

Алгоритм каскаду — це процес, який браузер використовує, щоб вирішити, які CSS-правила застосувати, коли кілька стилів націлені на один і той самий елемент. Він гарантує, що будуть використані найбільш відповідні стилі на основі набору чітко визначених правил.

Процес починається з **релевантності**. Спочатку браузер фільтрує всі CSS-правила, щоб знайти ті, які дійсно застосовуються до конкретного елемента. Це включає відповідність селекторів і врахування медіазапитів, які можуть бути активними.

Медіазапит — це CSS-техніка, яка використовується для застосування стилів залежно від характеристик пристрою або вікна перегляду, таких як ширина, висота або орієнтація.

Далі алгоритм враховує **походження та важливість**. CSS може надходити з різних джерел: стилі за замовчуванням браузера (user-agent), стилі, встановлені користувачем, та стилі, написані автором (вами).

Після врахування походження алгоритм оцінює важливість кожного правила, надаючи пріоритет правилам, позначеним `!important`, які заміщають інші правила незалежно від їхнього джерела.

Після фільтрації за походженням і важливістю алгоритм розглядає **специфічність**. Коли застосовуються два правила з однаковим походженням і рівнем важливості, застосовується правило з вищою специфічністю.

Специфічність — це міра того, наскільки точно націлений селектор, причому більш специфічні селектори мають пріоритет над загальнішими.

Нарешті, якщо все інше рівне, враховується **порядок появи**. Коли два правила мають однакову специфічність, застосовується те, що з’являється останнім у CSS.

Ось чому порядок, у якому ви пишете свої стилі, іноді може впливати на результат.

Давайте розглянемо приклад.

У HTML-файлі є один елемент абзацу. Потім у CSS ми маємо два правила, кожне з яких націлене на елемент абзацу.

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<p>example paragraph</p>
```

```css
p {
  color: blue;
}

p {
  color: green; 
}
```

:::

Перше правило встановлює для всіх елементів абзацу колір тексту синій, а друге правило встановлює для всіх елементів абзацу колір тексту зелений.

Отже, який колір буде застосовано? Колір зелений буде застосовано до елементів абзацу.

Враховуючи релевантність, походження та важливість, специфічність, область видимості та порядок появи, алгоритм каскаду гарантує, що ваш CSS працюватиме передбачувано, дозволяючи створювати більш складні та тонкі вебсторінки.
