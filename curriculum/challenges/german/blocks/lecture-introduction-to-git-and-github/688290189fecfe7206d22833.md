# --description--

In einer vorherigen Lektion haben wir GitHub kurz erwähnt. Sowohl GitHub als auch GitLab sind Anbieter von Versionskontrolle. Das bedeutet, dass sie cloudbasierte Lösungen sind, die die Speicherung von versionierten Projekten in sogenannten „Repositories“ anbieten und Funktionen zur Zusammenarbeit für diese Projekte ermöglichen.

Bevor wir uns den Funktionen widmen, die diese Plattformen bieten, müssen wir die verschiedenen Arten von Projekten besprechen. Viele Projekte gelten als „Open Source“, was bedeutet, dass Menschen den veröffentlichten Code sehen, Änderungen vorschlagen, Probleme melden und sogar eine modifizierte Version ausführen können.

Manchmal möchten Sie jedoch, dass nur Ihre Teammitglieder den Code sehen und daran arbeiten. Sie können ein Projekt auch als „Closed Source“ veröffentlichen, was bedeutet, dass nur die Personen, die Sie ausdrücklich autorisieren, das Projekt sehen und damit interagieren können.

Unabhängig davon, welchen Ansatz Ihr Projekt verfolgt, können Seiten wie GitHub und GitLab die Zusammenarbeit Ihres Teams erheblich verbessern.

Zum Beispiel könnte ein Teammitglied, das an einer Funktion arbeitet, feststecken und Fragen haben. Anstatt den Code in Discord kopieren und einfügen zu müssen, damit Sie ihn lesen, oder einen Anruf zu starten, um den Bildschirm zu teilen, kann es die Änderungen direkt auf GitHub hochladen. Von dort aus können Sie die Änderungen im Browser ansehen oder sogar herunterladen, um sie lokal auszuführen und zu debuggen.

Wenn ein Teammitglied eine Funktion fertiggestellt hat, kann es die Änderungen über diese Websites vorschlagen. Sie können die Änderungen dann überprüfen und Kommentare zu bestimmten Dateien oder Zeilen hinterlassen, wenn Sie Gedanken oder Fragen haben. Sie können sogar vorgeschlagene Änderungen machen, die sie direkt im Browser auf die Änderungen anwenden können!

Wenn Sie mit den Änderungen zufrieden sind, können Sie sie akzeptieren und in Ihren Hauptzweig zusammenführen. Das bedeutet, dass das gesamte Team die neuesten Änderungen lokal herunterladen kann, sodass alle den aktuellsten Code haben, wenn sie mit ihrer nächsten Aufgabe beginnen.

Apropos Aufgaben: Diese Seiten bieten auch leistungsstarke Issue-Tracker. Sie können diese verwenden, um Tickets für Ihre Teammitglieder zu verfolgen und zuzuweisen, Bug-Reports aus der Öffentlichkeit anzunehmen und sogar komplette Sprints über Funktionen wie Projektboards zu planen.

Diese Seiten bieten noch viele weitere Funktionen. In den nächsten Lektionen werden wir die wichtigsten Aspekte behandeln, damit Sie direkt loslegen und zusammenarbeiten können!
