## 可访问性树

可访问性树是屏幕阅读器等辅助技术用来解释网页内容并与之交互的结构。

## `max()` 函数

`max()` 函数返回一组逗号分隔值中最大的值：

:::interactive_editor

```html
<head>
  <link rel="stylesheet" href="styles.css">
</head>
<body>
  <img src="https://cdn.freecodecamp.org/curriculum/css-photo-gallery/9.jpg" alt="A cat sitting on a table">
</body>
```

```css
img {
  width: max(250px, 25vw);
}
```

:::

在上例中，如果视口宽度小于 1000 像素，图片的宽度将为 250px。 如果视口宽度大于 1000 像素，图像宽度将为 25vw。 这是因为 25vw 相当于视口宽度的 25%。

## CSS 和可访问性最佳实践

- **`display: none;`**：使用 `display: none;` 意味着屏幕阅读器和其他辅助技术将无法访问此内容，因为它不包含在辅助功能树中。 因此，只有当你想从视觉呈现和可访问性两方面完全删除内容时，才必须使用这种方法。
- **`visibility: hidden;`**：该属性和值在视觉上隐藏内容，但将其保留在文档流中，这意味着它仍占据页面空间。 屏幕阅读器也无法读取这些元素，因为它们已经从辅助功能树中删除。
- **`.sr-only` CSS 类**：这是一种常用的技术，用于在视觉上隐藏内容，同时保持屏幕阅读器可以访问这些内容。

:::interactive_editor

```html
<head>
  <link rel="stylesheet" href="styles.css">
</head>
<body>
  <span>This span element can be read</span>
  <span class="sr-only">This span element can only be read by screen readers</span>
  <p style="display: none">
    This text is hidden with display: none and is NOT accessible to screen readers.
  </p>
  <p style="visibility: hidden">
    This text is hidden with visibility: hidden and is NOT read by screen readers,
    but it still takes up space.
  </p>
</body>

```

```css
.sr-only {
  position: absolute;
  width: 1px;
  height: 1px;
  padding: 0;
  margin: -1px;
  overflow: hidden;
  clip: rect(0, 0, 0, 0);
  white-space: nowrap;
  border: 0;
}

```

:::

## 使用 HTML 属性隐藏内容

- **`aria-hidden` 属性**：用于向使用屏幕阅读器等辅助技术的用户隐藏元素。 例如，这可用于隐藏不提供任何有意义内容的装饰图片。
- **`hidden` 属性**：大多数现代浏览器都支持该属性，它既隐藏内容的视觉显示，也从可访问性树中将其移除。 它可以通过 JavaScript 轻松切换。

:::interactive_editor

```html
<p aria-hidden>This paragraph is visible for sighted users, but is hidden from assistive technology.</p>
<p hidden>This paragraph is hidden from both sighted users and assistive technology.</p>
<p>This is a normal paragraph</p>

```

:::

## `placeholder` 属性的可访问性问题

使用占位符文本不利于可访问性。 用户经常会将占位符文本与实际输入值混淆，以为输入中已经有一个值。
