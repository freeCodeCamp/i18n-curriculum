# --interactive--

Давайте дізнаємося про деякі загальні проблеми, які виникають при спробі стилізувати спеціальні вхідні елементи, такі як `datetime-local` і `color`.

Ці спеціальні типи вхідних елементів покладаються на складні псевдоелементи для створення таких речей, як вибір дати та кольору. Це створює значні труднощі при стилізації цих елементів. Одна з проблем полягає в тому, що оскільки стилізація за замовчуванням повністю залежить від браузера, CSS, який робить вибір кольору правильним в одному браузері, може дати зовсім інший результат в іншому.

Ось приклад вхідного елемента для вибору кольору:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<form>
  <label for="favorite-color">Pick your favorite color:</label>
  <input type="color" id="favorite-color" name="favorite-color">
</form>
```

```css
input {
  padding: 8px 12px;
  margin: 8px 0;
  border-radius: 6px;
  border: 1px solid #ccc;
}

input[type="color"] {
  width: 60px;
  height: 40px;
  padding: 0;
  border: 2px solid #555;
  border-radius: 4px;
  cursor: pointer;
}

```

:::

Ще одна проблема може бути у складності псевдоелемента. Розглянемо селектор дати; тут багато рухомих частин, і складна структура псевдоелемента може створити значні труднощі при застосуванні стилів до потрібних областей.

Ось приклад вхідного елемента для вибору дати:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<form>
  <label for="birthdate">Select your birthdate:</label>
  <input type="date" id="birthdate" name="birthdate">
</form>
```

```css
input {
  padding: 8px 12px;
  margin: 8px 0;
  border-radius: 6px;
  border: 1px solid #ccc;
}

input[type="date"] {
  padding: 6px 10px;
  border: 2px solid #555;
  border-radius: 4px;
  font-size: 14px;
  cursor: pointer;
}

input[type="date"]::-webkit-calendar-picker-indicator {
  background-color: #4CAF50;
  color: white;
  border-radius: 4px;
  cursor: pointer;
}

```

:::

Звісно, з такими складними елементами існує ризик випадково втратити важливу функціональність при ручній стилізації. Ви можете не тільки втратити важливі індикатори, такі як стан фокусу або вибраний елемент, але й потенційно повністю зламати селектор.

З цих причин багато розробників повністю покладаються на JavaScript-бібліотеки або кастомні компоненти замість використання вбудованих компонентів браузера.
