# --interactive--

Nakala ya kipepeo ya safu ya data ni safu mpya ambayo ina vitu sawa na safu ya awali. Ikiwa safu hiyo ina thamani za msingi tu kama nambari au mfuatano wa herufi, safu mpya ni tofauti kabisa. Lakini ikiwa safu ina safu nyingine ndani yake, asili na nakala zote mbili zina rejea kwa safu hizo za ndani. Hii inamaanisha kwamba ukibadilisha kitu ndani ya safu ya ndani inayoshirikiwa, utaona mabadiliko hayo katika safu zote mbili.

Nakala za kipepeo ni msaada wakati unahitaji kubadilisha muundo wa ngazi ya juu, kama kuongeza, kuondoa, au kubadilisha mpangilio wa vipengele, bila kubadilisha safu ya awali au safu ya ndani.

Kuna njia kadhaa za kutengeneza nakala za kipepeo za safu za data, na tutachunguza baadhi ya njia maarufu: `concat()`, `slice()`, na opereta ya spread.

Tuanze na njia ya `concat()`. Njia hii hutengeneza safu mpya kwa kuunganisha safu mbili au zaidi. Inapotumika na safu moja tu, hutengeneza nakala ya kipepeo kwa ufanisi. Hapa kuna mfano:

:::interactive_editor

```js
const originalArray = [1, 2, 3];
const copyArray = [].concat(originalArray);

console.log(copyArray); // [1, 2, 3]
console.log(copyArray === originalArray); // false
```

:::

Katika mfano huu, tunatumia njia ya `concat()` kuunganisha safu tupu na `originalArray`. Hii itatengeneza safu mpya ambayo ni nakala ya kipepeo ya `originalArray`.

`copyArray` ina vipengele sawa na `originalArray`, lakini ni kitu tofauti cha safu, ndiyo maana ukaguzi mkali wa usawa (`===`) unarudisha `false`.

Njia nyingine ya kutengeneza nakala ya kipepeo ni njia ya `slice()`. Inapoitwa bila hoja, `slice()` inarudisha nakala ya kipepeo ya safu nzima. Hivi ndivyo inavyofanya kazi:

:::interactive_editor

```js
const originalArray = [1, 2, 3];
const copyArray = originalArray.slice();

console.log(copyArray); // [1, 2, 3]
console.log(copyArray === originalArray); // false
```

:::

Katika kesi hii, `originalArray.slice()` inatengeneza safu mpya ambayo ni nakala ya kipepeo ya `originalArray`. Tena, `copyArray` ina vipengele sawa lakini ni kitu tofauti cha safu.

Opereta ya spread (`...`), iliyotangazwa katika ES6, inatoa njia nyingine fupi ya kutengeneza nakala za kipepeo za safu. Hapa kuna mfano:

:::interactive_editor

```js
const originalArray = [1, 2, 3];
const copyArray = [...originalArray];

console.log(copyArray); // [1, 2, 3]
console.log(copyArray === originalArray); // false
```

:::

Opereta ya spread (`...`) hueneza vipengele vya `originalArray` ndani ya safu mpya, kwa ufanisi kutengeneza nakala ya kipepeo. Ni muhimu kuelewa kwamba njia hizi zote hutengeneza vitu vipya vya safu, ambayo inamaanisha unaweza kubadilisha nakala bila kuathiri safu ya awali. Kwa mfano:

:::interactive_editor

```js
const originalArray = [1, 2, 3];
const copyArray = [...originalArray];

copyArray.push(4);
console.log(originalArray); // [1, 2, 3]
console.log(copyArray);     // [1, 2, 3, 4]
```

:::

Katika mfano huu, kuongeza kipengele kwenye `copyArray` hakutaathiri `originalArray`.

Kwa muhtasari, nakala za kipepeo za safu zinaweza kutengenezwa kwa urahisi kwa kutumia njia kama `concat()`, `slice()`, au opereta ya spread. Njia hizi ni muhimu kwa kutengeneza safu mpya ambazo zinaweza kubadilishwa huru bila kuathiri safu ya awali.
