## ما هي البرمجة كائنية التوجه؟

- **البرمجة كائنية التوجه**: أسلوب برمجي يعامل فيه المطوّرون كل شيء في شفرتهم ككائن من العالم الحقيقي. تُعرف اختصارًا بـ OOP. المبادئ الأربعة الأساسية التي تساعدك على تنظيم وإدارة الشفرة بفعالية هي **التغليف**، **الوراثة**، **تعدد الأشكال**، و**التجريد**.
- **الأصناف**: هي المخطط لإنشاء الكائنات. كل كائن يُنشأ من صنف يحتوي على خصائص تحدد البيانات وطرق تحدد سلوكيات الكائنات.

## ما هو التغليف؟

- **التغليف**: تجميع الخصائص والطرق الخاصة بالكائن في وحدة واحدة. يسمح لك بإخفاء الحالة الداخلية للكائن خلف مجموعة بسيطة من الطرق والخصائص العامة التي تعمل كأبواب. خلف هذه الأبواب توجد خصائص وطرق خاصة تتحكم في كيفية تغير البيانات ومن يمكنه رؤيتها.
- **مثال على التغليف**: إذا أردت تتبع رصيد محفظة، ستسمح بالإيداع والسحب، لكنك لن تسمح لأي شخص بالتلاعب برصيد المحفظة نفسه:

```py
class Wallet:
   def __init__(self, balance):
       self.__balance = balance # Private attribute

   def deposit(self, amount):
       if amount > 0:
           self.__balance += amount # Add to the balance safely

   def withdraw(self, amount):
       if 0 < amount <= self.__balance:
           self.__balance -= amount # Remove from the balance safely

account = Wallet(500)
print(account.__balance) # AttributeError: 'Wallet' object has no attribute '__balance'
```

- **الفرق بين بادئة الخصائص بواصلة واحدة وواصلة مزدوجة**: بادئة الخصائص والطرق بواصلة واحدة تعني أنها مخصصة للاستخدام الداخلي فقط. هذا اصطلاح ولا يمنع الوصول إلى الخصائص من الخارج. أما بادئة الخصائص والطرق بواصلة مزدوجة فتمنع فعليًا الوصول إليها من خارج الصنف.

## ما هي الدوال getter و setter؟

- **الدوال getter و setter**: طرق تتيح لك التحكم في كيفية الوصول إلى خصائص الصنف وتعديلها. تسترجع القيم باستخدام getters وتعيّن القيم باستخدام setters.
- **الخصائص**: تربط بين getters و setters، وتسمح بالوصول إلى البيانات. تنفذ منطقًا إضافيًا خلف الكواليس عند الحصول على القيم أو تعيينها أو حذفها.
- **لماذا الخصائص بدلًا من الطرق؟** تُستخدم الخصائص بدلاً من الطرق لقراءة أفضل وشفرة أنظف. تتيح لك الوصول إلى القيم باستخدام ترميز النقطة، مثل الخصائص العادية، بدون أقواس عادية.
- **إنشاء getter**: لإنشاء getter، تستخدم المزخرف `@property`. هذا مثال على getter يسترجع نصف قطر دائرة:

```py
class Circle:
    def __init__(self, radius):
        self._radius = radius

    @property
    def radius(self): # A getter to get the radius
        return self._radius
  
    @property
    def area(self):  # A getter to calculate area
        return 3.14 * (self._radius ** 2)

my_circle = Circle(3)

print(my_circle.radius) # 3
print(my_circle.area) # 28.26
```

- **إنشاء setter**: لإنشاء setter الذي يعيّن نصف القطر، عليك تعريف طريقة أخرى بنفس الاسم واستخدام المزخرف `@<property_name>.setter` فوقها:

```py
class Circle:
    def __init__(self, radius):
        self._radius = radius

    @property
    def radius(self):  # A getter to get the radius
        return self._radius

    @radius.setter
    def radius(self, value):  # A setter to set the radius
        if value <= 0:
            raise ValueError('Radius must be positive')
        self._radius = value

my_circle = Circle(3)
print('Initial radius:', my_circle.radius) # Initial radius: 3

my_circle.radius = 8
print('After modifying the radius:', my_circle.radius) # After modifying the radius: 8
```

- **كيف يتعامل Python مع getters و setters**: بمجرد تعريف getters و setters، يستدعيها Python تلقائيًا خلف الكواليس كلما استخدمت صيغة الخاصية العادية بهذه الطريقة:

```py
my_circle.radius # This will call the getter
my_circle.radius = 4 # This will call the setter
```

عند تعيين قيمة، لا يجب تعيينها إلى اسم الخاصية نفسه لأن ذلك يسبب `RecursionError`. استخدم اسمًا داخليًا منفصلًا، غالبًا مع واصلة سفلية، لتخزين القيمة.

- **المحذف (deleter)**: بعد تعيين واسترجاع قيمة باستخدام setter و getter، يمكنك التحكم في كيفية حذفها باستخدام `deleter`. ينفذ المحذف منطقًا مخصصًا عند استخدام عبارة `del` على خاصية. لإنشاء محذف، تستخدم المزخرف `@<property_name>.deleter`.

```py
  # Deleter
    @radius.deleter
    def radius(self):
        print("Deleting radius...")
        del self._radius
```

## ما هي الوراثة وكيف تعزز إعادة استخدام الشفرة؟

- **الوراثة**: العملية التي يستخدم فيها صنف فرعي خصائص وطرق صنف أبوي. تعزز الوراثة إعادة استخدام الشفرة، توفر تسلسلات هرمية واضحة، وتخصص السلوك بدون إعادة كتابة كل شيء. لتنفيذ الوراثة، يأخذ الصنف الفرعي اسم الصنف الأبوي:

```py
class Parent:
    # Parent attributes and methods

class Child(Parent):
    # Child inherits, extends, and/or overrides where necessary
```

- **الوراثة المفردة والمتعددة**: عندما يرث صنف فرعي خصائص وطرق من صنف أبوي واحد، كما في المثال أعلاه، تسمى هذه **الوراثة المفردة**. عندما يرث صنف فرعي خصائص وطرق من أكثر من صنف أبوي، تسمى **الوراثة المتعددة**. هذه هي الصيغة لذلك:

```py
class Parent:
    # Attributes and methods for Parent

class Child:
    # Attributes and methods for Child

class GrandChild(Parent, Child):
    # GrandChild inherits from both Parent and Child
    # GrandChild can combine or override behavior from each
```

- **دالة `super()`**: دالة تتيح لك **استدعاء** طريقة من صنف أبوي، عندما يكون للصنف تنفيذ مختلف لتلك الطريقة، أو يوسعها، بدون تكرار الشفرة.

## ما هو تعدد الأشكال وكيف يعزز إعادة استخدام الشفرة؟

- **تعدد الأشكال**: مبدأ في البرمجة كائنية التوجه يسمح لأصناف مختلفة باستخدام نفس اسم الطريقة، لكن كل صنف ينفذها بطريقة مختلفة عند استدعائها. هذه هي الصيغة لذلك:

```py
class A:
   def action(self): ...

class B:
   def action(self): ...

class C:
   def action(self): ...

Class().method()  # Works for A, B, or C
```

- **تعدد الأشكال القائم على الوراثة**: يحدد الصنف الأبوي طريقة، ويعدل كل صنف فرعي استخدامها حسب حاجته.

## ما هو تشويش الأسماء وكيف يعمل؟

- **تشويش الأسماء**: عملية يعيد فيها Python داخليًا تسمية خاصية تبدأ بواصلة مزدوجة بإضافة واصلة واسم الصنف كبادئة، محولًا `__attribute` إلى `_ClassName__attribute`.
- **الغرض من تشويش الأسماء**: الغرض الرئيسي من تشويش الأسماء هو منع تجاوز الخصائص والطرق عن طريق الخطأ عند استخدام الوراثة. هذا مثال شفرة لتوضيح ذلك:

```py
class Parent:
    def __init__(self):
        self.__data = 'Parent data'

class Child(Parent):
    def __init__(self):
        super().__init__()
        self.__data = 'Child data'

c = Child()
print(c.__dict__) # {'_Parent__data': 'Parent data', '_Child__data': 'Child data'}
```

## ما هو التجريد وكيف يساعد في تنظيم الأنظمة المعقدة؟

- **التجريد**: مفهوم برمجي يخفي تفاصيل التنفيذ المعقدة لكائن أو نظام ويُظهر فقط الميزات الأساسية. في Python ولغات برمجة أخرى، يبسط التجريد الأنظمة المعقدة بزيادة إعادة الاستخدام.
- **مثال على التجريد**: مثال جيد على التجريد في الحياة اليومية هو السيارة التي تتيح لك استخدام المقود والدواسات وذراع التغيير فقط دون معرفة كيفية عمل المحرك أو المكابح.
- **كيف ينفذ Python التجريد**: ينفذ Python التجريد عبر وحدة `abc`. توفر الوحدة الصنف `ABC` (الصنف الأساسي المجرد) والمزخرف `@abstractmethod`. يحدد الصنف الأساسي المجرد الطرق والخصائص المشتركة التي يجب على الأصناف الفرعية تنفيذها. لا يمكن إنشاء كائن منه.
- **كيفية تعريف الطريقة المجردة**: تُعرف الطريقة المجردة باستخدام `@abstractmethod` ويجب تجاوزها في الأصناف الفرعية، حتى لو كان لها تنفيذ افتراضي. الصيغة الأساسية للتجريد تبدو هكذا:

```py
from abc import ABC, abstractmethod

# Define an abstract base class
class AbstractClass(ABC):
    @abstractmethod
    def abstract_method(self):
        pass

# Concrete subclass that implements the abstract method
class ConcreteClassOne(AbstractClass):
    def abstract_method(self):
        print('Implementation in ConcreteClassOne')

# Another concrete subclass
class ConcreteClassTwo(AbstractClass):
    def abstract_method(self):
        print('Implementation in ConcreteClassTwo')
```
