# --description--

Ikiwa kitendakazi kitaitwa, lakini hoja haijatolewa, basi kigezo kitapewa thamani ya `undefined`. Hii inaweza kusababisha matokeo yasiyotarajiwa ikiwa kitendakazi hakijatengenezwa kushughulikia hali hii.

```js
function favoriteAnimal(animal) {
  return animal + " is my favorite animal!"
}

console.log(favoriteAnimal()); // Output: undefined is my favorite animal!
```

Katika mfano ulio juu, kitendakazi `favoriteAnimal` kinatelekezwa bila hoja. Kigezo `animal` kimepewa thamani ya `undefined`, ambayo kisha huunganishwa na mfuatano wa herufi `" is my favorite animal!"`.

Ikiwa thamani haijatolewa unaweza kutumia vigezo vya msingi. Vigezo vya msingi vinakuwezesha kuweka thamani ya msingi kwa kigezo ikiwa thamani haijatolewa wakati kitendakazi kinapoitwa.

```js
function favoriteAnimal(animal = 'Giant Panda') {
  return animal + " is my favorite animal!"
}
```

Katika mfano ulio juu, thamani ya msingi ya kigezo `animal` imewekwa kuwa `'Giant Panda'`. Ikiwa kitendakazi `favoriteAnimal` kinatelekezwa bila hoja, kigezo `animal` kitapewa thamani ya `'Giant Panda'`.

# --questions--

## --text--

Vigezo vya msingi katika vitendakazi vya JavaScript vinaboresha vipi tabia za vitendakazi kama `favoriteAnimal` vinapoitwa bila hoja yoyote?

## --answers--

---
