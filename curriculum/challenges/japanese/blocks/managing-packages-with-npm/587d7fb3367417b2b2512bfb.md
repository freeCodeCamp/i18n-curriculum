# --description--

このチャレンジに取り組むには、次のいずれかの方法でコードを書いてください。

- <a href="https://github.com/freeCodeCamp/boilerplate-npm/" target="_blank" rel="noopener noreferrer nofollow">このGitHubリポジトリ</a>をクローンして、<a href="https://www.freecodecamp.org/news/how-to-run-the-freecodecamp-backend-challenges-locally/" target="_blank" rel="noopener noreferrer nofollow">このガイド</a>を使ってNode.js、npmをセットアップし、ローカルでバックエンドプロジェクトを実行しながらチャレンジを完了する。
- お好みのサイトビルダーを使ってプロジェクトを完成させる。必ずGitHubリポジトリのすべてのファイルを取り込んでください。

`package.json` ファイルは、どのNode.jsプロジェクトやnpmパッケージにおいても中心的な存在です。プロジェクトに関する情報を保存します。情報はキーと値のペアで構成された単一のJSONオブジェクトで保存されます。必須フィールドは`name`と`version`の2つだけですが、追加情報を記載するのが良い習慣です。

`package.json` ファイルは、ターミナルから`npm init` コマンドを使って作成できます。これによりガイド付きセットアップが実行されます。`npm init` に`-y` フラグを付けて実行すると、質問なしでファイルが生成されます。`npm init -y`。

プロジェクトのファイルツリーを見ると、トップレベルに`package.json` ファイルがあるのがわかります。これが次のチャレンジで改善していくファイルです。

このファイルで最もよく使われる情報の一つが`author` フィールドです。プロジェクトの作成者を指定し、文字列か連絡先などの詳細を含むオブジェクトで記述できます。大きなプロジェクトではオブジェクトが推奨されますが、このプロジェクトでは以下のようなシンプルな文字列で十分です。

```json
"author": "Jane Doe",
```

# --instructions--

`author` ファイルの`package.json` にあなたの名前を追加してください。

**注意:** JSONを書いているので、すべてのフィールド名はダブルクォーテーション（"）で囲み、カンマ（,）で区切る必要があります。
