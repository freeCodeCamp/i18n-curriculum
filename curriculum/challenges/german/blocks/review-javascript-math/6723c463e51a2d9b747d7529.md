# --interactive--

## Arbeiten mit dem Datentyp Number

- **Definition**: Der `Number`-Typ in JavaScript umfasst ganze Zahlen, Fließkommazahlen, `Infinity` und `NaN`. Fließkommazahlen sind Zahlen mit Dezimalpunkt. Positive `Infinity` ist eine Zahl, die größer ist als jede andere Zahl, während `-Infinity` eine Zahl ist, die kleiner ist als jede andere Zahl. `NaN` (`Not a Number`) steht für einen ungültigen numerischen Wert wie den String `"Jessica"`.

## Häufige arithmetische Operationen

- **Additionsoperator**: Dieser Operator (`+`) wird verwendet, um die Summe von zwei oder mehr Zahlen zu berechnen. 
- **Subtraktionsoperator**: Dieser Operator (`-`) wird verwendet, um die Differenz zwischen zwei Zahlen zu berechnen.
- **Multiplikationsoperator**: Dieser Operator (`*`) wird verwendet, um das Produkt von zwei oder mehr Zahlen zu berechnen. 
- **Divisionsoperator**: Dieser Operator (`/`) wird verwendet, um den Quotienten zwischen zwei Zahlen zu berechnen.
- **Division durch Null**: Wenn Sie versuchen, durch Null zu dividieren, gibt JavaScript `Infinity` zurück.
- **Restoperator**: Dieser Operator (`%`) gibt den Rest einer Division zurück. 
- **Exponentiationoperator**: Dieser Operator (`**`) hebt eine Zahl auf die Potenz einer anderen.

## Berechnungen mit Zahlen und Strings

- **Erklärung**: Wenn Sie den `+`-Operator mit einer Zahl und einem String verwenden, wird JavaScript die Zahl in einen String umwandeln und die beiden Werte verketten. Wenn Sie die Operatoren `-`, `*` oder `/` mit einem String und einer Zahl verwenden, wird JavaScript den String in eine Zahl umwandeln, und das Ergebnis ist eine Zahl. Für `null` und `undefined` behandelt JavaScript `null` als 0 und undefined als `NaN` bei mathematischen Operationen.

:::interactive_editor

```js
const result = 5 + '10';

console.log(result); // "510"
console.log(typeof result); // string

const subtractionResult = '10' - 5;
console.log(subtractionResult); // 5
console.log(typeof subtractionResult); // number

const multiplicationResult = '10' * 2;
console.log(multiplicationResult); // 20
console.log(typeof multiplicationResult); // number

const divisionResult = '20' / 2;
console.log(divisionResult); // 10
console.log(typeof divisionResult); // number

const result1 = null + 5;
console.log(result1); // 5
console.log(typeof result1); // number

const result2 = undefined + 5;
console.log(result2); // NaN
console.log(typeof result2); // number
```

:::

## Operatorpriorität

- **Definition**: Die Operatorpriorität bestimmt die Reihenfolge, in der Operationen in einem Ausdruck ausgewertet werden. Operatoren mit höherer Priorität werden vor denen mit niedrigerer Priorität ausgewertet. Werte in Klammern werden zuerst ausgewertet, und Multiplikation/Division hat eine höhere Priorität als Addition/Subtraktion. Haben die Operatoren die gleiche Priorität, verwendet JavaScript die Assoziativität.

:::interactive_editor

```js
const result = (2 + 3) * 4;

console.log(result); // 20

const result2 = 10 - 2 + 3;

console.log(result2); // 11

const result3 = 2 ** 3 ** 2;

console.log(result3); // 512
```

:::

- **Definition**: Die Assoziativität gibt an, in welche Richtung ein Ausdruck ausgewertet wird, wenn mehrere Operatoren desselben Typs vorhanden sind. Sie definiert, ob der Ausdruck von links nach rechts (`left-associative`) oder von rechts nach links (`right-associative`) ausgewertet wird. Zum Beispiel ist der Exponentenoperator ebenfalls rechts-assoziativ:

:::interactive_editor

```js
const result4 = 5 ** 4 ** 1; 

console.log(result4); // 625
```

:::

## Inkrement- und Dekrementoperatoren

- **Inkrementoperator**: Dieser Operator wird verwendet, um den Wert um eins zu erhöhen. Die Präfixnotation `++num` erhöht zuerst den Wert der Variablen und gibt dann den neuen Wert zurück. Die Postfixnotation `num++` gibt zuerst den aktuellen Wert der Variablen zurück und erhöht ihn dann.

:::interactive_editor

```js
let x = 5;

console.log(++x); // 6
console.log(x); // 6


let y = 5;

console.log(y++); // 5
console.log(y); // 6
```

:::

- **Dekrementoperator**: Dieser Operator wird verwendet, um den Wert um eins zu verringern. Die Präfix- und Postfixnotation funktionieren genauso wie beim Inkrementoperator.

:::interactive_editor

```js
let num = 5;

console.log(--num); // 4
console.log(num--); // 4
console.log(num); // 3
```

:::

## Zusammengesetzte Zuweisungsoperatoren

- **Additionszuweisungsoperator (`+=`)**: Dieser Operator führt eine Addition der Werte durch und weist das Ergebnis der Variablen zu.
- **Subtraktionszuweisungsoperator (`-=`)**: Dieser Operator führt eine Subtraktion der Werte durch und weist das Ergebnis der Variablen zu.
- **Multiplikationszuweisungsoperator (`*=`)**: Dieser Operator führt eine Multiplikation der Werte durch und weist das Ergebnis der Variablen zu.
- **Divisionszuweisungsoperator (`/=`)**: Dieser Operator führt eine Division der Werte durch und weist das Ergebnis der Variablen zu.
- **Restzuweisungsoperator (`%=`)**: Dieser Operator dividiert eine Variable durch die angegebene Zahl und weist den Rest der Variablen zu.
- **Exponentationszuweisungsoperator (`**=`)**: Dieser Operator hebt eine Variable auf die Potenz der angegebenen Zahl und weist das Ergebnis der Variablen zu.

## Booleans und Gleichheit

- **Boolean-Definition**: Ein Boolean ist ein Datentyp, der nur zwei Werte haben kann: `true` oder `false`.
- **Gleichheitsoperator (`==`)**: Dieser Operator verwendet Typumwandlung, bevor geprüft wird, ob die Werte gleich sind.

:::interactive_editor

```js
console.log(5 == '5'); // true
```

:::

- **Strikter Gleichheitsoperator (`===`)**: Dieser Operator führt keine Typumwandlung durch und prüft, ob sowohl Typ als auch Wert gleich sind.

:::interactive_editor

```js
console.log(5 === '5'); // false
```

:::

- **Ungleichheitsoperator (`!=`)**: Dieser Operator verwendet Typumwandlung, bevor geprüft wird, ob die Werte ungleich sind.
- **Strikter Ungleichheitsoperator (`!==`)**: Dieser Operator führt keine Typumwandlung durch und prüft, ob sowohl Typ als auch Wert ungleich sind.

## Vergleichsoperatoren

- **Größer-als-Operator (`>`)**: Dieser Operator prüft, ob der Wert links größer ist als der Wert rechts.
- **Größer-als-oder-gleich-Operator (`>=`)**: Dieser Operator prüft, ob der Wert links größer oder gleich dem Wert rechts ist.
- **Kleiner-als-Operator (`<`)**: Dieser Operator prüft, ob der Wert links kleiner ist als der Wert rechts.
- **Kleiner-als-oder-gleich-Operator (`<=`)**: Dieser Operator prüft, ob der Wert links kleiner oder gleich dem Wert rechts ist.

## Unäre Operatoren

- **Unärer Plusoperator**: Dieser Operator wandelt seinen Operanden in eine Zahl um. Ist der Operand bereits eine Zahl, bleibt er unverändert.

:::interactive_editor

```js
const str = '42';
const num = +str;

console.log(num); // 42
console.log(typeof num); // number
```

:::

- **Unärer Negationsoperator (`-`)**: Dieser Operator negiert den Operanden.

:::interactive_editor

```js
const num = 4;
console.log(-num); // -4
```

:::

- **Logischer NICHT-Operator (`!`)**: Dieser Operator kehrt den Boolean-Wert seines Operanden um. Wenn der Operand `true` ist, wird er zu `false`, und wenn er `false` ist, wird er zu `true`.

## Bitweise Operatoren

- **Bitweises UND (`&`)**: Dieser Operator gibt an jeder Bitposition eine 1 zurück, für die die entsprechenden Bits beider Operanden 1 sind.
- **Bitweises UND-Zuweisungsoperator (`&=`)**: Dieser Operator führt eine `bitwise AND`-Operation mit der angegebenen Zahl aus und weist das Ergebnis der Variablen zu.
- **Bitweises ODER (`|`)**: Dieser Operator gibt an jeder Bitposition eine 1 zurück, für die die entsprechenden Bits eines oder beider Operanden 1 sind.
- **Bitweises ODER-Zuweisungsoperator (`|=`)**: Dieser Operator führt eine `bitwise OR`-Operation mit der angegebenen Zahl aus und weist das Ergebnis der Variablen zu.
- **Bitweises XOR (`^`)**: Dieser Operator gibt an jeder Bitposition eine 1 zurück, für die die entsprechenden Bits entweder des einen oder des anderen, aber nicht beider Operanden 1 sind.
- **Bitweises NICHT (`~`)**: Dieser Operator invertiert die binäre Darstellung einer Zahl.
- **Linksverschiebungsoperator (`<<`)**: Dieser Operator verschiebt alle Bits um eine angegebene Anzahl von Positionen nach links.
- **Rechtsverschiebungsoperator (`>>`)**: Dieser Operator verschiebt alle Bits nach rechts.

## Bedingte Anweisungen, truthy-Werte, falsy-Werte und der ternäre Operator

- **`if/else if/else`**: Eine `if`-Anweisung nimmt eine Bedingung und führt einen Codeblock aus, wenn diese Bedingung `truthy` ist. Ist die Bedingung `false`, wird zum `else if`-Block gewechselt. Wenn keine dieser Bedingungen `true` ist, wird die `else`-Klausel ausgeführt. `Truthy`-Werte sind alle Werte, die in einem Boolean-Kontext wie einer `true`-Anweisung zu `if` ausgewertet werden. `Falsy`-Werte sind Werte, die in einem Boolean-Kontext zu `false` ausgewertet werden.

:::interactive_editor

```js
const score = 87;

if (score >= 90) {
 console.log('You got an A'); 
} else if (score >= 80) {
 console.log('You got a B'); // You got a B
} else if (score >= 70) {
 console.log('You got a C');
} else {
 console.log('You failed! You need to study more!');
}
```

:::

- **Ternärer Operator**: Dieser Operator wird oft als kürzere Schreibweise für `if else`-Anweisungen verwendet.

:::interactive_editor

```js
const temperature = 30;
const weather = temperature > 25 ? 'sunny' : 'cool';

console.log(`It's a ${weather} day!`); // It's a sunny day!
```

:::

## Binäre logische Operatoren

- **Logisches UND (`&&`)**: Dieser Operator prüft, ob beide Operanden `true` sind. Sind beide `true`, gibt er den zweiten Wert zurück. Ist einer der Operanden `falsy`, gibt er den `falsy`-Wert zurück. Sind beide Operanden `falsy`, gibt er den ersten `falsy`-Wert zurück.

:::interactive_editor

```js
const result = true && 'hello';

console.log(result); // hello
```

:::

- **Logisches ODER (`||`)**: Dieser Operator prüft, ob mindestens einer der Operanden truthy ist.
- **Nullish Coalescing (`??`)**: Dieser Operator gibt einen Wert nur zurück, wenn der erste `null` oder `undefined` ist.

:::interactive_editor

```js
const userSettings = {
 theme: null,
 volume: 0,
 notifications: false,
};

let theme = userSettings.theme ?? 'light';
console.log(theme); // light
```

:::

## Das `Math`-Objekt

- **Die `Math.random()`-Methode**: Diese Methode erzeugt eine zufällige Fließkommazahl zwischen 0 (einschließlich) und 1 (ausschließlich). Das bedeutet, dass die Ausgabe 0 sein kann, aber niemals tatsächlich 1 erreicht.
- **Die `Math.max()`-Methode**: Diese Methode nimmt eine Menge von Zahlen und gibt den Maximalwert zurück.
- **Die `Math.min()`-Methode**: Diese Methode nimmt eine Menge von Zahlen und gibt den Minimalwert zurück.
- **Die `Math.ceil()`-Methode**: Diese Methode rundet einen Wert auf die nächsthöhere ganze Zahl auf.
- **Die `Math.floor()`-Methode**: Diese Methode rundet einen Wert auf die nächstniedrigere ganze Zahl ab.
- **Die `Math.round()`-Methode**: Diese Methode rundet einen Wert auf die nächste ganze Zahl.

:::interactive_editor

```js
console.log(Math.round(2.3)); // 2
console.log(Math.round(4.5)); // 5
console.log(Math.round(4.8)); // 5
```

:::

- **Die `Math.trunc()`-Methode**: Diese Methode entfernt den Dezimalteil einer Zahl und gibt nur den ganzzahligen Anteil zurück, ohne zu runden.
- **Die `Math.sqrt()`-Methode**: Diese Methode gibt die Quadratwurzel einer Zahl zurück.
- **Die `Math.cbrt()`-Methode**: Diese Methode gibt die Kubikwurzel einer Zahl zurück.
- **Die `Math.abs()`-Methode**: Diese Methode gibt den Absolutwert einer Zahl zurück.
- **Die `Math.pow()`-Methode**: Diese Methode nimmt zwei Zahlen und hebt die erste auf die Potenz der zweiten.

## Häufige Methoden für Zahlen

- **`isNaN()`**: `NaN` steht für „Not-a-Number“. Es ist ein spezieller Wert, der ein nicht darstellbares oder undefiniertes numerisches Ergebnis repräsentiert. Die `isNaN()`-Funktionseigenschaft wird verwendet, um zu bestimmen, ob ein Wert `NaN` ist oder nicht. `Number.isNaN()` bietet eine zuverlässigere Möglichkeit, `NaN`-Werte zu überprüfen, insbesondere in Fällen, in denen Typumwandlung zu unerwarteten Ergebnissen mit der globalen `isNaN()`-Funktion führen kann.

:::interactive_editor

```js
console.log(isNaN(NaN));       // true
console.log(isNaN(undefined)); // true
console.log(isNaN({}));        // true

console.log(isNaN(true));      // false
console.log(isNaN(null));      // false
console.log(isNaN(37));        // false


console.log(Number.isNaN(NaN));        // true
console.log(Number.isNaN(Number.NaN)); // true
console.log(Number.isNaN(0 / 0));      // true

console.log(Number.isNaN("NaN"));      // false
console.log(Number.isNaN(undefined));  // false
```

:::

- **Die `parseFloat()`-Methode**: Diese Methode parst ein String-Argument und gibt eine Fließkommazahl zurück. Sie ist dafür ausgelegt, eine Zahl vom Anfang eines Strings zu extrahieren, auch wenn der String später nicht-numerische Zeichen enthält.
- **Die `parseInt()`-Methode**: Diese Methode parst ein String-Argument und gibt eine ganze Zahl zurück. `parseInt()` stoppt das Parsen beim ersten nicht-ziffern Zeichen. Bei Fließkommazahlen gibt sie nur den ganzzahligen Teil zurück. Wenn sie am Anfang des Strings keine gültige ganze Zahl findet, gibt sie `NaN` zurück.
- **Die `toFixed()`-Methode**: Diese Methode wird auf einer Zahl aufgerufen und nimmt ein optionales Argument, das die Anzahl der Stellen nach dem Dezimalpunkt angibt. Sie gibt eine String-Darstellung der Zahl mit der angegebenen Anzahl Dezimalstellen zurück.
