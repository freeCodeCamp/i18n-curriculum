# --description--

Il file `.env` e la libreria `dotenv` sono strumenti usati nello sviluppo software moderno per separare i dati di configurazione sensibili dal codice dell'applicazione.

Un file `.env` è un file di testo semplice posizionato nella radice della directory di un progetto che archivia impostazioni di configurazione e segreti come coppie chiave-valore.

Invece di inserire direttamente nel codice sorgente chiavi API, password del database o numeri di porta rischiando di esporli se il codice viene caricato su un repository pubblico come GitHub, gli sviluppatori li salvano localmente in questo file.

La struttura di un file `.env` è semplice:

```bash
PORT=3000
DB_HOST=localhost
API_KEY=xyz123secretkey
```

Poiché questo file contiene chiavi altamente sensibili, è prassi comune aggiungere `.env` al file `.gitignore` del progetto in modo che non venga mai tracciato da Git.

Il pacchetto `dotenv` è un modulo leggero che collega il file `.env` alla tua applicazione.

Quando il programma parte, `dotenv` legge il file `.env`, analizza le chiavi e le inserisce nell'ambiente di esecuzione. In un'applicazione Node.js, ad esempio, inizializzi il pacchetto all'inizio del file di ingresso. Una volta caricate, le variabili diventano accessibili globalmente tramite l'oggetto `process.env`.

```javascript
const express = require('express');
const app = express();

// initialize and configure dotenv
require('dotenv').config();

// access the variables securely from the environment
const port = process.env.PORT || 5000;
const apiKey = process.env.API_KEY;

app.get('/', (req, res) => {
  res.send(`Connected successfully using API Key: ${apiKey}`);
});

app.listen(port, () => {
  console.log(`Server running on port ${port}`);
});
```

Questo sistema offre due grandi vantaggi. Primo, migliora la sicurezza mantenendo le credenziali private fuori dal codice sorgente. Secondo, offre flessibilità permettendo allo stesso codice di comportarsi diversamente tra ambienti di sviluppo, test e produzione semplicemente sostituendo il file `.env` locale su ciascun rispettivo server di hosting.
