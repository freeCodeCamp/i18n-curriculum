# --interactive--

In vorherigen Modulen haben Sie gelernt, mit String-Literalen wie diesem zu arbeiten:

```js
const greeting = "Hello, World!";
```

Aber JavaScript hat auch String-Objekte. Sowohl String-Objekte als auch String-Primitiven werden verwendet, um Text zu verarbeiten, funktionieren jedoch unter der Haube unterschiedlich. Ein String-Objekt wird mit der String-Konstruktorfunktion erstellt, die den primitiven Wert in einem Objekt kapselt. So erstellen Sie ein String-Objekt:

:::interactive_editor

```js
const greetingObject = new String("Hello, World!");

console.log(typeof greetingObject); // "object"
```

:::

Wenn wir den `typeof`-Operator verwenden, sehen wir, dass das Ergebnis vom Typ Objekt statt vom Typ String ist. Eine Frage, die Sie sich vielleicht gestellt haben, ist, wie Sie Eigenschaften wie die `.length`-Eigenschaft bei String-Primitiven verwenden können.

Wenn Sie die `length`-Eigenschaft bei einem String-Primitiv verwenden, kapselt JavaScript das String-Primitiv vorübergehend in einem String-Objekt, um die Operation auszuführen. Deshalb können Sie die `length`-Eigenschaft und verschiedene Methoden wie `repeat()`, `concat()` und `slice()` verwenden. Diese Arten von Methoden und Eigenschaften werden als Instanzmethoden, Instanzeigenschaften und statische Methoden bezeichnet. Wie das genau funktioniert, lernen Sie in zukünftigen Modulen.

Ein wesentlicher Unterschied zwischen einem String-Objekt und einem String-Primitiv liegt in Bezug auf Speicher und Leistung. String-Primitiven sind in der Regel speichereffizienter und schneller als String-Objekte. Obwohl Sie hauptsächlich mit String-Primitiven in Ihrem Code arbeiten werden, ist es dennoch nützlich zu verstehen, wie String-Objekte funktionieren.
