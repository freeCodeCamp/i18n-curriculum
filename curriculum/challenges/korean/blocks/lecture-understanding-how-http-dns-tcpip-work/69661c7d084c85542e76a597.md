# --description--

서버는 현대 웹에서 필수적입니다. 서버는 사용자가 매일 의존하는 온라인 경험을 지원하며, 개발자가 만들 대부분의 경험도 서버가 지원할 것입니다.

서버에 대해 더 알아봅시다.

알다시피, 인터넷은 상호 연결된 컴퓨터 네트워크의 전 세계 시스템입니다. 서버는 데이터를 교환하는 과정을 처리하여 이 통신을 가능하게 하는 중심 컴포넌트입니다.

**서버**는 다른 컴퓨터로부터 요청을 받고 네트워크를 통해 데이터나 서비스를 응답으로 보내는 컴퓨터 또는 컴퓨터 프로그램입니다.

서버는 이중 개념입니다. 왜냐하면 다음을 모두 가리킬 수 있기 때문입니다:

* 이러한 서비스를 효율적이고 지속적으로 제공하도록 설계된 실제 물리적이고 강력한 컴퓨터 **하드웨어**. 이 기계들은 신뢰성을 위해 구축되고 설계되었습니다.
    
* 또는 이러한 서비스를 제공하기 위해 해당 물리적 하드웨어에서 실행되는 실제 컴퓨터 **소프트웨어**.
    
“서버”라고 부르는 물리적 컴퓨터는 보통 데이터 센터에 보관되며, 데이터 센터는 환경이 통제된 크고 안전한 시설입니다.

서버는 **클라이언트-서버 모델**의 필수 부분입니다.

클라이언트-서버 모델은 컴포넌트가 클라이언트 또는 서버인 분산 시스템 아키텍처입니다.

* **클라이언트**는 데이터나 서비스를 위해 서버에 요청을 보내는 프로세스나 애플리케이션입니다. 예를 들어 웹 브라우저입니다.
    
* **서버**는 이러한 요청을 처리하고 응답을 반환하는 물리적 장치나 컴퓨터 프로그램입니다.

본질적으로 클라이언트와 서버는 요청-응답 주기의 일부로, 클라이언트가 서버에 요청을 보내면 서버가 클라이언트에 응답을 반환합니다.

예를 들어, 브라우저(클라이언트)를 사용해 사용자 이름과 비밀번호로 애플리케이션에 인증할 때, 정보가 안전하게 서버로 전송됩니다. 서버는 입력한 사용자 이름과 비밀번호가 플랫폼의 데이터베이스에 있는 정보와 일치하는지 확인합니다. 일치하면 서버는 클라이언트(브라우저)에 성공 응답을 반환하고 사이트에 성공적으로 인증됩니다.

일치하지 않으면 서버는 실패 응답을 클라이언트에 반환하며, 보통 로그인 시도가 실패한 이유를 설명하는 오류나 피드백을 보게 됩니다.

이 모든 인증 로직은 서버에서 실행됩니다. 서버는 자격 증명을 검증하고, 데이터베이스와 상호작용하며, 각 요청에 어떻게 응답할지 결정하는 책임이 있습니다.

각 서버 응답에는 HTTP 상태 코드가 포함되어 클라이언트에 무슨 일이 있었는지 알려줍니다. 이 상태 코드는 요청이 성공했는지 실패했는지, 그리고 그 이유를 클라이언트에 알립니다.

예를 들어:

* 200은 모든 것이 정상임을 의미합니다.
    
* 404는 서버에서 리소스를 찾을 수 없음을 의미합니다.
    
* 500은 내부 서버 오류가 있음을 의미합니다.
    
* 503은 서버가 일시적으로 사용 불가능함을 의미합니다.
    
* 기타 등등…
    

각 상태 코드는 고유한 의미를 가집니다.

서버를 사용하는 실제 애플리케이션 예로는 전자상거래, 검색 엔진, 게임, 비디오 스트리밍, 클라우드 저장소 및 협업 플랫폼, 소셜 미디어 플랫폼, 금융 애플리케이션 등이 있습니다.

데이터가 동적으로 업데이트되거나 애플리케이션이 데이터베이스와 통신해야 할 경우, 서버는 통신 과정의 핵심 컴포넌트일 가능성이 높습니다.

주요 서버 유형은 다음과 같습니다:

* **웹 서버**는 HTML 페이지, 이미지, CSS 파일 같은 웹 콘텐츠를 브라우저에 전달합니다.
    
* **애플리케이션 서버**는 비즈니스 로직을 처리하고 사용자 입력을 처리하며 응답을 조정합니다.
    
* **데이터베이스 서버**는 데이터베이스 관리 시스템을 실행하고 데이터를 저장, 검색, 관리합니다.
    
* **메일 서버**는 이메일 메시지를 보내고 받고 저장합니다.
    
* **파일 서버**는 파일 접근을 관리하며 네트워크를 통해 파일을 저장하고 공유할 수 있게 합니다.
    
* **프록시 서버**는 클라이언트와 다른 서버 사이에서 중개자 역할을 하며, 캐싱, 보안, 트래픽 제어에 자주 사용됩니다.

서버는 우리가 매일 사용하는 강력한 온라인 플랫폼의 중추입니다. 서버 덕분에 복잡한 로직을 실행하고, 대량의 데이터를 저장 및 처리하며, 동시에 수백만 사용자를 지원할 수 있습니다. 서버가 없다면 우리가 의존하는 현대 디지털 세계는 존재하지 않을 것입니다.
