# --description--

React надає багато вбудованих хуків, які дозволяють реалізовувати різні функції у ваших проєктах. Серед них є `useState`, `useEffect`, `useContext` та інші.

Але іноді потрібно додати функцію, з якою жоден із вбудованих хуків не допоможе. На щастя, ви можете створити власні кастомні хуки в React.

Кастомні хуки не такі складні, як здаються. Це просто повторно використовувані функції, які дозволяють ділитися логікою між кількома компонентами. Тобто повторне використання — ще одна причина, чому варто створити свій хук.

За допомогою кастомного хука ви можете винести логіку з будь-яких компонентів, які його використовують, наприклад, отримання даних, керування станом, перемикання, побічні ефекти, як-от перевірка онлайн- чи офлайн-статусу користувачів, і так далі.

Потім ви можете імпортувати цей хук для використання в будь-якому компоненті, щоб зосередитися на відтворенні та презентації в цих компонентах. Це означає менше повторень і дублювань, а отже, менше місць, де потрібно вносити зміни під час оновлень.

Тепер давайте подивимося, як створити власний кастомний хук.

У React усі вбудовані хуки починаються зі слова `use`, тому ваш кастомний хук має дотримуватися тієї ж конвенції. Назва вашого кастомного хука також має чітко передавати, що він робить.

Отже, якщо ваш кастомний хук…

- отримує дані, ви можете назвати його `useFetch`

- перемикає щось увімкнено/вимкнено, ви можете назвати його `useToggle`

- або якщо він реалізує дебаунсінг, `useDebounce` — гарна назва

Припустимо, ви хочете створити кастомний хук для додавання дебаунсінгу у ваш застосунок.

Дебаунсінг — це техніка програмування, яка обмежує, як часто функція виконується. Вона працює, чекаючи, поки користувач припинить виконувати дію на певний період часу, перш ніж виконати функцію. Наприклад, у пошуковому полі замість того, щоб робити API-запит на кожен натиск клавіші, дебаунсінг чекає, поки користувач припинить друкувати, скажімо, на 500 мілісекунд.

Щоб створити кастомний хук для дебаунсінгу, спочатку потрібно створити `useDebounce.jsx` або `useDebounce.js` файл. Зазвичай файли для кастомних хуків зберігають у папці `hooks`.

Ви можете використовувати деякі вбудовані хуки всередині свого кастомного хука. Для дебаунсінгу потрібні хуки `useState` і `useEffect`, тому імпортуйте їх на початку файлу:

```js
import { useState, useEffect } from "react";
```

Далі створіть функцію `useDebounce`, яка приймає `value` і `delay` як параметри. `value` — це ресурс, на який ви хочете чекати, а `delay` — період часу, який ви хочете чекати. Оскільки потрібно чекати певний час, функції `setTimeout` і `clearTimeout` будуть корисними:

```js
function useDebounce(value, delay) {
  const [debouncedValue, setDebouncedValue] = useState(value);

  useEffect(() => {
    const handler = setTimeout(() => {
      setDebouncedValue(value);
    }, delay);

    return () => {
      clearTimeout(handler);
    };
  }, [value, delay]);

  return debouncedValue;
}

export { useDebounce };
```

Стан `debouncedValue` зберігає і повертає затримане значення, яке оновлюється лише після вказаного таймауту.

`useEffect` — це місце, де відбувається справжня магія. Якщо пригадати попередній урок, усе, що існує поза циклом відтворення React, як-от встановлення та очищення таймера, є побічним ефектом, і для їх обробки слід використовувати хук `useEffect`.

У цьому хуку `useEffect` ви використовуєте `setTimeout`, щоб встановити `debouncedValue`. Потім повертаєте функцію очищення, яка використовує `clearTimeout` для скасування попереднього таймауту щоразу, коли змінюються `value` або `delay`, або коли компонент демонтується.

А ось компонент `FootballerSearch`, який використовує хук `useDebounce` для затримки пошуку на 1 секунду після того, як користувач припиняє друкувати:

```jsx
import { useState, useEffect } from "react";
import { useDebounce } from "./hooks/useDebounce";
import footballers from "./footballers";

const FootballerSearch = () => {
  const [query, setQuery] = useState("");
  const debouncedQuery = useDebounce(query, 1000); // Start searching 1 second after the user stops typing

  useEffect(() => {
    if (debouncedQuery) {
      const results = footballers.filter((footballer) =>
        footballer.toLowerCase().includes(debouncedQuery.toLowerCase()),
      );
      console.log("Search results:", results);
    } else {
      console.log("Search results: []");
    }
  }, [debouncedQuery]);

  return (
    <>
      <h1 style={{ textAlign: "center" }}>Footballer Search App</h1>
      <div style={{ textAlign: "center" }}>
        <input
          style={{ padding: "0.5rem", width: "30%" }}
          type="text"
          value={query}
          onChange={(e) => setQuery(e.target.value)}
          placeholder="Search for a footballer..."
        />
      </div>
    </>
  );
};

export default FootballerSearch;
```

Як бачите, змінна `debouncedQuery` ініціалізує хук `useDebounce` зі станом запиту (те, що вводить користувач) і затримкою в 1 000 мілісекунд, або 1 секунду. Сам пошук обробляється всередині хуку `useEffect`, а результати пошуку виводяться в консоль.
