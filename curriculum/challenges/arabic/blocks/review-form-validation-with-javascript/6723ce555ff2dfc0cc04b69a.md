# --interactive--

## التحقق من صحة النماذج باستخدام JavaScript

- **واجهة برمجة التحقق من القيود (Constraint Validation API)**: بعض عناصر HTML، مثل `textarea` و`input`، تعرض واجهة برمجة تحقق من القيود. تتيح هذه الواجهة التأكد من أن القيمة التي يقدمها المستخدم لهذا العنصر تجتاز أي تحقق على مستوى HTML كتبته، مثل الحد الأدنى للطول أو مطابقة النمط.
- **طريقة `checkValidity()`**: تُرجع هذه الطريقة `true` إذا تطابق العنصر مع كل تحقق HTML (بناءً على خصائصه)، و`false` إذا فشل.

:::محرر_تفاعلي

```html
<form>
  <label>
    Email:
    <input
      id="email"
      type="email"
      required
      pattern=".+\.com$"
      placeholder="example@site.com"
    />
  </label>
</form>

<script>
  const input = document.getElementById("email");

  input.addEventListener("input", (e) => {
    if (!e.target.checkValidity()) {
      e.target.setCustomValidity("You must use a .com email.");
    } else {
      e.target.setCustomValidity("");
    }
  });
</script>
```

:::

- **طريقة `reportValidity()`**: تخبر هذه الطريقة المتصفح بأن `input` غير صالح.

:::محرر_تفاعلي

```html
<form>
  <label>
    Email:
    <input
      id="email2"
      type="email"
      required
      pattern=".+\.com$"
      placeholder="example@site.com"
    />
  </label>
</form>

<script>
  const input = document.getElementById("email2");

  input.addEventListener("input", (e) => {
    if (!e.target.checkValidity()) {
      e.target.reportValidity();
    }
  });
</script>
```

:::

- **خاصية `validity`**: تُستخدم هذه الخاصية للحصول على حالة صلاحية عناصر التحكم في الموذج (مثل `<input>`, `<select>`, وغيرها) وتوفر معلومات حول ما إذا كان مُدخَل المستخدم يفي بالقيود المحددة لذلك العنصر (مثل الحقول `required`، قيود النمط، الحد الأقصى للطول، وغيرها).

:::محرر_تفاعلي

```html
<input
  id="age"
  type="number"
  min="18"
  placeholder="Enter age (18+)"
/>

<script>
  const input = document.getElementById("age");

  input.addEventListener("input", (e) => {
    console.log(e.target.validity);
  });
</script>
```

:::

- **خاصية `patternMismatch`**: ستكون `true` إذا لم تتطابق القيمة مع نمط التعبير النمطي المحدد.

## طريقة `preventDefault()`

- **تعريف**: لكل حدث يحدث في DOM سلوك افتراضي معين. على سبيل المثال، نقر مربع اختيار يبدل حالته افتراضيًا. الضغط على مفتاح المسافة على زر مركّز يُفعّل الزر. تمنع طريقة `preventDefault()` على كائنات `Event` هذا السلوك من الحدوث.

:::محرر_تفاعلي

```html
<form id="form">
  <input type="text" placeholder="Try to submit" />
  <button type="submit">Submit</button>
</form>

<p id="status"></p>

<script>
  const form = document.getElementById("form");
  const status = document.getElementById("status");

  form.addEventListener("submit", (event) => {
    event.preventDefault();
    status.textContent = "Form submission prevented.";
  });
</script>
```

:::

## إرسال النماذج

- **تعريف**: هناك ثلاث طرق لإرسال نموذج. الأولى عندما ينقر المستخدم زرًا في النموذج له الخاصية `type` مضبوطة على `submit`. الثانية عندما يضغط المستخدم مفتاح `Enter` في أي حقل `input` قابل للتحرير في النموذج. الثالثة من خلال استدعاء JavaScript لطريقتي `requestSubmit()` أو `submit()` لعناصر `form`.
- **خاصية `action`**: يجب أن تحتوي خاصية `action` على عنوان URL أو مسار نسبي للنطاق الحالي. يحدد هذا المكان الذي يحاول النموذج إرسال البيانات إليه - إذا لم تضبط خاصية `action`، سيرسل النموذج البيانات إلى عنوان URL للصفحة الحالية.

```html
<form action="https://freecodecamp.org" method="GET">
  <input
    type="number"
    name="number"
    placeholder="Enter a number"
  />
  <button type="submit">Submit</button>
</form>
```

- **الخاصية `method`**: تقبل هذه الخاصية طريقة `HTTP` قياسية، مثل `GET` أو `POST`، وتستخدم تلك الطريقة عند إجراء الطلب إلى عنوان URL الخاص بالإجراء. وعندما لا تُضبط طريقة، فإن النموذج يلجأ افتراضيًا إلى طلب `GET`. وستُرمَّز بيانات النموذج بترميز URL على هيئة أزواج `name=value` وتُلحَق بعنوان URL الخاص بالإجراء كمعلمات استعلام.

```html
<form action="/data" method="POST">
  <input
    type="number"
    id="input"
    placeholder="Enter a number"
    name="number"
  />
  <button type="submit">Submit</button>
</form>
```

- **خاصية `enctype`**: يقبل عنصر `form` خاصية `enctype` التي تمثل نوع الترميز المستخدم للبيانات. تقبل هذه الخاصية ثلاثة قيم فقط: `application/x-www-form-urlencoded` (الافتراضي، يرسل البيانات كجسم نموذج مشفر في URL)، `text/plain` (يرسل البيانات كنص عادي، في أزواج `name=value` مفصولة بأسطر جديدة)، أو `multipart/form-data`، المخصصة للتعامل مع النماذج التي تحتوي على رفع ملفات.
