# --interactive--

Katika mafundisho yaliyopita, ulijifunza kuhusu programu isiyo ya wakati mmoja ambayo inaruhusu msimbo mwingine kuendelea kuendeshwa wakati tunasubiri kazi zinazochukua muda kama vile kupata data kutoka seva, kusoma data kutoka faili, na kadhalika.

`async`/`await`, iliyojengwa juu ya ahadi, inafanya uandishi na usomaji wa msimbo isiyo ya wakati mmoja kuwa rahisi. Unapoweka neno kuu la `async` mbele ya kitendakazi, ina maana kitendakazi hicho kitaendelea kila wakati kurudisha `Promise`. Neno kuu la `await`, ambalo linakuwezesha kusubiri `Promise` kutatuliwa kabla ya kuendelea na mstari unaofuata wa msimbo, linaweza kutumika tu ndani ya vitendakazi isiyo ya wakati mmoja au viungo vya ngazi ya juu vya moduli. Hapa kuna mfano unaoonyesha jinsi `async`/`await` inavyofanya kazi:

:::interactive_editor

```js
async function delayedGreeting(name) {
  console.log("A Messenger entered the chat...");
  await new Promise(resolve => setTimeout(resolve, 2000));
  console.log(`Hello, ${name}!`);
}

delayedGreeting("Alice");
console.log("First Printed Message!");
```

:::

Katika msimbo huu, tunafafanua kitendakazi cha `async` kinachoitwa `delayedGreeting`. Ndani ya kitendakazi hiki, tunatumia `await` kusitisha utekelezaji kwa sekunde 2. Baada ya kuchelewa, inachapisha salamu.

Tunapoitisha kitendakazi hiki, utaona `First Printed Message!` ikionekana kabla ya salamu. Hii ni kwa sababu kitendakazi ni isiyo ya wakati mmoja - hakizuizi msimbo mwingine kuendelea kuendeshwa.

Moja ya faida kubwa za `async`/`await` ni usimamizi wa makosa. Kwa ahadi, mara nyingi tulilazimika kutumia njia ya `.catch()` kushughulikia makosa. Kwa `async`/`await`, tunaweza kutumia vifungu vya `try`/`catch`. Hapa kuna mfano:

```js
async function fetchUserData() {
  try {
    let response = await fetch(`https://api.example.com/users`);
    let userData = await response.json();
    console.log(userData);
  } catch (error) {
    console.log("Error fetching user data:", error);
  }
}

fetchUserData();
```

Katika mfano huu, tunatumia `async`/`await` kupata data za mtumiaji kutoka API. Neno kuu la `await` linatumika mara mbili: mara moja kusubiri operesheni ya fetch kukamilika, na tena kusubiri uchambuzi wa JSON kumalizika. Ikiwa kuna kosa lolote litakalotokea wakati wa mchakato huu, litatambuliwa katika kifungu cha `catch`.
