# --description--

У попередньому уроці ми дізналися про роботу з `localStorage` і коротко познайомилися з `sessionStorage`.

Нагадаємо, що `sessionStorage` означає, що дані очищуються одразу після того, як користувач закриває вкладку або вікно, у якому працює веб-застосунок. Це ідеально підходить для ситуацій, коли дані потрібно зберігати лише протягом однієї сесії, наприклад, для збереження даних форми під час навігації або тимчасового збереження стану під час оформлення замовлення.

Подібно до `localStorage`, `sessionStorage` використовує пари ключ-значення для збереження та отримання даних. Методи, які застосовуються з `sessionStorage`, також такі ж, як і з `localStorage`, єдина реальна різниця — це тривалість зберігання даних.

Ось кілька прикладів роботи з різними методами:

- `sessionStorage.setItem()`: Зберігає пару ключ-значення в `sessionStorage`.

```js
sessionStorage.setItem('cart', '3 items');
```

- `sessionStorage.getItem()`: Отримує значення за заданим ключем з `sessionStorage`.

```js
let cart = sessionStorage.getItem('cart');
console.log(cart); // Outputs: '3 items'
```

- `sessionStorage.removeItem()`: Видаляє конкретний елемент з `sessionStorage` за його ключем.

```js
sessionStorage.removeItem('cart');
```

- `sessionStorage.clear()`: Очищає всі дані, збережені в `sessionStorage`.

```js
sessionStorage.clear();
```

Розглянемо приклад, де ми зберігаємо дані в `sessionStorage`, які зберігаються лише поки відкрита вкладка або вікно браузера:

```js
// Store data in sessionStorage
sessionStorage.setItem('currentUser', 'JohnDoe');

// Retrieve the stored data
const user = sessionStorage.getItem('currentUser');
console.log(user); // 'JohnDoe'

// Remove a specific key from sessionStorage
sessionStorage.removeItem('currentUser');

// Clear all sessionStorage data
sessionStorage.clear();
```

У цьому прикладі ми:

1. Зберігаємо ім’я поточного користувача (`JohnDoe`) у `sessionStorage`.

2. Отримуємо його та відображаємо.

3. Видаляємо елемент, пов’язаний із ключем `currentUser`.

4. Очищаємо всі дані `sessionStorage`.

Головна відмінність від `localStorage` полягає в тому, що як тільки користувач закриває вкладку, усі збережені сесійні дані втрачаються.

`sessionStorage` особливо корисний у таких випадках:

- Збереження тимчасових даних, наприклад, записів форми під час багатосторінкового процесу.

- Збереження тимчасових виборів або налаштувань, які не потрібно зберігати між сесіями.

- Підтримка стану в односторінковому застосунку, який не потрібно пам’ятати після закриття вкладки.

`sessionStorage` гарантує, що після того, як користувач залишає сторінку, сесійні дані очищуються, що чудово підходить для випадків, коли не потрібно зберігати інформацію понад поточну сесію.
