# --description--

Express에서 라우트 매개변수는 URL 경로에 임베드된 동적 값입니다. 애플리케이션이 각 요청마다 달라질 수 있는 URL의 일부를 캡처할 수 있게 해줍니다. 이는 특정 사용자, 블로그 게시물, 또는 URL에 따라 달라지는 제품을 표시하는 동적 페이지를 만드는 데 유용합니다.

예를 들어 다음과 같습니다:

```javascript
// Route with a parameter in the path
app.get("/post/:postId", (req, res) => {
  const postId = req.params.postId;
  res.send(`Viewing post with ID: ${postId}`);
});
```

여기서 라우트는 `/post/:postId`에 대한 GET 요청을 수신합니다.

* `:postId`은 라우트 매개변수로, URL에서 실제 값으로 교체될 자리 표시자입니다. 예를 들어, `/post/42`에 대한 요청은 이 라우트와 일치하며 `42`이 `postId`의 값이 됩니다.
    
* `req.params`를 사용해 라우트 매개변수에 접근할 수 있습니다. `req.params` 객체는 모든 라우트 매개변수를 포함하며, 이 경우 `req.params.postId`은 URL에서 온 값인 `42`을 포함합니다.
    

마지막으로 `res.send()`가 클라이언트에게 응답을 보냅니다. 따라서 URL이 `/post/42`이라면 클라이언트는 `Viewing post with ID: 42`을 보게 됩니다.

이처럼 라우트 매개변수는 모든 가능한 값마다 별도의 라우트를 만들 필요 없이 라우트를 유연하고 동적으로 만듭니다.

라우트 매개변수가 어떻게 작동하는지 이해했으니, 지난 수업의 예제로 돌아가서 다양한 메뉴 유형에 동적 라우트를 어떻게 사용할 수 있는지 살펴보겠습니다:

```javascript
app.get("/menu/:category", (req, res) => {
  const category = req.params.category;
  res.send(`You are viewing the ${category} menu!`);
});
```

이 작동 방식을 간단히 설명하면 다음과 같습니다:

1. 사용자가 [`http://localhost:3000/menu/drinks`](http://localhost:3000/menu/drinks)를 방문합니다.
    
2. 서버는 이 요청을 라우트 `/menu/:category`와 일치시키고 `category` 매개변수를 `drinks`로 캡처합니다.
    
3. 서버는 이 응답을 브라우저로 보냅니다: `You are viewing the drinks menu!`
    

마찬가지로 사용자가 [`http://localhost:3000/menu/food`](http://localhost:3000/menu/food)을 방문하면 서버는 다음과 같이 응답합니다: `You are viewing the food menu!`

라우트 매개변수를 사용하면 URL에 따라 동적 콘텐츠를 표시할 수 있습니다. 모든 카테고리에 대해 별도의 라우트를 만들지 않고도 URL만 바꿔서 음료, 음식, 디저트 등 여러 카테고리를 하나의 라우트로 처리할 수 있습니다.
