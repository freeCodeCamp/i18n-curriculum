# --interactive--

Le proprietà e i metodi statici appartengono alla classe stessa, non alle singole istanze della classe. Puoi consultarli direttamente sul nome della classe senza creare un'istanza della classe. Sono definiti all'interno delle classi per racchiudere funzionalità correlate.

Puoi definire un metodo statico scrivendo la parola chiave `static` prima del nome del metodo.

```js
class MyClass {
  static staticMethod() { ... }
}
```

Poi, puoi chiamare il metodo statico direttamente sulla classe, usando la notazione a punto e passando gli argomenti necessari:

```js
MyClass.staticMethod();
```

Nota che puoi chiamare il metodo senza creare un'istanza della classe. Questa è una delle caratteristiche principali dei metodi statici.

Ecco un esempio. Supponiamo di creare un'applicazione di film e vogliamo poter confrontare i film in base al loro voto.

Potremmo considerare questo metodo di confronto come un metodo di livello superiore che non è specifico per nessun film:

```js
if (movieA.rating < movieB.rating) {
  console.log(`${movieB.title} has a higher rating.`);
} 
```

È come un metodo più generale relativo alla classe `Movie`.

Per motivi di leggibilità e manutenibilità, sarebbe utile definirlo all'interno della classe `Movie` per tenere tutti i metodi correlati relativamente vicini tra loro.

Quindi, questo è un candidato perfetto per un metodo statico. Lo puoi vedere qui, subito sotto il costruttore:

:::interactive_editor

```js
class Movie {

  constructor(title, rating) {
    this.title = title;
    this.rating = rating;
  }

  static compareMovies(movieA, movieB) {
    if (movieA.rating > movieB.rating) {
      console.log(`${movieA.title} has a higher rating.`);
    } else if (movieA.rating < movieB.rating) {
      console.log(`${movieB.title} has a higher rating.`);
    } else {
      console.log("These movies have the same rating.");
    }
  }

}

let movieA = new Movie("Movie A", 80);
let movieB = new Movie("Movie B", 45);
console.log(movieA);
```

:::

Il metodo statico è definito con la parola chiave `static` e si chiama `compareMovies`. Ha due parametri: `movieA` e `movieB`. Questi saranno istanze della classe `Movie`.

Li confronteremo in base al loro voto, su una scala da `0` a `100`. Questa logica è implementata con una condizione e stamperà un messaggio appropriato in base a quale film ha un voto più alto.

Una volta definito il metodo, puoi chiamarlo sulla classe. Ma prima, devi avere gli argomenti pronti e definiti nel tuo programma.

In questo caso, il metodo prende due istanze di film come argomenti, quindi definiamo queste istanze qui:

```js
let movieA = new Movie("Movie A", 80);
let movieB = new Movie("Movie B", 45);
```

Puoi vedere che `movieA` ha un voto più alto di `movieB`. Vediamo l'output di questo metodo.

Per chiamare il metodo, devi solo usare la notazione a punto sulla classe stessa.
Scrivi il nome della classe (`Movie`), seguito da un punto, e poi il nome del metodo statico (`compareMovies`).

Poi, passi gli argomenti tra parentesi tonde. In questo caso, sono le due istanze di film che il metodo richiede.

```js
Movie.compareMovies(movieA, movieB);
```

Ecco l'esempio aggiornato:

:::interactive_editor

```js
class Movie {

  constructor(title, rating) {
    this.title = title;
    this.rating = rating;
  }

  static compareMovies(movieA, movieB) {
    if (movieA.rating > movieB.rating) {
      console.log(`${movieA.title} has a higher rating.`);
    } else if (movieA.rating < movieB.rating) {
      console.log(`${movieB.title} has a higher rating.`);
    } else {
      console.log("These movies have the same rating.");
    }
  }

}

let movieA = new Movie("Movie A", 80);
let movieB = new Movie("Movie B", 45);
Movie.compareMovies(movieA, movieB);
console.log(movieA);
```

:::

I metodi statici sono utili anche per implementare i metodi "factory". Un metodo factory è un metodo che definisci oltre al costruttore per creare oggetti basati su criteri specifici.

Ecco un esempio con una classe `Pizza`. Il metodo statico `createMargherita` è un metodo factory che puoi chiamare per creare un'istanza di pizza Margherita con il suo tipo e prezzo già impostati.

```js
class Pizza {
  constructor(type, price) {
    this.type = type;
    this.price = price;
  }

  static createMargherita() {
    return new this("Margherita", 6.99);
  }
}
```

Questo porta anche a qualcosa di molto importante riguardo ai metodi statici.

Il valore della parola chiave `this` nei metodi statici è la classe stessa, dato che il metodo statico appartiene alla classe.

Ecco perché possiamo usare `this` per creare una nuova istanza della classe `Pizza`.

Se chiami questo metodo sulla classe `Pizza` stessa e assegni l'istanza restituita a una variabile, come in questo esempio:

```js
let myPizza = Pizza.createMargherita();
```

Puoi usarla ovunque nel tuo codice. Per esempio, puoi stamparla sulla console:

```js
console.log(myPizza);
```

Questo è l'output:

```js
Pizza { type: 'Margherita', price: 6.99 }
```

Puoi anche usare la notazione a punto per chiamare i suoi metodi e consultare le sue proprietà, così:

```js
console.log(myPizza.type);
```

Ecco l'esempio completo:

```js
class Pizza {
  constructor(type, price) {
    this.type = type;
    this.price = price;
  }

  static createMargherita() {
    return new this("Margherita", 6.99);
  }
}

let myPizza = Pizza.createMargherita();
console.log(myPizza);
console.log(myPizza.type);
```

Oltre ai metodi, puoi anche definire proprietà statiche con la parola chiave `static`.

In questo esempio, abbiamo una proprietà statica `numberOfPizzasSold`.

```js
class Pizza {
  static numberOfPizzasSold = 0;

  constructor(type) {
    this.type = type;
    Pizza.numberOfPizzasSold++;
  }
}
```

È statica perché non appartiene a nessuna particolare istanza di pizza, ma alla classe stessa.

Ha un valore iniziale di `0` ed è aggiornata ogni volta che viene creata una nuova istanza.

Se crei due istanze di pizza, il valore sarà aggiornato due volte:

```js
let pizza1 = new Pizza("Margherita");
let pizza2 = new Pizza("Neapolitan");
```

Per consultare il valore di una proprietà statica, devi solo usare la notazione a punto sulla classe stessa, dato che la proprietà appartiene alla classe.

```js
class Pizza {
  static numberOfPizzasSold = 0;

  constructor(type) {
    this.type = type;
    Pizza.numberOfPizzasSold++;
  }
}

let pizza1 = new Pizza("Margherita");
let pizza2 = new Pizza("Neapolitan");

console.log(Pizza.numberOfPizzasSold);
```

In questo caso, l'output è `2` perché sono state vendute due pizze.

Questi sono i fondamenti delle proprietà e dei metodi statici in JavaScript. Capire i membri statici è essenziale per creare classi riutilizzabili ed efficienti.

## --text--

Qual è la differenza principale tra un metodo statico e un metodo di istanza in JavaScript?

## --answers--

I metodi statici possono essere usati solo all'interno della classe, mentre i metodi di istanza possono essere usati all'esterno della classe.
