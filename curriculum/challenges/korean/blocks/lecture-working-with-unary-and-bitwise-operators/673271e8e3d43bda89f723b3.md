# --interactive--

단항 연산자는 하나의 피연산자에 작용하여 타입 변환, 값 조작, 특정 조건 확인 같은 작업을 수행합니다. 자주 쓰이는 단항 연산자와 그 작동 방식을 살펴보겠습니다.

단항 덧셈 연산자는 피연산자를 숫자로 변환합니다. 피연산자가 이미 숫자라면 변하지 않습니다.

:::interactive_editor

```js
const str = '42';
const strToNum = +str;

console.log(strToNum); // 42
console.log(typeof str); // string
console.log(typeof strToNum); // number
```

:::

단항 덧셈은 값이 숫자인지 확실히 하고 싶을 때 유용합니다. 예상했겠지만, 단항 부정 연산자도 있습니다. 이 연산자는 피연산자의 값을 부정합니다. 단항 덧셈과 비슷하게 작동하지만 부호를 반전시킵니다.

:::interactive_editor

```js
const str = '42';
const strToNegativeNum = -str;

console.log(strToNegativeNum); // -42
console.log(typeof str); // string
console.log(typeof strToNegativeNum); // number
```

:::

논리 NOT 연산자는 느낌표(`!`)로 나타내는 또 다른 단항 연산자입니다. 피연산자의 불리언 값을 반전시킵니다. 따라서 피연산자가 `true`이면 `false`가 되고, `false`이면 `true`가 됩니다.

:::interactive_editor

```js
let isOnline = true;
console.log(!isOnline); // false

let isOffline = false;
console.log(!isOffline); // true
```

:::

비트 단위 NOT 연산자는 덜 자주 쓰이는 단항 연산자입니다. 물결표 `~`로 나타내며, 숫자의 이진 표현을 반전시킵니다. 컴퓨터는 숫자를 이진 형식(1과 0)으로 저장합니다. `~` 연산자는 모든 비트를 뒤집어 1은 0으로, 0은 1로 바꿉니다. 이진수와 비트에 대해서는 앞으로 수업에서 더 배우게 됩니다.

:::interactive_editor

```js
const num = 5; // The binary for 5 is 00000101

console.log(~num); // -6
```

:::

이 예에서 `5`은 `-6`이 되었는데, `~`에 `5` 연산자를 적용하면 `- (5 + 1)`이 되고, 이는 2의 보수 표현법 때문에 `-6`와 같습니다. 2의 보수는 컴퓨터가 음수를 이진수로 나타내는 방법입니다. 비트 단위 NOT은 비트를 직접 조작하는 저수준 프로그래밍 작업이 아니라면 자주 쓰지 않을 것입니다.

`void` 키워드는 단항 연산자로, 표현식을 평가하고 `undefined`를 반환합니다.

:::interactive_editor

```js
const result = void (2 + 2);

console.log(result); // undefined
```

:::

`void`는 하이퍼링크에서 이동을 막기 위해 자주 사용됩니다:

```js
<a href="javascript:void(0);">Click Me</a>
```

마지막으로 이전 수업에서 배운 `typeof` 연산자가 있습니다. 이 연산자는 피연산자의 타입을 문자열로 반환합니다.

:::interactive_editor

```js
const value = 'Hello world';

console.log(typeof value); // string
```

:::
