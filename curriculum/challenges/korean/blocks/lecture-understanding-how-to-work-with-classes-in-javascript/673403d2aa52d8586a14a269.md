# --interactive--

프로그래밍에서 상속은 한 클래스가 다른 클래스의 속성과 메서드를 물려받도록 정의할 수 있게 합니다.

예를 들어, `Car`는 `Vehicle`의 한 종류이므로, 모든 종류의 차량에 공통적인 가장 일반적인 속성과 메서드를 가진 `Vehicle` 클래스를 정의한 다음, `Car` 클래스의 속성과 메서드를 상속받는 `Vehicle` 클래스를 정의할 수 있습니다.

이 "is a" 관계가 상속의 특징입니다. 이 예에서 `Car`는 `Vehicle`의 더 전문화된 형태입니다.

상속을 시작하려면 알아야 할 두 가지 주요 개념이 있습니다: 상위 클래스와 하위 클래스입니다.

상위 클래스는 다른 클래스의 청사진 역할을 하는 클래스입니다. 다른 클래스가 상속받는 속성과 메서드를 정의합니다.

하위 클래스는 다른 클래스의 속성과 메서드를 상속받는 클래스입니다. 하위 클래스는 새로운 속성과 메서드를 추가하여 상위 클래스의 기능을 확장할 수도 있습니다.

앞선 예에서 `Vehicle`는 `Car` 클래스의 상위 클래스이고, `Car` 클래스는 `Vehicle`의 하위 클래스입니다.

상위 클래스는 여러 하위 클래스를 가질 수 있습니다.

JavaScript에서는 상속을 구현하기 위해 `extends` 키워드를 사용합니다. 이 키워드는 한 클래스가 다른 클래스의 하위 클래스임을 나타냅니다.

이 예에서는 클래스 정의의 첫 줄만 있지만, `Car` 클래스가 `Vehicle` 클래스를 확장하므로 `Car`는 `Vehicle`로부터 상속받습니다.

```js
class Vehicle {
  // Implementation of Vehicle...
}

class Car extends Vehicle {
  // Implementation of Car...
}
```

`Vehicle` 클래스에 `brand`와 `year` 두 개의 속성이 있다고 가정해 봅시다. 업데이트된 코드에서 확인할 수 있습니다.

`Car`에는 `honk`라는 메서드가 있습니다. 하지만 `Car` 클래스에는 자체 생성자가 없습니다. 속성을 상속받는지 확인해 봅시다.

```js
class Vehicle {
  constructor(brand, year) {
    this.brand = brand;
    this.year = year;
  }
}

class Car extends Vehicle {
  honk() {
    console.log("Honk! Honk!");
  }
}
```

`Car`의 인스턴스를 정의하려면 `brand`와 `year` 두 인수를 전달해야 합니다. 이 경우 브랜드는 `freeCodeCamp Motors`이고 연도는 `2019`입니다.

```js
let myCar = new Car("freeCodeCamp Motors", 2019);
```

점 표기법으로 이 속성들에 접근하고 메서드를 호출하면:

```js
console.log(myCar.brand);
console.log(myCar.year);
myCar.honk();
```

전체 예시는 다음과 같습니다:

:::interactive_editor

```js
class Vehicle {
  constructor(brand, year) {
    this.brand = brand;
    this.year = year;
  }
}

class Car extends Vehicle {
  honk() {
    console.log("Honk! Honk!");
  }
}

let myCar = new Car("freeCodeCamp Motors", 2019);
console.log(myCar.brand);
console.log(myCar.year);
myCar.honk();
```

:::

출력이 올바른 것을 보면, 이 속성과 메서드는 `Car` 인스턴스에 정의되어 있으며, `Vehicle` 클래스에서 이 속성을 상속받아 코드 반복을 많이 줄였음을 알 수 있습니다.

이 예에서 하위 클래스는 추가 속성이 없었습니다. 그래서 생성자가 없고 메서드만 있었습니다.

추가 속성이 필요하면 생성자를 정의해야 합니다.

예를 살펴봅시다.

```js
class Vehicle {
  constructor(brand, year) {
    this.brand = brand;
    this.year = year;
  }
}

class Car extends Vehicle {
  constructor(brand, year, numDoors) {
    super(brand, year);
    this.numDoors = numDoors;
  }
}
```

이번에는 `numDoors` 하위 클래스에 `Car` 속성을 추가합니다.

업데이트된 코드에서 이 클래스에 생성자가 있는 것을 볼 수 있습니다. 생성자 내에서 `super()`를 호출하며 `brand`와 `year` 인수를 전달합니다.

`super()`는 상위 클래스의 생성자를 호출하므로 `super(brand, year)`를 사용하면 하위 클래스에서 상위 클래스의 속성을 정의하는 셈입니다. 이 용어도 사용할 수 있습니다 — 상위 클래스는 부모 클래스이고 하위 클래스는 자식 클래스입니다.

그 다음 `this.numDoors = numDoors` 줄은 `Car` 클래스에만 배타적으로 속하는 새로운 속성, 즉 문 개수(`numDoors`)를 정의합니다.

이것은 `Vehicle` 클래스를 확장하여 `Car` 인스턴스에만 배타적인 속성을 추가하는 방법입니다.

이것이 상속의 근본적인 장점 중 하나인 확장성의 예입니다.

이제 `Car` 클래스의 인스턴스를 생성하면 `brand`, `year`, `numDoors` 세 가지 속성을 가진 것을 볼 수 있습니다. 인스턴스를 생성할 때 필요한 인수도 전달해야 합니다.

```js
let myCar = new Car("freeCodeCamp Motors", 2019, 4);
```

`console.log()`로 이 속성들의 값을 출력하면:

```js
console.log(myCar.brand);
console.log(myCar.year);
console.log(myCar.numDoors);
```

전체 예시는 다음과 같습니다:

:::interactive_editor

```js
class Vehicle {
  constructor(brand, year) {
    this.brand = brand;
    this.year = year;
  }
}

class Car extends Vehicle {
  constructor(brand, year, numDoors) {
    super(brand, year);
    this.numDoors = numDoors;
  }
}

let myCar = new Car("freeCodeCamp Motors", 2019, 4);
console.log(myCar.brand);
console.log(myCar.year);
console.log(myCar.numDoors);
```

:::

상속의 주요 장점은 코드 재사용성, 모듈화, 확장성, 그리고 코드 구조 개선입니다.

계층 구조를 구현하면 상위 클래스에 이미 작성한 코드를 하위 클래스에서 재사용하여 반복을 피할 수 있습니다.

상속은 또한 복잡한 시스템을 계층 내 더 단순한 컴포넌트로 분해하여 모듈화를 촉진합니다.

또한 상위 클래스의 기능을 확장할 수 있어 개발 과정에서 요구사항 변화에 쉽게 적응하고 새로운 기능을 추가할 수 있습니다.

마지막으로 계층 구조는 코드를 더 이해하기 쉽고 유지보수하기 쉽게 만듭니다.

이것이 상속의 기본입니다. 상속이 어떻게 작동하는지 이해하면 잘 구조화되고 유지보수 가능하며 확장 가능한 객체지향 프로그램을 설계할 수 있습니다.
