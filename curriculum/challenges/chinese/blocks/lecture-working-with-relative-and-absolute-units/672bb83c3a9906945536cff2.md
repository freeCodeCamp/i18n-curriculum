# --interactive--

在 CSS 中，`vh` 和 `vw` 是相对于视口的单位，允许你根据浏览器窗口的维度调整元素大小。这些单位对于创建适应不同屏幕尺寸的响应式设计特别有用。

`vh` 代表“视口高度”，`1vh` 等于视口高度的 1%。

同样，`vw` 代表“viewport width”，`1vw` 等于视口宽度的 1%。 

这意味着如果你将一个元素的高度设置为 `100vh`，它将占据整个视口的高度，而不管设备的实际像素维度。

当你想要创建全屏版型或维护屏幕特定比例的元素时，这些单位特别方便。 

例如，你可能想使用它们来创建一个始终填满整个屏幕的英雄区（要与示例交互，你需要启用交互式编辑器）。 

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css" />

<section class="hero">
  <h1>100vh / 100vw Example</h1>
  <p>This section fills the entire browser window.</p>
</section>
```

```css
body {
  margin: 0;
  font-family: sans-serif;
  border: 5px dashed #333;
}

.hero {
  height: 100vh;
  width: 100vw;
  background-color: #add8e6; 
  padding: 2em;
}

.hero h1 {
  font-size: 2em;
  margin-bottom: 0.5em;
}

.hero p {
  font-size: 1em;
}
```

:::

此 CSS 确保英雄部分始终与视口大小完全相同，无论设备的屏幕尺寸如何。

`vh` 和 `vw` 单位也可以用于排版，以创建响应式文本大小。

启用交互式编辑器，尝试调整预览窗口的大小，以查看文本如何随视口大小缩放：

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css" />
<h1>Responsive Heading</h1>
<p>The h1 heading scales with the viewport width.</p>
```

```css
h1 {
  font-size: 5vw;
}
```

:::

这将把 `h1` 元素的字体大小设置为视口宽度的 5％，使文本能够随着浏览器窗口大小平滑缩放。

`vh` 和 `vw` 单位的一个优点是它们能够实时响应视口大小的变化。这意味着如果用户调整他们的浏览器窗口大小，使用这些单位设置尺寸的元素会相应调整，而无需重新加载页面。然而，明智地使用这些单位非常重要。例如，仅用 `vw` 单位设置字体大小，可能导致文本在窄屏幕上变得过小，或在宽屏幕上变得过大。 

另一个需要考虑的是，在移动设备上，当浏览器的地址栏出现或消失时，视口高度可能会变化，如果你大量使用 `vh` 单位，可能会导致意外的版型移动。

总之，`vh` 和 `vw` 单位是创建响应式版型和适应视口大小的元素的强大工具。它们对于全屏部分、维护宽高比以及创建平滑缩放的设计特别有用。然而，应当谨慎使用，并且通常与其他 CSS 技术结合使用，以确保在所有设备上获得最佳用户体验。
