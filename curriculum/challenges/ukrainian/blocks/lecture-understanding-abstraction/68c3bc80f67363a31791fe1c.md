# --description--

Тепер, коли ми розглянули інкапсуляцію, успадкування та поліморфізм, давайте обговоримо наступну ключову концепцію об’єктно-орієнтованого програмування – абстракцію.

Абстракція — це процес приховування складних деталей реалізації та показу лише основних функцій об’єкта або системи. Уявіть це як фокусування на тому, що щось робить, а не на тому, як воно це робить.

Абстракція не обмежується Python. Це концепція програмування, яку можна реалізувати у багатьох мовах, що підтримують об’єктно-орієнтоване програмування.

Щоб проілюструвати абстракцію, уявіть, що ви керуєте автомобілем. Коли ви сидите за кермом, ви взаємодієте з основними частинами, такими як кермо, важіль перемикання передач і педалі газу та гальма. Вам не потрібно знати складні деталі роботи двигуна, як працює коробка передач або фізику гальмівної системи, адже всі ці речі — це складні деталі реалізації.

Ось це і є абстракція! Вона надає вам спрощений інтерфейс для взаємодії зі складною системою.

У випадку з автомобілем спрощеним інтерфейсом є кермо, гальма та педаль газу, а складною системою — сам автомобіль.

Щодо того, як Python реалізує абстракцію, він робить це через модуль `abc`.

Цей модуль надає клас `ABC` (що означає «абстрактний базовий клас») і декоратор `@abstractmethod`.

`ABC` — це клас, від якого слід успадковуватися, але без можливості створювати безпосередні об’єкти цього класу. Він визначає загальний інтерфейс методів і властивостей, які мають реалізувати його підкласи.

З іншого боку, абстрактний метод — це метод, оголошений в абстрактному базовому класі (ABC) із використанням декоратора `@abstractmethod`. Він може не мати реалізації або мати базову реалізацію за замовчуванням. Проте будь-який підклас повинен заміщати цей метод, щоб вважатися конкретним і придатним для створення екземплярів, навіть якщо реалізація за замовчуванням надана.

Ось базовий синтаксис абстрактного класу в Python:

```py
from abc import ABC, abstractmethod

# Define an abstract base class
class AbstractClass(ABC):
    @abstractmethod
    def abstract_method(self):
        pass

# Concrete subclass that implements the abstract method
class ConcreteClassOne(AbstractClass):
    def abstract_method(self):
        print('Implementation in ConcreteClassOne')

# Another concrete subclass
class ConcreteClassTwo(AbstractClass):
    def abstract_method(self):
        print('Implementation in ConcreteClassTwo')
```

Ось базовий приклад:

```py
from abc import ABC, abstractmethod

class Animal(ABC): # Inherits from abstract base class
   @abstractmethod # Abstract method decorator
   def make_sound(self):  # The method subclasses must override
       pass

# Concrete class that will override the abstract method
class Dog(Animal):
   def make_sound(self):
       print('Woof!')

# Another concrete class that will override the abstract method
class Cat(Animal):
   def make_sound(self):
       print('Meow!')

# Another concrete class that will override the abstract method
class Monkey(Animal):
   def make_sound(self):
       print('Ooh ooh aah aah!')

# Create instances of each concrete class
animals = [Dog(), Cat(), Monkey()]

# Loop through the instances to call the make_sound method
for animal in animals:
   animal.make_sound()

# Output:
# Woof!
# Meow!
# Ooh ooh aah aah!
```

У цьому прикладі:

- Ми імпортуємо клас `ABC` і `abstractmethod` з модуля `abc`.
- Потім створюємо клас `Animal`, який успадковується від `ABC`, і оголошуємо в ньому абстрактний метод `make_sound`, який кожен підклас `Animal` має заміщати.
- Створюємо конкретні класи `Dog`, `Cat` і `Monkey`, які мають заміщати абстрактний метод `make_sound`.
- Створюємо екземпляри конкретних класів і викликаємо їхній метод `make_sound`, щоб показати, як кожен із них реалізує абстрактний метод `make_sound` по-своєму.

Пам’ятайте, що ви не можете створити екземпляр класу Animal. Ось що станеться, якщо ви спробуєте це зробити:

```py
dog = Animal() 
# TypeError: Can't instantiate abstract class Animal 
# without an implementation for abstract method 'make_sound'
```

Те саме правило стосується підкласів, які не реалізують абстрактний метод. Навіть якщо ви визначите підклас Animal, ви не зможете створити його екземпляр, доки він не замістить make_sound. Ось приклад, який це демонструє:

```py
class Bird(Animal):
    pass

bird = Bird()
# TypeError: Can't instantiate abstract class Bird 
# without an implementation for abstract method 'make_sound'
```

Ось ще один приклад, цього разу з атрибутом екземпляра, який можна передати екземплярам конкретних класів:

```py
from abc import ABC, abstractmethod

# The blueprint for any toy that can speak
class TalkingToy(ABC):
   def __init__(self, name):
       self.name = name
   @abstractmethod
   def speak(self):
       pass

class RobotToy(TalkingToy):
   def speak(self):
       print(f'{self.name} says beep boop! I am a robot!')

class TeddyBearToy(TalkingToy):
   def speak(self):
       print(f"{self.name} says hug me! I'm cuddly!")

class DinosaurToy(TalkingToy):
   def speak(self):
       print(f'{self.name} says ROOOOAR!')

# Create toys
rusty = RobotToy('Rusty')
fluffy = TeddyBearToy('Fluffy')
rex = DinosaurToy('Rex')

toys = [rusty, fluffy, rex]
for toy in toys:
   toy.speak()

# Output:
# Rusty says beep boop! I am a robot!
# Fluffy says hug me! I'm cuddly!
# Rex says ROOOOAR!
```

У цьому прикладі:

- У нас є абстрактний базовий клас `TalkingToy`, який визначає шаблон для будь-якої іграшки, що може говорити.
- Підкласи `RobotToy`, `TeddyBearToy` і `DinosaurToy` реалізують метод `speak` по-своєму.
- Коли ми створюємо екземпляри цих підкласів і викликаємо метод `speak`, кожна іграшка говорить унікальним способом.

Підсумовуючи, абстракція в Python спрощує складні системи, підвищуючи повторне використання.

Ви побачили, як можна повторно використовувати один метод з абстрактного класу в кількох підкласах, змушуючи кожен підклас надавати свою конкретну поведінку.

Такий підхід допомагає тримати ваш код організованим, гнучким і легшим у підтримці, особливо коли ваш застосунок зростає.
