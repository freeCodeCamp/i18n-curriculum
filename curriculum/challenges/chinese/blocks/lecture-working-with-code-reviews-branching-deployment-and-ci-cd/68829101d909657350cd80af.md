# --description--

GitHub Pages 是部署静态站点或不需要后端服务器来处理逻辑的应用程序的一个非常可靠的选项。也就是说，完全在客户端或用户浏览器中运行的应用程序，可以完全部署在这个平台上。

此外，GitHub Pages 对开源项目完全免费。将你的项目开源有很多充分的理由，而 Pages 又增加了一个理由。

但是你到底如何部署到 GitHub Pages？这取决于你的应用如何工作。

让我们从一个基本的超文本标记语言页面开始。如果你愿意，可以将下面的内容添加到你现有的仓库中：

```html
<!DOCTYPE html>
<html>
  <head>
    <title>freeCodeCamp</title>
    <link rel="stylesheet" href="./style.css" />
  </head>
  <body>
    <h1>Welcome to freeCodeCamp's Lessons!</h1>
    <p>Let's have some fun learning about Git and GitHub!</p>
  </body>
</html>
```

现在我们需要访问我们的仓库并点击 "Settings" 标签（页）以进入仓库设置。然后我们需要从侧边栏选择 "Pages"。

最初只有少数几个选项，但随着我们开始配置设置，这将会改变。首先，我们需要告诉 GitHub 从哪里部署。

我们想保持为 "Deploy from a branch"，因为我们没有使用任何框架，也没有配置任何 GitHub 操作。你将在即将到来的课程中了解更多关于它们的内容。

接下来，将我们部署的分支设置为 `main`，以部署该分支上的内容。

然后你可以选择从你的项目根文件夹或 `docs` 文件夹部署。这个特色被像 freeCodeCamp 这样的项目使用——我们过去将文档与应用程序打包在一起，因此我们可以通过 `docs` 文件夹仅将文档部署到 GitHub 页面。现在，保持为 `/`（根）文件夹，然后点击此下拉菜单旁边的“储存”按钮以储存你的更改。

一旦你储存后，你将看到一个新建的输入，用于配置自定义域。如果你已经有了，并且想要使用它，你可以点击链接查看文档来操作。然而，对于本课，我们将保持此项为空。

你还可以看到你的新建 GitHub Pages 站点已部署到的域。没有自定义域时，默认是 `<username>.github.io`，其中 `<username>` 是你的账户名（如果你是从组织拥有的仓库部署，则是组织账户名）。仓库将通过匹配仓库名称的路径提供服务，例如 `<username>.github.io/<repo-name>`。如果你访问该 URL，你将看到你的站点！

现在，当你将新更改推送到你的 `main` 分支时，GitHub Pages 会自动使用最新更改更新我们已部署的网站。此过程可能需要几分钟，具体取决于服务器装载情况——你可能需要进行强制刷新以清除缓存。

至此，我们已成功将一个基本的网页应用部署到 GitHub Pages。你也可以在你的 Pages 部署中包含 CSS 和 JavaScript 文件，这样你就可以构建美观且交互性强的网站。一旦你学会了像 React 或 Angular 这样的框架，你甚至可以将它们部署到 GitHub Pages 上！
