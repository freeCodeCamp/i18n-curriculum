# --description--

现在你已经构建了一个基础的 Express 应用，是时候更深入了解路由及其工作原理了。

在 Express 中，路由是你定义 Web 应用端点的方式。你可以把路由看作是告诉应用当用户访问特定 URL 时如何响应的规则。

让我们再看一下这个简单的 Express 应用：

```javascript
const express = require("express");
const app = express();
const port = 3000;

app.get("/", (req, res) => {
  res.send("Hello World!");
});

app.listen(port, () => {
  console.log(`Example app listening on port ${port}`);
});
```

在这个例子中，应用为根 URL（`/`）定义了一个路由。当客户发送一个 GET 请求到这个路径时，服务器会响应文本 `Hello World!`。然后应用开始监听端口 `3000` 上的传入请求。

但还有其他路由方法。

Express 允许你通过不同的路由方法来处理不同类型的超文本传输协议请求。最常见的有：

* `app.get()` – 获取数据
    
* `app.post()` – 创建新数据
    
* `app.put()` – 更新已有数据
    
* `app.delete()` – 删除数据
    

每个方法定义了你的应用应该如何响应特定路由上的特定超文本传输协议请求。

这里有几个简单的例子：

```javascript
app.post("/submit", (req, res) => {
  res.send("Form submitted!");
});

app.put("/update", (req, res) => {
  res.send("Item updated!");
});

app.delete("/delete", (req, res) => {
  res.send("Item deleted!");
});
```

在每种情况下，Express 都监听带有特定超文本传输协议方法和 URL 路径的请求，然后运行相应的处理程序函数，将响应发送回客户。

你可以使用 Postman 或 `curl` 等工具测试这些路由，这些工具允许你向服务器发送不同类型的超文本传输协议请求。

这就是 Express 中的路由——它就是定义你的应用如何响应不同请求。
