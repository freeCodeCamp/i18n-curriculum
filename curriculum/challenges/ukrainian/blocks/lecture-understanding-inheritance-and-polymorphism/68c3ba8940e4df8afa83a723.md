# --description--

Успадкування — це наступна ключова концепція об’єктно-орієнтованого програмування (ООП), яку ми розглянемо.

Давайте глибше зануримося в цю концепцію і дізнаємося, як вона дозволяє писати повторно використовуваний код.

За допомогою успадкування підклас (або дочірній клас) може використовувати атрибути та методи базового класу (або батьківського класу). Це дозволяє повторно використовувати код, створювати чіткі ієрархії класів і налаштовувати поведінку без переписування всього заново. Ви можете налаштовувати, розширюючи існуючі методи або заміщаючи їх у дочірньому класі.

Ось базовий синтаксис для успадкування:

```py
class Parent:
    # Parent attributes and methods

class Child(Parent):
    # Child inherits, extends, and/or overrides where necessary
```

Щоб клас `Child` успадкувався від класу `Parent`, потрібно передати Батьківський клас Дочірньому.

Цей стиль називається одиночним успадкуванням, оскільки дочірній клас успадковується від рівно одного батьківського класу.

Ось приклад:

```py
class Animal:
    def __init__(self, name):
        self.name = name

    def sound(self):
        return f'{self.name} makes a sound'

class Dog(Animal):
    bark = 'woof! woof!! woof!!!'

jack = Dog('Jack')
print(jack.sound())  # Jack makes a sound
print(jack.bark)  # woof! woof!! woof!!!
```

Ви можете побачити, що ми можемо повторно використовувати атрибут `self.name` і метод `sound()` з батьківського класу `Animal` у дочірньому класі `Dog`.

Давайте замістимо метод `sound()` з батьківського класу `Animal` у дочірньому класі `Dog`, щоб `sound()` міг використовувати змінну класу `bark`:

```py
class Animal:
    def __init__(self, name):
        self.name = name

    def sound(self):
        return f'{self.name} makes a sound.'

class Dog(Animal):
    bark = 'woof! woof!! woof!!!'

    # Override sound() to use bark class variable
    def sound(self):
        return f'{self.name} barks {self.bark}'

jack = Dog('Jack')
print(jack.sound())  # Jack barks woof! woof!! woof!!!
```

Якщо ви хочете зберегти повернене значення `sound()` і додати змінну класу bark пізніше, ви можете розширити `sound()`, використовуючи функцію `super()`:

```py
class Animal:
    def __init__(self, name):
        self.name = name

    def sound(self):
        return f'{self.name} makes a sound'

class Dog(Animal):
    bark = 'woof! woof!! woof!!!'

    # Call Animal.sound(), then append bark
    def sound(self):
        base = super().sound()
        return f'{base}, then {self.name} barks {self.bark}'

jack = Dog('Jack')
print(jack.sound())  # Jack makes a sound, then Jack barks woof! woof!! woof!!!
```

У цьому прикладі `base` — це результат виклику методу `sound()` з класу `Animal`, а потім ми додаємо специфічний звук класу `Dog`. Таким чином, ви можете розширити функціональність батьківського класу `Animal`, зберігаючи при цьому його оригінальну поведінку.

Існує також множинне успадкування, коли дочірній клас може успадковувати від більше ніж одного батьківського класу.

Ось базовий синтаксис множинного успадкування:

```py
class Parent:
    # Attributes and methods for Parent

class Child:
    # Attributes and methods for Child

class GrandChild(Parent, Child):
    # GrandChild inherits from both Parent and Child
    # GrandChild can combine or override behavior from each
```

Простий спосіб продемонструвати множинне успадкування — це жаба, яка може і ходити по суші, і плавати у воді:

```py
class Walker:
    def walk(self):
        return 'I can walk on land'

class Swimmer:
    def swim(self):
        return 'I can swim in water'

# Amphibian inherits from both Walker and Swimmer
class Amphibian(Walker, Swimmer):
    def __init__(self, name):
        self.name = name

    def introduce(self):
        return f"I'm {self.name} the frog. {self.walk()} and {self.swim()}."

frog = Amphibian('Freddy')
print(frog.introduce())
# Output: I'm Freddy the frog. I can walk on land and I can swim in water.
```
