# --interactive--

Lernen wir etwas über Quantifizierer in regulären Ausdrücken.

Stellen Sie sich vor, Sie möchten einen vierstelligen Identifikationscode finden. Sie wissen, dass Sie die `\d` Zeichenklasse verwenden können, also könnten Sie diese viermal schreiben. Und um keine zusätzlichen Zeichen zuzulassen, fügen Sie sowohl den Start- als auch den Endanker hinzu:

```js
const regex = /^\d\d\d\d$/;
```

Und das funktioniert – es wird genau vier numerische Zeichen finden. Aber anstatt die gleiche Klasse mehrfach zu schreiben, können Sie einen Quantifizierer verwenden.

Quantifizierer werden durch geschweifte Klammern definiert, die eine oder zwei Zahlen enthalten. Verwenden wir einen Quantifizierer in unserem Muster:

```js
const regex = /^\d{4}$/;
```

Beachten Sie, dass unser Quantifizierer nur die Zahl `4` enthält. Diese Syntax bedeutet „das vorherige Zeichen genau viermal finden“. Sehen wir, wie sich das verhält:

:::interactive_editor

```js
const regex = /^\d{4}$/;
console.log(regex.test("123")); // false
console.log(regex.test("1234")); // true
console.log(regex.test("12345")); // false
console.log(regex.test("123456")); // false
console.log(regex.test("1234567")); // false
```

:::

Das Muster findet nur den String mit genau vier Ziffern, weil wir die Anker verwendet haben und unser Quantifizierer nur genau vier Ziffern erlaubt. Aber vielleicht muss der Identifikationscode nur mindestens vier Ziffern haben.

Um vier oder mehr Ziffern zu erlauben, fügen Sie ein Komma nach der Zahl in Ihrem Quantifizierer hinzu:

```js
const regex = /^\d{4,}$/;
```

Jetzt erlaubt unsere Syntax, dass das Muster vier oder mehr Ziffern findet. Testen wir es:

:::interactive_editor

```js
const regex = /^\d{4,}$/;
console.log(regex.test("123")); // false
console.log(regex.test("1234")); // true
console.log(regex.test("12345")); // true
console.log(regex.test("123456")); // true
console.log(regex.test("1234567")); // true
```

:::

Ein siebenstelliger Identifikator ist ziemlich lang. Diese Identifikatoren sollten höchstens 6 Ziffern und mindestens 4 Ziffern haben. Um das zu erreichen, können Sie Ihrem Quantifizierer nach dem Komma eine zweite Zahl hinzufügen:

```js
const regex = /^\d{4,6}$/;
```

Und jetzt findet unser Muster den siebenstelligen Identifikator nicht mehr, weil er größer als unser sechsstelliges Maximum ist.

:::interactive_editor

```js
const regex = /^\d{4,6}$/;
console.log(regex.test("123")); // false
console.log(regex.test("1234")); // true
console.log(regex.test("12345")); // true
console.log(regex.test("123456")); // true
console.log(regex.test("1234567")); // false
```

:::

Beachten Sie, dass Sie diese Syntax nicht verwenden können, um nur ein Maximum festzulegen – Sie müssen immer ein Minimum angeben. Aber wenn Sie das Minimum auf `1` setzen, erreichen Sie effektiv dasselbe Ergebnis.

Wir haben aktualisierte Anforderungen von unseren Nutzenden erhalten. Identifikatoren können jetzt optional mit einem Buchstaben beginnen. Wir kennen bereits die Zeichenklasse dafür, also fügen wir sie unserem regulären Ausdruck hinzu:

```js
const regex = /^[a-zA-Z]\d{4,6}$/;
```

Aber jetzt verlangen wir zwingend einen Buchstaben. Wie können wir ihn optional machen?

Sie könnten die Quantifizierer-Syntax mit `0` als Minimum und `1` als Maximum verwenden:

```js
const regex = /^[a-zA-Z]{0,1}\d{4,6}$/;
```

Aber es gibt tatsächlich einen speziellen Kurzquantifizierer für ein einzelnes optionales Zeichen – das Fragezeichen (`?`). Ersetzen wir unseren Quantifizierer durch das Fragezeichen:

```js
const regex = /^[a-zA-Z]?\d{4,6}$/;
```

Wir sollten das Ergebnis validieren:

:::interactive_editor

```js
const regex = /^[a-zA-Z]?\d{4,6}$/;
console.log(regex.test("123")); // false
console.log(regex.test("a1234")); // true
console.log(regex.test("12345")); // true
console.log(regex.test("az12345")); // false
console.log(regex.test("X123456")); // true
console.log(regex.test("1234567")); // false
```

:::

Unser Muster erlaubt jetzt einen einzelnen optionalen Buchstaben, gefolgt von vier bis sechs Ziffern.

Leider haben wir gerade festgestellt, dass wir die Anforderungen falsch gelesen haben. Wir müssen beliebig viele Buchstaben vor den Zahlen erlauben. Wir können unseren Quantifizierer mit einem Minimum von `0` und ohne definiertes Maximum verwenden:

```js
const regex = /^[a-zA-Z]{0,}\d{4,6}$/;
```

Aber unser Muster wird wieder lang. Zum Glück gibt es eine weitere Kurzschreibweise für „das vorherige Zeichen null oder mehrmals finden“ – das Sternchen (`*`). Ersetzen wir unseren Quantifizierer durch dieses Symbol im Muster und testen es:

:::interactive_editor

```js
const regex = /^[a-zA-Z]*\d{4,6}$/;
console.log(regex.test("123")); // false
console.log(regex.test("a1234")); // true
console.log(regex.test("12345")); // true
console.log(regex.test("az12345")); // true
console.log(regex.test("X123456")); // true
console.log(regex.test("1234567")); // false
```

:::

Jetzt finden wir erfolgreich jeden Identifikator mit null oder mehr Buchstaben, gefolgt von vier bis sechs Zahlen. Aber es stellt sich heraus, dass das unser System zum Absturz bringt – wir müssen tatsächlich mindestens einen Buchstaben verlangen.

Wieder könnten wir einen Quantifizierer mit einem Minimum von eins und ohne definiertes Maximum verwenden, oder wir nutzen eine weitere spezielle Syntax – das Pluszeichen (`+`):

:::interactive_editor

```js
const regex = /^[a-zA-Z]+\d{4,6}$/;
console.log(regex.test("123")); // false
console.log(regex.test("a1234")); // true
console.log(regex.test("12345")); // false
console.log(regex.test("az12345")); // true
console.log(regex.test("X123456")); // true
console.log(regex.test("1234567")); // false
```

:::

Jetzt schlagen Identifikatoren fehl, die nicht mit mindestens einem Buchstaben beginnen, egal wie viele Zahlen folgen.

Sie können Quantifizierer verwenden, um die Kürze und Lesbarkeit Ihrer regulären Ausdrücke erheblich zu verbessern.
