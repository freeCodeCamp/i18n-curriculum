# --description--

在上一课中，我们了解了 Fetch API 是什么以及如何使用它。在本课中，我们将讨论 Fetch API 的 `GET`、`POST`、`PUT` 和 `DELETE` HTTP 方法。

让我们从最常见的 HTTP 方法开始，那就是 `GET` 方法。它用于从服务器检索数据。当你使用 `fetch()` 而不指定方法时，它默认是 `GET`。

```js
fetch('https://api.example.com/data')
```

在这段代码中，我们正在向 `GET` 发起一个 `https://api.example.com/data` 请求。请注意，你不能直接使用这些数据；你必须将响应转换为 JSON 格式。只有这样，你才能在项目中的任何地方使用它。下面是如何操作的示例：

```js
fetch('https://api.example.com/data')
  .then(response => response.json())
  .then(data => console.log(data))
```

在这段代码中，来自 Fetch API 的响应是一个 promise，我们使用 `.then` 处理程序将响应转换为 JSON 格式。在之前的课程中，你已经学过 `Promise` 是一个对象，表现了异步进程最终完成（或失败）及其值。

promise 的值在创建时是未知的。只有当异步进程完成时才知道。当我们将两个 `.then` 处理程序链式调用到 fetch 调用时，这就是所谓的 promise 链式调用，下一课将讲解。

到目前为止，我们一直在从服务器检索资源。但你知道吗，我们也可以向服务器发送数据？`POST` 方法用于向服务器发送数据以创建资源。下面是一个使用 `POST` 方法在服务器上创建资源的示例：

```js
fetch('https://api.example.com/users', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({
    name: 'John Doe',
    email: 'john@example.com'
  })
})
```

在这个示例中，我们发送了一个 `POST` 请求来创建一个新用户。我们将方法指定为 `POST`，设置了适当的头部，并包含了我们想发送的数据的 `body`。body 需要是一个字符串，所以我们使用 `JSON.stringify()` 将对象转换为 JSON 字符串。

`PUT` 方法用于更新服务器上的现有资源。示例如下：

```js
fetch('https://api.example.com/users/45', {
  method: 'PUT',
  headers: {
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({
    name: 'John Smith',
    email: 'john@example.com'
  })
})
```

在这个示例中，仔细看 URL，你会看到末尾有一个 `45`。这通常用作唯一 ID 来标识我们要更新的数据。代码中使用了 `PUT` 方法，并且指定了作为 `body` 的数据，用于更新所标识的数据。

`DELETE` 方法用于从服务器删除资源。示例如下：

```js
fetch('https://api.example.com/users/45', {
  method: 'DELETE',
})
```

在这段代码中，我们包含了 `DELETE` 方法，并在 URL 末尾添加了一个 ID 来标识需要删除的数据。
