# --description--

문자열에서 패턴의 모든 항목을 일치시키거나 교체하는 방법을 배워보겠습니다.

이전에 `replace()`와 `match()` 메서드, 그리고 전역 `g` 수식어에 대해 배웠습니다. 이제 그 지식을 결합해 문자열 내 모든 패턴을 처리할 수 있습니다. 원래의 일치 코드부터 다시 살펴보겠습니다:

```js
const regex = /freeCodeCamp/;
const match = "freeCodeCamp".match(regex);
console.log(match);
```

그리고 결과로 나온 일치 객체입니다:

```js
// [
//   'freeCodeCamp',
//   index: 0,
//   input: 'freeCodeCamp',
//   groups: undefined
// ]
```

그런데 만약 `freecodecamp`가 여러 번 나타나는 문자열이 있다면 어떻게 될까요? `match()`가 어떻게 동작하는지 살펴보겠습니다. 비교를 위해 이전 `replace()` 예제도 함께 넣어보겠습니다:

```js
const regex = /freecodecamp/;
const str = "freecodecamp is the best we love freecodecamp";
const matched = str.match(regex);
const replaced = str.replace(regex, "freeCodeCamp");
console.log(matched);
console.log(replaced);
```

결과는 다음과 같습니다:

```js
// [
//   'freecodecamp',
//   index: 0,
//   input: 'freecodecamp is the best we love freecodecamp',
//   groups: undefined
// ]
// freeCodeCamp is the best we love freecodecamp
```

어머나! `match()`는 첫 번째 일치 항목만 반환했고, `replace()`는 첫 번째 일치 항목만 교체했습니다. 이는 기본적으로 `match()`와 `replace()`가 첫 번째 패턴 항목에만 작동하기 때문입니다.

다행히도 정규 표현식에 전역 수식어를 사용하면 이 문제를 피할 수 있습니다. 우리 코드에 추가해 보겠습니다:

```js
const regex = /freecodecamp/g;
const str = "freecodecamp is the best we love freecodecamp";
const matched = str.match(regex);
const replaced = str.replace(regex, "freeCodeCamp");
console.log(matched);
console.log(replaced);
```

결과를 확인해 보겠습니다:

```js
// [ 'freecodecamp', 'freecodecamp' ]
// freeCodeCamp is the best we love freeCodeCamp
```

잘 됐네요! replace 호출이 소문자 `freecodecamp` 문자열 모두를 교체했고, `match()` 메서드는 두 개 모두 일치시켰습니다.

흥미로운 점은 전역 수식어를 `match()`와 함께 사용하면, 일치 배열에 포함되는 캡처 그룹과 문자열 인덱스에 관한 추가 정보가 사라진다는 것입니다.

다행히도 2019년 ECMAScript 업데이트에서 두 가지 새로운 메서드가 도입되었습니다: `matchAll()`와 `replaceAll()`. 이들은 단일 메서드와 마찬가지로 문자열이나 정규 표현식을 인수로 받고, `replaceAll()`는 두 번째 인수로 교체할 문자열도 받습니다.

하지만 이전 메서드와 달리 `replaceAll()`와 `matchAll()`는 전역 수식어가 없는 정규 표현식을 인수로 주면 오류를 발생시킵니다. 이 새로운 메서드를 사용하도록 코드를 업데이트해 보겠습니다:

```js
const pattern = "freecodecamp";
const str = "freecodecamp is the best we love freecodecamp";
const matched = str.matchAll(pattern);
const replaced = str.replaceAll(pattern, "freeCodeCamp");
console.log(matched);
console.log(replaced);
```

결과는 다음과 같습니다:

```js
// {}
// freeCodeCamp is the best we love freeCodeCamp
```

좋은 소식입니다! `replaceAll()`가 정확히 원하는 대로 작동했습니다 – 소문자 `freecodecamp`의 모든 항목을 올바른 카멜 케이스 버전으로 교체했습니다.

그런데 빈 객체는 무엇일까요? `matchAll()`는 freeCodeCamp 콘솔이 처리할 준비가 되어 있지 않은 `Iterator`라는 특별한 유형의 객체를 반환합니다.

브라우저 콘솔을 살펴보면 `Iterator`에 `next()` 메서드가 있어 다음 값을 얻기 위해 호출할 수 있습니다:

```js
// RegExpStringIterator {  }
//  <prototype>: RegExp String Iterator {
//    next: ƒ next(),
//    Symbol(Symbol.toStringTag): "RegExp String Iterator"
//    <prototype>: Object { ... }
//  }
```

`matched.next()`를 호출하고 결과를 기록해 보겠습니다:

```js
// {
//   "done": false,
//   "value": [
//     0: "freecodecamp"
//     groups: undefined
//     index: 0
//     input: "freecodecamp is the best we love freecodecamp"
//   ]
// }
```

여기 일치 배열이 있습니다! `next()`는 두 가지 값을 가진 객체를 반환하는데, `done`는 반복기에 더 많은 요소가 있을 때 `false`이고, `value`는 방금 반복한 값입니다. 그러니 한 번 더 호출해 보겠습니다:

```js
const regex = /freecodecamp/g;
const str = "freecodecamp is the best we love freecodecamp";
const matched = str.matchAll(regex);
const replaced = str.replaceAll(regex, "freeCodeCamp");
console.log(matched);
console.log(replaced);
console.log(matched.next());
console.log(matched.next());
```

```js
// {
//   "done": false,
//   "value": Array [ "freecodecamp"]
// }
//
// {
//   "done": false,
//   "value": Array [ "freecodecamp"]
// }
```

왜 `done`가 아직도 `false`라고 나올까요? 배열에는 두 개의 일치 항목만 있어야 하지 않나요? 세 번째 호출해 결과를 확인해 보겠습니다:

```js
// {
//   "done": false,
//   "value": Array [ "freecodecamp"]
// }
//
// {
//   "done": false,
//   "value": Array [ "freecodecamp"]
// }
//
// {
//   "done": true,
//   "value": undefined
// }
```

`done`는 마침내 `true`가 되었지만, 왜 그 값이 `undefined`일까요? 사실 `matchAll()` 반복기는 게으릅니다. 한꺼번에 모든 일치 항목을 찾지 않습니다. `next()`를 호출할 때마다 일치 항목을 찾습니다.

일치 항목을 찾는 한 `done`가 아니고, 일치 항목을 찾지 못해 `undefined`를 반환하면 `done`가 됩니다. 불편할 수 있지만, 정규 표현식이 계산 비용이 많이 들 때는 매우 유용할 수 있습니다.

우리 예제처럼 덜 복잡한 경우에는 이 기능을 건너뛰고 한 번에 모든 일치 항목을 배열로 추출할 수 있습니다. 이는 `Array.from()`를 호출하고 반복기를 인수로 전달하면 됩니다.

코드를 업데이트해 보겠습니다 – `replaceAll` 호출은 잘 작동하니 정리하겠습니다:

```js
const regex = /freecodecamp/g;
const str = "freecodecamp is the best we love freecodecamp";
const matched = str.matchAll(regex);
console.log(Array.from(matched));
```

마침내 일치 항목 배열을 얻었습니다:

```js
// [
//   'freecodecamp',
//   index: 0,
//   input: 'freecodecamp is the best we love freecodecamp',
//   groups: undefined
// ]
//
// [
//   'freecodecamp',
//   index: 33,
//   input: 'freecodecamp is the best we love freecodecamp',
//   groups: undefined
// ]
```

이 강력한 메서드들은 성능이나 가독성을 희생하지 않고 문자열에서 데이터를 조작하고 추출하는 데 도움을 줍니다.
