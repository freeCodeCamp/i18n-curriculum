# --description--

No Express, o objeto `res` (response) é usado para enviar respostas de volta ao cliente após processar requisições recebidas. O Express oferece vários métodos embutidos para enviar diferentes tipos de respostas. Por exemplo, você pode enviar texto simples, dados JSON, redirecionar um usuário para outra URL ou renderizar um template.

Um dos métodos de resposta mais usados é `res.send()`:

```javascript
app.get("/text", (req, res) => {
  res.send("Hello, World!");
});
```

O método `res.send()` envia uma resposta de volta ao cliente. Ele pode enviar texto simples, HTML, JSON ou até dados binários. Neste exemplo, quando um cliente visita `/text`, o servidor responde com o texto `Hello, World!`.

Outro método de resposta comumente usado é `res.json()`.

Esse método envia uma resposta JSON para o cliente. É especialmente útil ao construir APIs, onde você frequentemente precisa retornar dados estruturados como objetos ou arrays. O Express converte automaticamente um objeto JavaScript em uma string formatada em JSON:

```javascript
app.get("/user", (req, res) => {
  res.json({ name: "John Doe", age: 30 });
});
```

Se um cliente enviar uma requisição para `/user`, o servidor responde com o seguinte:

```json
{
  "name": "John Doe",
  "age": 30
}
```

Quando você precisa controlar o código de status HTTP de uma resposta, pode usar `res.status()`. Esse método define o código de status e normalmente é encadeado com outro método de resposta como `res.send()` ou `res.json()`:

```javascript
app.get("/notfound", (req, res) => {
  res.status(404).send("Not Found");
});
```

Aqui, o servidor define o código de status para `404` e envia a mensagem `Not Found` de volta ao cliente.

Se você quiser enviar o usuário para uma URL diferente, pode usar `res.redirect()`:

```javascript
app.get("/oldpage", (req, res) => {
  res.redirect("/newpage");
});
```

Quando um cliente visita `/oldpage`, ele é automaticamente redirecionado para `/newpage`.

Por fim, `res.render()` é usado ao trabalhar com motores de template como Pug, EJS ou Handlebars. Ele renderiza um template de visualização e envia o HTML resultante para o cliente:

```javascript
app.get("/profile", (req, res) => {
  res.render("profile", { username: "john_doe" });
});
```

Neste exemplo, o servidor renderiza uma visualização `profile` e passa uma variável `username` para ela, que pode então ser exibida no HTML.

Esses métodos de resposta permitem que você retorne diferentes tipos de dados com base na natureza da requisição. Seja enviando texto, dados JSON ou redirecionando o cliente, o Express tem um conjunto flexível de ferramentas para lidar com isso.
