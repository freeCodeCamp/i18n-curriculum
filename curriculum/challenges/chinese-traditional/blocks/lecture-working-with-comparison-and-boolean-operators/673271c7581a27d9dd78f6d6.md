# --interactive--

在之前的課程中，你首次接觸到布林值的概念，但在本課程中，我們將更深入探討布林值的運作方式以及等號運算子和不等號運算子的運作方式。

布林值是一種只有 `true` 和 `false` 值的資料型別。它們很有用，因為你可以根據某些條件來執行某些操作。當你想要評估某件事是否應該發生時，布林值是不可或缺的，例如決定某人是否可以存取你應用程式中的某個特性。以下是一個將值 `true` 指定給名為 `isOldEnoughToDrive` 的變數的範例：

:::interactive_editor

```js
let isOldEnoughToDrive = true;

console.log(isOldEnoughToDrive); // true
```

:::

你可以像這樣在條件式中使用這個變數：

:::interactive_editor

```js
let isOldEnoughToDrive = true;

if (isOldEnoughToDrive) {
 console.log("You're old enough to drive"); // You're old enough to drive
} else {
 console.log("Sorry, you are not old enough to drive");
}
```

:::

條件式可幫助你根據條件在你的程式碼中做出決策。此範例使用所謂的 `if/else` 述語。

如果 `isOldEnoughToDrive` 是 `true`，則句子 `You're old enough to drive` 將會被記錄到主控台。否則，如果 `isOldEnoughToDrive` 是 `false`，則句子 `Sorry, you are not old enough to drive` 將會被記錄到主控台。由於 `isOldEnoughToDrive` 變數被設定為 `true`，第一個句子將會被記錄到主控台。你將在未來的課程中學習更多關於 `if/else` 述語。

要比較兩個值，你可以使用等號運算子或嚴格等號運算子。比較的結果將會是 `true` 或 `false` 的布林值。以下是使用等號運算子比較字串和數字的範例。等號運算子是由雙等號符號（`==`）表述。

:::interactive_editor

```js
console.log(5 == "5"); // true
```

:::

在這個範例中，JavaScript 將字串 `"5"` 轉換成數字 `5`，然後檢查它們是否相等。由於兩個值現在相同，結果為 true。等號運算子在檢查每個值是否相等之前會使用型別強制轉換。

這與嚴格等號運算子不同，嚴格等號運算子不會執行型別強制轉換。嚴格等號運算子會檢查型別是否相同以及值是否相同。以下是使用嚴格等號運算子比較數字和字串的範例。此運算子由三個等號符號（`===`）表述。

:::interactive_editor

```js
console.log(5 === '5'); // false
```

:::

以下比較會是 `false`，因為字串資料型別與數字資料型別不相同。如果你需要檢查某個值是否不相等於另一個值，則可以使用不等號或嚴格不等號運算子。以下是使用不等號運算子（`!=`）比較數字與字串的範例。 

:::interactive_editor

```js
console.log(5 != "5"); // false
```

:::

在此範例中，結果會是 `false`，因為不相等運算子會先將字串值轉換為數字，然後再比較這些值。由於這些值會相等，因此它會傳回 `false`。如果你嘗試使用嚴格不相等運算子，則會得到不同的結果。嚴格不相等運算子是由驚嘆號後接兩個等號（`!==`）所表述。

:::interactive_editor

```js
console.log(5 !== "5"); // true
```

:::

結果會是 `true`，因為嚴格不相等運算子不會執行任何型別強制轉換。由於數字 `5` 不相等於字串 `"5"`，所以結果是 `true`。

在可能的情況下使用嚴格不相等和嚴格相等運算子被視為最佳實務，因為它們不會執行型別強制轉換。在專業專案中，大多數時間你會看到程式碼庫通常偏好使用這兩個運算子，而非不相等和相等運算子。
