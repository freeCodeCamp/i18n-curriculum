# --description--

Il debugging è una competenza essenziale per ogni sviluppatore Python. Comprendere le tecniche fondamentali può aiutarti a identificare e risolvere i problemi in modo efficiente.

Il debugging è il processo di identificare e risolvere errori o bug nel tuo codice. Consiste nell’esaminare il codice, capire il flusso e usare strumenti per individuare la fonte dei problemi.

In questa lezione, vedremo le tecniche di debugging comuni che puoi usare nel tuo prossimo progetto Python.

## Usare la funzione print e le f-string

Per prima cosa, usare la funzione `print()` e le f-string in vari punti del tuo codice può aiutarti a capire il flusso e lo stato delle variabili. Per esempio:

```py
def add(a, b):
    result = a + b
    print(f'Adding {a} and {b} gives {result}')
    return result
```

Stampando i valori di `a`, `b` e `result`, puoi verificare che la funzione si comporti come previsto.

## Debugging interattivo con il modulo `pdb`

Poi, puoi utilizzare il modulo `pdb` integrato in Python per il debugging interattivo:

```py
import pdb

def divide(a, b):
    pdb.set_trace()
    return a / b

print(divide(10, 2))
```

Impostando un trace con il metodo `.set_trace()`, puoi eseguire il codice passo dopo passo, ispezionare le variabili e capire il comportamento del programma.

Se esegui il codice sopra, vedrai un output che mostra la posizione del file che stai eseguendo, la riga in cui hai chiamato il metodo `.set_trace()` e il codice subito dopo, e un prompt interattivo `pdb`:

```py
> /Users/fcc/Desktop/debugging.py(5)divide()
-> return a / b
(Pdb)
```

Se digiti help nel prompt, vedrai una lista di comandi che puoi usare:

```py
(Pdb) help

========================================
EOF    c          d        h         list      q        rv       undisplay
a      cl         debug    help      ll        quit     s        unt      
alias  clear      disable  ignore    longlist  r        source   until    
args   commands   display  interact  n         restart  step     up       
b      condition  down     j         next      return   tbreak   w        
break  cont       enable   jump      p         retval   u        whatis   
bt     continue   exit     l         pp        run      unalias  where    

Miscellaneous help topics:
==========================

```

Poi puoi usare i comandi per eseguire il debug del tuo codice.

Per esempio, se vuoi vedere il tipo degli elementi nel tuo codice in quel momento, puoi usare il comando `whatis`:

```py
(Pdb) whatis a
<class 'int'>
(Pdb) whatis divide
Function divide
```

Come vedi, al momento in cui esegui `.set_trace()`, il tipo del parametro a è un intero, e divide è una funzione.

Per continuare l’esecuzione del codice, puoi usare il comando `continue` o uno dei suoi alias, `cont` o `c`:

```py
(Pdb) continue
5.0
```

## Strumenti di debugging negli IDE

Molti ambienti di sviluppo integrati (IDE) offrono strumenti avanzati di debugging, come breakpoint, esecuzione passo passo e ispezione delle variabili.

## Usare il debugger di VS Code

Se usi VS Code, puoi impostare breakpoint nel tuo codice ed eseguire il debugger per mettere in pausa l’esecuzione in quei punti. Ecco come fare il debug della stessa funzione `divide`:

**Passaggio 1: Prepara il tuo codice** Crea un file chiamato `main.py` con il seguente contenuto:

```py
def divide(a, b):
    result = a / b
    return result

print(divide(10, 2))
print(divide(15, 3))
```

**Passaggio 2: Imposta un breakpoint**

1. Clicca nel margine interno (a sinistra) accanto alla riga 2 (`result = a / b`) per impostare un breakpoint

2. Apparirà un punto rosso che indica che il breakpoint è stato impostato

**Passaggio 3: Avvia il debugging**

1. Premi `F5` oppure vai su Esegui > Avvia debugging

2. Seleziona "Python File" quando richiesto

3. Il debugger metterà in pausa l’esecuzione al breakpoint

**Passaggio 4: Ispeziona le variabili**

- Passa il mouse sulle variabili per vedere i loro valori attuali

- Usa il pannello Variabili a sinistra per vedere tutte le variabili locali

- Usa la console di debug in basso per valutare espressioni

**Passaggio 5: Esegui il codice passo passo**

Usa la barra degli strumenti del debugger per:

- **Continua (F5)**: Riprendi l’esecuzione fino al prossimo breakpoint

- **Step Over (F10)**: Esegui la riga corrente e passa alla successiva

- **Step Into (F11)**: Entra nelle chiamate di funzione

- **Step Out (Shift+F11)**: Esci dalla funzione corrente

Gli strumenti di debugging degli IDE offrono un’interfaccia visiva per esaminare lo stato del programma, rendendo più facile identificare e risolvere problemi rispetto all’uso solo di istruzioni print.

Padroneggiando queste tecniche fondamentali di debugging — usando istruzioni `print()`, il modulo pdb e gli strumenti degli IDE — puoi identificare e risolvere efficacemente i problemi nel tuo codice Python. Ogni tecnica ha il suo ruolo: le istruzioni `print()` per controlli rapidi, pdb per esplorazioni interattive e i debugger degli IDE per ispezioni visive.
