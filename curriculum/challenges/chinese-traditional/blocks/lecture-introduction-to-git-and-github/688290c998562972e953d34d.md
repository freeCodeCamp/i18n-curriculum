# --description--

你之前已經學會如何提交變更並將它們推送到你的遠端儲存庫，但這些都是在你的預設 `main` 分支上進行。這對較小的專案來說可行，但當你為其他開放原始碼專案貢獻，或當你的專案成長時，分支是必不可少的。

例如，如果你從一個儲存庫分支並對你分支的 `main` 分支進行提交，你可能會在你的分支中產生原始儲存庫未接受的變更。這會讓你從分支中接受的變更更新你的分支變得非常困難。或者，當你在自己的專案中工作時，如果你沒有使用分支，切換不同任務會變得更加困難。相反地，你必須手動追蹤哪些變更與哪些特性或修正有關。

但究竟什麼是分支？將你的 `main` 分支視為一條高速公路，每個 commit 就像一輛進入高速公路的車輛。你可能決定想去看看某些東西，例如建立一個新特性，所以你需要暫時離開高速公路去看看。

分支本質上是從你的主要歷史中偏離，在那裡你可以自由地嘗試針對新特性或錯誤修正進行更改。如果你對你的更改感到滿意，你可以透過將分支合併到 `main` 來回到主線。如果你對你的更改不滿意，你可以刪除分支並切換回 `main`，而不接受這些更改。

在建立新分支之前，你可以使用 `git branch` 指令查看你的分支。輸出可能會像這樣：

```sh
* main
```

目前我們只有一個分支，我們的預設 `main` 分支。星號（`*`）表示那是你目前「檢出」的分支。你稍後會學到更多關於這個的內容。

你可以使用 `git branch` 指令創建一個新分支，並使用引數來指定新分支的名稱：

```sh
git branch feature
```

這會創建一個名為 `feature` 的新分支。讓我們再次使用 `git branch` 查看我們的分支：

```sh
  feature
* main
```

我們現在可以看到我們的新分支，但我們仍然在 `main` 分支上。我們需要透過「檢出」來切換到我們的新分支。當你檢出一個新分支時，你是在告訴 Git 載入當時檔案的精確狀態。

讓我們使用以下指令檢查我們的新分支：

```sh
git checkout feature
```

或

```sh
git switch feature
```

然後再次執行 `git branch` 以查看：

```sh
* feature
  main
```

請注意，星號現在已移動到你目前「檢出」的分支。
 
通常當你創建一個新分支時，你會想要立即切換到該分支。你可以用一個指令同時創建並切換到分支，而不是先創建分支再用兩個指令切換：

```sh
git checkout -b feature
```

或

```sh
git switch -c feature
```

現在我們已經創建並切換到一個新的分支。這個分支是 `main` 分支的複製，它擁有你創建分支時 `main` 所有相同的程式碼。現在，我們可以在這個分支添加任何程式碼，而不會影響 `main` 分支。我們本質上是從主線路上脫離出來以創建一個特性。如果你檢查你的 `git status`，你會看到類似以下的內容：

```sh
On branch feature
nothing to commit, working tree clean
```

透過執行此指令來創建一個新的 `feature.md` 檔案：

```sh
echo "This is our new feature" > feature.md
```

現在狀態將會看起來像這樣：

```sh
On branch feature
Untracked files:
  (use "git add <file>..." to include in what will be committed)
	feature.md

nothing added to commit but untracked files present (use "git add" to track)
```

接下來，是時候提交你的新特性。首先，將檔案添加到暫存區：

```sh
git add feature.md
```

然後，提交變更：

```sh
git commit -m "my new feature"
```

最後，將你的 `feature` 分支推送到你的儲存庫：

```sh
git push -u origin feature
```

`-u` 旗標會將你的區域的 `feature` 分支連接到遠端分支，這樣未來推送時只需使用 `git push` 即可。
