# --description--

提升“time to usable”，换句话说，就是从用户发出请求到他们能够有意义地与页面交互之间的时间间隔，对于增强用户体验至关重要。

让我们探索实现更快可用时间的有效策略。

首先专注于用户首先看到的内容。这意味着立即装载关键元素，并推迟非关键组件。

例如，你可以尝试实现惰性装载。

我们已经介绍过惰性装载，但这里有一个惰性装载图像的示例以供复习：

```html
<img src="image.jpg" loading="lazy" alt="Description">
```

你只需为 `loading="lazy"` 属性添加到图像标签。这会推迟图像的装载，直到它们被需要，通常是在它们进入用户视图时。通过仅在图像和视频进入视图时装载，我们既节省了带宽，也加快了初始渲染速度。

通常，网页会一次性装载所有图像，即使那些未在屏幕上可见的图像。使用惰性装载时，折叠以下的图像，换句话说，屏幕外的图像，只有当用户向下滚动到它们时才会被装载。

下面是惰性装载 `iframe` 的示例：

```html
<iframe src="video.html" loading="lazy"></iframe>
```

你可以将之前应用的相同属性应用到 `iframes`，相同的概念也将适用。

你也可以使用 `defer` 属性来推迟非关键 JavaScript，直到初始页面装载完成。下面是我们这样做的一个示例：

```html
<script src="non-critical.js" defer></script>
```

你也可以考虑最小化阻塞渲染的资源。

阻塞渲染的资源会延迟页面变得可交互。

为避免这种情况，你可以尝试异步地装载 CSS。对于非关键 CSS，使用 `media` 属性来条件式装载样式表。下面是我们这样做的一个示例：

```html
<link rel="stylesheet" href="print.css" media="print">
```

通过将媒体值指定为 `print`，`print.css` 仅在页面打印或在屏幕上预览时装载。在下面的示例中，通过将 `min-width: 800px` 作为 `media` 属性的值传递，我们表示只想为宽度大于 800px 的屏幕装载 `desktop.css`：

```html
<!-- This stylesheet is only loaded on screens wider than 800px -->
<link rel="stylesheet" href="desktop.css" media="(min-width: 800px)">
```

这通过跳过不必要的样式来节省带宽。

总之，通过实现这些策略，你可以显著减少网站的 `time to usable`，确保用户能够及时与内容互动。
