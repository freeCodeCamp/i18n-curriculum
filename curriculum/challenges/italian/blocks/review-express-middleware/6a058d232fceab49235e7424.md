## Che cos’è il middleware?

- Il **middleware** è una funzione che viene eseguita durante il ciclo di vita di una richiesta al server.
- Il middleware può consultare l'**oggetto request** (`req`), l'**oggetto response** (`res`) e la **funzione `next`**.
- Usi comuni: registrare le richieste, gestire gli errori, analizzare i dati in arrivo e servire file statici.
- Il middleware viene eseguito nell’ordine in cui viene aggiunto all’app.

### Come funziona il middleware

1. Viene inviata una richiesta al server.
2. Le funzioni middleware elaborano la richiesta.
3. Se il middleware chiama `next()`, il controllo passa al middleware o gestore di rotta successivo.
4. Se il middleware non chiama `next()`, il ciclo richiesta-risposta si interrompe.

```javascript
app.use((req, res, next) => {
  console.log('Request URL:', req.url);
  console.log('Request Method:', req.method);
  next();
});
```

- `app.use()` aggiunge middleware globalmente all’applicazione.

## Middleware a livello di applicazione

- Il **middleware a livello di applicazione** è associato a un’istanza dell’app Express.
- Si registra usando `app.use()` o metodi specifici per la rotta come `app.get()`, `app.post()` ecc.
- Viene eseguito per ogni richiesta in arrivo a meno che non sia limitato a una rotta specifica.
- Casi d’uso: registrazione, autenticazione e gestione degli errori su più rotte.

```javascript
app.use((req, res, next) => {
  console.log('Request received at:', new Date());
  next();
});
```

## Middleware a livello di router

- Il **middleware a livello di router** è associato a un’istanza di `express.Router()` invece che all’app principale.
- Si applica usando `router.use()` e interessa solo le rotte gestite da quel router.
- Aiuta a modularizzare l’app in moduli di rotta isolati.
- Monta un router con `app.use('/path', router)` per limitare il middleware a rotte specifiche.

```javascript
const router = express.Router();

router.use((req, res, next) => {
  console.log('Request made to /menu route');
  next();
});

router.get('/drinks', (req, res) => {
  res.send('Welcome to the drinks menu!');
});

app.use('/menu', router);
```

## Middleware per la gestione degli errori

- Il **middleware per la gestione degli errori** è un middleware speciale usato per intercettare errori durante il ciclo richiesta-risposta.
- Ha **quattro parametri**: `err`, `req`, `res` e `next`. La firma a quattro parametri indica a Express che si tratta di un gestore di errori.
- Viene invocato quando un errore viene passato a `next(err)`.
- Deve essere posizionato **alla fine** dello stack di middleware, dopo tutti gli altri middleware e rotte.

```javascript
app.use((err, req, res, next) => {
  console.error(err.stack);
  res.status(500).send('Something went wrong!');
});
```

## Middleware integrato e di terze parti

### Middleware integrato

- **`express.json()`**: analizza le richieste in arrivo con payload JSON.

```javascript
app.use(express.json());
```

- **`express.static()`**: serve file statici come immagini, CSS e JavaScript.

```javascript
app.use(express.static('public'));
```

- **`express.urlencoded()`**: analizza le richieste in arrivo con payload codificati in URL (il formato inviato dai moduli HTML); rende i dati analizzati disponibili su `req.body`.

```javascript
app.use(express.urlencoded({ extended: true }));
```

### Middleware di terze parti

- **`cors`**: abilita il Cross-Origin Resource Sharing.

```javascript
app.use(cors());
```

- **`morgan`**: registra le richieste HTTP.

```javascript
app.use(morgan('tiny'));
```

- Sia i middleware integrati sia quelli di terze parti vengono aggiunti nell’ordine in cui vengono chiamati con `app.use()`.
