# --interactive--

Ein CSS-Reset ist ein Stylesheet, das alle oder einige der standardmäßigen Formatierungen entfernt, die Webbrowser auf HTML-Elemente anwenden. Zum Beispiel ist Ihnen vielleicht aufgefallen, dass bestimmte Elemente wie Absätze und Überschriften standardmäßig bereits Außenabstände haben, auch wenn Sie diese nicht explizit in Ihren eigenen Stylesheets definieren.

Das sehen Sie auch bei verschiedenen Eigenschaften in einer Vielzahl von HTML-Elementen. Unterschiedliche Browser können außerdem unterschiedliche Standardstile auf HTML-Elemente anwenden. Die Standardstile in Google Chrome sind möglicherweise nicht genau dieselben wie in Mozilla Firefox oder Microsoft Edge. Das kann zu inkonsistenten Stilen in verschiedenen Browsern führen, was Sie so weit wie möglich vermeiden sollten.

Um das zu handhaben, entfernt ein CSS-Reset die Standardstile. Indem Sie alle oder einige der Standardstile entfernen, erhalten Sie eine einheitliche Ausgangsbasis für Ihr Design und minimieren potenzielle Inkonsistenzen zwischen Browsern und Geräten. Das Entfernen der Standardstile kann den Styling-Prozess auch erleichtern, weil Sie dann nur die Stile sehen, die Sie explizit in Ihren eigenen Stylesheets angewendet haben. Es gibt zwei Hauptansätze für CSS-Resets: Sie können entweder eigene CSS-Resets definieren oder CSS-Resets von Drittanbietern verwenden.

Eigene CSS-Resets sind Stylesheets, die Sie von Grund auf neu erstellen, um den Anforderungen Ihres Projekts gerecht zu werden. So können Sie genau steuern, welche Stile zurückgesetzt werden, und haben viel Flexibilität. Allerdings müssen Sie auch Zeit investieren, um die Stylesheets zu entwickeln und zu pflegen. Hier ist ein Beispiel für eine sehr gebräuchliche CSS-Regel, um den Außen- und Innenabstand aller HTML-Elemente zurückzusetzen:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<h1>Example Heading</h1>
<p>This is a paragraph.</p>
```

```css
* {
  margin: 0;
  padding: 0;
}
```

:::

Diese Regel wird üblicherweise am Anfang des CSS-Stylesheets geschrieben. Der Asterisk-Selektor ist ein Wildcard-Selektor, der alle HTML-Elemente auswählt, sodass sie einen Außenabstand von null und einen Innenabstand von null an allen vier Seiten haben. Das gibt Ihnen einen Ausgangspunkt, den Sie dann in spezifischeren CSS-Selektoren weiter unten im Stylesheet anpassen können.

Sie können diesen Ansatz verwenden, um jedes HTML-Element auszuwählen und seine Standardstile zurückzusetzen. Erstellen Sie einfach ein eigenes Stylesheet und verwenden Sie die passenden CSS-Selektoren, um die Elemente auszuwählen und die Stile festzulegen.

Das kann jedoch zeitaufwendig sein. Wenn Sie Zeit sparen möchten, können Sie auch einen CSS-Reset von Drittanbietern verwenden. Diese Stylesheets sind bereits vorgefertigt, sodass Sie sie einfach herunterladen und direkt in Ihr Projekt einbinden können. Ein großartiges Beispiel für einen CSS-Reset von Drittanbietern ist `Normalize.css`. Dieses Stylesheet normalisiert die Stile für eine breite Palette von HTML-Elementen, behält dabei aber einige nützliche Standardstile bei, insbesondere solche, die für die Barrierefreiheit wichtig sind. Es korrigiert auch häufige Bugs und Stilinkonsistenzen.

Eine weitere Option ist `sanitize.css`. Dabei handelt es sich um eine CSS-Bibliothek, mit der Sie sicherstellen können, dass die Standardstile in allen wichtigen modernen Browsern konsistent sind. Diese Bibliothek wird zusammen mit `Normalize.css` entwickelt, sodass sie sich gemeinsam weiterentwickeln. Sie bietet auch einzelne Stylesheets, die Sie für spezielle Zwecke herunterladen können, etwa zur Normalisierung von Formularen und Typografie.

Es gibt viele Optionen, aber Sie sollten diejenigen auswählen, die am besten zu den Anforderungen Ihres Projekts passen. Sie können auch beide Ansätze kombinieren, indem Sie CSS-Resets von Drittanbietern mit eigenen Resets verwenden.

Beim Arbeiten mit CSS-Resets ist es außerdem wichtig, die Barrierefreiheit zu berücksichtigen. Ihre Webanwendung sollte für alle zugänglich sein. Deshalb sollten Sie keine Stile zurücksetzen, die für Screenreader oder andere unterstützende Technologien hilfreich sein könnten. Sie sollten auch die Auswirkungen dieser zusätzlichen Stylesheets auf die Leistung Ihrer Anwendung bedenken, da sie heruntergeladen werden müssen, bevor die eigenen Stile angewendet werden.

Indem CSS-Resets die Standardstile entfernen, geben sie Ihnen einen leeren Ausgangspunkt, um Ihr Design zu implementieren. Das führt zu einer einheitlicheren und konsistenteren Benutzererfahrung über verschiedene Browser und Geräte hinweg.
