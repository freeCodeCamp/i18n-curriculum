# --description--

الـ State هو أحد أهم أساسيات React وإطارات العمل الأمامية الأخرى. يشبه دماغ المكوِّن، إذ يحتفظ بمعلومات يمكن أن تتغير مع الوقت ويتحكم في كيفية تصرف المكوِّنات ومظهرها.

لنستعرض ما هو الـ State وكيف يسمح لك الـ `useState` hook بالتعامل معه.

يمثل الـ State البيانات الديناميكية في مكوِّن React الخاص بك، مثل القيمة من مُدخَل المستخدم، أو البيانات المسترجعة من API، أو عنصر في قائمة المهام.

كلما تغير الـ State، يعيد React عرض المكوِّن دون إعادة تحميل الصفحة ليعكس تلك التغييرات في واجهة المستخدم. هذه التفاعلية تجعل تطبيقك تفاعليًا.

الـ `useState` hook هو دالة تتيح لك إعلان متغيرات State في المكوِّنات الوظيفية.

قبل وجود الـ hooks، كان يمكنك استخدام الـ State فقط في مكوِّنات الصنف. لكن مع ظهور الـ hooks منذ React 16.8، يمكنك استخدام الـ State في المكوِّنات الوظيفية عبر استخدام الـ `useState` hook.

لاستخدام الـ `useState` hook، تحتاج إلى استيراده من React:

```js
import { useState } from "react";
```

يمكنك أيضًا استيراد React نفسه والحصول على الوصول إلى الـ `useState` hook كخاصية:

```js
import React from "react";
```

إليك كيف تعلن عن متغير State عند استيراد `useState`:

```js
const [stateVariable, setStateFunction] = useState(initialValue);
```

وهنا كيف تعلن عن متغير State عند استيراد React:

```js
const [stateVariable, setStateFunction] = React.useState(initialValue);
```

في متغير الـ State لديك ما يلي:

- `stateVariable` يحتفظ بالقيمة الحالية للـ State
- `setStateFunction` (دالة التعيين) تُحدّث متغير الـ State
- `initialValue` يحدد القيمة الابتدائية للـ State

لاحظ أن الـ State في مكوِّن React خاص ومعزول لكل نسخة من المكوِّن. هذا يعني أنه إذا عرضت نفس المكوِّن مرتين، فإن حالة أحدهما لا تؤثر على الآخر. كما يعني أنه إذا أردت مشاركة الـ State بين المكوِّنات، يجب رفع الـ State إلى مكوِّن أب مشترك وتمريره كـ خصائص.

شيء آخر هو أن الـ hooks يجب استدعاؤها في المستوى الأعلى للمكوِّن، قبل كلمة `return` مباشرة، للحفاظ على ثبات الـ State والتأثيرات عبر عمليات العرض. هذا يعني أنه لا يمكنك استخدام الـ State داخل الحلقات أو الشروط أو الدوال المسكنة.

إليك مثالًا على إدارة الـ State باستخدام الـ `useState` hook في مكوِّن `Counter`:

```jsx
// Importing the useState hook
import { useState } from "react";

function Counter() {
  const initialValue = 0;

  // The state variable and setter function
  const [count, setCount] = useState(initialValue);

  return (
    <div>
      {/* Display current state value */}
      <h2>{count}</h2>

      <button onClick={() => setCount(count - 1)}>Decrement</button>
      <button onClick={() => setCount(count + 1)}>Increment</button>
    </div>
  );
}

export default Counter;
```

في الشفرة أعلاه، استوردنا الـ `useState` hook من React. في مكوِّن `Counter`، يمثل `count` الـ State الحالي بينما `setCount` هي دالة التعيين المسؤولة عن تحديث الـ State. القيمة الحالية للـ State هي `0`. يحتوي تعبير `return` على العدد وزرين لتقليل وزيادة العدد بمقدار `1`.

يمكنك إدارة عدة أجزاء من الـ State باستدعاء الـ `useState` hook عدة مرات. هذا مهم خصوصًا عندما تكون متغيرات الـ State غير مرتبطة:

```jsx
function UserProfile() {
  const [isOnline, setIsOnline] = useState(false);
  const [notifications, setNotifications] = useState(0);

  // The rest of the component logic
}
```

يمكنك أيضًا استدعاء الـ `useState` hook عدة مرات عند إدارة عدة حالات تتغير بشكل منفصل، مثل حقول النموذج:

```jsx
function SignUpForm() {
  const [name, setName] = useState("");
  const [username, setUsername] = useState("");
  const [email, setEmail] = useState("");

  // The rest of the component logic
}
```

لكن في هذه الحالة، من الأفضل دمج الحالات لأنها كلها جزء من نفس النموذج:

```jsx
function SignUpForm() {
  const [formData, setFormData] = useState({
    name: "",
    username: "",
    email: "",
  });

  // The rest of the component logic
}
```

هذا هو الـ State وكيف يمكنك استخدام الـ `useState` hook.
