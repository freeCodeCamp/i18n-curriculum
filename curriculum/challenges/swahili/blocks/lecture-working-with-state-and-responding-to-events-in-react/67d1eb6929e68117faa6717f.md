# --description--

Katika React, rendering ni mchakato ambao sehemu zinaonekana katika kiolesura cha mtumiaji (UI), kawaida ni kivinjari.

React huchukua msimbo wako wote wa JavaScript, JSX, na CSS, huamua jinsi unavyopaswa kuonekana, kisha huonyesha katika kiolesura cha mtumiaji.

Mchakato wa rendering una hatua tatu: trigger, render, na commit. Hebu tuchunguze hizi kwa undani zaidi.

Hatua ya trigger hutokea wakati React inapotambua kuwa kitu kimebadilika na kwamba kiolesura cha mtumiaji kinaweza kuhitaji kusasishwa. Mabadiliko haya mara nyingi hutokana na sasisho katika state au vigezo vinavyopitishwa.

Kwa mfano, kutambua kuwa ni wakati wa chakula cha jioni kunaweza kusababisha uingie jikoni kuanza kupika.

Katika mfano wa `Counter` hapa chini, kubofya kitufe cha kuongeza au kupunguza husababisha React kuonyesha thamani mpya ya `count`:

```jsx
import { useState } from "react";

function Counter() {
  const [count, setCount] = useState(0);

  return (
    <div>
      <h1>{count}</h1>
      <button onClick={() => setCount(count - 1)}>Decrement</button>
      <button onClick={() => setCount(count + 1)}>Increment</button>
    </div>
  );
}

export default Counter;
```

Mara tu trigger inapotokea, React inaingia katika hatua ya render. Hapa, React hupima tena sehemu zako na huamua nini cha kuonyesha.

Ili kufanya hivyo, React hutumia nakala nyepesi ya DOM "halisi". Hii huitwa virtual DOM. Kwa kutumia virtual DOM, React inaweza haraka kuangalia ni nini kinachohitaji kubadilika katika sehemu.

Fikiria hatua hii kama wakati uko jikoni, umekusanya viungo vyako, na unaanza kupika chakula chako cha jioni.

Kwa sehemu ya `Counter`, hatua ya render ni wakati ambapo React inafanya tena wito wa kitendakazi na thamani mpya ya `count`. React hupima upya jinsi sehemu ya `<h1>{count}</h1>` inavyopaswa kuonekana kulingana na thamani iliyosasishwa ya `count`, lakini hutaona mabadiliko yoyote kwenye skrini hadi hatua inayofuata – commit.

Hatua ya commit ni pale ambapo React huchukua mabadiliko yaliyotayarishwa kutoka virtual DOM na kuyaweka kwenye DOM halisi. Kwa maneno mengine, hii ni hatua ambapo unaona matokeo ya mwisho kwenye skrini.

Ili kufanya hivi, React inalinganisha virtual DOM na DOM halisi, hutambua sehemu tu zinazohitaji sasisho, na huweka mabadiliko hayo kwenye DOM halisi ili kusasisha kiolesura cha mtumiaji.

Unaweza kufikiria hatua hii kama wakati unavyoweka chakula ulichopika mezani, ukifanya kionekane kama vile React inavyofanya inapoweka sasisho kwenye DOM halisi.

Kuhusu sehemu ya `Counter`, hatua ya commit ni wakati ambapo thamani mpya ya `count` inawekwa kwenye kipengele cha `h1`, na unaweza kuona mabadiliko kwenye ukurasa.

Mchakato huu wa tatu ni haraka sana kwa sababu React hupunguza uendeshaji wa moja kwa moja wa DOM kwa kuhesabu mabadiliko kwanza katika virtual DOM, kisha husasisha sehemu tu zinazohitaji kubadilika katika DOM halisi.
