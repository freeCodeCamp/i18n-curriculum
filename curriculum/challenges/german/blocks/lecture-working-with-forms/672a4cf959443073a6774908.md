# --interactive--

In HTML können Formularsteuerelemente wie Eingaben verschiedene Stadien oder Zustände haben, zum Beispiel einen `focused`-Zustand, `readonly`-Zustand oder `disabled`-Zustand.

Der erste Zustand wird als `default`-Zustand betrachtet. Der Standardzustand eines E-Mail-Eingabefelds ist eine leere Eingabe. So sieht das E-Mail-Eingabefeld aus, wenn es zum ersten Mal auf der Seite gerendert wird.

:::interactive_editor

```html
<input type="email" name="email" id="email" />
```

:::

Wenn der Benutzer auf ein Formularsteuerelement klickt oder es mit der Tabulatortaste der Tastatur auswählt, befindet es sich im `focused`-Zustand. Wenn eine Eingabe im `focused`-Zustand ist, zeigen die meisten Browser einen blau hervorgehobenen Rahmen um die Eingabe an. Sie können jedoch zusätzliche Stile in CSS hinzufügen.

Klicken Sie auf einen beliebigen Bereich des Leerraums im Vorschaufenster und drücken Sie dann die `tab`-Taste, um den Fokuszustand zu sehen. Um die Vorschauen zu sehen, müssen Sie den interaktiven Editor aktivieren.

:::interactive_editor

```html
<input type="email" name="email" id="email" />
```

:::

Ein weiterer Formularzustand ist der `disabled`-Zustand. Dieser Zustand zeigt den Nutzenden, dass eine Eingabe nicht fokussiert oder aktiviert werden kann.

Aktivieren Sie den interaktiven Editor und versuchen Sie, auf das E-Mail-Eingabefeld zu klicken. Sie werden feststellen, dass es keinen Fokus mehr erhält.

:::interactive_editor

```html
<input disabled type="email" name="email" id="email" />
```

:::

Ähnlich wie beim `focused`-Zustand können Sie zusätzliche Stile für den `disabled`-Zustand mit CSS hinzufügen.

Ein weiterer Formularzustand ist der `readonly`-Zustand. Das ist, wenn ein Formularsteuerelement wie eine Eingabe für den Nutzenden nicht bearbeitbar ist. Hier ist ein Beispiel, wie man ein E-Mail-Eingabefeld auf Nur-Lesen setzt. Das `value`-Attribut wird verwendet, um den Wert festzulegen, der im Eingabefeld angezeigt wird.

Aktivieren Sie den interaktiven Editor und versuchen Sie, den aktuellen Wert von `example@email.com` im Vorschaufenster zu bearbeiten. Sie werden feststellen, dass das nicht möglich ist.

:::interactive_editor

```html
<input
  readonly
  type="email"
  name="email"
  id="email"
  value="example@email.com"
/>
```

:::

Ein wichtiger Unterschied zwischen dem `disabled`-Zustand und dem `readonly`-Zustand ist, dass `readonly` fokussiert werden kann, während der `disabled`-Zustand das nicht kann.

Das Verständnis der verschiedenen Formularzustände ist wichtig, weil sie eine reibungslose Benutzererfahrung sicherstellen, indem sie klares Feedback und Orientierung bieten und Fehler behandeln.
