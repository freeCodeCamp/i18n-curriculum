# --description--

У попередніх уроках ми навчились створювати невеликі компоненти в React, як-от цей:

```jsx
function Greeting() {
  const developerName = "Jessica";
  return <h1>Hi {developerName}!</h1>;
}
```

Ми можемо вкласти цей компонент всередину іншого батьківського компонента або кореневого компонента так:

```jsx
function App() {
  return <Greeting />;
}

function Greeting() {
  const developerName = "Jessica";
  return <h1>Hi {developerName}!</h1>;
}
```

Хоч цей код і працюватиме та відобразить результат `Hi Jessica!` на екрані, цей компонент не дуже гнучкий.

Що, якби ми хотіли показати інше ім’я, наприклад Naomi, Tom або Oliver? Саме тут на допомогу приходять пропси React. Пропси, що є скороченням від properties, — це спосіб, яким батьківські компоненти передають дані дочірнім. Пропси можуть бути будь-якого типу: рядки, числа, булеві значення, об’єкти або масиви.

Оновімо наш приклад, щоб тепер він приймав проп `name`:

```jsx
function App() {
  return <Greeting name="Jessica" />;
}

export default App;

function Greeting(props) {
  console.log(props);
  return <h1>Hi {props.name}!</h1>;
}
```

Для дочірнього компонента `Greeting` ми тепер використовуємо `props.name` замість жорстко прописаного імені `"Jessica"`. Також ми виводимо `props` у консоль, де він показується як об’єкт.

Потім, всередині батьківського компонента `App`, ми передаємо значення пропсу `name`, щоб воно могло бути передане дочірньому. Результат на екрані буде таким самим, як і раніше, але тепер ми створили більш гнучкий компонент.

Тепер ми можемо повторно використовувати дочірній компонент кілька разів, передаючи різні імена щоразу:

```jsx
function App() {
  return (
    <>
      <Greeting name="Naomi" />
      <Greeting name="Tom" />
      <Greeting name="Jessica" />
      <Greeting name="Oliver" />
    </>
  );
}
```

Ви також можете використати деструктуризацію об’єкта в пропсах, щоб зробити код більш читабельним. Ось як можна переписати компонент `Greeting`:

```jsx
function Greeting({ name }) {
  return <h1>Hi {name}!</h1>;
}
```

Цей код дає той самий результат, але чіткіше показує, які пропси очікує отримати компонент.

React також надає спеціальний проп `children`. Будь-який JSX, який ви розміщуєте між відкриваючим і кінцевим тегами компонента, передається цьому компоненту як `children`.

Наприклад:

```jsx
function Card({ children }) {
  return <div className="card">{children}</div>;
}

function App() {
  return (
    <Card>
      <h2>Hello</h2>
    </Card>
  );
}
```

У цьому випадку JSX всередині `<Card>...</Card>` стає пропом `children`. Відтворений результат у DOM буде:

```html
<div class="card">
  <h2>Hello</h2>
</div>
```

Цей шаблон часто використовується для композиції компонентів, коли компонент обгортає інші елементи інтерфейсу.

Іноді у вас може бути багато властивостей, які потрібно передати як пропси. Замість того, щоб передавати їх по одному, можна використати оператор розширення (`...`) після перетворення їх в об’єкт.

Ось приклад нового дочірнього компонента `DeveloperCard`:

```jsx
function DeveloperCard({ name, age, country }) {
  return (
    <div className="developer-card">
      <h1>Developer: {name}</h1>
      <p>Age: {age}</p>
      <p>Country: {country}</p>
    </div>
  );
}
```

Цей компонент `DeveloperCard` приймає три пропси: `name`, `age` і `country`.

У батьківському компоненті `App` ми можемо використати синтаксис розширення, щоб передати всі властивості об’єкта як окремі пропси дочірньому компоненту:

```jsx
function App() {
  const developerObj = {
    name: "Alice",
    age: 30,
    country: "USA",
  };

  return (
    <div className="App">
      <DeveloperCard {...developerObj} />
    </div>
  );
}
```

Це особливо корисно при роботі з масивами об’єктів і передачею множинних множин властивостей дочірнім компонентам. Наприклад, у вас може бути список розробників, де кожен об’єкт у масиві має однакову структуру, але представляє різну людину.

Ви дізнаєтесь більше про відтворення списків у масивах у майбутніх уроках.

Використання пропсів у React робить ваші компоненти більш гнучкими та багаторазовими, дозволяючи створювати складніші інтерфейси. Однак важливо пам’ятати, що пропси є незмінними, тобто їх не можна змінювати після передачі компоненту. Якщо потрібно обробляти введення користувача та змінювати дані, слід використовувати стан.
