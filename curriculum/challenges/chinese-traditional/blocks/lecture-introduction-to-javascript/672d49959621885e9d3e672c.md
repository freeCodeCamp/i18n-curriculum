# --interactive--

在使用 JavaScript 時，你經常會宣告變數來儲存你計劃在整個程式中使用的資料。

在現代 JavaScript 中，`let` 和 `const` 是宣告變數的首選方式，但它們在控制代碼值的指定與重新指定方面有所不同。

在本課程中，我們將探討 `let` 和 `const` 在變數宣告、指定及重新指定上的差異。

`let` 關鍵字允許你宣告可以稍後更新或重新指定的變數。你可以將 `let` 視為一個彈性的容器－一旦你在其中儲存了值，就可以在整個程式中根據需要變更該值。

以下是使用 `let` 宣告並指定變數的範例：

```js
let score = 10;
```

在這種情況下，變數 `score` 被宣告並指定值為 `10`。如果你想稍後更新該值，你可以輕鬆做到：

:::interactive_editor

```js
let score = 10;
console.log(score); // 10
score = 20;
console.log(score); // 20
```

:::

現在，`score` 持有值 `20`。當你知道變數的值會隨著你的程式執行而改變時，`let` 特別有用。

另一方面，`const` 用於宣告常數變數。一旦你為使用 `const` 宣告的變數指定值，就無法重新指定。

這使得 `const` 成為你不想在程式執行期間意外更改的值的理想選擇。

以下是使用 `const` 宣告並指定變數的範例：

:::interactive_editor

```js
const maxScore = 100;
console.log(maxScore); // 100
```

:::

一旦 `maxScore` 被指定值為 `100`，它就無法更改：

```js
maxScore = 200; // This will result in an error
```

嘗試重新指定 `const` 變數的值會在你的 JavaScript 主控台發出錯誤，因為 `const` 變數一旦被指定後就是不可變的。

你可以宣告一個 `let` 變數而不立即指定值，並且你可以稍後指定值：

:::interactive_editor

```js
let age;
console.log(age); // undefined
age = 25;
console.log(age); // 25
```

:::

使用 `const` 宣告的變數必須在宣告時指定值。如果你嘗試宣告一個未指定值的 `const` 變數，將會產生錯誤：

```js
const age; // Error: Missing initializer in const declaration
```

當你需要宣告稍後會被重新指定的變數時，應該使用 `let`。例如，在你的程式中追蹤不斷變動的分數或隨時間更新的值。

當你想宣告應保持不變的變數時，例如設定值或不應意外更改的組態，請使用 `const`。

你也可以使用 `var` 關鍵字，但現在不太建議使用。`var` 有點像 `let`，只是它的範圍比較廣，這比較容易在你的程式中造成問題。
