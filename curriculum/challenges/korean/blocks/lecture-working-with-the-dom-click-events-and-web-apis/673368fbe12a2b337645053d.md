# --description--

`Event` 객체는 사용자가 웹 페이지와 어떤 방식으로든 상호작용할 때 발생하는 페이로드입니다. 이러한 상호작용은 버튼을 클릭하거나 입력에 포커스를 맞추는 것부터 모바일 기기를 흔드는 것까지 다양할 수 있습니다.

모든 JavaScript 객체처럼 `Event` 객체도 유용할 수 있는 여러 속성을 가지고 있습니다. 사용 가능한 속성은 이 페이로드를 발생시킨 이벤트에 따라 달라집니다.

모든 `Event` 객체는 `type` 속성을 가집니다. 이 속성은 `"keydown"` 또는 `"click"` 같은 페이로드를 발생시킨 이벤트 유형을 알려줍니다. 이 값들은 `addEventListener()`에 전달할 수 있는 값과 일치하며, 여기서 `Event` 객체를 캡처하고 활용할 수 있습니다.

`Event` 객체는 항상 `target` 속성을 가집니다. `target` 속성은 이벤트를 발생시킨 객체에 대한 참조입니다. 대부분은 어떤 종류의 `HTMLElement` 객체이거나 `Document` 또는 `Window` 객체일 것입니다. 하지만 `AudioContext`처럼 더 구체적인 경우도 있습니다.

이벤트에는 메서드도 있는데, 이는 객체의 속성으로 노출된 함수입니다. 자주 사용되는 메서드 중 하나는 `preventDefault()`로, 호출 시 이벤트의 기본 동작을 막습니다.

예를 들어, 폼 제출을 직접 처리하고 싶다면 `preventDefault()`를 호출해 브라우저가 폼 데이터를 `POST` 요청으로 제출하려는 것을 막을 수 있습니다. `POST` 요청에 대해서는 앞으로의 수업에서 더 배우게 됩니다.

또한 `stopPropagation()` 메서드를 자주 만나게 될 것입니다. 이 메서드는 이벤트가 부모 요소로 버블링되거나 전파되는 것을 막습니다. 이것이 무엇을 의미하는지는 앞으로의 수업에서 더 자세히 배우게 됩니다.

`Event` 객체의 특정 구현에만 해당하는 많은 속성도 있습니다. 예를 들어 `FetchEvent`는 이벤트를 발생시킨 요청을 담는 request 속성을 가집니다.

어떤 속성이 사용 가능한지 확실하지 않으면, 해당 `Event` 객체를 로그로 출력하거나 문서를 확인하세요.
