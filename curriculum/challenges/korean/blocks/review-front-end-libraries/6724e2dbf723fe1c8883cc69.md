## JavaScript 라이브러리와 프레임워크

- JavaScript 라이브러리와 프레임워크는 공통 문제에 대한 빠른 해결책을 제공하고, 미리 작성된 코드를 통해 개발 속도를 높입니다.
- 라이브러리는 일반적으로 DOM 조작, 이벤트 처리, AJAX 요청 관리와 같은 특정 작업에 대한 솔루션 제공에 더 집중합니다.
- JavaScript 라이브러리의 예로는 jQuery와 React가 있습니다.
- 반면 프레임워크는 애플리케이션을 구축하기 위한 더 명확한 구조를 제공합니다. 개발자가 따라야 할 규칙과 관례 세트를 포함하는 경우가 많습니다.
- 프레임워크의 예로는 Angular와 React용 메타 프레임워크인 Next.js가 있습니다.
- **싱글 페이지 애플리케이션**(SPA)은 단일 HTML 페이지를 로드하고, 사용자가 애플리케이션과 상호작용할 때 전체 페이지를 다시 로드하지 않고 해당 페이지를 동적으로 업데이트하는 웹 애플리케이션입니다.
- SPA는 애플리케이션 상태를 관리하고 콘텐츠를 렌더링하기 위해 JavaScript를 사용합니다. 이는 복잡한 사용자 인터페이스를 구축하는 데 유용한 도구를 제공하는 프레임워크를 사용해 수행하는 경우가 많습니다.
- SPA와 관련된 몇 가지 문제는 다음과 같습니다:

  - 화면 읽기 프로그램이 동적으로 업데이트된 콘텐츠를 처리하는 데 어려움을 겪습니다.
  - 사용자가 애플리케이션 내에서 이동해도 URL이 변경되지 않아 특정 페이지를 북마크하거나 뒤로 가기, 공유하기가 어렵습니다.
  - 애플리케이션이 크면 초기 로드 시간이 느릴 수 있는데, 모든 자산을 미리 로드해야 하기 때문입니다.

## React

- React는 사용자 인터페이스와 웹 애플리케이션을 구축하기 위한 인기 있는 JavaScript 라이브러리입니다.
- React의 핵심 개념은 데이터가 변경될 때 독립적으로 업데이트되고 렌더링되는 재사용 가능한 UI 컴포넌트를 만드는 것입니다.
- React는 애플리케이션 상태에 따라 UI가 어떻게 보여야 하는지 설명할 수 있게 해줍니다. 데이터나 상태가 변경되면 React가 적절한 컴포넌트를 업데이트하고 렌더링합니다.

## React 컴포넌트

- 컴포넌트는 복잡한 사용자 인터페이스를 더 작고 관리하기 쉬운 조각으로 나누도록 해주는 React 애플리케이션의 구성 요소입니다.
- UI는 JSX라는 JavaScript 구문 확장을 사용해 설명하는데, 이는 개발자가 JavaScript 내에서 HTML과 유사한 코드를 작성할 수 있게 해줍니다.
- 컴포넌트는 기본적으로 UI 조각을 반환하는 JS 함수나 클래스입니다.

다음은 인사 메시지를 렌더링하는 간단한 React 컴포넌트 예시입니다:

```jsx
function Greeting() {
  const name = 'Anna';
  return <h1>Welcome, {name}!</h1>;
}
```

컴포넌트를 사용하려면 다음과 같이 호출하세요:
    
```jsx
  <Greeting />
```

## React 컴포넌트 가져오기와 내보내기

- export는 컴포넌트를 다른 파일에서 사용할 수 있게 하고, import는 다른 곳에서 export된 컴포넌트를 파일에서 사용할 수 있게 합니다.
- React에는 기본 내보내기(default export)와 명명 내보내기(named export) 두 가지 유형이 있습니다.

**기본 내보내기**

- 파일당 하나의 기본 내보내기만 가질 수 있습니다. 주로 단일 컴포넌트를 포함하는 파일에 적합합니다.
- 컴포넌트를 기본 내보내기로 내보내는 방법은 다음과 같습니다:

```jsx
// City.js
function City() {
  return <p>New York</p>;
}

export default City;
```

- 컴포넌트 정의와 같은 줄에서 내보낼 수도 있습니다:

```jsx
export default function City() {
  return <p>New York</p>;
}
```

- 기본 내보내기를 가져올 때는 중괄호 없이 `import` 키워드를 사용하세요:

```jsx
// App.js
import City from './City';

function App() {
  return (
    <div>
      <h1>My favorite city is:</h1>
      <City />
    </div>
  );
}
```

**명명 내보내기**

- 명명 내보내기는 파일에서 여러 컴포넌트를 공유할 수 있게 합니다. 기본 내보내기와 달리 내보낸 정확한 이름으로 가져와야 합니다.
- 같은 파일에서 여러 컴포넌트를 내보내는 방법은 다음과 같습니다:

```jsx
// Animals.jsx
export function Cat() {
  return <h2>Mr. Whiskers</h2>;
}

export function Dog() {
  return <h2>Fido</h2>;
}
```

- 명명 내보내기를 가져올 때는 내보낸 정확한 이름을 중괄호 안에 넣어 사용하세요:

```jsx
import { Cat, Dog } from './Animals';
```

- 가져올 때 `as` 키워드를 사용해 명명 내보내기의 이름을 바꿀 수도 있습니다:

```jsx
import { Cat as Kitty } from './Animals';
```

**기본 내보내기와 명명 내보내기 혼합**

- 파일은 하나의 기본 내보내기와 여러 명명 내보내기를 동시에 가질 수 있습니다:

```jsx
// Animals.jsx
export default function Cat() {
  return <h2>Mr. Whiskers</h2>;
}

export function Dog() {
  return <h2>Fido</h2>;
}
```

- 혼합 내보내기를 가져올 때는 기본 내보내기를 먼저(중괄호 없이), 그 다음 명명 내보내기를 중괄호 안에 넣어 가져옵니다:

```jsx
import Cat, { Dog } from './Animals';
```

## Vite를 사용한 React 프로젝트 설정

- 프로젝트 설정 도구와 CLI는 새 프로젝트를 빠르고 쉽게 시작할 수 있게 해주어, 개발자가 설정 대신 코드 작성에 집중할 수 있게 합니다.
- Vite는 인기 있는 프로젝트 설정 도구이며 React와 함께 사용할 수 있습니다.
- 터미널에서 다음 명령어로 Vite를 사용해 새 프로젝트를 생성할 수 있습니다:

```bash
npm create vite@latest my-react-app -- --template react
```

이 명령어는 Vite의 React 템플릿을 사용해 `my-react-app`라는 새 React 프로젝트를 만듭니다. 프로젝트 디렉터리에는 프로젝트 의존성과 명령어가 나열된 `package.json` 파일이 있습니다.

- 프로젝트를 실행하려면 프로젝트 디렉터리로 이동해 다음 명령어를 실행하세요:

```bash
cd my-react-app # path to the project directory
npm install # installs the dependencies listed in the package.json file
```

- 의존성이 설치되면 프로젝트 내에 `node_modules`라는 새 폴더가 생성된 것을 확인할 수 있습니다.

- `node_modules` 폴더는 프로젝트에 필요한 모든 패키지와 라이브러리가 저장되는 곳입니다.

- 프로젝트를 실행하려면 다음 명령어를 사용하세요:

```bash
npm run dev
```

- 그런 다음 브라우저를 열고 `http://localhost:5173`로 이동하면 React 애플리케이션이 실행되는 것을 볼 수 있습니다.

- 시작 템플릿 코드를 실제로 보려면 프로젝트 내 `src` 폴더로 들어가 `App.jsx` 파일을 확인하세요.

## React 컴포넌트에 props 전달하기

- React에서 props(속성)는 부모 컴포넌트에서 자식 컴포넌트로 데이터를 전달하는 방법입니다. 이는 재사용 가능하고 동적인 UI 요소를 만들기 위해 필요합니다.
- props는 어떤 JavaScript 값도 될 수 있습니다. 부모에서 자식 컴포넌트로 props를 전달하려면 부모 JSX에서 자식 컴포넌트를 사용할 때 속성으로 props를 추가하면 됩니다. 간단한 예시는 다음과 같습니다:

```jsx
// Parent component
function Parent() {
  const name = 'Anna';
  return <Child name={name} />;
}

// Child component
function Child(props) {
  return <h1>Hello, {props.name}!</h1>;
}
```

객체로 변환한 후 전개 연산자 `(...)`를 사용해 여러 props를 전달할 수 있습니다. 예시는 다음과 같습니다:
    
```jsx
// Parent component
function Parent() {
  const person = {
    name: 'Anna',
    age: 25,
    city: 'New York'
  };
  return <Child {...person} />;
}
```

이 코드에서 전개 연산자 `{...person}`는 person 객체를 개별 props로 변환해 Child 컴포넌트에 전달합니다.

## React에서 조건부 렌더링

- React의 조건부 렌더링은 애플리케이션 내 특정 조건이나 상태에 따라 다른 콘텐츠를 보여주는 동적 UI를 만드는 데 사용됩니다.
- 조건부 렌더링 방법은 여러 가지가 있습니다. 흔한 방법 중 하나는 삼항 연산자를 사용하는 것입니다. 예시는 다음과 같습니다:

```jsx
function Greeting({ isLoggedIn }) {
  return (
    <div>
      {isLoggedIn ? <h1>Welcome back!</h1> : <h1>Please log in</h1>}
    </div>
  );
}
```

- 또 다른 방법은 논리 AND 연산자(`&&`)를 사용하는 것입니다. 특정 조건이 참일 때만 콘텐츠를 렌더링할 때 유용합니다. 예시는 다음과 같습니다:

```jsx
function Greeting({ user }) {
  return (
    <div>
      {user && <h1>Welcome, {user.name}!</h1>}
    </div>
  );
}
```

위 코드에서 `h1` 요소는 user 객체가 참일 때만 렌더링됩니다.

직접 `if` 문을 이렇게 사용할 수도 있습니다:

```js
function Greeting({ isLoggedIn }) {
  if (isLoggedIn) {
    return <h1>Welcome back!</h1>;
  }
  return <h1>Please sign in</h1>;
}
```

## React에서 리스트 렌더링

- React에서 리스트 렌더링은 사용자 인터페이스를 구축할 때 흔히 하는 작업입니다.
- JS 배열 `map()` 메서드를 사용해 배열 항목을 순회하며 JSX 요소의 새 배열을 반환해 리스트를 렌더링할 수 있습니다.
- 예를 들어, 이름 배열을 리스트로 렌더링하려면 다음과 같이 할 수 있습니다:

```jsx
function NameList({ names }) {
  return (
    <ul>
      {names.map((name, index) => (
        <li key={`${name}-${index}`}>{name}</li>
      ))}
    </ul>
  );
}
```

- 각 리스트 항목에 고유한 키를 제공해 React가 업데이트와 렌더링을 효율적으로 관리할 수 있게 하세요. 이 기법들로 React 애플리케이션에서 유연하고 효율적이며 동적인 리스트를 만들 수 있습니다.

## React에서 인라인 스타일

- React에서 인라인 스타일은 JavaScript 객체를 사용해 JSX 요소에 직접 CSS 스타일을 적용할 수 있게 합니다.
- 인라인 스타일을 적용하려면 JSX 요소에 style 속성을 사용하세요. style 속성은 키가 카멜 케이스인 CSS 속성이고 값이 해당 값인 객체를 받습니다. 예시는 다음과 같습니다:

```js
function Greeting() {
  return (
    <h1
      style={{ color: 'blue', fontSize: '24px', backgroundColor: 'lightgray' }}
    >
      Hello, world!
    </h1>
  );
}

export default Greeting;
```

스타일을 별도의 객체로 추출해 `style` 속성에서 참조할 수도 있습니다:

```jsx
function Greeting() {

  const styles = {
    color: 'blue',
    fontSize: '24px',
    backgroundColor: 'lightgray'
  };

  return <h1 style={styles}>Hello, world!</h1>;
}

export default Greeting;
```

- 인라인 스타일은 props나 상태에 따라 조건부로 스타일을 적용할 수 있어 동적 스타일링을 지원합니다. 다음은 prop에 따라 조건부 스타일을 적용하는 예시입니다:

```jsx
function Greeting({ isImportant }) {

  const styles = {
    color: isImportant ? 'red' : 'black',
    fontSize: isImportant ? '24px' : '16px'
  };

  return <h1 style={styles}>Hello, world!</h1>;
}

export default Greeting;
```

- 위 코드에서 `color`과 `fontSize` 스타일은 `isImportant` prop에 따라 조건부로 설정됩니다.

## React에서 이벤트 다루기

- **합성 이벤트 시스템**: React가 이벤트를 처리하는 방식입니다. `click`, `keydown`, `submit` 같은 네이티브 이벤트를 감싸는 래퍼 역할을 합니다. React의 이벤트 핸들러는 카멜 케이스 명명 규칙을 따릅니다. (예: `onClick`, `onSubmit` 등)

다음은 React에서 `onClick` 요소에 `button` 속성을 사용하는 예시입니다:

```jsx
function handleClick() {
  console.log("Button clicked!");
}

<button onClick={handleClick}>Click Me</button>;
```

React에서 이벤트 핸들러 함수는 보통 `handle` 접두어로 시작해 이벤트를 처리함을 나타냅니다. 예: `handleClick`, `handleSubmit`.

사용자 동작이 이벤트를 트리거하면 React는 합성 이벤트 객체를 핸들러에 전달합니다. 이 객체는 일반 JavaScript의 네이티브 Event 객체처럼 동작하며 `type`, `target`, `currentTarget` 같은 속성을 제공합니다.

예를 들어 `onSubmit` 이벤트 중 브라우저 새로 고침 같은 기본 동작을 막으려면 `preventDefault()` 메서드를 호출하세요:

```jsx
function handleSubmit(event) {
  event.preventDefault();
  console.log("Form submitted!");
}

<form onSubmit={handleSubmit}>
  <input type="text" />
  <button>Submit</button>
</form>;
```

화살표 함수로 핸들러 함수를 감쌀 수도 있습니다:

```jsx
function handleDelete(id) {
  console.log("Deleting item:", id);
}

<button onClick={() => handleDelete(1)}>Delete Item</button>;
```

## 상태와 `useState` 훅 다루기

- **상태 정의**: React에서 상태는 컴포넌트의 데이터를 담은 객체입니다. 상태가 업데이트되면 컴포넌트가 다시 렌더링됩니다. React는 상태를 불변으로 취급해 직접 수정하지 말아야 합니다.
- **`useState()` 훅**: `useState` 훅은 함수형 컴포넌트에서 상태 변수를 선언할 수 있게 해줍니다. 기본 구문은 다음과 같습니다:

```js
const [stateVariable, setStateFunction] = useState(initialValue);
```

상태 변수에는 다음이 포함됩니다:

- `stateVariable`: 현재 상태 값
- `setStateFunction`: 상태 변수를 업데이트하는 설정 함수
- `initialValue`: 초기 상태 설정

다음은 `Counter` 컴포넌트의 완전한 예시입니다:

```jsx
import { useState } from "react";

function Counter() {
  const [count, setCount] = useState(0);

  return (
    <div>
      <h2>{count}</h2>

      <button onClick={() => setCount(count - 1)}>Decrement</button>
      <button onClick={() => setCount(count + 1)}>Increment</button>
    </div>
  );
}

export default Counter;
```

## 렌더링과 React 컴포넌트

- **정의**: React에서 렌더링은 컴포넌트가 사용자 인터페이스(UI), 보통은 브라우저에 나타나는 과정입니다. 렌더링 과정은 트리거, 렌더, 커밋 세 단계로 이루어집니다.

트리거 단계는 React가 변경 사항을 감지해 UI를 업데이트해야 할 때 발생합니다. 이 변경은 주로 상태나 props 업데이트 때문입니다.

트리거가 발생하면 React는 렌더 단계에 들어갑니다. 여기서 컴포넌트를 재평가해 무엇을 보여줄지 결정합니다. React는 가벼운 복사본인 가상 DOM을 사용해 변경할 부분을 빠르게 확인합니다.

커밋 단계는 가상 DOM에서 준비된 변경 사항을 실제 DOM에 적용하는 단계입니다. 즉, 화면에 최종 결과가 나타나는 단계입니다.

## 상태에서 객체와 배열 업데이트

- **상태에서 객체 업데이트**: 상태의 객체를 업데이트할 때는 새 객체를 만들거나 기존 객체를 복사한 후 새 객체로 상태를 설정해야 합니다. 상태에 넣는 객체는 읽기 전용으로 간주해야 합니다. 다음은 사용자의 이름, 나이, 도시를 설정하는 예시입니다. `handleChange` 함수가 사용자 정보 업데이트를 처리합니다:

```jsx
import { useState } from "react";

function Profile() {
  const [user, setUser] = useState({ name: "John Doe", age: 31, city: "LA" });

  const handleChange = (e) => {
    const { name, value } = e.target;

    setUser((prevUser) => ({...prevUser, [name]: value}));
  };

  return (
    <div>
      <h1>User Profile</h1>
      <p>Name: {user.name}</p>
      <p>Age: {user.age}</p>
      <p>City: {user.city}</p>

      <h2>Update User Age </h2>
      <input type="number" name="age" value={user.age} onChange={handleChange} />

      <h2>Update User Name </h2>
      <input type="text" name="name" value={user.name} onChange={handleChange} />

      <h2>Update User City </h2>
      <input type="text" name="city" value={user.city} onChange={handleChange} />
    </div>
  );
}

export default Profile;
```

- **상태에서 배열 업데이트**: 상태의 배열을 업데이트할 때는 `push()`나 `pop()` 같은 메서드로 배열을 직접 수정하지 말고 새 배열을 만들어 상태를 업데이트해야 합니다:

```jsx
const addItem = () => {
  const newItem = {
    id: items.length + 1,
    name: `Item ${items.length + 1}`,
  };

  // Creates a new array
  setItems((prevItems) => [...prevItems, newItem]);
};
```

배열에서 항목을 제거하려면 필터링 후 새 배열을 반환하는 `filter()` 메서드를 사용하세요:

```jsx
const removeItem = (id) => {
  setItems((prevItems) => prevItems.filter((item) => item.id !== id));
};
```

## 참조값 사용하기 (Refs)

- **`ref` 속성**: React에서 DOM 노드에 접근하려면 `ref` 속성을 사용하세요. 다음은 `ref`를 사용해 `input` 요소에 포커스하는 예시입니다. `current` 속성은 해당 `ref`의 현재 값을 접근하는 데 사용됩니다:

```jsx
import { useRef } from "react";

const Focus = () => {
  const inputRef = useRef(null);

  const handleFocus = () => {
    if (inputRef.current) {
      inputRef.current.focus();
    }
  };

  return (
    <div>
      <input ref={inputRef} type="text" placeholder="Enter text" />
      <button onClick={handleFocus}>Focus Input</button>
    </div>
  );
};

export default Focus;
```

## `useEffect` 훅 다루기

- **`useEffect()` 훅**: React에서 이펙트는 컴포넌트 렌더링 과정 외부에서 발생하는 모든 작업입니다. 예를 들어 데이터 가져오기, 브라우저 탭 제목 업데이트, 로컬 스토리지 읽기/쓰기, 사용자 위치 가져오기 등이 있습니다. 이런 작업은 부수 효과(side effects)라고 하며, React는 `useEffect` 훅을 제공해 이를 처리할 수 있게 합니다. `useEffect`는 컴포넌트가 렌더링되거나 업데이트된 후 함수를 실행합니다.

```js
import { useEffect } from "react";

useEffect(() => {
 // Your side effect logic (usually a function) goes here
}, [dependencies]);
```

이펙트 함수는 컴포넌트 렌더링 후 실행되며, 선택적 `dependencies` 인수는 이펙트 실행 시기를 제어합니다.

`dependencies`는 "반응형 값"(상태, props, 함수, 변수 등)의 배열, 빈 배열, 또는 생략할 수 있습니다. 각 옵션이 `useEffect` 작동 방식을 제어하는 방법은 다음과 같습니다:

- `dependencies`가 하나 이상의 반응형 값을 포함하는 배열이면, 해당 값이 변경될 때마다 이펙트가 실행됩니다.

- `dependencies`가 빈 배열이면, `useEffect`는 컴포넌트가 처음 렌더링될 때 한 번만 실행됩니다.

- `dependencies`를 생략하면, 컴포넌트가 렌더링되거나 업데이트될 때마다 이펙트가 실행됩니다.

## 커스텀 훅 만들기

- **커스텀 훅**: 커스텀 훅은 데이터 가져오기, 상태 관리, 토글, 온라인 상태 추적 같은 재사용 가능한 로직을 컴포넌트에서 분리해낼 수 있게 합니다. React의 내장 훅은 모두 `use`로 시작하므로, 커스텀 훅도 같은 관례를 따라야 합니다.

다음은 `useDebounce` 훅을 만드는 예시입니다:

```jsx
function useDebounce(value, delay) {
  const [debouncedValue, setDebouncedValue] = useState(value);

  useEffect(() => {
    const handler = setTimeout(() => {
      setDebouncedValue(value);
    }, delay);

    return () => {
      clearTimeout(handler);
    };
  }, [value, delay]);

  return debouncedValue;
}

export { useDebounce };
```

## React에서 폼 다루기

- **제어된 입력**: 입력 필드 값을 상태에 저장하고 `onChange` 이벤트를 통해 업데이트하는 방식입니다. 폼 데이터를 완전히 제어할 수 있어 즉시 유효성 검사와 조건부 렌더링이 가능합니다.

```jsx
import { useState } from "react";

function App() {
  const [name, setName] = useState("");

  const handleChange = (e) => {
    setName(e.target.value);
  };

  const handleSubmit = (e) => {
    e.preventDefault();
    console.log(name);
  };

  return (
    <>
      <form onSubmit={handleSubmit}>
        <label htmlFor="name">Your name</label> <br />
        <input value={name} id="name" onChange={handleChange} type="text" />
        <button type="submit">Submit</button>
      </form>
    </>
  );
}

export default App;
```

- **비제어 입력**: `useState` 훅 대신 HTML의 DOM이 자체 상태를 관리하는 방식입니다. DOM이 입력 값을 제어하므로, 입력 필드 값을 가져오려면 `ref`를 사용해야 합니다.

```jsx
import { useRef } from "react";

function App() {
 const nameRef = useRef();

 const handleSubmit = (e) => {
   e.preventDefault();
   console.log(nameRef.current.value);
 };

 return (
   <form onSubmit={handleSubmit}>
     <label htmlFor="name">Your</label>{" "}
     <input type="text" ref={nameRef} id="name" />
     <button type="submit">Submit</button>
   </form>
 );
}

export default App;
```

## `useActionState` 훅 다루기

- **서버 액션**: 서버에서 실행되어 API 엔드포인트 없이도 서버에서 폼 처리를 가능하게 하는 함수입니다. Next.js 애플리케이션 예시는 다음과 같습니다:

```js
"use server";

async function submitForm(formData) {
 const name = formData.get("name");
 return { message: `Hello, ${name}!` };
}
```

`"use server"` 지시어는 함수를 서버 액션으로 표시합니다.

- **`useActionState` 훅**: 폼 제출 결과에 따라 상태를 업데이트하는 훅입니다. `useActionState` 훅의 기본 구문은 다음과 같습니다:

```js
const [state, action, isPending] = useActionState(actionFunction, initialState, permalink);
```

- `state`: 액션이 반환하는 현재 상태
- `action`: 서버 액션을 트리거하는 함수
- `isPending`: 액션이 실행 중인지 나타내는 불리언
- `actionFunction`: 서버 액션 자체
- `initialState`: 액션 실행 전 상태의 시작점
- `permalink`: 폼이 수정하는 고유 페이지 URL을 담은 선택적 문자열

## React에서 데이터 가져오기

- **데이터 가져오기 옵션**: React에서 데이터를 가져오는 방법은 다양합니다. 네이티브 Fetch API, Axios, SWR 같은 서드파티 도구를 사용할 수 있습니다.
- **데이터 가져올 때 흔히 사용하는 상태 변수**: 어떤 방법을 쓰든 데이터 자체, 데이터가 아직 로딩 중인지 추적하는 상태, 데이터 가져오기 중 발생할 수 있는 오류를 담는 상태 변수가 필요합니다.

```js
const [data, setData] = useState(null);
const [loading, setLoading] = useState(true);
const [error, setError] = useState(null);
```

데이터 가져오기는 부수 효과이므로 `Fetch API`를 `useEffect` 훅 안에서 사용하는 것이 좋습니다.

```js
useEffect(() => {
  const fetchData = async () => {
    try {
      const res = await fetch("https://jsonplaceholder.typicode.com/posts");
      
      if (!res.ok) {
        throw new Error("Network response was not ok");
      }

      const data = await res.json();
      setData(data);
    } catch (err) {
      setError(err);
    } finally {
      setLoading(false);
    }
  };

  fetchData();
}, []);
```

데이터 가져오기가 완료되지 않았으면 로딩 메시지를, 오류가 있으면 오류 메시지를, 그렇지 않으면 결과를 렌더링할 수 있습니다.

```jsx
if (loading) {
  return <p>Loading...</p>;
}

if (error) {
  return <p>{error.message}</p>;
}

return (
  <ul>
    {data.map((post) => (
      <li key={post.id}>{post.title}</li>
    ))}
  </ul>
);
```

Axios를 사용하려면 설치하고 가져와야 합니다:

```bash
npm i axios
```

```js
import axios from "axios";
```

그다음 `axios.get`를 사용해 데이터를 가져올 수 있습니다:

```js
const [data, setData] = useState(null);
const [loading, setLoading] = useState(true);
const [error, setError] = useState(null);

useEffect(() => {
  const fetchData = async () => {
    try {
      const res = await axios.get(
        "https://jsonplaceholder.typicode.com/users"
      );
      setData(res.data);
    } catch (err) {
      setError(err);
    } finally {
      setLoading(false);
    }
  };

  fetchData();
}, []);
```

`useSWR` 훅을 사용해 데이터를 가져오려면 먼저 설치하고 가져와야 합니다.

```bash
npm i swr
```

```js
import useSWR from "swr";
```

훅을 사용해 데이터를 가져오는 방법은 다음과 같습니다:

```jsx
import useSWR from "swr";

const fetcher = (url) => fetch(url).then((res) => res.json());

const FetchTodos = () => {
 const { data, error } = useSWR(
   "https://jsonplaceholder.typicode.com/todos",
   fetcher
 );

 if (!data) {
   return <h2>Loading...</h2>;
 }
 if (error) {
   return <h2>Error: {error.message}</h2>;
 }

 return (
   <>
     <h2>Todos</h2>
     <div>
       {data.map((todo) => (
         <h3 key={todo.id}>{todo.title}</h3>
       ))}
     </div>
   </>
 );
};

export default FetchTodos;
```

## `useOptimistic` 훅 다루기

- **`useOptimistic` 훅**: 비동기 작업이 백그라운드에서 완료되는 동안 UI를 반응형으로 유지하는 데 사용됩니다. 서버 응답을 기다리는 등 작업의 예상 결과에 기반해 즉각적인 UI 업데이트를 제공하는 "낙관적 업데이트" 전략을 관리합니다.

기본 구문은 다음과 같습니다:

```js
const [optimisticState, addOptimistic] = useOptimistic(actualState, updateFunction);
```

- `optimisticState`: 더 나은 사용자 경험을 위해 즉시 업데이트되는 임시 상태
- `addOptimistic`: 실제 상태 변경 전에 낙관적 업데이트를 적용하는 함수
- `actualState`: 서버에서 데이터를 가져오는 등 작업 결과로부터 오는 실제 상태 값
- `updateFunction`: 호출 시 낙관적 상태가 어떻게 업데이트될지 결정하는 함수

다음은 `useOptimistic` 훅을 `TaskList` 컴포넌트에서 사용하는 예시입니다:

```js
"use client";

import { useOptimistic } from "react";

export default function TaskList({ tasks, addTask }) {
  const [optimisticTasks, addOptimisticTask] = useOptimistic(
    tasks,
    (state, newTask) => [...state, { text: newTask, pending: true }]
  );

  async function handleSubmit(e) {
    e.preventDefault();
    const formData = new FormData(e.target);

    addOptimisticTask(formData.get("task"));

    addTask(formData);
    e.target.reset();
  }

  return <>{/* UI */}</>;
}
```

- **`startTransition`**: UI 일부를 렌더링하고 상태 업데이트를 비긴급 전환으로 표시하는 데 사용됩니다. 비용이 많이 드는 업데이트 중에도 UI가 반응형으로 유지되게 합니다. 기본 구문은 다음과 같습니다:

```js
startTransition(action)
```

`action`는 상태 업데이트를 수행하거나 전환 관련 로직을 트리거합니다. 이를 통해 타이핑이나 클릭 같은 긴급 UI 업데이트가 차단되지 않도록 보장합니다.

## `useMemo` 훅 다루기

- **메모이제이션**: 비용이 많이 드는 함수 호출 결과를 특정 인수에 따라 캐시해, 같은 인수가 다시 주어지면 재계산 없이 캐시된 결과를 반환하는 최적화 기법입니다.
- **`useMemo` 훅**: 계산된 값을 메모이제이션하는 데 사용됩니다. 다음은 큰 배열을 정렬한 결과를 메모이제이션하는 예시입니다. `expensiveSortFunction`는 `largeArray`가 변경될 때만 실행됩니다:

```js
const memoizedSortedArray = useMemo(
  () => expensiveSortFunction(largeArray),
  [largeArray]
);
```

## `useCallback` 훅 다루기

- **`useCallback` 훅**: 함수 참조를 메모이제이션하는 데 사용됩니다.

```js
const handleClick = useCallback(() => {
 // code goes here
}, [dependency]);
``` 

- **`React.memo`**: prop이 변경되지 않았을 때 불필요한 재렌더링을 방지하기 위해 컴포넌트를 메모이제이션하는 데 사용됩니다.

```js
const MemoizedComponent = React.memo(({ prop }) => {
 return (
   <>
     {/* Presentation */}
   </>
 )
});
``` 

## 의존성 관리 도구

- **의존성 정의**: 소프트웨어에서 의존성은 애플리케이션 내 한 컴포넌트나 모듈이 제대로 작동하려면 다른 컴포넌트나 모듈에 의존하는 것을 의미합니다. React 프로젝트에 필요한 두 가지 핵심 의존성은 `react`와 `react-dom` 패키지입니다:

```json
"dependencies": {
  "react": "^18.3.1",
  "react-dom": "^18.3.1"
}
```

- **패키지 관리자 정의**: 프로젝트에서 소프트웨어 의존성을 관리하려면 패키지 관리자를 사용해야 합니다. 패키지 관리자는 의존성 설치, 업데이트, 제거를 위한 도구입니다. JavaScript, Python, Ruby, Java 등 많은 인기 프로그래밍 언어가 패키지 관리자를 사용합니다. JavaScript용 인기 패키지 관리자로는 npm, Yarn, pnpm이 있습니다.
- **`package.json` 파일**: 프로젝트 이름, 버전, 의존성 등 메타데이터를 포함하는 주요 구성 파일입니다. 스크립트, 라이선스 정보, 기타 설정도 정의해 프로젝트와 의존성 관리를 돕습니다.
- **`package-lock.json` 파일**: 프로젝트에서 사용하는 모든 패키지의 정확한 버전을 고정하는 파일입니다. 패키지를 업데이트하면 잠금 파일도 업데이트됩니다.
- **`node_modules` 폴더**: `package.json` 파일에 나열된 의존성의 실제 코드를 포함하는 폴더로, 프로젝트의 직접 의존성과 그 의존성의 의존성을 모두 포함합니다.

## React 라우터

- **소개**: React Router는 React 애플리케이션에 라우팅 기능을 추가하는 서드파티 라이브러리입니다. 기존 React 프로젝트에 React Router를 설치하려면 다음과 같이 하세요:

```bash
npm i react-router
```

그런 다음 `main.jsx` 또는 `index.jsx` 파일 내에서 라우트 구조를 다음과 같이 설정해야 합니다:

```jsx
import { StrictMode } from "react";
import { createRoot } from "react-dom/client";
import { BrowserRouter, Routes, Route } from "react-router";
import App from "./App.jsx";

import "./index.css";

createRoot(document.getElementById("root")).render(
  <StrictMode>
    <BrowserRouter>
      <Routes>
        <Route path="/" element={<App />} />
      </Routes>
    </BrowserRouter>
  </StrictMode>
);
```

`path`와 `element`는 URL과 UI 컴포넌트를 연결하는 데 사용됩니다. 여기서는 홈페이지 경로에 `App` 컴포넌트를 연결합니다.

- **다중 뷰와 라우트 설정**: 대규모 애플리케이션에서는 여러 뷰와 라우트를 다음과 같이 설정하는 경우가 많습니다:

```jsx
<Routes>
  <Route index element={<Home />} />
  <Route path="about" element={<About />} />

  <Route path="products">
    <Route index element={<ProductsHome />} />
    <Route path=":category" element={<Category />} />
    <Route path=":category/:productId" element={<ProductDetail />} />
    <Route path="trending" element={<Trending />} />
  </Route>
</Routes>
```

이 예시에서 `index` 프로프는 특정 경로 세그먼트의 기본 라우트를 나타냅니다. 따라서 `Home` 컴포넌트는 루트 `/` 경로에서, `ProductsHome` 컴포넌트는 `/products` 경로에서 보여집니다.

- **라우트 중첩**: 라우트를 다른 라우트 안에 중첩할 수 있으며, 이 경우 자식 라우트 경로가 부모 라우트 경로에 추가됩니다.

```jsx
<Route path="products">
  <Route path="trending" element={<Trending />} />
</Route>
```

위 예시에서 트렌딩 상품 경로는 products/trending이 됩니다.

- **동적 세그먼트**: URL 경로의 일부가 동적으로 변하는 부분입니다.

```jsx
<Route path=":category" element={<Category />} />
```

이 예시에는 `category`라는 동적 세그먼트가 있습니다. 사용자가 products/brass-instruments 같은 URL로 이동하면 뷰가 `Category` 컴포넌트로 바뀌고, 세그먼트에 따라 적절한 데이터를 동적으로 가져올 수 있습니다.

- **`useParams` 훅**: URL 경로에서 동적 매개변수를 가져오는 데 사용됩니다.

```js
import { useParams } from "react-router";

export default function Category() {
  let params = useParams();
   {/* Accessing the category param: params.category */}
   {/* rest of code goes here */}
}
```

- **소개**: React 프레임워크는 라우팅, 이미지 최적화, 데이터 가져오기, 인증 등 기능을 제공합니다. 이로 인해 특정 사용 사례에서는 별도의 프론트엔드와 백엔드 애플리케이션을 설정할 필요가 없을 수 있습니다. React 프레임워크 예로는 Next.js와 Remix가 있습니다.

다음은 커스텀 요청 핸들러를 만드는 예시입니다:

```js
export async function GET() {

  return Response.json({ data });
}
```


## Prop Drilling

- **정의**: Prop drilling은 부모 컴포넌트에서 깊게 중첩된 자식 컴포넌트로 props를 전달하는 과정으로, 일부 자식 컴포넌트는 props가 필요 없더라도 전달됩니다.

## 상태 관리

- **Context API**: 부모 컴포넌트가 명시적으로 props를 통해 전달하지 않고도 자식 컴포넌트에 정보를 제공하는 방법입니다. `createContext`는 다른 컴포넌트가 읽을 컨텍스트 객체를 생성하는 데 사용되고, `Provider`는 자식 컴포넌트에 컨텍스트 값을 공급하는 데 사용됩니다.

```js
import { useState, createContext } from "react";

const CounterContext = createContext();

const CounterProvider = ({ children }) => {
  const [count, setCount] = useState(0);

  return (
    <CounterContext.Provider value={{ count, setCount }}>
      {children}
    </CounterContext.Provider>
  );
};

export { CounterContext, CounterProvider };
```

- **Redux**: Redux는 중앙 저장소와 상태 업데이트에 대한 엄격한 제어를 제공해 상태 관리를 처리합니다. 액션, 리듀서, 미들웨어라는 예측 가능한 패턴을 사용합니다. 액션은 사용자 상호작용으로부터 Redux 저장소에 데이터를 보내는 정보 페이로드입니다. 리듀서는 상태가 불변 방식으로 어떻게 변경될지 지정하는 함수입니다. 미들웨어는 액션 디스패치와 리듀서 사이의 다리 역할을 하며, 로깅이나 비동기 작업 처리 같은 Redux 기능 확장을 가능하게 합니다.

- **Zustand**: 이 상태 관리 솔루션은 소규모에서 중간 규모 애플리케이션에 적합합니다. `useStore` 훅을 사용해 컴포넌트와 페이지에서 상태에 직접 접근할 수 있어 액션, 리듀서, 프로바이더 없이 데이터를 수정하고 접근할 수 있습니다.

- **React 개발자 도구**: Chrome, Firefox, Edge에서 React 컴포넌트를 검사하고 성능 문제를 식별하는 브라우저 확장입니다. Safari에서는 `react-devtools` npm 패키지를 설치해야 합니다. React DevTools를 설치하고 React 앱을 브라우저에서 열면, 개발자 도구에서 컴포넌트와 프로파일러 탭을 사용할 수 있습니다.

## INP 개선

- **정의**: INP(Interaction to Next Paint)는 사용자가 클릭이나 키 입력 같은 상호작용을 한 후 브라우저가 다음 화면 업데이트를 할 때까지 걸리는 시간을 측정해 페이지의 전반적 반응성을 평가합니다. INP가 낮을수록 더 반응성이 좋은 페이지입니다.

INP를 개선하는 방법:

- 긴 JavaScript 작업을 분할해 메인 스레드 작업 줄이기
- 비핵심 스크립트에 `requestIdleCallback()` 사용. 브라우저 유휴 시간에 함수 호출 예약
- 앞서 다룬 무거운 자산 지연 로딩 또는 연기
- 이벤트 핸들러 최적화. 너무 자주 실행되거나 무거운 작업을 수행하면 페이지가 느려지고 INP가 증가합니다. 해결책은 디바운싱으로, 사용자가 타이핑을 멈춘 후 일정 시간(예: 300ms) 후에만 함수가 실행되도록 해 불필요한 계산을 방지하고 성능을 향상합니다.

## 브라우저에서 렌더링 작동 방식

- **렌더링 작동 방식**: 먼저 브라우저가 HTML을 파싱해 DOM을 만듭니다. 다음으로 CSS를 처리해 CSS 객체 모델(CSSOM)을 구성합니다. CSSOM은 요소 스타일링 방식을 결정하는 또 다른 트리 구조입니다. 마지막으로 브라우저가 계산된 스타일과 레이아웃을 기반으로 픽셀을 화면에 그립니다. 복잡한 페이지에서는 여러 레이어가 합성되어 최종 시각 출력을 만듭니다.

## 성능이 지속 가능성에 미치는 영향

- **배경 정보**: 인터넷은 전 세계 탄소 배출량의 약 2%를 차지하는데, 이는 항공 산업과 비슷한 수준입니다! 데이터 센터부터 사용자 기기까지 모든 바이트 전송에는 전기가 필요합니다. 큰 파일과 비효율적 스크립트는 더 많은 전력 소비를 의미합니다. 고성능 웹사이트는 단순히 빠른 것뿐 아니라 불필요한 처리와 에너지 사용을 줄입니다.

## 페이지 로딩 시간 줄이는 방법

- **미디어 자산 최적화**: 큰 이미지와 비디오는 느린 로드 시간의 주범입니다. 압축, WebP 같은 최신 포맷 사용, 자산 지연 로딩으로 사이트 속도를 크게 높일 수 있습니다.
- **브라우저 캐싱 활용**: 캐싱은 브라우저가 웹사이트 일부를 로컬에 저장해 재방문 시 로드 시간을 줄입니다.
- **파일 축소 및 압축**: 파일 크기를 줄이면 다운로드가 빨라집니다. 전송 파일 크기 축소, CSS와 JavaScript 파일 축소가 포함됩니다.

## "사용 가능 시간" 개선

- **정의**: "사용 가능 시간"은 사용자가 페이지를 요청한 시점부터 의미 있게 상호작용할 수 있을 때까지의 간격입니다. 이를 개선하려면 자산을 지연 로드하거나 렌더링 차단 리소스를 최소화하세요.

## 성능 측정 주요 지표

- **첫 번째 콘텐츠 풀 페인트(FCP)**: 화면에 첫 번째 콘텐츠(텍스트나 이미지)가 나타나는 속도를 측정합니다. 좋은 FCP는 1.8초 이하, 나쁜 FCP는 3초 이상으로 간주합니다. Chrome DevTools의 성능 탭에서 FCP를 확인할 수 있습니다.
- **총 차단 시간(TBT)**: 메인 스레드가 무거운 JavaScript 작업으로 차단된 시간을 보여줍니다. TBT가 높으면 사용자 상호작용이 느려집니다. TBT를 개선하려면 긴 작업을 분할하고 비필수 스크립트를 지연하세요.
- **이탈률**: 상호작용 없이 사이트를 떠난 방문자 비율입니다. 이탈률이 높으면 페이지가 너무 느릴 수 있습니다.
- **고유 사용자 수**: 사이트를 방문한 개별 방문자 수를 추적합니다. 이탈률과 고유 사용자 수는 Google Analytics에서 확인할 수 있으며, 이를 통해 참여도를 개선할 수 있습니다.

## 성능 측정 도구

- **Chrome DevTools**: Google Chrome 내장 도구로 실시간으로 성능을 분석하고 디버깅할 수 있습니다. 로드 시간, CPU 사용량, 렌더 지연을 보여줍니다. 특히 FCP 측정에 유용하며, 사이트가 느리면 병목 현상을 찾는 데 도움을 줍니다.
- **Lighthouse**: 성능, SEO, 접근성을 자동으로 검사하는 도구입니다.
- **WebPageTest**: 다양한 위치와 기기에서 사이트 로드 방식을 테스트할 수 있습니다. 사이트의 속도 지수, 총 차단 시간 등 주요 성능 지표를 상세히 분석합니다. 전 세계 실제 사용자의 경험을 알고 싶다면 WebPageTest가 적합합니다.
- **PageSpeed Insights**: 모바일과 데스크톱 모두에 대해 사이트를 분석하고 빠른 개선책을 제안합니다. 이미지 최적화, 렌더 차단 스크립트 제거, 서버 응답 시간 단축 등 구체적 권고를 제공합니다. Google이 사이트 성능을 어떻게 보는지 빠르고 쉽게 확인할 수 있습니다.
- **실제 사용자 모니터링(RUM)**: 실제 사용자 행동을 추적해 사이트 경험을 보여줍니다. Google Analytics, New Relic, Datadog 같은 도구가 인기 있습니다. 실제 사용자 데이터를 원한다면 RUM 도구가 필수입니다.

## 성능 웹 API 다루기

- **정의**: 성능 웹 API는 개발자가 코드 내에서 웹페이지 로드와 응답 효율성을 추적할 수 있게 합니다. 페이지 로드 시간, 렌더링 및 상호작용 지연, JavaScript 실행 시간을 측정할 수 있습니다.
- **`performance.now()`**: 사이트의 각 부분이 로드되는 시간을 밀리초 단위로 고정밀 타임스탬프로 제공합니다.

```js
const start = performance.now();  
// Run some code here  
const end = performance.now();  

console.log(`Execution time: ${end - start}ms`);
```

- **Performance Timing API**: DNS 조회부터 `DOMContentLoaded`까지 페이지 로드의 모든 단계를 세분화해 보여줍니다.

```js
const timing = performance.timing;  

const pageLoadTime = timing.loadEventEnd - timing.navigationStart;  
console.log(`Page load time: ${pageLoadTime}ms`);
```

- **`PerformanceObserver`**: 레이아웃 이동, 긴 작업, 사용자 상호작용 같은 성능 이벤트를 감지합니다.

```js
const observer = new PerformanceObserver((list) => {  
  list.getEntries().forEach((entry) => {  
    console.log(`Long task detected: ${entry.duration}ms`);  
  });  
});  

observer.observe({ type: "longtask", buffered: true });
```

## CSS 성능 향상 기법

- **CSS 애니메이션**: 크기, 위치, 레이아웃 같은 CSS 속성을 애니메이션하면 브라우저가 "리플로우" 과정을 거쳐 요소 위치와 기하학을 재계산합니다. 이는 다시 칠하기를 유발하며 계산 비용이 큽니다. 따라서 CSS 애니메이션 수를 가능한 줄이거나 사용자에게 켜고 끌 수 있는 옵션을 제공하는 것이 좋습니다.

## JavaScript 성능 향상 기법

- **코드 분할**: JavaScript 코드를 핵심 작업과 비핵심 작업 모듈로 나누면 핵심 모듈을 최대한 빨리 프리로드하고 비핵심 모듈은 지연해 페이지 렌더링 속도를 높일 수 있습니다.
- **DOM 조작**: DOM 조작은 문서 객체 모델(DOM)과 상호작용해 페이지 내용을 동적으로 변경하는 과정입니다. DOM 조작은 계산 비용이 크므로 JavaScript 코드에서 DOM 조작을 줄이면 성능이 향상됩니다.

## CSS 프레임워크

- **CSS 프레임워크**: 워크플로우를 가속화하고, 웹사이트 전반에 일관된 시각 스타일을 만들며, 여러 브라우저에서 디자인 일관성을 유지하고, CSS 코드를 더 체계적으로 관리할 수 있게 합니다.
- **인기 CSS 프레임워크**: Tailwind CSS, Bootstrap, Materialize, Foundation 등이 있습니다.
- **잠재적 단점**:
  - 프레임워크가 제공하는 CSS가 사용자 정의 CSS와 충돌할 수 있습니다.
  - 같은 프레임워크를 사용하는 다른 웹사이트와 비슷해 보일 수 있습니다.
  - 큰 프레임워크는 성능 문제를 일으킬 수 있습니다.

## 두 가지 유형의 CSS 프레임워크

- **유틸리티 우선 CSS 프레임워크**: 여백, 패딩, 배경색 설정 같은 특정 목적의 작은 클래스를 제공합니다. 필요에 따라 HTML 요소에 직접 할당해 디자인을 만듭니다. Tailwind CSS가 이에 해당합니다.

다음은 Tailwind CSS로 버튼을 스타일링하는 예시입니다.

```html
<button class="bg-blue-500 text-white font-bold py-2 px-4 rounded-full hover:bg-blue-700">
  Button
</button>
```

- **컴포넌트 기반 CSS 프레임워크**: 미리 정의된 스타일을 가진 컴포넌트를 제공하며, 공식 문서에서 컴포넌트를 복사해 프로젝트에 붙여넣을 수 있습니다. Bootstrap이 이에 해당합니다.

다음은 Bootstrap을 사용해 리스트 그룹을 만드는 예시입니다. 작은 클래스를 HTML 요소에 적용하는 대신 HTML 구조 전체를 추가합니다.

```html
<div class="card" style="width: 25rem;">
  <ul class="list-group list-group-flush">
    <li class="list-group-item">HTML</li>
    <li class="list-group-item">CSS</li>
    <li class="list-group-item">JavaScript</li>
  </ul>
</div>
```

## Tailwind CSS

Tailwind는 유틸리티 우선 CSS 프레임워크입니다. 사용자 정의 CSS 규칙을 작성하는 대신 HTML 내에서 작은 유틸리티 클래스를 조합해 디자인을 만듭니다.

### 반응형 디자인 유틸리티

Tailwind는 `sm:`, `md:`, `lg:` 같은 접두사를 사용해 다양한 화면 크기에서 스타일을 적용합니다.

```html
<div class="w-full md:w-1/2 lg:flex-row">
  Responsive layout
</div>
```

### Flexbox 유틸리티

`flex`, `flex-col`, `justify-around`, `items-center` 같은 클래스는 유연한 레이아웃을 쉽게 만듭니다.

```html
<div class="flex flex-col md:flex-row justify-around items-center">
  <p>Column on small screens</p>
  <p>Row on medium and larger screens</p>
</div>
```

### 그리드 유틸리티

Tailwind는 `grid`, `grid-cols-1`, `md:grid-cols-3` 같은 CSS 그리드 유틸리티를 포함합니다.

```html
<div class="grid grid-cols-1 md:grid-cols-3 gap-8">
  <div class="bg-gray-100 p-4">Column 1</div>
  <div class="bg-gray-100 p-4">Column 2</div>
  <div class="bg-gray-100 p-4">Column 3</div>
</div>
```

### 간격 유틸리티

`mt-8`, `mx-auto`, `p-4`, `gap-4` 같은 유틸리티는 CSS를 작성하지 않고도 일관된 간격을 만듭니다.

```html
<div class="mt-8 p-4 bg-indigo-600 text-white">
  Spaced content
</div>
```

### 타이포그래피 유틸리티

`uppercase`, `font-bold`, `font-semibold`, `text-4xl` 같은 유틸리티는 텍스트 모양을 제어합니다.

`text-3xl`와 `md:text-5xl` 같은 브레이크포인트에서 조정되는 폰트 크기를 설정할 수 있습니다.

```html
<h1 class="text-3xl md:text-5xl font-semibold text-center">
  Responsive Heading
</h1>
```

### 색상과 호버 상태

Tailwind는 `text-red-700`, `bg-indigo-600`, `bg-gray-100` 같은 다양한 색상 팔레트를 제공합니다.

`hover:bg-pink-600` 같은 클래스는 인터랙티브 효과를 간단하게 만듭니다.

```html
<a href="#" class="bg-pink-500 hover:bg-pink-600 text-white px-4 py-2 rounded-md">
  Hover Me
</a>
```

### 테두리, 링, 효과

- **테두리**: `border-2 border-red-300`는 지정된 두께와 색상의 테두리를 추가합니다.
- **링**: `ring-1 ring-gray-300`는 포커스나 카드에 자주 쓰이는 윤곽선 같은 효과를 만듭니다.
- **둥근 모서리와 스케일링**: `rounded-md`, `rounded-xl`, `scale-105` 같은 클래스는 세련된 효과를 더합니다.

```html
<div class="p-6 rounded-xl ring-2 ring-fuchsia-500 scale-105">
  Highlighted card
</div>
```

### 그라디언트

Tailwind는 `bg-gradient-to-r from-fuchsia-500 to-indigo-600` 같은 그라디언트 유틸리티를 지원합니다.

```html
<div class="p-4 text-white bg-gradient-to-r from-fuchsia-500 to-indigo-600">
  Gradient background
</div>
```

## CSS 전처리기

- **CSS 전처리기**: 표준 CSS를 확장하는 도구로, 확장된 구문을 네이티브 CSS 파일로 컴파일합니다. 복잡한 프로젝트에서 더 깔끔하고 재사용 가능하며 반복을 줄인 확장 가능한 CSS를 작성하는 데 유용합니다.
- **특징**: 변수, 믹스인, 중첩, 선택자 상속 같은 기능을 제공합니다.
- **인기 CSS 전처리기**: Sass, Less, Stylus 등이 있습니다.
- **잠재적 단점**:
  - CSS 규칙을 표준 CSS로 컴파일하는 데 오버헤드가 발생할 수 있습니다.
  - 컴파일된 코드는 디버깅이 어려울 수 있습니다.

## Sass

- **Sass**: 가장 인기 있는 CSS 전처리기 중 하나로, "Syntactically Awesome Style Sheets"의 약자입니다.
- **Sass가 지원하는 기능**: 변수, 중첩 CSS 규칙, 모듈, 믹스인, 상속, 기본 수학 연산자 등을 지원합니다.

## Sass가 지원하는 두 가지 구문

- **SCSS 구문**: CSS 기본 구문을 확장한 것으로, Sass에서 가장 널리 사용됩니다. SCSS 파일은 `.scss` 확장자를 가집니다.

다음은 SCSS에서 변수를 정의하고 사용하는 예시입니다.

```scss
$primary-color: #3498eb;

header {
  background-color: $primary-color;
}
```

- **들여쓰기 구문**: Sass의 원래 구문으로 "Sass 구문"이라고도 합니다.

다음은 들여쓰기 구문에서 변수를 정의하고 사용하는 예시입니다.

```sass
$primary-color: #3498eb

header
  background-color: $primary-color
```

### 믹스인

- **믹스인**: 여러 CSS 속성과 값을 그룹화해 이름으로 묶고, 스타일시트 전반에서 재사용할 수 있게 합니다.

다음은 SCSS 구문으로 믹스인을 정의하는 예시입니다. 이 믹스인 `center-flex`는 플렉스박스를 사용해 요소를 중앙에 배치하는 세 가지 CSS 속성을 가집니다.

```scss
@mixin center-flex {
  display: flex;
  justify-content: center;
  align-items: center;
}
```

정의한 믹스인을 사용하는 예시는 다음과 같습니다.

```scss
section {
  @include center-flex;
  height: 500px;
  background-color: #3289a8;
}
```

## 수동 및 자동 테스트

- **수동 테스트**: 테스터가 애플리케이션 각 부분을 직접 점검하며 기능이 제대로 작동하는지 확인합니다. 버그가 발견되면 소프트웨어 팀에 보고해 수정합니다.
- **자동 테스트**: 별도의 프로그램을 작성해 애플리케이션이 기대한 대로 동작하는지 자동으로 검사합니다.

## 단위 테스트

- **단위 테스트**: 각 함수를 테스트해 모든 기능이 예상대로 작동하는지 확인합니다. 단위 테스트는 코드의 예상 동작을 나타내므로 문서 역할도 합니다.
- **단일 책임 원칙**: 각 함수는 작고 하나의 책임만 가지도록 유지하는 것이 좋습니다.
- **일반적인 JavaScript 테스트 프레임워크**: Jest, Mocha, Vitest 등이 있습니다. Jest는 단위 테스트에 인기 있는 프레임워크입니다.

다음은 Jest를 사용한 단위 테스트 예시입니다.

먼저 새로 포맷된 문자열을 반환하는 함수를 만듭니다:

```javascript
export function getFormattedWord(str) {
  if (!str) return '';
  return str.charAt(0).toUpperCase() + str.slice(1);
}
```

별도의 `getFormattedWord.test.js` 파일에서 함수가 제대로 작동하는지 확인하는 테스트를 작성할 수 있습니다. `getFormattedWord.test.js` 파일은 다음과 같습니다:

```javascript
import { getFormattedWord } from "./getFormattedWord.js";
import { test, expect } from "jest";

test('capitalizes the first letter of a word', () => {
  expect(getFormattedWord('hello')).toBe('Hello');
});
```

- **`expect` 함수**: `expect` 함수는 값을 테스트하는 데 사용됩니다.
- **매처**: 매처는 값이 예상대로 동작하는지 확인하는 함수입니다. 위 예시에서 매처는 `toBe()`입니다. Jest에는 다양한 매처가 있습니다.

Jest를 사용하려면 먼저 `jest` 패키지를 `npm i jest`로 설치해야 합니다. `package.json` 파일에 다음과 같이 스크립트를 추가해야 합니다:

```js
"scripts": {
  "test": "jest"
},
```

그런 다음 `npm run test` 명령어로 테스트를 실행할 수 있습니다.

## 소프트웨어 개발 생명주기

- **소프트웨어 개발 생명주기의 다양한 단계**:
  - **기획 단계**: 개발팀이 이해관계자로부터 요구사항을 수집합니다.
  - **설계 단계**: 소프트웨어 팀이 요구사항을 분해하고 최적의 해결책을 결정합니다.
  - **구현 단계**: 소프트웨어 팀이 요구사항을 관리 가능한 작업으로 나누고 구현합니다.
  - **테스트 단계**: 수동 및 자동 테스트를 통해 새 작업을 검증합니다. 때로는 개발 전반에 걸쳐 애플리케이션을 테스트해 문제를 발견하고 수정합니다.
  - **배포 단계**: 새 변경사항을 빌드 또는 테스트 환경에 배포합니다.
  - **유지보수 단계**: 프로덕션 애플리케이션에서 발생하는 문제를 수정합니다.

- **소프트웨어 개발 생명주기의 다양한 모델**:
  - **워터폴 모델**: 각 단계가 완료되어야 다음 단계로 넘어갑니다.
  - **애자일 모델**: 작업을 스프린트로 나누어 반복적으로 개발합니다.

## TypeScript란?

- **JavaScript**: JavaScript는 동적 타입 언어로, 변수에 런타임에 어떤 값도 할당할 수 있습니다. 동적 타입 언어의 문제는 타입 안전성이 부족해 오류가 발생할 수 있다는 점입니다.

예를 들어, JavaScript 함수가 배열을 기대해도 숫자를 인수로 호출할 수 있습니다:

```javascript
const getRandomValue = (array) => {
  return array[Math.floor(Math.random() * array.length)];
}

console.log(getRandomValue(10));
```

위 예시의 `console` 출력은 `undefined`입니다.

- **TypeScript**: TypeScript는 JavaScript에 정적 타입을 추가해 타입 불일치로 인한 오류를 코드 실행 전에 잡아줍니다.

예를 들어 `array` 매개변수의 타입을 다음과 같이 정의할 수 있습니다:

```typescript
const getRandomValue = (array: string[]) => {
  return array[Math.floor(Math.random() * array.length)];
}
```

이 타입 정의는 `array` 매개변수가 문자열 배열이어야 함을 TypeScript에 알립니다. 그런 다음 `getRandomValue`를 호출하며 숫자를 전달하면 컴파일러 오류가 발생합니다.

- **컴파일러**: TypeScript 코드를 일반 JavaScript로 컴파일해야 합니다. 컴파일러는 코드를 평가해 타입이 맞지 않는 부분에 오류를 발생시킵니다.

## TypeScript의 데이터 타입

- **원시 데이터 타입**: `string`, `null`, `undefined`, `number`, `boolean`, `bigint`에 대응하는 타입 키워드를 제공합니다.

```typescript
let str: string = "Naomi";
let num: number = 42;
let bool: boolean = true;
let nope: null = null;
let nada: undefined = undefined;
```

- **배열**: 특정 타입 배열을 두 가지 구문으로 정의할 수 있습니다.

```typescript
const arrOne: string[] = ["Naomi"];
const arrTwo: Array<string> = ["Camperchan"];
```

- **객체**: 객체의 정확한 구조를 정의할 수 있습니다.

```typescript
const obj: { a: string, b: number } = { a: "Naomi", b: 42 };
```

모든 키가 있지만 값이 모두 문자열이어야 하는 객체를 정의하는 두 가지 방법은 다음과 같습니다:

```typescript
const objOne: Record<string, string> = {};
const objTwo: { [key: string]: string } = {};
```

- **TypeScript의 기타 유용한 타입**:
  - **`any`**: `any`는 값이 어떤 타입이든 될 수 있음을 나타냅니다. 컴파일러에게 해당 변수 타입을 신경 쓰지 말라고 지시합니다.
  - **`unknown`**: `unknown`는 값 타입을 신경 쓰지만 실제 타입은 모를 때 사용합니다. `unknown`가 일반적으로 `any`보다 선호됩니다.
  - **`void`**: 주로 반환값이 없는 함수를 정의할 때 사용하는 특별한 타입입니다. 반환 타입으로 `void`를 사용합니다.
  - **`never`**: 절대 존재하지 않는 타입을 나타냅니다.

- **`type` 키워드**: `const`와 비슷하지만 변수 선언 대신 타입을 선언할 때 사용합니다.

합집합 타입이나 특정 값만 포함하는 타입 같은 커스텀 타입 선언에 유용합니다:

```typescript
type stringOrNumber = string | number;
type bot = "camperchan" | "camperbot" | "naomi";
```

- **`interface`**: 인터페이스는 타입용 클래스와 같습니다. 다른 인터페이스를 구현하거나 확장할 수 있고, 주로 객체 타입이며, `type` 선언에서 제공하는 특정 기능이 필요하지 않으면 선호됩니다.

```typescript
interface wowie {
  zowie: boolean;
  method: () => void;
}
```

- **반환 타입 정의**: 함수의 반환 타입도 정의할 수 있습니다.

아래 예시는 반환값을 문자열로 정의합니다. 다른 값을 반환하면 컴파일러 오류가 발생합니다.

```typescript
const getRandomValue = (array: string[]): string => {
  return array[Math.floor(Math.random() * array.length)];
}
```

## 제네릭

- **제네릭 타입 정의**: 함수의 타입 정의 동작을 제어하기 위해 특별한 매개변수처럼 제네릭 타입을 정의할 수 있습니다.

다음은 함수에 제네릭 타입을 정의하는 예시입니다:

```typescript
const getRandomValue = <T>(array: T[]): T => {
  return array[Math.floor(Math.random() * array.length)];
}
const val = getRandomValue([1, 2, 4])
```

`<T>` 구문은 함수에 대해 제네릭 타입 `T`를 정의함을 TypeScript에 알립니다. `T`는 제네릭 타입의 일반적인 이름이지만 아무 이름이나 사용할 수 있습니다.

그런 다음 `array` 매개변수가 제네릭 타입에 맞는 값 배열이며, 반환값은 같은 타입의 단일 요소임을 알립니다.

- **함수 호출 시 타입 인수 지정**: 함수 이름과 매개변수 사이에 꺾쇠 괄호를 사용해 타입 인수를 전달할 수 있습니다.

다음은 함수 호출에 타입 인수를 전달하는 예시입니다:

```typescript
const email = document.querySelector<HTMLInputElement>("#email");
console.log(email.value);
```

이는 TypeScript에 찾으려는 요소가 입력 요소임을 알려줍니다.

## 타입 좁히기

- **진리값에 의한 좁히기**: 아래 예시에서 `value` 속성에 접근하려 하면 컴파일러 오류가 발생합니다. `email`의 `email` 속성이 `null`일 수도 있기 때문입니다.

```typescript
const email = document.querySelector<HTMLInputElement>("#email");
console.log(email.value);
```

조건문을 사용해 `email`가 참인지 확인한 후 속성에 접근할 수 있습니다:

```typescript
const email = document.querySelector<HTMLInputElement>("#email");
if (email) {
  console.log(email.value);
}
```

진리값 검사는 반대 방향으로도 작동합니다:

```typescript
const email = document.querySelector<HTMLInputElement>("#email");
if (!email) {
  throw new ReferenceError("Could not find email element!")
}
console.log(email.value);
```

오류를 발생시키면 논리 실행이 종료되어 `console.log()` 호출 시 TypeScript는 `email`가 `null`일 수 없음을 알게 됩니다.

- **선택적 체이닝**: 선택적 체이닝 `?.`도 타입 좁히기의 한 형태로, `email` 값이 `null`이면 속성 접근이 발생하지 않습니다.

```typescript
const email = document.querySelector<HTMLInputElement>("#email");
console.log(email?.value);
```

- **`typeof` 연산자**: `typeof` 연산자를 사용해 변수의 타입을 조건문으로 확인할 수 있습니다.

```typescript
const myVal = Math.random() > 0.5 ? 222 : "222";
if (typeof myVal === "number") {
  console.log(myVal / 10);
}
```

- **`instanceof` 키워드**: 객체가 클래스에서 왔으면 `instanceof` 키워드를 사용해 타입을 좁힐 수 있습니다.

```typescript
const email = document.querySelector("#email");

if (email instanceof HTMLInputElement) {
    console.log(email.value);
}
```

- **캐스팅**: TypeScript가 아래 예시의 `request.json()` 메서드 결과 타입을 자동으로 결정하지 못하면 컴파일러 오류가 발생합니다. 이를 해결하는 한 방법은 타입 캐스팅이지만, 이는 TypeScript의 오류 감지 능력을 약화시킵니다.

```typescript
interface User {
    name: string;
    age: number;
}

const printAge = (user: User) => 
  console.log(`${user.name} is ${user.age} years old!`)

const request = await fetch("url")
const myUser = await request.json() as User;
printAge(myUser);
```

- **타입 가드**: 타입 캐스팅 대신 타입 가드를 작성할 수 있습니다:

```typescript
interface User {
    name: string;
    age: number;
}

const isValidUser = (user: unknown): user is User => {
  return !!user && 
    typeof user === "object" &&
    "name" in user &&
    "age" in user;
}
```

`user is User` 구문은 함수가 불리언 값을 반환하며, true일 때 `user` 값이 `User` 인터페이스를 만족함을 나타냅니다.

## `tsconfig` 파일

- **`tsconfig.json`**: TypeScript 컴파일러 설정은 프로젝트 루트 디렉터리의 `tsconfig.json` 파일에 있습니다.

```json
{
  "compilerOptions": {
    "rootDir": "./src",
    "outDir": "./prod",
    "lib": ["ES2023"],
    "target": "ES2023",
    "module": "ES2022",
    "moduleResolution": "Node",
    "esModuleInterop": true,
    "skipLibCheck": true,
    "strict": true
  },
  "exclude": ["test/"]
}
```

위 예시에서 사용된 컴파일러 옵션 설명은 다음과 같습니다:

- **`compilerOptions`**: `compilerOptions` 속성은 TypeScript 컴파일러 동작을 제어합니다.
- **`rootDir`와 `outDir`**: `rootDir`와 `outDir`는 소스 파일이 있는 디렉터리와 트랜스파일된 JavaScript 코드가 들어갈 디렉터리를 지정합니다.
- **`lib`**: `lib` 속성은 컴파일러가 사용할 타입 정의를 결정하며, 특정 ES 릴리스, DOM 지원 등을 포함할 수 있습니다.
- **`module`와 `moduleResolution`**: `module`와 `moduleResolution`는 CommonJS 또는 ECMAScript 모듈 사용 방식을 관리합니다.
- **`esModuleInterop`**: `esModuleInterop`는 CommonJS와 ES 모듈 간 원활한 상호 운용성을 위해 자동으로 네임스페이스 객체를 생성합니다.
- **`skipLibCheck`**: `skipLibCheck` 옵션은 코드에서 가져오지 않는 `.d.ts` 파일 검증을 건너뜁니다.
- **`strict`**: `strict` 플래그는 널 가능 타입 처리 보장, TypeScript가 `any`로 대체될 때 경고 등 여러 검사를 활성화합니다.
- **`exclude`**: 최상위 `exclude` 속성은 컴파일 시 이 TypeScript 파일들을 무시하도록 컴파일러에 지시합니다.
