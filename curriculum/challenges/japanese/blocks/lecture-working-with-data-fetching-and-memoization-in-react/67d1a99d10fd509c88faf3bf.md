# --description--

Reactアプリは動的なコンテンツのために外部のAPIやデータベースに依存することがよくあります。これらのAPIやデータベースからデータにアクセスするには、いくつかのデータフェッチングの手法を使う必要があります。

Reactでのデータフェッチングの仕組みと、データを取得するために利用できるさまざまな選択肢を見ていきましょう。

Reactはデータの取得方法について特に決まりを設けていません。基本的には、すべての最新ブラウザがサポートしている組み込みのFetch APIを使うことができます。

また、AxiosやSWRを使うこともできます。AxiosはXMLHttpRequestオブジェクトの上に構築されたPromiseベースのHTTPリクエストライブラリで、SWRはVercelチームが作成したReactのデータフェッチング用フックです。

まずは例から始めましょう。最初に`useState`と`useEffect`のフックをインポートする必要があります。

```js
import { useState, useEffect } from "react";
```

次に、`loading`、`data`、`error`という3つの状態変数を作成します。

```js
const [data, setData] = useState(null);
const [loading, setLoading] = useState(true);
const [error, setError] = useState(null);
```

`loading`変数はデータがまだ取得中かどうかを追跡します。`data`変数はデータそのものを表し、`error`変数はデータフェッチング中に発生する可能性のあるエラーをキャッチします。

データフェッチングは副作用なので、Fetch APIは`useEffect`フックの中で使うのが最適です。

以下はその例です。

```js
useEffect(() => {
  fetch("https://jsonplaceholder.typicode.com/posts")
    .then((res) => res.json())
    .then((data) => {
      setData(data);
      setLoading(false);
    })
    .catch((err) => {
      setError(err);
      setLoading(false);
    });
}, []);
```

この`useEffect`はFetch APIでデータを取得し、すべての状態を設定します。

`async`構文の代わりに`await`/`.then()`を使うことでより良くできます。つまり、`useEffect`を`useEffect`キーワードで接頭辞できないため、`async`の中に別の関数を用意する必要があります。

```js
useEffect(() => {
  const fetchData = async () => {
    try {
      const res = await fetch("https://jsonplaceholder.typicode.com/posts");
       
      if (!res.ok) {
        throw new Error("Network response was not ok");
      }

      const data = await res.json();
      setData(data);
    } catch (err) {
      setError(err);
    } finally {
      setLoading(false);
    }
  };

  fetchData();
}, []);
```

その後、これらの状態を使ってAPIからのデータをレンダリングできます。

完全なコードはこちらです。

```jsx
import { useState, useEffect } from "react";

const FetchPosts = () => {
  const [data, setData] = useState(null);
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState(null);

  useEffect(() => {
    const fetchData = async () => {
      try {
        const res = await fetch("https://jsonplaceholder.typicode.com/posts");
       
        if (!res.ok) {
          throw new Error("Network response was not ok");
        }

        const data = await res.json();
        setData(data);
      } catch (err) {
        setError(err);
      } finally {
        setLoading(false);
      }
    };

    fetchData();
  }, []);

  if (loading) {
    return <p>Loading...</p>;
  }

  if (error) {
    return <p>{error.message}</p>;
  }

  return (
    <ul>
      {data.map((post) => (
        <li key={post.id}>{post.title}</li>
      ))}
    </ul>
  );
};

export default FetchPosts;
```

UIでは、データ取得中は`Loading...`が画面に表示され、データ取得が成功すればデータが、失敗すればエラーが表示されます。

AxiosとSWRを使ったデータフェッチングについても話しましたね。次にAxiosを使った例を見てみましょう。

まずはコマンドラインからAxiosをインストールする必要があります。

```sh
npm i axios
```

次にAxiosをこのようにインポートします。

```js
import axios from "axios";
```

先ほどの状態変数を使い、`axios.get`を使ってAPIからデータを取得できます。

```js
const [data, setData] = useState(null);
const [loading, setLoading] = useState(true);
const [error, setError] = useState(null);

useEffect(() => {
  const fetchData = async () => {
    try {
      const res = await axios.get(
        "https://jsonplaceholder.typicode.com/users"
      );
      setData(res.data);
    } catch (err) {
      setError(err);
    } finally {
      setLoading(false);
    }
  };

  fetchData();
}, []);
```

この例には`await res.json()`の行がないことに気づいたかもしれません。AxiosはJSONを自動的にパースするため、その必要がないからです。

最後の例は`useSWR`フックを使ってデータを取得する方法です。

Axiosと同様に、まずSWRをこのようにインストールします。

```sh
npm install swr
```

次に`useSWR`フックをファイルにインポートします。

```js
import useSWR from "swr";
```

前の例と比べて、SWRの構文はずっと短いです。fetcher関数を作成し、それを`useSWR`フックの第2パラメータとして渡します（エンドポイントが第1パラメータです）。

また、`useSWR`フックからdataとerrorの両方の状態を分割代入できるため、`useState`フックは不要です。

構文は以下の通りです。

```js
const fetcher = (url) => fetch(url).then((res) => res.json());
const { data, error } = useSWR(endpoint, fetcher);
```

ここでの「fetcher」という名前は慣例に過ぎないので、好きな名前を付けてかまいません。

JSON Placeholder APIからtodosを取得するコンポーネントはこちらです。

```jsx
import useSWR from "swr";

const fetcher = (url) => fetch(url).then((res) => res.json());

const FetchTodos = () => {
  const { data, error } = useSWR(
    "https://jsonplaceholder.typicode.com/todos",
    fetcher
  );

  if (!data) {
    return <h2>Loading...</h2>;
  }
  if (error) {
    return <h2>Error: {error.message}</h2>;
  }

  return (
    <>
      <h2>Todos</h2>
      <div>
        {data.map((todo) => (
          <h3 key={todo.id}>{todo.title}</h3>
        ))}
      </div>
    </>
  );
};

export default FetchTodos;
```

カスタムフックに関する前のレッスンで学んだように、データフェッチングはカスタムフックに抽出できるロジックです。複数のコンポーネントやページでデータを取得する場合は、`useFetch`フックを作成するのが最適です。

SWRを使った`useFetch`フックはこちらです。

```jsx
import useSWR from "swr";

const fetcher = (url) => fetch(url).then((res) => res.json());

const useFetch = (url) => {
  const { data, error } = useSWR(url, fetcher);

  return {
    data,
    loading: !data && !error,
    error,
  };
};

export default useFetch;
```

そして、JSON Placeholder APIから投稿を取得する最初の例を`useFetch`フックを使って書き換える方法はこちらです。

```jsx
import useFetch from "./useFetch";

const FetchPosts = () => {
  const { data, loading, error } = useFetch(
    "https://jsonplaceholder.typicode.com/posts"
  );

  if (loading) {
    return <h2>Loading...</h2>;
  }

  if (error) {
    return <h2>{error.message}</h2>;
  }

  return (
    <>
      <h2>Posts</h2>
      <ul>
        {data.map((post) => (
          <li key={post.id}>{post.title}</li>
        ))}
      </ul>
    </>
  );
};

export default FetchPosts;
```
