# --description--

`Provider` コンポーネントは、Reactコンポーネントに `state` と `dispatch` を提供できますが、どの状態とアクションを使いたいかを正確に指定する必要があります。こうすることで、各コンポーネントが必要な状態にのみアクセスできるようになります。これを実現するには、`mapStateToProps()` と `mapDispatchToProps()` という2つの関数を作成します。

これらの関数では、アクセスしたい状態の部分とディスパッチできるようにしたいアクションクリエーターを宣言します。これらの関数が用意できたら、別のチャレンジで React Redux の `connect` メソッドを使ってコンポーネントに接続する方法を学びます。

**注意:** 裏側では、React Redux は `store.subscribe()` メソッドを使って `mapStateToProps()` を実装しています。

# --instructions--

`mapStateToProps()` という関数を作成してください。この関数は `state` を引数に取り、その状態を特定のプロパティ名にマッピングしたオブジェクトを返す必要があります。これらのプロパティは `props` を通じてコンポーネントからアクセス可能になります。この例ではアプリの状態全体を単一の配列で保持しているので、その状態全体をコンポーネントに渡せます。返されるオブジェクトに `messages` というプロパティを作成し、それを `state` に設定してください。

# --hints--

const `state` は空の配列であるべきです。

```js
assert(Array.isArray(state) && state.length === 0);
```

`mapStateToProps` は関数であるべきです。

```js
assert(typeof mapStateToProps === 'function');
```

`mapStateToProps` はオブジェクトを返すべきです。

```js
assert(typeof mapStateToProps() === 'object');
```

配列を状態として `mapStateToProps` に渡すと、その配列が `messages` のキーに代入されて返されるべきです。

```js
assert(mapStateToProps(['messages']).messages.pop() === 'messages');
```
