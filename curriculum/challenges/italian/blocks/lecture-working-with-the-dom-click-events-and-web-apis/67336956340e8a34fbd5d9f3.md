# --interactive--

Creare animazioni fluide su una pagina web può essere complicato, soprattutto se non sai come gestire correttamente la temporizzazione. La buona notizia è che l’API `requestAnimationFrame()` rende tutto più semplice.

`requestAnimationFrame()` è un metodo che ti permette di programmare il passaggio successivo della tua animazione prima del prossimo aggiornamento dello schermo, ottenendo così un’esperienza visiva fluida e piacevole.

Il prossimo aggiornamento dello schermo indica il momento in cui il browser aggiorna la visualizzazione della pagina web. Questo avviene più volte al secondo, tipicamente circa 60 volte (o 60 frame al secondo) sulla maggior parte dei display.

Per usare il metodo `requestAnimationFrame()`, tutto ciò che devi fare è chiamarlo e passargli una funzione di callback:

```js
requestAnimationFrame(callback);
```

La chiamata a `requestAnimationFrame()` deve avvenire all’interno di una funzione che gestisce l’animazione, come `animate()`, insieme a una funzione per aggiornare l’animazione, tradizionalmente chiamata `update()`:

```js
function animate() {
 // Update the animation...
 // for example, move an element, change a style, and more.
 update();
 // Request the next frame
 requestAnimationFrame(animate);
}
```

La funzione `update()` è dove avviene la magia. Al suo interno puoi modificare tutto ciò che vuoi animare. Per esempio, aggiornare uno stile o cambiare la posizione di un elemento:

```js
function update() {
 element.style.transform = `translateX(${position}px)`;
 position += 2;
}
```

L’animazione parte infine chiamando `requestAnimationFrame()` e passando la funzione `animate`, questa volta all’esterno della funzione `animate`:

```js
requestAnimationFrame(animate);
```

Il loop continuerà finché non lo fermerai.

Ora, diamo un’occhiata a un altro esempio. L’HTML di questo esempio è un elemento `div` con il testo `freeCodeCamp is Awesome`. Il CSS trasforma il `div` in un rettangolo e nasconde tutto ciò che esce dal viewport a sinistra o a destra. Il codice JavaScript sposta il rettangolo `2px` verso destra a ogni chiamata di `requestAnimationFrame(animate)` con la funzione `animate` come callback:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css" />
<div id="rect" class="rect">freeCodeCamp is Awesome</div>
<script src="index.js"></script>
```

```css
body {
  overflow-x: hidden;
}

.rect {
  display: flex;
  align-items: center;
  justify-content: center;
  width: 400px;
  height: 250px;
  border-radius: 5px;
  background-color: #1b1b32;
  color: #f5f6f7;
  font-size: 2rem;
  position: absolute;
}
```

```js
const rect = document.getElementById("rect");

let position = 0;

function update() {
  // Move the rectangle 2px to the right
  rect.style.left = position + "px";
  position += 2;

  if (position > window.innerWidth) {
    // Move the rectangle just outside the left side of the screen
    position = -rect.offsetWidth;
  }
}

function animate() {
  update();

  //request the next frame
  requestAnimationFrame(animate);
}

// Start the animation
requestAnimationFrame(animate);
```

:::

Il risultato nel browser sarà una scheda titolo animata che scorre sullo schermo.
