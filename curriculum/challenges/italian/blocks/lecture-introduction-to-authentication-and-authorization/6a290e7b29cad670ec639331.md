# --description--

Configurare un'autenticazione personalizzata richiede di scrivere una quantità significativa di logica manuale. Devi gestire la lettura delle intestazioni della richiesta, l'estrazione dei cookie, la verifica delle firme e la gestione degli errori.

Costruirlo da solo è ottimo per imparare, ma farlo per ogni applicazione in produzione richiede tempo ed è soggetto a errori di sicurezza. Per questo gli sviluppatori usano Passport.js.

Passport.js è un middleware di autenticazione per Node.js. Funziona come un framework organizzato che standardizza il modo in cui autentichi le richieste. Gestisce i dettagli di basso livello dell'autenticazione, permettendoti di concentrarti sulla logica di business della tua applicazione.

L'idea principale dietro Passport.js è che ogni applicazione richiede modi diversi per effettuare il login, ma il flusso di autenticazione sottostante dovrebbe rimanere lo stesso. Passport ottiene questa flessibilità tramite plugin modulari chiamati **Strategies**.

Invece di scrivere codice personalizzato per ogni metodo di autenticazione, installi la strategy specifica di cui hai bisogno. Ci sono oltre 500 strategie disponibili, che coprono quasi tutti i metodi di autenticazione usati oggi. Alcune di queste sono:

- **passport-local:** Gestisce l'autenticazione tradizionale con nome utente e password contro il tuo database.

- **passport-jwt:** Protegge gli endpoint API estraendo e validando JSON Web Token dalle richieste in arrivo.

- **passport-google-oauth20:** Permette agli utenti di accedere immediatamente usando i loro account Google esistenti.

- **passport-facebook:** Accesso con account Facebook tramite OAuth 2.0.

- **passport-github2:** Permette agli utenti di accedere immediatamente usando i loro account GitHub.

Suddividendo l'autenticazione in strategie distinte, Passport permette alla tua applicazione di crescere senza problemi. Se inizi oggi con un sistema username/password e decidi di aggiungere "Accedi con Google" il mese prossimo, non devi riscrivere tutto il codice di autenticazione. Devi solo installare la strategy Google e collegarla a Passport.

Per usare Passport in un'app Express, devi installare il pacchetto core di Passport insieme alla strategy specifica che intendi usare. Per una configurazione standard con username e password, esegui questo comando nel terminale:

```bash
npm install passport passport-local
```

Una volta installato, devi integrare Passport nella configurazione della tua app Express. Questo deve avvenire prima di definire le tue rotte:

```js
const express = require('express');
const passport = require('passport');
const LocalStrategy = require('passport-local').Strategy;

const app = express();

// initialize Passport middleware
app.use(passport.initialize());

// configure Passport to use the Local Strategy
passport.use(new LocalStrategy((username, password, done) => {
    // your database look-up and password verification logic goes here
}));
```

Passport si integra perfettamente nel sistema di routing standard di Express come middleware. Quando un utente invia le sue credenziali alla tua rotta di login, Passport intercetta la richiesta, passa i dati alla strategy configurata e valuta il risultato.

```js
app.post('/login', passport.authenticate('local'), (req, res) => {
  // if this function runs, then authentication was successful
  res.send(`Welcome back, ${req.user.username}!`);
});
```

Se la strategy verifica con successo l'utente, Passport gestisce automaticamente due attività critiche:

1. **Gestione della sessione:** Attiva funzioni di supporto per memorizzare un piccolo pezzo di dati utente all'interno del cookie di sessione.

2. **Decorazione della richiesta:** Allegare l'oggetto utente autenticato direttamente alla richiesta in arrivo come `req.user`.

Una volta che `req.user` è popolato, qualsiasi middleware o gestore di rotta successivo nella tua applicazione può vedere immediatamente chi è loggato e prendere decisioni di autorizzazione basate sulla loro identità.

Passport fornisce una base coerente, sicura e pronta per la produzione per gestire le identità degli utenti nelle tue applicazioni Node.js.
