# --interactive--

La ricorsione è una funzionalità complessa che ti permette di chiamare ripetutamente una funzione finché non si raggiunge un caso base. A differenza di un loop tradizionale, la ricorsione ti consente di gestire qualcosa con una profondità sconosciuta, come oggetti/array annidati in profondità o un albero dei file. Ma puoi usarla anche per attività più semplici, come contare a ritroso da un numero dato.

Costruiamo una funzione che faccia proprio questo. La chiameremo `recursiveCountdown` e deve accettare un numero. La faremo stampare questo numero nella console:

:::interactive_editor

```js
const recursiveCountdown = (number) => {
  console.log(number);
};

recursiveCountdown(5);
```

:::

Ora, se la chiamiamo passando il numero 5, vedremo il numero stampato nel terminale. Ma non succede altro – e il numero 5 sicuramente non è un conto alla rovescia.

Prima di iniziare a costruire la parte ricorsiva della nostra funzione, dobbiamo prima stabilire il caso base. Se non hai un caso base stabilito, il tuo codice continuerà a girare finché non supera la memoria disponibile e va in crash.

:::interactive_editor

```js
const recursiveCountdown = (number) => {
  if (number < 1) {
    return;
  }
  console.log(number);
};

recursiveCountdown(5);
```

:::

Per il nostro caso base, vogliamo che il conto alla rovescia si fermi se il numero è minore di 1. Quando raggiungiamo quel caso base, possiamo usare return per uscire dall’esecuzione della funzione.

Ora che abbiamo preparato in sicurezza un caso base, possiamo impostare la ricorsione. Il punto chiave che rende una funzione ricorsiva è che chiama se stessa durante l’esecuzione. In questo caso, vogliamo chiamare la funzione dopo aver stampato il numero. Ma per contare a ritroso, il nostro nuovo numero deve essere uno in meno:

:::interactive_editor

```js
const recursiveCountdown = (number) => {
  if (number < 1) {
    return;
  }
  console.log(number);
  recursiveCountdown(number - 1);
};

recursiveCountdown(5); 
```

:::

Questo stamperà i numeri 5, 4, 3, 2 e 1 nella console.

Otteniamo i nostri cinque numeri! Ma cosa succede se volessimo contare in avanti invece? Invece di scrivere una funzione completamente nuova, possiamo invertire l’ordine della stampa e della chiamata ricorsiva:

:::interactive_editor

```js
const recursiveCountdown = (number) => {
    if (number < 1) {
        return;
    }
    recursiveCountdown(number - 1);
    console.log(number);
  };

recursiveCountdown(5);
```

:::

Questo stamperà i numeri 1, 2, 3, 4 e 5 nella console.

Ma perché funziona così? Per capirlo devi comprendere lo stack delle chiamate. Lo stack delle chiamate è come JavaScript tiene traccia e risolve le chiamate di funzione. Lo stack funziona un po’ come una coda last-in-first-out. Per capire meglio, aggiungiamo qualche log alla nostra funzione:

:::interactive_editor

```js
const recursiveCountdown = (number) => {
  console.log(`Function execution started for number: ${number}`);
  if (number < 1) {
    console.log(`Base case reached, begin resolving stack`);
    return;
  }
  console.log(`Calling recursiveCountdown with number: ${number - 1}`);
  recursiveCountdown(number - 1);
  console.log(`Function execution completed for number: ${number}`);
};

recursiveCountdown(5);
```

:::

Abbiamo aggiunto quattro istruzioni chiave qui. Il primo log viene eseguito quando una chiamata di funzione inizia l’esecuzione. Il terzo log viene eseguito subito prima che la funzione ricorsiva venga chiamata. E il quarto log viene eseguito quando l’esecuzione della funzione è terminata. Il risultato è:

```md
Function execution started for number: 5
Calling recursiveCountdown with number: 4
Function execution started for number: 4
Calling recursiveCountdown with number: 3
Function execution started for number: 3
Calling recursiveCountdown with number: 2
Function execution started for number: 2
Calling recursiveCountdown with number: 1
Function execution started for number: 1
Calling recursiveCountdown with number: 0
Function execution started for number: 0
Base case reached, begin resolving stack
Function execution completed for number: 1
Function execution completed for number: 2
Function execution completed for number: 3
Function execution completed for number: 4
Function execution completed for number: 5
```

Ma come succede questo? Qui entra in gioco lo stack delle chiamate. Quando chiamiamo `recursiveCountdown(5)`, quella chiamata di funzione viene aggiunta allo stack delle chiamate.

Quando quella chiamata arriva al punto in cui deve chiamare `recursiveCountdown(4)`, deve fermarsi e aspettare quel risultato. Nel frattempo, `recursiveCountdown(4)` viene aggiunto allo stack, sopra `recursiveCountdown(5)`.

Quando quella chiamata arriva al punto in cui deve chiamare `recursiveCountdown(3)`, deve fermarsi e aspettare quel risultato. Nel frattempo, `recursiveCountdown(3)` viene aggiunto allo stack, sopra `recursiveCountdown(4)`.

Quando quella chiamata arriva al punto in cui deve chiamare `recursiveCountdown(2)`, deve fermarsi e aspettare quel risultato. Nel frattempo, `recursiveCountdown(2)` viene aggiunto allo stack, sopra `recursiveCountdown(3)`.

Quando quella chiamata arriva al punto in cui deve chiamare `recursiveCountdown(1)`, deve fermarsi e aspettare quel risultato. Nel frattempo, `recursiveCountdown(1)` viene aggiunto allo stack, sopra `recursiveCountdown(2)`.

E infine, quando quella chiamata arriva al punto in cui deve chiamare `recursiveCountdown(0)`, deve fermarsi e aspettare quel risultato. Nel frattempo, `recursiveCountdown(0)` viene aggiunto allo stack, sopra `recursiveCountdown(1)`.

Ma `recursiveCountdown(0)` non chiama un’altra funzione – raggiunge il nostro caso base, dove ritorna subito. Poiché l’esecuzione di quella funzione è terminata, la chiamata può essere considerata “risolta”. Quando la chiamata è risolta, viene rimossa dallo stack.

Ora `recursiveCountdown(1)` non aspetta più quella chiamata – è in cima allo stack e può riprendere l’esecuzione.

`recursiveCountdown(1)` si risolve, viene rimossa dallo stack e permette a `recursiveCountdown(2)` di riprendere l’esecuzione.

`recursiveCountdown(2)` si risolve, viene rimossa dallo stack e permette a `recursiveCountdown(3)` di riprendere l’esecuzione.

`recursiveCountdown(3)` si risolve, viene rimossa dallo stack e permette a `recursiveCountdown(4)` di riprendere l’esecuzione.

`recursiveCountdown(4)` si risolve, viene rimossa dallo stack e permette a `recursiveCountdown(5)` di riprendere l’esecuzione.

E `recursiveCountdown(5)` si risolve e viene rimossa dallo stack. Ora il nostro stack delle chiamate è vuoto, quindi la ricorsione è completa!

Questa è una panoramica base di come funziona la ricorsione in JavaScript. È un concetto complesso, e dovresti sperimentare con del codice e delle istruzioni di log finché non ti senti a tuo agio con il comportamento dello stack delle chiamate.

Per una curiosità, abbiamo detto che la mancanza di un caso base può far crashare il codice quando finisce la memoria. Questo succede perché la ricorsione continua ad accumulare sempre più chiamate di funzione nello stack fino a quando lo stack fa overflow. Proprio come il nome della popolare comunità di programmazione.
