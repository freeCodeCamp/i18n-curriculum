## Sifa za kawaida za ARIA za ufikikaji

- **Sifa ya `aria-expanded`**: Inatumika kuonyesha hali ya kipengele cha kubadili (au kufichua) kwa watumiaji wa kisomaji cha skrini.

:::interactive_editor

```html
<button id="menuBtn" aria-expanded="false">Menu</button>

<script>
  const btn = document.getElementById("menuBtn");

  btn.addEventListener("click", () => {
    const expanded = btn.getAttribute("aria-expanded") === "true";
    btn.setAttribute("aria-expanded", String(!expanded));
  });
</script>
```

:::

- **Sifa ya `aria-haspopup`**: Hali hii inatumika kuonyesha kuwa kipengele cha mwingiliano kitasababisha kuonekana kwa kipengele cha pop-up wakati kitakapozimwa. Unaweza kutumia sifa ya `aria-haspopup` tu wakati pop-up ina mojawapo ya majukumu haya: `menu`, `listbox`, `tree`, `grid`, au `dialog`. Thamani ya `aria-haspopup` lazima iwe mojawapo ya majukumu haya au `true`, ambayo ni sawa na `menu`.

:::interactive_editor

```html
<button
  id="menubutton"
  aria-haspopup="menu"
  aria-controls="filemenu"
  aria-expanded="false"
>
  File
</button>

<ul
  id="filemenu"
  role="menu"
  aria-labelledby="menubutton"
  hidden
>
  <li role="menuitem" tabindex="-1">Open</li>
  <li role="menuitem" tabindex="-1">New</li>
  <li role="menuitem" tabindex="-1">Save</li>
  <li role="menuitem" tabindex="-1">Delete</li>
</ul>

<script>
  const button = document.getElementById("menubutton");
  const menu = document.getElementById("filemenu");

  button.addEventListener("click", () => {
    const expanded = button.getAttribute("aria-expanded") === "true";

    button.setAttribute("aria-expanded", String(!expanded));
    menu.hidden = expanded;
  });
</script>
```

:::

- **Sifa ya `aria-checked`**: Sifa hii inatumika kuonyesha kama kipengele kiko katika hali ya kuchaguliwa. Inatumiwa zaidi wakati wa kuunda visanduku vya kuchagua maalum, vitufe vya radio, vitufe vya kubadilisha, na jedwali za orodha.

:::interactive_editor

```html
<div
  id="checkbox"
  role="checkbox"
  aria-checked="true"
  tabindex="0"
  style="
    display: inline-flex;
    align-items: center;
    gap: 6px;
    cursor: pointer;
  "
>
  <span
    id="box"
    aria-hidden="true"
    style="
      width: 16px;
      height: 16px;
      border: 2px solid blue;
      background: blue;
      display: inline-block;
    "
  ></span>
  Checkbox
</div>

<script>
  const checkbox = document.getElementById("checkbox");
  const box = document.getElementById("box");

  const toggle = () => {
    const checked = checkbox.getAttribute("aria-checked") === "true";
    checkbox.setAttribute("aria-checked", String(!checked));
    box.style.background = checked ? "white" : "black";
  };

  checkbox.addEventListener("click", toggle);

  checkbox.addEventListener("keydown", (e) => {
    if (e.key === " " || e.key === "Enter") {
      e.preventDefault();
      toggle();
    }
  });
</script>
```

:::

- **Sifa ya `aria-disabled`**: Hali hii inatumika kuonyesha kuwa kipengele kimezimwa kwa watu wanaotumia teknolojia saidizi, kama kisomaji cha skrini.

:::interactive_editor

```html
<div
  id="editBtn"
  role="button"
  tabindex="-1"
  aria-disabled="true"
  style="opacity: 0.5; cursor: not-allowed;"
>
  Edit
</div>

<button id="toggle">Toggle Disabled</button>

<script>
  const editBtn = document.getElementById("editBtn");
  const toggleBtn = document.getElementById("toggle");

  toggleBtn.addEventListener("click", () => {
    const disabled = editBtn.getAttribute("aria-disabled") === "true";

    editBtn.setAttribute("aria-disabled", String(!disabled));
    editBtn.tabIndex = disabled ? 0 : -1;
    editBtn.style.opacity = disabled ? "1" : "0.5";
    editBtn.style.cursor = disabled ? "pointer" : "not-allowed";
  });
</script>
```

:::

- **Sifa ya `aria-selected`**: Hali hii inatumika kuonyesha kuwa kipengele kimechaguliwa. Unaweza kutumia hali hii kwenye vidhibiti maalum kama kiolesura cha tabo, jedwali la orodha, au gridi.

:::interactive_editor

```html
<div role="tablist">
  <button role="tab" aria-selected="true">Tab 1</button>
  <button role="tab" aria-selected="false">Tab 2</button>
  <button role="tab" aria-selected="false">Tab 3</button>
</div>

<script>
  const tabs = document.querySelectorAll('[role="tab"]');

  tabs.forEach((tab) => {
    tab.addEventListener("click", () => {
      tabs.forEach(t => t.setAttribute("aria-selected", "false"));
      tab.setAttribute("aria-selected", "true");
    });
  });
</script>
```

:::

- **Sifa ya `aria-controls`**: Inatumika kuhusisha kipengele na kipengele kingine kinachodhibiti. Hii husaidia watu wanaotumia teknolojia saidizi kuelewa uhusiano kati ya vipengele hivyo.

:::interactive_editor

```html
<div role="tablist">
  <button 
    role="tab"
    id="tab1"
    aria-controls="section1"
    aria-selected="true"
  >
    Tab 1
  </button>
  <button
    role="tab"
    id="tab2"
    aria-controls="section2"
    aria-selected="false"
  >
    Tab 2
  </button>
  <button
    role="tab"
    id="tab3"
    aria-controls="section3"
    aria-selected="false"
  >
    Tab 3
  </button>
</div>
```

:::

- **Sifa ya `hidden`**: Huficha paneli zisizotumika kutoka kwa watumiaji wa kuona na teknolojia saidizi.

## Kufanya kazi na Maeneo Hai na Maudhui Yanayobadilika

- **Sifa ya `aria-live`**: Hufanya sehemu ya ukurasa wa mtandao kuwa eneo hai, ikimaanisha masasisho yoyote ndani ya eneo hilo yatatangazwa na kisomaji cha skrini ili watumiaji wasikose mabadiliko muhimu.
- **Thamani ya `polite`**: Maeneo mengi hai hutumia thamani hii. Thamani hii ina maana kuwa sasisho si la dharura, hivyo kisomaji cha skrini kinaweza kusubiri hadi kitakapo maliza tangazo lolote linaloendelea au mtumizi atakapomaliza kitendo chake cha sasa kabla ya kutangaza sasisho hilo.

Hapa kuna mfano wa eneo hai linalosasishwa kwa nguvu na JavaScript:

:::interactive_editor

```html
<div aria-live="polite" id="status"></div>

<button id="updateStatus">Update Status</button>

<script>
  const statusEl = document.getElementById("status");
  const btn = document.getElementById("updateStatus");

  btn.addEventListener("click", () => {
    statusEl.textContent = "Your file has been successfully uploaded.";
  });
</script>
```

:::

- **Sifa ya `contenteditable`**: Hubadilisha kipengele kuwa mhariri hai, kuruhusu watumiaji kusasisha maudhui yake kana kwamba ni sehemu ya maandishi. Wakati hakuna lebo au kichwa cha sehemu kinachoonekana kwa eneo la contenteditable, ongeza jina linalopatikana kwa kutumia sifa ya `aria-label` kusaidia watumiaji wa kisomaji cha skrini kuelewa kusudi la eneo linalohaririwa.

:::interactive_editor

```html
<div
  contenteditable="true"
  aria-label="Note editor"
  id="editor"
  style="border: 1px solid #ccc; padding: 8px;"
>
  Editable content goes here
</div>

<p id="status" aria-live="polite"></p>

<script>
  const editor = document.getElementById("editor");
  const status = document.getElementById("status");

  editor.addEventListener("input", () => {
    status.textContent = "Content updated";
  });
</script>
```

:::

## Tukio la `focus` na `blur`

- **Tukio la `blur`**: Hutokea wakati kipengele kinapopoteza makini.

:::interactive_editor

```html
<input
  id="nameInput"
  type="text"
  placeholder="Type here and click outside"
  aria-label="Name input"
/>

<p id="status" aria-live="polite"></p>

<script>
  const input = document.getElementById("nameInput");
  const status = document.getElementById("status");

  input.addEventListener("blur", () => {
    status.textContent = "Input lost focus";
  });
</script>
```

:::

- **Tukio la `focus`**: Hutokea wakati kipengele kinapopata makini.

:::interactive_editor

```html
<input
  id="emailInput"
  type="email"
  placeholder="Click or tab into this field"
  aria-label="Email input"
/>

<p id="status" aria-live="polite"></p>

<script>
  const input = document.getElementById("emailInput");
  const status = document.getElementById("status");

  input.addEventListener("focus", () => {
    status.textContent = "Input received focus";
  });
</script>
```
