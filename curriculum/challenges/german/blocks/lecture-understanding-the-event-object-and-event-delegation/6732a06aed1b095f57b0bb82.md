# --interactive--

Das `change`-Ereignis ist ein spezielles Ereignis, das ausgelöst wird, wenn der Benutzer den Wert bestimmter Eingabeelemente ändert. Genauer gesagt:

- Wenn ein Kontrollkästchen aktiviert oder deaktiviert wird.

- Wenn eine Optionsschaltfläche aktiviert wird.

- Wenn der Benutzer eine Auswahl aus etwas wie einem Datumsauswahlfeld oder Dropdown-Menü trifft.

- Wenn ein Eingabefeld den Fokus verliert (der Benutzer mit Tab zum nächsten Feld wechselt oder außerhalb des Formulars klickt), nachdem der Wert geändert wurde.

- Wenn der Benutzer den Wert anderweitig bestätigt, zum Beispiel durch Drücken der Eingabetaste nach der Texteingabe.

Beachten Sie, dass das `change`-Ereignis NICHT ausgelöst wird, wenn Ihr Benutzer in ein Eingabefeld tippt. Das `change`-Ereignis wird erst ausgelöst, nachdem der Fokus auf ein anderes Element gewechselt wurde.

Hier ist ein Beispiel für die Verwendung des `change`-Ereignisses mit einem Dropdown-Menü:

:::interactive_editor

```html
<select id="select-menu">
  <option value="option1">Option 1</option>
  <option value="option2">Option 2</option>
  <option value="option3">Option 3</option>
</select>
<script src="index.js"></script>
```

```js
const selectMenu = document.getElementById("select-menu");
selectMenu.addEventListener("change", (event) => {
  console.log(`You selected: ${event.target.value}`);
});
```

:::

Jedes Mal, wenn der Benutzer eine andere Option aus dem Dropdown-Menü auswählt, wird das `change`-Ereignis ausgelöst, und der ausgewählte Wert wird in der Konsole protokolliert.

Das `change`-Ereignis erzeugt weiterhin ein `Event`-Objekt, aber im Gegensatz zu den meisten anderen Ereignissen erzeugt es keine benutzerdefinierte Implementierung – die einzigen Eigenschaften und Methoden, auf die Sie zugreifen können, sind die des Basis-`Event`-Objekts.

Dies unterscheidet sich vom `input`-Ereignis, das ein dediziertes `InputEvent`-Objekt erzeugt. Das Change-Ereignis unterscheidet sich auch in einigen anderen Punkten. Ein `input`-Ereignis wird zum Beispiel ausgelöst, wenn ein Benutzer Inhalte in ein Feld eingibt.

Diese Unterschiede sind wichtig zu beachten, da Sie sonst durch das Timing der Ereignisauslösung verwirrt werden könnten.

Worin unterscheidet sich das `Event`-Objekt für das `change`-Ereignis vom `Event`-Objekt für das `input`-Ereignis?

## --answers--

Das `change`-Ereignis hat mehr Eigenschaften als das `input`-Ereignis.
