# --description--

**REST** significa transferência de estado representacional. Pense nisso como um conjunto de diretrizes que descrevem como clientes e servidores web devem se comunicar entre si pela internet.

Nesta lição, você aprenderá o que é `REST`, seus conceitos-chave e como ele funciona.

REST define uma forma para diferentes sistemas interagirem usando tecnologias web padrão. Com REST, clientes como navegadores web, apps móveis e APIs se comunicam com um servidor usando métodos HTTP comuns como `GET`, `POST`, `PUT`, `PATCH` e `DELETE`, que discutimos em uma lição anterior.

Alguns conceitos centrais do REST incluem **sem estado**, **recursos**, **métodos HTTP** e **códigos de resposta ou status**.

**Sem estado** significa que cada requisição que um cliente envia ao servidor é tratada de forma independente. O servidor não lembra requisições anteriores. Por causa disso, toda requisição deve incluir todas as informações necessárias para processá-la. Por exemplo, se um usuário está autenticado, seu token de autenticação deve ser enviado com cada requisição.

No REST, tudo é tratado como um recurso. Um recurso é um pedaço de dado que pode ser acessado ou modificado. Cada recurso é identificado por uma URL, como `/products` para todos os produtos ou `/products/1` para um produto específico.

Abaixo está uma visão geral dos métodos HTTP, códigos de status e um exemplo passo a passo de como a arquitetura REST funciona.

## Métodos HTTP

Os métodos HTTP definem a ação a ser realizada em um recurso:

* `GET`: Recuperar um recurso
    
* `POST`: Criar um novo recurso
    
* `PUT / PATCH`: Atualizar um recurso existente
    
* `DELETE`: Remover um recurso
    

## Códigos de Status HTTP

Após processar uma requisição, o servidor responde com um **status code** que indica o resultado:

* `200` **(OK)**: A requisição foi bem-sucedida
    
* `201` **(Created)**: Um novo recurso foi criado
    
* `400` **(Bad Request)**: O cliente enviou dados inválidos
    
* `404` **(Não Encontrado)**: O recurso solicitado não existe
    
* `500` **(Erro Interno do Servidor)**: Ocorreu um erro no servidor
    

## Como o REST Funciona: Um Exemplo Passo a Passo

### 1\. Cliente Envia uma Requisição

* O cliente envia uma requisição HTTP usando um método como `GET` ou `POST`
    
* A requisição inclui uma URL que identifica o recurso
    
* Também pode incluir `headers`, `metadata` e um corpo de `request`
    

### 2\. Servidor Recebe a Requisição

* O servidor valida o formato da requisição e a autenticação (se necessário)
    
* Ele identifica o recurso alvo a partir da URL
    
* Ele determina a ação solicitada com base no método HTTP
    
* Ele verifica permissões e controle de acesso
    

### 3\. Servidor Executa a Ação

* `GET`: Recuperar o recurso
    
* `POST`: Criar um novo recurso
    
* `PUT / PATCH`: Atualizar o recurso
    
* `DELETE`: Remover o recurso
    
* Qualquer outra lógica necessária é executada
    

### 4\. O `server` constrói a `response`

* O servidor seleciona um código de status apropriado (por exemplo, `200`, `201` ou `404`)
    
* Ele formata os dados da resposta, geralmente como **JSON**
    
* Ele adiciona cabeçalhos relevantes como `Content-Type` ou `Cache-Control`
    

### 5\. Servidor Envia a Resposta

* A resposta inclui um código de status, cabeçalhos e um corpo opcional
    
* Nenhum estado do cliente é armazenado no servidor (sem estado)
    

### 6\. Cliente manipula a resposta

* O cliente verifica o status code para determinar sucesso ou falha
    
* Ele processa quaisquer dados retornados pelo servidor
