# --description--

在上一课中，你学习了如何使用并集类型。使用并集类型的一种常见方法是使用类型别名。 

类型别名是类型的名称。下面是为一个并集类型提供名称的示例：

```ts
type ID = number | string;
```

在你的项目中，你经常会创建 `type` 别名并在整个 `Program` 中像这样使用它们：

```ts
type ID = number | string;

type User = {
  id: ID;
  name: string;
};

function getUserId(user: User): ID {
  return user.id;
}
```

此示例使用了两个名为 `ID` 和 `User` 的类型。`ID` 将是 `number` 或 `string` 的并集类型，而 `User` 是具有 `id` 和 `name` 属性的对象类型。`id` 指定为 `ID` 类型，而 `name` 则具有 `string` 类型。 

`getUserId` 函数有一个名为 `user` 的参数，其类型是 `User`。`getUserId` 函数的返回类型是 `ID`。 

类型别名是描述对象形状的好方法。通常会创建对象类型并像这样在数组中使用它们：

```ts
type User = {
  id: number | string;
  name: string;
};

const users: User[] = [
  { id: 101, name: "Alice" },
  { id: "A102", name: "Bob" },
  { id: 103, name: "Charlie" }
];
```

在此示例中，`users` 数组必须仅填充 `User` 对象。如果你忘记了某个对象中的属性，那么 TypeScript 会抛出如下误差：

```ts
type User = {
  id: number | string;
  name: string;
};

const users: User[] = [
  // Property 'name' is missing in type '{ id: number; }' but required in type 'User'.
  { id: 101 },
  { id: "A102", name: "Bob" },
  { id: 103, name: "Charlie" }
];
```

你可能已经注意到在命名类型别名时使用了 PascalCase。这是因为按照惯例，使用 PascalCase 有助于区分类型和变量。它也有助于提高可读性。当你看到 `User[]` 时，你会知道它是一个 `User` 类型对象的数组。
