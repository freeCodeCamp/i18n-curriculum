# --description--

La crittografia RSA si basa sulla seguente procedura:

Genera due numeri primi distinti `p` e `q`.
Calcola `n=p*q` e `φ=(p-1)(q-1)`.
Trova un intero `e`, `1 < e < φ`, tale che `gcd(e,φ) = 1`

Un messaggio in questo sistema è un numero nell'intervallo `[0,n-1]`.
Un testo da crittografare viene quindi in qualche modo convertito in messaggi (numeri nell'intervallo `[0,n-1]`).
Per crittografare il testo, per ogni messaggio, `m`, si calcola c=m<sup>e</sup> mod n.

Per decrittografare il testo, è necessaria la seguente procedura: calcola `d` tale che `ed=1 mod φ`, poi per ogni messaggio crittografato, `c`, calcola m=c<sup>d</sup> mod n.

Esistono valori di `e` e `m` tali che m<sup>e</sup> mod n = m.
Chiamiamo messaggi `m` per i quali m<sup>e</sup> mod n=m messaggi non nascosti.

Un problema nella scelta di `e` è che non dovrebbero esserci troppi messaggi non nascosti.
Per esempio, sia `p=19` e `q=37`.
Allora `n=19*37=703` e `φ=18*36=648`.
Se scegliamo `e=181`, allora, anche se `gcd(181,648)=1` risulta che tutti i messaggi possibili
m `(0≤m≤n-1)` sono non nascosti quando si calcola m<sup>e</sup> mod n.
Per ogni scelta valida di `e` esistono alcuni messaggi non nascosti.
È importante che il numero di messaggi non nascosti sia al minimo.

Per ogni dato `p` e `q`, trova la somma di tutti i valori di `e`, `1 < e < φ(p,q)` e `gcd(e,φ)=1`, in modo che il numero di messaggi non nascosti per questo valore di `e` sia al minimo.

# --solutions--

```js
function gcd(a, b) {
    if (b)
        return gcd(b, a % b);
    else
        return a;
}

function RSAEncryption(p, q) {
    let phi = (p - 1) * (q - 1);

    let best = Number.MAX_SAFE_INTEGER;
    let sum = 0;

    for (let e = 0; e < phi; ++e) {
        if (!(gcd(e, phi) == 1))
            continue;

        let msg = (gcd(p - 1, e - 1) + 1) * (gcd(q - 1, e - 1) + 1);

        if (best == msg) {
            sum += e;
        } else if (best > msg) {
            best = msg;
            sum = e;
        }
    }

    return sum;
}
```
