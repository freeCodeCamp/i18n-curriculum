# --description--

ゲッターとセッターは、クラスの属性がどのようにアクセスされ、変更されるかを制御するメソッドです。ゲッターでは値を取得し、セッターでは値を設定します。

これらの操作はプロパティと呼ばれるものを通じて行われます。プロパティはゲッターとセッターをつなぎ、データへのアクセスを可能にします。

プロパティは属性のように振る舞いますが、内部ではメソッドのように動作します。メソッドのように定義したデータを属性のように扱うイメージです。つまり、丸括弧ではなくドット記法でプロパティにアクセスできます。

プロパティの主な役割は、値を取得、設定、または削除するときに裏で追加のロジックを実行することです。これにより、オブジェクト内のデータにアクセスしたり操作したりするのに最適な選択肢となります。

では、なぜメソッドではなくプロパティを使うのでしょうか？主に可読性と慣例のためです。コードがよりきれいで読みやすくなります。

メソッドを使う場合は常に丸括弧で呼び出す必要がありますが、プロパティなら通常の属性のようにドット記法でアクセスできます。これにより、裏で追加の処理をしていてもコードがシンプルに見えます。

例えば、値を計算したり、新しい値が有効かどうかを保存前にチェックしたりしたい場合があります。そうした処理をメソッドで呼び出す代わりに、属性のように扱うことができます。

Pythonでは、デコレーターは他の関数やクラスの機能を元のコードを変えずに修正する関数です。

カスタムデコレーターを作ることも可能ですが、このレッスンの範囲外です。プロパティを作るには、メソッドを定義し、その上に`@property`デコレーターを置くだけです。これによりメソッドがプロパティになり、内部的にはデコレートされたメソッドを呼び出しつつ、属性のようにアクセスできます。

ここでゲッターの作り方を`@property`デコレーターを使って紹介します。

```py
class Circle:
    def __init__(self, radius):
        self._radius = radius

    @property
    def radius(self): # A getter to get the radius
        return self._radius
  
    @property
    def area(self):  # A getter to calculate area
        return 3.14 * (self._radius ** 2)

my_circle = Circle(3)

print(my_circle.radius) # 3
print(my_circle.area) # 28.26
```

この例では半径と円の面積を取得しています。

クラス内でradiusの代わりに`_radius`を使っていることに注目してください。アンダースコアはPythonの慣例で、その属性がプライベートであることを示します。つまり、内部用でありクラスの外から直接アクセスすべきでないことを意味します。

例えば半径を設定するセッターを作るには、同じ名前の別のメソッドを定義し、その上に`@<property_name>.setter`を置きます。

`self.radius`内で`__init__`を使うと、オブジェクト生成時にセッターが実行され、不正な半径値をすぐに検出できます。

```py
class Circle:
    def __init__(self, radius):
        self.radius = radius # Calling the setter

    @property
    def radius(self):  # A getter to get the radius
        return self._radius

    @radius.setter
    def radius(self, value):  # A setter to set the radius
        if value <= 0:
            raise ValueError('Radius must be positive')
        self._radius = value

my_circle = Circle(3)
print('Initial radius:', my_circle.radius) # Initial radius: 3

my_circle.radius = 8
print('After modifying the radius:', my_circle.radius) # After modifying the radius: 8
```

この例では、半径のセッターは単に半径を設定するだけでなく、半径が負の数でないかをチェックするバリデーションも行っています。

ゲッターとセッターを定義すると、Pythonは通常の属性構文を使うたびに自動的にそれらを裏で呼び出します。

```py
my_circle.radius # This will call the getter
my_circle.radius = 4 # This will call the setter
```

セッター内で新しい値を代入するときにプロパティと同じ名前を使えないことに注意してください。これは`self.radius = value`がセッター内でセッターを呼び出し、無限再帰と`RecursionError`を引き起こすためです。必ずアンダースコア付きの`self._radius = value`を使う必要があります。

ゲッターで属性のアクセス方法を制御し、セッターで変更方法を制御できるように、削除方法もdeleterで制御できます。

deleterはdel文でプロパティを削除するときにカスタムロジックを実行します。作るには`@<property_name>.deleter`デコレーターを使います。

```py
class Circle:
    def __init__(self, radius):
        self.radius = radius

    # Getter
    @property
    def radius(self):
        return self._radius

    # Setter
    @radius.setter
    def radius(self, value):
        if value <= 0:
            raise ValueError("Radius must be positive")
        self._radius = value

    # Deleter
    @radius.deleter
    def radius(self):
        print("Deleting radius...")
        del self._radius
```

deleterの使い方は次の通りです。

```py
# Create circle object with a radius
my_circle = Circle(33)
print("Initial radius:", my_circle.radius)  # 33

# Delete the radius
# This calls the deleter
del my_circle.radius # Deleting radius...
print("Radius deleted!") # Radius deleted!

# Try to access radius after deletion
try:
    print(my_circle.radius)
except AttributeError as e:
    print("Error:", e) # Error: 'Circle' object has no attribute '_radius'
```

まとめると：

- ゲッターは値を取得したり、その場で計算したりできます。
- セッターは代入前にチェックを行い、安全に値を変更できます。
- プロパティはゲッターとセッターをつなぎ、ドット記法を使いながらロジックを記述できます。
- deleterは属性が削除されたときの動作を定義できます。
