## O que é npm?

- **Definição**: npm é um registro online de software open-source, funcionando como uma "loja de apps" para código.
- **Significado oficial**: npm não significa oficialmente "Node Package Manager"; é uma abreviação recursiva que significa "npm is not an acronym".
- **Três partes relacionadas**:

  - **Site (npmjs.com)**: Para consultar pacotes no registro.
  - **Registro**: Um grande banco de dados público que armazena todos os pacotes.
  - **CLI (Interface de linha de comando)**: Uma ferramenta instalada com Node.js para gerenciar dependências do projeto.

- **Alternativas**: Embora npm seja o mais usado, alternativas populares incluem **Yarn**, **PNPM** e **Bun**.

## O arquivo package.json

- **Propósito**: Um arquivo de configuração que armazena metadados, dependências e scripts para um projeto Node.js.
- **Campos principais**:

  - `name` e `version`: Identificam o projeto.
  - `main`: Especifica o arquivo de entrada, geralmente `index.js`.
  - `license`: Define termos legais (padrão é `ISC`).
  - `dependencies`: Pacotes necessários para executar o app em produção.
  - `devDependencies`: Ferramentas usadas apenas durante o desenvolvimento, como frameworks de teste.

- **Criação**:

  - `npm init`: Inicia um processo interativo de configuração.
  - `npm init -y`: Instala um `package.json` com valores padrão instantaneamente.

## Versionamento semântico (SemVer)

- **Formato**: `MAJOR.MINOR.PATCH`.
- **Regras**:

  - **MAJOR**: Mudanças incompatíveis que podem afetar código existente.
  - **MINOR**: Novos recursos compatíveis com versões anteriores.
  - **PATCH**: Pequenas correções de bugs e melhorias.

- **Símbolos de intervalo**:

  - `^` (Cursor): Permite atualizações de versões minor e patch (ex.: `^1.2.3` permite `1.x.x`).
  - `~` (Til): Permite apenas atualizações patch (ex.: `~1.2.3` permite `1.2.x`).
  - `*` (Asterisco): Corresponde a qualquer versão; usado principalmente para testes.

## Gerenciamento de dependências

- **Instalação**:

  - `npm install <name>`: Adiciona uma dependência de produção.
  - `npm install <name> -D`: Adiciona uma dependência de desenvolvimento.
  - `npm install <name>@<version>`: Instala uma versão específica.

- **Node modules e locks**:

  - `node_modules/`: Contém o código real dos pacotes instalados.
  - `package-lock.json`: Trava as versões exatas de todas as dependências e "dependências filhas" para garantir consistência entre ambientes.

- **Remoção**: `npm uninstall <name>` remove o pacote e atualiza `package.json`.
- **Manutenção**:

  - `npm outdated`: Identifica quais pacotes têm versões mais recentes disponíveis.
  - `npm update`: Atualiza pacotes dentro do intervalo permitido pelo SemVer.

## Scripts npm

- **Função**: Comandos personalizados definidos na seção `scripts` do `package.json` para automatizar tarefas.
- **Execução**: Rodados via `npm run <script_name>`.
- Existem atalhos para `start`, `test`, `stop` e `restart` (ex.: `npm start`).
- **Execução sequencial**:
- `&&`: Executa o segundo comando somente se o primeiro for bem-sucedido.
- `;`: Executa comandos um após o outro independentemente do sucesso.
- **Execução concorrente**: Use o operador `&` para rodar processos em segundo plano.
- **Argumentos**: Passe argumentos para scripts usando o separador `--` (ex.: `npm start -- 8000`).

## Publicação no registro npm

- **Acesso ao registro**: Um banco de dados de pacotes públicos e privados.
- **Passos para publicar**:

1. Crie uma conta gratuita em `npmjs.com`.
2. Faça login pelo terminal usando `npm login`.
3. Garanta um `name` único e um `version` válido no `package.json`.
4. Crie um `README.md` e um `.npmignore` (para excluir arquivos sensíveis).
5. Rode `npm publish` (ou `npm publish --access public` para pacotes com escopo).

- **Escopos**: Usados para evitar conflitos de nome publicando sob `@username/package-name`.

## Sistemas de módulos no Node.js

### CommonJS (CJS)

- **Sistema padrão**: A forma original como Node.js lida com módulos; carrega de forma síncrona.
- **Sintaxe**:
- **Importar**: `const module = require('./path')`.
- **Exportar**: `module.exports = { functionName }`.
- **Identificação**: Usado para arquivos `.cjs` ou arquivos `.js` quando não há `type` especificado no `package.json`.

### ES Modules (ESM)

- **Formato padronizado**: Padrão moderno do JavaScript; carrega de forma assíncrona para maior eficiência.
- **Sintaxe**:

  - **Exportar**: `export { name }` ou `export default function`.
  - **Importar**: `import { name } from './path.mjs'`.
  - **Identificação**: Usado para arquivos `.mjs` ou arquivos `.js` quando `package.json` contém `"type": "module"`.
