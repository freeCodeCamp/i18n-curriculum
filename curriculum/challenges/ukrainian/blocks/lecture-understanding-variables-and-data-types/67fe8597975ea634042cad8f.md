# --description--

У Python змінні — це як блок із міткою для зберігання та посилання на дані різних типів. Щоб оголосити змінні в Python, ви призначаєте значення ідентифікатору за допомогою оператора присвоєння (`=`). Вам не потрібно використовувати спеціальні ключові слова, як `let` або `const` у JavaScript, або `char` у C#.

У Python ви просто пишете ім’я змінної зліва, потім оператор присвоєння, а справа — значення, яке хочете призначити змінній. Ось приклад, як оголосити змінні `name` і `age`:

```python
name = 'John Doe'
age = 25
```

У наведеному прикладі змінна `name` містить значення `'John Doe'`. Це значення — рядок, тобто послідовність символів, яка використовується для представлення тексту. Рядки пишуться в одинарних або подвійних лапках, наприклад `'Hello'` або `"Hello"`. У наступних уроках ви дізнаєтеся більше про роботу з рядками в Python.

При іменуванні змінних у Python слід пам’ятати кілька важливих правил:

- Імена змінних можуть починатися лише з літери або підкреслення (`_`), а не з числа.
- Імена змінних можуть містити лише алфавітно-цифрові символи (`a-z`, `A-Z`, `0-9`) і підкреслення (`_`).
- Імена змінних чутливі до регістру — `age`, `Age` і `AGE` вважаються різними.
- Імена змінних не можуть бути зарезервованими ключовими словами Python, такими як `if`, `class` або `def`.

Якщо ви порушите будь-яке з цих правил, ваша програма на Python видасть `SyntaxError`:

```bash
    5variable_name = 5
     ^
SyntaxError: invalid syntax
```

Тепер розглянемо деякі загальні конвенції іменування змінних у Python.

По-перше, імена змінних мають бути в нижньому регістрі, а окремі слова розділяти підкресленням. Це називається snake case:

```python
my_variable_name = 'freeCodeCamp'
```

Далі слід використовувати описові імена змінних. Наприклад, якщо ви хочете зберегти вік користувача у змінній, `user_age` краще, ніж `age` або скорочення на кшталт `ua`:

```python
user_age = 30
```

Так ви легко зможете пояснити призначення змінної іншим членам команди (або собі в майбутньому) у великому коді.

Ще одна конвенція — уникати використання імен змінних з однією літерою. Це дуже поширено в Python, але слід уникати, бо такі імена не передають жодного призначення чи значення:

```python
x = 56 # What do you mean by x?
```

Це інша справа, якщо ви в циклі або чомусь подібному, бо імена змінних на кшталт `i`, `j`, `k` тощо є загальноприйнятими і допустимими.

Також символ фунта (`#`) і текст, що йде після нього в наведеному прикладі, називаються коментарем. Ви, можливо, вже знайомі з коментарями, тому швидко пройдемося по них і пояснимо, як вони працюють.

У Python коментарі починаються з символу фунта (`#`), і мова ігнорує все, що йде після символу `#` у цій лінії:

```python
# This is a single-line comment
```

Багаторядкові коментарі можна створити, використовуючи послідовні однорядкові коментарі:

```python
# This is a
# multi-line
# comment
```

Ви можете використовувати коментарі, щоб пояснити свій код, залишити собі нагадування або пояснити, чому існує певний рядок. Коментарі особливо корисні, коли ви вчитеся або працюєте в команді.

Однак не слід використовувати коментарі, щоб пояснювати значення імен змінних. Натомість імена, які ви обираєте для змінних, мають бути описовими і передавати їх призначення, а також дотримуватися інших правил іменування, згаданих раніше, щоб уникнути синтаксичних помилок.
