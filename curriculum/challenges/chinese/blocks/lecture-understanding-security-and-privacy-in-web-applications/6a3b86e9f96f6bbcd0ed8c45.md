# --description--

HTTPS 代表超文本传输协议安全版。它是 HTTP 的安全版本。

正如你在之前的几节课中学到的，HTTP 在你的浏览器和网站之间传输数据。标准的 HTTP 以明文形式发送这些数据，因此任何人都可以拦截这些明文数据。

因此，HTTPS 使用一种叫做传输层安全性（TLS）的加密协议来保护数据传输。TLS 对两个点之间的通信通道进行加密。旧系统使用安全套接字层（SSL），许多网络托管平台至今仍在使用。

加密会将数据混淆成只有用户的浏览器和网络服务器持有密钥才能解开的不可读代码。

例如，当你在网站上输入密码时，HTTPS 会立即将其加密，并保护信用卡信息、个人地址、登录凭据和其他敏感信息。拦截数据的黑客只能看到随机字符，因此无法利用被盗信息。

这种保护在公共 Wi-Fi 网络上尤为重要。咖啡馆、机场和其他公共场所的开放 Wi-Fi 网络缺乏基本的安全性，使网络犯罪分子能够监控这些网络上的未加密流量。HTTPS 在公共网络中创建一个私密通道，保护你的浏览历史免受附近恶意行为者的窥视。

HTTPS 还提供认证，证明网站的合法性。要使用 HTTPS，网站所有者必须获得由可信组织颁发的数字证书。该证书验证网站所有者的真实身份。

当你访问一个地址仅以“http”开头的网站时，这表明网站所有者未获得该证书，使网站不安全。现代浏览器也会对此类网站发出警告。

最后，数据完整性是 HTTPS 的另一个关键好处。这意味着在传输过程中无人能篡改你的数据。黑客无法在网站文件传输到你的设备时注入恶意代码。你收到的正是网站所有者发送的原始数据。

像 Google 这样的搜索引擎优先考虑 HTTPS 网站，因为安全网站在搜索结果中排名更高。现代浏览器还要求 HTTPS 才能使用高级功能，如地理定位、推送通知和网络摄像头。
