# --description--

I server sono essenziali per il web moderno. Alimentano le esperienze online su cui facciamo affidamento ogni giorno come utenti e saranno alla base della maggior parte delle esperienze che creerai come sviluppatore.

Scopriamone di più.

Come forse sai, internet è un sistema globale di reti di computer interconnesse. I server sono i componenti centrali che permettono questa comunicazione gestendo il processo di scambio dati.

Un **server** è un computer o un programma per computer che riceve richieste da altri computer e invia dati o servizi come risposta attraverso la rete.

Il concetto di server è duplice perché può riferirsi a:

* L’**hardware** fisico, potente, progettato per funzionare in modo efficiente e continuo per fornire questi servizi. Queste macchine sono costruite e progettate per essere affidabili.
    
* Oppure al **software** vero e proprio che gira su quell’hardware fisico per fornire i servizi.
    
I computer fisici che chiamiamo “server” sono solitamente conservati in data center, strutture grandi e sicure con ambienti controllati.

I server sono una parte essenziale del **modello client-server**.

Il modello client-server è un’architettura di sistema distribuito in cui i componenti sono o client o server.

* Il **client** è il processo o l’applicazione che invia richieste ai server per dati o servizi. Per esempio, un browser web.
    
* Il **server** è il dispositivo fisico o il programma per computer che gestisce quelle richieste e restituisce una risposta.

In sostanza, client e server fanno parte di un ciclo richiesta-risposta, dove il client invia una richiesta al server e il server restituisce una risposta al client.

Per esempio, quando usi il browser (il client) per autenticarti su un’applicazione con il tuo nome utente e password, le tue informazioni vengono inviate in modo sicuro a un server. Il server verifica se il nome utente e la password inseriti corrispondono alle informazioni che la piattaforma ha attualmente nel suo database. Se c’è corrispondenza, il server restituisce una risposta di successo al client (il tuo browser) e vieni autenticato con successo sul sito.

Se non viene trovata corrispondenza, il server restituisce una risposta di errore al client e di solito vedrai un messaggio di errore o un feedback che spiega perché il tentativo di accesso non è riuscito.

Tutta questa logica di autenticazione gira sul server. Il server è responsabile di validare le credenziali, interagire con il database e decidere come rispondere a ogni richiesta.

Ogni risposta del server include un codice di stato HTTP, che indica al client cosa è successo. Questi codici di stato informano il client se una richiesta è riuscita o fallita e perché.

Per esempio:

* 200 significa che tutto è andato bene.
    
* 404 significa che la risorsa non è stata trovata sul server.
    
* 500 significa che c’è stato un errore interno del server.
    
* 503 significa che il server è temporaneamente non disponibile.
    
* E così via…
    

Ogni codice di stato ha un significato unico.

Esempi di applicazioni reali che usano server includono e-commerce, motori di ricerca, giochi, streaming video, piattaforme di archiviazione cloud e collaborazione, social media, applicazioni finanziarie e altro.

Se i dati vengono aggiornati dinamicamente o l’applicazione necessita di qualche tipo di comunicazione con un database, i server sono quasi sicuramente componenti chiave del processo di comunicazione.

Questi sono alcuni dei principali tipi di server:

* **Server web** forniscono contenuti web, come pagine HTML, immagini e file CSS, ai browser.
    
* **Server applicativi** gestiscono la logica di business, elaborano gli input degli utenti e coordinano le risposte.
    
* **Server database** eseguono sistemi di gestione database e archiviano, recuperano e gestiscono i dati.
    
* **Server di posta** inviano, ricevono e archiviano messaggi email.
    
* **Server file** gestiscono l’accesso ai file e permettono che siano archiviati e condivisi in rete.
    
* **Server proxy** agiscono da intermediari tra client e altri server, spesso usati per caching, sicurezza o controllo del traffico.

I server sono la spina dorsale delle potenti piattaforme online che usiamo ogni giorno. Rendono possibile eseguire logiche complesse, archiviare ed elaborare grandi quantità di dati e supportare milioni di utenti contemporaneamente. Senza i server, il mondo digitale moderno su cui facciamo affidamento non esisterebbe.
