# --description--

为了衡量和提升网页性能，我们使用关键性能指标，如 First Contentful Paint（或 FCP）、Speed Index、Total Blocking Time（或 TBT）、弹跳率和独立用户数。让我们逐一了解这些指标。

首次内容绘制，简称 FCP。它衡量第一个内容块（如文本或图像）在屏幕上出现的速度。良好的 FCP 通常低于 1.8 秒，而较差的 FCP 通常超过 3 秒。你可以使用 Chrome DevTools 这样查看你的 FCP：

第 1 步：打开 Chrome DevTools

- 打开 Google Chrome。

- 打开你想测试的网页。

- 按下 "F12" 或右键点击并选择 "Inspect"。

第 2 步：在 DevTools 中查找并点击 `Lighthouse` 标签（页）。

第 3 步：点击 `"Analyze page load"` 按钮并等待分析完成。

现在我们可以分析结果。我们通过向下滚动到 `Metrics` 部分并查找 `First Contentful Paint` 标记来完成此操作。我们可以看到以毫秒或秒为单位的准确时间。

FCP 和 Interaction to Next Paint（或 INP），我们之前已经看过，都是性能指标，但它们衡量的是用户体验的不同方面。重要的是不要将它们混淆。

作为提醒，FCP 是关于用户在装载后多快看到内容，INP 是关于用户点击或类型指派后多快看到反馈。

接下来是 Speed Index。

Speed Index 衡量页面可见部分的绘制速度。分数越低表示性能越好！你可以在 Lighthouse Reports 中查看你的 Speed Index。

我们还有总阻塞时间（Total Blocking Time，简称 TBT）。这显示了主线程被繁重的 JavaScript 任务阻塞了多长时间。如果 TBT 较高，用户会体验到交互延迟。为改善 TBT，请拆分长任务并推迟非必要的脚本。

接下来是弹跳率。这是指未进行任何互动就离开的访问者百分比。如果你的网站弹跳率很高，可能是因为你的页面太慢。

最后，我们有唯一用户。此指标跟踪访问你网站的独立访客数量。要查看弹跳率和唯一用户，你可以使用 Google Analytics。它将允许你监控这些指标并提升参与度。
