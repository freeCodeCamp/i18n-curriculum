# --interactive--

Le asserzioni lookahead e lookbehind ti permettono di far corrispondere pattern specifici in base alla presenza o all'assenza di pattern circostanti. Esistono quattro varianti di queste asserzioni.

La prima è l'asserzione lookahead positiva. Questa asserzione corrisponde a un pattern quando il pattern è seguito da un altro pattern.

Per costruire un lookahead positivo, devi iniziare con il pattern che vuoi far corrispondere. Poi, usa le parentesi tonde per racchiudere il pattern che vuoi usare come condizione. Dopo la parentesi di apertura, usa `?=` per definire quel pattern come un lookahead positivo.

Questo pattern corrisponderà solo alla parola `free` se è seguita da `code`:

```js
const regex = /free(?=code)/i;
```

Proviamo il comportamento del nostro pattern:

:::interactive_editor

```js
const regex = /free(?=code)/i;
console.log(regex.test("freeCodeCamp")); // true
console.log(regex.test("free code camp")); // false
console.log(
  regex.test("I need someone for free to write code for me")
); // false
```

:::

Nota come solo la stringa in cui `free` è immediatamente seguito da `code` supera il test.

Ma cosa succede se vuoi far corrispondere la presenza di `free` quando NON è seguito da `code`? Puoi trasformare il tuo lookahead positivo in un lookahead negativo per invertire il comportamento. Per farlo, cambia il tuo `?=` in `?!`:

```js
const regex = /free(?!code)/i;
```

Proviamo questo con le stesse stringhe:

:::interactive_editor

```js
const regex = /free(?!code)/i;
console.log(regex.test("freeCodeCamp")); // false
console.log(regex.test("free code camp")); // true
console.log(
  regex.test("I need someone for free to write code for me")
); // true
```

:::

Come previsto, i risultati sono invertiti. L'unica stringa che non passa è la prima, dove `free` è immediatamente seguito da `code`.

Le asserzioni lookbehind funzionano in modo simile alle lookahead, tranne che, invece di far corrispondere condizionalmente in base a un pattern che segue, fanno corrispondere condizionalmente in base a un pattern che precede. Diamo un'occhiata a un lookbehind positivo.

Un lookbehind positivo è indicato con `?<=` invece di `?=`. Facciamo corrispondere la nostra espressione regolare a `code` quando è preceduto da `free`:

```js
const regex = /(?<=free)code/i;
```

Proprio come con il nostro lookahead positivo, il nostro lookbehind positivo corrisponde alla prima stringa perché `code` è immediatamente preceduto da `free`:

:::interactive_editor

```js
const regex = /(?<=free)code/i;
console.log(regex.test("freeCodeCamp")); // true
console.log(regex.test("free code camp")); // false
console.log(
  regex.test("I need someone for free to write code for me")
); // false
```

:::

Per far corrispondere `code` quando NON è preceduto da `free`, possiamo usare un lookbehind negativo. Un lookbehind negativo si definisce sostituendo `?<=` con `?<!`:

```js
const regex = /(?<!free)code/i;
```

Questo corrisponderebbe a qualsiasi occorrenza di `code` che NON è immediatamente preceduta da `free`:

:::interactive_editor

```js
const regex = /(?<!free)code/i;
console.log(regex.test("freeCodeCamp")); // false
console.log(regex.test("free code camp")); // true
console.log(
  regex.test("I need someone for free to write code for me")
); // true
```

:::

Ricorda che `Regex.prototype.test` conferma solo se una stringa corrisponde all'espressione regolare. Usiamo il nostro lookbehind negativo con `String.prototype.match` per vedere come le asserzioni influenzano questo:

```js
const regex = /(?<!free)code/i;
console.log("freeCodeCamp".match(regex)); // null
console.log("free code camp".match(regex)); // ['code', index: 5, input: 'free code camp', groups: undefined]
console.log(
  "I need someone for free to write code for me".match(regex)
); // ['code', index: 33, input: 'I need someone for free to write code for me', groups: undefined]
```

Nota come, anche se la nostra espressione regolare usa un lookbehind per verificare la presenza di `free`, non corrisponde a `free`. L'unico testo incluso nella corrispondenza è `code`.

Lookahead e lookbehind sono incredibilmente utili per far corrispondere testo condizionalmente senza influenzare il valore restituito dalla corrispondenza.
