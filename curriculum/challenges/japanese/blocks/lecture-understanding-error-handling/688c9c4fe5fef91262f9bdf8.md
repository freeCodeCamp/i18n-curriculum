# --description--

Pythonでは、`raise`文はコード内で例外を手動で発生させる強力なツールです。エラーが発生するタイミングや方法を制御できるため、カスタムのエラー条件を作成し、特定のプログラム動作を強制できます。

`raise`文は、プログラムの任意の場所で明示的に例外をスローするために使われ、エラー条件が発生したことや特定の要件が満たされていないことを示すことができます。

Pythonの`raise`文は、例外を発生させるためにいくつかの方法で使えます。基本的には、組み込み例外を発生させたり、カスタムのエラーメッセージを作成したりできます。以下は簡単な例です。

```py
def check_age(age):
    if age < 0:
        raise ValueError('Age cannot be negative')
    return age

try:
    check_age(-5)
except ValueError as e:
    print(f'Error: {e}') # Error: Age cannot be negative
```

ここで`raise`が例外を発生させるキーワードであることがわかります。

この例では、無効な年齢が与えられたときにカスタムメッセージ付きで`ValueError`を発生させています。

`raise`文は現在の例外を再スローするためにも使え、例外処理で特に役立ちます。

```py
def process_data(data):
    try:
        result = int(data)
        return result * 2
    except ValueError:
        print('Logging: Invalid data received')
        raise  # Re-raises the same ValueError

try:
    process_data('abc')
except ValueError:
    print('Handled at higher level')
```

ここでは引数なしの`raise`キーワードが、処理中の現在の例外を再スローしています。

これにより、ログ記録やクリーンアップを行いながらも、エラーを呼び出し元に伝播させることができます。

独自の例外クラスを定義してカスタム例外を作成し、発生させることも可能です。

```py
class InsufficientFundsError(Exception):
    def __init__(self, balance, amount):
        self.balance = balance
        self.amount = amount
        super().__init__(f'Insufficient funds: ${balance} available, ${amount} requested')

def withdraw(balance, amount):
    if amount > balance:
        raise InsufficientFundsError(balance, amount)
    return balance - amount

try:
    new_balance = withdraw(100, 150)
except InsufficientFundsError as e:
    print(f'Transaction failed: {e}')
```

ここではカスタム例外クラスが`Exception`またはそのサブクラスから継承していることがわかります。

クラスや継承については今後のレッスンで詳しく学びます。今は、これがカスタムロジックを持つ独自例外を作る方法だと理解してください。

`raise`文は`from`キーワードと組み合わせて使うことで、異なるエラー間の関係を示す例外の連鎖を作れます。

```py
def parse_config(filename):
    try:
        with open(filename, 'r') as file:
            data = file.read()
            return int(data)
    except FileNotFoundError:
        raise ValueError('Configuration file is missing') from None
    except ValueError as e:
        raise ValueError('Invalid configuration format') from e

config = parse_config('config.txt')
```

ここでは`raise ... from None`が元の例外コンテキストを抑制していることがわかります。

```bash
Traceback (most recent call last):
  File "main.py", line 12, in <module>
    config = parse_config('config.txt')
             ^^^^^^^^^^^^^^^^^^^^^^^^^^
  File "main.py", line 7, in parse_config
    raise ValueError('Configuration file is missing') from None
ValueError: Configuration file is missing
```

そして`raise ... from e`は新しい例外を元の例外に連鎖させ、エラーの経緯を保持します。

```bash
Traceback (most recent call last):
  File "main.py", line 5, in parse_config
    return int(data)
           ^^^^^^^^^
ValueError: invalid literal for int() with base 10: ''

The above exception was the direct cause of the following exception:

Traceback (most recent call last):
  File "main.py", line 12, in <module>
    config = parse_config('config.txt')
             ^^^^^^^^^^^^^^^^^^^^^^^^^^
  File "main.py", line 9, in parse_config
    raise ValueError('Invalid configuration format') from e
ValueError: Invalid configuration format
```

また、`assert`文を使って条件付きで例外を発生させることもでき、これは`raise`と`AssertionError`の省略形のようなものです。

```py
def calculate_square_root(number):
    assert number >= 0, 'Cannot calculate square root of negative number'
    return number ** 0.5

try:
    result = calculate_square_root(-4)
except AssertionError as e:
    print(f'Assertion failed: {e}')
```

`raise`文は、ビジネスルールの強制、入力の検証、有意義なエラーメッセージの提供が必要な堅牢なアプリケーションを作る上で不可欠です。戦略的に`raise`を使うことで、コードをより予測可能でデバッグしやすくし、ユーザーに何が問題だったのかを明確に伝えられます。
