# --description--

У попередньому уроці ми розглянули, що таке Fetch API і як його використовувати. У цьому уроці ми поговоримо про `GET`, `POST`, `PUT` та `DELETE` HTTP-методи Fetch API.

Почнемо з найпоширенішого HTTP-методу — це метод `GET`. Він використовується для отримання даних із сервера. Коли ви використовуєте `fetch()` без вказання методу, за замовчуванням використовується `GET`.

```js
fetch('https://api.example.com/data')
```

У цьому коді ми робимо `GET` запит до `https://api.example.com/data`. Зверніть увагу, що ви не можете використовувати ці дані напряму, потрібно конвертувати відповідь у формат JSON. Лише тоді ви зможете використовувати їх у будь-якому місці вашого проєкту. Ось приклад, як це зробити:

```js
fetch('https://api.example.com/data')
  .then(response => response.json())
  .then(data => console.log(data))
```

У цьому коді відповідь, що надходить від Fetch API, є промісом, і ми використовуємо обробник `.then`, щоб конвертувати відповідь у формат JSON. У попередньому уроці ви дізналися, що `Promise` — це об’єкт, який представляє остаточне завершення (або невдачу) асинхронного процесу та його значення.

Значення промісу невідоме в момент створення промісу. Воно стає відомим лише після завершення асинхронного процесу. Коли ми ланцюжимо два обробники `.then` до виклику fetch, це називається ланцюжком промісів, про який ми поговоримо в наступному уроці.

Поки що ми отримували ресурси з сервера. Але чи знали ви, що ми також можемо надсилати дані на сервер? Метод `POST` використовується для надсилання даних на сервер для створення ресурсу. Ось приклад методу `POST`, який використовується для створення даних на сервері:

```js
fetch('https://api.example.com/users', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({
    name: 'John Doe',
    email: 'john@example.com'
  })
})
```

У цьому прикладі ми надсилаємо `POST` запит для створення нового користувача. Ми вказуємо метод `POST`, встановлюємо відповідні заголовки та додаємо `body` з даними, які хочемо надіслати. Тіло має бути рядком, тому ми використовуємо `JSON.stringify()`, щоб конвертувати наш об’єкт у JSON-рядок.

Метод `PUT` використовується для оновлення існуючих ресурсів на сервері. Ось приклад:

```js
fetch('https://api.example.com/users/45', {
  method: 'PUT',
  headers: {
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({
    name: 'John Smith',
    email: 'john@example.com'
  })
})
```

У цьому прикладі уважно подивіться на URL, ви побачите `45` в кінці. Зазвичай це унікальний ідентифікатор для визначення даних, які ми хочемо оновити. У коді ми використали метод `PUT` і також вказали дані як `body`, які будуть використані для оновлення визначених даних.

Метод `DELETE` використовується для видалення ресурсу з сервера. Ось приклад:

```js
fetch('https://api.example.com/users/45', {
  method: 'DELETE',
})
```

У цьому коді ми вказуємо метод `DELETE` і ідентифікатор в кінці URL, щоб визначити дані, які потрібно видалити.
