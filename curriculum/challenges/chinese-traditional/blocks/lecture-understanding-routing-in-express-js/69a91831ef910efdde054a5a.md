# --description--

在 Express 中，路由參數是嵌入在 URL 路徑中的動態值。它們讓你的應用程式能擷取每次請求中會改變的 URL 部分。這對於建置動態頁面非常有用，例如根據 URL 顯示特定使用者、部落格文章或產品。

這裡有一個範例：

```javascript
// Route with a parameter in the path
app.get("/post/:postId", (req, res) => {
  const postId = req.params.postId;
  res.send(`Viewing post with ID: ${postId}`);
});
```

在這裡，路由監聽對 `/post/:postId` 的 GET 請求。

* `:postId` 是路由參數，是 URL 中的佔位符，會被實際的值取代。例如，對 `/post/42` 的請求符合此路由，`42` 就成為 `postId` 的值。
    
* 你可以使用 `req.params` 來存取路由參數。`req.params` 物件包含所有路由參數，在這個例子中，`req.params.postId` 包含來自 URL 的值 `42`。
    

最後，`res.send()` 將回應傳回給客戶端。所以如果 URL 是 `/post/42`，客戶端會看到 `Viewing post with ID: 42`。

如你所見，路由參數讓你的路由變得靈活且動態，無需為每個可能的值建立獨立路由。

現在你了解路由參數的運作方式，讓我們回到上一課的範例，看看如何為不同類型的選單使用動態路由：

```javascript
app.get("/menu/:category", (req, res) => {
  const category = req.params.category;
  res.send(`You are viewing the ${category} menu!`);
});
```

以下是這個運作方式的解析：

1. 使用者造訪 [`http://localhost:3000/menu/drinks`](http://localhost:3000/menu/drinks)。
    
2. 伺服器將此請求與路由 `/menu/:category` 匹配，並將 `category` 參數擷取為 `drinks`。
    
3. 伺服器將此回應傳回瀏覽器：`You are viewing the drinks menu!`
    

同樣地，如果使用者造訪 [`http://localhost:3000/menu/food`](http://localhost:3000/menu/food)，伺服器會回應以下內容：`You are viewing the food menu!`

路由參數讓你能根據 URL 顯示動態內容。你不必為每個類別建立獨立路由，只要透過改變 URL，就能用單一路由處理多個類別，如飲料、食物、甜點等。
