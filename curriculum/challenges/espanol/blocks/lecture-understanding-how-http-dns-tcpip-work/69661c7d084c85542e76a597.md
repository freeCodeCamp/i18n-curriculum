# --description--

Los servidores son esenciales para la web moderna. Alimentan las experiencias en línea de las que dependemos todos los días como usuarios, y alimentarán la mayoría de las experiencias que crearás como desarrollador.

Aprendamos más sobre ellos.

Como sabes, internet es un sistema global de redes de computadoras interconectadas. Los servidores son los componentes centrales que permiten esta comunicación al manejar el proceso de intercambio de datos.

Un **servidor** es una computadora o programa de computadora que recibe solicitudes de otras computadoras y les envía datos o servicios a través de la red como respuesta.

Servidor es un concepto dual porque puede referirse a:

* El hardware físico real y potente que está diseñado para funcionar de manera eficiente y continua para proporcionar estos servicios. Estas máquinas están construidas y diseñadas para la confiabilidad.
    
* O al **software** real de la computadora que se está ejecutando en ese hardware físico para proporcionar los servicios.
    
Los computadores físicos a los que nos referimos como “servidores” generalmente se almacenan en centros de datos, que son instalaciones grandes y seguras con ambientes controlados.

Los servidores son una parte esencial de **El Modelo Cliente-Servidor**.

El Modelo Cliente-Servidor es una arquitectura de sistema distribuido donde los componentes son clientes o servidores.

* El **cliente** es el proceso o aplicación que envía solicitudes a los servidores para obtener datos o servicios. Por ejemplo, un navegador web.
    
* El **servidor** es el dispositivo físico o programa de computadora que maneja esas solicitudes y devuelve una respuesta.

Esencialmente, el cliente y el servidor son parte de un ciclo de solicitud-respuesta, donde el cliente envía una solicitud al servidor, y el servidor devuelve una respuesta al cliente.

Por ejemplo, cuando usas tu navegador (el cliente) para autenticarte en una aplicación con tu nombre de usuario y contraseña, tu información se envía de forma segura a un servidor. Luego, el servidor verifica si el nombre de usuario y la contraseña que ingresaste coinciden con la información que la plataforma tiene actualmente en su base de datos. Si hay una coincidencia, el servidor devuelve una respuesta exitosa al cliente (tu navegador), y te autenticas correctamente en el sitio.

Si no se encuentra ninguna coincidencia, el servidor devuelve una respuesta de fallo al cliente, y normalmente verás un error o algún tipo de retroalimentación que explica por qué tu intento de inicio de sesión no tuvo éxito.

Toda esta lógica de autenticación se ejecuta en el servidor. El servidor es responsable de validar las credenciales, interactuar con la base de datos y decidir cómo responder a cada solicitud.

Cada respuesta del servidor incluye un código de estado HTTP, que le indica al cliente qué ocurrió. Estos códigos de estado permiten al cliente saber si una solicitud tuvo éxito o falló y por qué.

Por ejemplo:

* 200 significa que todo estuvo bien.
    
* 404 significa que el recurso no fue encontrado en el servidor.
    
* 500 significa que hubo un error interno del servidor.
    
* 503 significa que el servidor está temporalmente no disponible.
    
* Y así sucesivamente…
    

Cada código de estado tiene un significado único.

Ejemplos de aplicaciones del mundo real que usan servidores incluyen comercio electrónico, motores de búsqueda, juegos, transmisión de video, almacenamiento en la nube y plataformas de colaboración, plataformas de redes sociales, aplicaciones financieras y más.

Si los datos se actualizan dinámicamente o la aplicación necesita algún tipo de comunicación con una base de datos, los servidores son muy probablemente componentes clave del proceso de comunicación.

Estos son algunos de los principales tipos de servidores:

* **Los servidores web** entregan contenido web, como páginas HTML, imágenes y archivos CSS, a los navegadores.
    
* **Los servidores de aplicaciones** manejan la lógica de negocio, procesan la entrada del usuario y coordinan las respuestas.
    
* **Los servidores de bases de datos** ejecutan sistemas de gestión de bases de datos y almacenan, recuperan y gestionan datos.
    
* **Los servidores de correo** envían, reciben y almacenan mensajes de correo electrónico.
    
* **Los servidores de archivos** gestionan el acceso a archivos y permiten que se almacenen y compartan a través de una red.
    
* **Los servidores proxy** actúan como intermediarios entre clientes y otros servidores, a menudo usados para almacenamiento en caché, seguridad o control de tráfico.

Los servidores son la columna vertebral de las poderosas plataformas en línea que usamos todos los días. Hacen posible ejecutar lógica compleja, almacenar y procesar grandes cantidades de datos, y soportar millones de usuarios al mismo tiempo. Sin servidores, el mundo digital moderno en el que confiamos no existiría.
