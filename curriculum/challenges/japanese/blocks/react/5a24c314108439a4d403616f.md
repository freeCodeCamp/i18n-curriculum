# --description--

最後のチャレンジを除いて、ステートレス関数コンポーネントにプロップスを渡してきました。これらのコンポーネントは純粋な関数のように動作します。プロップスを入力として受け取り、同じプロップスが渡されるたびに同じビューを返します。ステートとは何か疑問に思うかもしれませんが、次のチャレンジで詳しく扱います。その前に、コンポーネントの用語の復習をしましょう。

*ステートレス関数コンポーネント*とは、プロップスを受け取りJSXを返す任意の関数のことです。一方、*ステートレスコンポーネント*は`React.Component`を継承するクラスですが、内部ステートを使いません（これは次のチャレンジで扱います）。最後に、*ステートフルコンポーネント*は自身の内部ステートを持つクラスコンポーネントです。ステートフルコンポーネントは単にコンポーネントやReactコンポーネントと呼ばれることもあります。

一般的なパターンとしては、ステートフルを最小限に抑え、可能な限りステートレス関数コンポーネントを作成することです。これにより、ステート管理をアプリケーションの特定のエリアに限定できます。その結果、ステートの変更が動作にどう影響するかを追いやすくなり、アプリの開発やメンテナンスが改善されます。

# --instructions--

コードエディターには`CampSite`コンポーネントがあり、子として`Camper`コンポーネントをレンダリングしています。`Camper`コンポーネントを定義し、デフォルトプロップスとして`{ name: 'CamperBot' }`を代入してください。`Camper`コンポーネント内では任意のコードをレンダリングして構いませんが、`p`として渡された`name`の値だけを含む`prop`要素を1つ必ず含めてください。最後に、`propTypes`コンポーネントに`Camper`を定義し、`name`がプロップとして提供され、かつ`string`型であることを検証してください。

```js
    __helpers.removeWhiteSpace(code)
  )
);
```

```js
    return (
      mockedComponent.find('p').text() ===
```
