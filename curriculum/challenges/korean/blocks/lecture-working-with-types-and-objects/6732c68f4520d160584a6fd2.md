# --interactive--

`toString()` 메서드는 JavaScript에서 값을 문자열로 변환하는 기본 기능입니다. 숫자, 불리언, 배열, 객체에 사용할 수 있는 메서드입니다. `toString()`의 가장 일반적인 용도 중 하나는 숫자를 문자열로 변환하는 것입니다. 예를 들어 다음과 같습니다:

:::interactive_editor

```js
const num = 10;
console.log(num.toString()); // "10"
```

:::

이 메서드는 `2`부터 `36`까지의 숫자인 선택적 기수를 인수로 받습니다. 이 기수는 2진수의 경우 2, 8진수의 경우 8과 같은 진법을 나타냅니다. 기수를 지정하지 않으면 기본적으로 10진수인 10이 사용됩니다. `2` 메서드에 `toString()`를 인수로 전달하는 예는 다음과 같습니다:

:::interactive_editor

```js
const num = 10;
console.log(num.toString(2)); // "1010"
```

:::

결과는 `1010`이며, 이는 10진수 `10`의 2진수 표현입니다.

배열과 객체를 문자열로 변환할 때도 `toString()` 메서드를 사용할 수 있습니다. 배열은 각 요소를 문자열로 변환하고 쉼표로 연결하는 `toString()`의 사용자 정의 구현을 가지고 있습니다. 예를 들어 다음과 같습니다:

:::interactive_editor

```js
const arr = [1, 2, 3];
console.log(arr.toString()); // "1,2,3"
```

:::

이 예제에서는 배열의 모든 요소가 연결되어 문자열 `1,2,3`을 만듭니다.

`toString()` 메서드를 객체에 사용하면 결과는 객체 프로퍼티의 문자열화된 버전이 아닙니다.

:::interactive_editor

```js
const person = {
  name: "John",
  age: 30,
  isStudent: true
};

console.log(person.toString()); // "[object Object]"
```

:::

이 예제에서 결과는 객체의 기본 문자열 표현인 `[object Object]`입니다. `person` 객체 프로퍼티의 문자열화된 버전을 얻으려면 앞으로 수업에서 배우게 될 `JSON.stringify()`를 사용해야 합니다.

결론적으로 `toString()` 메서드는 값을 문자열로 변환하는 데 사용됩니다. 다양한 데이터 타입에서 어떻게 작동하는지, 그리고 자신만의 객체에 맞게 어떻게 커스터마이징하는지 이해하면 JavaScript 애플리케이션에서 데이터를 조작하고 표시하는 능력이 크게 향상됩니다.
