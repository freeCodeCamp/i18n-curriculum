# --interactive--

在上一課中，你學會了如何使用 JavaScript 中的 `Date` 物件。但在 JavaScript 中有幾種不同的方式來格式化日期。

在本課程中，我們將探討如何使用 `toISOString()`、`toString()` 和 `toLocaleDateString()` 方法來格式化 JavaScript 中的日期。

在我們檢視不同的方法之前，先來回顧一下 `Date` 物件的樣子：

:::interactive_editor

```js
const date = new Date();
console.log(date);
```

:::

當你將 `date` 物件記錄到主控台時，你會看到根據使用者系統設定的當前日期和時間。以下是你可能會看到的輸出範例：

```js
Sun Sep 29 2024 19:45:37 GMT-0700 (Pacific Daylight Time)
```

如果你對 `date` 物件使用 `toString()` 方法，將會看到與上述相同的輸出。

要以擴展的 ISO 格式（ISO 8601）格式化日期，你可以像這樣使用 `toISOString()` 方法：

:::interactive_editor

```js
const date = new Date();
console.log(date.toISOString());
```

:::

ISO 8601 是一個用於表述日期和時間的國際標準。格式為 `YYYY-MM-DDTHH:mm:ss.sssZ`。

以下是你可能會看到的輸出範例：

```js
2024-09-30T02:47:20.292Z
```

另一種格式化日期的方法是使用 `toLocaleDateString()` 方法。此方法允許你根據使用者的區域格式化日期。以下是使用 `toLocaleDateString()` 方法的基本語法：

:::interactive_editor

```js
const date = new Date();
console.log(date.toLocaleDateString());
```

:::

以下是你可能會看到的輸出範例：

```js
9/29/2024
```

`toLocaleDateString()` 方法接受兩個選用參數：`locales` 和 `options`。

`locales` 參數是一個表述要使用的語系的字串。例如，你可以傳入 `en-US` 代表英文（美國）或 `fr-FR` 代表法文（法國）。如果你沒有傳入 `locales` 參數，將會使用預設語系。以下是如何使用 `locales` 參數的範例：

:::interactive_editor

```js
const date = new Date();
console.log(date.toLocaleDateString("fr-FR"));
```

:::

第二個選用參數是 `options` 參數。此參數是一個物件，允許你指定日期字串的格式。以下是如何使用 `options` 參數的範例：

:::interactive_editor

```js
const date = new Date();
const options = {
  weekday: "long",
  year: "numeric",
  month: "long",
  day: "numeric",
};
console.log(date.toLocaleDateString("en-GB", options));
```

:::

在上述範例中，我們指定了選項，可選方案為英語（英國）並包含完整的星期幾、年份、月份和日期。 

以下是你可能會看到的輸出範例：

```js
Sunday, September 29, 2024
```

在本課程中，我們只涵蓋了在 JavaScript 中格式化日期的幾種方法。還有許多其他方法和函式庫可用來幫助你格式化 JavaScript 中的日期。但 `toISOString()` 和 `toLocaleDateString()` 方法是格式化 JavaScript 日期的良好起點。
