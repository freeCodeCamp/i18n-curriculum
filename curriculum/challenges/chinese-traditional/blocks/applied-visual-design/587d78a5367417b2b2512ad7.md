# --description--

`repeating-linear-gradient()` 函式（程式）與 `linear-gradient()` 非常相似，主要差別在於它會重複指定的漸層樣式。`repeating-linear-gradient()` 接受各種值，但為了簡化，你將在此挑戰中使用角度值和顏色停止值。

角度值是漸層的方向。顏色停止點就像標記過渡位置的寬度值，並以百分比或像素數表示。

在程式碼編輯器中示範的範例中，漸層從 0 像素處的顏色 `yellow` 開始，並與距離起點 40 像素處的第二個顏色 `blue` 混合。由於下一個顏色停點也在 40 像素處，漸層會立即變為第三個顏色 `green`，該顏色本身會與距離漸層起點 80 像素處的第四個顏色值 `red` 混合。

在這個範例中，有助於將色彩停點視為配對，每兩個顏色會彼此混合。

```css
0px [yellow -- blend -- blue] 40px [green -- blend -- red] 80px
```

如果每兩個顏色停止值是相同的顏色，混合效果不明顯，因為它是在相同顏色之間，接著是到下一個顏色的硬切換，所以你最終會得到條紋。

# --instructions--

透過將 `repeating-linear-gradient()` 的漸層角度改為 `45deg` 來製作條紋，然後將前兩個顏色停點設為 `yellow`，最後將後兩個顏色停點設為 `black`。

# --hints--

`repeating-linear-gradient()` 的角度應該是 45deg。

```js
assert.match(code,/background:\s*?repeating-linear-gradient\(\s*?45deg/gi);
```

`repeating-linear-gradient()` 的角度不應再是 90deg

```js
assert.notMatch(code, /90deg/gi);
```

0 像素處的顏色停止點應該是 `yellow`。

```js
assert.match(code, /yellow\s+?0(px)?/gi);
```

第一個顏色停點在 40 像素處應該是 `yellow`。

```js
assert.match(code, /yellow\s+?40px/gi);
```

第二個顏色停止點在 40 像素處應該是 `black`。

```js
assert.match(code, /yellow\s+?40px,\s*?black\s+?40px/gi);
```

最後一個顏色停靠點在 80 像素處應該是 `black`。

```js
assert.match(code, /black\s+?80px/gi);
```
