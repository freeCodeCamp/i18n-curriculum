# --description--

في Python، تُعد معالجة الاستثناءات جزءًا أساسيًا من كتابة برامج قوية وقادرة على تحمل الأخطاء. تتيح لك توقع الأخطاء، والتقاطها، والاستجابة لها بطريقة منظمة.

معالجة الاستثناءات هي عملية التقاط الأخطاء التي تحدث أثناء تنفيذ البرنامج وإدارتها، حتى لا تتعطل الشفرة بشكل غير متوقع.

يوفر Python كتل `try` و `except` و `else` و `finally` للتعامل مع الأخطاء بسلاسة. إليك مثالًا أساسيًا:

```py
try:
    x = 10 / 0
except ZeroDivisionError:
    print("You can't divide by zero!")
```

- `try`: كتلة الشفرة التي تتوقع فيها حدوث خطأ محتمل.

- `except`: تُنفذ هذه الكتلة إذا حدث خطأ من النوع المحدد داخل كتلة `try`.

- في هذه الحالة، القسمة على صفر ترفع خطأ `ZeroDivisionError`، الذي يتم التقاطه ومعالجته بعد ذلك.

وهنا مثال يوضح أيضًا كيفية استخدام كتلتي `else` و `finally`:

```py
try:
    x = 10 / 2
except ZeroDivisionError:
    print("You can't divide by zero!")
else:
    print('Division successful:', x)
finally:
    print('This block always runs.')
```

- `else`: تُنفذ إذا لم يحدث أي استثناء في كتلة try.

- `finally`: تُنفذ بغض النظر عما إذا حدث استثناء أم لا. مفيدة لمهام التنظيف مثل إغلاق الملفات أو تحرير الموارد.

يمكنك أيضًا التقاط عدة استثناءات باستخدام كتل `except` منفصلة:

```py
try:
    number = int('abc')
    result = 10 / number
except ValueError:
    print('That was not a valid number.')
except ZeroDivisionError:
    print("Can't divide by zero.")
```

باستخدام بنود `except` منفصلة، يمكنك جعل استجابات الأخطاء أكثر تحديدًا وفائدة.

يمكنك أيضًا استخدام كائن الاستثناء، الذي يُعطى عادة اسمًا مستعارًا باستخدام الكلمة المفتاحية `as`. هنا نستخدم `e` كاسم مستعار لكائن الخطأ:

```py
try:
    x = 1 / 0
except ZeroDivisionError as e:
    print(f'Error occurred: {e}')
```

استخدام `e` يتيح لك الوصول إلى رسالة الخطأ الفعلية أو الكائن لتسجيله أو تصحيح الأخطاء.

يمكنك أيضًا التقاط عدة استثناءات في بند `except` واحد عن طريق تحديد الاستثناءات كصف مرتب:

```py
try:
    number = int(input('Enter a number: '))
    result = 10 / number
except (ValueError, ZeroDivisionError) as e:
    print(f'Error occurred: {e}')
```

تسمح معالجة الاستثناءات لبرامجك بالتعافي من الأخطاء بسلاسة. باستخدام `try` و `except` و `else` و `finally`، يمكنك توقع المشكلات المحتملة وبناء تطبيقات أكثر متانة.
