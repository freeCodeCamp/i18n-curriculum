# --interactive--

HTMLの`form`要素は、名前やメールアドレスなどのユーザー情報を収集するために使われます。以下は`form`要素の例です。

```html
<form action="url-goes-here">
  <!-- input elements go here -->
</form>
```

`action`属性は、提出時にフォームデータが送信される場所を指定します。名前やメールアドレスなど特定の情報を収集するには、`input`要素を使います。以下は`input`要素の使用例です。

インタラクティブエディターを有効にして、プレビューウィンドウの`input`要素の入力フィールドに名前を入力してみてください。

:::interactive_editor

```html
<form action="">
  <input type="text" />
</form>
```

:::

`input`要素は空要素で終了タグがありません。`type`属性はユーザーから期待されるデータの種類を定義します。この場合、データはプレーンテキストです。入力にラベルを追加するには、`label`要素を使います。以下は`label`要素を使い、テキストに`Full Name:`を表示した例です。

プレビューウィンドウのテキスト`Full Name:`をクリックすると、入力フィールドにフォーカスが移動します。例とやり取りするにはインタラクティブエディターを有効にしてください。

:::interactive_editor

```html
<form action="">
  <label>
    Full Name:
    <input type="text" />
  </label>
</form>
```

:::

`input`を`label`要素内にネストすると、`label`と`input`入力フィールドの間に暗黙の関連付けが作られます。「暗黙的」とは、追加の属性や要素で明示的に指定しなくても理解または推測されることを指します。`label`と`input`を明示的に関連付けるには、`for`属性を使います。以下はメールアドレスのラベルに`for`属性を使った例です。

プレビューウィンドウの`input`要素に、`jane@example.com`のような架空のメールアドレスを入力してみてください。例とやり取りするにはインタラクティブエディターを有効にしてください。

:::interactive_editor

```html
<form action="">
  <label for="email"> Email Address: </label>
  <input type="email" id="email" />
</form>
```

:::

明示的な関連付けを使う場合、`for`属性と`id`の値は同じである必要があります。この例では両方とも`email`に設定されています。入力の`email`タイプは、正しくフォーマットされたメールアドレスの基本的なバリデーションを提供します。ユーザーに期待される入力について追加のヒントを表示したい場合は、`placeholder`属性を使います。以下はメール入力内で`placeholder`属性を使った例です。

インタラクティブエディターを有効にし、メール入力をクリックしてメールを入力し始めると、プレースホルダーテキストが消えます。

:::interactive_editor

```html
<form action="">
  <label for="email"> Email Address: </label>
  <input type="email" id="email" placeholder="example@email.com" />
</form>
```

:::

プレースホルダーテキストには、ユーザーに期待するデータの形式や種類を示す役立つ短いテキストを提供してください。この例では、プレースホルダーテキスト`example@email.com`が、正しくフォーマットされたメールアドレスを入力する必要があることをユーザーに示しています。
