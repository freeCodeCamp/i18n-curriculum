# --description--

CRUD è un acronimo che sta per Create, Read, Update e Delete. Queste sono le quattro operazioni dello storage persistente.

Lo storage persistente si riferisce al salvataggio dei dati in modo che siano disponibili anche dopo che l'alimentazione è stata spenta o il dispositivo riavviato.

Capire come funzionano le operazioni di base di CRUD è fondamentale per lo sviluppo web perché costituiscono la base per lavorare con i database e costruire applicazioni in cui gli utenti possono aggiungere, visualizzare, modificare ed eliminare dati.

Ora, diamo un’occhiata più da vicino a ogni parte di CRUD

- Create si riferisce al processo di creare nuovi dati. Per esempio, in un’app web, questo può essere quando un utente aggiunge un nuovo post a un blog.

- Read è l’operazione in cui i dati vengono recuperati da un database. Per esempio, quando visiti un post del blog o visualizzi il tuo profilo su un sito web, stai eseguendo un’operazione di lettura per recuperare e mostrare i dati archiviati nel database.

- Update consiste nel modificare dati esistenti nel database. Un esempio sarebbe modificare un post del blog o aggiornare le informazioni del tuo profilo.

- Delete è l’operazione che rimuove dati da un database. Per esempio, quando elimini un post del blog o un account, stai eseguendo un’operazione di eliminazione.

CRUD si usa quando si lavora con database, l’interfaccia utente e le API RESTful. Le API RESTful sono un insieme di convenzioni per costruire servizi web che permettono al client di interagire con un database o un sistema back-end eseguendo operazioni CRUD tramite metodi HTTP standard.

HTTP sta per Hypertext Transfer Protocol ed è la base per la comunicazione dei dati sul web. Ci sono metodi HTTP che definiscono le azioni che si possono eseguire sulle risorse via web. I metodi comuni sono `GET`, `POST`, `PUT`, `PATCH`, `DELETE`.

Imparerai di più sulle API RESTful e HTTP nelle lezioni future, ma ecco una breve spiegazione di come CRUD si mappa ai diversi metodi HTTP.

- `POST` si usa per creare una nuova risorsa.

- `GET` si usa per recuperare o leggere dati.

- `PUT` si usa per aggiornare una risorsa sostituendola completamente.

- `PATCH` si usa per aggiornare parzialmente una risorsa.

- `DELETE` si usa per rimuovere una risorsa.

Ecco un esempio di come le operazioni CRUD potrebbero essere rappresentate nel codice usando un semplice array in JavaScript:

```js
let items = [];

// Create
function createItem(item) {
  items.push(item);
}

// Read
function readItems() {
  return items;
}

// Update
function updateItem(index, newItem) {
  items[index] = newItem;
}

// Delete
function deleteItem(index) {
  items.splice(index, 1);
}

// Example Usage
createItem('Book');
console.log(readItems()); // ['Book']
updateItem(0, 'Magazine');
console.log(readItems()); // ['Magazine']
deleteItem(0);
console.log(readItems()); // []
```

In questo esempio:

- Creiamo un elemento aggiungendolo a un array.

- Leggiamo gli elementi restituendo l’array.

- Aggiorniamo un elemento modificando l’elemento dell’array a un indice specifico.

- Eliminiamo un elemento rimuovendolo dall’array usando `splice()`.

Questa è una rappresentazione di base di come funzionano le operazioni CRUD a livello concettuale.
