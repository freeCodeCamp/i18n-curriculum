# --interactive--

Якщо ви пишете вираз із кількома операторами в JavaScript, як JavaScript вирішує, який з них обчислювати першим? Тут і вступає в дію пріоритет операторів. Давайте детально розглянемо пріоритет операторів на прикладах коду, а також що відбувається, коли оператори мають однаковий пріоритет.

Пріоритет операторів визначає порядок, у якому обчислюються операції у виразі. Оператори з вищим пріоритетом обчислюються раніше за оператори з нижчим пріоритетом. Уявіть пріоритет операторів як у математиці, де ділення і множення відбуваються раніше за додавання і віднімання.

Без дотримання цього правила базові рівняння давали б неправильну відповідь. JavaScript працює так само. У нього є власні правила, які оператори йдуть першими, другими і так далі. Наприклад, подивіться на вираз нижче:

:::interactive_editor

```js
const result = 2 + 3 * 4;

console.log(result); // 14
```

:::

Якби JavaScript обчислював цей вираз зліва направо, ви очікували б `2 + 3 = 5`, а потім `5 * 4 = 20`. Але оскільки множення має вищий пріоритет, ніж додавання, JavaScript спочатку обчислює частину `3 * 4`, що дає результат `2 + 12 = 14`.

Іноді ви хочете, щоб певні частини вашого виразу обчислювалися першими, незалежно від правил пріоритету. Ви можете використати круглі дужки, `()`, щоб це зробити. Все, що всередині круглих дужок, обчислюється першим, незалежно від усього. Давайте змусимо частину `2 + 3` з попереднього прикладу обчислитися першою:

:::interactive_editor

```js
const result = (2 + 3) * 4;

console.log(result); // 20
```

:::

У наведеному прикладі круглі дужки змушують JavaScript спочатку обчислити `2 + 3`, а потім помножити результат на `4`. Це дає вам `20` замість `14`.

Оператор ділення також має вищий пріоритет, ніж додавання чи віднімання:

:::interactive_editor

```js
const result = 2 + 6 / 3;

console.log(result); // 4
```

:::

Якби JavaScript обчислював цей вираз зліва направо, ви могли б очікувати `2 + 6 = 8`, а потім `8 / 3 = 2.67`. Але оскільки ділення має вищий пріоритет, ніж додавання, JavaScript спочатку обчислює ділення: `6 / 3 = 2`, а потім додає `2 + 2`, отримуючи результат `4`.

Отже, і в множенні, і в діленні ці операції завжди відбуваються раніше за додавання і віднімання, якщо ви не використаєте круглі дужки, щоб змінити порядок. А що відбувається, якщо оператори мають однаковий пріоритет? JavaScript використовує асоціативність, щоб визначити порядок їх обчислення.

Асоціативність — це те, що каже JavaScript, чи обчислювати оператори зліва направо, чи справа наліво. Для більшості операторів, як додавання і множення, асоціативність — зліва направо. Тож JavaScript обробляє їх з лівого краю виразу до правого:

:::interactive_editor

```js
const result = 10 - 2 + 3;

console.log(result); // 11
```

:::

Спочатку `10 - 2 = 8`, потім `8 + 3 = 11`. У цьому випадку JavaScript рухається зліва направо. Деякі оператори, як присвоєння (`=`), мають асоціативність справа наліво. Це означає, що права частина виразу обчислюється першою:

:::interactive_editor

```js
let a, b;
a = b = 5;

console.log(a); // 5
console.log(b); // 5
console.log(a + b); // 10
```

:::

У наведеному прикладі JavaScript спочатку призначає `5` `b`, а потім призначає `b` (яке тепер `5`) `a`.

Оператор піднесення до степеня також має асоціативність справа наліво:

:::interactive_editor

```js
const result = 2 ** 3 ** 2;

console.log(result); // 512
```

:::

Спочатку JavaScript обчислює `3 ** 2`, що дорівнює `9`, потім обчислює `2 ** 9`, що дорівнює `512`. Якби оператор піднесення до степеня мав асоціативність зліва направо, JavaScript спочатку обчислив би `2 ** 3`, щоб отримати `8`, а потім `8 ** 2`, щоб отримати `64`.
