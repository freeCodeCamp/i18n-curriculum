# --description--

In React ist Rendering der Prozess, bei dem Komponenten in der Benutzeroberfläche (UI) erscheinen, normalerweise im Browser.

React nimmt all Ihren JavaScript-, JSX- und CSS-Code, ermittelt, wie er aussehen soll, und zeigt ihn dann in der Benutzeroberfläche an.

Der Rendering-Prozess besteht aus drei Phasen: Auslösen, Rendern und Commit. Schauen wir uns diese genauer an.

Die Auslösephase tritt ein, wenn React erkennt, dass sich etwas geändert hat und die Benutzeroberfläche möglicherweise aktualisiert werden muss. Diese Änderung ist oft auf ein Update im State oder in den Props zurückzuführen.

Zum Beispiel kann die Erkenntnis, dass es Zeit fürs Abendessen ist, Sie dazu veranlassen, in die Küche zu gehen und mit dem Kochen zu beginnen.

Im untenstehenden `Counter`-Beispiel löst das Klicken auf die Erhöhen- oder Verringern-Schaltfläche aus, dass React den neuen `count`-Wert anzeigt:

```jsx
import { useState } from "react";

function Counter() {
  const [count, setCount] = useState(0);

  return (
    <div>
      <h1>{count}</h1>
      <button onClick={() => setCount(count - 1)}>Decrement</button>
      <button onClick={() => setCount(count + 1)}>Increment</button>
    </div>
  );
}

export default Counter;
```

Sobald das Auslösen erfolgt ist, geht React in die Render-Phase über. Hier wertet React Ihre Komponenten erneut aus und ermittelt, was angezeigt werden soll.

Dazu verwendet React eine leichte Kopie des „echten“ DOM. Diese wird als virtuelles DOM bezeichnet. Mit dem virtuellen DOM kann React schnell prüfen, was sich in der Komponente ändern muss.

Man kann sich diese Phase so vorstellen, als wären Sie in der Küche, hätten Ihre Zutaten gesammelt und kochen nun Ihr Abendessen.

Für die `Counter`-Komponente ist die Render-Phase der Punkt, an dem React die Funktionen mit dem neuen `count`-Wert erneut ausführt. React berechnet neu, wie der `<h1>{count}</h1>`-Teil der Komponente basierend auf dem aktualisierten `count`-Wert aussehen soll, aber Sie sehen auf dem Bildschirm erst im nächsten Schritt – Commit – Änderungen.

Die Commit-Phase ist der Punkt, an dem React die vorbereiteten Änderungen aus dem virtuellen DOM nimmt und auf das echte DOM anwendet. Anders gesagt, hier sehen Sie das endgültige Ergebnis auf dem Bildschirm.

Um dies zu erreichen, vergleicht React das virtuelle DOM mit dem tatsächlichen DOM, identifiziert nur die Teile, die aktualisiert werden müssen, und wendet diese Änderungen auf das echte DOM an, um die Benutzeroberfläche zu aktualisieren.

Man kann sich diese Phase so vorstellen, als würden Sie das gekochte Essen servieren und sichtbar machen, genau wie React es macht, wenn es Updates im echten DOM committet.

Für die `Counter`-Komponente ist die Commit-Phase der Punkt, an dem der neue `count`-Wert auf das `h1`-Element angewendet wird und Sie die Änderung auf der Seite sehen.

Diese drei Prozesse sind extrem schnell, weil React direkte DOM-Manipulation minimiert, indem es Änderungen zuerst im virtuellen DOM berechnet und dann nur die Teile im echten DOM aktualisiert, die sich ändern müssen.
