# --description--

JavaScript 엔진은 코드를 읽고 이해하며 실행할 수 있는 능력을 가지고 있습니다. 코드를 컴퓨터가 이해하고 그에 따라 작동할 수 있는 명령어로 변환하는 변환기처럼 작동합니다.

가장 잘 알려진 JavaScript 엔진 중 하나는 구글이 개발한 V8로, Chrome과 Node.js에서 사용됩니다. JavaScript 엔진은 몇 가지 단계로 작동합니다. 먼저 코드를 파싱하여 한 줄씩 읽으면서 JavaScript 코드에 오류가 없는지 확인합니다. 그다음 이 코드를 바이트코드로 변환하는데, 바이트코드는 컴퓨터가 이해하고 실행하기 더 쉬운 간단한 중간 형태의 코드입니다. 마지막으로 이 바이트코드를 실행하여 프로그램의 명령을 수행합니다. 다음은 JavaScript 코드 예시입니다:

```js
const greeting = "Hello, World!";
console.log(greeting);
```

이 코드를 실행하면 JavaScript 엔진은 먼저 구문 오류가 있는지 확인하기 위해 코드를 파싱합니다. 파싱이란 엔진이 코드를 읽고 이해할 수 있는 구조로 분해하며, 그 과정에서 오류를 검사하는 것을 의미합니다.

그다음 코드를 중간 형식(엔진에 따라 바이트코드나 머신 코드)으로 컴파일합니다. 컴파일은 사람이 읽을 수 있는 코드를 컴퓨터가 더 빠르게 실행할 수 있는 효율적인 형식으로 변환하는 과정입니다.

마지막으로 엔진이 코드를 실행하여 `Hello, World!`를 콘솔에 출력합니다.

이제 JavaScript 런타임에 대해 이야기해 보겠습니다. JavaScript 런타임은 JavaScript 코드가 실행되는 환경입니다. 여기에는 코드를 처리하고 실행하는 JavaScript 엔진(예: Chrome의 V8 또는 Firefox의 SpiderMonkey)뿐 아니라, 환경(웹 브라우저나 Node.js 등)에서 제공하는 추가 기능도 포함됩니다. 이 추가 기능에 대해서는 앞으로 수업에서 더 배우게 됩니다.

JavaScript 언어 자체는 변수, 반복문, 함수 같은 기본적인 부분을 다루지만, 런타임은 DOM(웹 페이지용)이나 Fetch API(네트워크 요청용)처럼 언어 외부와 상호작용할 수 있는 도구를 제공합니다.

요약하자면, 런타임은 JavaScript가 기본 프로그래밍 작업을 넘어서 웹 페이지와 상호작용하거나 시간 기반 작업을 처리하는 등 더 많은 일을 할 수 있도록 언어 자체 외의 기능을 제공하는 환경입니다.

JavaScript를 작성하는 데 엔진과 런타임의 모든 세부 사항을 알 필요는 없지만, 기본적인 이해는 더 효율적인 코드를 작성하고 문제를 디버그하는 데 도움이 됩니다.
