# --description--

쿠키는 웹 쿠키 또는 브라우저 쿠키라고도 하며, 서버가 사용자의 웹 브라우저로 보내는 작은 데이터 조각입니다. 이 쿠키들은 사용자의 기기에 저장되고 이후 요청 시 서버로 다시 전송됩니다.

쿠키는 HTTP가 무상태 프로토콜이기 때문에 웹 애플리케이션이 상태를 유지하고 사용자 정보를 기억하는 데 필수적입니다.

쿠키는 사용자 환경 설정, 세션 데이터 또는 추적 정보 등 다양한 정보를 저장할 수 있습니다.

쿠키는 항상 이름-값 쌍으로 저장됩니다. 즉, 각 쿠키는 이름(키)과 연결된 값이 있습니다.

예를 들어, 쿠키는 사용자의 세션 ID를 다음과 같이 저장할 수 있습니다: `sessionId=abc123`. 키는 `sessionId`이고 값은 `abc123`입니다. 브라우저가 서버와 통신할 때마다 이 쿠키들은 이름-값 쌍 형태로 전송됩니다.

사용자가 웹사이트를 방문하면 서버는 HTTP 응답에 Set-Cookie 헤더를 포함하여 하나 이상의 쿠키를 사용자의 브라우저로 보낼 수 있습니다. 헤더는 HTTP 요청 또는 응답에 대한 추가 정보를 제공하는 키-값 쌍입니다. HTTP 요청과 응답에 대해서는 이후 수업에서 더 배우게 됩니다.

쿠키가 설정되면 브라우저는 이를 저장하고 이후 동일 도메인에 대한 모든 요청에 Cookie 헤더에 쿠키를 자동으로 포함합니다. 이를 통해 서버는 저장된 쿠키에 접근하여 사용자 세션 유지나 환경 설정 추적 등에 활용할 수 있습니다.

다음은 HTTP 응답에서 쿠키가 설정되는 예시입니다:

```http
Set-Cookie: sessionId=abc123; Expires=Wed, 21 Oct 2021 07:28:00 GMT; Secure; HttpOnly
```

브라우저는 쿠키를 저장하고 이후 동일 서버에 요청할 때 `Cookie` 헤더에 쿠키를 포함합니다:

```http
Cookie: sessionId=abc123
```

서버는 쿠키를 읽고 저장된 세션 ID를 사용해 사용자가 로그인했는지 등의 정보를 조회할 수 있습니다.

다음은 쿠키의 다양한 유형에 대한 설명입니다.

- 세션 쿠키는 사용자의 웹사이트 세션 동안에만 지속됩니다. 사용자가 브라우저나 탭을 닫으면 세션 쿠키는 삭제됩니다. 이 쿠키들은 방문 중 사용자를 로그인 상태로 유지하는 등의 작업에 주로 사용됩니다.

- 영속 쿠키는 만료일이 있으며, 만료일까지 사용자의 기기에 저장됩니다. 영속 쿠키는 세션 간 사용자 환경 설정이나 로그인 정보를 기억하는 데 자주 사용됩니다.

- 보안 쿠키는 HTTPS를 통해서만 전송되어 전송 중 공격자가 가로채지 못하도록 보장합니다.

- HttpOnly 쿠키는 브라우저에서 실행되는 JavaScript가 접근하거나 수정할 수 없어 교차 사이트 스크립팅(XSS) 공격에 대해 더 안전합니다. XSS 공격은 공격자가 악성 스크립트를 웹 페이지에 주입해 다른 사용자가 볼 때 실행되도록 하는 공격입니다. 이 스크립트는 피해자 브라우저에서 쿠키나 세션 데이터를 훔치거나 사용자의 동의 없이 악의적 행동을 할 수 있습니다. 쿠키를 HttpOnly로 표시하면 JavaScript를 통한 접근이 차단되어 이런 공격 위험이 줄어듭니다.

쿠키는 서버 응답에서 `Set-Cookie` 헤더를 사용하거나 JavaScript에서 `document.cookie`를 사용해 생성할 수 있습니다.

다음은 JavaScript로 쿠키를 설정하는 예시입니다:

```js
document.cookie = "username=JohnDoe; expires=Fri, 31 Dec 2021 23:59:59 GMT; path=/";
```

이 명령은 `username`라는 이름의 쿠키를 값 `"JohnDoe"`으로 설정하며 2021년 말에 만료됩니다. 기존 쿠키는 새 값으로 다시 설정해 업데이트할 수 있습니다.

쿠키를 삭제하려면 만료일을 과거로 설정하세요:

```js
document.cookie = "username=; expires=Thu, 01 Jan 1970 00:00:00 GMT; path=/";
```

이렇게 하면 브라우저에서 `username` 쿠키가 삭제됩니다.
