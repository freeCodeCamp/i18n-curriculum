# --interactive--

`Number` 型別是 JavaScript 和其他程式語言中最常使用的資料型別之一。數字看起來很簡單，但在 JavaScript 中關於數字有很多值得探索的地方。所以，讓我們深入了解。在 JavaScript 中，`Number` 資料型別表述數值。

與許多其他將整數和浮點數分為不同型別的程式語言不同，JavaScript 使用一個統一的 `Number` 型別來處理數字。這表示你可以在同一個 `Number` 資料型別下操作整數、小數，甚至特殊數值。

這是一個基本範例，向你展示整數、浮點數和負數都是 `number` 型別。

:::interactive_editor

```js
const wholeNumber = 50;
const decimalNumber = 4.5;
const negativeNumber = -7;

console.log(typeof wholeNumber); // number
console.log(typeof decimalNumber); // number
console.log(typeof negativeNumber); // number
```

:::

JavaScript 的 `Number` 型別包含各種數值，從簡單的整數和浮點數到特殊情況如 `Infinity` 和 `NaN`，或稱為「非數值」。讓我們拆解你會遇到的主要型別。整數是沒有任何分數或小數部分的整數字。它們可以是正數、負數或零。以下是一些範例：

:::interactive_editor

```js
const positiveInteger = 100;
const negativeInteger = -25;
const zero = 0;

console.log(typeof positiveInteger); // number
console.log(typeof negativeInteger); // number
console.log(typeof zero); // number
```

:::

浮點數是帶有小數點的數字。JavaScript 開發者通常簡稱為「float」。當你需要更高精度時，例如處理測量值或貨幣，浮點數非常有用。以下是一些範例：

:::interactive_editor

```js
const floatingPointNumber = 4.5;
const anotherFloat = 89.56;
const oneMoreFloat = 16.462;

console.log(typeof floatingPointNumber); // number
console.log(typeof anotherFloat); // number
console.log(typeof oneMoreFloat); // number
```

:::

JavaScript 可以用 `Infinity` 表述超出最大限制的數字。當你嘗試用零除一個數字，或在少數情況下超過 `Number` 型別的上限時，就會遇到這種情況。以下是一個範例：

:::interactive_editor

```js
const infiniteNumber = 1 / 0;
console.log(infiniteNumber); // Infinity
console.log(typeof infiniteNumber); // number
```

:::

有時在 JavaScript 中，某些數學操作不會產生有效的數字。例如，如果你嘗試對非數字的東西執行數學操作，你會得到 `NaN`，代表「不是數字」：

:::interactive_editor

```js
const notANumber = 'hello world' / 2;
console.log(notANumber); // NaN
```

:::

令人驚訝的是，`NaN` 的型別也是 `Number`：

:::interactive_editor

```js
const notANumber = 'hello world' / 2;
console.log(typeof notANumber); // number
```

:::

除了標準的十進位制（base 10）之外，JavaScript 也支援不同進位制的數字，例如二進位、八進位和十六進位。二進位是使用僅有數字 1 和 0 的 base-2 制度。八進位是使用僅有數字 0 到 7 的 base-8 制度。十六進位是使用數字 0 到 9 以及字母 a 到 f 的 base-16 制度，就像你在 CSS 十六進位顏色中看到的那樣。
