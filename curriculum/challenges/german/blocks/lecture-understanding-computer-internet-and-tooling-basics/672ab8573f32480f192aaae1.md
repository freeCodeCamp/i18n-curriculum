# --description--

Professionelle Entwickler verlassen sich auf eine Vielzahl von Tools, um Produktivität und Codequalität zu steigern. Lernen wir diese Tools kennen, einschließlich derjenigen, die sehr offensichtlich erscheinen.

An erster Stelle unter den Tools stehen Computer. Ein Computer ist die primäre Entwicklungsumgebung. Es kann ein leistungsstarker Desktop oder ein tragbarer Laptop mit Windows, macOS oder Linux als Betriebssystem sein.

Professionelle Entwickler entscheiden sich oft für Computer mit schneller Rechenleistung und viel RAM, um ressourcenintensive Aufgaben zu bewältigen.

Nach dem Computer folgt ein Programm zum Schreiben und Bearbeiten von Code direkt auf diesem Computer. Das ist ein Code-Editor oder eine integrierte Entwicklungsumgebung (IDE).

IDEs wie Visual Studio, IntelliJ IDEA, JetBrains und PyCharm bieten leistungsstarke Funktionen wie Codevervollständigung, Debugging und integrierte Terminalunterstützung.

Visual Studio Code (VS Code) ist im Wesentlichen ein Code-Editor, bietet aber auch diese Funktionen mit seiner umfangreichen Erweiterungsbibliothek.

Weitere Code-Editoren sind Sublime Text, Atom, Notepad++ und Brackets.

Wenn Sie Code mit Code-Editoren und IDEs schreiben, müssen Sie die Änderungen, die Sie daran vornehmen, verfolgen. Das Tool, das Ihnen erlaubt, diese Änderungen zu verfolgen, ist ein Versionskontrollsystem.

Git ist das am weitesten verbreitete Versionskontrollsystem in der Entwickler-Community.

Plattformen wie GitHub, GitLab und Bitbucket bieten cloudbasierte Hosting-Dienste für Ihre Git-Repositories. Diese Plattformen ermöglichen die Zusammenarbeit mit anderen Entwicklern, erlauben das Arbeiten an mehreren Branches und erleichtern das Zusammenführen von Codeänderungen.

Paketmanager sind ein weiteres wichtiges Tool. Sie helfen Entwicklern, den Prozess des Hinzufügens, Aktualisierens und Entfernens von Bibliotheken und Projektabhängigkeiten zu vereinfachen.

Beispiele für beliebte Paketmanager sind:

* NPM, Yarn und PNPM für JavaScript
* PIP für Python
* Composer für PHP
* Maven für Java

Nachdem Entwickler Code mit verschiedenen Tools geschrieben haben, testen sie diesen Code, um sicherzustellen, dass er wie erwartet funktioniert.

Dafür verwenden Entwickler Test-Frameworks wie Cypress, Playwright, Selenium und andere. Es gibt auch sprachspezifische Test-Frameworks. Beispiele sind:

* Jest für JavaScript
* pytest für Python
* JUnit für Java
* PHPUnit für PHP

Sie müssen nicht nur den Code testen, um sicherzustellen, dass er wie erwartet funktioniert. Sie müssen auch testen, wie der Code visuell für die Endnutzer aussieht. Entwickler verwenden dafür Webbrowser.

Moderne Browser wie Chrome, Firefox, Edge und Safari bieten Entwickler-Tools zum Inspizieren von HTML-, CSS- und JavaScript-Code. Es gibt auch Tools zum Debuggen und zur Leistungsanalyse.

Diese Tools helfen Entwicklern, ihre Webanwendungen für die Endnutzer in verschiedenen Browsern zu testen und zu optimieren.
