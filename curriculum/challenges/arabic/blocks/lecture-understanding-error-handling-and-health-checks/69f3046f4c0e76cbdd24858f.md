# --description--

هل حدث أن تعطل تطبيق Express الخاص بك دون ظهور رسالة خطأ واضحة؟ تساعدك معالجة الأخطاء وتصحيحها وتسجيل الأحداث على رؤية ما يحدث خلف الكواليس بالضبط، لتتمكن من العثور على المشكلات وإصلاحها بسرعة أكبر.

في هذا الدرس، سنستكشف كيف يتعامل Express مع معالجة الأخطاء وتصحيحها وتسجيل الأحداث—وكيف يمكنك استخدام ذلك لتحسين سير عمل التطوير لديك.

## معالجة الأخطاء المدمجة في Express

يتضمن Express أداة معالجة أخطاء مدمجة تسمى `debug`. لا تحتاج إلى تثبيت أي شيء لاستخدامها.

لتشغيل معالجة الأخطاء، كل ما عليك فعله هو تعيين متغير البيئة `DEBUG` عند تشغيل التطبيق.

إليك مثالاً:

```js
DEBUG=express:* node index.js
```

يُمكّن هذا الأمر جميع رسائل معالجة الأخطاء تحت مساحة الأسماء `express:*`. ستبدأ برؤية مخرجات مثل:

```js
express:router:route dispatching GET /
express:router:layer processing GET /
```

تساعدك هذه الرسائل على تتبع توجيه المسارات، واستدعاءات البرمجيات الوسيطة، والمزيد. إنها طريقة رائعة لفهم تدفق الطلبات في تطبيقك.

هل تريد أن تكون أكثر تحديدًا؟ يمكنك تضييق مساحة الأسماء:

```js
DEBUG=express:router node index.js
```

سيعرض هذا فقط السجلات المتعلقة بمنطق الموجِّه في Express.

على نظام Windows، الصيغة مختلفة قليلاً في PowerShell:

```js
$env:DEBUG = "express:*"; node index.js
```

أما في CMD، فتبدو هكذا:

```js
set DEBUG=express:* && node index.js
```

الفكرة نفسها، فقط طريقة مختلفة لتعيين متغيرات البيئة.

الآن لننظر إلى تسجيل الأحداث في Express.

لتسجيل طلبات HTTP، يستخدم العديد من المطوّرين برمجيات وسيطة مثل `morgan`.

إليك كيفية استخدامها:

```bash
npm install morgan
```

ثم أضفها إلى تطبيقك:

```js
const express = require('express')
const morgan = require('morgan')
const app = express()

app.use(morgan('dev'))s
```

الآن، سيتم تسجيل كل طلب وارد في الطرفية مع تفاصيل مثل الطريقة، وعنوان URL، ورمز الحالة، ووقت الاستجابة.

ولماذا هذا مهم؟

تمنحك معالجة الأخطاء نظرة على كيفية تعامل Express مع الطلبات، والبرمجيات الوسيطة، والتوجيه دون إفساد سجلات الإنتاج.

يوفر التسجيل سجلًا لما يحدث عبر الطلبات، وهو أمر حاسم لتصحيح الأخطاء، وفهم حركة المرور، واكتشاف المشكلات مبكرًا.

معًا، يمنحانك السيطرة والوضوح في عملية التطوير.
