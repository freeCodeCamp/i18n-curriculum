# --description--

Per mostrare un codice boilerplate HTML, prima ti serve un file HTML con cui lavorare.

Crea una nuova cartella sul tuo computer e chiamala `html-boilerplate`. All’interno di quella cartella crea un nuovo file e chiamalo `index.html`.

Probabilmente conosci già molti tipi diversi di file, per esempio file doc, pdf e immagini.

Per far capire al computer che vuoi creare un file HTML, devi aggiungere al nome del file l’estensione `.html` come hai fatto quando hai creato il file `index.html`.

Vale la pena notare che hai chiamato il tuo file HTML index. Dovresti sempre chiamare il file HTML che conterrà la homepage dei tuoi siti `index.html`. Questo perché i server web per impostazione predefinita cercano una pagina `index.html` quando gli utenti arrivano sui tuoi siti — e non averla può causare grossi problemi.

## The DOCTYPE

Ogni pagina HTML inizia con una dichiarazione doctype. Lo scopo del doctype è dire al browser quale versione di HTML deve usare per renderizzare il documento. L’ultima versione di HTML è HTML5, e il doctype per quella versione è semplicemente `<!DOCTYPE html>`.

I doctype per le versioni più vecchie di HTML erano un po’ più complicati. Per esempio, questa è la dichiarazione doctype per HTML4:

```html
<!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.01 Transitional//EN" "http://www.w3.org/TR/html4/loose.dtd">
```

Tuttavia, probabilmente non vorrai mai usare una versione più vecchia di HTML, quindi userai sempre `<!DOCTYPE html>`.

Apri il file `index.html` creato prima nel tuo editor di testo e aggiungi `<!DOCTYPE html>` alla primissima riga.
