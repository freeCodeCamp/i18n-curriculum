# --description--

الـ **طابور الأولوية** هو نوع مجرد من هياكل البيانات (ADT) يعمل بطريقة مشابهة للطابور أو المكدس، لكن مع فرق رئيسي واحد.

كما قد تعرف بالفعل، تتبع الطوابير العادية نهج FIFO (الأول داخل، الأول خارج)، حيث يكون العنصر الأول الذي أُضيف إلى الطابور هو أول عنصر يُزال منه.

تتبع المكدسات نهج LIFO (الأخير داخل، الأول خارج)، حيث يكون العنصر الأخير الذي أُضيف إلى المكدس هو أول عنصر يُزال منه.

تأخذ الطوابير والمكدسات بعين الاعتبار فقط ترتيب إدخال العناصر.

لكن طوابير الأولوية تأخذ "الأولوية" للعناصر في الحسبان. تُستخدم الأولوية لتحديد أي عنصر يجب إزالته بعد ذلك.

عادةً، يُزال العنصر ذو الأولوية الأعلى أولاً، لكن بعض التنفيذات قد تختار أيضًا إزالة العنصر ذو الأولوية الأدنى أولاً. هذا يعتمد على متطلبات برنامجك.

طوابير الأولوية مفيدة جدًا في التطبيقات العملية مثل إيجاد أقصر مسار بين موقعين، جدولة المهام في أنظمة التشغيل، محاكاة حركة المرور، ضغط البيانات، وإدارة الشبكات.

في الواقع، تُنفذ طوابير الأولوية عادةً باستخدام هيكل بيانات الحزمة.

الـ **حزمة** هي هيكل بيانات شجري له خاصية محددة جدًا تُسمى **خاصية الحزمة**. تحدد هذه الخاصية العلاقة بين كل عقدة وأولادها، بناءً على نوع الحزمة.

هناك نوعان رئيسيان من الحزم:

* الحزمة العظمى
    
* الحزمة الصغرى
    

في **الحزمة العظمى**، تكون قيمة كل عقدة أكبر من أو تساوي قيمة أولادها.

في هذا المثال، يمكنك رؤية هيكل شجري يحتوي على العقد 8، 7، 5، 2، و1. لاحظ أن العقدة 7 أكبر من العقدتين 2 و1، وفقًا لخاصية الحزمة. وهذا صحيح لجميع العقد الأخرى أيضًا.

<img src="https://cdn.freecodecamp.org/curriculum/lecture-transcripts/how-do-priority-queues-and-heaps-work-1.png" alt="A max-heap tree structure showing nodes with values 8 at the root, 7 and 5 as children of 8, and 2 and 1 as children of 7, demonstrating that each parent node is greater than its children.">

على العكس، في **الحزمة الصغرى**، تكون قيمة كل عقدة أقل من أو تساوي قيمة أولادها.

في هذا المثال، لدينا عقد بقيم 4، 7، 9، 12، و15. على سبيل المثال، العقدة 7 أقل من العقدتين 12 و15، وفقًا لخاصية الحزمة. وهذا صحيح أيضًا لجميع العقد الأخرى.

<img src="https://cdn.freecodecamp.org/curriculum/lecture-transcripts/how-do-priority-queues-and-heaps-work-2.png" alt="A min-heap tree structure showing nodes with values 4 at the root, 7 and 9 as children of 4, and 12 and 15 as children of 7, demonstrating that each parent node is less than its children.">

خاصية الحزمة مهمة لأنها تضمن بقاء العنصر الأقصى (أو الأدنى) دائمًا في الأعلى، مما يجعل إزالته سهلة جدًا.

في الواقع، تُنفذ الحزم عادةً كمصفوفات للوصول بكفاءة إلى العقد الأم والأبناء.

استخدام المصفوفات يبسط منطق الوصول إلى هذه القيم أو "العقد" لأنه خلف الكواليس، إذا حافظت الحزمة على هيكل شجرة ثنائية كاملة، فإن تنفيذ المصفوفة يتطلب فقط عمليات رياضية بسيطة بناءً على الفهارس لتحديد مكان العناصر في الذاكرة.

لا يحتوي JavaScript على وحدة حزمة مدمجة، لكن يمكنك تنفيذ حزمة صغرى باستخدام مصفوفة.

إليك تنفيذًا أساسيًا للحزمة الصغرى في JavaScript:

```js
class MinHeap {
  constructor(compare = (a, b) => a - b) {
    this.data = [];
    this.compare = compare;
  }

  peek() {
    return this.data[0];
  }

  push(value) {
    this.data.push(value);
    this.#bubbleUp(this.data.length - 1);
  }

  pop() {
    if (this.data.length === 0) return undefined;

    const top = this.data[0];
    const last = this.data.pop();

    if (this.data.length > 0) {
      this.data[0] = last;
      this.#bubbleDown(0);
    }

    return top;
  }

  pushPop(value) {
    if (this.data.length === 0) return value;

    if (this.compare(this.data[0], value) < 0) {
      const top = this.data[0];
      this.data[0] = value;
      this.#bubbleDown(0);
      return top;
    }

    return value;
  }

  heapify(arr) {
    this.data = arr.slice();
    for (let i = Math.floor(this.data.length / 2) - 1; i >= 0; i--) {
      this.#bubbleDown(i);
    }
  }

  #bubbleUp(i) {
    while (i > 0) {
      const p = Math.floor((i - 1) / 2);
      if (this.compare(this.data[i], this.data[p]) >= 0) break;
      [this.data[i], this.data[p]] = [this.data[p], this.data[i]];
      i = p;
    }
  }

  #bubbleDown(i) {
    const n = this.data.length;

    while (true) {
      let smallest = i;
      const l = 2 * i + 1;
      const r = 2 * i + 2;

      if (l < n && this.compare(this.data[l], this.data[smallest]) < 0) smallest = l;
      if (r < n && this.compare(this.data[r], this.data[smallest]) < 0) smallest = r;

      if (smallest === i) break;

      [this.data[i], this.data[smallest]] = [this.data[smallest], this.data[i]];
      i = smallest;
    }
  }
}
````

لاستخدام هذه الحزمة، يمكنك إنشاء حزمة فارغة. ستكون هذه هيكل البيانات الأساسي للحزمة:

```js
const myHeap = new MinHeap();
```

لإضافة عناصر إلى الحزمة، تستدعي `push()`. سيضيف هذا العنصر تلقائيًا في المكان المناسب للحفاظ على خاصية الحزمة:

```js
myHeap.push(9);
```

للحصول على العنصر ذو الأولوية الأعلى (في هذه الحالة، القيمة الأصغر)، تستدعي `pop()`:

```js
myHeap.pop();
```

`pushPop()` يجمع بين العمليتين في استدعاء واحد.

هذا أكثر كفاءة من استدعائهما بشكل منفصل، خاصة عندما تكون الحزمة كبيرة، لأنه ينفذ عملية إعادة ترتيب واحدة فقط:

```js
myHeap.pushPop(15);
```

إذا كان لديك مصفوفة بالفعل وتريد تحويلها إلى حزمة، يمكنك استدعاء `heapify()`:

```js
myHeap.heapify([9, 2, 7, 1]);
```

لكننا حاليًا نرتب العناصر حسب قيمها، أليس كذلك؟

ماذا لو أردنا ترتيبها حسب "الأولوية" بدلاً من ذلك؟

يمكنك فعل ذلك بتخزين مصفوفات بهذا الهيكل: `[priority, element]`.

في JavaScript، لا تُقارن المصفوفات تلقائيًا عنصرًا عنصرًا للترتيب، لذا تمرر دالة مقارنة إلى الحزمة تقارن الأولويات أولاً.

يرجى ملاحظة أنه في هذه الحالة، القيم الأقل تمثل أولويات أعلى. هذا يعني أن عنصرًا بأولوية 1 سيكون له **أولوية أعلى** من عنصر بأولوية 3:

```js
const myHeap = new MinHeap((a, b) => a[0] - b[0]);

myHeap.push([3, "A"]);
myHeap.push([2, "B"]);
myHeap.push([1, "C"]);
```

إذا كنت تحتاج إلى إزالة العناصر ذات الأولوية نفسها حسب ترتيب إدخالها، يمكنك تضمين عداد فريد كالعنصر الثاني لكسر التعادل، مثل هذا `[priority, counter, element]`.

على سبيل المثال، يمكنك المقارنة حسب الأولوية أولاً، ثم حسب العداد:

```js
let counter = 0;
const myHeap = new MinHeap((a, b) => (a[0] - b[0]) || (a[1] - b[1]));

myHeap.push([3, counter++, "A"]);
myHeap.push([2, counter++, "B"]);
myHeap.push([2, counter++, "C"]);
```

الآن دعنا نتحدث عن كفاءة الحزم.

تعقيدات الوقت في المتوسط وأسوأ الحالات لإدخال واستخراج القيمة الدنيا أو العليا من الحزمة (حسب نوع الحزمة) هي لوغاريتمية، `O(log n)`, لأن عدد التبديلات المطلوبة عادةً ما يكون متناسبًا مع ارتفاع الحزمة، وهو log(n).

تعقيد الوقت في المتوسط وأسوأ الحالات لعملية "النظر" هو وقت ثابت، O(1). النظر يعني الحصول على القيمة الدنيا أو العليا (حسب نوع الحزمة) دون إزالتها.

عملية "heapify"، حيث تُبنى الحزمة من قائمة غير مرتبة، لها تعقيد زمني خطي، O(n)، في المتوسط وأسوأ الحالات.

بنفس الطريقة، كل من البحث عن عنصر عشوائي وحذفه لهما تعقيد زمني خطي في المتوسط وأسوأ الحالات O(n)، لأنهما قد يتطلبان اجتياز الحزمة بأكملها.

وكم من المساحة تحتاج الحزم؟

تعقيد المساحة للحزمة خطي، O(n)، حيث `n` هو عدد العناصر التي تحتويها. تحتاج فقط إلى تخزين العناصر وبعض الحمل الإضافي الصغير لكائن المصفوفة نفسه.

طوابير الأولوية والحزم مهمة جدًا في علوم الحاسوب. تتيح لك العثور بسرعة على أهم العناصر في قائمة واستخدامها. هذه الكفاءة حاسمة للعديد من برامج الحاسوب التي تنفذ مهامًا حقيقية حرجة، مثل إيجاد أسرع مسار على الخريطة.
