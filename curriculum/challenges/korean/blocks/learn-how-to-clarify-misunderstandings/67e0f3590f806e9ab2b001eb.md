# --explanation--

`Sit in on`는 수업이나 회의 같은 것에 적극적으로 참여하지 않고 참석하는 것을 의미합니다. 예를 들어:

`She asked if she could sit in on our design session.` – 그녀는 세션을 주도하거나 많이 말하지 않고 관찰하고 싶어 했습니다.

`Code review`는 개발자가 서로의 코드를 오류와 개선점 측면에서 점검하는 과정입니다. 예를 들어:

`We found a small bug during the code review.` – 팀이 코드를 검토하고 문제를 발견했습니다.
