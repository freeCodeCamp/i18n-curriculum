# --interactive--

`input`、`select`、`textarea`、`button`、`fieldset`のような意味論的なフォームコントロール要素には、アクセシビリティ技術に伝えられる組み込みの状態があります。

たとえば、`disabled`属性を使ってボタンを無効化したり、`checked`属性でチェックボックスがチェックされていることを示したりできます。

しかし、カスタムコントロール要素を作成する場合は、ARIA属性を使ってコントロールの状態をアクセシビリティ技術に伝える必要があります。

このレッスンでは、カスタムコントロール要素で使えるいくつかの一般的なARIA状態について説明します。

最初に説明するARIA状態は`aria-selected`です。この状態は要素が選択されていることを示すために使われます。タブ付きインターフェース、リストボックス、グリッドなどのカスタムコントロールで使えます。

以下はカスタムタブコントロールで`aria-selected`を使う例です。

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<div role="tablist">
  <button role="tab" aria-selected="true">Tab 1</button>
  <button role="tab" aria-selected="false">Tab 2</button>
  <button role="tab" aria-selected="false">Tab 3</button>
</div>
<script src="index.js"></script>
```

```css
[role="tablist"] {
  display: flex;
  border-bottom: 2px solid #ddd;
  gap: 0.25rem;
  font-family: system-ui, sans-serif;
}

[role="tab"] {
  appearance: none;
  border: none;
  background: none;
  padding: 0.5rem 1rem;
  cursor: pointer;
  font-size: 1rem;
  color: #444;
  border-radius: 4px 4px 0 0;
  transition: background-color 0.2s, color 0.2s;
}

[role="tab"]:hover {
  background-color: #f3f3f3;
}

[role="tab"][aria-selected="true"] {
  background-color: #fff;
  color: #0078d4;
  border: 2px solid #0078d4;
  border-bottom: 2px solid #fff; 
  font-weight: 600;
  position: relative;
  z-index: 1;
}

[role="tab"]:focus {
  outline: 2px solid #0078d4;
  outline-offset: 2px;
}

```

```js
document.addEventListener("click", (event) => {
  const clickedTab = event.target.closest('[role="tab"]');
  if (!clickedTab) return;

  const tablist = clickedTab.closest('[role="tablist"]');
  const tabs = tablist.querySelectorAll('[role="tab"]');

  tabs.forEach((tab) => {
    const isSelected = tab === clickedTab;
    tab.setAttribute("aria-selected", isSelected);
    tab.tabIndex = isSelected ? 0 : -1;
  });
});

document.addEventListener("keydown", (event) => {
  const activeTab = document.activeElement;
  if (activeTab.getAttribute("role") !== "tab") return;

  const tablist = activeTab.closest('[role="tablist"]');
  const tabs = Array.from(tablist.querySelectorAll('[role="tab"]'));
  const index = tabs.indexOf(activeTab);

  let newIndex = index;
  if (event.key === "ArrowRight") newIndex = (index + 1) % tabs.length;
  if (event.key === "ArrowLeft") newIndex = (index - 1 + tabs.length) % tabs.length;

  if (newIndex !== index) {
    tabs[newIndex].focus();
    tabs[newIndex].click();
  }
});

```

:::

タブは限られたスペースで複数のコンテンツパネルを表示するために使われます。`aria-selected`状態は現在選択されているタブを示すために使われます。

ユーザーがタブを選択すると、選択されたタブの`aria-selected`状態は`true`に設定され、他のタブの`aria-selected`状態は`false`に設定されます。

もう一つの一般的なARIA状態は`aria-disabled`です。この状態はスクリーンリーダーなどのアクセシビリティ技術を使う人に対して要素が無効であることを示します。重要なのは、`aria-disabled`は実際に要素を無効化しないことです。見た目や動作を無効のようにするのは開発者の役割です。この属性はネイティブHTML要素で`disabled`属性の代わりに使われることも多いです。どちらを使うかはボタンの使われる文脈によります。

以下はカスタム編集ボタンで`aria-disabled`を使う例です。

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<div role="button" tabindex="-1" aria-disabled="true">Edit</div>
```

```css
[role="button"] {
  display: inline-block;
  background-color: #0078d4;
  color: #fff;
  padding: 0.5rem 1rem;
  border-radius: 6px;
  font-family: system-ui, sans-serif;
  font-size: 1rem;
  text-align: center;
  cursor: pointer;
  user-select: none;
  transition: background-color 0.2s, transform 0.1s, opacity 0.2s;
}

[role="button"]:not([aria-disabled="true"]):hover {
  background-color: #005fa3;
}

[role="button"]:not([aria-disabled="true"]):focus {
  outline: 2px solid #005fa3;
  outline-offset: 2px;
}

[role="button"]:not([aria-disabled="true"]):active {
  transform: scale(0.97);
}

[role="button"][aria-disabled="true"] {
  opacity: 0.5;
  pointer-events: none;
  cursor: not-allowed;
  background-color: #b0b0b0;
  color: #f2f2f2;
}
```

:::

`aria-disabled`属性は編集ボタンが無効で操作できないことをスクリーンリーダーユーザーに伝えます。繰り返しますが、実際にボタンを無効化するわけではありません。`aria-disabled`を使う場合は、見た目や動作を無効のボタンのようにするためにスタイルやJavaScriptを適用する必要があります。

ほとんどの場合はネイティブのbutton要素を使うでしょうが、カスタムコントロールが必要な場合もあります。だから、コントロールの状態をアクセシビリティ技術に伝える方法を知っておくことが重要です。

次に説明するARIA状態は`aria-haspopup`です。この状態はインタラクティブな要素が起動されたときにポップアップ要素を表示することを示します。`aria-haspopup`属性はポップアップが`menu`、`listbox`、`tree`、`grid`、または`dialog`のいずれかの役割を持つ場合にのみ使えます。`aria-haspopup`の値はこれらの役割のいずれかか、デフォルトで`true`役割となる`menu`でなければなりません。

以下は`aria-haspopup`を使ったファイルエディターメニューの例です。

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<button id="menubutton" aria-haspopup="menu" aria-controls="filemenu" aria-expanded="false">File</button>
<ul id="filemenu" role="menu" aria-labelledby="menubutton" hidden>
  <li role="menuitem" tabindex="-1">Open</li>
  <li role="menuitem" tabindex="-1">New</li>
  <li role="menuitem" tabindex="-1">Save</li>
  <li role="menuitem" tabindex="-1">Delete</li>
</ul>
```

```css
#menubutton {
  background-color: #0078d4;
  color: #fff;
  border: none;
  padding: 8px 14px;
  border-radius: 4px;
  cursor: pointer;
  font-size: 14px;
  position: relative;
}

#menubutton:hover,
#menubutton:focus {
  background-color: #005ea2;
  outline: none;
}

#filemenu {
  list-style: none;
  padding: 4px 0;
  margin: 4px 0 0;
  border: 1px solid #ccc;
  border-radius: 4px;
  background-color: #fff;
  width: 160px;
  position: absolute;
  z-index: 1000;
  box-shadow: 0 4px 6px rgba(0,0,0,0.1);
}

#filemenu[hidden] {
  display: none;
}

#filemenu [role="menuitem"] {
  display: block;
  padding: 8px 12px;
  font-size: 14px;
  color: #333;
  cursor: pointer;
}

#filemenu [role="menuitem"]:hover,
#filemenu [role="menuitem"]:focus {
  background-color: #e5f1fb;
  outline: none;
}

#filemenu [role="menuitem"]:focus-visible {
  box-shadow: inset 0 0 0 2px #0078d4;
}

```

:::

`aria-haspopup`状態は`File`メニューボタンが起動されたときにポップアップメニューを開くことを示します。スクリーンリーダーユーザーはボタンに移動したときにこの追加情報を聞く場合があります。

ポップアップメニューの表示・非表示や、メニューとやりとりするための適切なキーボードサポートはJavaScriptで実装する必要があります。また、ARIAの`menu`役割は非常に特定のタイプのメニューを指します。これはデスクトップアプリのメニューのようにユーザーが呼び出せるアクションのリストを指し、ナビゲーションメニューのような一般的な「メニュー」には含まれません。実際にはウェブで作るほとんどの「メニュー」はARIAメニューではなく、`aria-haspopup`は使いません。

次に説明するARIA状態は`aria-required`です。`aria-required`属性はフォームが提出される前に入力が必要なフィールドであることを示します。

以下はカスタムフォームコントロールで`aria-required`属性を使う例です。

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<div id="name-label">Full Name*</div>
<div role="textbox" contenteditable aria-labelledby="name-label" aria-required="true" id="name"></div>
```

```css
#name-label {
  font-family: system-ui, sans-serif;
  font-size: 0.95rem;
  font-weight: 600;
  margin-bottom: 0.25rem;
  color: #333;
}

[role="textbox"] {
  display: block;
  width: 100%;
  min-height: 2rem;
  padding: 0.5rem 0.75rem;
  border: 1.5px solid #ccc;
  border-radius: 4px;
  font-family: system-ui, sans-serif;
  font-size: 1rem;
  color: #222;
  background-color: #fff;
  line-height: 1.4;
  transition: border-color 0.2s, box-shadow 0.2s;
}

[role="textbox"]:hover {
  border-color: #999;
}

[role="textbox"]:focus {
  outline: none;
  border-color: #0078d4;
  box-shadow: 0 0 0 3px rgba(0, 120, 212, 0.25);
}

[role="textbox"]:empty::before {
  content: attr(data-placeholder);
  color: #aaa;
  pointer-events: none;
}

[role="textbox"][aria-required="true"] {
  border-left: 3px solid #e81123;
  padding-left: calc(0.75rem - 3px);
}
```

:::

ユーザーが入力できるように`contenteditable`属性を使う必要があります。また、このカスタムフォームコントロールが必須であることを示すために`aria-required`属性を`true`に設定しています。

フォームコントロールを通常のフォームコントロールのように見せるにはCSSを追加する必要があります。内容がないままフォームが提出されるのを防ぐためにJavaScriptも追加する必要があります。

ラベルにすでに`required`という単語が含まれている場合は、`aria-required`属性は省略してください。これによりスクリーンリーダーが「必須」という単語を一度だけ読み上げます。

ほとんどの場合はネイティブの`label`と`form`要素に`required`属性を使うでしょう。しかしカスタムフォームコントロールを作る場合は、必要に応じて`aria-required`属性を追加することが重要です。

さらに、`aria-required`属性は`input`、`textarea`、`select`などのネイティブフォーム入力にも使えます。これはネイティブのrequired属性より好まれることが多いです。なぜならrequired属性はブラウザのデフォルトのエラーハンドリングにより使い勝手やアクセシビリティに問題がある場合があるからです。最終的にはどの属性が最適かはテストして判断してください。

最後に説明するARIA状態は`aria-checked`です。この属性は要素がチェックされた状態かどうかを示します。カスタムチェックボックス、ラジオボタン、スイッチ、リストボックスを作るときに最もよく使われます。

以下はカスタムチェックボックスコントロールで`aria-checked`を使う例です。

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<div role="checkbox" aria-checked="true" tabindex="0">Checkbox</div>
```

```css
[role="checkbox"] {
  display: inline-flex;
  align-items: center;
  gap: 0.5rem;
  font-family: system-ui, sans-serif;
  font-size: 1rem;
  cursor: pointer;
  user-select: none;
  color: #222;
}

[role="checkbox"]::before {
  content: "";
  display: inline-block;
  width: 1rem;
  height: 1rem;
  border: 2px solid #666;
  border-radius: 4px;
  background-color: #fff;
  transition: all 0.2s ease;
  box-sizing: border-box;
}

[role="checkbox"]:hover::before {
  border-color: #0078d4;
}

[role="checkbox"]:focus::before {
  outline: 2px solid #0078d4;
  outline-offset: 2px;
}

[role="checkbox"][aria-checked="true"]::before {
  background-color: #0078d4;
  border-color: #0078d4;
  background-image: url("data:image/svg+xml;utf8,<svg xmlns='http://www.w3.org/2000/svg' width='14' height='14' viewBox='0 0 14 14'><path fill='white' d='M5.2 10.4L2 7.2l1.1-1.1 2.1 2.1L10.9 2.5 12 3.6z'/></svg>");
  background-repeat: no-repeat;
  background-position: center;
}

[role="checkbox"][aria-disabled="true"] {
  opacity: 0.5;
  pointer-events: none;
  cursor: not-allowed;
}

```

:::

ネイティブのチェックボックス要素にはアクセシビリティ技術に伝えられる組み込みの`checked`状態があります。しかしカスタムチェックボックスコントロールを作る場合は、状態を示すために`aria-checked`属性を使う必要があります。

ユーザーがカスタムチェックボックスコントロールを操作すると、チェックボックスの新しい状態を反映するために`aria-checked`状態を使う必要があります。チェックされているときは`aria-checked`属性が`true`に設定され、チェックが外れているときは`false`に設定されます。

ネイティブ要素は通常、より良いサポートと組み込みのアクセシビリティ機能を持っています。

しかし、カスタムコントロールを作る必要がある場合は、ARIA属性を使ってこれらのコントロールの状態をアクセシビリティ技術に効果的に伝えることが不可欠です。

いつも通り、ARIA属性が正しく適用されているか、カスタムコントロールがアクセシブルで使いやすく機能しているかをテストしてください。
