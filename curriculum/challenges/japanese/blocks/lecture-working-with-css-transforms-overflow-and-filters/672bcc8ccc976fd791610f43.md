# --interactive--

CSSの`transform`プロパティは、ウェブページ上の要素の見た目を変更できる強力なツールで、他の要素のレイアウトには影響を与えません。要素を回転させたり、拡大縮小したり、傾けたり、2Dや3D空間で移動（平行移動）させたりするさまざまな変形を適用できます。

`transform`プロパティは、要素の座標系に数学的な変換を適用することで機能します。つまり、要素の形や位置を操作しつつ、元の位置やドキュメントの流れはそのまま保たれます。

よく使われる変形関数を見てみましょう。以下はボックス要素の例です。

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<div class="box"></div>
```

```css
body {
  border: 2px solid black;
}

.box {
  width: 200px;
  height: 200px;
  background-color: red;
}
```

:::

`body`に黒の実線ボーダーを設定して、`.box`の中にネストされた`body`要素が見えるようにしています。

`translate`関数は、要素を現在の位置から移動させます。次は`translate`関数を使った更新例です。

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<div class="box"></div>
```

```css
body {
  border: 2px solid black;
}

.box {
  width: 200px;
  height: 200px;
  background-color: red;
  transform: translate(50px, 100px);
}
```

:::

このCSSルールは、`box`クラスの要素を元の位置から右に50ピクセル、下に100ピクセル移動させます。

`rotate`関数は、要素を固定された点の周りで回転させます。これは先ほどの`rotate`要素に`.box`関数を使った例です。

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<div class="box"></div>
```

```css
.box {
  margin: 100px;
  width: 200px;
  height: 200px;
  background-color: red;
  transform: rotate(45deg);
}
```

:::

この設定で要素は時計回りに45度回転します。

`scale`関数は、要素のサイズを変更できます。例を見てみましょう。

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<div class="box"></div>
```

```css
.box {
  margin: 100px;
  width: 200px;
  height: 200px;
  background-color: red;
  transform: scale(1.5, 2);
}
```

:::

これにより、要素は元の幅の1.5倍、高さは2倍になります。

複数の変形を1つの宣言で組み合わせることもできます。

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<div class="box"></div>
```

```css
.box {
  margin: 100px;
  width: 200px;
  height: 200px;
  background-color: red;
  transform: translate(50px, 50px) rotate(45deg) scale(1.5);
}
```

:::

この例では、要素を右下に50ピクセル移動し、45度回転させ、元のサイズの1.5倍に拡大します。

`transform`プロパティは視覚的に魅力的なデザインを作るのに強力ですが、使用時にはアクセシビリティを考慮することが重要です。以下に重要なアクセシビリティ上の注意点を挙げます。

スクリーンリーダーは変形されたコンテンツを正確に伝えられない場合があります。例えば、`transform`を使って要素の見た目の順序を入れ替えても、スクリーンリーダーは元のDOMの順序で読み上げます。これにより、スクリーンリーダー利用者に混乱を招くことがあります。

`scale`でテキストのサイズを変更する場合は、小さすぎたり大きすぎたりしないよう注意してください。極端に小さいテキストは読みづらく、大きすぎるとコンテナからはみ出して読めなくなることがあります。テキストのサイズ変更には適切なフォントスタイリング手法を使うことをお勧めします。

`transform`をアニメーション効果に使う場合は、動きに敏感なユーザーに配慮してください。過剰または急激なアニメーションは不快感を与えたり、発作を引き起こす場合があります。アニメーションを減らしたりオフにできる方法を用意しましょう。3D変形を使う場合は、すべてのユーザーが同じように奥行きを認識するわけではないことを覚えておいてください。3D効果で伝える重要な情報は、2D形式やテキストでも提供してください。

`transform`でコンテンツを隠したり表示したりする場合は、そのコンテンツがスクリーンリーダーやキーボード操作でアクセス可能なままであることを確認してください。隠す場合は、単に視覚的に画面外に移動させるのではなく、`display: none;`や`visibility: hidden;`のように本当に隠す方法を使いましょう。

`transform`をボタンやリンクなどのインタラクティブな要素に適用する場合は、クリック可能な範囲が直感的で簡単にターゲットできる状態を保ってください。大きく変形したボタンは視覚的に混乱を招いたり、運動機能に制限のあるユーザーにはクリックしづらくなることがあります。

まとめると、CSSの`transform`プロパティは視覚的に動的なウェブデザインを作る強力なツールですが、アクセシビリティを考慮して責任を持って使うことが大切です。変形した要素はさまざまな支援技術でテストし、変形によって影響を受ける情報や機能に代替手段を用意してください。
