# --interactive--

У CSS `vh` і `vw` — це одиниці, що залежать від вікна перегляду, які дозволяють задавати розміри елементів залежно від розмірів вікна браузера. Ці одиниці особливо корисні для створення адаптивних дизайнів, які підлаштовуються під різні розміри екранів.

`vh` означає «висоту вікна перегляду», а `1vh` дорівнює 1% висоти вікна перегляду.

Аналогічно, `vw` означає «ширину вікна перегляду», а `1vw` дорівнює 1% ширини вікна перегляду.

Це означає, що якщо ви встановите висоту елемента в `100vh`, він займатиме всю висоту вікна перегляду, незалежно від фактичних піксельних розмірів пристрою.

Ці одиниці особливо зручні, коли ви хочете створити макети на весь екран або елементи, які зберігають певне співвідношення розмірів екрану.

Наприклад, ви можете використати їх, щоб створити головний розділ, який завжди заповнює весь екран (щоб взаємодіяти з прикладом, потрібно увімкнути інтерактивний редактор).

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css" />

<section class="hero">
  <h1>100vh / 100vw Example</h1>
  <p>This section fills the entire browser window.</p>
</section>
```

```css
body {
  margin: 0;
  font-family: sans-serif;
  border: 5px dashed #333;
}

.hero {
  height: 100vh;
  width: 100vw;
  background-color: #add8e6; 
  padding: 2em;
}

.hero h1 {
  font-size: 2em;
  margin-bottom: 0.5em;
}

.hero p {
  font-size: 1em;
}
```

:::

Цей CSS гарантує, що головний розділ завжди буде точно розміром вікна перегляду, незалежно від розміру екрану пристрою.

Одиниці `vh` і `vw` також можна використовувати для типографіки, щоб створювати адаптивні розміри тексту.

Увімкніть інтерактивний редактор і спробуйте змінити розмір вікна попереднього перегляду, щоб побачити, як текст масштабуватиметься залежно від розміру вікна перегляду:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css" />
<h1>Responsive Heading</h1>
<p>The h1 heading scales with the viewport width.</p>
```

```css
h1 {
  font-size: 5vw;
}
```

:::

Це встановить розмір шрифту для елементів `h1` на 5% ширини вікна перегляду, що дозволить тексту плавно масштабуватися разом із розміром вікна браузера.

Однією з переваг одиниць `vh` і `vw` є те, що вони реагують на зміни розміру вікна перегляду в реальному часі. Це означає, що якщо користувач змінить розмір вікна браузера, елементи, розмір яких задано цими одиницями, відповідно підлаштуються без необхідності перезавантаження сторінки. Однак важливо використовувати ці одиниці обережно. Наприклад, встановлення розмірів шрифту виключно за допомогою одиниць `vw` може призвести до того, що текст стане занадто маленьким на вузьких екранах або занадто великим на широких.

Ще один момент — на мобільних пристроях висота вікна перегляду може змінюватися, коли з’являється або зникає адресний рядок браузера, що може спричинити несподівані зсуви макету, якщо ви широко використовуєте одиниці `vh`.

Підсумовуючи, одиниці `vh` і `vw` — це потужні інструменти для створення адаптивних макетів і елементів, які підлаштовуються під розмір вікна перегляду. Вони особливо корисні для розділів на весь екран, збереження співвідношень сторін і створення плавно масштабованих дизайнів. Однак їх слід використовувати обдумано і часто в поєднанні з іншими CSS-техніками, щоб забезпечити найкращий користувацький досвід на всіх пристроях.
