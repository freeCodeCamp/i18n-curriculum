# --description--

JavaScript спочатку створювався для запуску у веббраузерах, де його основною роллю було зробити вебсторінки інтерактивними. Через це ранній JavaScript зосереджувався переважно на обробці тексту у формах і маніпуляції з Document Object Model (DOM).

Однак не всі дані в інтернеті — це текст. Файли, зображення та відео — це двійкові дані, які потребують інших механізмів обробки. У браузерах такі типи даних зазвичай обробляють спеціалізовані компоненти браузера, а не сам JavaScript.

Сучасні браузери використовують рушії відтворення та JavaScript-рушії для керування цими завданнями — наприклад, Blink (з V8) у Chrome, WebKit (з JavaScriptCore) у Safari та Gecko (з SpiderMonkey) у Firefox.

Node.js не працює у браузері, тому йому потрібен власний спосіб обробки двійкових даних, особливо при роботі з вхідними та вихідними файлами (I/O) і TCP-потоками, де дані надходять частинами. Саме тут і з’являється модуль `Buffer`.

Модуль `Buffer` у Node.js дозволяє працювати з двійковими даними, такими як файли, зображення або мережеві потоки, безпосередньо. За його допомогою ви можете зберігати та маніпулювати двійковими даними прямо в пам’яті.

Як і модуль `fs`, `Buffer` є одним із базових модулів Node.js, тому його не потрібно встановлювати окремо перед використанням.

Щоб використати його, спочатку імпортуйте модуль за допомогою деструктуризації:

```js
const { Buffer } = require("buffer");
```

Потім викликайте `Buffer` з методами, які він надає. Наприклад, `Buffer.from()` дозволяє створити буфер із рядка, масиву або інших сирих даних.

Ось як створити `Buffer` із рядка та масиву:

```js
// Create a buffer from a string
const myStrBuffer = Buffer.from("freeCodeCamp");
console.log(myStrBuffer); // <Buffer 66 72 65 65 43 6f 64 65 43 61 6d 70>

// Create a buffer from an array of numbers
const myNumBuffer = Buffer.from([
  70, 82, 69, 69, 67, 79, 68, 69, 67, 65, 77, 80,
]);

console.log(myNumBuffer); // <Buffer 46 52 45 45 43 4f 44 45 43 41 4d 50>
```

Хоча деякі методи модуля `Buffer` можна використовувати без попереднього імпорту, інші методи недоступні, якщо ви явно не імпортували `Buffer`. Тому рекомендується завжди імпортувати Buffer, коли ви його використовуєте у своїх проєктах.

Ви можете отримати доступ до окремих елементів `buffer` так само, як до масиву:

```js
console.log(myNumBuffer[0]); // 70
console.log(myStrBuffer[0]); // 102
```

Також можна використати метод `toString()` для буферів, щоб побачити, як вони виглядають насправді:

```js
console.log(myStrBuffer.toString()); // freeCodeCamp
console.log(myNumBuffer.toString()); // FREECODECAMP
```

`Buffer.alloc()` дозволяє створити новий буфер заданого розміру (кількості байтів). Кожен байт у ньому автоматично заповнюється `0`:

```js
const someBuffer = Buffer.alloc(10);
console.log(someBuffer); // <Buffer 00 00 00 00 00 00 00 00 00 00>
```

Ви бачите, що буфер ініціалізується нулями, залежно від розміру, переданого у метод `alloc()`.

Далі можна використати метод `Buffer.write()`, щоб записати у цей буфер:

```js
someBuffer.write("Hello fCC");

console.log(someBuffer); // <Buffer 48 65 6c 6c 6f 20 66 43 43 00>
console.log(someBuffer.toString()); // Hello fCC
```

Якщо записати більше даних, ніж буфер може вмістити, вони будуть обрізані:

```js
someBuffer.write("Hello freeCodeCamp");

console.log(someBuffer); // <Buffer 48 65 6c 6c 6f 20 66 72 65 65>
console.log(someBuffer.toString()); // Hello free
```

Нарешті, можна використати `Buffer.byteLength()`, щоб показати кількість байтів, необхідних для збереження рядка у певному кодуванні:

```js
console.log(Buffer.byteLength("Hello freeCodeCamp")); // 18
```

Інші методи `Buffer` включають:

- `Buffer.isBuffer()`: перевіряє, чи є заданий об’єкт буфером
- `Buffer.compare()`: порівнює два буфери і повертає їх порядок сортування
- `Buffer.concat()`: об’єднує кілька буферів в один
