# --description--

У попередньому уроці ви дізналися, що таке залежності. Тут ви дізнаєтеся більше про них і про те, на що варто звертати увагу при виборі зовнішнього пакета для свого проєкту.

Залежність — це інша частина коду, яка потрібна вашому проєкту для роботи. Зазвичай це зовнішні пакети, які створили й поділилися інші розробники в реєстрі npm.

Використання залежностей економить вам багато часу. Замість того, щоб будувати все самостійно, ви можете встановити пакет, який уже вирішує загальну проблему — наприклад, створення веб-сервера, обробку маршрутизації, отримання даних, керування автентифікацією тощо.

У вашому файлі `package.json` усі пакети, від яких залежить ваш проєкт, перераховані під ключем `dependencies`:

```json
{
  "name": "package-dependencies",
  "version": "1.0.0",
  "description": "a demo on how package dependencies work",
  "main": "index.js",
  "scripts": {
    "test": "echo \"Error: no test specified\" && exit 1"
  },
  "keywords": [
    "Node",
    "NPM"
  ],
  "author": "Kolade Chris",
  "license": "ISC",
  "dependencies": {
    "chalk": "^5.6.2",
    "lodash": "^4.17.21"
  }
}
```

У наведеному вище прикладі файлу `package.json` `chalk` і `lodash` вказані як залежності проєкту. Це означає, що проєкт покладається на ці пакети для роботи. Якщо вони не встановлені — і не вказані у файлі `package.json` — будь-які функції, які їх використовують, перестануть працювати.

Існує ще одна категорія залежностей, яку називають `devDependencies`. Це пакети, які потрібні лише під час розробки або тестування проєкту, а не під час його роботи в продакшені. Загальні приклади:

* Nodemon для автоматичного перезапуску проєкту при оновленнях
    
* Jest або Mocha для тестування
    
* ESLint для лінтингу коду
    

Як і очікувалося, `devDependencies` з’являються в розділі `devDependencies` файлу `package.json`:

```json
{
  "name": "package-dependencies",
  "version": "1.0.0",
  "description": "a demo on how package dependencies work",
  "main": "index.js",
  "scripts": {
    "test": "echo \"Error: no test specified\" && exit 1"
  },
  "keywords": [
    "Node",
    "NPM"
  ],
  "author": "Kolade Chris",
  "license": "ISC",
  "dependencies": {
    "chalk": "^5.6.2",
    "lodash": "^4.17.21"
  },
  "devDependencies": {
    "jest": "^30.2.0",
    "nodemon": "^3.1.10"
  }
}
```

При виборі пакетів для свого проєкту варто враховувати кілька важливих моментів.

Безпека завжди має бути на першому місці. Навіть один небезпечний пакет може створити проблеми для всього проєкту. Також варто звертати увагу на продуктивність, бо великий або погано оптимізований пакет може уповільнити ваш проєкт.

Корисно обирати популярні та добре підтримувані пакети. Популярні пакети зазвичай мають активні спільноти, хорошу документацію та часті оновлення. Один із простих способів перевірити популярність — подивитися на кількість щотижневих скачувань у npm. Наприклад, пакет `chalk` отримує понад 300 000 скачувань щотижня:

<img src="https://cdn.freecodecamp.org/curriculum/lecture-transcripts/what-are-package-dependencies-and-what-are-some-considerations-for-choosing-an-external-package-1.webp" alt="screenshot from npm of chalk package statistics">

Обслуговування також важливе. Добре підтримувані пакети регулярно оновлюються і краще працюють із новими версіями іншого програмного забезпечення.

Нарешті, не ігноруйте документацію. Чітка, детальна документація значно полегшує розуміння, як користуватися пакетом, і заощаджує багато часу при інтеграції його у ваш проєкт.
