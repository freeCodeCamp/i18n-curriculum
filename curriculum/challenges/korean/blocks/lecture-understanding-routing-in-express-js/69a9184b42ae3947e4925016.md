# --description--

Express에서 `express.Router()`는 라우트를 더 작고 모듈화된 단위로 정리하는 데 도움이 되는 강력한 기능입니다. 애플리케이션이 커질수록 특히 유용합니다.

모든 라우트를 하나의 큰 파일에 정의하는 대신, 라우터를 사용해 별도의 파일로 나눌 수 있습니다. 이렇게 하면 코드를 더 읽기 쉽고 유지 관리하며 확장하기 편해집니다.

`express.Router()` 메서드는 새로운 라우터 객체를 만듭니다.

라우터는 미니 Express 애플리케이션처럼 작동합니다. 자체 라우트와 미들웨어를 처리할 수 있지만 완전한 Express 애플리케이션은 아닙니다. 라우터에 라우트를 정의한 후에는 `app.use()`를 사용해 메인 앱에 마운트할 수 있습니다.

즉, 라우터는 관련된 라우트를 함께 그룹화한 다음 나중에 메인 애플리케이션에 통합할 수 있게 해줍니다.

라우터를 사용하면 여러 가지 장점이 있습니다:

**관심사의 분리**

라우트를 별도의 파일에 정의할 수 있어 애플리케이션의 구성과 가독성이 향상됩니다.

**코드 재사용성**  
라우터는 애플리케이션의 여러 부분이나 심지어 다른 프로젝트에서도 재사용할 수 있습니다.

**더 깔끔한 메인 파일**  
메인 `app.js` 또는 `server.js` 파일에 많은 라우트 정의를 채우는 대신, 라우터로 옮겨 메인 파일은 구성과 설정에 집중할 수 있습니다.

`express.Router()`를 사용해 `/users` 경로에 대한 라우트를 정의하는 간단한 예제를 살펴보겠습니다.

먼저, `app.js` 파일입니다:

```javascript
const express = require("express");
const app = express();
const port = 3000;

// Importing the user routes
const userRoutes = require("./userRoutes");

// Using the userRoutes router for /users path
app.use("/users", userRoutes);

app.listen(port, () => {
  console.log(`Server running at http://localhost:${port}`);
});
```

그리고 `userRoutes.js`입니다:

```javascript
const express = require("express");
const router = express.Router();

// GET request for /users
router.get("/", (req, res) => {
  res.send("List of users");
});

// POST request for /users
router.post("/", (req, res) => {
  res.send("Creating a new user");
});

// GET request for /users/:id
router.get("/:id", (req, res) => {
  const userId = req.params.id;
  res.send(`User details for user with id: ${userId}`);
});

module.exports = router;
```

이제 모든 것이 어떻게 작동하는지 자세히 살펴보겠습니다.

`app.js`에서 `express`을 가져와 Express 애플리케이션 인스턴스를 만들고 `app`이라는 변수에 할당합니다.

다음으로 `userRoutes`에서 `userRoutes.js` 라우터를 가져와 `app.use("/users", userRoutes)`를 사용해 라우터를 마운트합니다. 이는 요청이 `/users`로 시작하면 `userRoutes` 라우터로 전달하라는 Express에 알리는 것입니다.

그런 다음 `userRoutes.js`에서 `express.Router()`를 사용해 새 라우터를 만들고 `router.get()`, `router.post()`, `router.get("/:id")`를 사용해 라우트를 정의합니다.

여기서 정의한 라우트는 라우터가 마운트된 `/users`에 상대적임을 주의하세요. 따라서:

*   `router.get("/")` → `GET /users`: 사용자 목록을 응답합니다
    
*   `router.post("/")` → `POST /users`: 새 사용자 생성에 관한 메시지를 응답합니다
    
*   `router.get("/:id")` → `GET /users/:id`: 주어진 사용자 ID에 대한 사용자 세부 정보를 응답합니다
    

`express.Router()`를 사용하면 메인 파일이 깔끔하고 구성에 집중할 수 있습니다. 모든 사용자 관련 라우트는 `userRoutes.js`에 함께 그룹화됩니다.

애플리케이션이 커지면 `/products`, `/orders`, 또는 `/auth` 같은 다른 섹션을 위한 추가 라우터를 만들 수 있습니다.

각 섹션은 자체 파일에 정리되어 있어 애플리케이션을 더 쉽게 유지 관리하고 확장할 수 있습니다.
