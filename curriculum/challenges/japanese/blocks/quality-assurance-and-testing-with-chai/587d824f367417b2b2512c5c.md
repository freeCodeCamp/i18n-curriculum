# --description--

このプロジェクトは、<a href="https://github.com/freeCodeCamp/boilerplate-mochachai/" target="_blank" rel="noopener noreferrer nofollow">GitHub</a>からクローンした以下のスタータープロジェクトを基に構築されています。

次のチャレンジでは、ヘッドレスブラウザーを使ってページ上の人間の操作をシミュレーションします。

ヘッドレスブラウザーとは、GUIを持たないウェブブラウザーのことです。通常のブラウザーと同じようにHTML、CSS、JavaScriptをレンダリングし解釈できるため、ウェブページのテストに特に役立ちます。

これからのチャレンジでは、追加のバイナリをインストールする必要がない軽量なヘッドレスブラウザーであるZombie.jsを使います。ただし、他にもより強力なヘッドレスブラウザーの選択肢は多数あります。

Mochaでは、実際のテストが始まる前にコードを実行することができます。これは、テストの残りで使うデータベースへのエントリー追加などに便利です。

ヘッドレスブラウザーを使う場合、テストを実行する前にテスト対象のページを**訪問**する必要があります。

`suiteSetup` フックは、テストスイートの最初に一度だけ実行されます。

他にも、各テストの前後やテストスイートの最後にコードを実行するフックがいくつかあります。詳細はMochaのドキュメントを参照してください。

# --instructions--

`tests/2_functional-tests.js` 内で、`Browser` 宣言の直後に、変数の`site` プロパティにプロジェクトのURLを追加してください。

```js
Browser.site = 'http://0.0.0.0:3000'; // Your URL here
```

次に、`'Functional Tests with Zombie.js'` スイートのルートレベルで、以下のコードを使って`Browser` オブジェクトの新しいインスタンスを作成してください。

```js
const browser = new Browser();
```

そして、`suiteSetup` フックを使って、`browser` を`/` ルートに案内するコードを以下のように書いてください。**注意**：`done` は`browser.visit` にコールバックとして渡されているため、自分で呼び出してはいけません。

```js
suiteSetup(function(done) {
  return browser.visit('/', done);
});
```
