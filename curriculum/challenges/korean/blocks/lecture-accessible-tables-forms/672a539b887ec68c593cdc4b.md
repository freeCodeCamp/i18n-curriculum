# --interactive--

표를 보면 우리는 즉시 데이터와 헤더 사이의 시각적 연관성을 만들기 시작합니다.

예를 들어, 애완동물을 나타내는 표가 있다고 가정해 보겠습니다. 개 두 마리와 고양이 두 마리가 있고, 표는 그들의 이름과 나이를 보여줍니다. 시각이 정상인 사람은 이 표에서 관계를 이해할 수 있지만, 화면 읽기 프로그램을 사용하는 사람들은 값과 헤더 사이의 연결을 이해하기가 훨씬 어렵습니다.

웹 개발자로서 이러한 연관성을 만들고 HTML 마크업을 화면 읽기 프로그램 사용자도 쉽게 해석할 수 있도록 구조화하는 책임이 있습니다.

그럼 모두가 이해할 수 있는 접근 가능한 표를 만드는 방법을 살펴보겠습니다. 첫 번째 모범 사례는 표 캡션을 사용하는 것입니다. `caption` 요소를 사용하면 표의 캡션(또는 제목)을 작성할 수 있어, 특히 보조 기술을 사용하는 사용자가 표의 목적과 내용을 빠르게 이해할 수 있습니다. `caption` 요소는 `table` 요소의 여는 태그 바로 다음에 배치해야 합니다. 이렇게 하면 화면 읽기 프로그램과 다른 보조 기술이 내용을 읽기 전에 캡션을 알림으로써 더 많은 맥락을 제공할 수 있습니다.

```html
<table>
  <caption>Our Pets</caption>
  <!-- Table Rows and Columns -->
</table>
```

이제 행과 열 헤더에 대해 이야기해 보겠습니다. 헤더는 일반적으로 행이나 열의 시작 부분에 있는 특별한 셀로, 해당 행이나 열에 저장된 데이터 유형을 설명합니다. 행 또는 열 헤더는 표 헤더 요소인 `th`로 정의할 수 있습니다.

예를 들어, 아래 코드는 두 마리 애완동물에 대한 표를 만듭니다. 각 행에는 행 헤더(애완동물 이름)가 있고, 각 열에는 열 헤더가 있어 열의 데이터가 무엇을 나타내는지 설명합니다(나이와 종류).

:::interactive_editor

```html
<table>
  <caption>Our Pets</caption>
  <thead>
    <tr>
      <!-- Column Headers -->
      <th>Name</th>
      <th>Age</th>
      <th>Type</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <th>Nora</th> <!-- Row Header -->
      <td>5</td>
      <td>Dog</td>
    </tr>
    <tr>
      <th>Gino</th> <!-- Row Header -->
      <td>2</td>
      <td>Cat</td>
    </tr>
  </tbody>
</table>
```

:::

위 코드에서 `caption` 요소의 여는 태그 바로 다음에 `table` 요소가 있는 것을 확인하세요. 그리고 표 헤드(`thead`) 요소 안에는 열 헤더(`Name`, `Age`, `Type`)가 있습니다. 두 번째와 세 번째 행에서는 표 본문(`tbody`) 요소 안에 각 애완동물의 데이터가 있습니다. 애완동물 이름은 표 헤더 요소(`th`) 안에 있으므로 행 헤더입니다.

데이터 셀과 해당 헤더를 연결하는 것도 화면 읽기 프로그램에 매우 중요합니다. `scope` 속성은 헤더가 행 헤더인지 열 헤더인지 결정합니다.
화면 읽기 프로그램은 표 구조에서 이를 올바르게 추측할 수 있지만, 명확성을 위해 `scope`을 명시적으로 표시하는 것이 일반적으로 권장됩니다.

`scope` 속성에는 네 가지 가능한 값이 있습니다. 가장 자주 사용하는 두 가지는 열을 나타내는 `col`와 행을 나타내는 `row`입니다.
아래 코드에서는 열과 행 헤더에 `scope` 속성을 추가한 것을 볼 수 있습니다. 세 개의 열 헤더(`Name`, `Age`, `Type`)는 `scope`이 `col`, 즉 열입니다.

두 개의 행 헤더(`Nora`, `Gino`)는 `scope`가 `row`입니다.

:::interactive_editor

```html
<table>
  <caption>Our Pets</caption>
  <thead>
    <tr>
      <!-- Now they have scope -->
      <th scope="col">Name</th>
      <th scope="col">Age</th>
      <th scope="col">Type</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <th scope="row">Nora</th>
      <td>5</td>
      <td>Dog</td>
    </tr>
    <tr>
      <th scope="row">Gino</th>
      <td>2</td>
      <td>Cat</td>
    </tr>
  </tbody>
</table>
```

:::

열 또는 행 헤더가 여러 셀에 걸쳐 있으면 `scope`도 각 셀에 개별적으로 적용됩니다. 다음은 그 예입니다:

:::interactive_editor

```html
<table>
  <tbody>
    <tr>
      <td></td>
      <th scope="col">Name</th>
      <th scope="col">Age</th>
    </tr>
    <tr>
      <th rowspan="2" scope="row">Dogs</th>
      <th scope="row">Nora</th>
      <td>5</td>
    </tr>
    <tr>
      <th scope="row">Gino</th>
      <td>2</td>
    </tr>
    <tr>
      <th rowspan="2" scope="row">Cats</th>
      <th scope="row">Lulu</th>
      <td>10</td>
    </tr>
    <tr>
      <th scope="row">Elizabeth</th>
      <td>6</td>
    </tr>
  </tbody>
</table>
```

:::

이 표에서 `Nora`의 나이(`5`) 셀은 하나의 열 헤더(`Age`)와 두 개의 행 헤더(`Dogs`, `Nora`)를 가집니다. `Gino`의 나이(`2`)도 하나의 열 헤더(`Age`)와 두 개의 행 헤더(`Dogs`, `Gino`)를 가집니다.

하지만 일부 화면 읽기 프로그램은 복잡한 구조의 표를 해석하지 못할 수 있으므로, 여러 셀에 걸친 행과 열 헤더를 피하기 위해 표를 최대한 평평하게 만드는 것이 좋습니다.

목표는 화면 읽기 프로그램이 복잡한 표 구조를 처리할 수 없더라도 사용자가 이 정보를 접근할 수 있도록 하는 것입니다.

이제 셀 너비에 대해 이야기하겠습니다. 고정 값을 사용하는 것은 피하는 것이 좋습니다. 대신 백분율과 같은 상대 값을 사용하세요. 또한 셀 높이 정의도 피하는 것이 좋습니다. 이렇게 하면 사용자가 필요에 따라 텍스트 크기를 조절할 수 있습니다.

마지막으로, 가능한 한 브라우저가 표 너비를 결정하도록 하여 가로 스크롤 필요성을 줄이세요.

HTML 표는 구조화된 데이터를 접근 가능하고 이해하기 쉬운 형식으로 제시하는 데 필수적입니다. 이러한 접근성 지침을 따르면 모두가 쉽게 이해할 수 있는 표를 만들 수 있습니다.
