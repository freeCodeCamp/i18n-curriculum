# --description--

In Express wird das `res` (Antwort-)Objekt verwendet, um nach der Verarbeitung eingehender Anfragen Antworten an den Client zurückzusenden. Express stellt mehrere eingebaute Methoden bereit, um verschiedene Arten von Antworten zu senden. Zum Beispiel können Sie einfachen Text, JSON-Daten senden, einen Benutzer zu einer anderen URL umleiten oder eine Vorlage rendern.

Eine der am häufigsten verwendeten Antwortmethoden ist `res.send()`:

```javascript
app.get("/text", (req, res) => {
  res.send("Hello, World!");
});
```

Die `res.send()`-Methode sendet eine Antwort zurück an den Client. Sie kann einfachen Text, HTML, JSON oder sogar Binärdaten senden. In diesem Beispiel antwortet der Server mit dem Text `/text`, wenn ein Client `Hello, World!` besucht.

Eine weitere häufig verwendete Antwortmethode ist `res.json()`.

Diese Methode sendet eine JSON-Antwort an den Client. Sie ist besonders nützlich beim Erstellen von APIs, bei denen Sie oft strukturierte Daten wie Objekte oder Arrays zurückgeben müssen. Express wandelt ein JavaScript-Objekt automatisch in einen JSON-formatierten String um:

```javascript
app.get("/user", (req, res) => {
  res.json({ name: "John Doe", age: 30 });
});
```

Wenn ein Client eine Anfrage an `/user` sendet, antwortet der Server mit Folgendem:

```json
{
  "name": "John Doe",
  "age": 30
}
```

Wenn Sie den HTTP-Statuscode einer Antwort steuern müssen, können Sie `res.status()` verwenden. Diese Methode setzt den Statuscode und wird typischerweise mit einer anderen Antwortmethode wie `res.send()` oder `res.json()` verkettet:

```javascript
app.get("/notfound", (req, res) => {
  res.status(404).send("Not Found");
});
```

Hier setzt der Server den Statuscode auf `404` und sendet die Nachricht `Not Found` zurück an den Client.

Wenn Sie den Benutzer zu einer anderen URL schicken möchten, können Sie `res.redirect()` verwenden:

```javascript
app.get("/oldpage", (req, res) => {
  res.redirect("/newpage");
});
```

Wenn ein Client `/oldpage` besucht, wird er automatisch zu `/newpage` umgeleitet.

Schließlich wird `res.render()` verwendet, wenn Sie mit Template-Engines wie Pug, EJS oder Handlebars arbeiten. Es rendert eine View-Vorlage und sendet das resultierende HTML an den Client:

```javascript
app.get("/profile", (req, res) => {
  res.render("profile", { username: "john_doe" });
});
```

In diesem Beispiel rendert der Server eine `profile`-View und übergibt ihr eine `username`-Variable, die dann im HTML angezeigt werden kann.

Diese Antwortmethoden ermöglichen es Ihnen, verschiedene Arten von Daten basierend auf der Art der Anfrage zurückzugeben. Egal, ob Sie Text, JSON-Daten senden oder den Client umleiten, Express bietet ein flexibles Set an Werkzeugen, um dies zu handhaben.
