# --description--

يمتلك محرك JavaScript القدرة على قراءة شفرتك وفهمها وتنفيذها. يعمل كمحول يأخذ شفرتك ويحوّلها إلى تعليمات يمكن للكمبيوتر فهمها وتنفيذها.

أحد أشهر محركات JavaScript هو V8، الذي طورته Google، ويُستخدم في Chrome وNode.js. يعمل محرك JavaScript بعدة خطوات. أولاً، يُحلل شفرتك، يقرأها سطرًا بسطر ليتأكد من عدم وجود أخطاء في شفرة JavaScript. ثم يحول هذه الشفرة إلى bytecode، وهو نسخة أبسط ومتوسطة من شفرتك يسهل على الكمبيوتر فهمها وتنفيذها. وأخيرًا، يشغّل هذا الـ bytecode لتنفيذ تعليمات برنامجك. إليك مثالًا على شفرة JavaScript:

```js
const greeting = "Hello, World!";
console.log(greeting);
```

عندما تشغّل هذه الشفرة، يبدأ محرك JavaScript بتحليلها أولًا للتحقق من وجود أي خطأ نحوي. التحليل يعني أن المحرك يقرأ الشفرة ويفككها إلى بنية يمكنه فهمها، مع التحقق من الأخطاء أثناء ذلك.

بعدها، يُجمّع الشفرة إلى صيغة وسيطة (غالبًا bytecode أو شفرة آلة، حسب المحرك). التجميع هو عملية تحويل الشفرة القابلة للقراءة البشرية إلى صيغة أكثر كفاءة يمكن للكمبيوتر تنفيذها بسرعة أكبر.

أخيرًا، ينفذ المحرك الشفرة، ويطبع `Hello, World!` في وحدة التحكم.

الآن، لنتحدث عن بيئة تشغيل JavaScript. بيئة التشغيل هي البيئة التي تُنفذ فيها شفرة JavaScript الخاصة بك. تشمل محرك JavaScript (مثل V8 في Chrome أو SpiderMonkey في Firefox)، الذي يعالج وينفذ الشفرة، بالإضافة إلى ميزات إضافية يوفرها بيئة العمل (مثل متصفح الويب أو Node.js، والتي ستتعلم عنها أكثر في الدروس القادمة).

بينما تتعامل لغة JavaScript الأساسية مع أشياء مثل المتغيرات والحلقات والدوال، توفر بيئة التشغيل أدوات إضافية تسمح لـ JavaScript بالتفاعل مع أشياء خارج اللغة نفسها، مثل DOM (لصفحات الويب) أو Fetch API (لإجراء طلبات الشبكة).

باختصار، بيئة التشغيل هي التي تسمح لـ JavaScript بأداء أكثر من مجرد مهام برمجية أساسية – مثل التفاعل مع صفحات الويب أو التعامل مع الإجراءات الزمنية – من خلال توفير هذه الميزات الإضافية خارج اللغة نفسها.

لا تحتاج إلى معرفة كل تفاصيل المحركات وبيئات التشغيل لكتابة JavaScript، لكن فهمًا أساسيًا يمكن أن يساعدك على كتابة شفرة أكثر كفاءة ومعالجة الأخطاء بشكل أفضل.
