# --interactive--

Область видимості в програмуванні означає видимість і доступність змінних у різних частинах вашого коду. Вона визначає, де змінні можна отримати або змінити. В JavaScript розуміння області видимості є ключовим для написання чистого, ефективного та безпомилкового коду. Існує три основні типи області видимості: глобальна область видимості, локальна область видимості та блочна область видимості.

Глобальна область видимості — це найзовнішніша область видимості в програмі на JavaScript. Змінні, оголошені в глобальній області видимості, доступні з будь-якого місця вашого коду, включно з функціями та блоками. Такі змінні часто називають глобальними змінними. Хоча глобальні змінні можуть бути зручними, їх слід використовувати помірно, оскільки вони можуть призводити до конфліктів імен і ускладнювати підтримку коду. Ось приклад глобальної змінної:

:::interactive_editor

```js
let globalVar = "I'm a global variable";

function printGlobalVar() {
    console.log(globalVar);
}

printGlobalVar(); // "I'm a global variable"
```

:::

У цьому прикладі `globalVar` оголошена в глобальній області видимості і може бути доступна всередині функції `printGlobalVar`.

Локальна область видимості, навпаки, означає змінні, які доступні лише всередині функції. Ось приклад локальної області видимості:

:::interactive_editor

```js
function greet() {
    let message = "Hello, local scope!";
    console.log(message);
}

greet(); // "Hello, local scope!"
// console.log(message); // This will throw an error
```

:::

У цьому коді `message` — це локальна змінна всередині функції `greet`. Вона може використовуватися всередині функції, але спроба отримати до неї доступ поза межами функції призведе до помилки.

Блочна область видимості — це концепція, введена з ключовими словами `let` та `const` у ES6. Блок — це будь-який розділ коду в фігурних дужках, `{}`, наприклад у інструкціях `if`, циклах `for` або циклах `while`. Концепцію циклів буде викладено в одному з наступних уроків.

Змінні, оголошені за допомогою `let` або `const` всередині блоку, доступні лише в межах цього блоку. Ось приклад блочної області видимості:

```js
if (true) {
    let blockVar = "I'm in a block";
    console.log(blockVar); // "I'm in a block"
}
console.log(blockVar); // This will throw an error
```

У цьому прикладі `blockVar` доступна лише всередині блоку `if`. Спроба отримати доступ поза межами блоку призведе до помилки. Розуміння цих різних типів області видимості є необхідним для керування доступністю змінних і уникнення небажаних побічних ефектів у вашому коді.

Глобальні змінні слід використовувати помірно, оскільки вони можуть призводити до конфліктів імен і ускладнювати підтримку коду. Локальні змінні допомагають ізолювати різні частини вашого коду, що особливо корисно у великих програмах. Блочна область видимості з `let` та `const` дає ще більш точний контроль над доступністю змінних, допомагаючи запобігти помилкам і роблячи код більш передбачуваним. Оволодіння цими базовими поняттями глобальної, локальної та блочної області видимості забезпечить міцну основу для розуміння більш складних тем.
