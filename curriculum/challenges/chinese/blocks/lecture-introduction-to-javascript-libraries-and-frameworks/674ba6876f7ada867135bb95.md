# --description--

在之前的课程中，你学习了如何在 JavaScript 中使用导入和导出。在本课中，我们将了解如何在 React 中导入和导出组件。

导出使组件可供其他文件使用。导入允许文件使用在其他地方导出的组件。在 React 项目中，这就是组件被重用和组合以构建用户界面的方式。

在本课中，你将学习如何使用默认导出和命名导出导出 React 组件，以及如何在需要的地方导入它们。

在这个示例中，我们有一个属于名为 `Cat.jsx` 的文件的 `Cat` 组件。对于文件扩展名，我们使用 `.jsx` 扩展名，因为该文件主要处理 JSX。

这个 `Cat` 组件 `return` 一个包含标题和名为 Mr. Whiskers 的猫的图像的 JSX 标记：

```jsx
function Cat() {
  return (
    <div className="card">
      <h2>Mr. Whiskers</h2>
      <img
        src="https://cdn.freecodecamp.org/curriculum/cat-photo-app/running-cats.jpg"
        alt="Tuxedo cats running on dirt ground."
      />
    </div>
  );
}
```

如果我们想在另一个文件中使用我们的 `Cat` 组件，首先需要像这样导出它：

```jsx
function Cat() {
  return (
    <div className="card">
      <h2>Mr. Whiskers</h2>
      <img
        src="https://cdn.freecodecamp.org/curriculum/cat-photo-app/running-cats.jpg"
        alt="Tuxedo cats running on dirt ground."
      />
    </div>
  );
}

export default Cat;
```

我们使用 `default` 关键字，因为这将是该模块的默认导出。你也可以选择在组件定义的同一行导出该组件，如下所示：

```jsx
export default function Cat() {
  return (
    <div className="card">
      <h2>Mr. Whiskers</h2>
      <img
        src="https://cdn.freecodecamp.org/curriculum/cat-photo-app/running-cats.jpg"
        alt="Tuxedo cats running on dirt ground."
      />
    </div>
  );
}
```

你可以选择在其他父组件的文件中导入子组件，或者在根组件的文件中导入它们。对于此示例，我们将在根组件的文件中导入 `Cat` 组件。

每个 React 项目都会有一个顶层组件，通常称为 `App.jsx`：

```jsx
export default function App() {
  return (
    // return component here
  );
}
```

此文件通常位于你的项目的 `src` 目录中。你将在后续课程中了解更多关于常见项目版型的内容。

要在根 `App` 组件内使用 `Cat` 组件，你需要这样导入它：

```jsx
import Cat from "./Cat";

export default function App() {
  return (
    // return component here
  );
}
```

现在，你可以像这样在 `App` 组件内 `return` `Cat` 组件：

```jsx
import Cat from "./Cat";

export default function App() {
  return (
    <Cat />
  );
}
```

当一个文件负责导出单个主要组件时，这种方法效果很好，保持了导入语法的简洁和易读。请记住，一个文件只能有一个默认导出，这使得它非常适合主要包含单个组件的文件。


既然你知道如何使用默认导出，现在让我们来看命名导出。命名导出允许一个文件共享多个组件或函数。与默认导出不同，这些必须使用它们导出时的确切名称导入（除非你使用 `as` 术语重命名它们）。 

当一个文件作为组件库时，这非常有用。例如，下面是一个包含两个组件 Cat 和 Dog 的文件：

```jsx
// Animals.jsx
export function Cat() {
  return <h2>Mr. Whiskers</h2>;
}

export function Dog() {
  return <h2>Fido</h2>;
}
```

我们在每个组件上单独使用 `export` 关键字。要在另一个文件中使用这些组件，我们使用花括弧 `import` 它们：


```jsx
import { Cat, Dog } from "./Animals";

export default function App() {
  return (
    <div>
      <Cat />
      <Dog />
    </div>
  );
}
```

`{ Cat, Dog }` 语法告诉 JavaScript 从 `Animals.jsx` 文件中导入特定的命名导出。花括弧中的名称必须与导出的名称完全匹配。如果你需要避免命名冲突，或者你希望在当前文件中使用不同的名称，可以使用 as 关键字在导入时重命名组件。

```jsx
import { Cat as Kitty } from "./Animals";

export default function App() {
  return <Kitty />;
}
```

最后，一个文件可以有一个默认导出和多个命名导出。

```jsx
// Animals.jsx
export default function Cat() {
  return <h2>Mr. Whiskers</h2>;
}

export function Dog() {
  return <h2>Fido</h2>;
}
```

当导入混合导出时，默认导出首先出现（无花括弧），随后是命名导出（在花括弧内）：

```jsx
// App.jsx
import Cat, { Dog } from "./Animals";

export default function App() {
  return (
    <div>
      <Cat />
      <Dog />
    </div>
  );
}
```

在此示例中，`Cat` 是默认导出，因此它不使用括弧状的花括弧。`Dog` 是命名导出，因此它需要使用花括弧。

通过理解 `default` 和命名导出如何工作，你可以在多个文件中组织 React 组件，并在需要时重用它们。随着你构建更大的应用，选择正确的导出方式将有助于保持你的代码清晰且易于维护，并使与他人创建的组件协作更加轻松。
