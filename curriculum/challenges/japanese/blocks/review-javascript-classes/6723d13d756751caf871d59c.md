## クラスの基本操作

- **定義**: JavaScriptのクラスは、オブジェクトを作成するための設計図を定義し、データをカプセル化するために使われます。クラスにはコンストラクタが含まれており、これはクラスから新しいオブジェクトが作成されると自動的に呼び出される特別なメソッドです。オブジェクトのプロパティを初期化するために使われます。ここで`this`キーワードはクラスの現在のインスタンスを指します。コンストラクタの下にはメソッドと呼ばれるものがあります。メソッドはクラス内で定義された関数で、クラスのデータや状態に対して操作や処理を行います。インスタンスが実行できる振る舞いを定義するために使われます。

:::interactive_editor

```js
class Dog {
  constructor(name) {
    this.name = name;
  }

  bark() {
    console.log(`${this.name} says woof!`);
  }
}

const dog = new Dog("Gino");
console.log(dog.name);  // Gino
```

:::

クラスの新しいインスタンスを作成するには、`new`キーワードの後にクラス名を続けて使います。

```js
const dog = new Dog("Gino");
```

クラスはクラス式としても作成できます。これはクラスが匿名で変数に代入される形です。

:::interactive_editor

```js
const Dog = class {
  constructor(name) {
    this.name = name;
  }

  bark() {
    console.log(`${this.name} says woof!`);
  }
};

const dog = new Dog("Gino");
console.log(dog.name);  // Gino
dog.bark();  // Gino says woof!
```

:::

## クラスの継承

- **定義**: プログラミングにおいて継承は、親クラスからプロパティやメソッドを受け継ぐクラスを定義できる仕組みです。これによりコードの再利用が促進され、クラス間の階層的な関係が確立されます。親クラスは他のクラスの設計図のような役割を果たし、他のクラスに継承されるプロパティやメソッドを定義します。子クラスは別のクラスのプロパティやメソッドを継承するクラスです。子クラスは親クラスの機能を拡張して、新しいプロパティやメソッドを追加することもできます。JavaScriptでは`extends`キーワードを使って継承を実装します。このキーワードはあるクラスが別のクラスの子クラスであることを示します。

:::interactive_editor

```js
class Vehicle {
  constructor(brand, year) {
    this.brand = brand;
    this.year = year;
  }
}

class Car extends Vehicle {
  honk() {
    console.log("Honk! Honk!");
  }
}

const myCar = new Car("freeCodeCamp Motors", 2019);
console.log(myCar.brand);  // freeCodeCamp Motors
console.log(myCar.year);  // 2019
myCar.honk();  // Honk! Honk!
```

:::

親クラスのメソッド、コンストラクタ、フィールドにアクセスするには`super`キーワードを使います。

:::interactive_editor

```js
class Vehicle {
  constructor(brand, year) {
    this.brand = brand;
    this.year = year;
  }
}

class Car extends Vehicle {
  constructor(brand, year, numDoors) {
    super(brand, year);
    this.numDoors = numDoors;
  }
}

const myCar = new Car("freeCodeCamp Motors", 2019, 4);
console.log(myCar.brand);  // freeCodeCamp Motors
console.log(myCar.year);  // 2019
console.log(myCar.numDoors);  // 4
```

:::

## 静的メソッドと静的プロパティの操作

- **静的メソッド**: これらのメソッドは、特定のオブジェクトの状態にアクセスする必要がないユーティリティ関数としてよく使われます。関連する機能をカプセル化するためにクラス内で定義されます。

:::interactive_editor

```js
class Movie {
  constructor(title, rating) {
    this.title = title;
    this.rating = rating;
  }

  static compareMovies(movieA, movieB) {
    if (movieA.rating > movieB.rating) {
      console.log(`${movieA.title} has a higher rating.`);
    } else if (movieA.rating < movieB.rating) {
      console.log(`${movieB.title} has a higher rating.`);
    } else {
      console.log("These movies have the same rating.");
    }
  }
}

let movieA = new Movie("Movie A", 80);
let movieB = new Movie("Movie B", 45);

Movie.compareMovies(movieA, movieB);  // Movie A has a higher rating.
```

:::

静的メソッドは「ファクトリ」メソッドの実装にも役立ちます。ファクトリメソッドは、コンストラクタに加えて特定の条件に基づいてオブジェクトを作成するために定義するメソッドです。

```js
class Pizza {
  constructor(type, price) {
    this.type = type;
    this.price = price;
  }

  static createMargherita() {
    return new this("Margherita", 6.99);
  }
}

let myPizza = Pizza.createMargherita();
console.log(myPizza);  // Pizza { type: "Margherita", price: 6.99 }
console.log(myPizza.type);  // Margherita
```

- **静的プロパティ**: これらのプロパティはクラス自体に関連付けられた値や属性を定義するために使われます。静的プロパティはクラスのすべてのインスタンスで共有され、インスタンスを作成しなくてもアクセスできます。

:::interactive_editor

```ts
class Car {
  // Static property
  static numberOfWheels = 4;

  constructor(make, model) {
    this.make = make;
    this.model = model;
  }

  // Instance method
  getCarInfo() {
    return `${this.make} ${this.model}`;
  }

  // Static method
  static getNumberOfWheels() {
    return Car.numberOfWheels;
  }
}

// Accessing static property directly from the class
console.log(Car.numberOfWheels);  // 4
```

:::
