# --description--

Ora che hai capito il sistema di moduli CommonJS, diamo un’occhiata al sistema dei moduli ES.

I moduli ES, o moduli ECMAScript, sono il formato ufficiale e standardizzato per i moduli in JavaScript definito dalla specifica ECMAScript. Usano le istruzioni `import` e `export` e in Node.js vengono caricati in modo asincrono, il che è più efficiente.

Node.js tratta i file come moduli ES nei seguenti casi:

* File con estensione `.mjs`.
    
* E file con estensione `.js` quando il file `package.json` genitore più vicino contiene un campo `"type"` a livello superiore con valore `"module"`.
    

Qui abbiamo la stessa funzione `multiply` che hai già visto, ma ora useremo il sistema ESM. Per questo, assegniamo l’estensione `.mjs` a entrambi i moduli.

Per esportare la funzione `multiply` dal modulo `math.mjs`, usa la parola chiave `export` seguita dal nome della funzione, variabile o oggetto che vuoi esportare racchiuso tra parentesi graffe:

```javascript
function multiply(a, b) {
  return a * b;
}

export { multiply };
```

Puoi anche esportare la funzione direttamente usando la parola chiave `export` prima della parola chiave `function`. Questo è noto come esportazione nominata. Puoi avere più esportazioni nominate per modulo:

```javascript
export function multiply(a, b) {
  return a * b;
}
```

Poi, per importare questo modulo, usa la parola chiave `import` seguita dalle parentesi graffe. Nota che stiamo usando `import` invece di `require()`. Questa è una delle differenze chiave tra CommonJS e i moduli ES.

All’interno delle parentesi graffe scriviamo il nome della funzione, variabile o oggetto che vogliamo importare. In questo caso è `multiply`. Poi usa la parola chiave `from` e il percorso relativo al modulo. Questo codice verrà eseguito con successo e l’output sarà `42`.

```javascript
import { multiply } from './math.mjs';

console.log(multiply(6, 7)); // 42
```

Nel sistema ES puoi impostare un’esportazione predefinita per modulo. In questo modo non serve usare le parentesi graffe per importarla.

Ecco un esempio in cui impostiamo la funzione `multiply` come esportazione predefinita per l’intero modulo `math.mjs`:

```javascript
export default function multiply(a, b) {
  return a * b;
}
```

Ora non servono più le parentesi graffe, dato che la funzione verrà importata e assegnata per impostazione predefinita:

```javascript
import multiply from './math.mjs';
```

Se non usi le estensioni dei file o il file `package.json` per indicare esplicitamente a Node.js quale tipo di sistema di moduli vuoi usare, esso controllerà il codice sorgente per cercare la sintassi dei moduli ES. Se la sintassi viene trovata, eseguirà il codice come modulo ES; altrimenti, lo eseguirà come modulo CommonJS.

Mentre CommonJS è stato lo standard predefinito per Node.js nei suoi primi tempi, dato che gli sviluppatori adottano sempre più ESM per la sua efficienza e compatibilità con JavaScript moderno, è importante conoscere le loro differenze e come usare entrambi.
