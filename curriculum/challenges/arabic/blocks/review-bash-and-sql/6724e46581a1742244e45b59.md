## تطبيع قواعد البيانات

هذه عملية تنظيم قاعدة بيانات علائقية لتقليل تكرار البيانات وتحسين التكامل.

تشمل فوائدها:

- تقليل البيانات المكررة، مما يوفر مساحة تخزين ويقلل التناقضات.
- فرض تكامل البيانات باستخدام المفاتيح الأساسية والأجنبية.
- جعل قواعد البيانات أسهل في الصيانة والفهم.

### الأشكال الطبيعية

- **1NF (الشكل الطبيعي الأول)**
  - كل خلية تحتوي على قيمة واحدة (ذرية).
  - كل سجل فريد (يفرض بواسطة المفتاح الأساسي).
  - ترتيب الصفوف/الأعمدة غير مهم.
  - مثال: نقل أرقام هواتف متعددة من جدول `students` إلى جدول منفصل `student_phones`.

- **2NF (الشكل الطبيعي الثاني)**
  - يحقق متطلبات 1NF.
  - لا توجد **تبعيات جزئية**: يجب أن تعتمد كل خاصية غير مفتاحية على المفتاح الأساسي المركب بالكامل.
  - مثال: تقسيم جدول `orders` إلى `order_header` و `order_items` لتجنب اعتماد الخصائص على جزء من المفتاح فقط.

- **3NF (الشكل الطبيعي الثالث)**
  - يحقق متطلبات 2NF.
  - لا توجد **تبعيات انتقالية**: لا يمكن أن تعتمد الخصائص غير المفتاحية على خصائص غير مفتاحية أخرى.
  - مثال: نقل `city_postal_code` إلى جدول `cities` بدلاً من تخزينه مع كل طلب.

- **BCNF (شكل بويز-كود الطبيعي)**
  - يحقق متطلبات 3NF.
  - يجب أن يكون كل محدد (الجانب الأيسر من التبعية الوظيفية) مفتاحًا فائقًا.

**نصيحة**: استهدف 3NF في معظم التصاميم لتحقيق توازن جيد بين التكامل والأداء.

## مفاهيم SQL الأساسية

- SQL تعني لغة الاستعلامات الهيكلية، وهي لغة للتواصل مع قواعد البيانات العلائقية.
- **الأوامر الأساسية** → `SELECT`، `INSERT`، `UPDATE`، `DELETE`، `CREATE TABLE`، `ALTER TABLE`، وغيرها.
- **الانضمامات** → تجمع البيانات من جداول متعددة (`INNER JOIN`، `LEFT JOIN`، `RIGHT JOIN`، `FULL JOIN`).

## تشغيل أوامر SQL في Bash

يمكنك تشغيل أوامر SQL مباشرة من سطر الأوامر باستخدام عميل سطر الأوامر `psql` لـ PostgreSQL أو أدوات مماثلة لقواعد بيانات أخرى.

على سبيل المثال، لتشغيل ملف SQL في PostgreSQL:

```bash
psql -U username -d database_name -c "SELECT * FROM students;"
```

يمكنك أيضًا تنفيذ أوامر MySQL مباشرة:

```bash
mysql -u username -p database_name -e "SELECT * FROM students;"
```

### تشغيل SQL من ملف

```bash
# PostgreSQL
psql -U username -d database_name -f script.sql

# MySQL
mysql -u username -p database_name < script.sql
```

### تضمين SQL في سكريبت Bash

```bash
#!/bin/bash
DB_USER="school_admin"
DB_NAME="school"

# Insert student data
psql -U "$DB_USER" -d "$DB_NAME" -c \
"INSERT INTO students (name, age, major) VALUES ('Alice', 20, 'CS');"
```

### استخدام المتغيرات في SQL

```bash
#!/bin/bash
DB_USER="school_admin"
DB_NAME="school"
STUDENT_NAME="Bob"
AGE=21

psql -U "$DB_USER" -d "$DB_NAME" -c \
"INSERT INTO students (name, age) VALUES ('$STUDENT_NAME', $AGE);"
```

**نصيحة**: نظف المتغيرات لتجنب حقن SQL.

## استرجاع واستخدام نتائج استعلام SQL في Bash

عند تشغيل استعلامات SQL عبر `psql`، يمكنك **التقاط** و**معالجة** القيم المرجعة في سكريبتات Bash الخاصة بك.

### التقاط قيمة واحدة

```bash
#!/bin/bash
DB_USER="school_admin"
DB_NAME="school"

# Get total student count
STUDENT_COUNT=$(psql -U "$DB_USER" -d "$DB_NAME" -t -A -c \
"SELECT COUNT(*) FROM students;")

echo "Total students: $STUDENT_COUNT"
```

الإخراج → 42

### استرجاع أعمدة متعددة

```bash
#!/bin/bash
DB_USER="school_admin"
DB_NAME="school"

# Get top 3 students' names and ages
RESULTS=$(psql -U "$DB_USER" -d "$DB_NAME" -t -A -F"," -c \
"SELECT name, age FROM students LIMIT 3;")

echo "Top 3 students:"
echo "$RESULTS"
```

الإخراج

```bash
Alice,20
Bob,21
Charlie,22
```

### التكرار خلال نتائج الاستعلام

```bash
#!/bin/bash
DB_USER="school_admin"
DB_NAME="school"

# Get student names and majors
psql -U "$DB_USER" -d "$DB_NAME" -t -A -F"," -c \
"SELECT name, major FROM students;" | while IFS="," read -r name major
do
    echo "Student: $name | Major: $major"
done
```

شكل الإخراج

```bash
Student: Alice | Major: CS
Student: Bob   | Major: Math
Student: Carol | Major: Physics
```

## حقن SQL

هو ثغرة أمنية في الويب حيث يدخل المهاجمون شفرة SQL خبيثة في حقول الإدخال للتحكم في قاعدة البيانات.

يمكن أن يؤدي ذلك إلى أفعال خطرة مثل:

- تجاوز التوثيق.
- سرقة بيانات حساسة.
- تعديل أو حذف السجلات.

مثال على هجوم حقن SQL:

```sql
SELECT * FROM users WHERE username = '' OR '1'='1' --' AND password = 'anything';
```

سيُرجع هذا الاستعلام جميع المستخدمين لأن الشرط `OR '1'='1'` صحيح دائمًا، مما يسمح للمهاجمين بتجاوز فحوصات تسجيل الدخول.

### منع حقن SQL

1. **استخدام العبارات المحضرة**: تفصل بين شفرة SQL والبيانات، مما يمنع الحقن. إليك مثالًا (Node.js مع pg):

    ```js
    client.query('SELECT * FROM users WHERE username = $1 AND password = $2', [username, password]);
    ```

2. **التحقق من الإدخال**: نظف وتحقق من كل مدخلات المستخدم لضمان توافقها مع الصيغ المتوقعة.

3. **أقل الامتيازات**: استخدم حسابات قاعدة بيانات بأقل الأذونات اللازمة للتطبيق.

**ملاحظة**: لا تمنح أبدًا حقوق المسؤول لحسابات التطبيق.

## مشكلة N+1

تحدث مشكلة N+1 عندما يقوم التطبيق بعمل استعلام واحد لاسترجاع قائمة عناصر (N) ثم يقوم بعمل استعلام إضافي لكل عنصر لاسترجاع بيانات مرتبطة، مما ينتج عنه N+1 استعلامات.

**لماذا هي مشكلة**

- كل استعلام يضيف حملًا على الشبكة والمعالجة.
- الاستعلامات الصغيرة المتعددة أبطأ من استعلام واحد محسن.

### مثال على نمط N+1

```sql
-- 1: Get list of orders
SELECT * FROM orders LIMIT 50;

-- N: For each order, get customer
SELECT * FROM customers WHERE customer_id = ...;
```

**الحل**: استخدم `JOINs` أو عمليات قائمة على المجموعات.

```sql
SELECT 
  orders.order_id,
  orders.product,
  orders.quantity,
  customers.customer_id,
  customers.name,
  customers.email,
  customers.address
FROM orders
JOIN customers 
  ON orders.customer_id = customers.customer_id
WHERE orders.order_id IN (SELECT order_id FROM orders LIMIT 50);
```

ابحث دائمًا عن فرص لدمج البيانات المرتبطة في استعلام واحد.
