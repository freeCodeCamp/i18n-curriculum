## JavaScript del lado del cliente

- **JavaScript del lado del cliente**: Código que se ejecuta en el lado del cliente de una aplicación web, usualmente en un navegador web.
- **Limitaciones de JavaScript del lado del cliente**: Aunque es poderoso, JavaScript del lado del cliente tiene ciertas limitaciones:
  - Acceso muy restringido a archivos locales.
  - No está diseñado para manejar lógica compleja de aplicaciones.
  - Posibles preocupaciones de seguridad (por ejemplo, exponer información sensible como credenciales de base de datos).
- **Razón de las limitaciones**: Estas limitaciones existen porque JavaScript del lado del cliente fue diseñado inicialmente para ejecutarse exclusivamente en navegadores web.

## Entorno del navegador

- **Motor de JavaScript**: Los navegadores web proporcionan el entorno necesario para ejecutar código JavaScript, incluyendo el Motor de JavaScript.
- **Modelo de Objetos del Documento (DOM)**: Los navegadores proporcionan acceso al DOM, permitiéndote acceder a los elementos HTML en tu código.
- **Contexto histórico**: Antes, JavaScript no podía ejecutarse fuera de un navegador, pero ahora puede hacerlo con Node.js.

## ¿Qué es Node.js?

- **Node.js**: Un entorno de ejecución de JavaScript. Definido oficialmente como "un entorno de ejecución de JavaScript de código abierto y multiplataforma."
- **Código abierto**: El código de Node.js está disponible públicamente y es mantenido por una gran comunidad de desarrolladores.
- **Multiplataforma**: Funciona en cualquier sistema operativo, incluyendo Windows, macOS y Linux.
- **Entorno de Ejecución de JavaScript**: Te permite ejecutar código JavaScript fuera de un navegador.

## Casos de uso de Node.js

- **Aplicaciones Comunes**: Node.js se utiliza en una amplia variedad de industrias para:
  - Construyendo servidores web y APIs que manejan solicitudes HTTP.
  - Desarrollando aplicaciones web y móviles.
  - Manejo de datos complejos e interacciones con bases de datos.
- **Adopción en la industria**: Si estás usando o desarrollando un sitio web o una aplicación web que interactúa con una base de datos o maneja datos complejos, hay una alta probabilidad de que uses Node.js detrás de escena.
- **Características clave**: Node.js es eficiente, escalable y cuenta con una gran comunidad de desarrolladores y mantenedores alrededor del mundo.

## Entorno de ejecución de navegador vs. Node

- **Entorno del Navegador**: Diseñado principalmente para desarrollo web front-end, ejecuta JavaScript del lado del cliente.
- **Entorno de Ejecución Node**: Diseñado principalmente para desarrollo web back-end, ejecuta JavaScript del lado del servidor.
- **Diferencias de API**: El entorno determina las APIs disponibles:
  - **Navegador**: Acceso a la API del DOM, pero con restricciones para acceder al sistema de archivos local.
  - **Node**: Acceso a casi todos los recursos del sistema, incluido el sistema de archivos, pero no al DOM.

## Objetos Globales

- **Objeto Global del Navegador (`window`)**: Proporciona acceso a funcionalidades relacionadas con el navegador:
  - Métodos para manipular el DOM.
  - Administrar cookies.
  - Manejo de eventos del navegador.

- **Objeto Global de Node.js (`global`)**: Proporciona acceso a funcionalidades específicas de Node.js:
  - Módulos integrados para trabajar con archivos locales.
  - Capacidades de red.
  - Interactuando con el sistema operativo.

## Control de versiones

- **Versión del Navegador**: No tienes control sobre la versión del entorno del navegador que tus usuarios usarán para visitar tu sitio web.
- **Versión de Node.js**: Puedes elegir la versión de Node.js que quieres usar en tu servidor.

## Importancia de Node.js

- **Desarrollo Full-Stack**: Node.js ha transformado el desarrollo web al permitir que los desarrolladores usen JavaScript para construir tanto el frontend como el backend de aplicaciones full-stack.
- **Eficiencia**: El proceso de desarrollo es más eficiente porque los desarrolladores no necesitan aprender un nuevo lenguaje de programación solo para desarrollar el backend.
- **Impacto**: Node.js es una herramienta poderosa, escalable y versátil que se ha vuelto muy importante para desarrolladores en todo el mundo.

## Ventajas de Node.js

- **Lenguaje único para Full-Stack**: Puedes implementar tanto el front-end como el back-end de una aplicación web usando JavaScript.
  - Reduce la curva de aprendizaje inicial.
  - No es necesario cambiar entre lenguajes de programación.
  - Aumenta la eficiencia y la productividad.

- **Arquitectura no bloqueante y orientada a eventos**: Ideal para desarrollar aplicaciones en tiempo real donde la capacidad de respuesta y la eficiencia son esenciales para crear una buena experiencia de usuario.

- **Hilo único y bucle de eventos**: Puede manejar eficazmente una gran cantidad de solicitudes simultáneas y operaciones de Entrada-Salida.
  - Perfecto para aplicaciones que requieren manejar múltiples solicitudes simultáneamente.

- **Gran Comunidad**: Miles de desarrolladores alrededor del mundo mantienen constantemente Node.js y agregan nuevas características.
  - Muchos recursos de aprendizaje disponibles.
  - Fácil encontrar respuestas a preguntas.

- **npm (Node Package Manager)**: Una herramienta poderosa que te permite instalar y gestionar paquetes y módulos para tus proyectos.
  - Paquetes disponibles para muchos propósitos: manejo de solicitudes HTTP, trabajo con bases de datos, formato de fechas y horas, pruebas de código y más.
  - Reutiliza código escrito, probado y compartido por otros desarrolladores.
  - Hace que el flujo de trabajo sea más rápido y eficiente.

- **Rentable**: Node.js es gratuito y de código abierto.
  - Los paquetes, bibliotecas y herramientas asociadas también son principalmente gratuitas.
  - Factor importante al optimizar el presupuesto.

## Desventajas de Node.js

- **Limitación de un solo hilo**: Solo ejecuta un hilo a la vez, lo que significa que solo puede manejar una operación a la vez.
  - Aunque la arquitectura permite un manejo eficiente de solicitudes concurrentes (simultáneas), las tareas intensivas en CPU pueden bloquear el hilo principal y causar problemas de rendimiento.
  - Ejemplos de tareas intensivas para la CPU: operaciones matemáticas complejas, procesamiento de imágenes y videos, criptografía.
  - Existen formas de superar esto, pero generalmente aumentan la complejidad de la aplicación.

- **Programación Asíncrona**: Node.js se basa en la programación asíncrona.
  - Se inicia una tarea que puede tardar mucho en ejecutarse, pero en lugar de esperar hasta que se complete, el programa principal continúa ejecutándose mientras la tarea asincrónica se ejecuta en paralelo.
  - Cuando la tarea se completa, el programa maneja el resultado.
  - A menudo implica "callbacks", que son funciones que definen qué sucede cuando las operaciones asincrónicas se completan.
  - Puede potencialmente hacer que el código sea más difícil de leer, entender y depurar.

- **Preocupaciones sobre la calidad del paquete**: Al elegir paquetes de npm, algunos pueden no mantenerse constantemente y pueden introducir vulnerabilidades en tu aplicación.
  - Los desarrolladores deben evaluar cada paquete cuidadosamente.
  - Verifica si los paquetes siguen las mejores prácticas de calidad y seguridad.

## Hilos

- **Hilo**: Un camino de ejecución dentro de un proceso, como un programa de computadora que se está ejecutando actualmente.

## NVM (Administrador de Versiones de Node)

- **NVM**: Una herramienta para gestionar múltiples versiones de Node.js en la misma computadora.
- **Propósito**: Te ayuda a cambiar fácilmente entre diferentes versiones de Node.js para distintos proyectos.
- **Recomendación**: Se recomienda instalar NVM antes de Node.js para mayor flexibilidad futura.

## Instalando NVM en macOS/Linux

- **Xcode Command Line Tools (macOS)**: Deben instalarse manualmente antes de ejecutar el script de instalación de NVM en macOS.

```bash
xcode-select --install
```

- **Verifica la instalación de Xcode**:

```bash
xcode-select --version
```

- **Descarga y ejecuta el script de instalación de NVM**: Ejecuta uno de estos comandos:

```bash
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.3/install.sh | bash
```

```bash
wget -qO- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.3/install.sh | bash
```

- **Reinicio del Terminal**: Puede que necesites reiniciar tu terminal cuando el proceso haya terminado para cargar el nuevo comando y configuración.

- **Verifica la instalación de NVM**:

```bash
nvm -v
```

## Instalando Node.js con NVM en macOS/Linux

- **Instala la Última Versión LTS**: Para instalar la última versión de Soporte a Largo Plazo (LTS) de Node.js:

```bash
nvm install --lts
```

- **LTS (Long-Term Support)**: Esta versión generalmente se recomienda porque prioriza la estabilidad, confiabilidad y seguridad sobre características nuevas o experimentales.
  - Probado a fondo y listo para producción.
  - Garantizado para tener períodos de soporte más largos con un enfoque en correcciones de errores y parches de seguridad.

- **Instalar versión específica**: Puedes instalar una versión específica de Node.js especificando el número de versión:

```bash
nvm install <version>
```

Ejemplo:

```bash
nvm install 20
```

- **Cambiar a una Versión**:

```bash
nvm use <version>
```

Ejemplo:

```bash
nvm use 20
```

- **Verificar la versión actual de Node.js**:

```bash
node -v
```

## Instalando NVM en Windows

- **Soporte oficial de NVM**: Las versiones oficiales de NVM solo soportan Windows en casos específicos.
  - Si tienes Windows Subsystem for Linux (WSL), puedes seguir la guía de macOS/Linux arriba.

- **nvm-windows**: Una utilidad de gestión de versiones de Node.js para Windows.
  - Proyecto completamente separado del proyecto oficial NVM.
  - Mismo propósito: ayudarte a trabajar con múltiples versiones de Node.js en el mismo dispositivo.

- **Antes de instalar**: Se recomienda desinstalar cualquier instalación previa de Node antes de instalar NVM para Windows.

- **Pasos de instalación**:
  - Ve a la sección Releases del repositorio oficial: `https://github.com/coreybutler/nvm-windows/releases`.
  - Descarga el instalador más reciente.
  - Sigue las instrucciones.

- **Reinicio del Terminal**: Puede que necesites reiniciar tu terminal si el comando `nvm` no está disponible de inmediato.

- **Verificar Instalación**:

```powershell
nvm -v
```

## Comandos básicos de NVM y Node.js

- **Nota sobre los comandos**: Todos los comandos funcionan para NVM instalado en macOS/Linux, incluyendo WSL. Pueden funcionar de manera diferente o no funcionar en absoluto para `nvm-windows`. Se proporcionan comandos alternativos para `nvm-windows` cuando es posible.

### Política de Ejecución de PowerShell (Windows)

- **Política de Ejecución Predeterminada**: La política de ejecución predeterminada de PowerShell es `Restricted`, lo que impide que los scripts se ejecuten como medida de seguridad.

- **Ejecuta PowerShell como Administrador**: Haz clic derecho en PowerShell, luego haz clic en "Run as Administrator".

- **Verificar la Política de Ejecución Actual**:

```powershell
Get-ExecutionPolicy
```

- **Establece la Política de Ejecución en RemoteSigned**: Permite que los scripts locales y `npm` se ejecuten:

```powershell
Set-ExecutionPolicy RemoteSigned
```

- **Después de configurar la política**: Cierra la ventana de administrador de PowerShell y abre una ventana de PowerShell sin privilegios de administrador.

### Lista Todas las Versiones Instaladas

- **Listar Versiones Instaladas**: Para listar todas las versiones instaladas de Node.js y verificar cuál se está usando actualmente:

```bash
nvm ls
```

### Establecer la versión predeterminada de Node.js (NVM solo en macOS/Linux)

- **Establecer versión predeterminada**: Establece la versión predeterminada de Node.js para nuevas sesiones de terminal:

```bash
nvm alias default <version>
```

Ejemplo:

```bash
nvm alias default 22.20.0
```

- **Crear Alias Personalizados**: Asigna un nombre más específico a una versión:

```bash
nvm alias <new_name> <version>
```

Ejemplo:

```bash
nvm alias new-api 18.20.8
```

- **Usa Alias Personalizado**:

```bash
nvm use <new_name>
```

Ejemplo:

```bash
nvm use new-api
```

### Eliminar una versión de Node.js

- **Desinstalar una Versión**:

```bash
nvm uninstall <version>
```

Ejemplo:

```bash
nvm uninstall 20.19.5
```

- **Eliminar por Versión Principal (NVM en macOS/Linux)**: Puedes eliminar una versión ejecutando el comando `uninstall` con una versión principal como `22` o `20`:

```bash
nvm uninstall 20
```

Si tienes múltiples versiones para esa versión mayor instaladas, como `20.14.0` y `20.12.2`, la versión más reciente se eliminará primero. Ejecutar `nvm uninstall 20` eliminará primero `20.14.0`.

### Verificar la versión de Node.js

- **Verificar Versión Actual**:

```bash
node -v
```

### Verificar versión de npm

- **npm**: Un gestor de paquetes para Node.js usado para instalar, publicar y gestionar paquetes de software. Se instala automáticamente cuando instalas Node.js.

- **Verificar la versión de npm**:

```bash
npm -v
```

### Ejecuta un archivo JavaScript

- **Ejecutar archivo JavaScript**: Ejecuta el archivo JavaScript en el runtime de Node.js activo actualmente:

```bash
node <file.js>
```

Ejemplo:

```bash
node app.js
```

### Crea un proyecto de Node.js

- **Inicializar Proyecto**: Crea un nuevo proyecto de Node.js creando un archivo `package.json` que rastrea sus detalles y dependencias:

```bash
npm init
```

- **Configuración interactiva**: El comando `npm init` te guía a través de una interfaz CLI interactiva para configurar el archivo `package.json`.
  - Hace preguntas sobre el nombre del proyecto, descripción, versión, etc.

- **Configuración predeterminada**: Para comenzar con un archivo `package.json` predeterminado:

```bash
npm init -y
```

o

```bash
npm init --yes
```

### Instalar un Paquete

- **Instalar Paquete Específico**:

```bash
npm install <package>
```

Ejemplo:

```bash
npm install express
```

### Instala todas las dependencias

- **Instalar Dependencias**: Para instalar todas las dependencias listadas en el archivo `package.json` de un proyecto:

```bash
npm install
```
