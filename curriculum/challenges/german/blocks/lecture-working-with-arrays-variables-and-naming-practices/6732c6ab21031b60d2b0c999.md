# --description--

Das Benennen von Variablen und Funktionen ist ein wichtiger Aspekt beim Schreiben von sauberem, lesbarem und wartbarem Code. Gute Benennungspraktiken machen Ihren Code selbstdokumentierend, reduzieren den Bedarf an umfangreichen Kommentaren und erleichtern es anderen Entwicklern, einschließlich Ihnen selbst in der Zukunft, Ihren Code zu verstehen.

Beginnen wir mit allgemeinen Benennungskonventionen in JavaScript. In vorherigen Lektionen haben Sie gelernt, für Variablennamen camel case zu verwenden. Für Boolean-Variablen ist es üblich, Präfixe wie `is`, `has` oder `can` zu verwenden. Das zeigt dem Leser sofort, dass die Variable ein Boolean ist:

```js
let isLoading = true;
let hasPermission = false;
let canEdit = true;
```

Bei Funktionen sollte der Name klar angeben, was die Funktion tut. Es ist oft hilfreich, mit einem Verb zu beginnen:

```js
function getUserData(){
  /* ... */
}

function calculateTotal(){
  /* ... */
}

function validateInput(){
  /* ... */
}
```

Für Funktionen, die einen Boolean zurückgeben und oft Prädikate genannt werden, können Sie dieselben `is`, `has` oder `can` Präfixe verwenden:

```js
function isValidEmail(email) {
  /* ... */
}

function hasRequiredFields(form) {
  /* ... */
}
```

Wenn Sie Funktionen haben, die Daten abrufen, ist es üblich, mit dem Wort `get` zu beginnen:

```js
function getProductDetails(productId) {
  /* ... */
}

function getUserProfile(userId) {
  /* ... */
}
```

Wenn Sie Funktionen haben, die Daten setzen, ist es üblich, mit dem Wort `set` zu beginnen:

```js
function setUserPreferences(preferences) {
  /* ... */
}

function setPageTitle(title) {
  /* ... */
}
```

Für Event-Handler-Funktionen können Sie mit `handle` ein Präfix setzen oder mit `handler` ein Suffix anhängen:

```js
function handleClick(){
  /* ... */
}

function onSubmit(){
  /* ... */
}

function keyPressHandler(){
  /* ... */
}
```

Ein Event-Handler ist eine Aktion, die nach dem Eintreten eines Ereignisses passiert, wie zum Beispiel ein Button-Klick. Das werden Sie in zukünftigen Lektionen lernen.

Beim Benennen von Iterator-Variablen und Schleifen ist es üblich, einzelne Buchstaben wie `i`, `j` oder `k` zu verwenden, aber für verschachtelte Schleifen oder komplexere Iterationen können aussagekräftigere Namen hilfreich sein:

```js
for (let i = 0; i < array.length; i++) {
  /* ... */
}

for (let studentIndex = 0; studentIndex < students.length; studentIndex++) {
  /* ... */
}
```

Für Array-Namen sollten Sie in Betracht ziehen, Pluralformen zu verwenden, um anzuzeigen, dass die Variable mehrere Elemente enthält:

```js
const colors = ['red', 'green', 'blue'];
const userNames = ['Alice', 'Bob', 'Charlie'];
```

Denken Sie daran, dass das Ziel ist, Ihren Code so selbsterklärend wie möglich zu machen. Eine gute Faustregel ist, dass wenn Sie einen Kommentar hinzufügen müssen, um zu erklären, was eine Variable oder Funktion tut, Sie vielleicht in Erwägung ziehen sollten, sie in etwas Beschreibenderes umzubenennen.

Zuletzt sollten Sie in Ihrem Code oder Team konsistent sein. Wenn Ihr Team bereits Benennungskonventionen etabliert hat, halten Sie sich daran. Konsistenz macht den Code für alle Beteiligten lesbarer und wartbarer.
