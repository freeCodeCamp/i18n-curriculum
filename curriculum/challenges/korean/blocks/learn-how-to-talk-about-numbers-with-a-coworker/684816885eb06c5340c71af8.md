# --explanation--

`Initial unit tests`는 테스트의 첫 단계입니다. 개발자는 코드의 작은 부분(`units`)이 제대로 작동하는지 확인합니다. 예를 들어:

`We wrote initial unit tests to check the main functions.` — 팀은 가장 중요한 코드 부분이 올바르게 작동하는지 확인하기 위해 초기 테스트를 만들었습니다.

`Quality assurance (QA) checks`는 품질 보증 팀이 수행하는 최종 테스트로, 소프트웨어가 신뢰할 수 있고 사용자 친화적이며 출시 준비가 되었는지 확인합니다. 예를 들어:

`Regular QA checks help prevent bugs from reaching users.` — 품질 보증의 빈번한 테스트는 오류를 조기에 발견하는 데 도움이 되어 사용자가 문제를 겪지 않도록 합니다.

Alice는 테스트 예산이 초기 코딩 검사부터 출시 전 최종 테스트까지 모든 단계를 포함한다고 설명하고 있습니다.
