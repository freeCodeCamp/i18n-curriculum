# --description--

Impariamo cos'è Bash e alcuni dei suoi comandi di base.

Bash sta per Bourne Again SHell, ed è probabilmente la shell più comune che incontrerai in ambienti Unix-like. Per questo motivo, può essere molto utile capire come navigare in questa shell e usare i comandi comuni.

Il primo comando è `pwd`, che stampa la directory di lavoro corrente nel terminale. La "directory di lavoro" indica la directory a cui il terminale è attualmente puntato, ed è una cosa importante da sapere per molti di questi altri comandi.

È del tutto possibile che il tuo terminale sia puntato alla directory sbagliata. Il comando `cd` ti permette di cambiare directory. Puoi specificare un percorso assoluto, con prefisso la barra (`/`), un percorso relativo senza prefisso, e usare la sintassi con il doppio punto (`..`) per salire alla directory genitore.

Il comando successivo è `ls`, che elenca il contenuto della directory di lavoro corrente. Questo è utile per vedere se un file o una cartella esiste nella tua directory. Il comando `ls` accetta anche flag per mostrare file nascosti (il flag `-a`), i permessi dei file (il flag `-l`) e altre funzionalità. Imparerai di più sui flag dei comandi nella prossima lezione. Puoi anche usare `less` o `cat` per visualizzare il contenuto di un file.

E se la directory non contiene il file o la cartella di cui hai bisogno? Ci sono comandi chiamati `mkdir` e `touch` che ti permettono di crearli. `mkdir` crea una nuova directory, o cartella, e `touch` crea un nuovo file. Entrambi questi comandi accettano il nome del file o della cartella che vuoi creare. Per esempio, `touch readme.md` crea un nuovo file Markdown chiamato `readme.md`.

Forse il tuo nuovo file non avrebbe dovuto chiamarsi `readme.md`, ma `readthis.md`. Per fortuna, il comando `mv` ti permette di spostare (o rinominare) un file. Prende il vecchio nome del file e il nuovo nome.

Il comando `rm` ti permette di rimuovere (o eliminare) un file, e con il flag `-r` eliminerà una directory. A volte un file potrebbe essere protetto, e dovrai includere il flag `-f`.

Il comando `cp` può essere usato per copiare un file o una directory in una nuova posizione. A differenza del comando `mv`, il comando `cp` non rimuove il file originale — e per copiare una directory, dovrai passare il flag `-r`.

Il comando `echo` può essere considerato l'equivalente Bash di una funzione `console.log()` o `print()`. `echo` prende un argomento stringa, racchiuso tra virgolette, e lo stampa nel terminale. Potrebbe sembrare sciocco eseguirlo nel terminale solo per ottenere testo nel terminale, ma puoi effettivamente concatenare `echo` a un simbolo di controllo. Il simbolo `>` ti permette di specificare un nome file da creare o sovrascrivere con la nuova stringa, come `echo "Naomi was here." > readme.md`, e il simbolo `>>` aggiungerà al file.

Ci sono molti altri comandi Bash disponibili, come `head`, `tail`, `chown` e `chmod`. Sarebbe impossibile coprirli tutti in questa lezione. Ma puoi usare il comando `man` per vedere il manuale o la pagina di aiuto per quasi ogni comando.

Probabilmente ricorderai i comandi che usi più frequentemente. E per gli altri, puoi sempre consultare la pagina man.
