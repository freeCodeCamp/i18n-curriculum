# --description--

In regulären Ausdrücken findet ein <dfn>gieriger</dfn> Match den längstmöglichen Teil eines Strings, der zum Regex-Muster passt, und gibt ihn als Treffer zurück. Die Alternative wird <dfn>lazy</dfn> Match genannt, bei dem der kleinstmögliche Teil des Strings gefunden wird, der das Regex-Muster erfüllt.

Sie können den Regex `/t[a-z]*i/` auf den String `"titanic"` anwenden. Dieser Regex ist im Grunde ein Muster, das mit `t` beginnt, mit `i` endet und dazwischen einige Buchstaben enthält.

Reguläre Ausdrücke sind standardmäßig gierig, daher würde der Treffer `["titani"]` zurückgeben. Er findet die größtmögliche Teilzeichenkette, die zum Muster passt.

Sie können jedoch das `?`-Zeichen verwenden, um auf lazy matching umzuschalten. `"titanic"`, angewendet auf den angepassten Regex `/t[a-z]*?i/`, gibt `["ti"]` zurück.

**Hinweis:** Das Parsen von HTML mit regulären Ausdrücken sollte vermieden werden, aber das Musterabgleichen eines HTML-Strings mit regulären Ausdrücken ist völlig in Ordnung.

# --instructions--

Korrigieren Sie den Regex `/<.*>/`, damit der HTML-Tag `<h1>` und nicht der Text `"<h1>Winter is coming</h1>"` zurückgegeben wird. Denken Sie daran, dass das Wildcard-Zeichen `.` in einem regulären Ausdruck für jedes Zeichen steht.
