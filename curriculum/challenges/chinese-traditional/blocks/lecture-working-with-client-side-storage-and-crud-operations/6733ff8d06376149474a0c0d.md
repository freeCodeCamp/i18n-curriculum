# --description--

`Cache` API 用於儲存網路請求和回應，使網頁應用程式運作更高效，甚至能離線運作。它是更廣泛的 Service Worker API 的一部分，對於建立能在不穩定或緩慢網路條件下運作的漸進式網頁應用程式（PWA）至關重要。 

在我們繼續使用 `Cache` API 之前，我們首先需要了解服務工作者、PWA 和網路請求在基本層次上的運作方式。 

網路請求是由網頁瀏覽器或應用程式向伺服器發出的請求，用以透過網際網路擷取資料或資源。例如，當你造訪網站時，你的瀏覽器會向網頁伺服器發送網路請求，以取得顯示頁面所需的檔案（例如 HTML、影像或影片）。

Service worker 是一種特殊型別的 JavaScript 檔案，會在網頁應用程式的背景中執行，與主要瀏覽器執行緒分開。它充當網頁與網路之間的中介，讓開發者能攔截網路請求、快取資源，並處理推播通知或背景同步等功能。

PWA，或稱為進階網頁應用程式，是一種使用現代網頁技術，在網路上提供原生應用程式般體驗的網頁應用程式。PWA 設計成能在任何裝置上可靠運作，無論網路狀況如何，並且可以透過瀏覽器存取，或像傳統應用程式一樣安裝在使用者的裝置上。 

既然我們對 service workers、PWAs 和網路請求有了更深入的了解，我們就可以更深入探討 Cache API。

`Cache` API 是一種儲存機制，用來儲存 `Request` 和 `Response` 物件。當向伺服端發出請求時，應用程式可以將回應存放起來，之後從快取中取回，而不必重新發出新的網路請求。這樣可以減少載入時間、節省頻寬，並提升整體使用者體驗。

瀏覽器提供一個稱為 `CacheStorage` 的儲存區域，開發者可以在此存檔網路請求及其對應回應的鍵值對組。

使用 `Cache-Control` 標頭，開發者可以指定快取資源應該儲存多長時間，以及是否應該重新驗證或直接從快取提供。 

透過使用 `Cache` API，開發者可以建置離線優先的網頁應用程式，允許 PWA 例如在使用者與網路斷線時提供快取的資產。
