# --interactive--

У попередньому уроці ви вперше познайомилися з роботою з масивами та доступом до різних елементів масивів. Ось нагадування, як отримати доступ до другого елемента масиву:

:::interactive_editor

```js
const fruits = ["apple", "banana", "cherry"];

console.log(fruits[1]); // "banana"
```

:::

Оскільки індексація масивів починається з нуля, перший елемент буде за індексом `0`, другий елемент — за індексом `1` і так далі. Важливо пам’ятати, що якщо ви спробуєте отримати доступ до індексу, якого немає в масиві, JavaScript поверне `undefined`.

:::interactive_editor

```js
let fruits = ["apple", "banana", "cherry"];
console.log(fruits[3]); // undefined
```

:::

У цьому прикладі немає індексу `3` у масиві `fruits`. Тож у консолі з’явиться `undefined`. Тепер подивімося, як оновити елементи в масиві. Ви можете оновити елемент, призначивши нове значення певному індексу.

:::interactive_editor

```js
let fruits = ["apple", "banana", "cherry"];
fruits[1] = "blueberry";
console.log(fruits); // ["apple", "blueberry", "cherry"]
```

:::

У цьому прикладі ми замінили `banana` на `blueberry` за індексом `1`. Цей метод дозволяє змінити будь-який елемент масиву, якщо ви знаєте його індекс. Ви також можете додати нові елементи в масив, призначивши значення індексу, якого ще немає:

:::interactive_editor

```js
let fruits = ["apple", "banana", "cherry"];
fruits[3] = "date";
console.log(fruits); // ["apple", "banana", "cherry", "date"]
```

:::

Однак будьте обережні при цьому. Якщо ви призначите значення індексу, який значно більший за поточну довжину масиву, між існуючими індексами з’являться невизначені елементи, що може призвести до непередбачуваної поведінки. Продовжуючи працювати з JavaScript, ви побачите, що ці способи доступу та оновлення елементів масиву є фундаментальними для багатьох програмних завдань. Чи створюєте ви простий список справ, чи обробляєте складні структури даних, ці навички будуть надзвичайно корисними.
