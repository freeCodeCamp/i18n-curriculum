# --description--

افترض أن لديك سلسلتين نصيتين تريد دمجهما لإنشاء رسالة واحدة، كما في المثال أدناه:

```js
const str1 = "Hello";
const str2 = "World!";
const message = str1.push(str2);
```

<img src="https://cdn.statically.io/gh/TheOdinProject/curriculum/4ed59981b4ce2c60b5b83bf7415d3127b61821f5/foundations/javascript_basics/understanding_errors/imgs/03.png" style="width:100%" alt="an example of a type error in the Chrome developer console">

هنا، ستحصل على `TypeError` مع رسالة تفيد بأن `str1.push is not a function`. هذه رسالة خطأ شائعة تشتت المتعلمين لأنك قد تعرف أن `.push()` هي دالة بالتأكيد (على سبيل المثال، إذا استخدمتها سابقًا لإضافة عناصر إلى المصفوفات).

لكن هذه هي النقطة الأساسية - `.push()` ليست طريقة من طرق السلسلة النصية، بل هي طريقة من طرق المصفوفة. لذلك، فهي "ليست دالة" يمكنك إيجادها كطريقة للسلسلة النصية. إذا غيرت `.push()` إلى `.concat()`، وهي طريقة صحيحة للسلسلة النصية، ستعمل الشفرة كما هو مقصود!

ملاحظة جيدة يجب تذكرها عند مواجهة `TypeError` هي التفكير في نوع البيانات التي تحاول تشغيل طريقة أو عملية عليها. ستجد على الأرجح أنها ليست كما تعتقد، أو أن العملية أو الطريقة غير متوافقة مع ذلك النوع.

## --text--

لماذا ينتج عن شفرة JavaScript التالية `TypeError`؟

```js
const str1 = "Hello";
const str2 = "World!";
const message = str1.push(str2);
```

## --answers--

طريقة `.push()` يمكن استخدامها فقط على الكائنات، وليس على السلاسل النصية.

---

طريقة `.push()` غير متوفرة للسلاسل النصية لأنها طريقة من طرق المصفوفة.

---

طريقة `.push()` مكتوبة بشكل خاطئ ويجب أن تكون .pusch().

---

`TypeError` ناتج عن خطأ نحوي في شفرة JavaScript.
