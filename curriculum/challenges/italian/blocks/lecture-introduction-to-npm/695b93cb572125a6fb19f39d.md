# --description--

Mantenere aggiornate le tue dipendenze—e sapere come installare una versione specifica—è importante per mantenere stabile il tuo progetto. Ti aiuta a evitare bug imprevisti e ti dà più controllo quando vengono rilasciate nuove versioni.

Per installare una versione specifica di un pacchetto, usa questo formato:

```bash
npm install package@version-number
```

Per esempio, se il tuo progetto ha bisogno di una versione più vecchia di Express, diciamo `4.21.2`, puoi installarla così:

```bash
npm install express@4.21.2
```

Se non includi un numero di versione, npm installerà automaticamente l’ultima versione disponibile.

Se non sei sicuro di quali versioni esistano, puoi controllarle con questo comando:

```bash
npm view package-name versions
```

Per esempio:

```bash
npm view express versions
```

Per aggiornare i pacchetti, puoi aggiornarli singolarmente o tutti insieme. Prima di farlo, è utile vedere quali sono obsoleti:

```bash
npm outdated
```

Questo ti mostrerà quali dipendenze hanno versioni più recenti disponibili, e dovresti vedere un output simile a questo:

```bash
user@Kolade ~/Desktop/fCC/script-code/node/npm/install-spec-version  % npm outdated
Package  Current  Wanted  Latest  Location              Depended by
chalk      5.5.0   5.5.0   5.6.2  node_modules/chalk    install-spec-version
cleave     1.0.1   1.0.1   1.0.2  node_modules/cleave   install-spec-version
express   4.21.2  4.21.2   5.2.1  node_modules/express  install-spec-version
```

Una volta controllato quali dipendenze sono obsolete, puoi iniziare ad aggiornarle.

Per aggiornare una singola dipendenza, usa il seguente comando:

```bash
npm update package-name
```

Questo aggiorna il pacchetto solo entro l’intervallo di versioni consentito dal tuo file `package.json`. Per esempio, per aggiornare Express, puoi usare questo:

```bash
npm update express
```

Se vuoi saltare l’intervallo di versioni e installare l’ultima versione di un pacchetto, usa questo comando:

```bash
npm install package-name@latest
```

Per aggiornare tutte le dipendenze che rientrano negli intervalli di versione consentiti, esegui:

```bash
npm update
```

Tieni però presente che `npm update` non aggiorna un pacchetto a una nuova versione major se il tuo `package.json` non lo permette. Le versioni major spesso contengono cambiamenti incompatibili, quindi npm non le applica automaticamente.

Se vuoi specificamente la versione major più recente, devi eseguire:

```bash
npm install package-name@latest
```

Questo forza npm a installare l’ultima release, indipendentemente dall’intervallo di versioni nel tuo `package.json`.
