# --interactive--

In der vorherigen Lektion haben Sie gelernt, wie Sie mit dem `arguments`-Objekt arbeiten, das ein array-ähnliches Objekt ist und die Werte der an die Funktion übergebenen Argumente enthält.

:::interactive_editor

```js
function logArgs() {
  for (const arg of arguments) {
    console.log(arg);
  }
}

logArgs(1, 2, 3);
// result:
// 1
// 2
// 3
```

:::

Obwohl dies eine gültige Methode ist, um auf eine variable Anzahl von Argumenten einer Funktion zuzugreifen und mit ihnen zu arbeiten, verwenden moderne JavaScript-Anwendungen stattdessen die Syntax der Rest-Parameter.

Hier ist ein aktualisiertes Beispiel, das die Syntax der Rest-Parameter anstelle des `arguments`-Objekts verwendet:

:::interactive_editor

```js
function logArgs(...args) {
  for (const arg of args) {
    console.log(arg);
  }
}

logArgs(1, 2, 3);
// result:
// 1
// 2
// 3
```

:::

Dieses aktualisierte Beispiel verweist nicht mehr direkt auf das `arguments`-Objekt. Stattdessen hat der letzte Parameter der Funktionsdefinition drei Punkte davor. Dadurch wird dieser Rest-Parameter in einem `Array`-Objekt abgelegt. Sie können diesen Rest-Parameter beliebig benennen. Achten Sie nur darauf, dass er der letzte Parameter in der Funktionsdefinition ist, so wie hier:

```js
function exampleFunction(a, b, ...restOfArgs) {
  // some code here
}

function anotherFunction(x, y, ...theArgs) {
  // some code here
}
```

Es gibt noch einige weitere Einschränkungen bei der Verwendung der Syntax für Rest-Parameter. Eine Einschränkung ist, dass Funktionsdefinitionen nur einen Rest-Parameter haben dürfen. Das folgende Beispiel hier wäre also ungültig.

```js
// Won't work

function badFunction(...args, ...moreArgs) {
  // some code here
}
```

Die nächste Einschränkung ist, dass nach dem Rest-Parameter keine abschließenden Kommas erlaubt sind: `function exampleFunction(a, b, ...restOfArgs, )`.

Eine weitere Einschränkung ist, dass der Rest-Parameter keinen Standardwert haben darf. Andernfalls wird ein `SyntaxError` geworfen.

```js
function badFunction(...args = [1,2]){
  // some code here
}
```

Was sind also einige Unterschiede zwischen dem `arguments`-Objekt und den `rest`-Parametern?

Ein wesentlicher Unterschied ist, dass das `arguments`-Objekt kein echtes Array ist, weshalb es Methoden wie `includes`, `pop` und `push` nicht unterstützt. Der Rest-Parameter ist jedoch eine `Array`-Instanz. Sie können also gültige eingebaute Array-Methoden verwenden, ohne ihn zuerst in ein echtes Array umwandeln zu müssen.

:::interactive_editor

```js
function hasCat(...args) {
  return args.includes("cat");
}

console.log(hasCat("dog", "chicken", "cat")); // true
console.log(hasCat("dog", "chicken", "horse")); // false
```

:::
