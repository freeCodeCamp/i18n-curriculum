# --interactive--

`for...in` ループは、オブジェクトのプロパティを繰り返し処理する必要があるときに最適です。このループは、継承されたプロパティや数値以外のプロパティを含む、オブジェクトのすべての列挙可能なプロパティを繰り返します。

継承されたプロパティとは、オブジェクトのプロトタイプチェーンから継承されたプロパティのことです。数値以外のプロパティとは、数値ではないか、数値に変換できる文字列でないプロパティのことです。

以下は `for...in` ループの基本的な構文です：

```js
for (variable in object) {
  // code block to be executed
}
```

例の変数は、繰り返し処理されているオブジェクトの現在のプロパティを表します。

`for...in` ループの動作をよりよく理解できるように、いくつか例を見てみましょう。

最初の例では `fruit` オブジェクトがあり、各プロパティを繰り返して値をコンソールに出力します。

:::interactive_editor

```js
const fruit = {
  name: 'apple',
  color: 'red',
  price: 0.99
};

for (const prop in fruit) {
  console.log(fruit[prop]);
}
```

:::

`prop` 変数はオブジェクトの現在のプロパティを表します。`fruit[prop]` は各プロパティの値にアクセスするために使われます。

最初の反復では `prop` が `name` になります。2回目の反復では `prop` が `color` になり、以下同様です。

コンソールに出力される結果は `apple`、`red`、`0.99` です。

2つ目の例では、ネストされたオブジェクトがあり、各プロパティを繰り返して値をコンソールに出力します。

:::interactive_editor

```js
const person = {
  name: 'John',
  age: 30,
  address: {
    street: '123 Main St',
    city: 'Anytown',
    state: 'CA'
  }
};

for (const prop in person) {
  console.log(person[prop]);
}
```

:::

`address` プロパティ自体がオブジェクトです。`for...in` ループは `person` オブジェクトのプロパティも繰り返し処理し、`address` オブジェクト全体をコンソールに出力します。

コンソールでの結果は以下のようになります：

```md
John
30
{ street: '123 Main St', city: 'Anytown', state: 'CA' }
```

`address` オブジェクトのプロパティを繰り返したい場合は、最初のループの中に別の `for...in` ループをネストしてください。

:::interactive_editor

```js
const person = {
  name: 'John',
  age: 30,
  address: {
    street: '123 Main St',
    city: 'Anytown',
    state: 'CA'
  }
};

function isObject(obj) {
  return typeof obj === 'object' && !Array.isArray(obj) && obj !== null;
}

for (const prop in person) {
  if (isObject(person[prop])) {
    for (const nestedProp in person[prop]) {
      console.log(person[prop][nestedProp]);
    }
  } else {
    console.log(person[prop]);
  }
}
```

:::

この例では、値がオブジェクトかどうかをチェックするカスタム関数 `isObject` を使っています。

`Array.isArray` メソッドは値が配列かどうかをチェックするために使われます。論理否定演算子（`!`）をメソッドの前に置くことで、値が配列でないことをチェックしています。

単に `typeof` が `'object'` と等しいかどうかを使えない理由は、配列も JavaScript ではオブジェクトとみなされるためです。配列をチェックから除外したいのです。

また、JavaScript の歴史的なバグのために `typeof null` は `'object'` を返します。したがって `null` の値もチェックから除外したいです。

条件が真の場合、ネストされたオブジェクトのプロパティを繰り返し処理して値をコンソールに出力する別の `for...in` ループをネストします。

`nestedProp` 変数はネストされたオブジェクトの現在のプロパティを表します。

修正後のコンソールの結果は以下のようになります：

```md
"John"
30
"123 Main St"
"Anytown"
"CA"
```

`for...in` ループは、オブジェクトのプロパティを繰り返し処理する必要があるときに便利です。

配列の要素を繰り返すために `for...in` ループを使うことは推奨されません。代わりに `for...of` ループや、今後のレッスンで学ぶ `forEach`、`map`、`filter`、`reduce` といった配列メソッドを使ってください。
