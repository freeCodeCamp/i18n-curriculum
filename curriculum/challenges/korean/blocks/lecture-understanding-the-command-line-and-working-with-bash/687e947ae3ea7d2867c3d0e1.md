# --description--

명령어 옵션과 플래그에 대해 배워보겠습니다.

이전 수업에서 `touch readme.md` 같은 명령어에 인수를 전달하는 방법을 배웠습니다. 옵션 또는 플래그는 명령어에 전달할 수 있는 특별한 인수로, 명령어의 동작 방식을 바꿉니다. 두 용어는 서로 바꿔 쓰이지만, "플래그"는 켜기/끄기 토글 역할을 하는 옵션에 좀 더 구체적으로 사용되는 경향이 있습니다.

옵션은 보통 하나 또는 두 개의 하이픈(`-`) 기호가 접두사로 붙어, 옵션과 인수를 시각적으로 구분하는 데 도움을 줍니다. 먼저 두 개의 하이픈, 즉 "롱 폼" 구문을 살펴보겠습니다.

`ls` 같은 많은 애플리케이션은 `--version` 플래그를 받아, 애플리케이션의 명령어를 실행하는 대신 현재 버전을 터미널에 출력합니다.

또한 많은 애플리케이션은 `--help` 플래그를 제공하여 애플리케이션 사용법에 대한 안내를 출력합니다.

하나의 하이픈, 즉 "숏 폼" 구문은 보통 한 글자로 된 옵션을 사용합니다. 예를 들어, `-a` 플래그와 `ls`는 점(`.`)으로 시작하는 숨김 파일을 포함해 모든 파일을 나열합니다. 예를 들어 `.env`가 있습니다.

이 짧은 옵션의 장점은 여러 플래그를 한꺼번에 연결할 수 있다는 점입니다. `ls --all --human-readable --size` 대신 `ls -ahs`처럼 한 번에 여러 글자를 사용할 수 있습니다.

일부 옵션은 값이 전달되기를 기대합니다.

롱 폼 옵션을 사용할 때는 보통 등호를 사용해야 합니다. 이 구문에서는 값이 등호(`=`) 기호와 함께 옵션에 바로 붙습니다. 다음은 `ls`의 동작을 색상 포함 여부로 바꾸는 예입니다:

```bash
ls --color=never
```

숏 폼 옵션을 사용할 때는 보통 값과 옵션을 공백으로 구분합니다. 예를 들어, `width` 결과의 `ls`을 설정하는 롱 폼과 숏 폼 옵션은 다음과 같습니다:

```bash
ls -w 50
```

```bash
ls --width=50
```

숏 폼은 `-w 50`를 사용하고 롱 폼은 `--width=50`를 사용하는 점을 주목하세요. 이 차이를 알아야 옵션 값을 위치 인수로 잘못 전달하는 일을 피할 수 있습니다.

하지만 확실하지 않을 때는 보통 `--help` 플래그를 사용해 옵션에 기대되는 구문을 확인할 수 있다는 점을 기억하세요!
