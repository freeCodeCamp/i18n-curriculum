# --description--

Expressでは、**ルートパス**はサーバーが受け取ったリクエストにどのように応答するかを決めるURLパターンです。ユーザーが特定のURLをリクエストすると、ExpressはそのURLをルートパスと照合し、対応するハンドラを実行します。

ルートパスは`app.get()`、`app.post()`などのルートメソッド内で定義されます。これらのパスは`/home`のような静的なものや、`/user/:id`のようにURLから値を取得するためのパラメータを使った動的なものがあります。

このレッスンでは静的ルートに焦点を当てます。以下はその例です。

```javascript
// Static route
app.get("/home", (req, res) => {
  res.send("Welcome to the homepage!");
});
```

この例では、サーバーは`/home`へのGETリクエストを待ち受け、シンプルなメッセージで応答します。

`app.get()`メソッドは2つの引数を取ります。

* 最初の引数`"/home"`は、サーバーが待ち受ける静的なルートパス（またはURL）です。
    
* 2つ目の引数は、そのパスにGETリクエストがあったときに実行されるコールバック関数です。
    

コールバック内で`res.send()`はクライアントにレスポンスを送ります。この場合、ブラウザに表示されるテキスト`Welcome to the homepage!`を送信します。

もう一つ静的ルートの例を示します。

```javascript
// Static route
app.get("/menu/drinks", (req, res) => {
  res.send("Welcome to the drinks menu!");
});
```

この場合、サーバーは`/menu/drinks`のURLへのGETリクエストを待ち受け、そのルートがリクエストされるとメッセージを返します。

このようなルートは、サイトに共通のパスの下に複数の関連ページがある場合に使うことがあります。例えば、レストランのウェブサイトがメニューページを次のように整理する場合です。

* `/menu/food`
    
* `/menu/drinks`
    
* `/menu/desserts`
    

この方法はURLをわかりやすく意味のあるものに保ち、アプリケーションが成長してもルートの整理や管理がしやすくなります。
