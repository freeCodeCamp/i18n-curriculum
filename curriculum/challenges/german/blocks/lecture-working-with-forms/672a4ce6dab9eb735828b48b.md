# --interactive--

Wenn ein Benutzer ein Formular auf Ihrer Website ausfüllt, ist es wichtig, dass er alle notwendigen Informationen im richtigen Format eingibt. HTML-Formularsteuerelemente wie Eingaben verfügen über viele eingebaute Validierungen, die Sie verwenden können, um ungültige Daten zu überprüfen. Das hilft sicherzustellen, dass der Benutzer diese Fehler behebt, bevor die Informationen abgesendet und vom Server verarbeitet werden.

Der Begriff „clientseitig“ bezieht sich auf alles, was auf dem Computer oder Gerät des Benutzers passiert, also auf den Teil einer Website oder Anwendung, mit dem Sie direkt interagieren. Dazu gehören das Layout, das Design und alle interaktiven Funktionen.

Der Begriff „serverseitig“ bezieht sich auf alles, was auf dem Server, dem Computer oder System, das die Website oder Anwendung hostet, passiert. Dazu gehören die Datenverarbeitung, das Ausführen von Anwendungen und das Bearbeiten von Anfragen, die vom Gerät des Benutzers kommen.

Während die clientseitige Validierung wichtig ist, benötigen Sie auch eine serverseitige Validierung für zusätzliche Sicherheit. Böswillige Benutzer können clientseitige Prüfungen umgehen, daher sind robuste serverseitige Maßnahmen unerlässlich. Mehr dazu lernen Sie in einem späteren Modul. Schauen wir uns jetzt einige Beispiele für clientseitige Formularvalidierung an.

Ein häufiges Beispiel für eingebaute Formularvalidierung ist die Verwendung des `required`-Attributs in Eingaben. Das `required`-Attribut gibt an, dass der Benutzer diesen Teil des Formulars ausfüllen muss, bevor es abgesendet wird. Hier ist ein Beispiel für die Verwendung des `required`-Attributs in einem E-Mail-Eingabefeld.

Klicken Sie auf die `Submit Form`-Schaltfläche, ohne eine E-Mail-Adresse anzugeben, und Sie sehen eine Meldung, die Sie auffordert, das Feld auszufüllen.

:::interactive_editor

```html
<form action="">
  <label for="email">Email Address (Required field):</label>
  <input required type="email" name="email" id="email" />
  <button type="submit">Submit Form</button>
</form>
```

:::

Jeder Browser hat seine eigenen Stile, um diese Warnmeldung anzuzeigen.

Ein weiterer Vorteil der Verwendung des E-Mail-Eingabefelds ist, dass E-Mail-Eingaben eine grundlegende Validierung haben, um korrekt formatierte E-Mail-Adressen sicherzustellen. Wenn Sie zum Beispiel zufällige Wörter eingeben und auf absenden klicken, zeigt der Browser eine Warnung an, dass ein `@`-Zeichen fehlt.

Geben Sie `abc` in das E-Mail-Feld ein und klicken Sie auf die Absenden-Schaltfläche. Es sollte eine Meldung erscheinen, weil es keine gültige E-Mail-Adresse ist.

:::interactive_editor

```html
<form action="">
  <label for="email">Email Address (Required field):</label>
  <input required type="email" name="email" id="email" />
  <button type="submit">Submit Form</button>
</form>
```

:::

Es ist wichtig zu beachten, dass Browser nur eine grundlegende Validierung für Standard-E-Mail-Adressen durchführen. Es liegt an Ihnen, zusätzliche Validierungsebenen hinzuzufügen, die Sie in späteren Modulen lernen werden.

Andere Formen der Validierung für E-Mail-Eingaben sind die Verwendung der `minlength`- und `maxlength`-Attribute. Hier ist ein Beispiel mit der zusätzlichen Validierung.

Geben Sie `b@m` in das Feld ein und klicken Sie auf die Absenden-Schaltfläche. Es erscheint eine Meldung, weil die Mindestlänge nicht erfüllt ist.

:::interactive_editor

```html
<form action="">
  <label for="email">Email Address (Required field):</label>
  <input
    required
    type="email"
    name="email"
    id="email"
    minlength="4"
    maxlength="64"
  />
  <button type="submit">Submit Form</button>
</form>
```

:::

Die `minlength`- und `maxlength`-Attribute werden verwendet, um die minimale und maximale Länge in Zeichen für die E-Mail-Eingabe festzulegen. Wenn Sie die Mindestlänge nicht angeben oder die maximale Zeichenanzahl überschreiten, zeigt der Browser eine Warnmeldung an.
