# --description--

У цьому уроці ви навчитеся додавати типи для об’єктів і масивів.

Спочатку розглянемо масиви.

Ось приклад масиву мов програмування:

```js
const programmingLanguages = ["Java", "C++", "Python"];
```

Якщо ви хочете явно вказати тип цього масиву як масив рядків, то можете призначити йому тип `string[]`:

```ts
const programmingLanguages:string[] = ["Java", "C++", "Python"];
```

Як ви вже дізналися раніше, TypeScript буде достатньо розумним, щоб вивести тип як масив рядків, якщо просто написати так:

```js
const programmingLanguages = ["Java", "C++", "Python"];
```

Але явне призначення типу допоможе у випадках, коли ви спробуєте додати до масиву інший тип, наприклад так:

```ts
// Type 'number' is not assignable to type 'string'.(2322)
const programmingLanguages:string[] = ["Java", 9, "Python"];
```

Ще один спосіб типізувати масиви — це використати такий синтаксис:

```ts
const programmingLanguages:Array<string> = ["Java", "C++", "Python"];
```

Фактично, ці два синтакси однакові, і вибір між ними зазвичай залежить від уподобань.

Тепер розглянемо типізацію об’єктів.

Ось приклад літерала об’єкта:

```js
const developer = {
  firstName: "Jessica",
  isEmployed: true
}
```

Щоб явно типізувати властивості об’єкта, можна використати такий синтаксис:

```ts
const developer: { firstName: string, isEmployed: boolean } = {
  firstName: "Jessica",
  isEmployed: true
}
```

Якщо ви спробуєте додати або видалити властивості з об’єкта, то отримаєте помилку такого вигляду:

```ts
// Property 'isEmployed' is missing in type '{ firstName: string; }' but required in type '{ firstName: string; isEmployed: boolean; }'
const developer: { firstName: string, isEmployed: boolean } = {
  firstName: "Jessica",
}
```

Ви можете зробити властивості об’єкта необов’язковими, використовуючи `?` після назви властивості, наприклад так:

```ts
const developer: { firstName: string, isEmployed?: boolean } = {
  firstName: "Jessica",
}
```

TypeScript більше не показуватиме помилку, бо властивість `isEmployed` позначена як необов’язкова.

Поки що всі приклади об’єктів мали обмежувальні властивості. Це означає, що ми явно перелічили кожну властивість, яку об’єкт може мати:

```ts
const developer: { firstName: string, isEmployed: boolean } = {
  firstName: "Jessica",
  isEmployed: true
}
```

Але що, якщо ви не знаєте всі назви властивостей наперед? Що, якщо ви хочете об’єкт, який може мати будь-яку кількість властивостей, але всі значення мають відповідати певному типу?

Існує два способи це зробити. Перший — використати `Record`, який є вбудованим універсальним утилітним типом у TypeScript.

Ось базовий синтаксис:

```ts
Record<Keys, ValueType>
```

Перший тип представляє тип ключів, а другий — тип значень.

Ось приклад використання `Record` для об’єкта `userRoles`:

```ts
const userRoles: Record<string, string> = {
  admin: "full-access",
  editor: "limited-access",
  viewer: "read-only"
};
```

Цей об’єкт `userRoles` може мати будь-які ключі рядкового типу, і всі значення мають бути рядками. Якщо ви спробуєте призначити значення не рядкового типу, отримаєте помилку:

```ts
// Type 'number' is not assignable to type 'string'
const userRoles: Record<string, string> = {
  admin: 1
};
```

Другий спосіб записати те саме — це індексний підпис:

```ts
const obj: { [key: string]: string } = {};
```

Синтаксис вище означає, що будь-яка властивість із ключем рядкового типу має мати значення рядкового типу. Ось приклад для об’єкта `settings`:

```ts
const settings: { [key: string]: string } = {
  theme: "dark",
  language: "en",
  layout: "grid"
};
```

Як і раніше, якщо у вас неправильні типи значень, TypeScript покаже повідомлення про помилку:

```ts
// Type 'boolean' is not assignable to type 'string'
const settings: { [key: string]: string } = {
  darkMode: true
};
```

І `Record`, і індексні підписи фактично однакові. Тож вибір між ними — це здебільшого справа уподобань.
