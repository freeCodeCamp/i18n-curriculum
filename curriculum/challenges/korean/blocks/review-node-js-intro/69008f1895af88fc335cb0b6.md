## 클라이언트 사이드 JavaScript

- **클라이언트 사이드 JavaScript**: 웹 애플리케이션의 클라이언트 측, 보통 웹 브라우저에서 실행되는 코드입니다.
- **클라이언트 사이드 JavaScript의 한계**: 강력하지만 클라이언트 사이드 JavaScript에는 몇 가지 제한이 있습니다:
  - 로컬 파일에 대한 접근이 매우 제한적입니다.
  - 복잡한 애플리케이션 로직을 처리하도록 설계되지 않았습니다.
  - 보안 문제 가능성 (예: 데이터베이스 자격 증명 같은 민감한 정보 노출).
- **한계의 이유**: 클라이언트 사이드 JavaScript는 처음에 웹 브라우저에서만 실행되도록 설계되었기 때문에 이러한 제한이 존재합니다.

## 브라우저 환경

- **JavaScript 엔진**: 웹 브라우저는 JavaScript 코드를 실행하는 데 필요한 환경과 JavaScript 엔진을 제공합니다.
- **문서 객체 모델(DOM)**: 브라우저는 DOM에 접근할 수 있게 하여 코드에서 HTML 요소에 접근할 수 있습니다.
- **역사적 배경**: 이전에는 JavaScript가 브라우저 밖에서 실행될 수 없었지만, 이제는 Node.js로 실행할 수 있습니다.

## Node.js란?

- **Node.js**: JavaScript 런타임 환경입니다. 공식적으로는 "오픈 소스이자 크로스 플랫폼 JavaScript 런타임 환경"으로 정의됩니다.
- **오픈 소스**: Node.js 코드는 공개되어 있으며 많은 개발자 커뮤니티가 유지 관리합니다.
- **크로스 플랫폼**: Windows, macOS, Linux 등 모든 운영체제에서 작동합니다.
- **JavaScript 런타임 환경**: 브라우저 밖에서 JavaScript 코드를 실행할 수 있게 합니다.

## Node.js 사용 사례

- **일반적인 애플리케이션**: Node.js는 다양한 산업 분야에서 다음과 같은 용도로 사용됩니다:
  - HTTP 요청을 처리하는 웹 서버와 API 구축.
  - 웹 및 모바일 애플리케이션 개발.
  - 복잡한 데이터 및 데이터베이스 상호작용 처리.
- **산업 채택**: 데이터베이스와 상호작용하거나 복잡한 데이터를 처리하는 웹사이트나 웹 애플리케이션을 사용하거나 개발한다면, Node.js가 백엔드에서 사용될 가능성이 높습니다.
- **주요 특징**: Node.js는 효율적이고 확장 가능하며 전 세계에 많은 개발자와 유지 관리자가 있습니다.

## 브라우저와 Node 런타임 환경 비교

- **브라우저 환경**: 주로 프론트엔드 웹 개발용으로 설계되었으며 클라이언트 사이드 JavaScript를 실행합니다.
- **Node 런타임 환경**: 주로 백엔드 웹 개발용으로 설계되었으며 서버 사이드 JavaScript를 실행합니다.
- **API 차이점**: 환경에 따라 사용 가능한 API가 다릅니다:
  - **브라우저**: DOM API에 접근 가능하지만 로컬 파일 시스템 접근은 제한됩니다.
  - **Node**: 파일 시스템을 포함한 거의 모든 시스템 자원에 접근 가능하지만 DOM에는 접근하지 않습니다.

## 전역 객체

- **브라우저 전역 객체 (`window`)**: 브라우저 관련 기능에 접근할 수 있습니다:
  - DOM 조작 메서드.
  - 쿠키 관리.
  - 브라우저 이벤트 처리.

- **Node.js 전역 객체 (`global`)**: Node.js 고유 기능에 접근할 수 있습니다:
  - 로컬 파일 작업을 위한 내장 모듈.
  - 네트워킹 기능.
  - 운영체제와 상호작용.

## 버전 관리

- **브라우저 버전**: 사용자가 웹사이트를 방문할 때 사용하는 브라우저 환경 버전을 제어할 수 없습니다.
- **Node.js 버전**: 서버에서 사용할 Node.js 버전을 선택할 수 있습니다.

## Node.js의 중요성

- **풀스택 개발**: Node.js는 개발자가 JavaScript로 프론트엔드와 백엔드 모두를 구축할 수 있게 하여 웹 개발을 혁신했습니다.
- **효율성**: 백엔드 개발을 위해 새로운 프로그래밍 언어를 배울 필요가 없어 개발 과정이 더 효율적입니다.
- **영향력**: Node.js는 강력하고 확장 가능하며 다재다능한 도구로 전 세계 개발자에게 매우 중요해졌습니다.

## Node.js의 장점

- **풀스택을 위한 단일 언어**: 웹 애플리케이션의 프론트엔드와 백엔드를 모두 JavaScript로 구현할 수 있습니다.
  - 초기 학습 곡선을 줄입니다.
  - 프로그래밍 언어를 전환할 필요가 없습니다.
  - 효율성과 생산성을 높입니다.

- **논블로킹, 이벤트 기반 아키텍처**: 반응성과 효율성이 중요한 실시간 애플리케이션 개발에 적합합니다.

- **단일 스레드와 이벤트 루프**: 동시에 많은 요청과 입출력 작업을 효과적으로 처리할 수 있습니다.
  - 여러 요청을 동시에 처리해야 하는 애플리케이션에 완벽합니다.

- **큰 커뮤니티**: 전 세계 수천 명의 개발자가 Node.js를 지속적으로 유지 관리하고 새로운 기능을 추가합니다.
  - 많은 학습 자료가 있습니다.
  - 질문에 대한 답을 쉽게 찾을 수 있습니다.

- **npm (Node 패키지 매니저)**: 프로젝트용 패키지와 모듈을 설치하고 관리할 수 있는 강력한 도구입니다.
  - HTTP 요청 처리, 데이터베이스 작업, 날짜 및 시간 포맷, 코드 테스트 등 다양한 용도의 패키지가 있습니다.
  - 다른 개발자가 작성, 테스트, 공유한 코드를 재사용할 수 있습니다.
  - 작업 흐름을 더 빠르고 효율적으로 만듭니다.

- **비용 효율적**: Node.js는 무료이자 오픈 소스입니다.
  - 패키지, 라이브러리, 관련 도구도 주로 무료입니다.
  - 예산 최적화 시 중요한 요소입니다.

## Node.js의 단점

- **단일 스레드 제한**: 한 번에 하나의 스레드만 실행하므로 한 번에 하나의 작업만 처리할 수 있습니다.
  - 아키텍처는 동시 요청을 효율적으로 처리할 수 있지만, CPU 집약적인 작업은 메인 스레드를 차단해 성능 문제를 일으킬 수 있습니다.
  - CPU 집약적 작업 예: 복잡한 수학 연산, 이미지 및 비디오 처리, 암호화.
  - 이를 극복하는 방법이 있지만 보통 애플리케이션 복잡도가 증가합니다.

- **비동기 프로그래밍**: Node.js는 비동기 프로그래밍에 의존합니다.
  - 오래 걸릴 수 있는 작업을 시작하고 완료될 때까지 기다리지 않고 메인 프로그램이 계속 실행됩니다.
  - 작업이 완료되면 결과를 처리합니다.
  - 종종 비동기 작업 완료 시 실행되는 콜백 함수가 포함됩니다.
  - 코드가 더 읽기 어렵고 이해 및 디버그가 어려워질 수 있습니다.

- **패키지 품질 문제**: npm에서 패키지를 선택할 때 일부는 지속적으로 유지 관리되지 않아 애플리케이션에 취약점을 도입할 수 있습니다.
  - 개발자는 각 패키지를 신중히 평가해야 합니다.
  - 품질과 보안 모범 사례 준수 여부를 확인하세요.

## 스레드

- **스레드**: 현재 실행 중인 컴퓨터 프로그램 같은 프로세스 내 실행 경로입니다.

## NVM (Node 버전 관리자)

- **NVM**: 같은 컴퓨터에서 여러 Node.js 버전을 관리하는 도구입니다.
- **목적**: 프로젝트별로 다른 Node.js 버전 간 쉽게 전환할 수 있게 도와줍니다.
- **권장 사항**: 향후 유연성을 위해 Node.js 설치 전에 NVM을 설치하는 것이 좋습니다.

## macOS/Linux에서 NVM 설치

- **Xcode 명령줄 도구(macOS)**: macOS에서 NVM 설치 스크립트를 실행하기 전에 수동으로 설치해야 합니다.

```bash
xcode-select --install
```

- **Xcode 설치 확인**:

```bash
xcode-select --version
```

- **NVM 설치 스크립트 다운로드 및 실행**: 다음 명령 중 하나를 실행하세요:

```bash
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.3/install.sh | bash
```

```bash
wget -qO- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.3/install.sh | bash
```

- **터미널 재시작**: 프로세스 완료 후 새 명령과 구성을 불러오기 위해 터미널을 재시작해야 할 수 있습니다.

- **NVM 설치 확인**:

```bash
nvm -v
```

## macOS/Linux에서 NVM으로 Node.js 설치

- **최신 LTS 버전 설치**: 최신 장기 지원(LTS) 버전 Node.js를 설치하려면:

```bash
nvm install --lts
```

- **LTS (장기 지원)**: 안정성, 신뢰성, 보안을 우선시하여 새 기능보다 검증된 버전을 권장합니다.
  - 철저히 테스트되어 프로덕션에 적합합니다.
  - 버그 수정과 보안 패치에 중점을 둔 긴 지원 기간을 보장합니다.

- **특정 버전 설치**: 버전 번호를 지정해 특정 Node.js 버전을 설치할 수 있습니다:

```bash
nvm install <version>
```

예시:

```bash
nvm install 20
```

- **버전 전환**:

```bash
nvm use <version>
```

예시:

```bash
nvm use 20
```

- **현재 Node.js 버전 확인**:

```bash
node -v
```

## Windows에서 NVM 설치

- **공식 NVM 지원**: 공식 NVM은 특정 경우에만 Windows를 지원합니다.
  - Windows Subsystem for Linux(WSL)가 있다면 위 macOS/Linux 가이드를 따르세요.

- **nvm-windows**: Windows용 Node.js 버전 관리 유틸리티입니다.
  - 공식 NVM 프로젝트와 완전히 별개입니다.
  - 목적은 동일: 같은 장치에서 여러 Node.js 버전을 관리할 수 있게 돕습니다.

- **설치 전**: Windows용 NVM 설치 전에 기존 Node 설치를 제거하는 것이 좋습니다.

- **설치 단계**:
  - 공식 저장소의 Releases 섹션으로 이동: `https://github.com/coreybutler/nvm-windows/releases`.
  - 최신 설치 프로그램 다운로드.
  - 안내에 따라 설치하세요.

- **터미널 재시작**: `nvm` 명령이 바로 사용 불가하면 터미널을 재시작하세요.

- **설치 확인**:

```powershell
nvm -v
```

## 기본 NVM 및 Node.js 명령어

- **명령어 참고**: 모든 명령어는 macOS/Linux 및 WSL에 설치된 NVM에서 작동합니다. `nvm-windows`에서는 다르게 작동하거나 작동하지 않을 수 있습니다. `nvm-windows`에 대한 대체 명령어가 가능한 경우 제공됩니다.

### PowerShell 실행 정책 (Windows)

- **기본 실행 정책**: PowerShell 기본 실행 정책은 `Restricted`로, 보안상 스크립트 실행을 차단합니다.

- **관리자 권한으로 PowerShell 실행**: PowerShell 아이콘을 우클릭하고 "관리자 권한으로 실행"을 클릭하세요.

- **현재 실행 정책 확인**:

```powershell
Get-ExecutionPolicy
```

- **실행 정책을 RemoteSigned로 설정**: 로컬 스크립트와 `npm` 실행을 허용합니다:

```powershell
Set-ExecutionPolicy RemoteSigned
```

- **정책 설정 후**: 관리자 PowerShell 창을 닫고 일반 PowerShell 창을 엽니다.

### 설치된 모든 버전 목록

- **설치된 버전 목록 확인**: 설치된 모든 Node.js 버전을 나열하고 현재 사용 중인 버전을 확인하려면:

```bash
nvm ls
```

### 기본 Node.js 버전 설정 (macOS/Linux NVM 전용)

- **기본 버전 설정**: 새 터미널 세션에서 사용할 기본 Node.js 버전을 설정합니다:

```bash
nvm alias default <version>
```

예시:

```bash
nvm alias default 22.20.0
```

- **사용자 지정 별칭 생성**: 버전에 더 구체적인 이름을 할당합니다:

```bash
nvm alias <new_name> <version>
```

예시:

```bash
nvm alias new-api 18.20.8
```

- **사용자 지정 별칭 사용**:

```bash
nvm use <new_name>
```

예시:

```bash
nvm use new-api
```

### Node.js 버전 제거

- **버전 제거**:

```bash
nvm uninstall <version>
```

예시:

```bash
nvm uninstall 20.19.5
```

- **주요 버전별 제거 (macOS/Linux NVM)**: `uninstall` 명령어로 주요 버전(예: `22`, `20`)을 제거할 수 있습니다:

```bash
nvm uninstall 20
```

해당 주요 버전의 여러 릴리스가 설치되어 있으면 `20.14.0`, `20.12.2`처럼 최신 버전부터 제거됩니다. `nvm uninstall 20` 명령어는 `20.14.0`부터 먼저 제거합니다.

### Node.js 버전 확인

- **현재 버전 확인**:

```bash
node -v
```

### npm 버전 확인

- **npm**: Node.js용 패키지 매니저로, 소프트웨어 패키지 설치, 게시, 관리를 담당합니다. Node.js 설치 시 자동으로 설치됩니다.

- **npm 버전 확인**:

```bash
npm -v
```

### JavaScript 파일 실행

- **JavaScript 파일 실행**: 현재 활성화된 Node.js 런타임에서 JavaScript 파일을 실행합니다:

```bash
node <file.js>
```

예시:

```bash
node app.js
```

### Node.js 프로젝트 생성

- **프로젝트 초기화**: 프로젝트 세부 정보와 의존성을 추적하는 `package.json` 파일을 생성하여 새 Node.js 프로젝트를 만듭니다:

```bash
npm init
```

- **대화형 설정**: `npm init` 명령어는 `package.json` 파일 설정을 위한 대화형 명령줄 가이드를 제공합니다.
  - 프로젝트 이름, 설명, 버전 등을 묻습니다.

- **기본 설정**: 기본 `package.json` 파일로 시작하려면:

```bash
npm init -y
```

또는

```bash
npm init --yes
```

```bash
npm install <package>
```

```bash
npm install express
```
