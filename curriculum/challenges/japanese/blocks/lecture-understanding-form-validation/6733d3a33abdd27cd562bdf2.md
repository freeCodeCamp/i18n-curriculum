# --interactive--

イベントの `preventDefault()` メソッドの目的について学びましょう。

DOMで発生するすべてのイベントには、何らかのデフォルトの動作があります。チェックボックスの `click` イベントは、デフォルトでそのチェックボックスの状態を切り替えます。フォーカスされたボタンでスペースキーを押すと、そのボタンがアクティブになります。これらのイベントオブジェクトの `preventDefault()` メソッドは、その動作が起こるのを止めます。

例を見てみましょう。ユーザーが入力できる `input` 要素を定義します。

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<label>Enter some characters:
  <input type="text">
</label>
```

```css
label {
  display: flex;
  flex-direction: column;
  font-family: Arial, sans-serif;
  font-size: 14px;
  color: #333;
  margin-bottom: 12px;
}

input[type="text"] {
  padding: 8px 12px;
  border: 1px solid #ccc;
  border-radius: 4px;
  font-size: 14px;
  margin-top: 4px;
  width: 100%;
  box-sizing: border-box;
  transition: border-color 0.2s, box-shadow 0.2s;
}

input[type="text"]:focus {
  border-color: #0078d4;
  box-shadow: 0 0 3px rgba(0, 120, 212, 0.5);
  outline: none;
}

```

:::

結果を見ると、期待通りに入力フィールドに文字を入力できます。でも、そうしたくない場合もあります。代わりに、ユーザーが入力した文字を別の要素に表示したいかもしれません。まず、そのための要素を定義しましょう。

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

  <input type="text">
</label>
<p id="output"></p>
```

```css

  flex-direction: column;
  align-items: center;

  margin: 0;

  font-size: 1.2rem;
  color: #333;
  margin-bottom: 1rem;
  display: flex;
  flex-direction: column;
}

input[type="text"] {
  padding: 0.5rem;
  font-size: 1rem;
  border: 2px solid #ccc;
  border-radius: 5px;
  margin-top: 0.5rem;
  outline: none;
}

input[type="text"]:focus {
  border-color: #007BFF;
}

  margin-top: 1rem;
  font-size: 1.1rem;
  color: #555;
}

```

:::

次に、キーボードで文字が入力されたことを検知するために `keydown` イベントにフックする必要があります。ここでは `change` や `input` イベントは使いません。キーボードの情報が必要だからです。

```js
const input = document.querySelector("input");

input.addEventListener("keydown", (e) => {
    
})
```

`keydown` イベントはキーボードのキーを押したときに発火します。これが起きたら、`p` 要素にその文字を表示しましょう。

```js
const input = document.querySelector("input");
const output = document.getElementById("output");

input.addEventListener("keydown", (e) => {
  output.innerText = `You pressed the ${e.key} key`;
});
```

:::

`e.key` は押されたキーの値を教えてくれます。例えば、`a` キーなら `a`、`Enter` キーなら `Enter` です。

上記のコードでは、`input` に文字を入力すると、入力した文字が `p` 要素に表示されます。

これは便利ですが、`input` にも文字が表示されてしまいます。ここで `preventDefault()` メソッドの出番です。`keydown` のデフォルトの動作は、入力フィールドに文字をレンダリングすることです。`e.preventDefault()` を呼び出してそれを防ぎましょう。

```js
const input = document.querySelector("input");
const output = document.getElementById("output");

input.addEventListener("keydown", (e) => {
  e.preventDefault();
  output.innerText = `You pressed the ${e.key} key`;
});
```

:::

こうして、デフォルトの動作を防いで、自分でカスタムなイベント処理を実装できるようになりました。

`e.preventDefault` メソッドを使うもう一つのよくある例はフォームの提出です。デフォルトでは、フォームを提出するとサーバーにデータが送信され、ページがリロードされます。`e.preventDefault()` を使うとこれを防げます。

```js
const form = document.querySelector("form");

form.addEventListener("submit", (e) => {
  e.preventDefault();
  // rest of code goes here
});
```

デフォルトの動作を防ぐのは、ユーザーのページとのやり取りをより細かく制御したいときに便利ですが、アクセシビリティのような点も考慮することが重要です。カスタムの動作でもデフォルトと同じ機能を提供する必要があります。
