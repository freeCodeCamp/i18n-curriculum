# --description--

前のレッスンでは、比較演算子と条件文を使ってプログラムの流れを制御する方法を学びました。

これらは非常に強力ですが、複数の値を一度に比較する必要がある状況にしばしば遭遇します。これにより、例えばネストされた条件文が発生することがあります。

```python
is_citizen = True
age = 25

if is_citizen:
    if age >= 18:
        print('You are eligible to vote') # You are eligible to vote
else:
    print('You are not eligible to vote')
```

上の例では、まず`is_citizen`が`True`かどうかをチェックします。もしそうなら、ネストされた`if`文に進み、`age`が`18`以上かどうかを確認します。`age`が`18`以上なので、端末に出力されるメッセージは`You are eligible to vote`になります。もし`is_citizen`が`False`だった場合は、端末に出力されるメッセージは`You are not eligible to vote`になっていたでしょう。

より複雑な条件文を扱う場合は、Pythonの`and`、`or`、および`not`演算子を使うことができます。

しかし、その演算子に入る前に、真値と偽値とは何かを見てみましょう。

Pythonでは、すべての値に固有のブール値、つまり論理的な文脈で`True`または`False`として扱われるべきかどうかの組み込みの感覚があります。多くの値は**真値**と見なされ、論理的な文脈で`True`に評価されます。その他は**偽値**で、`False`に評価されます。

いくつかの偽値を紹介します：

- `None`
- `False`
- 整数の`0`
- 浮動小数点数の`0.0`
- 空の文字列`""`

非ゼロの数値や空でない文字列などの他の値は真値です。

値が真値か偽値かを確認したい場合は、組み込みの`bool()`関数を使うことができます。これは値を明示的にブール値に変換し、真値の場合は`True`を、偽値の場合は`False`を返します。いくつかの例を示します：

```py
print(bool(False)) # False
print(bool(0))  # False
print(bool('')) # False

print(bool(True)) # True
print(bool(1)) # True
print(bool('Hello')) # True
```

真値と偽値が理解できたので、論理演算子とも呼ばれるブール演算子を見てみましょう。これらは複数の式を組み合わせて、コード内でより複雑な意思決定ロジックを作成できる特別な演算子です。

Pythonには3つのブール演算子があります：`and`、`or`、および`not`です。

まず`and`演算子を見てみましょう。

`and`演算子は2つのオペランドを取り、最初のオペランドが偽値ならそれを返し、そうでなければ2番目のオペランドを返します。両方のオペランドが真値でなければ、式は真値になりません。

例を示します：

```python
is_citizen = True
age = 25

print(is_citizen and age) # 25
```

上の例では、25が端末に出力されます。なぜなら`and`演算子は最初のオペランドが`True`の場合に2番目のオペランドを評価するからです。`and`演算子はショートサーキット演算子として知られています。ショートサーキットとは、Pythonが左から右へ値をチェックし、最終結果が決まった時点で評価をストップすることを意味します。

`and`は`if`文内で複数の条件が満たされているかをチェックするためによく使われます。先ほどの例をネストされた`and`文の代わりに`if`演算子を使って書き換える方法は次の通りです：

```python
is_citizen = True
age = 25

if is_citizen and age >= 18:
    print('You are eligible to vote') # You are eligible to vote
else:
    print('You are not eligible to vote')
```

上の例では、`is_citizen`は`True`で、`age >= 18`は`True`に評価されます。`and`演算子の両方のオペランドが真値なので、条件`is_citizen and age >= 18`は`True`に評価され、`print`ブロック内の`if`呼び出しが実行されます。

次に`or`演算子を見てみましょう。この演算子は最初のオペランドが真値ならそれを返し、そうでなければ2番目のオペランドを返します。`or`式は少なくとも1つのオペランドが真値であれば真値になります。`or`演算子もショートサーキット演算子として知られています。例を示します：

```python
age = 19
is_employed = False

print(age or is_employed) # 19
```

次のコードは19を出力します。なぜなら最初のオペランド`age`が`True`だからです。

1つ以上の式が`True`かどうかをチェックする必要がある場合は、次のように条件文で`or`演算子を使うことができます：

```python
age = 19
is_student = True

if age < 18 or is_student:
    print('You are eligible for a student discount') # You are eligible for a student discount
else:
    print('You are not eligible for a student discount')
```

この場合、`age < 18`は`False`ですが、`is_student`は`True`です。少なくとも1つの条件が真なので、全体の`or`式は`True`に評価され、`if`ブロック内の割引メッセージが出力されます。

最後に見ていくのは`not`演算子で、これは単一のオペランドを取り、そのブール値を反転させます。真値を`False`に、偽値を`True`に変換します。これまで見てきた演算子とは異なり、`not`は常に`True`か`False`を返します。

いくつか例を示します：

```python
print(not '') # True, because empty string is falsy
print(not 'Hello') # False, because non-empty string is truthy
print(not 0) # True, because 0 is falsy
print(not 1) # False, because 1 is truthy
print(not False) # True, because False is falsy
print(not True) # False, because True is truthy
```

`not`演算子は、何かが`True`または`False`でないかを条件文でチェックするのによく使われます。例えば次のように：

```python
is_admin = False

if not is_admin:
    print('Access denied for non-administrators.') # Access denied for non-administrators.
else:
    print('Welcome, Administrator!')
```

`is_admin`が`False`なので、`not is_admin`は`False`ではない、つまり`True`と言っています。したがってメッセージ`Access denied for non-administrators.`が出力されます。

真値と偽値、`and`、`or`、`not`演算子、そしてショートサーキットの仕組みが理解できたので、より柔軟で読みやすい条件ロジックを書けるようになります。
