# --description--

ターミナルとは何か、そしてコマンドラインとどう違うのかを学びましょう。

コマンドラインは、ユーザーが「コマンド」を入力し、通常は「Enter」キーを押して送信または実行できる基本的なテキスト入力インターフェースです。コマンドラインインターフェースは、最も一般的にターミナル内で見られます。

ターミナルは、基本的な読み書き操作を超えたシステムレベルのコマンドを実行するためのコマンドラインインターフェースを提供する特別なアプリケーションです。「ターミナルエミュレーター」という言葉も聞くかもしれませんが、これは基本的なターミナルインターフェースをラップして追加の機能や機能性を提供するアプリケーションのことです。しかし、一般的な用途では、これらも「ターミナル」と呼んで問題ありません。

最後に、「シェル」についても説明が必要です。シェルはコマンドラインをラップし、入力をコマンドとして解釈して出力を返すソフトウェアです。

では、システム上でターミナルにどうやってアクセスできるでしょうか？

もし最新のWindowsを使っているなら、2つの異なる選択肢があるでしょう。Windowsは標準で2つの異なるシェルアプリケーションを提供しています：Powershellとコマンドプロンプトです。

これらのシェルはそれぞれの名前を持つターミナルアプリケーションとセットになっているので、アプリケーションメニューからPowershellターミナルかコマンドプロンプトターミナルのどちらかを起動できます。

Windowsはまた、Microsoft Terminalという比較的新しくモダンなターミナルエミュレーターも提供しています。

このアプリケーションはWindowsストアからインストールでき、Powershellやコマンドプロンプトのターミナルと同じように起動できます。Microsoft Terminalの利点は、同じアプリケーションからPowershellとコマンドプロンプトの両方のシェル、さらにWindows Subsystems for Linuxを設定していればLinuxのシェルにもアクセスできることです。

MacOSは「Terminal」と名付けられたターミナルを提供しています。スポットライト検索やアプリケーションランチャーからアクセスできます。

MacOSはまた、iTermやGhosttyのようなサードパーティ製ターミナルエミュレーターもサポートしています。これらはそれぞれのホームページからインストールできます。

Linuxでは、デフォルトのターミナルや起動方法はディストリビューションやデスクトップ環境によって大きく異なります。サードパーティ製のターミナルエミュレーターも多くサポートされています。例えば、Arch Linuxでkittyを設定し、wofiのようなアプリケーションマネージャーで起動することもできます。

Linuxはおそらく最も多様なターミナルアプリケーションの選択肢があります。ほとんどはディストリビューションのパッケージマネージャーからインストール可能です。

「ターミナル」「シェル」「コマンドライン」の区別は時に細かく感じるかもしれません。ほとんどの場合、これらの用語はほぼ同じ意味で使われます。しかし、それぞれに特定の意味があり、その意味を知っておくことは重要です。
