# --description--

Express에서는 `app.route()` 메서드를 사용해 **체인 가능한 라우트 핸들러**를 만들 수 있습니다. 이는 같은 라우트 경로에 대해 GET, POST, PUT 같은 여러 HTTP 메서드를 각각 경로를 반복하지 않고 처리할 때 유용합니다.

`app.get("/user")`, `app.post("/user")`, `app.put("/user")` 같은 별도의 라우트 정의를 작성하는 대신, `app.route()`를 사용하면 이들을 하나의 깔끔한 코드 블록으로 묶을 수 있습니다.

`app.route()` 메서드는 같은 엔드포인트에 대해 여러 핸들러를 정의하는 지름길 역할을 합니다. 이는 RESTful API에서 단일 리소스(예: `/user`)가 여러 작업을 지원할 때 특히 유용합니다.

다음은 사용자를 관리하는 RESTful API를 시뮬레이션한 예입니다:

```javascript
const express = require("express");
const app = express();
const port = 3000;

app.use(express.json()); // Parse JSON request bodies

// Use app.route() to group handlers for the same path
app
  .route("/user")
  .get((req, res) => {
    res.send("Fetching user data");
  })
  .post((req, res) => {
    const { name } = req.body;
    res.send(`Creating a user with name: ${name}`);
  })
  .put((req, res) => {
    const { name } = req.body;
    res.send(`Updating the user to: ${name}`);
  })
  .delete((req, res) => {
    res.send("Deleting the user");
  });

app.listen(port, () => {
  console.log(`Server running at http://localhost:${port}`);
});
```

위 예제에서 `app.route("/user")`는 `/user` 경로에 대해 단일 라우트를 정의합니다. 그런 다음 여러 HTTP 메서드인 `.get()`, `.post()`, `.put()`, `.delete()`를 체인으로 연결해 같은 라우트에 대한 다양한 요청을 처리합니다.

간단한 RESTful 애플리케이션에서 각 라우트가 어떻게 작동하는지 살펴보면 다음과 같습니다:

* GET `/user`: 사용자 데이터를 가져와 반환합니다.
    
* POST `/user`: 요청 본문에 있는 데이터로 새 사용자를 생성합니다.
    
* PUT `/user`: 요청 본문에 있는 데이터로 기존 사용자를 업데이트합니다.
    
* DELETE `/user`: 사용자를 삭제합니다.
    

`app.route()`를 사용하면 같은 경로에 대해 여러 HTTP 메서드를 처리할 때 코드를 깔끔하게 정리할 수 있습니다. 매번 라우트를 반복하는 대신 관련 핸들러를 한곳에 모아둡니다.

이렇게 하면 중복이 줄고 코드를 읽고 유지 관리하기 쉬워집니다. 모든 메서드가 한 블록에 정의되면 해당 엔드포인트가 어떻게 동작하는지 훨씬 명확해집니다.

애플리케이션이 커지고 라우트가 복잡해질수록 `app.route()`는 관련 로직을 함께 묶는 간단한 구조를 제공합니다. 특히 GET, POST, PUT, DELETE 같은 여러 작업을 지원하는 단일 리소스를 설계할 때 유용합니다.
