# --description--

이전 수업에서는 `localStorage`를 다루었고 `sessionStorage`에 대해 간단히 소개했습니다.

`sessionStorage`는 웹 애플리케이션이 실행되는 탭이나 창을 사용자가 닫으면 데이터가 즉시 지워지는 것을 의미합니다. 이는 탐색 중 폼 데이터를 유지하거나 결제 과정 중 임시 상태 정보를 저장하는 등 단일 세션 동안만 데이터가 유지되어야 하는 상황에 이상적입니다.

`localStorage`와 마찬가지로 `sessionStorage`도 키-값 쌍을 사용해 데이터를 저장하고 가져옵니다. `sessionStorage`에서 사용하는 메서드도 `localStorage`와 동일하며, 실제 차이는 데이터가 저장되는 기간뿐입니다.

다음은 다양한 메서드를 사용하는 몇 가지 예시입니다:

- `sessionStorage.setItem()`: `sessionStorage`에 키-값 쌍을 저장합니다.

```js
sessionStorage.setItem('cart', '3 items');
```

- `sessionStorage.getItem()`: `sessionStorage`에서 주어진 키의 값을 가져옵니다.

```js
let cart = sessionStorage.getItem('cart');
console.log(cart); // Outputs: '3 items'
```

- `sessionStorage.removeItem()`: 키를 사용해 `sessionStorage`에서 특정 항목을 제거합니다.

```js
sessionStorage.removeItem('cart');
```

- `sessionStorage.clear()`: `sessionStorage`에 저장된 모든 데이터를 지웁니다.

```js
sessionStorage.clear();
```

이제 브라우저 탭이나 창이 열려 있는 동안에만 지속되는 `sessionStorage`에 데이터를 저장하는 예를 살펴보겠습니다:

```js
// Store data in sessionStorage
sessionStorage.setItem('currentUser', 'JohnDoe');

// Retrieve the stored data
const user = sessionStorage.getItem('currentUser');
console.log(user); // 'JohnDoe'

// Remove a specific key from sessionStorage
sessionStorage.removeItem('currentUser');

// Clear all sessionStorage data
sessionStorage.clear();
```

이 예시에서는 다음을 수행합니다:

1. 현재 사용자의 이름(`JohnDoe`)을 `sessionStorage`에 저장합니다.

2. 이를 가져와서 표시합니다.

3. `currentUser` 키와 연관된 항목을 제거합니다.

4. `sessionStorage`의 모든 데이터를 지웁니다.

`localStorage`와의 주요 차이점은 사용자가 탭을 닫으면 저장된 세션 데이터가 모두 사라진다는 점입니다.

`sessionStorage`는 다음과 같은 상황에서 특히 유용합니다:

- 다중 페이지 폼 과정 중 폼 항목과 같은 임시 데이터를 저장할 때.

- 세션 간에 유지할 필요가 없는 임시 선택이나 설정을 저장할 때.

- 탭이 닫히면 기억할 필요가 없는 단일 페이지 애플리케이션의 상태를 유지할 때.

`sessionStorage`는 사용자가 페이지를 떠나면 세션 데이터를 지우므로 현재 세션을 넘어 정보를 보관하고 싶지 않은 상황에 적합합니다.
