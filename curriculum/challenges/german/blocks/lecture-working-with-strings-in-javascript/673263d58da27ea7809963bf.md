# --interactive--

In JavaScript sind Template-Literale eine leistungsstarke und flexible Möglichkeit, mit Strings zu arbeiten. Im Gegensatz zu normalen Strings, die einfache (`'`) oder doppelte (`"`) Anführungszeichen verwenden, werden Template-Literale mit Backticks (`` ` ``) definiert.

Sie ermöglichen eine einfachere String-Manipulation, einschließlich des direkten Einbettens von Variablen in einen String, eine Funktion, die als String-Interpolation bekannt ist.

Template-Literale erleichtern es, Strings zu erstellen, die sich über mehrere Zeilen erstrecken oder Ausdrücke (wie Variablen oder sogar JavaScript-Code) direkt im String enthalten.

Hier ist ein Beispiel für ein Template-Literal:

:::interactive_editor

```js
const name = "Alice";
const greeting = `Hello, ${name}!`;

console.log(greeting);
```

:::

Beachten Sie die Verwendung von Backticks anstelle von einfachen oder doppelten Anführungszeichen. Die `${name}`-Syntax ist ein Beispiel für String-Interpolation, bei der der Wert der Variablen `name` direkt in den String eingefügt wird.

String-Interpolation ermöglicht es Ihnen, Variablen und Ausdrücke in einen String einzubetten. Dies ist eine bedeutende Verbesserung gegenüber der älteren Methode, bei der Sie Strings und Variablen mit dem `+`-Operator verketten mussten.

Hier ist ein Beispiel mit String-Verkettung und dem Plus-Operator (`+`):

:::interactive_editor

```js
const name = "Alice";
const age = 25;
const message = "My name is " + name + " and I am " + age + " years old.";
console.log(message); 
```

:::

Und hier ist ein Beispiel mit Template-Literalen und String-Interpolation:

:::interactive_editor

```js
const name = "Alice";
const age = 25;
const message = `My name is ${name} and I am ${age} years old.`;
console.log(message); 
```

:::

Wie Sie sehen, ist die String-Interpolation mit Template-Literalen viel sauberer und leichter zu lesen, besonders wenn Sie mit mehreren Variablen arbeiten.

Eine weitere großartige Funktion von Template-Literalen ist, dass sie mehrzeilige Strings unterstützen. Bei normalen Strings müssten Sie Escape-Zeichen (`\n`) verwenden, um neue Zeilen zu erzeugen. Mit Template-Literalen können Sie den String einfach über mehrere Zeilen schreiben, und die Formatierung bleibt erhalten:

:::interactive_editor

```js
let poem = `Roses are red,
Violets are blue,
JavaScript is fun,
And so are you.`;

console.log(poem);
```

:::

Eine weitere Funktion von Template-Literalen ist, dass Sie JavaScript-Ausdrücke direkt im String einbetten können, wie in diesem Beispiel:

:::interactive_editor

```js
const song = "Bohemian Rhapsody";
const score = 9.5;
const highestScore = 10;
const output = `One of my favorite songs is "${song}". I rated it ${
  (score / highestScore) * 100
}%.`;
console.log(output); 
```

:::

Template-Literale sind besonders nützlich, wenn Sie Variablen oder Ausdrücke in Strings einfügen, komplexe Strings formatieren oder mit mehrzeiligem Text arbeiten müssen. Sie sind prägnanter und besser lesbar als die traditionelle String-Verkettung.
