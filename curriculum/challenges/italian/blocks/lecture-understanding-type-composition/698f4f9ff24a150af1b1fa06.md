# --description--

Nelle lezioni precedenti, hai imparato a lavorare con i tipi primitivi, i tipi unione e le interfacce.

Ma TypeScript ha alcuni tipi speciali di cui dovresti essere consapevole. Iniziamo dando un’occhiata al tipo `any`.

Il tipo `any` viene usato per rappresentare qualsiasi tipo di valore.

```ts
let randomValue: any;

randomValue = 42;           

randomValue = "Hello";     
 
randomValue = true;        

randomValue = { name: "Alice" }; 
```

Con questo tipo, devi fare attenzione perché può essere facilmente usato eccessivamente o in modo errato. Non vuoi tipizzare tutto con il tipo `any` solo per far tacere i messaggi di errore di TypeScript. Questo vanifica lo scopo di usare TypeScript e aggiungere sicurezza di tipo fin dall’inizio.

Un equivalente più sicuro del tipo `any` sarebbe il tipo `unknown`. `unknown` è simile a `any` ma con il tipo `unknown` devi fare un controllo del tipo della variabile prima di usarla.

Ecco un esempio:

```ts
function doubleValue(value: unknown) {
  if (typeof value === "number") {
    console.log(value * 2);
  } else if (typeof value === "string") {
    console.log(value + value);
  }
}

doubleValue(10);    
doubleValue("Hi "); 
doubleValue(true);   
```

In questo esempio, i controlli di tipo assicurano che le operazioni vengano eseguite solo su valori del tipo corretto. I valori che non corrispondono ai tipi attesi vengono ignorati.

Il prossimo tipo che vedremo è il tipo `never`. Questo tipo rappresenta qualcosa che non accadrà mai.

Nella maggior parte delle situazioni, non scriverai annotazioni di tipo usando `never`. Invece, probabilmente vedrai il tipo `never` comparire nei messaggi di errore come questo:

```ts
function processValue(value: string | number) {
  if (typeof value === "string") {
    console.log("String value:", value.toUpperCase());
  } else if (typeof value === "number") {
    console.log("Number value:", value * 2);
  } else {
    console.log(value); 
  }
}
```

All’interno della clausola `else` dell’esempio sopra, `value` ha un tipo `never`. Questo perché il ramo `else` ora è impossibile da raggiungere e non rimane alcun tipo. Se provassi a chiamare la funzione con un valore diverso da `string` o `number` mostrerebbe un errore come questo:

```ts
function processValue(value: string | number) {
  if (typeof value === "string") {
    console.log("String value:", value.toUpperCase());
  } else if (typeof value === "number") {
    console.log("Number value:", value * 2);
  } else {
    console.log(value); 
  }
}

// Argument of type 'boolean' is not assignable to parameter of type 'string | number'.
processValue(true); 
```

L’ultimo tipo che vedremo è il tipo `void`. Quando hai una funzione che non restituisce un valore, quello sarebbe un tipo `void`.

Ecco un esempio:

```ts
type Status = "loading" | "success" | "error";

type Handler = {
  status: Status;
  onChange: (newStatus: Status) => void; 
};
```

In questo esempio, la proprietà `onChange` per il tipo `Handler` non restituisce nulla ed è per questo che si usa `void`.
