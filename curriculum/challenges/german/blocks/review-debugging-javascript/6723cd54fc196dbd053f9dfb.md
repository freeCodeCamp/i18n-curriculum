## Häufige Arten von Fehlermeldungen

- **SyntaxError**: Diese Fehler treten auf, wenn Sie etwas falsch in Ihrem Code schreiben, zum Beispiel eine Klammer oder eine eckige Klammer vergessen. Man kann es sich vorstellen wie einen Grammatikfehler in einem Satz.

```js
const arr = ["Beau", "Quincy" "Tom"]
```

- **ReferenceError**: Es gibt verschiedene Arten von Reference Errors, die auf unterschiedliche Weise ausgelöst werden. Die erste Art von Reference Error ist eine nicht definierte Variable. Ein weiteres Beispiel für einen ReferenceError ist der Versuch, auf eine Variable zuzugreifen, die mit `let` oder `const` deklariert wurde, bevor sie definiert wurde.

:::interactive_editor

```js
console.log(num);
const num = 50;
```

:::

- **TypeError**: Diese Fehler treten auf, wenn Sie versuchen, eine Operation mit einem falschen Typ auszuführen.

```js
const developerObj = {
  name: "Jessica",
  country: "USA",
  isEmployed: true
};

developerObj.map()
```

- **RangeError**: Diese Fehler entstehen, wenn Ihr Code versucht, einen Wert zu verwenden, der außerhalb des Bereichs liegt, den JavaScript verarbeiten kann.

```js
const arr = [];
arr.length = -1; 
```

## Die `throw`-Anweisung

- **Definition**: Die `throw`-Anweisung in JavaScript wird verwendet, um eine benutzerdefinierte Ausnahme zu werfen. Eine Ausnahme in der Programmierung ist ein unerwartetes Ereignis, das den normalen Ablauf des Programms unterbricht.

:::interactive_editor

```js
function validateNumber(input) {
  if (typeof input !== "number") {
    throw new TypeError("Expected a number, but received " + typeof input);
  }
  return input * 2;
}

console.log(validateNumber("Naomi"));   // TypeError: Expected a number, but received string
```

:::

## `try...catch...finally`

- **Definition**: Der `try`-Block wird verwendet, um Code einzuschließen, der einen Fehler werfen könnte. Er dient als sicherer Bereich, um etwas auszuprobieren, das fehlschlagen könnte. Der `catch`-Block fängt Fehler ab und behandelt sie, die im try-Block auftreten. Sie können das Fehlerobjekt innerhalb von catch verwenden, um zu untersuchen, was schiefgelaufen ist. Der `finally`-Block wird nach den try- und catch-Blöcken ausgeführt, unabhängig davon, ob ein Fehler aufgetreten ist. Er wird häufig für Aufräumarbeiten verwendet, wie das Schließen von Dateien oder das Freigeben von Ressourcen.

:::interactive_editor

```js
function processInput(input) {
  if (typeof input !== "string") {
    throw new TypeError("Input must be a string.");
  }

  return input.toUpperCase();
}

try {
  console.log("Starting to process input...");
  const result1 = processInput("hello");
  console.log("Processed result:", result1);  // HELLO
  const result2 = processInput(9);  // throws TypeError
  console.log("Processed result:", result2);  // not executed
} catch (error) {
  console.error("Error occurred:", error.message);
} 
```

:::

## Debugging-Techniken

- **`debugger`-Anweisung**: Diese Anweisung ermöglicht es Ihnen, Ihren Code an einer bestimmten Zeile anzuhalten, um zu untersuchen, was im Programm vor sich geht.

:::interactive_editor

```js
let firstNumber = 5;
let secondNumber = 10;

debugger; // Code execution pauses here
let sum = firstNumber + secondNumber;

console.log(sum);
```

:::

- **Breakpoints**: Breakpoints ermöglichen es Ihnen, die Ausführung Ihres Codes an einer bestimmten Zeile Ihrer Wahl anzuhalten. Nach der Pause können Sie Variablen inspizieren, Ausdrücke auswerten und den Aufrufstapel untersuchen.
- **Watchers**: Watch-Ausdrücke erlauben es Ihnen, die Werte von Variablen oder Ausdrücken während der Codeausführung zu überwachen, auch wenn sie außerhalb des aktuellen Gültigkeitsbereichs liegen.
- **Profiling**: Profiling hilft Ihnen, Leistungsengpässe zu identifizieren, indem Sie Screenshots aufnehmen und CPU-Auslastung, Funktionsaufrufe und Ausführungszeit aufzeichnen.
- **`console.dir()`**: Diese Methode wird verwendet, um eine interaktive Liste der Eigenschaften eines angegebenen JavaScript-Objekts anzuzeigen. Sie gibt eine hierarchische Auflistung aus, die erweitert werden kann, um alle verschachtelten Eigenschaften zu sehen.

```js
console.dir(document);
```

- **`console.table()`**: Diese Methode zeigt tabellarische Daten als Tabelle in der Konsole an. Sie nimmt ein obligatorisches Argument, das ein Array oder ein Objekt sein muss, und ein optionales Argument, um anzugeben, welche Eigenschaften (Spalten) angezeigt werden sollen.
