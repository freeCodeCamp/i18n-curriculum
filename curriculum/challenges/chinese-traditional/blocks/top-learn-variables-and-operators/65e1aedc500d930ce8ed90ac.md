# --description--

如果一個表達式有多個運算子，執行順序由它們的優先權定義，換句話說，就是運算子的預設優先順序。

從學校你就知道，表達式 `1 + 2 * 2` 中的乘法應該在加法之前計算。這正是優先的概念。乘法被認為比加法具有較高的優先。

小括弧會改寫任何優先，因此如果你對預設值的順序不滿意，你可以使用它們來改變順序。例如，寫 `(1 + 2) * 2`。

JavaScript 中有許多運算子。每個運算子都有對應的優先數字。數字較大的會先執行。如果優先相同，執行順序是從左到右。

以下是優先表的摘錄（你不需要記住這個，但請注意一元運算子的優先高於相應的二進位運算子）：

| Precedence | Name             | Sign |
|------------|------------------|------|
| ...        | ...              | ...  |
| 14         | unary plus       | +    |
| 14         | unary negation   | -    |
| 13         | exponentiation   | **   |
| 12         | multiplication   | *    |
| 12         | division         | /    |
| 11         | addition         | +    |
| 11         | subtraction      | -    |
| ...        | ...              | ...  |
| 2          | assignment       | =    |
| ...        | ...              | ...  |



# --questions--

## --text--

當在 JavaScript 表達式中處理多個運算子時，關於執行順序哪個述語是正確的？

## --answers--

運算子不論其優先順序，皆由右至左執行。

---

優先較高的運算子會先執行，而小括弧可以改寫預設的優先。

---

運算子的優先由運算子執行的速度決定。

---

乘法總是比加法具有較高的優先，一元運算子在二進位運算子之後執行。
