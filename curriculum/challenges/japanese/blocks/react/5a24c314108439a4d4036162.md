# --description--

コンポーネントはReactの核となる部分です。Reactのすべてはコンポーネントで構成されており、ここではその作り方を学びます。

Reactコンポーネントを作成する方法は2つあります。最初の方法はJavaScriptの関数を使うことです。この方法でコンポーネントを定義すると、*ステートレス関数コンポーネント*が作成されます。アプリケーションの状態（state）の概念は後のチャレンジで扱います。今は、ステートレスコンポーネントを、データを受け取ってレンダリングはするけれど、そのデータの変更を管理・追跡しないものと考えてください。（Reactコンポーネントを作成する2つ目の方法は次のチャレンジで扱います。）

関数でコンポーネントを作るには、JSXか`null`を返すJavaScript関数を書くだけです。重要な点は、Reactでは関数名を大文字で始める必要があることです。以下は、JSXでHTMLのクラスを割り当てたステートレス関数コンポーネントの例です。

```jsx
const DemoComponent = function() {
  return (
    <div className='customClass' />
  );
};
```

トランスパイル後、`<div>`は`customClass`というCSSクラスを持ちます。

JSXコンポーネントはHTMLを表すので、複数のコンポーネントを組み合わせてより複雑なHTMLページを作ることができます。これがReactのコンポーネントアーキテクチャの大きな利点の一つです。多くの独立したコンポーネントからUIを構成できるため、複雑なユーザーインターフェースの構築や保守が容易になります。

# --instructions--

コードエディターには`MyComponent`という関数があります。この関数を完成させて、文字列を含む単一の`div`要素を返すようにしてください。

**注意:** 文字列は`div`要素の子要素とみなされるため、自己終了タグは使えません。

# --hints--

`MyComponent`はJSXを返すべきです。

```js
assert(
  (function () {
    const mockedComponent = Enzyme.mount(React.createElement(MyComponent));
    return mockedComponent.length === 1;
  })()
);
```

`MyComponent`は`div`要素を返すべきです。

```js
assert(
  (function () {
    const mockedComponent = Enzyme.mount(React.createElement(MyComponent));
    return mockedComponent.children().type() === 'div';
  })()
);
```

`div`要素は文字列を含むべきです。

```js
assert(
  (function () {
    const mockedComponent = Enzyme.mount(React.createElement(MyComponent));
    return mockedComponent.find('div').text() !== '';
  })()
);
```
