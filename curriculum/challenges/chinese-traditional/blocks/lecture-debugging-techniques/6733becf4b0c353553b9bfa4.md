# --description--

`debugger` 述語是一個強大的 JavaScript 工具，讓你可以在特定行暫停你的程式碼，以調查程式中發生的狀況。當正確使用時，`debugger` 述語可以為你節省大量時間，幫助你找出為何某些功能無法正常運作。

JavaScript 從上到下執行你的程式碼。當 JavaScript 執行你的程式碼並遇到 `debugger` 述語時，它會立即在該行暫停執行。這讓你有機會檢查變數、查看函式，以及整體程式碼的流程。

這樣，你就能準確看到哪裡運作正常或出錯。然而，這只有在你的瀏覽器開發者工具開啟時才會發生。否則，`debugger` 述語會被忽略，程式碼會照常執行。

此外，當你使用 `debugger` 述語時，現代瀏覽器會在指定的列暫停程式碼執行。它們也允許你透過點擊播放按鈕來恢復執行，但頁面本身不會自動重新載入，除非手動觸發。

要使用 `debugger` 述語，請將它放在你想讓程式碼執行暫停的那一行。以下是一個範例：

```js
let firstNumber = 5;
let secondNumber = 10;
debugger; // Code execution pauses here
let sum = firstNumber + secondNumber;
console.log(sum);
```

在這個範例中，如果你在程式碼執行前沒有開啟主控台，當你最終開啟它時，你會看到 `15`。當主控台開啟且你重新載入頁面時，執行會暫停，讓你可以檢查程式碼。 

這裡有一個更複雜的範例：

```js
function calculateTotalPrice(price, discountPercentage) {
 debugger
 let discountAmount = (price * discountPercentage) / 100
 let totalPrice = price - discountAmount

 console.log(`Original Price: ${price}`)
 console.log(`Discount Amount: ${discountAmount}`)
 console.log(`Total Price after Discount: ${totalPrice}`)

 return totalPrice
}

let price = 100
let discount = 15

let finalPrice = calculateTotalPrice(price, discount)
console.log(`Final Price: ${finalPrice}`);
```

在此範例中，`debugger` 述語被放置在 `calculateTotalPrice` 的頂部。當函式被呼叫時，函式執行將會暫停。

如果你在主控台開啟的情況下重新載入頁面，你會看到頁面持續重新載入，確認程式碼的執行只是暫停，而不是停止。如果你想讓執行繼續，可以點擊播放按鈕。

當你繼續建置你的 JavaScript 程式時，測試程式碼中的 `debugger` 述語，看看它如何幫助你更好地理解執行流程、即時檢查變數，並快速找出可能出錯的地方。
