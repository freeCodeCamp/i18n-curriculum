## 响应式网页设计

- **定义**：响应式设计的核心原则是适应性，即网站能够根据屏幕尺寸和浏览设备的功能调整布局和内容。
- **流动网格**：流动网格使用百分比等相对单位，而不是像素等固定单位，允许内容根据屏幕尺寸调整大小和回流。
- **弹性图像**：弹性图像可在其包含的元素内调整大小，确保在较小的屏幕上不会溢出容器。

## 媒体查询

- **定义**：媒体查询允许开发人员根据设备的特性，主要是视口宽度，应用不同的样式。

```css
@media screen and (min-width: 768px) {
  /* Styles for screens at least 768px wide */
}
```

- **`all` 媒体类型**：适用于所有设备。 如果未指定媒体类型，则默认为该类型。
- **`print` 媒体类型**：适用于分页材料和在打印预览模式下在屏幕上查看的文档。
- **`screen` 媒体类型**：主要适用于屏幕。
- **`aspect-ratio`**：描述视口宽度和高度之间的比例。

```css
@media screen and (aspect-ratio: 16/9) {
  /* Styles for screens with a 16:9 aspect ratio */
}
```

- **`orientation`**：用于指示设备是横向还是纵向。

```css
@media screen and (orientation: landscape) {
  /* Styles for landscape orientation */
}
```

- **`resolution`**：用于描述输出设备的分辨率，单位是点/英寸（dpi）或点/厘米（dpcm）。

```css
@media screen and (min-resolution: 300dpi) {
  /* Styles for high-resolution screens */
}
```

- **`hover`**：用于测试主输入机制是否可以将鼠标悬停在元素上。

```css
@media (hover: hover) {
  /* Styles for devices that support hover */
}
```

- **`prefers-color-scheme`**：用于检测用户是否要求使用浅色或深色主题。
- **媒体查询和逻辑操作符**：`and` 操作符用于组合多个媒体特性，而 `not` 和 `only` 可用于否定或隔离媒体查询。

```css
@media screen and (min-width: 768px) and (orientation: landscape) {
  /* Styles for landscape screens at least 768px wide */
}
```

## 常见媒体断点

- **定义**：媒体断点是网站设计中的特定点，在这些点上，布局和内容会进行调整，以适应不同的屏幕尺寸。 你可以针对手机、平板电脑和桌面屏幕使用一些通用的断点。 但是，试图为不同设备寻找每一种可能的屏幕尺寸并不明智。

```css
/* Styles for screens wider than 768px */
@media screen and (min-width: 768px) {
  body {
    font-size: 1.125rem;
  }
}
```

- **小型设备（智能手机）**：最大 640px
- **中型设备（平板电脑）**： 641px 至 1024px
- **大型设备（台式机）**： 1025px 或更大

## 移动优先方法

- **定义**：移动优先方法是一种响应式网页设计中的设计理念和开发策略，优先为移动设备创建网站，然后再为更大屏幕设计。

```css
/* Base styles for mobile */
.container {
  width: 100%;
  padding: 10px;
}

/* Styles for larger screens */
@media screen and (min-width: 768px) {
  .container {
    width: 750px;
    margin: 0 auto;
    padding: 20px;
  }
}

@media screen and (min-width: 1024px) {
  .container {
    width: 960px;
  }
}
```
