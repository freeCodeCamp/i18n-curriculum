# --description--

```css
/* rule 1 */
.class.second-class {
  font-size: 12px;
}

/* rule 2 */
.class .second-class {
  font-size: 24px;
}
```

Hier haben sowohl Regel 1 als auch Regel 2 die gleiche Spezifität. Regel 1 verwendet einen Verkettungsselektor (kein Leerzeichen) und Regel 2 verwendet einen Nachfahrkombinator (das Leerzeichen). Beide Regeln haben jedoch zwei Klassen, und das Kombinatorsymbol selbst trägt nicht zur Spezifität bei.

```css
/* rule 1 */
.class.second-class {
  font-size: 12px;
}
/* rule 2 */
.class > .second-class {
  font-size: 24px;
}
```

Dieses Beispiel zeigt dasselbe. Obwohl Regel 2 einen Kindkombinator (`>`) verwendet, ändert dies den Spezifitätswert nicht. Beide Regeln haben weiterhin zwei Klassen, daher haben sie die gleichen Spezifitätswerte.

*Hinweis:* Nicht alles trägt zur Spezifität bei

Beim Vergleichen von Selektoren können Sie auf spezielle Symbole für den universellen Selektor (`*`) sowie Kombinatoren (`+`, `~`, `>` und ein Leerzeichen) stoßen. Diese Symbole erhöhen die Spezifität für sich genommen nicht.
