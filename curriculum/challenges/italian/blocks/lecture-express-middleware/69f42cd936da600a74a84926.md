# --description--

Il middleware è un concetto essenziale in Express e ci permette di gestire le richieste e le risposte tra client e server. Le funzioni middleware hanno accesso all’oggetto richiesta, all’oggetto risposta e alla funzione next nel ciclo richiesta-risposta dell’applicazione.

Vediamolo nel dettaglio:

## Cos’è il middleware?

Il middleware è fondamentalmente una funzione che viene eseguita durante il ciclo di vita di una richiesta al server. Svolge varie attività come registrare le richieste, gestire errori, analizzare le richieste in arrivo o servire file statici.

Express offre un modo comodo per definire middleware che elaborano le richieste in arrivo prima che raggiungano i gestori di rotta.

Come funziona il middleware:

Quando viene effettuata una richiesta a un’applicazione Express, il middleware viene eseguito nell’ordine in cui è stato aggiunto. Il flusso è il seguente:

* La richiesta viene inviata al server.
* Le funzioni middleware elaborano la richiesta.
* Se il middleware chiama `next()`, passa il controllo al middleware o gestore di rotta successivo.
* Se il middleware non chiama `next()`, il ciclo richiesta-risposta si interrompe.

```js
const express = require('express')
const app = express()

// Simple middleware that logs request info
app.use((req, res, next) => {
  console.log('Request URL:', req.url)
  console.log('Request Method:', req.method)
  next() // Passes control to the next middleware
})

app.get('/', (req, res) => {
  res.send('Hello World!')
})

app.listen(3000, () => {
  console.log('Server running on http://localhost:3000')
})
```

`app.use()`: Aggiunge middleware globalmente alla tua applicazione.

In questo esempio, il middleware registra i dettagli della richiesta e chiama `next()` per procedere al middleware o gestore di rotta successivo.

Il middleware è una funzionalità potente in Express che ti permette di intercettare e modificare le richieste in vari punti. Puoi usare il middleware per gestire cose come la registrazione, l’autenticazione o anche modificare gli oggetti richiesta e risposta prima che raggiungano i gestori di rotta.
