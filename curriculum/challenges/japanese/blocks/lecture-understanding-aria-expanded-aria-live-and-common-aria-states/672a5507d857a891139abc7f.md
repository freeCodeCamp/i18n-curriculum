# --interactive--

`aria-controls`属性は、ページ上の別の要素の表示を制御する要素と、その制御される要素との間にプログラム的な関係を作るために使われます。この関係はスクリーンリーダーユーザーがコントロールの仕組みをよりよく理解するのに役立ちます。よくある使い方としては、異なるコンテンツのセクションの可視性を制御するタブのセットや、メニューの可視性を切り替えるボタンなどがあります。

この仕組みがどのように機能するか、例を見てみましょう。このタブ付きインターフェースの例では、3つのボタンがある`div`要素があります。

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<div role="tablist">
  <button role="tab" id="tab1" aria-controls="section1" aria-selected="true">
    Tab 1
  </button>
  <button role="tab" id="tab2" aria-controls="section2" aria-selected="false">
    Tab 2
  </button>
  <button role="tab" id="tab3" aria-controls="section3" aria-selected="false">
    Tab 3
  </button>
</div>
```

```css
[role="tablist"] {
  display: flex;
  border-bottom: 2px solid #ccc;
  margin-bottom: 1em;
  gap: 4px;
}

[role="tab"] {
  background: #f5f5f5;
  border: 1px solid #ccc;
  border-bottom: none;

  cursor: pointer;

  transition: background 0.2s, color 0.2s;
}

[role="tab"]:hover,
[role="tab"]:focus {
  background: #e8f0fe;
  outline: none;
}

[role="tab"][aria-selected="true"] {
  background: #fff;
  border-color: #ccc;
  border-bottom: 2px solid #fff;
  color: #000;
  font-weight: 600;
  position: relative;
  top: 1px; 
}

  outline: 2px solid #0078d4;
  outline-offset: 2px;
}

```

:::

`div`は`role="tablist"`を使って、タブのグループのコンテナ要素であることを示しています。

各ボタンはタブを表し、`role`属性に`tab`が設定されています。ほとんどのタブ付きインターフェースでは、最初のタブパネルがデフォルトでは表示されているため、最初のタブボタンには`aria-selected`属性に`true`が設定されていて、関連するコンテンツのセクションが現在表示されていることを示しています。`aria-controls`属性は、各ボタンとそのボタンが制御するコンテンツのセクションを関連付けるために使われます。

タブに対応する3つのコンテンツセクションは次の通りです。

```js
const tabs = document.querySelectorAll('[role="tab"]');
const tabList = document.querySelector('[role="tablist"]');

tabs.forEach(tab => {
  tab.addEventListener('click', () => {
    activateTab(tab);
  });

  tab.addEventListener('keydown', (e) => {
    const key = e.key;
    const currentIndex = Array.from(tabs).indexOf(tab);
    let newIndex = null;

    if (key === 'ArrowRight') {
      newIndex = (currentIndex + 1) % tabs.length;
    } else if (key === 'ArrowLeft') {
      newIndex = (currentIndex - 1 + tabs.length) % tabs.length;
    } else if (key === 'Home') {
      newIndex = 0;
    } else if (key === 'End') {
      newIndex = tabs.length - 1;
    }

    if (newIndex !== null) {
      tabs[newIndex].focus();
      activateTab(tabs[newIndex]);
    }
  });
});

function activateTab(tab) {
  const tabPanels = document.querySelectorAll('[role="tabpanel"]');

  tabs.forEach(t => {
    t.setAttribute('aria-selected', 'false');
    t.setAttribute('tabindex', '-1');
  });

  tabPanels.forEach(panel => panel.hidden = true);

  tab.setAttribute('aria-selected', 'true');
  tab.removeAttribute('tabindex');

  const panelId = tab.getAttribute('aria-controls');
  const panel = document.getElementById(panelId);
  panel.hidden = false;
  tab.focus();
}

```

:::

各コンテンツセクションには`role`属性に`tabpanel`が設定されており、対応するタブを指す`aria-labelledby`属性があり、パネルにアクセシブルな名前を与えています。`hidden`属性は、現在選択されていないコンテンツセクションを非表示にするために使われます。各セクションのIDは対応するボタンの`aria-controls`属性の値と一致しています。`section1`のIDは最初のボタンの`aria-controls`属性と一致し、`section2`は2番目のボタンの`aria-controls`属性と一致し、`section3`は3番目のボタンの`aria-controls`属性と一致しています。これがタブとそのコンテンツセクションの関連付けの仕組みです。

異なるセクションを切り替えるには、どのセクションが現在表示されているかに基づいて、JavaScriptでセクションの`hidden`属性とボタンの`aria-selected`属性を更新する必要があります。一度に表示できるセクションは1つだけで、そのセクションには`hidden`属性がなく、対応するボタンの`aria-selected`は`true`に設定されている必要があります。残りの非表示のセクションにはすべて`hidden`属性があり、それらのボタンの`aria-selected`は`false`に設定されている必要があります。

タブは`aria-controls`属性の一般的な使用例ですが、1つの要素が別の要素の可視性や動作を制御する他のシナリオでも使えます。ほかの例としてはアコーディオン、ドロップダウンメニュー、モーダルなどがあります。

次に要素の可視性を制御するユーザーインターフェースを設計するときは、制御する要素と制御される要素の関係を確立するために`aria-controls`属性を使うことを検討してください。
