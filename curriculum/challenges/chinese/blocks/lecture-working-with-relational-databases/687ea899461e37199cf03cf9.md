# --description--

要在 SQL 中定义一个表格，你需要指定每个列的数据类型。

请记住，这是在 SQL 中创建 `table` 的基本语法。每个 `column` 都有自己的 `data type`。

```sql
CREATE TABLE table_name(
  column1 data_type column_constraint,
  column2 data_type column_constraint,
  column3 data_type column_constraint,
  ... etc
);
```

作为开发者，你的目标是尽可能准确地选择这些数据类型。

SQL 中有六种常见的数据类型类别：

- 数值类型，例如 `INTEGER`、`FLOAT`、`SERIAL` 和 `DECIMAL`。
- 日期和时间，例如 `TIMESTAMP`、`DATE` 和 `TIME`。
- 字符和字串。这些包括 `CHAR`、`VARCHAR` 和 `TEXT`。
- Unicode，包括 `NTEXT` 和 `NVARCHAR`。这些用于确保文本无论字符来源如何，都能被正确保存和检索。
- 二进制，用于保存非文本数据，如图像、音频和视频文件。
- 以及其他杂项数据类型，例如 `XML` 和 `TABLE`。

在本课中，我们将介绍一些在 PostgreSQL 中最广泛使用的类型。数据类型会因数据库管理系统而异，但它们通常非常相似。所以你需要在文档中查看完整列表。

我们将从数值开始。

首先，我们有 `INTEGER` 数据类型：

```sql
units_sold INTEGER
```

在此示例中，我们将此数据类型赋值给 `units_sold` 列。

`INTEGER` 类型在数据库中占用 4 字节，值的范围从 -2,147,483,648 到 2,147,483,647。官方 PostgreSQL 文档将其描述为“整数的典型选择”。

我们还有 `SMALLINT` 和 `BIGINT`，它们基本上与 `INTEGER` 相同，只是由于在数据库中分配的大小不同，分别具有较小和较大的数字范围。

PostgreSQL 中一个有用的特色是使用 `SERIAL` 关键字创建一个列。这里，使用 `SERIAL` 创建了一个 `id` 列：

```sql
id SERIAL
```

虽然它不是真正的数据类型，但它对于创建唯一标识符列非常有帮助，因为该列将具有 `INTEGER` 类型，不允许 `NULL` 值，并且在添加行时会自动递增。

例如，第一行将自动具有 `id` 为 `1`，第二行将具有 `id` 为 `2`，依此类推，为每个记录创建唯一的 ID。

在 MySQL 中，`SERIAL` 的等价物是 `AUTO_INCREMENT` 属性，用于自动生成循序的整数：

```sql
id INT AUTO_INCREMENT
```

你可以看到，不同的数据库管理系统可能有不同的方法来实现你数据库中完全相同的功能性。

这些是最常用的数字数据类型，但有时你可能需要表现文本或字符序列。

为此，你有 `VARCHAR` 数据类型。此数据类型用于可变长度的字串。你可以在括号内设置最大字符长度：

```sql
name VARCHAR(50)
```

在此示例中，我们定义了一个 `name` 列，其中的值最长可达 50 个字符。

此数据类型为字串设置最大长度，但如果你需要保存任意长度的字串，可以改用 `TEXT` ：

```sql
name TEXT
```

除了数字和字串，保存日期和时间也很常见。

例如，如果你创建一个表格来保存事件，你可能需要以特定形式保存每个事件的日期。为此，你可以使用 `DATE` 数据类型：

```sql
event_date DATE
```

要保存时间，你也可以使用 `TIME` 数据类型。例如，你可以保存事件开始的时间：

```sql
start_time TIME
```

`TIMESTAMP` 数据类型结合了两者。它包含日期和时间。如果你写 `TIMESTAMP WITH TIME ZONE`，它还可能包含时区：

```sql
event_timestamp TIMESTAMP
```

```sql
event_timestamp TIMESTAMP WITH TIME ZONE
```

最后，如果你需要保存布尔值 `TRUE` 或 `FALSE`，你可以使用 `BOOLEAN` 数据类型：

```sql
is_active BOOLEAN
```

现在你已经了解了 SQL 中一些最常见的数据类型，但还有更多。你可以查看你的数据库管理系统的文档，以查找有关你可以使用的数据类型的更多信息。
