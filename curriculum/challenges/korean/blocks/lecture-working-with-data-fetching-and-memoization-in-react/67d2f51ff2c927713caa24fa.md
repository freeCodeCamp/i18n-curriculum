# --description--

지난 수업에서 메모이제이션과 `useMemo` 훅이 어떻게 작동하는지 배웠습니다.

이번 수업에서는 `useCallback` 훅과 `React.memo`가 어떻게 작동하는지 배우겠습니다.

지난 수업에서 `useCallback`가 함수 참조를 메모이제이션하는 데 사용된다고도 언급했습니다.

`React.memo`는 프로프가 변경되지 않았을 때 불필요한 재렌더링을 방지하기 위해 컴포넌트를 메모이제이션할 수 있게 해줍니다.

다음은 `useCallback` 훅의 기본 구문입니다:

```js
const handleClick = useCallback(() => {
  // code goes here
}, [dependency]);
```

그리고 `React.memo`의 기본 구문은 다음과 같습니다:

```jsx
const MemoizedComponent = React.memo(({ prop }) => {
  return (
    <>
      {/* Presentation */}
    </>
  )
});
```

`useCallback` 훅의 예제를 살펴보겠습니다:

```jsx
import { useState, useEffect } from "react";

function Counter() {
  const [count, setCount] = useState(0);

  const handleClick = () => {
    setCount((prevCount) => prevCount + 1);
  };

  useEffect(() => {
    console.log("useEffect runs");
  }, [handleClick]);

  return (
    <div>
      <p>Count: {count}</p>
      <button onClick={handleClick}>Increment</button>
    </div>
  );
}

export default Counter;
```

컴포넌트에서는 `handleClick`가 변경될 때마다 effect가 실행되는데, 이는 `handleClick` 함수가 매 렌더링마다 새로 생성되기 때문입니다.

이를 고치려면 `handleClick` 함수를 렌더링 간에 동일한 것으로 취급하도록 `useCallback` 훅으로 메모이제이션해야 합니다. 이렇게 하면 함수가 새로 생성되지 않습니다:

```jsx
import { useState, useEffect, useCallback } from "react";

function Counter() {
  const [count, setCount] = useState(0);

  // Memoize the handleClick function with useCallback
  const handleClick = useCallback(() => {
    setCount((prevCount) => prevCount + 1);
  }, []);

  useEffect(() => {
    console.log("useEffect runs");
  }, [handleClick]);

  return (
    <div>
      <p>Count: {count}</p>
      <button onClick={handleClick}>Increment</button>
    </div>
  );
}

export default Counter;
```

이제 `handleClick` 함수는 매 렌더링마다 새로 생성되지 않습니다.

`React.memo` (또는 `memo`) 고차 함수와 `useCallback` 훅이 함께 작동하는 방식을 보여주기 위해, `Counter` 컴포넌트와 `handleClick` 함수가 있는데 이 함수는 `useCallback`가 필요하지만 현재 사용하지 않고 있습니다:

```jsx
import { useState, useEffect, useCallback } from "react";
import CounterChild from "./CounterChild";

function Counter() {
  const [count, setCount] = useState(0);
  const [timer, setTimer] = useState(new Date().toLocaleTimeString());

  const handleClick = () => {
    setCount(count + 1);
  };

  useEffect(() => {
    const interval = setInterval(() => {
      setTimer(new Date().toLocaleTimeString());
    }, 1000);

    return () => clearInterval(interval);
  }, []);

  return (
    <div>
      <h1>Time: {timer}</h1>
      <p>Count: {count}</p>
      <button onClick={handleClick}>Increment</button>
      <CounterChild onClick={handleClick} />
    </div>
  );
}

export default Counter;
```

이 함수는 또한 상태에 타이머를 가지고 있어 매초 업데이트됩니다. 이로 인해 `timer`가 변경될 때마다 컴포넌트가 재렌더링되고 `handleClick` 함수가 매 렌더링마다 새로 생성됩니다.

그래서 `handleClick`는 `useCallback`로 메모이제이션해야 합니다.

다음은 `CounterChild` 컴포넌트입니다:

```jsx
const CounterChild = ({ onClick }) => {
  console.log("CounterChild component rendered");
  return <button onClick={onClick}>Increment from Child</button>;
};

export default CounterChild;
```

이 `CounterChild` 컴포넌트는 `onClick` 프로프를 받아서 카운터를 증가시킬 수도 있습니다.

`CounterChild` 컴포넌트가 `Counter` 컴포넌트의 자식이므로, 타이머가 변경되어 `Counter`가 재렌더링될 때마다 이 컴포넌트도 렌더링됩니다. 따라서 `CounterChild`도 메모이제이션해야 합니다.

메모이제이션이 없으면, 타이머가 매초 업데이트되어 컴포넌트가 재렌더링될 때 `CounterChild` 컴포넌트도 함께 재렌더링됩니다.

이를 방지하려면 `CounterChild` 컴포넌트를 `React.memo`로 메모이제이션해야 합니다:

```jsx
import React from "react";

const CounterChild = React.memo(({ onClick }) => {
  console.log("CounterChild component rendered");
  return <button onClick={onClick}>Increment from Child</button>;
});

export default CounterChild;
```

`CounterChild`를 `React.memo`로 메모이제이션한 후에도 최적의 작동은 아닙니다.

이는 `handleClick` 함수가 매 렌더링마다 새로 생성되기 때문이며, 이 함수도 `useCallback`로 메모이제이션해야 React에 렌더링 간에 동일한 함수임을 알려줄 수 있습니다:

```js
const handleClick = useCallback(() => {
  setCount((prevCount) => prevCount + 1);
}, [count]);
```

이제 컴포넌트는 `count` 상태가 변경될 때만 재렌더링됩니다.
