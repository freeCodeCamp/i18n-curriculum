# --interactive--

Quando inizi a lavorare con HTML e CSS, noterai che alcuni stili vengono applicati alle tue pagine web anche prima di scrivere qualsiasi CSS. Questi stili si chiamano "stili predefiniti del browser" o "stili user-agent".

Questi stili predefiniti del browser forniscono una formattazione di base per garantire che gli elementi HTML vengano visualizzati in modo leggibile su tutti i browser.

Tuttavia, questi stili possono variare leggermente da un browser all’altro, ed è per questo che è importante conoscerli quando progetti un aspetto coerente per il tuo sito web.

Diamo un’occhiata ad alcuni stili predefiniti comuni applicati a vari elementi HTML.

Le intestazioni sono uno degli elementi HTML più usati e sono stilizzate per impostazione predefinita con dimensioni e pesi diversi.

Per esempio, `h1` (l’intestazione di livello più alto) è solitamente in grassetto e con un carattere tipografico più grande rispetto alle intestazioni di livello inferiore come `h2`, `h3` e così via.

Questi tag di intestazione aiutano a definire la gerarchia e la struttura del tuo contenuto.

L’esempio seguente mostra tutti e sei gli elementi di intestazione da `h1` a `h6`.

:::interactive_editor

```html
<h1>Heading 1</h1>
<h2>Heading 2</h2>
<h3>Heading 3</h3>
<h4>Heading 4</h4>
<h5>Heading 5</h5>
<h6>Heading 6</h6>
```

:::

Un altro elemento con stili predefiniti è l’elemento `hr`, che crea una linea orizzontale spesso usata per separare visivamente le sezioni di contenuto. I browser generalmente applicano a questo elemento uno stile di linea semplice.

Per capirlo meglio, diamo un’occhiata a questo esempio di codice:

:::interactive_editor

```html
<p>Paragraph 1</p>
<p>Paragraph 2</p>
<hr>
<p>Paragraph 3</p>
<p>Paragraph 4</p>
```

:::

La linea orizzontale appare come una linea sottile con spaziatura sopra e sotto il testo per distinguere le sezioni di contenuto.

Successivamente, vedremo l’elemento `blockquote`.

Le citazioni sono usate per indicare una sezione di testo che è una citazione da un’altra fonte. I browser di solito aggiungono un rientro e a volte mettono il testo in corsivo.

Il rientro aiuta a distinguere le citazioni dal resto del testo, rendendo chiaro che questo contenuto è preso da un’altra fonte.

Diamo un’occhiata a questo esempio di codice:

:::interactive_editor

```html
<p>Paragraph 1</p>
<p>Paragraph 2</p>
<blockquote>I think, therefore I am. (Rene Descartes)</blockquote>
<p>Paragraph 3</p>
<p>Paragraph 4</p>
```

:::

Nell’esempio sopra, l’elemento `blockquote` produrrà un rientro sul testo, applicato dagli stili predefiniti del browser.

Un altro elemento con stili predefiniti è l’elemento àncora (`<a>`), che è stilizzato con un colore blu predefinito e sottolineatura per renderlo riconoscibile come collegamento.

Diamo un’occhiata al seguente esempio HTML:

:::interactive_editor

```html
<p>Paragraph 1</p>
<p>Paragraph 2</p>
<a href="https://freecodecamp.org/">Visit the freeCodeCamp learn page</a>
<p>Paragraph 3</p>
<p>Paragraph 4</p>
```

:::

Il codice sopra contiene quattro elementi paragrafo con un elemento àncora in mezzo. L’elemento àncora è stilizzato in blu con una sottolineatura per indicare agli utenti di andare alla pagina di apprendimento di freeCodeCamp.

Infine, vedremo gli elementi lista non ordinata e lista ordinata.

I browser aggiungono una formattazione di base alle liste, inclusi rientro e punti elenco o numeri, a seconda che tu stia usando una lista non ordinata (`ul`) o una lista ordinata (`ol`).

Diamo un’occhiata a un esempio:

:::interactive_editor

```html
<p>Sample Paragraph</p>
<ol>
  <li>item 1</li>
  <li>item 2</li>
  <li>item 3</li>
</ol>
<ul>
  <li>item</li>
  <li>another item</li>
  <li>yet another item</li>
</ul>
<p>Ending Paragraph</p>
```

:::

Nell’esempio di codice sopra, stiamo usando una lista non ordinata e una lista ordinata. Entrambe queste liste saranno leggermente rientrate a destra per impostazione predefinita.

Per tutti questi esempi, hai visto come il browser applica spaziatura predefinita, dimensioni e pesi di carattere variabili a questi elementi HTML.

Nelle lezioni successive imparerai come resettare alcuni di questi stili predefiniti usando un reset CSS.
