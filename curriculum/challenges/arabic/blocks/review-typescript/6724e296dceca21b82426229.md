## ما هو TypeScript

- **JavaScript**: جافاسكريبت لغة ذات نوع ديناميكي. هذا يعني أن المتغيرات يمكن أن تستقبل أي قيم أثناء وقت التشغيل. التحدي في لغة ذات نوع ديناميكي هو أن غياب أمان النوع قد يُدخل أخطاء.

على سبيل المثال، حتى لو كانت دالتك في جافاسكريبت تتوقع مصفوفة، يمكنك مع ذلك استدعاؤها برقم:

```javascript
const getRandomValue = (array) => {
  return array[Math.floor(Math.random() * array.length)];
}

console.log(getRandomValue(10));
```

الناتج في الـ`console` للمثال أعلاه سيكون `undefined`.

- **TypeScript**: يوسع TypeScript لغة جافاسكريبت ليشمل النوع الثابت، مما يساعد على اكتشاف الأخطاء الناتجة عن عدم تطابق الأنواع قبل تشغيل الشفرة.

على سبيل المثال، يمكنك تعريف نوع للمعلمة `array` كما يلي:

```typescript
const getRandomValue = (array: string[]) => {
  return array[Math.floor(Math.random() * array.length)];
}
```

تعريف النوع هذا يخبر TypeScript أن المعلمة `array` يجب أن تكون مصفوفة من سلاسل نصية. ثم، عندما تستدعي `getRandomValue` وتمرر لها رقمًا، تحصل على خطأ يُسمى خطأ المترجم.

- **المترجم**: تحتاج أولًا إلى ترجمة شفرة TypeScript إلى جافاسكريبت عادية. عند تشغيل المترجم، يقوم TypeScript بتقييم الشفرة ويرمي خطأً لأي مشاكل في عدم تطابق الأنواع.

## أنواع البيانات في TypeScript

- **أنواع البيانات الأولية في TypeScript**: لأنواع البيانات الأولية `string`، `null`، `undefined`، `number`، `boolean`، و`bigint`، يقدم TypeScript الكلمات المفتاحية المناسبة للنوع.

```typescript
let str: string = "Naomi";
let num: number = 42;
let bool: boolean = true;
let nope: null = null;
let nada: undefined = undefined;
```

- **مصفوفة**: يمكنك تعريف مصفوفة من نوع محدد باستخدام صيغتين مختلفتين.

```typescript
const arrOne: string[] = ["Naomi"];
const arrTwo: Array<string> = ["Camperchan"];
```

- **الكائنات**: يمكنك تعريف الهيكل الدقيق لكائن.

```typescript
const obj: { a: string, b: number } = { a: "Naomi", b: 42 };
```

إذا أردت كائنًا بأي مفاتيح، لكن يجب أن تكون كل القيم سلاسل نصية، هناك طريقتان لتعريفه:

```typescript
const objOne: Record<string, string> = {};
const objTwo: { [key: string]: string } = {};
```

- **أنواع أخرى مفيدة في TypeScript**:
  - **`any`**: `any` تشير إلى أن القيمة يمكن أن تكون من أي نوع. تخبر المترجم بالتوقف عن الاهتمام بنوع ذلك المتغير.
  - **`unknown`**: `unknown` تخبر TypeScript بأنك تهتم بنوع القيمة، لكنك لا تعرف ما هو فعليًا. يُفضل استخدام `unknown` على `any`.
  - **`void`**: هذا نوع خاص تستخدمه عادة عند تعريف الدوال. الدوال التي لا تُرجع قيمة تستخدم نوع إرجاع `void`.
  - **`never`**: يمثل نوعًا لن يوجد أبدًا.

- **الكلمة المفتاحية `type`**: هذه الكلمة تشبه `const`، لكن بدلًا من إعلان متغير، يمكنك إعلان نوع.

مفيدة لإعلان أنواع مخصصة مثل أنواع الاتحاد أو الأنواع التي تشمل قيمًا محددة فقط:

```typescript
type stringOrNumber = string | number;
type bot = "camperchan" | "camperbot" | "naomi";
```

- **`interface`**: الواجهات تشبه الأصناف للأنواع. يمكنها تنفيذ أو توسيع واجهات أخرى، وهي أنواع كائنات بشكل خاص، وعادة ما تُفضل إلا إذا كنت تحتاج ميزة محددة تقدمها إعلان `type`.

```typescript
interface wowie {
  zowie: boolean;
  method: () => void;
}
```

- **تعريف نوع الإرجاع**: يمكنك أيضًا تعريف _نوع الإرجاع_ للدالة.

المثال أدناه يحدد قيمة الإرجاع كسلسلة نصية. إذا حاولت إرجاع أي شيء آخر، سيعطي TypeScript خطأ مترجم.

```typescript
const getRandomValue = (array: string[]): string => {
  return array[Math.floor(Math.random() * array.length)];
}
```

## الأنواع العامة (Generics)

- **تعريف النوع العام**: يمكنك تعريف نوع عام والإشارة إليه في دالتك. يمكن اعتباره معلمة خاصة تقدمها للدالة للتحكم في سلوك تعريف نوع الدالة.

إليك مثالًا لتعريف نوع عام لدالة:

```typescript
const getRandomValue = <T>(array: T[]): T => {
  return array[Math.floor(Math.random() * array.length)];
}
const val = getRandomValue([1, 2, 4])
```

الصيغة `<T>` تخبر TypeScript بأنك تعرف نوعًا عامًا اسمه `T` للدالة. `T` اسم شائع للأنواع العامة، لكن يمكنك استخدام أي اسم.

ثم تخبر TypeScript أن المعلمة `array` هي مصفوفة من القيم التي تطابق النوع العام، وأن القيمة المرجعة هي عنصر واحد من نفس النوع.

- **تحديد معلمة النوع في استدعاء الدالة**: يمكنك تمرير معلمة نوع لاستدعاء دالة باستخدام أقواس زاوية بين اسم الدالة ومعلماتها.

إليك مثالًا لتمرير معلمة نوع لاستدعاء دالة:

```typescript
const email = document.querySelector<HTMLInputElement>("#email");
console.log(email.value);
```

هذا يخبر TypeScript أن العنصر الذي تتوقع العثور عليه سيكون عنصر إدخال.

## تضييق النوع (Type Narrowing)

- **التضييق بالتحقق من الحقيقة**: في المثال أدناه، تحصل على خطأ مترجم عند محاولة الوصول إلى خاصية `value` لـ`email` لأن `email` _قد_ تكون `null`.

```typescript
const email = document.querySelector<HTMLInputElement>("#email");
console.log(email.value);
```

يمكنك استخدام شرط للتأكد من أن `email` هي _حقيقية_ قبل الوصول إلى الخاصية:

```typescript
const email = document.querySelector<HTMLInputElement>("#email");
if (email) {
  console.log(email.value);
}
```

يمكن أن تعمل فحوصات الحقيقة أيضًا في الاتجاه العكسي:

```typescript
const email = document.querySelector<HTMLInputElement>("#email");
if (!email) {
  throw new ReferenceError("Could not find email element!")
}
console.log(email.value);
```

رمي خطأ ينهي التنفيذ المنطقي لهذا الكود، مما يعني أنه عند الوصول إلى استدعاء `console.log()`، يعرف TypeScript أن `email` _لا يمكن_ أن تكون `null`.

- **التسلسل الاختياري**: التسلسل الاختياري `?.` هو أيضًا شكل من أشكال تضييق النوع، على نفس المبدأ بأن الوصول إلى الخاصية لا يمكن أن يحدث إذا كانت قيمة `email` هي `null`.

```typescript
const email = document.querySelector<HTMLInputElement>("#email");
console.log(email?.value);
```

- **عامل `typeof`**: يمكنك استخدام شرط للتحقق من نوع المتغير باستخدام عامل `typeof`.

```typescript
const myVal = Math.random() > 0.5 ? 222 : "222";
if (typeof myVal === "number") {
  console.log(myVal / 10);
}
```

- **الكلمة المفتاحية `instanceof`**: إذا كان الكائن من صنف، يمكنك استخدام الكلمة المفتاحية `instanceof` لتضييق النوع.

```typescript
const email = document.querySelector("#email");

if (email instanceof HTMLInputElement) {
    console.log(email.value);
}
```

- **التحويل (Casting)**: عندما لا يستطيع TypeScript تحديد نوع القيمة تلقائيًا، مثل نتيجة دالة `request.json()` في المثال أدناه، ستحصل على خطأ مترجم. إحدى الطرق لحل ذلك هي تحويل النوع، لكن هذا يضعف قدرة TypeScript على اكتشاف الأخطاء المحتملة.

```typescript
interface User {
    name: string;
    age: number;
}

const printAge = (user: User) => 
  console.log(`${user.name} is ${user.age} years old!`)

const request = await fetch("url")
const myUser = await request.json() as User;
printAge(myUser);
```

- **حارس النوع (Type Guard)**: بدلًا من تحويل النوع، يمكنك كتابة حارس نوع:

```typescript
interface User {
    name: string;
    age: number;
}

const isValidUser = (user: unknown): user is User => {
  return !!user && 
    typeof user === "object" &&
    "name" in user &&
    "age" in user;
}
```

الصيغة `user is User` تشير إلى أن دالتك تُرجع قيمة منطقية، وعندما تكون صحيحة تعني أن قيمة `user` تفي بواجهة `User`.

## ملف `tsconfig`

- **`tsconfig.json`**: إعدادات مترجم TypeScript موجودة في ملف `tsconfig.json` في الدليل الجذري لمشروعك.

```json
{
  "compilerOptions": {
    "rootDir": "./src",
    "outDir": "./prod",
    "lib": ["ES2023"],
    "target": "ES2023",
    "module": "ES2022",
    "moduleResolution": "Node",
    "esModuleInterop": true,
    "skipLibCheck": true,
    "strict": true
  },
  "exclude": ["test/"]
}
```

إليك وصفًا لخيارات المترجم المستخدمة في المثال أعلاه:

- **`compilerOptions`**: خاصية `compilerOptions` هي المكان الذي تتحكم فيه بسلوك مترجم TypeScript.
- **`rootDir` و `outDir`**: تخبر `rootDir` و `outDir` TypeScript بأي دليل يحتوي على ملفات المصدر، وأي دليل يجب أن يحتوي على شفرة جافاسكريبت المترجمة.
- **`lib`**: تحدد خاصية `lib` تعريفات النوع التي يستخدمها المترجم، وتسمح لك بتضمين دعم لإصدارات ES محددة، وDOM، والمزيد.
- **`module` و `moduleResolution`**: يعمل `module` و `moduleResolution` معًا لإدارة كيفية استخدام حزمتك للوحدات - إما CommonJS أو ECMAScript.
- **`esModuleInterop`**: يسمح `esModuleInterop` بتشغيل أكثر سلاسة بين وحدات CommonJS وES عن طريق إنشاء كائنات مساحة أسماء تلقائيًا للاستيراد.
- **`skipLibCheck`**: خيار `skipLibCheck` يتخطى التحقق من ملفات `.d.ts` التي لا يتم الإشارة إليها بالاستيراد في شفرتك.
- **`strict`**: تفعيل العلم `strict` يُمكّن عدة فحوصات، مثل ضمان التعامل الصحيح مع الأنواع القابلة لأن تكون فارغة وتحذير عند رجوع TypeScript إلى `any`.
- **`exclude`**: خاصية `exclude` على المستوى الأعلى تخبر المترجم بتجاهل هذه الملفات TypeScript أثناء الترجمة.
