# --description--

ستتطلب منك هذه التحديات كتابة شفرتك باستخدام إحدى الطرق التالية:

- استنسخ <a href="https://github.com/freeCodeCamp/boilerplate-npm/" target="_blank" rel="noopener noreferrer nofollow">مستودع GitHub هذا</a> وأكمل هذه التحديات محليًا، باستخدام <a href="https://www.freecodecamp.org/news/how-to-run-the-freecodecamp-backend-challenges-locally/" target="_blank" rel="noopener noreferrer nofollow">هذا الدليل</a> لإعداد Node.js و npm وتشغيل مشاريع الواجهة الخلفية على جهازك.
- استخدم منشئ مواقع من اختيارك لإكمال المشروع. تأكد من دمج جميع الملفات من مستودع GitHub الخاص بنا.

ملف `package.json` هو مركز أي مشروع Node.js أو حزمة npm. يخزن معلومات عن مشروعك. يتكون من كائن JSON واحد حيث تُخزن المعلومات في أزواج المفتاح-القيمة. هناك حقلان مطلوبان فقط؛ `name` و `version`، لكن من الممارسات الجيدة توفير معلومات إضافية.

يمكنك إنشاء ملف `package.json` من الطرفية باستخدام الأمر `npm init`. سيشغل هذا إعدادًا موجهًا. استخدام `npm init` مع العلامة `-y` سينشئ الملف دون طرح أي أسئلة، `npm init -y`.

إذا نظرت إلى شجرة الملفات لمشروعك، ستجد ملف `package.json` في المستوى الأعلى من الشجرة. هذا هو الملف الذي ستعمل على تحسينه في التحديات القادمة.

واحدة من أكثر المعلومات شيوعًا في هذا الملف هي حقل `author`. يحدد من أنشأ المشروع، ويمكن أن يتكون من سلسلة نصية أو كائن يحتوي على تفاصيل الاتصال أو غيرها. يُنصح باستخدام كائن للمشاريع الأكبر، لكن سلسلة نصية بسيطة مثل المثال التالي تكفي لهذا المشروع.

```json
"author": "Jane Doe",
```

# --instructions--

أضف اسمك كـ `author` للمشروع في ملف `package.json`.

**ملاحظة:** تذكر أنك تكتب JSON، لذا يجب أن تستخدم جميع أسماء الحقول علامات اقتباس مزدوجة (") وأن تفصل بينها بفاصلة (,).
