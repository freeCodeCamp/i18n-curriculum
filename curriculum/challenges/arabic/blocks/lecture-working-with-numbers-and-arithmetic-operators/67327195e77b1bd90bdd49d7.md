# --interactive--

JavaScript هي لغة تعمل أحيانًا بطرق مفاجئة أو حتى غريبة. من هذه المفاجآت ما يحدث عند خلط الأعداد والسلاسل النصية في العمليات الحسابية. أول شيء ستجربه على الأرجح هو جمع عدد مع سلسلة نصية. في JavaScript، يقوم العامل `+` بوظيفتين. فهو يتعامل مع الجمع وأيضًا ربط السلاسل النصية، وهي طريقة لضم سلسلتين نصيتين معًا.

عندما تستخدم `+` مع عدد وسلسلة نصية، يقرر JavaScript معاملة كلاهما كسلاسل نصية ويربطهما معًا. إذا تحققت من نوع النتيجة باستخدام العامل `typeof`، سترى أنها بالفعل سلسلة نصية:

:::interactive_editor

```js
const result = 5 + '10';

console.log(result); // 510
console.log(typeof result); // string
```

:::

ما رأيك سيحدث إذا غيرت ترتيب `5` و `'10'`؟

:::interactive_editor

```js
const result = '10' + 5;

console.log(result); // 105
console.log(typeof result); // string
```

:::

يمكنك أن ترى أن نفس الشيء حدث. يرى JavaScript سلسلة نصية في `'10'` وعددًا في `5`، لذا يحول العدد إلى سلسلة نصية ويربطهما. هذا ما يُعرف بإجبار النوع. إجبار النوع هو تحويل قيمة من نوع بيانات إلى نوع آخر.

تصبح الأمور أكثر إثارة عندما تحاول إجراء عمليات حسابية أخرى مثل الطرح أو الضرب أو القسمة مع سلسلة نصية وعدد. في هذه الحالات، يحاول JavaScript تحويل السلسلة النصية إلى عدد قبل إجراء الحساب — وهو إجبار نوع آخر! هذا ما يحدث:

:::interactive_editor

```js
const subtractionResult = '10' - 5;
console.log(subtractionResult); // 5
console.log(typeof subtractionResult); // number

const multiplicationResult = '10' * 2;
console.log(multiplicationResult); // 20
console.log(typeof multiplicationResult); // number

const divisionResult = '20' / 2;
console.log(divisionResult); // 10
console.log(typeof divisionResult); // number
```

:::

في الأمثلة أعلاه، يحول JavaScript السلسلة النصية `'10'` أو `'20'` بنجاح إلى عدد ثم ينفذ العملية الحسابية. كنتيجة لذلك، `'10' - 5` يعطي `5`، و`'10' * 2` يعطي `20`، و`'20' / 2` ينتج `10`.

لكن ماذا لو لم تكن السلسلة النصية تشبه عددًا؟ لنرَ ماذا يحدث في هذه الحالة:

:::interactive_editor

```js
const subtractionResult = 'abc' - 5;
console.log(subtractionResult); // NaN
console.log(typeof subtractionResult); // number

const multiplicationResult = 'abc' * 2;
console.log(multiplicationResult); // NaN
console.log(typeof multiplicationResult); // number

const divisionResult = 'abc' / 2;
console.log(divisionResult); // NaN
console.log(typeof divisionResult); // number
```

:::

في الأمثلة أعلاه، السلسلة النصية `'abc'` لا تمثل قيمة عددية صحيحة، لذا لا يستطيع JavaScript تحويلها إلى عدد ذي معنى. عندما يفشل هذا التحويل، يرجع JavaScript القيمة `NaN`، التي تعني "ليست عددًا". `NaN` هي قيمة خاصة من نوع `Number` تمثل عددًا غير صالح أو غير قابل للتمثيل.

ماذا لو أجريت عمليات حسابية باستخدام قيمة منطقية (`true` أو `false`)? لنرَ ماذا يحدث. يعامل JavaScript القيم المنطقية كأعداد في العمليات الحسابية: `true` تصبح `1`، و`false` تصبح `0`.

:::interactive_editor

```js
const result1 = true + 1;
console.log(result1); // 2
console.log(typeof result1); // number

const result2 = false + 1;
console.log(result2); // 1
console.log(typeof result2); // number

const result3 = 'Hello' + true;
console.log(result3); // "Hellotrue"
console.log(typeof result3); // string
```

:::

في المثالين الأولين، كانت نتيجة `true + 1` هي `2`، و`false + 1` هي `1`. في المثال الثالث، `'Hello' + true`، حول JavaScript القيمة `true` إلى سلسلة نصية وربطها مع `'Hello'`، مما أدى إلى `'Hellotrue'`، وهي سلسلة نصية.

بالنسبة لـ `null` و `undefined`، يعامل JavaScript `null` كـ `0` و`undefined` كـ `NaN` في العمليات الحسابية:

:::interactive_editor

```js
const result1 = null + 5;
console.log(result1); // 5
console.log(typeof result1); // number

const result2 = undefined + 5;
console.log(result2); // NaN
console.log(typeof result2); // number
```

:::

يقوم JavaScript غالبًا بإجبار النوع، محولًا تلقائيًا أنواع البيانات مثل الأعداد والسلاسل النصية والقيم المنطقية بطرق قد تكون غير متوقعة أحيانًا. فهم هذه التحويلات ضروري لتجنب الأخطاء البرمجية وكتابة شفرة قوية في مشاريعك.
