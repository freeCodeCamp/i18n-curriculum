# --description--

In Express, `express.Router()` è una funzionalità potente che ti aiuta a organizzare le tue rotte in parti più piccole e modulari. Questo è particolarmente utile man mano che la tua applicazione cresce.

Invece di definire tutte le rotte in un unico file grande, puoi dividerle in file separati usando i router. Questo mantiene il tuo codice più facile da leggere, mantenere e scalare.

Il metodo `express.Router()` crea un nuovo oggetto router.

Un router funziona come una mini applicazione Express. Può gestire le proprie rotte e middleware, ma non è una vera e propria applicazione Express. Una volta definite le rotte su un router, puoi montarlo nella tua app principale usando `app.use()`.

In altre parole, un router ti permette di raggruppare rotte correlate insieme e poi integrarle nella tua applicazione principale.

Usare i router ha diversi vantaggi:

**Separazione delle responsabilità**

Puoi definire le rotte in file separati, migliorando l’organizzazione e la leggibilità della tua applicazione.

**Riutilizzo del codice**  
I router possono essere riutilizzati in diverse parti di un’applicazione o anche tra progetti diversi.

**File principale più pulito**  
Invece di riempire il tuo file principale `app.js` o `server.js` con molte definizioni di rotte, puoi spostarle nei router e mantenere il file principale focalizzato sulla configurazione e l’impostazione.

Vediamo un esempio semplice in cui usiamo `express.Router()` per definire rotte per il percorso `/users`.

Prima, ecco il file `app.js`:

```javascript
const express = require("express");
const app = express();
const port = 3000;

// Importing the user routes
const userRoutes = require("./userRoutes");

// Using the userRoutes router for /users path
app.use("/users", userRoutes);

app.listen(port, () => {
  console.log(`Server running at http://localhost:${port}`);
});
```

Ecco invece `userRoutes.js`:

```javascript
const express = require("express");
const router = express.Router();

// GET request for /users
router.get("/", (req, res) => {
  res.send("List of users");
});

// POST request for /users
router.post("/", (req, res) => {
  res.send("Creating a new user");
});

// GET request for /users/:id
router.get("/:id", (req, res) => {
  const userId = req.params.id;
  res.send(`User details for user with id: ${userId}`);
});

module.exports = router;
```

Ora diamo un’occhiata più da vicino a come funziona tutto.

In `app.js` importiamo `express` e creiamo un’istanza dell’applicazione Express assegnandola a una variabile chiamata `app`.

Poi importiamo il router `userRoutes` da `userRoutes.js` e usiamo `app.use("/users", userRoutes)` per montare il router. Questo dice a Express che, se una richiesta inizia con `/users`, deve passarla al router `userRoutes`.

Poi in `userRoutes.js` creiamo un nuovo router usando `express.Router()` e definiamo le rotte usando `router.get()`, `router.post()` e `router.get("/:id")`.

Nota che le rotte definite qui sono relative a `/users` perché è lì che il router è montato. Quindi:

*   `router.get("/")` → `GET /users`: risponde con una lista di utenti
    
*   `router.post("/")` → `POST /users`: risponde con un messaggio sulla creazione di un nuovo utente
    
*   `router.get("/:id")` → `GET /users/:id`: risponde con i dettagli di un utente dato un id utente
    

Usare `express.Router()` mantiene il file principale pulito e focalizzato sulla configurazione. Tutte le rotte relative agli utenti sono raggruppate in `userRoutes.js`.

Man mano che la tua applicazione cresce, puoi creare router aggiuntivi per altre sezioni, come `/products`, `/orders` o `/auth`.

Ogni sezione resta organizzata nel proprio file, rendendo l’applicazione più facile da mantenere e scalare.
