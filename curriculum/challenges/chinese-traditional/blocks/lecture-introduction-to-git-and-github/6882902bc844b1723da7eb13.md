# --description--

在你真正開始使用版本控制和像 GitHub 這樣的網站之前，你需要先安裝並準備好使用 Git。

要檢查你的機器上是否已經安裝 Git，你可以在終端機中執行以下指令：

```sh
git --version
```

如果你看到版本號，表示已安裝 Git。若沒有，則你需要安裝它。

對於 Linux 系統，Git 通常隨大多數發行版預先安裝。如果你沒有預先安裝 Git，應該可以使用你的軟體包管理指令來安裝，例如 `sudo apt-get install git` 或 `sudo pacman -S git`。

對於 Mac 使用者，你可以透過 Homebrew 使用 `brew install git` 安裝 Git，或者你可以從 Git 的網站下載可執行安裝程式。

對於 Windows，你可以從 Git 的網站下載可執行安裝程式。或者，如果你已經設定好 Chocolatey，你可以在 PowerShell 中執行 `choco install git.install`。請注意，在 Windows 上，你可能也想下載 GitBash，以便擁有類 Unix 的 shell 環境。

為了確保安裝成功，請在終端機中再次執行 `git --version` 指令。

安裝好 Git 後，你需要對設定做一些更改。你只需要在你的電腦上做這些更改一次，升級時就不需要持續更改。

`git config` 用於設定負責 Git 在你的機器上如何運作的組態變數。若要查看你目前的設定變數以及它們在系統上的儲存位置，你可以執行以下指令：

```sh
git config --list --show-origin
```

現在如果你是第一次安裝 Git，應該只會看到系統層級的設定。若你在本課程之前已經安裝過 Git，則可能會看到使用者名稱和電子郵件的使用者設定。

要設定你的使用者名稱，你可以執行以下指令：

```sh
git config --global user.name "Jane Doe"
```

這裡使用 `--global` 旗標來設定系統中所有使用 Git 的專案的使用者名稱。如果你需要為特定專案改寫使用者名稱，則可以在該特定專案目錄中執行該指令，且不使用 `--global` 旗標。

要設定使用者電子郵件位址，你可以執行以下指令：

```sh
git config --global user.email janedoe@example.com
```

設定你的使用者名稱和電子郵件很重要，因為 Git 會在你對專案所做的每次 commit 中使用這些資訊。你將在未來的課程中學習更多關於 commit 的內容。

另一個你可以設定的組態是你希望 Git 使用的偏好編輯器。以下是如何將你的偏好編輯器設定為 Emacs 的範例：

```sh
git config --global core.editor emacs
```

其他選項包括 Vim、Nano、VS Code 等。 例如，如果你使用 VS Code 作為你的編輯器，則你可以這樣組態 Git 設定：

```sh
git config --global core.editor "code --wait"
```

如果你選擇不設定偏好的編輯器，則 Git 將使用你系統的預設編輯器。

如果你使用的是 Windows 電腦並且想要更改你的編輯器，你需要像這樣提供可執行檔的完整路徑：

```sh
git config --global core.editor "'C:/Program Files/Notepad++/notepad++.exe' -multiInst -notabbar -nosession -noPlugin"
```

要查看你的設定列表，你可以執行以下指令：

```sh
git config --list
```

既然你已經安裝並設定好 Git，現在你已準備好開始為你的專案添加版本控制。
