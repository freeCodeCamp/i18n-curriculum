# --description--

前回のレッスンでは、メモ化と `useMemo` フックの動作について学びました。

今回のレッスンでは、`useCallback` フックと `React.memo` の動作について学びます。

前回のレッスンでは、`useCallback` が関数の参照をメモ化するためのものだとも説明しました。

`React.memo` は、プロップが変わっていない場合に不要な再レンダリングを防ぐためにコンポーネントをメモ化できます。

`useCallback` フックの基本的な構文は以下の通りです：

```js
const handleClick = useCallback(() => {
  // code goes here
}, [dependency]);
```

そして `React.memo` の基本的な構文は以下の通りです：

```jsx
const MemoizedComponent = React.memo(({ prop }) => {
  return (
    <>
      {/* Presentation */}
    </>
  )
});
```

`useCallback` フックの例を見てみましょう：

```jsx
import { useState, useEffect } from "react";

function Counter() {
  const [count, setCount] = useState(0);

  const handleClick = () => {
    setCount((prevCount) => prevCount + 1);
  };

  useEffect(() => {
    console.log("useEffect runs");
  }, [handleClick]);

  return (
    <div>
      <p>Count: {count}</p>
      <button onClick={handleClick}>Increment</button>
    </div>
  );
}

export default Counter;
```

このコンポーネントでは、`handleClick` が変わるたびに effect が実行されます。なぜなら `handleClick` 関数が毎回レンダリングのたびに再作成されているからです。

これを修正するには、`handleClick` 関数を `useCallback` フックでメモ化して、レンダリング間で同じものとして扱うように React に伝える必要があります。そうすれば再作成されません：

```jsx
import { useState, useEffect, useCallback } from "react";

function Counter() {
  const [count, setCount] = useState(0);

  // Memoize the handleClick function with useCallback
  const handleClick = useCallback(() => {
    setCount((prevCount) => prevCount + 1);
  }, []);

  useEffect(() => {
    console.log("useEffect runs");
  }, [handleClick]);

  return (
    <div>
      <p>Count: {count}</p>
      <button onClick={handleClick}>Increment</button>
    </div>
  );
}

export default Counter;
```

これで `handleClick` 関数は毎回レンダリングのたびに再作成されなくなりました。

`React.memo`（または `memo`）の高階関数の動作と `useCallback` フックが連携する様子を示すために、`Counter` コンポーネントと `handleClick` 関数の例を示します。この関数は `useCallback` が必要ですが、現在は使われていません：

```jsx
import { useState, useEffect, useCallback } from "react";
import CounterChild from "./CounterChild";

function Counter() {
  const [count, setCount] = useState(0);
  const [timer, setTimer] = useState(new Date().toLocaleTimeString());

  const handleClick = () => {
    setCount(count + 1);
  };

  useEffect(() => {
    const interval = setInterval(() => {
      setTimer(new Date().toLocaleTimeString());
    }, 1000);

    return () => clearInterval(interval);
  }, []);

  return (
    <div>
      <h1>Time: {timer}</h1>
      <p>Count: {count}</p>
      <button onClick={handleClick}>Increment</button>
      <CounterChild onClick={handleClick} />
    </div>
  );
}

export default Counter;
```

この関数には、1秒ごとに更新されるタイマーが state にあります。これにより `timer` が変わるたびにコンポーネントが再レンダリングされ、`handleClick` 関数が毎回再作成されてしまいます。

だからこそ `handleClick` は `useCallback` でメモ化する必要があります。

こちらが `CounterChild` コンポーネントです：

```jsx
const CounterChild = ({ onClick }) => {
  console.log("CounterChild component rendered");
  return <button onClick={onClick}>Increment from Child</button>;
};

export default CounterChild;
```

この `CounterChild` コンポーネントは `onClick` プロップを受け取り、そこからカウンターを増やすこともできます。

`CounterChild` コンポーネントは `Counter` コンポーネントの子なので、タイマーの更新によって `Counter` が再レンダリングされるたびにこちらも再レンダリングされます。したがって `CounterChild` もメモ化が必要です。

メモ化しなければ、タイマーが毎秒更新されるたびに `CounterChild` コンポーネントも再レンダリングされてしまいます。

これを防ぐために `CounterChild` コンポーネントを `React.memo` でメモ化してください：

```jsx
import React from "react";

const CounterChild = React.memo(({ onClick }) => {
  console.log("CounterChild component rendered");
  return <button onClick={onClick}>Increment from Child</button>;
});

export default CounterChild;
```

`CounterChild` を `React.memo` でメモ化しても、まだ最適に動作しません。

これは `handleClick` 関数が毎回レンダリングのたびに再作成されているためで、こちらも `useCallback` でメモ化して、レンダリング間で同じ関数を使うように React に伝える必要があります：

```js
const handleClick = useCallback(() => {
  setCount((prevCount) => prevCount + 1);
}, [count]);
```

これでコンポーネントは `count` state が変わったときだけ再レンダリングされます。
