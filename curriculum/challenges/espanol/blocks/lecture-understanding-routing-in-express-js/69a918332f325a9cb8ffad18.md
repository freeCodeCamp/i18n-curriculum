# --description--

En Express, los **manejadores de rutas** (también llamados funciones callback de ruta o funciones controlador) son funciones que definen qué sucede cuando un cliente envía una solicitud a una ruta específica.

Un manejador de ruta procesa la solicitud entrante, realiza la lógica necesaria como consultar una base de datos o hacer cálculos, y luego envía una respuesta de vuelta al cliente.

Vamos a desglosarlo con un ejemplo sencillo (que en realidad has estado usando todo el tiempo):

```javascript
app.get("/menu", (req, res) => {
  res.send("Welcome to the menu!");
});
```

El método `.get()` es uno de los métodos de ruta de Express. Escucha solicitudes **GET** en una ruta especificada. En este ejemplo, escucha solicitudes a `/menu`.

La función `(req, res) => { ... }` es el **manejador de ruta**. Se ejecuta cada vez que un cliente envía una solicitud GET a `/menu`.

Dentro del manejador:

* `req` (request) es un objeto que contiene información sobre la solicitud entrante, como parámetros de ruta, cadenas de consulta, encabezados y datos del cuerpo.
    
* `res` (response) es un objeto que se usa para enviar una respuesta de vuelta al cliente.
    

En este caso, `res.send("Welcome to the menu!")` envía una respuesta en texto plano al cliente.

Cuando un usuario navega a `/menu` en el navegador — o envía una solicitud GET a esa ruta — el servidor ejecuta este manejador y responde con el mensaje `Welcome to the menu!`.

Los manejadores de rutas son el núcleo de una aplicación Express. Permiten definir cómo responderán las diferentes rutas de tu aplicación web a las solicitudes. También puedes agregar lógica a estos manejadores, como procesar datos de formularios, interactuar con bases de datos o realizar validaciones.
