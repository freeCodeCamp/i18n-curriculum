# --description--

Un’altra parte importante di un errore è lo stack trace. Questo ti aiuta a capire quando l’errore è stato lanciato nella tua applicazione e quali funzioni sono state chiamate che hanno portato all’errore. Quindi, per esempio, se hai il seguente codice:

```javascript
const a = 5;
const b = 10;

function add() {
  return c;
}

function print() {
  add();
}

print();
```

La nostra funzione `print()` dovrebbe chiamare `add()`, che restituisce una variabile chiamata `c`, che attualmente non è stata dichiarata. L’errore corrispondente è il seguente:

<img src="https://cdn.statically.io/gh/TheOdinProject/curriculum/284f0cdc998be7e4751e29e8458323ad5d320303/foundations/javascript_basics/understanding_errors/imgs/01.png" width="100%" alt="Un errore mostrato con uno stacktrace nella console dello sviluppatore">

Lo stack trace ci dice che:

1. `c is not defined` è nell’ambito di `add()`, che è dichiarato alla riga 5.
1. `add()` è stato chiamato da `print()`, che è stato dichiarato alla riga 9.
1. `print()` stesso è stato chiamato alla riga 12.

Quindi lo stack trace ti permette di tracciare l’evoluzione di un errore fino alla sua origine, che qui è la dichiarazione di `add()`.

# --questions--

## --text--

In che modo lo stack trace aiuta a eseguire il debug dell’errore nel codice JavaScript fornito?

## --answers--  

Lo stack trace mostra che la funzione `add()` restituisce correttamente il valore di `c`.

---

Lo stack trace indica quale riga del codice deve essere modificata per correggere un errore di sintassi.

---

Lo stack trace fornisce un percorso dettagliato delle chiamate di funzione che portano all’errore, aiutando a identificare dove `c` è referenziato in modo errato.

---

Lo stack trace indica solo che `print()` è stato chiamato, ma non fornisce dettagli sull’errore in `add()`.
