# --description--

Wenn Sie beginnen, Ihre eigenen Webanwendungen zu erstellen, müssen Sie darauf achten, wie Sie Ihre Dateien benennen.

Sie sollten Ihre Dateien so benennen, dass sie leicht verständlich und wartbar sind.

Schauen wir uns einige Beispiele für gute und schlechte Dateinamen für HTML-Dateien an. Beachten Sie, dass die `.html`-Erweiterung in diesen Beispielen uns sagt, dass es sich um eine HTML-Datei handelt.

Hier ist ein Beispiel für einen schlechten Dateinamen:

```md
index1234.html
```

Allein am Namen ist nicht klar, worum es in der Datei geht. Diese Datei könnte für eine Produktseite, eine Blogseite oder jede andere Art von Seite sein, aber das wissen wir anhand des Dateinamens nicht.

Hier ist ein Beispiel für einen besseren Dateinamen für eine HTML-Datei:

```md
about-us.html
```

Durch die Verwendung eines beschreibenderen Namens wie `about-us.html` ist klar, worum es in der Datei geht.

Sie werden auch bemerken, dass in diesem Dateinamen keine Leerzeichen verwendet werden. Stattdessen verwenden Sie einen Bindestrich, `-`, um die Wörter zu trennen.

Manchmal sehen Sie einen Dateinamen namens `index.html`. Dies ist ein spezieller Dateiname, der verwendet wird, um die Hauptseite einer Website darzustellen. Wenn Sie eine Website besuchen, wird die `index.html`-Datei als erste geladen.

Eine weitere wichtige Überlegung bei der Benennung von Dateien ist die Verwendung von Sonderzeichen.

Die Verwendung einer Mischung aus Sonderzeichen, Zahlen und Buchstaben kann es schwierig machen zu verstehen, worum es in der Datei geht.

Hier ist ein Beispiel für einen schlechten Dateinamen:

```md
file-1!@.html
```

Dieser Name wirkt unnötig kompliziert und gibt uns keine Informationen darüber, worum es in der Datei geht.

Es ist also am besten, nur Buchstaben und Bindestriche in Ihren Dateinamen zu verwenden.

Wenn Sie anfangen, weitere Sprachen zu lernen, werden Sie sehen, dass es auch für diese Sprachen übliche Konventionen für die Benennung von Dateien gibt.

In diesen Fällen ist es immer am besten, die offizielle Dokumentation oder andere Ressourcen zu konsultieren, um die bewährten Vorgehensweisen für die Benennung von Dateien in dieser Sprache zu verstehen.

Außerdem werden Sie in Situationen sein, in denen Sie in einem Team arbeiten und die vom Team festgelegten Konventionen für die Benennung von Dateien befolgen müssen.

Konsultieren Sie immer den Styleguide des Teams, um sicherzustellen, dass Ihre Dateinamen mit denen des Teams übereinstimmen.
