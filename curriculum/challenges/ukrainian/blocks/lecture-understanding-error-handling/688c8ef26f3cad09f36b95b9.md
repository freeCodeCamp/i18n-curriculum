# --description--

Налагодження — це необхідна навичка для будь-якого розробника на Python. Розуміння базових технік допоможе вам ефективно знаходити та виправляти проблеми.

Налагодження — це процес виявлення та усунення помилок або багів у вашому коді. Він включає в себе аналіз коду, розуміння потоку виконання та використання інструментів для визначення джерела проблем.

У цьому уроці ми розглянемо загальні техніки налагодження, які ви можете застосувати у своєму наступному проєкті на Python.

## Використання функції print і f-рядків

По-перше, використання функції `print()` та f-рядків у різних місцях вашого коду допоможе зрозуміти потік виконання та стан змінних. Наприклад:

```py
def add(a, b):
    result = a + b
    print(f'Adding {a} and {b} gives {result}')
    return result
```

Виводячи значення `a`, `b` та `result`, ви можете перевірити, чи функція працює так, як очікується.

## Інтерактивне налагодження з модулем `pdb`

Далі ви можете скористатися вбудованим модулем `pdb` для інтерактивного налагодження:

```py
import pdb

def divide(a, b):
    pdb.set_trace()
    return a / b

print(divide(10, 2))
```

Встановивши трасування за допомогою методу `.set_trace()`, ви можете крокувати по коду, переглядати змінні та розуміти поведінку програми.

Якщо запустити наведений код, ви побачите вихідні дані, які показують розташування файлу, який ви запускаєте, лінію, де ви викликали метод `.set_trace()`, та код безпосередньо після нього, а також інтерактивний запит `pdb`:

```py
> /Users/fcc/Desktop/debugging.py(5)divide()
-> return a / b
(Pdb)
```

Якщо ввести help у запиті, ви побачите список команд, які можна використовувати:

```py
(Pdb) help

Documented commands (type help <topic>):
========================================
EOF    c          d        h         list      q        rv       undisplay
a      cl         debug    help      ll        quit     s        unt      
alias  clear      disable  ignore    longlist  r        source   until    
args   commands   display  interact  n         restart  step     up       
b      condition  down     j         next      return   tbreak   w        
break  cont       enable   jump      p         retval   u        whatis   
bt     continue   exit     l         pp        run      unalias  where    

Miscellaneous help topics:
==========================
exec  pdb
```

Потім ви можете застосовувати ці команди для налагодження коду.

Наприклад, якщо ви хочете подивитися типи елементів у вашому коді в цей момент, можна використати команду `whatis`:

```py
(Pdb) whatis a
<class 'int'>
(Pdb) whatis divide
Function divide
```

Як бачите, до моменту виконання `.set_trace()` тип параметра a — ціле число, а divide — функція.

Щоб продовжити виконання коду, можна використати команду `continue` або її псевдоніми `cont` чи `c`:

```py
(Pdb) continue
5.0
```

## Інструменти налагодження в IDE

Багато інтегрованих середовищ розробки (IDE) пропонують розширені інструменти налагодження, такі як точки зупинки, покрокове виконання та перегляд змінних.

## Використання налагоджувача VS Code

Якщо ви користуєтеся VS Code, ви можете встановлювати точки зупинки у своєму коді та запускати налагоджувач, щоб призупинити виконання в цих точках. Ось як налагодити ту саму функцію `divide`:

**Крок 1: Підготуйте код** Створіть файл з назвою `main.py` з таким вмістом:

```py
def divide(a, b):
    result = a / b
    return result

print(divide(10, 2))
print(divide(15, 3))
```

**Крок 2: Встановіть точку зупинки**

1. Натисніть у вертикальному розриві (лівому полі) поруч із лінією 2 (`result = a / b`), щоб встановити точку зупинки

2. З’явиться червона крапка, що означає встановлену точку зупинки

**Крок 3: Почніть налагодження**

1. Натисніть `F5` або перейдіть у Run > Start Debugging

2. Виберіть "Python File" при запиті

3. Налагоджувач призупинить виконання на вашій точці зупинки

**Крок 4: Перегляньте змінні**

- Наведіть курсор на змінні, щоб побачити їхні поточні значення

- Використовуйте панель Variables зліва, щоб побачити всі локальні змінні

- Використовуйте Debug Console внизу для оцінки виразів

**Крок 5: Крокуйте по коду**

Використовуйте панель налагодження для:

- **Continue (F5)**: Продовжити виконання до наступної точки зупинки

- **Step Over (F10)**: Виконати поточну лінію та перейти до наступної

- **Step Into (F11)**: Увійти у виклики функцій

- **Step Out (Shift+F11)**: Вийти з поточної функції

Інструменти налагодження в IDE надають візуальний інтерфейс для перегляду стану вашої програми, що значно полегшує виявлення та виправлення проблем у порівнянні з використанням лише інструкцій print.

Опанувавши ці базові техніки налагодження — використання інструкцій `print()`, модуля pdb та інструментів IDE — ви зможете ефективно знаходити та усувати проблеми у своєму Python-коді. Кожна техніка має своє призначення: інструкції `print()` для швидких перевірок, pdb для інтерактивного дослідження та налагоджувачі IDE для візуального аналізу.
