# --description--

في الدروس السابقة، تعلمت كيفية العمل مع الاستيراد والتصدير في JavaScript. في هذا الدرس، سنلقي نظرة على كيفية استيراد وتصدير المكوّنات في React.

التصدير يجعل المكوّن متاحًا لملفات أخرى. الاستيراد يسمح لملف باستخدام مكوّن تم تصديره في مكان آخر. في مشاريع React، هذه هي الطريقة التي تُعاد بها استخدام المكوّنات وتُدمج لبناء واجهات المستخدم.

في هذا الدرس، ستتعلم كيفية تصدير مكوّنات React باستخدام الصادرات الافتراضية والمُسماة، وكيف تستوردها حيثما تحتاجها.

في هذا المثال، لدينا مكوّن `Cat` ينتمي إلى ملف يسمى `Cat.jsx`. نستخدم امتداد الملف `.jsx` لأن هذا الملف يعمل بشكل رئيسي مع JSX.

هذا المكوّن `Cat` يُرجع ترميز JSX يحتوي على عنوان وصورة لقطة تُدعى Mr. Whiskers:

```jsx
function Cat() {
  return (
    <div className="card">
      <h2>Mr. Whiskers</h2>
      <img
        src="https://cdn.freecodecamp.org/curriculum/cat-photo-app/running-cats.jpg"
        alt="Tuxedo cats running on dirt ground."
      />
    </div>
  );
}
```

إذا أردنا استخدام مكوّن `Cat` في ملف آخر، يجب أولًا تصديره بهذا الشكل:

```jsx
function Cat() {
  return (
    <div className="card">
      <h2>Mr. Whiskers</h2>
      <img
        src="https://cdn.freecodecamp.org/curriculum/cat-photo-app/running-cats.jpg"
        alt="Tuxedo cats running on dirt ground."
      />
    </div>
  );
}

export default Cat;
```

نستخدم الكلمة المفتاحية `default` لأنها ستكون التصدير الافتراضي من الوحدة. يمكنك أيضًا اختيار تصدير المكوّن في نفس سطر تعريف المكوّن بهذا الشكل:

```jsx
export default function Cat() {
  return (
    <div className="card">
      <h2>Mr. Whiskers</h2>
      <img
        src="https://cdn.freecodecamp.org/curriculum/cat-photo-app/running-cats.jpg"
        alt="Tuxedo cats running on dirt ground."
      />
    </div>
  );
}
```

يمكنك اختيار استيراد المكوّنات الفرعية في ملفات المكوّنات الأصلية الأخرى، أو استيرادها في ملف المكوّن الجذري. في هذا المثال، سنستورد مكوّن `Cat` داخل ملف المكوّن الجذري.

كل مشروع React يحتوي على مكوّن على المستوى الأعلى، يُسمى عادةً `App.jsx`:

```jsx
export default function App() {
  return (
    // return component here
  );
}
```

عادةً ما يكون هذا الملف موجودًا في دليل `src` الخاص بمشروعك. ستتعلم المزيد عن تخطيطات المشاريع الشائعة في درس لاحق.

لاستخدام مكوّن `Cat` داخل المكوّن الجذري `App`، ستحتاج إلى استيراده بهذا الشكل:

```jsx
import Cat from "./Cat";

export default function App() {
  return (
    // return component here
  );
}
```

الآن، يمكنك إرجاع مكوّن `Cat` داخل مكوّن `App` بهذا الشكل:

```jsx
import Cat from "./Cat";

export default function App() {
  return (
    <Cat />
  );
}
```

تعمل هذه الطريقة جيدًا عندما يكون الملف مسؤولًا عن تصدير مكوّن رئيسي واحد، مما يحافظ على صيغة الاستيراد بسيطة وسهلة القراءة. تذكر، يمكن للملف أن يحتوي على تصدير افتراضي واحد فقط، مما يجعله مثاليًا لملف يحتوي بشكل أساسي على مكوّن واحد.


الآن بعد أن عرفت كيفية استخدام التصدير الافتراضي، دعنا نلقي نظرة على التصدير المسمي. يسمح التصدير المسمي لملف بمشاركة عدة مكوِّنات أو دوال. على عكس التصدير الافتراضي، يجب استيرادها باستخدام الاسم الدقيق الذي تم تصديرها به (إلا إذا قمت بإعادة تسميتها باستخدام الكلمة المفتاحية `as`).

هذا مفيد عندما يكون الملف بمثابة مكتبة مكوّنات. على سبيل المثال، هنا ملف يحتوي على مكوّنين، Cat و Dog:

```jsx
// Animals.jsx
export function Cat() {
  return <h2>Mr. Whiskers</h2>;
}

export function Dog() {
  return <h2>Fido</h2>;
}
```

نستخدم كلمة export بشكل فردي على كل مكوّن. لاستخدامها في ملف آخر، نستوردها باستخدام الأقواس المتعرجة:


```jsx
import { Cat, Dog } from "./Animals";

export default function App() {
  return (
    <div>
      <Cat />
      <Dog />
    </div>
  );
}
```

الصيغة `{ Cat, Dog }` تخبر جافاسكريبت باستيراد التصديرات المسماة المحددة من ملف `Animals.jsx`. يجب أن تتطابق الأسماء داخل الأقواس المتعرجة مع الأسماء المصدرة تمامًا. إذا كنت بحاجة لتجنب تعارض الأسماء، أو تفضل اسمًا مختلفًا في الملف الحالي، يمكنك إعادة تسمية المكوِّن أثناء الاستيراد باستخدام الكلمة المفتاحية `as`.

```jsx
import { Cat as Kitty } from "./Animals";

export default function App() {
  return <Kitty />;
}
```

أخيرًا، يمكن للملف أن يحتوي على تصدير افتراضي واحد وعدة صادرات مُسماة.

```jsx
// Animals.jsx
export default function Cat() {
  return <h2>Mr. Whiskers</h2>;
}

export function Dog() {
  return <h2>Fido</h2>;
}
```

عند استيراد الصادرات المختلطة، يأتي التصدير الافتراضي أولًا (بدون أقواس)، يليه الصادرات المُسماة (داخل الأقواس):

```jsx
// App.jsx
import Cat, { Dog } from "./Animals";

export default function App() {
  return (
    <div>
      <Cat />
      <Dog />
    </div>
  );
}
```

في هذا المثال، `Cat` هو التصدير الافتراضي، لذا لا يستخدم الأقواس المتعرجة. أما `Dog` فهو تصدير مُسمى، لذا يتطلب الأقواس.

بفهم كيفية عمل الصادرات الافتراضية والمُسماة، يمكنك تنظيم مكوّنات React عبر ملفات متعددة وإعادة استخدامها حيثما تحتاج. مع بناء تطبيقات أكبر، اختيار نمط التصدير المناسب يساعد في الحفاظ على شفرتك واضحة وقابلة للصيانة، ويسهل العمل مع المكوّنات التي أنشأها الآخرون.
