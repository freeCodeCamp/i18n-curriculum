# --description--

리스트 렌더링은 React 웹 애플리케이션에서 기본적인 작업이며, 사용자에게 데이터를 보여줄 때 사용됩니다. React에서는 `map` 메서드를 사용해 데이터 배열을 UI에서 렌더링할 수 있는 JSX 요소 배열로 변환합니다.

다음은 과일 목록을 보여주는 `FruitList` 컴포넌트 예시입니다:

```jsx
function FruitList() {
  const fruits = ['Apple', 'Banana', 'Cherry', 'Date'];
  return (
    <ul>
      {fruits.map(fruit => <li>{fruit}</li>)}
    </ul>
  );
}
```

이 예시에서 `map` 함수는 `fruits` 배열의 각 항목을 반복합니다. 각 과일마다 과일 이름을 포함하는 새로운 `li` 요소를 만듭니다. 새로 생성된 `li` 요소 배열은 `ul` 부모 태그 안에 표시됩니다.

하지만 React에서 리스트를 렌더링할 때는 각 요소에 `key` 프로프를 꼭 잊지 말아야 합니다. key는 항상 고유해야 하며, React가 어떤 항목이 변경되었고, 추가되었으며, 제거되었는지 식별하는 데 도움을 줍니다. 이는 리스트를 효율적으로 렌더링하고 업데이트하는 데 필수적입니다.

key를 잊으면 React가 콘솔에 경고를 표시하지만 치명적인 오류를 발생시키지는 않습니다. 애플리케이션은 여전히 렌더링되고 작동할 수 있지만, 특히 리스트가 변경될 때 미묘한 버그가 발생할 수 있습니다. 이러한 버그는 UI가 처음에는 올바르게 보이기 때문에 디버그하기 어려울 수 있습니다.

일반적인 실수는 배열 인덱스를 key로 사용하는 것입니다. 예를 들어 다음과 같습니다:

```jsx
{fruits.map((fruit, index) => <li key={index}>{fruit}</li>)}
```

이렇게 하면 경고는 사라지지만, 일반적으로 안티패턴으로 간주됩니다. 인덱스를 key로 사용하면 리스트가 재정렬되거나 정렬, 필터링될 때 문제가 발생할 수 있습니다. React는 key를 사용해 요소를 추적합니다. 리스트 순서가 바뀌면 React가 컴포넌트 상태를 잘못 재사용하거나, 인덱스가 같아도 내용이 바뀌었는데도 DOM을 효율적으로 업데이트하지 못할 수 있습니다.

모범 사례는 각 항목에 대해 안정적이고 고유한 식별자를 사용하는 것입니다. 보통 데이터베이스의 UUID나 데이터베이스 ID 같은 ID를 사용합니다.

데이터에 고유 ID가 없다면, 데이터를 생성할 때 (예: `crypto.randomUUID()` 또는 `uuid` 같은 라이브러리를 사용해) 생성하거나, 고유함이 보장되는 필드 조합을 사용할 수 있습니다. 하지만 렌더링 중에 키를 즉석에서 생성하는 것(예: `key={Math.random()}`)은 피해야 합니다. 이렇게 하면 React가 매 렌더링마다 DOM 요소를 다시 만들고 상태를 초기화하기 때문입니다.

예제를 키를 포함하도록 수정해 보겠습니다:

```jsx
function FruitList() {
  const fruits = ["Apple", "Banana", "Cherry", "Date"];
  return (
    <ul>
      {fruits.map((fruit, index) => (
        <li key={`${fruit}-${index}`}>{fruit}</li>
      ))}
    </ul>
  );
}
```

이 리팩터링된 예시에서는 과일 이름과 인덱스를 연결해 각 리스트 항목에 고유한 키를 만듭니다. 이렇게 하면 각 리스트 항목이 고유한 키를 가져 React가 항목이 추가, 제거, 재정렬될 때 리스트를 효율적으로 관리하고 업데이트할 수 있습니다.

React는 더 복잡한 구조도 렌더링할 수 있습니다. 예를 들어, 여러 속성을 가진 사용자 객체 배열이 있고, 이를 표시하고 싶을 때 다음과 같이 할 수 있습니다:

```jsx
function UserList() {
  const users = [
    { id: "user-001-employee", name: "Alice", email: "alice@example.com" },
    { id: "user-002-employee", name: "Bob", email: "bob@example.com" },
    { id: "user-003-employee", name: "John", email: "john@example.com" },
  ];
  return (
    <div>
      {users.map((user) => (
        <div key={user.id}>
          <h3>{user.name}</h3>
          <p>{user.email}</p>
        </div>
      ))}
    </div>
  );
}
```

이 예시에서는 각 사용자에 대해 이름과 이메일 주소를 모두 보여주는 더 복잡한 JSX 구조를 만듭니다. 사용자의 `id`를 `key`로 사용하고 있는데, 이는 좋은 모범 사례입니다.

결론적으로 React에서 리스트를 렌더링하는 것은 보통 `map` 함수를 사용해 데이터 배열을 JSX 요소로 변환하는 작업입니다.
