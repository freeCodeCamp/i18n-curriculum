# --interactive--

在使用 CSS 时，你经常会遇到元素的三种不同显示行为类型：`inline`、`block` 和 `inline-block`。

这些属性中的每一个都会影响元素的定位方式以及它们与页面上其他元素的交互方式。

在本课中，我们将重点讲解 `inline-block` 属性的工作原理以及它与内联和块级元素的区别。

块级元素表现得像大型容器或占据其父容器全宽的“块”。它们总是在新行开始，并且可以调整其高度和宽度。

内联元素只占用它们所需的空间。它们在周围内容中流动，不会换到新行。

`inline-block` 属性是这两种行为的混合体。像内联元素一样，`inline-block` 元素保持在文本流中而不会另起一行。

但是，与内联元素不同，你可以像调整块级元素一样调整 `inline-block` 元素的宽度和高度。

简而言之，`inline` 和 `inline-block` 之间的关键区别在于 `inline` 元素无法控制其大小，而 `inline-block` 元素则允许完全控制维度，同时仍保持与其他内容内联。

让我们来看一个例子。

:::interactive_editor

```html
<link href="styles.css" rel="stylesheet">

<div class="container">
  <span class="inline-block-element element1">Inline-Block</span>
  <span class="inline-block-element element2">Inline-Block</span>
</div>
```

```css
.inline-block-element {
  display: inline-block;
  width: 150px;
  height: 100px;
}

.element1 {
  background-color: lightblue;
}

.element2 {
  background-color: lightgreen;
}
```

:::

在上述示例中，我们有一个类为 `container` 的 `div`。在该 `div` 元素内部，我们有两个 `span` 元素。

每个 span 元素都设置为 `display:inline-block`，并且还设置了宽度和高度。

内联块元素会并排显示，因为它们表现得像内联元素，但它们也有指定的宽度和高度，这赋予了它们块状的属性。

但是，如果你移除 `display: inline-block` 属性，即使你明确定义了，`height` 和 `width` 也都不会被应用。

这是修订后的 CSS：

:::interactive_editor

```html
<link href="styles.css" rel="stylesheet">

<div class="container">
  <span class="inline-block-element element1">Span element</span>
  <span class="inline-block-element element2">Span element</span>
</div>
```

```css
.inline-block-element {
  width: 150px;
  height: 100px;
}

.element1 {
  background-color: lightblue;
}

.element2 {
  background-color: lightgreen;
}
```

:::

在这段代码中，我们移除了 `display: inline-block;` 属性，但保持了其他所有内容不变。在这里，`span` 元素恢复为其作为内联元素的默认行为。

因此，指定的宽度和高度将被忽略，元素仅占用其内容所需的空间。

理解 `inline-block` 的工作原理很有用，因为你可以用它来创建既需要对齐又需要尺寸控制的版型，同时保持连续的文本流。
