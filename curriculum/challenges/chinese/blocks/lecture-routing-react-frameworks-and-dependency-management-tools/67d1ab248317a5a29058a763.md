# --description--

在软件中，依赖是指应用中的一个组件或模块依赖另一个组件或模块才能正常运行。依赖在软件应用中很常见，因为它允许开发者使用由其他人创建的预构建函数或工具。

如果你想构建一个 React 应用，你需要安装 React 依赖项。没有这些依赖项，你的代码将无法正常工作，应用将显示一系列误差。

当你处理更复杂的项目时，你通常需要依赖许多依赖。如果一组依赖在项目中没有得到良好管理或定义，那么这将导致所谓的依赖地狱。

要管理项目中的软件依赖关系，你需要使用包管理器。包管理器是一种用于安装、更新和移除依赖项的工具。许多流行的编程语言，如 JavaScript、Python、Ruby 和 Java，都使用包管理器。

在之前的课程中，你曾简要了解过一个流行的包管理器，叫做 npm。

要使用 Vite 和 npm 创建一个新的 React 项目，你可以运行以下命令：

```sh
npm create vite@latest my-react-app -- --template react
```

正如你从之前的课程中回忆的，这将创建启动一个新 React 应用所需的所有公式化代码。在你能够启动应用之前，你需要通过运行 `npm install` 或简写的 `npm i` 来安装依赖项。

你可以查看位于你项目根目录中的 `package.json` 文件中的所有依赖项。

`package.json` 文件是项目中的关键配置文件，包含有关你的项目的元数据，包括其名称、版本和依赖项。它还定义了脚本、许可信息以及帮助管理项目及其依赖项的其他设置。

当你安装依赖时，`node_modules` 文件夹将被添加到你的项目。

`node_modules` 文件夹是存放你的项目所需的所有包和库的地方。该文件夹包含 `package.json` 文件中列出的依赖项的实际代码，包括你项目的直接依赖项以及这些依赖项的任何依赖项。

React 项目所需的两个核心依赖将是 `react` 和 `react-dom` 包：

```json
"dependencies": {
  "react": "^18.3.1",
  "react-dom": "^18.3.1"
}
```

`package.json` 会列出你当前使用的那些包的版本。如果你需要在局部更新任何包，可以运行 `npm update` 命令。或者你可以通过运行 `npm update -g` 命令来全局更新所有包。

除了 `package.json` 文件，你还会有一个 `package-lock.json` 文件。该文件会锁定你的项目正在使用的所有包的确切版本。当你更新包时，锁定文件中的新版本也会被更新。

`package.json` 文件的另一个重要方面是开发依赖项：

```json
"devDependencies": {
  "@eslint/js": "^9.17.0",
  "@types/react": "^18.3.18",
  "@types/react-dom": "^18.3.5",
  "@vitejs/plugin-react": "^4.3.4",
  "eslint": "^9.17.0",
  "eslint-plugin-react": "^7.37.2",
  "eslint-plugin-react-hooks": "^5.0.0",
  "eslint-plugin-react-refresh": "^0.4.16",
  "globals": "^15.14.0",
  "vite": "^6.0.5"
}
```

开发依赖是仅用于开发而不在生产环境中使用的包。一个例子是像 Jest 这样的测试库。你会将 Jest 安装为开发依赖，因为你只在本地测试你的项目时使用它，而生产环境中应用运行时不需要它。

在本课的大部分时间里，我们一直专注于 npm。但还有其他包管理器，比如 yarn 和 pnpm。那么你应该为你的项目使用哪个包管理器？

嗯，简短的回答是这取决于情况。

如果你正在加入一个已有团队的项目，那么所有这些决定都会为你做出，你将使用现有的包管理器。如果你是从零开始创建一个项目，那么你需要研究每个管理器的优缺点，并决定哪个更适合你的需求。
