## Validierung von Formularen mit JavaScript

- **Constraint Validation API**: Bestimmte HTML-Elemente, wie die `textarea` und `input` Elemente, stellen eine Constraint Validation API bereit. Diese API erlaubt es Ihnen zu prüfen, ob der vom Benutzer eingegebene Wert für dieses Element alle HTML-Validierungen erfüllt, die Sie definiert haben, wie Mindestlänge oder Mustererkennung.
- **`checkValidity()` Methode**: Diese Methode gibt `true` zurück, wenn das Element alle HTML-Validierungen (basierend auf seinen Attributen) besteht, und `false`, wenn es fehlschlägt.

:::interactive_editor

```html
<form>
  <label>
    Email:
    <input
      id="email"
      type="email"
      required
      pattern=".+\.com$"
      placeholder="example@site.com"
    />
  </label>
</form>

<script>
  const input = document.getElementById("email");

  input.addEventListener("input", (e) => {
    if (!e.target.checkValidity()) {
      e.target.setCustomValidity("You must use a .com email.");
    } else {
      e.target.setCustomValidity("");
    }
  });
</script>
```

:::

- **`reportValidity()` Methode**: Diese Methode teilt dem Browser mit, dass das `input` ungültig ist.

:::interactive_editor

```html
<form>
  <label>
    Email:
    <input
      id="email2"
      type="email"
      required
      pattern=".+\.com$"
      placeholder="example@site.com"
    />
  </label>
</form>

<script>
  const input = document.getElementById("email2");

  input.addEventListener("input", (e) => {
    if (!e.target.checkValidity()) {
      e.target.reportValidity();
    }
  });
</script>
```

:::

- **`validity` Eigenschaft**: Diese Eigenschaft wird verwendet, um den Gültigkeitsstatus von Formularsteuerelementen (wie `<input>`, `<select>`, etc.) zu erhalten und liefert Informationen darüber, ob die Benutzereingabe die für dieses Element definierten Einschränkungen erfüllt (z. B. `required` Felder, Musterbeschränkungen, maximale Länge usw.).

:::interactive_editor

```html
<input
  id="age"
  type="number"
  min="18"
  placeholder="Enter age (18+)"
/>

<script>
  const input = document.getElementById("age");

  input.addEventListener("input", (e) => {
    console.log(e.target.validity);
  });
</script>
```

:::

- **`patternMismatch` Eigenschaft**: Diese ist `true`, wenn der Wert nicht dem angegebenen regulären Ausdrucksmuster entspricht.

## `preventDefault()` Methode

- **Definition**: Jedes Ereignis, das im DOM ausgelöst wird, hat eine Art Standardverhalten. Das Klick-Ereignis auf ein Kontrollkästchen schaltet standardmäßig den Zustand dieses Kontrollkästchens um. Das Drücken der Leertaste auf einem fokussierten Button aktiviert den Button. Die `preventDefault()` Methode bei diesen `Event` Objekten verhindert, dass dieses Verhalten eintritt.

:::interactive_editor

```html
<form id="form">
  <input type="text" placeholder="Try to submit" />
  <button type="submit">Submit</button>
</form>

<p id="status"></p>

<script>
  const form = document.getElementById("form");
  const status = document.getElementById("status");

  form.addEventListener("submit", (event) => {
    event.preventDefault();
    status.textContent = "Form submission prevented.";
  });
</script>
```

:::

## Absenden von Formularen

- **Definition**: Es gibt drei Möglichkeiten, wie ein Formular abgesendet werden kann. Die erste ist, wenn der Benutzer auf einen Button im Formular klickt, der das `type` Attribut mit dem Wert `submit` hat. Die zweite ist, wenn der Benutzer die `Enter` Taste in einem beliebigen bearbeitbaren `input` Feld im Formular drückt. Die dritte ist ein JavaScript-Aufruf der `requestSubmit()` oder `submit()` Methoden des `form` Elements.
- **`action` Attribut**: Das `action` Attribut sollte entweder eine URL oder einen relativen Pfad für die aktuelle Domain enthalten. Dieser Wert bestimmt, wohin das Formular versucht, die Daten zu senden – wenn Sie kein `action` Attribut setzen, sendet das Formular die Daten an die URL der aktuellen Seite.

```html
<form action="https://freecodecamp.org" method="GET">
  <input
    type="number"
    name="number"
    placeholder="Enter a number"
  />
  <button type="submit">Submit</button>
</form>
```

- **`method` Attribut**: Dieses Attribut akzeptiert eine Standard-`HTTP` Methode, wie `GET` oder `POST`, und verwendet diese Methode, wenn die Anfrage an die Aktions-URL gesendet wird. Wenn keine Methode gesetzt ist, verwendet das Formular standardmäßig eine `GET` Anfrage. Die Daten im Formular werden als URL-kodierte `name=value` Paare kodiert und als Abfrageparameter an die Aktions-URL angehängt.

```html
<form action="/data" method="POST">
  <input
    type="number"
    id="input"
    placeholder="Enter a number"
    name="number"
  />
  <button type="submit">Submit</button>
</form>
```

- **`enctype` Attribut**: Das `form` Element akzeptiert ein `enctype` Attribut, das den zu verwendenden Kodierungstyp für die Daten angibt. Dieses Attribut akzeptiert nur drei Werte: `application/x-www-form-urlencoded` (Standard, sendet die Daten als URL-kodierten Formular-Body), `text/plain` (sendet die Daten im Klartext, als `name=value` Paare, getrennt durch Zeilenumbrüche) oder `multipart/form-data`, das speziell für Formulare mit Datei-Uploads gedacht ist.
