# --description--

<dfn>Emit</dfn> ist die gebräuchlichste Art der Kommunikation, die Sie verwenden werden. Wenn Sie vom Server aus etwas an 'io' emittieren, senden Sie den Namen eines Ereignisses und Daten an alle verbundenen Sockets. Ein gutes Beispiel für dieses Konzept wäre, bei jeder neuen Verbindung eines Nutzers die aktuelle Anzahl der verbundenen Nutzer zu emittieren!

Beginnen Sie damit, eine Variable hinzuzufügen, um die Nutzer zu zählen, direkt vor der Stelle, an der Sie aktuell auf Verbindungen hören.

```js
let currentUsers = 0;
```

Wenn sich nun jemand verbindet, sollten Sie die Anzahl vor dem Emittieren erhöhen. Fügen Sie also den Inkrementierer innerhalb des Verbindungs-Listeners hinzu.

```js
++currentUsers;
```

Schließlich sollten Sie nach dem Erhöhen der Anzahl das Ereignis emittieren (immer noch innerhalb des Verbindungs-Listeners). Das Ereignis sollte 'user count' heißen, und die Daten sollten einfach der `currentUsers` sein.

```js
io.emit('user count', currentUsers);
```

Jetzt können Sie eine Möglichkeit implementieren, damit Ihr Client auf dieses Ereignis hört! Ähnlich wie beim Lauschen auf eine Verbindung auf dem Server verwenden Sie das Schlüsselwort `on`.

```js
socket.on('user count', function(data) {
  console.log(data);
});
```

Laden Sie nun Ihre App, authentifizieren Sie sich, und Sie sollten in der Konsole Ihres Clients eine '1' sehen, die die aktuelle Nutzeranzahl darstellt! Versuchen Sie, weitere Clients zu laden und sich zu authentifizieren, um zu sehen, wie die Zahl steigt.

Senden Sie Ihre Seite ab, wenn Sie denken, dass alles richtig ist. Wenn Sie auf Fehler stoßen, können Sie <a href="https://forum.freecodecamp.org/t/advanced-node-and-express/567135#communicate-by-emitting-7" target="_blank" rel="noopener noreferrer nofollow">das Projekt bis zu diesem Punkt fertiggestellt ansehen</a>.
