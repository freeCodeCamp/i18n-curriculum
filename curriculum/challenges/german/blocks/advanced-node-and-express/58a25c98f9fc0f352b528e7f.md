# --description--

Zurück zum Abschnitt Informationssicherheit: Sie erinnern sich vielleicht, dass das Speichern von Klartext-Passwörtern *niemals* in Ordnung ist. Jetzt ist es an der Zeit, BCrypt zu implementieren, um dieses Problem zu lösen.

`bcrypt@~5.0.0` wurde bereits als Abhängigkeit hinzugefügt, also importieren Sie es in Ihrem Server. Sie müssen das Hashen an zwei wichtigen Stellen handhaben: dort, wo Sie ein neues Konto registrieren/speichern, und wenn Sie überprüfen, ob ein Passwort beim Login korrekt ist.

Derzeit fügen Sie in Ihrer Registrierungs-Route das Klartext-Passwort eines Benutzers so in die Datenbank ein: `password: req.body.password`. Hashen Sie stattdessen die Passwörter, indem Sie Folgendes vor Ihrer Datenbank-Logik hinzufügen: `const hash = bcrypt.hashSync(req.body.password, 12);`, und ersetzen Sie das `req.body.password` beim Speichern in der Datenbank durch einfach `password: hash`.

In Ihrer Authentifizierungsstrategie prüfen Sie vor Abschluss des Prozesses Folgendes in Ihrem Code: `if (password !== user.password) return done(null, false);`. Nach den vorherigen Änderungen ist `user.password` nun ein Hash. Bevor Sie den bestehenden Code ändern, beachten Sie, dass die Anweisung überprüft, ob das Passwort **nicht** übereinstimmt, und dann nicht authentifiziert zurückgibt. Ändern Sie den Code daher wie folgt, um das eingegebene Passwort korrekt mit dem Hash zu vergleichen:

```js
if (!bcrypt.compareSync(password, user.password)) { 
  return done(null, false);
}
```

Das ist alles, was es braucht, um eine der wichtigsten Sicherheitsfunktionen zu implementieren, wenn Sie Passwörter speichern müssen.

Senden Sie Ihre Seite ab, wenn Sie denken, dass alles richtig ist. Wenn Sie auf Fehler stoßen, können Sie <a href="https://forum.freecodecamp.org/t/advanced-node-and-express/567135#hashing-your-passwords-1" target="_blank" rel="noopener noreferrer nofollow">das bis zu diesem Punkt abgeschlossene Projekt ansehen</a>.
