# --description--

CSSプリプロセッサは、変数、ミックスイン、ネスト、セレクターの継承などの強力な機能で標準のCSSを拡張するツールです。

これらの機能のうち、変数やネストは現在ネイティブCSSでサポートされているか、より多くのサポートが進んでいるところで、この傾向は今後も続く可能性が高いです。しかし、以前のバージョンのネイティブCSSではこれらの機能はサポートされていませんでした。

だからこそ、CSSプリプロセッサが最初に広く使われるようになったのです。プリプロセッサは、あなたが書くCSSルールやスタイルとその拡張構文を受け取り、ブラウザが理解できるネイティブCSSファイルにコンパイルします。

CSSプリプロセッサを使うことで、CSSコードをより再利用可能で論理的な構造にできます。また、CSSで直接サポートされていないミックスインのような強力な機能も利用できます。

最も人気のあるCSSプリプロセッサには、Sass、Less、Stylusがあります。

ここではSassについて話しましょう。

Sassは「Syntactically Awesome Style Sheets」の略です。すべてのCSSバージョンと互換性があり、多くの開発者コミュニティによってメンテナンスされています。

Sassは以下のような機能をサポートしています：

- 変数：値を保存してスプレッドシート全体で再利用できます。

- ネストされたCSSルール：ファイル内に視覚的な階層を作成できます。

- モジュール：スタイルを複数のスタイルシートに分割できます。

- ミックスイン：サイト全体でCSS宣言を再利用できます。

- 継承：複数のCSSセレクターがプロパティを共有できます。

- 演算子：基本的な数学演算ができます。

Sassは2つの構文もサポートしています。

SCSS構文はSassy CSSの略で、CSSのスーパーセットです。つまり、SCSSはCSSの基本構文を拡張しています。

SCSSはSassで最も一般的に使われる構文で、Sassを使う際によく見かけます。ネイティブCSSと同様に、CSSプロパティの周りに波括弧、中括弧（`{` と `}`）を使い、CSS宣言の末尾にセミコロン（`;`）が必要です。

これらのファイルは`.scss`拡張子を持ちます。例を示します：

```scss
$primary-color: #3498eb;

header {
  background-color: $primary-color;
}
```

ここで上部に見えるのは、SCSSで定義された変数です。この変数は下のCSSルールで使われています。

SCSSとあまり使われない構文であるインデント構文を比較してみましょう。これは「Sass構文」とも呼ばれ、Sassの元々の構文です。

この構文はインデントでルールを定義します。例を示します：

```scss
$primary-color: #3498eb

header
  background-color: $primary-color
```

CSSルールの周りに波括弧やセミコロンがないことに注目してください。上部には変数も見えます。この構文では変数名がドル記号（`$`）で始まっており、ネイティブCSSの変数とは異なりますが、非常によく似た動作をします。

Sassはミックスインという強力な機能もサポートしています。

ミックスインは複数のCSSプロパティとその値を名前の下にまとめ、そのCSSコードのブロックをスタイルシート全体で再利用できます。

これにより、CSSコードの繰り返しが減り、メンテナンスが簡単になります。ブロック内の何かを変更すれば、そのミックスインを使っているすべての場所に変更が適用されます。

ミックスインはカスタムで説明的な名前を付けられるため、各CSSコードのブロックが何をしているか理解しやすくなり、スタイルシートの理解が容易になります。

また、ミックスインはスタイルシートの一貫性を促進し、均一なスタイルを適用します。

これはSCSS構文でのミックスインの例です：

```scss
@mixin center-flex {
  display: flex;
  justify-content: center;
  align-items: center;
}
```

ミックスインを定義するには、まず`@mixin`アットルールを書き、続けてミックスインの名前を書きます。この場合、ミックスインの名前は`center-flex`です。Flexboxを使って要素を中央に配置する3つのCSSプロパティがあります。

ミックスインを定義したら、`@include`アットルールを使ってCSSルール内にそのプロパティを含めます。`@include`の後にミックスインの名前を書くだけです。この場合、名前は`center-flex`です：

```scss
section {
  @include center-flex;
  height: 500px;
  background-color: #3289a8;
}
```

ミックスインとCSSルールを含む完全なコードは以下の通りです：

```scss
@mixin center-flex {
  display: flex;
  justify-content: center;
  align-items: center;
}

section {
  @include center-flex;
  height: 500px;
  background-color: #3289a8;
}
```

必要なだけ多くのCSSルールにミックスインを含めることができます。変更が必要な場合はミックスイン自体を変更するだけで、その変更はすべての使用箇所に自動的に適用されます。この例はSCSS構文で、波括弧とセミコロンがあることに注目してください。

同じものをインデント構文（Sass構文とも呼ばれます）で示すと、波括弧やセミコロンはありません：

```scss
@mixin center-flex
  display: flex
  justify-content: center
  align-items: center

section
  @include center-flex
  height: 500px
  background-color: #3289a8
```

そしてこれはコンパイルされたCSSコードで、ファイルがネイティブCSSにコンパイルされた後にブラウザが実際に解釈するものです：

```css
section {
  display: flex;
  justify-content: center;
  align-items: center;
  height: 500px;
  background-color: #3289a8;
}
```

上部にミックスインの3つのプロパティ、`display`、`justify-content`、`align-items`が含まれていることに注目してください。

これはCSSプリプロセッサができることの単純な例にすぎません。学んで日々の作業に活かす価値のある多くの強力な機能があります。

しかし、知っておくべき潜在的なデメリットもあります。

まず、CSSプリプロセッサはCSSルールを標準のCSSにコンパイルする必要があります。このコンパイル処理のオーバーヘッドは時間やリソースの面で通常は最小限ですが、実際のプロジェクトでは考慮すべき点です。

次に、ブラウザはコンパイルされたCSSを直接使うため、デバッグが難しくなる場合があります。拡張構文から問題のあるスタイルを特定するには、標準CSSと比べて少し手間がかかることがあります。

しかし、CSSプリプロセッサの利点は通常、特に複雑なプロジェクトではデメリットを上回ります。よりクリーンで再利用可能、繰り返しが少なく、スケーラブルなCSSを書くのに非常に役立ちます。
