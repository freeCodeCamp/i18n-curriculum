# --interactive--

Die `audio`- und `video`-Elemente ermöglichen es Ihnen, Ton- und Videoinhalte in Ihre HTML-Dokumente einzufügen. Das `audio`-Element unterstützt beliebte Audioformate wie mp3, wav und ogg.
Das `video`-Element unterstützt die Formate mp4, ogg und webm.

Um Audioinhalte auf Ihrer Webseite einzubinden, können Sie das `audio`-Element mit dem `src`-Attribut verwenden, das auf den Speicherort Ihrer Audiodatei verweist.

Wie Sie im Vorschaufenster sehen, wird auf der Seite nichts angezeigt. Um die Vorschauen zu sehen, müssen Sie den interaktiven Editor aktivieren.

:::interactive_editor

```html
<audio src="https://cdn.freecodecamp.org/curriculum/js-music-player/cruising-for-a-musing.mp3"></audio>
```

:::

Wenn Sie den Audioplayer auf der Seite sehen möchten, können Sie das `audio`-Element mit dem `controls`-Attribut hinzufügen.

Drücken Sie im Vorschaufenster auf Wiedergabe, um eines von Quincy Larsons Liedern zu hören. Um ein anderes Lied zu hören, ändern Sie den Wert von `src` zu `"https://cdn.freecodecamp.org/curriculum/js-music-player/never-not-favored.mp3"`. Um die Vorschauen zu sehen, müssen Sie den interaktiven Editor aktivieren.

:::interactive_editor

```html
<audio src="https://cdn.freecodecamp.org/curriculum/js-music-player/cruising-for-a-musing.mp3" controls></audio>
```

:::

Das `controls`-Attribut ermöglicht es Benutzern, die Audiowiedergabe zu steuern, einschließlich Pausieren oder Fortsetzen der Wiedergabe. Das `controls`-Attribut ist ein Boolean-Attribut, das einem Element hinzugefügt werden kann, um eingebaute Wiedergabesteuerungen zu aktivieren. Wenn es weggelassen wird, werden keine Steuerelemente angezeigt.

Hinweis: Einige Browser, wie Safari, zeigen standardmäßig möglicherweise keine Lautstärkeregelung an, selbst wenn das `controls`-Attribut vorhanden ist.

Abgesehen von den `src`- und `controls`-Attributen gibt es mehrere andere Attribute, die die Funktionalität des `audio`-Elements verbessern. Das `loop`-Attribut ist ein Boolean-Attribut, das die Audio-Wiedergabe kontinuierlich wiederholen lässt.

Hier ist ein Beispiel für die Verwendung des `loop`-Attributs, um eines von Quincy Larsons Liedern mit dem Titel „Can't stay down“ abzuspielen. Um die Schleifenfunktion zu sehen, aktivieren Sie den interaktiven Editor, ziehen Sie den Wiedergabekopf nahe ans Ende des Liedes, und es wird nach dem Ende erneut gestartet.

:::interactive_editor

```html
<audio
  src="https://cdn.freecodecamp.org/curriculum/js-music-player/can't-stay-down.mp3"
  loop
  controls
></audio>
```

:::

Ein weiteres Attribut, das Sie verwenden können, ist das `muted`-Attribut. Wenn es im `audio`-Element vorhanden ist, startet dieses Boolean-Attribut den Ton stummgeschaltet. Hier ist ein Beispiel für die Verwendung des `muted`-Attributs.

Um die Musik zu hören, aktivieren Sie den interaktiven Editor und klicken Sie auf das Lautstärkesymbol im Audioplayer.

:::interactive_editor

```html
<audio
  src="https://cdn.freecodecamp.org/curriculum/js-music-player/can't-stay-down.mp3"
  loop
  controls
  muted
></audio>
```

:::

Was die Audio-Dateitypen betrifft, gibt es Unterschiede darin, welche Browser welchen Typ unterstützen. Um dem gerecht zu werden, können Sie `source`-Elemente innerhalb des `audio`-Elements verwenden, und der Browser wählt die erste Quelle aus, die er versteht. Hier ist ein Beispiel für die Verwendung mehrerer `source`-Elemente für ein `audio`-Element:

```html
<audio controls>
  <source src="audio.ogg" type="audio/ogg" />
  <source src="audio.wav" type="audio/wav" />
  <source src="audio.mp3" type="audio/mpeg" />
</audio>
```

Der Browser beginnt zuerst mit dem ogg-Typ, und wenn er die Audiodatei nicht abspielen kann, wechselt er zum nächsten Typ in der Liste.

Alle Attribute, die wir bisher gelernt haben, werden auch im `video`-Element unterstützt. Hier ist ein Beispiel für die Verwendung eines `video`-Elements mit den Attributen `loop`, `controls` und `muted`.

Fügen Sie das `autoplay`-Attribut zum öffnenden `video`-Tag hinzu, damit das Video automatisch abgespielt wird. Um mit dem Beispiel zu interagieren, müssen Sie den interaktiven Editor aktivieren.

**HINWEIS**: Das `width`-Attribut wird hier verwendet, um das Video kleiner zu machen und besser ins Vorschaufenster zu passen. Mehr über das `width`-Attribut lernen Sie in zukünftigen Lektionen.

:::interactive_editor

```html
<video
  src="https://archive.org/download/BigBuckBunny_124/Content/big_buck_bunny_720p_surround.mp4"
  loop
  controls
  muted
  width="400"
></video>
```

:::

Für das `src`- oder Quellattribut verwenden wir ein Video namens „Big Buck Bunny“ von archive.org. Wenn Sie während des Herunterladens des Videos ein Bild anzeigen möchten, können Sie das `poster`-Attribut verwenden. Dieses Attribut ist für `audio`-Elemente nicht verfügbar und ist einzigartig für das `video`-Element. Hier ist ein Beispiel für die Verwendung des `poster`-Attributs mit Inhalten von peach.blender.org.

```html
<video
  src="https://archive.org/download/BigBuckBunny_124/Content/big_buck_bunny_720p_surround.mp4"
  loop
  controls
  muted
  poster="https://peach.blender.org/wp-content/uploads/title_anouncement.jpg?x11217"
  width="400"
></video>
```

Sie können auch das `source`-Element innerhalb eines `video`-Elements verwenden, genau wie Sie es mit dem `audio`-Element getan haben. So können Sie dasselbe Video in mehreren Formaten bereitstellen, und der Browser wählt das erste abspielbare aus.

```html
<video
  controls
  width="400"
  poster="https://peach.blender.org/wp-content/uploads/title_anouncement.jpg?x11217"
>
  <source
    src="https://archive.org/download/BigBuckBunny_124/Content/big_buck_bunny_720p_surround.mp4"
    type="video/mp4"
  />
  <source
    src="https://archive.org/download/BigBuckBunny_124/Content/big_buck_bunny_720p_surround.webm"
    type="video/webm"
  />
  Your browser does not support the video tag.
</video>
```
