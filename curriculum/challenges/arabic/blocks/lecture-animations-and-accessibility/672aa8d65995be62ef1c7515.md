# --interactive--

تسمح تحريكات CSS بإنشاء تأثيرات ديناميكية وجذابة بصريًا على صفحات الويب دون الحاجة إلى JavaScript أو برمجة معقدة. توفر طريقة للانتقال السلس للعناصر بين أنماط مختلفة خلال مدة محددة.

في جوهرها، تتكون تحريكات CSS من مكوِّنَين رئيسيين: قاعدة `@keyframes` وخاصية التحريك.

تعرف قاعدة `@keyframes` مراحل وأنماط التحريك. تحدد الأنماط التي يجب أن يمتلكها العنصر في نقاط مختلفة أثناء التحريك.

إليك مثالًا:

```css
@keyframes slide-in {
  0% {
    transform: translateX(-100%);
  }
  100% {
    transform: translateX(0);
  }
}
```

تعرف قاعدة `@keyframes` هذه، المسماة `slide-in`، تحريكًا ينقل العنصر من اليسار إلى اليمين. تمثل النسب المئوية تقدم التحريك، حيث `0%` هو البداية و`100%` هو النهاية.

تتحكم الدالة `translateX` في تحريك `@keyframes` في الموضع الأفقي للعنصر أثناء تحريكه للظهور.

لتطبيق هذا التحريك على عنصر، تستخدم خاصية `animation`. هذا المثال يكرر التحريك إلى ما لا نهاية لتتمكن من رؤيته أثناء التشغيل (ستحتاج إلى تفعيل المحرر التفاعلي لرؤية المعاينات):

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<div class="sliding-element">Hello, I slide in!</div>
```

```css
@keyframes slide-in {
  0% {
    transform: translateX(-100%);
  }
  100% {
    transform: translateX(0);
  }
}

.sliding-element {
  animation: slide-in 2s ease-out infinite;
}
```

:::

يطبق هذا التحريك `slide-in` على العنصر مع مدة 2 ثانية ودالة توقيت `ease-out`.

في الواقع، خاصية `animation` هي اختصار لعدة خصائص فردية:

`animation-name` التي تحدد قاعدة `@keyframes` التي ستُستخدم.

`animation-duration` التي تحدد مدة إكمال التحريك.

`animation-timing-function` التي تعرف كيف يتقدم التحريك مع الوقت - مثل `ease`، `linear`، `ease-in-out`.

`animation-delay` التي تحدد تأخيرًا قبل بدء التحريك.

`animation-iteration-count` التي تحدد عدد مرات تكرار التحريك.

`animation-direction` التي تحدد ما إذا كان التحريك يلعب للأمام، للخلف، أو بالتناوب.

`animation-fill-mode` التي تحدد كيف يُنسَّق العنصر قبل وبعد التحريك.

`animation-play-state` التي تسمح بإيقاف واستئناف التحريك.

يمكنك استخدام هذه الخصائص بشكل فردي للتحكم الدقيق:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<div class="complex-animation">Watch my colors change!</div>
```

```css
.complex-animation {
  animation-name: color-change;
  animation-duration: 3s;
  animation-timing-function: linear;
  animation-delay: 1s;
  animation-iteration-count: infinite;
  animation-direction: alternate;
}

@keyframes color-change {
  0% {
    background-color: red;
  }
  50% {
    background-color: blue;
  }
  100% {
    background-color: green;
  }
}
```

:::

ينشئ هذا تحريكًا يغير لون خلفية العنصر باستمرار بين الأحمر والأزرق والأخضر.

يمكن تحفيز تحريكات CSS بواسطة أحداث مختلفة، مثل التمرير فوق عنصر:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<button class="button">Hover over me!</button>
```

```css
.button {
  background-color: blue;
  transition: background-color 0.3s;
}

.button:hover {
  background-color: red;
}
```

:::

بينما يستخدم هذا المثال خاصية `transition`، وهي أبسط للتأثيرات الأساسية، فإنه يوضح كيف يمكن لـ CSS إنشاء عناصر متحركة وتفاعلية.

من المهم ملاحظة أنه رغم قوة تحريكات CSS، يجب استخدامها باعتدال. الإفراط في التحريكات قد يؤدي إلى ضعف الأداء وقد يشتت أو يسبب مشاكل لمستخدمي إمكانية الوصول. دائمًا فكر في توفير خيارات لتقليل أو تعطيل التحريكات للمستخدمين الذين يفضلون حركة أقل.

توفر تحريكات CSS طريقة لإنشاء تجارب ويب تفاعلية وجذابة دون الاعتماد على JavaScript. بفهم مبادئ `@keyframes` وخصائص `animation`، يمكنك إحياء تصاميم الويب بطريقة فعالة وقابلة للوصول.
