# --description--

`Promise` هو كائن يمثل إتمام أو فشل عملية غير متزامنة في المستقبل. يكون في البداية في حالة انتظار. ثم يمكن أن ينتقل إلى حالة مُنجزة عند نجاح العملية، أو حالة مرفوضة عند فشل العملية. إليك مثالًا على إنشاء `Promise`:

```js
const aPromise = new Promise((resolve, reject) => {
  setTimeout(() => {
    resolve("Operation successful!");
  }, 1000);
});
```

في هذا المثال، ننشئ وعدًا يحاكي عملية غير متزامنة باستخدام `setTimeout`. بعد ثانية واحدة، يُحل الوعد مع الرسالة `Operation successful!`.

طريقة أخرى للعمل مع الوعود هي استخدام الطريقتين `.then` و `.catch`.

تُستخدم الطريقة `.then()` في الوعد لتحديد ما يجب أن يحدث عند إتمام الوعد، بينما تُستخدم `.catch()` لمعالجة أي أخطاء تحدث. لنرَ كيف يمكننا استخدام `.then` و `.catch` مع وعدنا:

```js
aPromise.then((result) => {
  console.log(result);  // Outputs: "Operation successful!"
}).catch((error) => {
  console.error(error);
});
```

في هذا الكود، تُستدعى الدالة الممررة إلى `.then()` بالقيمة التي تم حل الوعد بها. إذا حدث خطأ، تُستدعى الدالة الممررة إلى `.catch()` بدلاً من ذلك.

الآن، لنتحدث عن تسلسل الوعود. من الميزات القوية للوعود أننا نستطيع تسلسل عدة عمليات غير متزامنة معًا. كل `.then()` يمكن أن يُرجع وعدًا جديدًا، مما يسمح لك بأداء متتالية من العمليات غير المتزامنة واحدة تلو الأخرى. إليك مثالًا:

```js
fetch('https://api.example.com/data')
  .then(response => response.json())
  .then(data => {
    console.log(data);
    return fetch('https://api.example.com/data2');
  })
  .then(response => response.json())
  .then(data2 => console.log(data2))
  .catch(error => console.error('Error:', error));
```

في هذا المثال، نجري مكالمتين لواجهة برمجة التطبيقات (API) بالتتابع. يقوم أول `.then()` بتحليل الاستجابة كـ JSON. يقوم ثاني `.then()` بتسجيل البيانات ويجري مكالمة API أخرى. يقوم ثالث `.then()` بتحليل استجابة المكالمة الثانية، ويقوم رابع `.then()` بتسجيل تلك البيانات. إذا حدث خطأ في أي نقطة من هذه المتتالية، سيتم التقاطه بواسطة `.catch()` في النهاية.

من المهم ملاحظة أن `.catch()` يلتقط الأخطاء من أي من الخطوات السابقة في المتتالية. هذا يعني أنك لا تحتاج إلى إضافة معالجة أخطاء لكل خطوة على حدة، مما يبسط شفرتك كثيرًا.
