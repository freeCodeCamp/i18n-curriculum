# --interactive--

在之前的課程中，你學會了如何使用 `addEventListener()` 方法來處理事件。但在 JavaScript 中，還有另一種不建議使用的方式來處理事件。

行內事件處理常式是 HTML 元素上的特殊屬性，用於在事件發生時執行 JavaScript 程式碼。

這是一個帶有行內點擊事件處理常式的 `button` 元素範例：

:::interactive_editor

```html
<button onclick="alert('Hello World!')">Show alert</button>
```

:::

當使用者點擊按鈕時，會呼叫 `alert` 函式（程式），並顯示帶有訊息 `Hello World!` 的警示對話盒。

另一種使用行內事件處理常式的方法是呼叫在 HTML 文件中的 `script` 頁籤定義的函式。 

以下是定義一個名為 `changeBgColor` 的函式（程式）並從行內點擊事件處理常式中呼叫它的範例：

:::interactive_editor

```html
<script>
  function changeBgColor() {
    document.body.style.backgroundColor = "lightblue";
  }
</script>

<button onclick="changeBgColor()">Change background color</button>
```

:::

當使用者點擊按鈕時，會呼叫 `changeBgColor` 函式（程式），並將頁面的背景色更改為淺藍色。雖然可以像這樣使用行內事件處理常式，但這並不被視為最佳做法。 

有一個原因是，行內事件處理常式只能用來為元素附加一個事件監聽器。如果你想為同一個元素附加多個事件監聽器，你需要使用 `addEventListener()`。另一個原因是行內事件處理常式會將 HTML 和 JavaScript 程式碼混合在一起，這會讓你的程式碼更難閱讀和維護。最好透過使用 `addEventListener()` 將事件監聽器附加到元素，來將你的 HTML 程式碼和 JavaScript 程式碼分開。

不建議在現代 JavaScript 中使用行內事件處理常式。因此，在使用 JavaScript 的事件時，最佳做法是堅持使用 `addEventListener()` 方法。
