# --description--

在 Python 中，变量就像一个带标签的盒子，用于存储和引用不同类型的数据。要在 Python 中声明变量，你使用赋值（`=`）操作符将一个值赋值给标识符。你不需要像 JavaScript 中使用 `let` 或 `const`，或 C# 中使用 `char` 这样的特殊关键字。

在 Python 中，你只需在左侧写变量名，后跟赋值操作符，再在右侧写你想赋值给变量的值。下面是如何声明 `name` 和 `age` 变量的示例：

```python
name = 'John Doe'
age = 25
```

在上面的例子中，变量 `name` 保存了值 `'John Doe'`。该值是一个字串，字串是一串用于表现文本的字符。字串用单引号或双引号书写，例如 `'Hello'` 或 `"Hello"`。在后续课程中，你将学习更多关于在 Python 中操作字串的内容。

在 Python 中命名变量时，有一些重要规则你应该牢记：

- 变量名只能以字母或下划线（`_`）开头，不能以数字开头。
- 变量名只能包含字母数字字符（`a-z`、`A-Z`、`0-9`）和下划线（`_`）。
- 变量名区分分支——`age`、`Age` 和 `AGE` 都被视为唯一。
- 变量名不能是 Python 的保留关键字之一，例如 `if`、`class` 或 `def`。

如果你违反任何这些规则，你的 Python 程序将抛出一个 `SyntaxError`：

```bash
    5variable_name = 5
     ^
SyntaxError: invalid syntax
```

现在让我们来了解一些 Python 中变量的常见命名约定。

首先，变量名应全部小写，多个单词之间用下划线分隔。这称为蛇形命名法：

```python
my_variable_name = 'freeCodeCamp'
```

接下来，你应该为变量使用描述性名称。例如，如果你想将用户的年龄储存为变量，`user_age` 比 `age` 或缩略语如 `ua` 更好：

```python
user_age = 30
```

这样，你可以在大型代码库中轻松地向其他团队成员（或你未来的自己）传达一个变量的用途。

另一种约定是避免使用单字母变量名。这在 Python 中非常常见，但应避免使用，因为单字母变量名无法传达任何目的或含义：

```python
x = 56 # What do you mean by x?
```

如果你处于循环或类似情况，这会有所不同，因为像 `i`、`j`、`k` 等变量名是常见且可接受的。

此外，上述示例中的井号符号（`#`）及其后面的文本称为评论。你可能已经熟悉评论，所以我们快速浏览一下并解释它们的工作原理。

在 Python 中，评论以井号符号（`#`）开头，语言会忽略该行中 `#` 符号之后的所有内容：

```python
# This is a single-line comment
```

多行评论可以通过连续的单行评论来创建：

```python
# This is a
# multi-line
# comment
```

你可以使用评论来解释你的代码，为自己留下提醒，或说明某行代码存在的原因。当你学习或团队协作时，评论尤其有用。

但是，你不应该使用评论来解释你的变量名的含义。相反，你为变量选择的名称应该具有描述性，能够传达它们的用途，并遵循前面提到的其他命名规则以防止语法误差。
