# --interactive--

In JavaScript moderno, le classi sono come modelli che puoi definire nel codice per creare più oggetti dello stesso tipo.

Per esempio, se stai modellando un sistema di inventario, avrai bisogno di creare più oggetti per ogni tipo di prodotto.

Ogni singolo prodotto avrà le sue proprietà e azioni, ma i prodotti dello stesso tipo possono essere creati dallo stesso "modello". È proprio a questo che servono le classi.

Le classi possono essere definite come espressione di classe o dichiarazione di classe.

Ecco la sintassi base per una dichiarazione di classe:

```js
class MyClassName {
  // Class Methods
  constructor() { ... }
  method1() { ... }
  method2() { ... }
  ...
}
```

Inizi scrivendo la parola chiave `class` seguita dal nome della classe. Quando nomini le classi, dovresti usare la convenzione di denominazione `PascalCase` invece di `camelCase`.

Il contenuto all’interno delle parentesi graffe è noto come corpo della classe. Qui vanno il costruttore e i metodi.

Il costruttore è un metodo speciale che viene chiamato automaticamente quando viene creato un nuovo oggetto dalla classe. Viene tipicamente usato per inizializzare le proprietà dell’oggetto.

In questo esempio, puoi vedere una classe `Dog` con un costruttore.

```js
class Dog {
  constructor(name) {
    this.name = name;
  }
}
```

La parola chiave `this` viene usata qui per riferirsi all’istanza corrente della classe `Dog`, permettendoti di assegnare la proprietà `name` all’oggetto che viene creato. Imparerai di più sulla parola chiave `this` in una lezione futura.

Sotto il costruttore, puoi avere quello che si chiamano metodi. I metodi sono funzioni definite all’interno di una classe che eseguono azioni o operazioni. Servono a definire comportamenti che le istanze della classe possono eseguire.

Ecco un esempio di aggiunta di un metodo chiamato `bark` alla classe `Dog`.

```js
class Dog {
  constructor(name) {
    this.name = name;
  }

  bark() {
    console.log(`${this.name} says woof!`);
  }
}
```

Quando il metodo `bark` viene chiamato, il messaggio verrà stampato nella console con il nome effettivo che è stato passato. Se crei un’istanza `Dog` con il nome `"Buddy"`, chiamare il metodo `bark()` su quell’istanza stamperà `"Buddy says woof!"` nella console.

Una volta definita la tua classe, puoi creare un’istanza della classe con la parola chiave `new`, il nome della classe e i suoi argomenti.

In questo esempio, creiamo una nuova istanza `Dog` con il nome `Gino` e poi la assegniamo alla variabile `dog`.

```js
const dog = new Dog("Gino");
```

Quando scrivi `new Dog("Gino")` nel tuo codice, il nuovo oggetto viene creato e il costruttore viene chiamato automaticamente con gli argomenti dati.

Puoi quindi consultare questa proprietà usando la notazione a punto, come puoi vedere nella riga di codice seguente:

```js
console.log(dog.name);
```

Ecco l’esempio completo:

:::interactive_editor

```js
class Dog {
  constructor(name) {
    this.name = name;
  }

  bark() {
    console.log(`${this.name} says woof!`);
  }
}

const dog = new Dog("Gino");
console.log(dog.name);
```

:::

Puoi anche chiamare i metodi. In questo esempio, stiamo chiamando il metodo `bark()` sull’istanza `dog`.

```js
dog.bark();
```

Ecco l’esempio completo con la chiamata al metodo inclusa:

:::interactive_editor

```js
class Dog {
  constructor(name) {
    this.name = name;
  } 
  bark() {
    console.log(`${this.name} says woof!`);
  }
}

const dog = new Dog("Gino");
console.log(dog.name);
dog.bark();
```

:::

Come detto prima, puoi anche definire le classi come espressione di classe. Qui la classe è anonima e assegnata a una variabile.

Ecco come appare l’esempio precedente come espressione di classe:

```js
const Dog = class {
  constructor(name) {
    this.name = name;
  }

  bark() {
    console.log(`${this.name} says woof!`);
  }
};
```

La maggior parte delle volte userai dichiarazioni di classe, ma capire le espressioni di classe è comunque utile, perché possono offrire più flessibilità, specialmente in situazioni di programmazione dinamica.

Usando le classi e comprendendo la loro sintassi base, puoi creare codice riutilizzabile, organizzarne il contenuto in unità logiche e promuovere la modularità del codice.
