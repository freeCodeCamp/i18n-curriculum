# --description--

Web Storage APIは、ブラウザ内にキーと値のペアを保存する仕組みを提供し、開発者が異なるページのリロードやセッションをまたいで利用できる情報を保存できるようにします。

Web Storage APIの主な2つのコンポーネントは`localStorage`と`sessionStorage`です。

`localStorage`は、ブラウザのウィンドウが閉じられたりページが更新された後もデータを保持できるWeb Storage APIの部分です。このデータは、アプリケーションやユーザーによって明示的に削除されるまで利用可能なままです。

`sessionStorage`は、ページセッションの間だけデータを保存するWeb Storage APIの別の部分で、ブラウザのタブやウィンドウが開いている間はデータが利用可能です。ただし、`localStorage`とは異なり、`sessionStorage`のデータはタブやウィンドウが閉じられるとクリアされます。`sessionStorage`については次のレッスンで詳しく学びます。

`localStorage`の一般的な利用例には、テーマや言語設定などのユーザー設定の保存、ブラウザセッションをまたいだフォームデータの記憶、ウェブアプリのパフォーマンス向上のための小さな情報のキャッシュがあります。

キャッシュとは、頻繁にアクセスされるデータを一時的な保存場所（キャッシュ）に保存し、その後のリクエストをより速く処理できるようにすることで、データベースや外部サーバーなどの遅いデータソースから再取得や再計算を避けることを指します。

一般的な`localStorage`のメソッドには、`setItem`、`getItem`、`removeItem`、`clear`があります。

以下は、`setItem()`メソッドを使って`localStorage`にキーと値のペアを保存する例です。

```js
localStorage.setItem('username', 'JaneDoe');
```

次に、`localStorage`から特定のキーの値を取得したい場合は、`getItem()`メソッドを使います。

```js
let username = localStorage.getItem('username');
console.log(username); // JaneDoe
```

`localStorage`からキーを使ってアイテムを削除するには、`removeItem()`メソッドを使います。

```js
localStorage.removeItem('username');
```

`localStorage`のすべてのデータをクリアするには、`clear()`メソッドを使います。

```js
localStorage.clear();
```

では、`localStorage`を使ってユーザーの好みのテーマを保存する例を見てみましょう。

```js
// Store the user's theme preference
localStorage.setItem('theme', 'dark');

// Retrieve the stored theme preference
const userTheme = localStorage.getItem('theme');
console.log(userTheme); // 'dark'

// Remove the theme preference
localStorage.removeItem('theme');

// Clear all localStorage data
localStorage.clear();
```

この例では：

- まずユーザーのテーマの選択（`dark`）を保存します。

- 次にそのテーマを取得してコンソールに出力します。

- 最後に特定のアイテムを削除する方法や、保存されたすべてのデータをクリアする方法を示します。

`localStorage`はセッション間で保持する必要がある小さなデータを保存するのに非常に便利ですが、`localStorage`はパスワードなどの機密情報を保存するのには使わないほうがよいことに注意してください。セキュリティ上のリスクがあるためです。
