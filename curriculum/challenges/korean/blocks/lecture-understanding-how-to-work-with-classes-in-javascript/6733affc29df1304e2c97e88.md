# --interactive--

현대 JavaScript에서 클래스는 같은 종류의 여러 객체를 만들기 위해 코드로 정의할 수 있는 청사진과 같습니다.

예를 들어, 재고 시스템을 모델링할 때는 제품 종류별로 여러 객체를 만들어야 합니다.

각 개별 제품은 고유한 속성과 동작을 가지지만, 같은 종류의 제품은 동일한 "청사진"에서 만들어질 수 있습니다. 이것이 클래스가 사용되는 이유입니다.

클래스는 클래스 표현식이나 클래스 선언으로 정의할 수 있습니다.

다음은 클래스 선언의 기본 구문입니다:

```js
class MyClassName {
  // Class Methods
  constructor() { ... }
  method1() { ... }
  method2() { ... }
  ...
}
```

`class` 키워드 다음에 클래스 이름을 작성하는 것으로 시작합니다. 클래스 이름을 지을 때는 `PascalCase` 명명 관례를 사용해야 하며 `camelCase`는 사용하지 않습니다.

중괄호 안의 내용은 클래스 본문이라고 하며, 생성자와 메서드가 들어가는 곳입니다.

생성자는 클래스에서 새 객체가 만들어질 때 자동으로 호출되는 특별한 메서드입니다. 보통 객체의 속성을 초기화하는 데 사용합니다.

이 예제에서는 생성자가 있는 `Dog` 클래스를 볼 수 있습니다.

```js
class Dog {
  constructor(name) {
    this.name = name;
  }
}
```

여기서 `this` 키워드는 `Dog` 클래스의 현재 인스턴스를 가리키며, 생성 중인 객체에 `name` 속성을 할당할 수 있게 합니다. `this` 키워드에 대해서는 이후 수업에서 더 배우게 됩니다.

생성자 아래에는 메서드라고 하는 것들이 있을 수 있습니다. 메서드는 클래스 내부에 정의된 함수로, 동작이나 작업을 수행합니다. 클래스 인스턴스가 수행할 수 있는 동작을 정의하는 데 사용합니다.

다음은 `bark` 클래스에 `Dog`라는 메서드를 추가한 예입니다.

```js
class Dog {
  constructor(name) {
    this.name = name;
  }

  bark() {
    console.log(`${this.name} says woof!`);
  }
}
```

`bark` 메서드가 호출되면, 전달된 실제 이름과 함께 메시지가 콘솔에 출력됩니다. 만약 `Dog`라는 이름으로 `"Buddy"` 인스턴스를 만들고, 그 인스턴스에서 `bark()` 메서드를 호출하면 `"Buddy says woof!"`가 콘솔에 출력됩니다.

클래스를 정의한 후에는 `new` 키워드, 클래스 이름, 그리고 인수를 사용해 클래스 인스턴스를 만들 수 있습니다.

이 예제에서는 `Dog`라는 이름으로 새 `Gino` 인스턴스를 만들고, 이를 `dog` 변수에 할당합니다.

```js
const dog = new Dog("Gino");
```

코드에 `new Dog("Gino")`를 작성하면 새 객체가 생성되고, 생성자가 주어진 인수와 함께 자동으로 호출됩니다.

그다음 점 표기법을 사용해 이 속성에 접근할 수 있습니다. 다음 코드 줄에서 볼 수 있습니다:

```js
console.log(dog.name);
```

전체 예제는 다음과 같습니다:

:::interactive_editor

```js
class Dog {
  constructor(name) {
    this.name = name;
  }

  bark() {
    console.log(`${this.name} says woof!`);
  }
}

const dog = new Dog("Gino");
console.log(dog.name);
```

:::

메서드도 호출할 수 있습니다. 이 예제에서는 `bark()` 인스턴스에서 `dog` 메서드를 호출하고 있습니다.

```js
dog.bark();
```

메서드 호출이 포함된 전체 예제는 다음과 같습니다:

:::interactive_editor

```js
class Dog {
  constructor(name) {
    this.name = name;
  } 
  bark() {
    console.log(`${this.name} says woof!`);
  }
}

const dog = new Dog("Gino");
console.log(dog.name);
dog.bark();
```

:::

앞서 언급했듯이, 클래스는 클래스 표현식으로도 정의할 수 있습니다. 이 경우 클래스는 익명이며 변수에 할당됩니다.

앞서 예제를 클래스 표현식으로 바꾸면 다음과 같습니다:

```js
const Dog = class {
  constructor(name) {
    this.name = name;
  }

  bark() {
    console.log(`${this.name} says woof!`);
  }
};
```

대부분은 클래스 선언을 사용하지만, 클래스 표현식을 이해하는 것도 중요합니다. 특히 동적 프로그래밍 상황에서 더 유연성을 제공할 수 있기 때문입니다.

클래스를 사용하고 기본 구문을 이해하면 재사용 가능한 코드를 만들고, 논리적 단위로 조직하며, 코드 모듈화를 촉진할 수 있습니다.
