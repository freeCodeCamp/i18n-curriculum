# --description--

Crypto ni moduli nyingine kuu iliyojengwa ndani ya Node.js. Inajumuisha zana kwa ajili ya mambo kama vile hashing, encryption, decryption, na kuunda saini za kidijitali, zote zinazotumika kulinda taarifa nyeti na kuweka programu yako salama.

Hata hivyo, `crypto` inakupa vipande vya msingi vya chini, si usalama wa plug-and-play. Kuandika msimbo wako wa encryption au uthibitisho wa utambulisho kunaweza kuwa hatari ikiwa huna makini. Katika hali nyingi, ni bora kutumia maktaba zilizojaribiwa vizuri kama `bcrypt` kwa hashing ya nywila au `jsonwebtoken` (JWT) kwa kushughulikia kuingia na tokeni.

Hata hivyo, bado ni muhimu kuelewa jinsi baadhi ya njia katika moduli ya crypto zinavyofanya kazi.

Ili kutumia njia hizi, unahitaji import moduli ya `crypto`:

```js
const crypto = require("crypto");
```

Baadhi ya njia hutumika kwa madhumuni ya mabadiliko ya data, kama zile za hashing na encryption ya data, na nyingine ni kwa usimamizi wa funguo na usalama, kama zile za kuzalisha thamani za nasibu na kuunda siri.

Tuchunguze kwanza njia za hashing na encryption ya data.

Njia ya `createHash()` huunda kitu cha hash kwa kutumia algoriti kama `sha256`, `sha512`, au `md5`. Ni operesheni ya njia moja, hivyo huwezi kuirudisha nyuma.

`createHash()` ni muhimu kwa hashing ya nywila na kuchapisha alama za faili. Ili kuitumia, unapitisha algoriti yako, tumia njia ya `update()` kuingiza data, na hatimaye tumia `digest()` na encoding kupata thamani ya hash:

```js
const crypto = require("crypto");

const hashedPassword = crypto
  .createHash("sha256")
  .update("myStrongPassword")
  .digest("hex");

console.log("createHash result:", hashedPassword);
// createHash result: f92c9cfa0ead1bcec05ca75888a4074ba994ad237e5e2a8c7cc6a620378c061d
```

`createHmac()` hufanya karibu sawa na `createHash()`, lakini huongeza kiwango kwa kupokea funguo ya siri, hivyo ni mtu tu mwenye funguo hiyo anayeweza kuthibitisha hash. Ni bora kwa uthibitisho wa utambulisho na kuthibitisha usahihi wa data:

```js
const crypto = require("crypto");

const hashedMessage = crypto
  .createHmac("sha256", "secretkey")
  .update("important-secret-message")
  .digest("hex");

console.log("createHmac result:", hashedMessage);
// createHmac result: da48d6f026b6036286b1fb872c63264130d5cc4271f3a213bb6ddca5a023e77e
```

Njia za `createCipheriv()` na `createDecipheriv()` hufanya encryption na decryption ya data. Zote zinapokea algoriti, funguo, na `iv`, ambayo ni kipande cha data ya nasibu au ya kipekee inayotumika mwanzoni mwa mchakato wa encryption:

```js
createCipheriv(algorithm, key, iv);
createDecipheriv(algorithm, key, iv);
```

Ili kufungua data, funguo lazima iwe ile ile, vinginevyo, decryption itashindwa, na utapata kosa au maandishi yasiyosomeka badala ya ujumbe wa awali.

Hapa ni hizi mbili zikiendeshwa:

```js
const crypto = require("crypto");

// A key must match the algorithm length. Here AES-256 is 32 bytes
const key = Buffer.from("12345678901234567890123456789012");

// A fixed IV, 16 bytes for AES
const iv = Buffer.from("1234567890123456");

const cipher = crypto.createCipheriv("aes-256-cbc", key, iv);

let encrypted = cipher.update("Hello campers!", "utf8", "hex");
encrypted += cipher.final("hex");

console.log("Encrypted data:", encrypted);
// Encrypted data: 4ee93aa398ab44e3540e4a67ca96bc8c

// Decrypt the "Hello campers!" message
const decipher = crypto.createDecipheriv("aes-256-cbc", key, iv);
let decrypted = decipher.update(encrypted, "hex", "utf8");
decrypted += decipher.final("utf8");

console.log("Decrypted data:", decrypted);
// Decrypted data: Hello campers!
```

Njia nyingine ya crypto kwa mabadiliko ya data ni `sign()` na `verify()`.

`sign()` huunda saini ya kidijitali kutoka kwa data fulani kwa kutumia funguo binafsi. Saini hii inathibitisha kuwa data ilitoka kwa mwenye funguo binafsi na haijabadilishwa.

`verify()` kisha huangalia saini hiyo, na inashindwa ikiwa data au saini haitalingani.

Sasa, tuchunguze njia za crypto za kuzalisha thamani za nasibu na kuunda siri.

`randomBytes()` hupokea `size` na kuzalisha tokeni zilizo salama kwa njia ya cryptography. Hii inafanya iwe nzuri kwa kuzalisha UUIDs (vitambulisho vya kipekee duniani kote). Zaidi ya hayo, ni mbadala mzuri kwa `Math.random()`, ambayo si salama kwa tokeni na funguo.

```js
console.log("Random Bytes:", crypto.randomBytes(16));
// Random Bytes: <Buffer 01 88 aa 1e 2c 38 48 39 26 e1 6b a9 d8 c5 ed 49>
```

Matokeo ni buffer kwa chaguo-msingi. Kama ulivyojifunza katika funzo kuhusu moduli ya Buffer, unaweza kubadilisha Buffer hiyo kuwa mfuatano wa herufi kwa kutumia njia ya `toString()`:

```js
console.log("Random Bytes:", crypto.randomBytes(16).toString("hex"));
// Random Bytes: a6154ef5a296fa176ad0f332bd94d712
```

Hoja ya `'hex'` katika `toString('hex')` hapa inamwambia Node kuandika data ya binary kutoka Buffer kama mfuatano wa herufi wa hexadecimal.

Njia ya `randomInt()` hupokea `min` na `max` na kuzalisha nambari kamili ya nasibu salama kati yao. Ni muhimu kwa OTPs na uteuzi wa nasibu.

```js
console.log("Random Int:", crypto.randomInt(0, 100)); // 89
```

Tena, kuboreshwa juu ya `Math.random()` na `Math.floor()` ni kwamba njia hii hutumia nasibu salama ya cryptography chini ya hood, hivyo wadukuzi hawawezi kutabiri nambari ya nasibu itakayotokea.

Njia nyingine ni `createSecretKey()`. Inapokea buffer na kuzalisha byte ghafi iliyofunikwa katika `KeyObject`:

```js
const crypto = require("crypto");

const secret = crypto.createSecretKey(crypto.randomBytes(32));
console.log(secret); // SecretKeyObject [KeyObject] { [Symbol(kKeyType)]: 'secret' }
```

Kisha unaweza kutumia njia ya `export` kutuma `KeyObject` hiyo:

```js
const secret = crypto.createSecretKey(crypto.randomBytes(32));
console.log(secret.export());
// <Buffer 53 06 a1 c7 75 69 8b 38 8b a4 b2 f7 1b bc b8
// ae e2 d1 bf 67 af 1a 6a 0a 6e a0 29 62 bb 52 52 32>
```

Na hatimaye, tumia `toString()` kwenye buffer kuona uwakilishi wake wa mfuatano wa herufi:

```js
const secret = crypto.createSecretKey(crypto.randomBytes(32));
console.log(secret.export().toString('hex'));
// 32dfe5917668580160986f1623bf8152913329c71163be9c3404a110cd78efd6
```

Mbali na hizi, kuna:

- `createPublicKey()` na `createPrivateKey()` zinazokuwezesha kufanya kazi na funguo zilizozalishwa mahali pengine
- `createDiffieHellman()` kwa pande mbili kuzalisha siri ya pamoja bila kutuma siri moja kwa moja
- `Certificate()` kwa kufanya kazi na ile inayotumika katika HTTPS, hivyo unaweza kuchambua, kuhamisha, na kuthibitisha maudhui ya vyeti
