# --description--

Express.js або просто Express — це мінімалістичний і гнучкий веб-фреймворк, побудований **на основі** Node.js. Він спрощує процес створення вебсерверів і API.

Замість того, щоб писати купу шаблонного коду з використанням вбудованого модуля `http` у Node, Express дозволяє створювати надійні вебзастосунки всього за кілька рядків.

Можна уявити його як швидкий, неупереджений шар, який робить Node.js простішим і потужнішим.

Ось три ключові причини, чому розробники люблять Express:

1. **Мінімалістичний і швидкий** – Express надає лише найнеобхідніше, даючи повний контроль і дозволяючи додавати функції за потребою через підпрограмне забезпечення та плагіни.

2. **Велика екосистема** – має величезну колекцію підпрограмного забезпечення та плагінів для логування, автентифікації, безпеки, сесій та іншого.

3. **Чудово підходить для API** – його проста модель маршрутизації та обробки запитів робить Express особливо зручним для створення RESTful API.

За допомогою Express ви можете створювати:

* Вебзастосунки

* RESTful API

* Бекенди для односторінкових застосунків (SPA)

* Сервіси з великою кількістю підпрограмного забезпечення

* Будь-що серверне з HTTP


## Як встановити Express

Спершу переконайтеся, що у вас встановлені Node.js і npm. Ось як перевірити, чи встановлені node і npm:

```bash
node -v
npm -v
```

Якщо ви бачите номер версії, це означає, що вони встановлені. Якщо ні, вам потрібно їх встановити. Інструкції з встановлення можна знайти в одному з попередніх уроків або на офіційному сайті Node.js.

Далі у вашому терміналі введіть такі команди:

```bash
mkdir my-app
cd my-app
npm init -y
npm install express
```

Ви дізнаєтеся більше про підпрограмне забезпечення та плагіни в майбутніх уроках, але коротко: підпрограмне забезпечення — це код, який виконується між отриманням запиту і відправленням відповіді. Його часто використовують для обробки помилок, валідації вхідних даних, модифікації даних у запиті або відповіді тощо. Ви можете ланцюжити підпрограмне забезпечення, щоб створювати гнучку та повторно використовувану логіку для обробки запитів.

Плагіни зазвичай є зовнішніми, сторонніми модулями Node.js, які розширюють функціональність Express. Їх можна встановити окремо через npm, і більшість реалізовані як підпрограмне забезпечення, яке можна використовувати у вашому застосунку з мінімальним налаштуванням. Замість того, щоб писати загальні функції з нуля, ви можете використати плагін для обробки таких речей, як логування, автентифікація або найкращі практики безпеки.

Express — популярний фреймворк для серверної розробки на Node.js. Він дає вам будівельні блоки, щоб швидко запустити сервер, обробляти маршрути, керувати підпрограмним забезпеченням і створювати API — усе це з чистим і зрозумілим кодом.
