# --interactive--

Operator `typeof` katika JavaScript ni chombo rahisi lakini chenye nguvu kinachokuwezesha kuona aina ya data ya variable au thamani. Daima hurudisha mfuatano wa herufi unaoonyesha aina hiyo.

Tuchunguze mifano michache:

:::interactive_editor

```js
let num = 42;
console.log(typeof num); // "number"
```

:::

Katika mfano huu wa kwanza, tumetengeneza variable inayoitwa `num` na tumeiweka thamani ya nambari `42`. Unapotumia operator `typeof` kwenye variable inayoitwa `num`, itarudisha mfuatano wa herufi `number`.

Hapa kuna mfano mwingine wa kutumia operator `typeof` kwenye variable inayoitwa `isUserLoggedIn`:

:::interactive_editor

```js
let isUserLoggedIn = true;
console.log(typeof isUserLoggedIn); // "boolean"
```

:::

Unapotumia operator `typeof` kwenye variable `isUserLoggedIn`, itarudisha mfuatano wa herufi `boolean` kwa sababu thamani ya kweli au siyo kweli `true` imepewa variable hiyo.

Kutumia operator `typeof` kunaweza kuwa na manufaa hasa unapotafuta makosa au kujaribu kuelewa aina gani ya data unayofanya nayo kazi katika msimbo wako.

Hata hivyo, kuna hitilafu inayojulikana sana katika JavaScript linapokuja suala la `null`.

Tuchunguze mfano:

:::interactive_editor

```js
let exampleVariable = null;
console.log(typeof exampleVariable); // "object"
```

:::

Katika mfano huu, tuna variable inayoitwa `exampleVariable` na tumeiweka thamani ya `null`. Lakini tunapotumia operator `typeof`, inarudisha mfuatano wa herufi `object`.

Hii inachukuliwa sana kuwa hitilafu katika JavaScript, ambayo imetokea tangu enzi zake za mwanzo. Sababu ya tabia hii iko katika jinsi JavaScript ilivyoundwa awali.

Lugha ilipotekelezwa kwa mara ya kwanza, thamani kama `null` zilikuwa zinawakilishwa kama aina maalum ya kitu, na kusababisha matokeo haya yasiyotegemewa.

Kwa bahati mbaya, hii imekuwa sehemu ya lugha, na ingawa ni ya kuchanganya, ni jambo ambalo utahitaji kuzingatia.
