# --description--

Crypto ist ein weiteres Kernmodul, das in Node.js eingebaut ist. Es enthält Werkzeuge für Dinge wie Hashing, Verschlüsselung, Entschlüsselung und das Erstellen digitaler Signaturen, die alle dazu dienen, sensible Informationen zu schützen und Ihre App sicher zu halten.

Das heißt, `crypto` bietet Ihnen Bausteine auf niedriger Ebene, keine Plug-and-Play-Sicherheit. Eigene Verschlüsselungs- oder Authentifizierungscodes zu schreiben, kann unsicher sein, wenn Sie nicht vorsichtig sind. In den meisten Fällen ist es besser, gut getestete Bibliotheken wie `bcrypt` für Passwort-Hashing oder `jsonwebtoken` (JWT) für die Handhabung von Logins und Tokens zu verwenden.

Dennoch ist es nützlich zu verstehen, wie einige der Methoden im Crypto-Modul funktionieren.

Um diese Methoden zu verwenden, müssen Sie das `crypto`-Modul importieren:

```js
const crypto = require("crypto");
```

Einige der Methoden werden für Datenumwandlungen verwendet, wie die für Hashing und Verschlüsselung von Daten, und andere sind für Schlüssel- und Sicherheitsverwaltung, wie die zur Erzeugung zufälliger Werte und zum Erstellen von Geheimnissen.

Schauen wir uns zuerst die Methoden zum Hashing und Verschlüsseln von Daten an.

Die `createHash()`-Methode erstellt ein Hash-Objekt, indem sie Algorithmen wie `sha256`, `sha512` oder `md5` übernimmt. Es ist eine Einwegoperation, die nicht umkehrbar ist.

`createHash()` ist nützlich zum Hashen von Passwörtern und zum Fingerabdruck von Dateien. Um es zu verwenden, übergeben Sie Ihren Algorithmus, nutzen die `update()`-Methode, um die Daten einzuspeisen, und schließlich `digest()` mit einer Kodierung, um den Hash-Wert zu erhalten:

```js
const crypto = require("crypto");

const hashedPassword = crypto
  .createHash("sha256")
  .update("myStrongPassword")
  .digest("hex");

console.log("createHash result:", hashedPassword);
// createHash result: f92c9cfa0ead1bcec05ca75888a4074ba994ad237e5e2a8c7cc6a620378c061d
```

`createHmac()` macht fast dasselbe wie `createHash()`, geht aber einen Schritt weiter, indem es einen geheimen Schlüssel akzeptiert, sodass nur jemand mit diesem Schlüssel den Hash verifizieren kann. Es ist ideal für Authentifizierung und die Überprüfung der Datenintegrität:

```js
const crypto = require("crypto");

const hashedMessage = crypto
  .createHmac("sha256", "secretkey")
  .update("important-secret-message")
  .digest("hex");

console.log("createHmac result:", hashedMessage);
// createHmac result: da48d6f026b6036286b1fb872c63264130d5cc4271f3a213bb6ddca5a023e77e
```

Die `createCipheriv()`- und `createDecipheriv()`-Methoden verschlüsseln und entschlüsseln Daten. Beide nehmen einen Algorithmus, einen Schlüssel und ein `iv` entgegen, das ein Block aus zufälligen oder einzigartigen Daten ist, der zu Beginn des Verschlüsselungsprozesses verwendet wird:

```js
createCipheriv(algorithm, key, iv);
createDecipheriv(algorithm, key, iv);
```

Um die Daten zu entschlüsseln, muss der Schlüssel derselbe sein, sonst schlägt die Entschlüsselung fehl, und Sie erhalten entweder einen Fehler oder unleserlichen Kauderwelsch anstelle der ursprünglichen Nachricht.

Hier sind die beiden in Aktion:

```js
const crypto = require("crypto");

// A key must match the algorithm length. Here AES-256 is 32 bytes
const key = Buffer.from("12345678901234567890123456789012");

// A fixed IV, 16 bytes for AES
const iv = Buffer.from("1234567890123456");

const cipher = crypto.createCipheriv("aes-256-cbc", key, iv);

let encrypted = cipher.update("Hello campers!", "utf8", "hex");
encrypted += cipher.final("hex");

console.log("Encrypted data:", encrypted);
// Encrypted data: 4ee93aa398ab44e3540e4a67ca96bc8c

// Decrypt the "Hello campers!" message
const decipher = crypto.createDecipheriv("aes-256-cbc", key, iv);
let decrypted = decipher.update(encrypted, "hex", "utf8");
decrypted += decipher.final("utf8");

console.log("Decrypted data:", decrypted);
// Decrypted data: Hello campers!
```

Eine weitere Crypto-Methode für Datenumwandlungen ist `sign()` und `verify()`.

`sign()` erstellt eine digitale Signatur aus einigen Daten mit einem privaten Schlüssel. Diese Signatur beweist, dass die Daten vom Inhaber des privaten Schlüssels stammen und nicht manipuliert wurden.

`verify()` überprüft dann diese Signatur, und sie schlägt fehl, wenn die Daten oder die Signatur nicht übereinstimmen.

Schauen wir uns nun die Crypto-Methoden zum Erzeugen zufälliger Werte und zum Erstellen von Geheimnissen an.

`randomBytes()` nimmt ein `size` entgegen und erzeugt kryptografisch gesicherte Tokens. Das macht es gut geeignet für die Erzeugung von UUIDs (universell eindeutige IDs). Außerdem ist es ein guter Ersatz für `Math.random()`, das für Tokens und Schlüssel nicht sicher ist.

```js
console.log("Random Bytes:", crypto.randomBytes(16));
// Random Bytes: <Buffer 01 88 aa 1e 2c 38 48 39 26 e1 6b a9 d8 c5 ed 49>
```

Die Ausgabe ist standardmäßig ein Puffer. Wie Sie in der Lektion zum Buffer-Modul gelernt haben, können Sie diesen Puffer mit der `toString()`-Methode in einen String umwandeln:

```js
console.log("Random Bytes:", crypto.randomBytes(16).toString("hex"));
// Random Bytes: a6154ef5a296fa176ad0f332bd94d712
```

Das `'hex'`-Argument in `toString('hex')` hier sagt Node, die Binärdaten aus dem Puffer als hexadezimalen String zu kodieren.

Die `randomInt()`-Methode nimmt zwei Werte `min` und `max` entgegen und erzeugt eine sichere zufällige Ganzzahl zwischen ihnen. Sie ist nützlich für OTPs und zufällige Auswahl.

```js
console.log("Random Int:", crypto.randomInt(0, 100)); // 89
```

Auch hier ist die Verbesserung gegenüber `Math.random()` und `Math.floor()`, dass die Methode kryptografisch sichere Zufälligkeit verwendet, sodass Angreifer die resultierende Zufallszahl nicht vorhersagen können.

Eine weitere Methode ist `createSecretKey()`. Sie nimmt einen Puffer und erzeugt ein rohes Byte, das in ein `KeyObject` eingewickelt ist:

```js
const crypto = require("crypto");

const secret = crypto.createSecretKey(crypto.randomBytes(32));
console.log(secret); // SecretKeyObject [KeyObject] { [Symbol(kKeyType)]: 'secret' }
```

Sie können dann die `export`-Methode verwenden, um dieses `KeyObject` zu senden:

```js
const secret = crypto.createSecretKey(crypto.randomBytes(32));
console.log(secret.export());
// <Buffer 53 06 a1 c7 75 69 8b 38 8b a4 b2 f7 1b bc b8
// ae e2 d1 bf 67 af 1a 6a 0a 6e a0 29 62 bb 52 52 32>
```

Und schließlich verwenden Sie `toString()` auf dem Puffer, um die String-Darstellung davon zu sehen:

```js
const secret = crypto.createSecretKey(crypto.randomBytes(32));
console.log(secret.export().toString('hex'));
// 32dfe5917668580160986f1623bf8152913329c71163be9c3404a110cd78efd6
```

Zusätzlich zu diesen gibt es:

- `createPublicKey()` und `createPrivateKey()`, mit denen Sie mit Schlüsseln arbeiten können, die anderswo generiert wurden
- `createDiffieHellman()` für zwei Parteien, um ein gemeinsames Geheimnis zu erzeugen, ohne das Geheimnis direkt zu senden
- `Certificate()` für die Arbeit mit dem in HTTPS verwendeten Schlüssel, sodass Sie Zertifikatsinhalte parsen, exportieren und verifizieren können
