# --description--

在你的學習過程中，你應該致力於培養強大的問題解決能力。這些核心技能將對你在日常工作中處理現實世界的問題至關重要。

解決演算法挑戰是練習的絕佳方式。它需要分析性的思考方式，能夠將問題拆解成其核心組件，並找到一個能高效產生正確輸出的解決方案。

但是你該從哪裡開始？

有幾種解決問題的技巧可以讓你開始著手處理這些挑戰。

作為範例，我們將在 Python 中反轉一個字串。

這是挑戰：

給定一個字串，撰寫一個演算法，傳回一個字元順序相反的新字串。

當你遇到這種類型的問題時，首先應該多次閱讀描述，以確保你了解其內容。如果跳過這個步驟或閱讀過快，你可能會錯過關鍵資訊。

然後，一旦你熟悉了問題，開始將它拆解成其核心組件。

問問你自己：

* 輸入是什麼？
    
* 預期輸出是什麼？
    
* 我該如何將輸入轉換成預期的輸出？
    

在這個問題中，你可以判斷輸入是一個字串，因為挑戰是以「Given a string…」開始。

輸出是「一個字元順序反轉的新字串」。

所以你需要取得原始字串並將它反轉。

這個初步分析一開始可能看起來有點重複，但它對確保你完全理解需求非常有幫助。

然後，你應該開始思考你將開發的演算法如何將輸入轉換成輸出。

在這個規劃與分析階段，通常會使用偽程式碼來規劃所需的步驟。

**Pseudocode** 是對演算法邏輯的高層次描述，具有通用性，且不基於任何特定的程式語言。

偽程式碼不像實際的程式碼那麼正式，因為它僅供人類閱讀。它應該一目了然且易於理解。它的主要目的是清楚表達將要執行的步驟順序。

偽程式碼通常是一般書寫語言（例如英文）與程式設計構件（例如 `IF`、`ELSE`、`FOR` 和 `WHILE`）的混合。

這是一個你可能會撰寫來解決「反轉字串」挑戰的偽程式碼範例。

```md
GET original_string

SET reversed_string = ""

FOR EACH character IN original_string:
  ADD character TO THE BEGINNING OF reversed_string

DISPLAY reversed_string
```

請注意步驟的說明方式易於理解。字、單字和構件本身可能會根據你所遵循的標準而有所不同。

如果你想，你可以依照相同的邏輯，使用多種程式語言實作這些步驟，因為這段偽程式碼與程式語言無關。

到這個時候，你可能已經意識到這個問題可以用許多不同的方法來解決。這並不是反轉字串的唯一方法。

但請記住，選擇正確的演算法很重要。

在先前的課程中，你學習了演算法複雜度以及為什麼選擇在時間和空間方面高效的演算法很重要。

這就是你作為開發者將扮演重要角色的地方。你需要選擇最高效的演算法來解決這個挑戰。

思考不同可用的演算法是你應該練習的重要解決問題技能。花點時間問問自己，你在偽程式碼中提出的解法是否為最佳解法。

例如，有許多不同的演算法用於排序元素，但其中有些比其他更高效。泡沫排序，例如，對於排序大型列表來說非常低效，而快速排序通常更高效。

對於我們的「反轉字串」挑戰，假設我們計劃在 Python 中實作我們的演算法，我們可以使用這些方法中的任一種：

* 使用擴展切片語法 `[::-1]` 來取得新的反轉字串。
    
* 從左到右迴圈遍歷字元，並將新字元添加到新字串的開頭。
    
* 呼叫 `reversed()` 函式（程式）以取得包含所有字元反向順序的迭代器，然後使用 `““.join()` 方法將它們串接回字串。
    

你應該使用哪一個？那是你的選擇。

根據你的知識和經驗做出這些決策，能對你的應用程式最終的效率產生巨大影響。考慮不同的方法、它們的效率、影響及實作。

問問你自己：

* 我將如何處理這個問題？
    
* 我會使用哪些資料結構？
    
* 我選擇的資料結構是針對手頭問題最高效的嗎？
    
* 我是否涵蓋所有可能的邊緣案例？
    

邊界情況是發生在演算法應該控制代碼範圍邊界上的特定且有效的輸入或條件。

例如，在「反轉字串」挑戰中，邊界情況會是將空字串作為輸入。你有正確控制代碼嗎？如果沒有，請考慮控制這個邊界情況的最佳方法，並將它添加到你的偽程式碼中。

然後，一旦你對你的計畫感到滿意，你就可以進入實作階段。在此階段，你將使用程式語言實作你的演算法。

在結構化你的程式時，你應該撰寫易於閱讀和理解的模組化程式碼。

根據你目前的知識，使用該程式語言的工具。有些程式語言包含針對常見問題和任務的內建解決方案，若可能，請使用它們。

為了保持一致性，請遵循你所選程式語言的最佳實務。

在你撰寫程式碼時測試它，並確保你適當地處理邊界情況。

一旦你的解決方案被實作，請檢查它是否對所有範例都能正確運作，並且可能重構你的程式碼以使其更清晰或更簡單。

回到你的解決方案是非常重要的。開發不一定是線性、一步一步的過程。你總是可以回到你的程式碼，並運用你的批判性思考能力來改進它。

這些是你可以遵循的一些常見解決問題技巧，以應對演算法挑戰。如果你持續練習，你將逐漸培養你的解決問題能力。
