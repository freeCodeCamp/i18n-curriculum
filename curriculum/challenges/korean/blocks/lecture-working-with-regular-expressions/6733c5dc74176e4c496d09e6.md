# --interactive--

전방 탐색과 후방 탐색 어서션은 주변 패턴의 존재 여부에 따라 특정 패턴을 매칭할 수 있게 해줍니다. 이 어서션에는 네 가지 변형이 있습니다.

첫 번째는 긍정 전방 탐색 어서션입니다. 이 어서션은 특정 패턴 뒤에 다른 패턴이 올 때 매칭합니다.

긍정 전방 탐색을 만들려면, 먼저 매칭하려는 패턴을 작성하세요. 그다음 조건으로 사용할 패턴을 소괄호로 감쌉니다. 소괄호 바로 뒤에 `?=`를 써서 이 패턴이 긍정 전방 탐색임을 정의하세요.

이 패턴은 `free`가 바로 뒤따를 때만 `code`이라는 단어를 매칭합니다:

```js
const regex = /free(?=code)/i;
```

패턴의 동작을 테스트해봅시다:

:::interactive_editor

```js
const regex = /free(?=code)/i;
console.log(regex.test("freeCodeCamp")); // true
console.log(regex.test("free code camp")); // false
console.log(
  regex.test("I need someone for free to write code for me")
); // false
```

:::

`free`가 `code` 바로 뒤에 올 때만 문자열이 테스트를 통과하는 것을 확인하세요.

그런데 `free`가 뒤따르지 않을 때 `code`의 존재를 매칭하고 싶다면 어떻게 할까요? 긍정 전방 탐색을 부정 전방 탐색으로 바꾸면 동작이 반전됩니다. 이를 위해 `?=`를 `?!`로 바꾸세요:

```js
const regex = /free(?!code)/i;
```

같은 문자열에 대해 테스트해봅시다:

:::interactive_editor

```js
const regex = /free(?!code)/i;
console.log(regex.test("freeCodeCamp")); // false
console.log(regex.test("free code camp")); // true
console.log(
  regex.test("I need someone for free to write code for me")
); // true
```

:::

예상대로 결과가 반전되었습니다. `free`가 `code` 바로 뒤에 오는 첫 번째 문자열만 실패합니다.

후방 탐색 어서션은 전방 탐색과 비슷하지만, 뒤따르는 패턴이 아니라 앞에 오는 패턴을 조건으로 매칭합니다. 긍정 후방 탐색을 살펴보겠습니다.

긍정 후방 탐색은 `?<=`로 표시하며 `?=`와 다릅니다. `code`가 앞에 올 때 `free`을 매칭하도록 정규 표현식을 만들어봅시다:

```js
const regex = /(?<=free)code/i;
```

긍정 전방 탐색과 마찬가지로, `code`가 `free` 바로 앞에 있으므로 첫 번째 문자열과 매칭됩니다:

:::interactive_editor

```js
const regex = /(?<=free)code/i;
console.log(regex.test("freeCodeCamp")); // true
console.log(regex.test("free code camp")); // false
console.log(
  regex.test("I need someone for free to write code for me")
); // false
```

:::

`code`가 앞에 오지 않을 때 `free`을 매칭하려면 부정 후방 탐색을 사용하세요. 부정 후방 탐색은 `?<=`를 `?<!`로 바꾸어 정의합니다:

```js
const regex = /(?<!free)code/i;
```

이것은 `code`가 바로 앞에 오지 않는 `free`의 모든 발생을 매칭합니다:

:::interactive_editor

```js
const regex = /(?<!free)code/i;
console.log(regex.test("freeCodeCamp")); // false
console.log(regex.test("free code camp")); // true
console.log(
  regex.test("I need someone for free to write code for me")
); // true
```

:::

`Regex.prototype.test`는 문자열이 정규 표현식과 매칭되는지만 확인한다는 점을 기억하세요. `String.prototype.match`와 함께 부정 후방 탐색을 사용해 어서션이 어떻게 작용하는지 봅시다:

```js
const regex = /(?<!free)code/i;
console.log("freeCodeCamp".match(regex)); // null
console.log("free code camp".match(regex)); // ['code', index: 5, input: 'free code camp', groups: undefined]
console.log(
  "I need someone for free to write code for me".match(regex)
); // ['code', index: 33, input: 'I need someone for free to write code for me', groups: undefined]
```

정규 표현식이 `free`의 존재를 확인하기 위해 후방 탐색을 사용해도 `free`과는 매칭되지 않는 것을 확인하세요. 매칭에 포함되는 텍스트는 `code`뿐입니다.

전방 탐색과 후방 탐색은 매칭 결과에 영향을 주지 않고 조건부로 텍스트를 매칭하는 데 매우 유용합니다.
