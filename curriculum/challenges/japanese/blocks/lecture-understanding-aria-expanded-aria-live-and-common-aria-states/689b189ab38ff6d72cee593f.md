# --interactive--

以前、セマンティックマークアップとARIA属性がコンテンツをアクセシブルにする方法を学びました。実際のアプリでは、コンテンツはほとんど静的ではなく、ページはJavaScriptによって動的に更新されることが多いです。このような場合、これらの変更がHTMLにも反映されていることが重要です。これにより、スクリーンリーダーが更新された状態を正確にユーザーに伝えられます。これがないと、支援技術を使う人は何かが変わったことに気づかなかったり、古い情報や誤解を招く情報を受け取ったりする可能性があります。

JavaScriptが要素の状態を変更する一般的なシナリオと、ARIA属性を更新してHTMLのアクセシビリティを保つ方法を見てみましょう。

ドロップダウンメニューを開閉するボタンがあるとします。

:::interactive_editor

```html
<button id="menuButton" aria-expanded="false" aria-controls="menuList">
  Menu
</button>

<ul id="menuList" hidden>
  <li>List item 1</li>
  <li>List item 2</li>
</ul>
<script src="index.js"></script>
```

```js
const button = document.getElementById("menuButton");
const menu = document.getElementById("menuList");

button.addEventListener("click", () => {
  const expanded = button.getAttribute("aria-expanded") === "true";
  button.setAttribute("aria-expanded", String(!expanded));
  menu.hidden = expanded;
});
```

:::

HTMLはドロップダウンメニューを制御するボタンを設定しています。ボタンは`aria-expanded`を使ってメニューが開いているかどうかを示し、`aria-controls`を使ってメニューの`id`を参照し、メニューとリンクしています。

JavaScriptはメニューボタンのクリックを監視しています。クリックされると、`aria-expanded`の値を切り替え、メニューの`hidden`プロパティを更新します。これにより、メニューの可視性とアクセシビリティ情報が同期され、視覚ユーザーとスクリーンリーダーユーザーの両方がメニューが開いているか閉じているかを知ることができます。

これらのARIA属性については今後のレッスンで詳しく学びます。今は、JavaScriptが要素の状態を変更したときは、関連するARIA属性も更新して、支援技術が画面上の実際の状態を正確に反映できるようにすることを覚えておいてください。
