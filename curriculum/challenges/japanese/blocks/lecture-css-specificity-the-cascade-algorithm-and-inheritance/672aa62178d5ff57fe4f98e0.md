# --interactive--

CSSの特異性は、複数のルールが適用される可能性がある場合に、どのスタイルが要素に適用されるかを決定する基本的な概念です。

特異性を理解することで、開発者は異なるCSSルール間の競合を解決し、望ましいスタイルが一貫して適用されるようにできます。

CSSの特異性は、使用されるセレクターの種類に基づいて計算されます。

最も高い特異性は、`style`属性を通じて要素に直接適用されるインラインスタイルにあります。

この例では、最初の段落が赤くなり、他の`p`要素は青くなります。これは、インラインスタイルが`p`ファイルに示されている`styles.css`のような型セレクターよりも高い特異性を持つためです。

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<p style="color: red;">Red paragraph</p>
<p>Other paragraph</p>
<p>Another paragraph</p>
<p>Yet another paragraph</p>
```

```css
p {
  color: blue;
}
```

:::

次に、IDセレクターは高い特異性を持ちます。

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<p id="para">Red paragraph</p>
<p>Other paragraph</p>
<p>Another paragraph</p>
<p>Yet another paragraph</p>
```

```css
#para {
  color: red; 
}

p {
  color: blue;
}
```

:::

次に、クラスセレクター、属性セレクター、疑似クラスが登場します。

例として、`.container`や`.button`のようなクラスセレクター、`[type="text"]`のような属性セレクター、`:hover`のような疑似クラスがあります。これらは中程度の特異性を持ちます。

**注意**：疑似クラスについては今後のレッスンで詳しく学びます。

この例では、最初の段落が赤くなります。なぜなら`id`セレクターがクラスや型セレクターよりも高い特異性を持つためです。`.example-para`要素は緑色になり、残りの段落要素は青色になります。

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<p id="para">Example paragraph</p>
<p class="example-para">Other paragraph</p>
<p class="example-para">Another paragraph</p>
<p>Yet another paragraph</p>
```

```css
#para {
  color: red; 
}

.example-para {
  color: green;
}

p {
  color: blue;
}
```

:::

型セレクター（`div`や`h1`）や疑似要素（`::before`や`::after`）は、前述のグループよりも低い特異性を持ちます。

**注意**：疑似要素については今後のレッスンで詳しく学びます。

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<p id="para">Example paragraph</p>
<p>Other paragraph</p>
<p>Yet another paragraph</p>
```

```css
#para {
  color: red; 
}

p {
  color: blue;
}
```

:::

最後に、アスタリスクで表されるユニバーサルセレクター`*`は、すべての中で最も低い特異性を持ちます。

ここでは、`*`セレクターを使ってすべての要素の色を赤に設定する例を示します。ただし、IDセレクターや型セレクターがこれらのスタイルをオーバーライドするため、赤い要素は表示されません。これは`*`セレクターの特異性が低いことを示しています。

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<p id="para">Example paragraph</p>
<p>Other paragraph</p>
<p>Yet another paragraph</p>
```

```css
* {
  color: red;
}

#para {
  color: green; 
}

p {
  color: blue;
}
```

:::

特異性の値は4つの部分からなる値`(a, b, c, d)`として計算されます：

- `a`：インラインスタイル（`1`または`0`）。
- `b`：IDセレクターの数。
- `c`：クラスセレクター、属性セレクター、疑似クラスの数。
- `d`：型セレクター、疑似要素、ユニバーサルセレクターの数。

特異性値の各部分は異なる重みを持ちます：

- **インラインスタイル**（`a`）は最も重みが高く、特異性値の最初の部分に`1`の値を加えます。
    
- **IDセレクター**（`b`）は次に高い重みを持ち、各IDが特異性値の2番目の部分に`1`を加えます。
    
- **クラスセレクター、属性セレクター、疑似クラス**（`c`）は中程度の重みを持ち、各々が特異性値の3番目の部分に`1`を加えます。
    
- **型セレクターと疑似要素**（`d`）は最も低い重みで、各々が特異性値の4番目の部分に`1`を加えます。
    
- **ユニバーサルセレクター**（`*`）：ユニバーサルセレクターは特異性計算に`0`を加え、特異性には影響しません。セレクターに含まれても特異性値は変わりません。

インラインCSSは要素に直接適用されるため、最も高い特異性を持ちます。内部や外部のCSSをオーバーライドします。インラインスタイルの特異性値は`(1, 0, 0, 0)`です。

別のインラインスタイルの例を示します：

:::interactive_editor

```html
<p style="color: red;">This text is red.</p>
```

:::

内部CSSはHTMLドキュメントの`style`セクション内の`head`要素内で定義されます。インラインスタイルより特異性は低いです。内部と外部のCSSは同じ特異性レベルを共有します — 特異性はCSSが定義されている場所ではなく、使用されるセレクターによって決まります。2つのルールが同じ特異性の場合、ドキュメント内で後に現れる方が優先されます。したがって、外部スタイルシートが内部`style`ブロックの後にリンクされていれば、外部ルールが優先されます。前にリンクされていれば、内部ルールが優先されます。

内部スタイルの特異性値は使用されるセレクターによって決まります。例えば、内部CSS内のIDセレクターは特異性値`(0, 1, 0, 0)`を持ちます。

:::interactive_editor

```html
<head>
  <style>
    #text {
      color: blue;
    }
  </style>
</head>
<body>
  <div id="text">This text is blue.</div>
</body>
```

:::

この例では、インラインスタイルやより特異性の高いセレクターが適用されない限り、テキストは青色になります。

外部CSSは`link`セクションの`head`要素を通じてリンクされ、別の`.css`ファイルに記述されます。内部CSSと同様に、外部スタイルの特異性は使用されるセレクターによって決まります。外部CSSは大規模なプロジェクトでの保守性に優れています。

外部スタイルの特異性値も使用されるセレクターによって決まります。例えば、外部CSS内のクラスセレクターは特異性値`(0, 0, 1, 0)`を持ちます。

:::interactive_editor

```html
<head>
  <link rel="stylesheet" href="styles.css">
</head>
<body>
  <div class="text">This text's color is defined in an external CSS file.</div>
</body>
```

```css
.text {
  color: purple;
}
```

:::

この例では、テキストの色は`styles.css`ファイルで定義されており、より特異性の高いセレクターやインラインスタイルでオーバーライドされない限り適用されます。
