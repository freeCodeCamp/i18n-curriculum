# --description--

Vererbung ist das nächste wichtige Konzept der objektorientierten Programmierung (OOP), das wir behandeln.

Werfen wir einen genaueren Blick auf dieses Konzept und darauf, wie es Ihnen ermöglicht, wiederverwendbaren Code zu schreiben.

Mit Vererbung kann eine Unterklasse (oder Kindklasse) die Attribute und Methoden einer Basisklasse (oder Elternklasse) verwenden. Dadurch können Sie Code wiederverwenden, klare Klassenhierarchien erstellen und das Verhalten anpassen, ohne alles neu schreiben zu müssen. Sie können anpassen, indem Sie bestehende Methoden erweitern oder sie in der Kindklasse überschreiben.

Hier ist die grundlegende Syntax für Vererbung:

```py
class Parent:
    # Parent attributes and methods

class Child(Parent):
    # Child inherits, extends, and/or overrides where necessary
```

Damit die `Child`-Klasse von der `Parent`-Klasse erbt, müssen Sie die Elternklasse an die Kindklasse übergeben.

Dieser Stil wird Einzelvererbung genannt, da eine Kindklasse genau von einer Elternklasse erbt.

Hier ist ein Beispiel:

```py
class Animal:
    def __init__(self, name):
        self.name = name

    def sound(self):
        return f'{self.name} makes a sound'

class Dog(Animal):
    bark = 'woof! woof!! woof!!!'

jack = Dog('Jack')
print(jack.sound())  # Jack makes a sound
print(jack.bark)  # woof! woof!! woof!!!
```

Sie sehen, dass wir das `self.name`-Attribut und die `sound()`-Methode aus der Elternklasse `Animal` in der Kindklasse `Dog` wiederverwenden können.

Lassen Sie uns die `sound()`-Methode aus der Elternklasse `Animal` in der Kindklasse `Dog` überschreiben, damit `sound()` die `bark`-Klassenvariable verwenden kann:

```py
class Animal:
    def __init__(self, name):
        self.name = name

    def sound(self):
        return f'{self.name} makes a sound.'

class Dog(Animal):
    bark = 'woof! woof!! woof!!!'

    # Override sound() to use bark class variable
    def sound(self):
        return f'{self.name} barks {self.bark}'

jack = Dog('Jack')
print(jack.sound())  # Jack barks woof! woof!! woof!!!
```

Wenn Sie den Rückgabewert von `sound()` beibehalten und später die Klassenvariable bark hinzufügen möchten, können Sie `sound()` erweitern, indem Sie die `super()`-Funktion verwenden:

```py
class Animal:
    def __init__(self, name):
        self.name = name

    def sound(self):
        return f'{self.name} makes a sound'

class Dog(Animal):
    bark = 'woof! woof!! woof!!!'

    # Call Animal.sound(), then append bark
    def sound(self):
        base = super().sound()
        return f'{base}, then {self.name} barks {self.bark}'

jack = Dog('Jack')
print(jack.sound())  # Jack makes a sound, then Jack barks woof! woof!! woof!!!
```

In diesem Beispiel ist `base` das Ergebnis des Aufrufs der `sound()`-Methode aus der `Animal`-Klasse, und dann fügen wir den spezifischen Klang der `Dog`-Klasse hinzu. So können Sie die Funktionalität der Elternklasse `Animal` erweitern und gleichzeitig ihr ursprüngliches Verhalten beibehalten.

Es gibt auch Mehrfachvererbung, bei der eine Kindklasse von mehr als einer Elternklasse erben kann.

Hier ist die grundlegende Syntax der Mehrfachvererbung:

```py
class Parent:
    # Attributes and methods for Parent

class Child:
    # Attributes and methods for Child

class GrandChild(Parent, Child):
    # GrandChild inherits from both Parent and Child
    # GrandChild can combine or override behavior from each
```

Ein einfaches Beispiel für Mehrfachvererbung ist ein Frosch, der sowohl an Land gehen als auch im Wasser schwimmen kann:

```py
class Walker:
    def walk(self):
        return 'I can walk on land'

class Swimmer:
    def swim(self):
        return 'I can swim in water'

# Amphibian inherits from both Walker and Swimmer
class Amphibian(Walker, Swimmer):
    def __init__(self, name):
        self.name = name

    def introduce(self):
        return f"I'm {self.name} the frog. {self.walk()} and {self.swim()}."

frog = Amphibian('Freddy')
print(frog.introduce())
# Output: I'm Freddy the frog. I can walk on land and I can swim in water.
```
