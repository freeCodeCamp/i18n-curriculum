# --description--

Bei diesen Aufgaben schreiben Sie Ihren Code mit einer der folgenden Methoden:

- Klonen Sie <a href="https://github.com/freeCodeCamp/boilerplate-npm/" target="_blank" rel="noopener noreferrer nofollow">dieses GitHub-Repository</a> und bearbeiten Sie die Aufgaben lokal, indem Sie <a href="https://www.freecodecamp.org/news/how-to-run-the-freecodecamp-backend-challenges-locally/" target="_blank" rel="noopener noreferrer nofollow">diese Anleitung</a> verwenden, um Node.js, npm einzurichten und die Backend-Projekte auf Ihrem Rechner auszuführen.
- Verwenden Sie einen Website-Builder Ihrer Wahl, um das Projekt abzuschließen. Stellen Sie sicher, dass Sie alle Dateien aus unserem GitHub-Repository einbinden.

Die `package.json`-Datei ist das Herzstück jedes Node.js-Projekts oder npm-Pakets. Sie speichert Informationen über Ihr Projekt. Sie besteht aus einem einzigen JSON-Objekt, in dem die Informationen in Schlüssel-Wert-Paaren gespeichert sind. Es gibt nur zwei Pflichtfelder; `name` und `version`, aber es ist gute Praxis, weitere Informationen anzugeben.

Sie können die `package.json`-Datei über das Terminal mit dem `npm init`-Befehl erstellen. Dieser führt Sie durch eine geführte Einrichtung. Die Verwendung von `npm init` mit dem `-y`-Flag erzeugt die Datei, ohne dass Fragen gestellt werden, `npm init -y`.

Wenn Sie sich den Dateibaum Ihres Projekts ansehen, finden Sie die `package.json`-Datei auf der obersten Ebene des Baums. Diese Datei werden Sie in den nächsten Aufgaben verbessern.

Eines der häufigsten Informationen in dieser Datei ist das `author`-Feld. Es gibt an, wer das Projekt erstellt hat, und kann aus einem String oder einem Objekt mit Kontakt- oder anderen Details bestehen. Für größere Projekte wird ein Objekt empfohlen, aber ein einfacher String wie im folgenden Beispiel reicht für dieses Projekt aus.

```json
"author": "Jane Doe",
```

# --instructions--

Fügen Sie Ihren Namen als `author` des Projekts in der `package.json`-Datei hinzu.

**Hinweis:** Denken Sie daran, dass Sie JSON schreiben, daher müssen alle Feldnamen doppelte Anführungszeichen (") verwenden und mit einem Komma (,) getrennt sein.
