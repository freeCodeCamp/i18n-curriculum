## ما هي البرمجيات الوسيطة؟

- **البرمجيات الوسيطة** هي دالة تُنفذ خلال دورة حياة طلب إلى الخادم.
- تمتلك البرمجيات الوسيطة وصولًا إلى **كائن الطلب** (`req`)، و**كائن الاستجابة** (`res`)، و**الدالة `next`**.
- الاستخدامات الشائعة: تسجيل الطلبات، معالجة الأخطاء، تحليل البيانات الواردة، وتقديم الملفات الثابتة.
- تُنفذ البرمجيات الوسيطة بالترتيب الذي تُضاف به إلى التطبيق.

### كيف تعمل البرمجيات الوسيطة

1. يُرسل طلب إلى الخادم.
2. تعالج دوال البرمجيات الوسيطة الطلب.
3. إذا استدعت البرمجيات الوسيطة `next()`، تنتقل السيطرة إلى البرمجيات الوسيطة التالية أو معالِج المسار.
4. إذا لم تستدع البرمجيات الوسيطة `next()`، تتوقف دورة الطلب والاستجابة.

```javascript
app.use((req, res, next) => {
  console.log('Request URL:', req.url);
  console.log('Request Method:', req.method);
  next();
});
```

- تضيف `app.use()` البرمجيات الوسيطة على مستوى التطبيق بشكل عام.

## برمجيات وسيطة على مستوى التطبيق

- **البرمجيات الوسيطة على مستوى التطبيق** مرتبطة بنسخة من تطبيق Express.
- تُسجل باستخدام `app.use()` أو طرق خاصة بالمسارات مثل `app.get()`، `app.post()`، وغيرها.
- تُنفذ مع كل طلب وارد ما لم تُقيد لمسار معين.
- حالات الاستخدام: تسجيل الدخول، التوثيق، ومعالجة الأخطاء عبر عدة مسارات.

```javascript
app.use((req, res, next) => {
  console.log('Request received at:', new Date());
  next();
});
```

## برمجيات وسيطة على مستوى الموجِّه

- **البرمجيات الوسيطة على مستوى الموجِّه** مرتبطة بنسخة من `express.Router()` بدلاً من التطبيق الرئيسي.
- تُطبق باستخدام `router.use()` وتؤثر فقط على المسارات التي يعالجها ذلك الموجِّه.
- تساعد في تقسيم التطبيق إلى وحدات مسارات معزولة.
- اربط الموجِّه باستخدام `app.use('/path', router)` لتقييد البرمجيات الوسيطة لمسارات محددة.

```javascript
const router = express.Router();

router.use((req, res, next) => {
  console.log('Request made to /menu route');
  next();
});

router.get('/drinks', (req, res) => {
  res.send('Welcome to the drinks menu!');
});

app.use('/menu', router);
```

## برمجيات وسيطة لمعالجة الأخطاء

- **برمجيات وسيطة لمعالجة الأخطاء** هي برمجيات وسيطة خاصة تُستخدم لالتقاط الأخطاء خلال دورة الطلب والاستجابة.
- تحتوي على **أربع معلمات**: `err`، `req`، `res`، و`next`. توقيع المعلمات الأربعة هو ما يخبر Express بأنها معالِج أخطاء.
- تُستدعى عند تمرير خطأ إلى `next(err)`.
- يجب وضعها **في نهاية** مكدس البرمجيات الوسيطة، بعد كل البرمجيات الوسيطة والمسارات الأخرى.

```javascript
app.use((err, req, res, next) => {
  console.error(err.stack);
  res.status(500).send('Something went wrong!');
});
```

## البرمجيات الوسيطة المدمجة والجهات الخارجية

### البرمجيات الوسيطة المدمجة

- **`express.json()`**: يحلل الطلبات الواردة التي تحتوي على حمولات JSON.

```javascript
app.use(express.json());
```

- **`express.static()`**: يقدم الملفات الثابتة مثل الصور وCSS وJavaScript.

```javascript
app.use(express.static('public'));
```

- **`express.urlencoded()`**: يحلل الطلبات الواردة التي تحتوي على حمولات مشفرة بصيغة URL (الصيغة التي ترسلها نماذج HTML)؛ يجعل البيانات المحللة متاحة في `req.body`.

```javascript
app.use(express.urlencoded({ extended: true }));
```

### البرمجيات الوسيطة من جهات خارجية

- **`cors`**: يفعّل مشاركة الموارد عبر الأصول المختلفة.

```javascript
app.use(cors());
```

- **`morgan`**: يسجل طلبات HTTP.

```javascript
app.use(morgan('tiny'));
```

- تُضاف دوال البرمجيات الوسيطة المدمجة والخارجية بالترتيب الذي تُستدعى به باستخدام `app.use()`.
