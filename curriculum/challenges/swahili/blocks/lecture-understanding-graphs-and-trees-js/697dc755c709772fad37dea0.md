# --description--

**Foleni ya kipaumbele** ni aina ya muundo wa data wa dhahania (ADT) unaofanya kazi kama foleni au rundo, lakini na tofauti moja muhimu.

Kama unavyoweza tayari kujua, foleni za kawaida hufuata mtindo wa FIFO (Kwanza kuingia, kwanza kutoka), ambapo kipengele cha kwanza kuongezwa kwenye foleni ndicho kinachotolewa kwanza kutoka kwenye foleni.

Rundo hufuata mtindo wa LIFO (Mwisho kuingia, kwanza kutoka), ambapo kipengele cha mwisho kuongezwa kwenye rundo ndicho kinachotolewa kwanza kutoka kwenye rundo.

Foleni na rundo huzingatia tu mpangilio wa kuingizwa kwa vipengele.

Hata hivyo, foleni za kipaumbele huzingatia "kipaumbele" cha vipengele. Kipaumbele hutumika kuamua ni kipengele gani kinapaswa kutolewa baadaye.

Kawaida, kipengele chenye kipaumbele cha juu hutolewa kwanza, lakini utekelezaji mwingine unaweza pia kuchagua kutoa kipengele chenye kipaumbele cha chini kwanza. Hii itategemea mahitaji ya programu yako.

Foleni za kipaumbele ni msaada mkubwa kwa matumizi halisi kama vile kupata njia fupi kati ya maeneo mawili, kupanga ratiba za kazi katika mifumo ya uendeshaji, kuiga trafiki, kubana data, na kusimamia mitandao.

Katika vitendo, foleni za kipaumbele kawaida hufanywa kwa kutumia muundo wa data wa rundo.

**Rundo** ni muundo wa data wa mti wenye sifa maalum sana inayoitwa **sifa ya rundo**. Sifa hii huamua uhusiano kati ya kila nodi na nodi zake za watoto, kulingana na aina ya rundo.

Kuna aina kuu mbili za rundo:

* Rundo kubwa (max-heap)
    
* Rundo ndogo (min-heap)
    

Katika **rundo kubwa**, thamani ya kila nodi ni kubwa au sawa na thamani ya nodi zake za watoto.

Katika mfano huu, unaweza kuona muundo wa mti wenye nodi 8, 7, 5, 2, na 1. Kumbuka kuwa nodi 7 ni kubwa kuliko nodi 2 na nodi 1, ikifuata sifa ya rundo. Hii ni kweli kwa nodi zote nyingine pia.

Kinyume chake, katika **rundo ndogo**, thamani ya kila nodi ni ndogo au sawa na thamani ya nodi zake za watoto.

Katika mfano huu, tuna nodi zenye thamani 4, 7, 9, 12, na 15. Kwa mfano, nodi 7 ni ndogo kuliko nodi 12 na nodi 15, ikifuata sifa ya rundo. Hii pia ni kweli kwa nodi zote nyingine.

Sifa ya rundo ni muhimu kwa sababu inahakikisha kuwa kipengele cha juu kabisa (au cha chini kabisa) (kulingana na aina ya rundo) hubaki juu kila wakati, jambo linalorahisisha sana kutoa kipengele hicho.

Katika vitendo, rundo kawaida hufanywa kama safu za data ili kufikia nodi za mzazi na watoto kwa ufanisi.

Kutumia safu za data kunarahisisha mantiki ya kufikia thamani hizi au "nodi" kwa sababu nyuma ya pazia, ikiwa rundo linaendelea kuwa na muundo wa mti wa kibainari kamili, utekelezaji wa safu unahitaji tu operesheni rahisi za kihesabu kulingana na vigezo vyao ili kupata mahali vipengele vipo katika kumbukumbu.

JavaScript haina moduli ya rundo iliyojengwa ndani, lakini unaweza kutekeleza rundo ndogo kwa kutumia safu ya data.

Hapa kuna utekelezaji wa msingi wa rundo ndogo katika JavaScript:

```js
class MinHeap {
  constructor(compare = (a, b) => a - b) {
    this.data = [];
    this.compare = compare;
  }

  peek() {
    return this.data[0];
  }

  push(value) {
    this.data.push(value);
    this.#bubbleUp(this.data.length - 1);
  }

  pop() {
    if (this.data.length === 0) return undefined;

    const top = this.data[0];
    const last = this.data.pop();

    if (this.data.length > 0) {
      this.data[0] = last;
      this.#bubbleDown(0);
    }

    return top;
  }

  pushPop(value) {
    if (this.data.length === 0) return value;

    if (this.compare(this.data[0], value) < 0) {
      const top = this.data[0];
      this.data[0] = value;
      this.#bubbleDown(0);
      return top;
    }

    return value;
  }

  heapify(arr) {
    this.data = arr.slice();
    for (let i = Math.floor(this.data.length / 2) - 1; i >= 0; i--) {
      this.#bubbleDown(i);
    }
  }

  #bubbleUp(i) {
    while (i > 0) {
      const p = Math.floor((i - 1) / 2);
      if (this.compare(this.data[i], this.data[p]) >= 0) break;
      [this.data[i], this.data[p]] = [this.data[p], this.data[i]];
      i = p;
    }
  }

  #bubbleDown(i) {
    const n = this.data.length;

    while (true) {
      let smallest = i;
      const l = 2 * i + 1;
      const r = 2 * i + 2;

      if (l < n && this.compare(this.data[l], this.data[smallest]) < 0) smallest = l;
      if (r < n && this.compare(this.data[r], this.data[smallest]) < 0) smallest = r;

      if (smallest === i) break;

      [this.data[i], this.data[smallest]] = [this.data[smallest], this.data[i]];
      i = smallest;
    }
  }
}
````

Ili kutumia rundo hili, unaweza kuunda rundo tupu. Hii itakuwa muundo wa data wa msingi kwa rundo:

```js
const myHeap = new MinHeap();
```

Ili kuongeza vipengele kwenye rundo, utaita `push()`. Hii itaongeza kipengele moja kwa moja mahali panapostahili, ili kuhifadhi sifa ya rundo:

```js
myHeap.push(9);
```

Ili kupata kipengele chenye kipaumbele cha juu (katika kesi hii, thamani ndogo kabisa), utaita `pop()`:

```js
myHeap.pop();
```

`pushPop()` huunganisha operesheni zote mbili katika wito mmoja.

Hii ni bora zaidi kuliko kuwaita kwa mfululizo tofauti, hasa wakati rundo ni kubwa, kwa sababu hufanya operesheni moja tu ya kupanga upya:

```js
myHeap.pushPop(15);
```

Ikiwa tayari una safu ya data na unataka kuibadilisha kuwa rundo, unaweza kuita `heapify()`:

```js
myHeap.heapify([9, 2, 7, 1]);
```

Lakini kwa sasa, tunapanga vipengele kwa thamani zao, sivyo?

Je, tungependa kuzipanga kwa "kipaumbele" badala yake?

Unaweza kufanya hivi kwa kuhifadhi safu za data zenye muundo huu: `[priority, element]`.

Katika JavaScript, safu za data hazilinganishwi moja kwa moja kipengele kwa kipengele kwa ajili ya upangaji, hivyo unapitisha kitendakazi cha kulinganisha kwa rundo kinacholinganishwa kipaumbele kwanza.

Tafadhali kumbuka kuwa, katika kesi hii, thamani ndogo zitawakilisha vipaumbele vya juu. Hii inamaanisha kuwa kipengele chenye kipaumbele cha 1 kitakuwa na **kipaumbele cha juu** kuliko kipengele chenye kipaumbele cha 3:

```js
const myHeap = new MinHeap((a, b) => a[0] - b[0]);

myHeap.push([3, "A"]);
myHeap.push([2, "B"]);
myHeap.push([1, "C"]);
```

Ikiwa unahitaji vipengele vyenye kipaumbele sawa zitoe kwa mpangilio waliowekwa, unaweza kuzingatia kujumuisha kielezo cha kipekee kama kipengele cha pili cha kipengele chako ili kuvunja usawa, kama hii `[priority, counter, element]`.

Kwa mfano, unaweza kulinganisha kwa kipaumbele kwanza, kisha kwa kielezo:

```js
let counter = 0;
const myHeap = new MinHeap((a, b) => (a[0] - b[0]) || (a[1] - b[1]));

myHeap.push([3, counter++, "A"]);
myHeap.push([2, counter++, "B"]);
myHeap.push([2, counter++, "C"]);
```

Sasa tuzungumzie kuhusu ufanisi wa rundo.

Muda wa wastani na mbaya zaidi wa kuingiza na kutoa thamani ndogo au kubwa kutoka kwenye rundo (kulingana na aina ya rundo) ni wa logaritmiki, `O(log n)`, kwa sababu idadi ya kubadilishana mara nyingi ni sawia na urefu wa rundo, ambao ni log(n).

Muda wa wastani na mbaya zaidi wa operesheni ya "kuangalia" ni wa wakati wa kudumu, O(1). Kuangalia kunahusisha kupata thamani ndogo au kubwa (kulingana na aina ya rundo) bila kuiondoa.

Operesheni ya "heapify", ambapo rundo linajengwa kutoka kwenye orodha isiyo pangwa, ina ugumu wa wakati wa mstari, O(n), katika wastani na hali mbaya zaidi.

Vivyo hivyo, utaftaji na kufuta kipengele chochote vina ugumu wa wakati wa mstari katika wastani na hali mbaya zaidi wa O(n), kwa sababu vinaweza kuhitaji kupitia rundo lote.

Na zinahitaji nafasi gani?

Ugumu wa nafasi wa rundo ni wa mstari, O(n), ambapo `n` ni idadi ya vipengele vinavyomo. Zinahitaji kuhifadhi vipengele na mzigo mdogo wa ziada kwa ajili ya kitu cha safu ya data yenyewe.

Foleni za kipaumbele na rundo ni muhimu sana katika sayansi ya kompyuta. Zinakuwezesha kupata na kutumia vipengele muhimu zaidi kutoka kwenye orodha haraka. Ufanisi huu ni muhimu kwa programu nyingi za kompyuta zinazofanya kazi muhimu za dunia halisi, kama vile kupata njia ya haraka zaidi kwenye ramani.
