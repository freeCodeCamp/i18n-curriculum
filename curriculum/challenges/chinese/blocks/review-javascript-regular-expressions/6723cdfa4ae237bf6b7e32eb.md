## 正则表达式和常用方法

- **定义**：正则表达式（Regex）用于创建一个“模式”，你可以用它来检查字符串、提取文本等。

```js
const regex = /freeCodeCamp/;
```

- **`test()` 方法**：该方法接受一个字符串，用于测试该字符串是否与正则表达式匹配。该方法返回一个布尔值，表示字符串是否匹配正则表达式。

:::interactive_editor

```js
const regex = /freeCodeCamp/;
const test = regex.test("e");
console.log(test); // false
```

:::

- **`match()` 方法**：该方法接受一个正则表达式，也可以传入一个字符串，该字符串会被构造为正则表达式。`match` 方法返回字符串的匹配数组。

```js
const regex = /freeCodeCamp/;
const match = "freeCodeCamp".match(regex);
console.log(match); // ['freeCodeCamp', index: 0, input: 'freeCodeCamp', groups: undefined]
```

- **`replace()` 方法**：该方法接受两个参数：要匹配的正则表达式（或字符串），以及用于替换匹配项的字符串（或对每个匹配项运行的函数）。

:::interactive_editor

```js
const regex = /Jessica/;
const str = "Jessica is rly kewl";
const replaced = str.replace(regex, "freeCodeCamp");
console.log(replaced); // "freeCodeCamp is rly kewl"
```

:::

- **`replaceAll` 方法**：该方法用于将指定模式的所有出现替换为新字符串。如果传入的正则表达式没有全局修饰符，则会抛出错误。

:::interactive_editor

```js
const text = "I hate JavaScript! I hate programming!";
const newText = text.replaceAll("hate", "love");
console.log(newText);  // "I love JavaScript! I love programming!"
```

:::

- **`matchAll` 方法**：该方法用于检索字符串中给定正则表达式的所有匹配项，包括捕获组，并以迭代器形式返回。迭代器是一个允许你遍历集合的对象。

:::interactive_editor

```js
const str = "JavaScript, Python, JavaScript, Swift, JavaScript";
const regex = /JavaScript/g;

const iterator = str.matchAll(regex);

for (let match of iterator) {
  console.log(match[0]); // "JavaScript" for each match
}
```

:::

## 正则表达式修饰符

- **定义**：修饰符，通常称为“标记”，用于修改正则表达式的行为。
- **`i` 标记**：该标记使正则表达式忽略大小写。

:::interactive_editor

```js
const regex = /freeCodeCamp/i;
console.log(regex.test("freecodecamp")); // true
console.log(regex.test("FREECODECAMP")); // true
```

:::

- **`g` 标记**：该标记，也称为全局修饰符，允许正则表达式多次匹配模式。

:::interactive_editor

```js
const regex = /freeCodeCamp/gi;
console.log(regex.test("freeCodeCamp")); // true
console.log(regex.test("freeCodeCamp is great")); // false
```

:::

- **锚点定义**：正则表达式开头的 `^` 锚点表示“匹配字符串开头”。正则表达式结尾的 `$` 锚点表示“匹配字符串结尾”。

:::interactive_editor

```js
const start = /^freeCodeCamp/i;
const end = /freeCodeCamp$/i;
console.log(start.test("freecodecamp")); // true
console.log(end.test("freecodecamp")); // true
```

:::

- **`m` 标记**：锚点用于查找整个字符串的开头和结尾。但你可以使用 `m` 标记（多行修饰符）让正则表达式处理多行。

:::interactive_editor

```js
const start = /^freecodecamp/im;
const end = /freecodecamp$/im;
const str = `I love 
freecodecamp
it's my favorite
`;
console.log(start.test(str)); // true
console.log(end.test(str)); // true
```

:::

- **`d` 标记**：该标记扩展了匹配对象中可获得的信息。

```js
const regex = /freecodecamp/di;
const string = "we love freecodecamp isn't freecodecamp great?";
console.log(string.match(regex));
```

结果如下：

```js
// [
//   'freecodecamp',
//   index: 8,
//   input: "we love freecodecamp isn't freecodecamp great?",
//   groups: undefined,
//   indices: [
//     0: [8, 20],
//     groups: undefined
//   ]
// ]
```

- **`u` 标记**：该标记扩展了正则表达式的功能，使其能够匹配特殊的 Unicode 字符。`u` 标记允许访问特殊类别，如 `Extended_Pictographic` 用于匹配大多数表情符号。还有 `v` 标记，进一步扩展了 Unicode 匹配功能。
- **`y` 标记**：粘滞修饰符的行为与全局修饰符非常相似，但有一些例外。最大区别是，全局正则表达式会从 lastIndex 开始搜索字符串剩余部分的匹配，而粘滞正则表达式如果在上一个 lastIndex 位置没有立即匹配，则返回 null 并将 lastIndex 重置为 0。
- **`s` 标记**：单行修饰符允许正则表达式中的通配符字符（用 `.` 表示）匹配换行符——实际上将字符串视为单行文本。

## 字符类

- **通配符 `.`**：字符类是一种特殊语法，用于匹配字符的集合或子集。你应该学习的第一个字符类是通配符类。通配符用句点（点号）表示，匹配除换行符之外的任意单个字符。要让通配符匹配换行符，记得需要使用 `s` 标记。

```js
const regex = /a./;
```

- **`\d`**：这将匹配字符串中的所有数字（`0-9`）。

```js
const regex = /\d/;
```

- **`\w`**：用于匹配字串中的任何单词字符（`a-zA-Z0-9_`）。单词字符定义为任意字母（a 到 z 或 A 到 Z）、数字（0 到 9）或下划线字符。

```js
const regex = /\w/;
```

- **`\s`**：空白字符类 `\s`，用反斜线加 `s` 表示。该字符类匹配任何空白字符，包括换行符、空格、制表符和特殊的 Unicode 空格字符。
- **否定特殊字符类**：要否定这些字符类之一，可以使用大写字母替代反斜线后的小写字母。下面的例子不匹配数字字符，而是匹配任何非数字字符的单个字符。

```js
const regex = /\D/;
```

- **自定义字符类**：你可以通过将想匹配的字符放入方括号内，创建自定义字符类。

```js
const regex = /[abcdf]/;
```

## 前瞻和后顾断言

- **定义**：前瞻和后顾断言允许你根据周围模式的存在或缺失来匹配特定模式。
- **正向前瞻断言**：当某个模式后面跟着另一个模式时，该断言匹配该模式。构造正向前瞻时，先写要匹配的模式，然后用括号包裹你想用作条件的模式。在开括号后使用 `?=` 来定义该模式为正向前瞻。

```js
const regex = /free(?=code)/i;
```

- **负向前瞻断言**：这是一种条件，用于检查字符串前方不出现某个特定模式。

```js
const regex = /free(?!code)/i;
```

- **正向后顾断言**：该断言仅在某模式前面有另一个特定模式时匹配，但不包括前面的模式。

```js
const regex = /(?<=free)code/i;
```

- **负向后顾断言**：该断言确保某模式前面不紧跟另一个特定模式。仅当指定模式前面没有给定序列时匹配，且不包括前面的序列。

```js
const regex = /(?<!free)code/i;
```

## 正则表达式量词

- **定义**：量词指定模式（或模式的一部分）应出现的次数。它们帮助控制匹配中字符或分组的出现次数。下面的例子用于匹配前一个字符恰好出现四次。

```js
const regex = /^\d{4}$/;
```

- **`*`** ：匹配前一个元素出现 0 次或多次。
- **`+`**：匹配前一个元素出现 1 次或多次。
- **`?`**：匹配前一个元素出现 0 次或 1 次。
- **`{n}`**：匹配前一个元素恰好出现 n 次。
- **`{n,}`**：匹配前一个元素出现 n 次或更多次。
- **`{n,m}`**：匹配前一个元素出现 n 到 m 次。

## 捕获组和反向引用

- **捕获组**：捕获组允许你“捕获”匹配字符串的一部分，以便你需要时使用。捕获组由包含要捕获模式的括号定义，不带前导字符如前瞻。

```js
const regex = /free(code)camp/i;
```

- **非捕获组**：非捕获组类似于捕获组，但不会保存匹配的字串部分以供后续使用。非捕获组由 `(?:...)` 定义。

```js
const regex = /free(?:code)camp/i;
```

- **反向引用**：正则表达式中的反向引用是指在同一表达式中重用之前匹配的部分。它允许你通过编号引用捕获组（括号内的模式）。例如，`$1` 指代第一个捕获组。

:::interactive_editor

```js
const regex = /free(co+de)camp/i;
console.log("freecoooooooodecamp".replace(regex, "paid$1world"));
```

:::

- 你可以在正则表达式内部使用反向引用，通过斜线和捕捉组编号来匹配之前捕捉的相同文本。例如：

:::interactive_editor

```js
const regex = /(hello) \1/i;
console.log(regex.test("hello hello")); // true
console.log(regex.test("hello world")); // false
```

:::
