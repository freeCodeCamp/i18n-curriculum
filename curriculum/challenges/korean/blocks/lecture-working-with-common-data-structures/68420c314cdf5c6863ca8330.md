# --description--

사용자의 기기에서 실행되는 모든 컴퓨터 프로그램은 특정 작업을 완료하기 위해 특정 순서로 실행되는 명확한 지침 집합을 가지고 있습니다.

작업은 숫자 집합을 정렬하거나, 이미지를 수정하거나, 재고를 추적하거나, 심지어 좋아하는 비디오 게임을 실행하는 것일 수 있습니다.

바로 이때 알고리즘이 등장합니다. **알고리즘**은 문제를 해결하거나 작업을 수행하기 위한 명확한 지침 집합입니다.

알고리즘을 "레시피"로 생각할 수 있습니다. 요리할 때 레시피는 필요한 모든 재료를 나열하고 요리를 준비하는 단계별 지침을 제공합니다.

마찬가지로 알고리즘은 컴퓨터가 무엇을 어떻게 해야 하는지 정확히 알려주는 "레시피"라고 생각할 수 있습니다.

알고리즘에는 두 가지 주요 특징이 있습니다:

* 무한히 계속될 수 없습니다. 유한한 단계 내에 끝나야 합니다.
    
* 각 단계는 정확하고 명확해야 합니다.
    

알고리즘은 입력이 없을 수도 있고, 하나 이상일 수도 있으며, 하나 이상의 출력을 생성할 수 있습니다.

알고리즘의 단계는 어떤 프로그래밍 언어와도 독립적입니다.

하지만 실제로 컴퓨터에서 실행하려면 Python이나 JavaScript 같은 프로그래밍 언어로 구현해야 합니다.

알고리즘이 올바르면, 어떤 유효한 입력에 대해서도 출력이 예상한 결과와 일치해야 합니다.

올바른 것 외에도 알고리즘은 효율적이어야 합니다.

알고리즘 효율성은 실행 시간과 작업을 완료하는 데 필요한 메모리 공간 측면에서 측정할 수 있습니다.

알고리즘의 효율성을 아는 것은 입력 크기가 커질 때 얼마나 잘 작동할지 가늠할 수 있기 때문에 매우 중요합니다.

예를 들어, 15개의 정수를 정렬하는 것과 100만 개의 정수를 정렬하는 것은 다릅니다.

처리 크기와 복잡도가 커질수록 알고리즘이 이를 처리할 만큼 효율적이지 않으면, 매우 느린 컴퓨터 프로그램이 되거나 시스템 전체가 다운될 수도 있습니다.

그래서 가능한 가장 효율적인 알고리즘을 개발하고 선택하는 것이 매우 중요합니다.

이때 빅오 표기법이 매우 중요해집니다.

빅오 표기법은 입력 크기가 커질 때 알고리즘의 최악의 성능 또는 성장률을 나타냅니다.

알고리즘의 성장률은 입력 크기가 커짐에 따라 필요한 자원이 얼마나 증가하는지를 의미합니다.

빅오 표기법은 최악의 성능에 초점을 맞춥니다. 이는 입력과 상관없이 최악의 상황에서도 알고리즘이 얼마나 효율적인지 이해하는 데 중요합니다.

앞서 정렬 예시로 돌아가면, 100만 개의 정수를 정렬하는 것이 15개를 정렬하는 것보다 직관적으로 더 많은 시간과 자원이 필요합니다.

하지만 얼마나 더 많이 필요할까요?

이는 선택한 알고리즘에 따라 다릅니다.

빅오 표기법은 알고리즘 효율성을 정확한 숫자로 알려주지는 않지만, 알고리즘이 수행하는 연산 수를 기준으로 입력 크기가 커질 때 어떻게 확장되는지 알려줍니다.

빅오 표기법에서 입력 크기는 보통 `n`로 나타냅니다. 예를 들어, 입력이 리스트라면 `n`은 그 리스트의 요소 수를 나타냅니다.

상수 계수와 낮은 차수 항은 연산 수를 기준으로 알고리즘의 시간 복잡도를 구할 때 고려하지 않습니다. `n` 크기가 커질수록 이러한 작은 항들이 전체 연산 수에 미치는 영향이 점점 작아지기 때문입니다.

알고리즘의 전체 동작을 지배하는 항은 `n`인 입력 크기를 가진 최고 차수 항이 됩니다.

예를 들어, 알고리즘이 `7n + 20` 연산을 수행한다면, 상수 `20`가 최종 결과에 미치는 영향은 `n`가 커질수록 점점 작아집니다. `7n` 항이 지배적이 되어 알고리즘의 전체 동작과 효율성을 정의합니다.

또 다른 예로, `20n² + 15n + 7` 연산을 수행하는 알고리즘이 있습니다. `20n²`가 커질수록 `n` 항이 지배적이 되어 이 알고리즘은 `n²`를 가진 항 때문에 이차 시간 복잡도를 갖습니다.

이차 시간 복잡도는 알고리즘 세계에서 볼 수 있는 여러 시간 복잡도 유형 중 하나입니다.

가장 흔한 유형 몇 가지를 배워봅시다.

**`O(1)`**는 "상수 시간 복잡도"로 알려져 있습니다. 상수 시간 복잡도를 가진 알고리즘은 입력 크기와 상관없이 실행 시간이 항상 같습니다.

예를 들어, 숫자가 짝수인지 홀수인지 확인하는 작업은 숫자 자체와 상관없이 항상 같은 시간이 걸립니다.

```python
def check_even_or_odd(number):
    if number % 2 == 0:
        return 'Even'
    else:
        return 'Odd'
```

**`O(log n)`**는 "로그 시간 복잡도"로 알려져 있습니다. 이는 입력 크기가 커질수록 알고리즘이 필요한 시간이 천천히 증가함을 의미합니다. 문제 크기를 일정 비율로 반복해서 줄이는 문제에서 흔히 나타납니다.

예를 들어, 이진 탐색(Binary Search)이라는 인기 있는 탐색 알고리즘은 `O(log n)` 최악의 시간 복잡도를 가집니다. 이는 비교할 때마다 남은 요소의 절반을 제거하기 때문에 전체적으로 더 효율적입니다.

**`O(n)`**는 "선형 시간 복잡도"로 알려져 있습니다. 이 시간 복잡도를 가진 알고리즘의 실행 시간은 입력 크기에 비례해 증가합니다.

예를 들어, 리스트의 모든 요소를 반복하는 `for` 루프는 리스트 요소 수가 증가할수록 반복 횟수가 늘어납니다. 리스트 크기가 두 배가 되면 연산 수 역시 대략 두 배가 됩니다.

```python
for grade in grades:  # grades is a list.
    print(grade)
```

**`O(n log n)`**는 "로그-선형 시간 복잡도"로 알려져 있습니다. 이는 병합 정렬(Merge Sort)과 퀵 정렬(Quick Sort) 같은 효율적인 정렬 알고리즘에서 흔히 볼 수 있는 시간 복잡도입니다.

**`O(n²)`**는 "이차 시간 복잡도"로 알려져 있습니다. 이 알고리즘의 실행 시간은 입력 크기에 대해 제곱 비례로 증가하며, 일반적으로 실제 문제에 비효율적입니다.

중첩 루프는 이차 시간 복잡도의 흔한 예입니다. 내부 루프는 외부 루프의 `n` 반복마다 `n` 반복을 수행하여 `n` 제곱만큼 반복합니다.

```python
for i in range(n):
    for j in range(n):
        print("Hello, World!")
```

다른 시간 복잡도로는 "지수 시간 복잡도"인 `O(2^n)`와 "팩토리얼 시간 복잡도"인 `O(n!)`가 있습니다. 둘 다 실제 상황에서는 비효율적입니다.

이 그래프에서는 가장 흔한 시간 복잡도를 나타내는 수학 함수들의 성장을 비교할 수 있습니다. x축(가로)은 입력 크기, y축(세로)은 알고리즘의 실행 시간을 나타냅니다.

이차 시간 복잡도(`O(n²)`) (노란색)가 다른 것들보다 훨씬 빠르게 증가하는 반면, 상수 시간 복잡도(`O(1)`) (빨간색)는 입력이 커져도 일정하게 유지되는 것을 볼 수 있습니다.

<img src="https://cdn.freecodecamp.org/curriculum/lecture-transcripts/what-is-an-algorithm-and-how-does-big-o-notation-work-1.png" alt="graph comparing time complexity">

좋습니다. 지금까지 시간 요구 사항 측면에서 빅오 표기법에 대해 배웠지만, 이 표기법은 공간 요구 사항 맥락에도 적용할 수 있습니다.

이 맥락에서는 입력 크기가 커질 때 알고리즘이 필요로 하는 메모리 공간이 어떻게 증가하는지를 설명합니다.

"상수 공간 복잡도" `O(1)`를 가진 알고리즘은 입력이 커져도 항상 일정한 메모리 공간을 필요로 합니다.

예를 들어, 메모리에 몇 개의 변수만 생성하고 저장하는 알고리즘이 이에 해당합니다.

반면, "선형 공간 복잡도" `O(n)`를 가진 알고리즘은 입력 크기가 커짐에 따라 필요한 공간이 비례해서 증가합니다.

예를 들어, 길이가 `n`인 리스트의 복사본을 생성하고 저장하는 알고리즘이 이에 해당합니다.

마지막으로, "이차 공간 복잡도" `O(n²)`를 가진 알고리즘은 입력 크기가 커질수록 공간 요구가 제곱 비례로 증가합니다.

예를 들어, 입력 크기로 결정되는 차원의 2D 행렬을 생성하여 가능한 모든 쌍을 저장하는 경우입니다.

알고리즘은 컴퓨터 프로그램의 기본 구성 요소이며, 빅오 표기법은 입력 크기가 커질 때 최악의 경우 시간과 공간 요구 사항이 어떻게 확장되는지 분석하는 강력한 프레임워크입니다. 효율성을 이해하는 것은 실제 상황에서 효율적으로 작동하는 소프트웨어를 개발하는 데 매우 중요합니다.
