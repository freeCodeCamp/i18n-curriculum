# --interactive--

Властивість `grid-template-areas` дозволяє створювати візуальний макет сітки за допомогою іменованих міток.

Потім ви призначаєте ці мітки конкретним елементам сітки за допомогою властивості grid-area. Іншими словами, ці іменовані мітки також називають "іменами областей сітки".

Ось базовий синтаксис властивості `grid-template-areas`:

```css
grid-template-areas:
 'header header header'
 'left-sidebar main right-sidebar'
 'footer footer footer';
```

Ось основні моменти, які варто запам’ятати з базового синтаксису:

- Значення на кшталт `header` і `main` — це імена областей сітки.

- Кожне значення, розділене пробілом у рядку, відповідає стовпцю.

- Кожен рядок представляє рядок у сітці.

Отже, у цьому синтаксисі ми маємо контейнер сітки розміром 3 на 3.

Після визначення шаблону потрібно використати властивість `grid-area` і іменовані мітки (або області) як значення, щоб повідомити CSS, що конкретний елемент належить до цієї області сітки.

Властивість `grid-area` пов’язує елемент сітки з іменованою областю, яку ви визначаєте у `grid-template-areas`.

Популярний спосіб продемонструвати можливості властивості `grid-template-areas` — створити класичний макет Holy Grail.

Якщо ви не знаєте, що це таке, макет Holy Grail — це шаблон вебдизайну з заголовком, нижнім колонтитулом, двома бічними панелями та основною областю вмісту. Він гарантує, що основний вміст має пріоритет, а бічні панелі та інші розділи адаптивно підлаштовуються в межах макета.

Існує багато способів реалізувати макет Holy Grail, але використання `grid-template-areas` і властивості `grid-area` — найпростіший спосіб його створити.

Ось приклад для макета Holy Grail:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<div class="grid-container">
  <div class="header">
    <h2>Header</h2>
  </div>
  <div class="sidebar-left">
    <h2>Left Sidebar</h2>
  </div>
  <div class="main"><h2>Main Content</h2></div>
  <div class="sidebar-right">
    <h2>Right Sidebar</h2>
  </div>
  <div class="footer">
    <h2>Footer</h2>
  </div>
</div>
```

```css
.grid-container {
  display: grid;
  grid-template-areas:
    'header header header'
    'sidebar-left main sidebar-right'
    'footer footer footer';
  grid-gap: 10px;
  background-color: #2196F3;
  padding: 10px;
}

.header {
  grid-area: header;
  background-color: rgba(255, 255, 255, 0.8);
  padding: 20px;
  text-align: center;
}

.sidebar-left {
  grid-area: sidebar-left;
  background-color: rgba(255, 255, 255, 0.8);
  padding: 20px;
  text-align: center;
}

.main {
  grid-area: main;
  background-color: rgba(255, 255, 255, 0.8);
  padding: 20px;
  text-align: center;
}

.sidebar-right {
  grid-area: sidebar-right;
  background-color: rgba(255, 255, 255, 0.8);
  padding: 20px;
  text-align: center;
}

.footer {
  grid-area: footer;
  background-color: rgba(255, 255, 255, 0.8);
  padding: 20px;
  text-align: center;
}
```

:::

Зверніть увагу, що властивості `grid-template-areas` і `grid-area` можна використовувати незалежно одна від одної.

Властивість `grid-template-areas` спеціально використовується для визначення візуального макета шляхом проєктування іменованих областей сітки всередині контейнера сітки.

З іншого боку, властивість `grid-area` використовується для позиціювання окремих елементів сітки, або вказуючи їхні позиції рядків і стовпців, або посилаючись на іменовані області, визначені властивістю `grid-template-areas`.
