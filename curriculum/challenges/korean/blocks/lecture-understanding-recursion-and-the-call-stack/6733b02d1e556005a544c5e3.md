# --interactive--

재귀는 함수가 기본 사례에 도달할 때까지 반복해서 자신을 호출할 수 있게 하는 복잡한 기능입니다. 전통적인 반복문과 달리, 재귀는 깊이가 알려지지 않은 중첩 객체/배열이나 파일 트리 같은 것을 처리할 수 있습니다. 하지만 주어진 숫자에서 카운트다운하는 기본 작업에도 사용할 수 있습니다.

정확히 그 작업을 수행하는 함수를 만들어 보겠습니다. 함수 이름은 `recursiveCountdown`로 하며, 숫자를 인자로 받아야 합니다. 이 숫자를 콘솔에 출력하게 하겠습니다:

:::interactive_editor

```js
const recursiveCountdown = (number) => {
  console.log(number);
};

recursiveCountdown(5);
```

:::

이제 이 함수를 호출하고 숫자 5를 전달하면, 터미널에 숫자가 출력됩니다. 하지만 그 외에는 아무 일도 일어나지 않습니다 — 숫자 5는 분명히 카운트다운이 아닙니다.

재귀 부분을 만들기 전에, 먼저 기본 사례를 정해야 합니다. 기본 사례가 없으면, 코드가 메모리 할당을 초과해 충돌할 때까지 계속 실행됩니다.

:::interactive_editor

```js
const recursiveCountdown = (number) => {
  if (number < 1) {
    return;
  }
  console.log(number);
};

recursiveCountdown(5);
```

:::

기본 사례로, 숫자가 1보다 작으면 카운트다운을 멈추게 하겠습니다. 이 기본 사례에 도달하면, 함수 실행을 종료하기 위해 return을 사용합니다.

기본 사례를 안전하게 준비했으니, 이제 재귀를 설정할 수 있습니다. 함수를 재귀적으로 만드는 핵심은 함수가 자기 자신을 호출한다는 점입니다. 여기서는 숫자를 출력한 후에 함수를 호출하고 싶습니다. 하지만 카운트다운하려면, 새 숫자는 1만큼 줄어야 합니다:

:::interactive_editor

```js
const recursiveCountdown = (number) => {
  if (number < 1) {
    return;
  }
  console.log(number);
  recursiveCountdown(number - 1);
};

recursiveCountdown(5); 
```

:::

이 코드는 콘솔에 5, 4, 3, 2, 1을 출력할 것입니다.

숫자 다섯 개를 얻었네요! 그런데 만약 카운트업을 하고 싶다면 어떻게 할까요? 완전히 새로운 함수를 작성하는 대신, 출력과 재귀 호출 순서를 바꾸면 됩니다:

:::interactive_editor

```js
const recursiveCountdown = (number) => {
    if (number < 1) {
        return;
    }
    recursiveCountdown(number - 1);
    console.log(number);
  };

recursiveCountdown(5);
```

:::

이 코드는 콘솔에 1, 2, 3, 4, 5를 출력할 것입니다.

그런데 왜 이렇게 작동할까요? 이를 이해하려면 호출 스택을 알아야 합니다. 호출 스택은 JavaScript가 함수 호출을 추적하고 해결하는 방식입니다. 스택은 후입선출 구조로 작동합니다. 이를 더 잘 이해하려면 함수에 로그를 추가해 보겠습니다:

:::interactive_editor

```js
const recursiveCountdown = (number) => {
  console.log(`Function execution started for number: ${number}`);
  if (number < 1) {
    console.log(`Base case reached, begin resolving stack`);
    return;
  }
  console.log(`Calling recursiveCountdown with number: ${number - 1}`);
  recursiveCountdown(number - 1);
  console.log(`Function execution completed for number: ${number}`);
};

recursiveCountdown(5);
```

:::

여기 네 가지 주요 로그 문을 추가했습니다. 첫 번째 로그는 함수 호출이 실행되기 시작할 때 실행됩니다. 세 번째 로그는 재귀 함수가 호출되기 직전에 실행됩니다. 네 번째 로그는 함수 실행이 끝났을 때 실행됩니다. 결과는 다음과 같습니다:

```md
Function execution started for number: 5
Calling recursiveCountdown with number: 4
Function execution started for number: 4
Calling recursiveCountdown with number: 3
Function execution started for number: 3
Calling recursiveCountdown with number: 2
Function execution started for number: 2
Calling recursiveCountdown with number: 1
Function execution started for number: 1
Calling recursiveCountdown with number: 0
Function execution started for number: 0
Base case reached, begin resolving stack
Function execution completed for number: 1
Function execution completed for number: 2
Function execution completed for number: 3
Function execution completed for number: 4
Function execution completed for number: 5
```

그런데 이게 어떻게 일어날까요? 바로 호출 스택이 작동하는 부분입니다. `recursiveCountdown(5)`을 호출하면, 그 함수 호출이 호출 스택에 추가됩니다.

그 함수 호출이 `recursiveCountdown(4)`를 호출해야 할 때, 결과를 기다리기 위해 멈춥니다. 그 사이에 `recursiveCountdown(4)`이 호출 스택에 `recursiveCountdown(5)` 위에 추가됩니다.

그 함수 호출이 `recursiveCountdown(3)`를 호출해야 할 때, 결과를 기다리기 위해 멈춥니다. 그 사이에 `recursiveCountdown(3)`이 호출 스택에 `recursiveCountdown(4)` 위에 추가됩니다.

그 함수 호출이 `recursiveCountdown(2)`을 호출해야 할 때, 결과를 기다리기 위해 멈춥니다. 그 사이에 `recursiveCountdown(2)`이 호출 스택에 `recursiveCountdown(3)` 위에 추가됩니다.

그 함수 호출이 `recursiveCountdown(1)`을 호출해야 할 때, 결과를 기다리기 위해 멈춥니다. 그 사이에 `recursiveCountdown(1)`이 호출 스택에 `recursiveCountdown(2)` 위에 추가됩니다.

마지막으로, 그 함수 호출이 `recursiveCountdown(0)`를 호출해야 할 때, 결과를 기다리기 위해 멈춥니다. 그 사이에 `recursiveCountdown(0)`이 호출 스택에 `recursiveCountdown(1)` 위에 추가됩니다.

하지만 `recursiveCountdown(0)`은 다른 함수를 호출하지 않고, 기본 사례에 도달해 일찍 반환합니다. 그 함수 실행이 끝났으므로, 함수 호출은 “해결됨”으로 간주됩니다. 호출이 해결되면, 스택에서 제거됩니다.

이제 `recursiveCountdown(1)`은 더 이상 그 호출을 기다리지 않고, 스택 맨 위에 있어 실행을 재개할 수 있습니다.

`recursiveCountdown(1)`이 해결되어 스택에서 제거되고, `recursiveCountdown(2)`이 실행을 재개합니다.

`recursiveCountdown(2)`이 해결되어 스택에서 제거되고, `recursiveCountdown(3)`가 실행을 재개합니다.

`recursiveCountdown(3)`이 해결되어 스택에서 제거되고, `recursiveCountdown(4)`가 실행을 재개합니다.

`recursiveCountdown(4)`이 해결되어 스택에서 제거되고, `recursiveCountdown(5)`가 실행을 재개합니다.

마지막으로 `recursiveCountdown(5)`이 해결되어 스택에서 제거됩니다. 호출 스택이 이제 비어 재귀가 완료되었습니다!

이것이 JavaScript에서 재귀가 작동하는 기본 개요입니다. 복잡한 개념이니, 호출 스택의 동작에 익숙해질 때까지 코드를 작성하고 로그를 찍어 보세요.

재미있는 사실 하나를 알려드리자면, 기본 사례가 없으면 코드가 메모리 부족으로 충돌할 수 있다고 했죠. 이는 재귀가 계속해서 함수 호출을 호출 스택에 쌓아 스택 오버플로우가 발생하기 때문입니다. 인기 있는 프로그래밍 커뮤니티 이름도 여기서 따온 것입니다.
