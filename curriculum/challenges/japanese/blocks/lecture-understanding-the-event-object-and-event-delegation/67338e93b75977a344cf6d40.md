# --description--

イベントバブリング、または伝播とは、イベントが発生したときに親オブジェクトへ「バブルアップ」する仕組みのことです。例えば、次のコードを考えてみましょう。

```html
<p>
  <span>Click me~!</span>
</p>
```

ここで `p` 要素は `span` 要素の親とみなされます。

`span` 要素をクリックすると、指示通りにすると、`span` 要素が `click` イベントのターゲットになります。しかし、そのイベントは親にもバブルアップし、`p` 要素が必要に応じてそのイベントを受け取り処理できます。

では、これは実際にはどういう意味でしょうか？例えば、`p` 要素にイベントリスナーを付けることができます。

```js
const p = document.querySelector("p");
p.addEventListener("click", (event) => {
  console.log(event.target);
});
```

そうすると、`span` 要素をクリックしたときに、コンソールに `Click me~!` というテキストが表示されます。

イベントは親の `p` 要素に伝播し、そのイベントリスナーでイベントのターゲットを表示するために処理されます。

ターゲットがまだ `span` 要素のままであることに注目してください。これは `span` 要素が最初のクリックを受け取ったためです。

動作をより確かめるために、コードを拡張してみましょう。

```js
const p = document.querySelector("p");
const span = document.querySelector("span");
p.addEventListener("click", (event) => {
  console.log("P listener: ");
  console.log(event.target);
});
span.addEventListener("click", (event) => {
  console.log("Span listener: ");
  console.log(event.target);
});
```

イベントがどのようにバブルアップするかイメージできるように、`span` 要素をクリックした後にコンソールに表示される内容は次の通りです。

```html
"Span listener: "
<span>Click me~!</span>
"P listener: "
<span>Click me~!</span>
```

次に、`stopPropagation()` を使ってイベントの伝播を止めた場合に何が起きるか見てみましょう。`span` のイベントリスナー内で呼び出します。

```js
const p = document.querySelector("p");
const span = document.querySelector("span");
p.addEventListener("click", (event) => {
  console.log("P listener: ");
  console.log(event.target);
});
span.addEventListener("click", (event) => {
  console.log("Span listener: ");
  console.log(event.target);
  event.stopPropagation();
});
```

そして再び `span` をクリックします。

```html
"Span listener: "
<span>Click me~!</span>
```

今回は `p` のリスナーが発火しません。子の `p` 要素の処理中に伝播を止めるよう指示したため、`span` 要素にはイベントが届かないのです。

イベントデリゲーションはこれとは逆の考え方です。キャプチャされたイベントを別の要素に委譲するプロセスです。

コードに戻り、`span` 要素をクリックすると赤くなるように更新してみましょう。

```js
const p = document.querySelector("p");
const span = document.querySelector("span");
p.addEventListener("click", (event) => {});
span.addEventListener("click", (event) => {
  event.target.style.color = "red";
});
```

しかし、もし `span` 要素が20個あったらどうでしょう？あるいは JavaScript で動的に `span` 要素をさらに作成した場合は？

すべての `span` 要素にイベントリスナーを付ける代わりに、親の `p` 要素にリスナーを付けておけば済みます。つまり、`span` のクリック処理を親の `p` 要素に委譲できるのです。

コードは次のようになるかもしれません。

```js
const p = document.querySelector("p");
p.addEventListener("click", (event) => {
  event.target.style.color = "red";
});
```

`span` 要素にはリスナーが一切付いていないことに注目してください。イベント処理を正しく `p` 要素に委譲できています。では、動作はどうでしょうか？

いくつか `span` 要素を追加して試してみましょう。

```html
<p>
  <span>Click me~!</span>
  <span>Click me~!</span>
  <span>Click me~!</span>
  <span>Click me~!</span>
</p>
```

これで `span` をクリックするたびに、その要素のテキストが赤くなります。

このように、単一のイベントリスナーで `click` 要素から親の `span` へ `p` イベントをバブルアップさせ、その `click` イベントの処理を `p` 要素に委譲できました。

イベントの伝播とデリゲーションは、特にテーブルのように深くネストされた要素がある場合は複雑なトピックです。ここで紹介したコードを使ってさらに探求し、実験してみることをお勧めします。
