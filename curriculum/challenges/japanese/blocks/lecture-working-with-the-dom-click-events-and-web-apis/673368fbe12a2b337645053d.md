# --description--

`Event` オブジェクトは、ユーザーがウェブページと何らかの形でやり取りしたときに発生するペイロードです。これらのやり取りは、ボタンのクリックや入力へのフォーカス、モバイルデバイスの振動など、さまざまなものがあります。

すべての JavaScript オブジェクトと同様に、`Event` オブジェクトには役立つプロパティがいくつかあります。利用できるプロパティは、このペイロードを発生させたイベントによって異なります。

すべての `Event` オブジェクトには `type` プロパティがあります。このプロパティは、`"keydown"` や `"click"` のように、ペイロードを発生させたイベントの種類を示します。これらの値は、`addEventListener()` に渡して `Event` オブジェクトを取得・利用できる値と対応しています。

`Event` オブジェクトには常に `target` プロパティがあります。`target` プロパティは、イベントを発生させたオブジェクトへの参照です。多くの場合、これは何らかの `HTMLElement` オブジェクトや `Document`、`Window` オブジェクトになりますが、`AudioContext` のようにより具体的なものの場合もあります。

イベントにはメソッドもあり、これはオブジェクトのプロパティとして公開されている関数です。よく使われるメソッドの一つに `preventDefault()` があります。これは呼び出すとイベントのデフォルトの動作を防ぎます。

たとえば、フォームの提出を自分で処理したい場合、`preventDefault()` を呼び出してブラウザがフォームデータを `POST` リクエストとして送信しようとするのを防ぐことができます。`POST` リクエストについては今後のレッスンで詳しく学びます。

また、`stopPropagation()` メソッドに出会うことも多いでしょう。このメソッドはイベントのバブリングや親要素への伝播を防ぎます。これが何を意味するかは今後のレッスンで学びます。

`Event` オブジェクトの特定の実装に特有の多くのプロパティもあります。たとえば、`FetchEvent` にはイベントを発生させたリクエストを含む request プロパティがあります。

利用可能なプロパティがわからない場合は、対象の `Event` オブジェクトをログに出力したり、ドキュメントを確認したりしてください。
