# --description--

前回のチャレンジでは、`duck` オブジェクトにメソッドを追加しました。return文内で `duck.name` のドット記法を使って `name` プロパティの値にアクセスしていました。

```js
sayName: function() {return "The name of this duck is " + duck.name + ".";}
```

これはオブジェクトのプロパティにアクセスする有効な方法ですが、落とし穴があります。変数名が変わると、元の名前を参照しているコードもすべて更新する必要があります。短いオブジェクト定義では問題になりませんが、プロパティへの参照が多いオブジェクトではエラーが起きやすくなります。

これらの問題を避ける方法が `this` キーワードです。

```js
let duck = {
  name: "Aflac",
  numLegs: 2,
  sayName: function() {return "The name of this duck is " + this.name + ".";}
};
```

`this` は奥が深いトピックで、上記の例は使い方の一例にすぎません。ここでの文脈では、`this` はメソッドが関連付けられているオブジェクト、つまり `duck` を指します。オブジェクトの名前を `mallard` に変えても、コード内の `duck` へのすべての参照を探す必要はありません。これによりコードが再利用可能で読みやすくなります。

# --instructions--

`dog.sayLegs` メソッドを修正して `dog` への参照をすべて削除してください。`duck` の例を参考にしてください。

# --hints--

`dog.sayLegs()` は与えられた文字列を返す必要があります。

```js
assert(dog.sayLegs() === 'This dog has 4 legs.');
```

コードは `this` キーワードを使って `numLegs` の `dog` プロパティにアクセスする必要があります。

```js
assert(__helpers.removeJSComments(code).match(/this\.numLegs/g));
```
