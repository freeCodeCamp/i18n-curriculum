# --description--

Scopriamo cos’è il terminale e in cosa si differenzia dalla riga di comando.

La riga di comando è un’interfaccia di input testuale di base che permette a un utente di inserire “comandi”, solitamente sotto forma di una serie di caratteri, e inviarli o eseguirli, di solito premendo il tasto “Invio”. Le interfacce a riga di comando si trovano più comunemente all’interno di un terminale.

Un terminale è un’applicazione speciale che offre un’interfaccia a riga di comando per eseguire comandi a livello di sistema oltre alle operazioni di lettura/scrittura di base. Potresti anche sentire parlare di “emulatori di terminale”, che sono essenzialmente applicazioni che racchiudono un’interfaccia terminale di base per offrire funzionalità o caratteristiche aggiuntive. Ma per la maggior parte degli usi generali, puoi tranquillamente chiamarli “terminali”.

Infine, dobbiamo parlare anche delle “shell”. Una shell è il software che racchiude la riga di comando e interpreta i tuoi input come comandi, restituendo l’output.

Ma come puoi consultare il terminale sul tuo sistema?

Se usi una versione moderna di Windows, probabilmente hai due opzioni diverse. Windows offre due applicazioni shell distinte già pronte all’uso: Powershell e Prompt dei comandi.

Entrambe queste shell hanno le loro applicazioni terminal (che portano il nome della rispettiva shell), quindi puoi usare il menu delle applicazioni per avviare un terminale Powershell o un terminale Prompt dei comandi.

Windows offre anche un emulatore di terminale relativamente nuovo e modernizzato chiamato Microsoft Terminal.

Puoi installare questa applicazione dal Windows Store e poi avviarla come faresti con i terminali Powershell o Prompt dei comandi. Il vantaggio di Microsoft Terminal è che ti permette di consultare sia le shell Powershell sia Prompt dei comandi dalla stessa applicazione, oltre alla rispettiva shell Linux se hai configurato i Windows Subsystem for Linux.

MacOS offre un terminale chiamato semplicemente “Terminale”. Puoi consultarlo tramite la ricerca Spotlight o tramite il launcher delle applicazioni.

MacOS supporta anche emulatori di terminale di terze parti, come iTerm o Ghostty. Puoi installare queste applicazioni dalle loro pagine ufficiali.

Su Linux, il terminale predefinito (e come lo avvii) dipende molto dalla distribuzione e dall’ambiente desktop che usi. Esiste anche molto supporto per emulatori di terminale di terze parti. Per esempio, potresti configurare kitty su Arch Linux e avviarlo con un gestore di applicazioni come wofi.

Linux probabilmente offre la più ampia varietà di applicazioni terminal tra cui scegliere. La maggior parte può essere installata tramite il gestore di pacchetti della tua distribuzione.

La distinzione tra “terminale”, “shell” e “riga di comando” può sembrare a volte piuttosto pedante. E per la maggior parte del tempo, i termini possono essere usati in modo intercambiabile. Ma è importante sapere che hanno significati specifici e quali sono questi significati.
