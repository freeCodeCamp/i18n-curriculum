# --interactive--

**ПРИМІТКА**: Деякі інтерактивні приклади можуть використовувати CSS, який ви ще не вивчали. Не хвилюйтеся, якщо не розумієте весь код. Мета прикладів — показати вам попередній перегляд цих концепцій дизайну, щоб ви краще розуміли, як все працює. Щоб побачити попередній перегляд, потрібно увімкнути інтерактивний редактор.

Коли йдеться про вебдизайн, існує багато типів навігаційних допоміжних засобів, які ви можете використовувати. Прикладами є верхні навігаційні панелі, бічні панелі та нижні колонтитули. Але якщо ваш сайт досить складний і має глибші рівні навігації, варто розглянути використання breadcrumbs.

Breadcrumbs — це навігаційний помічник, який показує користувачу, де він знаходиться в ієрархії сайту. Ось приклад того, як виглядають breadcrumbs для макету сайту електроніки:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<div class="breadcrumbs">
  Home / Electronics / Phones / Smartphone XYZ
</div>
```

```css
.breadcrumbs {
  font-size: 0.9rem;
  color: #555;
  margin: 1rem 0;
}
```

:::

На більшості сайтів breadcrumbs розміщують у верхній частині сторінки, показуючи користувачу шлях, яким він пройшов, щоб потрапити на поточну сторінку. Починаючи з `Homepage`, користувач перейшов до категорії `Electronics`, потім до категорії `Phones` і нарешті до продукту `Smartphone XYZ`. Ви, мабуть, стикалися з breadcrumbs на сайті, коли шукали продукт або конкретну інформацію.

Використання breadcrumbs корисне, бо допомагає користувачам зрозуміти, де вони знаходяться в ієрархії сайту, і як повернутися на попередні сторінки. Це особливо важливо, коли користувач прийшов із результатів пошуку або зовнішнього посилання і потребує розуміння контексту сторінки, на якій він перебуває.

При проєктуванні breadcrumbs варто врахувати кілька моментів. Перший — визначити, який буде роздільник. Роздільник — це символ, який відокремлює різні рівні ієрархії. Загальними роздільниками є знак більше (`>`), праві лапки (`»`) і коса риска (`/`).

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<div class="breadcrumbs">
  Home &gt; Electronics &gt; Phones &gt; Smartphone XYZ
</div>
```

```css
.breadcrumbs {
  font-size: 0.9rem;
  color: #555;
  margin: 1rem 0;
}
```

:::

Другий момент — розміщення breadcrumbs. Зазвичай їх розміщують у верхній частині сторінки, або над, або під головною навігаційною панеллю. Користувачам не повинно бути складно знайти breadcrumbs, тому переконайтеся, що вони видимі та легко доступні.

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<nav class="main-nav">
  <a href="#">Home</a>
  <a href="#">Electronics</a>
  <a href="#">Phones</a>
</nav>
<div class="breadcrumbs">
  Home &gt; Electronics &gt; Phones &gt; Smartphone XYZ
</div>
```

```css
.main-nav {
  background-color: #333;
  padding: 1rem;
}

.main-nav a {
  color: white;
  margin-right: 1rem;
  text-decoration: none;
}

.breadcrumbs {
  font-size: 0.9rem;
  color: #555;
  margin: 1rem 0;
}
```

:::

Ще один момент — розмір breadcrumbs. Вони мають бути достатньо великими, щоб їх легко читати, але не настільки великими, щоб займати забагато місця на сторінці. Пам’ятайте, що breadcrumbs не призначені для основної навігації, а слугують допоміжним навігаційним засобом. На сайтах із великою кількістю інформації на сторінці користувачі можуть легко бачити, де вони знаходяться в ієрархії, і повертатися на попередні сторінки за допомогою breadcrumbs.

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<div class="breadcrumbs">
  Home &gt; Electronics &gt; Phones &gt; Smartphone XYZ
</div>
<p>
  Lorem ipsum dolor sit amet, consectetur adipiscing elit. Sed do eiusmod tempor incididunt ut labore et dolore magna aliqua.
</p>
```

```css
.breadcrumbs {
  font-size: 1rem;
  color: #555;
  margin: 1rem 0;
}   

p {
  font-size: 1.2rem;
  line-height: 1.6;
}
```

:::

Отже, коли варто використовувати breadcrumbs на вашому сайті? Якщо ваш сайт простий і має неглибоку ієрархію, breadcrumbs можуть бути не потрібні. Неглибока ієрархія означає, що сайт має лише кілька рівнів навігації, тому breadcrumbs можуть не додати значної користі. Зазвичай breadcrumbs використовують на сайтах електронної комерції, новинних сайтах та інших сайтах із глибокою ієрархією, наприклад, на сайтах технічної документації.
