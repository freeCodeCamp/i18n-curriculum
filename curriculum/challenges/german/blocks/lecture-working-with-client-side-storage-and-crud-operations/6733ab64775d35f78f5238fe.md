# --description--

CRUD ist ein Akronym, das für Create, Read, Update und Delete steht. Dies sind die vier Operationen der persistenten Speicherung.

Persistente Speicherung bedeutet, Daten so zu speichern, dass sie auch nach dem Ausschalten des Geräts oder einem Neustart verfügbar bleiben.

Zu verstehen, wie die Grundoperation von CRUD funktioniert, ist entscheidend für die Webentwicklung, da sie die Grundlage für die Arbeit mit Datenbanken und den Aufbau von Anwendungen bildet, bei denen Nutzer Daten hinzufügen, anzeigen, ändern und löschen können.

Schauen wir uns nun jeden Teil von CRUD genauer an:

- Create bezieht sich auf den Prozess, neue Daten zu erstellen. Zum Beispiel könnte dies in einer Webanwendung der Fall sein, wenn ein Nutzer einen neuen Beitrag in einem Blog hinzufügt.

- Read ist die Operation, bei der Daten aus einer Datenbank abgerufen werden. Wenn Sie beispielsweise einen Blogbeitrag besuchen oder Ihr Profil auf einer Website ansehen, führen Sie eine Read-Operation aus, um die in der Datenbank gespeicherten Daten abzurufen und anzuzeigen.

- Update beinhaltet das Ändern vorhandener Daten in der Datenbank. Ein Beispiel wäre das Bearbeiten eines Blogbeitrags oder das Aktualisieren Ihrer Profilinformationen.

- Delete ist die Operation, die Daten aus einer Datenbank entfernt. Wenn Sie beispielsweise einen Blogbeitrag oder ein Konto löschen, führen Sie eine Delete-Operation aus.

CRUD wird bei der Arbeit mit Datenbanken, der Benutzeroberfläche und RESTful APIs verwendet. RESTful APIs sind eine Reihe von Konventionen zum Erstellen von Webdiensten, die es dem Client ermöglichen, mit einer Datenbank oder einem Back-End-System zu interagieren, indem CRUD-Operationen über standardisierte HTTP-Methoden ausgeführt werden.

HTTP steht für Hypertext Transfer Protocol und ist die Grundlage für die Datenkommunikation im Web. Es gibt HTTP-Methoden, die die Aktionen definieren, die über das Web auf Ressourcen ausgeführt werden können. Die gängigen Methoden sind `GET`, `POST`, `PUT`, `PATCH`, `DELETE`.

Sie werden in zukünftigen Lektionen mehr über RESTful APIs und HTTP lernen, aber hier ist eine kurze Übersicht, wie CRUD auf die verschiedenen HTTP-Methoden abgebildet wird.

- `POST` wird verwendet, um eine neue Ressource zu erstellen.

- `GET` wird verwendet, um Daten abzurufen oder zu lesen.

- `PUT` wird verwendet, um eine Ressource durch vollständiges Ersetzen zu aktualisieren.

- `PATCH` wird verwendet, um eine Ressource teilweise zu aktualisieren.

- `DELETE` wird verwendet, um eine Ressource zu entfernen.

Hier ist ein Beispiel, wie CRUD-Operationen in Code mit einem einfachen Array in JavaScript dargestellt werden könnten:

```js
let items = [];

// Create
function createItem(item) {
  items.push(item);
}

// Read
function readItems() {
  return items;
}

// Update
function updateItem(index, newItem) {
  items[index] = newItem;
}

// Delete
function deleteItem(index) {
  items.splice(index, 1);
}

// Example Usage
createItem('Book');
console.log(readItems()); // ['Book']
updateItem(0, 'Magazine');
console.log(readItems()); // ['Magazine']
deleteItem(0);
console.log(readItems()); // []
```

In diesem Beispiel:

- Erstellen wir ein Element, indem wir es in ein Array einfügen.

- Lesen wir die Elemente, indem wir das Array zurückgeben.

- Aktualisieren wir ein Element, indem wir das Array-Element an einem bestimmten Index ändern.

- Löschen wir ein Element, indem wir es mit `splice()` aus dem Array entfernen.

Dies ist eine grundlegende Darstellung, wie CRUD-Operationen auf konzeptioneller Ebene funktionieren.
