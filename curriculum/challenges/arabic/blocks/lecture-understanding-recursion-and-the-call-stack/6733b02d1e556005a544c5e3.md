# --interactive--

التكرار ميزة معقدة تتيح لك استدعاء دالة بشكل متكرر حتى تصل إلى حالة أساسية. على عكس الحلقة التقليدية، يسمح التكرار بالتعامل مع شيء ذو عمق غير معروف، مثل الكائنات/المصفوفات المسكنة بعمق، أو شجرة الملفات. لكن يمكنك أيضًا استخدامه لمهام أبسط، مثل العد التنازلي من رقم معين.

لننشئ دالة تفعل ذلك بالضبط. سنسمي دالتنا `recursiveCountdown`، ويجب أن تقبل رقمًا. سنجعلها تطبع هذا الرقم في وحدة التحكم:

:::interactive_editor

```js
const recursiveCountdown = (number) => {
  console.log(number);
};

recursiveCountdown(5);
```

:::

الآن إذا استدعينا هذه الدالة ومررنا الرقم 5، سنرى الرقم يُطبع في الطرفية. لكن لا يحدث شيء آخر – والرقم 5 بالتأكيد ليس عدًا تنازليًا.

قبل أن نبدأ ببناء الجزء التكراري من دالتنا، نحتاج أولًا إلى تحديد الحالة الأساسية. إذا لم تحدد حالة أساسية، ستستمر الشفرة في العمل حتى تتجاوز تخصيص الذاكرة وتنهار.

:::interactive_editor

```js
const recursiveCountdown = (number) => {
  if (number < 1) {
    return;
  }
  console.log(number);
};

recursiveCountdown(5);
```

:::

لحالتنا الأساسية، نريد أن يتوقف العد التنازلي إذا كان الرقم أقل من 1. عندما نصل إلى تلك الحالة الأساسية، يمكننا استخدام return للخروج من تنفيذ الدالة.

الآن بعد أن أعددنا الحالة الأساسية بأمان، يمكننا إعداد التكرار. النقطة الأساسية التي تجعل الدالة تكرارية هي أنها تستدعي نفسها أثناء التنفيذ. في هذه الحالة، نريد استدعاء الدالة بعد طباعة الرقم. لكن لكي نعد تنازليًا، يجب أن يكون رقمنا الجديد أقل بواحد:

:::interactive_editor

```js
const recursiveCountdown = (number) => {
  if (number < 1) {
    return;
  }
  console.log(number);
  recursiveCountdown(number - 1);
};

recursiveCountdown(5); 
```

:::

هذا سيطبع الأرقام 5، 4، 3، 2، و1 في وحدة التحكم.

لقد حصلنا على أرقامنا الخمسة! لكن ماذا لو أردنا العد تصاعديًا بدلًا من ذلك؟ بدلًا من كتابة دالة جديدة بالكامل، يمكننا تبديل ترتيب الطباعة والاستدعاء التكراري:

:::interactive_editor

```js
const recursiveCountdown = (number) => {
    if (number < 1) {
        return;
    }
    recursiveCountdown(number - 1);
    console.log(number);
  };

recursiveCountdown(5);
```

:::

هذا سيطبع الأرقام 1، 2، 3، 4، و5 في وحدة التحكم.

لكن لماذا يعمل ذلك؟ لفهم هذا تحتاج إلى فهم مكدس الاستدعاء. مكدس الاستدعاء هو كيف يتتبع JavaScript ويحل استدعاءات الدوال. يعمل المكدس كطابور من نوع آخر: آخر ما يدخل هو أول من يخرج. لفهم هذا بشكل أفضل، دعنا نضيف بعض الطباعة إلى دالتنا:

:::interactive_editor

```js
const recursiveCountdown = (number) => {
  console.log(`Function execution started for number: ${number}`);
  if (number < 1) {
    console.log(`Base case reached, begin resolving stack`);
    return;
  }
  console.log(`Calling recursiveCountdown with number: ${number - 1}`);
  recursiveCountdown(number - 1);
  console.log(`Function execution completed for number: ${number}`);
};

recursiveCountdown(5);
```

:::

أضفنا هنا أربع عبارات رئيسية. الطباعة الأولى تحدث عند بدء تنفيذ استدعاء الدالة. الطباعة الثالثة تحدث قبل استدعاء الدالة التكرارية مباشرة. والطباعة الرابعة تحدث عند انتهاء تنفيذ الدالة. النتيجة هي:

```md
Function execution started for number: 5
Calling recursiveCountdown with number: 4
Function execution started for number: 4
Calling recursiveCountdown with number: 3
Function execution started for number: 3
Calling recursiveCountdown with number: 2
Function execution started for number: 2
Calling recursiveCountdown with number: 1
Function execution started for number: 1
Calling recursiveCountdown with number: 0
Function execution started for number: 0
Base case reached, begin resolving stack
Function execution completed for number: 1
Function execution completed for number: 2
Function execution completed for number: 3
Function execution completed for number: 4
Function execution completed for number: 5
```

لكن كيف يحدث هذا؟ هنا يأتي دور مكدس الاستدعاء. عندما نستدعي `recursiveCountdown(5)`، يُضاف هذا الاستدعاء إلى مكدس الاستدعاء.

عندما يصل هذا الاستدعاء إلى النقطة التي يحتاج فيها لاستدعاء `recursiveCountdown(4)`، يجب أن يتوقف وينتظر النتيجة. في هذه الأثناء، يُضاف `recursiveCountdown(4)` إلى مكدس الاستدعاء، فوق `recursiveCountdown(5)`.

عندما يصل هذا الاستدعاء إلى النقطة التي يحتاج فيها لاستدعاء `recursiveCountdown(3)`، يجب أن يتوقف وينتظر النتيجة. في هذه الأثناء، يُضاف `recursiveCountdown(3)` إلى مكدس الاستدعاء، فوق `recursiveCountdown(4)`.

عندما يصل هذا الاستدعاء إلى النقطة التي يحتاج فيها لاستدعاء `recursiveCountdown(2)`، يجب أن يتوقف وينتظر النتيجة. في هذه الأثناء، يُضاف `recursiveCountdown(2)` إلى مكدس الاستدعاء، فوق `recursiveCountdown(3)`.

عندما يصل هذا الاستدعاء إلى النقطة التي يحتاج فيها لاستدعاء `recursiveCountdown(1)`، يجب أن يتوقف وينتظر النتيجة. في هذه الأثناء، يُضاف `recursiveCountdown(1)` إلى مكدس الاستدعاء، فوق `recursiveCountdown(2)`.

وأخيرًا، عندما يصل هذا الاستدعاء إلى النقطة التي يحتاج فيها لاستدعاء `recursiveCountdown(0)`، يجب أن يتوقف وينتظر النتيجة. في هذه الأثناء، يُضاف `recursiveCountdown(0)` إلى مكدس الاستدعاء، فوق `recursiveCountdown(1)`.

لكن `recursiveCountdown(0)` لا يستدعي دالة أخرى – بل يصل إلى الحالة الأساسية، حيث يرجع مبكرًا. وبما أن تنفيذ تلك الدالة انتهى، يمكن اعتبار استدعاء الدالة "محلولًا". وعندما يُحل الاستدعاء، يُزال من المكدس.

الآن لم يعد `recursiveCountdown(1)` ينتظر ذلك الاستدعاء – فهو في أعلى المكدس ويمكنه استئناف التنفيذ.

`recursiveCountdown(1)` يُحل، ويُزال من المكدس، ويسمح لـ `recursiveCountdown(2)` باستئناف التنفيذ.

`recursiveCountdown(2)` يُحل، ويُزال من المكدس، ويسمح لـ `recursiveCountdown(3)` باستئناف التنفيذ.

`recursiveCountdown(3)` يُحل، ويُزال من المكدس، ويسمح لـ `recursiveCountdown(4)` باستئناف التنفيذ.

`recursiveCountdown(4)` يُحل، ويُزال من المكدس، ويسمح لـ `recursiveCountdown(5)` باستئناف التنفيذ.

و`recursiveCountdown(5)` يُحل ويُزال من المكدس. الآن مكدس الاستدعاء فارغ، لذا اكتمل التكرار!

هذه نظرة عامة أساسية على كيفية عمل التكرار في JavaScript. إنه مفهوم معقد، ويجب أن تجرب بعض الشفرات وعبارات الطباعة حتى تشعر بالراحة مع سلوك مكدس الاستدعاء.

لمعلومة ممتعة، تحدثنا عن كيف أن غياب الحالة الأساسية يمكن أن يسبب تعطل الشفرة عند نفاد الذاكرة. هذا لأن التكرار يستمر في تكديس المزيد والمزيد من استدعاءات الدوال في مكدس الاستدعاء حتى يحدث تجاوز السعة. تمامًا مثل اسم مجتمع البرمجة الشهير.
