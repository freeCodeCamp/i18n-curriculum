# --interactive--

CSSはウェブページに適用する方法が主に3つあります：インライン、内部、外部です。

それぞれの方法には使いどころ、利点、制限があり、どの方法をいつ使うかを知ることは、きれいで効率的、かつ保守しやすいコードを書くために重要です。

3種類のCSSの違いと使うべきタイミングを見ていきましょう。

インラインCSSはHTML要素の中に直接`style`属性を使って書きます。特定の要素にスタイルを適用します。

インラインCSSの例はこちらです：

:::interactive_editor

```html
<p style="color: green;">This is an inline-styled paragraph.</p>
```

:::

この例では、`style`属性を使って段落のテキスト色を`green`に設定しています。

インラインCSSは、素早く一時的なスタイルを適用したり、特定の要素のスタイルを上書きしたりする場合に使われます。

しかし、HTMLが散らかりやすくなり、コードの保守が難しくなるため、ほとんどの場合は避けるべきです。

多くの場合、スタイルを整理して保守しやすくするために、内部CSSか外部CSSを使うほうが良いです。

内部CSSはHTMLドキュメントの`style`セクション内の`head`タグの中に書きます。ページ全体にスタイルを適用し、単一のドキュメントをスタイリングしたいときに便利です。

内部CSSの例はこちらです：

:::interactive_editor

```html
<head>
  <style>
    p {
      color: blue;
    }
  </style>
</head>
<body>
  <p>This paragraph is styled using internal CSS.</p>
</body>
```

:::

上の例では、内部CSSがページ内のすべての`p`要素に青色のテキストを適用しています。

内部CSSは複数ページにまたがらず、特定のページだけにスタイルを適用したい場合に最適です。単一ページのウェブサイトや、他で再利用しないスタイルに便利です。

ただし、複数ページでの再利用性が低いことや、インラインCSSと同様にHTMLとCSSが混ざるため、大規模なプロジェクトでは保守が難しくなるという欠点もあります。

外部CSSは別の`.css`ファイルに書かれ、`link`セクションの`head`要素でHTMLドキュメントにリンクされます。

複数ページにわたって一貫したスタイルを適用でき、プロのウェブ開発では推奨される方法です。

すべての段落要素にスタイルを適用する例はこちらです：

:::interactive_editor

```html
<head>
  <link rel="stylesheet" href="styles.css">
</head>
<body>
  <p>This paragraph is styled using external CSS.</p>
</body>
```

```css
p {
  color: red;
}
```

:::

前のレッスンで学んだように、`link`要素には現在のドキュメントとリンクされたリソースの関係を指定する`rel`属性があり、スタイルシートや外部リソースへのリンクに使われます。

一方、`href`属性はリンクされたリソースのURLを指定し、どこからリソースを取得するかを示します。

外部CSSは複数ページにわたって一貫したスタイルを維持したい大規模なプロジェクトに最適です。

HTMLが構造を担当し、CSSがスタイルを担当するという関心の分離を促進し、コードの保守性と拡張性を高めます。

どのタイプのCSSをいつ使うかを理解することは、効率的で効果的なウェブ開発に不可欠です。

ほとんどの場合、特に大規模で複雑なプロジェクトでは、外部CSSを使うことをお勧めします。
