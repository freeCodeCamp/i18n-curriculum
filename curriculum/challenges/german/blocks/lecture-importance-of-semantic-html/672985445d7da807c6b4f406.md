# --interactive--

Der wichtigste Aspekt beim Erstellen einer strukturellen Hierarchie ist die korrekte Verwendung von Überschriftselementen. Überschriftselemente sind nummeriert als `h1`, `h2`, `h3` und so weiter. Diese Zahlen geben die Überschriftenebene für dieses Element an.

Ähnlich wie bei einem Textdokument sollten Sie Überschriftenebenen nicht in der falschen Reihenfolge verwenden. Ihr `h1`-Element ist Ihre Überschrift auf oberster Ebene. Sie werden selten mehr als eine davon auf einer Seite haben, und sie sollte normalerweise vor all Ihren Inhalten stehen.

Ihr `h2`-Element ist Ihre Unterüberschrift. Es sollte immer nach Ihrem `h1` kommen und kann nach etwas einleitendem Text stehen. Im Gegensatz zu einem `h1`-Element können Sie mehrere `h2`-Elemente auf Ihrer Seite haben. Dies tun Sie oft, um verschiedene Inhaltsabschnitte voneinander abzugrenzen.

Dem Muster folgend sollte Ihr `h3`-Element immer nach einem `h2`-Element kommen. Das heißt, Sie sollten niemals direkt von `h1` zu `h3` springen. Sie können jedoch mehrere Überschriftselemente auf derselben Ebene haben. Zum Beispiel ist dieser Code korrekt:

```html
<section>
  <h1>freeCodeCamp</h1>
  <h2>Learn Front-End Development</h2>
  <h2>Learn Back-End Development</h2>
</section>
```

Aber dieser Code wäre nicht korrekt, weil `h3` vor `h2` kommt.

Verschieben Sie das `h2`-Element über das `h3`, damit es semantisch korrekt ist. Um mit dem Code zu interagieren, müssen Sie den interaktiven Editor aktivieren.

:::interactive_editor

```html
<section>
  <h1>freeCodeCamp</h1>
  <h3>Introduction to HTML</h3>
  <h2>Learn Front-End Development</h2>
</section>
```

:::

Es mag verlockend sein, ein bestimmtes Überschriftselement wegen seines Stils zu verwenden, wie zum Beispiel `h1` für großen Text:

:::interactive_editor

```html
<article>
  <p>
    Here is some
    <h1>Large Text</h1>
  </p>
</article>
```

:::

Stattdessen sollten Sie das passende Element für Ihre Dokumentstruktur auswählen und CSS verwenden, um das gewünschte Styling zu erreichen.

Die richtige Hierarchie ist wichtig für die Barrierefreiheit. Hilfstechnologien wie Screenreader verlassen sich auf die Struktur einer Webseite, um zu bestimmen, wie sie diese Webseite für den Nutzer vorlesen und ankündigen. Die Verwendung eines `h3`-Elements nach einem `h1` könnte dazu führen, dass ein Screenreader-Nutzer glaubt, er habe versehentlich wichtigen Inhalt übersprungen, weil ein `h2`-Element fehlt.

Eine korrekte Struktur ist auch wichtig für SEO. Suchmaschinen verwenden automatisierte Verfahren, um den Inhalt Ihrer Webseite zu analysieren und zu bestimmen, wann und wo sie in den Suchergebnissen angezeigt werden soll. Wenn Ihre Struktur fehlerhaft ist, können Suchmaschinen Sie möglicherweise nicht gut in den relevanten Suchergebnissen platzieren.

Schließlich kann Ihr HTML, je nachdem wie falsch Ihre Struktur ist, technisch gesehen sogar ungültig sein. In diesem Fall muss der Webbrowser erraten, was Sie eigentlich beabsichtigt haben. Und was er errät, entspricht möglicherweise überhaupt nicht Ihren Wünschen.

Wie Sie heute gelernt haben, gibt es viele Gründe, die richtige strukturelle Hierarchie für Ihre Seite zu verwenden. Behalten Sie das im Hinterkopf, wenn Sie neue Projekte erstellen.
