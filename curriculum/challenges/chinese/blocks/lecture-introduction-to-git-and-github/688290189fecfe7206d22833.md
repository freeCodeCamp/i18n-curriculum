# --description--

在之前的课程中，我们简要提到了 GitHub。GitHub 和 Gitlab 都是版本控件提供商。这意味着它们是基于云的方案，提供将版本控件项目存储在称为 `repositories` 的地方，并启用用于这些项目的协作特色。

在深入了解这些平台提供的特色之前，我们需要讨论不同类型的项目。许多项目被认为是“开源”的，这意味着人们可以查看你发布的代码，提出更改建议，报告问题，甚至运行修改后的版本。

但有时你可能只希望你的队友查看和处理代码。你也可以将一个项目发布为“闭源”，这意味着只有你显式授权的人才能查看和操作该项目。

无论你的项目采用哪种方法，像 GitHub 和 Gitlab 这样的网站都能大幅提升你团队的协作能力。

例如，正在开发某个特色的队友可能遇到困难并有疑问。他们无需将代码拷贝粘贴到 Discord 让你读取，或跳到通话中共享屏幕工作内容，而是可以直接将更改上传到 GitHub。之后，你可以在浏览器中查看更改，甚至下载更改以便局部调试和运行。

当队友完成一个特色时，他们可以通过这些网站提出更改。然后你可以审查这些更改，如果你有想法和问题，可以对特定的文件或行留下评论。你甚至可以提出建议的编辑，他们可以直接在浏览器中将这些编辑应用到更改中！

如果你对更改感到满意，可以接受它们并将其合并到你的主分支中。这意味着整个团队随后可以在本地下载最新的更改，以便在开始下一个任务时拥有最新的代码。

说到任务，这些网站还提供强大的问题跟踪工具。你可以使用这些工具来跟踪并为你的团队成员赋值工单，接受来自公众的错误报告，甚至通过诸如项目看板之类的功能规划完整的冲刺。

这些网站还提供许多其他特色。在接下来的几节课中，我们将介绍关键方面，以便你可以立即开始协作！
