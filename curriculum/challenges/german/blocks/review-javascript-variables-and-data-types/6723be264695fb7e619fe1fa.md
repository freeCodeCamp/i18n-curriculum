## Arbeiten mit HTML, CSS und JavaScript

Während HTML und CSS die Struktur einer Website bereitstellen, sorgt JavaScript für Interaktivität, indem es komplexe Funktionen ermöglicht, wie die Verarbeitung von Benutzereingaben, das Animieren von Elementen und sogar das Erstellen kompletter Webanwendungen.

## Datentypen in JavaScript

Datentypen helfen dem Programm zu verstehen, mit welcher Art von Daten es arbeitet, ob es sich um eine Zahl, Text oder etwas anderes handelt.

- **Number**: Eine Zahl stellt sowohl ganze Zahlen als auch Fließkommazahlen dar. Beispiele für ganze Zahlen sind 7, 19 und 90. 
- **Fließkommazahl**: Eine Fließkommazahl ist eine Zahl mit Dezimalpunkt. Beispiele sind 3,14, 0,5 und 0,0001.
- **String**: Ein String ist eine Folge von Zeichen oder Text, der in Anführungszeichen eingeschlossen ist. `"I like coding"` und `'JavaScript is fun'` sind Beispiele für Strings.
- **Boolean**: Ein Boolean stellt einen von zwei möglichen Werten dar: `true` oder `false`. Sie können einen Boolean verwenden, um eine Bedingung darzustellen, wie zum Beispiel `isLoggedIn = true`.
- **Undefined und Null**: Ein `undefined`-Wert ist eine Variable, die deklariert, aber nicht mit einem Wert belegt wurde. Ein `null`-Wert ist ein leerer Wert oder eine Variable, der absichtlich der Wert `null` zugewiesen wurde.
- **Object**: Ein Objekt ist eine Sammlung von Schlüssel-Wert-Paaren. Der Schlüssel ist der Name der Eigenschaft, und der Wert ist der Wert der Eigenschaft.

Hier hat das `pet`-Objekt drei Eigenschaften oder Schlüssel: `name`, `age` und `type`. Die Werte sind jeweils `Fluffy`, `3` und `dog`.

```js
let pet = {
  name: "Fluffy",
  age: 3,
  type: "dog"
};
```

- **Symbol**: Der Datentyp Symbol ist ein einzigartiger und unveränderlicher Wert, der als Bezeichner für Objekteigenschaften verwendet werden kann.

Im folgenden Beispiel werden zwei Symbole mit derselben Beschreibung erstellt, die jedoch nicht gleich sind.

:::interactive_editor

```js
const crypticKey1= Symbol("saltNpepper");
const crypticKey2= Symbol("saltNpepper");
console.log(crypticKey1 === crypticKey2); // false
```

:::

- **BigInt**: Wenn die Zahl zu groß für den Datentyp `Number` ist, können Sie den Datentyp BigInt verwenden, um ganze Zahlen beliebiger Länge darzustellen.

Indem Sie ein `n` an das Ende der Zahl anhängen, können Sie ein BigInt erstellen.

```js
const veryBigNumber = 1234567890123456789012345678901234567890n;
```

## Variablen in JavaScript

- Variablen können mit dem Schlüsselwort `let` deklariert werden. 

```js
let cityName;
```

- Um einer Variable einen Wert zuzuweisen, können Sie den Zuweisungsoperator `=` verwenden.

```js
cityName = "New York";
```

- Variablen, die mit `let` deklariert wurden, können einen neuen Wert zugewiesen bekommen.

:::interactive_editor

```js
let cityName = "New York";
cityName = "Los Angeles";
console.log(cityName); // Los Angeles
```

:::

- Neben `let` können Sie auch `const` verwenden, um eine Variable zu deklarieren. Allerdings kann einer `const`-Variablen kein neuer Wert zugewiesen werden.

```js
const cityName = "New York";
cityName = "Los Angeles"; // TypeError: Assignment to constant variable.
```

- Variablen, die mit `const` deklariert wurden, werden verwendet, um Konstanten zu deklarieren, die im Code nicht geändert werden dürfen, wie `PI` oder `MAX_SIZE`.

## Konventionen zur Benennung von Variablen

- Variablennamen sollten beschreibend und aussagekräftig sein.
- Variablennamen sollten CamelCase sein, wie `cityName`, `isLoggedIn` und `veryBigNumber`.
- Variablennamen dürfen nicht mit einer Zahl beginnen. Sie müssen mit einem Buchstaben, `_` oder `$` anfangen.
- Variablennamen dürfen keine Leerzeichen oder Sonderzeichen enthalten, außer `_` und `$`.
- Variablennamen dürfen keine reservierten Schlüsselwörter sein.
- Variablennamen sind case-sensitiv. `age` und `Age` sind unterschiedliche Variablen.

## Strings und die Unveränderlichkeit von Strings in JavaScript

- Strings sind Zeichenfolgen, die in Anführungszeichen eingeschlossen sind. Sie können mit einfachen oder doppelten Anführungszeichen erstellt werden.

```js
let correctWay = 'This is a string';
let alsoCorrect = "This is also a string";
```

- Strings sind in JavaScript unveränderlich. Das bedeutet, dass Sie, sobald ein String erstellt wurde, die Zeichen im String nicht ändern können. Sie können jedoch Strings neu zuweisen.

```js
let firstName = "John";
firstName = "Jane"; // Reassigning the string to a new value
```

## String-Verkettung in JavaScript

- Verkettung ist der Prozess, mehrere Strings zu verbinden oder Strings mit Variablen zu kombinieren, die Text enthalten. Der `+`-Operator ist eine der einfachsten und am häufigsten verwendeten Methoden, um Strings zu verketten.

:::interactive_editor

```js
let studentName = "Asad";
let studentAge = 25;
let studentInfo = studentName + " is " + studentAge + " years old.";
console.log(studentInfo); // Asad is 25 years old.
```

:::

- Wenn Sie einen bestehenden String erweitern oder anhängen möchten, können Sie den `+=`-Operator verwenden. Das ist hilfreich, wenn Sie einen String nach und nach mit mehr Text erweitern wollen.

:::interactive_editor

```js
let message = "Welcome to programming, ";
message += "Asad!";
console.log(message); // Welcome to programming, Asad!
```

:::

- Eine weitere Möglichkeit, Strings zu verketten, ist die Verwendung der `concat()`-Methode. Diese Methode verbindet zwei oder mehr Strings miteinander.

:::interactive_editor

```js
let firstName = "John";
let lastName = "Doe";
let fullName = firstName.concat(" ", lastName);
console.log(fullName); // John Doe
```

:::

## Nachrichten mit `console.log()` protokollieren

- Die `console.log()`-Methode wird verwendet, um Nachrichten in der Konsole zu protokollieren. Sie ist ein hilfreiches Werkzeug zum Debuggen und Testen Ihres Codes.

:::interactive_editor

```js
console.log("Hello, World!");
// Output: Hello, World!
```

## Semikolons in JavaScript

- Semikolons werden hauptsächlich verwendet, um das Ende einer Anweisung zu markieren. Das hilft der JavaScript-Engine, die einzelnen Anweisungen zu trennen, was für die korrekte Ausführung entscheidend ist.

```js
let message = "Hello, World!"; // first statement ends here
let number = 42; // second statement starts here
```

- Semikolons helfen, Mehrdeutigkeiten bei der Codeausführung zu vermeiden und stellen sicher, dass Anweisungen korrekt beendet werden.

## Kommentare in JavaScript

- Jede auskommentierte Codezeile wird von der JavaScript-Engine ignoriert. Kommentare werden verwendet, um Code zu erklären, Notizen zu machen oder Code vorübergehend zu deaktivieren.
- Einzeilige Kommentare werden mit `//` erstellt.

```js
// This is a single-line comment and will be ignored by the JavaScript engine
```

- Mehrzeilige Kommentare werden mit `/*` begonnen und mit `*/` beendet.

```js
/*
This is a multi-line comment.
It can span multiple lines.
*/
```

## JavaScript als dynamisch typisierte Sprache

- JavaScript ist eine dynamisch typisierte Sprache, was bedeutet, dass Sie den Datentyp einer Variable bei der Deklaration nicht angeben müssen. Die JavaScript-Engine bestimmt den Datentyp automatisch anhand des zugewiesenen Werts.

```js
let error = 404; // JavaScript treats error as a number
error = "Not Found"; // JavaScript now treats error as a string
```

- Andere Sprachen wie C#, die nicht dynamisch typisiert sind, würden einen Fehler verursachen:

```csharp
int error = 404; // value must always be an integer
error = "Not Found"; // This would cause an error in C#
```

## Verwendung des `typeof`-Operators

- Der `typeof`-Operator wird verwendet, um den Datentyp einer Variable zu überprüfen. Er gibt einen String zurück, der den Typ der Variable angibt.

:::interactive_editor

```js
let age = 25;
console.log(typeof age); // "number"

let isLoggedIn = true;
console.log(typeof isLoggedIn); // "boolean"
```

:::

- Es gibt jedoch eine bekannte Eigenheit in JavaScript bezüglich `null`. Der `typeof`-Operator gibt für `"object"`-Werte `null` zurück.

:::interactive_editor

```js
let user = null;
console.log(typeof user); // "object"
```

:::
