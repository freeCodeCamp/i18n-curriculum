## 관계형 데이터베이스 소개

- **관계형 데이터베이스**: 데이터를 행과 열로 구성된 관련 표로 조직합니다. 각 행은 레코드를 나타내고, 각 열은 데이터의 속성을 나타냅니다.
- **관계형 데이터베이스의 장점**: 확장 가능하며, 의료, 비즈니스, 게임 등 다양한 분야에 널리 적용되고, 신뢰할 수 있는 데이터를 유지하도록 구조화되어 있습니다.
- **일반적인 사용 사례**: 웹 개발, 재고 시스템, 전자상거래, 의료, 기업용 애플리케이션 등입니다.

## 주요 개념

- **스키마**: 관계형 데이터베이스는 구조를 정의하는 스키마가 필요합니다—표, 열, 데이터 유형, 제약 조건, 관계 등이 포함됩니다.
- **기본 키**: 표의 각 행을 고유하게 식별하는 키입니다. 데이터 무결성에 필수적이며, 외래 키를 통해 표 간 레코드를 연결하는 데 사용됩니다.
- **외래 키**: 다른 표의 기본 키를 참조하여 관련 데이터를 연결하는 데 사용됩니다.
- **관계**: 기본 키와 외래 키를 통해 표를 연결하여 표준화된 데이터를 구조화하고 의미 있는 쿼리를 수행할 수 있습니다.
- **엔터티 관계 다이어그램(ERD)**: 데이터베이스 스키마에서 엔터티(표) 간의 관계를 시각화합니다.
- **데이터 무결성**: 키와 데이터 유형을 사용해 강제하며, 저장된 데이터의 일관성과 정확성을 보장합니다.

## SQL 기본

- **쿼리**: 데이터베이스에서 특정 데이터를 검색하는 요청입니다.

```sql
SELECT * FROM dogs WHERE age < 3;
```

- **WHERE 절**: 조건에 따라 결과를 필터링합니다. `<`, `=`, `>` 등의 비교 연산자를 사용하세요.
- **ORDER BY와 함께 선택**: 열을 기준으로 결과를 검색하고 정렬합니다.

```sql
SELECT columns FROM table_name ORDER BY column_name;
```

## 표 작업

- **`CREATE TABLE` 문**: 데이터베이스에 새 표를 생성하는 데 사용합니다.

```sql
CREATE TABLE first_table();
```

- **`ALTER TABLE ADD COLUMN` 문**: 기존 표에 열을 추가하는 데 사용합니다.

```sql
ALTER TABLE table_name ADD COLUMN column_name DATATYPE;
```

- **`ALTER TABLE DROP COLUMN` 문**: 기존 표에서 열을 삭제하는 데 사용합니다.

```sql
ALTER TABLE table_name DROP COLUMN column_name;
```

- **`ALTER TABLE RENAME COLUMN` 문**: 표의 열 이름을 변경하는 데 사용합니다.

```sql
ALTER TABLE table_name RENAME COLUMN column_name TO new_name;
```

- **`DROP TABLE` 문**: 데이터베이스에서 전체 표를 삭제하는 데 사용합니다.

```sql
DROP TABLE table_name;
```

- **`ALTER DATABASE RENAME` 문**: 데이터베이스 이름을 변경하는 데 사용합니다.

```sql
ALTER DATABASE database_name RENAME TO new_database_name;
```

- **`DROP DATABASE` 문**: 전체 데이터베이스를 삭제하는 데 사용합니다.

```sql
DROP DATABASE database_name;
```

## 제약 조건 및 데이터 무결성

- **`ALTER TABLE ADD COLUMN`와 제약 조건**: 기존 표에 제약 조건이 있는 열을 추가하는 데 사용합니다.

```sql
ALTER TABLE table_name ADD COLUMN column_name DATATYPE CONSTRAINT;
```

- **`NOT NULL` 제약 조건**: 열에 NULL 값을 허용하지 않도록 보장합니다.

```sql
column_name VARCHAR(50) NOT NULL
```

- **`ALTER TABLE ADD PRIMARY KEY` 문**: 표에 기본 키 제약 조건을 추가하는 데 사용합니다.

```sql
ALTER TABLE table_name ADD PRIMARY KEY(column_name);
```

- **`ALTER TABLE DROP CONSTRAINT` 문**: 표에서 제약 조건을 삭제하는 데 사용합니다.

```sql
ALTER TABLE table_name DROP CONSTRAINT constraint_name;
```

- **`ALTER TABLE ADD COLUMN`와 외래 키**: 다른 표를 참조하는 외래 키 열을 추가하는 데 사용합니다.

```sql
ALTER TABLE table_name ADD COLUMN column_name DATATYPE REFERENCES referenced_table_name(referenced_column_name);
```

- **`ALTER TABLE ADD UNIQUE` 문**: 열에 UNIQUE 제약 조건을 추가하는 데 사용합니다.

```sql
ALTER TABLE table_name ADD UNIQUE(column_name);
```

- **`ALTER TABLE ALTER COLUMN SET NOT NULL` 문**: 기존 열에 NOT NULL 제약 조건을 설정하는 데 사용합니다.

```sql
ALTER TABLE table_name ALTER COLUMN column_name SET NOT NULL;
```

- **NULL 값이 포함된 `INSERT` 문**: 표에 NULL 값을 삽입하는 방법을 보여줍니다.

```sql
INSERT INTO table_name(column_a) VALUES(NULL);
-- or
INSERT INTO table_name(column_b) VALUES('value'); -- if column_a allows nulls
```

- **복합 기본 키**: 여러 열로 구성된 기본 키를 정의하는 제약 조건입니다.

```sql
CREATE TABLE course_enrollments (
    student_id INT,
    course_id INT,
    PRIMARY KEY (student_id, course_id)
);
```

## 데이터 조작 (CRUD)

- **`INSERT` 문**: 표에 단일 행을 삽입하는 데 사용합니다.

```sql
INSERT INTO table_name(column_1, column_2) VALUES(value1, value2);
```

- **열 이름 생략이 포함된 `INSERT` 문**: 열 이름을 명시하지 않고 표의 기본 열 순서에 따라 값을 삽입하는 방법을 보여줍니다.

```sql
INSERT INTO dogs VALUES ('Gino', 3);
```

- **여러 행이 포함된 `INSERT` 문**: 한 번에 여러 행을 표에 삽입하는 데 사용합니다.

```sql
INSERT INTO dogs (name, age) VALUES 
('Gino', 3),
('Nora', 2);
```

- **`UPDATE` 문**: 조건에 따라 표의 기존 데이터를 업데이트하는 데 사용합니다.

```sql
UPDATE table_name SET column_name=new_value WHERE condition;
```

- **`DELETE` 문**: 조건에 따라 표에서 행을 삭제하는 데 사용합니다.

```sql
DELETE FROM table_name WHERE condition;
```

## 데이터 유형

- **`NUMERIC` 데이터 유형**: 지정된 정밀도와 소수 자릿수를 가진 정확한 십진수를 저장하는 데 사용합니다.

```sql
price NUMERIC(10, 2)
```

- **`TEXT` 데이터 유형**: 길이 제한이 없는 가변 길이 문자열을 저장하는 데 사용합니다.

```sql
column_name TEXT
```

- **`INTEGER` 데이터 유형**: 소수점 없는 정수를 저장하는 데 사용합니다.

```sql
units_sold INTEGER
```

- **`SMALLINT` 및 `BIGINT` 데이터 유형**: 각각 더 작거나 더 큰 범위를 가진 INTEGER 변형입니다.

- **`SERIAL` 데이터 유형**: PostgreSQL에서 자동 증가하는 정수 열을 생성하는 데 사용합니다.

```sql
id SERIAL
```

- **`AUTO_INCREMENT` 속성**: MySQL에서 자동 증가하는 정수 열을 생성하는 데 사용합니다.

```sql
id INT AUTO_INCREMENT
```

- **`VARCHAR` 데이터 유형**: 최대 길이가 지정된 가변 길이 문자열을 저장하는 데 사용합니다.

```sql
name VARCHAR(50)
```

- **`DATE` 데이터 유형**: 날짜 값(년, 월, 일)을 저장하는 데 사용합니다.

```sql
event_date DATE
```

- **`TIME` 데이터 유형**: 시간 값(시, 분, 초)을 저장하는 데 사용합니다.

```sql
start_time TIME
```

- **`TIMESTAMP` 데이터 유형**: 날짜와 시간 값을 저장하며, 선택적으로 시간대 정보를 포함할 수 있습니다.

```sql
event_timestamp TIMESTAMP
event_timestamp TIMESTAMP WITH TIME ZONE
```

- **`BOOLEAN` 데이터 유형**: 참/거짓 값을 저장하는 데 사용합니다.

```sql
is_active BOOLEAN
```

## 데이터베이스 관계

- **관계 유형**: 관계형 데이터베이스에서 표가 서로 연결되는 다양한 방식입니다.
  - 일대일
  - 일대다
  - 다대일
  - 다대다
  - 자기 참조(재귀)

- **일대일 관계**: 한 표의 각 레코드가 다른 표의 정확히 한 레코드와 대응하는 관계입니다.

```md
One employee is assigned exactly one vehicle.
Tables: employees, vehicles
```

- **일대다 관계**: 한 표의 한 레코드가 다른 표의 여러 레코드와 연결될 수 있는 관계입니다.

```md
One customer can have many orders.
Tables: customers → orders
```

- **조인 표를 통한 다대다 관계**: 두 관련 표의 외래 키를 포함하는 조인 표를 사용해 구현합니다.

```sql
CREATE TABLE books_authors (
    author_id INT REFERENCES authors(id),
    book_id INT REFERENCES books(id)
);
```

- **자기 참조 관계**: 표가 자기 자신을 참조하여 계층 구조를 만드는 관계입니다.

```md
An employee table where each employee may report to another employee.
```

## 고급 SQL (조인)

- **`INNER JOIN` 문**: 두 표 모두에서 일치하는 값이 있는 행만 반환하는 조인입니다.

```sql
SELECT *
FROM products
INNER JOIN sales ON products.product_id = sales.product_id;
```

- **`FULL OUTER JOIN` 문**: 두 표의 모든 행을 반환하며, 어느 한쪽 표에 일치하지 않는 행도 포함합니다.

```sql
SELECT *
FROM products
FULL OUTER JOIN sales ON products.product_id = sales.product_id;
```

- **`LEFT OUTER JOIN` 문**: 왼쪽 표의 모든 행과 오른쪽 표의 일치하는 행을 반환합니다.

```sql
SELECT *
FROM products
LEFT JOIN sales ON products.product_id = sales.product_id;
```

- **`RIGHT OUTER JOIN` 문**: 오른쪽 표의 모든 행과 왼쪽 표의 일치하는 행을 반환합니다.

```sql
SELECT *
FROM products
RIGHT JOIN sales ON products.product_id = sales.product_id;
```

- **`SELF JOIN` 문**: 같은 표 내에서 행을 비교하기 위해 표를 자기 자신과 조인하는 데 사용합니다.

```sql
SELECT A.column_name, B.column_name
FROM table_name A
JOIN table_name B ON A.related_column = B.related_column;
```

- **`CROSS JOIN` 문**: 두 표의 카르테시안 곱을 반환하여 첫 번째 표의 모든 행과 두 번째 표의 모든 행을 조합합니다.

```sql
SELECT *
FROM table1
CROSS JOIN table2;
```

## PostgreSQL 전용 명령어

- **`psql` 로그인 명령어**: 특정 사용자 이름과 데이터베이스로 PostgreSQL에 로그인하는 데 사용합니다.

```bash
psql --username=freecodecamp --dbname=postgres
```

- **`\l` 명령어**: PostgreSQL 인스턴스의 모든 데이터베이스를 나열합니다.

```sql
\l
```

- **`CREATE DATABASE` 및 `\c` 명령어**: 새 데이터베이스를 생성하고 연결하는 데 사용합니다.

```sql
CREATE DATABASE database_name;
\c database_name
```

- **`\d` 명령어**: 현재 데이터베이스의 모든 표를 나열합니다.

```sql
\d
```

- **`\d table_name` 명령어**: 특정 표의 스키마/구조를 표시합니다.

```sql
\d table_name
```

- **`\q` 명령어**: PostgreSQL 클라이언트를 종료합니다.

```bash
\q
```

## 관계형과 비관계형

- **비관계형(NoSQL) 데이터베이스**: 비정형 또는 반정형 데이터를 저장합니다. 엄격한 스키마가 필요 없으며, 진화하는 데이터 모델에 더 유연합니다.
- **관계형과 비관계형 선택**: 데이터와 애플리케이션 요구 사항의 특성에 따라 결정합니다.
- **관계형 대 비관계형**: 구조화된 데이터와 일관성이 필요하면 관계형을, 유연성과 빠른 변화가 필요하면 NoSQL을 선택하세요.

## 인기 있는 RDBMS 시스템

- **MySQL**: 오픈 소스, 신뢰성 높고, 웹 개발에 널리 사용되며, 큰 커뮤니티가 지원합니다.
- **PostgreSQL**: 오픈 소스, 고급 기능과 확장성을 갖추었으며, 사용자 정의 데이터 유형과 서버 측 프로그래밍을 지원합니다.
- **SQLite**: 경량, 파일 기반, 서버리스로 작은 애플리케이션에 적합합니다.
- **Microsoft SQL Server**: 상용, 기업용 데이터베이스입니다.
- **Oracle Database**: 대규모 성능과 확장성으로 알려진 상용 관계형 데이터베이스입니다.
