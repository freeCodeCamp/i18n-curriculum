# --description--

你已經學會如何建立提交並將它們推送到你所控制的自己的儲存庫。你也可以對你不維護的其他專案執行相同操作，但需要一些額外步驟。

首先，通常你不會對別人的專案擁有「寫入存取權」。這表示你無法直接推送提交到他們的專案。相反地，你需要創建該儲存庫的副本來進行作業。這個過程稱為「forking」。

要分叉一個儲存庫，請造訪你想要貢獻的 GitHub 儲存庫。在頂部，你會看到一個「Fork」按鈕。點擊此按鈕即可進入建立分叉的使用者介面。

你會看到一個類似於你創建自己儲存庫時的螢幕，但選項較少。你可以選擇一個擁有者，無論是你的個人帳戶或你有存取權的組織，為 fork 選擇一個新名稱，以及一個新的描述。

你也有一個選項，可以只複製 `main` 分支（或儲存庫組態的預設分支）。這通常是個好主意，除非你需要對特定的功能分支做出貢獻。你將在未來的課程中學習更多關於分支的知識。現在，請保持該方框勾選。

點擊「Create fork」，你的新儲存庫將會被創建，並且你會自動被導向該儲存庫。

接著，像操作你自己的儲存庫一樣，使用 SSH 或 GitHub CLI 將它複製到你的區域電腦。

現在，讓我們來看看你的區域複製所組態的遠端。打開終端機，進入區域的儲存庫目錄，並執行 `git remote -v` 以取得完整列表，應該會長這樣：

```sh
origin git@github.com:your-username/your-repo-name.git (fetch)
origin git@github.com:your-username/your-repo-name.git (pull)
```

透過從 GitHub 複製一個儲存庫，Git 已自動將你的分支設定為 `origin` 遠端。然而，當你從分支工作時，你經常需要更新你的副本以引入來自原始儲存庫的變更。

為此，讓我們將原始的 repository 添加為另一個 remote。回到你 fork 該 repository 的頁面，點擊「Code」按鈕，並取得 SSH URL。然後執行指令來創建一個新的 upstream remote：

```sh
git remote add upstream git@github.com:original-repo-username/repo-name.git
```

`upstream` 是遠端的名稱－你可以將它命名為其他名稱，但 `upstream` 通常是慣例。如果你再次查看你的遠端，你會看到你新的 upstream 遠端：

```sh
origin git@github.com:your-username/your-repo-name.git (fetch)
origin git@github.com:your-username/your-repo-name.git (pull)
upstream git@github.com:original-repo-username/your-repo-name.git (fetch)
upstream git@github.com:original-repo-username/your-repo-name.git (pull)
```

使用 `upstream` 遠端，你將能夠執行像是 `git fetch upstream` 和 `git merge upstream/main` 這類操作，將原始專案的最新變更拉取到你的區域分支。

分支仍然是一個儲存庫，你以相同的方式創建提交並推送更改。然而，在深入了解之前，你應該明白在對別人的專案進行更改時，有某些策略是你應該遵循的。你將在接下來的課程中學習更多關於這些策略的內容。

將原始碼公開給大眾、允許你修改程式碼並接受程式碼貢獻的專案稱為「開放原始碼」專案。開放原始碼有許多利益，例如讓社群幫助你識別和修正錯誤、從他人對你工作的貢獻中學習，以及培養一個你可以轉化為網路的開發者社群。

當你對某人的專案進行更改時，你會被視為「開放原始碼貢獻者」。為開放原始碼專案做出貢獻，可以是體驗專業開發團隊工作方式的絕佳預覽。而這裡的一些策略將幫助你做到這一點，並延續到你的第一份有薪工作。

如果你對如何貢獻有任何疑問，大多數開源專案都會有貢獻指南，裡面包含你需要知道的所有概述。但如果你仍然不確定，開啟一個問題（issue）在儲存庫中詢問你的問題絕對沒有壞處－或者，如果他們在像 Discord 這樣的平台上有社群空間，加入並在那裡提出你的問題。
