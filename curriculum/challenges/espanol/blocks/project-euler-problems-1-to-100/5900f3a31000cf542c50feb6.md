# --description--

Si tomamos 47, invertimos y sumamos, 47 + 74 = 121, que es palindrómico.

No todos los números producen palíndromos tan rápido. Por ejemplo,

<div style="margin-left: 4em;">
  349 + 943 = 1292,<br>
  1292 + 2921 = 4213<br>
  4213 + 3124 = 7337<br>
</div>

Es decir, 349 tomó tres iteraciones para llegar a un palíndromo.

Aunque nadie lo ha probado aún, se piensa que algunos números, como 196, nunca producen un palíndromo. Un número que nunca forma un palíndromo mediante el proceso de invertir y sumar se llama número Lychrel. Debido a la naturaleza teórica de estos números, y para el propósito de este problema, asumiremos que un número es Lychrel hasta que se demuestre lo contrario. Además, se te da que para cada número menor de diez mil, este o bien (i) se convierte en un palíndromo en menos de cincuenta iteraciones, o (ii) nadie, con todo el poder computacional que existe, ha logrado hasta ahora mapearlo a un palíndromo. De hecho, 10677 es el primer número que se ha demostrado que requiere más de cincuenta iteraciones antes de producir un palíndromo: 4668731596684224866951378664 (53 iteraciones, 28 dígitos).

Sorprendentemente, hay números palindrómicos que son ellos mismos números Lychrel; el primer ejemplo es 4994.

¿Cuántos números Lychrel hay por debajo de `num`?

**Nota:** La redacción fue modificada ligeramente el 24 de abril de 2007 para enfatizar la naturaleza teórica de los números Lychrel.

# --solutions--

```js
const countLychrelNumbers = (size) => {
  const numReverse = (num) => {
    return Number(num.toString().split('').reverse().join(''));
  };
  const isPalin = (num) => {
    if (numReverse(num) === num) {
      return true;
    }
    return false;
  };
  let total = 0;
  for (let i = 1; i < size; i++) {
    let loopCount = 1;
    let sum = i;
    while (loopCount < 50) {
      sum = sum + numReverse(sum);
      if (isPalin(sum)) {
        break;
      } else {
        loopCount++;
      }
    }
    if (loopCount === 50) {
      total++;
    }
  }
  return total;
}
```
