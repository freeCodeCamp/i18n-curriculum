# --description--

Crypto è un altro modulo core integrato in Node.js. Include strumenti per cose come hashing, crittografia, decrittografia e creazione di firme digitali, tutti usati per proteggere informazioni sensibili e mantenere sicura la tua app.

Detto questo, `crypto` ti offre blocchi di costruzione a basso livello, non una sicurezza plug-and-play. Scrivere il proprio codice di crittografia o autenticazione può essere pericoloso se non stai attento. Nella maggior parte dei casi, è meglio usare librerie ben testate come `bcrypt` per l’hashing delle password o `jsonwebtoken` (JWT) per gestire login e token.

Detto ciò, è comunque utile capire come funzionano alcuni metodi del modulo crypto.

Per usare questi metodi, devi importare il modulo `crypto`:

```js
const crypto = require("crypto");
```

Alcuni metodi sono usati per trasformare dati, come quelli per l’hashing e la crittografia, mentre altri sono per la gestione di chiavi e sicurezza, come quelli per generare valori casuali e creare segreti.

Iniziamo dai metodi per l’hashing e la crittografia dei dati.

Il metodo `createHash()` crea un oggetto hash accettando algoritmi come `sha256`, `sha512` o `md5`. È un’operazione unidirezionale, quindi non puoi invertirla.

`createHash()` è utile per l’hashing delle password e per creare impronte digitali di file. Per usarlo, passi l’algoritmo, usi il metodo `update()` per fornire i dati e infine `digest()` con una codifica per ottenere il valore hash:

```js
const crypto = require("crypto");

const hashedPassword = crypto
  .createHash("sha256")
  .update("myStrongPassword")
  .digest("hex");

console.log("createHash result:", hashedPassword);
// createHash result: f92c9cfa0ead1bcec05ca75888a4074ba994ad237e5e2a8c7cc6a620378c061d
```

`createHmac()` fa quasi la stessa cosa di `createHash()`, ma porta le cose al livello successivo accettando una chiave segreta, così solo chi ha quella chiave può verificare l’hash. È ideale per l’autenticazione e per verificare l’integrità dei dati:

```js
const crypto = require("crypto");

const hashedMessage = crypto
  .createHmac("sha256", "secretkey")
  .update("important-secret-message")
  .digest("hex");

console.log("createHmac result:", hashedMessage);
// createHmac result: da48d6f026b6036286b1fb872c63264130d5cc4271f3a213bb6ddca5a023e77e
```

I metodi `createCipheriv()` e `createDecipheriv()` crittografano e decrittografano i dati. Entrambi accettano un algoritmo, una chiave e un `iv`, che è un blocco di dati casuali o unici usato all’inizio del processo di crittografia:

```js
createCipheriv(algorithm, key, iv);
createDecipheriv(algorithm, key, iv);
```

Per decrittografare i dati, la chiave deve essere la stessa, altrimenti la decrittografia fallirà e otterrai un errore o un insieme di caratteri illeggibili invece del messaggio originale.

Ecco i due in azione:

```js
const crypto = require("crypto");

// A key must match the algorithm length. Here AES-256 is 32 bytes
const key = Buffer.from("12345678901234567890123456789012");

// A fixed IV, 16 bytes for AES
const iv = Buffer.from("1234567890123456");

const cipher = crypto.createCipheriv("aes-256-cbc", key, iv);

let encrypted = cipher.update("Hello campers!", "utf8", "hex");
encrypted += cipher.final("hex");

console.log("Encrypted data:", encrypted);
// Encrypted data: 4ee93aa398ab44e3540e4a67ca96bc8c

// Decrypt the "Hello campers!" message
const decipher = crypto.createDecipheriv("aes-256-cbc", key, iv);
let decrypted = decipher.update(encrypted, "hex", "utf8");
decrypted += decipher.final("utf8");

console.log("Decrypted data:", decrypted);
// Decrypted data: Hello campers!
```

Un altro metodo crypto per la trasformazione dei dati è `sign()` e `verify()`.

`sign()` crea una firma digitale da alcuni dati usando una chiave privata. Questa firma dimostra che i dati provengono dal possessore della chiave privata e non sono stati manomessi.

`verify()` verifica poi quella firma, e fallisce se i dati o la firma non corrispondono.

Ora, vediamo i metodi crypto per generare valori casuali e creare segreti.

`randomBytes()` accetta un `size` e genera token crittograficamente sicuri. Questo lo rende adatto per generare UUID (ID univoci universali). Inoltre, è un buon sostituto di `Math.random()`, che non è sicuro per token e chiavi.

```js
console.log("Random Bytes:", crypto.randomBytes(16));
// Random Bytes: <Buffer 01 88 aa 1e 2c 38 48 39 26 e1 6b a9 d8 c5 ed 49>
```

L’output è un buffer per impostazione predefinita. Come hai imparato nella lezione sul modulo Buffer, puoi convertire quel buffer in una stringa con il metodo `toString()`:

```js
console.log("Random Bytes:", crypto.randomBytes(16).toString("hex"));
// Random Bytes: a6154ef5a296fa176ad0f332bd94d712
```

L’argomento `'hex'` in `toString('hex')` qui dice a Node di codificare i dati binari del buffer come stringa esadecimale.

Il metodo `randomInt()` accetta valori `min` e `max` e genera un intero casuale sicuro tra di essi. È utile per OTP e selezioni casuali.

```js
console.log("Random Int:", crypto.randomInt(0, 100)); // 89
```

Ancora una volta, il miglioramento rispetto a `Math.random()` e `Math.floor()` è che il metodo usa casualità crittograficamente sicura sotto il cofano, quindi gli attaccanti non possono prevedere il numero casuale risultante.

Un altro metodo è `createSecretKey()`. Prende un buffer e genera un byte grezzo incapsulato in un `KeyObject`:

```js
const crypto = require("crypto");

const secret = crypto.createSecretKey(crypto.randomBytes(32));
console.log(secret); // SecretKeyObject [KeyObject] { [Symbol(kKeyType)]: 'secret' }
```

Puoi poi usare il metodo `export` per inviare quel `KeyObject`:

```js
const secret = crypto.createSecretKey(crypto.randomBytes(32));
console.log(secret.export());
// <Buffer 53 06 a1 c7 75 69 8b 38 8b a4 b2 f7 1b bc b8
// ae e2 d1 bf 67 af 1a 6a 0a 6e a0 29 62 bb 52 52 32>
```

E infine, usa `toString()` sul buffer per vedere la sua rappresentazione in stringa:

```js
const secret = crypto.createSecretKey(crypto.randomBytes(32));
console.log(secret.export().toString('hex'));
// 32dfe5917668580160986f1623bf8152913329c71163be9c3404a110cd78efd6
```

Oltre a questi, ci sono:

- `createPublicKey()` e `createPrivateKey()` che ti permettono di lavorare con chiavi generate altrove
- `createDiffieHellman()` per due parti che generano un segreto condiviso senza inviare direttamente il segreto
- `Certificate()` per lavorare con quello usato in HTTPS, così puoi analizzare, esportare e verificare il contenuto dei certificati
