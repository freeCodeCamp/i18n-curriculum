## جافاسكريبت على جانب العميل

- **جافاسكريبت على جانب العميل**: شفرة تُنفذ على جانب العميل في تطبيق الويب، عادةً في متصفح الويب.
- **قيود جافاسكريبت على جانب العميل**: رغم قوتها، لجافاسكريبت على جانب العميل بعض القيود:
  - وصول محدود جدًا إلى الملفات المحلية.
  - غير مصممة للتعامل مع منطق تطبيق معقد.
  - مخاوف أمنية محتملة (مثل كشف معلومات حساسة كبيانات اعتماد قاعدة البيانات).
- **سبب القيود**: هذه القيود موجودة لأن جافاسكريبت على جانب العميل صُممت في الأصل لتعمل حصريًا على متصفحات الويب.

## بيئة المتصفح

- **محرك جافاسكريبت**: توفر متصفحات الويب بيئة تشغيل شفرة جافاسكريبت، بما في ذلك محرك جافاسكريبت.
- **نموذج كائن المستند (DOM)**: توفر المتصفحات وصولًا إلى DOM، مما يسمح لك بالوصول إلى عناصر HTML في الشفرة.
- **السياق التاريخي**: سابقًا، لم يكن بإمكان جافاسكريبت العمل خارج المتصفح، لكنها الآن تستطيع ذلك مع Node.js.

## ما هو Node.js؟

- **Node.js**: بيئة تشغيل جافاسكريبت. تُعرف رسميًا بأنها "بيئة تشغيل جافاسكريبت مفتوحة المصدر وعبر المنصات."
- **مفتوح المصدر**: شفرة Node.js متاحة للجمهور وتُدار بواسطة مجتمع كبير من المطورين.
- **عبر المنصات**: يعمل على أي نظام تشغيل، بما في ذلك Windows وmacOS وLinux.
- **بيئة تشغيل جافاسكريبت**: تتيح لك تشغيل شفرة جافاسكريبت خارج المتصفح.

## حالات استخدام Node.js

- **التطبيقات الشائعة**: يُستخدم Node.js في العديد من الصناعات لـ:
  - بناء خوادم الويب وواجهات برمجة التطبيقات التي تتعامل مع طلبات HTTP.
  - تطوير تطبيقات الويب والهاتف المحمول.
  - التعامل مع البيانات المعقدة والتفاعل مع قواعد البيانات.
- **اعتماد الصناعة**: إذا كنت تستخدم أو تطور موقع ويب أو تطبيق ويب يتفاعل مع قاعدة بيانات أو يتعامل مع بيانات معقدة، فمن المحتمل جدًا أن تستخدم Node.js في الخلفية.
- **الخصائص الرئيسية**: Node.js فعال، قابل للتوسع، وله مجتمع كبير من المطورين والمُصَانِعين حول العالم.

## بيئة المتصفح مقابل بيئة تشغيل Node

- **بيئة المتصفح**: مصممة أساسًا لتطوير الواجهة الأمامية، تُشغّل جافاسكريبت على جانب العميل.
- **بيئة تشغيل Node**: مصممة أساسًا لتطوير الواجهة الخلفية، تُشغّل جافاسكريبت على جانب الخادم.
- **اختلافات API**: تحدد البيئة واجهات برمجة التطبيقات المتاحة:
  - **المتصفح**: وصول إلى API الخاص بـ DOM، لكن مع قيود على الوصول إلى نظام الملفات المحلي.
  - **Node**: وصول إلى معظم موارد النظام، بما في ذلك نظام الملفات، لكن ليس DOM.

## الكائنات العالمية

- **الكائن العالمي في المتصفح (`window`)**: يوفر وصولًا إلى وظائف متعلقة بالمتصفح:
  - طرق للتعامل مع DOM.
  - إدارة ملفات تعريف الارتباط.
  - التعامل مع أحداث المتصفح.

- **الكائن العالمي في Node.js (`global`)**: يوفر وصولًا إلى وظائف خاصة بـ Node.js:
  - وحدات مدمجة للعمل مع الملفات المحلية.
  - قدرات الشبكات.
  - التفاعل مع نظام التشغيل.

## التحكم بالإصدار

- **إصدار المتصفح**: لا يمكنك التحكم في إصدار بيئة المتصفح التي يستخدمها زوار موقعك.
- **إصدار Node.js**: يمكنك اختيار إصدار Node.js الذي تريد استخدامه على الخادم الخاص بك.

## أهمية Node.js

- **تطوير كامل المكدس**: غيّر Node.js تطوير الويب بتمكين المطورين من استخدام جافاسكريبت لبناء الواجهة الأمامية والخلفية لتطبيقات كاملة المكدس.
- **الكفاءة**: عملية التطوير أكثر كفاءة لأن المطورين لا يحتاجون لتعلم لغة برمجة جديدة لتطوير الواجهة الخلفية.
- **التأثير**: Node.js أداة قوية وقابلة للتوسع ومتعددة الاستخدامات أصبحت مهمة جدًا للمطورين حول العالم.

## مزايا Node.js

- **لغة واحدة لتطوير كامل المكدس**: يمكنك تنفيذ الواجهة الأمامية والخلفية لتطبيق ويب باستخدام جافاسكريبت.
  - يقلل من منحنى التعلم الأولي.
  - لا حاجة للتبديل بين لغات البرمجة.
  - يزيد الكفاءة والإنتاجية.

- **هيكلية غير حابسة وموجهة بالأحداث**: ممتازة لتطوير التطبيقات الزمن الحقيقي حيث الاستجابة والكفاءة ضرورية لتجربة مستخدم جيدة.

- **خيط واحد وحلقة أحداث**: يمكنه التعامل بفعالية مع عدد كبير من الطلبات المتزامنة وعمليات الإدخال والإخراج.
  - مثالي للتطبيقات التي تتطلب معالجة طلبات متعددة في نفس الوقت.

- **مجتمع كبير**: آلاف المطورين حول العالم يحافظون باستمرار على Node.js ويضيفون ميزات جديدة.
  - توفر العديد من الموارد التعليمية.
  - سهولة العثور على إجابات للأسئلة.

- **npm (مدير حزم Node)**: أداة قوية تتيح لك تثبيت وإدارة الحزم والوحدات لمشاريعك.
  - حزم متوفرة لأغراض متعددة: التعامل مع طلبات HTTP، العمل مع قواعد البيانات، تنسيق التواريخ والأوقات، اختبار الشفرة، وأكثر.
  - إعادة استخدام الشفرة المكتوبة والمختبرة والمشتركة من مطورين آخرين.
  - يجعل سير العمل أسرع وأكثر كفاءة.

- **فعالية من حيث التكلفة**: Node.js مجاني ومفتوح المصدر.
  - الحزم والمكتبات والأدوات المرتبطة به مجانية في الغالب.
  - عامل مهم عند تحسين الميزانية.

## عيوب Node.js

- **قيود الخيط الواحد**: يعمل بخيط واحد فقط في كل مرة، مما يعني أنه يمكنه معالجة عملية واحدة فقط في الوقت نفسه.
  - رغم أن الهيكلية تسمح بمعالجة فعالة للطلبات المتزامنة، قد تؤدي المهام التي تستهلك وحدة المعالجة المركزية إلى حجب الخيط الرئيسي وتسبب مشاكل في الأداء.
  - أمثلة على المهام التي تستهلك وحدة المعالجة المركزية: العمليات الرياضية المعقدة، معالجة الصور والفيديو، التشفير.
  - توجد طرق لتجاوز ذلك، لكنها عادةً تزيد من تعقيد التطبيق.

- **البرمجة غير المتزامنة**: يعتمد Node.js على البرمجة غير المتزامنة.
  - تبدأ مهمة قد تستغرق وقتًا طويلًا للتنفيذ، لكن بدلاً من الانتظار حتى تكتمل، يستمر البرنامج الرئيسي في العمل بينما تعمل المهمة غير المتزامنة بالتوازي.
  - عند اكتمال المهمة، يتعامل البرنامج مع النتيجة.
  - غالبًا ما تتضمن "استدعاءات راجعة"، وهي دوال تحدد ما يحدث عند اكتمال العمليات غير المتزامنة.
  - قد تجعل الشفرة أصعب في القراءة والفهم ومعالجة الأخطاء.

- **مخاوف جودة الحزم**: عند اختيار حزم من npm، قد لا يتم صيانتها باستمرار وقد تُدخل ثغرات أمنية في تطبيقك.
  - يجب على المطورين تقييم كل حزمة بعناية.
  - التحقق من اتباع الحزم لأفضل الممارسات في الجودة والأمان.

## الخيوط

- **الخيط**: مسار تنفيذ داخل عملية، مثل برنامج كمبيوتر يعمل حاليًا.

## NVM (مدير إصدارات Node)

- **NVM**: أداة لإدارة إصدارات متعددة من Node.js على نفس الكمبيوتر.
- **الغرض**: تساعدك على التبديل بسهولة بين إصدارات مختلفة من Node.js لمشاريع مختلفة.
- **التوصية**: يُنصح بتثبيت NVM قبل Node.js لمرونة مستقبلية.

## تثبيت NVM على macOS/Linux

- **أدوات سطر أوامر Xcode (macOS)**: يجب تثبيتها يدويًا قبل تشغيل سكريبت تثبيت NVM على macOS.

```bash
xcode-select --install
```

- **التحقق من تثبيت Xcode**:

```bash
xcode-select --version
```

- **تحميل وتنفيذ سكريبت تثبيت NVM**: شغّل أحد هذه الأوامر:

```bash
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.3/install.sh | bash
```

```bash
wget -qO- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.3/install.sh | bash
```

- **إعادة تشغيل الطرفية**: قد تحتاج لإعادة تشغيل الطرفية عند اكتمال العملية لتحميل الأمر والتكوين الجديد.

- **التحقق من تثبيت NVM**:

```bash
nvm -v
```

## تثبيت Node.js باستخدام NVM على macOS/Linux

- **تثبيت أحدث إصدار LTS**: لتثبيت أحدث إصدار دعم طويل الأمد (LTS) من Node.js:

```bash
nvm install --lts
```

- **LTS (الدعم طويل الأمد)**: يُوصى بهذا الإصدار عادةً لأنه يعطي الأولوية للاستقرار والموثوقية والأمان على الميزات الجديدة أو التجريبية.
  - مُختبر جيدًا وجاهز للإنتاج.
  - مضمون بفترات دعم أطول مع التركيز على إصلاح الأخطاء وتحديثات الأمان.

- **تثبيت إصدار محدد**: يمكنك تثبيت إصدار معين من Node.js بتحديد رقم الإصدار:

```bash
nvm install <version>
```

مثال:

```bash
nvm install 20
```

- **التبديل إلى إصدار**:

```bash
nvm use <version>
```

مثال:

```bash
nvm use 20
```

- **التحقق من إصدار Node.js الحالي**:

```bash
node -v
```

## تثبيت NVM على Windows

- **الدعم الرسمي لـ NVM**: الإصدارات الرسمية من NVM تدعم Windows فقط في حالات محددة.
  - إذا كان لديك Windows Subsystem for Linux (WSL)، يمكنك اتباع دليل macOS/Linux أعلاه.

- **nvm-windows**: أداة إدارة إصدارات Node.js لنظام Windows.
  - مشروع منفصل تمامًا عن مشروع NVM الرسمي.
  - نفس الغرض: مساعدتك على العمل مع إصدارات متعددة من Node.js على نفس الجهاز.

- **قبل التثبيت**: يُنصح بإلغاء تثبيت أي نسخة Node سابقة قبل تثبيت NVM لنظام Windows.

- **خطوات التثبيت**:
  - اذهب إلى قسم الإصدارات في المستودع الرسمي: `https://github.com/coreybutler/nvm-windows/releases`.
  - حمّل أحدث مثبت.
  - اتبع التعليمات.

- **إعادة تشغيل الطرفية**: قد تحتاج لإعادة تشغيل الطرفية إذا لم يكن أمر `nvm` متاحًا فورًا.

- **التحقق من التثبيت**:

```powershell
nvm -v
```

- **تثبيت إصدار محدد**:


- **التبديل إلى إصدار**:

```powershell
nvm use <version>
```

مثال:

```powershell
nvm use 20
```

- **التحقق من إصدار Node.js الحالي**:

## أوامر NVM وNode.js الأساسية

- **ملاحظة حول الأوامر**: جميع الأوامر تعمل مع NVM المثبت على macOS/Linux، بما في ذلك WSL. قد تعمل بشكل مختلف أو لا تعمل على الإطلاق مع `nvm-windows`. تُقدم أوامر بديلة لـ `nvm-windows` عند الإمكان.

### سياسة تنفيذ PowerShell (Windows)

- **سياسة التنفيذ الافتراضية**: سياسة تنفيذ PowerShell الافتراضية هي `Restricted`، مما يمنع تشغيل السكريبتات كإجراء أمني.

- **تشغيل PowerShell كمسؤول**: انقر بزر الماوس الأيمن على PowerShell، ثم اختر "تشغيل كمسؤول".

- **التحقق من سياسة التنفيذ الحالية**:

```powershell
Get-ExecutionPolicy
```

- **تعيين سياسة التنفيذ إلى RemoteSigned**: يسمح بتشغيل السكريبتات المحلية و`npm`:

```powershell
Set-ExecutionPolicy RemoteSigned
```

- **بعد تعيين السياسة**: أغلق نافذة PowerShell الخاصة بالمسؤول وافتح نافذة PowerShell عادية.

### قائمة جميع الإصدارات المثبتة

- **قائمة الإصدارات المثبتة**: لعرض جميع إصدارات Node.js المثبتة والتحقق من الإصدار المستخدم حاليًا:

```bash
nvm ls
```

### تعيين إصدار Node.js الافتراضي (NVM على macOS/Linux فقط)

- **تعيين الإصدار الافتراضي**: يحدد الإصدار الافتراضي لـ Node.js لجلسات الطرفية الجديدة:

```bash
nvm alias default <version>
```

مثال:

```bash
nvm alias default 22.20.0
```

- **إنشاء أسماء مستعارة مخصصة**: تعيين اسم أكثر تحديدًا لإصدار معين:

```bash
nvm alias <new_name> <version>
```

مثال:

```bash
nvm alias new-api 18.20.8
```

- **استخدام الاسم المستعار المخصص**:

```bash
nvm use <new_name>
```

مثال:

```bash
nvm use new-api
```

### إزالة إصدار من Node.js

- **إلغاء تثبيت إصدار**:

```bash
nvm uninstall <version>
```

مثال:

```bash
nvm uninstall 20.19.5
```

- **الإزالة حسب الإصدار الرئيسي (NVM على macOS/Linux)**: يمكنك إزالة إصدار بتشغيل أمر `uninstall` مع رقم الإصدار الرئيسي مثل `22` أو `20`:

```bash
nvm uninstall 20
```

إذا كان لديك عدة إصدارات لذلك الإصدار الرئيسي مثبتة، مثل `20.14.0` و`20.12.2`، سيتم إزالة أحدث إصدار أولاً. تشغيل `nvm uninstall 20` سيزيل `20.14.0` أولاً.

### التحقق من إصدار Node.js

- **التحقق من الإصدار الحالي**:

```bash
node -v
```

### التحقق من إصدار npm

- **npm**: مدير حزم لـ Node.js يُستخدم لتثبيت ونشر وإدارة حزم البرامج. يُثبت تلقائيًا عند تثبيت Node.js.

- **التحقق من إصدار npm**:

```bash
npm -v
```

### تشغيل ملف جافاسكريبت

- **تشغيل ملف جافاسكريبت**: يشغّل ملف جافاسكريبت في بيئة تشغيل Node.js النشطة حاليًا:

```bash
node <file.js>
```

مثال:

```bash
node app.js
```

### إنشاء مشروع Node.js

- **تهيئة المشروع**: ينشئ مشروع Node.js جديدًا بإنشاء ملف `package.json` الذي يتتبع تفاصيله واعتمادياته:

```bash
npm init
```

- **الإعداد التفاعلي**: يأخذك أمر `npm init` خلال دليل CLI تفاعلي لإعداد ملف `package.json`.
  - يطرح أسئلة عن اسم المشروع، الوصف، الإصدار، وغيرها.

- **الإعداد الافتراضي**: للبدء بملف `package.json` افتراضي:

```bash
npm init -y
```

أو

```bash
npm init --yes
```

### تثبيت حزمة

- **تثبيت حزمة محددة**:

```bash
npm install <package>
```

مثال:

```bash
npm install express
```

### تثبيت جميع الاعتماديات

- **تثبيت التبعيات**: لتثبيت جميع التبعيات المدرجة في ملف `package.json` الخاص بالمشروع:

```bash
npm install
```
