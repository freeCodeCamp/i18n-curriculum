## ما هو npm؟

- **التعريف**: npm هو سجل إلكتروني للبرمجيات مفتوحة المصدر، يعمل مثل "متجر تطبيقات" للشفرة.
- **المعنى الرسمي**: npm لا تعني رسميًا "Node Package Manager"؛ بل هي اختصار تكراري يعني "npm ليست اختصارًا".
- **ثلاثة أجزاء مرتبطة:**

- **الموقع الإلكتروني (npmjs.com)**: للبحث عن الوحدات في السجل.
  - **السجل**: قاعدة بيانات عامة كبيرة تخزن كل الوحدات.
  - **واجهة سطر الأوامر (CLI)**: أداة تُثبت مع Node.js لإدارة تبعيات المشروع.

- **البدائل**: رغم أن npm هو الأكثر استخدامًا، تشمل البدائل الشهيرة **Yarn**، **PNPM**، و**Bun**.

## ملف package.json

- **الغرض**: ملف إعدادات يخزن البيانات الوصفية، التبعيات، والسكريبتات لمشروع Node.js.
- **الحقول الرئيسية:**

- `name` و`version`: لتحديد المشروع.
  - `main`: يحدد ملف نقطة الدخول، غالبًا `index.js`.
  - `license`: يحدد الشروط القانونية (الافتراضي هو `ISC`).
  - `dependencies`: الوحدات المطلوبة لتشغيل التطبيق في الإنتاج.
  - `devDependencies`: الأدوات اللازمة فقط أثناء التطوير، مثل أُطُر الاختبار.


- **الإنشاء:**

- `npm init`: يبدأ عملية إعداد تفاعلية.
  - `npm init -y`: ينشئ `package.json` بقيم افتراضية فورًا.

## الإصدار الدلالي (Semantic Versioning - SemVer)

- **الصيغة**: `MAJOR.MINOR.PATCH`.
- **القواعد:**

- **MAJOR**: تغييرات جذرية قد تؤثر على الشفرة الحالية.
  - **MINOR**: ميزات جديدة متوافقة مع الإصدارات السابقة.
  - **PATCH**: إصلاحات أخطاء وتحسينات صغيرة.

- **رموز النطاق:**

- `^` (مؤشر الإدراج): يسمح بتحديثات MINOR وPATCH (مثلاً، `^1.2.3` يسمح بـ `1.x.x`).
  - `~` (مد): يسمح فقط بتحديثات PATCH (مثلاً، `~1.2.3` يسمح بـ `1.2.x`).
  - `*` (نجمة): يطابق أي إصدار؛ يستخدم أساسًا للاختبار.

## إدارة التبعيات

- **التثبيت:**

- `npm install <name>`: يضيف تبعية للإنتاج.
  - `npm install <name> -D`: يضيف تبعية للتطوير.
  - `npm install <name>@<version>`: يثبت إصدارًا محددًا.

- **وحدات Node وقفل الإصدارات:**

- `node_modules/`: يحتوي على الشفرة الفعلية للوحدات المثبتة.
  - `package-lock.json`: يقفل الإصدارات الدقيقة لكل التبعيات و"التبعيات الفرعية" لضمان التوافق عبر بيئات العمل.

- **الإزالة**: `npm uninstall <name>` يحذف الوحدة ويحدث `package.json`.
- **الصيانة:**

- `npm outdated`: يحدد الوحدات التي لها إصدارات أحدث متاحة.
  - `npm update`: يحدث الوحدات ضمن نطاق الإصدار الدلالي المسموح.

## سكريبتات npm

- **الوظيفة**: أوامر مخصصة معرفة في قسم `scripts` داخل `package.json` لأتمتة المهام.
- **التنفيذ**: تُشغل عبر `npm run <script_name>`.
- توجد اختصارات لـ `start`، `test`، `stop`، و`restart` (مثلاً، `npm start`).
- **التنفيذ المتسلسل:**
- `&&`: يشغل الأمر الثاني فقط إذا نجح الأول.
- `;`: يشغل الأوامر واحدًا تلو الآخر بغض النظر عن النجاح.
- **التنفيذ المتزامن**: استخدم عامل `&` لتشغيل العمليات في الخلفية.
- **المعلمات**: مرر المعلمات للسكريبتات باستخدام الفاصل `--` (مثلاً، `npm start -- 8000`).

## النشر إلى سجل npm

- **الوصول إلى السجل**: قاعدة بيانات للوحدات العامة والخاصة.
- **خطوات النشر:**

1. أنشئ حسابًا مجانيًا على `npmjs.com`.
2. سجّل الدخول عبر الطرفية باستخدام `npm login`.
3. تأكد من وجود `name` فريد و`version` صالح في `package.json`.
4. أنشئ `README.md` و`.npmignore` (لاستبعاد الملفات الحساسة).
5. شغّل `npm publish` (أو `npm publish --access public` للوحدات ذات النطاق).

- **النطاقات**: تُستخدم لتجنب تعارض الأسماء عبر النشر تحت `@username/package-name`.

## أنظمة الوحدات في Node.js

### CommonJS (CJS)

- **النظام الافتراضي**: الطريقة الأصلية التي يتعامل بها Node.js مع الوحدات؛ تحميل متزامن.
- **الصيغة:**
- **الاستيراد**: `const module = require('./path')`.
- **التصدير**: `module.exports = { functionName }`.
- **التعريف**: يستخدم للملفات `.cjs` أو `.js` عندما لا يُحدد `type` في `package.json`.

### وحدات ES (ESM)

- **الصيغة الموحدة**: معيار جافاسكريبت الحديث؛ تحميل غير متزامن لتحسين الأداء.
- **الصيغة:**

- **التصدير**: `export { name }` أو `export default function`.
  - **الاستيراد**: `import { name } from './path.mjs'`.
  - **التعريف**: يستخدم للملفات `.mjs` أو `.js` عندما يحتوي `package.json` على `"type": "module"`.
