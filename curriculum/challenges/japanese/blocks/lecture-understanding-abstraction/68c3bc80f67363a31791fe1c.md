# --description--

カプセル化、継承、ポリモーフィズムについて学んだので、次にオブジェクト指向プログラミングの重要な概念である抽象化について説明します。

抽象化とは、複雑な実装の詳細を隠し、オブジェクトやシステムの本質的な機能だけを見せるプロセスです。何をするかにフォーカスし、どのようにそれをするかは隠すイメージです。

抽象化はPythonに限った話ではありません。オブジェクト指向プログラミングをサポートする多くの言語で実装できるプログラミングの概念です。

抽象化を説明するために、車を運転している状況を想像してください。運転席に座っているときは、ハンドル、シフトレバー、アクセルやブレーキペダルなどの重要な部分とだけやり取りします。エンジンの仕組みやトランスミッションのギアチェンジの仕組み、ブレーキシステムの物理的な詳細を知る必要はありません。これらはすべて複雑な実装の詳細だからです。

これが抽象化の働きです！複雑なシステムとやり取りするための簡略化されたインターフェースを提供しています。

車の場合、簡略化されたインターフェースはハンドルやブレーキ、アクセルであり、複雑なシステムは車そのものです。

Pythonが抽象化をどのように実装しているかというと、`abc` モジュールを通じて行われます。

このモジュールは`ABC` クラス（「抽象基底クラス」を表す）と`@abstractmethod` デコレーターを提供します。

`ABC` は継承されることを目的としたクラスですが、直接インスタンス化することはできません。これはサブクラスが実装しなければならない共通のメソッドやプロパティのインターフェースを定義します。

一方、抽象メソッドは`@abstractmethod` デコレーターを使って抽象基底クラス（ABC）内で宣言されるメソッドです。実装がないか基本的なデフォルト実装がある場合があります。しかし、どのサブクラスも具体的なクラスとしてインスタンス化可能になるためには、このメソッドをオーバーライドしなければなりません。

Pythonの抽象クラスの基本的な構文は以下の通りです：

```py
from abc import ABC, abstractmethod

# Define an abstract base class
class AbstractClass(ABC):
    @abstractmethod
    def abstract_method(self):
        pass

# Concrete subclass that implements the abstract method
class ConcreteClassOne(AbstractClass):
    def abstract_method(self):
        print('Implementation in ConcreteClassOne')

# Another concrete subclass
class ConcreteClassTwo(AbstractClass):
    def abstract_method(self):
        print('Implementation in ConcreteClassTwo')
```

基本的な例はこちらです：

```py
from abc import ABC, abstractmethod

class Animal(ABC): # Inherits from abstract base class
   @abstractmethod # Abstract method decorator
   def make_sound(self):  # The method subclasses must override
       pass

# Concrete class that will override the abstract method
class Dog(Animal):
   def make_sound(self):
       print('Woof!')

# Another concrete class that will override the abstract method
class Cat(Animal):
   def make_sound(self):
       print('Meow!')

# Another concrete class that will override the abstract method
class Monkey(Animal):
   def make_sound(self):
       print('Ooh ooh aah aah!')

# Create instances of each concrete class
animals = [Dog(), Cat(), Monkey()]

# Loop through the instances to call the make_sound method
for animal in animals:
   animal.make_sound()

# Output:
# Woof!
# Meow!
# Ooh ooh aah aah!
```

この例では：

- `ABC` モジュールから`abstractmethod` クラスと`abc` をインポートしています。
- `Animal` を継承した`ABC` クラスを作成し、その中に`make_sound` の各サブクラスがオーバーライドしなければならない抽象メソッド`Animal` を宣言しています。
- `Dog`、`Cat`、`Monkey` という具体的なクラスを作成し、それぞれが`make_sound` 抽象メソッドをオーバーライドしています。
- 具体的なクラスのインスタンスを作成し、それぞれの`make_sound` メソッドを呼び出して、各クラスが`make_sound` 抽象メソッドを独自に実装している様子を示しています。

Animalクラスのインスタンスを作成しようとするとどうなるか覚えておいてください：

```py
dog = Animal() 
# TypeError: Can't instantiate abstract class Animal 
# without an implementation for abstract method 'make_sound'
```

抽象メソッドの実装を提供しないサブクラスにも同じルールが適用されます。Animalのサブクラスを定義しても、make_soundをオーバーライドしない限りインスタンス化できません。以下はその例です：

```py
class Bird(Animal):
    pass

bird = Bird()
# TypeError: Can't instantiate abstract class Bird 
# without an implementation for abstract method 'make_sound'
```

次は、具体的なメソッドのインスタンスに渡せるインスタンス属性を持つ例です：

```py
from abc import ABC, abstractmethod

# The blueprint for any toy that can speak
class TalkingToy(ABC):
   def __init__(self, name):
       self.name = name
   @abstractmethod
   def speak(self):
       pass

class RobotToy(TalkingToy):
   def speak(self):
       print(f'{self.name} says beep boop! I am a robot!')

class TeddyBearToy(TalkingToy):
   def speak(self):
       print(f"{self.name} says hug me! I'm cuddly!")

class DinosaurToy(TalkingToy):
   def speak(self):
       print(f'{self.name} says ROOOOAR!')

# Create toys
rusty = RobotToy('Rusty')
fluffy = TeddyBearToy('Fluffy')
rex = DinosaurToy('Rex')

toys = [rusty, fluffy, rex]
for toy in toys:
   toy.speak()

# Output:
# Rusty says beep boop! I am a robot!
# Fluffy says hug me! I'm cuddly!
# Rex says ROOOOAR!
```

この例では：

- 抽象基底クラス`TalkingToy` が話すことができるおもちゃの設計図を定義しています。
- サブクラス`RobotToy`、`TeddyBearToy`、`DinosaurToy` はそれぞれ独自の方法で`speak` メソッドを実装しています。
- これらのサブクラスのインスタンスを作成し、`speak` メソッドを呼び出すと、それぞれのおもちゃが独自の方法で話します。

まとめると、Pythonの抽象化は複雑なシステムを簡略化し、再利用性を高めます。

抽象クラスの単一のメソッドを複数のサブクラスで再利用しつつ、各サブクラスに特有の振る舞いを強制的に実装させる方法を見てきました。

この方法はコードを整理しやすく、柔軟で、特にアプリケーションが大きくなるにつれて保守しやすくなります。
