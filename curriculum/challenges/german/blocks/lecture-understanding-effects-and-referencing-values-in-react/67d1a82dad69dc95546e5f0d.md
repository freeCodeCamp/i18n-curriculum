# --description--

In React kann es Situationen geben, in denen Sie direkten Zugriff auf ein DOM-Element benötigen. Hier kommen „Refs“ ins Spiel. Refs können auch veränderliche Werte speichern, aber für solche Fälle ist der State die bessere Wahl.

In reinem JavaScript haben Sie die `getElementById()` und `querySelector()` Methoden verwendet, um auf DOM-Elemente zuzugreifen. In React verwenden Sie jedoch refs, um Elemente im DOM zu referenzieren.

Ein wesentlicher Unterschied ist, dass Sie mit refs keine Bezeichner wie IDs und Klassen benötigen, um Elemente zu referenzieren.

Wie können Sie also refs erstellen und verwenden? React stellt einen `useRef()` Hook bereit, mit dem Sie genau das tun können.

Der erste Schritt ist, den Hook aus React zu importieren:

```js
import { useRef } from "react";
```

Als Nächstes müssen Sie eine Variable erstellen, die die ref mit dem Anfangswert der ref innerhalb des `useRef` Hooks hält, zum Beispiel eine `sectionRef`, die mit `null` initialisiert wird:

```js
const sectionRef = useRef(null);
```

Das Letzte, was Sie tun müssen, ist, die ref-Variable mit dem `ref` Attribut an das Element in Ihrem JSX anzuhängen:

```jsx
<section ref={sectionRef}>
  {/* Section content */}
</section>
```

Wenn Sie die ref in der Konsole ausgeben, sehen Sie, dass es sich um ein Objekt mit dem aktuellen Wert handelt, in diesem Fall `null`:

```js
console.log(sectionRef); // { current: null }
```

Sie können auch den aktuellen Wert mit der `current` Eigenschaft in der Konsole ausgeben, damit Sie den Wert direkt sehen:

```js
console.log(sectionRef.current); // null
```

Die nachfolgenden Werte der ref hängen vom Lebenszyklus der Komponente ab.

Zum Beispiel ist der Anfangswert von `sectionRef` immer `null`, weil er so initialisiert wurde. Nachdem die Komponente gemountet wurde, ist der Wert der ref das `section` Element, an das die ref angehängt ist.

Wenn die Komponente unmountet wird, geht der Wert der ref zurück auf den Anfangswert `null`.

Ein typisches Beispiel, um eine ref zu demonstrieren, ist das Setzen des Fokus auf ein Eingabeelement beim Rendern oder durch Klicken auf einen Button.

So machen Sie das, wenn Sie auf einen Button klicken:

```jsx
import { useRef } from "react";

const Focus = () => {
  const inputRef = useRef(null);

  const handleFocus = () => {
    if (inputRef.current) {
      inputRef.current.focus();
    }
  };

  return (
    <div>
      <input ref={inputRef} type="text" placeholder="Enter text" />
      <button onClick={handleFocus}>Focus Input</button>
    </div>
  );
};

export default Focus;
```

Im obigen Code wird die `inputRef` erstellt und an das `input` Element angehängt. Es gibt auch einen Button mit einem `onClick` Ereignis, das eine `handleFocus` Funktion aufruft.

Alles, was die `handleFocus` Funktion tut, ist, die `focus()` Methode auf dem `input` Element aufzurufen. Beachten Sie, dass `input` eine eingebaute Komponente von React ist, weshalb das tatsächliche `input` DOM-Element in der `current` Eigenschaft der ref gespeichert ist. Deshalb rufen Sie die `focus()` Methode mit `input.current.focus()` auf.

Hier sind einige bewährte Vorgehensweisen, die Sie beim Arbeiten mit refs beachten sollten:

- Verwenden Sie refs hauptsächlich, um mit dem DOM zu interagieren. Sie können sie auch für veränderliche Daten verwenden, aber für diese ist der State die bessere Wahl.

- Verwenden Sie refs nicht für grundlegendes State-Management – dafür ist `useState` gedacht.

- Stellen Sie sicher, dass `ref.current` existiert, bevor Sie auf seine Eigenschaften zugreifen. So machen Sie das noch einmal:

```js
const handleFocus = () => {
  if (inputRef.current) {
    inputRef.current.focus();
  }
};
```

Das verhindert Fehler, falls auf die ref zugegriffen wird, bevor sie an das DOM angehängt oder nachdem sie entfernt wurde.
