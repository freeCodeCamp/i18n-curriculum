# --description--

最後の要素、決定打の決定打です。すべての要素を考慮した後でも、まだ複数の競合するルールが同じ要素に適用される場合があります。カスケードはどのルールを適用するかどうやって決めるのでしょうか？

実はとてもシンプルです。最後に定義されたルールが勝ちです。

```css
/* styles.css */

.alert {
  color: red;
}

.warning {
  color: yellow;
}
```

`alert`と`warning`の両方のクラスを持つ要素の場合、カスケードは継承（ここではなし）や特異性（どちらのルールもより特異ではない）など、他のすべての要素を順に確認します。`.warning`のルールが最後に定義されていて、他の要素でどのルールを適用するか決まらなかったため、そのルールが要素に適用されます。
