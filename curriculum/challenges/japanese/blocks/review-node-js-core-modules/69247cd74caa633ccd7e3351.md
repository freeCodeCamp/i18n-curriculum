## Node.js コアモジュールの紹介

- **Node.js コアモジュール**：Node.js に組み込みで付属しているモジュールで、ファイル操作、暗号処理、OSとの連携などの基本機能を提供します。
- **標準モジュール**：別途インストール不要で、Node.js がインストールされていれば利用可能です。
- **インポート構文**：`require()` を使ってコアモジュールをアプリケーションにインポートします。

```javascript
const fs = require("fs");
const crypto = require("crypto");
const os = require("os");
```

## ファイルシステム（fs）モジュール

- **目的**：ファイルやフォルダーの操作を行うメソッドを提供し、開く、閉じる、読み込み、書き込み、削除などが可能です。
- **インポート**：`const fs = require("fs");`
- **柔軟性**：コールバック、Promise、同期の3つの形式でメソッドが利用できます。

### メソッドの使用パターン

- **コールバックによる非同期**：デフォルトの動作で、ブロックしません。

```javascript
fs.writeFile("filePath", "content", "utf8", (err) => {
  if (err) throw err;
  console.log("File written to!");
});
```

- **Promise ベース**：モダンなアプリケーションに推奨され、`async/await` を使います。

```javascript
const fs = require("fs/promises");

async function writeToFile() {
  try {
    await fs.writeFile("article.md", "## Node fs Module");
    console.log("File written to!");
  } catch (err) {
    console.error(err);
  }
}
```

- **同期処理**：ブロッキング操作で、単純なスクリプトに適しています。

```javascript
try {
  fs.writeFileSync("article.md", "## Node fs Module", "utf8");
  console.log("File written to!");
} catch (err) {
  console.error(err);
}
```

### 主要なファイル操作

- **`writeFile()` メソッド**：指定した内容でファイルを作成または上書きします。

```javascript
await fs.writeFile("article.md", "## Node fs Module: The Complete Guide");
```

- **`appendFile()` メソッド**：既存のファイルの末尾に内容を追加します。

```javascript
await fs.appendFile("article.md", "\n\nIn this article, you will learn...");
```

- **`readFile()` メソッド**：ファイルの内容を読み取ります。

```javascript
const content = await fs.readFile("article.md", "utf8");
console.log("File content:", content);
```

- **`unlink()` メソッド**：ファイルシステムからファイルを削除します。

```javascript
await fs.unlink("article.md");
console.log("File deleted successfully");
```

### 文字エンコーディング

- **UTF-8 エンコーディング**：ファイルをテキスト文字列として読み込むには `"utf8"` を指定します。
- **バッファ形式**：エンコーディングを指定しない場合、ファイルはバイナリのバッファオブジェクトとして読み込まれます。

```javascript
// Returns string
const textContent = await fs.readFile("file.txt", "utf8");

// Returns Buffer
const bufferContent = await fs.readFile("file.txt");
```

## バッファモジュール

- **目的**：ファイル、画像、ネットワークストリームなどのバイナリデータをメモリ上で直接扱います。
- **インポート**：`const { Buffer } = require("buffer");`
- **利用例**：ファイル入出力、TCPストリーム、画像処理、バイナリデータ操作など。

### バッファの作成

- **`Buffer.from()` メソッド**：文字列、配列、その他のデータからバッファを作成します。

```javascript
// From string
const myStrBuffer = Buffer.from("freeCodeCamp");
console.log(myStrBuffer); // <Buffer 66 72 65 65 43 6f 64 65 43 61 6d 70>

// From array of numbers
const myNumBuffer = Buffer.from([70, 82, 69, 69, 67, 79, 68, 69, 67, 65, 77, 80]);
```

- **`Buffer.alloc()` メソッド**：指定サイズのゼロ埋めバッファを作成します。

```javascript
const someBuffer = Buffer.alloc(10);
console.log(someBuffer); // <Buffer 00 00 00 00 00 00 00 00 00 00>
```

### バッファ操作

- **配列のようなアクセス**：インデックス表記でバッファの個々の要素にアクセスできます。

```javascript
console.log(myStrBuffer[0]); // 102 (byte value for 'f')
```

- **`toString()` メソッド**：バッファを読みやすい文字列に変換します。

```javascript
console.log(myStrBuffer.toString()); // "freeCodeCamp"
```

- **`Buffer.write()` メソッド**：割り当てられたバッファにデータを書き込みます。

```javascript
someBuffer.write("Hello fCC");
console.log(someBuffer.toString()); // "Hello fCC"
```

- **`Buffer.byteLength()` メソッド**：文字列を格納するのに必要なバイト数を返します。

```javascript
console.log(Buffer.byteLength("Hello freeCodeCamp")); // 18
```

### 追加のバッファメソッド

- **`Buffer.isBuffer()`**：オブジェクトがバッファかどうかを判定します。
- **`Buffer.compare()`**：2つのバッファを比較し、ソート順を返します。
- **`Buffer.concat()`**：複数のバッファを1つに結合します。

## Crypto モジュール

- **目的**：ハッシュ化、暗号化、復号、デジタル署名などの暗号機能を提供します。
- **インポート**：`const crypto = require("crypto");`
- **セキュリティ注意**：本番の認証には `bcrypt` や `jsonwebtoken` のような十分に検証されたライブラリを使うことをお勧めします。

### ハッシュ化メソッド

- **`createHash()` メソッド**：SHA256、SHA512、MD5 などのアルゴリズムで一方向ハッシュを作成します。

```javascript
const hashedPassword = crypto
  .createHash("sha256")
  .update("myPassword123")
  .digest("hex");

console.log("createHash result:", hashedPassword);
```

- **`createHmac()` メソッド**：認証やデータ整合性のために秘密鍵を使ったハッシュを作成します。

```javascript
const hashedMessage = crypto
  .createHmac("sha256", "secret-key")
  .update("Hello World")
  .digest("hex");
```

### 暗号化と復号

- **`createCipheriv()` と `createDecipheriv()` メソッド**：アルゴリズム、鍵、初期化ベクトルを使ってデータを暗号化・復号します。

```javascript
const key = Buffer.from("12345678901234567890123456789012"); // 32 bytes for AES-256
const iv = Buffer.from("1234567890123456"); // 16 bytes for AES

// Encryption
const cipher = crypto.createCipheriv("aes-256-cbc", key, iv);
let encrypted = cipher.update("Hello campers!", "utf8", "hex");
encrypted += cipher.final("hex");

// Decryption
const decipher = crypto.createDecipheriv("aes-256-cbc", key, iv);
let decrypted = decipher.update(encrypted, "hex", "utf8");
decrypted += decipher.final("utf8");
```

### ランダム値生成

- **`randomBytes()` メソッド**：暗号学的に安全なランダムバイトを生成します。

```javascript
console.log("Random Bytes:", crypto.randomBytes(16).toString("hex"));
// Output: a6154ef5a296fa176ad0f332bd94d712
```

- **`randomInt()` メソッド**：指定範囲内の安全なランダム整数を生成します。

```javascript
console.log("Random Int:", crypto.randomInt(0, 100)); // Random number 0-99
```

### 鍵管理

- **`createSecretKey()` メソッド**：生のバイトから暗号鍵オブジェクトを作成します。

```javascript
const secret = crypto.createSecretKey(crypto.randomBytes(32));
console.log(secret.export().toString('hex'));
```

### デジタル署名

- **`sign()` と `verify()` メソッド**：秘密鍵・公開鍵ペアを使ってデジタル署名を作成・検証します。
- **利用例**：データの真正性と整合性を証明します。

### 追加の Crypto メソッド

- **`createPublicKey()` と `createPrivateKey()`**：外部で生成された鍵を扱います。
- **`createDiffieHellman()`**：安全な共有秘密の生成を可能にします。
- **`Certificate()`**：HTTPS 証明書の操作を行います。

## OS モジュール

- **目的**：システム情報やクロスプラットフォーム対応のためのOS関連ユーティリティを提供します。
- **インポート**：`const os = require("os");`
- **利用例**：システム監視、クロスプラットフォームスクリプト、ハードウェア情報取得。

### システム情報メソッド

- **`platform()` メソッド**：OSのプラットフォームを返します。

```javascript
console.log(os.platform()); // 'darwin', 'win32', 'linux', etc.

// Cross-platform usage
if (os.platform() === 'win32') {
  // Windows-specific code
} else {
  // Unix-like systems
}
```

- **`arch()` メソッド**：CPUアーキテクチャを返します。

```javascript
console.log(os.arch()); // 'arm64', 'x64', 'arm', 'ia32', etc.
```

- **`type()` メソッド**：公式のOS名を返します。

```javascript
console.log(os.type()); // 'Darwin', 'Linux', 'Windows_NT'
```

### バージョン情報

- **`release()` メソッド**：OSカーネルのバージョンを表示します。

```javascript
console.log(os.release()); // '25.0.0'
```

- **`version()` メソッド**：詳細なOSバージョン情報を返します。

```javascript
console.log(os.version());
// Darwin Kernel Version 25.0.0: Wed Sep 17 21:41:39 PDT 2025
```

### ハードウェア情報

- **`cpus()` メソッド**：CPUコア情報の配列を返します。

```javascript
console.log(os.cpus()); // Array of CPU core objects with model, speed, times
```

- **`uptime()` メソッド**：システムの稼働時間（秒）を返します。

```javascript
console.log(os.uptime()); // 23047 (seconds since system boot)
```

### メモリ情報

- **`totalmem()` メソッド**：システムの総メモリ容量（バイト）を返します。
- **`freemem()` メソッド**：利用可能なメモリ容量（バイト）を返します。

## ベストプラクティス

### パフォーマンスの考慮点

- **同期メソッドの回避**：本番アプリではブロックを防ぐため非同期メソッドを使ってください。
- **Promise の推奨**：Promise ベースの API である `async/await` を使うと、コードがよりクリーンで保守しやすくなります。
- **エラーハンドリング**：try-catch ブロックやエラーコールバックで適切にエラー処理を実装してください。

### セキュリティガイドライン

- **Crypto モジュール**：独自の暗号化ソリューションを作るより、認証には実績のあるライブラリを使ってください。
- **入力バリデーション**：ファイルパスやユーザー入力を検証し、セキュリティリスクを防止してください。
- **鍵管理**：暗号鍵は安全に保管し、ソースコードにハードコードしないでください。

### クロスプラットフォーム開発

- **OS モジュールの活用**：クロスプラットフォーム対応には OS モジュールのメソッドを使ってください。
- **パス操作**：クロスプラットフォームのファイルパス操作には `path` モジュールを使います。
- **環境変数**：プラットフォーム固有の環境変数や設定を確認してください。
