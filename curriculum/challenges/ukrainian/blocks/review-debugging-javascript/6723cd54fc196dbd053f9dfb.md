## Загальні типи повідомлень про помилки

- **SyntaxError**: Ці помилки виникають, коли ви неправильно пишете щось у своєму коді, наприклад, пропускаєте дужку або квадратну дужку. Уявіть це як граматичну помилку в реченні.

```js
const arr = ["Beau", "Quincy" "Tom"]
```

- **ReferenceError**: Існує кілька типів помилок ReferenceError, які виникають різними способами. Перший тип помилки посилання — це невизначені змінні. Інший приклад ReferenceError — це спроба отримати доступ до змінної, оголошеної з `let` або `const`, до того, як вона була визначена.

:::interactive_editor

```js
console.log(num);
const num = 50;
```

:::

- **TypeError**: Ці помилки виникають, коли ви намагаєтеся виконати операцію над неправильним типом.

```js
const developerObj = {
  name: "Jessica",
  country: "USA",
  isEmployed: true
};

developerObj.map()
```

- **RangeError**: Ці помилки трапляються, коли ваш код намагається використати значення, яке виходить за межі того, що JavaScript може обробити.

```js
const arr = [];
arr.length = -1; 
```

## Інструкція `throw` 

- **Визначення**: Інструкція `throw` у JavaScript використовується для викидання користувацького виключення. Виключення в програмуванні — це коли трапляється несподівана подія, яка порушує нормальний потік програми.

:::interactive_editor

```js
function validateNumber(input) {
  if (typeof input !== "number") {
    throw new TypeError("Expected a number, but received " + typeof input);
  }
  return input * 2;
}

console.log(validateNumber("Naomi"));   // TypeError: Expected a number, but received string
```

:::

## `try...catch...finally`

- **Визначення**: Блок `try` використовується для обгортання коду, який може викликати помилку. Він слугує безпечною зоною для спроби виконати щось, що може не вдатися. Блок `catch` ловить і обробляє помилки, які виникають у блоці try. Ви можете використовувати об’єкт помилки всередині catch, щоб перевірити, що пішло не так. Блок `finally` виконується після блоків try і catch, незалежно від того, чи сталася помилка. Його часто використовують для завдань очищення, наприклад, закриття файлів або звільнення ресурсів.

:::interactive_editor

```js
function processInput(input) {
  if (typeof input !== "string") {
    throw new TypeError("Input must be a string.");
  }

  return input.toUpperCase();
}

try {
  console.log("Starting to process input...");
  const result1 = processInput("hello");
  console.log("Processed result:", result1);  // HELLO
  const result2 = processInput(9);  // throws TypeError
  console.log("Processed result:", result2);  // not executed
} catch (error) {
  console.error("Error occurred:", error.message);
} 
```

:::

## Техніки налагодження

- **Інструкція `debugger`**: Ця інструкція дозволяє призупинити виконання коду на певній лінії, щоб дослідити, що відбувається в програмі.

:::interactive_editor

```js
let firstNumber = 5;
let secondNumber = 10;

debugger; // Code execution pauses here
let sum = firstNumber + secondNumber;

console.log(sum);
```

:::

- **Точки зупинки**: Точки зупинки дозволяють призупинити виконання коду на вибраній вами лінії. Після паузи ви можете перевірити змінні, оцінити вирази та дослідити стек викликів.
- **Спостерігачі**: Спостереження за виразами дозволяє відстежувати значення змінних або виразів під час виконання коду, навіть якщо вони поза поточною областю видимості.
- **Профілювання**: Профілювання допомагає виявити вузькі місця продуктивності, дозволяючи робити знімки екрана та записувати використання центрального процесора, виклики функцій і час виконання.
- **`console.dir()`**: Цей метод використовується для відображення інтерактивного списку властивостей заданого об’єкта JavaScript. Він виводить ієрархічний перелік, який можна розгортати, щоб побачити всі вкладені властивості.

```js
console.dir(document);
```

- **`console.table()`**: Цей метод відображає табличні дані у вигляді таблиці в консолі. Він приймає один обов’язковий аргумент, який має бути масивом або об’єктом, і один необов’язковий аргумент для вказівки, які властивості (стовпці) відображати.
