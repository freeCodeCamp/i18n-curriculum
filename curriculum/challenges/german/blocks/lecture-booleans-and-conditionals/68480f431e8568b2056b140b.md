# --description--

In der vorherigen Lektion haben Sie gelernt, wie Sie Vergleichsoperatoren und bedingte Anweisungen verwenden, um den Ablauf Ihrer Programme zu steuern.

Obwohl diese sehr mächtig sind, stoßen Sie oft auf Situationen, in denen Sie mehrere Werte gleichzeitig vergleichen müssen. Dies kann zu verschachtelten bedingten Anweisungen führen, zum Beispiel:

```python
is_citizen = True
age = 25

if is_citizen:
    if age >= 18:
        print('You are eligible to vote') # You are eligible to vote
else:
    print('You are not eligible to vote')
```

Das obige Beispiel prüft zuerst, ob `is_citizen` `True` ist. Falls ja, wird die verschachtelte `if`-Anweisung ausgeführt und überprüft, ob `age` größer oder gleich `18` ist. Da `age` größer oder gleich `18` ist, wird die Nachricht `You are eligible to vote` im Terminal ausgegeben. Wäre `is_citizen` `False` gewesen, hätte die Nachricht `You are not eligible to vote` ausgegeben werden sollen.

Wenn Sie mit komplexeren bedingten Anweisungen arbeiten, können Sie die `and`-, `or`- und `not`-Operatoren von Python verwenden.

Doch bevor wir uns diese Operatoren ansehen, schauen wir uns an, was truthy- und falsy-Werte sind.

In Python hat jeder Wert einen inhärenten Boolean-Wert, also ein eingebautes Verständnis dafür, ob er in einem logischen Kontext als `True` oder `False` behandelt werden sollte. Viele Werte gelten als **truthy**, das heißt, sie werten in einem logischen Kontext zu `True` aus. Andere sind **falsy**, was bedeutet, dass sie zu `False` auswerten.

Hier sind einige falsy-Werte:

- `None`
- `False`
- Ganzzahl `0`
- Fließkommazahl `0.0`
- Leere Strings `""`

Andere Werte wie nicht-null Zahlen und nicht-leere Strings sind truthy.

Wenn Sie prüfen möchten, ob ein Wert truthy oder falsy ist, können Sie die eingebaute `bool()`-Funktion verwenden. Sie wandelt einen Wert explizit in sein Boolean-Äquivalent um und gibt `True` für truthy-Werte und `False` für falsy-Werte zurück. Hier sind einige Beispiele:

```py
print(bool(False)) # False
print(bool(0))  # False
print(bool('')) # False

print(bool(True)) # True
print(bool(1)) # True
print(bool('Hello')) # True
```

Jetzt, wo Sie truthy- und falsy-Werte verstehen, können wir uns die Boolean-Operatoren ansehen, die auch als logische Operatoren bekannt sind. Diese speziellen Operatoren ermöglichen es Ihnen, mehrere Ausdrücke zu kombinieren, um komplexere Entscheidungslogik in Ihrem Code zu erstellen.

Es gibt drei Boolean-Operatoren in Python: `and`, `or` und `not`.

Schauen wir uns zuerst den `and`-Operator an.

Der `and`-Operator nimmt zwei Operanden und gibt den ersten Operanden zurück, wenn dieser falsy ist, andernfalls gibt er den zweiten Operanden zurück. Beide Operanden müssen truthy sein, damit ein Ausdruck zu einem truthy-Wert auswertet.

Hier ist ein Beispiel:

```python
is_citizen = True
age = 25

print(is_citizen and age) # 25
```

Im obigen Beispiel wird die Zahl 25 im Terminal ausgegeben, weil der `and`-Operator den zweiten Operanden auswertet, wenn der erste Operand `True` ist. Der `and`-Operator ist als Short-Circuit-Operator bekannt. Short-Circuiting bedeutet, dass Python Werte von links nach rechts prüft und stoppt, sobald das endgültige Ergebnis feststeht.

Sie werden `and` häufig innerhalb von `if`-Anweisungen verwenden, um zu prüfen, ob mehrere Bedingungen erfüllt sind. So können Sie das frühere Beispiel umschreiben, um den `and`-Operator anstelle verschachtelter `if`-Anweisungen zu verwenden:

```python
is_citizen = True
age = 25

if is_citizen and age >= 18:
    print('You are eligible to vote') # You are eligible to vote
else:
    print('You are not eligible to vote')
```

Im obigen Beispiel ist `is_citizen` `True` und `age >= 18` wertet zu `True` aus. Da beide Operanden des `and`-Operators truthy sind, wertet die Bedingung `is_citizen and age >= 18` zu `True` aus, und der `print`-Aufruf im `if`-Block wird ausgeführt.

Sehen wir uns nun den `or`-Operator an. Dieser Operator gibt den ersten Operanden zurück, wenn er truthy ist, andernfalls den zweiten Operanden. Ein `or`-Ausdruck ergibt einen truthy-Wert, wenn mindestens ein Operand truthy ist. Der `or`-Operator ist ebenfalls als Short-Circuit-Operator bekannt. Hier ist ein Beispiel:

```python
age = 19
is_employed = False

print(age or is_employed) # 19
```

Der folgende Code gibt die Zahl 19 aus, weil der erste Operand `age` truthy ist.

Wenn Sie prüfen müssen, ob ein oder mehrere Ausdrücke `True` sind, können Sie den `or`-Operator in einer Bedingung wie folgt verwenden:

```python
age = 19
is_student = True

if age < 18 or is_student:
    print('You are eligible for a student discount') # You are eligible for a student discount
else:
    print('You are not eligible for a student discount')
```

In diesem Fall ist `age < 18` `False`, aber `is_student` ist `True`. Da mindestens eine Bedingung wahr ist, wertet der gesamte `or`-Ausdruck zu `True` aus, und die Rabattnachricht im `if`-Block wird ausgegeben.

Der letzte Operator, den wir uns ansehen, ist der `not`-Operator, der einen einzelnen Operanden nimmt und dessen Boolean-Wert invertiert. Er wandelt truthy-Werte in `False` und falsy-Werte in `True` um. Im Gegensatz zu den vorherigen Operatoren gibt `not` immer `True` oder `False` zurück.

Hier sind einige Beispiele:

```python
print(not '') # True, because empty string is falsy
print(not 'Hello') # False, because non-empty string is truthy
print(not 0) # True, because 0 is falsy
print(not 1) # False, because 1 is truthy
print(not False) # True, because False is falsy
print(not True) # False, because True is truthy
```

Es ist üblich, den `not`-Operator in Bedingungen zu verwenden, um zu prüfen, ob etwas nicht `True` oder `False` ist, so wie hier:

```python
is_admin = False

if not is_admin:
    print('Access denied for non-administrators.') # Access denied for non-administrators.
else:
    print('Welcome, Administrator!')
```

Da `is_admin` `False` ist, sagt `not is_admin` „nicht `False`“, was `True` ist. Daher wird die Nachricht `Access denied for non-administrators.` ausgegeben.

Jetzt, wo Sie truthy- und falsy-Werte, die `and`-, `or`- und `not`-Operatoren sowie wissen, wie Short-Circuiting funktioniert, können Sie flexiblere und besser lesbare bedingte Logik schreiben.
