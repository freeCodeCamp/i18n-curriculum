# --interactive--

在現代 JavaScript 中，類別就像藍圖，你可以在程式碼中定義它們，以建立多個相同種類的物件。

例如，如果你正在模式化一個庫存系統，你將需要為每種產品型別創建多個物件。

每個獨立的產品都會有自己的屬性和動作，但同種類的產品可以從相同的「藍圖」生成。這就是類別的用途。

類別可以定義為類別運算式或類別宣告式。

以下是類別宣告的基本語法：

```js
class MyClassName {
  // Class Methods
  constructor() { ... }
  method1() { ... }
  method2() { ... }
  ...
}
```

你從撰寫 `class` 關鍵字開始，接著是類別名稱。命名類別時，你應該使用 `PascalCase` 命名慣例，而非 `camelCase`。

大括號內的內容稱為類別本體。這裡是建構式和方法將會放置的地方。

建構式是一個特殊的行為，當從類別產生新物件時會自動被呼叫。它通常用來初始化物件的屬性。

在這個範例中，你可以看到一個帶有建構式的 `Dog` 類別。

```js
class Dog {
  constructor(name) {
    this.name = name;
  }
}
```

這裡使用 `this` 關鍵字來指涉 `Dog` 類別的當前實體，讓你可以將 `name` 屬性指定給正在創建的物件。你將在未來的課程中學習更多關於 `this` 關鍵字的內容。

在建構式下方，你可以擁有所謂的方法。方法是在類別內定義的函式，用來執行動作或操作行為。它們用於定義類別實體可以執行的行為。

這是一個為 `Dog` 類別添加名為 `bark` 的方法的範例。

```js
class Dog {
  constructor(name) {
    this.name = name;
  }

  bark() {
    console.log(`${this.name} says woof!`);
  }
}
```

當 `bark` 方法被呼叫時，訊息將會以傳入的實際名稱記錄到主控台。如果你創建一個名稱為 `"Buddy"` 的 `Dog` 實體，對該實體呼叫 `bark()` 方法將會在主控台列印 `"Buddy says woof!"`。

一旦你定義了你的類別，你就可以使用 `new` 關鍵字、類別名稱及其引數來創建該類別的實體。

在此範例中，我們創建一個名稱為 `Gino` 的新 `Dog` 實體，然後將它指定給變數 `dog`。

```js
const dog = new Dog("Gino");
```

當你在程式碼中寫 `new Dog("Gino")` 時，新的物件會被建立，且建構式會自動以給定的引數被呼叫。

你接著可以使用點記法存取這個屬性，就像你在以下這行程式碼中看到的那樣：

```js
console.log(dog.name);
```

以下是完整範例：

:::interactive_editor

```js
class Dog {
  constructor(name) {
    this.name = name;
  }

  bark() {
    console.log(`${this.name} says woof!`);
  }
}

const dog = new Dog("Gino");
console.log(dog.name);
```

:::

你也可以呼叫方法。在此範例中，我們正在對 `dog` 實體呼叫 `bark()` 方法。

```js
dog.bark();
```

這裡是包含方法呼叫的完整範例：

:::interactive_editor

```js
class Dog {
  constructor(name) {
    this.name = name;
  } 
  bark() {
    console.log(`${this.name} says woof!`);
  }
}

const dog = new Dog("Gino");
console.log(dog.name);
dog.bark();
```

:::

如前所述，你也可以將類別定義為類別表達式。這是指該類別是匿名的，並指定給一個變數。

以下是先前範例作為類別表達式的樣子：

```js
const Dog = class {
  constructor(name) {
    this.name = name;
  }

  bark() {
    console.log(`${this.name} says woof!`);
  }
};
```

大多數時間你會使用類別宣告，但了解類別表達式也很有價值，因為它們能提供更多彈性，特別是在動態程式設計情況下。

透過使用類別並理解其基本語法，你可以創建可重複使用的程式碼，將其組織成邏輯單元，並促進程式碼模組化。
