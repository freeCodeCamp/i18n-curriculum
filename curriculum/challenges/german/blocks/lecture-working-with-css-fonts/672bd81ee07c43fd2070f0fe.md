# --description--

Web-sichere Schriftarten sind eine Teilmenge von Schriftarten, die sehr wahrscheinlich auf einem Computer oder Gerät installiert sind. Sie werden von verschiedenen Betriebssystemen und Webbrowsern breit unterstützt, sodass sie höchstwahrscheinlich konsistent gerendert und angezeigt werden. Schauen wir uns an, wie das funktioniert.

Browser sind dafür verantwortlich, Schriftarten auf einer Website zu interpretieren und anzuzeigen. Wenn der Browser eine Schriftart rendern muss, versucht er, die Schriftartdatei auf dem System des Nutzers zu finden. Wird die Schriftart nicht gefunden, greift er normalerweise auf eine Standardsystemschriftart zurück. So ist sichergestellt, dass der Inhalt weiterhin lesbar bleibt, auch wenn die spezifische Schriftart, die auf der Website gerendert werden soll, fehlt.

Die vom Browser ausgewählte Fallback-Schriftart kann jedoch sehr unterschiedlich aussehen im Vergleich zur ursprünglich vorgesehenen Schriftart. Das kann einen entscheidenden Einfluss auf das Gesamtdesign und die Benutzererfahrung haben. Um das zu vermeiden, sollten Sie wann immer möglich web-sichere Schriftarten verwenden. Sie haben zwei Möglichkeiten: Sie können sie entweder als Ihre primären Schriftarten verwenden oder benutzerdefinierte Schriftarten mit einer web-sicheren Schriftart als Fallback-Option einsetzen. So können Sie steuern, wie die Website aussieht, falls die benutzerdefinierte Schriftart nicht gefunden wird.

Schauen wir uns einige Beispiele für web-sichere Schriftarten an. In einer vorherigen Lektion haben Sie gelernt, dass serifenlose Schriftarten häufig für die Webentwicklung verwendet werden, weil sie keine kleinen „Füßchen“ oder Linien am Ende der Zeichen haben und daher auf dem Bildschirm leicht lesbar sind. Einige Beispiele für web-sichere serifenlose Schriftarten sind:

Im Gegensatz dazu haben Serifenschriften kleine „Füßchen“ am Ende der Zeichen und werden daher häufig für traditionelle Druckerzeugnisse verwendet. Wenn Sie sie jedoch für Webentwicklungszwecke einsetzen müssen, gehören zu den web-sicheren Serifenschriften:

Times New Roman

<img src="https://cdn.freecodecamp.org/curriculum/lecture-transcripts/what-are-web-safe-fonts-5.png" alt="The text 'Hello, World!' displayed in Georgia, a serif font with wider letterforms and stroke contrast optimized for screen readability" />

Durch die Verwendung web-sicherer Schriftarten können Sie sicherstellen, dass Ihr Design auf Geräten und Plattformen konsistent aussieht. Sie können auch die Barrierefreiheit für Nutzer mit Sehbehinderungen verbessern, da sie einfach und gut lesbar sind. Und schließlich können web-sichere Schriftarten die Ladezeit der Seite verkürzen, da sie nicht heruntergeladen werden müssen, wenn sie bereits installiert sind.

Jetzt, da Sie wissen, was web-sichere Schriftarten sind, können Sie sie verwenden und als Fallbacks zuweisen, um verlässliche Benutzererfahrungen über Browser und Plattformen hinweg zu schaffen.
