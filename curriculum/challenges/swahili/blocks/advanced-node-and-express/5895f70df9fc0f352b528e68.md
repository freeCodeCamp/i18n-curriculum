# --description--

Mkakati ni njia ya kuthibitisha mtumizi. Unaweza kutumia mkakati kuruhusu watumizi kuthibitishwa kulingana na taarifa zilizohifadhiwa ndani ya kompyuta (kama umewaandikisha kwanza) au kutoka kwa watoa huduma mbalimbali kama Google au GitHub. Kwa mradi huu, tutatumia programu ya kati ya Passport. Passport hutoa seti kamili ya mikakati inayounga mkono uthibitisho wa utambulisho kwa kutumia jina la mtumizi na nenosiri, GitHub, Google, na zaidi.

`passport-local@~1.0.0` tayari imeongezwa kama utegemezi. Iongeze kwenye seva yako kama ifuatavyo:

```javascript
const LocalStrategy = require('passport-local');
```

Waambie passport **itumiwe** mfano wa `LocalStrategy` ulioanzishwa na mipangilio michache iliyowekwa. Hakikisha hii (pamoja na kila kitu kuanzia hapa) imefungwa ndani ya muunganisho wa hifadhidata kwa sababu inategemea hilo!:

```javascript
passport.use(new LocalStrategy((username, password, done) => {
  myDataBase.findOne({ username: username }, (err, user) => {
    console.log(`User ${username} attempted to log in.`);
    if (err) return done(err);
    if (!user) return done(null, false);
    if (password !== user.password) return done(null, false);
    return done(null, user);
  });
}));
```

Hii inaelezea mchakato wa kutumia unapo jaribu kuthibitisha mtu kwa njia ya ndani. Kwanza, inajaribu kutafuta mtumizi katika hifadhidata yako kwa jina la mtumizi lililoingizwa. Kisha, inakagua kama nenosiri linaendana. Mwishowe, kama hakuna makosa yaliyotokea ambayo umeangalia (mfano nenosiri lisilo sahihi), mfano wa `user` unarudishwa na mtumizi anathibitishwa.

Mikakati mingi huwekwa kwa kutumia mipangilio tofauti. Kwa ujumla, ni rahisi kuianzisha kulingana na README katika hifadhidata ya mkakati huo. Mfano mzuri ni mkakati wa GitHub ambapo huna haja ya kuwa na wasiwasi kuhusu jina la mtumizi au nenosiri kwa sababu mtumizi atapelekwa kwenye ukurasa wa uthibitisho wa GitHub kuthibitisha. Mradi tu wapo wameingia na wanakubali basi GitHub hurudisha wasifu wao kwa ajili yako kutumia.

Katika hatua inayofuata, utaweka jinsi ya kuitisha mkakati wa uthibitisho wa utambulisho kuthibitisha mtumizi kulingana na data ya fomu.

Wasilisha ukurasa wako unapoona umefanya vizuri. Ikiwa unakutana na makosa, unaweza <a href="https://forum.freecodecamp.org/t/advanced-node-and-express/567135#authentication-strategies-6" target="_blank" rel="noopener noreferrer nofollow">kagua mradi uliofanyika hadi hatua hii</a>.
