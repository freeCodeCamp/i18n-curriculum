## String コンストラクタと `toString()` メソッド

- **定義**: 文字列オブジェクトは文字のシーケンスを表すために使われます。文字列オブジェクトは `String` コンストラクタ関数を使って作成され、プリミティブ値をオブジェクトでラップします。

:::interactive_editor

```js
const greetingObject = new String("Hello, world!");

console.log(typeof greetingObject); // "object"
```

:::

- **`toString()` メソッド**: このメソッドは値を文字列の表現に変換します。数値、ブール値、配列、オブジェクトに対して使えるメソッドです。

:::interactive_editor

```js
const num = 10;
console.log(num.toString()); // "10"

const arr = [1, 2, 3];
console.log(arr.toString()); // "1,2,3"
```

:::

このメソッドはオプションの基数を受け入れます。基数は2から36までの数字で、基数は2進数や8進数のような基数を表します。基数が指定されない場合は、デフォルトで10進数（基数10）になります。

:::interactive_editor

```js
const num = 10;
console.log(num.toString(2)); // "1010"(binary)
```

:::

## Number コンストラクタ

- **定義**: `Number` コンストラクタは数値オブジェクトを作成するために使われます。数値オブジェクトには `isNaN` や `toFixed` メソッドのような便利なプロパティやメソッドが含まれています。ほとんどの場合、他のデータ型を数値型に変換するために `Number` コンストラクタを使います。

:::interactive_editor

```js
const myNum = new Number("34");
console.log(typeof myNum); // "object"

const num = Number('100');
console.log(num); // 100

console.log(typeof num); // number
```

:::

## 変数や関数の命名に関するベストプラクティス

- **キャメルケース**: 慣例として、JavaScriptの開発者は変数や関数の命名にキャメルケースを使います。キャメルケースは最初の単語がすべて小文字で、続く単語の先頭が大文字になる命名法です。例：`isLoading`。

- **ブール値の命名**: ブール値の変数には「is」「has」「can」のような接頭辞を使うのが一般的です。

```js
let isLoading = true;
let hasPermission = false;
let canEdit = true;
```

- **関数の命名**: 関数名はその関数が何をするかを明確に示すべきです。ブール値を返す関数（しばしば述語関数と呼ばれる）には「is」「has」「can」の接頭辞を使えます。データを取得する関数には「get」で始めるのが一般的です。データを設定する関数には「set」で始めるのが一般的です。イベントハンドラ関数には「handle」を接頭辞にしたり、「Handler」を接尾辞にしたりします。

```js
function getUserData() { /* ... */ }

function isValidEmail(email) { /* ... */ }

function getProductDetails(productId) { /* ... */ }

function setUserPreferences(preferences) { /* ... */ }

function handleClick() { /* ... */ }
```

- **ループ内の変数命名**: ループのイテレータ変数には `i`、`j`、`k` のような単一文字を使うのが一般的です。

```js
for (let i = 0; i < array.length; i++) { /* ... */ }
```

## スパース配列の扱い方

- **定義**: 空のスロットを持つ配列を作ることができます。空のスロットとは何も入っていないスロットのことです。これは `undefined` の値を持つ配列スロットとは異なります。このような配列はスパース配列と呼ばれます。

:::interactive_editor

```js
const sparseArray = [1, , , 4];
console.log(sparseArray.length); // 4
```

:::

## リンターとフォーマッター

- **リンター**: リンターは静的コード解析ツールで、プログラミングエラー、バグ、スタイルの誤り、疑わしい構造を検出します。一般的なリンターの例は ESLint です。
- **フォーマッター**: フォーマッターはコードを特定のスタイルガイドに従って自動的に整形するツールです。一般的なフォーマッターの例は Prettier です。

## メモリ管理

- **定義**: メモリ管理はメモリを制御し、必要なときに割り当て、不要になったら解放するプロセスです。JavaScriptは自動メモリ管理を使っています。つまり、JavaScript（より正確にはウェブブラウザのJavaScriptエンジン）がメモリの割り当てと解放を自動で行います。コード内で明示的にメモリを解放する必要はありません。この自動プロセスは「ガベージコレクション」と呼ばれます。

## クロージャ

- **定義**: クロージャは、外側の関数が戻った後でも、その外側のレキシカルスコープ内の変数にアクセスできる関数です。

:::interactive_editor

```js
function outerFunction(x) {
  let y = 10;
  function innerFunction() {
    console.log(x + y);
  }
  return innerFunction;
}

let closure = outerFunction(5);
closure(); // 15
```

:::

## `var` キーワードとホイスティング

- **定義**: `var` は2015年以前に変数を宣言するための元々の方法でした。しかし、スコープや再宣言などの問題が `var` にありました。だからこそ、現代のJavaScriptプログラミングでは `let` と `const` を使います。
- **`var` での変数の再宣言**: `let` を使って変数を再宣言しようとすると `SyntaxError` が発生します。しかし `var` では変数の再宣言が許可されています。

```js
// Uncaught SyntaxError: Identifier 'num' has already been declared 
let num = 19;
let num = 18;

var myNum = 5;
var myNum = 10; // This is allowed and doesn't throw an error

console.log(myNum) // 10
```

- **`var` とスコープ**: `var` でブロック内（例えば `if` 文や `for` ループ）で宣言された変数は、そのブロックの外でもアクセス可能です。

:::interactive_editor

```js
if (true) {
  var num = 5;
}
console.log(num); // 5
```

:::

- **ホイスティング**: これはJavaScriptのデフォルトの動作で、コードが実行される前のコンパイル段階で宣言をそれぞれのスコープの先頭に移動させます。`var` キーワードで変数を宣言すると、JavaScriptはその宣言をスコープの先頭にホイストします。

:::interactive_editor

```js
console.log(num); // undefined
var num = 5;
console.log(num); // 5
```

:::

関数宣言構文で関数を宣言すると、関数名と関数本体の両方がホイストされます。つまり、コード内で宣言する前に関数を呼び出すことができます。

:::interactive_editor

```js
sayHello(); // "Hello, World!"

function sayHello() {
  console.log("Hello, World!");
}
```

:::

`let` または `const` で行われた変数宣言はホイストされますが、初期化されず、コード内の実際の宣言より前にアクセスできません。この動作は「一時的デッドゾーン」と呼ばれます。

:::interactive_editor

```js
console.log(num); // Throws a ReferenceError
let num = 10;
```

:::

## インポート、エクスポート、モジュールの扱い方

- **モジュール**: 関連する関数、クラス、変数をまとめた自己完結型のコード単位です。モジュールを作成するには、JavaScriptコードを別のファイルに書きます。
- **エクスポート**: アプリケーションの他の部分で使いたい変数、関数、クラスは `export` キーワードを使って明示的にエクスポートする必要があります。エクスポートには名前付きエクスポートとデフォルトエクスポートの2種類があります。
- **インポート**: エクスポートされたものをアプリケーションの別の部分で使うには、`import` キーワードを使ってインポートします。インポートには名前付きインポート、デフォルトインポート、名前空間インポートがあります。

```js
// Within a file called math.js, we export the following functions:

// Named export
export function add(num1, num2) {
  return num1 + num2;
}

// Default export
export default function subtract(num1, num2) {
  return num1 - num2;
}

// Within another file, we can import the functions from math.js.

// Named import - This line imports the add function.
// The name of the function must exactly match the one exported from math.js.
import { add } from './math.js';

// Default import - This line imports the subtract function.
// The name of the function can be anything.
import subtractFunc from './math.js';

// Namespace import - This line imports everything from the file.
import * as Math from './math.js';

console.log(add(5, 3)); // 8
console.log(subtractFunc(5, 3)); // 2
console.log(Math.add(5, 3)); // 8
console.log(Math.subtract(5, 3)); // 2
```
