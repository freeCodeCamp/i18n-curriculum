## 동적 프로그래밍 소개

- **정의**: 동적 프로그래밍은 복잡한 문제를 더 간단한 하위 문제로 나누고 결과를 저장하여 중복 계산을 피하는 알고리즘 기법입니다. 
- **중복되는 하위 문제**: 더 큰 문제를 해결할 때 동일한 작은 문제가 여러 번 나타납니다. 이 하위 문제를 반복해서 다시 계산하는 대신, 그 해답을 저장합니다.
- **최적 부분 구조**: 문제의 최적 해답은 하위 문제들의 최적 해답을 포함합니다. 즉, 더 작은 부분들의 최적 해답을 결합해 최적 해답을 만들 수 있습니다.

## 동적 프로그래밍 해법

- **메모이제이션 (상향식 접근법)**: 메모이제이션은 비용이 많이 드는 함수 호출 결과를 저장하고, 같은 입력이 다시 발생하면 저장된 결과를 반환합니다.

```js
function climbStairsMemo(n, memo = {}) {
  // If already calculated, return cached value
  if (memo[n] !== undefined) {
    return memo[n];
  }

  // Base cases: 1 way for 1 step, 2 ways for 2 steps
  if (n <= 2) {
    return n;
  }

  // Calculate once and store in memo for future use
  memo[n] =
    climbStairsMemo(n - 1, memo) +
    climbStairsMemo(n - 2, memo);

  return memo[n];
}
```

- **테이블화 (하향식 접근법)**: 테이블화는 하위 문제들의 해답을 채워가며 바닥부터 해답을 구축합니다.

```js
function climbStairsTabulation(n) {
  if (n <= 2) {
    return n;
  }

  // Create an array to store solutions from 0 to n
  const dp = new Array(n + 1).fill(0);

  // Base cases
  dp[1] = 1; // 1 way to reach step 1
  dp[2] = 2; // 2 ways to reach step 2

  // Build up the solution iteratively
  for (let i = 3; i <= n; i++) {
    // Ways to reach step i = ways to reach (i-1) + ways to reach (i-2)
    dp[i] = dp[i - 1] + dp[i - 2];
  }

  return dp[n];
}
```

## 동적 프로그래밍을 활용한 실제 애플리케이션

- **라우트 최적화**: GPS 시스템은 위치 간 최단 경로를 찾기 위해 동적 프로그래밍 알고리즘을 사용합니다.
- **텍스트 처리**: 맞춤법 검사기와 자동 완성 기능은 단어 간 편집 거리를 계산하기 위해 동적 프로그래밍에 의존합니다.
- **금융 모델링**: 투자 전략과 포트폴리오 최적화에 동적 프로그래밍 기법이 자주 활용됩니다.
- **자원 할당**: 배낭 문제와 그 변형은 일정 관리, 예산 편성, 자원 관리에 나타납니다.
