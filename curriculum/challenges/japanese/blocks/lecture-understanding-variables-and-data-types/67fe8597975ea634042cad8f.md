# --description--

Pythonでは、変数は異なる型のデータを格納し参照するためのラベル付きの箱のようなものです。Pythonで変数を宣言するには、代入演算子（`=`）を使って識別子に値を代入します。JavaScriptの`let`や`const`、C#の`char`のような特別なキーワードを使う必要はありません。

Pythonでは、変数名を左に書き、代入演算子を続けて、右側に変数に代入したい値を書くだけです。以下は`name`と`age`の変数を宣言する例です。

```python
name = 'John Doe'
age = 25
```

上の例では、変数`name`に`'John Doe'`という値が格納されています。この値は文字列で、テキストを表すために使われる一連の文字です。文字列はシングルクォートまたは二重引用符で囲んで書きます。例えば`'Hello'`や`"Hello"`です。今後のレッスンでPythonの文字列の扱いについてさらに学びます。

Pythonで変数名を付ける際には、いくつか重要なルールがあります：

- 変数名は文字またはアンダースコア（`_`）で始まらなければなりません。数字で始めることはできません。
- 変数名には英数字（`a-z`、`A-Z`、`0-9`）とアンダースコア（`_`）のみを含めることができます。
- 変数名は大文字と小文字を区別します。`age`、`Age`、`AGE`はすべて異なるものとして扱われます。
- 変数名はPythonの予約キーワード（`if`、`class`、`def`など）を使うことはできません。

これらのルールに違反すると、Pythonプログラムは`SyntaxError`をスローします。

```bash
    5variable_name = 5
     ^
SyntaxError: invalid syntax
```

では、Pythonの変数名の一般的な命名規則を見ていきましょう。

まず、変数名は小文字で書き、単語の区切りはアンダースコアでつなぐべきです。これをスネークケースと呼びます。

```python
my_variable_name = 'freeCodeCamp'
```

次に、変数には説明的な名前を使うべきです。例えばユーザーの年齢を保存する場合、`user_age`は`age`や`ua`のような略語よりも適切です。

```python
user_age = 30
```

こうすることで、大きなコードベースで他のチームメンバーや将来の自分に変数の目的を簡単に伝えられます。

もう一つの慣例は、単一文字の変数名を避けることです。Pythonではよく使われますが、単一文字の変数名は目的や意味を伝えないため避けるべきです。

```python
x = 56 # What do you mean by x?
```

シャープ記号（`#`）とその後のテキストはコメントと呼ばれます。コメントに慣れているかもしれませんが、簡単に説明し仕組みを解説します。

Pythonでは、コメントはシャープ記号（`#`）で始まり、その行の`#`以降のすべての内容は無視されます。

```python
# This is a single-line comment
```

複数行のコメントは連続した単一行コメントで作成できます。

```python
# This is a
# multi-line
# comment
```

コメントはコードの説明や自分へのメモ、なぜその行があるのかを明確にするために使えます。特に学習中やチーム作業で役立ちます。

ただし、変数名の意味を説明するためにコメントを使うべきではありません。代わりに、変数名自体が説明的で目的を伝え、前述の命名ルールに従って構文エラーを防ぐべきです。
