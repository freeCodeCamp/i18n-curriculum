# --interactive--

In der Programmierung ermöglicht Vererbung, Klassen zu definieren, die Eigenschaften und Methoden von anderen Klassen erben.

Zum Beispiel ist ein `Car` eine Art von `Vehicle`, daher könnten Sie eine `Vehicle`-Klasse mit den allgemeinsten Eigenschaften und Methoden definieren, die allen Fahrzeugtypen gemeinsam sind, und dann eine `Car`-Klasse definieren, die die Eigenschaften und Methoden der `Vehicle`-Klasse erbt.

Diese „ist ein“ Beziehung ist charakteristisch für Vererbung. In diesem Beispiel ist `Car` eine spezialisiertere Form von `Vehicle`.

Es gibt zwei Hauptkonzepte, die Sie kennen sollten, um mit Vererbung zu arbeiten: Elternklasse und Kindklasse.

Eine Elternklasse ist eine Klasse, die wie eine Vorlage für andere Klassen fungiert. Sie definiert Eigenschaften und Methoden, die von anderen Klassen geerbt werden.

Eine Kindklasse ist eine Klasse, die die Eigenschaften und Methoden einer anderen Klasse erbt. Kindklassen können die Funktionalität ihrer Elternklassen auch erweitern, indem sie neue Eigenschaften und Methoden hinzufügen.

In unserem vorherigen Beispiel wäre `Vehicle` die Elternklasse der `Car`-Klasse, während die `Car`-Klasse die Kindklasse von `Vehicle` wäre.

Eine Elternklasse kann mehrere Kindklassen haben.

In JavaScript verwenden wir das Schlüsselwort `extends`, um Vererbung zu implementieren. Dieses Schlüsselwort zeigt an, dass eine Klasse die Kindklasse einer anderen Klasse ist.

In diesem Beispiel, in dem wir nur die ersten Zeilen der Klassendefinitionen haben, sehen Sie, dass die `Car`-Klasse die `Vehicle`-Klasse erweitert, sodass `Car` von `Vehicle` erbt.

```js
class Vehicle {
  // Implementation of Vehicle...
}

class Car extends Vehicle {
  // Implementation of Car...
}
```

Angenommen, die `Vehicle`-Klasse hat zwei Eigenschaften: `brand` und `year`. Das sehen Sie im aktualisierten Code.

`Car` hat eine Methode namens `honk`. Beachten Sie jedoch, dass die `Car`-Klasse keinen eigenen Konstruktor hat. Schauen wir, ob sie die Eigenschaften erbt.

```js
class Vehicle {
  constructor(brand, year) {
    this.brand = brand;
    this.year = year;
  }
}

class Car extends Vehicle {
  honk() {
    console.log("Honk! Honk!");
  }
}
```

Um eine Instanz von `Car` zu definieren, müssen Sie zwei Argumente übergeben: `brand` und `year`. In diesem Fall ist die Marke `freeCodeCamp Motors` und das Jahr `2019`.

```js
let myCar = new Car("freeCodeCamp Motors", 2019);
```

Wenn Sie versuchen, mit Punktnotation auf diese Eigenschaften zuzugreifen und die Methode aufzurufen:

```js
console.log(myCar.brand);
console.log(myCar.year);
myCar.honk();
```

Hier ist das vollständige Beispiel:


Tatsächlich ist die Ausgabe korrekt, also sind diese Eigenschaften und die Methode in der `Car`-Instanz definiert, und Sie haben sich viel Codewiederholung erspart, indem Sie diese Eigenschaften von der `Vehicle`-Klasse erben.

In diesem Beispiel hatte die Kindklasse keine zusätzlichen Eigenschaften. Deshalb hatte die Klasse keinen Konstruktor, nur eine Methode.

Wenn Sie zusätzliche Eigenschaften hinzufügen müssen, müssen Sie einen Konstruktor definieren.

Sehen wir uns ein Beispiel an.


```js
class Vehicle {
  constructor(brand, year) {
    this.brand = brand;
    this.year = year;
  }
}

class Car extends Vehicle {
  constructor(brand, year, numDoors) {
    super(brand, year);
    this.numDoors = numDoors;
  }
}
```

Diesmal fügen wir der `numDoors`-Unterklasse eine `Car`-Eigenschaft hinzu.

Sie sehen, dass diese Klasse jetzt im aktualisierten Code einen Konstruktor hat. Innerhalb des Konstruktors gibt es einen Aufruf von `super()`, der die Argumente `brand` und `year` übergibt.

`super()` ruft den Konstruktor der Superklasse auf, sodass Sie durch die Verwendung von `super(brand, year)` im Wesentlichen die Eigenschaften der Superklasse in der Unterklasse definieren. Das ist eine weitere Terminologie, die Sie verwenden können – Superklasse ist die Elternklasse und Unterklasse ist die Kindklasse.

Dann definiert die `this.numDoors = numDoors`-Zeile eine neue Eigenschaft, die ausschließlich zur `Car`-Klasse gehört, nämlich die Anzahl der Türen (`numDoors`).

Das ist eine Möglichkeit, die `Vehicle`-Klasse zu erweitern, indem eine Eigenschaft hinzugefügt wird, die exklusiv für `Car`-Instanzen ist.

Das ist ein Beispiel für Erweiterbarkeit, einen der grundlegenden Vorteile der Vererbung.

Wenn Sie jetzt eine Instanz der `Car`-Klasse erstellen, sehen Sie, dass sie die drei Eigenschaften `brand`, `year` und `numDoors` hat. Sie müssen auch die notwendigen Argumente übergeben, wenn Sie die Instanz erstellen.

```js
let myCar = new Car("freeCodeCamp Motors", 2019, 4);
```

Wenn Sie die Werte dieser Eigenschaften mit `console.log()` ausgeben:

```js
console.log(myCar.brand);
console.log(myCar.year);
console.log(myCar.numDoors);
```

Hier ist das vollständige Beispiel:

:::interactive_editor


Die Hauptvorteile der Vererbung sind Code-Wiederverwendbarkeit, Modularität, Erweiterbarkeit und eine verbesserte Code-Struktur.

Durch die Implementierung einer Hierarchie können Sie Code, den Sie bereits für eine Elternklasse geschrieben haben, in der Kindklasse wiederverwenden und so Wiederholungen vermeiden.

Vererbung fördert auch die Modularität, indem komplexe Systeme in einfachere Komponenten innerhalb der Hierarchie aufgeteilt werden.

Außerdem macht es die Möglichkeit, die Funktionalität der Elternklasse zu erweitern, einfacher, sich an sich ändernde Anforderungen anzupassen und später im Entwicklungsprozess neue Funktionen hinzuzufügen.

Schließlich kann die hierarchische Struktur Ihren Code leichter verständlich und wartbar machen.

Das sind die Grundlagen der Vererbung. Wenn Sie verstehen, wie Vererbung funktioniert, können Sie gut strukturierte, wartbare und erweiterbare objektorientierte Programme entwerfen.
