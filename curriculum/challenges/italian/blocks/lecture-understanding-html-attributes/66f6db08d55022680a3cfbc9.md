# --interactive--

HTML, che sta per Hypertext Markup Language, è un linguaggio di markup per creare pagine web. Quando visiti un sito e vedi contenuti come paragrafi, intestazioni, collegamenti, immagini e video; quello è HTML.

Ecco un piccolo esempio che usa elementi HTML. Prova a modificare un po’ il testo nell’editor e guarda come cambiano i risultati nella finestra di anteprima.

:::interactive_editor

```html
<h1>Main heading element</h1>

<p>I am a paragraph element.</p>
```

:::

HTML rappresenta il contenuto e la struttura di una pagina web tramite l’uso di elementi. La maggior parte degli elementi ha un tag di apertura e un tag di chiusura. A volte questi tag sono chiamati tag di inizio e di fine. Tra questi due tag trovi il contenuto. Questo contenuto può essere testo o altri elementi HTML.

Ecco un altro esempio di un elemento paragrafo. Cambia il testo nell’editor in `I love coding!` e guarda i risultati nella finestra di anteprima.

:::interactive_editor

```html
<p>I am a paragraph element.</p>
```

:::

Sia i tag di apertura che quelli di chiusura iniziano con una parentesi angolare sinistra (`<`) e finiscono con una parentesi angolare destra (`>`), con il nome del tag posizionato tra queste parentesi. Anche se i nomi dei tag HTML non fanno distinzione tra maiuscole e minuscole, è una convenzione molto diffusa e una buona pratica scriverli in minuscolo.

Ecco uno sguardo più da vicino ai soli tag di apertura e chiusura del paragrafo:

```html
<p>
```

```html
</p>
```

Ciò che distingue un tag di apertura da uno di chiusura è la barra obliqua (`/`) posta subito dopo la parentesi angolare sinistra nel tag di chiusura. Alcuni elementi HTML non hanno un tag di chiusura. Questi sono noti come elementi void.

Ecco un esempio di un elemento immagine che è un elemento void:

```html
<img>
```

Nota che questo elemento immagine non ha il tag di chiusura e non ha alcun contenuto. Gli elementi void non possono avere contenuto e hanno solo un tag di apertura.

A volte vedrai elementi void che usano un `/` prima del `>` così:

```html
<img />
```

Anche se molti formatter di codice come Prettier scelgono di includere il `/` negli elementi void, la specifica HTML afferma che la presenza del `/` "non indica che il tag di apertura sia auto-chiudente ma è invece inutile e non ha alcun effetto".

Nello sviluppo reale, vedrai entrambe le forme quindi è importante conoscerle entrambe.

Se vuoi mostrare un’immagine, devi includere un paio di attributi dentro il tuo elemento immagine. Un attributo è un valore speciale usato per regolare il comportamento di un elemento HTML.

Ecco un esempio di un elemento immagine con un attributo `src`. Aggiorna il valore dell’attributo `src` a `"https://cdn.freecodecamp.org/curriculum/cat-photo-app/cats.jpg"` e vedrai l’immagine cambiare in due gatti che dormono pacificamente.

:::interactive_editor

```html
<img src="https://cdn.freecodecamp.org/curriculum/cat-photo-app/running-cats.jpg" />
```

:::

L’attributo `src` serve a specificare la posizione di quell’immagine. Per gli elementi immagine, è buona pratica includere un altro attributo chiamato attributo `alt`. L’attributo `alt` serve a fornire un testo breve e descrittivo per le immagini.

Ecco un esempio di un elemento immagine con gli attributi `src` e `alt`. Prova a rompere l’immagine aggiornando il valore `src` a `"https://.freecodecamp.org/curriculum/cat-photo-app/cats.jpg"`. Vedrai l’immagine sparire e apparire solo il testo `alt`.

:::interactive_editor

```html
<img src="https://cdn.freecodecamp.org/curriculum/cat-photo-app/cats.jpg" alt="Two tabby kittens sleeping together on a couch." />
```

:::

Quindi, ti starai chiedendo se l’HTML da solo sia sufficiente per costruire un sito web. Beh, la risposta è: dipende. Se stai costruendo un piccolo progetto di pratica che mostra solo testo e immagini, l’HTML da solo potrebbe bastare. Però, se stai creando un sito professionale moderno, avrai bisogno di HTML, CSS e JavaScript.

HTML serve per il contenuto e la struttura. CSS serve per lo stile. JavaScript serve per aggiungere interattività alle tue pagine web. Una buona analogia è paragonare HTML, CSS e JavaScript a un edificio completo.

HTML rappresenta i blocchi, il cemento e i ferri che formano le pareti. È la base che rende l’edificio solido. CSS rappresenta il design interno ed esterno che rende la casa bella. JavaScript rappresenta l’impianto elettrico e idraulico che garantisce l’accesso continuo all’acqua e all’elettricità.
