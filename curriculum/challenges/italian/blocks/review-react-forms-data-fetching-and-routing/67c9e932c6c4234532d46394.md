## Lavorare con i form in React

- **Input controllati**: si tratta di quando memorizzi il valore del campo di input nello stato e lo aggiorni tramite eventi `onChange`. Questo ti dà il controllo completo sui dati del form e permette una validazione istantanea e un rendering condizionale.

```jsx
import { useState } from "react";

function App() {
  const [name, setName] = useState("");

  const handleChange = (e) => {
    setName(e.target.value);
  };

  const handleSubmit = (e) => {
    e.preventDefault();
    console.log(name);
  };

  return (
    <>
      <form onSubmit={handleSubmit}>
        <label htmlFor="name">Your name</label> <br />
        <input value={name} id="name" onChange={handleChange} type="text" />
        <button type="submit">Submit</button>
      </form>
    </>
  );
}

export default App;
```

- **Input non controllati**: invece di gestire gli input tramite il hook `useState`, gli input non controllati in HTML mantengono il proprio stato interno grazie al DOM. Poiché il DOM controlla i valori degli input, devi prelevare i valori dei campi di input con un `ref`.

```jsx
import { useRef } from "react";

function App() {
 const nameRef = useRef();

 const handleSubmit = (e) => {
   e.preventDefault();
   console.log(nameRef.current.value);
 };

 return (
   <form onSubmit={handleSubmit}>
     <label htmlFor="name">Your</label>{" "}
     <input type="text" ref={nameRef} id="name" />
     <button type="submit">Submit</button>
   </form>
 );
}

export default App;
```

## Lavorare con il hook `useActionState`

- **Azioni server**: sono funzioni che vengono eseguite sul server per permettere la gestione dei form direttamente sul server senza bisogno di endpoint API. Ecco un esempio tratto da un'applicazione Next.js:

```js
"use server";

async function submitForm(formData) {
 const name = formData.get("name");
 return { message: `Hello, ${name}!` };
}
```

La direttiva `"user server"` indica che la funzione è un'azione server.

- **Hook `useActionState`**: questo hook aggiorna lo stato in base al risultato di un invio del form. Ecco la sintassi base del hook `useActionState`:

```js
const [state, action, isPending] = useActionState(actionFunction, initialState, permalink);
```

- `state` è lo stato corrente che l'azione restituisce.
- `action` è la funzione che attiva l'azione server.
- `isPending` è un booleano che indica se l'azione è in esecuzione o meno.
- Il parametro `actionFunction` è l'azione server stessa.
- `initialState` è il parametro che rappresenta il punto di partenza per lo stato prima che l'azione venga eseguita.
- `permalink` è una stringa opzionale che contiene l'URL unico della pagina che il form modifica.

## Recupero dati in React

- **Opzioni per recuperare dati**: ci sono molti modi diversi per recuperare dati in React. Puoi usare l'API Fetch nativa o strumenti di terze parti come Axios o SWR.
- **Variabili di stato comuni durante il recupero dati**: indipendentemente dal metodo scelto per recuperare dati in React, ci sono alcune parti di stato da tenere sotto controllo. La prima è proprio i dati. La seconda tiene traccia se i dati sono ancora in fase di recupero. La terza è una variabile di stato che cattura eventuali errori che possono verificarsi durante il processo di recupero dati.

```js
const [data, setData] = useState(null);
const [loading, setLoading] = useState(true);
const [error, setError] = useState(null);
```

Poiché il recupero dati è un effetto collaterale, è meglio usare `Fetch API` all'interno di un hook `useEffect`.

```js
useEffect(() => {
  const fetchData = async () => {
    try {
      const res = await fetch("https://jsonplaceholder.typicode.com/posts");
      
      if (!res.ok) {
        throw new Error("Network response was not ok");
      }

      const data = await res.json();
      setData(data);
    } catch (err) {
      setError(err);
    } finally {
      setLoading(false);
    }
  };

  fetchData();
}, []);
```

Poi puoi renderizzare un messaggio di caricamento se il recupero dati non è completo, un messaggio di errore se c'è stato un problema nel recupero, oppure i risultati.

```jsx
if (loading) {
  return <p>Loading...</p>;
}

if (error) {
  return <p>{error.message}</p>;
}

return (
  <ul>
    {data.map((post) => (
      <li key={post.id}>{post.title}</li>
    ))}
  </ul>
);
```

Se vuoi usare Axios, devi installarlo e importarlo:

```bash
npm i axios
```

```js
import axios from "axios";
```

Poi puoi recuperare i dati usando `axios.get`:

```jsx
const [data, setData] = useState(null);
const [loading, setLoading] = useState(true);
const [error, setError] = useState(null);

useEffect(() => {
  const fetchData = async () => {
    try {
      const res = await axios.get(
        "https://jsonplaceholder.typicode.com/users"
      );
      setData(res.data);
    } catch (err) {
      setError(err);
    } finally {
      setLoading(false);
    }
  };

  fetchData();
}, []);
```

Per recuperare dati usando il hook `useSWR`, devi prima installarlo e importarlo.

```bash
npm i swr
```

```js
import useSWR from "swr";
```

Ecco come puoi usare il hook per recuperare dati:

```jsx
import useSWR from "swr";

const fetcher = (url) => fetch(url).then((res) => res.json());

const FetchTodos = () => {
 const { data, error } = useSWR(
   "https://jsonplaceholder.typicode.com/todos",
   fetcher
 );

 if (!data) {
   return <h2>Loading...</h2>;
 }
 if (error) {
   return <h2>Error: {error.message}</h2>;
 }

 return (
   <>
     <h2>Todos</h2>
     <div>
       {data.map((todo) => (
         <h3 key={todo.id}>{todo.title}</h3>
       ))}
     </div>
   </>
 );
};

export default FetchTodos;
```

## Lavorare con il hook `useOptimistic`

- **Hook `useOptimistic`**: questo hook serve a mantenere le interfacce utente responsivi mentre si aspetta che un'azione asincrona termini in background. Aiuta a gestire gli "aggiornamenti ottimistici" nell'interfaccia, una tecnica in cui fornisci aggiornamenti immediati basati sull'esito previsto di un'azione, come l'attesa di una risposta dal server.

Ecco la sintassi base:

```js
const [optimisticState, addOptimistic] = useOptimistic(actualState, updateFunction);
```

- `optimisticState` è lo stato temporaneo che si aggiorna subito per migliorare l'esperienza utente.
- `addOptimistic` è la funzione che applica l'aggiornamento ottimistico prima che lo stato reale cambi.
- `actualState` è il valore reale dello stato che deriva dal risultato di un'azione, come il recupero dati da un server.
- `updateFunction` è la funzione che determina come lo stato ottimistico deve aggiornarsi quando viene chiamata.

Ecco un esempio di utilizzo del hook `useOptimistic` in un componente `TaskList`:

```jsx
"use client";

import { useOptimistic } from "react";

export default function TaskList({ tasks, addTask }) {
  const [optimisticTasks, addOptimisticTask] = useOptimistic(
    tasks,
    (state, newTask) => [...state, { text: newTask, pending: true }]
  );

  async function handleSubmit(e) {
    e.preventDefault();
    const formData = new FormData(e.target);

    addOptimisticTask(formData.get("task"));

    addTask(formData);
    e.target.reset();
  }

  return <>{/* UI */}</>;
}
```

- **`startTransition`**: serve a renderizzare una parte dell'interfaccia e a marcare un aggiornamento di stato come una transizione non urgente. Questo permette all'interfaccia di rimanere responsiva durante aggiornamenti costosi. Ecco la sintassi base:

```js
startTransition(action);
```

`action` esegue un aggiornamento di stato o attiva una logica legata alla transizione. Questo assicura che aggiornamenti urgenti dell'interfaccia (come digitare o cliccare) non vengano bloccati.

## Lavorare con il hook `useMemo`

- **Memoizzazione**: è una tecnica di ottimizzazione in cui il risultato di chiamate di funzione costose viene memorizzato in cache in base a specifici argomenti. Quando gli stessi argomenti vengono forniti di nuovo, viene restituito il risultato memorizzato invece di ricalcolare la funzione.
- **Hook `useMemo`**: questo hook serve a memoizzare valori calcolati. Ecco un esempio di memoizzazione del risultato di un ordinamento di un grande array. `expensiveSortFunction` verrà eseguito solo quando `largeArray` cambia:

```js
const memoizedSortedArray = useMemo(
  () => expensiveSortFunction(largeArray),
  [largeArray]
);
```

## Strumenti per la gestione delle dipendenze

- **Definizione di dipendenza**: nel software, una dipendenza è quando un componente o modulo in un'applicazione si basa su un altro per funzionare correttamente. Le dipendenze sono comuni nelle applicazioni software perché permettono agli sviluppatori di usare funzioni o strumenti predefiniti creati da altri. Le due dipendenze principali necessarie per un progetto React sono i pacchetti `react` e `react-dom`:

```json
"dependencies": {
  "react": "^18.3.1",
  "react-dom": "^18.3.1"
}
```

- **Definizione di package manager**: per gestire le dipendenze software in un progetto, devi usare un package manager. Un package manager è uno strumento usato per installare, aggiornare e rimuovere dipendenze. Molti linguaggi di programmazione popolari come JavaScript, Python, Ruby e Java usano package manager. I package manager più diffusi per JavaScript includono npm, Yarn e pnpm.
- **File `package.json`**: è un file di configurazione chiave nei progetti che contiene metadati sul progetto, inclusi nome, versione e dipendenze. Definisce anche script, informazioni sulla licenza e altre impostazioni che aiutano a gestire il progetto e le sue dipendenze.
- **File `package-lock.json`**: questo file blocca le versioni esatte di tutti i pacchetti usati nel progetto. Quando aggiorni un pacchetto, le nuove versioni vengono aggiornate anche nel file di lock.
- **Cartella `node_modules`**: questa cartella contiene il codice effettivo delle dipendenze elencate nel file `package.json`, incluse sia le dipendenze dirette del progetto sia quelle delle dipendenze stesse.
- **Dipendenze di sviluppo**: sono pacchetti usati solo per lo sviluppo e non in produzione. Un esempio è una libreria di test come Jest. Installeresti Jest come dipendenza di sviluppo perché serve per testare l'applicazione localmente ma non è necessaria in produzione.

## React Router

- **Introduzione**: React Router è una libreria di terze parti che permette di aggiungere il routing alle tue applicazioni React. Per iniziare, devi installare React Router in un progetto React esistente così:

```bash
npm i react-router
```

Poi, all'interno del file `main.jsx` o `index.jsx`, devi configurare la struttura delle rotte così:

```jsx
import { StrictMode } from "react";
import { createRoot } from "react-dom/client";
import { BrowserRouter, Routes, Route } from "react-router";
import App from "./App.jsx";

import "./index.css";

createRoot(document.getElementById("root")).render(
  <StrictMode>
    <BrowserRouter>
      <Routes>
        <Route path="/" element={<App />} />
      </Routes>
    </BrowserRouter>
  </StrictMode>
);
```

`path` e `element` servono ad associare l'URL ai componenti dell'interfaccia. In questo caso, stiamo configurando una rotta per la homepage che punta al componente `App`.

- **Più viste e configurazione delle rotte**: è comune in applicazioni più grandi avere più viste e rotte configurate così:

```jsx
<Routes>
  <Route index element={<Home />} />
  <Route path="about" element={<About />} />

  <Route path="products">
    <Route index element={<ProductsHome />} />
    <Route path=":category" element={<Category />} />
    <Route path=":category/:productId" element={<ProductDetail />} />
    <Route path="trending" element={<Trending />} />
  </Route>
</Routes>
```

La prop `index` in questi esempi rappresenta la rotta predefinita per un dato segmento di percorso. Quindi il componente `Home` sarà mostrato al percorso radice `/` mentre il componente `ProductsHome` sarà mostrato al percorso `/products`.

- **Annidare rotte**: puoi annidare rotte dentro altre rotte, il che fa sì che il percorso della rotta figlio venga aggiunto al percorso della rotta genitore.

```jsx
<Route path="products">
  <Route path="trending" element={<Trending />} />
</Route>
```

Nell'esempio sopra, il percorso per i prodotti di tendenza sarà products/trending.

- **Segmenti dinamici**: un segmento dinamico è quando una parte qualsiasi del percorso URL è dinamica.

```jsx
<Route path=":category" element={<Category />} />
```

In questo esempio abbiamo un segmento dinamico chiamato `category`. Quando un utente naviga a un URL come products/brass-instruments, la vista cambierà al componente `Category` e potrai recuperare dinamicamente i dati appropriati in base al segmento.

- **Hook `useParams`**: questo hook serve a consultare i parametri dinamici da un percorso URL.

```jsx
import { useParams } from "react-router";

export default function Category() {
  let params = useParams();
  {/* Accessing the category param: params.category */}
  {/* rest of code goes here */}
}
```

## Prop drilling

- **Definizione**: il prop drilling è il processo di passare props da un componente genitore a componenti figli profondamente annidati, anche quando alcuni di questi figli non hanno bisogno delle props.

## Gestione dello stato

- **Context API**: il contesto si riferisce a quando un componente genitore rende informazioni disponibili ai componenti figli senza doverle passare esplicitamente tramite props. `createContext` serve a creare un oggetto contesto che rappresenta il contesto che altri componenti leggeranno. `Provider` serve a fornire i valori del contesto ai componenti figli.

```jsx
import { useState, createContext } from "react";

const CounterContext = createContext();

const CounterProvider = ({ children }) => {
  const [count, setCount] = useState(0);

  return (
    <CounterContext.Provider value={{ count, setCount }}>
      {children}
    </CounterContext.Provider>
  );
};

export { CounterContext, CounterProvider };
```

- **Redux**: Redux gestisce lo stato fornendo un archivio centrale e un controllo rigoroso sugli aggiornamenti di stato. Usa un pattern prevedibile con azioni, reducer e middleware. Le azioni sono payload di informazioni che inviano dati dall'applicazione all'archivio Redux, spesso attivate da interazioni utente. I reducer sono funzioni che specificano come lo stato deve cambiare in risposta a quelle azioni, assicurando che lo stato venga aggiornato in modo immutabile. Il middleware, invece, agisce da ponte tra il dispatch delle azioni e il reducer, permettendo di estendere le funzionalità di Redux (ad esempio logging, gestione di operazioni asincrone) senza modificare il flusso principale.

- **Zustand**: questa soluzione per la gestione dello stato è ideale per applicazioni di piccole e medie dimensioni. Funziona usando un hook `useStore` per consultare lo stato direttamente in componenti e pagine. Questo ti permette di modificare e consultare dati senza bisogno di azioni, reducer o provider.

- **React Developer Tools**: è un'estensione del browser che puoi usare in Chrome, Firefox e Edge per ispezionare i componenti React e identificare problemi di prestazioni. Per Safari, devi installare il pacchetto npm `react-devtools`. Dopo aver installato React DevTools e aperto un'app React nel browser, apri gli strumenti per sviluppatori del browser per accedere alle due schede extra fornite per il debugging di React – Componenti e Profiler.
- **Scheda Componenti**: questa scheda mostra ogni componente in formato ad albero. Ecco alcune cose che puoi fare in questa scheda:
  - vedere la gerarchia dei componenti dell'app
  - controllare e modificare props, stati e valori di contesto in tempo reale

## React Server Components

- **Definizione**: i React Server Components sono componenti React che vengono renderizzati esclusivamente sul server, inviando solo l'HTML finale al client. Questo significa che quei componenti possono accedere direttamente a risorse server-side e ridurre drasticamente la quantità di JavaScript inviata al browser.
