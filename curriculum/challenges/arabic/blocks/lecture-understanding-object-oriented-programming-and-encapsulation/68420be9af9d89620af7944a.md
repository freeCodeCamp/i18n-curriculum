# --description--

البرمجة الكائنية التوجه، المعروفة أيضًا بـ OOP، هي أسلوب برمجي يعامل فيه المطوِّرون كل شيء في شفرتهم ككائن من العالم الحقيقي.

الصنف يشبه المخطط لإنشاء الكائنات. كل كائن يُنشأ من صنف يحتوي على خصائص تُعرّف البيانات وطرق تُعرّف سلوكيات الكائنات.

في درس سابق، تعلمت كيفية إنشاء الأصناف. إليك تذكيرًا بالصِيغة النحوية:

```py
class ClassName:
   def __init__(self, parameters):
       attribute = value

   def method_name(self):
       # method logic
```

إليك مثالًا على صنف يستخدم الطريقة الخاصة `__init__` لتهيئة خاصيتي العلامة التجارية واللون في كل مرة يُنشأ فيها كائن باستخدام الصنف:

```py
class Car:
   def __init__(self, brand, color):
       self.brand = brand
       self.color = color

# create two objects from the Car class
car1 = Car('Toyota', 'red')
car2 = Car('Lambo', 'green')

print('Car 1 Brand:', car1.brand) # Car 1 Brand: Toyota
print('Car 1 Color:', car1.color) # Car 1 Color: red

print('Car 2 Brand:', car2.brand) # Car 2 Brand: Lambo
print('Car 2 Color:', car2.color) # Car 2 Color: green
```

البرمجة الكائنية التوجه لها أربعة مبادئ رئيسية تساعدك على تنظيم وإدارة الشفرة بفعالية. وهي التغليف، الوراثة، التعددية الشكلية، والتجريد.

سيركز بقية هذا الدرس على كيفية عمل التغليف.

التغليف هو تجميع الخصائص والطرق الخاصة بالكائن في وحدة واحدة، وهي الصنف.

مع التغليف، يمكنك إخفاء الحالة الداخلية للكائن خلف مجموعة بسيطة من الطرق والخصائص العامة التي تعمل كأبواب. خلف هذه الأبواب توجد خصائص وطرق خاصة تتحكم في كيفية تغير البيانات ومن يمكنه رؤيتها.

لنفترض أنك تريد تتبع رصيد محفظة. تريد السماح للناس بإيداع أو سحب المال من المحفظة، لكن لا يجب لأحد التلاعب بالرصيد مباشرة.

في هذه الحالة، يمكنك جعل `deposit()` و `withdraw()` طرقًا عامة، وتخفي الرصيد تحت الخاصية `_balance`:

```py
class Wallet:
   def __init__(self, balance):
       self._balance = balance # For internal use by convention

   def deposit(self, amount):
       if amount > 0:
           self._balance += amount # Add to the balance safely

   def withdraw(self, amount):
       if 0 < amount <= self._balance:
           self._balance -= amount # Remove from the balance safely
```

بموجب الاصطلاح، بادئة الخاصيات والطرق بشرطة سفلية واحدة تعني أنها مخصصة للاستخدام الداخلي. لا يجب لأحد الوصول إليها مباشرة من خارج الصنف لأنها تخالف مبادئ التغليف، مما قد يؤدي إلى أخطاء برمجية.

بينما بادئة الشرطة السفلية الواحدة مجرد اصطلاح، فإن بادئة الخاصيات والطرق بشرطتين سفليتين تمنع فعليًا الوصول إليها من خارج الصنف، مما يجعل تلك الخاصيات والطرق خاصة.

```py
class Wallet:
   def __init__(self, balance):
       self.__balance = balance # Private attribute

   def deposit(self, amount):
       if amount > 0:
           self.__balance += amount # Add to the balance safely

   def withdraw(self, amount):
       if 0 < amount <= self.__balance:
           self.__balance -= amount # Remove from the balance safely

account = Wallet(500)
print(account.__balance) # AttributeError: 'Wallet' object has no attribute '__balance'
```

للحصول على القيمة الحالية لـ `__balance`، يمكنك تعريف طريقة `get_balance`. على سبيل المثال:

```py
class Wallet:
   def __init__(self, balance):
       self.__balance = balance

   def deposit(self, amount):
       if amount > 0:
           self.__balance += amount

   def withdraw(self, amount):
       if 0 < amount <= self.__balance:
           self.__balance -= amount
  
   def get_balance(self):
       return self.__balance


acct_one = Wallet(100)
acct_one.deposit(50)
print(acct_one.get_balance()) # 150

acct_two = Wallet(450)
acct_two.withdraw(28)
print(acct_two.get_balance()) # 422

acct_two.deposit(150)
print(acct_two.get_balance()) # 572
```

يمكنك أيضًا تعريف طريقة خاصة `__validate` للتحقق من أن كل مبلغ إيداع أو سحب هو عدد موجب:

```py
class Wallet:
   def __init__(self):
       self.__balance = 0

   def __validate(self, amount):
       if amount < 0:
           raise ValueError('Amount must be positive')

   def deposit(self, amount):
       self.__validate(amount)
       self.__balance += amount

   def withdraw(self, amount):
       self.__validate(amount)
       if amount > self.__balance:
           raise ValueError('Insufficient funds')
       self.__balance -= amount

   def get_balance(self):
       return self.__balance

acct_one = Wallet()
acct_one.deposit(3)
print(acct_one.get_balance()) # 3

acct_one.deposit(50)
print(acct_one.get_balance()) # 53

acct_one.deposit(-4)  # ValueError: Amount must be positive
acct_one.withdraw(-8) # ValueError: Amount must be positive
acct_one.withdraw(58) # ValueError: Insufficient funds
```

كما ترى، الطريقة `__validate` خاصة، وتعمل خلف الكواليس في الطرق العامة `deposit()` و `withdraw()` لضمان أن المبلغ دائمًا صالح.

في درس قادم، ستتعلم المزيد عن كيفية عمل الخاصيات التي تبدأ بشرطتين سفليتين.

باختصار، التغليف يقفل البيانات الداخلية خلف طرق عامة واضحة. هكذا تحمي أصنافك من التلاعب وتُركّز التحقق في مكان واحد. يمكنك تحديث أو توسيع الشفرة بحرية، مع العلم أن الشفرة الخارجية تتعامل فقط مع الواجهات التي تتيحها.
