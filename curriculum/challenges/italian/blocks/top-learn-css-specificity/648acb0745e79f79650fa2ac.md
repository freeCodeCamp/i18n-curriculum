# --description--

L’ultimo fattore, la fine della linea, il decisore finale del decisore finale. Supponiamo che, dopo aver considerato tutti gli altri fattori, ci siano ancora più regole in conflitto che puntano a uno stesso elemento. Come fa la cascata a determinare quale regola applicare?

In realtà è molto semplice. Vince la regola che è stata definita per ultima.

```css
/* styles.css */

.alert {
  color: red;
}

.warning {
  color: yellow;
}
```

Per un elemento che ha sia la classe `alert` sia la classe `warning`, la cascata esaminerebbe tutti gli altri fattori, inclusa l’ereditarietà (qui assente) e la specificità (nessuna delle due regole è più specifica dell’altra). Poiché la regola `.warning` è stata definita per ultima, e nessun altro fattore è riuscito a stabilire quale regola applicare, quella è quella che viene applicata all’elemento.
