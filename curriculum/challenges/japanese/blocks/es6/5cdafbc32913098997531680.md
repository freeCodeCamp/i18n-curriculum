# --description--

Promise には `pending`、`fulfilled`、`rejected` の3つの状態があります。前のチャレンジで作成した Promise は、完了させる方法を追加しなかったため、永遠に `pending` 状態のままです。Promise の引数に渡される `resolve` と `reject` は、この完了を行うために使われます。`resolve` は Promise を成功させたいときに使い、`reject` は失敗させたいときに使います。これらは引数を取るメソッドで、以下のように使います。

```js
const myPromise = new Promise((resolve, reject) => {
  if(condition here) {
    resolve("Promise was fulfilled");
  } else {
    reject("Promise was rejected");
  }
});
```

上の例ではこれらの関数の引数に文字列を使っていますが、実際には何でもかまいません。多くの場合はオブジェクトで、それを使ってウェブサイトや他の場所にデータを表示したりします。

# --instructions--

Promise が成功と失敗の両方を処理するようにしてください。もし `responseFromServer` が `true` なら、`resolve` メソッドを呼び出して Promise を成功させてください。`resolve` に `We got the data` という値の文字列を渡します。もし `responseFromServer` が `false` なら、代わりに `reject` メソッドを使い、文字列 `Data not received` を渡してください。
