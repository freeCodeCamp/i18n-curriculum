# --description--

Bashとその基本的なコマンドについて学びましょう。

BashはBourne Again SHellの略で、Unix系環境で最もよく使われるシェルの一つです。そのため、このシェルの操作方法やよく使われるコマンドを理解することは非常に役立ちます。

最初のコマンドは`pwd`で、現在の作業ディレクトリをターミナルに出力します。「作業ディレクトリ」とは、ターミナルが現在指しているディレクトリのことで、多くの他のコマンドを使う上で重要な情報です。

ターミナルが間違ったディレクトリを指している可能性もあります。`cd`コマンドを使うとディレクトリを変更できます。絶対パスはスラッシュ（`/`）で始まり、相対パスは接頭辞なしで指定し、`..`構文を使うと親ディレクトリに移動できます。

次のコマンドは`ls`で、現在の作業ディレクトリの内容を一覧表示します。ディレクトリにファイルやフォルダが存在するか確認するのに便利です。`ls`コマンドは、隠しファイルを表示する`-a`フラグ、ファイルの権限を表示する`-l`フラグなどのオプションも使えます。コマンドのフラグについては次のレッスンで詳しく学びます。ファイルの内容を見るには`less`や`cat`も使えます。

もしディレクトリに必要なファイルやフォルダがなければ、`mkdir`や`touch`というコマンドで作成できます。`mkdir`は新しいディレクトリ（フォルダ）を作成し、`touch`は新しいファイルを作成します。どちらのコマンドも作成したいファイルやフォルダの名前を指定します。例えば、`touch readme.md`は`readme.md`という名前の新しいMarkdownファイルを作成します。

新しいファイルの名前が`readme.md`ではなく`readthis.md`にすべきだった場合もあります。幸い、`mv`コマンドを使うとファイルを移動（または名前変更）できます。古いファイル名と新しいファイル名を指定します。

`rm`コマンドはファイルを削除でき、`-r`フラグを付けるとディレクトリも削除できます。ファイルが保護されている場合は`-f`フラグを付ける必要があります。

`cp`コマンドはファイルやディレクトリを新しい場所にコピーできます。`mv`コマンドとは違い、`cp`コマンドは元のファイルを削除しません。ディレクトリをコピーするには`-r`フラグを付ける必要があります。

`echo`コマンドはBashでの`console.log()`や`print()`関数に相当すると考えられます。`echo`は文字列の引数を引用符で囲んで受け取り、ターミナルに出力します。ターミナルで文字列を出力するだけなので一見無意味に思えますが、`echo`を制御記号に連結できます。`>`記号は`echo "Naomi was here." > readme.md`のように新しいファイルを作成または上書きし、`>>`記号はファイルに追記します。

他にも`head`、`tail`、`chown`、`chmod`など多くのBashコマンドがありますが、このレッスンですべてを扱うのは不可能です。ほとんどのコマンドのマニュアルやヘルプページは`man`コマンドで確認できます。

よく使うコマンドは自然と覚えられます。その他のコマンドはいつでもmanページで調べてください。
