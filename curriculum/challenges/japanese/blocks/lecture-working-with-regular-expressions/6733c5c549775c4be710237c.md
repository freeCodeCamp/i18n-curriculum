# --description--

文字列内のパターンのすべての出現箇所をマッチまたは置き換える方法を学びましょう。

これまでに`replace()`と`match()`メソッド、およびグローバル`g`修飾子について学びました。これらの知識を組み合わせて、文字列内のすべてのパターンを扱えるようになります。まずは元のマッチコードを思い出しましょう。

```js
const regex = /freeCodeCamp/;
const match = "freeCodeCamp".match(regex);
console.log(match);
```

そして、得られたマッチオブジェクトはこちらです。

```js
// [
//   'freeCodeCamp',
//   index: 0,
//   input: 'freeCodeCamp',
//   groups: undefined
// ]
```

しかし、もし文字列に複数の`freecodecamp`の出現箇所があったらどうなるでしょうか？`match()`がどのように動作するか見てみましょう。比較のために以前の`replace()`の例も入れてみます。

```js
const regex = /freecodecamp/;
const str = "freecodecamp is the best we love freecodecamp";
const matched = str.match(regex);
const replaced = str.replace(regex, "freeCodeCamp");
console.log(matched);
console.log(replaced);
```

結果はこうなりました。

```js
// [
//   'freecodecamp',
//   index: 0,
//   input: 'freecodecamp is the best we love freecodecamp',
//   groups: undefined
// ]
// freeCodeCamp is the best we love freecodecamp
```

あらら！`match()`は最初のマッチだけを返し、`replace()`は最初のマッチだけを置き換えました。これは、デフォルトでは`match()`と`replace()`が最初のパターンの出現箇所に対してのみ動作するためです。

幸いにも、正規表現にグローバル修飾子を使うことでこれを回避できます。さっそく追加してみましょう。

```js
const regex = /freecodecamp/g;
const str = "freecodecamp is the best we love freecodecamp";
const matched = str.match(regex);
const replaced = str.replace(regex, "freeCodeCamp");
console.log(matched);
console.log(replaced);
```

結果を確認します。

```js
// [ 'freecodecamp', 'freecodecamp' ]
// freeCodeCamp is the best we love freeCodeCamp
```

うまくいきました！replaceの呼び出しは小文字の`freecodecamp`文字列すべてを置き換え、`match()`メソッドは両方をマッチさせました。

ここで興味深いのは、グローバル修飾子を`match()`に使うと、マッチ配列に含まれるキャプチャグループや文字列のインデックスに関する追加情報が失われることです。

しかし、2019年のECMAScriptアップデートで`matchAll()`と`replaceAll()`という2つの新しいメソッドが導入されました。これらは単数形のメソッドと同様に文字列または正規表現を受け取り、`replaceAll()`は置き換え用の文字列を第2引数として受け取ります。

ただし、以前のメソッドとは異なり、`replaceAll()`と`matchAll()`はグローバル修飾子なしの正規表現を渡すとエラーをスローします。これらの新しいメソッドを使うようにコードを更新しましょう。

```js
const pattern = "freecodecamp";
const str = "freecodecamp is the best we love freecodecamp";
const matched = str.matchAll(pattern);
const replaced = str.replaceAll(pattern, "freeCodeCamp");
console.log(matched);
console.log(replaced);
```

結果はこちらです。

```js
// {}
// freeCodeCamp is the best we love freeCodeCamp
```

良い知らせです！`replaceAll()`は期待通りに動作し、小文字の`freecodecamp`のすべての出現箇所を正しいキャメルケースのバージョンに置き換えました。

では、この空のオブジェクトは何でしょう？実は`matchAll()`は`Iterator`と呼ばれる特別なタイプのオブジェクトを返しますが、freeCodeCampコンソールはこれを処理できません。

ブラウザのコンソールを覗くと、`Iterator`には`next()`メソッドがあり、これを呼び出すと次の値を取得できます。

```js
// RegExpStringIterator {  }
//  <prototype>: RegExp String Iterator {
//    next: ƒ next(),
//    Symbol(Symbol.toStringTag): "RegExp String Iterator"
//    <prototype>: Object { ... }
//  }
```

さっそく`matched.next()`を呼び出して結果をログに出してみましょう。

```js
// {
//   "done": false,
//   "value": [
//     0: "freecodecamp"
//     groups: undefined
//     index: 0
//     input: "freecodecamp is the best we love freecodecamp"
//   ]
// }
```

これがマッチ配列です！`next()`は2つの値を持つオブジェクトを返します。`done`はイテレーターにさらに要素がある場合は`false`で、`value`は直前に反復した値です。では、もう一度呼び出してみましょう。

```js
const regex = /freecodecamp/g;
const str = "freecodecamp is the best we love freecodecamp";
const matched = str.matchAll(regex);
const replaced = str.replaceAll(regex, "freeCodeCamp");
console.log(matched);
console.log(replaced);
console.log(matched.next());
console.log(matched.next());
```

```js
// {
//   "done": false,
//   "value": Array [ "freecodecamp"]
// }
//
// {
//   "done": false,
//   "value": Array [ "freecodecamp"]
// }
```

なぜ`done`がまだ`false`のままなのでしょう？配列には2つのマッチしかないはずですよね？3回目を呼び出してみて結果を見てみましょう。

```js
// {
//   "done": false,
//   "value": Array [ "freecodecamp"]
// }
//
// {
//   "done": false,
//   "value": Array [ "freecodecamp"]
// }
//
// {
//   "done": true,
//   "value": undefined
// }
```

`done`はついに`true`になりましたが、なぜその値が`undefined`なのでしょう？実は`matchAll()`イテレーターは遅延評価です。一度にすべてのマッチを見つけるわけではありません。`next()`を呼び出して初めてマッチを見つけます。

マッチが見つかる限り`done`ではありません。マッチが見つからず`undefined`を返すと`done`になります。この動作は不便に思えるかもしれませんが、正規表現の計算コストが高い場合には非常に役立ちます。

今回のように計算コストが低い例では、この機能をスキップしてイテレーターを配列に変換し、一度にすべてのマッチを取得できます。これは`Array.from()`を呼び出してイテレーターを引数に渡すことで実現します。

コードを更新してこれを使いましょう。`replaceAll`の呼び出しは動作がわかっているので整理します。

```js
const regex = /freecodecamp/g;
const str = "freecodecamp is the best we love freecodecamp";
const matched = str.matchAll(regex);
console.log(Array.from(matched));
```

ついにマッチの配列が得られました。

```js
// [
//   'freecodecamp',
//   index: 0,
//   input: 'freecodecamp is the best we love freecodecamp',
//   groups: undefined
// ]
//
// [
//   'freecodecamp',
//   index: 33,
//   input: 'freecodecamp is the best we love freecodecamp',
//   groups: undefined
// ]
```

これらの強力なメソッドは、パフォーマンスや可読性を犠牲にせずに文字列からデータを操作・抽出するのに役立ちます。
