# --description--

Il middleware per la gestione degli errori in Express è un tipo speciale di middleware usato per gestire gli errori che si verificano durante il ciclo richiesta-risposta. È fondamentale per intercettare errori imprevisti, registrarli e fornire risposte appropriate al client.

Come hai visto in una lezione precedente sulla gestione degli errori, il middleware di Express per la gestione degli errori ha quattro parametri: `err`, `req`, `res` e `next`. Questo middleware viene invocato quando un errore viene passato alla funzione `next()`.

Ecco un promemoria della sintassi:

```js
app.use((err, req, res, next) => {
  console.error(err.message);
  res.status(500).send("Internal Server Error");
});
```

Ecco come appare in una app Express di base:

```js
const express = require('express')
const app = express()

// A simple route that throws an error
app.get('/', (req, res) => {
  throw new Error('Something went wrong!')
})

// Error-handling middleware
app.use((err, req, res, next) => {
  console.error(err.stack); // Logs the error stack
  res.status(500).send('Something went wrong!')
})

app.listen(3000, () => {
  console.log('Server running on http://localhost:3000')
})
```

* Gestore della rotta: il gestore della rotta lancia intenzionalmente un errore per simulare un problema.
* Middleware per la gestione degli errori: questo middleware è registrato con `app.use()`. Intercetta gli errori passati tramite `next(err)` e invia una risposta HTTP 500 al client con un messaggio di errore generico.
* `err`: questo è l’oggetto errore che viene passato al middleware.

Il middleware per la gestione degli errori garantisce che quando qualcosa va storto nella tua applicazione, gli utenti ricevano comunque una risposta adeguata. È importante includere sempre questo middleware alla fine della catena di middleware per intercettare eventuali errori.
