# --interactive--

선택적 체이닝 연산자(`?.`)는 JavaScript에서 객체 속성에 안전하게 접근하거나 메서드를 호출할 때 해당 속성이나 메서드가 존재하는지 걱정하지 않아도 되도록 도와주는 유용한 도구입니다. 마치 누락된 부분이 있을 수 있는 객체를 다룰 때 안전망 역할을 합니다.

:::interactive_editor

```js
const person = {
  name: "Alice",
  age: 30
};

console.log(person.name); // "Alice"
console.log(person.job); // undefined
```

:::

이 예제에서 `person.name`은 존재하므로 `Alice`를 출력합니다. 하지만 `person.job`은 존재하지 않아서 `undefined`를 반환합니다.

이제 존재하지 않을 수도 있는 객체의 속성에 접근하고 싶다고 가정해 봅시다:

```js
const person = {
  name: "Alice",
  age: 30
};

console.log(person.address.street); // This will throw an error!
```

이 예제는 `Uncaught TypeError`를 발생시킵니다. `person.address`이 `undefined`이기 때문에 `street` 속성에 접근할 수 없습니다. 이럴 때 선택적 체이닝 연산자가 유용합니다. 선택적 체이닝 연산자를 사용하는 예는 다음과 같습니다:

:::interactive_editor

```ts
const user = {
  name: "John",
  profile: {
    email: "john@example.com",
    address: {
      street: "123 Main St",
      city: "Somewhere"
    }
  }
};

console.log(user?.profile?.address?.street); // "123 Main St"
console.log(user?.profile?.phone?.number);   // undefined
```

:::

선택적 체이닝 연산자를 사용하면 객체(또는 `?.` 앞의 값)가 존재하고 `null`이나 `undefined`이 아닐 때만 연산을 계속하라고 JavaScript에 지시하는 것입니다.

만약 `?.` 앞의 값이 `null`이나 `undefined`라면, JavaScript는 연산을 진행하려고 시도해 오류를 발생시키는 대신 `undefined`를 반환합니다.

선택적 체이닝 연산자는 속성이나 메서드가 존재하는지 확실하지 않을 때 가장 유용합니다. 오류를 방지하고 코드를 더 견고하게 만듭니다.
