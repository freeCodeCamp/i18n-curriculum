# --interactive--

## HTMLの基本

- **HTMLの役割**：HTMLはウェブページのコンテンツと構造を表します。
- **HTML要素**：要素はHTMLドキュメントの構成要素です。見出し、段落、リンク、画像などを表します。ほとんどのHTML要素は開始タグ（`<elementName>`）と終了タグ（`</elementName>`）で構成されています。

基本的な構文は以下の通りです：

```html
<elementName>Content goes here</elementName>
```

- **空要素**：空要素は内容を持たず、開始タグのみを持ちます。例として`img`や`meta`要素があります。

```html
<img>
<meta>
```

空要素の中にスラッシュ`/`を含むコードベースもよく見られます。どちらも許容されます：

```html
<img>
<img/>
```

- **属性**：属性はHTML要素の開始タグ内に配置される値です。属性は要素に関する追加情報を提供したり、要素の動作を指定したりします。属性の基本的な構文は以下の通りです：

```html
<element attribute="value"></element>
```

ブール属性は、HTMLタグ内に存在するかしないかのどちらかで、その存在がtrue、存在しなければfalseを表します。ブール属性の例には`disabled`、`readonly`、`required`があります。

- **コメント**：コメントはプログラミングで自分や他の開発者のためにコード内にメモを残すために使います。HTMLのコメント構文は以下の通りです：

```html
<!--This is an HTML comment.-->
```

## よく使われるHTML要素

- **見出し要素**：HTMLには6つの見出し要素があります。`h1`から`h6`までの見出し要素は、その下のコンテンツの重要度を示します。数字が小さいほど重要度が高く、`h2`要素は`h1`要素より重要度が低いです。

:::interactive_editor

```html
<h1>most important heading element</h1>
<h2>second most important heading element</h2>
<h3>third most important heading element</h3>
<h4>fourth most important heading element</h4>
<h5>fifth most important heading element</h5>
<h6>least important heading element</h6>
```

:::

- **段落要素**：ウェブページの段落に使います。

:::interactive_editor

```html
<p>This is a paragraph element.</p>
```

:::

- **`img`要素**：`img`要素はウェブページに画像を追加するために使います。`src`属性はその画像の場所を指定します。画像要素には`alt`属性という別の属性を含めるのがベストプラクティスです。以下は`img`要素に`src`と`alt`属性を付けた例です：

:::interactive_editor

```html
<img src="https://cdn.freecodecamp.org/curriculum/cat-photo-app/lasagna.jpg" alt="A slice of lasagna on a plate.">
```

:::

- **`body`要素**：この要素はHTMLドキュメントのコンテンツを表します。

:::interactive_editor

```html
<body>
  <h1>CatPhotoApp</h1>
  <p>This is a paragraph element.</p>
</body>    
```

:::

- **`section`要素**：この要素はコンテンツを小さなセクションに分割するために使います。

:::interactive_editor

```html
<section>
  <h2>About Me</h2>
  <p>Hi, I am Jane Doe and I am a web developer.</p>
</section>
```

:::

- **`div`要素**：この要素は意味論的な意味を持たない汎用のHTML要素です。他のHTML要素をまとめるための汎用コンテナとして使います。

:::interactive_editor

```html
<div>
  <h1>I am a heading</h1>
  <p>I am a paragraph</p>
</div>
```

:::

- **アンカー（`a`）要素**：これらの要素はウェブページにリンクを適用するために使います。`href`属性はユーザーがクリックしたときにリンク先を指定します。

:::interactive_editor

```html
<a href="https://cdn.freecodecamp.org/curriculum/cat-photo-app/running-cats.jpg">cute cats</a>
```

:::

- **順不同リスト（`ul`）と番号付きリスト（`ol`）要素**：箇条書きリストを作成するには、`ul`要素の中に1つ以上の`li`要素をネストして使います。

:::interactive_editor

```html
<ul>
  <li>catnip</li>
  <li>laser pointers</li>
  <li>lasagna</li>
</ul>
```

:::

番号付きリストを作成するには、`ol`要素を使います：

:::interactive_editor

```html
<ol>
  <li>flea treatment</li>
  <li>thunder</li>
  <li>other cats</li>
</ol>
```

:::

- **強調（`em`）要素**：テキストの一部に強調を付けるために使います。

:::interactive_editor

```html
<p>Cats <em>love</em> lasagna.</p>  
```

:::

- **強い重要性（`strong`）要素**：緊急性や重大さを示すためにテキストに強い強調を付けるために使います。

:::interactive_editor

```html
<p>
  <strong>Important:</strong> Before proceeding, make sure to wear your safety goggles. 
</p>
```

:::

- **`figure`と`figcaption`要素**：`figure`要素は画像や図などのコンテンツをグループ化するために使います。`figcaption`要素は`figure`要素内のそのコンテンツのキャプションを表します。

:::interactive_editor

```html
<figure>
  <img src="https://cdn.freecodecamp.org/curriculum/cat-photo-app/cats.jpg" alt="Two tabby kittens sleeping together on a couch.">
  <figcaption>Cats <strong>hate</strong> other cats.</figcaption>  
</figure>
```

:::

- **`main`要素**：この要素はウェブページのメインコンテンツを表します。
- **`footer`要素**：この要素はHTMLドキュメントの下部に置かれ、通常は著作権情報や重要なページリンクを含みます。

:::interactive_editor

```html
<footer>
  <p>
    No Copyright - <a href="https://www.freecodecamp.org">freeCodeCamp.org</a>
  </p>
</footer>
```

:::

## 識別子とグループ化

- **ID**：HTML要素の一意の識別子です。ID名はHTMLドキュメント内で1回だけ使う必要があります。

```html
<h1 id="title">Movie Review Page</h1>
```

ID名にスペースは使えません。複数の単語を含む場合は単語間にダッシュを使えます：

```html
<div id="red-box"></div>
```

- **クラス**：スタイルや動作のために要素をグループ化するために使います。

```html
<div class="box"></div>
```

IDとは異なり、同じクラス名をHTMLドキュメント内で何度でも使えます。`class`の値はスペースを含めることもできます：

```html
<div class="box red-box"></div>
<div class="box blue-box"></div>
```

## 特殊文字とリンク

- **HTMLエンティティ**：HTMLエンティティまたは文字参照は、HTMLで予約文字を表すために使う文字のセットです。例としてアンパサンド（`&amp;`）記号や小なり記号（`&lt;`）があります。

```html
<p>This is an &lt;img /&gt; element</p>
```

- **`link`要素**：この要素はスタイルシートやサイトアイコンなどの外部リソースにリンクするために使います。外部CSSファイルに`link`要素を使う基本構文は以下の通りです：

```html
<link rel="stylesheet" href="./styles.css" />
```

`rel`属性はリンクされたリソースとHTMLドキュメントの関係を指定します。`href`属性は外部リソースのURLの場所を指定します。

- **`script`要素**：この要素は実行可能なコードを埋め込むために使います。

```html
<body>
  <script>
    alert("Welcome to freeCodeCamp");
  </script>
</body>
```

技術的にはすべてのJavaScriptコードを`script`タグ内に書くこともできますが、外部JavaScriptファイルにリンクするのがベストプラクティスです。外部JavaScriptファイルにリンクするために`script`要素を使う例は以下の通りです：

```html
<script src="path-to-javascript-file.js"></script>
```

`src`属性は外部JavaScriptファイルの場所を指定します。

## ボイラープレートとエンコーディング

- **HTMLボイラープレート**：これはすべてのHTMLドキュメントに必要な基本構造と必須要素を含むボイラープレートです。

:::interactive_editor

```html
<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="utf-8" />
    <title>freeCodeCamp</title>
    <link rel="stylesheet" href="./styles.css" />
  </head>
  <body>
    <!--Headings, paragraphs, images, etc. go inside here-->
  </body>
</html>
```

:::

- **`DOCTYPE`**：ブラウザに使用しているHTMLのバージョンを伝えるために使います。
- **`html`要素**：HTMLドキュメントの最上位要素またはルートを表します。ドキュメントの言語を指定するには`lang`属性を使います。
- **`head`要素**：`head`セクションはブラウザや検索エンジンに必要な裏側の重要なメタデータを含みます。
- **`meta`要素**：これらの要素はサイトのメタデータを表します。文字エンコーディングやTwitterのようなウェブサイトがページのリンクをどのようにプレビューするかなどの詳細を含みます。
- **`title`要素**：ブラウザのタブやウィンドウに表示されるテキストを設定するために使います。
- **UTF-8文字エンコーディング**：UTF-8（UCS変換フォーマット8）はウェブで広く使われている標準化された文字エンコーディングです。文字エンコーディングはコンピューターが文字をデータとして保存する方法です。`charset`属性は`meta`要素内で文字エンコーディングをUTF-8に設定するために使います。

## SEOとソーシャル共有

- **SEO**：検索エンジン最適化はウェブページを最適化して検索エンジンでより目立ち、上位にランクされるようにする手法です。
- **メタ（`description`）要素**：ウェブページの短い説明を提供し、SEOに影響を与えるために使います。

```html
<meta
  name="description"
  content="Discover expert tips and techniques for gardening in small spaces, choosing the right plants, and maintaining a thriving garden."
/>
```

- **オープングラフタグ**：オープングラフプロトコルはFacebookやLinkedInなどのさまざまなソーシャルメディアプラットフォームでウェブサイトのコンテンツの表示方法を制御できます。これらのオープングラフプロパティを設定することで、ユーザーがクリックしてコンテンツに関わりたくなるように誘導できます。これらのプロパティはHTMLの`meta`セクション内の複数の`head`要素で設定できます。
- **`og:title`プロパティ**：ソーシャルメディア投稿に表示されるタイトルを設定するために使います。

```html
<meta content="freeCodeCamp.org" property="og:title" />
```

- **`og:type`プロパティ**：`type`プロパティはソーシャルメディアで共有されるコンテンツの種類を表します。例として記事、ウェブサイト、動画、音楽などがあります。

```html
<meta property="og:type" content="website" />
```

- **`og:image`プロパティ**：ソーシャルメディア投稿に表示される画像を設定するために使います。

```html
<meta
  content="https://cdn.freecodecamp.org/platform/universal/fcc_meta_1920X1080-indigo.png"
  property="og:image"
/>
```

- **`og:url`プロパティ**：ユーザーがソーシャルメディア投稿でクリックするURLを設定するために使います。

```html
<meta property="og:url" content="https://www.freecodecamp.org" />
```

## メディア要素と最適化

- **置き換えられた要素**：置き換えられた要素はCSS自体ではなく外部リソースによって内容が決まる要素です。例として`iframe`要素があります。`iframe`はインラインフレームの略で、HTMLページ内に他のHTMLコンテンツを直接埋め込むためのインライン要素です。

```html
<iframe src="https://www.example.com" title="Example Site"></iframe>
```

ユーザーがiframeを全画面表示できるようにする`allowfullscreen`属性を含めることができます。

```html
<iframe
  src="video-url"
  width="width-value"
  height="height-value"
  allowfullscreen
></iframe>
```

`iframe`内に動画を埋め込むには、YouTubeやVimeoなどの人気動画サービスから直接コピーするか、`src`属性で動画のURLを指定して自分で定義できます。以下はYouTubeのfreeCodeCampの人気コースを埋め込んだ例です：

:::interactive_editor

```html
<h1>A freeCodeCamp YouTube Video Embedded with the iframe Element</h1>

<iframe
  width="560"
  height="315"
  src="https://www.youtube.com/embed/PkZNo7MFNFg?si=-UBVIUNM3csdeiWF"
  title="YouTube video player"
  allow="accelerometer; autoplay; clipboard-write; encrypted-media; gyroscope; picture-in-picture; web-share"
  referrerpolicy="strict-origin-when-cross-origin"
  allowfullscreen
></iframe>
```

:::

他にも`video`や`embed`などの置き換えられた要素があります。また特定の状況下で置き換えられた要素として振る舞う要素もあります。以下は`input`要素に`type`属性を`image`に設定した例です：

```html
<input type="image" alt="Descriptive text goes here" src="example-img-url">
```

- **メディアの最適化**：画像などのメディアを使う際に考慮すべき3つのポイントはサイズ、フォーマット、圧縮です。圧縮アルゴリズムはファイルやデータのサイズを小さくするために使います。
- **画像フォーマット**：PNGやJPGはよく使われるファイルフォーマットですが、もはや画像配信に最適なフォーマットではありません。古いブラウザのサポートが必要ない限り、WEBPやAVIFのようなより最適化されたフォーマットを検討してください。
- **画像のライセンス**：パブリックドメインの画像は著作権がなく、制限なく自由に使えます。Creative Commons 0ライセンスの画像はパブリックドメインとみなされます。Creative CommonsライセンスやfreeCodeCampが使うBSDライセンスのような寛容なライセンスで公開されている画像もあります。
- **SVG**：スケーラブルベクターグラフィックスはパスや方程式に基づいて点、線、曲線をプロットするデータを追跡します。つまり、SVGのようなベクターグラフィックは品質を損なうことなく任意のサイズに拡大縮小できます。

## マルチメディア統合

- **`audio`と`video`要素**：`audio`と`video`要素はHTMLドキュメントに音声や動画コンテンツを追加できます。`audio`要素はmp3、wav、oggなどの一般的な音声フォーマットをサポートします。`video`要素はmp4、ogg、webmフォーマットをサポートします。

```html
<audio src="CrystalizeThatInnerChild.mp3"></audio>
```

ページ上にオーディオプレーヤーを表示したい場合は、`audio`要素に`controls`属性を追加できます：

:::interactive_editor

```html
<audio src="CrystalizeThatInnerChild.mp3" controls></audio>
```

:::

`controls`属性は音声の再生管理（音量調整、一時停止、再開など）をユーザーに可能にします。`controls`属性は組み込みの再生コントロールを有効にするブール属性です。省略するとコントロールは表示されません。

- **`loop`属性**：`loop`属性は音声を連続再生させるブール属性です。

:::interactive_editor

```html
<audio
  src="https://cdn.freecodecamp.org/curriculum/js-music-player/can't-stay-down.mp3"
  loop
  controls
></audio>
```

:::

- **`muted`属性**：`audio`要素に存在すると、`muted`ブール属性は音声をミュート状態で開始します。

:::interactive_editor

```html
<audio
  src="https://cdn.freecodecamp.org/curriculum/js-music-player/can't-stay-down.mp3"
  loop
  controls
  muted
></audio>
```

:::

- **`source`要素**：音声ファイルの種類によってブラウザの対応が異なります。これに対応するため、`source`要素内に複数の`audio`要素を使い、ブラウザは理解できる最初のソースを選択します。以下は`source`要素に複数の`audio`要素を使う例です：

```html
<audio controls>
  <source src="audio.ogg" type="audio/ogg" />
  <source src="audio.wav" type="audio/wav" />
  <source src="audio.mp3" type="audio/mpeg" />
</audio>
```

これまで学んだすべての属性は`video`要素でもサポートされています。以下は`video`要素に`loop`、`controls`、`muted`属性を使った例です：

:::interactive_editor

```html
<video
  src="https://archive.org/download/BigBuckBunny_124/Content/big_buck_bunny_720p_surround.mp4"
  loop
  controls
  muted
></video>
```

:::

- **`poster`属性**：動画がダウンロード中に表示する画像を指定したい場合、`poster`属性を使えます。この属性は`audio`要素にはなく、`video`要素に固有です。

:::interactive_editor

```html
<video
  src="https://archive.org/download/BigBuckBunny_124/Content/big_buck_bunny_720p_surround.mp4"
  loop
  controls
  muted
  poster="https://peach.blender.org/wp-content/uploads/title_anouncement.jpg?x11217"
  width="620"
></video>
```

:::

## target属性の種類

- **`target`属性**：この属性はアンカー要素のURLをどこで開くかブラウザに指示します。重要な4つの値は`_self`、`_blank`、`_parent`、`_top`です。5つ目の値`_unfencedTop`は現在実験的な`FencedFrame` APIで使われています。今のところ使う機会はほとんどないでしょう。
- **`_self`値**：`target`属性のデフォルト値です。リンクを現在の閲覧コンテキストで開きます。ほとんどの場合、現在のタブやウィンドウです。

```html
<a href="https://freecodecamp.org" target="_self">Visit freeCodeCamp</a>
```

- **`_blank`値**：リンクを新しい閲覧コンテキストで開きます。通常は新しいタブですが、ユーザーによっては新しいウィンドウで開く設定にしている場合もあります。

```html
<a href="https://freecodecamp.org" target="_blank">Visit freeCodeCamp</a>
```

- **`_parent`値**：リンクを現在のコンテキストの親で開きます。例えば、ウェブサイトにiframeがある場合、そのiframe内の`_parent`値はウェブサイトのタブやウィンドウで開き、埋め込みフレーム内では開きません。

```html
<a href="https://freecodecamp.org" target="_parent">Visit freeCodeCamp</a>
```

- **`_top`値**：リンクを最上位の閲覧コンテキストで開きます。つまり「親の親」のようなものです。`_parent`に似ていますが、ネストされた埋め込みフレームでも常にブラウザのフルタブやウィンドウで開きます。

```html
<a href="https://freecodecamp.org" target="_top">Visit freeCodeCamp</a>
```

## 絶対パスと相対パス

- **パスの定義**：パスはファイルシステム内のファイルやディレクトリの場所を指定する文字列です。ウェブ開発では、画像、スタイルシート、スクリプト、他のウェブページなどのリソースにリンクするために使います。
- **パスの構文**：知っておくべき3つの構文があります。1つ目はスラッシュで、OSによってバックスラッシュ（`\`）かフォワードスラッシュ（`/`）です。2つ目はシングルドット（`.`）、3つ目はダブルドット（`..`）です。スラッシュは「パス区切り文字」と呼ばれ、フォルダやファイル名の間の区切りを示します。シングルドットは現在のディレクトリを指し、ダブルドットは親ディレクトリを指します。

```md
public/index.html
./favicon.ico
../src/index.css
```

- **絶対パス**：絶対パスはリソースへの完全なリンクです。ルートディレクトリから始まり、すべてのディレクトリ、最後にファイル名と拡張子を含みます。「ルートディレクトリ」とは階層の最上位のディレクトリやフォルダを指します。絶対パスにはプロトコル（`http`、`https`、`file`など）や、リソースがウェブ上にある場合はドメイン名も含まれます。以下はfreeCodeCampのロゴにリンクする絶対パスの例です：

:::interactive_editor

```html
<a href="https://design-style-guide.freecodecamp.org/img/fcc_secondary_small.svg">
  View fCC Logo
</a>
```

:::

- **相対パス**：相対パスは現在のファイルのディレクトリから見たファイルの場所を指定します。プロトコルやドメイン名は含まれず、同じウェブサイト内の内部リンクに対して短く柔軟に使えます。以下は同じフォルダ内の`about.html`ページから`contact.html`ページにリンクする例です：

```html
<p>
  Read more on the
  <a href="about.html">About Page</a>
</p>
```

## リンクの状態

- **`:link`**：これはデフォルトの状態です。ユーザーがまだ訪問、クリック、操作していないリンクを表します。この状態はページ上のすべてのリンクの基本スタイルを提供すると考えられます。他の状態はこれに上乗せされます。
- **`:visited`**：ユーザーがリンク先のページをすでに訪問した場合に適用されます。デフォルトではリンクが紫色になりますが、CSSを活用して別の視覚的な表示に変更できます。
- **`:hover`**：ユーザーがリンク上にカーソルをホバーしているときに適用されます。リンクに注目を集め、ユーザーがクリックする意図があることを示すのに役立ちます。
- **`:focus`**：リンクにフォーカスが当たっているときに適用されます。
- **`:active`**：ユーザーがリンクをアクティブにしているときに適用されます。通常はマウスの左クリックでリンクをクリックすることを意味します。
