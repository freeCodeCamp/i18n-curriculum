# --description--

`README` 檔案是許多專案中重要的部分，特別是在軟體開發中。當人們查看專案時，它通常是第一個看到的文件。`README` 檔案說明專案的內容、如何使用它，以及有時如何為它做出貢獻。在專案文件的背景關係中，了解 `README` 檔案的用途至關重要。

# --questions--

## --text--

在專案中，`README` 檔案通常用來做什麼？

## --answers--

儲存程式碼

### --feedback--

雖然程式碼是專案的重要部分，`README` 檔案並不是用來儲存程式碼，而是用來提供關於專案的資訊。

---

關於專案的說明

---

聯絡資訊

### --feedback--

即使可以包含聯絡資訊，`README` 檔案的主要目的是提供關於專案的指示和資訊，而不僅僅是聯絡資料。

---

變更記錄

### --feedback--

變更記錄的日誌，通常稱為 changelog，通常與 `README` 檔案分開。`README` 檔案專注於介紹和說明專案。
