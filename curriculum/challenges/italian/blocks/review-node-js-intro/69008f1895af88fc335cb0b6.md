## JavaScript lato client

- **JavaScript lato client**: codice che viene eseguito sul lato client di un'applicazione web, solitamente in un browser web.
- **Limitazioni di JavaScript lato client**: pur essendo potente, JavaScript lato client ha alcune limitazioni:
  - Accesso molto limitato ai file locali.
  - Non progettato per gestire logiche complesse di applicazioni.
  - Potenziali problemi di sicurezza (ad esempio, esporre informazioni sensibili come le credenziali del database).
- **Motivo delle limitazioni**: queste limitazioni esistono perché JavaScript lato client è stato inizialmente progettato per essere eseguito esclusivamente nei browser web.

## Ambiente del browser

- **Motore JavaScript**: i browser web forniscono l'ambiente necessario per eseguire codice JavaScript, incluso il motore JavaScript.
- **Document Object Model (DOM)**: i browser permettono di consultare il DOM, consentendo di accedere agli elementi HTML nel codice.
- **Contesto storico**: in passato, JavaScript non poteva essere eseguito al di fuori di un browser, ma ora è possibile con Node.js.

## Cos'è Node.js?

- **Node.js**: un ambiente di esecuzione JavaScript. Definito ufficialmente come "un ambiente di esecuzione JavaScript open source e multipiattaforma".
- **Open source**: il codice di Node.js è pubblicamente disponibile e mantenuto da una vasta comunità di sviluppatori.
- **Multipiattaforma**: funziona su qualsiasi sistema operativo, inclusi Windows, macOS e Linux.
- **Ambiente di esecuzione JavaScript**: permette di eseguire codice JavaScript al di fuori di un browser.

## Casi d'uso di Node.js

- **Applicazioni comuni**: Node.js è usato in molti settori per:
  - Costruire server web e API che gestiscono richieste HTTP.
  - Sviluppare applicazioni web e mobile.
  - Gestire dati complessi e interazioni con database.
- **Adozione nel settore**: se usi o sviluppi un sito web o un'applicazione web che interagisce con un database o gestisce dati complessi, è molto probabile che Node.js venga usato dietro le quinte.
- **Caratteristiche chiave**: Node.js è efficiente, scalabile e ha una grande comunità di sviluppatori e manutentori in tutto il mondo.

## Browser vs. ambiente di esecuzione Node

- **Ambiente browser**: progettato principalmente per lo sviluppo front-end, esegue JavaScript lato client.
- **Ambiente di esecuzione Node**: progettato principalmente per lo sviluppo back-end, esegue JavaScript lato server.
- **Differenze API**: l'ambiente determina le API disponibili:
  - **Browser**: accesso all'API DOM, ma con restrizioni per consultare il file system locale.
  - **Node**: accesso quasi a tutte le risorse di sistema, incluso il file system, ma non al DOM.

## Oggetti globali

- **Oggetto globale del browser (`window`)**: fornisce accesso a funzionalità legate al browser:
  - Metodi per manipolare il DOM.
  - Gestione dei cookie.
  - Gestione degli eventi del browser.

- **Oggetto globale di Node.js (`global`)**: fornisce accesso a funzionalità specifiche di Node.js:
  - Moduli integrati per lavorare con file locali.
  - Capacità di rete.
  - Interazione con il sistema operativo.

## Controllo delle versioni

- **Versione del browser**: non hai controllo sulla versione dell'ambiente browser che gli utenti useranno per visitare il tuo sito.
- **Versione di Node.js**: puoi scegliere la versione di Node.js da usare nel tuo server.

## Importanza di Node.js

- **Sviluppo full-stack**: Node.js ha trasformato lo sviluppo web permettendo agli sviluppatori di usare JavaScript sia per il front-end sia per il back-end delle applicazioni full-stack.
- **Efficienza**: il processo di sviluppo è più efficiente perché non serve imparare un nuovo linguaggio di programmazione solo per il back-end.
- **Impatto**: Node.js è uno strumento potente, scalabile e versatile che è diventato molto importante per gli sviluppatori in tutto il mondo.

## Vantaggi di Node.js

- **Un solo linguaggio per full-stack**: puoi implementare sia il front-end sia il back-end di un'applicazione web usando JavaScript.
  - Riduce la curva di apprendimento iniziale.
  - Non serve passare da un linguaggio di programmazione all'altro.
  - Aumenta efficienza e produttività.

- **Architettura non bloccante e basata su eventi**: ottima per sviluppare applicazioni in tempo reale dove reattività ed efficienza sono essenziali per una buona esperienza utente.

- **Thread singolo e event loop**: può gestire efficacemente un gran numero di richieste simultanee e operazioni di input-output.
  - Perfetto per applicazioni che richiedono la gestione di molte richieste contemporaneamente.

- **Grande comunità**: migliaia di sviluppatori in tutto il mondo mantengono costantemente Node.js e aggiungono nuove funzionalità.
  - Molte risorse di apprendimento disponibili.
  - Facile trovare risposte alle domande.

- **npm (Node Package Manager)**: uno strumento potente che permette di installare e gestire pacchetti e moduli per i tuoi progetti.
  - Pacchetti disponibili per molti scopi: gestire richieste HTTP, lavorare con database, formattare date e orari, testare codice e altro.
  - Riutilizzo di codice scritto, testato e condiviso da altri sviluppatori.
  - Rende il flusso di lavoro più veloce e efficiente.

- **Economico**: Node.js è gratuito e open source.
  - Anche pacchetti, librerie e strumenti associati sono principalmente gratuiti.
  - Fattore importante quando si ottimizza il budget.

## Svantaggi di Node.js

- **Limitazione del thread singolo**: esegue un solo thread alla volta, quindi può gestire un'operazione alla volta.
  - Anche se l'architettura permette di gestire efficientemente richieste concorrenti (simultanee), operazioni intensive per la CPU possono bloccare il thread principale e causare problemi di prestazioni.
  - Esempi di operazioni intensive per la CPU: operazioni matematiche complesse, elaborazione di immagini e video, crittografia.
  - Esistono modi per superare questo limite, ma di solito aumentano la complessità dell'applicazione.

- **Programmazione asincrona**: Node.js si basa sulla programmazione asincrona.
  - Un'attività che può richiedere molto tempo viene avviata, ma invece di aspettare che finisca, il programma principale continua a girare mentre l'attività asincrona procede in parallelo.
  - Quando l'attività è completata, il programma gestisce il risultato.
  - Spesso coinvolge "callback", cioè funzioni che definiscono cosa succede quando le operazioni asincrone sono completate.
  - Può rendere il codice più difficile da leggere, capire e eseguire il debug.

- **Qualità dei pacchetti**: scegliendo pacchetti da npm, alcuni potrebbero non essere mantenuti costantemente e introdurre vulnerabilità nell'applicazione.
  - Gli sviluppatori dovrebbero valutare attentamente ogni pacchetto.
  - Verificare che i pacchetti seguano buone pratiche di qualità e sicurezza.

## Thread

- **Thread**: un percorso di esecuzione all'interno di un processo, come un programma per computer che sta girando.

## NVM (Node Version Manager)

- **NVM**: uno strumento per gestire più versioni di Node.js sullo stesso computer.
- **Scopo**: ti aiuta a passare facilmente tra diverse versioni di Node.js per progetti diversi.
- **Raccomandazione**: si consiglia di installare NVM prima di Node.js per maggiore flessibilità futura.

## Installare NVM su macOS/Linux

- **Xcode Command Line Tools (macOS)**: devono essere installati manualmente prima di eseguire lo script di installazione di NVM su macOS.

```bash
xcode-select --install
```

- **Verifica installazione di Xcode**:

```bash
xcode-select --version
```

- **Scarica ed esegui lo script di installazione di NVM**: esegui uno di questi comandi:

```bash
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.3/install.sh | bash
```

```bash
wget -qO- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.3/install.sh | bash
```

- **Riavvio del terminale**: potresti dover riavviare il terminale al termine del processo per caricare il nuovo comando e la configurazione.

- **Verifica installazione di NVM**:

```bash
nvm -v
```

## Installare Node.js con NVM su macOS/Linux

- **Installa l'ultima versione LTS**: per installare l'ultima versione Long-Term Support (LTS) di Node.js:

```bash
nvm install --lts
```

- **LTS (Long-Term Support)**: questa versione è solitamente raccomandata perché privilegia stabilità, affidabilità e sicurezza rispetto a funzionalità nuove o sperimentali.
  - Ampiamente testata e pronta per la produzione.
  - Garantisce periodi di supporto più lunghi con focus su correzioni di bug e patch di sicurezza.

- **Installa versione specifica**: puoi installare una versione specifica di Node.js specificando il numero di versione:

```bash
nvm install <version>
```

Esempio:

```bash
nvm install 20
```

- **Passa a una versione**:

```bash
nvm use <version>
```

Esempio:

```bash
nvm use 20
```

- **Controlla la versione corrente di Node.js**:

```bash
node -v
```

## Installare NVM su Windows

- **Supporto ufficiale NVM**: le versioni ufficiali di NVM supportano Windows solo in casi specifici.
  - Se hai Windows Subsystem for Linux (WSL), puoi seguire la guida per macOS/Linux sopra.

- **nvm-windows**: un'utilità di gestione versioni Node.js per Windows.
  - Progetto completamente separato da quello ufficiale di NVM.
  - Stesso scopo: aiutarti a lavorare con più versioni di Node.js sullo stesso dispositivo.

- **Prima dell'installazione**: si consiglia di disinstallare eventuali installazioni precedenti di Node prima di installare NVM per Windows.

- **Passaggi di installazione**:
  - Vai alla sezione Releases del repository ufficiale: `https://github.com/coreybutler/nvm-windows/releases`.
  - Scarica l'installer più recente.
  - Segui le istruzioni.

- **Riavvio del terminale**: potresti dover riavviare il terminale se il comando `nvm` non è subito disponibile.

- **Verifica installazione**:

```powershell
nvm -v
```

## Comandi base di NVM e Node.js

- **Nota sui comandi**: tutti i comandi funzionano per NVM installato su macOS/Linux, incluso WSL. Potrebbero funzionare diversamente o non funzionare affatto per `nvm-windows`. Sono forniti comandi alternativi per `nvm-windows` quando possibile.

### Politica di esecuzione PowerShell (Windows)

- **Politica di esecuzione predefinita**: la politica di esecuzione predefinita di PowerShell è `Restricted`, che impedisce l'esecuzione di script per motivi di sicurezza.

- **Esegui PowerShell come amministratore**: clic destro su PowerShell, poi clicca su "Esegui come amministratore".

- **Controlla politica di esecuzione corrente**:

```powershell
Get-ExecutionPolicy
```

- **Imposta politica di esecuzione su RemoteSigned**: permette l'esecuzione di script locali e `npm`:

```powershell
Set-ExecutionPolicy RemoteSigned
```

- **Dopo aver impostato la politica**: chiudi la finestra di PowerShell amministratore e apri una finestra PowerShell non amministratore.

### Elenca tutte le versioni installate

- **Elenca versioni installate**: per elencare tutte le versioni di Node.js installate e controllare quella attualmente in uso:

```bash
nvm ls
```

### Imposta versione Node.js predefinita (solo NVM su macOS/Linux)

- **Imposta versione predefinita**: imposta la versione predefinita di Node.js per nuove sessioni terminale:

```bash
nvm alias default <version>
```

Esempio:

```bash
nvm alias default 22.20.0
```

- **Crea alias personalizzati**: assegna un nome più specifico a una versione:

```bash
nvm alias <new_name> <version>
```

Esempio:

```bash
nvm alias new-api 18.20.8
```

- **Usa alias personalizzato**:

```bash
nvm use <new_name>
```

Esempio:

```bash
nvm use new-api
```

### Rimuovi una versione di Node.js

- **Disinstalla una versione**:

```bash
nvm uninstall <version>
```

Esempio:

```bash
nvm uninstall 20.19.5
```

- **Rimuovi per versione principale (NVM su macOS/Linux)**: puoi rimuovere una versione eseguendo il comando `uninstall` con una versione principale come `22` o `20`:

```bash
nvm uninstall 20
```

Se hai più release per quella versione principale installate, come `20.14.0` e `20.12.2`, verrà rimossa prima la versione più recente. Eseguendo `nvm uninstall 20` verrà rimossa prima `20.14.0`.

### Controlla versione Node.js

node -v

### Controlla versione npm

- **npm**: un gestore di pacchetti per Node.js usato per installare, pubblicare e gestire pacchetti software. Installato automaticamente con Node.js.

- **Controlla versione npm**:

```bash
npm -v
```

### Esegui un file JavaScript

- **Esegui file JavaScript**: esegue il file JavaScript nell'ambiente Node.js attivo:

```bash
node <file.js>
```

Esempio:

```bash
node app.js
```

### Crea un progetto Node.js

- **Inizializza progetto**: crea un nuovo progetto Node.js creando un file `package.json` che traccia dettagli e dipendenze:

```bash
npm init
```

- **Configurazione interattiva**: il comando `npm init` ti guida con un'interfaccia a riga di comando interattiva per configurare il file `package.json`.
  - Pone domande su nome progetto, descrizione, versione ecc.

- **Configurazione predefinita**: per iniziare con un file `package.json` predefinito:

```bash
npm init -y
```

o

```bash
npm init --yes
```

### Installa un pacchetto

- **Installa pacchetto specifico**:

```bash
npm install <package>
```

Esempio:

```bash
npm install express
```

### Installa tutte le dipendenze

- **Installa dipendenze**: per installare tutte le dipendenze elencate nel file `package.json` di un progetto:

```bash
npm install
```
