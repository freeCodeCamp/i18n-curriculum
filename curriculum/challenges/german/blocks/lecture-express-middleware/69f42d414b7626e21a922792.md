# --description--

Middleware auf Anwendungsebene ist Middleware, die an eine Instanz einer Express-Anwendung gebunden ist. Sie wird typischerweise verwendet, um Anfragen zu bearbeiten, die eine Verarbeitung über mehrere Routen hinweg erfordern. Sie kann global oder für bestimmte Routen angewendet werden.

Diese Middleware wird mit `app.use()` oder spezifischen Routemethoden wie `app.get()`, `app.post()` usw. registriert. Sie wird bei jeder eingehenden Anfrage ausgeführt, es sei denn, sie ist auf bestimmte Routen beschränkt.

```js
const express = require('express')
const app = express()

// Application-level middleware that logs all incoming requests
app.use((req, res, next) => {
  console.log('Request received at:', new Date())
  next(); // Passes control to the next middleware
})

app.get('/', (req, res) => {
  res.send('Hello World!')
})

app.listen(3000, () => {
  console.log('Server running on http://localhost:3000')
})
```

* `app.use()`: Dies ist Middleware auf Anwendungsebene. Sie protokolliert jede Anfrage, die an den Server gesendet wird, unabhängig von der Route.
* Die Middleware läuft bei jeder eingehenden Anfrage und liefert dem Server Details wie den aktuellen Zeitstempel.

Middleware auf Anwendungsebene ist nützlich für Aufgaben, die über viele oder alle Routen in der Anwendung angewendet werden müssen. Dazu gehören beispielsweise Protokollierung, Authentifizierung und Fehlerbehandlung.
