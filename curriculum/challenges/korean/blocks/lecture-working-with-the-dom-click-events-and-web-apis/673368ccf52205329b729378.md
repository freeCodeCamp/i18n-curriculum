# --interactive--

`innerText`, `textContent`, `innerHTML`에 대해 배워봅시다.

이들은 JavaScript에서 HTML 요소의 내용을 가져오거나 변경할 때 접근할 수 있는 속성입니다. 처음에는 매우 비슷해 보일 수 있지만, 중요한 차이점이 있습니다. 어떤 것을 선택할지는 특정 사용 사례에 따라 다르니 자세히 살펴보겠습니다.

먼저 `innerText`부터 시작하겠습니다.

`innerText`는 HTML 요소와 그 자손 요소의 보이는 텍스트 내용을 나타냅니다. 이 속성은 숨겨진 텍스트나 HTML 태그는 포함하지 않고, 렌더링된 텍스트만 포함합니다.

예를 들어, 여기 두 개의 단락을 포함한 `div` 요소가 있습니다:

```html
<div id="container">
  <p>Hello, World!</p>
  <p>I'm learning JavaScript</p>
</div>
```

JavaScript 코드에서 `getElementById()`를 사용해 이 HTML 요소에 대한 참조를 얻으면, 이 요소의 `innerText` 속성에 접근할 수 있습니다:

:::interactive_editor

```html
<div id="container">
  <p>Hello, World!</p>
  <p>I'm learning JavaScript</p>
</div>
<script src="index.js"></script>
```

```js
const container = document.getElementById("container");
console.log(container.innerText);
```

:::

이것이 이 요소의 내부 텍스트입니다:

```md
Hello, World!
I'm learning JavaScript
```

이 속성은 요소 내에 포함된 텍스트를 문자열로 반환하며, 자손 요소의 텍스트도 포함합니다.

`innerText`는 문자열이 요청된 순간에 보이는 텍스트만 반환한다는 점을 알아두세요. 자식 요소가 숨겨져 있으면 그 텍스트는 보이지 않습니다.

두 번째 단락이 숨겨진 예시는 다음과 같습니다:

:::interactive_editor

```html
<div id="container">
  <p>Hello, World!</p>
  <p hidden>I'm learning JavaScript</p>
</div>
```

:::

`innerText`를 다시 출력하면, 이제 두 번째 단락의 텍스트가 출력되지 않습니다:

:::interactive_editor

```html
<div id="container">
  <p>Hello, World!</p>
  <p hidden>I'm learning JavaScript</p>
</div>
<script src="index.js"></script>
```

```js
const container = document.getElementById("container");
console.log(container.innerText);
```

:::

HTML 요소의 `innerText` 속성은 이렇게 설정할 수 있지만, 기존 텍스트를 대체하고 줄바꿈마다 줄바꿈 요소(`br`)를 추가합니다:

:::interactive_editor

```html
<div id="container">
  <p>Hello, World!</p>
  <p>I'm learning JavaScript</p>
</div>
<script src="index.js"></script>
```

```js
const container = document.getElementById("container");
container.innerText = "JavaScript is awesome!";
```

:::

`innerText`는 가시성을 고려하기 때문에, 값을 가져오면 웹사이트 내 특정 요소의 위치를 다시 계산하는 "리플로우"라는 과정이 발생합니다. 이 과정은 계산 비용이 많이 들 수 있으니 가능하면 피하는 것이 좋습니다.

좋습니다. 이제 `textContent`에 대해 이야기해봅시다.

`textContent`는 요소와 그 자손 내 모든 텍스트를 포함한 일반 텍스트 내용을 반환합니다.

`innerText`와 `textContent`의 가장 중요한 차이점은 `textContent`는 보이든 숨겨져 있든 관계없이 항상 HTML 요소와 자손의 전체 텍스트 내용을 반환한다는 점입니다.

여기 동일한 HTML 예시가 있습니다:

```html
<div id="container">
  <p>Hello, World!</p>
  <p>I'm learning JavaScript</p>
</div>
```

이 속성에 접근하면 들여쓰기와 공백을 유지한 채 요소와 자손의 텍스트가 출력됩니다:

:::interactive_editor

```html
<div id="container">
  <p>Hello, World!</p>
  <p>I'm learning JavaScript</p>
</div>
<script src="index.js"></script>
```

```js
const container = document.getElementById("container");
console.log(container.textContent);
```

:::

HTML 요소가 보이지 않는 경우에도, 예를 들어 두 번째 단락을 숨긴 경우에도, 이 속성에는 텍스트가 포함됩니다:

:::interactive_editor

```html
<div id="container">
  <p>Hello, World!</p>
  <p hidden>I'm learning JavaScript</p>
</div>
<script src="index.js"></script>
```

```js
const container = document.getElementById("container");
console.log(container.textContent);
```

:::
  
`textContent`는 `script`와 `style` 같은 요소의 내용도 포함합니다.

노드에서 `textContent` 값을 바꾸면 모든 자식 노드를 제거하고 새 문자열을 포함하는 단일 텍스트 노드로 대체합니다:

:::interactive_editor

```html
<div id="container">
  <p>Hello, World!</p>
  <p hidden>I'm learning JavaScript</p>
</div>
<script src="index.js"></script>
```

```js
const container = document.getElementById("container");
container.textContent = "New content";
```

:::

마지막으로 `textContent`와 `innerText`가 `innerHTML`와 어떻게 다른지 알아봅시다.

`innerHTML`를 사용하면 요소의 내부 HTML 내용을 설정할 수 있습니다. 이는 DOM에 새 HTML을 동적으로 삽입할 때 유용합니다.

하지만 문자열을 제어할 수 없을 때, 예를 들어 사용자가 입력한 데이터를 포함하는 문자열이라면 보안 위험이 발생할 수 있습니다. 악의적인 데이터가 포함되면 심각한 보안 문제가 생길 수 있습니다.

이를 피하려면 일반 텍스트를 삽입할 때 `textContent` 속성을 사용하는 것이 권장됩니다.

JavaScript의 `innerText`, `textContent`, `innerHTML` 속성은 HTML 요소의 내용을 접근하고 조작하는 다양한 방법을 제공합니다. HTML 내용을 효과적으로 다루려면 이 속성들의 차이를 이해하는 것이 중요합니다.
