# --description--

Arrays sind JavaScript-Objekte, die viele verschiedene Elemente enthalten können.

```js
var complexArr = [1, 5, "2", "Word", {"name": "James"}];
```

Im Grunde passiert im Hintergrund, dass Ihr Browser automatisch den richtigen Speicherplatz für dieses Array bereitstellt. Dieser wird auch bei Bedarf angepasst, wenn Sie Daten hinzufügen oder entfernen.

Im Bereich der Hochleistung und bei verschiedenen Elementtypen müssen Sie jedoch manchmal genauer angeben, wie viel Speicher einem Array zugewiesen wird.

<dfn>Typisierte Arrays</dfn> sind die Lösung für dieses Problem. Sie können nun angeben, wie viel Speicher Sie einem Array zuweisen möchten. Nachfolgend finden Sie eine grundlegende Übersicht über die verschiedenen verfügbaren Array-Typen und die Größe in Bytes für jedes Element in diesem Array.

Es gibt zwei Möglichkeiten, solche Arrays zu erstellen. Eine Möglichkeit ist, es direkt zu erstellen. Nachfolgend sehen Sie, wie Sie ein `Int16Array` mit der Länge 3 erstellen.

```js
var i8 = new Int16Array(3);
console.log(i8);
// Returns [0, 0, 0]
```

Sie können auch einen <dfn>Puffer</dfn> erstellen, um festzulegen, wie viele Daten (in Bytes) das Array belegen soll. **Hinweis**  
Um typisierte Arrays mit Puffern zu erstellen, müssen Sie die Anzahl der Bytes als Vielfaches der oben angegebenen Bytes zuweisen.

```js
// Create same Int16Array array differently
var byteSize = 6; // Needs to be multiple of 2
var buffer = new ArrayBuffer(byteSize);
var i8View = new Int16Array(buffer);
buffer.byteLength; // Returns 6
i8View.byteLength; // Returns 6
console.log(i8View); // Returns [0, 0, 0]
```

<dfn>Puffer</dfn> sind allgemeine Objekte, die einfach Daten enthalten. Sie können nicht direkt auf sie zugreifen. Um darauf zuzugreifen, müssen Sie zuerst eine <dfn>Ansicht</dfn> erstellen.

```js
i8View[0] = 42;
console.log(i8View); // Returns [42, 0, 0]
```

**Hinweis**  
Typisierte Arrays verfügen nicht über einige der Methoden, die traditionelle Arrays haben, wie `.pop()` oder `.push()`. Typisierte Arrays schlagen auch bei `Array.isArray()` fehl, das überprüft, ob etwas ein Array ist. Obwohl einfacher, kann dies ein Vorteil für weniger komplexe JavaScript-Engines sein, die sie implementieren.

# --instructions--

Erstellen Sie zuerst einen `buffer`, der 64 Bytes groß ist. Erstellen Sie dann ein typisiertes `Int32Array`-Array mit einer Ansicht darauf namens `i32View`.

# --hints--

Ihr `buffer` sollte 64 Bytes groß sein.

```js
assert(buffer.byteLength === 64);
```

Ihre `i32View`-Ansicht Ihres Puffers sollte 64 Bytes groß sein.

```js
assert(i32View.byteLength === 64);
```

Ihre `i32View`-Ansicht Ihres Puffers sollte 16 Elemente lang sein.

```js
assert(i32View.length === 16);
```
