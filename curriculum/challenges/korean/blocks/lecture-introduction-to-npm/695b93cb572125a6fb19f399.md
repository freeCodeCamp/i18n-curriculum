# --description--

npm은 프로젝트에 관한 중요한 세부 정보를 저장할 방법이 필요합니다. 예를 들어 어떤 패키지를 사용하는지, 그 패키지들의 버전, 실행할 수 있는 명령어 등이 있습니다. 이 모든 것을 `package.json` 파일로 처리합니다.

`package.json` 파일은 프로젝트를 위한 npm 구성 파일이라고 생각하세요. 이 파일은 간단한 JSON 파일로, 키-값 쌍을 사용해 다음과 같은 내용을 기록합니다:

* 프로젝트 이름과 버전
    
* 작성자
    
* 가장 먼저 실행해야 할 주요 파일
    
* 라이선스
    
* 코드 저장소, 키워드, 설명
    
* 프로젝트가 필요로 하는 의존성
    

다음은 `package.json` 파일의 예시입니다:

```json
{
  "name": "my-app",
  "version": "1.0.0",
  "description": "A simple NPM project",
  "main": "index.js",
  "scripts": {
    "start": "node index.js",
    "test": "echo \"Running tests...\""
  },
  "repository": {
    "type": "git",
    "url": "git+https://github.com/username/project.git"
  },
  "keywords": ["fCC", "js"],
  "author": "Kolade Chris",
  "license": "ISC",
  "dependencies": {
    "react": "^19.2.1"
  },
  "devDependencies": {
    "cypress": "^15.7.1"
  }
}
```

`main` 필드는 보통 `index.js`로 설정되며, Node.js가 가장 먼저 실행할 파일을 알려줍니다.

`license` 필드는 다른 사람이 패키지의 코드를 사용, 수정, 배포할 때 적용되는 법적 조건을 명시하는 데 사용됩니다. `MIT`, `ISC`, `GPL` 같은 다양한 라이선스가 있으며, 각각 조건과 권한이 다릅니다. 기본값은 `ISC`로, 매우 관대한 오픈 소스 라이선스인 Internet Systems Consortium License입니다.

프로젝트가 커지고 패키지를 설치하면, npm은 두 섹션 중 하나에 패키지를 추가합니다:

* `dependencies`: 앱이 프로덕션에서 실행될 때 필요한 패키지
    
* `devDependencies`: 개발 중에만 필요한 패키지
    

예를 들어, 프로젝트가 UI에 React를 사용한다면, 배포 후 프로젝트가 작동하는 데 필요하므로 `dependencies`에 들어갑니다. Cypress 같은 도구는 프로덕션 배포 전에만 사용하는 종단 간 테스트용이므로 `devDependencies`에 속합니다.

`package.json` 파일을 만드는 쉬운 방법이 몇 가지 있습니다. 가장 일반적인 방법은 다음 명령어를 실행하는 것입니다:

```bash
npm init
```

이 명령어는 프로젝트의 루트 폴더에서 실행해야 합니다. 이 명령어는 대화형 설정 과정을 시작하며, npm이 여러 질문을 합니다. 답변은 프로젝트 이름, 버전, 설명, 스크립트, 키워드, 작성자, 라이선스 같은 필드를 채우는 데 사용됩니다.

질문을 건너뛰고 기본값으로 `package.json` 파일을 만들고 싶다면 다음 명령어를 실행하세요:

```bash
npm init -y
```

또는

```bash
npm init --yes
```

이 명령어들은 여러 필드에 기본값을 넣은 `package.json` 파일을 즉시 생성합니다.

일반적으로 `npm init`를 `-y` 또는 `--yes` 플래그 없이 사용하는 것이 좋습니다. 그래야 각 필드를 살펴보고 프로젝트 정보를 맞춤 설정할 수 있습니다. 질문에 답하지 않으면 npm이 기본값을 사용합니다. 예를 들어 라이선스 질문을 비워두면 기본적으로 `ISC`로 설정됩니다.

다음은 `npm init` 명령어를 실행할 때 보게 될 예시입니다:


`dependencies`와 `devDependencies`이 아직 목록에 없는 것을 알 수 있습니다. 이 섹션들은 첫 번째 패키지를 설치한 후에만 추가되기 때문입니다.

또한 `repository` 필드가 없다는 것도 눈치챌 수 있습니다. npm이 `npm init` 중에 물어보긴 하지만, 나중에 자동으로 채우지 않습니다. 예를 들어 `git remote add origin` 또는 `git push` 같은 명령어를 실행해도 npm이 저장소 필드를 업데이트하지 않으니, 포함시키려면 직접 추가해야 합니다.

다음 수업에서는 `package.json`의 의존성에 대해 더 배우고, 설치, 업데이트, 삭제 방법도 익힐 것입니다.
