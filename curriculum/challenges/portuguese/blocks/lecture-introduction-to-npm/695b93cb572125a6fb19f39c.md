# --description--

Em uma lição anterior, você aprendeu o que são dependências e como escolher as certas para o seu projeto.

Nesta lição, você aprenderá como instalar e remover dependências usando npm.

Para instalar uma dependência, use o comando `npm install` assim:

```bash
npm install dependencyName
```

Você também pode abreviar `install` para `i` assim:

```bash
npm i dependencyName
```

Após instalar um pacote, o npm adiciona uma nova seção `dependencies` ao seu arquivo `package.json` (se ainda não estiver lá) e lista o pacote que você instalou. Por exemplo, veja como fica quando você instala o pacote `chalk`:

```json
{
  "name": "install-uninstall-deps",
  "version": "1.0.0",
  "description": "a simple demo on how to install and uninstall dependencies",
  "main": "index.js",
  "scripts": {
    "test": "echo \"Error: no test specified\" && exit 1"
  },
  "keywords": ["npm", "deps"],
  "author": "Kolade Chris",
  "license": "ISC",
  "dependencies": {
    "chalk": "^5.6.2"
  }
}
```

Quando você instala uma dependência, duas coisas novas aparecem na pasta raiz do seu projeto:

* `node_modules/`: uma pasta que contém o código real dos pacotes que você instalou.
    
* `package-lock.json`: um arquivo que armazena informações detalhadas sobre as versões exatas desses pacotes, incluindo suas próprias dependências. (Você aprenderá mais sobre esse arquivo em uma lição posterior.)
    

<img src="https://cdn.freecodecamp.org/curriculum/lecture-transcripts/how-can-you-install-and-remove-dependencies-1.png" alt="">

A maioria dos projetos precisa de mais de uma dependência. Em vez de instalá-las uma a uma, o npm permite que você instale vários pacotes de uma vez digitando uma lista de nomes de pacotes separados por espaço:

```bash
npm install express lodash bcrypt
```

Assim como antes, quando você instala múltiplas dependências de uma vez, o npm atualiza seus arquivos `package.json` e `package-lock.json` e seus `node_modules` para incluí-las. Veja como o `package.json` ficaria:

```json
{
  "name": "install-uninstall-deps",
  "version": "1.0.0",
  "description": "a simple demo on how to install and uninstall dependencies",
  "main": "index.js",
  "scripts": {
    "test": "echo \"Error: no test specified\" && exit 1"
  },
  "keywords": ["npm", "deps"],
  "author": "Kolade Chris",
  "license": "ISC",
  "dependencies": {
    "bcrypt": "^6.0.0",
    "chalk": "^5.6.2",
    "express": "^5.1.0",
    "lodash": "^4.17.21"
  }
}
```

Para instalar uma `devDependency`, adicione a flag `--save-dev` ao comando `install`. Você também pode usar a versão mais curta `-D`:

```bash
npm i nodemon -D
```

Após executar este comando, o npm atualiza seu arquivo `package.json` e adiciona o pacote na seção `devDependencies`. Aqui está um exemplo de como isso fica:

```json
{
  "name": "install-uninstall-deps",
  "version": "1.0.0",
  "description": "a simple demo on how to install and uninstall dependencies",
  "main": "index.js",
  "scripts": {
    "test": "echo \"Error: no test specified\" && exit 1"
  },
  "keywords": ["npm", "deps"],
  "author": "Kolade Chris",
  "license": "ISC",
  "dependencies": {
    "bcrypt": "^6.0.0",
    "chalk": "^5.6.2",
    "express": "^5.1.0",
    "lodash": "^4.17.21"
  },
  "devDependencies": {
    "nodemon": "^3.1.10"
  }
}
```

Sempre que você executar um comando `npm install`, o npm mostrará uma mensagem informando se a instalação foi bem-sucedida ou não. Uma mensagem de sucesso geralmente se parece com isto:

```bash

added 72 packages, and audited 74 in 8s

  run `npm fund` for details

found 0 vulnerabilities

user@Kolade ~/Desktop/fCC/script-code/node/npm/install-uninstall-deps  % npm install nodemon -D

added 27 packages, and audited 101 in 16s

21 packages are looking for funding
  run `npm fund` for details

found 0 vulnerabilities
```

Para desinstalar ou remover um pacote, use o comando `npm uninstall`:

```bash
npm uninstall dependencyName
```

Por exemplo, para remover `chalk` do seu projeto, você deve executar este comando:

```bash
npm uninstall chalk
```

Você também pode desinstalar vários pacotes de uma vez listando-os com espaços entre cada nome de pacote:

```bash
npm uninstall express lodash bcrypt
```

Você também receberá uma mensagem mostrando se o comando de desinstalação foi bem-sucedido ou falhou:

```bash
user@Kolade ~/Desktop/fCC/script-code/node/npm/install-uninstall-deps  % npm uninstall chalk

added 1 package, removed 1 package, and audited 101 packages in 1s

20 packages are looking for funding
  run `npm fund` for details

found 0 vulnerabilities

user@Kolade ~/Desktop/fCC/script-code/node/npm/install-uninstall-deps  % npm uninstall express lodash bcrypt

removed 70 packages, and audited 31 in 1s

4 packages are looking for funding
  run `npm fund` for details

found 0 vulnerabilities
```

E seu arquivo `package.json` será atualizado automaticamente:

```json
{
  "name": "install-uninstall-deps",
  "version": "1.0.0",
  "description": "a simple demo on how to install and uninstall dependencies",
  "main": "index.js",
  "scripts": {
    "test": "echo \"Error: no test specified\" && exit 1"
  },
  "keywords": ["npm", "deps"],
  "author": "Kolade Chris",
  "license": "ISC",
  "devDependencies": {
    "nodemon": "^3.1.10"
  }
}
```

Para desinstalar uma `devDependency`, você pode simplesmente executar:

```bash
npm uninstall nodemon
```

Observe que você não precisa da flag `-D` ao desinstalar. O npm removerá automaticamente o pacote da seção correta, `devDependencies` neste caso.

Após desinstalar o `devDependency`, seu `package.json` será atualizado novamente:

```json
{
  "name": "install-uninstall-deps",
  "version": "1.0.0",
  "description": "a simple demo on how to install and uninstall dependencies",
  "main": "index.js",
  "scripts": {
    "test": "echo \"Error: no test specified\" && exit 1"
  },
  "keywords": ["npm", "deps"],
  "author": "Kolade Chris",
  "license": "ISC"
}
```
