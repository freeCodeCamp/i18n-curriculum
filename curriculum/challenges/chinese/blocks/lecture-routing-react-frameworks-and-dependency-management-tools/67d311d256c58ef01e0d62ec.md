# --description--

到目前为止，你一直在使用 React 构建用户界面。如果你需要像路由这样的额外特色，那么你必须导入第三方库，比如 React Router，才能在不同的视图之间切换。

但是当你需要构建一个全栈网页应用时会发生什么？如果你只想坚持使用 JavaScript，那么你可以使用 React 作为前端，使用 Node 和 Express 处理后端逻辑。或者你也可以为后端使用其他语言，比如 Go、Python 或 Java。

虽然所有这些都是可行的选项，但有时你可能想使用 React 框架。React 框架提供路由、图像优化、数据获取、认证等特色。这意味着对于某些用例，你可能不需要设置单独的前端和后端应用。

让我们更详细地了解 Next.js，它是一个流行的 React 框架。Next.js 的主要特色之一是基于文件系统的路由器。该路由系统包括对动态路由、并行路由、路由处理程序、重定向、国际化等的支持。

下面是创建自定义请求处理器的示例：

```js
export async function GET() {
  const res = await fetch("https://example-api.com");
  const data = await res.json();

  return Response.json({ data });
}
```

你可以在 `app/api` 目录下名为 `route.js` 的文件中定义 GET 或 POST 请求的路由处理器。

Next.js 的另一个特色是自动的图像和字体优化。

下面是一个在 `page.js` 文件中使用 `Image` 组件的示例：

```jsx
import Image from "next/image";

export default function Page() {
  return (
    <Image src="link-to-image-goes-here" alt="descriptive-title-goes-here" />
  );
}
```

`Image` 组件扩展了本地的超文本标记语言 `img` 元素，允许更快的页面装载和尺寸优化。这意味着图像只有在进入视图时才会装载，且 `Image` 组件会自动为每个设备提供正确尺寸的图像。

虽然这些只是 Next.js 的一些特色，但你还可以使用许多其他特色来构建健壮的全栈网页应用。还有其他 React 框架，比如 Remix，也能提供构建现代全栈网页应用的相同能力。

尽管 JavaScript 库和框架的格局不断变化，了解你可用的选项并学习每个框架的优缺点仍然很重要。
