# --description--

現代のウェブ開発では、コードを「モジュール」と呼ばれる複数のファイルに分割できます。

モジュールは自己完結型で再利用可能なコードの塊と考えられます。

モジュールを他のモジュールにインポートすることで、その機能を再利用しつつ、コードを別々のファイルに分けて管理できるため、大規模で複雑なウェブアプリケーションの保守性が向上します。

Node.jsでは、各ファイルがモジュールとみなされます。

Nodeにはモジュールを扱うための2つの異なるシステムがあります：CommonJSとESです。

CommonJSはNode.jsでパッケージを扱う元々のデフォルトの方法です。モジュールを同期的に読み込むため、次のモジュールの読み込みやコードの実行を続ける前に、モジュールが完全に読み込まれるのを待つ必要があります。

現在も広く使われていますが、徐々に新しいES構文に置き換えられつつあります。

CommonJSは`require()`関数を使ってモジュールをインポートし、その相対パスを引数として受け取ります。

例えば、基本的な数学演算を定義した`math.js`というモジュールがあるとしましょう。

この例をシンプルに保つために、ここでは掛け算の関数だけを使います：

```js
function multiply(a, b) {
  return a * b;
}
```

この関数を現在のモジュールの外で再利用するにはどうすればよいでしょうか？デフォルトでは、Node.jsモジュール内で定義された関数や変数は**プライベート**であり、明示的にエクスポートしない限り他のモジュールからアクセスできません。これにはいくつかの方法があります。

一般的な方法の一つは、`module.exports`に値を代入することです。このオブジェクトは、モジュールが他の場所でインポートされたときに返されるもので、公開したい関数や変数、オブジェクトへのアクセスを他のモジュールに提供します。

この例では、`multiply`プロパティを持つオブジェクトを`module.exports`に代入しています。そのプロパティの値は、先ほど定義した`multiply`関数への参照です。これにより関数がエクスポートされ、必要に応じて他のモジュールから呼び出せるようになります。

```javascript
function multiply(a, b) {
  return a * b;
}

module.exports = {
  multiply: multiply
};
```

別のモジュールに`math.js`モジュールをインポートしようとするとしましょう。`require()`関数を呼び出し、相対パスを引数として渡します。次に、`require()`が返すオブジェクトを`math`変数に代入します。

```javascript
const math = require('./math');
```

コンソールで`math`の値を`console.log()`で出力して確認しましょう：

```javascript
console.log(math);
```

これは出力結果です。`multiply`プロパティを持つオブジェクトで、その値は関数です：

```javascript
{ multiply: [Function: multiply] }
```

この関数を`math.multiply()`で呼び出し、引数に`6`と`7`を渡すと、出力は`42`になります。関数は正常に実行され、しかも別のファイルにあります！つまり、インポート文は正しく機能しています。

```javascript
const math = require('./math');

console.log(math.multiply(6, 7)); // 42
```

しかし、関数や変数、オブジェクトをエクスポートする方法は他にもあります。

例えば、単一の関数や変数、オブジェクトだけをエクスポートしたい場合は、それを直接`module.exports`の値として代入できます：

```javascript
function multiply(a, b) {
  return a * b;
}

module.exports = multiply;
```

複数の関数をエクスポートしたい場合は、カンマで区切って`module.exports`オブジェクトに追加するだけです。

ここでは、エクスポートされたオブジェクトが`subtract`と`multiply`という2つのプロパティを持っています。それらの値は対応する関数への参照なので、`math.js`をインポートするどのモジュールでも両方にアクセスできます：

```javascript
function subtract(a, b) {
  return a - b;
}

function multiply(a, b) {
  return a * b;
}

module.exports = {
  subtract: subtract,
  multiply: multiply
};
```

`module.exports`オブジェクトに複数のプロパティがある場合、すべてをインポートするか、個別にプロパティをインポートするか選べます。

この例では、`math.js`がエクスポートしたオブジェクト全体をインポートしているので、`subtract`と`multiply`の両方にアクセスできます：

```javascript
const math = require('./math');

console.log(math.subtract(6, 7));  // -1
console.log(math.multiply(6, 7));  // 42
```

もし`multiply`だけをインポートしたい場合は、`const`キーワードの後に波括弧で名前を書くだけです。基本的に、分割代入構文を使ってエクスポートされたオブジェクトから値を取得しています。その後はコード内で名前を直接参照できます：

```javascript
const { multiply } = require('./math');

console.log(multiply(6, 7));
```

これは複数の要素にも使えます。波括弧内でカンマで区切るだけです。ここでは`subtract`と`multiply`の両方の関数をインポートしています：

```javascript
const { subtract, multiply } = require('./math');
```

デフォルトでは、Node.jsは以下をCommonJSモジュールとして扱います：

* `.cjs`拡張子のファイル。
    
* 最も近い親の`.js`ファイルにトップレベルフィールド`package.json`が`"type"`の値で含まれている場合の`"commonjs"`拡張子のファイル。
    
* 最も近い親の`.js`ファイルにトップレベルフィールド`package.json`が含まれていない、または親フォルダに`"type"`が存在しない場合の`package.json`拡張子のファイルや拡張子なしのファイル。
    
* `.mjs`、`.cjs`、`.json`、`.node`、`.js`以外の拡張子のファイル。
    

ESモジュールが徐々にCommonJSモジュールに取って代わりつつありますが、既存のNode.jsコードベースや多くのnpmパッケージを扱う開発者にとっては、CommonJSの理解は依然として重要です。
