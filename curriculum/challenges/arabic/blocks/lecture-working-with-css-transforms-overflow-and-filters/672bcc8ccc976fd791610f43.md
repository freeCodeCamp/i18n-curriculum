# --interactive--

خاصية CSS `transform` هي أداة قوية تتيح لك تعديل العرض البصري للعناصر على صفحة الويب دون التأثير على تخطيط العناصر الأخرى. تمكنك من تطبيق تحولات مختلفة على العناصر مثل التدوير، التكبير/التصغير، الإمالة، أو الترجمة (التحريك) في فضاء ثنائي أو ثلاثي الأبعاد.

تعمل خاصية `transform` عن طريق تطبيق تحويل رياضي على نظام إحداثيات العنصر. هذا يعني أنه يمكنك تعديل شكل وموقع العنصر مع الحفاظ على مكانه الأصلي وتدفق المستند كما هو.

لنستعرض بعض دوال التحويل الشائعة. إليك مثالًا على عنصر صندوق:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<div class="box"></div>
```

```css
body {
  border: 2px solid black;
}

.box {
  width: 200px;
  height: 200px;
  background-color: red;
}
```

:::

قمنا بتعيين حدود سوداء صلبة لعنصر `body` حتى تتمكن من رؤية عنصر `.box` المسكن داخل عنصر `body`.

دالة `translate` تحرك العنصر من موقعه الحالي. إليك مثالًا محدثًا يستخدم دالة `translate`:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<div class="box"></div>
```

```css
body {
  border: 2px solid black;
}

.box {
  width: 200px;
  height: 200px;
  background-color: red;
  transform: translate(50px, 100px);
}
```

:::

تقوم قاعدة CSS هذه بتحريك العنصر الذي يحمل الصنف `box` بمقدار 50 بكسل إلى اليمين و100 بكسل إلى الأسفل من موقعه الأصلي.

دالة `rotate` تدور العنصر حول نقطة ثابتة، وهذا مثال على استخدام دالة `rotate` لعنصر `.box` من المثال السابق:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<div class="box"></div>
```

```css
.box {
  margin: 100px;
  width: 200px;
  height: 200px;
  background-color: red;
  transform: rotate(45deg);
}
```

:::

سيقوم هذا بتدوير العنصر بزاوية خمسة وأربعين درجة في اتجاه عقارب الساعة.

دالة `scale` تتيح لك تغيير حجم العنصر. إليك مثالًا:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<div class="box"></div>
```

```css
.box {
  margin: 100px;
  width: 200px;
  height: 200px;
  background-color: red;
  transform: scale(1.5, 2);
}
```

:::

سيجعل هذا العنصر أعرض بمقدار مرة ونصف وأطول بمقدار مرتين من حجمه الأصلي.

يمكنك دمج عدة تحولات في إعلان واحد:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<div class="box"></div>
```

```css
.box {
  margin: 100px;
  width: 200px;
  height: 200px;
  background-color: red;
  transform: translate(50px, 50px) rotate(45deg) scale(1.5);
}
```

:::

سيحرك هذا العنصر 50 بكسل إلى اليمين والأسفل، ويديره 45 درجة، ويكبره ليصبح مرة ونصف من حجمه الأصلي.

بينما خاصية `transform` قوية لإنشاء تصاميم جذابة بصريًا، من المهم مراعاة إمكانية الوصول عند استخدامها. إليك بعض النقاط المهمة المتعلقة بإمكانية الوصول يجب الانتباه لها.

قد لا تنقل قارئات الشاشة المحتوى المحول بدقة. على سبيل المثال، إذا استخدمت `transform` لإعادة ترتيب الترتيب البصري للعناصر، ستظل قارئات الشاشة تقرأ المحتوى حسب ترتيب DOM الأصلي. هذا قد يسبب ارتباكًا للمستخدمين الذين يعتمدون على قارئات الشاشة.

عند استخدام `scale` لتغيير حجم النص، كن حذرًا ألا تجعله صغيرًا جدًا أو كبيرًا جدًا. النص الصغير جدًا قد يصعب قراءته، بينما النص الكبير جدًا قد يتجاوز الحاوية ويصبح غير قابل للقراءة. من الأفضل عمومًا استخدام تقنيات تنسيق الخط المناسبة لتغيير حجم النص.

إذا كنت تستخدم `transform` لتأثيرات الرسوم المتحركة، انتبه للمستخدمين الحساسين للحركة. يمكن أن تسبب الرسوم المتحركة المفرطة أو السريعة انزعاجًا أو حتى نوبات لبعض الأشخاص. فكر في توفير طريقة للمستخدمين لتقليل أو إيقاف الرسوم المتحركة. عند استخدام التحولات ثلاثية الأبعاد، تذكر أن ليس كل المستخدمين يدركون العمق بنفس الطريقة. تأكد من أن أي معلومات مهمة تُنقل عبر تأثيرات ثلاثية الأبعاد متاحة أيضًا بصيغة ثنائية الأبعاد أو نصية.

إذا كنت تستخدم `transform` لإخفاء أو إظهار المحتوى، تأكد من أن المحتوى لا يزال متاحًا لقارئات الشاشة والتنقل عبر لوحة المفاتيح. يجب أن يكون المحتوى المخفي مخفيًا فعليًا مثل استخدام `display: none;` أو `visibility: hidden;`، وليس مجرد تحريكه بصريًا خارج الشاشة.

عند تطبيق `transform` على عناصر تفاعلية مثل الأزرار أو الروابط، تأكد من أن مساحة النقر تظل بديهية وسهلة الاستهداف. قد يكون الزر المحول بشكل كبير مربكًا بصريًا أو صعب النقر، خاصة للمستخدمين الذين يعانون من إعاقات حركية.

في الختام، بينما خاصية CSS `transform` أداة قوية لإنشاء تصاميم ويب ديناميكية بصريًا، من الضروري استخدامها بمسؤولية مع مراعاة إمكانية الوصول. اختبر دائمًا العناصر المحولة باستخدام تقنيات المساعدة المختلفة وفكر في توفير طرق بديلة للوصول إلى المعلومات أو الوظائف التي قد تتأثر بالتحولات.
