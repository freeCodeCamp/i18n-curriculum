# --description--

Hier steigen wir von binären Suchbäumen um und betrachten eine andere Art von Baumstruktur, die Trie genannt wird. Ein Trie ist ein geordneter Suchbaum, der häufig verwendet wird, um Strings zu speichern oder allgemeiner assoziative Arrays oder dynamische Datensätze, bei denen die Schlüssel Strings sind. Sie sind sehr gut darin, Datensätze zu speichern, wenn viele Schlüssel überlappende Präfixe haben, zum Beispiel alle Wörter in einem Wörterbuch. Im Gegensatz zu einem binären Baum sind Knoten nicht mit tatsächlichen Werten verbunden. Stattdessen stellt der Pfad zu einem Knoten einen bestimmten Schlüssel dar. Wenn wir zum Beispiel den String code in einem Trie speichern wollten, hätten wir vier Knoten, einen für jeden Buchstaben: c — o — d — e. Wenn man diesem Pfad durch alle diese Knoten folgt, entsteht code als String – dieser Pfad ist der Schlüssel, den wir gespeichert haben. Wenn wir dann den String coding hinzufügen wollten, würde er die ersten drei Knoten von code teilen, bevor er sich nach dem d verzweigt. Auf diese Weise können große Datensätze sehr kompakt gespeichert werden. Außerdem kann die Suche sehr schnell sein, da sie effektiv auf die Länge des Strings begrenzt ist, den Sie speichern. Darüber hinaus kann ein Knoten im Gegensatz zu binären Bäumen eine beliebige Anzahl von Kindknoten speichern. Wie Sie aus dem obigen Beispiel vielleicht schon vermutet haben, werden an Knoten, die das Ende eines Schlüssels halten, häufig Metadaten gespeichert, damit bei späteren Durchläufen dieser Schlüssel weiterhin abgerufen werden kann. Wenn wir zum Beispiel codes in unserem obigen Beispiel hinzufügen würden, bräuchten wir eine Möglichkeit zu wissen, dass das e in code das Ende eines zuvor eingegebenen Schlüssels darstellt. Andernfalls gingen diese Informationen beim Hinzufügen von codes effektiv verloren.

# --instructions--

Lassen Sie uns einen Trie erstellen, um Wörter zu speichern. Er soll Wörter über eine `add`-Methode akzeptieren und diese in einer Trie-Datenstruktur speichern. Außerdem soll er uns erlauben, mit einer `isWord`-Methode abzufragen, ob ein gegebener String ein Wort ist, und mit einer `print`-Methode alle in den Trie eingegebenen Wörter abzurufen. `isWord` sollte einen Boolean-Wert zurückgeben und `print` sollte ein Array aller dieser Wörter als String-Werte zurückgeben. Damit wir überprüfen können, ob diese Datenstruktur korrekt implementiert ist, haben wir eine `Node`-Struktur für jeden Knoten im Baum bereitgestellt. Jeder Knoten wird ein Objekt mit einer `keys`-Eigenschaft sein, die ein JavaScript Map-Objekt ist. Dieses hält die einzelnen Buchstaben, die gültige Schlüssel jedes Knotens sind. Wir haben außerdem eine `end`-Eigenschaft an den Knoten erstellt, die auf `true` gesetzt werden kann, wenn der Knoten das Ende eines Wortes darstellt.
