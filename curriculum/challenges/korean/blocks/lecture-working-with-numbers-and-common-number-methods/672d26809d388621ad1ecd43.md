# --interactive--

JavaScript에서 `NaN`는 "숫자가 아님"을 의미합니다. 이는 나타낼 수 없거나 정의되지 않은 숫자 결과를 나타내는 특별한 값입니다. `NaN`는 전역 객체의 속성이며, JavaScript에서는 숫자의 한 종류로 간주되기도 하는데, 처음에는 직관에 반할 수 있습니다.

`NaN`는 일반적으로 숫자를 반환해야 하지만 의미 있는 숫자 값을 생성할 수 없는 연산의 결과입니다. 예를 들어:

:::interactive_editor

```js
let result = 0 / 0;
console.log(result); // NaN
```

:::

이 경우, 0을 0으로 나누는 것은 수학적으로 정의되지 않으므로 JavaScript는 `NaN`를 반환합니다. `NaN`의 특이한 속성 중 하나는 자기 자신을 포함해 어떤 것과도 같지 않다는 점입니다:

:::interactive_editor

```js
console.log(NaN === NaN); // false
```

:::

이 독특한 동작 때문에 표준 비교 연산자로 값이 `NaN`인지 확인하는 것이 어렵습니다. 이를 해결하기 위해 JavaScript는 `isNaN()` 함수를 제공합니다. `isNaN()` 함수 속성은 값이 `NaN`인지 아닌지 판단하는 데 사용됩니다. 하지만 `isNaN()`가 어떻게 작동하는지 이해하는 것이 중요한데, 때때로 예상치 못한 결과를 낼 수 있기 때문입니다. `isNaN()`의 동작 방식은 다음과 같습니다:

:::interactive_editor

```js
console.log(isNaN(NaN));       // true
console.log(isNaN(undefined)); // true
console.log(isNaN({}));        // true

console.log(isNaN(true));      // false
console.log(isNaN(null));      // false
console.log(isNaN(37));        // false

console.log(isNaN("37"));      // false: "37" is converted to 37
console.log(isNaN("37.37"));   // false: "37.37" is converted to 37.37
console.log(isNaN(""));        // false: empty string is converted to 0
console.log(isNaN(" "));       // false: string with a space is converted to 0

console.log(isNaN("blabla"));  // true: "blabla" is not a number
```

:::

보시다시피 `isNaN()`는 먼저 매개변수를 숫자로 변환하려 시도합니다. 변환할 수 없으면 `true`를 반환합니다. 이 동작은 특히 숫자로 강제 변환될 수 있는 문자열을 다룰 때 놀라운 결과를 초래할 수 있습니다.

이러한 잠재적 불일치 때문에 ES6(2015년에 출시된 JavaScript 6번째 판)는 `Number.isNaN()`를 도입했습니다. 이 메서드는 테스트 전에 매개변수를 숫자로 변환하려 하지 않습니다. 값이 정확히 `true`일 때만 `NaN`를 반환합니다:

:::interactive_editor

```js
console.log(Number.isNaN(NaN));        // true
console.log(Number.isNaN(Number.NaN)); // true
console.log(Number.isNaN(0 / 0));      // true

console.log(Number.isNaN("NaN"));      // false
console.log(Number.isNaN(undefined));  // false
console.log(Number.isNaN({}));         // false
console.log(Number.isNaN("blabla"));   // false
```

:::

`Number.isNaN()`는 특히 타입 강제 변환으로 인해 전역 `NaN` 함수에서 예상치 못한 결과가 발생할 수 있는 경우 `isNaN()` 값을 확인하는 더 신뢰할 수 있는 방법을 제공합니다. 실제로 숫자 연산이나 숫자여야 하는 사용자 입력을 다룰 때는 오류나 예상치 못한 입력을 우아하게 처리하기 위해 `NaN`를 확인하는 것이 종종 필요합니다. 예를 들어:

:::interactive_editor

```js
let a = 0;
let b = 0;
let result = a / b;

if (Number.isNaN(result)) {
  result = "Error: Division resulted in NaN";
}

console.log(result); // "Error: Division resulted in NaN"
```

:::

이 예제에서는 `Number.isNaN()`를 사용해 나눗셈 연산 결과가 `NaN`인 경우를 잡아내어 이 상황을 적절히 처리할 수 있게 합니다. `NaN`를 이해하고 이를 올바르게 확인하는 방법은 특히 수학 연산이나 사용자 입력 구문 분석을 다룰 때 견고한 JavaScript 코드를 작성하는 데 매우 중요합니다.
