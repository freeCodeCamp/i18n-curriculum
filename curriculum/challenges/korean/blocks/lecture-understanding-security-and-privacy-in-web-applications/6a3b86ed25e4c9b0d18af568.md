# --description--

`.env` 파일과 `dotenv` 라이브러리는 민감한 구성 데이터를 애플리케이션 코드와 분리하기 위해 현대 소프트웨어 개발에서 사용하는 도구입니다.

`.env` 파일은 프로젝트 디렉터리의 루트에 위치한 일반 텍스트 파일로, 구성 설정과 비밀 정보를 키-값 쌍으로 저장합니다.

API 키, 데이터베이스 비밀번호, 포트 번호를 소스 코드에 직접 하드코딩하여 GitHub 같은 공개 저장소에 코드를 올릴 때 노출 위험을 감수하는 대신, 개발자들은 이 파일에 로컬로 저장합니다.

`.env` 파일의 구조는 간단합니다:

```bash
PORT=3000
DB_HOST=localhost
API_KEY=xyz123secretkey
```

이 파일에는 매우 민감한 키가 포함되어 있으므로, `.env`를 프로젝트 `.gitignore` 파일에 추가하여 Git에서 추적하지 않도록 하는 것이 일반적인 관행입니다.

`dotenv` 패키지는 `.env` 파일을 애플리케이션에 연결하는 가벼운 모듈입니다.

프로그램이 시작되면 `dotenv`가 `.env` 파일을 읽고 키를 파싱하여 런타임 환경에 주입합니다. 예를 들어 Node.js 애플리케이션에서는 진입 파일 맨 위에서 패키지를 초기화합니다. 로드되면 변수들은 `process.env` 객체를 통해 전역에서 접근할 수 있게 됩니다.

```javascript
const express = require('express');
const app = express();

// initialize and configure dotenv
require('dotenv').config();

// access the variables securely from the environment
const port = process.env.PORT || 5000;
const apiKey = process.env.API_KEY;

app.get('/', (req, res) => {
  res.send(`Connected successfully using API Key: ${apiKey}`);
});

app.listen(port, () => {
  console.log(`Server running on port ${port}`);
});
```

이 설정을 사용하면 두 가지 주요 이점이 있습니다. 첫째, 비공개 자격 증명을 소스 코드에서 분리해 보안을 강화합니다. 둘째, 각 호스팅 서버에서 로컬 `.env` 파일만 교체하여 개발, 테스트, 프로덕션 환경에서 동일한 코드베이스가 다르게 동작하도록 유연성을 제공합니다.
