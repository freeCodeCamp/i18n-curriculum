# --description--

你是否曾經點擊過網頁卻感覺什麼都沒發生？這種延遲會影響使用者體驗，並以一個稱為 Interaction to Next Paint，或 INP 的指標來衡量。

INP 透過測量使用者交談（例如點擊或按鍵）到瀏覽器下一次更新顯示的時間，來評估頁面的整體回應速度。較低的 INP 表示頁面回應更快。

但是為什麼 `INP` 很重要？高 `INP` 會讓使用者感覺頁面沒有回應，導致挫折感。優化 `INP` 可確保使用者獲得即時回授，提升他們的體驗。

像 Chrome 的 DevTools 和網頁效率、性能 API 這類工具可以透過追蹤互動延遲來協助測量 INP。

為了提升 INP，請將主執行緒工作量降到最低，最佳化事件處理常式，並確保使用者互動能觸發即時的視覺更新。

透過理解和優化 INP，你可以創建感覺快速且回應靈敏的網頁體驗，讓你的用戶保持滿意。
