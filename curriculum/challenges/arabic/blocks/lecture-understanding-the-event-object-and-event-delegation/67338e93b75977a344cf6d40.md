# --description--

تشير فقاعات الأحداث، أو الانتشار، إلى كيفية "تصاعد" الحدث إلى الكائنات الأصلية عند تشغيله. على سبيل المثال، انظر إلى هذه الشفرة:

```html
<p>
  <span>Click me~!</span>
</p>
```

يُعتبر عنصر `p` هنا هو الأصل لعنصر `span`.

عندما تنقر على عنصر `span`، كما هو موضح لك، يصبح عنصر `span` هدف حدث `click`. لكن هذا الحدث أيضًا يتصاعد إلى الأصل – يمكن لعنصر `p` استقبال هذا الحدث واستهلاكه حسب الحاجة.

لكن ماذا يعني هذا فعليًا؟ حسنًا، يمكنك إرفاق مستمع حدث إلى عنصر `p`:

```js
const p = document.querySelector("p");
p.addEventListener("click", (event) => {
  console.log(event.target);
});
```

ثم، عندما تنقر على عنصر `span` سترى النص `Click me~!` مسجلاً في وحدة التحكم.

ينتشر الحدث إلى عنصر `p` الأصل، الذي يستهلكه في مستمع الحدث ليعرض هدف الحدث.

لاحظ كيف أن الهدف لا يزال هو عنصر `span`. هذا لأن عنصر `span` تلقى النقر الأولي.

للتأكد من كيفية عمل الأمور، دعنا نوسع شفرتنا:

```js
const p = document.querySelector("p");
const span = document.querySelector("span");
p.addEventListener("click", (event) => {
  console.log("P listener: ");
  console.log(event.target);
});
span.addEventListener("click", (event) => {
  console.log("Span listener: ");
  console.log(event.target);
});
```

لنعطيك فكرة عن كيفية تصاعد الحدث، هذا ما سترى في وحدة التحكم بعد النقر على عنصر `span`:

```html
"Span listener: "
<span>Click me~!</span>
"P listener: "
<span>Click me~!</span>
```

الآن لنرَ ماذا يحدث عندما تمنع انتشار حدث باستخدام `stopPropagation()`. سنستدعيها في مستمع حدث `span` الخاص بنا:

```js
const p = document.querySelector("p");
const span = document.querySelector("span");
p.addEventListener("click", (event) => {
  console.log("P listener: ");
  console.log(event.target);
});
span.addEventListener("click", (event) => {
  console.log("Span listener: ");
  console.log(event.target);
  event.stopPropagation();
});
```

ثم انقر على `span` مرة أخرى:

```html
"Span listener: "
<span>Click me~!</span>
```
 
هذه المرة، لا نرى مستمع `p` يعمل. الحدث لا يُشغل أبدًا لعنصر `p`، لأننا أخبرناه بإيقاف الانتشار أثناء معالجته لعنصر `span` الفرعي.

يمكن اعتبار تفويض الأحداث عكس ذلك. إنها عملية أخذ حدث تم التقاطه، وتفويضه إلى عنصر آخر.

بالعودة إلى شفرتنا، دعنا نحدثها بحيث يؤدي النقر على عنصر `span` إلى تغييره إلى اللون الأحمر:

```js
const p = document.querySelector("p");
const span = document.querySelector("span");
p.addEventListener("click", (event) => {});
span.addEventListener("click", (event) => {
  event.target.style.color = "red";
});
```

لكن ماذا لو كان لديك عشرون عنصر `span`؟ أو ربما تستخدم JavaScript لإنشاء المزيد من عناصر `span` أثناء التشغيل؟

بدلاً من إرفاق مستمع حدث لكل عنصر `span` على حدة، يمكنك فعليًا استخدام المستمع على عنصر `p` لجميعها. بعبارة أخرى، يمكنك تفويض معالجة نقرات `span` إلى عنصر `p` الأصل.

قد تبدو شفرتنا الآن شيئًا مثل هذا:

```js
const p = document.querySelector("p");
p.addEventListener("click", (event) => {
  event.target.style.color = "red";
});
```

لاحظ كيف لم يعد لدينا أي مستمع مرتبط بعنصر `span` على الإطلاق. لقد فوضت معالجة الحدث بشكل صحيح إلى عنصر `p`. لكن هل يعمل ذلك؟

دعنا ننشئ بعض عناصر `span` الإضافية ونرى:

```html
<p>
  <span>Click me~!</span>
  <span>Click me~!</span>
  <span>Click me~!</span>
  <span>Click me~!</span>
</p>
```

الآن، في كل مرة تنقر على `span`، يصبح نص ذلك العنصر أحمر.

وبهذه الطريقة، باستخدام مستمع حدث واحد فقط سمحنا بشكل صحيح لحدث `click` أن يتصاعد من عناصر `span` إلى العنصر الأب `p`، وفوضنا منطق ذلك الحدث `click` إلى عنصر `p`.

يمكن أن يكون انتشار الأحداث وتفويضها موضوعًا معقدًا، خاصة مع العناصر المتداخلة بكثافة مثل الجداول. ننصحك باستكشاف هذا أكثر وتجربة بعض الشفرات التي كتبناها هنا.
