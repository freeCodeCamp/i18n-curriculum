# --interactive--

프로그래밍에서 문자가 숫자로 어떻게 나타나는지 이해하는 것은 기본입니다. 여기서 ASCII가 등장합니다. ASCII는 American Standard Code for Information Interchange의 약자로, 컴퓨터에서 텍스트를 나타내기 위해 사용하는 문자 인코딩 표준입니다. 각 문자에 숫자 값을 할당하며, 이 값은 기계가 보편적으로 인식합니다.

이번 수업에서는 ASCII가 무엇인지, 어떻게 작동하는지, 그리고 JavaScript 메서드인 `charCodeAt()`와 `fromCharCode()`가 문자 인코딩과 어떻게 관련되는지 살펴보겠습니다. JavaScript 문자열은 내부적으로 유니코드(UTF-16)를 사용하지만, ASCII 값은 유니코드의 처음 128자와 일치하기 때문에 ASCII 기반 예제가 JavaScript에서 작동합니다.

ASCII는 문자(글자, 숫자, 기호 등)를 숫자 값으로 인코딩하는 체계입니다. 각 문자는 특정 숫자에 매핑됩니다.

예를 들어, 대문자 `A`는 ASCII에서 숫자 `65`로 나타내며, 소문자 `a`는 `97`로 나타냅니다. 이 인코딩 덕분에 컴퓨터는 텍스트를 저장하고 조작할 수 있습니다.

ASCII 표준은 다음 128자를 포함합니다:

- 대문자 및 소문자 영어 알파벳 (A-Z, a-z).
- 숫자 (0-9).
- 일반적인 구두점 및 기호 (!, @, # 등).
- 제어 문자 (줄 바꿈, 탭 등).

JavaScript에서는 `charCodeAt()` 메서드를 사용해 문자의 숫자 코드를 접근할 수 있습니다. 이 메서드는 지정한 인덱스의 문자의 UTF-16 코드 단위를 반환합니다. 처음 128자에 대해서는 이 값이 ASCII 코드와 일치합니다.

예제를 살펴보겠습니다:

:::interactive_editor

```js
let letter = "A";
console.log(letter.charCodeAt(0));  // 65
```

:::

이 예제에서 `A`는 문자열의 첫 번째 문자이며, `charCodeAt(0)`를 호출하면 그 숫자 코드(기본 라틴 문자에 대해 ASCII 값과 일치하는)인 `65`가 반환됩니다.

다른 문자에 대해서도 이 메서드를 사용해 숫자 코드 값을 찾을 수 있습니다:

:::interactive_editor

```js
let symbol = "!";
console.log(symbol.charCodeAt(0));  // 33
```

:::

여기서는 느낌표 `!`의 숫자 코드가 `33`로 반환되며, 이는 ASCII 값과 일치합니다.

`charCodeAt()`는 문자의 숫자 코드를 가져오는 데 도움을 주는 반면, `fromCharCode()` 메서드는 그 반대로 작동합니다: UTF-16 코드 단위(기본 문자에 대해 ASCII와 일치)를 해당 문자로 변환합니다.

실제로 어떻게 작동하는지 보겠습니다:

:::interactive_editor

```js
let char = String.fromCharCode(65);
console.log(char);  //  A
```

:::

이 예제에서 `fromCharCode(65)`는 숫자 코드 `65` (`A`의 ASCII 값과 일치)를 문자 `A`로 변환합니다.

또 다른 예로 숫자 `97`를 해당 소문자 문자로 변환할 수 있습니다:

:::interactive_editor

```js
let char = String.fromCharCode(97);
console.log(char);  // a
```

:::

이 메서드들은 문자의 숫자 코드 값을 기반으로 문자를 조작하거나 비교할 때 특히 유용합니다.

예를 들어, `charCodeAt()`를 사용해 문자가 대문자인지, 소문자인지, 숫자인지 ASCII 값을 비교해 확인할 수 있습니다.

반면 `fromCharCode()`는 ASCII 코드에서 문자를 동적으로 생성하는 데 사용할 수 있습니다.
