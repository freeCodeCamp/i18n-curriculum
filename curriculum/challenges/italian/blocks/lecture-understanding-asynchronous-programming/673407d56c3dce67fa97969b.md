# --interactive--

Nelle lezioni precedenti, hai imparato la programmazione asincrona che permette ad altro codice di essere eseguito mentre aspetti che alcune attività che richiedono tempo vengano completate, come recuperare dati da un server, leggere dati da un file e così via.

`async`/`await`, basato sulle promise, rende più facile scrivere e leggere codice asincrono. Quando metti la parola chiave `async` davanti a una funzione, significa che quella funzione restituirà sempre una `Promise`. Solo all’interno di una funzione `async` puoi usare la parola chiave `await`, che ti permette di aspettare che una `Promise` venga risolta prima di passare alla riga successiva di codice. Ecco un esempio per illustrare come funziona `async`/`await`:

:::interactive_editor

```js
async function delayedGreeting(name) {
  console.log("A Messenger entered the chat...");
  await new Promise(resolve => setTimeout(resolve, 2000));
  console.log(`Hello, ${name}!`);
}

delayedGreeting("Alice");
console.log("First Printed Message!");
```

:::

In questo codice, definiamo una funzione `async` chiamata `delayedGreeting`. All’interno di questa funzione, usiamo `await` per mettere in pausa l’esecuzione per 2 secondi. Dopo il ritardo, stampa un saluto.

Quando chiamiamo questa funzione, vedrai apparire `First Printed Message!` prima del saluto. Questo perché la funzione è asincrona: non blocca l’esecuzione del resto del codice.

Uno dei maggiori vantaggi di `async`/`await` è la gestione degli errori. Con le promise, spesso dovevamo usare il metodo `.catch()` per gestire gli errori. Con `async`/`await` possiamo usare i blocchi `try`/`catch`. Ecco un esempio:

```js
async function fetchUserData() {
  try {
    let response = await fetch(`https://api.example.com/users`);
    let userData = await response.json();
    console.log(userData);
  } catch (error) {
    console.log("Error fetching user data:", error);
  }
}

fetchUserData();
```

In questo esempio, usiamo `async`/`await` per recuperare dati utente da un’API. La parola chiave `await` è usata due volte: una per aspettare che l’operazione fetch sia completata, e un’altra per aspettare che il parsing JSON finisca. Se si verifica un errore durante questo processo, verrà catturato nel blocco `catch`.
