# --interactive--

**참고**: 일부 인터랙티브 예제는 아직 배우지 않은 CSS와 JavaScript를 사용할 수 있습니다. 모든 코드를 이해하려고 걱정하지 마세요. 이 예제들의 목적은 디자인 개념을 미리 보여줘서 어떻게 작동하는지 더 잘 이해할 수 있게 하는 것입니다. 미리 보기를 보려면 인터랙티브 편집기를 활성화하세요.

무한 스크롤은 사용자가 페이지를 아래로 스크롤할 때 더 많은 콘텐츠를 불러오는 디자인 패턴입니다. 종종 Twitter 같은 소셜 미디어 사이트에서 사용됩니다. 예를 들어, 로그인한 상태에서 더 많은 트윗을 보고 싶으면 아래로 스크롤하면 더 많은 트윗이 로드됩니다. 이것이 무한 스크롤의 예입니다.

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<div class="infinite-scroll">
  <div class="post">Post 1</div>
  <div class="post">Post 2</div>
  <div class="post">Post 3</div>
  <div class="post">Post 4</div>
  <div class="post">Post 5</div>
  <div class="post">Post 6</div>
  <div class="post">Post 7</div>
  <div class="post">Post 8</div>
  <div class="post">Post 9</div>
  <div class="post">Post 10</div>
  <!-- More posts will load here as the user scrolls down -->
</div>

<script src="index.js"></script>
```

```css
.infinite-scroll {
  display: flex;
  flex-direction: column;
  gap: 12px;
}

.post {
  padding: 12px;
  border: 1px solid #ccc;
  border-radius: 4px;
}
```

```js
// Simulate loading more content as the user scrolls
window.addEventListener('scroll', () => {
  if (window.innerHeight + window.scrollY >= document.body.offsetHeight) {
    loadMorePosts();
  }
});

function loadMorePosts() {
  const container = document.querySelector('.infinite-scroll');
  for (let i = 0; i < 3; i++) {
    const post = document.createElement('div');
    post.className = 'post';
    post.textContent = `Post ${container.children.length + 1}`;
    container.appendChild(post);
  }
}
```

무한 스크롤은 페이지 나누기 대신 사용되기도 합니다. 페이지 나누기는 콘텐츠를 여러 페이지로 나누는 디자인 패턴입니다. 많은 콘텐츠를 표시할 때 자주 사용됩니다. 예를 들어 Google에서 검색할 때 여러 페이지에 검색 결과가 나오는 것이 페이지 나누기입니다. 페이지 나누기는 다음 페이지로 가려면 버튼을 클릭해야 합니다. 무한 스크롤은 계속 아래로 스크롤하면 더 많은 콘텐츠가 로드됩니다.

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<div class="pagination">
  <div class="post">Post 1</div>
  <div class="post">Post 2</div>
  <div class="post">Post 3</div>
  <!-- More posts will be on the next pages -->
</div>
<div class="pagination-controls">
  <button class="prev" disabled>Previous</button>
  <button class="next">Next</button>
</div>
<script src="index.js"></script>
```

```css
.pagination {
  display: flex;
  flex-direction: column;
  gap: 12px;
}

.post {
  padding: 12px;
  border: 1px solid #ccc;
  border-radius: 4px;
}

.pagination-controls {
  margin-top: 12px;
}

.pagination-controls button {
  padding: 8px 16px;
  margin-right: 8px;
}
```

```js
let currentPage = 1;
const postsPerPage = 3;
const totalPosts = 9; 
const totalPages = Math.ceil(totalPosts / postsPerPage);
const container = document.querySelector('.pagination');
const prevButton = document.querySelector('.prev');
const nextButton = document.querySelector('.next'); 

function renderPosts() {
  container.innerHTML = '';
  const start = (currentPage - 1) * postsPerPage;
  const end = start + postsPerPage;
  for (let i = start; i < end && i < totalPosts; i++) {
    const post = document.createElement('div');
    post.className = 'post';
    post.textContent = `Post ${i + 1}`;
    container.appendChild(post);
  }
  prevButton.disabled = currentPage === 1;
  nextButton.disabled = currentPage === totalPages;
}

prevButton.addEventListener('click', () => {
  if (currentPage > 1) {
    currentPage--;
    renderPosts();
  }
});

nextButton.addEventListener('click', () => {
  if (currentPage < totalPages) {
    currentPage++;
    renderPosts();
  }
});

renderPosts();
```

무한 스크롤을 디자인에 적용할 때 몇 가지 모범 사례를 기억하세요. 첫 번째 고려사항은 사용자가 클릭할 때 다음 결과 세트를 불러오는 "더 보기" 버튼을 제공하는 것입니다. 이는 사용자가 언제 더 많은 콘텐츠를 보고 싶은지 제어할 수 있게 하는 좋은 방법입니다.

:::interactive_editor

```html

  <div class="post">Post 2</div>
  <div class="post">Post 3</div>
  <!-- More posts will load when the user clicks "Load More" -->
</div>
<button class="load-more">Load More</button>
<script src="index.js"></script>
```

```css
.infinite-scroll {
  display: flex;
  flex-direction: column;
  gap: 12px;
}

.post {
  padding: 12px;
  border: 1px solid #ccc;
  border-radius: 4px;

  margin-top: 12px;

```

```js
const loadMoreButton = document.querySelector('.load-more');
const container = document.querySelector('.infinite-scroll');   
loadMoreButton.addEventListener('click', () => {
  loadMorePosts();
});

function loadMorePosts() {
  for (let i = 0; i < 3; i++) {
    const post = document.createElement('div');
    post.className = 'post';
    post.textContent = `Post ${container.children.length + 1}`;
    container.appendChild(post);
  }
}
```

:::

또 다른 고려사항은 "뒤로" 버튼을 추가하는 것입니다. 이 버튼은 사용자가 스크롤을 모두 올리지 않고도 이전 페이지로 돌아갈 수 있게 해줍니다. 이는 더 나은 사용자 경험을 만들고 탐색을 더 편리하게 합니다.

때때로 "맨 위로" 버튼이 있어 사용자를 결과 페이지 맨 위로 안내하는 디자인도 볼 수 있습니다. 또 다른 고려사항은 로딩 표시기를 제공하는 것입니다. 사용자는 더 많은 콘텐츠가 로드되고 있다는 명확한 표시를 받아야 합니다. 그렇지 않으면 페이지가 고장 난 것으로 생각할 수 있습니다.

:::interactive_editor

```js
const container = document.querySelector('.infinite-scroll');
const loadingIndicator = document.querySelector('.loading-indicator');
const backToTopBtn = document.getElementById('back-to-top');

window.addEventListener('scroll', () => {
  if (window.scrollY > 400) {
    backToTopBtn.style.display = 'block';
  } else {
    backToTopBtn.style.display = 'none';
  }

  if (window.innerHeight + window.scrollY >= document.body.offsetHeight) {
    loadMorePosts();
  }
});

backToTopBtn.addEventListener('click', () => {
  window.scrollTo({ top: 0, behavior: 'smooth' });
});

function loadMorePosts() {
  loadingIndicator.style.display = 'block';
  setTimeout(() => {
    for (let i = 0; i < 3; i++) {
      const post = document.createElement('div');
      post.className = 'post';
      post.textContent = `Post ${container.children.length + 1}`;
      container.appendChild(post);
    }
    loadingIndicator.style.display = 'none';
  }, 1000);
}
``` 

:::

마지막 고려사항 중 하나는 푸터를 항상 접근 가능하게 유지하는 것입니다. 푸터에 중요한 정보가 포함되어 있다면 사용자가 언제든지 접근할 수 있어야 합니다.

결론적으로, 무한 스크롤은 웹사이트에서 콘텐츠를 표시하는 훌륭한 방법입니다. 하지만 무한 스크롤을 디자인할 때 모범 사례를 기억해서 최상의 사용자 경험을 제공하세요.
