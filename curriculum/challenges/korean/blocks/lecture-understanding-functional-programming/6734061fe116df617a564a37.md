# --interactive--

커링은 여러 인수를 받는 함수를 하나의 인수만 받는 함수들의 수열로 변환하는 기법입니다.

기본 예제로 시작해 보겠습니다. 두 수를 더하는 함수가 있다고 상상해 보세요:

:::interactive_editor

```js
function add(a, b) {
  return a + b;
}

console.log(add(3, 4)); // 7
```

:::

이 함수는 두 개의 인수를 받아 그 합을 반환합니다. 이제 이 함수를 커링하는 방법을 살펴보겠습니다:

:::interactive_editor

```js
function curriedAdd(a) {
  return function(b) {
    return a + b;
  }
}

console.log(curriedAdd(3)(4)); // 7
```

:::

이 커링된 코드에서는 두 인수를 한 번에 받는 대신, 첫 번째 인수를 받고 또 다른 함수를 반환합니다. 반환된 함수가 두 번째 인수를 받아 덧셈을 수행합니다. `curriedAdd(3)(4)`처럼 호출하는데, 각 소괄호 쌍이 함수 호출을 나타냅니다.

그런데 왜 이렇게 할까요?

커링을 사용하면 특별한 함수를 쉽게 만들 수 있습니다. 예를 들어, 어떤 수에든 항상 5를 더하는 함수를 만들 수 있습니다:

:::interactive_editor

```js
function curriedAdd(a) {
  return function(b) {
    return a + b;
  }
}

const addFive = curriedAdd(5);
console.log(addFive(10)); // 15
console.log(addFive(20)); // 25
```

:::

여기서 `addFive`은 어떤 수를 주어도 항상 5를 더할 준비가 된 함수입니다. 이것은 부분 적용의 간단한 예로, 함수에 일부 인수를 고정하여 더 적은 인수를 받는 다른 함수를 만드는 것입니다.

우리 예제는 두 인수를 받는 함수에 초점을 맞췄지만, 커링은 어떤 수의 인수를 받는 함수에도 적용할 수 있습니다.

JavaScript 학습을 계속하면서 커링이 코드를 더 깔끔하고 표현력 있게 만드는 많은 상황을 만나게 될 것입니다.

## --text--

다음 중 JavaScript에서 커링된 함수의 예는 무엇인가요?

## --answers--

```js
function add(a, b, c) {
  return a + b + c;
}
```

### --feedback--

이것은 모든 매개변수를 한 번에 받는 일반 함수입니다.

---

```js
function add(a) {
  return function (b) {
    return function (c) {
      return a + b + c;
    };
  };
}
```

---

```js
function add(a, b, c) {
  return a() + b() + c();
}
```

### --feedback--

이 함수는 모든 매개변수를 한 번에 받고 함수처럼 호출하는데, 이는 커링된 함수의 정의에 맞지 않습니다.

---

```js
function add(a) {
  function inner(b) {
    return a + b;
  }
}
```

### --feedback--

내부 함수가 반환되지 않으므로 `add(2)`는 `undefined`을 반환하고 `add(2)(3)`는 오류를 발생시킬 것입니다.
