# --description--

웹 성능을 측정하고 개선하려면 First Contentful Paint(FCP), Speed Index, Total Blocking Time(TBT), 이탈률, 고유 사용자 수 같은 주요 성능 지표를 사용합니다. 이제 이 지표들을 하나씩 살펴보겠습니다.

First Contentful Paint, 즉 FCP는 텍스트나 이미지 같은 첫 번째 콘텐츠가 화면에 얼마나 빨리 나타나는지를 측정합니다. 좋은 FCP는 1.8초 이하로 간주되고, 나쁜 FCP는 보통 3초 이상입니다. Chrome 개발자 도구를 사용해 FCP를 확인하는 방법은 다음과 같습니다.

단계 1: Chrome 개발자 도구 열기

- Google Chrome을 엽니다.

- 테스트할 웹페이지로 이동합니다.

- "F12"를 누르거나 마우스 오른쪽 버튼을 클릭하고 "검사"를 선택합니다.

단계 2: 개발자 도구에서 "Lighthouse" 탭을 찾아 클릭합니다.

단계 3: "페이지 로드 분석" 버튼을 클릭하고 분석이 완료될 때까지 기다립니다.

이제 결과를 분석할 수 있습니다. "Metrics" 섹션으로 스크롤하여 "First Contentful Paint" 표시를 찾으세요. 정확한 시간은 밀리초 또는 초 단위로 표시됩니다.

FCP와 이전에 살펴본 Interaction to Next Paint(INP)는 모두 성능 지표지만 사용자 경험의 다른 측면을 측정합니다. 혼동하지 않는 것이 중요합니다.

다시 말해, FCP는 사용자가 로드 후 콘텐츠를 얼마나 빨리 보는지에 관한 것이고, INP는 클릭이나 입력 후 피드백을 얼마나 빨리 받는지에 관한 것입니다.

다음은 Speed Index입니다.

Speed Index는 페이지의 보이는 부분이 얼마나 빨리 그려지는지를 측정합니다. 점수가 낮을수록 성능이 더 좋습니다! Lighthouse 보고서에서 Speed Index를 확인할 수 있습니다.

또한 Total Blocking Time(TBT)이 있습니다. 이는 무거운 JavaScript 작업으로 인해 메인 스레드가 얼마나 오래 차단되는지를 보여줍니다. TBT가 높으면 사용자가 느린 반응을 경험합니다. TBT를 개선하려면 긴 작업을 나누고 필수적이지 않은 스크립트는 지연시키세요.

다음은 이탈률입니다. 이는 방문자가 상호작용 없이 사이트를 떠나는 비율입니다. 이탈률이 높다면 페이지가 너무 느릴 수 있습니다.

마지막으로 고유 사용자 수가 있습니다. 이 지표는 사이트를 방문하는 개별 방문자 수를 추적합니다. 이탈률과 고유 사용자 수는 Google Analytics를 사용해 확인할 수 있습니다. 이를 통해 지표를 모니터링하고 참여도를 높일 수 있습니다.
