# --interactive--

在 JavaScript 中，`null` 和 `undefined` 是两种不同的数据类型，它们都表示没有值，但在比较中的表现却不同。 了解这些类型在各种比较场景中的交互方式，对于编写健壮且无错误的代码至关重要。

让我们从 `undefined` 类型开始。 当变量已被声明但尚未赋值时，它就是 `undefined` 变量。 它是未初始化变量和未提供参数的函数参数的默认值。

`null` 类型是一个赋值，它代表一个有意的非值。 通常用来表示变量故意没有值。

在使用相等运算符（`==`）比较 `null` 和 `undefined` 时，JavaScript 会执行类型强制。 这意味着在进行比较之前，它会尝试将操作数转换为相同的类型。 在这种情况下，`null` 和 `undefined` 被认为是相等的：

:::interactive_editor

```js
console.log(null == undefined); // true
```

:::

但是，在使用严格相等运算符（`===`）时，该运算符同时检查值和类型而不执行类型强制，`null` 和 `undefined` 是不相等的：

:::interactive_editor

```js
console.log(null === undefined); // false
```

:::

在编写条件语句或在代码中执行相等检查时，必须牢记这一区别。 当使用相等运算符（`==`）将 `null` 或 `undefined` 与其他值进行比较时，行为可能会出乎意料。 例如：

:::interactive_editor

```js
console.log(null == 0);  // false
console.log(null == ''); // false
console.log(undefined == 0); // false
console.log(undefined == ''); // false
```

:::

这些比较会返回 `false`，因为在使用相等运算符时，`null` 和 `undefined` 只会彼此相等（以及自身相等）。 `null` 在其他比较中的行为尤其棘手：

:::interactive_editor

```js
console.log(null > 0);  // false
console.log(null == 0); // false
console.log(null >= 0); // true
```

:::

而 `undefined` 在数字上下文中总是转换为 `NaN`，这将使所有与 `undefined` 进行的数值比较返回 `false`：

:::interactive_editor

```js
console.log(undefined > 0);  // false
console.log(undefined < 0);  // false
console.log(undefined == 0); // false
```

:::

鉴于这些细微差别，通常建议在比较值时使用严格相等运算符，尤其是在处理 `null` 和 `undefined` 时。 这种方法有助于避免意外的类型强制，使代码的行为更可预测。

总之，虽然 `null` 和 `undefined` 都用来表示没有值，但它们在比较中的表现不同。 了解这些差异是编写清晰无误的 JavaScript 代码的关键。
