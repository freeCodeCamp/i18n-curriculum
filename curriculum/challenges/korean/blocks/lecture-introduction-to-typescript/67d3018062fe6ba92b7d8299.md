# --description--

이 수업에서는 기본 원시 타입을 다루는 방법과 변수에 대한 타입 주석이 어떻게 작동하는지 배우세요.

원시 데이터 타입 `string`, `null`, `undefined`, `number`, `boolean`, 그리고 `bigint`에 대해 TypeScript는 해당하는 타입 키워드를 제공합니다.

이번 수업에서는 자주 사용하는 원시 타입에 집중하고 `bigint`에 대해서는 자세히 다루지 않습니다. `bigint` 데이터 타입은 JavaScript에서 표준 정수 타입보다 큰 숫자를 나타내는 숫자 데이터 타입입니다.

먼저 `string`부터 시작해 각 원시 타입을 살펴보겠습니다. TypeScript에서 문자열을 타입 지정하는 방법은 다음과 같습니다:

```ts
let firstName: string = "Angie";
```

일반 JavaScript와 달리 변수 `firstName`는 `string` 타입을 기대합니다. 그래서 숫자 값을 다시 할당하려 하면 다음과 같은 오류가 발생합니다:

```ts
let firstName: string = "Angie";
firstName = 9 // Type 'number' is not assignable to type 'string'.
```

다음으로 살펴볼 타입은 `number` 타입입니다. 예시는 다음과 같습니다:

```ts
let age: number = 16;
```

`number` 타입에는 정수와 부동소수점을 할당할 수 있습니다. 하지만 `bigint` 타입 값을 다시 할당하려 하면 다음과 같은 오류가 발생합니다:

```ts
let age: number = 16;

// Type 'bigint' is not assignable to type 'number'.
age = 123n;
```

다음으로 살펴볼 타입은 `boolean` 타입입니다. 몇 가지 예시는 다음과 같습니다:

```ts
let isLoggedIn: boolean = true;
let isAdmin: boolean = false;
```

마지막 두 타입은 `null`와 `undefined` 타입입니다.

이전에 배운 JavaScript 수업에서 기억하듯이, `null`은 의도적으로 값이 없음을 나타냅니다. `undefined`은 변수가 선언되었지만 값이 할당되지 않았음을 의미합니다.

이 타입들을 다루는 예시는 함수 타입과 유니언 타입을 배울 때 다시 보게 될 것입니다.

변수에 타입 주석이 어떻게 작동하는지 살펴봤으니, 항상 변수에 명시적으로 타입을 지정해야 하는지 의문이 생길 수 있습니다.

만약 다음과 같이 간단한 타입 주석이 있다면 명시적 타입을 사용하지 않아도 될 수 있습니다.

```ts
let firstName: string = "Angie";
```

그 이유는 TypeScript가 값에 따라 타입을 자동으로 추론하려 하기 때문입니다. 그래서 다음과 같이 작성하면 TypeScript는 이것이 문자열임을 이해합니다.

```js
let firstName = "Angie";
```

TypeScript가 타입을 명확히 추론하지 못하거나 함수 매개변수나 복잡한 객체 같은 상황에서 특정 타입을 강제하고 싶을 때는 변수에 명시적으로 타입을 지정해야 합니다.
