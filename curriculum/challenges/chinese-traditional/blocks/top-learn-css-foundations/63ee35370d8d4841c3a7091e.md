# --description--

組合子允許我們以不同於群組或串鏈的方式組合多個選擇器，因為它們表現出選擇器之間的關係。組合子總共有四種型別，但現在我們只會向你展示後代組合子，它在 CSS 中由選擇器之間的一個空格表示。後代組合子只有在元素同時具有符合前一個選擇器的祖先（父元素、祖父元素等）時，才會選取符合最後一個選擇器的元素。

所以像 `.ancestor .child` 這樣的寫法會選取具有 `child` 類別的元素，前提是它有一個具有 `ancestor` 類別的祖先。另一種想法是，只有當 `child` 嵌套在 `ancestor` 裡面時，不論層級多深，才會被選取。快速看看下面的範例，試著判斷根據提供的 CSS 規則會選取哪些元素：

```html
<!-- index.html -->

<div class="ancestor"> <!-- A -->
  <div class="contents"> <!-- B -->
    <div class="contents"> <!-- C -->
    </div>
  </div>
</div>

<div class="contents"></div> <!-- D -->
```

```css
/* styles.css */

.ancestor .contents {
  /* some declarations */
}
```

在上述範例中，具有 `contents` 類別的前兩個元素（`B` 和 `C`）會被選取，但最後一個元素（`D`）不會。你的猜測正確嗎？

你可以為規則添加的組合器數量實際上沒有限制，所以 `.one .two .three .four` 是完全有效的。這會選擇一個具有 `four` 類別的元素，前提是它有一個具有 `three` 類別的祖先，且該祖先又有一個具有 `two` 類別的祖先，依此類推。不過，你通常會想避免嘗試選擇需要這種層次巢狀的元素，因為這會變得相當混亂且冗長，並且在特異性方面可能會引起問題。

## --text--

後代組合器的作用是什麼？

## --answers--

它將具有相同宣告的某些類別群組在一起。

---

它提供選擇具有相同 `class` 和 `id` 的元素的能力。

---

它允許你根據元素與其祖先（父元素、祖父元素等）的關係來選取元素。
