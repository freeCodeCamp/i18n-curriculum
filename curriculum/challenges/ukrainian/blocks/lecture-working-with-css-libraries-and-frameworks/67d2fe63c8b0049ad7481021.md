# --description--

CSS-препроцесор — це інструмент, який розширює стандартний CSS потужними функціями, такими як змінні, міксини, вкладеність і успадкування селекторів.

Деякі з цих функцій, наприклад змінні та вкладеність, зараз підтримуються або отримують більше підтримки в нативному CSS, і ця тенденція, ймовірно, триватиме. Однак у ранніх версіях нативного CSS ці функції не підтримувалися.

Саме тому CSS-препроцесори стали настільки популярними спочатку. Вони беруть CSS-правила і стилі, які ви пишете, разом із їх розширеним синтаксисом, і компілюють їх у нативний CSS-файл, який розуміють браузери.

Використовуючи CSS-препроцесор, ви можете структурувати свій CSS-код більш логічно та багаторазово. Ви також отримаєте доступ до потужних функцій, таких як міксини, які безпосередньо не підтримуються CSS.

Деякі з найпопулярніших CSS-препроцесорів — це Sass, Less і Stylus.

Поговоримо про Sass.

Sass означає «Syntactically Awesome Style Sheets». Він сумісний з усіма версіями CSS і підтримується великою спільнотою розробників.

Sass підтримує такі функції:

- Змінні, щоб зберігати і повторно використовувати значення по всьому стилю.

- Вкладені CSS-правила, щоб створити візуальну ієрархію у вашому файлі.

- Модулі, щоб розбити стилі на кілька стилістичних файлів.

- Міксини, щоб повторно використовувати оголошення CSS по всьому сайту.

- Успадкування, щоб кілька CSS-селекторів могли ділитися властивостями.

- І оператори для базових математичних операцій.

Sass також підтримує два синтаксиси.

Синтаксис SCSS означає Sassy CSS, що є надмножиною CSS. Це означає, що SCSS розширює базовий синтаксис CSS.

SCSS — це найпоширеніший синтаксис, який ви будете використовувати і зустрічати при роботі з Sass. Він вимагає використання фігурних дужок (`{` і `}`) навколо CSS-властивостей і крапок з комою (`;`) в кінці оголошень CSS, так само як і нативний CSS.

Ці файли мають розширення `.scss`. Ось приклад:

```scss
$primary-color: #3498eb;

header {
  background-color: $primary-color;
}
```

Те, що ви бачите тут зверху, — це змінна, визначена в SCSS. Ця змінна використовується в CSS-правилі нижче.

Порівняймо SCSS із менш поширеним синтаксисом — відступним синтаксисом. Його також називають «синтаксисом Sass», оскільки це був оригінальний синтаксис Sass.

Цей синтаксис базується на відступах для визначення правил. Ось приклад:

```scss
$primary-color: #3498eb

header
  background-color: $primary-color
```

Зверніть увагу, що тут немає фігурних дужок навколо CSS-правила або крапок з комою в кінці оголошень CSS. Зверху також видно змінну. Зверніть увагу, що її ім’я починається з знака долара (`$`) у цьому синтаксисі, що відрізняється від змінних нативного CSS, але працює дуже схоже.

Sass також підтримує потужну функцію, яка називається міксинами.

Міксини дозволяють згрупувати кілька CSS-властивостей і їх значень під одним ім’ям і повторно використовувати цей блок CSS-коду по всьому стилю.

Це робить ваш CSS-код менш повторюваним і легшим у підтримці, тому що, якщо ви зміните щось у цьому блоці, зміни застосуються скрізь, де використовується міксин.

Оскільки міксини можуть мати кастомні та описові імена, вони допомагають зрозуміти, що робить кожен блок CSS-коду, що робить ваші стилі легшими для розуміння.

І міксини також сприяють послідовності стилів, застосовуючи однакові стилі.

Ось приклад міксина в синтаксисі SCSS:

```scss
@mixin center-flex {
  display: flex;
  justify-content: center;
  align-items: center;
}
```

Щоб визначити міксин, починайте з написання директиви `@mixin`, за якою йде ім’я міксина. У цьому випадку міксин називається `center-flex`. Він має три CSS-властивості для центрування елементів за допомогою гнучкого контейнера.

Потім, коли міксин визначено, ви використовуєте директиву `@include`, щоб включити ці властивості в CSS-правило. Просто напишіть `@include` і потім ім’я міксина. У цьому випадку ім’я — `center-flex`:

```scss
section {
  @include center-flex;
  height: 500px;
  background-color: #3289a8;
}
```

Ось повний код із міксином і CSS-правилом:

```scss
@mixin center-flex {
  display: flex;
  justify-content: center;
  align-items: center;
}

section {
  @include center-flex;
  height: 500px;
  background-color: #3289a8;
}
```

Ви можете включати міксин у стільки CSS-правил, скільки потрібно. Якщо потрібно внести зміни, достатньо змінити сам міксин, і зміни автоматично застосуються скрізь. Цей приклад у синтаксисі SCSS. Зверніть увагу, що тут є фігурні дужки і крапки з комою.

Ось еквівалент у відступному синтаксисі, також відомому як синтаксис Sass, без фігурних дужок і крапок з комою:

```scss
@mixin center-flex
  display: flex
  justify-content: center
  align-items: center

section
  @include center-flex
  height: 500px
  background-color: #3289a8
```

А це скомпільований CSS-код, який браузер фактично інтерпретує після компіляції файлу у нативний CSS:

```css
section {
  display: flex;
  justify-content: center;
  align-items: center;
  height: 500px;
  background-color: #3289a8;
}
```

Зверніть увагу, що він включає три властивості міксина зверху: `display`, `justify-content` і `align-items`.

Це лише простий приклад того, на що здатні CSS-препроцесори. Вони мають багато потужних функцій, які варто вивчати і застосовувати у вашому щоденному робочому процесі.

Однак у них є потенційні недоліки, про які варто знати.

По-перше, CSS-препроцесори повинні компілювати CSS-правила у стандартний CSS. Хоча потенційне навантаження від цього процесу компіляції зазвичай мінімальне з точки зору часу і ресурсів, це варто враховувати у реальних проєктах.

По-друге, вони можуть створювати складнощі з налагодженням, оскільки браузери використовують безпосередньо скомпільований CSS. Визначити, що саме генерує проблемний стиль із розширеного синтаксису, може зайняти кілька додаткових кроків у порівнянні зі стандартним CSS.

Проте переваги CSS-препроцесорів зазвичай переважають їхні недоліки, особливо для складних проєктів. Вони дуже допомагають писати чистіший, багаторазовий, менш повторюваний і масштабований CSS.
