# --description--

У розробці програмного забезпечення бібліотека — це як набір інструментів для розробників.

Замість того, щоб реалізовувати кожну частину коду з нуля, бібліотека надає вам заздалегідь написаний і багаторазово використовуваний код, такий як функції, класи та структури даних, які ви можете використовувати у своїх проєктах.

Python має велику стандартну бібліотеку з багатьма різними вбудованими модулями. Вони всі стандартизовані, добре перевірені рішення для багатьох проблем і завдань, з якими ви щодня стикаєтеся як програміст, наприклад:

* Взаємодія з операційною системою.
    
* Робота з файлами.
    
* Мережеві операції.
    
* Робота з датою і часом.
    
* Виконання математичних операцій.
    
* Використання регулярних виразів.
    
* Тестування та налагодження вашого коду.
    
* І багато іншого!
    

Деякі приклади популярних вбудованих модулів — це `math`, `random`, `re` (скорочення від "регулярні вирази") та `datetime`.

Модуль `math` має корисні функції для виконання більш складних математичних операцій.

Модуль `random` допомагає генерувати випадкові числа.

Модуль `re` використовується для роботи з регулярними виразами.

А модуль `datetime` корисний для роботи з датами і часом у Python.

Але як отримати доступ до змінних, констант, функцій і класів, визначених у цих вбудованих модулях?

Ви використовуєте інструкцію import. Ці інструкції дозволяють імпортувати модулі у ваш Python-скрипт. Зазвичай інструкції import пишуться на початку файлу. Також ви можете налаштовувати їх відповідно до своїх потреб. Спочатку використовується інструкція `import`, після якої йде назва модуля:

```python
import module_name
```

Припустимо, ви хочете імпортувати модуль `math`. У такому разі ви напишете це на початку свого файлу:

```python
import math
```

Потім, якщо вам потрібно викликати функцію з цього модуля у вашому Python-скрипті, ви використовуєте точкову нотацію: назва модуля, потім крапка, а після неї — назва функції:

```python
module_name.function_name() 
```

Наприклад, щоб отримати квадратний корінь із 36, ви напишете `math`, потім крапку, а далі `sqrt`, скорочення від квадратного кореня, і в круглих дужках передасте потрібні аргументи. У цьому випадку нам потрібно передати лише число, з якого хочемо взяти квадратний корінь:

```python
math.sqrt(36)
```

Це найпростіший варіант інструкції import, але є й інші альтернативи.

Якщо потрібно імпортувати модуль під іншим іменем (так званим "аліасом"), ви можете використати такий синтаксис: `as`, а в кінці інструкції import вказати аліас:

```python
import module_name as module_alias
```

Це часто використовують, щоб скоротити довгі назви модулів або уникнути конфліктів імен.

Наприклад, щоб звертатися до модуля `math` як `m` у вашому коді, можна призначити йому аліас так:

```python
import math as m 
```

Потім ви можете отримувати доступ до елементів модуля, використовуючи цей аліас:

```python
m.sqrt(36)
```

Але іноді не потрібно імпортувати весь модуль. Можливо, вам потрібні лише одна або дві конкретні функції чи класи. У Python є саме те, що вам потрібно в такому випадку.

Інструкція import починається з `from`, потім назва модуля, а далі ключове слово `import` і перелік імен елементів, які ви хочете імпортувати:

```python
from module_name import name1, name2
```

Потім ви можете використовувати ці імена без префікса модуля у вашому Python-скрипті.

Якщо хочете призначити аліаси цим іменам, це можна зробити за допомогою ключового слова `as` після кожного, а потім вказати потрібний аліас:

```python
from module_name import name1 as alias1, name2 as alias2
```

Припустимо, ви хочете імпортувати лише функції radians, sine і cosine з модуля `math`. Ви напишете:

```python
from math import radians, sin, cos
```

Тепер ви можете викликати ці функції безпосередньо у вашому коді, без префікса модуля `math`.

Ось більш детальний приклад:

Щоб знайти синус і косинус конкретного кута, спочатку заданого в градусах, ми можемо викликати функцію `radians`, щоб конвертувати його в радіани, а потім викликати функції sine і cosine, передаючи кут у радіанах:

```python
from math import radians, sin, cos

angle_degrees = 40
angle_radians = radians(angle_degrees)

sine_value = sin(angle_radians)
cos_value = cos(angle_radians)

print(sine_value) # 0.6427876096865393
print(cos_value)  # 0.766044443118978
```

Зверніть увагу, що ми викликаємо функції без назви модуля як префікса. Це тому, що ми імпортували функції за допомогою альтернативного синтаксису.

Це зручно, але може призвести до конфліктів імен, якщо у вашому Python-скрипті вже є функції або змінні з такими ж іменами. Тож це варто враховувати, обираючи тип інструкції import.

І нарешті, є інструкція import, що закінчується зірочкою. Зірочка каже Python, що ви хочете імпортувати все з цього модуля, але так, щоб не потрібно було використовувати назву модуля як префікс:

```python
from module_name import *
```

Наприклад, якщо ви зробите це при імпорті модуля `math`, ви зможете викликати будь-яку функцію з цього модуля без вказування назви модуля як префікса. Ось кілька прикладів:

```python
from math import *
print(sqrt(36))  # 6.0
print(pow(5, 2)) # 25.0
print(exp(1))    # 2.718281828459045
```

Однак це зазвичай не рекомендується, бо може призвести до колізій у просторі імен і ускладнити розуміння, звідки беруться певні імена.

Інструкції import працюють однаково для функцій, класів, констант, змінних і будь-яких інших елементів, визначених у модулі.

Ось приклад константи з модуля `math`, числа `pi`:

```python
import math
print(math.pi)
```

А ось приклад класу з модуля `datetime`. Ми створюємо об’єкт дати, що представляє 15 липня 1959 року. Потім призначаємо цей об’єкт `date` змінній і отримуємо доступ до дня, місяця і року окремо, використовуючи точкову нотацію:

```python
import datetime
birthday = datetime.date(1959, 7, 15)
print(birthday.day)    # 15
print(birthday.month)  # 7
print(birthday.year)   # 1959
```

Більше інформації про вміст модуля можна знайти в офіційній документації Python для цього модуля.

Чудово. Тепер, коли ви знаєте більше про модулі, вам також варто знати про цей дуже важливий ідіом у Python-скриптах, бо вони дуже тісно пов’язані:

```python
if __name__ == '__main__': 
    # Code
```

`__name__` — це спеціальна вбудована змінна в Python.

Коли Python-файл виконується безпосередньо, Python встановлює значення цієї змінної як рядок `"__main__"`.

Але якщо Python-файл імпортується як модуль у інший Python-скрипт, значення змінної `__name__` встановлюється як назва цього модуля (зазвичай ім’я файлу без розширення `.py`).

Ось чому ви часто знайдете цю умову в Python-скриптах. Вона містить код, який ви хочете виконати **тільки якщо** Python-скрипт запускається як головна програма:

```python
if __name__ == '__main__': 
    # Code
```

Але якщо скрипт імпортується як модуль, код у цьому блоці не виконується.

Це зручно, бо дозволяє Python-скриптам мати дві цілі. Вони можуть запускатися безпосередньо для виконання основної логіки або імпортуватися в інший модуль без виконання основної логіки.
