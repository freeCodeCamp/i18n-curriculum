# --description--

컴퓨터 과학에서 분할 정복 패러다임은 문제를 재귀적으로 더 작은 하위 문제로 나누는 기법입니다. 이 기법의 핵심 요소 중 하나는 재귀로, 함수가 기본 사례에 도달할 때까지 자신을 반복해서 호출하는 것을 말합니다. 이 수업에서는 병합 정렬 알고리즘을 살펴보면서 분할 정복 기법이 어떻게 작동하는지 더 잘 이해해 보겠습니다.

다음과 같은 숫자 목록이 있다고 가정해 보겠습니다:

```md
42 37 53 17
```

목표는 병합 정렬 알고리즘을 사용해 이 목록을 가장 작은 수부터 가장 큰 수 순으로 정렬하는 것입니다. 첫 번째 단계는 목록을 반으로 나누는 것입니다:

```md
42 37 | 53 17
```

그다음 목록의 왼쪽 부분을 살펴봅니다:

```md
42 37
```

그 하위 목록을 다시 반으로 나누어 각 하위 목록에 항목이 하나만 남을 때까지 나눕니다:

```md
42 | 37
```

항목이 하나만 있는 목록은 기본적으로 정렬된 상태입니다. 다음으로, 각 한 개 요소 하위 목록을 병합하여 정렬된 목록으로 만듭니다:

```md
37 42
```

원래 목록의 오른쪽 부분에 대해서도 같은 과정을 따릅니다:

```javascript
// right side of original list
53 17

// divide the list in half
53 | 17

// merge the lists in sorted order
17 53
```

이제 원래 목록의 두 절반이 모두 정렬되었으니, 이 두 절반을 병합하여 요소들을 정렬합니다:

```markdown
17 37 42 53
```

알고리즘이 코드로 표현된 모습은 다음과 같습니다:

```javascript
function mergeSort(arr) {
    if (arr.length <= 1) {
        return arr;
    }

    const mid = Math.floor(arr.length / 2);
    const left = mergeSort(arr.slice(0, mid));
    const right = mergeSort(arr.slice(mid));

    const sorted = [];
    let i = 0;
    let j = 0;

    while (i < left.length && j < right.length) {
        if (left[i] <= right[j]) {
            sorted.push(left[i]);
            i += 1;
        } else {
            sorted.push(right[j]);
            j += 1;
        }
    }

    return sorted.concat(left.slice(i)).concat(right.slice(j));
}
```

병합 정렬의 시간 복잡도는 `O(n log n)`입니다. 목록이 계속 반으로 나누어지기 때문(`log n`)이며, 그 후 병합되기 때문입니다(`O(n)`). 버블 정렬 같은 다른 정렬 알고리즘과 달리 병합 정렬은 제자리 정렬이 아니며 공간 복잡도는 `O(n)`입니다.
