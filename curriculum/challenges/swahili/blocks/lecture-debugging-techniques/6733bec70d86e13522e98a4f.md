# --description--

Sentensi ya `throw` katika JavaScript hutumika kutupa exception iliyobainishwa na mtumizi. Exception katika programu ni tukio lisilotarajiwa linalotokea na kuingilia mtiririko wa kawaida wa programu.

Kama waprogramu ni muhimu kushughulikia exceptions hizi, ili programu zako zisivunjike ghafla wakati makosa yanapotokea. Hapa kuna sintaksia ya msingi ya sentensi ya `throw`:

```js
throw expression;
```

`expression` katika kesi hii itakuwa ni kitu au thamani inayowakilisha exception unayotaka kutupa. Mifano ya hii ni darasa la exceptions zilizojengwa kama darasa la `Error`, `TypeError`, au `RangeError`. Hapa kuna mfano wa kutumia sentensi ya `throw` kutupa `TypeError`:

```js
function validateNumber(input) {
  if (typeof input !== "number") {
    throw new TypeError("Expected a number, but received " + typeof input);
  }
  return input * 2;
}
```

Katika mfano huu, tunakagua kama aina ya `input` si ya aina ya `number`. Ikiwa siyo, basi tunatupa `TypeError` yenye ujumbe maalum. Vinginevyo, kitendakazi kitarejesha matokeo ya kuzidisha `input` kwa `2`.

Ikiwa ungependa kutupa ujumbe wa makosa wa jumla zaidi, basi unaweza kurejea kijenzi cha `Error` kama hivi:

```js
function divide(numerator, denominator) {
  if (denominator === 0) {
    throw new Error("Cannot divide by zero");
  }
  return numerator / denominator;
}
```

Hapa kuna mfano wa kitendakazi kitakachokagua kama denominator ni `0`. Ikiwa ni hivyo, basi kitatupa ujumbe wa makosa maalum unaosema `Cannot divide by zero`.

Katika funzo lijalo, tutaangalia jinsi ya kutupa ujumbe wa makosa ndani ya muktadha wa blokki ya `try`/`catch` ambayo hutumika kushughulikia exceptions kwa heshima katika JavaScript.
