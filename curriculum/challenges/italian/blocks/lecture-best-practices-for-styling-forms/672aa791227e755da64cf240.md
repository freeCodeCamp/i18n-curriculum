# --interactive--

Come per tutti gli elementi di testo, devi assicurarti che gli stili applicati all'input di testo siano accessibili. Questo significa che il carattere tipografico deve avere una dimensione adeguata e il colore deve avere un contrasto sufficiente con lo sfondo.  

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<form class="accessible-form">
  <label for="username">Username</label>
  <input type="text" id="username" name="username" placeholder="Enter your username">
  <button type="submit">Submit</button>
</form>

```

```css
body {
  background-color: #f9fafb;
  color: #222;
  padding: 2rem;
}

.accessible-form {
  max-width: 320px;
  margin: 0 auto;
}

label {
  display: block;
  font-weight: 600;
  margin-bottom: 0.5rem;
}

input[type="text"] {
  width: 100%;
  padding: 0.6rem 0.8rem;
  font-size: 1rem;
  border: 2px solid #555;
  border-radius: 4px;
  background-color: #fff;
  color: #111;
}

input[type="text"]:focus {
  outline: 3px solid #1e90ff;
  border-color: #1e90ff;
}

button {
  margin-top: 1rem;
  padding: 0.6rem 1rem;
  font-size: 1rem;
  background-color: #1e90ff;
  color: #fff;
  border: none;
  border-radius: 4px;
  cursor: pointer;
}

button:hover,
button:focus {
  background-color: #187bcd;
}

```

:::

Il segnaposto, però, viene spesso dimenticato. È importante ricordare che anche questo è testo e probabilmente dovrai modificare lo stile per garantirne la leggibilità.

Un'altra cosa da tenere a mente è che dovresti permettere all'utente di modificare l'input. Per esempio, se si tratta di un'area di testo non dovresti rimuovere la possibilità di ridimensionarla. L'input dovrebbe anche scalare di conseguenza quando l'utente ingrandisce la pagina.

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<form class="accessible-form">
  <label for="message">Your message</label>
  <textarea id="message" name="message" placeholder="Type your message here..."></textarea>
  <button type="submit">Send</button>
</form>

```

body {

  margin: 0 auto;

  display: block;

  color: #555;
  opacity: 1;

Gli elementi input sono anche focalizzabili. Quando modifichi i tuoi stili devi fare attenzione a preservare un indicatore ben visibile quando l'elemento ha il focus, come un bordo in grassetto. 

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<form class="accessible-form">
  <label for="name">Full name</label>
  <input type="text" id="name" name="name" placeholder="Enter your full name">

  <label for="email">Email address</label>
  <input type="email" id="email" name="email" placeholder="you@example.com">

  <button type="submit">Submit</button>
</form>

```

  color: #222;
  padding: 2rem;

  margin: 0 auto;
}

  display: block;

Inoltre c'è da considerare lo stato di errore. Quando il testo dell'utente non supera la validazione dell'input, c'è un indicatore visivo che mostra che c'è qualcosa che non va. 

Per aggiornare dinamicamente il messaggio di errore, dovresti usare JavaScript, che imparerai nelle lezioni future. 

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<form class="accessible-form">
  <label for="email">Email address</label>
  <input 
    type="email" 
    id="email" 
    name="email" 
    placeholder="you@example.com" 
    aria-describedby="email-error"
  >
  <p id="email-error" class="error-message">
    Please enter a valid email address.
  </p>

  <button type="submit">Submit</button>
</form>

<script src="index.js"></script>
```

Mentre dai al tuo elemento il tuo tocco personale, vuoi essere sicuro che lo stato di errore sia ancora ben visibile e che non assomigli all'indicatore di focus. Tenendo tutto questo a mente, ora puoi creare moduli utente accattivanti.
