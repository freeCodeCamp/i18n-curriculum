## خدمات الويب وواجهات برمجة التطبيقات

- **API (واجهة برمجة التطبيقات)**: طريقة عامة لتواصل البرمجيات مع بعضها.
- **خدمة ويب**: نوع محدد من API يستخدم تقنيات الويب مثل HTTP لتبادل البيانات عبر الشبكة. كل خدمة ويب هي API، لكن ليس كل API هي خدمة ويب.
- **الخصائص الرئيسية لخدمة الويب**: تعمل على خادم، تتيح التواصل بين الآلات، غير مرتبطة بلغة برمجة معينة، غير مرتبطة بمنصة معينة، وتستخدم بروتوكولات ويب قياسية (HTTP/HTTPS، FTP، SMTP).
- **REST (نقل الحالة التمثيلية)**: أكثر أنواع خدمات الويب استخدامًا اليوم. تتيح الوصول إلى الموارد المحددة بمسارات URL مثل `/users` أو `/users/20`. عادةً ما تكون الاستجابات بصيغة JSON، لكنها قد تكون XML أو نصًا عاديًا.
- **SOAP (بروتوكول الوصول البسيط للكائنات)**: يستخدم XML فقط. شائع في أنظمة المؤسسات التي تحتاج إلى معايير وأمان أكثر صرامة.
- **XML-RPC**: يستخدم XML لترميز الطلبات والاستجابات. خفيف الوزن لكنه أقل مرونة من REST.
- **القسم العلوي `Accept`**: يستخدمه العميل ليخبر الخادم بصيغة الاستجابة المطلوبة، مثل `Accept: application/json`.

## بنية REST

- **عديم الحالة**: يجب أن يحتوي كل طلب على كل المعلومات التي يحتاجها الخادم لمعالجته. الخادم لا يتذكر الطلبات السابقة.
- **المورد**: قطعة بيانات يمكن الوصول إليها أو تعديلها، محددة بواسطة URL (مثل `/products/1`).
- **طرق HTTP في REST**:
  - `GET` — استرجاع مورد
  - `POST` — إنشاء مورد جديد
  - `PUT` / `PATCH` — تحديث مورد موجود
  - `DELETE` — حذف مورد

## رموز حالة HTTP

بعد معالجة الطلب، يرد الخادم برمز حالة يشير إلى النتيجة:

- `200` **(OK)**: تم تنفيذ الطلب بنجاح
- `201` **(Created)**: تم إنشاء مورد جديد
- `400` **(Bad Request)**: أرسل العميل بيانات غير صحيحة
- `404` **(Not Found)**: المورد المطلوب غير موجود
- `500` **(Internal Server Error)**: حدث خطأ في الخادم

### كيف يعمل طلب REST

1. يرسل العميل طلبًا مع طريقة، URL، رؤوس اختيارية، وجسم اختياري.
2. يتحقق الخادم من صحة الطلب ويحدد المورد.
3. ينفذ الخادم الإجراء (استرجاع، إنشاء، تحديث، أو حذف).
4. يبني الخادم استجابة مع رمز حالة وجسم اختياري (عادة JSON).
5. يرسل الخادم الاستجابة — لا يتم تخزين حالة العميل.
6. يتحقق العميل من رمز الحالة ويعالج البيانات المستلمة.

## التطبيقات الأحادية مقابل الخدمات المصغرة

- **التطبيق الأحادي**: قاعدة شفرة واحدة كبيرة حيث تتواجد واجهة المستخدم، قاعدة البيانات، وAPI معًا. يعمل جيدًا للتطبيقات الصغيرة لكنه يصبح أصعب في الصيانة والتوسع مع النمو.
- **الخدمات المصغرة**: بنية تقسم التطبيق الكبير إلى خدمات أصغر مستقلة. كل خدمة تتعامل مع قدرة عمل واحدة (مثل المنتجات، الطلبات، المدفوعات) ويمكن أن يكون لها قاعدة شفرة، قاعدة بيانات، وفريق خاص بها.
- **كيف تتواصل الخدمات المصغرة**: عبر APIs أو أنظمة قائمة على الرسائل.
- **مزايا الخدمات المصغرة**: تطوير أسرع، عزل أفضل للأخطاء، وحرية استخدام لغات أو أُطُر مختلفة لكل خدمة.
- **تحديات الخدمات المصغرة**: تصحيح أخطاء أكثر تعقيدًا، إدارة بنية تحتية أصعب، وزيادة الحمل الشبكي بين الخدمات.

## ما هو Express.js؟

- **Express**: إطار عمل ويب بسيط ومرن مبني على Node.js يسهل بناء خوادم الويب وواجهات برمجة التطبيقات.
- **تثبيت Express**: `npm install express`
- **ما يمكنك بناؤه باستخدام Express**: تطبيقات ويب، APIs بنمط REST، خلفيات تطبيقات الصفحة الواحدة، خدمات تعتمد على برمجيات وسيطة، وأي شيء على الخادم باستخدام HTTP.
- **البرمجيات الوسيطة**: شفرة تعمل بين استقبال الطلب وإرسال الاستجابة. تُستخدم لمعالجة الأخطاء، التحقق من المُدخلات، تعديل بيانات الطلب أو الاستجابة، وأكثر. يمكن ربط عدة دوال برمجيات وسيطة معًا.
- **الإضافات**: وحدات npm خارجية توسع Express، عادةً ما تُنفذ كبرمجيات وسيطة.

## إنشاء تطبيق Express أساسي

```javascript
const express = require("express");
const app = express();
const port = 3000;

app.get("/", (req, res) => {
  res.send("Hello World!");
});

app.listen(port, () => {
  console.log(`Example app listening on port ${port}`);
});
```

- `require("express")` — يحمل مكتبة Express.
- `express()` — دالة مصنع تنشئ وتُرجع نسخة جديدة من تطبيق Express.
- `app` — كائن الخادم الرئيسي الذي يتيح الوصول إلى طرق مثل `.get()`, `.post()`, و `.listen()`.
- `port` — نقطة النهاية الشبكية التي يستمع عليها التطبيق. المنفذ `3000` شائع للتطوير المحلي.
- `app.listen(port, callback)` — يبدأ الخادم ويبدأ الاستماع لطلبات HTTP الواردة.

## التوجيه في Express

- **المسار**: قاعدة تخبر التطبيق كيف يستجيب عندما يطلب العميل URL معين بطريقة HTTP محددة.
- **طرق المسار**: `app.get()`, `app.post()`, `app.put()`, `app.delete()` — كل منها يتعامل مع الطلبات التي تستخدم طريقة HTTP المطابقة.
- **مسار ثابت**: URL ثابت مثل `/home` أو `/menu/drinks`.
- **مسار ديناميكي (معلمة المسار)**: عنصر نائب في URL يلتقط قيمة، مسبوقة بـ `:`, مثل `/post/:postId`.
- **`req.params`**: كائن يحتوي القيم الديناميكية الملتقطة من URL. مثلاً، طلب إلى `/post/42` يعطي `req.params.postId === "42"`.

### معالجات المسار

- **معالج المسار** هو دالة الاستدعاء الراجعة `(req, res) => { ... }` التي تُنفذ عندما يتطابق الطلب مع مسار.
- `req` — يحتوي معلومات عن الطلب الوارد (المعلمات، سلاسل الاستعلام، الرؤوس، الجسم).
- `res` — يُستخدم لإرسال استجابة إلى العميل.

## طرق الاستجابة

| الطريقة | ما تقوم به |
| --- | --- |
| `res.send()` | يرسل استجابة — نص عادي، HTML، JSON، أو بيانات ثنائية. |
| `res.json()` | يرسل استجابة JSON؛ يحول تلقائيًا كائن JavaScript. |
| `res.status(code)` | يحدد رمز حالة HTTP؛ عادةً ما يُرتبط مع `res.send()` أو `res.json()`. |
| `res.redirect(url)` | يعيد توجيه العميل إلى URL مختلف. |
| `res.render(view, data)` | يعرض قالبًا باستخدام محرك القوالب ويرسل HTML الناتج. |

## معالجات المسار القابلة للربط مع `app.route()`

- `app.route(path)` يجمع عدة معالجات لطرق HTTP لنفس المسار لتجنب تكراره.

```javascript
app
  .route("/user")
  .get((req, res) => { res.send("Fetching user data"); })
  .post((req, res) => { res.send("Creating a user"); })
  .put((req, res) => { res.send("Updating the user"); })
  .delete((req, res) => { res.send("Deleting the user"); });
```

## التوجيه المعياري مع `express.Router()`

- `express.Router()` ينشئ كائن موجِّه — مثل تطبيق Express صغير — يمكنك تعريف المسارات عليه ثم تركيبه في التطبيق الرئيسي.
- ركب الموجِّه باستخدام `app.use(path, router)`. المسارات داخل الموجِّه تكون نسبية لمسار التركيب.
- **الفوائد**: فصل الاهتمامات، الحفاظ على نظافة الملف الرئيسي، وجعل المسارات قابلة لإعادة الاستخدام وأسهل في التوسع.

```javascript
// userRoutes.js
const express = require("express");
const router = express.Router();
router.get("/", (req, res) => { res.send("List of users"); });
router.get("/:id", (req, res) => { res.send(`User ${req.params.id}`); });
module.exports = router;

// app.js
app.use("/users", userRoutes); // mounts the router at /users
```

## تقديم الملفات الثابتة

- **الملفات الثابتة**: أصول مثل الصور، CSS، وجافاسكريبت على جانب العميل تُقدم مباشرة بدون معالجة على الخادم.
- `express.static(directory)` — برمجيات وسيطة مدمجة تقدم الملفات من مجلد محدد.
- إذا لم يُعثر على ملف مطلوب، يرد Express تلقائيًا برمز 404.
- لا حاجة لتعريف مسارات منفصلة للملفات الثابتة الفردية.

```javascript
app.use(express.static(path.join(__dirname, "public")));
```

- يمكنك تقديم الملفات من عدة مجلدات عبر استدعاء `app.use()` عدة مرات، مع مسار تركيب اختياري:

```javascript
app.use(express.static(path.join(__dirname, "public")));
app.use("/images", express.static(path.join(__dirname, "images")));
```
