# --interactive--

이전 모듈에서는 다음과 같은 문자열 리터럴을 다뤄왔습니다:

```js
const greeting = "Hello, World!";
```

하지만 JavaScript에는 문자열 객체도 있습니다. 문자열 객체와 문자열 원시값은 모두 텍스트를 처리하는 데 사용되지만 내부적으로 작동 방식이 다릅니다. 문자열 객체는 문자열 생성자 함수를 사용해 생성되며, 원시값을 객체로 감쌉니다. 문자열 객체를 생성하는 방법은 다음과 같습니다:

:::interactive_editor

```js
const greetingObject = new String("Hello, World!");

console.log(typeof greetingObject); // "object"
```

:::

`typeof` 연산자를 사용하면 결과가 문자열 타입이 아니라 객체 타입임을 알 수 있습니다. 문자열 원시값에서 `.length` 속성과 같은 속성을 어떻게 사용할 수 있는지 궁금했을 수도 있습니다.

문자열 원시값에서 `length` 속성을 사용할 때, JavaScript는 임시로 문자열 원시값을 문자열 객체로 감싸서 작업을 수행합니다. 그래서 `length` 속성과 `repeat()`, `concat()`, `slice()` 같은 다양한 메서드를 사용할 수 있습니다. 이러한 메서드와 속성은 인스턴스 메서드, 인스턴스 속성, 정적 메서드라고 부릅니다. 이 작동 방식은 앞으로의 모듈에서 자세히 배우게 됩니다.

문자열 객체와 문자열 원시값의 주요 차이점 중 하나는 메모리와 성능과 관련이 있습니다. 문자열 원시값이 보통 문자열 객체보다 메모리를 덜 사용하고 더 빠릅니다. 코드에서는 주로 문자열 원시값을 사용하지만, 문자열 객체가 어떻게 작동하는지 이해하는 것도 유용합니다.
