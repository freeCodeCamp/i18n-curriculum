# --interactive--

JavaScriptを使うとき、プログラム全体で使うデータを保存するために変数を宣言することがよくあります。

現代のJavaScriptでは、`let` と `const` が変数を宣言するための推奨される方法ですが、値の代入や再代入の扱い方に違いがあります。

このレッスンでは、`let` と `const` が変数の宣言、代入、再代入においてどのように異なるかを探っていきます。

`let` キーワードは、後で更新や再代入ができる変数を宣言することを可能にします。`let` は柔軟なコンテナのようなもので、一度値を保存したら、プログラムの中で必要に応じてその値を変更できます。

`let` を使って変数を宣言し、代入する例を見てみましょう。

```js
let score = 10;
```

この場合、変数 `score` が宣言され、値 `10` が代入されています。後で値を更新したい場合は、簡単にできます。

:::interactive_editor

```js
let score = 10;
console.log(score); // 10
score = 20;
console.log(score); // 20
```

:::

今、`score` は値 `20` を持っています。これは、プログラムの実行中に変数の値が変わることがわかっている場合に、`let` が特に役立つ理由です。

一方、`const` は定数を宣言するために使われます。`const` で宣言された変数に一度値を代入すると、再代入はできません。

これは、プログラムの実行中に誤って変更したくない値に対して、`const` が理想的である理由です。

`const` を使って変数を宣言し、代入する例を見てみましょう。

:::interactive_editor

```js
const maxScore = 100;
console.log(maxScore); // 100
```

:::

一度 `maxScore` に値 `100` が代入されると、変更できません。

```js
maxScore = 200; // This will result in an error
```

`const` 変数に値を再代入しようとすると、JavaScriptのコンソールでエラーがスローされます。`const` 変数は一度代入されると不変だからです。

`let` 変数は、すぐに値を代入せずに宣言することもでき、後で値を代入できます。

:::interactive_editor

```js
let age;
console.log(age); // undefined
age = 25;
console.log(age); // 25
```

:::

`const` で宣言された変数は、宣言時に値を代入する必要があります。値を代入せずに `const` 変数を宣言しようとすると、エラーになります。

```js
const age; // Error: Missing initializer in const declaration
```

後で再代入する必要がある変数を宣言するときは、`let` を使うべきです。例えば、変化するスコアを追跡したり、プログラム内で値を時間とともに更新したりする場合です。

変更してはいけない設定値や誤って変更したくない設定を宣言するときは、`const` を使ってください。

`var` キーワードも使えますが、今ではあまり推奨されていません。`var` は `let` に似ていますが、スコープが広いため、プログラムで問題を引き起こしやすいです。
