# --description--

Обробка подій — це важлива частина кожного інтерактивного вебсайту.

React надає потужний і послідовний спосіб обробки подій через свою систему Synthetic Event, яка є обгорткою над нативними подіями, такими як `click`, `keydown` і `submit`, про які ви дізналися в попередніх уроках.

Ця кросбраузерна обгортка гарантує, що події працюють однаково у всіх браузерах, тому немає жодних розбіжностей.

Давайте подивимося, як працюють події в React, щоб ви могли почати їх використовувати у своїх проєктах.

У React обробники подій працюють подібно до нативних браузерних подій, але з кількома відмінностями.

Замість використання імен атрибутів подій у нижньому регістрі, як `onclick` і `onsubmit`, React використовує верблюжий регістр, наприклад `onClick` і `onSubmit`.

Крім того, замість рядків для вказівки типу події React очікує функцію як обробник події.

Функція обробника передається елементу як проп, а тип події, наприклад `onClick` або `onSubmit`, використовується як атрибут у JSX.

Ось нагадування, як працювати з подією click у звичайному HTML:

```html
<button onclick="alert('Button clicked!')">Click Me</button>
```

А ось як це робиться в React:

```jsx
function handleClick() {
  console.log("Button clicked!");
}

<button onClick={handleClick}>Click Me</button>;
```

У цьому прикладі `handleClick` виводить повідомлення в консоль, коли користувач натискає на кнопку.

Зверніть увагу, що вам не потрібно ставити круглі дужки після `handleClick` в атрибуті `onClick`, оскільки ви передаєте референс на функцію, а не викликаєте її.

У React функції обробників подій зазвичай починаються з префікса `handle`, щоб показати, що вони відповідають за обробку подій, наприклад `handleClick` або `handleSubmit`.

Коли користувацька дія викликає подію, React передає обробнику об’єкт Synthetic Event. Цей об’єкт поводиться дуже схоже на нативний об’єкт події у ванільному JavaScript, надаючи властивості, такі як `type`, `target` і `currentTarget`.

Ви можете передати `event` як параметр у функцію обробника і вивести його в консоль, щоб подивитися на його структуру:

```js
function handleClick(event) {
  console.log(event);
}
```

Щоб запобігти стандартним діям, наприклад оновленню сторінки браузера під час події `onSubmit`, ви можете викликати метод `preventDefault()`:

```jsx
function handleSubmit(event) {
  event.preventDefault();
  console.log("Form submitted!");
}

<form onSubmit={handleSubmit}>
  <input type="text" />
  <button>Submit</button>
</form>;
```

Ви також можете зупинити поширення події до батьківських елементів, викликавши `event.stopPropagation()`.

Іноді, обробляючи спеціальні випадки, як функції видалення та редагування, ви можете захотіти передати додаткові дані в обробник події. Це можна зробити, обгорнувши обробник у рядкову стрілкову функцію:

```jsx
function handleDelete(id) {
  console.log("Deleting item:", id);
}

<button onClick={() => handleDelete(1)}>Delete Item</button>;
```

Використовувати рядкові обробники подій у React цілком нормально, оскільки React ефективно керує повторними відтвореннями і уникає проблем з продуктивністю.

У ванільному JavaScript рядкові обробники подій можуть призводити до проблем з продуктивністю, створюючи нові функції при кожному відтворенні, оскільки немає віртуального DOM для оптимізації цього процесу.
