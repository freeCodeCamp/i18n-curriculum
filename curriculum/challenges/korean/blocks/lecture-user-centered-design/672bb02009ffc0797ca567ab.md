진행 상황 표시는 사용자가 프로세스에서 얼마나 진행했는지 보여주는 방법입니다. 양식, 등록, 설정 과정에서 사용할 수 있습니다. 목표는 사용자가 현재 위치와 남은 작업량을 이해하도록 돕는 것입니다.

예를 들어, 양식을 작성할 때 남은 작업을 보여주기 위해 진행 상황 표시 바를 사용할 수 있습니다. 사용자가 긴 양식을 작성해야 하는데 몇 단계가 남았는지 모르는 상황을 만들고 싶지 않습니다. 투명성이 중요하며, 사용자가 충분한 시간을 가지고 앉아서 양식을 완료할지 아니면 나중에 다시 와야 할지 알 수 있어야 합니다.

```js
const form = document.getElementById('multiStepForm');
const steps = form.querySelectorAll('.form-step');
const progressBar = form.querySelector('.progress-bar');
const progressText = form.querySelector('.progress-text');
const totalSteps = steps.length;

let currentStep = 0;

function updateProgress() {
  const percent = ((currentStep + 1) / totalSteps) * 100;
  progressBar.style.width = percent + '%';
  progressText.textContent = `Step ${currentStep + 1} of ${totalSteps}`;
}

function showStep(index) {
  steps.forEach((step, i) => {
    step.classList.toggle('active', i === index);
  });
  updateProgress();
}

form.querySelectorAll('.next-btn').forEach(btn => {
  btn.addEventListener('click', () => {
    if (currentStep < totalSteps - 1) {
      currentStep++;
      showStep(currentStep);
    }
  });
});

form.querySelectorAll('.prev-btn').forEach(btn => {
  btn.addEventListener('click', () => {
    if (currentStep > 0) {
      currentStep--;
      showStep(currentStep);
    }
  });
});

showStep(currentStep);

form.addEventListener('submit', e => {
  e.preventDefault();
  alert('Form submitted!');
});
```

진행 상황 표시 섹션을 설계할 때 몇 가지 모범 사례를 기억하세요. 첫 번째 고려 사항은 단순하게 유지하는 것입니다. 너무 많은 정보로 사용자를 압도해 좌절하고 사이트를 떠나지 않도록 해야 합니다.

두 번째 고려 사항은 이전 단계로 돌아갈 수 있게 하는 것입니다. 사용자가 이전 답변을 확인하거나 변경하고 싶어할 수 있기 때문에 중요합니다.

또 다른 고려 사항은 진행 상황 표시 섹션을 쉽게 찾을 수 있게 하는 것입니다. 사용자가 찾지 못하면 현재 위치를 알 수 없습니다.

마지막 고려 사항은 명확한 섹션 제목, 백분율 또는 단계를 표시하는 것입니다. 단순히 진행 바만 있고 맥락이 없으면 사용자가 의미를 알 수 없습니다.
