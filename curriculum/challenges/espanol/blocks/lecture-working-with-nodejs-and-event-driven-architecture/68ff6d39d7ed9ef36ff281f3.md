# --description--

Ahora estás listo para dar tus primeros pasos en el mundo de Node.js instalándolo en tu computadora. Pero primero, antes de instalar Node.js, te recomendamos instalar una herramienta llamada NVM (Node Version Manager). Como su nombre indica, NVM es una herramienta para gestionar múltiples versiones de Node.js en la misma computadora.

Configurar esta herramienta desde el principio te ayudará en el futuro porque podrías necesitar cambiar entre diferentes versiones de Node.js para distintos proyectos. Con NVM, puedes hacer esto muy fácilmente.

Veamos cómo puedes instalar NVM en macOS/Linux y Windows, y cómo puedes usarlo para instalar Node.js.

## Cómo instalar NVM en macOS/Linux

En macOS y Linux, puedes instalar NVM directamente a través de un script de shell.

Pero para macOS, ten en cuenta que la documentación oficial de NVM menciona que “necesitas instalar manualmente las herramientas de línea de comandos de Xcode antes de ejecutar el script de instalación, de lo contrario, fallará.”

En macOS, puedes instalar las herramientas de línea de comandos de Xcode ejecutando este comando en tu terminal:

```bash
xcode-select --install
```

Una vez que la instalación haya terminado, puedes verificar el número de versión y confirmar la instalación con este comando:

```bash
xcode-select --version
```

Luego, es momento de instalar NVM. Puedes descargar y ejecutar el script de instalación de NVM ejecutando uno de estos comandos:

```bash
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.3/install.sh | bash
```

```bash
wget -qO- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.3/install.sh | bash
```

Para obtener la versión más reciente de NVM en el momento en que leas este artículo, consulta la documentación oficial de NVM: `https://github.com/nvm-sh/nvm?tab=readme-ov-file#install--update-script`. Allí encontrarás estos comandos con el número de versión más reciente.

Es posible que necesites reiniciar tu terminal cuando el proceso haya terminado para cargar el nuevo comando y la nueva configuración.

Luego, ejecuta este comando para confirmar que NVM se instaló correctamente y para verificar tu versión actual:

```bash
nvm -v
```

Deberías ver tu versión de NVM en la salida.

## Cómo instalar Node.js con NVM en macOS/Linux

Ahora que tienes NVM instalado en macOS/Linux, puedes instalar Node.js.

Para instalar la última versión de Soporte a Largo Plazo (LTS) de Node.js, ejecuta este comando:

```bash
nvm install --lts
```

Observa la bandera `--lts`. Esta bandera indica que quieres instalar la versión actual de Long-Term Support (LTS) de Node.js. Esta versión generalmente se recomienda porque prioriza la estabilidad, confiabilidad y seguridad sobre características nuevas o experimentales. Está completamente probada y lista para producción. También se garantiza que tendrá períodos de soporte más largos con un enfoque en correcciones de errores y parches de seguridad.

Alternativamente, puedes instalar una versión específica de Node.js especificando el número de versión en el comando:

```bash
nvm install <version>
```

Por ejemplo, aquí tienes cómo puedes instalar Node.js v20:

```bash
nvm install 20
```

Una vez que termines de instalar una versión de Node.js, puedes cambiar a esa versión con este comando:

```bash
nvm use <version>
```

Por ejemplo, aquí te mostramos cómo cambiar a la versión 20:

```bash
nvm use 20
```

Finalmente, puedes verificar la versión actual de Node.js con este comando:

```bash
node -v
```

## Cómo instalar NVM en Windows

Ahora veamos cómo puedes instalar y usar NVM en Windows.

Las versiones oficiales de NVM solo soportan Windows en casos específicos. Si tienes Windows Subsystem for Linux (WSL) configurado en tu dispositivo, puedes seguir la guía de macOS/Linux arriba para instalar y usar NVM.

Sin embargo, para Windows en sí, hay algunas alternativas. Una de ellas es `nvm-windows`, una utilidad de gestión de versiones de Node.js para Windows. Este es un proyecto completamente separado del proyecto oficial de NVM. Pero el propósito de estas herramientas es el mismo: ayudarte a trabajar con múltiples versiones de Node.js en el mismo dispositivo.

Antes de instalar `nvm-windows`, debes saber que el repositorio oficial recomienda desinstalar cualquier instalación previa de Node antes de instalar NVM para Windows. Si es la primera vez que instalas Node.js, deberías estar bien. Si ya tienes otras versiones de Node.js instaladas, se recomienda desinstalarlas primero.

Para instalar `nvm-windows`:

- Ve a la sección Releases del repositorio oficial: `https://github.com/coreybutler/nvm-windows/releases`.
- Descarga el instalador más reciente.
- Sigue las instrucciones.

Después de completar la instalación, es posible que necesites reiniciar tu terminal si el comando `nvm` no está disponible de inmediato.

Puedes verificar si la instalación fue exitosa y ver tu versión actual de `nvm-windows` ejecutando este comando:

```powershell
nvm -v
```

## Cómo instalar Node.js usando NVM para Windows

Ahora que tienes `nvm-windows` instalado en Windows, estás listo para instalar Node.js.

Para instalar la última versión de Soporte a Largo Plazo (LTS) de Node.js, ejecuta este comando:

```powershell
nvm install lts
```

La versión de Soporte a Largo Plazo (LTS) suele ser recomendada, pero puedes instalar una versión diferente especificando su número de versión en el comando:

```powershell
nvm install <version>
```

Por ejemplo:

```powershell
nvm install 20
```

Cuando la instalación esté completada, puedes cambiar a esa versión de Node.js con este comando:

```powershell
nvm use <version>
```

Por ejemplo:

```powershell
nvm use 20
```

También puedes verificar tu versión actual de Node.js con este comando:

```bash
node -v
```

## Comandos básicos

Ahora que sabes cómo instalar NVM y Node.js, veamos algunos comandos básicos que te serán útiles al trabajar con Node.js y NVM.

Además, aunque todos estos comandos funcionarán para NVM instalado en macOS/Linux, incluyendo WSL, pueden funcionar de manera diferente o no funcionar en absoluto para `nvm-windows`. Dejaremos comandos alternativos para `nvm-windows` siempre que sea posible.

### Cómo ajustar la política de ejecución de Powershell

Si estás usando Windows y sigues junto con Powershell y `nvm-windows`, puede que necesites ajustar la política de ejecución de Powershell. La política de ejecución predeterminada para Powershell es `Restricted`, que es una medida de seguridad para evitar que se ejecuten scripts.

Primero, ejecuta Powershell como administrador haciendo clic derecho en Powershell y luego seleccionando “Ejecutar como administrador“.

Luego, para ver la política de ejecución actual de la sesión de Powershell actual, ejecuta el siguiente comando:

```powershell
Get-ExecutionPolicy
```

Finalmente, para establecer la política de ejecución de Powershell en `RemoteSigned`, que permite ejecutar scripts locales y `npm`, usa el siguiente comando:

```powershell
Set-ExecutionPolicy RemoteSigned
```

Una vez que hayas ejecutado el comando anterior, puedes cerrar la ventana de administrador de Powershell y abrir una ventana de Powershell sin privilegios de administrador.

### Listar Todas las Versiones

Para listar todas las versiones instaladas de Node.js y verificar cuál se está usando actualmente, ejecuta este comando:

```bash
nvm ls
```

### Establecer la versión predeterminada de Node.js (NVM solo en macOS/Linux)

Este comando establece la versión predeterminada de Node.js que se usará para nuevas sesiones de terminal:

```bash
nvm alias default <version>
```

Por ejemplo:

```bash
nvm alias default 22.20.0
```

También puedes crear alias personalizados para asignar un nombre más específico a una versión:

```bash
nvm alias <new_name> <version>
```

Por ejemplo:

```bash
nvm alias new-api 18.20.8
```

Después de configurar el alias, puedes usar este nuevo nombre para cambiar a esa versión:

```bash
nvm use <new_name>
```

Por ejemplo:

```bash
nvm use new-api
```

### Eliminar una versión de Node.js

Para eliminar una versión de Node.js, ejecuta este comando:

```bash
nvm uninstall <version>
```

Por ejemplo:

```bash
nvm uninstall 20.19.5
```

Con NVM para macOS/Linux, puedes eliminar una versión de Node.js ejecutando el comando `uninstall` con una versión mayor como `22` o `20`. Por ejemplo:

```bash
nvm uninstall 20
```

Si tienes múltiples versiones para esa versión mayor instaladas como `20.14.0` y `20.12.2`, la versión más reciente se eliminará primero. En este caso, ejecutar `nvm uninstall 20` eliminará primero `20.14.0`.

### Verifica tu versión de Node.js

Para verificar tu versión actual de Node.js, puedes usar lo siguiente:

```bash
node -v
```

### Verifica tu versión de npm

`npm` es un gestor de paquetes para Node.js. Puedes usarlo para instalar, publicar y gestionar paquetes de software. Se instala automáticamente cuando instalas Node.js.

Para verificar la versión actual de `npm`, ejecuta este comando:

```bash
npm -v
```

### Ejecuta un archivo JavaScript

Este comando ejecuta el archivo JavaScript que especificas en el runtime activo de Node.js:

```bash
node <file.js>
```

Por ejemplo:

```bash
node app.js
```

### Crea un proyecto de Node.js

Este comando crea un nuevo proyecto de Node.js creando un archivo `package.json` que rastrea sus detalles y dependencias.

```bash
npm init
```

Cuando ejecutas el comando `npm init`, te guiará a través de una guía interactiva de CLI para configurar el archivo `package.json`. Te hará preguntas sobre el nombre del proyecto, descripción, versión, etc. Si quieres comenzar con un archivo `package.json` predeterminado, puedes usar el comando `npm init -y` o `npm init --yes`.

### Instalar un Paquete

Para instalar un paquete específico, ejecuta:

```bash
npm install <package>
```

Por ejemplo:

```bash
npm install express
```

### Instalar dependencias

Para instalar todas las dependencias listadas en el archivo `package.json` de un proyecto, ejecuta:

```bash
npm install
```
