# --interactive--

Sie haben bereits gelernt, wie semantische Auszeichnung und ARIA-Attribute Inhalte zugänglich machen. In realen Anwendungen sind Inhalte selten statisch, und Seiten werden oft dynamisch durch JavaScript aktualisiert. Wenn das passiert, ist es wichtig sicherzustellen, dass diese Änderungen auch im HTML widergespiegelt werden. So können Screenreader den aktualisierten Zustand genau an die Nutzerinnen und Nutzer weitergeben. Ohne das könnten Menschen, die auf unterstützende Technologien angewiesen sind, nie erfahren, dass sich etwas geändert hat, oder sie erhalten veraltete oder irreführende Informationen.

Schauen wir uns ein häufiges Szenario an, bei dem JavaScript den Zustand eines Elements ändert, und wie Sie Ihr HTML zugänglich halten, indem Sie ARIA-Attribute aktualisieren.

Angenommen, Sie haben einen Button, der ein Dropdown-Menü öffnet und schließt:

:::interactive_editor

```html
<button id="menuButton" aria-expanded="false" aria-controls="menuList">
  Menu
</button>

<ul id="menuList" hidden>
  <li>List item 1</li>
  <li>List item 2</li>
</ul>
<script src="index.js"></script>
```

```js
const button = document.getElementById("menuButton");
const menu = document.getElementById("menuList");

button.addEventListener("click", () => {
  const expanded = button.getAttribute("aria-expanded") === "true";
  button.setAttribute("aria-expanded", String(!expanded));
  menu.hidden = expanded;
});
```

:::

Das HTML richtet einen Button ein, der ein Dropdown-Menü steuert. Der Button verwendet `aria-expanded`, um anzuzeigen, ob das Menü geöffnet ist, und `aria-controls`, um es mit dem Menü zu verknüpfen, indem es auf die `id` des Menüs verweist.

Das JavaScript hört auf Klicks auf den Menü-Button. Wenn darauf geklickt wird, schaltet es den Wert von `aria-expanded` um und aktualisiert die `hidden`-Eigenschaft am Menü. So bleiben die Sichtbarkeit des Menüs und die Barrierefreiheitsinformationen synchron, damit sowohl sehende Nutzerinnen und Nutzer als auch Screenreader-Nutzerinnen und -Nutzer wissen, ob das Menü geöffnet oder geschlossen ist.

Sie werden in den kommenden Lektionen mehr über diese ARIA-Attribute lernen. Merken Sie sich vorerst nur: Wenn Ihr JavaScript den Zustand eines Elements ändert, sollten Sie auch die relevanten ARIA-Attribute aktualisieren, damit unterstützende Technologien genau das widerspiegeln, was tatsächlich auf dem Bildschirm zu sehen ist.
