# --description--

Bevor wir loslegen, sollten Sie wissen, was eine At-Regel in CSS ist. At-Regeln sind Anweisungen, die dem Browser bestimmte Vorgaben machen. Sie können sie verwenden, um verschiedene Aspekte des Stylesheets zu definieren, wie zum Beispiel Media Queries, Keyframes, Schriftarten und mehr. Konzepte wie Media Queries und Keyframes lernen Sie in zukünftigen Lektionen kennen. Dieses Mal lernen Sie die `@font-face` At-Regel kennen.

Mit `@font-face` können Sie eine benutzerdefinierte Schriftart definieren, indem Sie die Schriftartdatei, das Format und weitere wichtige Eigenschaften wie Gewicht und Stil angeben. Das ist die grundlegende Syntax. Sie sehen `@font-face`, gefolgt von einem Block, der von geschweiften Klammern eingeschlossen ist:

```css
@font-face {
  /* Descriptors */
}
```

Innerhalb der geschweiften Klammern müssen Sie Deskriptoren einfügen, um Ihre Schriftart anzupassen. Schauen wir uns einige der am häufigsten verwendeten an. Der Deskriptor font-family gibt den Namen an, den Sie im gesamten Stylesheet verwenden, um auf diese Schriftart zu verweisen. Zum Beispiel definieren Sie diese `@font-face rule`. Sie hat den `font-family` Deskriptor definiert, und dessen Wert ist `MyCustomFont`:

```css
@font-face {
  font-family: "MyCustomFont";
}
```

In diesem Fall müssten Sie diesen Namen in Ihrem Stylesheet verwenden, wo immer Sie diese benutzerdefinierte Schriftfamilie allen Elementen zuweisen möchten, die von einem CSS-Selektor erfasst werden. Zum Beispiel würden Sie ihn als Wert von `font-family` verwenden:

```css
body { 
  font-family: "MyCustomFont"; 
}
```

Damit die `@font-face` At-Regel gültig ist, müssen Sie außerdem die `src` angeben. Diese enthält Referenzen zu den Schriftressourcen. Es ist im Grunde eine Liste externer Referenzen oder lokal installierter Schriftartnamen, getrennt durch Kommas. Sie kann auch Hinweise zum Format und zur Technologie der Schriftressourcen enthalten.

Im folgenden Code ist die `src` definiert. Der Wert ist eine Liste von URLs, getrennt durch Kommas und auf separaten Zeilen geschrieben, um die Lesbarkeit zu verbessern:

```css
@font-face {
  font-family: "MyCustomFont"; 
  src: url("path/to/font.woff2"),
    url("path/to/font.woff"),
    url("path/to/font.otf");
}
```

Sie können die `url()` Funktion aufrufen, um eine Datei in Ihr Stylesheet einzubinden. In diesem Fall binden wir die Schriftartdateien ein. Sie müssen den Dateipfad in runden Klammern und Anführungszeichen schreiben, einschließlich der Dateiendung. Zur besseren Lesbarkeit können Sie jede Ressource in einer eigenen Zeile schreiben, aber die letzte muss mit einem Semikolon enden.

Für jede Schriftressource können Sie auch das Format angeben. Das ist optional. Es ist ein Hinweis für den Browser zum Schriftformat. Wenn das Format weggelassen wird, wird die Ressource heruntergeladen und das Format nach dem Download erkannt. Ist das Format ungültig, wird die Ressource nicht heruntergeladen.

Mögliche Schriftformate sind `collection`, `embedded-opentype`, `opentype`, `svg`, `truetype`, `woff` und `woff2`.

Hier ein Beispiel mit Schriftformaten. Beachten Sie, wie wir das spezifische Format in runden Klammern und Anführungszeichen schreiben:

```css
@font-face {
  font-family: "MyCustomFont"; 
  src: url("path/to/font.woff2") format("woff2"),
    url("path/to/font.otf") format("opentype"),
    url("path/to/font.woff") format("woff");
}
```

In diesem Beispiel sehen Sie, dass wir das WOFF2-Format, das OpenType-Format und das WOFF-Format angeben.

woff steht für „Web Open Font Format“. Der Unterschied zwischen WOFF und WOFF2 liegt im Algorithmus zur Datenkompression. OpenType ist ein Format für skalierbare Computerschriften, entwickelt von Microsoft und Adobe, das es Nutzern ermöglicht, zusätzliche Funktionen einer Schriftart zu nutzen. Es wird auf den wichtigsten Betriebssystemen weit verbreitet verwendet.

Neben der Angabe des Formats können Sie auch die Technologie der Schriftressource angeben. Das ist ebenfalls optional. Hier ein Beispiel, in dem wir die Technologie der zweiten Schriftressource angeben.

```css
@font-face {
  font-family: "MyCustomFont"; 
  src: url("path/to/font.woff2") format("woff2"),
    url("path/to/font.otf") format("opentype") tech(color-COLRv1),
    url("path/to/font.woff") format("woff");
}
```

Das sind die Grundlagen der `@font-face` Regel. Mit dieser At-Regel können Sie die Schriftartdatei angeben, um benutzerdefinierte Schriftarten für Ihre einzigartigen Designs zu definieren.
