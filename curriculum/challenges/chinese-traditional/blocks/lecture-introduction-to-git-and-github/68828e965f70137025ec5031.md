# --description--

如果你曾經處理過某些東西，例如論文，在做出重大更改後存檔多個副本並將檔案命名為 `Thesis-Draft`、`Thesis-Final`、`Thesis-Final-For-Real` 和 `Thesis-Absolute-Final-DO-NOT-CHANGE`，你就是使用了一種非常原始的版本控制方法。但如果我告訴你有更好的方法呢？

你可能聽過像 Git、SVN 或 Mercurial 這類工具。這些是版本控制系統，它們是每個開發者工作流程中的關鍵組件。版本控制系統允許你追蹤和管理你專案中的變更。就這些課程的目的而言，我們將專注於 Git。

Git 提供強大的工具，讓你管理專案的編輯歷史。你將在接下來的課程中學習更多關於如何使用 Git 及其提供的特性。現在，這是你真正需要知道的：

Git 允許你執行像是創建 commit，這是你程式碼庫特定狀態的螢幕快照。你接著可以比較 commit 以查看你所做的變更、還原包含不當變更的 commit，等等。

你也可以創建「branches」，你可以將它們視為你程式碼中的不同路徑。雖然 branch 可能是你走的道路，但 commits 是沿途的地標。branches 讓你能夠獨立處理新的特性，這樣如果你的工作出錯，你可以刪除 branch 而不會影響核心程式碼庫。

這些特性使版本控制成為與其他開發人員協作的重要組件。當你有多個人在同一個專案中工作時，分支允許他們各自處理自己的程式碼，而不會互相覆蓋彼此的變更。

你經常會將像 Git 這樣的版本控制系統與像 GitHub 或 Gitlab 這樣的版本控制提供者配對，這些提供者為你提供額外的協作工具以及將你的專案備份到雲端的能力。

即使沒有基於雲端的儲存解決方案，版本控制系統仍然可以為你的工作流程帶來革命性的改變。不再需要 `THIS-IS-THE-FINAL-COPY-FOREVER` 檔案。

關於使用這些系統有很多很酷的資訊要涵蓋，所以讓我們在接下來的幾堂課中深入探討！
