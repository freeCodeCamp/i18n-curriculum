# --description--

自分のウェブアプリケーションを作り始めるとき、ファイルの名前の付け方に注意する必要があります。

ファイル名は、理解しやすくメンテナンスしやすい方法で付けることが重要です。

HTMLファイルの良いファイル名と悪いファイル名の例を見てみましょう。これらの例にある`.html`の拡張子は、このファイルがHTMLファイルであることを示しています。

悪いファイル名の例はこちらです：

```md
index1234.html
```

名前だけではファイルの内容が何か分かりません。このファイルは商品ページ、ブログページ、または他のどんなページのものかもしれませんが、ファイル名からは分かりません。

HTMLファイルのより良いファイル名の例はこちらです：

```md
about-us.html
```

`about-us.html`のようにより説明的な名前を使うことで、ファイルの内容が明確になります。

また、このファイル名にはスペースがなく、代わりにハイフン`-`で単語を区切っていることに気づくでしょう。

時々、`index.html`というファイル名を見ることがあります。これはウェブサイトのメインページを表す特別なファイル名です。ウェブサイトを訪れると、最初に`index.html`ファイルが読み込まれます。

ファイル名を付ける際のもう一つの重要なポイントは、特殊文字の使用です。

特殊文字、数字、文字が混ざっていると、ファイルの内容が分かりにくくなります。

悪いファイル名の例はこちらです：

```md
file-1!@.html
```

この名前は不必要に複雑に見え、ファイルの内容についての情報を与えてくれません。

ですので、ファイル名には文字とハイフンだけを使うのがベストです。

他の言語を学び始めると、その言語でのファイル命名の一般的な慣例があることに気づくでしょう。

そういった場合は、公式ドキュメントや他のリソースを参照して、その言語でのファイル命名のベストプラクティスを理解することをお勧めします。

また、チームで作業する場合は、チームが定めたファイル命名の慣例に従う必要があります。

チームのスタイルガイドを必ず確認して、ファイル名がチーム全体と一貫しているかを確認してください。
