# --interactive--

Los modificadores, a menudo llamados "flags", modifican el comportamiento de una expresión regular. Recordemos nuestro ejemplo de una lección anterior:

:::interactive_editor

```js
const regex = /freeCodeCamp/;
console.log(regex.test("freeCodeCamp")); // true
console.log(regex.test("freeCodeCamp is great")); // true
console.log(regex.test("I love freeCodeCamp")); // true
console.log(regex.test("freecodecamp")); // false
console.log(regex.test("FREECODECAMP")); // false
console.log(regex.test("free")); // false
console.log(regex.test("code")); // false
console.log(regex.test("camp")); // false
```

:::

Si recuerdas, las cadenas de `freeCodeCamp` en minúsculas y mayúsculas no coincidieron con el patrón. Esto se debe a que, por defecto, las expresiones regulares distinguen entre mayúsculas y minúsculas.

Pero, ¿qué pasaría si pudiéramos decirle a la expresión regular que no distinga entre mayúsculas y minúsculas? Bueno, hay un modificador para eso. La bandera `i` hace que una regex ignore las mayúsculas. ¿Cómo podemos usarla? Las banderas van después de la barra inclinada de cierre en una expresión regular:

```js
const regex = /freeCodeCamp/i;
```

Observa el cambio en la expresión regular en la primera línea. Ahora podemos verificar cómo esto cambia las cosas:

:::interactive_editor

```js
const regex = /freeCodeCamp/i;

console.log(regex.test("freeCodeCamp")); // true
console.log(regex.test("freeCodeCamp is great")); // true
console.log(regex.test("I love freeCodeCamp")); // true
console.log(regex.test("freecodecamp")); // true
console.log(regex.test("FREECODECAMP")); // true
console.log(regex.test("free")); // false
console.log(regex.test("code")); // false
console.log(regex.test("camp")); // false
```

:::

Debido a que ahora nuestra expresión regular no distingue entre mayúsculas y minúsculas, las cadenas en minúsculas y mayúsculas han "pasado" la prueba. Esto también puede funcionar para una cadena con una mezcla aleatoria de letras mayúsculas y minúsculas:

:::interactive_editor

```js
const regex = /freeCodeCamp/i;

console.log(regex.test("dO yOu LoVe fReEcOdEcAmP?")); // true
```

:::

Hay bastantes otras banderas que puedes usar. La bandera `g`, o el modificador global, permite que tu expresión regular coincida con un patrón más de una vez.

Veamos cómo eso afecta nuestro código. Notarás que mantuvimos la bandera `i`: una expresión regular puede usar múltiples banderas (tantas como sean necesarias) para lograr el comportamiento deseado:

```js
const regex = /freeCodeCamp/gi;
```

Espera un segundo... ¿qué es esto? Parecería que el modificador global está haciendo que algunas de nuestras cadenas que deberían estar pasando fallan en su lugar:

:::interactive_editor

```js
const regex = /freeCodeCamp/gi;

console.log(regex.test("freeCodeCamp")); // true
console.log(regex.test("freeCodeCamp is great")); // false
console.log(regex.test("I love freeCodeCamp")); // true
console.log(regex.test("freecodecamp")); // false
console.log(regex.test("FREECODECAMP")); // true
console.log(regex.test("free")); // false
console.log(regex.test("code")); // false
console.log(regex.test("camp")); // false
```

:::

¿Por qué? Bueno, el modificador global hace que tu expresión regular sea stateful. Esto significa que realiza un seguimiento de dónde ha coincidido previamente un patrón. Por lo tanto, cuando coincide la primera cadena de `freeCodeCamp`, recuerda que encontró una coincidencia que comienza en el índice `0`.

Luego la probamos con `freeCodeCamp is great`, pero no comienza en el índice `0`. La expresión regular "sabe" que ya encontró una coincidencia en el índice `0`, así que aunque esta sea una cadena diferente, comienza desde el índice final de la coincidencia.

`freeCodeCamp` tiene `12` caracteres, así que una coincidencia en `0` termina en el índice `11`. La búsqueda continuará en el índice `12`. Y como `is great` no coincide con `freeCodeCamp`, devuelve `false`.

Luego, porque no encuentra una coincidencia, "pierde" su estado y comienza la coincidencia siguiente desde `0`.

Si intercambiamos nuestros registros para que una cadena con la coincidencia en `0` sea seguida inmediatamente por una cadena que tiene una coincidencia más allá del índice `11`:

:::interactive_editor

```js
const regex = /freeCodeCamp/gi;

console.log(regex.test("freeCodeCamp")); // true
console.log(regex.test("I loooooooove freeCodeCamp")); // true
```

:::

Cuando una expresión regular es global, obtiene una nueva propiedad llamada `lastIndex`. Tomando nuestro código anterior, veamos cómo funciona esta propiedad:

:::interactive_editor

```js
const regex = /freeCodeCamp/gi;

console.log(regex.lastIndex); // 0
console.log(regex.test("freeCodeCamp")); // true
console.log(regex.lastIndex); // 12
console.log(regex.test("freeCodeCamp is great")); // false
console.log(regex.lastIndex); // 0
console.log(regex.test("I love freeCodeCamp")); // true
console.log(regex.lastIndex); // 19
console.log(regex.test("freecodecamp")); // false
console.log(regex.lastIndex); // 0
console.log(regex.test("FREECODECAMP")); // true
console.log(regex.lastIndex); // 12
console.log(regex.test("free")); // false
console.log(regex.lastIndex); // 0
console.log(regex.test("code")); // false
console.log(regex.lastIndex); // 0
console.log(regex.test("camp")); // false
```

:::

Al mirar este ejemplo, puedes ver cómo cambia el estado de la expresión regular con cada llamada de prueba, usando el `lastIndex` para rastrear sus coincidencias previas.

El flag global es excelente cuando necesitas obtener múltiples coincidencias de una sola cadena. Pero si estás probando múltiples cadenas con la misma expresión regular, es mejor dejar la bandera `g` desactivada.

Antes de aprender sobre la siguiente bandera, necesitas aprender sobre los anclajes. El anclaje caret (`^`), al principio de la expresión regular, indica "coincidir con el inicio de la cadena":

```js
const start = /^freecodecamp/i;
```

El ancla del signo de dólar (`$`), al final de la expresión regular, dice "coincide el fin de la cadena":

```js
const end = /freecodecamp$/i;
```

Tómate un momento para comparar los resultados en la derecha:

:::interactive_editor

```js
const start = /^freecodecamp/i;
const end = /freecodecamp$/i;
console.log(start.test("freecodecamp")); // true
console.log(end.test("freecodecamp")); // true
console.log(start.test("freecodecamp is great")); // true
console.log(end.test("freecodecamp is great")); // false
console.log(start.test("i love freecodecamp")); // false
console.log(end.test("i love freecodecamp")); // true
console.log(start.test("have met freecodecamp's founder")); // false
console.log(end.test("have met freecodecamp's founder")); // false
```

:::

¿Ves cómo el ancla de inicio solo coincide al comienzo de la cadena y el ancla final solo coincide al final de la cadena? ¿Pero qué hay de la coincidencia a través de múltiples líneas? Echemos un vistazo a eso:

:::interactive_editor

```js
const start = /^freecodecamp/i;
const end = /freecodecamp$/i;
const string = `I really love
freecodecamp
it's my favorite`;
console.log(start.test(string)); // false
console.log(end.test(string)); // false
```

:::

Aunque `freecodecamp` está allí por su propia línea, falla ambas pruebas. Esto se debe a que, por defecto, los anclajes buscan el inicio y el final de toda la cadena.

Pero puedes hacer que una regex maneje múltiples líneas con la bandera `m`, o el modificador de líneas múltiples. Agreguemos eso a nuestras expresiones regulares para ver qué obtenemos:

:::interactive_editor

```js
const start = /^freecodecamp/im;
const end = /freecodecamp$/im;
const string = `I really love
freecodecamp
it's my favorite`;
console.log(start.test(string)); // true
console.log(end.test(string)); // true
```

:::

¡Ahora coinciden ambas! Dado que `freecodecamp` está completamente en su propia línea, el ancla de inicio coincide con el inicio de esa línea y el ancla final coincide con el final de esa línea.

Finalmente, tienes la bandera `d`, o el modificador de índices. Recuerda que la bandera `i` es para insensibilidad a mayúsculas, por lo que el modificador de índices necesita una bandei diferente.

La bandera `d` expande la información que obtienes en un objeto de coincidencia. Agreguémoslo a nuestra expresión regular:

```js
const regex = /freecodecamp/di;
const string = "we love freecodecamp isn't freecodecamp great?";
console.log(string.match(regex));
```

Y el resultado es:

```js
// [
//   'freecodecamp',
//   index: 8,
//   input: "we love freecodecamp isn't freecodecamp great?",
//   groups: undefined,
//   indices: [
//     0: [8, 20],
//     groups: undefined
//   ]
// ]
```

¡Nuestro objeto de coincidencia obtiene una nueva propiedad `indices`! Esta propiedad es un array de dos números, el primero es el índice en la cadena original donde comienza la coincidencia, y el segundo es el índice después de que terminó la coincidencia. Este array también tiene una propiedad `groups` extra, que también es para grupos de captura por nombre.

Hay algunas otras banderas que deberías saber que están disponibles para ti, pero son menos comunes en el código típico.

La primera es el modificador unicode, o la bandei `u`. Esto expande la funcionalidad de una expresión regular para permitir que coincida con caracteres especiales unicode.

Aprenderás más sobre las clases de caracteres en una lección futura, pero la bandera `u` te da acceso a clases especiales como `Extended_Pictographic` para coincidir con la mayoría de los emojis:

:::interactive_editor

```js
const regex = /🍎/u;

const str = "I have an apple 🍎";
console.log(regex.test(str)); // true
```

:::

También hay un indicador `v`, que amplía aún más la funcionalidad de la coincidencia unicode.

La segunda es el modificador sticky, o la bandei `y`. El modificador sticky se comporta de manera muy similar al modificador global, pero con algunas excepciones.

La mayor es que una expresión regular global comenzará desde `lastIndex` y buscará a través de toda la cadena restante otra coincidencia, pero una expresión regular sticky retornará `null` y restablecerá el `lastIndex` a `0` si no hay inmediatamente una coincidencia en el `lastIndex` anterior.

Y la última es el modificador de línea única, o la bandera `s`. Recuerda que el modificador de múltiples líneas permite que los anclajes de inicio y fin coincidan con el inicio y fin de una línea, en lugar de toda la cadena.

El modificador de línea única permite que un carácter comodín, representado por un punto (`.`) en regex, coincida con saltos de línea, tratando efectivamente la cadena como una sola línea de texto.

Hay bastantes de estos modificadores, pero las banderas `i` y `g` son las que usarás más frecuentemente, y son las más importantes de recordar.
