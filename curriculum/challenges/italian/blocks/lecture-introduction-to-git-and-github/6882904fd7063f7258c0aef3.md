# --description--

Prima di poter inviare il tuo repository su GitHub, devi essere autenticato. Sia le chiavi SSH sia quelle GPG usano una coppia pubblica-privata. Questo significa che tieni una chiave privata sulla tua macchina locale, che usi per autenticarti. Condividi una chiave pubblica con le persone o i servizi che devono convalidare la tua autenticazione. In altre parole, la chiave privata serve per eseguire un’azione, mentre la chiave pubblica serve per verificarla.

Le chiavi GPG, o Gnu Privacy Guard, sono tipicamente usate per firmare file o commit. Qualcuno può poi usare la tua chiave pubblica GPG per verificare che la firma del file provenga dalla tua chiave e che il contenuto del file non sia stato modificato o manomesso. Le chiavi SSH, o Secure SHell, sono solitamente usate per autenticare una connessione remota a un server — tramite l’utilità `ssh`. Tuttavia, come imparerai in questa lezione, puoi anche usare una chiave SSH per firmare i commit.

Per generare una chiave GPG, devi eseguire:

```sh
gpg --full-generate-key
```

Per una chiave SSH, esegui:

```sh
ssh-keygen -t ed25519 -C "your_email@example.com"
```

Assicurati di fornire la tua email reale.

Entrambi questi comandi dovrebbero richiedere una passphrase per proteggere la chiave. Sono opzionali, ma fortemente consigliate.

Per le chiavi SSH, otterrai due file — uno per la tua chiave privata e uno per quella pubblica. I nomi dei file della chiave pubblica finiranno con `.pub`, così non condividerai accidentalmente la chiave privata. Questi file saranno in `~/.ssh`.

Le chiavi GPG, invece, saranno in una directory nascosta e criptata. Per elencare le tue chiavi pubbliche, esegui:

```sh
gpg --list-secret-keys --keyid-format=long
```

Poi, per ottenere la chiave pubblica, usa:

```sh
gpg --armor --export "<key id>"
```

Ma cosa puoi fare con queste nuove chiavi? Abbiamo parlato di firmare i tuoi commit, e questa è una cosa che puoi fare con entrambe le chiavi.

Per firmare i tuoi commit con la chiave GPG, devi caricare la tua chiave pubblica, non quella privata, sul tuo account GitHub. Poi, prendi l’ID breve che hai ottenuto elencando le chiavi prima e esegui questo comando per impostarlo come chiave di firma git:

```sh
git config --global user.signingkey <your_gpg_key_id>
```

Poi, puoi passare il flag `-S` al comando `git commit` per firmare un commit specifico — dovrai fornire la tua passphrase. In alternativa, se vuoi firmare automaticamente ogni commit, puoi impostare la configurazione autosign su `true`:

```sh
git config --global commit.gpgsign true
```

Per firmare con una chiave SSH, che è una funzionalità relativamente nuova su GitHub, devi prima caricare la chiave sul tuo account GitHub. Poi devi impostare la modalità di firma per git su SSH:

```sh
git config --global gpg.format ssh
```

Poi, per impostare la chiave di firma, passerai il percorso del file invece di un ID:

```sh
git config --global user.signingkey <path_to_your_ssh_keys>
```

Infine, puoi abilitare la firma automatica come faresti per una chiave GPG.

Con uno di questi metodi configurati, ora sei pronto per avere commit verificati che riceveranno un badge speciale su GitHub.
