# --description--

**보안은 무단 접근, 도난, 악의적 손상으로부터 데이터를 보호합니다.** 온라인에서 정보 주위에 강력한 디지털 벽을 만듭니다.

데이터 보안을 현관문에 단단한 자물쇠가 달린 것으로 생각하세요. 자물쇠는 침입자를 집 밖에 막아줍니다. 디지털 세계에서 보안은 데이터 기밀성, 데이터 무결성, 시스템 가용성을 보호하는 데 집중합니다.

사이버보안 도구에는 네트워크 방화벽, 강력한 비밀번호, 데이터 암호화, 안티바이러스 소프트웨어가 포함됩니다. 외부 해커가 파일을 훔치려 하면 보안 시스템이 시도를 차단합니다.

보안은 데이터가 의미하는 바를 분석하지 않습니다. 오직 악의적인 행위자로부터 데이터를 안전하게 지키는 데만 집중합니다. 보안이 없으면 온라인에 저장한 모든 데이터가 즉시 도난당할 위험에 노출됩니다.

**개인정보 보호는 조직이 개인 데이터를 수집, 저장, 사용, 공유하는 방식을 통제합니다.** 누가 개인 정보를 볼 수 있는지 결정할 권한을 줍니다.

데이터 개인정보 보호를 창문 커튼으로 생각하세요. 창문에 단단한 자물쇠가 있어도 커튼을 활짝 열어두면 사람들이 안을 볼 수 있습니다.

개인정보 보호는 기업이 개인 권리를 존중하도록 보장합니다. 기업이 사용자의 검색 기록을 판매하거나 GPS 위치를 추적하거나 이메일 주소를 제3자 광고주와 공유할 수 있는지 결정합니다.

유럽연합(EU)의 GDPR 같은 글로벌 법률이 데이터 개인정보 보호를 엄격히 규제합니다. 이 법률들은 앞으로 강의에서 배우게 됩니다.

이제 보안과 개인정보 보호의 관계를 살펴보겠습니다. 두 개념은 다르지만 서로 밀접하게 의존합니다.

개인정보 보호 없이 보안만 갖추는 것은 쉽습니다. 예를 들어, 스마트폰 제조업체가 일일 위치 데이터를 매우 안전한 클라우드 데이터베이스에 저장할 수 있습니다. 고급 해커도 이 서버를 뚫을 수 없으니 데이터는 완전히 안전합니다.

하지만 제조업체가 명시적 허락 없이 그 데이터를 광고 대행사에 판매하면 개인정보 보호가 침해됩니다. 보안은 완벽하지만 개인정보 보호는 전혀 없는 상태입니다.

반대로, 강력한 보안 없이는 진정한 개인정보 보호를 달성할 수 없습니다. 의료 클리닉이 건강 기록을 절대 공유하지 않겠다고 약속하면 개인정보 보호를 약속하는 것입니다. 그러나 클리닉이 로컬 서버를 안전하게 지키지 못하면 악성 해커가 쉽게 의료 기록을 훔칠 수 있습니다.

이처럼 보안이 완전히 없으면 즉시 개인정보 보호가 무너집니다. 따라서 강력한 보안은 효과적인 개인정보 보호의 필수 기반입니다.

보안은 기술적 방어 메커니즘을 제공하고, 개인정보 보호는 데이터 사용에 관한 법적 권리와 정책을 관리합니다. 이 차이를 이해하면 디지털 공간을 안전하게 탐색할 수 있습니다.

보안 팀은 적극적인 사이버 공격에 대한 기술적 방어를 구축하고, 개인정보 보호 팀은 개인정보 보호법 준수와 사용자 선택 존중을 보장합니다.

두 요소 모두 오늘날 온라인에서 개인을 보호하는 데 필수적입니다. 보안은 현관문을 잠그고, 개인정보 보호는 창문 블라인드를 내립니다. 함께 개인 정보를 위한 안전하고 신뢰할 수 있는 환경을 만듭니다.
