# --description--

A volte ci saranno situazioni in cui conosci più informazioni su un tipo rispetto a TypeScript. Qui entrano in gioco le asserzioni di tipo.

Ecco un esempio che usa il metodo `querySelector()` per consultare un elemento con l'id `id` di `submit`:

```ts
const submitBtn = document.querySelector("#submit");
```

Al momento, TypeScript sa solo che questo `submitBtn` è una qualche forma di `Element` o forse `null`. TypeScript non controllerà l'HTML corrispondente per capirlo.

Questo è un buon momento per usare un'asserzione di tipo per dire a TypeScript che tipo di elemento è. Ecco un esempio:

```ts
const submitBtn = document.querySelector("#submit") as HTMLButtonElement;
```

Ora, TypeScript tratterà questo come un elemento `button` grazie a quell'asserzione di tipo.

Ci sono dei limiti nell'usare la parola chiave `as`. Per esempio, non puoi fare cose come questa senza che TypeScript lanci un errore:

```ts
// Conversion of type 'string' to type 'number' may be a mistake because neither type sufficiently overlaps with the other. If this was intentional, convert the expression to 'unknown' first.
const age = "awesome" as number;
```

In questo esempio, non ha molto senso cercare di asserire che la stringa `"awesome"` debba essere vista come un tipo number. TypeScript permette asserzioni di tipo solo tra tipi compatibili o sovrapposti.

Un modo per rimuovere quell'errore sarebbe usare una doppia asserzione come questa:

```ts
const age = "awesome" as unknown as number;
```

Anche se questo funziona in TypeScript, non è comunque raccomandato perché logicamente non ha molto senso.

Un altro modo per scrivere asserzioni di tipo è usare la sintassi con parentesi angolari così:

```ts
const submitBtn = <HTMLButtonElement>document.querySelector("#submit");
```

Devi stare attento con questa sintassi quando usi file `tsx` perché TypeScript potrebbe pensare che stai cercando di renderizzare un componente e lanciare un errore come questo:

```ts
// file: index.tsx
// This JSX tag requires 'React' to be in scope, but it could not be found.
// JSX element 'HTMLButtonElement' has no corresponding closing tag.
const submitBtn =  <HTMLButtonElement>document.querySelector("#submit");
```

Nella maggior parte delle situazioni, vedrai usare la prima sintassi con la parola chiave `as` piuttosto che la seconda opzione.
