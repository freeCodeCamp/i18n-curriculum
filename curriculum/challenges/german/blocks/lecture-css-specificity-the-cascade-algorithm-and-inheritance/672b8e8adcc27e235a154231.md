# --interactive--

Typselektoren, auch bekannt als Elementselektoren, richten sich an Elemente basierend auf ihrem Tag-Namen.

Diese Selektoren sind grundlegend in CSS und ermöglichen es Ihnen, Stile auf alle Instanzen eines bestimmten HTML-Elements anzuwenden.

Typselektoren sind einfach zu verwenden und werden einfach als der Tag-Name des Elements geschrieben, das Sie stylen möchten.

Hier ist ein Beispiel für einen Typselektor, der alle Absatz-Elemente auf der Seite anspricht:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<p>Paragraph one</p>
<p>Paragraph two</p>
<p>Paragraph three</p>
```

```css
p {
  color: blue;
}
```

:::

In diesem Beispiel wird bei allen `p`-Elementen die Textfarbe auf `blue` gesetzt.

Typselektoren haben eine relativ niedrige Spezifität im Vergleich zu anderen Selektoren. Der Spezifitätswert für einen Typselektor ist `(0, 0, 0, 1)`.

Das bedeutet, dass Typselektoren von Klassenselektoren, ID-Selektoren und inline-Stilen überschrieben werden, aber dennoch Stile anwenden können, sofern keine Regeln mit höherer Spezifität vorhanden sind.

Sehen wir uns ein Beispiel an, bei dem Klassenselektoren die Stile des Typselektors überschreiben.

Hier ist ein Beispiel mit zwei Absatz-Elementen:

:::interactive_editor

```html
<p class="para">I am a paragraph</p>
<p class="para">Here is another paragraph</p>
```

:::

Beide Absatz-Elemente haben eine Klasse namens `para`.

Im CSS-File spricht der Typselektor Absätze an, und der Klassenselektor spricht Elemente mit der Klasse `para` an.

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<p class="para">I am a paragraph</p>
<p class="para">Here is another paragraph</p>
```

```css
p {
  color: blue;
}

.para {
  color: red;
}
```

:::

Alle Absätze auf der Seite mit der Klasse `para` haben die Textfarbe `red` statt `blue`, weil Klassenselektoren eine höhere Spezifität als Typselektoren haben.
