# --interactive--

Quando costruisci i tuoi programmi, spesso vorrai che qualcosa accada dopo un ritardo o si ripeta a intervalli regolari. È qui che entrano in gioco i metodi `setTimeout()` e `setInterval()`. Entrambi i metodi accettano due parametri: una funzione e un ritardo.

Vediamo ciascuno dei metodi in dettaglio così puoi iniziare a usarli nei tuoi progetti web.

`setTimeout()` ti permette di ritardare un'azione per un tempo specificato. Ecco la sintassi base del metodo `setTimeout()`:

```js
setTimeout(functionToRun, delay);
```

`functionToRun` è il codice che vuoi eseguire dopo un ritardo specificato, e `delay` è il tempo (in millisecondi) da aspettare prima di eseguire la funzione. Ecco un esempio:

:::interactive_editor

```js
setTimeout(function () {
 console.log("This runs after 3 seconds");
}, 3000);
```

:::

Puoi anche usare una funzione freccia come callback, così:

:::interactive_editor

```js
setTimeout(() => {
 console.log("This runs after 3 seconds"); 
}, 3000);
```

:::

`setInterval()` continua a ripetere una funzione a intervalli regolari. È come avere una sveglia che suona ogni pochi secondi. `setInterval()` è perfetto quando vuoi che qualcosa accada più e più volte, come aggiornare dati o creare un'animazione che si muove in modo continuo.

Proprio come `setTimeout()`, prende una funzione e un ritardo:

```js
setInterval(functionToRun, delay);
```

`functionToRun` è il codice che vuoi eseguire ripetutamente, mentre `delay` è il tempo in millisecondi tra ogni esecuzione di quella funzione. Ecco un esempio:

:::interactive_editor

```js
setInterval(() => {
 console.log("This runs every 2 seconds");
}, 2000);

// Result:
// This runs every 2 seconds
// This runs every 2 seconds
// This runs every 2 seconds
// ...
```

:::

Dato che `setInterval()` continua a eseguire la funzione fornita all'intervallo specificato, potresti volerlo fermare. Per farlo, devi usare il metodo `clearInterval()`.

Per funzionare, `clearInterval()` prende l’ID del `setInterval()` che vuoi fermare. Questo potrebbe essere una variabile a cui assegni l’intervallo. Un modo per fermare l’intervallo è dentro un `setTimeout()`, così si ferma dopo un certo tempo:

:::interactive_editor

```js
const intervalID = setInterval(() => {
 console.log("This will stop after 5 seconds");
}, 1000);

setTimeout(() => {
 clearInterval(intervalID);
}, 5000);
```

:::

Puoi anche fermare un timeout con il metodo `clearTimeout()`:

:::interactive_editor

```js
let timeoutID = setTimeout(() => {
 console.log("This will not run");
}, 5000);

clearTimeout(timeoutID);
```

:::

Con quel codice, il `setTimeout()` non verrà eseguito affatto. Quindi, un modo più elegante per farlo è tramite il DOM, aggiungendo un listener per l’evento clic su un pulsante per fermare il timeout:

:::interactive_editor

```html
<h1>Cancel Timeout Example</h1>
<button id="cancelButton">Cancel Timeout</button>
<script src="index.js"></script>
```

```js
let timeoutID = setTimeout(() => {
  console.log("This will run if not canceled");
}, 5000);

document
.querySelector("#cancelButton")
.addEventListener("click", () => {
  clearTimeout(timeoutID);
  console.log("Timeout canceled!");
});
```

:::
