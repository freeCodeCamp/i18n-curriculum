# --description--

`Promise` は、非同期操作の最終的な完了または失敗を表すオブジェクトです。最初は保留中の状態にあります。その後、操作が成功したときに完了状態に、失敗したときに拒否状態に移行します。以下は`Promise`を作成する例です：

```js
const aPromise = new Promise((resolve, reject) => {
  setTimeout(() => {
    resolve("Operation successful!");
  }, 1000);
});
```

この例では、`setTimeout`を使って非同期操作をシミュレートするpromiseを作成しています。1秒後に、メッセージ`Operation successful!`でpromiseが解決されます。

promiseを扱うもう一つの方法は、`.then`と`.catch`メソッドを使うことです。

`.then()`メソッドはpromiseが完了したときに何をするかを指定するために使われ、`.catch()`は発生したエラーを処理するために使われます。`.then`と`.catch`を使った例を見てみましょう：

```js
aPromise.then((result) => {
  console.log(result);  // Outputs: "Operation successful!"
}).catch((error) => {
  console.error(error);
});
```

このコードでは、promiseが完了したときに実行する指示、つまり`.then()`に渡された関数がpromiseの解決値で呼び出されます。エラーが発生した場合は、`.catch()`に渡された関数が代わりに呼び出されます。

次にpromiseチェーンについて説明します。promiseの強力な機能の一つは、複数の非同期操作を連結できることです。各`.then()`は新しいpromiseを返すことができ、非同期操作のシーケンスを順番に実行できます。例を示します：

```js
fetch('https://api.example.com/data')
  .then(response => response.json())
  .then(data => {
    console.log(data);
    return fetch('https://api.example.com/data2');
  })
  .then(response => response.json())
  .then(data2 => console.log(data2))
  .catch(error => console.error('Error:', error));
```

この例では、2つのAPI呼び出しを順番に行っています。最初の`.then()`はレスポンスをJSONとしてパースします。2番目の`.then()`はデータをログに出力し、さらに別のAPI呼び出しを行います。3番目の`.then()`は2回目のAPI呼び出しのレスポンスをパースし、4番目の`.then()`はそのデータをログに出力します。チェーンのどこかでエラーが発生した場合は、最後の`.catch()`でキャッチされます。

重要なのは、`.catch()`がチェーン内のどのステップのエラーもキャッチすることです。これにより、各ステップに個別のエラーハンドリングを追加する必要がなくなり、コードが大幅に簡単になります。
