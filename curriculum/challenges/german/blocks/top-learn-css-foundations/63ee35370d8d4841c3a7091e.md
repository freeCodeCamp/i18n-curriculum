# --description--

Kombinatoren ermöglichen es uns, mehrere Selektoren anders zu kombinieren als durch Gruppierung oder Verkettung, da sie eine Beziehung zwischen den Selektoren darstellen. Es gibt insgesamt vier Arten von Kombinatoren, aber im Moment zeigen wir Ihnen nur den Nachfahren-Kombinator, der in CSS durch ein einzelnes Leerzeichen zwischen Selektoren dargestellt wird. Ein Nachfahren-Kombinator bewirkt, dass nur Elemente ausgewählt werden, die dem letzten Selektor entsprechen, wenn sie auch einen Vorfahren (Elternteil, Großelternteil usw.) haben, der dem vorherigen Selektor entspricht.

Etwas wie `.ancestor .child` würde also ein Element mit der Klasse `child` auswählen, wenn es einen Vorfahren mit der Klasse `ancestor` hat. Eine andere Möglichkeit, es zu betrachten, ist: Ein Kind wird nur ausgewählt, wenn es innerhalb von `ancestor` verschachtelt ist, egal wie tief. Werfen Sie einen kurzen Blick auf das folgende Beispiel und prüfen Sie, ob Sie erkennen können, welche Elemente basierend auf der angegebenen CSS-Regel ausgewählt würden:

```html
<!-- index.html -->

<div class="ancestor"> <!-- A -->
  <div class="contents"> <!-- B -->
    <div class="contents"> <!-- C -->
    </div>
  </div>
</div>

<div class="contents"></div> <!-- D -->
```

```css
/* styles.css */

.ancestor .contents {
  /* some declarations */
}
```

Im obigen Beispiel würden die ersten beiden Elemente mit der Klasse `contents` (`B` und `C`) ausgewählt, aber das letzte Element (`D`) nicht. War Ihre Vermutung richtig?

Es gibt eigentlich keine Begrenzung, wie viele Kombinatoren Sie zu einer Regel hinzufügen können, daher wäre `.one .two .three .four` völlig gültig. Dies würde einfach ein Element auswählen, das die Klasse `four` hat, wenn es einen Vorfahren mit der Klasse `three` hat, und wenn dieser Vorfahre wiederum einen eigenen Vorfahren mit der Klasse `two` hat, und so weiter. Sie sollten jedoch generell vermeiden, Elemente auszuwählen, die eine so tiefe Verschachtelung benötigen, da dies ziemlich verwirrend und langwierig werden kann und Probleme bei der Spezifität verursachen kann.
