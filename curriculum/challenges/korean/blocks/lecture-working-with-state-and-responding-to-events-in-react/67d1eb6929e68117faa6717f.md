# --description--

React에서 렌더링은 컴포넌트가 사용자 인터페이스(UI), 보통은 브라우저에 나타나는 과정입니다.

React는 모든 JavaScript, JSX, CSS 코드를 받아서 어떻게 보여야 하는지 계산한 후 사용자 인터페이스에 표시합니다.

렌더링 과정은 트리거, 렌더, 커밋의 세 단계로 이루어집니다. 이 단계를 좀 더 자세히 살펴보겠습니다.

트리거 단계는 React가 무언가 변경되었고 사용자 인터페이스를 업데이트해야 할 수도 있다고 감지할 때 발생합니다. 이 변경은 보통 상태(state)나 프로프스(props)의 업데이트 때문입니다.

예를 들어, 저녁 시간이 되었다는 것을 알게 되면 부엌에 가서 요리를 시작하는 것과 같습니다.

아래 `Counter` 예시에서, 증가 또는 감소 버튼을 클릭하면 React가 새로운 `count` 값을 보여주도록 트리거됩니다:

```jsx
import { useState } from "react";

function Counter() {
  const [count, setCount] = useState(0);

  return (
    <div>
      <h1>{count}</h1>
      <button onClick={() => setCount(count - 1)}>Decrement</button>
      <button onClick={() => setCount(count + 1)}>Increment</button>
    </div>
  );
}

export default Counter;
```

트리거가 발생하면 React는 렌더 단계로 들어갑니다. 이 단계에서 React는 컴포넌트를 다시 평가하고 무엇을 표시할지 결정합니다.

이를 위해 React는 "실제" DOM의 가벼운 복사본인 가상 DOM을 사용합니다. 가상 DOM을 통해 React는 컴포넌트에서 변경해야 할 부분을 빠르게 확인할 수 있습니다.

이 단계를 부엌에서 재료를 모아 저녁을 요리하는 시점으로 생각하세요.

`Counter` 컴포넌트의 경우, 렌더 단계는 React가 새로운 `count` 값으로 함수를 다시 실행하는 시점입니다. React는 업데이트된 `<h1>{count}</h1>` 값을 기반으로 `count` 컴포넌트 부분이 어떻게 보여야 하는지 다시 계산하지만, 화면에는 다음 단계인 커밋이 진행될 때까지 아무 변화도 보이지 않습니다.

커밋 단계는 React가 가상 DOM에서 준비한 변경 사항을 실제 DOM에 적용하는 단계입니다. 즉, 최종 결과가 화면에 나타나는 단계입니다.

이 과정을 위해 React는 가상 DOM과 실제 DOM을 비교하여 업데이트가 필요한 부분만 찾아내고, 그 부분만 실제 DOM에 적용해 사용자 인터페이스를 업데이트합니다.

이 단계를 요리한 음식을 서빙하여 눈에 보이게 하는 시점으로 생각할 수 있습니다. React가 실제 DOM에 업데이트를 커밋하는 것과 같습니다.

`Counter` 컴포넌트의 경우, 커밋 단계는 새로운 `count` 값이 `h1` 요소에 적용되어 페이지에서 변화를 볼 수 있는 시점입니다.

이 세 과정은 매우 빠르게 진행되는데, React가 직접 DOM을 조작하는 것을 최소화하고 먼저 가상 DOM에서 변경 사항을 계산한 후 실제 DOM에서 변경이 필요한 부분만 업데이트하기 때문입니다.
