# --interactive--

تداخل الهامش هو مفهوم أساسي في CSS يربك المبتدئين في تطوير الويب كثيرًا.

يحدث هذا السلوك عندما تتداخل الهوامش الرأسية للعناصر المجاورة، مما يؤدي إلى هامش واحد يساوي الأكبر بينهما.

فهم تداخل الهامش مهم للتحكم الدقيق في التباعد والتخطيط في تصميم الويب. لذا، دعنا نتعرف على كيفية عمل تداخل الهامش ونستعرض بعض السيناريوهات الشائعة التي يحدث فيها.

في CSS، عندما تتلامس هوامش رأسية مع بعضها، فإنها تتداخل، وهذا يعني أنه بدلًا من أن تُجمع، يفوز الهامش الأكبر ويحدد المسافة بين العناصر. ينطبق هذا السلوك فقط على الهوامش الرأسية (الأعلى والأسفل) وليس على الهوامش الأفقية (اليسار واليمين). إليك مثالًا لتوضيح هذا المفهوم:

:::interactive_editor

```html
<style>
  .box1 {
    margin-bottom: 20px;
    background-color: lightblue;
  }
  .box2 {
    margin-top: 30px;
    background-color: lightgreen;
  }
</style>

<div class="box1">Box 1</div>
<div class="box2">Box 2</div>
```

:::

في هذا المثال، قد تتوقع أن تكون المسافة الكلية بين `.box1` و `.box2` هي 50 بكسل (20 بكسل زائد 30 بكسل). لكن بسبب تداخل الهامش، ستكون المسافة الفعلية 30 بكسل، وهو الأكبر بين الهوامش.

كما رأينا في المثال السابق، هوامش العناصر الشقيقة المجاورة تتداخل. هذه أبسط حالة لتداخل الهامش. دعنا نستكشف المزيد من الحالات التي يمكن أن يحدث فيها تداخل الهامش.

يمكن أن تتداخل الهوامش أيضًا بين عنصر أبوي وأول أو آخر عنصر فرعي له. إذا لم يكن هناك حدود، أو حشو، أو محتوى مضمّن، أو مسافة فاصلة تفصل هامش الأب عن هامش الطفل، فإنهما يتداخلان.

:::interactive_editor

```html
<style>
  .parent {
    margin-top: 40px;
    background-color: lightyellow;
  }
  .child {
    margin-top: 30px;
    background-color: lightpink;
  }
</style>

<div class="parent">
  <div class="child">Child element</div>
</div>
```

:::

في هذه الحالة، قد تتوقع أن يكون الطفل على بعد 70 بكسل من الأعلى (40 بكسل زائد 30 بكسل). لكن الهوامش تتداخل ويُستخدم الهامش الأكبر وهو 40 بكسل.

إذا لم يكن للعنصر محتوى أو حشو أو حدود، يمكن أن تتداخل هوامشه العلوية والسفلية لتصبح هامشًا واحدًا.

:::interactive_editor

```html
<style>
  .empty-block {
    margin-top: 20px;
    margin-bottom: 10px;
    height: 0;
  }
  .next-block {
    background-color: lightgray;
  }
</style>

<div class="empty-block"></div>
<div class="next-block">Next block</div>
```

:::

في هذا المثال، تتداخل هوامش الأعلى والأسفل للعنصر `empty-block` لتصبح هامشًا واحدًا بقيمة 20 بكسل، وهو الأكبر بينهما.

إليك مثالًا لمنع التداخل باستخدام الحشو:

:::interactive_editor

```html
<style>
  .parent {
    margin-top: 40px;
    padding-top: 1px;
    background-color: lightyellow;
  }
  .child {
    margin-top: 30px;
    background-color: lightpink;
  }
</style>

<div class="parent">
  <div class="child">Child element</div>
</div>
```

:::

في هذه الحالة، يمنع الحشو البالغ بكسل واحد على العنصر الأب تداخل الهامش، مما يؤدي إلى مساحة كلية تبلغ 71 بكسل من أعلى العنصر الأب إلى أعلى محتوى الطفل.

فهم تداخل الهامش مهم للتحكم الدقيق في التخطيط والتباعد في CSS. رغم أنه قد يؤدي أحيانًا إلى نتائج غير متوقعة، فهو ميزة مصممة لإنشاء تباعد أكثر جمالية واتساقًا في المستندات. بمعرفة متى يحدث تداخل الهامش وكيفية منعه عند الحاجة، يمكنك إنشاء تخطيطات أكثر توقعًا وسهولة في الصيانة في تصاميم الويب الخاصة بك.
