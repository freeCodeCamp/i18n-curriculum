# --description--

Montieren Sie einen POST-Handler am Pfad `/name`. Es ist derselbe Pfad wie zuvor. Wir haben ein Formular auf der HTML-Startseite vorbereitet. Es wird dieselben Daten wie in Übung 10 (Query string) absenden. Wenn der Body-Parser korrekt konfiguriert ist, sollten Sie die Parameter im Objekt `req.body` finden. Werfen Sie einen Blick auf das übliche Beispiel aus der Bibliothek:

<blockquote>route: POST '/library'<br>urlencoded_body: userId=546&#x26;bookId=6754 <br>req.body: {userId: '546', bookId: '6754'}</blockquote>

Antworten Sie mit demselben JSON-Objekt wie zuvor: `{name: 'firstname lastname'}`. Testen Sie, ob Ihr Endpunkt mit dem HTML-Formular funktioniert, das wir auf der Startseite der App bereitgestellt haben.

Hinweis: Es gibt mehrere andere HTTP-Methoden neben GET und POST. Und gemäß Konvention besteht eine Entsprechung zwischen dem HTTP-Verb und der Operation, die Sie auf dem Server ausführen möchten. Die konventionelle Zuordnung lautet:

POST (manchmal PUT) – Erstellen einer neuen Ressource mit den Informationen, die mit der Anfrage gesendet werden,

GET – Lesen einer vorhandenen Ressource ohne sie zu verändern,

PUT oder PATCH (manchmal POST) – Aktualisieren einer Ressource mit den gesendeten Daten,

DELETE – Löschen einer Ressource.

Es gibt auch einige andere Methoden, die verwendet werden, um eine Verbindung mit dem Server auszuhandeln. Mit Ausnahme von GET können alle anderen oben genannten Methoden eine Nutzlast haben (also die Daten im Anfrage-Body). Die Middleware Body-Parser funktioniert auch mit diesen Methoden.

Test 1 : Ihr API-Endpunkt sollte mit dem korrekten Namen antworten

```js
  const response = await fetch(code + '/name', {
    method: 'POST',
    headers: {
      'Content-Type': 'application/x-www-form-urlencoded'
    },
    body: new URLSearchParams({ first: 'Mick', last: 'Jagger' }).toString()
  });
  if (!response.ok) {
    throw new Error(await response.text());
  }
  const data = await response.json();
  assert.equal(
    data.name,
    'Mick Jagger',
    'Test 1: "POST /name" route does not behave as expected'
  );
```
