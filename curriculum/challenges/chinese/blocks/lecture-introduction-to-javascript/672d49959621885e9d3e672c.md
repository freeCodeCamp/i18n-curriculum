# --interactive--

在使用 JavaScript 时，你经常会声明变量来存储计划在整个程序中使用的数据。

在现代 JavaScript 中，`let` 和 `const` 是首选的变量声明方式，但它们在处理值赋值和重新赋值的方式上有所不同。

在本课中，我们将探讨 `let` 和 `const` 在变量声明、赋值和重新赋值方面的区别。

`let` 关键字允许你声明可在以后更新或重新赋值的变量。 你可以将 `let` 视为一个灵活的容器——一旦你在其中存储了一个值，你就可以在整个程序中根据需要更改该值。

下面是一个使用 `let` 声明和赋值变量的示例：

```js
let score = 10;
```

在本例中，变量 `score` 被声明并赋值为 `10`。 如果你想稍后更新该值，也很容易做到：

:::interactive_editor

```js
let score = 10;
console.log(score); // 10
score = 20;
console.log(score); // 20
```

:::

现在，`score` 存储的值为 `20`。 当你知道变量的值会随着程序的运行而改变时，`let` 就会特别有用。

而 `const` 用于声明常数变量。 一旦为使用 `const` 声明的变量赋值，就不能重新赋值。

因此，`const` 非常适合用于你不想在程序执行过程中意外更改的值。

下面是一个使用 `const` 声明和赋值变量的示例：

:::interactive_editor

```js
const maxScore = 100;
console.log(maxScore); // 100
```

:::

一旦 `maxScore` 赋值为 `100`，就不能更改：

```js
maxScore = 200; // This will result in an error
```

如果尝试向 `const` 变量重新赋值，则会在 JavaScript 控制台中显示错误，因为 `const` 变量一旦赋值就不可更改。

你可以声明一个 `let` 变量而不立即为其赋值，你可以稍后为其赋值：

:::interactive_editor

```js
let age;
console.log(age); // undefined
age = 25;
console.log(age); // 25
```

:::

使用 `const` 声明的变量必须在声明时赋值。 如果你尝试声明一个 `const` 变量而不为其赋值，则会报错：

```js
const age; // Error: Missing initializer in const declaration
```

当你需要声明稍后将被重新赋值的变量时，应使用 `let` 。 例如，在程序中跟踪随时间变化的分数或更新数值。

当你要声明应保持不变的变量时，如配置值或不应被意外更改的设置，请使用 `const` 。

你也可以使用 `var` 关键字，但不再推荐使用。 `var` 有点像 `let`，只是它的范围更大，更容易在程序中造成问题。
