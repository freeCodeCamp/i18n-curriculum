# --description--

Bevor Sie mit Python-Variablen arbeiten, ist es wichtig, Datentypen zu verstehen. Ein Datentyp beschreibt, welche Art von Wert eine Variable enthält. Zum Beispiel eine Zahl, ein Textstück oder eine Liste von Elementen. Programmiersprachen verwenden Datentypen, damit sie wissen, wie sie verschiedene Arten von Informationen speichern und verarbeiten.

Python ist eine dynamisch typisierte Sprache wie JavaScript, was bedeutet, dass Sie die Typen von Variablen nicht explizit deklarieren müssen. Die Sprache erkennt den Datentyp einer Variable anhand dessen, was Sie ihr zuweisen.

Hier sind einige Beispiele:

```python
name = 'John Doe' # Python knows this is a string
age = 25 # Python knows this is an integer
```

Das steht im Gegensatz zu einigen statisch typisierten Sprachen wie C#, Java und C++, bei denen Sie Typen mit Variablen deklarieren müssen, so:

```csharp
string name = 'John Doe'
int age = 25
```

Die dynamische Typisierung von Python macht das Programmieren sehr schnell und flexibler, kann aber zu unerwarteten Bugs führen, weil Typfehler erst zur Laufzeit erkannt werden, nicht beim Kompilieren.

Da Python die Datentypen während der **Ausführung** Ihres Programms bestimmt, werden typbezogene Fehler erst in diesem Moment entdeckt. Wenn ein Programm läuft, führt Python Ihren Code Zeile für Zeile aus. Wenn es auf eine Zeile trifft, in der ein bestimmtes Objekt sich auf eine Weise verhalten soll, die es nicht kann, stoppt Python und zeigt einen Fehler an.

Im Gegensatz dazu **kompilieren** einige Sprachen Ihr Programm, bevor es ausgeführt wird. Kompilieren bedeutet, dass der Computer Ihren Code im Voraus überprüft und für die Ausführung vorbereitet. Während dieses Schritts können diese Sprachen Typfehler erkennen, bevor das Programm überhaupt startet.

Sie müssen diese Sprachen noch nicht kennen. Die wichtige Idee ist einfach:

- **In Python können sich Typfehler während der Ausführung zeigen**, wenn das Programm tatsächlich läuft und Ihren Code verwendet.

- **Kompilierte Sprachen erkennen Typfehler während des Kompiliervorgangs**, bevor das Programm ausgeführt wird.

Deshalb erfahren Sie in Python möglicherweise erst von einem Typfehler, wenn das Programm genau diese Codezeile erreicht.

Hier sind die gängigsten Datentypen, die Sie in Python verwenden:

- Integer: Eine ganze Zahl ohne Dezimalstellen, zum Beispiel `10` oder `-5`.

```python
my_integer_var = 10
print('Integer:', my_integer_var) # Integer: 10
```

- Float: Eine Zahl mit Dezimalpunkt, wie `4.41` oder `-0.4`.

```python
my_float_var = 4.50
print('Float:', my_float_var) # Float: 4.5
```

- String: Eine Folge von Zeichen, eingeschlossen in einfache oder doppelte Anführungszeichen wie `'Hello world!'`.

```python
my_string_var = 'hello'
print('String:', my_string_var) # String: hello
```

- Boolean: Ein Wahrheitswert, geschrieben als `True` oder `False`.

```python
my_boolean_var = True
print('Boolean:', my_boolean_var) # Boolean: True
```

- Set: Eine ungeordnete Sammlung einzigartiger Elemente, wie `{0.5, 4, 'apple'}`.
 
```python
my_set_var = {7, 'hello', 8.5}
print('Set:', my_set_var) # Set: {7, 'hello', 8.5}
```

- Dictionary: Eine Sammlung von Schlüssel-Wert-Paaren, eingeschlossen in geschweifte Klammern, wie `{'name': 'John Doe', 'age': 28}`. 

```python
my_dictionary_var = {'name': 'Alice', 'age': 25}
print('Dictionary:', my_dictionary_var) # Dictionary: {'name': 'Alice', 'age': 25}
```

- Tuple: Eine unveränderliche geordnete Sammlung, eingeschlossen in runde Klammern, wie `('apple', 4.5, 7)`.

```python
my_tuple_var = (7, 'hello', 8.5)
print('Tuple:', my_tuple_var) # Tuple: (7, 'hello', 8.5)
```

- Range: Eine Folge von Zahlen, oft in Schleifen verwendet, zum Beispiel `range(5)`.

```python
my_range_var = range(5)
print('Range:', my_range_var) # Range: range(0, 5)
```

- List: Eine geordnete Sammlung von Elementen, die verschiedene Datentypen unterstützen.
    
```python
my_list = [22, 'Hello world', 3.14, True]
print(my_list) # [22, 'Hello world', 3.14, True]
```

- None: Ein spezieller Wert, der das Fehlen eines Werts darstellt.

```python
my_none_var = None
print('None:', my_none_var) # None: None
```

Um den Datentyp einer Variable zu ermitteln, können Sie die `type()`-Funktion verwenden:

```python
my_var_1 = 'Hello world'
my_var_2 = 21

print(type(my_var_1)) # <class 'str'>
print(type (my_var_2)) # <class 'int'>
```

Und hier sind alle in dieser Lektion behandelten Datentypen zusammen mit ihren Typen im Terminal:

```python
my_integer_var = 10
print(type(my_integer_var))  # <class 'int'>

my_float_var = 4.50
print(type(my_float_var))  # <class 'float'>

my_string_var = 'hello'
print(type(my_string_var))  # <class 'str'>

my_boolean_var = True
print(type(my_boolean_var))  # <class 'bool'>

my_set_var = {7, 'hello', 8.5}
print(type(my_set_var))  # <class 'set'>

my_dictionary_var = {'name': 'Alice', 'age': 25}
print(type(my_dictionary_var))  # <class 'dict'>

my_tuple_var = (7, 'hello', 8.5)
print(type(my_tuple_var))  # <class 'tuple'>

my_range_var = range(5)
print(type(my_range_var))  # <class 'range'>

my_list = [22, 'Hello world', 3.14, True]
print(type(my_list)) # <class 'list'>

my_none_var = None
print(type(my_none_var))  # <class 'NoneType'>
```

Die eingebaute `isinstance()`-Funktion ermöglicht es Ihnen zu prüfen, ob eine Variable einem bestimmten Datentyp entspricht. Sie nimmt ein Objekt und den Typ, gegen den geprüft werden soll, und gibt einen Boolean zurück. Hier sind einige Beispiele:

```python
isinstance('Hello world', str) # True
isinstance(True, bool) # True
isinstance(42, int) # True
isinstance('John Doe', int) # False
```
