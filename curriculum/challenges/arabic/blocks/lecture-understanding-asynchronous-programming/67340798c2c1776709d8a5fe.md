# --description--

تلعب الخاصيتان `async` و`defer` في عناصر `script` بلغة HTML دورًا مهمًا في كيفية تحميل وتنفيذ ملفات JavaScript في صفحات الويب. فهمهما يحسن أداء موقعك وتجربة المستخدم. عند تضمين `script` في ملف HTML، يكون شكله كالتالي:

```js
<script src="example.js"></script>
```

عندما يجد المتصفح وسم `script` هذا، يتوقف عن تحليل HTML، يحمل الشفرة، ينفذها، ثم يستأنف تحليل HTML. هذا قد يبطئ تحميل صفحة الويب، خاصة إذا كانت الشفرات كبيرة.

هنا تأتي الخاصيتان `async` و`defer` لتوفر طرقًا لتحميل الشفرات بشكل أكثر كفاءة.

لنبدأ بالخاصية `async`:

```js
<script src="example.js" async></script>
```

بإضافة الخاصية `async` إلى وسم `script`، يستمر المتصفح في تحليل HTML أثناء تحميل الشفرة. بمجرد اكتمال تحميل الشفرة، يوقف المتصفح تحليل HTML، ينفذ الشفرة، ثم يستأنف تحليل HTML. هذا يسرع تحميل الصفحة بشكل كبير.

من المهم ملاحظة أن السكربتات غير المتزامنة تُنفَّذ بمجرد تنزيلها، مما يعني أنها قد لا تعمل بالترتيب الصحيح الذي نرغب فيه. وهنا يأتي دور الخاصية `defer` للإنقاذ. لنلقِ نظرة على شكل الخاصية `defer`:

```js
<script src="example.js" defer></script>
```

خاصية `defer` تشبه خاصية `async`. مع ذلك، لا تُنفذ السكربتات التي تحمل خاصية defer فور تحميلها. بل تنتظر حتى يكتمل تحليل HTML. بالإضافة إلى ذلك، تُنفذ السكربتات التي تحمل defer بالترتيب الذي تظهر به في شفرة HTML.

باختصار، استخدم `async` للشفرات التي لا يهم ترتيب تنفيذها، واستخدم `defer` عندما تحتاج لضمان تنفيذ الشفرات بالترتيب الصحيح. كلا الخاصيتين تحسنان أوقات تحميل الصفحة بشكل كبير بالسماح للمتصفح بالاستمرار في تحليل HTML أثناء تحميل الشفرات في الخلفية.
