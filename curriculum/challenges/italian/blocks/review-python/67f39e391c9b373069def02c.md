## Variabili

- **Dichiarare variabili**: Per dichiarare una variabile, inizia con il nome della variabile seguito dall'operatore di assegnazione (`=`) e poi dal valore. Questo può essere un numero, una stringa, un booleano, ecc. Ecco alcuni esempi:

```py
name = 'John Doe'
age = 25
```

- **Convenzioni di denominazione per le variabili**: Ecco le convenzioni che dovresti usare per i nomi delle variabili:

  - I nomi delle variabili possono iniziare solo con una lettera o un underscore (_), non con un numero.
  - I nomi delle variabili possono contenere solo caratteri alfanumerici (a-z, A-Z, 0-9) e underscore (_).
  - I nomi delle variabili fanno distinzione tra maiuscole e minuscole — `age`, `Age` e `AGE` sono tutti considerati unici.
  - I nomi delle variabili non possono essere una delle parole chiave riservate di Python come `if`, `class` o `def`.
  - I nomi delle variabili composti da più parole sono separati da underscore. Ad esempio, `snake_case`.

## Tipi di dato comuni in Python

- **Introduzione**: Python è un linguaggio a tipizzazione dinamica come JavaScript, il che significa che non devi dichiarare esplicitamente i tipi delle variabili. Il linguaggio riconosce il tipo di una variabile in base a ciò che le assegni.
- **Integer**: Un numero intero senza decimali:

```py
my_integer_var = 10
print('Integer:', my_integer_var) # Integer: 10
```

- **Float**: Un numero con decimali:

```py
my_float_var = 4.50
print('Float:', my_float_var) # Float: 4.5
```

- **String**: Una sequenza di caratteri racchiusa tra virgolette:

```py
my_string_var = 'hello'
print('String:', my_string_var) # String: hello
```

- **Boolean**: Un valore che rappresenta `True` o `False`:

```py
my_boolean_var = True
print('Boolean:', my_boolean_var) # Boolean: True
```

- **Set**: Una collezione non ordinata di elementi unici:

```py
my_set_var = {7, 5, 8}
print('Set:', my_set_var) # Set: {7, 5, 8}
```

- **Dizionario**: Una collezione di coppie chiave-valore, racchiusa tra parentesi graffe:

```py
my_dictionary_var = {"name": "Alice", "age": 25}
print('Dictionary:', my_dictionary_var) # Dictionary: {'name': 'Alice', 'age': 25}
```

- **Tupla**: Una collezione ordinata e immutabile, racchiusa tra parentesi tonde:

```py
my_tuple_var = (7, 5, 8)
print('Tuple:', my_tuple_var) # Tuple: (7, 5, 8)
```

- **Range**: Una sequenza di numeri, spesso usata nei loop:

```py
my_range_var = range(5)
print(my_range_var) # range(0, 5)
```

- **Lista**: Una collezione ordinata di elementi che supporta tipi di dato diversi:

```py
my_list = [22, 'Hello world', 3.14, True]
print(my_list) # [22, 'Hello world', 3.14, True]
```

- **None**: Un valore speciale che rappresenta l'assenza di un valore:

```py
my_none_var = None
print('None:', my_none_var) # None: None
```

## Tipi immutabili e mutabili

- **Tipi immutabili**: Questi tipi non possono cambiare una volta dichiarati, anche se puoi riassegnare la variabile a qualcos'altro, operazione chiamata riassegnazione. Includono integer, float, complex, boolean, string, tuple, range e `None`.
- **Tipi mutabili**: Questi tipi possono cambiare una volta dichiarati. Puoi aggiungere, rimuovere o aggiornare i loro elementi. Includono tipi di collezione come list, set e dictionary.

## Lavorare con le stringhe

- **Definizione**: Come ricordi da JavaScript, le stringhe sono immutabili, il che significa che non puoi modificarle dopo che sono state create. In Python puoi usare sia le virgolette singole che doppie. È consigliato scegliere una regola e mantenerla:

```py
developer = 'Jessica'
city = "Los Angeles"
```

- **Consultare caratteri da una stringa**: Puoi consultare i caratteri di una stringa usando la notazione a parentesi in questo modo:

```py
my_str = 'Hello world'

print(my_str[0])  # H
print(my_str[6])  # w

print(my_str[-1])  # d
print(my_str[-2]) # l
```

- **Escape delle stringhe**: Puoi usare una barra rovesciata (`\`) se la tua stringa contiene virgolette, come in questo esempio:

```py
msg = 'It\'s a sunny day'
quote = "She said, \"Hello!\""
```

- **Concatenazione di stringhe**: Per concatenare stringhe puoi usare l'operatore `+` in questo modo:

```py
developer = 'Jessica'
print('My name is ' + developer + '.') # My name is Jessica.
```

Un altro modo per concatenare stringhe è usare l'operatore `+=`. Questo viene usato per concatenare e assegnare nello stesso passaggio, come in questo esempio:

```py
greeting = 'My name is '
developer = 'Jessica.'

greeting += developer
print(greeting) # My name is Jessica.
```

- **`f-strings`**: È l'abbreviazione di stringhe letterali formattate. Ti permette di gestire l'interpolazione e anche di fare un po' di concatenazione con una sintassi compatta e leggibile:

```py
developer = 'Jessica'
greeting = f'My name is {developer}.'
print(greeting) # My name is Jessica.
```

- **Slicing delle stringhe**: Serve per estrarre porzioni di una stringa. Ecco la sintassi di base:

```py
str[start:stop:step]
```

La posizione di inizio rappresenta l'indice da cui deve iniziare l'estrazione. La posizione di stop è dove lo slice deve terminare. Questa posizione è non inclusiva. La posizione di step rappresenta l'intervallo di incremento per lo slicing. Ecco alcuni esempi:

```py
message = 'Python is fun!'

print(message[0:6])  # Python
print(message[7:])  # is fun!
print(message[::2])  # Pto sfn
```

- **Ottenere la lunghezza di una stringa**: La funzione `len()` viene usata per restituire il numero di caratteri nella stringa:

```py
developer = 'Jessica'

print(len(developer)) # 7
```

## Lavorare con l'operatore `in`

## Metodi comuni delle stringhe

- **`upper()`**: Questo metodo restituisce una nuova stringa con tutti i caratteri convertiti in maiuscolo:

```py
developer = 'Jessica'

print(developer.upper()) # JESSICA
```

- **`lower()`**: Questo metodo restituisce una nuova stringa con tutti i caratteri convertiti in minuscolo:

```py
developer = 'Jessica'

print(developer.lower()) # jessica
```

- **`strip()`**: Questo metodo restituisce una copia della stringa con i caratteri iniziali e finali specificati rimossi (se non viene passato nessun argomento, rimuove gli spazi bianchi iniziali e finali).

```py
greeting = '  hello world  '

trimmed_my_str = greeting.strip()
print(trimmed_my_str)  # 'hello world'
```

- **`replace()`**: Questo restituisce una nuova stringa con tutte le occorrenze della vecchia stringa sostituite da una nuova.

```py
greeting = 'hello world'

replaced_my_str = greeting.replace('hello', 'hi')
print(replaced_my_str)  # 'hi world'
```

- **`split()`**: Serve per dividere una stringa in una lista usando un separatore specificato. Un separatore è una stringa che specifica dove deve avvenire la divisione.

```py
dashed_name = 'example-dashed-name'

split_words = dashed_name.split('-')
print(split_words)  # ['example', 'dashed', 'name']
```

- **`join()`**: Serve per unire gli elementi di un iterabile in una stringa con un separatore. Un iterabile è una collezione di elementi su cui si può iterare, come una lista, una stringa o una tupla.

```py
example_list = ['example', 'dashed', 'name']

joined_str = ' '.join(example_list)
print(joined_str)  # example dashed name
```

- **`startswith(prefix)`**: Questo metodo restituisce un booleano che indica se una stringa inizia con il prefisso specificato:

```py
developer = 'Naomi'

result = developer.startswith('N')
print(result)  # True
```

- **`endswith(suffix)`**: Questo metodo restituisce un booleano che indica se una stringa termina con il suffisso specificato:

```py
developer = 'Naomi'

result = developer.endswith('N')
print(result)  # False
```

- **`find()`**: Questo metodo restituisce l'indice della prima occorrenza di una sottostringa. Se non viene trovata, viene restituito `-1`:

```py
developer = 'Naomi'

result = developer.find('N')
print(result)  # 0

city = 'Los Angeles'
print(city.find('New')) # -1
```

- **`count(substring)`**: Questo metodo conta quante volte una sottostringa appare in una stringa:

```py
city = 'Los Angeles'
print(city.count('e')) # 2
```

- **`capitalize()`**: Questo metodo restituisce una nuova stringa con il primo carattere maiuscolo e gli altri caratteri in minuscolo:

```py
dessert = 'chocolate cake'
print(dessert.capitalize()) # Chocolate cake
```

- **`isupper()`**: Questo metodo restituisce `True` se tutte le lettere nella stringa sono maiuscole e `False` altrimenti:

```py
dessert = 'chocolate cake'
print(dessert.isupper()) # False
```

- **`islower()`**: Questo metodo restituisce `True` se tutte le lettere nella stringa sono minuscole e `False` altrimenti:

```py
dessert = 'chocolate cake'
print(dessert.islower()) # True
```

- **`title()`**: Questo metodo restituisce una nuova stringa con la prima lettera di ogni parola maiuscola:

```py
city = 'los angeles'
print(city.title()) # Los Angeles
```

- **`maketrans()`**: Questo metodo viene usato per creare una tabella di mappature carattere per carattere per la traduzione. Spesso viene usato insieme al metodo `translate()`, che applica quella tabella a una stringa e restituisce il risultato tradotto.

```py
trans_table = str.maketrans('abc', '123')
print(trans_table) # {97: 49, 98: 50, 99: 51}

result = 'abcabc'.translate(trans_table)
print(result)  # 123123
```

## Lavorare con le funzioni

```py
def get_sum(num_1, num_2=2):
```

Se chiami la funzione senza il numero corretto di argomenti, otterrai un `TypeError`:

```py
def calculate_sum(a, b):
```

## Ambito in Python

- **Ambito locale**: Si verifica quando una variabile dichiarata all'interno di una funzione o classe può essere consultata solo all'interno di quella funzione o classe.

```py
def my_func():
    num = 10
    print(num)
```

- **Ambito di chiusura**: Si verifica quando una funzione annidata all'interno di un'altra funzione può consultare le variabili della funzione in cui è annidata.

```py
def outer_func():
    msg = 'Hello there!'
    
    def inner_func():
        print(msg)
    inner_func()

print(outer_func()) # Hello there!
```

- **Ambito globale**: Si riferisce alle variabili dichiarate all'esterno di qualsiasi funzione o classe, che possono essere consultate da qualsiasi punto del programma.

```py
tax = 0.70 

def get_total(subtotal):
    total = subtotal + (subtotal * tax)
    return total

print(get_total(100))  # 170.0
```

- **Ambito integrato**: Nomi riservati in Python per funzioni, moduli, parole chiave e oggetti predefiniti.

```py
print(str(45)) # '45'
print(type(3.14)) # <class 'float'>
print(isinstance(3, str)) # False
```

## Lavorare con le istruzioni `if`, `elif` e `else`

- **Istruzioni `if`**: Sono condizioni usate per determinare se qualcosa è vero o meno. Se la condizione restituisce `True`, allora quel blocco di codice verrà eseguito.

```py
age = 18
```

```py
age = 16

```

```py
age = 12

if age >= 18:
    print('You are an adult')
elif age >= 13:
    print('You are a teenager')
else:
    print('You are a child')  # You are a child
```

Puoi anche usare istruzioni `if` annidate come in questo esempio:

```py
is_citizen = True
age = 25

if is_citizen:
    if age >= 18:

    print('You are not eligible to vote')
```

## Valori truthy e falsy

- **Definizione**: In Python, ogni valore ha un valore booleano intrinseco, ovvero un senso integrato di se debba essere trattato come `True` o `False` in un contesto logico. Molti valori sono considerati truthy, cioè restituiscono `True` in un contesto logico. Altri sono falsy, cioè restituiscono `False`. Ecco alcuni esempi di valori falsy:

```md
None
False
Integer 0
Float 0.0
Empty strings ''
```

Altri valori come numeri diversi da zero e stringhe non vuote sono truthy.

## Lavorare con la funzione `bool()`

- **Definizione**: Se vuoi verificare se un valore è truthy o falsy, puoi usare la funzione integrata `bool()`. Converte esplicitamente un valore nel suo equivalente booleano e restituisce `True` per i valori truthy e `False` per quelli falsy. Ecco alcuni esempi:

```py
print(bool(False)) # False
print(bool(0))  # False
print(bool('')) # False

print(bool(True)) # True
print(bool(1)) # True
print(bool('Hello')) # True
```

## Operatori booleani e short-circuiting

- **Definizione**: Sono operatori speciali che ti permettono di combinare più espressioni per creare logiche decisionali più complesse nel tuo codice. In Python ci sono tre operatori booleani: `and`, `or` e `not`.
- **Operatore `and`**: Questo operatore prende due operandi e restituisce il primo operando se è falsy, altrimenti restituisce il secondo operando. Entrambi gli operandi devono essere truthy affinché l'espressione restituisca un valore truthy.

```py
is_citizen = True
age = 25

print(is_citizen and age) # 25
```

Puoi anche usare l'operatore `and` nelle condizioni come in questo esempio:

```py
is_citizen = True
age = 25

if is_citizen and age >= 18:
    print('You are eligible to vote') # You are eligible to vote
else:
    print('You are not eligible to vote')
```

- **Operatore `or`**: Questo operatore restituisce il primo operando se è truthy, altrimenti restituisce il secondo operando. Un'espressione `or` restituisce un valore truthy se almeno uno degli operandi è truthy. Ecco un esempio:

```py
age = 19
is_employed = False

print(age or is_employed) # 19
```

Come per l'operatore `and`, puoi usare l'operatore `or` nelle condizioni come in questo esempio:

```py
age = 19
is_student = True

if age < 18 or is_student:
    print('You are eligible for a student discount') # You are eligible for a student discount
else:
    print('You are not eligible for a student discount')
```

- **Short-circuiting**: Gli operatori `and` e `or` sono detti operatori short-circuit. Short-circuiting significa che Python controlla i valori da sinistra a destra e si ferma appena determina il risultato finale.
- **Operatore `not`**: Questo operatore prende un solo operando e ne inverte il valore booleano. Converte i valori truthy in `False` e i valori falsy in `True`. A differenza degli operatori precedenti, `not` restituisce sempre `True` o `False`. Ecco alcuni esempi:

```py
print(not '') # True, because empty string is falsy
print(not 'Hello') # False, because non-empty string is truthy
print(not 0) # True, because 0 is falsy
print(not 1) # False, because 1 is truthy
print(not False) # True, because False is falsy
print(not True) # False, because True is truthy
```

Ecco un esempio dell'operatore `not` in una condizione:

```py
is_admin = False

if not is_admin:
    print('Access denied for non-administrators.') # Access denied for non-administrators.
else:
    print('Welcome, Administrator!')
```


## Liste in Python

- **Introduzione**: In Python, il tipo di dato list è una sequenza ordinata di elementi che può essere composta da stringhe, numeri o anche altre liste. Le liste sono mutabili e indicizzate a base zero.

```python
cities = ['Los Angeles', 'London', 'Tokyo']
```

- **Consultare elementi in una lista**: Per consultare un elemento dalla lista `cities`, puoi fare riferimento al suo numero di indice nella sequenza:

```python
cities = ['Los Angeles', 'London', 'Tokyo']
cities[0] # Los Angeles
```

- **Consultare elementi usando l'indicizzazione negativa**: Per consultare l'ultimo elemento di una lista, puoi usare `-1` come numero di indice:

```python
cities = ['Los Angeles', 'London', 'Tokyo']
cities[-1] # Tokyo
```

- L'indicizzazione negativa viene usata per consultare elementi a partire dalla fine della lista invece che dall'inizio all'indice `0`.

- **Creare liste usando il costruttore `list()`**: Le liste possono essere create anche usando il costruttore `list()`. Il costruttore `list()` viene usato per convertire un iterabile in una lista:

```python
developer = 'Jessica'

print(list(developer)) 
# Result: ['J', 'e', 's', 's', 'i', 'c', 'a']
```

- **Trovare la lunghezza di una lista**: Puoi usare la funzione `len()` per ottenere la lunghezza di una lista:

```python
numbers = [1, 2, 3, 4, 5]
len(numbers) # 5
```

- **Mutabilità delle liste**: Le liste sono mutabili, il che significa che puoi aggiornare qualsiasi elemento della lista purché passi un indice valido. Per aggiornare le liste a un indice specifico, puoi assegnare un nuovo valore a quell'indice:

```python
programming_languages = ['Python', 'Java', 'C++', 'Rust']
programming_languages[0] = 'JavaScript'
print(programming_languages) # ['JavaScript', 'Java', 'C++', 'Rust']
```

- **Errore di indice fuori intervallo**: Se passi un indice (positivo o negativo) che è fuori dai limiti della lista, riceverai un `IndexError`:

```python
programming_languages = ['Python', 'Java', 'C++', 'Rust']
programming_languages[10] = 'JavaScript'

"""
Traceback (most recent call last):
  File "<stdin>", line 1, in <module>
IndexError: list assignment index out of range
"""
```

- **Rimuovere elementi da una lista**: Gli elementi possono essere rimossi da una lista usando la parola chiave `del`:

```python
developer = ['Jane Doe', 23, 'Python Developer']
del developer[1]
print(developer) # ['Jane Doe', 'Python Developer']
```

- **Verificare se un elemento esiste in una lista**: La parola chiave `in` può essere usata per verificare se un elemento esiste in una lista:

```py
programming_languages = ['Python', 'Java', 'C++', 'Rust']

'Rust' in programming_languages # True
'JavaScript' in programming_languages # False
```

- **Annidare liste**: Le liste possono essere annidate all'interno di altre liste:

```python
developer = ['Alice', 25, ['Python', 'Rust', 'C++']]
```

- Per consultare la lista annidata, dovrai consultarla usando l'indice `2` poiché le liste sono indicizzate a base zero.

```python
developer = ['Alice', 25, ['Python', 'Rust', 'C++']]
developer[2] # ['Python', 'Rust', 'C++']
```

- Per consultare ulteriormente la seconda lingua da quella lista annidata, dovrai consultarla usando l'indice `1`:

```python
developer = ['Alice', 25, ['Python', 'Rust', 'C++']]
developer[2][1] # Rust
```

- **Unpacking dei valori da una lista:** L'unpacking dei valori da una lista è una tecnica usata per assegnare i valori di una lista a nuove variabili. Ecco un esempio per fare l'unpacking della lista `developer` in nuove variabili chiamate `name`, `age` e `job` in questo modo:

```py
developer = ['Alice', 34, 'Rust Developer']
name, age, job = developer
```

- Se il numero di variabili a sinistra dell'operatore di assegnazione non corrisponde al numero totale di elementi nella lista, riceverai un `ValueError`.

- **Raccogliere gli elementi rimanenti da una lista**: Per raccogliere eventuali elementi rimanenti da una lista, puoi usare l'operatore asterisco (`*`) in questo modo:

```py
developer = ['Alice', 34, 'Rust Developer']
name, *rest = developer
```

- **Slicing delle liste**: Lo slicing è il concetto di consultare una porzione di una lista usando l'operatore di slice `:`. Per fare lo slicing di una lista che parte dall'indice `1` e termina prima dell'indice `3`, puoi usare la seguente sintassi:

```python
desserts = ['Cake', 'Cookies', 'Ice Cream', 'Pie']
desserts[1:3] # ['Cookies', 'Ice Cream']
```

- **Intervalli di step**: È anche possibile specificare un intervallo di step che determina di quanto incrementare tra gli indici. Ecco un esempio se vuoi estrarre una lista di soli numeri pari usando lo slicing:

```python
numbers = [1, 2, 3, 4, 5, 6]
numbers[1::2] # [2, 4, 6]
```

## Metodi delle liste

- **append()**: Usato per aggiungere un elemento alla fine della lista. Ecco un esempio di utilizzo del metodo `append()` per aggiungere il numero `6` a questa lista `numbers`:

```py
numbers = [1, 2, 3, 4, 5]
numbers.append(6)
print(numbers) # [1, 2, 3, 4, 5, 6]
```

- **Aggiungere liste**: Il metodo `append()` può essere usato anche per aggiungere una lista alla fine di un'altra:

```py
numbers = [1, 2, 3, 4, 5]
even_numbers = [6, 8, 10]

numbers.append(even_numbers)
print(numbers) # [1, 2, 3, 4, 5, [6, 8, 10]]
```

- **extend()**: Usato per aggiungere più elementi alla fine di una lista. Ecco un esempio di aggiunta dei numeri `6`, `8` e `10` alla fine della lista `numbers`:

```py
numbers = [1, 2, 3, 4, 5]
even_numbers = [6, 8, 10]

numbers.extend(even_numbers)
print(numbers) # [1, 2, 3, 4, 5, 6, 8, 10]
```

- **insert()**: Usato per inserire un elemento a un indice specifico nella lista. Ecco un esempio di utilizzo del metodo `insert()`:

```py
numbers = [1, 2, 3, 4, 5]
numbers.insert(2, 2.5)

print(numbers) # [1, 2, 2.5, 3, 4, 5]
```

- **remove():** Usato per rimuovere un elemento dalla lista. Il metodo `remove()` rimuoverà solo la prima occorrenza di un elemento nella lista:

```py
numbers = [1, 2, 3, 4, 5, 5, 5]
numbers.remove(5)

print(numbers) # [1, 2, 3, 4, 5, 5]

```

- **pop()**: Usato per rimuovere un elemento specifico dalla lista e restituirlo:

```py
numbers = [1, 2, 3, 4, 5]
numbers.pop(1) # The number 2 is returned
```

- Se non specifichi un elemento per il metodo `pop`, allora verrà rimosso l'ultimo elemento.

```py
numbers = [1, 2, 3, 4, 5]
numbers.pop() # The number 5 is returned
```

- **clear()**: Usato per rimuovere tutti gli elementi dalla lista:

```py
numbers = [1, 2, 3, 4, 5]
numbers.clear()

print(numbers) # []
```

- **sort()**: Il metodo `sort()` viene usato per ordinare gli elementi in loco. Ecco un esempio di ordinamento in loco di una lista casuale di `numbers`:

```py
numbers = [19, 2, 35, 1, 67, 41]
numbers.sort()

print(numbers) # [1, 2, 19, 35, 41, 67]
```

- **sorted()**: Una funzione integrata che restituisce una nuova lista ordinata invece di modificare la lista originale:

```py
numbers = [19, 2, 35, 1, 67, 41]
sorted_numbers = sorted(numbers)

print(sorted_numbers) # [1, 2, 19, 35, 41, 67]
print(numbers) # [19, 2, 35, 1, 67, 41]
```

- **reverse()**: Usato per invertire l'ordine degli elementi in una lista:

```py
numbers = [6, 5, 4, 3, 2, 1]
numbers.reverse()

print(numbers) # [1, 2, 3, 4, 5, 6]
```

- **index()**: Usato per trovare il primo indice in cui un elemento può essere trovato in una lista:

```python
programming_languages = ['Rust', 'Java', 'Python', 'C++']
programming_languages.index('Java') # 1
```

- Se l'elemento non può essere trovato usando il metodo `index()`, il risultato sarà un `ValueError`.

## Tuple in Python

- **Definizione**: Una tupla è un tipo di dato Python usato per creare una sequenza ordinata di valori. Le tuple possono contenere un insieme misto di tipi di dato:

```py
developer = ('Alice', 34, 'Rust Developer')
```

- Le tuple sono immutabili, cioè gli elementi nella tupla non possono essere modificati una volta creati. Se provi ad aggiornare uno degli elementi nella tupla, otterrai un `TypeError`:

```py
programming_languages = ('Python', 'Java', 'C++', 'Rust')
programming_languages[0] = 'JavaScript'

"""
Traceback (most recent call last):
  File "<stdin>", line 1, in <module>
TypeError: "tuple" object does not support item assignment
"""
```

- **Consultare elementi da una tupla**: Per consultare un elemento da una tupla, usa la notazione a parentesi e il numero di indice:

```py
developer = ('Alice', 34, 'Rust Developer')
developer[1] # 34
```

- L'indicizzazione negativa può essere usata per consultare elementi a partire dalla fine della tupla:

```py
numbers = (1, 2, 3, 4, 5)
numbers[-2] # 4
```

- Se provi a passare un numero di indice che supera o è uguale alla lunghezza della tupla, riceverai un `IndexError`:

```py
numbers = (1, 2, 3, 4, 5)
numbers[7]

"""
Traceback (most recent call last):
  File "<stdin>", line 1, in <module>
IndexError: tuple index out of range
"""
```

- Una tupla può essere creata anche usando il costruttore `tuple()`. All'interno del costruttore puoi passare diversi iterabili come stringhe, liste e anche altre tuple.

```python
developer = 'Jessica'

print(tuple(developer)) 
# Result: ('J', 'e', 's', 's', 'i', 'c', 'a')
```

- **Verificare elementi in una tupla**: Per verificare se un elemento è presente in una tupla, puoi usare la parola chiave `in` in questo modo:

```py
programming_languages = ('Python', 'Java', 'C++', 'Rust')

'Rust' in programming_languages # True
'JavaScript' in programming_languages # False
```

- **Unpacking delle tuple**: Gli elementi possono essere estratti da una tupla in questo modo:

```py
developer = ('Alice', 34, 'Rust Developer')
name, age, job = developer
```

- Se hai bisogno di raccogliere eventuali elementi rimanenti da una tupla, puoi usare l'operatore asterisco (`*`) in questo modo:

```py
developer = ('Alice', 34, 'Rust Developer')
name, *rest = developer
```

- **Slicing delle tuple**: Lo slicing può essere usato per estrarre una porzione di una tupla. Ad esempio, gli elementi `pie` e `cookies` possono essere estratti in una tupla separata:

```py
desserts = ('cake', 'pie', 'cookies', 'ice cream')
desserts[1:3] # ('pie', 'cookies')
```

- **Rimuovere elementi da una tupla**: Rimuovere un elemento da una tupla genererà un `TypeError` poiché le tuple sono immutabili:

```py
developer = ('Jane Doe', 23, 'Python Developer')
del developer[1]

"""
Traceback (most recent call last):
  File "<stdin>", line 1, in <module>
TypeError: "tuple" object doesn't support item deletion
"""
```

- **Quando usare una tupla o una lista?**: Se hai bisogno di una collezione dinamica di elementi in cui puoi aggiungere, rimuovere e aggiornare elementi, allora dovresti usare una lista. Se sai che stai lavorando con una collezione fissa e immutabile di dati, allora dovresti usare una tupla.

## Metodi comuni delle tuple

- **`count()`**: Usato per determinare quante volte un elemento appare in una tupla. Ad esempio, puoi verificare quante volte la lingua `'Rust'` appare nella tupla:

```py
programming_languages = ('Rust', 'Java', 'Python', 'C++', 'Rust')
programming_languages.count('Rust') # 2
```

- Se l'elemento specificato nel metodo `count()` non è presente nella tupla, il valore di ritorno sarà `0`:

```py
programming_languages = ('Rust', 'Java', 'Python', 'C++', 'Rust')
programming_languages.count('JavaScript') # 0
```

- Se non vengono passati argomenti al metodo `count()`, Python restituirà un `TypeError`.

- **index()**: Usato per trovare l'indice in cui un elemento specifico è presente nella tupla. Ecco un esempio di utilizzo del metodo `index()` per trovare l'indice della lingua `'Java'`:

```py
programming_languages = ('Rust', 'Java', 'Python', 'C++', 'Rust')
programming_languages.index('Java') # 1
```

- Se l'elemento specificato non può essere trovato, Python restituirà un `ValueError`.

- Puoi passare un indice di inizio opzionale al metodo `index()` per specificare da dove iniziare la ricerca dell'elemento nella tupla:

```py
programming_languages = ('Rust', 'Java', 'Python', 'C++', 'Rust', 'Python')
programming_languages.index('Python', 3) # 5
```

- Puoi anche passare un indice di fine opzionale al metodo `index()` per specificare dove terminare la ricerca dell'elemento nella tupla:

```py
programming_languages = ('Rust', 'Java', 'Python', 'C++', 'Rust', 'Python', 'JavaScript', 'Python')
programming_languages.index('Python', 2, 5) # 2
```

- **`sorted()`**: Usato per ordinare gli elementi di qualsiasi iterabile e restituire una nuova lista ordinata. Ecco un esempio di creazione di una nuova lista di numeri usando la funzione `sorted()`:

```py
numbers = (13, 2, 78, 3, 45, 67, 18, 7)
sorted(numbers) # [2, 3, 7, 13, 18, 45, 67, 78]
```

- **Modificare il comportamento dell'ordinamento**: Puoi personalizzare il comportamento dell'ordinamento per un iterabile usando gli argomenti opzionali `reverse` e `key`. Ecco un esempio di utilizzo dell'argomento `key` per ordinare gli elementi di una tupla in base alla lunghezza:

```py
programming_languages = ('Rust', 'Java', 'Python', 'C++', 'Rust', 'Python')
sorted(programming_languages, key=len)

# Result
# ['C++', 'Rust', 'Java', 'Rust', 'Python', 'Python']
```

- Puoi creare una nuova lista di valori in ordine inverso usando l'argomento `reverse` in questo modo:

```py
programming_languages = ('Rust', 'Java', 'Python', 'C++', 'Rust', 'Python')

print(sorted(programming_languages, reverse=True))

# Result
# ['Rust', 'Rust', 'Python', 'Python', 'Java', 'C++']
```

## Loop in Python

- **Definizione**: I loop vengono usati per ripetere un blocco di codice per un numero definito di volte.

- **Loop `for`**: Usato per iterare su una sequenza (come una lista, una tupla o una stringa) ed eseguire un blocco di codice per ogni elemento di quella sequenza. Ecco un esempio di utilizzo di un loop `for` per iterare su una lista e stampare ogni lingua sulla console:

```py
programming_languages = ['Rust', 'Java', 'Python', 'C++']

for language in programming_languages:
    print(language)

"""
Result 

Rust
Java
Python
C++
"""
```

- Ecco un esempio di utilizzo di un loop `for` per iterare sulla stringa `code` e stampare ogni carattere:

```py
for char in 'code':
    print(char)

"""
Result 

c
o
d
e
"""
```

- I loop `for` possono essere annidati. Ecco un esempio di utilizzo di un loop `for` annidato:

```py
categories = ['Fruit', 'Vegetable']
foods = ['Apple', 'Carrot', 'Banana']

for category in categories:
    for food in foods:
        print(category, food)

Result

Fruit Apple
```

```py
secret_number = 3
guess = 0

while guess != secret_number:
    guess = int(input('Guess the number (1-5): '))
    if guess != secret_number:
        print('Wrong! Try again.')

print('You got it!')

"""
Result

Guess the number (1-5): 2
```

- **Istruzioni `break` e `continue`**: Usate nei loop per modificare l'esecuzione di un loop.

- L'istruzione `break` viene usata per uscire immediatamente dal loop quando viene soddisfatta una certa condizione. Ecco un esempio di utilizzo dell'istruzione `break` per una lista di `developer_names`:

```py
developer_names = ['Jess', 'Naomi', 'Tom']

for developer in developer_names:

        break
    print(developer)
```

- L'istruzione `continue` viene usata per saltare l'iterazione corrente e passare alla successiva del loop. Ecco un esempio di utilizzo dell'istruzione `continue` invece di una `break`:

```python
words = ['sky', 'apple', 'rhythm', 'fly', 'orange']
```

## Range e il loro uso nei loop

- **La funzione `range()`**: Usata per generare una sequenza di numeri interi.

```py
range(start, stop, step)
```

- L'argomento obbligatorio `stop` è un intero (non incluso) che rappresenta il punto finale della sequenza di numeri generata. Ecco un esempio di utilizzo della funzione `range()`:

```py
for num in range(3):
    print(num)
```

- Se non viene specificato un argomento `start`, il valore predefinito sarà `0`. Per impostazione predefinita, la sequenza di numeri interi incrementerà di `1`. Puoi usare l'argomento opzionale `step` per cambiare il valore di incremento predefinito. Ecco un esempio di generazione di una sequenza di numeri pari da 2 fino a ma non incluso 11 (cioè include 10)

```py
for num in range(2, 11, 2):
    print(num)
```

- Se non fornisci nessun argomento alla funzione `range()`, otterrai un `TypeError`.

- La funzione `range()` accetta solo numeri interi come argomenti e non float. Usare float genererà anche un `TypeError`:

```py
ERROR!
Traceback (most recent call last):
  File "<main.py>", line 1, in <module>
TypeError: 'float' object cannot be interpreted as an integer
```

- Puoi usare un intero negativo per l'argomento `step` per generare una sequenza di numeri interi in ordine decrescente:

```py
for num in range(40, 0, -10):
    print(num)
```

- La funzione `range()` può essere usata anche per creare una lista di numeri interi usandola con il costruttore `list`. Il costruttore `list` viene usato per convertire un iterabile in una lista. Ecco un esempio di generazione di una lista di numeri pari tra 2 e 10 inclusi:

```py
numbers = list(range(2, 11, 2))
print(numbers) # [2, 4, 6, 8, 10]
```

## Funzioni `enumerate()` e `zip()` in Python

- **`enumerate()`**: usata per iterare su una sequenza e tenere traccia dell'indice di ogni elemento in quella sequenza. La funzione `enumerate()` prende un iterabile come argomento e restituisce un oggetto `enumerate` che consiste nell'indice e nel valore di ogni elemento dell'iterabile.

```python
languages = ['Spanish', 'English', 'Russian', 'Chinese']

for index, language in enumerate(languages):
    print(f'Index {index} and language {language}')

# Result
# Index 0 and language Spanish
# Index 1 and language English
# Index 2 and language Russian
# Index 3 and language Chinese

```

- La funzione `enumerate()` può essere usata anche al di fuori di un loop `for`:

```python
languages = ['Spanish', 'English', 'Russian', 'Chinese']

print(list(enumerate(languages)))
# [(0, 'Spanish'), (1, 'English'), (2, 'Russian'), (3, 'Chinese')]
```

- La funzione `enumerate()` accetta anche un argomento opzionale `start` che specifica il valore iniziale del conteggio. Se questo argomento viene omesso, il conteggio inizierà da `0`.

-  **`zip()`** : Usata per iterare su più iterabili in parallelo. Ecco un esempio di utilizzo della funzione `zip()` per iterare su `developers` e `ids`:

```python
developers = ['Naomi', 'Dario', 'Jessica', 'Tom']
ids = [1, 2, 3, 4]

for name, id in zip(developers, ids):
    print(f'Name: {name}')
    print(f'ID: {id}')


"""
Result

Name: Naomi
ID: 1
Name: Dario
ID: 2
Name: Jessica
ID: 3
Name: Tom
ID: 4
"""

```

## List comprehension in Python

- **Definizione**: La list comprehension ti permette di creare una nuova lista in una sola riga combinando il loop e la condizione direttamente tra parentesi quadre. Questo rende il codice più corto e spesso più leggibile.

```py
even_numbers = [num for num in range(21) if num % 2 == 0]
print(even_numbers)
```

## Funzioni sugli iterabili

- **`filter()`**: Usata per filtrare elementi da un iterabile in base a una condizione. Restituisce un iteratore che contiene solo gli elementi che soddisfano la condizione. Ecco un esempio di creazione di una nuova lista di sole parole più lunghe di quattro caratteri:

```py
words = ['tree', 'sky', 'mountain', 'river', 'cloud', 'sun']

def is_long_word(word):
    return len(word) > 4

long_words = list(filter(is_long_word, words))
print(long_words) # ['mountain', 'river', 'cloud']
```

- **`map()`**: Usata per applicare una funzione a ogni elemento di un iterabile e restituire un nuovo iterabile con i risultati. Ecco un esempio di utilizzo della funzione `map()` per convertire una lista di temperature in celsius in fahrenheit:

```py
celsius = [0, 10, 20, 30, 40]

def to_fahrenheit(temp):
    return (temp * 9/5) + 32

fahrenheit = list(map(to_fahrenheit, celsius))
print(fahrenheit) # [32.0, 50.0, 68.0, 86.0, 104.0]
```

- **`sum()`**: Usata per ottenere la somma da un iterabile come una lista o una tupla. Ecco un esempio di utilizzo della funzione `sum()`:

```py
numbers = [5, 10, 15, 20]
total = sum(numbers)
print(total) # Result: 50
```

- Puoi anche passare un argomento opzionale `start` che imposta il valore iniziale per la somma. Ecco un esempio aggiornato usando l'argomento `start` come argomento posizionale:

```py
numbers = [5, 10, 15, 20]
total = sum(numbers, 10) # positional argument
print(total) # 60
```

- Puoi anche scegliere di usare l'argomento `start` come argomento keyword in questo modo:

```py
numbers = [5, 10, 15, 20]
total = sum(numbers, start=10) # keyword argument
print(total) # 60
```

## Funzioni lambda

- **Definizione**: Una funzione lambda in Python è un modo conciso per creare una funzione senza nome (una funzione anonima).
- Le funzioni lambda sono spesso usate come argomento di un'altra funzione. Ecco un esempio di funzione lambda:

```py
numbers = [1, 2, 3, 4, 5]

even_numbers = list(filter(lambda x: x % 2 == 0, numbers))
print(even_numbers)  # [2, 4]
```

- Le buone pratiche per l'uso delle funzioni lambda includono il non assegnarle a una variabile, mantenerle semplici e leggibili e usarle per funzioni brevi e di una sola volta.

- **Dizionari**: I dizionari sono strutture dati integrate che archiviano collezioni di coppie chiave-valore. Le chiavi devono essere tipi di dato immutabili. Questa è la sintassi generale di un dizionario Python:

```python
dictionary = {
    key1: value1,
    key2: value2
}
```

- **Costruttore `dict()`**: Il costruttore `dict()` è un modo alternativo per costruire il dizionario. Passi una lista di tuple come argomento al costruttore `dict()`. Queste tuple contengono la chiave come primo elemento e il valore come secondo elemento.

```python
pizza = dict([('name', 'Margherita Pizza'), ('price', 8.9), ('calories_per_slice', 250), ('toppings', ['mozzarella', 'basil'])])
```

- **Notazione a parentesi**: Per consultare il valore di una coppia chiave-valore, puoi usare la sintassi nota come notazione a parentesi.

```python
dictionary[key]
```

## Metodi comuni dei dizionari

- **Metodo `get()`**: Il metodo `get()` recupera il valore associato a una chiave. È simile alla notazione a parentesi, ma ti permette di impostare un valore predefinito, evitando errori se la chiave non esiste.

```python
dictionary.get(key, default)
```

- **Metodi `keys()` e `values()`**: I metodi `keys()` e `values()` restituiscono un oggetto view con tutte le chiavi e i valori del dizionario, rispettivamente. Un oggetto view è un modo per vedere il contenuto di un dizionario senza creare una copia separata dei dati.

```python
pizza = {
    'name': 'Margherita Pizza',
    'price': 8.9,
    'calories_per_slice': 250
}

pizza.keys()
# dict_keys(['name', 'price', 'calories_per_slice'])

pizza.values()
# dict_values(['Margherita Pizza', 8.9, 250])
```

- **Metodo `items()`**: Il metodo `items()` restituisce un oggetto view con tutte le coppie chiave-valore del dizionario, incluse sia le chiavi che i valori.

```python
pizza.items()
# dict_items([('name', 'Margherita Pizza'), ('price', 8.9), ('calories_per_slice', 250)])
```

- **Metodo `clear()`**: Il metodo `clear()` rimuove tutte le coppie chiave-valore dal dizionario.

```python
pizza.clear()
```

- **Metodo `pop()`**: Il metodo `pop()` rimuove la coppia chiave-valore con la chiave specificata come primo argomento e ne restituisce il valore. Se la chiave non esiste, restituisce il valore predefinito specificato come secondo argomento. Se la chiave non esiste e il valore predefinito non è specificato, viene generato un `KeyError`.

```python
pizza.pop('price', 10)
pizza.pop('total_price') # KeyError
```

- **Metodo `popitem()`**: In Python 3.7 e versioni successive, il metodo `popitem()` rimuove l'ultimo elemento inserito.

```python
pizza.popitem()
```

- **Metodo `update()`**: Il metodo `update()` aggiorna le coppie chiave-valore con quelle di un altro dizionario. Se hanno chiavi in comune, i loro valori vengono sovrascritti. Le nuove chiavi verranno aggiunte come nuove coppie chiave-valore.

```python
pizza.update({ 'price': 15, 'total_time': 25 })
```

## Iterare su un dizionario

- **Iterare sui valori**: Se hai bisogno di iterare sui valori di un dizionario, puoi scrivere un loop `for` con `values()` per ottenere tutti i valori di un dizionario.

```python
products = {
    'Laptop': 990,
    'Smartphone': 600,
    'Tablet': 250,
    'Headphones': 70,
}

for price in products.values():
    print(price)
```

Output:

```md
990
600
250
70
```

- **Iterare sulle chiavi**: Se hai bisogno di iterare sulle chiavi del dizionario `products` sopra, puoi scrivere `products.keys()` o `products` direttamente.

```python
for product in products.keys():
    print(product)
    
# Or

for product in products:
    print(product)
```

Output:

```md
Laptop
Smartphone
Tablet
Headphones
```

- **Iterare sulle coppie chiave-valore**: Se hai bisogno di iterare sulle chiavi e sui rispettivi valori contemporaneamente, puoi iterare su `products.items()`. Ottieni singole tuple con le chiavi e i rispettivi valori.

```python
for product in products.items():
    print(product)
```

Output:

```md
('Laptop', 990)
('Smartphone', 600)
('Tablet', 250)
('Headphones', 70)
```

Per archiviare la chiave e il valore in variabili separate nel loop, devi separarle con una virgola. La prima variabile archivia la chiave, la seconda il valore.

```python
for product, price in products.items():
    print(product, price)
```

Output:

```md
Laptop 990
Smartphone 600
Tablet 250
Headphones 70
```

- **Funzione `enumerate()`**: Se hai bisogno di iterare su un dizionario tenendo traccia di un contatore, puoi chiamare la funzione `enumerate()`. La funzione restituisce un oggetto `enumerate`, che assegna un intero a ogni elemento, come un contatore. Puoi far partire il contatore da qualsiasi numero, ma per impostazione predefinita parte da 0.

Assegnare l'indice e l'elemento a variabili separate nel loop è il modo comune di usare `enumerate()`. Ad esempio, con `products.items()`, puoi ottenere l'intera coppia chiave-valore oltre all'indice:

```python
for index, product in enumerate(products.items()):
    print(index, product)
```

Output:

```md
0 ('Laptop', 990)
1 ('Smartphone', 600)
2 ('Tablet', 250)
3 ('Headphones', 70)
```

Per personalizzare il valore iniziale del conteggio, puoi passare un secondo argomento a `enumerate()`. Ad esempio, qui il conteggio parte da 1.

```python
for index, product in enumerate(products.items(), 1):
    print(index, product)
```

Output:

```md
1 ('Laptop', 990)
2 ('Smartphone', 600)
3 ('Tablet', 250)
4 ('Headphones', 70)
```

## Set

- **Set**: I set sono strutture dati integrate in Python che non consentono valori duplicati. I set sono mutabili e non ordinati, il che significa che i loro elementi non sono archiviati in un ordine specifico, quindi non puoi usare indici o chiavi per consultarli. Inoltre, i set possono contenere solo valori di tipi di dato immutabili, come numeri, stringhe e tuple.

- **Definire un set**: Per definire un set, devi scrivere i suoi elementi tra parentesi graffe e separarli con virgole.

```python
my_set = {1, 2, 3, 4, 5}
```

- **Definire un set vuoto**: Se hai bisogno di definire un set vuoto, devi usare la funzione `set()`. Scrivere solo parentesi graffe vuote creerà automaticamente un dizionario.

```python
set() # Set
{}    # Dictionary
```

## Metodi comuni dei set

- **Metodo `add()`**: Puoi aggiungere un elemento a un set con il metodo `add()`, passando il nuovo elemento come argomento.

```python
my_set.add(6)
```

- **Metodi `remove()` e `discard()`**: Per rimuovere un elemento da un set, puoi usare il metodo `remove()` o il metodo `discard()`, passando l'elemento che vuoi rimuovere come argomento. Il metodo `remove()` genererà un `KeyError` se l'elemento non viene trovato, mentre il metodo `discard()` no.

```python
my_set.remove(4)
my_set.discard(4)
```

- **Metodo `clear()`**: Il metodo `clear()` rimuove tutti gli elementi dal set.

```python
my_set.clear()
```

## Operazioni matematiche sui set

- **Metodi `issubset()` e `issuperset()`**: I metodi `issubset()` e `issuperset()` verificano se un set è un sottoinsieme o un sovrainsieme di un altro set, rispettivamente.

```python
my_set = {1, 2, 3, 4, 5}
your_set = {2, 3, 4, 5}

print(your_set.issubset(my_set)) # True
print(my_set.issuperset(your_set)) # True
```

- **Metodo `isdisjoint()`**: Il metodo `isdisjoint()` verifica se due set sono disgiunti, cioè se non hanno elementi in comune.

```python
my_set = {1, 2, 3}
your_set = {4, 5, 6}

print(my_set.isdisjoint(your_set)) # True
```

- **Operatore di unione (`|`)**: L'operatore di unione `|` restituisce un nuovo set con tutti gli elementi di entrambi i set.

```python
my_set = {1, 2, 3}
your_set = {4, 5, 6}

my_set | your_set # {1, 2, 3, 4, 5, 6}
```

- **Operatore di intersezione (`&`)**: L'operatore di intersezione `&` restituisce un nuovo set con solo gli elementi che i set hanno in comune.

```python
my_set = {1, 2, 3, 4, 5}
your_set = {2, 3, 4, 6}

my_set & your_set # {2, 3, 4}
```

- **Operatore di differenza (`-`)**: L'operatore di differenza `-` restituisce un nuovo set con gli elementi del primo set che non sono nell'altro set.

```python
my_set = {1, 2, 3, 4, 5}
your_set = {2, 3, 4, 6}

my_set - your_set # {1, 5}
```

- **Operatore di differenza simmetrica (`^`)**: L'operatore di differenza simmetrica `^` restituisce un nuovo set con gli elementi che sono o nel primo o nel secondo set, ma non in entrambi.

```python
my_set = {1, 2, 3, 4, 5}
your_set = {2, 3, 4, 6}

my_set ^ your_set # {1, 5, 6}
```

- **Operatore `in`**: Puoi verificare se un elemento è presente o meno in un set con l'operatore `in`.

```python
print(5 in my_set)
```

## Libreria standard di Python

- **Libreria standard di Python**: Una libreria ti offre codice già scritto e riutilizzabile, come funzioni, classi e strutture dati, che puoi riutilizzare nei tuoi progetti. Python ha un'ampia libreria standard con moduli integrati che implementano soluzioni standardizzate per molti problemi e attività. Alcuni esempi di moduli integrati popolari sono `math`, `random`, `re` (abbreviazione di "regular expressions") e `datetime`.

## Istruzione import

- **Istruzione import**: Per consultare gli elementi definiti nei moduli integrati, usi un'istruzione import. Le istruzioni import vengono generalmente scritte all'inizio del file. Le istruzioni import funzionano allo stesso modo per funzioni, classi, costanti, variabili e qualsiasi altro elemento definito nel modulo.

- **Istruzione import di base**: Puoi usare la parola chiave `import` seguita dal nome del modulo:

```python
import module_name
```

Poi, se hai bisogno di chiamare una funzione da quel modulo, userai la notazione a punto, con il nome del modulo seguito dal nome della funzione.

```python
module_name.function_name()
```

Ad esempio, scriveresti quanto segue nel tuo codice per importare il modulo `math` e ottenere la radice quadrata di 36:

```python
import math

math.sqrt(36)
```

- **Importare un modulo con un nome diverso**: Se hai bisogno di importare il modulo con un nome diverso (detto anche "alias"), puoi usare `as` seguito dall'alias alla fine dell'istruzione import. Questo viene spesso usato per nomi di moduli lunghi o per evitare conflitti di nomi.

```python
import module_name as module_alias
```

Ad esempio, per riferirti al modulo `math` come `m` nel tuo codice, puoi assegnare un alias in questo modo:

```python
import math as m
```

Poi puoi consultare gli elementi del modulo usando l'alias:

```python
m.sqrt(36)
```

- **Importare elementi specifici**: Se non hai bisogno di tutto da un modulo, puoi importare elementi specifici usando `from`. In questo caso, l'istruzione import inizia con `from`, seguita dal nome del modulo, poi dalla parola chiave `import` e infine dai nomi degli elementi che vuoi importare.

```python
from module_name import name1, name2
```

Poi puoi usare questi nomi senza il prefisso del modulo nel tuo script Python. Ad esempio:

```python
from math import radians, sin, cos

angle_degrees = 40
angle_radians = radians(angle_degrees)

sine_value = sin(angle_radians)
cos_value = cos(angle_radians)

print(sine_value) # 0.6427876096865393
print(cos_value)  # 0.766044443118978
```
 
Questo è utile, ma può causare conflitti di nomi se hai già funzioni o variabili con lo stesso nome. Tienilo a mente quando scegli quale tipo di istruzione import usare.

Se hai bisogno di assegnare alias a questi nomi, puoi farlo anche usando la parola chiave `as` seguita dall'alias.

```python
from module_name import name1 as alias1, name2 as alias2
```

- **Istruzione import con asterisco (`*`)**: L'asterisco dice a Python che vuoi importare tutto da quel modulo, ma vuoi importarlo in modo da non dover usare il nome del modulo come prefisso.

```python
from module_name import *
```

Ad esempio, se usi questo per importare il modulo `math`, potrai chiamare qualsiasi funzione definita in quel modulo senza specificare il nome del modulo come prefisso.

```python
from math import *
print(sqrt(36))  # 6.0
```

Tuttavia, questo è generalmente sconsigliato perché può portare a collisioni di nomi e rendere più difficile capire da dove provengono i nomi.

## `if __name__ == '__main__'`

Quando un file Python viene eseguito direttamente, Python imposta il valore di questa variabile sulla stringa `"__main__"`. Ma se il file Python viene importato come modulo in un altro script Python, il valore della variabile `__name__` viene impostato sul nome di quel modulo.

Ecco perché spesso troverai questa condizione negli script Python. Contiene il codice che vuoi eseguire **solo** se lo script Python viene eseguito come programma principale.

```python
if __name__ == '__main__': 
    # Code
```

## Errori comuni in Python

- **SyntaxError**: L'errore che Python solleva quando il tuo codice non segue le sue regole di sintassi. Ad esempio, il codice `print("Hello there"` genererà un errore di sintassi con il messaggio `SyntaxError: '(' was never closed`, perché manca una parentesi di chiusura.
- **NameError**: Python solleva un `NameError` quando provi a consultare una variabile o funzione che non hai definito. Ad esempio, se hai la riga `print(username)` nel tuo codice senza aver prima definito la variabile `username`, otterrai un errore di nome con il messaggio `NameError: name 'username' is not defined`.
- **TypeError**: Questo è l'errore che Python genera quando esegui un'operazione su due o più tipi di dato incompatibili. Ad esempio, se provi ad aggiungere una stringa a un numero, otterrai l'errore `TypeError: can only concatenate str (not "int") to str`.
- **IndexError**: Otterrai un `IndexError` se consulti un indice che non esiste in una lista o altre sequenze come tuple e stringhe. Ad esempio, in una stringa `Hello world`, l'indice dell'ultimo carattere è `10`. Se consulti un carattere in questo modo, `greet = "hello world"; print(greet[11])`, otterrai un errore con il messaggio `IndexError: string index out of range`.
- **AttributeError**: Python solleva questo errore quando provi a usare un metodo o una proprietà che non esiste in un oggetto di quel tipo. Ad esempio, chiamare `.append()` su una stringa come `"hello".append("!")` genererà un errore con il messaggio `AttributeError: 'str' object has no attribute 'append'`.

## Gestione delle eccezioni

- **`try...except`**: Viene usato per eseguire un blocco di codice che potrebbe sollevare un'eccezione. Il blocco `try` è dove prevedi che possa verificarsi un errore, mentre il blocco `except` prende un'eccezione specificata e viene eseguito se quell'errore viene sollevato. Ecco un esempio:

  ```py
  try:
    print(22 / 0)
  except ZeroDivisionError:
    print('You can\'t divide by zero!')
    # You can't divide by zero!
  ```

  Puoi anche concatenare più blocchi `except` per gestire più tipi di eccezioni:

  ```py
  try:
    number = int(input('Enter a number: '))
    print(22 / number)
  except ZeroDivisionError:
    print('You cannot divide by zero!')
    # You cannot divide by zero! prints when you enter 0
  except ValueError:
    print('Please enter a valid number!')
    # Please enter a valid number! prints when you don't enter an integer
  ```

- **`else` e `finally`**: Questi blocchi estendono `try...except`. Se non si verifica nessuna eccezione, viene eseguito il blocco `else`. Il blocco `finally` viene sempre eseguito indipendentemente dagli errori.

  ```py
  try:
    result = 100 / 4
  except ZeroDivisionError:
    print('You cannot divide by zero!') # This will not run
  else:
    print(f'Result is {result}') # Result is 25.0
  finally:
    print('Execution complete!') # Execution complete!
  ```

- **Oggetto eccezione**: Ti permette di consultare direttamente l'eccezione per un debugging migliore e stampare il messaggio di errore diretto. Per consultare l'oggetto eccezione, devi usare la parola chiave `as`. Ecco un esempio:

  ```py
  try:
      value = int('This will raise an error')
  except ValueError as e:
      print(f'Caught an error: {e}')
      # Caught an error: invalid literal for int() with base 10: 'This will raise an error'
  ```

- **L'istruzione `raise`**: Ti permette di sollevare manualmente un'eccezione. Puoi usarla per lanciare un'eccezione quando viene soddisfatta una certa condizione. Ecco un esempio:

  ```py
  def divide(a, b):
      if b == 0:
          raise ZeroDivisionError('You cannot divide by zero')
      return a / b
  ```
  
## Segnalazione delle eccezioni
  
  L'istruzione `raise` è utile anche quando crei le tue eccezioni personalizzate, perché puoi usarla per lanciare un'eccezione con un messaggio personalizzato. Ecco un esempio:

  ```py
  class InvalidCredentialsError(Exception):
      def __init__(self, message="Invalid username or password"):
          self.message = message
          super().__init__(self.message)

  def login(username, password):
      stored_username = "admin"
      stored_password = "password123"
      
      if username != stored_username or password != stored_password:
          raise InvalidCredentialsError()
      
      return f"Welcome, {username}!"
  ```  

  Ecco come puoi usare la funzione `login` con l'eccezione `InvalidCredentialsError`:

  ```py
  # failed login attempt
  try:
      message = login("user", "wrongpassword")
  except InvalidCredentialsError as e:
      print(f"Login failed: {e}")
  else:
      print(message)

  # successful login attempt
  try:
      message = login("admin", "password123")
  except InvalidCredentialsError as e:
      # This block is not executed because the login was successful
      print(f"Login failed: {e}")
  else:
      # The else block runs if the 'try' block completes without an exception
      print(message)
  ```

  L'istruzione `raise` può essere usata anche con la parola chiave `from` per concatenare eccezioni, mostrando la relazione tra errori diversi:

  ```py
  def parse_config(filename):
    try:
        with open(filename, 'r') as file:
            data = file.read()
            return int(data)
    except FileNotFoundError:
        raise ValueError('Configuration file is missing') from None
    except ValueError as e:
        raise ValueError('Invalid configuration format') from e

  config = parse_config('config.txt')
```

## Classi e oggetti in Python

- **Definizione di classe**: Una classe è un modello per creare oggetti. Definisce il comportamento che un oggetto avrà attraverso i suoi attributi e metodi. Ecco un esempio base di definizione di una classe in Python:

```python
class Dog:
    def __init__(self, name, age):
        self.name = name
        self.age = age

    def bark(self):
        print(f'{self.name.upper()} says woof woof!')
```

- **Creare oggetti**: Gli oggetti sono istanze di una classe. Vengono creati chiamando la classe con gli argomenti necessari.

```python
dog1 = Dog('Jack', 3)
dog2 = Dog('Thatcher', 5)

dog1.bark()  # JACK says woof woof!
dog2.bark()  # THATCHER says woof woof!
```

- **Chiamare metodi sugli oggetti**: Puoi chiamare metodi sugli oggetti per eseguire azioni o recuperare informazioni.

```python
object_name1.method_name()
object_name2.method_name()
```

- **Differenza tra classe e oggetto**: Una classe è un modello riutilizzabile, mentre un oggetto è un'istanza specifica di quella classe con dati reali.

## Attributi

- **Attributi di istanza**: Definiti in `__init__()` usando `self`, e unici per ogni oggetto.
- **Attributi di classe**: Definiti direttamente all'interno della classe e condivisi da tutte le istanze.

```python
class Dog:
    species = 'French Bulldog'  # Class attribute

    def __init__(self, name):
        self.name = name  # Instance attribute

print(Dog.species) # French Bulldog

jack = Dog('Jack')
print(jack.name)     # Jack
print(jack.species)  # French Bulldog
```

## Metodi

- **Metodi**: Funzioni definite all'interno di una classe che operano sugli attributi dell'oggetto.

```python
class Car:
    def __init__(self, color, model):
        self.color = color
        self.model = model

    def describe(self):
        return f'This car is a {self.color} {self.model}'

my_car_1 = Car('red', 'Tesla Model S')
print(my_car_1.describe())  # This car is a red Tesla Model S
```

- **Consultare i metodi**: Chiama i metodi sugli oggetti usando la notazione a punto. Ecco un esempio di chiamata del metodo `describe` su due diversi oggetti auto:

```python
class Car:
    def __init__(self, color, model):
        self.color = color  
        self.model = model  

    def describe(self):
        return f'This car is a {self.color} {self.model}'

my_car_1 = Car('red', 'Tesla Model S')
my_car_2 = Car('green', 'Lamborghini Revuelto')

print(my_car_1.describe()) # Calling method using the dot notation

print(my_car_2.describe()) # Calling method using the dot notation
```

## Metodi dunder (magici)

- **Definizione**: Metodi speciali che iniziano e finiscono con un doppio underscore (ad esempio `__init__`, `__len__`, `__str__`, `__eq__`). Python li usa internamente per operazioni integrate.

```python
class Book:
    def __init__(self, title, pages):
        self.title = title
        self.pages = pages

    def __len__(self):
        return self.pages

    def __str__(self):
        return f"'{self.title}' has {self.pages} pages"

    def __eq__(self, other):
        return self.pages == other.pages

book1 = Book('Built Wealth Like a Boss', 420)
print(len(book1))        # 420
print(str(book1))        # 'Built Wealth Like a Boss' has 420 pages
```

- **Chiamare i metodi dunder indirettamente**: Non è necessario chiamare direttamente i metodi dunder. Python li chiama automaticamente quando avvengono certe azioni. Queste operazioni includono:

  - **operazioni aritmetiche come addizione, sottrazione, moltiplicazione, divisione e altre**. `__add__()` viene chiamato per l'addizione, `__sub__()` per la sottrazione, `__mul__()` per la moltiplicazione e `__truediv__()` per la divisione.
        
  - **operazioni sulle stringhe come concatenazione, ripetizione, formattazione e conversione in testo**. `__add__()` viene chiamato per la concatenazione, `__mul__()` per la ripetizione, `__format__()` per la formattazione, `__str__()` e `__repr__()` per la conversione in testo, e così via.
        
  - **operazioni di confronto come uguaglianza, minore di, maggiore di e altre**. `__eq__()` viene chiamato per i controlli di uguaglianza, `__lt__()` per minore di, `__gt__()` per maggiore di, e così via.
        
  - **operazioni di iterazione come rendere un oggetto iterabile e avanzare tra gli elementi**. `__iter__()` viene chiamato per restituire un iteratore e `__next__()` per ottenere l'elemento successivo.

## Esempio reale: carrello della spesa

- **Classe Cart con metodi dunder**: Permette di aggiungere, rimuovere, iterare e controllare il contenuto con comportamento integrato.

```python
class Cart:
    def __init__(self):
        self.items = []

    def add(self, item):
        self.items.append(item)

    def remove(self, item):
        if item in self.items:
            self.items.remove(item)
        else:
            print(f'{item} is not in cart')

    def list_items(self):
        return self.items

    def __len__(self):
        return len(self.items)

    def __getitem__(self, index):
        return self.items[index]

    def __contains__(self, item):
        return item in self.items

    def __iter__(self):
        return iter(self.items)

cart = Cart()
cart.add('Laptop')
print(len(cart))        # 1
print('Laptop' in cart) # True
```

## Cos’è la programmazione orientata agli oggetti?

- **Programmazione orientata agli oggetti**: Uno stile di programmazione in cui gli sviluppatori trattano tutto nel codice come un oggetto del mondo reale. È comunemente chiamata OOP. I quattro principi chiave che ti aiutano a organizzare e gestire il codice efficacemente sono **incapsulamento**, **ereditarietà**, **polimorfismo** e **astrazione**.
- **Classi**: Il modello per creare oggetti. Ogni singolo oggetto creato da una classe ha attributi che definiscono i dati e metodi che determinano i comportamenti dell’oggetto.

## Cos’è l’incapsulamento?

- **Incapsulamento**: Il raggruppamento degli attributi e dei metodi di un oggetto in un’unità singola. Ti permette di nascondere lo stato interno dell’oggetto dietro un semplice insieme di metodi e attributi pubblici che agiscono come porte. Dietro quelle porte ci sono attributi e metodi privati che controllano come i dati cambiano e chi può vederli.
- **Esempio di incapsulamento**: Se vuoi tenere traccia del saldo di un portafoglio, permetterai depositi e prelievi, ma non vorrai che qualcuno manometta direttamente il saldo del portafoglio:

```py
class Wallet:
   def __init__(self, balance):
       self.__balance = balance # Private attribute

   def deposit(self, amount):
       if amount > 0:
           self.__balance += amount # Add to the balance safely

   def withdraw(self, amount):
       if 0 < amount <= self.__balance:
           self.__balance -= amount # Remove from the balance safely

account = Wallet(500)
print(account.__balance) # AttributeError: 'Wallet' object has no attribute '__balance'
```

- **Differenza tra prefissare attributi con singolo o doppio underscore**: Prefissare attributi e metodi con un singolo underscore significa che sono destinati all’uso interno. Questa è una convenzione e non impedisce di consultare gli attributi dall’esterno. Prefissare attributi e metodi con un doppio underscore impedisce effettivamente che vengano consultati dall’esterno della loro classe.

## Cosa sono getter e setter?

- **Getter e setter**: Metodi che ti permettono di controllare come gli attributi di una classe vengono consultati e modificati. Recuperi i valori con i getter e li imposti con i setter.
- **Proprietà**: Collegano getter e setter e permettono di consultare i dati. Eseguono logica extra dietro le quinte quando ottieni, imposti o elimini valori.
- **Perché usare proprietà invece di metodi**: Le proprietà si usano invece dei metodi per una migliore leggibilità e un codice più pulito. Permettono di consultare i valori con la notazione a punto, come normali attributi, senza parentesi.
- **Creare un getter**: Per creare un getter, usi il decoratore `@property`. Ecco un getter che ottiene il raggio di un cerchio:

```py
class Circle:
    def __init__(self, radius):
        self._radius = radius

    @property
    def radius(self): # A getter to get the radius
        return self._radius
  
    @property
    def area(self):  # A getter to calculate area
        return 3.14 * (self._radius ** 2)

my_circle = Circle(3)

print(my_circle.radius) # 3
print(my_circle.area) # 28.26
```

- **Creare un setter**: Per creare il setter che imposta il raggio, devi definire un altro metodo con lo stesso nome e usare `@<property_name>.setter` sopra di esso:

```py
class Circle:
    def __init__(self, radius):
        self._radius = radius

    @property
    def radius(self):  # A getter to get the radius
        return self._radius

    @radius.setter
    def radius(self, value):  # A setter to set the radius
        if value <= 0:
            raise ValueError('Radius must be positive')
        self._radius = value

my_circle = Circle(3)
print('Initial radius:', my_circle.radius) # Initial radius: 3

my_circle.radius = 8
print('After modifying the radius:', my_circle.radius) # After modifying the radius: 8
```

- **Come Python gestisce getter e setter**: Una volta definiti getter e setter, Python li chiama automaticamente sotto il cofano ogni volta che usi la sintassi normale degli attributi in questo modo:

```py
my_circle.radius # This will call the getter
my_circle.radius = 4 # This will call the setter
```

 Quando imposti un valore, non dovresti assegnare direttamente al nome della proprietà perché questo causerebbe un `RecursionError`. Usa un nome interno separato, spesso con un underscore, per archiviare il valore.

- **Deleter**: Dopo aver impostato e ottenuto un valore con setter e getter, puoi controllare come viene eliminato con un `deleter`. Un deleter esegue logica personalizzata quando usi l’istruzione `del` su una proprietà. Per creare un deleter, usi il decoratore `@<property_name>.deleter`.

```py
  # Deleter
    @radius.deleter
    def radius(self):
        print("Deleting radius...")
        del self._radius
```

## Cos’è l’ereditarietà e come favorisce il riuso del codice?

- **Ereditarietà**: Il processo con cui una classe figlia usa gli attributi e i metodi di una classe genitore. L’ereditarietà favorisce il riuso del codice, fornisce gerarchie chiare e personalizza il comportamento senza riscrivere tutto. Per implementare l’ereditarietà, una classe figlia prende il nome di una classe genitore:

```py
class Parent:
    # Parent attributes and methods

class Child(Parent):
    # Child inherits, extends, and/or overrides where necessary
```

- **Ereditarietà singola e multipla**: Quando una classe figlia eredita proprietà e metodi da un solo genitore, come vedi sopra, il processo si chiama **ereditarietà singola**. Quando una classe figlia eredita proprietà e metodi da più genitori, si chiama **ereditarietà multipla**. Ecco la sintassi per questo:

```py
class Parent:
    # Attributes and methods for Parent

class Child:
    # Attributes and methods for Child

class GrandChild(Parent, Child):
    # GrandChild inherits from both Parent and Child
    # GrandChild can combine or override behavior from each
```

- **Funzione `super()`**: Una funzione che ti permette di **chiamare** un metodo da una classe genitore, quando una classe ha un’implementazione diversa di quel metodo o lo estende, senza duplicare codice.

## Cos’è il polimorfismo e come favorisce il riuso del codice?

- **Polimorfismo**: Il principio OOP che permette a classi diverse di usare lo stesso nome di metodo, ma ogni classe lo implementa in modo diverso quando viene chiamato. Ecco la sintassi per questo:

```py
class A:
   def action(self): ...

class B:
   def action(self): ...

class C:
   def action(self): ...

Class().method()  # Works for A, B, or C
```

- **Polimorfismo basato sull’ereditarietà**: Un genitore definisce un metodo, e ogni classe figlia lo adatta al proprio uso.

## Cos’è il name mangling e come funziona?

- **Name mangling**: Un processo in cui Python rinomina internamente un attributo prefissato con doppio underscore aggiungendo un underscore e il nome della classe come prefisso, trasformando `__attribute` in `_ClassName__attribute`.
- **Scopo del name mangling**: Lo scopo principale del name mangling è prevenire la sovrascrittura accidentale di attributi e metodi quando usi l’ereditarietà. Ecco un codice che rende questo più comprensibile:

```py
class Parent:
    def __init__(self):
        self.__data = 'Parent data'

class Child(Parent):
    def __init__(self):
        super().__init__()
        self.__data = 'Child data'

c = Child()
print(c.__dict__) # {'_Parent__data': 'Parent data', '_Child__data': 'Child data'}
```

## Cos’è l’astrazione e come aiuta a mantenere organizzati sistemi complessi?

- **Astrazione**: Un concetto di programmazione in cui i dettagli complessi di implementazione di un oggetto o sistema sono nascosti e vengono mostrati solo gli aspetti essenziali. In Python e altri linguaggi di programmazione, l’astrazione semplifica sistemi complessi aumentando il riuso.
- **Esempio di astrazione**: Un buon esempio di astrazione nella vita quotidiana è un’auto che ti permette di usare solo il volante, i pedali e il cambio senza sapere come funzionano il motore o i freni.
- **Come Python implementa l’astrazione**: Python implementa l’astrazione tramite il modulo `abc`. Il modulo fornisce la classe `ABC` (classe base astratta) e il decoratore `@abstractmethod`. Una classe base astratta (ABC) definisce i metodi e le proprietà comuni che le sottoclassi devono implementare. Non può essere istanziata.
- **Come si definisce un metodo astratto**: Un metodo astratto si definisce con `@abstractmethod` e deve essere sovrascritto nelle sottoclassi, anche se ha un’implementazione predefinita. La sintassi base dell’astrazione è questa:

```py
from abc import ABC, abstractmethod

# Define an abstract base class
class AbstractClass(ABC):
    @abstractmethod
    def abstract_method(self):
        pass

# Concrete subclass that implements the abstract method
class ConcreteClassOne(AbstractClass):
    def abstract_method(self):
        print('Implementation in ConcreteClassOne')

# Another concrete subclass
class ConcreteClassTwo(AbstractClass):
    def abstract_method(self):
        print('Implementation in ConcreteClassTwo')
```

## Algoritmi e notazione Big O

- **Algoritmi**: Un insieme di istruzioni non ambigue per risolvere un problema o svolgere un’attività. Gli algoritmi devono terminare in un numero finito di passaggi e ogni passaggio deve essere preciso e non ambiguo.

- **Notazione Big O**: Descrive la prestazione nel caso peggiore, o il tasso di crescita, di un algoritmo all’aumentare della dimensione dell’input. Si concentra su come l’uso delle risorse cresce con la dimensione dell’input, ignorando fattori costanti e termini di ordine inferiore.

### Complessità temporali comuni

- **O(1) - Tempo costante**: L’algoritmo impiega lo stesso tempo indipendentemente dalla dimensione dell’input.

```python
def check_even_or_odd(number):
    if number % 2 == 0:
        return 'Even'
    else:
        return 'Odd'
```

- **O(log n) - Tempo logaritmico**: Il tempo aumenta lentamente all’aumentare dell’input. Comune negli algoritmi che riducono ripetutamente la dimensione del problema di una frazione (come la ricerca binaria).

- **O(n) - Tempo lineare**: Il tempo di esecuzione cresce proporzionalmente alla dimensione dell’input.

```python
for grade in grades:
    print(grade)
```

- **O(n log n) - Tempo log-lineare**: Complessità temporale comune di algoritmi di ordinamento efficienti come merge sort e quick sort.

- **O(n²) - Tempo quadratico**: Il tempo di esecuzione cresce quadraticamente. Spesso visto in loop annidati.

```python
for i in range(n):
    for j in range(n):
        print("Hello, World!")
```

### Complessità spaziale

- **O(1) - Spazio costante**: L’algoritmo usa la stessa quantità di memoria indipendentemente dalla dimensione dell’input.
- **O(n) - Spazio lineare**: L’uso della memoria cresce proporzionalmente alla dimensione dell’input.
- **O(n²) - Spazio quadratico**: L’uso della memoria cresce quadraticamente con la dimensione dell’input.

## Tecniche di risoluzione dei problemi

- **Comprendere il problema**: Leggi più volte l’enunciato del problema. Identifica l’input, l’output atteso e come trasformare l’input in output.

- **Pseudocodice**: Descrizione ad alto livello della logica dell’algoritmo indipendente dal linguaggio. Usa un linguaggio scritto comune mescolato a costrutti di programmazione come `IF`, `ELSE`, `FOR`, `WHILE`.

```md
GET original_string
SET reversed_string = ""
FOR EACH character IN original_string:
  ADD character TO THE BEGINNING OF reversed_string
DISPLAY reversed_string
```

- **Casi limite**: Input specifici e validi che si verificano ai confini di ciò che un algoritmo dovrebbe gestire. Considera e testa sempre i casi limite.

## Array

- **Array statici**: Hanno una dimensione fissa determinata all’inizializzazione. Gli elementi sono archiviati in posizioni di memoria adiacenti. La dimensione non può essere cambiata durante l’esecuzione del programma.

- **Array dinamici**: Possono crescere o ridursi automaticamente durante l’esecuzione del programma. Gestiscono il ridimensionamento tramite copia automatica in array più grandi quando necessario.

### Liste Python (array dinamici)

```python
numbers = [3, 4, 5, 6]

# Access elements
numbers[0]  # 3

# Update elements
numbers[2] = 16

# Add elements
numbers.append(7)
numbers.insert(3, 15)  # Insert at specific index

# Remove elements
numbers.pop(2)  # Remove at specific index
numbers.pop()   # Remove last element
```

### Complessità temporali per array dinamici

- **Consultare**: O(1)
- **Inserire alla fine**: O(1) in media, O(n) quando serve ridimensionare
- **Inserire in mezzo**: O(n)
- **Eliminare**: O(n) per il mezzo, O(1) per la fine

## Stack

- **Stack**: Struttura dati Last-In, First-Out (LIFO). Gli elementi si aggiungono e rimuovono solo dalla cima.

- **Operazione push**: Aggiungere un elemento in cima allo stack. Complessità temporale: O(1).

- **Operazione pop**: Rimuovere un elemento dalla cima dello stack. Complessità temporale: O(1).

```python
# Using Python list as stack
stack = []

# Push operations
stack.append(1)
stack.append(2)
stack.append(3)

# Pop operations
top_element = stack.pop()  # Returns 3
```

## Code

- **Code**: Struttura dati First-In, First-Out (FIFO). Gli elementi si aggiungono in coda e si rimuovono dalla testa.

- **Operazione enqueue**: Aggiungere un elemento in coda alla coda. Complessità temporale: O(1).

- **Operazione dequeue**: Rimuovere un elemento dalla testa della coda. Complessità temporale: O(1).

```python
from collections import deque

# Using deque for efficient queue operations
queue = deque()

# Enqueue operations
queue.append(1)
queue.append(2)
queue.append(3)

# Dequeue operations
first_element = queue.popleft()  # Returns 1
```

## Liste concatenate

- **Liste concatenate**: Struttura dati lineare in cui ogni nodo contiene dati e un riferimento al nodo successivo. I nodi sono collegati come una catena.

### Liste concatenate singole

- **Struttura**: Ogni nodo ha dati e un riferimento al nodo successivo.
- **Attraversamento**: Puoi muoverti solo in avanti dalla testa alla coda.
- **Nodo testa**: Primo nodo della lista, solitamente l’unico nodo accessibile direttamente.
- **Nodo coda**: Ultimo nodo della lista, punta a `None`.

### Operazioni e complessità temporali

- **Inserire all’inizio**: O(1)
- **Inserire alla fine**: O(n) - bisogna attraversare fino alla fine
- **Inserire in mezzo**: O(n) - bisogna attraversare fino alla posizione
- **Eliminare dall’inizio**: O(1)
- **Eliminare dalla fine**: O(n) - bisogna attraversare per trovare il nodo precedente
- **Eliminare dal mezzo**: O(n) - bisogna attraversare per trovare il nodo

### Liste concatenate doppie

- **Struttura**: Ogni nodo ha dati e due riferimenti: al nodo successivo e al nodo precedente.
- **Attraversamento**: Puoi muoverti in entrambe le direzioni.
- **Memoria**: Richiede più memoria rispetto alle liste concatenate singole a causa del riferimento extra.

## Mappe hash e insiemi

### Mappe e mappe hash

- **Mappa (tipo astratto di dato)**: Gestisce collezioni di coppie chiave-valore. Ogni chiave deve essere unica, ma i valori possono ripetersi.

- **Mappa hash**: Implementazione concreta della mappa ADT usando la tecnica di hashing. Usa una funzione hash per generare valori hash per le chiavi, che determinano la posizione di archiviazione nell’array sottostante.

### Dizionari Python (mappe hash)

```python
# Creating dictionaries
my_dictionary = {
    "A": 1,
    "B": 2, 
    "C": 3
}

# Alternative creation
my_dictionary = dict(A=1, B=2, C=3)

# Access and modify
value = my_dictionary["A"]  # 1
my_dictionary["A"] = 4      # Update value
del my_dictionary["A"]      # Remove key-value pair

# Check membership
"C" in my_dictionary

# Get keys, values, items
my_dictionary.keys()
my_dictionary.values()
my_dictionary.items()
```

### Complessità temporali per mappe hash

- **Caso medio**: O(1) per inserimento, consultazione, eliminazione
- **Caso peggiore**: O(n) quando si verificano molte collisioni hash

### Insiemi

- **Insiemi**: Collezioni non ordinate di elementi unici. Non sono ammessi duplicati, non viene mantenuto un ordine specifico.

- **Solo elementi immutabili**: Gli insiemi possono contenere solo tipi di dati immutabili (numeri, stringhe, tuple) perché i valori hash devono rimanere costanti.

```python
# Creating sets
numbers = {1, 2, 3, 4}
empty_set = set()  # Must use set(), not {}

# Add and remove elements
numbers.add(5)
numbers.remove(4)      # Raises KeyError if not found
numbers.discard(4)     # No error if not found

# Set operations
set_a = {1, 2, 3, 4}
set_b = {2, 3, 4, 5, 6}

# Union, intersection, difference, symmetric difference
set_a.union(set_b)                    # or set_a | set_b
set_a.intersection(set_b)             # or set_a & set_b
set_a.difference(set_b)               # or set_a - set_b
set_a.symmetric_difference(set_b)     # or set_a ^ set_b

# Subset and superset checks
set_a.issubset(set_b)
set_a.issuperset(set_b)
set_a.isdisjoint(set_b)

# Membership testing
5 in numbers
```

### Complessità temporali per insiemi

- **Caso medio**: O(1) per aggiungere, rimuovere, testare appartenenza
- **Caso peggiore**: O(n) a causa di collisioni hash

## Collisioni hash

- **Collisione hash**: Si verifica quando due chiavi diverse producono lo stesso valore hash.

- **Strategie di risoluzione delle collisioni**:
  - **Concatenamento**: Ogni indice dell’array punta a una lista concatenata che memorizza tutti gli elementi con lo stesso valore hash
  - **Indirizzamento aperto**: Cerca il prossimo indice disponibile usando una sequenza predefinita

## Quando usare ogni struttura dati

- **Liste**: Quando serve accesso ordinato e indicizzato e non si conosce la dimensione in anticipo
- **Stack**: Per operazioni LIFO (funzionalità annulla, valutazione espressioni, backtracking)
- **Code**: Per operazioni FIFO (pianificazione attività, ricerca in ampiezza)
- **Liste concatenate**: Quando servono inserimenti/eliminazioni frequenti all’inizio, dimensione sconosciuta, nessun accesso casuale necessario
- **Mappe hash**: Per ricerche rapide chiave-valore, conteggio occorrenze, caching
- **Insiemi**: Per controllare unicità, operazioni matematiche su insiemi, rimuovere duplicati

## Algoritmi di ricerca

Gli algoritmi di ricerca ti permettono di cercare un obiettivo all’interno di una certa lista di elementi.

In informatica, ci sono due algoritmi di ricerca che dovresti conoscere. Sono gli algoritmi di **ricerca lineare** e **ricerca binaria**. È importante capire le differenze tra i due algoritmi e quando usare ciascuno.

### Ricerca lineare

- La ricerca lineare itera su una lista di elementi, controllando ogni elemento dall’inizio finché non trova l’elemento obiettivo.
- Se l’elemento obiettivo viene trovato, viene restituito l’indice in cui si trova nella lista.
- Se l’obiettivo non viene trovato, restituisce `-1`, che significa **indice non valido** nella maggior parte dei linguaggi di programmazione.
- Poiché la ricerca lineare controlla ogni elemento finché trova l’obiettivo, non è efficiente per una lista grande di elementi.
- La complessità temporale della ricerca lineare è `O(n)` perché il tempo necessario per cercare nella lista cresce linearmente con la dimensione della lista.
- La complessità spaziale della ricerca lineare è `O(1)` perché non richiede spazio aggiuntivo per cercare nella lista.

### Ricerca binaria

- La ricerca binaria funziona dividendo una lista di elementi a metà e controllando se il valore obiettivo è nel mezzo della lista.
- La condizione perché la ricerca binaria funzioni è che gli elementi nella lista devono essere in ordine crescente.
- La ricerca binaria è un algoritmo più efficiente per cercare in una lista grande di elementi perché divide la lista a metà e ignora qualsiasi metà in cui l’obiettivo non si trova.
- Se l’elemento obiettivo viene trovato nel mezzo della lista, viene restituito l’indice dell’elemento obiettivo.
- Se l’elemento non viene trovato, l’algoritmo controlla se l’elemento obiettivo è nella metà sinistra o destra della lista.
- Continua a dividere le parti rimanenti della lista in metà finché l’elemento obiettivo non viene trovato.
- Se l’elemento obiettivo alla fine non viene trovato nella lista, restituisce `-1`.
- La complessità temporale della ricerca binaria è `O(log n)` perché il tempo necessario per cercare nella lista cresce logaritmicamente con la dimensione della lista.
- La complessità spaziale della ricerca binaria è `O(1)` perché non richiede spazio aggiuntivo per cercare nella lista.

### Come la ricerca lineare differisce dalla ricerca binaria

- La ricerca binaria è più adatta per una lista grande di elementi rispetto alla ricerca lineare.
- La complessità temporale della ricerca lineare è `O(n)` perché il tempo necessario per cercare nella lista cresce linearmente con la dimensione della lista.
- La complessità temporale della ricerca binaria è `O(log n)` perché il tempo necessario per cercare nella lista cresce logaritmicamente con la dimensione della lista.

## Algoritmi di ordinamento e divide-et-impera

In informatica, divide-et-impera è una tecnica usata per suddividere un problema in sotto-problemi più piccoli così da renderli più facili da risolvere. La ricorsione è la tecnica spesso impiegata in divide-et-impera, ed è una strategia potente usata per implementare molti algoritmi di ordinamento efficienti come merge sort.

### Merge sort

- Merge sort è un algoritmo di ordinamento che segue l’approccio divide-et-impera.
- Funziona dividendo ricorsivamente una lista in sotto-liste più piccole finché ogni sotto-lista contiene un solo elemento.
- Poi unisce ripetutamente le sotto-liste insieme in ordine ordinato.
- La complessità temporale di merge sort è `O(n log n)` perché la lista viene continuamente divisa a metà `(log n)` e poi unita insieme `(O(n))`.
- La complessità spaziale di merge sort è `O(n)` perché non è un algoritmo di ordinamento in-place.

## Panoramica sui grafi

Un grafo è un insieme di nodi (vertici) collegati da archi (connessioni). Ogni nodo può collegarsi a più altri nodi, formando una rete. I diversi tipi di grafi includono:

- Diretti: gli archi hanno una direzione (da un nodo a un altro), spesso rappresentati con linee rette e frecce.
- Indiretti: gli archi non hanno direzione, rappresentati con linee semplici.
- Vertice: ogni nodo è associato a un’etichetta o identificatore.
- Ciclici: contengono cicli (un percorso che inizia e finisce nello stesso nodo).
- Aciclici (DAG): non contengono cicli.
- Archi etichettati: ogni arco ha un’etichetta solitamente disegnata accanto all’arco corrispondente.
- Pesati: gli archi hanno pesi (valori) associati, che possono essere usati per operazioni aritmetiche.
- Disconnessi: contengono due o più nodi che non sono collegati da alcun arco.

I grafi sono usati in varie applicazioni come mappe, reti, sistemi di raccomandazione, risoluzione di dipendenze.

## Attraversamenti di grafi

Questo consiste nel visitare tutti i nodi di un grafo. I due algoritmi principali sono:

- **Ricerca in ampiezza (BFS)**
  - Usa una coda.
  - Esplora livello per livello.
  - Trova il percorso più breve in grafi non pesati.

- **Ricerca in profondità (DFS)**
  - Usa uno stack (o ricorsione).
  - Esplora completamente un ramo prima di tornare indietro.
  - Utile per rilevare cicli e trovare percorsi.

## Rappresentazioni dei grafi

I grafi possono essere rappresentati in due modi principali:

- **Lista di adiacenza**
  - Ogni nodo ha una lista dei suoi vicini.
  - Spazio efficiente per grafi sparsi.
  - Facile iterare sui vicini.

- **Matrice di adiacenza**
  - Un array 2D dove righe e colonne rappresentano i nodi.
  - Spazio intensivo per grafi grandi.
  - Veloce per controllare se esiste un arco tra due nodi.

## Alberi

Un albero è un tipo speciale di grafo che è aciclico e connesso. Le proprietà chiave includono:

- Non hanno loop o cicli (percorsi in cui i nodi di partenza e arrivo sono gli stessi).
- Devono essere connessi (ogni nodo può essere raggiunto da ogni altro nodo).

### Tipi comuni di alberi

I tipi più comuni di alberi sono:

- Alberi binari
  - Ogni nodo ha al massimo due figli, un figlio sinistro e un figlio destro.

- Alberi binari di ricerca (BST)
  - Un albero binario in cui ogni figlio sinistro è minore del genitore, e ogni figlio destro è maggiore del genitore.

## Trie

Conosciuti anche come alberi prefisso, sono usati per archiviare insiemi di stringhe, dove ogni nodo rappresenta un carattere.

I prefissi condivisi sono archiviati una sola volta, rendendoli efficienti per attività come completamento automatico e controllo ortografico.

Le operazioni di ricerca e inserimento hanno complessità temporale O(L), dove L è la lunghezza della stringa.

## Code di priorità

Una coda di priorità è un tipo astratto di dato in cui ogni elemento ha una priorità.

Code e stack considerano solo l’ordine di inserimento, mentre le code di priorità considerano la priorità degli elementi.

Le code standard seguono FIFO (First In First Out) e gli stack seguono LIFO (Last In First Out). Tuttavia, in una coda di priorità, gli elementi con priorità più alta vengono serviti prima di quelli con priorità più bassa, indipendentemente dall’ordine di inserimento.

## Heap

È una struttura dati ad albero specializzata con una proprietà molto specifica chiamata proprietà di heap.

La proprietà di heap determina la relazione tra nodi genitori e figli. Ci sono due tipi di heap:

- Max-Heap
  - Il valore di ogni nodo genitore è maggiore o uguale ai valori dei suoi figli.
  - L’elemento più grande è alla radice.

- Min-Heap
  - Il valore di ogni nodo genitore è minore o uguale ai valori dei suoi figli.
  - L’elemento più piccolo è alla radice.

### Esempio del modulo `heapq` in Python

```py
import heapq

# Create empty heap
my_heap = []

# Insert elements
heapq.heappush(my_heap, 9)
heapq.heappush(my_heap, 3)
heapq.heappush(my_heap, 5)
print(my_heap) # [3, 9, 5]

# Remove smallest element
print(heapq.heappop(my_heap))  # 3
print(my_heap) # [5, 9]

# Push + Pop in one step
print(heapq.heappushpop(my_heap, 7)) # 5
print(my_heap) # [7, 9]

# Transform list into heap
nums = [5, 7, 3, 1]
heapq.heapify(nums)
```

### Usare le priorità

```py
my_heap = []
heapq.heappush(my_heap, (3, "A"))
heapq.heappush(my_heap, (2, "B"))
heapq.heappush(my_heap, (1, "C"))

# Removes lowest number = highest priority
print(heapq.heappop(my_heap))  # (1, "C")
```

## Introduzione alla programmazione dinamica

- **Definizione**: La programmazione dinamica è una tecnica algoritmica che risolve problemi complessi suddividendoli in sotto-problemi più semplici e memorizzando i risultati per evitare calcoli ridondanti.
- **Sovrapposizione di sotto-problemi**: Gli stessi sotto-problemi più piccoli appaiono più volte quando si risolve il problema più grande. Invece di ricalcolare questi sotto-problemi ripetutamente, memorizziamo le loro soluzioni.
- **Sottostruttura ottimale**: La soluzione ottimale al problema contiene soluzioni ottimali ai suoi sotto-problemi. Questo significa che possiamo costruire la migliore soluzione combinando le migliori soluzioni alle parti più piccole.

## Soluzioni di programmazione dinamica

- **Memoizzazione (approccio top-down)**: La memoizzazione memorizza i risultati di chiamate di funzione costose e restituisce il risultato memorizzato nella cache quando gli stessi input si ripresentano.

```py
def climb_stairs_memo(n, memo={}):
    """Dynamic programming with memoization"""
    # Check if we've already calculated this value
    if n in memo:
        return memo[n]  # Return cached result - O(1) lookup!
    
    # Base cases
    if n <= 2:
        return n
    
    # Calculate once and store in memo for future use
    memo[n] = climb_stairs_memo(n-1, memo) + climb_stairs_memo(n-2, memo)
    return memo[n]
```

- **Tabulazione (approccio bottom-up)**: La tabulazione costruisce la soluzione dal basso verso l’alto, riempiendo una tabella con le soluzioni ai sotto-problemi.

```py
def climb_stairs_tabulation(n):
    """Dynamic programming with tabulation"""
    if n <= 2:
        return n
    
    # Create array to store results for all steps from 0 to n
    dp = [0] * (n + 1)
    dp[1] = 1  # 1 way to reach step 1
    dp[2] = 2  # 2 ways to reach step 2
    
    # Build up the solution iteratively
    for i in range(3, n + 1):
        # Ways to reach step i = ways to reach (i-1) + ways to reach (i-2)
        dp[i] = dp[i-1] + dp[i-2]
    
    return dp[n]
```

## Applicazioni reali che usano la programmazione dinamica

- **Ottimizzazione delle rotte**: I sistemi GPS usano algoritmi di programmazione dinamica per trovare i percorsi più brevi tra le località.
- **Elaborazione del testo**: I correttori ortografici e le funzioni di completamento automatico spesso si basano sulla programmazione dinamica per calcolare le distanze di modifica tra le parole.
- **Modellazione finanziaria**: Strategie di investimento e ottimizzazione di portafoglio impiegano frequentemente tecniche di programmazione dinamica.
- **Allocazione delle risorse**: Il problema dello zaino e le sue varianti compaiono nella pianificazione, nel budgeting e nella gestione delle risorse.

## Quando usare la programmazione dinamica

Dovresti considerare di usare la programmazione dinamica nei seguenti scenari:

- Il problema può essere suddiviso in sotto-problemi sovrapposti.
- Il problema presenta una sottostruttura ottimale.
- Una soluzione ricorsiva ingenua comporterebbe calcoli ripetuti.
- Hai bisogno di ottimizzare la complessità temporale a scapito della complessità spaziale.
