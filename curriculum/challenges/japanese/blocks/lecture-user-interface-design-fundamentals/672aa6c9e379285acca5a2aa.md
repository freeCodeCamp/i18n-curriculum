# --interactive--

**注意**: いくつかのインタラクティブな例では、まだ学んでいないCSSを使っている場合があります。コードのすべてを理解しようと心配しないでください。これらの例の目的は、デザインの概念をプレビューで示し、仕組みをよりよく理解してもらうことです。

開発者として、デザイナーと密に連携する必要があるかもしれません。基本的なデザイン用語を理解していれば、共通の語彙ができ、デザインプロセスにより積極的に参加できます。これにより作業効率が上がり、より良いユーザー体験につながります。

では始めましょう。まずは「レイアウト」という用語からです。レイアウトとは、メッセージを伝えるためにページや画面上の視覚要素がどのように配置されているかを指します。これらの要素にはテキスト、画像、余白などが含まれます。レイアウトはデザインの設計図のようなものです。デザイナーは各要素の配置、サイズ、階層を考慮しなければなりません。

:::interactive_editor

レイアウトに密接に関連する用語が「アライメント」です。アライメントとは、要素同士がどのように配置されているかを指します。アライメントを正しく使うことで、デザインがすっきりと整理された印象になります。デザイナーは想像上の線や端、中心点に沿って要素を整列させることで視覚的な秩序を作り出します。

:::interactive_editor

    <h2>Center-Aligned</h2>

  </section>
</div>

よいですね。次に「コンポジション」について話しましょう。コンポジションは、要素を配置して調和のとれたデザインを作る技術です。画像、テキスト、形などの要素が互いにどう関係し、芸術的にデザインに寄与するかを決めます。レイアウトが主に要素の配置に焦点を当てるのに対し、コンポジションはその配置が全体のデザインに与える芸術的な影響も考慮します。

コンポジションは「バランス」の概念と密接に関連しています。バランスとは、コンポジション内で視覚的な重みがどのように分布しているかを指します。デザイナーは左右対称または非対称の配置で均衡を作り出そうとします。バランスのとれたデザインは調和が感じられます。

:::interactive_editor

これらはよく見かける一般的なデザインの概念ですが、ソフトウェア開発により密接に関連するデザイン用語もあります。

ユーザーインターフェース（UI）は、人間がコンピューターとやり取りする方法です。ユーザーインターフェースには、画面上でユーザーが見たり操作したりできるアイコン、画像、テキスト、メニュー、リンク、ボタンなどの視覚的かつインタラクティブな要素が含まれます。

ユーザーエクスペリエンス（UX）は、製品やサービスを使うときのユーザーの感覚に関するものです。よく設計されたユーザーエクスペリエンスを持つアプリケーションは直感的で使いやすく、効率的でアクセシブル、そして楽しいものです。ユーザーインターフェースはユーザーエクスペリエンスをできるだけ簡単で楽しいものにする上で重要な役割を果たすため、両者は非常に密接に関連しています。

これらはデザイナーとコミュニケーションをとるために知っておくべき一般的な用語です。この知識があれば、デザインプロセスにより積極的に参加できるようになります。
