# --description--

JavaScriptは動的型付け言語と考えられています。これは、変数が実行時にどんな値でも受け取れることを意味します。数値として変数を宣言しても、後で文字列の値を代入することができ、関数のパラメータにもどんな値でも渡せます。

動的型付け言語の課題は、型の安全性がないために注意しないとエラーが発生しやすいことです。TypeScriptはJavaScript言語を拡張して静的型付けを導入し、コードをデプロイする前にそうしたエラーを検出できるようにします。でも、どうやって動くのでしょうか？

このJavaScriptの関数を考えてみましょう：

```js
const getRandomValue = (array) => {
  return array[Math.floor(Math.random() * array.length)];
}
```

私たちの`getRandomValue`関数は配列を受け取り、その配列からランダムな値を返します。しかしJavaScriptは型を検証しないため、数値を渡して関数を呼び出しても何も止められません：

```js
console.log(getRandomValue(10));
```

この例のコンソール出力は`undefined`になります。配列を期待していたのに数値が渡されたためです。

しかしTypeScriptでは配列パラメータの型を定義できます：

```js
const getRandomValue = (array: string[]) => {
  return array[Math.floor(Math.random() * array.length)];
}
```

この型定義は、配列の引数は文字列の配列でなければならないとTypeScriptに伝えます。`getRandomValue`を呼び出して数値を渡すと、コンパイラーエラーという別の種類のエラーが発生します。

ブラウザやNode.jsなどの多くのJavaScript実行環境はTypeScriptをネイティブに実行できません。代わりに、まずTypeScriptコードを通常のJavaScriptにコンパイル（翻訳）する必要があります。TypeScript言語に組み込まれているコンパイラーを使ってそれができます。コンパイラーを実行すると、型が合わない問題がある場合にエラーをスローします。例えば、配列を期待する関数に数値を渡す場合などです。

これらの型がどのように機能するかは、次のいくつかのレッスンで詳しく学びます。しかし、この追加の安全性が多くの開発者や組織にとってTypeScriptを魅力的な言語にしています。
