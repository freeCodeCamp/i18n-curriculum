# --explanation--

`patch`는 문제를 고치거나 성능을 개선하거나 보안 문제를 해결하기 위해 개발자가 배포하는 작은 업데이트입니다. 예를 들어:

`The company released a patch to fix the login error.` – 문제에 대한 수정을 게시했다는 뜻입니다.

`To exploit`는 특히 약점을 이용해 부당하거나 해로운 방식으로 무언가를 사용하는 것을 의미합니다. 예를 들어:

`Hackers exploited a flaw in the email server.` – 약점을 이용해 접근하거나 피해를 입혔다는 뜻입니다.
