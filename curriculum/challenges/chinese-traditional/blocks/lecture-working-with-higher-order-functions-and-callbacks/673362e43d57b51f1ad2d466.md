# --interactive--

當你在 JavaScript 中操作陣列時，經常會想要檢查陣列中的所有元素是否都符合某個條件，或者是否至少有一個元素符合該條件。

這就是 `every()` 和 `some()` 方法派上用場的地方。這些方法是強大的工具，可以簡化你的程式碼並使其更易讀。

讓我們從 `every()` 方法開始。此方法會測試陣列中的所有元素是否通過由提供的函式實作的測試。簡單來說，它會檢查陣列中的每一個項目是否符合你指定的條件。

如果提供的函式對陣列中所有元素都傳回 `true`，則 `every()` 方法會傳回 `true`。如果有任何元素未通過測試，該方法會立即傳回 `false` 並停止檢查剩餘的元素。

以下是一個說明 `every()` 如何運作的範例：

:::interactive_editor

```js
const numbers = [2, 4, 6, 8, 10];
const hasAllEvenNumbers = numbers.every((num) => num % 2 === 0);

console.log(hasAllEvenNumbers); // true
```

:::

在這個範例中，我們正在檢查陣列中的所有數字是否都是偶數。我們提供給 `every()` 的函式會檢查每個數字是否能被 `2` 整除且沒有餘數。由於我們陣列中的所有數字確實都是偶數，`hasAllEvenNumbers` 將會是 true。

現在，讓我們來看看 `some()` 方法。

當 `every()` 檢查所有元素是否通過測試時，`some()` 則檢查是否至少有一個元素通過測試。`some()` 方法一旦找到通過測試的元素就會傳回 `true`。如果沒有元素通過測試，則傳回 `false`。

以下是 `some()` 如何運作的範例：

:::interactive_editor

```js
const numbers = [1, 3, 5, 7, 8, 9];
const hasSomeEvenNumbers = numbers.some((num) => num % 2 === 0);

console.log(hasSomeEvenNumbers); // true
```

:::

在這個範例中，我們正在檢查陣列中是否有任何數字是偶數。我們傳遞給 `some()` 的函式（程式）與之前相同。即使我們陣列中的大多數數字是奇數，`hasEven` 仍會是 `true`，因為陣列中至少有一個偶數（`8`）。

當你需要驗證資料或檢查陣列中的特定條件時，`every()` 和 `some()` 都非常有用。它們常常可以取代較冗長的迴圈和條件述語，使你的程式碼更簡潔且更具表達力。

重要的是要注意，這兩種方法在能夠確定結果時就會停止執行。對於 `every()`，這表示一旦找到 `false` 結果就會停止。對於 `some()`，一旦找到 `true` 結果就會停止。這對效率特別有利，尤其是在處理大型陣列時。
