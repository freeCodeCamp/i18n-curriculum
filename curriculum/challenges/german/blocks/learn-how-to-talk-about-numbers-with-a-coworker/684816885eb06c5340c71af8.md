# --explanation--

`Initial unit tests` sind der erste Schritt beim Testen. Entwickler überprüfen kleine Teile des Codes (`units`), um sicherzustellen, dass sie das tun, was sie sollen. Zum Beispiel:

`We wrote initial unit tests to check the main functions.` — Das Team hat frühe Tests erstellt, um sicherzustellen, dass die wichtigsten Teile des Codes korrekt funktionieren.

`Quality assurance (QA) checks` sind abschließende Tests, die von QA-Teams durchgeführt werden, um sicherzustellen, dass die Software zuverlässig, benutzerfreundlich und startbereit ist. Zum Beispiel:

`Regular QA checks help prevent bugs from reaching users.` — Häufiges Testen durch QA hilft, Fehler frühzeitig zu erkennen, damit Benutzer keine Probleme haben.
