# --description--

In vorherigen Lektionen haben Sie Bash und einige der häufig verwendeten Befehle kennengelernt. Aber Bash unterstützt eine vollständige Skriptsprache, mit der Sie komplexere automatisierte Aufgaben ausführen können.

Bash-Scripting bedeutet, eine Folge von Bash-Befehlen in einer Datei zu schreiben, die Sie dann mit Bash ausführen können, um den Inhalt der Datei ablaufen zu lassen.

Schauen wir uns an, wie ein Bash-Skript aussehen kann:

```bash
#!/bin/bash

servers=("prod" "dev")

for server in "${servers[@]}"
do
  echo "Pulling $server"
  rsync --archive --verbose $server:/etc/nginx/conf.d/server.conf configs/$server.conf
done
```

Dieses Skript lädt NGINX-Konfigurationsdateien von einer Liste entfernter Server herunter, was nützlich sein kann, um diese Konfigurationen in einem lokalen Repository zu sichern. NGINX ist ein beliebter Webserver, und Sie werden in einem zukünftigen Modul mehr über die Arbeit mit Servern lernen.

Die erste Zeile wird „shebang“ (`#!`) genannt und teilt dem System mit, welcher Interpreter zum Ausführen des Skripts verwendet werden soll. Übliche Werte hier sind `/usr/bin/bash`, `/bin/bash` oder `/bin/sh`.

Die erste Zeile, `servers=("prod" "dev")`, erstellt eine Liste von Strings, die die Servernamen darstellen. `"prod"` wird oft verwendet, um die Produktionsseite zu bezeichnen, während `"dev"` die Entwicklungsseite meint, die typischerweise zum Testen neuer Funktionen und Änderungen genutzt wird, bevor sie in der Live-Produktionsumgebung bereitgestellt werden.

Dann wird die Liste mit einer `for`-Schleife durchlaufen – die `servers[@]`-Syntax erweitert die Liste in einzelne Elemente. Jedes Element wird bei jeder Iteration der Variablen server zugewiesen.

Die `do`-Anweisung markiert den Beginn des logischen Blocks der Schleife, also den Code, der bei jeder Iteration ausgeführt werden soll.

`echo "Pulling $server"` interpoliert den aktuellen Wert der Variablen server, der in diesem Fall prod oder dev sein würde, und gibt ihn im Terminal aus.

Dann wird `rsync` verwendet, um die eigentliche Datei vom Server herunterzuladen und lokal zu speichern, wobei weitere Interpolation zur Steuerung dieses Befehls genutzt wird.

`done` markiert das Ende des logischen Blocks der Schleife.

Es gibt eine Reihe bedeutender Vorteile, Bash-Scripting zu verstehen und zu nutzen.

Der erste ist, dass nahezu jede Unix-Umgebung Bash standardmäßig bereitstellt, was bedeutet, dass Sie Ihr Skript unverändert ausführen können, ohne Laufzeitumgebungen wie Node.js oder Python konfigurieren zu müssen.

Der zweite Vorteil ist, dass Sie mit minimalem Aufwand auf alle binären Anwendungen zugreifen können, die auf dem System verfügbar sind. Wenn Ihr System doctl installiert hat, kann Ihr Skript dieses Binary ausführen. Ein Binary ist ein kompiliertes Programm, das der Computer ausführt. Mit einem Bash-Skript können Sie das Binary ausführen, das das Programm startet.

Und schließlich ist die Syntax, die Sie für Ihr Skript verwenden, auch direkt im Terminal ausführbar. Das bedeutet, dass das Testen eines kleinen Teils Ihres Skripts viel weniger kompliziert wird – Sie können diesen Teil einfach kopieren und direkt in Ihr Terminal einfügen und das Ergebnis sehen!

Bash zu verstehen wird nicht nur Ihre Skripting-Fähigkeiten verbessern, sondern Sie zu einem wahren Meister der Kommandozeile machen!
