# --interactive--

Цикл `for...in` найкраще використовувати, коли потрібно ітерувати над властивостями об’єкта. Цей цикл пройде всі перелічувані властивості об’єкта, включно з успадкованими властивостями та нечисловими властивостями.

Успадкована властивість — це властивість, успадкована від прототипного ланцюжка об’єкта. Нечислова властивість — це властивість, яка не є числом або рядком, який можна перетворити на число.

Ось базовий синтаксис циклу `for...in`:

```js
for (variable in object) {
  // code block to be executed
}
```

Змінна в прикладі представляє поточну властивість об’єкта, по якій ітерують.

Давайте розглянемо кілька прикладів, щоб краще зрозуміти, як працює цикл `for...in`.

У першому прикладі у нас є об’єкт `fruit`, і ми хочемо пройтися по кожній властивості та вивести значення в консоль.

:::interactive_editor

```js
const fruit = {
  name: 'apple',
  color: 'red',
  price: 0.99
};

for (const prop in fruit) {
  console.log(fruit[prop]);
}
```

:::

Змінна `prop` представляє поточну властивість об’єкта. `fruit[prop]` використовується для доступу до значення кожної властивості.

Під час першої ітерації `prop` буде `name`. Під час другої ітерації `prop` буде `color`, і так далі.

Результати, які виведуться в консоль, будуть `apple`, `red` та `0.99`.

У другому прикладі у нас вкладений об’єкт, і ми хочемо пройтися по кожній властивості та вивести значення в консоль.

:::interactive_editor

```js
const person = {
  name: 'John',
  age: 30,
  address: {
    street: '123 Main St',
    city: 'Anytown',
    state: 'CA'
  }
};

for (const prop in person) {
  console.log(person[prop]);
}
```

:::

Властивість `address` сама є об’єктом. Цикл `for...in` також пройде по властивостях об’єкта `person` і виведе весь об’єкт `address` у консоль.

Ось як виглядатиме результат у консолі:

```md
John
30
{ street: '123 Main St', city: 'Anytown', state: 'CA' }
```

Якщо ви хочете пройтися по властивостях об’єкта `address`, можна вкласти ще один цикл `for...in` всередину першого.

:::interactive_editor

```js
const person = {
  name: 'John',
  age: 30,
  address: {
    street: '123 Main St',
    city: 'Anytown',
    state: 'CA'
  }
};

function isObject(obj) {
  return typeof obj === 'object' && !Array.isArray(obj) && obj !== null;
}

for (const prop in person) {
  if (isObject(person[prop])) {
    for (const nestedProp in person[prop]) {
      console.log(person[prop][nestedProp]);
    }
  } else {
    console.log(person[prop]);
  }
}
```

:::

У цьому прикладі у нас є кастомна функція `isObject`, яка перевіряє, чи є значення об’єктом.

Метод `Array.isArray` використовується, щоб перевірити, чи є значення масивом. Поміщаючи логічний оператор НЕ (`!`) перед методом, ми перевіряємо, чи значення не є масивом.

Причина, чому ми не можемо просто використати `typeof` рівне `'object'`, полягає в тому, що масиви також вважаються об’єктами в JavaScript. Ми хочемо виключити масиви з перевірки.

Також через історичну помилку в JavaScript `typeof null` повертає `'object'`. Тому ми також хочемо виключити значення `null` з перевірки.

Якщо умова істинна, ми вкладаємо ще один цикл `for...in`, який пройде по властивостях вкладеного об’єкта і виведе значення в консоль.

Змінна `nestedProp` представляє поточну властивість вкладеного об’єкта.

Ось як виглядатиме змінений результат у консолі:

```md
"John"
30
"123 Main St"
"Anytown"
"CA"
```

Цикл `for...in` корисний, коли потрібно пройтися по властивостях об’єкта.

Не рекомендується використовувати цикл `for...in` для ітерації по елементах масиву. Замість цього використовуйте цикл `for...of` або інші методи масивів, такі як `forEach`, `map`, `filter` та `reduce`, про які ви дізнаєтеся в майбутніх уроках.
