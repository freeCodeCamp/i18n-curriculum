# --description--

我们已经稍微谈过一些关于 Git 和 GitHub 的内容，现在我们可以开始将它们整合起来。为了做到这一点，你需要一个仓库。但什么是仓库呢？你可以将仓库看作是一个项目的容器——如果你正在开发一个应用程序，你会将该应用程序的文件集中保存在一个仓库中。

仓库可以是你计算机上的局部的，也可以是像 GitHub 这样的服务上的远程仓库。要创建一个仓库，你可以从局部的或远程的开始——结果是相同的，所以选择最适合你工作流程的方法。对于从远程仓库开始的工作流程，你可以通过遵循这些说明直接在 GitHub 网站上创建一个仓库：

访问<a href="https://github.com" target="_blank">GitHub 的网站</a>，确保你已登录，然后点击导航栏右上角的加号图标。你会看到很多选项，但你关心的是“New Repository”。选择它以打开创建新仓库的界面。

你首先会看到的选项是选择一个 `template`。你可能还没有设置这样的 `template`，但 `template` 是一种特殊类型的仓库，可以作为你新仓库的跳板。这些对于在你所有的项目中包含文档或问题模板非常有帮助。

你的下一个选项是选择一个所有者，你可以选择你的用户账户或你拥有或有访问权限的任何组织。然后你可以为仓库命名——通常，你会想使用一个简洁描述你正在创建的项目的名称。作为通用的最佳实践，你应避免在仓库名称中使用空格。GitHub 会自动将它们替换为连字符。

接下来，你可以可选地添加一个描述，该描述将显示在仓库的主页上。

你还可以选择仓库是公开的还是私有的。公开仓库可以被任何人查看和下载。私有仓库只能被你和你显式授予访问权限的任何人访问。

最后，如果你不选择模板，你将看到自动在仓库中生成文件的选项。一个 README 文件将出现在仓库主页上，是提供项目文档和描述的绝佳方式。`.gitignore` 文件允许你指定 Git 将“忽略”的文件——这对于包依赖、环境文件以及其他你不想被储存的内容非常有用。GitHub 会为该文件提供模板，排除指定语言或运行期的常见项。许可证是一份法律文件，概述了谁可以使用你的软件，以及他们可以对其做什么。

当你将你的设置全部配置好后，你可以点击 `"Create Repository"` 按钮完成该进程。你将自动被重定向到你的新仓库的登录页面！

从这里，你可以点击 `Code` 按钮以获取将你的仓库克隆到你的计算机的选项。

你可以忽略本课中的 "Codespaces" 和 "Copilot" 标签（页）。相反，我们将重点关注 "Local" 标签（页）中的三个选项。前两个，HTTPS 和 SSH，使用方式相同。你使用带有框中提供的 URL 的 `git clone` 命令将远程仓库克隆到你的计算机。这两者之间的区别在于连接和认证方法。

克隆公共仓库时，实际上你不需要进行身份验证。但要克隆私有仓库，或向公共或私有仓库推送，你必须经过身份验证。使用 HTTPS 远程 URL 通过你的 GitHub 用户名和密码对你进行身份验证。然而，出于安全性考虑，这种方式已不再支持，你需要改为使用用户名和访问令牌。即便如此，这种方法的安全性远不如使用私钥和公钥的 SSH。这还有一个额外好处，就是可以自动登录，而不必每次都提供令牌。将有一节单独课程深入讲解 SSH 及其工作原理，你也可以阅读 GitHub 文档了解如何设置 SSH。

要克隆仓库，你可以在终端中运行此命令：

```sh
git clone git@github.com:your-username/your-repository-name
```

它将在你的计算机上创建一个以你的仓库名称命名的文件夹，并包含仓库中的所有文件。然后你可以使用 `cd`（切换目录）进入你的克隆仓库，命令如下：

```sh
cd name-of-cloned-repo
```

此外，你的仓库的局部拷贝将有一个指向 GitHub 上拷贝的远程。要查看你的远程，可以运行以下 `Command`：

```sh
git remote -v
```

克隆你的仓库的另一个选项是使用 GitHub CLI。该工具用于在不离开命令行的情况下执行 GitHub 特定的任务。如果你尚未安装它，可以从 GitHub 的文档中获取安装说明——但你应该能够在系统的包管理器中找到它。

要使用 CLI 工具克隆一个仓库，我会使用：

```sh
gh repo clone your-username/your-repository-name
```

这些命令的最终结果是相同的：你将在本地拥有你的仓库的拷贝。

你也可以使用 GitHub CLI 来创建一个仓库：

```sh
gh repo create
```

如果你不传递任何参数，CLI 会为你提供一个交互式向导来设置你的仓库。该向导将为你提供与网页 UI 相同的所有选项。使用此工具，你还将获得立即克隆新建仓库的选项，因此你无需再执行前面描述的手动步骤。

你现在已经对仓库的工作原理有了基本的了解，以及如何创建一个仓库以开始正确地管理你的版本。在以后的课程中，你将学习如何将现有的局部的仓库推送到 GitHub，以及如何将更改推送到远程仓库。
