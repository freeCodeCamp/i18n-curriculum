## WebSockets 与 HTTP 的区别

- **HTTP** 遵循请求-响应模型：客户发送请求，服务器发送响应，然后连接关闭。
- **WebSockets** 在客户和服务器之间创建一个永久的双向连接，直到任一方发送关闭信号才会关闭。
- HTTP 非常适合加载页面或获取数据，但对于实时应用来说会变得缓慢且低效。
- 每个 WebSocket 连接都以一个 **HTTP 握手** 开始：客户请求升级，服务器响应 `101 Switching Protocols`。

### WebSocket 连接流程

1. 客户通过 HTTP 连接服务器并请求升级。
2. 服务器接受并响应 `101 Switching Protocols`。
3. 连接保持打开状态。
4. 双方随时交换数据，直到连接关闭。

### HTTP 与 WebSocket 通信对比

- **HTTP**：`Client → Request → Server → Response`（一个周期，然后关闭）
- **WebSocket**：`Client ↔ Server`（持久的，双向的）

### 一个有用的比喻

- **HTTP** 就像交换信件：你写一封，等待回复，然后对话结束。
- **WebSockets** 就像打电话：双方实时通话，持续所需时间。

### WebSockets 的实际应用

- 聊天应用
- 在线多人游戏
- 实时体育比分更新
- 股票市场仪表盘
- 协作工具（例如 Google Docs）

## 发布/订阅架构

- **发布/订阅**（Pub/Sub）是一种消息架构，允许系统各部分在不直接了解彼此的情况下通信。
- 主要有三个部分：**发布者**、**主题** 和 **订阅者**。
  - **发布者** 向主题发送消息。
  - **主题**（也称为频道或组）充当中间人。
  - **订阅者** 接收其订阅主题的消息。

### 发布/订阅消息流程

1. 订阅者订阅一个主题。
2. 发布者向该主题发送消息。
3. 消息系统将消息传递给该主题的所有订阅者。

### 直接通信与发布/订阅对比

- **无发布/订阅**：`Application A → Application B` — 紧耦合；B 的变化可能导致 A 出错。
- **有发布/订阅**：`Application A → Topic → Subscribers` — A 只负责发布；不需要知道谁接收。

### 发布/订阅的实际应用

- 多聊天室的聊天应用
- 在线多人游戏
- 通知系统
- 实时仪表盘
- 事件驱动应用

### WebSockets 与发布/订阅

- **WebSockets** 提供通信通道（道路）。
- **发布/订阅** 提供消息模式，决定谁接收特定消息（交通系统）。
- 许多实时应用同时使用两者：WebSockets 保持连接打开，发布/订阅将消息路由到正确的订阅者。

## 在 Node.js 中使用 WebSockets

- Node.js 没有内置的 WebSocket 服务器。`ws` 库是标准的外部包。
- 安装命令：`npm install ws`

### 创建 WebSocket 服务器

```js
const WebSocket = require("ws");

const server = new WebSocket.Server({ port: 3000 });
```

### WebSocket 事件

| 事件 | 触发条件 |
|---|---|
| `connection` | 新客户连接服务器 |
| `message` | 已连接客户发送数据 |
| `close` | 连接关闭 |
| `error` | 连接出现问题 |

### 向客户发送消息

```js
socket.send("Hello from the server");
```

### 向所有已连接客户广播

- `server.clients` 保存所有已连接客户。
- 发送前查看 `readyState === WebSocket.OPEN` 以避免向断开连接的客户发送消息导致错误。

```js
server.on("connection", (socket) => {
  socket.on("message", (message) => {
    server.clients.forEach((client) => {
      if (client.readyState === WebSocket.OPEN) {
        client.send(message.toString());
      }
    });
  });
});
```

- **广播** 是向多个已连接客户同时发送消息的模式——用于聊天应用、实时通知、体育更新和多人游戏事件。
