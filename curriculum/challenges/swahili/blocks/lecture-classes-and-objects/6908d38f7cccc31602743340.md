# --description--

Katika funzo la awali, ulijifunza kuhusu sifa za kipengele kuwa ni vigezo vinavyomilikiwa na kipengele. Hii inamaanisha zinashikilia data inayofafanua hali au tabia ya kipengele hicho.

Kwa mfano, gari kawaida lina chapa na mfano. Chapa na mfano vinaweza kuwa sifa za kipengele kwa darasa la `Car`:

```python
class Car: 
    def __init__(self, brand, model): 
        self.brand = brand 
        self.model = model 

my_car = Car('Lamborghini', 'Gallardo') 
print(my_car.brand) # Lamborghini 
print(my_car.model) # Gallardo 
```

Lakini wakati mwingine, hujui ni sifa gani unazohitaji hadi programu yako ianze kutekelezwa. Fikiria unaandika script inayopokea majina ya sifa kutoka kwa mtumizi au faili la usanidi. Hizo si sifa unazoweza kuweka msimbo thabiti kwa wakati wa awali.

Hapa ndipo kushughulikia sifa kwa njia ya mabadiliko kunapoingia. Kwa njia hii, unaweza kupata upatikanaji, kubadilisha, kuangalia, au hata kufuta sifa kwa kutumia majina yao kama vigezo, na si kama majina thabiti katika msimbo wako. Hii inampa programu yako uwezo wa kubadilika ili kujibu data tofauti au ingizo la mtumizi kwa wakati halisi.

Python inakupa vitendakazi vinne vilivyojengwa ndani vya kufanya kazi na sifa za kipengele kwa njia ya mabadiliko. Ni `getattr()`, `setattr()`, `hasattr()`, na `delattr()`.

Vinakuwezesha kupata upatikanaji, kuunda, kuangalia, na kufuta sifa kwa kutumia majina ya vigezo. Hebu tuchunguze kila moja ikitenda kazi.

`getattr()` inafanya iwezekane kusoma sifa kutoka kwa kipengele wakati hujui jina lake hadi wakati wa utekelezaji. Ikiwa sifa haipo, hutoa `AttributeError`, isipokuwa ukitoa thamani ya chaguo-msingi.

Ili kuitumia, unapitisha kipengele, jina la sifa, na thamani ya chaguo-msingi kama inavyohitajika:

```python
getattr(object, attribute_name, default_value) 
```

Hapa kuna mfano:

```python
class Person: 
    def __init__(self, name, age): 
        self.name = name 
        self.age = age 

person = Person('John Doe', 30) 
 
print(getattr(person, 'name')) # John Doe 
print(getattr(person, 'age')) # 30 
print(getattr(person, 'city', 'Milano')) # Milano
```

Katika mfano hapo juu, `Milano` ni thamani ya chaguo-msingi kwa sababu `city` haipo katika darasa la `Person`.

Kama tulivyosema awali, nguvu halisi ya `getattr()` inaonekana pale jina la sifa linapotoka kwenye kigezo, kama vile kutoka kwa ingizo la mtumizi au faili fulani.

Katika hali hiyo, huwezi kutumia sintaksia ya kawaida ya `object.attribute_name` kwa sababu jina la sifa haliko thabiti.

```python
class Person: 
    def __init__(self, name, age): 
        self.name = name 
        self.age = age 

person = Person('John Doe', 30)

attr_name = input('Enter the attribute you want to see: ')
print(getattr(person, attr_name, 'Attribute not found'))
```

Katika kesi hii, ikiwa mtumizi ataandika `name`, wanaona `John Doe`, na ikiwa wataandika `age`, wanaona `30`. Na ikiwa wataandika kitu ambacho hakipo katika darasa kama `email`, wanaona `Attribute not found`.

Hii ndiyo hasa faida ya kushughulikia sifa kwa njia ya mabadiliko. Inaruhusu msimbo wako kujibu ingizo na data ambayo haujawahi kuiona kabla.

Zaidi ya hayo, unaweza kutaka kupitia sifa zote ambazo kipengele kina, si zile tu unazojua tayari. Kitendakazi kilichojengwa ndani cha `dir()` kinakuruhusu kufanya hivyo. Kinarejesha orodha ya majina yote ya sifa kwenye kipengele. Hapa ni jinsi ya kuitumia:

```python
class Person: 
    def __init__(self, name, age): 
        self.name = name 
        self.age = age 

person = Person('John Doe', 30)

# Loop through all attributes of the person object with dir() function
for attr in dir(person):
    # Ignore dunder methods like __init__ or __str__ and regular methods
    if not attr.startswith('__') and not callable(getattr(person, attr)): 
        value = getattr(person, attr)
        print(f'{attr}: {value}')

# Output
# age: 30
# name: John Doe
```

Katika mzunguko hapo juu, `callable()` ni kitendakazi kilicho jengwa ndani kinachorejesha `True` ikiwa kipengele kilichopitishwa kinaweza kuitwa kama kitendakazi au njia, na `False` vinginevyo. Kwa kuangalia `not callable(getattr(person, attr))`, mzunguko unaruka njia na kuchapisha tu sifa za data kama `name` na `age`.

Kitendakazi cha `setattr()` kinakuruhusu kuunda sifa mpya au kusasisha ile iliyopo kwa njia ya mabadiliko. Sintaksia ni kama ifuatavyo:

```python
setattr(object, attribute_name, value) 
```

Hapa kuna mfano unaoweka sifa za usanidi kulingana na data kutoka kwa faili la usanidi au kigezo cha mazingira:

```python
class Configuration:
    pass

# Data loaded at runtime (like from a config or env file)
settings_data = {
    'server_url': 'https://api.example.com',
    'timeout_sec': 30,
    'max_retries': 5
}

config_obj = Configuration()

# Dynamically set attributes using dictionary keys and values
for attr_name, attr_value in settings_data.items():
    setattr(config_obj, attr_name, attr_value)

print(config_obj.server_url) # https://api.example.com
print(config_obj.timeout_sec) # 30
```

Pia kuna `hasattr()`. Kabla hujafanya kitu na sifa au kuifuta, ni desturi nzuri kuangalia kama ipo. Hilo ndilo `hasattr()` linakuruhusu kufanya. Linaangalia kama sifa ipo na kurejesha `True` au `False` kulingana na matokeo.

Hii ni sintaksia ya msingi:

```python
hasattr(object, attribute_name)  
```

Na hapa kuna mfano unaoangalia kwa njia ya mabadiliko uwepo wa sifa katika mfano wa darasa la `Product`:

```python
class Product:
    def __init__(self, name, price):
        self.name = name
        self.price = price

product_a = Product('T-Shirt', 25)

required_attributes = ['name', 'price', 'inventory_id']

for attr in required_attributes:
    if not hasattr(product_a, attr):
        print(f"ERROR: Product is missing the required attribute: '{attr}'")
    else:
        # Access the attributes dynamically once their existence is confirmed
        print(f'{attr}: {getattr(product_a, attr)}')

# Output:
# name: T-Shirt
# price: 25
# ERROR: Product is missing the required attribute: 'inventory_id'
```

Matokeo ya ERROR yalitokea kwa sababu `inventory_id` haipo katika darasa la `Product` na mfano wake.

Mwishowe, `delattr()` inakuwezesha kufuta sifa kwa njia ya mabadiliko:

```python
delattr(object, attribute_name) 
```

Kwa mfano, fikiria kipengele kimekamilika kusindika, kisha unaamua kusafisha sifa zozote nyeti au za muda ambazo zinaweza kuwepo kabla ya kuhifadhi toleo la mwisho. Baada ya hapo, unaweza kutumia `dir()` kuzunguka sifa zilizobaki:

```python
class UserSession:
    def __init__(self, user_id, token):
        self.user_id = user_id
        self.auth_token = token # sensitive
        self.temp_counter = 0 # temporary

session = UserSession(101, 'a1b2c3d4e5')

# List of attributes to remove dynamically before "saving" the session
attributes_to_clean = ['auth_token', 'temp_counter']

# Dynamically remove specified attributes
for attr in attributes_to_clean:
    if hasattr(session, attr):
        delattr(session, attr)
        print(f'Removed attribute: {attr}')

print('\nFinal attributes remaining:')

# Loop through the remaining attributes with dir()
for attr in dir(session):
    # Ignore dunder methods like __init__ or __str__ and regular methods
    if not attr.startswith('__') and not callable(getattr(session, attr)):
        print(f' - {attr}: {getattr(session, attr)}')

# Output:
# Removed attribute: auth_token
# Removed attribute: temp_counter

# Final attributes remaining:
#  - user_id: 101
```

Na ndivyo unavyoweza kushughulikia sifa kwa njia ya mabadiliko!
