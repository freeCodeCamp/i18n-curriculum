# --description--

Sie haben vielleicht bemerkt, dass Sie bisher nur die Benutzeranzahl erhöht haben. Das Behandeln einer Benutzertrennung ist genauso einfach wie das Behandeln der ersten Verbindung, nur dass Sie diesmal auf jedem Socket einzeln darauf hören müssen, statt auf dem gesamten Server.

Fügen Sie dazu einen weiteren Listener innerhalb des bestehenden `'connect'`-Listeners hinzu, der auf `'disconnect'` am Socket hört, ohne dass Daten übergeben werden. Sie können diese Funktion testen, indem Sie einfach in der Konsole protokollieren, dass ein Benutzer die Verbindung getrennt hat.

```js
socket.on('disconnect', () => {
  /*anything you want to do on disconnect*/
});
```

Um sicherzustellen, dass die Clients stets die aktuelle Anzahl der Benutzer haben, sollten Sie `currentUsers` um 1 verringern, wenn die Trennung eintritt, und dann das `'user count'`-Ereignis mit der aktualisierten Anzahl senden.

**Hinweis:** Genau wie bei `'disconnect'` sollten alle anderen Ereignisse, die ein Socket an den Server senden kann, innerhalb des Verbindungs-Listeners behandelt werden, in dem wir 'socket' definiert haben.

Senden Sie Ihre Seite ab, wenn Sie denken, dass alles richtig ist. Wenn Sie auf Fehler stoßen, können Sie <a href="https://forum.freecodecamp.org/t/advanced-node-and-express/567135#handle-a-disconnect-8" target="_blank" rel="noopener noreferrer nofollow">das bis zu diesem Punkt fertiggestellte Projekt ansehen</a>.
