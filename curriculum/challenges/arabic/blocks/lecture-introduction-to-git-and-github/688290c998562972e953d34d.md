# --description--

لقد تعلمت سابقًا كيفية الالتزام بالتغييرات ودفعها إلى مستودعك البعيد، لكن كل ذلك كان على الفرع الافتراضي `main`. هذا قد يناسب مشروعًا صغيرًا، لكن مع مساهمتك في مشاريع مفتوحة المصدر أخرى، أو مع نمو مشروعك، تصبح الفروع ضرورية.

على سبيل المثال، إذا قمت بعمل فورك لمستودع وأجريت التزامات على فرع `main` في فوركك، قد ينتهي بك الأمر مع تغييرات في فوركك لم يقبلها المستودع الأصلي. هذا يجعل من الصعب جدًا تحديث فرعك بالتغييرات التي قُبلت في الفورك الأصلي. أو، عند العمل على مشروعك الخاص، يصبح التنقل بين المهام المختلفة أصعب كثيرًا إذا لم تستخدم الفروع. بدلاً من ذلك، عليك تتبع التغييرات يدويًا لتعرف أي منها يتعلق بأي ميزة أو إصلاح.

لكن ما هو الفرع بالضبط؟ اعتبر فرع `main` كطريق سريع، وكل التزام هو سيارة تدخل الطريق. قد تقرر أنك تريد أن تذهب لتفحص شيئًا ما، مثل إنشاء ميزة جديدة، لذا تحتاج إلى الخروج من الطريق السريع لفترة لتفقد الأمر.

الفرع هو في الأساس انحراف عن تاريخك الرئيسي، حيث يمكنك التجربة بحرية مع تغييرات لميزة جديدة أو إصلاح خطأ برمجي. إذا كنت راضيًا عن تغييراتك، يمكنك العودة إلى الطريق السريع بدمج الفرع في `main`. وإذا لم تكن راضيًا، يمكنك حذف الفرع والعودة إلى `main` دون قبول التغييرات.

قبل إنشاء فرع جديد، يمكنك عرض الفروع باستخدام الأمر `git branch`. قد يبدو الناتج هكذا:

```sh
* main
```

الآن لدينا فرع واحد فقط، وهو فرعنا الافتراضي `main`. النجمة (`*`) تعني أن هذا هو الفرع الذي لديك "مفعل" حاليًا. ستتعلم المزيد عن ذلك لاحقًا.

يمكنك إنشاء فرع جديد باستخدام الأمر `git branch` مع معلمة لتحديد اسم الفرع الجديد:

```sh
git branch feature
```

هذا ينشئ فرعًا جديدًا باسم `feature`. لنلق نظرة على فروعنا مرة أخرى باستخدام `git branch`:

```sh
  feature
* main
```

يمكننا الآن رؤية فرعنا الجديد، لكننا ما زلنا على فرع `main`. نحتاج إلى التبديل إلى فرعنا الجديد عن طريق "تفعيله". عند تفعيل فرع جديد، تخبر Git بتحميل الملفات بالحالة التي كانت عليها في تلك اللحظة.

لنفعّل فرعنا الجديد باستخدام:

```sh
git checkout feature
```

أو

```sh
git switch feature
```

وشغّل `git branch` مرة أخرى لترى:

```sh
* feature
  main
```

لاحظ أن النجمة انتقلت الآن إلى الفرع الذي لديك "مفعل" حاليًا.
 
غالبًا عندما تنشئ فرعًا جديدًا، سترغب في التحقق منه فورًا. بدلًا من إنشاء الفرع ثم التحقق منه باستخدام أمرين، يمكنك إنشاء الفرع والتبديل إليه بأمر واحد باستخدام:

```sh
git checkout -b feature
```

أو

```sh
git switch -c feature
```

الآن أنشأنا وتحققنا من فرع جديد. هذا الفرع هو نسخة من فرع `main`. يحتوي على كل الشفرة التي كانت في `main` وقت إنشاء الفرع. الآن، يمكننا إضافة أي شفرة إلى هذا الفرع دون التأثير على فرع `main`. لقد خرجنا فعليًا من الطريق السريع لإنشاء ميزة. إذا تحققت من `git status`، سترى شيئًا مثل هذا:

```sh
On branch feature
nothing to commit, working tree clean
```

أنشئ ملفًا جديدًا باسم `feature.md` بتشغيل هذا الأمر:

```sh
echo "This is our new feature" > feature.md
```

والآن ستبدو الحالة هكذا:

```sh
On branch feature
Untracked files:
  (use "git add <file>..." to include in what will be committed)
	feature.md

nothing added to commit but untracked files present (use "git add" to track)
```

بعدها، حان وقت الالتزام بميزتك الجديدة. أولًا، أضف الملف إلى منطقة الإعداد (staging):

```sh
git add feature.md
```

ثم، التزم بالتغييرات:

```sh
git commit -m "my new feature"
```

وأخيرًا، ادفع فرع `feature` إلى مستودعك:

```sh
git push -u origin feature
```

العلم `-u` يربط فرع `feature` المحلي بالفرع البعيد، بحيث يمكن تنفيذ عمليات الدفع المستقبلية باستخدام `git push` فقط.

بعدها، سنتعلم كيفية عمل طلب سحب لميزتنا الجديدة!
