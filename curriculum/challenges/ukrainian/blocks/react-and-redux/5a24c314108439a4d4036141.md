# --description--

Ця серія завдань знайомить із тим, як використовувати Redux разом із React. Спершу — повторення деяких ключових принципів кожної технології. React — це бібліотека для відтворення інтерфейсу, якій ви надаєте дані, а вона ефективно та передбачувано відтворює вигляд. Redux — це фреймворк для керування станом, який допомагає спростити управління станом вашого застосунку. Зазвичай у застосунку React Redux створюють один Redux store, який керує станом усього застосунку. Ваші React компоненти підписуються лише на ті частини даних у store, які стосуються їхньої ролі. Потім ви виконуєте dispatch дій безпосередньо з React компонентів, що запускає оновлення store.

Хоча React компоненти можуть керувати власним локальним станом, у складних застосунках зазвичай краще зберігати стан застосунку в одному місці за допомогою Redux. Є винятки, коли окремі компоненти можуть мати локальний стан, специфічний лише для них. Нарешті, оскільки Redux не призначений для роботи з React «з коробки», потрібно використовувати пакет `react-redux`. Він надає спосіб передавати Redux `state` і `dispatch` вашим React компонентам як `props`.

У наступних кількох завданнях спершу ви створите простий React компонент, який дозволяє вводити нові текстові повідомлення. Вони додаються до масиву, який відображається у вигляді. Це буде гарне повторення того, що ви вивчили в уроках React. Потім ви створите Redux store і дії, які керують станом масиву повідомлень. Нарешті, ви використаєте `react-redux`, щоб підключити Redux store до вашого компонента, таким чином винісши локальний стан у Redux store.
