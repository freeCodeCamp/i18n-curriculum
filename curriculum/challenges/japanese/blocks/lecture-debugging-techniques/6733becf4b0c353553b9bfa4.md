# --description--

`debugger` ステートメントは、プログラムの特定の行でコードを一時停止し、何が起こっているかを調査できる強力なJavaScriptのツールです。正しく使うと、`debugger` ステートメントは、なぜ何かが期待通りに動作しないのかを理解するための時間を大幅に節約できます。

JavaScriptはコードを上から下へ順に実行します。JavaScriptがコードを実行中に`debugger` ステートメントに到達すると、その行で即座に実行が一時停止します。これにより、変数の確認や関数のチェック、コードの流れ全体を調べることができます。

これによって、何が正しく動いているのか、あるいは間違っているのかを正確に把握できます。ただし、これはブラウザの開発者ツールが開いている場合に限ります。そうでなければ、`debugger` ステートメントは無視され、コードは通常通り実行され続けます。

さらに、`debugger` ステートメントを使うと、最新のブラウザは指定した行でコードの実行を一時停止します。また、再生ボタンをクリックすれば実行を再開できますが、ページ自体は手動でリロードしない限り自動的にリロードされません。

`debugger` ステートメントを使うには、コードの実行を一時停止したい行に配置してください。例を示します。

```js
let firstNumber = 5;
let secondNumber = 10;
debugger; // Code execution pauses here
let sum = firstNumber + secondNumber;
console.log(sum);
```

この例では、コードが実行される前にコンソールを開いていなければ、後からコンソールを開くと`15` が表示されます。コンソールが開いている状態でページをリロードすると、コードの実行が一時停止し、コードを調査できます。

より複雑な例を示します。

```js
function calculateTotalPrice(price, discountPercentage) {
 debugger
 let discountAmount = (price * discountPercentage) / 100
 let totalPrice = price - discountAmount

 console.log(`Original Price: ${price}`)
 console.log(`Discount Amount: ${discountAmount}`)
 console.log(`Total Price after Discount: ${totalPrice}`)

 return totalPrice
}

let price = 100
let discount = 15

let finalPrice = calculateTotalPrice(price, discount)
console.log(`Final Price: ${finalPrice}`);
```

この例では、`debugger` ステートメントが`calculateTotalPrice` の先頭に置かれています。関数が呼び出されると、その関数の実行が一時停止します。

コンソールを開いた状態でページをリロードすると、ページがリロードされ続けることがわかります。これはコードの実行が停止しているのではなく、一時停止しているだけであることを示しています。実行を続けたい場合は、再生ボタンをクリックしてください。

JavaScriptプログラムを作成し続ける中で、`debugger` ステートメントをコードに試してみてください。これにより、実行の流れをよりよく理解し、リアルタイムで変数を調査し、問題がどこで起きているかを素早く特定できます。
