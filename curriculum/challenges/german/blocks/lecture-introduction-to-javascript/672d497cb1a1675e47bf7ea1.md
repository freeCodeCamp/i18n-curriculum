# --interactive--

In JavaScript fungieren Variablen als Container zum Speichern von Daten, auf die Sie im gesamten Programm zugreifen und die Sie ändern können.

Sie können sich Variablen wie Boxen vorstellen, die Werte enthalten. Mit Variablen können Sie Dinge wie Zahlen oder Text verfolgen und auf diese Werte jederzeit in Ihrem Programm verweisen.

Eine Möglichkeit, eine Variable in JavaScript zu deklarieren, ist die Verwendung des Schlüsselworts `let`. Mehr über das Schlüsselwort `let` sowie andere Möglichkeiten, Variablen zu deklarieren, lernen Sie in zukünftigen Lektionen.

Hier ist ein Beispiel für die Verwendung von `let`, um eine Variable namens `age` zu deklarieren:

```js
let age;
```

Im Moment hat die Variable `age` keinen Wert zugewiesen. Wenn Sie versuchen, sie zu verwenden, gibt sie `undefined` zurück, was bedeutet, dass sie keinen Wert hat.

Hier ein Beispiel.

**HINWEIS**: `console.log()` ist eine Funktion, die Informationen in der Konsole ausgibt, einem Teil Ihres Webbrowsers, der zum Debuggen von Code verwendet wird. Mehr über `console.log()` lernen Sie in zukünftigen Lektionen. Außerdem werden die Symbole `//` verwendet, um Kommentare in Ihrem Code hinzuzufügen. Kommentare sind Notizen für Sie selbst oder andere Programmierende, die beim Ausführen des Codes ignoriert werden.

:::interactive_editor
 
```js
let age;
console.log(age); // undefined
```

:::

Um einer Variable einen Wert zuzuweisen, müssen Sie den Zuweisungsoperator wie folgt verwenden:

```js
let age = 25;
```

Wenn Sie nun die Variable `age` verwenden, gibt sie den Wert `25` zurück.

:::interactive_editor

```js
let age = 25;
console.log(age); // 25
```

:::

Der Zuweisungsoperator sieht aus wie ein Gleichheitszeichen (`=`), prüft aber nicht auf Gleichheit. Die richtigen Operatoren zum Prüfen auf Gleichheit lernen Sie in zukünftigen Lektionen kennen.

Der Zuweisungsoperator wird verwendet, um einer Variable einen Wert zuzuweisen. Dieser Vorgang der Wertzuweisung zu einer Variable wird als Initialisierung bezeichnet.

Ein Vorteil der Verwendung des Schlüsselworts `let` zur Deklaration von Variablen ist, dass Sie ihnen Werte neu zuweisen können. In der Programmierung bedeutet Neu-Zuweisung, einer Variable, die bereits einen Wert hat, einen neuen Wert zu geben.

Hier ein Beispiel für die Neu-Zuweisung des Werts für die Variable `age`.

:::interactive_editor

```js
let age = 25;
console.log(age); // 25
age = 30;
console.log(age); // 30
```

:::

Jetzt hält die Variable `age` den Wert `30`. Beachten Sie, dass das Schlüsselwort `let` nicht erneut benötigt wurde, weil die Variable `age` bereits deklariert war und daher nicht ein zweites Mal deklariert werden muss.

Bei der Neu-Zuweisung müssen Sie nur den Variablennamen referenzieren. Neu-Zuweisung ist nützlich, weil sie es Ihnen erlaubt, den in einer Variable gespeicherten Wert während der Programmausführung zu aktualisieren und zu ändern. Ein gutes Beispiel dafür ist das Aktualisieren von Punkten in einem Spiel.

Die Benennung von Variablen scheint einfach, aber es gibt einige Regeln und bewährte Vorgehensweisen, um sicherzustellen, dass Ihr Code lesbar und funktional ist.

Ihre Variablennamen sollten beschreiben, was die Daten darstellen. Anstatt zum Beispiel einen Namen wie `x` zu verwenden, macht ein beschreibenderer Name wie `age` oder `points` Ihren Code leichter verständlich.

```js
// Bad variable names
let x = 10;
let y = "John";

// Good variable names
let age = 10;
```

Variablen in JavaScript müssen mit einem Buchstaben, einem Unterstrich (`_`) oder einem Dollarzeichen (`$`) beginnen. Sie dürfen nicht mit einer Zahl starten.

```js
// Valid variable names
let age;
let _score;
let $total;

// Invalid variable names
let 1stPlace; // starts with a number
```

Variablennamen sind case-sensitiv, das heißt, das Wort `age` in Kleinbuchstaben und das Wort `Age` mit einem Großbuchstaben `A` werden als unterschiedliche Variablen betrachtet.

:::interactive_editor

```js
let age = 25;
let Age = 30;
console.log(age); // 25
console.log(Age); // 30
```

:::

Deshalb ist es wichtig, eine konsistente Benennungskonvention wie CamelCase einzuhalten. CamelCase bedeutet, dass das erste Wort komplett in Kleinbuchstaben geschrieben wird und jedes folgende Wort mit einem Großbuchstaben beginnt.

Hier ein Beispiel für die Verwendung der CamelCase-Benennungskonvention für eine Variable:

```js
let thisIsCamelCase;
let anotherExampleVariable;
let freeCodeCampStudents;
```

Es gibt bestimmte Schlüsselwörter in JavaScript, die Sie nicht als Variablennamen verwenden dürfen, wie `let`, `const`, `function` oder `return`, da sie für die Sprache reserviert sind.

Sie sollten auch vermeiden, Sonderzeichen wie Ausrufezeichen (`!`) oder At-Zeichen (`@`) in Ihren Variablennamen zu verwenden. Es ist am besten, Variablennamen lesbar zu halten, indem Sie Buchstaben, Zahlen, Unterstriche oder Dollarzeichen verwenden.

Wenn Sie diese Richtlinien befolgen, wird Ihr Code sauberer und besser handhabbar, wenn er komplexer wird.
