# --interactive--

У попередньому уроці ви дізналися, як працювати з об’єктом `arguments`, який є подібним до масиву об’єктом, що містить значення аргументів, переданих у функцію.

:::interactive_editor

```js
function logArgs() {
  for (const arg of arguments) {
    console.log(arg);
  }
}

logArgs(1, 2, 3);
// result:
// 1
// 2
// 3
```

:::

Хоча це дійсний спосіб отримати доступ і працювати з довільною кількістю аргументів функції, сучасні застосунки JavaScript використовують синтаксис параметрів залишку.

Ось оновлений приклад із використанням синтаксису параметрів залишку замість об’єкта `arguments`:

:::interactive_editor

```js
function logArgs(...args) {
  for (const arg of args) {
    console.log(arg);
  }
}

logArgs(1, 2, 3);
// result:
// 1
// 2
// 3
```

:::

У цьому оновленому прикладі більше немає прямого звернення до об’єкта `arguments`. Замість цього останній параметр у визначенні функції має три крапки перед ним. Це призводить до того, що цей параметр залишку поміщається в об’єкт `Array`. Ви можете назвати цей параметр залишку як завгодно. Головне, щоб він був останнім параметром у визначенні функції, як тут:

```js
function exampleFunction(a, b, ...restOfArgs) {
  // some code here
}

function anotherFunction(x, y, ...theArgs) {
  // some code here
}
```

Існує кілька додаткових обмежень при роботі з синтаксисом параметрів залишку. Одне з них — у визначенні функції може бути лише один параметр залишку. Тож наступний приклад вважається недійсним.

```js
// Won't work

function badFunction(...args, ...moreArgs) {
  // some code here
}
```

Ще одне обмеження — після параметра залишку не можна ставити коми: `function exampleFunction(a, b, ...restOfArgs, )`.

Ще одне обмеження полягає в тому, що параметр залишку не може мати значення за замовчуванням. Інакше буде викинуто `SyntaxError`.

```js
function badFunction(...args = [1,2]){
  // some code here
}
```

Отже, які ж відмінності між об’єктом `arguments` і параметрами залишку `rest`?

Головна відмінність у тому, що об’єкт `arguments` не є справжнім масивом, тому він не підтримує методи, такі як `includes`, `pop` і `push`. Натомість параметр залишку є екземпляром `Array`. Тож ви можете використовувати вбудовані методи масивів без необхідності спочатку конвертувати його у справжній масив.

:::interactive_editor

```js
function hasCat(...args) {
  return args.includes("cat");
}

console.log(hasCat("dog", "chicken", "cat")); // true
console.log(hasCat("dog", "chicken", "horse")); // false
```

:::
