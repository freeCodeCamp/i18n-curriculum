# --interactive--

يُعد مظهر وسلوك عناصر الإدخال مهمين عند بناء نماذج الويب. النموذج الذي يحتوي على مُدخلات تستجيب لأفعال المستخدم يحسّن تجربة المستخدم كثيرًا، ويجب ألا يسبب ارتباكًا أو إحباطًا.

يوفر CSS عدة فئات مزيفة للمُدخَل يمكنها جعل نماذجك أكثر بديهية وسهولة في الاستخدام. دعنا نلقي نظرة مفصلة على هذه الفئات المزيفة.

تسمح لك الفئة المزيفة `:focus` باستهداف عنصر الإدخال عندما يختاره المستخدم أو ينقر عليه، مما يجذب الانتباه إلى حقل الإدخال النشط. هذا يساعد المستخدمين على تحديد مكان الكتابة الحالي بسهولة:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css" />
<form>
  <input type="text" />
</form>
```

```css
input:focus {
  border: 2px solid crimson;
  outline: none;
}
```

:::

كما يوحي الاسم، تُفعّل الفئة المزيفة `:hover` عندما يضع المستخدم المؤشر فوق عنصر ما. توفر رد فعل بصري قبل تفاعل المستخدم مع الإدخال، مما ينبهه بفعالية إلى أن الإدخال جاهز للعمل.

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css" />
<form>
  <input type="text" />
</form>
```

```css
input:hover {
  background-color: orange;
}
```

:::

في المثال أعلاه، يتغير خلفية عنصر الإدخال إلى اللون البرتقالي عندما يحوم المستخدم فوقه، مما يخبره بأن الحقل جاهز للتفاعل.

تتيح لك الفئة المزيفة `:checked` تنسيق زر اختيار أو مربع اختيار بشكل مختلف عندما يختاره المستخدم. بهذه الطريقة، يمكن للمستخدم رؤية الخيار الذي اختاره بسهولة.

إليك مثالًا مع مربع اختيار للموافقة على الشروط في موقع ويب.

**ملاحظة**: بعض خصائص CSS في هذا المثال لم تُشرح بعد. هذا فقط ليعطيك فكرة عن كيفية إنشاء مربع اختيار مخصص. ستتعلم كيف يعمل كل هذا في الدروس وورش العمل القادمة.

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css" />
<form>
  <label>
  Agree <input class="checkbox" type="checkbox" />
  </label>
</form>
```

```css
.checkbox {
  appearance: none;
  width: 18px;
  height: 18px;
  border: 2px solid #ccc;
  border-radius: 4px;
  display: inline-block;
  position: relative;
  cursor: pointer;
  transition: all 0.25s ease;
  vertical-align: middle; 
}

.checkbox:hover {
  border-color: #888;
}

.checkbox:checked {
  background-color: #4caf50;
  border-color: #4caf50;
}

.checkbox:checked::after {
  content: "";
  position: absolute;
  left: 4px;
  top: 0px;
  width: 5px;
  height: 10px;
  border: solid white;
  border-width: 0 2px 2px 0;
  transform: rotate(45deg);
}

.checkbox:focus {
  outline: 2px solid #90caf9;
  outline-offset: 2px;
}

```

:::

تستهدف الفئة المزيفة `:required` عناصر الإدخال التي تحتوي على الخاصية `required`. تشير إلى أن المستخدم يجب أن يملأ الحقل لإرسال النموذج.

إليك مثالًا مع نموذج يحتوي على بعض حقول الإدخال المطلوبة:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css" />
<form action="#">
  <label for="name">Name:</label>
  <input type="text" id="name" name="name" required />

  <label for="email">Email:</label>
  <input type="email" id="email" name="email" required />

  <label for="phone">Phone Number:</label>
  <input type="tel" id="phone" name="phone" />

  <button type="submit">Submit</button>
</form>
```

```css
input:required {
  border: 2px solid orange;
}
```

:::

عند التحقق من صحة النماذج، يمكنك استخدام الفئة المزيفة `:valid` لتنسيق حقول الإدخال التي تستوفي معايير التحقق، و`:invalid` لتنسيق الحقول التي لا تستوفي المعايير. عادةً ما تستخدم اللون الأخضر للإدخال الصحيح والأحمر للإدخال غير الصحيح.

إليك مثالًا باستخدام الفئة المزيفة `:valid`:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css" />
<form>
  <label for="email">Email:</label>
  <input type="email" id="email" name="email" />
</form> 
```

```css
input:valid {
  border-color: green;
}
```

:::

إليك مثالًا على استخدام الفئة المزيفة `:invalid`:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css" />
<form>  
  <label for="email">Email:</label>
  <input type="email" id="email" name="email" />
</form>
```

```css
input:invalid {
  border-color: crimson;
}
```

:::

تسمح لك الفئة المزيفة `:disabled` باختيار وتنسيق عناصر الإدخال المعطلة. تحتوي هذه العناصر على الخاصية `disabled` التي تمنع المستخدمين من التفاعل معها. عندما يكون الإدخال معطلاً، لا يمكن النقر عليه أو تركيزه أو تعديله.

إليك مثالًا على مسمى وعنصر إدخال معطل.

في CSS، نستهدف هذا الإدخال المعطل ونعطيه لون خلفية `lightgray` ونغير مؤشر المؤشر إلى `not-allowed`.

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css" />
<form>
  <label for="name">Name:</label>
  <input class="text-input" type="text" id="name" name="name" disabled />
</form>
```

```css
.text-input:disabled {
  background-color: lightgray;
  cursor: not-allowed;
}
```

:::

تغير خاصية CSS `cursor: not-allowed;` مظهر المؤشر لتشير إلى أن الإجراء غير مسموح به.

عند تطبيقها على عنصر، تعرض مؤشرًا برمز دائرة مع شريط مائل (عادة دائرة مع خط قطري) لتنبيه المستخدمين أن العنصر غير تفاعلي أو لا يمكن النقر عليه أو التفاعل معه.

فيما يلي بعض الأمثلة الأخرى على فئات مزيفة للمُدخَل وما تقوم به:

- `:autofill`: يطبق أنماطًا على حقول الإدخال التي يملأها المتصفح تلقائيًا بالبيانات المحفوظة.
- `:optional`: ينسق عناصر الإدخال التي ليست مطلوبة ويمكن تركها فارغة.
- `:in-range` و `:out-of-range`: تنسق العناصر بناءً على ما إذا كانت قيمها ضمن أو خارج قيود النطاق المحدد.
