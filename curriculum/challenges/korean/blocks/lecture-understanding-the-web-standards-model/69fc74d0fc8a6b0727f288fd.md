# --description--

오픈 웹은 스스로 작동하지 않습니다. 브라우저가 페이지를 렌더링하거나, 비디오를 재생하거나, 3D 장면을 그릴 때마다, 이는 수년간 엔지니어와 회사 대표들이 작업 그룹에서 정한 지침을 따르고 있는 것입니다.

이 지침을 웹 표준이라고 합니다. 그리고 이를 정하는 그룹을 표준 기구라고 부릅니다. 인터넷의 입법자라고 생각하세요.

오늘날 웹 표준 작업의 대부분을 담당하는 조직에는 **W3C**, **WHATWG**, **ECMA (TC39)**, 그리고 **Khronos Group**이 있습니다. 각 조직은 스택의 다른 계층을 관리하며, 운영 방식도 조금씩 다릅니다.

**웹 하이퍼텍스트 애플리케이션 기술 작업 그룹**인 WHATWG는 HTML과 DOM을 단일한 지속적으로 업데이트되는 "라이빙 스탠다드"로 유지합니다. 버전별 릴리스 대신, 사양이 실시간으로 진화하며, 소수의 강력한 편집 권한을 가진 그룹이 편집합니다. 의사 결정 권한은 Chromium, Gecko, WebKit 같은 주요 브라우저 엔진 팀에 있습니다.

**월드 와이드 웹 컨소시엄**인 W3C는 웹의 주요 표준 기구입니다. 브라우저 공급자, 정부, 연구 기관 등 광범위한 회원을 보유하고 있습니다.

W3C는 CSS, SVG, 접근성 지침, 그리고 수많은 웹 API를 감독합니다. 제안이 공식 표준이 되기 전 여러 검토 단계를 거치는 느리고 합의 중심의 절차를 통해 작업합니다.

**Ecma International**(이전의 유럽 컴퓨터 제조업체 협회, ECMA)은 원래 컴퓨팅과 통신 기술에 초점을 맞춘 범용 표준 기구입니다. 현재는 JavaScript 구현의 기반이 되는 ECMAScript 표준을 공식적으로 게시합니다. 언어 표준의 기술 개발은 Ecma의 기술 위원회 39(**TC39**)가 담당하며, Ecma International은 조직적 틀, 게시 절차, 표준 관리 역할을 합니다.

이 위원회는 Google, Apple, Mozilla, Microsoft 등 대표들이 약 두 달마다 모입니다. 새로운 언어 기능은 5단계 제안 과정을 거치며, 0단계의 대략적인 스케치에서 4단계의 완전 명세 및 구현까지 진행됩니다. async/await, optional chaining, ES 모듈 같은 기능들이 이 과정을 거쳐 JavaScript 언어에 포함되었습니다.

**Khronos Group**은 그래픽, 컴퓨트, 미디어 표준에 집중하는 회원 기반 컨소시엄입니다. NVIDIA, AMD 같은 GPU 제조사 대표들과 브라우저 공급자가 회원으로 참여합니다. 웹에서는 GPU 가속 3D 렌더링을 브라우저에 제공하는 WebGL과 WebGL 2가 가장 중요한 기여입니다.
