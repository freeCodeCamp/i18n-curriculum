# --description--

Chrome開発者ツールを使って、Interaction to Next Paint（INP）を実際に測定する方法を見ていきましょう。

まずは、「F12」を押すか、右クリックして「検証」をクリックして開発者ツールを開きます。

例えば、あなたのウェブページに検索バーがあるとします。

ユーザーがクエリを入力してから、サジェストのドロップダウンが表示されるまでに600msかかると想像してください。Chrome開発者ツールはこの高いINP値を示します。

重いJavaScriptを遅延させたり、長いタスクを減らしたり、イベント処理を改善したりして最適化できます。そうするとINPは150msまで下がり、操作が瞬時に感じられます。

すごいですよね？

では、操作の遅延を特定して、より速くスムーズなユーザー体験を作るのに役立つ実践的な測定手法を見ていきましょう！

まず、Chrome開発者ツールを開きます。もう一度言うと、Google Chromeを開き、分析したいウェブページに移動して、「F12」を押すか、ページを右クリックして「検証」をクリックします。次に「パフォーマンス」タブに移動しましょう。ここに必要なすべてがあります。

これでInteraction to Next Paint（INP）を特定する準備ができました。記録された最長の操作遅延を探します。良いINPは一般的に200ms以下で、悪いINPは500ms以上です。以上です！
