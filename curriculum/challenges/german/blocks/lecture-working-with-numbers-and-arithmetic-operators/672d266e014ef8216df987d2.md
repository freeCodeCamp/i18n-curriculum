# --interactive--

Der `Number`-Typ ist einer der am häufigsten verwendeten Datentypen in JavaScript und anderen Programmiersprachen. Zahlen wirken vielleicht einfach, aber es gibt viel zu entdecken, wenn es um Zahlen in JavaScript geht. Schauen wir uns das also genauer an. In JavaScript stellt der `Number`-Datentyp einen numerischen Wert dar.

Im Gegensatz zu vielen anderen Programmiersprachen, die ganze Zahlen und Gleitkommazahlen in verschiedene Typen aufteilen, verwendet JavaScript einen einheitlichen `Number`-Typ für Zahlen. Das bedeutet, Sie können mit ganzen Zahlen, Dezimalzahlen und sogar speziellen numerischen Werten alle unter demselben `Number`-Datentyp arbeiten.

Hier ist ein einfaches Beispiel, das zeigt, dass ganze Zahlen, Gleitkommazahlen und negative Zahlen alle vom Typ number sind:

:::interactive_editor

```js
const wholeNumber = 50;
const decimalNumber = 4.5;
const negativeNumber = -7;

console.log(typeof wholeNumber); // number
console.log(typeof decimalNumber); // number
console.log(typeof negativeNumber); // number
```

:::

Der `Number`-Typ von JavaScript umfasst verschiedene Arten von numerischen Werten, von einfachen ganzen Zahlen und Gleitkommazahlen bis hin zu Spezialfällen wie `Infinity` und `NaN` oder „Not a Number“. Schauen wir uns die Haupttypen an, denen Sie begegnen werden. Ganze Zahlen sind Zahlen ohne Bruch- oder Dezimalanteil. Sie können positiv, negativ oder null sein. Hier sind einige Beispiele:

:::interactive_editor

```js
const positiveInteger = 100;
const negativeInteger = -25;
const zero = 0;

console.log(typeof positiveInteger); // number
console.log(typeof negativeInteger); // number
console.log(typeof zero); // number
```

:::

Gleitkommazahlen sind Zahlen mit Dezimalstellen. JavaScript-Entwickler nennen sie oft einfach „Floats“. Floats sind nützlich, wenn Sie mehr Genauigkeit benötigen, zum Beispiel bei Messungen oder Währungen. Hier sind einige Beispiele:

:::interactive_editor

```js
const floatingPointNumber = 4.5;
const anotherFloat = 89.56;
const oneMoreFloat = 16.462;

console.log(typeof floatingPointNumber); // number
console.log(typeof anotherFloat); // number
console.log(typeof oneMoreFloat); // number
```

:::

JavaScript kann Zahlen darstellen, die über das maximale Limit hinausgehen, mit `Infinity`. Dies tritt auf, wenn Sie versuchen, eine Zahl durch null zu teilen oder in seltenen Fällen die obere Grenze des `Number`-Typs zu überschreiten. Hier ist ein Beispiel:

:::interactive_editor

```js
const infiniteNumber = 1 / 0;
console.log(infiniteNumber); // Infinity
console.log(typeof infiniteNumber); // number
```

:::

Manchmal führen mathematische Operationen in JavaScript nicht zu einer gültigen Zahl. Wenn Sie zum Beispiel versuchen, eine mathematische Operation mit etwas auszuführen, das keine Zahl ist, erhalten Sie `NaN`, was für „Not a Number“ steht:

:::interactive_editor

```js
const notANumber = 'hello world' / 2;
console.log(notANumber); // NaN
```

:::

Überraschenderweise ist der Typ von `NaN` ebenfalls `Number`:

:::interactive_editor

```js
const notANumber = 'hello world' / 2;
console.log(typeof notANumber); // number
```

:::

Neben dem Standard-Dezimalsystem (Basis 10) unterstützt JavaScript auch Zahlen in anderen Basen wie Binär, Oktal und Hexadezimal. Binär ist ein Basis-2-System, das nur die Ziffern 1 und 0 verwendet. Oktal ist ein Basis-8-System, das nur die Ziffern 0 bis 7 verwendet. Hexadezimal ist ein Basis-16-System, das die Ziffern 0 bis 9 und die Buchstaben a bis f verwendet, wie Sie es bei CSS-Hex-Farben sehen.
