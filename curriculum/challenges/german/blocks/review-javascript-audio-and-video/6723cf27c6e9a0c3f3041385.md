## `Audio` Konstruktor und gängige Methoden

- **Definition**: Der `Audio`-Konstruktor ist wie andere Konstruktoren eine spezielle Funktion, die mit dem Schlüsselwort `new` aufgerufen wird. Er gibt ein `HTMLAudioElement` zurück, das Sie dann verwenden können, um Audio für den Benutzer abzuspielen oder es dem DOM hinzuzufügen, damit der Benutzer es selbst steuern kann. Wenn Sie den Konstruktor aufrufen, können Sie optional eine URL als (einziges) Argument übergeben. Diese URL sollte auf die Quelle der Audiodatei zeigen, die Sie abspielen möchten. Oder, wenn Sie die Quelle dynamisch ändern müssen, können Sie die URL der `src`-Eigenschaft des zurückgegebenen Audioelements zuweisen.
- **`play()`-Methode**: Diese Methode wird mit den Elementen `audio` oder `video` verwendet, um die Wiedergabe der Medien zu starten.

:::interactive_editor

```html
<audio id="audio" controls>
  <source src="https://www.w3schools.com/html/horse.mp3" type="audio/mpeg">
</audio>

<button id="playBtn">Play Audio</button>

<script>
  const audio = document.getElementById("audio");
  const playBtn = document.getElementById("playBtn");

  playBtn.addEventListener("click", () => {
    audio.play();
  });
</script>
```

:::

- **`pause()`-Methode**: Diese Methode wird mit den Elementen `audio` oder `video` verwendet, um die Wiedergabe der Medien zu pausieren. Sie behält die aktuelle Position bei, sodass bei erneutem Aufruf von `play()` die Wiedergabe an dieser Stelle fortgesetzt wird.

:::interactive_editor

```html
<audio id="myAudio" controls>
  <source src="https://www.w3schools.com/html/horse.mp3" type="audio/mpeg">
</audio>

<br>

<button id="pauseBtn">Pause Audio</button>

<script>
  document.addEventListener("DOMContentLoaded", () => {
    const audio = document.getElementById("myAudio");
    const pauseBtn = document.getElementById("pauseBtn");

    pauseBtn.addEventListener("click", () => {
      audio.pause();
    });
  });
</script>
```

:::

- **`addTextTrack()` Methode**: Diese Methode erlaubt es Ihnen, eine Textspur mit dem Medienelement zu verknüpfen – was besonders hilfreich ist, um Untertitel zu einem Video hinzuzufügen.
- **`fastSeek()` Methode**: Diese Methode erlaubt es Ihnen, die Wiedergabeposition auf eine bestimmte Zeit innerhalb der Medien zu verschieben.

Der `Audio()`-Konstruktor hat ebenso Eigenschaften wie Methoden. Zu diesen Eigenschaften gehören:

- `currentTime`: um die aktuelle Wiedergabezeit eines Audios zu erhalten
- `loop`: um das Audio kontinuierlich abzuspielen, indem es automatisch neu startet, wenn es das Ende erreicht
- `muted`: um die Audioausgabe eines Medienelements stummzuschalten, unabhängig von der Lautstärkeeinstellung

## Verschiedene Audio- und Videoformate

- **MIME-Typ**: Ein MIME-Typ, kurz für Multipurpose Internet Mail Extensions, ist eine standardisierte Möglichkeit, programmgesteuert einen Dateityp anzugeben. Der MIME-Typ kann einer Anwendung, wie Ihrem Browser, mitteilen, wie eine bestimmte Datei behandelt werden soll. Im Fall von Audio und Video zeigt der MIME-Typ an, dass es sich um ein Multimediaformat handelt, das in die Webseite eingebettet werden kann.
- **`source`-Element**: Dieses wird verwendet, um einen Dateityp und eine Quelle anzugeben – und kann mehrere verschiedene Typen enthalten, indem mehrere Source-Elemente verwendet werden. Wenn Sie dies tun, bestimmt der Browser das beste Format für die aktuelle Umgebung des Benutzers.
- **MP3**: Dies ist ein digitaler Dateityp, der verwendet wird, um Musik, Audio oder Ton zu speichern. Es handelt sich um eine komprimierte Version einer Tonaufnahme, die die Dateigröße verkleinert, sodass sie leichter gespeichert und geteilt werden kann. MP3 hat die breiteste Browserunterstützung und den MIME-Typ `audio/mpeg`.
- **MP4**: Ein MP4 ist ein digitaler Dateityp, der verwendet wird, um Video und Audio zu speichern. Er dient als Container, der sowohl das Video (Bilder) als auch den Ton (Musik oder Sprache) in einer Datei enthält. Ein MP4 kann den MIME-Typ `audio/mp4` ODER `video/mp4` haben, je nachdem, ob es sich um eine Videodatei oder nur Audio handelt.
- Weitere Formate sind `WMV`, das mit der Windows Media Player-Anwendung verbunden ist, `OGG`, `MKV`, `AVI` und mehr.

## Codecs

- **Definition**: Ein Codec, kurz für „Encoder/Decoder“, ist ein Algorithmus oder eine Software, die Audio und Video zwischen analogen und digitalen Formaten konvertieren kann. Codecs können als Teil des MIME-Typs angegeben werden. Die grundlegende Syntax zur Definition eines Codecs besteht darin, nach dem Medientyp ein Semikolon zu setzen, dann `codecs=` und den Codec.

## HTMLMediaElement-API

- **Definition**: Die `HTMLMediaElement`-API wird verwendet, um das Verhalten von Audio- und Videoelementen auf Ihrer Seite zu steuern. Sie erweitert das Basisinterface `HTMLElement`, sodass Sie Zugriff auf die Basiseigenschaften, -methoden sowie hilfreiche Ereignisse wie `waiting`, `ended`, `canplay`, `canplaythrough` und mehr haben. Beispiele für Methoden sind `play()`, `fastSeek()`, `pause()` und `canPlayType()`, mit denen Sie überprüfen können, ob ein Browser wahrscheinlich eine Audiodatei abspielen kann.

## Media-Capture- und Streams-API

- **Definition**: Die Media Capture and Streams API, oder MediaStream API, wird verwendet, um Audio und Video von Ihrem Gerät aufzunehmen. Um die API zu verwenden, müssen Sie das `MediaStream` Objekt erstellen. Sie könnten dies mit dem Konstruktor tun, aber es wäre nicht an die Hardware des Benutzers gebunden. Stattdessen hat die `mediaDevices` Eigenschaft des `global` Navigator-Objekts eine `getUserMedia()` Methode, die Sie verwenden können.

```js
window.navigator.mediaDevices.getUserMedia({
  audio: true,
  video: {
    width: {
      min: 1280,
      ideal: 1920,
      max: 3840
    },
    height: {
      min: 720,
      ideal: 1080,
      max: 2160
    }
  }
});
```

## Bildschirmaufnahme-API

- **Definition**: Die Screen Capture API wird verwendet, um den Bildschirm eines Benutzers aufzuzeichnen. Diese API wird durch den Aufruf der `getDisplayMedia()` Methode des `mediaDevices` Objekts verfügbar gemacht und verwendet den zurückgegebenen Medienstream.

## MediaStream-Aufnahme-API

- **Definition**: Die MediaStream Recording API arbeitet zusammen mit den MediaStreams APIs und ermöglicht es Ihnen, einen MediaStream (oder sogar direkt ein `HTMLMediaElement`) aufzuzeichnen.

## Media Source Extensions API

- **Thema**: Die Media Source Extensions API ermöglicht es Ihnen, den Webcam-Feed eines Benutzers direkt mit der `srcObject`-Eigenschaft an ein Videoelement weiterzugeben.

## Web Audio API

- **Definition**: Die Web Audio API ist die Grundlage für alles, was im Web hörbar ist. Diese API umfasst wichtige Objekte wie ein `AudioBuffer` (das speziell einen Puffer mit Audiodaten darstellt) oder den `AudioContext`, der verwendet wird, um einen Audioverarbeitungsgraphen darzustellen.
