# --description--

Cuando visitas un sitio web, lo primero que haces es escribir una dirección web en tu navegador.

Por ejemplo, para ir a freeCodeCamp, ingresarías `freecodecamp.org` en la barra de direcciones de tu navegador.

Esto se conoce como un nombre de dominio. Es un identificador único legible para humanos de ese recurso en internet.

Los nombres de dominio están muy relacionados con el concepto de DNS, que es el tema principal de esta lección. Comencemos hablando un poco sobre la jerarquía de nombres de dominio, para que puedas entender cómo funciona un DNS detrás de escena.

La jerarquía de nombres de dominio se lee de derecha a izquierda, moviéndose desde la categoría más general hasta la más específica.

La estructura principal de un nombre de dominio es, de derecha a izquierda:

* **Dominio de nivel superior (TLD):** el último segmento del nombre de dominio, ubicado a la derecha. En el caso de `freecodecamp.org`, eso sería `.org`. Otros ejemplos incluyen `.com` y `.edu`.
    
* **Dominio de Segundo Nivel (SLD):** el nombre registrado por la persona u organización. Ese sería `freecodecamp` en `freecodecamp.org`. Esto es lo que usualmente notas primero cuando ves una dirección web.
    
* **Dominio de tercer nivel (subdominio):** se usa para identificar una sección específica del dominio principal. Por ejemplo, `www`, que significa “World Wide Web.” Hoy en día, los servidores web modernos manejan automáticamente el enrutamiento de dominios, por lo que puedes acceder a un sitio web con o sin ingresar `www`. Otros ejemplos de subdominios incluyen `blog.`, `shop.`, `support.` y `api.`.
    

Los nombres de dominio nos son muy útiles a los humanos porque son fáciles de recordar y escribir en un navegador. Sin embargo, como sabes, las computadoras trabajan con números. Tu navegador tiene que transformar el nombre de dominio que ingresaste en algo conocido como una dirección IP para encontrar el sitio web que estás buscando.

Una **dirección IP** es una secuencia única de números que identifica una computadora en una red, como internet.

Las direcciones IP son esenciales para permitir que los clientes envíen solicitudes a los servidores correctos y obtengan las respuestas adecuadas. Un ejemplo de una dirección IPv4 es `127.0.0.1`. La dirección IP equivalente en un estándar más reciente llamado IPv6 sería `0:0:0:0:0:0:0:1`.

Hay mucho que aprender sobre las direcciones IP, y puede que quieras explorarlas con más profundidad a medida que sigas aprendiendo. Pero por ahora, solo necesitas entender que son las direcciones numéricas únicas que se usan para identificar computadoras en una red.

Desde los conceptos de nombres de dominio y direcciones IP, ahora podemos llegar al concepto de DNS.

DNS significa **Domain Name System**. Esto suele referirse como la “guía telefónica” de internet.

El DNS traduce el nombre de dominio que escribes en tu navegador a su dirección IP correspondiente. Es un sistema de nombres jerárquico y descentralizado para computadoras y dispositivos que están conectados a internet o a una red específica.

Cuando el usuario ingresa un nombre de dominio en el navegador, como `freecodecamp.org`, el DNS asigna ese nombre a la dirección IP de su servidor correspondiente. Una vez que se ha identificado el servidor, la solicitud del cliente se envía al servidor. El servidor maneja esa solicitud según cómo fue programado y finalmente envía una respuesta al cliente. Esto completa el ciclo de solicitud-respuesta.

Todo esto es posible gracias al DNS que realizó esa conversión inicial a la dirección IP objetivo.

Vamos a profundizar en los detalles del proceso de resolución DNS.

## El Proceso de Resolución DNS

* La resolución DNS funciona enviando una consulta a través de una cadena de servidores DNS.

* Cuando tú ingresas un nombre de dominio en el navegador, el **cliente** envía una solicitud a un servidor DNS local. Este servidor DNS local se conoce como el **Recursive Resolver**. Generalmente, esto es gestionado por tu proveedor de servicios de internet (ISP).
    
* El Recursive Resolver reenvía esa solicitud a los **Root Servers**. Estos son servidores generales que no tienen información directa sobre direcciones IP. Funcionan dirigiendo el Recursive Resolver a un servidor más específico, específicamente al Top-Level Domain (TLD) Server para el dominio de nivel superior particular de la solicitud. Por ejemplo, el servidor responsable de todos los dominios `.org`.
    
* El Resolver Recursivo luego consulta el **Servidor de Dominio de Nivel Superior (TLD)**. Este servidor devuelve la dirección del Servidor de Nombres Autoritativo específico que contiene los registros para el dominio solicitado (por ejemplo, `freecodecamp.org`).
    
* El Recursive Resolver inicia una consulta final al correspondiente **Authoritative Name Server.** Si se encuentra el **DNS record**, se devuelve al Recursive Resolver.
    
* Cuando el Recursive Resolver recibe la dirección IP, la entrega al cliente, para que puedas comenzar a navegar por el sitio web.

El Servidor de Nombres Autoritativo devuelve **Registros DNS**, no solo direcciones IP. Los Registros DNS son entradas de texto estructuradas que contienen información importante sobre un dominio, como información de subdominios, cuánto tiempo debe almacenarse en caché la información y la información necesaria para realizar el servicio solicitado.

Para optimizar el proceso, el Recursive Resolver almacena temporalmente en caché el DNS Record para acelerar el proceso. De esta manera, no estarás haciendo la misma solicitud repetidamente en un corto período de tiempo. Eso hace que el proceso general sea mucho más eficiente.

Como puedes ver, los servidores DNS son esenciales porque proporcionan el paso inicial y crítico de traducir nombres de dominio en direcciones IP. Este proceso de traducción permite que cada cliente localice y se conecte al servidor correcto. Sin este sistema de directorio, el modelo cliente-servidor, la columna vertebral de la web moderna, no sería funcional.
