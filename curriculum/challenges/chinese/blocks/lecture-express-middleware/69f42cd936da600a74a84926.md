# --description--

中间件是 Express 中的一个核心概念，它允许我们在客户和服务器之间处理请求和响应。中间件函数可以访问请求对象、响应对象以及应用请求-响应周期中的下一个函数。

让我们来拆解一下：

## 什么是中间件？

中间件本质上是在请求服务器的生命周期中执行的函数。它执行各种任务，比如记录请求日志、处理误差、解析传入请求，或者提供静态文件。

Express 提供了一种方便的方式来定义中间件，处理传入请求，在它们到达路由处理程序之前。

中间件的工作原理：

当向 Express 应用发出请求时，中间件会按照添加的顺序执行。流程如下：

* 请求发送到服务器。
* 中间件函数处理请求。
* 如果中间件调用 `next()`，它会将控制权传递给下一个中间件或路由处理程序。
* 如果中间件不调用 `next()`，请求-响应周期将停止。

```js
const express = require('express')
const app = express()

// Simple middleware that logs request info
app.use((req, res, next) => {
  console.log('Request URL:', req.url)
  console.log('Request Method:', req.method)
  next() // Passes control to the next middleware
})

app.get('/', (req, res) => {
  res.send('Hello World!')
})

app.listen(3000, () => {
  console.log('Server running on http://localhost:3000')
})
```

`app.use()`：为你的应用全局添加中间件。

在这个例子中，中间件记录请求详情并调用 `next()` 以继续执行下一个中间件或路由处理程序。

中间件是 Express 中一个强大的特色，它允许你在不同阶段拦截和修改请求。你可以使用中间件来处理日志记录、认证，甚至在请求和响应对象到达路由处理程序之前修改它们。
