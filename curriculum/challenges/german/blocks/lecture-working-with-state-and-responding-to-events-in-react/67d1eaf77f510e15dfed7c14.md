# --description--

State ist eines der wichtigsten Grundlagen von React und anderen Frontend-Frameworks. Es ist wie das Gehirn einer Komponente, das heißt, es hält Informationen, die sich im Laufe der Zeit ändern können, und steuert, wie sich die Komponenten verhalten und aussehen.

Schauen wir uns an, was State ist und wie der `useState`-Hook Ihnen erlaubt, damit zu arbeiten.

State stellt die dynamischen Daten in Ihrer React-Komponente dar, wie zum Beispiel den Wert einer Benutzereingabe, Daten, die von einer API abgerufen wurden, oder ein Element in einer To-do-Liste.

Wann immer sich der State ändert, rendert React die Komponente neu, ohne die Seite neu zu laden, um diese Änderungen in der Benutzeroberfläche widerzuspiegeln. Diese Reaktivität macht Ihre App interaktiv.

Der `useState`-Hook ist eine Funktion, mit der Sie State-Variablen in funktionalen Komponenten deklarieren können.

Vor Hooks konnten Sie State nur in Klassenkomponenten verwenden. Aber mit der Einführung der Hooks seit React 16.8 können Sie State in funktionalen Komponenten mit dem `useState`-Hook verwenden.

Um den `useState`-Hook zu verwenden, müssen Sie ihn aus React importieren:

```js
import { useState } from "react";
```

Sie können auch React selbst importieren und erhalten Zugriff auf den `useState`-Hook als Eigenschaft:

```js
import React from "react";
```

So können Sie eine State-Variable deklarieren, wenn Sie `useState` importieren:

```js
const [stateVariable, setStateFunction] = useState(initialValue);
```

Und so deklarieren Sie eine State-Variable, wenn Sie React importieren:

```js
const [stateVariable, setStateFunction] = React.useState(initialValue);
```

In der State-Variable haben Sie Folgendes:

- `stateVariable` hält den aktuellen State-Wert
- `setStateFunction` (die Setter-Funktion) aktualisiert die State-Variable
- `initialValue` setzt den Anfangs-State

Beachten Sie, dass der State in einer React-Komponente privat ist und auf jede Komponenteninstanz isoliert ist. Das bedeutet, wenn Sie dieselbe Komponente zweimal rendern, beeinflusst der State der einen Komponente nicht die andere. Das bedeutet auch, dass Sie, wenn Sie State zwischen Komponenten teilen möchten, den State zu einem gemeinsamen Elternteil hochheben und als Props weitergeben müssen.

Ein weiterer Punkt ist, dass Hooks immer auf der obersten Ebene einer Komponente aufgerufen werden müssen, direkt vor dem `return`-Schlüsselwort, um den State und die Effekte über die Renderzyklen hinweg konsistent zu halten. Das bedeutet, Sie können State nicht innerhalb von Schleifen, Bedingungen oder verschachtelten Funktionen verwenden.

Hier ist ein Beispiel, wie Sie State mit dem `useState`-Hook in einer `Counter`-Komponente verwalten:

```jsx
// Importing the useState hook
import { useState } from "react";

function Counter() {
  const initialValue = 0;

  // The state variable and setter function
  const [count, setCount] = useState(initialValue);

  return (
    <div>
      {/* Display current state value */}
      <h2>{count}</h2>

      <button onClick={() => setCount(count - 1)}>Decrement</button>
      <button onClick={() => setCount(count + 1)}>Increment</button>
    </div>
  );
}

export default Counter;
```

Im obigen Code haben wir den `useState`-Hook aus React importiert. In der `Counter`-Komponente stellt `count` den aktuellen State dar, während `setCount` die Set-Funktion ist, die für die Aktualisierung des States verantwortlich ist. Der aktuelle State-Wert ist `0`. Die `return`-Anweisung enthält den Zähler und zwei Buttons, um den Zähler um `1` zu verringern oder zu erhöhen.

Sie können mehrere State-Werte verwalten, indem Sie den `useState`-Hook mehrfach aufrufen. Das ist besonders wichtig, wenn Sie unabhängige State-Variablen haben:

```jsx
function UserProfile() {
  const [isOnline, setIsOnline] = useState(false);
  const [notifications, setNotifications] = useState(0);

  // The rest of the component logic
}
```

Sie können den `useState`-Hook auch mehrfach aufrufen, wenn Sie mehrere States verwalten, die separat aktualisiert werden, wie Formularfelder:

```jsx
function SignUpForm() {
  const [name, setName] = useState("");
  const [username, setUsername] = useState("");
  const [email, setEmail] = useState("");

  // The rest of the component logic
}
```

In diesem Fall ist es jedoch besser, die States zu kombinieren, da sie alle Teil desselben Formulars sind:

```jsx
function SignUpForm() {
  const [formData, setFormData] = useState({
    name: "",
    username: "",
    email: "",
  });

  // The rest of the component logic
}
```

So funktioniert State und so können Sie den `useState`-Hook verwenden.
