### --question--

#### --text--

HTTP 通訊與 WebSocket 通訊有何不同？

#### --distractors--

HTTP 是雙向的，而 WebSocket 是單向的。

---

HTTP 保持永久連線，而 WebSocket 每次交換後關閉。

---

HTTP 和 WebSocket 都遵循相同的請求-回應模式。

#### --answer--

HTTP 採用 `Client --> Request --> Server --> Response`，而 WebSocket 是 `Client <--> Server`。

### --question--

#### --text--

Pub/Sub 的訊息流程是什麼？

#### --distractors--

發布者發送給訂閱者 --> 訂閱者轉發給主題 --> 主題儲存訊息。

---

主題發送訊息 --> 發布者接收 --> 發布者轉發給訂閱者。

---

訂閱者連接發布者 --> 發布者發送資料 --> 訂閱者確認收到。

#### --answer--

訂閱者訂閱主題 --> 發布者向該主題發送訊息 --> 訊息系統將訊息傳送給該主題的所有訂閱者。

以下哪項正確描述 WebSocket 連線流程？

#### --distractors--

客戶端透過 TCP 連線 --> 伺服器傳送資料 --> 客戶端接收 --> 連線關閉。

---

客戶端傳送資料 --> 伺服器請求升級 --> 連線開啟 --> 客戶端確認。

---

伺服器主動發起連線 --> 客戶端接受 --> 雙方交換資料 --> 連線關閉。
