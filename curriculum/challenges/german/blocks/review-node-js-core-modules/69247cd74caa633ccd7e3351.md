## Einführung in die Node.js-Core-Module

- **Node.js-Core-Module**: Eingebaute Module, die mit Node.js vorinstalliert sind und grundlegende Funktionen für Dateioperationen, Kryptografie, Betriebssysteminteraktionen und mehr bereitstellen.
- **Standardmodule**: Müssen nicht separat installiert werden; sind verfügbar, solange Node.js installiert ist.
- **Import-Syntax**: Verwenden Sie `require()`, um Core-Module in Ihre Anwendung zu importieren.

```javascript
const fs = require("fs");
const crypto = require("crypto");
const os = require("os");
```

## File-System-(fs)-Modul

- **Zweck**: Bietet Methoden zum Arbeiten mit Dateien und Ordnern, einschließlich Öffnen, Schließen, Lesen, Schreiben und Löschen.
- **Import**: `const fs = require("fs");`
- **Flexibilität**: Methoden sind in drei Varianten verfügbar: Callback, Promise und synchron.

### Muster für Methodenaufrufe

- **Asynchron mit Callbacks**: Standardverhalten, nicht blockierend.

```javascript
fs.writeFile("filePath", "content", "utf8", (err) => {
  if (err) throw err;
  console.log("File written to!");
});
```

- **Promise-basiert**: Bevorzugt für moderne Anwendungen, verwendet `async/await`.

```javascript
const fs = require("fs/promises");

async function writeToFile() {
  try {
    await fs.writeFile("article.md", "## Node fs Module");
    console.log("File written to!");
  } catch (err) {
    console.error(err);
  }
}
```

- **Synchron**: Blockierende Operation, geeignet für einfache Skripte.

```javascript
try {
  fs.writeFileSync("article.md", "## Node fs Module", "utf8");
  console.log("File written to!");
} catch (err) {
  console.error(err);
}
```

### Wichtige Dateioperationen

- **`writeFile()`-Methode**: Erstellt oder überschreibt eine Datei mit angegebenem Inhalt.

```javascript
await fs.writeFile("article.md", "## Node fs Module: The Complete Guide");
```

- **`appendFile()`-Methode**: Fügt Inhalt am Ende einer bestehenden Datei hinzu.

```javascript
await fs.appendFile("article.md", "\n\nIn this article, you will learn...");
```

- **`readFile()`-Methode**: Liest den Inhalt einer Datei.

```javascript
const content = await fs.readFile("article.md", "utf8");
console.log("File content:", content);
```

- **`unlink()`-Methode**: Löscht eine Datei aus dem Dateisystem.

```javascript
await fs.unlink("article.md");
console.log("File deleted successfully");
```

### Zeichencodierung

- **UTF-8-Codierung**: Geben Sie `"utf8"` an, um Dateien als Textstrings zu lesen.
- **Pufferformat**: Ohne Codierungsangabe werden Dateien als binäre Pufferobjekte gelesen.

```javascript
// Returns string
const textContent = await fs.readFile("file.txt", "utf8");

// Returns Buffer
const bufferContent = await fs.readFile("file.txt");
```

## Buffer-Modul

- **Zweck**: Verarbeitet binäre Daten direkt im Speicher für Dateien, Bilder und Netzwerkstreams.
- **Import**: `const { Buffer } = require("buffer");`
- **Anwendungsfälle**: Datei-I/O, TCP-Streams, Bildverarbeitung und Manipulation binärer Daten.

### Buffer-Erstellung

- **`Buffer.from()`-Methode**: Erstellt Puffer aus Strings, Arrays oder anderen Daten.

```javascript
// From string
const myStrBuffer = Buffer.from("freeCodeCamp");
console.log(myStrBuffer); // <Buffer 66 72 65 65 43 6f 64 65 43 61 6d 70>

// From array of numbers
const myNumBuffer = Buffer.from([70, 82, 69, 69, 67, 79, 68, 69, 67, 65, 77, 80]);
```

- **`Buffer.alloc()`-Methode**: Erstellt Puffer einer angegebenen Größe, gefüllt mit Nullen.

```javascript
const someBuffer = Buffer.alloc(10);
console.log(someBuffer); // <Buffer 00 00 00 00 00 00 00 00 00 00>
```

### Buffer-Operationen

- **Array-ähnlicher Zugriff**: Greifen Sie mit Indexnotation auf einzelne Puffer-Elemente zu.

```javascript
console.log(myStrBuffer[0]); // 102 (byte value for 'f')
```

- **`toString()`-Methode**: Wandelt Puffer in lesbaren String um.

```javascript
console.log(myStrBuffer.toString()); // "freeCodeCamp"
```

- **`Buffer.write()`-Methode**: Schreibt Daten in einen zugewiesenen Puffer.

```javascript
someBuffer.write("Hello fCC");
console.log(someBuffer.toString()); // "Hello fCC"
```

- **`Buffer.byteLength()`-Methode**: Gibt die Anzahl der Bytes zurück, die zum Speichern eines Strings benötigt werden.

```javascript
console.log(Buffer.byteLength("Hello freeCodeCamp")); // 18
```

### Weitere Buffer-Methoden

- **`Buffer.isBuffer()`**: Prüft, ob ein Objekt ein Puffer ist.
- **`Buffer.compare()`**: Vergleicht zwei Puffer und gibt die Sortierreihenfolge zurück.
- **`Buffer.concat()`**: Verbindet mehrere Puffer zu einem.

## Crypto-Modul

- **Zweck**: Bietet kryptografische Funktionen wie Hashing, Verschlüsselung, Entschlüsselung und digitale Signaturen.
- **Import**: `const crypto = require("crypto");`
- **Sicherheitshinweis**: Verwenden Sie gut getestete Bibliotheken wie `bcrypt` oder `jsonwebtoken` für die Authentifizierung in der Produktion.

### Hashing-Methoden

- **`createHash()`-Methode**: Erstellt Einweg-Hashes mit Algorithmen wie SHA256, SHA512 oder MD5.

```javascript
const hashedPassword = crypto
  .createHash("sha256")
  .update("myPassword123")
  .digest("hex");

console.log("createHash result:", hashedPassword);
```

- **`createHmac()`-Methode**: Erstellt einen Hash mit einem geheimen Schlüssel für Authentifizierung und Datenintegrität.

```javascript
const hashedMessage = crypto
  .createHmac("sha256", "secret-key")
  .update("Hello World")
  .digest("hex");
```

### Verschlüsselung und Entschlüsselung

- **`createCipheriv()`- und `createDecipheriv()`-Methoden**: Verschlüsseln und entschlüsseln Daten mit Algorithmen, Schlüsseln und Initialisierungsvektoren.

```javascript
const key = Buffer.from("12345678901234567890123456789012"); // 32 bytes for AES-256
const iv = Buffer.from("1234567890123456"); // 16 bytes for AES

// Encryption
const cipher = crypto.createCipheriv("aes-256-cbc", key, iv);
let encrypted = cipher.update("Hello campers!", "utf8", "hex");
encrypted += cipher.final("hex");

// Decryption
const decipher = crypto.createDecipheriv("aes-256-cbc", key, iv);
let decrypted = decipher.update(encrypted, "hex", "utf8");
decrypted += decipher.final("utf8");
```

### Erzeugung zufälliger Werte

- **`randomBytes()`-Methode**: Erzeugt kryptografisch sichere Zufallsbytes.

```javascript
console.log("Random Bytes:", crypto.randomBytes(16).toString("hex"));
// Output: a6154ef5a296fa176ad0f332bd94d712
```

- **`randomInt()`-Methode**: Erzeugt sichere Zufallszahlen innerhalb eines angegebenen Bereichs.

```javascript
console.log("Random Int:", crypto.randomInt(0, 100)); // Random number 0-99
```

### Schlüsselverwaltung

- **`createSecretKey()`-Methode**: Erstellt kryptografische Schlüsselobjekte aus Rohbytes.

```javascript
const secret = crypto.createSecretKey(crypto.randomBytes(32));
console.log(secret.export().toString('hex'));
```

### Digitale Signaturen

- **`sign()`- und `verify()`-Methoden**: Erstellen und validieren digitale Signaturen mit privaten/öffentlichen Schlüsselpaaren.
- **Anwendungsfall**: Beweist Datenauthentizität und -integrität.

### Weitere Crypto-Methoden

- **`createPublicKey()` und `createPrivateKey()`**: Arbeiten mit extern generierten Schlüsseln.
- **`createDiffieHellman()`**: Ermöglicht sichere gemeinsame Geheimniserzeugung.
- **`Certificate()`**: Verarbeitet HTTPS-Zertifikatsoperationen.

## OS-Modul

- **Zweck**: Bietet Betriebssystem-bezogene Hilfsfunktionen für Systeminformationen und plattformübergreifende Kompatibilität.
- **Import**: `const os = require("os");`
- **Anwendungsfälle**: Systemüberwachung, plattformübergreifende Skripterstellung, Hardwareinformationen.

### Methoden für Systeminformationen

- **`platform()`-Methode**: Gibt die Betriebssystemplattform zurück.

```javascript
console.log(os.platform()); // 'darwin', 'win32', 'linux', etc.

// Cross-platform usage
if (os.platform() === 'win32') {
  // Windows-specific code
} else {
  // Unix-like systems
}
```

- **`arch()`-Methode**: Gibt die CPU-Architektur zurück.

```javascript
console.log(os.arch()); // 'arm64', 'x64', 'arm', 'ia32', etc.
```

- **`type()`-Methode**: Gibt den offiziellen Betriebssystemnamen zurück.

```javascript
console.log(os.type()); // 'Darwin', 'Linux', 'Windows_NT'
```

### Versionsinformationen

- **`release()`-Methode**: Zeigt die Kernel-Version des Betriebssystems an.

```javascript
console.log(os.release()); // '25.0.0'
```

- **`version()`-Methode**: Gibt detaillierte Versionsinformationen des Betriebssystems zurück.

```javascript
console.log(os.version());
// Darwin Kernel Version 25.0.0: Wed Sep 17 21:41:39 PDT 2025
```

### Hardwareinformationen

- **`cpus()`-Methode**: Gibt ein Array mit Informationen zu CPU-Kernen zurück.

```javascript
console.log(os.cpus()); // Array of CPU core objects with model, speed, times
```

- **`uptime()`-Methode**: Gibt die Systemlaufzeit in Sekunden zurück.

```javascript
console.log(os.uptime()); // 23047 (seconds since system boot)
```

### Speicherinformationen

- **`totalmem()`-Methode**: Gibt den gesamten Systemspeicher in Bytes zurück.
- **`freemem()`-Methode**: Gibt den verfügbaren Systemspeicher in Bytes zurück.

## Bewährte Vorgehensweisen

### Leistungsaspekte

- **Vermeiden Sie synchrone Methoden**: Verwenden Sie asynchrone Methoden in Produktionsanwendungen, um Blockierungen zu verhindern.
- **Bevorzugen Sie Promises**: Nutzen Sie `async/await` mit der Promise-basierten API für sauberen, wartbaren Code.
- **Fehlerbehandlung**: Implementieren Sie immer eine ordnungsgemäße Fehlerbehandlung mit try-catch-Blöcken oder Fehler-Callbacks.

### Sicherheitsrichtlinien

- **Crypto-Modul**: Verwenden Sie etablierte Bibliotheken für Authentifizierung statt eigener Kryptolösungen.
- **Eingabevalidierung**: Validieren Sie Dateipfade und Benutzereingaben, um Sicherheitslücken zu vermeiden.
- **Schlüsselverwaltung**: Speichern Sie kryptografische Schlüssel sicher, niemals hartkodiert im Quellcode.

### Plattformübergreifende Entwicklung

- **Verwendung des OS-Moduls**: Nutzen Sie OS-Modul-Methoden für plattformübergreifende Kompatibilität.
- **Pfadbehandlung**: Verwenden Sie das `path`-Modul für plattformübergreifende Datei-Pfadoperationen.
- **Umgebungsvariablen**: Prüfen Sie plattformspezifische Umgebungsvariablen und Konfigurationen.
