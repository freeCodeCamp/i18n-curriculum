# --interactive--

정적 속성과 메서드는 클래스 자체에 속하며, 클래스의 개별 인스턴스에는 속하지 않습니다. 클래스 인스턴스를 생성하지 않고도 클래스 이름으로 직접 접근할 수 있습니다. 관련 기능을 캡슐화하기 위해 클래스 내에 정의합니다.

메서드 이름 앞에 `static` 키워드를 작성하여 정적 메서드를 정의할 수 있습니다.

```js
class MyClass {
  static staticMethod() { ... }
}
```

그런 다음, 점 표기법을 사용해 클래스에서 직접 정적 메서드를 호출하고 필요한 인수를 전달할 수 있습니다:

```js
MyClass.staticMethod();
```

클래스 인스턴스를 생성하지 않고도 메서드를 호출할 수 있다는 점에 주목하세요. 이것이 정적 메서드의 핵심 특징 중 하나입니다.

예를 들어 보겠습니다. 영화 애플리케이션을 만들고 있다고 가정하고, 영화 평점을 기준으로 영화를 비교할 수 있기를 원합니다.

이 비교 메서드는 특정 영화에 국한되지 않는 상위 수준 메서드로 생각할 수 있습니다:

```js
if (movieA.rating < movieB.rating) {
  console.log(`${movieB.title} has a higher rating.`);
} 
```

이는 `Movie` 클래스와 관련된 보다 일반적인 메서드와 같습니다.

가독성과 유지보수성을 위해 관련 메서드를 서로 가깝게 유지하려면 `Movie` 클래스 내에 정의하는 것이 좋습니다.

따라서 이것은 정적 메서드로 완벽한 후보입니다. 생성자 바로 아래에 정의된 것을 볼 수 있습니다:

:::interactive_editor

```js
class Movie {

  constructor(title, rating) {
    this.title = title;
    this.rating = rating;
  }

  static compareMovies(movieA, movieB) {
    if (movieA.rating > movieB.rating) {
      console.log(`${movieA.title} has a higher rating.`);
    } else if (movieA.rating < movieB.rating) {
      console.log(`${movieB.title} has a higher rating.`);
    } else {
      console.log("These movies have the same rating.");
    }
  }

}

let movieA = new Movie("Movie A", 80);
let movieB = new Movie("Movie B", 45);
console.log(movieA);
```

:::

정적 메서드는 `static` 키워드로 정의되며, 이름은 `compareMovies`입니다. 두 개의 매개변수 `movieA`와 `movieB`를 받습니다. 이들은 `Movie` 클래스의 인스턴스가 될 것입니다.

평점 범위 `0`부터 `100`까지를 기준으로 비교합니다. 이 로직은 조건문으로 구현되어 어떤 영화가 더 높은 평점을 받았는지에 따라 적절한 메시지를 출력합니다.

메서드가 정의되면 클래스에서 호출할 수 있습니다. 하지만 먼저 인수를 프로그램 내에 준비하고 정의해야 합니다.

이 경우, 메서드는 두 개의 영화 인스턴스를 인수로 받으므로 여기서 인스턴스를 정의합니다:

```js
let movieA = new Movie("Movie A", 80);
let movieB = new Movie("Movie B", 45);
```

`movieA`이 `movieB`보다 평점이 더 높다는 것을 알 수 있습니다. 이 메서드의 출력을 확인해 보겠습니다.

메서드를 호출하려면 클래스 자체에서 점 표기법을 사용하면 됩니다.
클래스 이름(`Movie`)을 작성하고 점을 찍은 다음 정적 메서드 이름(`compareMovies`)을 씁니다.

그런 다음 소괄호 안에 인수를 전달합니다. 이 경우, 메서드가 요구하는 두 영화 인스턴스입니다.

```js
Movie.compareMovies(movieA, movieB);
```

업데이트된 예시는 다음과 같습니다:

:::interactive_editor

```js
class Movie {

  constructor(title, rating) {
    this.title = title;
    this.rating = rating;
  }

  static compareMovies(movieA, movieB) {
    if (movieA.rating > movieB.rating) {
      console.log(`${movieA.title} has a higher rating.`);
    } else if (movieA.rating < movieB.rating) {
      console.log(`${movieB.title} has a higher rating.`);
    } else {
      console.log("These movies have the same rating.");
    }
  }

}

let movieA = new Movie("Movie A", 80);
let movieB = new Movie("Movie B", 45);
Movie.compareMovies(movieA, movieB);
console.log(movieA);
```

:::

정적 메서드는 "팩토리" 메서드를 구현할 때도 유용합니다. 팩토리 메서드는 생성자 외에 특정 기준에 따라 객체를 생성하기 위해 정의하는 메서드입니다.

`Pizza` 클래스를 예로 들어 보겠습니다. 정적 메서드 `createMargherita`는 타입과 가격이 이미 설정된 마르게리타 피자 인스턴스를 생성하기 위해 호출할 수 있는 팩토리 메서드입니다.

```js
class Pizza {
  constructor(type, price) {
    this.type = type;
    this.price = price;
  }

  static createMargherita() {
    return new this("Margherita", 6.99);
  }
}
```

이것은 정적 메서드에 대해 매우 중요한 점을 알려 줍니다.

정적 메서드 내 `this` 키워드의 값은 정적 메서드가 클래스에 속하므로 클래스 자체입니다.

그래서 `this`를 사용해 `Pizza` 클래스의 새 인스턴스를 생성할 수 있습니다.

이 메서드를 `Pizza` 클래스 자체에서 호출하고 반환된 인스턴스를 변수에 할당하면, 다음 예시처럼 됩니다:

```js
let myPizza = Pizza.createMargherita();
```

코드 어디에서든 필요할 때 사용할 수 있습니다. 예를 들어 콘솔에 출력할 수도 있습니다:

```js
console.log(myPizza);
```

출력 결과는 다음과 같습니다:

```js
Pizza { type: 'Margherita', price: 6.99 }
```

점 표기법을 사용해 메서드를 호출하고 속성에 접근할 수도 있습니다:

```js
console.log(myPizza.type);
```

전체 예시는 다음과 같습니다:

```js
class Pizza {
  constructor(type, price) {
    this.type = type;
    this.price = price;
  }

  static createMargherita() {
    return new this("Margherita", 6.99);
  }
}

let myPizza = Pizza.createMargherita();
console.log(myPizza);
console.log(myPizza.type);
```

메서드뿐 아니라 `static` 키워드로 정적 속성도 정의할 수 있습니다.

이 예시에서는 정적 `numberOfPizzasSold` 속성이 있습니다.

```js
class Pizza {
  static numberOfPizzasSold = 0;

  constructor(type) {
    this.type = type;
    Pizza.numberOfPizzasSold++;
  }
}
```

특정 피자 인스턴스에 속하지 않고 클래스 자체에 속하기 때문에 정적입니다.

초기 값은 `0`이며 새 인스턴스가 생성될 때마다 업데이트됩니다.

피자 인스턴스 두 개를 생성하면 값이 두 번 업데이트됩니다:

```js
let pizza1 = new Pizza("Margherita");
let pizza2 = new Pizza("Neapolitan");
```

정적 속성 값을 접근하려면 속성이 클래스에 속하므로 클래스 자체에서 점 표기법을 사용하면 됩니다.

```js
class Pizza {
  static numberOfPizzasSold = 0;

  constructor(type) {
    this.type = type;
    Pizza.numberOfPizzasSold++;
  }
}

let pizza1 = new Pizza("Margherita");
let pizza2 = new Pizza("Neapolitan");

console.log(Pizza.numberOfPizzasSold);
```

이 경우, 두 개의 피자가 판매되어 출력값은 `2`입니다.

이것이 JavaScript에서 정적 속성과 메서드의 기본입니다. 정적 멤버를 이해하는 것은 재사용 가능하고 효율적인 클래스를 만드는 데 필수적입니다.
