# --description--

Una base de datos relacional es una colección de datos organizados en tablas. Estas tablas están compuestas por filas y columnas.

Las filas representan registros individuales. Por ejemplo, si estás almacenando datos de usuarios, cada fila podría representar a un usuario específico.

Las columnas representan los atributos o campos que describen cada registro. La tabla podría incluir columnas para atributos comunes del usuario, como nombre y número de identificación nacional.

Una de las características clave de las bases de datos relacionales es que nos permiten conectar diferentes piezas de información vinculando tablas a través de atributos comunes, facilitando ver cómo se relacionan entre sí.

Las bases de datos relacionales requieren un esquema. El esquema define la estructura general de la base de datos, incluyendo sus tablas, columnas, tipos de datos, relaciones y restricciones.

Todas las tablas en una base de datos relacional tienen una clave primaria, que es un identificador único para cada fila. Permiten relaciones entre tablas a través de claves foráneas. Aprenderás más sobre claves primarias, claves foráneas y cómo funcionan en las próximas lecciones. Por ahora, solo debes saber que son fundamentales para enlazar registros entre tablas.

Por ejemplo, supongamos que estás creando una base de datos para una organización que administra un centro de conservación de vida silvestre para animales en peligro crítico.

Una de las tablas podría almacenar información general sobre los animales, podrías llamarla `animals`. Cada fila podría representar un animal, mientras que cada columna podría representar un atributo de ese animal, como especie, tamaño, peso, edad y otras características únicas.

Para identificar de manera única a cada animal, normalmente agregarías una columna `id` a la tabla. Esta sería la clave primaria de la tabla porque es única para cada animal. El ID de cada animal podría usarse para vincular su registro en esta tabla con registros relacionados en otras tablas.

También podrías tener una tabla `species` para almacenar información sobre cada especie, como su hábitat y estado de conservación. Cada animal en la tabla `animals` podría referenciar una especie de esta tabla usando un ID de especie.

También podrías crear una tabla para almacenar los registros veterinarios de los animales. De esta manera, el centro puede hacer seguimiento de su salud, tratamientos y medicamentos.

Una vez que tengas este esquema general, podrías vincular estas tablas a la tabla `animals`, que actúa como el punto central para los datos relacionados con animales.

De esta manera, no tienes que mantener toda la información sobre un animal en la misma tabla. En cambio, puedes almacenar datos relacionados en tablas separadas y vincularlas mediante relaciones. Al crear estas relaciones, puedes obtener los datos específicos que necesitas con una sola consulta.

Una consulta es una solicitud de datos específicos de la base de datos. Ejemplos de consultas que podrías hacer en el contexto de nuestro ejemplo serían encontrar todos los animales que necesitan un chequeo veterinario, encontrar las especies en peligro crítico más importantes del centro, encontrar el número de animales que nacieron en cautiverio, y más. Puedes personalizar las consultas para que se ajusten a tus necesidades.

Nuestro ejemplo estaba relacionado con la conservación de la vida silvestre, pero las bases de datos relacionales tienen una amplia gama de aplicaciones en industrias como la salud, la construcción, los negocios, los videojuegos, la educación, el gobierno, el comercio electrónico, las redes sociales y más.

Ofrecen varias ventajas, incluida la escalabilidad. Pueden manejar grandes conjuntos de datos, por lo que son perfectos para aplicaciones complejas del mundo real. También garantizan la integridad de los datos mediante claves primarias, claves foráneas y tipos de datos, lo que asegura que los datos sean consistentes y precisos.

Además de las bases de datos relacionales, puedes trabajar con bases de datos no relacionales.

La principal diferencia entre las bases de datos relacionales y no relacionales es cómo se almacenan los datos. Las bases de datos no relacionales, también conocidas como bases de datos NoSQL, son más flexibles. Almacenan datos en archivos individuales que no están conectados. Su modelo de datos también es más flexible.

Aunque algunas bases de datos no relacionales pueden tener un esquema básico o modelo de datos, a menudo es menos rígido que el esquema definido en bases de datos relacionales. Puedes agregar, modificar o eliminar campos de datos si es necesario.

La decisión de usar una base de datos relacional o no relacional depende de varios factores, incluyendo la naturaleza de los datos y los requisitos específicos de la aplicación. Debes evaluar las compensaciones entre estos dos enfoques para elegir el mejor para tu aplicación.
