# --description--

في React، التأثير هو أي شيء يحدث خارج عملية عرض المكوِّن. أي أي شيء لا يتعامل معه React مباشرة كجزء من عرض واجهة المستخدم.

تشمل الأمثلة الشائعة جلب البيانات، تحديث عنوان تبويب المتصفح، القراءة من التخزين المحلي للمتصفح أو الكتابة فيه، الحصول على موقع المستخدم، وأكثر من ذلك. هذه العمليات تتفاعل مع العالم الخارجي وتُعرف بالتأثيرات الجانبية.

يوفر React دالة `useEffect` لتتمكن من التعامل مع تلك التأثيرات الجانبية. تتيح لك `useEffect` تشغيل دالة بعد عرض المكوِّن أو تحديثه.

لنرَ كيف تعمل دالة `useEffect` ولماذا هي ضرورية لتطوير React الحديث.

لاستخدام دالة `useEffect`، تحتاج أولًا إلى استيرادها:

```js
import { useEffect } from "react";
```

ثم تستخدمها كدالة، هكذا:

```js
useEffect(() => {
  // Your side effect logic (usually a function) here
}, [dependencies]);
```

تُنفذ دالة التأثير بعد عرض المكوِّن، بينما تتحكم المعلمة الاختيارية `dependencies` في توقيت تنفيذ التأثير.

لاحظ أن `dependencies` يمكن أن تكون مصفوفة من "القيم التفاعلية" (الحالة، الخصائص، الدوال، المتغيرات، وهكذا)، أو مصفوفة فارغة، أو تُحذف تمامًا. إليك كيف تتحكم كل هذه الخيارات في عمل `useEffect`:

- إذا كانت `dependencies` مصفوفة تحتوي على قيمة تفاعلية واحدة أو أكثر، فسيُنفذ التأثير كلما تغيرت تلك القيم.

- إذا كانت `dependencies` مصفوفة فارغة، فإن `useEffect` يُنفذ مرة واحدة فقط عند العرض الأول للمكوِّن.

- إذا حذفت `dependencies`، يُنفذ التأثير في كل مرة يُعرض أو يُحدّث فيها المكوِّن.

على سبيل المثال، في تطبيق `Counter` هذا، لا نمرر معلمة `dependencies`، لذلك يُنفذ التأثير عند عرض المكوِّن وكل مرة يُحدّث فيها:

```jsx
import { useState, useEffect } from "react";

const Counter = () => {
  const [count, setCount] = useState(0);

  useEffect(() => {
    console.log("Component renders");
  });

  return (
    <div
      style={{
        display: "flex",
        alignItems: "center",
        flexDirection: "column",
      }}
    >
      <h2>{count}</h2>
      <div>
        <button onClick={() => setCount(count + 1)}>Increase</button>
        <button onClick={() => setCount(count - 1)}>Decrease</button>
      </div>
    </div>
  );
};

export default Counter;
```

لكن إذا مررنا مصفوفة فارغة كاعتماد، يُنفذ التأثير فقط عند العرض الأول:

```js
useEffect(() => {
  console.log('Component renders');
}, []);
```

إذا مررت حالة `count` كاعتماد، يُنفذ التأثير عند العرض الأول للمكوِّن، وعندما يتغير `count`:

```js
useEffect(() => {
  document.title = `The current count is ${count}`;
  console.log('component renders');
}, [count]);
```

لاحظ أنه إذا استمر التأثير الذي أنشأته بعد دورة حياة عرض المكوِّن، فقد تحتاج إلى دالة أخرى لـ"تنظيف" ذلك التأثير بعد عرض المكوِّن أو تحديثه.

على سبيل المثال، إذا استخدمت دالة التأثير `setInterval()`، أو أضفت مستمع حدث مثل `window.addEventListener()`، أو اتصلت بخادم، فستحتاج إلى دالة تنظيف لتشغيل `clearInterval()`، و`window.removeEventListener()`، وفصل الاتصال بالخادم على التوالي.

إليك الصيغة لإرجاع دالة تنظيف من دالة `useEffect`:

```js
useEffect(() => {
  // Your side effect logic here
  return () => {
    // Cleanup logic here (optional)
  };
}, [dependencies]);
```

على سبيل المثال، إذا أضفت مستمع حدث للتمرير، يمكنك تنظيفه بإزالته في دالة التنظيف:

```js
useEffect(() => {
  const handleScroll = () => {
    // Handle scroll logic
  };
  window.addEventListener("scroll", handleScroll);

  return () => {
    window.removeEventListener("scroll", handleScroll);
  };
}, []);
```
