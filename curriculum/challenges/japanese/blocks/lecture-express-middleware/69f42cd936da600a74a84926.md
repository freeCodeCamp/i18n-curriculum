# --description--

ミドルウェアはExpressにおける重要な概念であり、クライアントとサーバー間のリクエストとレスポンスを処理することを可能にします。ミドルウェア関数はリクエストオブジェクト、レスポンスオブジェクト、そしてアプリケーションのリクエスト-レスポンスサイクル内の次の関数にアクセスできます。

詳しく見ていきましょう：

## ミドルウェアとは何か？

ミドルウェアは基本的に、サーバーへのリクエストのライフサイクル中に実行される関数です。リクエストのログ記録、エラー処理、受信リクエストの解析、静的ファイルの提供など、さまざまなタスクを実行します。

Expressは、ルートハンドラに到達する前に受信リクエストを処理するミドルウェアを定義する便利な方法を提供しています。

ミドルウェアの動作：

Expressアプリケーションにリクエストが送られると、ミドルウェアは追加された順に実行されます。流れは次のようになります：

* リクエストがサーバーに送信されます。
* ミドルウェア関数がリクエストを処理します。
* ミドルウェアが`next()`を呼び出すと、次のミドルウェアまたはルートハンドラに制御が渡されます。
* ミドルウェアが`next()`を呼び出さない場合、リクエスト-レスポンスサイクルはストップします。

```js
const express = require('express')
const app = express()

// Simple middleware that logs request info
app.use((req, res, next) => {
  console.log('Request URL:', req.url)
  console.log('Request Method:', req.method)
  next() // Passes control to the next middleware
})

app.get('/', (req, res) => {
  res.send('Hello World!')
})

app.listen(3000, () => {
  console.log('Server running on http://localhost:3000')
})
```

`app.use()`：アプリケーションにグローバルにミドルウェアを追加します。

この例では、ミドルウェアがリクエストの詳細をログに記録し、次のミドルウェアまたはルートハンドラに進むために`next()`を呼び出しています。

ミドルウェアはExpressの強力な機能であり、さまざまなポイントでリクエストを傍受し、変更することができます。ログ記録、認証の処理、あるいはルートハンドラに到達する前にリクエストやレスポンスオブジェクトを変更するためにミドルウェアを使うことができます。
