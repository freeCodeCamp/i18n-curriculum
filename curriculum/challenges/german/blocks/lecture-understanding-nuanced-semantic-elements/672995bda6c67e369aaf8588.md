# --interactive--

Beschreibunglisten sind perfekt, um Begriffe und Definitionen in einem organisierten und leicht lesbaren Format darzustellen, wie in einem Glossar oder einem echten Wörterbuch, in dem Sie Wörter mit ihren entsprechenden Definitionen finden können.

Dies ist ein Beispiel für eine Beschreibungsliste in HTML mit zwei Begriffen und ihren entsprechenden Details.

Aktivieren Sie den interaktiven Editor und versuchen Sie, den Code auszukommentieren, um das neue Detail-Element im Vorschaufenster anzuzeigen.

:::interactive_editor

```html
<dl>
  <dt>HTML</dt>
  <dd>HyperText Markup Language</dd>
  <dt>CSS</dt>
  <dd>Cascading Style Sheets</dd>
  <!-- <dt>JS</dt>
  <dd>JavaScript</dd> -->
</dl>
```

:::

In diesem Fall sind die Begriffe die Akronyme HTML und CSS, und die Details sind deren Ausschreibungen. Die Details könnten auch Definitionen oder andere Informationen zu den Begriffen sein.

Sie benötigen drei HTML-Elemente, um eine Beschreibungsliste zu definieren. Zuerst das Beschreibungsliste-Element, `dl`, das der Container für die gesamte Liste ist. Sie sehen, dass es alle anderen Elemente der Beschreibungsliste im Beispiel umschließt.

Dann ein Beschreibungsterm-Element, `dt`, für jeden Begriff. In diesem Fall hat die Beschreibungsliste zwei Begriffe, HTML und CSS, also gibt es zwei dieser Elemente.

Und schließlich finden Sie nach jedem Begriff ein Beschreibungselement, `dd`, für die Beschreibung oder Details, die mit diesem Begriff verbunden sind. In diesem Beispiel sind das HyperText Markup Language und Cascading Style Sheets.

Im Browser würden Sie jeden Begriff gefolgt von seiner entsprechenden Beschreibung sehen. Standardmäßig sind die Beschreibungen leicht nach rechts eingerückt, um sie visuell zu unterscheiden.

Aber Beschreibunglisten sind nicht nur auf Begriffe und Definitionen beschränkt. Sie sind viel vielseitiger. Hier haben wir ein Rezept mit zwei Zutaten.

Aktivieren Sie den interaktiven Editor und versuchen Sie, den Code auszukommentieren, um das neue Detail-Element im Vorschaufenster anzuzeigen.

:::interactive_editor

```html
<dl>
  <dt>Flour</dt>
  <dd>2 cups</dd>
  <dt>Sugar</dt>
  <dd>1/2 cup</dd>
  <!-- <dt>Vegetable Oil</dt>
  <dd>2 tablespoons</dd> -->
</dl>
```

:::

Die gesamte Beschreibungsliste befindet sich innerhalb eines Beschreibungslistelements. Die erste Zutat, `Flour`, befindet sich innerhalb eines Beschreibungsterm-Elements. Dann sehen Sie, wie viel von dieser Zutat Sie benötigen: `2 cups`. Dies befindet sich direkt nach der entsprechenden Zutat in einem Beschreibungselement.

Und dieselbe Struktur wird für `Sugar` wiederholt. In diesem Fall hat das Rezept nur zwei Zutaten, aber wenn es mehr gäbe, könnte dieselbe Struktur in der gesamten Beschreibungsliste wiederholt werden.

Im Browser würden Sie die Zutaten linksbündig und die Mengenangaben eingerückt sehen, um sie visuell zu trennen.

Weitere Anwendungsfälle für Beschreibunglisten sind Produktspezifikationen, häufig gestellte Fragen, Kontaktinformationen und Metadaten. Grundsätzlich können Sie eine Beschreibungsliste verwenden, wenn Sie zwei zusammenhängende Informationen im Schlüssel-Wert-Paar-Format haben, wobei eine als Bezeichnung, der Schlüssel, und die andere als zusätzliche verwandte Information, der Wert, fungiert.
