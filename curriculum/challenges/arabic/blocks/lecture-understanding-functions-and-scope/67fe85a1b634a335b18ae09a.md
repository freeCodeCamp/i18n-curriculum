# --description--

في Python، يحدد النطاق النقطة التي يمكنك عندها الوصول إلى متغير. إنه ما يتحكم في مدة حياة المتغير وكيف يُحل في أجزاء مختلفة من الشفرة.

لتحديد النطاق بشكل صحيح، يتبع Python قاعدة **LEGB**، التي تعني ما يلي:

- **النطاق المحلي (L)**: المتغيرات المعرفة داخل الدوال أو الأصناف.
    
- **النطاق المحيط (E)**: المتغيرات المعرفة في الدوال المحيطة أو المسكنة.
    
- **النطاق العام (G):** المتغيرات المعرفة في المستوى الأعلى من الوحدة أو الملف.
    
- **النطاق المدمج (B)**: الأسماء المحجوزة في Python للدوال والوحدات والكلمات المفتاحية والكائنات المعرفة مسبقًا.

يستخدم Python قاعدة LEGB لحل نطاق المتغيرات في برنامجك. سنغوص في كل من هذه القواعد لتفهم العملية بشكل أفضل.

**النطاق المحلي** يعني أن المتغير المعلن داخل دالة أو صنف يمكن الوصول إليه فقط داخل تلك الدالة أو الصنف.

إليك مثالًا:

```python
def my_func():
    my_var = 10
    print(my_var)
```

في هذه الحالة، للدالة `my_func` نطاق خاص بها لا يمكن الوصول إليه من خارج الدالة. استدعاء `my_func` سيُخرج `10`، لكن طباعة `my_var` خارج الدالة ستؤدي إلى خطأ `NameError`:

```python
def my_func():
    my_var = 10 # Locally scoped to my_func
    print(my_var)

my_func() # 10

print(my_var) # NameError: name 'my_var' is not defined
```

**النطاق المحيط** يعني أن الدالة التي تسكن داخل دالة أخرى يمكنها الوصول إلى المتغيرات في الدالة التي تسكن بداخلها.

على سبيل المثال:

```python
def outer_func():
    msg = 'Hello there!'

    def inner_func():
        print(msg)

    inner_func()

outer_func() # Hello there!
```

في هذا المثال، يمكن للدالة الداخلية `inner_func` الوصول بحرية إلى المتغير `msg` المعرفة في الدالة الخارجية `outer_func`. مع ذلك، لاحظ أن الدوال الخارجية لا يمكنها الوصول إلى المتغيرات المعرفة داخل أي دوال مسكنة:

```python
def outer_func():
    msg = 'Hello there!'
    print(res)

    def inner_func():
        res = 'How are you?'
        print(msg)

    inner_func()

outer_func() # NameError: name 'res' is not defined
```

ذلك لأن `res` نطاقه محلي داخل `inner_func`. أيضًا، لاحظ أن `outer_func` تحاول طباعة `res` قبل استدعاء `inner_func`.

الحل هو تهيئة `res` كسلسلة نصية فارغة في النطاق المحيط، وهو داخل `outer_func`. ثم داخل `inner_func`، اجعل `res` متغيرًا غير محلي باستخدام الكلمة المفتاحية `nonlocal`:

```python
def outer_func():
    msg = 'Hello there!'
    res = ""  # Declare res in the enclosing scope

    def inner_func():
        nonlocal res  # Allow modification of an enclosing variable
        res = 'How are you?'
        print(msg)  # Accessing msg from outer_func()

    inner_func()
    print(res)  # Now res is accessible and modified

outer_func()

# Output:
# Hello there!
# How are you?
```

**النطاق العام** يشير إلى المتغيرات المعلنة خارج أي دوال أو أصناف والتي يمكن الوصول إليها من أي مكان في البرنامج. هنا، يمكن الوصول إلى `my_var` في أي مكان، حتى داخل دالة لم تُعرف فيها:

```python
my_var = 100

def show_var():
    print(my_var)

show_var() # 100
print(my_var) # 100
```

وإذا أردت جعل متغير محلي معرف داخل دالة متاحًا على مستوى عام، يمكنك استخدام الكلمة المفتاحية `global`:

```python
my_var_1 = 7

def show_vars():
    global my_var_2
    my_var_2 = 10
    print(my_var_1)
    print(my_var_2)

show_vars() # 7 10

# my_var_2 is now a global variable and can be accessed anywhere in the program
print(my_var_2) # 10
```

يمكنك أيضًا استخدام الكلمة المفتاحية `global` لتعديل متغير عام:

```python
my_var = 10  # A global variable

def change_var():
    global my_var  # Allows modification of a global variable
    my_var = 20

change_var()

print(my_var)  # my_var is now modified globally to 20
```

أخيرًا، **النطاق المدمج** يشير إلى جميع دوال Python المدمجة، والوحدات، والكلمات المفتاحية، وهي متاحة في أي مكان في برنامجك:

```python
print(str(45)) # '45'
print(type(3.14)) # <class 'float'>
print(isinstance(3, str)) # False
```
