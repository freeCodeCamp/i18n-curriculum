# --interactive--

修饰符，通常称为“标记”，用于修改正则表达式的行为。让我们回顾一下之前课程中的示例：

:::interactive_editor

```js
const regex = /freeCodeCamp/;
console.log(regex.test("freeCodeCamp")); // true
console.log(regex.test("freeCodeCamp is great")); // true
console.log(regex.test("I love freeCodeCamp")); // true
console.log(regex.test("freecodecamp")); // false
console.log(regex.test("FREECODECAMP")); // false
console.log(regex.test("free")); // false
console.log(regex.test("code")); // false
console.log(regex.test("camp")); // false
```

:::

如果你还记得，全部小写和全部大写的 `freeCodeCamp` 字串未能匹配该模式。这是因为，默认情况下，正则表达式是区分分支的。

但是如果我们能告诉正则表达式忽略分支呢？好吧，有一个修饰符可以做到这一点。`i` 标记使正则表达式忽略分支。我们如何使用它？标记放在正则表达式的结束斜线后面：

```js
const regex = /freeCodeCamp/i;
```

注意第一行中正则表达式的变化。现在我们可以查看这如何改变情况：

:::interactive_editor

```js
const regex = /freeCodeCamp/i;

console.log(regex.test("freeCodeCamp")); // true
console.log(regex.test("freeCodeCamp is great")); // true
console.log(regex.test("I love freeCodeCamp")); // true
console.log(regex.test("freecodecamp")); // true
console.log(regex.test("FREECODECAMP")); // true
console.log(regex.test("free")); // false
console.log(regex.test("code")); // false
console.log(regex.test("camp")); // false
```

:::

因为我们的正则表达式现在不区分分支，所以全小写和全大写的字串都“通过”了测试。这也适用于包含随机大小写字母混合的字串：

:::interactive_editor

```js
const regex = /freeCodeCamp/i;

console.log(regex.test("dO yOu LoVe fReEcOdEcAmP?")); // true
```

:::

还有许多其他标记你可以使用。`g` 标记，或全局修饰符，允许你的正则表达式匹配一个模式多次。

让我们看看这如何影响我们的代码。你会注意到我们保留了 `i` 标记：一个正则表达式可以使用多个标记（根据需要的数量）来实现你想要的行为：

```js
const regex = /freeCodeCamp/gi;
```

等一下……这是什么？看起来全局的修饰符让我们一些本应通过的字串反而失败了：

:::interactive_editor

```js
const regex = /freeCodeCamp/gi;

console.log(regex.test("freeCodeCamp")); // true
console.log(regex.test("freeCodeCamp is great")); // false
console.log(regex.test("I love freeCodeCamp")); // true
console.log(regex.test("freecodecamp")); // false
console.log(regex.test("FREECODECAMP")); // true
console.log(regex.test("free")); // false
console.log(regex.test("code")); // false
console.log(regex.test("camp")); // false
```

:::

为什么？因为全局修饰符使你的正则表达式具有状态。这意味着它会跟踪之前匹配过模式的位置。所以当它匹配第一个 `freeCodeCamp` 字串时，它会记住在索引 `0` 处找到了匹配。

然后我们用 `freeCodeCamp is great` 测试它，但它并不是从索引 `0` 开始。正则表达式“知道”它已经在索引 `0` 找到匹配，所以即使这是一个不同的字串，它也是从匹配的结束索引开始。

`freeCodeCamp` 长度为 `12` 个字符，所以在 `0` 处的匹配结束于索引 `11`。匹配将在索引 `12` 处继续。并且由于 `is great` 不匹配 `freeCodeCamp`，它返回 `false`。

然后，因为它未能查找匹配，它“丢失”了状态，并从 `0` 重新开始以下匹配。

如果我们交换日志，使得在 `0` 处有匹配的字串紧跟着一个在索引 `11` 之后有匹配的字串：

:::interactive_editor

```js
const regex = /freeCodeCamp/gi;

console.log(regex.test("freeCodeCamp")); // true
console.log(regex.test("I loooooooove freeCodeCamp")); // true
```

:::

当一个正则表达式是全局的时候，它会获得一个名为 `lastIndex` 的新建属性。拿我们之前的代码来看一下这个属性是如何工作的：

:::interactive_editor

```js
const regex = /freeCodeCamp/gi;

console.log(regex.lastIndex); // 0
console.log(regex.test("freeCodeCamp")); // true
console.log(regex.lastIndex); // 12
console.log(regex.test("freeCodeCamp is great")); // false
console.log(regex.lastIndex); // 0
console.log(regex.test("I love freeCodeCamp")); // true
console.log(regex.lastIndex); // 19
console.log(regex.test("freecodecamp")); // false
console.log(regex.lastIndex); // 0
console.log(regex.test("FREECODECAMP")); // true
console.log(regex.lastIndex); // 12
console.log(regex.test("free")); // false
console.log(regex.lastIndex); // 0
console.log(regex.test("code")); // false
console.log(regex.lastIndex); // 0
console.log(regex.test("camp")); // false
```

:::

看这个例子，你可以看到正则表达式的状态如何随着每次测试调用而变化，使用 `lastIndex` 来跟踪之前的匹配。

当你需要从单个字串中获取多个匹配时，全局标记非常有用。但如果你用相同的正则表达式测试多个字串，最好关闭 `g` 标记。

在学习下一个标记之前，你需要了解锚点。插入符号（`^`）锚点，位于正则表达式的开头，表示“匹配字串的开始”：

```js
const start = /^freecodecamp/i;
```

美元符号（`$`）锚点，位于正则表达式的末尾，表示“匹配字串的结尾”：

```js
const end = /freecodecamp$/i;
```

花点时间比较右侧的输出：

:::interactive_editor

```js
const start = /^freecodecamp/i;
const end = /freecodecamp$/i;
console.log(start.test("freecodecamp")); // true
console.log(end.test("freecodecamp")); // true
console.log(start.test("freecodecamp is great")); // true
console.log(end.test("freecodecamp is great")); // false
console.log(start.test("i love freecodecamp")); // false
console.log(end.test("i love freecodecamp")); // true
console.log(start.test("have met freecodecamp's founder")); // false
console.log(end.test("have met freecodecamp's founder")); // false
```

:::

看看起始锚点只在字串开头匹配，结束锚点只在字串结尾匹配？但是跨多行匹配呢？让我们来看一下：

:::interactive_editor

```js
const start = /^freecodecamp/i;
const end = /freecodecamp$/i;
const string = `I really love
freecodecamp
it's my favorite`;
console.log(start.test(string)); // false
console.log(end.test(string)); // false
```

:::

尽管 `freecodecamp` 独自占据一行，但它未通过两个测试。这是因为锚点默认查找整个字串的开头和结尾。

但是你可以使用 `m` 标记，或多行修饰符，使正则表达式处理多行。让我们为我们的正则表达式添加它，看看会得到什么：

:::interactive_editor

```js
const start = /^freecodecamp/im;
const end = /freecodecamp$/im;
const string = `I really love
freecodecamp
it's my favorite`;
console.log(start.test(string)); // true
console.log(end.test(string)); // true
```

:::

现在它们都匹配了！因为 `freecodecamp` 完全位于它自己的独立一行，起始锚点匹配该行的开头，结束锚点匹配该行的结尾。

最后，你有 `d` 标记，或索引修饰符。记住 `i` 标记是用于分支不敏感的，所以索引修饰符需要一个不同的标记。

`d` 标记扩展了你在匹配对象中获得的信息。让我们将它添加到我们的正则表达式中：

```js
const regex = /freecodecamp/di;
const string = "we love freecodecamp isn't freecodecamp great?";
console.log(string.match(regex));
```

结果如下：

```js
// [
//   'freecodecamp',
//   index: 8,
//   input: "we love freecodecamp isn't freecodecamp great?",
//   groups: undefined,
//   indices: [
//     0: [8, 20],
//     groups: undefined
//   ]
// ]
```

我们的匹配对象获得了一个新的 `indices` 属性！该属性是一个包含两个数字的数组，第一个数字是匹配在原始字串中开始的位置索引，第二个数字是匹配结束后的位置索引。该数组还有一个额外的 `groups` 属性，专门用于命名捕捉组。

还有一些其他的标记你应该知道是可用的，但在典型代码中较少见。

第一个是 unicode 修饰符，或 `u` 标记。它扩展了正则表达式的功能性，使其能够匹配特殊的 unicode 字符。

你将在以后的课程中学习更多关于字符类的内容，但 `u` 标记让你可以访问像 `Extended_Pictographic` 这样的特殊类，以匹配大多数表情符号：

:::interactive_editor

```js
const regex = /🍎/u;

const str = "I have an apple 🍎";
console.log(regex.test(str)); // true
```

:::

还有一个 `v` 标记，它进一步扩展了 unicode 匹配的功能性。

第二个是粘性修饰符，或者说是 `y` 标记。粘性修饰符的行为与全局修饰符非常相似，但有一些例外。

最大的问题是，全局的正则表达式将从 `lastIndex` 开始并在字串的剩余部分查找另一个匹配，但粘性正则表达式如果在之前的 `lastIndex` 处没有立即匹配，则会返回 `null` 并将 `lastIndex` 重置为 `0`。

最后一个是单行修饰符，或者说是 `s` 标记。记住，多行修饰符允许起始和结束锚点匹配每一行的开始和结束，而不是整个字串的开始和结束。

单行修饰符允许通配符字符（在正则表达式中用句点 `.` 表示）匹配换行符，有效地将字串视为单行文本。

这些修饰符有很多，但你最常用且最重要记住的是 `i` 和 `g` 标记。
