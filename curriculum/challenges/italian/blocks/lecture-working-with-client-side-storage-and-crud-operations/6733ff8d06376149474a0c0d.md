# --description--

La `Cache` API viene usata per archiviare richieste e risposte di rete, rendendo le applicazioni web più efficienti e perfino funzionanti offline. Fa parte della più ampia Service Worker API ed è fondamentale per creare Progressive Web App (PWA) che possano funzionare anche in condizioni di rete instabili o lente.

Prima di continuare con la `Cache` API, dobbiamo prima capire come funzionano a livello base i service worker, le PWA e le richieste di rete.

Una richiesta di rete è una richiesta fatta da un browser web o da un’applicazione a un server per recuperare dati o risorse tramite internet. Per esempio, quando visiti un sito web, il tuo browser invia una richiesta di rete al server web per ottenere i file (come HTML, immagini o video) necessari a mostrare la pagina.

Un service worker è un tipo speciale di file JavaScript che gira in background in un’applicazione web, separato dal thread principale del browser. Funziona come un intermediario tra la pagina web e la rete, permettendo agli sviluppatori di intercettare le richieste di rete, memorizzare risorse nella cache e gestire funzionalità come notifiche push o sincronizzazione in background.

Una PWA, o Progressive Web App, è un tipo di applicazione web che usa tecnologie web moderne per offrire un’esperienza simile a un’app nativa sul web. Le PWA sono progettate per funzionare in modo affidabile su qualsiasi dispositivo, indipendentemente dalle condizioni di rete, e possono essere consultate tramite browser o installate sul dispositivo dell’utente come un’app tradizionale.

Ora che abbiamo una migliore comprensione di service worker, PWA e richieste di rete, possiamo approfondire la Cache API.

La `Cache` API è un meccanismo di archiviazione che conserva oggetti `Request` e `Response`. Quando viene fatta una richiesta a un server, l’applicazione può salvare la risposta e poi recuperarla dalla cache invece di fare una nuova richiesta di rete. Questo riduce i tempi di caricamento, risparmia banda e migliora l’esperienza complessiva dell’utente.

Il browser fornisce un’area di archiviazione nota come `CacheStorage`, dove gli sviluppatori possono salvare coppie chiave-valore di richieste di rete e delle loro risposte corrispondenti.

Con l’intestazione `Cache-Control`, gli sviluppatori possono specificare per quanto tempo una risorsa memorizzata nella cache deve rimanere archiviata, e se deve essere riconsultata o servita direttamente dalla cache.

Usando la `Cache` API, gli sviluppatori possono costruire applicazioni web offline-first, permettendo a una PWA, per esempio, di servire risorse memorizzate nella cache quando l’utente è disconnesso dalla rete.
