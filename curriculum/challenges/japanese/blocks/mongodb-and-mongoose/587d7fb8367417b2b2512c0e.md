# --description--

昔ながらの方法では、ドキュメントを編集して何らかの形で使いたい場合（例えばサーバーレスのレスポンスとして返すなど）、この手順が必要でした。Mongooseには専用の更新メソッド`Model.update()`があります。これは低レベルのmongoドライバーに結びついています。特定の条件に合う多くのドキュメントを一括編集できますが、更新されたドキュメント自体は返さず、'ステータス'メッセージのみを返します。さらに、mongoドライバーを直接呼び出すため、モデルのバリデーションが難しくなります。

# --instructions--

`findEditThenSave`関数を修正して、`_id`を検索キーとして使い、`personId`のいずれかの方法で人物を検索してください。人物の`"hamburger"`のリストに`favoriteFoods`を追加してください（`Array.push()`を使えます）。その後、findのコールバック内で更新された`save()`を`Person`してください。

**注意:** スキーマで`favoriteFoods`を型指定なしの配列（つまり`[String]`）として宣言している場合は少し難しいかもしれません。その場合、`favoriteFoods`はMixed型にデフォルト設定され、`document.markModified('edited-field')`を使って手動で編集済みとしてマークする必要があります。詳細は<a href="https://www.freecodecamp.org/news/introduction-to-mongoose-for-mongodb-d2a7aa593c57/" target="_blank" rel="noopener noreferrer nofollow">Mongooseの記事</a>をご覧ください。

# --hints--

アイテムの検索・編集・更新は成功するはずです

```js
  const response = await fetch(code + '/_api/find-edit-save', {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json'
    },
    body: JSON.stringify({
      name: 'Poldo',
      age: 40,
      favoriteFoods: ['spaghetti']
    })
  });
  if (!response.ok) {
    throw new Error(await response.text());
  }
  const data = await response.json();
  assert.equal(data.name, 'Poldo', 'item.name is not what is expected');
  assert.equal(data.age, 40, 'item.age is not what expected');
  assert.deepEqual(
    data.favoriteFoods,
    ['spaghetti', 'hamburger'],
    'item.favoriteFoods is not what expected'
  );
  assert.equal(data.__v, 1, 'The item should be previously edited');
```
