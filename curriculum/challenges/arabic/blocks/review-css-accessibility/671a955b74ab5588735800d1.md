## شجرة إمكانية الوصول

شجرة إمكانية الوصول هي بنية تستخدمها تقنيات المساعدة، مثل قارئ الشاشة، لفهم المحتوى على صفحة الويب والتفاعل معه.

## دالة `max()`

تعيد دالة `max()` أكبر قيمة من مجموعة قيم مفصولة بفواصل:

:::interactive_editor

```html
<head>
  <link rel="stylesheet" href="styles.css">
</head>
<body>
  <img src="https://cdn.freecodecamp.org/curriculum/css-photo-gallery/9.jpg" alt="A cat sitting on a table">
</body>
```

```css
img {
  width: max(250px, 25vw);
}
```

:::

في المثال أعلاه، سيكون عرض الصورة 250 بكسل إذا كان عرض إطار العرض أقل من 1000 بكسل. إذا كان عرض إطار العرض أكبر من 1000 بكسل، سيكون عرض الصورة 25vw. ذلك لأن 25vw تساوي 25% من عرض إطار العرض.

## أفضل الممارسات مع CSS وإمكانية الوصول

- **`display: none;`**: استخدام `display: none;` يعني أن قارئات الشاشة وتقنيات المساعدة الأخرى لن تتمكن من الوصول إلى هذا المحتوى، لأنه غير مدرج في شجرة إمكانية الوصول. لذلك، من المهم استخدام هذه الطريقة فقط عندما تريد إزالة المحتوى تمامًا من العرض البصري وإمكانية الوصول.
- **`visibility: hidden;`**: تخفي هذه الخاصية والقيمة المحتوى بصريًا لكنها تبقيه في تدفق المستند، مما يعني أنه لا يزال يشغل مساحة على الصفحة. كما أن هذه العناصر لن تُقرأ بعد ذلك بواسطة قارئات الشاشة لأنها تُزال من شجرة إمكانية الوصول.
- **فئة CSS `.sr-only`**: هذه تقنية شائعة تُستخدم لإخفاء المحتوى بصريًا مع إبقائه متاحًا لقارئات الشاشة.

:::interactive_editor

```html
<head>
  <link rel="stylesheet" href="styles.css">
</head>
<body>
  <span>This span element can be read</span>
  <span class="sr-only">This span element can only be read by screen readers</span>
  <p style="display: none">
    This text is hidden with display: none and is NOT accessible to screen readers.
  </p>
  <p style="visibility: hidden">
    This text is hidden with visibility: hidden and is NOT read by screen readers,
    but it still takes up space.
  </p>
</body>

```

```css
.sr-only {
  position: absolute;
  width: 1px;
  height: 1px;
  padding: 0;
  margin: -1px;
  overflow: hidden;
  clip: rect(0, 0, 0, 0);
  white-space: nowrap;
  border: 0;
}

```

:::

## إخفاء المحتوى باستخدام خصائص HTML

- **خاصية `aria-hidden`**: تُستخدم لإخفاء عنصر عن الأشخاص الذين يستخدمون تقنية مساعدة مثل قارئ الشاشة. على سبيل المثال، يمكن استخدامها لإخفاء الصور الزخرفية التي لا تقدم محتوى ذا معنى.
- **خاصية `hidden`**: تدعمها معظم المتصفحات الحديثة وتخفي المحتوى بصريًا ومن شجرة إمكانية الوصول. يمكن تبديلها بسهولة باستخدام JavaScript.

:::interactive_editor

```html
<p aria-hidden="true">This paragraph is visible for sighted users, but is hidden from assistive technology.</p>
<p hidden>This paragraph is hidden from both sighted users and assistive technology.</p>
<p>This is a normal paragraph</p>

```

:::

## مشكلة إمكانية الوصول في خاصية `placeholder`

استخدام نص العنصر النائب ليس مثاليًا لإمكانية الوصول. كثيرًا ما يخلط المستخدمون بين نص العنصر النائب وقيمة المُدخَل الفعلية، ويظنون أن هناك قيمة موجودة بالفعل في المُدخَل.
