# --description--

JavaScript 성능을 향상시키기 위한 가장 중요한 기법들을 함께 살펴보겠습니다.

놀라울 수 있지만, 첫 번째 기법은 가능한 한 JavaScript 사용을 줄이는 것입니다.

간단한 정적 웹사이트를 개발한다면, JavaScript에 의존하는 프레임워크 없이도 똑같이 구현할 수 있을지도 모릅니다.

또한 사용하지 않는 JavaScript 코드를 제거하고, 가능하다면 더 간단한 해결책을 사용하는 것이 좋습니다.

예를 들어, 직접 만든 기능 대신 내장 브라우저 기능을 사용하는 것을 고려해 보세요. 내장 브라우저 기능에는 폼 유효성 검사, 브라우저 내장 비디오 플레이어, 그리고 커스텀 JavaScript 애니메이션 대신 CSS 애니메이션이 포함됩니다.

애니메이션은 가능한 한 적게 사용하세요.

JavaScript 코드에서 DOM 조작을 줄이세요. DOM 조작은 계산 비용이 많이 듭니다.

JavaScript 코드를 중요한 작업과 중요하지 않은 작업을 수행하는 모듈로 분리하는 것도 도움이 됩니다. 이렇게 하면 중요한 모듈을 가능한 빨리 미리 불러오고, 중요하지 않은 모듈은 페이지 렌더링을 최대한 빠르게 하기 위해 나중으로 미룰 수 있습니다.

파일을 미리 불러온다고 해서 사용 시점에 바로 사용할 수 있다는 보장은 없지만, 다운로드가 더 빨리 시작되므로 전체 대기 시간이 줄어듭니다.

코드를 여러 파일로 분리한 후에는 빌드 과정에서 자동으로 파일 크기를 줄이는 압축(minify)을 하여 크기를 줄이고 로드 시간을 최적화할 수 있습니다.

더 이상 필요 없는 이벤트 리스너는 반드시 제거하세요:

```js
element.removeEventListener("mousemove", handleMouseMove);
```

이 예제에서는 `mousemove` 이벤트에 대한 이벤트 리스너를 제거하여, 이벤트가 발생해도 `handleMouseMove` 함수가 더 이상 호출되지 않도록 했습니다. 이벤트 리스너를 계속 유지하면 더 이상 필요 없는 계산을 계속 수행하게 되어 전체 성능에 영향을 미칩니다. 여러 개의 활성 이벤트 리스너가 동시에 있을 경우 이 영향은 더욱 뚜렷해집니다.

이벤트 이야기가 나왔으니, 가능하면 이벤트 위임을 사용하는 것이 좋습니다.

이벤트 위임은 개별 자식 요소에 이벤트 리스너를 설정할 필요 없이, 부모 요소에 직접 설정하면 이벤트가 버블링되어 처리할 수 있는 부모를 찾을 때까지 올라간다는 뜻입니다.

JavaScript 최적화는 빠르고 반응형인 웹 애플리케이션 개발에 매우 중요합니다. 실제 프로젝트에서는 웹사이트 성능을 지속적으로 분석하여 개선할 영역을 찾아야 합니다.
