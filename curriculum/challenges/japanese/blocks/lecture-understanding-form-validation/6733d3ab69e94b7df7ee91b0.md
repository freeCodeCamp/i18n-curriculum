# --description--

`submit` イベントが HTML フォームでどのように動作するかを学びましょう。まず、フォームの提出方法を理解する必要があります。フォームが提出される方法は3つあります。

1つ目は、フォーム内のボタンをユーザーがクリックし、そのボタンに `type` 属性が `submit` に設定されている場合です。

2つ目は、ユーザーがフォーム内の編集可能な入力フィールドで Enter キーを押した場合です。

3つ目は、JavaScript でフォーム要素の `requestSubmit()` または `submit()` メソッドを呼び出す場合です。

では、フォームが提出されると何が起こるのでしょうか？いくつかのことが起こり、フォームの動作はその属性によって決まります。

最初に見るべき属性は `action` 属性です。`action` 属性には URL または現在のドメインの相対パスが含まれている必要があります。この値はフォームがデータを送信しようとする場所を決定します。`action` 属性を設定しない場合、フォームは現在のページの URL にデータを送信します。以下は `action` 属性が特定の URL に設定されたフォームの例です。

```html
<form action="https://freecodecamp.org">
  <input type="number" id="input" placeholder="Enter a number" name="number" />
  <button type="submit">Submit</button>
</form>
```

このフォームが提出されると、freeCodeCamp のホームページにデータを送信しますが、おそらく何も起こりません。

次に、相対パスに送信する別のフォームです。

```html
<form action="/data">
  <input type="number" id="input" placeholder="Enter a number" name="number" />
  <button type="submit">Submit</button>
</form>
```

これは現在のドメインの `/data` パスに送信します。今回の場合、`http://127.0.0.1:5500/data` に送信されます。

提出の動作を制御する2つ目の属性は `method` 属性です。この属性は `GET` や `POST` のような標準的な HTTP メソッドを受け入れ、そのメソッドを使って `action` URL にリクエストを行います。

HTTP は Hypertext Transfer Protocol の略で、ウェブ上でデータを転送するために使われます。

HTTP メソッドは、`GET`、`POST`、`PUT`、`DELETE` などのリソースに対して実行できる操作を定義します。これらのメソッドについては今後のレッスンで詳しく学びます。

メソッドが設定されていない場合、フォームはデフォルトで `GET` リクエストを送信します。`GET` リクエストは指定されたリソースからデータを取得するために使われ、変更は加えず、パラメータは通常クエリ文字列の形で URL に追加されます。

フォーム内のデータは `name=value` ペアとして URL エンコードされ、`action` URL にクエリパラメータとして追加されます。例えば、前の例のフォームを `3342` の値で `input` フィールドに入力して提出すると、フォームは `GET` しようとします URL `http://127.0.0.1:5500/data?number=3342`。

これはユーザーがデータを検索するような検索フォームには最適です。しかし、ユーザーに新しいデータを提出してもらいたい場合、例えば登録フォームでは、慣例的に `POST` メソッドを使います。`POST` メソッドはサーバーにデータを送信してリソースを作成または更新するために使われます。

`method` 属性を `POST` に設定してみましょう。

```xml
<form action="/data" method="POST">
  <input type="number" id="input" placeholder="Enter a number" name="number" />
  <button type="submit">Submit</button>
</form>
```

`POST` リクエストを送信するときは、リクエストのデータを含むボディを含めることができます。したがって `GET` とは異なり、データは URL に追加されず、フォームは `POST` リクエストを `http://127.0.0.1:5500/data` に送信します。データはリクエストのボディ内にあり、URL エンコードされたフォームデータのままです。

URL エンコードされたフォームデータとは、フォームデータがキーと値のペアの文字列に変換され、`name=John+Doe&email=john%40example.com` のように特殊文字がエンコードされたバージョンに置き換えられてウェブ上で安全に送信されるものです。これについては今後のレッスンでさらに学びます。

しかし、データを URL エンコードされたフォームペイロードとして送信したくない場合もあるかもしれません。

`form` 要素は `enctype` 属性を受け入れ、これはデータに使用するエンコードタイプを表します。

この属性は3つの値のみを受け入れます。`application/x-www-form-urlencoded`（デフォルトで、データを URL エンコードされたフォームボディとして送信）、`text/plain`（データをプレーンテキスト形式で、改行で区切られた `name=value` ペアとして送信）、または `multipart/form-data` で、これはファイルアップロードを含むフォームを扱うためのものです。

このレッスンではフォーム提出の基本を学びました。今後のレッスンでフォームの操作やサーバーとのやりとりについてさらに深く掘り下げます。
