# --description--

在软件开发中，你应该了解两种测试方法。这些方法是 TDD（测试驱动开发）和 BDD（行为驱动开发）。

让我们先来看看 TDD。测试驱动开发是一种方法论，强调在为一个应用创建特色之前先编写测试。

以下是 TDD 的典型进程：

1. 编写测试

2. 确保它失败

3. 编写该测试的代码

4. 确保这次测试通过

5. 对代码进行任何必要的重构

重复步骤 1-5。

在创建特色之前编写测试为开发过程中的开发者提供实时反馈。它还有助于在开发阶段更早地发现错误。

虽然一些软件团队会选择遵循 TDD 方法，但 TDD 在行业内是一个极具争议的话题。你会发现大量文章和视频既有支持 TDD 的，也有批评 TDD 的。无论你对这种方法持何种看法，了解它都很重要，因为你可能会在实际场景中遇到它。

我们将要看的第二种测试方法是行为驱动开发（BDD）。行为驱动开发是一种将一系列测试与业务目标对齐的方法。这些测试应以技术人员和非技术人员都能理解的语言编写。

在对软件应用中的特定行为进行测试时，BDD 通常使用 Gherkin 语言，该语言在企业中用于描述行为和场景。Gherkin 语言在描述场景时使用 Given、When 和 Then 语句。

这是一个示例：

**场景：新建博客文章**

**假设** 用户已登录他们的账户并且处于 `"Create New Post"` 页面，

**当** 用户 输入帖子标题和内容并点击 `"Publish"` 按钮时，

**然后** 博客文章应被储存并在主页上显示正确的标题、内容和时间戳。

你首先要定义一个场景。示例场景包括用户登录一个应用或在网站上下订单。在此示例中，场景是用户在网站上创建一个新博客文章。

在定义场景之后，你应该从一个 Given 语句开始。这为用户在执行操作之前的起始上下文提供了说明。When 语句用于表现用户所采取的操作。

最后，Then 语句用于表现用户之前采取的操作所期望的结果。一旦你定义了所有的场景，就可以为每个场景编写测试。

BDD 测试框架的示例包括 Cucumber、JBehave 和 SpecFlow。
