# --description--

Sifa za `async` na `defer` katika vipengele vya HTML vya `script` zina jukumu muhimu katika jinsi mafaili ya JavaScript yanavyopakuliwa na kutekelezwa katika kurasa za mtandao. Kuelewa hizi kunaweza kuboresha utendaji wa tovuti yako na uzoefu wa mtumizi. Unapojumuisha `script` katika faili yako ya HTML, inaonekana kama hii:

```js
<script src="example.js"></script>
```

Wakati kivinjari kinapokutana na lebo hii ya `script`, kinakoma kusoma HTML, kinapakua script, kinatekeleza, kisha kinaendelea kusoma HTML. Hii inaweza kuchelewesha upakiaji wa ukurasa wako wa mtandao, hasa ikiwa una script kubwa.

Hapa ndipo sifa za `async` na `defer` zinapoingia. Zinatoa njia za kupakua script kwa ufanisi zaidi.

Tuanze na sifa ya `async`:

```js
<script src="example.js" async></script>
```

Kwa kuongeza sifa ya `async` kwenye lebo ya `script`, kivinjari kitaendelea kusoma HTML wakati script inapakuliwa. Mara script inapopakuliwa kikamilifu, kivinjari kitasitisha kusoma HTML, kinatekeleza script, kisha kinaendelea kusoma HTML. Hii inaweza kuharakisha sana upakiaji wa ukurasa.

Ni muhimu kuelewa kuwa script za async hufanyika mara tu zinapopakuliwa, ambayo ina maana huenda zisifanyike kwa mpangilio sahihi tunayotaka. Hapa ndipo sifa ya `defer` inapoleta msaada. Tazama jinsi sifa ya `defer` inavyoonekana:

```js
<script src="example.js" defer></script>
```

Sifa ya `defer` ni sawa na sifa ya `async`. Hata hivyo, script za defer hazifanyiki mara moja baada ya kupakuliwa. Badala yake, husubiri hadi kusoma HTML kukamilike. Zaidi ya hayo, script za defer hufanyika kwa mpangilio wa kuonekana kwao katika msimbo wa HTML.

Kwa kifupi, tumia `async` kwa script ambazo mpangilio wa utekelezaji haujali, na tumia `defer` unapohitaji kuhakikisha script zinafanyika kwa mpangilio sahihi. Sifa zote mbili zinaweza kuboresha sana muda wa upakiaji wa ukurasa kwa kuruhusu kivinjari kuendelea kusoma HTML wakati script zinapakuliwa nyuma ya pazia.
