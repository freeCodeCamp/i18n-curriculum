# --description--

Si has trabajado en algo, como una tesis, donde has guardado copias después de hacer cambios significativos y nombrado los archivos algo así como `Thesis-Draft`, `Thesis-Final`, `Thesis-Final-For-Real` y `Thesis-Absolute-Final-DO-NOT-CHANGE`, has usado un enfoque muy rudimentario para el control de versiones. ¿Pero qué tal si te digo que hay una mejor manera?

Puede que hayas oído hablar de herramientas como Git, SVN o Mercurial antes. Estos son sistemas de control de versiones, y son un componente clave en el flujo de trabajo de todo desarrollador. Un sistema de control de versiones te permite rastrear y gestionar los cambios en tu proyecto. Para los propósitos de estas lecciones, nos enfocaremos en Git.

Git ofrece herramientas poderosas que te permiten gestionar el historial de edición de un proyecto. Aprenderás más sobre cómo usar Git y las funciones que ofrece en las próximas lecciones. Por ahora, esto es lo que realmente necesitas saber:

Git te permite hacer cosas como crear commits, que son una instantánea de un estado específico de tu base de código. Luego puedes comparar commits para ver qué cambios has hecho, revertir un commit que tenga cambios incorrectos y más.

También puedes crear "branches", que puedes pensar como diferentes caminos en tu código. Mientras que una branch podría ser el camino por el que caminas, los commits son los puntos de referencia a lo largo del recorrido. Las branches te permiten hacer cosas como trabajar en una nueva función de forma aislada, de modo que si tu trabajo se complica, puedes eliminar la branch sin afectar la estructura principal del código.

Estas características hacen que el control de versiones sea un componente esencial para la colaboración con otros desarrolladores. Cuando tienes varias personas trabajando en el mismo proyecto, las ramas permiten que cada una trabaje en su propio código sin interferir con los cambios de los demás.

A menudo combinarás un sistema de control de versiones como Git con un proveedor de control de versiones como GitHub o Gitlab, que te ofrece herramientas adicionales de colaboración y la capacidad de respaldar tus proyectos en la nube.

Pero incluso sin una solución de almacenamiento en la nube, los sistemas de control de versiones pueden ser un cambio radical para tus flujos de trabajo. No más archivos `THIS-IS-THE-FINAL-COPY-FOREVER`.

¡Hay mucha información interesante para cubrir sobre el uso de estos sistemas, así que hagamos un análisis profundo en las próximas lecciones!
