# --description--

Il paradigma divide and conquer in informatica è una tecnica per suddividere ricorsivamente i problemi in sotto-problemi più piccoli. Uno degli aspetti chiave di questa tecnica è la ricorsione, che avviene quando una funzione si chiama ripetutamente fino a raggiungere un caso base. In questa lezione, esamineremo l’algoritmo merge sort per capire meglio come funziona la tecnica divide and conquer.

Supponiamo di avere questa lista di numeri:

```md
42 37 53 17
```

L’obiettivo è ordinare quella lista dal più piccolo al più grande usando l’algoritmo merge sort. Il primo passaggio è dividere quella lista a metà:

```md
42 37 | 53 17
```

Poi dobbiamo guardare il lato sinistro della lista:

```md
42 37
```

Prendiamo quella sotto-lista e la dividiamo di nuovo a metà finché ogni sotto-lista contiene solo un elemento:

```md
42 | 37
```

Una lista con un solo elemento è ordinata per impostazione predefinita. Successivamente dobbiamo unire ciascuna di quelle sotto-liste di un elemento in una lista ordinata:

```md
37 42
```

Poi seguiamo lo stesso processo per il lato destro della lista originale:

```py
# right side of original list
53 17

# divide the list in half
53 | 17

# merge the lists in sorted order
17 53
```

Ora che entrambe le metà della lista originale sono ordinate, uniamo quelle due metà insieme e ordiniamo gli elementi:

```markdown
17 37 42 53
```

Ecco come appare l’algoritmo nel codice:

```py
def merge_sort(arr):
    if len(arr) <= 1:
        return arr

    mid = len(arr) // 2
    left = merge_sort(arr[:mid])
    right = merge_sort(arr[mid:])

    sorted_list = []
    i = 0
    j = 0

    while i < len(left) and j < len(right):
        if left[i] <= right[j]:
            sorted_list.append(left[i])
            i += 1
        else:
            sorted_list.append(right[j])
            j += 1

    sorted_list.extend(left[i:])
    sorted_list.extend(right[j:])

    return sorted_list
```

La complessità temporale del merge sort sarebbe `O(n log n)` perché la lista viene continuamente divisa a metà (`log n`) e poi unita (`O(n)`). A differenza di altri algoritmi di ordinamento come il bubble sort, il merge sort non ordina in loco e ha una complessità spaziale di `O(n)`.
