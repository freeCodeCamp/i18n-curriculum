# --description--

在上一课中，你学习了服务器处理一个 HTTP 请求并发送回一个响应。该响应包含你在网页上看到的各种资源。

HTTP 响应返回的资源包括基于文本的资源、媒体资源、字体等。每个资源都带有一个 **Content-Type** 头部，告诉浏览器如何正确处理和显示该资源。

基于文本的资产一直是网络的基本组成部分。事实上，网络最初就是纯文本。如今，基于文本的资产包括以下内容：

* **超文本标记语言 (HTML)**：任何网页的基本结构。没有 HTML，网页无法存在。
    
* **CSS (层叠样式表)**：控制网页外观的资源。通过 CSS，你可以将颜色、字体和版型等样式应用到 `HTML`。
    
* **JavaScript**：JavaScript 为超文本标记语言和 CSS 添加交互性和动态内容。
    
* **JSON 和 XML**：这两种形式让你在服务器和浏览器之间传输结构化数据。
    
媒体资源包括图像、音频和视频。它们通过视觉和声音使网页生动起来。

许多网站依赖图像来传达视觉信息。图像有不同的格式，如 `JPG`、`JPEG`、`PNG`、`WEBP` 和 `GIF`。

像 YouTube 和 Vimeo 这样的网站提供 `MP4`、`MOV`、`AVI`、`WebM` 等格式的视频内容。

音频是许多平台的重要组成部分，因为它让用户能够流式传输音乐、播客和其他声音内容。常见的音频形式包括 `MP3`、`AAC`、Ogg 和 `WAV`。像 Spotify 和 YouTube Music 这样的平台依赖这些形式来提供高质量的音频体验。

在 HTTP `Response` 中返回的其他资源包括像 `PDF`、`DOCX` 和 `PAGES` 这样的文档，像 `ZIP` 和 `RAR` 这样的压缩档案，像 `WOFF`、`WOFF2` 和 `TTF` 这样的字体文件，以及像 `EXE` 这样的可执行文件。

这是这些常见资产及其内容类型的表格：

| Asset Type | Content-Type Header |
| --- | --- |
| HTML | text/html |
| CSS | text/css |
| JavaScript | application/javascript |
| JPEG | image/jpeg |
| PNG | image/png |
| GIF | image/gif |
| SVG | image/svg+xml |
| WebP | image/webp |
| WOFF | font/woff |
| WOFF2 | font/woff2 |
| TTF | font/ttf |
| OTF | font/otf |
| MP4 | video/mp4 |
| WebM | video/webm |
| Ogg (Video) | video/ogg |
| MP3 | audio/mpeg |
| AAC | audio/aac |
| WAV | audio/wav |
| Ogg (Audio) | audio/ogg |
| JSON | application/json |
| XML | application/xml |
| PDF | application/pdf |
| ZIP | application/zip |
