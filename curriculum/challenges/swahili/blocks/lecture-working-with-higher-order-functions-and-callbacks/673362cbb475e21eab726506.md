# --interactive--

Uunganishaji wa mfululizo wa njia ni mbinu ambapo unaita njia kadhaa mfululizo moja baada ya nyingine. Unaweza kutumia uunganishaji wa mfululizo wa njia kwa aina nyingi za thamani katika JavaScript ikiwa ni pamoja na mfuatano wa herufi, safu za data, na vitu. Ingawa mfuatano wa herufi ni thamani za msingi, JavaScript kwa muda mfupi huziweka ndani ya kitu cha `String` wakati unapotumia njia ya mfuatano wa herufi.

Tuchunguze mfano tukitumia njia tofauti za mfuatano wa herufi:

:::interactive_editor

```js
const result = "  Hello, World!  "
  .trim()
  .toLowerCase()
  .replace("world", "JavaScript");

console.log(result); // "hello, JavaScript!"
```

:::

Katika mfano huu, tunaanza na mfuatano wa herufi na kufanya operesheni tatu mfululizo: futa nafasi za ziada, badilisha kuwa herufi ndogo, na badilisha mfuatano wa herufi `world` na `JavaScript`. Kila njia hurudisha mfuatano mpya wa herufi, ambao hutumika kama lengwa la wito wa njia inayofuata.

Uunganishaji wa mfululizo wa njia unaweza kuboresha sana usomaji wa msimbo wakati unafanya kazi na operesheni ngumu.

Kwa mfano, chukulia mfano huu ukitumia njia za `filter`, `map` na `reduce`:

:::interactive_editor

```js
const transactions = [
  { amount: 100, type: "credit" },
  { amount: 20, type: "cash" },
  { amount: 150, type: "credit" },
  { amount: 50, type: "cash" },
  { amount: 75, type: "credit" }
];

const totalCreditWithBonus = transactions
  .filter((transaction) => transaction.type === "credit")
  .map((transaction) => transaction.amount * 1.1)
  .reduce((sum, amount) => sum + amount, 0);

console.log(totalCreditWithBonus); // 357.5
```

:::

Katika mfano huu, tuna safu ya data ya miamala ambapo kila kitu kina kiasi na aina ya kadi ya mkopo au pesa taslimu.

Kwanza tunachuja miamala na kuunda safu mpya ya miamala ya kadi ya mkopo pekee. Kisha, tunaunganisha njia ya `map` kwa matokeo yaliyopangwa na kwa kila kiasi cha muamala, tunazidisha kwa `1.1` ambayo inawakilisha bonasi ya `10%`.

Kisha, tunachukua matokeo hayo na kuunganisha njia ya `reduce` kuongeza kila kiasi ambacho husababisha `357.5`.

Ingawa uunganishaji wa mfululizo wa njia unaweza kufanya msimbo kuwa mfupi na rahisi kusoma, ni muhimu kuitumia kwa busara.

Mfululizo mrefu sana unaweza kuwa mgumu kurekebisha makosa, kwani si wazi mara moja ni hatua gani katika mfululizo inaweza kusababisha tatizo. Mara nyingi ni mbinu nzuri kugawanya mfululizo mrefu sana katika hatua nyingi kwa uwazi bora na kurekebisha makosa kwa urahisi.

Unaweza pia kuunganisha njia kwenye kitu. Katika kesi hii, kila njia hurudisha `this`, ambayo inahusu kitu cha sasa, kuruhusu njia inayofuata katika mfululizo ifanye kazi juu yake.

:::interactive_editor

```js
const calculator = {
  total: 0,
  add(n) {
    this.total += n;
    return this;
  },
  multiply(n) {
    this.total *= n;
    return this;
  },
  subtract(n) {
    this.total -= n;
    return this;
  },
  getResult() {
    return this.total;
  }
};

const result = calculator.add(5).multiply(2).subtract(3).getResult();
console.log(result); // 7
```

:::

Kwa njia hii, kila njia hurudisha kitu hicho hicho, hivyo unaweza kuendelea kuunganisha wito moja baada ya mwingine.

Nini kitakuwa matokeo ya msimbo ufuatao?

```js
let obj = {
    value: 1,
    increment: function() {
        this.value++;
        return this;
    },
    double: function() {
        this.value *= 2;
        return this;
    },
    getValue: function() {
        return this.value;
    }
};

let result = obj.increment().double().increment().getValue();
console.log(result);
```
