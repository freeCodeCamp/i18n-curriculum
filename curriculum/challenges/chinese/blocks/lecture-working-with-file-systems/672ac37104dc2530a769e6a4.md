# --description--

当你开始构建自己的网页应用时，你需要注意你为你的文件命名。

你会希望以一种易于理解和维护的方式命名你的 `files`。

让我们来看一些 HTML 文件的好与坏文件名示例。请注意，这些示例中的 `.html` 扩展名告诉我们这是一个 HTML 文件。

这是一个错误文件名的示例： 

```md
index1234.html
```

仅凭名称无法明确该文件的内容。该文件可能用于产品页面、博客页面或任何其他类型的页面，但我们无法从文件名中得知。

这是一个更好的 HTML 文件名示例：

```md
about-us.html
```

通过使用像 `about-us.html` 这样更具描述性的名称，可以清楚地知道该文件的内容。

你还会注意到这个文件名中没有空格。相反，你使用连字符 `-` 来分隔单词。 

有时你可能会看到一个名为 `index.html` 的文件名。这个特殊的文件名用于表现网站的主页。当你访问一个网站时，`index.html` 文件是第一个被装载的文件。

命名文件时另一个重要的考虑因素是特殊字符的使用。

使用特殊字符、数字和字母的混合可能会使理解该 `file` 的内容变得困难。

这是一个错误文件名的示例：

```md
file-1!@.html
```

这个名称看起来不必要地复杂，并且没有给我们任何关于该文件内容的简介。

因此，最好只在你的文件名中使用字母和短横线。

当你开始学习更多语言时，你会发现这些语言中也有命名 `file` 的通用约定。 

在这些情况下，最好始终查阅官方文档或其他资源，以了解该语言中命名文件的最佳实践。

你也会遇到在团队中工作并且需要遵循团队为命名文件所制定的约定的情况。 

始终参考团队的风格指南，以确保你的 `file` 名称与团队其他成员保持一致。
