# --interactive--

多くのユーザーは、身体的な障害、反復性の負傷、または個人的な好みのためにマウスではなくキーボードを使っています。これにはスクリーンリーダーのユーザーや、マウスが使えない場合も含まれます。  
キーボードアクセシビリティは、こうしたユーザーがウェブアプリケーションを障壁なく効果的に操作できるようにします。

ここでは、ウェブアプリケーションをキーボードでアクセスしやすくするために使える実践的な手法を見ていきましょう。

多くのユーザーは、ウェブページ上のインタラクティブな要素を移動するために`Tab`キーを使います。デフォルトでは、ブラウザはリンク、ボタン、フォームフィールドなどの要素をHTMLに現れる順番でタブ移動できるようにしています。これを自然なタブ順序と呼びます。

時には、フォーカス可能な要素を調整したり、フォーカスの順序を変えたい場合があります。`tabindex`属性を使うとこれができます。

基本的な構文は以下の通りです。

```html
<element tabindex="number">Element Text</element>
```

`tabindex`の値は、キーボードナビゲーションでの要素の振る舞いを決定します：

`tabindex="0"`は要素を自然なタブ順序に追加します。

タブ移動は`button`から`div`、そしてリンクへとHTMLの順序に従ってフォーカスが移動します。

プレビューウィンドウの空白部分をクリックしてください。次に`Tab`キーを使うと、異なる要素間でフォーカスが移動するのがわかります。例を操作するにはインタラクティブエディターを有効にしてください。

:::interactive_editor

```html
<button>First</button>
<div tabindex="0">Second</div>
<a href="#">Third</a>
```

:::

`tabindex="-1"`はプログラム的に要素をフォーカス可能にします。これは通常フォーカスできない見出し、コンテナ、ダイアログ、エラーメッセージなどの要素のフォーカス管理に役立ちます：

```html
<p tabindex="-1">Sorry, there was an error with your submission.</p>
```

この例では、段落は通常のタブ順序に含まれていないため、ユーザーは`Tab`キーを押してもそこに到達できません。しかし、スクリプトでこの要素にフォーカスを設定すれば、メッセージがユーザーの注意を引きます。この手法はJavaScriptのレッスンでさらに学びます。

`tabindex`が`0`より大きい場合、カスタムのタブ順序が設定されます。つまり、より小さい正の値を持つ要素が先にフォーカスされます。

この例では、タブ移動はHTMLの順序に関係なく、`input`（`tabindex="1"`付き）、次に`2`、そして`3`にフォーカスします。

```html
<input tabindex="2">
<input tabindex="1">
<input tabindex="3">
```

複雑なウィジェット、例えば特定のナビゲーション順序が必要なツールバーなどではカスタムの正の値が使われることがあります。しかし、この方法はページが大きくなったり変化したりするとナビゲーションが混乱しやすく、保守が難しくなるため推奨されません。

`accesskey`はウェブプロジェクトをキーボードでアクセスしやすくするために使える別の属性です。特定の要素にフォーカスを当てたり起動したりするキーを定義できます。

以下は、下記の説明に従って試せるインタラクティブな例です（まずインタラクティブエディターを有効にしてください）：

- `accesskey="s"`は`S`キーを`Save`ボタンに割り当てます。ほとんどのブラウザでは、`ALT + S`（Windows）や`CTRL + Option + S`（Mac）を押すとこのボタンが起動します。
- `accesskey="c"`は`C`キーを`Cancel`ボタンに設定し、`ALT + C`（Windows）や`CTRL + Option + C`（Mac）で起動できます。
- `accesskey="h"`は`H`キーを`Home`リンクに割り当て、`ALT + H`（Windows）や`CTRL + Option + H`（Mac）でホームページに移動できます。

アクセスキーを起動する正確なキーの組み合わせはブラウザやOSによって異なる場合があります。通常はWindowsでは`ALT + Specified Key`、Macでは`CTRL + Option + Specified Key`です。

:::interactive_editor

```html
<button accesskey="s">Save</button>
<button accesskey="c">Cancel</button>
<a href="index.html" accesskey="h">Home</a>
```

:::

アプリでキーボードアクセスを向上させるもう一つの方法は、明確なフォーカスインジケーターを提供することです。デフォルトのブラウザのフォーカスインジケーターが不十分だと感じたら、要素のフォーカス状態をターゲットにしてオーバーライドできます。

以下は`button`要素のフォーカス状態をスタイル付けした例です。プレビューウィンドウの空白部分をクリックし、続けて`Tab`キーを押してボタンにフォーカスしてください。例を操作するにはインタラクティブエディターを有効にしてください。

:::interactive_editor

```html
<link href="styles.css" rel="stylesheet">

<button>Example button</button>
```

```css
button:focus {
  outline: 2px solid #005fcc;
}
```

:::

`outline`プロパティは要素の周囲のアウトラインを定義するために使われます。この例ではアウトラインを2ピクセルの青の実線に設定しています。フォーカスインジケーターは、現在どの要素にフォーカスがあるかが明確にわかるようにスタイル付けする必要があります。アクセシビリティを確保するために、インジケーターは背景色と少なくとも3:1の最小色コントラストを持つ必要があります。

また、モーダルやポップアップなどのコンポーネントで、ユーザーが特定の要素からフォーカスを移動できなくなるキーボードトラップは避けてください。
