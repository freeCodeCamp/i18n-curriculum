# --description--

讓我們來看看在無障礙存取的背景關係中撰寫良好連結文字的利益，以及一些良好連結文字的範例。

良好聯結文字的第一個明顯利益是它讓每個人更容易快速找到資訊。描述性聯結幫助使用者知道他們將前往何處以及將存取什麼。這確保使用者不會感到迷失，並提升整體使用者體驗。

對於使用螢幕閱讀器的人來說，清晰且具描述性的聯結文字是必須的。螢幕閱讀器會朗讀聯結文字，因此像「閱讀我們的無障礙指南」這樣的文字比「點擊這裡」好得多。

使 `link text` 清晰且具描述性不僅對視覺障礙者有益。具描述性的聯結也透過提供清楚的背景關係幫助認知障礙者。

在撰寫聯結文字時，請記住以下一些最佳實踐：

- 確保聯結具有視覺上的區別，例如使用底線和其他視覺提示，讓使用者能輕鬆識別並導覽它們。
- 避免使用像「here」、「click here」和「more-info」這類泛型的聯結文字，因為它們不提供任何有用的資訊。
- 目標是簡潔且具描述性的聯結文字，理想長度為 2-5 個字，能傳達聯結的目的。
- 避免使用使用者可能不理解的行話和縮寫。
- 專注於目的地，而非動作。例如，「使用者行為結果」，而非「點擊此處閱讀更多」。
- 不要為不同的目的地重複相同的聯結文字。
- 將聯結放置在周圍文字中合理的位置。例如，使用「欲了解更多詳情，請造訪我們的事件頁面」而非「點擊這裡了解更多」。

以下是針對特定使用案例的良好聯結文字範例，與較不具幫助性的範例比較。假設你想要連結到提供關於某個事件詳細資訊的頁面，如下所示：

```html
<a href="webinar-details-link">Details</a>
```

`Details` 含糊不清，沒有提供使用者點擊該連結後會看到什麼具體資訊。若沒有額外的背景關係，使用者可能不知道該連結是導向有關網路研討會、產品、政策或其他內容的詳細資料。

現在，這裡有一個良好聯結文字的範例：

```html
<a href="webinar-details-link">
  Get details about our upcoming webinar
</a>
```

這段聯結文字為使用者提供他們將會找到的內容的背景關係，使他們更容易決定是否要點擊。它透過明確指出該聯結與網路研討會相關，減少了歧義。

這裡是另一個連結到部落格文章的範例：

```html
<a href="/blog-post-link">Read more</a>
```

聯結文字 `Read more` 在存取性方面並不理想，因為它缺乏背景關係。

這裡有一個更好的聯結文字範例：

```html
<a href="/blog-post-link">
   Read our latest blog post on web accessibility
</a>
```

此更新的聯結文字讓使用者更清楚知道預期內容及點擊該聯結的原因，這對使用螢幕閱讀器的使用者尤其重要。

讓我們再看一個範例。假設你想提供關於某個主題的更多資訊：

```html
<a href="/link-to-topic">More info</a>
```

`More info` 也含糊不清，且可能對無障礙性造成問題。它沒有提供關於聯結目的地的具體細節，這可能會讓依賴螢幕閱讀器的使用者或以非線性方式瀏覽網站的使用者感到困惑。

這裡有一個更好的聯結文字範例：

```html
<a href="/link-to-topic">
   Learn more about our accessibility efforts
</a>
```

這個更新過的聯結文字較佳，因為它清楚描述使用者如果點擊該聯結會找到什麼。使用者可以立即了解該聯結會導向關於你的無障礙努力的資訊。
