# --description--

이전 수업에서 단일 밑줄과 이중 밑줄로 속성에 접두사를 붙이는 방법을 배웠습니다.

둘의 차이를 다시 상기시키자면, 단일 밑줄은 클래스 내부에서만 사용하기 위한 속성임을 나타내는 관례로, 클래스 외부에서 직접 접근하지 말아야 한다는 의미입니다. 반면 이중 밑줄은 클래스 외부에서 해당 속성에 직접 접근하는 것을 방지합니다.

다음은 두 가지가 어떻게 작동하는지 보여주는 예시입니다:

```py
class Example:
    def __init__(self):
        self._internal = 'I can be accessed from outside the class, but should not'
        self.__private = 'You cannot access me directly from outside the class'

obj = Example()

print(obj._internal) # I can be accessed from outside the class, but should not
print(obj.__private)  # AttributeError: 'Example' object has no attribute '__private'
```

속성에 이중 밑줄을 접두사로 붙이면 Python의 이름 맹글링 과정이 시작되는데, 이 과정에서 Python은 내부적으로 속성 이름 앞에 밑줄과 클래스 이름을 붙여 `__attribute`를 `_ClassName__attribute`로 바꿉니다.

이를 확인하려면 클래스의 인스턴스를 만들고, 그 인스턴스의 `__dict__` 특수 속성을 사용하세요. 이 속성은 객체의 속성을 담은 사전입니다:

```py
class Example:
    def __init__(self, internal, private):
        self._internal = internal
        self.__private = private

example1 = Example(
    'I can be accessed from outside the class, but should not',
    'I cannot be accessed directly from outside the class'
)

print(example1.__dict__)
```

결과는 다음과 같습니다:

```py
{
  '_internal': 'I can be accessed from outside the class, but should not',
  '_Example__private': 'I cannot be accessed directly from outside the class'
}
```

보시다시피 `__private` 속성은 `_Example__private`로 저장되어 있습니다. 즉, 클래스 외부에서도 다음과 같이 해당 속성에 접근할 수 있습니다:

```py
class Example:
    def __init__(self, internal, private):
        self._internal = internal
        self.__private = private

example1 = Example(
    'I can be accessed from outside the class, but should not',
    'I cannot be accessed directly from outside the class'
)
example2 = Example(
    'I should not be accessed from outside the class',
    'But I can be accessed from outside the class with name mangling'
)

print(example1._Example__private) # I cannot be accessed directly from outside the class
print(example2._Example__private) # But I can be accessed from outside the class with name mangling
```

그렇다면 Python이 이름 맹글링을 하는 이유는 무엇일까요?

이름 맹글링의 주요 목적은 상속을 사용할 때 속성과 메서드가 실수로 재정의되는 것을 방지하는 것입니다. 다음 예시가 이를 명확히 보여줍니다:

```py
class Parent:
    def __init__(self):
        self.__data = 'Parent data'

class Child(Parent):
    def __init__(self):
        super().__init__()
        self.__data = 'Child data'

c = Child()
print(c.__dict__) # {'_Parent__data': 'Parent data', '_Child__data': 'Child data'}
```

`Parent` 클래스와 이를 상속받은 `Child` 모두 각각 별도의 `_class__data` 속성을 가지고 있음을 알 수 있습니다. 이는 이름 맹글링 덕분에 가능한 일입니다. 그렇지 않으면 `Child`가 부모 클래스의 데이터를 실수로 덮어썼을 것입니다.

만약 두 클래스의 속성에 이중 밑줄 접두사를 붙이지 않고 Python이 이름 맹글링을 하지 못하게 한다면 다음과 같은 일이 벌어집니다:

```py
class Parent:
   def __init__(self):
       self.data = 'Parent data'

class Child(Parent):
   def __init__(self):
       super().__init__()
       self.data = 'Child data'

c = Child()
print(c.__dict__)  # {'data': 'Child data'}
```

그렇다면 단일 밑줄(`_`)과 이중 밑줄(`__`) 중 어떤 것을 속성 접두사로 사용해야 할까요? 상황에 따라 다릅니다. 속성이 클래스 내부에서만 사용될 예정이라면 단일 밑줄을 사용하세요.

하지만 상속될 클래스라면 부모의 속성이 덮어쓰이지 않도록 이중 밑줄을 사용하는 것이 좋습니다.
