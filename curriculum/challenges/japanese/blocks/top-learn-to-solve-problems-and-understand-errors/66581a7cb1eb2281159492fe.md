# --description--

次のコードを書いたと仮定しましょう。

```javascript
const a = "Hello";
const b = "World";

console.log(c);
```

このコードは実行されますが、エラーが発生します。技術的には、これはエラーを<dfn>スローする</dfn>と言います。エラーの最初の部分はエラーの種類を表示します。これは何が起きているかの最初の手がかりになります。レッスンの後半でさまざまなエラーの種類について詳しく学びます。この例では、`ReferenceError`があります。

<img src="https://cdn.statically.io/gh/TheOdinProject/curriculum/175b5ef2a1b4758a7b75f4ef43d7e27203e5707b/foundations/javascript_basics/understanding_errors/imgs/00.png" width="100%" alt="開発者コンソールに表示されたエラー"/>

`ReferenceError`は、現在のスコープ内で宣言および初期化されていない変数を参照したときにスローされます。この場合、エラーメッセージは`c is not defined`が原因でエラーが発生したことを説明しています。

このタイプのエラーは、`ReferenceError`の原因によって異なるメッセージが表示されます。例えば、別のメッセージとして`ReferenceError: can't access lexical declaration 'X' before initialization`が表示される場合があります。

エラーの次の部分では、エラーが発生したファイル名（この場合は`script.js`）と行番号が示されます。

これにより、コードの問題のある行に簡単に移動できます。ここでは、エラーは`script.js`の4行目から発生しており、エラーメッセージの下に`script.js:4`というテキストのリンクとして表示されています。このリンクをクリックすると、ほとんどのブラウザは開発者ツールのSourcesタブで該当のコード行とスクリプトの残りの部分に移動します。

ブラウザのコンソールによっては、エラーが発生している行の列（または文字位置）も表示される場合があります。この例では、`script.js:4:13`となります。
