# --description--

在 Express 中，`res`（回應）物件用來在處理完收到的請求後，將回應送回給客戶端。Express 提供了多種內建方法來傳送不同類型的回應。舉例來說，你可以傳送純文字、JSON 資料、將使用者重新導向到另一個 URL，或是呈現模板。

其中最常用的回應方法之一是 `res.send()`：

```javascript
app.get("/text", (req, res) => {
  res.send("Hello, World!");
});
```

`res.send()` 方法會將回應送回給客戶端。它可以傳送純文字、HTML、JSON，甚至是二進位資料。在這個範例中，當客戶端造訪 `/text` 時，伺服器會回應文字 `Hello, World!`。

另一個常用的回應方法是 `res.json()`。

這個方法會傳送 JSON 回應給客戶端。當你在建置 API 時特別有用，因為你經常需要回傳結構化的資料，例如物件或陣列。Express 會自動將 JavaScript 物件轉換成 JSON 格式的字串：

```javascript
app.get("/user", (req, res) => {
  res.json({ name: "John Doe", age: 30 });
});
```

如果客戶端送出請求到 `/user`，伺服器會回應以下內容：

```json
{
  "name": "John Doe",
  "age": 30
}
```

當你需要控制回應的 HTTP 狀態碼時，可以使用 `res.status()`。這個方法會設定狀態碼，通常會與其他回應方法像是 `res.send()` 或 `res.json()` 串鏈使用：

```javascript
app.get("/notfound", (req, res) => {
  res.status(404).send("Not Found");
});
```

在這裡，伺服器將狀態碼設定為 `404`，並將訊息 `Not Found` 傳回給客戶端。

如果你想將使用者導向到不同的 URL，可以使用 `res.redirect()`：

```javascript
app.get("/oldpage", (req, res) => {
  res.redirect("/newpage");
});
```

當客戶端造訪 `/oldpage` 時，他們會自動被重新導向到 `/newpage`。

最後，`res.render()` 用於搭配模板引擎，例如 Pug、EJS 或 Handlebars。它會呈現一個視圖模板，並將產生的 HTML 傳送給客戶端：

```javascript
app.get("/profile", (req, res) => {
  res.render("profile", { username: "john_doe" });
});
```

在這個範例中，伺服器呈現 `profile` 視圖，並傳入 `username` 變數，該變數可以在 HTML 中顯示。

這些回應方法讓你能根據請求的性質回傳不同類型的資料。無論你是傳送文字、JSON 資料，還是重新導向客戶端，Express 都有一套靈活的工具來處理。
