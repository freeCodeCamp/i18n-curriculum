## 문자열 생성자와 `toString()` 메서드

- **정의**: 문자열 객체는 문자 수열을 나타내는 데 사용됩니다. 문자열 객체는 원시 값을 객체로 감싸는 `String` 생성자 함수를 사용해 만듭니다.

:::interactive_editor

```js
const greetingObject = new String("Hello, world!");

console.log(typeof greetingObject); // "object"
```

:::

- **`toString()` 메서드**: 이 메서드는 값을 문자열 표현으로 변환합니다. 숫자, 불리언, 배열, 객체에 사용할 수 있는 메서드입니다.

:::interactive_editor

```js
const num = 10;
console.log(num.toString()); // "10"

const arr = [1, 2, 3];
console.log(arr.toString()); // "1,2,3"
```

:::

이 메서드는 2에서 36 사이의 숫자인 선택적 기수(radix)를 받습니다. 이 기수는 2진수의 경우 2, 8진수의 경우 8처럼 진법을 나타냅니다. 기수를 지정하지 않으면 기본값은 10진수인 10입니다.


:::interactive_editor

```js
const num = 10;
console.log(num.toString(2)); // "1010"(binary)
```

:::

## 숫자 생성자

- **정의**: `Number` 생성자는 숫자 객체를 만드는 데 사용됩니다. 숫자 객체는 `isNaN`와 `toFixed` 메서드 같은 몇 가지 유용한 속성과 메서드를 포함합니다. 대부분의 경우 다른 데이터 타입을 숫자 타입으로 변환할 때 `Number` 생성자를 사용합니다.

:::interactive_editor

```js
const myNum = new Number("34");
console.log(typeof myNum); // "object"

const num = Number('100');
console.log(num); // 100

console.log(typeof num); // number
```

:::

## 변수와 함수 이름 짓기 모범 사례

- **카멜케이스**: 관례상 JavaScript 개발자는 변수와 함수 이름에 카멜케이스를 사용합니다. 카멜케이스는 첫 단어는 모두 소문자이고, 뒤따르는 단어들은 첫 글자가 대문자인 방식입니다. 예: `isLoading`.

- **불리언 이름 짓기**: 불리언 변수에는 "is", "has", "can" 같은 접두사를 붙이는 것이 일반적입니다.

```js
let isLoading = true;
let hasPermission = false;
let canEdit = true;
```

- **함수 이름 짓기**: 함수 이름은 함수가 하는 일을 명확히 나타내야 합니다. 불리언을 반환하는 함수(보통 술어 함수라고 함)에는 "is", "has", "can" 접두사를 사용할 수 있습니다. 데이터를 가져오는 함수는 보통 "get"으로 시작하고, 데이터를 설정하는 함수는 "set"으로 시작합니다. 이벤트 핸들러 함수는 "handle"을 접두사로 붙이거나 "Handler"를 접미사로 붙이는 경우가 많습니다.

```js
function getUserData() { /* ... */ }

function isValidEmail(email) { /* ... */ }

function getProductDetails(productId) { /* ... */ }

function setUserPreferences(preferences) { /* ... */ }

function handleClick() { /* ... */ }
```

- **반복문 내 변수 이름 짓기**: 반복문에서 반복자 변수 이름은 보통 `i`, `j`, `k` 같은 한 글자 변수를 사용합니다.

```js
for (let i = 0; i < array.length; i++) { /* ... */ }
```

## 희소 배열 다루기

- **정의**: 빈 슬롯이 있는 배열을 가질 수 있습니다. 빈 슬롯은 아무것도 없는 슬롯을 뜻합니다. 이는 값이 `undefined`인 배열 슬롯과 다릅니다. 이런 배열을 희소 배열이라고 합니다.

:::interactive_editor

```js
const sparseArray = [1, , , 4];
console.log(sparseArray.length); // 4
```

:::

## 린터와 포매터

- **린터**: 린터는 프로그래밍 오류, 버그, 스타일 오류, 의심스러운 구문을 표시하는 정적 코드 분석 도구입니다. 대표적인 린터로 ESLint가 있습니다.
- **포매터**: 포매터는 코드를 특정 스타일 가이드에 맞게 자동으로 포맷하는 도구입니다. 대표적인 포매터로 Prettier가 있습니다.

## 메모리 관리

- **정의**: 메모리 관리는 메모리를 제어하고 필요할 때 할당하며 더 이상 필요 없을 때 해제하는 과정입니다. JavaScript는 자동 메모리 관리를 사용합니다. 즉, JavaScript(더 정확히는 웹 브라우저의 JavaScript 엔진)가 메모리 할당과 해제를 대신 처리합니다. 코드에서 명시적으로 메모리를 해제할 필요가 없습니다. 이 자동 과정은 흔히 "가비지 컬렉션"이라고 부릅니다.

## 클로저

- **정의**: 클로저는 외부(포함하는) 렉시컬 범위에 있는 변수에 접근할 수 있는 함수로, 외부 함수가 반환된 후에도 그 변수를 참조할 수 있습니다.

:::interactive_editor

```js
function outerFunction(x) {
  let y = 10;
  function innerFunction() {
    console.log(x + y);
  }
  return innerFunction;
}

let closure = outerFunction(5);
closure(); // 15
```

:::

## `var` 키워드와 호이스팅

- **정의**: `var`는 2015년 이전에 변수를 선언하는 원래 방식이었습니다. 하지만 `var`는 범위, 재선언 등에서 문제가 있었습니다. 그래서 현대 JavaScript 프로그래밍에서는 `let`와 `const`를 대신 사용합니다.
- **`var`로 변수 재선언하기**: `let`로 변수를 재선언하면 `SyntaxError`가 발생합니다. 하지만 `var`로는 변수 재선언이 허용됩니다.

```js
// Uncaught SyntaxError: Identifier 'num' has already been declared 
let num = 19;
let num = 18;

var myNum = 5;
var myNum = 10; // This is allowed and doesn't throw an error

console.log(myNum) // 10
```

- **`var`와 범위**: `var`로 블록(예: `if` 문이나 `for` 반복문) 안에서 선언한 변수는 그 블록 밖에서도 접근할 수 있습니다.

:::interactive_editor

```js
if (true) {
  var num = 5;
}
console.log(num); // 5
```

:::

- **호이스팅**: 호이스팅은 JavaScript가 코드 실행 전에 컴파일 단계에서 선언을 해당 범위의 최상단으로 끌어올리는 기본 동작입니다. `var` 키워드로 변수를 선언하면 JavaScript가 선언을 범위 최상단으로 호이스팅합니다.

:::interactive_editor

```js
console.log(num); // undefined
var num = 5;
console.log(num); // 5
```

:::

함수 선언 구문으로 함수를 선언하면 함수 이름과 함수 본문 모두 호이스팅됩니다. 즉, 코드에서 함수 선언 전에 함수를 호출할 수 있습니다.

:::interactive_editor

```js
sayHello(); // "Hello, World!"

function sayHello() {
  console.log("Hello, World!");
}
```

:::

`let` 또는 `const`로 선언한 변수는 호이스팅되지만 초기화되지 않으며, 코드에서 실제 선언 전에 접근할 수 없습니다. 이 동작을 흔히 "일시적 사각지대"라고 부릅니다.

:::interactive_editor

```js
console.log(num); // Throws a ReferenceError
let num = 10;
```

:::

## 가져오기, 내보내기, 모듈 다루기

- **모듈**: 관련 함수, 클래스, 변수들을 캡슐화한 독립적인 코드 단위입니다. 모듈을 만들려면 JavaScript 코드를 별도의 파일에 작성합니다.
- **내보내기**: 애플리케이션의 다른 부분에서 사용하려는 변수, 함수, 클래스를 `export` 키워드로 명시적으로 내보내야 합니다. 내보내기에는 명명 내보내기와 기본 내보내기 두 가지 유형이 있습니다.
- **가져오기**: 내보낸 항목을 애플리케이션의 다른 부분에서 사용하려면 `import` 키워드로 가져와야 합니다. 가져오기 유형에는 명명 가져오기, 기본 가져오기, 네임스페이스 가져오기가 있습니다.

```js
// Within a file called math.js, we export the following functions:

// Named export
export function add(num1, num2) {
  return num1 + num2;
}

// Default export
export default function subtract(num1, num2) {
  return num1 - num2;
}

// Within another file, we can import the functions from math.js.

// Named import - This line imports the add function.
// The name of the function must exactly match the one exported from math.js.
import { add } from './math.js';

// Default import - This line imports the subtract function.
// The name of the function can be anything.
import subtractFunc from './math.js';

// Namespace import - This line imports everything from the file.
import * as Math from './math.js';

console.log(add(5, 3)); // 8
console.log(subtractFunc(5, 3)); // 2
console.log(Math.add(5, 3)); // 8
console.log(Math.subtract(5, 3)); // 2
```
