# --description--

設定訂製的身份驗證需要撰寫大量手動邏輯。你必須處理讀取請求標頭、擷取 cookies、驗證簽名以及管理錯誤。

自己建置這些功能對學習很有幫助，但每個生產應用都這麼做既耗時又容易忽略安全細節。這就是為什麼開發者會使用 Passport.js。

Passport.js 是 Node.js 的身份驗證中介層。它作為一個有組織的框架，標準化你如何驗證請求。它處理底層的身份驗證細節，讓你能專注於應用的業務邏輯。

Passport.js 的核心概念是每個應用需要不同的登入方式，但底層的身份驗證流程應該保持一致。Passport 透過稱為 **Strategies** 的模組化外掛程式實現這種彈性。

你不必為每種身份驗證方法撰寫訂製程式碼，而是安裝你需要的特定策略。現有超過 500 種策略，涵蓋幾乎所有現今使用的身份驗證方法。其中一些是：

- **passport-local：** 處理傳統的使用者名稱和密碼，對應你自己的資料庫進行驗證。

- **passport-jwt：** 透過擷取並驗證來自請求的 JSON Web Tokens 來保護 API 端點。

- **passport-google-oauth20：** 允許使用者使用現有的 Google 帳號立即登入。

- **passport-facebook：** 透過 OAuth 2.0 使用 Facebook 帳號登入。

- **passport-github2：** 允許使用者使用 GitHub 帳號立即登入。

透過將身份驗證拆解成不同策略，Passport 讓你的應用能無縫成長。如果你今天用使用者名稱/密碼系統，並決定下個月新增「用 Google 登入」，你不必重寫整個身份驗證程式碼。只要安裝 Google 策略並將它接入 Passport 即可。

要在 Express 應用中使用 Passport，你必須安裝核心 Passport 套件以及你計畫使用的特定策略。對於標準的使用者名稱和密碼設定，在終端機執行以下指令：

```bash
npm install passport passport-local
```

安裝完成後，你需要將 Passport 接入你的 Express 應用設定。這必須在你定義路由之前完成：

```js
const express = require('express');
const passport = require('passport');
const LocalStrategy = require('passport-local').Strategy;

const app = express();

// initialize Passport middleware
app.use(passport.initialize());

// configure Passport to use the Local Strategy
passport.use(new LocalStrategy((username, password, done) => {
    // your database look-up and password verification logic goes here
}));
```

Passport 作為中介層順利整合進標準的 Express 路由系統。當使用者提交登入路由的憑證時，Passport 攔截請求，將資料傳給你設定的策略，並評估結果。

```js
app.post('/login', passport.authenticate('local'), (req, res) => {
  // if this function runs, then authentication was successful
  res.send(`Welcome back, ${req.user.username}!`);
});
```

如果策略成功驗證使用者，Passport 會自動處理兩項關鍵任務：

1. **會話管理：** 它觸發輔助函式，將一小段使用者資料存入會話 cookie。

2. **請求裝飾：** 它將已驗證的使用者物件直接附加到傳入請求的 `req.user`。

一旦 `req.user` 被填入，應用中任何後續的中介層或路由處理常式都能立即知道誰已登入，並根據其身份做授權決策。

Passport 為你在 Node.js 應用中管理使用者身份提供了一個一致、安全且適合生產環境的基礎。
