# --description--

До цього моменту ви відтворювали компоненти React на клієнті. Зазвичай саме так ви й робитимете. Однак існують випадки, коли має сенс відтворювати компонент React на сервері. Оскільки React — це бібліотека для створення інтерфейсів на JavaScript, а JavaScript можна запускати на сервері за допомогою Node, це можливо. Насправді React надає метод `renderToString()`, який можна використати для цієї мети.

Існує дві ключові причини, чому у реальному застосунку може використовуватися відтворення на сервері. По-перше, без цього ваші React-застосунки складалися б із відносно порожнього HTML-файлу та великого пакету JavaScript при початковому завантаженні в браузер. Це може бути неідеально для пошукових систем, які намагаються індексувати вміст ваших сторінок, щоб люди могли вас знайти. Якщо ви відтворите ініціальну HTML-розмітку на сервері та надішлете її клієнту, початкове завантаження сторінки міститиме всю розмітку сторінки, яку можуть сканувати пошукові системи. По-друге, це створює швидший досвід початкового завантаження сторінки, оскільки відтворений HTML менший за JavaScript-код усього застосунку. React все одно зможе розпізнати ваш застосунок і керувати ним після початкового завантаження.

# --instructions--

Метод `renderToString()` надається на `ReactDOMServer`, який доступний тут як глобальний об’єкт. Метод приймає один аргумент — React-елемент. Використайте це, щоб відтворити `App` у рядок.

# --hints--
