# --description--

Mantener tus dependencias actualizadas—y saber cómo instalar una versión específica—es importante para mantener tu proyecto estable. Te ayuda a evitar errores inesperados y te da más control cuando se lanzan nuevas versiones.

Para instalar una versión específica de un paquete, usa este formato:

```bash
npm install package@version-number
```

Por ejemplo, si tu proyecto necesita una versión anterior de Express, digamos `4.21.2`, puedes instalarla así:

```bash
npm install express@4.21.2
```

Si no incluyes un número de versión, npm instalará automáticamente la última versión disponible.

Si no estás seguro de qué versiones existen, puedes verificarlas con este comando:

```bash
npm view package-name versions
```

Por ejemplo:

```bash
npm view express versions
```

Para actualizar paquetes, puedes actualizarlos individualmente o todos a la vez. Antes de hacerlo, es útil primero ver cuáles están desactualizados:

```bash
npm outdated
```

Esto te mostrará qué dependencias tienen versiones más recientes disponibles, y deberías ver una salida similar a esta:

```bash
user@Kolade ~/Desktop/fCC/script-code/node/npm/install-spec-version  % npm outdated
Package  Current  Wanted  Latest  Location              Depended by
chalk      5.5.0   5.5.0   5.6.2  node_modules/chalk    install-spec-version
cleave     1.0.1   1.0.1   1.0.2  node_modules/cleave   install-spec-version
express   4.21.2  4.21.2   5.2.1  node_modules/express  install-spec-version
```

Una vez que hayas verificado qué dependencias están desactualizadas, puedes comenzar a actualizarlas.

Para actualizar una sola dependencia, usa lo siguiente:

```bash
npm update package-name
```

Esto actualiza el paquete solo dentro del rango de versiones permitido por tu archivo `package.json`. Por ejemplo, para actualizar Express, puedes usar lo siguiente:

```bash
npm update express
```

Si quieres omitir el rango de versiones e instalar la última versión de un paquete, usa el siguiente comando:

```bash
npm install package-name@latest
```

Para actualizar todas las dependencias que estén dentro de los rangos de versión permitidos, ejecuta:

```bash
npm update
```

Sin embargo, ten en cuenta que `npm update` no actualiza un paquete a una nueva versión mayor si tu `package.json` no lo permite. Las versiones mayores a menudo contienen cambios incompatibles, por lo que npm no los aplicará automáticamente.

Si quieres específicamente la versión mayor más reciente, debes ejecutar:

```bash
npm install package-name@latest
```

Esto obliga a npm a instalar la versión más reciente, sin importar el rango de versiones que esté en tu `package.json`.
