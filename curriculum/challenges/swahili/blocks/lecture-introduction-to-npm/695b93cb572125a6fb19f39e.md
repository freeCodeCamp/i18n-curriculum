# --description--

Unapoweka utegemezi kwa kutumia npm, hasa katika mradi uliopo, npm huangalia faili lako la `package.json` kuona ni vifurushi gani mradi wako unahitaji.

Kila utegemezi katika `package.json` una nambari ya toleo kama hii:

```bash
"lodash": "^4.17.21"
```

Kama ulivyojifunza awali, **ishara ya caret (^)** huruhusu npm kuweka toleo jipya la kidogo na la marekebisho kama `4.18.0` au `4.17.22`. Uwezo huu ni msaada, lakini pia unaweza kusababisha matatizo.

Ikiwa wasanidi programu wawili watakimbia `npm install` kwa nyakati tofauti, npm inaweza kuweka matoleo tofauti kidogo ya kifurushi hicho hicho. Tofauti ndogo ndogo zinaweza kusababisha hitilafu au tabia zisizotarajiwa.

Faili la `package-lock.json` linatatua hili. Linaundwa moja kwa moja mara ya kwanza unapoendesha `npm install`, na linaweka vifungo vya toleo halisi la kila utegemezi uliowekwa—pamoja na utegemezi wote wa watoto, ambao ni utegemezi wa utegemezi.

Hii inahakikisha kuwa kila mtu anayefanya kazi kwenye mradi anapata matoleo sawa ya vifurushi kila wakati.

Hapa kuna faili rahisi la `package.json` lenye `lodash` kama utegemezi:

```bash
{
  "name": "package-lock-json",
  "version": "1.0.0",
  "description": "",
  "main": "index.js",
  "scripts": {
    "test": "echo \"Error: no test specified\" && exit 1"
  },
  "keywords": [],
  "author": "",
  "license": "ISC",
  "type": "commonjs",
  "dependencies": {
    "lodash": "^4.17.21"
  }
}
```

Na hapa ni sehemu inayolingana kutoka kwenye `package-lock.json`:

```bash
"node_modules/lodash": {
  "version": "4.17.21",
  "resolved": "https://registry.npmjs.org/lodash/-/lodash-4.17.21.tgz",
  "integrity": "sha512-v2kDEe57..."
}
```

Angalia tofauti:

* Katika `package.json`, `lodash` imeorodheshwa kama `^4.17.21` — kanuni yenye ufanisi.
    
* Katika `package-lock.json`, `lodash` imefungwa kwa toleo `4.17.21`.
    

Haijalishi mtu anapokimbia `npm install`, atapata toleo hilo halisi.

Kwa njia hii, haijalishi mtu anapokimbia `npm install`, atapata toleo sawa la Lodash kila wakati.

Kumbuka hili pia linahusu utegemezi wa watoto, ambao ni utegemezi wa utegemezi wowote unaotumia kwa mradi wako.

Pia angalia kwamba, ingawa `dependencies` katika `node_modules/express` hutumia ishara za caret kwa anuwai, lakini ukipata kipengele cha kila utegemezi wa mtoto kama `node_modules/etags`, zimefungwa kwa toleo maalum.

Kwa kifupi, faili la `package-lock.json` linahakikisha kila mtu yuko kwenye ukurasa mmoja na linahakikisha uthabiti kati ya mashine za wasanidi programu, seva za CI, usambazaji wa programu, na mazingira mengine.

Ni muhimu kila mara kuingiza faili lako la `package-lock.json` kwenye Git ili timu yako ishiriki usanidi sawa. Pia, usihariri faili la `package-lock.json` kwa mkono—npm italiboresha moja kwa moja kila wakati utegemezi utabadilika.
