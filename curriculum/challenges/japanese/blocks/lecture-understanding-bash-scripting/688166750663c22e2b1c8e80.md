# --description--

前のレッスンでは、Bashとよく使われるコマンドについて学びました。しかし、Bashは完全なスクリプト言語をサポートしており、より複雑な自動化タスクを実行するために使うことができます。

Bashスクリプトとは、Bashコマンドのシーケンスをファイルに書き、それをBashで実行してファイルの内容を動かすことを指します。

Bashスクリプトがどのようなものか見てみましょう。

```bash
#!/bin/bash

servers=("prod" "dev")

for server in "${servers[@]}"
do
  echo "Pulling $server"
  rsync --archive --verbose $server:/etc/nginx/conf.d/server.conf configs/$server.conf
done
```

このスクリプトは、リモートサーバーのリストからNGINXの設定ファイルを取得します。これは設定ファイルをローカルリポジトリにバックアップするのに便利です。NGINXは人気のあるウェブサーバーで、サーバーの扱いについては今後のモジュールでさらに学びます。

最初の行は「シェバン」と呼ばれ（`#!`）、スクリプトを実行するためにどのインタープリターを使うかをシステムに伝えます。ここでよく使われる値は`/usr/bin/bash`、`/bin/bash`、または`/bin/sh`です。

最初の行の`servers=("prod" "dev")`は、サーバー名を表す文字列のリストを作成しています。`"prod"`は本番サイトを指すことが多く、`"dev"`は開発サイトを指します。開発サイトは通常、新機能や変更を本番環境に展開する前にテストするために使われます。

次に、`for`ループでリストを反復します。`servers[@]`構文はリストを個々の要素に展開します。各要素は反復ごとにserverに代入されます。

`do`文はループの論理ブロックの開始を示し、反復ごとに実行されるコードです。

`echo "Pulling $server"`はserver変数の現在の値（この場合はprodまたはdev）を補間し、端末にログを出力します。

次に`rsync`を使ってサーバーから実際のファイルを取得し、補間を使ってコマンドを管理しながらローカルに保存します。

`done`はループの論理ブロックの終了を示します。

Bashスクリプトを理解し活用することには多くの大きな利点があります。

まず、ほぼすべてのUnix環境にBashが標準で用意されているため、Node.jsやPythonのような言語ランタイムを設定しなくても、そのままスクリプトを実行できます。

次に、システム上のすべてのバイナリアプリケーションに簡単にアクセスできることです。システムにdoctlがインストールされていれば、スクリプトからそのバイナリを実行できます。バイナリとはコンパイルされたプログラムで、コンピューターが実行します。Bashスクリプトがあれば、そのバイナリを実行してプログラムを起動できます。

最後に、スクリプトで使う構文は端末で直接実行可能です。つまり、スクリプトの一部をテストするのがずっと簡単になり、その部分をコピーして端末に貼り付けて結果をすぐに確認できます。

Bashを理解すれば、スクリプトの腕が上がるだけでなく、コマンドラインの達人にもなれます！
