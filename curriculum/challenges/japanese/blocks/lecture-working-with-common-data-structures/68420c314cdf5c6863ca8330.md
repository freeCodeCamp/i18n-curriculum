# --description--

あなたのデバイスで動作するすべてのコンピュータプログラムには、特定の順序で実行される一連の指示があります。

そのタスクは、数値の並べ替え、画像の編集、在庫の追跡、あるいはお気に入りのビデオゲームの実行などです。

ここでアルゴリズムが登場します。**アルゴリズム**とは、問題を解決したりタスクを実行したりするための明確な指示のセットです。

アルゴリズムは「レシピ」と考えることができます。料理をするとき、レシピには必要な材料がすべて記載されており、料理の準備手順がステップごとに示されています。

同様に、アルゴリズムはコンピューターに何をどうすべきかを正確に伝える「レシピ」と考えられます。

アルゴリズムには2つの重要な特徴があります：

* 無限に続くことはできません。有限のステップ数で終了する必要があります。
    
* 各ステップは正確で明確でなければなりません。
    

アルゴリズムは、入力がゼロ、1つ、または複数あり、出力も1つ以上生成します。

アルゴリズムのステップは、どのプログラミング言語にも依存しません。

しかし、実際にコンピューターで動かすには、PythonやJavaScriptなどのプログラミング言語で実装する必要があります。

アルゴリズムが正しければ、有効な入力に対する出力は期待される結果と一致します。

正しいだけでなく、アルゴリズムは効率的であることも重要です。

アルゴリズムの効率は、実行にかかる時間や、タスクを完了するために必要なメモリの量で測れます。

アルゴリズムの効率を知ることは非常に重要です。なぜなら、入力サイズが大きくなるにつれてどれだけ性能が維持できるかの目安になるからです。

例えば、15個の整数を並べ替えるのと、100万個の整数を並べ替えるのは同じではありません。

処理の規模や複雑さが増すにつれて、アルゴリズムがそれに対応できるほど効率的でなければ、非常に遅いプログラムになったり、システム全体がクラッシュすることもあります。

だからこそ、できるだけ効率的なアルゴリズムを開発し選ぶことが重要です。

ここでビッグO記法が非常に重要になります。

ビッグO記法は、入力サイズが増加したときのアルゴリズムの最悪の場合の性能や成長率を表します。

アルゴリズムの成長率とは、入力サイズが大きくなるにつれて必要なリソースがどのように増加するかを指します。

ビッグO記法は最悪の場合の性能に注目します。これは、入力に関係なく最悪のケースでもアルゴリズムがどれだけ効率的かを理解するために重要です。

先ほどの並べ替えの例に戻ると、100万個の整数を並べ替えるのは直感的に15個の整数を並べ替えるよりも時間とリソースがかかります。

しかし、どれだけ多くかかるかは選ぶアルゴリズムによって異なります。

ビッグO記法はアルゴリズムの効率を正確な数値で示すわけではありませんが、アルゴリズムが実行する操作の数に基づいて、入力サイズが増えたときにどのようにスケールするかの目安を示します。

ビッグO記法では、通常入力サイズを`n`で表します。例えば、入力がリストの場合、`n`はそのリストの要素数を表します。

定数項や低次の項は、操作数に基づくアルゴリズムの時間計算量を求める際に考慮されません。なぜなら、`n`のサイズが大きくなるにつれて、これらの小さな項の影響はどんどん小さくなるからです。

アルゴリズムの全体的な挙動を支配するのは、入力サイズ`n`を含む最高次の項です。

例えば、あるアルゴリズムが`7n + 20`回の操作で完了するとき、定数`20`の影響は`n`が大きくなるにつれて小さくなります。`7n`の項が支配的になり、これがアルゴリズムの全体的な挙動と効率を決定します。

別の例として、`20n² + 15n + 7`回の操作で完了するアルゴリズムがあります。`20n²`の項が`n`が大きくなるにつれて支配的になるため、このアルゴリズムは二次の時間計算量を持つことになります。なぜなら支配的な項に`n²`があるからです。

二次の時間計算量は、アルゴリズムの世界で見られる多くの時間計算量の一つです。

ここで、よくあるいくつかの時間計算量について学びましょう。

**`O(1)`**は「定数時間計算量」と呼ばれます。定数時間計算量のアルゴリズムは、入力サイズに関係なく同じ時間で実行されます。

例えば、数が偶数か奇数かを判定する処理は、数自体に関係なく常に同じ時間がかかります。

```python
def check_even_or_odd(number):
    if number % 2 == 0:
        return 'Even'
    else:
        return 'Odd'
```

**`O(log n)`**は「対数時間計算量」と呼ばれます。これは、入力サイズが増えてもアルゴリズムの実行時間がゆっくり増加することを意味します。問題のサイズが一定の割合で繰り返し減少する問題でよく見られます。

例えば、バイナリサーチという有名な探索アルゴリズムは`O(log n)`の最悪の場合の時間計算量を持ちます。これは比較ごとに残りの要素の半分を除外するため、全体として効率的です。

**`O(n)`**は「線形時間計算量」と呼ばれます。この時間計算量のアルゴリズムは、入力サイズに比例して実行時間が増加します。

例えば、リストのすべての要素を反復する`for`ループは、リストの要素数が増えると反復回数も増えます。リストのサイズが2倍になれば、操作数もほぼ2倍になります。

```python
for grade in grades:  # grades is a list.
    print(grade)
```

**`O(n log n)`**は「対数線形時間計算量」と呼ばれます。これは、マージソートやクイックソートのような効率的なソートアルゴリズムでよく見られる時間計算量です。

**`O(n²)`**は「二次時間計算量」と呼ばれます。これらのアルゴリズムの実行時間は入力サイズの二乗に比例して増加し、実際の問題には一般的に効率的ではありません。

ネストされたループは二次時間計算量の一般的な例です。内側のループは外側の`n`回の各反復に対して`n`回反復し、結果として`n`の二乗回の反復が行われます。

```python
for i in range(n):
    for j in range(n):
        print("Hello, World!")
```

その他の時間計算量には「指数時間計算量」`O(2^n)`や「階乗時間計算量」`O(n!)`があります。どちらも実際のシナリオでは非効率的です。

このグラフでは、最も一般的な時間計算量を表す数学関数の成長を比較できます。x軸（横軸）は入力サイズ、y軸（縦軸）はアルゴリズムの実行時間と考えてください。

二次時間計算量（`O(n²)`）（黄色）が他のものよりもはるかに速く増加し、定数時間計算量（`O(1)`）（赤）は入力が大きくなっても一定であることがわかります。

素晴らしいですね。ここまでで、ビッグO記法を時間の要件の観点から学びましたが、この記法は空間の要件の文脈にも適用できます。

この場合、アルゴリズムが必要とするメモリ空間が入力サイズの増加に伴ってどのように増加するかを表します。

「定数空間計算量」`O(1)`のアルゴリズムは、入力が大きくなっても常に一定のメモリ空間を必要とします。

例えば、メモリにいくつかの変数だけを作成して保存するアルゴリズムが該当します。

対照的に、「線形空間計算量」`O(n)`のアルゴリズムは、入力サイズが増えるにつれて必要な空間も比例して増加します。

例えば、長さ`n`のリストのコピーを作成して保存するアルゴリズムが該当します。

最後に、「二次空間計算量」`O(n²)`のアルゴリズムは、入力サイズが増えるにつれて必要な空間が二乗的に増加します。

例えば、入力サイズで決まる次元の2D行列を作成し、すべての可能なペアを保存する場合です。

アルゴリズムはコンピュータープログラムの基本単位であり、ビッグO記法は最悪の場合の時間と空間の要件が入力サイズの増加に伴ってどのようにスケールするかに基づいて効率を分析する強力なフレームワークです。効率を理解することは、実際のシナリオで効率的に動作するソフトウェアを開発するために非常に重要です。
