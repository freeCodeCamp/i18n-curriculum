# --description--

다음과 같은 코드를 작성했다고 가정해 보겠습니다:

```javascript
const a = "Hello";
const b = "World";

console.log(c);
```

이 코드는 실행되지만 오류가 발생합니다. 기술적으로 이것을 오류를 <dfn>발생시키는</dfn> 것이라고 합니다. 오류의 첫 부분은 오류 유형을 표시합니다. 이는 무엇을 다루고 있는지에 대한 첫 번째 단서를 제공합니다. 수업 후반부에서 다양한 오류 유형에 대해 더 배우게 됩니다. 이 예제에서는 `ReferenceError`가 있습니다.

<img src="https://cdn.statically.io/gh/TheOdinProject/curriculum/175b5ef2a1b4758a7b75f4ef43d7e27203e5707b/foundations/javascript_basics/understanding_errors/imgs/00.png" width="100%" alt="개발자 콘솔에 표시된 오류"/>

`ReferenceError`는 현재 범위 내에서 선언되거나 초기화되지 않은 변수를 참조할 때 발생합니다. 이 경우 오류 메시지는 `c is not defined` 때문에 오류가 발생했다고 설명합니다.

이 유형의 다른 오류들은 `ReferenceError`를 유발하는 원인에 따라 다른 메시지를 가집니다. 예를 들어, 마주칠 수 있는 또 다른 메시지는 `ReferenceError: can't access lexical declaration 'X' before initialization`입니다.

오류의 다음 부분은 오류가 발생한 파일 이름(이 경우 `script.js`)과 줄 번호를 알려줍니다.

이를 통해 코드에서 문제의 줄로 쉽게 이동할 수 있습니다. 여기서는 오류가 `script.js`의 네 번째 줄에서 발생했으며, 오류 메시지 아래에 `script.js:4`라는 텍스트로 링크가 표시됩니다. 이 링크를 클릭하면 대부분의 브라우저는 개발자 도구의 Sources 탭에서 정확한 코드 줄과 나머지 스크립트로 이동합니다.

때때로 브라우저 콘솔은 오류가 발생하는 줄의 열(또는 문자) 위치도 표시합니다. 이 예제에서는 `script.js:4:13`에 해당합니다.
