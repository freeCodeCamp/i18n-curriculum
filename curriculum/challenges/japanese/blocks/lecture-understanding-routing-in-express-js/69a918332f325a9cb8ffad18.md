# --description--

Expressでは、**ルートハンドラ**（ルートコールバック関数やコントローラー関数とも呼ばれます）は、クライアントが特定のルートにリクエストを送信したときに何が起こるかを定義する関数です。

ルートハンドラは、受信したリクエストを処理し、データベースのクエリや計算の実行など必要なロジックを行い、その後クライアントにレスポンスを返します。

これを簡単な例で説明しましょう（実はずっと使ってきた例です）：

```javascript
app.get("/menu", (req, res) => {
  res.send("Welcome to the menu!");
});
```

`.get()` メソッドはExpressのルートメソッドの一つです。指定されたルートでの**GET**リクエストを待ち受けます。この例では、`/menu` へのリクエストを待ち受けています。

関数 `(req, res) => { ... }` が**ルートハンドラ**です。クライアントが`/menu` にGETリクエストを送るたびに実行されます。

ハンドラの中では：

* `req`（リクエスト）は、ルートパラメータ、クエリ文字列、ヘッダー、ボディデータなど、受信したリクエストに関する情報を含むオブジェクトです。
    
* `res`（レスポンス）は、クライアントにレスポンスを送るために使うオブジェクトです。
    

この場合、`res.send("Welcome to the menu!")` はクライアントにプレーンテキストのレスポンスを送ります。

ユーザーがブラウザで`/menu` にアクセスするか、そのルートにGETリクエストを送ると、サーバーはこのハンドラを実行し、メッセージ`Welcome to the menu!`で応答します。

ルートハンドラはExpressアプリケーションの中核です。ウェブアプリケーションの異なるルートがリクエストにどう応答するかを定義できます。また、フォームデータの処理、データベースとのやり取り、バリデーションの実行などのロジックをこれらのハンドラに追加できます。
