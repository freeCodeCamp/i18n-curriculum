# --description--

HTML è un linguaggio molto tollerante: gli elementi vengono comunque renderizzati anche quando commetti errori, come dimenticare di includere un tag di chiusura.

Supponiamo che tu abbia un elemento `h2` senza un tag di chiusura:

```html
<h1>Article Topic</h1>
<h2>Subheading 1 </h2>
<h2>Subheading 2 </h2>

<!-- This h2 does not have a closing tag -->
<h2>Subheading 3
```

L'elemento `h2` senza tag di chiusura verrà comunque renderizzato correttamente. Questo accade perché i browser usano un algoritmo di parsing che gestisce gli errori comuni e cerca di renderizzare l'HTML il più possibile vicino all'intenzione dell'autore.

Ma questo a volte può ritorcersi contro. Aggiungiamo qualche paragrafo sotto i tag di intestazione 2 esistenti nel codice:

```html
<h1>Article Topic</h1>
<p>Lorem ipsum dolor sit amet consectetur adipisicing elit. Maiores, nisi.</p>

<h2>Subheading 1 </h2>
<p>Lorem, ipsum dolor sit amet consectetur adipisicing elit. At, doloremque.</p>

<h2>Subheading 2 </h2>
<p>Lorem ipsum dolor sit amet consectetur adipisicing elit. Unde, placeat.</p>

<!-- This h2 does not have a closing tag -->
<h2>Subheading 3
<p>Lorem ipsum dolor, sit amet consectetur adipisicing elit. Tempore, illum.</p>
```

Di conseguenza, l'elemento paragrafo sotto il `h2` senza tag di chiusura `h2` viene renderizzato come intestazione 2. Ecco perché ti serve un validatore HTML.

Un validatore HTML è uno strumento che verifica la validità del tuo codice HTML rispetto alle specifiche standard HTML. Ti aiuta a identificare errori e avvisi nel tuo codice HTML, assicurando che le tue pagine web siano strutturate correttamente e conformi agli standard web.

Usare un validatore HTML è utile non solo per te e per le revisioni future del codice, ma anche per chiunque altro esamini il tuo codice, come i tuoi compagni di squadra e i contributori open source.

Ci sono diversi validatori HTML che puoi usare. Il più accettato è il servizio di validazione markup `w3.org`.

Quando visiti il sito [`validator.w3.org`](https://validator.w3.org/), puoi cliccare sul pulsante `Validate by Direct Input` e incollare il tuo codice HTML.

Quando clicchi sul pulsante `Check`, verrà mostrata una lista di risultati con gli errori da correggere.

Un altro validatore HTML che puoi usare è [`jsonformatter.org`](https://jsonformatter.org/).

Puoi copiare e incollare il tuo codice HTML nel primo editor, e quando clicchi sul pulsante `Validate`, ti mostrerà eventuali errori presenti nel codice.
