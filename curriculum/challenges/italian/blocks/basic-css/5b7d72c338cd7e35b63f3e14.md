# --description--

Quando lavori con il CSS probabilmente ti imbatterai in problemi di compatibilità tra browser a un certo punto. Per questo è importante fornire fallback per i browser per evitare potenziali problemi.

Quando il tuo browser analizza il CSS di una pagina web, ignora tutte le proprietà che non riconosce o non supporta. Per esempio, se usi una variabile CSS per assegnare un colore di sfondo su un sito, Internet Explorer ignorerà il colore di sfondo perché non supporta le variabili CSS. In quel caso, il browser userà qualunque valore abbia per quella proprietà. Se non trova nessun altro valore impostato per quella proprietà, tornerà al valore predefinito, che di solito non è l’ideale.

Questo significa che se vuoi fornire un fallback per il browser, è sufficiente aggiungere un altro valore più ampiamente supportato subito prima della tua dichiarazione. In questo modo un browser più vecchio avrà qualcosa su cui fare fallback, mentre un browser più recente interpreterà semplicemente qualunque dichiarazione venga dopo nella cascata.

# --instructions--

Sembra che venga usata una variabile per impostare il colore di sfondo della classe `.red-box`. Miglioriamo la compatibilità con i browser aggiungendo un’altra dichiarazione `background` subito prima di quella esistente e impostandone il valore su `red`.

# --hints--

La tua regola `.red-box` dovrebbe includere un fallback con la proprietà `background` impostata su `red` subito prima della dichiarazione `background` esistente.

```js
const spacelessCode = __helpers.removeWhiteSpace(__helpers.removeCssComments(code));
assert.match(
  spacelessCode,
  /\.red-box{background:(red|#ff0000|#f00|rgb\(255,0,0\)|rgb\(100%,0%,0%\)|hsl\(0,100%,50%\));background:var\(--red-color\);height:200px;width:200px;}/gi
);
```
