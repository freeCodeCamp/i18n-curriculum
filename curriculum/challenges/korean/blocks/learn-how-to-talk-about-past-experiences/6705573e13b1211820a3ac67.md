# --explanation--

동사 `to be`의 부정형은 `Simple Past` 시제에서 단수 주어일 때 `was not` 또는 `wasn't`, 복수 주어일 때 `were not` 또는 `weren't`입니다. 이는 어떤 일이 일어나지 않았거나 사실이 아니었음을 설명할 때 사용합니다. 예를 들어:

- 단수: `The code wasn't running as expected.` - 코드에 문제가 있었습니다.

- 복수: `The changes weren't implemented correctly.` - 구현상의 오류들이 버그를 일으켰습니다.

James의 문장에서는 `wasn't`를 사용해 시스템이 데이터베이스에 연결하지 못하고 있음을 설명합니다. `wasn't` 또는 `weren't`를 포함한 부정문은 기대한 대로 일어나지 않았거나 작동하지 않은 것을 나타냅니다.
