# --interactive--

HTML 表格如今的使用頻率不如以往。但作為前端開發者，你仍然應該熟悉它們。表格是 1990 年代開發者在瀏覽器中顯示資料的最早方式之一。

以下是一個用來從美國勞工統計局產生表格的程式碼範例：

:::interactive_editor

```html
<table id="quickfacts">
  <thead>
    <tr>
      <th colspan="2">Quick Facts: Software Developers, Quality Assurance Analysts, and Testers</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <th>2023 Median Pay</th>
      <td>
        $130,160 per year
        <br>$62.58 per hour
      </td>
    </tr>
    <tr>
      <th>Typical Entry-Level Education</th>
      <td>Bachelor's degree</td>
    </tr>
    <tr>
      <th>Work Experience in a Related Occupation</th>
      <td>None</td>
    </tr>
    <tr>
      <th>On-the-job Training</th>
      <td>None</td>
    </tr>
    <tr>
      <th>Number of Jobs, 2022</th>
      <td>1,795,300</td>
    </tr>
    <tr>
      <th>Job Outlook, 2022-32</th>
      <td>25% (Much faster than average)</td>
    </tr>
    <tr>
      <th>Employment Change, 2022-32</th>
      <td>451,200</td>
    </tr>
  </tbody>
  <tfoot>
    <tr>
      <th>If this table had a footer it would be here.</th>
    </tr>
  </tfoot>
</table>
```

:::

如你所見，有一個主要的 `table` 元素，其 `id` 設定為 `quickfacts`。在其中，該表格包含一個表頭元素 `thead`、表身元素 `tbody` 和表尾元素 `tfoot`。

表格的表頭、表身和表尾元素各自可以包含若干個表格列，`tr`。而每個表格列可以包含一個表格標頭 `th`，用來標示該列中的資料。它也可以包含若干個稱為表格資料的個別資料儲存格，`td`。

現在，這是大量的 HTML 元素。但別被嚇到－這些遵循簡單的階層體系。

這是我們可以創建的最簡單的表格，包含所有這些元素：

:::interactive_editor

```html
<table>
  <thead>
    <tr>
      <th>The title of this table</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <th>First Row</th>
      <td>
        First Data Cell
      </td>
    </tr>
    <tr>
      <th>Second Row</th>
      <td>
        Second Data Cell
      </td>
    </tr>
  </tbody>
  <tfoot>
    <tr>
      <th>The footer of this table, which might contain date of publication, author credits, or other meta information.</th>
    </tr>
  </tfoot>
</table>
```

:::

如你所見，資料本身總是在一個 `tr` 中－而在該 `tr` 元素內有一個帶有標題的 `th` 元素，以及一個帶有資料的 `td` 元素。

有些網站會選擇使用 `div` 來建置自己的表格，而不是使用較為適合的 `table` 元素。

雖然可以使用泛型的 `div` 元素來顯示表格式資料，但仍然最好使用 `table` 元素。

多年前，開發者可能會使用 `table` 來定位網頁上的非資料元素。這從未被視為最佳實務。但你可能會遇到一些程式碼庫仍然以這種方式使用表格。

現今，開發者使用 CSS flexbox 和 grid 來佈局他們的設計。freeCodeCamp 將在稍後深入涵蓋這些工具。

目前，請僅將 HTML 表格用於其原本預期的用途：顯示表格式資料。
