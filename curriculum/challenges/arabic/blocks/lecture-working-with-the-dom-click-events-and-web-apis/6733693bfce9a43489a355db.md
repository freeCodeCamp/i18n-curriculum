# --interactive--

يُطلق حدث `DOMContentLoaded` عندما تُحمّل كل محتويات مستند HTML وتُحلل. إذا كان لديك أوراق أنماط خارجية أو صور، فلن ينتظر حدث `DOMContentLoaded` تحميلها. سينتظر فقط تحميل HTML.

هذا يختلف عن حدث `load`، الذي ينتظر تحميل كل شيء، بما في ذلك أوراق الأنماط الخارجية والصور وما إلى ذلك.

إليك صيغة المثال لاستخدام حدث `DOMContentLoaded`:

```js
document.addEventListener("DOMContentLoaded", function () {
  console.log("DOM is loaded.");
});
```

بمجرد تحميل DOM، تُنفذ الدالة ويُسجل في وحدة التحكم الرسالة `DOM is loaded.`.

الآن، لنلق نظرة على مثال آخر يستخدم حدث `DOMContentLoaded`. في هذا المثال، لدينا صورة داخل HTML. لتحديث الصورة، يمكننا إنشاء دالة تغير خاصيتي `src` و`alt` للصورة:

:::interactive_editor

```html
<h1>Image Change on DOM Loaded</h1>
<img
  id="example-img"
  src="https://cdn.freecodecamp.org/curriculum/cat-photo-app/relaxing-cat.jpg"
  alt="Cat relaxing"
/>

<script src="index.js"></script>
```

```js
function changeImg() {
  const img = document.getElementById("example-img");
  img.src =
    "https://cdn.freecodecamp.org/curriculum/responsive-web-design-principles/FCCStickers-CamperBot200x200.jpg";
  img.alt = "CamperBot sticker";
  console.log("image was just changed");
}

changeImg();
```

:::

يمكننا بعد ذلك التحقق مما إذا كان DOM لا يزال يحمل وإضافة مستمع حدث لحدث `DOMContentLoaded`. إذا كان حدث `DOMContentLoaded` قد أُطلق بالفعل، يمكننا استدعاء دالة `changeImg` مباشرة:

```js
if (document.readyState === "loading") {
  document.addEventListener("DOMContentLoaded", changeImg);
} else {
  console.log("DOMContentLoaded has already fired");
  changeImg();
}
```

إذا شغلت هذا الشفرة وأعدت تحميل الصفحة، سترى الصورة تتغير إلى الصورة الجديدة بعد تحميل DOM. يحدث ذلك بسرعة، لكن سترى وميضًا بسيطًا أثناء تغير الصورة.

يُفيد حدث `DOMContentLoaded` عندما تريد تشغيل بعض شفرة JavaScript بمجرد تحميل DOM، لكن قبل تحميل كل الموارد الخارجية مثل الصور وأوراق الأنماط.
