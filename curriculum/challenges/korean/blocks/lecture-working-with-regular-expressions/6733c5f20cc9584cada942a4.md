# --interactive--

캡처 그룹은 일치한 문자열의 일부를 "캡처"하여 필요에 따라 사용할 수 있게 합니다. 캡처 그룹은 전방 탐색 같은 선행 문자가 없는 괄호 안에 캡처할 패턴을 정의하여 만듭니다.

우리 `code`에서 `freeCodeCamp` 정규 표현식의 `code`를 캡처해 보겠습니다. 이를 위해 `code`를 괄호로 감싸 캡처 그룹으로 정의합니다:

```js
const regex = /free(code)camp/i;
```

동작을 확인하려면 `freecodecamp` 문자열에 대해 테스트할 수 있습니다:

:::interactive_editor

```js
const regex = /free(code)camp/i;
console.log(regex.test("freecodecamp")); // true
```

:::

하지만 이것은 실제로 캡처한 그룹을 사용하지 않습니다. 대신 `match`를 사용한 결과를 살펴보겠습니다:

```js
const regex = /free(code)camp/i;
console.log("freecodecamp".match(regex));
// [
//   'freecodecamp',
//   'code', <--
//   index: 0,
//   input: 'freecodecamp',
//   groups: undefined
// ]
```

여기서 `match` 배열에 두 번째 요소가 있는데, 이것이 캡처 그룹이 캡처한 문자열 부분입니다.

캡처 그룹이 `code`와 정확히 일치하는 패턴을 찾는 것을 볼 수 있습니다. 여기서 문자 클래스는 `c`, `o`, `d`, `e` 목록 중 하나의 단일 문자를 일치시킵니다.

그렇다면 실제로 이것을 어떻게 사용할 수 있을까요? 캡처 그룹은 문자열 내용을 교체할 때 자주 사용됩니다. 이를 위해 코드를 작성해 보겠습니다. `freecodecamp`을 `paidcodeworld`로 바꿀 것입니다:

:::interactive_editor

```js
const regex = /free(code)camp/i;
console.log("freecodecamp".replace(regex, "paidcodeworld"));
```

:::

이 코드는 자체로 작동하지만, `code`에 `o`가 몇 개 있는지 모른다면 어떻게 할까요? 하나 이상의 `o`에 대한 수량자가 필요하다면:

:::interactive_editor

```js
const regex = /free(co+de)camp/i;
console.log("freecoooooooodecamp".replace(regex, "paidcodeworld"));
```

:::

결과로 `paidcodeworld`를 얻고 있습니다. `o`의 개수를 유지하려면 정규 표현식이 캡처한 내용을 재사용해야 합니다.

여기서 백참조가 등장합니다. 교체 문자열의 `code` 부분을 하드코딩하는 대신 캡처 그룹을 직접 참조할 수 있습니다.

`replace` 메서드에서 백참조는 달러 기호(`$`) 다음에 사용할 캡처 그룹 번호를 붙여서 만듭니다. 여기서는 `code`가 첫 번째 캡처 그룹에 캡처되었으므로 `$1`가 됩니다:

:::interactive_editor

```js
const regex = /free(co+de)camp/i;
console.log("freecoooooooodecamp".replace(regex, "paid$1world")); // paidcoooooooodeworld
```

:::

이제 `freecodecamp`를 `paidcodeworld`로 변환할 때 알 수 없는 수의 `o` 문자를 성공적으로 유지했습니다. 하지만 백참조는 replace 메서드에만 국한되지 않습니다. 실제로 정규 표현식 내에서 직접 사용할 수도 있습니다.

이렇게 하면 정규 표현식에서 이전에 캡처한 패턴을 나중에 다시 일치시킬 수 있습니다.

예를 들어, `freecodecamp`를 두 번, 같은 수의 `o`와 함께 문자열 어디에서든 일치시키고 싶다고 합시다.

먼저 와일드카드 문자로 구분하고, 그 와일드카드에 어떤 문자든 여러 개가 일치하도록 허용해야 합니다:

```js
const regex = /free(co+de)camp.*free(co+de)camp/i;
```

하지만 이 표현식은 `o` 문자의 개수가 같은지 보장하지 않습니다. 이를 위해 두 번째 캡처 그룹을 첫 번째 캡처 그룹을 참조하는 것으로 바꿔야 합니다.

정규 표현식 내에서 백참조는 역슬래시 다음에 캡처 그룹 번호를 붙여서 나타냅니다:

:::interactive_editor

```js
const regex = /free(co+de)camp.*free\1camp/i;
console.log(regex.test("freecooooodecamp is great i love freecooooodecamp")); // true
console.log(regex.test("freecooooodecamp is great i love freecodecamp")); // false
```

:::

이제 `o` 개수가 맞는 문자열은 일치하지만, `o` 개수가 다른 두 문자열은 일치하지 않는 것을 볼 수 있습니다.

이 구문은 훌륭하지만 여러 캡처 그룹을 참조할 때는 금방 헷갈릴 수 있습니다. 다행히도 숫자 대신 그룹에 이름을 붙일 수 있습니다.

이름 있는 캡처 그룹을 정의하려면 그룹 시작 부분에 물음표(`?`)를 추가하고, 그 뒤에 이름을 꺾쇠 괄호 안에 넣습니다. 우리 캡처 그룹 이름을 `code`로 정해 보겠습니다:

```js
const regex = /free(?<code>co+de)camp.*free\1camp/i;
```

이제 정규 표현식 내 백참조를 이 그룹을 참조하도록 업데이트할 수 있습니다. 이름 있는 백참조는 JavaScript에서 역슬래시 다음에 문자 `k`를 붙이고, 그 뒤에 이름을 다시 꺾쇠 괄호(`<`, `>`) 안에 넣습니다. 살펴보겠습니다:

```js
const regex = /free(?<code>co+de)camp.*free\k<code>camp/i;
```

이제 `test()` 메서드를 확인하면 여전히 통과하는 것을 볼 수 있습니다:

:::interactive_editor

```js
const regex = /free(?<code>co+de)camp.*free\k<code>camp/i;
console.log(regex.test("freecooooodecamp is freecooooodecamp")); // true
```

:::

이름 있는 캡처 그룹을 `replace()` 메서드에서 사용하려면 문자열에 달러 기호를 넣고, 그 뒤에 이름을 꺾쇠 괄호 안에 넣으면 됩니다:

:::interactive_editor

```js
const regex = /free(?<code>co+de)camp/i;
console.log("freecooooodecamp".replace(regex, "paid$<code>camp")); // paidcooooodecamp
```

:::

마지막으로, 때로는 문자 그룹을 만들고 싶지만 캡처 결과는 필요하지 않을 때가 있습니다.

예를 들어 `freecodecamp` 또는 `freecandycamp` 중 하나와 일치시키고 싶다면 OR 어서션으로 구분된 두 패턴을 만들 수 있습니다:

```js
const regex = /freecodecamp|freecandycamp/i;
```

하지만 대규모 정규 표현식에서는 이것이 꽤 길어질 수 있습니다. 대신 OR가 필요한 문자 주위에 비캡처 그룹을 만들 수 있습니다:

```js
const regex = /free(?:code|candy)camp/i;
```

비캡처 그룹은 `code|candy` 일치를 별도로 메모리에 저장하지 않습니다. 하지만 가독성이나 성능을 희생하지 않고 대체 패턴을 만들 때 유용합니다.
