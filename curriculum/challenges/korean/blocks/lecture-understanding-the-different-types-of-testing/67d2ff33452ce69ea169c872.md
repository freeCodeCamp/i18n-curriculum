# --description--

이전 수업에서 단위 테스트를 통해 작은 코드 단위를 테스트하는 방법을 배웠습니다. 단위 테스트는 기능 테스트의 일부입니다. 기능 테스트는 애플리케이션의 기능과 함수가 예상대로 작동하는지 확인합니다.

이번 수업에서는 기능 테스트와 비기능 테스트의 차이점을 살펴보겠습니다.

기능 테스트의 목표는 여러 시나리오에 대해 시스템 전체를 테스트하여 시스템이 예상대로 작동하는지 확인하는 것입니다. 기능 테스트의 예로는 스모크 테스트가 있습니다. 스모크 테스트는 더 광범위한 테스트를 수행하기 전에 시스템의 기본적이거나 중요한 문제를 예비 점검하는 것입니다.

반면, 비기능 테스트는 성능과 신뢰성 같은 부분에 집중합니다.

기능 테스트의 실제 테스트 시나리오는 사용자 로그인입니다. 사용자는 사용자 이름과 비밀번호로 애플리케이션에 로그인할 수 있어야 합니다. 성공하면 계정 정보를 접근할 수 있어야 합니다. 문제가 발생하면 그 오류를 적절히 처리해야 합니다. 테스트를 작성할 때는 모든 성공과 실패 시나리오를 고려해야 합니다.

비기능 테스트의 예는 다양한 네트워크 조건에서 애플리케이션의 성능을 테스트하는 것입니다. 앞으로의 수업에서 성능 테스트에 대해 더 배우게 됩니다.

기능 테스트가 중요한 이유는 애플리케이션에서 버그를 조기에 발견하고, 기획 단계 초기에 설정한 요구사항을 충족하는지 확인하며, 사용자가 좋은 사용자 경험을 하도록 보장하기 위해서입니다. 사용자가 애플리케이션을 사용하는 데 어려움을 겪거나 기능 고장과 시스템 실패로 막히면 안 됩니다.
