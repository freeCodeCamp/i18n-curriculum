# --interactive--

Opereta za kimantiki za binary hukusaidia kutathmini usemi wa kihesabu wawili na kurudisha matokeo kulingana na uhalisia wao. Tuchunguze opereta tatu maarufu za kimantiki za binary: logical AND, logical OR, na opereta ya nullish coalescing.

Opereta ya logical AND inawakilishwa na alama mbili za ampersand (`&&`). Inakagua kama pande zote mbili ni kweli na kurudisha matokeo. Ikiwa pande zote mbili ni kweli, inarudisha thamani ya pili, yaani ile upande wa kulia:

:::interactive_editor

```js
const result = true && 'hello';

console.log(result); // hello
```

:::

Katika mfano hapo juu, maandishi `hello` yameandikwa kwenye konsoli kwa sababu pande zote mbili ni `true`. Ikiwa upande wowote ni thamani ya uongo, inarudisha thamani ya uongo:

:::interactive_editor

```js
const result = 0 && 3;

console.log(result); // 0
```

:::

Kwa kuwa `0` ni thamani ya uongo, nambari `0` imeandikwa kwenye konsoli. Na ikiwa pande zote mbili ni thamani za uongo, inarudisha thamani ya uongo ya kwanza:

:::interactive_editor

```js
const result = false && 0;

console.log(result); // false
```

:::

Kwa kuwa `false` ni thamani ya uongo, basi `false` imeandikwa kwenye konsoli. Opereta ya logical AND ni muhimu unapotaka kukagua masharti mengi na kuhakikisha kuwa yote ni kweli kabla ya kuendelea. Hapa kuna mfano:

:::interactive_editor

```js
if (2 < 3 && 3 < 4) {
 console.log('The if block runs'); 
} else {
 console.log('The else block runs');
} 
```

:::

Katika sharti, kwa kuwa `2` ni ndogo kuliko `3` NA `3` ni ndogo kuliko `4`, basi sentensi `The if block runs` itaandikwa kwenye konsoli.

Opereta ya logical OR inakagua kama angalau upande mmoja ni kweli. Ikiwa upande wa kwanza ni kweli, inarudisha thamani hiyo:

:::interactive_editor

```js
const result = 'This is truthy' || false;

console.log(result); // This is truthy
```

:::

Ikiwa upande wa kwanza ni thamani ya uongo lakini upande wa pili ni kweli, thamani ya pili itaandikwa kwenye konsoli:

:::interactive_editor

```js
const result = 0 || 'This is truthy';

console.log(result); // This is truthy
```

:::

Ni kawaida kutumia opereta ya logical OR katika taarifa za `if/else` kama hizi:

:::interactive_editor

```js
let userInput;

if (userInput || 'Guest') {
 console.log('A user is present');
} else {
 console.log('No user detected');
}
```

:::

Kwa kuwa hatukuweka thamani kwa mabadiliko ya `userInput`, kwa sasa ni `undefined`. Sharti katika taarifa ya `if` linakagua kama mabadiliko ya `userInput` au mfuatano wa herufi `Guest` ni kweli. Kwa kuwa mfuatano wa herufi `Guest` ni kweli katika muktadha wa thamani ya kweli kama huu, mfuatano wa herufi `A user is present` utaandikwa kwenye konsoli.

Opereta ya nullish coalescing ni ya hali ya juu zaidi kuliko logical OR na logical AND. Inawakilishwa na alama mbili za alama ya swali (`??`), husaidia katika hali ambapo unataka kurudisha thamani tu ikiwa ya kwanza ni `null` au `undefined`. Hapa kuna mfano wa kufanya kazi na opereta ya nullish coalescing:

:::interactive_editor

```ts
const result = null ?? 'default';

console.log(result); // default
```

:::

Kwa kuwa `null` ni thamani ya nullish, mfuatano wa herufi `default` utaandikwa kwenye konsoli. Opereta ya nullish coalescing ni ya maana sana katika hali ambapo `null` au `undefined` ndizo thamani pekee zinazopaswa kusababisha njia mbadala au thamani ya msingi. Hapa kuna mfano wa kushughulikia mipangilio ya upendeleo wa mtumizi:


:::interactive_editor

```ts
const userSettings = {
 theme: null,
 volume: 0,
 notifications: false,
};

let theme = userSettings.theme ?? 'light';
console.log(theme); // light
```

:::

Katika mfano hapo juu, tuna kitu kinachoitwa `userSettings` ambacho kina vigezo `theme`, `volume` na `notifications`. Tunapata upatikanaji wa `theme` kwa kutumia uandishi wa nukta kama `userSettings.theme`. Utajifunza zaidi kuhusu jinsi ya kufanya kazi na vitu katika funzo lijalo. Kwa kuwa upendeleo wa mtumizi wa `theme` kwa sasa umewekwa kuwa `null`, basi mfuatano wa herufi `light` utaandikwa kwenye konsoli.
