# --description--

No passado, a maioria das aplicações, sejam elas de grande ou pequeno porte, eram construídas como monólitos. Um monólito é uma única base de código gigante que contém tudo na `web app`.

Este sistema monolítico significa que a UI, o banco de dados e a API vivem todos em uma única base de código.

Imagine uma loja de eCommerce como a Amazon, onde as páginas de produto, sistema de checkout, processamento de pagamento, autenticação e autorização, junto com a API e os bancos de dados que os alimentam, residem todos na mesma base de código.

A abordagem monolítica funciona bem para aplicações pequenas e startups em estágio inicial. No entanto, à medida que uma aplicação cresce e a demanda dos usuários aumenta, os monólitos frequentemente se tornam mais difíceis de manter, mais lentos para atualizar e difíceis de escalar. Uma pequena mudança em uma área pode exigir o reimplantamento do sistema inteiro, e escalar um recurso frequentemente significa escalar a aplicação inteira.

Esses desafios levaram ao surgimento dos microservices, que é uma abordagem arquitetural projetada para resolver as limitações dos sistemas monolíticos.

Em uma arquitetura de microserviços, uma grande aplicação é dividida em serviços menores e independentes. Cada serviço é responsável por uma capacidade de negócio específica, como produtos, pedidos, autenticação, pagamentos e assim por diante. Esses serviços frequentemente têm suas próprias bases de código e podem até gerenciar seus próprios bancos de dados, e podem ter equipes totalmente diferentes trabalhando neles.

Os serviços se comunicam entre si por meio de APIs ou sistemas baseados em mensagens, permitindo que trabalhem juntos como uma única aplicação.

Com essa abordagem, tanto aplicações web quanto móveis são mais fáceis de manter, atualizar e escalar. Por exemplo, uma plataforma de e-commerce pode ter serviços separados para produtos, carrinhos, usuários, pedidos e pagamentos. Cada serviço pode ser construído, implantado e escalado independentemente sem afetar os outros.

Microserviços oferecem várias vantagens, incluindo desenvolvimento mais rápido, melhor isolamento de falhas e maior flexibilidade. Serviços diferentes podem usar linguagens de programação ou frameworks diferentes—por exemplo, um serviço de busca pode usar Java, um serviço de produto pode usar Node.js e um serviço de autenticação pode usar .NET.

No entanto, microserviços também introduzem novos desafios. A depuração se torna mais complexa porque issues podem abranger múltiplos serviços. A implantação e o gerenciamento da infraestrutura são mais complicados e a sobrecarga de rede aumenta devido à comunicação frequente entre os serviços.
