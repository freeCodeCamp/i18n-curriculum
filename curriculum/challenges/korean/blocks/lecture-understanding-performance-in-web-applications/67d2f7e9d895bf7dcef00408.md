# --description--

모든 프로젝트는 고유하므로 이 기법들 중 어떤 것이 웹 애플리케이션에 가장 도움이 될지 신중하게 고려하세요.

먼저, 모든 스타일이 파싱되기 때문에 CSS 스타일시트에서 불필요한 스타일을 제거해야 합니다.

또한 CSS 스타일을 여러 파일로 분할하세요. 이렇게 하면 특정 웹 페이지를 렌더링하는 데 필요하지 않은 파일은 로드되지 않아 렌더링 과정을 차단하지 않습니다.

기본적으로 브라우저는 모든 스타일시트가 렌더링 과정을 차단한다고 가정합니다. 하지만 `media` 속성과 미디어 쿼리를 사용해 스타일시트를 언제 로드할지 브라우저에 알려줄 수 있습니다.

이 예시에서 `print.css` 스타일시트는 `media` 속성에 `print` 값이 있기 때문에 문서가 출력될 때만 적용됩니다:

```html
<link rel="stylesheet" href="print.css" media="print" />
```

빌드 과정에서 CSS 파일을 자동으로 축소하여 크기를 줄이고, 웹 애플리케이션을 호스트하는 서버에서 이 파일들을 압축하는 것도 가능합니다.

CSS 선택자는 가능한 한 단순하게 유지하고, 필요한 것보다 더 많은 요소를 선택하려 하지 마세요.

가능한 한 CSS의 계단식 특성을 활용하세요. 일부 스타일은 상속됩니다.

`rel="preload"`를 사용해 리소스를 미리 로드하는 것도 중요한 기법입니다. 중요한 자산은 먼저 로드해야 합니다.

CSS 애니메이션이 성능에 영향을 줄 수 있다는 점도 기억하세요.

크기, 위치, 레이아웃 같은 특정 CSS 속성을 애니메이션하면 "리플로우"라는 과정이 발생하는데, 이 과정에서 브라우저가 페이지 내 일부 요소의 위치와 기하학을 다시 계산합니다. 이때 다시 그리기가 필요하며, 이는 계산 비용이 많이 듭니다.

따라서 CSS 애니메이션 수를 가능한 한 줄이거나 적어도 사용자가 애니메이션을 켜고 끌 수 있는 옵션을 제공하는 것이 좋습니다.

폰트는 웹 애플리케이션의 시각적 표현을 향상시키지만 성능에 영향을 줄 수 있습니다. 일부 폰트는 크기가 매우 커서 수 메가바이트에 달하기도 합니다. 최대 두세 개의 폰트만 사용하고 가능하면 웹 안전 폰트를 사용하세요.

가장 중요한 폰트를 미리 로드해 빠르게 사용할 수 있도록 하는 것도 고려해 보세요.

이것들이 웹 개발에서 CSS 최적화를 위해 가장 중요하고 흔히 사용되는 기법들입니다. 이를 적용하면 웹사이트 성능을 최적화해 부드러운 사용자 경험을 만들 수 있습니다.
