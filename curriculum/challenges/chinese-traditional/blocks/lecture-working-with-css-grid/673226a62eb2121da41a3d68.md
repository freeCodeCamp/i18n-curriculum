# --interactive--

`minmax()` 函式定義網格軌道大小的範圍，指定列或行可以佔用多少空間。

請記住，你可以使用像 `px`（像素）、`rem`，甚至 `em` 這樣的單位來設定軌道大小，還可以使用分數單位（`fr`）。

`minmax()` 函式（程式）更進一步，允許你為網格軌道設定最小尺寸和最大尺寸。

以下是 `minmax()` 函式的語法：

```css
minmax(min, max)
```

`min` 是網格軌道的最小尺寸，可以使用像素、百分比或 `auto` 來設定。而 `max` 是網格軌道的最大尺寸，你可以使用相同的單位來設定。

這兩個值以這種方式協同運作：

- `min` 值確保網格軌道不會縮小到低於設定大小。

- `max` 值限制網格軌道可以增長的最大尺寸。

網格軌道大小會根據內容和容器大小，在 `min` 和 `max` 值之間動態調整。

讓我們來看一個實際範例：

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<div class="grid-container">
  <div>
    <h2>Item 1</h2>
  </div>
  <div>
    <h2>Item 2</h2>
  </div>
</div>
```

```css
.grid-container {
  display: grid;
  grid-template-columns: minmax(150px, 300px) 1fr;
  gap: 20px;
}

.grid-container > div {
  background: crimson;
  padding: 20px;
  text-align: center;
}
```

:::

這裡發生了什麼？ 

第一行 `minmax(150px, 300px)` 將始終至少為 `150px`，且最多為 `300px`，視可用空間而定。

另一方面，第二行 `1fr` 將會佔用網格容器中任何可用的剩餘空間，因為沒有其他行可以共享該空間。

`minmax()` 函式相較於固定尺寸甚至 `fr` 單位的優點在於它更具彈性，非常適合用於適應性和響應式設計。
