## Основи CSS

- **Що таке CSS?**: Каскадні таблиці стилів (CSS) — це мова стилів, яка використовується для застосування стилів до HTML-елементів. CSS використовується для кольорів, фонових зображень, макетів і багато чого іншого.
- **Основна структура правила CSS**: Правило CSS складається з двох основних частин: селектора та блоку оголошень. Селектор — це шаблон, який використовується в CSS для ідентифікації та націлювання конкретних HTML-елементів для стилізації. Блок оголошень застосовує набір стилів для заданого селектора або селекторів.
- **Елемент `meta name="viewport"`**: Цей `meta` елемент дає браузеру інструкції щодо керування розмірами сторінки та масштабуванням на різних пристроях, особливо на мобільних телефонах і планшетах.
- **Стилі браузера за замовчуванням**: Кожен HTML-елемент матиме застосовані стилі браузера за замовчуванням. Зазвичай це включає такі речі, як стандартні поля та відступи.

## Рядковий, внутрішній та зовнішній CSS

- **Рядковий CSS**: ці стилі записані безпосередньо всередині HTML-елемента за допомогою атрибута `style`. Зазвичай ви не будете використовувати рядковий CSS через принцип розділення обов’язків.
- **Внутрішній CSS**: ці стилі записані всередині тегів `<style>` у розділі `head` HTML-документа. Це може бути корисно для створення коротких прикладів коду, але зазвичай ви не використовуватимете внутрішній CSS.
- **Зовнішній CSS**: ці стилі записані в окремому CSS-файлі та підключені до HTML-документа за допомогою елемента `link` у розділі `head`. Для більшості проєктів ви використовуватимете зовнішній CSS-файл замість внутрішнього чи рядкового CSS.

## Робота з властивостями `width` і `height`

- **Властивість `width`**: ця властивість задає ширину елемента. Якщо ви не вкажете ширину, то за замовчуванням встановлюється `auto`. Це дозволяє браузеру визначати ширину елемента залежно від його вмісту, батьківського елемента та типу відтворення.
- **Властивість `min-width`**: ця властивість задає мінімальну ширину елемента.
- **Властивість `max-width`**: ця властивість задає максимальну ширину елемента.
- **Властивість `height`**: ця властивість задає висоту елемента. Аналогічно, висотою за замовчуванням є `auto`, тобто вона підлаштовується під вміст всередині.
- **Властивість `min-height`**: ця властивість задає мінімальну висоту елемента.
- **Властивість `max-height`**: ця властивість задає максимальну висоту елемента.

## Різні типи CSS-комбінаторів

- **Комбінатор нащадка (` `)&nbsp;\*один пробіл між селекторами**: цей комбінатор використовується для вибору елементів, які є нащадками вказаного батьківського елемента.
- **Комбінатор дочірнього елемента (`>`)**: цей комбінатор використовується для вибору елементів, які є прямими дочірніми елементами вказаного батьківського елемента.
- **Комбінатор наступного братського елемента (`+`)**: цей комбінатор вибирає елемент, який йде безпосередньо після вказаного братського елемента.

- **Комбінатор наступних братських елементів (`~`)**: цей комбінатор вибирає всі братські елементи вказаного елемента, які йдуть після нього.

## Рядкові, блокові та рядково-блокові елементи

- **Рядкові елементи**: Рядкові елементи займають лише ту ширину, яка їм потрібна, і не починаються з нового рядка. Вони розміщуються в потоці вмісту, дозволяючи тексту та іншим рядковим елементам розташовуватися поруч. Загальними рядковими елементами є `span`, `a` та `img`.
- **Блочні елементи**: Блочні елементи за замовчуванням займають всю доступну ширину, розтягуючись на ширину свого контейнера. Загальними блочними елементами є `div`, `p` та `section`.
- **Елементи рівня inline-block**: Ви можете встановити елементу значення `inline-block` через властивість `display`. Такі елементи поводяться як рядкові, але можуть мати задані `width` і `height`, як блочні елементи.

## Поле і відступ

- **Властивість `margin`**: ця властивість використовується, щоб додати місце поза межами елемента, між кордоном елемента та навколишніми елементами.
- **Скорочений запис `margin`**: ви можете вказати від 1 до 4 значень, щоб встановити поля з усіх боків. Одне значення застосовується до всіх чотирьох сторін; два значення встановлюють `top` і `bottom`, потім `right` і `left`; три значення встановлюють `top`, горизонтальні (`right` і `left`), потім `bottom`; чотири значення встановлюють `top`, `right`, `bottom`, `left`.
- **Властивість `padding`**: ця властивість використовується, щоб додати місце всередині елемента, між вмістом і його кордоном.
- **Скорочений запис `padding`**: ви можете вказати від 1 до 4 значень, щоб встановити відступи з усіх боків. Одне значення застосовується до всіх чотирьох сторін; два значення встановлюють `top` і `bottom`, потім `right` і `left`; три значення встановлюють `top`, горизонтальні (`right` і `left`), потім `bottom`; чотири значення встановлюють `top`, `right`, `bottom`, `left`.

## Специфічність CSS

- **Специфічність рядкового CSS**: Рядковий CSS має найвищу специфічність, оскільки застосовується безпосередньо до елемента. Він перекриває будь-який внутрішній або зовнішній CSS. Значення специфічності для рядкових стилів — (1, 0, 0, 0).
- **Специфічність внутрішнього CSS**: Внутрішній CSS визначається у елементі `style` у секції `head` HTML-документа. Він має нижчу специфічність, ніж рядкові стилі. Внутрішній і зовнішній CSS мають однаковий рівень специфічності, який визначається їхніми селекторами, а не розташуванням.
- **Специфічність зовнішнього CSS**: Зовнішній CSS підключається через елемент `link` у секції `head` і записується у окремих `.css` файлах. Як і внутрішній CSS, його специфічність визначається селекторами. Якщо два правила мають однакову специфічність, перевагу має те, що йде пізніше у документі. Зовнішній CSS забезпечує найкращу підтримуваність для великих проєктів.
- **Універсальний селектор (`*`)**: Це особливий тип CSS-селектора, який відповідає будь-якому елементу в документі. Його часто використовують для застосування стилю до всіх елементів сторінки, що корисно для скидання або нормалізації стилів у різних браузерах. Універсальний селектор має найнижче значення специфічності — (0, 0, 0, 0).
- **Селектори типу**: Ці селектори націлені на елементи за їх тегом. Вони мають відносно низьку специфічність порівняно з іншими селекторами. Значення специфічності для селектора типу — (0, 0, 0, 1).
- **Селектори класу**: Визначаються крапкою (`.`) з іменем класу. Значення специфічності для селектора класу — (0, 0, 1, 0). Це означає, що селектори класу можуть перекривати селектори типу, але їх можуть перекривати селектори ID і рядкові стилі.
- **Селектори ID**: Визначаються символом решітки (`#`) з ідентифікатором. Вони мають дуже високу специфічність, вищу за селектори типу і класу, але нижчу за рядкові стилі. Значення специфічності для селектора ID — (0, 1, 0, 0).
- **Ключове слово `!important`**: Використовується, щоб надати правилу стилю найвищий пріоритет, дозволяючи йому перекривати будь-які інші оголошення властивості. При використанні воно змушує браузер застосувати вказаний стиль незалежно від специфічності інших селекторів. Використовуйте `!important` обережно, оскільки це ускладнює підтримку та налагодження CSS.
- **Алгоритм каскадування**: Алгоритм, який визначає, які CSS-правила застосовувати, коли кілька стилів націлені на один елемент. Він гарантує використання найбільш відповідних стилів на основі чітко визначених правил.
- **Успадкування CSS**: Процес, за якого стилі передаються від батьківських елементів до їхніх дочірніх. Успадкування дозволяє визначати стилі на вищому рівні дерева документа і застосовувати їх до кількох елементів без явного вказування для кожного.

## Терміни дизайну

- **Макет**: це спосіб розташування візуальних елементів на сторінці або екрані для передачі повідомлення. Ці елементи можуть включати текст, зображення та порожній простір.
- **Вирівнювання**: це спосіб розміщення елементів один відносно одного. Правильне вирівнювання допомагає зробити дизайн чистим і організованим.
- **Композиція**: це процес розташування елементів для створення гармонійного дизайну. Вона визначає, як такі елементи, як зображення, текст і форми, взаємодіють між собою та вносять художній внесок у дизайн.
- **Баланс**: це розподіл візуальної ваги в композиції. Дизайнери прагнуть створити рівновагу через симетричні або асиметричні розташування.
- **Масштаб**: це порівняння розмірів або габаритів одного елемента з іншим.
- **Ієрархія**: визначає порядок важливості елементів у дизайні. Йдеться про те, щоб найважливіша інформація була помічена першою.
- **Контраст**: це процес створення чітких відмінностей між елементами. Це можна зробити за допомогою варіацій кольору, розміру, форми, текстури або будь-якої іншої візуальної характеристики. Високий контраст також допомагає покращити читабельність.
- **Порожній простір**: це порожня область у дизайні. Це простір навколо елементів.
- **UI (інтерфейс користувача)**: UI включає візуальні та інтерактивні елементи, які користувачі бачать на своїх екранах, такі як іконки, зображення, текст, меню, посилання та кнопки.
- **UX (досвід користувача)**: UX стосується того, як користувачі відчувають себе під час використання продукту або послуги. Застосунок із добре продуманим досвідом користувача є інтуїтивним, простим у використанні, ефективним, доступним і приємним.
- **Технічне завдання дизайну**: це документ, який окреслює цілі, завдання та вимоги проєкту. Він є дорожньою картою, що керує процесом дизайну і гарантує, що кінцевий продукт відповідає потребам клієнта.
- **Векторний дизайн**: це створення цифрового мистецтва за допомогою математичних формул для визначення ліній, форм і кольорів.
- **Прототипування**: це процес створення інтерактивної моделі продукту або інтерфейсу користувача.

## Основи UI дизайну

- **Хороший контраст між фоном і переднім планом**: важливо забезпечити хороший контраст між кольорами фону і переднього плану, щоб текст був читабельним. Рекомендації Web Content Accessibility Guidelines (WCAG) пропонують мінімальний коефіцієнт контрастності 4.5:1 для звичайного тексту і 3:1 для великого тексту.
- **Хороша візуальна ієрархія в дизайні**: сильна візуальна ієрархія допомагає очам чітко слідувати за інформацією, забезпечуючи її сприйняття у потрібному порядку.
- **Адаптивні зображення**: адаптивні зображення — це зображення, які масштабуються відповідно до розміру екрану, на якому їх переглядають. Це важливо, бо гарантує хороший вигляд зображень на будь-яких пристроях — від великих екранів до мобільних телефонів.
- **Прогресивне покращення**: це підхід у дизайні, який гарантує, що всі користувачі, незалежно від браузера чи пристрою, матимуть доступ до основного вмісту та функціональності застосунку.
- **Дизайн, орієнтований на користувача**: це підхід, який ставить на перше місце кінцевого користувача — його потреби, вподобання та обмеження. Мета такого дизайну — створити вебсторінку, яка буде інтуїтивною, ефективною у використанні та приємною для взаємодії.
- **Дослідження користувачів**: це систематичне вивчення людей, які користуються продуктом. Мета — виміряти потреби, поведінку та проблемні моменти користувачів.
- **Тестування користувачами**: це практика збору даних від користувачів під час їхньої взаємодії із застосунком.
- **A/B-тестування**: це процес впровадження нової функції для випадково обраної підмножини користувачів. Потім можна використовувати аналітичні дані, щоб визначити, чи є ця функція корисною.
- **Вимоги користувачів**: це історії або критерії, яким має відповідати застосунок. Вимоги користувачів можуть визначатися дослідженнями, галузевими стандартами або внеском зацікавлених сторін.
- **Прогресивне розкриття**: це шаблон дизайну, який показує користувачам лише релевантний вміст залежно від їхньої поточної дії, приховуючи решту. Це робиться, щоб зменшити когнітивне навантаження і зробити досвід користувача більш інтуїтивним.
- **Відкладена/лінива реєстрація**: це шаблон UI-дизайну, який дозволяє користувачам переглядати і взаємодіяти із застосунком без необхідності реєструватися.

## Найкращі практики дизайну

- **Темний режим**: Особлива функція веб-застосунків, яка дозволяє змінити стандартну світлу кольорову схему на темну. У темному режимі слід використовувати десатуровані кольори — менш насичені та з нижчим рівнем насиченості.
- **Хлібні крихти**: Навігаційна підказка, що показує користувачу, де він знаходиться у ієрархії сайту. Найкраще розміщувати хлібні крихти у верхній частині сторінки, щоб користувачі легко їх знаходили. Важливо, щоб вони були достатньо великими для легкого читання, але не займали забагато місця.
- **Компонент картки**: Ваш компонент картки має бути простим у дизайні, без візуального перевантаження або надмірної інформації. Для медіа обирайте якісні зображення та відео, щоб покращити користувацький досвід.
- **Нескінченне прокручування**: Патерн дизайну, який завантажує більше вмісту під час прокручування сторінки користувачем. Розгляньте можливість додати кнопку «завантажити більше», щоб користувач міг контролювати, коли побачити більше вмісту. Також можна додати кнопку «назад», щоб користувач міг повернутися на попередню сторінку без прокручування вгору.
- **Модальне вікно**: Тип спливаючого вікна, яке відображається поверх існуючого вмісту сторінки. Зазвичай фон затемнюється, щоб допомогти користувачу краще зосередитися на модальному вмісті. Також завжди добре дозволити користувачу натиснути поза модальним вікном, щоб його закрити. Використання HTML-елемента `dialog` надає багато функціональності та переваг доступності.
- **Індикатор прогресу для реєстрації у формі**: Спосіб показати користувачам, наскільки далеко вони просунулися у процесі. Використовується у формах, реєстрації та налаштуваннях. Ваш дизайн має бути простим, легким для знаходження і дозволяти повертатися до попередніх кроків.
- **Кошик для покупок**: Місце, де користувачі бачать вибрані товари на платформі електронної комерції. Кошики мають бути завжди видимими, використовувати загальний значок, як кошик, сумка або корзина, і мати чітку кнопку заклику до дії для переходу до оформлення замовлення.

## Загальні інструменти дизайну

Він дозволяє командам дизайнерів і розробників співпрацювати з будь-якого місця, пропонуючи вбудовані функції, включно з векторним дизайном, автоматичним макетом, системою коментарів і відгуків та інше.
- **Sketch**: Популярний інструмент дизайну, відомий своїм інтуїтивним інтерфейсом і простотою, ідеальний для розробників, які хочуть швидко створювати прототипи. Також широко використовується дизайнерами для створення UI, іконок і вебмакетів.
- **Adobe XD**: Це векторний інструмент для дизайну та прототипування UI/UX, відомий своєю безшовною інтеграцією з іншими додатками Adobe, такими як Photoshop, Illustrator і After Effects.

## Абсолютні одиниці

- **`px` (пікселі)**: Ця абсолютна одиниця є фіксованою одиницею виміру в CSS, що забезпечує точний контроль розмірів. 1px завжди дорівнює 1/96 дюйма.
- **`in` (дюйм)**: Ця абсолютна одиниця дорівнює 96px.
- **`cm` (сантиметри)**: Ця абсолютна одиниця дорівнює 25.2/64 дюйма.
- **`mm` (міліметри)**: Ця абсолютна одиниця дорівнює 1/10 сантиметра.
- **`q` (четверть міліметра)**: Ця абсолютна одиниця дорівнює 1/40 сантиметра.
- **`pc` (піки)**: Ця абсолютна одиниця дорівнює 1/6 дюйма.
- **`pt` (пункти)**: Ця абсолютна одиниця дорівнює 1/72 дюйма.

## Відносні одиниці

- **Відсотки**: ці відносні одиниці дозволяють визначати розміри, виміри та інші властивості як пропорцію від батьківського елемента. Наприклад, якщо ви встановите `width: 50%;` для елемента, він займатиме половину ширини батьківського контейнера.
- **Одиниця `em`**: ці одиниці відносні до розміру шрифту елемента. Якщо ви використовуєте `ems` для властивості `font-size`, то розмір тексту буде відносним до розміру шрифту батьківського елемента.
- **Одиниця `rem`**: ці одиниці відносні до розміру шрифту кореневого елемента, яким є елемент `html`.
- **Одиниця `vh`**: `vh` означає `"viewport height"`, а `1vh` дорівнює 1% висоти вікна перегляду.
- **Одиниця `vw`**: `vw` означає `"viewport width"`, а `1vw` дорівнює 1% ширини вікна перегляду.

## Функція `calc`

- **Функція `calc()`**: за допомогою функції `calc()` ви можете виконувати обчислення безпосередньо у таблицях стилів, щоб динамічно визначати значення властивостей. Це означає, що ви можете створювати гнучкі та адаптивні інтерфейси користувача, обчислюючи розміри на основі розміру вікна перегляду або інших елементів.

## Псевдокласи дій користувача

- **Визначення псевдокласів**: це спеціальні ключові слова CSS, які дозволяють вибирати елемент залежно від його конкретного стану або позиції.
- **Псевдокласи дій користувача**: це спеціальні ключові слова, які дозволяють змінювати вигляд елементів залежно від взаємодії користувача, покращуючи загальний досвід користувача.
- **Псевдоклас `:active`**: цей псевдоклас дозволяє вибрати активний стан елемента, наприклад, при натисканні на кнопку.
- **Псевдоклас `:hover`**: цей псевдоклас визначає стан наведення на елемент.
- **Псевдоклас `:focus`**: цей псевдоклас застосовує стилі, коли елемент отримує фокус, зазвичай через навігацію клавіатурою або коли користувач натискає на вхідні дані форми.
- **Псевдоклас `:focus-within`**: цей псевдоклас використовується для застосування стилів до елемента, коли він або будь-який із його нащадків має фокус.

## Псевдокласи для вхідних даних

- **Псевдокласи для вхідних даних**: ці псевдокласи використовуються для націлювання HTML-елементів `input` залежно від стану, у якому вони перебувають до та після взаємодії користувача.
- **Псевдоклас `:enabled`**: цей псевдоклас використовується для націлювання кнопок форми або інших елементів, які наразі увімкнені.
- **Псевдоклас `:disabled`**: цей псевдоклас дозволяє стилізувати інтерактивний елемент у вимкненому режимі.
- **Псевдоклас `:checked`**: цей псевдоклас використовується, щоб показати користувачу, що елемент позначений.
- **Псевдоклас `:valid`**: цей псевдоклас націлює поля введення, які відповідають критеріям валідації.
- **Псевдоклас `:invalid`**: цей псевдоклас націлює поля введення, які не відповідають критеріям валідації.
- **Псевдокласи `:in-range` та `:out-of-range`**: ці псевдокласи застосовують стилі до елементів залежно від того, чи їхні значення знаходяться в межах або поза межами заданих обмежень діапазону.
- **Псевдоклас `:required`**: цей псевдоклас націлює елементи `input`, які мають атрибут `required`. Він сигналізує користувачу, що поле потрібно заповнити для відправлення форми.
- **Псевдоклас `:optional`**: цей псевдоклас застосовує стилі до вхідних елементів, які не є обов’язковими і можуть залишатися порожніми.
- **Псевдоклас `:autofill`**: цей псевдоклас застосовує стилі до полів введення, які браузер автоматично заповнює збереженими даними.

## Псевдокласи локації

- **Псевдокласи локації**: Ці псевдокласи використовуються для стилізації посилань і елементів, які є ціллю у поточному документі.
- **`:any-link` Псевдоклас**: Поєднує `:link` і `:visited`. Відповідає будь-якому елементу посилання з атрибутом `href`, незалежно від того, чи було його відвідано.
- **`:link` Псевдоклас**: Дозволяє націлитися на всі непереглянуті посилання на веб-сторінці. Використовуйте для стилізації посилань до кліку користувача.
- **`:local-link` Псевдоклас**: Націлюється на посилання, які ведуть до того ж документа. Корисний для розрізнення внутрішніх і зовнішніх посилань.
- **`:visited` Псевдоклас**: Націлюється на посилання, які користувач вже відвідав.
- **`:target` Псевдоклас**: Використовується для застосування стилів до елемента, який є ціллю фрагмента URL.

## Псевдокласи, що базуються на структурі дерева

- **Псевдокласи, що базуються на структурі дерева**: ці псевдокласи дозволяють націлюватися та стилізувати елементи залежно від їхнього розташування в дереві документа.
- **Псевдоклас `:root`**: цей псевдоклас зазвичай є кореневим `html`-елементом. Він допомагає націлитися на найвищий рівень у документі, щоб застосувати поширений стиль до всього документа.
- **Псевдоклас `:empty`**: порожні елементи, тобто елементи без дочірніх, окрім порожнього простору, також включені в дерево документа. Саме тому для націлювання на порожні елементи існує псевдоклас `:empty`.
- **Псевдоклас `:nth-child(n)`**: цей псевдоклас дозволяє вибирати елементи залежно від їхнього розташування всередині батьківського елемента.
- **Псевдоклас `:nth-last-child(n)`**: цей псевдоклас дає змогу вибирати елементи, рахуючи з кінця.
- **Псевдоклас `:first-child`**: цей псевдоклас вибирає перший елемент у батьківському елементі або в документі.
- **Псевдоклас `:last-child`**: цей псевдоклас вибирає останній елемент у батьківському елементі або в документі.
- **Псевдоклас `:only-child`**: цей псевдоклас вибирає єдиний елемент у батьківському елементі або в документі.
- **Псевдоклас `:first-of-type`**: цей псевдоклас вибирає перше входження певного типу елемента всередині його батьківського елемента.
- **Псевдоклас `:last-of-type`**: цей псевдоклас вибирає останнє входження певного типу елемента всередині його батьківського елемента.
- **Псевдоклас `:nth-of-type(n)`**: цей псевдоклас дозволяє вибрати конкретний елемент у межах батьківського елемента, базуючись на його позиції серед братських елементів того самого типу.
- **Псевдоклас `:only-of-type`**: цей псевдоклас вибирає елемент, якщо він є єдиним такого типу всередині свого батьківського елемента.


## Функціональні псевдокласи

- **Функціональні псевдокласи**: Функціональні псевдокласи дозволяють вибирати елементи на основі складніших умов або зв’язків. На відміну від звичайних псевдокласів, які орієнтуються на елемент за певним станом (наприклад, `:hover`, `:focus`), функціональні псевдокласи приймають аргументи.
- **Псевдоклас `:is()`**: Цей псевдоклас приймає список селекторів (наприклад, `ol`, `ul`) і вибирає елемент, який відповідає одному з селекторів у списку.
- **Псевдоклас `:where()`**: Цей псевдоклас приймає список селекторів (наприклад, `ol`, `ul`) і вибирає елемент, який відповідає одному з селекторів у списку. Різниця між `:is()` і `:where()` полягає в тому, що останній має специфічність 0.
- **Псевдоклас `:has()`**: Цей псевдоклас часто називають селектором `"parent"`, бо він дозволяє стилізувати елементи, які містять дочірні елементи, вказані у списку селекторів.

## Псевдоелементи

- **Псевдоелемент `::before`**: цей псевдоелемент використовує властивість `content` для вставлення косметичного вмісту (наприклад, іконок) безпосередньо перед елементом.
- **Псевдоелемент `::after`**: цей псевдоелемент використовує властивість `content` для вставлення косметичного вмісту (наприклад, іконок) безпосередньо після елемента.
- **Псевдоелемент `::first-letter`**: цей псевдоелемент націлений на першу літеру вмісту елемента, що дозволяє стилізувати її.
- **Псевдоелемент `::marker`**: цей псевдоелемент дає змогу вибрати маркер (крапку або нумерацію) елементів списку для стилізації.


## Теорія кольору

- **Визначення теорії кольору**: Вивчення того, як кольори взаємодіють між собою і як вони впливають на наше сприйняття. Охоплює відносини кольорів, гармонію кольорів і психологічний вплив кольору.
- **Основні кольори**: Жовтий, синій і червоний — базові відтінки, з яких утворюються всі інші кольори.
- **Вторинні кольори**: Утворюються змішуванням рівних частин двох основних кольорів. Прикладами є зелений, помаранчевий і фіолетовий.
- **Третинні кольори**: Утворюються поєднанням основного кольору з сусіднім вторинним. Прикладами є жовто-зелений, синьо-зелений і синьо-фіолетовий.
- **Теплі кольори**: Включають червоні, помаранчеві та жовті, викликають відчуття комфорту, тепла і затишку.
- **Холодні кольори**: Включають сині, зелені та фіолетові, викликають відчуття спокою, умиротворення і професіоналізму.
- **Колірне коло**: Кругова діаграма, що показує взаємозв’язки кольорів. Важливий інструмент для дизайнерів, який допомагає вибирати кольорові поєднання.
- **Аналогічні кольорові схеми**: Створюють гармонійний і заспокійливий ефект. Включають кольори, що розташовані поруч на колірному колі.
- **Доповнюючі кольорові схеми**: Створюють високий контраст і візуальний вплив. Кольори розташовані на протилежних кінцях колірного кола.
- **Тріадична кольорова схема**: Яскрава кольорова схема, утворена кольорами, приблизно рівновіддаленими один від одного. Якщо їх з’єднати, утвориться рівносторонній трикутник на колірному колі.
- **Монохроматична кольорова схема**: Всі кольори походять від одного базового кольору шляхом регулювання світлості, темряви та насиченості. Викликає відчуття єдності і гармонії, одночасно створюючи контраст.

## Різні способи роботи з кольорами в CSS

- **Іменовані кольори**: Попередньо визначені назви кольорів, які розпізнають браузери. Приклади: `blue`, `darkred`, `lightgreen`.
- **Функція `rgb()`**: RGB означає червоний, зелений і синій — основні кольори світла. Ці три кольори комбінуються з різною інтенсивністю для створення широкого спектра кольорів. Функція `rgb()` дозволяє визначати кольори за моделлю RGB.
- **Функція `rgba()`**: Додає четверте значення — альфа-канал, який контролює прозорість кольору.
- **Функція `hsl()`**: HSL означає відтінок (Hue), насиченість (Saturation) і світлість (Lightness) — три ключові компоненти, що визначають колір.
- **Функція `hsla()`**: Додає четверте значення — альфа-канал, який контролює непрозорість кольору.
- **Шістнадцятковий код**: Шістнадцятковий код (hex) — це рядок з шести символів, що використовується для представлення кольорів у моделі RGB. "Hex" означає систему числення з основою 16, яка використовує цифри 0–9 і літери A–F.

## Властивість `box-shadow`

- **Визначення**: властивість `box-shadow` застосовує одну або кілька тіней навколо елемента.
- **Значення зміщення**: ви повинні вказати горизонтальні (`offset-x`) та вертикальні (`offset-y`) значення. Додатне `offset-x` зміщує тінь вправо, тоді як від’ємні значення зміщують її вліво. Додатні `offset-y` зміщує тінь вниз, а від’ємні — вгору. Якщо значення `0`, не потрібно вказувати одиниці вимірювання.
- **Радіус розмиття**: це необов’язкове значення контролює, наскільки розмитою виглядає тінь. Якщо його не вказано, за замовчуванням використовується `0`, що створює чіткі краї. Чим більше значення, тим м’якша тінь.
- **Радіус розповсюдження**: це необов’язкове значення контролює, наскільки тінь розширюється або звужується. Якщо його не вказано, за замовчуванням використовується `0`.
- **Колір тіні**: ви можете вказати колір за допомогою іменованих кольорів, шістнадцяткових значень, `rgb()`, `rgba()`, `hsl()` або `hsla()`.
- **Ключове слово `inset`**: додавання ключового слова `inset` розміщує тінь всередині елемента замість поза межами.
- **Застосування кількох тіней блоку**: ви можете застосувати кілька тіней, розділяючи їх комами. Тіні накладаються шарами від переднього плану до заднього.

### Синтаксис

```css
box-shadow: offset-x offset-y blur-radius spread-radius color;
```

- **Радіальні градієнти**: ці градієнти створюють кругові або еліптичні градієнти, що розходяться від центральної точки.

## Найкращі практики стилізації вхідних даних

- **Стилізація вхідних даних**: Як і з усіма текстовими елементами, потрібно переконатися, що стилі, які ви застосовуєте до текстових вхідних даних, є доступними. Це означає, що шрифт має бути достатньо великим, а колір — мати достатній контраст із фоном. Елементи вхідних даних також можуть отримувати фокус. Під час редагування стилів слід подбати про збереження помітного індикатора, коли елемент у фокусі, наприклад, жирний кордон.

## Використання `appearance: none` для вхідних даних

- **`appearance: none`**: Браузери застосовують стилі за замовчуванням до багатьох елементів. Властивість CSS `appearance: none` дає повний контроль над стилізацією, але має певні застереження. При створенні кастомних стилів для елементів вхідних даних потрібно переконатися, що індикатори фокусу та помилок залишаються видимими.

## Загальні проблеми стилізації властивостей `datetime-local` і `color`

- **Загальні проблеми**: ці спеціальні типи вхідних даних використовують складні псевдоелементи для створення таких речей, як вибір дати та кольору. Це створює значні труднощі при стилізації цих вхідних даних. Одна з проблем полягає в тому, що стилі за замовчуванням повністю залежать від браузера, тому CSS, який ви пишете, може виглядати зовсім інакше в іншому браузері.

## CSS-властивість overflow

- **Визначення**: overflow (укр. переповнення) означає, як елементи обробляють вміст, який перевищує або «виходить за межі» розміру контейнера. Переповнення є двовимірним.
- **`overflow-x`**: вісь x визначає горизонтальне переповнення.
- **`overflow-y`**: вісь y визначає вертикальне переповнення.

## CSS-властивість transform

- **Визначення**: ця властивість дозволяє застосовувати різні трансформації до елементів, такі як обертання, масштабування, нахил або переміщення у 2D або 3D просторі.
- **Функція `translate()`**: ця функція використовується для переміщення елемента з його поточного положення.
- **Функція `scale()`**: ця функція дозволяє змінювати розмір елемента.
- **Трансформації та доступність**: якщо ви використовуєте transform для приховування або показу вмісту, то переконайтеся, що вміст залишається доступним для читачів екрана та навігації за допомогою клавіатури. Прихований вміст має бути справді прихованим, наприклад, за допомогою `display: none` або `visibility: hidden`, а не просто візуально переміщеним за межі екрану.

## Блокова модель

- **Визначення**: у блоковій моделі кожен елемент оточений блоком. Цей блок складається з чотирьох компонентів: області вмісту, `padding`, `border` та `margin`.
- **Область вмісту**: область вмісту — це найвнутрішніша частина блоку. Це простір, який містить фактичний вміст елемента (наприклад, текст або зображення).
- **`padding`**: відступ — це область безпосередньо після області вмісту. Це простір між областю вмісту та кордоном елемента.
- **`border`**: кордон — це зовнішній край або обриси елемента в блоковій моделі. Це візуальна межа елемента.
- **`margin`**: поле — це простір поза межами кордону елемента. Воно визначає відстань між елементом та іншими елементами навколо нього.

## Злиття полів

- **Визначення**: ця поведінка виникає, коли вертикальні поля суміжних елементів накладаються одне на одне, утворюючи одне поле, яке дорівнює більшому з двох. Така поведінка застосовується лише до вертикальних полів (верхнього та нижнього), а не до горизонтальних (лівого та правого).

## Значення властивостей `content-box` та `border-box`

- **Властивість `box-sizing`**: ця властивість використовується для визначення, як обчислюються фінальні `width` і `height` для HTML-елемента.
- **Значення `content-box`**: у моделі `content-box` `width` і `height`, які ви задаєте для елемента, визначають розміри області вмісту, але не включають `padding`, `border` або `margin`.
- **Значення `border-box`**: у моделі `border-box` `width` і `height` елемента включають область вмісту, `padding` і `border`, але не включають `margin`.

## Скидання CSS

- **Визначення**: скидання CSS — це таблиця стилів, яка видаляє всі або частину форматування за замовчуванням, яке веббраузери застосовують до HTML-елементів. Серед сторонніх варіантів скидання CSS є `sanitize.css` і `normalize.css`.

## CSS-властивість filter

- **Визначення**: цю властивість можна використовувати для створення різних ефектів, таких як розмиття, зміна кольору та правка контрасту.
- **Функція `blur()`**: ця функція застосовує гаусове розмиття до елемента. Кількість визначається в пікселях і представляє радіус розмиття.
- **Функція `brightness()`**: ця функція регулює яскравість елемента. Значення 0% зробить елемент повністю чорним, а значення понад 100% збільшать яскравість.
- **Функція `grayscale()`**: ця функція перетворює елемент у відтінки сірого. Кількість визначається у відсотках, де 100% — повністю у відтінках сірого, а 0% залишає зображення без змін.
- **Функція `sepia()`**: ця функція застосовує сепію до елемента. Як і для відтінків сірого, використовується відсоткове значення.
- **Функція `hue-rotate()`**: ця функція застосовує обертання відтінку до елемента. Значення визначається в градусах і представляє оберт навколо кола кольорів.

## Вступ до flexbox і моделі flex

- **Визначення**: CSS flexbox — це одномірна модель макета, яка дозволяє розташовувати елементи в рядках і стовпцях усередині контейнера.  
- **Модель flex**: ця модель визначає, як гнучкі елементи розташовуються всередині гнучкого контейнера. Кожен гнучкий контейнер має дві осі: головну і поперечну.

## Властивість `flex-direction`

- **Визначення**: ця властивість встановлює напрямок головної осі. Значення за замовчуванням `flex-direction` — `row`, що розміщує всі гнучкі елементи в одному рядку у напрямку мови браузера (зліва направо або справа наліво).
- **`flex-direction: row-reverse;`**: це змінює порядок елементів у рядку на зворотний.
- **`flex-direction: column;`**: це вирівнює гнучкі елементи вертикально замість горизонтально.
- **`flex-direction: column-reverse;`**: це змінює порядок гнучких елементів вертикально на зворотний.

## Властивість `flex-wrap`

- **Визначення**: Ця властивість визначає, як гнучкі елементи обгортаються всередині контейнера flex, щоб поміститися у доступний простір. `flex-wrap` може приймати три значення: `nowrap`, `wrap` і `wrap-reverse`.
- **`flex-wrap: nowrap;`**: Значення за замовчуванням. Гнучкі елементи не будуть переноситися на новий рядок, навіть якщо їх ширина перевищує ширину контейнера.
- **`flex-wrap: wrap;`**: Ця властивість дозволяє переносити елементи, коли вони перевищують ширину контейнера.
- **`flex-wrap: wrap-reverse;`**: Ця властивість переносить гнучкі елементи у зворотному порядку.
- **Властивість `flex-flow`**: Скорочена властивість для `flex-direction` і `flex-wrap`.

## Властивість `justify-content`

- **Визначення**: ця властивість вирівнює дочірні елементи вздовж головної осі гнучкого контейнера.
- **`justify-content: flex-start;`**: у цьому випадку гнучкі елементи будуть вирівняні на початку головної осі. Це може бути горизонтально або вертикально.
- **`justify-content: flex-end;`**: у цьому випадку гнучкі елементи вирівняні в кінці головної осі, горизонтально або вертикально.
- **`justify-content: center;`**: це центрує гнучкі елементи вздовж головної осі.
- **`justify-content: space-between;`**: це рівномірно розподіляє елементи вздовж головної осі.
- **`justify-content: space-around;`**: це рівномірно розподіляє гнучкі елементи всередині головної осі, додаючи пробіл перед першим елементом і після останнього.
- **`justify-content: space-evenly;`**: це рівномірно розподіляє елементи вздовж головної осі.
  
## Властивість `align-items`

- **Визначення**: ця властивість використовується для розподілу елементів вздовж поперечної осі. Пам’ятайте, що поперечна вісь перпендикулярна до головної осі.
- **`align-items: center;`**: використовується для центрування елементів вздовж поперечної осі.
- **`align-items: flex-start;`**: вирівнює елементи на початку поперечної осі.
- **`align-items: stretch;`**: використовується для розтягування гнучких елементів вздовж поперечної осі.

## Вступ до типографіки

- **Визначення**: типографіка — це мистецтво вибору правильних шрифтів і форматування, щоб текст був візуально привабливим і легким для читання. «Тип» означає, як окремі символи спроєктовані та розташовані.
- **Визначення гарнітури**: гарнітура — це загальний дизайн і стиль набору символів, чисел і знаків. Це як основа для сімейства шрифтів.
- **Визначення шрифту**: шрифт — це конкретний варіант гарнітури з певними характеристиками, такими як розмір, товщина, стиль і ширина.
- **Гарнітура з зарубками (serif)**: ця гарнітура має класичний стиль із маленькими лініями на кінцях символів. Гарнітури з зарубками зазвичай використовують для друкованих матеріалів (наприклад, книг).
- **Гарнітура без зарубок (sans serif)**: ця гарнітура має більш сучасний вигляд, без маленьких ліній на кінцях символів. Гарнітури без зарубок часто використовують у цифровому дизайні, бо їх легко читати на екрані. Прикладами є Helvetica, Arial і Roboto.
- **Товщина шрифту (font weight)**: це товщина символів, включно з light, regular, bold та black.
- **Стиль шрифту (font style)**: це нахил і орієнтація символів, наприклад, курсив і похилий.
- **Ширина шрифту (font width)**: це горизонтальний простір, який займають символи (наприклад, стиснутий і розширений).
- **Базова лінія (baseline)**: це уявна лінія, на якій стоять більшість символів.
- **Висота прописних літер (cap height)**: це висота великих літер, виміряна від базової лінії до верхньої межі.
- **Висота x (x-height)**: це середня висота малих літер, без урахування висхідних і низхідних елементів.
- **Висхідні елементи (ascenders)**: це частини малих літер, що виступають над висотою x (наприклад, верхівки літер 'h', 'b', 'd' і 'f').
- **Низхідні елементи (descenders)**: це частини малих літер, що виступають нижче базової лінії (наприклад, хвостики літер 'y', 'g', 'p' і 'q').
- **Кернінг (kerning)**: це регулювання простору між конкретними парами символів для покращення читабельності та естетики. Наприклад, зменшення простору між літерами A і V.
- **Трекінг (tracking)**: це регулювання простору між усіма символами в блоці тексту. Це відстань між символами, що впливає на щільність і відкритість тексту.
- **Інтерліньяж (leading)**: це вертикальний простір між рядками тексту. Вимірюється від базової лінії одного рядка до базової лінії наступного.
- **Найкращі практики з типографіки**: обирайте чіткі та прості шрифти, щоб дизайн був легким для розуміння. Це особливо важливо для основного тексту сайту. Користувачі частіше взаємодіють із вмістом, якщо шрифт легко читати. Використовуйте не більше двох-трьох шрифтів, щоб створити візуальну послідовність. Занадто багато шрифтів ускладнюють читання тексту і послаблюють ідентичність бренду. Це також може вплинути на досвід користувача, збільшуючи час завантаження сайту. Використовуйте розмір шрифту, щоб створити візуальну ієрархію для заголовків, підзаголовків, абзаців та інших елементів. Наприклад, головний заголовок на сторінці має бути більшим, а підзаголовки — меншими. Це дасть кожному елементу ієрархії конкретний розмір шрифту, що допоможе користувачам орієнтуватися у структурі візуально.

  
## CSS-властивість `font-family`

- **Визначення**: сімейство шрифтів — це група шрифтів, які мають один дизайн. Усі шрифти, що належать до одного сімейства, базуються на одному основному типі шрифту, але також мають варіації у стилі, товщині та ширині. Ви можете вказати кілька сімейств шрифтів у порядку пріоритету, від найвищого до найнижчого, розділяючи їх комами. Ці альтернативні шрифти будуть виконувати роль запасних варіантів. Завжди варто додавати загальне сімейство шрифтів наприкінці списку font-family.
- **Поширені сімейства шрифтів**: ось деякі поширені сімейства шрифтів, які використовуються в CSS: serif, sans-serif, monospace, cursive, fantasy.

## Веббезпечні шрифти

- **Визначення**: ці шрифти є підмножиною шрифтів, які дуже ймовірно встановлені на комп’ютері або пристрої. Коли браузер має відтворити шрифт, він намагається знайти файл шрифту в системі користувача. Але якщо шрифт не знайдено, зазвичай він переходить до шрифту за замовчуванням у системі.

- **Веббезпечні шрифти**: 
  - Sans-serif часто використовують для веброзробки, бо вони не мають маленьких «ніжок» або лінійок на кінцях символів, тому їх легко читати на екрані. Деякі приклади веббезпечних шрифтів sans-serif: Arial, Verdana та Trebuchet MS.  
  - Натомість, serif мають маленькі «ніжки» на кінцях символів, тому їх зазвичай використовують для традиційного друку. До них належать: Times New Roman і Georgia.

## Директиви та директива `@font-face`

- **Визначення**: директиви — це інструкції для браузера. Ви можете використовувати їх, щоб визначати різні аспекти таблиці стилів, такі як медіазапити, ключові кадри, шрифти та інше.  
- **`@font-face`**: це дозволяє визначити кастомний шрифт, вказавши файл шрифту, формат і інші важливі властивості, як-от вагу та стиль. Щоб директива `@font-face` була дійсною, також потрібно вказати властивість `src`. Ця властивість містить референси на ресурси шрифту.
- **Формати шрифтів**: для кожного ресурсу шрифту можна також вказати формат. Це необов’язково. Це підказка для браузера щодо формату шрифту. Якщо формат пропущено, то ресурс буде скачано, а формат визначать після завантаження. Якщо формат недійсний, ресурс не буде скачано. Можливі формати шрифтів включають `collection`, `embedded-opentype`, `opentype`, `svg`, `truetype`, `woff` та `woff2`.
- **`woff` та `woff2`**: `woff` означає «Web Open Font Format». Це широко використовуваний формат шрифтів, розроблений Mozilla у співпраці з Type Supply, LettError та іншими організаціями. Різниця між `woff` і `woff2` полягає в алгоритмі стиснення даних.
- **OpenType**: це формат масштабованих комп’ютерних шрифтів, розроблений Microsoft та Adobe, який дозволяє користувачам отримувати доступ до додаткових функцій шрифту. Він широко використовується в основних операційних системах.
- **`tech()`**: використовується, щоб вказати технології ресурсу шрифту. Це необов’язково.

## Робота з зовнішніми шрифтами

- **Визначення**: зовнішній шрифт — це файл шрифту, який не включений безпосередньо у файли проєкту. Зазвичай вони розміщені на окремому сервері. Щоб додати ці зовнішні шрифти до проєкту, ви можете використати елемент `link` або інструкцію `@import` у CSS.
- **Google Fonts**: це сервіс Google, який пропонує колекцію шрифтів, багато з яких спеціально розроблені для веброзробки.
- **Font Squirrel**: це ще один популярний ресурс, який можна використовувати для додавання кастомних зовнішніх шрифтів до проєктів.

## Властивість `text-shadow`

- **Визначення**: ця властивість використовується для додавання тіней до тексту. Потрібно вказати зміщення по X та Y, які відповідно представляють горизонтальну та вертикальну відстань тіні від тексту. Ці значення є обов’язковими. Додатні значення зміщення по X і Y перемістять тінь вправо і вниз відповідно, тоді як від’ємні значення перемістять тінь вліво і вгору.
- **Колір тіні**: ви також можете кастомізувати колір тіні, вказавши це значення до або після зміщення. Якщо колір не вказано, браузер визначить його автоматично, тому зазвичай краще встановити його явно.
- **Радіус розмиття**: радіус розмиття є необов’язковим, але зробить тінь значно м’якшою та більш тонкою. Значення за замовчуванням для радіуса розмиття — нуль. Чим більше значення, тим сильніше розмиття, а отже тінь стає світлішою.
- **Застосування кількох тіней до тексту**: текст може мати більше однієї тіні. Тіні накладаються шарами, спереду назад, причому перша тінь буде зверху.

Цей інструмент також має симулятор, що дозволяє побачити, як вебсайт або застосунок виглядає для людей із порушенням сприйняття кольору.

## Дерево доступності

Дерево доступності — це структура, яку використовують допоміжні технології, такі як читачі екрана, щоб інтерпретувати та взаємодіяти з вмістом на вебсторінці.

## Функція `max()`

Функція `max()` повертає найбільше значення з множини значень, розділених комами:

```css
img {
  width: max(250px, 25vw);
}
```

У наведеному вище прикладі шириною зображення буде 250px, якщо ширина вікна перегляду менша за 1000 пікселів (оскільки 250px більше за 25% цієї ширини вікна перегляду). Якщо ширина вікна перегляду більша за 1000 пікселів, шириною зображення буде 25vw. Це тому, що 25vw дорівнює 25% ширини вікна перегляду.

## Найкращі практики для CSS і доступності

- **`display: none;`**: використання `display: none;` означає, що читачі екрана та інші допоміжні технології не зможуть отримати доступ до цього вмісту, оскільки він не включений у дерево доступності. Тому цей метод важливо застосовувати лише тоді, коли ви хочете повністю видалити вміст як із візуального відображення, так і з доступності.
- **`visibility: hidden;`**: ця властивість і значення приховують вміст візуально, але залишають його в потоці документа, тобто він все ще займає місце на сторінці. Такі елементи також більше не будуть зчитуватися читачами екрана, оскільки вони будуть видалені з дерева доступності.
- **CSS-клас `.sr-only`**: це загальна техніка, яка використовується для візуального приховування вмісту, залишаючи його доступним для читачів екрана.

```css
.sr-only {
  position: absolute;
  width: 1px;
  height: 1px;
  padding: 0;
  margin: -1px;
  overflow: hidden;
  clip: rect(0, 0, 0, 0);
  white-space: nowrap;
  border: 0;
}
```

- **`scroll-behavior: smooth;`**: ця властивість і значення увімкнуть плавну поведінку прокрутки.
- **Функція `prefers-reduced-motion`**: це медіафункція, яка може використовуватися для визначення переваг анімації користувача.

```css
@media (prefers-reduced-motion: no-preference) {
  * {
    scroll-behavior: smooth;
  }
}
```

У наведеному вище прикладі плавна прокрутка увімкнена, якщо на пристрої користувача не встановлено перевагу анімації.

## Приховування вмісту за допомогою HTML-атрибутів

- **Атрибут `aria-hidden`**: використовується для приховування елемента від користувачів допоміжних технологій, таких як читачі екрана. Наприклад, його можна застосувати для приховування декоративних зображень, які не несуть жодного значущого вмісту.
- **Атрибут `hidden`**: цей атрибут підтримується більшістю сучасних браузерів і приховує вміст як візуально, так і з дерева доступності. Його легко можна переключати за допомогою JavaScript.

:::interactive_editor

```html
<p aria-hidden>This paragraph is visible for sighted users, but is hidden from assistive technology.</p>
<p hidden>This paragraph is hidden from both sighted users and assistive technology.</p>
```

:::

## Проблема доступності атрибута `placeholder`

Використання тексту-заповнювача не є оптимальним для доступності. Надто часто користувачі плутають текст-заповнювач із фактичним значенням у вхідних даних — вони думають, що в полі вже є значення.

## Робота з різними селекторами атрибутів і посиланнями

- **Визначення**: селектор `attribute` дозволяє націлюватися на HTML-елементи на основі їхніх атрибутів, таких як атрибути `href` або `title`.
- **Атрибут `title`**: цей атрибут надає додаткову інформацію про елемент.

## Націлення елементів з атрибутом `lang` та `data-lang`

- **Атрибут `lang`**: цей атрибут використовується в HTML для вказівки мови вмісту всередині елемента. Ви можете стилізувати елементи по-різному залежно від мови, якою вони написані, особливо на багатомовному сайті.
- **Атрибут `data-lang`**: кастомні атрибути даних, як-от атрибут `data-lang`, часто використовують для зберігання додаткової інформації в елементах (наприклад, щоб вказати мову, яка використовується в певному розділі тексту).


## Робота з селектором атрибутів, елементами впорядкованого списку та атрибутом `type`

- **Атрибут `type`**: при роботі з впорядкованими списками в HTML атрибут `type` дозволяє вказати стиль нумерації, який використовується (наприклад числовий, алфавітний або римські цифри).

## Робота з float

- **Визначення**: float використовується, щоб вивести елемент із його звичайного потоку на сторінці та розмістити його або зліва, або справа від контейнера. Коли це відбувається, текст обтікає цей плаваючий вміст.
- **Очищення float**: властивість `clear` використовується, щоб визначити, чи потрібно перемістити елемент під плаваючий вміст. Коли у вас є кілька плаваючих елементів, розташованих поруч, можуть виникати проблеми з накладанням і зсувом у макетах. Тому, щоб виправити цю проблему, був створений `clearfix`.

## Статичне, відносне та абсолютне позиціювання

- **Статичне позиціювання**: це звичайний потік документа. Елементи розташовуються зверху вниз і зліва направо один за одним.
- **Відносне позиціювання**: дозволяє використовувати властивості `top`, `left`, `right` і `bottom` для позиціювання елемента в межах нормального потоку документа. Також відносне позиціювання дає змогу елементам накладатися один на одного на сторінці.
- **Абсолютне позиціювання**: дозволяє вивести елемент із нормального потоку документа, роблячи його незалежним від інших елементів.

## Фіксоване та липке позиціонування

- **Фіксоване позиціонування**: коли елемент позиціонується з `position: fixed`, він вилучається з нормального потоку документа і розміщується відносно вікна перегляду, тобто залишається на одному й тому ж місці навіть під час прокручування користувачем. Це часто використовується для таких елементів, як заголовки або панелі навігації, які мають бути завжди видимими.
- **Липке позиціонування**: цей тип позиціонування є гібридом між відносним і фіксованим. Спочатку елемент поводиться так, ніби він позиціонується відносно, залишаючись у потоці документа. Однак, коли користувач прокручує елемент за певну точку, він «прилипає» до вікна перегляду (зазвичай до верху) і поводиться так, ніби він фіксований. Це чудово підходить для створення таких елементів, як липкі панелі навігації, які стають фіксованими лише після того, як користувач прокрутив до певної позиції.

## Робота з властивістю `z-index`

- **Визначення**: властивість `z-index` у CSS використовується для керування вертикальним порядком накладання позиційованих елементів, які перекриваються на сторінці.

## Адаптивний вебдизайн

- **Визначення**: Основний принцип адаптивного дизайну — це здатність сайту підлаштовувати свій макет і вміст залежно від розміру екрану та можливостей пристрою, на якому його переглядають.
- **Гнучкі сітки**: Використовують відносні одиниці, як відсотки, замість фіксованих, як пікселі, що дозволяє вмісту змінювати розмір і переформатовуватися залежно від розміру екрану.
- **Гнучкі зображення**: Налаштовані так, щоб змінювати розмір у межах своїх контейнерів, забезпечуючи, що вони не виходять за межі контейнера на менших екранах.

## Медіа-запити

- **Визначення**: це дозволяє розробникам застосовувати різні стилі залежно від характеристик пристрою, головним чином від ширини вікна перегляду.
- **Медіатип `all`**: підходить для всіх пристроїв. Це значення за замовчуванням, якщо тип не вказано.
- **Медіатип `print`**: призначений для матеріалів із розбиттям на сторінки та документів, які переглядають на екрані в режимі попереднього перегляду друку.
- **Медіатип `screen`**: призначений переважно для екранів.
- **`aspect-ratio`**: описує співвідношення між шириною та висотою вікна перегляду.
- **`orientation`**: використовується для вказівки, чи пристрій у ландшафтній чи портретній орієнтації.
- **`resolution`**: використовується для опису роздільності вихідного пристрою в точках на дюйм (dpi) або точках на сантиметр (dpcm).
- **`hover`**: використовується для перевірки, чи основний механізм введення може наводити курсор на елементи.
- **`prefers-color-scheme`**: використовується для визначення, чи користувач обрав світлу чи темну кольорову тему.
- **Медіазапити та логічні оператори**: оператор `and` використовується для поєднання кількох медіафункцій, тоді як `not` і `only` можна використовувати для заперечення або ізоляції медіазапитів.

## Кастомні CSS-властивості (CSS-змінні)

- **Визначення**: кастомні CSS-властивості, також відомі як CSS-змінні, — це сутності, які визначають автори CSS і які містять конкретні значення для повторного використання по всьому документу. Це потужна функція, що дозволяє створювати більш ефективні, підтримувані та гнучкі стилі. Кастомні властивості особливо корисні для створення дизайнів із підтримкою тем. Ви можете визначити множину властивостей для різних тем.

## Правило `@property`

- **Визначення**: правило `@property` — це потужна CSS-функція, яка дозволяє розробникам визначати кастомні властивості з більшим контролем над їхньою поведінкою, включно з анімацією та початковими значеннями.

```css
@property --property-name {
  syntax: '<type>';
  inherits: true | false;
  initial-value: <value>;
}
```

- **`--property-name`**: Назва кастомної властивості, яку ви визначаєте. Як і всі кастомні властивості, вона має починатися з двох дефісів.
- **`syntax`**: Визначає тип властивості, наприклад `<color>`, `<length>`, `<number>`, `<percentage>` або більш складні типи.
- **`inherits`**: Вказує, чи має властивість успадковувати значення від батьківського елемента.
- **`initial-value`**: Встановлює початкове значення властивості.
- **Запасні варіанти (Fallbacks)**: При використанні кастомної властивості можна вказати запасне значення через функцію `var()`, так само, як і зі стандартними кастомними властивостями.

## CSS-сітка

- **Визначення**: сітка — це двовимірна система макету, яка використовується для створення складних макетів на вебсторінках. Сітки мають рядки та стовпці з пробілами між ними. Щоб визначити макет сітки, потрібно встановити значення властивості `display` на `grid`.
- **Одиниця `fr` (дріб)**: ця одиниця представляє частку простору всередині контейнера сітки. Ви можете використовувати одиницю `fr` для створення гнучких сіток.
- **Створення пробілів між треками**: існує три способи створення пробілів між треками в сітці. Ви можете використовувати властивість `column-gap` для створення пробілів між стовпцями. Ви можете використовувати властивість `row-gap` для створення пробілів між рядками. Або ви можете використовувати скорочену властивість `gap` для створення пробілів і між рядками, і між стовпцями.
- **Функція `repeat()`**: ця функція використовується для повторення секцій у списку треків. Замість того, щоб писати `grid-template-columns: 1fr 1fr 1fr;`, можна використати функцію `repeat()`.
- **Явна сітка**: ви можете вказати кількість ліній або треків за допомогою властивостей `grid-template-columns` або `grid-template-rows`.
- **Прихована сітка**: коли елементи розміщуються поза межами сітки, для цих зовнішніх елементів автоматично створюються рядки та стовпці.
- **Функція `minmax()`**: ця функція використовується для встановлення мінімальних і максимальних розмірів трека.
- **Автоматичне розміщення сітки**: браузер автоматично розміщує елементи, використовуючи алгоритм автоматичного розміщення. Властивість `grid-auto-flow` контролює цю поведінку, використовуючи значення, такі як `row`, `column` або `dense`, щоб впливати на напрямок розміщення та чи заповнювати порожні пробіли.
- **Розміщення на основі ліній**: усі сітки мають лінії. Щоб вказати, де елемент починається на лінії, можна використовувати властивості `grid-column-start` і `grid-row-start`. Щоб вказати, де елемент закінчується на лінії, можна використовувати властивості `grid-column-end` і `grid-row-end`. Також можна вибрати використання скорочених властивостей `grid-column` або `grid-row`.
- **`grid-template-areas`**: ця властивість використовується для надання назви елементам, які ви збираєтеся розміщувати на сітці.
- **Вирівнювання:** використовуйте `justify-items` і `align-items` для контролю вирівнювання елементів у межах їхніх областей сітки. Використовуйте `justify-self` і `align-self` на окремих елементах сітки, щоб замістити вирівнювання, встановлене на контейнері.

## Анімації в CSS

- **Визначення**: анімації дозволяють створювати динамічні, візуально привабливі ефекти на вебсторінках без потреби в JavaScript або складному програмуванні. Вони забезпечують спосіб плавно переходити елементам між різними стилями протягом заданого часу.
- **Правило `@keyframes`**: це правило визначає етапи та стилі анімації. Воно вказує, які стилі має мати елемент у різні моменти анімації.
- **Властивість `animation`**: це скорочена властивість, яка використовується для застосування анімацій.
- **`animation-name`**: ця властивість задає назву правила `@keyframes`, яку потрібно використовувати.
- **`animation-duration`**: ця властивість встановлює, скільки часу має тривати анімація.
- **`animation-timing-function`**: ця властивість визначає, як анімація змінюється з часом (наприклад, ease, linear, ease-in-out).
- **`animation-delay`**: ця властивість задає затримку перед початком анімації.
- **`animation-iteration-count`**: ця властивість встановлює, скільки разів анімація має повторюватися.
- **`animation-direction`**: ця властивість визначає, чи анімація має відтворюватися вперед, назад або чергуватися.
- **`animation-fill-mode`**: ця властивість задає, як елемент має бути стилізований до та після анімації.
- **`animation-play-state`**: ця властивість дозволяє призупиняти та відновлювати анімацію.

## Доступність і медіазапит `prefers-reduced-motion`

- **Медіазапит `prefers-reduced-motion`**: однією з основних проблем доступності анімацій є те, що вони можуть викликати дискомфорт або навіть фізичну шкоду деяким користувачам. Люди з вестибулярними розладами або чутливістю до руху можуть відчувати запаморочення, нудоту або головний біль при впливі певних типів руху на екрані. Медіазапит `prefers-reduced-motion` дозволяє веброзробникам визначити, чи користувач на системному рівні запросив мінімальні анімації або ефекти руху.
