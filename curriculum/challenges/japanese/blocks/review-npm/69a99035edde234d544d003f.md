## npmとは何ですか？

- **定義**：npmはオープンソースソフトウェアのオンラインレジストリで、コードの「アプリストア」のように機能します。
- **正式な意味**：npmは公式には「Node Package Manager」の略ではなく、「npm is not an acronym」という再帰的な逆略語です。
- **3つの関連部分**：

  - **ウェブサイト（npmjs.com）**：レジストリ内のパッケージを検索するためのものです。
  - **レジストリ**：すべてのパッケージを保存する大規模な公開データベースです。
  - **CLI（コマンドラインインターフェース）**：Node.jsと一緒にインストールされ、プロジェクトの依存関係を管理するツールです。

- **代替ツール**：npmが最も広く使われていますが、人気のある代替には**Yarn**、**PNPM**、**Bun**があります。

## package.jsonファイル

- **目的**：Node.jsプロジェクトのメタデータ、依存関係、スクリプトを保存する設定ファイルです。
- **主なフィールド**：

  - `name` と `version`：プロジェクトを識別します。
  - `main`：エントリーポイントのファイルを指定し、多くの場合`index.js`です。
  - `license`：法的条件を定義し、デフォルトは`ISC`です。
  - `dependencies`：本番環境でアプリを実行するために必要なパッケージです。
  - `devDependencies`：テストフレームワークなど、開発中のみ必要なツールです。

- **作成方法**：

  - `npm init`：対話形式のセットアッププロセスを開始します。
  - `npm init -y`：デフォルト値で`package.json`を即座にインストールします。

## セマンティックバージョニング（SemVer）

- **形式**：`MAJOR.MINOR.PATCH`。
- **ルール**：

  - **MAJOR**：既存のコードに影響を与える破壊的変更。
  - **MINOR**：後方互換性のある新機能。
  - **PATCH**：小さなバグ修正や改善。

- **範囲指定記号**：

  - `^`（キャレット）：マイナーおよびパッチの更新を許可（例：`^1.2.3`は`1.x.x`を許可）。
  - `~`（チルダ）：パッチの更新のみ許可（例：`~1.2.3`は`1.2.x`を許可）。
  - `*`（アスタリスク）：任意のバージョンにマッチし、主にテストで使用。

## 依存関係管理

- **インストール**：

  - `npm install <name>`：本番用依存関係を追加します。
  - `npm install <name> -D`：開発用依存関係を追加します。
  - `npm install <name>@<version>`：特定のバージョンをインストールします。

- **node_modulesとロックファイル**：

  - `node_modules/`：インストールされたパッケージの実際のコードを含みます。
  - `package-lock.json`：すべての依存関係と「子依存関係」の正確なバージョンを固定し、環境間の一貫性を保証します。

- **削除**：`npm uninstall <name>`はパッケージを削除し、`package.json`を更新します。
- **メンテナンス**：

  - `npm outdated`：新しいバージョンが利用可能なパッケージを特定します。
  - `npm update`：許可されたSemVer範囲内でパッケージを更新します。

## npmスクリプト

- **機能**：`scripts`セクションの`package.json`に定義されたカスタムコマンドで、タスクを自動化します。
- **実行方法**：`npm run <script_name>`で実行します。
- `start`、`test`、`stop`、`restart`の省略形があり（例：`npm start`）。
- **順次実行**：
- `&&`：最初のコマンドが成功した場合のみ2番目のコマンドを実行します。
- `;`：成功に関係なくコマンドを順番に実行します。
- **並列実行**：`&`演算子を使ってプロセスをバックグラウンドで実行します。
- **引数**：`--`区切りでスクリプトに引数を渡します（例：`npm start -- 8000`）。

## npmレジストリへの公開

- **レジストリアクセス**：公開および非公開パッケージのデータベースです。
- **公開手順**：

1. `npmjs.com`で無料アカウントを作成します。
2. ターミナルで`npm login`を使ってログインします。
3. `name`に一意の`version`と有効な`package.json`があることを確認します。
4. `README.md`と`.npmignore`（機密ファイルを除外するため）を作成します。
5. `npm publish`（スコープ付きパッケージの場合は`npm publish --access public`）を実行します。

- **スコープ**：名前の衝突を避けるために`@username/package-name`の下で公開するために使います。

## Node.jsのモジュールシステム

### CommonJS（CJS）

- **デフォルトシステム**：Node.jsがモジュールを扱う元々の方法で、同期的に読み込みます。
- **構文**：
- **インポート**：`const module = require('./path')`。
- **エクスポート**：`module.exports = { functionName }`。
- **識別方法**：`.cjs`ファイルや、`.js`で`type`が指定されていない`package.json`ファイルに使われます。

### ESモジュール（ESM）

- **標準化された形式**：最新のJavaScript標準で、効率のため非同期に読み込みます。
- **構文**：

  - **エクスポート**：`export { name }`または`export default function`。
  - **インポート**：`import { name } from './path.mjs'`。
  - **識別方法**：`.mjs`ファイルや、`.js`に`package.json`が含まれる`"type": "module"`ファイルに使われます。
