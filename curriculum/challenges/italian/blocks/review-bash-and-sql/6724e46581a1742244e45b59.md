## Normalizzazione del database

È il processo di organizzazione di un database relazionale per ridurre la ridondanza dei dati e migliorare l'integrità.

I suoi vantaggi includono:

- Minimizzare i dati duplicati, risparmiando spazio di archiviazione e riducendo le incoerenze.
- Garantire l'integrità dei dati tramite l'uso di chiavi primarie e chiavi esterne.
- Rendere i database più facili da mantenere e comprendere.

### Forme normali

- **1NF (prima forma normale)**
  - Ogni cella contiene un singolo valore (atomico).
  - Ogni record è unico (garantito da una chiave primaria).
  - L'ordine di righe/colonne è irrilevante.
  - Esempio: spostare più numeri di telefono da una tabella `students` in una tabella separata `student_phones`.

- **2NF (seconda forma normale)**
  - Soddisfa i requisiti della 1NF.
  - Nessuna **dipendenza parziale**: ogni attributo non chiave deve dipendere dall'intera chiave primaria composta.
  - Esempio: dividere la tabella `orders` in `order_header` e `order_items` per evitare attributi che dipendono solo da una parte della chiave.

- **3NF (terza forma normale)**
  - Soddisfa i requisiti della 2NF.
  - Nessuna **dipendenza transitiva**: gli attributi non chiave non possono dipendere da altri attributi non chiave.
  - Esempio: spostare `city_postal_code` in una tabella `cities` invece di conservarlo con ogni ordine.

- **BCNF (forma normale di Boyce-Codd)**
  - Soddisfa i requisiti della 3NF.
  - Ogni determinante (lato sinistro di una dipendenza funzionale) deve essere una superchiave.

**Consiglio**: punta alla 3NF nella maggior parte dei progetti per un buon equilibrio tra integrità e prestazioni.

## Concetti chiave di SQL

- SQL è un linguaggio di query strutturato per comunicare con database relazionali.
- **Comandi base** → `SELECT`, `INSERT`, `UPDATE`, `DELETE`, `CREATE TABLE`, `ALTER TABLE`, ecc.
- `Joins` → combina dati da più tabelle (`INNER JOIN`, `LEFT JOIN`, `RIGHT JOIN`, `FULL JOIN`).

## Eseguire comandi SQL in Bash

Puoi eseguire comandi SQL direttamente dalla riga di comando usando il client `psql` per PostgreSQL o strumenti simili per altri database.

Ad esempio, per eseguire un file SQL in PostgreSQL:

```bash
psql -U username -d database_name -c "SELECT * FROM students;"
```

Puoi anche eseguire comandi MySQL direttamente:

```bash
mysql -u username -p database_name -e "SELECT * FROM students;"
```

### Eseguire SQL da un file

```bash
# PostgreSQL
psql -U username -d database_name -f script.sql

# MySQL
mysql -u username -p database_name < script.sql
```

### Incorporare SQL in uno script Bash

```bash
#!/bin/bash
DB_USER="school_admin"
DB_NAME="school"

# Insert student data
psql -U "$DB_USER" -d "$DB_NAME" -c \
"INSERT INTO students (name, age, major) VALUES ('Alice', 20, 'CS');"
```

### Uso di variabili in SQL

```bash
#!/bin/bash
DB_USER="school_admin"
DB_NAME="school"
STUDENT_NAME="Bob"
AGE=21

psql -U "$DB_USER" -d "$DB_NAME" -c \
"INSERT INTO students (name, age) VALUES ('$STUDENT_NAME', $AGE);"
```

**Consiglio**: sanifica le variabili per evitare l'iniezione SQL.

## Recuperare e usare i risultati delle query SQL in Bash

Quando esegui query SQL tramite `psql`, puoi **catturare** e **processare** i valori restituiti nei tuoi script Bash.

### Catturare un singolo valore

```bash
#!/bin/bash
DB_USER="school_admin"
DB_NAME="school"

# Get total student count
STUDENT_COUNT=$(psql -U "$DB_USER" -d "$DB_NAME" -t -A -c \
"SELECT COUNT(*) FROM students;")

echo "Total students: $STUDENT_COUNT"
```

Output → 42

### Recuperare più colonne

```bash
#!/bin/bash
DB_USER="school_admin"
DB_NAME="school"

# Get top 3 students' names and ages
RESULTS=$(psql -U "$DB_USER" -d "$DB_NAME" -t -A -F"," -c \
"SELECT name, age FROM students LIMIT 3;")

echo "Top 3 students:"
echo "$RESULTS"
```

Output

```bash
Alice,20
Bob,21
Charlie,22
```

### Iterare sui risultati di una query

```bash
#!/bin/bash
DB_USER="school_admin"
DB_NAME="school"

# Get student names and majors
psql -U "$DB_USER" -d "$DB_NAME" -t -A -F"," -c \
"SELECT name, major FROM students;" | while IFS="," read -r name major
do
    echo "Student: $name | Major: $major"
done
```

Forma dell'output

```bash
Student: Alice | Major: CS
Student: Bob   | Major: Math
Student: Carol | Major: Physics
```

## Iniezione SQL

È una vulnerabilità di sicurezza web in cui gli attaccanti inseriscono codice SQL malevolo nei campi di input per manipolare il database.

Questo può portare ad azioni rischiose come:

- Bypassare l'autenticazione.
- Rubare dati sensibili.
- Modificare o cancellare record.

Un esempio di attacco di iniezione SQL:

```sql
SELECT * FROM users WHERE username = ' " " OR "1"="1" -- ' AND password = 'anything';
```

Questa query restituirebbe tutti gli utenti perché la condizione `OR "1"="1"` è sempre vera, permettendo agli attaccanti di bypassare i controlli di accesso.

### Prevenire l'iniezione SQL

1. **Usa istruzioni preparate**: separano il codice SQL dai dati, prevenendo l'iniezione. Ecco un esempio (Node.js con pg):

    ```sql
    client.query('SELECT * FROM users WHERE username = $1 AND password = $2', [username, password]);
    ```

2. **Validazione degli input**: sanifica e valida tutti gli input degli utenti per assicurarti che rispettino i formati attesi.

3. **Minimo privilegio**: usa account database con i permessi minimi necessari per l'applicazione.

**Nota**: non concedere mai diritti di amministratore agli account dell'applicazione.

## Problema N+1

Il problema N+1 si verifica quando un'applicazione esegue una query per recuperare una lista di elementi (N) e poi esegue una query aggiuntiva per ogni elemento per recuperare dati correlati, risultando in N+1 query.

**Perché è negativo**

- Ogni query aggiunge overhead di rete e di elaborazione.
- Molte query piccole sono più lente di una query ottimizzata unica.

### Esempio del pattern N+1

```sql
-- 1: Get list of orders
SELECT * FROM orders LIMIT 50;

-- N: For each order, get customer
SELECT * FROM customers WHERE customer_id = ...;
```

**Soluzione**: usa `JOINs` o altre operazioni basate su insiemi.

```sql
SELECT 
  orders.order_id,
  orders.product,
  orders.quantity,
  customers.customer_id,
  customers.name,
  customers.email,
  customers.address
FROM orders
JOIN customers 
  ON orders.customer_id = customers.customer_id
WHERE orders.order_id IN (SELECT order_id FROM orders LIMIT 50);
```

Cerca sempre opportunità per combinare dati correlati in un'unica query.
