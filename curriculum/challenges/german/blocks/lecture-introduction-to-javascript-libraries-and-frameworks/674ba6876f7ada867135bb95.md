# --description--

In früheren Lektionen haben Sie gelernt, wie man Importe und Exporte in JavaScript verwendet. In dieser Lektion schauen wir uns an, wie man Komponenten in React importiert und exportiert.

Ein Export macht eine Komponente für andere Dateien verfügbar. Ein Import erlaubt einer Datei, eine Komponente zu verwenden, die anderswo exportiert wurde. In React-Projekten werden Komponenten so wiederverwendet und kombiniert, um Benutzeroberflächen zu erstellen.

In dieser Lektion lernen Sie, wie Sie React-Komponenten mit Standardexporten und benannten Exporten exportieren und wie Sie sie dort importieren, wo sie benötigt werden.

In diesem Beispiel haben wir eine `Cat`-Komponente, die in einer Datei namens `Cat.jsx` liegt. Für die Dateiendung verwenden wir die `.jsx`-Dateiendung, weil diese Datei hauptsächlich mit JSX arbeitet.

Diese `Cat`-Komponente gibt eine JSX-Auszeichnung mit einem Titel und einem Bild einer Katze namens Mr. Whiskers zurück:

```jsx
function Cat() {
  return (
    <div className="card">
      <h2>Mr. Whiskers</h2>
      <img
        src="https://cdn.freecodecamp.org/curriculum/cat-photo-app/running-cats.jpg"
        alt="Tuxedo cats running on dirt ground."
      />
    </div>
  );
}
```

Wenn wir unsere `Cat`-Komponente in einer anderen Datei verwenden wollen, müssen wir sie zuerst so exportieren:

```jsx
function Cat() {
  return (
    <div className="card">
      <h2>Mr. Whiskers</h2>
      <img
        src="https://cdn.freecodecamp.org/curriculum/cat-photo-app/running-cats.jpg"
        alt="Tuxedo cats running on dirt ground."
      />
    </div>
  );
}

export default Cat;
```

Wir verwenden das Schlüsselwort `default`, weil dies der Standardexport aus dem Modul sein wird. Sie können die Komponente auch in derselben Zeile wie die Komponentendefinition exportieren, so:

```jsx
export default function Cat() {
  return (
    <div className="card">
      <h2>Mr. Whiskers</h2>
      <img
        src="https://cdn.freecodecamp.org/curriculum/cat-photo-app/running-cats.jpg"
        alt="Tuxedo cats running on dirt ground."
      />
    </div>
  );
}
```

Sie können Kindkomponenten in anderen übergeordneten Komponenten-Dateien importieren oder sie in der Root-Komponenten-Datei importieren. Für dieses Beispiel importieren wir die `Cat`-Komponente in der Root-Komponenten-Datei.

Jedes React-Projekt hat eine oberste Komponente, die typischerweise `App.jsx` genannt wird:

```jsx
export default function App() {
  return (
    // return component here
  );
}
```

Diese Datei befindet sich normalerweise im `src`-Verzeichnis Ihres Projekts. Mehr über gängige Projektstrukturen lernen Sie in einer zukünftigen Lektion.

Um die `Cat`-Komponente innerhalb der Root-`App`-Komponente zu verwenden, müssen Sie sie so importieren:

```jsx
import Cat from "./Cat";

export default function App() {
  return (
    // return component here
  );
}
```

Jetzt können Sie die `Cat`-Komponente innerhalb der `App`-Komponente so zurückgeben:

```jsx
import Cat from "./Cat";

export default function App() {
  return (
    <Cat />
  );
}
```

Dieser Ansatz funktioniert gut, wenn eine Datei für den Export einer einzelnen Hauptkomponente verantwortlich ist, wodurch die Importsyntax einfach und gut lesbar bleibt. Denken Sie daran, dass eine Datei nur einen Standardexport haben kann, was sie ideal für eine Datei macht, die hauptsächlich eine einzelne Komponente enthält.


Jetzt, da Sie wissen, wie man Standardexporte verwendet, schauen wir uns benannte Exporte an. Benannte Exporte erlauben einer Datei, mehrere Komponenten oder Funktionen zu teilen. Im Gegensatz zu Standardexporten müssen diese mit genau dem Namen importiert werden, mit dem sie exportiert wurden (es sei denn, Sie benennen sie mit dem `as`-Schlüsselwort um).

Das ist nützlich, wenn eine Datei als Bibliothek von Komponenten dient. Zum Beispiel hier eine Datei mit zwei Komponenten, Cat und Dog:

```jsx
// Animals.jsx
export function Cat() {
  return <h2>Mr. Whiskers</h2>;
}

export function Dog() {
  return <h2>Fido</h2>;
}
```

Wir verwenden das export-Schlüsselwort einzeln bei jeder Komponente. Um diese in einer anderen Datei zu verwenden, importieren wir sie mit geschweiften Klammern:


```jsx
import { Cat, Dog } from "./Animals";

export default function App() {
  return (
    <div>
      <Cat />
      <Dog />
    </div>
  );
}
```

Die `{ Cat, Dog }`-Syntax sagt JavaScript, die spezifischen benannten Exporte aus der `Animals.jsx`-Datei zu importieren. Die Namen innerhalb der geschweiften Klammern müssen genau mit den exportierten Namen übereinstimmen. Wenn Sie Namenskonflikte vermeiden möchten oder einen anderen Namen in der aktuellen Datei bevorzugen, können Sie eine Komponente beim Import mit dem as-Schlüsselwort umbenennen.

```jsx
import { Cat as Kitty } from "./Animals";

export default function App() {
  return <Kitty />;
}
```

Schließlich kann eine Datei einen Standardexport und mehrere benannte Exporte haben.

```jsx
// Animals.jsx
export default function Cat() {
  return <h2>Mr. Whiskers</h2>;
}

export function Dog() {
  return <h2>Fido</h2>;
}
```

Beim Import gemischter Exporte steht der Standardexport zuerst (ohne Klammern), gefolgt von den benannten Exporten (in Klammern):

```jsx
// App.jsx
import Cat, { Dog } from "./Animals";

export default function App() {
  return (
    <div>
      <Cat />
      <Dog />
    </div>
  );
}
```

In diesem Beispiel ist `Cat` der Standardexport, daher verwendet er keine geschweiften Klammern. `Dog` ist ein benannter Export, daher benötigt er sie.

Wenn Sie verstehen, wie Standard- und benannte Exporte funktionieren, können Sie React-Komponenten über mehrere Dateien organisieren und sie dort wiederverwenden, wo sie gebraucht werden. Wenn Sie größere Anwendungen erstellen, hilft die Wahl des richtigen Exportstils, Ihren Code klar und wartbar zu halten und erleichtert die Zusammenarbeit mit Komponenten, die andere erstellt haben.
