# --interactive--

Es gibt viele verschiedene Möglichkeiten, Ihre Daten in der Programmierung zu sortieren. In dieser Lektion konzentrieren wir uns auf die eingebaute `sort`-Methode in JavaScript.

Die `sort`-Methode wird verwendet, um die Elemente eines Arrays anzuordnen, und gibt eine Referenz auf das sortierte Array zurück. Es wird keine Kopie erstellt, da die Elemente an Ort und Stelle sortiert werden.

Hier ist die grundlegende Syntax für die `sort`-Methode:

```js
array.sort(compareFunction);
```

Der `compareFunction` ist ein optionaler Parameter, der eine Funktion angibt, die die Sortierreihenfolge definiert. Später sehen wir uns an, wie man eine Vergleichsfunktion beim Sortieren von Zahlen verwendet.

## Sortieren von Strings

In diesem ersten Beispiel haben wir ein Array von Strings in zufälliger Reihenfolge.

```js
const fruits = ["Banana", "Orange", "Apple", "Mango"];
```

Unser Ziel ist es, das Array alphabetisch zu sortieren. Das können wir erreichen, indem wir die `sort`-Methode auf dem `fruits`-Array aufrufen.

:::interactive_editor

```js
const fruits = ["Banana", "Orange", "Apple", "Mango"];
fruits.sort();

console.log(fruits); // ["Apple", "Banana", "Mango", "Orange"]
```

:::

Das Ergebnis wird ein alphabetisch sortiertes Array von Früchten sein, das mit der Frucht `Apple` beginnt.

## Sortieren von Zahlen

Im nächsten Beispiel wollen wir das folgende Array von Zahlen sortieren:

```js
const numbers = [414, 200, 5, 10, 3];
```

Wenn wir versuchen, die `sort`-Methode auf diesem `numbers`-Array zu verwenden, erhalten wir unerwartete Ergebnisse.

:::interactive_editor

```js
const numbers = [414, 200, 5, 10, 3];
numbers.sort();

console.log(numbers); // [10, 200, 3, 414, 5]
```

:::

Wir erwarteten das Ergebnis `[3, 5, 10, 200, 414]`, aber stattdessen erhielten wir `[10, 200, 3, 414, 5]`.

Das liegt daran, dass die `sort`-Methode die Elemente in Strings umwandelt und dann deren Folgen von UTF-16-Codeeinheiten vergleicht.

UTF-16-Codeeinheiten sind die numerischen Werte, die die Zeichen im String darstellen. Beispiele für UTF-16-Codeeinheiten sind die Zahlen `65`, `66` und `67`, die die Zeichen `A`, `B` und `C` darstellen.

Deshalb erscheint die Zahl `200` vor der Zahl `3` im Array, weil der String `200` vor dem String `3` kommt, wenn man ihre UTF-16-Codeeinheiten vergleicht.

Die Lösung für dieses Problem ist, der `sort`-Methode eine Vergleichsfunktion zu übergeben.

Hier ist ein Beispiel, wie man das `numbers`-Array mit einer Vergleichsfunktion sortiert:

:::interactive_editor

```js
const numbers = [414, 200, 5, 10, 3];

numbers.sort((a, b) => a - b);

console.log(numbers); // [3, 5, 10, 200, 414]
```

:::

Die Parameter `a` und `b` sind die zwei zu vergleichenden Elemente. Die Vergleichsfunktion sollte einen negativen Wert zurückgeben, wenn `a` vor `b` stehen soll, einen positiven Wert, wenn `a` nach `b` stehen soll, und null, wenn `a` und `b` gleich sind.

Der erste Vergleich ist zwischen den Zahlen `414` und `200`. Das Ergebnis von `414 - 200` ist `214`, was ein positiver Wert ist. Das bedeutet, dass `414` nach `200` im sortierten Array stehen soll.

Der nächste Vergleich ist zwischen den Zahlen `200` und `5`. Das Ergebnis von `200 - 5` ist `195`, was ein positiver Wert ist. Das bedeutet, dass `200` nach `5` im sortierten Array stehen soll.

Wir wiederholen diesen Vorgang für alle Elemente im Array, und das Ergebnis ist ein sortiertes Array von Zahlen.

## Umgang mit `undefined`-Werten und leeren Stellen

Es ist auch wichtig zu verstehen, wie die `sort`-Methode mit `undefined`-Werten und leeren Stellen in einem Array umgeht.

Wenn die `sort`-Methode auf einen `undefined`-Wert trifft, übergibt sie diesen Wert nicht an die Vergleichsfunktion. Stattdessen werden `undefined`-Werte automatisch ans Ende des Arrays verschoben.

Hier ist ein Beispiel, das zeigt, wie `sort` mit einem Array umgeht, das einen `undefined`-Wert enthält:

:::interactive_editor

```js
const fruits = ["Banana", undefined, "Apple", "Mango"];
fruits.sort();

console.log(fruits); // ["Apple", "Banana", "Mango", undefined]
```

:::

Beachten Sie, dass `undefined` am Ende des sortierten Arrays nach allen anderen Elementen platziert wird.

Ähnlich verhält es sich, wenn die `sort`-Methode auf eine leere Stelle in einem dünn besetzten Array trifft: Diese leere Stelle bleibt erhalten und wird ans Ende des Arrays verschoben. Sind sowohl `undefined`-Werte als auch leere Stellen vorhanden, kommen zuerst die `undefined`-Werte, gefolgt von den leeren Stellen ganz am Ende.

Hier ist ein Beispiel, das dieses Verhalten demonstriert:

:::interactive_editor

```js
const arr = [, undefined, "banana", "apple"];
arr.sort();

console.log(arr); // ["apple", "banana", undefined, empty]
```

:::

In diesem Beispiel werden die Strings `"apple"` und `"banana"` zuerst alphabetisch sortiert. Danach wird der `undefined`-Wert platziert. Schließlich bleibt die leere Stelle ganz am Ende des Arrays erhalten.

Obwohl es viele weitere Möglichkeiten gibt, Daten in der Programmierung zu sortieren, kann die `sort`-Methode in JavaScript in vielen Fällen nützlich und effizient sein, wenn Sie ein Array von Elementen sortieren müssen.
