# --description--

이번 수업에서는 객체와 배열에 타입을 추가하는 방법을 배우세요.

먼저 배열을 살펴보겠습니다.

다음은 프로그래밍 언어들의 배열 예시입니다:

```js
const programmingLanguages = ["Java", "C++", "Python"];
```

이 배열을 문자열 배열로 명시적으로 타입 지정하고 싶다면 `string[]`로 타입을 지정할 수 있습니다:

```ts
const programmingLanguages:string[] = ["Java", "C++", "Python"];
```

앞서 배운 것처럼, TypeScript는 다음과 같이 작성하면 문자열 배열 타입을 똑똑하게 추론합니다:

```js
const programmingLanguages = ["Java", "C++", "Python"];
```

하지만 명시적으로 타입을 지정하면 다음과 같이 배열에 다른 타입을 추가하려 할 때 도움이 됩니다:

```ts
// Type 'number' is not assignable to type 'string'.(2322)
const programmingLanguages:string[] = ["Java", 9, "Python"];
```

배열에 타입을 지정하는 또 다른 방법은 다음 구문을 사용하는 것입니다:

```ts
const programmingLanguages:Array<string> = ["Java", "C++", "Python"];
```

본질적으로 이 두 구문은 같으며, 둘 중 어떤 것을 선택할지는 보통 개인 취향의 문제입니다.

이제 객체 타입 지정 방법을 살펴보겠습니다.

다음은 객체 리터럴 예시입니다:

```js
const developer = {
  firstName: "Jessica",
  isEmployed: true
}
```

객체 속성을 명시적으로 타입 지정하려면 다음 구문을 사용하세요:

```ts
const developer: { firstName: string, isEmployed: boolean } = {
  firstName: "Jessica",
  isEmployed: true
}
```

객체에 속성을 추가하거나 제거하려 하면 다음과 같은 오류가 발생합니다:

```ts
// Property 'isEmployed' is missing in type '{ firstName: string; }' but required in type '{ firstName: string; isEmployed: boolean; }'
const developer: { firstName: string, isEmployed: boolean } = {
  firstName: "Jessica",
}
```

속성 이름 뒤에 `?`를 사용하면 객체 속성을 선택 사항으로 만들 수 있습니다:

```ts
const developer: { firstName: string, isEmployed?: boolean } = {
  firstName: "Jessica",
}
```

`isEmployed` 속성이 선택 사항으로 표시되어 TypeScript가 더 이상 오류를 표시하지 않습니다.

지금까지 모든 객체 예시는 제한적인 속성을 가졌습니다. 즉, 객체가 가질 수 있는 모든 속성을 명시적으로 나열했습니다:

```ts
const developer: { firstName: string, isEmployed: boolean } = {
  firstName: "Jessica",
  isEmployed: true
}
```

그런데 만약 미리 모든 속성 이름을 모른다면 어떨까요? 속성 수는 제한 없지만 모든 값이 특정 타입을 따라야 하는 객체를 원한다면요?

이럴 때 두 가지 방법이 있습니다. 첫 번째는 TypeScript 내장 제네릭 유틸리티 타입인 `Record`를 사용하는 것입니다.

기본 구문은 다음과 같습니다:

```ts
Record<Keys, ValueType>
```

첫 번째 타입은 키의 타입을 나타내고, 두 번째 타입은 값의 타입을 나타냅니다.

다음은 `Record`를 `userRoles` 객체에 사용하는 예시입니다:

```ts
const userRoles: Record<string, string> = {
  admin: "full-access",
  editor: "limited-access",
  viewer: "read-only"
};
```

이 `userRoles` 객체는 어떤 문자열 키도 가질 수 있으며, 모든 값은 문자열이어야 합니다. 문자열이 아닌 값을 할당하려 하면 오류가 발생합니다:

```ts
// Type 'number' is not assignable to type 'string'
const userRoles: Record<string, string> = {
  admin: 1
};
```

두 번째 방법은 인덱스 시그니처를 사용하는 것입니다:

```ts
const obj: { [key: string]: string } = {};
```

위 구문은 문자열 키를 가진 모든 속성은 문자열 값을 가져야 한다는 뜻입니다. 다음은 `settings` 객체 예시입니다:

```ts
const settings: { [key: string]: string } = {
  theme: "dark",
  language: "en",
  layout: "grid"
};
```

앞서와 마찬가지로 값 타입이 올바르지 않으면 TypeScript가 오류 메시지를 표시합니다:

```ts
// Type 'boolean' is not assignable to type 'string'
const settings: { [key: string]: string } = {
  darkMode: true
};
```

`Record`와 인덱스 시그니처는 본질적으로 같습니다. 따라서 둘 중 선택은 주로 개인 취향에 달려 있습니다.
