# --interactive--

Функції — це багаторазові частини коду, які виконують певне завдання або обчислюють значення. Уявіть функції як машину, яка приймає деякі вхідні дані, виконує над ними операції, а потім видає вихідні дані. Ось приклад оголошення функції:

```js
function greet() {
  console.log("Hello, Jessica!");
}
```

У цьому прикладі ми оголосили функцію з назвою `greet`. Всередині цієї функції є `console.log`, який виводить повідомлення `Hello, Jessica!`. Якщо ми спробуємо запустити цей код, ми не побачимо повідомлення в консолі. Це тому, що нам потрібно викликати функцію.

Виклик функції, або інвокація, — це коли ми фактично використовуємо або виконуємо функцію. Щоб викликати функцію, потрібно звернутися до її назви, після якої йде множина круглих дужок:

:::interactive_editor

```js
function greet() {
  console.log("Hello, Jessica!");
}

greet(); // "Hello, Jessica!"
```

:::

Тепер повідомлення `Hello, Jessica!` буде виведено в консоль. Але що, якщо ми хочемо, щоб повідомлення було `Hello, Nick!` або `Hello, Anna!`? Ми не хочемо писати нову функцію щоразу, коли вітаємо різного користувача. Замість цього ми можемо створити багаторазову функцію, яка використовує параметри та аргументи.

Параметри виступають як заповнювачі для значень, які будуть передані функції під час її виклику. Вони дозволяють функціям приймати вхідні дані і працювати з ними. Аргументи — це фактичні значення, які передаються функції під час виклику. Ось оновлена версія функції `greet`, яка використовує параметри та аргументи:

:::interactive_editor

```js
function greet(name) {
  console.log("Hello, " + name + "!");
}

greet("Alice"); // Hello, Alice!
greet("Nick"); // Hello, Nick!
```

:::

`name` слугує параметром, а рядки `Alice` і `Nick` — аргументами. Тепер у нас є багаторазова функція, яку можна використовувати десятки разів у коді з різними аргументами.

Коли функція завершує виконання, вона завжди повертає значення. За замовчуванням повернене значення буде `undefined`. Ось приклад:

:::interactive_editor

```js
function doSomething() {
  console.log("Doing something...");
}

let result = doSomething();
console.log(result); // undefined
```

:::

Якщо потрібно, щоб функція повертала конкретне значення, потрібно використати інструкцію `return`. Ось приклад використання інструкції `return` для повернення суми двох значень:

:::interactive_editor

```js
function calculateSum(num1, num2) {
  return num1 + num2;
}

console.log(calculateSum(3, 4)); // 7
```

:::

Часто ви будете використовувати інструкцію `return`, бо зможете використати це повернене значення пізніше у коді.

Поки що ми працювали з іменованими функціями, але ви також можете створити так звану анонімну функцію. Анонімна функція — це функція без імені, яку можна призначити змінній ось так:

:::interactive_editor

```js
const sum = function (num1, num2) {
  return num1 + num2;
};

console.log(sum(3, 4)); // 7
```

:::

У цьому прикладі у нас є змінна `const` з назвою `sum`, якій ми призначаємо анонімну функцію, що повертає суму `num1` і `num2`. Потім ми можемо викликати `sum` і передати числа `3` і `4`, щоб отримати результат `7`.

Функції підтримують параметри за замовчуванням, що дозволяє встановлювати стандартні значення для параметрів. Ці значення використовуються, якщо функцію викликають без аргументу для цього параметра. Ось приклад:

:::interactive_editor

```js
function greetings(name = "Guest") {
  console.log("Hello, " + name + "!");
}

greetings(); // Hello, Guest!
greetings("Anna"); // Hello, Anna!
```

:::

У цьому прикладі, якщо аргумент для `name` не надано, він за замовчуванням дорівнює `Guest`.

Підсумовуючи, функції дозволяють писати багаторазовий і організований код. Вони можуть приймати вхідні дані (параметри), виконувати дії і повертати вихідні дані.
