# --description--

Se non conosci `let`, dai un’occhiata <a href="/learn/javascript-algorithms-and-data-structures/basic-javascript/explore-differences-between-the-var-and-let-keywords" target="_blank" rel="noopener noreferrer nofollow">a questa sfida sulle differenze tra <code>let</code> e <code>var</code></a>.

Quando dichiari una variabile con la parola chiave `var`, viene dichiarata globalmente o localmente se dichiarata all’interno di una funzione.

La parola chiave `let` si comporta in modo simile, ma con alcune funzionalità aggiuntive. Quando dichiari una variabile con la parola chiave `let` all’interno di un blocco, istruzione o espressione, il suo ambito è limitato a quel blocco, istruzione o espressione.

Per esempio:

```js
var numArray = [];
for (var i = 0; i < 3; i++) {
  numArray.push(i);
}
console.log(numArray);
console.log(i);
```

Qui la console mostrerà i valori `[0, 1, 2]` e `3`.

Con la parola chiave `var`, `i` viene dichiarata globalmente. Quindi quando `i++` viene eseguito, aggiorna la variabile globale. Questo codice è simile al seguente:

```js
var numArray = [];
var i;
for (i = 0; i < 3; i++) {
  numArray.push(i);
}
console.log(numArray);
console.log(i);
```

Qui la console mostrerà i valori `[0, 1, 2]` e `3`.

Questo comportamento può causare problemi se crei una funzione e la archivi per un uso successivo all’interno di un loop `for` che usa la variabile `i`. Questo perché la funzione archiviata farà sempre riferimento al valore aggiornato della variabile globale `i`.

```js
var printNumTwo;
for (var i = 0; i < 3; i++) {
  if (i === 2) {
    printNumTwo = function() {
      return i;
    };
  }
}
console.log(printNumTwo());
```

Qui la console mostrerà il valore `3`.

Come puoi vedere, `printNumTwo()` stampa 3 e non 2. Questo perché il valore assegnato a `i` è stato aggiornato e `printNumTwo()` restituisce la variabile globale `i` e non il valore che `i` aveva quando la funzione è stata creata nel loop for. La parola chiave `let` non segue questo comportamento:

```js
let printNumTwo;
for (let i = 0; i < 3; i++) {
  if (i === 2) {
    printNumTwo = function() {
      return i;
    };
  }
}
console.log(printNumTwo());
console.log(i);
```

Qui la console mostrerà il valore `2` e un errore che `i is not defined`.

`i` non è definita perché non è stata dichiarata nell’ambito globale. È dichiarata solo all’interno dell’istruzione loop `for`. `printNumTwo()` ha restituito il valore corretto perché sono state create tre diverse variabili `i` con valori unici (0, 1 e 2) dalla parola chiave `let` all’interno dell’istruzione loop.

# --instructions--

Correggi il codice in modo che `i` dichiarata nell’istruzione `if` sia una variabile separata da `i` dichiarata nella prima riga della funzione. Assicurati di non usare la parola chiave `var` da nessuna parte nel tuo codice.

Questo esercizio è pensato per illustrare la differenza tra come le parole chiave `var` e `let` assegnano l’ambito alla variabile dichiarata. Quando programmi una funzione simile a quella usata in questo esercizio, spesso è meglio usare nomi di variabili diversi per evitare confusione.
