# --interactive--

Currying 是一種技術，我們將接受多個引數的函式（程式）轉換成一連串的函式（程式），每個函式（程式）只接受一個引數。

讓我們從一個基本範例開始。想像我們有一個將兩個數字相加的函式（程式）：

:::interactive_editor

```js
function add(a, b) {
  return a + b;
}

console.log(add(3, 4)); // 7
```

:::

這是一個接受兩個引數並傳回它們總和的函式。現在，讓我們看看如何對這個函式進行柯里化：

:::interactive_editor

```js
function curriedAdd(a) {
  return function(b) {
    return a + b;
  }
}

console.log(curriedAdd(3)(4)); // 7
```

:::

在這個柯里轉換的程式碼中，我們不是一次接受兩個引數，而是有一個函式先接受第一個引數並傳回另一個函式。這個傳回的函式接著接受第二個引數並執行加法。我們像 `curriedAdd(3)(4)` 這樣呼叫它，其中每一對小括弧代表一次函式呼叫。

但是為什麼我們會想要這麼做？ 

Currying 讓我們能輕鬆地創建一些特殊的函式（程式）。例如，我們可以創建一個函式，總是將五加到任何數字上：

:::interactive_editor

```js
function curriedAdd(a) {
  return function(b) {
    return a + b;
  }
}

const addFive = curriedAdd(5);
console.log(addFive(10)); // 15
console.log(addFive(20)); // 25
```

:::

這裡，`addFive` 是一個函式（程式），它隨時準備將五加到我們給它的任何數字。這是一個偏函式應用的簡單範例，我們將一定數量的引數固定到一個函式（程式），產生另一個接受較少引數的函式（程式）。

雖然我們的範例集中在具有兩個引數的函式（程式）上，但 currying 可以應用於具有任意數量引數的函式（程式）。

當你繼續在 JavaScript 的旅程中前進時，你很可能會發現許多情況下，currying 可以讓你的程式碼更乾淨且更具表達力。

## --text--

以下哪一項是 JavaScript 中柯里化函式（函式）的範例？

## --answers--

```js
function add(a, b, c) {
  return a + b + c;
}
```

### --feedback--

這是一個一次接受所有參數的普通函式（程式）。

---

```js
function add(a) {
  return function (b) {
    return function (c) {
      return a + b + c;
    };
  };
}
```

---

```js
function add(a, b, c) {
  return a() + b() + c();
}
```

### --feedback--

此函式一次接受所有參數並像呼叫函式一樣呼叫它們，這不符合柯里化函式的定義。

---

```js
function add(a) {
  function inner(b) {
    return a + b;
  }
}
```

### --feedback--

內部函式（程式）永遠不會被傳回，因此 `add(2)` 會傳回 `undefined`，而 `add(2)(3)` 會丟出錯誤。
