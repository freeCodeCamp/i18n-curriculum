# --description--

Наступний тип циклу, який ви вивчите, називається `do...while` циклом. Його називають `do...while` циклом, тому що він спочатку `do` один прохід коду всередині циклу незалежно від обставин, а потім продовжує виконувати цикл `while`, доки вказана умова не оціниться як `true`.

```js
const ourArray = [];
let i = 0;

do {
  ourArray.push(i);
  i++;
} while (i < 5);
```

Приклад вище поводиться подібно до інших типів циклів, і отриманий масив виглядатиме як `[0, 1, 2, 3, 4]`. Однак те, що відрізняє `do...while` від інших циклів, — це те, як він поводиться, коли умова не виконується при першій перевірці. Давайте подивимось це на практиці. Ось звичайний `while` цикл, який виконуватиме код у циклі, доки `i < 5`:

```js
const ourArray = []; 
let i = 5;

while (i < 5) {
  ourArray.push(i);
  i++;
}
```

У цьому прикладі ми ініціалізуємо значення `ourArray` як порожній масив і значення `i` як 5. Коли ми виконуємо `while` цикл, умова оцінюється як `false`, тому що `i` не менше за 5, тож ми не виконуємо код всередині циклу. В результаті `ourArray` залишиться без доданих значень і виглядатиме як `[]` після завершення виконання всього коду в прикладі вище. Тепер подивіться на `do...while` цикл:

```js
const ourArray = []; 
let i = 5;

do {
  ourArray.push(i);
  i++;
} while (i < 5);
```

У цьому випадку ми ініціалізуємо значення `i` як 5, так само, як і в `while` циклі. Коли ми доходимо до наступного рядка, умови для перевірки немає, тож ми переходимо до коду всередині фігурних дужок і виконуємо його. Ми додамо один елемент до масиву, а потім збільшимо `i` перед тим, як дійти до перевірки умови. Коли ми нарешті оцінюємо умову `i < 5` у останньому рядку, бачимо, що `i` тепер дорівнює 6, що не проходить перевірку умови, тож ми виходимо з циклу і завершуємо роботу. Наприкінці наведеного прикладу значення `ourArray` дорівнює `[5]`. По суті, `do...while` цикл гарантує, що код всередині циклу виконається принаймні один раз. Спробуємо змусити `do...while` цикл працювати, додаючи значення до масиву.
