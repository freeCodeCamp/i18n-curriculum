# --description--

In Express, i **gestori di rotta** (chiamati anche funzioni di callback di rotta o funzioni controller) sono funzioni che definiscono cosa succede quando un client invia una richiesta a una rotta specifica.

Un gestore di rotta elabora la richiesta in arrivo, esegue la logica necessaria come interrogare un database o fare calcoli, e poi invia una risposta al client.

Vediamo un esempio semplice (che in realtà stai già usando da sempre):

```javascript
app.get("/menu", (req, res) => {
  res.send("Welcome to the menu!");
});
```

Il metodo `.get()` è uno dei metodi di rotta di Express. Ascolta le richieste **GET** su una rotta specificata. In questo esempio, ascolta le richieste a `/menu`.

La funzione `(req, res) => { ... }` è il **gestore di rotta**. Viene eseguita ogni volta che un client invia una richiesta GET a `/menu`.

All’interno del gestore:

* `req` (request) è un oggetto che contiene informazioni sulla richiesta in arrivo, come parametri di rotta, stringhe di query, intestazioni e dati del corpo.
    
* `res` (response) è un oggetto usato per inviare una risposta al client.
    

In questo caso, `res.send("Welcome to the menu!")` invia una risposta in testo semplice al client.

Quando un utente naviga a `/menu` nel browser — o invia una richiesta GET a quella rotta — il server esegue questo gestore e risponde con il messaggio `Welcome to the menu!`.

I gestori di rotta sono il cuore di un’applicazione Express. Ti permettono di definire come le diverse rotte della tua applicazione web risponderanno alle richieste. Puoi anche aggiungere logica a questi gestori, come elaborare dati di moduli, interagire con database o eseguire validazioni.
