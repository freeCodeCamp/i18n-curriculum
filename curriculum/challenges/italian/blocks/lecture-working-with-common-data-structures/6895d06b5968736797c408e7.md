# --description--

In questa lezione, vedremo mappe, hash map e set. Ma prima di farlo, definiamo i tipi di dato astratti.

Un tipo di dato astratto (ADT) è una rappresentazione concettuale di un tipo di dato, che include quali operazioni possono essere eseguite sui dati e le proprietà di quei dati.

I tipi di dato astratti sono come progetti che descrivono **cosa** si può fare, non **come** farlo. Separano l'interfaccia dall'implementazione effettiva delle operazioni.

Una **mappa** è un ADT che gestisce collezioni di coppie chiave-valore e le loro operazioni in modo molto specifico ed efficiente.

In una mappa, ogni valore è associato a una chiave specifica.

Una delle caratteristiche principali delle mappe è che ogni chiave deve essere unica. Questa unicità permette ricerche dirette, rendendo il processo di recupero delle informazioni molto più efficiente.

Solo le chiavi devono essere uniche, i valori possono essere ripetuti.

Il tipo di dato astratto mappa definisce anche operazioni importanti, come inserire coppie chiave-valore, ottenere il valore associato a una chiave, aggiornare il valore associato a una chiave, rimuovere una coppia chiave-valore e verificare se una chiave esiste nella mappa.

Non specifica come queste operazioni devono essere eseguite, le elenca solo come parte delle operazioni disponibili del tipo di dato.

Una **hash map**, nota anche come tabella hash, è un’implementazione concreta del tipo di dato astratto mappa.

Le hash map usano una tecnica chiamata "hashing" per eseguire le operazioni comuni in modo molto efficiente.

L’hashing funziona generando un valore hash per ogni elemento usando una funzione hash.

Il valore hash viene generato in base alla chiave della coppia chiave-valore ed è usato per calcolare un indice in un array sottostante, la struttura dati effettiva dove sono archiviate le coppie chiave-valore.

Ma ti starai chiedendo: cosa succede se due chiavi producono lo stesso indice?

Le hash map risolvono queste collisioni con strategie intelligenti.

Una opzione è usare la strategia del "concatenamento", dove ogni indice dell’array punta a una lista concatenata (un’altra struttura dati), dove sono archiviati tutti gli elementi con lo stesso indice.

Un’altra strategia è usare "indirizzamento aperto", che consiste nel cercare il prossimo indice disponibile nell’array seguendo una sequenza di ricerca predefinita.

La complessità temporale media delle hash map è "tempo costante" `O(1)` per inserire, recuperare e cancellare coppie chiave-valore.

La complessità temporale nel caso peggiore di queste operazioni è tempo lineare `O(n)`, che si verifica quando ci sono molte collisioni hash, quindi la strategia di risoluzione delle collisioni deve essere applicata più volte.

La complessità spaziale per inserire in una hash map è costante `O(1)` nel caso medio, una quantità costante di memoria per archiviare la nuova coppia. Tuttavia, nel caso peggiore, può avere complessità spaziale lineare `O(n)` a causa di un’operazione di ridimensionamento dell’array sottostante. In generale, rimuovere un elemento ha complessità spaziale costante `O(1)`.

Questo trasforma la tabella hash in qualcosa di simile a una struttura dati lineare dove `n` elementi devono essere scansionati per trovare la chiave obiettivo. Tuttavia, questo è relativamente raro se la hash map è implementata correttamente.

I **dizionari** di Python sono implementati come hash map dietro le quinte.

Per creare un dizionario Python, devi solo scrivere le coppie chiave-valore tra parentesi graffe e separarle con una virgola. Ogni chiave deve essere separata dal valore corrispondente con due punti.

```python
my_dictionary = {
  'A': 1,
  'B': 2, 
  'C': 3
}
```

In questo codice, `'A'` è la chiave e `1` è il valore:

```python
'A': 1
```

In alternativa, puoi usare `dict()`:

```python
my_dictionary = dict(A=1, B=2, C=3)
```

Puoi ottenere il valore tramite la sua chiave corrispondente:

```python
my_dictionary['A']  # 1
```

Puoi anche aggiornare il valore associato a una chiave:

```python
my_dictionary['A'] = 4
```

E puoi rimuovere una coppia chiave-valore:

```python
del my_dictionary['A']  
```

Puoi anche verificare se una chiave è nel dizionario (o no):

```python
'C' in my_dictionary
```

E puoi chiamare questi metodi per ottenere rispettivamente le chiavi, i valori e gli elementi del dizionario.

```python
my_dictionary.keys()
my_dictionary.values()
my_dictionary.items()
```

Ottimo. Ora che sai di più su mappe e hash map, parliamo dei set.

I **set** sono collezioni non ordinate di elementi unici.

Scomponiamo questo concetto nei suoi componenti chiave:

* I set sono non ordinati. Gli elementi di un set non sono archiviati in un ordine specifico, quindi non puoi accedervi tramite indici.
    
* I set contengono solo elementi unici. Se provi ad aggiungere lo stesso valore due volte, verrà mantenuta solo una copia del valore.
    

Sono analoghi ai set in matematica e implementano le stesse operazioni di insieme, come intersezione, unione e differenza.

Uno dei principali vantaggi dei set è che garantiscono che gli elementi siano unici (senza duplicati). Per questo sono spesso usati per rimuovere duplicati da liste e altre strutture dati.

Sono anche dinamici. Possono adattarsi al numero di elementi attualmente archiviati. Questo li rende molto potenti.

La complessità temporale media per aggiungere, rimuovere, ottenere la lunghezza del set e verificare se un elemento è nel set è "tempo costante" `O(1)`, che è molto efficiente.

Poiché i set sono implementati come tabelle hash, la complessità temporale nel caso peggiore per aggiungere, rimuovere e verificare l’appartenenza è "tempo lineare" `O(n)`. Questo può accadere quando ci sono molte collisioni hash, trasformando la tabella hash in qualcosa di simile a una struttura dati lineare, dove `n` scansioni sono necessarie per trovare la chiave.

In termini di complessità spaziale, nel caso medio, inserire un elemento avrebbe complessità costante `O(1)`, con un nuovo elemento unico che richiede una quantità costante di memoria. Tuttavia, nel caso peggiore, potrebbe esserci un’operazione di ridimensionamento dell’array sottostante, che potrebbe richiedere complessità spaziale lineare `O(n)`. In generale, rimuovere un elemento richiederebbe complessità spaziale costante `O(1)`.

Python ha una struttura dati `set` integrata che usi per lavorare con i set nei tuoi programmi.

Dietro le quinte, i set Python sono implementati usando una tabella hash dove sono archiviate solo le chiavi, senza valori associati.

I set possono archiviare solo oggetti di tipi di dato immutabili perché i loro valori hash rimangono sempre gli stessi. Al contrario, i valori hash degli oggetti mutabili possono cambiare quando vengono modificati. Per questo non possono far parte dei set. Se il valore hash di un oggetto archiviato nel set cambia, il programma non sarebbe più in grado di trovarlo.

Per definire un set in Python, devi solo racchiudere gli elementi tra parentesi graffe e separarli con virgole:

```python
numbers = {1, 2, 3, 4}
```

Per creare un set vuoto, puoi chiamare `set()`:

```python
numbers = set()
```

Nota che se usi parentesi graffe vuote, questo creerà automaticamente un dizionario Python, non un set, quindi devi chiamare la funzione `set()` per creare un set vuoto.

Puoi aggiungere un elemento a un set con il metodo `.add()`:

```python
numbers.add(5)
```

Puoi anche rimuovere elementi dal set con il metodo `.remove()`:

```python
numbers.remove(5)
```

Questo lancerà un `KeyError` se l’elemento non viene trovato. Ma se non vuoi lanciare un errore in quel caso, puoi usare invece il metodo `.discard()`.

Il metodo `.pop()` restituisce un elemento arbitrario dal set, mentre il metodo `.clear()` rimuove tutti gli elementi dal set.

Puoi verificare se un elemento è in un set con l’operatore `in`:

```python
5 in numbers
```

Python supporta anche operazioni di insieme, inclusi unione, differenza, differenza simmetrica e intersezione, che puoi eseguire con questi metodi:

```python
set_a = {1, 2, 3, 4}
set_b = {2, 3, 4, 5, 6}

set_a.union(set_b)
set_a.intersection(set_b)
set_a.symmetric_difference(set_b)
set_a.difference(set_b)
```

O con i loro operatori equivalenti:

```python
set_a | set_b
set_a & set_b
set_a ^ set_b
set_a - set_b 
```

La complessità temporale media per aggiungere, rimuovere e testare l’appartenenza è "tempo costante" `O(1)`.

La complessità temporale nel caso peggiore per queste operazioni è "tempo lineare" `O(n)` a causa dello scenario peggiore di collisione nella hash map.

Puoi anche verificare se un set è un sottoinsieme o un sovrainsieme di un altro:

```python
set_a.issubset(set_b)
set_a.issuperset(set_b)
```

In generale, dovresti usare i set quando hai bisogno di archiviare una collezione di elementi unici e controllare frequentemente la presenza di un elemento.

Mappe, hash map e set sono potenti strutture dati progettate per organizzare e recuperare dati in modo efficiente. Ognuna di esse ha caratteristiche e casi d’uso unici. Come sviluppatore, dovrai scegliere la migliore per il tuo progetto.
