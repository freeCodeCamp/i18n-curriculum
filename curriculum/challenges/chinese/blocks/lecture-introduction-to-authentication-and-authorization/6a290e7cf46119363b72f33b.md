# --description--

在构建网页应用时，你的后端服务器不仅仅是发送数据，它还以 HTTP 响应头的形式向浏览器发送隐藏信息。

默认情况下，Express 的配置会暴露有关你的服务器技术的细节。攻击者可以利用这些默认头部查找你应用中的漏洞。这就是开发者使用 Helmet.js 的原因。

Helmet.js 是一个针对 Node.js 应用的安全性中间件集合。它通过自动设置、修改或移除特定的 HTTP 头部，充当保护盾牌。

通过管理这些头部，Helmet 帮助保护你的应用免受常见的网页漏洞、恶意脚本和数据嗅探攻击。

Helmet 不需要你手动配置数十个安全协议，而是开箱即用地配置了一套较小的中间件函数。安装只需一个终端命令：

```bash
npm install helmet
```

要将这层安全保护应用到整个 Express 应用，你只需一行代码：

```js
const express = require('express');
const helmet = require('helmet');

const app = express();

// apply Helmet security middleware
app.use(helmet());
```

将 `app.use(helmet())` 放在中间件栈的顶部，Helmet 会拦截每个发出的响应并安全地配置你的 HTTP 头部。

初始化 Helmet 后，它会自动处理几个关键的安全任务。以下是它设置的最重要配置：

**隐藏服务器技术（X-Powered-By）**：默认情况下，Express 会通过发送头部 `X-Powered-By: Express` 来自我宣传。这会告诉黑客你的后端运行的具体技术，使他们能够针对特定版本的错误进行攻击。Helmet 会完全移除这个头部，迫使攻击者盲猜你的底层设置。

**缓解跨站脚本攻击（Content-Security-Policy）**：内容安全策略（CSP）是防御跨站脚本（XSS）攻击最强有力的手段之一。Helmet 设定严格规则，定义浏览器允许从哪里加载脚本、图像和样式。这防止恶意注入者在你的用户浏览器中运行未授权的脚本。

**防止点击劫持（X-Frame-Options）**：点击劫持发生在恶意网站将你的网站嵌入一个不可见的 iframe 中，诱骗用户点击他们本不想点击的按钮。Helmet 将 `X-Frame-Options` 头部设置为 `SAMEORIGIN`，确保你的网站不能被外部不受信任的域嵌入。

**强制安全连接（Strict-Transport-Security）**：Helmet 迫使浏览器仅通过安全的 HTTPS 连接与你的服务器通信，而非不安全的 HTTP。这保护数据不被同一网络上的攻击者截获。

总之，Helmet 为生产环境应用提供了重要的防护层。虽然它不会修复你自定义代码中的错误，但它能立即强化你的服务器配置，抵御自动扫描器和常见的利用技术。
