# --interactive--

Das `script`-Element wird verwendet, um ausführbaren Code einzubetten. Die meisten Entwickler nutzen es, um JavaScript-Code auszuführen. JavaScript wird verwendet, um Interaktivität zu Ihren Webseiten hinzuzufügen. Häufige Beispiele für die Verwendung von JavaScript sind interaktive Spiele, Bild-Slider und dynamische Formulare, die Benutzereingaben in Echtzeit validieren.

Hier ist ein Beispiel für die Verwendung des `script`-Elements in einem HTML-Dokument. Entfernen Sie das `//` vor dem `alert("Welcome to freeCodeCamp");`, und es sollte ein Alert-Fenster erscheinen. Um die Vorschauen zu sehen, müssen Sie den interaktiven Editor aktivieren.

:::interactive_editor

```html
<body>
  <script>
    // alert("Welcome to freeCodeCamp");
  </script>
</body>
```

:::

Obwohl Sie technisch gesehen Ihren gesamten JavaScript-Code innerhalb der `script`-Tags schreiben können, gilt es als bewährte Vorgehensweise, stattdessen eine externe JavaScript-Datei zu verknüpfen. Hier ist ein Beispiel für die Verwendung des `script`-Elements, um eine externe JavaScript-Datei zu verknüpfen:

```html
<script src="path-to-javascript-file.js"></script>
```

Das `src`-Attribut wird hier verwendet, um den Speicherort dieser externen JavaScript-Datei anzugeben. `src` steht für „source“. Der Grund, warum es nicht empfohlen wird, den gesamten JavaScript-Code im HTML-Dokument zu platzieren, ist die Trennung der Anliegen. Die Trennung der Anliegen ist ein Gestaltungsprinzip, bei dem Sie Ihre Programme in verschiedene Abschnitte unterteilen und jeder Abschnitt ein separates Anliegen behandelt. In diesem Fall möchten wir unseren JavaScript-Code vom HTML-Code trennen.
