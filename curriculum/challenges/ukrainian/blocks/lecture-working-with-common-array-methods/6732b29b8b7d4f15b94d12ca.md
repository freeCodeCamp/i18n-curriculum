# --interactive--

Поверхнева копія масиву — це новий масив, який містить ті ж елементи, що й оригінал. Якщо масив містить лише примітивні значення, як-от числа або рядки, новий масив є повністю окремим. Але якщо масив містить інші масиви всередині, і оригінал, і копія посилаються на ті самі внутрішні масиви. Це означає, що якщо ви зміните щось у спільному внутрішньому масиві, ця зміна буде видна в обох масивах.

Поверхневі копії корисні, коли потрібно змінити структуру верхнього рівня, наприклад додати, видалити або змінити порядок елементів, не змінюючи оригінальний масив або внутрішні масиви.

Існує кілька методів створення поверхневих копій масивів, і ми розглянемо деякі з найпоширеніших: `concat()`, `slice()` та оператор розширення.

Почнемо з методу `concat()`. Цей метод створює новий масив шляхом об’єднання двох або більше масивів. Якщо його застосувати до одного масиву, він фактично створює поверхневу копію. Ось приклад:

:::interactive_editor

```js
const originalArray = [1, 2, 3];
const copyArray = [].concat(originalArray);

console.log(copyArray); // [1, 2, 3]
console.log(copyArray === originalArray); // false
```

:::

У цьому прикладі ми використовуємо метод `concat()` для конкатенації порожнього масиву з `originalArray`. Це створить новий масив, який є поверхневою копією `originalArray`.

`copyArray` містить ті ж елементи, що й `originalArray`, але це інший об’єкт масиву, тому перевірка суворої рівності (`===`) повертає `false`.

Інший метод створення поверхневої копії — це метод `slice()`. Викликаний без аргументів, `slice()` повертає поверхневу копію всього масиву. Ось як це працює:

:::interactive_editor

```js
const originalArray = [1, 2, 3];
const copyArray = originalArray.slice();

console.log(copyArray); // [1, 2, 3]
console.log(copyArray === originalArray); // false
```

:::

У цьому випадку `originalArray.slice()` створює новий масив, який є поверхневою копією `originalArray`. Знову ж таки, `copyArray` містить ті ж елементи, але це інший об’єкт масиву.

Оператор розширення (`...`), представлений у ES6, надає ще один лаконічний спосіб створення поверхневих копій масивів. Ось приклад:

:::interactive_editor

```js
const originalArray = [1, 2, 3];
const copyArray = [...originalArray];

console.log(copyArray); // [1, 2, 3]
console.log(copyArray === originalArray); // false
```

:::

Оператор розширення (`...`) розгортає елементи `originalArray` у новий масив, фактично створюючи поверхневу копію. Важливо зазначити, що всі ці методи створюють нові об’єкти масивів, що означає, що ви можете змінювати копію, не впливаючи на оригінальний масив. Наприклад:

:::interactive_editor

```js
const originalArray = [1, 2, 3];
const copyArray = [...originalArray];

copyArray.push(4);
console.log(originalArray); // [1, 2, 3]
console.log(copyArray);     // [1, 2, 3, 4]
```

:::

У цьому прикладі додавання елемента до `copyArray` не впливає на `originalArray`.

Підсумовуючи, поверхневі копії масивів можна легко створити за допомогою методів `concat()`, `slice()` або оператора розширення. Ці методи корисні для створення нових масивів, які можна змінювати незалежно від оригіналу.
