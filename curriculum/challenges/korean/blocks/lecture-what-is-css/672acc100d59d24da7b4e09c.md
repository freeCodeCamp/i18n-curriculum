# --interactive--

CSS 결합자는 CSS에서 선택자 간의 관계를 정의하는 데 사용됩니다. 이들은 다른 요소와의 관계를 기반으로 요소를 선택하는 데 도움을 주어 더 정확하고 효율적인 스타일링을 가능하게 합니다.

먼저 자손 결합자를 시작으로 주요 CSS 결합자와 그 사용 사례를 살펴보겠습니다.

자손 결합자는 첫 번째 선택자와 일치하는 조상 요소 안에 중첩된 두 번째 선택자와 일치하는 요소를 대상으로 지정하는 데 사용됩니다. 조상은 부모 요소나 부모의 부모일 수 있습니다.

이 작동 방식을 더 잘 이해하려면 예제를 살펴보겠습니다.

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<figure>
  <img src="https://cdn.freecodecamp.org/curriculum/cat-photo-app/relaxing-cat.jpg" alt="Relaxing Cat">
  <figcaption>A relaxing cat with a border</figcaption>
</figure>
```

```css
figure img {
  border: 4px solid red;
}
```

:::

위 예제에서는 자손 결합자를 사용해 `figure` 요소 안의 모든 이미지 요소를 선택하고 네 면 모두에 `solid red` 테두리를 적용했습니다.

부모 선택자와 자식 선택자 사이에 `space`가 사용된 점에 주목하세요.

이 경우 `figure`가 부모이고 `img`가 자식입니다.

만약 `figure` 요소 안에 여러 이미지가 있다면, 자손 결합자를 사용해 각 이미지 주위에 검은색 실선 테두리를 적용하는 것이 좋습니다.

다른 유형의 결합자는 자식 결합자입니다.

CSS에서 자식 결합자(`>`)는 지정된 부모 요소의 직접 자식인 요소를 선택하는 데 사용됩니다.

이 결합자는 특정 부모를 가진 요소만 대상으로 하여 CSS 규칙을 더 정확하게 만들고 더 깊이 중첩된 요소에 의도치 않은 스타일이 적용되는 것을 방지합니다.

다음 HTML 예제를 살펴보겠습니다:

:::interactive_editor

```html
<div class="container">
  <p>First</p>
  <div>
    <p>Second</p>
  </div>
  <div>
    <p>Third</p>
  </div>
</div>
```

:::

위 HTML 구조에서 `container` 클래스가 `div` 요소에 적용되어 있습니다.

이 컨테이너 안에는 직접 자식인 `p` 요소("First")가 있고, 그 뒤로 각각 `div` 요소("Second"와 "Third")를 포함하는 두 개의 추가 `p` 요소가 있습니다.

첫 번째 `p` 요소는 `.container` 요소의 직접 자식인 반면, 나머지 두 `p` 요소는 다른 `div` 요소 안에 중첩되어 있어 더 깊은 자손 요소입니다.

`container` 클래스의 직접 자식에게만 스타일을 적용하려면 다음과 같이 자식 결합자를 사용할 수 있습니다:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<div class="container">
  <p>First</p>
  <div>
    <p>Second</p>
  </div>
  <div>
    <p>Third</p>
  </div>
</div>
```

```css
.container > p {
  color: blue;
}
```

:::

위 예제에서는 `container` 클래스의 직접 자식만 대상으로 지정했습니다. 이로 인해 직접 자식은 `blue` 색상의 텍스트를 갖게 됩니다.

다른 두 단락 요소는 `div` 요소 안에 중첩되어 있으므로 `container` 클래스의 직접 자식으로 간주되지 않아 파란색 텍스트 색상이 적용되지 않습니다.

또 다른 일반적인 결합자는 바로 다음 형제 결합자입니다.

CSS에서 바로 다음 형제 결합자(`+`)는 지정된 형제 요소 바로 다음에 오는 요소를 선택합니다. 이 결합자는 한 요소 바로 다음에 오는 요소에 스타일을 적용할 때 유용하며, 형제 요소 간 위치에 따라 타깃팅된 스타일링을 가능하게 합니다.

다음 HTML 예제를 살펴보겠습니다:

:::interactive_editor

```html
<figure>
  <img
    src="https://cdn.freecodecamp.org/curriculum/cat-photo-app/relaxing-cat.jpg"
    alt="A cute orange cat lying on its back."
  />
  <figcaption>A cute orange cat lying on its back.</figcaption>
</figure>
```

:::

여기서는 `figure` 요소 안에 `img` 요소가 있고, 그 뒤에 `figcaption` 요소가 있습니다. `figcaption` 요소는 `img` 요소의 바로 다음 형제입니다.

`figcaption` 요소 주위에 검은색 테두리를 적용하려면 다음과 같이 바로 다음 형제 결합자를 사용할 수 있습니다:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<figure>
  <img
    src="https://cdn.freecodecamp.org/curriculum/cat-photo-app/relaxing-cat.jpg"
    alt="A cute orange cat lying on its back."
  />
  <figcaption>A cute orange cat lying on its back.</figcaption>
</figure>
```

```css
img + figcaption {
  border: 4px solid black;
}
```

:::

이 예제에서 바로 다음 형제 결합자(`+`)는 `figcaption` 요소 바로 다음에 오는 `img` 요소를 선택합니다. 적용된 CSS 규칙은 `4px solid black border` 주위에 `figcaption`를 추가합니다.

또 다른 일반적인 결합자는 이후 형제 결합자입니다.

CSS에서 이후 형제 결합자(`~`)는 지정된 요소 다음에 오는 모든 형제 요소를 선택합니다.

바로 다음 형제 결합자가 즉시 다음 형제만 대상으로 하는 것과 달리, 이후 형제 결합자(`~`)는 지정된 요소 다음에 오는 모든 형제를 대상으로 하여 스타일링에 더 큰 유연성을 제공합니다.

다음 HTML 예제를 살펴보겠습니다:

:::interactive_editor

```html
<div class="container">
  <h2>Subheading</h2>
  <p>First paragraph.</p>
  <p>Second paragraph.</p>
  <p>Third paragraph.</p>
  <p>Another paragraph element</p>
</div>
```

:::

이 HTML 구조에서는 `h2` 요소 다음에 네 개의 단락 요소가 있습니다. 단락 요소들은 `h2` 요소의 형제입니다.

`h2` 요소 다음에 오는 모든 단락 요소에 스타일을 적용하려면 이후 형제 결합자를 다음과 같이 사용할 수 있습니다:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<div class="container">
  <h2>Subheading</h2>
  <p>First paragraph.</p>
  <p>Second paragraph.</p>
  <p>Third paragraph.</p>
  <p>Another paragraph element</p>
</div>
```

```css
h2 ~ p {
  color: green;
}
```

:::

이 예제에서 `h2` 요소 다음에 오는 모든 단락 요소는 녹색 텍스트 색상을 갖게 됩니다.

이후 형제 결합자(`~`)는 `h2` 요소 다음에 나타나는 모든 단락 형제를 대상으로 하며, 즉시 다음 형제인지 여부는 상관하지 않습니다.

결론적으로, 다양한 CSS 결합자를 이해하고 사용하면 HTML 요소에 정확한 스타일을 적용할 수 있어 디자인의 제어력과 유연성을 높일 수 있습니다.

이 선택자들을 숙달하면 웹 페이지의 외관과 기능을 모두 향상시키는 더 정교하고 타깃팅된 스타일 규칙을 만들 수 있습니다.
