# --interactive--

In JavaScript ist ein Objekt eine grundlegende Datenstruktur, mit der Sie zusammengehörige Daten und Funktionen speichern und organisieren können.

Sie können sich ein Objekt wie einen Container vorstellen, der verschiedene Informationen enthält, ähnlich wie ein Aktenschrank verschiedene Ordner und Dokumente aufbewahrt.

Diese Informationen nennt man Eigenschaften, und sie bestehen aus einem Namen (oder Schlüssel) und einem Wert.

```js
const exampleObject = {
  propertyName: value,
}
```

Objekte sind unglaublich vielseitig und bilden das Rückgrat von JavaScript. Tatsächlich ist fast alles in JavaScript ein Objekt oder kann wie eines behandelt werden. Dazu gehören Arrays, Funktionen und sogar primitive Datentypen wie Strings und Zahlen, wenn sie auf bestimmte Weise verwendet werden.

Diese objektorientierte Natur von JavaScript ist einer der Gründe, warum es eine so flexible und leistungsstarke Sprache ist. Schauen wir uns an, wie Sie ein Objekt erstellen können:

```js
const person = {
  name: "Alice",
  age: 30,
  city: "New York"
};
```

In diesem Beispiel haben wir ein Objekt namens `person` mit drei Eigenschaften erstellt: `name`, `age` und `city`. Jede Eigenschaft hat einen Namen und einen Wert, getrennt durch einen Doppelpunkt.

Nun wollen wir erkunden, wie Sie auf diese Eigenschaften zugreifen können. Es gibt zwei Hauptmethoden, um auf Objekteigenschaften in JavaScript zuzugreifen: Punktnotation und Klammernotation.

Die Punktnotation ist die gebräuchlichste und einfachste Methode, um auf Objekteigenschaften zuzugreifen. Hier ist die grundlegende Syntax der Punktnotation:

```js
objectName.propertyName
```

So würden Sie die Punktnotation mit unserem `person`-Objekt verwenden:

:::interactive_editor

```js
const person = {
  name: "Alice",
  age: 30,
  city: "New York"
};

console.log(person.name);  // Alice
console.log(person.age);   // 30
```

:::

Die Punktnotation ist kurz und leicht lesbar, weshalb sie bevorzugt wird, wenn Sie den genauen Namen der Eigenschaft kennen, auf die Sie zugreifen möchten, und dieser Name ein gültiger JavaScript-Bezeichner ist (das heißt, er beginnt nicht mit einer Zahl und enthält keine Sonderzeichen oder Leerzeichen).

Die Klammernotation hingegen ermöglicht es Ihnen, auf Objekteigenschaften mit einem String innerhalb von eckigen Klammern zuzugreifen. So verwenden Sie die Klammernotation:

:::interactive_editor

```js
const person = {
  name: "Alice",
  age: 30,
  city: "New York"
};

console.log(person["name"]); // Alice
console.log(person["age"]); //  30
```

:::

Die Klammernotation ist flexibler als die Punktnotation, weil sie Eigenschaftsnamen erlaubt, die keine gültigen JavaScript-Bezeichner sind. Zum Beispiel, wenn Sie einen Eigenschaftsnamen mit Leerzeichen oder einen, der mit einer Zahl beginnt, haben, müssen Sie die Klammernotation verwenden:

:::interactive_editor

```js
const oddObject = {
  "1stProperty": "Hello",
  "property with spaces": "World"
};

console.log(oddObject["1stProperty"]);  // Hello
console.log(oddObject["property with spaces"]);  // World
```

:::

Ein weiterer Vorteil der Klammernotation ist, dass Sie Variablen verwenden können, um Eigenschaften dynamisch zuzugreifen:

:::interactive_editor

```js
const person = {
  name: "Alice",
  age: 30,
  city: "Wonderland"
};

let propertyName = "city";
console.log(person[propertyName]); // Wonderland
```

:::

Diese Flexibilität macht die Klammernotation besonders nützlich, wenn Sie den genauen Eigenschaftsnamen zum Zeitpunkt des Schreibens des Codes nicht kennen oder wenn Sie mit Eigenschaftsnamen arbeiten, die von Benutzereingaben oder einer anderen dynamischen Quelle stammen.

Es ist erwähnenswert, dass Objekte in JavaScript unglaublich mächtig und vielseitig sind. Sie können nicht nur einfache Werte wie Strings und Zahlen enthalten, sondern auch Arrays oder andere Objekte.

Das Verständnis von Objekten und wie man mit ihnen arbeitet, ist in JavaScript entscheidend, weil sie in der gesamten Sprache und in vielen JavaScript-Bibliotheken und Frameworks intensiv verwendet werden.

Während Sie weiter JavaScript lernen und damit arbeiten, werden Sie feststellen, dass das Beherrschen von Objekten eine Welt voller Möglichkeiten eröffnet, um komplexe und leistungsstarke Anwendungen zu erstellen.
