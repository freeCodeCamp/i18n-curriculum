# --description--

의존성을 최신 상태로 유지하고 특정 버전을 설치하는 방법을 아는 것은 프로젝트를 안정적으로 유지하는 데 중요합니다. 이렇게 하면 예상치 못한 버그를 피할 수 있고 새 버전이 출시될 때 더 많은 제어권을 가질 수 있습니다.

패키지의 특정 버전을 설치하려면 다음 형식을 사용하세요:

```bash
npm install package@version-number
```

예를 들어, 프로젝트에 Express의 이전 버전인 `4.21.2`가 필요하다면 이렇게 설치할 수 있습니다:

```bash
npm install express@4.21.2
```

버전 번호를 포함하지 않으면 npm이 자동으로 사용 가능한 최신 버전을 설치합니다.

어떤 버전이 있는지 확실하지 않다면 다음 명령어로 확인할 수 있습니다:

```bash
npm view package-name versions
```

예를 들면:

```bash
npm view express versions
```

패키지를 업데이트할 때는 개별적으로 하거나 한꺼번에 할 수 있습니다. 그 전에 어떤 패키지가 오래되었는지 먼저 확인하는 것이 도움이 됩니다:

```bash
npm outdated
```

이 명령어는 어떤 의존성에 새 버전이 있는지 보여주며, 다음과 비슷한 출력이 나타납니다:

```bash
user@Kolade ~/Desktop/fCC/script-code/node/npm/install-spec-version  % npm outdated
Package  Current  Wanted  Latest  Location              Depended by
chalk      5.5.0   5.5.0   5.6.2  node_modules/chalk    install-spec-version
cleave     1.0.1   1.0.1   1.0.2  node_modules/cleave   install-spec-version
express   4.21.2  4.21.2   5.2.1  node_modules/express  install-spec-version
```

오래된 의존성을 확인한 후 업데이트를 시작할 수 있습니다.

단일 의존성을 업데이트하려면 다음을 사용하세요:

```bash
npm update package-name
```

이 명령어는 `package.json` 파일에 허용된 버전 범위 내에서만 패키지를 업데이트합니다. 예를 들어 Express를 업데이트하려면 다음을 사용할 수 있습니다:

```bash
npm update express
```

버전 범위를 무시하고 패키지의 최신 버전을 설치하려면 다음 명령어를 사용하세요:

```bash
npm install package-name@latest
```

허용된 버전 범위 내의 모든 의존성을 업데이트하려면 다음을 실행하세요:

```bash
npm update
```

하지만 `npm update`는 `package.json`에 허용되지 않은 경우 패키지를 새 메이저 버전으로 업그레이드하지 않습니다. 메이저 버전은 종종 호환성에 영향을 주는 변경사항을 포함하므로 npm이 자동으로 적용하지 않습니다.

최신 메이저 버전을 명시적으로 원한다면 다음을 실행해야 합니다:

```bash
npm install package-name@latest
```

이 명령어는 `package.json`에 어떤 버전 범위가 있든 상관없이 npm이 가장 최근 릴리스를 설치하도록 강제합니다.
