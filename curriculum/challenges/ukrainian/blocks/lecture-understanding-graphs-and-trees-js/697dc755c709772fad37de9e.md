# --description--

Графи — це дуже потужні структури даних, які складаються з множини вузлів, також відомих як вершини, і ребер, що їх з’єднують.

Існує два загальні способи реалізації графів у вашому коді:

* матриці суміжності
    
* списки суміжності
    

Давайте розглянемо їх детальніше, а також їхні переваги та обмеження.

## Матриці суміжності

Почнемо з матриць суміжності.

Матриця суміжності — це двовимірний масив, у якому рядки та стовпці представляють вершини графа.

Значення в матриці представляють ребра або зв’язки між вузлами.

Наприклад, якщо у вас є матриця, збережена у змінній `matrix`, значення, збережене в `matrix[i][j]`, де `i` — це рядок, а `j` — стовпець, представляє ребро або зв’язок між вузлом `i` і вузлом `j`.

Значення можуть мати різне значення залежно від того, чи є граф зваженим чи незваженим:

* Якщо граф незважений, значення `1` означає, що існує ребро, яке з’єднує ці вузли, тоді як значення `0` означає, що між ними немає ребра.
    
* Якщо граф зважений, значення представлятиме вагу ребра, що з’єднує вузли.

Однією з великих переваг використання матриці суміжності є те, що перевірка наявності ребра між двома вузлами займає сталий час `O(1)`. Це тому, що програмі потрібно лише знайти конкретне значення в двовимірному масиві.

Однак ця ефективність у пошуку ребер має свою ціну. Матриці суміжності потребують великого квадратичного обсягу пам’яті `O(V²)`, де `V` — це кількість вузлів у графі.

Це неефективно для розріджених графів, тобто графів, які мають лише кілька ребер. Чому? Тому що якщо граф розріджений, ви зберігатимете багато нулів у матриці, щоб представити відсутність ребер між вузлами, і ці нулі все одно займатимуть місце в пам’яті.

Матриці суміжності також неефективні для пошуку сусідів вузла, оскільки програмі доводиться ітерувати весь рядок або стовпець, щоб знайти нулі та одиниці, які представляють ребра. У найгіршому випадку цей процес може зайняти `O(V)` часу, де `V` — це кількість вузлів у графі.

Давайте подивимося приклад матриці суміжності для цього конкретного графа:

<img src="https://cdn.freecodecamp.org/curriculum/lecture-transcripts/how-do-matrices-and-adjacency-lists-work-1.png" alt="Граф із чотирма вузлами, позначеними A, B, C і D. Вузол A з’єднується з B, C і D. Вузол B з’єднується з A і D. Вузол C з’єднується з A. Вузол D з’єднується з A і B.">

У матриці суміжності:

* Кожен рядок представляє вузол. Перший рядок представляє вузол A, другий рядок — вузол B і так далі.
    
* Кожен стовпець також представляє вузол.
    
* Кожне значення в матриці показує, чи існує ребро між парою вузлів. Значення 0 означає, що ребра між цими вузлами немає, а значення 1 означає, що ребро є.
    
Значення на діагоналі показують, чи має кожен вузол петлю на самому собі — ребро, що з’єднує вузол із самим собою. У нашому прикладі всі вони 0, бо граф не має таких петель.

Ось візуальне представлення матриці суміжності, щоб показати, як рядки та стовпці відповідають відповідним вузлам.

Наприклад, перший рядок — це `[0, 1, 1, 1]`, бо вузол A має ребра, що з’єднують його з вузлами B, C і D:

```markdown
#      A  B  C  D
# A   [0, 1, 1, 1],
# B   [1, 0, 0, 1],
# C   [1, 0, 0, 0],
# D   [1, 1, 0, 0]
```

А це та сама матриця суміжності, але реалізована у коді JavaScript:

```javascript
const adjacencyMatrix = [
  [0, 1, 1, 1], // The neighbors of A are B, C, and D
  [1, 0, 0, 1], // The neighbors of B are A and D
  [1, 0, 0, 0], // The only neighbor of C is A
  [1, 1, 0, 0]  // The neighbors of D are A and B
];
```

## Списки суміжності

Інший загальний спосіб представлення графів — використання списків суміжності.

Список суміжності — це масив або об’єкт (або `Map`), який зберігає всіх сусідів кожного вузла.

Існує два загальні способи реалізації списків суміжності:

* Як масив, де кожен індекс представляє вузол, а масив, збережений за цим індексом, містить його сусідів.
* Як об’єкт (або `Map`), де кожен ключ представляє вузол, а значення, пов’язане з цим ключем (масив), містить його сусідів.

Списки суміжності є більш ефективними за матриці суміжності з точки зору вимог до пам’яті. Вони мають просторову складність `O(V + E)`, де `V` — кількість вершин (вузлів), а `E` — кількість ребер.

Вони також ефективні для пошуку всіх сусідніх вузлів, оскільки ця операція вимагає лише доступу до списку, пов’язаного з вузлом.

Однак є і компроміс.

Списки суміжності менш ефективні, ніж матриці суміжності, для визначення, чи існує ребро між двома вузлами.

Процес пошуку зазвичай займає `O(deg(v))` часу (де `deg(v)` — кількість сусідів вузла, який ви перевіряєте), а в найгіршому випадку це може бути `O(V)`, якщо вузол з’єднаний з усіма іншими вузлами графа.

Ось приклад списку суміжності для цього графа:

<img src="https://cdn.freecodecamp.org/curriculum/lecture-transcripts/how-do-matrices-and-adjacency-lists-work-1.png" alt="Граф із чотирма вузлами, позначеними A, B, C і D. Вузол A з’єднується з B, C і D. Вузол B з’єднується з A і D. Вузол C з’єднується з A. Вузол D з’єднується з A і B. Це те саме зображення, що й раніше.">

Цей список суміжності реалізовано як об’єкт. Кожен ключ в об’єкті представляє вузол, а значення, пов’язане з цим ключем, — це масив із усіма сусідами відповідного вузла:

```javascript
const adjacencyList = {
  A: ["B", "C", "D"],
  B: ["A", "D"],
  C: ["A"],
  D: ["A", "B"]
};
```

Альтернативно, ми могли б реалізувати його як двовимірний масив, де кожен індекс представляє вузол. Наприклад, індекс 0 представляє вузол A, індекс 1 — вузол B і так далі:

```javascript
const adjacencyList = [
  ["B", "C", "D"], // Neighbors of A (index 0)
  ["A", "D"],      // Neighbors of B (index 1)
  ["A"],           // Neighbors of C (index 2)
  ["A", "B"]       // Neighbors of D (index 3)
];
```

Зверніть увагу, що навіть якщо цей двовимірний масив може виглядати схожим на матрицю суміжності, вони досить різні.

* Матриця суміжності зберігає 0, 1 або інші значення, які представляють ребра або ваги ребер у графі.

* Список суміжності зберігає фактичний список усіх сусідів кожного вузла.

Це дуже важлива різниця, з якою ви повинні бути знайомі.

І матриці суміжності, і списки суміжності дуже важливі для реалізації графів. Вибір між ними залежить від розміру графа та того, як ви плануєте використовувати дані. Матриці суміжності корисні для щільних графів із багатьма ребрами, тоді як списки суміжності зазвичай є кращим вибором для реальних сценаріїв, де розріджені графи зустрічаються частіше.
