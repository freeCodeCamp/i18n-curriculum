# --description--

В Express `express.Router()` — це потужна функція, яка допомагає організувати ваші маршрути у менші, модульні частини. Це особливо корисно, коли ваш застосунок зростає.

Замість того, щоб визначати всі маршрути в одному великому файлі, ви можете розбити їх на окремі файли за допомогою маршрутизаторів. Це робить ваш код легшим для читання, підтримки та масштабування.

Метод `express.Router()` створює новий об’єкт маршрутизатора.

Маршрутизатор працює як міні-застосунок Express. Він може обробляти власні маршрути та підпрограмне забезпечення, але не є повноцінним застосунком Express. Після того, як ви визначите маршрути на маршрутизаторі, ви можете підключити його до свого головного застосунку за допомогою `app.use()`.

Інакше кажучи, маршрутизатор дозволяє групувати пов’язані маршрути разом, а потім інтегрувати їх у головний застосунок.

Використання маршрутизаторів має кілька переваг:

**Розділення обов’язків**

Ви можете визначати маршрути в окремих файлах, що покращує організацію та читабельність вашого застосунку.

**Повторне використання коду**  
Маршрутизатори можна повторно використовувати в різних частинах застосунку або навіть у різних проєктах.

**Чистіший головний файл**  
Замість того, щоб заповнювати головний файл `app.js` або `server.js` багатьма визначеннями маршрутів, ви можете перемістити їх у маршрутизатори і тримати головний файл зосередженим на конфігурації та налаштуванні.

Розглянемо простий приклад, де ми використовуємо `express.Router()` для визначення маршрутів для шляху `/users`.

Спершу ось файл `app.js`:

```javascript
const express = require("express");
const app = express();
const port = 3000;

// Importing the user routes
const userRoutes = require("./userRoutes");

// Using the userRoutes router for /users path
app.use("/users", userRoutes);

app.listen(port, () => {
  console.log(`Server running at http://localhost:${port}`);
});
```

А ось `userRoutes.js`:

```javascript
const express = require("express");
const router = express.Router();

// GET request for /users
router.get("/", (req, res) => {
  res.send("List of users");
});

// POST request for /users
router.post("/", (req, res) => {
  res.send("Creating a new user");
});

// GET request for /users/:id
router.get("/:id", (req, res) => {
  const userId = req.params.id;
  res.send(`User details for user with id: ${userId}`);
});

module.exports = router;
```

Тепер давайте детальніше розглянемо, як усе працює.

У `app.js` ми імпортуємо `express` і створюємо екземпляр застосунку Express, призначаючи його змінній `app`.

Далі ми імпортуємо маршрутизатор `userRoutes` з `userRoutes.js` і використовуємо `app.use("/users", userRoutes)` для підключення маршрутизатора. Це повідомляє Express, що якщо запит починається з `/users`, його потрібно передати маршрутизатору `userRoutes`.

Потім у `userRoutes.js` ми створюємо новий маршрутизатор за допомогою `express.Router()` і визначаємо маршрути за допомогою `router.get()`, `router.post()` і `router.get("/:id")`.

Зверніть увагу, що маршрути, визначені тут, відносні до `/users`, оскільки саме там підключено маршрутизатор. Отже:

*   `router.get("/")` → `GET /users`: Відповідає списком користувачів
    
*   `router.post("/")` → `POST /users`: Відповідає повідомленням про створення нового користувача
    
*   `router.get("/:id")` → `GET /users/:id`: Відповідає деталями користувача за заданим ідентифікатором
    

Використання `express.Router()` допомагає тримати головний файл чистим і зосередженим на конфігурації. Всі маршрути, пов’язані з користувачами, згруповані в `userRoutes.js`.

Коли ваш застосунок зростає, ви можете створювати додаткові маршрутизатори для інших розділів, таких як `/products`, `/orders` або `/auth`.

Кожен розділ залишається організованим у власному файлі, що полегшує підтримку та масштабування застосунку.
