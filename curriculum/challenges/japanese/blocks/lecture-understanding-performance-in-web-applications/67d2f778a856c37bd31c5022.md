# --description--

ウェブパフォーマンスを測定し改善するために、First Contentful Paint（FCP）、Speed Index、Total Blocking Time（TBT）、Bounce Rate、Unique Users といった主要なパフォーマンス指標を使います。これらの指標をそれぞれ見ていきましょう。

First Contentful Paint、略してFCPは、テキストや画像など最初のコンテンツが画面に表示されるまでの速さを測定します。良いFCPは1.8秒未満とされ、悪いFCPは通常3秒以上です。Chromeの開発者ツールを使ってFCPを確認する方法は以下の通りです。

ステップ1：Chromeの開発者ツールを開く

- Google Chromeを開きます。

- テストしたいウェブページにアクセスします。

- 「F12」を押すか、右クリックして「検証」を選択します。

ステップ2：開発者ツール内の「Lighthouse」タブを見つけてクリックします。

ステップ3：「Analyze page load」ボタンをクリックし、解析が終わるまで待ちます。

結果を分析しましょう。下にスクロールして「Metrics」セクションの「First Contentful Paint」マーカーを探します。ミリ秒または秒単位で正確な時間が表示されます。

FCPと以前に見たInteraction to Next Paint（INP）はどちらもパフォーマンス指標ですが、ユーザー体験の異なる側面を測定しています。混同しないようにしましょう。

復習すると、FCPは読み込み後にユーザーがコンテンツをどれだけ早く見るかを示し、INPはクリックや入力後にフィードバックがどれだけ早く表示されるかを示します。

次にSpeed Indexです。

Speed Indexはページの見える部分がどれだけ速く描画されるかを測定します。スコアが低いほどパフォーマンスが良いことを意味します。Speed IndexはLighthouseレポートで確認できます。

次にTotal Blocking Time（TBT）です。これは重いJavaScriptタスクによってメインスレッドがどれだけ長くブロックされているかを示します。TBTが高いとユーザーは操作がもたつく体験をします。TBTを改善するには、長いタスクを分割し、重要でないスクリプトは遅延させましょう。

次にBounce Rateです。これは訪問者が操作せずに離脱する割合を示します。サイトの離脱率が高い場合、ページの読み込みが遅すぎる可能性があります。

最後にUnique Usersです。この指標はサイトに訪れる個別の訪問者数を追跡します。Bounce RateやUnique UsersはGoogle Analyticsで確認でき、これらの指標を監視してエンゲージメントを改善できます。
