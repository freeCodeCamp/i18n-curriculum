# --description--

Aprendamos sobre los códigos comunes de respuesta HTTP.

Los códigos de respuesta HTTP son códigos numéricos de tres dígitos que los servidores web usan para comunicarse con los clientes y hacerles saber qué ocurrió con sus solicitudes.

Proporcionan información sobre si la solicitud se manejó con éxito o si hubo un error al manejar la solicitud.

Cubriré algunos de los más comunes en esta lección.

## Códigos de éxito

Comencemos con los **códigos de éxito**. Estos códigos comienzan con el número 2.

El código **200 OK** es el código de respuesta HTTP más común. Significa que la solicitud fue exitosa y el servidor devolvió la información solicitada por el cliente.

El código **201 Created** es devuelto por el servidor cuando el cliente solicita crear un recurso y el proceso se completa con éxito. Por ejemplo, si se crea una nueva cuenta de usuario correctamente, el servidor devuelve este código de respuesta al cliente.

El código **204 No Content** se devuelve cuando la solicitud se ha manejado con éxito pero el servidor no tiene contenido que devolver. Esto puede usarse para procesos que eliminan o actualizan un recurso en una base de datos pero no tienen datos que devolver.

## **Códigos de redirección**

Ahora hablemos de los **códigos de redirección**.

Estos códigos son devueltos por el servidor cuando la solicitud de recurso por parte del cliente ha sido movida a una ubicación diferente o cuando el cliente necesita tomar pasos adicionales para completar la solicitud.

Estos códigos comienzan con el número 3.

El código **301 Moved Permanently** se devuelve cuando el recurso solicitado por el cliente ha sido movido permanentemente a otra ubicación. En este caso, el servidor debe incluir una URL a la nueva ubicación en la respuesta y una vez que el cliente reciba la respuesta, el usuario debe ser redirigido a esta URL.

El código **302 Found** indica que el recurso solicitado por el cliente ha sido movido temporalmente a otra ubicación. El servidor debe proporcionar la nueva URL en la respuesta y el cliente debe redirigir al usuario a esta URL. Sin embargo, la URL original podría estar disponible en el futuro ya que el cambio es temporal. Esa es la principal diferencia entre los códigos de respuesta 301 y 302.

El código **304 Not Modified** es devuelto por el servidor para indicar al cliente que el recurso solicitado no ha sido modificado desde la última vez que se solicitó. El servidor envía este código para evitar enviar datos que ya han sido enviados si el contenido no ha cambiado. Es una técnica de optimización de rendimiento.

## Códigos de error del cliente

Ahora comenzaremos a profundizar en los **códigos de error**.

Los errores pueden ser causados por el cliente y por el servidor. Primero hablaremos de los códigos de error del cliente.

Los códigos de error del cliente se devuelven cuando la solicitud no pudo completarse debido a un problema generado por el cliente.

Estos códigos de error comienzan con el número 4.

El código **400 Bad Request** se devuelve cuando el servidor no pudo entender la solicitud enviada por el cliente. Esto puede ocurrir cuando el cliente envía una solicitud con sintaxis incorrecta, con parámetros inválidos o datos inválidos.

El código **401 Unauthorized** está relacionado con la autenticación. Indica que el cliente debe proporcionar ciertas credenciales de autenticación para acceder al recurso que se está solicitando. Estas credenciales pueden incluir un nombre de usuario y una contraseña.

El código **403 Forbidden** se devuelve cuando el cliente no está autorizado para acceder al recurso solicitado. La diferencia entre 401 y 403 es que en 403, el cliente no tiene los permisos necesarios para el recurso particular que está solicitando, incluso si tiene credenciales de autenticación válidas.

El código **404 Not Found** se devuelve cuando no se pudo encontrar el recurso en el servidor. Tal vez fue eliminado o hay un error tipográfico en la URL que impide que el servidor encuentre el recurso. El servidor también puede estar experimentando un problema temporal que resulta en este código de error.

## Códigos de error del servidor

A continuación, tenemos los **códigos de error del servidor**. Estos códigos son devueltos por el servidor cuando la solicitud no pudo ser procesada debido a un problema en el servidor.

El código de error **500 Internal Server Error** indica que hubo un error inesperado en el servidor. Este error podría ser causado por una amplia variedad de problemas, que van desde problemas de configuración hasta problemas temporales.

El código de error **502 Bad Gateway** se devuelve cuando el servidor intentaba comunicarse con otro servidor pero recibió una respuesta inválida de ese servidor. Hay una amplia variedad de causas para este tipo de error.

El código de error **503 Service Unavailable** indica que el servidor no podrá manejar la solicitud temporalmente, por lo que el cliente debe intentar de nuevo más tarde. Las causas comunes de este error incluyen tareas de mantenimiento y un alto volumen de solicitudes que el servidor actualmente no puede manejar.

Los códigos de respuesta HTTP son esenciales para entender la comunicación entre clientes y servidores web. Al comprender estos códigos, puedes solucionar problemas y depurar tus aplicaciones web.
