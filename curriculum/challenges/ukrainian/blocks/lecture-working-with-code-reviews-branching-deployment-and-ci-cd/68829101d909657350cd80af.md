# --description--

GitHub Pages — це дуже надійний варіант для розгортання статичних сайтів або застосунків, які не потребують серверної логіки. Тобто застосунки, що повністю працюють на стороні клієнта або в браузері користувача, можна повністю розгорнути на цій платформі.

Крім того, GitHub Pages повністю безкоштовні для проєктів з відкритим кодом. Існує багато вагомих причин зробити свій проєкт відкритим, і Pages додає ще одну.

Але як насправді розгорнути застосунок на GitHub Pages? Це залежить від того, як працює ваш застосунок.

Почнемо з базової HTML-сторінки. Ось приклад, який ви можете додати до свого існуючого репозиторію, якщо хочете:

```html
<!DOCTYPE html>
<html>
  <head>
    <title>freeCodeCamp</title>
    <link rel="stylesheet" href="./style.css" />
  </head>
  <body>
    <h1>Welcome to freeCodeCamp's Lessons!</h1>
    <p>Let's have some fun learning about Git and GitHub!</p>
  </body>
</html>
```

Тепер потрібно відвідати свій репозиторій і натиснути на вкладку "Settings", щоб перейти до налаштувань репозиторію. Потім виберіть "Pages" у бічному меню.

Спочатку там лише кілька опцій, але це зміниться, коли ми почнемо налаштовувати параметри. Спершу потрібно вказати GitHub, що саме розгортати.

Ми залишимо вибір "Deploy from a branch", бо ми не використовуємо фреймворк і не налаштовували жодних GitHub actions. Ви дізнаєтеся більше про них у наступному уроці.

Далі встановіть гілку, з якої ми розгортаємо, на `main`, щоб розгорнути вміст цієї гілки.

Потім вам запропонують вибрати, чи розгортати з кореневої папки проєкту, чи з папки `docs`. Ця функція використовується в проєктах на кшталт freeCodeCamp — ми раніше об’єднували документацію із застосунком, щоб можна було розгортати лише документацію на GitHub Pages через папку `docs`. Наразі залиште вибір на папці `/` (корінь) і натисніть кнопку "Save" поруч із цим випадаючим меню, щоб зберегти зміни.

Після збереження з’явиться поле для налаштування кастомного домену. Якщо у вас він є і ви хочете його використати, можете перейти за посиланням у документацію, щоб налаштувати. Для цього уроку залишимо це поле порожнім.

Також ви побачите домен, на якому розгорнуто ваш новий сайт GitHub Pages. Без кастомного домену за замовчуванням це `<username>.github.io`, де `<username>` — це ім’я вашого акаунта (або організації, якщо ви розгортаєте з репозиторію, що належить організації). Репозиторій буде доступний за шляхом, що відповідає імені репозиторію, наприклад `<username>.github.io/<repo-name>`. Якщо відвідати цю URL-адресу, ви побачите свій сайт!

Тепер, коли ви пушите нові зміни у гілку `main`, GitHub Pages автоматично оновить розгорнутий сайт із останніми змінами. Цей процес може зайняти кілька хвилин залежно від навантаження на сервер — і вам може знадобитися зробити жорстке оновлення сторінки, щоб очистити кеш.

Отже, ми успішно розгорнули базовий веб-застосунок на GitHub Pages. Ви також можете додавати CSS і JavaScript файли до розгортання Pages, щоб створювати гарні та інтерактивні сайти. А коли вивчите фреймворк на кшталт React або Angular, зможете розгортати їх теж на GitHub Pages!
