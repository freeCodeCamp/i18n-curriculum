# --interactive--

CSS-Custom-Properties, auch bekannt als CSS-Variablen, sind von CSS-Autoren definierte Entitäten, die bestimmte Werte enthalten, die im gesamten Dokument wiederverwendet werden können. Sie sind eine leistungsstarke Funktion, die effizientere, wartbarere und flexiblere Stylesheets ermöglicht.

Die Syntax zur Deklaration einer Custom Property ist einfach. Sie beginnt mit zwei Bindestrichen (`--`) gefolgt vom Eigenschaftsnamen:

```css
:root {
  --main-color: #3498db;
}
```

In diesem Beispiel deklarieren wir eine Custom Property namens `--main-color` mit dem Wert `#3498db`. Die Pseudoklasse `:root` wird häufig verwendet, um globale Custom Properties zu deklarieren, da sie das oberste Elternelement im DOM-Baum darstellt.

Um eine Custom Property zu verwenden, nutzen Sie die `var()`-Funktion:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<button class="button">Click Me</button>
```

```css
:root {
  --main-color: #3498db;
}

.button {
  background-color: var(--main-color);
}
```

:::

Dies setzt die Hintergrundfarbe von Elementen mit der Klasse `button` auf den in `--main-color` gespeicherten Wert.

Eine der wichtigsten Funktionen von Custom Properties ist, dass sie der CSS-Kaskade folgen. Das bedeutet, dass Sie sie für bestimmte Elemente oder Kontexte neu definieren können:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<div class="alert">This is an alert message.</div>
```

```css
.alert {
  --main-color: #e74c3c;
  background-color: var(--main-color);
}
```

:::

In diesem Fall verwenden Elemente mit der Klasse `alert` einen anderen Wert für `--main-color` und überschreiben damit die globale Definition.

Custom Properties unterstützen auch Fallback-Werte. Wenn eine Custom Property nicht definiert oder ungültig ist, können Sie einen Fallback-Wert angeben:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<div class="text">This is some text.</div>
```

```css
:root {
  --text-color: green;
}

.text {
  color: var(--text-color, green);
}
```

:::

Hier wird, falls `--text-color` nicht definiert ist, `color` standardmäßig auf `green` gesetzt.

Custom Properties sind besonders nützlich, um themenfähige Designs zu erstellen. Sie können eine Reihe von Eigenschaften für verschiedene Themes definieren:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<div class="light-theme">
  <p>This is some light themed text.</p>
</div>
```

```css
:root {
  --bg-color: black;
}

body {
  background-color: var(--bg-color);
}

.light-theme {
  --bg-color: white;
  --text-color: black;
  background-color: var(--bg-color);
  color: var(--text-color);
  padding: 10px;
}

```

:::

Das Wechseln der Themes wird so einfach wie das Hinzufügen oder Entfernen einer Klasse vom `body`-Element.

Custom Properties können auch mit Media Queries verwendet werden, um responsives Design zu erstellen:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<div class="card">
  <h2>Responsive Design</h2>
  <p>Resize the window to see the card adapt!</p>
</div>

```

```css
:root {
  --card-width: 90%;
  --card-bg: #f0f0f0;
  --card-padding: 1rem;
  --text-color: #333;
}

/* Tablet screens and up */
@media (min-width: 600px) {
  :root {
    --card-width: 70%;
    --card-bg: #e8f5e9;
    --card-padding: 1.5rem;
  }
}

/* Desktop screens and up */
@media (min-width: 1024px) {
  :root {
    --card-width: 50%;
    --card-bg: #d0f0ff;
    --card-padding: 2rem;
  }
}

body {
  font-family: system-ui, sans-serif;
  display: flex;
  justify-content: center;
  align-items: center;
  min-height: 100vh;
  background-color: #fafafa;
}

.card {
  width: var(--card-width);
  background-color: var(--card-bg);
  padding: var(--card-padding);
  color: var(--text-color);
  border-radius: 8px;
  box-shadow: 0 2px 6px rgba(0,0,0,0.1);
  text-align: center;
  transition: all 0.3s ease;
}

```

:::

Dieser Ansatz ermöglicht wartbarere responsive Layouts, da Sie Werte an einer Stelle ändern können, anstatt im gesamten Stylesheet. Das kann die Wiederholung in Ihren Stylesheets erheblich reduzieren, besonders bei häufig verwendeten, komplexen Werten.

Custom Properties können auch andere Custom Properties referenzieren:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<button class="button">Click Me</button>
```

```css
:root {
  --primary-color: #3498db;
  --button-bg: var(--primary-color);
}

button {
  background-color: var(--button-bg);
}

```

:::

Das ermöglicht es, Beziehungen zwischen verschiedenen Stil-Aspekten herzustellen und erleichtert die konsistente Pflege von Themes auf Ihrer Website.

Obwohl Custom Properties viele Vorteile bieten, sollten Sie sich ihrer Browser-Unterstützung bewusst sein. Sie werden in modernen Browsern gut unterstützt, aber ältere Browser erkennen sie möglicherweise nicht. Haben Sie immer einen Fallback-Plan, wenn Sie moderne Funktionen verwenden.

Zusammenfassend bieten CSS-Custom-Properties eine leistungsstarke Möglichkeit, dynamischere, flexiblere und wartbarere Stylesheets zu erstellen. Sie ermöglichen die Erstellung von themenfähigen Designs, vereinfachen responsive Layouts und erlauben die Laufzeit-Manipulation von Styles.

Da sich die Webentwicklung ständig weiterentwickelt, werden Custom Properties zu einem immer wichtigeren Werkzeug im Werkzeugkasten von Entwicklern und eröffnen neue Möglichkeiten für anpassungsfähiges und effizientes CSS.
