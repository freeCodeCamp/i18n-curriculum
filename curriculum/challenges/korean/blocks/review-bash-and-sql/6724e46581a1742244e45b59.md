## 데이터베이스 정규화

관계형 데이터베이스에서 데이터 중복을 줄이고 무결성을 향상시키기 위해 구조를 조직하는 과정입니다.

이점:

- 중복 데이터 최소화로 저장 공간 절약 및 불일치 감소.
- 기본 키와 외래 키를 통한 데이터 무결성 강제.
- 데이터베이스 유지보수와 이해가 쉬워짐.

### 정규형

- **1NF (제1정규형)**
  - 각 셀은 단일(원자) 값을 가짐.
  - 각 레코드는 고유함(기본 키로 강제).
  - 행과 열의 순서는 중요하지 않음.
  - 예: `students` 테이블의 여러 전화번호를 별도의 `student_phones` 테이블로 분리.

- **2NF (제2정규형)**
  - 1NF 조건 충족.
  - **부분 함수 종속성 없음**: 모든 비키 속성은 복합 기본 키 전체에 의존해야 함.
  - 예: `orders` 테이블을 `order_header`와 `order_items`으로 분리해 일부 속성이 키의 일부에만 의존하는 것을 방지.

- **3NF (제3정규형)**
  - 2NF 조건 충족.
  - **이행적 종속성 없음**: 비키 속성이 다른 비키 속성에 의존하지 않음.
  - 예: `city_postal_code`을 모든 주문에 저장하지 않고 `cities` 테이블로 이동.

- **BCNF (Boyce-Codd 정규형)**
  - 3NF 조건 충족.
  - 모든 결정자는 슈퍼키여야 함.

**팁**: 무결성과 성능의 균형을 위해 대부분 설계에서 3NF를 목표로 하세요.

## 주요 SQL 개념

- SQL은 관계형 데이터베이스와 소통하기 위한 구조화된 쿼리 언어입니다.
- **기본 명령어** → `SELECT`, `INSERT`, `UPDATE`, `DELETE`, `CREATE TABLE`, `ALTER TABLE` 등
- **조인** → 여러 표에서 데이터를 결합합니다 (`INNER JOIN`, `LEFT JOIN`, `RIGHT JOIN`, `FULL JOIN`).

## Bash에서 SQL 명령 실행

PostgreSQL용 `psql` 명령줄 클라이언트나 다른 데이터베이스 도구를 사용해 명령줄에서 직접 SQL 명령을 실행할 수 있습니다.

예를 들어 PostgreSQL에서 SQL 파일을 실행하려면:

```bash
psql -U username -d database_name -c "SELECT * FROM students;"
```

MySQL 명령도 직접 실행할 수 있습니다:

```bash
mysql -u username -p database_name -e "SELECT * FROM students;"
```

### 파일에서 SQL 실행

```bash
# PostgreSQL
psql -U username -d database_name -f script.sql

# MySQL
mysql -u username -p database_name < script.sql
```

### Bash 스크립트에 SQL 임베드

```bash
#!/bin/bash
DB_USER="school_admin"
DB_NAME="school"

# Insert student data
psql -U "$DB_USER" -d "$DB_NAME" -c \
"INSERT INTO students (name, age, major) VALUES ('Alice', 20, 'CS');"
```

### SQL에서 변수 사용

```bash
#!/bin/bash
DB_USER="school_admin"
DB_NAME="school"
STUDENT_NAME="Bob"
AGE=21

psql -U "$DB_USER" -d "$DB_NAME" -c \
"INSERT INTO students (name, age) VALUES ('$STUDENT_NAME', $AGE);"
```

## Bash에서 SQL 쿼리 결과 가져오기 및 사용

`psql`를 통해 SQL 쿼리를 실행하면 반환된 값을 Bash 스크립트에서 **캡처**하고 **처리**할 수 있습니다.

### 단일 값 캡처

```bash
#!/bin/bash
DB_USER="school_admin"
DB_NAME="school"

# Get total student count
STUDENT_COUNT=$(psql -U "$DB_USER" -d "$DB_NAME" -t -A -c \
"SELECT COUNT(*) FROM students;")

echo "Total students: $STUDENT_COUNT"
```

출력 → 42

### 여러 열 가져오기

```bash
#!/bin/bash
DB_USER="school_admin"
DB_NAME="school"

# Get top 3 students' names and ages
RESULTS=$(psql -U "$DB_USER" -d "$DB_NAME" -t -A -F"," -c \
"SELECT name, age FROM students LIMIT 3;")

echo "Top 3 students:"
echo "$RESULTS"
```

출력

```bash
Alice,20
Bob,21
Charlie,22
```

### 쿼리 결과 반복 처리

```bash
#!/bin/bash
DB_USER="school_admin"
DB_NAME="school"

# Get student names and majors
psql -U "$DB_USER" -d "$DB_NAME" -t -A -F"," -c \
"SELECT name, major FROM students;" | while IFS="," read -r name major
do
    echo "Student: $name | Major: $major"
done
```

출력 형태

```bash
Student: Alice | Major: CS
Student: Bob   | Major: Math
Student: Carol | Major: Physics
```

## SQL 인젝션

입력 필드에 악의적인 SQL 코드를 삽입해 데이터베이스를 조작하는 웹 보안 취약점입니다.

위험한 행위 예:

- 인증 우회.
- 민감 데이터 탈취.
- 레코드 수정 또는 삭제.

SQL 인젝션 공격 예:

```sql
SELECT * FROM users WHERE username = '' OR '1'='1' --' AND password = 'anything';
```

이 쿼리는 조건 `OR '1'='1'`이 항상 참이기 때문에 모든 사용자를 반환하며, 공격자가 로그인 검사를 우회할 수 있게 합니다.

### SQL 인젝션 방지

1. **준비된 문 사용**: SQL 코드와 데이터를 분리해 인젝션을 방지합니다. 예 (Node.js pg 사용):

    ```js
    client.query('SELECT * FROM users WHERE username = $1 AND password = $2', [username, password]);
    ```

2. **입력 유효성 검사**: 모든 사용자 입력을 정제하고 예상 형식에 맞는지 검증하세요.

3. **최소 권한 원칙**: 애플리케이션에 필요한 최소 권한만 가진 데이터베이스 계정을 사용하세요.

**주의**: 애플리케이션 계정에 관리자 권한을 절대 부여하지 마세요.

## N+1 문제

애플리케이션이 아이템 목록(N)을 조회하는 쿼리 하나를 실행한 후, 각 아이템마다 관련 데이터를 조회하는 추가 쿼리를 실행해 총 N+1개의 쿼리가 발생하는 문제입니다.

**문제점**

- 각 쿼리가 네트워크와 처리 오버헤드를 증가시킵니다.
- 여러 작은 쿼리가 최적화된 하나의 쿼리보다 느립니다.

### N+1 패턴 예

```sql
-- 1: Get list of orders
SELECT * FROM orders LIMIT 50;

-- N: For each order, get customer
SELECT * FROM customers WHERE customer_id = ...;
```

**해결책**: `JOINs` 또는 다른 집합 기반 연산을 사용하세요.

```sql
SELECT 
  orders.order_id,
  orders.product,
  orders.quantity,
  customers.customer_id,
  customers.name,
  customers.email,
  customers.address
FROM orders
JOIN customers 
  ON orders.customer_id = customers.customer_id
WHERE orders.order_id IN (SELECT order_id FROM orders LIMIT 50);
```

관련 데이터를 하나의 쿼리로 병합할 기회를 항상 찾으세요.
