# --description--

開放圖譜協定啟用你控制你網站內容在各種社群媒體平台（例如 Facebook、LinkedIn 等）上的顯示方式。透過設定這些開放圖譜屬性，你可以吸引用戶點擊並與你的內容互動。你可以透過 HTML `head` 區段內的一組 `meta` 元素來設定這些屬性。

第一個重要的 OG 屬性是 `title`。以下是為 freeCodeCamp 首頁設定 OG `title` 的範例：

```html
<meta content="freeCodeCamp.org" property="og:title" />
```

對於 `property` 屬性，你需要指定它為 `og:title`。`content` 屬性是你將為社群媒體網站顯示的標題寫入的位置。

下一個重要的 OG 屬性是 `type`。以下是使用 OG `type` 為 freeCodeCamp 首頁的範例：

```html
<meta property="og:type" content="website" />
```

`type` 屬性用於表述在社群媒體上分享的內容型別。此類內容的範例包括文章、網站、影片或音樂。

第三個重要的 OG 屬性是 `image`。以下是為 freeCodeCamp 首頁設定 OG `image` 的範例：

```html
<meta
  content="https://cdn.freecodecamp.org/platform/universal/fcc_meta_1920X1080-indigo.png"
  property="og:image"
/>
```

在這個範例中，open graph 影像指向 freeCodeCamp 標誌。所有這些影像都應該具有高品質且尺寸與比例良好。大多數社群媒體平台會包含影像需求的標準，以幫助你確保你的內容能在他們的網站上良好顯示。例如，developers.facebook.com 文件頁面指出：

為了在高解析度裝置上獲得最佳顯示效果，請使用至少 1200 x 630 像素的影像。至少，你應該使用 600 x 315 像素的影像來顯示具有較大影像的連結頁面貼文。

第四個重要的 OG 屬性是 `url`。以下是為 freeCodeCamp 首頁設定 OG `url` 的範例：

```html
<meta property="og:url" content="https://www.freecodecamp.org" />
```

你可以設定更多的 OG 屬性，例如 `description`、`audio`、`video` 和 `locale`。然而，開放圖譜的 `url`、`image`、`type` 和 `title` 是最重要要包含的。

那麼這些 open graph 屬性如何影響搜尋引擎優化？當你的內容在社群媒體上被分享時，精心設計的 OG 屬性可以提升內容在使用者動態中的呈現效果。這可能導致更高的點擊率，進而向搜尋引擎傳達你的內容具有相關性且具吸引力。
