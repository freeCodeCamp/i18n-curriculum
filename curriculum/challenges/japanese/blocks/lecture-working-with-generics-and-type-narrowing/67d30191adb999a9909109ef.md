# --description--

実は、前のレッスンでジェネリック型を見たことがあります：`Array<string>`。しかし、通常は関数でジェネリック型を使います。実際、ジェネリック型は関数の型定義の動作を制御するために関数に渡す特別なパラメータのようなものと考えられます。

前回の配列からランダムな値を取得する関数の例に戻りましょう：

```js
const getRandomValue = (array: string[]): string => {
  return array[Math.floor(Math.random() * array.length)];
}
```

この関数は文字列の配列を受け取り、文字列を返します。しかし、もし数値の配列を渡して数値を返したい場合はどうでしょうか？

ユニオン型を使うこともできます：

```js
const getRandomValue = (array: (string|number)[]): (string|number) => {
  return array[Math.floor(Math.random() * array.length)];
}
const val = getRandomValue([1, 2, 4])
```

しかしこの方法では、数値だけの配列を渡しても、TypeScriptは`val`変数が数値か文字列のどちらかになる可能性があると判断します。

代わりに、関数にジェネリック型を定義できます：

```js
const getRandomValue = <T>(array: T[]): T => {
  return array[Math.floor(Math.random() * array.length)];
}
const val = getRandomValue([1, 2, 4])
```

新しい例にはいくつか重要な要素があります。まず、`<T>`構文はTypeScriptに対して関数のジェネリック型`T`を定義していることを伝えます。`T`はジェネリック型の一般的な名前ですが、JavaScriptの変数慣例に従う限り好きな名前を付けられます。

次に、配列のパラメータがジェネリック型に一致する値の配列であること、返される値が同じ型の単一の要素であることをTypeScriptに伝えます。これにより、渡した配列が数値のみで構成されているため、TypeScriptは`val`が数値であると正しく検出できます。

では、自分で制御できない関数はどうでしょう？`document.querySelector()`メソッドを見てみましょう：

```js
const email = document.querySelector("#email");
console.log(email.value);
```

IDセレクターでクエリしているため、TypeScriptはどの種類の要素をクエリしているのか分かりません。何らかの`Element`であることは分かっていますが、トップレベルの要素インターフェースに`value`プロパティがないため、そのプロパティにアクセスしようとするとエラーになります。

幸い、関数呼び出しに型引数を渡せます：

```js
const email = document.querySelector<HTMLInputElement>("#email");
console.log(email.value);
```

ここでも角括弧構文（`<HTMLInputElement>`）を使っていますが、今回は関数呼び出しの中です。これにより、TypeScriptは見つけることを期待する要素が`input`要素であると認識し、input要素にはvalueプロパティがあるためエラーがなくなります。もちろん、`querySelector`は要素が見つからない場合に`null`を返す可能性があり、その場合はこのコードで処理していないためTypeScriptはコンパイルエラーを出します。

しかし、その点は次のレッスンで学びます。
