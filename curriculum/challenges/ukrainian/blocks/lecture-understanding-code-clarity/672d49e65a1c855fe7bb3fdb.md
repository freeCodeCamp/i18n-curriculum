# --description--

Коментарі в програмуванні використовують, щоб надати додатковий контекст для коду або залишити нотатки для себе та інших.

Коментарі — це рядки або блоки тексту, які ігнорує JavaScript-двигун під час виконання коду. Вони існують виключно для користі людей, які читають код, чи то ви, чи хтось інший.

JavaScript надає два способи додати коментарі до коду: однорядкові коментарі та багаторядкові коментарі.

Однорядкові коментарі створюються за допомогою двох косих рисок (`//`). Ось приклад:

```js
// I am a single line comment in JavaScript
```

Цей тип коментаря чудово підходить для коротких пояснень або уточнень.

Ось реальний приклад із файлів навчальної програми freeCodeCamp:

```js
// This is to allow English to build without having to download the i18n files.
// It fails when trying to resolve the i18n-curriculum path if they don't exist.
const curriculumLocale = process.env.CURRICULUM_LOCALE ?? 'english';
const I18N_CURRICULUM_DIR = path.resolve(
  __dirname,
  curriculumLocale === 'english' ? '.' : 'i18n-curriculum/curriculum'
);
```

Не хвилюйтеся, якщо не розумієте, що саме робить цей код, бо він складніший за те, що ви вивчили досі. Замість цього зверніть увагу на коментар розробника. Цей коментар надає важливий контекст, чому цей код існує.

Такі коментарі важливі для командної роботи з двох причин:

1. Інші розробники, які працюють над проєктом, зрозуміють призначення цього коду.

1. Це допомагає уникнути непотрібних змін або видалень без консультації з командою, що могло б призвести до помилок або проблем.

Інший тип коментарів — багаторядкові коментарі. Ось базовий синтаксис:

```js
/*
 I am a multiline comment.
 This is helpful for longer explanations.
*/
```

Багаторядкові коментарі корисні, коли потрібно написати довші описи, пояснення або нотатки у вашому коді.

Давайте ще раз подивимося на файли навчальної програми freeCodeCamp, щоб побачити, як багаторядкові коментарі можна використовувати на практиці.

```js
/* Since there can be more than one way to complete a certification (using the
legacy curriculum or the new one, for instance), we need a certification
field to track which certification this belongs to. */
const dupeCertifications = [
  {
    certification: 'responsive-web-design',
    dupe: '2022/responsive-web-design'
  }
];
const hasDupe = dupeCertifications.find(
  cert => cert.dupe === meta.superBlock
);
```

Як і раніше, ігноруйте весь JavaScript-код, бо він використовує концепції, які ще не вивчали. Зверніть увагу на коментар розробника.

Розробник у команді або навіть новий учасник проєкту може зрозуміти, чому цей фрагмент коду тут і отримати повний контекст перед роботою над цією областю проєкту.

Хоча коментарі корисні в програмуванні, важливо уникати надмірного коментування. Не потрібно коментувати кожен рядок коду, особливо якщо код простий і зрозумілий сам по собі.

Ось приклад використання коментарів для пояснення очевидного:

```js
// This code uses the const keyword to create a new variable called price.
// We are assigning the number 10 to the price variable.
const price = 10;
```

У цій ситуації немає потреби додавати коментарі, бо код і так зрозумілий. Мета — покращити читабельність, а не засмічувати код зайвими поясненнями.

Якщо ви хочете додавати коментарі у свої особисті проєкти під час навчання програмуванню, це нормально. Але коли починаєте працювати над реальними проєктами з іншими розробниками, важливо не використовувати коментарі для коду, який і так зрозумілий.

Також важливо не використовувати коментарі, щоб пояснювати заплутаний, надто складний або погано написаний код. У таких випадках краще рефакторити, тобто змінювати код, щоб інші розробники краще розуміли, що відбувається.

Коментарі — це потужний інструмент для документування коду і полегшення його розуміння. Використовуйте коментарі, щоб надавати контекст або залишати нотатки для себе та інших.
