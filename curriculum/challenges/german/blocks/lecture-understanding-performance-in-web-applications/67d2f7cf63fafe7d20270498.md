# --description--

Während Tools wie Lighthouse und Chrome Entwicklerwerkzeuge großartig sind, benötigen Sie manchmal maßgeschneiderte Einblicke, die speziell auf Ihre Website zugeschnitten sind.

Hier kommen Performance-Web-APIs ins Spiel!

In dieser Lektion erklären wir, wie Sie drei wichtige Web-APIs nutzen können, um die Geschwindigkeit Ihrer Website zu messen und zu analysieren.

Aber zuerst: Was sind Performance-Web-APIs?

Performance-Web-APIs ermöglichen es Entwicklerinnen und Entwicklern, direkt im Code zu verfolgen, wie effizient eine Webseite lädt und reagiert. Diese APIs erlauben es Ihnen, Ladezeiten von Seiten zu messen, Verzögerungen beim Rendern und bei Interaktionen zu verfolgen und die Ausführungszeit von JavaScript zu analysieren.

Mit diesen APIs können Sie Ihre eigenen Performance-Überwachungstools bauen, ohne auf Software von Drittanbietern angewiesen zu sein!

Lassen Sie uns drei leistungsstarke Web-APIs erkunden, die Sie heute schon nutzen können.

Zuerst `performance.now()`.

Diese API liefert hochpräzise Zeitstempel (in Millisekunden), um zu messen, wie lange verschiedene Teile Ihrer Seite zum Laden brauchen.

Angenommen, Sie möchten messen, wie schnell eine Funktion läuft:

```js
const start = performance.now();  
// Run some code here  
const end = performance.now();  

console.log(`Execution time: ${end - start}ms`);
```

Das ist genauer als die Verwendung von `Date.now()`, weil es die Zeit in Millisekunden mit Mikrosekunden-Genauigkeit misst und so Probleme mit der Systemuhr vermeidet. Sie können es nutzen, um die Ausführungszeit von Skripten, Verzögerungen bei Ereignisantworten und die Performance von Animationen zu verfolgen.

Als nächstes die Performance Timing API.

Diese API gibt Ihnen eine Aufschlüsselung jeder einzelnen Phase des Seitenladens, vom DNS-Lookup bis zu `DOMContentLoaded`.

Möchten Sie messen, wie lange Ihre Seite zum vollständigen Laden braucht?

```js
let [navigationTiming] = performance.getEntriesByType("navigation");

if (navigationTiming instanceof PerformanceNavigationTiming) {
  // Calculate time from navigation start to DOM content loaded
  const pageLoadTime =
    navigationTiming.domContentLoadedEventEnd - navigationTiming.startTime;

  console.log("DOM Content Loaded Time:", pageLoadTime, "ms");
}
```

Wichtige Kennzahlen, die Sie mit dieser API verfolgen können, sind die DNS-Lookup-Zeit – also die Verbindungsgeschwindigkeit –, Time to First Byte (TTFB) – also die Antwortgeschwindigkeit des Servers – und `DOMContentLoaded` – also wann die Seite für Interaktionen bereit ist.

Wenn Ihre Ladezeiten langsam sind, zeigt Ihnen diese API genau, wo die Verzögerung auftritt!

Und schließlich sprechen wir über `PerformanceObserver`.

Diese API hört auf Performance-Ereignisse wie Layout-Verschiebungen, lange Tasks und Benutzerinteraktionen.

Möchten Sie lange laufende JavaScript-Tasks überwachen?

```js
const observer = new PerformanceObserver((list) => {  
  list.getEntries().forEach((entry) => {  
    console.log(`Long task detected: ${entry.duration}ms`);  
  });  
});  

observer.observe({ type: "longtask", buffered: true });
```

Und was kann diese API verfolgen? Sie kann lange Tasks erfassen – also JavaScript, das das Rendern blockiert –, Layout-Verschiebungen erkennen, um UI-Ruckler zu identifizieren, und First Input Delay (FID) – also wie schnell eine Seite auf Benutzereingaben reagiert.

Wenn Sie Performance in Echtzeit verfolgen wollen, ist diese API ein echter Game-Changer.

Welche API sollten Sie also verwenden? Hier ein kurzer Vergleich:

| Performance API        | Am besten geeignet für                              |
| :--------------------- | :------------------------------------------------- |
| `performance.now()`    | Präzise Zeitmessung von Funktionen und Skripten    |
| Performance Timing API | Messung der vollständigen Seitenlade-Performance   |
| Performance Observer   | Echtzeitüberwachung von Interaktionen und Rendering |

Indem Sie diese APIs kombinieren, können Sie Ihre eigenen Performance-Messwerkzeuge bauen und genau das verfolgen, was für Ihre Seite am wichtigsten ist.
