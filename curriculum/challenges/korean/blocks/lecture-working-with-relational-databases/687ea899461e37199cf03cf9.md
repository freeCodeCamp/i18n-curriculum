# --description--

SQL에서 표를 정의하려면 각 열의 데이터 유형을 지정해야 합니다.

이것이 SQL에서 표를 생성하는 기본 구문임을 기억하세요. 각 열마다 고유한 데이터 유형이 있습니다.

```sql
CREATE TABLE table_name(
  column1 data_type column_constraint,
  column2 data_type column_constraint,
  column3 data_type column_constraint,
  ... etc
);
```

개발자로서 목표는 이러한 데이터 유형을 가능한 한 정확하게 선택하는 것입니다.

SQL에는 여섯 가지 인기 있는 데이터 유형 범주가 있습니다:

- 숫자형, 예를 들어 `INTEGER`, `FLOAT`, `SERIAL`, `DECIMAL` 등이 있습니다.
- 날짜 및 시간형, 예를 들어 `TIMESTAMP`, `DATE`, `TIME` 등이 있습니다.
- 문자 및 문자열형. 여기에는 `CHAR`, `VARCHAR`, `TEXT`가 포함됩니다.
- 유니코드형, `NTEXT`, `NVARCHAR` 등이 있습니다. 이는 문자 출처와 상관없이 텍스트가 올바르게 저장되고 검색되도록 보장하는 데 사용됩니다.
- 바이너리형, 이미지, 오디오, 비디오 파일과 같은 비텍스트 데이터를 저장하는 데 사용됩니다.
- 그리고 `XML`, `TABLE`과 같은 기타 다양한 데이터 유형이 있습니다.

이번 수업에서는 PostgreSQL에서 가장 널리 사용되는 일부 데이터 유형을 다룹니다. 데이터 유형은 데이터베이스 관리 시스템마다 다를 수 있지만, 일반적으로 매우 비슷합니다. 따라서 문서에서 전체 목록을 확인해야 합니다.

먼저 숫자 값을 살펴보겠습니다.

먼저 `INTEGER` 데이터 유형이 있습니다:

```sql
units_sold INTEGER
```

이 예에서는 이 데이터 유형을 `units_sold` 열에 할당합니다.

`INTEGER` 유형은 데이터베이스에서 4바이트를 사용하며 값 범위는 -2,147,483,648부터 2,147,483,647까지입니다. 공식 PostgreSQL 문서에서는 이를 "정수에 대한 일반적인 선택"이라고 설명합니다.

또한 `SMALLINT`과 `BIGINT`이 있는데, 이들은 기본적으로 `INTEGER`와 같지만 데이터베이스에서 할당된 크기 때문에 각각 더 작거나 더 큰 숫자 범위를 가집니다.

PostgreSQL에서 유용한 기능 중 하나는 `SERIAL` 키워드를 사용해 열을 생성하는 것입니다. 여기서는 `id`를 사용해 `SERIAL` 열을 생성합니다:

```sql
id SERIAL
```

진정한 데이터 유형은 아니지만, 이 열은 `INTEGER` 유형을 가지며, `NULL` 값을 허용하지 않고 행이 추가될 때 자동으로 증가하므로 고유 식별자 열을 만드는 데 매우 유용합니다.

예를 들어, 첫 번째 행은 자동으로 `id`의 `1`를 가지며, 두 번째 행은 `id`의 `2`를 가지는 식으로 각 레코드에 고유 ID를 생성합니다.

MySQL에서는 `SERIAL`에 해당하는 것이 `AUTO_INCREMENT` 속성으로, 순차적인 정수를 자동으로 생성하는 데 사용됩니다:

```sql
id INT AUTO_INCREMENT
```

서로 다른 데이터베이스 관리 시스템이 데이터베이스에서 동일한 기능을 달성하는 방법이 다를 수 있음을 알 수 있습니다.

이것들이 가장 일반적으로 사용되는 숫자 데이터 유형이지만, 때로는 텍스트나 문자 수열을 나타내야 할 때도 있습니다.

이럴 때 `VARCHAR` 데이터 유형을 사용합니다. 이 데이터 유형은 가변 문자열 길이에 사용됩니다. 소괄호 안에 최대 문자 길이를 설정할 수 있습니다:

```sql
name VARCHAR(50)
```

이 예에서는 최대 50자까지 값을 가질 수 있는 `name` 열을 정의합니다.

이 데이터 유형은 문자열의 최대 길이를 설정하지만, 문자열 길이에 제한이 없도록 저장하려면 대신 `TEXT`를 사용할 수 있습니다:

```sql
name TEXT
```

숫자와 문자열 외에도 날짜와 시간을 저장하는 것도 일반적입니다.

예를 들어, 이벤트를 저장하는 표를 만든다면 각 이벤트의 날짜를 특정 형식으로 저장해야 할 수 있습니다. 이를 위해 `DATE` 데이터 유형이 있습니다:

```sql
event_date DATE
```

시간을 저장하려면 `TIME` 데이터 유형도 사용할 수 있습니다. 예를 들어, 이벤트가 시작하는 시간을 저장할 수 있습니다:

```sql
start_time TIME
```

`TIMESTAMP` 데이터 유형은 날짜와 시간을 모두 포함합니다. `TIMESTAMP WITH TIME ZONE`를 작성하면 시간대도 포함할 수 있습니다:

```sql
event_timestamp TIMESTAMP
```

```sql
event_timestamp TIMESTAMP WITH TIME ZONE
```

마지막으로 불리언 값을 저장해야 한다면, `TRUE` 또는 `FALSE`를 위해 `BOOLEAN` 데이터 유형을 사용할 수 있습니다:

```sql
is_active BOOLEAN
```

이제 SQL에서 가장 일반적인 데이터 유형 몇 가지를 알게 되었지만, 더 많은 유형이 있습니다. 사용 중인 데이터베이스 관리 시스템의 문서를 확인해 사용할 수 있는 데이터 유형에 대해 더 알아보세요.
