# --description--

`Versions` dei pacchetti npm nella sezione dependencies del tuo file package.json seguono quello che si chiama versionamento semantico (SemVer), uno standard del settore per il versionamento del software che mira a facilitare la gestione delle dipendenze. Librerie, framework o altri strumenti pubblicati su npm dovrebbero usare SemVer per comunicare chiaramente che tipo di cambiamenti i progetti possono aspettarsi se aggiornano.

Conoscere SemVer può essere utile quando sviluppi software che usa dipendenze esterne (cosa che succede quasi sempre). Un giorno, la tua comprensione di questi numeri ti salverà dall’introdurre accidentalmente cambiamenti incompatibili nel tuo progetto senza capire perché cose che funzionavano ieri improvvisamente oggi non funzionano più. Ecco come funziona il versionamento semantico secondo il sito ufficiale:

```json
"package": "MAJOR.MINOR.PATCH"
```

La versione MAJOR dovrebbe aumentare quando fai cambiamenti incompatibili all’API. La versione MINOR dovrebbe aumentare quando aggiungi funzionalità in modo retrocompatibile. La versione PATCH dovrebbe aumentare quando fai correzioni di bug retrocompatibili. Questo significa che le PATCH sono correzioni di bug e le MINOR aggiungono nuove funzionalità ma nessuna delle due rompe ciò che funzionava prima. Infine, le MAJOR aggiungono cambiamenti che non funzioneranno con versioni precedenti.
