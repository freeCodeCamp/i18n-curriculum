# --interactive--

У сучасному JavaScript класи — це як шаблони, які ви можете визначити в коді для створення кількох об’єктів одного типу.

Наприклад, якщо ви моделюєте систему інвентаризації, вам потрібно створити кілька об’єктів для кожного типу продукту.

Кожен окремий продукт матиме свої властивості та дії, але продукти одного типу можна створити з одного й того самого «шаблону». Саме для цього використовують класи.

Класи можна визначати як вираз класу або оголошення класу.

Ось базовий синтаксис оголошення класу:

```js
class MyClassName {
  // Class Methods
  constructor() { ... }
  method1() { ... }
  method2() { ... }
  ...
}
```

Починаєте з написання ключового слова `class`, за яким іде назва класу. При іменуванні класів слід використовувати конвенцію іменування `PascalCase` замість `camelCase`.

Вміст у фігурних дужках називають тілом класу. Саме тут розміщуються конструктор і методи.

Конструктор — це спеціальний метод, який викликається автоматично, коли створюється новий об’єкт із класу. Зазвичай його використовують для ініціалізації властивостей об’єкта.

У цьому прикладі ви бачите клас `Dog` із конструктором.

```js
class Dog {
  constructor(name) {
    this.name = name;
  }
}
```

Ключове слово `this` тут використовується для посилання на поточний екземпляр класу `Dog`, що дозволяє призначити властивість `name` об’єкту, який створюється. Ви дізнаєтеся більше про ключове слово `this` у майбутньому уроці.

Під конструктором можуть бути методи. Методи — це функції, визначені всередині класу, які виконують дії або операції. Вони використовуються для визначення поведінки, яку можуть виконувати екземпляри класу.

Ось приклад додавання методу з назвою `bark` до класу `Dog`.

```js
class Dog {
  constructor(name) {
    this.name = name;
  }

  bark() {
    console.log(`${this.name} says woof!`);
  }
}
```

Коли викликається метод `bark`, повідомлення буде виведено в консоль із фактичним ім’ям, яке було передано. Якщо створити екземпляр `Dog` з ім’ям `"Buddy"`, виклик методу `bark()` на цьому екземплярі виведе в консоль `"Buddy says woof!"`.

Після визначення класу ви можете створити його екземпляр за допомогою ключового слова `new`, назви класу та його аргументів.

У цьому прикладі ми створюємо новий екземпляр `Dog` з ім’ям `Gino` і призначаємо його змінній `dog`.

```js
const dog = new Dog("Gino");
```

Коли ви пишете `new Dog("Gino")` у коді, створюється новий об’єкт, і конструктор викликається автоматично з переданими аргументами.

Потім ви можете отримати доступ до цієї властивості, використовуючи точкову нотацію, як видно в наступному рядку коду:

```js
console.log(dog.name);
```

Ось повний приклад:

:::interactive_editor

```js
class Dog {
  constructor(name) {
    this.name = name;
  }

  bark() {
    console.log(`${this.name} says woof!`);
  }
}

const dog = new Dog("Gino");
console.log(dog.name);
```

:::

Ви також можете викликати методи. У цьому прикладі ми викликаємо метод `bark()` на екземплярі `dog`.

```js
dog.bark();
```

Ось повний приклад із викликом методу:

:::interactive_editor

```js
class Dog {
  constructor(name) {
    this.name = name;
  } 
  bark() {
    console.log(`${this.name} says woof!`);
  }
}

const dog = new Dog("Gino");
console.log(dog.name);
dog.bark();
```

:::

Як уже згадувалося, класи можна також визначати як вираз класу. Це коли клас анонімний і призначений змінній.

Ось як виглядає попередній приклад у вигляді виразу класу:

```js
const Dog = class {
  constructor(name) {
    this.name = name;
  }

  bark() {
    console.log(`${this.name} says woof!`);
  }
};
```

Здебільшого ви будете використовувати оголошення класів, але розуміння виразів класів також корисне, оскільки вони можуть надати більше гнучкості, особливо в динамічних програмних ситуаціях.

Використовуючи класи та розуміючи їхній базовий синтаксис, ви можете створювати повторно використовуваний код, організовувати його в логічні одиниці та сприяти модульності коду.
