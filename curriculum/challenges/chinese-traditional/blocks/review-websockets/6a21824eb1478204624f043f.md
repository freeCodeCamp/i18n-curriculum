## WebSockets 與 HTTP 的比較

- **HTTP** 採用請求-回應模型：客戶端發送請求，伺服端回應，然後連線關閉。
- **WebSockets** 創建一個永久且雙向的連線，客戶端和伺服端之間保持開啟，直到任一方發送關閉信號。
- HTTP 非常適合載入頁面或取得資料，但在即時應用中會變得緩慢且低效。
- 每個 WebSocket 連線都從 **HTTP 握手** 開始：客戶端請求升級，伺服端回應 `101 Switching Protocols`。

### WebSocket 連線流程

1. 客戶端透過 HTTP 連接伺服端並請求升級。
2. 伺服端接受並回應 `101 Switching Protocols`。
3. 連線保持開啟。
4. 雙方可隨時交換資料，直到連線關閉。

### HTTP 與 WebSocket 通訊比較

- **HTTP**：`Client → Request → Server → Response`（一次循環，然後關閉）
- **WebSocket**：`Client ↔ Server`（持續且雙向）

### 一個有用的比喻

- **HTTP** 就像交換信件：你寫一封，等待回覆，然後對話結束。
- **WebSockets** 就像電話通話：雙方即時交談，持續所需時間。

### WebSockets 的實際應用

- 聊天應用程式
- 線上多人遊戲
- 即時體育比分更新
- 股市儀表板
- 協作工具（例如 Google Docs）

## Pub/Sub 架構

- **Pub/Sub**（發布/訂閱）是一種訊息架構，允許系統的各部分在不直接了解彼此的情況下通訊。
- 主要有三個部分：**發布者**、**主題** 和 **訂閱者**。
  - **發布者** 將訊息發送到主題。
  - **主題**（也稱為頻道或群組）充當中介者。
  - **訂閱者** 從它訂閱的主題接收訊息。

### Pub/Sub 訊息流程

1. 訂閱者訂閱一個主題。
2. 發布者將訊息發送到該主題。
3. 訊息系統將訊息傳送給該主題的所有訂閱者。

### 直接通訊與 Pub/Sub 的比較

- **沒有 Pub/Sub**：`Application A → Application B` — 緊密耦合；B 的變動可能會破壞 A。
- **有 Pub/Sub**：`Application A → Topic → Subscribers` — A 只發布，不需要知道誰接收。

### Pub/Sub 的實際應用

- 多聊天室的聊天應用
- 線上多人遊戲
- 通知系統
- 即時儀表板
- 事件驅動應用

### WebSockets 與 Pub/Sub 的比較

- **WebSockets** 提供通訊通道（道路）。
- **Pub/Sub** 提供訊息樣式，決定誰接收特定訊息（交通系統）。
- 許多即時應用同時使用兩者：WebSockets 保持連線開啟，Pub/Sub 將訊息導向正確的訂閱者。

## 在 Node.js 中使用 WebSockets

- Node.js 沒有內建 WebSocket 伺服器。`ws` 函式庫是標準的外部套件。
- 安裝指令：`npm install ws`

### 創建 WebSocket 伺服器

```js
const WebSocket = require("ws");

const server = new WebSocket.Server({ port: 3000 });
```

### WebSocket 事件

| 事件 | 觸發條件 |
|---|---|
| `connection` | 新客戶端連接到伺服器 |
| `message` | 已連接的客戶端發送資料 |
| `close` | 連線關閉 |
| `error` | 連線發生錯誤 |

### 傳送訊息給客戶端

```js
socket.send("Hello from the server");
```

### 廣播給所有已連接的客戶端

- `server.clients` 儲存所有已連接的客戶端。
- 傳送前檢查 `readyState === WebSocket.OPEN`，避免對已斷線的客戶端發生錯誤。

```js
server.on("connection", (socket) => {
  socket.on("message", (message) => {
    server.clients.forEach((client) => {
      if (client.readyState === WebSocket.OPEN) {
        client.send(message.toString());
      }
    });
  });
});
```

- **廣播** 是同時向多個已連接客戶端發送訊息的樣式 — 用於聊天應用、即時通知、體育更新和多人遊戲事件。
