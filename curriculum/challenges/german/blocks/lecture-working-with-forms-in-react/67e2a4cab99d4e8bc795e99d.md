# --description--

React 19 brachte zwei bemerkenswerte neue Funktionen mit sich, die server components und server actions genannt werden.

Ab dieser Version wurden server components in Frameworks wie Next.js, die sie direkt unterstützen, zum Standard.

Server actions hingegen sind Funktionen, die auf dem Server ausgeführt werden, um die Formularverarbeitung direkt auf dem Server zu ermöglichen, ohne dass API-Endpunkte benötigt werden.

Eine server action sieht so aus:

```js
"use server";

async function submitForm(formData) {
  const name = formData.get("name");
  return { message: `Hello, ${name}!` };
}
```

Diese server action extrahiert ein `name`-Feld aus einem Formular und gibt einen String zurück, der diesen Namen begrüßt.

Um die Zustandsverwaltung für server actions zu vereinfachen und die Notwendigkeit von clientseitigem JavaScript für einfache Formulare zu beseitigen, hat das React-Team in Version 19 den `useActionState`-Hook eingeführt.

Werfen wir einen genaueren Blick auf diesen Hook und sehen wir, wie er funktioniert.

Die React-Dokumentation beschreibt den `useActionState`-Hook als einen Hook, der „es Ihnen ermöglicht, den Zustand basierend auf dem Ergebnis einer Formularaktion zu aktualisieren.“

Das bedeutet aber nicht, dass Sie den `useActionState`-Hook nur mit Formularen verwenden können. Sie können ihn auch nutzen, um Button-Klicks und andere Ereignisse zu verwalten, solange eine Aktion vorhanden ist.

Und bedenken Sie, dass Sie den `useActionState` als Hook nicht innerhalb einer server component verwenden können.

Hier ist die grundlegende Syntax des `useActionState`-Hooks:

```js
const [state, action, isPending] = useActionState(actionFunction, initialState, permalink);
```

- `state` ist der aktuelle Zustand, den die Aktion zurückgibt.

- `action` ist die Funktion, die die server action auslöst.

- `isPending` ist ein Boolean, der anzeigt, ob die Aktion gerade ausgeführt wird oder nicht.

- Der `actionFunction`-Parameter ist die server action selbst.

- `initialState` ist der Parameter, der den Ausgangszustand vor Ausführung der Aktion darstellt.

- `permalink` ist ein optionaler String, der die eindeutige Seiten-URL enthält, die das Formular ändert.

Um den `useActionState`-Hook zu verwenden, stellen Sie zuerst sicher, dass eine Aktion vorhanden ist. Nutzen wir dafür die Aktion aus dem vorherigen Beispiel mit einer kleinen Änderung:

```js
"use server";

export async function submitForm(_, formData) {
  const name = formData.get("name");

  const hour = new Date().getHours();
  let greeting;

  if (hour < 12) {
    greeting = "Good morning";
  } else if (hour < 18) {
    greeting = "Good afternoon";
  } else {
    greeting = "Good evening";
  }

  return { message: `${greeting}, ${name}` };
}
```

In Ihrer Komponente müssen Sie dann den `useActionState`-Hook importieren und ihn auf oberster Ebene des Komponentenrumpfs (vor der return-Anweisung) aufrufen, genau wie andere Hooks. Sie sollten auch die Aktion importieren:

```jsx
"use client";

// Import the useActionState hook
import { useActionState } from "react";

// Import the submitForm action
import { submitForm } from "./actions/submitForm";

const Greeter = () => {

 // Initialize the hook
 const [state, submit, isPending] = useActionState(submitForm, {
   message: "",
 });

  return (
    <div className="flex flex-col items-center justify-center min-h-screen bg-gray-100 p-6">
      {/* Rest of component */}
    </div>
  );
};

export default Greeter;
```

So sieht der vollständige Code mit etwas Styling aus:

```jsx
"use client";

import { useActionState } from "react";
import { submitForm } from "./actions/submitForm";

const Greeter = () => {
  const [state, submit, isPending] = useActionState(submitForm, {
    message: "",
  });

  return (
    <div className="flex flex-col items-center justify-center min-h-screen bg-gray-100 p-6">
      <form
        action={submit}
        className="bg-white p-6 rounded-2xl shadow-md w-full max-w-md"
      >
        <h2 className="text-2xl text-center font-semibold text-gray-700 mb-4">
          Greet Someone
        </h2>

        <input
          type="text"
          name="name"
          placeholder="Enter your name"
          required
          className="w-full p-3 border border-gray-300 rounded-lg focus:outline-none focus:ring-2 focus:ring-green-400"
        />

        <button
          type="submit"
          disabled={isPending}
          className="w-full mt-4 p-3 bg-green-500 text-white font-semibold rounded-lg hover:bg-green-600 disabled:bg-gray-400 transition-all"
        >
          {isPending ? "Greeting..." : "Greet"}
        </button>

        {state.message && (
          <p className="mt-4 text-green-600 text-center font-medium">
            {state.message}
          </p>
        )}
      </form>
    </div>
  );
};

export default Greeter;
```

Im Browser würden Sie sehen, wie sich die Schaltfläche Ihres Formulars von `Greet` zu `Greeting...` ändert, während die Aktion `isPending` ausgeführt wird – und die Begrüßung würde `Good morning, {name}`, `Good afternoon, {name}` oder `Good evening, {name}` anzeigen, je nachdem, zu welcher Tageszeit das Formular abgesendet wurde.

Erinnern Sie sich daran, dass Sie den `useActionState`-Hook auch außerhalb eines Formulars verwenden können?

In diesem Beispiel holen wir fünf Benutzer von JSONPlaceholder mit einem Button-Klick:

```js
"use server";

export async function getUsers() {
  const res = await fetch(
    "https://jsonplaceholder.typicode.com/users?_start=0&_limit=5/"
  );
  return await res.json();
}
```

So sieht die gestylte Benutzeroberfläche aus:

```jsx
"use client";

import { useActionState } from "react";
import { getUsers } from "./actions/getUsers";

export default function FetchUsers() {
  const [users, fetchAction, isPending] = useActionState(getUsers, []);

  return (
    <div className="p-6 max-w-lg mx-auto">
      <button
        onClick={fetchAction}
        disabled={isPending}
        className="px-4 py-2 cursor-pointer bg-green-500 text-white rounded-lg hover:bg-green-600 disabled:bg-gray-400 font-bold"
      >
        {isPending ? "Fetching Users..." : "Fetch Users"}
      </button>

      <ul className="mt-4 space-y-2">
        {users.map((user) => (
          <li key={user.id} className="p-3 bg-gray-100 rounded-lg">
            <p className="font-semibold">{user.name}</p>
            <p className="text-sm text-gray-600">{user.email}</p>
          </li>
        ))}
      </ul>
    </div>
  );
}
```

Im Browser würden Sie sehen, dass der Text des Buttons nach dem Klicken niemals auf `Fetching Users...` aktualisiert wird.

Das passiert, weil React das Datenabrufen und Rendern als höher priorisiert als den `isPending`-Zustand behandelt, was `isPending` blockiert und dabei einen Fehler wirft.

Um dieses Problem zu beheben, müssen Sie die Aktion in `startTransition` einhüllen:

```jsx
"use client";

// import startTransition from React
import { useActionState, startTransition } from "react";
import { getUsers } from "./actions/getUsers";

export default function FetchUsers() {
  const [users, fetchAction, isPending] = useActionState(getUsers, []);

  return (
    <div className="p-6 max-w-lg mx-auto">
      <button
        {/* wrap fetchAction in startTransition */}
        onClick={() => startTransition(() => fetchAction())}
        disabled={isPending}
        className="px-4 py-2 bg-green-500 font-bold cursor-pointer text-white rounded-lg hover:bg-green-600 disabled:bg-gray-400"
      >
        {isPending ? 'Fetching Users...' : 'Fetch Users'}
      </button>

      <ul className="mt-4 space-y-2">
        {users.map((user) => (
          <li key={user.id} className="p-3 bg-gray-100 rounded-lg">
            <p className="font-semibold">{user.name}</p>
            <p className="text-sm text-gray-600">{user.email}</p>
          </li>
        ))}
      </ul>
    </div>
  );
}
```

Falls Sie sich fragen, was `startTransition` ist: Es ist eine Funktion, die React mitteilt, dass ein Zustandsupdate eine niedrige Priorität hat und unterbrochen werden kann. So bleibt die Benutzeroberfläche responsiv, während asynchrone Updates wie server actions verarbeitet werden.

So verwenden Sie den `useActionState`-Hook innerhalb und außerhalb eines Formulars.

## --text--

Was ist der Zweck des `useActionState`-Hooks in React?

## --answers--

Er verwaltet den Zustand basierend auf dem Ergebnis einer Aktion, einschließlich Formularabsendungen, Button-Klicks und anderen ereignisgesteuerten Aktualisierungen.

---

Er aktualisiert den Zustand nur, wenn ein Formular abgesendet wird, und unterstützt keine anderen Arten von Aktionen wie Button-Klicks oder benutzerdefinierte Ereignishandler.

### --feedback--

Er funktioniert mit Aktionen, nicht nur mit Formularen.

---

Er übernimmt die globale Zustandsverwaltung über mehrere Komponenten hinweg und ersetzt damit Zustandsverwaltungsbibliotheken wie Redux oder Zustand.

### --feedback--

Er funktioniert mit Aktionen, nicht nur mit Formularen.

---

Er ersetzt den `useState`-Hook vollständig, indem er eine eingebaute Möglichkeit bietet, sowohl lokalen als auch globalen Zustand ohne zusätzliche Abhängigkeiten zu verwalten.
