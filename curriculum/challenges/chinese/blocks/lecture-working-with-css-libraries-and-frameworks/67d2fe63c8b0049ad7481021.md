# --description--

CSS 预处理器是一种工具，它通过变量、混入、嵌套和选择器继承等强大特色扩展了标准 CSS。 

其中一些特色，比如变量和嵌套，现在在本地 CSS 中已经被支持或正在获得更多支持，这一趋势很可能会继续。然而，早期版本的本地 CSS 并不支持这些特色。

这就是为什么 CSS 预处理器最初会被广泛使用的原因。它们将你编写的 CSS 规则和样式，以及它们扩展的语法，编译成浏览器可以理解的本地的 CSS 文件。

通过使用 CSS 预处理器，你可以以更可重用和逻辑化的方式构建你的 CSS 代码。你还将能够访问强大的特色，例如 CSS 不直接支持的 mixins。

一些最流行的 CSS 预处理器是 Sass、Less 和 Stylus。

让我们谈谈 Sass。

Sass 代表 “Syntactically Awesome Style Sheets”。它兼容所有版本的 CSS，并由一个庞大的开发者社区维护。

Sass 支持如下特色：

- 变量，这样你就可以在整个电子表格中保存和重复使用值。

- 嵌套的 CSS 规则，使你可以在你的 `file` 中创建视觉层次结构。

- 模块，这样你就可以将样式拆分到多个样式表中。

- Mixin，这样你就可以在整个网站中重用 CSS 声明。

- 继承，因此多个 CSS 选择器可以共享属性。

- 和操作符，用于基本数学操作。

Sass 也支持两种语法。

SCSS 语法代表 Sassy CSS，它是 CSS 的超集。这意味着 SCSS 扩展了 CSS 的基本语法。

SCSS 是你在使用 Sass 时最常用和最常见的语法。它要求在 CSS 属性周围使用括弧状的花括弧（`{` 和 `}`），并且在 CSS 声明末尾使用分号（`;`），就像本地 CSS 一样。

这些文件具有 `.scss` 扩展名。以下是一个示例：

```scss
$primary-color: #3498eb;

header {
  background-color: $primary-color;
}
```

你在顶部看到的是在 SCSS 中定义的变量。这个变量用于下面的 CSS 规则中。

让我们将 SCSS 与一种较少使用的语法——缩排语法进行比较。这也被称为 `"Sass syntax"`，因为它是 Sass 的原始语法。

此语法依赖缩排来定义规则。以下是一个示例：

```scss
$primary-color: #3498eb

header
  background-color: $primary-color
```

注意 CSS 规则周围没有括弧状的花括弧，CSS 声明末尾也没有分号。在顶部，你还可以看到一个变量。注意它的名称在这种语法中以美元符号（`$`）开头，这与本地的 CSS 变量不同，但工作方式非常相似。

Sass 还支持一个强大的特色，称为 mixins。 

Mixin 允许你将多个 CSS 属性及其值组合在一个名称下，并在整个样式表中重用该块 CSS 代码。

这使你的 CSS 代码不那么重复且更易于维护，因为如果你更改该块中的内容，更改会应用到你使用该 mixin 的所有地方。

由于 mixins 可以有自定义且描述性的名称，它们有助于理解每个 CSS 代码块的作用，从而使你的样式表更易于理解。

并且 mixins 通过应用统一的样式促进样式表的一致性。

这是 SCSS 语法中 mixin 的一个示例：

```scss
@mixin center-flex {
  display: flex;
  justify-content: center;
  align-items: center;
}
```

要定义一个 mixin，首先编写 `@mixin` 规则，然后是 mixin 的名称。在这个例子中，mixin 名称是 `center-flex`。它有三个 CSS 属性，用于使用 Flexbox 居中元素。

然后，一旦你定义了你的 mixin，你就可以使用 `@include` 规则将那些属性包含到 CSS 规则中。你只需写 `@include`，后面跟上 mixin 的名称。在这个例子中，名称是 `center-flex`：

```scss
section {
  @include center-flex;
  height: 500px;
  background-color: #3289a8;
}
```

这是包含 mixin 和 CSS 规则的完整代码：

```scss
@mixin center-flex {
  display: flex;
  justify-content: center;
  align-items: center;
}

section {
  @include center-flex;
  height: 500px;
  background-color: #3289a8;
}
```

你可以在任意多个 CSS 规则中包含该 mixin。如果你需要进行任何更改，只需更改 mixin 本身，所有地方都会自动应用这些更改。此示例使用 SCSS 语法。注意它包含括弧状的花括弧和分号。

以下是等效的缩排语法，也称为 Sass 语法，没有括弧状的花括弧或分号：

```scss
@mixin center-flex
  display: flex
  justify-content: center
  align-items: center

section
  @include center-flex
  height: 500px
  background-color: #3289a8
```

这就是编译后的 CSS 代码，浏览器在文件被编译成本地 CSS 后实际会解释的内容：

```css
section {
  display: flex;
  justify-content: center;
  align-items: center;
  height: 500px;
  background-color: #3289a8;
}
```

请注意，它包含顶部的三个混入属性：`display`、`justify-content` 和 `align-items`。

这只是 CSS 预处理器功能的一个简单示例。它们有许多强大的特色，值得你在日常工作流程中学习和应用。

但是，它们确实有你应该注意的潜在缺点。

首先，CSS 预处理器必须将 CSS 规则编译成标准 CSS。虽然这个编译过程的潜在额外开销在时间和资源方面通常很小，但你应该在实际项目中考虑这一点。

接下来，他们可以创建调试挑战，因为浏览器直接使用编译后的 CSS。与标准 CSS 相比，查找由扩展语法生成的问题样式可能需要额外的几个步骤。

然而，CSS 预处理器的优点通常超过其缺点，尤其是对于复杂的项目。它们对于编写更简洁、可重用、减少重复且可扩展的 CSS 非常有帮助。
