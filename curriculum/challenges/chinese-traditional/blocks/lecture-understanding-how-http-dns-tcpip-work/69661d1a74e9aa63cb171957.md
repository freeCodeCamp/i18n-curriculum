# --description--

當你造訪網站時，第一件事就是在瀏覽器中輸入網頁位址。

例如，要前往 freeCodeCamp，你會在瀏覽器的位址列輸入 `freecodecamp.org`。

這稱為網域名稱。它是在網際網路上該資源的唯一可供人類閱讀的識別字。

網域名稱與 DNS 這個本課程的主要主題密切相關。讓我們先談談網域名稱階層體系，這樣你就能了解 DNS 在幕後如何運作。

網域名稱階層體系是從右到左讀取，從最一般的類別移動到最具體的類別。

網域名稱的主要結構是從右到左：

* **頂層網域（TLD）：** 網域名稱的最後一段，位於最右側。以 `freecodecamp.org` 為例，該段為 `.org`。其他範例包括 `.com` 和 `.edu`。
    
* **第二層網域 (SLD)：** 由個人或組織註冊的名稱。在 `freecodecamp.org` 中即為 `freecodecamp`。這通常是你在看到網路位址時最先注意到的部分。
    
* **第三層級網域（子網域）：** 用來識別主網域的特定區段。例如，`www`，代表「World Wide Web」。現今，現代網頁伺服器會自動控制代碼網域路由，因此你可以輸入或不輸入 `www` 來存取網站。其他子網域的例子包括 `blog.`、`shop.`、`support.` 和 `api.`。
    

網域名稱對我們人類非常有幫助，因為它們容易記憶且輸入到瀏覽器中。然而，如你所知，電腦是以數字運作的。你的瀏覽器必須將你輸入的網域名稱轉換成稱為 IP 位址的東西，以找到你正在尋找的網站。

**IP 位址** 是一組獨特的數字序列，用來識別網路上（例如網際網路）的電腦。

IP 位址對於讓客戶端將請求傳送到正確的伺服器並獲得適當的回應是必不可少的。IPv4 位址的範例是 `127.0.0.1`。在一個稱為 IPv6 的較新標準中，等同的 IP 位址是 `0:0:0:0:0:0:0:1`。

關於 IP 位址有很多需要學習的地方，隨著你繼續學習，你可能會想更深入地探索它們。但目前，你只需要了解它們是用來識別網路上電腦的唯一數字位址。

從網域名稱和 IP 位址的概念出發，現在我們可以達到 DNS 的概念。

DNS 代表 **Domain Name System**。這通常被稱為網際網路的「電話簿」。

DNS 將你在瀏覽器中輸入的網域名稱轉換為其對應的 IP 位址。它是一個分層且去中心化的命名系統，用於連接到網際網路或特定網路的電腦和裝置。

當使用者在瀏覽器中輸入網域名稱，例如 `freecodecamp.org`，DNS 會將該名稱對應到其相應伺服端的 IP 位址。一旦伺服端被識別，客戶端的請求就會被送到該伺服端。伺服端根據其程式設計來處理該請求，並最終將回應傳送給客戶端。這完成了請求-回應循環。

這一切都要歸功於 DNS 執行了最初的轉換，將其轉換為標的 IP 位址。

讓我們深入了解 DNS 決議過程的細節。

## DNS 決議過程

* DNS 決議過程是透過將查詢傳送到一串鏈的 DNS 伺服器來運作。

* 當你（使用者）在瀏覽器輸入網域名稱時，**客端**會向區域的 DNS 伺服器發送請求。這個區域的 DNS 伺服器稱為 **Recursive Resolver**。這通常由你的網際網路服務提供者（ISP）管理。
    
* 遞迴解析器會將該請求轉發給**Root Servers**。這些是一般伺服器，沒有關於 IP 位址的直接資訊。它們的運作方式是將遞迴解析器導向更特定的伺服器，特別是該請求所屬頂層網域的頂層網域（TLD）伺服器。例如，負責所有 `.org` 網域的伺服器。
    
* 遞迴解析器接著查詢**頂層網域（TLD）伺服器**。此伺服器傳回持有所請求網域（例如 `freecodecamp.org`）記錄的特定權威名稱伺服器的位址。
    
* 遞迴解析器會向對應的 **Authoritative Name Server** 發出最終查詢。如果找到 **DNS record**，則會傳回給遞迴解析器。
    
* 當 Recursive Resolver 收到 IP Address 時，它會將其傳送給客戶端，讓你可以開始瀏覽網站。

權威名稱伺服器傳回的是 **DNS Records**，而不僅僅是 IP 位址。DNS Records 是包含重要資訊的結構化文字條目，關於一個網域，例如子網域資訊、資訊應該快取多久，以及執行所請求服務所需的資訊。

為了最佳化處理程序，Recursive Resolver 臨時快取 DNS Record 以加快處理程序。這樣，你就不會在短時間內重複發出相同的請求。這使整體處理程序更加高效。

如你所見，DNS 伺服器是不可或缺的，因為它們提供將網域名稱轉換為 IP 位址的初始且關鍵步驟。這個轉換過程讓每個客戶端能夠定位並連接到正確的伺服器。若沒有這個目錄系統，作為現代網路骨幹的客戶端/伺服器架構將無法運作。
