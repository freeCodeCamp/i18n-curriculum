# --description--

Katika Express, **njia za data** ni mifumo ya URL inayobainisha jinsi seva inavyotumia majibu kwa maombi yanayoingia. Wakati mtumizi anapouliza URL fulani, Express huoanisha URL hiyo na njia ya data na kuendesha mshughulikiaji husika.

Njia za data huainishwa ndani ya njia za data kama `app.get()`, `app.post()`, na kadhalika. Njia hizi zinaweza kuwa za kudumu, kama `/home`, au za mabadiliko, ambapo vigezo hutumika kunasa thamani kutoka kwenye URL, kama `/user/:id`.

Katika funzo hili tutazingatia njia za data za kudumu. Hapa kuna mfano mmoja:

```javascript
// Static route
app.get("/home", (req, res) => {
  res.send("Welcome to the homepage!");
});
```

Katika mfano huu, seva husikiliza maombi ya GET kwa `/home` na hutoa jibu la ujumbe rahisi.

Njia ya `app.get()` inachukua hoja mbili:

* Hoja ya kwanza, `"/home"`, ni njia ya data ya kudumu (au URL) ambayo seva husikiliza.
    
* Hoja ya pili ni kitendakazi cha mwitiko kinachoendeshwa wakati ombi la GET linapotumwa kwa njia hiyo.
    

Ndani ya mwitiko, `res.send()` hutuma jibu kurudi kwa mteja. Katika kesi hii, hutuma maandishi `Welcome to the homepage!`, ambayo yanaonyeshwa kwenye kivinjari.

Hapa kuna mfano mwingine wa njia ya data ya kudumu:

```javascript
// Static route
app.get("/menu/drinks", (req, res) => {
  res.send("Welcome to the drinks menu!");
});
```

Katika kesi hii, seva husikiliza maombi ya GET kwa URL ya `/menu/drinks` na hutuma ujumbe kurudi wakati njia hiyo ya data inapoombwa.

Unaweza kutumia njia za data kama hizi wakati tovuti yako ina kurasa nyingi zinazohusiana zilizokusanywa chini ya njia ya kawaida. Kwa mfano, tovuti ya mgahawa inaweza kupanga kurasa zake za menyu kama hizi:

* `/menu/food`
    
* `/menu/drinks`
    
* `/menu/desserts`
    

Njia hii husaidia kuweka URL zako wazi na zenye maana, na pia hufanya njia zako za data ziwe rahisi kupanga na kudumisha wakati programu yako inakua.

Katika funzo lijalo, tutaangalia njia za data za mabadiliko zenye vigezo.
