# --description--

提供された `index.pug` ファイルにはログインフォームがあります。インライン JavaScript の `if showLogin` によってフォームが隠されており、その後にフォームがインデントされています。

そのページの `res.render` に、新しい変数 `showLogin: true` をオブジェクトに追加してください。ページを更新するとフォームが表示されるはずです！このフォームは `/login` に対して **POST** するように設定されています。したがって、ここで POST リクエストを受け入れてユーザーを認証する設定を行う必要があります。

このチャレンジでは、POST リクエストを受け入れるルート `/login` を追加してください。このルートで認証するには、レスポンスを送る前に認証用のミドルウェアを追加する必要があります。これは、レスポンスの前にミドルウェアを引数として渡すだけで実現できます。使用するミドルウェアは `passport.authenticate('local')` です。

`passport.authenticate` は `{ failureRedirect: '/' }` のようなオプションを引数として受け取ることもでき、とても便利なので必ず追加してください。ミドルウェアを使った後（認証ミドルウェアが成功した場合のみ呼ばれます）、ユーザーを `/profile` にリダイレクトするレスポンスを追加してください。そのルートも追加し、ビュー `profile.pug` をレンダリングするようにしてください。

認証が成功すると、ユーザーオブジェクトは `req.user` に保存されます。

この時点で、フォームにユーザー名とパスワードを入力するとホームページ `/` にリダイレクトされ、サーバーのコンソールには `'User {USERNAME} attempted to log in.'` が表示されるはずです。現在は登録されていないユーザーはログインできないためです。

正しくできたと思ったらページを提出してください。エラーが出る場合は、<a href="https://forum.freecodecamp.org/t/advanced-node-and-express/567135#how-to-use-passport-strategies-7" target="_blank" rel="noopener noreferrer nofollow">ここまで完成したプロジェクトを確認できます</a>。

`/login` への POST リクエストは `/` に正しくリダイレクトされる必要があります。

```js
  const url = new URL("/login", code);
  const res = await fetch(url, { method: 'POST' });
  const data = await res.text();
  assert.match(
    data,
    /Looks like this page is being rendered from Pug into HTML!/,
    'A login attempt at this point should redirect to the homepage since we do not have any registered users'
  );
```
