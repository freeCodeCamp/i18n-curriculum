# --interactive--

L’operatore di concatenamento opzionale (`?.`) è uno strumento utile in JavaScript che ti permette di consultare in sicurezza le proprietà di un oggetto o chiamare metodi senza preoccuparti se esistono o meno. È come una rete di sicurezza per lavorare con oggetti che potrebbero avere parti mancanti.

:::interactive_editor

```js
const person = {
  name: "Alice",
  age: 30
};

console.log(person.name); // "Alice"
console.log(person.job); // undefined
```

:::

In questo esempio, `person.name` esiste, quindi stampa `Alice`. Ma `person.job` non esiste, quindi ci restituisce `undefined`.

Ora, supponiamo di voler consultare una proprietà di un oggetto che potrebbe non esistere:

```js
const person = {
  name: "Alice",
  age: 30
};

console.log(person.address.street); // This will throw an error!
```

Questo esempio lancerà un `Uncaught TypeError`. Poiché `person.address` è `undefined`, non possiamo consultare la proprietà `street`. Qui entra in gioco l’operatore di concatenamento opzionale. Ecco un esempio di come usarlo:

:::interactive_editor

```ts
const user = {
  name: "John",
  profile: {
    email: "john@example.com",
    address: {
      street: "123 Main St",
      city: "Somewhere"
    }
  }
};

console.log(user?.profile?.address?.street); // "123 Main St"
console.log(user?.profile?.phone?.number);   // undefined
```

:::

Usando l’operatore di concatenamento opzionale, diciamo a JavaScript di continuare l’operazione solo se l’oggetto (o il valore prima del `?.`) esiste e non è `null` o `undefined`.

Se il valore prima del `?.` è `null` o `undefined`, JavaScript restituisce `undefined` invece di tentare di procedere con l’operazione e lanciare un errore.

Ricorda, l’operatore di concatenamento opzionale è più utile quando non sei sicuro che una proprietà o un metodo esista. Aiuta a prevenire errori e rende il tuo codice più robusto.
