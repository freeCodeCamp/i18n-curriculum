# --description--

Express 提供了一系列内置中间件函数，帮助执行常见任务，比如提供静态文件服务、解析 JSON 请求体等。你也可以使用第三方中间件来添加功能。

**以下是一些内置中间件的示例**：

1. `express.json()`：一个内置中间件，用于解析传入的 JSON 负载。

```js
app.use(express.json())
```

1. `express.static()`：用于提供静态文件服务的中间件，比如图像、CSS 和 JavaScript 文件。

```js
app.use(express.static('public'))
```

1. `express.urlencoded()`：一个解析传入 URL 编码负载的中间件，这种格式通常由超文本标记语言表单发送。它会将解析后的数据放在 `req.body` 上。

```js
app.use(express.urlencoded({ extended: true }))
```

**现在，以下是一些第三方中间件的示例：**

1. `cors`：一个流行的第三方中间件，用于启用跨域资源共享。

```bash
npm install cors
```

```js
const cors = require('cors')
app.use(cors())
```

1. `morgan`：一个用于 HTTP 请求的日志中间件。

```bash
npm install morgan
```

```js
const morgan = require('morgan')
app.use(morgan('tiny'))
```

现在，让我们看一个同时使用内置和第三方中间件的示例：

```js
const express = require('express')
const cors = require('cors')
const morgan = require('morgan')
const app = express()

app.use(express.json()) // Built-in middleware to parse JSON
app.use(cors())       // Third-party middleware for CORS
app.use(morgan('tiny')) // Third-party middleware for logging

app.get('/', (req, res) => {
  res.send('Hello, World!')
})

app.listen(3000, () => {
  console.log('Server running on http://localhost:3000')
})
```

* 内置中间件 `express.json()` 解析传入的 JSON 负载。
* 第三方中间件 `cors` 允许跨域请求，`morgan` 记录传入的 HTTP 请求。

需要注意的是，中间件函数是按照调用 `app.use()` 的顺序添加的。

Express 提供了处理常见任务（如 JSON 解析和静态文件服务）的核心内置中间件。你也可以通过第三方中间件扩展功能，获得更多能力。
