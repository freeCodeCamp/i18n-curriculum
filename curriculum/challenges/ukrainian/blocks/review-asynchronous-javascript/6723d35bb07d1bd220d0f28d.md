# --description--

- **Синхронний JavaScript** виконується послідовно і чекає на завершення попередньої операції перед тим, як перейти до наступної.
- **Асинхронний JavaScript** дозволяє виконувати кілька операцій у фоновому режимі, не блокуючи головний потік.
- **Потік** — це послідовність інструкцій, які можуть виконуватися незалежно від основного потоку програми.
- **Функції зворотного виклику** — це функції, які передаються як аргументи іншим функціям і виконуються після завершення операції або в результаті події.

## JavaScript-рушій і середовище виконання JavaScript

- **JavaScript-рушій** — це програма, яка виконує JavaScript-код у веббраузері. Він працює як конвертер, який бере ваш код, перетворює його на інструкції, зрозумілі комп’ютеру, і виконує їх.
- V8 — приклад JavaScript-рушія, розробленого Google.
- **Середовище виконання JavaScript** — це середовище, у якому виконується JavaScript-код. Воно включає JavaScript-рушій, який обробляє та виконує код, а також додаткові функції, як-от веббраузер або Node.js.

## Fetch API

- Fetch API дозволяє вебзастосункам робити мережеві запити, зазвичай для отримання або відправлення даних на сервер. Він надає `fetch()` метод, який ви можете використовувати для цих запитів.
- За допомогою Fetch API можна отримувати текст, зображення, аудіо, JSON та інші типи даних.

## HTTP-методи для Fetch API

Fetch API підтримує різні HTTP-методи для взаємодії з сервером. Найпоширеніші методи:

- **GET**: Використовується для отримання даних із сервера. За замовчуванням Fetch API використовує метод `GET` для отримання даних.

```js
fetch('https://api.example.com/data')
```

Щоб використати отримані дані, їх потрібно конвертувати у формат JSON за допомогою методу `.json()`:

```js
fetch('https://api.example.com/data')
  .then(response => response.json())
  .then(data => console.log(data))
```

У цьому коді відповідь, що надходить від Fetch API, є промісом, а обробник `.then` конвертує відповідь у формат JSON.

- **POST**: Використовується для відправлення даних на сервер. Метод `POST` використовується для створення нових ресурсів на сервері.

```js
fetch('https://api.example.com/users', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({
    name: 'John Doe',
    email: 'john@example.com'
  })
})
```

У цьому прикладі ми відправляємо запит `POST` для створення нового користувача. Ми вказали метод `POST`, встановили відповідні заголовки та додали тіло з даними, які хочемо відправити. Тіло має бути рядком, тому ми використовуємо `JSON.stringify()` для конвертації об’єкта у JSON-рядок.

- **PUT**: Використовується для оновлення даних на сервері. Метод `PUT` використовується для оновлення існуючих ресурсів на сервері.

```js
fetch('https://api.example.com/users/45', {
  method: 'PUT',
  headers: {
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({
    name: 'John Smith',
    email: 'john@example.com'
  })
})
```

У цьому прикладі ми оновлюємо ID `45`, який вказаний у кінці URL. Ми використали метод `PUT` у коді та також вказали дані у тілі, які будуть використані для оновлення ідентифікованих даних.

- **DELETE**: Використовується для видалення даних на сервері. Метод `DELETE` використовується для видалення ресурсів на сервері.

```js
fetch('https://api.example.com/users/45', {
  method: 'DELETE'
})
```

У цьому прикладі ми відправляємо запит `DELETE` для видалення користувача з ID `45`.

## Проміси та ланцюжки промісів

- **Проміси** — це об’єкти, які представляють майбутнє завершення або невдачу асинхронної операції та її результативне значення. Значення промісу відоме лише після завершення `async` операції.
- Ось приклад створення простого промісу:

```js
const promise = new Promise((resolve, reject) => {
  setTimeout(() => {
    resolve('Data received successfully');
  }, 2000);
});
```

- Метод `.then()` використовується в промісі, щоб вказати, що має статися, коли проміс буде виконано, а `.catch()` — для обробки будь-яких помилок.
- Ось приклад використання `.then()` і `.catch()` з промісом:

```js
promise
  .then(data => {
    console.log(data);
  })
  .catch(error => {
    console.error(error);
  });
```

У наведеному прикладі метод `.then()` використовується для виведення даних, отриманих із промісу, а метод `.catch()` — для виведення помилок, якщо вони виникають.

- **Ланцюжки промісів**: Одна з потужних функцій промісів — можливість ланцюжити кілька асинхронних операцій. Кожен `.then()` може повертати новий проміс, що дозволяє виконувати послідовність асинхронних операцій одна за одною.
- Ось приклад ланцюжка промісів:

```js
fetch('https://api.example.com/data')
  .then(response => response.json())
  .then(data => {
    console.log(data);
    return fetch('https://api.example.com/other-data');
  })
  .then(response => response.json())
  .then(otherData => {
    console.log(otherData);
  })
  .catch(error => {
    console.error(error);
  });
```

У наведеному прикладі ми спочатку отримуємо дані з одного URL, потім отримуємо дані з іншого URL на основі першої відповіді і нарешті виводимо другі отримані дані.

Метод `catch` обробляє будь-які помилки, що виникають під час процесу. Це означає, що вам не потрібно додавати обробку помилок на кожному кроці, що значно спрощує код.

## Використання `async/await` для обробки промісів

Async/await робить написання та читання асинхронного коду простішим, він побудований на основі промісів.

- **async**: Ключове слово `async` використовується для визначення асинхронної функції. Асинхронна функція `async` повертає проміс, який розв’язується зі значенням, поверненим `async` функцією.
- **await**: Ключове слово `await` використовується всередині `async` функції, щоб призупинити виконання функції, доки проміс не буде розв’язано. Його можна використовувати лише всередині `async` функції.
- Ось приклад використання `async/await`:

```js
async function delayedGreeting(name) {
  console.log("A Messenger entered the chat...");
  await new Promise(resolve => setTimeout(resolve, 2000));
  console.log(`Hello, ${name}!`);
}

delayedGreeting("Alice");
console.log("First Printed Message!");
```

У наведеному прикладі функція `delayedGreeting` є `async` функцією, яка призупиняється на 2 секунди перед виведенням привітального повідомлення. Ключове слово `await` використовується, щоб призупинити виконання функції, доки `Promise` не буде розв’язано.

- Однією з найбільших переваг `async/await` є обробка помилок за допомогою блоків `try/catch`. Ось приклад:

```js
async function fetchData() {
  try {
    const response = await fetch('https://api.example.com/data');
    const data = await response.json();
    console.log(data);
  } catch (error) {
    console.error(error);
  }
}

fetchData();
```

У наведеному прикладі блок `try` містить код, який може викликати помилку, а блок `catch` обробляє помилку, якщо вона виникає. Це робить обробку помилок більш простою та зрозумілою.

## Атрибут `async`

- Атрибут `async` повідомляє браузеру скачати файл скрипта асинхронно, продовжуючи парсинг HTML-документа.
- Після того, як скрипт завантажено, парсинг HTML призупиняється, скрипт виконується, а потім парсинг HTML відновлюється.
- Використовуйте `async` для незалежних скриптів, де порядок виконання не має значення.

## Атрибут `defer`

- Атрибут `defer` також завантажує скрипт асинхронно, але відкладає виконання скрипта до повного парсингу HTML-документа.
- Скрипти `defer` зберігають порядок виконання відповідно до їх розташування в HTML-документі.

- Важливо зазначити, що атрибути `async` і `defer` ігноруються для рядкових скриптів і працюють лише для зовнішніх файлів скриптів.

- Якщо присутні обидва атрибути `async` і `defer`, перевага надається атрибуту `async`.

## Geolocation API

- Geolocation API надає спосіб для вебсайтів запитувати місцезнаходження користувача.

- Наведений нижче приклад демонструє метод `getCurrentPosition()` API, який використовується для отримання поточного місцезнаходження користувача.

```js
navigator.geolocation.getCurrentPosition(
  (position) => {
    console.log("Latitude: " + position.coords.latitude);
    console.log("Longitude: " + position.coords.longitude);
  },
  (error) => {
    console.log("Error: " + error.message);
  }
);
```

У цьому коді ми викликаємо `getCurrentPosition` і передаємо їй функцію, яка буде викликана, коли позиція буде успішно отримана.

Об’єкт `position` містить різноманітну інформацію, але тут ми вибрали лише `latitude` і `longitude`.

Якщо виникає проблема з отриманням `position`, помилка буде виведена в консоль.

- Важливо поважати приватність користувача і запитувати його місцезнаходження лише тоді, коли це необхідно.
