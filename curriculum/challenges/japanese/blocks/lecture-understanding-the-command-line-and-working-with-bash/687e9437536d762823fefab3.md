# --description--

生産性を上げるために使えるコマンドラインのショートカットについて学びましょう。

ターミナルやコマンドラインのショートカットは、作業の効率化や生産性の最大化に役立ちます。LinuxとmacOSはどちらもUnixにルーツがあるため、多くのショートカットは共通しています。しかしWindowsではいくつか違いがあります。

最初のショートカットは上矢印キーです。このキーを押すと、コマンド履歴を1つずつ遡っていけます。下矢印キーは逆にコマンド履歴を順に進めるのに使います。この2つのキーで、以前に実行したコマンドを素早く見つけて再実行できます。

多くのシェルにはオートコンプリート機能もあり、入力を始めたコマンドに基づいて候補を提案します。Tabキーを使うと提案された内容を補完し、コマンドラインに完全な構文を素早く入力できます。提案の生成方法はシェルによって異なります。例えばzshでは最近のコマンド履歴に基づいていますが、PowerShellでは利用可能なコマンド、変数、引数に基づいており、最近の使用頻度はあまり重視されません。

時にはコマンドの出力が大量になり、ターミナル画面が見づらくなることがあります。LinuxやmacOSの*nix系ターミナルでは、Control + Lを押すと画面全体がクリアされ、きれいなプロンプトから再開できます。PowerShellでは`cls`コマンドを実行する必要があり、これをControl + Lなどのキーに割り当てることもできます。

コマンドの実行を中断したい場合は、Control + Cでプロセスを終了できます。これによりコマンドが終了し、新しいプロンプトが表示されて作業を続けられます。PowerShellユーザーの場合、Control + Cはテキストのコピーにも使われるため、コピー対象のテキストが選択されていない場合にのみプロセス終了が実行されます。

次のいくつかのショートカットは*nix系ターミナルでのみ利用可能で、PowerShellには対応するものがありません。

複数の作業を同時に行いたい場合、あるプロセスやコマンドをバックグラウンドで実行しながら別の作業を続けられます。Control + Zを押すと現在のプロセスがバックグラウンドタスクに移され、コマンドラインに戻って作業を続けられます。バックグラウンドタスクにフォーカスを戻したいときは、`fg`を使って復元できます。

最後に、同じコマンドを連続して実行したい場合があります。上矢印キーで直前のコマンドに遡ることもできますが、コマンドラインに感嘆符を2つ（`!!`）入力してEnterを押すと、直前に実行したコマンドが自動的に再実行されます。

多くのターミナルにはさらに多くのショートカットがあり、使っている環境のドキュメントを読むことをお勧めします。ですが、ここで紹介した基本的な共通ショートカットは、生産性を最大化しターミナルの達人になるための良い出発点になるでしょう。
