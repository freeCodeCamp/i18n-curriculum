# --description--

如果我們取 47，反轉並相加，47 + 74 = 121，這是回文數。

並非所有數字都能如此快速產生回文數，例如，

<div style="margin-left: 4em;">
  349 + 943 = 1292,<br>
  1292 + 2921 = 4213<br>
  4213 + 3124 = 7337<br>
</div>

也就是說，349 花了三次迭代才得到回文數。

雖然沒有人證明過，但有人認為某些數字，例如 196，永遠不會產生回文。永遠不會透過反轉並相加的處理程序形成回文的數字稱為 Lychrel 數。由於這些數字的理論性質，且為了這個問題的目的，我們將假設一個數字是 Lychrel，直到證明不是。此外，你被告知對於所有小於一萬的數字，它們要麼 (i) 在少於五十 次迭代內變成回文，要麼 (ii) 目前所有存在的運算能力都無法將其映射成回文。事實上，10677 是第一個被證明需要超過五十 次迭代才能產生回文的數字：4668731596684224866951378664（53 次迭代，28 位數）。

令人驚訝的是，有些回文數本身就是 Lychrel 數；第一個例子是 4994。

在 `num` 以下有多少個 Lychrel 數？

**注意：** 文字於 2007 年 4 月 24 日稍作修改，以強調 Lychrel 數的理論性質。

```js
const countLychrelNumbers = (size) => {
  const numReverse = (num) => {
    return Number(num.toString().split('').reverse().join(''));
  };
  const isPalin = (num) => {
    if (numReverse(num) === num) {
      return true;
    }
    return false;
  };
  let total = 0;
  for (let i = 1; i < size; i++) {
    let loopCount = 1;
    let sum = i;
    while (loopCount < 50) {
      sum = sum + numReverse(sum);
      if (isPalin(sum)) {
        break;
      } else {
        loopCount++;
      }
    }
    if (loopCount === 50) {
      total++;
    }
  }
  return total;
}
```
