## アルゴリズムとビッグO記法

- **アルゴリズム**：問題を解決したりタスクを実行したりするための明確で曖昧さのない手順の集合です。アルゴリズムは有限のステップ数で終了し、各ステップは正確かつ明確である必要があります。

- **ビッグO記法**：入力サイズが増加したときのアルゴリズムの最悪の性能や成長率を表します。リソース使用量が入力サイズに伴ってどのように増加するかに注目し、定数因子や低次の項は無視します。

### よくある時間計算量

- **O(1) - 定数時間**：入力サイズに関係なく同じ時間がかかります。

:::interactive_editor

```js
function checkEvenOrOdd(number) {
  if (number % 2 === 0) {
    return "Even";
  } else {
    return "Odd";
  }
}

console.log(checkEvenOrOdd(4)) // "Even"
console.log(checkEvenOrOdd(5)) // "Odd"
```

:::

- **O(log n) - 対数時間**：入力が増えても時間の増加がゆっくりです。問題サイズを繰り返し分割するアルゴリズム（例えば二分探索）でよく見られます。

- **O(n) - 線形時間**：実行時間が入力サイズに比例して増加します。

```js
for (const grade of grades) { // grades is an array
  console.log(grade);
}
```

- **O(n log n) - 線形対数時間**：マージソートやクイックソートなど効率的なソートアルゴリズムの一般的な時間計算量です。

- **O(n²) - 二次時間**：実行時間が二次的に増加します。ネストされたループでよく見られます。

:::interactive_editor

```js
const n = 3;

for (let i = 0; i < n; i++) {
  for (let j = 0; j < n; j++) {
    console.log("Hello, World!");
  }
}
```

:::

### 空間計算量

- **O(1) - 定数空間**：入力サイズに関係なく同じ量のメモリを使用します。
- **O(n) - 線形空間**：メモリ使用量が入力サイズに比例して増加します。
- **O(n²) - 二次空間**：メモリ使用量が二次的に増加します。

## 問題解決の手法

- **問題の理解**：問題文を何度も読みます。入力、期待される出力、入力から出力への変換方法を特定します。

- **疑似コード**：言語に依存しないアルゴリズムの論理を高レベルで説明します。一般的な文章とプログラミング構造（`IF`、`ELSE`、`FOR`、`WHILE`など）を混ぜて使います。

```md
GET original_string
SET reversed_string = ""
FOR EACH character IN original_string:
  ADD character TO THE BEGINNING OF reversed_string
DISPLAY reversed_string
```

- **エッジケース**：アルゴリズムが扱うべき境界にある特定の有効な入力です。常に考慮しテストしてください。

## 配列

- **静的配列**：初期化時にサイズが固定されます。要素は隣接したメモリ領域に格納されます。実行中にサイズを変更できません。

- **動的配列**：実行中に自動的にサイズが増減します。必要に応じて大きな配列に自動コピーしてサイズ変更を処理します。

### JavaScriptでの動的配列の実装

:::interactive_editor

```js
let numbers = [3, 4, 5, 6];
// Access elements
console.log(numbers[0])  // 3

// Update elements
numbers[2] = 16
console.log([...numbers]);  // [3, 4, 16, 6]

// Add elements
numbers.push(7)
console.log([...numbers]);  // [3, 4, 16, 6, 7] 

// Add at a specific index
numbers.splice(3, 0, 15);
console.log([...numbers]);  // [3, 4, 16, 15, 6, 7]

// Remove elements
numbers.splice(2, 1);  // Remove at specific index
console.log([...numbers]);  // [3, 4, 15, 6, 7]

numbers.pop()   // Remove last element
console.log([...numbers]); // [3, 4, 15, 6]
```

:::

### 動的配列の時間計算量

- **アクセス**：O(1)
- **末尾への挿入**：平均O(1)、サイズ変更時はO(n)
- **途中への挿入**：O(n)
- **削除**：途中はO(n)、末尾はO(1)

## スタック

- **スタック**：後入れ先出し（LIFO）のデータ構造です。要素は上部から追加・削除されます。

- **プッシュ操作**：スタックの上に要素を追加します。時間計算量はO(1)です。

- **ポップ操作**：スタックの上から要素を削除します。時間計算量はO(1)です。

:::interactive_editor

```js
// Using JavaScript array as a stack

let stack = [];

// Push operations
stack.push(1);
stack.push(2);
stack.push(3);
console.log([...stack]);  // [1, 2, 3]

// Pop operations
let topElement = stack.pop();
console.log(topElement);  // 3
console.log([...stack]);  // [1, 2]
```

:::

## キュー

- **キュー**：先入れ先出し（FIFO）のデータ構造です。要素は後ろから追加され、前から削除されます。

- **エンキュー操作**：キューの後ろに要素を追加します。時間計算量はO(1)です。

- **デキュー操作**：キューの前から要素を削除します。時間計算量はO(1)です。

:::interactive_editor

```js
// Using JavaScript array as a queue
let queue = [];

// Enqueue operations
queue.push(1);
queue.push(2);
queue.push(3);
console.log([...queue]);  // [1, 2, 3]

// Dequeue operations
let firstElement = queue.shift();
console.log(firstElement);  // 1
console.log([...queue]);  // [2, 3]
```

:::

## 連結リスト

- **連結リスト**：各ノードがデータと次のノードへの参照を持つ線形のデータ構造です。ノードは鎖のように繋がっています。

### 単方向連結リスト

- **構造**：各ノードはデータと次のノードへの1つの参照を持ちます。
- **走査**：先頭から末尾へ一方向にしか進めません。
- **先頭ノード**：リストの最初のノードで、通常は唯一直接アクセスできるノードです。
- **末尾ノード**：リストの最後のノードで、`null`を指します。

### 操作と時間計算量

- **先頭への挿入**：O(1)
- **末尾への挿入**：O(n) - 末尾まで走査が必要
- **途中への挿入**：O(n) - 挿入位置まで走査が必要
- **先頭からの削除**：O(1)
- **末尾からの削除**：O(n) - 前のノードを探すため走査が必要
- **途中からの削除**：O(n) - 削除ノードを探すため走査が必要

### 双方向連結リスト

- **構造**：各ノードはデータと2つの参照（次のノードと前のノード）を持ちます。
- **走査**：両方向に移動できます。
- **メモリ**：単方向連結リストより多くのメモリを使います（参照が増えるため）。

## 各データ構造の使いどころ

- **配列（動的配列）**：順序付きでインデックスによるアクセスが必要で、サイズが事前にわからない場合
- **スタック**：LIFO操作が必要な場合（元に戻す機能、式の評価、バックトラッキング）
- **キュー**：FIFO操作が必要な場合（タスクスケジューリング、幅優先探索）
- **連結リスト**：先頭での頻繁な挿入・削除があり、サイズが不明でランダムアクセスが不要な場合
