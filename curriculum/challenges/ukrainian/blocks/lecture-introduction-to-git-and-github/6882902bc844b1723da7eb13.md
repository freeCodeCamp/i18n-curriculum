# --description--

Перш ніж почати справді користуватися системою контролю версій і такими сайтами, як GitHub, потрібно встановити Git і підготувати його до роботи.

Щоб перевірити, чи встановлений Git на вашому комп’ютері, можна виконати таку команду в терміналі:

```sh
git --version
```

Якщо побачите номер версії, Git встановлений. Якщо ні — потрібно встановити.

Для Linux Git часто встановлений за замовчуванням. Якщо ні, встановіть його через менеджер пакетів, наприклад `sudo apt-get install git` або `sudo pacman -S git`.

Для Mac можна встановити Git через Homebrew командою `brew install git` або скачати інсталятор з сайту Git.

Для Windows можна скачати інсталятор з сайту Git. Якщо встановлено Chocolatey, можна виконати `choco install git.install` у PowerShell. Для Windows також рекомендується встановити Git Bash для Unix-подібного середовища.

Щоб переконатися, що встановлення пройшло успішно, знову виконайте `git --version`.

Після встановлення Git потрібно внести кілька змін у конфігурацію. Ці зміни потрібно зробити лише один раз на комп’ютері, і не доведеться повторювати їх при оновленнях.

`git config` використовується для встановлення змінних конфігурації, які визначають, як Git працює на вашому комп’ютері. Щоб переглянути поточні налаштування і де вони зберігаються у вашій системі, виконайте таку команду:

```sh
git config --list --show-origin
```

Зараз ви повинні бачити лише системні налаштування, якщо Git встановлений вперше. Якщо Git був встановлений раніше, ви можете бачити налаштування користувача, такі як ім’я користувача та е-пошта.

Щоб встановити ім’я користувача, виконайте:

```sh
git config --global user.name "Jane Doe"
```

Прапорець `--global` встановлює ім’я користувача для всіх проєктів на системі, що використовують Git. Щоб змінити ім’я для конкретного проєкту, виконайте команду у каталозі проєкту без прапорця `--global`.

Щоб встановити електронну пошту користувача, виконайте:

```sh
git config --global user.email janedoe@example.com
```

Налаштування імені користувача та е-пошти важливе, бо Git використовуватиме цю інформацію для кожного коміту у проєкті. Ви дізнаєтеся більше про коміти у наступних уроках.

Ще можна встановити улюблений редактор для Git. Приклад для Emacs:

```sh
git config --global core.editor emacs
```

Інші варіанти включають Vim, Nano, VS Code та інші. Наприклад, якщо ви використовуєте VS Code як редактор, можна налаштувати Git так:

```sh
git config --global core.editor "code --wait"
```

Якщо не встановити редактор, Git використовуватиме редактор за замовчуванням системи.

Якщо ви користуєтеся Windows і хочете змінити редактор, потрібно вказати повний шлях до виконуваного файлу, наприклад:

```sh
git config --global core.editor "'C:/Program Files/Notepad++/notepad++.exe' -multiInst -notabbar -nosession -noPlugin"
```

Щоб побачити список ваших налаштувань, виконайте таку команду:

```sh
git config --list
```

Тепер, коли Git встановлений і налаштований, ви готові почати додавати контроль версій до своїх проєктів.
