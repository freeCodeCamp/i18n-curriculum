# --description--

Graphen sind sehr mächtige Datenstrukturen, die aus einer Menge von Knoten, auch als Scheitelpunkte bekannt, und Kanten bestehen, die diese verbinden.

Es gibt zwei gängige Methoden, Graphen in Ihrem Code zu implementieren:

* Adjazenzmatrizen
    
* Adjazenzlisten
    

Schauen wir uns diese genauer an und besprechen ihre Vorteile und Einschränkungen.

## Adjazenzmatrizen

Wir beginnen mit Adjazenzmatrizen.

Eine Adjazenzmatrix ist ein zweidimensionales Array, bei dem die Zeilen und Spalten die Scheitelpunkte des Graphen darstellen.

Die Werte in der Matrix repräsentieren die Kanten oder Verbindungen zwischen den Knoten.

Zum Beispiel, wenn Sie eine Matrix in einer Variablen namens `matrix` gespeichert haben, repräsentiert der Wert an `matrix[i][j]`, wobei `i` die Zeile und `j` die Spalte ist, die Kante oder Verbindung zwischen Knoten `i` und Knoten `j`.

Die Werte können je nach Gewichtung des Graphen unterschiedliche Bedeutungen haben:

* Wenn der Graph ungewichtet ist, bedeutet ein Wert von `1`, dass eine Kante diese Knoten verbindet, während ein Wert von `0` bedeutet, dass keine Kante zwischen ihnen existiert.
    
* Wenn der Graph gewichtet ist, stellt der Wert das Gewicht der Kante dar, die die Knoten verbindet.

Ein großer Vorteil der Verwendung einer Adjazenzmatrix ist, dass die Überprüfung, ob eine Kante zwischen zwei Knoten existiert, in konstanter Zeit `O(1)` erfolgt. Das liegt daran, dass das Programm nur diesen bestimmten Wert im 2D-Array finden muss.

Diese Effizienz beim Finden der Kanten hat jedoch einen Nachteil. Adjazenzmatrizen benötigen einen großen quadratischen Speicherplatz `O(V²)`, wobei `V` die Anzahl der Knoten im Graphen ist.

Das ist ineffizient für dünn besetzte Graphen, also Graphen mit nur wenigen Kanten. Warum? Weil bei einem dünn besetzten Graphen viele 0en in der Matrix gespeichert werden, um das Fehlen von Kanten zwischen den Knoten darzustellen, und diese 0en dennoch Speicherplatz im Arbeitsspeicher beanspruchen.

Adjazenzmatrizen sind auch ineffizient, um die Nachbarn eines Knotens zu finden, da das Programm die gesamte Zeile oder Spalte durchlaufen muss, um die 0en und 1en zu finden, die die Kanten darstellen. Im schlimmsten Fall kann dieser Vorgang `O(V)` Zeit in Anspruch nehmen, wobei `V` die Anzahl der Knoten im Graphen ist.

Sehen wir uns ein Beispiel für eine Adjazenzmatrix dieses speziellen Graphen an:

<img src="https://cdn.freecodecamp.org/curriculum/lecture-transcripts/how-do-matrices-and-adjacency-lists-work-1.png" alt="Ein Graph mit vier Knoten, bezeichnet mit A, B, C und D. Knoten A verbindet sich mit B, C und D. Knoten B verbindet sich mit A und D. Knoten C verbindet sich mit A. Knoten D verbindet sich mit A und B.">

In der Adjazenzmatrix:

* Jede Zeile stellt einen Knoten dar. Die erste Zeile steht für Knoten A, die zweite für Knoten B und so weiter.
    
* Jede Spalte stellt ebenfalls einen Knoten dar.
    
* Jeder Wert in der Matrix zeigt an, ob eine Kante zwischen jedem Knotenpaar existiert. Ein Wert von 0 bedeutet, dass keine Kante diese Knoten verbindet, während ein Wert von 1 bedeutet, dass eine Kante existiert.
    
Die Werte auf der Diagonalen zeigen an, ob jeder Knoten eine Selbstschleife hat, also eine Kante, die den Knoten mit sich selbst verbindet. In unserem Beispiel sind alle Werte 0, da der Graph keine Selbstschleifen hat.

Dies ist eine visuelle Darstellung der Adjazenzmatrix, um zu zeigen, wie die Zeilen und Spalten die entsprechenden Knoten darstellen.

Zum Beispiel ist die erste Zeile `[0, 1, 1, 1]`, weil Knoten A Kanten zu den Knoten B, C und D hat:

```markdown
#      A  B  C  D
# A   [0, 1, 1, 1],
# B   [1, 0, 0, 1],
# C   [1, 0, 0, 0],
# D   [1, 1, 0, 0]
```

Und dies ist dieselbe Adjazenzmatrix, aber in JavaScript-Code implementiert:

```javascript
const adjacencyMatrix = [
  [0, 1, 1, 1], // The neighbors of A are B, C, and D
  [1, 0, 0, 1], // The neighbors of B are A and D
  [1, 0, 0, 0], // The only neighbor of C is A
  [1, 1, 0, 0]  // The neighbors of D are A and B
];
```

## Adjazenzlisten

Eine weitere gängige Methode, Graphen darzustellen, ist die Verwendung von Adjazenzlisten.

Eine Adjazenzliste ist ein Array oder Objekt (oder ein `Map`), das alle Nachbarn jedes Knotens speichert.

Es gibt zwei gängige Methoden, Adjazenzlisten zu implementieren:

* Als Array, bei dem jeder Index einen Knoten darstellt und das Array an diesem Index seine Nachbarn enthält.
* Als Objekt (oder `Map`), bei dem jeder Schlüssel einen Knoten darstellt und der Wert, der diesem Schlüssel zugeordnet ist (ein Array), seine Nachbarn enthält.

Adjazenzlisten sind im Hinblick auf den Speicherbedarf effizienter als Adjazenzmatrizen. Sie haben eine `O(V + E)` Speicherkomplexität, wobei `V` die Anzahl der Scheitelpunkte (Knoten) und `E` die Anzahl der Kanten ist.

Es ist auch effizient, alle Nachbarknoten zu finden, da dieser Vorgang nur den Zugriff auf die Liste erfordert, die dem Knoten zugeordnet ist.

Es gibt jedoch auch einen Nachteil.

Adjazenzlisten sind weniger effizient als Adjazenzmatrizen, wenn es darum geht, festzustellen, ob eine Kante zwischen zwei Knoten existiert.

Der Suchvorgang benötigt typischerweise `O(deg(v))` Zeit (wobei `deg(v)` die Anzahl der Nachbarn des zu überprüfenden Knotens ist), und im schlimmsten Fall kann es `O(V)` sein, wenn ein Knoten mit allen anderen Knoten im Graphen verbunden ist.

Hier ist ein Beispiel für eine Adjazenzliste dieses Graphen:

<img src="https://cdn.freecodecamp.org/curriculum/lecture-transcripts/how-do-matrices-and-adjacency-lists-work-1.png" alt="Ein Graph mit vier Knoten, bezeichnet mit A, B, C und D. Knoten A verbindet sich mit B, C und D. Knoten B verbindet sich mit A und D. Knoten C verbindet sich mit A. Knoten D verbindet sich mit A und B. Es ist dasselbe Bild wie zuvor.">

Diese Adjazenzliste ist als Objekt implementiert. Jeder Schlüssel im Objekt stellt einen Knoten dar, und der Wert, der diesem Schlüssel zugeordnet ist, ist ein Array mit allen Nachbarn des entsprechenden Knotens:

```javascript
const adjacencyList = {
  A: ["B", "C", "D"],
  B: ["A", "D"],
  C: ["A"],
  D: ["A", "B"]
};
```

Alternativ könnten wir sie als 2D-Array implementieren, bei dem jeder Index einen Knoten darstellt. Zum Beispiel steht Index 0 für Knoten A, Index 1 für Knoten B und so weiter:

```javascript
const adjacencyList = [
  ["B", "C", "D"], // Neighbors of A (index 0)
  ["A", "D"],      // Neighbors of B (index 1)
  ["A"],           // Neighbors of C (index 2)
  ["A", "B"]       // Neighbors of D (index 3)
];
```

Beachten Sie, dass dieses 2D-Array zwar ähnlich wie die Adjazenzmatrix aussieht, sie aber ganz unterschiedlich sind.

* Die Adjazenzmatrix speichert 0en, 1en oder andere Werte, die die Kanten oder Gewichte der Kanten im Graphen darstellen.

* Die Adjazenzliste speichert die tatsächliche Liste aller Nachbarn jedes Knotens.

Das ist ein sehr wichtiger Unterschied, mit dem Sie vertraut sein sollten.

Sowohl Adjazenzmatrizen als auch Adjazenzlisten sind sehr wichtig für die Implementierung von Graphen. Die Wahl zwischen ihnen hängt von der Größe des Graphen und davon ab, wie Sie die Daten verwenden müssen. Adjazenzmatrizen sind hilfreich für dichte Graphen mit vielen Kanten, während Adjazenzlisten in der Regel die bevorzugte Wahl für reale Szenarien sind, in denen dünn besetzte Graphen häufiger vorkommen.
