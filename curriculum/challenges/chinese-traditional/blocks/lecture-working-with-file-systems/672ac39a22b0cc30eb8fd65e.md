# --description--

在建置網頁應用程式時，思考你的檔案和資料夾的組織非常重要。這將有助於保持你的程式碼有條理且更容易維護。

讓我們來看看一個 HTML 和 CSS 專案的範例資料夾結構：

```bash
.
├── /assets
│   ├── /images
│   │   ├── logo.png
│   │   ├── banner.jpg
│   │   └── icons.svg
│   ├── /fonts
│   │   ├── custom-font.woff
│   │   └── custom-font.woff2
├── /css
│   ├── main.css
│   ├── about.css
│   └── contact.css
├── index.html
├── about.html
├── contact.html
└── README.md
```

在範例的頂部，有一個單一的點，表述根目錄。

根目錄是檔案系統中的最高層級目錄，作為所有檔案路徑的起點，並包含所有其他目錄和檔案。目錄是資料夾的另一種稱呼。

在根目錄中，顯示的第一個目錄是 `assets` 目錄。`assets` 一詞指的是專案中使用的任何檔案，例如影像、字型或其他資源。

在此範例中，`assets` 目錄內有兩個目錄，分別稱為 `images` 和 `fonts`。

下一個目錄會是 `css` 目錄。有時這也被稱為樣式表目錄。這裡是你會儲存所有 CSS 檔案的地方。

在 `css` 目錄下方是 HTML 檔案。`index.html` 檔案表述首頁，而 `about.html` 和 `contact.html` 分別表述關於和聯絡頁面。

最後，有一個 `README.md` 檔案。`README` 檔案通常用來提供關於專案的有用資訊，例如它的功能、如何使用，以及其他相關細節。

`.md` 擴充套件代表 Markdown，這是一種輕量級標記語言，常用於撰寫文件。

透過將你的專案組織成資產、CSS 和 HTML 檔案，可以輕鬆找到你正在尋找的內容。這也讓其他開發者更容易為你的專案做出貢獻。

重要的是要注意，這只是你如何組織你的檔案和資料夾的一個範例。專案的結構有許多不同的方式。
