# --description--

你已经了解了 `HTMLMediaElement` 和 MediaStreams API，但还有一些 API 在处理视频和音频时非常有用。

第一个是屏幕捕捉 API。顾名思义，该 API 允许你记录用户的屏幕。通过调用 `mediaDevices` 对象的 `getDisplayMedia()` 方法并使用返回的媒体流即可访问此 API。

然后是 MediaStream Recording API。该 API 与 MediaStreams API 协同工作，允许你录制 MediaStream（甚至直接录制 `HTMLMediaElement`）。然后它会触发带有 Blob 负载的 `dataavailable` 事件，你可以将其写入局部的文件存储。

所有这些技术的基础是 Media Source Extensions API。Media Source Extensions API 允许你通过 `srcObject` 属性将用户的摄像头视频直接传递给视频元素。 

最后，是驱动网页上所有可听内容的 Web Audio API。该 API 包含重要的对象，如 `AudioBuffer`（专门表现包含音频数据的缓冲区）或 `AudioContext`。 

这就是我们关于音频和视频的所有课程内容！
