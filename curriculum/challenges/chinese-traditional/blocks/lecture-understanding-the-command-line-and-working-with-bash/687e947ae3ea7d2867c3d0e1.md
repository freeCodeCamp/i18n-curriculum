# --description--

讓我們來了解指令選項和旗標。

在之前的課程中，你學習了如何將引數傳遞給像是 `touch readme.md` 這樣的指令。選項，或稱旗標，是你可以傳遞給指令的特殊引數，會影響指令的行為。這兩個詞彙可以互換使用，儘管「旗標」通常較特指作為開關切換的選項。

選項通常以一或兩個連字號（`-`）符號作為前置式，這有助於在選項和引數之間提供視覺上的區別。首先，讓我們來看看兩個連字號，或稱為「長表單」語法。

許多應用程式，例如 `ls`，接受 `--version` 旗標，該旗標會將應用程式的當前版本列印到終端機，而不是執行應用程式的指令。

許多應用程式也提供 `--help` 旗標，該旗標會列印如何使用該應用程式的說明。

一個連字元的短格式語法通常使用單一字母的選項。 例如，`ls` 的 `-a` 旗標會列出所有檔案，包括以點（`.`）開頭的隱藏檔案，如 `.env`。

這些短選項的優點是你可以將多個旗標串鏈在一起。你可以使用 `ls -ahs` 一次性使用所有單個字母，而不是 `ls --all --human-readable --size`。

某些選項可選方案需要傳遞一個值給它們。

當使用長格式選項時，通常需要使用等號。在此語法中，值會直接與選項使用等號（`=`）符號連接。以下是修改 `ls` 行為以包含或排除顏色的範例：

```bash
ls --color=never
```

使用短格式選項時，通常會用空格分隔值。例如，以下是設定 `ls` 結果中 `width` 的長格式和短格式選項：

```bash
ls -w 50
```

```bash
ls --width=50
```

請注意簡短形式使用 `-w 50`，而完整形式使用 `--width=50`。了解這個區別很重要，以避免將選項值誤傳為位置引數。

但如果你不確定，請記得你通常可以使用 `--help` 旗標來查看選項的預期語法！

## --text--

「長表單」和「短表單」語法在命令選項中有什麼差異？

## --answers--

長格式使用單一連字號，而短格式使用兩個連字號。
