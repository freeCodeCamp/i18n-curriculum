# --description--

У React рядкові стилі використовують для безпосереднього застосування CSS-стилів до React-елементів у вашому JSX-коді замість визначення їх у окремих CSS-файлах.

Підхід React до рядкових стилів полягає у використанні JavaScript-об’єктів для визначення стилів, а не традиційних CSS-рядків. Це означає, що замість того, щоб писати стилі так, як у CSS-файлі, ви створюєте JavaScript-об’єкт, де ключі — це версії назв CSS-властивостей у camel case, а значення — рядки CSS-значень.

Ось приклад того, як можна використовувати рядкові стилі для компонента `Button`:

```jsx
function Button({ buttonText }) {
  const defaultStyles = {
    backgroundColor: "#007BFF",
    color: "white",
    border: "none",
    borderRadius: "4px",
    padding: "10px 20px",
    fontSize: "16px",
    fontWeight: "bold",
    cursor: "pointer",
    transition: "background-color 0.3s ease",
  };

  return <button style={defaultStyles}>{buttonText}</button>;
}
```

У цьому прикладі ми визначаємо об’єкт стилю з назвою `defaultStyles`. Потім застосовуємо ці стилі до елемента button за допомогою атрибута `style`. React подбає про застосування цих стилів до елемента під час його відтворення.

Ви також можете передати об’єкт безпосередньо в атрибут `style`. Ось як виглядатиме змінений приклад:

```jsx
function Button({ buttonText }) {
  return (
    <button
      style={{
        backgroundColor: "#007BFF",
        color: "white",
      }}
    >
      {buttonText}
    </button>
  );
}
```

Зверніть увагу на подвійні фігурні дужки `{{}}` в атрибуті `style`. Зовнішні дужки позначають JavaScript-вираз у JSX, а внутрішні — літерал JavaScript-об’єкта. Цей синтаксис дозволяє вставляти JavaScript-об’єкти безпосередньо в атрибути JSX.

Іноді варто передавати об’єкт безпосередньо, якщо властивостей небагато, як тут. Інакше краще передавати ім’я об’єкта, як у першому прикладі.

Важливо зауважити, що хоча назви CSS-властивостей зазвичай пишуть у kebab case, як `font-size`, у рядкових стилях React ми використовуємо camel case, як `fontSize`. Це тому, що об’єкт стилю — це JavaScript-об’єкт, і назви у kebab case не є дійсними ключами об’єкта в JavaScript без лапок.

Велика перевага рядкових стилів у React — це підтримка динамічного стилювання на основі стану компонента або пропсів. Наприклад:

```jsx
function DynamicButton({ isActive }) {
  const buttonStyles = {
    backgroundColor: isActive ? "green" : "red",
    color: "white",
    padding: "10px 15px",
    border: "none",
    cursor: "pointer",
  };

  return <button style={buttonStyles}>Login</button>;
}
```

У цьому прикладі колір фону кнопки змінюється залежно від пропса `isActive`. Такий тип динамічного стилювання може бути потужним для створення інтерактивних і адаптивних інтерфейсів користувача.

Підсумовуючи, рядкові стилі в React дають потужний спосіб застосовувати та керувати стилями безпосередньо у ваших компонентах. Вони використовують JavaScript-об’єкти замість CSS-рядків, вимагають camel case для назв властивостей і легко інтегрують динамічні значення. Це незамінний інструмент у наборі розробника React, особливо для створення висококастомізованих і інтерактивних інтерфейсів користувача.
