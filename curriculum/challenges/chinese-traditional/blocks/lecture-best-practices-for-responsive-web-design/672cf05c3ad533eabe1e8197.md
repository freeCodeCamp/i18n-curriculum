# --interactive--

媒體查詢是響應式網頁設計的基本組件，允許開發者根據使用者裝置或瀏覽器的特性套用不同的樣式。它們提供了一種方法，可以在不更改內容本身的情況下，為各種裝置量身打造內容的呈現方式。

媒體查詢的核心由媒體型別和一個或多個用來檢查特定條件的表達式組成。如果這些條件為真，則會套用對應的樣式。這個機制允許產生能夠適應不同螢幕尺寸、解析度和裝置能力的響應式設計。

CSS 中媒體查詢的基本語法如下：

```css
@media mediatype and (feature: value) {
  /* CSS rules go here */
}
```

在此結構體中，`mediatype` 指定查詢所適用的媒體型別，而 `feature: value` 配對定義必須符合的條件，才能套用樣式。

媒體型別描述裝置的一般類別。讓我們來談談最常用的媒體型別：

- `all` 適用於所有裝置。如果未指定媒體型別，則這是預設值。

- `print` 是用於分頁材料和在列印預覽模式下於螢幕上檢視的文件。

- `screen` 主要是為螢幕設計的。

過去有更多媒體型別，例如 `handheld` 和 `tv`，但大多數已被棄用，改為使用特性來更精確地標的裝置。

媒體特性描述使用者代理、輸出裝置或環境的特定特性。讓我們來談談一些最常用的媒體特性。`width` 和 `height` 指的是視窗的寬度和高度，且經常與 `min-` 或 `max-` 前置式一起用於區間查詢。以下是一個範例：

```css
@media screen and (min-width: 768px) {
  /* Styles for screens at least 768px wide */
}
```

`aspect-ratio` 描述視窗寬度與高度之間的比例。以下是一個範例：

```css
@media screen and (aspect-ratio: 16/9) {
  /* Styles for screens with a 16:9 aspect ratio */
}
```

`orientation` 特性顯示裝置是處於橫向還是直向。以下是一個範例：

```css
@media screen and (orientation: landscape) {
  /* Styles for landscape orientation */
}
```

`resolution` 特性描述輸出裝置的解析度，單位為每英吋點數（dpi）或每公分點數（dpcm）。以下是一個範例：

```css
@media screen and (min-resolution: 300dpi) {
  /* Styles for high-resolution screens */
}
```

`hover` 特性會測試主要輸入機制是否能暫留在元素上。以下是一個範例：

```css
@media (hover: hover) {
  /* Styles for devices that support hover */
}
```

`prefers-color-scheme` 特性會偵測使用者是否要求淺色或深色主題。以下是一個範例：

```css
@media (prefers-color-scheme: dark) {
  /* Styles for dark mode */
}
```

媒體查詢也可以使用邏輯運算子來結合多個條件。`and` 運算子用於結合多個媒體特性，而 `not` 和 `only` 則可用來否定或隔離媒體查詢。以下是一個結合多個特性的範例：

```css
@media screen and (min-width: 768px) and (orientation: landscape) {
  /* Styles for landscape screens at least 768px wide */
}
```

也可以針對以逗號分隔的多個查詢，這會像「或」運算子一樣運作：

```css
@media screen and (min-width: 768px), print {
  /* Styles for screens at least 768px wide OR for print */
}
```

在使用媒體查詢時，考慮層疊非常重要。媒體查詢不會增加特異性－它們只是將條件規則分組。在每個媒體查詢中，CSS 層疊的正常規則仍然適用。

在實務中，媒體查詢經常用於創建響應式佈局。一個常見的樣式是為行動裝置定義基礎樣式，然後使用媒體查詢來強化較大螢幕的佈局：

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<div class="container">
  <p>Lorem ipsum dolor sit amet, consectetur adipiscing elit. Integer nec odio. Praesent libero. Sed cursus ante dapibus diam.</p>
  <p>Sed nisi. Nulla quis sem at nibh elementum imperdiet. Duis sagittis ipsum. Praesent mauris. Fusce nec tellus sed augue semper porta.</p>
  <p>Mauris massa. Vestibulum lacinia arcu eget nulla. Class aptent taciti sociosqu ad litora torquent per conubia nostra, per inceptos himenaeos.</p>
</div>
```

```css
/* Base styles for mobile */
.container {
  width: 100%;
  padding: 15px;
}

/* Styles for tablets */
@media screen and (min-width: 768px) {
  .container {
    width: 750px;
    margin: 0 auto;
  }
}

/* Styles for desktops */
@media screen and (min-width: 1024px) {
  .container {
    width: 960px;
  }
}
```

:::

這種方法稱為「行動優先」響應式設計，確保基礎樣式適用於行動裝置，並為較大螢幕添加增強功能。

總結來說，媒體查詢是 CSS 中一個強大的工具，允許創建響應式且可適應的網頁設計。透過了解如何使用不同的媒體型別和特性，開發者可以創建在各種裝置和偏好中提供最佳使用者體驗的網站。隨著網頁技術持續演進，持續更新對新媒體特性的了解有助於創建更細緻且友善使用者的響應式設計。
