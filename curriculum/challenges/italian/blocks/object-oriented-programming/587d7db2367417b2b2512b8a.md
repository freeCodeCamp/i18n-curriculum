# --description--

Nella sfida precedente, `bird` aveva una proprietà pubblica `name`. È considerata pubblica perché può essere consultata e modificata al di fuori della definizione di `bird`.

```js
bird.name = "Duffy";
```

Quindi, qualsiasi parte del tuo codice può facilmente cambiare il nome di `bird` a qualsiasi valore. Pensa a cose come password e conti bancari che potrebbero essere modificati facilmente da qualsiasi parte del tuo codice. Questo potrebbe causare molti problemi.

Il modo più semplice per rendere questa proprietà pubblica privata è creare una variabile all’interno della funzione costruttore. Questo cambia l’ambito di quella variabile rendendolo interno alla funzione costruttore invece che disponibile globalmente. In questo modo, la variabile può essere consultata e modificata solo dai metodi anch’essi all’interno della funzione costruttore.

```js
function Bird() {
  let hatchedEgg = 10;

  this.getHatchedEggCount = function() { 
    return hatchedEgg;
  };
}
let ducky = new Bird();
ducky.getHatchedEggCount();
```

Qui `getHatchedEggCount` è un metodo privilegiato, perché ha accesso alla variabile privata `hatchedEgg`. Questo è possibile perché `hatchedEgg` è dichiarato nello stesso contesto di `getHatchedEggCount`. In JavaScript, una funzione ha sempre accesso al contesto in cui è stata creata. Questo si chiama `closure`.
