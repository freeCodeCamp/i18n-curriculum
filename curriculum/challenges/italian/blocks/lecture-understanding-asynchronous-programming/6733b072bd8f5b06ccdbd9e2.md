# --description--

Impariamo cos’è JavaScript asincrono. Gli eventi asincroni si verificano in momenti diversi, indipendentemente l’uno dall’altro.

Nel contesto dello sviluppo software, "asincrono" indica attività che vengono eseguite in background, indipendentemente dal flusso principale del programma. Il vantaggio principale dei processi asincroni è che non bloccano l’esecuzione del programma principale.

Questo è particolarmente utile per attività che possono richiedere molto tempo, come recuperare dati da un server remoto, elaborare file di grandi dimensioni, gestire input dell’utente ed eseguire calcoli complessi. Questo è ciò che chiamiamo programmazione asincrona.

Questo approccio è in contrasto con la tecnica tradizionale di programmazione sincrona con cui hai lavorato finora. Quando scrivi più righe di codice, di solito puoi prevedere cosa succederà e quando succederà. La prima riga verrà eseguita, poi la seconda, e così via nell’ordine in cui le scrivi.

In questo esempio, la prima riga in cui definisci la variabile verrà eseguita per prima, poi la seconda riga, e infine la terza riga in cui la stringa viene mostrata nella console:

```js
const topic = "JavaScript";
const learning = `I'm learning ${topic}!`;
console.log(learning);
```

Tutto è sequenziale e prevedibile. Ogni riga viene completata prima che inizi la successiva. Questo tipo di programma JavaScript è noto come single-threaded.

Il concetto di thread è molto importante. Un thread è un’unità di esecuzione all’interno di un processo. È come un flusso di controllo separato all’interno del programma.

Nella programmazione sincrona, i thread vengono eseguiti in sequenza, uno dopo l’altro. Se un thread è bloccato, ad esempio in attesa di un input dell’utente, l’intero processo è bloccato finché il thread non termina.

Nella programmazione asincrona, i thread possono essere eseguiti contemporaneamente, facendo girare più thread allo stesso tempo. In questo modo, il programma può continuare a eseguire più attività simultaneamente senza rendere il programma principale non responsivo, anche se uno dei thread è bloccato.

La programmazione asincrona spesso coinvolge callback, promise o async/await per gestire operazioni non bloccanti.

Come hai imparato nelle lezioni precedenti, una callback è una funzione che passi a un’altra funzione per farla chiamare in un secondo momento. I gestori di eventi sono un tipo particolare di callback con cui hai già lavorato. Erano il modo più comune per implementare funzioni asincrone in JavaScript. Tuttavia, se la funzione di callback accetta a sua volta altre funzioni di callback, il codice e la logica possono diventare molto complicati molto rapidamente.

Attualmente, la tecnica più usata per implementare la programmazione asincrona in JavaScript è la promise. Una `Promise` è un oggetto che rappresenta il completamento eventuale (o il fallimento) di un processo asincrono e il suo valore.

Il valore di una promise non è noto quando la promise viene creata. Lo si conosce solo quando il processo asincrono è completato. Per esempio, il processo di recupero dati da un’API remota per la tua applicazione web può richiedere un po’ di tempo. Nel frattempo, vuoi che i tuoi utenti abbiano una buona esperienza d’uso, giusto?

Per implementare questo, potresti creare una promise per mantenere l’interfaccia utente attiva e interattiva mentre il processo asincrono è in esecuzione. Quando il processo è completato, la promise conterrà i dati restituiti dall’API, così l’applicazione potrà gestirli o renderizzarli correttamente quando saranno disponibili.

La programmazione asincrona è uno strumento potente per costruire applicazioni JavaScript efficienti. Capendo le differenze tra programmazione sincrona e asincrona, puoi scegliere l’approccio giusto per la tua applicazione e scrivere codice più efficiente.
