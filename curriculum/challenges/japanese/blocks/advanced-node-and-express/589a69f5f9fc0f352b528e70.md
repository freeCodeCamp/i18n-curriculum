# --description--

この種の認証がアプリでたどる基本的な流れは次の通りです：

1. ユーザーが特定の戦略（例：GitHub）を使って認証するためのルートに送るボタンやリンクをクリックします。
2. ルートが`passport.authenticate('github')`を呼び出し、GitHubへリダイレクトします。
3. ユーザーがGitHubのページに着くと、まだログインしていなければログインを促されます。その後、あなたのアプリがプロフィールにアクセスすることを承認するかどうか尋ねられます。
4. 承認されると、ユーザーは特定のコールバックURLであなたのアプリにプロフィール情報とともに戻されます。
5. これで認証されている状態になり、アプリはそのプロフィールが既存のものかどうかを確認し、なければデータベースに保存します。

OAuthを使う戦略では、少なくとも*クライアントID*と*クライアントシークレット*が必要です。これはサービス側が認証リクエストの発信元を確認し、有効かどうかを判断するためのものです。これらはGitHubなど認証を実装するサイトから取得し、あなたのアプリ固有のものです。**絶対に共有してはいけません**し、公開リポジトリにアップロードしたりコードに直接書いたりしてはいけません。一般的には`.env`ファイルに入れて、次のように参照します：`process.env.GITHUB_CLIENT_ID`。このチャレンジではGitHub戦略を使います。

<a href="https://www.freecodecamp.org/news/how-to-set-up-a-github-oauth-application/" target="_blank" rel="noopener noreferrer nofollow">こちらの手順</a>に従ってGitHubから*クライアントIDとシークレット*を取得してください。ホームページURLはあなたのホームページ（**プロジェクトコードのURLではありません**）に設定し、コールバックURLは同じホームページURLの末尾に`/auth/github/callback`を付けて設定してください。クライアントIDとクライアントシークレットはプロジェクトの`.env`ファイルに`GITHUB_CLIENT_ID`と`GITHUB_CLIENT_SECRET`として保存してください。

`routes.js`ファイルでは、ホームページルートに`showSocialAuth: true`の後に`showRegistration: true`を追加してください。次に、GETリクエストを受け付ける2つのルートを作成します：`/auth/github`と`/auth/github/callback`です。最初のルートはpassportを呼び出して`'github'`の認証だけを行います。2つ目のルートはpassportを呼び出して`'github'`の認証を行い、失敗した場合は`/`にリダイレクトし、成功した場合は`/profile`にリダイレクトします（前のプロジェクトと似ています）。

`/auth/github/callback`の例は通常のログイン処理に似ています：

```js
app.route('/login')
  .post(passport.authenticate('local', { failureRedirect: '/' }), (req,res) => {
    res.redirect('/profile');
  });
```

正しくできたと思ったらページを提出してください。エラーが出る場合は<a href="https://forum.freecodecamp.org/t/advanced-node-and-express/567135#implementation-of-social-authentication-3" target="_blank" rel="noopener noreferrer nofollow">ここまでのプロジェクトを確認できます</a>。
