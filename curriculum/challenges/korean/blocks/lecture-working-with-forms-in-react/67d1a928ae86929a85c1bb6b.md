# --description--

폼은 사용자 입력을 처리하고 데이터를 수집하며 동작을 실행할 수 있게 해주기 때문에 모든 웹 애플리케이션에서 기본적입니다.

React에서는 상태나 refs를 사용해 폼을 관리하며, 이를 통해 폼 동작과 유효성 검사를 완전히 제어할 수 있습니다. 폼을 관리하는 이 두 가지 방법을 각각 "제어된" 입력과 "비제어된" 입력이라고 합니다.

제어된 입력과 비제어된 입력이 무엇인지 살펴보겠습니다.

제어된 입력은 폼 입력을 처리하는 가장 "React다운" 방법입니다. 제어된 입력에서는 입력 필드 값을 상태에 저장하고 `onChange` 이벤트를 통해 값을 업데이트합니다. 이렇게 하면 폼 데이터를 완전히 제어할 수 있고 즉각적인 유효성 검사와 조건부 렌더링이 가능합니다.

과정은 다음과 같습니다: React는 `useState` 훅으로 폼 상태를 유지하고, 사용자가 입력 필드에 타이핑할 때마다 `onChange` 이벤트가 발생해 상태를 업데이트하며 React가 새 값으로 컴포넌트를 다시 렌더링합니다.

```jsx
import { useState } from "react";

function App() {
  const [name, setName] = useState("");

  const handleChange = (e) => {
    setName(e.target.value);
  };

  const handleSubmit = (e) => {
    e.preventDefault();
    console.log(name);
  };

  return (
    <>
      <form onSubmit={handleSubmit}>
        <label htmlFor="name">Your name</label> <br />
        <input value={name} id="name" onChange={handleChange} type="text" />
        <button type="submit">Submit</button>
      </form> 
    </>
  );
}

export default App;
```

제어된 입력의 장점은 다음과 같습니다:

- 폼 데이터에 즉시 접근할 수 있습니다.

- 즉각적인 유효성 검사를 구현할 수 있습니다.

- 제출 버튼을 조건부로 비활성화할 수 있습니다.

- 입력 값을 프로그래밍 방식으로 제어할 수 있습니다.

반면 비제어된 입력은 전통적인 HTML 폼에서 더 자주 볼 수 있습니다. 즉, `useState` 훅을 통해 입력을 처리하는 대신, HTML의 비제어된 입력은 DOM의 도움으로 자체 내부 상태를 유지합니다.

DOM이 입력 값을 제어하기 때문에, 입력 필드 값을 가져오려면 ref를 사용해야 합니다. 이 방법은 코드가 적게 들고 refs가 React를 다시 렌더링하지 않기 때문에 성능이 더 좋습니다.

비제어된 입력 예시는 다음과 같습니다:

```jsx
import { useRef } from "react";

function App() {
  const nameRef = useRef();

  const handleSubmit = (e) => {
    e.preventDefault();
    console.log(nameRef.current.value);
  };

  return (
    <form onSubmit={handleSubmit}>
      <label htmlFor="name">Your</label>{" "}
      <input type="text" ref={nameRef} id="name" />
      <button type="submit">Submit</button>
    </form>
  );
}

export default App;
```

비제어된 입력의 매우 눈에 띄는 장점은 코드가 적게 든다는 점입니다. 또한 성능이 더 좋고 HTML에 익숙한 React 초보자에게 더 자연스럽게 느껴집니다.

그렇다면 제어된 입력과 비제어된 입력 중 어느 것을 사용해야 할까요?

동적 폼 업데이트, 실시간 유효성 검사, 입력 값을 상태와 동기화해야 할 때는 제어된 입력을 사용하세요. 제어된 입력은 더 나은 제어를 제공하지만 더 많은 재렌더링이 필요합니다.

더 단순한 폼이 필요하거나, 제출 시에만 값을 접근하거나, React가 아닌 코드를 다룰 때는 비제어된 입력을 사용하세요.

제어된 입력이든 비제어된 입력이든 React에서 폼을 만들 때 반드시 지켜야 할 모범 사례는 다음과 같습니다:

- 항상 기본 폼 제출을 방지하세요.

- 제출 전에 입력을 반드시 유효성 검사하세요.

- 로딩, 유효성 검사 오류 또는 기타 관련 상태에 대해 사용자에게 명확한 피드백을 항상 제공하세요.
