# --description--

在 JavaScript 中，模块是一个自包含的代码单元，封装了相关的函数、类或变量。

将模块视为你的应用的构建块，就像书中的章节一样。每个模块专注于特定的功能性，使你的代码更加有序、易于维护且可重用。模块有助于防止命名冲突，并允许你将应用结构化为独立且相互关联的部分。

JavaScript 中的模块概念随着时间演变，但最广泛使用和支持的方法是 ES6（ECMAScript 2015）模块系统。该系统提供了一种在不同 JavaScript 环境中定义和使用模块的标准化方式。

要创建一个模块，你需要将你的 JavaScript 代码写在一个单独的文件中。任何你想让应用其他部分使用的变量、函数或类都需要显式导出。你可以使用 `export` 关键字来实现这一点。

例如，假设你有一个名为 `math.js` 的文件，其中包含一些数学函数：

```js
export function add(a, b) {
    return a + b;
}

export function subtract(a, b) {
    return a - b;
}

const PI = 3.14159;
export { PI };
```

在这个示例中，我们导出两个函数（`add` 和 `subtract`）和一个常量（`PI`）。你可以从单个模块导出任意数量的项。

要在你的应用的其他部分使用这些导出的项，你需要导入它们。这是通过使用 `import` 关键字完成的。

假设你想在名为 `app.js` 的文件中使用这些数学函数：

```js
import { add, subtract, PI } from './math.js';

console.log(add(5, 3));        // Outputs: 8
console.log(subtract(10, 4));  // Outputs: 6
console.log(PI);               // Outputs: 3.14159
```

这里，我们从 `math.js` 模块导入了我们需要的特定函数和常量。`'./math.js'` 部分告诉 JavaScript 相对于当前文件在哪里查找模块文件。

有时，你可能想导入一个模块导出的所有内容。你可以使用星号（`*`）语法来实现：

```js
import * as Math from './math.js';

console.log(Math.add(5, 3));        // Outputs: 8
console.log(Math.subtract(10, 4));  // Outputs: 6
console.log(Math.PI);               // Outputs: 3.14159
```

在这种情况下，所有来自 `math.js` 的导出都作为名为 `Math` 的对象的属性被导入。

另一个常见的模式是在模块中有一个默认导出。通常当模块主要导出单个函数时使用。每个模块只能有一个默认导出。

这就是它的样子：

```js
// In math.js
export default function multiply(a, b) {
    return a * b;
}

// In app.js
import multiply from './math.js';

console.log(multiply(4, 5));  // Outputs: 20
```

请注意，当导入一个默认导出时，你不需要使用括弧状的花括弧，并且你可以随意命名该导入。

重要的是要注意，要在浏览器中使用 ES6 模块，你需要在你的 `script` 标签中将 `type` 指定为 `module`：

```html
<script type="module" src="app.js"></script>
```

模块提供了一种强大的方式来组织和构建你的 JavaScript 代码。它们允许你将应用拆分成更小、更易管理的部分，促进代码重用，并帮助维护清晰的关注点分离。当你构建更大型的应用时，你会发现模块成为你 JavaScript 开发工具包中的一个必备工具。
