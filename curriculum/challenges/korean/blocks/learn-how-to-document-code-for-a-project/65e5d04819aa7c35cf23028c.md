# --description--

단어 `so`는 원인과 결과를 연결하거나 어떤 이유를 설명할 때 쓰는 연결어입니다. 기술과 프로그래밍 분야에서는 `so`가 문제나 상황과 해결책 또는 결과를 연결하는 데 자주 사용됩니다.

# --questions--

## --text--

기술 맥락에서 원인과 결과를 올바르게 연결하는 데 `so`가 사용된 문장을 고르세요.

## --answers--

`The program crashed, so we must reboot the computer.`

---

`We wrote the code, so but it didn't compile.`

### --feedback--

여기서 `so`의 사용이 잘못되었습니다. `But`는 `so` 뒤에 올 수 없습니다. `The code didn't compile, so we rewrote it` 또는 `We wrote the code, but it didn't compile.` 중 하나여야 합니다.

---

`The server is online, so therefore we can't connect.`

### --feedback--

`so`과 `therefore`를 함께 쓰는 것은 중복입니다. `The server is online, so we can connect` 또는 `The server is online; therefore, we can connect.` 중 하나만 써야 합니다.

---

`We need more data, so we can ignore the results.`

### --feedback--

이 문장은 원인과 결과를 논리적으로 연결하지 못했습니다. `so`를 더 잘 쓰려면, `We need more data, so we will perform more tests.`가 적절합니다.
