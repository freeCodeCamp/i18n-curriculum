# --description--

React は、コンポーネントが正しい型のプロップスを受け取っているかを検証する便利な型チェック機能を提供しています。たとえば、アプリケーションが API を呼び出して配列であることを期待するデータを取得し、それをプロップとしてコンポーネントに渡す場合があります。コンポーネントに `propTypes` を設定して、そのデータが `array` 型であることを要求できます。これにより、データが他の型の場合に役立つ警告がスローされます。

プロップの型が事前にわかっている場合は、`propTypes` を設定することがベストプラクティスとされています。コンポーネントに対して `propTypes` プロパティを、`defaultProps` を定義したのと同じ方法で定義できます。これにより、特定のキーのプロップが指定された型で存在するかどうかをチェックします。以下は、`function` という名前のプロップに対して `handleClick` 型を要求する例です。

```js
MyComponent.propTypes = { handleClick: PropTypes.func.isRequired }
```

上の例では、`PropTypes.func` の部分が `handleClick` が関数であることをチェックしています。`isRequired` を追加すると、そのコンポーネントに `handleClick` が必須のプロパティであることを React に伝えます。そのプロップが提供されていない場合、警告が表示されます。また、`func` は `function` を表しています。JavaScript の7つのプリミティブ型のうち、`function` と `boolean`（`bool` と書かれる）は唯一、綴りが特殊です。プリミティブ型に加えて、他の型も利用可能です。たとえば、プロップが React 要素であることをチェックできます。すべてのオプションについては<a href="https://reactjs.org/docs/typechecking-with-proptypes.html#proptypes" target="_blank" rel="noopener noreferrer nofollow">ドキュメント</a>を参照してください。

**注意:** React v15.5.0 以降、`PropTypes` は React から独立してインポートされます。以下のようにします：`import PropTypes from 'prop-types';`

# --instructions--

`propTypes` コンポーネントに対して `Items` を定義し、`quantity` をプロップとして必須にし、その型が `number` であることを検証してください。


`Items` コンポーネントには、`propTypes` に値が必須であり、その値が数値であることを要求する `quantity` チェックが含まれているべきです。

```js
const noWhiteSpace = __helpers.removeWhiteSpace(code);
assert(noWhiteSpace.includes('quantity:PropTypes.number.isRequired') && noWhiteSpace.includes('Items.propTypes='));
```
