# --description--

在计算机科学中，遍历列表中的项是常见的操作。关于查找，有两个你应该了解的关键算法：线性查找和二分查找。

线性查找从列表的开头开始，迭代每一项，直到它找到要查找的目标值。

如果找到目标值，则返回它在列表中的索引。如果未找到目标值，则返回 `-1`。我们返回 `-1` 是因为它在大多数编程语言中不是有效的索引。

下面是线性查找的代码示例：

```javascript
function linearSearch(arr, target) {
    for (let i = 0; i < arr.length; i++) {
        if (arr[i] === target) {
            return i;
        }
    }
    return -1;
}
```

如果我们要查找的列表是 `[13, 4, 7, 9, 10]` 且目标值是 `9`，函数将返回 `3`，因为 `9` 位于索引 `3`。

如果我们将目标值更改为 `5`，函数将返回 `-1`，因为 `5` 不在列表中。

虽然这是一个相对简单的算法，但它不是最高效的。如果你的列表中有大量项，线性查找可能需要很长时间才能找到目标值。

线性查找的时间复杂度是 `O(n)`，因为查找列表所需的时间随着列表大小线性增长。

线性查找的空间复杂度是 `O(1)`，因为它不需要任何额外空间来查找列表。

二分查找是一种更高效的算法，用于在大量项的列表中进行查找。这里的条件是列表必须按升序排序。

二分查找通过将列表分成两半并查看目标值是否在列表中间来工作。如果目标值在列表中间，则返回目标值的索引。否则，算法会查看目标值是在列表的左半部分还是右半部分。

它继续将列表的剩余部分分成两半，直到找到目标值。如果目标值不在列表中，它返回 `-1`。

以下是二分查找的代码示例：

```javascript
function binarySearch(arr, target) {
    let low = 0;
    let high = arr.length - 1;

    while (low <= high) {
        const mid = Math.floor((low + high) / 2);

        if (arr[mid] === target) {
            return mid;
        } else if (arr[mid] < target) {
            low = mid + 1;
        } else {
            high = mid - 1;
        }
    }

    return -1;
}
```

我们首先确定一个 `low` 和 `high` 索引。这表现了我们正在查找的列表的范围。

然后我们查看 `low` 是否小于或相等于 `high`。如果 `low` 大于 `high`，说明我们已经查找了整个列表且未找到目标值。在这种情况下，我们停止查找并返回 `-1`。

如果 `low` 索引小于或相等于 `high` 索引，我们计算列表的中间索引 `mid`。然后我们查看目标值是否在中间索引处。如果是，我们返回中间索引。

否则，我们查看中点处的值是否小于目标。如果是，我们将低索引更新为中间索引加一。这意味着我们将查找列表的右半部分。

最后，如果其他条件都不是 `True`，我们将把 `high` 索引更新为中间索引减一。这意味着我们将在列表的左半部分进行查找。

我们继续重复此进程，直到我们查找 `target` 或确定 `target` 不在 `list` 中。

二分查找的时间复杂度是 `O(log n)`，因为查找列表所需的时间随着列表大小呈对数增长。

二分查找的空间复杂度是 `O(1)`，因为它在查找列表时不需要任何额外空间。

二分查找和线性查找可用于你在计算机科学中遇到的各种问题。理解这两种算法之间的区别以及何时使用每一种算法非常重要。
