## Алгоритми та нотація Big O

- **Алгоритми**: Набір однозначних інструкцій для розв’язання задачі або виконання завдання. Алгоритми повинні завершуватися за скінченну кількість кроків, і кожен крок має бути точним і однозначним.

- **Нотація Big O**: Описує найгірший випадок продуктивності або швидкість зростання алгоритму при збільшенні розміру вхідних даних. Зосереджується на тому, як зростає використання ресурсів із розміром вхідних даних, ігноруючи сталий множник та члени нижчого порядку.

### Загальні часові складності

- **O(1) — константний час**: Алгоритм виконується за однаковий час незалежно від розміру вхідних даних.

```python
def check_even_or_odd(number):
    if number % 2 == 0:
        return 'Even'
    else:
        return 'Odd'
```

- **O(log n) — логарифмічний час**: Час зростає повільно зі збільшенням вхідних даних. Зустрічається в алгоритмах, які багаторазово зменшують розмір задачі на частку (наприклад, бінарний пошук).

- **O(n) — лінійний час**: Час виконання зростає пропорційно розміру вхідних даних.

```python
for grade in grades:
    print(grade)
```

- **O(n log n) — лог-лінійний час**: Загальна складність ефективних алгоритмів сортування, таких як Merge Sort і Quick Sort.

- **O(n²) — квадратичний час**: Час виконання зростає квадратично. Часто зустрічається у вкладених циклах.

```python
for i in range(n):
    for j in range(n):
        print("Hello, World!")
```

### Просторова складність

- **O(1) — константний простір**: Алгоритм використовує однаковий обсяг пам’яті незалежно від розміру вхідних даних.
- **O(n) — лінійний простір**: Використання пам’яті зростає пропорційно розміру вхідних даних.
- **O(n²) — квадратичний простір**: Використання пам’яті зростає квадратично з розміром вхідних даних.

## Техніки розв’язання задач

- **Розуміння задачі**: Прочитайте умову задачі кілька разів. Визначте вхідні дані, очікуваний вихід і як перетворити вхідні дані у вихідні.

- **Псевдокод**: Опис логіки алгоритму на високому рівні, незалежний від мови програмування. Використовує загальновживану мову, змішану з програмними конструкціями, такими як `IF`, `ELSE`, `FOR`, `WHILE`.

```md
GET original_string
SET reversed_string = ""
FOR EACH character IN original_string:
  ADD character TO THE BEGINNING OF reversed_string
DISPLAY reversed_string
```

- **Крайні випадки**: Конкретні, дійсні вхідні дані, що знаходяться на межах того, що алгоритм повинен обробляти. Завжди враховуйте та тестуйте крайні випадки.

## Масиви

- **Статичні масиви**: Мають фіксований розмір, визначений під час ініціалізації. Елементи зберігаються у суміжних ділянках пам’яті. Розмір не можна змінити під час виконання програми.

- **Динамічні масиви**: Можуть автоматично збільшуватися або зменшуватися під час виконання програми. Зміна розміру відбувається через автоматичне копіювання у більші масиви за потреби.

### Списки Python (динамічні масиви)

```python
numbers = [3, 4, 5, 6]

# Access elements
numbers[0]  # 3

# Update elements
numbers[2] = 16

# Add elements
numbers.append(7)
numbers.insert(3, 15)  # Insert at specific index

# Remove elements
numbers.pop(2)  # Remove at specific index
numbers.pop()   # Remove last element
```

### Часові складності для динамічних масивів

- **Доступ**: O(1)
- **Вставка в кінець**: O(1) у середньому, O(n) при зміні розміру
- **Вставка посередині**: O(n)
- **Видалення**: O(n) для середини, O(1) для кінця

## Стек

- **Стек**: Структура даних типу останній зайшов — перший вийшов (LIFO). Елементи додаються і видаляються лише з вершини.

- **Операція push**: Додавання елемента на вершину стеку. Часова складність: O(1).

- **Операція pop**: Видалення елемента з вершини стеку. Часова складність: O(1).

```python
# Using Python list as stack
stack = []

# Push operations
stack.append(1)
stack.append(2)
stack.append(3)

# Pop operations
top_element = stack.pop()  # Returns 3
```

## Черги

- **Черги**: Структура даних типу перший зайшов — перший вийшов (FIFO). Елементи додаються в кінець і видаляються з початку.

- **Операція enqueue**: Додавання елемента в кінець черги. Часова складність: O(1).

- **Операція dequeue**: Видалення елемента з початку черги. Часова складність: O(1).

```python
from collections import deque

# Using deque for efficient queue operations
queue = deque()

# Enqueue operations
queue.append(1)
queue.append(2)
queue.append(3)

# Dequeue operations
first_element = queue.popleft()  # Returns 1
```

## Зв’язані списки

- **Зв’язані списки**: Лінійна структура даних, де кожен вузол містить дані та референс на наступний вузол. Вузли з’єднані як ланцюг.

### Однозв’язні списки

- **Структура**: Кожен вузол має дані і один референс на наступний вузол.
- **Перебір**: Можна рухатися лише вперед від голови до хвоста.
- **Головний вузол**: Перший вузол у списку, зазвичай єдиний, до якого є прямий доступ.
- **Хвостовий вузол**: Останній вузол у списку, вказує на `None`.

### Операції та часові складності

- **Вставка на початок**: O(1)
- **Вставка в кінець**: O(n) — потрібно пройти до кінця
- **Вставка посередині**: O(n) — потрібно пройти до позиції
- **Видалення з початку**: O(1)
- **Видалення з кінця**: O(n) — потрібно знайти попередній вузол
- **Видалення посередині**: O(n) — потрібно знайти вузол

### Двозв’язні списки

- **Структура**: Кожен вузол має дані і два референси: на наступний і попередній вузол.
- **Перебір**: Можна рухатися в обох напрямках.
- **Пам’ять**: Потребує більше пам’яті, ніж однозв’язні списки через додатковий референс.

## Хеш-мапи та множини

### Мапи та хеш-мапи

- **Мапа (абстрактний тип даних)**: Керує колекціями пар ключ-значення. Кожен ключ має бути унікальним, але значення можуть повторюватися.

- **Хеш-мапа**: Конкретна реалізація мапи з використанням техніки хешування. Використовує хеш-функцію для генерації хеш-значень ключів, які визначають місце зберігання у базовому масиві.

### Словники Python (хеш-мапи)

```python
# Creating dictionaries
my_dictionary = {
    "A": 1,
    "B": 2, 
    "C": 3
}

# Alternative creation
my_dictionary = dict(A=1, B=2, C=3)

# Access and modify
value = my_dictionary["A"]  # 1
my_dictionary["A"] = 4      # Update value
del my_dictionary["A"]      # Remove key-value pair

# Check membership
"C" in my_dictionary

# Get keys, values, items
my_dictionary.keys()
my_dictionary.values()
my_dictionary.items()
```

### Часові складності для хеш-мап

- **Середній випадок**: O(1) для вставки, отримання, видалення
- **Найгірший випадок**: O(n) при великій кількості колізій хешів

### Множини

- **Множини**: Неупорядковані колекції унікальних елементів. Дублікатів немає, порядок не підтримується.

- **Тільки незмінні елементи**: Множини можуть містити лише незмінні типи даних (числа, рядки, кортежі), оскільки хеш-значення мають залишатися сталими.

```python
# Creating sets
numbers = {1, 2, 3, 4}
empty_set = set()  # Must use set(), not {}

# Add and remove elements
numbers.add(5)
numbers.remove(4)      # Raises KeyError if not found
numbers.discard(4)     # No error if not found

# Set operations
set_a = {1, 2, 3, 4}
set_b = {2, 3, 4, 5, 6}

# Union, intersection, difference, symmetric difference
set_a.union(set_b)                    # or set_a | set_b
set_a.intersection(set_b)             # or set_a & set_b
set_a.difference(set_b)               # or set_a - set_b
set_a.symmetric_difference(set_b)     # or set_a ^ set_b

# Subset and superset checks
set_a.issubset(set_b)
set_a.issuperset(set_b)
set_a.isdisjoint(set_b)

# Membership testing
5 in numbers
```

### Часові складності для множин

- **Середній випадок**: O(1) для додавання, видалення, перевірки належності
- **Найгірший випадок**: O(n) через колізії хешів

## Колізії хешів

- **Колізія хешів**: Виникає, коли два різні ключі дають однакове хеш-значення.

- **Стратегії розв’язання колізій**:
  - **Зв’язування (chaining)**: Кожен індекс масиву вказує на зв’язаний список, що зберігає всі елементи з однаковим хеш-значенням
  - **Відкрите адресування (open addressing)**: Пошук наступного доступного індексу за заздалегідь визначеною послідовністю

## Коли використовувати кожну структуру даних

- **Списки**: Коли потрібен впорядкований, індексований доступ і розмір невідомий заздалегідь
- **Стек**: Для операцій LIFO (функціонал скасування, обчислення виразів, повернення назад)
- **Черги**: Для операцій FIFO (планування завдань, пошук у ширину)
- **Зв’язані списки**: Коли часто вставляють/видаляють на початку, розмір невідомий, не потрібен випадковий доступ
- **Хеш-мапи**: Для швидкого пошуку пар ключ-значення, підрахунку появ, кешування
- **Множини**: Для перевірки унікальності, математичних операцій над множинами, видалення дублікатів
