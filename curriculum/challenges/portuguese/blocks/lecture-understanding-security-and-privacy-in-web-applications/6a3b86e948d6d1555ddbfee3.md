# --description--

A política de mesma origem é uma medida de segurança embutida nos seus navegadores que impede que um site leia dados privados de outro site. Essa regra protege suas contas pessoais contra hackers.

Uma origem tem três partes específicas: o protocolo web, o nome do domínio e o número da porta.

Por exemplo, no endereço `https://example.com:443`, o protocolo é HTTPS, o nome do domínio é `example.com` e o número da porta é `443`.

Os navegadores frequentemente ocultam o número da porta se for o padrão. HTTPS usa a porta 443 por padrão, e HTTP usa a porta 80. Mesmo que esteja oculta, o navegador ainda verifica. Dois sites têm a mesma origem somente quando essas três partes coincidem exatamente.

Sob essa regra rígida, o navegador bloqueia um site de bisbilhotar outro site diferente. Imagine que você abre sua conta bancária online em uma aba do navegador e abre um site malicioso em outra aba. A política de mesma origem bloqueia o site malicioso de roubar suas informações bancárias.

No entanto, sites modernos frequentemente precisam compartilhar dados entre origens diferentes. Por exemplo, uma página frontend pode precisar buscar dados de um servidor backend separado. As regras de mesma origem nos navegadores bloqueiam essas requisições seguras, então os desenvolvedores precisaram de uma forma de compartilhar dados com segurança. É aí que entra o CORS.

CORS é um sistema que permite aos servidores abrir acesso através das regras do navegador. Ele usa cabeçalhos especiais para conceder acesso a dados para sites confiáveis.

Veja o que acontece quando seu navegador pede dados de outro site:

* Ele inclui um cabeçalho `Origin` que nomeia o site que está fazendo a requisição
* O servidor que recebe então verifica esse nome
* Se o servidor confiar no site, ele responde com um cabeçalho especial de permissão que diz ao navegador que o site pode ler os dados recebidos.

Se o servidor não enviar essa permissão, o navegador bloqueia os dados imediatamente. O site frontend então recebe uma mensagem de erro em vez dos arquivos privados.

Em aplicações Node.js usando Express, o CORS é implementado com o pacote `cors`:

```javascript
const express = require('express');
const cors = require('cors');
const app = express();

// this enables CORS for all origins and all routes for development
app.use(cors());
```

Em produção, você pode precisar passar um objeto de configuração para restringir o acesso a um domínio específico:

```javascript
const corsOptions = {
  origin: ['https://yourapp.com', 'https://admin.yourapp.com'],
  credentials: true
};

app.use(cors(corsOptions));
```

Se você estiver usando Node.js puro, terá que lidar com isso manualmente definindo os cabeçalhos de resposta CORS você mesmo em cada requisição:

```javascript
res.setHeader('Access-Control-Allow-Origin', 'https://example.com');
res.setHeader('Access-Control-Allow-Methods', 'GET, POST, PUT, DELETE, OPTIONS');
res.setHeader('Access-Control-Allow-Headers', 'Content-Type, Authorization');
```
