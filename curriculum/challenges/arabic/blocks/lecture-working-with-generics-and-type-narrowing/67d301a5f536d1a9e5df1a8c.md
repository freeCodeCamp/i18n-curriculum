# --description--

ستواجه أحيانًا قيمة بنوع عام وتحتاج إلى تضييقها إلى نوع أكثر تحديدًا. على سبيل المثال، قد تحتاج إلى التأكد من أن كائنًا يطابق واجهة قمت بتعريفها. أو أن سلسلة نصية موجودة ضمن قائمة محددة من القيم. هناك عدة طرق لتحقيق ذلك.

الأولى هي التضييق بناءً على القيمة الصحيحة. انظر إلى مثالنا من الدرس السابق:

```js
const email = document.querySelector<HTMLInputElement>("#email");
console.log(email.value);
```

نحصل على خطأ من المُجمّع عند محاولة الوصول إلى خاصية `value` في `email`، لأن `email` قد تكون `null`. لكن يمكننا استخدام جملة شرطية لتأكيد أن `email` قيمة صحيحة قبل الوصول إلى الخاصية:

```js
const email = document.querySelector<HTMLInputElement>("#email");
if (email) {
  console.log(email.value);
}
```

في هذا المثال المحدث، بما أن `null` ليست قيمة صحيحة، يستطيع TypeScript استنتاج أن `email` يجب أن يكون عنصر `input` داخل كتلة الشرط. لذلك لم يعد يرمي خطأ من المُجمّع.

يمكن لفحوصات القيمة الصحيحة أن تعمل أيضًا في الاتجاه العكسي:

```js
const email = document.querySelector<HTMLInputElement>("#email");
if (!email) {
  throw new ReferenceError("Could not find email element!")
}
console.log(email.value);
```

بهذه الطريقة، نرمي خطأ إذا كانت `email` قيمة خاطئة. `null` هي قيمة خاطئة. رمي الخطأ ينهي التنفيذ المنطقي لهذا الكود، مما يعني أنه عند الوصول إلى استدعاء `console.log()` يعرف TypeScript أن `email` لا يمكن أن تكون `null`.

التسلسل الاختياري هو أيضًا شكل من أشكال تضييق النوع، على نفس المبدأ بأن الوصول إلى الخاصية لا يمكن أن يحدث إذا كانت قيمة `email` هي `null`.

```js
const email = document.querySelector<HTMLInputElement>("#email");
console.log(email?.value);
```

لكن ماذا عن الأنواع الأخرى؟ يمكنك أيضًا تضييق الأنواع باستخدام عامل `typeof`. لننظر إلى مثال لمتغير قد يكون سلسلة نصية أو رقمًا:

```js
const myVal = Math.random() > 0.5 ? 222 : "222";
console.log(myVal / 10)
```

في هذا المثال، نرى خطأ من المُجمّع لأننا لا نستطيع إجراء عمليات حسابية على قيمة سلسلة نصية. لكن يمكننا استخدام شرط لفحص `typeof` المتغير `myVal`:

```js
const myVal = Math.random() > 0.5 ? 222 : "222";
if (typeof myVal === "number") {
  console.log(myVal / 10);
}
```

بما أننا استخدمنا كلمة `typeof`، يعرف TypeScript الآن أن `myVal` يجب أن يكون رقمًا، ويمكننا إجراء العمليات الحسابية بأمان.

لكن ماذا عن أنواع الكائنات الأكثر تعقيدًا؟ إذا كان الكائن من صنف، يمكنك استخدام كلمة `instanceof` لتضييق النوع. نعود إلى مثالنا مع `querySelector()`:

```js
const email = document.querySelector("#email");
```

بدلاً من تمرير نوع عام وإخبار TypeScript ما هو العنصر، يمكننا استخدام `instanceof` لتضييق النوع وكتابة شفرة أكثر أمانًا:

```js
const email = document.querySelector("#email");

if (email instanceof HTMLInputElement) {
    console.log(email.value);
}
```

قد يبدو هذا الأسلوب مشابهًا للسابق، لكن `instanceof` هو تحقق وقت التشغيل - مما يعني، إذا أخطأنا في نوع TypeScript، فإن شفرة JavaScript ستؤكد أن `email` هو عنصر `input`.

بعد ذلك، لننظر إلى مثال حيث نسترجع كائن `User` من API ونحاول طباعة المعلومات:

```js
interface User {
    name: string;
    age: number;
}

const printAge = (user: User) => 
  console.log(`${user.name} is ${user.age} years old!`)

const request = await fetch("url")
const myUser = await request.json();
printAge(myUser);
```

سنحصل على خطأ من المُجمّع عند محاولة تمرير `myUser` إلى الدالة، لأنه رغم معرفتنا أن API يعيد الكائن الصحيح، فإن TypeScript لا يعرف ذلك. وطريقة `.json()` لا تقبل نوعًا عامًا.

الطريقة "السريعة" لحل هذه المشكلة هي تحويل النوع:

```js
interface User {
    name: string;
    age: number;
}

const printAge = (user: User) => 
  console.log(`${user.name} is ${user.age} years old!`)

const request = await fetch("url")
const myUser = await request.json() as User;
printAge(myUser);
```

لكن كلما حولت النوع، فأنت تضعف قدرة TypeScript على اكتشاف الأخطاء المحتملة. لذلك بدلًا من تحويل النوع، يمكنك كتابة حارس نوع:

```js
interface User {
    name: string;
    age: number;
}

const isValidUser = (user: unknown): user is User => {
  return !!user && 
    typeof user === "object" &&
    "name" in user &&
    "age" in user;
}
```

نوع الإرجاع هنا هو العنصر الأساسي في تعريف هذه الدالة. صيغة `user is User` تشير إلى أن دالتنا تُرجع قيمة منطقية، وعندما تكون `true` فهذا يعني أن قيمة `user` تحقق واجهة `User`. بعد ذلك نقوم ببعض الفحوصات الأساسية للتأكد من أن هيكل كائن `user` يتطابق — لاحظ استخدام تضييق القيمة المنطقية (`!!user`) وتضييق باستخدام `typeof`. يجب أن نفعل هذا لأن `typeof null` تُرجع `"object"`:

```js
interface User {
    name: string;
    age: number;
}

const isValidUser = (user: unknown): user is User => {
  return !!user && 
    typeof user === "object" &&
    "name" in user &&
    "age" in user;
}

const printAge = (user: User) => 
  console.log(`${user.name} is ${user.age} years old!`)

const request = await fetch("url")
const myUser = await request.json() as User;
if (isValidUser(myUser)) {
    printAge(myUser);
}
```

الآن، إذا جمعنا كل منطقنا معًا، لن نحصل على أخطاء من المُجمّع ويمكننا بناء شفرتنا بنجاح.

تضييق النوع ميزة قوية تساعدك على كتابة شفرة أكثر أمانًا وأقل عرضة للأخطاء - لكن تذكر أن أنواع TypeScript ليست صارمة تمامًا، فتجنب ممارسات مثل تحويل نوع قيمة دون تضييقها.
