# --interactive--

条件文はJavaScriptのコードで意思決定を可能にします。特定の条件に基づいてプログラムの流れを制御できます。`if`、`else if`、`else`、および三項演算子がどのようにコードの流れを制御するか見てみましょう。

`if`文は条件を受け取り、その条件が真値の場合にコードのブロックを実行します。真値とは、`true`文のようなブール文脈で評価したときに`if`となる値のことです。真値の例をいくつか挙げます：

- 空でない文字列、例えば `hello`

- `0` と `-0` 以外の任意の数値、例えば `4`、`-5` など

- 配列

- オブジェクト

- ブール値の `true`

一方、偽値とはブール文脈で`false`と評価される値です。JavaScriptには偽値が少なく、覚えやすいです。いくつかの偽値を挙げます：

- ブール値の `false`

- `0`（ゼロ）

- `""`（空文字列）

- `null`

- `undefined`

- `NaN`（Not a Number）

真値と偽値の基本を理解したところで、`if`文での動作を見てみましょう。最初の例では、いくつかの`if`文を使って真値と偽値をチェックしています：

:::interactive_editor

```js
if (null) {
  console.log("This will not run.");
}

if ("freeCodeCamp") {
  console.log("This will run.");
}
```

:::

`null`は偽値なので、ブロック内のメッセージはコンソールに表示されません。しかし2つ目の`if`文では、文字列`freeCodeCamp`が真値であり、`true`文のブール文脈で`if`とみなされます。その結果、メッセージ`This will run.`がコンソールに表示されます。

次に、`if`文が異なる比較演算子とどのように動作するかの例をいくつか見てみましょう。以下は、ユーザーが投票資格があるかどうかをチェックする`if`文の例です：

:::interactive_editor

```js
const age = 22;

if (age >= 18) {
 console.log("You're eligible to vote"); // You're eligible to vote
}
```

:::

この例では、`age`が現在`22`なので、条件は`true`と評価されます。なぜなら`22`が`18`以上だからです。したがって、メッセージ`You're eligible to vote`がコンソールに表示されます。もし`age`を`15`に変更すると、条件は`false`と評価され、メッセージはコンソールに表示されません：

:::interactive_editor

```js
const age = 15;

if (age >= 18) {
 console.log("You're eligible to vote"); // Code not running because age is less than 18
}
```

:::

条件が`false`の場合、`else`節を使うことができます：

:::interactive_editor

```js
const age = 15;

if (age >= 18) {
 console.log("You're eligible to vote");
} else {
 console.log("You're not eligible to vote"); // You're not eligible to vote
}
```

:::

この例では、`15`は`18`以上ではないため、条件は`false`となります。この場合、`else`ブロック内のコードが実行されます。

複数の条件をチェックしたい場合は、`else if`ブロックを使えます。これにより、プログラムは2つ以上の経路から選択できます。

:::interactive_editor

```js
const score = 87;

if (score >= 90) {
 console.log('You got an A'); 
} else if (score >= 80) {
 console.log('You got a B'); // You got a B
} else if (score >= 70) {
 console.log('You got a C');
} else {
 console.log('You failed! You need to study more!');
}
```

:::

`score`が現在`87`なので、メッセージ`You got a B`がコンソールに表示されます。

三項演算子は、単純な`if/else`文を書くためのコンパクトな方法です。3つの部分から成り立っています：条件、条件が真の場合の結果、条件が偽の場合の結果。基本的な構文は以下の通りです：

```js
condition ? expressionIfTrue : expressionIfFalse;
```

摂氏の気温を扱う例を見てみましょう：

:::interactive_editor

```js
const temperature = 30;
const weather = temperature > 25 ? 'sunny' : 'cool';

console.log(`It's a ${weather} day!`);
```

:::

`temperature`が`25`より大きい場合、上のコードは`It's a sunny day!`をログに出します。`temperature`が`25`以下の場合は、`It's a cool day!`をログに出します。

では、if文と三項演算子のどちらを使うべきでしょうか？単一の条件や単一の式を扱う場合、または単純なロジックのコンパクトな構文が欲しい場合は三項演算子を使ってください。複雑な条件や複数の文を扱う場合は`if/else`文を使うことをお勧めします。三項演算子をネストすると読みにくくなるからです。
