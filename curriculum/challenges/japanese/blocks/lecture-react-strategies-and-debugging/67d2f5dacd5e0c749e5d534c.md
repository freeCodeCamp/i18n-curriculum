# --description--

ブラウザには HTML、CSS、JavaScript をデバッグするための組み込みの開発者ツールがあります。

しかし、React アプリのバグを見つけて修正するにはあまり適していません。そこで React チームは「React Developer Tools」（通称 React DevTools）というツールを開発し、React アプリを検査、デバッグ、プロファイルできるようにしました。

React DevTools は Chrome、Edge、Firefox 用のブラウザ拡張機能として利用できます。Chrome または Edge を使っている場合は、Chrome ウェブストアにアクセスし、「React Developer Tools」を検索してブラウザに追加してください。

Firefox を使っている場合は、Firefox アドオンページにアクセスし、同じようにツールを検索してブラウザに追加してください。

Safari を使っている場合は、npm から React DevTools をインストールできます。`npm install -g react-devtools` または `yarn global add react-devtools` を実行してください。

React DevTools をインストールして有効にした後、ブラウザで React アプリを開き、ブラウザの開発者ツールを開くと、追加で「Components」と「Profiler」という2つのタブが表示されます。

Components タブはコンポーネントをツリー形式で表示します。これにより、以下のことができます：

- アプリのコンポーネント階層を確認する。
- プロップス、ステート、コンテキストの値をリアルタイムで確認・変更する。
- 選択したコンポーネントのソースコードを確認する。
- コンポーネントのデータをコンソールにログ出力する。
- コンポーネントの DOM 要素を検査する。

一方、Profiler タブはコンポーネントのパフォーマンスを記録・分析し、不要な再レンダリングを特定したり、コミット時間を確認したり、最適化できるポイントを見つけたりするのに役立ちます。

ここでは、コンポーネントやそのプロップスやステートを検査する方法を示す簡単なアプリを紹介します。これは以前のプロップドリリングに関するレッスンで使ったコードに似ています：

```jsx
import { useState } from "react";

export default function App() {
  const greeting = "Hello, Prop Drilling!";
  const response = "I'm not here to play!";

  return <Parent greeting={greeting} response={response} />;
}

const Parent = ({ greeting, response }) => {
  return <Child greeting={greeting} response={response} />;
};

const Child = ({ greeting, response }) => {
  return <Grandchild greeting={greeting} response={response} />;
};

const Grandchild = ({ greeting, response }) => {
  const [count, setCount] = useState(0);

  return (
    <>
      <h1>{greeting}</h1>
      <h2>{response}</h2>

      <button onClick={() => setCount(count + 1)}>Increase Count</button>
      <h2>Count: {count}</h2>
      <button onClick={() => setCount(count - 1)}>Decrease Count</button>
    </>
  );
};
```

React DevTools の Components タブを見ると、コンポーネントのツリー表示が確認できます。`App` コンポーネントがトップにあり、その下に `Parent`、`Child`、`Grandchild` コンポーネントが続きます。

これらのコンポーネントのいずれかを選択すると、その中のプロップスやステートが見えます。`Parent` コンポーネントを選択すると、`greeting` と `response` のプロップスが表示され、それぞれ `Hello, Prop Drilling!` と `I'm not here to play!` です。

プロップスやステートはリアルタイムで確認・更新でき、必要に応じて変更も可能です。例えば、`Grandchild` コンポーネントを選択し、`greeting` プロップを `Hello, Prop Drilling!` から `Hello, Welcome to Prop Drilling!` に変更すると、ページに即座に反映されます。

コンポーネントのデータをコンソールにログ出力したり、対応する DOM 要素を検査したり、コンポーネントのソースコードを確認したりするには、右上のアイコンを使います。`Grandchild` コンポーネントを選択し、`Log the component data to the console` ボタンをクリックすると、プロップス、ステート、フック、ノードなどのデータがコンソールにログ出力されます。

React でよくあるバグの一つにプロップスの不一致があります。

例えば、`Child` コンポーネントに対して、`reply` の代わりに誤って `response` をプロップとして渡した場合：

```jsx
const Child = ({ greeting, response }) => {
  return <Grandchild greeting={greeting} reply={response} />;
};
```

`Grandchild` は `response` プロップを期待しています。異なるプロップが渡されたため、そのテキストはページに表示されず、空の `h2` が DOM に追加されるだけです。結果として、`h1` というテキストがある `Hello, Prop Drilling!` 要素と、他のボタンやテキストだけがページに表示されます。空の `h2` 要素は存在しますが、中身が空なので DOM を検査しないと見えません。

これを修正するには、`Parent` コンポーネントから `Child` までのプロップの流れを検査し、プロップ名を直接編集します。Components タブで `Child` コンポーネントを選択し、`reply` プロップを `response` に変更すると、ページに `h2` 要素が `I'm not here to play!` というテキスト付きで表示されます。
