# --interactive--

Gli operatori logici binari ti aiutano a valutare due espressioni e a restituire un risultato basato sulla loro veridicità. Vediamo i tre operatori logici binari più comuni: l'AND logico, l'OR logico e l'operatore di coalescenza nullish.

L'operatore AND logico è rappresentato da un doppio e commerciale (`&&`). Controlla se entrambi gli operandi sono veri e restituisce un risultato. Se entrambi gli operandi sono truthy, restituisce il secondo valore, cioè quello a destra:

:::interactive_editor

```js
const result = true && 'hello';

console.log(result); // hello
```

:::

Nell'esempio sopra, il testo `hello` viene mostrato nella console perché entrambi gli operandi sono `true`. Se uno degli operandi è falsy, restituisce il valore falsy:

:::interactive_editor

```js
const result = 0 && 3;

console.log(result); // 0
```

:::

Poiché `0` è un valore falsy, il numero `0` viene mostrato nella console. E se entrambi gli operandi sono falsy, restituisce il primo valore falsy:

:::interactive_editor

```js
const result = false && 0;

console.log(result); // false
```

:::

Poiché `false` è un valore falsy, allora `false` viene mostrato nella console. L'operatore AND logico è utile quando vuoi verificare più condizioni e assicurarti che tutte siano vere prima di procedere. Ecco un esempio:

:::interactive_editor

```js
if (2 < 3 && 3 < 4) {
 console.log('The if block runs'); 
} else {
 console.log('The else block runs');
} 
```

:::

Nella condizione, dato che `2` è minore di `3` E `3` è minore di `4`, allora la frase `The if block runs` verrà mostrata nella console.

L'operatore OR logico verifica se almeno uno degli operandi è truthy. Se il primo operando è truthy, restituisce quel valore:

:::interactive_editor

```js
const result = 'This is truthy' || false;

console.log(result); // This is truthy
```

:::

Se il primo operando è falsy ma il secondo è truthy, il secondo valore verrà mostrato nella console:

:::interactive_editor

```js
const result = 0 || 'This is truthy';

console.log(result); // This is truthy
```

:::

È comune usare l'operatore OR logico in istruzioni `if/else` come questa:

:::interactive_editor

```js
let userInput;

if (userInput || 'Guest') {
 console.log('A user is present');
} else {
 console.log('No user detected');
}
```

:::

Poiché non abbiamo assegnato un valore alla variabile `userInput`, attualmente è `undefined`. La condizione nell'istruzione `if` verifica se la variabile `userInput` o la stringa `Guest` sono truthy. Poiché la stringa `Guest` è vera in un contesto booleano come questo, la stringa `A user is present` verrà mostrata nella console.

L'operatore di coalescenza nullish è più sofisticato dell'OR logico e dell'AND logico. Rappresentato da un doppio punto interrogativo (`??`), aiuta in scenari in cui vuoi restituire un valore solo se il primo è `null` o `undefined`. Ecco un esempio di utilizzo dell'operatore di coalescenza nullish:

:::interactive_editor

```ts
const result = null ?? 'default';

console.log(result); // default
```

:::

Poiché `null` è un valore nullish, la stringa `default` verrebbe mostrata nella console. L'operatore di coalescenza nullish è incredibilmente utile in situazioni in cui `null` o `undefined` sono gli unici valori che dovrebbero attivare un valore fallback o predefinito. Ecco un esempio di gestione delle impostazioni di preferenza di un utente:


:::interactive_editor

```ts
const userSettings = {
 theme: null,
 volume: 0,
 notifications: false,
};

let theme = userSettings.theme ?? 'light';
console.log(theme); // light
```

:::

Nell'esempio sopra, abbiamo un oggetto chiamato `userSettings` che contiene le proprietà `theme`, `volume` e `notifications`. Stiamo consultando la prop `theme` usando la notazione a punto come `userSettings.theme`. Imparerai di più su come lavorare con gli oggetti in una futura lezione. Poiché la `theme` dell'utente è attualmente impostata su `null`, allora la stringa `light` verrà mostrata nella console.
