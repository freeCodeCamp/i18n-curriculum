# --description--

Reactコンポーネントを定義するもう一つの方法は、ES6の`class`構文を使うことです。以下の例では、`Kitten`が`React.Component`を拡張しています。

```jsx
class Kitten extends React.Component {
  constructor(props) {
    super(props);
  }

  render() {
    return (
      <h1>Hi</h1>
    );
  }
}
```

これは、`Kitten`というES6クラスを作成し、`React.Component`クラスを拡張しています。これにより、`Kitten`クラスはローカルステートやライフサイクルフックなど、多くの便利なReactの機能にアクセスできるようになります。これらの用語にまだ慣れていなくても心配いりません。後のチャレンジで詳しく扱います。また、`Kitten`クラスには`constructor`が定義されており、`super()`を呼び出していることに注目してください。これは親クラス、ここでは`super()`のコンストラクタを呼び出すために`React.Component`を使っています。コンストラクタは、`class`キーワードで作成されたオブジェクトの初期化時に使われる特別なメソッドです。コンポーネントの`constructor`を`super`で呼び出し、`props`を両方に渡すのがベストプラクティスです。これによりコンポーネントが正しく初期化されます。今はこのコードが標準的に含まれていることを知っておいてください。すぐにコンストラクタの他の使い方や`props`も学びます。

コードエディタで`MyComponent`がクラス構文で定義されています。`render`メソッドの記述を完成させて、テキスト`div`を含む`h1`を返す`Hello React!`要素を返してください。

# --hints--

Reactコンポーネントは`div`要素を返すべきです。

```js
assert(Enzyme.shallow(React.createElement(MyComponent)).type() === 'div');
```

返される`div`は、その中に`h1`の見出し要素をレンダリングする必要があります。

```js
assert(
  /<div><h1>.*<\/h1><\/div>/.test(
    Enzyme.shallow(React.createElement(MyComponent)).html()
  )
);
```

`h1`の見出し要素は文字列`Hello React!`を含むべきです。

```js
assert(
  Enzyme.shallow(React.createElement(MyComponent)).html() ===
    '<div><h1>Hello React!</h1></div>'
);
```
