# --interactive--

In einer vorherigen Lektion haben Sie gelernt, wie Sie HTML verwenden, um die Werte einzuschränken, die Ihre Nutzer in Ihrem Formular absenden können. Aber manchmal reicht das nicht aus. Wenn Sie komplexere Anforderungen haben, wie zum Beispiel eigene Fehlermeldungen für den Nutzer anzuzeigen, müssen Sie JavaScript verwenden.

Bestimmte HTML-Elemente, wie die `textarea` und `input` Elemente, machen eine Constraint Validation API verfügbar. Diese API erlaubt es Ihnen zu überprüfen, ob der vom Nutzer eingegebene Wert für dieses Element alle HTML-Validierungen erfüllt, die Sie definiert haben, wie zum Beispiel Mindestlänge oder Musterabgleich.

Aber wie können Sie diese API tatsächlich nutzen? Angenommen, Sie möchten, dass Mitarbeitende eines Unternehmens Feedback-Nachrichten über ein Formular wie dieses senden:

Im obigen Beispiel haben wir unser Eingabefeld aus dem DOM abgefragt und einen Event-Listener für Eingaben hinzugefügt.

Wir wissen, dass `e.target` sich auf das Element bezieht, das das Ereignis ausgelöst hat. In diesem Fall unser `input`. Aber was macht die `checkValidity()`-Methode?

Diese ist Teil der Constraint Validation API. Die `checkValidity()`-Methode gibt `true` zurück, wenn das Element alle HTML-Validierungen (basierend auf seinen Attributen) besteht, und `false`, wenn es fehlschlägt.

Wenn wir eine ungültige Eingabe testen, sehen wir, dass `false` in der Konsole protokolliert wird. Nun, da wir wissen, dass die Eingabe ungültig ist, melden wir die Ungültigkeit:

Es meldet den ungültigen Zustand sofort, anstatt zu warten, bis wir das Formular absenden. Es verwendet jedoch weiterhin die Standardmeldung. Das liegt daran, dass die `reportValidity()`-Methode dem Browser nur mitteilt, dass die Eingabe ungültig ist. Der Browser entscheidet weiterhin, wie er anzeigt, warum sie ungültig ist. input.addEventListener("input", (e) => {
```js
  if (!e.target.checkValidity()) {
    e.target.setCustomValidity(
      "You must use a company email address that ends in @sampleCompany.com"
    );
  }
});
```

Die `validity`-Eigenschaft ist eine Instanz des `ValidityState`-Objekts. So könnte das Objekt im Browser aussehen:

```js
ValidityState {
  badInput: false,
  customError: false,
  patternMismatch: true,
  rangeOverflow: false,
  rangeUnderflow: false,
  stepMismatch: false,
  tooLong: false,
  tooShort: false,
  typeMismatch: true,
  valueMissing: false,
  valid: false
}
```

Es gibt mehrere hilfreiche Eigenschaften, die alle einen Boolean-Wert von `true` oder `false` haben.

Einige dieser hilfreichen Eigenschaften, die Sie selbst weiter erkunden können, sind die `valueMissing`-Eigenschaft, die `true` ist, wenn ein erforderliches Eingabefeld leer gelassen wird, oder die `patternMismatch`-Eigenschaft, die `true` ist, wenn der Wert nicht dem angegebenen regulären Ausdruck entspricht.

Nach dieser Lektion empfehle ich Ihnen, mit den Beispielen in dieser Lektion zu experimentieren und mehr über die verschiedenen Validitätseigenschaften zu erfahren.
