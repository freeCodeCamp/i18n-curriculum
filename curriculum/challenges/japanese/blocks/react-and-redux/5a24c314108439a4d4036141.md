# --description--

このチャレンジシリーズでは、ReactでReduxを使う方法を紹介します。まず、それぞれの技術の重要な原則を復習しましょう。Reactはデータを渡すと効率的かつ予測可能な方法でビューをレンダリングするビューライブラリです。Reduxはアプリケーションの状態管理を簡単にするためのフレームワークです。通常、React Reduxアプリでは、アプリ全体の状態を管理する単一のReduxストアを作成します。Reactコンポーネントは、自分の役割に関連するストア内のデータの一部だけを購読します。そして、Reactコンポーネントから直接アクションをディスパッチし、それがストアの更新を引き起こします。

Reactコンポーネントはローカルに自分の状態を管理できますが、複雑なアプリの場合は、Reduxでアプリの状態を単一の場所にまとめておく方が一般的に良いです。個々のコンポーネントだけに特有のローカル状態を持つ例外もあります。最後に、ReduxはReactとすぐに連携できるようには設計されていないため、`react-redux`パッケージを使う必要があります。これにより、Reduxの`state`と`dispatch`をReactコンポーネントに`props`として渡す方法が提供されます。

次のいくつかのチャレンジでは、まず新しいテキストメッセージを入力できるシンプルなReactコンポーネントを作成します。これらは配列に追加され、ビューに表示されます。これはReactのレッスンで学んだことの良い復習になるでしょう。次に、メッセージ配列の状態を管理するReduxストアとアクションを作成します。最後に、`react-redux`を使ってReduxストアとコンポーネントを接続し、ローカル状態をReduxストアに移行します。

# --instructions--

`DisplayMessages`コンポーネントから始めてください。このコンポーネントにコンストラクタを追加し、状態を初期化します。状態には2つのプロパティを持たせます。`input`は空の文字列に設定し、`messages`は空の配列に設定してください。
