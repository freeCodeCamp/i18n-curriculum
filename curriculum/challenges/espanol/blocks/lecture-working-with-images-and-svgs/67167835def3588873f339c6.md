# --description--

Hay tres herramientas a considerar al usar medios, como imágenes, en las páginas web: el tamaño, el formato y la compresión.

Primero, hablemos del tamaño de las imágenes. Cuando construyas un sitio web, a menudo darás estilo a las imágenes para mostrarlas en un tamaño específico.  For example, you might have an image display at a 640x480 resolution. 640 represents the width while 480 represents the height in pixels. Al preparar tu imagen, quieres escalarla a un tamaño de 640x480 para coincidir con ese estilo. Si sirves una imagen de 1920x1080 pero la estás estilizando para ser mucho más pequeña, estás requiriendo que los usuarios descarguen datos innecesarios. Una resolución más pequeña resulta en un tamaño de archivo más pequeño.

Lo siguiente a considerar es el formato de archivo. Dos de los formatos de archivo más comunes son PNG y JPG, pero ya no son los formatos más ideales para servir imágenes. A menos que necesites soporte para navegadores antiguos, deberías considerar usar un formato más optimizado, como WEBP o AVIF.

Finalmente, puedes ejecutar algoritmos de compresión en tus imágenes para reducir el tamaño del archivo. Herramientas como pngcrush funcionan bien para formatos sin pérdida como PNG, que pueden comprimirse sin ninguna pérdida de calidad ya que los datos originales pueden reconstruirse perfectamente. Sin embargo, no todos los formatos son sin pérdida. JPG, por ejemplo, usa compresión con pérdida: cada vez que un JPG se vuelve a guardar o re-comprimir, algunos datos de la imagen se descartan permanentemente, lo que resulta en una calidad degradada. Debes tener todo esto en cuenta al seleccionar y preparar imágenes para tus páginas web.
