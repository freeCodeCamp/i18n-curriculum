# --description--

`Versions` 在你的 `package.json` 檔案中相依性區段的 npm 軟體包遵循所謂的語意版本控制（SemVer），這是一種軟體版本控制的業界標準，旨在讓管理相依性更容易。在 npm 上發佈的函式庫、框架或其他工具應該使用 SemVer，以明確傳達專案在更新時可以預期的變更類型。

了解 SemVer 在你開發使用外部相依性的軟體時會很有幫助（你幾乎總是會這麼做）。有一天，你對這些數字的理解將會救你一命，避免你在不明白為何昨天可行的東西今天突然無法運作的情況下，意外地對你的專案引入破壞性變更。這就是根據官方網站所述的語意版本控制運作方式：

```json
"package": "MAJOR.MINOR.PATCH"
```

當你進行不相容的 API 變更時，應該遞增 MAJOR 版本。當你以向後相容的方式添加功能時，應該遞增 MINOR 版本。當你進行向後相容的錯誤修正時，應該遞增 PATCH 版本。這表示 PATCH 是錯誤修正，MINOR 則是新增特性，但兩者都不會破壞先前可正常運作的部分。最後，MAJOR 則是新增無法與早期版本相容的變更。

# --instructions--

在你的 `package.json` 檔案的相依性區段中，將 `@freecodecamp/example` 的版本更改為符合 MAJOR 版本 1、MINOR 版本 2 和 PATCH 版本 13。

# --hints--

`"dependencies"` 應該包含 `"@freecodecamp/example"`。

```js
fetch(code + '/_api/package.json')
  .then(response => response.json())
  .then(
    data => {
      assert.property(
        data.dependencies,
        '@freecodecamp/example',
        '"dependencies" does not include "@freecodecamp/example"'
      );
    },
    error => {
      throw new Error(error.message || error.responseText);
    }
  );
```

`"@freecodecamp/example"` 版本應該是 `"1.2.13"`。

```js
fetch(code + '/_api/package.json')
  .then(response => response.json())
  .then(
    data => {
      assert.equal(
        data.dependencies['@freecodecamp/example'],
        '1.2.13',
        'Wrong version of "@freecodecamp/example". It should be 1.2.13'
      );
    },
    error => {
      throw new Error(error.message || error.responseText);
    }
  );
```
