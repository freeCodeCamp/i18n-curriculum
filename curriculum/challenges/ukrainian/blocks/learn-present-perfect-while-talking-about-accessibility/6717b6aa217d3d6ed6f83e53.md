# --explanation--

Правка `tweak` означає зробити невелику правку або модифікацію. Наприклад:

`The developer tweaked the code to improve the app's performance.` — це означає, що розробник вніс невеликі зміни в код, щоб покращити роботу застосунку.

`Layouts` — це розташування або структура елементів на сторінці чи інтерфейсі. Наприклад:

`The layout of the homepage was changed to make it more user-friendly.` — це означає, що розташування елементів на головній сторінці було відкориговано, щоб користувачам було легше орієнтуватися.

`Guidelines` — це правила або принципи, які дають вказівки. Наприклад:

`The team followed the design guidelines to maintain consistency across pages.` — це означає, що команда дотримувалася конкретних правил, щоб забезпечити послідовний вигляд і структуру на різних сторінках.
