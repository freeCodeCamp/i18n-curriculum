# --description--

Node.js hat keinen eingebauten WebSocket-Server. Entwickler verwenden typischerweise externe Pakete wie `ws`, um das WebSocket-Protokoll zu implementieren und die Echtzeitkommunikation zwischen Clients und Servern zu handhaben.

Die `ws`-Bibliothek ist leichtgewichtig, schnell und einfach zu verwenden. Sie implementiert das WebSocket-Protokoll, sodass Sie WebSocket-Verbindungen erstellen und verwalten können.

Sie können sie mit npm installieren:

```bash
npm install ws
```

Nach der Installation erstellen Sie einen WebSocket-Server, indem Sie die `ws`-Bibliothek importieren, ihre `Server`-Methode aufrufen und einen Port übergeben:

```js
const WebSocket = require("ws");

const server = new WebSocket.Server({
  port: 3000,
});
```

Dies startet einen WebSocket-Server, der auf Verbindungen am Port `3000` hört.

Wenn sich ein Client verbindet, sendet der Server ein `connection`-Ereignis.

```js
server.on("connection", (socket) => {
  console.log("Client connected");
});
```

Betrachten Sie dieses Ereignis als die Art des Servers zu sagen: „Ein neuer Client ist beigetreten“.

Der `socket`-Parameter stellt die Verbindung zwischen dem Server und diesem Client dar. Sie verwenden ihn, um Nachrichten zu senden und zu empfangen.

Wenn ein verbundener Client Daten sendet, sendet der Server ein `message`-Ereignis.

```js
server.on("connection", (socket) => {
  socket.on("message", (message) => {
    console.log(message.toString());
  });
});
```

In diesem Beispiel hört der Server auf eingehende Nachrichten und protokolliert sie in der Konsole.

Sie können Daten an einen Client zurücksenden, indem Sie die `send()`-Methode verwenden:

```js
socket.send("Hello from the server");
```

Dies sendet eine Nachricht über die offene WebSocket-Verbindung.

Solange die Verbindung offen bleibt, können sowohl der Client als auch der Server weiterhin Nachrichten in Echtzeit austauschen.

Neben `connection` und `message` senden WebSocket-Verbindungen auch die `close`- und `error`-Ereignisse.

`close` wird ausgelöst, wenn eine Verbindung geschlossen wird:

```js
socket.on("close", () => {
  console.log("Disconnected");
});
```

`error` wird ausgelöst, wenn etwas schiefgeht:

```js
socket.on("error", (error) => {
  console.error(error);
});
```

Zusammen bilden diese Ereignisse die Grundlage der meisten WebSocket-Anwendungen.

Eine häufige Anforderung in Chat-Anwendungen ist das Broadcasting. Broadcasting bedeutet, eine Nachricht an mehrere verbundene Clients zu senden, nicht nur an einen.

Stellen Sie sich einen Chatraum mit drei Nutzern namens **John Doe**, **Jane Doe** und **Wes Smith** vor.

Wenn John Doe eine Nachricht sendet, muss der Server diese Nachricht möglicherweise an Jane Doe und Wes Smith weiterleiten.

Dazu verfolgt der Server die verbundenen Clients und durchläuft sie, wenn eine Nachricht eintrifft.

Bevor eine Nachricht gesendet wird, prüft der Server, ob die Client-Verbindung noch offen ist:

```js
if (client.readyState === WebSocket.OPEN) {
  // do something
}
```

Jede WebSocket-Verbindung hat eine `readyState`-Eigenschaft, die ihren aktuellen Zustand beschreibt.

`WebSocket.OPEN` bedeutet, dass die Verbindung aktiv ist und bereit, Daten zu senden und zu empfangen.

Indem der Server dies prüft, vermeidet er es, Nachrichten an Clients zu senden, die bereits getrennt sind oder deren Verbindungen noch aufgebaut werden:

```js
server.on("connection", (socket) => {
  socket.on("message", (message) => {
    server.clients.forEach((client) => {
      if (client.readyState === WebSocket.OPEN) {
        client.send(message.toString());
      }
    });
  });
});
```

In diesem Beispiel erhält jeder verbundene Client die eingehende Nachricht. Das nennt man Broadcasting.

Viele Echtzeit-Anwendungen verwenden Broadcasting, um Daten wie Chatnachrichten, Live-Benachrichtigungen, Sport-Updates und Multiplayer-Spielereignisse zu übermitteln.

Mit der `ws`-Bibliothek ist es unkompliziert, WebSocket-Server in Node.js zu erstellen.

Alles, was Sie tun müssen, ist, einen Server zu erstellen, auf Ereignisse zu hören, Nachrichten auszutauschen und Updates an verbundene Clients zu senden.

Diese Bausteine bilden die Grundlage vieler Echtzeit-Anwendungen.
