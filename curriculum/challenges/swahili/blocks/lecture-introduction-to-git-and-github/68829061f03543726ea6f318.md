# --description--

Tumezungumza kidogo kuhusu Git na GitHub, sasa tunaweza kuanza kuyaunganisha yote pamoja. Ili kufanya hivyo, unahitaji hifadhi. Lakini hifadhi ni nini? Unaweza kufikiria hifadhi kama kontena la mradi - ikiwa unafanya kazi kwenye programu, ungehifadhi mafaili ya programu hiyo pamoja katika hifadhi.

Hifadhi zinaweza kuwa za eneo la kompyuta yako, au za mbali kwenye huduma kama GitHub. Kuunda hifadhi, unaweza kuanza na eneo la kompyuta au hifadhi ya mbali - matokeo yatakuwa sawa, hivyo chagua njia inayofaa zaidi kwa mtiririko wako wa kazi. Kwa mtiririko wa kazi ambapo unaanza na hifadhi ya mbali, unaweza kuunda moja moja kupitia tovuti ya GitHub kwa kufuata maelekezo haya:

Tembelea <a href="https://github.com" target="_blank">tovuti ya GitHub</a>, hakikisha umeingia kwenye akaunti yako, kisha bofya ikoni ya alama ya kuongeza juu kulia kwenye upau wa urambazaji. Utaona chaguzi nyingi, lakini ile unayohitaji ni "New Repository". Chagua hiyo kufungua kiolesura cha kuunda hifadhi mpya.

Chaguo la kwanza utakaloona hapo ni uwezo wa kuchagua kiolezo. Huenda bado huna kiolezo cha aina hii kilichowekwa, lakini kiolezo ni aina maalum ya hifadhi unayoweza kutumia kama msingi wa hifadhi zako mpya. Hizi ni msaada kwa kujumuisha nyaraka au violezo vya matatizo katika miradi yako yote.

Chaguo lako linalofuata ni kuchagua mmiliki, unaweza kuchagua akaunti yako ya mtumiaji au mashirika yoyote unayomiliki au unayo upatikanaji nayo. Kisha unaweza kumtaja jina la hifadhi - kwa ujumla, utataka kutumia jina linaloelezea mradi unaojenga kwa ufupi. Kama mbinu bora kwa ujumla, unapaswa kuepuka nafasi katika jina la hifadhi yako. GitHub itazibadilisha moja kwa moja nafasi hizo kuwa alama za msalaba (-).

Ifuatayo, unaweza hiari kuongeza maelezo ambayo yataonekana kwenye ukurasa wa nyumbani wa hifadhi.

Pia unaweza kuchagua kama hifadhi itakuwa ya umma au ya faragha. Hifadhi ya umma inaweza kutazamwa na kupakuliwa na mtu yeyote. Hifadhi ya faragha inaweza kufikiwa na wewe tu, na mtu yeyote uye mpe upatikanaji waziwazi.

Mwishowe, kama hutachagua kiolezo, utaona chaguzi za kuzalisha mafaili katika hifadhi moja kwa moja. Faili la README litaonekana kwenye ukurasa wa nyumbani wa hifadhi, na ni njia nzuri ya kutoa nyaraka na maelezo ya mradi wako. Faili la `.gitignore` linakuwezesha kubainisha mafaili ambayo Git itaya "puuza" - hii ni nzuri kwa vitu kama utegemezi wa vifurushi, mafaili ya mazingira, na vitu vingine ambavyo hutaki kuhifadhiwa. GitHub itatoa violezo kwa faili hii, vinavyotenganisha vitu vya kawaida kwa lugha au mazingira yaliyotajwa. Na leseni ni hati ya kisheria inayobainisha nani anaweza kutumia programu yako, na wanachoweza kufanya nayo.

Unapokuwa na mipangilio yako yote imewekwa kama unavyotaka, unaweza kubofya kitufe cha "Create Repository" kumaliza mchakato. Utaelekezwa moja kwa moja kwenye ukurasa wa kutua wa hifadhi yako mpya!

Kutoka hapa, unaweza kubofya kitufe cha "Code" kupata chaguzi za kunakili hifadhi yako kwenye kompyuta yako.

Unaweza kupuuza tabo za "Codespaces" na "Copilot" kwa funzo hili. Badala yake, tutazingatia chaguzi tatu katika tabo ya "Local". Mbili za kwanza, HTTPS na SSH, hutumika kwa njia ile ile. Unatumia amri ya `git clone` pamoja na URL iliyotolewa kwenye kisanduku kunakili hifadhi ya mbali kwenye kompyuta yako. Tofauti kati ya hizi mbili ni njia za kuunganishwa na uthibitisho wa utambulisho.

Kwa kunakili hifadhi ya umma, hautahitaji kuthibitisha utambulisho. Lakini kunakili hifadhi ya faragha, au kusukuma mabadiliko kwenye hifadhi za umma au faragha, utahitaji kuthibitishwa. URL ya mbali ya HTTPS hutumika kuthibitisha kupitia jina lako la mtumiaji la GitHub na nenosiri. Hata hivyo, kutokana na masuala ya usalama hii haitegemewe tena, na badala yake unahitajika kutumia jina la mtumiaji na tokeni ya upatikanaji. Hata hivyo, hii ni njia isiyo salama kama SSH, ambayo hutumia funguo zako binafsi na za umma. Hii ina faida ya ziada ya kukuingia moja kwa moja, badala ya kutoa tokeni kila wakati. Kutakuwa na funzo tofauti linaloelezea kwa kina SSH na jinsi inavyofanya kazi lakini pia unaweza kusoma nyaraka za GitHub kuhusu jinsi ya kusanidi SSH.

Kunakili hifadhi, unaweza kuendesha hii kwenye terminal:

```sh
git clone git@github.com:your-username/your-repository-name
```

Itaunda folda kwenye kompyuta yako yenye jina la hifadhi yako na itajumuisha mafaili yote kutoka hifadhi yako ndani yake. Kutoka hapo unaweza `cd` (badilisha saraka) kwenda kwenye hifadhi yako iliyonakiliwa kwa:

```sh
cd name-of-cloned-repo
```

Zaidi ya hayo, nakala ya eneo la hifadhi yako itakuwa na remote inayoelekeza kwenye nakala ya GitHub. Kuangalia remotes zako unaweza kuendesha amri ifuatayo:

```sh
git remote -v
```

Chaguo jingine la kunakili hifadhi yako ni kutumia GitHub CLI. Zana hii hutumika kufanya kazi maalum za GitHub bila kuondoka kwenye mstari wa amri. Kama huna imewekwa, unaweza kupata maelekezo ya jinsi ya kuisakinisha kutoka kwenye nyaraka za GitHub - lakini inapaswa kuwa inapatikana katika meneja wa vifurushi wa mfumo wako.

Kunakili hifadhi kwa kutumia zana ya CLI, ningetumia:

```sh
gh repo clone your-username/your-repository-name
```

Matokeo ya amri hizi ni sawa: utakuwa na nakala ya hifadhi yako eneo la kompyuta yako.

Pia unaweza kutumia GitHub CLI kuunda hifadhi:

```sh
gh repo create
```

Kama hutapita hoja yoyote, CLI itakupa mchawi wa kuingiliana ili kusanidi hifadhi yako. Mchawi huyu atakupa chaguzi zote kama kiolesura cha wavuti. Kwa zana hii, pia utapata chaguo la kunakili hifadhi mpya mara moja, hivyo hutahitaji kupitia hatua za mkono zilizotajwa awali.

Sasa una uelewa wa msingi wa jinsi hifadhi zinavyofanya kazi, na jinsi unavyoweza kuunda moja kuanza kuweka toleo la kazi yako ipasavyo. Katika mafundisho yajayo, utajifunza jinsi ya kusukuma hifadhi ya eneo la kompyuta iliyopo kwenda GitHub na jinsi ya kusukuma mabadiliko kwenye hifadhi za mbali.
