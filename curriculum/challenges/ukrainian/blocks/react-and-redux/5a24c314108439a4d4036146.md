# --description--

Функція `mapDispatchToProps()` використовується для надання конкретних творців дій вашим React-компонентам, щоб вони могли виконувати dispatch дій у Redux store. Вона схожа за структурою на функцію `mapStateToProps()`, яку ви писали в останньому завданні. Вона повертає об’єкт, який спроєктовує dispatch дії на імена властивостей, які стають `props` компонента. Однак, замість того, щоб повертати частину `state`, кожна властивість повертає функцію, яка виконує виклик `dispatch` з творцем дії та будь-якими відповідними даними дії. Ви маєте доступ до цього `dispatch`, бо він передається у `mapDispatchToProps()` як параметр під час визначення функції, так само як ви передавали `state` у `mapStateToProps()`. Під капотом React Redux використовує Redux `store.dispatch()` для виконання цих dispatch з `mapDispatchToProps()`. Це схоже на те, як він використовує `store.subscribe()` для компонентів, які спроєктовані на `state`.

Наприклад, у вас є творець дії `loginUser()`, який приймає `username` як корисне навантаження дії. Об’єкт, що повертається з `mapDispatchToProps()` для цього творця дії, виглядав би приблизно так:

```jsx
{
  submitLoginUser: function(username) {
    dispatch(loginUser(username));
  }
}
```

# --instructions--

Редактор коду надає творця дії під назвою `addMessage()`. Напишіть функцію `mapDispatchToProps()`, яка приймає `dispatch` як аргумент, а потім повертає об’єкт. Об’єкт повинен мати властивість `submitNewMessage`, встановлену в функцію dispatch, яка приймає параметр для нового повідомлення, щоб додати його під час виконання dispatch `addMessage()`.

      testAction = fn;
