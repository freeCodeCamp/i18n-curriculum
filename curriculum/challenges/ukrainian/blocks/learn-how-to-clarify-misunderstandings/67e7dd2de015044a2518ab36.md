# --explanation--

`Think of it as` використовується, щоб допомогти комусь зрозуміти концепцію, порівнюючи її з тим, що він уже знає. Це заохочує уявити концепцію так, щоб її було легше зрозуміти. Наприклад:

`Think of a function as a recipe.` — це допомагає уявити функцію як множину інструкцій, подібно до того, як рецепт надає кроки для приготування страви.

`Wrapping code` означає обгортання або упаковку коду таким чином, щоб контролювати доступ до нього. Це `limit` доступ до коду. Наприклад:

`We're wrapping the database access code inside a function to hide the complexity from the rest of the program.` — це означає, що взаємодії з базою даних приховані ззовні, а функція надає спрощений інтерфейс для роботи з ними.
