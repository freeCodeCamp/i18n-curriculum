# --description--

In Express sind Route-Parameter dynamische Werte, die in den URL-Pfad eingebettet sind. Sie ermöglichen es Ihrer Anwendung, Teile der URL zu erfassen, die sich bei jeder Anfrage ändern können. Das ist nützlich, um dynamische Seiten zu erstellen, wie zum Beispiel die Anzeige eines bestimmten Benutzers, Blogbeitrags oder Produkts basierend auf der URL.

Hier ist ein Beispiel:

```javascript
// Route with a parameter in the path
app.get("/post/:postId", (req, res) => {
  const postId = req.params.postId;
  res.send(`Viewing post with ID: ${postId}`);
});
```

Hier hört die Route auf GET-Anfragen an `/post/:postId`.

* `:postId` ist ein Route-Parameter, ein Platzhalter in der URL, der durch einen tatsächlichen Wert ersetzt wird. Zum Beispiel entspricht eine Anfrage an `/post/42` dieser Route, und `42` wird zum Wert von `postId`.
    
* Sie können auf Route-Parameter mit `req.params` zugreifen. Das `req.params`-Objekt enthält alle Route-Parameter, und in diesem Fall enthält `req.params.postId` den Wert aus der URL, `42`.
    

Schließlich sendet `res.send()` eine Antwort zurück an den Client. Wenn also die URL `/post/42` ist, sieht der Client `Viewing post with ID: 42`.

Wie Sie sehen, machen Route-Parameter Ihre Routen flexibel und dynamisch, ohne dass Sie für jeden möglichen Wert eine separate Route benötigen.

Jetzt, da Sie verstanden haben, wie Route-Parameter funktionieren, kehren wir zu einem Beispiel aus der letzten Lektion zurück und sehen, wie Sie dynamische Routen für verschiedene Arten von Menüs verwenden können:

```javascript
app.get("/menu/:category", (req, res) => {
  const category = req.params.category;
  res.send(`You are viewing the ${category} menu!`);
});
```

Und hier eine Erklärung, wie das funktioniert:

1. Ein Benutzer besucht [`http://localhost:3000/menu/drinks`](http://localhost:3000/menu/drinks).
    
2. Der Server ordnet diese Anfrage der Route `/menu/:category` zu und erfasst den Parameter `category` als `drinks`.
    
3. Der Server sendet diese Antwort zurück an den Browser: `You are viewing the drinks menu!`
    

Ähnlich, wenn der Benutzer [`http://localhost:3000/menu/food`](http://localhost:3000/menu/food) besucht, antwortet der Server mit folgendem: `You are viewing the food menu!`

Route-Parameter ermöglichen es Ihnen, dynamische Inhalte basierend auf der URL anzuzeigen. Anstatt für jede Kategorie eine separate Route zu erstellen, können Sie eine einzige Route verwenden, um mehrere Kategorien wie Getränke, Essen, Desserts und mehr zu bedienen, indem Sie einfach die URL ändern.

## --text--

Welche Rolle hat `:postId` in der Route `/post/:postId`?

## --answers--

Es ist ein statischer Teil der URL.

### --feedback--

`:` zeigt einen Parameter an, keinen festen Abschnitt.

---

Es erfasst einen dynamischen Wert aus der URL.

---

Es ist ein Query-Parameter.

### --feedback--

Query-Parameter erscheinen nach `?`, nicht im Pfad.

---

Es definiert eine benutzerdefinierte Kopfzeile.

### --feedback--

Route-Parameter befinden sich im URL-Pfad, nicht in Kopfzeilen.

## --text--

Wie können Sie Route-Parameter in Ihrer Anwendung verwenden?

## --answers--

Um statische Werte in der URL zu speichern.

### --feedback--

Statische Werte profitieren nicht von Parameter-Platzhaltern.

---

Um Teile der URL dynamisch zu erfassen und zu verarbeiten.

---

Um Benutzereingaben zu validieren.

### --feedback--

Validierung kann nach der Erfassung des Werts erfolgen, aber der Parameter selbst erfasst Daten.

---

Um Kopfzeilen für Anfragen zu definieren.

### --feedback--

Route-Parameter gehören zum Pfad und definieren keine Kopfzeilen.
