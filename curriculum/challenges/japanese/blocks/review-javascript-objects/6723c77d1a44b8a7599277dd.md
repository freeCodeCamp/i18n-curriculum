# --interactive--

## オブジェクトの基本

- **定義**: オブジェクトはプロパティで構成されるデータ構造です。プロパティはキーと値から成ります。オブジェクトのデータにアクセスするには、ドット記法かブラケット記法を使います。

:::interactive_editor

```js
const person = {
  name: "Alice",
  age: 30,
  city: "New York"
};

console.log(person.name);  // Alice
console.log(person["name"]); // Alice
```

:::

既存のオブジェクトのプロパティを設定するには、ドット記法かブラケット記法と代入演算子を使います。

:::interactive_editor

```js
const person = {
  name: "Alice",
  age: 30
};

person.job = "Engineer"
person["hobby"] = "Knitting"
console.log(person);  // {name: 'Alice', age: 30, job: 'Engineer', hobby: 'Knitting'}
```

:::

## オブジェクトからプロパティを削除する

- **`delete` 演算子**: この演算子はオブジェクトからプロパティを削除するために使います。

:::interactive_editor

```js
const person = {
  name: "Alice",
  age: 30,
  job: "Engineer"
};

delete person.job;

console.log(person.job); // undefined
```

:::

## オブジェクトにプロパティがあるか確認する

- **`hasOwnProperty()` メソッド**: このメソッドはオブジェクトが指定したプロパティを自身のプロパティとして持っているかどうかをブール値で返します。

:::interactive_editor

```js
const person = {
  name: "Alice",
  age: 30
};

console.log(person.hasOwnProperty("name")); // true
console.log(person.hasOwnProperty("job")); // false
```

:::

- **`Object.hasOwn()` メソッド**: これはオブジェクトが自身のプロパティを持っているか（継承されたものではなく）を確認する、現代的で推奨される方法です。構文は `Object.hasOwn(object, propertyName)` です。プロパティがオブジェクトに存在すれば `true` を返し、存在しなければ `false` を返します。プロパティの値に関係なく判定されるため、`hasOwnProperty()` より安全で、値が `0`、`false`、`null`、`undefined` の場合でも `if (obj.prop)` より信頼性があります。

:::interactive_editor

```js
const person = {
  name: "Alice",
  age: 30
};

console.log(Object.hasOwn(person, "name")); // true
console.log(Object.hasOwn(person, "job")); // false
```

:::

以下は、なぜ `Object.hasOwn()` が値を直接チェックするより信頼できるかを示す詳しい例です。

:::interactive_editor

```js
const settings = {
  darkMode: false,
  fontSize: 0,
  language: null
};

// Object.hasOwn() correctly sees these properties exist
console.log(Object.hasOwn(settings, "darkMode")); // true (value is false, but exists)
console.log(Object.hasOwn(settings, "fontSize")); // true (value is 0, but exists)
console.log(Object.hasOwn(settings, "theme"));    // false (property was never added)

// Using if() directly is unsafe for falsy values!
if (settings.darkMode) {
  console.log("Dark mode on"); // Does NOT print — misleading!
}

// Object.hasOwn() is the safe way
if (Object.hasOwn(settings, "darkMode")) {
  console.log("darkMode exists, value is:", settings.darkMode); // darkMode exists, value is: false
}
```

:::

- **`in` 演算子**: この演算子はオブジェクトにプロパティが存在すれば `true` を返します。

:::interactive_editor

```js
const person = {
  name: "Bob",
  age: 25
};

console.log("name" in person);  // true
```

:::

## ネストされたオブジェクトのプロパティにアクセスする

- **データアクセス**: ネストされたオブジェクトのプロパティにアクセスするには、単純なオブジェクトのプロパティアクセスと同様にドット記法かブラケット記法を使います。ただし、ネスト構造を掘り下げるためにこれらのアクセスをチェーンする必要があります。

:::interactive_editor

```js
const person = {
  name: "Alice",
  age: 30,
  contact: {
    email: "alice@example.com",
    phone: {
      home: "123-456-7890",
      work: "098-765-4321"
    }
  }
};

console.log(person.contact.phone.work); // "098-765-4321"
```

:::

## プリミティブ型と非プリミティブ型のデータ型

- **プリミティブ型データ**: これらのデータ型には数値、文字列、ブール値、`null`、`undefined`、シンボルが含まれます。これらは単一の値を表し、オブジェクトではないため「プリミティブ」と呼ばれます。プリミティブ値は不変で、一度作成されると値を変更できません。
- **非プリミティブ型データ**: JavaScript ではこれらはオブジェクトで、通常のオブジェクト、配列、関数が含まれます。プリミティブとは異なり、非プリミティブ型は複数の値をプロパティや要素として保持できます。

## オブジェクトメソッド

- **定義**: オブジェクトメソッドはオブジェクトに関連付けられた関数です。オブジェクトのプロパティとして定義され、オブジェクトのデータにアクセスし操作できます。メソッド内の `this` キーワードはオブジェクト自身を指し、そのプロパティにアクセス可能にします。

:::interactive_editor

```js
const person = {
  name: "Bob",
  age: 30,
  sayHello: function() {
    return "Hello, my name is " + this.name;
  }
};

console.log(person.sayHello()); // "Hello, my name is Bob"
```

:::

## オブジェクトコンストラクタ

- **定義**: JavaScript ではコンストラクタはオブジェクトを作成・初期化する特別な関数です。`new` キーワードで呼び出され、新しく作成されたオブジェクトのプロパティやメソッドを初期化できます。`Object()` コンストラクタは新しい空のオブジェクトを作成します。

```js
new Object()
```

## オプショナルチェーン演算子（`?.`）の使い方

- **定義**: この演算子は、存在するかどうかを気にせずにオブジェクトのプロパティに安全にアクセスしたりメソッドを呼び出したりできます。

:::interactive_editor

```ts
const user = {
  name: "John",
  profile: {
    email: "john@example.com",
    address: {
      street: "123 Main St",
      city: "Somewhere"
    }
  }
};

console.log(user.profile?.address?.street); // "123 Main St"
console.log(user.profile?.phone?.number);   // undefined
```

:::

## オブジェクトの分割代入

- **定義**: オブジェクトの分割代入は、オブジェクトから値を抽出して変数に簡潔かつ読みやすく代入できる機能です。

:::interactive_editor

```js
const person = { name: "Alice", age: 30, city: "New York" };

const { name, age } = person;

console.log(name); // Alice
console.log(age);  // 30
```

:::

## JSONの扱い方

- **定義**: JSON は JavaScript Object Notation の略で、サーバーとウェブアプリケーション間でデータを交換するためによく使われる軽量のテキストベースのデータ形式です。

```js
{
  "name": "Alice",
  "age": 30,
  "isStudent": false,
  "list of courses": ["Mathematics", "Physics", "Computer Science"]
}
```

- **`JSON.stringify()`**: このメソッドは JavaScript オブジェクトを JSON 文字列に変換します。データをシステム間で簡単に共有・転送したい場合に便利です。

:::interactive_editor

```js
const user = {
  name: "John",
  age: 30,
  isAdmin: true
};

const jsonString = JSON.stringify(user);
console.log(jsonString); // '{"name":"John","age":30,"isAdmin":true}'
```

:::

- **`JSON.parse()`**: このメソッドは JSON 文字列を JavaScript オブジェクトに戻します。ウェブサーバーや localStorage から JSON データを取得し、アプリケーション内で操作したい場合に使います。

:::interactive_editor

```js
const jsonString = '{"name":"John","age":30,"isAdmin":true}';
const userObject = JSON.parse(jsonString);

// result: { name: 'John', age: 30, isAdmin: true }
console.log(userObject);
```

:::
