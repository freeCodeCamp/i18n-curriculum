Überprüfen Sie die folgenden Konzepte, um sich auf die bevorstehende Prüfung vorzubereiten.

## CSS-Grundlagen

- **Was ist CSS?**: Cascading Style Sheets (CSS) ist eine Stylesheet-Sprache, die verwendet wird, um HTML-Elementen Stile zuzuweisen. CSS wird für Farben, Hintergrundbilder, Layouts und mehr verwendet.
- **Grundaufbau einer CSS-Regel**: Eine CSS-Regel besteht aus zwei Hauptteilen: einem Selektor und einem Deklarationsblock. Ein Selektor ist ein Muster, das in CSS verwendet wird, um bestimmte HTML-Elemente für die Gestaltung zu identifizieren und anzusprechen. Ein Deklarationsblock wendet eine Reihe von Stilen für einen bestimmten Selektor oder mehrere Selektoren an.
- **`meta name="viewport"`-Element**: Dieses `meta`-Element gibt dem Browser Anweisungen, wie die Abmessungen und Skalierung der Seite auf verschiedenen Geräten, insbesondere auf Mobiltelefonen und Tablets, gesteuert werden sollen.
- **Standard-Browser-Stile**: Jedem HTML-Element werden standardmäßig Browser-Stile zugewiesen. Dies umfasst in der Regel Elemente wie voreingestellte Außenabstände und Innenabstände.

## Inline-, interne und externe CSS

- **Inline-CSS**: Diese Stile werden direkt innerhalb eines HTML-Elements mit dem `style` Attribut geschrieben. Meistens verwenden Sie kein Inline-CSS, um die Trennung der Verantwortlichkeiten zu wahren.
- **Internes CSS**: Diese Stile werden innerhalb der `<style>` Tags im `head` Abschnitt eines HTML-Dokuments geschrieben. Dies kann nützlich sein, um kurze Codebeispiele zu erstellen, aber normalerweise verwenden Sie kein internes CSS.
- **Externes CSS**: Diese Stile werden in einer separaten CSS-Datei geschrieben und mit dem `link` Element im `head` Abschnitt mit dem HTML-Dokument verknüpft. Für die meisten Projekte verwenden Sie eine externe CSS-Datei anstelle von internem oder Inline-CSS.

## Arbeiten mit den `width` und `height` Eigenschaften

- **`width` Eigenschaft**: Diese Eigenschaft legt die Breite eines Elements fest. Wenn Sie keine Breite angeben, ist der Standardwert `auto`. Dadurch bestimmt der Browser die Breite des Elements basierend auf seinem Inhalt, dem übergeordneten Element und dem Anzeige-Typ.
- **`min-width` Eigenschaft**: Diese Eigenschaft legt die minimale Breite eines Elements fest.
- **`max-width` Eigenschaft**: Diese Eigenschaft legt die maximale Breite eines Elements fest.
- **`height` Eigenschaft**: Diese Eigenschaft legt die Höhe eines Elements fest. Ebenso ist die Höhe standardmäßig `auto`, was bedeutet, dass sie sich an den Inhalt anpasst.
- **`min-height` Eigenschaft**: Diese Eigenschaft legt die minimale Höhe eines Elements fest.
- **`max-height` Eigenschaft**: Diese Eigenschaft legt die maximale Höhe eines Elements fest.

## Verschiedene Arten von CSS-Kombinatoren

- **Nachfahre-Kombinator (` `)&nbsp;\*ein einzelnes Leerzeichen zwischen Selektoren**: Dieser Kombinator wird verwendet, um Elemente anzusprechen, die Nachfahren eines bestimmten übergeordneten Elements sind.
- **Kind-Kombinator (`>`)**: Dieser Kombinator wird verwendet, um Elemente auszuwählen, die direkte Kinder eines bestimmten übergeordneten Elements sind.
- **Nächstes Geschwister-Kombinator (`+`)**: Dieser Kombinator wählt ein Element aus, das unmittelbar auf ein bestimmtes Geschwisterelement folgt.

- **Nachfolgendes Geschwister-Kombinator (`~`)**: Dieser Kombinator wählt alle Geschwister eines bestimmten Elements aus, die nach ihm kommen.

## Inline-, Block- und Inline-Block-Level-Elemente

- **Inline-Level-Elemente**: Inline-Elemente nehmen nur so viel Breite ein, wie sie benötigen, und beginnen nicht auf einer neuen Zeile. Diese Elemente fließen innerhalb des Inhalts, sodass Text und andere Inline-Elemente neben ihnen erscheinen können. Häufige Inline-Elemente sind `span`, `anchor` und `img` Elemente.
- **Block-Level-Elemente**: Block-Level-Elemente nehmen standardmäßig die gesamte verfügbare Breite ein und erstrecken sich über die Breite ihres Containers. Einige häufige Block-Level-Elemente sind `div`, `p` und `section` Elemente.
- **Inline-Block-Level-Elemente**: Sie können ein Element mit der `inline-block` Eigenschaft auf `display` setzen. Diese Elemente verhalten sich wie Inline-Elemente, können aber wie Block-Level-Elemente einen `width` und `height` haben.

## Außen- und Innenabstand

- **`margin` Eigenschaft**: Diese Eigenschaft wird verwendet, um Abstand außerhalb des Elements anzuwenden, zwischen dem Rahmen des Elements und den umliegenden Elementen.
- **`margin` Kurzschreibweise**: Sie können 1–4 Werte angeben, um die Außenabstände festzulegen. Ein Wert gilt für alle vier Seiten; zwei Werte setzen `top` und `bottom`, dann `right` und `left`; drei Werte setzen `top`, horizontal (`right` und `left`), dann `bottom`; vier Werte setzen `top`, `right`, `bottom`, `left`.
- **`padding` Eigenschaft**: Diese Eigenschaft wird verwendet, um Abstand innerhalb des Elements anzuwenden, zwischen dem Inhalt und seinem Rahmen.
- **`padding` Kurzschreibweise**: Sie können 1–4 Werte angeben, um die Innenabstände festzulegen. Ein Wert gilt für alle vier Seiten; zwei Werte setzen `top` und `bottom`, dann `right` und `left`; drei Werte setzen `top`, horizontal (`right` und `left`), dann `bottom`; vier Werte setzen `top`, `right`, `bottom`, `left`.

## CSS-Spezifität

- **Spezifität von Inline-CSS**: Inline-CSS hat die höchste Spezifität, da es direkt auf das Element angewendet wird. Es überschreibt internes oder externes CSS. Der Spezifitätswert für Inline-Stile ist (1, 0, 0, 0).
- **Spezifität von internem CSS**: Internes CSS wird innerhalb eines `style` Elements im `head` Abschnitt des HTML-Dokuments definiert. Es hat eine geringere Spezifität als Inline-Stile. Internes und externes CSS teilen sich denselben Spezifitätsgrad, der durch ihre Selektoren bestimmt wird, nicht durch ihren Ort.
- **Spezifität von externem CSS**: Externes CSS wird über ein `link` Element im `head` Abschnitt verknüpft und in separaten `.css` Dateien geschrieben. Wie beim internen CSS wird die Spezifität durch die verwendeten Selektoren bestimmt. Wenn zwei Regeln dieselbe Spezifität haben, entscheidet die Reihenfolge im Quelltext – die Regel, die später im Dokument erscheint, hat Vorrang. Externes CSS bietet die beste Wartbarkeit für größere Projekte.
- **Universalselektor (`*`)**: Dies ist ein spezieller CSS-Selektor, der jedes Element im Dokument auswählt. Er wird oft verwendet, um einen Stil auf alle Elemente der Seite anzuwenden, was nützlich sein kann, um Stile in verschiedenen Browsern zurückzusetzen oder zu normalisieren. Der Universalselektor hat den niedrigsten Spezifitätswert aller Selektoren. Er trägt 0 zu allen Teilen des Spezifitätswerts bei (0, 0, 0, 0).
- **Typselektoren**: Diese Selektoren sprechen Elemente basierend auf ihrem Tag-Namen an. Typselektoren haben eine relativ niedrige Spezifität im Vergleich zu anderen Selektoren. Der Spezifitätswert für einen Typselektor ist (0, 0, 0, 1).
- **Klassenselektoren**: Diese Selektoren werden durch einen Punkt (`.`) gefolgt vom Klassennamen definiert. Der Spezifitätswert für einen Klassenselektor ist (0, 0, 1, 0). Das bedeutet, dass Klassenselektoren Typselektoren überschreiben können, aber von ID-Selektoren und Inline-Stilen überschrieben werden können.
- **ID-Selektoren**: ID-Selektoren werden durch ein Rautezeichen (`#`) gefolgt vom ID-Namen definiert. ID-Selektoren haben eine sehr hohe Spezifität, höher als Typ- und Klassenselektoren, aber niedriger als Inline-Stile. Der Spezifitätswert für einen ID-Selektor ist (0, 1, 0, 0).
- **`!important` Schlüsselwort**: Wird verwendet, um einer Stilregel die höchste Priorität zu geben, sodass sie alle anderen Deklarationen für eine Eigenschaft überschreibt. Wenn es verwendet wird, zwingt es den Browser, den angegebenen Stil anzuwenden, unabhängig von der Spezifität anderer Selektoren. Sie sollten `!important` mit Vorsicht verwenden, da es Ihr CSS schwerer wartbar und debugbar machen kann.
- **Kaskadenalgorithmus**: Ein Algorithmus, der entscheidet, welche CSS-Regeln angewendet werden, wenn mehrere Stile dasselbe Element ansprechen. Er stellt sicher, dass die passendsten Stile verwendet werden, basierend auf einem Satz klar definierter Regeln.
- **CSS-Vererbung**: Der Prozess, bei dem Stile von übergeordneten Elementen an ihre Kinder weitergegeben werden. Vererbung ermöglicht es, Stile auf einer höheren Ebene im Dokumentbaum zu definieren und sie auf mehrere Elemente anzuwenden, ohne sie explizit für jedes Element anzugeben.

## Design-Terminologie

- **Layout**: So werden visuelle Elemente auf einer Seite oder einem Bildschirm angeordnet, um eine Botschaft zu vermitteln. Diese Elemente können Text, Bilder und Leerraum umfassen.
- **Ausrichtung**: So werden die Elemente zueinander platziert. Die korrekte Verwendung von Ausrichtung hilft, das Design sauber und organisiert wirken zu lassen.
- **Komposition**: Das Anordnen von Elementen, um ein harmonisches Design zu schaffen. Es bestimmt, wie Elemente wie Bilder, Text und Formen zueinander in Beziehung stehen und künstlerisch zum Design beitragen.
- **Balance**: So wird das visuelle Gewicht innerhalb einer Komposition verteilt. Designer streben ein Gleichgewicht durch symmetrische oder asymmetrische Anordnungen an.
- **Skalierung**: Bezieht sich auf den Vergleich der Dimensionen oder Größe eines Elements mit einem anderen.
- **Hierarchie**: Legt die Reihenfolge der Wichtigkeit der Elemente in einem Design fest. Es geht darum sicherzustellen, dass die wichtigsten Informationen zuerst wahrgenommen werden.
- **Kontrast**: Der Prozess, klare Unterschiede zwischen den Elementen zu schaffen. Dies kann durch Variationen in Farbe, Größe, Form, Textur oder anderen visuellen Merkmalen erfolgen. Starker Kontrast verbessert auch die Lesbarkeit.
- **Weißraum (Negativraum)**: Der leere Raum in einem Design. Es ist der Bereich um die Elemente herum.
- **UI (Benutzeroberfläche)**: UI umfasst die visuellen und interaktiven Elemente, die Benutzer auf ihren Bildschirmen sehen können, wie Symbole, Bilder, Text, Menüs, Links und Schaltflächen.
- **UX (Benutzererfahrung)**: UX beschreibt, wie Benutzer sich bei der Nutzung eines Produkts oder einer Dienstleistung fühlen. Eine Anwendung mit gut gestaltetem Benutzererlebnis ist intuitiv, einfach zu bedienen, effizient, barrierefrei und angenehm.
- **Design-Brief**: Ein Dokument, das die Ziele, Anforderungen und Vorgaben eines Projekts umreißt. Es ist eine Roadmap, die den Designprozess leitet und sicherstellt, dass das Endprodukt die Bedürfnisse des Kunden erfüllt.
- **Vektorbasierte Gestaltung**: Dabei wird digitale Kunst mit mathematischen Formeln erstellt, um Linien, Formen und Farben zu definieren.
- **Prototyping**: Der Prozess, ein interaktives Modell eines Produkts oder einer Benutzeroberfläche zu erstellen.

## Grundlagen des UI-Designs

- **Guter Kontrast für Hintergrund- und Vordergrundfarben**: Es ist wichtig sicherzustellen, dass Hintergrund- und Vordergrundfarben einen guten Kontrast haben, damit der Text lesbar ist. Die Web Content Accessibility Guidelines (WCAG) empfehlen ein Mindest-Kontrastverhältnis von 4,5:1 für normalen Text und 3:1 für großen Text.
- **Gute visuelle Hierarchie im Design**: Eine starke visuelle Hierarchie bietet einen klaren Pfad für das Auge, sodass die Informationen in der beabsichtigten Reihenfolge aufgenommen werden.
- **Responsive Bilder**: Responsive Bilder passen sich an die Größe des Bildschirms an, auf dem sie betrachtet werden. Das ist wichtig, damit Ihre Bilder auf allen Geräten gut aussehen, von Desktops bis zu Mobiltelefonen.
- **Progressive Enhancement**: Ein Designansatz, der sicherstellt, dass alle Benutzer, unabhängig von Browser oder Gerät, auf die wesentlichen Inhalte und Funktionen einer Anwendung zugreifen können.
- **Nutzerzentriertes Design**: Ein Ansatz, der den Endbenutzer priorisiert, von seinen Bedürfnissen bis zu seinen Vorlieben und Einschränkungen. Ziel ist es, eine Webseite zu gestalten, die intuitiv, effizient nutzbar und angenehm für die Nutzer ist.
- **Nutzerforschung**: Die systematische Untersuchung der Personen, die Ihr Produkt verwenden. Ziel ist es, Nutzerbedürfnisse, Verhaltensweisen und Schmerzpunkte zu messen.
- **Nutzertests**: Die Praxis, Daten von Nutzern zu erfassen, während sie mit Ihrer Anwendung interagieren.
- **A/B-Tests**: Der Prozess, eine neue Funktion an eine zufällig ausgewählte Teilmenge Ihrer Nutzerbasis auszuliefern. Anschließend können Sie Analysedaten nutzen, um zu bestimmen, ob die Funktion vorteilhaft ist.
- **Nutzeranforderungen**: Die Geschichten oder Vorgaben, denen Ihre Anwendung folgen muss. Nutzeranforderungen können durch Nutzerforschung, Branchenstandards oder Stakeholder-Eingaben definiert werden.
- **Progressive Offenlegung**: Ein Designmuster, das nur relevante Inhalte basierend auf der aktuellen Aktivität des Nutzers anzeigt und den Rest verbirgt. Dies reduziert die kognitive Belastung und macht die Nutzererfahrung intuitiver.
- **Deferred/Lazy Registration**: Ein UI-Designmuster, das es Nutzern erlaubt, Ihre Anwendung zu durchsuchen und zu nutzen, ohne sich sofort registrieren zu müssen.

## Bewährte Design-Vorgehensweisen

- **Dark Mode**: Das ist eine spezielle Funktion bei Webanwendungen, bei der Sie das standardmäßige helle Farbschema in ein dunkles Farbschema ändern können. Sie sollten in Dark Mode entsättigte Farben verwenden. Entsättigte Farben sind Farben, die weniger intensiv sind und einen niedrigeren Sättigungsgrad haben.
- **Breadcrumbs**: Das ist eine Navigationshilfe, die dem Nutzer zeigt, wo er sich in der Hierarchie der Seite befindet. Es ist am besten, Breadcrumbs oben auf der Seite zu platzieren, damit Nutzer sie leicht finden. Außerdem sollten die Breadcrumbs groß genug sein, um gut lesbar zu sein, aber nicht so groß, dass sie zu viel Platz auf der Seite einnehmen.
- **Card-Komponente**: Ihre Card-Komponente sollte einfach gestaltet sein, nicht visuell überladen oder mit zu vielen Informationen gefüllt. Für Medien sollten Sie hochwertige Bilder und Videos wählen, um das Nutzererlebnis zu verbessern.
- **Infinite Scroll**: Das ist ein Designmuster, das mehr Inhalte lädt, während der Nutzer die Seite nach unten scrollt. Sie sollten in Erwägung ziehen, eine „Mehr laden“-Schaltfläche zu verwenden, da sie dem Nutzer Kontrolle darüber gibt, wann er mehr Inhalte sehen möchte. Sie können auch eine Zurück-Schaltfläche hinzufügen, damit Nutzer zur vorherigen Seite zurückkehren können, ohne ganz nach oben scrollen zu müssen.
- **Modal-Dialog**: Das ist eine Art Pop-up, das über dem bestehenden Seiteninhalt angezeigt wird. Üblicherweise hat der Hintergrund eine abgedunkelte Farbüberlagerung, um dem Nutzer zu helfen, sich besser auf den Modal-Inhalt zu konzentrieren. Es ist außerdem immer eine gute Idee, dem Nutzer zu erlauben, durch Klicken außerhalb des Modals dieses zu schließen. Wenn Sie das HTML `dialog` Element verwenden, erhalten Sie viele der Funktionalitäts- und Barrierefreiheitsvorteile eingebaut.
- **Fortschrittsanzeige für Formularregistrierung**: Das ist eine Möglichkeit, Nutzern zu zeigen, wie weit sie in einem Prozess sind. Es kann in Formularen, Registrierungen und Einrichtungsprozessen verwendet werden. Ihr Design sollte einfach, leicht zu finden sein und es ermöglichen, zu vorherigen Schritten zurückzukehren.
- **Einkaufswagen**: Einkaufswagen sind ein Bereich, in dem Nutzer sehen können, welche Artikel sie auf einer E-Commerce-Plattform bereits ausgewählt haben. Ihre Einkaufswagen sollten immer für den Nutzer sichtbar sein, ein gängiges Symbol wie einen Wagen, eine Tasche oder einen Korb verwenden und eine klare Handlungsaufforderungsschaltfläche haben, damit Nutzer zur Kasse gehen können.

Es ermöglicht Design- und Entwicklungsteams, von überall zusammenzuarbeiten, und bietet eingebaute Funktionen wie vektorbasierte Gestaltung, automatisches Layout, ein Kommentierungs- und Feedbacksystem und mehr.

## Absolute Einheiten

- **`px` (Pixel)**: Diese absolute Einheit ist eine feste Maßeinheit in CSS, die präzise Kontrolle über Dimensionen bietet. 1px entspricht immer 1/96 Zoll.
- **`in` (Zoll)**: Diese absolute Einheit entspricht 96px.
- **`cm` (Zentimeter)**: Diese absolute Einheit entspricht 25,2/64 Zoll.
- **`mm` (Millimeter)**: Diese absolute Einheit entspricht 1/10 Zentimeter.
- **`q` (Viertelmillimeter)**: Diese absolute Einheit entspricht 1/40 Zentimeter.
- **`pc` (Picas)**: Diese absolute Einheit entspricht 1/6 Zoll.
- **`pt` (Punkte)**: Diese absolute Einheit entspricht 1/72 Zoll.

## Relative Einheiten

- **Prozentwerte**: Diese relativen Einheiten erlauben es Ihnen, Größen, Dimensionen und andere Eigenschaften als Anteil des übergeordneten Elements zu definieren. Wenn Sie beispielsweise `width: 50%;` an einem Element setzen, nimmt es die halbe Breite seines übergeordneten Containers ein.
- **`em` Einheit**: Diese Einheiten beziehen sich auf die Schriftgröße des Elements. Wenn Sie `ems` für die `font-size` Eigenschaft verwenden, ist die Textgröße relativ zur Schriftgröße des übergeordneten Elements.
- **`rem` Einheit**: Diese Einheiten beziehen sich auf die Schriftgröße des Wurzelelements, also des `html` Elements.
- **`vh` Einheit**: `vh` steht für `"viewport height"` und `1vh` entspricht 1 % der Höhe des Viewports.
- **`vw` Einheit**: `vw` steht für `"viewport width"` und `1vw` entspricht 1 % der Breite des Viewports.

## `calc` Funktion

- **`calc()` Funktion**: Mit der `calc()` Funktion können Sie Berechnungen direkt in Ihren Stylesheets durchführen, um Eigenschaftswerte dynamisch zu bestimmen. So können Sie flexible und responsive Benutzeroberflächen erstellen, indem Sie Dimensionen basierend auf der Viewport-Größe oder anderen Elementen berechnen.

## Pseudoklassen für Benutzeraktionen

- **Definition von Pseudoklassen**: Dies sind spezielle CSS-Schlüsselwörter, mit denen Sie ein Element basierend auf seinem spezifischen Zustand oder seiner Position auswählen können.
- **Pseudoklassen für Benutzeraktionen**: Diese speziellen Schlüsselwörter ermöglichen es, das Aussehen von Elementen basierend auf Benutzerinteraktionen zu ändern und verbessern so die gesamte Benutzererfahrung.
- **`:active` Pseudoklasse**: Diese Pseudoklasse ermöglicht die Auswahl des aktiven Zustands eines Elements, z. B. beim Klicken auf eine Schaltfläche.
- **`:hover` Pseudoklasse**: Diese Pseudoklasse definiert den Hover-Zustand eines Elements.
- **`:focus` Pseudoklasse**: Diese Pseudoklasse wendet Stile an, wenn ein Element den Fokus erhält, typischerweise durch Tastaturnavigation oder wenn ein Nutzer in ein Formulareingabefeld klickt.
- **`:focus-within` Pseudoklasse**: Diese Pseudoklasse wird verwendet, um Stile auf ein Element anzuwenden, wenn es selbst oder eines seiner Nachfahren den Fokus hat.

## Pseudoklassen für Eingaben

- **Pseudoklassen für Eingaben**: Diese Pseudoklassen werden verwendet, um HTML `input` Elemente basierend auf ihrem Zustand vor und nach der Benutzerinteraktion anzusprechen.
- **`:enabled` Pseudoklasse**: Diese Pseudoklasse wird verwendet, um Formularschaltflächen oder andere Elemente anzusprechen, die derzeit aktiviert sind.
- **`:disabled` Pseudoklasse**: Diese Pseudoklasse ermöglicht das Stylen eines interaktiven Elements im deaktivierten Modus.
- **`:checked` Pseudoklasse**: Diese Pseudoklasse zeigt dem Nutzer an, dass ein Element ausgewählt ist.
- **`:valid` Pseudoklasse**: Diese Pseudoklasse spricht Eingabefelder an, die die Validierungskriterien erfüllen.
- **`:invalid` Pseudoklasse**: Diese Pseudoklasse spricht Eingabefelder an, die die Validierungskriterien nicht erfüllen.
- **`:in-range` und `:out-of-range` Pseudoklassen**: Diese Pseudoklassen wenden Stile auf Elemente an, basierend darauf, ob ihre Werte innerhalb oder außerhalb bestimmter Bereichsgrenzen liegen.
- **`:required` Pseudoklasse**: Diese Pseudoklasse spricht `input` Elemente an, die das `required` Attribut besitzen. Sie signalisiert dem Nutzer, dass das Feld ausgefüllt werden muss, um das Formular abzusenden.
- **`:optional` Pseudoklasse**: Diese Pseudoklasse wendet Stile auf Eingabeelemente an, die nicht erforderlich sind und leer bleiben können.
- **`:autofill` Pseudoklasse**: Diese Pseudoklasse wendet Stile auf Eingabefelder an, die der Browser automatisch mit gespeicherten Daten füllt.

## Ortsbezogene Pseudoklassen

- **Ortsbezogene Pseudoklassen**: Diese Pseudoklassen werden verwendet, um Links und Elemente zu stylen, die innerhalb des aktuellen Dokuments angesprochen werden.
- **`:any-link` Pseudoklasse**: Diese Pseudoklasse ist eine Kombination aus den `:link` und `:visited` Pseudoklassen. Sie trifft also auf jedes Ankerelement mit einem href-Attribut zu, unabhängig davon, ob es besucht wurde oder nicht.
- **`:link` Pseudoklasse**: Diese Pseudoklasse ermöglicht es, alle nicht besuchten Links auf einer Webseite anzusprechen. Sie können sie verwenden, um Links vor dem Klicken unterschiedlich zu stylen.
- **`:local-link` Pseudoklasse**: Diese Pseudoklasse spricht Links an, die auf dasselbe Dokument verweisen. Sie ist nützlich, um interne Links von externen zu unterscheiden.
- **`:visited` Pseudoklasse**: Diese Pseudoklasse spricht einen Link an, den der Nutzer bereits besucht hat.
- **`:target` Pseudoklasse**: Diese Pseudoklasse wird verwendet, um Stile auf ein Element anzuwenden, das Ziel eines URL-Fragments ist.

## Baumstrukturbezogene Pseudoklassen

- **Baumstrukturbezogene Pseudoklassen**: Diese Pseudoklassen ermöglichen es, Elemente basierend auf ihrer Position im Dokumentbaum anzusprechen und zu stylen.
- **`:root` Pseudoklasse**: Diese Pseudoklasse ist üblicherweise das Wurzel-`html` Element. Sie hilft, die höchste Ebene im Dokument anzusprechen, um einen gemeinsamen Stil auf das gesamte Dokument anzuwenden.
- **`:empty` Pseudoklasse**: Leere Elemente, also Elemente ohne Kinder außer Leerraum, sind ebenfalls im Dokumentbaum enthalten. Deshalb gibt es die `:empty` Pseudoklasse, um leere Elemente anzusprechen.
- **`:nth-child(n)` Pseudoklasse**: Diese Pseudoklasse ermöglicht die Auswahl von Elementen basierend auf ihrer Position innerhalb eines Elternteils.
- **`:nth-last-child(n)` Pseudoklasse**: Diese Pseudoklasse ermöglicht die Auswahl von Elementen, indem vom Ende gezählt wird.
- **`:first-child` Pseudoklasse**: Diese Pseudoklasse wählt das erste Element in einem Elternteil oder im Dokument aus.
- **`:last-child` Pseudoklasse**: Diese Pseudoklasse wählt das letzte Element in einem Elternteil oder im Dokument aus.
- **`:only-child` Pseudoklasse**: Diese Pseudoklasse wählt das einzige Element in einem Elternteil oder im Dokument aus.
- **`:first-of-type` Pseudoklasse**: Diese Pseudoklasse wählt das erste Vorkommen eines bestimmten Elementtyps innerhalb seines Elternteils aus.
- **`:last-of-type` Pseudoklasse**: Diese Pseudoklasse wählt das letzte Vorkommen eines bestimmten Elementtyps innerhalb seines Elternteils aus.
- **`:nth-of-type(n)` Pseudoklasse**: Diese Pseudoklasse ermöglicht die Auswahl eines bestimmten Elements innerhalb seines Elternteils basierend auf seiner Position unter Geschwistern desselben Typs.
- **`:only-of-type` Pseudoklasse**: Diese Pseudoklasse wählt ein Element aus, wenn es das einzige seiner Art innerhalb seines Elternteils ist.


## Funktionale Pseudoklassen

- **Funktionale Pseudoklassen**: Funktionale Pseudoklassen ermöglichen die Auswahl von Elementen basierend auf komplexeren Bedingungen oder Beziehungen. Im Gegensatz zu regulären Pseudoklassen, die Elemente basierend auf einem Zustand ansprechen (z. B. `:hover`, `:focus`), akzeptieren funktionale Pseudoklassen Argumente.
- **`:is()` Pseudoklasse**: Diese Pseudoklasse nimmt eine Liste von Selektoren (z. B. `ol`, `ul`) und wählt ein Element aus, das einem der Selektoren in der Liste entspricht.
- **`:where()` Pseudoklasse**: Diese Pseudoklasse nimmt eine Liste von Selektoren (z. B. `ol`, `ul`) und wählt ein Element aus, das einem der Selektoren in der Liste entspricht. Der Unterschied zwischen `:is` und `:where` ist, dass letztere eine Spezifität von 0 hat.
- **`:has()` Pseudoklasse**: Diese Pseudoklasse wird oft als `"parent"` Selektor bezeichnet, da sie es ermöglicht, Elemente zu stylen, die Kind-Elemente enthalten, die in der Selektorliste angegeben sind.

## Pseudoelemente

- **`::before` Pseudoelement**: Dieses Pseudoelement verwendet die `content` Eigenschaft, um kosmetische Inhalte wie Symbole direkt vor dem Element einzufügen.
- **`::after` Pseudoelement**: Dieses Pseudoelement verwendet die `content` Eigenschaft, um kosmetische Inhalte wie Symbole direkt nach dem Element einzufügen.
- **`::first-letter` Pseudoelement**: Dieses Pseudoelement spricht den ersten Buchstaben des Inhalts eines Elements an und ermöglicht dessen Styling.
- **`::marker` Pseudoelement**: Dieses Pseudoelement ermöglicht die Auswahl des Markers (Aufzählungszeichen oder Nummerierung) von Listenelementen zum Stylen.


## Farbtheorie

- **Definition der Farbtheorie**: Die Farbtheorie untersucht, wie Farben miteinander interagieren und wie sie unsere Wahrnehmung beeinflussen. Sie umfasst Farbbeziehungen, Farbharmonie und die psychologische Wirkung von Farben.
- **Primärfarben**: Diese Farben – Gelb, Blau und Rot – sind die grundlegenden Farbtöne, aus denen alle anderen Farben abgeleitet werden.
- **Sekundärfarben**: Diese Farben entstehen durch Mischen gleicher Mengen von zwei Primärfarben. Grün, Orange und Lila sind Beispiele für Sekundärfarben.
- **Tertiärfarben**: Diese Farben entstehen durch Kombinieren einer Primärfarbe mit einer benachbarten Sekundärfarbe. Gelb-Grün, Blau-Grün und Blau-Violett sind Beispiele für Tertiärfarben.
- **Warme Farben**: Diese Farben, zu denen Rot, Orange und Gelb gehören, vermitteln Gefühle von Komfort, Wärme und Gemütlichkeit.
- **Kühle Farben**: Diese Farben, zu denen Blau, Grün und Violett gehören, vermitteln Gefühle von Ruhe, Gelassenheit und Professionalität.
- **Farbrad**: Das Farbrad ist ein kreisförmiges Diagramm, das zeigt, wie Farben zueinander in Beziehung stehen. Es ist ein wichtiges Werkzeug für Designer, da es bei der Auswahl von Farbkombinationen hilft.
- **Analoge Farbschemata**: Diese Farbschemata schaffen kohärente und beruhigende Erlebnisse. Sie bestehen aus Farben, die im Farbrad nebeneinander liegen.
- **Komplementäre Farbschemata**: Diese Farbschemata erzeugen hohen Kontrast und visuelle Wirkung. Ihre Farben liegen sich im Farbrad gegenüber.
- **Triadisches Farbschema**: Dieses Farbschema verwendet lebendige Farben, die ungefähr gleich weit voneinander entfernt sind. Verbunden bilden sie ein gleichseitiges Dreieck im Farbrad.
- **Monochromatisches Farbschema**: Bei diesem Farbschema stammen alle Farben von derselben Grundfarbe, indem Helligkeit, Dunkelheit und Sättigung angepasst werden. Es vermittelt Einheit und Harmonie und schafft dennoch Kontrast.

## Verschiedene Möglichkeiten, mit Farben in CSS zu arbeiten

- **Benannte Farben**: Diese Farben sind vordefinierte Farbnamen, die von Browsern erkannt werden. Beispiele sind `blue`, `darkred`, `lightgreen`.
- **`rgb()` Funktion**: RGB steht für Rot, Grün und Blau – die Primärfarben des Lichts. Diese drei Farben werden in unterschiedlichen Intensitäten kombiniert, um eine breite Farbpalette zu erzeugen. Die `rgb()` Funktion erlaubt es, Farben mit dem RGB-Farbmodell zu definieren.
- **`rgba()` Funktion**: Diese Funktion fügt einen vierten Wert – Alpha – hinzu, der die Transparenz der Farbe steuert.
- **`hsl()` Funktion**: HSL steht für Farbton, Sättigung und Helligkeit – drei wichtige Komponenten, die eine Farbe definieren.
- **`hsla()` Funktion**: Diese Funktion fügt einen vierten Wert – Alpha – hinzu, der die Deckkraft der Farbe steuert.
- **Hexadezimal**: Ein Hex-Code (kurz für hexadezimaler Code) ist eine sechsstellige Zeichenfolge, die Farben im RGB-Farbmodell darstellt. „Hex“ bezieht sich auf das Zahlensystem zur Basis 16, das die Ziffern 0 bis 9 und die Buchstaben A bis F verwendet.

## `box-shadow` Eigenschaft

- **Definition**: Die `box-shadow` Eigenschaft wendet einen oder mehrere Schatten um ein Element herum an.
- **Versatzwerte**: Sie müssen horizontale (`offset-x`) und vertikale (`offset-y`) Werte angeben. Positive `offset-x` verschieben den Schatten nach rechts, negative Werte nach links. Positive `offset-y` verschieben den Schatten nach unten, negative nach oben. Wenn ein Wert `0` ist, müssen Sie keine Einheit angeben.
- **Unschärferadius**: Dieser optionale Wert steuert, wie verschwommen der Schatten erscheint. Wenn er nicht angegeben wird, ist der Standardwert `0`, was scharfe Kanten erzeugt. Je höher der Wert, desto weicher der Schatten.
- **Ausbreitungsradius**: Dieser optionale Wert steuert, wie stark der Schatten sich ausdehnt oder verkleinert. Wenn er nicht angegeben wird, ist der Standardwert `0`.
- **Schattenfarbe**: Sie können die Farbe mit benannten Farben, Hex-Werten, `rgb()`, `rgba()`, `hsl()` oder `hsla()` Funktionen angeben.
- **`inset` Schlüsselwort**: Das Hinzufügen des `inset` Schlüsselworts platziert den Schatten innerhalb des Elements statt außerhalb.
- **Mehrere Box-Schatten anwenden**: Sie können mehrere Schatten anwenden, indem Sie sie mit Kommas trennen. Die Schatten werden von vorne nach hinten geschichtet.

### Syntax

```css
box-shadow: offset-x offset-y blur-radius spread-radius color;
```

## Lineare und radiale Verläufe

- **Lineare Verläufe**: Diese Verläufe erzeugen einen allmählichen Übergang zwischen Farben entlang einer geraden Linie. Sie können die Richtung dieser Linie und die verwendeten Farben steuern.
- **Radiale Verläufe**: Diese Verläufe erzeugen kreisförmige oder elliptische Verläufe, die von einem zentralen Punkt ausgehen.

## Bewährte Vorgehensweisen für das Gestalten von Eingaben

- **Gestalten von Eingaben**: Wie bei allen Textelementen müssen Sie sicherstellen, dass die Stile, die Sie auf Texteingaben anwenden, zugänglich sind. Das bedeutet, die Schriftart muss ausreichend groß sein, und die Farbe muss einen ausreichenden Kontrast zum Hintergrund haben. Eingabeelemente sind außerdem fokussierbar. Wenn Sie Ihre Stile bearbeiten, sollten Sie darauf achten, dass ein gut sichtbarer Indikator erhalten bleibt, wenn das Element den Fokus hat, zum Beispiel ein fetter Rahmen.

## Verwendung von `appearance: none` für Eingaben

- **`appearance: none`**: Browser wenden auf viele Elemente eine Standardgestaltung an. Die CSS-Eigenschaft `appearance: none` gibt Ihnen die vollständige Kontrolle über die Gestaltung, bringt aber einige Einschränkungen mit sich. Wenn Sie benutzerdefinierte Stile für Eingabeelemente erstellen, müssen Sie sicherstellen, dass Fokus- und Fehlerindikatoren weiterhin vorhanden sind.

## Häufige Probleme beim Styling von `datetime-local` und `color` Eigenschaften

- **Häufige Probleme**: Diese speziellen Eingabetypen verwenden komplexe Pseudoelemente, um Dinge wie Datums- und Farbauswähler zu erstellen. Das stellt eine große Herausforderung für das Styling dieser Eingaben dar. Eine Herausforderung ist, dass das Standardstyling vollständig vom Browser abhängt, sodass das CSS, das Sie schreiben, um den Auswähler wie gewünscht aussehen zu lassen, in einem anderen Browser ganz anders sein kann.

## CSS Overflow-Eigenschaft

- **Definition**: Overflow beschreibt, wie Elemente mit Inhalten umgehen, die die Größe des enthaltenden Elements überschreiten oder „überlaufen“. Overflow ist zweidimensional.
- **`overflow-x`**: Die x-Achse bestimmt den horizontalen Überlauf.
- **`overflow-y`**: Die y-Achse bestimmt den vertikalen Überlauf.

## CSS Transform-Eigenschaft

- **Definition**: Diese Eigenschaft ermöglicht es, verschiedene Transformationen auf Elemente anzuwenden, wie Drehen, Skalieren, Schrägstellen oder Verschieben (Translatieren) im 2D- oder 3D-Raum.
- **`translate()` Funktion**: Diese Funktion wird verwendet, um ein Element von seiner aktuellen Position zu verschieben.
- **`scale()` Funktion**: Diese Funktion erlaubt es, die Größe eines Elements zu ändern.
- **Transformationen und Barrierefreiheit**: Wenn Sie transform verwenden, um Inhalte zu verbergen oder anzuzeigen, stellen Sie sicher, dass die Inhalte weiterhin für Screenreader und Tastaturnavigation zugänglich sind. Versteckte Inhalte sollten wirklich verborgen sein, z. B. durch Verwendung von `display: none` oder `visibility: hidden`, und nicht nur visuell aus dem Bildschirm verschoben werden.

## Das Box-Modell

- **Definition**: Im CSS-Box-Modell ist jedes Element von einer Box umgeben. Diese Box besteht aus vier Komponenten: dem Inhaltsbereich, `padding`, `border`, `margin`.
- **Inhaltsbereich**: Der Inhaltsbereich ist der innerste Teil der Box. Er ist der Raum, der den eigentlichen Inhalt eines Elements enthält, wie Text oder Bilder.
- **`padding`**: Der Innenabstand ist der Bereich direkt nach dem Inhaltsbereich. Er ist der Raum zwischen dem Inhaltsbereich und dem Rahmen eines Elements.
- **`border`**: Der Rahmen ist der äußere Rand oder die Umrandung eines Elements im CSS-Box-Modell. Er ist die sichtbare Begrenzung des Elements.
- **`margin`**: Der Außenabstand ist der Raum außerhalb des Rahmens eines Elements. Er bestimmt den Abstand zwischen einem Element und anderen Elementen in seiner Umgebung.

## Außenabstandszusammenfall

- **Definition**: Dieses Verhalten tritt auf, wenn sich die vertikalen Außenabstände benachbarter Elemente überlappen und zu einem einzigen Außenabstand führen, der dem größeren der beiden entspricht. Dieses Verhalten gilt nur für vertikale Außenabstände (oben und unten), nicht für horizontale (links und rechts).

## Die `content-box` und `border-box` Eigenschaftswerte

- **`box-sizing` Eigenschaft**: Diese Eigenschaft wird verwendet, um zu bestimmen, wie die endgültigen `width` und `height` für ein HTML-Element berechnet werden.
- **`content-box` Wert**: Im `content-box` Modell bestimmen die `width` und `height`, die Sie für ein Element festlegen, die Dimensionen des Inhaltsbereichs, schließen aber nicht `padding`, `border` oder `margin` ein.
- **`border-box` Wert**: Mit `border-box` umfassen die `width` und `height` eines Elements den Inhaltsbereich, den `padding` und den `border`, schließen aber den `margin` nicht ein.

## CSS-Reset

- **Definition**: Ein CSS-Reset ist ein Stylesheet, das alle oder einige der Standardformatierungen entfernt, die Webbrowser auf HTML-Elemente anwenden. Drittanbieter-Optionen für CSS-Resets umfassen `sanitize.css` und `normalize.css`.

## CSS Filter-Eigenschaft

- **Definition**: Diese Eigenschaft kann verwendet werden, um verschiedene Effekte wie Unschärfe, Farbverschiebung und Kontrastanpassung zu erzeugen.
- **`blur()` Funktion**: Diese Funktion wendet eine Gaußsche Unschärfe auf das Element an. Die Menge wird in Pixeln definiert und stellt den Radius der Unschärfe dar.
- **`brightness()` Funktion**: Diese Funktion passt die Helligkeit des Elements an. Ein Wert von 0 % macht das Element komplett schwarz, während Werte über 100 % die Helligkeit erhöhen.
- **`grayscale()` Funktion**: Diese Funktion wandelt das Element in Graustufen um. Die Menge wird als Prozentsatz definiert, wobei 100 % komplett Graustufen bedeutet und 0 % das Bild unverändert lässt.
- **`sepia()` Funktion**: Diese Funktion wendet einen Sepia-Ton auf das Element an. Wie bei Graustufen wird ein Prozentwert verwendet.
- **`hue-rotate()` Funktion**: Diese Funktion wendet eine Farbtonrotation auf das Element an. Der Wert wird in Grad angegeben und stellt eine Drehung im Farbkreis dar.

## Einführung in CSS Flexbox und Flex-Modell

- **Definition**: CSS Flexbox ist ein eindimensionales Layoutmodell, mit dem Sie Elemente in Reihen und Spalten innerhalb eines Containers anordnen können.
- **Flex-Modell**: Dieses Modell definiert, wie Flex-Elemente innerhalb eines Flex-Containers angeordnet werden. Jeder Flex-Container hat zwei Achsen: die Hauptachse und die Querachse.

## Die `flex-direction` Eigenschaft

- **Definition**: Diese Eigenschaft legt die Richtung der Hauptachse fest. Der Standardwert von `flex-direction` ist `row`, wodurch alle Flex-Elemente in derselben Reihe in der Richtung der Standardsprache Ihres Browsers (von links nach rechts oder von rechts nach links) angeordnet werden.
- **`flex-direction: row-reverse;`**: Dies kehrt die Reihenfolge der Elemente in der Reihe um.
- **`flex-direction: column;`**: Dies richtet die Flex-Elemente vertikal statt horizontal aus.
- **`flex-direction: column-reverse;`**: Dies kehrt die Reihenfolge der Flex-Elemente vertikal um.

## Die `flex-wrap` Eigenschaft

- **Definition**: Diese Eigenschaft bestimmt, wie Flex-Elemente innerhalb eines Flex-Containers umgebrochen werden, um in den verfügbaren Platz zu passen. `flex-wrap` kann drei mögliche Werte annehmen: `nowrap`, `wrap` und `wrap-reverse`.
- **`flex-wrap: nowrap;`**: Dies ist der Standardwert. Flex-Elemente werden nicht auf eine neue Zeile umgebrochen, selbst wenn ihre Breite die Containerbreite überschreitet.
- **`flex-wrap: wrap;`**: Diese Eigenschaft bewirkt, dass die Elemente umgebrochen werden, wenn sie die Breite ihres Containers überschreiten.
- **`flex-wrap: wrap-reverse;`**: Diese Eigenschaft bewirkt, dass Flex-Elemente in umgekehrter Reihenfolge umgebrochen werden.
- **`flex-flow` Eigenschaft**: Diese Eigenschaft ist eine Kurzschreibweise für `flex-direction` und `flex-wrap`.

## Die `justify-content` Eigenschaft

- **Definition**: Diese Eigenschaft richtet die Kindelemente entlang der Hauptachse des Flex-Containers aus.
- **`justify-content: flex-start;`**: In diesem Fall werden die Flex-Elemente am Anfang der Hauptachse ausgerichtet. Dies kann horizontal oder vertikal sein.
- **`justify-content: flex-end;`**: In diesem Fall sind die Flex-Elemente am Ende der Hauptachse ausgerichtet, horizontal oder vertikal.
- **`justify-content: center;`**: Dies zentriert die Flex-Elemente entlang der Hauptachse.
- **`justify-content: space-between;`**: Dies verteilt die Elemente gleichmäßig entlang der Hauptachse.
- **`justify-content: space-around;`**: Dies verteilt die Flex-Elemente gleichmäßig innerhalb der Hauptachse und fügt einen Abstand vor dem ersten und nach dem letzten Element hinzu.
- **`justify-content: space-evenly;`**: Dies verteilt die Elemente gleichmäßig entlang der Hauptachse.
  
## Die `align-items` Eigenschaft

- **Definition**: Diese Eigenschaft wird verwendet, um Elemente entlang der Querachse zu verteilen. Denken Sie daran, dass die Querachse senkrecht zur Hauptachse steht.
- **`align-items: center;`**: Dies wird verwendet, um die Elemente entlang der Querachse zu zentrieren.
- **`align-items: flex-start;`**: Dies richtet die Elemente am Anfang der Querachse aus.
- **`align-items: stretch;`**: Dies wird verwendet, um die Flex-Elemente entlang der Querachse zu strecken.

## Einführung in Typografie

- **Definition**: Typografie ist die Kunst, die richtigen Schriftarten und Formate auszuwählen, um Text visuell ansprechend und leicht lesbar zu machen. „Type“ bezieht sich darauf, wie die einzelnen Zeichen gestaltet und angeordnet sind.
- **Definition Schriftart**: Eine Schriftart ist das Gesamtdesign und der Stil einer Gruppe von Zeichen, Zahlen und Symbolen. Sie ist wie ein Bauplan für eine Schriftfamilie.
- **Definition Font**: Ein Font ist eine spezifische Variation einer Schriftart mit bestimmten Merkmalen wie Größe, Gewicht, Stil und Breite.
- **Serifenschrift**: Diese Schriftart hat einen klassischen Stil mit kleinen Linien am Ende der Zeichen. Serifenschriften werden häufig für gedruckte Materialien wie Bücher verwendet.
- **Serifenlose Schrift**: Diese Schriftart hat ein moderneres Aussehen, ohne die kleinen Linien am Ende der Zeichen. Serifenlose Schriften werden häufig im digitalen Design verwendet, da sie auf Bildschirmen gut lesbar sind. Beispiele sind Helvetica, Arial und Roboto.
- **Schriftstärke**: Dies ist die Dicke der Zeichen, einschließlich leicht, normal, fett und schwarz.
- **Schriftstil**: Dies ist die Neigung und Ausrichtung der Zeichen, wie kursiv und schräg.
- **Schriftbreite**: Dies ist der horizontale Raum, den Zeichen einnehmen, wie kondensiert und erweitert.
- **Grundlinie**: Dies ist die imaginäre Linie, auf der die meisten Zeichen ruhen.
- **Versalhöhe**: Dies ist die Höhe der Großbuchstaben, gemessen von der Grundlinie bis zur Oberkante.
- **x-Höhe**: Dies ist die durchschnittliche Höhe der Kleinbuchstaben, ohne Auf- und Abstriche.
- **Aufstriche**: Dies sind die Teile von Kleinbuchstaben, die über die x-Höhe hinausgehen, wie die Oberteile der Buchstaben „h“, „b“, „d“ und „f“.
- **Abstriche**: Dies sind die Teile von Kleinbuchstaben, die unter die Grundlinie reichen, wie die Schwänze von „y“, „g“, „p“ und „q“.
- **Kerning**: Dies ist die Anpassung des Abstands zwischen bestimmten Buchstabenpaaren, um die Lesbarkeit und Ästhetik zu verbessern. Zum Beispiel das Verringern des Abstands zwischen den Buchstaben A und V.
- **Tracking**: Dies ist die Anpassung des Abstands zwischen allen Zeichen in einem Textblock. Es beeinflusst, wie dicht oder luftig der Text wirkt.
- **Zeilenabstand (Leading)**: Dies ist der vertikale Abstand zwischen Textzeilen. Er wird von der Grundlinie einer Zeile bis zur Grundlinie der nächsten gemessen.
- **Bewährte Praktiken bei Typografie**: Sie sollten klare und einfache Schriftarten wählen, um Ihre Designs leicht verständlich zu machen. Das ist besonders wichtig für den Haupttext Ihrer Webseite. Nutzer beschäftigen sich eher mit Ihrem Inhalt, wenn die Schrift gut lesbar ist. Verwenden Sie höchstens zwei oder drei Schriftarten, um visuelle Konsistenz zu schaffen. Zu viele Schriftarten erschweren die Lesbarkeit und schwächen die Markenidentität. Dies kann auch die Nutzererfahrung beeinträchtigen, indem die Ladezeit der Webseite erhöht wird. Sie können die Schriftgröße verwenden, um eine visuelle Hierarchie für Überschriften, Unterüberschriften, Absätze und andere Elemente zu schaffen. Zum Beispiel sollte die Hauptüberschrift auf einer Webseite eine größere Schriftgröße haben, gefolgt von Unterüberschriften mit kleineren Schriftgrößen. So erhält jedes Element in der Hierarchie eine spezifische Schriftgröße, die Nutzern hilft, die Struktur visuell zu erfassen.

  
## CSS `font-family` Eigenschaft

- **Definition**: Eine Schriftfamilie ist eine Gruppe von Fonts, die ein gemeinsames Design teilen. Alle Fonts, die zur gleichen Familie gehören, basieren auf derselben Grundschriftart, haben aber Variationen in Stil, Gewicht und Breite. Sie können mehrere Schriftfamilien in der Reihenfolge der Priorität von höchster zu niedrigster angeben, getrennt durch Kommas. Diese alternativen Fonts dienen als Fallback-Optionen. Sie sollten immer eine generische Schriftfamilie am Ende der font-family-Liste angeben.
- **Gängige Schriftfamilien**: Hier sind einige gängige Schriftfamilien, die in CSS verwendet werden: serif, sans-serif, monospace, cursive, fantasy

## Web-sichere Schriftarten

- **Definition**: Diese Schriftarten sind eine Teilmenge von Fonts, die sehr wahrscheinlich auf einem Computer oder Gerät installiert sind. Wenn der Browser eine Schriftart rendern muss, versucht er, die Schriftdatei im System des Nutzers zu finden. Wenn die Schriftart nicht gefunden wird, fällt er normalerweise auf eine Standardsystemschriftart zurück.

- **Web-sichere Schriftarten**:
  - Serifenlose Schriftarten werden häufig für Webentwicklung verwendet, da sie keine kleinen „Füße“ oder Linien am Ende der Zeichen haben und daher auf Bildschirmen gut lesbar sind. Beispiele für web-sichere serifenlose Schriftarten sind Arial, Verdana und Trebuchet MS.
  - Im Gegensatz dazu haben Serifenschriften kleine „Füße“ am Ende der Zeichen und werden häufig für traditionelle Druckerzeugnisse verwendet. Web-sichere Serifenschriften sind Times New Roman und Georgia.

## At-Regeln und die `@font-face` At-Regel

- **Definition**: At-Regeln sind Anweisungen, die dem Browser Anweisungen geben. Sie können verwendet werden, um verschiedene Aspekte des Stylesheets zu definieren, wie Media Queries, Keyframes, Font Faces und mehr.
- **`@font-face`**: Diese At-Regel ermöglicht es, eine benutzerdefinierte Schriftart zu definieren, indem die Schriftdatei, das Format und andere wichtige Eigenschaften wie Gewicht und Stil angegeben werden. Damit die `@font-face` At-Regel gültig ist, müssen Sie auch die `src` Eigenschaft angeben. Diese Eigenschaft enthält Referenzen auf die Schriftressourcen.
- **Schriftformate**: Für jede Schriftressource können Sie optional auch das Format angeben. Dies ist ein Hinweis für den Browser zum Schriftformat. Wenn das Format weggelassen wird, wird die Ressource heruntergeladen und das Format nach dem Download erkannt. Wenn das Format ungültig ist, wird die Ressource nicht heruntergeladen. Mögliche Schriftformate sind `collection`, `embedded-opentype`, `opentype`, `svg`, `truetype`, `woff` und `woff2`.
- **`woff` und `woff2`**: `woff` steht für „Web Open Font Format“. Es ist ein weit verbreitetes Schriftformat, das von Mozilla in Zusammenarbeit mit Type Supply, LettError und anderen Organisationen entwickelt wurde. Der Unterschied zwischen `woff` und `woff2` liegt im verwendeten Kompressionsalgorithmus.
- **OpenType**: Dies ist ein Format für skalierbare Computerschriften, entwickelt von Microsoft und Adobe, das es Nutzern ermöglicht, zusätzliche Funktionen in einer Schriftart zu nutzen. Es wird auf den wichtigsten Betriebssystemen weit verbreitet verwendet.
- **`tech()`**: Dies wird verwendet, um die Technologie der Schriftressource anzugeben. Dies ist optional.

## Arbeiten mit externen Schriftarten

- **Definition**: Eine externe Schriftart ist eine Schriftdatei, die nicht direkt in Ihren Projektdateien enthalten ist. Sie wird normalerweise auf einem separaten Server gehostet. Um diese externen Schriftarten in Ihr Projekt einzubinden, können Sie ein `link` Element oder eine `@import` Anweisung in Ihrem CSS verwenden.
- **Google Fonts**: Ein Google-Dienst, der eine Sammlung von Schriftarten anbietet, von denen viele speziell für die Webentwicklung entworfen wurden.
- **Font Squirrel**: Eine weitere beliebte Ressource, die Sie für das Hinzufügen benutzerdefinierter externer Schriftarten zu Ihren Projekten verwenden können.

## `text-shadow` Eigenschaft

- **Definition**: Diese Eigenschaft wird verwendet, um Schatten auf Text anzuwenden. Sie müssen den X- und Y-Versatz angeben, die den horizontalen und vertikalen Abstand des Schattens vom Text darstellen. Diese Werte sind erforderlich. Positive Werte für X- und Y-Versatz verschieben den Schatten nach rechts bzw. unten, negative Werte nach links bzw. oben.
- **Schattenfarbe**: Sie können die Farbe des Schattens auch anpassen, indem Sie diesen Wert vor oder nach dem Versatz angeben. Wenn die Farbe nicht angegeben wird, bestimmt der Browser die Farbe automatisch, daher ist es meist besser, sie explizit zu setzen.
- **Unschärferadius**: Der Unschärferadius ist optional, macht den Schatten aber viel weicher und subtiler. Der Standardwert für den Radius ist null. Je höher der Wert, desto größer die Unschärfe, was den Schatten heller macht.
- **Mehrere Textschatten anwenden**: Der Text kann mehr als einen Schatten haben. Die Schatten werden in Schichten von vorne nach hinten angewendet, wobei der erste Schatten oben liegt.

## Barrierefreiheitsbaum

Der Barrierefreiheitsbaum ist eine Struktur, die von Hilfstechnologien wie Screenreadern verwendet wird, um den Inhalt einer Webseite zu interpretieren und mit ihm zu interagieren.

## `max()` Funktion

Die `max()` Funktion gibt den größten Wert aus einer Menge von durch Kommas getrennten Werten zurück:

```css
img {
  width: max(250px, 25vw);
}
```

Im obigen Beispiel beträgt die Breite des Bildes 250px, wenn die Viewport-Breite weniger als 1000 Pixel beträgt (da 250px größer als 25 % dieser Viewport-Breite ist). Wenn die Viewport-Breite größer als 1000 Pixel ist, beträgt die Breite des Bildes 25vw. Das liegt daran, dass 25vw 25 % der Viewport-Breite entspricht.

## Beste Praktiken mit CSS und Barrierefreiheit

- **`display: none;`**: Die Verwendung von `display: none;` bedeutet, dass Screenreader und andere Hilfstechnologien nicht auf diesen Inhalt zugreifen können, da er nicht im Barrierefreiheitsbaum enthalten ist. Daher ist es wichtig, diese Methode nur zu verwenden, wenn Sie Inhalte vollständig aus der visuellen Darstellung und der Barrierefreiheit entfernen möchten.
- **`visibility: hidden;`**: Diese Eigenschaft und ihr Wert verbergen den Inhalt visuell, behalten ihn aber im Dokumentfluss, sodass er weiterhin Platz auf der Seite einnimmt. Diese Elemente werden auch nicht mehr von Screenreadern gelesen, da sie aus dem Barrierefreiheitsbaum entfernt wurden.
- **`.sr-only` CSS-Klasse**: Dies ist eine gängige Technik, um Inhalte visuell zu verbergen, während sie für Screenreader zugänglich bleiben.

```css
.sr-only {
  position: absolute;
  width: 1px;
  height: 1px;
  padding: 0;
  margin: -1px;
  overflow: hidden;
  clip: rect(0, 0, 0, 0);
  white-space: nowrap;
  border: 0;
}
```

- **`scroll-behavior: smooth;`**: Diese Eigenschaft und ihr Wert ermöglichen ein sanftes Scrollverhalten.
- **`prefers-reduced-motion` Feature**: Dies ist ein Media-Feature, mit dem die Animationspräferenz des Nutzers erkannt werden kann.

```css
@media (prefers-reduced-motion: no-preference) {
  * {
    scroll-behavior: smooth;
  }
}
```

Im obigen Beispiel ist sanftes Scrollen aktiviert, wenn der Nutzer keine Animationspräferenz auf seinem Gerät eingestellt hat.

## Inhalte mit HTML-Attributen verbergen

- **`aria-hidden` Attribut**: Wird verwendet, um ein Element für Personen mit Hilfstechnologien wie Screenreadern zu verbergen. Zum Beispiel kann es verwendet werden, um dekorative Bilder zu verbergen, die keinen bedeutungsvollen Inhalt liefern.
- **`hidden` Attribut**: Dieses Attribut wird von den meisten modernen Browsern unterstützt und verbirgt Inhalte sowohl visuell als auch im Barrierefreiheitsbaum. Es kann einfach mit JavaScript umgeschaltet werden.

:::interactive_editor

```html
<p aria-hidden>This paragraph is visible for sighted users, but is hidden from assistive technology.</p>
<p hidden>This paragraph is hidden from both sighted users and assistive technology.</p>
```

:::

## Barrierefreiheitsproblem des `placeholder` Attributs

Die Verwendung von Platzhaltertext ist für die Barrierefreiheit nicht ideal. Zu oft verwechseln Nutzer den Platzhaltertext mit einem tatsächlichen Eingabewert – sie denken, es sei bereits ein Wert im Eingabefeld vorhanden.

## Arbeiten mit verschiedenen Attributselektoren und Links

- **Definition**: Der `attribute` Selektor ermöglicht es, HTML-Elemente basierend auf ihren Attributen wie `href` oder `title` anzusprechen.
- **`title` Attribut**: Dieses Attribut liefert zusätzliche Informationen über ein Element.

## Elemente mit den `lang` und `data-lang` Attributen ansprechen

- **`lang` Attribut**: Dieses Attribut wird in HTML verwendet, um die Sprache des Inhalts innerhalb eines Elements anzugeben. Sie möchten Elemente möglicherweise unterschiedlich stylen, basierend auf der Sprache, in der sie geschrieben sind, insbesondere auf einer mehrsprachigen Webseite.
- **`data-lang` Attribut**: Benutzerdefinierte Datenattribute wie das `data-lang` Attribut werden häufig verwendet, um zusätzliche Informationen in Elementen zu speichern, z. B. um die Sprache innerhalb eines bestimmten Textabschnitts anzugeben.


## Arbeiten mit Attributselektoren, geordneten Listen und dem `type` Attribut

- **`type` Attribut**: Beim Arbeiten mit geordneten Listen in HTML ermöglicht das `type` Attribut, den Stil der Nummerierung festzulegen, z. B. numerisch, alphabetisch oder römisch.

## Arbeiten mit Floats

- **Definition**: Floats werden verwendet, um ein Element aus dem normalen Fluss der Seite zu entfernen und es entweder links oder rechts im Container zu positionieren. Wenn dies geschieht, umfließt Text den gefloateten Inhalt.
- **Floats löschen**: Die `clear` Eigenschaft wird verwendet, um zu bestimmen, ob ein Element unter dem gefloateten Inhalt platziert werden muss. Wenn Sie mehrere gefloatete Elemente nebeneinander stapeln, kann es zu Überlappungen und Zusammenbrüchen im Layout kommen. Deshalb wurde ein `clearfix` Hack entwickelt, um dieses Problem zu beheben.

## Statische, relative und absolute Positionierung

- **Statische Positionierung**: Dies ist der normale Fluss des Dokuments. Elemente werden von oben nach unten und von links nach rechts nacheinander positioniert.
- **Relative Positionierung**: Diese erlaubt es, die `top`, `left`, `right` und `bottom` Eigenschaften zu verwenden, um das Element innerhalb des normalen Dokumentflusses zu positionieren. Sie können relative Positionierung auch verwenden, um Elemente über andere Elemente auf der Seite zu legen.
- **Absolute Positionierung**: Diese erlaubt es, ein Element aus dem normalen Dokumentfluss zu nehmen, sodass es unabhängig von anderen Elementen agiert.

## Feste und klebende Positionierung

- **Feste Positionierung**: Wenn ein Element mit `position: fixed` positioniert wird, wird es aus dem normalen Dokumentfluss entfernt und relativ zum Viewport platziert, sodass es an derselben Position bleibt, auch wenn der Nutzer scrollt. Dies wird oft für Elemente wie Kopfzeilen oder Navigationsleisten verwendet, die immer sichtbar bleiben sollen.
- **Klebende Positionierung**: Diese Positionierungsart wirkt als Hybrid zwischen relativer und fester Positionierung. Anfangs verhält sich das Element wie relativ positioniert und bleibt im Dokumentfluss. Sobald der Nutzer das Element an eine bestimmte Position scrollt, „klebt“ es am Viewport (meist oben) und verhält sich wie fest positioniert. Dies ist ideal für Elemente wie klebende Navigationsleisten, die erst beim Scrollen fixiert werden.

## Arbeiten mit der `z-index` Eigenschaft

- **Definition**: Die `z-index` Eigenschaft in CSS wird verwendet, um die vertikale Stapelreihenfolge von positionierten Elementen zu steuern, die sich auf der Seite überlappen.

## Responsives Webdesign

- **Definition**: Das Kernprinzip des responsiven Designs ist Anpassungsfähigkeit – die Fähigkeit einer Webseite, ihr Layout und ihren Inhalt basierend auf der Bildschirmgröße und den Fähigkeiten des Geräts, auf dem sie betrachtet wird, anzupassen.
- **Flüssige Raster**: Diese verwenden relative Einheiten wie Prozentsätze anstelle von festen Einheiten wie Pixeln, sodass Inhalte basierend auf der Bildschirmgröße skaliert und umgebrochen werden.
- **Flexible Bilder**: Diese werden so eingestellt, dass sie sich innerhalb ihrer enthaltenden Elemente skalieren, um sicherzustellen, dass sie auf kleineren Bildschirmen nicht überlaufen.

## Media Queries

- **Definition**: Damit können Entwickler unterschiedliche Stile basierend auf den Eigenschaften des Geräts anwenden, hauptsächlich der Viewport-Breite.
- **`all` Medientyp**: Dieser ist für alle Geräte geeignet. Er ist der Standard, wenn kein Medientyp angegeben ist.
- **`print` Medientypen**: Dieser ist für paginierte Materialien und Dokumente gedacht, die im Druckvorschau-Modus auf einem Bildschirm angezeigt werden.
- **`screen` Medientypen**: Dieser ist hauptsächlich für Bildschirme gedacht.
- **`aspect-ratio`**: Dies beschreibt das Verhältnis zwischen Breite und Höhe des Viewports.
- **`orientation`**: Dies wird verwendet, um anzugeben, ob das Gerät im Quer- oder Hochformat ist.
- **`resolution`**: Dies beschreibt die Auflösung des Ausgabegeräts in Punkten pro Zoll (dpi) oder Punkten pro Zentimeter (dpcm).
- **`hover`**: Dies wird verwendet, um zu testen, ob der primäre Eingabemechanismus über Elemente hovern kann.
- **`prefers-color-scheme`**: Dies wird verwendet, um zu erkennen, ob der Nutzer ein helles oder dunkles Farbschema angefordert hat.
- **Medienabfragen und logische Operatoren**: Der `and` Operator wird verwendet, um mehrere Media Features zu kombinieren, während `not` und `only` verwendet werden können, um Media Queries zu negieren oder zu isolieren.

## Übliche Media Breakpoints

- **Definition**: Media Breakpoints sind spezifische Punkte im Design einer Webseite, an denen Layout und Inhalt angepasst werden, um unterschiedliche Bildschirmgrößen zu berücksichtigen. Es gibt einige allgemeine Breakpoints, die Sie verwenden können, um Telefone, Tablets und Desktop-Bildschirme anzusprechen. Es ist jedoch nicht ratsam, jede einzelne mögliche Bildschirmgröße für verschiedene Geräte zu verfolgen.
- **Kleine Geräte (Smartphones)**: bis 640px
- **Mittlere Geräte (Tablets)**: 641px bis 1024px
- **Große Geräte (Desktops)**: 1025px und größer

## Mobile-first-Ansatz

- **Definition**: Der Mobile-first-Ansatz ist eine Designphilosophie und Entwicklungsstrategie im responsiven Webdesign, die es priorisiert, Websites zuerst für mobile Geräte zu erstellen, bevor sie für größere Bildschirme gestaltet werden.

## CSS-Custom-Properties (CSS-Variablen)

- **Definition**: CSS-Custom-Properties, auch CSS-Variablen genannt, sind von CSS-Autoren definierte Entitäten, die bestimmte Werte enthalten, die im gesamten Dokument wiederverwendet werden können. Sie sind eine leistungsstarke Funktion, die effizientere, wartbarere und flexiblere Stylesheets ermöglicht. Custom Properties sind besonders nützlich, um themenfähige Designs zu erstellen. Sie können einen Satz von Eigenschaften für verschiedene Themen definieren.

## Die `@property` Regel

- **Definition**: Die `@property` Regel ist eine mächtige CSS-Funktion, die Entwicklern erlaubt, benutzerdefinierte Eigenschaften mit größerer Kontrolle über ihr Verhalten zu definieren, einschließlich ihrer Animationen und Anfangswerte.

```css
@property --property-name {
  syntax: '<type>';
  inherits: true | false;
  initial-value: <value>;
}
```

- **`--property-name`**: Dies ist der Name der benutzerdefinierten Eigenschaft, die Sie definieren. Wie alle benutzerdefinierten Eigenschaften muss sie mit zwei Bindestrichen beginnen.
- **`syntax`**: Dies definiert den Typ der Eigenschaft, der Dinge wie `<color>`, `<length>`, `<number>`, `<percentage>` oder komplexere Typen sein kann.
- **`inherits`**: Dies gibt an, ob die Eigenschaft ihren Wert vom übergeordneten Element erben soll.
- **`initial-value`**: Dies legt den Standardwert der Eigenschaft fest.
- **Fallbacks**: Beim Verwenden der benutzerdefinierten Eigenschaft können Sie einen Fallback-Wert mit der `var()` Funktion angeben, genau wie bei Standard-Custom-Properties.

## CSS Grid-Grundlagen

- **Definition**: CSS Grid ist ein zweidimensionales Layoutsystem, das verwendet wird, um komplexe Layouts auf Webseiten zu erstellen. Grids haben Reihen und Spalten mit Abständen dazwischen. Um ein Grid-Layout zu definieren, setzen Sie die `display` Eigenschaft auf `grid`.
- **Die `fr` (Bruch-) Einheit**: Diese Einheit repräsentiert einen Bruchteil des Raums innerhalb des Grid-Containers. Sie können die `fr` Einheit verwenden, um flexible Grids zu erstellen.
- **Abstände zwischen Tracks erstellen**: Es gibt drei Möglichkeiten, Abstände zwischen Tracks in CSS Grid zu erzeugen. Sie können die `column-gap` Eigenschaft verwenden, um Abstände zwischen Spalten zu schaffen. Die `row-gap` Eigenschaft erzeugt Abstände zwischen Reihen. Oder Sie verwenden die `gap` Kurzschreibweise, um Abstände zwischen Reihen und Spalten zu erzeugen.
- **`repeat()` Funktion**: Diese Funktion wird verwendet, um Abschnitte in der Track-Liste zu wiederholen. Anstatt `grid-template-columns: 1fr 1fr 1fr;` zu schreiben, können Sie stattdessen die `repeat()` Funktion verwenden.
- **Explizites Grid**: Sie können die Anzahl der Linien oder Tracks mit den `grid-template-columns` oder `grid-template-rows` Eigenschaften angeben.
- **Implizites Grid**: Wenn Elemente außerhalb des Grids platziert werden, werden automatisch Reihen und Spalten für diese außerhalb liegenden Elemente erstellt.
- **`minmax()` Funktion**: Diese Funktion wird verwendet, um die minimalen und maximalen Größen für einen Track festzulegen.
- **Automatische Grid-Platzierung**: Der Browser platziert Elemente automatisch mit dem Auto-Placement-Algorithmus. Die `grid-auto-flow` Eigenschaft steuert dieses Verhalten und verwendet Werte wie `row`, `column` oder `dense`, um die Platzierungsrichtung zu beeinflussen und zu bestimmen, ob leere Lücken aufgefüllt werden.
- **Linienbasierte Platzierung**: Alle Grids haben Linien. Um anzugeben, wo ein Element auf einer Linie beginnt, können Sie die `grid-column-start` und `grid-row-start` Eigenschaften verwenden. Um anzugeben, wo das Element auf der Linie endet, verwenden Sie die `grid-column-end` und `grid-row-end` Eigenschaften. Alternativ können Sie die `grid-column` oder `grid-row` Kurzschreibweise verwenden.
- **`grid-template-areas`**: Diese Eigenschaft wird verwendet, um einem Element, das Sie im Grid positionieren, einen Namen zu geben.
- **Ausrichtung**: Verwenden Sie `justify-items` und `align-items`, um zu steuern, wie Elemente innerhalb ihrer Grid-Bereiche ausgerichtet werden. Verwenden Sie `justify-self` und `align-self` bei einzelnen Grid-Elementen, um die Ausrichtung des Containers zu überschreiben.

## CSS-Animationsgrundlagen

- **Definition**: CSS-Animationen ermöglichen es, dynamische, visuell ansprechende Effekte auf Webseiten zu erstellen, ohne JavaScript oder komplexe Programmierung zu benötigen. Sie bieten eine Möglichkeit, Elemente über eine bestimmte Dauer sanft zwischen verschiedenen Stilen zu überblenden.
- **Die `@keyframes` Regel**: Diese Regel definiert die Phasen und Stile der Animation. Sie legt fest, welche Stile das Element zu verschiedenen Zeitpunkten während der Animation haben soll.
- **`animation` Eigenschaft**: Dies ist die Kurzschreibweise, um Animationen anzuwenden.
- **`animation-name`**: Dies legt den Namen der `@keyframes` Regel fest, die verwendet werden soll.
- **`animation-duration`**: Dies legt fest, wie lange die Animation dauern soll.
- **`animation-timing-function`**: Dies definiert, wie die Animation über die Zeit verläuft (z. B. ease, linear, ease-in-out).
- **`animation-delay`**: Dies legt eine Verzögerung fest, bevor die Animation startet.
- **`animation-iteration-count`**: Dies legt fest, wie oft die Animation wiederholt werden soll.
- **`animation-direction`**: Dies bestimmt, ob die Animation vorwärts, rückwärts oder abwechselnd abgespielt wird.
- **`animation-fill-mode`**: Dies legt fest, wie das Element vor und nach der Animation gestylt wird.
- **`animation-play-state`**: Dies ermöglicht das Pausieren und Fortsetzen der Animation.

## Barrierefreiheit und die `prefers-reduced-motion` Media Query

- **Die `prefers-reduced-motion` Media Query**: Eine der Hauptbedenken bei Animationen in Bezug auf Barrierefreiheit ist, dass sie bei manchen Nutzern Unwohlsein oder sogar körperliche Beschwerden verursachen können. Menschen mit vestibulären Störungen oder Bewegungsempfindlichkeit können Schwindel, Übelkeit oder Kopfschmerzen erleben, wenn sie bestimmten Bewegungen auf dem Bildschirm ausgesetzt sind. Die `prefers-reduced-motion` Media Query ermöglicht es Webentwicklern, zu erkennen, ob der Nutzer auf Systemebene minimale Animationen oder Bewegungseffekte angefordert hat.
