# --interactive--

Bedingte Anweisungen ermöglichen es Ihnen, Entscheidungen in Ihrem JavaScript-Code zu treffen. Sie lassen Ihr Programm auf eine bestimmte Weise ablaufen, basierend auf bestimmten Bedingungen. Schauen wir uns an, wie `if`, `else if`, `else` und der ternäre Operator funktionieren, um Ihnen die Steuerung des Programmflusses zu ermöglichen.

Eine `if`-Anweisung nimmt eine Bedingung und führt einen Codeblock aus, wenn diese Bedingung einen truthy-Wert hat. Truthy-Werte sind alle Werte, die in einem Boolean-Kontext wie einer `true`-Anweisung zu `if` ausgewertet werden. Hier sind Beispiele für truthy-Werte:

- Nicht-leere Strings, zum Beispiel `hello`

- Jede Zahl außer `0` und `-0`, zum Beispiel `4`, `-5` und andere

- Arrays

- Objekte

- Der Boolean `true`

Andererseits sind falsy-Werte Werte, die in einem Boolean-Kontext zu `false` ausgewertet werden. JavaScript hat nur wenige falsy-Werte, was sie leicht merkbar macht. Hier sind einige falsy-Werte:

- Boolean `false`

- `0` (Null)

- `""` (leerer String)

- `null`

- `undefined`

- `NaN` (Not a Number)

Jetzt, da wir ein grundlegendes Verständnis von truthy- und falsy-Werten haben, sehen wir, wie das mit `if`-Anweisungen funktioniert. Im ersten Beispiel verwenden wir ein paar `if`-Anweisungen, um gegen truthy- und falsy-Werte zu prüfen:

:::interactive_editor

```js
if (null) {
  console.log("This will not run.");
}

if ("freeCodeCamp") {
  console.log("This will run.");
}
```

:::

Da `null` ein falsy-Wert ist, wird die Nachricht im Block niemals in der Konsole ausgegeben. Aber bei der zweiten `if`-Anweisung ist der String `freeCodeCamp` ein truthy-Wert und wird in diesem Boolean-Kontext der `true`-Anweisung als `if` betrachtet. Folglich wird die Nachricht `This will run.` in der Konsole ausgegeben.

Schauen wir uns noch ein paar weitere Beispiele an, wie `if`-Anweisungen mit verschiedenen Vergleichsoperatoren funktionieren. Hier ist ein Beispiel für die Verwendung einer `if`-Anweisung, um zu prüfen, ob der Benutzer wahlberechtigt ist:

:::interactive_editor

```js
const age = 22;

if (age >= 18) {
 console.log("You're eligible to vote"); // You're eligible to vote
}
```

:::

In diesem Beispiel, da `age` derzeit `22` ist, bedeutet das, dass die Bedingung zu `true` ausgewertet wird, weil `22` größer oder gleich `18` ist. Also wird die Nachricht `You're eligible to vote` in der Konsole ausgegeben. Wenn wir das Beispiel ändern, sodass `age` jetzt `15` ist, wird die Bedingung zu `false` ausgewertet und die Nachricht wird nicht in der Konsole ausgegeben:

:::interactive_editor

```js
const age = 15;

if (age >= 18) {
 console.log("You're eligible to vote"); // Code not running because age is less than 18
}
```

:::

Wenn eine Bedingung `false` ist, können Sie eine `else`-Klausel verwenden:

:::interactive_editor

```js
const age = 15;

if (age >= 18) {
 console.log("You're eligible to vote");
} else {
 console.log("You're not eligible to vote"); // You're not eligible to vote
}
```

:::

In diesem Beispiel ist `15` nicht größer oder gleich `18`, also wäre die Bedingung `false`. Der Code im `else`-Block wird in diesem Fall ausgeführt.

Wenn Sie mehrere Bedingungen prüfen möchten, können Sie einen `else if`-Block verwenden. Dadurch kann Ihr Programm zwischen mehr als zwei Pfaden wählen.

:::interactive_editor

```js
const score = 87;

if (score >= 90) {
 console.log('You got an A'); 
} else if (score >= 80) {
 console.log('You got a B'); // You got a B
} else if (score >= 70) {
 console.log('You got a C');
} else {
 console.log('You failed! You need to study more!');
}
```

:::

Da `score` derzeit `87` ist, würde die Nachricht `You got a B` in der Konsole ausgegeben.

Der ternäre Operator ist eine kompakte Möglichkeit, einfache `if/else`-Anweisungen zu schreiben. Er besteht aus drei Teilen: einer Bedingung, einem Ergebnis, wenn die Bedingung wahr ist, und einem Ergebnis, wenn sie falsch ist. Hier ist die grundlegende Syntax:

```js
condition ? expressionIfTrue : expressionIfFalse;
```

Hier ist ein Beispiel, das mit Wettertemperaturen in Celsius arbeitet:

:::interactive_editor

```js
const temperature = 30;
const weather = temperature > 25 ? 'sunny' : 'cool';

console.log(`It's a ${weather} day!`);
```

:::

Wenn `temperature` größer als `25` ist, gibt der obige Code `It's a sunny day!` aus. Wenn `temperature` jemals kleiner oder gleich `25` ist, gibt er `It's a cool day!` aus.

Welche sollten Sie also zwischen einer if-Anweisung und einem ternären Operator verwenden? Verwenden Sie einen ternären Operator, wenn Sie mit einer einzelnen Bedingung oder einzelnen Ausdrücken arbeiten oder wenn Sie eine kompakte Syntax für einfache Logik wünschen. Verwenden Sie `if/else`-Anweisungen, wenn Sie mit komplexen Bedingungen und mehreren Anweisungen arbeiten, da es unleserlich wird, wenn Sie ternäre Operatoren verschachteln.
