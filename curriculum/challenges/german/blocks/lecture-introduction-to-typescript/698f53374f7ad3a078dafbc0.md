# --description--

In dieser Lektion lernen Sie, wie Sie Typen für Objekte und Arrays hinzufügen.

Schauen wir uns zuerst Arrays an.

Hier ist ein Beispiel für ein Array von Programmiersprachen:

```js
const programmingLanguages = ["Java", "C++", "Python"];
```

Wenn Sie dieses Array explizit als Array von Strings typisieren möchten, können Sie es als `string[]` schreiben:

```ts
const programmingLanguages:string[] = ["Java", "C++", "Python"];
```

Wie Sie bereits gelernt haben, ist TypeScript intelligent genug, den Typ als Array von Strings zu erkennen, wenn Sie einfach Folgendes schreiben:

```js
const programmingLanguages = ["Java", "C++", "Python"];
```

Aber eine explizite Typisierung hilft in Situationen, in denen Sie versuchen, einen anderen Typ zum Array hinzuzufügen, wie hier:

```ts
// Type 'number' is not assignable to type 'string'.(2322)
const programmingLanguages:string[] = ["Java", 9, "Python"];
```

Eine andere Möglichkeit, Arrays zu typisieren, ist die Verwendung dieser Syntax:

```ts
const programmingLanguages:Array<string> = ["Java", "C++", "Python"];
```

Grundsätzlich sind diese beiden Syntaxen gleichwertig, und die Wahl zwischen ihnen ist meist eine Frage der Präferenz.

Kommen wir nun zur Typisierung von Objekten.

Hier ist ein Beispiel für ein Objektliteral:

```js
const developer = {
  firstName: "Jessica",
  isEmployed: true
}
```

Um die Eigenschaften des Objekts explizit zu typisieren, können Sie diese Syntax verwenden:

```ts
const developer: { firstName: string, isEmployed: boolean } = {
  firstName: "Jessica",
  isEmployed: true
}
```

Wenn Sie versuchen, Eigenschaften zum Objekt hinzuzufügen oder zu entfernen, erhalten Sie einen Fehler wie diesen:

```ts
// Property 'isEmployed' is missing in type '{ firstName: string; }' but required in type '{ firstName: string; isEmployed: boolean; }'
const developer: { firstName: string, isEmployed: boolean } = {
  firstName: "Jessica",
}
```

Sie können Objekteigenschaften optional machen, indem Sie nach dem Eigenschaftsnamen das `?` verwenden, so:

```ts
const developer: { firstName: string, isEmployed?: boolean } = {
  firstName: "Jessica",
}
```

TypeScript zeigt keinen Fehler mehr an, weil die `isEmployed`-Eigenschaft als optional markiert wurde.

Bisher hatten alle Objektbeispiele restriktive Eigenschaften. Das bedeutet, wir haben explizit jede Eigenschaft aufgelistet, die das Objekt haben darf:

```ts
const developer: { firstName: string, isEmployed: boolean } = {
  firstName: "Jessica",
  isEmployed: true
}
```

Aber was, wenn Sie nicht alle Eigenschaftsnamen im Voraus kennen? Was, wenn Sie ein Objekt möchten, das beliebig viele Eigenschaften haben kann, aber alle Werte einem bestimmten Typ folgen müssen?

Nun, es gibt zwei Möglichkeiten, dies zu erreichen. Die erste Möglichkeit ist die Verwendung eines `Record`, eines eingebauten generischen Utility-Typs in TypeScript.

Hier ist die grundlegende Syntax:

```ts
Record<Keys, ValueType>
```

Der erste Typ steht für den Typ der Schlüssel, während der zweite Typ den Typ der Werte darstellt.

Hier ist ein Beispiel für die Verwendung eines `Record` für ein `userRoles`-Objekt:

```ts
const userRoles: Record<string, string> = {
  admin: "full-access",
  editor: "limited-access",
  viewer: "read-only"
};
```

Dieses `userRoles`-Objekt kann beliebige String-Schlüssel haben, und alle Werte müssen Strings sein. Wenn Sie versuchen, einen Nicht-String-Wert zuzuweisen, erhalten Sie einen Fehler:

```ts
// Type 'number' is not assignable to type 'string'
const userRoles: Record<string, string> = {
  admin: 1
};
```

Die zweite Möglichkeit, dieselbe Idee auszudrücken, ist mit einer Index-Signatur:

```ts
const obj: { [key: string]: string } = {};
```

Die obige Syntax besagt, dass jede Eigenschaft mit einem String-Schlüssel einen String-Wert haben muss. Hier ist ein Beispiel für ein `settings`-Objekt:

```ts
const settings: { [key: string]: string } = {
  theme: "dark",
  language: "en",
  layout: "grid"
};
```

Wie zuvor zeigt TypeScript eine Fehlermeldung an, wenn die Werttypen nicht korrekt sind:

```ts
// Type 'boolean' is not assignable to type 'string'
const settings: { [key: string]: string } = {
  darkMode: true
};
```

Sowohl `Record` als auch Index-Signaturen sind im Grunde dasselbe. Die Wahl zwischen ihnen ist also meist eine Frage der Präferenz.
