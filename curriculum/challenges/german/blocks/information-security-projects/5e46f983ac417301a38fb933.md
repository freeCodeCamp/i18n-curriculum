# --instructions--

Passwörter sollten niemals im Klartext gespeichert werden. Sie sollten als Hashes gespeichert werden, falls die Passwortliste entdeckt wird. Allerdings sind nicht alle Hashes gleichwertig.

In diesem Projekt lernen Sie die Bedeutung guter Sicherheit kennen, indem Sie einen Passwort-Cracker erstellen, der Passwörter herausfindet, die mit SHA-1 gehasht wurden.

Erstellen Sie eine Funktion, die einen SHA-1-Hash eines Passworts entgegennimmt und das Passwort zurückgibt, wenn es zu den 10.000 meistgenutzten Passwörtern gehört. Wenn der SHA-1-Hash NICHT zu einem Passwort in der Datenbank gehört, geben Sie "PASSWORD NOT IN DATABASE" zurück.

Die Funktion soll jedes Passwort aus `top-10000-passwords.txt` hashen und mit dem übergebenen Hash vergleichen.

Die Funktion soll ein optionales zweites Argument namens `use_salts` annehmen. Wenn es auf true gesetzt ist, soll jeder Salt-String aus der Datei `known-salts.txt` vor und nach jedem Passwort aus `top-10000-passwords.txt` angehängt werden, bevor es gehasht und mit dem übergebenen Hash verglichen wird.

Hier sind einige gehashte Passwörter, mit denen Sie die Funktion testen können:

- `b305921a3723cd5d70a375cd21a61e60aabb84ec` sollte "sammy123" zurückgeben
- `c7ab388a5ebefbf4d550652f1eb4d833e5316e3e` sollte "abacab" zurückgeben
- `5baa61e4c9b93f3f0682250b6cf8331b7ee68fd8` sollte "password" zurückgeben

Hier sind einige gehashte Passwörter, mit denen Sie die Funktion testen können, wenn `use_salts` auf `True` gesetzt ist:

- `53d8b3dc9d39f0184144674e310185e41a87ffd5` sollte "superman" zurückgeben
- `da5a4e8cf89539e66097acd2f8af128acae2f8ae` sollte "q1w2e3r4t5" zurückgeben
- `ea3f62d498e3b98557f9f9cd0d905028b3b019e1` sollte "bubbles1" zurückgeben

Die `hashlib`-Bibliothek wurde für Sie importiert. Sie sollten in Erwägung ziehen, sie in Ihrem Code zu verwenden. <a href="https://docs.python.org/3/library/hashlib.html" target="_blank" rel="noopener noreferrer nofollow">Erfahren Sie hier mehr über "hashlib"</a>.
