# --description--

SQL significa Structured Query Language. Fue desarrollado en los años 70, cuando inicialmente se conocía como el lenguaje de consulta estructurado en inglés (SEQUEL). Este término fue luego abreviado a SQL.

SQL es un lenguaje de programación usado para almacenar y gestionar datos en bases de datos relacionales.

Como aprendiste en una lección anterior, las bases de datos relacionales organizan los datos en tablas. Cada tabla tiene un conjunto de filas y columnas, donde se representan los atributos de los datos y sus relaciones. Cada fila representa un registro mientras que cada columna representa un atributo o campo específico de ese registro.

Con SQL, puedes agregar, cambiar, eliminar, buscar y recuperar datos de una base de datos relacional. Proporciona un conjunto completo de comandos para consultar, filtrar, ordenar y agregar datos.

SQL puede ayudar a optimizar el rendimiento de la base de datos, consultando solo los datos necesarios, y puede integrarse fácilmente con una amplia gama de lenguajes de programación, para que puedas usarlo para interactuar con bases de datos directamente desde tus aplicaciones.

También se ha adoptado como un estándar de la industria. Garantiza la integridad de los datos e incluye muchas funciones de seguridad, como soporte para la autenticación de usuarios y el cifrado de datos.

También es escalable, portátil y compatible con una amplia gama de sistemas de gestión de bases de datos.

SQL se basa en comandos, comúnmente conocidos como sentencias SQL o consultas SQL.

Como desarrollador, escribirás estas sentencias usando elementos específicos del lenguaje SQL o palabras clave. Te permiten realizar las operaciones necesarias en tu base de datos.

Antes de que puedas ingresar comandos SQL, necesitas abrir la consola psql para que puedas interactuar directamente con la base de datos PostgreSQL. En el símbolo del sistema o terminal, ingresa:

```bash
psql -U <username> -d <database_name>
```

Reemplaza `username` con tu nombre de usuario y `database_name` con la base de datos a la que quieres conectarte. Si aún no has creado tu propia base de datos, usa `postgres`, que viene con la instalación.

Una vez conectado, verás que el prompt cambia a:

```bash
postgres=#
```

Muestra el nombre de la base de datos a la que estás conectado y espera comandos SQL.

Repasemos algunos de los comandos SQL más fundamentales con los que deberías estar familiarizado. Notarás que, por convención, los comandos SQL terminan con un punto y coma (`;`).

Primero, puedes usar este comando para crear una base de datos llamada `my_database`:

```sql
CREATE DATABASE my_database;
```

Puedes escribir `CREATE` seguido de `DATABASE`, y el nombre de la base de datos que quieres crear. Observa que las palabras clave SQL del comando están escritas en mayúsculas.

Luego, una vez que tengas tu base de datos, puedes conectarte a ella.

Cómo te conectas a la base de datos realmente depende del entorno y la herramienta que estés usando.

En el símbolo del sistema o terminal, puedes usar el método mostrado anteriormente. En la consola interactiva de psql, puedes cambiar de base de datos con el comando abreviado `\c` seguido del nombre de la base de datos así:

```bash
\c my_database
```

Ten en cuenta que el comando `\c` y otros comandos abreviados (los que tienen una barra invertida delante) son parte del shell psql y no forman parte del lenguaje SQL en sí, por lo que no requieren un punto y coma para completar el comando.

Ahora que te has conectado a tu nueva base de datos, verás que el prompt ha cambiado:

```bash
my_database=#
```

Una vez que hayas creado y conectado a una base de datos, puedes crear una tabla con las palabras clave `CREATE TABLE`:

```sql
CREATE TABLE products (
  id SERIAL,
  name VARCHAR(255)
);
```

En el comando, escribes `CREATE TABLE`, seguido del nombre de la tabla que quieres crear. En este caso, creará una tabla llamada `products`.

Luego, entre paréntesis, escribes los nombres de las columnas que debe tener la tabla y especificas el tipo de datos de los valores que se almacenarán en cada columna.

En el ejemplo, la tabla tendrá dos columnas, una columna para el ID del producto y otra para el nombre del producto.

La convención estándar de nombres para tablas y columnas es snake case, escribiendo las palabras en minúsculas y separándolas con un guion bajo (`_`). Por ejemplo: `delivery_orders`

Estos son algunos comandos SQL esenciales que debes conocer para comenzar.

Con SQL, puedes consultar, manipular y analizar datos fácilmente para tomar decisiones informadas y resolver problemas del mundo real.
