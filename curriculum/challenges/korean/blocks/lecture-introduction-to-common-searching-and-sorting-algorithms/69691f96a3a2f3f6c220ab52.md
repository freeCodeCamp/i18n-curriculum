# --description--

목록에서 항목을 검색하는 것은 컴퓨터 과학에서 흔한 일입니다. 검색과 관련해 알아야 할 두 가지 주요 알고리즘은 선형 검색과 이진 검색입니다.

선형 검색은 목록의 시작부터 시작해 찾고자 하는 대상 값을 찾을 때까지 각 항목을 반복합니다.

대상 값을 찾으면 목록에서 위치한 인덱스를 반환합니다. 대상 값을 찾지 못하면 `-1`를 반환합니다. 대부분의 프로그래밍 언어에서 유효한 인덱스가 아니기 때문에 `-1`를 반환합니다.

선형 검색의 코드는 다음과 같습니다:

```javascript
function linearSearch(arr, target) {
    for (let i = 0; i < arr.length; i++) {
        if (arr[i] === target) {
            return i;
        }
    }
    return -1;
}
```

검색할 목록이 `[13, 4, 7, 9, 10]`이고 대상 값이 `9`라면, 함수는 `3`를 반환합니다. 왜냐하면 `9`가 인덱스 `3`에 있기 때문입니다.

대상 값을 `5`로 바꾸면, 함수는 `-1`를 반환합니다. `5`가 목록에 없기 때문입니다.

이 알고리즘은 비교적 간단하지만 가장 효율적인 것은 아닙니다. 항목이 많은 목록에서는 선형 검색이 대상 값을 찾는 데 시간이 오래 걸릴 수 있습니다.

선형 검색의 시간 복잡도는 `O(n)`입니다. 목록 크기에 따라 검색 시간이 선형적으로 증가하기 때문입니다.

선형 검색의 공간 복잡도는 `O(1)`입니다. 목록을 검색하는 데 추가 공간이 필요하지 않기 때문입니다.

이진 검색은 많은 항목이 있는 목록을 검색할 때 더 효율적인 알고리즘입니다. 단, 목록이 오름차순으로 정렬되어 있어야 합니다.

이진 검색은 목록을 반으로 나누고 대상 값이 목록 중간에 있는지 확인하는 방식으로 작동합니다. 대상 값이 중간에 있으면 대상 값의 인덱스를 반환합니다. 그렇지 않으면 대상 값이 목록의 왼쪽 절반에 있는지 오른쪽 절반에 있는지 확인합니다.

대상 값을 찾을 때까지 목록의 남은 부분을 계속 반으로 나눕니다. 대상 값이 목록에 없으면 `-1`를 반환합니다.

이진 검색의 코드는 다음과 같습니다:

```javascript
function binarySearch(arr, target) {
    let low = 0;
    let high = arr.length - 1;

    while (low <= high) {
        const mid = Math.floor((low + high) / 2);

        if (arr[mid] === target) {
            return mid;
        } else if (arr[mid] < target) {
            low = mid + 1;
        } else {
            high = mid - 1;
        }
    }

    return -1;
}
```

먼저 `low` 인덱스와 `high` 인덱스를 지정합니다. 이는 검색할 목록 범위를 나타냅니다.

그다음 `low`가 `high`보다 작거나 같은지 조건을 확인합니다. `low`가 `high`보다 크면 목록 전체를 검색한 것이고 대상 값을 찾지 못한 것입니다. 이 경우 검색을 중단하고 `-1`를 반환합니다.

`low` 인덱스가 `high` 인덱스보다 작거나 같으면 목록의 중간 인덱스인 `mid`를 계산합니다. 그리고 대상 값이 중간 인덱스에 있는지 확인합니다. 있으면 중간 인덱스를 반환합니다.

그렇지 않으면 중간값이 대상 값보다 작은지 확인합니다. 작으면 낮은 인덱스를 중간 인덱스에 1을 더한 값으로 업데이트합니다. 즉, 목록의 오른쪽 절반을 검색합니다.

마지막으로 다른 조건이 모두 `True`이면 `high` 인덱스를 중간 인덱스에서 1을 뺀 값으로 업데이트합니다. 즉, 목록의 왼쪽 절반을 검색합니다.

이 과정을 대상 값을 찾거나 목록에 없다고 판단할 때까지 반복합니다.

이진 검색의 시간 복잡도는 `O(log n)`입니다. 목록 크기에 따라 검색 시간이 로그 함수 형태로 증가하기 때문입니다.

이진 검색의 공간 복잡도는 `O(1)`입니다. 목록을 검색하는 데 추가 공간이 필요하지 않기 때문입니다.

이진 검색과 선형 검색은 컴퓨터 과학에서 마주칠 다양한 문제에 사용할 수 있습니다. 두 알고리즘의 차이점과 각각 언제 사용하는지 이해하는 것이 중요합니다.
