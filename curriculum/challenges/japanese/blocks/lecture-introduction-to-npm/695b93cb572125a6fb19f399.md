# --description--

npm はプロジェクトに関する重要な情報、たとえば使用しているパッケージやそのバージョン、実行できるコマンドなどを保存する方法が必要です。これを `package.json` ファイルで行います。

`package.json` ファイルはプロジェクトの npm 設定ファイルのようなものだと考えてください。これはシンプルな JSON ファイルで、キーと値のペアを使って次のような情報を管理します：

* プロジェクトの名前とバージョン
    
* 作者
    
* 最初に実行すべきメインファイル
    
* ライセンス
    
* コードリポジトリ、キーワード、説明
    
* プロジェクトが必要とする依存関係
    

`package.json` ファイルの例は次のようになります：

```json
{
  "name": "my-app",
  "version": "1.0.0",
  "description": "A simple NPM project",
  "main": "index.js",
  "scripts": {
    "start": "node index.js",
    "test": "echo \"Running tests...\""
  },
  "repository": {
    "type": "git",
    "url": "git+https://github.com/username/project.git"
  },
  "keywords": ["fCC", "js"],
  "author": "Kolade Chris",
  "license": "ISC",
  "dependencies": {
    "react": "^19.2.1"
  },
  "devDependencies": {
    "cypress": "^15.7.1"
  }
}
```

`main` フィールドはよく `index.js` に設定され、Node.js にどのファイルを最初に実行すべきかを伝えます。

`license` フィールドは、他の人があなたのパッケージのコードを使用、変更、配布する際の法的条件を指定するために使われます。`MIT`、`ISC`、`GPL` のようにさまざまなライセンスがあり、それぞれ異なる条件と許可があります。デフォルトは `ISC`（Internet Systems Consortium License）で、とても寛容なオープンソースライセンスです。

プロジェクトが成長しパッケージをインストールすると、npm はそれらを次の2つのセクションのいずれかに追加します：

* `dependencies` はアプリが本番環境で動作するために必要なパッケージ
    
* `devDependencies` は開発中にのみ必要なパッケージ
    

たとえば、プロジェクトが UI に React を使う場合、それは `dependencies` に入ります。これはプロジェクトがデプロイ後に動作するために必要です。Cypress のようなツールは、本番環境にデプロイする前のエンドツーエンドテストにのみ使われるため、`devDependencies` に入ります。

`package.json` ファイルを作成する簡単な方法がいくつかあります。最も一般的なのは次のコマンドを実行することです：

```bash
npm init
```

これはプロジェクトのルートフォルダーで実行してください。このコマンドは対話形式のセットアップを開始し、npm が一連の質問をします。回答はプロジェクトの名前、バージョン、説明、スクリプト、キーワード、作者、ライセンスなどのフィールドに使われます。

質問をスキップしてデフォルト値で `package.json` ファイルを作成したい場合は、次のコマンドを実行できます：

```bash
npm init -y
```

または

```bash
npm init --yes
```

これで多くのフィールドにデフォルト値が入った `package.json` ファイルが即座に作成されます。

通常は `npm init` を `-y` や `--yes` フラグなしで使い、各フィールドを確認してプロジェクト情報をカスタマイズすることをお勧めします。質問に答えなかった場合、npm は単にデフォルト値を使います。たとえば、ライセンスの質問を空欄にすると、npm はデフォルトで `ISC` に設定します。

`dependencies` と `devDependencies` がまだリストにないことに気づくかもしれません。これはこれらのセクションが最初のパッケージをインストールした後にのみ追加されるためです。

また、`repository` フィールドが欠けていることにも気づくでしょう。npm は `npm init` の際にこれについて尋ねますが、その後自動的に埋めることはありません。たとえば、`git remote add origin` や `git push` のようなコマンドを実行しても、npm はリポジトリフィールドを更新しません。含めたい場合は手動で追加する必要があります。
