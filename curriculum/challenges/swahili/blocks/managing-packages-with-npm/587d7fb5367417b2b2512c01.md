# --description--

`Versions` ya vifurushi vya npm katika sehemu ya utegemezi ya faili yako ya package.json hufuata kile kinachoitwa utoaji wa toleo la kimaelezo (Semantic Versioning, SemVer), kiwango cha tasnia kwa utoaji wa matoleo ya programu kinacholenga kurahisisha usimamizi wa utegemezi. Maktaba, mifumo ya kazi au zana nyingine zilizochapishwa kwenye npm zinapaswa kutumia SemVer ili kuwasilisha wazi aina ya mabadiliko ambayo miradi inaweza kutarajia ikiwa itasasisha.

Kujua SemVer kunaweza kuwa na manufaa unapotengeneza programu inayotumia utegemezi wa nje (ambayo karibu kila mara hutumia). Siku moja, uelewa wako wa nambari hizi utakukuokoa usije kwa bahati mbaya ukaleta mabadiliko yanayovunja mradi wako bila kuelewa kwa nini mambo yaliyofanya kazi jana ghafla hayafanyi kazi leo. Hivi ndivyo utoaji wa toleo la kimaelezo unavyofanya kazi kulingana na tovuti rasmi:

```json
"package": "MAJOR.MINOR.PATCH"
```

Toleo LA KUBWA linapaswa kuongezeka unapofanya mabadiliko ya API yasiyolingana. Toleo LA NDOGO linapaswa kuongezeka unapoongeza kipengele kwa njia inayolingana na matoleo ya awali. Toleo LA KIDOGO linapaswa kuongezeka unapofanya marekebisho ya hitilafu yanayolingana na matoleo ya awali. Hii inamaanisha kuwa matoleo ya LA KIDOGO ni marekebisho ya hitilafu na matoleo ya LA NDOGO huongeza vipengele vipya lakini hakuna kati yao vinavyovunja kile kilichofanya kazi awali. Mwishowe, matoleo ya LA KUBWA huongeza mabadiliko ambayo hayatakuwa yanayolingana na matoleo ya awali.

# --instructions--

Katika sehemu ya utegemezi ya faili yako ya `package.json`, badilisha toleo la `@freecodecamp/example` ili lilingane na toleo LA KUBWA 1, toleo LA NDOGO 2 na toleo LA KIDOGO 13

# --hints--

`"dependencies"` inapaswa kujumuisha `"@freecodecamp/example"`.

```js
fetch(code + '/_api/package.json')
  .then(response => response.json())
  .then(
    data => {
      assert.property(
        data.dependencies,
        '@freecodecamp/example',
        '"dependencies" does not include "@freecodecamp/example"'
      );
    },
    error => {
      throw new Error(error.message || error.responseText);
    }
  );
```

Toleo la `"@freecodecamp/example"` linapaswa kuwa `"1.2.13"`.

```js
fetch(code + '/_api/package.json')
  .then(response => response.json())
  .then(
    data => {
      assert.equal(
        data.dependencies['@freecodecamp/example'],
        '1.2.13',
        'Wrong version of "@freecodecamp/example". It should be 1.2.13'
      );
    },
    error => {
      throw new Error(error.message || error.responseText);
    }
  );
```
