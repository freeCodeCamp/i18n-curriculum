# --description--

체감 성능은 단순히 사이트가 실제로 얼마나 빠른지가 아니라, 사용자에게 얼마나 빠르게 느껴지는지에 관한 것입니다. 예를 들어, 실제로는 느리지 않은데 느리게 느껴졌던 웹사이트를 사용해본 적 있나요? 그것이 바로 체감 성능입니다 — 사이트가 사용자에게 얼마나 빠르게 느껴지는지입니다.

작은 조정만으로도 사용자 경험에 큰 차이를 만들 수 있습니다. 체감 성능을 향상하는 실용적인 방법들을 살펴보겠습니다!

스피너는 사용자가 실제보다 더 오래 기다리는 것처럼 느끼게 만듭니다. 대신, 실제 콘텐츠를 흉내 내면서 점차 로드되는 회색 자리 표시자인 스켈레톤 스크린을 사용해보세요:

```html
<div class="skeleton"></div>
<script>
  setTimeout(() => {
    document.querySelector(".skeleton").innerHTML = "✅ Content Loaded!";
  }, 2000);
</script>
```

사용자는 어둠 속에서 기다리는 것이 아니라 페이지가 점진적으로 로드되는 것처럼 느낍니다.

이와 더불어, 사용자가 상호작용을 시작하기 전에 모든 것이 로드될 때까지 기다릴 필요가 없어야 합니다. 중요한 콘텐츠를 먼저 로드하고 나머지는 백그라운드에서 로드하세요. 이전에 지연 로딩으로 이 부분을 살펴봤습니다:

```html
<img src="placeholder.jpg" data-src="real-image.jpg" class="lazy">
<script>
  document.addEventListener("DOMContentLoaded", function () {
    const lazyImages = document.querySelectorAll("img.lazy");
    lazyImages.forEach(img => {
      img.src = img.dataset.src;
    });
  });
</script>
```

이렇게 하면 모든 것이 로드되지 않아도 페이지가 반응형으로 느껴집니다.

그렇다면, 사용자가 다음에 무엇을 클릭할지 사이트가 예측할 수 있다면 어떨까요? 프리로딩을 사용하면 사용자가 요청하기 전에 다음 페이지를 미리 가져올 수 있습니다:

```html
<link rel="preload" href="next-page.html" as="document">
```

이렇게 하면 사용자가 이동할 때 체감 로딩 시간이 거의 0에 가깝게 줄어듭니다.

마지막으로, 사용자는 불확실함을 싫어합니다. 버튼을 클릭하면 즉시 그 행동을 인지시켜주세요:

```html
<button onclick="this.innerText='⏳ Processing...'; setTimeout(() => this.innerText='✅ Done!', 2000);">
  Click Me
</button>
```

이렇게 하면 지연이 있더라도 사용자는 무언가 진행되고 있다고 느낍니다.

결론적으로, 체감 성능을 향상하는 것은 단순한 속도 향상이 아니라, 속도가 사용자에게 어떻게 느껴지는가에 관한 것입니다. 스켈레톤 스크린을 사용하고, 중요한 콘텐츠를 우선순위로 두며, 스마트하게 프리로딩하고, 즉각적인 피드백을 제공해 사이트가 더 빠르게 느껴지도록 만드세요.
