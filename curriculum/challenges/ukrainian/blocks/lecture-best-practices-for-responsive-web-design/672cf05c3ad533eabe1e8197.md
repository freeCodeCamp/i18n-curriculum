# --interactive--

Медіазапити — це фундаментальний компонент адаптивного вебдизайну, який дозволяє розробникам застосовувати різні стилі залежно від характеристик пристрою або браузера користувача. Вони дають змогу налаштовувати відображення вмісту для різних пристроїв без зміни самого вмісту.

У своїй основі медіазапит складається з типу медіа та одного або кількох виразів, які перевіряють певні умови. Якщо ці умови виконуються, застосовуються відповідні стилі. Цей механізм дозволяє створювати адаптивні дизайни, які підлаштовуються під різні розміри екранів, роздільність і можливості пристроїв.

Базовий синтаксис медіазапиту в CSS виглядає так:

```css
@media mediatype and (feature: value) {
  /* CSS rules go here */
}
```

У цій структурі mediatype вказує тип медіа, до якого застосовується запит, а пара feature: value визначає умову, яка має бути виконана для застосування стилів.

Типи медіа описують загальну категорію пристрою. Розглянемо найпоширеніші типи медіа:

- `all` підходить для всіх пристроїв. Це значення за замовчуванням, якщо тип медіа не вказано.

- `print` призначений для сторінок і документів, які переглядаються на екрані в режимі попереднього перегляду друку.

- `screen` призначений переважно для екранів.

Раніше існувало більше типів медіа, наприклад `handheld` і `tv`, але більшість із них застаріли на користь використання функцій для точнішого націлювання пристроїв.

Функції медіа описують конкретні характеристики користувацького агента, пристрою виводу або середовища. Розглянемо деякі з найпоширеніших функцій медіа. `width` і `height` відповідають за ширину та висоту вікна перегляду, і часто використовуються з префіксами `min-` або `max-` для діапазонних запитів. Ось приклад:

```css
@media screen and (min-width: 768px) {
  /* Styles for screens at least 768px wide */
}
```

`aspect-ratio` описує співвідношення між шириною та висотою вікна перегляду. Ось приклад:

```css
@media screen and (aspect-ratio: 16/9) {
  /* Styles for screens with a 16:9 aspect ratio */
}
```

Функція `orientation` вказує, чи пристрій знаходиться в альбомній чи книжковій орієнтації. Ось приклад:

```css
@media screen and (orientation: landscape) {
  /* Styles for landscape orientation */
}
```

Функція `resolution` описує роздільність пристрою виводу в точках на дюйм (dpi) або точках на сантиметр (dpcm). Ось приклад:

```css
@media screen and (min-resolution: 300dpi) {
  /* Styles for high-resolution screens */
}
```

Функція `hover` перевіряє, чи може основний механізм введення наводити курсор на елементи. Ось приклад:

```css
@media (hover: hover) {
  /* Styles for devices that support hover */
}
```

Функція `prefers-color-scheme` визначає, чи користувач обрав світлу чи темну кольорову тему. Ось приклад:

```css
@media (prefers-color-scheme: dark) {
  /* Styles for dark mode */
}
```

Медіазапити також можуть комбінувати кілька умов за допомогою логічних операторів. Оператор `and` використовується для поєднання кількох функцій медіа, тоді як `not` і `only` можуть застосовуватися для заперечення або ізоляції медіазапитів. Ось приклад, який поєднує кілька функцій:

```css
@media screen and (min-width: 768px) and (orientation: landscape) {
  /* Styles for landscape screens at least 768px wide */
}
```

Також можна націлюватися на кілька запитів у списку, розділеному комами, що працює як оператор "або":

```css
@media screen and (min-width: 768px), print {
  /* Styles for screens at least 768px wide OR for print */
}
```

Працюючи з медіазапитами, важливо враховувати каскадність. Медіазапити не збільшують специфічність — вони лише групують умовні правила. Звичайні правила каскаду CSS все ще застосовуються в межах кожного медіазапиту.

На практиці медіазапити часто використовують для створення адаптивних макетів. Загальний шаблон — визначити базовий стиль для мобільних пристроїв, а потім за допомогою медіазапитів покращувати макет для більших екранів:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<div class="container">
  <p>Lorem ipsum dolor sit amet, consectetur adipiscing elit. Integer nec odio. Praesent libero. Sed cursus ante dapibus diam.</p>
  <p>Sed nisi. Nulla quis sem at nibh elementum imperdiet. Duis sagittis ipsum. Praesent mauris. Fusce nec tellus sed augue semper porta.</p>
  <p>Mauris massa. Vestibulum lacinia arcu eget nulla. Class aptent taciti sociosqu ad litora torquent per conubia nostra, per inceptos himenaeos.</p>
</div>
```

```css
/* Base styles for mobile */
.container {
  width: 100%;
  padding: 15px;
}

/* Styles for tablets */
@media screen and (min-width: 768px) {
  .container {
    width: 750px;
    margin: 0 auto;
  }
}

/* Styles for desktops */
@media screen and (min-width: 1024px) {
  .container {
    width: 960px;
  }
}
```

:::

Такий підхід, відомий як адаптивний дизайн "mobile-first", гарантує, що базові стилі підходять для мобільних пристроїв, а покращення додаються для більших екранів.

Підсумовуючи, медіазапити — це потужний інструмент у CSS, який дозволяє створювати адаптивні, гнучкі вебдизайни. Розуміючи, як використовувати різні типи медіа та функції, розробники можуть створювати сайти, які забезпечують оптимальний користувацький досвід на широкому спектрі пристроїв і вподобань. Оскільки вебтехнології продовжують розвиватися, оновлення знань про нові функції медіа допоможе створювати більш тонкі та зручні адаптивні дизайни.
