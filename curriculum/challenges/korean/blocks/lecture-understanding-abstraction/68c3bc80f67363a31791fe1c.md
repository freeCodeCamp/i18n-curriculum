# --description--

캡슐화, 상속, 다형성에 대해 살펴봤으니, 이제 객체지향 프로그래밍의 다음 핵심 개념인 추상화에 대해 이야기해 보겠습니다.

추상화는 복잡한 구현 세부사항을 숨기고 객체나 시스템의 필수 기능만 보여주는 과정입니다. 어떻게 작동하는지보다는 무엇을 하는지에 집중하는 것이라고 생각하세요.

추상화는 Python에만 국한된 개념이 아닙니다. 객체지향 프로그래밍을 지원하는 여러 언어에서 구현할 수 있는 프로그래밍 개념입니다.

추상화를 설명하기 위해 자동차를 운전하는 상황을 상상해 보세요. 운전석에 앉으면 핸들, 변속기, 가속 및 브레이크 페달 같은 필수 부품과 상호작용합니다. 엔진이 어떻게 작동하는지, 변속기가 어떻게 기어를 바꾸는지, 브레이크 시스템의 물리학적 원리를 알 필요가 없습니다. 이 모든 것은 복잡한 구현 세부사항이기 때문입니다.

이것이 바로 추상화가 작동하는 방식입니다! 복잡한 시스템과 상호작용할 수 있는 단순화된 인터페이스를 제공합니다.

자동차의 경우, 단순화된 인터페이스는 핸들, 브레이크, 가속 페달이고, 복잡한 시스템은 자동차 자체입니다.

Python에서 추상화를 구현하는 방법은 `abc` 모듈을 통해서입니다.

이 모듈은 `ABC` 클래스(“abstract base class”의 약자)와 `@abstractmethod` 데코레이터를 제공합니다.

`ABC`는 상속을 위해 만들어진 클래스이지만, 직접 객체를 생성할 수는 없습니다. 이 클래스는 하위 클래스가 반드시 구현해야 하는 메서드와 속성의 공통 인터페이스를 정의합니다.

반면, 추상 메서드는 `@abstractmethod` 데코레이터를 사용해 추상 기본 클래스(ABC)에서 선언된 메서드입니다. 구현이 없거나 기본 구현만 있을 수 있습니다. 하지만 하위 클래스가 이를 재정의해야만 구체적이고 인스턴스화할 수 있습니다.

Python에서 추상 클래스의 기본 구문은 다음과 같습니다:

```py
from abc import ABC, abstractmethod

# Define an abstract base class
class AbstractClass(ABC):
    @abstractmethod
    def abstract_method(self):
        pass

# Concrete subclass that implements the abstract method
class ConcreteClassOne(AbstractClass):
    def abstract_method(self):
        print('Implementation in ConcreteClassOne')

# Another concrete subclass
class ConcreteClassTwo(AbstractClass):
    def abstract_method(self):
        print('Implementation in ConcreteClassTwo')
```

기본 예시는 다음과 같습니다:

```py
from abc import ABC, abstractmethod

class Animal(ABC): # Inherits from abstract base class
   @abstractmethod # Abstract method decorator
   def make_sound(self):  # The method subclasses must override
       pass

# Concrete class that will override the abstract method
class Dog(Animal):
   def make_sound(self):
       print('Woof!')

# Another concrete class that will override the abstract method
class Cat(Animal):
   def make_sound(self):
       print('Meow!')

# Another concrete class that will override the abstract method
class Monkey(Animal):
   def make_sound(self):
       print('Ooh ooh aah aah!')

# Create instances of each concrete class
animals = [Dog(), Cat(), Monkey()]

# Loop through the instances to call the make_sound method
for animal in animals:
   animal.make_sound()

# Output:
# Woof!
# Meow!
# Ooh ooh aah aah!
```

이 예시에서:

- `ABC` 모듈에서 `abstractmethod` 클래스와 `abc`를 가져옵니다.
- `Animal`를 상속하는 `ABC` 클래스를 만들고, `make_sound`의 각 하위 클래스가 반드시 재정의해야 하는 추상 메서드 `Animal`를 선언합니다.
- `Dog`, `Cat`, `Monkey`라는 구체 클래스를 만들고, 이들은 `make_sound` 추상 메서드를 반드시 재정의해야 합니다.
- 구체 클래스의 인스턴스를 생성하고 `make_sound` 메서드를 호출해 각 클래스가 `make_sound` 추상 메서드를 자신만의 방식으로 구현하는 모습을 보여줍니다.

Animal 클래스의 인스턴스를 생성할 수 없다는 점을 기억하세요. 시도하면 다음과 같은 오류가 발생합니다:

```py
dog = Animal() 
# TypeError: Can't instantiate abstract class Animal 
# without an implementation for abstract method 'make_sound'
```

추상 메서드에 대한 구현을 제공하지 않는 하위 클래스에도 같은 규칙이 적용됩니다. Animal의 하위 클래스를 정의해도 make_sound를 재정의하지 않으면 인스턴스화할 수 없습니다. 다음 예시를 참고하세요:

```py
class Bird(Animal):
    pass

bird = Bird()
# TypeError: Can't instantiate abstract class Bird 
# without an implementation for abstract method 'make_sound'
```

이번에는 구체 메서드 인스턴스에 전달할 수 있는 인스턴스 속성을 포함한 또 다른 예시입니다:

```py
from abc import ABC, abstractmethod

# The blueprint for any toy that can speak
class TalkingToy(ABC):
   def __init__(self, name):
       self.name = name
   @abstractmethod
   def speak(self):
       pass

class RobotToy(TalkingToy):
   def speak(self):
       print(f'{self.name} says beep boop! I am a robot!')

class TeddyBearToy(TalkingToy):
   def speak(self):
       print(f"{self.name} says hug me! I'm cuddly!")

class DinosaurToy(TalkingToy):
   def speak(self):
       print(f'{self.name} says ROOOOAR!')

# Create toys
rusty = RobotToy('Rusty')
fluffy = TeddyBearToy('Fluffy')
rex = DinosaurToy('Rex')

toys = [rusty, fluffy, rex]
for toy in toys:
   toy.speak()

# Output:
# Rusty says beep boop! I am a robot!
# Fluffy says hug me! I'm cuddly!
# Rex says ROOOOAR!
```

이 예시에서:

- 말을 할 수 있는 장난감의 청사진을 정의하는 추상 기본 클래스 `TalkingToy`가 있습니다.
- 하위 클래스 `RobotToy`, `TeddyBearToy`, `DinosaurToy`는 `speak` 메서드를 자신만의 방식으로 구현합니다.
- 이 하위 클래스의 인스턴스를 만들고 `speak` 메서드를 호출하면, 각 장난감이 고유한 방식으로 말을 합니다.

결론적으로, Python의 추상화는 재사용성을 높여 복잡한 시스템을 단순화합니다.

추상 클래스의 단일 메서드를 여러 하위 클래스에서 재사용하면서 각 하위 클래스가 고유한 동작을 제공하도록 강제하는 방법을 보았습니다.

이 접근법은 애플리케이션이 커질수록 코드를 체계적이고 유연하며 유지보수하기 쉽게 만듭니다.
