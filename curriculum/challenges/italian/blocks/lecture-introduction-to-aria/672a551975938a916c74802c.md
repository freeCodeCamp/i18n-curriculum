# --interactive--

L'attributo `aria-describedby` serve a fornire informazioni aggiuntive su un elemento agli utenti di lettori di schermo facendo riferimento a contenuti già presenti nella pagina. Crea un’associazione programmata tra l’elemento e il contenuto (tecnicamente chiamato descrizione accessibile), che i lettori di schermo possono usare per informare gli utenti delle informazioni aggiuntive quando interagiscono con l’elemento.

L’uso più comune di `aria-describedby` è associare istruzioni e messaggi di errore ai campi di input dei moduli. A causa dei vari metodi con cui gli utenti di lettori di schermo navigano una pagina, potrebbero non sentire questi messaggi passando da un campo all’altro. Usare `aria-describedby` aiuta a garantire che li ascoltino.

Diamo un’occhiata a qualche esempio per capire come funziona. In questo primo esempio, abbiamo un elemento `form` che accetta una password.

Attiva l’editor interattivo e digita qualche carattere nel campo di input della password. Vedrai che la password è mascherata nella finestra di anteprima. Dovresti anche vedere che il testo `password-help` rimane rosso finché non inserisci 8 o più caratteri nel campo.

NOTA: Questo esempio interattivo usa CSS e JavaScript per aggiornare dinamicamente il colore del testo `password-help`. Non preoccuparti di capire il codice JavaScript perché lo imparerai nei moduli futuri.

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<form>
  <label for="password">Password:</label>
  <input type="password" id="password" aria-describedby="password-help" />
  <p id="password-help">Your password must be at least 8 characters long.</p>
</form>

<script src="index.js"></script>
```

```css
#password-help {
  color: red;
}
```

```js
const passwordEl = document.getElementById("password");
const passwordHelpText = document.getElementById("password-help");

passwordEl.addEventListener("input", (e) => { 
  const userInput = e.target.value;
  passwordHelpText.style.color = userInput.length >= 8 ? "green" : "red";
});
```

:::

Stiamo usando un elemento `label` per il testo `Password` e lo associamo al campo `input` della password.

Abbiamo anche un elemento paragrafo che descrive i requisiti della password. Usiamo l’attributo `aria-describedby` per associare il campo `input` della password ai requisiti indicati nel paragrafo. Quando un utente di lettore di schermo interagisce con questo `input`, il suo lettore annuncerà il nome del `input` (`Password`) e potrebbe anche annunciare i requisiti della password. Tuttavia, non è una garanzia assoluta perché alcuni lettori di schermo potrebbero non annunciare automaticamente il contenuto aggiuntivo o farlo solo in circostanze specifiche. Questo è normale. Ogni lettore di schermo è diverso e gestisce gli attributi ARIA a modo suo. Questo non sminuisce l’uso di `aria-describedby`, che comunque aiuterà gli utenti di lettori di schermo in generale.

Un altro buon caso d’uso per l’attributo `aria-describedby` è quando hai un pulsante di eliminazione `button`. Ecco un esempio di pulsante di eliminazione `button` seguito da un messaggio che descrive cosa succederà quando il pulsante viene cliccato:

:::interactive_editor

```html
<button aria-describedby="delete-message">Delete</button>

<p id="delete-message">Warning! All deletions are permanent.</p>
```

:::

Proprio come nell’esempio precedente, associamo il pulsante di eliminazione al messaggio usando l’attributo `aria-describedby`. Il valore `id` e il valore dell’attributo `aria-describedby` devono corrispondere.

L’attributo `aria-describedby` è un attributo potente che può aiutare a garantire che informazioni aggiuntive su un elemento vengano fornite agli utenti di lettori di schermo quando interagiscono con l’elemento. Viene usato più comunemente per associare istruzioni e messaggi di errore ai campi di input dei moduli, per ridurre la possibilità che gli utenti di lettori di schermo perdano questi messaggi mentre navigano nel modulo.
