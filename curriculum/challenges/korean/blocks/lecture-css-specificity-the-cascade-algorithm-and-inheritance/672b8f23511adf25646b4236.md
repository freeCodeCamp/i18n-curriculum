# --interactive--

캐스케이드 알고리즘은 브라우저가 동일한 요소를 대상으로 하는 여러 CSS 규칙 중 어떤 규칙을 적용할지 결정하는 과정입니다. 잘 정의된 규칙 집합을 바탕으로 가장 적절한 스타일이 사용되도록 보장합니다.

이 과정은 **관련성**에서 시작합니다. 브라우저는 먼저 해당 요소에 실제로 적용되는 CSS 규칙만 필터링합니다. 여기에는 선택자 일치와 적용 중일 수 있는 미디어 쿼리 고려가 포함됩니다.

미디어 쿼리는 장치나 뷰포트의 너비, 높이, 방향 같은 특성에 따라 스타일을 적용하는 CSS 기법입니다.

다음으로 알고리즘은 **출처와 중요도**를 고려합니다. CSS는 브라우저 기본 스타일(사용자 에이전트), 사용자 설정 스타일, 작성자(사용자)가 작성한 스타일 등 다양한 출처에서 올 수 있습니다.

출처를 고려한 후, 알고리즘은 각 규칙의 중요도를 평가하며 `!important`로 표시된 규칙에 우선순위를 부여해 출처와 상관없이 다른 규칙을 재정의합니다.

출처와 중요도로 필터링한 뒤, 알고리즘은 **특이성**을 살펴봅니다. 동일한 출처와 중요도 수준의 두 규칙이 적용될 때는 특이성이 더 높은 규칙이 적용됩니다.

특이성은 선택자가 얼마나 구체적인지를 나타내는 척도로, 더 구체적인 선택자가 더 일반적인 선택자보다 우선합니다.

마지막으로, 모든 조건이 같으면 **출현 순서**가 작용합니다. 특이성이 같은 두 규칙 중 CSS에서 나중에 나타난 규칙이 적용됩니다.

이 때문에 스타일을 작성하는 순서가 결과에 영향을 줄 수 있습니다.

예제를 살펴보겠습니다.

HTML 파일에는 단일 문단 요소가 있습니다. CSS에는 문단 요소를 대상으로 하는 두 개의 규칙이 있습니다.

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<p>example paragraph</p>
```

```css
p {
  color: blue;
}

p {
  color: green; 
}
```

:::

첫 번째 규칙은 모든 문단 요소의 텍스트 색상을 파란색으로 설정하고, 두 번째 규칙은 모든 문단 요소의 텍스트 색상을 초록색으로 설정합니다.

그렇다면 어떤 색상이 적용될까요? 초록색이 문단 요소에 적용됩니다.

관련성, 출처와 중요도, 특이성, 범위, 출현 순서를 고려해 캐스케이드 알고리즘은 CSS가 예측 가능하게 동작하도록 하여 더 복잡하고 세밀한 웹 페이지 디자인이 가능하게 합니다.
