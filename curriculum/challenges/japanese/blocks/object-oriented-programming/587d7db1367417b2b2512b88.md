# --description--

前のレッスンでは、オブジェクトが `prototype` オブジェクトを参照することで、別のオブジェクトから動作（メソッド）を継承できることを学びました。

```js
ChildObject.prototype = Object.create(ParentObject.prototype);
```

そして `ChildObject` は `prototype` にメソッドをチェーンすることで、自身のメソッドを持つようになりました。

```js
ChildObject.prototype.methodName = function() {...};
```

継承されたメソッドをオーバーライドすることも可能です。やり方は同じで、オーバーライドしたいメソッド名と同じ名前のメソッドを `ChildObject.prototype` に追加します。以下は `Bird` が `eat()` から継承した `Animal` メソッドをオーバーライドしている例です。

```js
function Animal() { }
Animal.prototype.eat = function() {
  return "nom nom nom";
};
function Bird() { }

Bird.prototype = Object.create(Animal.prototype);

Bird.prototype.eat = function() {
  return "peck peck peck";
};
```

もし `let duck = new Bird();` のインスタンスがあり `duck.eat()` を呼び出した場合、JavaScript は `prototype` の `duck` チェーン上で次のようにメソッドを検索します。

1.  `duck` => ここに `eat()` は定義されていますか？いいえ。
2.  `Bird` => ここに `eat()` は定義されていますか？はい。実行して検索を停止します。
3.  `Animal` => `eat()` も定義されていますが、JavaScript はこのレベルに到達する前に検索を停止しました。
4.  Object => JavaScript はこのレベルに到達する前に検索を停止しました。

# --instructions--

`fly()` の `Penguin` メソッドをオーバーライドして、文字列 `Alas, this is a flightless bird.` を返すようにしてください。

`bird.fly()` メソッドは文字列 `I am flying!` を返すべきです。
