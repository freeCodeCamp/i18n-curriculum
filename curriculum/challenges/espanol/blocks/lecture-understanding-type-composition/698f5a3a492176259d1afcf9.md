# --description--

En una lección anterior, aprendiste cómo trabajar con alias de tipo y viste ejemplos de cómo nombrar tipos de objetos. 

Como recuerdas, una forma de nombrar un tipo de objeto sería usar la palabra clave `type` así:

```ts
type ID = number | string;

type User = {
  id: ID;
  name: string;
};
```

Otra forma de nombrar un tipo de objeto sería usar una declaración `interface` como esta:

```ts
type ID = number | string;

interface User {
  id: ID;
  name: string;
}
```

`type` e `interface` son similares, pero una diferencia clave es que las interfaces usan la palabra clave `extends` para construir sobre otras interfaces, mientras que los alias de `type` usan tipos de intersección (`&`) para combinar tipos.

Veamos algunos ejemplos:

```ts
interface User {
  id: number | string;
  name: string;
}

interface Admin extends User {
  role: string;
}

const adminUser: Admin = {
  id: 101,
  name: "Alice",
  role: "superadmin"
};
```

En este ejemplo, `User` comienza con `id` y `name`. La interfaz `Admin` extiende `User`, lo que significa que incluye `id` y `name` y añade una nueva propiedad llamada `role`.

Si quisieras extender un `type`, usarías un tipo de intersección (`&`) así:

```ts
type User = {
  id: number | string;
  name: string;
};

type Admin = User & {
  role: string;
};

const adminUser: Admin = {
  id: 101,
  name: "Alice",
  role: "superadmin"
};
```

`Admin` combina el tipo `User` con un nuevo tipo de objeto que incluye `role`. El resultado tiene la misma estructura que el ejemplo de la interfaz, que es un objeto con `id`, `name` y `role`.

Tanto `type` como `interface` pueden describir la forma de un objeto. Si no estás seguro de cuál usar, recuerda que `interface` está diseñado específicamente para formas de objetos, mientras que `type` es más flexible y también puede representar tipos unión, primitivos y combinaciones de tipos más complejas.
