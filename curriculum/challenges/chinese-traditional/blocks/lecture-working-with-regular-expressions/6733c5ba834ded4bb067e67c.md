# --interactive--

修飾詞，通常稱為「旗標」，會修改正規表達式的行為。讓我們回想一下先前課程中的範例：

:::interactive_editor

```js
const regex = /freeCodeCamp/;
console.log(regex.test("freeCodeCamp")); // true
console.log(regex.test("freeCodeCamp is great")); // true
console.log(regex.test("I love freeCodeCamp")); // true
console.log(regex.test("freecodecamp")); // false
console.log(regex.test("FREECODECAMP")); // false
console.log(regex.test("free")); // false
console.log(regex.test("code")); // false
console.log(regex.test("camp")); // false
```

:::

如果你還記得，全部小寫和全部大寫的 `freeCodeCamp` 字串未能符合樣式。這是因為，預設情況下，正規表達式是區分大小寫的。

但是如果我們可以告訴正規表達式忽略大小寫呢？有一個修飾詞可以做到這點。`i` 旗標讓正規表達式忽略大小寫。我們要如何使用它？旗標放在正規表達式結尾的斜線後面：

```js
const regex = /freeCodeCamp/i;
```

注意第一行中正規表達式的變更。現在我們可以檢查這如何改變事情：

:::interactive_editor

```js
const regex = /freeCodeCamp/i;

console.log(regex.test("freeCodeCamp")); // true
console.log(regex.test("freeCodeCamp is great")); // true
console.log(regex.test("I love freeCodeCamp")); // true
console.log(regex.test("freecodecamp")); // true
console.log(regex.test("FREECODECAMP")); // true
console.log(regex.test("free")); // false
console.log(regex.test("code")); // false
console.log(regex.test("camp")); // false
```

:::

因為我們的正規表達式現在不區分大小寫，全部小寫和全部大寫的字串都已「通過」測試。這也適用於大小寫字母隨機混合的字串：

:::interactive_editor

```js
const regex = /freeCodeCamp/i;

console.log(regex.test("dO yOu LoVe fReEcOdEcAmP?")); // true
```

:::

還有相當多其他你可以使用的旗標。`g` 旗標，或稱全域的修飾詞，允許你的正規表達式多次匹配一個樣式。

讓我們看看這如何影響我們的程式碼。你會注意到我們保留了 `i` 旗標：一個正規表達式可以使用多個旗標（需要多少就用多少）來達成你想要的行為：

```js
const regex = /freeCodeCamp/gi;
```

等一下……這是什麼？看起來全域的修飾詞讓我們一些本應通過的字串反而失敗了：

:::interactive_editor

```js
const regex = /freeCodeCamp/gi;

console.log(regex.test("freeCodeCamp")); // true
console.log(regex.test("freeCodeCamp is great")); // false
console.log(regex.test("I love freeCodeCamp")); // true
console.log(regex.test("freecodecamp")); // false
console.log(regex.test("FREECODECAMP")); // true
console.log(regex.test("free")); // false
console.log(regex.test("code")); // false
console.log(regex.test("camp")); // false
```

:::

為什麼？因為全域的修飾詞會讓你的正規表達式具有狀態性。這表示它會記住先前匹配過樣式的位置。所以當它匹配到第一個 `freeCodeCamp` 字串時，它會記住在索引 `0` 開始找到匹配。

接著我們用 `freeCodeCamp is great` 來測試它，但它並沒有從索引 `0` 開始。正規表達式「知道」它已經在索引 `0` 找到匹配，因此即使這是不同的字串，它仍然從匹配的結束索引開始。

`freeCodeCamp` 長度為 `12` 個字元，所以在 `0` 的比對結束於索引值 `11`。比對將在索引值 `12` 繼續。且因為 `is great` 不符合 `freeCodeCamp`，它傳回 `false`。

然後，因為找不到匹配，它會「失去」其狀態並從 `0` 開始下一次匹配。

如果我們調換日誌，使得在 `0` 有匹配的字串緊接著跟隨一個在索引 `11` 之後有匹配的字串：

:::interactive_editor

```js
const regex = /freeCodeCamp/gi;

console.log(regex.test("freeCodeCamp")); // true
console.log(regex.test("I loooooooove freeCodeCamp")); // true
```

:::

當一個正規表達式是全域的時候，它會有一個名為 `lastIndex` 的新屬性。拿我們之前的程式碼來看，讓我們看看這個屬性如何運作：

:::interactive_editor

```js
const regex = /freeCodeCamp/gi;

console.log(regex.lastIndex); // 0
console.log(regex.test("freeCodeCamp")); // true
console.log(regex.lastIndex); // 12
console.log(regex.test("freeCodeCamp is great")); // false
console.log(regex.lastIndex); // 0
console.log(regex.test("I love freeCodeCamp")); // true
console.log(regex.lastIndex); // 19
console.log(regex.test("freecodecamp")); // false
console.log(regex.lastIndex); // 0
console.log(regex.test("FREECODECAMP")); // true
console.log(regex.lastIndex); // 12
console.log(regex.test("free")); // false
console.log(regex.lastIndex); // 0
console.log(regex.test("code")); // false
console.log(regex.lastIndex); // 0
console.log(regex.test("camp")); // false
```

:::

從這個範例中，你可以看到正規表達式的狀態如何隨著每次使用 `lastIndex` 追蹤先前匹配的測試呼叫而改變。

當你需要從單一字串取得多個符合項目時，全域的旗標非常有用。但如果你要用相同的正規表達式測試多個字串，最好不要開啟 `g` 旗標。

在學習下一個旗標之前，你需要先了解錨點。插入符號（`^`）錨點，位於正規表達式的開頭，表示「比對字串的開頭」：

```js
const start = /^freecodecamp/i;
```

美元符號（`$`）錨點，在正規表達式的結尾，表示「比對字串的結尾」。

```js
const end = /freecodecamp$/i;
```

花點時間比較右側的輸出：

:::interactive_editor

```js
const start = /^freecodecamp/i;
const end = /freecodecamp$/i;
console.log(start.test("freecodecamp")); // true
console.log(end.test("freecodecamp")); // true
console.log(start.test("freecodecamp is great")); // true
console.log(end.test("freecodecamp is great")); // false
console.log(start.test("i love freecodecamp")); // false
console.log(end.test("i love freecodecamp")); // true
console.log(start.test("have met freecodecamp's founder")); // false
console.log(end.test("have met freecodecamp's founder")); // false
```

:::

看看起始錨點只會在字串的開頭匹配，而結束錨點只會在字串的結尾匹配？但是跨多行匹配呢？讓我們來看看那個：

:::interactive_editor

```js
const start = /^freecodecamp/i;
const end = /freecodecamp$/i;
const string = `I really love
freecodecamp
it's my favorite`;
console.log(start.test(string)); // false
console.log(end.test(string)); // false
```

:::

即使 `freecodecamp` 獨自位於一行中，它仍未通過兩個測試。這是因為錨點預設會尋找整個字串的開始和結束。

但是你可以使用 `m` 旗標，或多行修飾詞，讓正規表達式處理多行。讓我們將它添加到我們的正規表達式中，看看會得到什麼：

:::interactive_editor

```js
const start = /^freecodecamp/im;
const end = /freecodecamp$/im;
const string = `I really love
freecodecamp
it's my favorite`;
console.log(start.test(string)); // true
console.log(end.test(string)); // true
```

:::

現在它們兩者都匹配了！因為 `freecodecamp` 完全位於自己的獨立行，起始錨點匹配該行的開頭，結束錨點匹配該行的結尾。

最後，你有 `d` 旗標，或稱為索引修飾詞。請記住，`i` 旗標是用於不區分大小寫，因此索引修飾詞需要不同的旗標。

`d` 旗標會擴展你在比對物件中取得的資訊。讓我們將它添加到我們的正規表達式中：

```js
const regex = /freecodecamp/di;
const string = "we love freecodecamp isn't freecodecamp great?";
console.log(string.match(regex));
```

結果如下：

```js
// [
//   'freecodecamp',
//   index: 8,
//   input: "we love freecodecamp isn't freecodecamp great?",
//   groups: undefined,
//   indices: [
//     0: [8, 20],
//     groups: undefined
//   ]
// ]
```

我們的 match 物件獲得了一個新的 `indices` 屬性！這個屬性是一個包含兩個數字的陣列，第一個數字是匹配開始於原始字串中的索引，第二個數字是匹配結束後的索引。這個陣列還有一個額外的 `groups` 屬性，這也是用於命名捕獲群組。

還有一些其他旗標你應該知道是可用的，但在典型的程式碼中較不常見。

第一個是 unicode 修飾詞，或稱為 `u` 旗標。這擴展了正規表達式的機能，使其能匹配特殊的 unicode 字元。

你將在未來的課程中學習更多關於字元類別的內容，但 `u` 旗標讓你可以存取像 `Extended_Pictographic` 這樣的特殊類別，以匹配大多數表情符號：

:::interactive_editor

```js
const regex = /🍎/u;

const str = "I have an apple 🍎";
console.log(regex.test(str)); // true
```

:::

還有一個 `v` 旗標，進一步擴展了 unicode 匹配的功能。

第二個是 sticky 修飾詞，或稱為 `y` 旗標。sticky 修飾詞的行為與全域的修飾詞非常相似，但有一些例外。

最大的差別是，全域的正規表達式會從 `lastIndex` 開始，搜尋字串剩餘部分的整個範圍以尋找下一個匹配，但 sticky 正規表達式如果在先前的 `lastIndex` 沒有立即匹配，會傳回 `null` 並將 `lastIndex` 重設為 `0`。

最後是單行修飾詞，或稱 `s` 旗標。記得多行修飾詞允許起始和結尾錨點匹配行的開始和結束，而不是整個字串。

單行修飾詞允許正規表達式中的萬用字元（用句點 `.` 表示）匹配換行字元，實際上將字串視為單行文字。

這些修飾詞有相當多，但你最常使用且最重要記住的是 `i` 和 `g` 旗標。
