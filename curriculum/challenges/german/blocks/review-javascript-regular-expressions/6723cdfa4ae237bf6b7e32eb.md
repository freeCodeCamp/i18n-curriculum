## Reguläre Ausdrücke und gängige Methoden

- **Definition**: Reguläre Ausdrücke, oder Regex, werden verwendet, um ein „Muster“ zu erstellen, mit dem Sie dann einen String überprüfen, Text extrahieren und mehr können.

```js
const regex = /freeCodeCamp/;
```

- **`test()` Methode**: Diese Methode akzeptiert einen String, der auf Übereinstimmungen mit dem regulären Ausdruck getestet wird. Diese Methode gibt einen Boolean zurück, wenn der String dem Regex entspricht.

:::interactive_editor

```js
const regex = /freeCodeCamp/;
const test = regex.test("e");
console.log(test); // false
```

:::

- **`match()` Methode**: Diese Methode akzeptiert einen regulären Ausdruck, obwohl Sie auch einen String übergeben können, der in einen regulären Ausdruck umgewandelt wird. Die `match` Methode gibt das Übereinstimmungs-Array für den String zurück.

```js
const regex = /freeCodeCamp/;
const match = "freeCodeCamp".match(regex);
console.log(match); // ['freeCodeCamp', index: 0, input: 'freeCodeCamp', groups: undefined]
```

- **`replace()` Methode**: Diese Methode akzeptiert zwei Argumente: den regulären Ausdruck, der übereinstimmen soll (oder einen String), und den String, mit dem die Übereinstimmung ersetzt werden soll (oder eine Funktion, die für jede Übereinstimmung ausgeführt wird).

:::interactive_editor

```js
const regex = /Jessica/;
const str = "Jessica is rly kewl";
const replaced = str.replace(regex, "freeCodeCamp");
console.log(replaced); // "freeCodeCamp is rly kewl"
```

:::

- **`replaceAll` Methode**: Diese Methode wird verwendet, um alle Vorkommen eines bestimmten Musters durch einen neuen String zu ersetzen. Diese Methode wirft einen Fehler, wenn Sie ihr einen regulären Ausdruck ohne den globalen Modifikator übergeben.

:::interactive_editor

```js
const text = "I hate JavaScript! I hate programming!";
const newText = text.replaceAll("hate", "love");
console.log(newText);  // "I love JavaScript! I love programming!"
```

:::

- **`matchAll` Methode**: Diese Methode wird verwendet, um alle Übereinstimmungen eines gegebenen regulären Ausdrucks in einem String abzurufen, einschließlich der Capturing-Gruppen, und gibt sie als Iterator zurück. Ein Iterator ist ein Objekt, das es Ihnen ermöglicht, eine Sammlung von Elementen durchzugehen (oder „zu iterieren“).

:::interactive_editor

```js
const str = "JavaScript, Python, JavaScript, Swift, JavaScript";
const regex = /JavaScript/g;

const iterator = str.matchAll(regex);

for (let match of iterator) {
  console.log(match[0]); // "JavaScript" for each match
}
```

:::

## Modifikatoren für reguläre Ausdrücke

- **Definition**: Modifikatoren, oft „Flags“ genannt, verändern das Verhalten eines regulären Ausdrucks.
- **`i` Flag**: Dieses Flag lässt einen Regex die Groß- und Kleinschreibung ignorieren.

:::interactive_editor

```js
const regex = /freeCodeCamp/i;
console.log(regex.test("freecodecamp")); // true
console.log(regex.test("FREECODECAMP")); // true
```

:::

- **`g` Flag**: Dieses Flag, oder globaler Modifikator, erlaubt Ihrem regulären Ausdruck, ein Muster mehr als einmal zu finden.

:::interactive_editor

```js
const regex = /freeCodeCamp/gi;
console.log(regex.test("freeCodeCamp")); // true
console.log(regex.test("freeCodeCamp is great")); // false
```

:::

- **Definition Anker**: Der `^` Anker am Anfang des regulären Ausdrucks bedeutet „beginne am Anfang des Strings“. Der `$` Anker am Ende des regulären Ausdrucks bedeutet „ende am Ende des Strings“.

:::interactive_editor

```js
const start = /^freeCodeCamp/i;
const end = /freeCodeCamp$/i;
console.log(start.test("freecodecamp")); // true
console.log(end.test("freecodecamp")); // true
```

:::

- **`m` Flag**: Anker suchen den Anfang und das Ende des gesamten Strings. Aber Sie können einen Regex mit dem `m` Flag, dem Mehrzeilen-Modifikator, so einstellen, dass er mehrere Zeilen behandelt.

:::interactive_editor

```js
const start = /^freecodecamp/im;
const end = /freecodecamp$/im;
const str = `I love 
freecodecamp
it's my favorite
`;
console.log(start.test(str)); // true
console.log(end.test(str)); // true
```

:::

- **`d` Flag**: Dieses Flag erweitert die Informationen, die Sie in einem Match-Objekt erhalten.

```js
const regex = /freecodecamp/di;
const string = "we love freecodecamp isn't freecodecamp great?";
console.log(string.match(regex));
```

Und das Ergebnis ist:

```js
// [
//   'freecodecamp',
//   index: 8,
//   input: "we love freecodecamp isn't freecodecamp great?",
//   groups: undefined,
//   indices: [
//     0: [8, 20],
//     groups: undefined
//   ]
// ]
```

- **`u` Flag**: Dieses Flag erweitert die Funktionalität eines regulären Ausdrucks, sodass er spezielle Unicode-Zeichen erkennen kann. Das `u` Flag gibt Ihnen Zugriff auf spezielle Klassen wie die `Extended_Pictographic`, um die meisten Emojis zu erkennen. Es gibt auch ein `v` Flag, das die Unicode-Erkennung noch weiter ausbaut.
- **`y` Flag**: Der Sticky-Modifikator verhält sich sehr ähnlich wie der globale Modifikator, aber mit einigen Ausnahmen. Die wichtigste ist, dass ein globaler regulärer Ausdruck ab lastIndex startet und den gesamten Rest des Strings nach einer weiteren Übereinstimmung durchsucht, während ein Sticky-Regulärer Ausdruck null zurückgibt und lastIndex auf 0 zurücksetzt, wenn nicht sofort eine Übereinstimmung an der vorherigen lastIndex-Position gefunden wird.
- **`s` Flag**: Der Single-Line-Modifikator erlaubt einem Wildcard-Zeichen, dargestellt durch ein `.` im Regex, Zeilenumbrüche zu erkennen – wodurch der String effektiv als eine einzige Textzeile behandelt wird.

## Zeichenklassen

- **Wildcard `.`**: Zeichenklassen sind eine spezielle Syntax, mit der Sie Mengen oder Teilmengen von Zeichen erkennen können. Die erste Zeichenklasse, die Sie lernen sollten, ist die Wildcard-Klasse. Die Wildcard wird durch einen Punkt dargestellt und erkennt JEDES einzelne Zeichen AUSSER Zeilenumbrüchen. Um die Wildcard-Klasse Zeilenumbrüche erkennen zu lassen, müssen Sie das `s` Flag verwenden.

```js
const regex = /a./;
```

- **`\d`**: Dies erkennt alle Ziffern (`0-9`) in einem String.

```js
const regex = /\d/;
```

- **`\w`**: Dies wird verwendet, um jedes Wortzeichen (`a-zA-Z0-9_`) in einem String zu finden. Ein Wortzeichen ist definiert als jeder Buchstabe von a bis z oder A bis Z, eine Zahl von 0 bis 9 oder das Unterstrichzeichen.

```js
const regex = /\w/;
```

- **`\s`**: Die Weißraum-Klasse `\s`, dargestellt durch einen Backslash gefolgt von einem `s`. Diese Zeichenklasse erkennt jeden Weißraum, einschließlich neuer Zeilen, Leerzeichen, Tabs und spezieller Unicode-Leerzeichen.
- **Negieren spezieller Zeichenklassen**: Um eine dieser Zeichenklassen zu negieren, verwenden Sie statt eines Kleinbuchstabens nach dem Backslash den entsprechenden Großbuchstaben. Das folgende Beispiel erkennt kein numerisches Zeichen. Stattdessen erkennt es jedes einzelne Zeichen, das KEIN numerisches Zeichen ist.

```js
const regex = /\D/;
```

- **Benutzerdefinierte Zeichenklassen**: Sie können benutzerdefinierte Zeichenklassen erstellen, indem Sie die Zeichen, die Sie erkennen möchten, in eckige Klammern setzen.

```js
const regex = /[abcdf]/;
```

## Lookahead- und Lookbehind-Zusicherungen

- **Definition**: Lookahead- und Lookbehind-Zusicherungen erlauben es Ihnen, bestimmte Muster basierend auf dem Vorhandensein oder Fehlen umliegender Muster zu erkennen.
- **Positive Lookahead-Zusicherung**: Diese Zusicherung erkennt ein Muster, wenn es von einem anderen Muster gefolgt wird. Um ein positives Lookahead zu erstellen, beginnen Sie mit dem Muster, das Sie erkennen möchten. Dann verwenden Sie runde Klammern, um das Muster einzuschließen, das Sie als Bedingung verwenden möchten. Nach der öffnenden Klammer verwenden Sie `?=`, um dieses Muster als positives Lookahead zu definieren.

```js
const regex = /free(?=code)/i;
```

- **Negative Lookahead-Zusicherung**: Dies ist eine Art Bedingung in regulären Ausdrücken, um zu prüfen, dass ein bestimmtes Muster nicht vor dem aktuellen Punkt im String vorkommt.

```js
const regex = /free(?!code)/i;
```

- **Positive Lookbehind-Zusicherung**: Diese Zusicherung erkennt ein Muster nur, wenn es von einem anderen bestimmten Muster vorausgeht, ohne das vorausgehende Muster in die Übereinstimmung einzubeziehen.

```js
const regex = /(?<=free)code/i;
```

- **Negative Lookbehind-Zusicherung**: Diese Zusicherung stellt sicher, dass ein Muster nicht von einem anderen bestimmten Muster vorausgegangen wird. Sie erkennt nur, wenn das angegebene Muster nicht unmittelbar von der gegebenen Sequenz vorausgegangen wird, ohne die vorausgehende Sequenz in die Übereinstimmung einzubeziehen.

```js
const regex = /(?<!free)code/i;
```

## Regex-Quantifizierer

- **Definition**: Quantifizierer in regulären Ausdrücken geben an, wie oft ein Muster (oder ein Teil eines Musters) erscheinen soll. Sie helfen dabei, die Anzahl der Vorkommen von Zeichen oder Gruppen in einer Übereinstimmung zu steuern. Das folgende Beispiel wird verwendet, um das vorherige Zeichen genau viermal zu erkennen.

```js
const regex = /^\d{4}$/;
```

- **`*`**: Erkennt 0 oder mehr Vorkommen des vorhergehenden Elements.
- **`+`**: Erkennt 1 oder mehr Vorkommen des vorhergehenden Elements.
- **`?`**: Erkennt 0 oder 1 Vorkommen des vorhergehenden Elements.
- **`{n}`**: Erkennt genau n Vorkommen des vorhergehenden Elements.
- **`{n,}`**: Erkennt n oder mehr Vorkommen des vorhergehenden Elements.
- **`{n,m}`**: Erkennt zwischen n und m Vorkommen des vorhergehenden Elements.

## Capturing-Gruppen und Backreferences

- **Capturing-Gruppen**: Eine Capturing-Gruppe erlaubt es Ihnen, einen Teil des erkannten Strings „einzufangen“, um ihn nach Bedarf zu verwenden. Capturing-Gruppen werden durch runde Klammern definiert, die das zu erfassende Muster enthalten, ohne führende Zeichen wie ein Lookahead.

```js
const regex = /free(code)camp/i;
```

- **Nicht-erfassende Gruppen**: Eine nicht-erfassende Gruppe ist ähnlich wie eine erfassende Gruppe, speichert aber den erkannten Teil des Strings nicht für die spätere Verwendung. Nicht-erfassende Gruppen werden durch `(?:...)` definiert.

```js
const regex = /free(?:code)camp/i;
```

- **Backreferences**: Eine Backreference in regulären Ausdrücken bezieht sich auf eine Möglichkeit, einen Teil des Musters, der zuvor in derselben Expression erkannt wurde, wiederzuverwenden. Sie erlaubt es, auf eine erfasste Gruppe (einen Teil des Musters in Klammern) durch ihre Nummer zu verweisen. Zum Beispiel bezieht sich `$1` auf die erste erfasste Gruppe.

:::interactive_editor

```js
const regex = /free(co+de)camp/i;
console.log("freecoooooooodecamp".replace(regex, "paid$1world"));
```

:::

- Sie können Rückverweise innerhalb des Regex selbst verwenden, um denselben Text zu erkennen, der von einer vorherigen Gruppe mit einem Backslash und der Nummer der Erfassungsgruppe erfasst wurde. Zum Beispiel:

:::interactive_editor

```js
const regex = /(hello) \1/i;
console.log(regex.test("hello hello")); // true
console.log(regex.test("hello world")); // false
```

:::
