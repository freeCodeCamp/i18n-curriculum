# --description--

このチャレンジでは、以下のいずれかの方法でコードを書いて進めていきます。

- <a href="https://github.com/freeCodeCamp/boilerplate-advancednode/" target="_blank" rel="noopener noreferrer nofollow">このGitHubリポジトリ</a>をクローンして、ローカルでチャレンジを完了させる。
- お好きなサイトビルダーを使ってプロジェクトを完成させる。ただし、GitHubリポジトリのすべてのファイルを取り込むことを忘れないでください。

テンプレートエンジンは、アプリで静的なテンプレートファイル（*Pug*で書かれたものなど）を使えるようにします。実行時にテンプレートエンジンはテンプレートファイル内の変数を、サーバーから渡された実際の値に置き換えます。そしてテンプレートを静的なHTMLファイルに変換し、クライアントに送信します。この方法によりHTMLページの設計が簡単になり、クライアントからAPIを呼び出さずにページ上に変数を表示できます。

`pug@~3.0.0` はすでにインストールされており、`package.json` ファイルの依存関係としてリストされています。

Expressはどのテンプレートエンジンを使うかを知る必要があります。`set` メソッドを使って、`pug` を `view engine` プロパティの値として代入してください。

```javascript
app.set('view engine', 'pug');
```

その後、もう一つ`set` メソッドを追加し、`views` の `app` プロパティを `./views/pug` ディレクトリに設定してください。これによりExpressはすべてのビューをそのディレクトリを基準にレンダリングします。

最後に、ホームページのルートで`res.render()` を使い、最初の引数に`index` を渡してください。これで`pug` テンプレートがレンダリングされます。

すべてがうまくいけば、アプリのホームページは空白ではなくなり、Pugテンプレートが正常にレンダリングされたことを示すメッセージが表示されます！

正しくできたと思ったらページを提出してください。エラーが出る場合は、<a href="https://forum.freecodecamp.org/t/advanced-node-and-express/567135#set-up-a-template-engine-1" target="_blank" rel="noopener noreferrer nofollow">ここまで完成したプロジェクトを確認できます</a>。
