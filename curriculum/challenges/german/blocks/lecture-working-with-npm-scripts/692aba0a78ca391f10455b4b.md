# --description--

In der modernen Webentwicklung können Sie Ihren Code in mehrere Dateien aufteilen, die „Module“ genannt werden.

Sie können sich ein Modul als einen eigenständigen und wiederverwendbaren Codeblock vorstellen.

Indem Sie Module in andere Module importieren, können Sie deren Funktionalität wiederverwenden und gleichzeitig den Code in separaten Dateien behalten, was besonders bei großen und komplexen Webanwendungen die Wartbarkeit verbessert.

In Node.js wird jede Datei als Modul betrachtet.

Node hat zwei verschiedene Systeme für die Arbeit mit Modulen: CommonJS und ES.

CommonJS ist die ursprüngliche und standardmäßige Methode, um mit Paketen in Node.js zu arbeiten. Es lädt Module synchron, was bedeutet, dass es warten muss, bis ein Modul vollständig geladen ist, bevor es das nächste lädt oder den Code weiter ausführt.

Es wird immer noch weit verbreitet genutzt, wird aber allmählich durch die neuere ES-Syntax ersetzt.

CommonJS verwendet die `require()`-Funktion, um ein Modul zu importieren, wobei der relative Pfad als Argument übergeben wird.

Zum Beispiel nehmen wir an, wir haben ein Modul namens `math.js`, in dem wir grundlegende mathematische Operationen definieren.

Wir verwenden hier nur eine Multiplikationsfunktion, um dieses Beispiel einfach und fokussiert zu halten:

```js
function multiply(a, b) {
  return a * b;
}
```

Wie können wir diese Funktion außerhalb unseres aktuellen Moduls wiederverwenden? Standardmäßig sind Funktionen und Variablen, die in einem Node.js-Modul definiert sind, **privat** — andere Module können nicht auf sie zugreifen, es sei denn, wir exportieren sie explizit. Es gibt mehrere Möglichkeiten, dies zu tun.

Eine gängige Methode ist, einem `module.exports`-Objekt einen Wert zuzuweisen. Dieses Objekt repräsentiert, was das Modul zurückgibt, wenn es an anderer Stelle importiert wird, und macht so Funktionen, Variablen oder Objekte verfügbar, die Sie freigeben möchten.

In diesem Beispiel weisen wir ein Objekt mit einer `multiply`-Eigenschaft `module.exports` zu. Der Wert dieser Eigenschaft ist eine Referenz auf die zuvor definierte `multiply`-Funktion. Dadurch wird die Funktion exportiert, sodass wir sie bei Bedarf aus anderen Modulen aufrufen können.

```javascript
function multiply(a, b) {
  return a * b;
}

module.exports = {
  multiply: multiply
};
```

Nehmen wir an, wir versuchen, das `math.js`-Modul in ein anderes Modul zu importieren. Wir rufen die `require()`-Funktion auf und übergeben den relativen Pfad als Argument. Dann weisen wir das von `require()` zurückgegebene Objekt der Variablen `math` zu.

```javascript
const math = require('./math');
```

Schauen wir uns den Wert von `math` in der Konsole an, indem wir ihn mit `console.log()` ausgeben:

```javascript
console.log(math);
```

Das ist die Ausgabe. Es ist ein Objekt mit einer `multiply`-Eigenschaft und einer Funktion als Wert:

```javascript
{ multiply: [Function: multiply] }
```

Wenn wir diese Funktion mit `math.multiply()` aufrufen und die Zahlen `6` und `7` als Argumente übergeben, ist die Ausgabe `42`. Die Funktion läuft erfolgreich, obwohl sie sich in einer anderen Datei befindet! Unser Import hat also korrekt funktioniert.

```javascript
const math = require('./math');

console.log(math.multiply(6, 7)); // 42
```

Es gibt aber auch andere Möglichkeiten, eine Funktion, Variable oder ein Objekt zu exportieren.

Wenn Sie zum Beispiel nur eine einzelne Funktion, Variable oder ein Objekt exportieren müssen, können Sie es direkt als Wert von `module.exports` zuweisen:

```javascript
function multiply(a, b) {
  return a * b;
}

module.exports = multiply;
```

Wenn Sie mehrere Funktionen exportieren möchten, müssen Sie sie einfach dem `module.exports`-Objekt hinzufügen, getrennt durch ein Komma.

Hier sehen Sie, dass das exportierte Objekt zwei Eigenschaften hat: `subtract` und `multiply`. Ihre Werte sind Referenzen auf die entsprechenden Funktionen, sodass Sie in jedem Modul, das `math.js` importiert, Zugriff auf beide haben:

```javascript
function subtract(a, b) {
  return a - b;
}

function multiply(a, b) {
  return a * b;
}

module.exports = {
  subtract: subtract,
  multiply: multiply
};
```

Wenn das `module.exports`-Objekt mehrere Eigenschaften hat, können Sie wählen, ob Sie alles importieren oder die Eigenschaften einzeln importieren möchten.

In diesem Beispiel importieren wir das gesamte Objekt, das von `math.js` exportiert wurde, sodass wir Zugriff auf sowohl `subtract` als auch `multiply` haben:

```javascript
const math = require('./math');

console.log(math.subtract(6, 7));  // -1
console.log(math.multiply(6, 7));  // 42
```

Wenn wir nur `multiply` importieren möchten, müssen wir einfach seinen Namen in geschweifte Klammern schreiben, nach dem Schlüsselwort `const`. Im Grunde verwenden Sie die Destrukturierungszuweisung, um seinen Wert aus dem exportierten Objekt zu erhalten. Dann können wir direkt mit seinem Namen im Code darauf zugreifen:

```javascript
const { multiply } = require('./math');

console.log(multiply(6, 7));
```

Das funktioniert auch für mehrere Elemente. Sie müssen sie nur durch Kommas innerhalb der geschweiften Klammern trennen. Hier importieren wir sowohl die Funktionen `subtract` als auch `multiply`:

```javascript
const { subtract, multiply } = require('./math');
```

Standardmäßig behandelt Node.js Folgendes als CommonJS-Module:

* Dateien mit der `.cjs`-Erweiterung.
    
* Dateien mit der `.js`-Erweiterung, wenn die nächstgelegene übergeordnete `package.json`-Datei ein Top-Level-Feld `"type"` mit dem Wert `"commonjs"` enthält.
    
* Dateien mit der `.js`-Erweiterung oder ohne Erweiterung, wenn die nächstgelegene übergeordnete `package.json`-Datei kein Top-Level-Feld `"type"` enthält oder kein `package.json` in einem übergeordneten Ordner vorhanden ist.
    
* Dateien mit einer Erweiterung, die nicht `.mjs`, `.cjs`, `.json`, `.node` oder `.js` ist.
    

Während ES-Module CommonJS-Module allmählich ersetzen, ist das Verständnis von CommonJS weiterhin wichtig für Entwickler, die mit bestehenden Node.js-Codebasen und vielen npm-Paketen arbeiten.
