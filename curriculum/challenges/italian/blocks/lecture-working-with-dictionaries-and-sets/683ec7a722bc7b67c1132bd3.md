# --description--

In Python, i dizionari sono strutture dati integrate che archiviano collezioni di coppie chiave-valore. Funzionano in modo molto simile ai dizionari reali, dove cerchi una parola per trovare il suo significato corrispondente.

Con i dizionari Python, usi una chiave per trovare il valore corrispondente. Dovresti usare i dizionari quando hai bisogno di associare valori a chiavi uniche. Questo è utile quando devi trovare un valore rapidamente basandoti sulla chiave e quando devi rappresentare dati strutturati.

Questa è la sintassi generale di un dizionario Python:

```python
dictionary = {
    key1: value1,
    key2: value2
}
```

Per prima cosa troviamo la variabile che contiene il dizionario. Non è necessario assegnare il dizionario a una variabile, ma è molto comune farlo per tenerlo in memoria e usarlo più avanti nel codice.

Segue poi una coppia di parentesi graffe, chiamate a volte parentesi graffe. All’interno delle parentesi graffe ci sono coppie chiave-valore.

Ogni chiave è associata a un valore, così puoi usare la chiave per consultare quel valore. Dopo ogni valore, tranne l’ultimo, c’è una virgola per separare le diverse coppie chiave-valore. Le chiavi devono essere uniche nel dizionario e devono essere di un tipo di dato immutabile. Tuttavia, i valori possono essere ripetuti e possono essere di qualsiasi tipo di dato.

Qui abbiamo un esempio di dizionario che archivia informazioni su una ricetta della pizza Margherita:

```python
pizza = {
    'name': 'Margherita Pizza',
    'price': 8.9,
    'calories_per_slice': 250,
    'toppings': ['mozzarella', 'basil']
}
```

Il dizionario è assegnato alla variabile `pizza`. Ha quattro coppie chiave-valore: `name`, `price`, `calories_per_slice` e `toppings`.

Un’altra alternativa sarebbe usare il costruttore `dict()`, che costruisce il dizionario da una sequenza di coppie chiave-valore.

Questa sarebbe la sintassi equivalente per il nostro esempio della pizza. Passiamo una lista di tuple come argomento al costruttore `dict()`. Queste tuple contengono la chiave come primo elemento e il valore come secondo elemento.

```python
pizza = dict([('name', 'Margherita Pizza'), ('price', 8.9), ('calories_per_slice', 250), ('toppings', ['mozzarella', 'basil'])])
```

Per consultare il valore di una coppia chiave-valore, puoi usare questa sintassi, nota come notazione a parentesi. È il nome della variabile che contiene il dizionario, seguito da parentesi quadre, e la chiave che vuoi consultare all’interno delle parentesi quadre:

```python
dictionary[key]
```

Nel nostro esempio della pizza, se vuoi consultare il valore di `name`, scriveresti il nome della variabile, `pizza`, seguito da parentesi quadre, e la chiave, `name`, tra virgolette:

```python
pizza['name']
```

Questo restituirà:

```python
'Margherita Pizza'
```

Per aggiornare un valore, devi solo aggiungere l’operatore di assegnazione, seguito dal nuovo valore.

Se la chiave non esiste nel dizionario, verrà creata una nuova coppia chiave-valore. Nelle versioni recenti di Python, i dizionari preservano l’ordine di inserimento. Questo è utile quando devi iterare sul dizionario:

```python
pizza['name'] = 'Margherita'
```

Ora il valore della chiave `name` è `'Margherita'`:

```python
print(pizza['name']) # 'Margherita'
```

I dizionari hanno anche metodi utili per eseguire operazioni comuni.

Il metodo `.get()` recupera il valore associato a una chiave. È simile alla notazione a parentesi che abbiamo appena usato, ma il suo vantaggio è che puoi impostare un valore predefinito, così non otterrai un errore se la chiave non esiste:

```python
dictionary.get(key, default)
```

In questo esempio, se la chiave `toppings` non esiste, restituirà una lista vuota, che è il valore predefinito che stiamo passando qui come secondo argomento. Ma se `toppings` esiste, restituirà quel valore:

```python
pizza.get('toppings', []) # ['mozzarella', 'basil']
```

I metodi `.keys()` e `.values()` restituiscono un oggetto view con tutte le chiavi e i valori nel dizionario, rispettivamente:

```python
pizza.keys()
# dict_keys(['name', 'price', 'calories_per_slice'])

pizza.values()
# dict_values(['Margherita Pizza', 8.9, 250])
```

Un oggetto view è solo un modo per vedere il contenuto di un dizionario senza creare una copia separata dei dati.

Il metodo `.items()` restituisce un oggetto view con tutte le coppie chiave-valore nel dizionario, includendo sia le chiavi sia i valori:

```python
pizza.items()
# dict_items([('name', 'Margherita Pizza'), ('price', 8.9), ('calories_per_slice', 250)])
```

Il metodo `.clear()` rimuove tutte le coppie chiave-valore dal dizionario:

```python
pizza.clear()
```

Il metodo `.pop()` rimuove la coppia chiave-valore con la chiave che specifichi come primo argomento e restituisce il suo valore. Se la chiave non esiste, restituisce il valore predefinito che specifichi come secondo argomento. Se la chiave non esiste e non passi un valore predefinito, viene lanciata un’eccezione `KeyError`:

```python
pizza.pop('price', 10)
pizza.pop('total_price') # KeyError
```

In Python 3.7 e nelle versioni più recenti, il metodo `.popitem()` rimuove l’ultimo elemento inserito:

```python
pizza.popitem()
```

E infine, il metodo `.update()` aggiorna le coppie chiave-valore con le coppie chiave-valore di un altro dizionario. Se hanno chiavi in comune, i loro valori vengono sovrascritti.

In questo esempio, stiamo aggiornando il dizionario `pizza`. La chiave `price` esiste in entrambi, quindi il suo valore sarà sostituito con `15`.

Ma `total_time` è nuova, quindi verrà aggiunta al dizionario `pizza` come nuova coppia chiave-valore:

```python
pizza.update({ 'price': 15, 'total_time': 25 })
```

Questo è il nuovo dizionario con `price` aggiornato e la nuova `total_time`. Nota come il prezzo ora è `15` e `total_time` è una nuova coppia chiave-valore:

```python
{
    'name': 'Margherita Pizza', 
    'price': 15, 
    'calories_per_slice': 250, 
    'toppings': ['mozzarella', 'basil'], 
    'total_time': 25
}
```

Questi sono alcuni dei metodi per dizionari più comunemente usati, ma ce ne sono molti altri. Scegliere quello giusto può essere utile per eseguire operazioni complesse in modo efficiente.
