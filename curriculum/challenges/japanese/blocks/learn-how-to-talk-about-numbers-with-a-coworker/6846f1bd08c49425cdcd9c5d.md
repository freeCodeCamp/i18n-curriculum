# --explanation--

`aim` は、目標やターゲットを達成することを計画することを意味します。例えば：

`We aim to release the app next month.` — 彼らはそれを望んだり計画しています。

`Beta testing` は、ほぼ完成した製品を実際のユーザーがテストしてバグを見つける段階です。例えば：

`The app is now in beta testing.` — 人々が問題を見つけるためにアプリを試しています。

`Critical issues` は、リリース前に修正しなければならない非常に深刻な問題です。例えば：

`The app crashing on login is a critical issue.` — それは使用を妨げる深刻な問題です。
