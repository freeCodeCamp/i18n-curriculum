# --description--

Lernen wir den `Audio`-Konstruktor und seine gängigen Methoden kennen.

Der `Audio`-Konstruktor ist wie andere Konstruktoren eine spezielle Funktion, die mit dem `new`-Schlüsselwort aufgerufen wird. Er gibt ein `HTMLAudioElement` zurück, das Sie dann verwenden können, um Audio für den Benutzer abzuspielen oder in das DOM einzufügen, damit der Benutzer es selbst steuern kann.

Wenn Sie den Konstruktor aufrufen, können Sie optional eine URL als (einziges) Argument übergeben. Diese URL sollte auf die Quelle der Audiodatei zeigen, die Sie abspielen möchten. Oder, wenn Sie die Quelle dynamisch ändern müssen, können Sie die URL der `src`-Eigenschaft des zurückgegebenen Audioelements zuweisen.

Das zurückgegebene Audioelement bietet verschiedene Methoden zur Steuerung des Audios. Am wahrscheinlichsten verwenden Sie die `play()`-Methode, die die Audiowiedergabe startet. Es gibt auch die `pause()`-Methode, die die Audiowiedergabe pausiert, aber die aktuelle Position im Track beibehält, sodass `play()` die Wiedergabe von diesem Punkt aus fortsetzen kann.

Man könnte erwarten, dass es eine `stop()`-Methode gibt, um die Audiowiedergabe zu pausieren und den Track auf den Anfang zurückzusetzen. Stattdessen sollten Sie jedoch die `pause()`-Methode aufrufen und die `currentTime`-Eigenschaft direkt setzen.

Schließlich kann die `canPlayType()`-Methode verwendet werden, um festzustellen, ob ein Browser wahrscheinlich in der Lage ist, Ihr spezielles Audioformat abzuspielen. Über Audio- und Videoformate lernen Sie in der nächsten Lektion.
