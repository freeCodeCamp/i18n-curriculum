# --interactive--

在上一課中，你學會了如何使用函式（程式），它們是可重複使用的程式碼片段，有助於讓你的程式碼更具模組化、更容易維護且更高效。所有先前的範例都使用了常規的函式語法，如下所示：

```js
function greetings(name) {
  console.log("Hello, " + name + "!");
}
```

但在 JavaScript 中撰寫函式的另一種方式是創建箭頭函式表達式。以下是你如何重構先前範例以改用箭頭函式語法：

```js
const greetings = (name) => {
  console.log("Hello, " + name + "!");
};
```

在這個修訂的範例中，我們正在建立一個名為 `greetings` 的 `const` 變數並指定一個匿名函式（程式）。大部分語法對你來說會很熟悉，除了缺少了 `function` 關鍵字以及在 `name` 參數和函式主體之間新增了箭頭（`=>`）。如果你的參數列中只有一個參數，那麼你可以像這樣移除小括弧：

```js
const greetings = name => {
  console.log("Hello, " + name + "!");
};
```

如果你的箭頭函式（arrow function）沒有參數，則你必須像這樣使用小括弧：

```js
const greetings = () => {
  console.log("Hello");
};
```

當你第一次學習函式（程式）時，你必須用大括號包住函式（程式）主體。但如果你的函式（程式）主體只包含一行程式碼，你可以像這樣移除大括號：

```js
const greetings = name => console.log("Hello, " + name + "!");
```

重要的是要注意，移除一般函式語法中的小括弧和大括弧是不行的。如果你嘗試這樣做，會出現錯誤：

```js
// This will produce syntax errors 
function greetings name console.log("Hello, " + name + "!");
```

這些單行函式型別只有在你使用箭頭函式語法時才有效。另一個關鍵概念是 `return` 述語。以下是使用箭頭函式語法計算面積的範例：

:::interactive_editor

```js
const calculateArea = (width, height) => {
  const area = width * height;
  return area;
};

console.log(calculateArea(5, 3)); // 15
```

:::

我們在名為 `area` 的函式內建立一個變數，然後傳回該變數。但我們可以稍微整理一下程式碼，直接傳回計算結果：

:::interactive_editor

```js
const calculateArea = (width, height) => {
  return width * height;
}; 

console.log(calculateArea(5, 3)); // 15
```

:::

如果你嘗試移除大括號並將計算放在同一行，則你會收到 `Uncaught SyntaxError: Unexpected token 'return'` 訊息：

```js
const calculateArea = (width, height) => return width * height;
```

你收到此錯誤的原因是你需要移除 `return` 述語。當你移除那個 `return` 述語後，錯誤將會消失，且函式仍會隱喻地傳回計算結果。 

```js
const calculateArea = (width, height) => width * height;
```

那麼你應該在什麼時候使用箭頭函式語法呢？這取決於情況。許多開發者會在他們的個人專案中持續使用它。然而，在團隊合作時，選擇通常取決於現有程式碼庫是使用一般函式還是箭頭函式。在未來的課程中，我們將涵蓋何時使用箭頭函式以及何時避免使用。
