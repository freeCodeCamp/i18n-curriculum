# --description--

Lernen wir asynchrones JavaScript kennen. Asynchrone Ereignisse treten zu unterschiedlichen Zeiten auf, unabhängig voneinander.

Im Kontext der Softwareentwicklung bezeichnet „asynchron“ Aufgaben, die im Hintergrund laufen, unabhängig vom Hauptablauf des Programms. Der Hauptvorteil asynchroner Prozesse besteht darin, dass sie die Ausführung des Hauptprogramms nicht blockieren.

Das ist besonders hilfreich bei Aufgaben, die lange dauern können, wie das Abrufen von Daten von einem entfernten Server, das Verarbeiten großer Dateien, das Verarbeiten von Benutzereingaben und das Durchführen komplexer Berechnungen. Das ist das, was wir als asynchrone Programmierung kennen.

Dieser Ansatz steht im Gegensatz zur traditionellen synchronen Programmierechnik, mit der Sie bisher gearbeitet haben. Wenn Sie mehrere Codezeilen schreiben, können Sie normalerweise vorhersagen, was passiert und wann es passiert. Die erste Zeile wird ausgeführt, dann die zweite Zeile und so weiter in der Reihenfolge, in der Sie sie schreiben.

In diesem Beispiel wird die erste Zeile, in der Sie die Variable definieren, zuerst ausgeführt, dann die zweite Zeile und schließlich die dritte Zeile, in der der String in der Konsole ausgegeben wird:

```js
const topic = "JavaScript";
const learning = `I'm learning ${topic}!`;
console.log(learning);
```

Alles ist sequenziell und vorhersehbar. Jede Zeile wird abgeschlossen, bevor die nächste beginnt. Diese Art von JavaScript-Programm ist als Single-Threaded bekannt.

Das Konzept der Threads ist sehr wichtig. Ein Thread ist eine Ausführungseinheit innerhalb eines Prozesses. Es ist wie ein separater Kontrollfluss innerhalb des Programms.

In der synchronen Programmierung werden Threads nacheinander ausgeführt. Wenn ein Thread blockiert ist, zum Beispiel weil er auf eine Benutzereingabe wartet, wird der gesamte Prozess blockiert, bis der Thread abgeschlossen ist.

JavaScript ist Single-Threaded und verwendet eine Event-Loop, um asynchrone Operationen zu handhaben. Dadurch kann das Programm andere Aufgaben weiter ausführen, während es auf den Abschluss asynchroner Operationen wie Netzwerk-Anfragen oder Timer wartet, ohne den Haupt-Thread zu blockieren.

Asynchrone Programmierung beinhaltet oft Callbacks, Promises oder async/await, um nicht blockierende Operationen zu handhaben.

Wie Sie in früheren Lektionen gelernt haben, ist ein Callback eine Funktion, die Sie an eine andere Funktion übergeben, damit sie später aufgerufen wird. Ereignis-Handler sind eine besondere Art von Callback, mit denen Sie bereits gearbeitet haben. Sie waren früher die gebräuchlichste Methode, um asynchrone Funktionen in JavaScript zu implementieren. Wenn die Callback-Funktion jedoch weitere Callback-Funktionen verwendet, kann der Code und die Logik sehr schnell kompliziert werden.

Derzeit ist die am häufigsten verwendete Technik zur Implementierung asynchroner Programmierung in JavaScript das Promise. Ein `Promise` ist ein Objekt, das die endgültige Fertigstellung (oder das Scheitern) eines asynchronen Prozesses und dessen Wert darstellt.

Der Wert eines Promise ist nicht bekannt, wenn das Promise erstellt wird. Er ist erst bekannt, wenn der asynchrone Prozess abgeschlossen ist. Zum Beispiel kann der Prozess, Daten von einer entfernten API für Ihre Webanwendung abzurufen, einige Zeit dauern. In der Zwischenzeit möchten Sie, dass Ihre Nutzer eine angenehme Benutzererfahrung haben, richtig?

Um dies zu realisieren, könnten Sie ein Promise erstellen, um die Benutzeroberfläche aktiv und interaktiv zu halten, während der asynchrone Prozess läuft. Wenn der Prozess abgeschlossen ist, enthält das Promise die Daten, die von der API zurückgesendet wurden, sodass die Anwendung sie verarbeiten oder angemessen rendern kann, sobald sie verfügbar sind.

Asynchrone Programmierung ist ein mächtiges Werkzeug zum Erstellen effizienter JavaScript-Anwendungen. Wenn Sie die Unterschiede zwischen synchroner und asynchroner Programmierung verstehen, können Sie den richtigen Ansatz für Ihre Anwendung wählen und effizienteren Code schreiben.
