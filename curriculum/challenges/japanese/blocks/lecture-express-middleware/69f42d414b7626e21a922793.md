# --description--

ルーター・レベルのミドルウェアはアプリケーション・レベルのミドルウェアに似ていますが、メインのアプリインスタンスではなく`express.Router()`のインスタンスにバインドされます。これにより、Expressアプリケーションを分割して、独立したルートモジュールを作成できます。

ルーター・レベルのミドルウェアはアプリケーション・レベルのミドルウェアと同じように動作しますが、特定のルーターインスタンスに適用されます。これにより、ミドルウェアをグローバルではなく特定のルートセットにのみ適用しやすくなります。

```js
const express = require('express')
const app = express()
const router = express.Router()

// Router-level middleware to log requests to '/menu' routes
router.use((req, res, next) => {
  console.log('Request made to /menu route')
  next() // Pass control to the next handler
})

// Define routes
router.get('/drinks', (req, res) => {
  res.send('Welcome to the drinks menu!')
})

app.use('/menu', router) // Apply the router

app.listen(3000, () => {
  console.log('Server running on http://localhost:3000')
})
```

* `router.use()`: これはルーター・レベルのミドルウェアで、routerが処理するルートにのみ適用されます。
* `app.use('/menu', router)`: ミドルウェアは`/menu`で始まるルートにのみ適用されます（例：`/menu/drinks`）。

ルーター・レベルのミドルウェアを使うと、アプリをモジュール化されたコンポーネントに分割し、特定のルートグループに選択的にミドルウェアを適用できます。これにより、構成やコードの明確さが向上します。
