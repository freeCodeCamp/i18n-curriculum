## 문제 이해하기
문제를 해결하는 첫 번째 단계는 문제 자체를 정확히 이해하는 것입니다. 문제를 이해하지 못하면 성공적으로 해결했는지 알 수 없고, 잘못된 해결책에 많은 시간을 낭비할 수 있습니다.

문제를 명확히 이해하려면 종이에 적고, 이해가 될 때까지 평이한 영어로 다시 표현하며, 도움이 된다면 도표를 그리세요. 문제를 다른 사람에게 평이한 영어로 설명할 수 있다면 이해한 것입니다.

### 계획 세우기
무엇을 해결할지 알게 되었으니 바로 코딩에 뛰어들지 마세요. 먼저 어떻게 해결할지 계획할 시간입니다. 이 과정에서 답해야 할 질문들은 다음과 같습니다:

- 프로그램에 사용자 인터페이스가 있나요? 어떻게 생겼나요? 인터페이스에 어떤 기능이 있나요? 종이에 스케치해 보세요.

- 프로그램에 어떤 입력이 있나요? 사용자가 데이터를 입력하나요, 아니면 다른 곳에서 입력을 받나요?

- 원하는 출력은 무엇인가요?

- 입력을 고려했을 때, 원하는 출력을 내기 위해 필요한 단계는 무엇인가요?

마지막 질문에서 문제를 해결할 알고리즘을 작성합니다. 알고리즘은 특정 문제를 해결하는 요리법과 같습니다. 컴퓨터가 문제를 해결하기 위해 따라야 할 단계를 의사 코드로 정의합니다.
