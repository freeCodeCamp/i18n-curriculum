# --description--

npm 需要一种方式来保存关于你的项目的重要细节——比如它使用了哪些包，这些包的版本，以及它可以运行哪些命令。它通过 `package.json` 文件来实现这一点。

将 `package.json` 文件视为项目的 npm 配置文件。它是一个简单的 JSON 文件，使用键值点对描述你的项目，以跟踪诸如：

* 项目的名称和版本
    
* 作者
    
* 应该首先运行的主文件
    
* 许可证
    
* 代码仓库、关键字和描述
    
* 项目所需的任何依赖项
    

下面是一个 `package.json` 文件的示例：

```json
{
  "name": "my-app",
  "version": "1.0.0",
  "description": "A simple NPM project",
  "main": "index.js",
  "scripts": {
    "start": "node index.js",
    "test": "echo \"Running tests...\""
  },
  "repository": {
    "type": "git",
    "url": "git+https://github.com/username/project.git"
  },
  "keywords": ["fCC", "js"],
  "author": "Kolade Chris",
  "license": "ISC",
  "dependencies": {
    "react": "^19.2.1"
  },
  "devDependencies": {
    "cypress": "^15.7.1"
  }
}
```

`main` 字段通常设置为 `index.js`，并告诉 Node.js 首先运行哪个文件。

`license` 字段用于指定其他人可以使用、修改和分发你包中代码的法律条款。有许多不同的许可证，如 `MIT`、`ISC` 或 `GPL`，它们都有不同的条款和权限。默认是 `ISC`，即互联网系统联盟许可证，这是一种非常宽松的开源许可证。

随着你的项目增长并安装包，npm 会将它们添加到两个部分之一：

* `dependencies`，是你的应用在生产环境中运行所需的包
    
* `devDependencies`，这些是你仅在开发期间需要的包
    

例如，如果你的项目使用 React 作为其 UI，那么它应放在 `dependencies` 中，因为它是项目部署后正常运行所必需的。像 Cypress 这样的工具，仅用于部署到生产环境前的端到端测试，应放在 `devDependencies` 中。

有几种简单的方法可以创建 `package.json` 文件。最常见的方法是运行以下命令：

```bash
npm init
```

你应该在你的项目根文件夹中运行此命令。该命令启动一个交互式设置进程，npm 会向你提出一系列问题。你的回答将用于填写诸如项目名称、版本、描述、脚本、关键字、作者和许可证等字段。

如果你想跳过这些问题并创建一个带有默认值的 `package.json` 文件，你可以运行：

```bash
npm init -y
```

或者

```bash
npm init --yes
```

这会立即为你创建一个带有许多字段默认值的 `package.json` 文件。

通常最好使用不带 `-y` 或 `--yes` 标记的 `npm init`，这样你可以逐个字段浏览并自定义你的项目的信息。如果你不回答某个问题，npm 会使用默认值。例如，如果你将许可证问题留空，npm 会默认将其设置为 `ISC`。

你可能会注意到 `dependencies` 和 `devDependencies` 还没有被列出。那是因为这些部分只有在你安装第一个包之后才会被添加。

你还会注意到 `repository` 字段缺失。尽管 npm 在 `npm init` 期间询问了它，但之后不会自动填写。例如，当你运行像 `git remote add origin` 或 `git push` 这样的命令时，npm 不会为你更新 repository 字段——如果你想包含它，必须手动添加。
