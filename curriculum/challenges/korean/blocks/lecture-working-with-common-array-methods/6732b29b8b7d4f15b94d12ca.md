# --interactive--

배열의 얕은 복사란 원본과 같은 항목을 가진 새 배열을 말합니다. 배열이 숫자나 문자열 같은 원시값만 포함한다면 새 배열은 완전히 별개입니다. 하지만 배열 안에 다른 배열이 포함되어 있다면, 원본과 복사본 모두 같은 내부 배열을 참조합니다. 즉, 공유된 내부 배열 안의 내용을 변경하면 두 배열 모두에서 그 변경 사항을 볼 수 있습니다.

얕은 복사는 원본 배열이나 내부 배열을 수정하지 않고, 요소를 추가, 제거, 재배열하는 등 최상위 구조만 수정해야 할 때 유용합니다.

배열의 얕은 복사를 만드는 여러 방법이 있으며, 여기서는 가장 일반적인 `concat()`, `slice()`, 그리고 전개 연산자 방법을 살펴보겠습니다.

먼저 `concat()` 메서드부터 시작하겠습니다. 이 메서드는 두 개 이상의 배열을 병합하여 새 배열을 만듭니다. 단일 배열과 함께 사용하면 사실상 얕은 복사를 만듭니다. 예시는 다음과 같습니다:

:::interactive_editor

```js
const originalArray = [1, 2, 3];
const copyArray = [].concat(originalArray);

console.log(copyArray); // [1, 2, 3]
console.log(copyArray === originalArray); // false
```

:::

이 예제에서는 `concat()` 메서드를 사용해 빈 배열을 `originalArray`에 연결하고 있습니다. 이렇게 하면 `originalArray`의 얕은 복사본인 새 배열이 만들어집니다.

`copyArray`은 `originalArray`과 같은 요소를 포함하지만, 서로 다른 배열 객체이기 때문에 엄격한 동등성 검사(`===`)는 `false`를 반환합니다.

얕은 복사를 만드는 또 다른 방법은 `slice()` 메서드입니다. 인수 없이 호출하면 `slice()`은 전체 배열의 얕은 복사본을 반환합니다. 작동 방식은 다음과 같습니다:

:::interactive_editor

```js
const originalArray = [1, 2, 3];
const copyArray = originalArray.slice();

console.log(copyArray); // [1, 2, 3]
console.log(copyArray === originalArray); // false
```

:::

이 경우 `originalArray.slice()`는 `originalArray`의 얕은 복사본인 새 배열을 만듭니다. 다시 말해 `copyArray`는 같은 요소를 포함하지만 다른 배열 객체입니다.

ES6에서 도입된 전개 연산자(`...`)는 배열의 얕은 복사를 만드는 또 다른 간결한 방법을 제공합니다. 예시는 다음과 같습니다:

:::interactive_editor

```js
const originalArray = [1, 2, 3];
const copyArray = [...originalArray];

console.log(copyArray); // [1, 2, 3]
console.log(copyArray === originalArray); // false
```

:::

전개 연산자(`...`)는 `originalArray`의 요소를 새 배열로 펼쳐서 얕은 복사를 만듭니다. 이 모든 방법은 새 배열 객체를 생성하므로 복사본을 수정해도 원본 배열에 영향을 주지 않는다는 점이 중요합니다. 예를 들어:

:::interactive_editor

```js
const originalArray = [1, 2, 3];
const copyArray = [...originalArray];

copyArray.push(4);
console.log(originalArray); // [1, 2, 3]
console.log(copyArray);     // [1, 2, 3, 4]
```

:::

이 예제에서 `copyArray`에 요소를 추가해도 `originalArray`에는 영향을 주지 않습니다.

요약하자면, 배열의 얕은 복사는 `concat()`, `slice()`, 또는 전개 연산자 같은 메서드를 사용해 쉽게 만들 수 있습니다. 이 방법들은 원본 배열과 독립적으로 조작할 수 있는 새 배열을 만드는 데 유용합니다.
