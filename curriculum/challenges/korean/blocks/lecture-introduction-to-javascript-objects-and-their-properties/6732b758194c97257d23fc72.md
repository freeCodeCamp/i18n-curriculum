# --interactive--

JavaScript에서 생성자는 객체를 생성하고 초기화하는 데 사용되는 특별한 유형의 함수입니다. `new` 키워드로 호출되며 새로 생성된 객체에 프로퍼티와 메서드를 초기화할 수 있습니다.

이번 수업에서는 `Object()` 생성자를 다루는 방법을 살펴보겠습니다. `Object()` 생성자는 새로운 빈 객체를 만듭니다. 다음은 예시입니다:

```js
new Object()
```

`new Object()`를 호출하면 값을 저장하는 데 사용할 수 있는 새 객체가 반환됩니다.

`Object()` 생성자는 `new` 키워드와 함께 또는 없이 사용할 수 있습니다. `new` 없이 함수로 호출하면 전달된 값의 유형에 따라 다르게 동작합니다. 다음은 `Object()` 생성자를 `new` 키워드 없이 사용하는 예시입니다:

:::interactive_editor

```js
const num = 42;
const numObj = Object(num); // Creates an object wrapper for the number

console.log(numObj);
console.log(typeof numObj); // "object"
```

:::

두 번째 `console.log`에서 볼 수 있듯이 `numObj`은 객체입니다. 이는 숫자 입력을 객체로 바꾸기 위해 `Object()` 생성자를 사용했기 때문입니다.

`null` 또는 `undefined`을 `Object()` 생성자에 전달하면 어떻게 될까요?

:::interactive_editor

```js
const newObj = new Object(undefined);
console.log(newObj); // {}
```

:::

결과는 빈 객체가 됩니다. `Object()` 생성자를 사용하는 또 다른 경우는 값의 유형을 알 수 없을 때 해당 값이 객체인지 확실히 해야 할 때입니다. 다음 예시를 살펴보겠습니다:

:::interactive_editor

```js
function toObject(value) {
  if (value === null || value === undefined) {
    return {};
  }

  if (typeof value === "object") {
    return value;
  }

  return Object(value);
}

console.log(toObject(null));

console.log(toObject(true));

console.log(toObject([1, 2, 3]));
```

:::

이 예시에는 `toObject`라는 함수가 있습니다. 두 번째 조건은 값이 객체 유형인지 확인하고 조건이 `true`일 경우 값을 반환합니다. 이 조건은 배열도 객체의 특별한 유형이므로 배열도 검사합니다.

두 조건 모두 참이 아니면 함수는 `Object(value)`을 반환하는데, 이는 입력을 객체로 변환합니다. 이 방법은 숫자, 문자열, 불리언 같은 값에 적용됩니다.

대부분의 경우 새 객체를 만들 때 `Object()` 생성자를 사용하지 않고 객체 리터럴 구문(e.g., `const objectLiteral = { name: "Beau" }`)을 사용합니다. 하지만 Object 생성자를 사용하는 기본 개념을 이해하는 것은 여전히 중요합니다.
