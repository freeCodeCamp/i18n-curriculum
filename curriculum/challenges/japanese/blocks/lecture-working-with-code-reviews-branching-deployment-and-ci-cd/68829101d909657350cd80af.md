# --description--

GitHub Pagesは、静的サイトやバックエンドサーバーでロジックを処理する必要のないアプリケーションをデプロイするのに非常に優れた選択肢です。つまり、完全にクライアント側、つまりユーザーのブラウザ上で動作するアプリケーションは、このプラットフォーム上で完全にデプロイできます。

さらに、GitHub Pagesはオープンソースプロジェクトに対して完全に無料です。プロジェクトをオープンソースにする理由はたくさんありますが、Pagesはその理由のひとつをさらに増やします。

では、実際にGitHub Pagesにどうやってデプロイするのでしょうか？それはアプリケーションの動作方法によります。

まずは基本的なHTMLページから始めましょう。もしよければ、既存のリポジトリに追加できるものを用意しました：

```html
<!DOCTYPE html>
<html>
  <head>
    <title>freeCodeCamp</title>
    <link rel="stylesheet" href="./style.css" />
  </head>
  <body>
    <h1>Welcome to freeCodeCamp's Lessons!</h1>
    <p>Let's have some fun learning about Git and GitHub!</p>
  </body>
</html>
```

次にリポジトリにアクセスし、「Settings」タブをクリックしてリポジトリの設定画面に移動します。次にサイドバーから「Pages」を選択してください。

最初はオプションが少しだけですが、設定を進めるにつれて変わっていきます。まず、GitHubにどこからデプロイするかを伝える必要があります。

「Deploy from a branch」のままにしておきましょう。なぜなら、フレームワークを使っておらず、GitHub Actionsも設定していないからです。これらについては今後のレッスンで詳しく学びます。

次に、デプロイ元のブランチを`main`に設定して、その内容をデプロイします。

その後、プロジェクトのルートフォルダーからデプロイするか、`docs`フォルダーからデプロイするかを選べます。この機能はfreeCodeCampのようなプロジェクトで使われています。私たちは以前、ドキュメントをアプリと一緒にまとめていたので、`docs`フォルダーを使ってドキュメントだけをGitHub Pagesにデプロイしていました。今回は`/`（ルート）フォルダーのままにして、このドロップダウンの横にある「Save」ボタンをクリックして変更を保存してください。

保存すると、カスタムドメインを設定するための新しい入力欄が表示されます。すでにドメインを持っていて使いたい場合は、ドキュメントへのリンクをたどって設定できます。ただし、このレッスンでは空欄のままにします。

また、新しいGitHub Pagesサイトがデプロイされたドメインも確認できます。カスタムドメインがない場合、デフォルトは`<username>.github.io`で、`<username>`はあなたのアカウント名（または組織所有のリポジトリからデプロイしている場合は組織のアカウント名）です。リポジトリはリポジトリ名に対応するパス、例えば`<username>.github.io/<repo-name>`で提供されます。URLにアクセスすると、サイトが表示されます！

これで`main`ブランチに新しい変更をプッシュすると、GitHub Pagesが自動的に最新の変更でデプロイ済みサイトを更新します。この処理はサーバーの負荷によって数分かかる場合があり、キャッシュをクリアするためにハードリフレッシュが必要な場合があります。

これで基本的なウェブアプリケーションをGitHub Pagesに無事デプロイできました。CSSやJavaScriptファイルもPagesのデプロイに含められるので、美しくインタラクティブなサイトを作れます。ReactやAngularのようなフレームワークを学べば、それらもGitHub Pagesにデプロイできます！
