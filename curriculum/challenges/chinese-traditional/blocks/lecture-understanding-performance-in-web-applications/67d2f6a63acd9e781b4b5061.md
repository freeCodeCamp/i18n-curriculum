# --description--

為什麼有些網站感覺反應迅速且靈敏，而有些則感覺遲緩？答案在於影響頁面載入和呈現的關鍵效率概念。理解關鍵的網頁效率概念對於建造快速、流暢且使用者友善的網站至關重要。 

讓我們拆解原始碼順序、關鍵呈現路徑、延遲等。

我們先來看看原始碼順序。

原始碼順序是指 HTML 元素在文件中的結構方式。這決定了什麼會先載入，並且會顯著影響效率和存取。

一些最佳實務包括：

- 將標題、導覽或主要文字等關鍵內容置於 HTML 結構的較高位置。

- 延緩非必要的腳本，例如用於分析或第三方小工具的腳本，以免它們阻礙呈現。

- 使用漸進式增強，以確保即使在樣式和腳本載入之前，核心體驗仍能運作。漸進式增強是一種建置網站和應用程式的方法，基於你應該先讓你的頁面能夠使用 HTML 運作的理念。

這裡是一個良好原始碼順序的範例，使用我們剛剛介紹的最佳實務。

```html
<!-- Good source order: Essential content first -->
<h1>Welcome to FastSite!</h1>
<p>Critical information loads first.</p>
<script src="slow-script.js" defer></script>
```

透過優化原始碼順序，我們確保使用者能盡快看到重要內容。

現在讓我們來看看關鍵呈現路徑。

關鍵呈現路徑是瀏覽器將程式碼轉換成螢幕上像素所遵循的步驟序列。

以下是我們稍後將詳細說明的關鍵步驟：

1. 解析 HTML：建置 DOM（文件物件模型）

2. 解析 CSS：建置 CSSOM（CSS 物件模型）

3. JavaScript 執行：可以修改 DOM 和 CSSOM

4. 呈現樹構建：結合 DOM 和 CSSOM

5. 版面配置與繪製：決定元素大小並繪製像素

6. 優化：

    - 將阻礙呈現的資源降到最低（例如，大型 CSS 檔案、未使用的 JS）。

    - 為腳本使用 async 和 defer 屬性：

    - 首先只載入必要的樣式；延緩非關鍵的 CSS。

整體而言，更短的關鍵呈現路徑等於更快的感知效率。我們稍後會更詳細地說明這一點。

最後，讓我們來看看延遲。

延遲是指請求在瀏覽器和伺服器之間傳輸所花費的時間。換句話說，延遲高代表頁面載入緩慢。

減少延遲的一些方法包括：

- 使用 CDN，換句話說，內容傳遞網路，從較近的位置提供檔案。

- 啟用壓縮，例如使用 Gzip，以減少檔案大小。

- 優化影像並使用延遲載入－我們稍後也會詳細說明。

透過降低延遲，我們讓互動感覺即時。

總結來說，透過優化原始碼順序、縮短關鍵路徑以及降低延遲，你可以讓你的網站感覺更快速。
