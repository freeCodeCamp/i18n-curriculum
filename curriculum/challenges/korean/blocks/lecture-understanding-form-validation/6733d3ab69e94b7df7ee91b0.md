# --description--

`submit` 이벤트가 HTML 폼과 어떻게 작동하는지 배워봅시다. 먼저, 폼을 제출하는 방법을 이해해야 합니다. 폼을 제출하는 방법은 세 가지가 있습니다.

첫 번째는 사용자가 `type` 속성이 `submit`로 설정된 폼 내 버튼을 클릭할 때입니다.

두 번째는 사용자가 폼 내 편집 가능한 입력 필드에서 Enter 키를 누를 때입니다.

세 번째는 JavaScript로 폼 요소의 `requestSubmit()` 또는 `submit()` 메서드를 호출할 때입니다.

그런데 폼이 제출되면 무슨 일이 일어날까요? 몇 가지 일이 발생하며, 폼의 동작은 속성에 따라 달라집니다.

먼저 살펴봐야 할 속성은 `action` 속성입니다. `action` 속성에는 URL이나 현재 도메인에 대한 상대 경로가 들어가야 합니다. 이 값은 폼이 데이터를 보내려고 시도하는 위치를 결정합니다. `action` 속성을 설정하지 않으면 폼은 현재 페이지의 URL로 데이터를 보냅니다. 다음은 `action` 속성이 특정 URL로 설정된 폼의 예입니다:

```html
<form action="https://freecodecamp.org">
  <input type="number" id="input" placeholder="Enter a number" name="number" />
  <button type="submit">Submit</button>
</form>
```

이 폼이 제출되면 freeCodeCamp의 홈페이지로 데이터를 보내는데, 아마 아무 일도 일어나지 않을 것입니다.

다음은 상대 경로로 제출하는 또 다른 폼입니다:

```html
<form action="/data">
  <input type="number" id="input" placeholder="Enter a number" name="number" />
  <button type="submit">Submit</button>
</form>
```

이 폼은 현재 도메인의 `/data` 경로로 제출합니다. 우리 경우에는 `http://127.0.0.1:5500/data`로 제출하는 셈입니다.

제출 동작을 제어하는 두 번째 속성은 `method` 속성입니다. 이 속성은 `GET`나 `POST` 같은 표준 HTTP 메서드를 받아들이며, `action` URL로 요청할 때 그 메서드를 사용합니다.

HTTP는 하이퍼텍스트 전송 프로토콜의 약자로, 웹에서 데이터를 전송하는 데 사용됩니다.

HTTP 메서드는 `GET`, `POST`, `PUT`, `DELETE` 등과 같이 리소스에 수행할 수 있는 작업을 정의하는 데 사용됩니다. 이 메서드들은 앞으로 수업에서 더 자세히 배우게 됩니다.

메서드가 설정되지 않으면 폼은 기본적으로 `GET` 요청을 사용합니다. `GET` 요청은 지정된 리소스에서 데이터를 변경하지 않고 가져오는 데 사용되며, 매개변수는 일반적으로 쿼리 문자열 형태로 URL에 추가됩니다.

폼의 데이터는 `name=value` 쌍으로 URL 인코딩되어 `action` URL에 쿼리 매개변수로 추가됩니다. 예를 들어, 이전 예제의 폼에서 `3342` 필드에 `input` 값을 입력하고 제출하면, 폼은 `GET` URL을 `http://127.0.0.1:5500/data?number=3342`하려고 시도합니다.

이 방법은 사용자가 데이터를 쿼리하는 검색 폼 같은 경우에 적합합니다. 하지만 사용자가 등록 폼처럼 새 데이터를 제출하려면, 관례적으로 `POST` 메서드를 사용하는 것이 좋습니다. `POST` 메서드는 서버에 데이터를 보내 리소스를 생성하거나 업데이트하는 데 사용됩니다.

`method` 속성을 `POST` 값으로 설정해봅시다:

```xml
<form action="/data" method="POST">
  <input type="number" id="input" placeholder="Enter a number" name="number" />
  <button type="submit">Submit</button>
</form>
```

`POST` 요청을 보낼 때는 요청에 필요한 데이터를 담은 본문을 포함할 수 있습니다. 그래서 `GET`와 달리 데이터가 URL에 추가되지 않고, 폼은 `POST`에 `http://127.0.0.1:5500/data` 요청을 보냅니다. 데이터는 여전히 URL 인코딩된 폼 데이터로 요청 본문에 포함됩니다.

URL 인코딩된 폼 데이터는 폼 데이터를 `name=John+Doe&email=john%40example.com`처럼 키-값 쌍 문자열로 변환한 것으로, 특수 문자는 안전하게 전송할 수 있도록 인코딩된 버전으로 교체됩니다. 이 부분은 앞으로 수업에서 더 자세히 배우게 됩니다.

하지만 데이터를 URL 인코딩된 폼 페이로드로 보내고 싶지 않을 수도 있겠죠?

`form` 요소는 데이터에 사용할 인코딩 유형을 나타내는 `enctype` 속성을 받습니다.

이 속성은 세 가지 값만 허용합니다: 기본값인 `application/x-www-form-urlencoded` (데이터를 URL 인코딩된 폼 본문으로 보내는 방식), `text/plain` (데이터를 평문 형태로, 줄바꿈으로 구분된 `name=value` 쌍으로 보내는 방식), 그리고 `multipart/form-data`로, 파일 업로드가 포함된 폼을 처리할 때 사용됩니다.
