## アクセシビリティの紹介

- **Web Content Accessibility Guidelines**: Web Content Accessibility Guidelines（WCAG）は、障害のある人がウェブコンテンツによりアクセスしやすくするためのガイドラインのセットです。WCAGの4つの原則はPOURで、Perceivable（知覚可能）、Operable（操作可能）、Understandable（理解可能）、Robust（堅牢）を表します。

## アクセシビリティのための支援技術

- **スクリーンリーダー**: コンピューター画面の内容を音声で読み上げるソフトウェアプログラムです。視覚障害のある人がウェブにアクセスするために使います。
- **大きな文字や点字キーボード**: 視覚障害のある人がウェブにアクセスするために使います。
- **画面拡大ソフト**: コンピューター画面の内容を拡大するソフトウェアプログラムです。視力が低い人がウェブにアクセスするために使います。
- **代替ポインティングデバイス**: 運動障害のある人がウェブにアクセスするために使います。ジョイスティック、トラックボール、タッチパッドなどのデバイスが含まれます。
- **音声認識**: 運動障害のある人がウェブにアクセスするために使います。ユーザーが声でコンピューターを操作できます。

## アクセシビリティ監査ツール

- **一般的なアクセシビリティツール**: Google Lighthouse、Wave、IBM Equal Accessibility Checker、axe DevToolsは、ウェブサイトのアクセシビリティを監査するためによく使われるツールです。

## アクセシビリティのベストプラクティス

- **適切な見出しレベルの構造**: コンテンツの論理的な構造を作るために適切な見出しレベルを使うべきです。これにより支援技術を使う人がウェブサイトの内容を理解しやすくなります。
- **アクセシビリティとテーブル**: テーブルを使う場合は、ヘッダーセルを定義するために`th`要素、データセルを定義するために`td`要素を使うべきです。これにより支援技術を使う人がテーブルの構造を理解しやすくなります。`caption`要素を使うとテーブルのキャプション（タイトル）を書けるので、特に支援技術を使うユーザーがテーブルの目的や内容をすぐに理解できます。`caption`要素は`table`要素の開始タグの直後に置くべきです。こうすることでスクリーンリーダーなどの支援技術がキャプションを読み上げてから内容を伝えられ、より多くの文脈を提供できます。
- **入力に関連付けられたラベルの重要性**: フォームの入力にラベルを関連付けるために`label`要素を使うべきです。これにより支援技術を使う人が入力の目的を理解しやすくなります。
- **適切な`alt`テキストの重要性**: 画像の代替テキストを提供するために`alt`属性を使うべきです。これにより支援技術を使う人が画像の内容を理解しやすくなります。
- **適切なリンクテキストの重要性**: リンクの目的を理解しやすくするために説明的なリンクテキストを使うべきです。これにより支援技術を使う人がリンクの目的を理解しやすくなります。
- **音声および動画コンテンツをアクセシブルにするベストプラクティス**: 聴覚障害のある人がアクセスできるように、音声および動画コンテンツにはキャプションとトランスクリプトを提供すべきです。視覚障害のある人がアクセスできるように、動画コンテンツには音声説明も提供すべきです。
- **`tabindex`属性**: 要素をフォーカス可能にし、キーボードでの移動順序を定義するために使います。`tabindex`属性に0より大きい値を使ってはいけません。代わりに0か-1の値を使うべきです。

:::interactive_editor

```html
<p tabindex="-1">Sorry, there was an error with your submission.</p>
```

:::

- **`accesskey`属性**: 要素のキーボードショートカットを定義するために使います。これにより運動障害のあるユーザーがウェブサイトをより簡単に操作できます。

:::interactive_editor

```html
<button accesskey="s">Save</button>
<button accesskey="c">Cancel</button>
<a href="index.html" accesskey="h">Home</a>
```

:::

## WAI-ARIA、役割、属性

- **WAI-ARIA**: Web Accessibility Initiative - Accessible Rich Internet Applicationsの略です。アクセシビリティを向上させるためにHTML要素に追加できる属性のセットです。支援技術にコンテンツの目的や構造について追加情報を提供します。
- **ARIAの役割**: HTML要素に追加できる事前定義された役割のセットで、要素の目的を定義します。これにより支援技術を使う人がウェブサイトの内容を理解しやすくなります。例として`role="tab"`、`role="menu"`、`role="alert"`があります。

ARIAの役割には6つの主なカテゴリがあります：

- **文書構造の役割**: ウェブページ全体の構造を定義します。これらの役割により支援技術は異なるセクション間の関係を理解し、ユーザーがコンテンツをナビゲートしやすくなります。
- **ウィジェットの役割**: スクロールバーのようなインタラクティブな要素の目的と機能を定義します。
- **ランドマークの役割**: ウェブページの主要なセクションを分類しラベル付けします。スクリーンリーダーはこれらを使って重要なセクションへの便利なナビゲーションを提供します。
- **ライブリージョンの役割**: 動的に内容が変わる要素を定義します。これによりスクリーンリーダーなどの支援技術が視覚障害のあるユーザーに変更を通知できます。
- **ウィンドウの役割**: ポップアップモーダルダイアログのようなサブウィンドウを定義します。`alertdialog`と`dialog`が含まれます。
- **抽象的な役割**: 文書の整理に役立ちます。ブラウザー内部でのみ使われるもので、開発者はウェブサイトやウェブアプリで使うべきではありませんが、存在は知っておくべきです。

- **`aria-label`と`aria-labelledby`属性**: 要素にプログラム的（アクセシブルな）名前を付けるために使います。これによりスクリーンリーダーなどの支援技術を使う人が要素の目的を理解しやすくなります。視覚的なラベルが画像や記号の場合によく使われます。`aria-label`は属性内に直接名前を定義でき、`aria-labelledby`はページ上の既存のテキストを参照できます。

```html
<button aria-label="Search">
    <i class="fas fa-search"></i>
</button>
```

:::interactive_editor

```html
<input type="text" aria-labelledby="search-btn">
<button type="button" id="search-btn">Search</button>
```

:::

- **`aria-hidden`属性**: スクリーンリーダーなどの支援技術から要素を隠すために使います。例えば、意味のない装飾画像を隠す場合に使います。

:::interactive_editor

```html
<button>
    <i class="fa-solid fa-gear" aria-hidden="true"></i>
    <span class="label">Settings</span>
</button>
```

:::

- **`aria-describedby`属性**: 要素に関する追加情報を別の要素と関連付けて提供するために使います。これによりスクリーンリーダーを使う人がその要素に移動したときに追加情報にすぐアクセスできます。一般的な使い方は、テキスト入力に書式設定の指示を関連付けたり、無効なフォーム提出後にエラーメッセージを入力に関連付けたりすることです。

:::interactive_editor

```html
<form>
    <label for="password">Password:</label>
    <input type="password" id="password" aria-describedby="password-help" />
    <p id="password-help">Your password must be at least 8 characters long.</p>
</form>
```

:::
