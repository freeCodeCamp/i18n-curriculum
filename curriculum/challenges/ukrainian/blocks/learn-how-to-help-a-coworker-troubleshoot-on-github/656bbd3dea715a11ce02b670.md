# --description--

У програмуванні, особливо при використанні систем контролю версій, таких як Git, `push` означає завантаження змін з локального репозиторію до віддаленого.

Минулотенсовий `pushed` вказує, що ця дія вже була виконана.

Наприклад, `He pushed his code to the GitHub repository` означає, що він завантажив свої останні зміни коду на GitHub.

# --questions--

## --text--

Що означає `push` у контексті контролю версій?

## --answers--

Скачати зміни з віддаленого репозиторію

### --feedback--

`Push` насправді означає завантажити зміни до віддаленого репозиторію, а не скачати.

---

Видалити зміни з репозиторію

### --feedback--

`Push` не стосується видалення змін з репозиторію.

---

Переглянути зміни в репозиторії

### --feedback--

`push` не стосується перегляду змін у репозиторії.

---

Завантажити зміни до віддаленого репозиторію
