# --interactive--

Конструктор `Number()` використовується для створення об’єкта числа. Об’єкт числа містить кілька корисних властивостей і методів, таких як `isNaN` і метод `toFixed`. Ось приклад використання конструктора `Number()` з ключовим словом `new`:

:::interactive_editor

```js
const myNum = new Number("34");
console.log(typeof myNum); // "object" 
```

:::

У цьому прикладі ми передаємо рядковий літерал у конструктор `Number()`, і повернене значення має тип `object` замість `string`.

Коли конструктор `Number()` викликається як функція без ключового слова `new`, тоді повернене значення буде примітивним типом числа. Здебільшого ви будете використовувати конструктор `Number()` для перетворення інших типів даних у числовий тип. Ось приклад перетворення рядка на число:

:::interactive_editor

```js
const myNum = Number("100");
console.log(myNum); // 100

console.log(typeof myNum); // number
```

:::

Це корисно, коли ви отримуєте вхідні дані від користувача і потрібно перетворити цей рядок у число, щоб виконувати математичні обчислення.

Якщо ви спробуєте викликати конструктор `Number()` з порожнім рядком, результатом буде число `0`:

:::interactive_editor

```js
const num = Number("");
console.log(num); // 0
```

:::

Це тому, що JavaScript намагається аналізувати рядок, і оскільки він не містить жодних цифр, результатом буде нуль.

Якщо ви спробуєте передати рядок із випадковими символами, результатом буде `NaN` або «Не число».

:::interactive_editor

```js
const num = Number("random");
console.log(num); // NaN
```

:::

При роботі з булевими значеннями `true` повертає `1`, тому що `true` перетворюється на одиницю, а `false` повертає `0`, бо `false` перетворюється на нуль.

:::interactive_editor

```js
const boolTrue = Number(true);
const boolFalse = Number(false);

console.log(boolTrue); // 1
console.log(boolFalse); // 0
```

:::

Якщо передати `null`, результатом буде `0`, а якщо передати `undefined`, результатом буде `NaN`.

:::interactive_editor

```js
const undefinedNum = Number(undefined);
const nullNum = Number(null);

console.log(undefinedNum); // NaN
console.log(nullNum); // 0
```

:::

При роботі з масивами слід враховувати кілька моментів.

Порожній масив поверне `0`. Масив з одним числом поверне це число. Масив з кількома числами повертає `NaN`. А масив із рядками також поверне `NaN`.

:::interactive_editor

```js
const emptyArr = Number([]);
const arrOneNum = Number([7]);
const arrMultiNum = Number([7, 36, 12]);
const arrStr = Number(["str1"]);
const arrMultiStr = Number(["str1", "str2"]);

console.log(emptyArr); // 0
console.log(arrOneNum); // 7
console.log(arrMultiNum); // NaN
console.log(arrStr); // NaN
console.log(arrMultiStr); // NaN
```

:::

При роботі з об’єктами результат завжди буде `NaN`.

:::interactive_editor

```js
const obj1 = Number({});
const obj2 = Number({2: 2});
const obj3 = Number({key: "val"});
const obj4 = Number({key: true});

console.log(obj1); // NaN
console.log(obj2); // NaN
console.log(obj3); // NaN
console.log(obj4); // NaN
```

:::

Підсумовуючи, ви здебільшого будете використовувати конструктор `Number()` для перетворення типів, а не для створення числа чи об’єкта числа.
