# --description--

게터와 세터는 클래스의 속성에 어떻게 접근하고 수정할지 제어할 수 있게 해 주는 메서드입니다. 게터는 값을 가져오고, 세터는 값을 설정합니다.

이 작업들은 프로퍼티라고 알려진 것을 통해 이루어집니다. 프로퍼티는 게터와 세터를 연결하고 데이터에 접근할 수 있게 해 줍니다.

프로퍼티는 속성처럼 동작하지만 내부적으로는 메서드처럼 작동합니다. 메서드처럼 정의하지만 속성처럼 작동하는 데이터라고 생각하세요. 즉, 소괄호 대신 점 표기법으로 프로퍼티에 접근할 수 있습니다.

프로퍼티가 하는 주요 역할은 값을 가져오거나 설정하거나 삭제할 때 추가 로직을 내부에서 실행한다는 점입니다. 이 때문에 객체 내 데이터를 접근하거나 조작할 때 완벽한 선택이 됩니다.

그렇다면 왜 메서드 대신 프로퍼티를 사용할까요? 주로 가독성과 관례 때문입니다. 프로퍼티는 코드를 더 깔끔하고 읽기 쉽게 만듭니다.

메서드를 사용할 때는 항상 소괄호를 붙여 호출해야 합니다. 하지만 프로퍼티는 점 표기법으로 일반 속성처럼 접근할 수 있습니다. 덕분에 내부에서 추가 작업을 하더라도 코드가 단순해 보입니다.

예를 들어, 값을 계산하거나 새 값이 유효한지 확인한 후 저장하고 싶을 때 메서드를 호출하는 대신 속성처럼 사용할 수 있습니다.

Python에서 데코레이터는 원래 코드를 변경하지 않고 다른 함수나 클래스의 기능을 수정하는 함수입니다.

사용자 정의 데코레이터를 만드는 것도 가능하지만, 이 수업 범위를 벗어납니다. 프로퍼티를 만들려면 메서드를 정의하고 그 위에 `@property` 데코레이터를 붙이면 됩니다. 이렇게 하면 메서드가 프로퍼티가 되어 내부적으로 데코레이터가 붙은 메서드를 호출하면서 속성처럼 접근할 수 있습니다.

이제 게터를 만드는 방법을 `@property` 데코레이터와 함께 살펴보겠습니다:

```py
class Circle:
    def __init__(self, radius):
        self._radius = radius

    @property
    def radius(self): # A getter to get the radius
        return self._radius
  
    @property
    def area(self):  # A getter to calculate area
        return 3.14 * (self._radius ** 2)

my_circle = Circle(3)

print(my_circle.radius) # 3
print(my_circle.area) # 28.26
```

이 예제는 원의 반지름과 넓이를 구합니다.

클래스 내부에서 radius 대신 `_radius`를 사용한 점에 주목하세요. 밑줄은 해당 속성이 비공개임을 나타내는 Python 관례입니다. 즉, 내부 용도로만 사용하며 클래스 외부에서 직접 접근하지 말라는 의미입니다.

예를 들어 반지름을 설정하는 세터를 만들려면 같은 이름의 메서드를 하나 더 정의하고 그 위에 `@<property_name>.setter`를 붙여야 합니다:

`self.radius`를 `__init__` 안에서 사용하면 객체 생성 시 세터가 실행되어 잘못된 반지름 값이 즉시 잡힙니다.

```py
class Circle:
    def __init__(self, radius):
        self.radius = radius # Calling the setter

    @property
    def radius(self):  # A getter to get the radius
        return self._radius

    @radius.setter
    def radius(self, value):  # A setter to set the radius
        if value <= 0:
            raise ValueError('Radius must be positive')
        self._radius = value

my_circle = Circle(3)
print('Initial radius:', my_circle.radius) # Initial radius: 3

my_circle.radius = 8
print('After modifying the radius:', my_circle.radius) # After modifying the radius: 8
```

이 예제에서 반지름 세터는 단순히 반지름을 설정하는 것뿐 아니라 반지름이 음수가 아닌지 확인하는 유효성 검사를 실행합니다.

게터와 세터를 정의하면 Python은 일반 속성 구문을 사용할 때마다 내부적으로 자동 호출합니다:

```py
my_circle.radius # This will call the getter
my_circle.radius = 4 # This will call the setter
```

세터 내부에서 새 값을 할당할 때 프로퍼티 이름을 그대로 사용할 수 없다는 점에 주의하세요. 세터 내에서 `self.radius = value`가 세터를 다시 호출해 무한 재귀와 `RecursionError`가 발생하기 때문입니다. 따라서 항상 밑줄이 붙은 `self._radius = value` 형태를 사용해야 합니다.

속성에 접근하는 방식을 게터로, 수정하는 방식을 세터로 제어할 수 있듯이 삭제하는 방식을 deleter로 제어할 수도 있습니다.

deleter는 del 문으로 프로퍼티를 삭제할 때 사용자 정의 로직을 실행합니다. 만들려면 `@<property_name>.deleter` 데코레이터를 사용하세요:

```py
class Circle:
    def __init__(self, radius):
        self.radius = radius

    # Getter
    @property
    def radius(self):
        return self._radius

    # Setter
    @radius.setter
    def radius(self, value):
        if value <= 0:
            raise ValueError("Radius must be positive")
        self._radius = value

    # Deleter
    @radius.deleter
    def radius(self):
        print("Deleting radius...")
        del self._radius
```

deleter를 활용하는 방법은 다음과 같습니다:

```py
# Create circle object with a radius
my_circle = Circle(33)
print("Initial radius:", my_circle.radius)  # 33

# Delete the radius
# This calls the deleter
del my_circle.radius # Deleting radius...
print("Radius deleted!") # Radius deleted!

# Try to access radius after deletion
try:
    print(my_circle.radius)
except AttributeError as e:
    print("Error:", e) # Error: 'Circle' object has no attribute '_radius'
```

요점은 다음과 같습니다:

- 게터는 값을 가져오거나 즉석에서 계산할 수 있게 해 줍니다.
- 세터는 할당 전에 검사를 실행해 값을 안전하게 수정할 수 있게 해 줍니다.
- 프로퍼티는 게터와 세터를 묶어 점 표기법을 유지하면서 로직을 작성할 수 있게 해 줍니다.
- deleter는 속성이 삭제될 때 어떤 일이 일어날지 정의할 수 있게 해 줍니다.
