# --description--

關聯式資料庫是組織成表格的資料群集。這些表格由列和行組成。

列表述個別的記錄。例如，如果你正在儲存使用者資料，每列可以表述特定的使用者。

行表述每個記錄的屬性或欄位。該表格可能包含用於常見使用者屬性的行，例如姓名和身分證字號。

關聯式資料庫的一個主要特性是它們允許我們透過共同的屬性將不同的資料表連結起來，使我們更容易看出它們之間的關係。

關聯式資料庫需要一個 schema。schema 定義資料庫的整體結構，包括其資料表、行、資料型別、關聯和條件。

關聯式資料庫中的所有資料表都有一個主鍵，它是每一列的唯一識別符。它們透過外鍵啟用資料表之間的關係。你將在接下來的課程中學習更多關於主鍵、外鍵及其運作方式。現在，只需知道它們是跨資料表聯結記錄的基礎。

例如，假設你正在為一個經營瀕危動物野生動物保育中心的組織建立資料庫。

其中一個資料表可以儲存關於動物的一般資訊，你可以將它命名為 `animals`。每一列可以表述一隻動物，而每一行可以表述該動物的屬性，例如物種、大小、重量、年齡以及其他獨特的特徵。

為了唯一識別每隻動物，你通常會為資料表添加一個 `id` 行。這將是資料表的主鍵，因為它對每隻動物都是唯一的。每隻動物的 ID 可以用來將此資料表中的記錄與其他資料表中的相關記錄聯結。

你也可以有一個 `species` 表格來儲存每個物種的資訊，例如牠們的棲息地和保育狀況。`animals` 表格中的每個動物都可以使用物種 ID 來引用這個表格中的物種。

你也可以創建一個資料表來儲存動物的獸醫記錄。這樣，中心就能追蹤牠們的健康狀況、治療和用藥。

一旦你有了這個通用架構，你就可以將這些資料表連結到作為動物相關資料中心點的 `animals` 資料表。

這樣，你就不必將所有關於動物的資訊保存在同一個資料表中。相反地，你可以將相關的資料儲存在不同的資料表中，並透過關聯將它們連結起來。透過建立這些關聯，你可以用單一查詢取得你需要的特定資料。

查詢是向資料庫請求特定資料的動作。在我們範例的背景關係中，你可以進行的查詢範例包括尋找所有需要獸醫檢查的動物、尋找中心中最瀕危的物種、尋找在圈養中出生的動物數量，等等。你可以自訂查詢以符合你的需求。

我們的範例與野生動物保育有關，但關聯式資料庫在醫療、建築、商務、遊戲、教育、政府、電子商務、社群媒體等多個產業中有廣泛的應用。

它們提供多項優點，包括可擴充性。它們能控制大量資料，因此非常適合複雜的實際應用。它們也透過主鍵、外鍵和資料型別來強制資料完整性，確保資料的一致性和準確性。

除了關聯式資料庫，你還可以使用非關聯式資料庫。

關聯式資料庫與非關聯式資料庫之間的主要差異在於資料的儲存方式。非關聯式資料庫，也稱為 NoSQL 資料庫，更具彈性。它們將資料儲存在彼此不相連的獨立檔案中。它們的資料模型也更具彈性。

雖然某些非關聯式資料庫可能有基本的結構或資料模型，但通常比關聯式資料庫中定義的結構更不嚴格。如果有必要，你可以添加、修改或移除資料欄。

使用關聯式或非關聯式資料庫的決定取決於多種因素，包括資料的性質和應用程式的具體需求。你應該評估這兩種方法之間的權衡，以選擇最適合你的應用程式的方案。
