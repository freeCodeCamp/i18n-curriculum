# --interactive--

In JavaScript, le variabili fungono da contenitori per archiviare dati che puoi consultare e modificare durante tutto il programma.

Puoi pensare alle variabili come a dei riquadri che contengono valori. Con le variabili puoi tenere traccia di cose come numeri o testo e fare riferimento a questi valori ogni volta che ti servono nel programma.

Un modo per dichiarare una variabile in JavaScript è usare la parola chiave `let`. Imparerai di più sulla parola chiave `let` e su altri modi per dichiarare variabili nelle lezioni future.

Ecco un esempio di utilizzo di `let` per dichiarare una variabile chiamata `age`:

```js
let age;
```

Al momento, la variabile `age` non ha un valore assegnato. Se provi a usarla, restituirà `undefined`, che significa che non ha valore.

Ecco un esempio.

**NOTA**: `console.log()` è una funzione che mostra informazioni nella console, che è una parte del tuo browser web usata per eseguire il debug del codice. Imparerai di più su `console.log()` nelle lezioni future. Inoltre, i simboli `//` sono usati per aggiungere commenti nel codice. I commenti sono note per te o per altri programmatori che vengono ignorate quando il codice viene eseguito.

:::interactive_editor
 
```js
let age;
console.log(age); // undefined
```

:::

Per assegnare un valore a una variabile devi usare l’operatore di assegnazione in questo modo:

```js
let age = 25;
```

Ora, quando usi la variabile `age`, restituirà il valore `25`.

:::interactive_editor

```js
let age = 25;
console.log(age); // 25
```

:::

L’operatore di assegnazione sembra un segno di uguale (`=`) ma non verifica l’uguaglianza. Imparerai gli operatori corretti per verificare l’uguaglianza nelle lezioni future.

L’operatore di assegnazione serve per assegnare un valore a una variabile. Questo processo di assegnare un valore a una variabile si chiama inizializzazione.

Un vantaggio dell’usare la parola chiave `let` per dichiarare variabili è che puoi riassegnare valori. In programmazione, riassegnare significa dare un nuovo valore a una variabile che ne ha già uno.

Ecco un esempio di riassegnazione del valore per la variabile `age`.

:::interactive_editor

```js
let age = 25;
console.log(age); // 25
age = 30;
console.log(age); // 30
```

:::

Ora la variabile `age` contiene il valore `30`. Nota che la parola chiave `let` non è stata necessaria di nuovo perché la variabile `age` era già stata dichiarata, quindi non serve dichiararla una seconda volta.

Quando usi la riassegnazione, devi solo fare riferimento al nome della variabile. La riassegnazione è utile perché ti permette di aggiornare e cambiare il valore archiviato in una variabile mentre il programma è in esecuzione. Un buon esempio è aggiornare i punti in un gioco.

Nominare le variabili può sembrare semplice, ma ci sono alcune regole e buone pratiche per assicurarti che il codice sia leggibile e funzionale.

I nomi delle variabili dovrebbero descrivere cosa rappresentano i dati. Per esempio, invece di usare un nome come `x`, un nome più descrittivo come `age` o `points` rende il codice più facile da capire.

```js
// Bad variable names
let x = 10;
let y = "John";

// Good variable names
let age = 10;
```

Le variabili in JavaScript devono iniziare con una lettera, un underscore (`_`) o un simbolo del dollaro (`$`). Non possono iniziare con un numero.

```js
// Valid variable names
let age;
let _score;
let $total;

// Invalid variable names
let 1stPlace; // starts with a number
```

I nomi delle variabili sono case-sensitive, cioè la parola `age` tutta in minuscolo e la parola `Age` con la `A` maiuscola sono considerate variabili diverse.

:::interactive_editor

```js
let age = 25;
let Age = 30;
console.log(age); // 25
console.log(Age); // 30
```

:::

Per questo è importante seguire una convenzione di denominazione coerente come camelCase. camelCase è quando la prima parola è tutta minuscola e ogni parola successiva inizia con una lettera maiuscola.

Ecco un esempio di utilizzo della convenzione camelCase per una variabile:

```js
let thisIsCamelCase;
let anotherExampleVariable;
let freeCodeCampStudents;
```

Ci sono alcune parole chiave in JavaScript che non puoi usare come nomi di variabili, come `let`, `const`, `function` o `return`, perché sono riservate al linguaggio stesso.

Dovresti anche evitare di usare caratteri speciali come punti esclamativi (`!`) o simboli at (`@`) nei nomi delle variabili. È meglio mantenere i nomi delle variabili leggibili usando lettere, numeri, underscore o simboli del dollaro.

Seguendo queste linee guida, il tuo codice sarà più pulito e più facile da gestire man mano che cresce in complessità.
