# --description--

In einer vorherigen Lektion haben Sie gelernt, wie man Attribute mit einem einzelnen Unterstrich und einem doppelten Unterstrich versieht.

Zur Erinnerung: Ein einzelner Unterstrich ist eine Konvention, die bedeutet, dass das Attribut nur für die interne Verwendung in der Klasse gedacht ist und nicht direkt von außerhalb der Klasse aufgerufen werden sollte. Ein doppelter Unterstrich hingegen verhindert, dass dieses Attribut direkt von außerhalb der Klasse aufgerufen wird.

Hier ist ein Beispiel, das zeigt, wie beide funktionieren:

```py
class Example:
    def __init__(self):
        self._internal = 'I can be accessed from outside the class, but should not'
        self.__private = 'You cannot access me directly from outside the class'

obj = Example()

print(obj._internal) # I can be accessed from outside the class, but should not
print(obj.__private)  # AttributeError: 'Example' object has no attribute '__private'
```

Das Voranstellen eines Attributs mit einem doppelten Unterstrich löst Pythons Name-Mangling-Prozess aus, bei dem Python das Attribut intern umbenennt, indem es einen Unterstrich und den Klassennamen als Präfix hinzufügt und so `__attribute` in `_ClassName__attribute` verwandelt.

Um das in Aktion zu sehen, erstellen Sie eine Instanz der Klasse und verwenden das spezielle `__dict__`-Attribut dieser Instanz, das ein Wörterbuch mit den Attributen des Objekts enthält:

```py
class Example:
    def __init__(self, internal, private):
        self._internal = internal
        self.__private = private

example1 = Example(
    'I can be accessed from outside the class, but should not',
    'I cannot be accessed directly from outside the class'
)

print(example1.__dict__)
```

Das Ergebnis wäre:

```py
{
  '_internal': 'I can be accessed from outside the class, but should not',
  '_Example__private': 'I cannot be accessed directly from outside the class'
}
```

Wie Sie sehen, wird das `__private`-Attribut als `_Example__private` gespeichert. Das bedeutet, Sie können dieses Attribut außerhalb der Klasse auf diese Weise weiterhin aufrufen:

```py
class Example:
    def __init__(self, internal, private):
        self._internal = internal
        self.__private = private

example1 = Example(
    'I can be accessed from outside the class, but should not',
    'I cannot be accessed directly from outside the class'
)
example2 = Example(
    'I should not be accessed from outside the class',
    'But I can be accessed from outside the class with name mangling'
)

print(example1._Example__private) # I cannot be accessed directly from outside the class
print(example2._Example__private) # But I can be accessed from outside the class with name mangling
```

Warum macht Python also Name Mangling?

Der Hauptzweck des Name Manglings ist es, versehentliches Überschreiben von Attributen und Methoden zu verhindern, wenn Sie Vererbung verwenden. Hier ist ein Beispiel, das das verdeutlicht:

```py
class Parent:
    def __init__(self):
        self.__data = 'Parent data'

class Child(Parent):
    def __init__(self):
        super().__init__()
        self.__data = 'Child data'

c = Child()
print(c.__dict__) # {'_Parent__data': 'Parent data', '_Child__data': 'Child data'}
```

Sie sehen, dass sowohl die `Parent`-Klasse als auch die `Child`, die von ihr erbt, jeweils eigene `_class__data`-Attribute haben. Das wird durch Name Mangling ermöglicht. Andernfalls hätte `Child` versehentlich die Daten der Elternklasse überschrieben.

So sähe es aus, wenn Python das Name Mangling nicht durchführen würde, also wenn Sie die Attribute in beiden Klassen nicht mit doppeltem Unterstrich versehen:

```py
class Parent:
   def __init__(self):
       self.data = 'Parent data'

class Child(Parent):
   def __init__(self):
       super().__init__()
       self.data = 'Child data'

c = Child()
print(c.__dict__)  # {'data': 'Child data'}
```

Welche Variante sollten Sie also verwenden, um Attribute zu präfixen: einen einzelnen Unterstrich (`_`) oder einen doppelten Unterstrich (`__`)? Das hängt davon ab. Wenn ein Attribut nur für die interne Verwendung innerhalb der Klasse gedacht ist, verwenden Sie einen einzelnen Unterstrich.

Wenn Sie jedoch mit einer Klasse arbeiten, die vererbt wird, sollten Sie einen doppelten Unterstrich verwenden, damit das Attribut der Elternklasse nicht überschrieben wird.
