# --description--

DOM 노드와 DOM 트리에서의 관계에 대해 배워보세요.

실제 나무가 크고 작은 가지들이 계층 구조로 연결되어 있듯이, DOM 노드들도 서로 직접적 또는 간접적인 관계를 가집니다. 이 예제를 통해 이러한 관계를 설명하겠습니다:

```html
<!DOCTYPE html>
<html>
  <head>
    <title>DOM Tree Example</title>
  </head>
  <body>
    <h1>Heading 1</h1>
    <p>Paragraph 1</p>
    <ul>
      <li>List item 1</li>
      <li>List item 2</li>
    </ul>
  </body>
</html>
```

맨 위부터 시작해 봅시다. DOM 트리의 루트는 `html` 요소입니다. HTML 문서의 모든 내용을 담는 최상위 컨테이너입니다. 다른 모든 노드는 이 루트 노드의 자손 요소입니다.

그다음 루트 노드 아래에는 계층 구조상 다른 노드들이 있습니다. 부모 노드는 다른 요소들을 포함하는 요소입니다. 자식 노드는 다른 요소 안에 포함된 요소입니다.

이 예제에서 `body` 요소는 문단 요소의 부모이고, 문단 요소는 `body` 요소의 자식입니다. 이 요소들은 DOM 트리에서 같은 관계로 노드로 나타납니다.

부모 노드와 자식 노드가 있듯이, 형제 노드도 있습니다. 형제 노드는 같은 부모를 공유하는 요소입니다. 예제에서 두 개의 목록 항목 요소는 같은 순서 없는 목록 부모를 공유하므로 형제입니다.

마찬가지로 `h1` 요소와 문단 요소도 같은 부모인 `body` 요소를 공유하므로 형제입니다.

또한 계층 구조의 서로 다른 수준 사이에 간접적인 관계도 있습니다. 자손 노드는 다른 요소 안에 직접 또는 간접적으로 포함된 요소입니다. 어떤 요소가 다른 요소의 자손으로 간주되려면, 조상 노드에서 DOM 트리를 따라 내려가서 도달할 수 있어야 합니다.

예제에서 목록 항목 `li` 요소들은 `body` 요소의 자손입니다. 왜냐하면 간접적으로 그 안에 포함되어 있기 때문입니다.

조상 노드는 다른 요소보다 DOM 트리 계층에서 더 위에 있는 요소입니다. 예제에서 `body` 요소는 목록 항목 `li` 요소들의 조상입니다.

이러한 관계를 이해하는 것은 JavaScript로 DOM 트리를 조작하고 탐색하는 데 필수적입니다.
