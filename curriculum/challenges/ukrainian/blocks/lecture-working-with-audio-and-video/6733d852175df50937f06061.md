# --description--

Замість відтворення аудіо та відео, іноді ви можете захотіти захопити аудіо або відео. API захоплення медіа та потоків, або MediaStream API, дозволяє це зробити.

Щоб використати цей API, вам потрібно створити об’єкт `MediaStream`. Ви могли б зробити це за допомогою конструктора, але він не буде пов’язаний із апаратним забезпеченням користувача. Замість цього властивість `mediaDevices` глобального об’єкта `navigator` має метод `getUserMedia()`, який ви можете використати.

Цей метод приймає один об’єкт `constraints`, який визначає тип медіа, яке ви хочете отримати. Цей об’єкт має властивості `audio` та `video`, що відображають аудіо- та відеопотоки. Ці властивості можуть бути `false`, якщо ви не хочете отримувати цей тип потоку, `true`, якщо хочете, або об’єктами, що визначають додаткові обмеження.

Наприклад, ви можете вимагати конкретну роздільність відео:

```js
window.navigator.mediaDevices.getUserMedia({
  audio: true,
  video: {
    width: {
      min: 1280,
      ideal: 1920,
      max: 3840
    },
    height: {
      min: 720,
      ideal: 1080,
      max: 2160
    }
  }
});
```

Цей об’єкт обмежень задає мінімальну та максимальну роздільність для відеопотоку. Властивість `ideal` вказує роздільність, яку ви найбільше бажаєте мати — і потік надасть роздільність, найближчу до вашої ідеальної.

Після створення MediaStream (за умови, що користувач схвалить автоматизований запит на доступ до свого апаратного забезпечення), ви можете використовувати дані потоку так, як вам потрібно.

Зверніть увагу, що `getUserMedia()` повертає проміс, а це означає, що вам знадобиться функція зворотного виклику для обробки потоку або використання синтаксису async/await. Ви дізнаєтеся більше про проміси та асинхронне програмування в майбутніх уроках.

Ось базовий приклад, який відтворює відеопотік з вебкамери користувача на сторінці.

```js
const video = document.querySelector("video");
const stream =
  await window.navigator.mediaDevices.getUserMedia({ video: true });
video.srcObject = stream;
await video.play();
```

Варто зазначити, що цей API не надає доступу до захоплення екрану. Ви дізнаєтеся про цей API в наступному уроці.
