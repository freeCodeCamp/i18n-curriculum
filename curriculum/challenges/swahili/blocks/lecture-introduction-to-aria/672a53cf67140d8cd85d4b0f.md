# --interactive--

Kufanya maudhui yasiyobadilika kupatikana kwa urahisi kunaweza kuwa rahisi, lakini maudhui yanayobadilika mara kwa mara yanaweza kuwa changamoto zaidi. Hapa ndipo WAI-ARIA inapoingia.

Tuchunguze ni nini WAI-ARIA, kusudi lake, jinsi inavyofanya kazi, na mifano kadhaa.

WAI-ARIA inasimama kwa Web Accessibility Initiative - Accessible Rich Internet Applications. Ni maelekezo yanayoongeza ufikikaji kwa maudhui yanayobadilika na sehemu za kiolesura cha mtumiaji (UI).

Kumbuka kuwa WCAG na WAI-ARIA si sawa. WCAG hutoa miongozo ya jumla kwa ufikikaji wa mtandao, wakati WAI-ARIA hutoa sheria maalum za kufanya maudhui yanayobadilika na ya mwingiliano kupatikana kwa watumiaji wa teknolojia saidizi.

Kwa hivyo, kusudi kuu la WAI-ARIA ni kuboresha ufikikaji kwa maudhui yanayobadilika na sehemu za kiolesura cha mtumiaji ambazo hazina sawa asilia za HTML.

WAI-ARIA hufanya kazi kwa kuanzisha seti ya sifa za kipengele unazoweza kuongeza kwenye vipengele vya HTML ili kutoa taarifa za maana zaidi. Sifa hizi za kipengele zimegawanywa katika majukumu, hali, na sifa.

Jukumu la ARIA linafafanua jukumu la kipengele ndani ya tovuti au programu ya mtandao. Hapa kuna mfano wa kuweka jukumu kuwa `button` kwa kipengele cha `div`.

```html
<div role="button">Click Me</div>
```

Kwa kufanya hivi unaonyesha kwa teknolojia saidizi kwamba kipengele ni kitufe. Hata hivyo, majukumu hayatoi kitendakazi chochote. Kutoa tu `div` hii `role` ya `button` haitafanya itende kama kitufe. Ili ionekane na itende kama kitufe, unahitaji kutumia CSS na JavaScript kupata matokeo unayotaka.

Hapa kuna mfano wa kutumia HTML, CSS na JavaScript kuunda kipengele maalum cha `button`.

**KUMBUKA**: Usijali kuelewa msimbo wa CSS na JavaScript. Utajifunza lugha hizo katika moduli zijazo.

:::interactive_editor

```html
<link href="styles.css" rel="stylesheet">
<div id="custom-btn" role="button">Click Me</div>
<script src="index.js"></script>
```

```css

```

```js
const button = document.getElementById("custom-btn");

button.addEventListener("click", () => {
  alert("Button clicked!");
});

button.addEventListener("keydown", (e) => {
  if (e.key === "Enter" || e.key === " ") {
    e.preventDefault(); 
    button.click();
  }
});
```

Daima ni bora kutumia kipengele asilia cha `button` au `input` pamoja na `type="button"` badala yake.

Sifa za ARIA hutoa maelezo ya ziada kuhusu vipengele. Kwa mfano, sifa ya `aria-labelledby` inakuwezesha kuunganisha kipengele na lebo maalum:

```html
<h2 id="header-id">About freeCodeCamp</h2>
<button id="button-id" aria-labelledby="header-id button-id">Learn More</button>
```

Hii itafanya vipengele kueleweka na kuvinuliwa kwa watumiaji wa teknolojia saidizi.

Ili kupata faida bora kutoka kwa WAI-ARIA, jaribu kutumia HTML asilia kadri inavyowezekana kwa sababu kwa kawaida hutoa ufikikaji zaidi moja kwa moja.

Tumia WAI-ARIA tu wakati HTML haifanyi kazi ipasavyo, na usisahau kujaribu na teknolojia saidizi kama kisomaji cha skrini, au kuwa na watu wenye ulemavu wakajaribu kazi yako. Pia, hakikisha hali na vigezo vya WAI-ARIA vinasasishwa pamoja na maudhui kwa wakati halisi. Epuka kutumia ARIA kupita kiasi, kwani mara nyingi inaweza kuchanganya.
