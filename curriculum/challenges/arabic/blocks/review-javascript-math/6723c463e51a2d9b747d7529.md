# --interactive--

## العمل مع نوع بيانات Number

- **تعريف**: نوع `Number` في JavaScript يشمل الأعداد الصحيحة، الأعداد العشرية، `Infinity` و`NaN`. الأعداد العشرية هي أعداد تحتوي على فاصلة عشرية. `Infinity` الموجب هو عدد أكبر من أي عدد آخر، بينما `-Infinity` هو عدد أصغر من أي عدد آخر. `NaN` (`Not a Number`) يمثل قيمة رقمية غير صالحة مثل السلسلة النصية `"Jessica"`.

## العمليات الحسابية الشائعة

- **معامل الجمع**: يستخدم هذا المعامل (`+`) لحساب مجموع رقمين أو أكثر.
- **معامل الطرح**: يستخدم هذا المعامل (`-`) لحساب الفرق بين رقمين.
- **معامل الضرب**: يستخدم هذا المعامل (`*`) لحساب حاصل ضرب رقمين أو أكثر.
- **معامل القسمة**: يستخدم هذا المعامل (`/`) لحساب حاصل القسمة بين رقمين.
- **القسمة على صفر**: إذا حاولت القسمة على صفر، سترجع JavaScript القيمة `Infinity`.
- **معامل الباقي**: يعيد هذا المعامل (`%`) باقي القسمة.
- **معامل الأس**: يرفع هذا المعامل (`**`) رقمًا إلى قوة رقم آخر.

## الحسابات مع الأعداد والسلاسل النصية

- **الشرح**: عند استخدام المعامل `+` مع رقم وسلسلة نصية، تجبر JavaScript الرقم على التحول إلى سلسلة نصية وتدمج القيمتين. عند استخدام المعاملات `-`، `*` أو `/` مع سلسلة نصية ورقم، تجبر JavaScript السلسلة النصية على التحول إلى رقم وتكون النتيجة رقمًا. بالنسبة لـ `null` و `undefined`، تعامل JavaScript `null` كـ 0 و `undefined` كـ `NaN` في العمليات الحسابية.

:::interactive_editor

```js
const result = 5 + '10';

console.log(result); // "510"
console.log(typeof result); // string

const subtractionResult = '10' - 5;
console.log(subtractionResult); // 5
console.log(typeof subtractionResult); // number

const multiplicationResult = '10' * 2;
console.log(multiplicationResult); // 20
console.log(typeof multiplicationResult); // number

const divisionResult = '20' / 2;
console.log(divisionResult); // 10
console.log(typeof divisionResult); // number

const result1 = null + 5;
console.log(result1); // 5
console.log(typeof result1); // number

const result2 = undefined + 5;
console.log(result2); // NaN
console.log(typeof result2); // number
```

:::

## أسبقية المعاملات

- **التعريف**: تحدد أسبقية المعاملات ترتيب تقييم العمليات في التعبير. تُقيّم المعاملات ذات الأسبقية الأعلى قبل تلك ذات الأسبقية الأقل. تُقيّم القيم داخل الأقواس أولًا، وتكون أسبقية الضرب والقسمة أعلى من الجمع والطرح. إذا كانت المعاملات لها نفس الأسبقية، تستخدم JavaScript الترابط.

:::interactive_editor

```js
const result = (2 + 3) * 4;

console.log(result); // 20

const result2 = 10 - 2 + 3;

console.log(result2); // 11

const result3 = 2 ** 3 ** 2;

console.log(result3); // 512
```

:::

- **التعريف**: الترابط يحدد اتجاه تقييم التعبير عندما توجد معاملات متعددة من نفس النوع. يحدد ما إذا كان يجب تقييم التعبير من اليسار إلى اليمين (`left-associative`) أو من اليمين إلى اليسار (`right-associative`). على سبيل المثال، معامل الأس هو أيضًا ترابط من اليمين إلى اليسار:

:::interactive_editor

```js
const result4 = 5 ** 4 ** 1; 

console.log(result4); // 625
```

:::

## معاملات التزايد والتقليل

- **معامل التزايد**: يستخدم هذا المعامل لزيادة القيمة بمقدار واحد. الصيغة البادئة `++num` تزيد قيمة المتغير أولًا ثم تُرجع القيمة الجديدة. الصيغة اللاحقة `num++` تُرجع القيمة الحالية للمتغير أولًا ثم تزيدها.

:::interactive_editor

```js
let x = 5;

console.log(++x); // 6
console.log(x); // 6


let y = 5;

console.log(y++); // 5
console.log(y); // 6
```

:::

- **معامل التقليل / معامل التخفيض**: يستخدم هذا المعامل لتقليل القيمة بمقدار واحد. تعمل الصيغتان البادئة واللاحقة بنفس طريقة معامل التزايد.

:::interactive_editor

```js
let num = 5;

console.log(--num); // 4
console.log(num--); // 4
console.log(num); // 3
```

:::

## معاملات التعيين المركب

- **معامل التعيين الجمعي (`+=`)**: يجري هذا المعامل عمليات الجمع على القيم ويُعيّن النتيجة للمتغير.
- **معامل التعيين الطرحي (`-=`)**: يجري هذا المعامل عمليات الطرح على القيم ويُعيّن النتيجة للمتغير.
- **معامل التعيين الضربي (`*=`)**: يجري هذا المعامل عمليات الضرب على القيم ويُعيّن النتيجة للمتغير.
- **معامل التعيين القسمي (`/=`)**: يجري هذا المعامل عمليات القسمة على القيم ويُعيّن النتيجة للمتغير.
- **معامل التعيين بباقي القسمة (`%=`)**: يقسم هذا المعامل المتغير على الرقم المحدد ويُعيّن الباقي للمتغير.
- **معامل التعيين الأسّي (`**=`)**: يرفع هذا المعامل المتغير إلى قوة الرقم المحدد ويُعيد تعيين النتيجة للمتغير.

## القيم المنطقية والمساواة

- **تعريف القيمة المنطقية**: القيمة المنطقية هي نوع بيانات يمكن أن يحتوي فقط على قيمتين: `true` أو `false`.
- **معامل المساواة (`==`)**: يستخدم هذا المعامل إجبار النوع قبل التحقق من تساوي القيم.

:::interactive_editor

```js
console.log(5 == '5'); // true
```

:::

- **معامل المساواة الصارمة (`===`)**: لا يجري هذا المعامل إجبار النوع ويتحقق من تساوي النوعين والقيم معًا.

:::interactive_editor

```js
console.log(5 === '5'); // false
```

:::

- **معامل عدم المساواة (`!=`)**: يستخدم هذا المعامل إجبار النوع قبل التحقق من عدم تساوي القيم.
- **معامل عدم المساواة الصارمة (`!==`)**: لا يجري هذا المعامل إجبار النوع ويتحقق من عدم تساوي النوعين والقيم معًا.

## معاملات المقارنة

- **معامل أكبر من (`>`)**: يتحقق هذا المعامل مما إذا كانت القيمة على اليسار أكبر من القيمة على اليمين.
- **معامل أكبر من أو يساوي (`>=`)**: يتحقق هذا المعامل مما إذا كانت القيمة على اليسار أكبر من أو تساوي القيمة على اليمين.
- **معامل أقل من (`<`)**: يتحقق هذا المعامل مما إذا كانت القيمة على اليسار أقل من القيمة على اليمين.
- **معامل أقل من أو يساوي (`<=`)**: يتحقق هذا المعامل مما إذا كانت القيمة على اليسار أقل من أو تساوي القيمة على اليمين.

## المعاملات الأحادية

- **معامل الجمع الأحادي**: يحول هذا المعامل المعامل إلى رقم. إذا كان المعامل رقمًا بالفعل، يبقى كما هو.

:::interactive_editor

```js
const str = '42';
const num = +str;

console.log(num); // 42
console.log(typeof num); // number
```

:::

- **معامل النفي الأحادي (`-`)**: ينفي هذا المعامل المعامل.

:::interactive_editor

```js
const num = 4;
console.log(-num); // -4
```

:::

- **معامل النفي المنطقي (`!`)**: يعكس هذا المعامل القيمة المنطقية للمعامل. فإذا كانت القيمة `true` تصبح `false`، وإذا كانت `false` تصبح `true`.

## المعاملات الثنائية على مستوى البت

- **عامل `&` (bitwise AND)**: يُرجع هذا العامل 1 في كل خانة بت حيث تكون الخانات المقابلة لكلا العاملين 1.
- **عامل التعيين `&=` (bitwise AND Assignment)**: يُنفذ هذا العامل عملية `bitwise AND` مع الرقم المحدد ويُعيد تعيين النتيجة إلى المتغير.
- **عامل `|` (bitwise OR)**: يُرجع هذا العامل 1 في كل خانة بت حيث تكون الخانات المقابلة لأحد العاملين أو كلاهما 1.
- **عامل التعيين `|=` (bitwise OR Assignment)**: يُنفذ هذا العامل عملية `bitwise OR` مع الرقم المحدد ويُعيد تعيين النتيجة إلى المتغير.
- **عامل `^` (bitwise XOR)**: يُرجع هذا العامل 1 في كل خانة بت حيث تكون الخانات المقابلة لأحد العاملين فقط 1، وليس كلاهما.
- **عامل `~` (bitwise NOT)**: يعكس هذا العامل التمثيل الثنائي للرقم.
- **عامل الإزاحة لليسار `<<` (Left Shift)**: يحرك هذا العامل كل البتات إلى اليسار بعدد محدد من المواقع.
- **عامل الإزاحة لليمين `>>` (Right Shift)**: يحرك هذا العامل كل البتات إلى اليمين.

## العبارات الشرطية، القيم الصحيحة، القيم الخاطئة والمعامل الثلاثي

- **`if/else if/else`**: يأخذ تعبير `if` شرطًا وينفذ كتلة شفرة إذا كان هذا الشرط `truthy`. إذا كان الشرط `false`، ينتقل إلى البند `else if`. إذا لم يكن أي من هذه الشروط `true`، ينفذ البند `else`. القيم `Truthy` هي أي قيم تُقيّم إلى `true` في سياق القيمة المنطقية مثل تعبير `if`. القيم `Falsy` هي القيم التي تُقيّم إلى `false` في سياق القيمة المنطقية.

:::interactive_editor

```js
const score = 87;

if (score >= 90) {
 console.log('You got an A'); 
} else if (score >= 80) {
 console.log('You got a B'); // You got a B
} else if (score >= 70) {
 console.log('You got a C');
} else {
 console.log('You failed! You need to study more!');
}
```

:::

- **المعامل الثلاثي**: يُستخدم هذا المعامل غالبًا كطريقة مختصرة لكتابة عبارات `if else`.

:::interactive_editor

```js
const temperature = 30;
const weather = temperature > 25 ? 'sunny' : 'cool';

console.log(`It's a ${weather} day!`); // It's a sunny day!
```

:::

## المعاملات المنطقية الثنائية

- **عامل AND المنطقي (`&&`)**: يتحقق هذا العامل مما إذا كانت كلتا القيمتين `true`. إذا كانت كلتاهما `true`، فسيُعيد القيمة الثانية. إذا كانت إحدى القيمتين `falsy`، فسيُعيد القيمة `falsy`. إذا كانت كلتا القيمتين `falsy`، فسيُعيد أول قيمة `falsy`.

:::interactive_editor

```js
const result = true && 'hello';

console.log(result); // hello
```

:::

- **عامل OR المنطقي (`||`)**: يتحقق هذا العامل مما إذا كان أحد المعاملين على الأقل صحيحًا (truthy). 
- **عامل الدمج الصفري (`??`)**: يعيد هذا العامل قيمة فقط إذا كانت القيمة الأولى `null` أو `undefined`.

:::interactive_editor

```js
const userSettings = {
 theme: null,
 volume: 0,
 notifications: false,
};

let theme = userSettings.theme ?? 'light';
console.log(theme); // light
```

:::

## كائن `Math`

- **طريقة `Math.random()`**: تولد هذه الطريقة رقمًا عشريًا عشوائيًا بين 0 (شامل) و1 (غير شامل). هذا يعني أن الناتج قد يكون 0، لكنه لن يصل أبدًا إلى 1.
- **طريقة `Math.max()`**: تأخذ هذه الطريقة مجموعة أرقام وترجع القيمة القصوى.
- **طريقة `Math.min()`**: تأخذ هذه الطريقة مجموعة أرقام وترجع القيمة الأدنى.
- **طريقة `Math.ceil()`**: تقرب هذه الطريقة القيمة إلى أقرب عدد صحيح أكبر أو يساويها.
- **طريقة `Math.floor()`**: تقرب هذه الطريقة القيمة إلى أقرب عدد صحيح أقل أو يساويها.
- **طريقة `Math.round()`**: تقرب هذه الطريقة القيمة إلى أقرب عدد صحيح.

:::interactive_editor

```js
console.log(Math.round(2.3)); // 2
console.log(Math.round(4.5)); // 5
console.log(Math.round(4.8)); // 5
```

:::

- **طريقة `Math.trunc()`**: تزيل هذه الطريقة الجزء العشري من الرقم، وتُرجع الجزء الصحيح فقط بدون تقريب.
- **طريقة `Math.sqrt()`**: تُرجع هذه الطريقة الجذر التربيعي لرقم.
- **طريقة `Math.cbrt()`**: تُرجع هذه الطريقة الجذر التكعيبي لرقم.
- **طريقة `Math.abs()`**: تُرجع هذه الطريقة القيمة المطلقة لرقم.
- **طريقة `Math.pow()`**: تأخذ هذه الطريقة رقمين وترفع الأول إلى قوة الثاني.

## طرق الأعداد الشائعة

- **`isNaN()`**: تعني `NaN` "ليس عددًا". هي قيمة خاصة تمثل نتيجة رقمية غير قابلة للتمثيل أو غير معرفة. تُستخدم دالة `isNaN()` لتحديد ما إذا كانت القيمة `NaN` أم لا. توفر `Number.isNaN()` طريقة أكثر موثوقية للتحقق من قيم `NaN`، خاصة في الحالات التي قد يؤدي فيها إجبار النوع إلى نتائج غير متوقعة مع الدالة العامة `isNaN()`.

:::interactive_editor

```js
console.log(isNaN(NaN));       // true
console.log(isNaN(undefined)); // true
console.log(isNaN({}));        // true

console.log(isNaN(true));      // false
console.log(isNaN(null));      // false
console.log(isNaN(37));        // false


console.log(Number.isNaN(NaN));        // true
console.log(Number.isNaN(Number.NaN)); // true
console.log(Number.isNaN(0 / 0));      // true

console.log(Number.isNaN("NaN"));      // false
console.log(Number.isNaN(undefined));  // false
```

:::

- **طريقة `parseFloat()`**: تحلل هذه الطريقة معلمة سلسلة نصية وتُرجع عددًا عشريًا عائمًا. صُممت لاستخراج رقم من بداية السلسلة النصية، حتى لو احتوت السلسلة على أحرف غير رقمية لاحقًا.
- **طريقة `parseInt()`**: تحلل هذه الطريقة معلمة سلسلة نصية وتُرجع عددًا صحيحًا. تتوقف `parseInt()` عن التحليل عند أول توقف غير رقمي تواجهه. بالنسبة للأعداد العشرية العائمة، تُرجع الجزء الصحيح فقط. إذا لم تجد عددًا صحيحًا صالحًا في بداية السلسلة، تُرجع `NaN`.
- **طريقة `toFixed()`**: تُستدعى هذه الطريقة على رقم وتأخذ معلمة اختيارية واحدة، وهي عدد الأرقام التي تظهر بعد الفاصلة العشرية. تُرجع تمثيلًا نصيًا للرقم بعدد الأماكن العشرية المحددة.
