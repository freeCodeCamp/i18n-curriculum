# --description--

Ora sei pronto per muovere i primi passi nel mondo di Node.js installandolo sul tuo computer. Ma prima, prima di installare Node.js, ti consigliamo di installare uno strumento chiamato NVM (Node Version Manager). Come suggerisce il nome, NVM è uno strumento per gestire più versioni di Node.js sullo stesso computer.

Configurare questo strumento fin dall’inizio ti sarà utile in futuro perché potresti dover passare da una versione all’altra di Node.js per progetti diversi. Con NVM puoi farlo molto facilmente.

Vediamo come puoi installare NVM su macOS/Linux e Windows, e come usarlo per installare Node.js.

## Come installare NVM su macOS/Linux

Su macOS e Linux puoi installare NVM direttamente tramite uno script shell.

Ma per macOS, nota che la documentazione ufficiale di NVM menziona che “devi installare manualmente gli strumenti da riga di comando di Xcode prima di eseguire lo script di installazione, altrimenti fallirà.”

Su macOS puoi installare gli strumenti da riga di comando di Xcode eseguendo questo comando nel terminale:

```bash
xcode-select --install
```

Una volta terminata l’installazione, puoi controllare il numero di versione e verificare l’installazione con questo comando:

```bash
xcode-select --version
```

Poi, è il momento di installare NVM. Puoi scaricare ed eseguire lo script di installazione di NVM eseguendo uno di questi comandi:

```bash
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.3/install.sh | bash
```

```bash
wget -qO- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.3/install.sh | bash
```

Per ottenere l’ultima versione di NVM al momento in cui leggi questo articolo, consulta la documentazione ufficiale di NVM: `https://github.com/nvm-sh/nvm?tab=readme-ov-file#install--update-script`. Lì troverai questi comandi con il numero di versione più aggiornato.

Potresti dover riavviare il terminale al termine del processo per caricare il nuovo comando e la nuova configurazione.

Poi, esegui questo comando per confermare che NVM è stato installato correttamente e per controllare la versione attuale:

```bash
nvm -v
```

Dovresti vedere la tua versione di NVM nell’output.

## Come installare Node.js con NVM su macOS/Linux

Ora che hai installato NVM su macOS/Linux, puoi installare Node.js.

Per installare l’ultima versione Long-Term Support (LTS) di Node.js, esegui questo comando:

```bash
nvm install --lts
```

Nota il flag `--lts`. Questo flag indica che vuoi installare la versione Long-Term Support (LTS) corrente di Node.js. Questa versione è solitamente consigliata perché dà priorità a stabilità, affidabilità e sicurezza rispetto a funzionalità nuove o sperimentali. È ampiamente testata e pronta per la produzione. Ha anche periodi di supporto più lunghi con un focus su correzioni di bug e patch di sicurezza.

In alternativa, puoi installare una versione specifica di Node.js specificando il numero di versione nel comando:

```bash
nvm install <version>
```

Ad esempio, ecco come puoi installare Node.js v20:

```bash
nvm install 20
```

Una volta terminata l’installazione di una versione di Node.js, puoi passare a quella versione con questo comando:

```bash
nvm use <version>
```

Ad esempio, ecco come passare alla versione 20:

```bash
nvm use 20
```

Infine, puoi controllare la versione corrente di Node.js con questo comando:

```bash
node -v
```

## Come installare NVM su Windows

Ora vediamo come puoi installare e usare NVM su Windows.

Le versioni ufficiali di NVM supportano Windows solo in casi specifici. Se hai configurato Windows Subsystem for Linux (WSL) sul tuo dispositivo, puoi seguire la guida per macOS/Linux sopra per installare e usare NVM.

Tuttavia, per Windows stesso, ci sono alcune alternative. Una di queste è `nvm-windows`, un’utilità per la gestione delle versioni di Node.js per Windows. Questo è un progetto completamente separato dal progetto ufficiale NVM. Ma lo scopo di questi strumenti è lo stesso — aiutarti a lavorare con più versioni di Node.js sullo stesso dispositivo.

Prima di installare `nvm-windows`, devi sapere che il repository ufficiale raccomanda di disinstallare qualsiasi installazione precedente di Node prima di installare NVM per Windows. Se è la prima volta che installi Node.js, dovresti essere a posto. Se hai già altre versioni di Node.js installate, è consigliato disinstallarle prima.

Per installare `nvm-windows`:

- Vai alla sezione Releases del repository ufficiale: `https://github.com/coreybutler/nvm-windows/releases`.
- Scarica l’installer più recente.
- Segui le istruzioni.  

Dopo aver completato l’installazione, potresti dover riavviare il terminale se il comando `nvm` non è subito disponibile.

Puoi verificare se l’installazione è andata a buon fine e vedere la tua versione attuale di `nvm-windows` eseguendo questo comando:

```powershell
nvm -v
```

## Come installare Node.js usando NVM per Windows

Ora che hai `nvm-windows` installato su Windows, sei pronto per installare Node.js.

Per installare l’ultima versione Long-Term Support (LTS) di Node.js, esegui questo comando:

```powershell
nvm install lts
```

La versione Long-Term Support (LTS) è solitamente consigliata, ma puoi installare una versione diversa specificando il numero di versione nel comando:

```powershell
nvm install <version>
```

Ad esempio:

```powershell
nvm install 20
```

Quando l’installazione è completata, puoi passare a quella versione di Node.js con questo comando:

```powershell
nvm use <version>
```

Ad esempio:

```powershell
nvm use 20
```

Puoi anche controllare la tua versione corrente di Node.js con questo comando:

```bash
node -v
```

## Comandi base

Ora che sai come installare NVM e Node.js, vediamo alcuni comandi base che ti saranno utili quando lavori con Node.js e NVM.

Inoltre, mentre tutti questi comandi funzioneranno per NVM installato su macOS/Linux, incluso WSL, potrebbero funzionare in modo diverso o non funzionare affatto per `nvm-windows`. Lasceremo comandi alternativi per `nvm-windows` quando possibile.

### Come modificare la policy di esecuzione di Powershell

Se usi Windows e stai seguendo con Powershell e `nvm-windows`, potresti dover modificare la policy di esecuzione di Powershell. La policy di esecuzione predefinita per Powershell è `Restricted`, che è una misura di sicurezza per impedire l’esecuzione di script.

Per prima cosa, esegui Powershell come amministratore cliccando con il tasto destro su Powershell, poi cliccando su “Esegui come amministratore“.

Poi, per vedere la policy di esecuzione attuale della sessione Powershell corrente, esegui questo comando:

```powershell
Get-ExecutionPolicy
```

Infine, per impostare la policy di esecuzione di Powershell su `RemoteSigned`, che permette l’esecuzione di script locali e `npm`, usa questo comando:

```powershell
Set-ExecutionPolicy RemoteSigned
```

Dopo aver eseguito il comando sopra, puoi chiudere la finestra di Powershell amministratore e aprire una finestra di Powershell senza privilegi amministrativi.

### Elencare tutte le versioni

Per elencare tutte le versioni di Node.js installate e controllare quella attualmente in uso, esegui questo comando:

```bash
nvm ls
```

### Impostare la versione predefinita di Node.js (solo NVM su macOS/Linux)

Questo comando imposta la versione predefinita di Node.js che sarà usata per nuove sessioni terminale:

```bash
nvm alias default <version>
```

Ad esempio:

```bash
nvm alias default 22.20.0
```

Puoi anche creare alias personalizzati per assegnare un nome più specifico a una versione:

```bash
nvm alias <new_name> <version>
```

Ad esempio:

```bash
nvm alias new-api 18.20.8
```

Dopo aver impostato l’alias, puoi usare questo nuovo nome per passare a quella versione:

```bash
nvm use <new_name>
```

Ad esempio:

```bash
nvm use new-api
```

### Rimuovere una versione di Node.js

Per rimuovere una versione di Node.js, esegui questo comando:

```bash
nvm uninstall <version>
```

Ad esempio:

```bash
nvm uninstall 20.19.5
```

Con NVM per macOS/Linux, puoi rimuovere una versione di Node.js eseguendo il comando `uninstall` con una versione principale come `22` o `20`. Ad esempio:

```bash
nvm uninstall 20
```

Se hai più release per quella versione principale installate come `20.14.0` e `20.12.2`, la versione più recente sarà rimossa per prima. In questo caso, eseguire `nvm uninstall 20` rimuoverà prima `20.14.0`.

### Controllare la versione di Node.js

Per controllare la tua versione corrente di Node.js, puoi usare:

```bash
node -v
```

### Controllare la versione di npm

`npm` è un gestore di pacchetti per Node.js. Puoi usarlo per installare, pubblicare e gestire pacchetti software. Viene installato automaticamente quando installi Node.js.

Per controllare la tua versione corrente di `npm`, esegui questo comando:

```bash
npm -v
```

### Eseguire un file JavaScript

Questo comando esegue il file JavaScript che specifichi nel runtime Node.js attivo:

```bash
node <file.js>
```

Ad esempio:

```bash
node app.js
```

### Creare un progetto Node.js

Questo comando crea un nuovo progetto Node.js creando un file `package.json` che tiene traccia dei dettagli e delle dipendenze.

```bash
npm init
```

Quando esegui il comando `npm init`, ti guiderà attraverso una guida CLI interattiva per configurare il file `package.json`. Ti farà domande sul nome del progetto, descrizione, versione, ecc. Se vuoi partire con un file `package.json` predefinito, puoi usare il comando `npm init -y` o `npm init --yes`.

### Installare un pacchetto

Per installare un pacchetto specifico, esegui:

```bash
npm install <package>
```

Ad esempio:

```bash
npm install express
```

### Installare le dipendenze

Per installare tutte le dipendenze elencate nel file `package.json` di un progetto, esegui:

```bash
npm install
```
