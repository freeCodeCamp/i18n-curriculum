# --description--

在先前的課程中，我們簡要提到過 GitHub。GitHub 和 Gitlab 都是版本控制提供者。這表示它們是基於雲端的解決方案，提供將版本控制專案儲存在所謂「repositories」中的功能，並啟用可用於這些專案的協作特性。

在我們深入探討這些平台所提供的特性之前，我們需要討論不同類型的專案。許多專案被視為「開放原始碼」，這表示人們可以查看你發佈的程式碼、提出變更建議、回報問題，甚至執行修改過的版本。

但有時你可能只想讓你的隊友看到並處理程式碼。你也可以將專案發佈為「封閉原始碼」，這表示唯有你明確授權的人才能看到並互動該專案。

無論你的專案採用哪種方法，像 GitHub 和 Gitlab 這樣的網站都能大幅提升你團隊的協作層次。

例如，正在開發某個特性的隊友可能會卡住並有問題。與其必須將程式碼複製貼上到 Discord 讓你閱讀，或是跳上電話分享螢幕，他們可以直接將變更上傳到 GitHub。從那裡，你可以在瀏覽器中查看變更，甚至下載變更以便在區域執行和偵錯。

當隊友完成一項特性時，他們可以透過這些網站提出變更。你接著可以審查這些變更，若有想法和問題，可以在特定的檔案或列上留下評論。你甚至可以提出建議的編輯，他們可以直接在瀏覽器中將這些編輯套用到變更中！

如果你對這些更改感到滿意，你可以接受它們並將它們合併到你的主要分支。這表示整個團隊接著可以在區域的下載最新的更改，這樣他們在開始下一個任務時就擁有最新的程式碼。

說到任務，這些網站也提供強大的問題追蹤器。你可以使用這些追蹤器來追蹤並指定工單給你的團隊成員，接受來自大眾的錯誤報告，甚至透過像是專案看板這類工具來規劃完整的衝刺。

這些網站還提供更多特性。在接下來的幾堂課中，我們將涵蓋關鍵面向，讓你能立即投入並開始協作！
