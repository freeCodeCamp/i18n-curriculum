# --description--

Come promemoria, questo progetto si basa sul progetto iniziale clonato da <a href="https://github.com/freeCodeCamp/boilerplate-mochachai/" target="_blank" rel="noopener noreferrer nofollow">GitHub</a>.

Nelle prossime sfide simulerai l'interazione umana con una pagina usando un browser headless.

I browser headless sono browser web senza interfaccia grafica. Sono in grado di renderizzare e interpretare HTML, CSS e JavaScript allo stesso modo di un browser normale, rendendoli particolarmente utili per testare le pagine web.

Per le sfide seguenti userai Zombie.js, un browser headless leggero che non richiede l'installazione di binari aggiuntivi. Ma esistono molte altre opzioni di browser headless più potenti.

Mocha ti permette di eseguire del codice prima che vengano eseguiti i test veri e propri. Questo può essere utile per fare cose come aggiungere elementi a un database che verrà usato nel resto dei test.

Con un browser headless, prima di eseguire i test, devi **visitare** la pagina che testerai.

Il hook `suiteSetup` viene eseguito una sola volta all'inizio di una suite di test.

Esistono diversi altri tipi di hook che possono eseguire codice prima di ogni test, dopo ogni test o alla fine di una suite di test. Consulta la documentazione di Mocha per maggiori informazioni.

# --instructions--

All'interno di `tests/2_functional-tests.js`, subito dopo la dichiarazione `Browser`, aggiungi l'URL del progetto alla proprietà `site` della variabile:

```js
Browser.site = 'http://0.0.0.0:3000'; // Your URL here
```

Poi, al livello radice della suite `'Functional Tests with Zombie.js'`, istanzia una nuova istanza dell'oggetto `Browser` con il seguente codice:

```js
const browser = new Browser();
```

E usa il hook `suiteSetup` per indirizzare `browser` alla rotta `/` con il seguente codice. **Nota**: `done` viene passato come callback a `browser.visit`, non devi invocarlo.

```js
suiteSetup(function(done) {
  return browser.visit('/', done);
});
```
