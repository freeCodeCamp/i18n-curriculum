## 객체 기본

- **정의**: 객체는 속성들로 이루어진 자료 구조입니다. 속성은 키와 값으로 구성됩니다. 객체에서 데이터를 접근하려면 점 표기법이나 대괄호 표기법을 사용할 수 있습니다.

:::interactive_editor

```js
const person = {
  name: "Alice",
  age: 30,
  city: "New York"
};

console.log(person.name);  // Alice
console.log(person["name"]); // Alice
```

:::

기존 객체의 속성을 설정하려면 점 표기법이나 대괄호 표기법과 대입 연산자를 함께 사용하세요.

:::interactive_editor

```js
const person = {
  name: "Alice",
  age: 30
};

person.job = "Engineer"
person["hobby"] = "Knitting"
console.log(person);  // {name: 'Alice', age: 30, job: 'Engineer', hobby: 'Knitting'}
```

:::

## 객체에서 속성 제거하기

- **`delete` 연산자**: 이 연산자는 객체에서 속성을 제거하는 데 사용됩니다.

:::interactive_editor

```js
const person = {
  name: "Alice",
  age: 30,
  job: "Engineer"
};

delete person.job;

console.log(person.job); // undefined
```

:::

## 객체에 속성이 있는지 확인하기

- **`hasOwnProperty()` 메서드**: 이 메서드는 객체가 지정한 속성을 자신의 속성으로 가지고 있는지 여부를 불리언으로 반환합니다.

:::interactive_editor

```js
const person = {
  name: "Alice",
  age: 30
};

console.log(person.hasOwnProperty("name")); // true
console.log(person.hasOwnProperty("job")); // false
```

:::

- **`Object.hasOwn()` 메서드**: 객체가 자신의 속성(상속받은 것이 아닌)을 가지고 있는지 확인하는 현대적이고 권장되는 방법입니다. 구문은 `Object.hasOwn(object, propertyName)`입니다. 속성이 객체에 있으면 `true`를 반환하고 없으면 `false`를 반환합니다 — 속성 값과 상관없이 그렇습니다. 이로 인해 `hasOwnProperty()`보다 안전하고 `if (obj.prop)`보다 신뢰할 수 있습니다. 값이 `0`, `false`, `null`, 또는 `undefined`일 때 특히 그렇습니다.

:::interactive_editor

```js
const person = {
  name: "Alice",
  age: 30
};

console.log(Object.hasOwn(person, "name")); // true
console.log(Object.hasOwn(person, "job")); // false
```

:::

다음은 `Object.hasOwn()`가 값을 직접 확인하는 것보다 더 신뢰할 수 있는 이유를 보여주는 자세한 예입니다:

:::interactive_editor

```js
const settings = {
  darkMode: false,
  fontSize: 0,
  language: null
};

// Object.hasOwn() correctly sees these properties exist
console.log(Object.hasOwn(settings, "darkMode")); // true (value is false, but exists)
console.log(Object.hasOwn(settings, "fontSize")); // true (value is 0, but exists)
console.log(Object.hasOwn(settings, "theme"));    // false (property was never added)

// Using if() directly is unsafe for falsy values!
if (settings.darkMode) {
  console.log("Dark mode on"); // Does NOT print — misleading!
}

// Object.hasOwn() is the safe way
if (Object.hasOwn(settings, "darkMode")) {
  console.log("darkMode exists, value is:", settings.darkMode); // darkMode exists, value is: false
}
```

:::

- **`in` 연산자**: 이 연산자는 속성이 객체에 있으면 `true`를 반환합니다.

:::interactive_editor

```js
const person = {
  name: "Bob",
  age: 25
};

console.log("name" in person);  // true
```

:::

## 중첩 객체에서 속성 접근하기

- **데이터 접근**: 중첩 객체에서 속성에 접근하려면 점 표기법이나 대괄호 표기법을 사용합니다. 단순 객체에서 속성에 접근하는 것과 비슷하지만, 중첩 구조를 탐색하려면 이러한 접근자를 체이닝해야 합니다.

:::interactive_editor

```js
const person = {
  name: "Alice",
  age: 30,
  contact: {
    email: "alice@example.com",
    phone: {
      home: "123-456-7890",
      work: "098-765-4321"
    }
  }
};

console.log(person.contact.phone.work); // "098-765-4321"
```

:::

## 원시 및 비원시 데이터 타입

- **원시 데이터 타입**: 이 데이터 타입에는 숫자, 문자열, 불리언, `null`, `undefined`, 그리고 심볼이 포함됩니다. 이 타입들은 단일 값을 나타내고 객체가 아니기 때문에 "원시"라고 부릅니다. 원시 값은 불변이며, 한 번 생성되면 값이 변경될 수 없습니다.
- **비원시 데이터 타입**: JavaScript에서 비원시 타입은 객체이며, 일반 객체, 배열, 함수가 포함됩니다. 원시와 달리 비원시 타입은 속성이나 요소로 여러 값을 저장할 수 있습니다.

## 객체 메서드

- **정의**: 객체 메서드는 객체와 연관된 함수입니다. 객체의 속성으로 정의되며 객체 데이터를 접근하고 조작할 수 있습니다. 메서드 내부의 `this` 키워드는 객체 자체를 가리켜 속성에 접근할 수 있게 합니다.

:::interactive_editor

```js
const person = {
  name: "Bob",
  age: 30,
  sayHello: function() {
    return "Hello, my name is " + this.name;
  }
};

console.log(person.sayHello()); // "Hello, my name is Bob"
```

:::

## 객체 생성자

- **정의**: JavaScript에서 생성자는 객체를 생성하고 초기화하는 데 사용되는 특별한 유형의 함수입니다. `new` 키워드와 함께 호출되며 새로 생성된 객체에 속성과 메서드를 초기화할 수 있습니다. `Object()` 생성자는 새 빈 객체를 만듭니다.

```js
new Object()
```

## 선택적 체이닝 연산자(`?.`) 사용하기

- **정의**: 이 연산자는 객체 속성에 안전하게 접근하거나 메서드를 호출할 때, 해당 속성이나 메서드가 존재하는지 걱정하지 않아도 되게 합니다.

:::interactive_editor

```ts
const user = {
  name: "John",
  profile: {
    email: "john@example.com",
    address: {
      street: "123 Main St",
      city: "Somewhere"
    }
  }
};

console.log(user.profile?.address?.street); // "123 Main St"
console.log(user.profile?.phone?.number);   // undefined
```

:::

## 객체 구조 분해

- **정의**: 객체 구조 분해는 객체에서 값을 추출해 변수에 더 간결하고 읽기 쉽게 할당할 수 있게 합니다.

:::interactive_editor

```js
const person = { name: "Alice", age: 30, city: "New York" };

const { name, age } = person;

console.log(name); // Alice
console.log(age);  // 30
```

:::

## JSON 다루기

- **정의**: JSON은 JavaScript Object Notation의 약자입니다. 서버와 웹 애플리케이션 간에 데이터를 교환할 때 흔히 사용하는 가볍고 텍스트 기반의 데이터 형식입니다.

```js
{
  "name": "Alice",
  "age": 30,
  "isStudent": false,
  "list of courses": ["Mathematics", "Physics", "Computer Science"]
}
```

- **`JSON.stringify()`**: 이 메서드는 JavaScript 객체를 JSON 문자열로 변환하는 데 사용됩니다. 시스템 간에 쉽게 공유하거나 전송할 수 있는 형식으로 데이터를 저장하거나 전송할 때 유용합니다.

:::interactive_editor

```js
const user = {
  name: "John",
  age: 30,
  isAdmin: true
};

const jsonString = JSON.stringify(user);
console.log(jsonString); // '{"name":"John","age":30,"isAdmin":true}'
```

:::

- **`JSON.parse()`**: 이 메서드는 JSON 문자열을 다시 JavaScript 객체로 변환합니다. 웹 서버나 localStorage에서 JSON 데이터를 가져와 애플리케이션에서 데이터를 조작할 때 유용합니다.

:::interactive_editor

```js
const jsonString = '{"name":"John","age":30,"isAdmin":true}';
const userObject = JSON.parse(jsonString);

// result: { name: 'John', age: 30, isAdmin: true }
console.log(userObject);
```

:::
