# --description--
  
Динамічне програмування — це алгоритмічна техніка, яка розв’язує складні задачі, розбиваючи їх на простіші підзадачі та зберігаючи результати, щоб уникнути повторних обчислень. Такий підхід перетворює задачі, які зазвичай потребують експоненційного часу, на ті, що можна розв’язати за поліноміальний час.

## Основні принципи динамічного програмування

Динамічне програмування працює, коли в задачі присутні дві ключові умови.

- **Перекривні підзадачі**: однакові менші задачі з’являються кілька разів під час розв’язання більшої задачі. Замість того, щоб повторно обчислювати ці підзадачі, ми зберігаємо їхні рішення.
    
- **Оптимальна підструктура**: оптимальне рішення задачі містить оптимальні рішення її підзадач. Це означає, що ми можемо побудувати найкраще рішення, комбінуючи найкращі рішення менших частин.
    

Розглянемо ці поняття на класичній задачі "підйом по сходах".

## Проблема наївної рекурсії

Розглянемо задачу підйому по сходах: ви піднімаєтесь сходами з `n` сходинок і можете підніматися на 1 або 2 сходинки за раз. Скільки різних способів дістатися до вершини?

```js
function climbStairsRecursive(n) {
  // Base cases: 1 way for 1 step, 2 ways for 2 steps
  if (n <= 2) {
    return n;
  }

  // Recursive rule: to reach step n, you can come from step (n-1) or step (n-2)
  return climbStairsRecursive(n - 1) + climbStairsRecursive(n - 2);
}
```

Ця реалізація має експоненційну складність через величезну кількість повторних обчислень. При обчисленні `climbStairsRecursive(5)` відбувається таке:

- `climbStairsRecursive(5)` викликає `climbStairsRecursive(4)` і `climbStairsRecursive(3)`
    
- `climbStairsRecursive(4)` викликає `climbStairsRecursive(3)` і `climbStairsRecursive(2)`
    
- `climbStairsRecursive(3)` обчислюється **двічі**
    
- `climbStairsRecursive(3)` викликає `climbStairsRecursive(2)` і `climbStairsRecursive(1)`
    
- `climbStairsRecursive(2)` обчислюється **3 рази загалом**
    

Для `n=5` ми робимо 9 викликів функції, хоча потрібно лише 5 унікальних обчислень. З ростом `n` ця надмірність зростає експоненційно — `climbStairsRecursive(30)` вимагатиме понад кілька мільйонів викликів функції! Складність часу стає `O(2^n)`, що робить цей підхід неефективним і непридатним для більших значень `n`.

## Рішення за допомогою динамічного програмування

Динамічне програмування усуває цю надмірність обчислень двома основними підходами:

### Мемоізація (підхід зверху вниз)

Мемоізація зберігає результати дорогих викликів функцій і повертає кешовані результати, коли ті самі вхідні дані зустрічаються знову:

```js
function climbStairsMemo(n, memo = {}) {
  // If already calculated, return cached value
  if (memo[n] !== undefined) {
    return memo[n];
  }

  // Base cases: 1 way for 1 step, 2 ways for 2 steps
  if (n <= 2) {
    return n;
  }

  // Calculate once and store in memo for future use
  memo[n] =
    climbStairsMemo(n - 1, memo) +
    climbStairsMemo(n - 2, memo);

  return memo[n];
}
```

Мемоізація значно ефективніша, бо кожне унікальне значення від `1` до `n` обчислюється рівно один раз. Коли нам знову потрібен `climbStairsMemo(3)`, замість повторного обчислення (що викликало б додаткові рекурсивні виклики) ми просто шукаємо його в нашому словнику мемо в `O(1)` часу.

Пройдемося по виконанню `climbStairsMemo(5)` з підходом зверху вниз, щоб побачити, як мемоізація усуває повторну роботу:

```md
Call: climbStairsMemo(5)
  memo = {} (empty)
  
  Call: climbStairsMemo(4) 
    memo = {} (empty)
    
    Call: climbStairsMemo(3)
      memo = {} (empty)
      
      Call: climbStairsMemo(2) → returns 2 (base case)
      Call: climbStairsMemo(1) → returns 1 (base case)
      
      Result: 2 + 1 = 3
      memo = {3: 3} (stored!)
    
    Call: climbStairsMemo(2) → returns 2 (base case)
    
    Result: 3 + 2 = 5
    memo = {3: 3, 4: 5} (stored!)
  
  Call: climbStairsMemo(3) → returns 3 (FROM MEMO - no recursion!)
  
  Result: 5 + 3 = 8
  memo = {3: 3, 4: 5, 5: 8}
```

**Порівняння ефективності**

- **Наївна рекурсія**: робить 9 викликів функції з повторними обчисленнями
    
- **Мемоізація**: робить лише 5 унікальних обчислень, потім повторно використовує збережені результати
    
- **Складність часу**: скорочується з `O(2^n)` до `O(n)`, бо робиться лише `n` унікальних обчислень
    
- **Складність пам’яті**: `O(n)` для зберігання мемо та стеку викликів
    
- **Реальний вплив**: `climbStairsMemo(30)` падає з мільйонів викликів функції до близько 30 унікальних обчислень!
    

### Табуляція (підхід знизу вгору)

Табуляція будує рішення знизу вгору, заповнюючи таблицю рішеннями підзадач:

```js
function climbStairsTabulation(n) {
  if (n <= 2) {
    return n;
  }

  // Create an array to store solutions from 0 to n
  const dp = new Array(n + 1).fill(0);

  // Base cases
  dp[1] = 1; // 1 way to reach step 1
  dp[2] = 2; // 2 ways to reach step 2

  // Build up the solution iteratively
  for (let i = 3; i <= n; i++) {
    // Ways to reach step i = ways to reach (i-1) + ways to reach (i-2)
    dp[i] = dp[i - 1] + dp[i - 2];
  }

  return dp[n];
}
```

Табуляція повністю усуває рекурсію, будуючи рішення ітеративно від найменших підзадач до цілі.

Погляньмо на підхід знизу вгору в дії, щоб побачити, як систематично будується рішення. Ось ітеративна побудова для `climbStairsTabulation(5)`:

```md
Initial state:
dp = [0, 1, 2, 0, 0, 0]
     [0, 1, 2, 3, 4, 5] ← indices (step numbers)

Step by step construction:

i = 3:
  dp[3] = dp[2] + dp[1] = 2 + 1 = 3
  dp = [0, 1, 2, 3, 0, 0]
  
i = 4:
  dp[4] = dp[3] + dp[2] = 3 + 2 = 5
  dp = [0, 1, 2, 3, 5, 0]
  
i = 5:
  dp[5] = dp[4] + dp[3] = 5 + 3 = 8
  dp = [0, 1, 2, 3, 5, 8]

Final result: dp[5] = 8
```

**Ключові переваги табуляції**

- **Відсутність рекурсії**: на відміну від мемоізації, немає рекурсивного стеку викликів.
    
- **Передбачуване виконання**: обчислюємо значення у визначеному порядку (1, 2, 3, 4, 5...).
    
- **Оптимізація кешу**: послідовний доступ до масиву оптимізує використання пам’яті.
    
- **Легко оптимізувати**: можна зменшити складність пам’яті до `O(1)`, бо потрібні лише останні два значення.
    

```js
function climbStairsOptimized(n) {
  if (n <= 2) {
    return n;
  }

  // Store only the last two results to save space
  let prev2 = 1; // ways to reach step 1
  let prev1 = 2; // ways to reach step 2

  for (let i = 3; i <= n; i++) {
    const current = prev1 + prev2;

    // Shift values forward
    prev2 = prev1;
    prev1 = current;
  }

  return prev1;
}
```

**Порівняння ефективності**

- **Наївна рекурсія**: 9 викликів функції для `n=5`, експоненційне зростання.
    
- **Табуляція**: 3 прості додавання для `n=5`, лінійне зростання.
    
- **Складність часу**: `O(n)` замість `O(2^n)`.
    
- **Складність пам’яті**: `O(n)` для масиву, або `O(1)` з оптимізацією.
    
- **Передбачувана продуктивність**: немає ризику переповнення стеку для великих вхідних даних.
    

Обидва підходи знижують складність часу з експоненційної `O(2^n)` до лінійної `O(n)`, що є драматичним покращенням і робить різницю між розв’язанням задачі за мілісекунди та очікуванням роками для більших вхідних даних.

## Застосування в реальному світі

Динамічне програмування широко застосовується в інформатиці та інших сферах:

- **Оптимізація маршрутів**: GPS-системи використовують алгоритми динамічного програмування для пошуку найкоротших шляхів між локаціями.
    
- **Обробка тексту**: перевірка орфографії та функції автозаповнення часто покладаються на динамічне програмування для обчислення відстаней редагування між словами.
    
- **Фінансове моделювання**: інвестиційні стратегії та оптимізація портфоліо часто застосовують техніки динамічного програмування.
    
- **Розподіл ресурсів**: задача рюкзака та її варіанти зустрічаються у плануванні, бюджетуванні та управлінні ресурсами.
    

## Практичний приклад: задача про зміну монет

Задача про зміну монет — класичне завдання програмування, яке при розв’язанні за допомогою динамічного програмування демонструє обидва ключові принципи ДП: оптимальну підструктуру та перекривні підзадачі.

Задача про зміну монет ставить питання: "Яка мінімальна кількість монет потрібна, щоб отримати цільову суму?"

Ось одне з рішень за допомогою динамічного програмування:

```js
function minCoins(amount, coins) {
  // Create dp array from 0 to amount, filled with Infinity
  const dp = new Array(amount + 1).fill(Infinity);

  // Base case: 0 coins needed to make amount 0
  dp[0] = 0;

  // Build up from 1 to target amount
  for (let i = 1; i <= amount; i++) {
    // Try each coin denomination
    for (let coin of coins) {
      // Only use coin if it is not larger than current amount
      if (coin <= i) {
        // Choose the minimum: current value OR solution for remaining amount + 1 coin
        dp[i] = Math.min(dp[i], dp[i - coin] + 1);
      }
    }
  }

  // Return result if possible, -1 if impossible (infinity)
  return dp[amount] === Infinity ? -1 : dp[amount];
}

/*
 Example usage:
coins = [1, 3, 4], amount = 6

dp[0] = 0
dp[1] = dp[0] + 1 = 1
dp[2] = dp[1] + 1 = 2
dp[3] = min(dp[2]+1, dp[0]+1) = min(2+1, 0+1) = 1
dp[4] = min(dp[3]+1, dp[1]+1, dp[0]+1) = min(1+1, 1+1, 0+1) = 1
dp[5] = min(dp[4]+1, dp[2]+1, dp[1]+1) = min(1+1, 2+1, 1+1) = 2
dp[6] = min(dp[5]+1, dp[3]+1, dp[2]+1) = min(2+1, 1+1, 2+1) = 2

Result: 2 coins (3 + 3)
*/
```

І ось як алгоритм динамічного програмування для задачі про зміну монет працює крок за кроком для `coins = [1, 3, 4]`, `amount = 6`:

```md
Initial state:
dp = [0, ∞, ∞, ∞, ∞, ∞, ∞]
     [0, 1, 2, 3, 4, 5, 6] ← amounts

Building up the solution:

For amount = 1:
  Try coin 1: dp[1] = min(∞, dp[0] + 1) = min(∞, 0 + 1) = 1
  dp = [0, 1, ∞, ∞, ∞, ∞, ∞]

For amount = 2:
  Try coin 1: dp[2] = min(∞, dp[1] + 1) = min(∞, 1 + 1) = 2
  dp = [0, 1, 2, ∞, ∞, ∞, ∞]

For amount = 3:
  Try coin 1: dp[3] = min(∞, dp[2] + 1) = min(∞, 2 + 1) = 3
  Try coin 3: dp[3] = min(3, dp[0] + 1) = min(3, 0 + 1) = 1
  dp = [0, 1, 2, 1, ∞, ∞, ∞]

For amount = 4:
  Try coin 1: dp[4] = min(∞, dp[3] + 1) = min(∞, 1 + 1) = 2
  Try coin 3: dp[4] = min(2, dp[1] + 1) = min(2, 1 + 1) = 2
  Try coin 4: dp[4] = min(2, dp[0] + 1) = min(2, 0 + 1) = 1
  dp = [0, 1, 2, 1, 1, ∞, ∞]

For amount = 5:
  Try coin 1: dp[5] = min(∞, dp[4] + 1) = min(∞, 1 + 1) = 2
  Try coin 3: dp[5] = min(2, dp[2] + 1) = min(2, 2 + 1) = 2
  Try coin 4: dp[5] = min(2, dp[1] + 1) = min(2, 1 + 1) = 2
  dp = [0, 1, 2, 1, 1, 2, ∞]

For amount = 6:
  Try coin 1: dp[6] = min(∞, dp[5] + 1) = min(∞, 2 + 1) = 3
  Try coin 3: dp[6] = min(3, dp[3] + 1) = min(3, 1 + 1) = 2
  Try coin 4: dp[6] = min(2, dp[2] + 1) = min(2, 2 + 1) = 2
  dp = [0, 1, 2, 1, 1, 2, 2]

Final result: dp[6] = 2 (achieved with coins 3 + 3)
```

Це рішення демонструє обидва ключові принципи динамічного програмування. Тут є перекривні підзадачі, бо для знаходження мінімальної кількості монет для суми 6 потрібно знати рішення для сум 5, 3 і 2. Ці самі підзадачі з’являються при обчисленні інших сум. Є оптимальна підструктура, бо оптимальне рішення для будь-якої суми включає оптимальні рішення для менших сум. Якщо ми знаємо, що мінімальна кількість монет для суми 3 — це 1, то один зі способів отримати суму 6 — використати це рішення плюс ще одну монету номіналом 3.

Без ДП довелося б перебирати всі можливі комбінації монет — експоненційна кількість варіантів. З ДП ми систематично будуємо рішення:

- **Складність часу**: `O(amount × number of coins)` замість експоненційної.
    
- **Складність пам’яті**: `O(amount)` для масиву `dp`.
    
- **Відсутність повторної роботи**: кожна підзадача (знаходження мінімальної кількості монет для кожної суми) розв’язується рівно один раз.
    
- **Повторне використання результатів**: як тільки ми знаємо мінімальну кількість монет для суми 3, використовуємо це знання для всіх більших сум, які можуть від цього виграти.
    

## Коли використовувати динамічне програмування

Динамічне програмування ефективне, коли:

- Задачу можна розбити на перекривні підзадачі.
    
- Задача має оптимальну підструктуру.
    
- Наївне рекурсивне рішення передбачає повторні обчислення.
    
- Потрібно оптимізувати складність часу за рахунок складності пам’яті.
    

Загальні шаблони динамічного програмування включають задачі оптимізації (пошук мінімальних/максимальних значень), задачі підрахунку (кількість способів досягти чогось) та задачі прийняття рішень, які можна розбити на менші рішення.

Динамічне програмування перетворює складні задачі на керовані, систематично зберігаючи та повторно використовуючи рішення підзадач. Розуміння цієї техніки відкриває двері до ефективного розв’язання широкого спектра обчислювальних завдань.
