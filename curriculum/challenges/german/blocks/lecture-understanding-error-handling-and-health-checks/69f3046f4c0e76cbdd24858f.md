# --description--

Hatten Sie schon einmal, dass Ihre Express-App ohne klare Fehlermeldung abstürzte? Debugging und Logging können Ihnen helfen, genau zu sehen, was hinter den Kulissen passiert, damit Sie Probleme schneller finden und beheben können.

In dieser Lektion erkunden wir, wie Express Debugging und Logging handhabt – und wie Sie es nutzen können, um Ihren Entwicklungsworkflow zu verbessern.

## Eingebautes Debugging mit Express

Express enthält bereits ein Debugging-Tool namens `debug`. Sie müssen nichts installieren, um es zu verwenden.

Um Debugging zu aktivieren, müssen Sie nur die Umgebungsvariable `DEBUG` setzen, wenn Sie Ihre App starten.

Hier ein Beispiel:

```js
DEBUG=express:* node index.js
```

Dieser Befehl aktiviert alle Debugging-Nachrichten unter dem Namespace `express:*`. Sie werden Ausgaben wie diese sehen:

```js
express:router:route dispatching GET /
express:router:layer processing GET /
```

Diese Nachrichten helfen Ihnen, die Weiterleitung von Routen, Middleware-Aufrufe und mehr nachzuvollziehen. Es ist eine großartige Möglichkeit, den Ablauf von Anfragen in Ihrer App zu verstehen.

Möchten Sie spezifischer sein? Sie können den Namespace eingrenzen:

```js
DEBUG=express:router node index.js
```

Dies zeigt nur Protokolle, die mit der Router-Logik von Express zu tun haben.

Unter Windows ist die Syntax für PowerShell etwas anders:

```js
$env:DEBUG = "express:*"; node index.js
```

Und für CMD sieht es so aus:

```js
set DEBUG=express:* && node index.js
```

Dasselbe Prinzip, nur eine andere Art, Umgebungsvariablen zu setzen.

Schauen wir uns nun das Logging in Express an.

Für das Logging von HTTP-Anfragen verwenden viele Entwickler Middleware wie `morgan`.

So verwenden Sie sie:

```bash
npm install morgan
```

Dann fügen Sie sie Ihrer App hinzu:

```js
const express = require('express')
const morgan = require('morgan')
const app = express()

app.use(morgan('dev'))s
```

Jetzt wird jede eingehende Anfrage mit Details wie Methode, URL, Statuscode und Antwortzeit im Terminal protokolliert.

Und warum ist das wichtig?

Debugging gibt Ihnen Einblick, wie Express Anfragen, Middleware und Routing verarbeitet, ohne Ihre Produktionslogs zu überladen.

Logging liefert Ihnen eine Aufzeichnung dessen, was bei den Anfragen passiert – das ist entscheidend, um Bugs zu debuggen, den Traffic zu verstehen und Probleme frühzeitig zu erkennen.

Zusammen geben sie Ihnen Kontrolle und Klarheit in Ihrem Entwicklungsprozess.
