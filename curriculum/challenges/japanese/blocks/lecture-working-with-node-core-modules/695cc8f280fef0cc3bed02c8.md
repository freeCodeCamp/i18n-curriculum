# --description--

CryptoはNode.jsに組み込まれているコアモジュールの一つです。ハッシュ化、暗号化、復号、デジタル署名の作成など、機密情報を保護しアプリの安全性を保つためのツールが含まれています。

とはいえ、`crypto`はプラグアンドプレイのセキュリティではなく、低レベルの構成要素を提供します。自分で暗号化や認証コードを書くのは注意しないと安全でない場合があります。ほとんどの場合、パスワードのハッシュ化には`bcrypt`、ログインやトークンの処理には`jsonwebtoken`（JWT）など、よくテストされたライブラリを使うのが最善です。

とはいえ、cryptoモジュールのいくつかのメソッドがどのように動作するかを理解するのは役立ちます。

これらのメソッドを使うには、`crypto`モジュールをインポートしてください。

```js
const crypto = require("crypto");
```

メソッドの中には、ハッシュ化や暗号化のようにデータ変換のために使うものもあれば、乱数生成や秘密情報の作成のようにキーやセキュリティ管理のためのものもあります。

まずはデータのハッシュ化や暗号化のメソッドを見てみましょう。

`createHash()`メソッドは、`sha256`、`sha512`、`md5`などのアルゴリズムを受け取り、ハッシュオブジェクトを作成します。これは一方向の操作なので逆に戻すことはできません。

`createHash()`はパスワードのハッシュ化やファイルのフィンガープリントに便利です。使うにはアルゴリズムを渡し、`update()`メソッドでデータを入力し、最後にエンコーディングを指定して`digest()`でハッシュ値を取得します。

```js
const crypto = require("crypto");

const hashedPassword = crypto
  .createHash("sha256")
  .update("myStrongPassword")
  .digest("hex");

console.log("createHash result:", hashedPassword);
// createHash result: f92c9cfa0ead1bcec05ca75888a4074ba994ad237e5e2a8c7cc6a620378c061d
```

`createHmac()`は`createHash()`とほぼ同じですが、秘密鍵を受け取ることで、鍵を持つ人だけがハッシュを検証できるようにします。認証やデータの整合性確認に最適です。

```js
const crypto = require("crypto");

const hashedMessage = crypto
  .createHmac("sha256", "secretkey")
  .update("important-secret-message")
  .digest("hex");

console.log("createHmac result:", hashedMessage);
// createHmac result: da48d6f026b6036286b1fb872c63264130d5cc4271f3a213bb6ddca5a023e77e
```

`createCipheriv()`と`createDecipheriv()`メソッドはデータの暗号化と復号を行います。どちらもアルゴリズム、キー、そして`iv`（暗号化開始時に使うランダムまたはユニークなデータのブロック）を受け取ります。

```js
createCipheriv(algorithm, key, iv);
createDecipheriv(algorithm, key, iv);
```

復号するにはキーが同じでなければならず、そうでないと復号に失敗し、エラーが出るか元のメッセージとは読めない文字列になります。

実際の動作例はこちらです。

```js
const crypto = require("crypto");

// A key must match the algorithm length. Here AES-256 is 32 bytes
const key = Buffer.from("12345678901234567890123456789012");

// A fixed IV, 16 bytes for AES
const iv = Buffer.from("1234567890123456");

const cipher = crypto.createCipheriv("aes-256-cbc", key, iv);

let encrypted = cipher.update("Hello campers!", "utf8", "hex");
encrypted += cipher.final("hex");

console.log("Encrypted data:", encrypted);
// Encrypted data: 4ee93aa398ab44e3540e4a67ca96bc8c

// Decrypt the "Hello campers!" message
const decipher = crypto.createDecipheriv("aes-256-cbc", key, iv);
let decrypted = decipher.update(encrypted, "hex", "utf8");
decrypted += decipher.final("utf8");

console.log("Decrypted data:", decrypted);
// Decrypted data: Hello campers!
```

データ変換用の別のcryptoメソッドに`sign()`と`verify()`があります。

`sign()`は秘密鍵を使ってデータからデジタル署名を作成します。この署名はデータが秘密鍵の所有者から来て改ざんされていないことを証明します。

`verify()`はその署名を検証し、データや署名が一致しなければ失敗します。

次に、乱数生成や秘密情報作成のためのcryptoメソッドを見てみましょう。

`randomBytes()`は`size`を受け取り、暗号学的に安全なトークンを生成します。UUID（ユニバーサルユニークID）生成に適しています。また、トークンやキーには安全でない`Math.random()`の代わりとしても使えます。

```js
console.log("Random Bytes:", crypto.randomBytes(16));
// Random Bytes: <Buffer 01 88 aa 1e 2c 38 48 39 26 e1 6b a9 d8 c5 ed 49>
```

出力はデフォルトでバッファです。Bufferモジュールのレッスンで学んだように、そのバッファは`toString()`メソッドで文字列に変換できます。

```js
console.log("Random Bytes:", crypto.randomBytes(16).toString("hex"));
// Random Bytes: a6154ef5a296fa176ad0f332bd94d712
```

ここでの`'hex'`引数は、バッファのバイナリデータを16進数の文字列としてエンコードするようNodeに指示しています。

`randomInt()`メソッドは`min`と`max`の値を受け取り、その間の安全な乱数整数を生成します。OTPやランダム選択に便利です。

```js
console.log("Random Int:", crypto.randomInt(0, 100)); // 89
```

`Math.random()`や`Math.floor()`より優れているのは、このメソッドが内部で暗号学的に安全な乱数を使うため、攻撃者が結果の乱数を予測できない点です。

もう一つのメソッドは`createSecretKey()`です。バッファを受け取り、`KeyObject`でラップされた生のバイトを生成します。

```js
const crypto = require("crypto");

const secret = crypto.createSecretKey(crypto.randomBytes(32));
console.log(secret); // SecretKeyObject [KeyObject] { [Symbol(kKeyType)]: 'secret' }
```

その後、`export`メソッドでその`KeyObject`を送信できます。

```js
const secret = crypto.createSecretKey(crypto.randomBytes(32));
console.log(secret.export());
// <Buffer 53 06 a1 c7 75 69 8b 38 8b a4 b2 f7 1b bc b8
// ae e2 d1 bf 67 af 1a 6a 0a 6e a0 29 62 bb 52 52 32>
```

最後に、バッファに`toString()`を使うと、その文字列表現を確認できます。

```js
const secret = crypto.createSecretKey(crypto.randomBytes(32));
console.log(secret.export().toString('hex'));
// 32dfe5917668580160986f1623bf8152913329c71163be9c3404a110cd78efd6
```

これらに加えて、

- `createPublicKey()`と`createPrivateKey()`は外部で生成されたキーを扱うためのもの
- `createDiffieHellman()`は二者間で秘密を直接送らずに共有秘密を生成するためのもの
- `Certificate()`はHTTPSで使われるものを扱い、証明書の内容を解析、エクスポート、検証するためのもの

があります。
