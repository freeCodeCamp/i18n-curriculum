### --question--

#### --text--

In cosa differisce la comunicazione HTTP da quella WebSocket?

#### --distractors--

HTTP è bidirezionale mentre WebSocket è unidirezionale.

---

HTTP mantiene una connessione permanente, mentre WebSocket si chiude dopo ogni scambio.

---

HTTP e WebSocket seguono entrambi lo stesso pattern richiesta-risposta.

#### --answer--

HTTP segue `Client --> Request --> Server --> Response`, mentre WebSocket è `Client <--> Server`.

### --question--

#### --text--

Qual è il flusso di messaggi in Pub/Sub?

#### --distractors--

Un publisher invia a un subscriber --> il subscriber inoltra a un topic --> il topic archivia il messaggio.

---

Un topic invia un messaggio --> i publisher lo ricevono --> i publisher lo inoltrano ai subscriber.

---

Un subscriber si connette a un publisher --> il publisher invia dati --> il subscriber conferma la ricezione.

#### --answer--

Un subscriber si iscrive a un topic --> un publisher invia un messaggio a quel topic --> il sistema di messaggistica lo consegna a tutti i subscriber di quel topic.

Quale delle seguenti descrive correttamente il flusso di connessione WebSocket?

#### --distractors--

Il client si connette via TCP --> il server invia dati --> il client riceve --> la connessione si chiude.

---

Il client invia dati --> il server richiede l'upgrade --> la connessione si apre --> il client conferma.
