## 什么是中间件？

- **中间件** 是在服务器请求生命周期中执行的函数。
- 中间件可以访问 **请求对象** (`req`)、**响应对象** (`res`) 和 **`next` 函数**。
- 常见用途：记录请求日志、处理误差、解析传入数据和提供静态文件。
- 中间件按照添加到应用中的顺序执行。

### 中间件的工作原理

1. 请求发送到服务器。
2. 中间件函数处理请求。
3. 如果中间件调用 `next()`，控制权传递给下一个中间件或路由处理程序。
4. 如果中间件不调用 `next()`，请求-响应周期停止。

```javascript
app.use((req, res, next) => {
  console.log('Request URL:', req.url);
  console.log('Request Method:', req.method);
  next();
});
```

- `app.use()` 将中间件全局添加到应用中。

## 应用级中间件

- **应用级中间件** 绑定到 Express 应用的实例。
- 使用 `app.use()` 或路由特定方法如 `app.get()`、`app.post()` 等注册。
- 对每个传入请求执行，除非限制到特定路由。
- 用例：记录日志、认证和跨多个路由的误差处理。

```javascript
app.use((req, res, next) => {
  console.log('Request received at:', new Date());
  next();
});
```

## 路由器级中间件

- **路由器级中间件** 绑定到 `express.Router()` 的实例，而不是主应用。
- 使用 `router.use()` 应用，仅影响该路由器处理的路由。
- 有助于将应用模块化为独立的路由模块。
- 使用 `app.use('/path', router)` 挂载路由器，将中间件限定到特定路由。

```javascript
const router = express.Router();

router.use((req, res, next) => {
  console.log('Request made to /menu route');
  next();
});

router.get('/drinks', (req, res) => {
  res.send('Welcome to the drinks menu!');
});

app.use('/menu', router);
```

## 错误处理中间件

- **错误处理中间件** 是用于捕获请求-响应周期中错误的特殊中间件。
- 它有 **四个参数**：`err`、`req`、`res` 和 `next`。四参数签名告诉 Express 这是一个错误处理器。
- 当错误传递给 `next(err)` 时调用。
- 应该放置在中间件栈的 **末尾**，在所有其他中间件和路由之后。

```javascript
app.use((err, req, res, next) => {
  console.error(err.stack);
  res.status(500).send('Something went wrong!');
});
```

## 内置和第三方中间件

### 内置中间件

- **`express.json()`**：解析带有 JSON 负载的传入请求。

```javascript
app.use(express.json());
```

- **`express.static()`**：提供静态文件，如图像、CSS 和 JavaScript。

```javascript
app.use(express.static('public'));
```

- **`express.urlencoded()`**：解析带有 URL 编码负载的传入请求（HTML 表单发送的格式）；使解析后的数据可通过 `req.body` 访问。

```javascript
app.use(express.urlencoded({ extended: true }));
```

### 第三方中间件

- **`cors`**：启用跨源资源共享。

```javascript
app.use(cors());
```

- **`morgan`**：记录 HTTP 请求日志。

```javascript
app.use(morgan('tiny'));
```

- 内置和第三方中间件函数都按照调用 `app.use()` 的顺序添加。
