# --interactive--

## Grundlagen von HTML

- **Rolle von HTML**: HTML stellt den Inhalt und die Struktur der Webseite dar.
- **HTML-Elemente**: Elemente sind die Bausteine eines HTML-Dokuments. Sie stellen Überschriften, Absätze, Links, Bilder und mehr dar. Die meisten HTML-Elemente bestehen aus einem öffnenden Tag (`<elementName>`) und einem schließenden Tag (`</elementName>`).

Hier ist die Grundsyntax:

```html
<elementName>Content goes here</elementName>
```

- **Void-Elemente**: Void-Elemente können keinen Inhalt haben und besitzen nur ein Start-Tag. Beispiele sind `img` und `meta`-Elemente.

```html
<img>
<meta>
```

Es ist üblich, in manchen Codebasen einen Schrägstrich `/` innerhalb des Void-Elements zu sehen. Beide Varianten sind akzeptabel:

```html
<img>
<img/>
```

- **Attribute**: Ein Attribut ist ein Wert, der im öffnenden Tag eines HTML-Elements platziert wird. Attribute liefern zusätzliche Informationen über das Element oder geben an, wie sich das Element verhalten soll. Hier ist die grundlegende Syntax für ein Attribut:

```html
<element attribute="value"></element>
```

Ein Boolean-Attribut ist ein Attribut, das entweder in einem HTML-Tag vorhanden oder nicht vorhanden sein kann. Wenn es vorhanden ist, ist der Wert true, andernfalls false. Beispiele für Boolean-Attribute sind `disabled`, `readonly` und `required`.

- **Kommentare**: Kommentare werden in der Programmierung verwendet, um sich selbst und anderen Entwickler*innen Notizen im Code zu hinterlassen. Hier ist die Syntax für einen Kommentar in HTML:

```html
<!--This is an HTML comment.-->
```

## Häufige HTML-Elemente

- **Überschriften-Elemente**: Es gibt sechs Überschriften-Elemente in HTML. Die `h1` bis `h6` Überschriften-Elemente werden verwendet, um die Wichtigkeit des darunterliegenden Inhalts zu kennzeichnen. Je niedriger die Zahl, desto höher die Wichtigkeit, daher haben `h2`-Elemente weniger Bedeutung als `h1`-Elemente.

:::interactive_editor

```html
<h1>most important heading element</h1>
<h2>second most important heading element</h2>
<h3>third most important heading element</h3>
<h4>fourth most important heading element</h4>
<h5>fifth most important heading element</h5>
<h6>least important heading element</h6>
```

:::

- **Absatz-Elemente**: Dieses Element wird für Absätze auf einer Webseite verwendet.

:::interactive_editor

```html
<p>This is a paragraph element.</p>
```

:::

- **`img`-Elemente**: Das `img`-Element wird verwendet, um Bilder auf der Webseite hinzuzufügen. Das `src`-Attribut gibt den Speicherort dieses Bildes an. Für Bildelemente ist es eine bewährte Vorgehensweise, ein weiteres Attribut namens `alt`-Attribut einzufügen. Hier ist ein Beispiel für ein `img`-Element mit den Attributen `src` und `alt`:

:::interactive_editor

```html
<img src="https://cdn.freecodecamp.org/curriculum/cat-photo-app/lasagna.jpg" alt="A slice of lasagna on a plate.">
```

:::

- **`body`-Element**: Dieses Element wird verwendet, um den Inhalt des HTML-Dokuments darzustellen.

:::interactive_editor

```html
<body>
  <h1>CatPhotoApp</h1>
  <p>This is a paragraph element.</p>
</body>
```

:::

- **`section`-Elemente**: Dieses Element wird verwendet, um Inhalte in kleinere Abschnitte zu unterteilen.

:::interactive_editor

```html
<section>
  <h2>About Me</h2>
  <p>Hi, I am Jane Doe and I am a web developer.</p>
</section>
```

:::

- **`div`-Elemente**: Dieses Element ist ein generisches HTML-Element ohne semantische Bedeutung. Es wird als allgemeiner Container verwendet, um andere HTML-Elemente zu halten.

:::interactive_editor

```html
<div>
  <h1>I am a heading</h1>
  <p>I am a paragraph</p>
</div>
```

:::

- **Anker (`a`)-Elemente**: Diese Elemente werden verwendet, um Links auf einer Webseite anzuwenden. Das `href`-Attribut gibt an, wohin der Link führt, wenn der Benutzer darauf klickt.

:::interactive_editor

```html
<a href="https://cdn.freecodecamp.org/curriculum/cat-photo-app/running-cats.jpg">cute cats</a>
```

:::

- **Ungeordnete (`ul`) und geordnete (`ol`) Listen-Elemente**: Um eine Aufzählungsliste zu erstellen, sollten Sie das `ul`-Element mit einem oder mehreren verschachtelten `li`-Elementen verwenden, so:

:::interactive_editor

```html
<ul>
  <li>catnip</li>
  <li>laser pointers</li>
  <li>lasagna</li>
</ul>
```

:::

Um eine geordnete Liste zu erstellen, verwenden Sie das `ol`-Element:

:::interactive_editor

```html
<ol>
  <li>flea treatment</li>
  <li>thunder</li>
  <li>other cats</li>
</ol>
```

:::

- **Emphasis (`em`)-Element**: Dieses Element wird verwendet, um einen Textabschnitt hervorzuheben.

:::interactive_editor

```html
<p>Cats <em>love</em> lasagna.</p>
```

:::

- **Strong Importance (`strong`)-Element**: Dieses Element wird verwendet, um starken Nachdruck auf Text zu legen und eine Dringlichkeit oder Ernsthaftigkeit anzuzeigen.

:::interactive_editor

```html
<p>
  <strong>Important:</strong> Before proceeding, make sure to wear your safety goggles.
</p>
```

:::

- **`figure`- und `figcaption`-Elemente**: Das `figure`-Element wird verwendet, um Inhalte wie Bilder und Diagramme zu gruppieren. Das `figcaption`-Element stellt eine Bildunterschrift für den Inhalt innerhalb des `figure`-Elements dar.

:::interactive_editor

```html
<figure>
  <img src="https://cdn.freecodecamp.org/curriculum/cat-photo-app/cats.jpg" alt="Two tabby kittens sleeping together on a couch.">
  <figcaption>Cats <strong>hate</strong> other cats.</figcaption>
</figure>
```

:::

- **`main`-Element**: Dieses Element stellt den Hauptinhalt einer Webseite dar.  
- **`footer`-Element**: Dieses Element befindet sich am unteren Ende des HTML-Dokuments und enthält üblicherweise Copyright-Informationen und andere wichtige Seitenlinks.

:::interactive_editor

```html
<footer>
  <p>
    No Copyright - <a href="https://www.freecodecamp.org">freeCodeCamp.org</a>
  </p>
</footer>
```

:::

- **`button`-Element**: Dieses Element wird verwendet, um einen anklickbaren Button zu erstellen.

:::interactive_editor

```html
<button>Click me</button>
```

:::

## Identifikatoren und Gruppierung

- **IDs**: Eindeutige Element-Identifikatoren für HTML-Elemente. ID-Namen sollten nur einmal pro HTML-Dokument verwendet werden.

```html
<h1 id="title">Movie Review Page</h1>
```

ID-Namen dürfen keine Leerzeichen enthalten. Wenn Ihr ID-Name aus mehreren Wörtern besteht, können Sie Bindestriche oder Unterstriche zwischen den Wörtern verwenden, so wie hier:

```html
<div id="red-box"></div>
```

- **Klassen**: Klassen werden verwendet, um Elemente für Styling und Verhalten zu gruppieren.

```html
<div class="box"></div>
```

Im Gegensatz zu IDs können Sie denselben Klassennamen im gesamten HTML-Dokument mehrfach verwenden. Der `class`-Wert kann ebenfalls Leerzeichen enthalten, so:

```html
<div class="box red-box"></div>
<div class="box blue-box"></div>
```

## Sonderzeichen und Verlinkung

- **HTML-Entitäten**: Eine HTML-Entität oder Zeichenreferenz ist eine Zeichenfolge, die verwendet wird, um ein reserviertes Zeichen in HTML darzustellen. Beispiele sind das Ampersand-Symbol (`&amp;`) und das Kleiner-als-Zeichen (`&lt;`).

```html
<p>This is an &lt;img /&gt; element</p>
```

- **`link`-Element**: Dieses Element wird verwendet, um externe Ressourcen wie Stylesheets und Seiten-Icons zu verknüpfen. Hier ist die grundlegende Syntax für die Verwendung des `link`-Elements für eine externe CSS-Datei:

```html
<link rel="stylesheet" href="./styles.css" />
```

Das `rel`-Attribut gibt die Beziehung zwischen der verknüpften Ressource und dem HTML-Dokument an. Das `href`-Attribut gibt den Speicherort der URL für die externe Ressource an.

- **`script`-Element**: Dieses Element wird verwendet, um ausführbaren Code einzubetten.

```html
<body>
  <script>
    alert("Welcome to freeCodeCamp");
  </script>
</body>
```

Obwohl Sie technisch gesehen Ihren gesamten JavaScript-Code innerhalb der `script`-Tags schreiben können, gilt es als bewährte Vorgehensweise, stattdessen auf eine externe JavaScript-Datei zu verlinken. Hier ist ein Beispiel für die Verwendung des `script`-Elements, um auf eine externe JavaScript-Datei zu verlinken:

```html
<script src="path-to-javascript-file.js"></script>
```

Das `src`-Attribut gibt hier den Speicherort für diese externe JavaScript-Datei an.

## Boilerplate und Kodierung

- **HTML-Boilerplate**: Dies ist eine Boilerplate, die die Grundstruktur und wesentliche Elemente enthält, die jedes HTML-Dokument benötigt.

:::interactive_editor

```html
<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="utf-8" />
    <title>freeCodeCamp</title>
    <link rel="stylesheet" href="./styles.css" />
  </head>
  <body>
    <!--Headings, paragraphs, images, etc. go inside here-->
  </body>
</html>
```

:::

- **`DOCTYPE`**: Dies wird verwendet, um Browsern mitzuteilen, welche HTML-Version Sie verwenden.
- **`html`-Element**: Dieses stellt das oberste Element oder die Wurzel eines HTML-Dokuments dar. Um die Sprache für das Dokument anzugeben, sollten Sie das `lang`-Attribut verwenden.
- **`head`-Element**: Der `head`-Abschnitt enthält wichtige Metadaten, also Hintergrundinformationen, die für Browser und Suchmaschinen benötigt werden.
- **`meta`-Elemente**: Diese Elemente stellen die Metadaten Ihrer Webseite dar. Sie enthalten Details wie Zeichencodierung und wie Webseiten wie Twitter den Link Ihrer Seite in der Vorschau anzeigen sollen.
- **`title`-Element**: Dieses Element wird verwendet, um den Text festzulegen, der im Browser-Tab oder -Fenster angezeigt wird.
- **UTF-8-Zeichencodierung**: UTF-8, oder UCS Transformation Format 8, ist eine standardisierte Zeichencodierung, die im Web weit verbreitet ist. Zeichencodierung ist die Methode, mit der Computer Zeichen als Daten speichern. Das `charset`-Attribut wird innerhalb eines `meta`-Elements verwendet, um die Zeichencodierung auf UTF-8 festzulegen.

## SEO und Social Sharing

- **SEO**: Suchmaschinenoptimierung ist eine Praxis, die Webseiten so optimiert, dass sie sichtbarer werden und in Suchmaschinen höher ranken.
- **Meta-(`description`) Element**: Dieses wird verwendet, um eine kurze Beschreibung für die Webseite bereitzustellen und die SEO zu verbessern.

```html
<meta
  name="description"
  content="Discover expert tips and techniques for gardening in small spaces, choosing the right plants, and maintaining a thriving garden."
/>
```

- **Open Graph-Tags**: Das Open-Graph-Protokoll ermöglicht es Ihnen, zu steuern, wie der Inhalt Ihrer Webseite auf verschiedenen sozialen Medienplattformen wie Facebook, LinkedIn und vielen anderen angezeigt wird. Durch das Festlegen dieser Open-Graph-Eigenschaften können Sie Nutzer dazu verleiten, auf Ihre Inhalte zu klicken und sich damit zu beschäftigen. Sie können diese Eigenschaften über eine Sammlung von `meta`-Elementen im `head`-Abschnitt Ihres HTML-Dokuments setzen.
- **`og:title`-Eigenschaft**: Diese wird verwendet, um den Titel festzulegen, der für Social-Media-Beiträge angezeigt wird.

```html
<meta content="freeCodeCamp.org" property="og:title" />
```

- **`og:type`-Eigenschaft**: Die `type`-Eigenschaft wird verwendet, um den Typ des Inhalts darzustellen, der in sozialen Medien geteilt wird. Beispiele hierfür sind Artikel, Webseiten, Videos oder Musik.

```html
<meta property="og:type" content="website" />
```

- **`og:image`-Eigenschaft**: Diese wird verwendet, um das Bild festzulegen, das für Social-Media-Beiträge angezeigt wird.

```html
<meta
  content="https://cdn.freecodecamp.org/platform/universal/fcc_meta_1920X1080-indigo.png"
  property="og:image"
/>
```

- **`og:url`-Eigenschaft**: Diese wird verwendet, um die URL festzulegen, auf die Nutzer*innen für Social-Media-Beiträge klicken.

```html
<meta property="og:url" content="https://www.freecodecamp.org" />
```

## Medienelemente und Optimierung

- **Ersetzte Elemente**: Ein ersetztes Element ist ein Element, dessen Inhalt durch eine externe Ressource bestimmt wird und nicht durch CSS selbst. Ein Beispiel wäre ein `iframe`-Element. `iframe` steht für Inline-Frame. Es ist ein Inline-Element, das verwendet wird, um anderen HTML-Inhalt direkt innerhalb der HTML-Seite einzubetten.

```html
<iframe src="https://www.example.com" title="Example Site"></iframe>
```

Sie können das `allowfullscreen`-Attribut hinzufügen, das es dem Nutzer ermöglicht, das iframe im Vollbildmodus anzuzeigen.

```html
<iframe
  src="video-url"
  width="width-value"
  height="height-value"
  allowfullscreen
></iframe>
```

Um ein Video innerhalb eines `iframe` einzubetten, können Sie es direkt von beliebten Videodiensten wie YouTube und Vimeo kopieren oder selbst definieren, indem Sie das `src`-Attribut verwenden, das auf die URL dieses Videos zeigt. Hier ist ein Beispiel, wie man einen beliebten freeCodeCamp-Kurs von YouTube einbettet:

:::interactive_editor

```html
<h1>A freeCodeCamp YouTube Video Embedded with the iframe Element</h1>

<iframe
  width="560"
  height="315"
  src="https://www.youtube.com/embed/PkZNo7MFNFg?si=-UBVIUNM3csdeiWF"
  title="YouTube video player"
  allow="accelerometer; autoplay; clipboard-write; encrypted-media; gyroscope; picture-in-picture; web-share"
  referrerpolicy="strict-origin-when-cross-origin"
  allowfullscreen
></iframe>
```

:::

Es gibt noch weitere ersetzte Elemente, wie `video` und `embed`. Und einige Elemente verhalten sich unter bestimmten Umständen wie ersetzte Elemente. Hier ist ein Beispiel für ein `input`-Element mit dem `type`-Attribut, das auf `image` gesetzt ist:

```html
<input type="image" alt="Descriptive text goes here" src="example-img-url">
```

- **Medien optimieren**: Es gibt drei Werkzeuge, die Sie beim Einsatz von Medien wie Bildern auf Ihren Webseiten berücksichtigen sollten: die Größe, das Format und die Kompression. Ein Kompressionsalgorithmus wird verwendet, um die Größe von Dateien oder Daten zu reduzieren.
- **Bildformate**: Zwei der gebräuchlichsten Dateiformate sind PNG und JPG, aber diese sind nicht mehr die idealsten Formate für die Bereitstellung von Bildern. Sofern Sie keine Unterstützung für ältere Browser benötigen, sollten Sie ein optimierteres Format wie WEBP oder AVIF in Betracht ziehen.
- **Bildlizenzen**: Ein Bild, das gemeinfrei ist, hat kein Urheberrecht und kann ohne Einschränkungen verwendet werden. Bilder, die speziell unter der Creative Commons 0-Lizenz stehen, gelten als gemeinfrei. Einige Bilder können unter einer erlaubnisfreien Lizenz veröffentlicht sein, wie einer Creative Commons-Lizenz oder der BSD-Lizenz, die freeCodeCamp verwendet.
- **SVGs**: Scalable Vector Graphics speichern Daten basierend auf Pfaden und Gleichungen, um Punkte, Linien und Kurven zu zeichnen. Das bedeutet, dass eine Vektorgrafik wie eine SVG auf jede Größe skaliert werden kann, ohne die Qualität zu beeinträchtigen.

## Multimedia-Integration

- **`audio`- und `video`-Elemente**: Die `audio`- und `video`-Elemente ermöglichen es Ihnen, Ton- und Videoinhalte in Ihre HTML-Dokumente einzufügen. Das `audio`-Element unterstützt gängige Audioformate wie mp3, wav und ogg. Das `video`-Element unterstützt mp4-, ogg- und webm-Formate.

```html
<audio src="CrystalizeThatInnerChild.mp3"></audio>
```

Wenn Sie den Audioplayer auf der Seite sehen möchten, können Sie das `audio`-Element mit dem `controls`-Attribut hinzufügen:

:::interactive_editor

```html
<audio src="CrystalizeThatInnerChild.mp3" controls></audio>
```

:::

Das `controls`-Attribut ermöglicht es Nutzer*innen, die Audiowiedergabe zu steuern, einschließlich Lautstärkeanpassung sowie Pausieren oder Fortsetzen der Wiedergabe. Das `controls`-Attribut ist ein Boolean-Attribut, das einem Element hinzugefügt werden kann, um eingebaute Wiedergabesteuerungen zu aktivieren. Wenn es weggelassen wird, werden keine Steuerungen angezeigt.

- **`autoplay`-Attribut**: Das `autoplay`-Attribut ist ein Boolean-Attribut, das bewirkt, dass Audio oder Video automatisch abgespielt wird, sobald es ohne Unterbrechung möglich ist.
- **`loop`-Attribut**: Das `loop`-Attribut ist ein Boolean-Attribut, das bewirkt, dass das Audio kontinuierlich wiederholt wird.

:::interactive_editor

```html
<audio
  src="https://cdn.freecodecamp.org/curriculum/js-music-player/can't-stay-down.mp3"
  loop
  controls
></audio>
```

:::

- **`muted`-Attribut**: Wenn es im `audio`-Element vorhanden ist, startet das `muted`-Boolean-Attribut die Audioaufnahme stummgeschaltet.

:::interactive_editor

```html
<audio
  src="https://cdn.freecodecamp.org/curriculum/js-music-player/can't-stay-down.mp3"
  loop
  controls
  muted
></audio>
```

:::

- **`source`-Element**: Bei Audiodateitypen gibt es Unterschiede darin, welche Browser welche Typen unterstützen. Um dem gerecht zu werden, können Sie `source`-Elemente innerhalb des `audio`-Elements verwenden, und der Browser wählt die erste Quelle aus, die er versteht. Hier ist ein Beispiel für die Verwendung mehrerer `source`-Elemente für ein `audio`-Element:

```html
<audio controls>
  <source src="audio.ogg" type="audio/ogg" />
  <source src="audio.wav" type="audio/wav" />
  <source src="audio.mp3" type="audio/mpeg" />
</audio>
```

Alle bisher gelernten Attribute werden auch im `video`-Element unterstützt. Hier ist ein Beispiel für die Verwendung eines `video`-Elements mit den Attributen `loop`, `controls` und `muted`:

:::interactive_editor

```html
<video
  src="https://archive.org/download/BigBuckBunny_124/Content/big_buck_bunny_720p_surround.mp4"
  loop
  controls
  muted
></video>
```

:::

- **`poster`-Attribut**: Wenn Sie ein Bild anzeigen möchten, während das Video heruntergeladen wird, können Sie das `poster`-Attribut verwenden. Dieses Attribut ist für `audio`-Elemente nicht verfügbar und ist einzigartig für das `video`-Element.

:::interactive_editor

```html
<video
  src="https://archive.org/download/BigBuckBunny_124/Content/big_buck_bunny_720p_surround.mp4"
  loop
  controls
  muted
  poster="https://peach.blender.org/wp-content/uploads/title_anouncement.jpg?x11217"
  width="620"
></video>
```

:::

## Target Attribute Types

- **`target`-Attribut**: Dieses Attribut sagt dem Browser, wo die URL für das Ankerelement geöffnet werden soll. Es gibt vier wichtige mögliche Werte für dieses Attribut: `_self`, `_blank`, `_parent` und `_top`. Es gibt einen fünften Wert namens `_unfencedTop`, der derzeit für die experimentelle `FencedFrame`-API verwendet wird. Wahrscheinlich werden Sie diesen Wert noch nicht brauchen.
- **`_self`-Wert**: Dies ist der Standardwert für das `target`-Attribut. Er öffnet den Link im aktuellen Browserkontext. In den meisten Fällen ist das der aktuelle Tab oder das aktuelle Fenster.

```html
<a href="https://freecodecamp.org" target="_self">Visit freeCodeCamp</a>
```

- **`_blank`-Wert**: Dieser öffnet den Link in einem neuen Browsing-Kontext. Typischerweise wird dies in einem neuen Tab geöffnet. Manche Nutzer*innen konfigurieren ihre Browser jedoch so, dass stattdessen ein neues Fenster geöffnet wird.

```html
<a href="https://freecodecamp.org" target="_blank">Visit freeCodeCamp</a>
```

- **`_parent`-Wert**: Dieser öffnet den Link im übergeordneten Kontext des aktuellen. Zum Beispiel, wenn Ihre Webseite ein iframe enthält, würde ein `_parent`-Wert in diesem iframe im Tab/Fenster Ihrer Webseite geöffnet werden, nicht im eingebetteten Frame.

```html
<a href="https://freecodecamp.org" target="_parent">Visit freeCodeCamp</a>
```

- **`_top`-Wert**: Dieser öffnet den Link im obersten Browsing-Kontext, denken Sie an „der Elternteil des Elternteils“. Dies ist ähnlich wie `_parent`, aber der Link wird immer im vollständigen Browser-Tab oder -Fenster geöffnet, auch bei verschachtelten eingebetteten Frames.

```html
<a href="https://freecodecamp.org" target="_top">Visit freeCodeCamp</a>
```

## Absolute vs. relative Pfade

- **Pfaddefinition**: Ein Pfad ist ein String, der den Speicherort einer Datei oder eines Verzeichnisses in einem Dateisystem angibt. In der Webentwicklung ermöglichen Pfade Entwickler*innen, Ressourcen wie Bilder, Stylesheets, Skripte und andere Webseiten zu verknüpfen.  
- **Pfad-Syntax**: Es gibt drei wichtige Syntaxelemente. Zuerst der Schrägstrich, der je nach Betriebssystem ein Backslash (`\`) oder ein Vorwärtsschrägstrich (`/`) sein kann. Zweitens der einzelne Punkt (`.`). Und schließlich der doppelte Punkt (`..`). Der Schrägstrich wird als „Pfadtrenner“ bezeichnet. Er zeigt eine Trennung zwischen Ordner- oder Dateinamen an. Ein einzelner Punkt verweist auf das aktuelle Verzeichnis, zwei Punkte auf das übergeordnete Verzeichnis.

```md
public/index.html
./favicon.ico
../src/index.css
```

- **Absoluter Pfad**: Ein absoluter Pfad ist eine vollständige Route zu einer Ressource innerhalb eines Dateisystems. Er beginnt beim Stammverzeichnis, umfasst alle weiteren Verzeichnisse und schließlich den Dateinamen mit Erweiterung. Das „Stammverzeichnis“ bezeichnet das oberste Verzeichnis oder den obersten Ordner in einer Hierarchie.
- **Absolute URL**: Eine absolute URL baut auf dem absoluten Pfad für Ressourcen im Web auf. Zusätzlich zum Pfad enthält sie das Protokoll, das `http`, `https` oder `file` sein kann, sowie den Domainnamen. Hier ist ein Beispiel für eine absolute URL, die auf das freeCodeCamp-Logo verweist:

:::interactive_editor

```html
<a href="https://design-style-guide.freecodecamp.org/img/fcc_secondary_small.svg">
  View fCC Logo
</a>
```

:::

- **Relativer Pfad**: Ein relativer Pfad gibt den Speicherort einer Datei relativ zum Verzeichnis der aktuellen Datei an. Er enthält nicht das Protokoll oder den Domainnamen, wodurch er kürzer und flexibler für interne Links innerhalb derselben Webseite ist. Hier ist ein Beispiel, wie man von der `about.html`-Seite auf die `contact.html`-Seite verlinkt, die sich beide im selben Ordner befinden:

```html
<p>
  Read more on the
  <a href="about.html">About Page</a>
</p>
```

## Link-Zustände

- **`:link`**: Dies ist der Standardzustand. Dieser Zustand stellt einen Link dar, den der Nutzer noch nicht besucht, angeklickt oder mit dem er noch nicht interagiert hat. Sie können diesen Zustand als Basisstil für alle Links auf Ihrer Seite betrachten. Die anderen Zustände bauen darauf auf.
- **`:visited`**: Dieser Zustand gilt, wenn ein Nutzer die verlinkte Seite bereits besucht hat. Standardmäßig wird der Link dadurch lila, aber Sie können CSS nutzen, um dem Nutzer eine andere visuelle Rückmeldung zu geben.
- **`:hover`**: Dieser Zustand gilt, wenn ein Nutzer den Mauszeiger über einen Link bewegt. Dieser Zustand ist hilfreich, um einem Link zusätzliche Aufmerksamkeit zu verleihen und sicherzustellen, dass der Nutzer tatsächlich beabsichtigt, darauf zu klicken.
- **`:focus`**: Dieser Zustand gilt, wenn wir den Fokus auf einen Link setzen.
- **`:active`**: Dieser Zustand gilt für Links, die vom Nutzer aktiviert werden. Das bedeutet in den meisten Fällen, dass der Link mit der primären Maustaste durch Linksklick angeklickt wird.
