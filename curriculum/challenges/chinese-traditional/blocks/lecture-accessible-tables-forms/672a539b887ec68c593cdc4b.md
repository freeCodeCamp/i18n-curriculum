# --description--

當我們看到一個表格時，我們會立即開始在數據和標題之間建立視覺聯繫。

例如，假設我們有一個代表寵物的表格。 我們有兩隻狗和兩隻貓，表格展示了它們的名字和年齡。 雖然視力正常的人可能能夠理解這個表格中的關係，但對於使用屏幕閱讀器瀏覽表格的人來說，要建立值和標題之間的聯繫就難得多了。

作爲網絡開發人員，你需要負責創建這些關聯，並以屏幕閱讀器用戶易於理解的方式構建 HTML 標記。

因此，讓我們來看看如何創建每個人都能理解的無障礙表格。 我們要介紹的第一個最佳實踐是使用表格標題。 使用 `caption` 元素，你可以編寫表格的標題，以便用戶（尤其是使用輔助技術的用戶）可以快速瞭解表格的目的和內容。 你應將 `caption` 元素緊接在 `table` 開始標籤之後放置。 這樣，屏幕閱讀器和其他輔助技術就可以在閱讀內容之前先播報標題，從而提供更多的背景信息。

```html
<table>
  <caption>Our Pets</caption>
  <!-- Table Rows and Columns -->
</table>
```

現在我們來談談行表頭和列表頭。 表頭單元格是一種特殊單元格，通常位於行或列的開頭，用於描述行或列中存儲的數據類型。 你可以使用表頭單元格元素 `th` 定義行或列頭。

例如，下面的代碼爲兩隻寵物創建了一個表格。 每一行都有一個行表頭（寵物名稱），每一列都有一個列表頭，描述列中數據所代表的內容（年齡和種類）。

```html
<table>
  <caption>Our Pets</caption>
  <thead>
    <tr>
      <!-- Column Headers -->
      <th>Name</th>
      <th>Age</th>
      <th>Type</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <th>Nora</th> <!-- Row Header -->
      <td>5</td>
      <td>Dog</td>
    </tr>
    <tr>
      <th>Gino</th> <!-- Row Header -->
      <td>2</td>
      <td>Cat</td>
    </tr>
  </tbody>
</table>
```

注意，上述代碼將 `caption` 元素緊接在 `table` 開始標籤之後放置。 然後，在表頭（`thead`）元素內，它包含列標題（`Name`、`Age` 和 `Type`）。 在表體（`tbody`）元素中的第二行和第三行，我們可以找到每隻寵物的數據。 寵物的名稱是行表頭，因爲它們位於表頭單元格元素（`th`）中。

將數據單元格與相應的表頭關聯起來對屏幕閱讀器也非常重要。 `scope` 屬性決定表頭是行表頭還是列表頭。 屏幕閱讀器可能會從表格的結構中正確猜到這一點，但通常建議明確指出 `scope` 以確保清晰。

`scope` 屬性有四種可能的值。 你最常用的兩個是用於列的 `col` 和用於行的 `row`。 在下面的代碼中，你可以看到我們爲列和行表頭添加了 `scope` 屬性。 三個列表頭 (`Name`, `Age`, and `Type`) 具有列值爲 `col` 的 `scope`。

兩個行表頭（`Nora` 和 `Gino`）具有值爲 `row` 的 `scope`。

```html
<table>
  <caption>Our Pets</caption>
  <thead>
    <tr>
      <!-- Now they have scope -->
      <th scope="col">Name</th>
      <th scope="col">Age</th>
      <th scope="col">Type</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <th scope="row">Nora</th>
      <td>5</td>
      <td>Dog</td>
    </tr>
    <tr>
      <th scope="row">Gino</th>
      <td>2</td>
      <td>Cat</td>
    </tr>
  </tbody>
</table>
```

如果列或行表頭橫跨多個單元格，`scope` 也將分別應用於每個單元格。 這是一個示例：

<table>
  <tbody>
    <tr>
      <td></td>
      <th scope="col">名字</th>
      <th scope="col">年齡</th>
    </tr>
    <tr>
      <th rowspan="2" scope="row">狗</th>
      <th scope="row">Nora</th>
      <td>5</td>
    </tr>
    <tr>
      <th scope="row">Gino</th>
      <td>2</td>
    </tr>
    <tr>
      <th rowspan="2" scope="row">貓</th>
      <th scope="row">Lulu</th>
      <td>10</td>
    </tr>
    <tr>
      <th scope="row">Elizabeth</th>
      <td>6</td>
    </tr>
  </tbody>
</table>

在這個表格中，`Nora` 的年齡（`5`）這個單元格對應一個列表頭（`Age`）和兩個行表頭（`Dogs` 和 `Nora`）。 `Gino` 的年齡（`2`）也對應一個列表頭（`Age`）和兩個行表頭（`Dogs` 和 `Gino`）。

不過，有些屏幕閱讀器可能無法解釋結構複雜的表格，因此還應儘量將表格扁平化，避免行表頭和列表頭跨越多個單元格。

你的目標應該始終是確保用戶可以訪問這些信息，即使他們的屏幕閱讀器可以處理複雜的表格結構。

現在，對於單元格寬度，建議避免使用固定值。 你應該使用相對值，如百分比。 此外，儘量避免定義單元格高度。 這樣，用戶就可以根據自己的需要調整文字大小。

最後，應儘可能讓瀏覽器決定表格的寬度，以減少水平滾動的需要。

HTML 表格對於以易於訪問和理解的格式呈現結構化數據至關重要。 遵循這些無障礙指南，你就可以製作出讓每個人都易於理解的表格。
