# --description--

¿Alguna vez tu aplicación Express se ha roto sin un mensaje de error claro? La depuración y el registro pueden ayudarte a ver exactamente qué está pasando detrás de escena, para que puedas encontrar y solucionar problemas más rápido.

En esta lección, exploraremos cómo Express maneja la depuración y el registro, y cómo puedes usarlos para mejorar tu flujo de trabajo de desarrollo.

## Depuración integrada con Express

Express ya incluye una herramienta de depuración llamada `debug`. No necesitas instalar nada para usarla.

Para activar la depuración, solo tienes que establecer la variable de entorno `DEBUG` cuando ejecutes tu aplicación.

Aquí tienes un ejemplo:

```js
DEBUG=express:* node index.js
```

Este comando habilita todos los mensajes de depuración bajo el espacio de nombres `express:*`. Comenzarás a ver salidas como:

```js
express:router:route dispatching GET /
express:router:layer processing GET /
```

Estos mensajes te ayudan a rastrear el envío de rutas, llamadas a middleware y más. Es una excelente forma de entender el flujo de las solicitudes en tu aplicación.

¿Quieres ser más específico? Puedes limitar el espacio de nombres:

```js
DEBUG=express:router node index.js
```

Esto solo mostrará los registros relacionados con la lógica del enrutador de Express.

En Windows, la sintaxis es un poco diferente para PowerShell:

```js
$env:DEBUG = "express:*"; node index.js
```

Y para CMD, se ve así:

```js
set DEBUG=express:* && node index.js
```

La idea es la misma, solo que es otra forma de establecer variables de entorno.

Ahora veamos el registro en Express.

Para registrar solicitudes HTTP, muchos desarrolladores usan middleware como `morgan`.

Así es como se usa:

```bash
npm install morgan
```

Luego agrégalo a tu aplicación:

```js
const express = require('express')
const morgan = require('morgan')
const app = express()

app.use(morgan('dev'))s
```

Ahora, cada solicitud entrante se registrará en la terminal con detalles como el método, la URL, el código de estado y el tiempo de respuesta.

¿Y por qué importa esto?

La depuración te da una visión de cómo Express maneja las solicitudes, el middleware y el enrutamiento sin saturar tus registros de producción.

El registro te proporciona un historial de lo que sucede en las solicitudes, lo cual es crucial para depurar errores, entender el tráfico y detectar problemas temprano.

Juntos, te dan control y claridad en tu proceso de desarrollo.
