# --description--

CRUD 是一個縮寫，代表創建、讀取、更新和刪除。這些是持久性儲存的四種操作行為。

持久性儲存是指以一種方式保存資料，使其即使在關閉電源或重新啟動裝置後仍可取用。

了解 CRUD 的基本操作行為如何運作對於網頁開發至關重要，因為它構成了與資料庫互動及建置讓使用者能夠新增、檢視、修改和刪除資料的應用程式的基礎。 

現在，讓我們更仔細地看看 CRUD 的每個零件。

- 創建是指創建新資料的處理程序。 例如，在網頁應用程式中，這可能是當使用者為你的部落格添加新貼文時。

- 讀取是從資料庫擷取資料的操作行為。例如，當你瀏覽部落格文章或查看網站上的評測時，你正在執行讀取操作行為以擷取並顯示儲存在資料庫中的資料。

- 更新涉及修改資料庫中現有的資料。範例包括編輯部落格文章或更新你的評測資訊。

- 刪除是從資料庫中移除資料的操作行為。例如，當你刪除一篇部落格文章或帳號時，你正在執行刪除操作行為。

當處理資料庫、使用者介面和 RESTful API 時會使用 CRUD。RESTful API 是一組用於建置網路服務的慣例，允許客戶端透過標準 HTTP 方法執行 CRUD 操作行為，以與資料庫或後端系統互動。

HTTP 代表超文字傳輸協定，它是網路上資料通訊的基礎。有 HTTP 方法定義了可以對網路上的資源執行的動作。常見的方法有 `GET`、`POST`、`PUT`、`PATCH`、`DELETE`。

你將在未來的課程中學習更多關於 RESTful API 和 HTTP 的內容，但這裡是 CRUD 如何對應到不同 HTTP 方法的快速說明。

- `POST` 用於創建一個新的資源。

- `GET` 用於擷取或讀取資料。

- `PUT` 用於透過完全取代來更新資源。

- `PATCH` 用於部分更新資源。

- `DELETE` 用於移除資源。

以下是如何使用 JavaScript 中的簡單陣列在程式碼中表述 CRUD 操作的範例：

```js
let items = [];

// Create
function createItem(item) {
  items.push(item);
}

// Read
function readItems() {
  return items;
}

// Update
function updateItem(index, newItem) {
  items[index] = newItem;
}

// Delete
function deleteItem(index) {
  items.splice(index, 1);
}

// Example Usage
createItem('Book');
console.log(readItems()); // ['Book']
updateItem(0, 'Magazine');
console.log(readItems()); // ['Magazine']
deleteItem(0);
console.log(readItems()); // []
```

在此範例中：

- 我們透過將一個項目推入陣列來創建它。

- 我們透過傳回陣列來讀取條款。

- 我們透過修改指定索引的陣列元素來更新一個項目。

- 我們透過使用 `splice()` 從陣列中移除一個條款來刪除它。

這是 CRUD 操作在概念層級上如何運作的基本表述。
