# --interactive--

Se scrivi un’espressione con diversi operatori in JavaScript, come fa JavaScript a decidere quale valutare per primo? Qui entra in gioco la precedenza degli operatori. Esploriamo la precedenza degli operatori nel dettaglio con esempi di codice e vediamo anche cosa succede quando gli operatori hanno la stessa precedenza.

La precedenza degli operatori determina l’ordine in cui le operazioni vengono valutate in un’espressione. Gli operatori con precedenza più alta vengono valutati prima di quelli con precedenza più bassa. Pensa alla precedenza degli operatori come in matematica, dove divisione e moltiplicazione avvengono prima di addizione e sottrazione.

Senza questa regola, le equazioni di base darebbero risultati sbagliati. JavaScript funziona allo stesso modo. Ha le sue regole su quali operatori vengono prima, seconda e così via. Per esempio, guarda l’espressione qui sotto:

:::interactive_editor

```js
const result = 2 + 3 * 4;

console.log(result); // 14
```

:::

Se JavaScript valutasse questa espressione da sinistra a destra, ti aspetteresti `2 + 3 = 5`, poi `5 * 4 = 20`. Ma poiché la moltiplicazione ha una precedenza più alta dell’addizione, JavaScript valuta prima la parte `3 * 4`, ottenendo `2 + 12 = 14`.

A volte, potresti voler che certe parti della tua espressione vengano eseguite prima, indipendentemente dalle regole di precedenza. Puoi usare le parentesi tonde, `()`, per farlo. Qualsiasi cosa dentro le parentesi viene valutata prima, a prescindere da tutto. Facciamo in modo che la parte `2 + 3` dell’esempio precedente venga valutata per prima:

:::interactive_editor

```js
const result = (2 + 3) * 4;

console.log(result); // 20
```

:::

Nell’esempio sopra, le parentesi costringono JavaScript a valutare prima `2 + 3`, e poi moltiplicare il risultato per `4`. Questo ti dà `20` invece di `14`.

Anche l’operatore di divisione ha più precedenza dell’addizione o sottrazione:

:::interactive_editor

```js
const result = 2 + 6 / 3;

console.log(result); // 4
```

:::

Se JavaScript valutasse questa espressione da sinistra a destra, potresti aspettarti `2 + 6 = 8`, poi `8 / 3 = 2.67`. Ma dato che la divisione ha una precedenza più alta dell’addizione, JavaScript valuta prima la divisione: `6 / 3 = 2`, e poi aggiunge `2 + 2`, ottenendo il risultato `4`.

Quindi, sia nella moltiplicazione sia nella divisione, queste operazioni avvengono sempre prima di addizione e sottrazione a meno che non usi le parentesi per cambiare l’ordine. Ma cosa succede se gli operatori hanno la stessa precedenza? JavaScript usa l’associatività per capire in quale ordine valutarli.

L’associatività dice a JavaScript se valutare gli operatori da sinistra a destra o da destra a sinistra. Per la maggior parte degli operatori come addizione e moltiplicazione, l’associatività è da sinistra a destra. Quindi, JavaScript li elabora partendo dal lato più a sinistra dell’espressione verso destra:

:::interactive_editor

```js
const result = 10 - 2 + 3;

console.log(result); // 11
```

:::

Prima `10 - 2 = 8`, poi `8 + 3 = 11`. In questo caso JavaScript procede da sinistra a destra. Alcuni operatori, come l’assegnazione (`=`), sono associativi da destra a sinistra. Questo significa che il lato destro dell’espressione viene valutato per primo:

:::interactive_editor

```js
let a, b;
a = b = 5;

console.log(a); // 5
console.log(b); // 5
console.log(a + b); // 10
```

:::

Nell’esempio sopra, JavaScript assegna prima `5` a `b`, poi assegna `b` (che ora è `5`) a `a`.

Anche l’operatore di esponente è associativo da destra a sinistra:

:::interactive_editor

```js
const result = 2 ** 3 ** 2;

console.log(result); // 512
```

:::

Prima JavaScript valuta `3 ** 2`, che è uguale a `9`, poi valuta `2 ** 9`, che è uguale a `512`. Se l’operatore di esponente fosse associativo da sinistra a destra, JavaScript avrebbe calcolato prima `2 ** 3` per ottenere `8`, poi `8 ** 2` per ottenere `64`.
