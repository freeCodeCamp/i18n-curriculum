# --description--

Na lição anterior, você criou uma pasta de projeto e instalou o Express:

```bash
mkdir my-app
cd my-app
npm init -y
npm install express
```

Como lembrete, esses comandos criam uma nova pasta de projeto, `cd` dentro da pasta do projeto, inicializam o projeto e criam um arquivo `package.json` básico, e instalam o Express.

Em seguida, abra um editor de código, crie um arquivo chamado `index.js` e escreva seu primeiro servidor Express:

```javascript
const express = require("express");
const app = express();
const port = 3000;

app.get("/", (req, res) => {
  res.send("Hello World!");
});

app.listen(port, () => {
  console.log(`Example app listening on port ${port}`);
});
```

Você pode executar seu código com o seguinte comando no terminal:

```bash
node index.js
```

Abra seu navegador em [http://localhost:3000](http://localhost:3000) e você verá a mensagem **"Hello World!"**

É assim que é rápido começar com o Express! Agora vamos analisar o código passo a passo.

Na primeira linha de código, você está carregando a biblioteca Express e atribuindo isso a uma variável chamada `express`:

```javascript
const express = require("express");
```

A variável `express` é uma função fábrica. Quando você a chama, ela cria e retorna uma nova instância da aplicação Express:

```javascript
const app = express();
```

`app` agora tem acesso a todos os métodos fornecidos pelo Express, como `.get()`, `.post()`, `.listen()` e assim por diante. Este é o seu objeto principal do servidor.

A próxima linha define a variável `port`. Uma porta é um endpoint de rede que sua aplicação escuta, e `3000` é uma porta comumente usada para desenvolvimento local:

```javascript
const port = 3000;
```

Agora, vamos dar uma olhada na próxima parte aqui:

```javascript
app.get("/", (req, res) => {
  res.send("Hello World!");
});
```

Este bloco de código define uma rota e diz ao servidor como responder às requisições. `app.get()` registra um manipulador para requisições HTTP GET. Uma requisição GET é um método HTTP comumente usado para recuperar dados.

O argumento `"/"` dentro de `app.get()` representa o caminho da rota. Neste caso, ele aponta para a URL raiz da aplicação.

A parte `(req, res) => { ... }` é uma função de callback que é executada quando uma requisição corresponde a essa rota. O objeto `req` contém informações sobre a requisição recebida, e o objeto `res` é usado para enviar uma resposta de volta ao cliente.

Finalmente, `res.send("Hello World!")` envia uma resposta para o cliente. Neste exemplo, a resposta é o texto simples `"Hello World!"`.

Agora, vamos dar uma olhada na última parte do código aqui:

```javascript
app.listen(port, () => {
  console.log(`Example app listening on port ${port}`);
});
```

O método `app.listen()` inicia a aplicação Express e diz ao Node para escutar requisições HTTP recebidas na porta especificada, neste caso, `3000`.

Assim que o servidor inicia, a função de callback é executada e a mensagem `Example app listening on port 3000` é registrada no console, confirmando que o app Express foi iniciado com sucesso e está pronto para receber requisições.
