# --description--

في بايثون، تعمل الأصناف والكائنات معًا لتنظيم وإدارة البيانات. تبني صنفًا لتعريف سلوك مشترك، ثم تنشئ كائنات تستخدم تلك السلوكيات.

بمعنى آخر، الصنف يشبه المخطط أو القالب الذي تستخدمه لإنشاء الكائنات.

لنلقِ نظرة على ما هي الأصناف، وكيف تستخدمها لإنشاء الكائنات.

لإنشاء صنف، تستخدم الكلمة المفتاحية `class` متبوعة باسم الصنف ونقطتين. ثم داخل الصنف، يمكنك إضافة مُهيئ، بالإضافة إلى أي خصائص وطرق.

الخصائص تشبه المتغيرات داخل الصنف، وتُستخدم لتخزين البيانات. الطرق هي دوال معرفة داخل الصنف، وهي الأفعال التي يمكن للكائنات التي تُنشأ من الصنف تنفيذها.

إليك الصيغة النحوية الأساسية للصنف:

```python
class ClassName:
    def __init__(self, name, age):
        self.name = name
        self.age = age

    def sample_method(self):               
        print(self.name.upper())
```

* `class ClassName` تتكون من الكلمة المفتاحية `class` لإنشاء صنف، متبوعة باسم الصنف، هنا يسمى `ClassName`. من الشائع في بايثون استخدام اصطلاح **PascalCase** عند تسمية الأصناف.
    
* `def __init__(self, name, age)` هي الطريقة الخاصة التي تُستدعى تلقائيًا عند إنشاء كائن جديد. تُهيئ خصائص الكائنات التي ستُنشأ من الصنف.
    
بالإضافة إلى ذلك، المعلمة الأولى في `__init__` هي دائمًا مرجع للكائن المحدد الذي يُنشأ أو يُستخدم. حسب الاصطلاح، تُسمى هذه المعلمة `self`، لكن تقنيًا يمكنك استخدام أي اسم. يسمح لك `self` بالوصول إلى خصائص وطرق الكائن نفسه.
    
* `self.name = name` و `self.age = age` هما الخصائص التي ستمتلكها الكائنات.
    
* `def sample_method(self):` هي الطريقة التي يمكن لكل كائن منشأ استدعاؤها.
    
* `print(self.name.upper())` هو ما ستفعله طريقة `sample_method`، في هذه الحالة، تطبع الاسم بأحرف كبيرة.
    

إذا بدا لك كل هذا كثيرًا، لا تقلق. لنلقِ نظرة على مثال مشابه لصنف `Dog`، وكيف يمكنك إنشاء كائنات منه:

```python
class Dog:
    def __init__(self, name, age):
        self.name = name
        self.age = age

    def bark(self):
        print(f"{self.name.upper()} says woof woof!")
```

مع هذا الصنف `Dog`، يمكنك إنشاء كائن. إليك الصيغة النحوية الأساسية لإنشاء كائنات من صنف:

```python
object_1 = ClassName(attribute_1, attribute_2)
object_2 = ClassName(attribute_1, attribute_2)
```

يمكنك أيضًا استدعاء أي من الطرق المعرفة في الصنف من كل كائن:

```python
object_1.method_name()
object_2.method_name()
```

الآن لننشئ كلبين باستخدام صنف `Dog` كمخطط:

```python
class Dog:
    def __init__(self, name, age):
        self.name = name
        self.age = age

    def bark(self):
        print(f"{self.name.upper()} says woof woof! I'm {self.age} years old!")

dog_1 = Dog("Jack", 3)
dog_2 = Dog("Thatcher", 5)

# Call the bark method
dog_1.bark()  # JACK says woof woof! I'm 3 years old!
dog_2.bark()  # THATCHER says woof woof! I'm 5 years old!
```

كما ترى، ننشئ كائنين من الكلاب باستخدام صنف `Dog`. عند تهيئة `dog_1`، تُمرر السلسلة النصية `Jack` والرقم `3`، مما يحدد خصائص `name` و `age` لذلك الكائن. وتهيأ `dog_2` بالسلسلة النصية `Thatcher` والرقم `5` كخصائص `name` و `age` على التوالي.

ثم عند استدعاء طريقة `.bark()` على `dog_1` و `dog_2`، يمكنك رؤية كيف تختلف المخرجات، وتستخدم الخصائص الفريدة `name` و `age` التي مررتها عند إنشاء كل كائن.

باختصار، الفرق بين الصنف والكائن هو أن الصنف هو القالب أو المخطط، والكائن هو ما يُنشأ باستخدام ذلك القالب.

أيضًا، يحدد الصنف البيانات والسلوك الذي يجب أن يمتلكه الكائن، والكائن يحمل البيانات الفعلية ويستخدم ذلك السلوك. تكتب الصنف مرة واحدة، ويمكنك إنشاء العديد من الكائنات منه، كل منها ببيانات مختلفة.
