# --description--

调试你的 JavaScript 程序不仅仅是使用代码中的 `console.log()` 语句。一旦你习惯了，还有更高级的技术让调试变得轻而易举。让我们一起了解这些更高级的技术，这样当你遇到错误时，就会乐于调试你的 JavaScript 程序。

我们将首先了解的概念是使用断点。断点允许你在选择的特定代码行暂停代码执行。暂停后，你可以检查变量、计算表达式并查看调用栈。

要在 Chrome 浏览器中为代码的任意行添加断点，请打开开发者工具并导航到 Sources 标签（页），打开你想调试的 JavaScript 文件，然后点击行号以设置断点。

当执行到达断点并停止后，你可以使用右上角的图标逐步执行代码。你也可以通过右键点击某一行并选择“添加条件式断点…”来添加条件式断点。

现在，让我们继续了解监视表达式。监视表达式允许你在代码运行时监控变量或表达式的值，即使它们不在当前作用域内。

要添加监视表达式，请导航到开发者工具的 Sources 标签（页），在右侧查找监视窗格，然后点击加号（`+`）图标进行添加。

下一个我们将要了解的概念是性能分析。性能分析通过让你捕捉屏幕截图和记录 CPU 使用率、函数调用及执行时间，帮助你识别性能瓶颈。

为你的代码打开性能标签（页），点击记录，然后执行你想要评测的操作。执行完该操作后，停止记录以分析结果。

如果你的应用在某些操作期间出现延迟，这可能会很有用。通过评测器，你可以看到代码中哪个函数或其他数据消耗了最多资源。

现在，让我们继续检查网络请求。检查网络请求可以帮助你调试与 API 请求相关的问题，例如参数错误、地址错误或服务器错误。

要使用 `Network` 标签（页）进行调试，请打开开发者工具并切换到 `Network` 标签（页），然后点击各个请求以查看诸如头部、响应和负载等详细信息。

在本课的最后部分，我们将重点讲解 `console.table` 和 `console.dir`。

`console.table()` 在控制台中以表格形式显示表格数据。它接受一个必需的参数，该参数必须是数组或对象，以及一个选择性参数，用于指定要显示的属性或列。

另一方面，`console.dir()` 允许你显示指定 JavaScript 对象属性的交互式列表。它输出一个可展开以查看所有嵌套属性的分层列表。

既然你已经拥有了更丰富的调试技巧工具包，就在你即将进行的 JavaScript `project` 中测试这些新技能。通过应用这些策略，你将能够更高效地识别和解析问题，从而编写出更简洁、更可靠的 `code`。
