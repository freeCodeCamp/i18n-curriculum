# --description--

Imagina que entras a un edificio de oficinas con alta seguridad. En la recepción, el guardia de seguridad te pide tu tarjeta de identificación para comprobar quién eres. Una vez que el guardia verifica que tu rostro coincide con la tarjeta, atraviesas el vestíbulo. Sin embargo, cuando intentas pasar esa misma tarjeta para entrar a la sala de servidores ejecutiva, el lector se pone rojo y te niega el acceso.

Este escenario cotidiano ilustra perfectamente los dos pilares de la seguridad web: **autenticación** y **autorización**. Aunque suenan parecido y trabajan juntos de cerca, cumplen funciones completamente diferentes en un sistema backend.

La autenticación decide quién eres. Es el proceso de verificar la identidad de un usuario. Responde a una pregunta fundamental: *"¿Es esta persona quien dice ser?"*

Cuando un usuario se registra para una cuenta, proporciona un identificador único, como un nombre de usuario, correo electrónico, una contraseña y otras credenciales. Esta información se almacena de forma segura en la base de datos. La próxima vez que el usuario visite e intente iniciar sesión, el sistema backend compara las credenciales que escribe con los registros almacenados en la base de datos.

Los desarrolladores backend usan tres categorías principales de factores para autenticar usuarios:

- **Algo que sabes:** una contraseña, un PIN o la respuesta a una pregunta de seguridad.

- **Algo que tienes:** un smartphone físico que recibe un código de verificación por SMS, o un token temporal de una app autenticadora.

- **Algo que eres:** datos biométricos como una huella digital o un escaneo facial.

Una vez que el sistema autentica exitosamente al usuario, la **autorización** toma el control y decide qué puede hacer o a qué puede acceder el usuario. Responde a una pregunta diferente: *"¿Qué permisos tiene esta identidad específica?"*

Que un usuario inicie sesión con éxito no significa que deba tener control total sobre la aplicación. La autorización define las reglas, roles y límites para ese usuario. Considera estos roles comunes en aplicaciones:

- **Usuario estándar:** puede editar su propio perfil, pero no puede ver la información privada de facturación de otros usuarios.

- **Editor:** puede escribir y publicar entradas de blog, pero no puede eliminar todo el código del sitio web.

- **Administrador:** tiene permiso total para modificar bases de datos, eliminar usuarios y cambiar configuraciones del sistema.

En el código backend, la autorización suele ocurrir mediante funciones middleware que verifican un rol de usuario o permisos específicos antes de conceder acceso a ciertas páginas o datos.

Para entenderlo bien, recuerda esta regla simple: **la autenticación siempre debe ocurrir primero, luego la autorización.** No puedes decidir qué permisos darle a alguien hasta que sepas exactamente quién es.

La autenticación es la llave que abre la puerta principal, y la autorización decide a qué habitaciones puedes entrar una vez dentro.
