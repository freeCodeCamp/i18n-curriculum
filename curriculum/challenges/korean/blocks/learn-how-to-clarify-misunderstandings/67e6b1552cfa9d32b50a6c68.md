# --explanation--

`Breaking into`는 무언가를 더 작은 부분으로 나누거나 분할하는 것을 의미합니다. 예를 들어:

`We are breaking the project into smaller tasks.` - 이것은 프로젝트를 관리하기 쉬운 조각으로 나누는 것을 의미합니다.

`Encapsulating`는 무언가(데이터나 기능 등)를 하나의 단위나 구조로 감싸서 세부사항을 숨기고 필요한 부분만 노출하는 것을 말합니다. 예를 들어:

`Encapsulating the data in an object means users can only access the data through specific methods, not directly.` - 이것은 데이터가 의도하지 않은 변경이나 오용으로부터 안전하게 관리되도록 보장합니다.
