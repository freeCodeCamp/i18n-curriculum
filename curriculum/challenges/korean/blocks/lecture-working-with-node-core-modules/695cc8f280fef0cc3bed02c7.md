# --description--

JavaScript는 원래 웹 브라우저에서 실행되도록 만들어졌으며, 주된 역할은 웹 페이지를 인터랙티브하게 만드는 것이었습니다. 이 때문에 초기 JavaScript는 주로 폼 내 텍스트 처리와 문서 객체 모델(DOM) 조작에 집중했습니다.

하지만 웹상의 모든 데이터가 텍스트인 것은 아닙니다. 파일, 이미지, 비디오는 이진 데이터로, 이를 처리하려면 다른 방식이 필요합니다. 브라우저에서는 이러한 데이터 유형을 보통 JavaScript 자체가 아니라 브라우저의 특수 컴포넌트가 처리합니다.

최신 브라우저는 렌더링 엔진과 JavaScript 엔진을 사용해 이런 작업을 관리합니다 — 예를 들어 Chrome의 Blink(V8), Safari의 WebKit(JavaScriptCore), Firefox의 Gecko(SpiderMonkey)가 있습니다.

Node.js는 브라우저에서 실행되지 않으므로, 특히 파일 입출력(I/O)과 TCP 스트림처럼 데이터가 청크 단위로 들어오는 경우 이진 데이터를 처리할 자체적인 방법이 필요했습니다. 바로 여기서 `Buffer` 모듈이 등장합니다.

Node.js의 `Buffer` 모듈은 파일, 이미지, 네트워크 스트림 같은 이진 데이터를 직접 다룰 수 있게 해줍니다. 이를 통해 이진 데이터를 메모리에 직접 저장하고 조작할 수 있습니다.

`fs` 모듈과 마찬가지로 `Buffer`도 Node.js의 핵심 모듈 중 하나이므로, 사용하기 전에 별도로 설치할 필요가 없습니다.

사용하려면 먼저 구조 분해 할당으로 모듈을 가져오세요:

```js
const { Buffer } = require("buffer");
```

그다음 제공하는 메서드로 `Buffer`를 호출하세요. 예를 들어 `Buffer.from()`는 문자열, 배열 또는 기타 원시 데이터로부터 버퍼를 생성할 수 있게 해줍니다.

문자열과 배열로 `Buffer`를 생성하는 방법은 다음과 같습니다:

```js
// Create a buffer from a string
const myStrBuffer = Buffer.from("freeCodeCamp");
console.log(myStrBuffer); // <Buffer 66 72 65 65 43 6f 64 65 43 61 6d 70>

// Create a buffer from an array of numbers
const myNumBuffer = Buffer.from([
  70, 82, 69, 69, 67, 79, 68, 69, 67, 65, 77, 80,
]);

console.log(myNumBuffer); // <Buffer 46 52 45 45 43 4f 44 45 43 41 4d 50>
```

`Buffer` 모듈의 일부 메서드는 먼저 가져오지 않아도 사용할 수 있지만, 다른 메서드는 명시적으로 `Buffer`를 가져와야만 사용할 수 있습니다. 따라서 프로젝트에서 사용할 때는 항상 Buffer를 가져오는 것이 좋습니다.

배열처럼 개별 `buffer` 요소에 접근할 수도 있습니다:

```js
console.log(myNumBuffer[0]); // 70
console.log(myStrBuffer[0]); // 102
```

또한 버퍼에 `toString()` 메서드를 사용해 실제 내용을 확인할 수 있습니다:

```js
console.log(myStrBuffer.toString()); // freeCodeCamp
console.log(myNumBuffer.toString()); // FREECODECAMP
```

`Buffer.alloc()`는 주어진 크기(바이트 수)의 새 버퍼를 생성합니다. 내부의 모든 바이트는 자동으로 `0`로 채워집니다:

```js
const someBuffer = Buffer.alloc(10);
console.log(someBuffer); // <Buffer 00 00 00 00 00 00 00 00 00 00>
```

`alloc()` 메서드에 전달한 크기를 기준으로 버퍼가 0으로 초기화된 것을 확인할 수 있습니다.

이 버퍼에 `Buffer.write()` 메서드를 사용해 데이터를 쓸 수 있습니다:

```js
someBuffer.write("Hello fCC");

console.log(someBuffer); // <Buffer 48 65 6c 6c 6f 20 66 43 43 00>
console.log(someBuffer.toString()); // Hello fCC
```

버퍼가 저장할 수 있는 용량보다 더 많은 데이터를 쓰면, 초과된 부분은 잘립니다:

```js
someBuffer.write("Hello freeCodeCamp");

console.log(someBuffer); // <Buffer 48 65 6c 6c 6f 20 66 72 65 65>
console.log(someBuffer.toString()); // Hello free
```

마지막으로 `Buffer.byteLength()`를 사용해 특정 인코딩으로 문자열을 저장하는 데 필요한 바이트 수를 확인할 수 있습니다:

```js
console.log(Buffer.byteLength("Hello freeCodeCamp")); // 18
```

그 밖의 `Buffer` 메서드에는 다음이 포함됩니다:

- `Buffer.isBuffer()`: 주어진 객체가 버퍼인지 확인
- `Buffer.compare()`: 두 버퍼를 비교해 정렬 순서 반환
- `Buffer.concat()`: 여러 버퍼를 하나로 합침
