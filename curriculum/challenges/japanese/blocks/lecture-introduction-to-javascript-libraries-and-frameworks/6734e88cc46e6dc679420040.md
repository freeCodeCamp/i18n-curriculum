# --description--

小規模なバニラHTML、CSS、JavaScriptのプロジェクトとは異なり、新しいReactプロジェクトを始めるには、すべてが正しく動作するようにいくつかの追加のステップが必要です。必要な設定をすべて自分で行おうとする代わりに、それを自動で行ってくれるツールがあります。

プロジェクトのセットアップに最も人気のあるツールの一つがViteです。Viteはフランス語で「速い」という意味で、モダンなウェブプロジェクトのためにより高速な開発体験を提供することを目指したビルドツールです。ViteはReactだけでなく、VueやSvelteのような他のライブラリやフレームワークでも使えます。バニラJavaScriptのプロジェクトでも使うことができます。

Viteで新しいプロジェクトを作成するには、コマンドラインを使う必要があります。Windowsの場合はコマンドプロンプトやWindows PowerShellを使えます。Macの場合はターミナルアプリを使ってください。

npmを動かすために必要なNode.jsがインストールされていない場合は、公式Node.jsのウェブページ（https://nodejs.org）からインストールしてください。インストールが終わったらここに戻ってReactのインストールを続けてください。

コマンドラインを開いたら、次のコマンドを使えます。

```bash
npm create vite@latest my-react-app -- --template react
```

このコマンドはViteのReactテンプレートを使って、`my-react-app`という名前の新しいReactプロジェクトを作成します。

その後、新しいプロジェクトを開くと、Viteが用意したReactのボイラープレートコードを見ることができます。

Viteの素晴らしいところは、Reactプロジェクトを始めるのに絶対に必要なファイルだけを提供してくれる点です。

実際にプロジェクトをそのまま動かすには、次のコマンドをコマンドラインで使って依存関係をインストールする必要があります。

```bash
cd my-react-app
npm install
```

`cd my-react-app`コマンドは、プロジェクトがある正しいディレクトリに移動することを確認します。

`npm install`コマンドは、Reactプロジェクトを正しく動かすために必要な依存関係をインストールします。依存関係とは、React自体やReactDOM、その他の追加機能を提供するサードパーティのパッケージなど、Reactプロジェクトが正しく動作するために必要なライブラリやパッケージのことです。

`npm install`を実行すると、プロジェクトディレクトリ内の`package.json`ファイルを読み込み、そこにリストされたすべての依存関係をインストールします。これにより、必要なコンポーネントが欠けることなくReactアプリをビルドして実行できます。

`package.json`ファイルは、プロジェクトの名前、バージョン、依存関係などのメタデータを含む重要な設定ファイルです。スクリプトやライセンス情報、その他プロジェクトと依存関係の管理に役立つ設定も定義されています。

依存関係がインストールされると、プロジェクト内に`node_modules`という新しいフォルダーができているはずです。

`node_modules`フォルダーは、プロジェクトが必要とするすべてのパッケージやライブラリが保存される場所です。このフォルダーには、`package.json`ファイルにリストされた依存関係の実際のコードが含まれており、プロジェクトの直接の依存関係だけでなく、それらの依存関係の依存関係も含まれます。

プロジェクトを実行するには、`npm run dev`コマンドを実行し、`http://localhost:5173/`で新しいブラウザタブを開いてください。

Viteが用意したスターターテンプレートが表示されるはずです。

ローカルサーバーを停止したい場合は、コマンドラインで`CTRL + C`を押してください。

このスターターテンプレートのコードを実際に見たい場合は、プロジェクト内の`src`フォルダーに入り、`App.jsx`ファイルを確認してください。

そこからファイルを編集し、変更を保存すると、ブラウザに新しい変更が表示されます。

これでReactアプリケーションの開発を始める準備が整いました！
