# --description--

让我们了解一下 `submit` 事件如何与超文本标记语言表单一起工作。首先，我们需要了解如何提交表单。表单可以通过三种方式提交。

第一个情况是当用户点击表单中 `type` 属性设置为 `submit` 的按钮时。

第二种情况是当用户在表单中任何可编辑的输入字段上按下 Enter 键时。

第三种是通过 JavaScript 调用表单元素的 `requestSubmit()` 或 `submit()` 方法。

但是当一个 `form` 被提交时会发生什么？会发生几件事，且该 `form` 的行为取决于它的属性。

我们需要查看的第一个属性是 `action` 属性。`action` 属性应包含一个 URL 或当前域的相对路径。该值决定了表单尝试发送数据的位置。如果你没有设置 `action` 属性，表单将把数据发送到当前页面的 URL。以下是一个将 `action` 属性设置为特定 URL 的表单示例：

```html
<form action="https://freecodecamp.org">
  <input type="number" id="input" placeholder="Enter a number" name="number" />
  <button type="submit">Submit</button>
</form>
```

当此 `form` 被提交时，它将向 freeCodeCamp 的主页发送 `data`，这可能不会有任何作用。

这是另一个提交到相对 `path` 的 `form`：

```html
<form action="/data">
  <input type="number" id="input" placeholder="Enter a number" name="number" />
  <button type="submit">Submit</button>
</form>
```

这个会提交到当前域的 `/data` 路径。在我们的分支中，它会提交到 `http://127.0.0.1:5500/data`。

第二个用于控制提交行为的属性是 `method` 属性。该属性接受标准的超文本传输协议方法，例如 `GET` 或 `POST`，并在向 `action` URL 发起请求时使用该方法。

HTTP 代表超文本传输协议，它用于通过网络传输数据。

超文本传输协议方法用于定义可以对资源执行的操作，例如 `GET`、`POST`、`PUT`、`DELETE` 等。你将在后续课程中学习更多关于这些方法的内容。

当未设置方法时，表单将默认使用 `GET` 请求。`GET` 请求用于从指定资源检索数据而不对其进行任何更改，参数通常以查询字串的形式追加到 URL 中。

表单中的数据将作为 `name=value` 点对进行 URL 编码，并作为查询参数追加到 `action` URL。比如，通过提交我们之前示例中 `input` 字段值为 `3342` 的表单，表单将尝试 `GET` URL `http://127.0.0.1:5500/data?number=3342`。

这非常适合像查找形式这样的场景，在那里你的用户正在查询数据。但如果你想让你的用户提交新数据，比如在注册形式中，惯用的方法是使用 `POST` 方法。`POST` 方法用于向服务器发送数据以创建或更新资源。

让我们将 `method` 属性设置为 `POST` 的值：

```xml
<form action="/data" method="POST">
  <input type="number" id="input" placeholder="Enter a number" name="number" />
  <button type="submit">Submit</button>
</form>
```

当你发送一个 `POST` 请求时，可以包含一个体，其中包含你的请求数据。因此，与 `GET` 不同，数据不会追加到 URL 中，我们的表单将 `POST` 请求发送到 `http://127.0.0.1:5500/data`。数据则可以在请求的体中找到，仍然是 URL 编码的表单数据。

URL 编码的表单数据是指将表单数据转换成密钥-值点对的字串，例如 `name=John+Doe&email=john%40example.com`，其中特殊字符被替换为编码版本，以便安全地通过网络发送数据。你将在后续课程中学习更多内容。

但是也许你不想将数据作为 URL 编码的表单负载发送？

`form` 元素接受一个 `enctype` 属性，该属性表现用于数据的编码类型。

此属性仅接受三个值：`application/x-www-form-urlencoded`（这是默认值，将数据作为 URL 编码的表单体发送）、`text/plain`（以纯文本形式发送数据，使用换行符分隔的 `name=value` 点对），或 `multipart/form-data`，专门用于处理带有文件上传的表单。

在本课中，你已经学习了表单提交的基础知识。在以后的课程中，我们将深入探讨如何使用表单以及它们如何与服务器交互。
