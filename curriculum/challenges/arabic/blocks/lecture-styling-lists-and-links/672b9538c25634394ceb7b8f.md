# --interactive--

هناك حالات مختلفة للرابط، منها `link`، و`visited`، و`hover`، و`focus`، و`active`. هذه الحالات مهمة لمساعدة المستخدمين على التعرف على الروابط وتوفير ردود فعل واضحة بعد التفاعل، مما يحسن كل من سهولة الاستخدام وإمكانية الوصول.

تنسيق هذه الحالات المختلفة للروابط ضروري لسهولة الاستخدام وإمكانية الوصول، إذ يوفر إشارات بصرية عن الحالة الحالية للرابط. هذا يساعد المستخدمين على فهم الروابط التي زاروها، والرابط الذي يتفاعلون معه، وما سيحدث عند النقر عليه.

للمستخدمين الذين يعانون من إعاقة بصرية أو صعوبات إدراكية، يمكن لهذه الأنماط المميزة أن تجعل التنقل أسهل وأكثر بديهية.

بالإضافة إلى ذلك، تعزز حالات الروابط الواضحة تجربة المستخدم بشكل عام من خلال توفير ردود فعل فورية على تفاعلات المستخدم، مما يقلل الالتباس ويحسن قابلية التنقل في الموقع.

يمكن تنسيق هذه الحالات باستخدام ما يُسمى بـ `pseudo-classes` في CSS.

الفئة المزيفة هي كلمة مفتاحية تُضاف إلى المحدد لتحديد حالة خاصة للعنصر المحدد.

على سبيل المثال، يمكن لـ `:hover` تغيير لون زر عندما يحوم مؤشر المستخدم فوقه، بينما يمكن لـ `:visited` تغيير لون الرابط الذي تم زيارته بالفعل.

تسمح الفئات المزيفة بتنسيق العناصر بناءً على حالتها أو تفاعل المستخدم معها، دون الحاجة إلى ترميز إضافي في HTML.

صيغة `pseudo-class` النحوية تبدو شيئًا مثل هذا حيث يكون `A` هو المحدِّد و`:B` هو الـ `pseudo-class`:

```css
A:B {
  property: value;
}
```

لفهم أفضل لكيفية تنسيق حالات الروابط المختلفة، دعنا نلقي نظرة على بعض الأمثلة.

تنسق الفئة المزيفة `:link` الروابط غير المزارة، مشيرة إلى أنها قابلة للنقر.

إليك مثالًا على استهداف عنصر وسم الرابط واستخدام الفئة المزيفة `:link`:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<a href="/">Example link</a>
```

```css
/* Normal state (unvisited link) */
a:link { 
  color: red;
}
```

:::

المثال أعلاه سيغير اللون الأزرق الافتراضي للرابط إلى `red` عندما يكون غير مزار.

تنسق الفئة المزيفة `:visited` الروابط التي تم زيارتها أو النقر عليها مسبقًا، مما يساعد المستخدمين على تتبع الروابط التي نقروا عليها سابقًا. إليك مثالًا على استخدام الفئة المزيفة `:visited`:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<a href="https://www.freecodecamp.org/learn/" target="_blank">freeCodeCamp</a>
```

```css
/* Visited link */
a:visited {
  color: green;
}
```

:::

ستلون هذه الشفرة الرابط باللون `green` عند النقر عليه.

تغير الفئة المزيفة `:hover` نمط الرابط عندما يحوم المستخدم فوقه، مما يوفر إشارة بصرية بأن الرابط تفاعلي. إليك مثالًا على استخدام الفئة المزيفة `:hover`:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<a href="/">Example link</a>
```

```css
/* Hover state */
a:hover {
  color: green;
}
```

:::

ستلون هذه الشفرة الروابط باللون `green` عندما يحوم المؤشر فوقها.

تضيف الفئة المزيفة `:focus` أنماطًا حول الرابط عندما يكون في حالة تركيز، مثل التنقل باستخدام لوحة المفاتيح، أو لتعزيز إمكانية الوصول.

إليك مثالًا يستخدم خاصية `outline` لتطبيق إطار برتقالي صلب عندما يكون الرابط في حالة تركيز.

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<a href="/">Example link</a>
```

```css
/* Focus state */
a:focus {
  outline: 2px solid orange;
}
```

:::

تغير الفئة المزيفة `:active` أنماط الرابط أثناء النقر عليه، مما يوفر رد فعل فوري للمستخدم بأن إجراءه يُسجل. إليك مثالًا على استخدام الفئة المزيفة `:active`:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<a href="/">Example link</a>
```

```css
/* Active state */
a:active {
  color: pink;
}
```

:::

سيجعل هذا المثال لون الرابط `pink` فور النقر عليه.
