# --explanation--

`Critical` означає щось дуже важливе або необхідне, а `critical data` стосується важливої інформації, на яку команда покладається для ефективної роботи. Втрата такої інформації спричинить затримки або проблеми. Наприклад:

- `The backup includes all critical data in case of a failure.` — Збережена копія містить необхідну інформацію для відновлення після проблеми.

- `You must encrypt critical data to ensure it's secure.` — Захист важливої інформації за допомогою шифрування необхідний для безпеки.
