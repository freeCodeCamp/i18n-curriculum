# --interactive--

## الدوال النقية مقابل الدوال غير النقية

- الدالة النقية هي التي تُنتج دائمًا نفس الناتج لنفس المُدخَل ولا تُحدث أي تأثيرات جانبية. يعتمد ناتجها فقط على المُدخَل، ولا تُعدل أي حالة خارجية.
- الدوال غير النقية تُحدث تأثيرات جانبية، وهي تغييرات في حالة البرنامج يمكن ملاحظتها خارج الدالة.

## البرمجة الوظيفية

- البرمجة الوظيفية هي نهج في تطوير البرمجيات يركز على استخدام الدوال لحل المشكلات، مع التركيز على ما يجب فعله بدلاً من كيفية فعله.
- تشجع البرمجة الوظيفية على استخدام تقنيات تساعد في تجنب التأثيرات الجانبية، مثل استخدام هياكل بيانات غير قابلة للتغيير والدوال ذات الرتبة العليا.
- عند استخدامها بشكل صحيح، تؤدي مبادئ البرمجة الوظيفية إلى شفرة أنظف وأسهل في الصيانة.

## التطبيق الجزئي

- التطبيق الجزئي هو تقنية في البرمجة الوظيفية تحول دالة ذات معلمات متعددة إلى متتالية من الدوال، كل منها يأخذ معلمة واحدة.

إليك مثالًا على دالة عادية مقابل دالة مطبقة جزئيًا:

:::interactive_editor

```js
// Regular function
function average(a, b, c) {
  return (a + b + c) / 3;
}

console.log(average(2, 3, 4)); // 3

// Curried function
function curriedAverage(a) {
  return function(b) {
    return function(c) {
      return (a + b + c) / 3;
    };
  };
}

console.log(curriedAverage(2)(3)(4)); // 3
```

:::

- التطبيق الجزئي يكون قويًا بشكل خاص عند العمل مع دوال تأخذ العديد من المعلمات.
- يجعل التطبيق الجزئي شفرتك أكثر مرونة وأسهل في إعادة الاستخدام.
- يمكنك استخدام الدوال السهمية لإنشاء دوال مطبقة جزئيًا بشكل أكثر إيجازًا:

```js
const curriedAverage = a => b => c => (a + b + c) / 3;
```

- رغم أن التطبيق الجزئي يمكن أن يؤدي إلى شفرة أكثر مرونة وقابلية لإعادة الاستخدام، إلا أنه قد يجعل الشفرة أصعب في القراءة إذا أُفرط في استخدامه.
