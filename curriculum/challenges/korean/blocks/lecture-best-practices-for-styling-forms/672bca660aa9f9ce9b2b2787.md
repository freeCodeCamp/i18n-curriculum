# --interactive--

`datetime-local`와 `color` 입력 같은 특수 입력 요소를 스타일링할 때 흔히 발생하는 문제에 대해 알아보겠습니다.

이러한 특수 입력 유형은 날짜 선택기나 색상 선택기 같은 기능을 만들기 위해 복잡한 가상 요소에 의존합니다. 이로 인해 이러한 입력 요소를 스타일링하는 데 큰 어려움이 있습니다. 한 가지 문제는 기본 스타일링이 전적으로 브라우저에 의존하기 때문에, 한 브라우저에서 선택기가 제대로 보이도록 만든 CSS가 다른 브라우저에서는 매우 다르게 나타날 수 있다는 점입니다.

다음은 색상 입력의 예입니다:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<form>
  <label for="favorite-color">Pick your favorite color:</label>
  <input type="color" id="favorite-color" name="favorite-color">
</form>
```

```css
input {
  padding: 8px 12px;
  margin: 8px 0;
  border-radius: 6px;
  border: 1px solid #ccc;
}

input[type="color"] {
  width: 60px;
  height: 40px;
  padding: 0;
  border: 2px solid #555;
  border-radius: 4px;
  cursor: pointer;
}

```

:::

또 다른 문제는 가상 요소의 복잡성일 수 있습니다. 날짜 선택기를 생각해 보면, 많은 움직이는 부분이 있고 가상 요소의 복잡한 구조 때문에 올바른 영역에 스타일을 적용하는 데 큰 어려움이 있을 수 있습니다.

다음은 날짜 입력의 예입니다:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<form>
  <label for="birthdate">Select your birthdate:</label>
  <input type="date" id="birthdate" name="birthdate">
</form>
```

```css
input {
  padding: 8px 12px;
  margin: 8px 0;
  border-radius: 6px;
  border: 1px solid #ccc;
}

input[type="date"] {
  padding: 6px 10px;
  border: 2px solid #555;
  border-radius: 4px;
  font-size: 14px;
  cursor: pointer;
}

input[type="date"]::-webkit-calendar-picker-indicator {
  background-color: #4CAF50;
  color: white;
  border-radius: 4px;
  cursor: pointer;
}

```

:::

물론, 이러한 복잡한 요소를 수동으로 스타일링할 때 중요한 기능을 실수로 잃을 위험도 있습니다. 포커스 상태나 선택된 항목 같은 중요한 표시기를 잃을 뿐만 아니라 선택기 자체가 완전히 작동하지 않을 수도 있습니다.

이러한 이유로 많은 개발자는 브라우저 내장 컴포넌트 대신 JavaScript 라이브러리나 맞춤 컴포넌트를 전적으로 사용하는 경우가 많습니다.
