# --description--

Wenn ein Ausdruck mehr als einen Operator enthält, wird die Ausführungsreihenfolge durch deren Priorität bestimmt, also mit anderen Worten die Standard-Prioritätsreihenfolge der Operatoren.

Aus der Schule wissen Sie, dass die Multiplikation im Ausdruck `1 + 2 * 2` vor der Addition berechnet werden sollte. Genau das ist die Priorität. Die Multiplikation hat eine höhere Priorität als die Addition.

Runde Klammern überschreiben jede Priorität, sodass Sie, wenn Sie mit der Standardreihenfolge nicht zufrieden sind, diese verwenden können, um sie zu ändern. Schreiben Sie zum Beispiel `(1 + 2) * 2`.

Es gibt viele Operatoren in JavaScript. Jeder Operator hat eine entsprechende Prioritätszahl. Der mit der größeren Zahl wird zuerst ausgeführt. Haben zwei Operatoren die gleiche Priorität, erfolgt die Ausführung von links nach rechts.

Hier ein Auszug aus der Prioritätstabelle (Sie müssen sich das nicht merken, aber beachten Sie, dass unäre Operatoren eine höhere Priorität als die entsprechenden binären haben):

| Priorität | Name             | Zeichen |
|-----------|------------------|---------|
| ...       | ...              | ...     |
| 14        | unäres Plus      | +       |
| 14        | unäre Negation   | -       |
| 13        | Exponentiation   | **      |
| 12        | Multiplikation   | *       |
| 12        | Division         | /       |
| 11        | Addition         | +       |
| 11        | Subtraktion      | -       |
| ...       | ...              | ...     |
| 2         | Zuweisung        | =       |
| ...       | ...              | ...     |



# --questions--

## --text--

Wenn Sie in einem JavaScript-Ausdruck mehrere Operatoren verwenden, welche Aussage trifft bezüglich der Ausführungsreihenfolge zu?

## --answers--

Operatoren werden unabhängig von ihrer Priorität von rechts nach links ausgeführt.

---

Operatoren mit höherer Priorität werden zuerst ausgeführt, und runde Klammern können die vordefinierte Priorität überschreiben.

---

Die Priorität der Operatoren wird durch deren Ausführungsgeschwindigkeit bestimmt.

---

Multiplikation hat immer Vorrang vor Addition, und unäre Operatoren werden nach binären ausgeführt.
