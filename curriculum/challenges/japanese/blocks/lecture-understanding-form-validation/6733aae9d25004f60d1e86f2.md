# --interactive--

前のレッスンでは、HTMLを使ってユーザーがフォームに送信できる値を制限する方法を学びました。しかし、それだけでは不十分な場合もあります。独自のエラーメッセージをユーザーに表示するなど、より複雑な処理をしたい場合はJavaScriptを使う必要があります。

`textarea` や `input` といった特定のHTML要素は、Constraint Validation APIを公開しています。このAPIを使うと、ユーザーが入力した値が最小長やパターンマッチングなど、HTMLレベルで設定したバリデーションを通過しているかどうかを確認できます。

では、実際にどう使うのでしょうか？例えば、会社の従業員が次のようなフォームでフィードバックメッセージを送るとします。

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css" />
<form>
  <label>Enter your email: </label>
  <input required type="email" />

  <label>Enter your feedback: </label>
  <textarea required placeholder="Your feedback here..."></textarea>

  <button type="submit">Submit Feedback</button>
</form>
```

ここでは、基本的なバリデーション（入力に`input`記号が含まれているかなど）をチェックする組み込みのバリデーション機能があるemail `@`を使っています。

しかし、ユーザーが`example@email.com`のようなメールアドレスを入力したらどうでしょう？これは基本的なバリデーションは通りますが、会社のメールアドレスからのメールだけを受け付けたい場合は不十分です。

そこで、emailアドレスが会社のメールアドレスで終わる必要があることを指定するために`pattern`属性を使います。更新後の例は次のようになります。

上の例では、DOMから入力要素を取得し、inputイベントリスナーを追加しています。

`e.target`はイベントを発生させた要素を指します。ここでは`input`です。しかし、`checkValidity()`メソッドとは何でしょうか？

これはConstraint Validation APIの一部です。`checkValidity()`メソッドは、要素がすべてのHTMLバリデーション（属性に基づく）を満たしていれば`true`を返し、満たしていなければ`false`を返します。

無効な入力で試すと、`false`がコンソールに記録されるのがわかります。入力が無効であることがわかったので、不正状態を報告しましょう。

const input = document.querySelector("input");

input.addEventListener("input", (e) => {
  if (!e.target.checkValidity()) {
    e.target.reportValidity();
  }
})

```js
const input = document.querySelector("input");

input.addEventListener("input", (e) => {
  if (!e.target.checkValidity()) {
    e.target.setCustomValidity(
      "You must use a company email address that ends in @sampleCompany.com"
    );
  }
});
```

このメソッドはカスタムのエラーメッセージを受け取り、ユーザーに表示します。その結果、カスタムエラーメッセージ`You must use a company email address that ends in @sampleCompany.com`が表示されます。

`validity`プロパティは`ValidityState`オブジェクトのインスタンスです。ブラウザでの例は次のようになります。

```js
ValidityState {
  badInput: false,
  customError: false,
  patternMismatch: true,
  rangeOverflow: false,
  rangeUnderflow: false,
  stepMismatch: false,
  tooLong: false,
  tooShort: false,
  typeMismatch: true,
  valueMissing: false,
  valid: true
}
```

いくつかの便利なプロパティはすべて`true`か`false`のブール値を持っています。

例えば、必須の入力フィールドが空の場合に`valueMissing`プロパティが`true`になることや、値が指定した正規表現パターンに一致しない場合に`patternMismatch`が`true`になることなどがあります。
