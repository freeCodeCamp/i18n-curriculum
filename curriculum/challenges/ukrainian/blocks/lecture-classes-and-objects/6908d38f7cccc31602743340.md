# --description--

У попередньому уроці ви дізналися, що атрибути — це змінні, які належать об’єкту. Це означає, що вони містять дані, які описують стан або поведінку об’єкта.

Наприклад, у автомобіля зазвичай є бренд і модель. Бренд і модель можуть бути атрибутами класу `Car`:

```python
class Car: 
    def __init__(self, brand, model): 
        self.brand = brand 
        self.model = model 

my_car = Car('Lamborghini', 'Gallardo') 
print(my_car.brand) # Lamborghini 
print(my_car.model) # Gallardo 
```

Але іноді ви можете не знати, які атрибути потрібні, поки програма не запуститься. Уявіть, що ви пишете скрипт, який отримує імена атрибутів від користувача або з конфігураційного файлу. Це не ті атрибути, які можна заздалегідь жорстко прописати в коді.

Ось тут і стає в пригоді динамічна робота з атрибутами. Таким чином ви можете отримувати доступ, змінювати, перевіряти або навіть видаляти атрибути за їхніми іменами як змінними, а не як фіксованими іменами в коді. Це дає вашій програмі гнучкість реагувати на різні дані або введення користувача на ходу.

Python надає чотири зручні вбудовані функції для динамічної роботи з атрибутами об’єктів. Це `getattr()`, `setattr()`, `hasattr()` і `delattr()`.

Вони дозволяють отримувати доступ, створювати, перевіряти та видаляти атрибути за змінними іменами. Давайте розглянемо кожну з них на практиці.

`getattr()` дає змогу прочитати атрибут об’єкта, коли ви не знаєте його ім’я до моменту запуску. Якщо атрибут не існує, вона викликає `AttributeError`, якщо ви не вказали значення за замовчуванням.

Щоб її використати, передайте об’єкт, ім’я атрибута та необов’язкове значення за замовчуванням:

```python
getattr(object, attribute_name, default_value) 
```

Ось приклад:

```python
class Person: 
    def __init__(self, name, age): 
        self.name = name 
        self.age = age 

person = Person('John Doe', 30) 
 
print(getattr(person, 'name')) # John Doe 
print(getattr(person, 'age')) # 30 
print(getattr(person, 'city', 'Milano')) # Milano
```

У наведеному прикладі `Milano` — це значення за замовчуванням, бо `city` не існує в класі `Person`.

Як ми вже казали, справжня сила `getattr()` проявляється, коли ім’я атрибута надходить зі змінної, наприклад, від користувача або з файлу.

У такому випадку ви не можете використовувати звичайний синтаксис `object.attribute_name`, бо ім’я атрибута не є фіксованим.

```python
class Person: 
    def __init__(self, name, age): 
        self.name = name 
        self.age = age 

person = Person('John Doe', 30)

attr_name = input('Enter the attribute you want to see: ')
print(getattr(person, attr_name, 'Attribute not found'))
```

У цьому випадку, якщо користувач введе `name`, він побачить `John Doe`, а якщо введе `age`, побачить `30`. Якщо ж введе щось, чого немає в класі, наприклад `email`, побачить `Attribute not found`.

Саме тут динамічна робота з атрибутами показує свою перевагу. Вона дозволяє вашому коду реагувати на введення та дані, яких він раніше не бачив.

Крім того, ви можете захотіти переглянути всі атрибути об’єкта, а не лише ті, що ви вже знаєте. Вбудована функція `dir()` дозволяє це зробити. Вона повертає список усіх імен атрибутів об’єкта. Ось як її використовувати:

```python
class Person: 
    def __init__(self, name, age): 
        self.name = name 
        self.age = age 

person = Person('John Doe', 30)

# Loop through all attributes of the person object with dir() function
for attr in dir(person):
    # Ignore dunder methods like __init__ or __str__ and regular methods
    if not attr.startswith('__') and not callable(getattr(person, attr)): 
        value = getattr(person, attr)
        print(f'{attr}: {value}')

# Output
# age: 30
# name: John Doe
```

Функція `setattr()` дозволяє динамічно створювати новий атрибут або оновлювати існуючий. Синтаксис виглядає так:

```python
setattr(object, attribute_name, value) 
```

Ось приклад, який встановлює атрибути конфігурації на основі даних із конфігураційного файлу або змінної середовища:

```python
class Configuration:
    pass

# Data loaded at runtime (like from a config or env file)
settings_data = {
    'server_url': 'https://api.example.com',
    'timeout_sec': 30,
    'max_retries': 5
}

config_obj = Configuration()

# Dynamically set attributes using dictionary keys and values
for attr_name, attr_value in settings_data.items():
    setattr(config_obj, attr_name, attr_value)

print(config_obj.server_url) # https://api.example.com
print(config_obj.timeout_sec) # 30
```

Також є `hasattr()`. Перед тим, як щось робити з атрибутом або видаляти його, корисно перевірити, чи він існує. Саме це робить `hasattr()`. Вона перевіряє наявність атрибута і повертає `True` або `False` залежно від результату.

Ось базовий синтаксис:

```python
hasattr(object, attribute_name)  
```

А ось приклад, який динамічно перевіряє наявність атрибутів у екземпляра класу `Product`:

```python
class Product:
    def __init__(self, name, price):
        self.name = name
        self.price = price

product_a = Product('T-Shirt', 25)

required_attributes = ['name', 'price', 'inventory_id']

for attr in required_attributes:
    if not hasattr(product_a, attr):
        print(f"ERROR: Product is missing the required attribute: '{attr}'")
    else:
        # Access the attributes dynamically once their existence is confirmed
        print(f'{attr}: {getattr(product_a, attr)}')

# Output:
# name: T-Shirt
# price: 25
# ERROR: Product is missing the required attribute: 'inventory_id'
```

Помилка виникла, бо `inventory_id` відсутній у класі `Product` та його екземплярі.

Нарешті, `delattr()` дозволяє динамічно видаляти атрибут:

```python
delattr(object, attribute_name) 
```

Наприклад, уявіть, що об’єкт повністю оброблений, і ви хочете очистити будь-які чутливі або тимчасові атрибути, які можуть існувати, перед збереженням фінальної версії. Після цього можна використати `dir()`, щоб пройтися циклом по решті атрибутів:

```python
class UserSession:
    def __init__(self, user_id, token):
        self.user_id = user_id
        self.auth_token = token # sensitive
        self.temp_counter = 0 # temporary

session = UserSession(101, 'a1b2c3d4e5')

# List of attributes to remove dynamically before "saving" the session
attributes_to_clean = ['auth_token', 'temp_counter']

# Dynamically remove specified attributes
for attr in attributes_to_clean:
    if hasattr(session, attr):
        delattr(session, attr)
        print(f'Removed attribute: {attr}')

print('\nFinal attributes remaining:')

# Loop through the remaining attributes with dir()
for attr in dir(session):
    # Ignore dunder methods like __init__ or __str__ and regular methods
    if not attr.startswith('__') and not callable(getattr(session, attr)):
        print(f' - {attr}: {getattr(session, attr)}')

# Output:
# Removed attribute: auth_token
# Removed attribute: temp_counter

# Final attributes remaining:
#  - user_id: 101
```

Ось так ви можете динамічно працювати з атрибутами!
