# --description--

デバッグはPython開発者にとって欠かせないスキルです。基本的な手法を理解することで、問題を効率よく特定し修正できます。

デバッグとは、コードのエラーやバグを特定し解決するプロセスです。コードを調べ、処理の流れを理解し、問題の原因を突き止めるためのツールを使います。

このレッスンでは、次のPythonプロジェクトで使える一般的なデバッグ手法を紹介します。

## `print()`関数とf文字列の活用

まず、コードのさまざまな箇所で`print()`関数とf文字列を使うことで、処理の流れや変数の状態を把握できます。例えば：

```py
def add(a, b):
    result = a + b
    print(f'Adding {a} and {b} gives {result}')
    return result
```

`a`、`b`、`result`の値を出力することで、関数が期待通りに動作しているか確認できます。

## 組み込みの`pdb`モジュールを使った対話型デバッグ

次に、Pythonの組み込み`pdb`モジュールを使った対話型デバッグが利用できます：

```py
import pdb

def divide(a, b):
    pdb.set_trace()
    return a / b

print(divide(10, 2))
```

`set_trace()`関数でトレースを設定すると、コードをステップ実行し、変数を調べ、プログラムの動作を理解できます。

上記のコードを実行すると、実行中のファイルの場所、`set_trace()`関数を呼び出した行とその直後のコード、そして対話型の`pdb`プロンプトが表示されます：

```py
> /Users/fcc/Desktop/debugging.py(5)divide()
-> return a / b
(Pdb)
```

プロンプトにhelpと入力すると、使えるコマンドの一覧が表示されます：

```py
(Pdb) help

Documented commands (type help <topic>):
========================================
EOF    c          d        h         list      q        rv       undisplay
a      cl         debug    help      ll        quit     s        unt      
alias  clear      disable  ignore    longlist  r        source   until    
args   commands   display  interact  n         restart  step     up       
b      condition  down     j         next      return   tbreak   w        
break  cont       enable   jump      p         retval   u        whatis   
bt     continue   exit     l         pp        run      unalias  where    

Miscellaneous help topics:
==========================
exec  pdb
```

これらのコマンドを使ってコードをデバッグできます。

例えば、その時点でコード内の要素の型を調べたい場合は、`whatis`コマンドを使います：

```py
(Pdb) whatis a
<class 'int'>
(Pdb) whatis divide
Function divide
```

ご覧の通り、`.set_trace()`を実行した時点で、パラメータaの型は整数で、divideは関数です。

コードの実行を続けるには、`continue`コマンド、またはその別名である`cont`や`c`を使います：

```py
(Pdb) continue
5.0
```

## IDEのデバッグツール

多くの統合開発環境（IDE）は、ブレークポイント、ステップ実行、変数の検査などの高度なデバッグツールを提供しています。

## VS Codeデバッガーの使い方

VS Codeを使う場合、コードにブレークポイントを設定し、デバッガーを起動してその箇所で実行を一時停止できます。以下は同じ`divide`関数をデバッグする方法です。

**ステップ1: コードの準備** `main.py`というファイルを作成し、以下の内容を記述します：

```py
def divide(a, b):
    result = a / b
    return result

print(divide(10, 2))
print(divide(15, 3))
```

**ステップ2: ブレークポイントを設定**

1. 2行目（`result = a / b`）の左側のガターをクリックしてブレークポイントを設定します

2. 赤い点が表示され、ブレークポイントが設定されたことを示します

**ステップ3: デバッグ開始**

1. `F5`を押すか、メニューの「実行」＞「デバッグの開始」を選択します

2. プロンプトが表示されたら「Python File」を選びます

3. ブレークポイントで実行が一時停止します

**ステップ4: 変数の確認**

- 変数にカーソルを合わせると現在の値が表示されます

- 左側の変数パネルでローカル変数をすべて確認できます

- 下部のデバッグコンソールで式を評価できます

**ステップ5: コードのステップ実行**

デバッグツールバーを使って：

- **続行 (F5)**：次のブレークポイントまで実行を再開

- **ステップオーバー (F10)**：現在の行を実行し次の行へ移動

- **ステップイン (F11)**：関数呼び出しの中に入る

- **ステップアウト (Shift+F11)**：現在の関数から抜ける

IDEのデバッグツールはプログラムの状態を視覚的に確認できるため、`print()`文だけを使うよりも問題の特定と修正が簡単になります。

これらの基本的なデバッグ手法、すなわち`print()`文、pdbモジュール、IDEツールをマスターすれば、Pythonコードの問題を効果的に特定し解決できます。各手法には役割があり、`print()`文は素早い確認に、pdbは対話的な調査に、IDEデバッガーは視覚的な検査に適しています。
