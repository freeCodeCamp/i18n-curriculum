# --description--

Katika JavaScript, mara nyingi hatuhitaji kutoa majina kwa vitendakazi vyetu, hasa tunapopita kitendakazi kama hoja kwa kitendakazi kingine. Badala yake, tunaunda vitendakazi ndani ya mstari. Hatuhitaji kutoa majina kwa vitendakazi hivi kwa sababu hatutavitumia tena mahali pengine.

Ili kufanikisha hili, mara nyingi tunatumia sintaksia ifuatayo:

```js
const myFunc = function() {
  const myVar = "value";
  return myVar;
}
```

ES6 inatupatia sukari ya sintaksia ili tusilazimike kuandika vitendakazi visivyo na jina kwa njia hii. Badala yake, unaweza kutumia **sintaksia ya kitendakazi cha mshale**:

```js
const myFunc = () => {
  const myVar = "value";
  return myVar;
}
```

Wakati hakuna mwili wa kitendakazi, na kuna thamani ya return tu, sintaksia ya kitendakazi cha mshale inakuwezesha kuacha neno kuu `return` pamoja na mabano yanayozunguka msimbo. Hii husaidia kurahisisha vitendakazi vidogo kuwa sentensi moja ndani ya mstari mmoja:

```js
const myFunc = () => "value";
```

Msimbo huu bado utarudisha mfuatano wa herufi `value` kwa chaguo-msingi.

# --instructions--

Andika upya kitendakazi kilichopewa thamani kwa kigezo `magic` kinachorudisha `new Date()` kwa kutumia sintaksia ya kitendakazi cha mshale. Pia, hakikisha hakuna kitu kilichotangazwa kwa kutumia neno kuu `var`.
