# --description--

In una lezione precedente, hai imparato a lavorare con gli alias di tipo e hai visto esempi su come nominare i tipi di oggetto.

Come ricorderai, un modo per nominare un tipo di oggetto è usare la parola chiave `type` in questo modo:

```ts
type ID = number | string;

type User = {
  id: ID;
  name: string;
};
```

Un altro modo per nominare un tipo di oggetto è usare una dichiarazione `interface` in questo modo:

```ts
type ID = number | string;

interface User {
  id: ID;
  name: string;
}
```

`type` e `interface` sono simili ma una differenza chiave è che le interfacce usano la parola chiave `extends` per estendere altre interfacce, mentre gli alias `type` usano tipi di intersezione (`&`) per combinare tipi.

Vediamo alcuni esempi:

```ts
interface User {
  id: number | string;
  name: string;
}

interface Admin extends User {
  role: string;
}

const adminUser: Admin = {
  id: 101,
  name: "Alice",
  role: "superadmin"
};
```

In questo esempio, `User` inizia con `id` e `name`. L'interfaccia `Admin` estende `User`, il che significa che include `id` e `name` e aggiunge una nuova proprietà chiamata `role`.

Se volessi estendere un `type`, useresti un tipo di intersezione (`&`) in questo modo:

```ts
type User = {
  id: number | string;
  name: string;
};

type Admin = User & {
  role: string;
};

const adminUser: Admin = {
  id: 101,
  name: "Alice",
  role: "superadmin"
};
```

`Admin` combina il tipo `User` con un nuovo tipo di oggetto che include `role`. Il risultato ha la stessa forma dell'esempio con l'interfaccia, cioè un oggetto con `id`, `name` e `role`.

Sia `type` che `interface` possono descrivere la forma di un oggetto. Se non sei sicuro quale usare, ricorda che `interface` è progettato specificamente per le forme degli oggetti, mentre `type` è più flessibile e può anche rappresentare tipi unione, primitivi e combinazioni di tipi più complesse.
