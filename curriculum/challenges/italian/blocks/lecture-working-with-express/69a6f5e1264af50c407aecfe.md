# --description--

Nella lezione precedente, hai creato una cartella di progetto e installato Express:

```bash
mkdir my-app
cd my-app
npm init -y
npm install express
```

Per ricordarti, questi comandi creano una nuova cartella di progetto, `cd` nella cartella del progetto, inizializzano il progetto e creano un file `package.json` di base, e installano Express.

Ora, apri un editor di codice, crea un file chiamato `index.js` e scrivi il tuo primo server Express:

```javascript
const express = require("express");
const app = express();
const port = 3000;

app.get("/", (req, res) => {
  res.send("Hello World!");
});

app.listen(port, () => {
  console.log(`Example app listening on port ${port}`);
});
```

Puoi eseguire il tuo codice con il seguente comando nel terminale:

```bash
node index.js
```

Apri il browser su [http://localhost:3000](http://localhost:3000) e vedrai il messaggio **"Hello World!"**

È così veloce iniziare con Express! Ora vediamo il codice passo dopo passo.

Nella prima riga di codice, stai caricando la libreria Express e assegnandola a una variabile chiamata `express`:

```javascript
const express = require("express");
```

La variabile `express` è una funzione factory. Quando la chiami, crea e restituisce una nuova istanza di applicazione Express:

```javascript
const app = express();
```

`app` ora ha accesso a tutti i metodi forniti da Express, come `.get()`, `.post()`, `.listen()` e così via. Questo è il tuo oggetto server principale.

La riga successiva definisce la variabile `port`. Una porta è un endpoint di rete su cui la tua applicazione ascolta, e `3000` è una porta comunemente usata per lo sviluppo locale:

```javascript
const port = 3000;
```

Ora, diamo un’occhiata alla parte successiva:

```javascript
app.get("/", (req, res) => {
  res.send("Hello World!");
});
```

Questo blocco di codice definisce una rotta e dice al server come rispondere alle richieste. `app.get()` registra un gestore per le richieste HTTP GET. Una richiesta GET è un metodo HTTP comunemente usato per recuperare dati.

L’argomento `"/"` dentro `app.get()` rappresenta il percorso della rotta. In questo caso, punta all’URL radice dell’applicazione.

La parte `(req, res) => { ... }` è una funzione di callback che viene eseguita quando una richiesta corrisponde a questa rotta. L’oggetto `req` contiene informazioni sulla richiesta in arrivo, e l’oggetto `res` viene usato per inviare una risposta al client.

Infine, `res.send("Hello World!")` invia una risposta al client. In questo esempio, la risposta è il testo semplice `"Hello World!"`.

Ora, diamo un’occhiata all’ultima parte di codice:

```javascript
app.listen(port, () => {
  console.log(`Example app listening on port ${port}`);
});
```

Il metodo `app.listen()` avvia l’applicazione Express e dice a Node di ascoltare le richieste HTTP in arrivo sulla porta specificata, in questo caso `3000`.

Una volta che il server parte, la funzione di callback viene eseguita e il messaggio `Example app listening on port 3000` viene mostrato nella console, confermando che l’app Express è partita con successo ed è pronta a ricevere richieste.
