## Чисті та нечисті функції

- Чиста функція — це така, що завжди повертає однаковий результат для однакових вхідних даних і не має побічних ефектів. Її вихідні дані залежать лише від вхідних, і вона не змінює жодного зовнішнього стану.
- Нечисті функції мають побічні ефекти — це зміни стану програми, які можна спостерігати поза межами функції.

## Функціональне програмування

- Функціональне програмування — це підхід до розробки програмного забезпечення, який наголошує на використанні функцій для розв’язання задач, зосереджуючись на тому, що потрібно зробити, а не як це зробити.
- Функціональне програмування заохочує використання технік, які допомагають уникати побічних ефектів, наприклад, використання незмінних структур даних і функцій вищого порядку.
- При правильному застосуванні принципи функціонального програмування ведуть до чистішого та легшого для підтримки коду.

## Каррінг

- Каррінг — це техніка функціонального програмування, яка перетворює функцію з кількома аргументами у послідовність функцій, кожна з яких приймає один аргумент.

Ось приклад звичайної функції та каррінгової функції:

:::interactive_editor

```js
// Regular function
function average(a, b, c) {
  return (a + b + c) / 3;
}

console.log(average(2, 3, 4)); // 3

// Curried function
function curriedAverage(a) {
  return function(b) {
    return function(c) {
      return (a + b + c) / 3;
    };
  };
}

console.log(curriedAverage(2)(3)(4)); // 3
```

:::

- Каррінг особливо корисний при роботі з функціями, що приймають багато аргументів.
- Каррінг робить ваш код гнучкішим і легшим для повторного використання.
- Ви можете використовувати стрілкові функції для створення каррінгових функцій більш лаконічно:

```js
const curriedAverage = a => b => c => (a + b + c) / 3;
```

- Хоча каррінг робить код гнучкішим і більш повторно використовуваним, надмірне його використання може ускладнити читання коду.
