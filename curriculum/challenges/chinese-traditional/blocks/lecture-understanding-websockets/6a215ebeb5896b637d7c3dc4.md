# --description--

Node.js 沒有內建的 WebSocket 伺服器。開發者通常會使用外部套件，例如 `ws`，來實作 WebSocket 協定並處理客戶端與伺服器之間的即時通訊。

`ws` 函式庫輕量、快速且易於使用。它實作了 WebSocket 協定，讓你可以創建和管理 WebSocket 連線。

你可以用 npm 安裝它：

```bash
npm install ws
```

安裝完成後，透過匯入 `ws` 函式庫，呼叫其 `Server` 方法並傳入埠號，即可創建 WebSocket 伺服器：

```js
const WebSocket = require("ws");

const server = new WebSocket.Server({
  port: 3000,
});
```

這會啟動一個 WebSocket 伺服器，監聽 `3000` 埠的連線。

當有客戶端連線時，伺服器會觸發 `connection` 事件。

```js
server.on("connection", (socket) => {
  console.log("Client connected");
});
```

你可以把這個事件想成伺服器在說「有新客戶端加入了」。

`socket` 參數代表伺服器與該客戶端之間的連線。你可以用它來傳送和接收訊息。

當已連線的客戶端傳送資料時，伺服器會觸發 `message` 事件。

```js
server.on("connection", (socket) => {
  socket.on("message", (message) => {
    console.log(message.toString());
  });
});
```

在這個範例中，伺服器監聽傳入的訊息並將其記錄到主控台。

你可以使用 `send()` 方法將資料傳回給客戶端：

```js
socket.send("Hello from the server");
```

這會透過開啟的 WebSocket 連線傳送訊息。

只要連線保持開啟，客戶端和伺服器就能持續即時交換訊息。

除了 `connection` 和 `message`，WebSocket 連線還會觸發 `close` 和 `error` 事件。

`close` 會在連線關閉時觸發：

```js
socket.on("close", () => {
  console.log("Disconnected");
});
```

`error` 則在發生錯誤時觸發：

```js
socket.on("error", (error) => {
  console.error(error);
});
```

這些事件共同構成大多數 WebSocket 應用的基礎。

聊天應用中常見的需求是廣播。廣播指的是將訊息傳送給多個已連線的客戶端，而不只是單一客戶端。

想像一個聊天室裡有三位使用者，分別是 **John Doe**、**Jane Doe** 和 **Wes Smith**。

如果 John Doe 傳送訊息，伺服器可能需要將該訊息轉發給 Jane Doe 和 Wes Smith。

為此，伺服器會追蹤已連線的客戶端，並在訊息到達時逐一迴圈處理。

在傳送訊息前，伺服器會檢查客戶端連線是否仍然開啟：

```js
if (client.readyState === WebSocket.OPEN) {
  // do something
}
```

每個 WebSocket 連線都有一個 `readyState` 屬性，描述其目前狀態。

`WebSocket.OPEN` 表示連線是作用中的，準備好傳送和接收資料。

透過檢查這點，伺服器避免向已斷線或連線尚未建立完成的客戶端傳送訊息：

```js
server.on("connection", (socket) => {
  socket.on("message", (message) => {
    server.clients.forEach((client) => {
      if (client.readyState === WebSocket.OPEN) {
        client.send(message.toString());
      }
    });
  });
});
```

在這個範例中，每個已連線的客戶端都會收到傳入的訊息。這就是所謂的廣播。

許多即時應用使用廣播來傳遞資料，例如聊天訊息、即時通知、體育賽事更新和多人遊戲事件。

使用 `ws` 函式庫，在 Node.js 中建置 WebSocket 伺服器非常簡單。

你只需要創建伺服器、監聽事件、交換訊息，並將更新廣播給已連線的客戶端。

這些基礎構件是許多即時應用的根基。
