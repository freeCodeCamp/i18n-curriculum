# --interactive--

L'operatore `typeof` in JavaScript è uno strumento semplice ma potente che ti permette di vedere il tipo di dato di una variabile o di un valore. Restituisce sempre una stringa che indica il tipo.

Diamo un'occhiata a qualche esempio:

:::interactive_editor

```js
let num = 42;
console.log(typeof num); // "number"
```

:::

In questo primo esempio, abbiamo creato una variabile chiamata `num` e le abbiamo assegnato il numero `42`. Quando usi l'operatore `typeof` sulla variabile chiamata `num`, restituirà la stringa `number`.

Ecco un altro esempio di utilizzo dell'operatore `typeof` su una variabile chiamata `isUserLoggedIn`:

:::interactive_editor

```js
let isUserLoggedIn = true;
console.log(typeof isUserLoggedIn); // "boolean"
```

:::

Quando usi l'operatore `typeof` sulla variabile `isUserLoggedIn`, restituirà una stringa `boolean` perché il booleano `true` è stato assegnato alla variabile.

Usare l'operatore `typeof` può essere particolarmente utile quando stai eseguendo il debug o cercando di capire con che tipo di dati stai lavorando nel tuo codice.

Tuttavia, c'è una nota stranezza ben conosciuta in JavaScript quando si tratta di `null`.

Diamo un'occhiata a un esempio:

:::interactive_editor

```js
let exampleVariable = null;
console.log(typeof exampleVariable); // "object"
```

:::

In questo esempio, abbiamo una variabile chiamata `exampleVariable` e le abbiamo assegnato il valore di `null`. Ma quando usiamo l'operatore `typeof`, restituisce la stringa `object`.

Questo è ampiamente considerato un bug in JavaScript, risalente ai suoi primi giorni. La ragione di questo comportamento è radicata nel modo in cui JavaScript è stato originariamente progettato.

Quando il linguaggio è stato implementato per la prima volta, valori come `null` erano rappresentati come un tipo speciale di oggetto, portando a questo risultato inaspettato.

Sfortunatamente, questo è diventato parte del linguaggio e, anche se è confuso, è qualcosa di cui dovrai essere consapevole.
