# --description--

- **同期的なJavaScript**は順番に実行され、前の操作が終わるまで次の操作に進みません。
- **非同期的なJavaScript**はメインスレッドをブロックせずに複数の操作をバックグラウンドで実行できます。
- **スレッド**はメインプログラムの流れとは独立して実行できる命令のシーケンスです。
- **コールバック関数**は他の関数に引数として渡され、操作の完了後やイベントの結果として実行される関数です。

## JavaScriptエンジンとJavaScriptランタイム

- **JavaScriptエンジン**はウェブブラウザでJavaScriptコードを実行するプログラムです。コードを受け取り、コンピュータが理解して動作できる命令に変換します。
- V8はGoogleが開発したJavaScriptエンジンの例です。
- **JavaScriptランタイム**はJavaScriptコードが実行される環境で、コードを処理・実行するJavaScriptエンジンと、ウェブブラウザやNode.jsのような追加機能を含みます。

## Fetch API

- Fetch APIはウェブアプリがネットワークリクエストを行い、通常はサーバーからデータを取得または送信するために使います。`fetch()` メソッドを使ってこれらのリクエストを行えます。
- Fetch APIを使ってテキスト、画像、音声、JSON、その他の種類のデータを取得できます。

## Fetch APIのHTTPメソッド

Fetch APIはサーバーとやり取りするために様々なHTTPメソッドをサポートしています。最も一般的なメソッドは以下の通りです：

- **GET**：サーバーからデータを取得するために使います。デフォルトでは、Fetch APIは`GET` メソッドを使ってデータを取得します。

```js
fetch('https://api.example.com/data')
```

取得したデータを使うには、`.json()` メソッドを使ってJSON形式に変換する必要があります：

```js
fetch('https://api.example.com/data')
  .then(response => response.json())
  .then(data => console.log(data))
```

このコードでは、Fetch APIからのレスポンスはPromiseであり、`.then` ハンドラがレスポンスをJSON形式に変換しています。

- **POST**：サーバーにデータを送信するために使います。`POST` メソッドはサーバー上に新しいリソースを作成するために使われます。

```js
fetch('https://api.example.com/users', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({
    name: 'John Doe',
    email: 'john@example.com'
  })
})
```

この例では、新しいユーザーを作成するために`POST` リクエストを送信しています。メソッドを`POST` に設定し、適切なヘッダーを指定し、送信したいデータを含むボディを設定しています。ボディは文字列である必要があるため、オブジェクトをJSON文字列に変換するために`JSON.stringify()` を使っています。

- **PUT**：サーバー上のデータを更新するために使います。`PUT` メソッドは既存のリソースを更新するために使われます。

```js
fetch('https://api.example.com/users/45', {
  method: 'PUT',
  headers: {
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({
    name: 'John Smith',
    email: 'john@example.com'
  })
})
```

この例では、URLの最後に指定されたID `45` のデータを更新しています。コードでは`PUT` メソッドを使い、更新するデータをボディとして指定しています。

- **DELETE**：サーバー上のデータを削除するために使います。`DELETE` メソッドはリソースを削除するために使われます。

```js
fetch('https://api.example.com/users/45', {
  method: 'DELETE'
})
```

この例では、ID `DELETE` のユーザーを削除するために`45` リクエストを送信しています。

## PromiseとPromiseチェーン

- **Promise**は非同期操作の最終的な完了または失敗と、その結果の値を表すオブジェクトです。Promiseの値は`async` 操作が完了したときにのみわかります。
- シンプルなPromiseを作成する例はこちらです：

```js
const promise = new Promise((resolve, reject) => {
  setTimeout(() => {
    resolve('Data received successfully');
  }, 2000);
});
```

- Promiseでは、Promiseが成功したときに何をするかを指定するために`.then()` メソッドが使われ、エラーが発生した場合は`.catch()` で処理します。
- Promiseで`.then()` と`.catch()` を使う例はこちらです：

```js
promise
  .then(data => {
    console.log(data);
  })
  .catch(error => {
    console.error(error);
  });
```

上の例では、Promiseから受け取ったデータをログに出力するために`.then()` メソッドが使われ、エラーが発生した場合は`.catch()` メソッドでログに出力しています。

- **Promiseチェーン**：Promiseの強力な機能の一つは、複数の非同期操作を連鎖させられることです。各`.then()` は新しいPromiseを返すことができ、非同期操作を順番に実行できます。
- Promiseチェーンの例はこちらです：

```js
fetch('https://api.example.com/data')
  .then(response => response.json())
  .then(data => {
    console.log(data);
    return fetch('https://api.example.com/other-data');
  })
  .then(response => response.json())
  .then(otherData => {
    console.log(otherData);
  })
  .catch(error => {
    console.error(error);
  });
```

上の例では、最初に1つ目のURLからデータを取得し、次に最初のレスポンスに基づいて別のURLからデータを取得し、最後に2つ目のデータをログに出力しています。

`catch` メソッドは処理中に発生したエラーを処理します。これにより各ステップでエラー処理を書く必要がなくなり、コードが大幅に簡単になります。

## `async/await` を使ったPromiseの処理

Async/awaitはPromiseの上に構築されており、非同期コードの記述と読み取りを簡単にします。

- **async**：`async` キーワードは非同期関数を定義するために使います。`async` 関数はPromiseを返し、そのPromiseは`async` 関数が返す値で解決されます。
- **await**：`await` キーワードは`async` 関数内で使い、Promiseが解決されるまで関数の実行を一時停止します。`async` 関数内でのみ使えます。
- `async/await` を使った例はこちらです：

```js
async function delayedGreeting(name) {
  console.log("A Messenger entered the chat...");
  await new Promise(resolve => setTimeout(resolve, 2000));
  console.log(`Hello, ${name}!`);
}

delayedGreeting("Alice");
console.log("First Printed Message!");
```

上の例では、`delayedGreeting` 関数は`async` 関数で、2秒間待ってから挨拶メッセージを出力します。`await` キーワードは`Promise` が解決されるまで関数の実行を一時停止します。

- `async/await` の最大の利点の一つは`try/catch` ブロックによるエラー処理です。例はこちらです：

```js
async function fetchData() {
  try {
    const response = await fetch('https://api.example.com/data');
    const data = await response.json();
    console.log(data);
  } catch (error) {
    console.error(error);
  }
}

fetchData();
```

上の例では、`try` ブロックにエラーが発生する可能性のあるコードがあり、`catch` ブロックでエラーを処理しています。これによりエラー処理がより簡潔で読みやすくなります。

## `async` 属性

- `async` 属性はブラウザにスクリプトファイルを非同期にダウンロードしながらHTMLドキュメントの解析を続けるよう指示します。
- スクリプトがダウンロードされるとHTML解析は一時停止し、スクリプトが実行され、その後HTML解析が再開されます。
- 実行順序が重要でない独立したスクリプトには`async` を使うべきです。

## `defer` 属性

- `defer` 属性もスクリプトを非同期にダウンロードしますが、HTMLドキュメントの解析が完全に終わるまでスクリプトの実行を遅延させます。
- `defer` スクリプトはHTMLドキュメントに現れる順序で実行順序を維持します。

- `async` と`defer` 属性はインラインスクリプトには無視され、外部スクリプトファイルにのみ有効です。

- `async` と`defer` 属性が両方ある場合は、`async` 属性が優先されます。

## Geolocation API

- Geolocation APIはウェブサイトがユーザーの位置情報を取得する方法を提供します。

- 以下の例は、ユーザーの現在位置を取得するための`getCurrentPosition()` メソッドを示しています。

```js
navigator.geolocation.getCurrentPosition(
  (position) => {
    console.log("Latitude: " + position.coords.latitude);
    console.log("Longitude: " + position.coords.longitude);
  },
  (error) => {
    console.log("Error: " + error.message);
  }
);
```

このコードでは、`getCurrentPosition` を呼び出し、位置情報が正常に取得されたときに呼ばれる関数を渡しています。

`position` オブジェクトには様々な情報が含まれていますが、ここでは`latitude` と`longitude` のみを選択しています。

`position` の取得に問題がある場合は、エラーがコンソールに出力されます。

- ユーザーのプライバシーを尊重し、必要な場合にのみ位置情報をリクエストすることが重要です。
