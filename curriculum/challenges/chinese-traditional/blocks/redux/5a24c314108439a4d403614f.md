# --description--

`dispatch` 方法是你用來將動作分派到 Redux store 的方式。呼叫 `store.dispatch()` 並傳入從動作建立者傳回的值，會將動作傳回給 store。

請記得，action creators 傳回一個具有 type 屬性的物件，該屬性指定已發生的動作類型。然後該方法將動作物件分派到 Redux store。根據前一個挑戰的範例，以下幾行是等價的，且兩者都分派類型為 `LOGIN` 的動作：

```js
store.dispatch(actionCreator());
store.dispatch({ type: 'LOGIN' });
```

# --instructions--

程式碼編輯器中的 Redux store 有一個已初始化的狀態，該狀態是一個包含目前設定為 `false` 的 `login` 屬性的物件。還有一個名為 `loginAction()` 的動作創建函式，它會傳回一個型別為 `LOGIN` 的動作。透過呼叫 `dispatch` 方法，並傳入由 `loginAction()` 創建的動作，將 `LOGIN` 動作分派到 Redux store。

# --hints--

呼叫函式（程式）`loginAction`應該傳回一個`type`屬性設定為字串`LOGIN`的物件。

```js
assert(loginAction().type === 'LOGIN');
```

該 store 應該以一個 `login` 屬性設為 `false` 的物件初始化。

```js

```

應該使用 `store.dispatch()` 方法來分派型別為 `LOGIN` 的動作。
