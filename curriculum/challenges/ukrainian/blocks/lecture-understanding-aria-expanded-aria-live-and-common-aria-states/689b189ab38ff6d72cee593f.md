# --interactive--

Раніше ви дізналися, як семантична розмітка та атрибути ARIA роблять вміст доступним. У реальних застосунках вміст рідко буває статичним, і сторінки часто оновлюються динамічно за допомогою JavaScript. Коли це відбувається, важливо переконатися, що ці зміни також відображаються в HTML. Це дозволяє читачам екрана точно передавати оновлений стан користувачам. Без цього люди, які покладаються на допоміжні засоби, можуть ніколи не дізнатися, що щось змінилося, або отримати застарілу чи оманливу інформацію.

Розглянемо поширений сценарій, коли JavaScript змінює стан елемента, і як ви можете зберегти доступність HTML, оновлюючи атрибути ARIA.

Припустімо, у вас є кнопка, яка відкриває і закриває випадаюче меню:

:::interactive_editor

```html
<button id="menuButton" aria-expanded="false" aria-controls="menuList">
  Menu
</button>

<ul id="menuList" hidden>
  <li>List item 1</li>
  <li>List item 2</li>
</ul>
<script src="index.js"></script>
```

```js
const button = document.getElementById("menuButton");
const menu = document.getElementById("menuList");

button.addEventListener("click", () => {
  const expanded = button.getAttribute("aria-expanded") === "true";
  button.setAttribute("aria-expanded", String(!expanded));
  menu.hidden = expanded;
});
```

:::

HTML налаштовує кнопку, яка керує випадаючим меню. Кнопка використовує `aria-expanded` для вказівки, чи відкрите меню, і `aria-controls` для зв’язку з меню, посилаючись на `id` меню.

JavaScript слухає натискання на кнопку меню. Коли на неї натискають, він переключає значення `aria-expanded` і оновлює властивість `hidden` у меню. Це підтримує синхронізацію видимості меню та інформації про доступність, щоб і користувачі зору, і користувачі читачів екрана знали, чи меню відкрите чи закрите.

Ви дізнаєтеся більше про ці атрибути ARIA в наступних уроках. А поки просто пам’ятайте: коли ваш JavaScript змінює стан елемента, ви також повинні оновити відповідні атрибути ARIA, щоб допоміжні технології точно відображали те, що насправді є на екрані.
