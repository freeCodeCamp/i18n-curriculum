# --description--

组件是 React 应用的创建块，允许开发者将复杂的用户界面拆分成更小、更易管理的部分，从而更容易开发和维护大规模应用。

React 中的两种组件类型是函数组件和基于类的组件。在现代 React 中，开发者将使用函数组件，我们今天看到的所有示例都是函数组件。

在更高层次上，你可以将组件视为返回描述 UI 的元素的 JavaScript 函数。

该 UI 使用 JSX 描述，JSX 是 JavaScript 的一种语法扩展名，看起来类似于超文本标记语言，但允许你以更声明式的方式编写 UI 元素。

让我们来看一个 React 组件的示例：

```jsx
function Greeting() {
  const name = "John"
  {/* The result will be Hello John*/}
  return <h1 className="title">Hello {name}</h1>;
}
```

在此示例中，我们定义了一个名为 `Greeting` 的组件。`h1` 标签内的括弧状的花括弧 `{}` 使我们能够嵌入 JavaScript 表达式，从而访问 `h1` 元素内的 `name` 变量。

我们还将名为 `title` 的 `className` 应用到 `h1` 元素。

但是为什么我们使用 `className` 而不是像常规超文本标记语言元素那样使用 `class`？

这是因为在 JavaScript 中，`class` 是一个保留名称。所以，我们需要使用 `className`。

我们还在 JSX 中使用了一个评论，显示结果将是什么。重要的是要注意，你可以使用像这样常规的评论语法，但它需要用括弧状的花括弧包裹起来才能生效：

```jsx
{/* Block Comments */}
```

你可能还注意到，我们在组件名称的开头使用了大写字母。但为什么不能全部使用小写字母呢？

这是因为 React 将大写字母开头的组件视为自定义组件，而小写字母开头的元素则被视为内置的超文本标记语言元素。

当 React 遇到小写标签，如 `<div>` 或 `<span>`，它会假设这是一个标准的超文本标记语言元素。然而，如果组件名称以大写字母开头，React 会将其视为用户定义的组件并相应地渲染它。这个区分帮助 React 区分本地的超文本标记语言标签和你创建的组件。

要在我们的应用中使用这个 `Greeting` 组件，我们会这样写：

```jsx
<Greeting />
```

这将渲染一个带有文本 `Hello John` 的 `h1` 元素到页面上。但仔细看看这里的语法。当我们使用该组件时，它以一个斜线然后是大于符号结尾。

在使用 JSX 时，所有标签和组件的使用都需要显式关闭。因此，如果组件或标签没有任何子元素，则需要像这里所示那样显式关闭它：

```jsx
<Greeting /> {/* /> is required */}
```

到目前为止，我们只看了如何渲染单个 `h1` 元素。但你实际上可以渲染多个元素。

让我们来看下面的示例代码：

```jsx
function Greeting() {
  const name = "John";
  {/* This will throw an error */}
  return <h1>Hello {name}</h1> 
  <p>Nice to meet you.</p>
}
```

我们正尝试添加另一句 `Nice to meet you`，但它未能正确渲染在页面上。

似乎出现了错误信息。错误信息显示 `Adjacent JSX elements must be wrapped in an enclosing tag.`

你收到该错误信息的原因是多个兄弟元素需要被包裹在一个父元素中。虽然你可以用一个简单的 `div` 来包裹 `h1` 和 `p` 元素，但还有另一种方法可以消除该错误。

React 片段用于将元素组合在一起。以下是修改后示例的样子：

```jsx
function Greeting() {
  const name = "John";
  return (
    <Fragment>
      <h1>Hello {name}</h1>
      <p>Nice to meet you.</p>
    </Fragment>
  );
}
```

你也可以选择使用空的 JSX 标签，它们可以作为片段的简写：

```jsx
function Greeting() {
  const name = "John";
  return (
    <>
      <h1>Hello {name}</h1>
      <p>Nice to meet you.</p>
    </>
  );
}
```
