# --description--

加號 `+` 存在兩種表單：你在上一個挑戰中使用的二進位表單和一元表單。

一元加號，換句話說，就是套用於單一值的加號運算子 `+`，對數字不會有任何作用。但如果運算元不是數字，一元加號會將它轉換成數字。

例如：

```js
// No effect on numbers
let x = 1;
console.log( +x ); // 1

let y = -2;
console.log( +y ); // -2

// Converts non-numbers
console.log( +true ); // 1
console.log( +"" );   //
```

將字串轉換為數字的需求經常出現。<dfn>例如</dfn>，如果你從 HTML 表單欄位取得值，它們通常是字串。那麼如果你想將它們相加呢？

二進位加號會將它們當作字串相加：

```js
let apples = "2";
let oranges = "3";

alert( apples + oranges ); // "23", the binary plus concatenates strings
```

如果你想將它們視為數字，你需要先轉換然後再相加：

```js
let apples = "2";
let oranges = "3";

// both values converted to numbers before the binary plus
alert( +apples + +oranges ); // 5

// the longer variant
// alert( Number(apples) + Number(oranges) ); // 5
```

從數學家的角度來看，過多的加號可能顯得奇怪。但從程式設計師的角度來看，這沒什麼特別：一元加號會先被應用，它們會將字串轉換成數字，然後二進位加號將它們相加。

為什麼一元加號會先套用到值上，而不是二元加號？正如你將會看到的，這是因為它們具有*較高*的優先權。

# --questions--

## --text--
根據 JavaScript 中一元和二元 `+` 運算子的說明，如何使用一元 `+` 正確地將字串值 `'2'` 和 `'3'` 相加為數字？

## --answers--

`console.log('2' + '3'); // Outputs: '23'` 

---

`console.log(+2 + +3); // Outputs: 5`

---

`console.log(+'2' + +'3'); // Outputs: 5`

---

`console.log(Number('2') + Number('3')); // Outputs: '23'`


## --video-solution--

3
