# --explanation--

`Complexity` bezieht sich darauf, wie kompliziert oder schwierig etwas ist, besonders wenn es viele Teile oder Schritte hat. Zum Beispiel:

`The back-end logic added a lot of complexity to the app.` – Das bedeutet, dass der Backend-Code die Anwendung schwerer verständlich oder handhabbar gemacht hat.

`Keeping the details hidden` bedeutet, nicht alle internen Schritte oder Funktionsweisen zu zeigen, sondern nur das Notwendige. Zum Beispiel:

`The user interface hides the complexity of the system by keeping the details hidden.` – Das bedeutet, dass Nutzer nur das sehen, was sie brauchen, ohne den technischen Hintergrund.
