# --description--

Давайте дізнаємося, як працює подія `submit` з HTML-формами. Спочатку потрібно зрозуміти, як відбувається відправлення форми. Існує три способи відправити форму.

Перший — коли користувач натискає на кнопку у формі, у якої встановлено атрибут `type` зі значенням `submit`.

Другий — коли користувач натискає клавішу Enter у будь-якому редагованому полі введення форми.

Третій — через виклик JavaScript методів `requestSubmit()` або `submit()` елемента форми.

Але що відбувається, коли форму відправляють? Відбувається кілька речей, і поведінка форми залежить від її атрибутів.

Перший атрибут, на який потрібно звернути увагу, — це атрибут `action`. Атрибут `action` повинен містити або URL, або відносний шлях для поточного домену. Це значення визначає, куди форма намагається надіслати дані. Якщо атрибут `action` не встановлено, форма надішле дані на URL поточної сторінки. Ось приклад форми з атрибутом `action`, встановленим на конкретний URL:

```html
<form action="https://freecodecamp.org">
  <input type="number" id="input" placeholder="Enter a number" name="number" />
  <button type="submit">Submit</button>
</form>
```

Коли ця форма відправляється, вона надішле дані на головну сторінку freeCodeCamp, що, ймовірно, нічого не зробить.

Ось інша форма, яка відправляє дані на відносний шлях:

```html
<form action="/data">
  <input type="number" id="input" placeholder="Enter a number" name="number" />
  <button type="submit">Submit</button>
</form>
```

Вона відправляє дані на шлях `/data` поточного домену. У нашому випадку це буде `http://127.0.0.1:5500/data`.

Другий атрибут, який контролює поведінку відправлення, — це атрибут `method`. Цей атрибут приймає стандартний HTTP-метод, наприклад `GET` або `POST`, і використовує цей метод при виконанні запиту на URL `action`.

HTTP означає протокол передачі гіпертексту і використовується для передачі даних у мережі.

HTTP-методи визначають дії, які можна виконувати над ресурсами, такі як `GET`, `POST`, `PUT`, `DELETE` тощо. Ви дізнаєтеся більше про ці методи в майбутніх уроках.

Якщо метод не встановлено, форма за замовчуванням виконає запит `GET`. Запит `GET` використовується для отримання даних із вказаного ресурсу без внесення змін, а параметри зазвичай додаються до URL у вигляді рядка запиту.

Дані форми будуть кодуватися у форматі URL як пари `name=value` і додаватися до URL `action` як параметри запиту. Наприклад, при відправленні форми з нашого попереднього прикладу зі значенням `3342` для поля `input`, форма спробує `GET` URL `http://127.0.0.1:5500/data?number=3342`.

Це чудово підходить для таких форм, як пошукова, де користувач робить запит даних. Але якщо ви хочете, щоб користувач відправляв нові дані, наприклад у формі реєстрації, ідіоматичним методом буде використання методу `POST`. Метод `POST` використовується для надсилання даних на сервер для створення або оновлення ресурсу.

Давайте встановимо атрибут `method` зі значенням `POST`:

```xml
<form action="/data" method="POST">
  <input type="number" id="input" placeholder="Enter a number" name="number" />
  <button type="submit">Submit</button>
</form>
```

Коли ви надсилаєте запит `POST`, можна включити тіло, яке містить дані для вашого запиту. Тож на відміну від `GET`, дані не додаються до URL, і наша форма надсилає запит `POST` на `http://127.0.0.1:5500/data`. Дані натомість знаходяться в тілі запиту, все ще у вигляді URL-кодованих даних форми.

URL-кодовані дані форми — це коли дані форми перетворюються на рядок пар ключ-значення, як `name=John+Doe&email=john%40example.com`, де спеціальні символи замінюються на закодовані версії для безпечної передачі даних через мережу. Ви дізнаєтеся більше про це в майбутніх уроках.

Але можливо, ви не хочете надсилати дані у вигляді URL-кодованого корисного навантаження форми?

Елемент `form` приймає атрибут `enctype`, який представляє тип кодування, що використовується для даних.

Цей атрибут приймає лише три значення: `application/x-www-form-urlencoded` (що є за замовчуванням і надсилає дані як URL-кодоване тіло форми), `text/plain` (що надсилає дані у вигляді простого тексту у парах `name=value`, розділених новими рядками), або `multipart/form-data`, який спеціально призначений для обробки форм із завантаженням файлів.

У цьому уроці ви дізналися основи відправлення форм. У майбутніх уроках ми глибше розглянемо роботу з формами та їх взаємодію з сервером.
