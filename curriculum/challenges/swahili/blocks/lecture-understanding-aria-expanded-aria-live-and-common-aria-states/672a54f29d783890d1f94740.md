# --interactive--

Vipengele vya udhibiti wa fomu vya maana kama `input`, `select`, `textarea`, `button`, na `fieldset` vina hali zilizojengwa ndani ambazo huwasilishwa kwa teknolojia za kusaidia.

Kwa mfano, unaweza kutumia sifa ya `disabled` kuzima kitufe au sifa ya `checked` kuonyesha kuwa kisanduku cha kuchagua kimechaguliwa.

Lakini ikiwa unaunda kipengele cha udhibiti maalum, unahitaji kutumia sifa za ARIA kuwasilisha hali ya udhibiti kwa teknolojia za kusaidia.

Katika funzo hili, tutajadili hali chache za ARIA zinazotumika kawaida kwenye vipengele vya udhibiti maalum.

Hali ya kwanza ya ARIA tutakayojadili ni `aria-selected`. Hali hii hutumika kuonyesha kuwa kipengele kimechaguliwa. Unaweza kutumia hali hii kwenye vidhibiti maalum kama kiolesura cha tabo, kisanduku cha orodha, au gridi.

Hapa kuna mfano wa jinsi unavyoweza kutumia `aria-selected` kwenye kidhibiti maalum cha tabo:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<div role="tablist">
  <button role="tab" aria-selected="true">Tab 1</button>
  <button role="tab" aria-selected="false">Tab 2</button>
  <button role="tab" aria-selected="false">Tab 3</button>
</div>
<script src="index.js"></script>
```

```css
[role="tablist"] {

  border-bottom: 2px solid #ddd;
  gap: 0.25rem;
  font-family: system-ui, sans-serif;
}

[role="tab"] {
  appearance: none;
  border: none;
  background: none;
  padding: 0.5rem 1rem;
  cursor: pointer;
  font-size: 1rem;
  color: #444;
  border-radius: 4px 4px 0 0;
  transition: background-color 0.2s, color 0.2s;
}

[role="tab"]:hover {
  background-color: #f3f3f3;
}

[role="tab"][aria-selected="true"] {
  background-color: #fff;
  color: #0078d4;
  border: 2px solid #0078d4;
  border-bottom: 2px solid #fff;

```

```js
document.addEventListener("click", (event) => {
  const clickedTab = event.target.closest('[role="tab"]');
  if (!clickedTab) return;

  const tablist = clickedTab.closest('[role="tablist"]');
  const tabs = tablist.querySelectorAll('[role="tab"]');

  tabs.forEach((tab) => {
    const isSelected = tab === clickedTab;
    tab.setAttribute("aria-selected", isSelected);
    tab.tabIndex = isSelected ? 0 : -1;
  });
});

document.addEventListener("keydown", (event) => {
  const activeTab = document.activeElement;
  if (activeTab.getAttribute("role") !== "tab") return;

  const tablist = activeTab.closest('[role="tablist"]');
  const tabs = Array.from(tablist.querySelectorAll('[role="tab"]'));
  const index = tabs.indexOf(activeTab);

  let newIndex = index;
  if (event.key === "ArrowRight") newIndex = (index + 1) % tabs.length;
  if (event.key === "ArrowLeft") newIndex = (index - 1 + tabs.length) % tabs.length;

  if (newIndex !== index) {
    tabs[newIndex].focus();
    tabs[newIndex].click();
  }
});

```

Tabo hutumika kuonyesha paneli nyingi za maudhui katika nafasi ndogo. Hali ya `aria-selected` hutumika kuonyesha ni tabo gani imechaguliwa kwa sasa.

Wakati mtumizi anapochagua tabo, hali ya `aria-selected` ya tabo iliyochaguliwa huwekwa kuwa `true`, na hali ya `aria-selected` ya tabo zingine huwekwa kuwa `false`.

Hali nyingine ya kawaida ya ARIA ni `aria-disabled`. Hali hii hutumika kuonyesha kuwa kipengele kimezimwa kwa watu wanaotumia teknolojia za kusaidia, kama kisomaji cha skrini. Ni muhimu kuelewa kuwa `aria-disabled` haizimi kipengele kwa kweli. Ni jukumu lako, msanidi programu, kuifanya ionekane na itende kama kipengele kilichozimwa. Sifa hii pia hutumika mara nyingi kwenye vipengele vya asili vya HTML badala ya sifa ya `disabled`. Ni wewe utakaebeba uamuzi wa kutumia ipi kulingana na muktadha wa kitufe kinapotumika.

Hapa kuna mfano wa jinsi unavyoweza kutumia `aria-disabled` kwenye kitufe maalum cha kuhariri:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<div role="button" tabindex="-1" aria-disabled="true">Edit</div>
```

Sifa ya `aria-disabled` hutumika kuambia watumiaji wa kisomaji cha skrini kuwa kitufe cha kuhariri kimezimwa na hakiwezi kuingiliana nacho. Tena, haizimi kitufe kwa kweli. Unapotumia `aria-disabled`, utahitaji kutumia mtindo na JavaScript kuifanya kidhibiti ionekane na itende kama kitufe kilichozimwa.

Katika hali nyingi, huenda ukatumia kipengele cha kitufe cha asili, lakini kuna nyakati ambapo unaweza kuhitaji kutumia kidhibiti maalum. Kwa hivyo, ni muhimu kujua jinsi ya kuwasilisha hali ya kidhibiti kwa teknolojia za kusaidia.

Hali inayofuata ya ARIA tutakayojadili ni `aria-haspopup`. Hali hii hutumika kuonyesha kuwa kipengele cha kuingiliana kitasababisha kipengele cha popup kufunguka wakati kinapozimwa. Unaweza kutumia sifa ya `aria-haspopup` tu wakati popup ina mojawapo ya majukumu yafuatayo: `menu`, `listbox`, `tree`, `grid`, au `dialog`. Thamani ya `aria-haspopup` lazima iwe mojawapo ya majukumu haya au `true`, ambayo ni chaguo-msingi kwa jukumu la `menu`.

Hapa kuna mfano wa menyu ya mhariri wa faili inayotumia `aria-haspopup`:

Hali ya `aria-haspopup` hutumika kuonyesha kuwa kitufe cha menyu cha `File` kitafungua menyu ya popup wakati kinapozimwa. Watumiaji wa kisomaji cha skrini wanaweza kusikia taarifa hii ya ziada wanapofikia kitufe.

Utahitaji kutumia JavaScript kuonyesha na kuficha menyu ya popup, na kutekeleza msaada sahihi wa kibodi kwa kuingiliana na menyu. Pia, tafadhali fahamu kuwa jukumu la ARIA `menu` linahusu aina maalum sana ya menyu. Kwa kawaida linahusu orodha ya vitendo ambavyo mtumizi anaweza kuitisha, sawa na menyu kwenye programu za desktop. Halihusishi matumizi ya kawaida ya "menyu" kama tunavyoyafahamu, kama menyu za urambazaji. Kwa kweli, menyu nyingi unazotengeneza mtandaoni hazitakuwa menyu za ARIA na hutatumia `aria-haspopup` nazo.

Hali inayofuata ya ARIA tutakayojadili ni `aria-required`. Sifa ya `aria-required` hutumika kuonyesha kuwa sehemu fulani inahitaji kujazwa kabla ya fomu kuwasilishwa.

Hapa kuna mfano wa kufanya kazi na sifa ya `aria-required` kwa kidhibiti maalum cha fomu.

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<div id="name-label">Full Name*</div>
<div role="textbox" contenteditable aria-labelledby="name-label" aria-required="true" id="name"></div>
```

Tunahitaji kutumia sifa ya `contenteditable` ili watumiaji waweze kuandika ingizo lao. Pia tunatumia sifa ya `aria-required` iliyowekwa kuwa `true` kuonyesha kuwa kidhibiti hiki maalum cha fomu ni cha lazima.

Ili kufanya kidhibiti cha fomu kiwe kama kidhibiti cha kawaida, utahitaji kuongeza CSS. Pia utahitaji kuongeza JavaScript kuzuia fomu kuwasilishwa bila maudhui.

Kama lebo tayari ina neno `required`, basi unapaswa kuacha sifa ya `aria-required`. Hii inahakikisha kuwa visomaji vya skrini hutoa neno "required" mara moja tu.

Katika hali nyingi, huenda ukatumia vipengele vya asili vya `label` na `form` pamoja na sifa ya `required`. Lakini kama unahitaji kuunda kidhibiti maalum cha fomu, basi ni muhimu kuongeza sifa ya `aria-required` inapohitajika.

Zaidi ya hayo, sifa ya `aria-required` pia inaweza kutumika kwenye ingizo za fomu za asili, kama vipengele vya `input`, `textarea`, na `select`. Hii mara nyingi hupendelewa kuliko sifa ya required ya asili, kwa sababu sifa ya required inaweza kuwa na changamoto za matumizi na ufikikaji, hasa kwa usimamizi wa makosa wa chaguo-msingi unaotolewa na kivinjari. Mwisho wa siku, utahitaji kufanya majaribio ili kubaini sifa gani ni bora kwa hali yako.

Hali ya mwisho ya ARIA tutakayojadili ni `aria-checked`. Sifa hii hutumika kuonyesha kama kipengele kiko katika hali ya kuchaguliwa. Hutumika zaidi wakati wa kuunda visanduku maalum vya kuchagua, vitufe vya radio, vitufe vya kubadilisha, na kisanduku cha orodha.

Hapa kuna mfano wa jinsi unavyoweza kutumia `aria-checked` kwenye kidhibiti maalum cha kisanduku cha kuchagua:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<div role="checkbox" aria-checked="true" tabindex="0">Checkbox</div>
```

Vipengele vya kisanduku cha kuchagua vya asili vina hali ya `checked` iliyojengwa ndani ambayo huwasilishwa kwa teknolojia za kusaidia. Lakini ikiwa unaunda kidhibiti maalum cha kisanduku cha kuchagua, utahitaji kutumia sifa ya `aria-checked` kuonyesha hali yake.

Wakati mtumizi anaposhirikiana na kidhibiti maalum cha kisanduku cha kuchagua, utahitaji kutumia hali ya `aria-checked` kuonyesha hali mpya ya kisanduku. Wakati kisanduku kimechaguliwa, sifa ya `aria-checked` huwekwa kuwa `true`. Wakati hakijachaguliwa, huwekwa kuwa `false`.

Vipengele vya asili kwa kawaida vina msaada bora na vipengele vya ufikikaji vilivyojengwa ndani.

Hata hivyo, kama lazima uunde vidhibiti maalum, kutumia sifa za ARIA ni muhimu kuwasilisha hali ya vidhibiti hivi kwa teknolojia za kusaidia kwa ufanisi.

Kama kawaida, hakikisha unakagua kazi yako ili kuhakikisha sifa za ARIA zimetumika ipasavyo na kidhibiti maalum kinafanya kazi kwa njia inayofaa na rahisi kwa mtumizi.
