# --interactive--

수정자, 흔히 "플래그"라고도 하는 것은 정규 표현식의 동작 방식을 바꿉니다. 이전 수업에서 본 예제를 다시 살펴보겠습니다:

:::interactive_editor

```js
const regex = /freeCodeCamp/;
console.log(regex.test("freeCodeCamp")); // true
console.log(regex.test("freeCodeCamp is great")); // true
console.log(regex.test("I love freeCodeCamp")); // true
console.log(regex.test("freecodecamp")); // false
console.log(regex.test("FREECODECAMP")); // false
console.log(regex.test("free")); // false
console.log(regex.test("code")); // false
console.log(regex.test("camp")); // false
```

:::

기억하시겠지만, 모두 소문자와 모두 대문자인 `freeCodeCamp` 문자열은 패턴과 일치하지 않았습니다. 이는 기본적으로 정규 표현식이 대소문자를 구분하기 때문입니다.

그런데 정규 표현식이 대소문자를 구분하지 않도록 할 수 있다면 어떨까요? 그런 수정자가 있습니다. `i` 플래그는 정규식이 대소문자를 무시하게 만듭니다. 어떻게 사용할까요? 플래그는 정규 표현식의 닫는 슬래시 뒤에 붙습니다:

```js
const regex = /freeCodeCamp/i;
```

첫 번째 줄의 정규 표현식 변화를 확인하세요. 이제 이것이 어떻게 바뀌는지 확인해 봅시다:

:::interactive_editor

```js
const regex = /freeCodeCamp/i;

console.log(regex.test("freeCodeCamp")); // true
console.log(regex.test("freeCodeCamp is great")); // true
console.log(regex.test("I love freeCodeCamp")); // true
console.log(regex.test("freecodecamp")); // true
console.log(regex.test("FREECODECAMP")); // true
console.log(regex.test("free")); // false
console.log(regex.test("code")); // false
console.log(regex.test("camp")); // false
```

:::

정규 표현식이 이제 대소문자를 구분하지 않으므로, 모두 소문자와 모두 대문자인 문자열도 테스트를 "통과"했습니다. 대소문자가 섞인 문자열에도 적용할 수 있습니다:

:::interactive_editor

```js
const regex = /freeCodeCamp/i;

console.log(regex.test("dO yOu LoVe fReEcOdEcAmP?")); // true
```

:::

사용할 수 있는 플래그가 꽤 많습니다. `g` 플래그, 즉 글로벌 수정자는 정규 표현식이 패턴과 여러 번 일치하도록 허용합니다.

이것이 코드에 어떤 영향을 미치는지 봅시다. `i` 플래그를 유지한 것을 알 수 있습니다 — 정규 표현식은 원하는 동작을 위해 여러 플래그를 동시에 사용할 수 있습니다(필요한 만큼).

```js
const regex = /freeCodeCamp/gi;
```

잠깐만요... 이게 뭐죠? 글로벌 수정자가 일부 문자열이 통과해야 하는데 실패하게 만드는 것 같습니다:

:::interactive_editor

```js
const regex = /freeCodeCamp/gi;

console.log(regex.test("freeCodeCamp")); // true
console.log(regex.test("freeCodeCamp is great")); // false
console.log(regex.test("I love freeCodeCamp")); // true
console.log(regex.test("freecodecamp")); // false
console.log(regex.test("FREECODECAMP")); // true
console.log(regex.test("free")); // false
console.log(regex.test("code")); // false
console.log(regex.test("camp")); // false
```

:::

왜 그럴까요? 글로벌 수정자는 정규 표현식을 상태 유지형으로 만듭니다. 즉, 이전에 패턴과 일치한 위치를 기억합니다. 그래서 첫 번째 `freeCodeCamp` 문자열과 일치했을 때, `0` 인덱스에서 일치가 시작된 것을 기억합니다.

그다음 `freeCodeCamp is great` 문자열을 테스트하지만 `0` 인덱스에서 시작하지 않습니다. 정규 표현식은 이미 `0` 인덱스에서 일치가 있었다는 것을 "알고" 있어서, 비록 다른 문자열이지만 일치가 끝난 인덱스부터 시작합니다.

`freeCodeCamp`은 `12` 문자 길이이므로, `0`에서 시작한 일치는 `11` 인덱스에서 끝납니다. 일치는 `12` 인덱스에서 다시 시작합니다. 그리고 `is great`가 `freeCodeCamp`와 일치하지 않으므로 `false`을 반환합니다.

그 후 일치를 찾지 못해 상태를 "잃어버리고" 다음 일치는 `0`에서 다시 시작합니다.

만약 `0`에서 일치하는 문자열 다음에 `11`보다 나중에 일치하는 문자열이 바로 이어지도록 로그 순서를 바꾸면:

:::interactive_editor

```js
const regex = /freeCodeCamp/gi;

console.log(regex.test("freeCodeCamp")); // true
console.log(regex.test("I loooooooove freeCodeCamp")); // true
```

:::

글로벌 정규 표현식에는 `lastIndex`이라는 새 속성이 생깁니다. 이전 코드를 가져와 이 속성이 어떻게 작동하는지 봅시다:

:::interactive_editor

```js
const regex = /freeCodeCamp/gi;

console.log(regex.lastIndex); // 0
console.log(regex.test("freeCodeCamp")); // true
console.log(regex.lastIndex); // 12
console.log(regex.test("freeCodeCamp is great")); // false
console.log(regex.lastIndex); // 0
console.log(regex.test("I love freeCodeCamp")); // true
console.log(regex.lastIndex); // 19
console.log(regex.test("freecodecamp")); // false
console.log(regex.lastIndex); // 0
console.log(regex.test("FREECODECAMP")); // true
console.log(regex.lastIndex); // 12
console.log(regex.test("free")); // false
console.log(regex.lastIndex); // 0
console.log(regex.test("code")); // false
console.log(regex.lastIndex); // 0
console.log(regex.test("camp")); // false
```

:::

이 예제를 보면, 정규 표현식의 상태가 각 테스트 호출마다 `lastIndex`을 사용해 이전 일치를 추적하는 방식을 알 수 있습니다.

글로벌 플래그는 단일 문자열에서 여러 일치를 얻어야 할 때 유용합니다. 하지만 여러 문자열을 같은 정규 표현식으로 테스트할 때는 `g` 플래그를 끄는 것이 좋습니다.

다음 플래그를 배우기 전에 앵커에 대해 알아야 합니다. 캐럿(`^`) 앵커는 정규 표현식 시작 부분에 위치하며 "문자열 시작과 일치"하라는 뜻입니다:

```js
const start = /^freecodecamp/i;
```

달러 기호(`$`) 앵커는 정규 표현식 끝에 위치하며 "문자열 끝과 일치"하라는 뜻입니다:

```js
const end = /freecodecamp$/i;
```

오른쪽 출력 결과를 잠시 비교해 보세요:

:::interactive_editor

```js
const start = /^freecodecamp/i;
const end = /freecodecamp$/i;
console.log(start.test("freecodecamp")); // true
console.log(end.test("freecodecamp")); // true
console.log(start.test("freecodecamp is great")); // true
console.log(end.test("freecodecamp is great")); // false
console.log(start.test("i love freecodecamp")); // false
console.log(end.test("i love freecodecamp")); // true
console.log(start.test("have met freecodecamp's founder")); // false
console.log(end.test("have met freecodecamp's founder")); // false
```

:::

시작 앵커는 문자열 시작에서만 일치하고, 끝 앵커는 문자열 끝에서만 일치하는 것을 볼 수 있습니다. 그렇다면 여러 줄에 걸쳐 일치시키려면 어떻게 할까요? 살펴봅시다:

:::interactive_editor

```js
const start = /^freecodecamp/i;
const end = /freecodecamp$/i;
const string = `I really love
freecodecamp
it's my favorite`;
console.log(start.test(string)); // false
console.log(end.test(string)); // false
```

:::

`freecodecamp`가 독립된 줄에 있음에도 두 테스트 모두 실패했습니다. 이는 기본적으로 앵커가 전체 문자열의 시작과 끝만 찾기 때문입니다.

하지만 `m` 플래그, 즉 멀티라인 수정자를 사용하면 정규식이 여러 줄을 처리할 수 있습니다. 정규 표현식에 추가해 결과를 확인해 봅시다:

:::interactive_editor

```js
const start = /^freecodecamp/im;
const end = /freecodecamp$/im;
const string = `I really love
freecodecamp
it's my favorite`;
console.log(start.test(string)); // true
console.log(end.test(string)); // true
```

:::

이제 둘 다 일치합니다! `freecodecamp`가 완전히 독립된 줄에 있으므로, 시작 앵커는 그 줄의 시작과, 끝 앵커는 그 줄의 끝과 일치합니다.

마지막으로 `d` 플래그, 즉 인덱스 수정자가 있습니다. `i` 플래그는 대소문자 구분 안 함용이므로, 인덱스 수정자는 다른 플래그가 필요했습니다.

`d` 플래그는 일치 객체에서 얻는 정보를 확장합니다. 정규 표현식에 추가해 봅시다:

```js
const regex = /freecodecamp/di;
const string = "we love freecodecamp isn't freecodecamp great?";
console.log(string.match(regex));
```

결과는 다음과 같습니다:

```js
// [
//   'freecodecamp',
//   index: 8,
//   input: "we love freecodecamp isn't freecodecamp great?",
//   groups: undefined,
//   indices: [
//     0: [8, 20],
//     groups: undefined
//   ]
// ]
```

일치 객체에 `indices`이라는 새 속성이 생겼습니다! 이 속성은 두 숫자로 된 배열로, 첫 번째는 원본 문자열에서 일치가 시작된 인덱스이고, 두 번째는 일치가 끝난 다음 인덱스입니다. 이 배열에는 `groups`라는 추가 속성도 있는데, 이는 명명된 캡처 그룹용입니다.

알아두면 좋지만 일반 코드에서는 덜 자주 쓰이는 플래그도 몇 가지 있습니다.

첫 번째는 유니코드 수정자, 즉 `u` 플래그입니다. 이 플래그는 정규 표현식이 특수 유니코드 문자를 일치시키도록 기능을 확장합니다.

앞으로 수업에서 문자 클래스를 더 배우겠지만, `u` 플래그는 대부분의 이모지를 일치시키는 `Extended_Pictographic` 같은 특수 클래스를 사용할 수 있게 해줍니다:

:::interactive_editor

```js
const regex = /🍎/u;

const str = "I have an apple 🍎";
console.log(regex.test(str)); // true
```

:::

또 `v` 플래그도 있는데, 유니코드 일치 기능을 더 확장합니다.

두 번째는 스티키 수정자, 즉 `y` 플래그입니다. 스티키 수정자는 글로벌 수정자와 매우 비슷하게 동작하지만 몇 가지 예외가 있습니다.

가장 큰 차이는 글로벌 정규 표현식은 `lastIndex`부터 시작해 문자열 나머지 전체를 검색하지만, 스티키 정규 표현식은 이전 `null` 위치에서 즉시 일치하지 않으면 `lastIndex`을 반환하고 `0`을 `lastIndex`로 재설정한다는 점입니다.

마지막으로 싱글라인 수정자, 즉 `s` 플래그가 있습니다. 멀티라인 수정자는 시작과 끝 앵커가 전체 문자열 대신 각 줄의 시작과 끝에 일치하도록 허용합니다.

싱글라인 수정자는 정규식에서 점(`.`)으로 표현되는 와일드카드 문자가 줄바꿈 문자와도 일치하도록 하여, 문자열을 단일 행 텍스트처럼 처리합니다.

수정자는 꽤 많지만, `i`와 `g` 플래그가 가장 자주 사용되고 기억해야 할 가장 중요한 플래그입니다.
