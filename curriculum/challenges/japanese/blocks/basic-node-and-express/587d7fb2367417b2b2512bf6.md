# --description--

クライアントから入力を取得するもう一つの一般的な方法は、ルートパスの後にデータをエンコードしてクエリ文字列を使うことです。クエリ文字列は疑問符（?）で区切られ、field=value の組み合わせを含みます。各組み合わせはアンパサンド（&）で区切られます。Expressはクエリ文字列からデータを解析し、オブジェクト `req.query` に格納できます。パーセント（%）のようにURLに含められない文字は、送信前に別の形式でエンコードする必要があります。JavaScriptのAPIを使うと、これらの文字をエンコード／デコードする特定のメソッドを利用できます。

<blockquote>route_path: '/library'<br>actual_request_URL: '/library?userId=546&#x26;bookId=6754' <br>req.query: {userId: '546', bookId: '6754'}</blockquote>

# --instructions--

`GET /name` にマウントされたAPIエンドポイントを作成してください。`{ name: 'firstname lastname'}` の構造を持つJSONドキュメントで応答してください。first と last の名前パラメータはクエリ文字列でエンコードされるべきです。例：`?first=firstname&last=lastname`。

**注意：** 次の演習では、同じ`/name`ルートパスでPOSTリクエストからデータを受け取ります。必要に応じて、メソッド`app.route(path).get(handler).post(handler)`を使うことができます。この構文は同じルートパスに異なる動詞のハンドラをチェーンでき、タイプ量を減らしコードをすっきりさせます。

# --hints--

Test 1 : `{ "name": "Mick Jagger" }` エンドポイントが `/name` で呼び出されたとき、APIエンドポイントは `?first=Mick&last=Jagger` で応答する必要があります

```js
  const response = await fetch(code + '/name?first=Mick&last=Jagger');
  if (!response.ok) {
    throw new Error(await response.text());
  }
  const data = await response.json();
  assert.equal(
    data.name,
    'Mick Jagger',
    'Test 1: "GET /name" route does not behave as expected'
  );
```

Test 2 : `{ "name": "Keith Richards" }` エンドポイントが `/name` で呼び出されたとき、APIエンドポイントは `?first=Keith&last=Richards` で応答する必要があります

```js
  const response = await fetch(code + '/name?last=Richards&first=Keith');
  if (!response.ok) {
    throw new Error(await response.text());
  }
  const data = await response.json();
  assert.equal(
    data.name,
    'Keith Richards',
    'Test 2: "GET /name" route does not behave as expected'
  );
```
