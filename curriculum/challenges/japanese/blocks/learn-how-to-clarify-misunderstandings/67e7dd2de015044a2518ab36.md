# --explanation--

`Think of it as` は、すでに知っているものに例えて概念を理解しやすくするために使われます。概念を理解しやすい形で想像することを促します。例えば：

`Think of a function as a recipe.` - これは関数を、料理のレシピが料理を作る手順を示すのと同じように、一連の指示としてイメージするのに役立ちます。

`Wrapping code` は、コードを包み込んでアクセスを制御することを意味します。これによりコードへのアクセスが`limit`されます。例えば：

`We're wrapping the database access code inside a function to hide the complexity from the rest of the program.` - これはデータベースとのやり取りが外部から隠され、関数が簡単なインターフェースを提供することを意味します。
