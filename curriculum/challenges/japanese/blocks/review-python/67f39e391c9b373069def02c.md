## Pythonとは何ですか？

- **概要**: Pythonは、そのシンプルさと使いやすさで知られる汎用プログラミング言語です。Pythonはデータサイエンスや機械学習、ウェブ開発、スクリプトや自動化、組み込みシステムやIoTなど、多くの分野で使われています。
- **一般的な利用例**: Pythonはデータサイエンス、機械学習、ウェブ開発、サイバーセキュリティ、自動化、そしてRaspberry PiやMicroPython互換ボードのようなマイクロコンピュータで使われています。

## ローカル環境でのPython

- **インストール**：Windows、Mac、LinuxにPythonをインストールする最も良い方法は、公式Pythonウェブサイトからインストーラーをダウンロードすることです(`https://www.python.org/`)。

## 変数

- **変数の宣言**：変数を宣言するには、変数名の後に代入演算子（`=`）を置き、その後に値を代入します。値は数値、文字列、ブール値などが使えます。以下は例です。

```py
name = 'John Doe'
age = 25
```

- **変数名の慣例**：変数名には以下の慣例を守ってください。

  - 変数名は文字またはアンダースコア（_）で始めることができ、数字で始めることはできません。
  - 変数名には英数字（a-z、A-Z、0-9）とアンダースコア（_）のみ使用できます。
  - 変数名は大文字と小文字を区別します。`age`、`Age`、`AGE`はすべて別の名前として扱われます。
  - 変数名はPythonの予約キーワード（`if`、`class`、`def`など）と同じにできません。
  - 複数の単語からなる変数名はアンダースコアで区切ります。例：`snake_case`。

## コメント

- **シングルラインコメント**：コード内に短いメモを残したいときに使います。

```py
# This is a single line comment
```

- **複数行文字列**：より大きなメモを残したり、コードのセクションをコメントアウトしたりするのに使えます。

```py
"""
This is a multi-line string.
Here is some code commented out.

name = 'John Doe'
age = 25
"""
```

- **`print()` 関数**：データをコンソールに出力するには、`print()` 関数を次のように使います。

```py
print('Hello world!') # Hello world!
```

## Pythonの一般的なデータ型

- **導入**: PythonはJavaScriptのような動的型付け言語で、変数の型を明示的に宣言する必要はありません。変数に代入した値によって、その変数の型が自動的に判断されます。
- **整数**: 小数点のない整数です。

```py
my_integer_var = 10
print('Integer:', my_integer_var) # Integer: 10
```

- **浮動小数点数**: 小数点を含む数値です。

```py
my_float_var = 4.50
print('Float:', my_float_var) # Float: 4.5
```

- **文字列**: 引用符で囲まれた文字のシーケンスです。

```py
my_string_var = 'hello'
print('String:', my_string_var) # String: hello
```

- **ブール値**: `True` または `False` のいずれかを表す値です。

```py
my_boolean_var = True
print('Boolean:', my_boolean_var) # Boolean: True
```

- **セット**: 順序のない一意の要素の集合です。

```py
my_set_var = {7, 5, 8}
print('Set:', my_set_var) # Set: {7, 5, 8}
```

- **辞書**: 波括弧で囲まれたキーと値のペアの集合です。

```py
my_dictionary_var = {"name": "Alice", "age": 25}
print('Dictionary:', my_dictionary_var) # Dictionary: {'name': 'Alice', 'age': 25}
```

- **タプル**: 丸括弧で囲まれた変更不可能な順序付きの集合です。

```py
my_tuple_var = (7, 5, 8)
print('Tuple:', my_tuple_var) # Tuple: (7, 5, 8)
```

- **レンジ**: ループでよく使われる数値のシーケンスです。

```py
my_range_var = range(5)
print(my_range_var) # range(0, 5)
```

- **リスト**: 異なるデータ型を含めることができる順序付きの要素の集合です。

```py
my_list = [22, 'Hello world', 3.14, True]
print(my_list) # [22, 'Hello world', 3.14, True]
```

- **None**: 値が存在しないことを表す特別な値です。

```py
my_none_var = None
print('None:', my_none_var) # None: None
```

## 不変型と可変型

- **不変型**: 宣言後に変更できませんが、変数を新しいものに向けることはでき、これを再代入と呼びます。整数、浮動小数点数、複素数、ブール値、文字列、タプル、range、そして`None`が含まれます。
- **可変型**: 宣言後に変更できます。アイテムの追加、削除、更新が可能です。リスト、セット、辞書などのコレクション型が含まれます。
- **`type()` 関数**: 変数の型を確認するには、次のように`type()`関数を使います。

```py
greeting = 'Hello there!'
age = 21

print(type(greeting)) # <class 'str'>
print(type(age)) # <class 'int'>
```

- **`isinstance()` 関数**: 変数が特定のデータ型に一致するかどうかを確認するために使います。

```py
greeting = 'Hello world'
name = 'John Doe'

print(isinstance(greeting, str)) # True
print(isinstance(name, int)) # False
```

## 文字列の操作

- **定義**: JavaScriptで覚えているように、文字列は不変であり、一度作成した後に変更できません。Pythonでは、シングルクォートかダブルクォートのどちらかを使えます。ルールを決めて一貫して使うことをお勧めします。

```py
developer = 'Jessica'
city = "Los Angeles"
```

- **文字列から文字をアクセスする**: ブラケット記法を使って文字列から文字をアクセスできます。

```py
my_str = 'Hello world'

print(my_str[0])  # H
print(my_str[6])  # w

print(my_str[-1])  # d
print(my_str[-2]) # l
```

- **文字列のエスケープ**: 文字列に引用符が含まれる場合は、バックスラッシュ (`\`) を使えます。

```py
msg = 'It\'s a sunny day'
quote = "She said, \"Hello!\""
```

- **文字列の連結**: 文字列を連結するには、`+` 演算子を使います。

```py
developer = 'Jessica'
print('My name is ' + developer + '.') # My name is Jessica.
```

文字列を連結するもう一つの方法は、`+=` 演算子を使うことです。これは連結と代入を同時に行うために使います。

```py
greeting = 'My name is '
developer = 'Jessica.'

greeting += developer
print(greeting) # My name is Jessica.
```

- **`f-strings`**: これはフォーマット済み文字列リテラルの略称です。補間を扱いながら、コンパクトで読みやすい構文で連結もできます。

```py
developer = 'Jessica'
greeting = f'My name is {developer}.'
print(greeting) # My name is Jessica.
```

- **文字列のスライス**: 文字列の一部を抽出することです。基本的な構文は次の通りです。

```py
str[start:stop:step]
```

開始位置は抽出を始めるインデックスを表します。終了位置はスライスを終える場所で、この位置は含まれません。ステップ位置はスライスの増分間隔を表します。いくつかの例を示します。

```py
message = 'Python is fun!'

print(message[0:6])  # Python
print(message[7:])  # is fun!
print(message[::2])  # Pto sfn
```

- **文字列の長さを取得する**: `len()` 関数は文字列の文字数を返します。

```py
developer = 'Jessica'

print(len(developer)) # 7
```

## `in` 演算子の使い方

- **`in` 演算子**：これは、文字列にその文字または文字列が存在するかどうかを示すブール値を返します。

```py
my_str = 'Hello world'

print('Hello' in my_str)  # True
print('hey' in my_str)    # False
print('hi' in my_str)    # False
print('e' in my_str)  # True
print('f' in my_str)  # False
```

## Common String Methods

- **`upper()`**: この文字列メソッドは、すべての文字を大文字に変換した新しい文字列を返します。

```py
developer = 'Jessica'

print(developer.upper()) # JESSICA
```

- **`lower()`**: この文字列メソッドは、すべての文字を小文字に変換した新しい文字列を返します。

```py
developer = 'Jessica'

print(developer.lower()) # jessica
```

- **`strip()`**: この文字列メソッドは、指定された先頭と末尾の文字を削除した文字列のコピーを返します（引数が渡されない場合は、先頭と末尾の空白文字を削除します）。

```py
greeting = '  hello world  '

trimmed_my_str = greeting.strip()
print(trimmed_my_str)  # 'hello world'
```

- **`replace()`**: これは、古い文字列のすべての出現箇所を新しい文字列に置き換えた新しい文字列を返します。

```py
greeting = 'hello world'

replaced_my_str = greeting.replace('hello', 'hi')
print(replaced_my_str)  # 'hi world'
```

- **`split()`**: これは、指定された区切り文字を使って文字列をリストに分割するために使われます。区切り文字は分割する位置を指定する文字列です。

```py
dashed_name = 'example-dashed-name'

split_words = dashed_name.split('-')
print(split_words)  # ['example', 'dashed', 'name']
```

- **`join()`**: これは、イテラブルの要素を区切り文字でつなげて文字列にするために使われます。イテラブルとは、リスト、文字列、タプルのようにループ処理できる要素の集合です。

```py
example_list = ['example', 'dashed', 'name']

joined_str = ' '.join(example_list)
print(joined_str)  # example dashed name
```

- **`startswith(prefix)`**: この文字列メソッドは、文字列が指定された接頭辞で始まるかどうかを示すブール値を返します。

```py
developer = 'Naomi'

result = developer.startswith('N')
print(result)  # True
```

- **`endswith(suffix)`**: この文字列メソッドは、文字列が指定された接尾辞で終わるかどうかを示すブール値を返します。

```py
developer = 'Naomi'

result = developer.endswith('N')
print(result)  # False
```

- **`find()`**: この文字列メソッドは、部分文字列が最初に現れるインデックスを返します。見つからない場合は`-1`が返されます。

```py
developer = 'Naomi'

result = developer.find('N')
print(result)  # 0

city = 'Los Angeles'
print(city.find('New')) # -1
```

- **`count(substring)`**: この文字列メソッドは、部分文字列が文字列内に何回現れるかを数えます。

```py
city = 'Los Angeles'
print(city.count('e')) # 2
```

- **`capitalize()`**: この文字列メソッドは、最初の文字を大文字にし、他の文字を小文字にした新しい文字列を返します。

```py
dessert = 'chocolate cake'
print(dessert.capitalize()) # Chocolate cake
```

- **`isupper()`**: この文字列メソッドは、文字列内のすべての文字が大文字の場合は`True`を返し、そうでない場合は`False`を返します。

```py
dessert = 'chocolate cake'
print(dessert.isupper()) # False
```

- **`islower()`**: この文字列メソッドは、文字列内のすべての文字が小文字の場合は`True`を返し、そうでない場合は`False`を返します。

```py
dessert = 'chocolate cake'
print(dessert.islower()) # True
```

- **`title()`**: この文字列メソッドは、各単語の最初の文字を大文字にした新しい文字列を返します。

```py
city = 'los angeles'
print(city.title()) # Los Angeles
```

- **`maketrans()`**: この文字列メソッドは、1対1の文字対応表を作成するために使われます。通常は`translate()`メソッドと組み合わせて使い、その対応表を文字列に適用して翻訳結果を返します。

```py
trans_table = str.maketrans('abc', '123')
print(trans_table) # {97: 49, 98: 50, 99: 51}

result = 'abcabc'.translate(trans_table)
print(result)  # 123123
```

## 整数と浮動小数点数でよく使われる操作

- **基本的な算術演算**: Pythonでは、整数と浮動小数点数に対して加算、減算、乗算、除算などの基本的な算術演算ができます。

```py
int_1 = 56
int_2 = 12
float_1 = 5.4
float_2 = 12.0

# Addition

print('Integer Addition:', int_1 + int_2) # Integer Addition: 68
print('Float Addition:', float_1 + float_2) # Float Addition: 17.4

# Subtraction

print('Int Subtraction:', int_1 - int_2) # Int Subtraction: 44
print('Float Subtraction:',  float_2 - float_1) # Float Subtraction: 6.6

# Multiplication

print('Int Multiplication:', int_1 * int_2) # Int Multiplication: 672
print('Float Multiplication:', float_2 * float_1) # Float Multiplication: 64.80000000000001

# Division

print('Int Division:', int_1 / int_2) # Int Division: 4.666666666666667
print('Float Division:', float_2 / float_1) # Float Division: 2.222222222222222
```

浮動小数点数と整数を加算すると、結果は浮動小数点数に変換されます。

```py
int_1 = 56
float_1 = 5.4

print(int_1 + float_1) # 61.4
```

- **モジュラス演算子 (`%`)**: これは、ある数を別の数で割ったときの余りを返します。

```py
int_1 = 56
int_2 = 12

print(int_1 % int_2) # 8
```

- **切り捨て除算 (`//`)**: この演算子は2つの数を割り、その結果を最も近い小さい整数に切り捨てます。

```py
int_1 = 56
int_2 = 12

print(int_1 // int_2) # 4
```

- **べき乗演算子 (`**`)**: この演算子は、ある数を別の数のべき乗にします。

```py
int_1 = 4
int_2 = 2

print(int_1 ** int_2) # 16
```

- **`float()` 関数**: この関数は整数を浮動小数点数に変換するために使います。

```py
num = 4

print(float(num)) # 4.0
```

- **`int()` 関数**: この関数は浮動小数点数を整数に変換するために使います。

```py
num = 4.0

print(int(num)) # 4
```

- **`round()` 関数**: この関数は数値を最も近い整数に丸めるために使います。

```py
num_1 = 3.4
num_2 = 7.7

print(round(num_1)) # 3
print(round(num_2)) # 8
```

- **`abs()` 関数**: この関数は数値の絶対値を返すために使います。

```py
num = -13

print(abs(num)) # 13
```

- **`bin()` 関数**: この関数は整数を文字列としての2進数表現に変換するために使います。

```py
num = 56

print(bin(num))  # 0b111000
```

- **`oct()` 関数**: この関数は整数を文字列としての8進数表現に変換するために使います。

```py
num = 56

print(oct(num))  # 0o70
```

- **`hex()` 関数**: この関数は整数を文字列としての16進数表現に変換するために使います。

```py
num = 56

print(hex(num))  # 0x38
```

- **`pow()` 関数**: この関数は数値を別の数のべき乗にするために使います。

```py
result = pow(2, 3) 
print(result)  # 8
```

## 拡張代入

- **定義**：拡張代入は、二項演算と代入を1つのステップで組み合わせたものです。変数に対して別の値と演算を適用し、その結果を同じ変数に代入します。

```py
# Addition assignment 
my_var = 10
my_var += 5

print(my_var) # 15

# Subtraction assignment
count = 14
count -= 3

print(count) # 11

# Multiplication assignment 
product = 65
product *= 7

print(product) # 455

# Division assignment 
price = 100
price /= 4

print(price) # 25.0

# Floor Division assignment 
total_pages = 23
total_pages //= 5

print(total_pages) # 4

# Modulo assignment 
bits = 35
bits %= 2

print(bits) # 1

# Exponentiation assignment 
power = 2
power **= 3

print(power) # 8
```

ビット演算子用の拡張代入演算子もあります。これには `&=`、`^=`、`>>=`、`<<=` が含まれます。

## 関数の操作

- **定義**：関数は再利用可能なコードの断片で、入力（引数）を受け取り、出力を返します。関数を呼び出すには、関数名の後に丸括弧を付けて参照します。

```py
# Defining a function

def get_sum(num_1, num_2):
    return num_1 + num_2

result = get_sum(3, 4) # function call
print(result) # 7
```

関数が明示的に戻り値を返さない場合、デフォルトの戻り値は `None` です。

```py
def greet():
    print('hello') 

result = greet() # hello
print(result) # None
```

パラメータにデフォルト値を設定することもできます。

```py
def get_sum(num_1, num_2=2):
    return num_1 + num_2

result = get_sum(3) 
print(result) # 5
```

正しい数の引数を指定せずに関数を呼び出すと、`TypeError` が発生します。

```py
def calculate_sum(a, b):
    print(a + b)

calculate_sum()

# TypeError: calculate_sum() missing 2 required positional arguments: 'a' and 'b'
```

## Pythonのスコープ

- **ローカルスコープ**: 関数やクラスの中で宣言された変数が、その関数やクラスの中だけでアクセスできる場合です。

```py
def my_func():
    num = 10
    print(num)
```

- **囲むスコープ**: ある関数の中にネストされた関数が、その外側の関数の変数にアクセスできる場合です。

```py
def outer_func():
    msg = 'Hello there!'
    
    def inner_func():
        print(msg)
    inner_func()

print(outer_func()) # Hello there!
```

- **グローバルスコープ**: 関数やクラスの外で宣言された変数で、プログラムのどこからでもアクセスできるものを指します。

```py
tax = 0.70 

def get_total(subtotal):
    total = subtotal + (subtotal * tax)
    return total

print(get_total(100))  # 170.0
```

- **組み込みスコープ**: Pythonであらかじめ定義されている関数、モジュール、キーワード、オブジェクトの予約名です。

```py
print(str(45)) # '45'
print(type(3.14)) # <class 'float'>
print(isinstance(3, str)) # False
```

## 比較演算子

- **Equal (`==`)**: 2つの値が等しいかどうかをチェックします。

```py
print(3 == 4) # False
```

- **Not equal (`!=`)**: 2つの値が等しくないかどうかをチェックします。

```py
print(3 != 4) # True
```

- **Strictly greater than (`>`)**: ある値が別の値より大きいかどうかをチェックします。

```py
print(3 > 4) # False
```

- **Strictly less than (`<`)**: ある値が別の値より小さいかどうかをチェックします。

```py
print(3 < 4) # True
```

- **Greater than or equal(`>=`)**: ある値が別の値以上かどうかをチェックします。

```py
print(3 >= 4) # False
```

- **Less than or equal(`<=`)**: ある値が別の値以下かどうかをチェックします。

```py
print(3 <= 4) # True
```

## `if`、`elif`、`else` 文の使い方

- **`if` 文**：これは何かが真かどうかを判断するための条件です。条件が`True`と評価されると、そのコードブロックが実行されます。

```py
age = 18

if age >= 18:
    print('You are an adult') # You are an adult
```

- **`elif` 節**：これは`if` 文の後に続く条件です。`elif` の条件が`True`と評価されると、そのコードブロックが実行されます。

```py
age = 16

if age >= 18:
    print('You are an adult')
elif age >= 13:
    print('You are a teenager')  # You are a teenager
```

- **`else` 節**：他のどの条件も`True`と評価されなかった場合に実行されます。

```py
age = 12

if age >= 18:
    print('You are an adult')
elif age >= 13:
    print('You are a teenager')
else:
    print('You are a child')  # You are a child
```

このようにネストされた`if` 文も使えます：

```py
is_citizen = True
age = 25

if is_citizen:
    if age >= 18:
        print('You are eligible to vote') # You are eligible to vote
else:
    print('You are not eligible to vote')
```

## 真偽値（Truthy と Falsy）

- **定義**: Pythonでは、すべての値に固有のブール値があり、論理的な文脈で `True` または `False` として扱われるかどうかの組み込みの感覚があります。多くの値は truthy と見なされ、論理的な文脈で `True` と評価されます。その他は falsy で、`False` と評価されます。以下は falsy 値の例です。

```md
None
False
Integer 0
Float 0.0
Empty strings ''
```

非ゼロの数値や空でない文字列などの他の値は truthy です。

## `bool()` 関数の使い方

- **定義**: 値が真値か偽値かを確認したい場合、組み込みの `bool()` 関数を使うことができます。この関数は値を明示的にブール値に変換し、真値の場合は `True` を、偽値の場合は `False` を返します。いくつかの例を示します。

```py
print(bool(False)) # False
print(bool(0))  # False
print(bool('')) # False

print(bool(True)) # True
print(bool(1)) # True
print(bool('Hello')) # True
```

## ブール演算子とショートサーキット

- **定義**: これらは複数の式を組み合わせて、コード内でより複雑な意思決定ロジックを作成できる特別な演算子です。Pythonには3つのブール演算子があります：`and`、`or`、および`not`です。
- **`and` 演算子**: この演算子は2つのオペランドを取り、最初のオペランドが偽値の場合はそれを返し、そうでなければ2番目のオペランドを返します。式が真値になるには両方のオペランドが真値である必要があります。

```py
is_citizen = True
age = 25

print(is_citizen and age) # 25
```

条件文で`and` 演算子を次のように使うこともできます：

```py
is_citizen = True
age = 25

if is_citizen and age >= 18:
    print('You are eligible to vote') # You are eligible to vote
else:
    print('You are not eligible to vote')
```

- **`or` 演算子**: この演算子は最初のオペランドが真値であればそれを返し、そうでなければ2番目のオペランドを返します。`or` 式は少なくとも1つのオペランドが真値であれば真値になります。例を示します：

```py
age = 19
is_employed = False

print(age or is_employed) # 19
```

`and` 演算子と同様に、条件文で`or` 演算子を次のように使えます：

```py
age = 19
is_student = True

if age < 18 or is_student:
    print('You are eligible for a student discount') # You are eligible for a student discount
else:
    print('You are not eligible for a student discount')
```

- **ショートサーキット**: `and` と `or` 演算子はショートサーキット演算子として知られています。ショートサーキットとは、Pythonが左から右へ値をチェックし、最終結果が決まった時点で処理を停止することを意味します。
- **`not` 演算子**: この演算子は1つのオペランドを取り、そのブール値を反転させます。真値を`False` に、偽値を`True` に変換します。これまで見てきた演算子とは異なり、`not` は常に`True` または`False` を返します。いくつか例を示します：

```py
print(not '') # True, because empty string is falsy
print(not 'Hello') # False, because non-empty string is truthy
print(not 0) # True, because 0 is falsy
print(not 1) # False, because 1 is truthy
print(not False) # True, because False is falsy
print(not True) # False, because True is truthy
```

条件文での`not` 演算子の例はこちらです：

```py
is_admin = False

if not is_admin:
    print('Access denied for non-administrators.') # Access denied for non-administrators.
else:
    print('Welcome, Administrator!')
```

## Pythonのリスト

- **イントロダクション**: Pythonでは、リストのデータ型は文字列、数値、または他のリストで構成されることがある順序付きのシーケンスです。リストはミュータブルでゼロベースのインデックスを持ちます。

```python
cities = ['Los Angeles', 'London', 'Tokyo']
```

- **リストの要素へのアクセス**: `cities`リストから要素にアクセスするには、シーケンス内のインデックス番号を参照します。

```python
cities = ['Los Angeles', 'London', 'Tokyo']
cities[0] # Los Angeles
```

- **負のインデックスを使った要素へのアクセス**: 任意のリストの最後の要素にアクセスするには、インデックス番号として`-1`を使います。

```python
cities = ['Los Angeles', 'London', 'Tokyo']
cities[-1] # Tokyo
```

- 負のインデックスは、リストの先頭のインデックス`0`ではなく、末尾から要素にアクセスするために使われます。

- **`list()`コンストラクタを使ったリストの作成**: リストは`list()`コンストラクタを使っても作成できます。`list()`コンストラクタはイテラブルをリストに変換するために使われます。

```python
developer = 'Jessica'

print(list(developer)) 
# Result: ['J', 'e', 's', 's', 'i', 'c', 'a']
```

- **リストの長さを調べる**: `len()`関数を使ってリストの長さを取得できます。

```python
numbers = [1, 2, 3, 4, 5]
len(numbers) # 5
```

- **リストのミュータビリティ**: リストはミュータブルなので、有効なインデックス番号を指定すればリスト内の任意の要素を更新できます。特定のインデックスのリストを更新するには、そのインデックスに新しい値を代入します。

```python
programming_languages = ['Python', 'Java', 'C++', 'Rust']
programming_languages[0] = 'JavaScript'
print(programming_languages) # ['JavaScript', 'Java', 'C++', 'Rust']
```

- **インデックス範囲外エラー**: リストの範囲外のインデックス（正または負）を指定すると、`IndexError`が発生します。

```python
programming_languages = ['Python', 'Java', 'C++', 'Rust']
programming_languages[10] = 'JavaScript'

"""
Traceback (most recent call last):
  File "<stdin>", line 1, in <module>
IndexError: list assignment index out of range
"""
```

- **リストから要素を削除する**: `del`キーワードを使ってリストから要素を削除できます。

```python
developer = ['Jane Doe', 23, 'Python Developer']
del developer[1]
print(developer) # ['Jane Doe', 'Python Developer']
```

- **リストに要素が存在するか確認する**: `in`キーワードを使ってリストに要素が存在するか確認できます。

```py
programming_languages = ['Python', 'Java', 'C++', 'Rust']

'Rust' in programming_languages # True
'JavaScript' in programming_languages # False
```

- **リストのネスト**: リストは他のリストの中にネストできます。

```python
developer = ['Alice', 25, ['Python', 'Rust', 'C++']]
```

- ネストされたリストにアクセスするには、リストがゼロベースのインデックスなのでインデックス`2`を使います。

```python
developer = ['Alice', 25, ['Python', 'Rust', 'C++']]
developer[2] # ['Python', 'Rust', 'C++']
```

- さらにそのネストされたリストから2番目の言語にアクセスするには、インデックス`1`を使います。

```python
developer = ['Alice', 25, ['Python', 'Rust', 'C++']]
developer[2][1] # Rust
```

- **リストからの値のアンパック**: リストから値をアンパックする手法は、リストの値を新しい変数に代入するために使われます。例えば、`developer`リストを`name`、`age`、`job`という新しい変数にアンパックする例は以下の通りです。

```py
developer = ['Alice', 34, 'Rust Developer']
name, age, job = developer
```

- 代入演算子の左側の変数の数がリスト内のアイテムの総数と一致しない場合、`ValueError`が発生します。

- **リストの残りの要素をまとめて取得する**: リストの残りの要素をまとめて取得するには、アスタリスク（`*`）演算子を使います。

```py
developer = ['Alice', 34, 'Rust Developer']
name, *rest = developer
```

- **リストのスライス**: スライスはスライス演算子`:`を使ってリストの一部にアクセスする手法です。インデックス`1`から始まりインデックス`3`の直前までのリストをスライスするには、以下の構文を使います。

```python
desserts = ['Cake', 'Cookies', 'Ice Cream', 'Pie']
desserts[1:3] # ['Cookies', 'Ice Cream']
```

- **ステップ間隔**: インデックス間の増分を指定するステップ間隔を設定することも可能です。例えば、スライスを使って偶数だけのリストを抽出する例は以下の通りです。

```python
numbers = [1, 2, 3, 4, 5, 6]
numbers[1::2] # [2, 4, 6]
```

## List Methods

- **append()**: リストの末尾にアイテムを追加するために使います。以下は`append()`メソッドを使って、`6`という数字をこの`numbers`リストに追加する例です。

```py
numbers = [1, 2, 3, 4, 5]
numbers.append(6)
print(numbers) # [1, 2, 3, 4, 5, 6]
```

- **リストの追加**: `append()`メソッドは、あるリストの末尾に別のリストを追加するためにも使えます。

```py
numbers = [1, 2, 3, 4, 5]
even_numbers = [6, 8, 10]

numbers.append(even_numbers)
print(numbers) # [1, 2, 3, 4, 5, [6, 8, 10]]
```

- **extend()**: 複数のアイテムをリストの末尾に追加するために使います。以下は`6`、`8`、`10`という数字を`numbers`リストの末尾に追加する例です。

```py
numbers = [1, 2, 3, 4, 5]
even_numbers = [6, 8, 10]

numbers.extend(even_numbers)
print(numbers) # [1, 2, 3, 4, 5, 6, 8, 10]
```

- **insert()**: リストの特定のインデックスにアイテムを挿入するために使います。以下は`insert()`メソッドを使った例です。

```py
numbers = [1, 2, 3, 4, 5]
numbers.insert(2, 2.5)

print(numbers) # [1, 2, 2.5, 3, 4, 5]
```

- **remove()**: リストからアイテムを削除するために使います。`remove()`メソッドはリスト内の最初の該当アイテムだけを削除します。

```py
numbers = [1, 2, 3, 4, 5, 5, 5]
numbers.remove(5)

print(numbers) # [1, 2, 3, 4, 5, 5]

```

- **pop()**: リストから特定のアイテムを削除して、そのアイテムを返します。

```py
numbers = [1, 2, 3, 4, 5]
numbers.pop(1) # The number 2 is returned
```

- `pop`メソッドで要素を指定しない場合は、最後の要素が削除されます。

```py
numbers = [1, 2, 3, 4, 5]
numbers.pop() # The number 5 is returned
```

- **clear()**: リストのすべてのアイテムを削除するために使います。

```py
numbers = [1, 2, 3, 4, 5]
numbers.clear()

print(numbers) # []
```

- **sort()**: `sort()`メソッドは要素をその場でソートします。以下はランダムな`numbers`のリストをその場でソートする例です。

```py
numbers = [19, 2, 35, 1, 67, 41]
numbers.sort()

print(numbers) # [1, 2, 19, 35, 41, 67]
```

- **sorted()**: 元のリストを変更せずに、新しいソート済みリストを返す組み込み関数です。

```py
numbers = [19, 2, 35, 1, 67, 41]
sorted_numbers = sorted(numbers)

print(sorted_numbers) # [1, 2, 19, 35, 41, 67]
print(numbers) # [19, 2, 35, 1, 67, 41]
```

- **reverse()**: リストの要素の順序を逆にするために使います。

```py
numbers = [6, 5, 4, 3, 2, 1]
numbers.reverse()

print(numbers) # [1, 2, 3, 4, 5, 6]
```

- **index()**: リスト内で要素が最初に見つかるインデックスを探すために使います。

```python
programming_languages = ['Rust', 'Java', 'Python', 'C++']
programming_languages.index('Java') # 1
```

- `index()`メソッドで要素が見つからない場合、結果は`ValueError`になります。

## Pythonのタプル

- **定義**：タプルはPythonのデータ型で、順序付けられた値のシーケンスを作成するために使われます。タプルは異なるデータ型を混在させることができます。

```py
developer = ('Alice', 34, 'Rust Developer')
```

- タプルはイミュータブルで、作成後に要素を変更できません。タプル内のアイテムを更新しようとすると、`TypeError`が発生します。

```py
programming_languages = ('Python', 'Java', 'C++', 'Rust')
programming_languages[0] = 'JavaScript'

"""
Traceback (most recent call last):
  File "<stdin>", line 1, in <module>
TypeError: "tuple" object does not support item assignment
"""
```

- **タプルの要素へのアクセス**：タプルの要素にアクセスするには、ブラケット記法とインデックス番号を使います。

```py
developer = ('Alice', 34, 'Rust Developer')
developer[1] # 34
```

- 負のインデックスを使うと、タプルの末尾から要素にアクセスできます。

```py
numbers = (1, 2, 3, 4, 5)
numbers[-2] # 4
```

- インデックス番号がタプルの長さ以上の場合、`IndexError`が発生します。

```py
numbers = (1, 2, 3, 4, 5)
numbers[7]

"""
Traceback (most recent call last):
  File "<stdin>", line 1, in <module>
IndexError: tuple index out of range
"""
```

- タプルは`tuple()`コンストラクタを使って作成することもできます。コンストラクタ内には文字列、リスト、他のタプルなどのイテラブルを渡せます。

```python
developer = 'Jessica'

print(tuple(developer)) 
# Result: ('J', 'e', 's', 's', 'i', 'c', 'a')
```

- **タプル内のアイテムの確認**：アイテムがタプルに含まれているか確認するには、`in`キーワードを使います。

```py
programming_languages = ('Python', 'Java', 'C++', 'Rust')

'Rust' in programming_languages # True
'JavaScript' in programming_languages # False
```

- **タプルのアンパック**：タプルからアイテムをアンパックするには次のようにします。

```py
developer = ('Alice', 34, 'Rust Developer')
name, age, job = developer
```

- タプルの残りの要素をまとめて取得したい場合は、アスタリスク（`*`）演算子を使います。

```py
developer = ('Alice', 34, 'Rust Developer')
name, *rest = developer
```

- **タプルのスライス**：スライスを使ってタプルの一部を抽出できます。例えば、`pie`と`cookies`のアイテムを別のタプルにスライスできます。

```py
desserts = ('cake', 'pie', 'cookies', 'ice cream')
desserts[1:3] # ('pie', 'cookies')
```

- **タプルからのアイテムの削除**：タプルはイミュータブルなので、アイテムを削除しようとすると`TypeError`が発生します。

```py
developer = ('Jane Doe', 23, 'Python Developer')
del developer[1]

"""
Traceback (most recent call last):
  File "<stdin>", line 1, in <module>
TypeError: "tuple" object doesn't support item deletion
"""
```

- **タプルとリストの使い分け**：要素の追加、削除、更新が必要な動的なコレクションにはリストを使います。固定で変更しないコレクションの場合はタプルを使うべきです。

## 一般的なタプルのメソッド

- **count()**: タプル内に特定のアイテムが何回現れるかを調べるために使います。例えば、タプル内に言語 `'Rust'` が何回現れるかを確認できます。

```py
programming_languages = ('Rust', 'Java', 'Python', 'C++', 'Rust')
programming_languages.count('Rust') # 2
```

- `count()` メソッドで指定したアイテムがタプルに全く存在しない場合、戻り値は `0` になります。

```py
programming_languages = ('Rust', 'Java', 'Python', 'C++', 'Rust')
programming_languages.count('JavaScript') # 0
```

- `count()` メソッドに引数を渡さない場合、Pythonは `TypeError` を返します。

- **index()**: タプル内の特定のアイテムが存在するインデックスを見つけるために使います。以下は言語 `'Java'` のインデックスを見つけるために `index()` メソッドを使った例です。

```py
programming_languages = ('Rust', 'Java', 'Python', 'C++', 'Rust')
programming_languages.index('Java') # 1
```

- 指定したアイテムが見つからない場合、Pythonは `ValueError` を返します。

- `index()` メソッドには、検索を開始する位置を指定するためのオプションの開始インデックスを渡せます。

```py
programming_languages = ('Rust', 'Java', 'Python', 'C++', 'Rust', 'Python')
programming_languages.index('Python', 3) # 5
```

- `index()` メソッドには、検索を終了する位置を指定するためのオプションの終了インデックスも渡せます。

```py
programming_languages = ('Rust', 'Java', 'Python', 'C++', 'Rust', 'Python', 'JavaScript', 'Python')
programming_languages.index('Python', 2, 5) # 2
```

- **sorted()**: 任意のイテラブルの要素をソートして、新しいソート済みリストを返すために使います。以下は `sorted()` 関数を使って新しい数値リストを作成する例です。

```py
numbers = (13, 2, 78, 3, 45, 67, 18, 7)
sorted(numbers) # [2, 3, 7, 13, 18, 45, 67, 78]
```

- **ソートの動作を変更する**: オプションの `reverse` と `key` 引数を使って、イテラブルのソート動作をカスタマイズできます。以下は `key` 引数を使ってタプル内のアイテムを長さでソートする例です。

```py
programming_languages = ('Rust', 'Java', 'Python', 'C++', 'Rust', 'Python')
sorted(programming_languages, key=len)

# Result
# ['C++', 'Rust', 'Java', 'Rust', 'Python', 'Python']
```

- `reverse` 引数を使うと、値の新しいリストを逆順で作成できます。

```py
programming_languages = ('Rust', 'Java', 'Python', 'C++', 'Rust', 'Python')

print(sorted(programming_languages, reverse=True))

# Result
# ['Rust', 'Rust', 'Python', 'Python', 'Java', 'C++']
```

## Pythonのループ

- **定義**：ループはコードのブロックを決まった回数繰り返すために使います。

- **`for` ループ**：リスト、タプル、文字列のようなシーケンスを繰り返し処理し、そのシーケンスの各要素に対してコードのブロックを実行します。以下はリストを繰り返し処理して各言語をコンソールに出力する `for` ループの例です：

```py
programming_languages = ['Rust', 'Java', 'Python', 'C++']

for language in programming_languages:
    print(language)

"""
Result 

Rust
Java
Python
C++
"""
```

- 以下は文字列 `word` を繰り返し処理し、各文字を出力する `for` ループの例です：

```py
for char in 'code':
    print(char)

"""
Result 

c
o
d
e
"""
```

- `for` ループはネストできます。以下はネストされた `for` ループの例です：

```py
categories = ['Fruit', 'Vegetable']
foods = ['Apple', 'Carrot', 'Banana']

for category in categories:
    for food in foods:
        print(category, food)

"""
Result

Fruit Apple
Fruit Carrot
Fruit Banana
Vegetable Apple
Vegetable Carrot
Vegetable Banana
"""

```

- **`while` ループ**：条件が `True` の間、コードのブロックを繰り返します。以下は推測ゲームに `while` ループを使った例です：

```py
secret_number = 3
guess = 0

while guess != secret_number:
    guess = int(input('Guess the number (1-5): '))
    if guess != secret_number:
        print('Wrong! Try again.')

print('You got it!')

"""
Result

Guess the number (1-5): 2
Wrong! Try again.
Guess the number (1-5): 1
Wrong! Try again.
Guess the number (1-5): 3
You got it!
"""

```

- **`break` と `continue` 文**：ループの実行を変更するために使います。

- `break` 文は特定の条件が満たされたときにループを即座に終了します。以下は `break` 文を使ってリストの中の要素を処理する例です：

```py
developer_names = ['Jess', 'Naomi', 'Tom']

for developer in developer_names:
    if developer == 'Naomi':
        break
    print(developer)
```

- `continue` 文は現在の反復をスキップして次の反復に進みます。以下は `continue` 文を使って `if` 文の代わりに処理をスキップする例です：

```py
developer_names = ['Jess', 'Naomi', 'Tom']

for developer in developer_names:
    if developer == 'Naomi':
        continue
    print(developer)
```

- `for` ループと `while` ループの両方は `else` 節と組み合わせることができ、これはループが `break` によって終了しなかった場合にのみ実行されます：

```python
words = ['sky', 'apple', 'rhythm', 'fly', 'orange']

for word in words:
    for letter in word:
        if letter.lower() in 'aeiou':
            print(f"'{word}' contains the vowel '{letter}'")
            break
    else:
        print(f"'{word}' has no vowels")
```

## Ranges とループでの使い方

- **`range()` 関数**：整数のシーケンスを生成するために使います。

```py
range(start, stop, step)
```

- 必須の `stop` 引数は整数（非包含）で、生成される数列の終点を表します。以下は `range()` 関数を使った例です：

```py
for num in range(3):
    print(num)
```

- `start` 引数が指定されない場合、デフォルトは `0` になります。デフォルトでは整数のシーケンスは `1` ずつ増加します。オプションの `step` 引数を使ってデフォルトの増分値を変更できます。以下は 2 から 11 未満（つまり 10 を含む）までの偶数のシーケンスを生成する例です。

```py
for num in range(2, 11, 2):
    print(num)
```

- `range()` 関数に引数を何も渡さないと、`TypeError` が得られます。

- `range()` 関数は引数に整数のみを受け付け、浮動小数点数は受け付けません。浮動小数点数を使うと `TypeError` になります。

```py
ERROR!
Traceback (most recent call last):
  File "<main.py>", line 1, in <module>
TypeError: 'float' object cannot be interpreted as an integer
```

- `step` 引数に負の整数を使うと、減少順の整数シーケンスを生成できます。

```py
for num in range(40, 0, -10):
    print(num)
```

- `range()` 関数は `list` コンストラクタと組み合わせて使うことで、整数のリストを作成することもできます。`list` コンストラクタはイテラブルをリストに変換するために使います。以下は 2 から 10 までの偶数のリストを生成する例です。

```py
numbers = list(range(2, 11, 2))
print(numbers) # [2, 4, 6, 8, 10]
```

## Pythonの`enumerate()`と`zip()`関数

- **`enumerate()`**：シーケンスを繰り返し処理し、そのシーケンス内の各アイテムのインデックスを追跡するために使います。`enumerate()`関数はイテラブルを引数に取り、イテラブル内の各アイテムのインデックスと値からなる`enumerate`オブジェクトを返します。

```python
languages = ['Spanish', 'English', 'Russian', 'Chinese']

for index, language in enumerate(languages):
    print(f'Index {index} and language {language}')

# Result
# Index 0 and language Spanish
# Index 1 and language English
# Index 2 and language Russian
# Index 3 and language Chinese

```

- `enumerate()`関数は`for`ループの外でも使うことができます。

```python
languages = ['Spanish', 'English', 'Russian', 'Chinese']

print(list(enumerate(languages)))
# [(0, 'Spanish'), (1, 'English'), (2, 'Russian'), (3, 'Chinese')]
```

- `enumerate()`関数は、カウントの開始値を指定するオプションの`start`引数も受け取ります。この引数を省略すると、カウントは`0`から始まります。

- **`zip()`**：複数のイテラブルを並行して繰り返すために使います。以下は`zip()`関数を使って`developers`と`ids`を繰り返す例です。

```python
developers = ['Naomi', 'Dario', 'Jessica', 'Tom']
ids = [1, 2, 3, 4]

for name, id in zip(developers, ids):
    print(f'Name: {name}')
    print(f'ID: {id}')


"""
Result

Name: Naomi
ID: 1
Name: Dario
ID: 2
Name: Jessica
ID: 3
Name: Tom
ID: 4
"""

```

## Pythonのリスト内包表記

- **定義**: リスト内包表記は、ループと条件を大括弧の中に直接組み合わせて、1行で新しいリストを作成できる方法です。これによりコードが短くなり、読みやすくなることが多いです。

```py
even_numbers = [num for num in range(21) if num % 2 == 0]
print(even_numbers)
```

## イテラブル関数

- **`filter()`**: 条件に基づいてイテラブルから要素をフィルターするために使います。条件を満たす要素だけを含むイテレーターを返します。4文字より長い単語だけの新しいリストを作成する例を示します。

```py
words = ['tree', 'sky', 'mountain', 'river', 'cloud', 'sun']

def is_long_word(word):
    return len(word) > 4

long_words = list(filter(is_long_word, words))
print(long_words) # ['mountain', 'river', 'cloud']
```

- **`map()`**: イテラブルの各アイテムに関数を適用し、その結果を含む新しいイテラブルを返すために使います。摂氏温度のリストを華氏に変換するために`map()`関数を使う例を示します。

```py
celsius = [0, 10, 20, 30, 40]

def to_fahrenheit(temp):
    return (temp * 9/5) + 32

fahrenheit = list(map(to_fahrenheit, celsius))
print(fahrenheit) # [32.0, 50.0, 68.0, 86.0, 104.0]
```

- **`sum()`**: リストやタプルのようなイテラブルから合計を取得するために使います。`sum()`関数を使う例を示します。

```py
numbers = [5, 10, 15, 20]
total = sum(numbers)
print(total) # Result: 50
```

- また、合計の初期値を設定するオプションの`start`引数を渡すこともできます。位置引数として`start`引数を使った更新例を示します。

```py
numbers = [5, 10, 15, 20]
total = sum(numbers, 10) # positional argument
print(total) # 60
```

- 代わりにキーワード引数として`start`引数を使うこともできます。

```py
numbers = [5, 10, 15, 20]
total = sum(numbers, start=10) # keyword argument
print(total) # 60
```

## ラムダ関数

- **定義**: Pythonのラムダ関数は、名前のない関数（無名関数）を簡潔に作成する方法です。  
- ラムダ関数は、別の関数への引数としてよく使われます。以下はラムダ関数の例です。

```py
numbers = [1, 2, 3, 4, 5]

even_numbers = list(filter(lambda x: x % 2 == 0, numbers))
print(even_numbers)  # [2, 4]
```

- ラムダ関数を使う際のベストプラクティスは、変数に代入しないこと、シンプルで読みやすく保つこと、短く一度きりの関数に使うことです。

## 辞書

- **辞書**：辞書はキーと値のペアのコレクションをストアする組み込みのデータ構造です。キーは不変のデータ型である必要があります。Pythonの辞書の一般的な構文は次の通りです。

```python
dictionary = {
    key1: value1,
    key2: value2
}
```

- **`dict()` コンストラクタ**：`dict()` コンストラクタは辞書を作成する別の方法です。`dict()` コンストラクタにタプルのリストを引数として渡します。これらのタプルは、最初の要素にキー、2番目の要素に値を含みます。

```python
pizza = dict([('name', 'Margherita Pizza'), ('price', 8.9), ('calories_per_slice', 250), ('toppings', ['mozzarella', 'basil'])])
```

- **ブラケット記法**：キーと値のペアの値にアクセスするには、ブラケット記法として知られる構文を使うことができます。

```python
dictionary[key]
```

## Common Dictionary Methods

- **`get()` メソッド**: `get()` メソッドは、キーに関連付けられた値を取得します。ブラケット記法に似ていますが、デフォルト値を設定できるため、キーが存在しない場合のエラーを防げます。

```python
dictionary.get(key, default)
```

- **`keys()` と `values()` メソッド**: `keys()` と `values()` メソッドは、それぞれ辞書内のすべてのキーと値を含むビューオブジェクトを返します。ビューオブジェクトは、データの別コピーを作成せずに辞書の内容を確認する方法です。

```python
pizza = {
    'name': 'Margherita Pizza',
    'price': 8.9,
    'calories_per_slice': 250
}

pizza.keys()
# dict_keys(['name', 'price', 'calories_per_slice'])

pizza.values()
# dict_values(['Margherita Pizza', 8.9, 250])
```

- **`items()` メソッド**: `items()` メソッドは、キーと値の両方を含む辞書内のすべてのキーと値のペアを含むビューオブジェクトを返します。

```python
pizza.items()
# dict_items([('name', 'Margherita Pizza'), ('price', 8.9), ('calories_per_slice', 250)])
```

- **`clear()` メソッド**: `clear()` メソッドは、辞書からすべてのキーと値のペアを削除します。

```python
pizza.clear()
```

- **`pop()` メソッド**: `pop()` メソッドは、最初の引数で指定されたキーのキーと値のペアを削除し、その値を返します。キーが存在しない場合は、2番目の引数で指定されたデフォルト値を返します。キーが存在せず、デフォルト値が指定されていない場合は `KeyError` が発生します。

```python
pizza.pop('price', 10)
pizza.pop('total_price') # KeyError
```

- **`popitem()` メソッド**: Python 3.7以降では、`popitem()` メソッドは最後に挿入されたアイテムを削除します。

```python
pizza.popitem()
```

- **`update()` メソッド**: `update()` メソッドは、別の辞書のキーと値のペアで辞書を更新します。共通のキーがある場合は、その値が上書きされます。新しいキーは新しいキーと値のペアとして辞書に追加されます。

```python
pizza.update({ 'price': 15, 'total_time': 25 })
```

## 辞書のループ処理

- **値の繰り返し**: 辞書の値を繰り返す必要がある場合は、`for` ループを `values()` と一緒に書いて、辞書のすべての値を取得できます。

```python
products = {
    'Laptop': 990,
    'Smartphone': 600,
    'Tablet': 250,
    'Headphones': 70,
}

for price in products.values():
    print(price)
```

出力:

```md
990
600
250
70
```

- **キーの繰り返し**: 上記の `products` 辞書のキーを繰り返す必要がある場合は、`products.keys()` または `products` を直接書くことができます。

```python
for product in products.keys():
    print(product)
    
# Or

for product in products:
    print(product)
```

出力:

```md
Laptop
Smartphone
Tablet
Headphones
```

- **キーと値のペアの繰り返し**: キーとそれに対応する値を同時に繰り返す必要がある場合は、`products.items()` を繰り返すことができます。キーと対応する値を持つ個別のタプルが得られます。

```python
for product in products.items():
    print(product)
```

出力:

```md
('Laptop', 990)
('Smartphone', 600)
('Tablet', 250)
('Headphones', 70)
```

キーと値を別々のループ変数に格納するには、カンマで区切る必要があります。最初の変数がキーを格納し、2番目の変数が値を格納します。

```python
for product, price in products.items():
    print(product, price)
```

出力:

```md
Laptop 990
Smartphone 600
Tablet 250
Headphones 70
```

- **`enumerate()` 関数**: カウンターを追跡しながら辞書を繰り返す必要がある場合は、`enumerate()` 関数を呼び出せます。この関数は `enumerate` オブジェクトを返し、各アイテムに整数を割り当ててカウンターのように扱います。カウンターは任意の数字から開始できますが、デフォルトでは0から始まります。

インデックスとアイテムを別々のループ変数に代入するのが `enumerate()` の一般的な使い方です。例えば `products.items()` を使うと、インデックスに加えてキーと値のペア全体を取得できます。

```python
for index, product in enumerate(products.items()):
    print(index, product)
```

出力:

```md
0 ('Laptop', 990)
1 ('Smartphone', 600)
2 ('Tablet', 250)
3 ('Headphones', 70)
```

カウントの初期値をカスタマイズするには、`enumerate()` に2番目の引数を渡します。例えば、ここではカウントを1から始めています。

```python
for index, product in enumerate(products.items(), 1):
    print(index, product)
```

出力:

```md
1 ('Laptop', 990)
2 ('Smartphone', 600)
3 ('Tablet', 250)
4 ('Headphones', 70)
```

## Sets

- **セット**: セットはPythonの組み込みデータ構造で、重複する値を許しません。セットは変更可能で順序がないため、要素は特定の順序で格納されておらず、インデックスやキーでアクセスすることはできません。また、セットには数値、文字列、タプルのような変更不可能なデータ型の値のみを含めることができます。

- **セットの定義**: セットを定義するには、要素を波括弧の中に書き、カンマで区切ります。

```python
my_set = {1, 2, 3, 4, 5}
```

- **空のセットの定義**: 空のセットを定義する必要がある場合は、`set()`関数を使う必要があります。空の波括弧だけを書くと、自動的に辞書が作成されます。

```python
set() # Set
{}    # Dictionary
```

## Common Set Methods

- **`add()` メソッド**: 新しい要素を引数として渡すことで、`add()` メソッドを使ってセットに要素を追加できます。

```python
my_set.add(6)
```

- **`remove()` と `discard()` メソッド**: セットから要素を削除するには、削除したい要素を引数として渡して、`remove()` メソッドか `discard()` メソッドのどちらかを使えます。`remove()` メソッドは要素が見つからない場合に `KeyError` を発生させますが、`discard()` メソッドは発生させません。

```python
my_set.remove(4)
my_set.discard(4)
```

- **`clear()` メソッド**: `clear()` メソッドはセットからすべての要素を削除します。

```python
my_set.clear()
```

## 数学的な集合演算

- **`issubset()` と `issuperset()` メソッド**: `issubset()` メソッドと `issuperset()` メソッドは、それぞれある集合が別の集合の部分集合かどうか、または上位集合かどうかを確認します。

```python
my_set = {1, 2, 3, 4, 5}
your_set = {2, 3, 4, 5}

print(your_set.issubset(my_set)) # True
print(my_set.issuperset(your_set)) # True
```

- **`isdisjoint()` メソッド**: `isdisjoint()` メソッドは、2つの集合が互いに素（共通の要素を持たない）かどうかを確認します。

```python
my_set = {1, 2, 3}
your_set = {4, 5, 6}

print(my_set.isdisjoint(your_set)) # True
```

- **和集合演算子 (`|`)**: 和集合演算子 `|` は、両方の集合のすべての要素を含む新しい集合を返します。

```python
my_set = {1, 2, 3}
your_set = {4, 5, 6}

my_set | your_set # {1, 2, 3, 4, 5, 6}
```

- **積集合演算子 (`&`)**: 積集合演算子 `&` は、両方の集合に共通する要素だけを含む新しい集合を返します。

```python
my_set = {1, 2, 3, 4, 5}
your_set = {2, 3, 4, 6}

my_set & your_set # {2, 3, 4}
```

- **差集合演算子 (`-`)**: 差集合演算子 `-` は、最初の集合にあって他の集合にない要素を含む新しい集合を返します。

```python
my_set = {1, 2, 3, 4, 5}
your_set = {2, 3, 4, 6}

my_set - your_set # {1, 5}
```

- **対称差演算子 (`^`)**: 対称差演算子 `^` は、最初の集合か2番目の集合のどちらか一方にだけ含まれる要素を持つ新しい集合を返します。

```python
my_set = {1, 2, 3, 4, 5}
your_set = {2, 3, 4, 6}

my_set ^ your_set # {1, 5, 6}
```

- **`in` 演算子**: `in` 演算子を使って、要素が集合に含まれているかどうかを確認できます。

```python
print(5 in my_set)
```

## Python標準ライブラリ

- **Python標準ライブラリ**：ライブラリとは、関数、クラス、データ構造などの再利用可能なコードがあらかじめ用意されているもので、プロジェクトで再利用できます。Pythonには多くの問題やタスクに対して標準化された解決策を実装した組み込みモジュールを備えた充実した標準ライブラリがあります。よく使われる組み込みモジュールの例として、`math`、`random`、`re`（「正規表現」の略）、および`datetime`があります。

## Import文

- **Import文**：組み込みモジュールで定義された要素にアクセスするには、import文を使います。import文は通常、ファイルの先頭に書きます。import文は関数、クラス、定数、変数、その他モジュール内で定義されたあらゆる要素に対して同じように機能します。

- **基本的なImport文**：`import` キーワードの後にモジュール名を続けて使います。

```python
import module_name
```

その後、そのモジュールの関数を呼び出す必要がある場合は、モジュール名の後に関数名を続けてドット記法を使います。

```python
module_name.function_name()
```

例えば、コード内で `math` モジュールをインポートし、36の平方根を取得するには次のように書きます。

```python
import math

math.sqrt(36)
```

- **別名でモジュールをインポートする**：モジュールを別の名前（「エイリアス」とも呼ばれます）でインポートしたい場合は、import文の最後に `as` とエイリアスを続けて使います。これは長いモジュール名や名前の衝突を避けるためによく使われます。

```python
import module_name as module_alias
```

例えば、`math` モジュールをコード内で `m` として参照したい場合は、次のようにエイリアスを割り当てます。

```python
import math as m
```

その後、エイリアスを使ってモジュールの要素にアクセスできます。

```python
m.sqrt(36)
```

- **特定の要素だけをインポートする**：モジュールのすべてが必要ない場合は、`from` を使って特定の要素だけをインポートできます。この場合、import文は `from` で始まり、モジュール名、`import` キーワード、そしてインポートしたい要素名を続けます。

```python
from module_name import name1, name2
```

その後、Pythonスクリプト内でこれらの名前をモジュールの接頭辞なしで使えます。例えば：

```python
from math import radians, sin, cos

angle_degrees = 40
angle_radians = radians(angle_degrees)

sine_value = sin(angle_radians)
cos_value = cos(angle_radians)

print(sine_value) # 0.6427876096865393
print(cos_value)  # 0.766044443118978
```

これは便利ですが、同じ名前の関数や変数がすでにある場合に名前の衝突が起きる可能性があります。どのタイプのimport文を使うか選ぶ際に注意してください。

これらの名前にエイリアスを割り当てたい場合は、`as` キーワードの後にエイリアスを続けて指定できます。

```python
from module_name import name1 as alias1, name2 as alias2
```

- **アスタリスクを使ったImport文（`*`）**：アスタリスクは、そのモジュール内のすべてをインポートしたいが、モジュール名を接頭辞として使わずに済ませたいことをPythonに伝えます。

```python
from module_name import *
```

例えば、これを使って `math` モジュールをインポートすると、そのモジュール内で定義された関数をモジュール名を指定せずに呼び出せます。

```python
from math import *
print(sqrt(36))  # 6.0
```

ただし、これは名前空間の衝突を招きやすく、名前の由来がわかりにくくなるため、一般的には推奨されません。

## `if __name__ == '__main__'`

- **`__name__` 変数**: `__name__` はPythonの特別な組み込み変数です。Pythonファイルが直接実行されると、この変数の値は文字列 `"__main__"` に設定されます。しかし、そのPythonファイルが別のPythonスクリプトにモジュールとしてインポートされると、`__name__` 変数の値はそのモジュールの名前に設定されます。

このため、Pythonスクリプトでこの条件文をよく見かけます。これはPythonスクリプトがメインプログラムとして実行されている場合に**のみ**実行したいコードを含んでいます。

```python
if __name__ == '__main__': 
    # Code
```

## Pythonでよくあるエラー

- **SyntaxError**: Pythonの構文ルールに従っていないコードに対して発生するエラーです。例えば、コード`print("Hello there"`は閉じ括弧が欠けているため、`SyntaxError: '(' was never closed`というメッセージの構文エラーになります。
- **NameError**: 定義していない変数や関数にアクセスしようとするとPythonが発生させるエラーです。例えば、コードに`print(username)`という行があり、先に`username`という変数が定義されていなければ、`NameError: name 'username' is not defined`というメッセージの名前エラーが発生します。
- **TypeError**: 互換性のないデータ型同士で操作を行ったときにPythonがスローするエラーです。例えば、文字列と数値を足そうとすると、`TypeError: can only concatenate str (not "int") to str`というエラーが発生します。
- **IndexError**: リストやタプル、文字列などのシーケンスで存在しないインデックスにアクセスすると発生します。例えば、`Hello world`という文字列では、最後の文字のインデックスは`10`です。`greet = "hello world"; print(greet[11])`のようにアクセスすると、`IndexError: string index out of range`というメッセージのエラーになります。
- **AttributeError**: その型のオブジェクトに存在しないメソッドやプロパティを使おうとするとPythonが発生させるエラーです。例えば、`"hello".append("!")`のような文字列に対して`.append()`を呼び出すと、`AttributeError: 'str' object has no attribute 'append'`というメッセージのエラーになります。

## 例外処理

- **`try...except`**: これは例外が発生する可能性があるコードのブロックを実行するために使います。`try` ブロックはエラーが起こるかもしれない場所を示し、`except` ブロックは指定した例外が発生した場合に実行されます。例を見てみましょう。

  ```py
  try:
    print(22 / 0)
  except ZeroDivisionError:
    print('You can\'t divide by zero!')
    # You can't divide by zero!
  ```

  複数の `except` ブロックを連結して、より多くの種類の例外を処理することもできます。

  ```py
  try:
    number = int(input('Enter a number: '))
    print(22 / number)
  except ZeroDivisionError:
    print('You cannot divide by zero!')
    # You cannot divide by zero! prints when you enter 0
  except ValueError:
    print('Please enter a valid number!')
    # Please enter a valid number! prints when you don't enter an integer
  ```

- **`else` と `finally`**: これらのブロックは `try...except` を拡張します。例外が発生しなければ `else` ブロックが実行されます。`finally` ブロックはエラーの有無にかかわらず常に実行されます。

  ```py
  try:
    result = 100 / 4
  except ZeroDivisionError:
    print('You cannot divide by zero!') # This will not run
  else:
    print(f'Result is {result}') # Result is 25.0
  finally:
    print('Execution complete!') # Execution complete!
  ```

- **例外オブジェクト**: これは例外そのものにアクセスできるので、デバッグや直接エラーメッセージを出力するのに便利です。例外オブジェクトにアクセスするには `as` キーワードを使います。例を見てみましょう。

  ```py
  try:
      value = int('This will raise an error')
  except ValueError as e:
      print(f'Caught an error: {e}')
      # Caught an error: invalid literal for int() with base 10: 'This will raise an error'
  ```

- **`raise` 文**: これは手動で例外をスローするために使います。特定の条件が満たされたときに例外をスローすることができます。例を見てみましょう。

  ```py
  def divide(a, b):
      if b == 0:
          raise ZeroDivisionError('You cannot divide by zero')
      return a / b
  ```

## Exception Signaling
  
  `raise` 文は、独自のカスタム例外を作成するときにも便利です。カスタムメッセージ付きで例外をスローするために使えます。以下はその例です。

  ```py
  class InvalidCredentialsError(Exception):
      def __init__(self, message="Invalid username or password"):
          self.message = message
          super().__init__(self.message)

  def login(username, password):
      stored_username = "admin"
      stored_password = "password123"
      
      if username != stored_username or password != stored_password:
          raise InvalidCredentialsError()
      
      return f"Welcome, {username}!"
  ```  

  こちらは `login` 関数を `InvalidCredentialsError` 例外と一緒に使う方法です。

  ```py
  # failed login attempt
  try:
      message = login("user", "wrongpassword")
  except InvalidCredentialsError as e:
      print(f"Login failed: {e}")
  else:
      print(message)

  # successful login attempt
  try:
      message = login("admin", "password123")
  except InvalidCredentialsError as e:
      # This block is not executed because the login was successful
      print(f"Login failed: {e}")
  else:
      # The else block runs if the 'try' block completes without an exception
      print(message)
  ```

  `raise` 文は `from` キーワードと一緒に使うこともでき、異なるエラー間の関係を示す例外の連鎖を作れます。

  ```py
  def parse_config(filename):
    try:
        with open(filename, 'r') as file:
            data = file.read()
            return int(data)
    except FileNotFoundError:
        raise ValueError('Configuration file is missing') from None
    except ValueError as e:
        raise ValueError('Invalid configuration format') from e

  config = parse_config('config.txt')
  ```

- **オブジェクトのメソッド呼び出し**: オブジェクトに対してメソッドを呼び出し、動作を実行したり情報を取得したりできます。

## Dunder（マジック）メソッド

- **定義**：前後にダブルアンダースコアが付く特別なメソッド（例：`__init__`、`__len__`、`__str__`、`__eq__`）です。Pythonは組み込みの操作で内部的にこれらを使います。

```python
class Book:
    def __init__(self, title, pages):
        self.title = title
        self.pages = pages

    def __len__(self):
        return self.pages

    def __str__(self):
        return f"'{self.title}' has {self.pages} pages"

    def __eq__(self, other):
        return self.pages == other.pages

book1 = Book('Built Wealth Like a Boss', 420)
print(len(book1))        # 420
print(str(book1))        # 'Built Wealth Like a Boss' has 420 pages
```

- **ダンダーメソッドの間接的な呼び出し**：ダンダーメソッドを直接呼び出す必要はありません。特定の操作が行われるとPythonが自動的に呼び出します。これらの操作には以下が含まれます：

  - **加算、減算、乗算、除算などの算術演算**。加算には`__add__()`、減算には`__sub__()`、乗算には`__mul__()`、除算には`__truediv__()`が呼び出されます。
        
  - **連結、繰り返し、フォーマット、文字列への変換などの文字列操作**。連結には`__add__()`、繰り返しには`__mul__()`、フォーマットには`__format__()`、文字列変換には`__str__()`や`__repr__()`などが呼び出されます。
        
  - **等価比較、小なり、大なりなどの比較操作**。等価比較には`__eq__()`、小なりには`__lt__()`、大なりには`__gt__()`などが呼び出されます。
        
  - **イテラブルにしてアイテムを順に取得するイテレーション操作**。イテレーターを返すには`__iter__()`、次のアイテムを取得するには`__next__()`が呼び出されます。

## 実例：ショッピングカート

- **Dunderメソッドを備えたCartクラス**：追加、削除、反復処理、内容の確認を組み込みの動作で行えます。

```python
class Cart:
    def __init__(self):
        self.items = []

    def add(self, item):
        self.items.append(item)

    def remove(self, item):
        if item in self.items:
            self.items.remove(item)
        else:
            print(f'{item} is not in cart')

    def list_items(self):
        return self.items

    def __len__(self):
        return len(self.items)

    def __getitem__(self, index):
        return self.items[index]

    def __contains__(self, item):
        return item in self.items

    def __iter__(self):
        return iter(self.items)

cart = Cart()
cart.add('Laptop')
print(len(cart))        # 1
print('Laptop' in cart) # True
```

## オブジェクト指向プログラミングとは何ですか？

- **オブジェクト指向プログラミング**：開発者がコード内のすべてを現実世界のオブジェクトのように扱うプログラミングスタイルです。一般的にOOPと呼ばれます。コードを効果的に整理・管理するための4つの重要な原則は、**カプセル化**、**継承**、**ポリモーフィズム**、および**抽象化**です。
- **クラス**：オブジェクトを作成するための設計図です。クラスから作成されたすべてのオブジェクトは、データを定義する属性とオブジェクトの振る舞いを決定するメソッドを持っています。

## カプセル化とは何ですか？

- **カプセル化**：オブジェクトの属性とメソッドをひとつの単位にまとめることです。これにより、オブジェクトの内部状態を、ドアのように機能するシンプルな公開メソッドと属性のセットの背後に隠すことができます。そのドアの向こうには、データの変更方法や誰がそれを見られるかを制御するプライベートな属性とメソッドがあります。
- **カプセル化の例**：財布の残高を管理したい場合、入金と出金は許可しますが、財布の残高自体を誰かが勝手に操作することは許したくありません。

```py
class Wallet:
   def __init__(self, balance):
       self.__balance = balance # Private attribute

   def deposit(self, amount):
       if amount > 0:
           self.__balance += amount # Add to the balance safely

   def withdraw(self, amount):
       if 0 < amount <= self.__balance:
           self.__balance -= amount # Remove from the balance safely

account = Wallet(500)
print(account.__balance) # AttributeError: 'Wallet' object has no attribute '__balance'
```

- **属性にシングルアンダースコアとダブルアンダースコアを付ける違い**：属性やメソッドにシングルアンダースコアを付けるのは内部用であることを示す慣例であり、外部からのアクセスを強制的に防ぐものではありません。ダブルアンダースコアを付けると、そのクラスの外部からアクセスされることを実質的に防ぎます。

## ゲッターとセッターとは何ですか？

- **ゲッターとセッター**：クラスの属性へのアクセスや変更方法を制御できるメソッドです。ゲッターで値を取得し、セッターで値を設定します。
- **プロパティ**：ゲッターとセッターをつなぎ、データへのアクセスを可能にします。値の取得、設定、削除時に裏で追加の処理を実行します。
- **なぜメソッドではなくプロパティを使うのか**：プロパティはメソッドよりも読みやすく、コードがすっきりします。丸括弧なしでドット記法を使い、通常の属性のように値にアクセスできます。
- **ゲッターの作成**：ゲッターを作るには`@property`デコレーターを使います。以下は円の半径を取得するゲッターの例です。

```py
class Circle:
    def __init__(self, radius):
        self.radius = radius # Calling the setter

    @property
    def radius(self): # A getter to get the radius
        return self._radius
  
    @property
    def area(self):  # A getter to calculate area
        return 3.14 * (self._radius ** 2)

my_circle = Circle(3)

print(my_circle.radius) # 3
print(my_circle.area) # 28.26
```

`__init__()`内でself.radiusを使うことで、オブジェクト作成時にセッターが実行され、不正な半径の値をすぐに検出できます。

- **セッターの作成**：半径を設定するセッターを作るには、同じ名前の別のメソッドを定義し、その上に`@<property_name>.setter`を付けます。

```py
class Circle:
    def __init__(self, radius):
        self.radius = radius # Calling the setter

    @property
    def radius(self):  # A getter to get the radius
        return self._radius

    @radius.setter
    def radius(self, value):  # A setter to set the radius
        if value <= 0:
            raise ValueError('Radius must be positive')
        self._radius = value

my_circle = Circle(3)
print('Initial radius:', my_circle.radius) # Initial radius: 3

my_circle.radius = 8
print('After modifying the radius:', my_circle.radius) # After modifying the radius: 8
```

- **Pythonがゲッターとセッターを扱う方法**：ゲッターとセッターを定義すると、通常の属性構文でアクセスしたときにPythonが自動的にそれらを呼び出します。

```py
my_circle.radius # This will call the getter
my_circle.radius = 4 # This will call the setter
```

セッター内でプロパティ名自体に代入すると`RecursionError`が発生するため、避けてください。

- **デリーター**：セッターとゲッターで値の設定と取得を制御した後、`deleter`で削除方法も制御できます。デリーターはプロパティに対して`del`文を使ったときにカスタム処理を実行します。デリーターを作るには`@<property_name>.deleter`デコレーターを使います。

```py
  # Deleter
    @radius.deleter
    def radius(self):
        print("Deleting radius...")
        del self._radius
```

## 継承とは何か、そしてそれがコードの再利用をどのように促進するか？

- **継承**：子クラスが親クラスの属性やメソッドを利用する仕組みです。継承はコードの再利用を促進し、明確な階層構造を提供し、すべてを書き直すことなく動作をカスタマイズできます。継承を実装するには、子クラスが親クラスの名前を受け取ります。

```py
class Parent:
    # Parent attributes and methods

class Child(Parent):
    # Child inherits, extends, and/or overrides where necessary
```

- **単一継承と多重継承**：子クラスが単一の親から属性やメソッドを継承する場合、これを**単一継承**と呼びます。子クラスが複数の親から属性やメソッドを継承する場合は、**多重継承**です。構文は以下の通りです。

```py
class Parent:
    # Attributes and methods for Parent

class Child:
    # Attributes and methods for Child

class GrandChild(Parent, Child):
    # GrandChild inherits from both Parent and Child
    # GrandChild can combine or override behavior from each
```

- **`super()` 関数**：クラスがそのメソッドを異なる実装にしている場合やメソッドを拡張している場合に、コードを重複させずに親クラスのメソッドを**呼び出す**ことができる関数です。

## ポリモーフィズムとは何か、そしてそれがコードの再利用をどのように促進するか？

- **ポリモーフィズム**：異なるクラスが同じメソッド名を使いながらも、呼び出されたときにそれぞれのクラスが異なる実装をすることを可能にするOOPの原則です。構文は以下の通りです。

```py
class A:
   def action(self): ...

class B:
   def action(self): ...

class C:
   def action(self): ...

Class().method()  # Works for A, B, or C
```

- **継承に基づくポリモーフィズム**：親クラスがメソッドを設定し、各子クラスがそれを自分用に変えて使います。

## 名前マングリングとは何で、どのように機能するのか？

- **名前マングリング**：Pythonが内部的に、二重アンダースコアで始まる属性の名前を、接頭辞としてアンダースコアとクラス名を追加して変更するプロセスです。これにより、`__attribute` が `_ClassName__attribute` に変わります。
- **名前マングリングの目的**：名前マングリングの主な目的は、継承を使うときに属性やメソッドの誤った上書きを防ぐことです。以下のコードでそれがより理解しやすくなります。

```py
class Parent:
    def __init__(self):
        self.__data = 'Parent data'

class Child(Parent):
    def __init__(self):
        super().__init__()
        self.__data = 'Child data'

c = Child()
print(c.__dict__) # {'_Parent__data': 'Parent data', '_Child__data': 'Child data'}
```

## 抽象化とは何か、そして複雑なシステムを整理するのにどう役立つか？

- **抽象化**：オブジェクトやシステムの複雑な実装の詳細を隠し、必要な機能だけを示すプログラミングの概念です。Pythonや他のプログラミング言語では、抽象化によって複雑なシステムが簡素化され、再利用性が高まります。
- **抽象化の例**：日常生活での抽象化の良い例は、エンジンやブレーキの仕組みを知らなくても、車のハンドル、ペダル、シフトレバーだけを使って運転できることです。
- **Pythonでの抽象化の実装方法**：Pythonは`abc`モジュールを通じて抽象化を実装しています。このモジュールは`ABC`クラス（抽象基底クラス）と`@abstractmethod`デコレーターを提供します。抽象基底クラス（ABC）は、サブクラスが実装しなければならない共通のメソッドやプロパティを定義し、インスタンス化できません。
- **抽象メソッドの定義方法**：抽象メソッドは`@abstractmethod`で定義され、サブクラスで必ずオーバーライドしなければなりません。たとえデフォルトの実装があってもです。抽象化の基本的な構文は次のようになります。

```py
from abc import ABC, abstractmethod

# Define an abstract base class
class AbstractClass(ABC):
    @abstractmethod
    def abstract_method(self):
        pass

# Concrete subclass that implements the abstract method
class ConcreteClassOne(AbstractClass):
    def abstract_method(self):
        print('Implementation in ConcreteClassOne')

# Another concrete subclass
class ConcreteClassTwo(AbstractClass):
    def abstract_method(self):
        print('Implementation in ConcreteClassTwo')
```

## アルゴリズムとビッグO記法

- **アルゴリズム**：問題を解決したりタスクを実行したりするための、あいまいさのない一連の手順です。アルゴリズムは有限のステップ数で終了し、各ステップは正確かつあいまいさがあってはいけません。

- **ビッグO記法**：入力サイズが増加したときのアルゴリズムの最悪の性能や成長率を表します。リソース使用量が入力サイズに対してどのように増加するかに注目し、定数因子や低次の項は無視します。

### よくある時間計算量

- **O(1) - 定数時間**：入力サイズに関係なく、アルゴリズムの実行時間が一定です。

```python
def check_even_or_odd(number):
    if number % 2 == 0:
        return 'Even'
    else:
        return 'Odd'
```

- **O(log n) - 対数時間**：入力が増えても実行時間の増加がゆっくりです。問題のサイズを分数で繰り返し減らすアルゴリズム（例えば二分探索）でよく見られます。

- **O(n) - 線形時間**：実行時間が入力サイズに比例して増加します。

```python
for grade in grades:
    print(grade)
```

- **O(n log n) - 線形対数時間**：マージソートやクイックソートのような効率的なソートアルゴリズムで一般的な時間計算量です。

- **O(n²) - 二次時間**：実行時間が二次的に増加します。ネストされたループでよく見られます。

```python
for i in range(n):
    for j in range(n):
        print("Hello, World!")
```

### 空間計算量

- **O(1) - 定数空間**：入力サイズに関係なく、アルゴリズムが使用するメモリ量が一定です。
- **O(n) - 線形空間**：メモリ使用量が入力サイズに比例して増加します。
- **O(n²) - 二次空間**：メモリ使用量が入力サイズの二次関数的に増加します。

## 問題解決の手法

- **問題の理解**：問題文を何度も読みます。入力、期待される出力、そして入力から出力への変換方法を特定してください。

- **疑似コード**：言語に依存しないアルゴリズムのロジックを高レベルで説明するものです。一般的な文章と `IF`、`ELSE`、`FOR`、`WHILE` のようなプログラミング構造を組み合わせて使います。

```md
GET original_string
SET reversed_string = ""
FOR EACH character IN original_string:
  ADD character TO THE BEGINNING OF reversed_string
DISPLAY reversed_string
```

- **エッジケース**：アルゴリズムが扱うべき範囲の境界にある特定の有効な入力です。常にエッジケースを考慮し、テストしてください。

## 配列

- **静的配列**: 初期化時に決まったサイズを持ちます。要素は隣接したメモリ領域にストアされます。プログラム実行中にサイズを変更できません。

- **動的配列**: プログラム実行中に自動でサイズが増減します。必要に応じてより大きな配列に自動コピーしてリサイズを処理します。

### Pythonのリスト（動的配列）

```python
numbers = [3, 4, 5, 6]

# Access elements
numbers[0]  # 3

# Update elements
numbers[2] = 16

# Add elements
numbers.append(7)
numbers.insert(3, 15)  # Insert at specific index

# Remove elements
numbers.pop(2)  # Remove at specific index
numbers.pop()   # Remove last element
```

### 動的配列の時間計算量

- **アクセス**: O(1)
- **末尾への挿入**: 平均O(1)、リサイズ時はO(n)
- **途中への挿入**: O(n)
- **削除**: 中間はO(n)、末尾はO(1)

## スタック

- **スタック**：後入れ先出し（LIFO）のデータ構造。要素は上部からのみ追加および削除されます。

- **プッシュ操作**：スタックの上部に要素を追加する操作。時間計算量は O(1) です。

- **ポップ操作**：スタックの上部から要素を削除する操作。時間計算量は O(1) です。

```python
# Using Python list as stack
stack = []

# Push operations
stack.append(1)
stack.append(2)
stack.append(3)

# Pop operations
top_element = stack.pop()  # Returns 3
```

## キュー

- **キュー**：先入れ先出し（FIFO）のデータ構造です。要素は後ろに追加され、前から取り出されます。

- **エンキュー操作**：キューの後ろに要素を追加する操作です。時間計算量は O(1) です。

- **デキュー操作**：キューの前から要素を取り出す操作です。時間計算量は O(1) です。

```python
from collections import deque

# Using deque for efficient queue operations
queue = deque()

# Enqueue operations
queue.append(1)
queue.append(2)
queue.append(3)

# Dequeue operations
first_element = queue.popleft()  # Returns 1
```

## 連結リスト

- **連結リスト**：各ノードがデータと次のノードへの参照を持つ線形データ構造です。ノードは鎖のようにつながっています。

### 単方向連結リスト

- **構造**：各ノードはデータと次のノードへの1つの参照を持ちます。
- **走査**：先頭から末尾へ一方向にしか進めません。
- **先頭ノード**：リストの最初のノードで、通常は唯一直接アクセスできるノードです。
- **末尾ノード**：リストの最後のノードで、`None`を指します。

### 操作と時間計算量

- **先頭への挿入**：O(1)
- **末尾への挿入**：O(n) - 末尾まで走査する必要があります
- **途中への挿入**：O(n) - 挿入位置まで走査する必要があります
- **先頭からの削除**：O(1)
- **末尾からの削除**：O(n) - 前のノードを見つけるために走査する必要があります
- **途中からの削除**：O(n) - 削除するノードを見つけるために走査する必要があります

### 双方向連結リスト

- **構造**：各ノードはデータと2つの参照（次のノードと前のノード）を持ちます。
- **走査**：両方向に移動できます。
- **メモリ**：単方向連結リストよりも参照が1つ多いため、より多くのメモリを必要とします。

## ハッシュマップとセット

### マップとハッシュマップ

- **マップ（抽象データ型）**：キーと値のペアの集合を管理します。キーはすべて一意である必要がありますが、値は重複してもかまいません。

- **ハッシュマップ**：ハッシュ手法を使ったマップ抽象データ型の具体的な実装です。キーに対してハッシュ関数を使いハッシュ値を生成し、その値によって基盤となる配列の格納場所が決まります。

### Pythonの辞書（ハッシュマップ）

```python
# Creating dictionaries
my_dictionary = {
    "A": 1,
    "B": 2, 
    "C": 3
}

# Alternative creation
my_dictionary = dict(A=1, B=2, C=3)

# Access and modify
value = my_dictionary["A"]  # 1
my_dictionary["A"] = 4      # Update value
del my_dictionary["A"]      # Remove key-value pair

# Check membership
"C" in my_dictionary

# Get keys, values, items
my_dictionary.keys()
my_dictionary.values()
my_dictionary.items()
```

### ハッシュマップの時間計算量

- **平均ケース**：挿入、取得、削除はO(1)
- **最悪ケース**：多くのハッシュ衝突が発生するとO(n)

### セット

- **セット**：順序なしの一意な要素の集合です。重複は許されず、特定の順序は保持されません。

- **不変要素のみ**：セットには不変のデータ型（数値、文字列、タプル）だけが含まれます。ハッシュ値が一定である必要があるためです。

```python
# Creating sets
numbers = {1, 2, 3, 4}
empty_set = set()  # Must use set(), not {}

# Add and remove elements
numbers.add(5)
numbers.remove(4)      # Raises KeyError if not found
numbers.discard(4)     # No error if not found

# Set operations
set_a = {1, 2, 3, 4}
set_b = {2, 3, 4, 5, 6}

# Union, intersection, difference, symmetric difference
set_a.union(set_b)                    # or set_a | set_b
set_a.intersection(set_b)             # or set_a & set_b
set_a.difference(set_b)               # or set_a - set_b
set_a.symmetric_difference(set_b)     # or set_a ^ set_b

# Subset and superset checks
set_a.issubset(set_b)
set_a.issuperset(set_b)
set_a.isdisjoint(set_b)

# Membership testing
5 in numbers
```

### セットの時間計算量

- **平均ケース**：追加、削除、メンバーシップテストはO(1)
- **最悪ケース**：ハッシュ衝突によりO(n)

## ハッシュの衝突

- **ハッシュ衝突**：異なる2つのキーが同じハッシュ値を生成すると発生します。

- **衝突解決の方法**：
  - **チェーン**：各配列のインデックスが、同じハッシュ値を持つすべての要素を格納する連結リストを指します
  - **オープンアドレッシング**：あらかじめ決められたシーケンスを使って、次に利用可能なインデックスを探します

## 各データ構造を使うタイミング

- **リスト**：順序付きでインデックスによるアクセスが必要で、サイズが事前にわからない場合
- **スタック**：LIFO操作に使う場合（元に戻す機能、式の評価、バックトラッキング）
- **キュー**：FIFO操作に使う場合（タスクスケジューリング、幅優先探索）
- **連結リスト**：先頭での頻繁な挿入・削除があり、サイズが不明でランダムアクセスが不要な場合
- **ハッシュマップ**：高速なキーと値の検索、出現回数のカウント、キャッシュに使う場合
- **セット**：一意性のチェック、数学的な集合演算、重複の削除に使う場合

## Searching Algorithms

検索アルゴリズムは、特定の項目リストの中からターゲットを検索するためのものです。

コンピューターサイエンスでは、知っておくべき検索アルゴリズムが2つあります。それは**線形探索**と**二分探索**のアルゴリズムです。これら2つのアルゴリズムの違いと、それぞれをいつ使うべきかを理解することが重要です。

### Linear Search

- 線形探索は、リストの最初から順に項目を反復し、ターゲットの項目が見つかるまでチェックします。
- ターゲットの項目が見つかれば、その項目がリスト内にあるインデックスを返します。
- ターゲットが見つからなければ、`-1` を返します。これはほとんどのプログラミング言語で**無効なインデックス**を意味します。
- 線形探索はターゲットが見つかるまで全ての項目をチェックするため、大きなリストには効率的ではありません。
- 線形探索の時間計算量は `O(n)` です。なぜなら、リストのサイズに比例して検索にかかる時間が増えるからです。
- 線形探索の空間計算量は `O(1)` です。追加の空間を必要とせずに検索できるためです。

### Binary Search

- 二分探索は、リストを半分に分割し、ターゲットの値がリストの中央にあるかをチェックすることで動作します。
- 二分探索が機能する条件は、リストの項目が昇順に並んでいることです。
- 二分探索は、リストを半分に分割し、ターゲットが存在しない半分を無視するため、大きなリストの検索により効率的なアルゴリズムです。
- ターゲットの項目がリストの中央に見つかれば、その項目のインデックスを返します。
- 項目が見つからなければ、ターゲットがリストの左半分か右半分にあるかをチェックします。
- 残りのリストをさらに半分に分割しながら、ターゲットが見つかるまで続けます。
- 最終的にターゲットがリストに見つからなければ、`-1` を返します。
- 二分探索の時間計算量は `O(log n)` です。なぜなら、リストのサイズに対して対数的に検索時間が増えるからです。
- 二分探索の空間計算量は `O(1)` です。追加の空間を必要とせずに検索できるためです。

### How Linear Search Differs from Binary Search 

- 二分探索は線形探索よりも大きなリストに適しています。
- 線形探索の時間計算量は `O(n)` です。リストのサイズに比例して検索時間が増えるためです。
- 二分探索の時間計算量は `O(log n)` です。リストのサイズに対して対数的に検索時間が増えるためです。

## ソートアルゴリズムと分割統治法

コンピューターサイエンスでは、分割統治法は問題をより小さなサブ問題に分割して解決しやすくする手法です。分割統治法では再帰がよく使われ、マージソートのような効率的なソートアルゴリズムを実装するための強力な戦略です。

### マージソート

- マージソートは分割統治法に従うソートアルゴリズムです。
- リストを再帰的により小さなサブリストに分割し、各サブリストが1つの要素だけになるまで続けます。
- その後、サブリストを繰り返しマージしてソートされた順序に戻します。
- マージソートの時間計算量は`O(n log n)`です。リストが継続的に半分に分割され`(log n)`、その後マージされるため`(O(n))`。
- マージソートの空間計算量は`O(n)`です。これはインプレースソートアルゴリズムではないためです。

## グラフの概要

グラフは、エッジ（接続）でつながれたノード（頂点）の集合です。各ノードは複数の他のノードとつながり、ネットワークを形成します。グラフの種類には以下のものがあります：

- 有向グラフ：エッジに方向があり（一つのノードから別のノードへ）、直線と矢印で表されることが多いです。
- 無向グラフ：エッジに方向がなく、単純な線で表されます。
- 頂点グラフ：各ノードにラベルや識別子が関連付けられています。
- 循環グラフ：サイクル（同じノードで始まり終わる経路）を含みます。
- 非循環グラフ（DAG）：サイクルを含みません。
- エッジラベル付きグラフ：各エッジにラベルがあり、通常は対応するエッジの隣に描かれます。
- 重み付きグラフ：エッジに重み（値）が関連付けられており、算術演算に使用できます。
- 非連結グラフ：エッジでつながれていない2つ以上のノードを含みます。

グラフは地図、ネットワーク、推薦システム、依存関係の解決など、さまざまなアプリケーションで使われています。

## グラフトラバーサル

これはグラフ内のすべてのノードを訪問することを指します。主なアルゴリズムは次の2つです。

- **幅優先探索 (BFS)**
  - キューを使用します。
  - レベルごとに探索します。
  - 重みなしグラフで最短経路を見つけます。

- **深さ優先探索 (DFS)**
  - スタック（または再帰）を使用します。
  - ブランチを完全に探索してから戻ります。
  - サイクル検出や経路探索に役立ちます。

## グラフの表現方法

グラフは主に2つの方法で表現できます。

- **隣接リスト**
  - 各ノードは隣接するノードのリストを持ちます。
  - スペース効率が良く、疎なグラフに適しています。
  - 隣接ノードを繰り返すのが簡単です。

- **隣接行列**
  - 行と列がノードを表す2次元配列です。
  - 大きなグラフではスペースを多く使います。
  - 2つのノード間にエッジがあるかどうかを素早く確認できます。

## Trees

ツリーは、非巡回かつ連結された特別な種類のグラフです。主な特徴は以下の通りです。

- ループやサイクル（開始ノードと終了ノードが同じ経路）がありません。
- 連結されている必要があり（すべてのノードが他のすべてのノードから到達可能です）。

### よくあるツリーの種類

よく使われるツリーの種類は以下の通りです。

- 二分木
  - 各ノードは最大で2つの子を持ち、左の子と右の子があります。

- 二分探索木（BST）
  - すべての左の子が親より小さく、すべての右の子が親より大きい二分木です。

## トライ木

接頭辞木とも呼ばれ、文字列の集合を格納するために使われます。各ノードは1文字を表します。

共通の接頭辞は一度だけ格納されるため、オートコンプリートやスペルチェックのようなタスクに効率的です。

検索や挿入の操作は、文字列の長さLに対して時間計算量O(L)です。

## 優先キュー

優先キューは、各要素に優先度が割り当てられている抽象データ型です。

キューやスタックは挿入順序のみを考慮しますが、優先キューは要素の優先度を考慮します。

標準的なキューはFIFO（先入れ先出し）に従い、スタックはLIFO（後入れ先出し）に従います。しかし、優先キューでは、挿入順序に関係なく、優先度が高い要素が優先的に処理されます。

## ヒープ

ヒープ特性と呼ばれる非常に特定のプロパティを持つ、特殊な木構造のデータ構造です。

ヒープ特性は親ノードと子ノードの関係を決定します。ヒープには2種類あります：

- 最大ヒープ（Max-Heap）
  - 各親ノードの値は、その子ノードの値以上です。
  - 最大の要素がルートにあります。

- 最小ヒープ（Min-Heap）
  - 各親ノードの値は、その子ノードの値以下です。
  - 最小の要素がルートにあります。

### Python `heapq` モジュールの例

```py
import heapq

# Create empty heap
my_heap = []

# Insert elements
heapq.heappush(my_heap, 9)
heapq.heappush(my_heap, 3)
heapq.heappush(my_heap, 5)
print(my_heap) # [3, 9, 5]

# Remove smallest element
print(heapq.heappop(my_heap))  # 3
print(my_heap) # [5, 9]

# Push + Pop in one step
print(heapq.heappushpop(my_heap, 7)) # 5
print(my_heap) # [7, 9]

# Transform list into heap
nums = [5, 7, 3, 1]
heapq.heapify(nums)
```

### 優先度の使用

```py
my_heap = []
heapq.heappush(my_heap, (3, "A"))
heapq.heappush(my_heap, (2, "B"))
heapq.heappush(my_heap, (1, "C"))

# Removes lowest number = highest priority
print(heapq.heappop(my_heap))  # (1, "C")
```

## 動的計画法の紹介

- **定義**：動的計画法は、複雑な問題をより簡単な部分問題に分解し、その結果を保存して重複した計算を避けるアルゴリズムの手法です。
- **重複部分問題**：大きな問題を解く際に同じ小さな問題が何度も現れます。これらの部分問題を繰り返し計算する代わりに、その解を保存します。
- **最適部分構造**：問題の最適解は、その部分問題の最適解を含みます。つまり、小さな部分の最適解を組み合わせて最良の解を構築できます。

## 動的計画法の解法

- **メモ化（トップダウンアプローチ）**: メモ化は、コストの高い関数呼び出しの結果を保存し、同じ入力が再度現れたときにキャッシュされた結果を返します。

```py
def climb_stairs_memo(n, memo={}):
    """Dynamic programming with memoization"""
    # Check if we've already calculated this value
    if n in memo:
        return memo[n]  # Return cached result - O(1) lookup!
    
    # Base cases
    if n <= 2:
        return n
    
    # Calculate once and store in memo for future use
    memo[n] = climb_stairs_memo(n-1, memo) + climb_stairs_memo(n-2, memo)
    return memo[n]
```

- **タビュレーション（ボトムアップアプローチ）**: タビュレーションは、下から順に解を構築し、部分問題の解をテーブルに埋めていきます。

```py
def climb_stairs_tabulation(n):
    """Dynamic programming with tabulation"""
    if n <= 2:
        return n
    
    # Create array to store results for all steps from 0 to n
    dp = [0] * (n + 1)
    dp[1] = 1  # 1 way to reach step 1
    dp[2] = 2  # 2 ways to reach step 2
    
    # Build up the solution iteratively
    for i in range(3, n + 1):
        # Ways to reach step i = ways to reach (i-1) + ways to reach (i-2)
        dp[i] = dp[i-1] + dp[i-2]
    
    return dp[n]
```

## 動的計画法を使った実際のアプリケーション

- **ルート最適化**：GPSシステムは動的計画法のアルゴリズムを使って、場所間の最短経路を見つけます。
- **テキスト処理**：スペルチェッカーやオートコンプリート機能は、単語間の編集距離を計算するために動的計画法をよく利用します。
- **金融モデリング**：投資戦略やポートフォリオの最適化には、動的計画法の手法が頻繁に使われます。
- **資源配分**：ナップサック問題やその変種は、スケジューリング、予算編成、資源管理に現れます。

## 動的計画法を使うべき場合

動的計画法は次のような場合に検討してください。

- 問題が重複する部分問題に分解できる場合。
- 問題が最適部分構造を持つ場合。
- 単純な再帰的解法では計算が繰り返される場合。
- 時間計算量を優先して空間計算量を犠牲にしてもよい場合。
