## Vergleiche und die `null` und `undefined` Datentypen

- **Vergleiche und `undefined`**: Eine Variable ist `undefined`, wenn sie deklariert wurde, aber noch keinen Wert zugewiesen bekommen hat. Es ist der Standardwert von nicht initialisierten Variablen und Funktionsparametern, denen kein Argument übergeben wurde. `undefined` wird in numerischen Kontexten zu `NaN` konvertiert, was dazu führt, dass alle numerischen Vergleiche mit `undefined` `false` zurückgeben.

:::interactive_editor

```js
console.log(undefined < 0); // false (NaN < 0 is false)
console.log(undefined >= 0); // false (NaN >= 0 is false)
```

:::

- **Vergleiche und `null`**: Der `null` Typ stellt das absichtliche Fehlen eines Werts dar. `null` wird in numerischen Kontexten zu `0` konvertiert, was zu unerwartetem Verhalten bei numerischen Vergleichen führen kann:

:::interactive_editor

```js
console.log(null < 0); // false (0 < 0 is false)
console.log(null >= 0); // true (0 >= 0 is true)
```

:::

- Beim Verwenden des Gleichheitsoperators (`==`) sind `null` und `undefined` nur zueinander und zu sich selbst gleich:

:::interactive_editor

```js
console.log(null == undefined); // true
console.log(null == 0); // false
console.log(undefined == NaN); // false
```

:::

- Beim Verwenden des strikten Gleichheitsoperators (`===`), der sowohl Wert als auch Typ überprüft, ohne eine Typumwandlung durchzuführen, sind `null` und `undefined` nicht gleich:

:::interactive_editor

```js
console.log(null === undefined); // false
```

:::

## `switch` Anweisungen

- **Definition**: Eine `switch` Anweisung wertet einen Ausdruck aus und vergleicht dessen Wert mit einer Reihe von `case` Klauseln. Wenn eine Übereinstimmung gefunden wird, wird der mit diesem Fall verknüpfte Codeblock ausgeführt. Eine `break` Anweisung sollte am Ende jedes Falls stehen, um dessen Ausführung zu beenden und mit dem nächsten fortzufahren. Der `default` Fall ist ein optionaler Fall und wird nur ausgeführt, wenn keiner der anderen Fälle zutrifft. Der `default` Fall steht am Ende einer `switch` Anweisung.

:::interactive_editor

```js
const dayOfWeek = 3; 

switch (dayOfWeek) {
  case 1:
    console.log("It's Monday! Time to start the week strong.");
    break;
  case 2:
    console.log("It's Tuesday! Keep the momentum going.");
    break;
  case 3:
    console.log("It's Wednesday! We're halfway there.");
    break;
  case 4:
    console.log("It's Thursday! Almost the weekend.");
    break;
  case 5:
    console.log("It's Friday! The weekend is near.");
    break;
  case 6:
    console.log("It's Saturday! Enjoy your weekend.");
    break;
  case 7:
    console.log("It's Sunday! Rest and recharge.");
    break;
  default:
    console.log("Invalid day! Please enter a number between 1 and 7.");
}
```

:::
