# --interactive--

Bitweise Operatoren in JavaScript sind spezielle Operatoren, die auf den binären Darstellungen von Zahlen arbeiten. Um bitweise Operatoren zu verstehen, müssen wir zuerst das Konzept von Bits und binären Zahlen erfassen. In der Informatik ist ein Bit die kleinste Informationseinheit. Es kann nur zwei Werte haben: `0` oder `1`. Binär ist ein Zahlensystem, das nur diese beiden Ziffern verwendet, um alle Zahlen darzustellen.

Zum Beispiel ist die binäre Darstellung der Dezimalzahl `10` `1010`. In diesem System stellt jede Ziffer eine Potenz von `2` dar, beginnend mit der ganz rechten Ziffer und steigend, wenn wir nach links gehen.

In der obigen Tabelle zeigt die erste Zeile die binäre Zahl `1010`, die zweite Zeile zeigt die Potenz von `2`, die jede binäre Position darstellt, und die dritte Zeile zeigt das Ergebnis jeder Multiplikation. Wenn Sie alle Werte in der dritten Zeile addieren, ergeben sie insgesamt `10`.

Kommen wir nun zu den bitweisen Operatoren. Diese Operatoren führen Operationen auf den binären Darstellungen von Zahlen aus. JavaScript bietet mehrere bitweise Operatoren, darunter AND (`&`), OR (`|`), XOR (`^`), NOT (`~`), Linksverschiebung (`<<`) und Rechtsverschiebung (`>>`).

Der bitweise AND-Operator (`&`) gibt an jeder Bitposition eine `1` zurück, für die die entsprechenden Bits beider Operanden `1` sind. Hier ein Beispiel:

:::interactive_editor

```js
let a = 5;  // Binary: 101
let b = 3;  // Binary: 011
console.log(a & b);  // 1 (Binary: 001)
```

:::

In diesem Beispiel führen wir eine bitweise AND-Operation auf `5` (`101` in binär) und `3` (`011` in binär) durch. Das Ergebnis ist `1` (`001` in binär), weil nur das ganz rechte Bit in beiden Zahlen `1` ist.

Der bitweise OR-Operator (`|`) gibt an jeder Bitposition eine `1` zurück, für die die entsprechenden Bits eines oder beider Operanden `1` sind. Zum Beispiel:

:::interactive_editor

```js
let a = 5;  // Binary: 101
let b = 3;  // Binary: 011
console.log(a | b);  // 7 (Binary: 111)
```

:::

Hier ist das Ergebnis `7` (`111` in binär), weil mindestens eines der Bits an jeder Position `1` ist.

Der bitweise XOR-Operator (`^`) gibt an jeder Bitposition eine `1` zurück, für die die entsprechenden Bits eines, aber nicht beider, Operanden `1` sind. Zum Beispiel:

:::interactive_editor

```js
let a = 5;  // Binary: 101
let b = 3;  // Binary: 011
console.log(a ^ b);  // 6 (Binary: 110)
```

:::

Das Ergebnis ist `6` (`110` in binär), weil das erste und zweite Bit von rechts in den beiden Zahlen unterschiedlich sind.

Der bitweise NOT-Operator (`~`) kehrt alle Bits seines Operanden um. Zum Beispiel:

:::interactive_editor

```js
let a = 5;  // Binary: 101
console.log(~a);  // -6
```

:::

Das mag überraschend erscheinen, aber das liegt daran, wie negative Zahlen im Binärsystem mit Zweierkomplement dargestellt werden.

Der Linksverschiebungsoperator (`<<`) verschiebt alle Bits um eine bestimmte Anzahl von Positionen nach links. Zum Beispiel:

:::interactive_editor

```js
let a = 5;  // Binary: 101
console.log(a << 1);  // 10 (Binary: 1010)
```

:::

Hier werden alle Bits um eine Position nach links verschoben, was effektiv die Zahl mit `2` multipliziert.

Der Rechtsverschiebungsoperator (`>>`) verschiebt alle Bits nach rechts. Zum Beispiel:

:::interactive_editor

```js
let a = 5;  // Binary: 101
console.log(a >> 1);  // 2 (Binary: 10)
```

:::

Hier werden alle Bits um eine Position nach rechts verschoben, was effektiv die Zahl durch `2` teilt und abrundet.

Bitweise Operatoren werden oft in der Low-Level-Programmierung und Kryptografie verwendet. Auch wenn sie im alltäglichen JavaScript-Programmieren nicht so häufig vorkommen, kann das Verständnis dieser Operatoren bei bestimmten spezialisierten Aufgaben hilfreich sein und Ihr Verständnis dafür vertiefen, wie Computer auf grundlegender Ebene arbeiten.
