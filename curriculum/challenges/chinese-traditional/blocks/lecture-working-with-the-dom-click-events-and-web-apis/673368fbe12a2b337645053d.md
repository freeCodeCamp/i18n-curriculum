# --description--

`Event` 物件是當使用者以某種方式與你的網頁交談時觸發的有效負載。這些交談可以是從點擊按鈕、聚焦輸入欄位到搖動他們的行動裝置的任何行為。

像所有 JavaScript 物件一樣，`Event` 物件有許多可能有用的屬性。可用的屬性取決於觸發此有效負載的事件。

所有 `Event` 物件都會有 `type` 屬性。此屬性揭示觸發有效載荷的事件型別，例如 `"keydown"` 或 `"click"`。這些值會對應到你可能傳遞給 `addEventListener()` 的相同值，在那裡你可以捕捉並使用 `Event` 物件。

`Event` 物件將永遠具有 `target` 屬性。`target` 屬性是對觸發該事件的任何物件的引用。最常見的是某種 `HTMLElement` 物件，或是 `Document` 或 `Window` 物件。但它也可以是更特定的東西，例如 `AudioContext`。

事件也有方法，這些方法是作為物件屬性公開的函式（程式）。一個常用的方法是 `preventDefault()`，當呼叫時會阻止該事件的預設行為。

如果你想自己控制代碼表單提交，例如，你可能會呼叫 `preventDefault()` 以防止瀏覽器嘗試將表單資料以 `POST` 請求方式提交。你將在未來的課程中學習更多關於 `POST` 請求的內容。

你也很可能會遇到 `stopPropagation()` 方法。這個方法可以防止事件向上冒泡或傳播到父元素。你將在未來的課程中學到更多關於這個意思的內容。

還有大量特定於某些 `Event` 物件實作的屬性。<dfn>例如</dfn>，`FetchEvent` 會有一個 request 屬性，用來包含觸發該事件的請求。 

如果你不確定有哪些屬性可用，你可以記錄相關的 `Event` 物件，甚至檢查文件。
