# --description--

Графи — це дуже потужні структури даних, які складаються з множини вузлів, також відомих як вершини, та ребер, що їх з’єднують.

Існує два загальні способи реалізації графів у вашому коді:

* матриці суміжності
    
* списки суміжності
    

Давайте розглянемо їх детальніше, а також їхні переваги та обмеження.

## Матриці суміжності

Почнемо з матриць суміжності.

Матриця суміжності — це двовимірний список, у якому рядки та стовпці представляють вершини графа.

Значення в матриці представляють ребра або зв’язки між вузлами.

Наприклад, якщо у вас є матриця, збережена у змінній `matrix`, значення, що зберігається в `matrix[i][j]`, де `i` — це рядок, а `j` — стовпець, представляє ребро або зв’язок між вузлом `i` та вузлом `j`.

Значення можуть мати різне значення залежно від того, чи є граф зваженим чи незваженим:

* Якщо граф незважений, значення `1` означає, що існує ребро, яке з’єднує ці вузли, тоді як значення `0` означає, що між ними немає ребра.
    
* Якщо граф зважений, значення представлятиме вагу ребра, що з’єднує вузли.
    

Однією з великих переваг використання матриці суміжності є те, що перевірка наявності ребра між двома вузлами займає сталий час `O(1)`. Це тому, що програмі потрібно лише знайти конкретне значення у двовимірному списку.

Однак ця ефективність у пошуку ребер має свою ціну. Матриці суміжності потребують великого квадратичного обсягу пам’яті `O(V²)`, де `V` — це кількість вузлів у графі.

Це неефективно для розріджених графів, тобто графів, які мають лише кілька ребер. Чому? Тому що якщо граф розріджений, ви зберігатимете багато 0 у матриці, щоб позначити відсутність ребер між вузлами, і ці 0 все одно займатимуть місце в пам’яті.

Матриці суміжності також неефективні для пошуку сусідів вузла, оскільки програмі доводиться ітерувати весь рядок або стовпець, щоб знайти 0 та 1, які представляють ребра. У найгіршому випадку цей процес може зайняти час O(V), де V — кількість вузлів у графі.

Давайте подивимося приклад матриці суміжності для цього конкретного графа:

<img src="https://cdn.freecodecamp.org/curriculum/lecture-transcripts/how-do-matrices-and-adjacency-lists-work-1.png" alt="Граф із чотирма вузлами, позначеними як A, B, C та D. Вузол A з’єднаний з B, C та D. Вузол B з’єднаний з A та D. Вузол C з’єднаний з A. Вузол D з’єднаний з A та B.">

У матриці суміжності:

* Кожен рядок представляє вузол. Перший рядок — це вузол A, другий рядок — вузол B і так далі.
    
* Кожен стовпець також представляє вузол.
    
* Кожне значення в матриці показує, чи існує ребро між парою вузлів. Значення 0 означає, що ребра немає, а значення 1 означає, що ребро є.
    

Значення на діагоналі показують, чи має кожен вузол петлю на самому собі, тобто ребро, що з’єднує вузол із самим собою. У нашому прикладі всі вони 0, бо граф не має таких петель.

Це візуальне представлення матриці суміжності, щоб показати, як рядки та стовпці відповідають відповідним вузлам.

Наприклад, перший рядок — це `[0, 1, 1, 1]`, бо вузол A має ребра, що з’єднують його з вузлами B, C та D:

```markdown
#      A  B  C  D
# A   [0, 1, 1, 1],
# B   [1, 0, 0, 1],
# C   [1, 0, 0, 0],
# D   [1, 1, 0, 0]
```

І це та сама матриця суміжності, але реалізована у коді Python:

```python
adjacency_matrix = [
    [0, 1, 1, 1],  # The neighbors of A are B, C, and D
    [1, 0, 0, 1],  # The neighbors of B are A and D
    [1, 0, 0, 0],  # The only neighbor of C is A
    [1, 1, 0, 0]   # The neighbors of D are A and B
]
```

## Списки суміжності

Інший поширений спосіб представлення графів — це використання списків суміжності.

Список суміжності — це масив або словник, який зберігає всіх сусідів кожного вузла.

Існує два способи реалізації списків суміжності:

* Як масив, де кожен індекс представляє вузол, а список, збережений за цим індексом, містить його сусідів.
    
* Як словник, де кожен ключ представляє вузол, а значення, пов’язане з цим ключем (список), містить його сусідів.
    

Списки суміжності є більш ефективними за матриці суміжності з точки зору вимог до пам’яті. Вони мають складність простору `O(V + E)`, де `V` — кількість вершин (вузлів), а `E` — кількість ребер.

Також вони ефективні для пошуку всіх сусідніх вузлів, оскільки ця операція вимагає лише доступу до списку, пов’язаного з вузлом.

Однак є і компроміс.

Списки суміжності менш ефективні за матриці суміжності для визначення, чи існує ребро між двома вузлами.

Процес пошуку може зайняти `O(V)` часу у найгіршому випадку, оскільки доведеться шукати у дуже довгому списку сусідів, якщо вузол з’єднаний з усіма іншими вузлами графа.

Ось приклад списку суміжності для цього графа:

<img src="https://cdn.freecodecamp.org/curriculum/lecture-transcripts/how-do-matrices-and-adjacency-lists-work-1.png" alt="Граф із чотирма вузлами, позначеними як A, B, C та D. Вузол A з’єднаний з B, C та D. Вузол B з’єднаний з A та D. Вузол C з’єднаний з A. Вузол D з’єднаний з A та B. Це те саме зображення, що й раніше.">

Цей список суміжності реалізований як словник. Кожен ключ у словнику представляє вузол, а значення, пов’язане з цим ключем, — це список усіх сусідів відповідного вузла:

```python
adjacency_list = {
    'A': ['B', 'C', 'D'],
    'B': ['A', 'D'],
    'C': ['A'],
    'D': ['A', 'B']
}
```

Альтернативно, ми могли б реалізувати його як двовимірний список, де кожен індекс представляє вузол. Наприклад, індекс 0 представляє вузол A, індекс 1 — вузол B і так далі:

```python
adjacency_list = [
    ['B', 'C', 'D'],  # Neighbors of A (index 0)
    ['A', 'D'],       # Neighbors of B (index 1)
    ['A'],            # Neighbors of C (index 2)
    ['A', 'B']        # Neighbors of D (index 3)
]
```

Зверніть увагу, що навіть якщо цей двовимірний список може виглядати схожим на матрицю суміжності, вони досить різні.

* Матриця суміжності зберігає 0, 1 або інші значення, які представляють ребра або ваги ребер у графі.
    
* Список суміжності зберігає фактичний список усіх сусідів кожного вузла.
    

Це дуже важлива різниця, з якою ви повинні бути знайомі.

І матриці суміжності, і списки суміжності дуже важливі для реалізації графів. Вибір між ними залежить від розміру графа та того, як ви плануєте використовувати дані. Матриці суміжності корисні для щільних графів із багатьма ребрами, тоді як списки суміжності зазвичай є кращим вибором для реальних сценаріїв, де розріджені графи зустрічаються частіше.
