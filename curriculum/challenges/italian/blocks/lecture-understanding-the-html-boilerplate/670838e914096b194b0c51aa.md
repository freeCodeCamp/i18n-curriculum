# --description--

Impariamo cos’è il boilerplate HTML.

Cos’è il boilerplate HTML, ti chiedi? È come un modello già pronto per le tue pagine web. Pensalo come le fondamenta di una casa. Un boilerplate include la struttura di base e gli elementi essenziali di cui ogni documento HTML ha bisogno. Ti fa risparmiare tempo e aiuta a garantire che le tue pagine siano impostate correttamente. Ecco un esempio:

```html
<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="utf-8" />
    <meta
       name="viewport"
       content="width=device-width, initial-scale=1.0" />
    <title>freeCodeCamp</title>
    <link rel="stylesheet" href="./styles.css" />
  </head>
  <body>
  </body>
</html>
```

Analizziamo le parti principali di questo boilerplate. Prima di tutto, c’è la dichiarazione `DOCTYPE`:

```html
<!DOCTYPE html>
```

Indica ai browser quale versione di HTML stai usando. Poi, c’è il tag `html`:

```html
<!DOCTYPE html>
<html lang="en">
  <!--All other elements go inside here-->
</html>
```

Questo racchiude tutto il tuo contenuto e può specificare la lingua della tua pagina. All’interno del tag `html` trovi due sezioni principali: una `head` e una `body`:

```html
<!DOCTYPE html>
<html lang="en">
  <head>
    <!--Important metadata goes here-->
  </head>
  <body>
    <!--Headings, paragraphs, images, etc. go inside here-->
  </body>
</html>
```

La sezione `head` contiene informazioni importanti dietro le quinte:

```html
<head>
  <meta charset="UTF-8" />
  <meta name="viewport" content="width=device-width, initial-scale=1.0" />
  <title>Document Title Goes Here</title>
  <link rel="stylesheet" href="./styles.css" />
</head>
```

I metadati del tuo sito, contenuti negli elementi `meta`, includono dettagli come la codifica dei caratteri e come siti come Twitter dovrebbero mostrare l’anteprima del collegamento alla tua pagina. Il titolo del tuo sito, che si trova nell’elemento `title`, determina il testo che appare nella scheda o nella finestra del browser. Infine, di solito colleghi i fogli di stile esterni della tua pagina nella sezione `head` usando elementi `link`.

La sezione `body` è dove va tutto il tuo contenuto:

```html
<body>
  <h1>I am a main title</h1>
  <p>Example paragraph text</p>
</body>
```

Ora, perché un boilerplate è importante? Garantisce che le tue pagine siano strutturate correttamente e funzionino bene su diversi browser. Usare un boilerplate ti aiuta a evitare errori comuni e a seguire le buone pratiche. È un ottimo punto di partenza per qualsiasi progetto web. Ricorda, puoi personalizzare il tuo boilerplate per adattarlo alle tue esigenze. Man mano che acquisisci esperienza, potresti aggiungere i tuoi elementi preferiti o tag `meta`. Continuando a migliorare il tuo boilerplate personale, scoprirai che ti fa risparmiare tempo quando inizi nuovi progetti.

La prossima volta che inizi un nuovo file HTML, considera di usare un boilerplate. Ti darà sicuramente una base solida su cui costruire.
