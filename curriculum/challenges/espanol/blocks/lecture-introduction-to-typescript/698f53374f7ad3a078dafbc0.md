# --description--

En esta lección, aprenderás cómo agregar tipos para objetos y arreglos.

Veamos primero los arrays.

Aquí tienes un ejemplo de un arreglo de lenguajes de programación: 

```js
const programmingLanguages = ["Java", "C++", "Python"];
```

Si quieres tipar explícitamente ese arreglo como un arreglo de cadenas, entonces puedes tiparlo como `string[]`:

```ts
const programmingLanguages:string[] = ["Java", "C++", "Python"];
```

Como aprendiste antes, TypeScript será lo suficientemente inteligente para inferir el tipo como un arreglo de cadenas si solo escribes esto:

```js
const programmingLanguages = ["Java", "C++", "Python"];
```

Pero escribirlo explícitamente ayudará en situaciones si intentas agregar otro tipo al arreglo así:

```ts
// Type 'number' is not assignable to type 'string'.(2322)
const programmingLanguages:string[] = ["Java", 9, "Python"];
```

Otra forma de escribir arreglos es usar esta sintaxis:

```ts
const programmingLanguages:Array<string> = ["Java", "C++", "Python"];
```

Fundamentalmente, estas dos sintaxis son iguales, y la elección entre las dos es generalmente una cuestión de preferencia.

Ahora, echemos un vistazo a la escritura de objetos.

Aquí hay un ejemplo de un literal de objeto:

```js
const developer = {
  firstName: "Jessica",
  isEmployed: true
}
```

Para tipar explícitamente las propiedades del objeto, puedes usar esta sintaxis:

```ts
const developer: { firstName: string, isEmployed: boolean } = {
  firstName: "Jessica",
  isEmployed: true
}
```

Si intentas agregar o eliminar propiedades del objeto, entonces obtendrás un error como este:

```ts
// Property 'isEmployed' is missing in type '{ firstName: string; }' but required in type '{ firstName: string; isEmployed: boolean; }'
const developer: { firstName: string, isEmployed: boolean } = {
  firstName: "Jessica",
}
```

Puedes hacer que las propiedades del objeto sean opcionales usando el `?` después del nombre de la propiedad así:

```ts
const developer: { firstName: string, isEmployed?: boolean } = {
  firstName: "Jessica",
}
```

TypeScript ya no mostrará un error porque la propiedad `isEmployed` ha sido marcada como opcional.

Hasta ahora, todos los ejemplos de objetos han tenido propiedades restrictivas. Eso significa que enumeramos explícitamente cada propiedad que el objeto puede tener:

```ts
const developer: { firstName: string, isEmployed: boolean } = {
  firstName: "Jessica",
  isEmployed: true
}
```

¿Pero qué pasa si no conoces todos los nombres de las propiedades de antemano? ¿Y si quieres un objeto que pueda tener cualquier número de propiedades pero todos los valores deben seguir un tipo específico?

Bueno, hay dos maneras de lograr esto. La primera es usar un `Record`, que es un tipo genérico de utilidad incorporado en TypeScript.

Esta es la sintáxis básica:

```ts
Record<Keys, ValueType>
```

El primer tipo representa el tipo de las claves mientras que el segundo tipo representa el tipo de los valores.

Aquí tienes un ejemplo de cómo usar un `Record` para un objeto `userRoles`:

```ts
const userRoles: Record<string, string> = {
  admin: "full-access",
  editor: "limited-access",
  viewer: "read-only"
};
```

Este objeto `userRoles` puede tener cualquier clave de tipo cadena, y todos los valores deben ser cadenas. Si intentas asignar un valor que no sea cadena, obtendrás un error:

```ts
// Type 'number' is not assignable to type 'string'
const userRoles: Record<string, string> = {
  admin: 1
};
```

La segunda forma de escribir la misma idea es con una firma de índice:

```ts
const obj: { [key: string]: string } = {};
```

La sintaxis anterior dice que cualquier propiedad con una clave de cadena debe tener un valor de cadena. Aquí hay un ejemplo para un objeto `settings`:

```ts
const settings: { [key: string]: string } = {
  theme: "dark",
  language: "en",
  layout: "grid"
};
```

Al igual que antes, si tienes tipos de valores incorrectos, TypeScript mostrará un mensaje de error:

```ts
// Type 'boolean' is not assignable to type 'string'
const settings: { [key: string]: string } = {
  darkMode: true
};
```

Tanto `Record` como las firmas de índice son fundamentalmente lo mismo. Así que elegir entre ellas es principalmente una cuestión de preferencia.
