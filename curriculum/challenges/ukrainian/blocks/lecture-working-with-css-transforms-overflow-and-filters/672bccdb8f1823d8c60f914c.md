# --interactive--

CSS reset — це таблиця стилів, яка видаляє всі або деякі стилі за замовчуванням, які веббраузери застосовують до HTML-елементів. Наприклад, ви могли помітити, що певні елементи, як-от абзаци та заголовки, вже мають поля за замовчуванням, навіть якщо ви явно не визначали їх у своїх кастомних таблицях стилів.

Ви також побачите це з різними властивостями у великій кількості HTML-елементів. Різні браузери можуть застосовувати різні стилі за замовчуванням до HTML-елементів. Стилі за замовчуванням у Google Chrome можуть відрізнятися від тих, що у Mozilla Firefox або Microsoft Edge. Це може призвести до непослідовності стилів між браузерами, чого слід уникати якомога більше.

Щоб це вирішити, CSS reset видаляє стилі за замовчуванням. Видаляючи всі або деякі стилі за замовчуванням, ви отримуєте послідовну базу для свого дизайну і мінімізуєте потенційні розбіжності між браузерами та пристроями. Видалення стилів за замовчуванням також полегшує процес стилізації, бо ви бачитимете лише ті стилі, які явно застосували у своїх кастомних таблицях стилів. Існує два основні підходи до CSS reset: ви можете або створити кастомний CSS reset, або використати сторонні CSS reset.

Кастомні CSS reset — це таблиці стилів, які ви створюєте з нуля, щоб відповідати потребам вашого проєкту. Таким чином ви контролюєте конкретні стилі, які будуть скинуті, з великою гнучкістю. Проте вам доведеться витратити час на розробку та підтримку таких таблиць стилів. Ось приклад дуже загального CSS правила для скидання полів і відступів усіх HTML-елементів:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<h1>Example Heading</h1>
<p>This is a paragraph.</p>
```

```css
* {
  margin: 0;
  padding: 0;
}
```

:::

Зазвичай це пишуть на початку таблиці стилів CSS. Селектор зірочка — це універсальний селектор, який відповідає всім HTML-елементам, тому вони матимуть поле за замовчуванням нуль і відступ за замовчуванням нуль з усіх чотирьох сторін. Це дає вам початкову точку, а потім ви можете налаштовувати їх у більш конкретних CSS селекторах далі в таблиці стилів.

Ви можете використати цей підхід, щоб вибрати будь-який HTML-елемент і скинути його стилі за замовчуванням. Просто створіть кастомну таблицю стилів і використайте відповідні CSS селектори, щоб підібрати елементи і встановити стилі.

Але це може бути трудомістким процесом. Якщо хочете заощадити час, ви також можете використати сторонній CSS reset. Ці таблиці стилів уже готові, тож ви можете просто скачати їх і додати безпосередньо до свого проєкту. Чудовим прикладом стороннього CSS reset є `Normalize.css`. Ця таблиця стилів нормалізує стилі для великої кількості HTML-елементів, при цьому зберігаючи деякі корисні стилі за замовчуванням, особливо ті, що важливі для доступності. Вона також виправляє загальні помилки та стилістичні розбіжності.

Ще один варіант — `sanitize.css`. Це CSS бібліотека, яку можна використати, щоб забезпечити послідовність стилів за замовчуванням у всіх основних сучасних браузерах. Ця бібліотека розробляється разом із `Normalize.css`, тож вони розвиваються синхронно. Вона також має окремі таблиці стилів, які можна скачати для конкретних цілей, наприклад, для нормалізації форм і типографіки.

Існує багато варіантів, але слід обирати ті, що найкраще відповідають потребам вашого проєкту. Ви також можете поєднувати обидва підходи, використовуючи сторонні CSS reset разом із кастомними.

Працюючи з CSS reset, важливо також враховувати доступність. Ваш вебзастосунок має бути доступним для всіх. Тож не слід скидати стилі, які можуть бути корисними для читачів екрана або інших допоміжних технологій. Також варто враховувати вплив додаткових таблиць стилів на продуктивність застосунку, адже їх потрібно скачати перед застосуванням кастомних стилів.

Видаляючи стилі за замовчуванням, CSS reset дає вам чисту початкову точку для реалізації вашого дизайну. Це призводить до більш однорідного і послідовного користувацького досвіду між браузерами і пристроями.
