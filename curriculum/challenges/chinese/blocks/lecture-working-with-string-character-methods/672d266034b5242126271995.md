# --interactive--

在编程中，了解字符如何表示为数字是基础。 这就是 ASCII 的作用所在。 ASCII 是 American Standard Code for Information Interchange 的缩写，是一种计算机用于表示文本的字符编码标准。 它为每个字符分配一个数值，机器可以普遍识别。

在本课中，我们将探讨什么是 ASCII，它是如何工作的，以及像 `charCodeAt()` 和 `fromCharCode()` 这样的 JavaScript 方法如何与字符编码相关。虽然 JavaScript 字串在内部使用 Unicode（UTF-16），但 ASCII 值匹配前 128 个 Unicode 字符，这就是基于 ASCII 的示例在 JavaScript 中有效的原因。

ASCII 是一种将字母、数字和符号等字符编码成数值的系统。 每个字符都对应一个特定的数字。

例如，大写字母 `A` 在 ASCII 中对应数字 `65`，而小写字母 `a` 对应 `97`。 通过这种编码，计算机可以存储和处理文本。

ASCII 标准包含 128 个字符，包括：

- 大写和小写英文字母（A-Z、a-z）。
- 数字（0-9）。
- 常见标点符号和特殊字符（!、@、# 等）。
- 控制字符（如换行符和制表符）。

在 JavaScript 中，你可以使用 `charCodeAt()` 方法访问字符的数字代码。该方法返回指定索引处字符的 UTF-16 代码单元。对于前 128 个字符，该值与 ASCII 代码匹配。

让我们来看一个例子：

:::interactive_editor

```js
let letter = "A";
console.log(letter.charCodeAt(0));  // 65
```

:::

在此示例中，`A` 是字串的第一个字符，调用 `charCodeAt(0)` 返回它的数字代码（对于基本拉丁字符，该代码与其 ASCII 值匹配），即 `65`。

你也可以使用此方法对其他字符进行查找它们的数字代码值：

:::interactive_editor

```js
let symbol = "!";
console.log(symbol.charCodeAt(0));  // 33
```

:::

这里，感叹号 `!` 的数字代码被返回为 `33`（这与其 ASCII 值匹配）。

虽然 `charCodeAt()` 帮助你获取字符的数字代码，`fromCharCode()` 方法则允许你执行相反的操作：将 UTF-16 代码单元（对于基本字符与 ASCII 匹配）转换为对应的字符。

让我们来看看它的实际作用：

:::interactive_editor

```js
let char = String.fromCharCode(65);
console.log(char);  //  A
```

:::

在此示例中，`fromCharCode(65)` 将数字代码 `65`（它匹配 `A` 的 ASCII 值）转换回字符 `A`。

另一个例子是将数字 `97` 转换为相应的小写字母：

:::interactive_editor

```js
let char = String.fromCharCode(97);
console.log(char);  // a
```

:::

当你需要基于字符的数字代码值操作或比较字符时，这些方法特别有用。

例如，你可以使用 `charCodeAt()` 通过比较字符的 ASCII 值来检查该字符是大写、小写还是数字。

而 `fromCharCode()` 可用于根据 ASCII 码动态生成字符。
