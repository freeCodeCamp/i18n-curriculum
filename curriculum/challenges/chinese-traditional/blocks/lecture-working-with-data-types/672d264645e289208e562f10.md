# --description--

JavaScript 是一種動態型別語言，這表示你在宣告變數時不需要指定變數的資料型別。相反地，型別會根據在程式執行時指定給變數的值來決定。這讓你可以在整個程式中變更變數的型別。

讓我們來看一個範例：

```js
let example = "Hello";
example = 42;
```

在這個範例中，我們有一個名為 `example` 的變數，其資料型別為字串。但接著我們將值更新為數字。

動態類型的彈性使得 JavaScript 更寬容且易於快速撰寫腳本，但它也可能會為你的程式引入較難發現的錯誤，尤其是當你的程式規模變大時。

在像 C# 或 C++ 這類靜態型別語言中，你必須在創建變數時宣告該變數的資料型別，且該型別無法更改。

例如，如果你將變數宣告為 `integer`，你只能為它指定整數值。如果你嘗試指定不同的型別，程式將會發出錯誤。

以下是 C# 語言的範例：

```csharp
int data = 42; // data must always be an integer
data = "Hello"; // This would cause an error in C#
```

動態型別與靜態型別的差異在於你的程式碼的彈性與安全性。動態型別語言提供彈性，但代價是可能發生執行期錯誤。

靜態型別語言強制執行較嚴格的規則，可以防止某些錯誤，但它們需要更多的前置宣告，並且在變更型別時提供較少的彈性。

這是另一個建立型別設定為 `number` 的變數，然後稍後將其變更為 `string` 型別的範例：

```js
let data = 100;  // Initially a number
data = "New data";  // Dynamically changes to a string
```

在靜態型別語言中，不允許這種變更，因為資料型別會被固定。

總結來說，JavaScript 的動態型別允許變數自由變換型別，這提供了彈性，但可能導致執行期間出現意外錯誤。

靜態型別語言如 Java 需要你事先指定變數型別，這有助於在程式執行前捕捉錯誤，但彈性較低。
