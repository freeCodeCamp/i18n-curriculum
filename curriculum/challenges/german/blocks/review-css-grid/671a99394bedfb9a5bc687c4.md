## Grundlagen von CSS Grid

- **Definition**: CSS Grid ist ein zweidimensionales Layoutsystem, das verwendet wird, um komplexe Layouts auf Webseiten zu erstellen. Grids bestehen aus Zeilen und Spalten mit Abständen dazwischen. Um ein Grid-Layout zu definieren, setzen Sie die `display`-Eigenschaft auf `grid`.

```css
.container {
  display: grid;
}
```

- **Die `fr` (Bruch-)Einheit**: Diese Einheit stellt einen Bruchteil des Raums innerhalb des Grid-Containers dar. Sie können die `fr`-Einheit verwenden, um flexible Grids zu erstellen.
- **Abstände zwischen Tracks erstellen**: Es gibt drei Möglichkeiten, Abstände zwischen Tracks im CSS Grid zu erzeugen. Sie können die `column-gap`-Eigenschaft verwenden, um Abstände zwischen Spalten zu erzeugen. Die `row-gap`-Eigenschaft erzeugt Abstände zwischen Zeilen. Oder Sie verwenden die Kurzschreibweise-Eigenschaft `gap`, um Abstände sowohl zwischen Zeilen als auch Spalten zu erzeugen.
- **`grid-template-columns`**: Diese Eigenschaft wird verwendet, um Liniennamen und Größen für die Grid-Track-Spalten festzulegen.

```css
.container {
  display: grid;
  width: 100%;
  grid-template-columns: 30px 1fr;
}
```

- **`grid-template-rows`**: Diese Eigenschaft wird verwendet, um Liniennamen und Größen für die Grid-Track-Zeilen festzulegen.
- **`grid-auto-flow`**: Diese Eigenschaft bestimmt, wie automatisch platzierte Elemente im Grid angeordnet werden.

```css
.container {
  display: grid;
  width: 100%;
  grid-auto-flow: column;
}
```

- **`grid-auto-columns`**: Diese Eigenschaft wird verwendet, um die Größe für implizit erstellte Spalten festzulegen.

```css
.container {
  display: grid;
  width: 100%;
  grid-auto-columns: auto;
}
```

- **`place-items`**: Diese Eigenschaft wird verwendet, um Elemente sowohl in Block- als auch in Inline-Richtung auszurichten.
- **`align-items`**: Diese Eigenschaft wird verwendet, um die Ausrichtung der Elemente in einem Grid-Container festzulegen.
- **`repeat()`-Funktion**: Diese Funktion wird verwendet, um Abschnitte in der Track-Liste zu wiederholen. Anstatt `grid-template-columns: 1fr 1fr 1fr;` zu schreiben, können Sie stattdessen die `repeat()`-Funktion verwenden.

```css
.container {
  display: grid;
  grid-template-columns: repeat(3, 1fr);
}
```

- **Explizites Grid**: Sie können die Anzahl der Linien oder Tracks mit den Eigenschaften `grid-template-columns` oder `grid-template-rows` festlegen.
- **Implizites Grid**: Wenn Elemente außerhalb des Grids platziert werden, werden automatisch Zeilen und Spalten für diese außerhalb liegenden Elemente erstellt.
- **`minmax()`-Funktion**: Diese Funktion wird verwendet, um die Mindest- und Maximalgrößen für einen Track festzulegen.

```css
.container {
  display: grid;
  grid-template-columns: repeat(4, 1fr);
  grid-auto-rows: minmax(150px, auto);
}
```

- **Linienbasierte Platzierung**: Alle Grids haben Linien. Um anzugeben, wo das Element an einer Linie beginnt, können Sie die Eigenschaften `grid-column-start` und `grid-row-start` verwenden. Um anzugeben, wo das Element an der Linie endet, können Sie die Eigenschaften `grid-column-end` und `grid-row-end` verwenden. Alternativ können Sie auch die Kurzschreibweise-Eigenschaften `grid-column` oder `grid-row` verwenden.

Hier ist ein Beispiel für die Verwendung der `grid-column`-Eigenschaft, um ein Element über alle Spalten zu strecken.

```css
.element {
  grid-column: 1 / -1;
}
```

- **`grid-template-areas`**: Diese Eigenschaft wird verwendet, um den Elementen, die Sie im Grid positionieren möchten, einen Namen zu geben.

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<div class="container">
  <div class="header">Header</div>
  <div class="sidebar">Sidebar</div>
  <div class="main">Main Content</div>
  <div class="footer">Footer</div>
</div>

```

```css
.container {
  display: grid;
  grid-template-columns: 200px 1fr; 
  grid-template-rows: auto 1fr auto; 
  grid-template-areas:
    "header header"
    "sidebar main"
    "footer footer"; 
  gap: 20px; 
}

.header {
  grid-area: header; 
  background-color: #4CAF50;
  padding: 10px;
  color: white;
}

.sidebar {
  grid-area: sidebar;
  background-color: #f4f4f4;
  padding: 10px;
}

.main {
  grid-area: main; 
  background-color: #e0e0e0;
  padding: 10px;
}

.footer {
  grid-area: footer; 
  background-color: #4CAF50;
  padding: 10px;
  color: white;
}
```

:::

## Debugging von CSS

- **Entwicklerwerkzeuge (DevTools)**: Entwicklerwerkzeuge ermöglichen es Ihnen, Ihr CSS in Echtzeit zu inspizieren und zu ändern. Der Bereich „Styles“ zeigt alle CSS-Regeln an, die auf das ausgewählte Element angewendet werden, einschließlich vererbter Styles. Sie können einzelne Eigenschaften ein- und ausschalten, Werte bearbeiten und sogar neue Regeln direkt im Browser hinzufügen. Dieses unmittelbare Feedback ist äußerst nützlich, um mit verschiedenen Styles zu experimentieren, ohne Ihren Quellcode zu verändern.
- **CSS-Validatoren**: Validatoren werden verwendet, um Ihr CSS anhand der offiziellen Spezifikationen zu überprüfen und melden Fehler oder Warnungen. Ein bekannter Validator, den Sie verwenden können, ist der W3C CSS Validator.
- **Debugging von responsiven Designs**: Die Entwicklerwerkzeuge bieten eine Option, mit der Sie simulieren können, wie Ihre Webseite auf verschiedenen Bildschirmgrößen und Geräten aussieht. Dies hilft Ihnen, Probleme mit Breakpoints oder Styles zu erkennen, die sich nicht gut über verschiedene Viewport-Größen skalieren lassen.
