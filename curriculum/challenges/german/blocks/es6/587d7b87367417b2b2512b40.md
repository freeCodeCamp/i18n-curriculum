# --description--

Wenn Sie mit `let` nicht vertraut sind, sehen Sie sich <a href="/learn/javascript-algorithms-and-data-structures/basic-javascript/explore-differences-between-the-var-and-let-keywords" target="_blank" rel="noopener noreferrer nofollow">diese Aufgabe über den Unterschied zwischen <code>let</code> und <code>var</code></a> an.

Wenn Sie eine Variable mit dem Schlüsselwort `var` deklarieren, wird sie global deklariert oder lokal, wenn sie innerhalb einer Funktion deklariert wird.

Das Schlüsselwort `let` verhält sich ähnlich, aber mit einigen zusätzlichen Funktionen. Wenn Sie eine Variable mit dem Schlüsselwort `let` innerhalb eines Blocks, einer Anweisung oder eines Ausdrucks deklarieren, ist ihr Gültigkeitsbereich auf diesen Block, diese Anweisung oder diesen Ausdruck beschränkt.

Zum Beispiel:

```js
var numArray = [];
for (var i = 0; i < 3; i++) {
  numArray.push(i);
}
console.log(numArray);
console.log(i);
```

Hier zeigt die Konsole die Werte `[0, 1, 2]` und `3` an.

Mit dem Schlüsselwort `var` wird `i` global deklariert. Wenn also `i++` ausgeführt wird, aktualisiert es die globale Variable. Dieser Code ist ähnlich wie der folgende:

```js
var numArray = [];
var i;
for (i = 0; i < 3; i++) {
  numArray.push(i);
}
console.log(numArray);
console.log(i);
```

Hier zeigt die Konsole die Werte `[0, 1, 2]` und `3` an.

Dieses Verhalten verursacht Probleme, wenn Sie eine Funktion erstellen und sie zur späteren Verwendung innerhalb einer `for`-Schleife speichern, die die Variable `i` verwendet. Denn die gespeicherte Funktion bezieht sich immer auf den Wert der aktualisierten globalen `i`-Variable.

```js
var printNumTwo;
for (var i = 0; i < 3; i++) {
  if (i === 2) {
    printNumTwo = function() {
      return i;
    };
  }
}
console.log(printNumTwo());
```

Hier zeigt die Konsole den Wert `3` an.

Wie Sie sehen, gibt `printNumTwo()` 3 aus und nicht 2. Das liegt daran, dass der Wert, der `i` zugewiesen wurde, aktualisiert wurde und `printNumTwo()` die globale `i` zurückgibt und nicht den Wert, den `i` hatte, als die Funktion in der for-Schleife erstellt wurde. Das Schlüsselwort `let` folgt diesem Verhalten nicht:

```js
let printNumTwo;
for (let i = 0; i < 3; i++) {
  if (i === 2) {
    printNumTwo = function() {
      return i;
    };
  }
}
console.log(printNumTwo());
console.log(i);
```

Hier zeigt die Konsole den Wert `2` an und einen Fehler, dass `i is not defined`.

`i` ist nicht definiert, weil es nicht im globalen Gültigkeitsbereich deklariert wurde. Es ist nur innerhalb der `for`-Schleifenanweisung deklariert. `printNumTwo()` gab den korrekten Wert zurück, weil drei verschiedene `i`-Variablen mit eindeutigen Werten (0, 1 und 2) durch das Schlüsselwort `let` innerhalb der Schleifenanweisung erstellt wurden.

# --instructions--

Beheben Sie den Code so, dass `i`, das in der `if`-Anweisung deklariert ist, eine separate Variable ist als `i`, die in der ersten Zeile der Funktion deklariert wurde. Achten Sie darauf, das Schlüsselwort `var` nirgendwo in Ihrem Code zu verwenden.

Diese Übung soll den Unterschied veranschaulichen, wie die Schlüsselwörter `var` und `let` den Gültigkeitsbereich der deklarierten Variable zuweisen. Beim Programmieren einer Funktion, die der in dieser Übung verwendeten ähnelt, ist es oft besser, unterschiedliche Variablennamen zu verwenden, um Verwirrung zu vermeiden.
