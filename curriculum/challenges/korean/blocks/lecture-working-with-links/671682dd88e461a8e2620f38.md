# --description--

경로는 파일 시스템에서 파일이나 디렉터리의 위치를 지정하는 문자열입니다. 웹 개발에서는 경로를 사용해 개발자가 이미지, 스타일시트, 스크립트, 그리고 다른 웹 페이지 같은 리소스에 링크할 수 있습니다. 절대 경로와 상대 경로가 있는데, 두 가지 모두 파일 시스템 내에서 파일 위치를 지정할 때 필수적입니다. 두 가지를 살펴보고 언제 어떤 것을 사용해야 할지 결정하세요.

절대 경로는 리소스에 대한 완전한 링크입니다. 루트 디렉터리부터 시작해 모든 하위 디렉터리를 포함하고, 마지막으로 파일 이름과 확장자를 포함합니다. "루트 디렉터리"는 계층 구조에서 최상위 디렉터리나 폴더를 의미합니다.

로컬 컴퓨터에 있는 리소스에 링크할 때는 파일의 전체 디렉터리 위치를 포함하는 절대 경로를 사용하세요. 절대 경로로 `about.html` 파일에 링크하는 방법은 다음과 같습니다.

```html
<p>
  Read more on the
  <a
    href="/Users/user/Desktop/fCC/script-code/absolute-vs-relative-paths/pages/about.html"
    >About Page</a
    >
</p>
```

이렇게 보이는 이유는 루트에서 시작해 `Users`라는 폴더로 들어가고, 그다음 `user` 폴더, `Desktop` 폴더, `fCC` 폴더, `script-code` 폴더, `absolute-vs-relative-paths` 폴더, `pages` 폴더로 차례로 들어가서 최종적으로 `about.html` 파일에 도달하기 때문입니다.

절대 URL은 리소스에 접근하기 위한 완전한 주소입니다. 프로토콜(`http`, `https`, `file` 중 하나일 수 있음)과, 리소스가 웹에 있을 경우 도메인 이름을 포함합니다. 다음은 freeCodeCamp 로고에 링크하는 절대 URL 예시입니다.

```html
<a href="https://design-style-guide.freecodecamp.org/img/fcc_secondary_small.svg">
  View fCC Logo
</a>
```

이 예시에서 프로토콜은 `https`, 도메인 이름은 `design-style-guide.freecodecamp.org`, 파일 이름은 `fcc_secondary_small.svg`입니다.

브라우저 주소 표시줄에서 절대 URL은 다음과 같이 보입니다.

```sh
file:///Users/user/Desktop/fCC/script-code/absolute-vs-relative-paths/pages/about.html
```

URL에는 프로토콜 `file://`가 포함되어 있습니다. 또한 `/Users/user/Desktop/fCC/script-code/absolute-vs-relative-paths/pages/`처럼 보이는 경로가 포함되어 있는데, 이는 파일로 이어지는 폴더들의 연속을 나타냅니다. 마지막으로 `about.html`가 포함되어 있는데, 이는 파일 이름과 확장자입니다.

절대 경로는 파일 시스템 내에서 파일의 전체 위치를 보여주며, 로컬 컴퓨터의 리소스에 일반적으로 사용됩니다. 절대 URL은 프로토콜과 웹 리소스의 경우 도메인 이름 같은 접근 정보를 포함해 브라우저가 리소스를 어떻게 어디서 가져올지 알려줍니다.

이제 상대 경로를 살펴보겠습니다. 상대 경로는 현재 파일의 디렉터리를 기준으로 파일 위치를 지정합니다. 프로토콜이나 도메인 이름을 포함하지 않아 같은 웹사이트 내 내부 링크에 더 짧고 유연합니다. 다음은 같은 폴더에 있는 `about.html` 페이지에서 `contact.html` 페이지로 링크하는 예시입니다.

```html
<p>
  Read more on the
  <a href="about.html">About Page</a>
</p>
```

`contact.html` 페이지에 있다고 상상해 보세요. `about.html` 페이지가 같은 위치에 있으므로 파일 이름만 가져오면 됩니다. 이것이 상대 파일 경로를 사용하는 예입니다.

그렇다면 언제 절대 경로, 절대 URL, 상대 경로를 사용해야 할까요? 다음 규칙을 따르세요:

- 사이트 루트나 로컬 컴퓨터의 알려진 디렉터리 같은 고정 위치에서 리소스를 참조할 때는 절대 경로를 사용하세요.

- 외부 웹사이트에 호스트된 리소스에 링크할 때는 절대 URL을 사용하세요.

- 같은 웹사이트 내 리소스에 링크할 때는 상대 경로를 사용하세요.

- 개발 중 코드를 더 깔끔하고 유지 관리하기 쉽게 하려면 상대 경로를 사용하세요.

- 인터넷 연결 없이도 링크가 작동하는지 확인하려면 로컬 테스트 시 상대 경로를 사용하세요.
