# --description--

Jetzt lernen Sie npm-Skripte kennen.

Bevor wir loslegen, wiederholen wir einige wichtige Konzepte:

* **npm** ist ein Kommandozeilen-Tool, mit dem Sie JavaScript-Pakete installieren und verwalten können.
    
* Ein **Paket** ist ein Verzeichnis, das zusammengehörigen Code in einer logischen Struktur gruppiert.
    
* Ein **Skript** ist eine Folge von Anweisungen oder Befehlen, die in einer Skriptsprache geschrieben sind.
    

**npm-Skripte** sind benutzerdefinierte Befehle, die Sie in der `package.json`-Datei Ihrer Anwendung definieren können.

Sie sind hilfreich, um Aufgaben und Arbeitsabläufe zu automatisieren, die Sie während der Entwicklungs-, Test- und Deployment-Phasen Ihres Projekts wiederholt verwenden.

Ein Beispiel für die Verwendung von npm-Skripten ist die Anpassung von `npm start`, um Ihre gesamte Entwicklungsumgebung zu starten. Anstatt jedes Mal alle Befehle einzeln manuell einzugeben, wenn Sie Ihre Entwicklungsumgebung starten müssen, können Sie einfach `npm start` ausführen, und der gesamte Prozess wird automatisch ausgeführt.

Sie können npm-Skripte auch verwenden, um das Linting im gesamten Codebasis zu automatisieren und Test-Suiten auszuführen, unter anderem.

Sie sind sehr mächtig, also schauen wir uns ihre Grundlagen an.

## Wie man eine package.json-Datei erstellt

Um benutzerdefinierte npm-Skripte erstellen und ausführen zu können, benötigen Sie eine `package.json`-Datei.

Die `package.json`-Datei ist für Node.js-Projekte unerlässlich. Diese Datei enthält die Metadaten, Abhängigkeiten und Skripte des Projekts.

Zur Erinnerung: Um eine `package.json`-Datei zu erstellen, müssen Sie den `npm init`-Befehl im Stammverzeichnis Ihres Projekts ausführen. Dann können Sie den Paketnamen, die Version, die Beschreibung und andere Details manuell eingeben oder diese Felder leer lassen, um die Standardwerte zu verwenden.

Nachdem Sie diesen Prozess abgeschlossen haben, haben Sie eine `package.json`-Datei im Stammverzeichnis Ihres Projekts.

Wenn Sie alle Standardwerte beibehalten haben, sollten Sie diese Schlüssel-Wert-Paare sehen, wenn Sie `package.json` in Ihrem Code-Editor öffnen:

```json
{
  "name": "<project_name>",
  "version": "1.0.0",
  "main": "index.js",
  "scripts": {
    "test": "echo \"Error: no test specified\" && exit 1"
  },
  "author": "",
  "license": "ISC",
  "description": ""
}
```

Beachten Sie dieses Schlüssel-Wert-Paar hier:

```json
"scripts": {
  "test": "echo \"Error: no test specified\" && exit 1"
}
```

Hier geben Sie Ihre npm-Skripte unter dem Schlüssel `scripts` an.

Standardmäßig sehen Sie dort ein Skript, das Test-Skript:

```json
"test": "echo \"Error: no test specified\" && exit 1"
```

Dies ist ein Beispiel für ein npm-Skript, das in `package.json` definiert ist.

Hier ist die grundlegende Syntax zur Definition eines npm-Skripts:

```json
"<script_name>": "<commands>"
```

Links vom Doppelpunkt finden Sie den Skriptnamen. Übliche Namen sind `run`, `start`, `test`, `build`, `dev` und `lint`. Diese Namen fungieren als „Abkürzungen“ für die dahinterstehenden Befehle rechts.

Rechts vom Doppelpunkt finden Sie die eigentlichen Befehle.

Zum Beispiel:

```json
"scripts": {
  "start": "node app.js"
}
```

Um das Skript auszuführen, geben Sie dies in der Kommandozeile ein:

```bash
npm run <script_name>
```

So können Sie also beliebige benutzerdefinierte Skripte ausführen, die in `package.json` definiert sind.

Manchmal können Sie diese Kurzform ohne das Wort `run` verwenden, um Skripte auszuführen:

```bash
npm <script_name>
```

Das funktioniert aber nur für bestimmte npm-Skripte mit den Namen `start`, `test`, `stop` und `restart`.

Zurück zu unserem Beispiel: Wenn wir Folgendes in `package.json` haben:

```json
"scripts": {
  "start": "node app.js"
}
```

Um das npm-Skript auszuführen, müssen Sie `npm run start` in der Kommandozeile eingeben. Dabei wird tatsächlich der dahinterstehende Befehl `node app.js` ausgeführt.

Da dieses npm-Skript eine Kurzform hat, könnten Sie auch `npm start` verwenden, und das Ergebnis wäre dasselbe.

In diesem Fall ersetzen Sie einfach einen einfachen Befehl, `node app.js`, durch einen anderen, `npm start`. Aber stellen Sie sich vor, Sie hätten eine Folge komplexer Befehle, die nacheinander oder parallel ausgeführt werden müssen, und könnten einfach `npm start` verwenden, um sie alle auszuführen. Das würde Ihnen viel Zeit sparen, oder? Genau hier zeigt sich die Stärke von npm-Skripten.

Hier eine kurze Übersicht über einige der am häufigsten verwendeten npm-Skripte:

* `npm run`: wird verwendet, um ein benutzerdefiniertes npm-Skript auszuführen.
    
* `npm start`: wird verwendet, um die Hauptanwendung oder den Entwicklungsserver zu starten.
    
* `npm lint`: wird verwendet, um einen Code-Stil-Checker auszuführen, der überprüft, ob Ihr Code den Stilstandards und bewährten Vorgehensweisen entspricht.
    
* `npm test`: wird verwendet, um die Testsuite des Projekts auszuführen.
    
Es gibt viele weitere npm-Skripte für unterschiedliche Zwecke, aber dies sind einige der gebräuchlichsten.

Außerdem ist es eine gängige Konvention, einen Doppelpunkt zu verwenden, um verwandte Skripte unter demselben Namensraum zu gruppieren. Zum Beispiel `start:server` und `start:ui`.

## Wie man Befehle nacheinander ausführt

Wenn Sie mehrere Befehle nacheinander ausführen müssen, haben Sie zwei Möglichkeiten. Sie können entweder den logischen UND-Operator (`&&`) oder das Semikolon (`;`) verwenden. Schauen wir uns ihre Unterschiede an.

Der logische UND-Operator führt den nächsten Befehl nur aus, wenn der erste erfolgreich ausgeführt wurde.

Zum Beispiel:

```json
"scripts": {
  "build": "webpack --mode production",
  "start": "npm run build && node server.js"
}
```

Beachten Sie, dass im `start`-Befehl ein logischer UND-Operator verwendet wird:

```json
"start": "npm run build && node server.js"
```

Hier versucht `npm start` zuerst, `npm run build` auszuführen, und nur wenn dieses Skript erfolgreich läuft, wird `node server.js` ausgeführt.

Das ist eine sehr gebräuchliche Vorgehensweise, weil sie Zuverlässigkeit sicherstellt und verhindert, dass Zeit und Ressourcen für nachfolgende Befehle verschwendet werden, die fehlschlagen oder Fehler werfen könnten, wenn der vorherige Befehl fehlschlägt.

Alternativ können Sie das Semikolon verwenden, um Befehle nacheinander auszuführen, unabhängig davon, ob der vorherige Befehl erfolgreich war oder nicht.

In unserem Beispiel hätte diese einfache Änderung eine wichtige Auswirkung darauf, wie die Befehle ausgeführt werden:

```json
"scripts": {
  "build": "webpack --mode production",
  "start": "npm run build; node server.js"
}
```

Konzentrieren Sie sich noch einmal auf den `start`-Befehl:

```json
"start": "npm run build; node server.js"
```

Da wir jetzt ein Semikolon anstelle des `&&`-Operators verwenden, wird der `node server.js`-Befehl immer nach `npm run build` ausgeführt, selbst wenn `node server.js` nicht erfolgreich war.

```json
"start": "npm run build; node server.js"
```

## Wie man Befehle gleichzeitig ausführt

Wenn Sie Befehle gleichzeitig (parallel) ausführen müssen, können Sie den `&`-Operator verwenden.

```json
"scripts": {
  "start:server": "node server/index.js",
  "start:client": "npm run build:watch",
  "dev": "npm run start:server & npm run start:client"
}
```

Sie sollten jedoch wissen, dass diese Option die Prozesse im Hintergrund ausführt.

Das bedeutet, wenn Sie sie in derselben Terminal-Sitzung ausführen, sehen Sie möglicherweise deren kombinierten Output, der schwer zu verstehen sein kann.

Andere Optionen, um Befehle gleichzeitig auszuführen, sind Tools wie `concurrently` und `npm-run-all`.

## Wie man Argumente an npm-Skripte übergibt

Wenn Sie Argumente direkt an den Befehl übergeben müssen, der innerhalb des npm-Skripts ausgeführt wird, können Sie diese im Befehl angeben, indem Sie zwei Bindestriche (`--`) gefolgt von den Argumenten schreiben.

Die `--` sagt npm, dass alles, was danach kommt, Argumente für die dahinterstehenden Befehle sind, die das npm-Skript ausführt.

Das ist die allgemeine Syntax:

```bash
npm run <script_name> -- <argument_1> <argument_2> <argument_3>
```

Zum Beispiel, nehmen wir an, Sie definieren dieses `npm start`-Skript:

```json
{
  "name": "project-example",
  "scripts": {
    "start": "node app.js"
  }
}
```

Wie könnten Sie den Port angeben, auf dem Sie Ihren Server ausführen möchten, wenn Sie `npm start` verwenden?

Angenommen, wir wollen den Port direkt als Kommandozeilenargument übergeben. Dann schreiben Sie Folgendes:

```bash
npm start -- 8000
```

In Ihrem Code können Sie auf diese Argumente im `process.argv`-Array über Indizes zugreifen:

```bash
process.argv[0] # The path to the node executable.
process.argv[1] # The script file being executed.
process.argv[2] # The first argument (8000).
```

Daher könnten Sie die Portnummer überall dort verwenden, wo Sie sie brauchen, mit `process.argv[2]`. Das ist die Stärke von Kommandozeilenargumenten in Kombination mit npm-Skripten.

Wie Sie sehen, sind npm-Skripte unverzichtbare Werkzeuge, um den Entwicklungsworkflow zu vereinfachen. Sie helfen Ihnen, komplexe und wiederverwendbare Befehlsfolgen zu automatisieren und in einfache, konsistente Abkürzungen zu verwandeln, die Sie in Ihrer täglichen Arbeit verwenden können.
