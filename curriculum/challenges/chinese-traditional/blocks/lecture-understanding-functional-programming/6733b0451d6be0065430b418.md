# --interactive--

函式式程式設計是關於撰寫乾淨、可預測且較容易測試和偵錯的程式碼。今天我們將探討的兩個關鍵概念是純函式和避免副作用。

讓我們從純函式（程式）開始。 

純函式（程式）是指在給定相同輸入時，總是會傳回相同輸出，且不會修改自身以外的任何東西的函式（程式）。

以下是一個純函式（程式）的範例：

```js
function add(a, b) {
  return a + b;
}
```

這個函式（程式）是純粹的，因為它對相同的輸入總是傳回相同的結果，且不會改變自身以外的任何東西。無論你呼叫多少次 `add(2, 3)`，它都會傳回 `5`，且不會影響你程式的任何其他零件。

現在，讓我們將此與不純的函式（程式）做比較：

```js
let total = 0;
function addToTotal(value) {
  total += value;
  return total;
}
```

這個函式是不純的，因為它修改了自身外部的變數（`total`）。`addToTotal(5)` 的結果會依據函式被呼叫前 `total` 的值而不同。這種不可預測性會讓我們的程式碼更難理解和偵錯。

這帶我們進入下一個重要的概念，也就是副作用。

副作用是當呼叫函式時，程式狀態中發生的任何變更。這可能包括修改全域變數、寫入檔案或進行 API 呼叫。

純函式（程式）沒有副作用，但不純函式（程式）則有這些。

避免副作用是函式（程式）設計的一個關鍵原則。當我們將副作用降到最低時，我們的程式碼會變得更可預測且更容易測試。 

在撰寫程式碼時，我們的主要目標應該是讓函式只做一件事，並且非常好地完成它，而不改變我們程式的其他零件。

以下是一個具有副作用的函式（程式）範例：

:::interactive_editor

```js
function greet(name) {
  console.log(`Hello, ${name}!`);
}

greet("Alice");
```

:::

這個函式（程式）有副作用，因為它透過記錄到主控台與外部世界交談。 

在函式程式設計中，我們可能會將此重寫為：

:::interactive_editor

```js
function greet(name) {
  return `Hello, ${name}!`;
}
console.log(greet("Alice"));
```

:::

現在 `greet` 是一個純函式（程式），它僅傳回一個字串，而我們則分別控制副作用（記錄到主控台）。

函式式程式設計鼓勵我們使用純函式（程式）來組合我們的程式，透過將程式碼拆解成小且可預測的部分。因此，我們可以建置出更容易理解、測試和維護的程式。
