# --description--

Повертаючись до розділу інформаційної безпеки, ви, можливо, пам’ятаєте, що зберігати паролі у відкритому вигляді *ніколи* не можна. Тепер настав час реалізувати BCrypt, щоб вирішити цю проблему.

`bcrypt@~5.0.0` вже додано як залежність, тож імпортуйте його у свій сервер. Вам потрібно буде обробляти хешування у двох ключових областях: там, де ви реєструєте/зберігаєте новий акаунт, і коли перевіряєте, чи пароль правильний під час входу.

Наразі на вашому маршруті реєстрації ви вставляєте відкритий пароль користувача у базу даних так: `password: req.body.password`. Замість цього захешуйте паролі, додавши наступне перед логікою бази даних: `const hash = bcrypt.hashSync(req.body.password, 12);`, і замінивши `req.body.password` у збереженні в базі даних просто на `password: hash`.

У вашій стратегії автентифікації ви перевіряєте наступне у коді перед завершенням процесу: `if (password !== user.password) return done(null, false);`. Після внесення попередніх змін тепер `user.password` — це хеш. Перед тим, як змінювати існуючий код, зверніть увагу, що інструкція перевіряє, чи пароль **не** дорівнює, і тоді повертає неавтентифікований результат. З урахуванням цього змініть код так, щоб правильно перевіряти введений пароль проти хешу:

```js
if (!bcrypt.compareSync(password, user.password)) { 
  return done(null, false);
}
```

Ось і все, що потрібно, щоб реалізувати одну з найважливіших функцій безпеки при зберіганні паролів.

Відправте свою сторінку, коли вважатимете, що зробили все правильно. Якщо виникають помилки, ви можете <a href="https://forum.freecodecamp.org/t/advanced-node-and-express/567135#hashing-your-passwords-1" target="_blank" rel="noopener noreferrer nofollow">переглянути проєкт, виконаний до цього моменту</a>.
