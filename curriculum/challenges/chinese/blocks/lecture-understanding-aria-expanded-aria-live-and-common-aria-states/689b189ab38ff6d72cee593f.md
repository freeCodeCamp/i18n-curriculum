# --interactive--

你之前学习了语义标记和 ARIA 属性如何使内容可访问。在实际应用中，内容很少是静态的，页面通常通过 JavaScript 动态更新。当发生这种情况时，确保这些更改也反映在 `HTML` 中非常重要。这使得屏幕阅读器能够准确地向用户传达更新后的状态。否则，依赖辅助工具的人可能永远不会知道内容发生了变化，或者可能会收到过时或误导性的信息。

让我们来看一个常见场景，其中 JavaScript 更改元素的状态，以及你如何通过更新 ARIA 属性来保持你的超文本标记语言可访问。

假设你有一个按钮可以打开和关闭下拉 `menu`：

:::interactive_editor

```html
<button id="menuButton" aria-expanded="false" aria-controls="menuList">
  Menu
</button>

<ul id="menuList" hidden>
  <li>List item 1</li>
  <li>List item 2</li>
</ul>
<script src="index.js"></script>
```

```js
const button = document.getElementById("menuButton");
const menu = document.getElementById("menuList");

button.addEventListener("click", () => {
  const expanded = button.getAttribute("aria-expanded") === "true";
  button.setAttribute("aria-expanded", String(!expanded));
  menu.hidden = expanded;
});
```

:::

超文本标记语言 设置了一个控件下拉菜单的按钮。该按钮使用 `aria-expanded` 来指示菜单是否打开，并使用 `aria-controls` 通过引用菜单的 `id` 将其链接到菜单。

JavaScript 监听菜单按钮的点击。当点击时，它切换 `aria-expanded` 的值并更新菜单上的 `hidden` 属性。这保持菜单的可见性和无障碍信息同步，使有视力的用户和屏幕阅读器用户都能知道菜单是打开还是关闭。

你将在接下来的课程中了解更多关于那些 ARIA 属性的内容。现在，只需记住，当你的 JavaScript 更改了一个元素的状态时，你也应该更新相关的 ARIA 属性，以便辅助技术能够准确反映屏幕上的实际内容。
