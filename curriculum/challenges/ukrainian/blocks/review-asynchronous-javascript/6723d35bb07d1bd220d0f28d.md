# --description--

- **Синхронний JavaScript** виконується послідовно і чекає на завершення попередньої операції перед переходом до наступної.
- **Асинхронний JavaScript** дозволяє виконувати кілька операцій у фоновому режимі, не блокуючи головний потік.
- **Потік** — це послідовність інструкцій, яку можна виконувати незалежно від основного потоку програми.
- **Функції зворотного виклику** — це функції, які передаються як аргументи іншим функціям і виконуються після завершення операції або в результаті події.

## JavaScript-рушій та середовище виконання JavaScript

- **JavaScript-рушій** — це програма, яка виконує JavaScript-код у веббраузері. Він працює як конвертер, який бере ваш код, перетворює його на інструкції, зрозумілі комп’ютеру, і виконує їх.
- V8 — приклад JavaScript-рушія, розробленого Google.
- **Середовище виконання JavaScript** — це середовище, у якому виконується JavaScript-код. Воно включає JavaScript-рушій, який обробляє та виконує код, а також додаткові функції, як-от веббраузер або Node.js.

## Fetch API

- Fetch API дозволяє вебзастосункам робити мережеві запити, зазвичай для отримання або відправлення даних на сервер. Він надає метод `fetch()`, який можна використовувати для виконання цих запитів.
- Ви можете отримувати текст, зображення, аудіо, JSON та інші типи даних за допомогою Fetch API.

## HTTP-методи для Fetch API

Fetch API підтримує різні HTTP-методи для взаємодії з сервером. Найпоширеніші методи:

- **GET**: використовується для отримання даних із сервера. За замовчуванням Fetch API використовує метод `GET` для отримання даних.

```js
fetch('https://api.example.com/data')
```

Щоб використовувати отримані дані, їх потрібно конвертувати у формат JSON за допомогою методу `.json()`:

```js
fetch('https://api.example.com/data')
  .then(response => response.json())
  .then(data => console.log(data))
```

У цьому коді `fetch()` повертає проміс, а обробник `.then()` конвертує розв’язану відповідь у формат JSON.

- **POST**: використовується для відправлення даних на сервер. Метод `POST` використовується для створення нових ресурсів на сервері.

```js
fetch('https://api.example.com/users', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({
    name: 'John Doe',
    email: 'john@example.com'
  })
})
```

У цьому прикладі ми надсилаємо `POST`-запит для створення нового користувача. Ми вказали метод як `POST`, встановили відповідні заголовки та додали тіло з даними, які хочемо надіслати. Тіло має бути рядком, тому ми використовуємо `JSON.stringify()` для перетворення нашого об’єкта на JSON рядок.

- **PUT**: використовується для оновлення даних на сервері. Метод `PUT` використовується для оновлення існуючих ресурсів на сервері.

```js
fetch('https://api.example.com/users/45', {
  method: 'PUT',
  headers: {
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({
    name: 'John Smith',
    email: 'john@example.com'
  })
})
```

У цьому прикладі ми оновлюємо ідентифікатор `45`, який вказаний в кінці посилання. Ми використали метод `PUT` у коді, а також вказали дані як тіло, які будуть використані для оновлення ідентифікованих даних.

- **DELETE**: використовується для видалення даних на сервері. Метод `DELETE` використовується для видалення ресурсів на сервері.

```js
fetch('https://api.example.com/users/45', {
  method: 'DELETE'
})
```

У цьому прикладі ми надсилаємо `DELETE`-запит, щоб видалити користувача з ідентифікатором `45`.

## Проміс і ланцюжок промісів

- **Проміси** — це об’єкти, які представляють майбутнє завершення або невдачу асинхронної операції та її результативне значення. Значення промісу відоме лише після завершення асинхронної операції.
- Ось приклад створення простого промісу:

```js
const promise = new Promise((resolve, reject) => {
  setTimeout(() => {
    resolve('Data received successfully');
  }, 2000);
});
```

- Метод `.then()` використовується в промісі, щоб вказати, що має статися, коли проміс буде виконано, тоді як `.catch()` використовується для обробки будь-яких помилок, які виникають.
- Ось приклад використання `.then()` та `.catch()` з промісом:

```js
promise
  .then(data => {
    console.log(data);
  })
  .catch(error => {
    console.error(error);
  });
```

У наведеному вище прикладі метод `.then()` використовується для виведення даних, отриманих з промісу, тоді як метод `.catch()` використовується для виведення будь-яких помилок, які виникають.

- **Ланцюжок промісів**: одна з потужних функцій промісів полягає в тому, що ми можемо створити ланцюжок з кількох асинхронних операцій. Кожен `.then()` може повертати новий проміс, що дозволяє виконувати послідовність асинхронних операцій одна за одною.
- Ось приклад ланцюжка промісів:

```js
fetch('https://api.example.com/data')
  .then(response => response.json())
  .then(data => {
    console.log(data);
    return fetch('https://api.example.com/other-data');
  })
  .then(response => response.json())
  .then(otherData => {
    console.log(otherData);
  })
  .catch(error => {
    console.error(error);
  });
```

У наведеному вище прикладі ми спочатку отримуємо дані з одного посилання, потім отримуємо дані з іншого посилання на основі першої відповіді, і нарешті виводимо у журнал другі отримані дані.

Метод `catch` обробляє будь-які помилки, що виникають під час процесу. Це означає, що вам не потрібно додавати обробку помилок до кожного кроку, що значно спрощує код.

## Використання `async/await` для обробки промісів

`async`/`await` робить написання та читання асинхронного коду простішим, він побудований на основі промісів.

- **async**: ключове слово `async` використовується для визначення асинхронної функції. Функція `async` повертає проміс, який виконується зі значенням, повернутим функцією `async`.
- **await**: ключове слово `await` використовується всередині функції `async`, щоб призупинити виконання функції до того моменту, поки проміс не буде виконаний. Його можна використовувати лише всередині функції `async`.
- Ось приклад використання `async/await`:

```js
async function delayedGreeting(name) {
  console.log("A Messenger entered the chat...");
  await new Promise(resolve => setTimeout(resolve, 2000));
  console.log(`Hello, ${name}!`);
}

delayedGreeting("Alice");
console.log("First Printed Message!");
```

У наведеному вище прикладі функція `delayedGreeting` є функцією `async`, яка призупиняється на 2 секунди перед тим, як вивести привітальне повідомлення. Ключове слово `await` використовується для призупинення виконання функції до того моменту, поки `Promise` не буде виконано.

- Одна з найбільших переваг `async/await` — це обробка помилок за допомогою блоків `try/catch`. Ось приклад:

```js
async function fetchData() {
  try {
    const response = await fetch('https://api.example.com/data');
    const data = await response.json();
    console.log(data);
  } catch (error) {
    console.error(error);
  }
}

fetchData();
```

У наведеному вище прикладі блок `try` містить код, який може викликати помилку, а блок `catch` обробляє помилку, якщо вона виникає. Це робить обробку помилок простою та зрозумілою.

## Атрибут `async`

- Атрибут `async` повідомляє браузеру скачати файл скрипта асинхронно, продовжуючи парсинг HTML-документа.
- Після того, як скрипт завантажено, парсинг HTML призупиняється, скрипт виконується, а потім парсинг HTML відновлюється.
- Використовуйте `async` для незалежних скриптів, де порядок виконання не має значення.

## Атрибут `defer`

- Атрибут `defer` також завантажує скрипт асинхронно, але відкладає виконання скрипта до повного парсингу HTML-документа.
- Скрипти `defer` зберігають порядок виконання відповідно до їх розташування в HTML-документі.

- Важливо зазначити, що обидва атрибути `async` та `defer` ігноруються для рядкових скриптів і працюють лише для зовнішніх скриптових файлів.

- Коли присутні обидва атрибути `async` та `defer`, то перевага надається атрибуту `async`.

## Geolocation API

- Geolocation API дозволяє зробити запит про місцезнаходження користувача.

- Наведений нижче приклад демонструє метод API `getCurrentPosition()`, який використовується для отримання поточного місцезнаходження користувача.

```js
navigator.geolocation.getCurrentPosition(
  (position) => {
    console.log("Latitude: " + position.coords.latitude);
    console.log("Longitude: " + position.coords.longitude);
  },
  (error) => {
    console.log("Error: " + error.message);
  }
);
```

У цьому коді ми викликаємо `getCurrentPosition` і передаємо йому функцію, яка буде викликана, коли позиція буде успішно отримана.

Об’єкт `position` містить різноманітну інформацію, але тут ми вибрали лише `latitude` та `longitude`.

Якщо виникає проблема з отриманням `position`, помилка буде виведена в консоль.

- Важливо поважати приватність користувача і запитувати його місцезнаходження лише тоді, коли це необхідно.
