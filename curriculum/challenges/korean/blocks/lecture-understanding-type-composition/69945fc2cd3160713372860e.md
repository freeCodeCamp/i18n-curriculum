# --description--

이전 수업에서 유니언 타입을 다루는 방법을 배웠습니다. 유니언 타입을 다루는 일반적인 방법 중 하나는 타입 별칭을 사용하는 것입니다.

타입 별칭은 타입에 붙이는 이름입니다. 다음은 유니언 타입에 이름을 붙이는 예시입니다:

```ts
type ID = number | string;
```

프로젝트에서 타입 별칭을 만들고 프로그램 전반에 걸쳐 이렇게 사용하는 경우가 많습니다:

```ts
type ID = number | string;

type User = {
  id: ID;
  name: string;
};

function getUserId(user: User): ID {
  return user.id;
}
```

이 예시는 `ID`와 `User`라는 두 타입을 사용합니다. `ID`는 `number` 또는 `string`의 유니언 타입이며, `User`는 `id`와 `name` 속성을 가진 객체 타입입니다. `id`은 `ID` 타입으로 지정되었고, `name`은 `string` 타입을 가집니다.

`getUserId` 함수는 `user`이라는 매개변수를 하나 가지며, 그 타입은 `User`입니다. `getUserId` 함수의 반환 타입은 `ID`일 것입니다.

타입 별칭은 객체의 형태를 설명하는 훌륭한 방법입니다. 객체 타입을 만들고 배열에서 이렇게 사용하는 것이 일반적입니다:

```ts
type User = {
  id: number | string;
  name: string;
};

const users: User[] = [
  { id: 101, name: "Alice" },
  { id: "A102", name: "Bob" },
  { id: 103, name: "Charlie" }
];
```

이 예시에서 `users` 배열은 `User` 객체만 채워야 합니다. 객체 중 하나에 속성을 빠뜨리면 TypeScript가 다음과 같은 오류를 발생시킵니다:

```ts
type User = {
  id: number | string;
  name: string;
};

const users: User[] = [
  // Property 'name' is missing in type '{ id: number; }' but required in type 'User'.
  { id: 101 },
  { id: "A102", name: "Bob" },
  { id: 103, name: "Charlie" }
];
```

타입 별칭 이름을 지을 때 PascalCase를 사용하는 것을 눈치챘을 것입니다. 이는 관례상 PascalCase를 사용하면 타입과 변수를 구분하는 데 도움이 되고 가독성도 좋아지기 때문입니다. `User[]`를 보면 `User` 타입 객체의 배열임을 알 수 있습니다.
