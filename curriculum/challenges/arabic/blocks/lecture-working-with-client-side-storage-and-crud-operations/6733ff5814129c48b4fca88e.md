# --description--

في الدرس السابق، تعلمنا كيفية العمل مع `localStorage` وتعرّفنا بإيجاز على `sessionStorage`.

تذكّر أن `sessionStorage` يعني أن البيانات تُمسح بمجرد إغلاق المستخدم للعلامة أو النافذة التي يعمل فيها تطبيق الويب. هذا مثالي للحالات التي تحتاج فيها البيانات إلى الاستمرار فقط طوال جلسة واحدة، مثل الحفاظ على بيانات النموذج أثناء التنقل أو تخزين معلومات الحالة المؤقتة أثناء عملية الدفع.

تمامًا مثل `localStorage`، يستخدم `sessionStorage` أزواج المفتاح-القيمة لتخزين البيانات واسترجاعها. الطرق المستخدمة مع `sessionStorage` هي نفسها المستخدمة مع `localStorage`، والفرق الحقيقي الوحيد هو مدة تخزين البيانات.

إليك بعض الأمثلة على العمل مع الطرق المختلفة:

- `sessionStorage.setItem()`: يخزن زوج مفتاح-قيمة في `sessionStorage`.

```js
sessionStorage.setItem('cart', '3 items');
```

- `sessionStorage.getItem()`: يسترجع قيمة مفتاح معين من `sessionStorage`.

```js
let cart = sessionStorage.getItem('cart');
console.log(cart); // Outputs: '3 items'
```

- `sessionStorage.removeItem()`: يزيل عنصرًا محددًا من `sessionStorage` باستخدام مفتاحه.

```js
sessionStorage.removeItem('cart');
```

- `sessionStorage.clear()`: يمسح كل البيانات المخزنة في `sessionStorage`.

```js
sessionStorage.clear();
```

لننظر إلى مثال حيث نخزن بيانات في `sessionStorage` تدوم فقط طالما أن علامة المتصفح أو النافذة مفتوحة:

```js
// Store data in sessionStorage
sessionStorage.setItem('currentUser', 'JohnDoe');

// Retrieve the stored data
const user = sessionStorage.getItem('currentUser');
console.log(user); // 'JohnDoe'

// Remove a specific key from sessionStorage
sessionStorage.removeItem('currentUser');

// Clear all sessionStorage data
sessionStorage.clear();
```

في هذا المثال، نحن:

1. نخزن اسم المستخدم الحالي (`JohnDoe`) في `sessionStorage`.

2. نسترجعه ونعرضه.

3. نزيل العنصر المرتبط بالمفتاح `currentUser`.

4. نمسح كل بيانات `sessionStorage`.

الفرق الرئيسي عن `localStorage` هو أنه بمجرد إغلاق المستخدم للعلامة، ستُفقد كل بيانات الجلسة المخزنة.

`sessionStorage` مفيد بشكل خاص في السيناريوهات مثل:

- تخزين بيانات مؤقتة مثل مدخلات النماذج أثناء عملية نموذج متعددة الصفحات.

- تخزين اختيارات أو تفضيلات مؤقتة لا تحتاج إلى الاستمرار عبر الجلسات.

- الحفاظ على الحالة في تطبيق صفحة واحدة لا يحتاج إلى تذكره بعد إغلاق العلامة.

يضمن `sessionStorage` أنه بمجرد مغادرة المستخدم للصفحة، تُمسح بيانات الجلسة، وهذا مفيد في الحالات التي لا تريد فيها الاحتفاظ بالمعلومات بعد الجلسة الحالية.
