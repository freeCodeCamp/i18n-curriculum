# --description--

تُعتبر JavaScript لغة ذات نوع ديناميكي. هذا يعني أن المتغيرات يمكن أن تستقبل أي قيم أثناء وقت التشغيل - إعلان متغير كرقم لا يمنعك من تعيين قيمة سلسلة نصية له لاحقًا، ويمكن تمرير أي قيمة إلى معلمات الدالة.

التحدي في لغة ذات نوع ديناميكي هو أن غياب أمان النوع قد يُدخل أخطاء إذا لم تكن حذرًا. TypeScript توسع لغة JavaScript لتشمل النوع الثابت، مما يساعد على اكتشاف تلك الأخطاء قبل نشر شفرتك. لكن كيف يعمل ذلك؟

انظر إلى هذه الدالة في JavaScript:

```js
const getRandomValue = (array) => {
  return array[Math.floor(Math.random() * array.length)];
}
```

دالتنا `getRandomValue` تأخذ مصفوفة وتُرجع قيمة عشوائية منها. لكن لأن JavaScript لا تتحقق من الأنواع، لا يوجد ما يمنعك من استدعاء الدالة مع رقم:

```js
console.log(getRandomValue(10));
```

الناتج في وحدة التحكم للمثال الحالي سيكون `undefined` لأن الدالة كانت تتوقع مصفوفة بدلاً من رقم.

لكن مع TypeScript يمكنك تعريف نوع لمعامِل المصفوفة:

```js
const getRandomValue = (array: string[]) => {
  return array[Math.floor(Math.random() * array.length)];
}
```

تعريف النوع هذا يُخبر TypeScript أن المعلمة يجب أن تكون مصفوفة من سلاسل نصية. ثم عندما تستدعي `getRandomValue` وتمرر رقمًا، تحصل على نوع مختلف من الخطأ يُسمى خطأ المترجم.

معظم بيئات تشغيل JavaScript، مثل المتصفح أو Node.js، لا تستطيع تشغيل TypeScript بشكل مباشر. بدلاً من ذلك، تحتاج أولًا إلى ترجمة شفرة TypeScript إلى JavaScript عادي. يمكنك فعل ذلك باستخدام المترجم المدمج مع لغة TypeScript. عند تشغيل المترجم، يقوم TypeScript بتحليل شفرتك ويرمي خطأً لأي مشاكل في عدم تطابق الأنواع - مثل تمرير رقم إلى دالة تتوقع مصفوفة.

ستتعلم المزيد عن كيفية عمل هذه الأنواع في الدروس القادمة. لكن هذه الحماية الإضافية هي ما يجعل TypeScript لغة جذابة للعديد من المطورين والمؤسسات.
