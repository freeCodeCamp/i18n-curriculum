# --description--

惰性注册是一种 UI 设计模式，允许用户浏览和使用你的应用而无需注册。一个很好的例子是电子商务网站。用户应该能够浏览产品并将几项商品添加到他们的购物车中。然后，如果他们有兴趣购买，就需要注册。

原因是用户需要看到你的网站所提供的价值，才愿意提供他们的信息并进行注册。在设计你的应用时，用户应该能够看到价值并感觉应用是安全的，可以提供他们的信息。否则，他们将不愿意注册，你将失去潜在客户。

你需要确保传达用户的敏感数据将被保护和安全。在后续的模块中，我们将讨论如何保护你的应用和保护你的用户的数据。

惰性注册的另一个好例子是 YouTube。YouTube 是一个视频分享平台，拥有数百万个涵盖科技、流行文化和游戏等各类内容的视频。如果你访问 YouTube，用户可以在不需要登录或注册的情况下观看任意数量的视频。然而，如果他们想点赞、发表评论或订阅频道，就需要注册。

如果用户喜欢他们正在观看的内容或想参与对话，那么他们会更愿意注册。惰性注册是一种有用的设计模式，它允许用户在愿意提供他们的信息之前先看到你的应用的价值。

下次你设计一个应用时，考虑使用惰性注册来提高用户参与度和留存率。
