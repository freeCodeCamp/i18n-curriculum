# --description--

프로젝트를 만들다 보면 프로그램이 예상대로 작동하지 않는 문제를 자주 만나게 됩니다.

프로그래머들은 이런 문제를 버그라고 부릅니다. 버그를 찾고 고치는 과정을 디버깅이라고 합니다.

코드를 디버깅하려면 브라우저에서 제공하는 몇 가지 도구를 사용해야 합니다.

중요한 도구 두 가지는 DOM 검사기와 개발자 도구입니다.

DOM 검사기는 현재 페이지의 HTML 구조를 검사할 수 있게 해줍니다.

DOM은 Document Object Model의 약자입니다. 페이지의 요소를 나타내는 트리 구조입니다. DOM에 대해서는 이후 모듈에서 더 배우게 됩니다.

개발자 도구는 현재 페이지의 HTML, CSS, JavaScript를 검사할 수 있게 해줍니다.

앵커 요소에 작은 버그가 있는 HTML 예제를 살펴보겠습니다:

```html
<a href="https://www.freecodecamp.org/larn/">freeCodeCamp curriculum</a>
```

링크를 클릭하면 404 페이지로 이동합니다. 404 페이지는 사용자가 서버에 존재하지 않는 웹페이지에 접근하려 할 때 나타나는 오류 페이지입니다.

링크는 freeCodeCamp 커리큘럼으로 연결되는 것이 의도입니다.

문제가 무엇인지 확인하려면 개발자 도구를 사용할 수 있습니다.

브라우저에서 개발자 도구를 열려면 페이지에서 마우스 오른쪽 버튼을 클릭하고 `Inspect`를 선택하세요.

또한 PC 키보드에서는 `Control Shift I`를, Mac에서는 `Command Option I`를 사용할 수 있습니다.

Google Chrome에서 개발자 도구를 열면 여러 탭이 보입니다. 첫 번째 탭은 `Elements` 탭입니다. 이 탭은 현재 페이지의 HTML 구조를 보여줍니다.

두 번째 탭은 `Console` 탭입니다. 이 탭은 페이지에서 발생할 수 있는 오류를 보여줍니다.

링크가 깨진 경우 콘솔에서 해당 링크의 오류 메시지를 확인할 수 있습니다. 깨진 링크에 대해 계속 표시되는 일반적인 메시지는 404 오류입니다. 404 오류는 페이지를 찾을 수 없음을 나타냅니다.

이로써 문제는 앵커 요소의 URL에 있다는 것을 알 수 있습니다. `href` 값을 검사하면 오타가 있음을 확인할 수 있습니다.

현재 콘솔 메시지는 404 오류에 대해 `/larn`를 보여주지만, 올바른 URL은 `/learn`이어야 합니다. 링크가 수정되면 예상대로 작동합니다.

개발자 도구 사용법은 인증 과정 전반에서 더 배우게 되지만, 이것은 코드를 디버그하는 데 어떻게 도움이 되는지 보여주는 짧은 예시입니다.
