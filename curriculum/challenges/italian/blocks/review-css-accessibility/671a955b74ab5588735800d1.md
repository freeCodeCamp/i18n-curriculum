## Strumenti per il contrasto dei colori

- **WebAIM's Color Contrast Checker**: Questo strumento online ti permette di inserire i colori di primo piano e di sfondo del tuo design e vedere subito se rispettano gli standard delle Web Content Accessibility Guidelines (WCAG).
- **TPGi Colour Contrast Analyzer**: Questo è un controllo gratuito del contrasto dei colori che ti consente di verificare se i tuoi siti web e app rispettano gli standard delle Web Content Accessibility Guidelines (WCAG). Questo strumento include anche una funzionalità di simulazione daltonismo che ti permette di vedere come appare il tuo sito o app per persone con problemi di visione dei colori.

## Albero di accessibilità

L'albero di accessibilità è una struttura usata dalle tecnologie assistive, come i lettori di schermo, per interpretare e interagire con il contenuto di una pagina web.

## Funzione `max()`

La funzione `max()` restituisce il valore più grande tra un insieme di valori separati da virgola:

:::interactive_editor

```html
<head>
  <link rel="stylesheet" href="styles.css">
</head>
<body>
  <img src="https://cdn.freecodecamp.org/curriculum/css-photo-gallery/9.jpg" alt="A cat sitting on a table">
</body>
```

```css
img {
  width: max(250px, 25vw);
}
```

:::

Nell'esempio sopra, la larghezza dell'immagine sarà 250px se la larghezza della viewport è inferiore a 1000 pixel. Se la larghezza della viewport è superiore a 1000 pixel, la larghezza dell'immagine sarà 25vw. Questo perché 25vw corrisponde al 25% della larghezza della viewport.

## Buone pratiche con CSS e accessibilità

- **`display: none;`**: Usare `display: none;` significa che i lettori di schermo e altre tecnologie assistive non potranno consultare questo contenuto, poiché non è incluso nell'albero di accessibilità. Perciò, è importante usare questo metodo solo quando vuoi rimuovere completamente il contenuto sia dalla presentazione visiva sia dall'accessibilità.
- **`visibility: hidden;`**: Questa proprietà e valore nascondono il contenuto visivamente ma lo mantengono nel flusso del documento, quindi occupa ancora spazio nella pagina. Questi elementi non saranno più letti dai lettori di schermo perché saranno stati rimossi dall'albero di accessibilità.
- **Classe CSS `.sr-only`**: Questa è una tecnica comune usata per nascondere visivamente il contenuto mantenendolo accessibile ai lettori di schermo.

:::interactive_editor

```html
<head>
  <link rel="stylesheet" href="styles.css">
</head>
<body>
  <span>This span element can be read</span>
  <span class="sr-only">This span element can only be read by screen readers</span>
  <p style="display: none">
    This text is hidden with display: none and is NOT accessible to screen readers.
  </p>
  <p style="visibility: hidden">
    This text is hidden with visibility: hidden and is NOT read by screen readers,
    but it still takes up space.
  </p>
</body>

```

```css
.sr-only {
  position: absolute;
  width: 1px;
  height: 1px;
  padding: 0;
  margin: -1px;
  overflow: hidden;
  clip: rect(0, 0, 0, 0);
  white-space: nowrap;
  border: 0;
}

```

:::

## Nascondere contenuti con attributi HTML

- **Attributo `aria-hidden`**: Usato per nascondere un elemento alle persone che usano tecnologie assistive come i lettori di schermo. Per esempio, può essere usato per nascondere immagini decorative che non forniscono contenuti significativi.
- **Attributo `hidden`**: Questo attributo è supportato dalla maggior parte dei browser moderni e nasconde il contenuto sia visivamente sia dall'albero di accessibilità. Può essere facilmente attivato o disattivato con JavaScript.

:::interactive_editor

```html
<p aria-hidden>This paragraph is visible for sighted users, but is hidden from assistive technology.</p>
<p hidden>This paragraph is hidden from both sighted users and assistive technology.</p>
<p>This is a normal paragraph</p>

```

:::

## Problema di accessibilità dell'attributo `placeholder`

Usare il testo segnaposto non è ideale per l'accessibilità. Spesso gli utenti confondono il testo segnaposto con un valore effettivo dell'input — pensano che ci sia già un valore inserito nell'input.
