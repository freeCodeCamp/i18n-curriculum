# --explanation--

`The full scope of an issue`는 조치를 취하기 전에 문제의 원인과 결과를 포함한 모든 측면을 이해하는 것을 의미합니다. 예를 들어:

`Before fixing the bug, we need to assess the full scope of its impact on other features.` - 이는 버그가 시스템의 여러 부분에 어떤 영향을 미치는지 분석하는 것을 뜻합니다.

`Let's make sure`는 진행하기 전에 무언가를 확인하거나 검증하는 것을 강조할 때 사용됩니다. 예를 들어:

`Let's make sure we have all the requirements before starting development.` - 이는 개발 과정을 시작하기 전에 모든 준비가 완료되었는지 확인해야 한다는 의미입니다.
