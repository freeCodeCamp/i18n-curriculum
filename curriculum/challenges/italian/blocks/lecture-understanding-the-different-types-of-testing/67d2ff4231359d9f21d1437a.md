# --description--

Il test end-to-end, o E2E in breve, verifica scenari reali dal punto di vista dell’utente. Esempi di scenari di test E2E includono testare il processo di registrazione e login o testare il checkout di un sito di e-commerce.

Per capire meglio come funziona il test end-to-end, diamo un’occhiata a un esempio reale dal codice di freeCodeCamp. Questo esempio usa Playwright, un popolare framework di test end-to-end sviluppato da Microsoft.

All’interno del codice di freeCodeCamp, c’è una directory chiamata `e2e` che contiene test per donazioni, certificazioni, esami e altro. All’interno della directory, c’è un file `donate-page-donor.spec.ts` che ha test per la pagina del donatore autenticato.

Il primo passaggio è navigare alla pagina delle donazioni come utente demo che è già un sostenitore:

```js
test.beforeEach(async ({ page }) => {
  execSync("node ./tools/scripts/seed/seed-demo-user --set-true isDonating");
  await page.goto("/donate");
});
```

Poiché ci sono più test in questo file, il hook `beforeEach` verrà eseguito prima di ciascuno di essi.

Una volta che l’utente è sulla pagina delle donazioni, vedrà un messaggio di ringraziamento con suggerimenti su come supportare ulteriormente freeCodeCamp. Ecco solo alcuni dei test per verificare la presenza di alcuni testi sulla pagina delle donazioni:

```js
test("should render the donate page correctly", async ({ page }) => {
  await expect(
    page.getByText("Thank you for your continued support")
  ).toBeVisible();

  await expect(
    page.getByText(
      "Your contributions are crucial in creating resources that empower millions of people to learn new skills and support their families."
    )
  ).toBeVisible();

  ...
});
```

Ci sono anche test per verificare che il donatore abbia un collegamento da sostenitore nella barra del menu, oltre a un bordo stilizzato speciale intorno al suo avatar per indicare che è un sostenitore:

```js
test("The menu should have a supporters link", async ({ page }) => {
  const menuButton = page.getByTestId("header-menu-button");
  const menu = page.getByTestId("header-menu");

  await expect(menuButton).toBeVisible();
  await menuButton.click();

  await expect(menu).toBeVisible();

  await expect(page.getByRole("link", { name: "Supporters" })).toBeVisible();
});

test("The Avatar should have a special border for donors", async ({ page }) => {
  const container = page.locator(".avatar-container");
  await expect(container).toHaveClass("avatar-container gold-border");
});
```

Avere questo tipo di test dettagliati ti permette di aumentare la copertura dei test per la tua applicazione oltre ai soli test unitari. I test end-to-end aiutano anche a garantire che la tua applicazione si comporti correttamente e sia prevedibile per gli utenti. Altri esempi di strumenti per test end-to-end includono Cypress, Selenium e Puppeteer.

Anche se il test end-to-end è importante, è costoso perché richiede molto tempo per essere configurato, progettato e mantenuto. I test E2E sono ottimi per i flussi utente critici, mentre i test unitari sono ideali per testare piccole unità nell’applicazione.
