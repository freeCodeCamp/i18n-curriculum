# --description--

Il file `.env` è un file nascosto che serve a passare variabili d'ambiente alla tua applicazione. Questo file è segreto, solo tu puoi consultarlo, e può essere usato per archiviare dati che vuoi mantenere privati o nascosti. Per esempio, puoi archiviare chiavi API di servizi esterni o l'URI del tuo database. Puoi anche usarlo per archiviare opzioni di configurazione. Impostando opzioni di configurazione, puoi modificare il comportamento della tua applicazione senza dover riscrivere del codice.

Le variabili d'ambiente sono consultabili dall'app come `process.env.VAR_NAME`. L'oggetto `process.env` è un oggetto globale di Node, e le variabili sono passate come stringhe. Per convenzione, i nomi delle variabili sono tutti in maiuscolo, con le parole separate da un underscore. Il `.env` è un file shell, quindi non serve racchiudere nomi o valori tra virgolette. È anche importante notare che non ci possono essere spazi intorno al segno di uguale quando assegni valori alle variabili, per esempio `VAR_NAME=value`. Di solito, metterai ogni definizione di variabile su una riga separata.

# --instructions--

Aggiungiamo una variabile d'ambiente come opzione di configurazione.

Crea un file `.env` nella radice della directory del tuo progetto e archivia in esso la variabile `MESSAGE_STYLE=uppercase`.

Poi, nel gestore della rotta GET `/json` che hai creato nella sfida precedente, consulta `process.env.MESSAGE_STYLE` e trasforma la proprietà `message` dell'oggetto risposta in maiuscolo se la variabile è uguale a `uppercase`. L'oggetto risposta dovrebbe essere `{"message": "Hello json"}` o `{"message": "HELLO JSON"}`, a seconda del valore `MESSAGE_STYLE`. Nota che devi leggere il valore di `process.env.MESSAGE_STYLE` **all'interno** del gestore della rotta, non all'esterno, a causa del modo in cui funzionano i nostri test.

Avrai bisogno di usare il pacchetto `dotenv`. Questo carica le variabili d'ambiente dal tuo file `.env` dentro `process.env`. Il pacchetto `dotenv` è già stato installato ed è nel file `package.json` del tuo progetto. In cima al file `myApp.js`, aggiungi `require('dotenv').config()` per caricare le variabili d'ambiente.

# --hints--

La risposta dell'endpoint `/json` dovrebbe cambiare in base alla variabile d'ambiente `MESSAGE_STYLE`

    throw new Error(await response.text());
  }
