# --interactive--

HTML، التي تعني لغة ترميز النص الفائق، هي لغة ترميز لإنشاء صفحات الويب. عندما تزور موقعًا إلكترونيًا وترى محتوى مثل الفقرات، العناوين، الروابط، الصور، والفيديوهات؛ فهذا هو HTML.

إليك مثال صغير يستخدم عناصر HTML. جرّب تعديل بعض النصوص في المحرر وشاهد التغييرات تتحدث في نافذة المعاينة.

:::interactive_editor

```html
<h1>Main heading element</h1>

<p>I am a paragraph element.</p>
```

:::

تمثل HTML محتوى وبنية صفحة الويب من خلال استخدام العناصر. معظم العناصر تحتوي على وسم بداية ووسم إغلاق. أحيانًا يُشار إلى هذه الوسوم كوسم بداية ووسم نهاية. بين هذين الوسمين، يوجد المحتوى. يمكن أن يكون هذا المحتوى نصًا أو عناصر HTML أخرى.

إليك مثال آخر لعنصر فقرة. غيّر النص في المحرر ليقول `I love coding!` وشاهد النتائج في نافذة المعاينة.

:::interactive_editor

```html
<p>I am a paragraph element.</p>
```

:::

كل من وسم البداية ووسم الإغلاق يبدأان بقوس زاوية يسار (`<`) وينتهيان بقوس زاوية يمين (`>`)، مع وضع اسم الوسم بين هذين القوسين. رغم أن أسماء وسوم HTML غير حساسة لحالة الأحرف، إلا أن الاصطلاح المقبول وأفضل الممارسات هو كتابتها بأحرف صغيرة.

إليك نظرة أقرب على وسوم البداية والإغلاق لعنصر الفقرة فقط:

```html
<p>
```

```html
</p>
```

ما يميز وسم البداية عن وسم الإغلاق هو الشرطة المائلة (`/`) التي توضع مباشرة بعد القوس الزاوية اليسار في وسم الإغلاق. بعض عناصر HTML لا تحتوي على وسم إغلاق. تُعرف هذه بالعناصر الفارغة.

إليك مثال على عنصر صورة وهو عنصر فارغ:

```html
<img>
```

لاحظ أن عنصر الصورة هذا لا يحتوي على وسم إغلاق ولا يحتوي على أي محتوى. لا يمكن للعناصر الفارغة أن تحتوي على محتوى ولها وسم بداية فقط.

أحيانًا سترى العناصر الفارغة التي تستخدم `/` قبل `>` هكذا:

```html
<img />
```

بينما يختار العديد من منسقي الشفرة مثل Prettier تضمين `/` في العناصر الفارغة، تنص مواصفات HTML على أن وجود `/` "لا يجعل وسم البداية وسمًا ذاتي الإغلاق بل هو غير ضروري ولا يؤثر بأي شكل".

في تطوير العالم الحقيقي، سترى كلا الشكلين لذا من المهم أن تكون على دراية بهما.

إذا أردت عرض صورة، ستحتاج إلى تضمين بعض الخصائص داخل عنصر الصورة. الخاصية هي قيمة خاصة تُستخدم لضبط سلوك عنصر HTML.

إليك مثال على عنصر صورة مع خاصية `src`. حدّث قيمة خاصية `src` إلى `"https://cdn.freecodecamp.org/curriculum/cat-photo-app/cats.jpg"` وسترى الصورة تتغير إلى قطتين نائمتين بسلام.

:::interactive_editor

```html
<img src="https://cdn.freecodecamp.org/curriculum/cat-photo-app/running-cats.jpg" />
```

:::

تُستخدم خاصية `src` لتحديد موقع تلك الصورة. بالنسبة لعناصر الصورة، من الأفضل تضمين خاصية أخرى تسمى خاصية `alt`. تُستخدم خاصية `alt` لتوفير نص وصفي قصير للصور.

إليك مثال على عنصر صورة مع خاصيتي `src` و`alt`. جرّب كسر الصورة بتحديث قيمة `src` إلى `"https://.freecodecamp.org/curriculum/cat-photo-app/cats.jpg"`. سترى اختفاء الصورة ويظهر فقط نص `alt`.

:::interactive_editor

```html
<img src="https://cdn.freecodecamp.org/curriculum/cat-photo-app/cats.jpg" alt="Two tabby kittens sleeping together on a couch." />
```

:::

قد تتساءل هل HTML وحدها كافية لبناء موقع إلكتروني؟ الجواب: يعتمد. إذا كنت تبني مشروع ممارسة صغير يعرض نصوصًا وصورًا فقط، قد تكون HTML وحدها كافية. لكن إذا كنت تنشئ موقعًا حديثًا واحترافيًا، ستحتاج إلى HTML وCSS وJavaScript.

HTML للمحتوى والبنية. CSS للتصميم. JavaScript لإضافة التفاعلية إلى صفحات الويب. تشبيه جيد هو مقارنة HTML وCSS وJavaScript بمبنى كامل.

تمثل HTML الكتل والخرسانة والحديد التي تشكل الجدران. إنها الأساس الذي يجعل المبنى قويًا. تمثل CSS التصميم الداخلي والخارجي الذي يجعل المنزل جميلًا. تمثل JavaScript نظام الكهرباء والمياه الذي يضمن وصولًا مستمرًا للمياه والكهرباء.
