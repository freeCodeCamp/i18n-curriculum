# --interactive--

Lernen wir einige der häufigen Probleme kennen, die beim Stylen spezieller Eingabeelemente wie den `datetime-local` und `color` Eingaben auftreten.

Diese speziellen Eingabetypen basieren auf komplexen Pseudoelementen, um Dinge wie Datums- und Farbauswahlen zu erstellen. Das stellt eine große Herausforderung beim Stylen dieser Eingaben dar. Eine Schwierigkeit ist, dass das Standardstyling vollständig vom Browser abhängt, sodass CSS, das den Picker in einem Browser richtig aussehen lässt, in einem anderen Browser ein ganz anderes Ergebnis liefern kann.

Hier ist ein Beispiel für eine Farbeingabe:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<form>
  <label for="favorite-color">Pick your favorite color:</label>
  <input type="color" id="favorite-color" name="favorite-color">
</form>
```

```css
input {
  padding: 8px 12px;
  margin: 8px 0;
  border-radius: 6px;
  border: 1px solid #ccc;
}

input[type="color"] {
  width: 60px;
  height: 40px;
  padding: 0;
  border: 2px solid #555;
  border-radius: 4px;
  cursor: pointer;
}

```

:::

Ein weiteres Problem kann die Komplexität des Pseudoelements sein. Betrachten Sie den Datumsselektor; hier gibt es viele bewegliche Teile, und die komplexe Struktur des Pseudoelements kann eine große Herausforderung darstellen, um das Styling auf die richtigen Bereiche anzuwenden.

Hier ist ein Beispiel für eine Datumseingabe:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<form>
  <label for="birthdate">Select your birthdate:</label>
  <input type="date" id="birthdate" name="birthdate">
</form>
```

```css
input {
  padding: 8px 12px;
  margin: 8px 0;
  border-radius: 6px;
  border: 1px solid #ccc;
}

input[type="date"] {
  padding: 6px 10px;
  border: 2px solid #555;
  border-radius: 4px;
  font-size: 14px;
  cursor: pointer;
}

input[type="date"]::-webkit-calendar-picker-indicator {
  background-color: #4CAF50;
  color: white;
  border-radius: 4px;
  cursor: pointer;
}

```

:::

Natürlich besteht bei diesen komplexen Elementen auch das Risiko, dass wichtige Funktionen versehentlich verloren gehen, wenn Sie sie manuell stylen. Sie könnten nicht nur wichtige Indikatoren wie den Fokuszustand oder das ausgewählte Element verlieren, sondern möglicherweise den Selektor komplett beschädigen.

Aus diesen Gründen verlassen sich viele Entwickler stattdessen ganz auf JavaScript-Bibliotheken oder eigene Komponenten anstelle der eingebauten Browser-Komponenten.
