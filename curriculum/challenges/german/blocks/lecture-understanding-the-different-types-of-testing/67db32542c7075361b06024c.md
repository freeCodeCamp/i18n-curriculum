# --description--

Wenn eine Anwendung weiter wächst, kommt es häufig vor, dass neue Änderungen unbeabsichtigt bestehende Funktionalitäten beeinträchtigen. Dies wird als Regression bezeichnet.

Um diese Probleme frühzeitig zu erkennen, können Sie Regressionstests verwenden. Regressionstesten bedeutet, funktionale Tests erneut gegen Teile Ihrer Anwendung auszuführen, um sicherzustellen, dass alles weiterhin wie erwartet funktioniert. Wenn Sie Bugfixes, Verbesserungen oder sogar Updates an Ihrer Konfiguration implementiert haben, ist es ratsam, Regressionstests durchzuführen.

Es gibt verschiedene Tools, mit denen Sie Regressionstests durchführen können. Dazu gehören Puppeteer, Playwright, Selenium und Cypress.

Es gibt auch einige Testtechniken, die Sie beim Regressionstesten kennen sollten. Die erste heißt Unit Regression Testing. Dabei haben Sie eine Liste von Elementen, die jedes Mal getestet werden müssen, wenn größere Änderungen oder Fehlerbehebungen in die Anwendung implementiert werden.

Eine weitere Testtechnik heißt Partial Regression Testing. Dabei handelt es sich um gezielte Ansätze, um sicherzustellen, dass neue Änderungen keine bestimmten Aspekte der Anwendung beeinträchtigt haben.

Schließlich gibt es das Complete Regression Testing. Dies ist die zeitaufwändigste und detaillierteste Option, bei der alle Funktionalitäten betrachtet werden, die von den jüngsten Änderungen im Code betroffen sind, und Tests dagegen ausgeführt werden.

Es ist wichtig zu beachten, dass Retesting und Regressionstesting unterschiedlich sind. Retesting wird verwendet, um bekannte Probleme zu überprüfen und sicherzustellen, dass sie behoben wurden, während Regressionstesting nach unbekannten Problemen sucht, die durch jüngste Änderungen im Code entstanden sein könnten.
