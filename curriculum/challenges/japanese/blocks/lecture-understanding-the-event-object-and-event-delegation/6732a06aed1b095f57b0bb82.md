# --interactive--

`change` イベントは、ユーザーが特定の入力要素の値を変更したときに発生する特別なイベントです。具体的には：

- チェックボックスがオンまたはオフにされたとき。

- ラジオボタンが選択されたとき。

- 日付ピッカーやドロップダウンメニューのようなものからユーザーが選択したとき。

- ユーザーが値を変更した後に入力フィールドのフォーカスが外れたとき（ユーザーが次のフィールドにタブ移動したり、フォームの外をクリックした場合）。

- ユーザーがテキストを入力した後にエンターキーを押すなど、値を確定したとき。

`change` イベントは、ユーザーが入力中に発生しないことに注意してください。`change` イベントは、ユーザーが別の要素にフォーカスを移した後にのみ発生します。

以下は、ドロップダウンメニューで `change` イベントを使う例です：

:::interactive_editor

```html
<select id="select-menu">
  <option value="option1">Option 1</option>
  <option value="option2">Option 2</option>
  <option value="option3">Option 3</option>
</select>
<script src="index.js"></script>
```

```js
const selectMenu = document.getElementById("select-menu");
selectMenu.addEventListener("change", (event) => {
  console.log(`You selected: ${event.target.value}`);
});
```

:::

ユーザーがドロップダウンメニューから異なるオプションを選択するたびに、`change` イベントが発生し、選択された値がコンソールに記録されます。

`change` イベントも `Event` オブジェクトを生成しますが、ほとんどの他のイベントとは異なりカスタム実装は生成されません。アクセスできるプロパティやメソッドは、基本の `Event` オブジェクトのものだけです。

これは、専用の `input` オブジェクトを生成する `InputEvent` イベントとは異なります。change イベントは他にもいくつか違いがあります。例えば、ユーザーがフィールドに文字を入力したときは `input` イベントが発生します。

これらの違いは重要で、イベントが発生するタイミングで混乱しないように覚えておいてください。
