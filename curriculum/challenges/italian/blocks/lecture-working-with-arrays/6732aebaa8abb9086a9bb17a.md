# --interactive--

Nella lezione precedente, hai iniziato a lavorare con gli array e a consultare i diversi elementi degli array. Ecco un promemoria su come consultare il secondo elemento di un array:

:::interactive_editor

```js
const fruits = ["apple", "banana", "cherry"];

console.log(fruits[1]); // "banana"
```

:::

Poiché gli array hanno indice basato su zero, il primo elemento si trova all’indice `0`, il secondo elemento si trova all’indice `1`, e così via. È importante notare che se provi a consultare un indice che non esiste nell’array, JavaScript restituirà `undefined`. 

:::interactive_editor

```js
let fruits = ["apple", "banana", "cherry"];
console.log(fruits[3]); // undefined
```

:::

In questo esempio, non esiste l’indice `3` per l’array `fruits`. Quindi il log mostrerà `undefined`. Ora, vediamo come aggiornare gli elementi in un array. Puoi aggiornare un elemento assegnando un nuovo valore a un indice specifico. 

:::interactive_editor

```js
let fruits = ["apple", "banana", "cherry"];
fruits[1] = "blueberry";
console.log(fruits); // ["apple", "blueberry", "cherry"]
```

:::

In questo esempio, abbiamo sostituito `banana` con `blueberry` all’indice `1`. Questo metodo ti permette di modificare qualsiasi elemento nell’array, purché tu conosca il suo indice. Puoi anche aggiungere nuovi elementi a un array assegnando un valore a un indice che ancora non esiste:

:::interactive_editor

```js
let fruits = ["apple", "banana", "cherry"];
fruits[3] = "date";
console.log(fruits); // ["apple", "banana", "cherry", "date"]
```

:::

Tuttavia, fai attenzione quando lo fai. Se assegni un valore a un indice molto più grande della lunghezza attuale dell’array, creerai elementi undefined per gli indici intermedi, il che può portare a comportamenti inattesi. Man mano che continuerai a lavorare con JavaScript, scoprirai che questi modi di consultare e aggiornare gli elementi di un array sono fondamentali per molte attività di programmazione. Che tu stia costruendo una semplice lista di cose da fare o elaborando strutture dati complesse, queste competenze saranno preziose.
