# --interactive--

Michoro ya CSS inakuwezesha kuunda athari za kuona zinazobadilika kwa urahisi kwenye kurasa za mtandao bila hitaji la JavaScript au programu ngumu. Zinatoa njia ya kuhamisha vipengele kwa laini kati ya mitindo tofauti kwa muda uliobainishwa.

Kwa msingi wake, mchoro wa CSS una sehemu kuu mbili: kanuni ya `@keyframes` na sifa ya animation.

Kanuni ya `@keyframes` inaeleza hatua na mitindo ya mchoro. Inaeleza mitindo ambayo kipengele kinapaswa kuwa nayo katika sehemu mbalimbali za mchoro.

Hapa kuna mfano:

```css
@keyframes slide-in {
  0% {
    transform: translateX(-100%);
  }
  100% {
    transform: translateX(0);
  }
}
```

Kanuni hii ya `@keyframes`, yenye jina `slide-in`, inaeleza mchoro unaosogeza kipengele kutoka kushoto kwenda kulia. Asilimia zinawakilisha maendeleo ya mchoro, ambapo `0%` ni mwanzo na `100%` ni mwisho.

Kitendakazi cha `translateX` katika mchoro wako wa `@keyframes` kinadhibiti nafasi ya usawa ya kipengele kinapokuwa kinajitokeza.

Ili kutumia mchoro huu kwa kipengele, unatumia sifa ya `animation`. Mfano huu pia unarudia mchoro bila kikomo ili uone jinsi unavyofanya kazi (utahitaji kuwezesha mhariri wa mwingiliano ili kuona maonyesho):

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<div class="sliding-element">Hello, I slide in!</div>
```

```css
@keyframes slide-in {
  0% {
    transform: translateX(-100%);
  }
  100% {
    transform: translateX(0);
  }
}

.sliding-element {
  animation: slide-in 2s ease-out infinite;
}
```

:::

Hii inatumia mchoro wa `slide-in` kwa kipengele kwa muda wa sekunde 2 na kitendakazi cha `ease-out` cha wakati.

Sifa ya `animation` ni kifupi kwa sifa kadhaa za mtu mmoja mmoja:

`animation-name` ambayo inaeleza kanuni ya `@keyframes` itakayotumika.

`animation-duration` ambayo inaweka muda ambao mchoro unapaswa kuchukua kukamilika.

`animation-timing-function` ambayo inaeleza jinsi mchoro unavyosonga kwa muda - kama vile `ease`, `linear`, `ease-in-out`.

`animation-delay` ambayo inaeleza kuchelewa kabla ya mchoro kuanza.

`animation-iteration-count` ambayo inaweka ni mara ngapi mchoro unapaswa kurudiwa.

`animation-direction` ambayo inaamua kama mchoro unapaswa kuchezwa mbele, nyuma, au kubadilishana.

`animation-fill-mode` ambayo inaeleza jinsi kipengele kinapaswa kupambwa kabla na baada ya mchoro.

`animation-play-state` ambayo inakuwezesha kusitisha na kuendelea na mchoro.

Unaweza kutumia sifa hizi moja moja kwa udhibiti wa kina zaidi:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<div class="complex-animation">Watch my colors change!</div>
```

```css
.complex-animation {
  animation-name: color-change;
  animation-duration: 3s;
  animation-timing-function: linear;
  animation-delay: 1s;
  animation-iteration-count: infinite;
  animation-direction: alternate;
}

@keyframes color-change {
  0% {
    background-color: red;
  }
  50% {
    background-color: blue;
  }
  100% {
    background-color: green;
  }
}
```

:::

Hii huunda mchoro unaoendelea kubadilisha rangi ya nyuma ya kipengele kati ya nyekundu, buluu, na kijani.

Michoro ya CSS inaweza kuanzishwa na matukio mbalimbali, kama vile kuonyesha kipengele kwa kubeba kipanya juu yake:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<button class="button">Hover over me!</button>
```

```css
.button {
  background-color: blue;
  transition: background-color 0.3s;
}

.button:hover {
  background-color: red;
}
```

:::

Ingawa mfano huu unatumia sifa ya `transition`, ambayo ni rahisi kwa athari za msingi, unaonyesha jinsi CSS inavyoweza kuunda vipengele vinavyoshirikiana na kuwa na michoro.

Ni muhimu kutambua kwamba ingawa michoro ya CSS ni yenye nguvu, inapaswa kutumika kwa kiasi. Matumizi ya kupita kiasi ya michoro yanaweza kusababisha utendaji mbaya na inaweza kuwa kikwazo au tatizo kwa watumizi wenye mahitaji maalum ya ufikikaji. Daima zingatia kutoa chaguzi za kupunguza au kuzima michoro kwa watumizi wanaopendelea mwendo mdogo.

Michoro ya CSS hutoa njia ya kuunda uzoefu wa mtandao unaovutia na unaoshirikisha bila kutegemea JavaScript. Kwa kuelewa kanuni za `@keyframes` na sifa za `animation`, unaweza kuleta miundo yako ya mtandao kuwa hai kwa utendaji mzuri na upatikanaji.
