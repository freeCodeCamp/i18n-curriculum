# --description--

前のレッスンでは、Fetch APIとは何か、そしてその使い方を見ました。このレッスンでは、Fetch APIの`GET`、`POST`、`PUT`、`DELETE`のHTTPメソッドについて説明します。

まず、最も一般的なHTTPメソッドである`GET`メソッドから始めましょう。これはサーバーからデータを取得するために使われます。`fetch()`をメソッド指定なしで使うと、デフォルトで`GET`になります。

```js
fetch('https://api.example.com/data')
```

このコードでは、`GET`に対して`https://api.example.com/data`リクエストをしています。ここで注意してほしいのは、このデータを直接使うことはできず、レスポンスをJSON形式に変換する必要があるということです。そうして初めて、プロジェクトのどこでも使えるようになります。以下はその例です。

```js
fetch('https://api.example.com/data')
  .then(response => response.json())
  .then(data => console.log(data))
```

このコードでは、Fetch APIからのレスポンスはpromiseであり、レスポンスをJSON形式に変換するために`.then`ハンドラを使っています。前のレッスンで、`Promise`は非同期処理の最終的な完了（または失敗）とその値を表すオブジェクトであることを学びました。

promiseの値は、promiseが作成された時点ではわかりません。非同期処理が完了した時に初めてわかります。fetch呼び出しに2つの`.then`ハンドラをチェーンすると、これは次のレッスンで学ぶpromiseチェーンと呼ばれるものです。

これまではサーバーからリソースを取得してきましたが、サーバーにデータを送ることもできるのをご存じですか？`POST`メソッドは、サーバーにデータを送ってリソースを作成するために使われます。以下はサーバーにデータを作成するために使う`POST`メソッドの例です。

```js
fetch('https://api.example.com/users', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({
    name: 'John Doe',
    email: 'john@example.com'
  })
})
```

この例では、新しいユーザーを作成するために`POST`リクエストを送っています。メソッドを`POST`に指定し、適切なヘッダーを設定し、送信したいデータを含む`body`を追加しています。bodyは文字列である必要があるため、オブジェクトをJSON文字列に変換するために`JSON.stringify()`を使っています。

`PUT`メソッドは、サーバーの既存リソースを更新するために使われます。以下はその例です。

```js
fetch('https://api.example.com/users/45', {
  method: 'PUT',
  headers: {
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({
    name: 'John Smith',
    email: 'john@example.com'
  })
})
```

この例では、URLの最後に`45`があるのがわかります。これは通常、更新しようとしているデータを識別するための一意のIDとして使われます。コードでは`PUT`メソッドを使い、更新するデータとして`body`を指定しています。

`DELETE`メソッドは、サーバーからリソースを削除するために使われます。以下はその例です。

```js
fetch('https://api.example.com/users/45', {
  method: 'DELETE',
})
```

このコードでは、削除するデータを識別するためにURLの最後にIDを付けて`DELETE`メソッドを含めています。
