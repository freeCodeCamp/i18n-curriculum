# --description--

이전 수업들에서, 다음과 같이 React에서 작은 컴포넌트를 만드는 방법을 배웠습니다:

```jsx
function Greeting() {
  const developerName = "Jessica";
  return <h1>Hi {developerName}!</h1>;
}
```

이 컴포넌트를 다른 부모 컴포넌트나 루트 컴포넌트 안에 중첩할 수도 있습니다:

```jsx
function App() {
  return <Greeting />;
}

function Greeting() {
  const developerName = "Jessica";
  return <h1>Hi {developerName}!</h1>;
}
```

이 코드는 잘 실행되고 화면에 `Hi Jessica!` 결과를 보여주지만, 그렇게 유연한 컴포넌트는 아닙니다.

만약 Naomi, Tom, Oliver 같은 다른 이름을 표시하고 싶다면 어떻게 할까요? 이때 React 프로프스가 필요합니다. 프로프스는 properties의 줄임말로, 부모 컴포넌트가 자식 컴포넌트에 데이터를 전달하는 방법입니다. 프로프스는 문자열, 숫자, 불리언, 객체, 배열 등 어떤 타입도 될 수 있습니다.

앞서 예제를 업데이트해서 이제 `name` 프로프스를 받도록 해봅시다:

```jsx
function App() {
  return <Greeting name="Jessica" />;
}

export default App;

function Greeting(props) {
  console.log(props);
  return <h1>Hi {props.name}!</h1>;
}
```

`Greeting`라는 자식 컴포넌트에서는 이름 `props.name`를 하드코딩하는 대신 `"Jessica"`를 사용하고 있습니다. 또한 `props`를 콘솔에 출력하는데, 객체로 나타납니다.

그다음 부모 `App` 컴포넌트 안에서 `name` 프로프스에 값을 전달해 자식에게 내려보냅니다. 화면에 표시되는 결과는 이전과 같지만, 이제 더 유연한 컴포넌트를 만들었습니다.

이제 자식 컴포넌트를 여러 번 재사용하면서 매번 다른 이름을 전달할 수 있습니다:

```jsx
function App() {
  return (
    <>
      <Greeting name="Naomi" />
      <Greeting name="Tom" />
      <Greeting name="Jessica" />
      <Greeting name="Oliver" />
    </>
  );
}
```

프로프스에서 객체 구조 분해를 사용해 더 읽기 쉽게 만들 수도 있습니다. `Greeting` 컴포넌트를 이렇게 다시 작성할 수 있습니다:

```jsx
function Greeting({ name }) {
  return <h1>Hi {name}!</h1>;
}
```

이 코드는 같은 결과를 내지만, 컴포넌트가 어떤 프로프스를 받을지 더 명확하게 보여줍니다.

React는 `children`라는 특별한 프로프스도 제공합니다. 컴포넌트의 여는 태그와 닫는 태그 사이에 넣은 JSX는 `children` 프로프스로 컴포넌트에 전달됩니다.

예를 들어:

```jsx
function Card({ children }) {
  return <div className="card">{children}</div>;
}

function App() {
  return (
    <Card>
      <h2>Hello</h2>
    </Card>
  );
}
```

이 경우 `<Card>...</Card>` 안의 JSX가 `children` 프로프스가 됩니다. DOM에 렌더링된 결과는 다음과 같습니다:

```html
<div class="card">
  <h2>Hello</h2>
</div>
```

이 패턴은 컴포넌트 합성에 자주 사용되며, 컴포넌트가 다른 UI 요소들을 감쌀 때 쓰입니다.

때로는 전달해야 할 프로퍼티가 많을 수 있습니다. 하나씩 전달하는 대신, 객체로 변환한 후 전개 연산자(`...`)를 사용할 수 있습니다.

새 자식 컴포넌트 `DeveloperCard`의 예는 다음과 같습니다:

```jsx
function DeveloperCard({ name, age, country }) {
  return (
    <div className="developer-card">
      <h1>Developer: {name}</h1>
      <p>Age: {age}</p>
      <p>Country: {country}</p>
    </div>
  );
}
```

이 `DeveloperCard` 컴포넌트는 `name`, `age`, `country` 세 가지 프로프스를 받습니다.

부모 `App` 컴포넌트에서는 전개 구문을 사용해 객체의 모든 프로퍼티를 자식 컴포넌트에 개별 프로프스로 전달할 수 있습니다:

```jsx
function App() {
  const developerObj = {
    name: "Alice",
    age: 30,
    country: "USA",
  };

  return (
    <div className="App">
      <DeveloperCard {...developerObj} />
    </div>
  );
}
```

이 방법은 배열 안의 객체들을 다루거나 여러 프로퍼티 세트를 자식 컴포넌트에 전달할 때 특히 유용합니다. 예를 들어, 배열 안에 각기 다른 사람을 나타내는 동일한 구조의 개발자 객체 목록이 있을 수 있습니다.

배열에서 리스트를 렌더링하는 방법은 앞으로 수업에서 더 배우게 됩니다.

React에서 프로프스를 사용하면 컴포넌트를 더 유연하고 재사용 가능하게 만들어 복잡한 UI를 구축할 수 있습니다. 하지만 프로프스는 불변이라는 점을 기억하세요. 즉, 컴포넌트에 전달된 후에는 변경할 수 없습니다. 사용자 입력을 처리하고 데이터를 수정해야 한다면 상태(state)를 사용해야 합니다. 상태 관리는 앞으로 수업에서 더 배우게 됩니다.
