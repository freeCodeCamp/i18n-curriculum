# --description--

讓我們來了解 DOM 節點及其在 DOM 樹中的關係。

就像真實的樹有大枝和小枝以階層結構連接一樣，DOM 節點彼此之間也有直接和間接的關係。我們將使用此範例來說明這些關係：

```html
<!DOCTYPE html>
<html>
  <head>
    <title>DOM Tree Example</title>
  </head>
  <body>
    <h1>Heading 1</h1>
    <p>Paragraph 1</p>
    <ul>
      <li>List item 1</li>
      <li>List item 2</li>
    </ul>
  </body>
</html>
```

讓我們從最上層開始。DOM 樹的根節點是 `html` 元素。它是 HTML 文件中所有內容的最高層級容器。所有其他節點都是此根節點的後代。

然後，在根節點下方，我們會在階層體系中找到其他節點。父節點是包含其他元素的元素。子節點是被包含在另一個元素中的元素。

在此範例中，`body` 元素是段落元素的父元素，而段落元素是 `body` 元素的子元素。這些元素在 DOM 樹中以節點的形式表述，並具有相同的關係。

就像我們有父節點和子節點一樣，我們也有兄弟節點。兄弟節點是指共享相同父節點的元素。在我們的範例中，兩個列表項目元素共享相同的無序列表父節點，因此它們是兄弟節點。 

同樣地，`h1` 和段落元素是兄弟元素，因為它們共用相同的父元素 `body`。

我們在階層體系的不同層級之間也有間接關係。後代節點是包含在另一個元素內的元素，無論是直接還是間接。如果可以透過從祖先節點往下走 DOM 樹來到達某個元素，該元素即被視為另一個元素的後代。

在我們的範例中，列表項目 `li` 元素是 `body` 元素的後代，因為它們間接包含在其中。

祖先節點是 DOM 樹階層體系中位於另一個元素之上的元素。在我們的範例中，`body` 元素是列表項目 `li` 元素的祖先。

理解這些關係對於使用 JavaScript 操作和瀏覽 DOM 樹至關重要。
