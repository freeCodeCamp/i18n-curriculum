# --interactive--

تساعدك المعاملات المنطقية الثنائية على تقييم تعبيرين وإرجاع نتيجة بناءً على صحتهما. دعنا نلقي نظرة على أكثر ثلاثة معاملات منطقية ثنائية شيوعًا: المعامل المنطقي AND، والمعامل المنطقي OR، ومعامل الدمج الصفري.

يمثل المعامل المنطقي AND بواسطة علامتي العطف (`&&`). يتحقق مما إذا كان كلا العاملين صحيحين ويرجع نتيجة. إذا كان كلا العاملين صحيحين، فإنه يرجع القيمة الثانية، أي القيمة الموجودة على اليمين:

:::interactive_editor

```js
const result = true && 'hello';

console.log(result); // hello
```

:::

في المثال أعلاه، يتم تسجيل النص `hello` في وحدة التحكم لأن كلا العاملين هما `true`. إذا كان أي من العاملين قيمة خاطئة، فإنه يرجع تلك القيمة الخاطئة:

:::interactive_editor

```js
const result = 0 && 3;

console.log(result); // 0
```

:::

بما أن `0` قيمة خاطئة، يتم تسجيل الرقم `0` في وحدة التحكم. وإذا كان كلا العاملين قيمتين خاطئتين، فإنه يرجع أول قيمة خاطئة:

:::interactive_editor

```js
const result = false && 0;

console.log(result); // false
```

:::

بما أن `false` قيمة خاطئة، يتم تسجيل `false` في وحدة التحكم. المعامل المنطقي AND مفيد عندما تريد التحقق من شروط متعددة والتأكد من أن جميعها صحيحة قبل المتابعة. إليك مثالًا:

:::interactive_editor

```js
if (2 < 3 && 3 < 4) {
 console.log('The if block runs'); 
} else {
 console.log('The else block runs');
} 
```

:::

في الشرط، بما أن `2` أقل من `3` و`3` أقل من `4`، سيتم تسجيل الجملة `The if block runs` في وحدة التحكم.

يتحقق المعامل المنطقي OR مما إذا كان أحد العاملين على الأقل صحيحًا. إذا كان العامل الأول صحيحًا، فإنه يرجع تلك القيمة:

:::interactive_editor

```js
const result = 'This is truthy' || false;

console.log(result); // This is truthy
```

:::

إذا كان العامل الأول قيمة خاطئة لكن العامل الثاني صحيح، سيتم تسجيل القيمة الثانية في وحدة التحكم:

:::interactive_editor

```js
const result = 0 || 'This is truthy';

console.log(result); // This is truthy
```

:::

من الشائع استخدام المعامل المنطقي OR في عبارات `if/else` بهذا الشكل:

:::interactive_editor

```js
let userInput;

if (userInput || 'Guest') {
 console.log('A user is present');
} else {
 console.log('No user detected');
}
```

:::

بما أننا لم نُعيّن قيمة للمتغير `userInput`، فهو حاليًا `undefined`. يتحقق الشرط في عبارة `if` مما إذا كان المتغير `userInput` أو السلسلة النصية `Guest` صحيحين. بما أن السلسلة النصية `Guest` صحيحة في سياق القيمة المنطقية، سيتم تسجيل النص `A user is present` في وحدة التحكم.

معامل الدمج الصفري أكثر تعقيدًا من المعاملين المنطقيين OR وAND. يُمثّل بعلامتي استفهام مزدوجتين (`??`)، ويساعد في الحالات التي تريد فيها إرجاع قيمة فقط إذا كانت القيمة الأولى `null` أو `undefined`. إليك مثالًا على العمل مع معامل الدمج الصفري:

:::interactive_editor

```ts
const result = null ?? 'default';

console.log(result); // default
```

:::

بما أن `null` قيمة صفريّة، سيتم تسجيل السلسلة النصية `default` في وحدة التحكم. معامل الدمج الصفري مفيد جدًا في الحالات التي تكون فيها القيمتان `null` أو `undefined` فقط هي التي يجب أن تُفعّل النظام البديل أو القيمة الافتراضية. إليك مثالًا على التعامل مع إعدادات تفضيلات المستخدم:


:::interactive_editor

```ts
const userSettings = {
 theme: null,
 volume: 0,
 notifications: false,
};

let theme = userSettings.theme ?? 'light';
console.log(theme); // light
```

:::

في المثال أعلاه، لدينا كائن يسمى `userSettings` يحتوي على خصائص `theme` و`volume` و`notifications`. نصل إلى الخاصية `theme` باستخدام ترميز النقطة مثل `userSettings.theme`. ستتعلم المزيد عن كيفية العمل مع الكائنات في درس مستقبلي. بما أن خاصية `theme` للمستخدم حاليًا مضبوطة على `null`، سيتم تسجيل السلسلة النصية `light` في وحدة التحكم.
