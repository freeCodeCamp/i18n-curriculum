# --interactive--

호이스팅은 JavaScript에서 초보자를 자주 혼란스럽게 하는 동작이지만, 이를 이해하면 코드에서 미묘한 버그를 피하는 데 도움이 됩니다. 간단히 말해 호이스팅은 코드가 실행되기 전에 컴파일 단계에서 선언을 각자의 범위 최상단으로 끌어올리는 JavaScript의 기본 동작입니다.

호이스팅을 이해하려면 JavaScript가 컴파일 단계와 실행 단계, 두 단계로 실행된다는 점을 아는 것이 중요합니다. 컴파일 단계에서는 JavaScript 엔진이 코드를 훑으며 변수와 함수에 메모리 공간을 할당합니다. 바로 이때 호이스팅이 작동합니다.

변수 호이스팅부터 시작해 보겠습니다. `var` 키워드로 변수를 선언하면 JavaScript는 선언을 해당 범위 최상단으로 호이스팅합니다. 하지만 중요한 점은 선언만 호이스팅되고 초기화는 호이스팅되지 않는다는 것입니다. 즉, 변수를 선언하기 전에 코드에서 사용할 수는 있지만, 실제 값을 할당하기 전까지는 `undefined` 값을 가집니다.

:::interactive_editor

```js
console.log(x); // undefined
var x = 5;
console.log(x); // 5
```

:::

이 코드에서 `x`를 선언 전에 사용했지만 오류가 발생하지 않고 대신 `undefined` 값을 얻습니다. 이는 JavaScript가 선언 `var x`만 범위 최상단으로 호이스팅하고 초기화 `x = 5`는 호이스팅하지 않기 때문입니다. 마치 코드가 다음과 같이 다시 작성된 것과 같습니다:

:::interactive_editor

```js
var x;
console.log(x); // undefined
x = 5;
console.log(x); // 5
```

:::

함수 호이스팅은 약간 다르게 작동합니다. `function` 선언 구문으로 함수를 선언하면 함수 이름과 함수 본문 전체가 호이스팅됩니다. 따라서 코드에서 함수를 선언하기 전에 호출할 수 있습니다. 함수 호이스팅 예시는 다음과 같습니다:

:::interactive_editor

```js
sayHello(); // "Hello, World!"

function sayHello(){
  console.log("Hello, World!");
}
```

:::

이 경우 `sayHello()`를 선언 전에 호출할 수 있는데, 이는 함수 전체가 범위 최상단으로 호이스팅되기 때문입니다.

ES6에서 도입된 `let`와 `const` 선언은 호이스팅이 다르게 작동한다는 점을 주의하세요.

```js
console.log(y); // Throws a ReferenceError
let y = 10;
```

이 선언들은 호이스팅되지만 초기화되지 않고, 코드에서 실제 선언 전에 접근할 수 없습니다. 이를 흔히 일시적 사각지대(temporal dead zone)라고 부릅니다.

호이스팅을 이해하면 더 깔끔하고 예측 가능한 코드를 작성할 수 있습니다. 하지만 호이스팅에 의존하면 코드가 읽기 어렵고 유지보수가 힘들어질 수 있습니다. 모범 사례로, 변수는 범위 최상단에, 함수는 사용 전에 선언하는 것이 권장됩니다. 이렇게 하면 코드 동작이 더 명확해지고, 미래의 자신을 포함한 다른 사람이 이해하기 쉬워집니다.
