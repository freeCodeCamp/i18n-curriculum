# --description--

Crypto هي وحدة أساسية أخرى مدمجة في Node.js. تشمل أدوات لأشياء مثل التجزئة، التشفير، فك التشفير، وإنشاء التواقيع الرقمية، وكلها تُستخدم لحماية المعلومات الحساسة والحفاظ على أمان تطبيقك.

مع ذلك، توفر لك وحدة `crypto` لبنات بناء منخفضة المستوى، وليست أمانًا جاهزًا للتشغيل. كتابة شفرة التشفير أو التوثيق بنفسك قد تكون غير آمنة إذا لم تكن حذرًا. في معظم الحالات، من الأفضل استخدام مكتبات مجربة مثل `bcrypt` لتجزئة كلمات المرور أو `jsonwebtoken` (JWT) للتعامل مع تسجيل الدخول والرموز.

مع ذلك، من المفيد فهم كيفية عمل بعض الطرق في وحدة crypto.

لاستخدام هذه الطرق، تحتاج إلى استيراد وحدة `crypto`:

```js
const crypto = require("crypto");
```

بعض الطرق تُستخدم لأغراض تحويل البيانات، مثل تلك الخاصة بتجزئة وتشفير البيانات، وبعضها الآخر لإدارة المفاتيح والأمان، مثل توليد القيم العشوائية وإنشاء الأسرار.

لنلقِ نظرة أولاً على الطرق الخاصة بتجزئة وتشفير البيانات.

تُنشئ الطريقة `createHash()` كائن تجزئة باستخدام خوارزميات مثل `sha256`، `sha512`، أو `md5`. إنها عملية أحادية الاتجاه، فلا يمكنك عكسها.

تُستخدم `createHash()` لتجزئة كلمات المرور وتمييز الملفات. لاستخدامها، تمرر الخوارزمية، تستخدم الطريقة `update()` لإدخال البيانات، وأخيرًا تستخدم `digest()` مع ترميز للحصول على قيمة التجزئة:

```js
const crypto = require("crypto");

const hashedPassword = crypto
  .createHash("sha256")
  .update("myStrongPassword")
  .digest("hex");

console.log("createHash result:", hashedPassword);
// createHash result: f92c9cfa0ead1bcec05ca75888a4074ba994ad237e5e2a8c7cc6a620378c061d
```

تقوم `createHmac()` بنفس عمل `createHash()` تقريبًا، لكنها ترتقي للأعلى بقبول مفتاح سري، بحيث يمكن فقط لمن يملك هذا المفتاح التحقق من التجزئة. هي مثالية للتوثيق والتحقق من سلامة البيانات:

```js
const crypto = require("crypto");

const hashedMessage = crypto
  .createHmac("sha256", "secretkey")
  .update("important-secret-message")
  .digest("hex");

console.log("createHmac result:", hashedMessage);
// createHmac result: da48d6f026b6036286b1fb872c63264130d5cc4271f3a213bb6ddca5a023e77e
```

تُشفّر وتُفكّك الطرق `createCipheriv()` و `createDecipheriv()` البيانات. كلاهما يأخذ خوارزمية، مفتاح، و`iv`، وهو كتلة من البيانات العشوائية أو الفريدة تُستخدم في بداية عملية التشفير:

```js
createCipheriv(algorithm, key, iv);
createDecipheriv(algorithm, key, iv);
```

لفك تشفير البيانات، يجب أن يكون المفتاح نفسه، وإلا ستفشل عملية فك التشفير، وستحصل إما على خطأ أو على بيانات غير مقروءة بدلاً من الرسالة الأصلية.

إليك الطريقتان أثناء العمل:

```js
const crypto = require("crypto");

// A key must match the algorithm length. Here AES-256 is 32 bytes
const key = Buffer.from("12345678901234567890123456789012");

// A fixed IV, 16 bytes for AES
const iv = Buffer.from("1234567890123456");

const cipher = crypto.createCipheriv("aes-256-cbc", key, iv);

let encrypted = cipher.update("Hello campers!", "utf8", "hex");
encrypted += cipher.final("hex");

console.log("Encrypted data:", encrypted);
// Encrypted data: 4ee93aa398ab44e3540e4a67ca96bc8c

// Decrypt the "Hello campers!" message
const decipher = crypto.createDecipheriv("aes-256-cbc", key, iv);
let decrypted = decipher.update(encrypted, "hex", "utf8");
decrypted += decipher.final("utf8");

console.log("Decrypted data:", decrypted);
// Decrypted data: Hello campers!
```

طريقة أخرى في crypto لتحويل البيانات هي `sign()` و `verify()`.

تُنشئ `sign()` توقيعًا رقميًا من بعض البيانات باستخدام مفتاح خاص. يثبت هذا التوقيع أن البيانات جاءت من حامل المفتاح الخاص ولم يتم العبث بها.

ثم تتحقق `verify()` من هذا التوقيع، وتفشل إذا لم تتطابق البيانات أو التوقيع.

الآن، لننظر إلى طرق crypto لتوليد القيم العشوائية وإنشاء الأسرار.

تأخذ `randomBytes()` حجمًا (`size`) وتولد رموزًا مؤمنة تشفيرًا. هذا يجعلها جيدة لتوليد معرفات UUID (معرفات فريدة عالمياً). بالإضافة إلى ذلك، هي بديل جيد لـ `Math.random()`، الذي ليس آمنًا للرموز والمفاتيح.

```js
console.log("Random Bytes:", crypto.randomBytes(16));
// Random Bytes: <Buffer 01 88 aa 1e 2c 38 48 39 26 e1 6b a9 d8 c5 ed 49>
```

الإخراج هو مخزن مؤقت افتراضيًا. كما تعلمت في درس وحدة Buffer، يمكنك تحويل هذا المخزن المؤقت إلى سلسلة نصية باستخدام الطريقة `toString()`:

```js
console.log("Random Bytes:", crypto.randomBytes(16).toString("hex"));
// Random Bytes: a6154ef5a296fa176ad0f332bd94d712
```

الحجة `'hex'` في `toString('hex')` هنا تخبر Node بترميز البيانات الثنائية من المخزن المؤقت كسلسلة نصية سداسية عشرية.

تأخذ الطريقة `randomInt()` قيم `min` و `max` وتولد عددًا صحيحًا عشوائيًا مؤمنًا بينهما. هي مفيدة لأكواد التحقق (OTP) والاختيار العشوائي.

```js
console.log("Random Int:", crypto.randomInt(0, 100)); // 89
```

مرة أخرى، التحسين على `Math.random()` و `Math.floor()` هو أن الطريقة تستخدم عشوائية مؤمنة تشفيرًا تحت الغطاء، فلا يمكن للمهاجمين التنبؤ بالعدد العشوائي الناتج.

طريقة أخرى هي `createSecretKey()`. تأخذ مخزنًا مؤقتًا وتولد بايتًا خامًا ملفوفًا في `KeyObject`:

```js
const crypto = require("crypto");

const secret = crypto.createSecretKey(crypto.randomBytes(32));
console.log(secret); // SecretKeyObject [KeyObject] { [Symbol(kKeyType)]: 'secret' }
```

يمكنك بعد ذلك استخدام طريقة `export` لإرسال ذلك `KeyObject`:

```js
const secret = crypto.createSecretKey(crypto.randomBytes(32));
console.log(secret.export());
// <Buffer 53 06 a1 c7 75 69 8b 38 8b a4 b2 f7 1b bc b8
// ae e2 d1 bf 67 af 1a 6a 0a 6e a0 29 62 bb 52 52 32>
```

وأخيرًا، استخدم `toString()` على المخزن المؤقت لرؤية التمثيل النصي له:

```js
const secret = crypto.createSecretKey(crypto.randomBytes(32));
console.log(secret.export().toString('hex'));
// 32dfe5917668580160986f1623bf8152913329c71163be9c3404a110cd78efd6
```

بالإضافة إلى هذه، هناك:

- `createPublicKey()` و `createPrivateKey()` التي تتيح لك العمل مع المفاتيح المولدة في مكان آخر
- `createDiffieHellman()` لطرفين لتوليد سر مشترك دون إرسال السر مباشرة
- `Certificate()` للعمل مع الشهادة المستخدمة في HTTPS، بحيث يمكنك تحليل، تصدير، والتحقق من محتويات الشهادة
