# --interactive--

검색 입력, 체크박스, 라디오 버튼의 스타일 문제를 해결하기 위해 `appearance: none`를 언제 사용해야 하는지 배워보세요.

브라우저는 많은 요소에 기본 스타일을 적용합니다. 입력 요소의 경우 CSS로 스타일을 지정하는 데 제한이 느껴질 수 있습니다. 그래서 기본 요소의 일부를 숨기고 직접 스타일을 만들기 위해 `appearance: none`를 사용하고 싶을 수 있습니다. 예를 들어, 체크박스 입력의 기본 체크박스를 숨기면 상태를 보여주기 위해 초록색 체크 표시나 빨간 X 같은 사용자 지정 표시기를 사용할 수 있습니다. 검색 입력의 경우 WebKit 기반 브라우저는 기본 검색 아이콘과 취소 버튼을 표시합니다. 이를 숨기면 모든 브라우저에서 나타나는 자신만의 표시기를 만들 수 있습니다.

다음은 사용자 지정 체크박스 예시입니다:

**참고**: 이 예시의 일부 CSS는 아직 배우지 않은 속성을 사용합니다. 이는 사용자 지정 체크박스를 만드는 방법을 이해하기 위한 예시일 뿐입니다. 앞으로 수업과 워크숍에서 이 모든 내용을 배우게 됩니다.

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css" />
<form>
  <label>
    <input class="checkbox" type="checkbox" /> Agree
  </label>
</form>
```

```css
.checkbox {
  appearance: none;
  width: 18px;
  height: 18px;
  border: 2px solid #ccc;
  border-radius: 4px;
  display: inline-block;
  position: relative;
  cursor: pointer;
  transition: all 0.25s ease;
  vertical-align: middle; 
}

.checkbox:hover {
  border-color: #888;
}

.checkbox:checked {
  background-color: #4caf50;
  border-color: #4caf50;
}

.checkbox:checked::after {
  content: "";
  position: absolute;
  left: 4px;
  top: 0px;
  width: 5px;
  height: 10px;
  border: solid white;
  border-width: 0 2px 2px 0;
  transform: rotate(45deg);
}

.checkbox:focus {
  outline: 2px solid #90caf9;
  outline-offset: 2px;
}
```

:::

WebKit은 웹 브라우저가 웹사이트를 표시하도록 돕는 소프트웨어 엔진입니다. Safari 같은 브라우저는 웹 페이지가 올바르게 보이고 작동하도록 WebKit을 사용합니다. 이 `appearance: none` CSS 속성은 스타일링을 완전히 제어할 수 있게 해주지만 주의할 점도 있습니다. 입력 요소의 기본 인터랙티브 컴포넌트에는 포커스와 오류 표시 같은 기능이 포함되어 있어, 이 기능들이 사라지지 않도록 신경 써야 합니다.

플랫폼 간 일관된 스타일링을 만드는 것은 이 속성을 사용하는 좋은 이유입니다. 모바일 기기에서 탭 대상이 충분히 크도록 하거나 체크박스 색상이 충분한 대비를 갖도록 보장하는 데도 사용할 수 있습니다.
