# --description--

Es wird Situationen geben, in denen Sie einen Wert mit einem breiten Typ haben und diesen auf einen spezifischeren Typ eingrenzen müssen. Zum Beispiel müssen Sie vielleicht sicherstellen, dass ein Objekt einer von Ihnen definierten Schnittstelle entspricht. Oder ein String befindet sich innerhalb einer bestimmten Liste von Werten. Es gibt mehrere Möglichkeiten, dies zu erreichen.

Die erste ist die Eingrenzung durch Truthiness. Betrachten Sie unser Beispiel aus der letzten Lektion:

```js
const email = document.querySelector<HTMLInputElement>("#email");
console.log(email.value);
```

Wir erhalten einen Compilerfehler, wenn wir versuchen, auf die `value`-Eigenschaft von `email` zuzugreifen, weil `email` möglicherweise `null` ist. Wir können jedoch eine bedingte Anweisung verwenden, um zu bestätigen, dass `email` truthy ist, bevor wir auf die Eigenschaft zugreifen:

```js
const email = document.querySelector<HTMLInputElement>("#email");
if (email) {
  console.log(email.value);
}
```

In diesem aktualisierten Beispiel, da `null` kein truthy-Wert ist, kann TypeScript innerhalb des bedingten Blocks ableiten, dass `email` UNBEDINGT ein `input`-Element sein muss. Daher wird kein Compilerfehler mehr ausgelöst.

Truthiness-Prüfungen funktionieren auch in die entgegengesetzte Richtung:

```js
const email = document.querySelector<HTMLInputElement>("#email");
if (!email) {
  throw new ReferenceError("Could not find email element!")
}
console.log(email.value);
```

Mit diesem Ansatz werfen wir einen Fehler, wenn `email` falsy ist. `null` ist ein falsy-Wert. Das Werfen eines Fehlers beendet die logische Ausführung dieses Codes, was bedeutet, dass TypeScript beim Erreichen des `console.log()`-Aufrufs weiß, dass `email` nicht `null` sein kann.

Optional chaining ist ebenfalls eine Form des Type Narrowing, basierend auf der gleichen Prämisse, dass der Eigenschaftszugriff nicht stattfinden kann, wenn der `email`-Wert `null` ist.

```js
const email = document.querySelector<HTMLInputElement>("#email");
console.log(email?.value);
```

Aber wie sieht es mit anderen Typen aus? Nun, Sie können Typen auch mit dem `typeof`-Operator eingrenzen. Schauen wir uns ein Beispiel für eine Variable an, von der wir angenommen haben, dass sie entweder ein String ODER eine Zahl sein könnte:

```js
const myVal = Math.random() > 0.5 ? 222 : "222";
console.log(myVal / 10)
```

In diesem Beispiel sehen wir einen Compilerfehler, weil wir keine arithmetischen Operationen mit einem String-Wert durchführen können. Aber wir können eine Bedingung verwenden, um den `typeof` der `myVal`-Variablen zu prüfen:

```js
const myVal = Math.random() > 0.5 ? 222 : "222";
if (typeof myVal === "number") {
  console.log(myVal / 10);
}
```

Da wir das Schlüsselwort `typeof` verwendet haben, weiß TypeScript jetzt, dass `myVal` eine Zahl sein muss, und wir können sicher damit rechnen.

Aber wie sieht es mit komplexeren Objekttypen aus? Wenn das betreffende Objekt von einer Klasse stammt, können Sie tatsächlich das Schlüsselwort `instanceof` verwenden, um den Typ einzugrenzen. Zurück zu unserem `querySelector()`-Beispiel:

```js
const email = document.querySelector("#email");
```

Anstatt einen generischen Typ zu übergeben und TypeScript zu sagen, was das Element ist, können wir `instanceof` verwenden, um den Typ einzugrenzen und sichereren Code zu schreiben:

```js
const email = document.querySelector("#email");

if (email instanceof HTMLInputElement) {
    console.log(email.value);
}
```

Dieser Ansatz mag dem vorherigen ähnlich erscheinen, aber `instanceof` ist eine Laufzeit-Validierung – das bedeutet, wenn wir den TypeScript-Typ irgendwie falsch angegeben haben, bestätigt unser JavaScript-Code trotzdem, dass `email` ein `input`-Element ist.

Als Nächstes betrachten wir ein Beispiel, bei dem wir ein `User`-Objekt von einer API abrufen und versuchen, die Informationen auszugeben:

```js
interface User {
    name: string;
    age: number;
}

const printAge = (user: User) => 
  console.log(`${user.name} is ${user.age} years old!`)

const request = await fetch("url")
const myUser = await request.json();
printAge(myUser);
```

Wir erhalten einen Compilerfehler, wenn wir versuchen, `myUser` an die Funktion zu übergeben, weil TypeScript nicht weiß, dass die API das richtige Objekt zurückgibt. Und die `.json()`-Methode akzeptiert keinen generischen Typ.

Der „einfache“ Weg, dieses Problem zu lösen, wäre, den Typ zu casten:

```js
interface User {
    name: string;
    age: number;
}

const printAge = (user: User) => 
  console.log(`${user.name} is ${user.age} years old!`)

const request = await fetch("url")
const myUser = await request.json() as User;
printAge(myUser);
```

Aber wann immer Sie den Typ casten, schwächen Sie im Grunde die Fähigkeit von TypeScript, potenzielle Fehler zu erkennen. Statt den Typ zu casten, können Sie also eine Type Guard schreiben:

```js
interface User {
    name: string;
    age: number;
}

const isValidUser = (user: unknown): user is User => {
  return !!user && 
    typeof user === "object" &&
    "name" in user &&
    "age" in user;
}
```

Der Rückgabetyp ist hier der entscheidende Bestandteil dieser Funktionsdefinition. Die `user is User`-Syntax zeigt an, dass unsere Funktion einen Boolean zurückgibt, der, wenn `true`, bedeutet, dass der `user`-Wert die `User`-Schnittstelle erfüllt. Wir führen dann einige grundlegende Prüfungen durch, um sicherzustellen, dass die Struktur des `user`-Objekts übereinstimmt – beachten Sie die Verwendung einer Truthiness-(`!!user`) Eingrenzung und einer `typeof`-Eingrenzung. Das müssen wir tun, weil `typeof null` `"object"` zurückgibt:

```js
interface User {
    name: string;
    age: number;
}

const isValidUser = (user: unknown): user is User => {
  return !!user && 
    typeof user === "object" &&
    "name" in user &&
    "age" in user;
}

const printAge = (user: User) => 
  console.log(`${user.name} is ${user.age} years old!`)

const request = await fetch("url")
const myUser = await request.json() as User;
if (isValidUser(myUser)) {
    printAge(myUser);
}
```

Wenn wir nun unsere gesamte Logik zusammenführen, erhalten wir keine Compilerfehler mehr und können unseren Code erfolgreich erstellen.

Type Narrowing ist eine mächtige Funktion, die Ihnen hilft, sichereren und weniger fehleranfälligen Code zu schreiben – aber denken Sie daran, dass TypeScript-Typen nicht völlig starr sind, also vermeiden Sie Praktiken wie das Casten eines Typs ohne Eingrenzung.
