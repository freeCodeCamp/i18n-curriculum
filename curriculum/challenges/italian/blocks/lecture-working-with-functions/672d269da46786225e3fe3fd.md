# --interactive--

Le funzioni sono porzioni di codice riutilizzabili che eseguono un compito specifico o calcolano un valore. Pensa alle funzioni come a una macchina che prende un input, esegue alcune operazioni su di esso e poi produce un output. Ecco un esempio di dichiarazione di una funzione:

```js
function greet() {
  console.log("Hello, Jessica!");
}
```

In questo esempio, abbiamo dichiarato una funzione chiamata `greet`. All’interno di quella funzione, abbiamo un `console.log` che stampa il messaggio `Hello, Jessica!`. Se provassimo a eseguire questo codice, non vedremmo il messaggio apparire nella console. Questo perché dobbiamo chiamare la funzione.

Una chiamata di funzione, o invocazione, è quando effettivamente usiamo o eseguiamo la funzione. Per chiamare una funzione, devi fare riferimento al nome della funzione seguito da un insieme di parentesi tonde:

:::interactive_editor

```js
function greet() {
  console.log("Hello, Jessica!");
}

greet(); // "Hello, Jessica!"
```

:::

Ora il messaggio `Hello, Jessica!` verrà stampato nella console. Ma cosa succede se volessimo che il messaggio dicesse `Hello, Nick!` o `Hello, Anna!`? Non vogliamo scrivere una nuova funzione ogni volta che salutiamo un utente diverso. Invece, possiamo creare una funzione riutilizzabile che usa parametri e argomenti.

I parametri fungono da segnaposto per i valori che verranno passati alla funzione quando viene chiamata. Permettono alle funzioni di accettare input e lavorare con quell’input. Gli argomenti sono i valori effettivi passati alla funzione quando viene chiamata. Ecco una versione aggiornata della funzione `greet` che usa parametri e argomenti:

:::interactive_editor

```js
function greet(name) {
  console.log("Hello, " + name + "!");
}

greet("Alice"); // Hello, Alice!
greet("Nick"); // Hello, Nick!
```

:::

`name` funge da parametro mentre le stringhe `Alice` e `Nick` fungono da argomenti. Ora abbiamo una funzione riutilizzabile che può essere usata decine di volte nel nostro codice con argomenti diversi.

Quando una funzione termina la sua esecuzione, restituisce sempre un valore. Per impostazione predefinita, il valore di ritorno sarà `undefined`. Ecco un esempio:

:::interactive_editor

```js
function doSomething() {
  console.log("Doing something...");
}

let result = doSomething();
console.log(result); // undefined
```

:::

Se hai bisogno che la tua funzione restituisca un valore specifico, allora devi usare l’istruzione `return`. Ecco un esempio di utilizzo di un’istruzione `return` per restituire la somma di due valori:

:::interactive_editor

```js
function calculateSum(num1, num2) {
  return num1 + num2;
}

console.log(calculateSum(3, 4)); // 7
```

:::

Spesso userai l’istruzione `return`, perché puoi usare quel valore restituito dalla funzione più avanti nel tuo codice.

Finora abbiamo lavorato con funzioni nominate, ma puoi anche creare quella che si chiama funzione anonima. Una funzione anonima è una funzione senza nome che può essere assegnata a una variabile in questo modo:

:::interactive_editor

```js
const sum = function (num1, num2) {
  return num1 + num2;
};

console.log(sum(3, 4)); // 7
```

:::

In questo esempio, abbiamo una variabile `const` chiamata `sum` e le stiamo assegnando una funzione anonima che restituisce la somma di `num1` e `num2`. Possiamo quindi chiamare `sum` e passare i numeri `3` e `4` per ottenere il risultato `7`.

Le funzioni supportano parametri predefiniti, permettendoti di impostare valori predefiniti per i parametri. Questi valori predefiniti vengono usati se la funzione viene chiamata senza un argomento per quel parametro. Ecco un esempio:

:::interactive_editor

```js
function greetings(name = "Guest") {
  console.log("Hello, " + name + "!");
}

greetings(); // Hello, Guest!
greetings("Anna"); // Hello, Anna!
```

:::

In questo esempio, se non viene fornito un argomento per `name`, il valore predefinito è `Guest`.

In sintesi, le funzioni ti permettono di scrivere codice riutilizzabile e organizzato. Possono prendere input (parametri), eseguire azioni e restituire output.
