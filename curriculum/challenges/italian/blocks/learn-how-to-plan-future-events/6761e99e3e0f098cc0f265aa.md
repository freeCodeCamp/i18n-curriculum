# --explanation--

`SQL injection attacks` sono un tipo di attacco informatico in cui gli hacker inseriscono codice dannoso in una query del database per consultare o danneggiare i dati. Per esempio:

`The company added security measures to prevent SQL injection attacks on their website.` - Questo significa che hanno preso misure per impedire agli hacker di inserire codice dannoso.

Jake dice che le `additional measures` sono necessarie per `prevent SQL injection attacks`, cioè stanno cercando di fermare questo tipo specifico di minaccia.
