## アルゴリズムとビッグO記法

- **アルゴリズム**：問題を解決したりタスクを実行したりするための、あいまいさのない一連の手順です。アルゴリズムは有限のステップで終了し、各ステップは正確かつあいまいさがあってはいけません。

- **ビッグO記法**：入力サイズが増加したときのアルゴリズムの最悪の性能や成長率を表します。定数項や低次の項を無視し、入力サイズに対するリソース使用量の増え方に注目します。

### よくある時間計算量

- **O(1) - 定数時間**：入力サイズに関係なく同じ時間がかかります。

```python
def check_even_or_odd(number):
    if number % 2 == 0:
        return 'Even'
    else:
        return 'Odd'
```

- **O(log n) - 対数時間**：入力が増えても時間の増加がゆっくりです。問題サイズを繰り返し分割するアルゴリズム（例：二分探索）でよく見られます。

- **O(n) - 線形時間**：実行時間が入力サイズに比例して増加します。

```python
for grade in grades:
    print(grade)
```

- **O(n log n) - 線形対数時間**：マージソートやクイックソートなど効率的なソートアルゴリズムの一般的な時間計算量です。

- **O(n²) - 二次時間**：実行時間が二次的に増加します。ネストされたループでよく見られます。

```python
for i in range(n):
    for j in range(n):
        print("Hello, World!")
```

### 空間計算量

- **O(1) - 定数空間**：入力サイズに関係なく同じメモリ量を使用します。
- **O(n) - 線形空間**：メモリ使用量が入力サイズに比例して増加します。
- **O(n²) - 二次空間**：メモリ使用量が二次的に増加します。

## 問題解決の手法

- **問題の理解**：問題文を何度も読みます。入力、期待される出力、入力から出力への変換方法を特定してください。

- **疑似コード**：言語に依存しないアルゴリズムの高レベルな説明です。一般的な文章と`IF`、`ELSE`、`FOR`、`WHILE`のようなプログラミング構造を混ぜて使います。

```md
GET original_string
SET reversed_string = ""
FOR EACH character IN original_string:
  ADD character TO THE BEGINNING OF reversed_string
DISPLAY reversed_string
```

- **エッジケース**：アルゴリズムが扱うべき境界にある特定の有効な入力です。必ず考慮し、テストしてください。

## 配列

- **静的配列**：初期化時にサイズが固定されます。要素は隣接したメモリ領域に格納されます。実行中にサイズを変更できません。

- **動的配列**：実行中に自動的にサイズが増減します。必要に応じてより大きな配列に自動コピーしてサイズ変更を処理します。

### Pythonのリスト（動的配列）

```python
numbers = [3, 4, 5, 6]

# Access elements
numbers[0]  # 3

# Update elements
numbers[2] = 16

# Add elements
numbers.append(7)
numbers.insert(3, 15)  # Insert at specific index

# Remove elements
numbers.pop(2)  # Remove at specific index
numbers.pop()   # Remove last element
```

### 動的配列の時間計算量

- **アクセス**：O(1)
- **末尾への挿入**：平均O(1)、サイズ変更時はO(n)
- **途中への挿入**：O(n)
- **削除**：途中はO(n)、末尾はO(1)

## スタック

- **スタック**：後入れ先出し（LIFO）のデータ構造です。要素は上部からのみ追加・削除されます。

- **プッシュ操作**：スタックの上に要素を追加します。時間計算量はO(1)です。

- **ポップ操作**：スタックの上から要素を削除します。時間計算量はO(1)です。

```python
# Using Python list as stack
stack = []

# Push operations
stack.append(1)
stack.append(2)
stack.append(3)

# Pop operations
top_element = stack.pop()  # Returns 3
```

## キュー

- **キュー**：先入れ先出し（FIFO）のデータ構造です。要素は後ろに追加され、前から削除されます。

- **エンキュー操作**：キューの後ろに要素を追加します。時間計算量はO(1)です。

- **デキュー操作**：キューの前から要素を削除します。時間計算量はO(1)です。

```python
from collections import deque

# Using deque for efficient queue operations
queue = deque()

# Enqueue operations
queue.append(1)
queue.append(2)
queue.append(3)

# Dequeue operations
first_element = queue.popleft()  # Returns 1
```

## 連結リスト

- **連結リスト**：各ノードがデータと次のノードへの参照を持つ線形のデータ構造です。ノードはチェーンのようにつながっています。

### 単方向連結リスト

- **構造**：各ノードはデータと次のノードへの1つの参照を持ちます。
- **走査**：先頭から末尾へ一方向にしか進めません。
- **先頭ノード**：リストの最初のノードで、通常は唯一直接アクセスできるノードです。
- **末尾ノード**：リストの最後のノードで、`None`を指します。

### 操作と時間計算量

- **先頭への挿入**：O(1)
- **末尾への挿入**：O(n) - 末尾まで走査が必要
- **途中への挿入**：O(n) - 挿入位置まで走査が必要
- **先頭からの削除**：O(1)
- **末尾からの削除**：O(n) - 前のノードを探すため走査が必要
- **途中からの削除**：O(n) - 削除ノードを探すため走査が必要

### 双方向連結リスト

- **構造**：各ノードはデータと2つの参照（次のノードと前のノード）を持ちます。
- **走査**：両方向に移動できます。
- **メモリ**：単方向連結リストより多くのメモリを使います（参照が1つ多いため）。

## ハッシュマップとセット

### マップとハッシュマップ

- **マップ（抽象データ型）**：キーと値のペアの集合を管理します。キーはすべて一意でなければなりませんが、値は重複してもかまいません。

- **ハッシュマップ**：ハッシュ技術を使ったマップADTの具体的な実装です。キーのハッシュ値を生成するハッシュ関数を使い、そのハッシュ値で基盤となる配列の格納場所を決定します。

### Pythonの辞書（ハッシュマップ）

```python
# Creating dictionaries
my_dictionary = {
    "A": 1,
    "B": 2, 
    "C": 3
}

# Alternative creation
my_dictionary = dict(A=1, B=2, C=3)

# Access and modify
value = my_dictionary["A"]  # 1
my_dictionary["A"] = 4      # Update value
del my_dictionary["A"]      # Remove key-value pair

# Check membership
"C" in my_dictionary

# Get keys, values, items
my_dictionary.keys()
my_dictionary.values()
my_dictionary.items()
```

### ハッシュマップの時間計算量

- **平均ケース**：挿入、取得、削除はO(1)
- **最悪ケース**：多くのハッシュ衝突が起きた場合はO(n)

### セット

- **セット**：重複しない要素の順序なし集合です。重複は許されず、特定の順序は保持されません。

- **不変要素のみ**：セットには不変のデータ型（数値、文字列、タプル）しか含められません。ハッシュ値が一定である必要があるためです。

```python
# Creating sets
numbers = {1, 2, 3, 4}
empty_set = set()  # Must use set(), not {}

# Add and remove elements
numbers.add(5)
numbers.remove(4)      # Raises KeyError if not found
numbers.discard(4)     # No error if not found

# Set operations
set_a = {1, 2, 3, 4}
set_b = {2, 3, 4, 5, 6}

# Union, intersection, difference, symmetric difference
set_a.union(set_b)                    # or set_a | set_b
set_a.intersection(set_b)             # or set_a & set_b
set_a.difference(set_b)               # or set_a - set_b
set_a.symmetric_difference(set_b)     # or set_a ^ set_b

# Subset and superset checks
set_a.issubset(set_b)
set_a.issuperset(set_b)
set_a.isdisjoint(set_b)

# Membership testing
5 in numbers
```

### セットの時間計算量

- **平均ケース**：追加、削除、メンバーシップテストはO(1)
- **最悪ケース**：ハッシュ衝突によりO(n)

## ハッシュ衝突

- **ハッシュ衝突**：異なる2つのキーが同じハッシュ値を生成することです。

- **衝突解決の手法**：
  - **チェーン**：各配列インデックスが同じハッシュ値の要素を格納する連結リストを指します
  - **オープンアドレッシング**：あらかじめ定められたシーケンスを使って次の空きインデックスを探します

## 各データ構造の使いどころ

- **リスト**：順序付きでインデックスアクセスが必要で、サイズが事前にわからない場合
- **スタック**：LIFO操作（元に戻す機能、式の評価、バックトラッキング）に使います
- **キュー**：FIFO操作（タスクスケジューリング、幅優先探索）に使います
- **連結リスト**：先頭での頻繁な挿入・削除、サイズ不明、ランダムアクセス不要な場合
- **ハッシュマップ**：高速なキーと値の検索、出現回数のカウント、キャッシュに使います
- **セット**：一意性のチェック、集合演算、重複の除去に使います
