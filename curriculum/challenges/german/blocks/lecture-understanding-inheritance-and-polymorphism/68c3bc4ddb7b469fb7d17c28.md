# --description--

Polymorphismus ist das nächste Schlüsselkonzept der objektorientierten Programmierung (OOP), über das wir sprechen werden.

Mit Polymorphismus haben Sie Zugriff auf ein Interface, über das Sie mit vielen Objekten derselben Art interagieren können.

Werfen wir einen genaueren Blick auf Polymorphismus und darauf, wie er Ihnen ermöglicht, Code wiederzuverwenden.

Polymorphismus erlaubt es Methoden in verschiedenen Klassen, denselben Namen zu teilen, aber unterschiedliche Aufgaben auszuführen. Sie rufen denselben Methodennamen bei verschiedenen Objekten auf, und jedes reagiert auf seine eigene Weise.

Hier ist das grundlegende Beispiel für Polymorphismus:

```py
class A:
   def action(self): ...

class B:
   def action(self): ...

class C:
   def action(self): ...

Class().method()  # Works for A, B, or C
```

Hier ein Beispiel, das verschiedene Tiergeräusche verwendet, um Polymorphismus darzustellen:

```py
class Cat:
   def speak(self):
       return "A cat meow"

class Bird:
   def speak(self):
       return "A bird tweet"
  
class Monkey:
   def speak(self):
       return "A monkey ooh ooh aah aah ooh ooh aah aah"

def animal_sound(animal):
   print(animal.speak())

animal_sound(Cat())
animal_sound(Bird())
animal_sound(Monkey())
```

In diesem Beispiel ist `animal_sound()` eine Funktion, die jedes Objekt mit einer `speak()`-Methode akzeptiert.

Wenn Sie ein `Cat`, `Bird` oder `Monkey` übergeben, ruft sie die `speak()`-Methode des Objekts auf und gibt das Ergebnis aus. Da jede Klasse `speak()` unterschiedlich definiert, erhalten Sie unterschiedliche Ausgaben von derselben Funktion. Das ist Polymorphismus in Aktion.

Hier ein weiteres Beispiel, diesmal mit Instanzen und einem Attribut:

```py
class Twitter:
   def __init__(self, content):
       self.content = content

   def post(self):
       return f"🐦 Tweet: '{self.content}' (280 chars max)"

class Instagram:
   def __init__(self, content):
       self.content = content

   def post(self):
       return f"📸 Instagram Post: '{self.content}' + ✨ filters"

class LinkedIn:
   def __init__(self, content):
       self.content = content

   def post(self):
       return f"💼 LinkedIn Article: '{self.content}' (Professional Mode)"

def start(social_media):
   print(social_media.post())  # Calls .post() on any object

# Instances
tweet = Twitter('Just learned Python polymorphism!')
photo = Instagram('Sunset vibes 🌅')
article = LinkedIn('Why OOP matters in 2024')

# The polymorphic calls - same function, different outputs
start(tweet) # 🐦 Tweet: 'Just learned Python polymorphism!' (280 chars max)
start(photo) # 📸 Instagram Post: 'Sunset vibes 🌅' + ✨ filters
start(article) # 💼 LinkedIn Article: 'Why OOP matters in 2024' (Professional Mode)
```

Es gibt auch eine Art von Polymorphismus, die **vererbungsbasierter Polymorphismus** genannt wird.

Beim vererbungsbasierten Polymorphismus definiert eine Elternklasse eine Methode, und mehrere Kindklassen überschreiben diese Methode auf ihre eigene Weise. Sie können dann dieselbe Methode bei jedem Kindobjekt aufrufen, und sie verhält sich unterschiedlich, je nachdem, zu welcher Kindklasse es gehört.

Hier ein Beispiel:

```py
class Animal:
   def speak(self):
       return 'Some generic sound'

class Cat(Animal):
   def speak(self):
       return 'A cat meow'

class Dog(Animal):
   def speak(self):
       return 'A dog barks woof woof'

class Monkey(Animal):
   def speak(self):
       return 'A monkey ooh ooh aah aah ooh ooh aah aah'
  
print(Cat().speak()) # A cat meow
print(Dog().speak()) # A dog barks woof woof
print(Monkey().speak()) # A monkey ooh ooh aah aah ooh ooh aah aah
print(Animal().speak()) # Some generic sound
```

Sie sehen, dass jede Kindklasse der Elternklasse `Animal` die `speak()`-Methode überschreibt, um ihre eigene Implementierung bereitzustellen. Wenn Sie also die `speak()`-Methode bei einer Instanz jeder Unterklasse aufrufen, gibt sie den spezifischen Laut zurück, der mit diesem Tier verbunden ist.

Sie können auch noch weiter gehen und den Aufruf in einer Liste machen, dann durch die Liste iterieren, um anzuzeigen, was die `speak()`-Methode für jedes zurückgibt:

```py
animals = [Cat(), Dog(), Monkey()]

for animal in animals:
   print(animal.speak())

# Output:
# A cat meow
# A dog barks woof woof
# A monkey ooh ooh aah aah ooh ooh aah aah
```
