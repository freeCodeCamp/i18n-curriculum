# --description--

In una lezione precedente, hai imparato che gli attributi sono le variabili che appartengono a un oggetto. Questo significa che contengono dati che descrivono lo stato o il comportamento dell'oggetto.

Per esempio, un'auto normalmente ha una marca e un modello. La marca e il modello potrebbero essere attributi per una `Car` classe:

```python
class Car: 
    def __init__(self, brand, model): 
        self.brand = brand 
        self.model = model 

my_car = Car('Lamborghini', 'Gallardo') 
print(my_car.brand) # Lamborghini 
print(my_car.model) # Gallardo 
```

Ma a volte potresti non sapere quali attributi ti servono fino a quando il programma non è in esecuzione. Immagina di scrivere uno script che riceve i nomi degli attributi da un utente o da un file di configurazione. Questi non sono attributi che puoi codificare in modo fisso in anticipo.

È qui che entra in gioco la gestione dinamica degli attributi. In questo modo puoi consultare, modificare, controllare o persino eliminare attributi usando i loro nomi come variabili, e non come nomi fissi nel codice. Questo dà al tuo programma la flessibilità di rispondere a dati o input dell'utente diversi al volo.

Python ti offre quattro comode funzioni integrate per lavorare dinamicamente con gli attributi degli oggetti. Sono `getattr()`, `setattr()`, `hasattr()` e `delattr()`.

Ti permettono di consultare, creare, controllare ed eliminare attributi usando nomi variabili. Vediamo ciascuna in azione.

`getattr()` rende possibile leggere un attributo da un oggetto quando non conosci il suo nome fino al momento dell'esecuzione. Se l'attributo non esiste, genera un `AttributeError`, a meno che tu non fornisca un valore predefinito.

Per usarla, passi l'oggetto, il nome dell'attributo e un valore predefinito opzionale:

```python
getattr(object, attribute_name, default_value) 
```

Ecco un esempio:

```python
class Person: 
    def __init__(self, name, age): 
        self.name = name 
        self.age = age 

person = Person('John Doe', 30) 
 
print(getattr(person, 'name')) # John Doe 
print(getattr(person, 'age')) # 30 
print(getattr(person, 'city', 'Milano')) # Milano
```

Nell'esempio sopra, `Milano` è un valore predefinito perché `city` non esiste nella classe `Person`.

Come detto prima, la vera potenza di `getattr()` si vede quando il nome dell'attributo proviene da una variabile, come un input dell'utente o un file.

In quel caso, non puoi usare la sintassi `object.attribute_name` normale perché il nome dell'attributo non è fisso.

```python
class Person: 
    def __init__(self, name, age): 
        self.name = name 
        self.age = age 

person = Person('John Doe', 30)

attr_name = input('Enter the attribute you want to see: ')
print(getattr(person, attr_name, 'Attribute not found'))
```

In questo caso, se l'utente digita `name`, vede `John Doe`, e se digita `age`, vede `30`. E se digita qualcosa che non esiste nella classe come `email`, vede `Attribute not found`.

Qui è proprio dove la gestione dinamica degli attributi dà il meglio di sé. Permette al tuo codice di rispondere a input e dati che non ha mai visto prima.

Inoltre, potresti voler scorrere tutti gli attributi che un oggetto ha, non solo quelli che già conosci. La funzione integrata `dir()` ti permette di farlo. Restituisce una lista di tutti i nomi degli attributi sull'oggetto. Ecco come usarla:

```python
class Person: 
    def __init__(self, name, age): 
        self.name = name 
        self.age = age 

person = Person('John Doe', 30)

# Loop through all attributes of the person object with dir() function
for attr in dir(person):
    # Ignore dunder methods like __init__ or __str__ and regular methods
    if not attr.startswith('__') and not callable(getattr(person, attr)): 
        value = getattr(person, attr)
        print(f'{attr}: {value}')

# Output
# age: 30
# name: John Doe
```

La funzione `setattr()` ti permette di creare un nuovo attributo o aggiornare uno esistente in modo dinamico. La sintassi è questa:

```python
setattr(object, attribute_name, value) 
```

Ecco un esempio che imposta attributi di configurazione basandosi su dati provenienti da un file di configurazione o da una variabile d'ambiente:

```python
class Configuration:
    pass

# Data loaded at runtime (like from a config or env file)
settings_data = {
    'server_url': 'https://api.example.com',
    'timeout_sec': 30,
    'max_retries': 5
}

config_obj = Configuration()

# Dynamically set attributes using dictionary keys and values
for attr_name, attr_value in settings_data.items():
    setattr(config_obj, attr_name, attr_value)

print(config_obj.server_url) # https://api.example.com
print(config_obj.timeout_sec) # 30
```

C'è anche `hasattr()`. Prima di fare qualcosa con un attributo o eliminarlo, è buona pratica controllare se esiste. Questo è ciò che `hasattr()` ti permette di fare. Controlla se un attributo esiste e restituisce `True` o `False` in base al risultato.

Ecco la sintassi base:

```python
hasattr(object, attribute_name)  
```

Ecco un esempio che controlla dinamicamente l'esistenza di attributi in un'istanza della classe `Product`:

```python
class Product:
    def __init__(self, name, price):
        self.name = name
        self.price = price

product_a = Product('T-Shirt', 25)

required_attributes = ['name', 'price', 'inventory_id']

for attr in required_attributes:
    if not hasattr(product_a, attr):
        print(f"ERROR: Product is missing the required attribute: '{attr}'")
    else:
        # Access the attributes dynamically once their existence is confirmed
        print(f'{attr}: {getattr(product_a, attr)}')

# Output:
# name: T-Shirt
# price: 25
# ERROR: Product is missing the required attribute: 'inventory_id'
```

L'output ERROR è avvenuto perché `inventory_id` manca nella classe `Product` e nella sua istanza.

Infine, `delattr()` ti permette di rimuovere un attributo in modo dinamico:

```python
delattr(object, attribute_name) 
```

Per esempio, immagina che un oggetto sia stato completamente elaborato, poi decidi di pulire eventuali attributi sensibili o temporanei che potrebbero esistere prima di salvare la versione finale. Dopo di che, puoi usare `dir()` per iterare sugli attributi rimanenti:

```python
class UserSession:
    def __init__(self, user_id, token):
        self.user_id = user_id
        self.auth_token = token # sensitive
        self.temp_counter = 0 # temporary

session = UserSession(101, 'a1b2c3d4e5')

# List of attributes to remove dynamically before "saving" the session
attributes_to_clean = ['auth_token', 'temp_counter']

# Dynamically remove specified attributes
for attr in attributes_to_clean:
    if hasattr(session, attr):
        delattr(session, attr)
        print(f'Removed attribute: {attr}')

print('\nFinal attributes remaining:')

# Loop through the remaining attributes with dir()
for attr in dir(session):
    # Ignore dunder methods like __init__ or __str__ and regular methods
    if not attr.startswith('__') and not callable(getattr(session, attr)):
        print(f' - {attr}: {getattr(session, attr)}')

# Output:
# Removed attribute: auth_token
# Removed attribute: temp_counter

# Final attributes remaining:
#  - user_id: 101
```

Ecco come puoi gestire gli attributi dinamicamente!
