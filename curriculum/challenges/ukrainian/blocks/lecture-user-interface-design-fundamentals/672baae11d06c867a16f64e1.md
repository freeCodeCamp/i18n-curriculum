# --description--

Прогресивне покращення — це підхід до дизайну, який гарантує, що всі користувачі, незалежно від браузера чи пристрою, можуть отримати доступ до основного вмісту та функціональності застосунку.

Він зосереджується на наданні базового досвіду, який працює для всіх, одночасно пропонуючи додаткові функції та покращення користувачам із більш сучасними браузерами або кращим інтернет-з’єднанням.

Підхід прогресивного покращення базується на таких основних принципах:

- Весь базовий вміст і базова функціональність мають бути доступні у всіх браузерах
- Всі розширені макети мають надаватися через зовнішні CSS-стилі
- Вся розширена функціональність має надаватися через зовнішні JavaScript-файли
- Переваги браузера користувача мають поважатися

Використання підходу прогресивного покращення робить ваші застосунки більш доступними, оскільки весь базовий вміст і функціональність не повинні блокуватися в середовищах, які їх не підтримують.

Щодо швидкості, підхід прогресивного покращення також може допомогти покращити продуктивність ваших застосунків.

Користувачі з повільнішим інтернет-з’єднанням все одно зможуть отримати доступ до вмісту, оскільки браузер спочатку завантажить необхідні ресурси.

Що стосується SEO, прогресивне покращення також може допомогти покращити видимість ваших застосунків.

Пошукові системи зможуть сканувати вміст ваших застосунків, оскільки базовий вміст доступний у початковій HTML-відповіді.

Хоча деякі критикували цей підхід, вважаючи, що він не завжди реалістичний для застосунків, які сильно залежать від JavaScript для своєї функціональності, все ж це добра практика, якої варто дотримуватися при створенні застосунків.
