# --interactive--

Le espressioni regolari, o regex, sono una funzionalità supportata da molti linguaggi di programmazione diversi.

Un'espressione regolare è una sintassi speciale per creare un "pattern", che puoi poi usare per verificare una stringa, estrarre testo e altro.

Diamo un'occhiata a una espressione regolare di base:

```js
const regex = /freeCodeCamp/;
```

Nota come, in JavaScript, definisci un'espressione regolare creando il pattern tra due barre oblique (`/`). Cerca di non confonderla con un commento, dove il testo segue entrambe le barre oblique.

Questa particolare espressione regolare corrisponderà al testo `freeCodeCamp`, con `C` maiuscole, ovunque in una stringa. Ma come si fa effettivamente?

Questo ci porta al nostro primo metodo – il metodo `test()`. Il metodo `test()` è presente sugli oggetti `RegExp`, che sono oggetti che rappresentano un'espressione regolare (come quella che abbiamo appena definito).

Il metodo `test()` accetta una stringa, che è la stringa da testare per verificare se corrisponde all'espressione regolare. Per esempio, proviamo a testare la stringa `e`:

:::interactive_editor

```js
const regex = /freeCodeCamp/;
const test = regex.test("e");
console.log(test);
```

:::

Puoi vedere che abbiamo chiamato il metodo `test()` sulla nostra nuova regex, passando la stringa `e` come argomento.

Il metodo `test()` ha restituito `false` perché la stringa `e` non corrisponde al pattern `freeCodeCamp`. Anche se il pattern `freeCodeCamp` include la lettera `e`, è il verso opposto rispetto a come funzionano le espressioni regolari.

Diamo un'occhiata ad altri esempi. Prenditi un momento per considerarli:

:::interactive_editor

```js
const regex = /freeCodeCamp/;
console.log(regex.test("freeCodeCamp"));
console.log(regex.test("freeCodeCamp is great"));
console.log(regex.test("I love freeCodeCamp"));
console.log(regex.test("freecodecamp"));
console.log(regex.test("FREECODECAMP"));
console.log(regex.test("free"));
console.log(regex.test("code"));
console.log(regex.test("camp"));
```

:::

Nota come le prime tre stringhe abbiano restituito `true`. Queste stringhe contengono tutte esattamente il testo `freeCodeCamp`, da qualche parte nella stringa.

Le righe 5 e 6 restituiscono `false`. Anche se contengono il testo `freecodecamp`, il maiuscolo/minuscolo non corrisponde. Le espressioni regolari sono sensibili al maiuscolo/minuscolo per impostazione predefinita.

Infine, mentre le ultime tre contengono una parte del pattern, le stringhe non contengono l'intero pattern.

Il metodo `test()` restituisce un `boolean`, che indica se la stringa corrisponde o meno all'espressione regolare.

Ma se volessi più informazioni di così? Le stringhe hanno un metodo `match()`. Questo metodo accetta un'espressione regolare, anche se puoi passare anche una stringa che verrà convertita in un'espressione regolare.

`match()` restituisce l'array delle corrispondenze per la stringa. Cos'è un array di corrispondenze? Diamo un'occhiata:

```js
const regex = /freeCodeCamp/;
const match = "freeCodeCamp".match(regex);
console.log(match);
```

Se eseguiamo questo, otteniamo un array! Ma è un array un po' strano. Ha alcune proprietà extra:

```js
// [
//   'freeCodeCamp',
//   index: 0,
//   input: 'freeCodeCamp',
//   groups: undefined
// ]
```

La proprietà `groups` mostrerebbe eventuali gruppi catturati. Imparerai cosa significa in una lezione futura.

La proprietà `index` ti dice a quale carattere nella stringa è stata trovata la corrispondenza. Nel nostro caso, è stata trovata all'inizio della stringa.

La proprietà `input` ti dice la stringa su cui è stato chiamato il metodo `match()`.

Proviamo ancora qualche esempio e vediamo come cambia il risultato:

```js
const regex = /freeCodeCamp/;
console.log("freeCodeCamp".match(regex)); // ['freeCodeCamp', index: 0, input: 'freeCodeCamp', groups: undefined]
console.log("freeCodeCamp is great".match(regex)); // ['freeCodeCamp', index: 0, input: 'freeCodeCamp is great', groups: undefined]
console.log("I love freeCodeCamp".match(regex)); // ['freeCodeCamp', index: 7, input: 'I love freeCodeCamp', groups: undefined]
console.log("freecodecamp".match(regex)); // null
console.log("FREECODECAMP".match(regex)); // null
console.log("free".match(regex)); // null
```

Sappiamo già che le prime tre stringhe dovrebbero produrre una corrispondenza, quindi diamo un'occhiata a quelle:

```js
// ['freeCodeCamp', index: 0, input: 'freeCodeCamp', groups: undefined]
// ['freeCodeCamp', index: 0, input: 'freeCodeCamp is great', groups: undefined]
// ['freeCodeCamp', index: 7, input: 'I love freeCodeCamp', groups: undefined]
```

È quello che ti aspettavi? Puoi vedere come `input` e `index` siano cambiati a seconda della stringa fornita e della posizione della corrispondenza nella stringa.

Le altre tre righe, che non corrispondono, restituiscono `null` invece di un array.

Ora che possiamo testare e trovare corrispondenze nelle stringhe con la nostra espressione regolare, cosa succede se vogliamo sostituire il contenuto di una stringa? Forse qualcuno ha scritto `freecodecamp` tutto in minuscolo e vogliamo correggere automaticamente il maiuscolo/minuscolo per loro.

Prima, dobbiamo aggiornare la nostra espressione regolare per corrispondere alla forma minuscola di `freecodecamp` e creare la nostra stringa di test:

```js
const regex = /freecodecamp/;
const str = "freecodecamp is rly kewl";
```

Ora, le stringhe hanno un metodo `replace()` che accetta due argomenti: l'espressione regolare da cercare (o una stringa, se non ti servono tutte le funzionalità delle regex) e la stringa con cui sostituire la corrispondenza (o una funzione da eseguire su ogni corrispondenza).

Quindi, se volessimo sostituire il nostro `freecodecamp` con la forma corretta:

:::interactive_editor

```js
const regex = /freecodecamp/;
const str = "freecodecamp is rly kewl";
const replaced = str.replace(regex, "freeCodeCamp");
console.log(replaced);
```

:::

Puoi vedere che `replace()` restituisce la stringa aggiornata con il pattern corrispondente `replaced`.

Le espressioni regolari e tutti i metodi associati possono sembrare complesse e difficili da capire. Ma avrai la possibilità di esplorarle meglio in questa prossima serie di lezioni.
