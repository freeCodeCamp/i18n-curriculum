# --description--

Wie Sie in vorherigen Lektionen gelernt haben, sind Cookies einfache Daten, die Websites auf dem Gerät eines Benutzers speichern können.

Sie können jedoch tatsächlich komplexere Datenstrukturen in Cookies speichern. Eine gängige Methode ist die Verwendung von JSON, um Objekte oder Arrays zu speichern. Hier ist ein Beispiel:

```js
const userData = {
  name: "John Doe",
  age: 30,
  role: "admin"
};

document.cookie = "userInfo=" + JSON.stringify(userData) + "; path=/";
```

In diesem Beispiel erstellen wir ein Objekt mit Benutzerdaten, wandeln es in einen JSON-String um und speichern es dann in einem Cookie. Wenn wir diese Daten abrufen möchten, können wir den JSON-String mit `JSON.parse()` in ein Objekt parsen.

Nun fragen Sie sich vielleicht, was es mit dem Aspekt der „HTTP-Header“ in unserem Thema auf sich hat. Typischerweise werden Cookies vom Server über HTTP-Header gesetzt.

Zum Beispiel könnte ein Server einen Header wie diesen senden:

```http
Set-Cookie: username=John Doe; expires=Thu, 31 Dec 2024 6:00:00 IST; path=/
```

Dieser Header weist den Browser an, ein Cookie zu setzen.

Wir können Cookies auch direkt im Browser mit JavaScript setzen. Das ist nützlich, um Daten zu speichern, die nicht sofort an den Server gesendet werden müssen.

Bitte beachten Sie, dass Cookies eine Größenbeschränkung von etwa 4 KB haben, und das Speichern zu vieler Daten darin Ihre Webanwendung verlangsamen kann.

Das Speichern großer Datenmengen in Cookies kann den Netzwerkverkehr erhöhen, da Cookies mit jeder HTTP-Anfrage gesendet werden.
