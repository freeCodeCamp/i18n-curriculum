# --interactive--

在網頁設計中，背景在定義網頁的整體外觀和感受中扮演著重要角色。

然而，在使用背景設計時，必須考慮存取性，以確保你的內容對所有使用者（包括視覺障礙者）皆可用且易於閱讀。

與背景相關的主要存取性問題之一是確保背景與文字之間有足夠的對比度。

如果沒有足夠的對比度，包含低視力或色盲的視覺障礙使用者可能會難以閱讀你頁面上的內容。

對比是指兩種顏色之間明暗的差異。背景色與文字色之間有足夠的對比度對於可讀性至關重要。

網頁內容無障礙指引（WCAG）建議一般文字的最低對比度為 4.5:1，大字體文字為 3:1。

例如，將白色文字放在淺灰色背景上會導致對比度不足，使文字難以閱讀。然而，白色文字放在深藍色背景上則會提供良好的對比度，提升所有使用者的可讀性。

以下是一個對比度不足的範例：

:::interactive_editor

```html
<p style="color: lightgray; background-color: whitesmoke;">
   This is an example of poor contrast.
</p>
```

:::

現在，這裡有一個良好對比的範例：

:::interactive_editor

```html
<p style="color: white; background-color: darkslategray;">
  This is an example of good contrast.
</p>
```

:::

另一個考量是避免將文字放置在繁複或複雜的背景上，例如多色的影像或漸層。繁複的背景會使文字與背景難以區分，無論對比度如何。

在設計背景時，避免僅使用顏色作為傳達資訊的唯一方式。 例如，僅使用顏色來顯示錯誤或成功訊息（例如紅色代表錯誤或綠色代表成功）可能會對色盲使用者造成問題。

除了顏色，你還應該使用符號或文字來傳達資訊。例如，在紅色錯誤訊息旁邊，你可以使用圖示或粗體文字來清楚表示有錯誤。

雖然較少見，背景音訊或影片也可能影響存取性。

背景音樂或自動播放的影片可能會讓某些使用者分心，特別是有認知障礙的使用者。如果你包含背景音訊，請務必提供讓使用者靜音或暫停音訊的方法。

透過考慮這些存取性因素，你可以創建更具包容性的設計，確保你的內容對所有使用者皆可讀且可用，無論他們的能力為何。
