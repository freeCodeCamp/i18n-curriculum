# --description--

Lo stato è uno dei fondamenti più importanti di React e di altri framework front-end. È come il cervello di un componente, cioè contiene informazioni che possono cambiare nel tempo e controlla come i componenti si comportano e appaiono.

Vediamo cos’è lo stato e come il hook `useState` ti permette di lavorarci.

Lo stato rappresenta i dati dinamici nel tuo componente React, come il valore di un input dell’utente, dati recuperati da un’API o un elemento in una lista di cose da fare.

Ogni volta che lo stato cambia, React renderizza di nuovo il componente senza ricaricare la pagina per riflettere quei cambiamenti nell’interfaccia utente. Questa reattività rende la tua app interattiva.

Il hook `useState` è una funzione che ti permette di dichiarare variabili di stato nei componenti funzionali.

Prima dei hook, potevi usare lo stato solo nei componenti di classe. Ma con l’introduzione dei hook da React 16.8, puoi usare lo stato nei componenti funzionali usando il hook `useState`.

Per usare il hook `useState`, devi importarlo da React:

```js
import { useState } from "react";
```

Puoi anche importare React stesso e accedere al hook `useState` come proprietà:

```js
import React from "react";
```

Ecco come puoi dichiarare una variabile di stato quando importi `useState`:

```js
const [stateVariable, setStateFunction] = useState(initialValue);
```

Ecco come puoi dichiarare una variabile di stato quando importi React:

```js
const [stateVariable, setStateFunction] = React.useState(initialValue);
```

Nella variabile di stato hai quanto segue:

- `stateVariable` contiene il valore attuale dello stato
- `setStateFunction` (la funzione setter) aggiorna la variabile di stato
- `initialValue` imposta lo stato iniziale

Nota che lo stato in un componente React è privato e isolato per ogni istanza del componente. Questo significa che, se renderizzi lo stesso componente due volte, lo stato di uno non influenza l’altro. Significa anche che, se vuoi condividere lo stato tra componenti, devi sollevare lo stato a un genitore comune e passarlo come props.

Un’altra cosa è che i hook devono essere chiamati al livello superiore di un componente, subito prima della parola chiave `return`, per mantenere lo stato e gli effetti coerenti tra i render. Questo significa che non puoi usare lo stato dentro loop, condizioni o funzioni annidate.

Ecco un esempio di gestione dello stato con il hook `useState` in un componente `Counter`:

```jsx
// Importing the useState hook
import { useState } from "react";

function Counter() {
  const initialValue = 0;

  // The state variable and setter function
  const [count, setCount] = useState(initialValue);

  return (
    <div>
      {/* Display current state value */}
      <h2>{count}</h2>

      <button onClick={() => setCount(count - 1)}>Decrement</button>
      <button onClick={() => setCount(count + 1)}>Increment</button>
    </div>
  );
}

export default Counter;
```

Nel codice sopra, abbiamo il hook `useState` importato da React. Nel componente `Counter`, `count` rappresenta lo stato attuale mentre `setCount` è la funzione set responsabile di aggiornare lo stato. Il valore attuale dello stato è `0`. L’istruzione `return` contiene il count e due pulsanti per decrementare e incrementare il count di `1`.

Puoi gestire più pezzi di stato chiamando il hook `useState` più volte. Questo è particolarmente importante quando hai variabili di stato non correlate:

```jsx
function UserProfile() {
  const [isOnline, setIsOnline] = useState(false);
  const [notifications, setNotifications] = useState(0);

  // The rest of the component logic
}
```

Puoi anche chiamare il hook `useState` più volte quando gestisci stati multipli che si aggiornano separatamente, come i campi di un modulo:

```jsx
function SignUpForm() {
  const [name, setName] = useState("");
  const [username, setUsername] = useState("");
  const [email, setEmail] = useState("");

  // The rest of the component logic
}
```

Ma in questo caso, è meglio combinare gli stati dato che fanno tutti parte dello stesso modulo:

```jsx
function SignUpForm() {
  const [formData, setFormData] = useState({
    name: "",
    username: "",
    email: "",
  });

  // The rest of the component logic
}
```

Questo è cos’è lo stato e come puoi usare il hook `useState`.
