# --interactive--

`switch`-Anweisungen und `if/else if/else`-Ketten sind beides Kontrollflussstrukturen in der Programmierung, die es uns ermöglichen, verschiedene Codeblöcke basierend auf bestimmten Bedingungen auszuführen. Sie haben jedoch unterschiedliche Eigenschaften und Anwendungsfälle.

Eine `switch`-Anweisung wertet einen Ausdruck aus und vergleicht dessen Wert mit einer Reihe von `case`-Klauseln. Wenn eine Übereinstimmung gefunden wird, wird der mit dieser `case` verknüpfte Codeblock ausgeführt. Hier ist die Grundstruktur einer `switch`-Anweisung:

```js
switch (expression) {
  case value1:
    // code to be executed if expression === value1
    break;
  case value2:
    // code to be executed if expression === value2
    break;
  default:
    // code to be executed if expression doesn't match any case
}
```

Die `break`-Anweisung am Ende jeder `case` ist entscheidend. Sie weist das Programm an, den switch-Block zu verlassen, sobald eine passende `case` ausgeführt wurde. Ohne sie würde das Programm die nachfolgenden Fälle weiter ausführen, ein Verhalten, das als „fall-through“ bekannt ist.

`switch`-Anweisungen werden typischerweise verwendet, wenn Sie eine einzelne Variable mit mehreren möglichen Werten vergleichen. Sie sind besonders nützlich, wenn viele potenzielle Bedingungen für eine einzelne Variable geprüft werden müssen. Hier ist ein Beispiel mit einer `switch`-Anweisung für die Wochentage:

:::interactive_editor

```js
let dayOfWeek = 3; 

switch (dayOfWeek) {
    case 1:
        console.log("It's Monday! Time to start the week strong.");
        break;
    case 2:
        console.log("It's Tuesday! Keep the momentum going.");
        break;
    case 3:
        console.log("It's Wednesday! We're halfway there.");
        break;
    case 4:
        console.log("It's Thursday! Almost the weekend.");
        break;
    case 5:
        console.log("It's Friday! The weekend is near.");
        break;
    case 6:
        console.log("It's Saturday! Enjoy your weekend.");
        break;
    case 7:
        console.log("It's Sunday! Rest and recharge.");
        break;
    default:
        console.log("Invalid day! Please enter a number between 1 and 7.");
}
```

:::

`switch`-Anweisungen können lesbarer und prägnanter sein, wenn viele mögliche Werte für eine einzelne Variable behandelt werden.

`if/else if`-Anweisungen hingegen sind flexibler. Sie können komplexe Bedingungen und verschiedene Variablen in jeder Klausel auswerten. Das macht sie für ein breiteres Spektrum an Szenarien geeignet. Hier ist ein Beispiel, wann Sie eine `if/else`-Anweisung einer `switch`-Anweisung vorziehen könnten:

:::interactive_editor

```js
let creditScore = 720; 
let annualIncome = 60000; 
let loanAmount = 200000; 

let eligibilityStatus;

if (creditScore >= 750 && annualIncome >= 80000) {
    eligibilityStatus = "Eligible for premium loan rates.";
} else if (creditScore >= 700 && annualIncome >= 50000) {
    eligibilityStatus = "Eligible for standard loan rates.";
} else if (creditScore >= 650 && annualIncome >= 40000) {
    eligibilityStatus = "Eligible for subprime loan rates.";
} else if (creditScore < 650) {
    eligibilityStatus = "Not eligible due to low credit score.";
} else {
    eligibilityStatus = "Not eligible due to insufficient income.";
}

console.log(eligibilityStatus);
```

:::

In diesem Beispiel haben wir das Jahreseinkommen und die Kreditwürdigkeit einer Person und prüfen, für welche Arten von Krediten sie sich qualifizieren würde. Da wir es mit komplexeren logischen Auswertungen und mehreren Variablen zu tun haben, ist es hier besser, eine `if/else`-Anweisung anstelle einer `switch`-Anweisung zu verwenden.

Es ist erwähnenswert, dass `switch`-Anweisungen in JavaScript eine strikte Gleichheit (`===`) verwenden, was bedeutet, dass sie keine Typumwandlung durchführen. Dies kann in Bezug auf Vorhersagbarkeit und das Vermeiden subtiler Bugs von Vorteil sein.

Zusammenfassend lässt sich sagen, dass sowohl `switch`-Anweisungen als auch `if/else if`-Ketten mehrzweigige Logik in Ihrem Code ermöglichen, sie jedoch unterschiedliche Stärken haben. `switch`-Anweisungen sind hervorragend geeignet, um mehrere mögliche Werte für eine einzelne Variable zu behandeln, während `if/else if`-Ketten mehr Flexibilität für komplexe Bedingungen bieten. Die Wahl zwischen ihnen hängt oft von den spezifischen Anforderungen Ihres Codes und den persönlichen oder teaminternen Programmierstilpräferenzen ab.
