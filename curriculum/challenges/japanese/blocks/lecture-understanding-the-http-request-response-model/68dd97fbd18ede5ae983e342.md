# --description--

前回のレッスンでは、サーバーがHTTPリクエストを処理し、レスポンスを返すことを学びました。そのレスポンスには、ウェブ上で見るさまざまなアセットが含まれています。

HTTPレスポンスで返されるアセットには、テキストベースのアセット、メディアアセット、フォントなどがあります。これらの各アセットには**Content-Type**ヘッダーが付いており、ブラウザーがアセットを正しく処理し表示する方法を示しています。

テキストベースのアセットは常にウェブの基本的な部分でした。実際、ウェブは純粋なテキストとして始まりました。現在、テキストベースのアセットには以下のものが含まれます：

* **HTML（Hypertext Markup Language）**：あらゆるウェブページの基本構造です。HTMLがなければウェブページは存在できません。
    
* **CSS（Cascading Stylesheet）**：ウェブページの見た目を制御するアセットです。CSSを使ってHTMLに色、フォント、レイアウトなどのスタイルを適用します。
    
* **JavaScript**：HTMLとCSSにインタラクティブ性や動的なコンテンツを追加します。
    
* **JSONとXML**：これらの2つのフォーマットは、サーバーとブラウザー間で構造化されたデータを転送するために使われます。
    
メディアアセットには画像、音声、動画が含まれます。これらは視覚や音でウェブに命を吹き込みます。

多くのウェブサイトは視覚情報を伝えるために画像に依存しています。画像には`JPG`、`JPEG`、`PNG`、`WEBP`、`GIF`などのさまざまなフォーマットがあります。

YouTubeやVimeoのようなウェブサイトは、`MP4`、`MOV`、`AVI`、`WebM`などのフォーマットで動画コンテンツを配信しています。

音声は多くのプラットフォームで重要な役割を果たしており、ユーザーが音楽やポッドキャスト、その他の音声コンテンツをストリーミングできます。一般的な音声フォーマットには`MP3`、`AAC`、Ogg、`WAV`があります。SpotifyやYouTube Musicのようなプラットフォームはこれらのフォーマットを使って高品質な音声体験を提供しています。

HTTPレスポンスで返されるその他のアセットには、`PDF`、`DOCX`、`PAGES`のようなドキュメント、`ZIP`や`RAR`のような圧縮アーカイブ、`WOFF`、`WOFF2`、`TTF`のようなフォントファイル、`EXE`のような実行ファイルがあります。

以下はこれらの一般的なアセットとそのContent-Typeの一覧です：

| アセットの種類 | Content-Typeヘッダー |
| --- | --- |
| HTML | text/html |
| CSS | text/css |
| JavaScript | application/javascript |
| JPEG | image/jpeg |
| PNG | image/png |
| GIF | image/gif |
| SVG | image/svg+xml |
| WebP | image/webp |
| WOFF | font/woff |
| WOFF2 | font/woff2 |
| TTF | font/ttf |
| OTF | font/otf |
| MP4 | video/mp4 |
| WebM | video/webm |
| Ogg (動画) | video/ogg |
| MP3 | audio/mpeg |
| AAC | audio/aac |
| WAV | audio/wav |
| Ogg (音声) | audio/ogg |
| JSON | application/json |
| XML | application/xml |
| PDF | application/pdf |
| ZIP | application/zip |
