# --interactive--

CSSでは、`vh` と `vw` はビューポートに対する相対単位で、ブラウザウィンドウのサイズに基づいて要素のサイズを指定できます。これらの単位は、異なる画面サイズに対応するレスポンシブデザインを作成する際に特に役立ちます。

`vh` は「ビューポートの高さ」を表し、`1vh` はビューポートの高さの1%に相当します。

同様に、`vw` は「ビューポートの幅」を表し、`1vw` はビューポートの幅の1%に相当します。

つまり、要素の高さを `100vh` に設定すると、デバイスの実際のピクセル寸法に関係なく、ビューポートの高さ全体を占めることになります。

これらの単位は、全画面レイアウトや画面の特定の比率を維持する要素を作成したい場合に特に便利です。

例えば、ヒーローセクションを常に画面全体にフィットさせたい場合に使うことができます（例を操作するにはインタラクティブエディターを有効にしてください）。

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css" />

<section class="hero">
  <h1>100vh / 100vw Example</h1>
  <p>This section fills the entire browser window.</p>
</section>
```

```css
body {
  margin: 0;
  font-family: sans-serif;
  border: 5px dashed #333;
}

.hero {
  height: 100vh;
  width: 100vw;
  background-color: #add8e6; 
  padding: 2em;
}

.hero h1 {
  font-size: 2em;
  margin-bottom: 0.5em;
}

.hero p {
  font-size: 1em;
}
```

:::

このCSSは、ヒーローセクションがデバイスの画面サイズに関係なく常にビューポートと同じサイズになることを保証します。

`vh` と `vw` の単位は、タイポグラフィにも使えて、レスポンシブなテキストサイズを作成できます。

インタラクティブエディターを有効にして、プレビューウィンドウのサイズを調整し、テキストがビューポートのサイズに合わせてどのようにスケールするかを確認してください：

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css" />
<h1>Responsive Heading</h1>
<p>The h1 heading scales with the viewport width.</p>
```

```css
h1 {
  font-size: 5vw;
}
```

:::

これは `h1` 要素のフォントサイズをビューポート幅の5%に設定し、ブラウザウィンドウのサイズに応じてテキストが滑らかにスケールするようにします。

`vh` と `vw` の単位の利点の一つは、ビューポートサイズの変化にリアルタイムで反応することです。つまり、ユーザーがブラウザウィンドウのサイズを変更すると、これらの単位でサイズ指定された要素はページを再読み込みすることなく自動的に調整されます。ただし、これらの単位は慎重に使う必要があります。例えば、フォントサイズを `vw` 単位だけで設定すると、狭い画面では文字が小さくなりすぎたり、広い画面では大きくなりすぎたりすることがあります。

また、モバイルデバイスでは、ブラウザのアドレスバーが表示されたり消えたりするとビューポートの高さが変わるため、`vh` 単位を多用すると予期しないレイアウトのズレが起こる場合があります。

まとめると、`vh` と `vw` の単位は、ビューポートサイズに適応するレスポンシブなレイアウトや要素を作成するための強力なツールです。全画面セクションの作成、アスペクト比の維持、滑らかにスケールするデザインの作成に特に役立ちます。ただし、最良のユーザー体験を実現するために、他のCSS手法と組み合わせて慎重に使うことをお勧めします。
