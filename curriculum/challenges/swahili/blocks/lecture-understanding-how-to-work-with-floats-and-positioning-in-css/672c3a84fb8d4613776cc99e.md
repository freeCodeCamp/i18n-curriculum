# --interactive--

Katika CSS, uwekaji wa nafasi unaturuhusu kudhibiti jinsi vipengele vinavyopangwa kwenye ukurasa. Aina mbili za kawaida za uwekaji nafasi ni uwekaji nafasi wa static na uwekaji nafasi wa relative. Kwa chaguo-msingi, vipengele vimewekwa kwa static. Hii ina maana kwamba vinafuata mtiririko wa kawaida wa hati, moja baada ya nyingine, kutoka juu kwenda chini, kutoka kushoto kwenda kulia.

Uwekaji nafasi wa static ni chaguo-msingi kwa vipengele vyote na hauhitaji tamko maalum katika CSS. Hutaona tofauti yoyote unapotumia uwekaji nafasi wa static kwa sababu unahifadhi vipengele mahali ambako kawaida vinapatikana katika hati. Hapa kuna mfano wa aya inayotumia uwekaji nafasi wa static kwa chaguo-msingi:

:::interactive_editor

```html
<p>This paragraph is statically positioned.</p>
```

:::

Inapoonyeshwa kwenye kivinjari, aya hii itaonekana mahali pake pa asili, ikifuata mtiririko wa kawaida wa hati. Hakuna mtindo maalum unaotumika kuibadilisha nafasi, na inabaki mahali ambapo ingeonekana kawaida katika mpangilio.

Uwekaji nafasi wa relative, kwa upande mwingine, unaruhusu kipengele kusogezwa kutoka kwenye nafasi yake ya kawaida bila kuathiri mtiririko wa hati. Fikiria kama kusogeza kipengele kutoka kwenye nafasi yake ya static kwa chaguo-msingi kwa kumpa kuratibu mpya. Hapa ni jinsi unavyoweza kutumia uwekaji nafasi wa relative:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<p class="relative">This paragraph is positioned relatively.</p>
```

```css
body {
  border: solid 1px black;
}

.relative {
  position: relative;
  top: 30px;
  left: 30px;
}
```

:::

Katika mfano huu, aya itaonekana `30px` chini na `30px` kulia kutoka kwenye nafasi yake ya awali. Hata hivyo, nafasi ambayo ingeweza kuchukua katika mtiririko wa kawaida inabaki kuhifadhiwa. Uwekaji nafasi wa relative ni muhimu sana unapotaka kusogeza kipengele kidogo bila kuathiri mpangilio mzima.
