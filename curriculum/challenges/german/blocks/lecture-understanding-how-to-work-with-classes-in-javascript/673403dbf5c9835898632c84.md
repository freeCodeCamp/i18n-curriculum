# --interactive--

Statische Eigenschaften und Methoden gehören zur Klasse selbst, nicht zu den einzelnen Instanzen der Klasse. Sie können direkt über den Klassennamen auf sie zugreifen, ohne eine Instanz der Klasse zu erstellen. Sie werden innerhalb von Klassen definiert, um zusammengehörige Funktionalität zu kapseln.

Sie können eine statische Methode definieren, indem Sie das Schlüsselwort `static` vor den Namen der Methode schreiben.

```js
class MyClass {
  static staticMethod() { ... }
}
```

Dann können Sie die statische Methode direkt auf der Klasse aufrufen, indem Sie die Punktnotation verwenden und alle notwendigen Argumente übergeben:

```js
MyClass.staticMethod();
```

Beachten Sie, dass Sie die Methode aufrufen können, ohne eine Instanz der Klasse zu erstellen. Das ist eines der Hauptmerkmale statischer Methoden.

Hier ist ein Beispiel. Angenommen, wir erstellen eine Film-Anwendung und möchten Filme basierend auf ihrer Bewertung vergleichen können.

Wir könnten diese Vergleichsmethode als eine höherstufige Methode betrachten, die nicht spezifisch für einen einzelnen Film ist:

```js
if (movieA.rating < movieB.rating) {
  console.log(`${movieB.title} has a higher rating.`);
} 
```

Es ist wie eine allgemeinere Methode, die mit der `Movie`-Klasse zusammenhängt.

Aus Gründen der Lesbarkeit und Wartbarkeit wäre es hilfreich, sie innerhalb der `Movie`-Klasse zu definieren, um alle verwandten Methoden relativ nah beieinander zu halten.

Daher ist dies ein perfekter Kandidat für eine statische Methode. Sie sehen sie hier, direkt unter dem Konstruktor:

:::interactive_editor

```js
class Movie {

  constructor(title, rating) {
    this.title = title;
    this.rating = rating;
  }

  static compareMovies(movieA, movieB) {
    if (movieA.rating > movieB.rating) {
      console.log(`${movieA.title} has a higher rating.`);
    } else if (movieA.rating < movieB.rating) {
      console.log(`${movieB.title} has a higher rating.`);
    } else {
      console.log("These movies have the same rating.");
    }
  }

}

let movieA = new Movie("Movie A", 80);
let movieB = new Movie("Movie B", 45);
console.log(movieA);
```

:::

Die statische Methode ist mit dem Schlüsselwort `static` definiert und heißt `compareMovies`. Sie hat zwei Parameter: `movieA` und `movieB`. Diese werden Instanzen der `Movie`-Klasse sein.

Wir vergleichen sie basierend auf ihrer Bewertung, auf einer Skala von `0` bis `100`. Diese Logik wird mit einer Bedingung implementiert und gibt eine passende Nachricht aus, je nachdem, welcher Film eine höhere Bewertung hat.

Sobald die Methode definiert ist, können Sie sie auf der Klasse aufrufen. Zuerst müssen Sie jedoch die Argumente in Ihrem Programm bereitstellen und definieren.

In diesem Fall nimmt die Methode zwei Film-Instanzen als Argumente, also definieren wir diese Instanzen hier:

```js
let movieA = new Movie("Movie A", 80);
let movieB = new Movie("Movie B", 45);
```

Sie sehen, dass `movieA` eine höhere Bewertung als `movieB` hat. Sehen wir uns die Ausgabe dieser Methode an.

Um die Methode aufzurufen, müssen Sie nur die Punktnotation auf der Klasse selbst verwenden.
Sie schreiben den Namen der Klasse (`Movie`), gefolgt von einem Punkt, und dann den Namen der statischen Methode (`compareMovies`).

Dann übergeben Sie die Argumente in runden Klammern. In diesem Fall sind das die beiden Film-Instanzen, die die Methode benötigt.

```js
Movie.compareMovies(movieA, movieB);
```

Hier ist das aktualisierte Beispiel:

:::interactive_editor

```js
class Movie {

  constructor(title, rating) {
    this.title = title;
    this.rating = rating;
  }

  static compareMovies(movieA, movieB) {
    if (movieA.rating > movieB.rating) {
      console.log(`${movieA.title} has a higher rating.`);
    } else if (movieA.rating < movieB.rating) {
      console.log(`${movieB.title} has a higher rating.`);
    } else {
      console.log("These movies have the same rating.");
    }
  }

}

let movieA = new Movie("Movie A", 80);
let movieB = new Movie("Movie B", 45);
Movie.compareMovies(movieA, movieB);
console.log(movieA);
```

:::

Statische Methoden sind auch hilfreich, um „Factory“-Methoden zu implementieren. Eine Factory-Methode ist eine Methode, die Sie zusätzlich zum Konstruktor definieren, um Objekte basierend auf bestimmten Kriterien zu erstellen.

Hier ist ein Beispiel mit einer `Pizza`-Klasse. Die statische Methode `createMargherita` ist eine Factory-Methode, die Sie aufrufen können, um eine Margherita-Pizza-Instanz mit bereits festgelegtem Typ und Preis zu erstellen.

```js
class Pizza {
  constructor(type, price) {
    this.type = type;
    this.price = price;
  }

  static createMargherita() {
    return new this("Margherita", 6.99);
  }
}
```

Das bringt auch etwas sehr Wichtiges über statische Methoden zur Sprache.

Der Wert des Schlüsselworts `this` in statischen Methoden ist die Klasse selbst, da die statische Methode zur Klasse gehört.

Deshalb können wir `this` verwenden, um eine neue Instanz der `Pizza`-Klasse zu erstellen.

Wenn Sie diese Methode auf der `Pizza`-Klasse selbst aufrufen und die zurückgegebene Instanz einer Variablen zuweisen, wie in diesem Beispiel:

```js
let myPizza = Pizza.createMargherita();
```

können Sie sie überall in Ihrem Code verwenden. Zum Beispiel können Sie sie in der Konsole ausgeben:

```js
console.log(myPizza);
```

Das ist die Ausgabe:

```js
Pizza { type: 'Margherita', price: 6.99 }
```

Sie können auch die Punktnotation verwenden, um ihre Methoden aufzurufen und auf ihre Eigenschaften zuzugreifen, so:

```js
console.log(myPizza.type);
```

Hier ist das vollständige Beispiel:

```js
class Pizza {
  constructor(type, price) {
    this.type = type;
    this.price = price;
  }

  static createMargherita() {
    return new this("Margherita", 6.99);
  }
}

let myPizza = Pizza.createMargherita();
console.log(myPizza);
console.log(myPizza.type);
```

Zusätzlich zu Methoden können Sie auch statische Eigenschaften mit dem Schlüsselwort `static` definieren.

In diesem Beispiel haben wir eine statische `numberOfPizzasSold`-Eigenschaft.

```js
class Pizza {
  static numberOfPizzasSold = 0;

  constructor(type) {
    this.type = type;
    Pizza.numberOfPizzasSold++;
  }
}
```

Sie ist statisch, weil sie nicht zu einer bestimmten Pizza-Instanz gehört, sondern zur Klasse selbst.

Sie hat einen Anfangswert von `0` und wird jedes Mal aktualisiert, wenn eine neue Instanz erstellt wird.

Wenn Sie zwei Pizza-Instanzen erstellen, wird der Wert zweimal aktualisiert:

```js
let pizza1 = new Pizza("Margherita");
let pizza2 = new Pizza("Neapolitan");
```

Um auf den Wert einer statischen Eigenschaft zuzugreifen, müssen Sie nur die Punktnotation auf der Klasse selbst verwenden, da die Eigenschaft zur Klasse gehört.

```js
class Pizza {
  static numberOfPizzasSold = 0;

  constructor(type) {
    this.type = type;
    Pizza.numberOfPizzasSold++;
  }
}

let pizza1 = new Pizza("Margherita");
let pizza2 = new Pizza("Neapolitan");

console.log(Pizza.numberOfPizzasSold);
```

In diesem Fall ist die Ausgabe `2`, weil zwei Pizzas verkauft wurden.

Das sind die Grundlagen statischer Eigenschaften und Methoden in JavaScript. Das Verständnis statischer Mitglieder ist entscheidend, um wiederverwendbare und effiziente Klassen zu erstellen.

## --text--

Was ist der Hauptunterschied zwischen einer statischen Methode und einer Instanzmethode in JavaScript?

## --answers--

Statische Methoden können nur innerhalb der Klasse verwendet werden, während Instanzmethoden außerhalb der Klasse verwendet werden können.

### --feedback--
