# --description--

多くの言語やフレームワークは、ファイルやフォルダーを操作する方法を提供しています。Node.jsでは、`fs`、つまりファイルシステムモジュールを使ってこれを行えます。

このレッスンでは、Nodeの`fs`モジュールに焦点を当て、その動作と提供されるメソッドを使う3つの異なる方法について学びます。

Nodeの`fs`モジュールは、ファイルやフォルダーの操作に使えるメソッドを提供しており、開く・閉じる、読み書き、削除などの操作が含まれます。

`fs`モジュールはNode.jsの標準モジュールなので、Node.jsが環境にインストールされていれば利用可能です。モジュールを使うには、次のようにインポートします。

```js
const fs = require("fs");
```

`fs`モジュールの柔軟性は、メソッドを同期的にも非同期的にも使えることです。`fs`モジュールのメソッドはデフォルトで非同期ですが、すべてのメソッドに同期版もあります。

```js
fs.writeFile() // Asynchronous file writing
fs.writeFileSync() // Synchronous file writing

fs.readFile() // Asynchronous file reading
fs.readFileSync() // Synchronous file reading

fs.open() // Opens a file
fs.openAsBlob() // Opens as blob
fs.openSync() // Synchronous open
fs.opendir() // Opens directory
fs.opendirSync() // Synchronous directory open
```

それだけではありません。ニーズに応じて、メソッドは3つの方法で使えます。

- コールバックを使う方法、例えば`fs.writeFile()`（非同期）
- `async/await`構文を好む場合はPromiseを使う方法、例えば`fs.promises.writeFile()`
- 同期的に使う方法、例えば`fs.writeFileSync()`

メソッドの非同期使用の基本構文は次の通りです。

```js
fs.writeFile("filePath", "content", "utf8", (err) => {
  if (err) {
    throw err;
  }
  console.log("File written to!");
});
```

Promise版は`fs/promises`からインポートするか、`promises`のインポートに`fs`をチェーンできます。構文は次の通りです。

```js
async function promisesExample() {
  try {
    await fs.promises.writeFile("filePath", "content", "utf8");
    console.log("File written to!");
  } catch (err) {
    console.error("Error:", err);
  }
}

promisesExample();
```

同期版はこちらです。

```js
try {
  fs.writeFileSync("filePath", "content", "utf8");
  console.log("File written to!");
} catch (err) {
  console.error("Error:", err);
}
```

同期的にメソッドを使うとブロッキングになります。つまり、プログラムは処理が終わるまで停止し、次のコード行に進みません。

短いスクリプト、小規模なプロジェクト、一度きりのタスクには同期メソッドで問題ありません。しかし実際のアプリケーションでは、ブロッキングI/O（入力/出力）は問題になります。Nodeがファイルシステムの処理を待つ間、他の部分がフリーズしてしまい、パフォーマンスやユーザー体験が悪化します。

だから非同期メソッドが存在します。Nodeがバックグラウンドでファイル操作を処理している間、プログラムの他の部分は動き続けられます。従来はコールバックを使っていましたが、アプリが大きくなると複雑になりがちです。

Promiseと`async/await`はその問題を解決します。非ブロッキングのまま、コードは通常の同期コードのように読みやすく、保守も簡単です。

このため、Promiseベースの方法が現在は一般的に好まれており、このレッスンの残りでもそれを使います。

`writeFile()`メソッドは既存のファイルに書き込みます。ファイルがなければ、カレントディレクトリに作成して指定した内容を書き込みます。

`writeFile()`メソッドの使い方は次の通りです。

```js
const fs = require("fs/promises");

async function writeToFile() {
  try {
    await fs.writeFile(
      "article.md",
      "## Node `fs` Module: The Complete Guide",
      "utf8",
    );
    console.log("File written to!");
  } catch (err) {
    console.error("Error writing to file:", err);
  }
}

writeToFile(); // File written to!
```

その後のarticle.mdファイルの内容はこうなります。

```md
## Node `fs` Module: The Complete Guide
```

`appendFile()`メソッドは既存ファイルの内容に追加できます。使い方はこちらです。

```js
const fs = require("fs/promises");

async function appendToFile() {
  try {
    await fs.appendFile(
      "article.md",
      "\nIn this article, you will learn all there is to know about the Node fs module...",
      "utf8",
    );
    console.log("File appended to!");
  } catch (err) {
    console.log("Error appending to file:", err);
  }
}

appendToFile(); // File appended to!
```

`readFile()`メソッドはファイルの内容を確認できます。

```js
const fs = require("fs/promises");

async function readFileContent() {
  try {
    const fileContent = await fs.readFile("article.md", "utf8");
    console.log("File content:", fileContent);
  } catch (err) {
    console.log("File read successfully");
  }
}

readFileContent();

/*
File content: ## Node `fs` Module: The Complete Guide

In this article, you will learn all there is to know about the Node fs module...
*/
```

`utf8`文字エンコーディングを指定しないと、ファイルの内容はバッファとして取得されます。

```js
async function readFileContent() {
  try {
    const fileContent = await fs.readFile("article.md");
    console.log("File content:", fileContent);
  } catch (err) {
    console.log("File read successfully");
  }
}

readFileContent();

/*
File content: <Buffer 23 23 20 4e 6f 64 65 20 60 66 73 60 20 4c 69 62 72 61 72 79 3a 20
 54 68 65 20 43 6f 6d 70 6c 65 74 65 20 47 75 69 64 65 0a 0a 49 6e 20 74 68 69 73 20 ... 
72 more bytes>
*/
```

最後に、`unlink()`メソッドはファイルを削除できます。

```js
const fs = require("fs/promises");

async function deleteFile() {
  try {
    await fs.unlink("article.ts");
    console.log("File deleted successfully!");
  } catch (err) {
    console.error("Error deleting file:", err);
  }
}

deleteFile();
```

ファイルが存在しない場合は、次のようなエラーメッセージが表示されます。

```bash
Error deleting file: Error: ENOENT: no such file or directory, unlink 'article.ts'
```

存在すればファイルは削除され、設定した成功メッセージが表示されます。
