# --description--

API геолокації надає спосіб для вебсайтів запитувати місцезнаходження користувача. Важливо зазначити, що з міркувань конфіденційності користувач повинен надати дозвіл, перш ніж його місцезнаходження можна буде отримати через вебсайт.

Основний метод, на якому ми сьогодні зосередимося, — це `getCurrentPosition`. Цей метод використовується для збору географічного місцезнаходження пристрою. Ось приклад того, як ви можете використати `getCurrentPosition`:

```js
navigator.geolocation.getCurrentPosition(
  (position) => {
    console.log("Latitude: " + position.coords.latitude);
    console.log("Longitude: " + position.coords.longitude);
  },
  (error) => {
    console.log("Error: " + error.message);
  }
);
```

У цьому коді ми викликаємо `getCurrentPosition` і передаємо йому функцію, яка буде викликана, коли позиція буде успішно отримана. Цей об’єкт позиції містить різні дані, але ми зосереджуємося лише на `latitude` і `longitude`.

Якщо виникає проблема з отриманням позиції, помилка буде виведена в консоль.

Метод `getCurrentPosition` використовує GPS, Wi-Fi мережі або геолокацію за IP-адресою, залежно від пристрою та його налаштувань. Після того, як місцезнаходження знайдено, викликається функція зворотного виклику успіху з об’єктом позиції.

Об’єкт позиції містить різні властивості, найпоширенішими з яких є `latitude` і `longitude`, але також може включати `altitude`, `accuracy`, `speed` і `heading` тощо.

Одне важливе зауваження при використанні геолокації — це конфіденційність користувача. Поясніть своїм користувачам, чому вам потрібні їхні дані про місцезнаходження і як ви їх використовуватимете.
