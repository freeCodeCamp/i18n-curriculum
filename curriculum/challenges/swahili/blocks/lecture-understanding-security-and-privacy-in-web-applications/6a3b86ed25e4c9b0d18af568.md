# --description--

Faili ya `.env` na maktaba ya `dotenv` ni zana zinazotumika katika ukuzaji wa programu za kisasa kutenganisha data nyeti za usanidi kutoka kwenye msimbo wa programu.

Faili ya `.env` ni faili ya maandishi rahisi iliyowekwa kwenye mzizi wa saraka ya mradi ambayo huhifadhi mipangilio ya usanidi na siri kama jozi za ufunguo-thamani.

Badala ya kuweka funguo za API, nywila za hifadhidata, au nambari za port moja kwa moja kwenye msimbo wa chanzo na kuhatarisha kuzifunua ikiwa msimbo utawekwa kwenye hifadhidata ya umma kama GitHub, wasanidi programu huzihifadhi kwa ndani kwenye faili hii.

Muundo wa faili ya `.env` ni rahisi:

```bash
PORT=3000
DB_HOST=localhost
API_KEY=xyz123secretkey
```

Kwa kuwa faili hii ina funguo nyeti sana, ni desturi ya kawaida kuongeza `.env` kwenye faili `.gitignore` ya mradi ili isifuatiliwe na Git.

Kifurushi cha `dotenv` ni moduli nyepesi inayounganisha faili ya `.env` na programu yako.

Programu yako inapoanza, `dotenv` husoma faili ya `.env`, huchanganuzi funguo, na kuziingiza kwenye mazingira ya wakati wa utekelezaji. Katika programu ya Node.js, kwa mfano, unaanzisha kifurushi hicho juu kabisa ya faili yako ya kuingilia. Mara tu kinapopakuliwa, vigezo vinapatikana kwa kiwango cha ulimwengu kupitia kitu cha `process.env`.

```javascript
const express = require('express');
const app = express();

// initialize and configure dotenv
require('dotenv').config();

// access the variables securely from the environment
const port = process.env.PORT || 5000;
const apiKey = process.env.API_KEY;

app.get('/', (req, res) => {
  res.send(`Connected successfully using API Key: ${apiKey}`);
});

app.listen(port, () => {
  console.log(`Server running on port ${port}`);
});
```

Kutumia mpangilio huu kunatoa faida kuu mbili. Kwanza, kunaboresha usalama kwa kuweka nyaraka binafsi mbali na msimbo wa chanzo. Pili, kunatoa unyumbufu kwa kuruhusu msimbo uleule ufanye kazi tofauti katika mazingira ya ukuzaji, majaribio, na uzalishaji kwa kubadilisha faili ya `.env` ya ndani kwenye seva husika ya uhudumu wa tovuti.
