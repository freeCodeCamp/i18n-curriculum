# --description--

API ya Hifadhi ya Mtandao hutoa njia kwa vivinjari kuhifadhi jozi za ufunguo-thamani ndani ya kivinjari, ikiruhusu wasanidi programu kuhifadhi taarifa zinazoweza kutumika katika upakiaji wa kurasa na vikao tofauti.

Sehemu kuu mbili katika API ya Hifadhi ya Mtandao ni `localStorage` na `sessionStorage`.

`localStorage` ni sehemu ya API ya Hifadhi ya Mtandao inayoruhusu data kudumu hata baada ya dirisha la kivinjari kufungwa au ukurasa kuonyeshwa upya. Data hii hubaki kupatikana hadi itakapotolewa wazi na programu au mtumizi.

`sessionStorage` ni sehemu nyingine ya API ya Hifadhi ya Mtandao inayohifadhi data kwa muda wa kikao cha ukurasa, ikimaanisha data inapatikana mradi tu kichupo au dirisha la kivinjari bado likiwa wazi. Hata hivyo, tofauti na `localStorage`, data katika `sessionStorage` hufutwa wakati kichupo au dirisha linapofungwa. Utajifunza zaidi kuhusu `sessionStorage` katika funzo lijalo.

Matumizi ya kawaida ya `localStorage` ni pamoja na kuhifadhi mipangilio ya mtumizi, kama mandhari au mapendeleo ya lugha, kukumbuka data za fomu katika vikao vya kivinjari, na kuhifadhi vipande vidogo vya taarifa ili kuboresha utendaji wa programu za mtandao.

Kuhifadhi kwa muda (caching) kunahusu kuhifadhi data inayofikiliwa mara kwa mara katika eneo la hifadhi la muda, linalojulikana kama cache, ili maombi yanayofuata ya data hiyo yaweze kutimizwa haraka zaidi bila ya kuhitaji kuhesabu tena au kuipata kutoka chanzo cha data chenye kasi ya chini, kama hifadhidata au seva ya nje.

Baadhi ya njia za kawaida za `localStorage` ni pamoja na njia za `setItem`, `getItem`, `removeItem` na `clear`.

Hapa kuna mfano wa kutumia njia ya `setItem()` ambayo huhifadhi jozi ya ufunguo-thamani katika `localStorage`.

```js
localStorage.setItem('username', 'JaneDoe');
```

Kisha ikiwa tunataka kupata thamani ya ufunguo fulani kutoka `localStorage`, tunaweza kutumia njia ya `getItem()`.

```js
let username = localStorage.getItem('username');
console.log(username); // JaneDoe
```

Kutoa kipengee kutoka `localStorage` kwa kutumia ufunguo wake, unaweza kutumia njia ya `removeItem()`.

```js
localStorage.removeItem('username');
```

Kufuta data zote katika `localStorage`, unaweza kutumia njia ya `clear()`.

```js
localStorage.clear();
```

Sasa, tuchunguze mfano ambapo tunatumia `localStorage` kuhifadhi mandhari inayopendekezwa na mtumizi.

```js
// Store the user's theme preference
localStorage.setItem('theme', 'dark');

// Retrieve the stored theme preference
const userTheme = localStorage.getItem('theme');
console.log(userTheme); // 'dark'

// Remove the theme preference
localStorage.removeItem('theme');

// Clear all localStorage data
localStorage.clear();
```

Katika mfano huu:

- Kwanza tunahifadhi chaguo la mandhari (`dark`) kwa mtumizi.

- Kisha tunapata mandhari hiyo na kuionyesha kwenye konsoli.

- Mwishowe, tunaonyesha jinsi ya kuondoa kipengee maalum au kufuta data zote zilizohifadhiwa.

`localStorage` ni muhimu sana kuhifadhi vipande vidogo vya data vinavyohitaji kudumu kati ya vikao, lakini ni muhimu kutambua kuwa `localStorage` haipaswi kutumika kuhifadhi taarifa nyeti, kama nywila, kwa sababu inaweza kusababisha hatari za usalama.
