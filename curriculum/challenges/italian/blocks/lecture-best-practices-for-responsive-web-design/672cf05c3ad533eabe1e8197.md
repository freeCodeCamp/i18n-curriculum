# --interactive--

Le media query sono un componente fondamentale del design web responsivo, che permette agli sviluppatori di applicare stili diversi in base alle caratteristiche del dispositivo o del browser dell’utente. Offrono un modo per adattare la presentazione del contenuto a una varietà di dispositivi senza modificare il contenuto stesso.

Alla base, una media query consiste in un tipo di media e una o più espressioni che verificano condizioni specifiche. Se queste condizioni sono vere, vengono applicati gli stili corrispondenti. Questo meccanismo consente di creare design responsivi che si adattano a diverse dimensioni dello schermo, risoluzioni e capacità del dispositivo.

La sintassi base di una media query in CSS è la seguente:

```css
@media mediatype and (feature: value) {
  /* CSS rules go here */
}
```

In questa struttura, mediatype specifica il tipo di media a cui si applica la query, mentre la coppia feature: value definisce la condizione che deve essere soddisfatta per applicare gli stili.

I tipi di media descrivono la categoria generale di un dispositivo. Parliamo dei tipi di media più comunemente usati:

- `all` è adatto a tutti i dispositivi. Questo è il valore predefinito se non viene specificato alcun tipo di media.

- `print` è destinato a materiale impaginato e documenti visualizzati su schermo in modalità anteprima di stampa.

- `screen` è pensato principalmente per gli schermi.

In passato esistevano più tipi di media, come `handheld` e `tv`, ma la maggior parte di questi è stata deprecata a favore dell’uso delle funzionalità per indirizzare i dispositivi in modo più preciso.

Le funzionalità media descrivono caratteristiche specifiche dell’agente utente, del dispositivo di output o dell’ambiente. Parliamo di alcune delle funzionalità media più usate. `width` e `height` si riferiscono alla larghezza e all’altezza della viewport e sono spesso usate con i prefissi `min-` o `max-` per query di intervallo. Ecco un esempio:

```css
@media screen and (min-width: 768px) {
  /* Styles for screens at least 768px wide */
}
```

`aspect-ratio` descrive il rapporto tra larghezza e altezza della viewport. Ecco un esempio:

```css
@media screen and (aspect-ratio: 16/9) {
  /* Styles for screens with a 16:9 aspect ratio */
}
```

La funzionalità `orientation` indica se il dispositivo è in orientamento orizzontale o verticale. Ecco un esempio:

```css
@media screen and (orientation: landscape) {
  /* Styles for landscape orientation */
}
```

La funzionalità `resolution` descrive la risoluzione del dispositivo di output in punti per pollice (dpi) o punti per centimetro (dpcm). Ecco un esempio:

```css
@media screen and (min-resolution: 300dpi) {
  /* Styles for high-resolution screens */
}
```

La funzionalità `hover` verifica se il meccanismo di input principale può passare sopra gli elementi (hover). Ecco un esempio:

```css
@media (hover: hover) {
  /* Styles for devices that support hover */
}
```

La funzionalità `prefers-color-scheme` rileva se l’utente ha richiesto un tema chiaro o scuro. Ecco un esempio:

```css
@media (prefers-color-scheme: dark) {
  /* Styles for dark mode */
}
```

Le media query possono anche combinare più condizioni usando operatori logici. L’operatore `and` serve a combinare più funzionalità media, mentre `not` e `only` possono essere usati per negare o isolare media query. Ecco un esempio che combina più funzionalità:

```css
@media screen and (min-width: 768px) and (orientation: landscape) {
  /* Styles for landscape screens at least 768px wide */
}
```

È anche possibile indirizzare più query in un elenco separato da virgole, che funziona come un operatore "o":

```css
@media screen and (min-width: 768px), print {
  /* Styles for screens at least 768px wide OR for print */
}
```

Quando si lavora con le media query, è importante considerare la cascade. Le media query non aumentano la specificità: raggruppano solo regole condizionali. Le normali regole della cascade CSS si applicano ancora all’interno di ogni media query.

In pratica, le media query sono spesso usate per creare layout responsivi. Un pattern comune è definire uno stile base per i dispositivi mobili e poi usare le media query per migliorare il layout per schermi più grandi:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<div class="container">
  <p>Lorem ipsum dolor sit amet, consectetur adipiscing elit. Integer nec odio. Praesent libero. Sed cursus ante dapibus diam.</p>
  <p>Sed nisi. Nulla quis sem at nibh elementum imperdiet. Duis sagittis ipsum. Praesent mauris. Fusce nec tellus sed augue semper porta.</p>
  <p>Mauris massa. Vestibulum lacinia arcu eget nulla. Class aptent taciti sociosqu ad litora torquent per conubia nostra, per inceptos himenaeos.</p>
</div>
```

```css
/* Base styles for mobile */
.container {
  width: 100%;
  padding: 15px;
}

/* Styles for tablets */
@media screen and (min-width: 768px) {
  .container {
    width: 750px;
    margin: 0 auto;
  }
}

/* Styles for desktops */
@media screen and (min-width: 1024px) {
  .container {
    width: 960px;
  }
}
```

:::

Questo approccio, noto come design responsivo "mobile-first", garantisce che gli stili base siano adatti ai dispositivi mobili, con miglioramenti aggiunti per schermi più grandi.

In conclusione, le media query sono uno strumento potente in CSS che permette di creare design web responsivi e adattabili. Capendo come usare diversi tipi e funzionalità media, gli sviluppatori possono realizzare siti web che offrono esperienze ottimali su una vasta gamma di dispositivi e preferenze. Con l’evoluzione delle tecnologie web, rimanere aggiornati sulle nuove funzionalità media aiuta a creare design responsivi più raffinati e user-friendly.
