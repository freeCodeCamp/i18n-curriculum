# --description--

Поліморфізм — це наступна ключова концепція об’єктно-орієнтованого програмування (ООП), про яку ми поговоримо.

З поліморфізмом ви маєте доступ до інтерфейсу, де можна взаємодіяти з багатьма об’єктами одного типу.

Давайте детальніше розглянемо поліморфізм і те, як він дозволяє повторно використовувати код.

Поліморфізм дозволяє методам у різних класах мати однакову назву, але виконувати різні завдання. Ви викликаєте один і той самий метод на різних об’єктах, і кожен реагує по-своєму.

Ось базовий приклад поліморфізму:

```py
class A:
   def action(self): ...

class B:
   def action(self): ...

class C:
   def action(self): ...

Class().method()  # Works for A, B, or C
```

Ось приклад із різними звуками тварин, щоб показати поліморфізм:

```py
class Cat:
   def speak(self):
       return "A cat meow"

class Bird:
   def speak(self):
       return "A bird tweet"
  
class Monkey:
   def speak(self):
       return "A monkey ooh ooh aah aah ooh ooh aah aah"

def animal_sound(animal):
   print(animal.speak())

animal_sound(Cat())
animal_sound(Bird())
animal_sound(Monkey())
```

У цьому прикладі `animal_sound()` — це функція, яка приймає будь-який об’єкт із методом `speak()`.

Коли ви передаєте `Cat`, `Bird` або `Monkey`, вона викликає метод `speak()` об’єкта і виводить результат. Оскільки кожен клас визначає `speak()` по-різному, ви отримуєте різні вихідні дані з однієї й тієї самої функції. Ось як працює поліморфізм.

Ось ще один приклад, цього разу з екземплярами та атрибутом:

```py
class Twitter:
   def __init__(self, content):
       self.content = content

   def post(self):
       return f"🐦 Tweet: '{self.content}' (280 chars max)"

class Instagram:
   def __init__(self, content):
       self.content = content

   def post(self):
       return f"📸 Instagram Post: '{self.content}' + ✨ filters"

class LinkedIn:
   def __init__(self, content):
       self.content = content

   def post(self):
       return f"💼 LinkedIn Article: '{self.content}' (Professional Mode)"

def start(social_media):
   print(social_media.post())  # Calls .post() on any object

# Instances
tweet = Twitter('Just learned Python polymorphism!')
photo = Instagram('Sunset vibes 🌅')
article = LinkedIn('Why OOP matters in 2024')

# The polymorphic calls - same function, different outputs
start(tweet) # 🐦 Tweet: 'Just learned Python polymorphism!' (280 chars max)
start(photo) # 📸 Instagram Post: 'Sunset vibes 🌅' + ✨ filters
start(article) # 💼 LinkedIn Article: 'Why OOP matters in 2024' (Professional Mode)
```

Існує також вид поліморфізму, який називається **поліморфізм на основі успадкування**.

У поліморфізмі на основі успадкування батьківський клас оголошує метод, а кілька дочірніх класів заміщають цей метод по-своєму. Ви можете викликати той самий метод на будь-якому дочірньому об’єкті, і він поводитиметься по-різному залежно від того, до якого дочірнього класу він належить.

Ось приклад:

```py
class Animal:
   def speak(self):
       return 'Some generic sound'

class Cat(Animal):
   def speak(self):
       return 'A cat meow'

class Dog(Animal):
   def speak(self):
       return 'A dog barks woof woof'

class Monkey(Animal):
   def speak(self):
       return 'A monkey ooh ooh aah aah ooh ooh aah aah'
  
print(Cat().speak()) # A cat meow
print(Dog().speak()) # A dog barks woof woof
print(Monkey().speak()) # A monkey ooh ooh aah aah ooh ooh aah aah
print(Animal().speak()) # Some generic sound
```

Ви бачите, що кожен дочірній клас батьківського класу `Animal` заміщує метод `speak()`, щоб надати власну реалізацію. Тож коли ви викликаєте метод `speak()` на екземплярі кожного підкласу, він повертає конкретний звук, пов’язаний із цією твариною.

Ви також можете піти далі і викликати метод у списку, а потім пройтися циклом по списку, щоб показати, що метод `speak()` повертає для кожного:

```py
animals = [Cat(), Dog(), Monkey()]

for animal in animals:
   print(animal.speak())

# Output:
# A cat meow
# A dog barks woof woof
# A monkey ooh ooh aah aah ooh ooh aah aah
```
