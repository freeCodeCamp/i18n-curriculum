# --description--

このチャレンジでは、じゃんけんゲームをプレイするプログラムを作成します。ランダムに手を選ぶプログラムは通常50％の確率で勝ちます。このチャレンジをクリアするには、4つの異なるボットと対戦し、それぞれの対戦で少なくとも60％の勝率を達成する必要があります。

機械学習カリキュラムのインタラクティブな指導部分はまだ開発中です。今のところ、このチャレンジをクリアするには他のリソースを使って学習してください。

# --instructions--

`RPS.py` ファイルには `player` という関数が用意されています。この関数は相手の最後の手を表す文字列（"R"、"P"、または "S"）を引数として受け取ります。関数は次に出す手を表す文字列（"R"、"P"、または "S"）を返す必要があります。

対戦の最初のゲームでは前の手がないため、プレイヤー関数には空文字列が引数として渡されます。

`RPS.py` ファイルには更新が必要な例の関数が示されています。この例の関数は2つの引数（`player(prev_play, opponent_history = [])`）で定義されていますが、2番目の引数は呼び出されることがないため完全に省略可能です。例の関数に2番目の引数（`opponent_history = []`）があるのは、`player` 関数の連続呼び出し間で状態を保存する唯一の方法だからです。相手の履歴を追跡したい場合のみ`opponent_history` 引数が必要です。

*ヒント：4人の対戦相手すべてに勝つには、相手の手に応じて変わる複数の戦略を持つ必要があるかもしれません。*

## 開発

`RPS_game.py` は変更しないでください。すべてのコードは `RPS.py` に書いてください。開発中は `main.py` を使ってコードをテストできます。

`main.py` は `RPS_game.py` からゲーム関数とボットをインポートしています。

コードをテストするには、`play` 関数でゲームをプレイしてください。`play` 関数は4つの引数を取ります：

- 対戦する2人のプレイヤー（実際には関数です）
- 対戦するゲーム数
- 各ゲームのログを表示するオプションの引数。これを `True` に設定するとメッセージが表示されます。

```py
play(player1, player2, num_games[, verbose])
```

例えば、`player` と `quincy` が1000ゲーム対戦し、各ゲームの結果を見たい場合は次のように関数を呼び出します：

```py
play(player, quincy, 1000, verbose=True)
```
