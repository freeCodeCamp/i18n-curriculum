# --description--

スタックとキューはコンピューターサイエンスでよく使われるデータ構造です。

これらは線形のデータ構造で、要素の追加と削除に特定のルールが適用されます。

## Stacks

まずは**スタック**から始めましょう。

**スタック**は後入れ先出し（LIFO）のデータ構造です。

これは、スタックに最後に追加された要素が最初に取り出されることを意味します。

スタックには「トップ」と「ボトム」という2つの端があります。

要素はスタックのトップから追加および削除されます。

スタックは皿の山のようなもので、皿は山の上にしか置けず、山の上からしか取れないと考えてください。

<img src="https://cdn.freecodecamp.org/curriculum/lecture-transcripts/how-do-stacks-and-queues-work-1.png" alt="stack data structure visualization">

この要素の追加と削除の操作には特別な名前があります。

スタックに要素を追加する操作は「プッシュ」と呼ばれます。スタックのトップに要素を追加するときに「プッシュする」と言います。

<img src="https://cdn.freecodecamp.org/curriculum/lecture-transcripts/how-do-stacks-and-queues-work-2.png" alt="stack push operation visualization">

スタックから要素を削除する操作は「ポップ」と呼ばれます。スタックのトップから要素を取り除くときに「ポップする」と言います。

<img src="https://cdn.freecodecamp.org/curriculum/lecture-transcripts/how-do-stacks-and-queues-work-3.png" alt="stack pop operation visualization">

スタックのボトムでは実際に操作は行いませんが、参照として残しています。

プッシュとポップの操作の時間計算量は通常`O(1)`、定数時間計算量です。

要素をスタックにプッシュすると、その要素は単純にトップに追加されます。

要素をスタックからポップすると、トップの要素が削除されます。

したがって、これらの操作にかかる時間はスタックのサイズに関係なく一定です。

プッシュとポップの操作の空間計算量は通常定数`O(1)`です。これは、これらの操作に必要なメモリ量がスタックのサイズに関係なく一定であることを意味します。

## Queues

スタックについて理解できたので、次は**キュー**について学びましょう。

キューは先入れ先出し（FIFO）の線形データ構造です。これは、キューに最初に追加された要素が最初に取り出されることを意味します。

キューには「フロント」と「バック」という2つの端があります。

<img src="https://cdn.freecodecamp.org/curriculum/lecture-transcripts/how-do-stacks-and-queues-work-4.webp" alt="queue data structure visualization">

要素はキューのバックに追加され、フロントから削除されます。

キューはスーパーマーケットのレジ待ちの列のようなもので、列の最初の人が最初にレジに行き、新しい人は列の最後に並びます。

要素の追加と削除の操作にはキューの文脈で特別な名前があります。

キューのバックに要素を追加する操作は「エンキュー」と呼ばれます。

エンキュー操作では、新しい要素がキューの最後に追加され、列の最後の位置になります。

<img src="https://cdn.freecodecamp.org/curriculum/lecture-transcripts/how-do-stacks-and-queues-work-5.png" alt="queue enqueue operation visualization">

キューのフロントから要素を削除する操作は「デキュー」と呼ばれます。

デキュー操作では、キューのフロントの要素が削除され、次の要素が新しいフロントになります。

<img src="https://cdn.freecodecamp.org/curriculum/lecture-transcripts/how-do-stacks-and-queues-work-6.png" alt="queue dequeue operation visualization">

エンキューとデキューの操作の時間計算量は`O(1)`、定数時間です。これらの操作にかかる時間はキューのサイズに関係なく一定です。

エンキューとデキューの操作の空間計算量は通常定数`O(1)`です。これは、これらの操作に必要なメモリ量がキューのサイズに関係なく一定であることを意味します。

スタックとキューは、要素を整理・管理するためにコンピューターサイエンスで使われるデータ構造です。これらを理解することは、さまざまなプログラミングアプリケーションで効率的なアルゴリズムを作るために不可欠です。
