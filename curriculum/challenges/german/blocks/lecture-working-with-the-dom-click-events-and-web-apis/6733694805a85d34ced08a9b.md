# --interactive--

Beim Erstellen Ihrer Programme möchten Sie oft, dass etwas nach einer Verzögerung passiert oder in regelmäßigen Abständen wiederholt wird. Hier kommen die `setTimeout()` und `setInterval()` Methoden ins Spiel. Beide Methoden akzeptieren zwei Parameter: eine Funktion und eine Verzögerung.

Schauen wir uns jede der Methoden genauer an, damit Sie sie in Ihren Webprojekten verwenden können.

`setTimeout()` ermöglicht es Ihnen, eine Aktion für eine bestimmte Zeit zu verzögern. Hier ist die grundlegende Syntax der `setTimeout()` Methode:

```js
setTimeout(functionToRun, delay);
```

`functionToRun` ist der Code, den Sie nach einer bestimmten Verzögerung ausführen möchten, und `delay` ist die Zeit (in Millisekunden), die gewartet wird, bevor die Funktion ausgeführt wird. Hier ein Beispiel:

:::interactive_editor

```js
setTimeout(function () {
 console.log("This runs after 3 seconds");
}, 3000);
```

:::

Sie können auch eine Pfeilfunktion für das Callback verwenden, so wie hier:

:::interactive_editor

```js
setTimeout(() => {
 console.log("This runs after 3 seconds"); 
}, 3000);
```

:::

`setInterval()` wiederholt eine Funktion in einem festgelegten Intervall. Es ist wie ein Wecker, der alle paar Sekunden klingelt. `setInterval()` ist perfekt, wenn Sie möchten, dass etwas immer wieder passiert, wie das Aktualisieren von Daten oder das Erstellen einer Animation, die sich kontinuierlich bewegt.

Genau wie `setTimeout()` nimmt es eine Funktion und eine Verzögerung:

```js
setInterval(functionToRun, delay);
```

`functionToRun` ist der Code, den Sie wiederholt ausführen möchten, während `delay` die Zeit in Millisekunden zwischen jeder Ausführung dieser Funktion ist. Hier ein Beispiel:

:::interactive_editor

```js
setInterval(() => {
 console.log("This runs every 2 seconds");
}, 2000);

// Result:
// This runs every 2 seconds
// This runs every 2 seconds
// This runs every 2 seconds
// ...
```

:::

Da `setInterval()` die angegebene Funktion im festgelegten Intervall immer wieder ausführt, möchten Sie sie vielleicht stoppen. Dazu müssen Sie die `clearInterval()` Methode verwenden.

Um die Arbeit zu erledigen, nimmt `clearInterval()` die ID des `setInterval()`, den Sie stoppen möchten. Das könnte eine Variable sein, der Sie das Intervall zuweisen. Eine Möglichkeit, das Intervall zu stoppen, ist innerhalb eines `setTimeout()`, da dies das Intervall nach einer bestimmten Zeit stoppt:

:::interactive_editor

```js
const intervalID = setInterval(() => {
 console.log("This will stop after 5 seconds");
}, 1000);

setTimeout(() => {
 clearInterval(intervalID);
}, 5000);
```

:::

Sie können auch einen Timeout mit der `clearTimeout()` Methode stoppen:

:::interactive_editor

```js
let timeoutID = setTimeout(() => {
 console.log("This will not run");
}, 5000);

clearTimeout(timeoutID);
```

:::

Mit diesem Code wird der `setTimeout()` überhaupt nicht ausgeführt. Eine elegantere Möglichkeit ist es daher, dies über das DOM zu machen, indem Sie einen Klick-Ereignis-Listener zu einem Button hinzufügen, um den Timeout zu stoppen:

:::interactive_editor

```html
<h1>Cancel Timeout Example</h1>
<button id="cancelButton">Cancel Timeout</button>
<script src="index.js"></script>
```

```js
let timeoutID = setTimeout(() => {
  console.log("This will run if not canceled");
}, 5000);

document
.querySelector("#cancelButton")
.addEventListener("click", () => {
  clearTimeout(timeoutID);
  console.log("Timeout canceled!");
});
```

:::
