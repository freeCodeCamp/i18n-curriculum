# --description--

在本课中，你将学习如何使用基本的原生类型以及变量的类型注解如何工作。

对于原语数据类型 `string`、`null`、`undefined`、`number`、`boolean` 和 `bigint`，TypeScript 提供了相应的类型关键字。 

对于本课，我们将重点关注常用的原语，而不详细介绍 `bigint`。`bigint` 数据类型是 JavaScript 中的一种数字数据类型，用于表现比标准整数类型更大的数字。 

让我们从 `string` 开始，来看一下每个原生类型。以下是在 TypeScript 中为字串指定类型的方法：

```ts
let firstName: string = "Angie";
```

与原生 JavaScript 不同，变量 `firstName` 期望的是 `string` 类型。所以如果你尝试用数字值重新赋值它，那么你将得到如下的误差：

```ts
let firstName: string = "Angie";
firstName = 9 // Type 'number' is not assignable to type 'string'.
```

下一个我们将要看的 `number` 类型。这里有一个例子：

```ts
let age: number = 16;
```

你可以为 `number` 类型赋值整数和浮点数。但如果你尝试重新赋值一个 `bigint` 类型的值，那么你将得到如下的误差：

```ts
let age: number = 16;

// Type 'bigint' is not assignable to type 'number'.
age = 123n;
```

下一个我们将要看的类型是 `boolean` 类型。以下是一些示例：

```ts
let isLoggedIn: boolean = true;
let isAdmin: boolean = false;
```

最后两种类型是 `null` 和 `undefined` 类型。 

正如你从之前的 JavaScript 课程中回忆的，`null` 表现为有意缺少值。`undefined` 意味着一个变量已被声明但尚未赋值。

当你学习 `function` 类型和 `union` 类型时，你将看到如何使用这些类型的示例。

既然你已经了解了类型注解如何与变量一起使用，那么问题就来了，你是否总是需要显式地为你的变量指定类型。 

如果你有像这样简单的 `type` 注解，那么你可能不需要使用显式的 `type`。

```ts
let firstName: string = "Angie";
```

原因是 TypeScript 会尝试根据其值自动推导类型。所以如果你这样写，TypeScript 会理解它是一个 `string`。

```js
let firstName = "Angie";
```

当 TypeScript 无法清晰推导 `variable` 的 `type` 或者你想在诸如 `function` 的 `parameter` 或复杂 `object` 等情况下迫使特定 `type` 时，你应该显式地为你的 `variable` 指定 `type`。
