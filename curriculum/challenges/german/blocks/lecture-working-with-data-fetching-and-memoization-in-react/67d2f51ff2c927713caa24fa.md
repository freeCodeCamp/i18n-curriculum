# --description--

In der letzten Lektion haben Sie etwas über Memoisierung und die Funktionsweise des `useMemo`-Hooks gelernt.

In dieser Lektion erfahren Sie, wie der `useCallback`-Hook und `React.memo` funktionieren.

In der letzten Lektion haben wir auch erwähnt, dass `useCallback` dazu dient, Funktionsreferenzen zu memoisieren.

Für `React.memo` können Sie eine Komponente memoisieren, um unnötige Re-Renderings zu verhindern, wenn sich ihre Prop nicht geändert hat.

Hier ist die grundlegende Syntax des `useCallback`-Hooks:

```js
const handleClick = useCallback(() => {
  // code goes here
}, [dependency]);
```

Und hier ist die grundlegende Syntax von `React.memo`:

```jsx
const MemoizedComponent = React.memo(({ prop }) => {
  return (
    <>
      {/* Presentation */}
    </>
  )
});
```

Schauen wir uns ein Beispiel für den `useCallback`-Hook an:

```jsx
import { useState, useEffect } from "react";

function Counter() {
  const [count, setCount] = useState(0);

  const handleClick = () => {
    setCount((prevCount) => prevCount + 1);
  };

  useEffect(() => {
    console.log("useEffect runs");
  }, [handleClick]);

  return (
    <div>
      <p>Count: {count}</p>
      <button onClick={handleClick}>Increment</button>
    </div>
  );
}

export default Counter;
```

In der Komponente läuft der Effekt jedes Mal, wenn sich `handleClick` ändert, weil die `handleClick`-Funktion bei jedem Rendern neu erstellt wird.

Um das zu beheben, müssen Sie React mitteilen, dass die `handleClick`-Funktion über die Render hinweg als dieselbe behandelt werden soll, indem Sie sie mit dem `useCallback`-Hook memoisieren, damit sie nicht neu erstellt wird:

```jsx
import { useState, useEffect, useCallback } from "react";

function Counter() {
  const [count, setCount] = useState(0);

  // Memoize the handleClick function with useCallback
  const handleClick = useCallback(() => {
    setCount((prevCount) => prevCount + 1);
  }, []);

  useEffect(() => {
    console.log("useEffect runs");
  }, [handleClick]);

  return (
    <div>
      <p>Count: {count}</p>
      <button onClick={handleClick}>Increment</button>
    </div>
  );
}

export default Counter;
```

Jetzt wird die `handleClick`-Funktion nicht mehr bei jedem Render neu erstellt.

Um Ihnen zu zeigen, wie die `React.memo` (oder `memo`) höherwertige Funktion und der `useCallback`-Hook zusammenarbeiten, hier eine `Counter`-Komponente mit einer `handleClick`-Funktion, die `useCallback` benötigt, aber derzeit nicht verwendet:

```jsx
import { useState, useEffect, useCallback } from "react";
import CounterChild from "./CounterChild";

function Counter() {
  const [count, setCount] = useState(0);
  const [timer, setTimer] = useState(new Date().toLocaleTimeString());

  const handleClick = () => {
    setCount(count + 1);
  };

  useEffect(() => {
    const interval = setInterval(() => {
      setTimer(new Date().toLocaleTimeString());
    }, 1000);

    return () => clearInterval(interval);
  }, []);

  return (
    <div>
      <h1>Time: {timer}</h1>
      <p>Count: {count}</p>
      <button onClick={handleClick}>Increment</button>
      <CounterChild onClick={handleClick} />
    </div>
  );
}

export default Counter;
```

Diese Funktion hat auch einen Timer im State, der jede Sekunde aktualisiert wird. Das führt dazu, dass die Komponente bei jeder Änderung von `timer` neu gerendert wird, wodurch die `handleClick`-Funktion bei jedem Render neu erstellt wird.

Deshalb muss die `handleClick` mit `useCallback` memoisiert werden.

Hier ist die `CounterChild`-Komponente:

```jsx
const CounterChild = ({ onClick }) => {
  console.log("CounterChild component rendered");
  return <button onClick={onClick}>Increment from Child</button>;
};

export default CounterChild;
```

Diese `CounterChild`-Komponente nimmt eine `onClick`-Prop entgegen, die Ihnen die Möglichkeit gibt, den Zähler auch von dort aus zu erhöhen.

Da die `CounterChild`-Komponente ein Kind der `Counter`-Komponente ist, wird sie ebenfalls jedes Mal gerendert, wenn die `Counter` aufgrund des sich ändernden Timers neu gerendert wird. Deshalb muss auch die `CounterChild` memoisiert werden.

Ohne Memoisierung wird die `CounterChild`-Komponente ebenfalls neu gerendert, weil die Komponente aufgrund der Aktualisierung des Timers jede Sekunde neu gerendert wird.

Um dies zu verhindern, müssen Sie die `CounterChild`-Komponente mit `React.memo` memoisieren:

```jsx
import React from "react";

const CounterChild = React.memo(({ onClick }) => {
  console.log("CounterChild component rendered");
  return <button onClick={onClick}>Increment from Child</button>;
});

export default CounterChild;
```

Die Dinge funktionieren noch nicht optimal, selbst nachdem die `CounterChild` mit `React.memo` memoisiert wurde.

Das liegt daran, dass die `handleClick`-Funktion bei jedem Render neu erstellt wird, also muss sie ebenfalls mit `useCallback` memoisiert werden, um React mitzuteilen, dass die Funktion über die Render hinweg gleich bleiben soll:

```js
const handleClick = useCallback(() => {
  setCount((prevCount) => prevCount + 1);
}, [count]);
```

Jetzt rendert die Komponente nur noch, wenn sich der `count`-State ändert.
