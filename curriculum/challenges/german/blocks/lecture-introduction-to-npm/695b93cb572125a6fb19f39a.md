# --description--

In einer früheren Lektion haben Sie gelernt, was Abhängigkeiten sind. Hier erfahren Sie mehr über sie und worauf Sie achten sollten, wenn Sie ein externes Paket für Ihr Projekt auswählen.

Eine Abhängigkeit ist ein weiterer Codeabschnitt, den Ihr Projekt benötigt, um zu funktionieren. Dabei handelt es sich meist um externe Pakete, die andere Entwickler erstellt und im npm-Registry veröffentlicht haben.

Die Verwendung von Abhängigkeiten spart Ihnen viel Zeit. Statt alles selbst zu entwickeln, können Sie ein Paket installieren, das bereits ein häufiges Problem löst – zum Beispiel das Erstellen eines Webservers, das Verwalten von Routing, das Abrufen von Daten, die Handhabung der Authentifizierung und mehr.

In Ihrer `package.json`-Datei sind alle Pakete, von denen Ihr Projekt abhängt, unter dem `dependencies`-Schlüssel aufgelistet:

```json
{
  "name": "package-dependencies",
  "version": "1.0.0",
  "description": "a demo on how package dependencies work",
  "main": "index.js",
  "scripts": {
    "test": "echo \"Error: no test specified\" && exit 1"
  },
  "keywords": [
    "Node",
    "NPM"
  ],
  "author": "Kolade Chris",
  "license": "ISC",
  "dependencies": {
    "chalk": "^5.6.2",
    "lodash": "^4.17.21"
  }
}
```

Im obigen Beispiel der `package.json`-Datei sind `chalk` und `lodash` als Abhängigkeiten des Projekts aufgeführt. Das bedeutet, dass das Projekt auf diese Pakete angewiesen ist, um zu funktionieren. Wenn sie nicht installiert sind – und nicht in der `package.json`-Datei aufgeführt werden – funktionieren alle Funktionen, die sie verwenden, nicht mehr.

Es gibt auch eine weitere Kategorie von Abhängigkeiten, die `devDependencies` genannt wird. Diese Pakete benötigen Sie nur während der Entwicklung oder beim Testen Ihres Projekts, nicht aber im produktiven Betrieb. Häufige Beispiele sind:

* Nodemon, um das Projekt bei Änderungen automatisch neu zu starten
    
* Jest oder Mocha für Tests
    
* ESLint für Code-Linting
    

Wie zu erwarten, erscheinen `devDependencies` unter dem Abschnitt `devDependencies` in der `package.json`-Datei:

```json
{
  "name": "package-dependencies",
  "version": "1.0.0",
  "description": "a demo on how package dependencies work",
  "main": "index.js",
  "scripts": {
    "test": "echo \"Error: no test specified\" && exit 1"
  },
  "keywords": [
    "Node",
    "NPM"
  ],
  "author": "Kolade Chris",
  "license": "ISC",
  "dependencies": {
    "chalk": "^5.6.2",
    "lodash": "^4.17.21"
  },
  "devDependencies": {
    "jest": "^30.2.0",
    "nodemon": "^3.1.10"
  }
}
```

Bei der Auswahl der Pakete für Ihr Projekt gibt es einige wichtige Punkte zu beachten.

Die Sicherheit sollte immer an erster Stelle stehen. Schon ein unsicheres Paket kann Probleme für Ihr gesamtes Projekt verursachen. Außerdem sollten Sie die Leistung berücksichtigen, denn ein großes oder schlecht optimiertes Paket kann Ihr Projekt verlangsamen.

Es hilft auch, Pakete auszuwählen, die beliebt und gut unterstützt sind. Beliebte Pakete haben meist aktive Communities, gute Dokumentation und regelmäßige Updates. Eine einfache Möglichkeit, die Beliebtheit zu prüfen, ist die Anzahl der wöchentlichen Downloads auf npm. Zum Beispiel wird das `chalk`-Paket jede Woche über 300.000 Mal heruntergeladen:

<img src="https://cdn.freecodecamp.org/curriculum/lecture-transcripts/what-are-package-dependencies-and-what-are-some-considerations-for-choosing-an-external-package-1.webp" alt="Screenshot von npm mit Statistiken zum chalk-Paket">

Auch die Wartung ist wichtig. Gut gewartete Pakete werden regelmäßig aktualisiert und funktionieren besser mit neuen Versionen anderer Software.

Schließlich sollten Sie die Dokumentation nicht vernachlässigen. Klare und ausführliche Dokumentationen erleichtern das Verständnis der Nutzung eines Pakets und sparen Ihnen viel Zeit bei der Integration in Ihr Projekt.
