# --interactive--

`audio` と `video` 要素は、HTMLドキュメントに音声や動画コンテンツを追加することを可能にします。`audio` 要素は、mp3、wav、oggなどの一般的な音声フォーマットに対応しています。
`video` 要素は、mp4、ogg、webmフォーマットに対応しています。

ウェブページに音声コンテンツを含めるには、`audio` 要素を使い、`src` 属性で音声ファイルの場所を指定できます。

プレビューウィンドウを見ると、ページ上には何も表示されません。プレビューを確認するには、インタラクティブエディターを有効にしてください。

:::interactive_editor

```html
<audio src="https://cdn.freecodecamp.org/curriculum/js-music-player/cruising-for-a-musing.mp3"></audio>
```

:::

ページ上にオーディオプレーヤーを表示したい場合は、`audio` 要素に `controls` 属性を追加できます。

プレビューウィンドウで再生ボタンを押すと、Quincy Larsonの曲の一つが再生されます。別の曲を聴きたい場合は、`src` の値を `"https://cdn.freecodecamp.org/curriculum/js-music-player/never-not-favored.mp3"` に変更してください。プレビューを見るには、インタラクティブエディターを有効にしてください。

:::interactive_editor

```html
<audio src="https://cdn.freecodecamp.org/curriculum/js-music-player/cruising-for-a-musing.mp3" controls></audio>
```

:::

`controls` 属性は、ユーザーが音声の再生を一時停止したり再開したりできるようにします。`controls` 属性はブール値の属性で、要素に追加すると組み込みの再生コントロールが有効になります。省略すると、コントロールは表示されません。

注意：Safariなどの一部のブラウザでは、`controls` 属性があってもデフォルトでは音量コントロールが表示されない場合があります。

`src` と `controls` 属性以外にも、`audio` 要素の機能を強化する属性がいくつかあります。`loop` 属性はブール値の属性で、音声を連続再生させます。

以下は、Quincy Larsonの曲「Can't stay down」を再生するために`loop` 属性を使った例です。ループ再生を確認するには、インタラクティブエディターを有効にして、再生ヘッドを曲の終わり近くまで移動させてください。曲が終わると再び最初から再生されます。

:::interactive_editor

```html
<audio
  src="https://cdn.freecodecamp.org/curriculum/js-music-player/can't-stay-down.mp3"
  loop
  controls
></audio>
```

:::

もう一つ使える属性は`muted` 属性です。`audio` 要素にこのブール値属性があると、音声はミュート状態で再生を開始します。以下は`muted` 属性を使った例です。

音楽を聴くには、インタラクティブエディターを有効にしてオーディオプレーヤーの音量アイコンをクリックしてください。

:::interactive_editor

```html
<audio
  src="https://cdn.freecodecamp.org/curriculum/js-music-player/can't-stay-down.mp3"
  loop
  controls
  muted
></audio>
```

:::

音声ファイルの種類については、ブラウザごとに対応しているタイプが異なります。これに対応するために、`source` 要素の中に複数の`audio` 要素を使うことができ、ブラウザは理解できる最初のソースを選択します。以下は`source` 要素に複数の`audio` 要素を使った例です：

```html
<audio controls>
  <source src="audio.ogg" type="audio/ogg" />
  <source src="audio.wav" type="audio/wav" />
  <source src="audio.mp3" type="audio/mpeg" />
</audio>
```

ブラウザは最初にoggタイプから再生を試み、再生できなければリストの次のタイプに移ります。

これまで学んだすべての属性は`video` 要素でもサポートされています。以下は`video` 要素に `loop`、`controls`、`muted` 属性を使った例です。

動画を自動再生させるには、開始タグの`autoplay` に `video` 属性を追加してください。例を操作するにはインタラクティブエディターを有効にする必要があります。

**注意**：ここでは動画を小さくしてプレビューウィンドウに収まるように、`width` 属性を使っています。`width` 属性については今後のレッスンで詳しく学びます。

:::interactive_editor

```html
<video
  src="https://archive.org/download/BigBuckBunny_124/Content/big_buck_bunny_720p_surround.mp4"
  loop
  controls
  muted
  width="400"
></video>
```

:::

`src`、つまりソース属性には、archive.orgの「Big Buck Bunny」という動画を使っています。動画がダウンロード中に画像を表示したい場合は、`poster` 属性を使えます。この属性は`audio` 要素にはなく、`video` 要素に特有のものです。以下はpeach.blender.org提供のコンテンツで`poster` 属性を使った例です。

```html
<video
  src="https://archive.org/download/BigBuckBunny_124/Content/big_buck_bunny_720p_surround.mp4"
  loop
  controls
  muted
  poster="https://peach.blender.org/wp-content/uploads/title_anouncement.jpg?x11217"
  width="400"
></video>
```

`source` 要素を`video` 要素の中に使うこともできます。`audio` 要素と同様に、複数のフォーマットで同じ動画を提供し、ブラウザが再生可能な最初のものを選択します。

```html
<video
  controls
  width="400"
  poster="https://peach.blender.org/wp-content/uploads/title_anouncement.jpg?x11217"
>
  <source
    src="https://archive.org/download/BigBuckBunny_124/Content/big_buck_bunny_720p_surround.mp4"
    type="video/mp4"
  />
  <source
    src="https://archive.org/download/BigBuckBunny_124/Content/big_buck_bunny_720p_surround.webm"
    type="video/webm"
  />
  Your browser does not support the video tag.
</video>
```
