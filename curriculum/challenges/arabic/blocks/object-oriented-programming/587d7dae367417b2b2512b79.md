# --description--

عملت دوال البناء `Bird` و `Dog` من التحدٍّ السابق بشكل جيد. مع ذلك، لاحظ أن جميع `Birds` التي تُنشأ باستخدام دالة البناء `Bird` تُسمى تلقائيًا Albert، ولونها أزرق، ولها قدمان. ماذا لو أردت طيورًا بقيم مختلفة للاسم واللون؟ من الممكن تعديل خصائص كل طائر يدويًا، لكن ذلك سيكون عملاً كثيرًا:

```js
let swan = new Bird();
swan.name = "Carlos";
swan.color = "white";
```

افترض أنك تكتب برنامجًا لتتبع مئات أو حتى آلاف الطيور المختلفة في حظيرة. سيستغرق إنشاء جميع الطيور ثم تعديل خصائصها إلى قيم مختلفة لكل واحدة وقتًا طويلًا. لتسهيل إنشاء كائنات `Bird` مختلفة، يمكنك تصميم دالة البناء Bird لتستقبل معلمات:

```js
function Bird(name, color) {
  this.name = name;
  this.color = color;
  this.numLegs = 2;
}
```

ثم مرر القيم كمعلمات لتعريف كل طائر فريد في دالة البناء `Bird`: `let cardinal = new Bird("Bruce", "red");` هذا يعطي نسخة جديدة من `Bird` مع خصائص `name` و `color` مضبوطة على `Bruce` و `red` على التوالي. خاصية `numLegs` لا تزال مضبوطة على 2. الطائر `cardinal` له هذه الخصائص:

```js
cardinal.name
cardinal.color
cardinal.numLegs
```

أصبحت دالة البناء أكثر مرونة. أصبح من الممكن الآن تحديد الخصائص لكل `Bird` عند إنشائه، وهذه إحدى الطرق التي تجعل دوال البناء في JavaScript مفيدة جدًا. فهي تجمع الكائنات معًا بناءً على الخصائص والسلوكيات المشتركة وتحدد نموذجًا يُؤتمت عملية إنشائها.

# --instructions--

أنشئ دالة بناء أخرى لـ `Dog`. هذه المرة، جهزها لتأخذ المعلمات `name` و `color`، واجعل خاصية `numLegs` ثابتة عند 4. ثم أنشئ كائن `Dog` جديد محفوظ في متغير `terrier`. مرر له سلسلتين نصيتين كمعلمات لخصائص `name` و `color`.

# --hints--

يجب أن تستقبل دالة البناء `Dog` معلمة لـ `name`.

```js
assert(new Dog('Clifford').name === 'Clifford');
```

يجب أن تستقبل دالة البناء `Dog` معلمة لـ `color`.

```js
assert(new Dog('Clifford', 'yellow').color === 'yellow');
```

يجب أن تحتوي دالة البناء `Dog` على خاصية `numLegs` مضبوطة على 4.

```js
assert(new Dog('Clifford').numLegs === 4);
```

يجب إنشاء `terrier` باستخدام دالة البناء `Dog`.

```js
assert(terrier instanceof Dog);
```
