# --description--

JavaScriptエンジンは、あなたのコードを読み取り、理解し、実行する能力を持っています。これは、コードを受け取り、コンピューターが理解してそれに従って動作できる命令に変換するコンバーターのように機能します。

最もよく知られているJavaScriptエンジンの一つは、Googleが開発し、ChromeやNode.jsで使われているV8です。JavaScriptエンジンは、いくつかのステップで動作します。まず、コードをパースし、JavaScriptコードに間違いがないか行ごとに読み取ります。次に、このコードをバイトコードに変換します。バイトコードは、コンピューターが理解しやすく実行しやすい、より単純な中間形式のコードです。最後に、このバイトコードを実行してプログラムの命令を実行します。以下はJavaScriptコードの例です。

```js
const greeting = "Hello, World!";
console.log(greeting);
```

このコードを実行すると、JavaScriptエンジンは最初に構文エラーがないかチェックするためにパースします。パースとは、エンジンがコードを読み取り、理解できる構造に分解しながら間違いを確認することです。

次に、コードを中間形式（エンジンによってはバイトコードやマシンコード）にコンパイルします。コンパイルとは、人間が読めるコードをコンピューターがより高速に実行できる効率的な形式に変換するプロセスです。

最後に、エンジンはコードを実行し、コンソールに`Hello, World!`を出力します。

次にJavaScriptランタイムについて説明します。JavaScriptランタイムは、JavaScriptコードが実行される環境のことです。これは、コードを処理し実行するJavaScriptエンジン（ChromeのV8やFirefoxのSpiderMonkeyなど）に加え、環境が提供する追加の機能（ウェブブラウザやNode.jsなど、今後のレッスンで詳しく学びます）を含みます。

JavaScriptのコア言語は変数やループ、関数などを扱いますが、ランタイムは言語自体の外側とやり取りできる追加のツールを提供します。例えば、ウェブページのDOMやネットワークリクエストを行うFetch APIなどです。

つまり、ランタイムはJavaScriptが基本的なプログラミングタスク以上のことをできるようにし、ウェブページとのやり取りや時間に基づく処理など、言語自体を超えた機能を提供しています。

JavaScriptを書くのにエンジンやランタイムのすべての詳細を知る必要はありませんが、基本を理解しておくと、より効率的なコードを書いたり、問題をデバッグしたりするのに役立ちます。
