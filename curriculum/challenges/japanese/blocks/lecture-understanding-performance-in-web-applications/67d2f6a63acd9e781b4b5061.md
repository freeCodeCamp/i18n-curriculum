# --description--

なぜあるウェブサイトは素早くレスポンシブに感じられ、他のサイトは遅く感じるのでしょうか？その答えは、ページの読み込みやレンダリングに影響を与える主要なパフォーマンスの概念にあります。主要なウェブパフォーマンスの概念を理解することは、速くてスムーズ、かつユーザーフレンドリーなウェブサイトを作るために不可欠です。

まずはソースオーダー、クリティカルレンダリングパス、レイテンシーなどを分解して見ていきましょう。

まずはソースオーダーから見ていきます。

ソースオーダーとは、HTML要素がドキュメント内でどのように構造化されているかを指します。これが何が最初に読み込まれるかを決定し、パフォーマンスやアクセシビリティに大きな影響を与えます。

いくつかのベストプラクティスは以下の通りです：

- 見出し、ナビゲーション、メインテキストなどの重要なコンテンツをHTML構造の上位に配置する。

- アナリティクスやサードパーティのウィジェットなど、必須でないスクリプトは遅延させてレンダリングを妨げないようにする。

- プログレッシブエンハンスメントを使い、スタイルやスクリプトが読み込まれる前でもコアの体験が機能するようにする。プログレッシブエンハンスメントは、まずHTMLでページを動作させることを基本にウェブサイトやアプリケーションを構築する方法です。

以下は、先ほどのベストプラクティスを使った良いソースオーダーの例です。

```html
<!-- Good source order: Essential content first -->
<h1>Welcome to FastSite!</h1>
<p>Critical information loads first.</p>
<script src="slow-script.js" defer></script>
```

ソースオーダーを最適化することで、ユーザーが重要なコンテンツをできるだけ早く見ることができます。

次にクリティカルレンダリングパスを見ていきましょう。

クリティカルレンダリングパスとは、ブラウザがコードを画面上のピクセルに変換するためにたどる一連のステップです。

以下は後で詳しく説明する主要なステップです：

1. HTMLの解析：DOM（ドキュメントオブジェクトモデル）を構築する

2. CSSの解析：CSSOM（CSSオブジェクトモデル）を構築する

3. JavaScriptの実行：DOMとCSSOMを変更できる

4. レンダーツリーの構築：DOMとCSSOMを組み合わせる

5. レイアウトとペイント：要素のサイズを決定しピクセルを描画する

6. 最適化：

    - レンダーブロックするリソース（大きなCSSファイルや未使用のJSなど）を最小化する。

    - スクリプトにasyncやdefer属性を使う。

    - まず必須のスタイルだけを読み込み、重要でないCSSは遅延させる。

全体として、クリティカルレンダリングパスが短いほど、体感パフォーマンスは速くなります。これについては後ほど詳しく説明します。

最後にレイテンシーを見ていきましょう。

レイテンシーとは、ブラウザとサーバー間でリクエストが往復するのにかかる時間です。つまり、レイテンシーが高いとページが遅くなります。

レイテンシーを減らす方法には以下があります：

- CDN（コンテンツデリバリーネットワーク）を使い、ファイルをより近い場所から配信する。

- Gzipなどの圧縮を有効にしてファイルサイズを減らす。

- 画像を最適化し、遅延読み込み（レイジーローディング）を使う — これも後で説明します。

レイテンシーを減らすことで、操作が瞬時に感じられるようになります。

まとめると、ソースオーダーを最適化し、クリティカルパスを短縮し、レイテンシーを減らすことで、ウェブサイトを速く感じさせることができます。
