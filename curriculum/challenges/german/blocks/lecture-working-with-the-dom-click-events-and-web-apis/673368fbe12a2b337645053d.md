# --description--

Das `Event`-Objekt ist eine Nutzlast, die ausgelöst wird, wenn ein Benutzer auf irgendeine Weise mit Ihrer Webseite interagiert. Diese Interaktionen können alles Mögliche sein, vom Klicken auf einen Button oder dem Fokussieren einer Eingabe bis hin zum Schütteln des Mobilgeräts.

Wie alle JavaScript-Objekte hat das `Event`-Objekt eine Reihe von Eigenschaften, die hilfreich sein können. Die verfügbaren Eigenschaften hängen vom Ereignis ab, das diese Nutzlast ausgelöst hat.

Alle `Event`-Objekte haben die `type`-Eigenschaft. Diese Eigenschaft zeigt den Typ des Ereignisses an, das die Nutzlast ausgelöst hat, wie zum Beispiel `"keydown"` oder `"click"`. Diese Werte entsprechen den gleichen Werten, die Sie an `addEventListener()` übergeben könnten, wo Sie das `Event`-Objekt erfassen und nutzen können.

`Event`-Objekte haben immer die `target`-Eigenschaft. Die `target`-Eigenschaft ist eine Referenz auf das Objekt, das das Ereignis ausgelöst hat. Meistens ist dies eine Art `HTMLElement`-Objekt oder die `Document`- oder `Window`-Objekte. Es kann aber auch etwas Spezifischeres sein, wie ein `AudioContext`.

Ereignisse haben auch Methoden, das sind Funktionen, die als Eigenschaften auf dem Objekt verfügbar gemacht werden. Eine häufig verwendete Methode ist `preventDefault()`, die das Standardverhalten des Ereignisses verhindert, wenn sie aufgerufen wird.

Wenn Sie zum Beispiel Formular-Einsendungen selbst verarbeiten möchten, könnten Sie `preventDefault()` aufrufen, um zu verhindern, dass der Browser versucht, die Formulardaten als `POST`-Anfrage abzusenden. Mehr über `POST`-Anfragen lernen Sie in zukünftigen Lektionen.

Wahrscheinlich werden Sie auch auf die `stopPropagation()`-Methode stoßen. Diese Methode verhindert, dass das Ereignis nach oben weitergegeben oder an übergeordnete Elemente propagiert wird. Was das genau bedeutet, lernen Sie in einer späteren Lektion.

Es gibt außerdem eine große Anzahl von Eigenschaften, die spezifisch für bestimmte Implementierungen des `Event`-Objekts sind. Zum Beispiel hat ein `FetchEvent` eine request-Eigenschaft, die die Anfrage enthält, die das Ereignis ausgelöst hat.

Wenn Sie sich jemals unsicher sind, welche Eigenschaften verfügbar sind, können Sie das betreffende `Event`-Objekt protokollieren oder sogar die Dokumentation prüfen.
