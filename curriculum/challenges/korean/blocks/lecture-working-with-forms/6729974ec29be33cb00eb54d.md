# --interactive--

HTML의 `form` 요소는 이름이나 이메일 주소 같은 사용자 정보를 수집하는 데 사용됩니다. 다음은 `form` 요소의 예입니다:

```html
<form action="url-goes-here">
  <!-- input elements go here -->
</form>
```

`action` 속성은 제출 시 폼 데이터가 전송될 위치를 지정합니다. 이름과 이메일 주소 같은 특정 정보를 수집하려면 `input` 요소를 사용합니다. 다음은 `input` 요소를 사용하는 예입니다.

인터랙티브 편집기를 활성화하고 미리보기 창에서 `input` 요소의 입력 필드에 이름을 입력해 보세요.

:::interactive_editor

```html
<form action="">
  <input type="text" />
</form>
```

:::

`input` 요소는 빈 요소이며 닫는 태그가 없습니다. `type` 속성은 사용자로부터 기대하는 데이터 유형을 정의합니다. 이 경우 데이터는 일반 텍스트입니다. 입력에 레이블을 추가하려면 `label` 요소를 사용합니다. 다음은 `label` 요소를 사용하고 텍스트로 `Full Name:`를 포함한 예입니다.

미리보기 창에서 `Full Name:` 텍스트를 클릭하면 입력 필드에 포커스가 갑니다. 예제와 상호작용하려면 인터랙티브 편집기를 활성화해야 합니다.

:::interactive_editor

```html
<form action="">
  <label>
    Full Name:
    <input type="text" />
  </label>
</form>
```

:::

`input`를 `label` 요소 안에 중첩하면 `label`와 `input` 입력 필드 사이에 암묵적인 연관이 만들어집니다. "암묵적"이라는 용어는 추가 속성이나 요소 없이도 명시적으로 정의하지 않아도 이해되거나 추론되는 것을 의미합니다. `label`와 `input`를 명시적으로 연결하려면 `for` 속성을 사용할 수 있습니다. 다음은 이메일 주소 레이블에 `for` 속성을 사용하는 예입니다.

미리보기 창에서 `input` 요소에 `jane@example.com` 같은 가짜 이메일 주소를 입력해 보세요. 예제와 상호작용하려면 인터랙티브 편집기를 활성화해야 합니다.

:::interactive_editor

```html
<form action="">
  <label for="email"> Email Address: </label>
  <input type="email" id="email" />
</form>
```

:::

명시적 연관을 사용할 때 `for` 속성과 `id`의 값은 같아야 합니다. 이 경우 두 값 모두 `email`로 설정되어 있습니다. 입력의 `email` 유형은 올바른 형식의 이메일 주소에 대해 기본 유효성 검사를 제공합니다. 사용자에게 예상 입력에 대한 추가 힌트를 보여주고 싶다면 `placeholder` 속성을 사용할 수 있습니다. 다음은 이메일 입력 안에 `placeholder` 속성을 사용하는 예입니다.

인터랙티브 편집기를 활성화하고 이메일 입력 필드를 클릭한 후 이메일을 입력하면 자리 표시자 텍스트가 사라지는 것을 볼 수 있습니다.

:::interactive_editor

```html
<form action="">
  <label for="email"> Email Address: </label>
  <input type="email" id="email" placeholder="example@email.com" />
</form>
```

:::

자리 표시자 텍스트에는 사용자에게 기대하는 데이터 형식과 유형을 보여주는 짧고 유용한 텍스트를 제공해야 합니다. 이 경우 자리 표시자 텍스트인 `example@email.com`는 사용자가 올바른 형식의 이메일 주소를 입력해야 함을 알려줍니다.
