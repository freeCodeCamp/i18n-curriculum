# --description--

Maoni katika uandishi wa programu hutumika kutoa muktadha zaidi kwa msimbo au kuacha vidokezo kwa ajili yako na wengine.

Maoni ni mistari au sehemu za maandishi ambazo hazizingatiwi na injini ya JavaScript wakati msimbo wako unatekelezwa. Zipo kwa manufaa tu ya watu wanaosoma msimbo huo, iwe ni wewe au mtu mwingine.

JavaScript hutoa njia mbili za kuongeza maoni kwenye msimbo wako: maoni ya mstari mmoja na maoni ya mistari mingi.

Maoni ya mstari mmoja huundwa kwa kutumia alama mbili za slashi mbele (`//`). Hapa kuna mfano:

```js
// I am a single line comment in JavaScript
```

Aina hii ya maoni ni nzuri kwa maelezo mafupi au ufafanuzi.

Hapa kuna mfano halisi kutoka kwa faili za mradi wa mtaala wa freeCodeCamp:

```js
// This is to allow English to build without having to download the i18n files.
// It fails when trying to resolve the i18n-curriculum path if they don't exist.
const curriculumLocale = process.env.CURRICULUM_LOCALE ?? 'english';
const I18N_CURRICULUM_DIR = path.resolve(
  __dirname,
  curriculumLocale === 'english' ? '.' : 'i18n-curriculum/curriculum'
);
```

Usijali kuelewa kile msimbo unachofanya kwa sasa kwa sababu ni ngumu zaidi kuliko ulivyojifunza hadi sasa. Badala yake, zingatia maoni yaliyowekwa na msanidi programu. Maoni haya yanatoa muktadha muhimu kwa sababu msimbo huu upo.

Maoni kama haya ni muhimu kwa wale wanaofanya kazi kwa timu kwa sababu mbili:

1. Wasanidi programu wengine wanaofanya kazi kwenye mradi wataelewa kusudi la msimbo huu.

1. Husaidia kuzuia mabadiliko au kufutwa bila kushauriana na timu, jambo ambalo linaweza kusababisha hitilafu au matatizo.

Aina nyingine ya maoni ni maoni ya mistari mingi. Hapa ni sintaksia ya msingi:

```js
/*
 I am a multiline comment.
 This is helpful for longer explanations.
*/
```

Maoni ya mistari mingi ni muhimu unapohitaji kuandika maelezo marefu, ufafanuzi, au vidokezo kwenye msimbo wako.

Tuchukulie tena faili za mradi wa mtaala wa freeCodeCamp kuona jinsi maoni ya mistari mingi yanavyoweza kutumika katika hali halisi.

```js
/* Since there can be more than one way to complete a certification (using the
legacy curriculum or the new one, for instance), we need a certification
field to track which certification this belongs to. */
const dupeCertifications = [
  {
    certification: 'responsive-web-design',
    dupe: '2022/responsive-web-design'
  }
];
const hasDupe = dupeCertifications.find(
  cert => cert.dupe === meta.superBlock
);
```

Kama awali, puuza msimbo wote wa JavaScript kwa sababu unatumia dhana ambazo bado hazijafundishwa. Badala yake, zingatia maoni yaliyowekwa na msanidi programu.

Msanidi programu katika timu, au hata mchangiaji mpya anayefanya kazi kwenye mradi, anaweza kuelewa kwa nini kipande hiki cha msimbo kiko hapa na kuwa na muktadha kamili kabla ya kufanya kazi katika eneo hili la mradi.

Ingawa maoni ni muhimu katika uandishi wa programu, ni muhimu kuepuka kuandika maoni mengi sana. Huna haja ya kuweka maoni kwenye kila mstari wa msimbo, hasa kama msimbo ni rahisi na unaeleweka kwa urahisi.

Hapa kuna mfano wa kutumia maoni kuelezea mambo yanayoonekana wazi:

```js
// This code uses the const keyword to create a new variable called price.
// We are assigning the number 10 to the price variable.
const price = 10;
```

Katika hali hii, hakuna haja ya kuongeza maoni yoyote hapa kwa sababu msimbo unajieleza wenyewe. Lengo ni kuboresha usomaji, si kujaa msimbo na maelezo yasiyo ya lazima.

Kama unataka kuongeza maoni kwenye miradi yako binafsi unapoanza kujifunza kuandika msimbo, hilo ni sawa. Lakini mara unapoanza kufanya kazi kwenye miradi halisi na wasanidi programu wengine, ni muhimu usitumie maoni kwa msimbo unaoeleweka kwa urahisi.

Pia ni muhimu usitumie maoni kusaidia kuficha msimbo mgumu, tata sana, au uliyoandikwa vibaya. Katika hali hizo, ni bora kurekebisha, au kubadilisha, msimbo wako ili wasanidi programu wengine waweze kuelewa vyema kinachoendelea.

Maoni ni zana zenye nguvu za kuandika kumbukumbu za msimbo wako na kuufanya uwe rahisi kueleweka. Unapaswa kutumia maoni kutoa muktadha au kuacha vidokezo kwa ajili yako na wengine.
