## タイポグラフィの紹介

- **定義**：タイポグラフィとは、テキストを視覚的に魅力的で読みやすくするために適切なフォントやフォーマットを選ぶ技術です。「タイプ」とは、個々の文字がどのようにデザインされ配置されているかを指します。
- **書体の定義**：書体とは、文字、数字、記号のセット全体のデザインやスタイルのことです。フォントファミリーの設計図のようなものです。
- **フォントの定義**：フォントとは、書体の特定のバリエーションで、サイズ、太さ、スタイル、幅などの特徴を持ちます。
- **セリフ書体**：文字の端に小さな線が付いたクラシックなスタイルの書体です。セリフ書体は本などの印刷物でよく使われます。
- **サンセリフ書体**：文字の端に小さな線がない、よりモダンな見た目の書体です。サンセリフ書体は画面上で読みやすいためデジタルデザインでよく使われます。例としてHelvetica、Arial、Robotoがあります。
- **フォントの太さ**：文字の太さのことで、ライト、レギュラー、ボールド、ブラックなどがあります。
- **フォントスタイル**：文字の傾きや向きのことで、イタリックやオブリークなどがあります。
- **フォント幅**：文字が占める横幅のことで、コンデンスドやエクスパンドなどがあります。
- **ベースライン**：ほとんどの文字が乗る想像上の線です。
- **キャップハイト**：大文字の高さで、ベースラインから上端までの距離です。
- **x-ハイト**：小文字の平均的な高さで、アセンダーやディセンダーを除いた部分です。
- **アセンダー**：小文字のうち、x-ハイトより上に伸びる部分で、例えば「h」「b」「d」「f」の上部です。
- **ディセンダー**：小文字のうち、ベースラインより下に伸びる部分で、例えば「y」「g」「p」「q」の下部です。
- **カーニング**：特定の文字ペア間のスペースを調整して読みやすさや美しさを向上させることです。例えば「A」と「V」の間のスペースを狭くすることです。
- **トラッキング**：テキスト全体の文字間隔を調整することです。文字間の距離を変えることで、テキストの密度や開放感に影響します。
- **リーディング**：行間のことで、ある行のベースラインから次の行のベースラインまでの垂直距離です。
- **タイポグラフィのベストプラクティス**：デザインをわかりやすくするために、明瞭でシンプルなフォントを選ぶことをお勧めします。特にウェブサイトの本文では重要です。読みやすいフォントはユーザーの関心を引きやすくなります。フォントは多くても2～3種類に抑えて視覚的一貫性を保つべきです。多すぎると読みづらくなり、ブランドのアイデンティティも弱まります。また、ウェブサイトの読み込み時間が長くなることもあります。フォントサイズを使って見出し、サブ見出し、段落などの視覚的な階層を作ることもできます。例えば、ウェブページのメイン見出しは大きなフォントサイズにし、サブ見出しは小さめにすることで、階層ごとに異なるフォントサイズを設定し、ユーザーが構造を視覚的にたどりやすくなります。

  
## CSS `font-family` プロパティ

- **定義**：フォントファミリーとは、共通のデザインを持つフォントのグループです。同じファミリーに属するフォントは同じ基本書体に基づいていますが、スタイル、太さ、幅にバリエーションがあります。複数のフォントファミリーを優先度の高い順にカンマで区切って指定できます。これらの代替フォントはフォールバックとして機能します。フォントファミリーリストの最後には必ず総称フォントファミリーを含めるべきです。

```css
font-family: Arial, Lato;
```

- **一般的なフォントファミリー**：CSSでよく使われるフォントファミリーには、serif、sans-serif、monospace、cursive、fantasy があります。

## ウェブセーフフォント

- **定義**：これらのフォントは、コンピューターやデバイスにインストールされている可能性が非常に高いフォントの部分集合です。ブラウザがフォントをレンダリングする際、ユーザーのシステム上でフォントファイルを探します。見つからない場合は通常、デフォルトのシステムフォントにフォールバックします。

- **ウェブセーフフォント**：
  - サンセリフフォントは文字の端に小さな「足」や線がないため、画面上で読みやすく、ウェブ開発でよく使われます。ウェブセーフなサンセリフフォントの例はArial、Verdana、Trebuchet MSです。
  - 一方、セリフフォントは文字の端に小さな「足」があり、伝統的な印刷物でよく使われます。ウェブセーフなセリフフォントにはTimes New RomanやGeorgiaがあります。

## アットルールと`@font-face` アットルール

- **定義**：アットルールはブラウザに指示を与える文です。メディアクエリ、キーフレーム、フォントフェイスなど、スタイルシートのさまざまな側面を定義するために使えます。  
- **`@font-face`**：フォントファイル、フォーマット、太さやスタイルなどの重要なプロパティを指定してカスタムフォントを定義できます。`@font-face` アットルールを有効にするには、`src` プロパティも指定する必要があります。このプロパティにはフォントリソースへの参照が含まれます。

```css
@font-face {
  font-family: "MyCustomFont"; 
  src: url("path/to/font.woff2"),
       url("path/to/font.woff"),
       url("path/to/font.otf");
}
```

- **フォントフォーマット**：各フォントリソースに対してフォーマットを指定できます。これは任意で、ブラウザへのフォントフォーマットのヒントです。フォーマットが省略されると、リソースはダウンロードされ、ダウンロード後にフォーマットが検出されます。フォーマットが無効な場合はリソースはダウンロードされません。可能なフォントフォーマットには`collection`、`embedded-opentype`、`opentype`、`svg`、`truetype`、`woff`、`woff2`があります。

```css
@font-face {
  font-family: "MyCustomFont"; 
  src: url("path/to/font.woff2") format("woff2"),
       url("path/to/font.otf") format("opentype"),
       url("path/to/font.woff") format("woff");
}
```

- **`woff` と `woff2`**：`woff` は「Web Open Font Format」の略で、MozillaがType Supply、LettError、その他の組織と協力して開発した広く使われているフォントフォーマットです。`woff` と `woff2` の違いはデータ圧縮に使われるアルゴリズムです。
- **OpenType**：MicrosoftとAdobeが開発したスケーラブルなコンピュータフォントのフォーマットで、フォントの追加機能にアクセスできます。主要なOSで広く使われています。
- **`tech()`**：フォントリソースの技術を指定するために使われます。これは任意です。

```css
@font-face {
  font-family: "MyCustomFont"; 
  src: url("path/to/font.woff2") format("woff2"),
       url("path/to/font.otf") format("opentype") tech(color-COLRv1),
       url("path/to/font.woff") format("woff");
}
```

## 外部フォントの利用

- **定義**：外部フォントとは、プロジェクトファイルに直接含まれていないフォントファイルのことです。通常は別のサーバーにホストされています。これらの外部フォントをプロジェクトに含めるには、CSS内で`link` 要素や`@import` 文を使います。 
- **Google Fonts**：Googleが提供するサービスで、ウェブ開発向けに設計されたフォントのコレクションを提供しています。

`link` 要素を使った例：

```html
<link rel="preconnect" href="https://fonts.googleapis.com">
<link rel="preconnect" href="https://fonts.gstatic.com" crossorigin>
<link href="https://fonts.googleapis.com/css2?family=Roboto:ital,wght@0,100;0,300;0,400;0,500;0,700;0,900;1,100;1,300;1,400;1,500;1,700;1,900&display=swap" rel="stylesheet">
```

```css
.roboto-thin {
  font-family: "Roboto", sans-serif;
  font-weight: 100;
  font-style: normal;
}
```

`@import` 文を使った例：

```css
@import url('https://fonts.googleapis.com/css2?family=Roboto:ital,wght@0,100;0,300;0,400;0,500;0,700;0,900;1,100;1,300;1,400;1,500;1,700;1,900&display=swap');
```

- **Font Squirrel**：カスタム外部フォントをプロジェクトに追加するためのもう一つの人気のあるリソースです。

## `text-shadow` プロパティ

- **定義**：このプロパティはテキストに影を付けるために使います。影のテキストからの水平距離と垂直距離を表すXオフセットとYオフセットを指定する必要があります。これらは必須の値です。XオフセットとYオフセットの正の値は影を右と下に移動させ、負の値は左と上に移動させます。

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<p>Typography Shadow</p>
```

```css
  p {
    text-shadow: 3px 2px;
  }
```

:::

- **影の色**：オフセットの前後に影の色を指定してカスタマイズできます。色を指定しない場合はブラウザが自動的に決定するため、通常は明示的に設定することをお勧めします。

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<p>Colored Shadow</p>
```

```css
p {
  text-shadow: 3px 2px #00ffc3;
}
```

:::

- **ぼかし半径**：ぼかし半径は任意ですが、影をより滑らかで控えめに見せます。デフォルト値は0で、値が大きいほどぼかしが強くなり、影が薄くなります。

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<p>Blurred Shadow</p>
```

```css
p {
  text-shadow: 3px 2px 3px #00ffc3;
}
```

:::

- **複数のテキストシャドウの適用**：テキストには複数の影を付けることができます。影は前から後ろへ層状に適用され、最初の影が一番上に表示されます。

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<p>Multiple Shadows</p>
```

```css
p {
  text-shadow:
    3px 2px 3px #00ffc3,
    -3px -2px 3px #0077ff,
    5px 4px 3px #dee7e5;
}
```

:::
