# --description--

في JavaScript، `Map` هو كائن مدمج يخزن أزواج المفتاح-القيمة، مشابه للكائن. لكنه يختلف عن كائنات JavaScript العادية بالسماح باستخدام مفاتيح من أي نوع، بما في ذلك الكائنات والدوال.

`WeakMap` هو مجموعة من أزواج المفتاح-القيمة، مشابه لـ `Map`، لكنه يستخدم مراجع ضعيفة لمفاتيحه. يجب أن تكون المفاتيح كائنات، بينما يمكن أن تكون القيم من أي نوع.

فكر في العلاقة بين `WeakMap` و `Map` كما هي العلاقة بين `WeakSet` و `Set`، كما تعلمت في الدرس السابق.

لإنشاء `Map`، استخدم دالة البناء `Map()` مسبوقة بالكلمة المفتاحية `new`:

```js
const myFirstMap = new Map();
```

يمكنك تهيئة الـ Map بالقيم:

```js
const myTreesMap = new Map([
 [{ type: 'deciduous' }, 'Maple tree'],
 [['forest', 'grove'], 'Pine tree'],
 [42, 'Oak tree'],
 [true, 'Birch tree'],
 [function() { return 'I am a function key'; }, 'Willow tree'],
]);
```

إذا لم تهيئ الـ `Map` بالقيم، يمكنك استخدام الطريقة `set()` لإضافتها:

```js
const myTreesMap = new Map();
myTreesMap.set({ type: 'deciduous' }, 'Maple tree');
myTreesMap.set([1, 2], 'Pine tree');
myTreesMap.set(42, 'Oak tree');
myTreesMap.set(true, 'Birch tree');
myTreesMap.set(function() { return "I'm a function key"; }, 'Willow tree');

console.log(myTreesMap);
```

هذا شكل الـ `Map` في وحدة التحكم:

```js
/*
```

طرق أخرى يمكنك استخدامها للعمل مع `Map` هي:

- `get(key)` لاسترجاع القيمة المرتبطة بـ `key` المحدد.
- `has(key)` للتحقق مما إذا كان `key` موجودًا في الـ `Map`.
- `delete(key)` لحذف زوج المفتاح-القيمة من الـ `Map`.
- `clear()` لمسح جميع أزواج المفتاح-القيمة.
- `entries()` لعرض مدخلات الـ `Map` (تعيد المدخلات في `MapIterator`).
- `forEach()` للتكرار على مدخلات الـ `Map`.
- `size` للإشارة إلى عدد أزواج المفتاح-القيمة في الـ `Map`.
    

هناك دالة البناء `WeakMap()` يمكنك استخدامها لإنشاء `WeakMap`:

```js
const myFirstWeakMap = new WeakMap();
```

طرق `set()`، `get()`، `has()`، و `delete()` متاحة أيضًا للاستخدام مع `WeakMap`. على سبيل المثال، هذا كيف يمكنك تعيين عناصر إلى الـ `Map` باستخدام الطريقة `set()`:

```js
const myTreeWeakMap = new WeakMap();

myTreeWeakMap.set({ id: 1 }, 'Maple tree');
myTreeWeakMap.set({ id: 2 }, 'Pine tree');
myTreeWeakMap.set({ id: 3 }, 'Oak tree');
myTreeWeakMap.set({ id: 4 }, 'Birch tree');
myTreeWeakMap.set({ id: 5 }, 'Willow tree');

console.log(myTreeWeakMap);
```

هذا شكل الـ `WeakMap` في وحدة التحكم:

```js
/*
WeakMap {{…} => 'Willow tree', {…} => 'Maple tree', {…} => 'Pine tree', {…} => 'Oak tree'}
  [[Entries]]
    No properties
  [[Prototype]]: WeakMap
*/
```

إليك الفروقات بين `Map` و `WeakMap`:

| **الميزة** | **Map** | **WeakMap** |
| --- | --- | --- |
| **نوع المفتاح** | يمكن أن تكون المفاتيح من أي نوع بيانات، بما في ذلك السلاسل النصية، الأرقام، الكائنات، أو حتى الدوال. | يجب أن تكون المفاتيح كائنات. |
| **حالة الاستخدام** | استخدم `Map` عندما تحتاج إلى ربط بيانات بأي نوع من المفاتيح. | استخدم `WeakMap` عندما تحتاج فقط إلى ربط بيانات بالكائنات. |
| **التكرار** | يمكنك التكرار عبر `Map` باستخدام `forEach()`، `keys()`، `values()`، أو `entries()`. | `WeakMap` غير قابل للتكرار. |
| **خاصية الحجم** | يحتوي `Map` على خاصية `size` للحصول على عدد أزواج المفتاح-القيمة. | لا يحتوي `WeakMap` على خاصية `size`. |

في الجدول أعلاه، يمكنك رؤية الفروقات ملخصة بما في ذلك نوع المفتاح، حالة الاستخدام، التكرار، وخاصية الحجم. خذ لحظة لقراءة محتوى هذا الجدول لتتعرف أكثر على اختلافاتهما.
