# --description--

SQLでテーブルを定義するには、各カラムのデータ型を指定する必要があります。

これはSQLでテーブルを作成する基本的な構文であることを覚えておいてください。各カラムにはそれぞれのデータ型があります。

```sql
CREATE TABLE table_name(
  column1 data_type column_constraint,
  column2 data_type column_constraint,
  column3 data_type column_constraint,
  ... etc
);
```

開発者としてのあなたの目標は、これらのデータ型をできるだけ正確に選ぶことです。

SQLには6つの人気のあるデータ型のカテゴリがあります：

- 数値型。例えば`INTEGER`、`FLOAT`、`SERIAL`、`DECIMAL`などです。
- 日付と時間型。例えば`TIMESTAMP`、`DATE`、`TIME`などです。
- 文字型および文字列型。これには`CHAR`、`VARCHAR`、`TEXT`が含まれます。
- Unicode型。`NTEXT`や`NVARCHAR`などがあり、文字の起源に関係なくテキストが正しく保存および取得されるように使われます。
- バイナリ型。画像、音声、動画ファイルなどの非テキストデータを保存するために使われます。
- その他の雑多なデータ型。例えば`XML`や`TABLE`などです。

このレッスンでは、PostgreSQLで最も広く使われているものをいくつか紹介します。データ型はデータベース管理システムによって異なりますが、一般的にはかなり似ています。ですので、ドキュメントで完全なリストを確認する必要があります。

まずは数値型から始めましょう。

最初に、`INTEGER`データ型があります：

```sql
units_sold INTEGER
```

この例では、このデータ型を`units_sold`カラムに代入しています。

`INTEGER`型はデータベース内で4バイトを消費し、値の範囲は-2,147,483,648から2,147,483,647までです。PostgreSQLの公式ドキュメントでは「整数の典型的な選択肢」と説明されています。

また、`SMALLINT`と`BIGINT`もあり、これは基本的に`INTEGER`と同じですが、データベース内で割り当てられるサイズの違いにより、数値の範囲がそれぞれ小さく、または大きくなっています。

PostgreSQLで利用できる便利な機能として、`SERIAL`キーワードを使ってカラムを作成する方法があります。ここでは、`id`を使って`SERIAL`カラムが作成されています：

```sql
id SERIAL
```

これは真のデータ型ではありませんが、ユニークな識別子カラムを作成するのに非常に役立ちます。なぜなら、このカラムは`INTEGER`型になり、`NULL`値を許可せず、行が追加されると自動的にインクリメントされるからです。

例えば、最初の行は自動的に`id`の`1`を持ち、2行目は`id`の`2`を持つなど、各レコードにユニークなIDが作成されます。

MySQLでは、`SERIAL`に相当するのは`AUTO_INCREMENT`属性で、これは連続した整数を自動的に生成するために使われます：

```sql
id INT AUTO_INCREMENT
```

異なるデータベース管理システムでは、同じ機能を実現するために異なる方法があることがわかります。

これらは最も一般的に使われる数値データ型ですが、時にはテキストや文字のシーケンスを表す必要があります。

そのために`VARCHAR`データ型があります。このデータ型は可変長の文字列に使われます。丸括弧内で最大文字数を設定できます：

```sql
name VARCHAR(50)
```

この例では、最大50文字までの値を持つ`name`カラムを定義しています。

このデータ型は文字列の最大長を設定しますが、任意の長さの文字列を保存する必要がある場合は、代わりに`TEXT`を使うことができます：

```sql
name TEXT
```

数値や文字列に加えて、日付や時間を保存することも一般的です。

例えば、イベントを保存するテーブルを作成する場合、各イベントの日付を特定の形式で保存する必要があります。そのために`DATE`データ型があります：

```sql
event_date DATE
```

時間を保存するには、`TIME`データ型も使えます。例えば、イベントの開始時間を保存する場合です：

```sql
start_time TIME
```

`TIMESTAMP`データ型は両方を組み合わせています。日付と時間の両方を含みます。`TIMESTAMP WITH TIME ZONE`と書くとタイムゾーンも含まれる場合があります：

```sql
event_timestamp TIMESTAMP
```

```sql
event_timestamp TIMESTAMP WITH TIME ZONE
```

最後に、ブール値を保存する必要がある場合、`TRUE`または`FALSE`を使うことができる`BOOLEAN`データ型があります：

```sql
is_active BOOLEAN
```

これでSQLで最も一般的なデータ型のいくつかを知りましたが、他にもたくさんあります。使用しているデータベース管理システムのドキュメントで、利用可能なデータ型についてさらに詳しく調べてください。
