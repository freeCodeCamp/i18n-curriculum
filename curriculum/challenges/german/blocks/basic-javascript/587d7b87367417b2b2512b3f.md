# --description--

Eines der größten Probleme bei der Deklaration von Variablen mit dem Schlüsselwort `var` ist, dass Sie Variablendeklarationen leicht überschreiben können:

```js
var camper = "James";
var camper = "David";
console.log(camper);
```

Im obigen Code wird die Variable `camper` ursprünglich als `James` deklariert und dann überschrieben, sodass sie `David` ist. Die Konsole zeigt dann den String `David` an.

In einer kleinen Anwendung stoßen Sie möglicherweise nicht auf dieses Problem. Aber wenn Ihr Codeumfang größer wird, könnten Sie versehentlich eine Variable überschreiben, die Sie nicht überschreiben wollten. Da dieses Verhalten keinen Fehler auslöst, wird die Suche nach und das Beheben von Bugs schwieriger.

Ein Schlüsselwort namens `let` wurde in ES6 eingeführt, einem großen Update von JavaScript, um dieses potenzielle Problem mit dem Schlüsselwort `var` zu lösen. Weitere ES6-Funktionen lernen Sie in späteren Aufgaben kennen.

Wenn Sie `var` im obigen Code durch `let` ersetzen, führt das zu einem Fehler:

```js
let camper = "James";
let camper = "David";
```

Den Fehler können Sie in der Browserkonsole sehen.

Im Gegensatz zu `var` kann eine Variable mit demselben Namen bei Verwendung von `let` nur einmal deklariert werden.

# --instructions--

Aktualisieren Sie den Code so, dass nur das Schlüsselwort `let` verwendet wird.
