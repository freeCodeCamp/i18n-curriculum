# --interactive--

JavaScriptでは、プリミティブ型と非プリミティブ型のデータ型の違いを理解することが、効率的でバグのないコードを書くために重要です。

これら2つのデータ型のカテゴリは、メモリへの格納方法やプログラム内での操作方法が異なります。

プリミティブ型はJavaScriptで最も単純なデータの形態です。number、bigint、string、boolean、`null`、`undefined`、およびsymbolが含まれます。これらの型は「プリミティブ」と呼ばれ、単一の値を表し、オブジェクトではないためです。

プリミティブ型を扱うときは、その値を直接操作しています。例えば、プリミティブ値を持つ変数を作成すると、その値は変数に直接格納されます。

プリミティブ値は不変であり、一度作成されると値を変更できません。ただし、変数に新しい値を再代入することは可能です。プリミティブ型を扱う例を示します。

:::interactive_editor

```js
let num1 = 5;
let num2 = num1;
num1 = 10;

console.log(num2); // 5
```

:::

この例では、`5`から`num1`にプリミティブ値（`num2`）を代入しています。これは値の独立したコピーを作成します。そのため、元の変数（`num1`）に加えた変更はコピー（`num2`）に影響しません。

一方、非プリミティブ型はより複雑です。JavaScriptでは、これらはオブジェクトであり、通常のオブジェクト、配列、関数が含まれます。プリミティブとは異なり、非プリミティブ型は複数の値をプロパティや要素として保持できます。

非プリミティブ値を持つ変数を作成すると、その変数に格納されるのはオブジェクト自体ではなく、オブジェクトがメモリに格納されている場所への参照です。これにより、動作に重要な違いが生じます。非プリミティブ型の例を示します。

:::interactive_editor

```js
const originalPerson = { name: "John", age: 30 };
const copiedPerson = originalPerson;

originalPerson.age = 31;

console.log(copiedPerson.age); // 31
```

:::

この例では、`originalPerson`というオブジェクトがあり、`name`と`age`という2つのプロパティを持っています。次に、`originalPerson`オブジェクトを`copiedPerson`という変数に代入しています。

その後、`age`オブジェクトの`originalPerson`の値を更新します。`age`オブジェクトの`copiedPerson`プロパティをログに出力すると、更新された値が表示されます。

なぜそうなるのでしょうか？これは、`originalPerson`と`copiedPerson`の両方が同じメモリ上のオブジェクトを参照しているためです。

JavaScriptでは、オブジェクトを別の変数に代入すると、オブジェクト自体ではなくオブジェクトへの参照がコピーされます。これは参照による浅いコピーとして知られています。その結果、1つの参照を通じてオブジェクトに加えた変更は、そのオブジェクトへのすべての参照に反映されます。

JavaScriptを使い続けると、プリミティブ型と非プリミティブ型の違いを理解することが重要な状況に多く出会います。これは言語の多くの側面の基礎となる概念であり、効率的で正しいコードを書くための鍵です。
