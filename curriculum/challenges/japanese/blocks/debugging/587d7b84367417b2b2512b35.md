# --description--

`console.log()` と `typeof` メソッドは、プログラムの中間値や出力の型を確認する主な方法の2つです。ここからはバグのよくある形について見ていきましょう。高速タイピングをする人が共感できる構文レベルの問題の一つに、単純なスペルミスがあります。

変数名や関数名で文字が入れ替わったり、抜けていたり、大文字・小文字が間違っていると、ブラウザは存在しないオブジェクトを探そうとして、参照エラーとして警告します。JavaScriptの変数名と関数名は大文字・小文字を区別します。

# --instructions--

コード内の2つのスペルミスを修正して、`netWorkingCapital` の計算が正しく動作するようにしてください。

# --hints--

netWorkingCapital の計算で使われている2つの変数のスペルを確認してください。コンソールの出力は「Net working capital is: 2」と表示されるはずです。

```js
assert(netWorkingCapital === 2);
```

コード内にスペルミスのある変数が存在しないようにしてください。

```js
assert(!__helpers.removeJSComments(code).match(/recievables/g));
```

`receivables` 変数はコード内で正しく宣言され、使用されている必要があります。

```js
assert(__helpers.removeJSComments(code).match(/receivables/g).length == 2);
```

コード内にスペルミスのある変数が存在しないようにしてください。

```js
assert(!__helpers.removeJSComments(code).match(/payable;/g));
```

`payables` 変数はコード内で正しく宣言され、使用されている必要があります。

```js
assert(__helpers.removeJSComments(code).match(/payables/g).length == 2);
```
