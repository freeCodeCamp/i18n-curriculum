# --description--

クッキーは、ウェブクッキーやブラウザクッキーとも呼ばれ、サーバーがユーザーのウェブブラウザに送信する小さなデータの断片です。これらのクッキーはユーザーのデバイスに保存され、その後のリクエストとともにサーバーに返送されます。

クッキーは、HTTPがステートレスなプロトコルであるため、ウェブアプリケーションが状態を維持しユーザー情報を記憶するのに不可欠です。

クッキーは、ユーザーの設定、セッションデータ、トラッキング情報など、さまざまな情報を保存できます。

クッキーは常に名前と値のペアとして保存されます。つまり、各クッキーには名前（キー）とそれに対応する値があります。

例えば、クッキーはユーザーのセッションIDを次のように保存することがあります：`sessionId=abc123`。キーは`sessionId`で、値は`abc123`です。ブラウザがサーバーと通信するたびに、これらのクッキーは名前と値のペアの形で送信されます。

ユーザーがウェブサイトを訪れると、サーバーはHTTPレスポンスにSet-Cookieヘッダーを含めることで、1つ以上のクッキーをユーザーのブラウザに送信できます。ヘッダーはHTTPリクエストやレスポンスに関する追加情報を提供するキーと値のペアです。HTTPリクエストとレスポンスについては今後のレッスンで詳しく学びます。

クッキーが設定されると、ブラウザはそれらを保存し、同じドメインへの後続のリクエストごとにCookieヘッダーに自動的に含めます。これにより、サーバーは保存されたクッキーにアクセスして、ユーザーセッションの維持や設定の追跡などに利用できます。

以下はHTTPレスポンスでクッキーが設定される例です：

```http
Set-Cookie: sessionId=abc123; Expires=Wed, 21 Oct 2021 07:28:00 GMT; Secure; HttpOnly
```

ブラウザはクッキーを保存し、同じサーバーへの将来のリクエストでは`Cookie`ヘッダーにクッキーを含めます：

```http
Cookie: sessionId=abc123
```

サーバーはクッキーを読み取り、保存されたセッションIDを使ってユーザーがログインしているかどうかなどの情報を取得できます。

ここで、クッキーの種類を説明します。

- セッションクッキーは、ユーザーがウェブサイトを利用している間だけ有効です。ユーザーがブラウザやタブを閉じると、セッションクッキーは削除されます。これらのクッキーは、訪問中にユーザーをログイン状態に保つなどの用途に使われます。

- 永続クッキーは有効期限が設定されており、その期限までユーザーのデバイスに保存され続けます。永続クッキーは、ユーザーの設定やログイン情報をセッションをまたいで記憶するためによく使われます。

- セキュアクッキーはHTTPS経由でのみ送信され、通信途中で攻撃者に傍受されるリスクを減らします。

- HttpOnlyクッキーはブラウザで実行されるJavaScriptからアクセスや変更ができず、クロスサイトスクリプティング（XSS）攻撃に対してより安全です。XSS攻撃は、攻撃者が悪意のあるスクリプトをウェブページに注入し、他のユーザーが閲覧した際にそのスクリプトが被害者のブラウザ上で実行されることで、クッキーやセッションデータを盗んだり、ユーザーの知らないうちに悪意ある操作を行ったりする攻撃です。クッキーをHttpOnlyに設定すると、JavaScriptからのアクセスが防がれ、こうした攻撃のリスクが減ります。

クッキーはサーバーレスポンスの`Set-Cookie`ヘッダーやJavaScriptの`document.cookie`を使って作成できます。

JavaScriptでクッキーを設定する例を示します：

```js
document.cookie = "username=JohnDoe; expires=Fri, 31 Dec 2021 23:59:59 GMT; path=/";
```

このコマンドは、名前が`username`で値が`"JohnDoe"`のクッキーを2021年末まで有効に設定します。既存のクッキーは新しい値で再設定することで更新できます。

クッキーを削除するには、有効期限を過去の日付に設定します：

```js
document.cookie = "username=; expires=Thu, 01 Jan 1970 00:00:00 GMT; path=/";
```

これにより、ブラウザから`username`クッキーが削除されます。
