# --description--

قبل أن تبدأ فعليًا في استخدام التحكم في الإصدارات، ومواقع مثل GitHub، تحتاج إلى تثبيت Git وجعله جاهزًا للاستخدام.

للتحقق مما إذا كان Git مثبتًا بالفعل على جهازك، يمكنك تشغيل الأمر التالي في الطرفية:

```sh
git --version
```

إذا رأيت رقم إصدار، فهذا يعني أن Git مثبت. إذا لم يكن كذلك، فستحتاج إلى تثبيته.

في أنظمة Linux، غالبًا ما يأتي Git مثبتًا مسبقًا مع معظم التوزيعات. إذا لم يكن Git مثبتًا مسبقًا، يمكنك تثبيته باستخدام أوامر مدير الحزم مثل `sudo apt-get install git` أو `sudo pacman -S git`.

لمستخدمي Mac، يمكنك تثبيت Git عبر Homebrew باستخدام الأمر `brew install git`، أو يمكنك تنزيل مثبت التنفيذ من موقع Git الرسمي.

لمستخدمي Windows، يمكنك تنزيل مثبت التنفيذ من موقع Git الرسمي. أو، إذا قمت بإعداد Chocolatey، يمكنك تشغيل الأمر `choco install git.install` في PowerShell. لاحظ أنه على Windows، قد ترغب أيضًا في تنزيل Git Bash للحصول على بيئة شل شبيهة بـ Unix.

للتأكد من نجاح التثبيت، شغّل الأمر `git --version` مرة أخرى في الطرفية.

بمجرد تثبيت Git، ستحتاج إلى إجراء بعض التغييرات على الإعدادات. يجب أن تقوم بهذه التغييرات مرة واحدة فقط على جهازك ولن تحتاج إلى تعديلها باستمرار عند التحديثات.

يُستخدم الأمر `git config` لضبط متغيرات الإعداد التي تتحكم في كيفية عمل Git على جهازك. لعرض متغيرات الإعداد الحالية وأماكن تخزينها على نظامك، يمكنك تشغيل الأمر التالي:

```sh
git config --list --show-origin
```

الآن يجب أن ترى فقط إعدادات على مستوى النظام إذا كنت قد ثبتت Git لأول مرة. إذا كنت قد ثبت Git سابقًا قبل هذا الدرس، فقد ترى إعدادات المستخدم لاسم المستخدم والبريد الإلكتروني.

لتعيين اسم المستخدم، يمكنك تشغيل الأمر التالي:

```sh
git config --global user.name "Jane Doe"
```

يُستخدم الخيار `--global` هنا لتعيين اسم المستخدم لجميع المشاريع على جهازك التي تستخدم Git. إذا كنت تريد تجاوز اسم المستخدم لمشروع معين، يمكنك تشغيل الأمر داخل مجلد ذلك المشروع بدون الخيار `--global`.

لتعيين عنوان البريد الإلكتروني للمستخدم، يمكنك تشغيل الأمر التالي:

```sh
git config --global user.email janedoe@example.com
```

تكوين اسم المستخدم والبريد الإلكتروني مهم لأن Git سيستخدم هذه المعلومات مع كل عملية commit تقوم بها في المشروع. ستتعلم المزيد عن عمليات commit في درس لاحق.

يمكنك أيضًا تعيين محرر النصوص المفضل الذي تريد أن يستخدمه Git. إليك مثالًا على تعيين المحرر المفضل إلى Emacs:

```sh
git config --global core.editor emacs
```

تشمل الخيارات الأخرى Vim وNano وVS Code والمزيد. على سبيل المثال، إذا كنت تستخدم VS Code كمحرر، يمكنك تكوين إعدادات Git هكذا:

```sh
git config --global core.editor "code --wait"
```

إذا اخترت عدم تعيين محرر مفضل، فسيستخدم Git المحرر الافتراضي للنظام.

إذا كنت تستخدم جهاز Windows وترغب في تغيير المحرر، ستحتاج إلى توفير المسار الكامل لملف التنفيذ هكذا:

```sh
git config --global core.editor "'C:/Program Files/Notepad++/notepad++.exe' -multiInst -notabbar -nosession -noPlugin"
```

لعرض قائمة الإعدادات الخاصة بك، يمكنك تشغيل الأمر التالي:

```sh
git config --list
```

الآن بعد أن ثبتت وأعددت Git، أنت جاهز لبدء إضافة التحكم في الإصدارات إلى مشاريعك.
