# --interactive--

L'ambito nella programmazione indica la visibilità e la possibilità di consultare le variabili in diverse parti del tuo codice. Determina dove le variabili possono essere consultate o modificate. In JavaScript, comprendere l'ambito è fondamentale per scrivere codice pulito, efficiente e senza bug. Esistono tre tipi principali di ambito: ambito globale, ambito locale e ambito di blocco.

L'ambito globale è l'ambito più esterno in un programma JavaScript. Le variabili dichiarate nell'ambito globale sono consultabili da qualsiasi punto del codice, comprese le funzioni e i blocchi. Queste variabili sono spesso chiamate variabili globali. Anche se le variabili globali possono essere comode, dovrebbero essere usate con parsimonia perché possono causare conflitti di nomi e rendere il codice più difficile da mantenere. Ecco un esempio di variabile globale:

:::interactive_editor

```js
let globalVar = "I'm a global variable";

function printGlobalVar() {
    console.log(globalVar);
}

printGlobalVar(); // "I'm a global variable"
```

:::

In questo esempio, `globalVar` è dichiarata nell'ambito globale e può essere consultata all'interno della funzione `printGlobalVar`.

L'ambito locale, invece, si riferisce alle variabili consultabili solo all'interno di una funzione. Ecco un esempio di ambito locale:

:::interactive_editor

```js
function greet() {
    let message = "Hello, local scope!";
    console.log(message);
}

greet(); // "Hello, local scope!"
// console.log(message); // This will throw an error
```

:::

In questo codice, `message` è una variabile locale all'interno della funzione `greet`. Può essere usata dentro la funzione, ma tentare di consultarla fuori dalla funzione genererà un errore.

L'ambito di blocco è un concetto introdotto con le parole chiave `let` e `const` in ES6. Un blocco è qualsiasi sezione di codice racchiusa tra parentesi graffe, `{}`, come nelle istruzioni `if`, nei loop `for` o nei loop `while`. Il concetto di loop sarà insegnato in una prossima lezione.

Le variabili dichiarate con `let` o `const` all'interno di un blocco sono consultabili solo all'interno di quel blocco. Ecco un esempio di ambito di blocco:

```js
if (true) {
    let blockVar = "I'm in a block";
    console.log(blockVar); // "I'm in a block"
}
console.log(blockVar); // This will throw an error
```

In questo esempio, `blockVar` è consultabile solo all'interno del blocco `if`. Tentare di consultarla fuori dal blocco genererà un errore. Comprendere questi diversi tipi di ambito è essenziale per gestire la consultabilità delle variabili ed evitare effetti collaterali indesiderati nel codice.

Le variabili globali dovrebbero essere usate con parsimonia, perché possono causare conflitti di nomi e rendere il codice più difficile da mantenere. Le variabili locali aiutano a mantenere isolate le diverse parti del codice, cosa particolarmente utile in programmi più grandi. L'ambito di blocco con `let` e `const` offre un controllo ancora più preciso sulla consultabilità delle variabili, aiutando a prevenire errori e rendendo il codice più prevedibile. Padroneggiare questi concetti base di ambito globale, locale e di blocco fornirà una solida base per comprendere argomenti più avanzati.
