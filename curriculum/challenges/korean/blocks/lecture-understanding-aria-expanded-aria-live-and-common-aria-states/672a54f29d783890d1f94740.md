# --interactive--

`input`, `select`, `textarea`, `button`, `fieldset` 같은 의미론적 폼 컨트롤 요소들은 내장된 상태를 가지고 있어 보조 기술에 전달됩니다.

예를 들어, `disabled` 속성을 사용해 버튼을 비활성화하거나 `checked` 속성으로 체크박스가 선택되었음을 나타낼 수 있습니다.

하지만 사용자 정의 컨트롤 요소를 만들 때는 ARIA 속성을 사용해 컨트롤의 상태를 보조 기술에 전달해야 합니다.

이번 수업에서는 사용자 정의 컨트롤 요소에 사용할 수 있는 몇 가지 일반적인 ARIA 상태를 살펴보겠습니다.

첫 번째로 살펴볼 ARIA 상태는 `aria-selected`입니다. 이 상태는 요소가 선택되었음을 나타내는 데 사용됩니다. 탭 인터페이스, 리스트박스, 그리드 같은 사용자 정의 컨트롤에 사용할 수 있습니다.

다음은 사용자 정의 탭 컨트롤에서 `aria-selected`를 사용하는 예시입니다:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<div role="tablist">
  <button role="tab" aria-selected="true">Tab 1</button>
  <button role="tab" aria-selected="false">Tab 2</button>
  <button role="tab" aria-selected="false">Tab 3</button>
</div>
<script src="index.js"></script>
```

```css
[role="tablist"] {
  display: flex;
  border-bottom: 2px solid #ddd;
  gap: 0.25rem;
  font-family: system-ui, sans-serif;
}

[role="tab"] {
  appearance: none;
  border: none;
  background: none;
  padding: 0.5rem 1rem;
  cursor: pointer;
  font-size: 1rem;
  color: #444;
  border-radius: 4px 4px 0 0;
  transition: background-color 0.2s, color 0.2s;
}

[role="tab"]:hover {
  background-color: #f3f3f3;
}

[role="tab"][aria-selected="true"] {
  background-color: #fff;
  color: #0078d4;
  border: 2px solid #0078d4;
  border-bottom: 2px solid #fff; 
  font-weight: 600;
  position: relative;
  z-index: 1;
}

[role="tab"]:focus {
  outline: 2px solid #0078d4;
  outline-offset: 2px;
}

```

```js
document.addEventListener("click", (event) => {
  const clickedTab = event.target.closest('[role="tab"]');
  if (!clickedTab) return;

  const tablist = clickedTab.closest('[role="tablist"]');
  const tabs = tablist.querySelectorAll('[role="tab"]');

  tabs.forEach((tab) => {
    const isSelected = tab === clickedTab;
    tab.setAttribute("aria-selected", isSelected);
    tab.tabIndex = isSelected ? 0 : -1;
  });
});

document.addEventListener("keydown", (event) => {
  const activeTab = document.activeElement;
  if (activeTab.getAttribute("role") !== "tab") return;

  const tablist = activeTab.closest('[role="tablist"]');
  const tabs = Array.from(tablist.querySelectorAll('[role="tab"]'));
  const index = tabs.indexOf(activeTab);

  let newIndex = index;
  if (event.key === "ArrowRight") newIndex = (index + 1) % tabs.length;
  if (event.key === "ArrowLeft") newIndex = (index - 1 + tabs.length) % tabs.length;

  if (newIndex !== index) {
    tabs[newIndex].focus();
    tabs[newIndex].click();
  }
});

```

:::

탭은 제한된 공간에 여러 콘텐츠 패널을 표시하는 데 사용됩니다. `aria-selected` 상태는 현재 선택된 탭을 나타내는 데 사용됩니다.

사용자가 탭을 선택하면 선택된 탭의 `aria-selected` 상태가 `true`로 설정되고, 다른 탭들의 `aria-selected` 상태는 `false`로 설정됩니다.

또 다른 일반적인 ARIA 상태는 `aria-disabled`입니다. 이 상태는 화면 읽기 프로그램 같은 보조 기술 사용자에게만 요소가 비활성화되었음을 알리는 데 사용됩니다. `aria-disabled`는 실제로 요소를 비활성화하지 않는다는 점이 중요합니다. 비활성화된 요소처럼 보이고 작동하도록 만드는 것은 개발자의 몫입니다. 이 속성은 `disabled` 속성 대신 네이티브 HTML 요소에 자주 사용됩니다. 어떤 것을 선택할지는 버튼이 사용되는 상황에 따라 다릅니다.

다음은 사용자 정의 편집 버튼에서 `aria-disabled`를 사용하는 예시입니다:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<div role="button" tabindex="-1" aria-disabled="true">Edit</div>
```

```css
[role="button"] {
  display: inline-block;
  background-color: #0078d4;
  color: #fff;
  padding: 0.5rem 1rem;
  border-radius: 6px;
  font-family: system-ui, sans-serif;
  font-size: 1rem;
  text-align: center;
  cursor: pointer;
  user-select: none;
  transition: background-color 0.2s, transform 0.1s, opacity 0.2s;
}

[role="button"]:not([aria-disabled="true"]):hover {
  background-color: #005fa3;
}

[role="button"]:not([aria-disabled="true"]):focus {
  outline: 2px solid #005fa3;
  outline-offset: 2px;
}

[role="button"]:not([aria-disabled="true"]):active {
  transform: scale(0.97);
}

[role="button"][aria-disabled="true"] {
  opacity: 0.5;
  pointer-events: none;
  cursor: not-allowed;
  background-color: #b0b0b0;
  color: #f2f2f2;
}
```

:::

`aria-disabled` 속성은 화면 읽기 프로그램 사용자에게 편집 버튼이 비활성화되어 상호작용할 수 없음을 알립니다. 다시 말하지만, 실제로 버튼을 비활성화하지는 않습니다. `aria-disabled`를 사용할 때는 컨트롤이 비활성화된 버튼처럼 보이고 작동하도록 스타일과 JavaScript를 적용해야 합니다.

대부분의 경우 네이티브 버튼 요소를 사용하겠지만, 사용자 정의 컨트롤이 필요한 경우도 있습니다. 따라서 컨트롤의 상태를 보조 기술에 전달하는 방법을 아는 것이 중요합니다.

다음으로 살펴볼 ARIA 상태는 `aria-haspopup`입니다. 이 상태는 인터랙티브 요소가 활성화될 때 팝업 요소를 트리거한다는 것을 나타냅니다. `aria-haspopup` 속성은 팝업이 `menu`, `listbox`, `tree`, `grid`, 또는 `dialog` 역할 중 하나일 때만 사용할 수 있습니다. `aria-haspopup`의 값은 이 역할 중 하나이거나 기본값인 `true`이어야 하며, 기본값은 `menu` 역할입니다.

다음은 `aria-haspopup`를 사용하는 파일 편집기 메뉴 예시입니다:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<button id="menubutton" aria-haspopup="menu" aria-controls="filemenu" aria-expanded="false">File</button>
<ul id="filemenu" role="menu" aria-labelledby="menubutton" hidden>
  <li role="menuitem" tabindex="-1">Open</li>
  <li role="menuitem" tabindex="-1">New</li>
  <li role="menuitem" tabindex="-1">Save</li>
  <li role="menuitem" tabindex="-1">Delete</li>
</ul>
```

```css
#menubutton {
  background-color: #0078d4;
  color: #fff;
  border: none;
  padding: 8px 14px;
  border-radius: 4px;
  cursor: pointer;
  font-size: 14px;
  position: relative;
}

#menubutton:hover,
#menubutton:focus {
  background-color: #005ea2;
  outline: none;
}

#filemenu {
  list-style: none;
  padding: 4px 0;
  margin: 4px 0 0;
  border: 1px solid #ccc;
  border-radius: 4px;
  background-color: #fff;
  width: 160px;
  position: absolute;
  z-index: 1000;
  box-shadow: 0 4px 6px rgba(0,0,0,0.1);
}

#filemenu[hidden] {
  display: none;
}

#filemenu [role="menuitem"] {
  display: block;
  padding: 8px 12px;
  font-size: 14px;
  color: #333;
  cursor: pointer;
}

#filemenu [role="menuitem"]:hover,
#filemenu [role="menuitem"]:focus {
  background-color: #e5f1fb;
  outline: none;
}

#filemenu [role="menuitem"]:focus-visible {
  box-shadow: inset 0 0 0 2px #0078d4;
}

```

:::

`aria-haspopup` 상태는 `File` 메뉴 버튼이 활성화될 때 팝업 메뉴를 열 것임을 나타냅니다. 화면 읽기 프로그램 사용자는 버튼에 접근할 때 이 추가 정보를 들을 수 있습니다.

팝업 메뉴를 표시하고 숨기려면 JavaScript를 사용해야 하며, 메뉴와 상호작용할 때 적절한 키보드 지원도 구현해야 합니다. 또한 ARIA `menu` 역할은 매우 특정한 유형의 메뉴를 가리킵니다. 일반적으로 사용자가 호출할 수 있는 작업 목록을 의미하며, 데스크톱 애플리케이션의 메뉴와 유사합니다. 일반적으로 우리가 "메뉴"라고 부르는 내비게이션 메뉴 같은 경우는 포함하지 않습니다. 현실적으로 웹에서 만드는 대부분의 "메뉴"는 ARIA 메뉴가 아니며 `aria-haspopup`를 사용하지 않습니다.

다음으로 살펴볼 ARIA 상태는 `aria-required`입니다. `aria-required` 속성은 폼이 제출되기 전에 필드를 반드시 채워야 함을 나타냅니다.

다음은 사용자 정의 폼 컨트롤에서 `aria-required` 속성을 사용하는 예시입니다:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<div id="name-label">Full Name*</div>
<div role="textbox" contenteditable aria-labelledby="name-label" aria-required="true" id="name"></div>
```

```css
#name-label {
  font-family: system-ui, sans-serif;
  font-size: 0.95rem;
  font-weight: 600;
  margin-bottom: 0.25rem;
  color: #333;
}

[role="textbox"] {
  display: block;
  width: 100%;
  min-height: 2rem;
  padding: 0.5rem 0.75rem;
  border: 1.5px solid #ccc;
  border-radius: 4px;
  font-family: system-ui, sans-serif;
  font-size: 1rem;
  color: #222;
  background-color: #fff;
  line-height: 1.4;
  transition: border-color 0.2s, box-shadow 0.2s;
}

[role="textbox"]:hover {
  border-color: #999;
}

[role="textbox"]:focus {
  outline: none;
  border-color: #0078d4;
  box-shadow: 0 0 0 3px rgba(0, 120, 212, 0.25);
}

[role="textbox"]:empty::before {
  content: attr(data-placeholder);
  color: #aaa;
  pointer-events: none;
}

[role="textbox"][aria-required="true"] {
  border-left: 3px solid #e81123;
  padding-left: calc(0.75rem - 3px);
}
```

:::

사용자가 입력할 수 있도록 `contenteditable` 속성을 사용해야 합니다. 또한 이 사용자 정의 폼 컨트롤이 필수임을 나타내기 위해 `aria-required` 속성을 `true`로 설정했습니다.

폼 컨트롤이 일반 폼 컨트롤처럼 보이도록 하려면 CSS를 추가해야 합니다. 또한 내용 없이 폼이 제출되는 것을 막으려면 JavaScript도 추가해야 합니다.

레이블에 이미 `required`라는 단어가 있다면 `aria-required` 속성은 생략해야 합니다. 이렇게 하면 화면 읽기 프로그램이 필수임을 한 번만 알립니다.

대부분의 경우 네이티브 `label`와 `form` 요소를 `required` 속성과 함께 사용할 것입니다. 하지만 사용자 정의 폼 컨트롤을 만들어야 한다면 필요할 때 `aria-required` 속성을 추가하는 것이 중요합니다.

또한 `aria-required` 속성은 `input`, `textarea`, `select` 같은 네이티브 폼 입력에도 사용할 수 있습니다. 이는 네이티브 required 속성보다 선호되는 경우가 많습니다. required 속성은 특히 브라우저가 제공하는 기본 오류 처리와 관련해 사용성 및 접근성 문제를 일으킬 수 있기 때문입니다. 궁극적으로 어떤 속성이 상황에 가장 적합한지는 테스트를 통해 결정해야 합니다.

마지막으로 살펴볼 ARIA 상태는 `aria-checked`입니다. 이 속성은 요소가 선택된 상태인지 여부를 나타내는 데 사용됩니다. 주로 사용자 정의 체크박스, 라디오 버튼, 스위치, 리스트박스에서 사용됩니다.

다음은 사용자 정의 체크박스 컨트롤에서 `aria-checked`를 사용하는 예시입니다:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<div role="checkbox" aria-checked="true" tabindex="0">Checkbox</div>
```

```css
[role="checkbox"] {
  display: inline-flex;
  align-items: center;
  gap: 0.5rem;
  font-family: system-ui, sans-serif;
  font-size: 1rem;
  cursor: pointer;
  user-select: none;
  color: #222;
}

[role="checkbox"]::before {
  content: "";
  display: inline-block;
  width: 1rem;
  height: 1rem;
  border: 2px solid #666;
  border-radius: 4px;
  background-color: #fff;
  transition: all 0.2s ease;
  box-sizing: border-box;
}

[role="checkbox"]:hover::before {
  border-color: #0078d4;
}

[role="checkbox"]:focus::before {
  outline: 2px solid #0078d4;
  outline-offset: 2px;
}

[role="checkbox"][aria-checked="true"]::before {
  background-color: #0078d4;
  border-color: #0078d4;
  background-image: url("data:image/svg+xml;utf8,<svg xmlns='http://www.w3.org/2000/svg' width='14' height='14' viewBox='0 0 14 14'><path fill='white' d='M5.2 10.4L2 7.2l1.1-1.1 2.1 2.1L10.9 2.5 12 3.6z'/></svg>");
  background-repeat: no-repeat;
  background-position: center;
}

[role="checkbox"][aria-disabled="true"] {
  opacity: 0.5;
  pointer-events: none;
  cursor: not-allowed;
}

```

:::

네이티브 체크박스 요소는 보조 기술에 전달되는 내장 `checked` 상태를 가지고 있습니다. 하지만 사용자 정의 체크박스 컨트롤을 만드는 경우 `aria-checked` 속성을 사용해 상태를 나타내야 합니다.

사용자가 사용자 정의 체크박스 컨트롤과 상호작용하면 `aria-checked` 상태를 사용해 체크박스의 새 상태를 반영해야 합니다. 체크박스가 선택되면 `aria-checked` 속성이 `true`로 설정되고, 선택 해제되면 `false`로 설정됩니다.

네이티브 요소가 일반적으로 더 나은 지원과 내장된 접근성 기능을 제공합니다.

하지만 사용자 정의 컨트롤을 반드시 만들어야 한다면 ARIA 속성을 사용해 이러한 컨트롤의 상태를 보조 기술에 효과적으로 전달하는 것이 필수입니다.

항상 작업을 테스트해 ARIA 속성이 올바르게 적용되었는지, 사용자 정의 컨트롤이 접근 가능하고 사용자 친화적으로 작동하는지 확인하세요.
