# --description--

在先前的課程中，你已經了解什麼是相依性。在這裡，你將進一步了解它們以及在為你的專案選擇外部軟體包時需要考慮的事項。

相依性是你的專案為了運作所需要的另一段程式碼。這些通常是其他開發者已經創建並在 npm 登錄檔中分享的外部軟體包。

使用相依性可以為你節省大量時間。你可以安裝已經解決常見問題的軟體包，而不是自己建置所有東西——像是建立網頁伺服器、處理路由、擷取資料、管理驗證等等。

在你的 `package.json` 檔案中，所有你的專案所依賴的軟體包都列在 `dependencies` 鍵下：

```json
{
  "name": "package-dependencies",
  "version": "1.0.0",
  "description": "a demo on how package dependencies work",
  "main": "index.js",
  "scripts": {
    "test": "echo \"Error: no test specified\" && exit 1"
  },
  "keywords": [
    "Node",
    "NPM"
  ],
  "author": "Kolade Chris",
  "license": "ISC",
  "dependencies": {
    "chalk": "^5.6.2",
    "lodash": "^4.17.21"
  }
}
```

在上方的範例 `package.json` 檔案中，`chalk` 和 `lodash` 被列為專案的相依套件。這表示專案依賴這些套件才能運作。如果它們未安裝且未列在 `package.json` 檔案中，任何使用它們的特性將會失效。

還有另一類依賴稱為 `devDependencies`。這些是你只在建置或測試你的專案時需要的套件，而非在生產環境中執行時需要的。常見範例包括：

* Nodemon 用於在有更新時自動重新啟動專案
    
* Jest 或 Mocha 用於測試
    
* 用於程式碼檢查的 ESLint
    

如你所料，`devDependencies` 出現在 `package.json` 檔案的 `devDependencies` 區段下：

```json
{
  "name": "package-dependencies",
  "version": "1.0.0",
  "description": "a demo on how package dependencies work",
  "main": "index.js",
  "scripts": {
    "test": "echo \"Error: no test specified\" && exit 1"
  },
  "keywords": [
    "Node",
    "NPM"
  ],
  "author": "Kolade Chris",
  "license": "ISC",
  "dependencies": {
    "chalk": "^5.6.2",
    "lodash": "^4.17.21"
  },
  "devDependencies": {
    "jest": "^30.2.0",
    "nodemon": "^3.1.10"
  }
}
```

在選擇要在你的專案中使用哪些軟體包時，有幾件重要的事情需要記住。

安全性應該永遠擺在第一位。即使是一個不安全的軟體包也可能為你的整個專案帶來問題。你也應該考慮效率，因為大型或最佳化不良的軟體包可能會拖慢你的專案速度。

它也有助於選擇受歡迎且支援良好的軟體包。受歡迎的軟體包通常擁有作用中的社群、良好的文件，以及頻繁的更新。檢查受歡迎程度的一個簡單方法是查看 npm 上每週的下載次數。例如，`chalk` 軟體包每週下載次數超過 300,000 次：

<img src="https://cdn.freecodecamp.org/curriculum/lecture-transcripts/what-are-package-dependencies-and-what-are-some-considerations-for-choosing-an-external-package-1.webp" alt="screenshot from npm of chalk package statistics">

維護也很重要。維護良好的軟體包會定期更新，並且能更好地與其他軟體的新版本搭配使用。

最後，不要忽略文件。清晰且詳盡的文件讓你更容易理解如何使用軟體包，並在將它整合到你的專案時為你節省大量時間。
