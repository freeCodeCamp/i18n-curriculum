# --description--

In Python, le variabili sono come un riquadro etichettato per memorizzare e fare riferimento a dati di tipi diversi. Per dichiarare variabili in Python, assegni un valore a un identificatore con l'operatore di assegnazione (`=`). Non è necessario usare parole chiave speciali come `let` o `const` in JavaScript, o `char` in C#.

In Python, scrivi semplicemente il nome della variabile a sinistra, seguito dall'operatore di assegnazione, e il valore che vuoi assegnare alla variabile a destra. Ecco un esempio di come dichiarare variabili `name` e `age`:

```python
name = 'John Doe'
age = 25
```

Nell'esempio sopra, la variabile `name` contiene il valore `'John Doe'`. Questo valore è una stringa, che è una serie di caratteri usati per rappresentare testo. Le stringhe si scrivono con virgolette singole o doppie, per esempio `'Hello'` o `"Hello"`. Nelle lezioni future, imparerai di più su come lavorare con le stringhe in Python.

Quando nomini le variabili in Python, ci sono alcune regole importanti da tenere a mente:

- I nomi delle variabili possono iniziare solo con una lettera o un underscore (`_`), non con un numero.
- I nomi delle variabili possono contenere solo caratteri alfanumerici (`a-z`, `A-Z`, `0-9`) e underscore (`_`).
- I nomi delle variabili sono case-sensitive — `age`, `Age` e `AGE` sono considerati tutti unici.
- I nomi delle variabili non possono essere una delle parole chiave riservate di Python come `if`, `class` o `def`.

Se infrangi una di queste regole, il tuo programma Python lancerà un `SyntaxError`:

```bash
    5variable_name = 5
     ^
SyntaxError: invalid syntax
```

Ora vediamo alcune convenzioni comuni per nominare le variabili in Python.

Innanzitutto, i nomi delle variabili dovrebbero essere in minuscolo, con le parole separate da un underscore. Questo si chiama snake case:

```python
my_variable_name = 'freeCodeCamp'
```

Poi, dovresti usare nomi descrittivi per le variabili. Per esempio, se vuoi salvare l'età di un utente come variabile, `user_age` è meglio di `age` o di un'abbreviazione come `ua`:

```python
user_age = 30
```

In questo modo, puoi comunicare facilmente lo scopo di una variabile ad altri membri del team (o a te stesso in futuro) in un grande codice.

Un'altra convenzione è evitare di usare nomi di variabili con una sola lettera. Questo è molto comune in Python, ma dovrebbe essere evitato perché i nomi di variabili con una sola lettera non comunicano alcuno scopo o significato:

```python
x = 56 # What do you mean by x?
```

Questo è diverso se sei in un loop o qualcosa di simile, dato che nomi di variabili come `i`, `j`, `k` e così via sono comuni e accettabili.

Inoltre, il simbolo cancelletto (`#`) e il testo che segue nell'esempio sopra si chiamano commento. Potresti già conoscere i commenti, quindi facciamo una rapida panoramica e spieghiamo come funzionano.

In Python, i commenti iniziano con un simbolo cancelletto (`#`), e il linguaggio ignora tutto ciò che segue il simbolo `#` su quella riga:

```python
# This is a single-line comment
```

I commenti multilinea possono essere creati usando commenti singoli consecutivi:

```python
# This is a
# multi-line
# comment
```

Puoi usare i commenti per spiegare il tuo codice, lasciare promemoria per te stesso o chiarire perché esiste una riga. I commenti sono particolarmente utili quando stai imparando o lavori in team.

Tuttavia, non dovresti usare i commenti per spiegare cosa significano i nomi delle tue variabili. Invece, i nomi che scegli per le tue variabili dovrebbero essere descrittivi e comunicare a cosa servono, e seguire le altre regole di denominazione menzionate prima per evitare errori di sintassi.
