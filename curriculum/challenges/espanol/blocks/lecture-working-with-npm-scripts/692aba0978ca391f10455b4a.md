# --description--

Aprendamos sobre el registro de npm, donde puedes compartir tus paquetes de JavaScript con el mundo.

El **registro npm** es una base de datos de paquetes públicos y privados.

Los desarrolladores y las organizaciones pueden publicar paquetes en este registro para compartir sus herramientas con sus comunidades y miembros del equipo, y en muchos casos, con todos los demás desarrolladores de Node.js.

Los paquetes pueden ser públicos o privados. Los paquetes públicos pueden ser accedidos por cualquiera. Los paquetes privados solo pueden ser accedidos por usuarios específicos o miembros de una organización.

En el registro de npm, puedes compartir lo que has construido con el mundo con solo unos pocos clics. ¿Suena genial, verdad?

Veamos cómo puedes publicar un paquete paso a paso.

## Paso 1: Crea una cuenta

El primer paso es crear una cuenta gratuita en npm. Para crear tu cuenta:

* Go to [https://www.npmjs.com/signup](https://www.npmjs.com/signup).
    
* Rellena el formulario. Necesitarás ingresar tu nombre de usuario, correo electrónico y contraseña.
    

Ten en cuenta que, según la página de registro, "tu dirección de correo electrónico será añadida a los metadatos de los paquetes que publiques, por lo que puede ser vista públicamente."

## Paso 2: **Iniciar sesión**

Una vez que tengas una cuenta de npm, necesitarás iniciar sesión en tu terminal ejecutando este comando:

```bash
npm login
```

Luego necesitarás autenticarte con la información de tu cuenta.

## **Paso 3: Crea un** `package.json` file

Antes de publicar tu paquete, necesitarás crear un archivo `package.json` con este comando:

```bash
npm init
```

(Si tu proyecto ya tiene uno, puedes ir directamente al **paso 4**).

Necesitarás ingresar esta información para tu archivo `package.json`:

* **name**: Normalmente, este es el nombre de tu proyecto/directorio. Si planeas hacer público tu paquete, este nombre debe ser único, a menos que uses scopes (esto lo cubriremos en un momento).
    
* **version**: Debes comenzar con `"1.0.0"`, el valor predeterminado. Esto representa la versión inicial de tu paquete y puedes actualizarla a medida que haces cambios en el paquete.
    
* **main**: El punto de entrada de tu proyecto. Esto es `index.js` por defecto.
    
* **author**: Puedes escribir tu nombre o el alias que prefieras.
    
* **license**: Elige una licencia específica. Algunos ejemplos incluyen MIT e ISC.
    
* **description**: Un resumen breve de tu paquete.
    

Al elegir el nombre de tu paquete, asegúrate de que no esté ya tomado en el registro de npm. Para hacer esto, puedes ir a la siguiente URL, reemplazando `<your-package-name>` con el nombre de tu paquete:

```bash
https://www.npmjs.com/package/<your-package-name>
```

Si no hay un paquete en esa URL, el nombre del paquete está disponible y puedes usarlo.

Sin embargo, si el nombre ya está tomado y realmente quieres conservarlo, puedes usar lo que se conoce como un **scope** para publicarlo.

### **Ámbitos**

Cuando creas una cuenta de usuario u organización en el registro de npm, se te otorga un scope con tu nombre de usuario o el nombre de tu organización.

Los scopes son útiles para evitar conflictos de nombres porque puedes publicar tu paquete bajo tu scope, incluso si ya existe otro paquete con el mismo nombre.

El nombre de un paquete con ámbito comienza con el nombre del ámbito. Por ejemplo, en `package.json`, encontrarías este par clave-valor:

```bash
"name": "@<scope-name>/<your-package-name>"
```

Puedes hacer este cambio directamente en el archivo si lo necesitas, o puedes crear el archivo `package.json` con el scope desde el principio agregando `--scope=@<scope-name>` al comando `npm init`.

```bash
npm init --scope=@<scope-name>
```

El nombre del ámbito puede ser tu nombre de usuario si estás usando una cuenta personal, o el nombre de tu organización si estás usando una cuenta organizacional.

## Paso 4: Crea README.md

El README de tu paquete es un archivo markdown donde describes qué hace tu paquete y su uso previsto.

Algunos ejemplos de contenido importante que quizás quieras incluir son:

* Su propósito.
    
* Cómo instalarlo.
    
* Dependencias y consideraciones importantes.
    
* Cómo empezar.
    
* Fragmentos de código de ejemplo.
    
* Directrices para contribuciones (si aceptas contribuciones).
    
* Enlace a un archivo changelog donde rastreas los cambios que se han hecho en diferentes versiones.
    

La estructura del README es flexible, sin embargo. Puedes adaptarla para que se ajuste a tus necesidades.

## **Paso 5:** `.npmignore`

El archivo `.npmignore` le indica al registro de npm qué archivos incluir y qué archivos ignorar al publicar tu paquete. Es muy importante evitar compartir información sensible como claves privadas, contraseñas e información personal.

En este archivo, puedes escribir patrones para especificar qué archivos, directorios y tipos de archivos se ignoran. Sigue las mismas reglas de coincidencia de patrones que los archivos `.gitignore`.

Si has trabajado con Git, puede que pienses que este archivo suena muy similar a `.gitignore`, ¿verdad? Bueno, eso es porque lo es. Según la documentación de npm, "Si no hay un archivo `.npmignore`, pero sí hay un archivo `.gitignore`, entonces npm ignorará lo que coincida con el archivo `.gitignore`."

También debes saber que npm busca archivos `.npmignore` y `.gitignore` en todos los subdirectorios de tu paquete, no solo en el directorio raíz.

Por defecto, estos caminos se ignoran automáticamente, por lo que no necesitas agregarlos a `.npmignore`:

* `.*.swp`
    
* `._*`
    
* `.DS_Store`
    
* `.git`
    
* `.gitignore`
    
* `.hg`
    
* `.npmignore`
    
* `.npmrc`
    
* `.lock-wscript`
    
* `.svn`
    
* `.wafpickle-*`
    
* `config.gypi`
    
* `CVS`
    
* `npm-debug.log`
    

El directorio `node_modules` también se ignora, excepto las dependencias empaquetadas.

## **Paso 6: Publica tu paquete**

Antes de publicar tu paquete en el registro de npm, debes probarlo ejecutando este comando:

```bash
npm install <full-path-to-package-directory>
```

Una vez que estés listo para publicarlo, el comando dependerá del tipo de paquete: con ámbito o sin ámbito.

Si tu paquete no tiene ámbito, ejecuta este comando:

```bash
npm publish
```

Si tu paquete tiene un ámbito, se publicará como un paquete privado por defecto (si esta función está disponible en tu cuenta). Si quieres hacerlo público, necesitarás agregar `--access public` al comando.

```bash
npm publish --access public
```

Para ver tu paquete público, ve a esta URL:

```bash
https://npmjs.com/package/<your-package-name>
```

Deberías ver una indicación de la visibilidad de tu paquete (público o privado) debajo del nombre del paquete.

## **Paso 7 (Opcional): Instala Tu Paquete**

Ahora que ya estás listo, puedes instalar tu paquete en un proyecto Node.js con este comando:

```bash
npm install <your-package-name>
```

## Actualizando Tu Paquete

Si necesitas hacer cambios en tu paquete después de publicarlo, puedes actualizar la versión del paquete en `package.json`.

Esta es la convención estándar para el versionado semántico:

* **Primera versión:** `1.0.0`.
    
* **Lanzamiento de parche:** Incrementa el tercer dígito. Por ejemplo, actualizar de `1.0.0` a `1.0.1`.
    
* **Lanzamiento menor:** Incrementa el dígito del medio y reinicia el último dígito a cero. Por ejemplo, actualizar `1.0.0` a `1.1.0`.
    
* **Lanzamiento mayor:** Incrementa el primer dígito y reinicia el dígito del medio y el último a cero. Por ejemplo, actualizar `1.1.0` a `2.0.0`.
    
Buen trabajo. Ahora sabes cómo publicar tu paquete en el registro de npm y cómo actualizarlo para corregir errores y agregar nuevas funciones.
