## データベース正規化

リレーショナルデータベースのデータ冗長性を減らし、整合性を高めるための整理プロセスです。

利点は：

- 重複データを最小化し、ストレージ節約と不整合の減少を実現。
- 主キーと外部キーを使ってデータ整合性を強制。
- データベースの保守と理解が容易になる。

### 正規形

- **1NF（第一正規形）**
  - 各セルは単一（原子）値を持つ。
  - 各レコードは一意（主キーで強制）。
  - 行や列の順序は関係ない。
  - 例：複数の電話番号を`students`テーブルから別の`student_phones`テーブルに分割。

- **2NF（第二正規形）**
  - 1NFの要件を満たす。
  - **部分関数従属**なし：非キー属性は複合主キー全体に依存する必要がある。
  - 例：`orders`テーブルを`order_header`と`order_items`に分割し、属性がキーの一部にのみ依存するのを防ぐ。

- **3NF（第三正規形）**
  - 2NFの要件を満たす。
  - **推移的関数従属**なし：非キー属性は他の非キー属性に依存しない。
  - 例：`city_postal_code`を注文ごとに保存するのではなく、`cities`テーブルに移動。

- **BCNF（Boyce-Codd正規形）**
  - 3NFの要件を満たす。
  - すべての決定子（関数従属の左辺）はスーパーキーでなければならない。

**ヒント**：ほとんどの設計では3NFを目指すと、整合性とパフォーマンスのバランスが良いです。

## 主要なSQL概念

- SQL はリレーショナルデータベースと通信するための構造化クエリ言語です。
- **基本コマンド** → `SELECT`、`INSERT`、`UPDATE`、`DELETE`、`CREATE TABLE`、`ALTER TABLE` など。
- **Joins** → 複数のテーブルからデータを結合します（`INNER JOIN`、`LEFT JOIN`、`RIGHT JOIN`、`FULL JOIN`）。

## BashでSQLコマンドを実行

PostgreSQLの`psql`コマンドラインクライアントや他のデータベースツールを使い、コマンドラインから直接SQLを実行できます。

例：PostgreSQLでSQLファイルを実行する方法

```bash
psql -U username -d database_name -c "SELECT * FROM students;"
```

MySQLコマンドを直接実行する例

```bash
mysql -u username -p database_name -e "SELECT * FROM students;"
```

### ファイルからSQLを実行

```bash
# PostgreSQL
psql -U username -d database_name -f script.sql

# MySQL
mysql -u username -p database_name < script.sql
```

### BashスクリプトにSQLを埋め込む

```bash
#!/bin/bash
DB_USER="school_admin"
DB_NAME="school"

# Insert student data
psql -U "$DB_USER" -d "$DB_NAME" -c \
"INSERT INTO students (name, age, major) VALUES ('Alice', 20, 'CS');"
```

### SQLで変数を使う

```bash
#!/bin/bash
DB_USER="school_admin"
DB_NAME="school"
STUDENT_NAME="Bob"
AGE=21

psql -U "$DB_USER" -d "$DB_NAME" -c \
"INSERT INTO students (name, age) VALUES ('$STUDENT_NAME', $AGE);"
```

**ヒント**：SQLインジェクションを防ぐために変数は必ずサニタイズしてください。

## BashでSQLクエリ結果を取得・利用

`psql`経由でSQLクエリを実行すると、Bashスクリプト内で返された値を**取得**し**処理**できます。

### 単一値の取得

```bash
#!/bin/bash
DB_USER="school_admin"
DB_NAME="school"

# Get total student count
STUDENT_COUNT=$(psql -U "$DB_USER" -d "$DB_NAME" -t -A -c \
"SELECT COUNT(*) FROM students;")

echo "Total students: $STUDENT_COUNT"
```

出力 → 42

### 複数列の取得

```bash
#!/bin/bash
DB_USER="school_admin"
DB_NAME="school"

# Get top 3 students' names and ages
RESULTS=$(psql -U "$DB_USER" -d "$DB_NAME" -t -A -F"," -c \
"SELECT name, age FROM students LIMIT 3;")

echo "Top 3 students:"
echo "$RESULTS"
```

出力

```bash
Alice,20
Bob,21
Charlie,22
```

### クエリ結果のループ処理

```bash
#!/bin/bash
DB_USER="school_admin"
DB_NAME="school"

# Get student names and majors
psql -U "$DB_USER" -d "$DB_NAME" -t -A -F"," -c \
"SELECT name, major FROM students;" | while IFS="," read -r name major
do
    echo "Student: $name | Major: $major"
done
```

出力の形

```bash
Student: Alice | Major: CS
Student: Bob   | Major: Math
Student: Carol | Major: Physics
```

## SQLインジェクション

ウェブのセキュリティ脆弱性で、攻撃者が悪意あるSQLコードを入力フィールドに挿入し、データベースを操作します。

リスク例：

- 認証を回避。
- 機密データの窃取。
- レコードの改ざんや削除。

SQLインジェクション攻撃の例：

```sql
SELECT * FROM users WHERE username = '' OR '1'='1' --' AND password = 'anything';
```

このクエリは、条件 `OR '1'='1'` が常に真であるため、すべてのユーザーを返します。これにより攻撃者はログインチェックを回避できます。

### SQLインジェクションの防止

1. **プリペアドステートメントを使う**：SQLコードとデータを分離し、インジェクションを防ぎます。例（Node.jsのpg）：

    ```js
    client.query('SELECT * FROM users WHERE username = $1 AND password = $2', [username, password]);
    ```

2. **入力バリデーション**：すべてのユーザー入力をサニタイズし、期待される形式か検証します。

3. **最小権限の原則**：アプリケーションに必要最低限の権限を持つデータベースアカウントを使います。

**注意**：アプリケーションアカウントに管理者権限を絶対に与えないでください。

## N+1問題

アプリケーションがアイテムリスト（N）を取得するクエリを1回実行し、その後各アイテムに関連データを取得するために追加でN回クエリを実行し、合計N+1回のクエリになる問題です。

**問題点**

- 各クエリがネットワークと処理のオーバーヘッドを増やします。
- 複数の小さなクエリは、1つの最適化されたクエリより遅くなります。

### N+1パターンの例

```sql
-- 1: Get list of orders
SELECT * FROM orders LIMIT 50;

-- N: For each order, get customer
SELECT * FROM customers WHERE customer_id = ...;
```

**解決策**：`JOINs`や他のセットベースの操作を使います。

```sql
SELECT 
  orders.order_id,
  orders.product,
  orders.quantity,
  customers.customer_id,
  customers.name,
  customers.email,
  customers.address
FROM orders
JOIN customers 
  ON orders.customer_id = customers.customer_id
WHERE orders.order_id IN (SELECT order_id FROM orders LIMIT 50);
```

関連データを1つのクエリにまとめる機会を常に探しましょう。
