# --description--

ミドルウェアは特定のルートに `app.METHOD(path, middlewareFunction)` を使ってマウントできます。ミドルウェアはルート定義内で連結することも可能です。

次の例を見てください。

```js
app.get('/user', function(req, res, next) {
  req.user = getTheUserSync();  // Hypothetical synchronous operation
  next();
}, function(req, res) {
  res.send(req.user);
});
```

この方法はサーバーの処理をより小さな単位に分割するのに役立ちます。これによりアプリの構造が改善され、コードを異なる場所で再利用できるようになります。この方法はデータのバリデーションを行う場合にも使えます。ミドルウェアスタックの各ポイントで現在の連結の実行を停止し、エラー処理用に特別に設計された関数に制御を渡すことができます。または、次の一致するルートに制御を渡して特別なケースを処理することも可能です。これについては高度な Express セクションで説明します。

# --instructions--

ルート `app.get('/now', ...)` でミドルウェア関数と最終ハンドラを連結してください。ミドルウェア関数内で現在の時刻を `req.time` キーでリクエストオブジェクトに追加してください。`new Date().toString()` を使えます。ハンドラでは、`{time: req.time}` の構造を持つ JSON オブジェクトで応答してください。

**注意:** ミドルウェアを連結しないとテストは通りません。関数を他の場所にマウントすると、出力結果が正しくてもテストは失敗します。

# --hints--

/now エンドポイントにはミドルウェアがマウントされている必要があります

```js
  const response = await fetch(code + '/_api/chain-middleware-time');
  if (!response.ok) {
    throw new Error(await response.text());
  }
  const data = await response.json();
  assert.equal(
    data.stackLength,
    2,
    '"/now" route has no mounted middleware'
  );
```

`/now` エンドポイントは現在の時刻を返す必要があります。

```js
  const response = await fetch(code + '/_api/chain-middleware-time');
  if (!response.ok) {
    throw new Error(await response.text());
  }
  const data = await response.json();
  var now = new Date();
  assert.isAtMost(
    Math.abs(new Date(data.time) - now),
    20000,
    'the returned time is not between +- 20 secs from now'
  );
```
