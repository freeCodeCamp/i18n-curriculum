# --interactive--

في JavaScript، الدوال وطرق الكائنات هما طريقتان لتغليف شفرة قابلة لإعادة الاستخدام، لكن هناك اختلافات رئيسية في كيفية تعريفها واستخدامها والسياق الذي تعمل فيه. فهم هذه الاختلافات ضروري لكتابة شفرة JavaScript فعالة ومنظمة.

كما تعلمت في الوحدات السابقة، الدوال هي كتل شفرة قابلة لإعادة الاستخدام تؤدي مهمة محددة:

:::interactive_editor

```js
function greet(name) {
    return "Hello, " + name + "!";
}
console.log(greet("Alice")); // "Hello, Alice!"
```

:::

أما طرق الكائنات، فهي دوال مرتبطة بكائن معين. تُعرف كخصائص للكائن ويمكنها الوصول إلى بيانات الكائن والتعامل معها. إليك مثالًا على كائن يحتوي على طريقة:

:::interactive_editor

```js
const person = {
    name: "Bob",
    age: 30,
    sayHello: function() {
        return "Hello, my name is " + this.name;
    }
};

console.log(person.sayHello()); // "Hello, my name is Bob"
```

:::

في هذا المثال، `sayHello` هي طريقة للكائن `person`. تسمح الكلمة المفتاحية `this` لطريقة `sayHello` بالوصول إلى خصائص الكائن المسمى `person`. ستتعلم المزيد عن الكلمة المفتاحية `this` في الدروس القادمة.

الفرق بين الدوال والطرق يكمن في كيفية استدعائها. تُستدعى الدوال باسمها، بينما تُستدعى الطرق باستخدام ترميز النقطة على الكائن الذي تنتمي إليه. على سبيل المثال، نستدعي الدالة `greet` كـ `greet("Alice")`، لكن نستدعي الطريقة `sayHello` كـ `person.sayHello()`.

فرق مهم آخر هو السياق الذي تعمل فيه. الدوال العادية لها نطاق خاص بها، لكنها لا تملك مرجعًا مدمجًا لأي كائن معين. أما الطرق، فهي مرتبطة بكائنها ويمكنها الوصول إلى خصائصه وطرق أخرى باستخدام الكلمة المفتاحية `this`.

نقطة أساسية يجب ملاحظتها هي أن الطرق تساعد في تنظيم الشفرة ضمن كائنات منطقية، بينما تُستخدم الدوال لكتابة شفرة عامة وقابلة لإعادة الاستخدام.

فهم الفرق بين الدوال وطرق الكائنات مهم مع تقدمك في رحلة تعلم JavaScript. رغم تشابهها في البداية، فإن معرفة متى تستخدم كلًا منها يساعدك على كتابة شفرة أكثر تنظيمًا وكفاءة وجودة.
