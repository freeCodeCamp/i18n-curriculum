# --description--

Avrai notato nella sfida precedente che, oltre all'attributo `style` impostato su un oggetto JavaScript, ci sono diverse altre differenze di sintassi rispetto agli stili in linea HTML. Per prima cosa, i nomi di alcune proprietà CSS usano il camel case. Per esempio, nella sfida precedente la dimensione del carattere tipografico è stata impostata con `fontSize` invece di `font-size`. Le parole scritte con trattino come `font-size` non sono una sintassi valida per le proprietà degli oggetti JavaScript, quindi React usa il camel case. Come regola, tutte le proprietà di stile scritte con trattino si scrivono in camel case in JSX.

Tutte le unità di misura delle lunghezze dei valori delle proprietà (come `height`, `width` e `fontSize`) si intendono in `px` a meno che non sia specificato diversamente. Se vuoi usare `em`, per esempio, devi racchiudere il valore e le unità tra virgolette, come `{fontSize: "4em"}`. A parte i valori di lunghezza che di default sono in `px`, tutti gli altri valori delle proprietà devono essere racchiusi tra virgolette.

# --instructions--

Se hai un insieme ampio di stili, puoi assegnare un oggetto stile `object` a una costante per mantenere il codice organizzato. Dichiara la tua costante degli stili come variabile globale in cima al file. Inizializza la costante `styles` e assegnale un `object` con tre proprietà di stile e i loro valori. Dai al `div` un colore `purple`, una dimensione del carattere tipografico `40` e un bordo `2px solid purple`. Poi imposta l'attributo `style` uguale alla costante `styles`.
