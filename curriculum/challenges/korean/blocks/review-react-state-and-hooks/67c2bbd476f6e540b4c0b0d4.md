## React에서 이벤트 다루기

- **합성 이벤트 시스템**: 이것은 React가 이벤트를 처리하는 방식입니다. `click`, `keydown`, `submit` 같은 네이티브 이벤트를 감싸는 래퍼 역할을 합니다. React의 이벤트 핸들러는 카멜 케이스 명명 관례를 사용합니다. (예: `onClick`, `onSubmit` 등)

다음은 React에서 `onClick` 요소에 `button` 속성을 사용하는 예시입니다:

```jsx
function handleClick() {
  console.log("Button clicked!");
}

<button onClick={handleClick}>Click Me</button>;
```

React에서 이벤트 핸들러 함수는 보통 `handle` 접두사로 시작하여 이벤트를 처리함을 나타냅니다. 예를 들어 `handleClick` 또는 `handleSubmit` 같은 이름입니다.

사용자 동작이 이벤트를 발생시키면 React는 합성 이벤트 객체를 핸들러에 전달합니다. 이 객체는 일반 JavaScript의 네이티브 Event 객체처럼 동작하며 `type`, `target`, `currentTarget` 같은 속성을 제공합니다.

예를 들어 `onSubmit` 이벤트 중 브라우저 새로 고침 같은 기본 동작을 막으려면 `preventDefault()` 메서드를 호출하세요:

```jsx
function handleSubmit(event) {
  event.preventDefault();
  console.log("Form submitted!");
}

<form onSubmit={handleSubmit}>
  <input type="text" />
  <button>Submit</button>
</form>;
```

또한 핸들러 함수를 화살표 함수로 감쌀 수도 있습니다:

```jsx
function handleDelete(id) {
  console.log("Deleting item:", id);
}

<button onClick={() => handleDelete(1)}>Delete Item</button>;
```

## 상태와 `useState` 훅스 다루기

- **상태 정의**: React에서 상태는 컴포넌트의 데이터를 담는 객체입니다. 상태가 업데이트되면 컴포넌트가 다시 렌더링됩니다. React는 상태를 불변으로 취급하므로 직접 수정하지 않아야 합니다.  
- **`useState()` 훅스**: `useState` 훅스는 함수형 컴포넌트에서 상태 변수를 선언할 수 있게 해주는 함수입니다. 기본 구문은 다음과 같습니다:

```js
const [stateVariable, setStateFunction] = useState(initialValue);
```

상태 변수에는 다음이 포함됩니다:

- `stateVariable`는 현재 상태 값을 담습니다
- `setStateFunction` (설정 함수)는 상태 변수를 업데이트합니다
- `initialValue`는 초기 상태를 설정합니다

다음은 `Counter` 컴포넌트의 완전한 예시입니다:

```jsx
import { useState } from "react";

function Counter() {
  const [count, setCount] = useState(0);

  return (
    <div>
      <h2>{count}</h2>

      <button onClick={() => setCount(count - 1)}>Decrement</button>
      <button onClick={() => setCount(count + 1)}>Increment</button>
    </div>
  );
}

export default Counter;
```

## 렌더링과 React 컴포넌트

- **정의**: React에서 렌더링은 컴포넌트가 사용자 인터페이스(UI), 보통 브라우저에 나타나는 과정입니다. 렌더링 과정은 트리거, 렌더, 커밋 세 단계로 구성됩니다.

트리거 단계는 React가 변경 사항을 감지하여 UI를 업데이트해야 할 수 있음을 인지할 때 발생합니다. 이 변경은 보통 상태나 프로프스 업데이트 때문입니다.

트리거가 발생하면 React는 렌더 단계에 들어갑니다. 여기서 React는 컴포넌트를 재평가하고 무엇을 표시할지 결정합니다. 이를 위해 React는 가벼운 복사본인 가상 DOM을 사용합니다. 가상 DOM 덕분에 React는 컴포넌트에서 변경할 부분을 빠르게 확인할 수 있습니다.

커밋 단계는 React가 가상 DOM에서 준비한 변경 사항을 실제 DOM에 적용하는 단계입니다. 즉, 화면에서 최종 결과를 볼 수 있는 단계입니다.

## 상태에서 객체와 배열 업데이트하기

- **상태에서 객체 업데이트**: 상태의 객체를 업데이트해야 할 때는 새 객체를 만들거나 기존 객체를 복사한 후 그 새 객체로 상태를 설정해야 합니다. 상태에 넣는 모든 객체는 읽기 전용으로 간주해야 합니다. 다음은 사용자의 이름, 나이, 도시를 설정하는 예시입니다. `handleChange` 함수는 사용자 정보를 업데이트하는 데 사용됩니다:

```jsx
import { useState } from "react";

function Profile() {
  const [user, setUser] = useState({ name: "John Doe", age: 31, city: "LA" });

  const handleChange = (e) => {
    const { name, value } = e.target;

    setUser((prevUser) => ({...prevUser, [name]: value}));
  };

  return (
    <div>
      <h1>User Profile</h1>
      <p>Name: {user.name}</p>
      <p>Age: {user.age}</p>
      <p>City: {user.city}</p>

      <h2>Update User Age </h2>
      <input type="number" name="age" value={user.age} onChange={handleChange} />

      <h2>Update User Name </h2>
      <input type="text" name="name" value={user.name} onChange={handleChange} />

      <h2>Update User City </h2>
      <input type="text" name="city" value={user.city} onChange={handleChange} />
    </div>
  );
}

export default Profile;
```

- **상태에서 배열 업데이트**: 상태의 배열을 업데이트할 때는 `push()`이나 `pop()` 같은 메서드로 배열을 직접 수정하지 않는 것이 중요합니다. 대신 상태를 업데이트할 때 새 배열을 만들어야 합니다:

```js
const addItem = () => {
  const newItem = {
    id: items.length + 1,
    name: `Item ${items.length + 1}`,
  };

  // Creates a new array
  setItems((prevItems) => [...prevItems, newItem]);
};
```

배열에서 항목을 제거하려면 `filter()` 메서드를 사용하세요. 이 메서드는 제거할 항목을 필터링한 새 배열을 반환합니다:

```js
const removeItem = (id) => {
  setItems((prevItems) => prevItems.filter((item) => item.id !== id));
};
```

## Ref를 사용해 값 참조하기

- **`ref` 속성**: React에서 DOM 노드에 접근하려면 `ref` 속성을 사용하세요. 다음은 `ref`를 사용해 `input` 요소에 포커스를 맞추는 예시입니다. `current` 속성은 해당 `ref`의 현재 값을 접근하는 데 사용됩니다:

```jsx
import { useRef } from "react";

const Focus = () => {
  const inputRef = useRef(null);

  const handleFocus = () => {
    if (inputRef.current) {
      inputRef.current.focus();
    }
  };

  return (
    <div>
      <input ref={inputRef} type="text" placeholder="Enter text" />
      <button onClick={handleFocus}>Focus Input</button>
    </div>
  );
};

export default Focus;
```

## `useEffect` 훅스 다루기

- **`useEffect()` 훅스**: React에서 이펙트는 컴포넌트 렌더링 과정 외부에서 발생하는 모든 작업입니다. 즉, React가 UI 렌더링의 일부로 직접 처리하지 않는 모든 작업입니다. 일반적인 예로는 데이터 가져오기, 브라우저 탭 제목 업데이트, 로컬 스토리지 읽기/쓰기, 사용자 위치 가져오기 등이 있습니다. 이런 작업은 외부 세계와 상호작용하며 부수 효과라고 합니다. React는 이러한 부수 효과를 처리할 수 있도록 `useEffect` 훅스를 제공합니다. `useEffect`는 컴포넌트가 렌더링되거나 업데이트된 후 함수를 실행할 수 있게 합니다.

```js
import { useEffect } from "react";

useEffect(() => {
 // Your side effect logic (usually a function) goes here
}, [dependencies]);
```

이펙트 함수는 컴포넌트가 렌더링된 후 실행되며, 선택적 `dependencies` 인수는 이펙트 실행 시점을 제어합니다.

`dependencies`은 "반응 값"(상태, 프로프스, 함수, 변수 등)의 배열일 수도 있고, 빈 배열이거나 완전히 생략할 수도 있습니다. 다음은 이 옵션들이 `useEffect` 작동 방식을 제어하는 방법입니다:

- `dependencies`이 하나 이상의 반응 값을 포함하는 배열이면, 해당 값이 변경될 때마다 이펙트가 실행됩니다.

- `dependencies`이 빈 배열이면, `useEffect`는 컴포넌트가 처음 렌더링될 때 한 번만 실행됩니다.

- `dependencies`를 생략하면, 컴포넌트가 렌더링되거나 업데이트될 때마다 이펙트가 실행됩니다.

## 커스텀 훅스 만드는 방법

- **커스텀 훅스**: 커스텀 훅스는 데이터 가져오기, 상태 관리, 토글, 온라인 상태 추적 같은 부수 효과 등 컴포넌트에서 재사용 가능한 로직을 추출할 수 있게 합니다. React의 모든 내장 훅스는 `use`로 시작하므로, 커스텀 훅스도 같은 관례를 따라야 합니다.

다음은 `useDebounce` 훅스를 만드는 예시입니다:

```js
function useDebounce(value, delay) {
  const [debouncedValue, setDebouncedValue] = useState(value);

  useEffect(() => {
    const handler = setTimeout(() => {
      setDebouncedValue(value);
    }, delay);

    return () => {
      clearTimeout(handler);
    };
  }, [value, delay]);

  return debouncedValue;
}

export { useDebounce };
```
