# --description--

在 JavaScript 中，模組是一個自包含的程式碼單元，封裝相關的函式（程式）、類別或變數。

將模組視為你應用程式的建造區塊，就像書中的章節一樣。每個模組專注於特定的功能，使你的程式碼更有組織性、易於維護且可重複使用。模組有助於避免命名衝突，並允許你將應用程式結構化為獨立且相互連結的部分。

JavaScript 中的模組概念隨著時間演進，但最廣泛使用且被支援的方式是 ES6（ECMAScript 2015）模組系統。此系統提供了一種標準化的方法，在不同的 JavaScript 環境中定義和使用模組。

要創建一個模組，你需要將你的 JavaScript 程式碼寫在一個獨立的檔案中。任何你想讓應用程式其他部分使用的變數、函式（程式）或類別，都需要明確地匯出。你可以使用 `export` 關鍵字來做到這一點。

例如，假設你有一個名為 `math.js` 的檔案，裡面有一些數學函式（程式）：

```js
export function add(a, b) {
    return a + b;
}

export function subtract(a, b) {
    return a - b;
}

const PI = 3.14159;
export { PI };
```

在此範例中，我們正在匯出兩個函式（`add` 和 `subtract`）以及一個常數（`PI`）。你可以從單一模組匯出任意多的項目。

若要在應用程式的其他零件中使用這些匯出的項目，必須將它們匯入。這是使用 `import` 關鍵字完成的。

假設你想在名為 `app.js` 的檔案中使用這些數學函式。

```js
import { add, subtract, PI } from './math.js';

console.log(add(5, 3));        // Outputs: 8
console.log(subtract(10, 4));  // Outputs: 6
console.log(PI);               // Outputs: 3.14159
```

在這裡，我們從 `math.js` 模組匯入我們需要的特定函式（程式）和常數。`'./math.js'` 這部分告訴 JavaScript 如何相對於目前的檔案找到該模組檔案。

有時候，你可能想要匯入一個模組匯出的所有內容。你可以使用星號（`*`）語法來做到這一點：

```js
import * as Math from './math.js';

console.log(Math.add(5, 3));        // Outputs: 8
console.log(Math.subtract(10, 4));  // Outputs: 6
console.log(Math.PI);               // Outputs: 3.14159
```

在這種情況下，所有從 `math.js` 的匯出都被匯入為名為 `Math` 的物件屬性。

另一個常見的樣式是在模組中有一個預設匯出。當模組主要匯出單一函式（程式）時，通常會使用這種方式。每個模組只能有一個預設匯出。

這就是它的樣子：

```js
// In math.js
export default function multiply(a, b) {
    return a * b;
}

// In app.js
import multiply from './math.js';

console.log(multiply(4, 5));  // Outputs: 20
```

請注意，當匯入 `default` 匯出時，你不需要使用大括號，且你可以將匯入命名為任何你想要的名稱。

重要的是要注意，若要在瀏覽器中使用 ES6 模組，你需要在你的 `script` 頁籤中將 `type` 指定為 `module`：

```html
<script type="module" src="app.js"></script>
```

模組提供了一種強大的方式來組織和結構你的 JavaScript 程式碼。它們允許你將應用程式拆分成較小且易於管理的部分，促進程式碼重用，並有助於維持清晰的關注點分離。當你建置較大型的應用程式時，你會發現模組成為你 JavaScript 開發套件中不可或缺的工具。
