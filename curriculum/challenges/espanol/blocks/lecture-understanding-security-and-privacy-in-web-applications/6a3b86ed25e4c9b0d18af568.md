# --description--

El archivo `.env` y la biblioteca `dotenv` son herramientas usadas en el desarrollo de software moderno para separar datos sensibles de configuración del código de la aplicación.

Un archivo `.env` es un archivo de texto plano ubicado en la raíz del directorio de un proyecto que almacena configuraciones y secretos como pares clave-valor.

En lugar de codificar directamente claves API, contraseñas de base de datos o números de puerto en tu código fuente y arriesgarte a exponerlos si el código se sube a un repositorio público como GitHub, los desarrolladores los guardan localmente en este archivo.

La estructura de un archivo `.env` es sencilla:

```bash
PORT=3000
DB_HOST=localhost
API_KEY=xyz123secretkey
```

Como este archivo contiene claves altamente sensibles, es práctica estándar agregar `.env` al archivo `.gitignore` del proyecto para que nunca sea rastreado por Git.

El paquete `dotenv` es un módulo ligero que conecta el archivo `.env` con tu aplicación.

Cuando tu programa inicia, `dotenv` lee el archivo `.env`, interpreta las claves y las inyecta en el entorno de ejecución. En una aplicación Node.js, por ejemplo, inicializas el paquete al inicio del archivo de entrada. Una vez cargadas, las variables son accesibles globalmente a través del objeto `process.env`.

```javascript
const express = require('express');
const app = express();

// initialize and configure dotenv
require('dotenv').config();

// access the variables securely from the environment
const port = process.env.PORT || 5000;
const apiKey = process.env.API_KEY;

app.get('/', (req, res) => {
  res.send(`Connected successfully using API Key: ${apiKey}`);
});

app.listen(port, () => {
  console.log(`Server running on port ${port}`);
});
```

Usar esta configuración ofrece dos grandes beneficios. Primero, mejora la seguridad al mantener las credenciales privadas fuera del código fuente. Segundo, proporciona flexibilidad al permitir que la misma base de código se comporte diferente en entornos de desarrollo, prueba y producción simplemente cambiando el archivo `.env` local en cada respectivo servidor de hosting.
