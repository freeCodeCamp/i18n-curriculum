# --description--

讓我們了解 `submit` 事件如何與 HTML 表單一起運作。首先，我們需要了解如何提交表單。表單有三種提交方式。

第一種情況是當使用者點擊表單中 `type` 屬性設定為 `submit` 的按鈕時。

第二種情況是當使用者在表單中任何可編輯的輸入欄位按下 Enter 鍵時。

第三種是透過 JavaScript 呼叫表單元素的 `requestSubmit()` 或 `submit()` 方法。

但是當 `form` 被提交時會發生什麼？會發生幾件事，而 `form` 的行為取決於其屬性。

我們需要先查看的第一個屬性是 `action` 屬性。`action` 屬性應該包含 URL 或當前網域的相對路徑。此值決定表單嘗試將資料傳送到哪裡。如果你沒有設定 `action` 屬性，表單將會將資料傳送到當前頁面的 URL。以下是一個將 `action` 屬性設定為特定 URL 的表單範例：

```html
<form action="https://freecodecamp.org">
  <input type="number" id="input" placeholder="Enter a number" name="number" />
  <button type="submit">Submit</button>
</form>
```

當此表單提交時，它會將資料傳送到 freeCodeCamp 的首頁，該首頁可能不會有任何反應。

這是另一個表單，它會提交到相對路徑：

```html
<form action="/data">
  <input type="number" id="input" placeholder="Enter a number" name="number" />
  <button type="submit">Submit</button>
</form>
```

這個會提交到目前網域的 `/data` 路徑。在我們的例子中，它會提交到 `http://127.0.0.1:5500/data`。

第二個用來控制提交行為的屬性是 `method` 屬性。此屬性接受標準的 HTTP 方法，例如 `GET` 或 `POST`，並在向 `action` URL 發出請求時使用該方法。

HTTP 代表超文字傳輸協定，它用於在網路上傳輸資料。

HTTP 方法用來定義可對資源執行的動作，例如 `GET`、`POST`、`PUT`、`DELETE` 等。你將在未來的課程中學習更多關於這些方法的內容。

當未設定行為時，表單將預設為 `GET` 請求。`GET` 請求用於從指定資源擷取資料而不對其進行任何更改，且參數通常以查詢字串的形式附加在 URL 後。

表單中的資料將會以 `name=value` 對組的形式進行 URL 編碼，並作為查詢參數附加到 `action` URL。舉例來說，若從我們先前的範例中提交表單，且 `input` 欄位的值為 `3342`，該表單將嘗試以 `GET` 方法取得 URL `http://127.0.0.1:5500/data?number=3342`。

這非常適合像搜尋表單這類你的使用者正在查詢資料的情況。但如果你想讓你的使用者提交新的資料，例如在註冊表單中，慣用的行為是使用 `POST` 方法。`POST` 方法用於將資料傳送到伺服器以創建或更新資源。

讓我們將 `method` 屬性設定為 `POST` 的值：

```xml
<form action="/data" method="POST">
  <input type="number" id="input" placeholder="Enter a number" name="number" />
  <button type="submit">Submit</button>
</form>
```

當你發送 `POST` 請求時，可以包含一個主體，其中包含你請求的資料。因此，與 `GET` 不同，資料不會附加在 URL 後面，我們的表單會將 `POST` 請求發送到 `http://127.0.0.1:5500/data`。資料則可以在請求的主體中找到，仍然是以 URL 編碼的表單資料形式。

URL 編碼的表單資料是指將表單資料轉換成鍵值對組成的字串，例如 `name=John+Doe&email=john%40example.com`，其中特殊字元會被取代成編碼版本，以安全地透過網路傳送資料。你將在未來的課程中學習更多相關內容。

但是也許你不想將資料作為 URL 編碼的表單有效負載傳送？

`form` 元素接受一個 `enctype` 屬性，該屬性表述用於資料的編碼型別。

此屬性僅接受三個值：`application/x-www-form-urlencoded`（預設值，將資料作為 URL 編碼的表單主體傳送）、`text/plain`（以純文字形式傳送資料，使用以換行符號分隔的 `name=value` 對組），或 `multipart/form-data`，專門用於處理包含檔案上傳的表單。

在本課程中，你已經學會了表單提交的基礎知識。在未來的課程中，我們將更深入探討如何操作表單以及它們如何與伺服端互動。
