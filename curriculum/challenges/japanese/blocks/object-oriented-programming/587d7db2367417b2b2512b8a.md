# --description--

前のチャレンジでは、`bird` に公開プロパティ `name` がありました。これは `bird` の定義外からアクセスおよび変更できるため、公開されていると考えられます。

```js
bird.name = "Duffy";
```

したがって、コードのどの部分からでも `bird` の名前を簡単に任意の値に変更できます。パスワードや銀行口座のようなものがコードベースのどの部分からでも簡単に変更できてしまうことを想像してください。多くの問題を引き起こす可能性があります。

この公開プロパティをプライベートにする最も簡単な方法は、コンストラクタ関数内に変数を作成することです。これにより、その変数のスコープがコンストラクタ関数内に限定され、グローバルには利用できなくなります。こうすることで、その変数はコンストラクタ関数内のメソッドからのみアクセスおよび変更できます。

```js
function Bird() {
  let hatchedEgg = 10;

  this.getHatchedEggCount = function() { 
    return hatchedEgg;
  };
}
let ducky = new Bird();
ducky.getHatchedEggCount();
```

ここで `getHatchedEggCount` は特権メソッドです。なぜならプライベート変数 `hatchedEgg` にアクセスできるからです。これは `hatchedEgg` が `getHatchedEggCount` と同じコンテキストで宣言されているため可能です。JavaScript では、関数は常に作成されたコンテキストにアクセスできます。これを `closure` と呼びます。
