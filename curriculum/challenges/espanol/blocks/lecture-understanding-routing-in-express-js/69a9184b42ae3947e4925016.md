# --description--

En Express, `express.Router()` es una función poderosa que te ayuda a organizar tus rutas en piezas más pequeñas y modulares. Esto es especialmente útil a medida que tu aplicación crece.

En lugar de definir todas tus rutas en un solo archivo grande, puedes dividirlas en archivos separados usando routers. Esto mantiene tu código más fácil de leer, mantener y escalar.

El método `express.Router()` crea un nuevo objeto router.

Un router funciona como una mini aplicación Express. Puede manejar sus propias rutas y middleware, pero no es una aplicación Express completa. Una vez que defines rutas en un router, puedes montarlo en tu aplicación principal usando `app.use()`.

En otras palabras, un router te permite agrupar rutas relacionadas y luego integrarlas en tu aplicación principal.

Usar routers tiene varias ventajas:

**Separación de responsabilidades**

Puedes definir rutas en archivos separados, lo que mejora la organización y legibilidad de tu aplicación.

**Reutilización de código**  
Los routers pueden reutilizarse en diferentes partes de una aplicación o incluso en varios proyectos.

**Archivo principal más limpio**  
En lugar de llenar tu archivo principal `app.js` o `server.js` con muchas definiciones de rutas, puedes moverlas a routers y mantener tu archivo principal enfocado en la configuración y el arranque.

Veamos un ejemplo sencillo donde usamos `express.Router()` para definir rutas para la ruta `/users`.

Primero, aquí está el archivo `app.js`:

```javascript
const express = require("express");
const app = express();
const port = 3000;

// Importing the user routes
const userRoutes = require("./userRoutes");

// Using the userRoutes router for /users path
app.use("/users", userRoutes);

app.listen(port, () => {
  console.log(`Server running at http://localhost:${port}`);
});
```

Y aquí está `userRoutes.js`:

```javascript
const express = require("express");
const router = express.Router();

// GET request for /users
router.get("/", (req, res) => {
  res.send("List of users");
});

// POST request for /users
router.post("/", (req, res) => {
  res.send("Creating a new user");
});

// GET request for /users/:id
router.get("/:id", (req, res) => {
  const userId = req.params.id;
  res.send(`User details for user with id: ${userId}`);
});

module.exports = router;
```

Ahora veamos más de cerca cómo funciona todo.

En `app.js`, importamos `express` y creamos una instancia de la aplicación Express asignándola a una variable llamada `app`.

Luego, importamos el router `userRoutes` desde `userRoutes.js` y usamos `app.use("/users", userRoutes)` para montar el router. Esto le indica a Express que, si una solicitud comienza con `/users`, la pase al router `userRoutes`.

Después, en `userRoutes.js`, creamos un nuevo router usando `express.Router()` y definimos rutas usando `router.get()`, `router.post()` y `router.get("/:id")`.

Ten en cuenta que las rutas definidas aquí son relativas a `/users` porque ahí es donde se monta el router. Así que:

*   `router.get("/")` → `GET /users`: responde con una lista de usuarios
    
*   `router.post("/")` → `POST /users`: responde con un mensaje sobre la creación de un nuevo usuario
    
*   `router.get("/:id")` → `GET /users/:id`: responde con detalles del usuario para un ID dado
    

Usar `express.Router()` mantiene tu archivo principal limpio y enfocado en la configuración. Todas las rutas relacionadas con usuarios están agrupadas en `userRoutes.js`.

A medida que tu aplicación crece, puedes crear routers adicionales para otras secciones, como `/products`, `/orders` o `/auth`.

Cada sección se mantiene organizada en su propio archivo, haciendo que la aplicación sea más fácil de mantener y escalar.
