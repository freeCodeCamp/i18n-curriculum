# --description--

Router-Level-Middleware ähnelt der Middleware auf Anwendungsebene, ist jedoch an eine Instanz von `express.Router()` gebunden und nicht an die Haupt-App-Instanz. Sie kann verwendet werden, um Ihre Express-Anwendung zu modularisieren, indem isolierte Routinenmodule erstellt werden.

Router-Level-Middleware funktioniert genauso wie Middleware auf Anwendungsebene, wird jedoch auf bestimmte Router-Instanzen angewendet. Dadurch ist es einfacher, Middleware nur auf eine Gruppe von Routen anzuwenden, anstatt global.

```js
const express = require('express')
const app = express()
const router = express.Router()

// Router-level middleware to log requests to '/menu' routes
router.use((req, res, next) => {
  console.log('Request made to /menu route')
  next() // Pass control to the next handler
})

// Define routes
router.get('/drinks', (req, res) => {
  res.send('Welcome to the drinks menu!')
})

app.use('/menu', router) // Apply the router

app.listen(3000, () => {
  console.log('Server running on http://localhost:3000')
})
```

* `router.use()`: Dies ist Router-Level-Middleware, die nur auf Routen angewendet wird, die vom Router behandelt werden.
* `app.use('/menu', router)`: Die Middleware gilt nur für Routen, die mit `/menu` beginnen (z. B. `/menu/drinks`).

Router-Level-Middleware ermöglicht es Ihnen, Ihre Anwendung in modulare Komponenten zu unterteilen und Middleware selektiv auf eine Gruppe von Routen anzuwenden. Das hilft bei der Organisation und Klarheit des Codes.
