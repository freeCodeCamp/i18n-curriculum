# --description--

برمجيات وسيطة على مستوى الموجِّه تشبه برمجيات وسيطة على مستوى التطبيق، لكنها مرتبطة بنسخة من `express.Router()` بدلاً من نسخة التطبيق الرئيسية. يمكن استخدامها لتقسيم تطبيق Express إلى وحدات مسارات معزولة.

تعمل برمجيات وسيطة على مستوى الموجِّه بنفس طريقة برمجيات وسيطة على مستوى التطبيق، لكنها تُطبَّق على نسخ محددة من الموجِّه. هذا يسهل تطبيق البرمجيات الوسيطة على مجموعة من المسارات فقط بدلاً من التطبيق بشكل عام.

```js
const express = require('express')
const app = express()
const router = express.Router()

// Router-level middleware to log requests to '/menu' routes
router.use((req, res, next) => {
  console.log('Request made to /menu route')
  next() // Pass control to the next handler
})

// Define routes
router.get('/drinks', (req, res) => {
  res.send('Welcome to the drinks menu!')
})

app.use('/menu', router) // Apply the router

app.listen(3000, () => {
  console.log('Server running on http://localhost:3000')
})
```

* `router.use()`: هذه برمجيات وسيطة على مستوى الموجِّه، تُطبَّق فقط على المسارات التي يتعامل معها الموجِّه.
* `app.use('/menu', router)`: البرمجيات الوسيطة تُطبَّق فقط على المسارات التي تبدأ بـ `/menu` (مثلًا، `/menu/drinks`).

تسمح برمجيات وسيطة على مستوى الموجِّه بتقسيم التطبيق إلى مكوِّنات وحدات وتطبيق البرمجيات الوسيطة بشكل انتقائي على مجموعة من المسارات. هذا يساعد في التنظيم ووضوح الشفرة.
