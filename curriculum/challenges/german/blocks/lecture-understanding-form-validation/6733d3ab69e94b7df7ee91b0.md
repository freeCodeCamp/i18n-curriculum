# --description--

Lernen wir, wie das `submit`-Ereignis mit HTML-Formularen funktioniert. Zuerst müssen wir verstehen, wie man ein Formular absendet. Es gibt drei Möglichkeiten, wie ein Formular abgesendet werden kann.

Die erste ist, wenn der Benutzer auf eine Schaltfläche im Formular klickt, die das `type`-Attribut mit dem Wert `submit` besitzt.

Die zweite ist, wenn der Benutzer die Eingabetaste (Enter) in einem beliebigen bearbeitbaren Eingabefeld im Formular drückt.

Die dritte erfolgt durch einen JavaScript-Aufruf der `requestSubmit()`- oder `submit()`-Methoden des Formularelements.

Aber was passiert, wenn ein Formular abgesendet wird? Es passieren einige Dinge, und das Verhalten des Formulars hängt von seinen Attributen ab.

Das erste Attribut, das wir uns ansehen müssen, ist das `action`-Attribut. Das `action`-Attribut sollte entweder eine URL oder einen relativen Pfad für die aktuelle Domain enthalten. Dieser Wert bestimmt, wohin das Formular versucht, die Daten zu senden. Wenn Sie kein `action`-Attribut setzen, sendet das Formular die Daten an die URL der aktuellen Seite. Hier ist ein Beispiel für ein Formular mit einem `action`-Attribut, das auf eine bestimmte URL gesetzt ist:

```html
<form action="https://freecodecamp.org">
  <input type="number" id="input" placeholder="Enter a number" name="number" />
  <button type="submit">Submit</button>
</form>
```

Wenn dieses Formular abgesendet wird, sendet es die Daten an die Startseite von freeCodeCamp, was wahrscheinlich nichts bewirken wird.

Hier ist ein weiteres Formular, das an einen relativen Pfad sendet:

```html
<form action="/data">
  <input type="number" id="input" placeholder="Enter a number" name="number" />
  <button type="submit">Submit</button>
</form>
```

Dieses sendet an den `/data`-Pfad auf der aktuellen Domain. In unserem Fall würde es an `http://127.0.0.1:5500/data` senden.

Das zweite Attribut, das das Verhalten einer Einsendung steuert, ist das `method`-Attribut. Dieses Attribut akzeptiert eine Standard-HTTP-Methode, wie `GET` oder `POST`, und verwendet diese Methode, wenn die Anfrage an die `action`-URL gesendet wird.

HTTP steht für Hypertext Transfer Protocol und wird verwendet, um Daten über das Web zu übertragen.

HTTP-Methoden definieren die Aktionen, die auf Ressourcen ausgeführt werden können, wie `GET`, `POST`, `PUT`, `DELETE` und so weiter. Sie werden in zukünftigen Lektionen mehr über diese Methoden lernen.

Wenn keine Methode gesetzt ist, verwendet das Formular standardmäßig eine `GET`-Anfrage. Eine `GET`-Anfrage wird verwendet, um Daten von einer angegebenen Ressource abzurufen, ohne Änderungen daran vorzunehmen, und die Parameter werden typischerweise als Abfragezeichenfolge an die URL angehängt.

Die Daten im Formular werden als `name=value`-Paare URL-codiert und als Abfrageparameter an die `action`-URL angehängt. Wenn Sie beispielsweise das Formular aus unserem vorherigen Beispiel mit dem Wert `3342` für das `input`-Feld absenden, würde das Formular versuchen, die URL `GET` mit einer `http://127.0.0.1:5500/data?number=3342`-Anfrage aufzurufen.

Das ist ideal für etwas wie ein Suchformular, bei dem Ihr Benutzer Daten abfragt. Wenn Sie jedoch möchten, dass Ihr Benutzer neue Daten absendet, wie bei einem Registrierungsformular, ist die idiomatische Methode die `POST`-Methode. Die `POST`-Methode wird verwendet, um Daten an den Server zu senden, um eine Ressource zu erstellen oder zu aktualisieren.

Setzen wir das `method`-Attribut auf den Wert `POST`:

```xml
<form action="/data" method="POST">
  <input type="number" id="input" placeholder="Enter a number" name="number" />
  <button type="submit">Submit</button>
</form>
```

Wenn Sie eine `POST`-Anfrage senden, kann ein Body enthalten sein, der die Daten für Ihre Anfrage enthält. Im Gegensatz zu einer `GET` werden die Daten also nicht an die URL angehängt, und unser Formular sendet die `POST`-Anfrage an `http://127.0.0.1:5500/data`. Die Daten befinden sich stattdessen im Body der Anfrage, weiterhin als URL-codierte Formulardaten.

URL-codierte Formulardaten sind, wenn Formulardaten in eine Zeichenkette aus Schlüssel-Wert-Paaren umgewandelt werden, wie `name=John+Doe&email=john%40example.com`, wobei Sonderzeichen durch codierte Versionen ersetzt werden, um die Daten sicher über das Web zu senden. Mehr darüber lernen Sie in zukünftigen Lektionen.

Vielleicht möchten Sie die Daten aber nicht als URL-codierte Form-Nutzlast senden?

Das `form`-Element akzeptiert ein `enctype`-Attribut, das den zu verwendenden Codierungstyp für die Daten darstellt.

Dieses Attribut akzeptiert nur drei Werte: `application/x-www-form-urlencoded` (was der Standard ist und die Daten als URL-codierten Form-Body sendet), `text/plain` (was die Daten im Klartext in `name=value`-Paaren sendet, getrennt durch Zeilenumbrüche) oder `multipart/form-data`, das speziell für Formulare mit Datei-Uploads gedacht ist.

In dieser Lektion haben Sie die Grundlagen von Formular-Einsendungen gelernt. In zukünftigen Lektionen werden wir tiefer in die Arbeit mit Formularen und deren Interaktion mit dem Server eintauchen.
