# --description--

Wakati seva ya HTML inatumikia HTML, API hutumikia data. API ya <dfn>REST</dfn> (uhamishaji wa hali ya uwakilishi) huruhusu kubadilishana data kwa njia rahisi, bila mteja kujua maelezo yoyote kuhusu seva. Mteja anahitaji tu kujua wapi rasilimali iko (URL), na kitendo anachotaka kufanya nayo (kitenzi). Kitenzi cha GET hutumika unapochukua taarifa, bila kubadilisha chochote. Siku hizi, muundo wa data unaopendekezwa kwa kusafirisha taarifa mtandaoni ni JSON. Kwa kifupi, JSON ni njia rahisi ya kuwakilisha kitu cha JavaScript kama mfuatano wa herufi, ili kiweze kusafirishwa kwa urahisi.

Tufanye API rahisi kwa kuunda njia ya data inayojibu na JSON kwenye njia `/json`. Unaweza kufanya hivyo kama kawaida, kwa kutumia njia `app.get()`. Ndani ya mshughulikiaji wa njia ya data, tumia njia `res.json()`, ukipitisha kitu kama hoja. Njia hii inafunga mzunguko wa ombi-jibu, ikirudisha data. Chini ya pazia, hubadilisha kitu halali cha JavaScript kuwa mfuatano wa herufi, kisha kuweka vichwa vinavyofaa ili kuambia kivinjari chako kuwa unatumia JSON, na kutuma data hiyo nyuma. Kitu halali kina muundo wa kawaida `{key: data}`. `data` inaweza kuwa nambari, mfuatano wa herufi, kitu kilichopangwa ndani au safu ya data. `data` pia inaweza kuwa kigezo au matokeo ya wito wa kitendakazi, ambapo itahesabiwa kabla ya kubadilishwa kuwa mfuatano wa herufi.

# --instructions--

Tumikia kitu `{"message": "Hello json"}` kama jibu, kwa muundo wa JSON, kwa maombi ya GET kwenye njia ya data `/json`. Kisha elekeza kivinjari chako kwenye `your-app-url/json`, unapaswa kuona ujumbe kwenye skrini.

# --hints--

Kiishio `/json` kinapaswa kutumikia kitu cha JSON `{"message": "Hello json"}`

```js
  const response = await fetch(code + '/json');
  if (!response.ok) {
    throw new Error(await response.text());
  }
  const data = await response.json();
  assert.equal(
    data.message,
    'Hello json',
    "The '/json' endpoint does not serve the right data"
  );
```
