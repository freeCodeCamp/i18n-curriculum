# --explanation--

`SQL injection attacks` — це тип кібер-атаки, коли хакери вставляють шкідливий код у запит до бази даних, щоб отримати доступ або пошкодити дані. Наприклад:

`The company added security measures to prevent SQL injection attacks on their website.` — це означає, що вони вжили заходів, щоб зупинити хакерів від введення шкідливого коду.

Джейк каже, що `additional measures` потрібні, щоб `prevent SQL injection attacks`, тобто вони намагаються запобігти цьому конкретному виду загрози.
