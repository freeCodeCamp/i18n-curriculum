# --interactive--

CSS에서 `!important` 키워드는 스타일 규칙에 가장 높은 우선순위를 부여하여 해당 속성에 대한 다른 선언을 모두 재정의할 수 있게 합니다.

이 키워드를 사용하면 브라우저가 다른 선택자의 특이성에 상관없이 지정된 스타일을 적용하도록 강제합니다.

예를 들어, 다음과 같이 인라인 스타일이 적용된 HTML 문단 요소가 있다고 가정해 보겠습니다:

:::interactive_editor

```html
<p class="para" style="background-color: lightblue; color: black;">
  This is a paragraph.
</p>
```

:::

이 예제에서 문단 요소는 배경색이 `lightblue`로, 텍스트 색상이 `black`로 설정되어 있습니다.

CSS 파일은 문단 요소에 다음과 같은 스타일을 적용합니다:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<p class="para" style="background-color: lightblue; color: black;">
  This is a paragraph.
</p>
```

```css
.para {
  background-color: black;
  color: white;
}
```

:::

인라인 스타일은 클래스, ID, 타입 선택자보다 우선순위가 높기 때문에, 검은색 배경과 흰색 텍스트 색상은 해당 문단 요소에 적용되지 않습니다.

이 인라인 스타일을 재정의하려면 `!important` 키워드를 다음과 같이 사용할 수 있습니다:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<p class="para" style="background-color: lightblue; color: black;">
  This is a paragraph.
</p>
```

```css
.para {
  background-color: black !important;
  color: white !important;
}
```

:::

`!important` 키워드는 CSS 값 뒤, 세미콜론 앞에 사용됩니다.

이제 문단 요소에 검은색과 흰색 색상이 적용되는데, 이는 `!important` 키워드를 사용해 인라인 스타일을 재정의했기 때문입니다.

CSS에서 `!important` 키워드는 스타일 규칙에 가장 높은 우선순위를 부여하여, 더 높은 특이성을 가진 선언이나 인라인 스타일까지도 효과적으로 재정의합니다.

하지만 `!important` 키워드는 CSS 선택자의 특이성을 변경하지 않습니다. 단지 `!important`가 포함된 규칙이 더 높은 특이성을 가진 다른 충돌 규칙이 있어도 적용되도록 보장합니다.

`!important` 키워드를 적절히 사용하는 또 다른 경우는 원본 CSS를 제어할 수 없는 서드파티 라이브러리나 프레임워크의 스타일을 재정의할 때입니다.

하지만 `!important` 키워드를 과도하게 사용하면 스타일의 자연스러운 계단식 흐름이 깨지고, 유지보수와 디버깅이 어려워질 수 있으며, 의도치 않은 결과를 초래할 수 있습니다.

따라서 `!important` 키워드는 절제해서 사용하는 것이 가장 좋습니다.
