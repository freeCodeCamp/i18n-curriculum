# --interactive--

У попередніх уроках ви дізналися про асинхронне програмування, яке дозволяє іншому коду виконуватися, поки ми чекаємо на завершення деяких тривалих завдань, наприклад, отримання даних із сервера, читання даних із файлу тощо.

`async`/`await`, побудовані на основі промісів, роблять написання та читання асинхронного коду простішим. Коли ви ставите ключове слово `async` перед функцією, це означає, що ця функція завжди повертатиме `Promise`. Лише всередині `async` функції ви можете використовувати ключове слово `await`, яке дозволяє чекати на розв’язання `Promise` перед переходом до наступної лінії коду. Ось приклад, щоб показати, як працює `async`/`await`:

:::interactive_editor

```js
async function delayedGreeting(name) {
  console.log("A Messenger entered the chat...");
  await new Promise(resolve => setTimeout(resolve, 2000));
  console.log(`Hello, ${name}!`);
}

delayedGreeting("Alice");
console.log("First Printed Message!");
```

:::

У цьому коді ми визначаємо `async` асинхронну функцію з ім’ям `delayedGreeting`. Всередині цієї функції ми використовуємо `await`, щоб призупинити виконання на 2 секунди. Після затримки вона виводить привітання.

Коли ми викликаємо цю функцію, ви побачите, що `First Printed Message!` з’являється перед привітанням. Це тому, що функція асинхронна — вона не блокує виконання решти коду.

Однією з найбільших переваг `async`/`await` є обробка помилок. З промісами ми часто мусили використовувати метод `.catch()` для обробки помилок. З `async`/`await` ми можемо використовувати блоки `try`/`catch`. Ось приклад:

```js
async function fetchUserData() {
  try {
    let response = await fetch(`https://api.example.com/users`);
    let userData = await response.json();
    console.log(userData);
  } catch (error) {
    console.log("Error fetching user data:", error);
  }
}

fetchUserData();
```

У цьому прикладі ми використовуємо `async`/`await`, щоб отримати дані користувача з API. Ключове слово `await` використовується двічі: один раз, щоб дочекатися завершення операції fetch, і ще раз, щоб дочекатися завершення парсингу JSON. Якщо під час цього процесу виникне помилка, вона буде спіймана в блоці `catch`.
