# --description--

- **JavaScript sincrono** viene eseguito in sequenza e aspetta che l'operazione precedente finisca prima di passare alla successiva.
- **JavaScript asincrono** permette di eseguire più operazioni in background senza bloccare il thread principale.
- **Thread** è una sequenza di istruzioni che può essere eseguita indipendentemente dal flusso principale del programma.
- Le **funzioni di callback** sono funzioni passate come argomenti ad altre funzioni e vengono eseguite dopo il completamento dell’operazione o in seguito a un evento.

## Il motore JavaScript e il runtime JavaScript

- Il **motore JavaScript** è un programma che esegue codice JavaScript in un browser web. Funziona come un convertitore che prende il tuo codice, lo trasforma in istruzioni che il computer può capire ed eseguire di conseguenza.
- V8 è un esempio di motore JavaScript sviluppato da Google.
- Il **runtime JavaScript** è l’ambiente in cui il codice JavaScript viene eseguito. Include il motore JavaScript che elabora ed esegue il codice, e funzionalità aggiuntive come un browser web o Node.js.

## L’API Fetch

- L’API Fetch permette alle app web di effettuare richieste di rete, tipicamente per recuperare o inviare dati al server. Fornisce un metodo `fetch()` che puoi usare per fare queste richieste.
- Puoi recuperare testo, immagini, audio, JSON e altri tipi di dati usando l’API Fetch.

## Metodi HTTP per l’API Fetch

L’API Fetch supporta vari metodi HTTP per interagire con il server. I metodi più comuni sono:

- **GET**: usato per recuperare dati dal server. Per impostazione predefinita, l’API Fetch usa il metodo `GET` per recuperare dati.

```js
fetch('https://api.example.com/data')
```

Per usare i dati recuperati, devono essere convertiti in formato JSON usando il metodo `.json()`:

```js
fetch('https://api.example.com/data')
  .then(response => response.json())
  .then(data => console.log(data))
```

In questo codice, la risposta proveniente dall’API Fetch è una promise e il gestore `.then` sta convertendo la risposta in formato JSON.

- **POST**: usato per inviare dati al server. Il metodo `POST` viene usato per creare nuove risorse sul server.

```js
fetch('https://api.example.com/users', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({
    name: 'John Doe',
    email: 'john@example.com'
  })
})
```

In questo esempio, stiamo inviando una richiesta `POST` per creare un nuovo utente. Abbiamo specificato il metodo come `POST`, impostato le intestazioni appropriate e incluso un corpo con i dati che vogliamo inviare. Il corpo deve essere una stringa, quindi usiamo `JSON.stringify()` per convertire il nostro oggetto in una stringa JSON.

- **PUT**: usato per aggiornare dati sul server. Il metodo `PUT` viene usato per aggiornare risorse esistenti sul server.

```js
fetch('https://api.example.com/users/45', {
  method: 'PUT',
  headers: {
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({
    name: 'John Smith',
    email: 'john@example.com'
  })
})
```

In questo esempio, stiamo aggiornando l’ID `45` specificato alla fine dell’URL. Abbiamo usato il metodo `PUT` nel codice e specificato anche i dati come corpo che saranno usati per aggiornare i dati identificati.

- **DELETE**: usato per eliminare dati sul server. Il metodo `DELETE` viene usato per eliminare risorse sul server.

```js
fetch('https://api.example.com/users/45', {
  method: 'DELETE'
})
```

In questo esempio, stiamo inviando una richiesta `DELETE` per rimuovere un utente con l’ID `45`.

## Promise e concatenamento di promise

- Le **promise** sono oggetti che rappresentano il completamento eventuale o il fallimento di un’operazione asincrona e il suo valore risultante. Il valore della promise è noto solo quando l’operazione `async` è completata.
- Ecco un esempio per creare una promise semplice:

```js
const promise = new Promise((resolve, reject) => {
  setTimeout(() => {
    resolve('Data received successfully');
  }, 2000);
});
```

- Il metodo `.then()` viene usato in una Promise per specificare cosa deve succedere quando la Promise viene soddisfatta, mentre `.catch()` serve per gestire eventuali errori.
- Ecco un esempio di uso di `.then()` e `.catch()` con una Promise:

```js
promise
  .then(data => {
    console.log(data);
  })
  .catch(error => {
    console.error(error);
  });
```

Nell’esempio sopra, il metodo `.then()` viene usato per stampare i dati ricevuti dalla Promise, mentre il metodo `.catch()` serve per stampare eventuali errori che si verificano.

- **Concatenamento di promise**: una delle funzionalità potenti delle Promise è che possiamo concatenare più operazioni asincrone insieme. Ogni `.then()` può restituire una nuova Promise, permettendoti di eseguire una sequenza di operazioni asincrone una dopo l’altra.
- Ecco un esempio di concatenamento di Promise:

```js
fetch('https://api.example.com/data')
  .then(response => response.json())
  .then(data => {
    console.log(data);
    return fetch('https://api.example.com/other-data');
  })
  .then(response => response.json())
  .then(otherData => {
    console.log(otherData);
  })
  .catch(error => {
    console.error(error);
  });
```

Nell’esempio sopra, prima recuperiamo dati da un URL, poi recuperiamo dati da un altro URL basandoci sulla prima risposta, e infine stampiamo i secondi dati ricevuti.

Il metodo `catch` gestirebbe eventuali errori che si verificano durante il processo. Questo significa che non devi aggiungere la gestione degli errori a ogni passaggio, semplificando molto il tuo codice.

## Usare `async/await` per gestire le promise

Async/await rende più facile scrivere e leggere codice asincrono ed è costruito sopra le Promise.

- **async**: la parola chiave `async` viene usata per definire una funzione asincrona. Una funzione `async` restituisce una Promise, che si risolve con il valore restituito dalla funzione `async`.
- **await**: la parola chiave `await` viene usata all’interno di una funzione `async` per mettere in pausa l’esecuzione della funzione finché la Promise non viene risolta. Può essere usata solo dentro una funzione `async`.
- Ecco un esempio di uso di `async/await`:

```js
async function delayedGreeting(name) {
  console.log("A Messenger entered the chat...");
  await new Promise(resolve => setTimeout(resolve, 2000));
  console.log(`Hello, ${name}!`);
}

delayedGreeting("Alice");
console.log("First Printed Message!");
```

Nell’esempio sopra, la funzione `delayedGreeting` è una funzione `async` che si mette in pausa per 2 secondi prima di stampare il messaggio di saluto. La parola chiave `await` viene usata per mettere in pausa l’esecuzione della funzione finché `Promise` non viene risolta.

- Uno dei maggiori vantaggi di `async/await` è la gestione degli errori tramite blocchi `try/catch`. Ecco un esempio:

```js
async function fetchData() {
  try {
    const response = await fetch('https://api.example.com/data');
    const data = await response.json();
    console.log(data);
  } catch (error) {
    console.error(error);
  }
}

fetchData();
```

Nell’esempio sopra, il blocco `try` contiene il codice che potrebbe lanciare un errore, e il blocco `catch` gestisce l’errore se si verifica. Questo rende la gestione degli errori più semplice e leggibile.

## L’attributo `async`

- L’attributo `async` dice al browser di scaricare il file script in modo asincrono mentre continua a parsare il documento HTML.
- Una volta scaricato lo script, il parsing HTML viene messo in pausa, lo script viene eseguito, e poi il parsing HTML riprende.
- Dovresti usare `async` per script indipendenti dove l’ordine di esecuzione non è importante.

## L’attributo `defer`

- L’attributo `defer` scarica anch’esso lo script in modo asincrono, ma differisce l’esecuzione dello script fino a dopo che il documento HTML è stato completamente parsato.
- Gli script `defer` mantengono l’ordine di esecuzione come appaiono nel documento HTML.

- È importante notare che entrambi gli attributi `async` e `defer` vengono ignorati per script in linea e funzionano solo per file script esterni.

- Quando sono presenti entrambi gli attributi `async` e `defer`, l’attributo `async` ha la precedenza.

## API Geolocation

- L’API Geolocation fornisce un modo per i siti web di richiedere la posizione dell’utente.

- L’esempio qui sotto dimostra il metodo `getCurrentPosition()` dell’API, usato per ottenere la posizione attuale dell’utente.

```js
navigator.geolocation.getCurrentPosition(
  (position) => {
    console.log("Latitude: " + position.coords.latitude);
    console.log("Longitude: " + position.coords.longitude);
  },
  (error) => {
    console.log("Error: " + error.message);
  }
);
```

In questo codice, stiamo chiamando `getCurrentPosition` e passando una funzione che verrà chiamata quando la posizione viene ottenuta con successo.

L’oggetto `position` contiene varie informazioni, ma qui abbiamo selezionato solo `latitude` e `longitude`.

Se c’è un problema nel recuperare la `position`, l’errore verrà stampato nella console.

- È importante rispettare la privacy dell’utente e richiedere la sua posizione solo quando necessario.
