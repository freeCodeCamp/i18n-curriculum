# --interactive--

讓靜態內容可存取通常相對簡單，但動態內容可能更具挑戰性。這就是 WAI-ARIA 派上用場的地方。

讓我們來看看什麼是 WAI-ARIA、它的目的、它如何運作，以及一些範例。

WAI-ARIA 代表 Web Accessibility Initiative - Accessible Rich Internet Applications。它是一個增強動態內容和 UI（使用者介面）組件可及性的規格。

請注意，WCAG 與 WAI-ARIA 並不相同。WCAG 提供網頁無障礙的一般指導方針，而 WAI-ARIA 則為使用輔助技術的使用者提供使動態和互動內容可存取的具體規則。

因此，WAI-ARIA 的主要目的是改善動態內容和沒有原生 HTML 等價的使用者介面組件的可存取性。

WAI-ARIA 的運作方式是透過為 HTML 元素添加一組屬性，以提供額外的語意資訊。這些屬性分為角色、狀態和屬性。

ARIA 角色定義網站或網頁應用程式中元素的用途。以下是將 `div` 元素的角色設定為 `button` 的範例。

```html
<div role="button">Click Me</div>
```

透過這樣做，你是在向輔助技術表明該元素是一個按鈕。然而，角色並不提供任何功能。僅僅給這個 `div` 指定 `role` 為 `button` 並不會讓它像按鈕一樣運作。要讓它看起來並行為像按鈕，你需要使用 CSS 和 JavaScript 來達到預期的效果。 

這裡是一個使用 HTML、CSS 和 JavaScript 來創建訂製 `button` 元素的範例。 

**注意**：不用擔心嘗試理解 CSS 和 JavaScript 程式碼。你會在未來的模組中學習這些語言。 

:::interactive_editor

```html
<link href="styles.css" rel="stylesheet">
<div id="custom-btn" role="button">Click Me</div>
<script src="index.js"></script>
```

```css
#custom-btn {
  display: inline-block;
  padding: 0.4em 1em;
  font-size: 1rem;
  font-family: sans-serif;
  color: buttontext;
  background-color: buttonface;
  border: 1px solid #ccc;
  border-radius: 4px;
  cursor: pointer;
  user-select: none;
  text-align: center;
}

#custom-btn:focus {
  outline: 2px solid Highlight;
  outline-offset: 2px;
}

#custom-btn:active {
  background-color: #ddd;
}
```

```js
const button = document.getElementById("custom-btn");

button.addEventListener("click", () => {
  alert("Button clicked!");
});

button.addEventListener("keydown", (e) => {
  if (e.key === "Enter" || e.key === " ") {
    e.preventDefault(); 
    button.click();
  }
});
```

:::

最好改用原生的 `button` 或帶有 `type="button"` 的 `input` 元素。

ARIA 屬性提供關於元素的額外細節。例如，`aria-labelledby` 屬性讓你將元素連結到特定的標籤：

```html
<h2 id="header-id">About freeCodeCamp</h2>
<button id="button-id" aria-labelledby="header-id button-id">Learn More</button>
```

這將使輔助技術的使用者能夠理解並導航這些元素。

為了充分發揮 WAI-ARIA 的效用，請盡可能使用原生的 HTML，因為它通常能夠提供更多的預設存取性。

只有在 HTML 不足時才使用 WAI-ARIA，並且別忘了使用像是螢幕閱讀器等輔助技術進行測試，或讓身心障礙者測試你的作品。此外，確保你的 WAI-ARIA 狀態和屬性能隨內容即時更新。避免過度使用 ARIA，因為它常常會造成混淆。
