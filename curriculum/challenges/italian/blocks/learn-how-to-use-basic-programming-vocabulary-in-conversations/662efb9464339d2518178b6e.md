# --description--

La parola `issue` può riferirsi a un problema o questione che deve essere affrontata.

In programmazione, un `issue` spesso indica un bug o un errore nel codice che deve essere corretto. Per esempio, `We found an issue in the latest software update.`

# --questions--

## --text--

Cosa significa `issue` in un contesto di programmazione?

## --answers--

Significa che la documentazione è incompleta.

### --feedback--

Anche se i problemi di documentazione possono essere considerati `issues`, in programmazione `issue` si riferisce più direttamente a problemi all'interno del codice stesso.

### --feedback--

Un `issue` non riguarda i manuali utente ma problemi nel codice che devono essere risolti.
