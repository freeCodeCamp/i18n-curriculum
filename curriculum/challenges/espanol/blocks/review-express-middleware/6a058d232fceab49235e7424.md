## ¿Qué es middleware?

- **Middleware** es una función que se ejecuta durante el ciclo de vida de una solicitud al servidor.
- El middleware tiene acceso al **objeto solicitud** (`req`), al **objeto respuesta** (`res`) y a la **función `next`**.
- Usos comunes: registrar solicitudes, manejar errores, interpretar datos entrantes y servir archivos estáticos.
- El middleware se ejecuta en el orden en que se agrega a la app.

### Cómo funciona el middleware

1. Se envía una solicitud al servidor.
2. Las funciones middleware procesan la solicitud.
3. Si el middleware llama a `next()`, el control pasa al siguiente middleware o manejador de ruta.
4. Si el middleware no llama a `next()`, el ciclo solicitud-respuesta se detiene.

```javascript
app.use((req, res, next) => {
  console.log('Request URL:', req.url);
  console.log('Request Method:', req.method);
  next();
});
```

- `app.use()` agrega middleware globalmente a la aplicación.

## Middleware a nivel de aplicación

- El **middleware a nivel de aplicación** está vinculado a una instancia de la app Express.
- Se registra usando `app.use()` o métodos específicos de ruta como `app.get()`, `app.post()`, etc.
- Se ejecuta para cada solicitud entrante a menos que se limite a una ruta específica.
- Casos de uso: registro, autenticación y manejo de errores en múltiples rutas.

```javascript
app.use((req, res, next) => {
  console.log('Request received at:', new Date());
  next();
});
```

## Middleware a nivel de enrutador

- El **middleware a nivel de enrutador** está vinculado a una instancia de `express.Router()` en lugar de la app principal.
- Se aplica usando `router.use()` y solo afecta las rutas manejadas por ese enrutador.
- Ayuda a modularizar la app en módulos de rutas aislados.
- Monta un enrutador con `app.use('/path', router)` para limitar el middleware a rutas específicas.

```javascript
const router = express.Router();

router.use((req, res, next) => {
  console.log('Request made to /menu route');
  next();
});

router.get('/drinks', (req, res) => {
  res.send('Welcome to the drinks menu!');
});

app.use('/menu', router);
```

## Middleware para manejo de errores

- El **middleware para manejo de errores** es un middleware especial usado para capturar errores durante el ciclo solicitud-respuesta.
- Tiene **cuatro parámetros**: `err`, `req`, `res` y `next`. La firma de cuatro parámetros es lo que indica a Express que es un manejador de errores.
- Se invoca cuando se pasa un error a `next(err)`.
- Debe colocarse **al final** de la pila de middleware, después de todos los demás middleware y rutas.

```javascript
app.use((err, req, res, next) => {
  console.error(err.stack);
  res.status(500).send('Something went wrong!');
});
```

## Middleware integrado y de terceros

### Middleware integrado

- **`express.json()`**: interpreta solicitudes entrantes con cargas útiles JSON.

```javascript
app.use(express.json());
```

- **`express.static()`**: sirve archivos estáticos como imágenes, CSS y JavaScript.

```javascript
app.use(express.static('public'));
```

- **`express.urlencoded()`**: interpreta solicitudes entrantes con cargas útiles codificadas en URL (el formato enviado por formularios HTML); hace que los datos interpretados estén disponibles en `req.body`.

```javascript
app.use(express.urlencoded({ extended: true }));
```

### Middleware de terceros

- **`cors`**: habilita el intercambio de recursos de origen cruzado.

```javascript
app.use(cors());
```

- **`morgan`**: registra solicitudes HTTP.

```javascript
app.use(morgan('tiny'));
```

- Tanto las funciones middleware integradas como las de terceros se agregan en el orden en que se llaman con `app.use()`.
