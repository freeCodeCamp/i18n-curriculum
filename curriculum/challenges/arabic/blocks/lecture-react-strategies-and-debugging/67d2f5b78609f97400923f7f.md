# --description--

مع نمو تطبيقك، قد تصبح إدارة تدفق البيانات بين المكوّنات معقدة.

عند البدء، قد يكون خطاف `useState` في React كافيًا، لكن مع إضافة ميزات، قد تواجه مشكلات مثل:

- تمرير الخصائص عبر مكوّنات لا تحتاجها، والمعروف أيضًا باسم تمرير الخصائص العميق
- الحفاظ على تزامن البيانات عبر أجزاء مختلفة من التطبيق
- التعامل مع تحديثات معقدة تؤثر على عدة مكوّنات في نفس الوقت

قد تظهر هذه التحديات وغيرها، مما يؤدي إلى قاعدة شفرة أصعب في الصيانة، والمعالجة، والاختبار. هنا تأتي مكتبات إدارة الحالة – فهي توفر مكانًا مركزيًا يمكن للمكوّنات من خلاله الحصول على البيانات التي تحتاجها أو تحديثها.

لنلقِ نظرة على بعض خيارات إدارة الحالة المختلفة المتاحة لديك، ومتى تستخدمها.

واجهة برمجة التطبيقات Context API هي مدير حالة مدمج في React يتيح لك مشاركة الحالة عبر المكوّنات دون استخدام مكتبة خارجية. إنها ترقية معتمدة على خطاف `useState`، لذا فهي مثالية لحالات مثل تبديل السمات أو حالة توثيق المستخدم.

مع ذلك، لا تتعامل Context API بشكل جيد مع التحديثات المتكررة، وقد تسبب إعادة عرض غير ضرورية، مما يجعلها أقل ملاءمة لاحتياجات الحالة المعقدة في تطبيقات مثل التجارة الإلكترونية ومنصات التواصل الاجتماعي.

إليك مكوّن عداد يوضح الاستخدام الأساسي لـ Context API:

```jsx
import { useState, createContext } from 'react';

const CounterContext = createContext();

const CounterProvider = ({ children }) => {
  const [count, setCount] = useState(0);

  return (
    <CounterContext.Provider value={{ count, setCount }}>
      {children}
    </CounterContext.Provider>
  );
};

export { CounterContext, CounterProvider };
```

تنشئ هذه الشفرة سياقًا وموفرًا لمشاركة حالة `count` عبر التطبيق.

يستخدم `CounterProvider` خطاف `useState` لتهيئة وإدارة حالة `count` ودالة التعيين الخاصة بها. ثم تُمرر كلاهما إلى المكوّنات الفرعية عبر `Provider`.

لذا، عندما تغلف تطبيقك بالكامل بـ `CounterProvider`، تصبح حالة `count` متاحة في كل مكان داخل التطبيق.

إليك كيفية تغليف `CounterProvider` حول تطبيقك:

```jsx
import { CounterProvider } from './context/CounterContext';

function App() {
  return (
    <CounterProvider>
        {/* App components */}
    </CounterProvider>
  );
}

export default App;
```

وهنا كيفية استخدام حالة `count`:

```jsx
import React, { useContext } from 'react';
import { CounterContext } from '../context/CounterContext';

const Counter = () => {
  const { count, setCount } = useContext(CounterContext);

  return (
    <>
      <div style={{ textAlign: 'center' }}>
        <h1>Context API Counter</h1>
        <button style={{ marginRight: '5px' }} onClick={() => setCount(count - 1)}>
          Decrease
        </button>
        <span>{count}</span>
        <button style={{ marginLeft: '5px' }} onClick={() => setCount(count + 1)}>
          Increase
        </button>
      </div>
    </>
  );
};

export default Counter;
```

كما ترى، يتم تهيئة `count` ودالة التعيين الخاصة بها `setCount` عبر دالة `useContext`.

ثم تُعرض حالة `count` الحالية، وتُستخدم `setCount` لزيادة أو تقليل حالة `count` عند نقر المستخدم على أزرار النقص والزيادة على التوالي.

مكتبة إدارة الحالة الشهيرة الأخرى هي Redux، وهي واحدة من أكثر المكتبات استخدامًا مع React. موجودة منذ فترة طويلة، ومثالية للتطبيقات الكبيرة مثل التجارة الإلكترونية ومنصات التواصل الاجتماعي والمنتديات، وغيرها.

يدير Redux الحالة من خلال توفير مخزن مركزي وتحكم صارم في تحديثات الحالة. يستخدم نمطًا متوقعًا مع الإجراءات، والمخفضات، والبرمجيات الوسيطة.

الإجراءات هي حمولة معلومات ترسل البيانات من تطبيقك إلى مخزن Redux، وغالبًا ما تُفعّل بواسطة تفاعلات المستخدم.

المخفضات هي دوال تحدد كيف يجب أن تتغير الحالة استجابة لتلك الإجراءات، مع ضمان تحديث الحالة بطريقة غير قابلة للتغيير.

البرمجيات الوسيطة، من ناحية أخرى، تعمل كجسر بين إرسال الإجراءات والمخفض، مما يسمح لك بتوسيع وظائف Redux (مثل التسجيل، والتعامل مع العمليات غير المتزامنة) دون تعديل التدفق الأساسي.

أكثر الشكاوى شيوعًا عن Redux هي كمية قالب الشفرة المعقدة التي تحتاجها للبدء. ردًا على ذلك، قدم فريق Redux "Redux Toolkit" و"RTK Query"، اللذان يبسطان عملية الإعداد كثيرًا.

عادةً ما تعرف الإجراءات والمخفضات في ملف واحد باستخدام دالة `createSlice()`. من الشائع تسمية الملف بحيث ينتهي بكلمة `Slice`، مثل `productSlice`، `userSlice`، `counterSlice`، وهكذا.

إليك ملف `counterSlice` ليعرض لك الأساسيات:

```jsx
import { createSlice } from '@reduxjs/toolkit';

const counterSlice = createSlice({
  name: 'counter',

  initialState: { count: 0 },

  reducers: {
    increment: (state) => {
      state.count += 1;
    },
    decrement: (state) => {
      state.count -= 1;
    },
  },
});

export const { increment, decrement } = counterSlice.actions;

export default counterSlice.reducer;
```

من هنا، تحتاج إلى تغليف التطبيق بالكامل بـ `Provider`، اختيار جزء من الحالة من الشريحة باستخدام `useSelector()`، ثم استخدام `useDispatch()` لتنشيط الحالة.

خيار آخر للنظر فيه هو Zustand.

Zustand هي مكتبة إدارة حالة خفيفة الوزن ذات واجهة برمجة تطبيقات بسيطة. تعتمد على الخطافات، لذا هناك قالب شفرة أقل مقارنة بـ Redux، مما يجعل الإعداد أسهل وأسرع.

Zustand مثالية للتطبيقات الصغيرة إلى المتوسطة الحجم. تعمل باستخدام خطاف `useStore` للوصول إلى الحالة مباشرة في المكوّنات والصفحات. هذا يتيح لك تعديل والوصول إلى البيانات دون الحاجة إلى إجراءات، أو مخفضات، أو موفر.

إليك `useCounterStore` التي تنفذ وظيفة عداد أخرى:

```jsx
import { create } from 'zustand';

const useCounterStore = create((set) => ({
  count: 0,
  increment: () => set((state) => ({ count: state.count + 1 })),
  decrement: () => set((state) => ({ count: state.count - 1 })),
}));

export default useCounterStore;
```

وهنا كيفية تهيئة واستخدام الحالات في تطبيقك:

```jsx
// Import the useCounterStore (it's just a hook)
import useCounterStore from '../useCounterStore';

const Counter = () => {
  // Initialize the states with the useCounterStore hook
  const { count, increment, decrement } = useCounterStore();

  return (
    <>
      <div style={{ textAlign: 'center' }}>
        <h1>Zustand Counter</h1>
        <button style={{ marginRight: '5px' }} onClick={() => decrement()}>
          Decrease
        </button>
        <span>{count}</span>
        <button style={{ marginLeft: '5px' }} onClick={() => increment()}>
          Increase
        </button>
      </div>
    </>
  );
};

export default Counter;
```

على الرغم من أن نظام بيئة الواجهة الأمامية يتطور باستمرار وتظهر مكتبات إدارة حالة جديدة بانتظام، فإن المكتبات التي ناقشناها مستخدمة على نطاق واسع في الصناعة.
