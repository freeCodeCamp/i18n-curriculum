# --interactive--

缩略语是单词或短语的缩短形式。例如，紧跟句号的 "Dr" 是单词 "doctor" 的缩略语。

缩略语有两种常见形式：首字母缩略词和首字母拼写词。

首字母缩略词是由一个短语的首字母组成的单词，每个字母代表该短语中一个单词的首字母。

GUI 是一个缩写的例子。它代表 Graphical User Interface。通过取每个单词的首字母 G、U 和 I，你得到缩写 GUI。

首字母缩略词由短语的首字母组成，每个字母表现该短语中一个单词的首字母。

例如，`HTML` 是一个首字母缩略词；它代表 HyperText Markup Language，并通过拼读每个字母 H、T、M、L 来发音。

缩略语包括首字母缩略词和首字母拼写词两种类型。区别在于首字母缩略词作为单词发音，而首字母拼写词则逐个字母发音。

它们对于编写更简洁的文本非常有帮助，尤其是在特定上下文中它们是众所周知且易于理解的时候。 

If you need to display abbreviations such as acronyms or initialisms in HTML, the abbreviation element is exactly what you're looking for. 在第一次使用时，应解释其全部含义。 然后，你可以使用缩写元素来突出显示它们，并提供更多细节。 

下面是一个例子，你可以看到一个段落中的句子 `HTML is the foundation of the web`： 

:::interactive_editor

```html
<p><abbr>HTML</abbr> is the foundation of the web.</p>
```

:::

The initialism HTML is within an abbreviation element. 在浏览器中，你会发现没有任何变化。 它看起来仍像普通文本。  The abbreviation element is providing helpful context behind the scenes, but users will still see the initialism as normal text. 

如果你想帮助用户理解这个缩写的含义，你可以用 `title` 属性显示它的完整形式。

`title` 属性是可选的，但如果你决定包含它，必须是缩略语、首字母缩写或首字母拼写的可读描述。

让我们以之前相同的例子为例，但为你的元素添加 `title` 属性。它将是 `HyperText Markup Language`，即该缩写的完整形式：

:::interactive_editor

```html
<p><abbr title="HyperText Markup Language">HTML</abbr> is the foundation of the web.</p>
```

:::

通常，添加该属性后，缩写元素的样式会发生变化。 具体样式取决于浏览器。  Some browsers may display a dotted underline, while others may convert the contents to small caps.  When the user hovers over the abbreviation, the full form is displayed as a tooltip.

虽然你不一定需要为网页上的每个缩略语使用 `abbreviation` 元素，但建议为那些可能不清楚和可能需要额外上下文的缩略语使用该元素。

你应根据自己的判断，在信息和表述之间找到适当的平衡，既要简洁明了，又要避免杂乱无章。
