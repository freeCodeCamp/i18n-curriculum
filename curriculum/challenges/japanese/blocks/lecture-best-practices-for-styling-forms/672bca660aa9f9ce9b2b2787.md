# --interactive--

`datetime-local`や`color`のような特殊な入力要素をスタイリングする際によくある問題について学びましょう。

これらの特殊なタイプの入力は、日付やカラーピッカーのようなものを作るために複雑な疑似要素に依存しています。これがこれらの入力のスタイリングに大きな課題をもたらします。1つの課題は、デフォルトのスタイルが完全にブラウザーに依存しているため、あるブラウザーでピッカーが正しく見えるCSSが、別のブラウザーではまったく異なる結果になる場合があることです。

こちらはカラ―入力の例です：

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<form>
  <label for="favorite-color">Pick your favorite color:</label>
  <input type="color" id="favorite-color" name="favorite-color">
</form>
```

```css
input {
  padding: 8px 12px;
  margin: 8px 0;
  border-radius: 6px;
  border: 1px solid #ccc;
}

input[type="color"] {
  width: 60px;
  height: 40px;
  padding: 0;
  border: 2px solid #555;
  border-radius: 4px;
  cursor: pointer;
}

```

:::

もう1つの課題は疑似要素の複雑さかもしれません。日付セレクターを考えてみてください。ここには多くの動く部分があり、疑似要素の複雑な構造が適切なエリアにスタイルを適用する際に大きな課題となる場合があります。

こちらは日付入力の例です：

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<form>
  <label for="birthdate">Select your birthdate:</label>
  <input type="date" id="birthdate" name="birthdate">
</form>
```

```css
input {
  padding: 8px 12px;
  margin: 8px 0;
  border-radius: 6px;
  border: 1px solid #ccc;
}

input[type="date"] {
  padding: 6px 10px;
  border: 2px solid #555;
  border-radius: 4px;
  font-size: 14px;
  cursor: pointer;
}

input[type="date"]::-webkit-calendar-picker-indicator {
  background-color: #4CAF50;
  color: white;
  border-radius: 4px;
  cursor: pointer;
}

```

:::

もちろん、これらの複雑な要素を手動でスタイリングすると、重要な機能を誤って失うリスクもあります。フォーカス状態や選択された項目のような重要なインジケーターを失うだけでなく、セレクター自体が完全に壊れてしまう可能性もあります。

これらの理由から、多くの開発者はブラウザーの組み込みコンポーネントを使う代わりに、JavaScriptライブラリやカスタムコンポーネントに頼ることが多いです。
