# --description--

De manera algo confusa, cualquier elemento puede ser tanto un contenedor flex como un ítem flex. Dicho de otra forma, también puedes poner `display: flex` en un ítem flex y luego usar flexbox para organizar sus hijos.

<img src="https://cdn.freecodecamp.org/curriculum/odin-project/flex-box/flexbox-03.png" style="width: 95%; height: 95%;" alt="a flex container that has multiple flex items, within those flex items are nested flex items as well. Making the parent of those nested flex items also a flex container." />

Crear y anidar múltiples contenedores y elementos flex es la forma principal en que estarás construyendo estructuras complejas. La siguiente imagen se logró usando solo flexbox para organizar, dimensionar y colocar los distintos elementos. Flexbox es una herramienta muy poderosa.

<img src="https://cdn.freecodecamp.org/curriculum/odin-project/flex-box/flexbox-02.png" alt="a complex layout of flex items and flex containers. There are multiple flex containers nested into each other, thus making them flex items as well"/>

Ciertamente, la imagen presenta una representación de una estructura CSS Flexbox con contenedores flex anidados. El contenedor externo está denotado como "ALSO a flex container" resaltado en azul, y dentro de él hay otro contenedor marcado como "with flex items" en rojo. Dentro del contenedor rojo, hay tres elementos etiquetados como "flex items" en durazno. Esto demuestra que un contenedor flex puede estar anidado dentro de otro contenedor flex y contener sus propios elementos flex, mostrando la naturaleza recursiva de las estructuras de diseño Flexbox.
