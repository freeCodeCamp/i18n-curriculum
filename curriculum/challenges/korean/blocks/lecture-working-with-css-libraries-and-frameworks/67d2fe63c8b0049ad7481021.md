# --description--

CSS 전처리기는 변수, 믹스인, 중첩, 선택자 상속 같은 강력한 기능으로 표준 CSS를 확장하는 도구입니다.

이 중 변수와 중첩 같은 기능은 이제 네이티브 CSS에서 지원되거나 지원이 점점 확대되고 있으며, 이 추세는 앞으로도 계속될 가능성이 큽니다. 하지만 이전 버전의 네이티브 CSS는 이러한 기능을 지원하지 않았습니다.

그래서 CSS 전처리기가 처음에 널리 사용되기 시작했습니다. 이 도구들은 작성한 CSS 규칙과 스타일, 그리고 확장된 구문을 브라우저가 이해할 수 있는 네이티브 CSS 파일로 컴파일합니다.

CSS 전처리기를 사용하면 CSS 코드를 더 재사용 가능하고 논리적으로 구조화할 수 있습니다. 또한 CSS에서 직접 지원하지 않는 믹스인 같은 강력한 기능도 사용할 수 있습니다.

가장 인기 있는 CSS 전처리기 중 일부는 Sass, Less, Stylus입니다.

Sass에 대해 이야기해 보겠습니다.

Sass는 "Syntactically Awesome Style Sheets"의 약자입니다. 모든 버전의 CSS와 호환 가능하며, 많은 개발자 커뮤니티가 유지 관리합니다.

Sass는 다음과 같은 기능을 지원합니다:

- 변수를 사용해 값을 저장하고 재사용할 수 있습니다.

- 중첩된 CSS 규칙으로 파일 내에 시각적 계층 구조를 만들 수 있습니다.

- 모듈로 스타일을 여러 스타일시트로 분할할 수 있습니다.

- 믹스인으로 사이트 전체에서 CSS 선언을 재사용할 수 있습니다.

- 상속으로 여러 CSS 선택자가 속성을 공유할 수 있습니다.

- 그리고 기본 수학 연산을 위한 연산자를 지원합니다.

Sass는 두 가지 구문을 지원합니다.

SCSS 구문은 Sassy CSS의 약자로, CSS의 상위 집합입니다. 즉, SCSS는 CSS의 기본 구문을 확장합니다.

SCSS는 Sass 작업 시 가장 일반적으로 사용하고 접하게 될 구문입니다. 네이티브 CSS처럼 CSS 속성 주위에 중괄호(`{`와 `}`)를 사용하고, CSS 선언 끝에 세미콜론(`;`)을 요구합니다.

이 파일들은 `.scss` 확장자를 가집니다. 예시는 다음과 같습니다:

```scss
$primary-color: #3498eb;

header {
  background-color: $primary-color;
}
```

위 예시 상단에 SCSS에서 정의한 변수가 보입니다. 이 변수는 아래 CSS 규칙에서 사용됩니다.

덜 자주 사용되는 구문인 들여쓰기 구문과 SCSS를 비교해 보겠습니다. 이 구문은 Sass의 원래 구문이라 "Sass 구문"이라고도 합니다.

이 구문은 규칙을 정의할 때 들여쓰기에 의존합니다. 예시는 다음과 같습니다:

```scss
$primary-color: #3498eb

header
  background-color: $primary-color
```

CSS 규칙 주위에 중괄호나 CSS 선언 끝에 세미콜론이 없음을 주목하세요. 상단에 변수도 보입니다. 이 구문에서 변수 이름은 달러 기호(`$`)로 시작하는데, 네이티브 CSS 변수와 다르지만 매우 유사하게 작동합니다.

Sass는 믹스인이라는 강력한 기능도 지원합니다.

믹스인은 여러 CSS 속성과 값을 하나의 이름 아래 묶어 스타일시트 전체에서 그 CSS 코드 블록을 재사용할 수 있게 합니다.

이로 인해 CSS 코드가 덜 반복적이고 유지 관리가 쉬워집니다. 블록 내 내용을 변경하면 믹스인을 사용하는 모든 곳에 변경 사항이 적용되기 때문입니다.

믹스인은 사용자 정의하고 설명적인 이름을 가질 수 있어, 각 CSS 코드 블록이 하는 일을 이해하는 데 도움이 되고 스타일시트를 더 쉽게 이해할 수 있게 합니다.

또한 믹스인은 일관된 스타일을 적용해 스타일시트의 일관성을 높입니다.

다음은 SCSS 구문에서 믹스인 예시입니다:

```scss
@mixin center-flex {
  display: flex;
  justify-content: center;
  align-items: center;
}
```

믹스인을 정의하려면 `@mixin` @규칙을 작성한 뒤 믹스인 이름을 적습니다. 이 예시에서 믹스인 이름은 `center-flex`입니다. Flexbox를 사용해 요소를 가운데 정렬하는 세 가지 CSS 속성이 포함되어 있습니다.

믹스인을 정의한 후에는 `@include` @규칙을 사용해 CSS 규칙에 해당 속성을 포함합니다. `@include` 다음에 믹스인 이름을 적으면 됩니다. 이 경우 이름은 `center-flex`입니다:

```scss
section {
  @include center-flex;
  height: 500px;
  background-color: #3289a8;
}
```

믹스인과 CSS 규칙이 포함된 전체 코드는 다음과 같습니다:

```scss
@mixin center-flex {
  display: flex;
  justify-content: center;
  align-items: center;
}

section {
  @include center-flex;
  height: 500px;
  background-color: #3289a8;
}
```

필요한 만큼 많은 CSS 규칙에 믹스인을 포함할 수 있습니다. 변경이 필요하면 믹스인 자체만 수정하면 되고, 변경 사항은 자동으로 모든 곳에 적용됩니다. 이 예시는 SCSS 구문이며, 중괄호와 세미콜론이 포함되어 있습니다.

중괄호와 세미콜론이 없는 들여쓰기 구문(즉, Sass 구문)에서 동등한 예시는 다음과 같습니다:

```scss
@mixin center-flex
  display: flex
  justify-content: center
  align-items: center

section
  @include center-flex
  height: 500px
  background-color: #3289a8
```

컴파일된 CSS 코드, 즉 파일이 네이티브 CSS로 컴파일된 후 브라우저가 실제로 해석하는 코드는 다음과 같습니다:

```css
section {
  display: flex;
  justify-content: center;
  align-items: center;
  height: 500px;
  background-color: #3289a8;
}
```

상단에 믹스인의 세 가지 속성 `display`, `justify-content`, `align-items`가 포함된 것을 확인하세요.

이것은 CSS 전처리기가 할 수 있는 기능의 단순한 예시일 뿐입니다. 배워서 일상 작업에 적용할 가치가 있는 많은 강력한 기능이 있습니다.

하지만 알아야 할 잠재적인 단점도 있습니다.

첫째, CSS 전처리기는 CSS 규칙을 표준 CSS로 컴파일해야 합니다. 이 컴파일 과정의 잠재적 오버헤드는 시간과 자원 면에서 보통 최소이지만, 실제 프로젝트에서는 고려해야 할 사항입니다.

다음으로, 브라우저가 컴파일된 CSS를 직접 사용하기 때문에 디버깅에 어려움이 있을 수 있습니다. 확장된 구문에서 문제가 되는 스타일을 찾아내려면 표준 CSS보다 몇 단계 더 거쳐야 할 수 있습니다.

하지만 CSS 전처리기의 장점은 보통 단점보다 크며, 특히 복잡한 프로젝트에서 매우 유용합니다. 더 깔끔하고 재사용 가능하며 덜 반복적이고 확장 가능한 CSS를 작성하는 데 큰 도움이 됩니다.
