## 순수 함수와 비순수 함수

- 순수 함수는 같은 입력에 대해 항상 같은 출력을 내고 부작용이 없는 함수입니다. 출력은 오직 입력에만 의존하며 외부 상태를 변경하지 않습니다.
- 비순수 함수는 부작용이 있는데, 이는 함수 외부에서 관찰 가능한 프로그램 상태의 변화를 의미합니다.

## 함수형 프로그래밍

- 함수형 프로그래밍은 문제 해결을 위해 함수를 사용하는 소프트웨어 개발 접근법으로, 어떻게 할 것인가보다 무엇을 할 것인가에 집중합니다.
- 함수형 프로그래밍은 불변 자료 구조와 고차 함수 사용처럼 부작용을 피하는 기법을 장려합니다.
- 올바르게 사용하면 함수형 프로그래밍 원칙은 더 깔끔하고 유지보수하기 쉬운 코드를 만듭니다.

## 커링

- 커링은 여러 인수를 받는 함수를 하나의 인수만 받는 함수들의 수열로 변환하는 함수형 프로그래밍 기법입니다.

다음은 일반 함수와 커링 함수의 예입니다:

:::interactive_editor

```js
// Regular function
function average(a, b, c) {
  return (a + b + c) / 3;
}

console.log(average(2, 3, 4)); // 3

// Curried function
function curriedAverage(a) {
  return function(b) {
    return function(c) {
      return (a + b + c) / 3;
    };
  };
}

console.log(curriedAverage(2)(3)(4)); // 3
```

:::

- 커링은 인수를 많이 받는 함수 작업에 특히 강력할 수 있습니다.
- 커링은 코드를 더 유연하고 재사용하기 쉽게 만듭니다.
- 화살표 함수를 사용하면 커링 함수를 더 간결하게 만들 수 있습니다:

```js
const curriedAverage = a => b => c => (a + b + c) / 3;
```

- 커링은 코드를 더 유연하고 재사용 가능하게 만들지만, 과도하게 사용하면 코드를 읽기 어렵게 만들 수도 있습니다.
