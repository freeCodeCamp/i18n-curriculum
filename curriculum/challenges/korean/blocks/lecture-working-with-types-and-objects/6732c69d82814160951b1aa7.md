# --interactive--

`Number()` 생성자는 숫자 객체를 만드는 데 사용됩니다. 숫자 객체는 `isNaN`과 `toFixed` 메서드 같은 몇 가지 유용한 프로퍼티와 메서드를 포함합니다. 다음은 `Number()` 생성자를 `new` 키워드와 함께 사용하는 예입니다:

:::interactive_editor

```js
const myNum = new Number("34");
console.log(typeof myNum); // "object" 
```

:::

이 예제에서는 문자열 리터럴을 `Number()` 생성자에 전달하며 반환 타입은 `object`이 아닌 `string` 타입입니다.

`Number()` 생성자가 `new` 키워드 없이 함수로 호출되면 반환값은 원시 숫자 타입이 됩니다. 대부분의 경우 다른 데이터 타입을 숫자 데이터 타입으로 변환하기 위해 `Number()` 생성자를 사용하게 됩니다. 문자열을 숫자로 변환하는 예는 다음과 같습니다:

:::interactive_editor

```js
const myNum = Number("100");
console.log(myNum); // 100

console.log(typeof myNum); // number
```

:::

사용자로부터 입력을 받을 때 문자열 입력을 숫자로 변환하여 수학 계산을 수행해야 할 때 유용합니다.

빈 문자열을 `Number()` 생성자에 전달하면 결과는 숫자 `0`가 됩니다:

:::interactive_editor

```js
const num = Number("");
console.log(num); // 0
```

:::

이는 JavaScript가 문자열을 파싱하려고 시도하는데 숫자가 포함되어 있지 않으므로 결과가 0이 되기 때문입니다.

임의의 문자가 포함된 문자열을 전달하면 결과는 `NaN` 또는 "숫자가 아님"이 됩니다:

:::interactive_editor

```js
const num = Number("random");
console.log(num); // NaN
```

:::

불리언을 다룰 때 `true`는 `1`를 반환하는데, 이는 `true`이 1로 변환되기 때문이고 `false`는 `0`를 반환하는데, `false`가 0으로 변환되기 때문입니다.

:::interactive_editor

```js
const boolTrue = Number(true);
const boolFalse = Number(false);

console.log(boolTrue); // 1
console.log(boolFalse); // 0
```

:::

`null`를 전달하면 결과는 `0`이고 `undefined`를 전달하면 결과는 `NaN`입니다.

:::interactive_editor

```js
const undefinedNum = Number(undefined);
const nullNum = Number(null);

console.log(undefinedNum); // NaN
console.log(nullNum); // 0
```

:::

배열을 다룰 때는 몇 가지를 고려해야 합니다.

빈 배열은 `0`를 반환합니다. 숫자 하나만 있는 배열은 그 숫자를 반환합니다. 여러 숫자가 있는 배열은 `NaN`를 반환합니다. 문자열이 포함된 배열도 `NaN`를 반환합니다.

:::interactive_editor

```js
const emptyArr = Number([]);
const arrOneNum = Number([7]);
const arrMultiNum = Number([7, 36, 12]);
const arrStr = Number(["str1"]);
const arrMultiStr = Number(["str1", "str2"]);

console.log(emptyArr); // 0
console.log(arrOneNum); // 7
console.log(arrMultiNum); // NaN
console.log(arrStr); // NaN
console.log(arrMultiStr); // NaN
```

:::

객체를 다룰 때 결과는 항상 `NaN`입니다.

:::interactive_editor

```js
const obj1 = Number({});
const obj2 = Number({2: 2});
const obj3 = Number({key: "val"});
const obj4 = Number({key: true});

console.log(obj1); // NaN
console.log(obj2); // NaN
console.log(obj3); // NaN
console.log(obj4); // NaN
```

:::

결론적으로, 숫자나 숫자 객체를 생성하는 것보다 타입 변환을 위해 `Number()` 생성자를 주로 사용하게 됩니다.
