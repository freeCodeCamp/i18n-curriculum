# --explanation--

Mark는 `If the application has memory leaks, it can lead to crashes`라고 말합니다.

여기서 핵심 용어는 `memory leaks`입니다. 이는 프로그램이 임시 작업을 위해 메모리를 할당하지만 작업이 끝난 후 이를 해제하지 못할 때 발생합니다. 예를 들어:

- `If the application uses too much memory, it could crash.` - 이는 메모리 사용이 충돌을 일으키는 문제임을 시사합니다.

- `Memory leaks can cause performance issues and crashes.` - 이는 메모리 누수가 충돌 원인 중 하나임을 보여줍니다.
