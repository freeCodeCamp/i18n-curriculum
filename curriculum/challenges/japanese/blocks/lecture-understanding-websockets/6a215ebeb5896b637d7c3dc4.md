# --description--

Node.jsには組み込みのWebSocketサーバーはありません。開発者は通常、`ws`のような外部パッケージを使ってWebSocketプロトコルを実装し、クライアントとサーバー間のリアルタイム通信を処理します。

`ws`ライブラリは軽量で高速、かつ使いやすいです。WebSocketプロトコルを実装しているので、WebSocket接続を作成・管理できます。

npmでインストールできます：

```bash
npm install ws
```

インストール後、`ws`ライブラリをインポートし、その`Server`メソッドを呼び出してポートを渡すことでWebSocketサーバーを作成します：

```js
const WebSocket = require("ws");

const server = new WebSocket.Server({
  port: 3000,
});
```

これにより、ポート`3000`で接続を待ち受けるWebSocketサーバーが起動します。

クライアントが接続すると、サーバーは`connection`イベントを発します。

```js
server.on("connection", (socket) => {
  console.log("Client connected");
});
```

このイベントはサーバーが「新しいクライアントが参加した」と伝える方法だと考えてください。

`socket`パラメータはサーバーとそのクライアント間の接続を表します。これを使ってメッセージの送受信を行います。

接続されたクライアントがデータを送信すると、サーバーは`message`イベントを発します。

```js
server.on("connection", (socket) => {
  socket.on("message", (message) => {
    console.log(message.toString());
  });
});
```

この例では、サーバーが受信メッセージを待ち受けてコンソールにログを出力しています。

`send()`メソッドを使ってクライアントにデータを送信できます：

```js
socket.send("Hello from the server");
```

これは開いているWebSocket接続を通じてメッセージを送信します。

接続が開いたままであれば、クライアントとサーバーはリアルタイムでメッセージを交換し続けられます。

`connection`や`message`のほかに、WebSocket接続は`close`や`error`イベントも発します。

`close`は接続が閉じられたときに発生します：

```js
socket.on("close", () => {
  console.log("Disconnected");
});
```

`error`は何か問題が起きたときに発生します：

```js
socket.on("error", (error) => {
  console.error(error);
});
```

これらのイベントが多くのWebSocketアプリケーションの基盤を形成しています。

チャットアプリケーションでよくある要件の一つがブロードキャストです。ブロードキャストとは、1つのクライアントではなく複数の接続されたクライアントにメッセージを送ることを意味します。

3人のユーザー、**John Doe**、**Jane Doe**、**Wes Smith**がいるチャットルームを想像してください。

John Doeがメッセージを送ると、サーバーはそのメッセージをJane DoeとWes Smithに転送する必要があります。

そのために、サーバーは接続されたクライアントを管理し、メッセージが届くとそれらをループ処理します。

メッセージを送る前に、サーバーはクライアント接続がまだ開いているかどうかを確認します：

```js
if (client.readyState === WebSocket.OPEN) {
  // do something
}
```

すべてのWebSocket接続には現在の状態を表す`readyState`プロパティがあります。

`WebSocket.OPEN`は接続がアクティブで、データの送受信が可能な状態を意味します。

これを確認することで、サーバーはすでに切断されたか、まだ接続が確立されていないクライアントにメッセージを送るのを避けられます：

```js
server.on("connection", (socket) => {
  socket.on("message", (message) => {
    server.clients.forEach((client) => {
      if (client.readyState === WebSocket.OPEN) {
        client.send(message.toString());
      }
    });
  });
});
```

この例では、接続されたすべてのクライアントが受信メッセージを受け取ります。これがブロードキャストと呼ばれます。

多くのリアルタイムアプリケーションは、チャットメッセージ、ライブ通知、スポーツの更新情報、マルチプレイヤーゲームのイベントなどのデータ配信にブロードキャストを使っています。

`ws`ライブラリを使えば、Node.jsでWebSocketサーバーを簡単に構築できます。

サーバーを作成し、イベントを待ち受け、メッセージを交換し、接続されたクライアントに更新をブロードキャストするだけです。

これらの基本要素が多くのリアルタイムアプリケーションの基盤となっています。
