# --description--

Die Suche in einer Liste von Elementen ist ein häufiges Ereignis in der Informatik. Es gibt zwei wichtige Algorithmen, die Sie beim Thema Suche kennen sollten: lineare Suche und binäre Suche.

Die lineare Suche beginnt am Anfang einer Liste und iteriert durch jedes Element, bis der gesuchte Zielwert gefunden wird.

Wenn der Zielwert gefunden wird, wird der Index zurückgegeben, an dem er sich in der Liste befindet. Wenn der Zielwert nicht gefunden wird, wird `-1` zurückgegeben. Wir geben `-1` zurück, weil es in den meisten Programmiersprachen kein gültiger Index ist.

So sieht der Code für die lineare Suche aus:

```python
def linear_search(arr, target):
    for i in range(len(arr)):
        if arr[i] == target:
            return i
    return -1
```

Wenn die Liste, die wir durchsuchen, `[13, 4, 7, 9, 10]` ist und der Zielwert `9`, würde die Funktion `3` zurückgeben, weil `9` sich an Index `3` befindet.

Wenn wir den Zielwert auf `5` ändern, würde die Funktion `-1` zurückgeben, weil `5` nicht in der Liste ist.

Obwohl dies ein relativ einfacher Algorithmus ist, ist er nicht der effizienteste. Wenn Sie eine große Liste von Elementen haben, kann die lineare Suche lange dauern, um den Zielwert zu finden.

Die Zeitkomplexität der linearen Suche ist `O(n)`, weil die Zeit, die benötigt wird, um die Liste zu durchsuchen, linear mit der Größe der Liste wächst.

Die Speicherkomplexität der linearen Suche ist `O(1)`, weil kein zusätzlicher Speicher benötigt wird, um die Liste zu durchsuchen.

Die binäre Suche ist ein effizienterer Algorithmus, um eine große Liste von Elementen zu durchsuchen. Die Voraussetzung ist, dass die Liste in aufsteigender Reihenfolge sortiert sein muss.

Die binäre Suche funktioniert, indem die Liste halbiert wird und geprüft wird, ob sich der Zielwert in der Mitte der Liste befindet. Wenn der Zielwert in der Mitte der Liste ist, wird der Index des Zielwerts zurückgegeben. Andernfalls prüft der Algorithmus, ob sich der Zielwert in der linken oder rechten Hälfte der Liste befindet.

Er teilt die verbleibenden Teile der Liste weiterhin in Hälften, bis der Zielwert gefunden wird. Wenn der Zielwert nicht in der Liste ist, wird `-1` zurückgegeben.

So sieht der Code für die binäre Suche aus:

```python
def binary_search(arr, target):
    low = 0
    high = len(arr) - 1

    while low <= high:
        mid = (low + high) // 2  

        if arr[mid] == target:
            return mid
        elif arr[mid] < target:
            low = mid + 1
        else:
            high = mid - 1

    return -1
```

Wir beginnen damit, einen `low` und einen `high` Index zu bestimmen. Dies stellt den Bereich der Liste dar, den wir durchsuchen.

Dann prüfen wir die Bedingung, dass `low` kleiner oder gleich `high` ist. Wenn `low` größer als `high` ist, haben wir die gesamte Liste durchsucht und der Zielwert wurde nicht gefunden. In diesem Fall beenden wir die Suche und geben `-1` zurück.

Wenn der `low` Index kleiner oder gleich dem `high` Index ist, berechnen wir den mittleren Index der Liste, `mid`. Dann prüfen wir, ob sich der Zielwert am mittleren Index befindet. Wenn ja, geben wir den mittleren Index zurück.

Andernfalls prüfen wir, ob der Wert am Mittelpunkt kleiner als der Zielwert ist. Wenn ja, aktualisieren wir den unteren Index auf den mittleren Index plus eins. Das bedeutet, wir durchsuchen die rechte Hälfte der Liste.

Schließlich, wenn keine der anderen Bedingungen `True` ist, aktualisieren wir den `high` Index auf den mittleren Index minus eins. Das bedeutet, wir durchsuchen die linke Hälfte der Liste.

Wir wiederholen diesen Vorgang so lange, bis wir den Zielwert finden oder feststellen, dass der Zielwert nicht in der Liste ist.

Die Zeitkomplexität der binären Suche ist `O(log n)`, weil die Zeit, die benötigt wird, um die Liste zu durchsuchen, logarithmisch mit der Größe der Liste wächst.

Die Speicherkomplexität der binären Suche ist `O(1)`, weil kein zusätzlicher Speicher benötigt wird, um die Liste zu durchsuchen.

Binäre Suche und lineare Suche können für eine Vielzahl von Problemen verwendet werden, denen Sie in der Informatik begegnen werden. Es ist wichtig, die Unterschiede zwischen den beiden Algorithmen zu verstehen und zu wissen, wann welcher Algorithmus eingesetzt werden sollte.
