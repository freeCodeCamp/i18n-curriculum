## Вступ до динамічного програмування

- **Визначення**: Динамічне програмування — це алгоритмічна техніка, яка розв’язує складні задачі, розбиваючи їх на простіші підзадачі та зберігаючи результати, щоб уникнути зайвих обчислень.
- **Перекривні підзадачі**: Одні й ті самі менші задачі з’являються кілька разів під час розв’язання більшої задачі. Замість того, щоб повторно обчислювати ці підзадачі, ми зберігаємо їхні розв’язки.
- **Оптимальна підструктура**: Оптимальне розв’язання задачі містить оптимальні розв’язки її підзадач. Це означає, що ми можемо побудувати найкраще розв’язання, поєднуючи найкращі розв’язки менших частин.

## Розв’язки за допомогою динамічного програмування

- **Мемоізація (підхід зверху вниз)**: Мемоізація зберігає результати дорогих викликів функцій і повертає кешований результат, коли ті самі вхідні дані з’являються знову.

```js
function climbStairsMemo(n, memo = {}) {
  // If already calculated, return cached value
  if (memo[n] !== undefined) {
    return memo[n];
  }

  // Base cases: 1 way for 1 step, 2 ways for 2 steps
  if (n <= 2) {
    return n;
  }

  // Calculate once and store in memo for future use
  memo[n] =
    climbStairsMemo(n - 1, memo) +
    climbStairsMemo(n - 2, memo);

  return memo[n];
}
```

- **Табуляція (підхід знизу вгору)**: Табуляція будує розв’язок знизу вгору, заповнюючи таблицю розв’язками підзадач.

```js
function climbStairsTabulation(n) {
  if (n <= 2) {
    return n;
  }

  // Create an array to store solutions from 0 to n
  const dp = new Array(n + 1).fill(0);

  // Base cases
  dp[1] = 1; // 1 way to reach step 1
  dp[2] = 2; // 2 ways to reach step 2

  // Build up the solution iteratively
  for (let i = 3; i <= n; i++) {
    // Ways to reach step i = ways to reach (i-1) + ways to reach (i-2)
    dp[i] = dp[i - 1] + dp[i - 2];
  }

  return dp[n];
}
```

## Реальні застосування динамічного програмування

- **Оптимізація маршрутів**: GPS-системи використовують алгоритми динамічного програмування для пошуку найкоротших шляхів між локаціями.
- **Обробка тексту**: Перевірка орфографії та функції автозаповнення часто покладаються на динамічне програмування для обчислення відстаней редагування між словами.
- **Фінансове моделювання**: Інвестиційні стратегії та оптимізація портфоліо часто застосовують техніки динамічного програмування.
- **Розподіл ресурсів**: Задача рюкзака та її варіанти зустрічаються у плануванні, бюджетуванні та управлінні ресурсами.

## Коли варто використовувати динамічне програмування

Варто розглянути використання динамічного програмування у таких випадках:

- Задачу можна розбити на перекривні підзадачі.
- Задача має оптимальну підструктуру.
- Наївне рекурсивне розв’язання передбачає повторні обчислення.
- Потрібно оптимізувати часову складність за рахунок складності пам’яті.
