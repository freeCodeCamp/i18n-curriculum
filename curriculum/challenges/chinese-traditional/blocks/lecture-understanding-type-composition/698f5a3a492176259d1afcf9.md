# --description--

在先前的課程中，你學會了如何使用型別別名，並看到如何為物件型別命名的範例。 

如你所知，命名一個物件型別的一種方法是使用 `type` 關鍵字，如下所示：

```ts
type ID = number | string;

type User = {
  id: ID;
  name: string;
};
```

另一種命名物件型別的方法是使用像這樣的 `interface` 宣告：

```ts
type ID = number | string;

interface User {
  id: ID;
  name: string;
}
```

`type` 和 `interface` 類似，但一個主要差異是介面使用 `extends` 關鍵字來建造其他介面，而 `type` 別名則使用交集型別（`&`）來組合型別。

讓我們來看一些範例：

```ts
interface User {
  id: number | string;
  name: string;
}

interface Admin extends User {
  role: string;
}

const adminUser: Admin = {
  id: 101,
  name: "Alice",
  role: "superadmin"
};
```

在此範例中，`User` 從 `id` 和 `name` 開始。`Admin` 介面繼承 `User`，表示它包含 `id` 和 `name`，並新增一個名為 `role` 的屬性。

如果你想擴充一個 `type`，你會像這樣使用交集型別（`&`）：

```ts
type User = {
  id: number | string;
  name: string;
};

type Admin = User & {
  role: string;
};

const adminUser: Admin = {
  id: 101,
  name: "Alice",
  role: "superadmin"
};
```

`Admin` 將 `User` 型別與包含 `role` 的新物件型別結合。結果與介面範例相同形狀，該介面是一個具有 `id`、`name` 和 `role` 的物件。

`type` 和 `interface` 都可以描述物件的形狀。如果你不確定要使用哪一個，請記住 `interface` 是專門為物件形狀設計的，而 `type` 則更具彈性，也可以表述聯集型別、原始型別以及更複雜的型別組合。
