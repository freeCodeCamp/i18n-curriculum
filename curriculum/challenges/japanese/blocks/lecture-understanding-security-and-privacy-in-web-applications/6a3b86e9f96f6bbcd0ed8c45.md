# --description--

HTTPSはHypertext Transfer Protocol Secureの略です。これはHTTPの安全なバージョンです。

いくつかの前の講義で学んだように、HTTPはブラウザーとウェブサイト間でデータを転送します。標準のHTTPはこのデータをプレーンテキストで送信するため、誰でもそのプレーンテキストのデータを傍受できます。

このため、HTTPSはTransport Layer Security（TLS）という暗号化プロトコルを使ってこのデータ転送を保護します。TLSは2点間の通信チャネルを暗号化します。古いシステムではSecure Sockets Layer（SSL）が使われており、多くのウェブホスティングプラットフォームでも今なお使われています。

暗号化はそのデータを読み取れないコードに変換し、ユーザーのブラウザーとウェブサーバーだけが復号の鍵を持っています。

例えば、ウェブサイトにパスワードを入力すると、HTTPSは即座にそれを暗号化し、クレジットカード情報、住所、ログイン資格情報、その他の機密情報を保護します。データを傍受したハッカーはランダムな文字しか見えないため、盗んだ情報を使うことができません。

この保護は公共のWi-Fiネットワークで特に重要です。カフェや空港などのオープンWi-Fiネットワークは基本的なセキュリティがなく、サイバー犯罪者がこれらのネットワーク上の暗号化されていないトラフィックを監視できる状態です。HTTPSは公共ネットワーク上にプライベートなトンネルを作り、近くの悪意ある者から閲覧履歴を守ります。

HTTPSはまた認証も提供し、ウェブサイトが正当なものであることを証明します。HTTPSを使うには、ウェブサイトの所有者が信頼された組織から発行されるデジタル証明書を取得する必要があります。この証明書はウェブサイト所有者の真の身元を検証します。

ウェブサイトを訪れてアドレスが「http」だけで始まる場合、それはウェブサイト所有者がその証明書を取得していないことを示し、そのウェブサイトが安全でないことを意味します。最新のブラウザーはそのようなウェブサイトについて警告も表示します。

最後に、データの完全性もHTTPSの重要な利点です。これは、データが転送中に誰にも改ざんされないことを意味します。ハッカーがウェブサイトのファイルに悪意のあるコードを注入することはできません。受け取るデータはウェブサイト所有者が送ったままの正確なものです。

Googleのような検索エンジンはHTTPSのウェブサイトを優先し、安全なウェブサイトは検索結果で上位にランクされます。最新のブラウザーはジオロケーション、プッシュ通知、ウェブカメラなどの高度な機能にHTTPSを必須としています。
