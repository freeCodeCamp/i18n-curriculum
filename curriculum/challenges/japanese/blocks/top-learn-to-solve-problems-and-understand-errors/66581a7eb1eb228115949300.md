# --description--

例えば、2つの文字列を組み合わせて1つのメッセージを作りたいとします。以下のように：

```js
const str1 = "Hello";
const str2 = "World!";
const message = str1.push(str2);
```

<img src="https://cdn.statically.io/gh/TheOdinProject/curriculum/4ed59981b4ce2c60b5b83bf7415d3127b61821f5/foundations/javascript_basics/understanding_errors/imgs/03.png" style="width:100%" alt="Chrome開発者コンソールでのタイプエラーの例">

ここでは、`TypeError`が表示され、`str1.push is not a function`というメッセージが出ます。これは学習者を混乱させるよくあるエラーメッセージです。なぜなら、`.push()`が確かに関数であることを知っているかもしれないからです（例えば、以前に配列にアイテムを追加するために使ったことがある場合など）。

しかし、ここがポイントです。`.push()`は文字列のメソッドではなく、配列のメソッドです。したがって、文字列のメソッドとしては「関数ではない」ということになります。`.push()`を正しい文字列のメソッドである`.concat()`に変更すれば、コードは意図した通りに動作します！

`TypeError`に直面したときに覚えておくと良いのは、メソッドや操作を実行しようとしているデータ型を考慮することです。多くの場合、それは思っているものとは異なるか、その操作やメソッドがその型に互換性がないことがわかるでしょう。

const str1 = "Hello";

## --answers--

`.push()`メソッドはオブジェクトにのみ使え、文字列には使えません。

---

`.push()`メソッドは配列のメソッドなので、文字列には使えません。

---

`.push()`メソッドはスペルミスで、正しくは.pusch()です。

---

`TypeError`はJavaScriptコードの構文エラーによるものです。
